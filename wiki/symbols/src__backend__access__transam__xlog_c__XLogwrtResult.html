XLogwrtResult

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/xlog.c:326

目的

WAL（Write-Ahead Log）の書き込みおよびフラッシュ処理の結果状態を保持するために使用されます。

シグネチャまたは形状

typedef struct XLogwrtResult
{
    XLogRecPtr  Write;          /* last byte + 1 written out */
    XLogRecPtr  Flush;          /* last byte + 1 flushed */
} XLogwrtResult;

入力と出力

  • **入力**: 構造体の各フィールドは XLogRecPtr 型であり、WAL ストリーム上の位置（バイトオフセット）を表します。
  • **出力**: WAL の書き込みおよびフラッシュ処理の完了地点を示すデータとして利用されます。

前提条件と不変条件

  • Write は、ディスクへ書き出された最後のバイトの次の位置（last byte + 1）を指します。
  • Flush は、ディスクへフラッシュ（fsync 等）が完了した最後のバイトの次の位置（last byte + 1）を指します。
  • 不変条件として、常に Flush <= Write が成立する必要があります。

アルゴリズムまたは振る舞い

本構造体は、WAL の永続化状態を追跡するために使用されます。XLogWrite 等の関数において、現在の書き込みおよびフラッシュの進捗を更新し、呼び出し元へ現在の WAL 状態を通知する役割を担います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlog.c 内の WAL 管理プロセス。
  • **呼び出し先**: 該当なし（データ構造体のため）。
  • 補足: src/backend/access/transam/xlog.c:613 にて LogwrtResult という静的変数としてインスタンス化されています。

関連データ構造

  • XLogRecPtr: WAL 上の位置を特定するためのポインタ型。
  • XLogCtlData: WAL 管理の全体状態を保持する構造体。

失敗モードと境界条件

  • Write および Flush が初期値（0）のまま更新されない場合、WAL の永続化が進行していないことを示します。
  • 境界条件として、WAL セグメントの切り替わりや、LSN のオーバーフロー発生時の挙動は XLogRecPtr の仕様に依存します。

保守担当者向けメモ

  • 本構造体は xlog.c 内の WAL 書き込みフローにおいて頻繁に参照されます。
  • LogwrtResult 変数の更新タイミングを追跡することで、WAL のディスク書き込み遅延やフラッシュのボトルネックを特定可能です。
