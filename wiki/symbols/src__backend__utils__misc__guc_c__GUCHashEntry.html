GUCHashEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/misc/guc.c:206

目的

GUC（Grand Unified Configuration）システムにおいて、パラメータ名からその定義構造体へ高速にアクセスするためのハッシュテーブルエントリとして機能します。

シグネチャまたは形状

typedef struct
{
	const char *gucname;		/* hash key */
	struct config_generic *gucvar;	/* -> GUC's defining structure */
} GUCHashEntry;

入力と出力

  • 入力: ハッシュテーブルのキーとして gucname（文字列）を使用。
  • 出力: gucvar を介して、対応するパラメータの定義情報（config_generic）へのポインタを提供。

前提条件と不変条件

  • gucname はハッシュテーブルの検索キーとして一意である必要があります。
  • gucvar は有効な config_generic 構造体を指している必要があります。

アルゴリズムまたは振る舞い

  • guc_hashtab のエントリサイズとして sizeof(GUCHashEntry) が指定され、ハッシュテーブルの構築および検索時に使用されます。
  • hash_search を用いて、指定された名前のパラメータに対応するエントリの取得や挿入が行われます。
  • hash_seq_search を用いて、登録されている全パラメータを走査する際に利用されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照箇所）:
  • src/backend/utils/misc/guc.c 内の hash_search 呼び出し箇所（パラメータの検索・登録時）。
  • hash_seq_search を使用した全パラメータの反復処理箇所。
  • 呼び出し先:
  • config_generic 構造体（gucvar メンバを通じて参照）。

関連データ構造

  • config_generic: GUC パラメータの基底構造体。
  • guc_hashtab: 本構造体を格納するハッシュテーブル。

失敗モードと境界条件

  • ハッシュテーブルの初期化前や、メモリ不足により hash_search が失敗した場合の挙動は、呼び出し側の実装に依存します。

保守担当者向けメモ

  • 本構造体は guc.c 内のハッシュテーブル管理に特化しています。パラメータ定義の変更や追加を行う際は、guc_tables.c との整合性に注意してください。
