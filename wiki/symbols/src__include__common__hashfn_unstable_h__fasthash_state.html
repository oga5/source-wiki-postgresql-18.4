fasthash_state

定義

  • **種別**: struct
  • **定義箇所**: src/include/common/hashfn_unstable.h:93

目的

インクリメンタルなハッシュ計算において、計算の進行状況と中間状態を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct fasthash_state
{
    uint64      accum;  /* 入力チャンクのステージング領域 */
    uint64      hash;   /* 現在のハッシュ値 */
} fasthash_state;

入力と出力

  • **入力**: fasthash_init を介してシード値が設定され、fasthash_accum 等の関数を通じてデータが蓄積されます。
  • **出力**: 最終的に fasthash_final32 または fasthash_final64 を呼び出すことで、計算されたハッシュ値が返されます。

前提条件と不変条件

  • 使用前に fasthash_init による初期化が必須です。
  • 本構造体はメモリ内での高速ハッシュ計算専用であり、永続化（ディスク保存）には適していません。

アルゴリズムまたは振る舞い

1. **初期化**: fasthash_init で accum と hash が初期化されます。 2. **蓄積**: fasthash_accum 等により、入力データが accum にロードされ、ハッシュ値へ混合されます。 3. **終了**: fasthash_final 系関数が入力長などの値を「tweak」として適用し、最終的なハッシュ値を算出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/namespace.c および src/include/common/hashfn_unstable.h 内の各関数から参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、fasthash_init、fasthash_accum、fasthash_final などの関数によって操作されます。

関連データ構造

  • uint64: ハッシュ計算に使用される 64 ビット整数型。

失敗モードと境界条件

  • **プラットフォーム依存**: 本構造体を用いたハッシュ計算はエンディアンの影響を受けるため、プラットフォーム間で結果が一致する保証はありません。
  • **アライメント**: fasthash_accum_cstring_aligned を使用する場合、ポインタのアライメントが適切である必要があります。

保守担当者向けメモ

  • 本構造体は「不安定」なハッシュ実装の一部です。永続化が必要なデータには使用しないでください。
  • fasthash_accum_cstring は、SMHasher テストをパスするために文字列長をハッシュ計算に組み込む設計となっています。
