gbt_bit_l2n

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bit.c:93

目的

GiST インデックス内のリーフノードキーを正規化し、gbt_bit_xfrm を適用してインデックス格納に適した形式に変換する。

シグネチャまたは形状

GBT_VARKEY *gbt_bit_l2n(GBT_VARKEY *leaf, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • leaf: 変換対象の GBT_VARKEY 構造体。
  • flinfo: 関数呼び出し情報（現時点では使用されていない）。
  • **出力**:
  • 変換後の新しい GBT_VARKEY 構造体。

前提条件と不変条件

  • 入力 leaf は gbt_var_key_readable で読み取り可能な状態であること。
  • 内部で生成される一時的な bytea オブジェクトは、処理終了後に pfree で解放されること。

アルゴリズムまたは振る舞い

1. gbt_var_key_readable(leaf) を呼び出し、入力キーの読み取り可能な構造体 r を取得する。 2. gbt_bit_xfrm(r.lower) を呼び出し、ビット列をインデックス格納用に変換した bytea オブジェクト o を生成する。 3. 構造体 r の lower および upper メンバを o で更新する。 4. gbt_var_key_copy(&r) を呼び出し、更新されたキーを新しいメモリ領域にコピーして返す。 5. 一時オブジェクト o を pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は contrib/btree_gist/btree_bit.c:119 で確認されている。
  • **呼び出し先**:
  • gbt_var_key_readable
  • gbt_bit_xfrm
  • gbt_var_key_copy
  • pfree

関連データ構造

  • GBT_VARKEY: GiST インデックスで可変長データを扱うためのキー構造体。
  • GBT_VARKEY_R: GBT_VARKEY を読み取り専用として扱うためのラッパー構造体。

失敗モードと境界条件

  • メモリ不足時に palloc（gbt_var_key_copy 内部）が失敗する可能性がある。
  • gbt_bit_xfrm が NULL を返す場合、その後のポインタ操作で問題が発生する可能性がある（ソースコード上では明示的なチェックは省略されている）。

保守担当者向けメモ

  • 本関数は gbt_bit_xfrm に強く依存しているため、変換ロジックを変更する場合は両方の整合性を確認すること。
  • flinfo 引数は現在使用されていないが、将来的な拡張のためにシグネチャに含まれている。
