brin_xlog_revmap_extend

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_xlog.c:207

目的

BRINインデックスのREVMAP（範囲マップ）ページが拡張された際のWALリプレイ処理を実行し、メタページの更新と新しいREVMAPページの初期化を行う。

シグネチャまたは形状

static void brin_xlog_revmap_extend(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（WALレコード情報）
  • **出力**: なし（副作用としてバッファプールおよびディスク上のページを更新）

前提条件と不変条件

  • record には xl_brin_revmap_extend 構造体が含まれていること。
  • メタページ（ブロック0）の lastRevmapPage は、対象ブロック番号の1つ前であること（Assertによる検証）。
  • ターゲットブロックは新規に初期化されるため、フルページイメージは存在しない。

アルゴリズムまたは振る舞い

1. WALレコードから xl_brin_revmap_extend データを取得し、ターゲットブロック番号を確認する。 2. メタページを読み込み、BLK_NEEDS_REDO の場合に以下を実行する：

  • lastRevmapPage をターゲットブロック番号に更新する。
  • pd_lower をメタデータ終了位置に設定する（旧バージョンとの互換性維持のため）。
  • LSNを更新し、バッファをダーティにする。

    • 3. ターゲットブロックを XLogInitBufferForRedo で初期化し、brin_page_init を用いて BRIN_PAGETYPE_REVMAP 型として設定する。 4. ターゲットブロックのLSNを更新し、ダーティにする。 5. 最後に各バッファのロックを解除し、解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: brin_redo（src/backend/access/brin/brin_xlog.c 内のディスパッチャ）から呼び出される。
  • **呼び出し先**: XLogRecGetData, XLogRecGetBlockTag, XLogReadBufferForRedo, PageGetContents, PageSetLSN, MarkBufferDirty, XLogInitBufferForRedo, brin_page_init, UnlockReleaseBuffer。

関連データ構造

  • xl_brin_revmap_extend: WALレコードのデータ構造。
  • BrinMetaPageData: インデックスのメタページ構造体。

失敗モードと境界条件

  • Assert 失敗: メタページの lastRevmapPage が期待値と異なる場合、リカバリプロセスが停止する。
  • バッファ取得失敗: XLogReadBufferForRedo 等がエラーを返した場合、リプレイ処理は中断される。

保守担当者向けメモ

  • メタページの pd_lower 設定は、PostgreSQL v11以前の古いインデックスとの互換性を保つための重要な防衛的実装である。この処理を削除してはならない。
  • REVMAPの拡張はメタページとターゲットページの2つのバッファを操作するため、デッドロックや順序に注意が必要だが、本関数は固定順序（メタページ→ターゲットページ）で処理している。
