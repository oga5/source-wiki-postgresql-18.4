AppendSeconds

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/datetime.c:457

目的

秒数（整数部）および小数秒（fsec_t）を文字列バッファに書き込み、指定された精度でフォーマットする。

シグネチャまたは形状

char *AppendSeconds(char *cp, int sec, fsec_t fsec, int precision, bool fillzeros)

入力と出力

  • **入力**:
  • cp: 書き込み先の文字列バッファポインタ。
  • sec: 秒の整数部分。
  • fsec: 小数秒部分（int32）。
  • precision: 小数点以下の桁数。
  • fillzeros: 秒の整数部を2桁でゼロ埋めするかどうか。
  • **出力**:
  • 書き込み後のバッファの末尾を指すポインタを返す。

前提条件と不変条件

  • precision >= 0 であること（Assertにより保証）。
  • fsec_t は int32 として扱われる。

アルゴリズムまたは振る舞い

1. **整数部の書き込み**: fillzeros が真であれば pg_ultostr_zeropad を、偽であれば pg_ultostr を使用して秒の整数部をバッファに書き込む。 2. **小数部の処理**: fsec が 0 でない場合、小数点 . を追加する。 3. **小数部のフォーマット**: 指定された precision に基づき、末尾のゼロを除去するように逆順で数値を生成する。 4. **精度不足時の対応**: 指定された精度で小数部を表現しきれない場合（value が 0 にならない場合）、pg_ultostr を用いて最小限の有効幅で書き出す。 5. **戻り値**: 書き込みが完了したバッファの末尾ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/datetime.c 内の複数の箇所（timestamp や interval の出力処理）。
  • src/interfaces/ecpg/pgtypeslib/interval.c（ECPGライブラリ）。
  • **呼び出し先**:
  • pg_ultostr_zeropad
  • pg_ultostr
  • abs

関連データ構造

  • fsec_t: 小数秒を保持する型（int32）。

失敗モードと境界条件

  • precision が不足している場合、指定された桁数を超えて数値全体が出力される。
  • fsec が 0 の場合、小数部は出力されない。

保守担当者向けメモ

  • 本関数は datetime.c 内で頻繁に利用される低レベルな書式化ルーチンである。
  • 小数部の末尾ゼロ除去ロジックは、逆順での数値生成に基づいているため、変更時は桁数計算の整合性に注意すること。
