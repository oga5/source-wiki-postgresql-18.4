heap_vacuum_eager_scan_setup

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/vacuumlazy.c:487

目的

VACUUM実行時、将来のAggressive VACUUMの負荷を軽減するために、可視性マップでスキップ可能なページを「意欲的（Eager）」にスキャンして凍結（Freeze）を行うための管理状態を初期化する。

シグネチャまたは形状

static void heap_vacuum_eager_scan_setup(LVRelState *vacrel, VacuumParams *params)

入力と出力

  • **入力**:
  • vacrel: VACUUM実行中の状態を保持する LVRelState 構造体。
  • params: VACUUMの実行パラメータ。
  • **出力**: なし（vacrel 内の eager_scan_* フィールドを更新する）。

前提条件と不変条件

  • params->max_eager_freeze_failure_rate が 0 の場合、Eager Scan は無効化される。
  • vacrel->aggressive が真の場合、Aggressive VACUUM は全ページをスキャンするため Eager Scan は無効化される。
  • 対象リレーションのサイズが 2 * EAGER_SCAN_REGION_SIZE 未満の場合、効率が悪いため無効化される。
  • FreezeLimit または MultiXactCutoff が、現在の relfrozenxid または relminmxid を超えて進んでいる場合にのみ有効化される。

アルゴリズムまたは振る舞い

1. **初期化**: vacrel 内の Eager Scan 関連フィールドを無効値（InvalidBlockNumber や 0）にリセットする。 2. **無効化判定**: パラメータ、Aggressive フラグ、テーブルサイズ、凍結対象の有無に基づき、Eager Scan が不要な場合は早期リターンする。 3. **成功上限の設定**: visibilitymap_count を使用して、全可視だが未凍結のブロック数を取得し、MAX_EAGER_FREEZE_SUCCESS_RATE に基づいて許容成功数を算出する。 4. **領域境界の計算**: 乱数を用いて最初のスキャン領域の開始ブロックを決定し、その領域サイズに応じた許容失敗数を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/vacuumlazy.c 内の heap_vacuum_rel から呼び出される。
  • **呼び出し先**: visibilitymap_count（可視性マップの集計）、pg_prng_uint32（乱数生成）。

関連データ構造

  • LVRelState: VACUUMの実行状態を保持する構造体。Eager Scan の進捗管理フィールドを含む。
  • VacuumParams: VACUUMの動作設定。
  • EAGER_SCAN_REGION_SIZE: スキャン領域の単位サイズ。

失敗モードと境界条件

  • 全ての可視ページが既に凍結されている場合、eager_scan_remaining_successes が 0 となり、Eager Scan は無効化される。
  • 最初の領域が小さい場合、そのサイズ比率に応じて許容失敗数が調整される。

保守担当者向けメモ

  • 本関数は、VACUUMの各実行において「同じブロックで凍結に失敗し続ける」ことを防ぐため、乱数を用いてスキャン開始位置をずらす設計となっている。
  • MAX_EAGER_FREEZE_SUCCESS_RATE などの定数は、VACUUMのパフォーマンスと凍結の進捗のトレードオフを制御する。
