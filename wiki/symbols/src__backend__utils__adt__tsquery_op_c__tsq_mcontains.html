tsq_mcontains

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/tsquery_op.c:306

目的

2つの tsquery 型の間で、単語集合としての包含関係（@> または <@ 演算子）を判定します。構造的な論理演算ではなく、クエリに含まれる単語の集合がもう一方の集合を包含しているかを評価します。

シグネチャまたは形状

Datum tsq_mcontains(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの TSQuery 型引数を受け取ります。
  • **出力**: Datum 型として bool（包含関係が成立すれば true、そうでなければ false）を返します。

前提条件と不変条件

  • 入力された TSQuery は collectTSQueryValues によって単語リストに抽出されます。
  • 比較の前に、両方の単語リストは qsort でソートされ、qunique によって重複が排除されます。

アルゴリズムまたは振る舞い

1. collectTSQueryValues を使用して、両方の TSQuery から単語の配列を抽出します。 2. 各配列を cmp_string を用いてソートし、qunique で重複を除去します。 3. 比較対象の単語数（ex_nvalues）が、包含元（query_nvalues）より多い場合は false を返します。 4. そうでない場合、ex_values の各単語が query_values に含まれているかを線形探索で確認します。 5. すべての単語が見つかれば true、一つでも欠けていれば false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery_op.c:356（DirectFunctionCall2 を介した呼び出しの可能性あり）
  • **呼び出し先**:
  • collectTSQueryValues
  • qsort
  • qunique
  • cmp_string
  • strcmp

関連データ構造

  • TSQuery: 全文検索クエリの内部表現。
  • QTNode / QueryItem: tsearch サブシステムで使用されるクエリの構成要素。

失敗モードと境界条件

  • ex_nvalues > query_nvalues の場合、包含は不可能とみなされ即座に false を返します。
  • 単語リストが空の場合の挙動は、qunique の結果に依存します。

保守担当者向けメモ

  • 本関数は構造的な包含ではなく、単語集合としての包含を判定します。
  • collectTSQueryValues や qunique の実行コストがクエリの複雑さに比例するため、大規模なクエリに対する性能特性に注意してください。
  • メモリ管理については、PG_GETARG_TSQUERY で取得した引数が適切に処理されているか確認してください。
