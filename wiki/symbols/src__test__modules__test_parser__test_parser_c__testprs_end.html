testprs_end

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_parser/test_parser.c:98

目的

全文検索用カスタムパーサーの解析セッションを終了し、割り当てられたメモリリソースを解放する。

シグネチャまたは形状

Datum testprs_end(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_GETARG_POINTER(0) を介して ParserState 構造体へのポインタを受け取る。
  • 出力: PG_RETURN_VOID() により、戻り値なし（void）を返す。

前提条件と不変条件

  • 入力されたポインタは、testprs_start によって事前に確保された有効な ParserState 構造体を指している必要がある。

アルゴリズムまたは振る舞い

1. 引数として渡された ParserState 構造体へのポインタを取得する。 2. pfree を呼び出し、当該 ParserState 構造体が占有していたメモリを解放する。 3. PG_RETURN_VOID() を実行して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLの関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録済み）。
  • 呼び出し先: pfree（PostgreSQLのメモリ解放関数）。

関連データ構造

  • ParserState: 解析状態を保持する構造体。本関数により破棄される。

失敗モードと境界条件

  • 無効なポインタが渡された場合、pfree 実行時にメモリ破壊やセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は testprs_start と対になるリソース解放関数である。パーサーのライフサイクル管理において、解析終了時に必ず呼び出されることを前提としている。
