gin_tsquery_consistent_6args

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsginidx.c:327

目的

GINインデックスにおける全文検索の整合性チェックを行うための、後方互換性を維持したラッパー関数です。

シグネチャまたは形状

Datum gin_tsquery_consistent_6args(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (関数呼び出し情報 fcinfo)。
  • **出力**: Datum 型の真偽値（bool）。インデックスの検索結果がクエリ条件を満たすかどうかを返します。

前提条件と不変条件

  • PG_NARGS() が 8 未満である場合、エラーが発生します（実装上のガードレール）。
  • 内部的に gin_tsquery_consistent を呼び出すため、同関数が要求する引数構造を満たしている必要があります。

アルゴリズムまたは振る舞い

1. 引数の数をチェックし、8個未満であれば elog(ERROR) を発行して処理を中断します。 2. 正常な場合は、受け取った fcinfo をそのまま gin_tsquery_consistent に引き渡し、その戻り値を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（SQLインターフェース経由）。
  • **呼び出し先**: gin_tsquery_consistent。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約で使用される引数コンテナ。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • **引数不足**: PG_NARGS() < 8 の場合、ERROR レベルのログを出力してトランザクションを停止させます。

保守担当者向けメモ

  • 本関数は、古いバージョンの関数シグネチャに対するスタブです。
  • 新規開発において本関数を直接呼び出す必要はありません。
  • 整合性チェックのロジック本体は gin_tsquery_consistent に集約されています。
