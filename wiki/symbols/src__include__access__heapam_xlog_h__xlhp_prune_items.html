xlhp_prune_items

定義

  • 種別: struct
  • 定義箇所: src/include/access/heapam_xlog.h:378

目的

ヒープページの整理（Pruning）処理において、対象となるタプルのオフセット番号をWALレコード内に保持するためのデータ構造です。

シグネチャまたは形状

typedef struct xlhp_prune_items
{
    uint16      ntargets;
    OffsetNumber data[FLEXIBLE_ARRAY_MEMBER];
} xlhp_prune_items;

入力と出力

  • **入力**: ntargets（対象アイテム数）および data（可変長配列としてのオフセット番号群）。
  • **出力**: なし（データ格納用構造体）。

前提条件と不変条件

  • data フィールドは FLEXIBLE_ARRAY_MEMBER として定義されており、構造体の末尾に ntargets で指定された数だけ OffsetNumber が連続して配置されることを前提としています。

アルゴリズムまたは振る舞い

  • 本構造体は、WALレコードのデコード時に、特定のオフセット番号のリストを読み取るためのカーソルとして使用されます。
  • src/backend/access/rmgrdesc/heapdesc.c において、WALレコードのストリームから本構造体へキャストし、offsetof(xlhp_prune_items, data) を用いて後続のデータ位置を計算する処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元（参照）**:
  • src/backend/access/heap/pruneheap.c: redirect_items, dead_items, unused_items としてインスタンス化。
  • src/backend/access/rmgrdesc/heapdesc.c: WALレコードのデコード処理でキャストして使用。
  • **呼び出し先**: 情報なし。

関連データ構造

  • xl_heap_prune: 本構造体を含む、より広範なヒープ整理用WALレコード構造体。

失敗モードと境界条件

  • ntargets が 0 の場合、data 配列には要素が含まれません。
  • 可変長構造体であるため、メモリレイアウトを直接キャストして扱う際は、offsetof を用いたオフセット計算が必須です。

保守担当者向けメモ

  • 本構造体は xl_heap_prune の一部として使用されます。解析時には src/backend/access/heap/pruneheap.c におけるメモリ配置と、heapdesc.c でのポインタ演算の整合性を確認してください。
