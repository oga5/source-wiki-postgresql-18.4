hash_xlog_split_page

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash_xlog.c:427

目的

ハッシュインデックスの分割（Split）操作に関連するWALレコードをリプレイし、対象ページの物理的な状態を復元する。

シグネチャまたは形状

static void hash_xlog_split_page(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record - リプレイ対象のWALレコード情報。
  • **出力**: なし（副作用としてバッファの更新と解放を行う）。

前提条件と不変条件

  • WALレコードには、対象ページのフルページイメージが含まれている必要がある。
  • 含まれていない場合、エラー（elog(ERROR)）が発生する。

アルゴリズムまたは振る舞い

1. XLogReadBufferForRedo を呼び出し、WALレコードからインデックスページを読み込む。 2. 読み込み結果が BLK_RESTORED（フルページイメージによる復元）でない場合、エラーを投げる。 3. 読み込んだバッファに対して必要な処理が完了したとみなし、UnlockReleaseBuffer を呼び出してバッファのロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/hash/hash_xlog.c 内の hash_redo（推測されるディスパッチ処理）から呼び出される。
  • **呼び出し先**:
  • XLogReadBufferForRedo
  • elog
  • UnlockReleaseBuffer

関連データ構造

  • XLogReaderState: WALレコードの読み取り状態を保持する構造体。
  • Buffer: PostgreSQLのバッファ管理用ハンドル。

失敗モードと境界条件

  • WALレコード内にフルページイメージが存在しない場合、elog(ERROR) によりリカバリ処理が中断される。

保守担当者向けメモ

  • 本関数は、ハッシュインデックスの分割処理において、ページの状態を強制的に上書きする単純なリプレイロジックである。
  • 冪等性を維持するため、フルページイメージの適用のみに依存している。
