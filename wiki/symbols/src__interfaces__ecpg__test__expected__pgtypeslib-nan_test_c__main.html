目的

ECPG（Embedded SQL for C）における float8 型および numeric 型の特殊値（NaN、infinity）の取り扱いを検証する回帰テストのエントリポイント。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は使用しない）
  • 出力: 成功時に 0 を返し、テスト結果を標準出力（stdout）に書き出す。エラー発生時は sqlprint() により標準エラー出力にエラー情報が出力される。

前提条件と不変条件

  • データベース ecpg1_regression への接続が確立可能であること。
  • ECPGライブラリが正しく初期化され、SQL文の実行環境が整っていること。
  • numeric 型の操作には PGTYPESnumeric_new() によるメモリ確保が必要。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、データベースへ接続。テスト用テーブル nantest1（float8用）および nantest2（numeric用）を作成。 2. **float8 テスト**:

  • NaN、infinity、-infinity を含むデータを挿入。
  • カーソルを使用してデータをフェッチし、isnan() および isinf() を用いて特殊値を判定。結果を標準出力に表示。
  • 挿入した値を基に、演算結果を再度テーブルへ挿入して整合性を確認。

    • 3. **numeric テスト**:

  • numeric 型の NaN を挿入。
  • select で取得した numeric 構造体の sign メンバが NUMERIC_NAN であるかを判定し、結果を表示。

    • 4. **終了処理**: カーソルを閉じ、トランザクションの終了とデータベース切断を行う。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラム実行時）
  • 呼び出し先:
  • ECPGconnect, ECPGdo: ECPGライブラリ関数（SQL実行用）
  • PGTYPESnumeric_new, PGTYPESnumeric_free: 数値型管理関数
  • isnan, isinf: C標準ライブラリ（浮動小数点判定用）
  • sqlprint, printf: 出力関数

関連データ構造

  • numeric: pgtypes_numeric.h で定義される数値型構造体。
  • sqlca: ECPGの実行状態を保持する構造体。

失敗モードと境界条件

  • データベース接続失敗やSQL実行エラーが発生した場合、sqlca.sqlcode < 0 を検知して sqlprint() を呼び出し、エラー情報を出力する。
  • loopcount によるループ制限（100回）があり、カーソルフェッチが終了しない場合の安全装置となっている。
