ExecSimpleRelationUpdate

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execReplication.c:650

目的

論理レプリケーションにおいて、リモートから受信した更新データをローカルテーブルに適用する。トリガーの実行、制約チェック、および競合検出を含む一連の更新処理を行う。

シグネチャまたは形状

void ExecSimpleRelationUpdate(ResultRelInfo *resultRelInfo, EState *estate, EPQState *epqstate, TupleTableSlot *searchslot, TupleTableSlot *slot)

入力と出力

  • **入力**:
  • resultRelInfo: 対象テーブルの実行時情報。
  • estate: 実行状態。
  • epqstate: EPQ（EvalPlanQual）状態。
  • searchslot: 更新対象行を特定するためのTIDを含むスロット。
  • slot: 更新後のデータを含むスロット。
  • **出力**: なし（副作用としてテーブルを更新）。

前提条件と不変条件

  • 対象テーブルはシステムカタログ以外の通常テーブル（RELKIND_RELATION）であること。
  • CheckCmdReplicaIdentity により、対象テーブルが更新操作に必要なレプリカアイデンティティ要件を満たしていること。

アルゴリズムまたは振る舞い

1. **トリガー実行**: ExecBRUpdateTriggers を呼び出し、BEFORE ROW UPDATEトリガーを実行する。戻り値がfalseの場合は処理をスキップする。 2. **データ準備**: 生成列（Stored Generated Columns）の計算を行う。 3. **制約チェック**: テーブル制約およびパーティションチェックを実行する。 4. **更新実行**: simple_table_tuple_update を呼び出し、物理的な更新を行う。 5. **インデックス更新**: ExecInsertIndexTuples を呼び出し、関連するインデックスを更新する。 6. **競合報告**: 更新により競合が発生した場合は CheckAndReportConflict を呼び出す。 7. **トリガー実行**: ExecARUpdateTriggers を呼び出し、AFTER ROW UPDATEトリガーを実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/worker.c (論理レプリケーションの適用プロセス)
  • **呼び出し先**:
  • CheckCmdReplicaIdentity
  • ExecBRUpdateTriggers / ExecARUpdateTriggers
  • ExecComputeStoredGenerated
  • ExecConstraints / ExecPartitionCheck
  • simple_table_tuple_update
  • ExecInsertIndexTuples
  • CheckAndReportConflict

関連データ構造

  • ResultRelInfo: テーブルの実行時情報。
  • TupleTableSlot: 行データを保持する構造体。
  • EState: 実行状態管理。

失敗モードと境界条件

  • レプリカアイデンティティが不十分な場合、CheckCmdReplicaIdentity によりエラーが発生する。
  • 制約違反やパーティションチェック失敗時にエラーが発生する。
  • 同時実行による競合が発生した場合、CheckAndReportConflict によりエラー報告が行われる。

保守担当者向けメモ

  • 本関数は論理レプリケーションの適用エンジンの一部であり、トリガーの実行順序（BEFORE/AFTER）が厳密に管理されています。
  • simple_table_tuple_update は低レベルな更新インターフェースであり、ここでの更新結果に基づいてインデックスの再スキャンや競合チェックが行われる点に注意してください。
