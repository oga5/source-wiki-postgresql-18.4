remove_rel_from_query

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/analyzejoins.c:340

目的

クエリ最適化の過程で不要と判断されたリレーション（RelOptInfo）を、プランナの内部データ構造から削除し、関連する参照を調整します。

シグネチャまたは形状

static void remove_rel_from_query(PlannerInfo *root, RelOptInfo *rel, int subst, SpecialJoinInfo *sjinfo, Relids joinrelids)

入力と出力

  • root: プランナのグローバル情報。
  • rel: 削除対象のリレーション。
  • subst: 置換先の relid（自己結合排除時）。削除のみの場合は -1。
  • sjinfo: 削除対象の結合に関する情報（NULL の場合あり）。
  • joinrelids: 結合対象のリレーション集合。

前提条件と不変条件

  • 削除対象の relid が root->all_baserels 等に含まれていること。
  • sjinfo が指定されている場合、その ojrelid が関連する集合から正しく削除されること。
  • PlaceHolderInfo の ph_eval_at が空にならないことが保証されていること。

アルゴリズムまたは振る舞い

1. **リレーション集合の更新**: root->all_baserels および all_query_rels から対象 relid を削除または置換します。 2. **SpecialJoinInfo の調整**: join_info_list 内の各 SpecialJoinInfo について、min_lefthand 等の relid 集合を調整します。sjinfo がある場合は ojrelid を削除し、ない場合は ChangeVarNodesExtended で式内の参照を置換します。 3. **PlaceHolderVar の処理**: 不要になった PlaceHolderInfo を削除し、残存するものは ph_eval_at や ph_lateral を更新します。 4. **EquivalenceClass の更新**: remove_rel_from_eclass を呼び出し、等価クラス内の参照を調整します。 5. **再計算**: 最後に、attr_needed や ph_needed を再構築し、プランナの整合性を維持します。

関連データ構造

  • PlannerInfo: プランニング全体のコンテキスト。
  • RelOptInfo: 削除対象のリレーション情報。
  • SpecialJoinInfo: 結合の制約情報。
  • PlaceHolderInfo: プレースホルダ変数情報。
  • EquivalenceClass: 等価クラス。

失敗モードと境界条件

  • sjinfo が NULL の場合、自己結合排除として振る舞い、subst > 0 であることがアサートされます。
  • PlaceHolderVar の削除判定において、結合パートナーで使用されているものと結合条件で使用されているものの区別が困難な場合があり、慎重な ph_eval_at のチェックが行われます。

保守担当者向けメモ

  • 本関数はクエリ内の varno を書き換える複雑な処理を含みます。ChangeVarNodesExtended を使用した参照置換は、プランナのデータ構造を破壊しないよう設計されています。
  • 結合削除後に attr_needed を再計算する処理は、他の外結合の削除可能性を正しく判定するために不可欠です。
