overexplain_per_plan_hook

定義

  • 種別: function
  • 定義箇所: contrib/pg_overexplain/pg_overexplain.c:251

目的

PostgreSQL の EXPLAIN コマンド実行時に、クエリ全体（PlannedStmt）に対して拡張されたプラン情報やレンジテーブルの詳細情報を追加出力するためのフック関数です。

シグネチャまたは形状

static void overexplain_per_plan_hook(PlannedStmt *plannedstmt,
                                      IntoClause *into,
                                      ExplainState *es,
                                      const char *queryString,
                                      ParamListInfo params,
                                      QueryEnvironment *queryEnv)

入力と出力

  • **入力**:
  • plannedstmt: 実行計画のルートノード。
  • es: EXPLAIN コマンドの現在の状態を保持する ExplainState。
  • その他 (into, queryString, params, queryEnv): EXPLAIN コマンドの標準的な実行コンテキスト情報。
  • **出力**: なし（ExplainState を通じて EXPLAIN 出力バッファに情報を書き込みます）。

前提条件と不変条件

  • _PG_init にて explain_per_plan_hook に登録されている必要があります。
  • GetExplainExtensionState を通じて、ExplainState 内に overexplain_options がアタッチされている場合にのみ拡張出力が行われます。

アルゴリズムまたは振る舞い

1. **既存フックの実行**: prev_explain_per_plan_hook が存在する場合、まずそれを呼び出し、標準的なフック処理を継続させます。 2. **状態取得**: GetExplainExtensionState を使用して、現在の ExplainState から overexplain_options を取得します。 3. **条件付き出力**:

  • options->debug が有効な場合、overexplain_debug を呼び出しデバッグ情報を出力します。
  • options->range_table が有効な場合、overexplain_range_table を呼び出しレンジテーブル情報を出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の EXPLAIN サブシステム（src/backend/commands/explain.c 経由で呼び出されるフック）。
  • **呼び出し先**:
  • prev_explain_per_plan_hook（存在する場合）
  • overexplain_debug
  • overexplain_range_table

関連データ構造

  • PlannedStmt: 実行計画の構造体。
  • ExplainState: EXPLAIN コマンドの出力状態を管理する構造体。
  • overexplain_options: 本拡張モジュールが保持するオプション状態。

失敗モードと境界条件

  • GetExplainExtensionState が NULL を返した場合（本拡張オプションが指定されていない場合）、何も出力せずに正常終了します。

保守担当者向けメモ

  • 本関数は explain_per_plan_hook をオーバーライドする形式で実装されています。元のフックを呼び出す prev_explain_per_plan_hook の実行順序は、他の拡張モジュールとの競合を避けるため、この順序（既存フックを先に実行）が維持される必要があります。
