CatalogCloseIndexes

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/indexing.c:60

目的

CatalogOpenIndexes によって確保されたインデックス関連のリソースを解放し、クリーンアップを行います。

シグネチャまたは形状

void CatalogCloseIndexes(CatalogIndexState indstate)

入力と出力

  • **入力**: indstate (CatalogIndexState) - 解放対象となるインデックス状態を保持する構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • indstate は CatalogOpenIndexes によって正常に初期化されている必要があります。
  • 本関数呼び出し後、indstate が指すメモリ領域および関連するインデックスリソースは無効となります。

アルゴリズムまたは振る舞い

1. ExecCloseIndices(indstate) を呼び出し、開かれているインデックスのリレーションをクローズします。 2. pfree(indstate) を呼び出し、CatalogIndexState 構造体自体が使用していたメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c
  • src/backend/catalog/index.c
  • src/backend/catalog/indexing.c
  • src/backend/catalog/pg_depend.c
  • src/backend/catalog/pg_enum.c
  • **呼び出し先**:
  • ExecCloseIndices
  • pfree

関連データ構造

  • CatalogIndexState: インデックス情報を保持する不透明な構造体（実体は ResultRelInfo）。

失敗モードと境界条件

  • indstate が NULL の場合、pfree 等の内部関数によりセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数はカタログ操作の終了処理における定型的なクリーンアップを担当します。
  • CatalogOpenIndexes と対になる関数であるため、リソースリークを防ぐために必ずペアで呼び出される必要があります。
