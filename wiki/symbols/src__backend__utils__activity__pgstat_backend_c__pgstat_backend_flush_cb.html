pgstat_backend_flush_cb

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/activity/pgstat_backend.c:312

目的

バックエンドプロセス固有の統計情報（I/O および WAL 使用量）を、共有メモリ上の統計領域へフラッシュするためのコールバック関数です。

シグネチャまたは形状

bool pgstat_backend_flush_cb(bool nowait)

入力と出力

  • **入力**:
  • nowait (bool): true の場合、ロック取得時に待機せず即座に失敗を返します。
  • **出力**:
  • bool: フラッシュ処理が成功した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • 本関数は、統計サブシステムがバックエンド統計をフラッシュする際のコールバックとして登録されている必要があります。
  • 内部で呼び出される pgstat_flush_backend は、メモリ割り当てを行わない設計であるため、クリティカルセクション内からの呼び出しが可能です。

アルゴリズムまたは振る舞い

1. pgstat_flush_backend を呼び出し、蓄積された統計情報を共有メモリへ書き出します。 2. その際、フラッシュ対象として PGSTAT_BACKEND_FLUSH_ALL を指定し、I/O および WAL 統計の双方を対象とします。 3. pgstat_flush_backend の戻り値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/pgstat.c:378: 統計サブシステムのフラッシュ用コールバックテーブル（.flush_static_cb）に登録されています。
  • **呼び出し先**:
  • pgstat_flush_backend (同一ファイル内): 実際のフラッシュ処理を実行するメイン関数。

関連データ構造

  • PendingBackendStats: プロセスローカルに保持されている統計情報のバッファ。
  • PgStatShared_Backend: 共有メモリ上に存在するバックエンド統計の格納先。

失敗モードと境界条件

  • nowait が true の場合、共有メモリのロック取得に失敗すると false を返します。
  • 統計対象のバックエンド種別が pgstat_tracks_backend_bktype によって収集対象外と判定されている場合、フラッシュ処理はスキップされます。

保守担当者向けメモ

  • 本関数は pgstat_flush_backend への単なるラッパーです。統計のフラッシュロジックを変更する場合は、pgstat_flush_backend を参照してください。
  • WAL 統計の計算には prevBackendWalUsage を用いた差分更新方式が採用されています。
