heap_xlog_multi_insert

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_xlog.c:535

目的

WAL（Write-Ahead Log）リプレイ中に、ヒープページへの複数タプル挿入操作を再現し、ページ状態を更新する。

シグネチャまたは形状

static void heap_xlog_multi_insert(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record （リプレイ対象のWALレコード）
  • **出力**: なし（副作用としてバッファおよび可視性マップを更新）

前提条件と不変条件

  • XLH_INSERT_ALL_VISIBLE_CLEARED フラグと XLH_INSERT_ALL_FROZEN_SET フラグは同時に設定できない（Assertによる検証）。
  • 挿入操作はMVCCデータを上書きしないため、Hot Standbyにおけるコンフリクト処理は不要。

アルゴリズムまたは振る舞い

1. **可視性マップの更新**: XLH_INSERT_ALL_VISIBLE_CLEARED フラグがある場合、対象ブロックの可視性マップビットをクリアする。 2. **バッファの準備**: XLOG_HEAP_INIT_PAGE フラグがあればページを初期化し、それ以外は XLogReadBufferForRedo でバッファを取得する。 3. **タプルの挿入**:

  • WALレコードからタプルデータを取り出し、各タプルのヘッダー情報を構築する。
  • PageAddItem を使用してページへタプルを書き込む。

    • 4. **ページ状態の確定**:

  • ページLSNを更新し、バッファをダーティにする。
  • フラグに応じて PageClearAllVisible または PageSetAllVisible を実行。

    • 5. **後処理**: バッファをアンロック・解放し、空き容量が少ない（BLCKSZ / 5 未満）場合は XLogRecordPageWithFreeSpace を呼び出してFSM（フリースペースマップ）を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_xlog.c 内の heap_redo（または関連するリプレイディスパッチャ）から呼び出されることが示唆されている。
  • **呼び出し先**: visibilitymap_pin, visibilitymap_clear, XLogInitBufferForRedo, XLogReadBufferForRedo, PageAddItem, PageSetLSN, MarkBufferDirty, XLogRecordPageWithFreeSpace 等のアクセスメソッドおよびバッファ管理関数。

関連データ構造

  • xl_heap_multi_insert: WALレコードのヘッダー構造体。
  • xl_multi_insert_tuple: 各タプルのメタデータ構造体。
  • XLogReaderState: WAL読み取り状態。
  • Buffer, Page: ヒープページ管理用構造体。

失敗モードと境界条件

  • PageGetMaxOffsetNumber(page) + 1 < offnum: ページ内のオフセット番号が不正な場合に PANIC。
  • PageAddItem が InvalidOffsetNumber を返した場合に PANIC。
  • WALレコード内のタプルデータ長と期待される合計長が一致しない場合に PANIC。

保守担当者向けメモ

  • 本関数はページ内の物理的なタプル配置を扱う。論理的な整合性はWAL生成側で担保されている前提。
  • FSMの更新は、ページがフルページイメージから復元された場合にはスキップされる。
  • XLH_INSERT_ALL_FROZEN_SET が設定されている場合、すべてのタプルが可視であると見なされる点に注意。
