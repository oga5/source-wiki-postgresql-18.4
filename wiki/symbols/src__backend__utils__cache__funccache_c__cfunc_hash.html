cfunc_hash

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/funccache.c:84

目的

関数キャッシュ用ハッシュテーブルにおいて、CachedFunctionHashKey をハッシュ化し、テーブル内のエントリ検索を効率化するために使用されます。

シグネチャまたは形状

static uint32 cfunc_hash(const void *key, Size keysize)

入力と出力

  • **入力**:
  • key: CachedFunctionHashKey 構造体へのポインタ。
  • keysize: キーのサイズ（sizeof(CachedFunctionHashKey) である必要がある）。
  • **出力**:
  • uint32: 計算されたハッシュ値。

前提条件と不変条件

  • keysize は sizeof(CachedFunctionHashKey) と一致しなければなりません（Assert により検証）。

アルゴリズムまたは振る舞い

1. CachedFunctionHashKey の固定フィールド（callResultType を除く）を hash_any でハッシュ化します。 2. nargs > 0 の場合、argtypes 配列の内容を hash_combine を用いて既存のハッシュ値に結合します。 3. callResultType が存在する場合、hashRowType を用いてその結果をさらにハッシュ値に結合します。 4. 最終的なハッシュ値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/funccache.c 内で、ハッシュテーブルの初期化時に ctl.hash として設定されます。
  • **呼び出し先**: hash_any、hash_combine、hashRowType（いずれも PostgreSQL のハッシュユーティリティ関数）。

関連データ構造

  • CachedFunctionHashKey: ハッシュ計算の対象となるキー構造体。

失敗モードと境界条件

  • keysize が CachedFunctionHashKey のサイズと一致しない場合、アサーション違反により停止します。

保守担当者向けメモ

  • 本関数は cfunc_match と対になって動作します。キー構造体 CachedFunctionHashKey にフィールドが追加された場合は、本関数でのハッシュ計算ロジックも更新する必要があります。
