GISTIntArrayOptions

定義

  • 種別: struct
  • 定義箇所: contrib/intarray/_int.h:19

目的

GiST インデックスにおける intarray オペレータクラスの動的な設定パラメータを保持するための構造体です。

シグネチャまたは形状

typedef struct
{
	int32		vl_len_;
	int			num_ranges;
} GISTIntArrayOptions;

入力と出力

  • **入力**: init_local_reloptions を通じてメモリ確保および初期化が行われます。
  • **出力**: PG_GET_OPCLASS_OPTIONS() を介して、インデックス操作中に num_ranges の値が取得されます。

前提条件と不変条件

  • vl_len_ は可変長データ（varlena）のヘッダーであり、直接操作してはなりません。
  • num_ranges はインデックスの挙動を制御する設定値として利用されます。

アルゴリズムまたは振る舞い

本構造体は、PostgreSQL のリレーションオプション管理システムと統合されており、インデックス作成時や設定変更時に num_ranges を保持・提供します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_gist.c: init_local_reloptions を呼び出し、構造体サイズを渡して初期化を行います。
  • contrib/intarray/_int.h: PG_GET_OPCLASS_OPTIONS() の戻り値をキャストして num_ranges を参照します。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • GISTIntArrayBigOptions: 類似の目的を持つ、より大きなデータセット用のオプション構造体。

失敗モードと境界条件

  • メモリ確保時に sizeof(GISTIntArrayOptions) を正しく指定しない場合、メモリ破壊や不正な値の読み取りが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を含みませんが、varlena 型であるため、メモリ確保時には PostgreSQL の標準的なマクロ（init_local_reloptions 等）を介して適切にサイズを管理してください。
