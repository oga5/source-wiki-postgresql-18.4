cstring_to_text_with_len

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varlena.c:204

目的

C言語の文字列（char *）と指定された長さを基に、PostgreSQLの可変長データ型である text 型の値を動的に生成します。

シグネチャまたは形状

text *cstring_to_text_with_len(const char *s, int len)

入力と出力

  • **入力**:
  • s: コピー元のC文字列（const char *）。
  • len: コピーするバイト数（int）。
  • **出力**:
  • 生成された text 型のポインタ。メモリは palloc によって割り当てられます。

前提条件と不変条件

  • len は、s が指すメモリ領域の有効な長さである必要があります。
  • palloc を使用するため、呼び出し元は適切なメモリコンテキストにいる必要があります。

アルゴリズムまたは振る舞い

1. len + VARHDRSZ バイトのメモリを palloc で確保し、text 型の構造体を作成します。 2. SET_VARSIZE マクロを使用して、確保したメモリサイズをヘッダーに設定します。 3. memcpy を使用して、入力文字列 s の先頭 len バイトを VARDATA（データ領域）へコピーします。 4. 生成された text ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gin/btree_gin.c
  • contrib/fuzzystrmatch/daitch_mokotoff.c
  • contrib/hstore/hstore_io.c
  • contrib/hstore/hstore_op.c
  • **呼び出し先**:
  • palloc（メモリ割り当て）
  • SET_VARSIZE（ヘッダー設定マクロ）
  • memcpy（メモリコピー）
  • VARDATA（データ領域取得マクロ）

関連データ構造

  • text: PostgreSQLの可変長文字列型。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させ、処理が中断されます。
  • len に負の値を指定した場合の動作は未定義です。

保守担当者向けメモ

  • 本関数は text 型のヘッダーサイズ（VARHDRSZ）を自動的に加算してメモリを確保するため、呼び出し側でサイズ計算を行う必要はありません。
  • ヌル終端文字を含まないバイナリデータや、部分文字列を text 型に変換する際に適しています。
