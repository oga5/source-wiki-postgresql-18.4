gist_qtxt

定義

  • 種別: function
  • 定義箇所: contrib/ltree/_ltree_gist.c:425

目的

GiSTインデックスにおいて、ltreeのキーが特定のltxtquery条件を満たすかどうかを、シグネチャを用いて判定する。

シグネチャまたは形状

static bool gist_qtxt(ltree_gist *key, ltxtquery *query, int siglen)

入力と出力

  • **入力**:
  • key: 判定対象のltree_gist構造体。
  • query: 検索条件となるltxtquery構造体。
  • siglen: シグネチャの長さ。
  • **出力**:
  • 条件を満たす場合はtrue、それ以外はfalseを返す。

前提条件と不変条件

  • keyがLTG_ISALLTRUEフラグを持つ場合、クエリの内容に関わらず常にtrueを返す。

アルゴリズムまたは振る舞い

1. keyが全真（LTG_ISALLTRUE）であれば直ちにtrueを返す。 2. keyからシグネチャデータ（LTG_SIGN）を抽出し、LtreeSignature構造体を初期化する。 3. ltree_execute関数を呼び出し、queryの条件をcheckcondition_bit関数を用いてシグネチャに対して評価する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/_ltree_gist.c:533
  • contrib/ltree/ltree_gist.c:576
  • contrib/ltree/ltree_gist.c:700
  • **呼び出し先**:
  • ltree_execute（外部関数）
  • checkcondition_bit（外部関数）

関連データ構造

  • ltree_gist: GiSTインデックスで使用されるキー構造。
  • ltxtquery: 検索条件を保持する構造体。
  • LtreeSignature: シグネチャ情報（signとsiglen）を保持する構造体。

失敗モードと境界条件

  • keyがLTG_ISALLTRUEでない場合、シグネチャの照合結果に依存する。
  • siglenが不適切な場合、ltree_executeの内部処理に影響する可能性がある。

保守担当者向けメモ

  • 本関数はltree_gist.cと_ltree_gist.cの両方から参照されている。変更時は両方のコンテキストでの動作確認が必要。
