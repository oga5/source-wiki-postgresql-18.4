GinTupleGetFirst

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/access/gin_tuple.h:32

目的

GinTuple 構造体から、そのタプルが保持する最初の ItemPointer（TID）へのポインタを取得する。

シグネチャまたは形状

static inline ItemPointer GinTupleGetFirst(GinTuple *tup)

入力と出力

  • **入力**: GinTuple *tup - 走査対象のGINタプルへのポインタ。
  • **出力**: ItemPointer - タプル内のデータ領域に格納された最初のTIDを指すポインタ。

前提条件と不変条件

  • tup は有効な GinTuple 構造体を指している必要がある。
  • GinTuple は可変長構造体であるため、アクセス時には keylen を用いたオフセット計算が必須である。
  • SHORTALIGN マクロにより、データ領域の開始位置は適切なメモリアライメントに調整される。

アルゴリズムまたは振る舞い

1. tup->data（キー値の開始位置）に tup->keylen を加算し、キーデータ直後の位置を特定する。 2. SHORTALIGN を適用して、GinPostingList 構造体の開始位置をアライメント調整する。 3. 調整されたアドレスを GinPostingList * 型にキャストする。 4. その構造体のメンバである first（ItemPointer 型）のアドレスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gininsert.c（インデックス挿入および比較処理にて複数箇所で参照）
  • **呼び出し先**:
  • SHORTALIGN マクロ（アライメント調整用）

関連データ構造

  • GinTuple: GINインデックスのキー情報を保持する可変長構造体。
  • GinPostingList: ポスティングリストのヘッダー構造体。
  • ItemPointer: タプル識別子（TID）。

失敗モードと境界条件

  • tup が NULL の場合、または tup が指すメモリ領域が不正な場合、セグメンテーション違反が発生する可能性がある。
  • keylen が実際のデータ長と一致しない場合、誤ったメモリ位置を参照する。

保守担当者向けメモ

  • 本関数はインライン関数であり、パフォーマンスが重視されるパスで使用される。
  • GinTuple は FLEXIBLE_ARRAY_MEMBER を含む構造体であるため、メモリレイアウトを変更する際は SHORTALIGN の適用箇所とアライメント規則を厳密に維持すること。
