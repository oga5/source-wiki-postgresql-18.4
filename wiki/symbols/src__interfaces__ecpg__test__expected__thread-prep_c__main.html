main

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/thread-prep.c:192

目的

ECPG（Embedded SQL for C）におけるマルチスレッド環境下でのプリペアードステートメントの並行実行およびライフサイクル管理を検証する回帰テストのエントリポイント。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（コマンドライン引数は使用しない）
  • 出力: 成功時に 0 を返す。

前提条件と不変条件

  • 実行環境において、ECPGライブラリおよびデータベース接続（ecpg1_regression）が利用可能であること。
  • プラットフォームに応じて、POSIXスレッド（pthread）またはWindowsスレッドAPIが利用可能であること。

アルゴリズムまたは振る舞い

1. **データベース初期化**: ECPGconnect で接続し、drop table if exists T および create table T ( i int ) を実行してテーブルを準備した後、接続を切断する。 2. **スレッド生成**: THREADS 定数で指定された数のスレッドを生成する。

  • Windows環境: _beginthreadex を使用し、スレッドハンドルを配列に保持する。
  • POSIX環境: pthread_create を使用し、スレッドIDを配列に保持する。

    • 3. **並行実行**: 各スレッドは fn 関数を実行し、個別の接続確立、プリペアードステートメントの実行、解放、接続切断を行う。 4. **同期**: 全スレッドの終了を待機する。

  • Windows環境: WaitForMultipleObjects で待機し、CloseHandle でリソースを解放する。
  • POSIX環境: pthread_join で待機する。

    • 5. **エラーハンドリング**: 各SQL操作後に sqlca.sqlcode < 0 を確認し、エラー発生時は sqlprint() を呼び出す。

呼び出し元と呼び出し先

  • 呼び出し元: OSのプロセスローダー（エントリポイント）。
  • 呼び出し先:
  • ECPGライブラリ関数: ECPGconnect, ECPGsetcommit, ECPGdo, ECPGdisconnect
  • スレッド制御関数: _beginthreadex / WaitForMultipleObjects / CloseHandle (WIN32)、または pthread_create / pthread_join (POSIX)
  • テスト関数: fn

関連データ構造

  • sqlca: SQL実行結果を保持するグローバルな通信領域。
  • threads: スレッドハンドルまたはスレッドIDを格納する配列。

失敗モードと境界条件

  • データベース接続失敗やテーブル作成失敗時に sqlprint() が実行される。
  • THREADS 数が過剰な場合、OSのリソース制限によりスレッド生成が失敗する可能性がある。
  • プラットフォーム固有のAPI（WIN32 vs POSIX）のいずれかが利用できない環境ではコンパイルまたは実行が失敗する。

保守担当者向けメモ

  • 本関数はプリプロセッサが生成した期待値コードであるため、直接編集しないこと。
  • WindowsとPOSIXの条件分岐（#ifdef WIN32）が含まれているため、修正時は両プラットフォームへの影響を考慮すること。
