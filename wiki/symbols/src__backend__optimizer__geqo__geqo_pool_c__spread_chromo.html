spread_chromo

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_pool.c:186

目的

遺伝的アルゴリズムの個体群（Pool）に対し、新しい染色体（Chromosome）を挿入し、評価値（worth）に基づいて順序を維持しつつ、最も評価の低い個体を排除する。

シグネチャまたは形状

void spread_chromo(PlannerInfo *root, Chromosome *chromo, Pool *pool)

入力と出力

  • **入力**:
  • root: プランナ情報へのポインタ。
  • chromo: プールに挿入する新しい染色体。
  • pool: 染色体を保持するプール構造体。
  • **出力**: なし（副作用として pool 内のデータが更新される）。

前提条件と不変条件

  • pool 内の染色体は、評価値（worth）に基づいて昇順（コストが低い順）にソートされている必要がある。
  • 評価値は小さいほど優秀であると見なされる。

アルゴリズムまたは振る舞い

1. **初期判定**: 新しい染色体の worth がプール内の最悪の個体（末尾）より大きい場合、挿入せず即座に終了する。 2. **二分探索**: プール内での適切な挿入位置（index）を二分探索で特定する。 3. **挿入とシフト**:

  • geqo_copy を使用して、新しい染色体をプールの末尾にコピーする。
  • index 以降の既存の個体を一つずつ後ろにずらし、index の位置に新しい染色体を配置する。
  • この際、ポインタの入れ替え（スワップ）によって効率的にシフト処理を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/geqo/geqo_main.c:224
  • **呼び出し先**:
  • geqo_copy（遺伝情報の複製）

関連データ構造

  • Chromosome: 染色体データ（string と worth を保持）。
  • Pool: 染色体の配列（data）とサイズ（size）を保持する管理構造体。

失敗モードと境界条件

  • 新しい染色体が既存のどの個体よりも評価が低い（worth が大きい）場合、何も行わずに終了する。
  • プールサイズが小さい場合でも、二分探索のロジックにより適切に挿入位置が決定される。

保守担当者向けメモ

  • 本関数は D. Whitley の Genitor アルゴリズムに基づいている。
  • シフト処理はポインタの入れ替えで行われるため、メモリのコピーコストを最小限に抑えている。
  • 評価値の大小比較は worth フィールドに依存しているため、比較ロジックを変更する場合は注意が必要。
