SetHintBits

定義

  • 種別: static inline function
  • 定義箇所: src/backend/access/heap/heapam_visibility.c:113

目的

タプルのヘッダー情報（t_infomask）を更新し、トランザクションのコミット/アボート状態をキャッシュすることで、将来の可視性判定コストを削減する。

シグネチャまたは形状

static inline void SetHintBits(HeapTupleHeader tuple, Buffer buffer, uint16 infomask, TransactionId xid)

入力と出力

  • tuple: 更新対象のタプルヘッダー。
  • buffer: タプルが格納されているバッファ。
  • infomask: 設定するビットマスク（HEAP_XMIN_COMMITTED 等）。
  • xid: 関連するトランザクションID。有効な場合、コミット状態の検証に使用される。
  • 出力: なし（副作用としてタプルヘッダーとバッファを更新）。

前提条件と不変条件

  • 呼び出し元は、対象のバッファに対して少なくとも共有ロック（Shared Lock）を保持している必要がある。
  • xid が有効な場合、そのトランザクションは既にコミット済みであることが前提である。

アルゴリズムまたは振る舞い

1. xid が有効な場合、そのコミットLSNを取得する。 2. 永続的なバッファであり、かつコミットLSNが未フラッシュでバッファのLSNより進んでいる場合は、データ整合性を守るためヒントビットを設定せずに終了する。 3. 上記条件に該当しない場合、tuple->t_infomask に infomask を論理和で追加する。 4. MarkBufferDirtyHint(buffer, true) を呼び出し、バッファをダーティとしてマークする。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/heapam_visibility.c 内の可視性判定関数群（HeapTupleSatisfiesMVCC 等）および HeapTupleSetHintBits。
  • 呼び出し先: TransactionIdGetCommitLSN, BufferIsPermanent, XLogNeedsFlush, BufferGetLSNAtomic, MarkBufferDirtyHint。

関連データ構造

  • HeapTupleHeader: タプルのメタデータ（t_infomask を保持）。
  • Buffer: 共有バッファプール内のページ管理構造。
  • TransactionId: トランザクション識別子。

失敗モードと境界条件

  • バッファが永続的でない場合や、WALフラッシュの競合が発生する状況では、ヒントビットの更新がスキップされる（安全側に倒した設計）。

保守担当者向けメモ

  • 本関数は heapam_visibility.c 内の可視性判定ロジックにおける最適化の要である。
  • MarkBufferDirtyHint を使用しているため、バッファのロック状態が適切であることを確認すること。
  • xid がコミット済みであるという前提条件は、呼び出し側のロジックに強く依存している。
