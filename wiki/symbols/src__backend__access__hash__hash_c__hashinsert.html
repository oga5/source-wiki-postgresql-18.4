hashinsert

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash.c:257

目的

ハッシュインデックスへ新しいタプルを挿入します。

シグネチャまたは形状

bool hashinsert(Relation rel, Datum *values, bool *isnull,
                ItemPointer ht_ctid, Relation heapRel,
                IndexUniqueCheck checkUnique,
                bool indexUnchanged,
                IndexInfo *indexInfo)

入力と出力

  • **入力**:
  • rel: 対象のハッシュインデックスリレーション。
  • values, isnull: 挿入するキー値とその NULL 状態。
  • ht_ctid: ヒープタプルを指す TID。
  • heapRel: ヒープリレーション。
  • checkUnique, indexUnchanged, indexInfo: インデックス挿入の制約やメタ情報。
  • **出力**:
  • 常に false を返します。

前提条件と不変条件

  • _hash_convert_tuple が成功した場合のみ、インデックスへの挿入処理が進行します。
  • 挿入に使用した一時的な IndexTuple は、関数終了前に pfree により解放されます。

アルゴリズムまたは振る舞い

1. _hash_convert_tuple を呼び出し、入力値をハッシュインデックス用のキーに変換します。変換に失敗した場合は即座に false を返します。 2. index_form_tuple を使用して、変換後の値からインデックス用のタプル（itup）を生成し、ht_ctid を設定します。 3. _hash_doinsert を呼び出し、適切なバケットへタプルを配置します。 4. 生成した itup をメモリから解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/hash/hash.c 内の amroutine 構造体（aminsert フィールド）を通じて、PostgreSQL のインデックス挿入処理から呼び出されます。
  • **呼び出し先**:
  • _hash_convert_tuple
  • index_form_tuple
  • _hash_doinsert
  • pfree

関連データ構造

  • Relation: インデックスおよびヒープのメタデータ。
  • IndexTuple: ハッシュインデックスに格納されるタプル形式。

失敗モードと境界条件

  • _hash_convert_tuple が失敗した場合、インデックスへの挿入は行われず、関数は false を返します。

保守担当者向けメモ

  • 本関数はインターフェース層であり、具体的な挿入ロジックは _hash_doinsert に委譲されています。
  • 挿入処理の詳細については src/backend/access/hash/hashinsert.c を参照してください。
