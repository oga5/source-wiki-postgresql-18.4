PGEventResultCreate

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/libpq-events.h:52

目的

PGresult が生成された際に、イベントハンドラへ関連する接続情報と生成された結果オブジェクトを渡すためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	PGconn	   *conn;
	PGresult   *result;
} PGEventResultCreate;

入力と出力

  • **入力**: 本構造体はイベントハンドラ（PGEventProc）の引数として渡されます。
  • **出力**: 構造体自体はデータ保持用であり、直接的な戻り値はありません。

前提条件と不変条件

  • conn は有効な PGconn オブジェクトを指している必要があります。
  • result は新しく作成された PGresult オブジェクトを指しています。

アルゴリズムまたは振る舞い

libpq 内部で PGresult が生成された際、登録済みのイベントハンドラに対してこの構造体が通知されます。ハンドラはこの構造体を通じて、生成された結果セットに対して必要な初期化やカスタムデータの紐付け（PQresultInstanceData を使用）を行うことができます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/libpq-events.c 内のイベント発行ロジックから参照されていることが確認されています。
  • **呼び出し先**: 本構造体自体はデータコンテナであり、関数呼び出しは行いません。

関連データ構造

  • PGconn: データベース接続オブジェクト。
  • PGresult: クエリ結果オブジェクト。
  • PGEventProc: 本構造体を受け取るイベントハンドラの型定義。

失敗モードと境界条件

  • conn または result が NULL である場合の挙動については、本ソース excerpt からは判断できません。

保守担当者向けメモ

  • 本構造体は PGresult のライフサイクルイベントに関連しています。メモリ管理を行う際は、対応する PGEVT_RESULTDESTROY イベントで適切にリソースを解放してください。
