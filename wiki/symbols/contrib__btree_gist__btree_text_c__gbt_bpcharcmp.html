gbt_bpcharcmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_text.c:142

目的

bpchar（空白埋め固定長文字）型データに対する比較処理を、GiSTインデックスのサポート関数として提供します。

シグネチャまたは形状

int32 gbt_bpcharcmp(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象のデータポインタ。
  • collation: 照合順序（Collation）のOID。
  • flinfo: 関数マネージャ情報。
  • **出力**:
  • int32: a と b の比較結果（a < b なら負、a == b なら 0、a > b なら正）。

前提条件と不変条件

  • bpcharcmp 関数がシステムカタログに存在し、DirectFunctionCall2Coll を介して呼び出し可能であること。

アルゴリズムまたは振る舞い

1. PointerGetDatum を使用して、入力された void ポインタを Datum 型に変換します。 2. DirectFunctionCall2Coll を経由して、PostgreSQL標準の bpcharcmp 関数を呼び出します。 3. 戻り値を DatumGetInt32 で int32 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の参照情報に基づくと、contrib/btree_gist/btree_text.c 内の bptinfo 構造体定義（161行目）で参照されています。
  • **呼び出し先**: PostgreSQL標準の bpcharcmp 関数を呼び出します。

関連データ構造

  • gbtree_vinfo: bpchar 型の比較ロジックを保持するために使用される構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、PointerGetDatum および bpcharcmp の動作に依存します。
  • 照合順序（collation）が不正な場合、bpcharcmp の実行時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は bpchar 型専用の比較ラッパーです。text 型の比較には gbt_textcmp が使用されます。
  • DirectFunctionCall2Coll を使用しているため、標準的な比較ロジックを直接利用しており、GiST特有の複雑な比較ロジックは含まれていません。
