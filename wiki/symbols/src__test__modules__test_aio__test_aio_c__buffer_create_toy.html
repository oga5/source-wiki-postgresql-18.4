buffer_create_toy

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_aio/test_aio.c:509

目的

PostgreSQL の非同期 I/O (AIO) テスト用モジュールにおいて、指定されたリレーションとブロック番号に対応する「トイ（テスト用）バッファ」を生成するための SQL 呼び出し可能関数です。

シグネチャまたは形状

Datum buffer_create_toy(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): 対象リレーションの OID。
  • PG_GETARG_UINT32(1): 対象のブロック番号。
  • **出力**:
  • PG_RETURN_INT32(buf): 生成されたバッファの識別子（Buffer 型）。

前提条件と不変条件

  • 対象リレーションに対して AccessExclusiveLock を取得します。
  • 関数終了時に ReleaseBuffer を呼び出し、バッファの参照カウントを減らします。
  • リレーションは関数終了時に relation_close されます。

アルゴリズムまたは振る舞い

1. 引数からリレーション OID とブロック番号を取得します。 2. relation_open を使用して、対象リレーションを AccessExclusiveLock モードで開きます。 3. create_toy_buffer を呼び出し、指定されたブロックに対するバッファを確保します。 4. ReleaseBuffer を呼び出してバッファを解放します。 5. relation_close を呼び出してリレーションを閉じます。 6. 確保したバッファ識別子を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されることを想定しています（PG_FUNCTION_INFO_V1 マクロによる登録あり）。
  • **呼び出し先**:
  • relation_open / relation_close
  • create_toy_buffer
  • ReleaseBuffer

関連データ構造

  • Relation (リレーション記述子)
  • Buffer (バッファ識別子)

失敗モードと境界条件

  • リレーションのオープン失敗や、create_toy_buffer 内部でのバッファ確保失敗時にエラーが発生する可能性があります。
  • AccessExclusiveLock を取得するため、対象リレーションに対する他の並行処理と競合する可能性があります。

保守担当者向けメモ

  • 本関数はテスト専用のヘルパーであり、本番環境のコードでは使用しないでください。
  • create_toy_buffer の実装詳細については、同ファイル内の該当シンボルを参照してください。
