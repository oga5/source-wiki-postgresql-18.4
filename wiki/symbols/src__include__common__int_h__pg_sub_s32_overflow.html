pg_sub_s32_overflow

定義

  • 種別: function
  • 定義箇所: src/include/common/int.h:168

目的

符号付き32ビット整数（int32）の減算において、オーバーフローを安全に検出し、演算結果を適切に処理する。

シグネチャまたは形状

static inline bool pg_sub_s32_overflow(int32 a, int32 b, int32 *result)

入力と出力

  • **入力**:
  • a: 被減数（int32）
  • b: 減数（int32）
  • result: 演算結果を格納するポインタ（int32 *）
  • **出力**:
  • 戻り値: オーバーフローが発生した場合は true、それ以外は false を返す。
  • *result: 演算が成功した場合は計算結果が格納される。オーバーフロー時はダミー値 0x5EED が書き込まれる。

前提条件と不変条件

  • int32 型の範囲内での演算を想定している。
  • コンパイラが HAVE__BUILTIN_OP_OVERFLOW をサポートしている場合は、組み込み関数を利用する。

アルゴリズムまたは振る舞い

1. HAVE__BUILTIN_OP_OVERFLOW が定義されている場合、コンパイラ組み込みの __builtin_sub_overflow を使用して演算と判定を行う。 2. 未定義の場合、int64 型へキャストして減算を行い、結果が PG_INT32_MAX を超えるか PG_INT32_MIN を下回るかを判定する。 3. 範囲外であれば *result に 0x5EED を代入し true を返す。 4. 範囲内であれば *result に計算結果をキャストして代入し false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_int4.c
  • src/backend/utils/adt/array_userfuncs.c
  • src/backend/utils/adt/arrayfuncs.c
  • src/backend/utils/adt/formatting.c
  • **呼び出し先**:
  • コンパイラ組み込み関数（__builtin_sub_overflow）または標準的な算術演算子。

関連データ構造

  • int32（標準的な符号付き32ビット整数型）

失敗モードと境界条件

  • a - b の結果が int32 の表現範囲（-2,147,483,648 ～ 2,147,483,647）を超える場合にオーバーフローとして検出される。

保守担当者向けメモ

  • 本関数はインライン関数であり、ヘッダーファイル内で定義されている。
  • オーバーフロー時のダミー値 0x5EED は、デバッグ時の警告を回避するためのものである。
