AssertCheckRanges

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/brin/brin_minmax_multi.c:295

目的

Ranges 構造体内のデータ整合性を検証するデバッグ用アサーション関数。USE_ASSERT_CHECKING が有効な場合にのみ実行され、範囲の順序、重なりの有無、および値の包含関係が正しく維持されているかをチェックする。

シグネチャまたは形状

static void AssertCheckRanges(Ranges *ranges, FmgrInfo *cmpFn, Oid colloid)

入力と出力

  • **入力**:
  • ranges: 検証対象の Ranges 構造体へのポインタ。
  • cmpFn: 値の比較に使用する関数情報。
  • colloid: 照合順序（Collation）ID。
  • **出力**: なし（検証失敗時は Assert によりプロセスが停止する）。

前提条件と不変条件

  • ranges->nranges >= 0
  • ranges->nsorted >= 0
  • ranges->nvalues >= ranges->nsorted
  • ranges->maxvalues >= 2 * ranges->nranges + ranges->nvalues
  • ranges->typid != InvalidOid
  • 範囲境界値およびソート済み値は、cmpFn に基づいて昇順にソートされていること。

アルゴリズムまたは振る舞い

1. **基本整合性チェック**: 構造体内の各カウンタ値が前提条件を満たしているか確認する。 2. **順序検証**: AssertArrayOrder を使用し、範囲境界値および単一ポイント値がソートされていることを保証する。 3. **包含関係検証**:

  • 各値が定義された範囲内に含まれていないことを確認する（範囲内に含まれる場合は Assert(false) を発行）。
  • 二分探索を用いて、値が既存の範囲のいずれかに該当するかを判定する。

    • 4. **重複排除検証**: ソート済み部分と未ソート部分で値が重複していないことを bsearch_arg を用いて確認する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_minmax_multi.c 内の複数箇所（インデックスの追加、マージ、シリアライズ処理の前後で呼び出される）。
  • **呼び出し先**:
  • Assert (マクロ)
  • AssertArrayOrder
  • FunctionCall2Coll
  • bsearch_arg

関連データ構造

  • Ranges: 本関数が検証する主要なデータ構造。
  • FmgrInfo: 比較関数を保持する構造体。

失敗モードと境界条件

  • USE_ASSERT_CHECKING が定義されていない場合、本関数は何も行わない。
  • 範囲の境界値がソートされていない場合、または値が範囲内に誤って含まれていると判定された場合に Assert 失敗となる。

保守担当者向けメモ

  • 本関数はデバッグ専用であるため、本番環境のパフォーマンスには影響しない。
  • 複雑なデータ型（UUID や INET 等）を扱う際の論理エラーを早期発見するために重要である。
  • 範囲の追加やマージロジックを変更した際は、必ず本関数が通過するパスで整合性が保たれているか確認すること。
