GetEpochTime

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/dt_common.c:948

目的

Unixエポック（1970-01-01 00:00:00 UTC）の日時情報を取得し、引数として渡された struct tm 構造体に格納する。

シグネチャまたは形状

int GetEpochTime(struct tm *tm)

入力と出力

  • **入力**: tm（日時情報を格納するための struct tm へのポインタ）
  • **出力**: 成功時は 0 を返し、tm にエポック時の日時（年、月、日、時、分、秒）を書き込む。失敗時は -1 を返す。

前提条件と不変条件

  • 引数 tm は有効なメモリ領域を指している必要がある。
  • tm->tm_year には 1970 が、tm->tm_mon には 1 が格納される（gmtime_r の結果にオフセットを加算するため）。

アルゴリズムまたは振る舞い

1. time_t 型の変数 epoch を 0 で初期化する。 2. gmtime_r を呼び出し、エポック時の日時を tmbuf に取得する。 3. gmtime_r が成功した場合、以下の変換を行い tm に代入する。

  • tm_year: t0->tm_year + 1900
  • tm_mon: t0->tm_mon + 1
  • tm_mday, tm_hour, tm_min, tm_sec: そのまま代入

    • 4. 成功時は 0 を返し、失敗時は -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/pgtypeslib/datetime.c:83
  • src/interfaces/ecpg/pgtypeslib/timestamp.c:73
  • ※ツールにより、バックエンド側の src/backend/utils/adt/ 配下からの参照も示唆されているが、本モジュール（ecpg）内での直接的な呼び出し関係は上記のとおり。
  • **呼び出し先**:
  • gmtime_r（標準ライブラリ関数）

関連データ構造

  • struct tm: C標準ライブラリの日時構造体。

失敗モードと境界条件

  • gmtime_r が NULL を返した場合、関数は -1 を返し、tm の内容は更新されない。

保守担当者向けメモ

  • 本関数は gmtime_r の結果に対して年（+1900）と月（+1）の補正を行っている。利用側で struct tm の標準的な仕様（年が1900年起点、月が0起点）を期待しているか確認すること。
