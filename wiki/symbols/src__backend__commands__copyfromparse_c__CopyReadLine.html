CopyReadLine

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfromparse.c:1157

目的

入力ストリームから1行分のデータを読み込み、cstate->line_buf に格納する。

シグネチャまたは形状

static bool CopyReadLine(CopyFromState cstate, bool is_csv)

入力と出力

  • **入力**:
  • cstate: CopyFromState 構造体（現在の解析状態を保持）。
  • is_csv: CSVモードかどうかを示すフラグ。
  • **出力**:
  • 戻り値: bool 型。EOF（ファイルの終わり）に到達した場合は true、それ以外は false を返す。

前提条件と不変条件

  • cstate->line_buf は初期化済みであること。
  • cstate->eol_type は有効な値（EOL_NL, EOL_CR, EOL_CRNL）に設定されていること。

アルゴリズムまたは振る舞い

1. cstate->line_buf をリセットし、line_buf_valid を false に設定する。 2. CopyReadLineText を呼び出し、入力データから行データを line_buf へ転送する。 3. EOFに到達した場合（戻り値が true）:

  • ソースが COPY_FRONTEND の場合、プロトコルバージョン3の仕様に従い、\. 以降のデータを読み捨ててバッファをクリアする。

    • 4. EOFではない場合:

  • cstate->eol_type に基づき、line_buf の末尾に含まれる改行コード（\n, \r, \r\n）を削除し、文字列を終端する。

    • 5. 最後に cstate->line_buf_valid を true に設定し、戻り値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfromparse.c 内のメインループ（行単位の処理箇所）。
  • **呼び出し先**: CopyReadLineText（内部的な行解析）、CopyGetData（フロントエンドからのデータ読み込み）、resetStringInfo（バッファ初期化）。

関連データ構造

  • CopyFromState: COPY FROM 操作の全状態を管理する構造体。
  • StringInfo: line_buf の型。動的文字列バッファ。

失敗モードと境界条件

  • EOL_UNKNOWN: eol_type が未設定の場合、Assert(false) により停止する。
  • line_buf の長さ不足: 改行コード削除時に Assert でバッファ長を検証しており、期待される長さがない場合は異常終了する。

保守担当者向けメモ

  • CopyReadLineText が行末までを読み込む役割を担い、本関数はその後の後処理（改行除去やEOF処理）を担当する。
  • cstate->line_buf_valid は、エラー発生時にバッファの内容を安全に参照できることを示すフラグとして機能する。
