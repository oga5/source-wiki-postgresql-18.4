tidle

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tid.c:202

目的

2つの TID (Tuple Identifier) を比較し、第1引数が第2引数以下であるか（<=）を判定します。

シグネチャまたは形状

Datum tidle(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの ItemPointer（TID の内部表現）を受け取ります。
  • **出力**: bool 型の Datum。第1引数が第2引数以下であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力される ItemPointer は有効なポインタである必要があります。
  • ItemPointerCompare の仕様に基づき、ブロック番号およびオフセット番号の順序で比較が行われます。

アルゴリズムまたは振る舞い

1. PG_GETARG_ITEMPOINTER マクロを使用して、引数から2つの ItemPointer を取得します。 2. ItemPointerCompare 関数を呼び出し、両者の大小関係を評価します。 3. 比較結果が 0 以下（等しいか、第1引数の方が小さい）である場合に true を返し、それ以外の場合に false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL レベルの演算子呼び出しから間接的に呼び出されます（直接的な呼び出し元情報はソースコード上では明示されていません）。
  • **呼び出し先**: ItemPointerCompare を呼び出します。

関連データ構造

  • ItemPointer: TID の内部表現。BlockIdData（ブロック番号）と OffsetNumber（オフセット番号）で構成されます。

失敗モードと境界条件

  • ItemPointerCompare の挙動に依存します。無効な ItemPointer が渡された場合の動作は定義されていません。

保守担当者向けメモ

  • 本関数は TID 型の比較演算子の一部です。比較ロジックを変更する場合は、ItemPointerCompare の実装を確認してください。
