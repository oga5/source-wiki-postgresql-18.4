ReinitializeParallelWorkers

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/parallel.c:564

目的

並列クエリ実行において、起動するワーカー数を再設定する。

シグネチャまたは形状

void ReinitializeParallelWorkers(ParallelContext *pcxt, int nworkers_to_launch)

入力と出力

  • pcxt: 操作対象の ParallelContext 構造体へのポインタ。
  • nworkers_to_launch: 起動を試みるワーカー数。
  • 出力: なし（pcxt->nworkers_to_launch が更新される）。

前提条件と不変条件

  • pcxt は初期化済みであること。
  • 設定されるワーカー数は、pcxt の初期化時に定義された最大ワーカー数（pcxt->nworkers）を超えないこと。

アルゴリズムまたは振る舞い

1. 引数 nworkers_to_launch と pcxt->nworkers を比較する。 2. Min マクロを用いて、小さい方の値を pcxt->nworkers_to_launch に代入する。 3. InitializeParallelDSM 等でワーカー数が削減されている可能性があるため、呼び出し側が正確な上限を知らなくても安全に値をトリミングする。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/commands/vacuumparallel.c
  • 呼び出し先:
  • 情報なし（標準ライブラリの Min マクロのみ使用）。

関連データ構造

  • ParallelContext: 並列実行のコンテキストを保持する構造体。

失敗モードと境界条件

  • nworkers_to_launch に負の数や過大な値が渡された場合でも、Min マクロにより pcxt->nworkers の範囲内に収まるよう調整される。

保守担当者向けメモ

  • 本関数は、ワーカー起動の直前に動的なワーカー数の調整が必要な場合に使用される。
  • pcxt->nworkers は ParallelContext 作成時に決定される固定値であるため、本関数による変更は nworkers_to_launch に対してのみ行われる。
