build_join_rel

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/relnode.c:664

目的

2つのリレーション（outer_rel と inner_rel）を結合する新しい結合リレーションノード（RelOptInfo）を構築、または既存のノードを検索して返します。

シグネチャまたは形状

RelOptInfo *build_join_rel(PlannerInfo *root,
                           Relids joinrelids,
                           RelOptInfo *outer_rel,
                           RelOptInfo *inner_rel,
                           SpecialJoinInfo *sjinfo,
                           List *pushed_down_joins,
                           List **restrictlist_ptr)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • joinrelids: 結合対象となるリレーションの集合（Bitmapset）。
  • outer_rel, inner_rel: 結合する2つのリレーションノード。
  • sjinfo: 結合の種類や制約を含む情報。
  • pushed_down_joins: プッシュダウンされた結合のリスト。
  • restrictlist_ptr: 結合条件リストを格納するためのポインタ（任意）。
  • **出力**: 構築または検索された RelOptInfo 構造体へのポインタ。

前提条件と不変条件

  • outer_rel および inner_rel は、IS_OTHER_REL マクロが真となるような「他のリレーション（子リレーション等）」であってはならない。
  • 結合リレーションのターゲットリスト（tlist）の内容は、構築順序に関わらず一貫性が保たれる必要がある。

アルゴリズムまたは振る舞い

1. **検索**: find_join_rel を使用し、指定された joinrelids に対応する結合リレーションが既に存在するか確認する。存在する場合はそのノードを返し、必要に応じて restrictlist を計算して出力する。 2. **新規作成**: 存在しない場合、makeNode(RelOptInfo) で新規作成し、RELOPT_JOINREL として初期化する。 3. **プロパティ設定**:

  • lateral_relids を min_join_parameterization を用いて計算。
  • set_foreign_rel_properties を呼び出し、外部リレーション関連のプロパティを設定。
  • build_joinrel_tlist および add_placeholders_to_joinrel を用いて、ターゲットリストとプレースホルダー変数を構築。

    • 4. **制約と見積もり**:

  • build_joinrel_restrictlist および build_joinrel_joinlist で結合条件を構築。
  • has_relevant_eclass_joinclause で等価クラス結合の有無を確認。
  • build_joinrel_partition_info でパーティション情報を設定。
  • set_joinrel_size_estimates で行数やコストの見積もりを実行。

    • 5. **並列実行フラグ**: is_parallel_safe を用いて、結合リレーションが並列ワーカー内でスキャン可能かを判定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/joinrels.c
  • **呼び出し先**:
  • find_join_rel, makeNode, build_joinrel_restrictlist, build_joinrel_tlist, add_placeholders_to_joinrel, set_foreign_rel_properties, build_joinrel_joinlist, has_relevant_eclass_joinclause, build_joinrel_partition_info, set_joinrel_size_estimates, is_parallel_safe 等。

関連データ構造

  • RelOptInfo: 結合リレーションの基本構造体。
  • PlannerInfo: プランナの全体状態。
  • SpecialJoinInfo: 結合の特性（JOIN_FULL等）を保持。

失敗モードと境界条件

  • 既に存在する結合リレーションに対しては、再構築を行わず既存のノードを再利用する。
  • restrictlist_ptr が NULL の場合、結合条件リストの出力はスキップされる。
  • 並列実行の安全性判定において、inner_rel または outer_rel のいずれかが並列非対応であれば、結合リレーションも並列非対応となる。

保守担当者向けメモ

  • 本関数は RelOptInfo の多くのフィールドを初期化するため、構造体の定義変更時には注意が必要。
  • パーティションワイズ結合（Partitionwise Join）のロジックが組み込まれており、consider_partitionwise_join フラグの扱いに注意すること。
  • ターゲットリストの構築順序は呼び出し元に依存する可能性があるが、内容は一貫性を保つ設計となっている。
