makeDefaultBloomOptions

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blutils.c:83

目的

Bloom インデックスのデフォルト設定を保持する BloomOptions 構造体をメモリ上に生成し、初期化します。

シグネチャまたは形状

static BloomOptions *makeDefaultBloomOptions(void)

入力と出力

  • **入力**: なし（void）
  • **出力**: BloomOptions 構造体へのポインタ。メモリは palloc0 によって確保されます。

前提条件と不変条件

  • DEFAULT_BLOOM_LENGTH および DEFAULT_BLOOM_BITS が定義されている必要があります。
  • INDEX_MAX_KEYS までの各キーに対して bitSize が設定されます。
  • 返される構造体は SET_VARSIZE により適切な可変長サイズが設定されています。

アルゴリズムまたは振る舞い

1. palloc0 を使用して BloomOptions サイズのメモリをゼロクリアして確保します。 2. DEFAULT_BLOOM_LENGTH（ビット単位）を SIGNWORDBITS で除算し、bloomLength（ワード単位）を算出します。 3. INDEX_MAX_KEYS のループ内で、各キーの bitSize に DEFAULT_BLOOM_BITS を代入します。 4. SET_VARSIZE を呼び出し、構造体のサイズをメタデータとして記録します。 5. 初期化済みのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blutils.c:430（インデックスのオプション設定処理内）
  • **呼び出し先**:
  • palloc0 (PostgreSQL メモリ管理)
  • SET_VARSIZE (PostgreSQL マクロ)

関連データ構造

  • BloomOptions: Bloom インデックスのパラメータ（署名長、各属性のビット数）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • SIGNWORDBITS の値が 0 または負の場合、除算で例外が発生する可能性があります（通常は定数定義のため発生しません）。

保守担当者向けメモ

  • DEFAULT_BLOOM_LENGTH を変更した際は、この関数内の計算ロジックと contrib/bloom/blutils.c:46 のコメントを同期させる必要があります。
