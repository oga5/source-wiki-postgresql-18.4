sync_queue_push

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/relfilenumber.c:74

目的

pg_upgrade の --swap モードにおいて、カタログファイルの永続化（fsync）を効率的に行うため、対象ファイル名をキューにバッファリングします。

シグネチャまたは形状

static inline void sync_queue_push(const char *fname)

入力と出力

  • **入力**: fname (同期対象のファイルパスを示す文字列)
  • **出力**: なし（副作用としてキューへの追加および必要に応じた同期実行）

前提条件と不変条件

  • sync_queue_init() が呼び出され、キューが初期化されている必要があります（本関数内で自動的に呼び出されます）。
  • sync_queue_len が SYNC_QUEUE_MAX_LEN を超えないよう管理されます。

アルゴリズムまたは振る舞い

1. sync_queue_init() を呼び出し、キューの初期化状態を保証します。 2. pre_sync_fname(fname, false) を実行し、同期前の準備処理を行います。 3. 指定された fname を sync_queue 配列の末尾に追加し、sync_queue_len をインクリメントします。 4. キューの長さが SYNC_QUEUE_MAX_LEN に達した場合、sync_queue_sync_all() を呼び出して保留中の全ファイルを同期し、キューをフラッシュします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/relfilenumber.c 内の複数箇所（ファイル転送処理の過程で呼び出されます）。
  • **呼び出し先**:
  • sync_queue_init()
  • pre_sync_fname()
  • sync_queue_sync_all()

関連データ構造

  • sync_queue: 同期待ちファイル名を保持するグローバルな配列。
  • sync_queue_len: 現在キューに格納されているファイル数。
  • SYNC_QUEUE_MAX_LEN: キューの最大容量。

失敗モードと境界条件

  • キューが満杯（SYNC_QUEUE_MAX_LEN）に達した際、自動的に sync_queue_sync_all() が実行されるため、バッファオーバーフローは発生しません。

保守担当者向けメモ

  • 本関数は inline 指定されており、パフォーマンスを重視した設計です。
  • swap モードにおける同期の遅延実行を担うため、本関数を呼び出した後は、処理の完了時に必ず sync_queue_sync_all() が呼び出される設計になっていることを確認してください。
