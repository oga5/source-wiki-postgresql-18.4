test_nosync

定義

  • 種別: function
  • 定義箇所: src/test/modules/libpq_pipeline/libpq_pipeline.c:612

目的

パイプラインモードにおいて、PQpipelineSync を明示的に呼び出さずに複数のクエリを送信し、その後の結果取得および同期処理が正しく行われるかを検証するテスト関数です。

シグネチャまたは形状

static void test_nosync(PGconn *conn)

入力と出力

  • 入力: PGconn *conn（確立済みのデータベース接続）
  • 出力: なし（成功時は標準エラー出力に "ok" を出力し、失敗時は pg_fatal によりプロセスを終了）

前提条件と不変条件

  • conn は有効な接続状態であること。
  • 実行中に PQenterPipelineMode が成功していること。
  • 送信したクエリ数（10回）と、受信した PGRES_TUPLES_OK の数が一致すること。

アルゴリズムまたは振る舞い

1. パイプラインモードへ移行します。 2. 10回ループし、SELECT クエリを送信して PQflush を呼び出します。各ループ内で select() を使用し、サーバーからの応答があれば PQconsumeInput で読み込みます。 3. 全クエリ送信後、PQsendFlushRequest を呼び出してサーバー側の出力バッファをフラッシュさせます。 4. PQgetResult をループで呼び出し、結果を消費します。各クエリに対して PGRES_TUPLES_OK が返されること、およびその直後に NULL が返されることを確認します。 5. 全ての結果を処理し終えたら終了します。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/libpq_pipeline/libpq_pipeline.c:2364（main 関数内と推測される箇所）
  • 呼び出し先: PQsocket, pg_fatal, PQenterPipelineMode, PQsendQueryParams, PQflush, select, FD_ZERO, FD_SET, FD_ISSET, PQconsumeInput, PQsendFlushRequest, PQgetResult, PQresultStatus, PQclear, PQresStatus, exit_nicely

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • PGresult: クエリ実行結果を保持する構造体。
  • fd_set: select() で使用するファイル記述子セット。

失敗モードと境界条件

  • ソケットが無効な場合、またはパイプラインモードへの移行に失敗した場合は異常終了します。
  • select() がエラーを返した場合、exit_nicely を呼び出して終了します。
  • サーバーからの応答読み込みに失敗した場合、または期待される結果（PGRES_TUPLES_OK や NULL）が得られない場合は pg_fatal で終了します。

保守担当者向けメモ

  • 本関数は PQpipelineSync を使用しないケースをテストしています。パイプラインモードの仕様変更や、libpq の結果取得ロジックの変更時に、この非同期的な結果消費フローが正しく維持されているかを確認してください。
