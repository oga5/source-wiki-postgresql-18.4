intarray_add_elem

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_tool.c:353

目的

既存の整数配列（ArrayType）に新しい要素を追加し、その結果を新しい配列として返します。

シグネチャまたは形状

ArrayType *intarray_add_elem(ArrayType *a, int32 elem)

入力と出力

  • **入力**:
  • a: 操作対象の ArrayType ポインタ。
  • elem: 追加する int32 型の要素。
  • **出力**:
  • 要素が追加された新しい ArrayType 構造体へのポインタ。

前提条件と不変条件

  • CHECKARRVALID(a) マクロにより、入力配列 a が有効な整数配列であることが検証されます。
  • 返される配列は new_intArrayType によって動的に確保されるため、呼び出し側でメモリ管理（解放）を行う必要があります。

アルゴリズムまたは振る舞い

1. 入力配列 a の要素数 c を取得します。 2. new_intArrayType(c + 1) を呼び出し、要素数 c + 1 を保持できる新しい配列領域を確保します。 3. c > 0 の場合、元の配列 a の内容を新しい配列の先頭にコピーします。 4. 新しい配列の末尾（インデックス c）に elem を格納します。 5. 新しい配列ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_op.c:331
  • contrib/intarray/_int_op.c:385
  • **呼び出し先**:
  • new_intArrayType (メモリ確保)
  • CHECKARRVALID (マクロによる検証)
  • ARRNELEMS (マクロによる要素数取得)
  • ARRPTR (マクロによるデータポインタ取得)
  • memcpy (標準ライブラリ)

関連データ構造

  • ArrayType: PostgreSQL の標準的な配列構造体。

失敗モードと境界条件

  • 入力配列 a が無効な場合、CHECKARRVALID によりエラーが発生する可能性があります。
  • メモリ不足時には new_intArrayType 内部でエラーが発生します。
  • 空の配列（c == 0）に対しても正しく動作し、要素1つの配列を生成します。

保守担当者向けメモ

  • 本関数は要素のソート順序を維持しません。ソートされた状態が必要な場合は、呼び出し側で別途ソート処理を行う必要があります。
  • memcpy を使用しているため、要素の追加に伴うメモリコピーが発生します。大規模な配列に対して頻繁に呼び出す場合はパフォーマンスに注意してください。
