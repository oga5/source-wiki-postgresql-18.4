目的

Informix 互換モードにおける ECPG の NULL 値処理（risnull および rsetnull）の正当性を検証する回帰テストのエントリポイントです。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は使用しません）。
  • 出力: 成功時に 0 を返します。テストの進行状況や結果は標準出力および標準エラー出力に書き出されます。

前提条件と不変条件

  • Informix 互換モードでコンパイルされている必要があります。
  • ecpg1_regression データベースへの接続が可能である必要があります。
  • 実行環境において test テーブルの作成および削除権限が必要です。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、ECPGconnect でデータベースに接続します。 2. **テーブル構築**: create table test を実行し、各種データ型（char, short, int, bool, float, long, double, decimal, date, timestamp）を含むテーブルを作成します。 3. **データ挿入（通常）**: 初期値をテーブルに挿入し、コミットします。 4. **NULL 値設定**: rsetnull を使用して、各 C 変数を Informix 互換の NULL 状態に設定します。 5. **データ挿入（NULL）**: NULL 値が設定された変数をテーブルに挿入し、コミットします。 6. **検証**: select 文でデータを取得し、test_null 関数を用いて各フィールドが期待通り NULL であるかを判定します。 7. **終了**: データベース接続を終了します。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）。
  • 呼び出し先: ECPGconnect, ECPGdo, ECPGtrans, rsetnull, test_null, sqlprint, printf。

関連データ構造

  • sqlca: ECPG のエラーハンドリングに使用される構造体。
  • decimal, date, timestamp: Informix 互換のデータ型。

失敗モードと境界条件

  • データベース接続失敗時、または SQL 実行エラー時に sqlprint() が呼び出され、エラー情報が出力されます。
  • rsetnull による NULL 設定が正しく反映されない場合、後続の test_null による検証で不一致が検出されます。

保守担当者向けメモ

  • 本関数は rnull.pgc から生成されたコードです。ロジックの変更が必要な場合は、元の .pgc ファイルを修正し、ECPG プリプロセッサを再実行してください。
  • risnull および rsetnull は Informix 互換モード専用の関数であるため、標準的な ECPG の動作とは異なる点に注意してください。
