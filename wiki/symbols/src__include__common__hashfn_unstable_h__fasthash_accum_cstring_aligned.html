fasthash_accum_cstring_aligned

定義

  • **種別**: function
  • **定義箇所**: src/include/common/hashfn_unstable.h:252

目的

NUL終端文字列をワード単位（64bit）で効率的に読み込み、ハッシュ状態へ蓄積する。

シグネチャまたは形状

static inline size_t fasthash_accum_cstring_aligned(fasthash_state *hs, const char *str)

入力と出力

  • **入力**:
  • hs: ハッシュ計算の状態を保持する fasthash_state 構造体へのポインタ。
  • str: ハッシュ化対象のNUL終端文字列。
  • **出力**: 処理した文字列の長さ（バイト数）を size_t で返す。

前提条件と不変条件

  • str は uint64 型の境界にアライメントされている必要がある（Assert(PointerIsAligned(start, uint64)) により検証）。

アルゴリズムまたは振る舞い

1. 文字列を uint64 単位でループ処理する。 2. 各チャンクに対して haszero64() を実行し、NULバイトが含まれているか判定する。 3. NULバイトが見つからない場合、そのチャンクを hs->accum にロードし、fasthash_combine(hs) を呼び出してハッシュを更新する。 4. NULバイトが含まれるチャンクに到達した時点でループを終了する。 5. 最後に fasthash_accum_cstring_unaligned() を呼び出し、残りのバイトを処理して全体の長さを算出・返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/common/hashfn_unstable.h 内の fasthash_accum_cstring から呼び出される。
  • **呼び出し先**: fasthash_combine、fasthash_accum_cstring_unaligned、haszero64。

関連データ構造

  • fasthash_state: ハッシュ計算の進行状況を保持する構造体。

失敗モードと境界条件

  • アライメントされていないポインタが渡された場合、Assert が有効なビルドではアボートする。

保守担当者向けメモ

  • 本関数はアドレスサニタイザー（ASan）の誤検知を避けるため、pg_attribute_no_sanitize_address() で保護されている。
  • プラットフォーム依存のエンディアンの影響を受けるため、永続化データには使用しないこと。
