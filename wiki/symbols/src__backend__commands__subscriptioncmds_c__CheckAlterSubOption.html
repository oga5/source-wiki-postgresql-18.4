CheckAlterSubOption

定義

  • 種別: function
  • 定義箇所: src/backend/commands/subscriptioncmds.c:1045

目的

論理レプリケーションのサブスクリプション変更時における failover および two_phase オプションの適用可否を検証し、制約を強制する。

シグネチャまたは形状

static void CheckAlterSubOption(Subscription *sub, const char *option, bool slot_needs_update, bool isTopLevel)

入力と出力

  • sub: 検証対象のサブスクリプション構造体。
  • option: 検証するオプション名（"failover" または "two_phase"）。
  • slot_needs_update: スロットのプロパティ更新が必要かどうかを示すフラグ。
  • isTopLevel: トランザクションブロック内での実行を判定するためのフラグ。
  • 戻り値: なし。検証失敗時は ereport(ERROR, ...) により中断。

前提条件と不変条件

  • option 引数は "failover" または "two_phase" であること（Assert で検証）。
  • サブスクリプションが有効（sub->enabled が真）な場合、これらのオプション変更は許可されない。

アルゴリズムまたは振る舞い

1. **有効状態のチェック**: サブスクリプションが有効な場合、既存の walsender がスロットを保持している可能性があるため、エラーを発生させる。 2. **スロット更新の検証**: slot_needs_update が真の場合、以下の処理を行う。

  • サブスクリプションに slotname が設定されているか確認し、未設定であればエラーとする。
  • PreventInTransactionBlock を呼び出し、トランザクションブロック内での実行を禁止する（変更がロールバックできないため）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/subscriptioncmds.c 内の AlterSubscription 関数から、two_phase および failover オプションの更新処理時に呼び出される。
  • **呼び出し先**: Assert, ereport, initStringInfo, appendStringInfo, PreventInTransactionBlock, pfree。

関連データ構造

  • Subscription: サブスクリプション情報を保持する構造体。

失敗モードと境界条件

  • **ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE**:
  • サブスクリプションが有効な状態でオプションを変更しようとした場合。
  • スロット名が未設定の状態でスロットプロパティを更新しようとした場合。
  • **トランザクション制約**: トランザクションブロック内で実行された場合、PreventInTransactionBlock によりエラーが発生する。

保守担当者向けメモ

  • 本関数は failover および two_phase オプション専用のバリデーターです。将来的に他のオプションで同様の制約が必要になった場合は、Assert 文の拡張を検討してください。
  • two_phase は既存の walsender によって有効化されるケースがありますが、一貫性と単純性を維持するため、有効なサブスクリプションでの変更は一律禁止しています。
