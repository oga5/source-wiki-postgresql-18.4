xl_multi_insert_tuple

定義

  • 種別: struct
  • 定義箇所: src/include/access/heapam_xlog.h:190

目的

heap_multi_insert 操作において、WALレコード内に格納される個々のタプルのメタデータとデータを保持するために使用されます。

シグネチャまたは形状

typedef struct xl_multi_insert_tuple
{
    uint16      datalen;        /* 後続するタプルデータのサイズ */
    uint16      t_infomask2;    /* タプルのinfomask2 */
    uint16      t_infomask;     /* タプルのinfomask */
    uint8       t_hoff;         /* タプルヘッダーのオフセット */
    /* タプルデータは構造体の末尾に続く */
} xl_multi_insert_tuple;

入力と出力

  • **入力**: heap_multi_insert 実行時に生成されるタプルの属性情報および実データ。
  • **出力**: WALレコードの一部としてディスクに書き込まれ、リカバリ時に heap_redo 等で読み取られる。

前提条件と不変条件

  • 構造体の直後に実際のタプルデータが配置される可変長構造をとる。
  • 構造体へのアクセスには SHORTALIGN を用いたアライメント調整が必要である（heapam.c および heapam_xlog.c での利用実績に基づく）。

アルゴリズムまたは振る舞い

1. **書き込み**: heap_multi_insert が複数のタプルを挿入する際、各タプルに対して本構造体をヘッダーとして付与し、WALレコードのデータ領域に順次書き込む。 2. **読み込み**: リカバリ時、WALレコードをデコードする過程で、本構造体を読み取ることで後続するタプルデータのサイズ（datalen）を特定し、次のタプルエントリへのポインタを計算する。

呼び出し元と呼び出し先

  • **呼び出し元（生成）**: src/backend/access/heap/heapam.c (heap_multi_insert)
  • **呼び出し先（解析）**:
  • src/backend/access/heap/heapam_xlog.c (REDO処理)
  • src/backend/replication/logical/decode.c (論理デコード)

関連データ構造

  • xl_heap_multi_insert: xl_multi_insert_tuple を複数含むWALレコード全体の構造体。

失敗モードと境界条件

  • datalen の不整合: WALレコードのデータ長と実際のデータサイズが一致しない場合、リカバリ処理がメモリ破壊や不正なポインタ参照を引き起こす可能性がある。
  • アライメント: SHORTALIGN を使用しない場合、プラットフォーム固有の境界整列違反が発生する可能性がある。

保守担当者向けメモ

  • 本構造体は可変長データを含むため、ポインタ演算を行う際は必ず datalen を参照して次のエントリ位置を算出すること。
  • src/include/access/heapam_xlog.h 内のコメントにある通り、ブロック0のデータ領域に本構造体が配置される設計となっている。
