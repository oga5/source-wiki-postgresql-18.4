box_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_ops.c:465

目的

外部バイナリ形式のデータストリームから BOX 型のデータを読み込み、内部表現へ変換します。

シグネチャまたは形状

Datum box_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される StringInfo 型のバイナリバッファ。
  • **出力**: 変換された BOX 構造体へのポインタを Datum として返します。

前提条件と不変条件

  • 入力バッファには、float8 型の座標値が 4 つ（high.x, high.y, low.x, low.y の順）格納されている必要があります。
  • 戻り値の BOX 構造体は palloc によってメモリ確保されます。
  • 座標値の正規化が行われ、high 座標が常に low 座標以上となることが保証されます。

アルゴリズムまたは振る舞い

1. palloc を使用して BOX 構造体用のメモリを割り当てます。 2. pq_getmsgfloat8 を 4 回呼び出し、バッファから順に high.x, high.y, low.x, low.y を読み取ります。 3. float8_lt を用いて座標の大小関係を比較し、high が low より小さい場合は値を入れ替えて正規化します。 4. 最終的に構築された BOX 構造体を PG_RETURN_BOX_P マクロで返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のバイナリ入出力インターフェース（詳細な呼び出し元情報はソースコード内に明示されていません）。
  • **呼び出し先**: palloc, pq_getmsgfloat8, float8_lt, PG_RETURN_BOX_P。

関連データ構造

  • BOX: 2つの Point（high と low）を保持する幾何データ型。
  • StringInfo: バイナリデータストリームを保持する構造体。

失敗モードと境界条件

  • 入力バッファが不足している場合、pq_getmsgfloat8 がエラーを発生させる可能性があります。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数はバイナリ形式の入力のみを扱います。テキスト形式の入力については box_in を参照してください。
  • 座標の入れ替え処理は、入力データが必ずしも正規化された順序で提供されないことを前提としています。
