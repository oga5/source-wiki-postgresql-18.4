acquireLocksOnSubLinks_context

定義

  • 種別: struct
  • 定義箇所: src/backend/rewrite/rewriteHandler.c:56

目的

クエリツリー内のサブリンク（SubLink）を走査し、リレーションのロックを確保する処理において、実行モード（for_execute）の状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct acquireLocksOnSubLinks_context
{
	bool		for_execute;	/* AcquireRewriteLocks' forExecute param */
} acquireLocksOnSubLinks_context;

入力と出力

  • **入力**: for_execute（bool型）。AcquireRewriteLocks 関数から渡されるパラメータを格納します。
  • **出力**: なし（本構造体は状態保持用であり、関数呼び出しの引数として渡されます）。

前提条件と不変条件

  • for_execute は、クエリが実行目的であるか否かを示します。
  • 本構造体は、acquireLocksOnSubLinks 関数による再帰的なツリー走査の間、一貫して保持される必要があります。

アルゴリズムまたは振る舞い

  • AcquireRewriteLocks などの関数において、ローカル変数としてインスタンス化されます。
  • acquireLocksOnSubLinks 関数にポインタとして渡され、走査中の各ノードでロック確保の要否を判断するためのフラグとして参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • AcquireRewriteLocks (src/backend/rewrite/rewriteHandler.c:153)
  • その他、acquireLocksOnSubLinks を呼び出す関数群（行 362, 2261, 2362, 3515）
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • AcquireRewriteLocks: この構造体を利用してロック確保処理を行う主要関数。
  • acquireLocksOnSubLinks: この構造体を引数として受け取り、ツリーを走査する関数。

失敗モードと境界条件

  • 構造体自体に失敗モードはありません。
  • 走査対象のノードが NULL の場合、acquireLocksOnSubLinks 側で適切に処理されることが前提となります。

保守担当者向けメモ

  • for_execute フラグは、ロックの強さや範囲を決定する重要なパラメータです。この構造体を拡張する場合は、acquireLocksOnSubLinks のシグネチャおよび内部ロジックとの整合性に注意してください。
