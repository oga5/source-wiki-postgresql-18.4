flagInhIndexes

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/common.c:386

目的

パーティションテーブルのインデックスと、その親インデックスを紐付けるための DO_INDEX_ATTACH オブジェクトを生成し、ダンプ処理における依存関係を構築する。

シグネチャまたは形状

static void flagInhIndexes(Archive *fout, TableInfo tblinfo[], int numTables)

入力と出力

  • **入力**:
  • fout: アーカイブハンドル（使用箇所なし）。
  • tblinfo[]: ダンプ対象のテーブル情報の配列。
  • numTables: tblinfo 配列の要素数。
  • **出力**: なし（副作用として TableInfo 内の partattaches リストおよびグローバルなダンプ管理構造を更新）。

前提条件と不変条件

  • tblinfo 配列内の各テーブルは、パーティションである場合に numParents == 1 であること（Assert により検証）。
  • findIndexByOid が親インデックスを正しく解決できること。

アルゴリズムまたは振る舞い

1. tblinfo を走査し、パーティションかつ親を持つテーブルを特定する。 2. 各テーブルのインデックスを走査し、親インデックスを持つものに対して以下を実行する。

  • IndexAttachInfo オブジェクトをメモリ確保し、AssignDumpId でダンプIDを割り当てる。
  • パーティションインデックスと親インデックスの情報を格納する。
  • addObjectDependency を使用し、以下のオブジェクトに対する依存関係を明示的に追加する。
  • パーティションインデックス自身
  • 親インデックス
  • パーティションインデックスが属するテーブル
  • 親インデックスが属するテーブル
  • 親インデックスの partattaches リストに生成したオブジェクトを追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/common.c 内の getSchemaData（推測される文脈）。
  • **呼び出し先**:
  • findIndexByOid
  • pg_malloc_object
  • AssignDumpId
  • addObjectDependency
  • simple_ptr_list_append

関連データ構造

  • TableInfo: テーブルのメタデータとインデックスリストを保持。
  • IndxInfo: インデックスのメタデータ。
  • IndexAttachInfo: インデックスの継承関係を表現するダンプオブジェクト。

失敗モードと境界条件

  • findIndexByOid が NULL を返す場合、そのインデックスはスキップされる。
  • パーティションでないテーブルや親を持たないテーブルは処理対象外となる。

保守担当者向けメモ

  • 本関数は pg_depend に存在しない依存関係を補完するために DO_INDEX_ATTACH を生成している。
  • addObjectDependency による依存関係の追加は、並列リストア時にインデックスのアタッチ操作が他のテーブル操作と競合しないようにするための制御を含んでいる。
