LargeObjectDrop

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_largeobject.c:94

目的

指定された OID を持つラージオブジェクトのメタデータおよび関連するすべてのデータページをシステムカタログから物理的に削除します。

シグネチャまたは形状

void LargeObjectDrop(Oid loid)

入力と出力

  • **入力**: loid (削除対象のラージオブジェクトの OID)
  • **出力**: なし（成功時はカタログから削除、失敗時はエラーをスロー）

前提条件と不変条件

  • 呼び出し元は適切なトランザクションコンテキストにある必要があります。
  • pg_largeobject_metadata および pg_largeobject テーブルに対する RowExclusiveLock が取得されます。
  • 削除対象の OID が存在しない場合、エラーが発生します。

アルゴリズムまたは振る舞い

1. pg_largeobject_metadata および pg_largeobject テーブルを RowExclusiveLock モードで開きます。 2. pg_largeobject_metadata を走査し、指定された loid に一致するエントリを検索します。

  • エントリが見つからない場合は ERRCODE_UNDEFINED_OBJECT エラーを発生させます。
  • 見つかった場合は CatalogTupleDelete を使用してメタデータを削除します。

    • 3. pg_largeobject テーブルを走査し、同じ loid を持つすべてのデータページ（タプル）を検索します。 4. 該当するすべてのデータページを CatalogTupleDelete で順次削除します。 5. 両方のテーブルを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/dependency.c:1404
  • **呼び出し先**:
  • table_open, table_close
  • systable_beginscan, systable_getnext, systable_endscan
  • CatalogTupleDelete
  • ScanKeyInit

関連データ構造

  • pg_largeobject_metadata: ラージオブジェクトのメタデータ管理用カタログ。
  • pg_largeobject: ラージオブジェクトのデータページ格納用カタログ。
  • ScanKeyData: テーブル走査時の検索条件保持用。

失敗モードと境界条件

  • 指定した loid が存在しない場合、ereport(ERROR, ...) により処理が中断されます。
  • システムカタログのロック取得や走査中にエラーが発生した場合、トランザクションのロールバックに従います。

保守担当者向けメモ

  • 本関数はシステムキャッシュを使用せず、カタログテーブルを直接走査します。
  • メタデータの削除を先に行い、その後にデータページを削除する順序が保証されています。
  • 頻繁な呼び出しはカタログへの直接アクセスを伴うため、パフォーマンスに影響を与える可能性があります。
