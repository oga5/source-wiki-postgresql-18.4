pg_atomic_write_u32_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/generic.h:54

目的

pg_atomic_uint32 型の変数に対して、値を直接代入するアトミック操作のフォールバック実装を提供します。

シグネチャまたは形状

static inline void pg_atomic_write_u32_impl(volatile pg_atomic_uint32 *ptr, uint32 val)

入力と出力

  • **入力**:
  • ptr: 書き込み対象となる volatile pg_atomic_uint32 構造体へのポインタ。
  • val: 書き込む uint32 型の値。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • ptr は有効なメモリ領域を指している必要があります。
  • 本関数は atomics.h を経由して呼び出されることを前提としています。

アルゴリズムまたは振る舞い

ptr が指す構造体の value メンバに対して、引数 val を直接代入します。本実装はアトミック操作の汎用的なフォールバック層であり、アーキテクチャ固有の最適化が提供されていない環境で使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/port/atomics.h:280
  • src/include/port/atomics/generic.h 内の各所（初期化処理等で 0 を書き込む際に使用）。
  • **呼び出し先**:
  • 本関数は直接的な呼び出し先を持ちません。

関連データ構造

  • pg_atomic_uint32: アトミック操作対象となる 32 ビット整数を保持する構造体。

失敗モードと境界条件

  • 本関数自体は単純な代入操作であるため、関数内部での失敗モードはありません。
  • ただし、本関数はメモリバリアを明示的に発行しないため、メモリの可視性が重要な並行処理環境では、呼び出し側で適切なバリア処理がなされている必要があります。

保守担当者向けメモ

  • 本関数は generic.h 内のフォールバック実装です。特定のアーキテクチャでより効率的な命令（アトミックストア命令等）が利用可能な場合は、arch-*.h 等でオーバーライドされているか確認してください。
  • volatile 修飾子によりコンパイラによる最適化（レジスタへのキャッシュ等）を抑制していますが、ハードウェアレベルのメモリ順序付けについては保証されません。
