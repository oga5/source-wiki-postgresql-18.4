IndexBulkDeleteResult

定義

  • 種別: struct
  • 定義箇所: src/include/access/genam.h:98

目的

インデックスの VACUUM 操作（不要領域の回収）の結果を保持し、呼び出し元へ統計情報を報告するために使用されます。

シグネチャまたは形状

typedef struct IndexBulkDeleteResult
{
	BlockNumber num_pages;          /* インデックス内の残存ページ数 */
	bool        estimated_count;    /* num_index_tuples が推定値かどうか */
	double      num_index_tuples;   /* 残存タプル数 */
	double      tuples_removed;     /* VACUUM 操作で削除されたタプル数 */
	BlockNumber pages_newly_deleted;/* 今回削除マークが付与されたページ数 */
	BlockNumber pages_deleted;      /* 削除マークが付与されているページ数（合計） */
	BlockNumber pages_free;         /* 再利用可能なページ数 */
} IndexBulkDeleteResult;

入力と出力

  • **入力**: VACUUM 処理を行うインデックス AM の関数（ambulkdelete や amvacuumcleanup）に対し、統計情報を格納する領域としてポインタが渡されます。
  • **出力**: 処理完了後、上記構造体の各フィールドに統計値が書き込まれた状態で呼び出し元へ返されます。

前提条件と不変条件

  • num_index_tuples が正確なカウントであるか、推定値であるかは estimated_count フラグで示されます。
  • pages_deleted は pages_newly_deleted を含む合計値である必要があります。

アルゴリズムまたは振る舞い

インデックス AM は、VACUUM 処理中にこの構造体の各フィールドを更新します。処理の開始時に呼び出し元がメモリを確保し、そのポインタを AM に渡すのが一般的です。

呼び出し元と呼び出し先

  • **呼び出し元（例）**: brinbulkdelete (src/backend/access/brin/brin.c)、blbulkdelete / blvacuumcleanup (contrib/bloom/blvacuum.c) など。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • IndexVacuumInfo: VACUUM 操作のパラメータを保持する構造体。
  • IndexBulkDeleteCallback: インデックス内のタプルを削除すべきか判定するためのコールバック関数型。

失敗モードと境界条件

  • メモリ割り当てに失敗した場合、呼び出し元で palloc0 等の例外が発生する可能性があります。
  • 統計値のオーバーフローについては、double 型および BlockNumber 型の範囲に依存します。

保守担当者向けメモ

  • 本構造体はインデックス AM のインターフェースの一部であり、AM を新規追加する際は必ずこの統計情報を適切に設定する必要があります。
  • estimated_count の扱いは、各 AM の実装（正確なカウントが可能か否か）に委ねられています。
