attach_to_queues

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_shm_mq/worker.c:153

目的

ワーカープロセスが共有メモリ上のメッセージキュー（入力用および出力用）を特定し、プロセスにアタッチして通信可能な状態にする。

シグネチャまたは形状

static void attach_to_queues(dsm_segment *seg, shm_toc *toc, int myworkernumber,
                             shm_mq_handle **inqhp, shm_mq_handle **outqhp)

入力と出力

  • **入力**:
  • seg: 接続先の動的共有メモリセグメント。
  • toc: 共有メモリ内のデータ構造を検索するためのテーブル・オブ・コンテント。
  • myworkernumber: 自身のワーカー番号（キューの検索キーとして使用）。
  • **出力**:
  • inqhp: 入力用メッセージキューのハンドルへのポインタ。
  • outqhp: 出力用メッセージキューのハンドルへのポインタ。

前提条件と不変条件

  • toc は有効な shm_toc 構造体を指していること。
  • myworkernumber に対応する入力キューおよび myworkernumber + 1 に対応する出力キューが toc 内に存在すること。

アルゴリズムまたは振る舞い

1. shm_toc_lookup を使用し、myworkernumber をキーとして入力キュー（inq）を取得する。 2. shm_mq_set_receiver を呼び出し、現在のプロセス（MyProc）を入力キューの受信者に設定する。 3. shm_mq_attach を呼び出し、入力キューをアタッチしてハンドルを inqhp に格納する。 4. shm_toc_lookup を使用し、myworkernumber + 1 をキーとして出力キュー（outq）を取得する。 5. shm_mq_set_sender を呼び出し、現在のプロセス（MyProc）を出力キューの送信者に設定する。 6. shm_mq_attach を呼び出し、出力キューをアタッチしてハンドルを outqhp に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_shm_mq/worker.c 内の test_shm_mq_main 関数から呼び出される。
  • **呼び出し先**: shm_toc_lookup, shm_mq_set_receiver, shm_mq_attach, shm_mq_set_sender（いずれもPostgreSQLの共有メモリ関連API）。

関連データ構造

  • shm_mq: 共有メモリメッセージキュー。
  • shm_toc: 共有メモリ内のデータ構造を管理するテーブル。
  • shm_mq_handle: メッセージキューへの接続ハンドル。

失敗モードと境界条件

  • shm_toc_lookup で指定されたキーが見つからない場合、shm_toc_lookup の第3引数が false であるため、エラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はワーカーの初期化フェーズで一度だけ実行されることを想定している。
  • myworkernumber と myworkernumber + 1 という固定のオフセットでキューを特定しているため、共有メモリ上のレイアウト変更時には注意が必要である。
