PGSemaphoreShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/port/posix_sema.c:164

目的

セマフォサブシステムが共有メモリ上に確保すべきバイト数を算出する。

シグネチャまたは形状

Size PGSemaphoreShmemSize(int maxSemas)

入力と出力

  • **入力**: maxSemas (確保するセマフォの最大数)
  • **出力**: 必要な共有メモリサイズ（バイト単位の Size 型）

前提条件と不変条件

  • USE_NAMED_POSIX_SEMAPHORES が定義されている場合、共有メモリは不要であるため常に 0 を返す。
  • それ以外の場合、PGSemaphoreData 構造体の配列を格納するために必要なサイズを返す。

アルゴリズムまたは振る舞い

1. コンパイル時フラグ USE_NAMED_POSIX_SEMAPHORES を確認する。 2. 名前付きセマフォを使用する場合、共有メモリを消費しないため 0 を返す。 3. 名前なしセマフォを使用する場合、maxSemas に sizeof(PGSemaphoreData) を乗算した値を返す。計算には mul_size を使用し、オーバーフローを考慮する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/ipci.c: Postmaster 起動時の共有メモリサイズ計算時に呼び出される。
  • src/backend/port/posix_sema.c: 内部的なメモリ確保の基準として参照される。
  • **呼び出し先**:
  • mul_size (PostgreSQL ユーティリティ関数)

関連データ構造

  • PGSemaphoreData: セマフォの内部状態を保持する構造体。

失敗モードと境界条件

  • maxSemas が負の値や極端に大きい値の場合、mul_size の内部実装に依存する（通常はエラーまたは適切な上限チェックが行われる）。

保守担当者向けメモ

  • 本関数はプラットフォームごとに実装が異なる（sysv_sema.c や win32_sema.c にも同名の関数が存在する）。変更時は各実装間での一貫性に注意すること。
  • USE_NAMED_POSIX_SEMAPHORES の有無によってメモリレイアウトが根本的に変わるため、共有メモリの初期化ロジックと密接に関連している。
