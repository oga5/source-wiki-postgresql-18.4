big52euc_tw

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_tw_and_big5/euc_tw_and_big5.c:229

目的

Big5エンコーディングの文字列をEUC_TWエンコーディングに変換する内部変換関数。

シグネチャまたは形状

static int big52euc_tw(const unsigned char *big5, unsigned char *p, int len, bool noError)

入力と出力

  • **入力**:
  • big5: 変換元となるBig5エンコーディングの文字列ポインタ。
  • p: 変換後のEUC_TW文字列を格納するバッファ。
  • len: 入力文字列のバイト長。
  • noError: 変換エラー発生時の挙動フラグ（true の場合、エラー時に例外を投げず処理を中断する）。
  • **出力**:
  • 戻り値: 変換処理したバイト数。

前提条件と不変条件

  • 入力文字列はBig5エンコーディングに従っていること。
  • 出力バッファ p は、変換後の文字列を格納するのに十分なサイズがあること。
  • 変換完了後、出力バッファの末尾にはヌル終端文字が付与される。

アルゴリズムまたは振る舞い

1. 入力文字列を1文字ずつ走査し、pg_encoding_verifymbchar で文字境界を検証する。 2. ASCII文字（0x80未満）はそのまま出力バッファへコピーする。 3. マルチバイト文字の場合、BIG5toCNS 関数を用いてCNS11643コードポイントへ変換する。 4. 変換されたコードポイントと文字セット識別子（lc）に基づき、EUC_TW形式（SS2を用いた補助文字セット表現を含む）へエンコードして出力バッファへ書き込む。 5. noError が true の場合、不正な文字や変換不能な文字に遭遇した時点で処理を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の big5_to_euc_tw（src/backend/utils/mb/conversion_procs/euc_tw_and_big5/euc_tw_and_big5.c:78）から呼び出される。
  • **呼び出し先**:
  • pg_encoding_verifymbchar
  • BIG5toCNS
  • report_invalid_encoding / report_untranslatable_char（エラー発生時）

関連データ構造

  • PG_BIG5, PG_EUC_TW: エンコーディング識別子。
  • LC_CNS11643_1 ～ LC_CNS11643_7: CNS11643の文字セット識別子。
  • SS2: EUC_TWにおける補助文字セット用シングルシフトコード。

失敗モードと境界条件

  • 入力文字列が不正なバイトシーケンスである場合、noError が false ならエラーを報告し、true ならその時点までのバイト数を返して終了する。
  • 変換不能な文字に遭遇した場合も同様にエラー処理を行う。
  • 入力文字列が空、またはヌル文字が含まれる場合はエラーとして扱われる。

保守担当者向けメモ

  • 本関数は big5.c で定義されている BIG5toCNS ヘルパー関数に変換ロジックを依存している。
  • EUC_TWのマルチバイト構造（特にSS2を用いたCNS11643の各プレーンへのマッピング）の変更が必要な場合は、本関数の条件分岐ロジックを更新すること。
