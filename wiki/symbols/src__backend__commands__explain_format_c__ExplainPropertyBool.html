ExplainPropertyBool

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_format.c:263

目的

EXPLAIN コマンドの出力において、真偽値（boolean）のプロパティを現在のフォーマット（TEXT, XML, JSON, YAML）に合わせて出力バッファへ書き込みます。

シグネチャまたは形状

void ExplainPropertyBool(const char *qlabel, bool value, ExplainState *es)

入力と出力

  • qlabel (入力): プロパティのラベル名。
  • value (入力): 出力する真偽値。
  • es (入力/出力): 出力状態を管理する ExplainState 構造体。この構造体のバッファにフォーマット済みの文字列が追加されます。

前提条件と不変条件

  • es は初期化済みである必要があります。
  • qlabel は NULL または有効な文字列である必要があります。

アルゴリズムまたは振る舞い

1. value が true の場合は "true"、false の場合は "false" という文字列リテラルを選択します。 2. ExplainProperty 関数を呼び出し、選択した文字列を渡します。 3. ExplainProperty 内部で、es->format に基づき、各フォーマット固有の引用符付けやエスケープ処理が行われ、出力バッファへ書き込まれます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/explain.c: JIT 関連のフラグ出力などで使用。
  • contrib/pg_overexplain/pg_overexplain.c: リレーションの属性（Parallel Safe, Inherited, Lateral 等）の出力で使用。
  • **呼び出し先**:
  • ExplainProperty: 実際のフォーマット変換とバッファ書き込みを行う共通関数。

関連データ構造

  • ExplainState: 出力先、現在のフォーマット、インデントレベル、グループ状態を保持する中心的な構造体。

失敗モードと境界条件

  • es が NULL の場合、プログラムは異常終了する可能性があります。
  • 出力バッファのメモリ不足が発生した場合、ExplainProperty 経由でエラーが報告される可能性があります。

保守担当者向けメモ

  • 本関数は ExplainProperty のラッパーです。真偽値の表現をカスタマイズしたい場合は ExplainProperty の実装を確認してください。
  • 新しい出力フォーマットを追加する場合は、ExplainProperty 内の switch 文を更新する必要があります。
