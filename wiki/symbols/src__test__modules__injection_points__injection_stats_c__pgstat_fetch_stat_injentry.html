pgstat_fetch_stat_injentry

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_stats.c:93

目的

指定された名前のインジェクションポイントに対応する統計エントリ（PgStat_StatInjEntry）を、共有メモリ上の統計サブシステムから取得します。

シグネチャまたは形状

static PgStat_StatInjEntry *pgstat_fetch_stat_injentry(const char *name)

入力と出力

  • **入力**: name (インジェクションポイント名を示す文字列)
  • **出力**: 該当する PgStat_StatInjEntry 構造体へのポインタ。見つからない場合や統計機能が無効な場合は NULL を返します。

前提条件と不変条件

  • 統計機能がロード済み（inj_stats_loaded）かつ有効（inj_stats_enabled）である必要があります。
  • 内部的に PGSTAT_INJ_IDX(name) マクロを使用して、名前からハッシュ化されたインデックスを生成します。

アルゴリズムまたは振る舞い

1. 統計機能の有効性を確認し、無効であれば NULL を返します。 2. name を PGSTAT_INJ_IDX マクロでハッシュ化します。 3. pgstat_fetch_entry を呼び出し、PGSTAT_KIND_INJECTION 種別および生成したハッシュキーを用いて共有メモリからエントリを検索します。 4. 検索結果のポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: injection_points_stats_numcalls 関数（src/test/modules/injection_points/injection_stats.c:191）から呼び出されることが確認されています。
  • **呼び出し先**: pgstat_fetch_entry（PostgreSQL統計サブシステム）。

関連データ構造

  • PgStat_StatInjEntry: インジェクションポイントの統計データ（実行回数等）を保持する構造体。

失敗モードと境界条件

  • inj_stats_loaded または inj_stats_enabled が偽の場合、検索を行わず NULL を返します。
  • 該当する名前のインジェクションポイントが統計サブシステムに存在しない場合、pgstat_fetch_entry の仕様に従い NULL が返されます。

保守担当者向けメモ

  • 本関数は SQL インターフェース経由での統計取得時に使用されます。
  • PGSTAT_KIND_INJECTION は 25 とハードコードされているため、統計サブシステム側の変更時には整合性に注意してください。
