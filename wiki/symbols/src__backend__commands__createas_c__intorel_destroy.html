intorel_destroy

定義

  • 種別: function
  • 定義箇所: src/backend/commands/createas.c:633

目的

DestReceiver オブジェクトを解放し、関連するメモリをクリーンアップする。

シグネチャまたは形状

static void intorel_destroy(DestReceiver *self)

入力と出力

  • **入力**: DestReceiver *self - 解放対象の DestReceiver 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • self は palloc 等で割り当てられた有効な DestReceiver 構造体を指している必要がある。
  • 本関数呼び出し後、self が指すメモリ領域は無効となる。

アルゴリズムまたは振る舞い

1. pfree(self) を実行し、DestReceiver オブジェクトに割り当てられたメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: 直接的な呼び出し元はソースコードの文脈からは特定できないが、DestReceiver インターフェースの rDestroy フィールドに本関数が代入されている（src/backend/commands/createas.c:447）。
  • **呼び出し先**: pfree（メモリ解放関数）。

関連データ構造

  • DestReceiver: クエリ結果を受け取るためのインターフェース構造体。

失敗モードと境界条件

  • self が NULL の場合、pfree の実装に依存する（通常はエラーとなる）。

保守担当者向けメモ

  • 本関数は DestReceiver の破棄用コールバックとして使用される。intorel_* 系の関数群の一部であり、CREATE TABLE AS 等のクエリ結果処理の終了処理において重要な役割を果たす。
