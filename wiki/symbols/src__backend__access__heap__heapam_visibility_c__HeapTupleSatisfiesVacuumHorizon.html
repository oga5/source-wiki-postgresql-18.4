HeapTupleSatisfiesVacuumHorizon

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_visibility.c:1204

目的

VACUUM処理において、タプルが物理的に削除可能か、あるいは「最近削除された（Recently Dead）」状態であるかを判定します。

シグネチャまたは形状

HTSV_Result HeapTupleSatisfiesVacuumHorizon(HeapTuple htup, Buffer buffer, TransactionId *dead_after)

入力と出力

  • **入力**:
  • htup: 判定対象のタプル。
  • buffer: タプルが格納されているバッファ。
  • dead_after: 削除されたトランザクションIDを格納するためのポインタ（出力用）。
  • **出力**:
  • HTSV_Result: タプルの可視性状態（HEAPTUPLE_LIVE, HEAPTUPLE_DEAD, HEAPTUPLE_RECENTLY_DEAD, HEAPTUPLE_INSERT_IN_PROGRESS, HEAPTUPLE_DELETE_IN_PROGRESS）。

前提条件と不変条件

  • htup->t_self が有効であること。
  • htup->t_tableOid が InvalidOid でないこと。
  • dead_after ポインタが非NULLであること。
  • 対象バッファに対して少なくとも共有ロック（Shared Lock）を保持していること。

アルゴリズムまたは振る舞い

1. **挿入トランザクションの確認**: タプルの Xmin を確認し、挿入がコミット済みか判定します。未コミットの場合は、実行中かアボート済みかに応じて HEAPTUPLE_INSERT_IN_PROGRESS または HEAPTUPLE_DEAD を返します。 2. **削除トランザクションの確認**: 挿入がコミット済みであれば、Xmax を確認します。

  • Xmax が無効であれば HEAPTUPLE_LIVE を返します。
  • Xmax がロックのみを目的としている場合は、ロック状態を考慮しつつ HEAPTUPLE_LIVE を返します。
  • Xmax が更新（削除）を伴う場合、そのトランザクションが実行中なら HEAPTUPLE_DELETE_IN_PROGRESS を返し、コミット済みであれば dead_after に Xmax を設定し HEAPTUPLE_RECENTLY_DEAD を返します。

    • 3. **ヒントビットの更新**: 判定過程でトランザクションの確定状態（コミット/アボート）が判明した場合、SetHintBits を呼び出してタプルヘッダーを更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_visibility.c (内部の可視性判定ロジック)
  • src/backend/access/heap/pruneheap.c:922
  • **呼び出し先**:
  • SetHintBits
  • TransactionIdIsCurrentTransactionId, TransactionIdIsInProgress, TransactionIdDidCommit
  • MultiXactIdIsRunning 等のトランザクション状態確認関数

関連データ構造

  • HeapTuple: タプル本体。
  • Buffer: タプルが配置されたバッファ。
  • HTSV_Result: 可視性判定結果の列挙型。

失敗モードと境界条件

  • HEAP_MOVED_OFF や HEAP_MOVED_IN フラグが設定されている場合、古いバイナリアップグレード用の特殊な判定パスを通ります。
  • MultiXactId が関与する場合、ロック状態と更新状態を厳密に区別して判定します。

保守担当者向けメモ

  • 本関数は HeapTupleSatisfiesVisibility 等の標準的な可視性判定とは異なり、VACUUMの物理削除判断に特化しています。
  • dead_after は、VACUUMがタプルを削除してよいか判断する際の「 horizon（水平線）」を決定するために使用されます。
