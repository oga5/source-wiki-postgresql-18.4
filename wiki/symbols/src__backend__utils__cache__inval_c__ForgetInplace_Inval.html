ForgetInplace_Inval

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/cache/inval.c:1285

目的

インプレース更新に関連する無効化情報（inplaceInvalInfo）の追跡を終了し、ポインタを無効化（NULL化）するために使用されます。

シグネチャまたは形状

void ForgetInplace_Inval(void)

入力と出力

  • **入力**: なし
  • **出力**: なし（副作用としてグローバル変数 inplaceInvalInfo を NULL に設定）

前提条件と不変条件

  • 本関数は、インプレース更新処理の完了後に呼び出されることを想定しています。
  • 呼び出し後、inplaceInvalInfo は NULL となるため、後続の無効化処理でこのポインタを参照することはできません。

アルゴリズムまたは振る舞い

1. グローバル変数 inplaceInvalInfo に NULL を代入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c:6488
  • src/backend/access/heap/heapam.c:6657
  • **呼び出し先**:
  • 情報なし

関連データ構造

  • inplaceInvalInfo（グローバル変数）: インプレース更新時の無効化情報を保持する構造体へのポインタ。

失敗モードと境界条件

  • 本関数自体は単純な代入操作であるため、失敗モードは定義されていません。
  • inplaceInvalInfo が既に NULL である状態で呼び出された場合でも、安全に動作します。

保守担当者向けメモ

  • 本関数は、通常のトランザクション管理とは異なるパスを通る「インプレース更新」専用のクリーンアップ処理です。
  • PreInplace_Inval や AtInplace_Inval と組み合わせて使用されるライフサイクルの一部です。
