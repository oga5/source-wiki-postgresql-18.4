RelocationBufferInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gist/gistbuildbuffers.c:519

目的

GiSTインデックスのバッファリングビルドにおいて、ノード分割（Split）発生時に、既存のノードバッファ内のタプルを新しいノードへ再配置（リロケート）するための情報を保持する。

シグネチャまたは形状

typedef struct {
    GISTENTRY           entry[INDEX_MAX_KEYS];
    bool                isnull[INDEX_MAX_KEYS];
    GISTPageSplitInfo  *splitinfo;
    GISTNodeBuffer     *nodeBuffer;
} RelocationBufferInfo;

入力と出力

本構造体はデータ保持用であり、関数ではない。主に gistRelocateBuildBuffersOnSplit 内で、分割対象のバッファ情報と分割後の新しいノード情報を紐付けるために使用される。

前提条件と不変条件

  • nodeBuffer は有効な GISTNodeBuffer を指している必要がある。
  • splitinfo は、分割後のノード配置に関する情報を含んでいる必要がある。

アルゴリズムまたは振る舞い

GiSTのノード分割時、gistRelocateBuildBuffersOnSplit はこの構造体の配列を使用して、古いバッファから取り出したタプルをどの新しいノードへ振り分けるべきかを決定する。各要素は、特定のバッファとそれに対応する分割後のノード状態を管理する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistbuildbuffers.c 内の関数（主に gistRelocateBuildBuffersOnSplit）で、palloc を介してメモリ確保され、分割処理のコンテキストで使用される。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはない。

関連データ構造

  • GISTNodeBuffer: バッファリングビルドにおける各ノードのタプル保持領域。
  • GISTPageSplitInfo: ノード分割時の詳細情報。
  • GISTENTRY: GiSTインデックスのキーエントリ。

失敗モードと境界条件

  • メモリ不足時: palloc によるメモリ確保失敗時に elog(ERROR) が発生する。
  • 構造体内のポインタが NULL の場合、参照時にセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本構造体は gistRelocateBuildBuffersOnSplit の実行中のみ有効な一時的な管理用データである。
  • INDEX_MAX_KEYS に依存しているため、GiSTのキー構造に変更がある場合は整合性に注意すること。
