ConfigurePostmasterWaitSet

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/postmaster.c:1629

目的

Postmaster プロセスが待機すべきイベントセット（WaitEventSet）を再構築し、接続受付ソケットおよびラッチの監視状態を更新する。

シグネチャまたは形状

static void ConfigurePostmasterWaitSet(bool accept_connections)

入力と出力

  • **入力**:
  • accept_connections (bool): true の場合、既存のリスニングソケットを監視対象に含める。
  • **出力**: なし（副作用として静的変数 pm_wait_set を更新）。

前提条件と不変条件

  • pm_wait_set は、本関数呼び出し前に既存のイベントセットを保持している場合、適切に解放される。
  • NumListenSockets および ListenSockets 配列が有効な状態であること。

アルゴリズムまたは振る舞い

1. 既存の pm_wait_set が存在する場合は FreeWaitEventSet で解放し、ポインタを NULL にリセットする。 2. CreateWaitEventSet を呼び出し、新しいイベントセットを作成する。監視対象数は accept_connections が真であれば 1 + NumListenSockets、偽であれば 1 となる。 3. AddWaitEventToSet を使用し、常に WL_LATCH_SET イベント（MyLatch）をセットに追加する。 4. accept_connections が true の場合、ループ処理により ListenSockets 配列内のすべてのソケットを WL_SOCKET_ACCEPT イベントとしてセットに追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/postmaster.c 内の複数箇所（サーバー起動時、状態遷移時など）。
  • **呼び出し先**:
  • FreeWaitEventSet
  • CreateWaitEventSet
  • AddWaitEventToSet

関連データ構造

  • WaitEventSet: 待機イベントを管理する構造体。
  • ListenSockets: リッスン中のソケットを保持する配列。
  • MyLatch: Postmaster プロセスが使用するラッチ。

失敗モードと境界条件

  • CreateWaitEventSet がメモリ不足等で失敗した場合、PostgreSQL の標準的なエラーハンドリングによりプロセスが異常終了する可能性がある。
  • accept_connections が false の場合、ソケット監視は行われず、ラッチの待機のみが行われる。

保守担当者向けメモ

  • 本関数は ServerLoop が待機するイベントセットを動的に再構成するために使用される。接続許可状態が変更されるたびに呼び出されるため、パフォーマンスへの影響を考慮すること。
