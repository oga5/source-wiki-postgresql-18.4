pg_cryptohash_ctx

定義

  • 種別: struct
  • 定義箇所: src/common/cryptohash.c:51

目的

PostgreSQLにおける暗号学的ハッシュ計算（MD5, SHA1, SHA2系）の状態を保持するための不透明なコンテキスト構造体です。

シグネチャまたは形状

struct pg_cryptohash_ctx
{
    pg_cryptohash_type type;
    pg_cryptohash_errno error;

    union
    {
        pg_md5_ctx    md5;
        pg_sha1_ctx   sha1;
        pg_sha224_ctx sha224;
        pg_sha256_ctx sha256;
        pg_sha384_ctx sha384;
        pg_sha512_ctx sha512;
    }             data;
};

入力と出力

  • **入力**: pg_cryptohash_create() を介して初期化され、pg_cryptohash_update() に渡されることでハッシュ計算の対象データが供給されます。
  • **出力**: pg_cryptohash_final() を通じて、計算結果が指定されたバッファに書き出されます。

前提条件と不変条件

  • コンテキスト構造体は、サポートする全アルゴリズムの中で最大のサイズ（SHA512用）に合わせて固定確保されます。
  • pg_cryptohash_free() を呼び出す際、機密情報保護のため explicit_bzero を用いてメモリ内容が消去されます。

アルゴリズムまたは振る舞い

  • ハッシュアルゴリズムの抽象化レイヤーとして機能し、内部の union により選択されたアルゴリズムのコンテキストを保持します。
  • pg_cryptohash_create で型が記録され、以降の init, update, final 関数でその型に応じた内部実装が呼び出されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/uuid-ossp/uuid-ossp.c
  • src/backend/libpq/auth-scram.c
  • src/backend/utils/adt/cryptohashfuncs.c
  • src/common/cryptohash.c（内部関数群）
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出しません。操作は pg_cryptohash_* 系関数を介して行われます。

関連データ構造

  • pg_cryptohash_type: 使用するハッシュアルゴリズムの識別子。
  • pg_cryptohash_errno: エラー状態を保持する列挙型。
  • 各アルゴリズム固有のコンテキスト（pg_md5_ctx, pg_sha1_ctx, pg_sha256_ctx 等）。

失敗モードと境界条件

  • pg_cryptohash_final() における出力バッファサイズの検証失敗が主なエラー発生源です。
  • エラーの詳細は pg_cryptohash_error() を通じて取得可能です。

保守担当者向けメモ

  • 本構造体はコードの複雑性を避けるため、最大サイズで固定確保される設計となっています。
  • メモリ管理は環境に応じて palloc（バックエンド）または malloc（フロントエンド）が切り替わります。
