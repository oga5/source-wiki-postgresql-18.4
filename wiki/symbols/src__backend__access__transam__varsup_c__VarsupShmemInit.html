VarsupShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/varsup.c:46

目的

トランザクションIDおよびオブジェクトIDの生成状態を保持する共有メモリ構造体 TransamVariables を初期化またはアタッチします。

シグネチャまたは形状

void VarsupShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上の TransamVariables が初期化または既存の領域に割り当てられる）

前提条件と不変条件

  • ShmemInitStruct を使用して共有メモリ領域を確保します。
  • IsUnderPostmaster が偽（Postmaster起動時）の場合、共有メモリ領域はゼロクリアされます。
  • IsUnderPostmaster が真（バックエンドプロセス起動時）の場合、領域は既に存在している必要があります。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を呼び出し、"TransamVariables" という名前の共有メモリ領域を確保または取得します。 2. IsUnderPostmaster の値に基づき、以下のいずれかの処理を行います。

  • **Postmaster起動時**: 領域が新規作成されたことを確認し、TransamVariables 構造体をゼロクリアします。
  • **バックエンドプロセス起動時**: 領域が既に存在していることを確認します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:287（CreateSharedMemoryAndSemaphores 内から呼び出される）
  • 呼び出し先: ShmemInitStruct（PostgreSQL共有メモリ管理API）

関連データ構造

  • TransamVariables: ID生成状態を保持する共有メモリ上の構造体。
  • TransamVariablesData: TransamVariables の型定義。

失敗モードと境界条件

  • ShmemInitStruct が失敗した場合、PostgreSQLの共有メモリ確保のルールに従い、プロセスは異常終了します。
  • IsUnderPostmaster の状態と共有メモリの存在有無が不一致の場合、Assert によりデバッグビルドでは停止します。

保守担当者向けメモ

  • 本関数はシステム起動時の共有メモリ初期化シーケンスの一部です。
  • TransamVariables の構造に変更を加える場合は、VarsupShmemSize（共有メモリサイズ計算関数）との整合性に注意してください。
