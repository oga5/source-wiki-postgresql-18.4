remove_leftjoinrel_from_query

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/analyzejoins.c:559

目的

クエリ最適化において、不要と判定された左結合（LEFT JOIN）のリレーションをクエリから削除し、関連するデータ構造を整合させる。

シグネチャまたは形状

static void remove_leftjoinrel_from_query(PlannerInfo *root, int relid, SpecialJoinInfo *sjinfo)

入力と出力

  • root: プランナのグローバル情報構造体。
  • relid: 削除対象のリレーションID。
  • sjinfo: 削除対象の結合に関する SpecialJoinInfo。
  • 出力: なし（副作用として PlannerInfo を更新）。

前提条件と不変条件

  • sjinfo->ojrelid が 0 でないこと。
  • 削除対象のリレーションが結合条件に基づき不要であることが事前に証明されていること。
  • PlaceHolderVar の評価位置（ph_eval_at）が更新済みであること。

アルゴリズムまたは振る舞い

1. **リレーションの削除**: remove_rel_from_query を呼び出し、クエリツリーから対象リレーションを排除する。 2. **結合条件の再処理**:

  • 対象リレーションに関連する RestrictInfo を走査する。
  • 削除対象リレーションや結合（ojrelid）への参照を RestrictInfo から除去する。
  • 削除後も有効な制約（RINFO_IS_PUSHED_DOWN が真）は、distribute_restrictinfo_to_rels を通じて再配置する。

    • 3. **メタデータのクリーンアップ**:

  • root->simple_rel_array および simple_rte_array から対象リレーションを削除し、RelOptInfo を解放する。

    • 4. **属性必要性の再計算**: rebuild_placeholder_attr_needed 等を呼び出し、結合削除に伴う属性の必要性情報を再構築する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/analyzejoins.c 内の remove_useless_joins 等から呼び出される。
  • **呼び出し先**: remove_rel_from_query, remove_join_clause_from_rels, remove_rel_from_restrictinfo, distribute_restrictinfo_to_rels, rebuild_placeholder_attr_needed 等。

関連データ構造

  • PlannerInfo: プランナのメイン構造体。
  • SpecialJoinInfo: 結合の特性を保持する構造体。
  • RelOptInfo: リレーションの最適化情報を保持する構造体。
  • RestrictInfo: 結合条件やフィルタ条件を保持する構造体。

失敗モードと境界条件

  • 削除対象の結合が他のクエリ要素と複雑に依存している場合、rebuild 系の関数で整合性が維持されないリスクがある（ソースコード上のアサーションで保護されている）。
  • 外部結合の置換や削除に関連する PlaceHolderVar の参照が残っている場合、プランナがクラッシュする可能性がある。

保守担当者向けメモ

  • remove_join_clause_from_rels がリストを破壊するため、処理前に list_copy で複製を取る必要がある。
  • ChangeVarNodesExtended を用いた参照置換の副作用に注意すること。
  • 結合削除後の attr_needed 再計算は、プランナの整合性維持に不可欠である。
