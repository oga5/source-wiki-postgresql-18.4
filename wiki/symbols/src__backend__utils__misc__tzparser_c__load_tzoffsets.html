load_tzoffsets

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/tzparser.c:448

目的

タイムゾーン設定ファイルを読み込み、解析し、PostgreSQL が内部で使用する TimeZoneAbbrevTable 形式へ変換して返します。

シグネチャまたは形状

TimeZoneAbbrevTable *load_tzoffsets(const char *filename)

入力と出力

  • **入力**: filename (読み込むタイムゾーン設定ファイル名)
  • **出力**: 解析および変換に成功した場合は TimeZoneAbbrevTable 構造体へのポインタ。失敗した場合は NULL。

前提条件と不変条件

  • GUC（Grand Unified Configuration）のチェックフックから呼び出されることを想定しており、elog(ERROR) を発生させない設計であること。
  • 内部で一時的なメモリコンテキストを作成し、処理終了後に解放する。

アルゴリズムまたは振る舞い

1. AllocSetContextCreate を使用して一時的なメモリコンテキストを作成し、処理対象をそこに切り替えます。 2. tzEntry 配列を初期サイズ 128 で確保します。 3. ParseTzFile を呼び出し、指定されたファイルおよび @INCLUDE 指令で指定されたファイルを再帰的に解析します。 4. 解析結果が正常（n >= 0）であれば、ConvertTimeZoneAbbrevs を呼び出し、tzEntry 配列を最終的な TimeZoneAbbrevTable 形式へ変換します。 5. 変換に失敗した場合は GUC_check_errmsg を通じてエラーメッセージを設定します。 6. 最後にメモリコンテキストを元の状態に戻し、一時コンテキストを削除して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/variable.c:506
  • **呼び出し先**: ParseTzFile, ConvertTimeZoneAbbrevs, AllocSetContextCreate, MemoryContextSwitchTo, MemoryContextDelete, palloc, GUC_check_errmsg

関連データ構造

  • TimeZoneAbbrevTable: 最終的な変換結果を保持する構造体。
  • tzEntry: 解析された各タイムゾーン略称とオフセットを保持する構造体。

失敗モードと境界条件

  • ファイルの解析中にエラーが発生した場合、ParseTzFile が負の値を返し、変換処理はスキップされます。
  • メモリ不足時には ConvertTimeZoneAbbrevs が NULL を返し、GUC_check_errmsg に "out of memory" が設定されます。

保守担当者向けメモ

  • 本関数は GUC フックから呼ばれるため、エラー報告には elog ではなく GUC_check_errmsg を使用してください。
  • tzEntry 配列のソート順は datetime.c 側の検索ロジックと密結合しているため、変更時は注意が必要です。
