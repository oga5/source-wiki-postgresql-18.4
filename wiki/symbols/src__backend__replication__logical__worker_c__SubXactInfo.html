SubXactInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/worker.c:335

目的

論理レプリケーションのストリーミングトランザクションにおいて、サブトランザクション（サブトランザクションID）と、その変更内容が格納されている一時ファイル（BufFile）上の位置を対応付けるために使用されます。

シグネチャまたは形状

typedef struct SubXactInfo
{
    TransactionId xid;    /* サブトランザクションのXID */
    int           fileno;   /* BufFile内のファイル番号 */
    off_t         offset;   /* ファイル内のオフセット */
} SubXactInfo;

入力と出力

本構造体は、ストリーミングされたトランザクションデータを一時ファイルから再読み込みする際、特定のサブトランザクションの開始位置を特定するためのインデックスとして機能します。

前提条件と不変条件

  • fileno および offset は、BufFile API を用いて書き込まれた有効なファイル位置を指している必要があります。
  • xid は有効なサブトランザクションIDである必要があります。

アルゴリズムまたは振る舞い

論理レプリケーションワーカーが巨大なトランザクションを処理する際、メモリ制限を超えるとメッセージを一時ファイルに退避します。このとき、各サブトランザクションのメタデータを SubXactInfo 配列として保持し、後続の適用処理（apply_spooled_messages 等）でこの情報を参照して、該当するサブトランザクションの変更内容をファイルからシーク・読み込みします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/worker.c 内のストリーミングトランザクション処理ロジック（subxact_data 関連の操作）から参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • BufFile: ストリーミングされたメッセージが格納される一時ファイル。
  • ApplySubXactData: SubXactInfo の配列を管理し、ストリーミング中のサブトランザクション状態を保持する構造体。

失敗モードと境界条件

  • subxact_data.nsubxacts_max を超えるサブトランザクションが発生した場合、メモリ再割り当て（palloc）が必要となります。
  • ファイル操作中に fileno や offset が無効な場合、BufFile からの読み込み時にエラーが発生します。

保守担当者向けメモ

  • 本構造体は worker.c 内のストリーミングトランザクションのシリアライズ/デシリアライズ処理に深く依存しています。
  • subxact_data 関連のコード（worker.c:4134 以降）を修正する際は、SubXactInfo のサイズ計算が正しく行われているか確認してください。
