statext_ndistinct_build

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/mvdistinct.c:87

目的

拡張統計情報における多変量NDistinct（列の組み合わせごとの一意な値の数）を計算し、MVNDistinct 構造体として構築する。

シグネチャまたは形状

MVNDistinct *statext_ndistinct_build(double totalrows, StatsBuildData *data)

入力と出力

  • **入力**:
  • totalrows: 対象テーブルの総行数。
  • data: 統計構築に必要な属性情報やサンプルデータを含む StatsBuildData 構造体。
  • **出力**:
  • 構築された MVNDistinct 構造体へのポインタ。

前提条件と不変条件

  • data->nattnums で指定された列数に基づき、2列以上のすべての組み合わせに対してNDistinctを計算する。
  • 構築後の result->nitems は、生成された組み合わせの総数と一致しなければならない。
  • 各 MVNDistinctItem の属性番号は有効（AttributeNumberIsValid）であること。

アルゴリズムまたは振る舞い

1. num_combinations を用いて必要な組み合わせ総数を算出し、MVNDistinct 構造体をメモリ確保する。 2. k を2から numattrs までループさせ、CombinationGenerator を用いて列の全組み合わせを生成する。 3. 各組み合わせに対し、ndistinct_for_combination を呼び出してNDistinct値を推定する。 4. 推定結果を MVNDistinctItem 配列に格納する。 5. 最後に、全アイテムが正しく埋められたことをアサートして結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/extended_stats.c:205
  • **呼び出し先**:
  • num_combinations
  • generator_init / generator_next / generator_free
  • ndistinct_for_combination

関連データ構造

  • MVNDistinct: 構築される統計情報のメイン構造体。
  • MVNDistinctItem: 個別の列組み合わせとそのNDistinct値を保持する構造体。
  • StatsBuildData: 統計構築の入力データ。
  • CombinationGenerator: 列の組み合わせを列挙するためのイテレータ。

失敗モードと境界条件

  • 列数が増加すると組み合わせ爆発が発生するため、統計オブジェクト作成時の列数には制限がある。
  • メモリ不足時には palloc によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は ANALYZE 処理の一部として実行される。
  • ndistinct_for_combination が内部でソート処理を行うため、列数やデータ量に応じた計算コストに注意すること。
