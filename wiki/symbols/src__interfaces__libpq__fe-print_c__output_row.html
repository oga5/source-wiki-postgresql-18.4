output_row

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-print.c:561

目的

クエリ結果の1行分を、指定されたフォーマット（標準テキストまたはHTML）に従って出力ストリームへ書き出します。

シグネチャまたは形状

static void output_row(FILE *fout, const PQprintOpt *po, const int nFields, char **fields,
                       unsigned char *fieldNotNum, int *fieldMax, char *border,
                       const int row_index)

入力と出力

  • fout: 出力先のファイルポインタ。
  • po: 出力オプション（PQprintOpt）。HTML出力やフィールド区切り文字の制御に使用。
  • nFields: 行内のフィールド数。
  • fields: 全行のデータを含む文字列配列。
  • fieldNotNum: 各フィールドが数値以外かを示すフラグ配列。
  • fieldMax: 各フィールドの最大幅を保持する配列。
  • border: 標準出力時に使用する枠線文字列。
  • row_index: 出力対象の行番号。
  • 戻り値: なし（void）。

前提条件と不変条件

  • fields 配列は row_index * nFields + field_index で各フィールドにアクセス可能であること。
  • fieldNotNum および fieldMax は nFields 以上のサイズを持つこと。

アルゴリズムまたは振る舞い

1. **HTMLモード**: po->html3 が真の場合、<tr> タグで開始し、各フィールドを <td> タグで囲んで出力します。数値以外は左寄せ、数値は右寄せで配置されます。最後に </tr> を出力します。 2. **標準モード**: po->standard が真の場合、フィールド区切り文字（po->fieldSep）を挟みつつ、fieldMax に基づいて整形出力します。行末には border を出力します。 3. **共通**: 各フィールドの出力後、最後に改行文字を出力します。フィールド値が NULL の場合は空文字列として扱います。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-print.c 内の PQprint 関数から呼び出されます。
  • 呼び出し先: 標準ライブラリの fputs, fprintf, fputc を使用します。

関連データ構造

  • PQprintOpt: 出力形式を制御する構造体。

失敗モードと境界条件

  • fout が無効な場合、出力処理は失敗します。
  • fields 内のポインタが NULL の場合、空文字列として処理されます。
  • fieldMax の値が実際のデータ長より小さい場合、整形が崩れる可能性があります。

保守担当者向けメモ

  • 本関数は PQprint の内部補助関数であり、直接的な外部公開はされていません。
  • po->html3 と po->standard の両方が偽の場合、フィールド区切り文字のみを用いた単純な出力が行われます。
