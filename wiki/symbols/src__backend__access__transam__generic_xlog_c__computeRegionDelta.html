computeRegionDelta

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/generic_xlog.c:120

目的

2つのページイメージ（curpage と targetpage）の指定範囲を比較し、変更された領域を差分（デルタ）として抽出し、writeFragment を介して記録する。

シグネチャまたは形状

static void computeRegionDelta(GenericXLogPageData *pageData,
                               const char *curpage, const char *targetpage,
                               int targetStart, int targetEnd,
                               int validStart, int validEnd)

入力と出力

  • pageData: 構築中のWALレコード状態を保持する構造体。
  • curpage: 変更前のページイメージ。
  • targetpage: 変更後のページイメージ。
  • targetStart, targetEnd: 比較対象とするページ内の範囲。
  • validStart, validEnd: 有効なデータ領域の範囲。
  • 出力: なし（副作用として writeFragment を呼び出し、差分を記録する）。

前提条件と不変条件

  • targetStart から targetEnd までの範囲が比較対象となる。
  • validStart より前の領域、および validEnd より後の領域は、無効な領域として強制的に差分に含められる。
  • MATCH_THRESHOLD を超える連続した一致箇所は、差分として記録されない（圧縮される）。

アルゴリズムまたは振る舞い

1. **初期化**: 無効な開始領域（validStart > targetStart）を最初のフラグメントとしてマークする。 2. **走査**: targetStart から loopEnd（targetEnd と validEnd の小さい方）までバイト単位で比較を行う。 3. **不一致の検出**: 異なるバイトが見つかった場合、新しいフラグメントを開始し、一致するバイトが現れるまで継続する。 4. **一致の検出**: 一致するバイトが MATCH_THRESHOLD を超えて連続した場合、現在のフラグメントを終了し、writeFragment を呼び出して書き出す。 5. **終了処理**: ループ終了後、無効な終了領域（loopEnd < targetEnd）を最後のフラグメントに含め、残存するフラグメントがあれば書き出す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/generic_xlog.c 内の computeDelta 関数（行238, 242）。
  • 呼び出し先: writeFragment（差分の書き込み）。

関連データ構造

  • GenericXLogPageData: 汎用WALレコードのページデータを管理する構造体。

失敗モードと境界条件

  • MATCH_THRESHOLD 以下の短い一致は、不一致領域の一部としてマージされ、差分として記録される。
  • ページ内の「ホール」領域（pd_lower と pd_upper 間）の不一致は、この関数を通じて適切に処理される。

保守担当者向けメモ

  • 本関数は generic_xlog.c におけるデルタ圧縮の中核であり、パフォーマンスに直結する。
  • MATCH_THRESHOLD を超える一致の判定ロジックは、WALサイズ削減の効率を左右するため、変更時には注意が必要。
  • ページ内の無効領域（ホール）の扱いは、異なる環境間でのページ再構築時の整合性維持に不可欠である。
