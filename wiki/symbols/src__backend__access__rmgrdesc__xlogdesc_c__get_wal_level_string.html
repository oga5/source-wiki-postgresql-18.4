get_wal_level_string

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/xlogdesc.c:39

目的

wal_level の整数値を、人間が読み取り可能な文字列（設定名）に変換します。主にWALレコードの内容をデバッグ出力やログ表示する際に使用されます。

シグネチャまたは形状

static const char *get_wal_level_string(int wal_level)

入力と出力

  • **入力**: int wal_level（変換対象の wal_level 設定値）
  • **出力**: const char *（対応する文字列名。一致するものがない場合は "?" を返す）

前提条件と不変条件

  • wal_level_options 配列が有効な config_enum_entry のリストとして定義されている必要があります。
  • wal_level_options は name フィールドが NULL であるエントリで終端されている必要があります。

アルゴリズムまたは振る舞い

1. wal_level_options 配列を先頭から順に走査します。 2. 各エントリの val フィールドが引数の wal_level と一致するか確認します。 3. 一致した場合はその name を返し、ループを終了します。 4. 配列の終端まで一致するものが見つからなかった場合、デフォルト値である "?" を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/rmgrdesc/xlogdesc.c 内の各箇所（チェックポイント情報やパラメータ変更ログの出力時など、計5箇所で参照を確認）。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。wal_level_options 構造体配列を参照します。

関連データ構造

  • wal_level_options: wal_level の値と文字列の対応を保持する config_enum_entry 型の構造体配列。

失敗モードと境界条件

  • 引数 wal_level がどの定義済みオプションにも該当しない場合、エラーにはならず "?" を返します。

保守担当者向けメモ

  • 新しい wal_level の値が追加された場合は、wal_level_options 配列を更新する必要があります。本関数自体は配列を走査する汎用的な実装であるため、ロジックの変更は不要です。
