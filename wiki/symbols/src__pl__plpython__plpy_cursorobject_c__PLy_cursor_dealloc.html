PLy_cursor_dealloc

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_cursorobject.c:303

目的

PLyCursorObject インスタンスの破棄および関連リソースの解放を行います。

シグネチャまたは形状

static void PLy_cursor_dealloc(PLyCursorObject *self);

入力と出力

  • 入力: PLyCursorObject *self（解放対象のカーソルオブジェクト）
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • self は有効な PLyCursorObject へのポインタである必要があります。
  • Python 3.8 以降では、型オブジェクトの参照カウント管理が本関数内で行われます。

アルゴリズムまたは振る舞い

1. **カーソルのクローズ**: self->closed が false の場合、GetPortalByName でポータルを取得し、UnpinPortal および SPI_cursor_close を実行してポータルを無効化します。 2. **メモリ解放**: self->mcxt が存在する場合、MemoryContextDelete でメモリコンテキストを破棄し、ポインタを NULL に設定します。 3. **オブジェクト解放**: PyObject_Free(self) を呼び出し、Python オブジェクトのメモリを解放します。 4. **型参照カウントの調整**: Python 3.8 以降では、Py_DECREF(tp) を実行して型オブジェクトの参照カウントを減らします。

呼び出し元と呼び出し先

  • 呼び出し元: Python の C API（Py_tp_dealloc スロット経由で呼び出されることが示唆されています）。
  • 呼び出し先: GetPortalByName, PortalIsValid, UnpinPortal, SPI_cursor_close, MemoryContextDelete, PyObject_Free, Py_DECREF。

関連データ構造

  • PLyCursorObject: カーソルの状態（portalname, closed, mcxt）を保持する構造体。

失敗モードと境界条件

  • self が既にクローズされている場合、ポータルのクローズ処理はスキップされます。
  • self->mcxt が NULL の場合、メモリコンテキストの解放処理はスキップされます。

保守担当者向けメモ

  • 本関数は Python のデストラクタとして機能するため、メモリリークを防ぐために mcxt の解放と PyObject_Free の順序を維持してください。
  • Python 3.8 以降の型オブジェクト参照カウント管理については、Python の公式イシュー（#35810）に関連する変更であるため、安易な削除は避けてください。
