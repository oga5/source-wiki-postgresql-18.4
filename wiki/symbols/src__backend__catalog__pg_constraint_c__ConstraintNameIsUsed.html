ConstraintNameIsUsed

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_constraint.c:411

目的

指定されたリレーションまたはドメインにおいて、特定の制約名が既に使用されているかどうかを判定します。

シグネチャまたは形状

bool ConstraintNameIsUsed(ConstraintCategory conCat, Oid objId, const char *conname)

入力と出力

  • **入力**:
  • conCat: 制約のカテゴリ（CONSTRAINT_RELATION または CONSTRAINT_DOMAIN）。
  • objId: 対象となるリレーションまたはドメインの OID。
  • conname: チェック対象の制約名。
  • **出力**:
  • bool: 指定された名前が使用されていれば true、そうでなければ false を返します。

前提条件と不変条件

  • pg_constraint システムカタログに対する AccessShareLock を取得して操作を行います。
  • 呼び出し元は、適切なロックレベルでカタログ操作を行う必要があります。

アルゴリズムまたは振る舞い

1. pg_constraint テーブルを AccessShareLock で開きます。 2. ConstraintRelidTypidNameIndexId インデックスを使用し、conrelid（リレーションの場合）または contypid（ドメインの場合）と conname をキーとしてスキャン条件を構築します。 3. systable_beginscan を実行し、一致するタプルが存在するかを確認します。 4. systable_getnext でタプルが取得できれば true、できなければ false を返します。 5. スキャン終了後、テーブルをクローズして結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c
  • src/backend/catalog/index.c
  • src/backend/catalog/pg_constraint.c
  • src/backend/commands/tablecmds.c
  • src/backend/commands/typecmds.c
  • **呼び出し先**:
  • table_open, table_close
  • ScanKeyInit
  • systable_beginscan, systable_getnext, systable_endscan

関連データ構造

  • pg_constraint: 制約情報を保持するシステムカタログ。
  • ConstraintCategory: 制約の対象種別を定義する列挙型。

失敗モードと境界条件

  • 指定された objId が無効な場合や、カタログへのアクセス権限がない場合はエラーが発生する可能性があります。
  • 内部的なスキャン処理において、インデックスが破損している場合は予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は制約名の重複チェックに特化しています。制約名の生成が必要な場合は ChooseConstraintName を参照してください。
  • pg_constraint のインデックス構成に変更があった場合は、ScanKey の設定箇所を更新する必要があります。
