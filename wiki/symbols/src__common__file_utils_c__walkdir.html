walkdir

定義

  • 種別: function
  • 定義箇所: src/common/file_utils.c:289

目的

ディレクトリツリーを再帰的に走査し、各ファイルおよびディレクトリに対して指定されたアクション（fsync 等）を実行する。

シグネチャまたは形状

static void walkdir(const char *path,
                    int (*action) (const char *fname, bool isdir),
                    bool process_symlinks,
                    const char *exclude_dir);

入力と出力

  • path: 走査を開始するディレクトリパス。
  • action: 各エントリに対して実行する関数ポインタ。第2引数 isdir は対象がディレクトリの場合に true となる。
  • process_symlinks: シンボリックリンクを追跡するかどうか。
  • exclude_dir: 走査から除外するディレクトリパス。
  • 出力: なし（void）。

前提条件と不変条件

  • path で指定されたディレクトリはオープン可能であること。
  • action 関数は、ファイルまたはディレクトリのパスと、それがディレクトリであるかを示すフラグを受け取り、成功時に 0 を返すことが期待される。
  • 走査中、. および .. エントリはスキップされる。

アルゴリズムまたは振る舞い

1. exclude_dir と現在の path が一致する場合、即座に終了する。 2. opendir でディレクトリを開き、readdir でエントリを順次読み込む。 3. get_dirent_type を用いてエントリの型を判定する。

  • PGFILETYPE_REG (通常ファイル): action を実行。
  • PGFILETYPE_DIR (ディレクトリ): 再帰的に walkdir を呼び出す。
  • その他: 無視する。

    • 4. 読み込み完了後、closedir でディレクトリを閉じる。 5. 最後に、ディレクトリ自身に対して action を実行する（ディレクトリメタデータの永続化を保証するため）。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/file/fd.c 内の各関数（pre_sync_fname や datadir_fsync_fname をアクションとして渡す例が確認されている）。
  • 呼び出し先: opendir, readdir, closedir, get_dirent_type, snprintf, strcmp, pg_log_error。

関連データ構造

  • DIR: ディレクトリストリーム構造体。
  • struct dirent: ディレクトリ内のエントリ情報。
  • PGFILETYPE_REG, PGFILETYPE_DIR: get_dirent_type が返すファイル型定数。

失敗モードと境界条件

  • ディレクトリのオープン失敗時: pg_log_error を出力して終了する。
  • ディレクトリ読み込み中のエラー: errno を確認し、pg_log_error を出力する。
  • fsync の重要性: 個別のファイル同期だけではディレクトリ内のエントリ変更が保証されないため、最後にディレクトリ自身に対して action を実行する設計となっている。

保守担当者向けメモ

  • 本関数はフロントエンド用であり、バックエンド版の walkdir とは実装が異なる点に注意すること。
  • action 関数がディレクトリに対して実行される際は、isdir 引数が true となる。
  • MAXPGPATH * 2 のバッファサイズが subpath に使用されている。
