AutoVacuumSignal

定義

  • 種別: enum
  • 定義箇所: src/backend/postmaster/autovacuum.c:249

目的

自動バキュームシステム内において、ランチャープロセスやワーカープロセス間での非同期的なイベント通知や状態変更を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
	AutoVacForkFailed,			/* failed trying to start a worker */
	AutoVacRebalance,			/* rebalance the cost limits */
} AutoVacuumSignal;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし（列挙型定義）

前提条件と不変条件

  • AutoVacForkFailed: ワーカープロセスの起動試行が失敗した際に使用されます。
  • AutoVacRebalance: バキュームコスト制限の再計算（リバランス）が必要な際に使用されます。

アルゴリズムまたは振る舞い

本シンボルは、自動バキュームの制御フローにおいて、特定のイベントが発生したことを示すフラグとして機能します。提供されたソース情報からは、具体的なシグナル送信関数や受信処理の詳細は特定できません。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは、この列挙型を参照している具体的な関数は特定できません。
  • 呼び出し先: なし（列挙型定義のため）。

関連データ構造

  • AutoVacuumShmemStruct: 自動バキュームの共有メモリ制御構造体。本シグナルに関連する状態が保持されている可能性があります。

失敗モードと境界条件

  • 列挙型定義のため、本シンボル自体に失敗モードはありません。ただし、本シグナルが示すイベント（起動失敗やリバランス要求）は、システムのリソース状況や負荷に依存します。

保守担当者向けメモ

  • 本シンボルが実際にどの共有メモリ変数やシグナルハンドラで使用されているかについては、src/backend/postmaster/autovacuum.c 内の具体的なロジックを参照してください。
