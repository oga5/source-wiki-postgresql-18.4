xl_xact_xinfo

定義

  • 種別: struct
  • 定義箇所: src/include/access/xact.h:244

目的

WAL（Write-Ahead Logging）レコードにおいて、トランザクションに関する追加情報（xinfo）を保持するための構造体です。

シグネチャまたは形状

typedef struct xl_xact_xinfo
{
    uint32      xinfo;
} xl_xact_xinfo;

入力と出力

  • **入力**: WALレコードのデータストリームの一部として、特定のフラグ（XLOG_XACT_HAS_INFO）がセットされている場合にこの構造体が後続します。
  • **出力**: xinfo フィールドに格納されたビットフラグにより、トランザクションのメタデータや後続するレコードの有無を識別します。

前提条件と不変条件

  • xinfo フィールドは4バイト（uint32）で定義されており、後続するレコードのアライメントを考慮して設計されています。
  • 構造体の各フィールドは size(int) の倍数であることが推奨されます。

アルゴリズムまたは振る舞い

  • トランザクションのコミットやアボートのWALレコードにおいて、追加情報が存在することを示すフラグが立っている場合に、この構造体がレコード内に配置されます。
  • WAL解析時、data ポインタから本構造体をキャストして読み込み、その後 data += sizeof(xl_xact_xinfo) を実行してポインタを進めることで、後続のデータへアクセスします。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/rmgrdesc/xactdesc.c: WALレコードの記述および解析処理で使用。
  • src/backend/access/transam/xact.c: トランザクション終了時のWAL記録処理で使用。
  • **呼び出し先**:
  • 構造体自体はデータ保持用であり、関数呼び出しは含みません。

関連データ構造

  • xl_xact_commit: コミットレコードのメイン構造体。
  • xl_xact_abort: アボートレコードのメイン構造体。
  • XLOG_XACT_HAS_INFO: 本構造体の存在をWALレコード内で示すためのフラグ。

失敗モードと境界条件

  • WALレコードの解析時に、XLOG_XACT_HAS_INFO フラグの確認を怠ると、データストリームの読み取り位置がずれ、不正なメモリ参照や解析エラーを引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は現在2バイト分の情報しか必要としませんが、アライメント調整のため4バイト確保されています。将来的にフィールドを追加する場合も、このアライメント制約を維持してください。
  • 可変長データを含むレコードの解析時は、必ず Parse...Record 系関数を経由して安全にアクセスしてください。
