get_delete_query_def

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ruleutils.c:7354

目的

DELETE 文のパースツリー（Query 構造体）を、人間が読み取り可能な SQL ソーステキストに逆コンパイル（deparse）する。

シグネチャまたは形状

static void get_delete_query_def(Query *query, deparse_context *context)

入力と出力

  • **入力**:
  • query: 逆コンパイル対象の Query 構造体。
  • context: 逆コンパイルの状態（出力バッファやインデント設定など）を保持する deparse_context 構造体。
  • **出力**: なし（context->buf に SQL 文字列が追記される）。

前提条件と不変条件

  • query は DELETE 文を表す有効な Query 構造体であること。
  • context は初期化済みであること。
  • query->resultRelation が指す RangeTblEntry は RTE_RELATION である必要がある（Assert で検証済み）。

アルゴリズムまたは振る舞い

1. **WITH 句**: get_with_clause を呼び出し、存在すれば出力する。 2. **DELETE 句**: rt_fetch で対象リレーションを取得し、DELETE FROM [ONLY] relname をバッファに書き込む。 3. **エイリアス**: get_rte_alias を呼び出し、必要に応じてリレーションのエイリアスを出力する。 4. **USING 句**: get_from_clause を呼び出し、USING 句が存在すれば出力する。 5. **WHERE 句**: query->jointree->quals が存在する場合、 WHERE を付与し、get_rule_expr を用いて条件式を再帰的に逆コンパイルする。 6. **RETURNING 句**: query->returningList が存在する場合、get_returning_clause を呼び出して出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/ruleutils.c 内のクエリ逆コンパイル処理（pg_get_querydef 等の内部フローから参照）。
  • **呼び出し先**:
  • get_with_clause, rt_fetch, appendStringInfoChar, appendStringInfo, only_marker, generate_relation_name, get_rte_alias, get_from_clause, appendContextKeyword, get_rule_expr, get_returning_clause

関連データ構造

  • Query: 逆コンパイル対象のクエリツリー。
  • deparse_context: 逆コンパイルの出力先バッファおよび整形設定を保持。
  • RangeTblEntry: クエリ内のテーブル参照情報。

失敗モードと境界条件

  • query->resultRelation が RTE_RELATION ではない場合、Assert によりデバッグビルドで停止する。
  • jointree->quals が NULL の場合、WHERE 句は省略される。
  • returningList が空の場合、RETURNING 句は省略される。

保守担当者向けメモ

  • 本関数は ruleutils.c における DELETE 文専用の逆コンパイルロジックである。
  • PRETTY_INDENT マクロによる整形制御が含まれており、インデントレベルの変更が context に反映される点に注意すること。
