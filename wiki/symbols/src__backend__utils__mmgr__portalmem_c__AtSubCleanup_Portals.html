AtSubCleanup_Portals

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/portalmem.c:1091

目的

サブトランザクションの終了（アボート等）に伴い、そのサブトランザクション内で作成されたポータルを特定し、強制的に破棄およびクリーンアップを行う。

シグネチャまたは形状

void AtSubCleanup_Portals(SubTransactionId mySubid)

入力と出力

  • **入力**: mySubid (クリーンアップ対象となるサブトランザクションのID)
  • **出力**: なし（副作用としてポータルが破棄される）

前提条件と不変条件

  • PortalHashTable が初期化されていること。
  • 指定された mySubid に関連付けられたポータルのみが処理対象となる。

アルゴリズムまたは振る舞い

1. PortalHashTable を走査し、各ポータルの createSubid が引数の mySubid と一致するか確認する。 2. 一致するポータルに対して以下の処理を行う：

  • portalPinned が true の場合、強制的に false に設定する。
  • ポータルにクリーンアップフック（cleanup）が設定されている場合、ユーザー定義コードの実行を避けるため、警告を出力してフックを無効化（NULL に設定）する。
  • PortalDrop(portal, false) を呼び出し、ポータルを破棄する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c:5393
  • **呼び出し先**: hash_seq_init, hash_seq_search, PortalDrop, elog

関連データ構造

  • PortalHashTable: 全ポータルを管理するハッシュテーブル。
  • PortalHashEnt: ハッシュテーブルの各エントリ。
  • Portal: ポータルの状態を保持する構造体。

失敗モードと境界条件

  • portalPinned が設定されているポータルであっても、サブトランザクションのクリーンアップ時には強制的にピンを解除して破棄する。
  • クリーンアップフックが設定されている場合、実行せずにスキップし、警告ログを出力する。

保守担当者向けメモ

  • 本関数はサブトランザクションの異常終了時に呼び出されることを想定している。
  • PortalDrop を呼び出す前にピンを強制解除する設計となっているため、呼び出し側でポータルの生存期間を厳密に管理する必要がある。
