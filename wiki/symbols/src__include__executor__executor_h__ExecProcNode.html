ExecProcNode

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/executor/executor.h:309

目的

プランツリーの特定のノードを実行し、次のタプル（TupleTableSlot）を取得する。実行エンジンにおけるタプル伝播の主要なインターフェースである。

シグネチャまたは形状

static inline TupleTableSlot *ExecProcNode(PlanState *node)

入力と出力

  • **入力**: PlanState *node - 実行対象のプランノードの状態構造体。
  • **出力**: TupleTableSlot * - 処理結果として得られたタプルを格納するスロット。

前提条件と不変条件

  • node は有効な PlanState 構造体を指している必要がある。
  • node->ExecProcNode 関数ポインタが適切に初期化されていること。

アルゴリズムまたは振る舞い

1. node->chgParam が NULL でない場合、パラメータの変更を検知し、ExecReScan(node) を呼び出してノードを再スキャン状態にする。 2. ノード固有の実行関数である node->ExecProcNode(node) を呼び出し、その戻り値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/executor/execMain.c
  • src/backend/executor/execProcnode.c（再帰的な呼び出しを含む）
  • **呼び出し先**:
  • ExecReScan（chgParam が設定されている場合）
  • node->ExecProcNode（各ノード型固有の実装関数）

関連データ構造

  • PlanState: 実行中のプランノードの状態を保持する構造体。
  • TupleTableSlot: タプルデータを保持するコンテナ。

失敗モードと境界条件

  • node が NULL の場合、あるいは node->ExecProcNode が初期化されていない場合はセグメンテーション違反が発生する可能性がある。
  • ノードがタプルを生成し尽くした（EOF）場合、各ノードの実装に従い空のスロットまたは NULL が返される。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、static inline として定義されている。変更時はパフォーマンスへの影響を考慮すること。
  • ExecProcNode は実行エンジンのディスパッチハブであり、ここから ExecNestLoop 等の各ノード処理へ制御が移る。デバッグ時はこの関数にブレークポイントを置くことで、プランツリー全体の実行フローを追跡可能である。
