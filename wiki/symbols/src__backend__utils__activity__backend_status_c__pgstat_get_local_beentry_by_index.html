pgstat_get_local_beentry_by_index

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/backend_status.c:1278

目的

共有メモリからローカルメモリへ最新のバックエンド状態を同期（スナップショット作成）した上で、指定されたインデックスに対応するローカルなバックエンド状態エントリへのポインタを取得する。

シグネチャまたは形状

LocalPgBackendStatus *pgstat_get_local_beentry_by_index(int idx)

入力と出力

  • **入力**: idx (1から始まるバックエンドのインデックス)
  • **出力**: 指定されたインデックスが有効な範囲（1 ～ localNumBackends）であれば LocalPgBackendStatus 構造体へのポインタを返し、範囲外であれば NULL を返す。

前提条件と不変条件

  • 本関数を呼び出す前に、共有メモリ上の状態がローカルメモリへコピーされている必要がある（関数内で pgstat_read_current_status() を呼び出し、この不変条件を保証している）。
  • idx は 1-based インデックスとして扱われる。

アルゴリズムまたは振る舞い

1. pgstat_read_current_status() を呼び出し、共有メモリ上の全バックエンド状態をローカルの localBackendStatusTable にコピーする。 2. 引数 idx が 1 未満、または localNumBackends を超えている場合は無効なインデックスとみなし NULL を返す。 3. 有効な場合、localBackendStatusTable[idx - 1] のアドレスを返す。

呼び出し元と呼び出し先

  • **呼び出し先**: pgstat_read_current_status()
  • **呼び出し元**:
  • src/backend/utils/adt/pgstatfuncs.c 内の複数の関数（pg_stat_get_activity 等の統計情報取得関数）から直接呼び出されている。

関連データ構造

  • LocalPgBackendStatus: バックエンドの状態を保持するローカルメモリ上の構造体。
  • localBackendStatusTable: 全バックエンドの状態を格納するローカル配列。
  • localNumBackends: 現在保持しているバックエンドの総数。

失敗モードと境界条件

  • idx が 1 未満、または localNumBackends を超える場合、関数は NULL を返す。
  • 共有メモリからの読み取り（pgstat_read_current_status）において、楽観的並行制御によるリトライが発生する可能性がある。

保守担当者向けメモ

  • 本関数は 1-based インデックスを内部の 0-based 配列アクセスに変換するラッパーとして機能する。
  • 呼び出し元は、返されたポインタが NULL である可能性を常に考慮する必要がある。
