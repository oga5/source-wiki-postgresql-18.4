tt_process_call

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/wparser.c:76

目的

SQLのセットを返す関数（SRF）の実行中、キャッシュされたトークン型リストから現在の要素をタプルとして順次生成し、呼び出し元へ返却する。

シグネチャまたは形状

static Datum tt_process_call(FuncCallContext *funcctx)

入力と出力

  • **入力**: FuncCallContext *funcctx（SRFの状態を保持するコンテキスト。user_fctx に TSTokenTypeStorage へのポインタが格納されている必要がある）。
  • **出力**: Datum（成功時は構築されたタプル、リストの末尾に達した場合は (Datum) 0）。

前提条件と不変条件

  • funcctx->user_fctx が TSTokenTypeStorage 型の有効なポインタを指していること。
  • TSTokenTypeStorage 内の list が初期化済みであり、cur が現在のインデックスを正しく保持していること。

アルゴリズムまたは振る舞い

1. funcctx->user_fctx から TSTokenTypeStorage 構造体を取得する。 2. リストの現在位置（st->cur）に有効な lexid が存在するか確認する。 3. 存在する場合:

  • lexid、alias、descr を文字列配列に変換する。
  • BuildTupleFromCStrings を使用してタプルを構築する。
  • pfree で一時的な文字列領域を解放する。
  • st->cur をインクリメントし、構築したタプルを Datum として返す。

    • 4. 存在しない場合（リストの終端）:

  • (Datum) 0 を返し、SRFの終了を通知する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/wparser.c:119 (ts_token_type_byid 内)
  • src/backend/tsearch/wparser.c:142 (ts_token_type_byname 内)
  • **呼び出し先**:
  • BuildTupleFromCStrings（PostgreSQL内部関数）
  • pfree（PostgreSQLメモリ管理関数）

関連データ構造

  • FuncCallContext: SRFの状態管理用構造体。
  • TSTokenTypeStorage: パーサーのトークン型情報を保持する構造体。

失敗モードと境界条件

  • リストの終端に達した際は (Datum) 0 を返すことで安全に終了する。
  • funcctx が不正な場合や、user_fctx が初期化されていない場合はセグメンテーションフォールト等の未定義動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は ts_token_type_byid および ts_token_type_byname からのみ呼び出される。
  • values 配列の各要素は BuildTupleFromCStrings に渡されるため、メモリリークを防ぐために pfree の呼び出し順序と対象を維持すること。
