FreeVfd

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/fd.c:1475

目的

VFD（仮想ファイル記述子）キャッシュ内の特定のファイルレコードを解放し、そのスロットを空きリストへ戻す。

シグネチャまたは形状

static void FreeVfd(File file)

入力と出力

  • **入力**: file (File 型) - 解放対象となるVFDキャッシュのインデックス。
  • **出力**: なし（副作用としてキャッシュ状態を更新）。

前提条件と不変条件

  • file は VfdCache 配列の有効なインデックスである必要がある。
  • VfdCache[0] はLRUリングのアンカーとして機能しており、本関数はこれを利用して空きリストを管理する。

アルゴリズムまたは振る舞い

1. VfdCache[file] を参照し、デバッグログを出力する。 2. vfdP->fileName が割り当てられている場合、メモリを free し、ポインタを NULL に設定する。 3. vfdP->fdstate を 0x0 にリセットする。 4. 現在の VfdCache[0].nextFree を vfdP->nextFree に代入し、VfdCache[0].nextFree を現在の file インデックスで更新することで、当該スロットを空きリストの先頭に繋ぎ直す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/file/fd.c:1630
  • src/backend/storage/file/fd.c:2071
  • **呼び出し先**:
  • free (標準ライブラリ)
  • elog (PostgreSQLログ出力マクロ)

関連データ構造

  • Vfd: ファイルの状態を保持する構造体。
  • VfdCache: Vfd 構造体の配列。VfdCache[0] は空きリストの管理用アンカーとして使用される。

失敗モードと境界条件

  • file が無効なインデックスである場合、メモリ破壊や不正なメモリアクセスが発生する可能性がある。
  • vfdP->fileName が既に NULL の場合、free は呼び出されない（安全）。

保守担当者向けメモ

  • 本関数はVFDスロットの再利用を可能にするための低レベルなクリーンアップ処理である。
  • VfdCache[0] を空きリストの起点として使用する設計であるため、リスト操作の整合性を維持すること。
