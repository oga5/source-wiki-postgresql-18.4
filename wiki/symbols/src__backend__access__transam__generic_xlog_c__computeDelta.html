computeDelta

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/generic_xlog.c:227

目的

ページ変更の差分（デルタ）を計算し、GenericXLogPageData に記録する。これにより、ページ全体ではなく変更箇所のみをWALに格納し、WALサイズを削減する。

シグネチャまたは形状

static void computeDelta(GenericXLogPageData *pageData, Page curpage, Page targetpage)

入力と出力

  • pageData: 差分データおよびその長さを格納する出力用構造体。
  • curpage: 変更前のページイメージ。
  • targetpage: 変更後のページイメージ。
  • 戻り値: なし（pageData を更新）。

前提条件と不変条件

  • curpage および targetpage は有効なページイメージであること。
  • ページ内の pd_lower から pd_upper までの領域（空き領域）は、WAL適用時にゼロクリアされる前提で無視される。

アルゴリズムまたは振る舞い

1. pageData->deltaLen を 0 に初期化する。 2. ページ内の有効領域を「下部（0 から pd_lower）」と「上部（pd_upper から BLCKSZ）」に分割する。 3. computeRegionDelta を呼び出し、各領域の差分を計算して pageData に追加する。 4. WAL_DEBUG が有効な場合、計算されたデルタを curpage に適用し、結果が targetpage と一致するか検証する。不一致の場合はエラーを発生させる。

呼び出し元と呼び出し先

  • 呼び出し元: GenericXLogFinish（src/backend/access/transam/generic_xlog.c:370）
  • 呼び出し先: computeRegionDelta、およびデバッグ有効時は applyPageRedo

関連データ構造

  • GenericXLogPageData: 差分データとメタデータを保持する構造体。
  • PageHeader: ページ内の pd_lower および pd_upper を参照するために使用。

失敗モードと境界条件

  • WAL_DEBUG が有効な環境において、デルタの適用結果が targetpage と一致しない場合、elog(ERROR, ...) により処理が中断される。

保守担当者向けメモ

  • 本関数はページ内の「ホール（pd_lower〜pd_upper）」を意図的に無視する。これは、異なる環境でページを再構築した際に、ゴミデータによる不一致を防ぐための設計である。
  • デルタ圧縮の効率は computeRegionDelta の実装に依存する。
