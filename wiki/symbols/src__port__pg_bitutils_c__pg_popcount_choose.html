pg_popcount_choose

定義

  • 種別: function
  • 定義箇所: src/port/pg_bitutils.c:201

目的

ポピュレーションカウント（ビットカウント）処理の実行時最適化を初期化し、最適化された実装へ処理を委譲する。

シグネチャまたは形状

static uint64 pg_popcount_choose(const char *buf, int bytes)

入力と出力

  • 入力:
  • buf: カウント対象のデータが格納されたバッファへのポインタ。
  • bytes: バッファのサイズ（バイト単位）。
  • 出力:
  • uint64: バッファ内の1のビットの総数。

前提条件と不変条件

  • 本関数は、pg_popcount_optimized 関数ポインタが初期化されていない初回呼び出し時に実行されることを想定している。
  • choose_popcount_functions() が呼び出されることで、CPU機能に応じた最適な実装が選択される。

アルゴリズムまたは振る舞い

1. choose_popcount_functions() を呼び出し、CPUの機能判定（POPCNT命令の有無等）と関数ポインタの切り替えを行う。 2. 切り替え後の pg_popcount_optimized を呼び出し、バッファ内のビット数をカウントしてその結果を返す。

呼び出し元と呼び出し先

  • 呼び出し元: pg_popcount_optimized 関数ポインタの初期値として参照されている。
  • 呼び出し先:
  • choose_popcount_functions()
  • pg_popcount_optimized（切り替え後の実体）

関連データ構造

  • pg_popcount_optimized: 本関数が初期化時に書き換える関数ポインタ。

失敗モードと境界条件

  • bytes が 0 の場合、pg_popcount_optimized の実装に依存するが、通常は 0 を返す。
  • CPU機能判定が失敗または未サポートの場合、汎用的なフォールバック実装が選択される。

保守担当者向けメモ

  • 本関数は pg_popcount_optimized の初期値として設定されるため、直接呼び出されることは稀である。
  • src/port/pg_popcount_aarch64.c 等の別ファイルでも同名の関数が定義されており、コンパイル時の環境に応じて適切な実装がリンクされる。
