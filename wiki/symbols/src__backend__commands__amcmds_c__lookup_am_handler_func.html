lookup_am_handler_func

定義

  • 種別: function
  • 定義箇所: src/backend/commands/amcmds.c:233

目的

アクセスメソッド（AM）のハンドラ関数を名前で検索し、その戻り値の型が指定された AM タイプ（インデックスまたはテーブル）に対して適切であるかを検証する。

シグネチャまたは形状

static Oid lookup_am_handler_func(List *handler_name, char amtype)

入力と出力

  • **入力**:
  • handler_name: ハンドラ関数名を指定する List。
  • amtype: AM の種類を示す文字（AMTYPE_INDEX または AMTYPE_TABLE）。
  • **出力**:
  • 成功時: 検索されたハンドラ関数の Oid。
  • 失敗時: エラーが発生し、例外がスローされる。

前提条件と不変条件

  • ハンドラ関数は internal 型の引数を 1 つ取る必要がある。
  • amtype は AMTYPE_INDEX または AMTYPE_TABLE のいずれかである必要がある。

アルゴリズムまたは振る舞い

1. handler_name が NIL の場合、エラーを報告する。 2. LookupFuncName を使用し、internal 型の引数を 1 つ持つ関数として handler_name を検索する。 3. amtype に基づき、期待される戻り値の型（INDEX_AM_HANDLEROID または TABLE_AM_HANDLEROID）を決定する。 4. get_func_rettype を使用して、検索した関数の戻り値の型が期待値と一致するか確認する。一致しない場合はエラーを報告する。 5. 成功した場合、関数の Oid を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/amcmds.c 内の CreateAccessMethod 関数から呼び出される。
  • **呼び出し先**: LookupFuncName、get_func_rettype、get_func_name、format_type_extended を呼び出す。

関連データ構造

  • Oid: PostgreSQL のオブジェクト識別子。
  • List: 関数名を保持するリスト構造。
  • AMTYPE_INDEX / AMTYPE_TABLE: アクセスメソッドの種類を識別する定数。
  • INDEX_AM_HANDLEROID / TABLE_AM_HANDLEROID: ハンドラ関数が返すべき期待される型 OID。

失敗モードと境界条件

  • handler_name が空の場合: ERRCODE_UNDEFINED_FUNCTION でエラー。
  • 関数が見つからない場合: LookupFuncName によりエラー。
  • 戻り値の型が不一致の場合: ERRCODE_WRONG_OBJECT_TYPE でエラー。
  • 未定義の amtype が渡された場合: elog(ERROR) によりエラー。

保守担当者向けメモ

  • 本関数は CREATE ACCESS METHOD 文の処理において、ハンドラ関数の妥当性を保証する重要なバリデーション層である。
  • 新しい AM タイプを追加する場合は、switch 文に新しいケースを追加し、対応する期待される戻り値の型を定義する必要がある。
