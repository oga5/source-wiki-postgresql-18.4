bloom_prop_bits_set

定義

  • 種別: function
  • 定義箇所: src/backend/lib/bloomfilter.c:186

目的

ブルームフィルタのビットセットにおいて、現在セットされている（1である）ビットの割合を算出します。フィルタの飽和度を評価するために使用されます。

シグネチャまたは形状

double bloom_prop_bits_set(bloom_filter *filter)

入力と出力

  • **入力**: bloom_filter *filter - 状態を保持するブルームフィルタ構造体へのポインタ。
  • **出力**: double - ビットセット全体のうち、セットされているビットの割合（0.0 から 1.0 の範囲）。

前提条件と不変条件

  • filter は有効なブルームフィルタ構造体である必要があります。
  • filter->m はビットセットの総ビット数であり、filter->bitset はそのメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. filter->m をバイト単位に変換し、ビットセットのバイトサイズを算出します。 2. pg_popcount を使用して、ビットセット全体でセットされているビットの総数（bits_set）をカウントします。 3. bits_set を総ビット数 filter->m で除算し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_nbtree.c:594
  • src/test/modules/test_bloomfilter/test_bloomfilter.c:98
  • **呼び出し先**:
  • pg_popcount (外部ユーティリティ)

関連データ構造

  • bloom_filter: フィルタの状態（ビットセット、サイズ m 等）を保持する構造体。

失敗モードと境界条件

  • filter が NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • filter->m が 0 の場合、ゼロ除算が発生する可能性があります。

保守担当者向けメモ

  • 理想的な飽和度は 0.5 前後です。この関数はデバッグやフィルタの効率性評価に有用です。
  • pg_popcount を利用しているため、プラットフォーム固有の最適化されたビットカウント処理が適用されます。
