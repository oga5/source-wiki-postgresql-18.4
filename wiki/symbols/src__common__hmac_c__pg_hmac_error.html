pg_hmac_error

定義

  • 種別: function
  • 定義箇所: src/common/hmac.c:305

目的

HMAC 計算コンテキストに関連付けられたエラーメッセージを取得します。

シグネチャまたは形状

const char *pg_hmac_error(pg_hmac_ctx *ctx);

入力と出力

  • **入力**: pg_hmac_ctx *ctx（HMAC 計算の状態を保持するコンテキストへのポインタ）。
  • **出力**: エラー内容を示す静的な文字列（const char *）。

前提条件と不変条件

  • ctx が NULL の場合、メモリ不足（"out of memory"）を示す文字列を返します。
  • ctx が有効な場合、コンテキスト内に保持されているエラー理由（errreason）またはエラーコード（error）に基づいてメッセージを返します。

アルゴリズムまたは振る舞い

1. ctx が NULL であれば、即座に "out of memory" を返します。 2. ctx->errreason が設定されている場合、その文字列を優先して返します。 3. ctx->errreason がない場合、ctx->error の値に応じて以下のいずれかを返します。

  • PG_HMAC_ERROR_NONE: "success"
  • PG_HMAC_ERROR_INTERNAL: "internal error"
  • PG_HMAC_ERROR_OOM: "out of memory"

    • 4. 上記のいずれにも該当しない場合、Assert(false) により異常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth-scram.c
  • src/common/scram-common.c
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行いません。

関連データ構造

  • pg_hmac_ctx: HMAC 計算の状態を保持する不透明な構造体。

失敗モードと境界条件

  • ctx が NULL の場合はメモリ不足として扱われます。
  • 予期しない ctx->error 値が渡された場合は Assert により停止します。

保守担当者向けメモ

  • 本関数はローカライズされた文字列を返すため、呼び出し側で適切に処理してください。
  • src/common/hmac_openssl.c にも同名の関数が存在し、ビルド構成に応じて切り替わります。
