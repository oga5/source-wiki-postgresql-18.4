record_lt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rowtypes.c:1288

目的

2つのレコード（複合型）を比較し、左側のレコードが右側のレコードよりも小さいかどうかを判定します。

シグネチャまたは形状

Datum record_lt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（比較対象となる2つのレコードを含む関数呼び出し情報）。
  • **出力**: bool 型の Datum。左側のレコードが右側のレコードより小さい場合は true、それ以外は false を返します。

前提条件と不変条件

  • 内部的に record_cmp を呼び出すため、比較対象のレコード型が互換性を持つ必要があります。
  • record_cmp が実行時に型情報を解決し、各フィールドの比較関数を再帰的に適用します。

アルゴリズムまたは振る舞い

1. record_cmp(fcinfo) を呼び出し、比較結果の整数値を取得します。 2. 取得した値が 0 未満（負）であるかを確認します。 3. PG_RETURN_BOOL マクロを使用して、比較結果が 0 未満であれば true を、そうでなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL演算子（<）の実行時に呼び出されます。
  • **呼び出し先**: record_cmp を呼び出します。

関連データ構造

  • RecordCompareData: 比較実行時に使用される型情報や関数ポインタのキャッシュ構造体。

失敗モードと境界条件

  • 比較対象のレコードの型が一致しない場合や、比較不能な型が含まれる場合、record_cmp 経由でエラーが発生する可能性があります。
  • 削除された列（attisdropped）は比較ロジックにおいてスキップされます。

保守担当者向けメモ

  • 本関数は record_cmp のラッパーです。比較ロジックの変更が必要な場合は record_cmp を参照してください。
  • 再帰的なレコード比較を行うため、スタック深さの制限に注意が必要です。
