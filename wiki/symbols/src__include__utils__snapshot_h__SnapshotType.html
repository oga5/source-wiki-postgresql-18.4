SnapshotType

定義

  • 種別: enum
  • 定義箇所: src/include/utils/snapshot.h:31

目的

PostgreSQLにおけるタプルの可視性判定ルールを分類・定義する。

シグネチャまたは形状

typedef enum SnapshotType {
    SNAPSHOT_MVCC,
    SNAPSHOT_SELF,
    SNAPSHOT_ANY,
    SNAPSHOT_TOAST,
    SNAPSHOT_DIRTY,
    SNAPSHOT_HISTORIC_MVCC,
    SNAPSHOT_NON_VACUUMABLE
} SnapshotType;

入力と出力

  • 入力: 可視性判定を行う関数（HeapTupleSatisfiesMVCC 等）に対して、どのルールを適用すべきかを示す識別子として渡される。
  • 出力: SNAPSHOT_DIRTY の場合、判定対象の構造体が「出力引数」として機能し、進行中のトランザクションID（xmin, xmax）や推測的挿入トークンを呼び出し元に返す。

前提条件と不変条件

  • SNAPSHOT_NON_VACUUMABLE を使用する場合、事前に snapshot->min に適切な xmin ホライゾンが設定されている必要がある。
  • SNAPSHOT_DIRTY を使用する場合、snapshot 構造体のフィールドは可視性判定の結果によって上書きされる。

アルゴリズムまたは振る舞い

各列挙値は以下の可視性セマンティクスを規定する：

  • SNAPSHOT_MVCC: 標準的なMVCCルール。コミット済みトランザクションと現在のトランザクションの過去のコマンドを可視とする。
  • SNAPSHOT_SELF: 現在のトランザクションによる変更を含む、自身の可視性ルール。
  • SNAPSHOT_ANY: すべてのタプルを可視とする。
  • SNAPSHOT_TOAST: TOASTテーブル用の可視性ルール。
  • SNAPSHOT_DIRTY: 進行中のトランザクションを含むすべての変更を考慮する。特殊な挙動として、判定結果を構造体に書き戻す。
  • SNAPSHOT_HISTORIC_MVCC: 論理デコード等のタイムトラベルコンテキスト用。
  • SNAPSHOT_NON_VACUUMABLE: 真空掃除（VACUUM）の対象外となるタプルを判定する。

呼び出し元と呼び出し先

  • 呼び出し元: SnapshotData 構造体内で snapshot_type フィールドとして保持される（src/include/utils/snapshot.h:140）。
  • 呼び出し先: 提供されたソース抜粋には、この列挙型を直接呼び出す関数や、この型を引数に取る関数の具体的なリストは含まれていない。

関連データ構造

  • SnapshotData: この列挙型を保持し、可視性判定の状態を管理する構造体。

失敗モードと境界条件

  • SNAPSHOT_DIRTY 使用時、xmin/xmax がコミット済みまたは自身のトランザクションである場合、InvalidTransactionId が返される。
  • フィールドの解釈は snapshot_type に強く依存するため、型と構造体内容の不一致は誤った可視性判定を引き起こす。

保守担当者向けメモ

  • SnapshotData のフィールドは歴史的経緯により再利用されるため、snapshot_type に基づいた厳密なフィールド解釈が必要。
  • 将来的に NodeTag を用いたサブクラス化が検討されている。
