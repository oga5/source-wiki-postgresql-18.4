gimme_edge_table

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_erx.c:94

目的

2つの親個体（ツアー）からエッジ情報を抽出し、エッジテーブルを構築して、各都市間の接続関係を初期化・集計する。

シグネチャまたは形状

float gimme_edge_table(PlannerInfo *root, Gene *tour1, Gene *tour2, int num_gene, Edge *edge_table)

入力と出力

  • **入力**:
  • root: プランナ情報へのポインタ。
  • tour1, tour2: 交叉対象となる2つの親ツアー（遺伝子配列）。
  • num_gene: ツアー内の遺伝子（都市）数。
  • edge_table: エッジ情報を格納する配列（呼び出し元で確保済みであること）。
  • **出力**:
  • float: 各インデックスあたりの平均エッジ数を返す。

前提条件と不変条件

  • edge_table は num_gene 以上のサイズで確保されている必要がある。
  • ツアーは循環構造（1->2, 2->3, 3->1）として扱われる。
  • エッジは双方向（1->2 と 2->1 は同一）として処理される。

アルゴリズムまたは振る舞い

1. edge_table の全要素について total_edges と unused_edges を 0 にリセットする。 2. 各親ツアーを走査し、隣接する都市ペア（index1 と index2）を抽出する。 3. gimme_edge を呼び出し、各ペアの双方向エッジをテーブルに登録する。 4. 登録されたエッジの総数を累積し、最後に (edge_total * 2) / num_gene を計算して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/geqo/geqo_main.c
  • **呼び出し先**:
  • gimme_edge（各エッジの登録処理）

関連データ構造

  • Edge: 都市間の接続関係を保持する構造体。
  • Gene: ツアーを構成する遺伝子（都市）の型。

失敗モードと境界条件

  • num_gene が 0 または 1 の場合、ループ処理が正しく機能しない可能性がある（ソースコード上の明示的なガードはなし）。
  • gimme_edge が内部でメモリ不足や不正なインデックスに遭遇した場合の挙動は gimme_edge の実装に依存する。

保守担当者向けメモ

  • 本関数は ERX（エッジ組み換え交叉）アルゴリズムの初期化フェーズを担う。
  • edge_total の計算において、各エッジを双方向で2回ずつ gimme_edge に渡している点に注意すること。
