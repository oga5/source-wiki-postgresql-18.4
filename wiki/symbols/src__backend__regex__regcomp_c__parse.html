parse

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regcomp.c:717

目的

正規表現の構文を再帰的に解析し、subre（サブ式ツリー）構造を構築する。

シグネチャまたは形状

static struct subre *parse(struct vars *v, int stopper, int type, struct state *init, struct state *final)

入力と出力

  • **入力**:
  • v: コンパイル状態を保持する vars 構造体。
  • stopper: 解析を終了させる文字（')' または EOS）。
  • type: 解析対象の型（LACON または PLAIN）。
  • init, final: NFA の開始状態と終了状態。
  • **出力**:
  • 構築された subre 構造体へのポインタ。

前提条件と不変条件

  • stopper は ')' または EOS である必要がある。
  • NOERRN() マクロにより、エラー発生時は即座に NULL を返す。
  • 戻り値の subre は、指定された init および final 状態と論理的に整合している必要がある。

アルゴリズムまたは振る舞い

1. subre ノードを初期化し、init から final へのパスを準備する。 2. do-while ループ内で parsebranch() を呼び出し、正規表現の各枝（| で区切られた要素）を解析する。 3. 各枝を subre ツリーの兄弟ノードとして連結し、フラグを更新する。 4. EAT('|') が真である限りループを継続する。 5. 解析終了後、stopper が正しく消費されたかを確認する。 6. 最適化処理:

  • 枝が単一の場合は、中間ノードを削除して直接連結する。
  • 複雑な構造を含まない場合は、ノードを簡略化する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は、提供された抜粋からは特定できません。
  • **呼び出し先**:
  • subre(): ノードの生成。
  • newstate(): NFA 状態の生成。
  • EMPTYARC(): NFA 状態間の遷移追加。
  • parsebranch(): 各枝の解析。
  • freesrnode() / freesubreandsiblings(): メモリ解放。
  • NOERRN(), EAT(), SEE(), ERR(), UP(), MESSY(): マクロ群。

関連データ構造

  • struct subre: 正規表現の構造を表現するツリーノード。
  • struct vars: コンパイル中の状態を保持するコンテキスト。
  • struct state: NFA の状態。

失敗モードと境界条件

  • stopper が期待通りでない場合、REG_EPAREN エラーを発生させる。
  • NOERRN() マクロにより、内部の解析関数（parsebranch 等）でエラーが発生した場合、その状態を伝播して終了する。

保守担当者向けメモ

  • 本関数は regcomp.c における再帰的降下構文解析の核となる。
  • lastbranch の連結処理や branches->flags の更新ロジックは、正規表現の結合性と最適化に直結するため、変更時は注意が必要。
  • ツールが提示した contrib/ 配下の parse 関連シンボルは、本関数とは無関係な別個の識別子である可能性が高い。
