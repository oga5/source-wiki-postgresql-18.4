macaddr_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mac.c:139

目的

外部バイナリ形式のデータストリームから6バイトのMACアドレス（macaddr）を読み取り、内部表現へ変換します。

シグネチャまたは形状

Datum macaddr_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される StringInfo 型のバッファ。
  • **出力**: 変換された macaddr 構造体へのポインタを Datum 型として返却します。

前提条件と不変条件

  • 入力バッファには、少なくとも6バイトのデータが格納されている必要があります。
  • palloc を使用してメモリを確保するため、PostgreSQLのメモリコンテキストが有効である必要があります。

アルゴリズムまたは振る舞い

1. palloc を用いて macaddr 構造体分のメモリを確保します。 2. pq_getmsgbyte を6回連続で呼び出し、バッファから1バイトずつ順次読み取ります。 3. 読み取った各バイトを macaddr 構造体のメンバ a から f に順に格納します。 4. 構築した構造体を PG_RETURN_MACADDR_P マクロを介して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（Fmgr）経由で呼び出されます。具体的な呼び出し元コードの詳細は提供されていません。
  • **呼び出し先**:
  • palloc: メモリ確保に使用。
  • pq_getmsgbyte: libpq/pqformat.h で定義されたバイナリデータ読み取り関数。

関連データ構造

  • macaddr: 6バイトのMACアドレスを保持する構造体。
  • StringInfo: libpq/pqformat.h で定義される、バイナリデータ通信用のバッファ構造体。

失敗モードと境界条件

  • バッファの残量が6バイト未満の場合、pq_getmsgbyte がエラーを発生させ、処理が中断される可能性があります。

保守担当者向けメモ

  • 本関数はバイナリプロトコル専用です。テキスト形式の入力には macaddr_in が使用されます。
  • 構造体のメンバ a〜f への代入順序は、ネットワークバイトオーダー（ビッグエンディアン）のストリーム順序と一致しています。
