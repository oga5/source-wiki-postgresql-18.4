fetch_range_request

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_rewind/libpq_source.c:29

目的

リモートの PostgreSQL サーバーから取得すべきファイルデータの範囲（パス、オフセット、長さ）を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	const char *path;			/* データディレクトリからの相対パス */
	off_t		offset;			/* 読み取り開始位置 */
	size_t		length;			/* 読み取るバイト数 */
} fetch_range_request;

入力と出力

本構造体はデータ保持用であり、関数ではありません。libpq_source 構造体内の request_queue 配列の要素として使用されます。

前提条件と不変条件

  • path はデータディレクトリのルートからの相対パスである必要があります。
  • offset および length は、対象ファイルの有効な範囲内である必要があります。

アルゴリズムまたは振る舞い

本構造体自体はデータコンテナであり、ロジックを持ちません。libpq_queue_fetch_range 関数によってキューに追加され、process_queued_fetch_requests 関数によって、保持する情報に基づいた SQL クエリ（fetch_chunks_stmt）が生成・実行されます。

呼び出し元と呼び出し先

  • 呼び出し元: libpq_queue_fetch_range（キューへの追加）、process_queued_fetch_requests（キュー内のデータ参照）
  • 呼び出し先: なし（データ構造のため）

関連データ構造

  • libpq_source: 本構造体を配列（request_queue）として保持する親構造体。
  • MAX_CHUNKS_PER_QUERY: request_queue のサイズを決定する定数。

失敗モードと境界条件

  • path が NULL または無効なパスである場合、後続のクエリ実行時にリモートサーバー側でエラーが発生する可能性があります。
  • offset や length が負の値やファイルサイズを超える値である場合、リモートサーバーの pg_read_binary_file 実行時にエラーとなります。

保守担当者向けメモ

  • 本構造体は libpq を介したバッチ処理のために設計されています。フィールドを追加する場合は、process_queued_fetch_requests 内のクエリ生成ロジックとの整合性に注意してください。
