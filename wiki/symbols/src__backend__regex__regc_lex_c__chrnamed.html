chrnamed

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_lex.c:1021

目的

名前（文字列）に対応する文字（chr）を検索して返します。名前の解決に失敗した場合は、指定された代替値を返します。

シグネチャまたは形状

static chr chrnamed(struct vars *v, const chr *startp, const chr *endp, chr lastresort)

入力と出力

  • **入力**:
  • v: 正規表現コンパイル状態を保持する vars 構造体へのポインタ。
  • startp, endp: 検索対象の名前を表す文字列の開始位置と終了位置。
  • lastresort: 名前解決が失敗した場合に返す代替文字。
  • **出力**:
  • 解決された chr 型の文字、または解決失敗時の lastresort。

前提条件と不変条件

  • v->err の状態を一時的に退避・リセットして element 関数を呼び出し、処理後に元のエラー状態へ復元します。
  • 内部で呼び出される element および range 関数が正常に動作する環境である必要があります。

アルゴリズムまたは振る舞い

1. 現在の v->err を保存し、v->err を 0 にリセットします。 2. element(v, startp, endp) を呼び出し、名前から文字への変換を試みます。 3. 変換中にエラーが発生した場合、保存していたエラー状態を復元し、lastresort を返します。 4. エラーがない場合、range(v, c, c, 0) を呼び出して文字集合（cvec）を取得します。 5. 取得した集合が空であれば lastresort を返し、そうでなければ集合内の最初の文字を返します。 6. 最後に保存していたエラー状態を v->err に戻します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_lex.c:627
  • src/backend/regex/regc_lex.c:654
  • **呼び出し先**:
  • element (内部関数)
  • range (内部関数)

関連データ構造

  • struct vars: 正規表現コンパイルのコンテキストを保持。
  • struct cvec: 文字集合を保持する構造体。

失敗モードと境界条件

  • element 関数がエラーを返した場合、または range 関数が空の文字集合を返した場合は、引数 lastresort が戻り値として採用されます。

保守担当者向けメモ

  • 本関数は regcomp.c に展開される regc_lex.c 内で定義されています。
  • v->err の操作は、名前解決中のエラーが正規表現全体のコンパイルエラーとして誤って扱われないようにするための保護措置です。
