DirectFunctionCall9Coll

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/fmgr.c:1016

目的

9つの引数を取る関数を、FmgrInfo構造体の初期化を介さずに直接呼び出すためのオーバーヘッド最小化用ラッパー関数です。

シグネチャまたは形状

Datum DirectFunctionCall9Coll(PGFunction func, Oid collation, Datum arg1, Datum arg2, Datum arg3, Datum arg4, Datum arg5, Datum arg6, Datum arg7, Datum arg8, Datum arg9)

入力と出力

  • **入力**:
  • func: 実行対象の関数ポインタ（PGFunction）。
  • collation: 照合順序（Oid）。
  • arg1～arg9: 関数に渡す9つの引数（Datum）。
  • **出力**:
  • 関数の実行結果（Datum）。

前提条件と不変条件

  • 呼び出し元は、対象関数がNULLを返さないことを前提としています。
  • 引数はすべて非NULL（isnull = false）として扱われます。

アルゴリズムまたは振る舞い

1. LOCAL_FCINFOマクロを使用して、スタック上に9引数用のFunctionCallInfoBaseDataを確保します。 2. InitFunctionCallInfoDataを呼び出し、照合順序と引数情報を設定します。 3. 9つの引数すべてをfcinfo->args配列に格納し、isnullフラグをfalseに設定します。 4. 関数ポインタfuncをfcinfoを引数として呼び出します。 5. 実行結果を取得後、fcinfo->isnullを確認し、関数がNULLを返した場合はelog(ERROR)で異常終了します。 6. 正常終了時に結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/fmgr.h:699（マクロ経由の呼び出し）
  • **呼び出し先**: 引数として渡されたPGFunctionポインタが指す関数。

関連データ構造

  • FunctionCallInfoBaseData: 関数呼び出しのコンテキストを保持する構造体。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • **NULL戻り値**: 呼び出し先関数がNULLを返した場合、elog(ERROR)が発生します。
  • **ポリモーフィック関数**: 本関数はFmgrInfoを必要とするポリモーフィック関数には使用できません。

保守担当者向けメモ

  • 本関数はパフォーマンスが重要なパスで使用されます。FmgrInfoの再利用が不可能な場合にのみ利用してください。
  • 引数数が固定されているため、動的な引数処理が必要な場合は他のfmgrインターフェースを検討してください。
