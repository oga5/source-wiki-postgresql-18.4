output_deallocate_prepare_statement

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/output.c:178

目的

ECPGプリプロセッサにおいて、SQLの DEALLOCATE PREPARE 文に対応するCコードを生成し、出力ストリームへ書き出します。

シグネチャまたは形状

void output_deallocate_prepare_statement(const char *name)

入力と出力

  • **入力**:
  • name: 解放対象のプリペアードステートメント名（文字列）。"all" の場合は全ステートメントの解放を意味します。
  • **出力**:
  • なし（副作用として base_yyout ストリームへCコードを出力）。

前提条件と不変条件

  • グローバル変数 base_yyout が有効なファイルストリームとして初期化されていること。
  • グローバル変数 compat および connection が現在のコンパイル設定に従って設定されていること。

アルゴリズムまたは振る舞い

1. connection が未設定の場合は "NULL" を使用します。 2. name が "all" でない場合:

  • ECPGdeallocate 関数を呼び出すCコードを出力します。
  • output_escaped_str を使用してステートメント名をエスケープし、引数として渡します。

    • 3. name が "all" の場合:

  • ECPGdeallocate_all 関数を呼び出すCコードを出力します。

    • 4. 最後に whenever_action(2) を呼び出し、エラーハンドリング用のコードを生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は提供されていませんが、プリプロセッサの構文解析フェーズから呼び出されることが想定されます。
  • **呼び出し先**:
  • output_escaped_str（文字列のエスケープ処理）
  • whenever_action（エラーハンドリングコードの生成）
  • fprintf, fputs（標準ライブラリによる出力）

関連データ構造

  • base_yyout: 出力先のファイルストリーム。
  • compat: 互換性モードを保持するグローバル変数。
  • connection: 現在の接続名を保持するグローバル変数。

失敗モードと境界条件

  • name が "all" の場合とそれ以外で、呼び出すランタイム関数が異なります。
  • output_escaped_str 内で文字列内の特殊文字（" や \n など）が適切にエスケープされない場合、生成されるCコードが構文エラーとなる可能性があります。

保守担当者向けメモ

  • whenever_action(2) の引数 2 は、WHENEVER 処理のスコープ終了を制御するビットフラグです。
  • ECPGdeallocate および ECPGdeallocate_all はランタイムライブラリ側の関数であるため、インターフェースの変更時にはランタイム側の定義と整合性を取る必要があります。
