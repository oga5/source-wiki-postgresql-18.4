IoMethod

定義

  • 種別: enum
  • 定義箇所: src/include/storage/aio.h:32

目的

PostgreSQL の非同期 I/O (AIO) サブシステムにおいて、I/O 操作を実行するために使用する実装方式を選択するための列挙型です。

シグネチャまたは形状

typedef enum IoMethod
{
    IOMETHOD_SYNC = 0,
    IOMETHOD_WORKER,
#ifdef IOMETHOD_IO_URING_ENABLED
    IOMETHOD_IO_URING,
#endif
} IoMethod;

入力と出力

  • 入力: なし（列挙型の定義）
  • 出力: IOMETHOD_SYNC（同期実行）、IOMETHOD_WORKER（ワーカースレッド経由）、IOMETHOD_IO_URING（io_uring 使用、有効時のみ）のいずれかの値。

前提条件と不変条件

  • IOMETHOD_IO_URING は、コンパイル時に IOMETHOD_IO_URING_ENABLED が定義されている場合にのみ利用可能です。

アルゴリズムまたは振る舞い

  • IOMETHOD_SYNC: 同期的な I/O 処理を行います。
  • IOMETHOD_WORKER: 非同期 I/O をワーカースレッドに委譲して処理します。
  • IOMETHOD_IO_URING: Linux カーネルの io_uring インターフェースを使用して非同期 I/O を発行します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは特定できません。
  • 呼び出し先: なし（列挙型定義のため）。

関連データ構造

  • PgAioHandle: I/O 操作のコンテキストを保持する構造体で、この列挙型を使用して実装方式を管理します。

失敗モードと境界条件

  • io_uring は EXEC_BACKEND 環境と互換性がないため、当該環境下では IOMETHOD_IO_URING を選択できません。

保守担当者向けメモ

  • 新しい I/O 実装を追加する場合は、この列挙型に新しいメンバーを追加し、対応するロジックを src/backend/storage/aio/aio.c 等に実装してください。
