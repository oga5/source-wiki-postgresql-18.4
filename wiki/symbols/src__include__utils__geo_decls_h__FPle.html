FPle

定義

  • 種別: function
  • 定義箇所: src/include/utils/geo_decls.h:64

目的

浮動小数点数 A が B 以下であるかを、許容誤差 EPSILON を考慮して判定する。

シグネチャまたは形状

static inline bool FPle(double A, double B)

入力と出力

  • **入力**: 比較対象となる2つの double 型の値 A および B。
  • **出力**: A <= B + EPSILON が真であれば true、そうでなければ false を返す。

前提条件と不変条件

  • EPSILON を用いた曖昧な比較を行う。
  • NaN（非数）が含まれる場合、常に false を返す。
  • 推移律を満たさない可能性があるため、厳密な順序付けが必要な場面での使用には注意が必要である。

アルゴリズムまたは振る舞い

A と B + EPSILON を比較する。EPSILON を加算することで、浮動小数点演算における微小な誤差を許容した「以下」の判定を実現している。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistproc.c
  • src/backend/utils/adt/geo_ops.c
  • **呼び出し先**:
  • math.h（EPSILON の定義や浮動小数点演算に関連）

関連データ構造

  • 幾何データ型（Point, BOX 等）の座標比較に使用される。

失敗モードと境界条件

  • NaN が入力された場合、比較結果は常に false となる。
  • 非常に大きな値や極端に小さな値の比較において、EPSILON の影響により期待される数学的な順序と異なる結果を返す可能性がある。

保守担当者向けメモ

  • 本関数は数値解析の専門家によって設計されたものではない。厳密な数値計算が求められる箇所での利用は避けること。
  • geo_decls.h 内の他の比較関数（FPeq, FPne, FPlt, FPgt）と一貫した設計思想に基づいている。
