GetIndexAmRoutineByAmId

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/amapi.c:55

目的

指定されたアクセスメソッド（AM）のOIDに基づき、対応する IndexAmRoutine 構造体を取得します。

シグネチャまたは形状

IndexAmRoutine *GetIndexAmRoutineByAmId(Oid amoid, bool noerror)

入力と出力

  • **入力**:
  • amoid: 検索対象のアクセスメソッドのOID。
  • noerror: true の場合、検索失敗時にエラーを投げず NULL を返します。
  • **出力**:
  • 成功時: アクセスメソッドの関数ポインタを保持する IndexAmRoutine 構造体へのポインタ。
  • 失敗時: noerror が true なら NULL、それ以外はエラーを発生させます。

前提条件と不変条件

  • 呼び出し元は、返された IndexAmRoutine 構造体を適切にメモリ管理（解放）する必要があります。
  • pg_am システムカタログがアクセス可能である必要があります。

アルゴリズムまたは振る舞い

1. AMOID システムキャッシュを使用して pg_am カタログを検索します。 2. 検索結果が存在しない場合、または amtype が AMTYPE_INDEX でない場合、エラーを報告するか NULL を返します。 3. amhandler が有効なプロシージャであるかを確認します。 4. システムキャッシュを解放します。 5. 最後に GetIndexAmRoutine(amhandler) を呼び出し、AM固有のルーチン構造体を取得して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/index/amapi.c (内部の複数箇所)
  • src/backend/catalog/index.c
  • src/backend/commands/opclasscmds.c
  • **呼び出し先**:
  • SearchSysCache1
  • ReleaseSysCache
  • GetIndexAmRoutine
  • elog / ereport

関連データ構造

  • IndexAmRoutine: アクセスメソッドの操作関数ポインタを保持する構造体。
  • Form_pg_am: pg_am システムカタログの行構造。

失敗モードと境界条件

  • 指定した amoid が存在しない場合、noerror が false ならエラーとなります。
  • amtype が AMTYPE_INDEX ではない場合、エラーとなります。
  • amhandler が設定されていない（無効な）場合、エラーとなります。

保守担当者向けメモ

  • 本関数はシステムカタログのブートストラップ中にも使用可能です。
  • GetIndexAmRoutine を内部で呼び出しているため、メモリ割り当てのライフサイクルに注意してください。
