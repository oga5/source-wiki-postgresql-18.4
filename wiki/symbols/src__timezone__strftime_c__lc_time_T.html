lc_time_T

定義

  • 種別: struct
  • 定義箇所: src/timezone/strftime.c:48

目的

時刻フォーマット処理において、ロケール固有の文字列定数（月名、曜日名、各種フォーマット文字列）を保持するための構造体です。

シグネチャまたは形状

struct lc_time_T
{
	const char *mon[MONSPERYEAR];
	const char *month[MONSPERYEAR];
	const char *wday[DAYSPERWEEK];
	const char *weekday[DAYSPERWEEK];
	const char *X_fmt;
	const char *x_fmt;
	const char *c_fmt;
	const char *am;
	const char *pm;
	const char *date_fmt;
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。各フィールドは const char * 型の文字列ポインタであり、時刻変換処理の際に参照されます。

前提条件と不変条件

  • 本構造体は C ロケールを前提として定義されています。
  • 配列フィールド（mon, month, wday, weekday）は、それぞれ MONSPERYEAR および DAYSPERWEEK の要素数を持ちます。

アルゴリズムまたは振る舞い

本構造体自体はロジックを持ちませんが、strftime.c 内で C_time_locale という静的定数としてインスタンス化され、時刻の文字列変換時にロケール依存の定数を提供するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/timezone/strftime.c:64 にて C_time_locale の初期化に使用されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • MONSPERYEAR: 月の数（12）を定義する定数。
  • DAYSPERWEEK: 週の日数（7）を定義する定数。

失敗モードと境界条件

  • 構造体内の各ポインタが NULL でないことが期待されます。
  • 本構造体は C ロケール専用であるため、多言語対応が必要な環境での動的な切り替え機能は本構造体単体では提供されません。

保守担当者向けメモ

  • 本構造体は src/timezone/strftime.c 内で完結しており、PostgreSQL の他のレイヤーでロケール処理が行われることを前提としています。
  • フィールドの追加や変更を行う際は、C_time_locale の初期化定義も併せて更新する必要があります。
