BuildTupleHashTable

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execGrouping.c:166

目的

グループ化、ハッシュ集約、または重複排除処理で使用されるインメモリハッシュテーブルを初期化し、必要なハッシュ関数および等価比較関数を構築する。

シグネチャまたは形状

TupleHashTable BuildTupleHashTable(PlanState *parent, TupleDesc inputDesc, const TupleTableSlotOps *inputOps, int numCols, AttrNumber *keyColIdx, const Oid *eqfuncoids, FmgrInfo *hashfunctions, Oid *collations, long nbuckets, Size additionalsize, MemoryContext metacxt, MemoryContext tablecxt, MemoryContext tempcxt, bool use_variable_hash_iv);

入力と出力

  • **入力**: 実行プラン状態、タプル記述子、列インデックス、ハッシュ/比較関数情報、バケット数、メモリコンテキスト（メタ用、テーブル用、一時用）、および並列処理用フラグ。
  • **出力**: 初期化済みの TupleHashTable 構造体へのポインタ。

前提条件と不変条件

  • nbuckets > 0 であること。
  • additionalsize は MAXALIGN でアラインメントされる。
  • metacxt と tablecxt が同一の場合、JIT コンパイルは無効化される。
  • tempcxt は呼び出し側で適切にリセットされる必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: 指定された metacxt 上に TupleHashTableData を割り当て、各種コンテキストや関数情報を格納する。 2. **メモリ制限**: get_hash_memory_limit() に基づき、初期バケット数を制限する。 3. **並列処理対応**: use_variable_hash_iv が真の場合、murmurhash32 を用いて並列ワーカーごとに異なるハッシュ初期値（IV）を生成する。 4. **ハッシュテーブル構築**: tuplehash_create を呼び出し、lib/simplehash.h ベースのハッシュテーブルを生成する。 5. **式評価準備**: ExecBuildHash32FromAttrs でハッシュ計算用、ExecBuildGroupingEqual で等価比較用の ExprState を構築する。 6. **コンテキスト管理**: CreateStandaloneExprContext を使用して式評価用のコンテキストを作成する。

呼び出し元と呼び出し先

  • **呼び出し元**: nodeAgg.c (ハッシュ集約), nodeRecursiveunion.c, nodeSetOp.c, nodeSubplan.c 等の実行ノード。
  • **呼び出し先**: tuplehash_create, MakeSingleTupleTableSlot, ExecBuildHash32FromAttrs, ExecBuildGroupingEqual, CreateStandaloneExprContext 等。

関連データ構造

  • TupleHashTableData: ハッシュテーブルのメタデータと状態を保持する主要構造体。
  • TupleHashEntryData: ハッシュテーブル内の各エントリ。
  • ExprState: 式評価のための状態。

失敗モードと境界条件

  • nbuckets がメモリ制限を超える場合、制限値に切り詰められる。
  • metacxt と tablecxt が同一の場合、JIT が無効化され、パフォーマンスに影響を与える可能性がある。

保守担当者向けメモ

  • tuplehash_ で始まる関数は lib/simplehash.h により生成される。
  • tempcxt は計算用であり、メモリリークを防ぐために呼び出し側で適切にリセットすること。
  • 並列処理時は use_variable_hash_iv を活用し、ハッシュテーブルの偏りを回避している。
