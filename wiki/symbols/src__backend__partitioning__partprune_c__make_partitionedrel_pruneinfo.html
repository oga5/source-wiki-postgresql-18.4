make_partitionedrel_pruneinfo

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partprune.c:445

目的

パーティション分割されたリレーションの階層全体に対して、実行時プルーニングに必要な情報（PartitionedRelPruneInfo）のリストを構築します。

シグネチャまたは形状

static List *make_partitionedrel_pruneinfo(
    PlannerInfo *root, 
    RelOptInfo *parentrel,
    List *prunequal,
    Bitmapset *partrelids,
    int *relid_subplan_map,
    Bitmapset **matchedsubplans
)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • parentrel: プルーニング対象の親リレーション。
  • prunequal: プルーニングに使用する制約条件（RestrictInfo リスト）。
  • partrelids: プルーニング対象となるパーティションリレーションの ID セット。
  • **出力**:
  • 成功時: PartitionedRelPruneInfo 構造体のリスト。
  • 失敗時: 矛盾（contradiction）が検出された場合は NIL を返します。

前提条件と不変条件

  • partrelids に含まれる各リレーションに対して、adjust_appendrel_attrs または adjust_appendrel_attrs_multilevel を用いて、親から子への変数変換が適切に行われる必要があります。
  • 実行時プルーニングが不要と判断された場合、initial_pruning_steps および exec_pruning_steps は NIL になります。

アルゴリズムまたは振る舞い

1. partrelids を走査し、各パーティションに対して GeneratePruningStepsContext を用いてプルーニングステップを生成します。 2. PARTTARGET_INITIAL モードで初期プルーニングステップを生成し、可変演算子や引数が含まれるかを確認します。 3. PARTTARGET_EXEC モードで実行時プルーニングステップを生成し、実際に使用される exec Params を特定します。 4. 矛盾が検出された場合は即座に NIL を返します。 5. 生成されたステップ情報を PartitionedRelPruneInfo に格納し、リストに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/partitioning/partprune.c 内の関数（詳細はソースコードの文脈に依存）。
  • **呼び出し先**: gen_partprune_steps, adjust_appendrel_attrs, adjust_appendrel_attrs_multilevel, get_partkey_exec_paramids 等。

関連データ構造

  • PartitionedRelPruneInfo: 本関数が生成する主要な出力構造体。
  • GeneratePruningStepsContext: ステップ生成中の状態を保持する一時的な構造体。
  • PlannerInfo, RelOptInfo: プランニング時のリレーション情報。

失敗モードと境界条件

  • **矛盾の検出**: context.contradictory が真となる場合、プルーニング条件が論理的に不可能であることを示し、関数は NIL を返して終了します。
  • **パラメータの欠如**: 実行時に使用されるパラメータが存在しない場合、exec_pruning_steps は NIL に設定されます。

保守担当者向けメモ

  • 本関数は、プランニング段階で解決できない動的なプルーニング条件を抽出する重要な役割を担います。
  • relid_subpart_map の構築ロジックは 1-based index を使用しているため、インデックス操作時のオフセットに注意してください。
  • 複雑な階層構造を持つパーティションの場合、adjust_appendrel_attrs_multilevel による変数変換が正しく行われているかを確認してください。
