pg_euc_dsplen

定義

  • 種別: function
  • 定義箇所: src/common/wchar.c:164

目的

EUC（Extended Unix Code）エンコーディングにおける、指定された文字の表示幅（ディスプレイ長）を計算します。

シグネチャまたは形状

static inline int pg_euc_dsplen(const unsigned char *s)

入力と出力

  • **入力**: s - 評価対象のマルチバイト文字の先頭バイトを指すポインタ。
  • **出力**: 文字の表示幅を整数（int）で返します。

前提条件と不変条件

  • 入力ポインタ s は有効なメモリ領域を指している必要があります。
  • EUCの制御文字である SS2（Single Shift 2）および SS3（Single Shift 3）が正しく定義されている環境で動作します。

アルゴリズムまたは振る舞い

1. 入力バイト *s が SS2 または SS3 の場合、表示幅として 2 を返します。 2. IS_HIGHBIT_SET(*s) が真（最上位ビットが立っている）の場合、表示幅として 2 を返します。 3. 上記のいずれにも該当しない場合、pg_ascii_dsplen(s) を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/wchar.c 内の複数箇所から参照されています。
  • **呼び出し先**: pg_ascii_dsplen を呼び出します。

関連データ構造

  • mb/pg_wchar.h に定義されているエンコーディング関連の定数やマクロに依存しています。

失敗モードと境界条件

  • 入力が NULL ポインタである場合の挙動は定義されていません。
  • IS_HIGHBIT_SET マクロの定義に依存するため、実行環境の文字エンコーディング設定が不整合な場合、誤った幅を返す可能性があります。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスを重視して設計されています。
  • EUCの表示幅判定ロジックを変更する場合は、SS2/SS3 の扱いおよび IS_HIGHBIT_SET の判定条件が他のエンコーディング関数と整合しているか確認してください。
