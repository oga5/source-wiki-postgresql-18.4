idx

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_op.c:261

目的

整数配列内において、指定された値が最初に出現する位置（インデックス）を検索します。

シグネチャまたは形状

Datum idx(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_ARRAYTYPE_P(0): 検索対象の整数配列。
  • PG_GETARG_INT32(1): 検索する整数値。
  • **出力**:
  • int32: 値が見つかった場合はその位置（1始まり）、見つからない場合または配列が空の場合は 0 を返します。

前提条件と不変条件

  • 入力配列は CHECKARRVALID マクロにより、PostgreSQLの配列として有効であることが検証されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_ARRAYTYPE_P を使用して入力配列を取得します。 2. CHECKARRVALID で配列の整合性を確認します。 3. ARRNELEMS で配列の要素数を取得し、空でないことを確認します。 4. intarray_match_first を呼び出し、指定された値のインデックスを取得します。 5. PG_FREE_IF_COPY により、必要に応じてコピーされた配列メモリを解放します。 6. 結果を PG_RETURN_INT32 で返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に直接的な呼び出し元は特定されていません（ツールが検出した contrib/cube/cube.c 等の idx は変数名であり、本関数とは無関係です）。
  • **呼び出し先**: intarray_match_first を内部的に呼び出します。

関連データ構造

  • ArrayType: PostgreSQLの標準的な配列データ構造。

失敗モードと境界条件

  • 配列が空の場合、検索は行われず 0 が返されます。
  • 値が配列内に存在しない場合、intarray_match_first の仕様に従い 0 が返されます。

保守担当者向けメモ

  • 本関数はSQLから直接呼び出される関数として実装されています。
  • intarray_match_first は _int_tool.c で定義されているヘルパー関数です。
