Point

定義

  • 種別: struct
  • 定義箇所: src/include/utils/geo_decls.h:96

目的

2次元平面上の座標を表現するためのデータ構造。PostgreSQLの幾何データ型（point型）のC言語における表現として使用されます。

シグネチャまたは形状

typedef struct
{
    float8      x,
                y;
} Point;

入力と出力

  • 入力: 2つの float8 型の値（x座標、y座標）。
  • 出力: Point 構造体。

前提条件と不変条件

  • float8（double）型の数値で構成されます。
  • 幾何学的な計算において、数値解析上の誤差を許容する比較（FPeq 等）が推奨されます。

アルゴリズムまたは振る舞い

  • 本構造体自体はデータの保持のみを行います。
  • 実際の操作（距離計算や比較など）は、src/backend/utils/adt/geo_ops.c 等で定義された関数群によって行われます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • contrib/earthdistance/earthdistance.c: 距離計算の引数として使用。
  • src/backend/access/gist/gistproc.c: GiSTインデックスのキーとして使用。
  • 呼び出し先:
  • PostgreSQLの関数マネージャ（fmgr.h）を介して PG_GETARG_POINT_P や DatumGetPointP マクロによりデータ変換が行われます。

関連データ構造

  • LSEG, PATH, LINE, BOX, POLYGON, CIRCLE: 本構造体と同様に geo_decls.h で定義される他の幾何データ型。

失敗モードと境界条件

  • 数値計算において NaN が含まれる場合、比較関数は常に FALSE を返します。
  • 浮動小数点演算の性質上、推移律が満たされない可能性があるため、厳密な等価判定には注意が必要です。

保守担当者向けメモ

  • 数値解析の専門家によって設計されたルーチンではないため、厳密な計算が必要な場合は注意してください。
  • PATH や POLYGON と異なり、Point は固定長データ型です。
