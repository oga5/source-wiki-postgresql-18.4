toast_tuple_try_compression

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/toast_helper.c:226

目的

指定されたタプル属性に対して圧縮を試み、成功した場合は属性値を圧縮後のデータに置換し、失敗した場合は以降の圧縮対象から除外するフラグを立てる。

シグネチャまたは形状

void toast_tuple_try_compression(ToastTupleContext *ttc, int attribute)

入力と出力

  • **入力**:
  • ttc: 処理対象のタプル状態を保持する ToastTupleContext へのポインタ。
  • attribute: 圧縮を試みる属性のインデックス。
  • **出力**: なし（ttc 構造体内の値およびフラグを直接更新する）。

前提条件と不変条件

  • ttc は toast_tuple_init によって適切に初期化されていること。
  • attribute は ttc->ttc_values の有効なインデックスであること。

アルゴリズムまたは振る舞い

1. toast_compress_datum を呼び出し、対象属性の圧縮を試みる。 2. **圧縮成功時**:

  • 既存の属性値がメモリ確保されている場合（TOASTCOL_NEEDS_FREE フラグあり）、pfree で解放する。
  • ttc->ttc_values[attribute] を圧縮後の new_value で更新する。
  • tai_colflags に TOASTCOL_NEEDS_FREE を追加し、tai_size を更新する。
  • ttc->ttc_flags に TOAST_NEEDS_CHANGE および TOAST_NEEDS_FREE を設定する。

    • 3. **圧縮失敗時**:

  • attr->tai_colflags に TOASTCOL_INCOMPRESSIBLE を追加し、以降の圧縮パスでスキップされるようにする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heaptoast.c (複数箇所)
  • **呼び出し先**:
  • toast_compress_datum (内部的な圧縮処理)
  • pfree (メモリ解放)

関連データ構造

  • ToastTupleContext: TOAST処理中のタプル状態を保持する構造体。
  • ToastAttrInfo: 各属性のTOAST関連情報（フラグやサイズ）を保持する構造体。

失敗モードと境界条件

  • 圧縮が不可能なデータ（toast_compress_datum が NULL を返す場合）は、TOASTCOL_INCOMPRESSIBLE フラグが付与され、エラーにはならず正常に終了する。

保守担当者向けメモ

  • 本関数は ToastTupleContext の状態を直接変更するため、呼び出し側で ttc の整合性が保たれていることを確認すること。
  • TOASTCOL_NEEDS_FREE フラグの管理は、メモリリークを防ぐために重要である。
