get_distance

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/multirangetypes_selfuncs.c:917

目的

2つのレンジ境界（RangeBound）間の距離を計算し、統計的な選択率推定のための指標を算出します。

シグネチャまたは形状

float8 get_distance(TypeCacheEntry *typcache, const RangeBound *bound1, const RangeBound *bound2)

入力と出力

  • **入力**:
  • typcache: 型のキャッシュ情報（サブタイプ比較関数 rng_subdiff_finfo を含む）。
  • bound1, bound2: 距離を測定する対象のレンジ境界。
  • **出力**:
  • float8: 境界間の距離。計算不能な場合は 1.0、無限大の場合は get_float8_infinity() が返されます。

前提条件と不変条件

  • typcache は有効な型キャッシュエントリである必要があります。
  • rng_subdiff_finfo が定義されている場合、その関数は float8 を返す必要があります。

アルゴリズムまたは振る舞い

1. **有限値同士の比較**: 両方の境界が有限である場合、typcache 内の rng_subdiff 関数を使用して差分を計算します。

  • rng_subdiff が利用可能な場合、その結果を返します（ただし、NaN または負の値の場合は 1.0 に補正されます）。
  • rng_subdiff が利用できない場合は 1.0 を返します。

    • 2. **無限値の処理**:

  • 両方が無限大の場合: 同じ方向（下限/上限の属性が一致）であれば 0.0 を返し、そうでなければ無限大を返します。
  • 片方のみが無限大の場合: 無限大を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/multirangetypes_selfuncs.c および src/backend/utils/adt/rangetypes_selfuncs.c 内の複数の箇所から、ヒストグラムを用いた選択率計算の過程で呼び出されています。
  • **呼び出し先**: typcache->rng_subdiff_finfo（関数ポインタ経由）、get_float8_infinity()。

関連データ構造

  • RangeBound: レンジの境界値を保持する構造体。
  • TypeCacheEntry: 型固有の比較関数や照合順序情報を保持する構造体。

失敗モードと境界条件

  • rng_subdiff が負の値を返したり NaN を返したりする場合、安全策として 1.0 が返されます。
  • 無限大を含む境界の比較では、論理的な距離として inf が返されます。

保守担当者向けメモ

  • 本関数は rangetypes_selfuncs.c と同一のロジックを共有しています。修正時は両方のファイルでの整合性に注意してください。
  • rng_subdiff の実装が型ごとに異なるため、推定精度はサブタイプ側の定義に強く依存します。
