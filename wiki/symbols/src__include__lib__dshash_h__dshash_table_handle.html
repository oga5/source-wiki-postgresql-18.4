dshash_table_handle

定義

  • 種別: typedef
  • 定義箇所: src/include/lib/dshash.h:24

目的

動的共有メモリ（DSM）上に構築された dshash テーブルを、プロセス間で共有・識別するためのハンドルとして機能します。

シグネチャまたは形状

typedef dsa_pointer dshash_table_handle;

入力と出力

  • **入力**: dsa_pointer 型の値を保持します。
  • **出力**: dshash_attach 等の関数に渡すことで、共有メモリ上のハッシュテーブルへのアクセスを確立するために使用されます。

前提条件と不変条件

  • dsa_pointer を基盤としているため、有効な DSA（Dynamic Shared Area）コンテキスト内でのみ意味を持ちます。
  • DSHASH_HANDLE_INVALID（InvalidDsaPointer に相当）は、ハンドルが有効なハッシュテーブルを指していないことを示す値として定義されています。

アルゴリズムまたは振る舞い

  • 本質的には dsa_pointer のエイリアスであり、共有メモリ内の特定のデータ構造を指し示すポインタとして機能します。
  • プロセス間でハッシュテーブルを共有する際、このハンドルを介して各プロセスが dshash_attach を実行することで、同一のハッシュテーブルインスタンスに接続します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/lib/dshash.c
  • src/backend/replication/logical/launcher.c
  • src/backend/storage/ipc/dsm_registry.c
  • src/backend/utils/cache/typcache.c
  • src/include/utils/pgstat_internal.h
  • **呼び出し先**:
  • dshash_get_hash_table_handle 関数が本型を戻り値として返します。
  • dshash_attach 等の API が本型を引数として受け取ります。

関連データ構造

  • dsa_pointer: 基盤となる共有メモリポインタ型。
  • dshash_table: 本ハンドルによって参照される実体。

失敗モードと境界条件

  • DSHASH_HANDLE_INVALID を使用して、ハンドルが初期化されていない、または無効であることを表現します。
  • 共有メモリ領域がデタッチまたは破棄された後にこのハンドルを使用した場合の動作は未定義です。

保守担当者向けメモ

  • dshash_table_handle は不透明な識別子として扱ってください。内部の dsa_pointer 値を直接操作することは推奨されません。
  • プロセス間で共有されるため、ハンドル自体の値は各プロセスで一貫していますが、ハンドルが指す先のメモリ領域が各プロセスで正しくマッピングされている必要があります。
