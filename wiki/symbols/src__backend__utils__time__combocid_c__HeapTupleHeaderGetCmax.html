HeapTupleHeaderGetCmax

定義

  • 種別: function
  • 定義箇所: src/backend/utils/time/combocid.c:117

目的

タプルヘッダーから、そのタプルが削除された際のコマンドID（cmax）を取得します。タプルが「Combo CID」を使用している場合は、対応する実値を復元して返します。

シグネチャまたは形状

CommandId HeapTupleHeaderGetCmax(const HeapTupleHeaderData *tup)

入力と出力

  • **入力**: tup (タプルヘッダーへのポインタ)
  • **出力**: CommandId (タプルが削除されたコマンドID)

前提条件と不変条件

  • tup の t_infomask に HEAP_MOVED フラグが立っていないこと。
  • クリティカルセクション外である場合、tup の更新トランザクションIDが現在のトランザクションIDと一致していること。

アルゴリズムまたは振る舞い

1. HeapTupleHeaderGetRawCommandId を呼び出し、ヘッダーから生のコマンドIDを取得します。 2. HEAP_MOVED フラグのチェックを行い、アサーションで検証します。 3. 更新トランザクションIDの整合性を確認します。 4. t_infomask に HEAP_COMBOCID フラグが含まれているか判定します。

  • フラグがある場合: GetRealCmax(cid) を呼び出し、Combo CIDから実値を復元して返します。
  • フラグがない場合: 取得した生のコマンドIDをそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c (複数箇所)
  • src/backend/access/heap/heapam_visibility.c (複数箇所)
  • **呼び出し先**:
  • HeapTupleHeaderGetRawCommandId
  • HeapTupleHeaderGetUpdateXid
  • TransactionIdIsCurrentTransactionId
  • GetRealCmax

関連データ構造

  • HeapTupleHeaderData: タプルヘッダーの構造体。
  • CommandId: コマンドIDを表す型。

失敗モードと境界条件

  • HEAP_MOVED フラグが設定されているタプルに対して呼び出した場合、アサーションエラーが発生します。
  • クリティカルセクション外で、現在のトランザクションによって更新されていないタプルに対して呼び出した場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は HEAP_COMBOCID フラグの有無によって挙動が分岐します。Combo CIDのロジックを変更する際は、GetRealCmax との整合性に注意してください。
