pgstat_free_entry

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_shmem.c:862

目的

共有メモリ上の統計情報エントリを削除し、関連付けられた動的共有メモリ（DSA）領域を解放する。

シグネチャまたは形状

static void pgstat_free_entry(PgStatShared_HashEntry *shent, dshash_seq_status *hstat)

入力と出力

  • shent: 削除対象の共有ハッシュエントリへのポインタ。
  • hstat: dshash の走査状態（dshash_seq_status）。NULL の場合は直接エントリを削除し、指定されている場合は走査中の現在のエントリを削除する。
  • 出力: なし（void）。

前提条件と不変条件

  • pgStatLocal.shared_hash および pgStatLocal.dsa が初期化されていること。
  • 呼び出し元は、対象エントリに対する適切なロックを保持している必要がある。

アルゴリズムまたは振る舞い

1. エントリ shent が保持するDSAポインタ（shent->body）を退避する。 2. hstat が NULL であれば dshash_delete_entry を使用してハッシュテーブルからエントリを削除する。 3. hstat が指定されていれば dshash_delete_current を使用して走査中のエントリを削除する。 4. 退避しておいたDSAポインタを dsa_free に渡し、メモリ領域を解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/activity/pgstat_shmem.c:653
  • src/backend/utils/activity/pgstat_shmem.c:914
  • 呼び出し先:
  • dshash_delete_entry
  • dshash_delete_current
  • dsa_free

関連データ構造

  • PgStatShared_HashEntry: 共有ハッシュテーブル内のエントリ構造体。
  • dshash_seq_status: dshash の走査状態を保持する構造体。
  • pgStatLocal: バックエンドローカルな統計情報管理構造体。

失敗モードと境界条件

  • shent が無効なポインタである場合、または dshash や dsa が適切に初期化されていない状態で呼び出された場合、セグメンテーション違反やメモリ破壊が発生する可能性がある。

保守担当者向けメモ

  • 本関数は、エントリの削除とメモリ解放の順序を厳密に制御している。エントリを削除する前にDSAポインタを退避させることで、ロック解放後も安全にメモリを解放できるように設計されている。
