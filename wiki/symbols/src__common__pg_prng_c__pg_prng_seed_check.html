pg_prng_seed_check

定義

  • **種別**: function
  • **定義箇所**: src/common/pg_prng.c:113

目的

擬似乱数生成器（PRNG）の状態がすべてゼロ（不動点）になることを防ぎ、生成器がゼロを返し続ける事態を回避するための安全装置として機能します。

シグネチャまたは形状

bool pg_prng_seed_check(pg_prng_state *state)

入力と出力

  • **入力**: pg_prng_state *state（乱数生成器の状態を保持する構造体へのポインタ）
  • **出力**: bool（常に true を返します）

前提条件と不変条件

  • 入力される state は有効な pg_prng_state 構造体を指している必要があります。
  • 本関数は、乱数生成器の初期化処理の直後に呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. state->s0 および state->s1 の両方が 0 であるかを確認します。 2. 両者が 0 の場合、生成器が不動点に陥るのを防ぐため、Knuthの線形合同法（LCG）パラメータを用いて、それぞれ以下の非ゼロ値で上書きします。

  • state->s0 = 0x5851F42D4C957F2D
  • state->s1 = 0x14057B7EF767814F

    • 3. 最後に true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/pg_prng.c 内の初期化処理（pg_prng_seed 等）から呼び出されます。
  • **呼び出し先**: 内部で呼び出している関数はありません。

関連データ構造

  • pg_prng_state: 乱数生成器の内部状態を保持する構造体。

失敗モードと境界条件

  • **境界条件**: s0 と s1 が両方とも 0 の場合にのみ、状態の書き換えが発生します。
  • **失敗モード**: 入力ポインタ state が NULL の場合の挙動については、本ソース excerpt 内では定義されていません。

保守担当者向けメモ

  • 本関数は pg_prng_strong_seed マクロの利便性を高めるために true を返す設計となっています。
  • 暗号学的な安全性は保証されないため、パスワードや鍵生成には使用しないでください。
