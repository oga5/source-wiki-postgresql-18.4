pg_atomic_uint32

定義

  • 種別: struct
  • 定義箇所: src/include/port/atomics/arch-x86.h:61

目的

マルチコア環境において、32ビット符号なし整数に対するアトミックな読み書きおよび演算操作を可能にするためのデータ構造です。

シグネチャまたは形状

typedef struct pg_atomic_uint32
{
	volatile uint32 value;
} pg_atomic_uint32;

入力と出力

  • 入力: 構造体内の value フィールド（uint32 型）。
  • 出力: volatile 修飾子により、コンパイラによる最適化を抑制し、常にメモリ上の最新値へのアクセスを保証します。

前提条件と不変条件

  • volatile 修飾子により、コンパイラは本構造体へのアクセスをキャッシュせず、常にメモリから読み書きを行います。
  • アトミック性を保証するためには、本構造体に対する操作に arch-x86.h 等で定義された専用のアトミック関数（pg_atomic_compare_exchange_u32_impl 等）を使用する必要があります。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のみを行い、具体的なアトミック操作は lock プレフィックス付きのx86命令（cmpxchg, xadd 等）をインラインアセンブラで実行する関数群によって提供されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照例）:
  • src/backend/commands/vacuum.c
  • src/backend/commands/vacuumparallel.c
  • src/backend/nodes/tidbitmap.c
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/pgarch.c
  • src/backend/storage/buffer/freelist.c
  • src/backend/storage/ipc/procsignal.c
  • 呼び出し先: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。操作は外部のアトミック関数によって行われます。

関連データ構造

  • pg_atomic_uint64: 64ビット版のアトミック構造体。
  • pg_atomic_flag: フラグ操作用のアトミック構造体。

失敗モードと境界条件

  • 本構造体自体に失敗モードはありません。
  • 32ビット境界を超える値の操作や、アトミック関数を介さない直接的な代入を行った場合、アトミック性は保証されません。

保守担当者向けメモ

  • 本構造体は src/include/port/atomics/generic.h を通じて上位層へ提供されるインターフェースの基盤となります。
  • x86環境ではTSO（Total Store Order）モデルに基づき動作するため、メモリバリアの挿入箇所についてはアーキテクチャ固有の制約を考慮してください。
