FindPostmasterChildByPid

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/pmchild.c:273

目的

指定されたプロセスID（PID）に対応する PMChild 構造体を、現在稼働中の子プロセスリストから検索して返します。

シグネチャまたは形状

PMChild *FindPostmasterChildByPid(int pid)

入力と出力

  • **入力**: pid (int) - 検索対象のプロセスID。
  • **出力**: PMChild* - 見つかった場合は該当する構造体へのポインタ、見つからない場合は NULL。

前提条件と不変条件

  • ActiveChildList が初期化済みであること。
  • 本関数は Postmaster プロセス内で実行されることを前提としています。

アルゴリズムまたは振る舞い

1. ActiveChildList に対して dlist_foreach を使用し、リスト内の全要素を走査します。 2. 各要素から PMChild 構造体を取得します。 3. 構造体の pid フィールドと引数の pid を比較します。 4. 一致した場合はそのポインタを即座に返します。 5. リストの末尾まで走査しても一致しない場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/postmaster.c:2517 において、PID に基づくプロセス特定のために呼び出されています。
  • **呼び出し先**: dlist API（dlist_foreach, dlist_container）を利用してリスト操作を行います。

関連データ構造

  • PMChild: 子プロセスの情報を保持する構造体。
  • ActiveChildList: 現在稼働中の全子プロセスを保持する双方向リスト。

失敗モードと境界条件

  • 該当する PID がリスト内に存在しない場合、NULL を返します。
  • ActiveChildList が空の場合、走査は行われず直ちに NULL を返します。

保守担当者向けメモ

  • 本関数は線形探索を行うため、ActiveChildList のサイズが大きい場合には検索コストが増大します。
  • 戻り値が NULL になるケース（プロセスが既に終了している、または追跡対象外である等）を呼び出し側で適切に処理してください。
