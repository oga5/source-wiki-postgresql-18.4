list_insert_nth_oid

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/list.c:466

目的

Oid 型の要素を、既存の List の指定された位置（インデックス）に挿入する。

シグネチャまたは形状

List *list_insert_nth_oid(List *list, int pos, Oid datum)

入力と出力

  • **入力**:
  • list: 挿入対象の List ポインタ。
  • pos: 挿入位置を示す整数インデックス。
  • datum: 挿入する Oid 値。
  • **出力**:
  • 挿入後の List ポインタ。リストが再配置された場合、元のポインタとは異なる可能性がある。

前提条件と不変条件

  • list が NIL の場合、pos は 0 でなければならない。
  • list が NIL でない場合、IsOidList(list) が真であること（リストが Oid 型を保持する構造であること）。
  • 操作の終了時に check_list_invariants(list) が呼び出され、リストの整合性が検証される。

アルゴリズムまたは振る舞い

1. list が NIL の場合、list_make1_oid(datum) を呼び出して新しいリストを作成し、それを返す。 2. list が NIL でない場合、insert_new_cell(list, pos) を呼び出して指定位置に新しいセルを確保する。 3. 確保されたセルに対して lfirst_oid マクロを用いて datum を書き込む。 4. check_list_invariants を実行してリストの不変条件を確認し、リストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は特定されていません。
  • **呼び出し先**:
  • list_make1_oid (リストが空の場合)
  • IsOidList (型チェック)
  • insert_new_cell (セル確保と配列シフト)
  • lfirst_oid (値の代入)
  • check_list_invariants (整合性チェック)

関連データ構造

  • List: PostgreSQL の動的配列ベースの汎用リスト構造体。
  • Oid: PostgreSQL のオブジェクト識別子型。

失敗モードと境界条件

  • pos がリストの現在の範囲外である場合、insert_new_cell の内部実装に依存してエラーが発生する可能性がある。
  • メモリ不足時には palloc を経由してエラーが発生する。

保守担当者向けメモ

  • 本関数は Oid 型専用の挿入処理を行う。他の型（int, ptr 等）については、対応する list_insert_nth_... 関数を使用すること。
  • リストの挿入操作は配列のシフトを伴うため、リストのサイズが大きい場合、計算量は O(N) となる。
  • insert_new_cell により内部配列が再配置される可能性があるため、呼び出し元でリストの要素を指すポインタを保持し続けることは避けること。
