BlockRefTableReader

定義

  • 種別: struct
  • 定義箇所: src/include/common/blkreftable.h:34

目的

ディスク上のブロック参照テーブル（BlockRefTable）から、変更されたデータベースブロックの情報をストリーミング形式で読み込むためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct BlockRefTableReader BlockRefTableReader;

（構造体の内部メンバは公開ヘッダーでは不透明な型として定義されています）

入力と出力

  • **入力**: src/common/blkreftable.c の実装において、外部ストレージからのデータ読み込みを抽象化するコールバック関数（io_callback_fn）およびエラー報告関数（report_error_fn）を介してデータを受け取ります。
  • **出力**: BlockRefTableReaderNextRelation を通じてリレーション単位の反復処理を行い、BlockRefTableReaderGetBlocks を通じて変更ブロックのリストを出力します。

前提条件と不変条件

  • 読み込み対象のデータは、データベース、テーブルスペース、リレーション、フォークの順にソートされている必要があります。
  • 読み込み処理中に不整合やエラーが発生した場合、登録された report_error_fn が呼び出されます。

アルゴリズムまたは振る舞い

1. palloc0 等でメモリを確保し、読み込みコンテキストを初期化します。 2. BlockRefTableReaderNextRelation を呼び出し、次のリレーションの変更情報を取得します。 3. BlockRefTableReaderGetBlocks を呼び出し、当該リレーション内の変更ブロック情報を取得します。 4. 内部的には、変更ブロック数に応じて配列表現またはビットマップ表現を適切にデシリアライズします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_incremental.c
  • src/backend/backup/walsummaryfuncs.c
  • src/bin/pg_walsummary/pg_walsummary.c
  • **呼び出し先**:
  • src/common/blkreftable.c 内の BlockRefTableRead（内部的な読み込み処理）

関連データ構造

  • BlockRefTable: メモリ上の変更情報保持構造体。
  • RelFileLocator: リレーションの物理的識別子。
  • BlockNumber: ブロック番号の型定義。

失敗モードと境界条件

  • データの順序が不正な場合、読み込み処理が失敗する可能性があります。
  • 読み込み中に発生したエラーは、初期化時に設定された report_error_fn を通じて通知されます。

保守担当者向けメモ

  • 本構造体は不透明な型として定義されているため、内部メンバへの直接アクセスは避け、blkreftable.c で提供される API を介して操作してください。
  • 読み込み順序の制約（ソート済みであること）は、パフォーマンスとメモリ効率を維持するために重要です。
