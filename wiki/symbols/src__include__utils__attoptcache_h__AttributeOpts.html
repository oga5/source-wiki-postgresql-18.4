AttributeOpts

定義

  • 種別: struct
  • 定義箇所: src/include/utils/attoptcache.h:19

目的

システムカタログ pg_attribute に格納される属性オプション（統計情報など）をメモリ上で保持し、アクセスを高速化するための構造体です。

シグネチャまたは形状

typedef struct AttributeOpts
{
    int32       vl_len_;        /* varlena header */
    float8      n_distinct;
    float8      n_distinct_inherited;
} AttributeOpts;

入力と出力

  • 本構造体は get_attribute_options 関数を通じて取得されます。
  • キャッシュミス時にはシステムカタログから読み込まれ、キャッシュに格納されます。

前提条件と不変条件

  • 先頭メンバ vl_len_ は可変長データ（varlena）のヘッダーです。直接操作せず、PostgreSQL の標準的な varlena マクロを使用する必要があります。
  • キャッシュのライフサイクルは pg_attribute の更新と連動しており、システムキャッシュの無効化コールバックによって管理されます。

アルゴリズムまたは振る舞い

  • 属性に関連付けられた統計情報（n_distinct 等）を保持します。
  • キャッシュ層として機能し、リレーションの属性メタデータへのアクセス時にシステムカタログへの直接アクセスを回避します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/common/reloptions.c: 属性オプションの定義およびサイズ計算に使用。
  • src/backend/commands/analyze.c: 統計情報の解析処理。
  • src/backend/statistics/extended_stats.c: 拡張統計情報の処理。
  • src/backend/utils/cache/attoptcache.c: キャッシュの構築および取得処理。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、特定の関数を呼び出すことはありません。

関連データ構造

  • pg_attribute: 本キャッシュの元となるシステムカタログ。
  • AttoptCacheHash: attoptcache.c 内で本構造体を管理するために使用されるハッシュテーブル。

失敗モードと境界条件

  • キャッシュに該当するエントリが存在しない場合、get_attribute_options は NULL を返す可能性があります。
  • pg_attribute が更新された場合、InvalidateAttoptCacheCallback によりキャッシュエントリが破棄されるため、古い情報を保持し続けることはありません。

保守担当者向けメモ

  • 本構造体は src/backend/utils/cache/attoptcache.c で実装されるキャッシュ機構の核となります。
  • 構造体のメンバを追加・変更する場合は、reloptions.c での定義やサイズ計算箇所も併せて更新する必要があります。
