hex_encode

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/write_manifest.c:279

目的

バイナリデータを 16 進数文字列（小文字）に変換し、指定されたバッファに書き込む。

シグネチャまたは形状

size_t hex_encode(const uint8 *src, size_t len, char *dst)

入力と出力

  • **入力**:
  • src: 変換対象のバイナリデータ（uint8 配列）。
  • len: 変換するデータのバイト数。
  • dst: 変換後の文字列を格納するバッファ（呼び出し側で len * 2 バイト以上の領域を確保する必要がある）。
  • **出力**:
  • 戻り値: 書き込まれた文字数（len * 2）。

前提条件と不変条件

  • dst は少なくとも len * 2 バイトの書き込み可能な領域を指していること。
  • 入力データは任意のバイト列を許容する。

アルゴリズムまたは振る舞い

1. 入力データの各バイトを上位 4 ビットと下位 4 ビットに分割する。 2. 各 4 ビットの値を 0-9 または a-f の ASCII 文字に変換する。 3. 変換した 2 文字を dst バッファに順次書き込む。 4. 全バイトの処理が完了するまで繰り返し、最後に変換後の総文字数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_combinebackup/write_manifest.c 内の各箇所（mwriter->buf への書き込み時）。
  • **呼び出し先**:
  • なし（標準的な算術演算とポインタ操作のみで構成される）。
  • **注記**: ツールにより src/backend/utils/adt/encode.c 等の他ファイルでも同名の関数が定義されていることが示唆されているが、本ページは write_manifest.c 内の定義を対象とする。

関連データ構造

  • 特になし（プリミティブ型のみを使用）。

失敗モードと境界条件

  • len が 0 の場合、何も書き込まず 0 を返す。
  • dst のバッファサイズが不足している場合、バッファオーバーランが発生する（呼び出し側の責任）。

保守担当者向けメモ

  • 本関数は write_manifest.c 内で静的に定義されており、マニフェスト生成時のパスやチェックサムのエンコードに使用される。
  • PostgreSQL の他のモジュール（バックエンド側など）にも同名の関数が存在するが、本実装は write_manifest.c 内で完結している。
