ExecIndexScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexscan.c:520

目的

PostgreSQLの実行エンジンにおいて、インデックスを用いたテーブルスキャンのメインエントリーポイントとして機能し、インデックスからタプルを順次取得する。

シグネチャまたは形状

TupleTableSlot *ExecIndexScan(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（実行中のインデックススキャンノードの状態）
  • **出力**: TupleTableSlot *（取得されたタプル、またはスキャン終了時は NULL）

前提条件と不変条件

  • pstate は IndexScanState 型にキャスト可能であること。
  • ランタイムキーが存在し、かつ未準備（iss_RuntimeKeysReady が偽）の場合、ExecReScan を呼び出してキーを再構築する。

アルゴリズムまたは振る舞い

1. **ランタイムキーの確認**: iss_NumRuntimeKeys が 0 でなく、かつ iss_RuntimeKeysReady が偽であれば、ExecReScan を実行してスキャンキーを初期化する。 2. **スキャン実行**:

  • iss_NumOrderByKeys > 0 の場合: IndexNextWithReorder をアクセスメソッドとして ExecScan を呼び出し、再ソートを伴うスキャンを行う。
  • それ以外の場合: IndexNext をアクセスメソッドとして ExecScan を呼び出し、通常のインデックススキャンを行う。

    • 3. **再チェック**: いずれの場合も IndexRecheck を再チェックメソッドとして使用し、インデックスの損失性や条件の再評価を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（ExecProcNode を介して呼び出される）。
  • **呼び出し先**:
  • ExecReScan: ランタイムキーの再構築用。
  • ExecScan: スキャンの共通基盤。
  • IndexNext / IndexNextWithReorder: インデックスからのタプル取得。
  • IndexRecheck: タプルの再検証。

関連データ構造

  • IndexScanState: インデックススキャンの実行状態を保持する構造体。
  • PlanState: プランノードの実行状態。
  • TupleTableSlot: タプルを格納するスロット。

失敗モードと境界条件

  • インデックススキャンが完了した場合は NULL を返す。
  • ランタイムキーの評価失敗や、インデックスアクセス時のエラーは呼び出し先の関数群に依存する。

保守担当者向けメモ

  • 本関数は ExecProcNode のインターフェースを実装しており、nodeIndexscan.c 内の IndexNext や IndexNextWithReorder と密接に連携する。
  • 並列クエリ実行時の状態管理については、ExecIndexScanInitializeDSM 等の関連関数を参照すること。
