DefineRule

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteDefine.c:189

目的

CREATE RULE 文の解析結果を処理し、対象リレーションに対する書き換えルールの定義およびシステムカタログへの登録を統括するメインエントリポイントです。

シグネチャまたは形状

ObjectAddress DefineRule(RuleStmt *stmt, const char *queryString)

入力と出力

  • **入力**:
  • stmt: 解析対象の RuleStmt 構造体（ルール定義情報）。
  • queryString: 元の SQL クエリ文字列。
  • **出力**:
  • ObjectAddress: 作成されたルールオブジェクトの識別子。

前提条件と不変条件

  • 対象リレーションに対して AccessExclusiveLock を取得する必要があります。
  • 呼び出し前に RuleStmt が適切に構築されている必要があります。

アルゴリズムまたは振る舞い

1. transformRuleStmt を呼び出し、RuleStmt からルールのアクションリストと WHERE 句を抽出・解析します。 2. RangeVarGetRelid を使用して、対象リレーションの OID を取得し、AccessExclusiveLock を獲得します。 3. 取得した情報を基に DefineQueryRewrite を呼び出し、カタログ登録、依存関係の構築、およびキャッシュ無効化を実行します。 4. DefineQueryRewrite の戻り値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c: CREATE RULE 文の実行時に呼び出されます。
  • **呼び出し先**:
  • transformRuleStmt
  • RangeVarGetRelid
  • DefineQueryRewrite

関連データ構造

  • RuleStmt: パーサーが生成するルール定義のノード。
  • ObjectAddress: 作成されたルールの識別情報。

失敗モードと境界条件

  • 対象リレーションが存在しない場合や、ロック取得に失敗した場合はエラーが発生します。
  • transformRuleStmt による解析フェーズで構文や意味的な誤りがある場合、処理が中断されます。

保守担当者向けメモ

  • 本関数は DefineQueryRewrite への橋渡し役です。ロックレベルの整合性は DefineQueryRewrite と一致させる必要があります。
  • ルール定義はシステムカタログを書き換えるため、排他ロックが必須です。
