gistkillitems

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistget.c:37

目的

インデックススキャン中に不要と判断された（「殺された」）タプルに対し、GiST インデックスページ上で LP_DEAD フラグを立て、ガベージとしてマークする。

シグネチャまたは形状

static void gistkillitems(IndexScanDesc scan)

入力と出力

  • **入力**: IndexScanDesc scan - 現在のスキャン状態を保持する記述子。opaque フィールドに GISTScanOpaque 構造体が含まれている必要がある。
  • **出力**: なし（副作用としてインデックスページを更新）。

前提条件と不変条件

  • scan->opaque が GISTScanOpaque としてキャスト可能であること。
  • so->curBlkno が有効なブロック番号であること。
  • so->curPageLSN が有効な LSN であること。
  • so->killedItems が初期化されていること。
  • 対象ページがリーフページであること（GistPageIsLeaf(page)）。

アルゴリズムまたは振る舞い

1. so->curBlkno を読み込み、共有ロック（GIST_SHARE）を取得する。 2. 現在のページの LSN と so->curPageLSN を比較する。不一致の場合、ページが読み込み後に更新されたと判断し、処理を中断して so->numKilled をリセットする。 3. 一致する場合、so->killedItems に格納されたオフセット番号を順次走査し、ItemIdMarkDead を用いて各アイテムに LP_DEAD フラグを立てる。 4. アイテムがマークされた場合、GistMarkPageHasGarbage を呼び出し、バッファをダーティとしてマークする。 5. 最後に so->numKilled を 0 にリセットし、スキャン状態をクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistget.c 内のインデックススキャン処理（gistgettuple 等から呼び出される）。
  • **呼び出し先**: ReadBuffer, LockBuffer, gistcheckpage, BufferGetPage, BufferGetLSNAtomic, PageGetItemId, ItemIdMarkDead, GistMarkPageHasGarbage, MarkBufferDirtyHint, UnlockReleaseBuffer。

関連データ構造

  • IndexScanDesc: インデックススキャンの共通記述子。
  • GISTScanOpaque: GiST 固有のスキャン状態。curBlkno, curPageLSN, killedItems, numKilled を保持する。

失敗モードと境界条件

  • **ページ更新**: 読み込み後にページが更新された場合（LSN 不一致）、安全のため処理を行わず終了する。
  • **無効なバッファ**: ReadBuffer が有効なバッファを返さない場合は即座に終了する。

保守担当者向けメモ

  • 本関数は MVCC 下での安全な削除を保証するため、LSN チェックを厳密に行う。
  • so->numKilled は処理の最後に必ずリセットされ、同一アイテムの重複処理を防ぐ設計となっている。
