AtSubAbort_Portals

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/portalmem.c:978

目的

サブトランザクションのアボート時に、そのサブトランザクションに関連するポータルの状態を整理し、リソースリークを防ぐためのクリーンアップを行う。

シグネチャまたは形状

void AtSubAbort_Portals(SubTransactionId mySubid,
                        SubTransactionId parentSubid,
                        ResourceOwner myXactOwner,
                        ResourceOwner parentXactOwner)

入力と出力

  • **入力**:
  • mySubid: アボート対象のサブトランザクションID。
  • parentSubid: 親トランザクション（またはサブトランザクション）のID。
  • myXactOwner: アボート対象のサブトランザクションのリソースオーナー。
  • parentXactOwner: 親のリソースオーナー。
  • **出力**: なし（副作用としてポータル状態の更新とメモリ解放を行う）。

前提条件と不変条件

  • PortalHashTable が初期化されていること。
  • ポータルが ACTIVE 状態である場合、MarkPortalFailed を介して FAILED 状態へ遷移させる必要がある。

アルゴリズムまたは振る舞い

1. PortalHashTable を走査し、全ポータルを順次処理する。 2. **他サブトランザクションのポータル**: createSubid が mySubid でない場合、activeSubid が mySubid であれば parentSubid に更新する。また、PORTAL_ACTIVE であれば FAILED にし、FAILED かつ resowner が存在する場合は親のリソースオーナーへ付け替える。 3. **自サブトランザクションのポータル**: createSubid が mySubid である場合：

  • READY または ACTIVE 状態であれば MarkPortalFailed を呼び出す。
  • cleanup フックが存在すれば実行し、cleanup をクリアする。
  • PortalReleaseCachedPlan を呼び出し、キャッシュされたプランの参照を解放する。
  • resowner をクリアする。
  • MemoryContextDeleteChildren を呼び出し、ポータルの子メモリコンテキスト（エグゼキュータ状態など）を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c（サブトランザクションのアボート処理時）。
  • **呼び出し先**: MarkPortalFailed, ResourceOwnerNewParent, portal->cleanup (関数ポインタ), PortalReleaseCachedPlan, MemoryContextDeleteChildren。

関連データ構造

  • PortalHashTable: 全ポータルを管理するハッシュテーブル。
  • PortalHashEnt: ハッシュテーブルのエントリ。
  • Portal: ポータル構造体。

失敗モードと境界条件

  • PORTAL_ACTIVE 状態のポータルが存在する場合、強制的に FAILED 状態へ遷移させる。
  • cleanup フックが設定されている場合は、二重実行を防ぐために実行後に NULL にリセットする。

保守担当者向けメモ

  • 本関数はポータルデータ構造そのものを削除するわけではなく、あくまでサブトランザクション終了に伴う状態の整合性維持とメモリ解放を行う。
  • resowner の付け替えは、アボート処理中にリソースリークやアサーションエラーが発生するのを防ぐための重要な措置である。
