utf8_to_euc_tw

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_euc_tw/utf8_and_euc_tw.c:62

目的

UTF-8エンコーディングの文字列をEUC_TWエンコーディングに変換する、PostgreSQLの動的ロード可能な変換プロシージャです。

シグネチャまたは形状

Datum utf8_to_euc_tw(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて以下の引数を受け取ります。
  • src (CSTRING): 変換元のUTF-8文字列。
  • dest (CSTRING): 変換後のEUC_TW文字列を格納するバッファ。
  • len (INT32): 変換対象のバイト長。
  • noError (BOOL): 変換失敗時にエラーを発生させるか否かのフラグ。
  • **出力**: PG_RETURN_INT32 を介して、変換されたバイト数を返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_UTF8, PG_EUC_TW) により、ソースがUTF-8、宛先がEUC_TWであることを検証します。
  • 変換処理は euc_tw_from_unicode_tree マッピングテーブルに依存します。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得します。 2. UtfToLocal 関数を呼び出し、変換ロジックを委譲します。この際、euc_tw_from_unicode_tree を変換テーブルとして使用し、noError フラグを渡してエラー時の挙動を制御します。 3. 変換されたバイト数を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（fmgr）を介してデータベースエンジンから呼び出されます。
  • **呼び出し先**: UtfToLocal（mb/pg_wchar.h で定義された汎用変換関数）。

関連データ構造

  • euc_tw_from_unicode_tree: ../../Unicode/utf8_to_euc_tw.map からインクルードされる変換テーブル。

失敗モードと境界条件

  • noError が false の場合、変換不可能な文字が含まれるとエラーが発生します。
  • noError が true の場合、エラーを抑制し、成功したバイト数のみを返します。

保守担当者向けメモ

  • 本関数はラッパーであり、実際の変換ロジックは mb/wchar.c およびマッピングファイルに依存しています。
  • 新しいエンコーディング変換を追加する際は、本関数の実装パターン（PG_FUNCTION_INFO_V1 および UtfToLocal の呼び出し）を模倣してください。
