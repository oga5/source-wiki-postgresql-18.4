shift_jis_20042euc_jis_2004

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc2004_sjis2004/euc2004_sjis2004.c:256

目的

SHIFT_JIS_2004 エンコーディングの文字列を EUC_JIS_2004 エンコーディングへ変換する。

シグネチャまたは形状

static int shift_jis_20042euc_jis_2004(const unsigned char *sjis, unsigned char *p, int len, bool noError)

入力と出力

  • **入力**:
  • sjis: 変換元となる SHIFT_JIS_2004 文字列のポインタ。
  • p: 変換先を格納するバッファのポインタ。
  • len: 入力文字列のバイト長。
  • noError: 不正なバイト列検出時にエラーを報告せず中断するかどうかを指定するフラグ。
  • **出力**:
  • 戻り値: 変換されたバイト数。
  • p が指すバッファには、変換後の EUC_JIS_2004 文字列とヌル終端文字が書き込まれる。

前提条件と不変条件

  • sjis は有効なメモリ領域を指していること。
  • p は変換後の文字列を格納するのに十分な領域を確保していること。
  • 入力文字列は SHIFT_JIS_2004 規格に従っていること。

アルゴリズムまたは振る舞い

1. 入力文字列を 1 文字ずつ走査する。 2. ASCII 文字（高位ビットが未セット）はそのままコピーする。 3. pg_encoding_verifymbchar を使用してマルチバイト文字の妥当性を検証する。 4. 1 バイトの JIS X 0201 文字（半角カナ）は SS2 を付加して 2 バイトに変換する。 5. 2 バイト文字は、c1（第 1 バイト）の値に基づき、JIS X 0213 の面（plane）、区（ku）、点（ten）を算出する。

  • 第 2 面に該当する場合は SS3 を付加する。
  • get_ten 関数を用いて点（ten）と区（ku）のオフセットを計算する。

    • 6. 変換結果を p に書き込み、ポインタと長さを更新する。 7. 最後に p にヌル終端を付加し、処理した総バイト数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mb/conversion_procs/euc2004_sjis2004/euc2004_sjis2004.c 内の変換関数（shift_jis_2004_to_euc_jis_2004 等から参照されている）。
  • **呼び出し先**:
  • pg_encoding_verifymbchar: マルチバイト文字の検証。
  • get_ten: 点（ten）の算出。
  • report_invalid_encoding: 不正なバイト列検出時のエラー報告。

関連データ構造

  • PG_SHIFT_JIS_2004: エンコーディング識別子。
  • SS2, SS3: EUC における補助コードセット識別子。

失敗モードと境界条件

  • 不正なバイト列（pg_encoding_verifymbchar が負値を返す、または範囲外）に遭遇した場合:
  • noError が true の場合は変換を中断し、処理済みバイト数を返す。
  • noError が false の場合は report_invalid_encoding を呼び出しエラーを発生させる。
  • 入力文字列が空、または len が 0 の場合は何も変換せず 0 を返す。

保守担当者向けメモ

  • JIS X 0213:2004 の規格に基づいたビット演算を行っているため、ロジック変更時は規格書との照合が必須。
  • get_ten 関数による「点」の計算は、Shift_JIS のバイト配置と JIS コードのオフセットに依存している。
