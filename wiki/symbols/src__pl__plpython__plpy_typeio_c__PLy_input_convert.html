PLy_input_convert

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_typeio.c:80

目的

PostgreSQLの Datum 型からPythonの PyObject 型への変換を実行する最上位エントリーポイント。

シグネチャまたは形状

PyObject *PLy_input_convert(PLyDatumToOb *arg, Datum val);

入力と出力

  • **入力**:
  • arg: 変換ロジックと型情報を保持する PLyDatumToOb 構造体。
  • val: 変換対象のPostgreSQL Datum 値。
  • **出力**: 変換後の PyObject へのポインタ。

前提条件と不変条件

  • 呼び出し時に PLy_current_execution_context() が有効な実行コンテキストを返すこと。
  • 変換処理は scratch_context 上で行われるため、呼び出し元はメモリコンテキストの生存期間を保証する必要がある。

アルゴリズムまたは振る舞い

1. 現在の実行コンテキストから scratch_context を取得する。 2. MemoryContextReset(scratch_context) を実行し、変換サイクル開始前にメモリをクリアする。 3. MemoryContextSwitchTo(scratch_context) を行い、変換処理中のメモリ割り当てを一時領域へ集約する。 4. arg->func を呼び出し、再帰的に変換を実行する。 5. 元のメモリコンテキストへ復帰し、生成された PyObject を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_exec.c:457 (PL/Python関数実行時の引数変換)
  • **呼び出し先**:
  • PLy_current_execution_context()
  • PLy_get_scratch_context()
  • arg->func (変換構造体内に保持された関数ポインタ)

関連データ構造

  • PLyDatumToOb: 変換関数ポインタおよび型メタデータを保持する構造体。
  • PLyExecutionContext: 実行状態を管理するコンテキスト。

失敗モードと境界条件

  • MemoryContextReset 実行時にエラーが発生した場合、Pythonオブジェクトの参照カウント解放がスキップされる可能性がある（ソースコード内のコメントによる）。
  • 再帰的な変換において、内部の変換関数が scratch_context をリセットすると、上位の変換処理が参照しているデータを破壊する恐れがあるため、リセットは最上位レベルでのみ行う。

保守担当者向けメモ

  • 本関数は再帰的な変換の起点となるため、メモリ管理の整合性を維持するために scratch_context のリセットタイミングを厳守すること。
  • arg->func が内部でどのような変換を行うかは、PLy_input_setup_func 等によって構築された変換ツリーに依存する。
