TParserState

定義

  • 種別: enum
  • 定義箇所: src/backend/tsearch/wparser_def.c:119

目的

PostgreSQL のデフォルト・テキスト検索パーサーにおける有限オートマトン（FSM）の現在の解析状態を定義します。入力文字列をトークン化する際の遷移先を管理するために使用されます。

シグネチャまたは形状

TPS_Base から TPS_Null までの列挙値で構成されます。主な状態には、単語（TPS_InWord）、数値（TPS_InUnsignedInt）、XMLタグ（TPS_InTag）、URL/パス（TPS_InURLPath）、コメント（TPS_InComment）などの解析状態が含まれます。

入力と出力

  • **入力**: パーサーのメインループにおいて、現在の状態と入力文字に基づいて次の状態を決定するために参照されます。
  • **出力**: 状態遷移の結果として、次の解析ステップで使用される TParserState 値が返されます。

前提条件と不変条件

  • TParserState の順序は、wparser_def.c 内の TParserStateActionItem テーブルの定義順序と一致している必要があります。
  • TPS_Null は末尾を示すダミー値として使用されます。

アルゴリズムまたは振る舞い

パーサーは TParserState を保持し、入力文字クラスと現在の状態をキーとして Actions テーブルを検索します。このテーブルから得られるアクション（PUSH/POP/BINGOなど）と遷移先の TParserState に基づき、状態を更新しながらトークンを抽出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/wparser_def.c 内の解析ロジック（prsd_nexttoken 等）が、状態遷移の管理やアサーションチェックのためにこの型を参照しています。
  • **呼び出し先**: 本シンボル自体は列挙型であり、関数呼び出しは行いません。

関連データ構造

  • TParser: 解析状態を保持するメイン構造体。
  • TParserStateActionItem: 状態遷移ルールを定義するテーブル。
  • TParserPosition: 状態遷移のスタック管理に使用される構造体。

失敗モードと境界条件

  • 定義順序が TParserStateActionItem テーブルと不一致の場合、誤った遷移ルールが適用されるリスクがあります。
  • TPS_Null を超えて状態が遷移することはありません。

保守担当者向けメモ

  • 新しいトークンタイプを追加する場合は、本列挙型に新しい状態を追加し、対応する TParserStateActionItem テーブルの定義を更新してください。
  • 状態遷移のデバッグには WPARSER_TRACE マクロの利用が推奨されます。
