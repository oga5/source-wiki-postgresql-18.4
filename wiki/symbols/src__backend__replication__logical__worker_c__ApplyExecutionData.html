ApplyExecutionData

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/worker.c:200

目的

論理レプリケーションの適用ワーカーにおいて、特定のテーブルに対する変更適用時の実行状態（EState）やターゲットリレーション情報を保持するために使用されます。

シグネチャまたは形状

typedef struct ApplyExecutionData
{
    EState               *estate;           /* 実行状態、リソース追跡用 */
    LogicalRepRelMapEntry *targetRel;       /* レプリケーション対象リレーション */
    ResultRelInfo        *targetRelInfo;    /* 上記に対応するResultRelInfo */
    ModifyTableState     *mtstate;          /* パーティション用ダミーModifyTable状態 */
    PartitionTupleRouting *proute;          /* パーティションルーティング情報 */
} ApplyExecutionData;

入力と出力

  • 本構造体自体はデータ保持用であり、関数ではありません。
  • palloc0 を用いてメモリ確保され、finish_edata 関数によって終了処理が行われます。

前提条件と不変条件

  • targetRel および targetRelInfo は、適用対象のテーブルに対して適切に初期化されている必要があります。
  • パーティションテーブルを扱う場合、mtstate および proute が適切に設定されている必要があります。

アルゴリズムまたは振る舞い

  • 適用ワーカーが受信した変更（INSERT/UPDATE/DELETE）をローカルテーブルへ反映する際、実行環境としてこの構造体が参照されます。
  • apply_handle_insert_internal、apply_handle_update_internal、apply_handle_delete_internal などの内部関数に引数として渡され、実行コンテキストを提供します。
  • パーティションテーブルへの適用時には、apply_handle_tuple_routing を通じてルーティング情報が活用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/worker.c 内の適用処理関数群（apply_handle_insert_internal 等）および edata を生成する関数。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接関数を呼び出すことはありませんが、EState や ResultRelInfo を介して PostgreSQL の実行エンジン機能を利用します。

関連データ構造

  • LogicalRepRelMapEntry: レプリケーション対象リレーションのマップ情報。
  • ResultRelInfo: 実行エンジンにおけるターゲットリレーション情報。
  • EState: PostgreSQL 実行エンジンの状態管理構造体。
  • ModifyTableState, PartitionTupleRouting: パーティションテーブル操作に関連する構造体。

失敗モードと境界条件

  • メモリ不足時に palloc0 が失敗する可能性があります。
  • ターゲットリレーションの解決に失敗した場合や、パーティションルーティングが正しく構成されていない場合に、適用処理がエラーとなる可能性があります。

保守担当者向けメモ

  • 本構造体は worker.c 内の適用ロジックの中核を成します。
  • パーティションテーブルへの対応ロジックが複雑であるため、mtstate や proute のライフサイクル管理については、apply_handle_tuple_routing などの関連関数を参照してください。
