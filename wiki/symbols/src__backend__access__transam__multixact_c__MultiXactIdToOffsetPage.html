MultiXactIdToOffsetPage

定義

  • 種別: static inline 関数
  • 定義箇所: src/backend/access/transam/multixact.c:112

目的

指定された MultiXactId が格納されている SLRU（Simple Least Recently Used）領域のページ番号を算出します。

シグネチャまたは形状

int64 MultiXactIdToOffsetPage(MultiXactId multi)

入力と出力

  • **入力**: multi (MultiXactId) - ページ番号を特定したい対象の MultiXact ID。
  • **出力**: int64 - MultiXactOffsetCtl における論理ページ番号。

前提条件と不変条件

  • MULTIXACT_OFFSETS_PER_PAGE が 0 でないこと。
  • 入力される MultiXactId は有効な範囲内であること。

アルゴリズムまたは振る舞い

multi を MULTIXACT_OFFSETS_PER_PAGE で除算し、その商をページ番号として返します。これにより、MultiXact のオフセット情報を保持する SLRU ページを特定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/multixact.c 内の多数の関数（GetMultiXactIdMembers、MultiXactIdCreate、TruncateMultiXact 等）から、SLRU ページへのアクセス時に頻繁に呼び出されます。
  • **呼び出し先**:
  • 算術演算のみを行うため、他の関数呼び出しはありません。

関連データ構造

  • MultiXactId: MultiXact を識別する 32 ビット ID。
  • MultiXactOffsetCtl: オフセット情報を管理する SLRU 制御構造体。

失敗モードと境界条件

  • multi が 0 の場合、ページ 0 が返されます。
  • MULTIXACT_OFFSETS_PER_PAGE の値に依存するため、この定数が変更された場合、算出されるページ番号も変化します。

保守担当者向けメモ

  • 本関数は multixact.c 内の SLRU 操作の基盤となる計算ロジックです。
  • 呼び出し元で得られたページ番号は、さらに SLRU_PAGES_PER_SEGMENT を用いてセグメントファイル特定のために使用されるケースがあります。
