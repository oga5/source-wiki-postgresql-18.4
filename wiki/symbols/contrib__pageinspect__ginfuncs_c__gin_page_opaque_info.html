gin_page_opaque_info

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/ginfuncs.c:95

目的

GINインデックスの物理ページにおける不透明領域（Opaque）の情報を抽出し、人間が読み取れる形式で提供する。

シグネチャまたは形状

Datum gin_page_opaque_info(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: bytea 型の raw ページデータ（PG_GETARG_BYTEA_P(0)）。
  • **出力**: 以下の3つのフィールドを持つ複合型（HeapTuple）を返す。

    • 1. rightlink (int64): 右隣のページ番号。 2. maxoff (int32): ページ内の最大オフセット。 3. flags (text[]): ページフラグの人間が読み取れる名称の配列。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要。
  • 入力ページは GinPageOpaqueData 構造体と一致するサイズ（MAXALIGN 済み）の特殊領域を持つ必要がある。
  • PageIsNew である場合は NULL を返す。

アルゴリズムまたは振る舞い

1. **権限チェック**: superuser() を呼び出し、権限がない場合はエラーを報告する。 2. **ページ取得**: get_page_from_raw() で raw データを Page 構造体に変換する。 3. **検証**: ページが空でないこと、および特殊領域のサイズが GinPageOpaqueData と一致することを確認する。 4. **フラグ解析**: GinPageGetOpaque() で取得したフラグビットマスクを、data, leaf, deleted, meta, list, list_fullrow, incomplete_split, compressed の各文字列に変換する。認識できないビットは16進数文字列として追加する。 5. **結果構築**: rightlink, maxoff, およびフラグ配列を heap_form_tuple でタプル化して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（PG_FUNCTION_INFO_V1 マクロによる登録あり）。
  • **呼び出し先**: superuser(), get_page_from_raw(), PageIsNew(), PageGetSpecialSize(), GinPageGetOpaque(), get_call_result_type(), CStringGetTextDatum(), DirectFunctionCall1(), construct_array_builtin(), heap_form_tuple()。

関連データ構造

  • GinPageOpaqueData: GINページの特殊領域を定義する構造体。
  • Page: PostgreSQLのページ構造体。

失敗モードと境界条件

  • **権限不足**: スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー。
  • **無効なページ**: 特殊領域のサイズが期待値と異なる場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー。
  • **戻り値型不一致**: 呼び出し元の期待する型が複合型でない場合、エラー。

保守担当者向けメモ

  • 本関数は raw ページを直接操作するため、セキュリティ上の理由からスーパーユーザー権限を必須としている。
  • フラグの追加や変更を行う際は、GinPageOpaqueData の定義と整合性を取る必要がある。
