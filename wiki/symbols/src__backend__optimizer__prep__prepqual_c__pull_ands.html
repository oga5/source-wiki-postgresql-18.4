pull_ands

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepqual.c:322

目的

ネストされた AND 句を再帰的に平坦化し、単一のリスト構造に変換することで、最適化器が論理式を効率的に走査できるようにします。

シグネチャまたは形状

static List *pull_ands(List *andlist)

入力と出力

  • **入力**: andlist (論理式のリスト)
  • **出力**: 平坦化された Node のリスト (List *)

前提条件と不変条件

  • 入力リスト内の各要素は Node として扱われます。
  • AND 句がネストされている場合、再帰的に展開されます。

アルゴリズムまたは振る舞い

1. 入力された andlist の各要素を走査します。 2. 要素が AND 句 (is_andclause) である場合、その内部引数に対して pull_ands を再帰的に呼び出し、結果を現在のリストに結合 (list_concat) します。 3. 要素が AND 句でない場合、その要素をそのままリストの末尾に追加 (lappend) します。 4. 最終的に、ネストが解消された単一のリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/prep/prepqual.c 内の複数箇所（pull_ands 自身による再帰呼び出しを含む）。
  • 具体的な外部関数からの呼び出しについては、提供されたソース情報からは特定できません。
  • **呼び出し先**:
  • is_andclause (ノード判定)
  • list_concat / lappend (リスト操作)
  • pull_ands (再帰呼び出し)

関連データ構造

  • List: PostgreSQLのリスト構造。
  • BoolExpr: AND 句を表現するノード構造体。

失敗モードと境界条件

  • 空のリストが入力された場合、空のリストを返します。
  • AND 句を含まないリストが入力された場合、入力をそのまま保持したリストを返します。

保守担当者向けメモ

  • 本関数は、変換によって意図せずネストが生じた場合に呼び出される補助的な役割を担います。
  • pull_ors と対になる関数であり、論理式の構造を整理する前処理の一部として機能します。
