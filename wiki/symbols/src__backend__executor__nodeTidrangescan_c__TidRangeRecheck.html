TidRangeRecheck

定義

  • **種別**: function
  • **定義箇所**: src/backend/executor/nodeTidrangescan.c:274

目的

EvalPlanQual（EPQ）実行時に、対象タプルが現在も有効な TID 範囲内にあるかを再検証する。

シグネチャまたは形状

static bool TidRangeRecheck(TidRangeScanState *node, TupleTableSlot *slot)

入力と出力

  • **入力**:
  • node: TidRangeScanState 構造体へのポインタ。現在のスキャン状態および TID 範囲（trss_mintid, trss_maxtid）を保持。
  • slot: 検証対象のタプルを保持する TupleTableSlot。
  • **出力**:
  • bool: タプルが範囲内であれば true、範囲外または評価失敗時は false を返す。

前提条件と不変条件

  • slot->tts_tid に有効な ItemPointer が格納されている必要がある（Assert により検証済み）。

アルゴリズムまたは振る舞い

1. TidRangeEval(node) を呼び出し、最新の TID 範囲を評価・更新する。評価が失敗した場合は false を返す。 2. slot 内の ctid と、node 内の trss_mintid（下限）および trss_maxtid（上限）を ItemPointerCompare で比較する。 3. ctid が trss_mintid 未満、または trss_maxtid を超えている場合は false を返す。 4. 上記以外の場合は true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン内の ExecScanRecheckMtd インターフェース経由で呼び出される（src/backend/executor/nodeTidrangescan.c:312）。
  • **呼び出し先**:
  • TidRangeEval: TID 範囲の再計算を行う。
  • ItemPointerCompare: TID の大小比較を行う。
  • ItemPointerIsValid: TID の有効性チェックを行う。

関連データ構造

  • TidRangeScanState: スキャンノードの状態を管理する構造体。
  • TupleTableSlot: タプルデータと物理位置（ctid）を保持する構造体。

失敗モードと境界条件

  • TidRangeEval が範囲を算出できない場合、false を返す。
  • slot->tts_tid が無効な場合、Assert により異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は ExecScanRecheckMtd の実装として登録されており、EPQ 処理において一貫性を保つために使用される。
  • ItemPointerCompare の結果に基づき厳密な範囲判定を行っているため、範囲の包含関係（inclusive）の変更が必要な場合は TidRangeEval 側のロジックと整合性をとること。
