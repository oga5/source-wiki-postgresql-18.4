ExecGatherMerge

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGatherMerge.c:182

目的

並列クエリ実行において、複数のワーカープロセスから送られてくるソート済みのタプルストリームをマージし、単一の順序付きストリームとして出力する。

シグネチャまたは形状

TupleTableSlot *ExecGatherMerge(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate （GatherMergeState 構造体としてキャストされる）
  • **出力**: マージされた次の TupleTableSlot。ストリームが終了した場合は NULL を返す。

前提条件と不変条件

  • ノードが未初期化（node->initialized が false）の場合、ワーカーの起動とリーダーの参加設定が行われる。
  • 並列モードが有効かつワーカーが存在する場合、LaunchParallelWorkers によりワーカーが起動される。
  • 実行ごとに ResetExprContext が呼び出され、前回のタプルサイクルで割り当てられた式評価用メモリが解放される。

アルゴリズムまたは振る舞い

1. **初期化**: 初回実行時にワーカーを起動し、タプルキューリーダーを作成する。リーダープロセスが参加すべき場合は need_to_scan_locally を設定する。 2. **メモリリセット**: ResetExprContext を呼び出し、式評価用コンテキストをクリーンアップする。 3. **タプル取得**: gather_merge_getnext を呼び出し、バイナリヒープを用いて各ソースから最小のタプルを選択する。 4. **終了判定**: 取得したスロットが NULL であれば終了。 5. **投影**: ps_ProjInfo が存在する場合、ExecProject を適用して結果を整形し、それ以外の場合は取得したスロットをそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（ExecProcNode 経由での呼び出しが想定される）。
  • **呼び出し先**:
  • ExecInitParallelPlan, ExecParallelReinitialize, LaunchParallelWorkers, ExecParallelCreateReaders（並列実行制御）
  • gather_merge_getnext（タプル取得）
  • ExecProject（投影処理）
  • ResetExprContext（メモリ管理）

関連データ構造

  • GatherMergeState: 本ノードの実行状態を保持する構造体。
  • TupleTableSlot: タプルを保持するコンテナ。
  • ParallelContext: 並列実行のコンテキスト。
  • TupleQueueReader: ワーカーからのタプル読み込み用リーダー。

失敗モードと境界条件

  • **ワーカー不在**: ワーカーが起動できない、または存在しない場合、リーダープロセスのみで処理を行う。
  • **タプル枯渇**: 全てのソースからタプルが供給されなくなった場合、gather_merge_getnext は NULL を返し、本関数も NULL を返す。
  • **割り込み**: CHECK_FOR_INTERRUPTS() により、実行中に割り込み要求がある場合は処理を中断する。

保守担当者向けメモ

  • node->initialized フラグにより、ワーカーの起動は初回実行時まで遅延される。
  • リーダープロセスの参加可否は parallel_leader_participation 設定またはリーダーが唯一の読み込み元であるかによって決定される。
  • ps_ProjInfo が設定されている場合、投影処理が必須となるため、ExecProject の戻り値が最終的な出力となる。
