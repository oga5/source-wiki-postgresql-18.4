inet_aton

定義

  • 種別: function
  • 定義箇所: src/port/inet_aton.c:55

目的

ドット区切りの IPv4 アドレス文字列をバイナリ形式（struct in_addr）に変換し、ネットワークバイトオーダーで格納する。標準 C ライブラリに本関数が存在しない環境向けの移植性レイヤーとして機能する。

シグネチャまたは形状

int inet_aton(const char *cp, struct in_addr *addr);

入力と出力

  • **入力**:
  • cp: 解析対象の IPv4 アドレス文字列（例: 127.0.0.1）。
  • addr: 変換後のバイナリ値を格納する struct in_addr へのポインタ。
  • **出力**:
  • 戻り値: 成功時に 1、失敗時に 0 を返す。
  • addr: 成功時、ネットワークバイトオーダーに変換された 32 ビット値が格納される。

前提条件と不変条件

  • 入力文字列は、10進数、8進数（0 プレフィックス）、16進数（0x プレフィックス）の表記をサポートする。
  • 最後のパートに続く不要な文字（空白を除く）は不正とみなされる。
  • 各パートの値は、指定されたビット幅（8/16/24ビット）の範囲内に収まる必要がある。

アルゴリズムまたは振る舞い

1. **解析ループ**: 文字列を . で分割し、各パートを基数（8, 10, 16）に基づいて数値化する。 2. **検証**: パート数に応じて、各数値が許容範囲内かを確認する。 3. **構築**: パート数（1〜4）に応じてビットシフトを行い、32 ビット整数を組み立てる。 4. **変換**: pg_hton32 を用いてホストバイトオーダーからネットワークバイトオーダーへ変換し、addr->s_addr に書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq/fe-secure-common.c
  • src/interfaces/libpq/fe-secure-openssl.c
  • **呼び出し先**:
  • pg_hton32（port/pg_bswap.h）
  • isdigit, isxdigit, islower, isspace（ctype.h）

関連データ構造

  • struct in_addr: ネットワークアドレスを保持する構造体（<netinet/in.h>）。

失敗モードと境界条件

  • 文字列が不正な形式である場合、または末尾に空白以外の不要な文字が含まれる場合は 0 を返す。
  • 各パートの値が許容範囲（例: 8ビットを超える値）を超えている場合は 0 を返す。
  • addr が NULL の場合、変換処理はスキップされるが、構文解析は行われ、成功時には 1 が返される。

保守担当者向けメモ

  • GNU C ライブラリの古い実装をベースとしている。
  • inet_addr() と異なり、戻り値で成功・失敗を明確に区別できる。
  • 移植性レイヤーであるため、修正時は src/port の他のネットワーク関連関数との整合性に注意すること。
