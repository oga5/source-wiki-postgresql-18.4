ExecDeletePrologue

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeModifyTable.c:1459

目的

DELETE 操作の実行前処理として、BEFORE ROW DELETE トリガーの実行および保留中の挿入操作のフラッシュを行う。

シグネチャまたは形状

bool ExecDeletePrologue(ModifyTableContext *context, ResultRelInfo *resultRelInfo,
                        ItemPointer tupleid, HeapTuple oldtuple,
                        TupleTableSlot **epqreturnslot, TM_Result *result)

入力と出力

  • **入力**:
  • context: ModifyTableContext 構造体（実行状態やトランザクション情報）。
  • resultRelInfo: ターゲットリレーション情報。
  • tupleid: 削除対象タプルの ItemPointer。
  • oldtuple: 削除対象の HeapTuple。
  • **出力**:
  • epqreturnslot: トリガー実行時に EPQ (EvalPlanQual) が必要な場合、そのスロットが格納される。
  • result: TM_Result 型の実行結果（成功時は TM_Ok）。
  • 戻り値: トリガー処理が正常に完了した場合は true、それ以外は false。

前提条件と不変条件

  • result 引数が非 NULL の場合、初期値として TM_Ok が設定される。
  • context->estate が有効であること。

アルゴリズムまたは振る舞い

1. result が提供されている場合、TM_Ok で初期化する。 2. ターゲットリレーションに trig_delete_before_row (BEFORE ROW DELETE トリガー) が定義されているか確認する。 3. トリガーが存在する場合:

  • context->estate に保留中の挿入 (es_insert_pending_result_relations) があれば、ExecPendingInserts を呼び出してフラッシュし、トリガーから挿入行を参照可能にする。
  • ExecBRDeleteTriggers を呼び出し、トリガーを実行する。この際、現在の操作が CMD_MERGE かどうかを判定して渡す。

    • 4. トリガーが存在しない場合は、直ちに true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeModifyTable.c 内の ExecDelete 関数（複数箇所）。
  • **呼び出し先**:
  • ExecPendingInserts
  • ExecBRDeleteTriggers

関連データ構造

  • ModifyTableContext: 実行ノードの状態管理。
  • ResultRelInfo: ターゲットテーブルのメタデータとトリガー記述子。
  • TM_Result: テーブル操作の結果ステータス。

失敗モードと境界条件

  • トリガー実行中にエラーが発生した場合、ExecBRDeleteTriggers の挙動に従い例外がスローされる可能性がある。
  • 保留中の挿入がない場合、ExecPendingInserts はスキップされる。

保守担当者向けメモ

  • 本関数は ExecDelete の前処理をカプセル化している。トリガー実行の順序や、保留中の挿入の可視性制御（ExecPendingInserts）がこの関数の重要な責務である。
