TransformJsonStringValuesState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonfuncs.c:76

目的

JSONデータ内の各文字列値に対して、特定の変換処理を適用しながらJSONを再構築するための状態保持構造体です。

シグネチャまたは形状

typedef struct TransformJsonStringValuesState
{
	JsonLexContext *lex;
	StringInfo	strval;
	JsonTransformStringValuesAction action;
	void	   *action_state;
} TransformJsonStringValuesState;

入力と出力

  • **入力**: JsonLexContext を介して解析対象のJSONテキストを受け取ります。
  • **出力**: StringInfo 型の strval に、変換後のJSON文字列が格納されます。

前提条件と不変条件

  • lex は有効な JsonLexContext を指している必要があります。
  • action は、JSON内の各値に対して適用される関数ポインタ（JsonTransformStringValuesAction）として設定されている必要があります。
  • action_state は、変換処理に必要な任意のコンテキスト情報を保持するポインタです。

アルゴリズムまたは振る舞い

1. palloc0 を用いて構造体のメモリを確保し、初期化します。 2. JSONの解析中、各文字列値に遭遇するたびに action で指定された変換関数が呼び出されます。 3. 変換結果は strval に蓄積され、最終的なJSON構造が再構築されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonfuncs.c 内の複数の関数（行5858, 5888, 5898, 5908, 5918, 5928, 5946, 5957）から、この構造体のインスタンス化およびキャストが行われています。
  • **呼び出し先**: action フィールドに格納された関数ポインタを介して、具体的な変換ロジックが実行されます。

関連データ構造

  • JsonLexContext: JSON解析のコンテキスト。
  • StringInfo: 文字列構築用のバッファ。
  • JsonTransformStringValuesAction: 文字列値に対する変換アクションの型定義。

失敗モードと境界条件

  • メモリ不足時: palloc0 がエラーを発生させます。
  • 解析エラー: lex が不正なJSONを読み込んだ場合、JsonLexContext の仕様に従いエラーが発生します。

保守担当者向けメモ

  • 本構造体は、JSONの変換処理における状態管理を目的としています。action_state は汎用ポインタであるため、キャスト時の型安全性に注意してください。
  • 呼び出し元で TransformJsonStringValuesState * へのキャストが頻繁に行われているため、変更時はこれらの箇所との整合性を確認してください。
