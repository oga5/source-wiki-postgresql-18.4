LockTagType

定義

  • 種別: enum
  • 定義箇所: src/include/storage/lock.h:136

目的

PostgreSQLのロックマネージャにおいて、ロック対象となるオブジェクトの種類を識別するために使用されます。

シグネチャまたは形状

typedef enum LockTagType
{
    LOCKTAG_RELATION,
    LOCKTAG_RELATION_EXTEND,
    LOCKTAG_DATABASE_FROZEN_IDS,
    LOCKTAG_PAGE,
    LOCKTAG_TUPLE,
    LOCKTAG_TRANSACTION,
    LOCKTAG_VIRTUALTRANSACTION,
    LOCKTAG_SPECULATIVE_TOKEN,
    LOCKTAG_OBJECT,
    LOCKTAG_USERLOCK,
    LOCKTAG_ADVISORY,
    LOCKTAG_APPLY_TRANSACTION
} LockTagType;

入力と出力

  • 入力: なし（列挙型定数）
  • 出力: ロック対象の種別を識別する整数値（列挙値）

前提条件と不変条件

  • LOCKTAG 構造体内の locktag_type フィールドとして格納されます。
  • LOCKTAG は16バイトに収まるよう設計されており、本列挙型の追加や変更は LOCKTAG のサイズ制約に影響を与える可能性があります。

アルゴリズムまたは振る舞い

  • ロック取得時、LockAcquire 等の関数内で LOCKTAG からハッシュ値を計算する際の識別子として利用されます。
  • ロックの競合判定や管理において、対象オブジェクトの性質（リレーション、タプル、トランザクション等）を分岐させるためのスイッチ条件として機能します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/storage/lmgr/lmgr.c
  • src/backend/utils/adt/lockfuncs.c
  • src/backend/utils/cache/relcache.c
  • src/include/storage/lock.h (マクロ LOCK_LOCKTAG, LOCALLOCK_LOCKTAG 内)
  • 呼び出し先: なし（列挙型のため）

関連データ構造

  • LOCKTAG: 本型を保持する主要な構造体。
  • LOCK: 共有メモリ上のロックオブジェクト。
  • LOCALLOCK: バックエンドローカルのロックキャッシュ。

失敗モードと境界条件

  • 未定義の LockTagType 値が LOCKTAG に設定された場合、ロックマネージャのハッシュ計算や競合判定ロジックで予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 新しいロック対象を追加する場合は、本列挙型への追加に加え、関連する switch 文（lockfuncs.c や lmgr.c 等）の網羅的な更新が必要です。
  • src/backend/utils/adt/lockfuncs.c:24 にある通り、変更時はドキュメントの更新が推奨されています。
