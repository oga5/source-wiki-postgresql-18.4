_include_path

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/preproc/type.h:131

目的

ECPGプリプロセッサがヘッダーファイルを検索する際のパス情報を保持する連結リスト構造体です。

シグネチャまたは形状

struct _include_path
{
	char	   *path;
	struct _include_path *next;
};

入力と出力

  • path: 検索対象のディレクトリパスを指す文字列へのポインタ。
  • next: リスト内の次の要素を指すポインタ。

前提条件と不変条件

  • next ポインタは、リストの終端を示すために NULL で終了する必要があります。
  • path は有効なディレクトリパス文字列を指している必要があります。

アルゴリズムまたは振る舞い

  • プリプロセッサの実行時に、コマンドライン引数等で指定されたインクルードパスを順次格納する連結リストとして機能します。
  • プリプロセッサがファイルを検索する際、このリストを辿ることで探索対象のディレクトリを決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/preproc/ecpg.c において、インクルードパスのリスト管理（初期化、追加）に使用されています。
  • **呼び出し先**: mm_alloc（src/interfaces/ecpg/preproc/ecpg.c:73）により、構造体インスタンスのメモリ確保が行われます。

関連データ構造

  • struct _include_path *include_paths: src/interfaces/ecpg/preproc/preproc_extern.h で宣言されているグローバルなリストの起点。

失敗モードと境界条件

  • メモリ不足時: mm_alloc が失敗した場合、プリプロセッサの動作が停止する可能性があります。
  • リストが空の場合: include_paths が NULL で初期化されており、検索対象のパスが存在しない状態を許容します。

保守担当者向けメモ

  • 本構造体は ecpg.c 内で動的に生成・管理されます。リストの走査や解放を行う際は、next ポインタを辿る再帰的または反復的な処理が必要です。
