pgwin32_system

定義

  • 種別: function
  • 定義箇所: src/port/system.c:52

目的

Windows環境において、cmd.exe の特殊な引用符（クォート）処理ルールに起因する引数解釈エラーを回避し、安全にコマンドを実行するための標準関数 system() のラッパーです。

シグネチャまたは形状

int pgwin32_system(const char *command)

入力と出力

  • **入力**: 実行するコマンド文字列（const char *）。
  • **出力**: 実行されたコマンドの終了ステータス（int）。メモリ確保に失敗した場合は -1 を返し、errno を ENOMEM に設定します。

前提条件と不変条件

  • 本関数は WIN32 かつ __CYGWIN__ ではない環境でのみ有効です。
  • 呼び出し側で既に引用符が含まれる複雑なコマンドを渡す場合、呼び出し側で適切にエスケープされている必要があります。

アルゴリズムまたは振る舞い

1. 入力されたコマンド文字列の長さを取得します。 2. 元の文字列長に引用符2バイトと終端文字1バイトを加えたサイズのメモリを確保します。 3. 確保したバッファの先頭と末尾に " を挿入し、中央に元のコマンド文字列をコピーします。 4. 引用符で囲まれた新しい文字列を標準の system() 関数に渡して実行します。 5. 実行結果を保持し、確保したバッファを解放したのち、実行時の errno を復元して呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port.h にて system マクロとして定義されており、PostgreSQLの各所から標準の system() の代わりとして呼び出されます。
  • **呼び出し先**: 標準ライブラリの system() を呼び出します。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • malloc が失敗した場合、errno を ENOMEM に設定し、-1 を返します。
  • cmd.exe の仕様上、文字列全体を引用符で囲むアプローチをとるため、入力文字列自体に特殊な引用符が含まれる場合の挙動は呼び出し側のエスケープ処理に依存します。

保守担当者向けメモ

  • 本関数は src/include/port.h で system をオーバーライドしているため、Windows環境での移植性維持に不可欠です。
  • 修正を行う際は、src/port/system.c の他の関数（pgwin32_popen など）と一貫した引用符処理が行われているか確認してください。
