calc_hist_selectivity_scalar

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/multirangetypes_selfuncs.c:706

目的

境界値ヒストグラムを用いて、指定された定数（constbound）がレンジ型データの分布においてどの程度の割合（選択率）を占めるかを算出します。

シグネチャまたは形状

static double calc_hist_selectivity_scalar(
    TypeCacheEntry *typcache, 
    const RangeBound *constbound,
    const RangeBound *hist, 
    int hist_nvalues, 
    bool equal
)

入力と出力

  • **入力**:
  • typcache: 型情報のキャッシュ。
  • constbound: 比較対象の境界値。
  • hist: 境界値ヒストグラムの配列。
  • hist_nvalues: ヒストグラムのビン数。
  • equal: 比較演算に等号を含むかどうかのフラグ。
  • **出力**:
  • double: 0.0 から 1.0 の範囲で表される選択率。

前提条件と不変条件

  • hist_nvalues はヒストグラムの要素数であり、計算の分母として使用されるため、通常は 2 以上であることが想定されます。
  • typcache は有効な型情報キャッシュを指している必要があります。

アルゴリズムまたは振る舞い

1. **ビン特定**: rbound_bsearch を呼び出し、constbound がヒストグラムのどのビンに該当するかを特定します。 2. **基本選択率算出**: 特定されたビンインデックスに基づき、それより前の全ビン分を合計した選択率を算出します。 3. **線形補間**: index が有効な範囲内（ヒストグラムの最後尾ではない）にある場合、get_position を用いてビン内での相対位置を計算し、線形補間によって選択率を微調整します。 4. **戻り値**: 最終的な選択率を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/multirangetypes_selfuncs.c 内の複数の箇所から、レンジ型の比較演算（<, <=, >, >= 等）の選択率推定のために呼び出されています。
  • **呼び出し先**:
  • rbound_bsearch: ヒストグラム内の位置特定。
  • get_position: ビン内での相対位置の計算。
  • Max (マクロ): インデックスの最小値保証。

関連データ構造

  • RangeBound: レンジ型の境界値を保持する構造体。
  • TypeCacheEntry: 型の比較関数やサブタイプ情報を含むキャッシュ構造体。

失敗モードと境界条件

  • hist_nvalues が 1 以下の場合、除算によるゼロ除算や不正な計算が発生する可能性があります。
  • rbound_bsearch がヒストグラム外のインデックスを返した場合、補間ロジックがスキップされるか、境界値付近での精度が低下する可能性があります。

保守担当者向けメモ

  • 本関数は線形補間を前提としています。サブタイプごとの subdiff 関数が適切に実装されていない場合、補間精度が低下するため注意してください。
  • 統計情報の更新（ANALYZE）によりヒストグラムの精度が向上するため、推定値が不自然な場合は統計情報の鮮度を確認してください。
