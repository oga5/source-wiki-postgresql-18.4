CreateSocketLockFile

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/miscinit.c:1523

目的

Unixドメインソケットファイルに対応するロックファイルを作成し、ソケットの二重使用や競合を防止する。

シグネチャまたは形状

void CreateSocketLockFile(const char *socketfile, bool amPostmaster, const char *socketDir)

入力と出力

  • socketfile (入力): ロック対象となるソケットファイルのパス。
  • amPostmaster (入力): 呼び出し元がPostmasterプロセスであるかを示すフラグ。
  • socketDir (入力): ソケットファイルが配置されているディレクトリパス。
  • 出力: なし（成功時はロックファイルを作成、失敗時はエラーをスロー）。

前提条件と不変条件

  • socketfile に .lock を付加したパスがロックファイルとして使用される。
  • 内部で CreateLockFile を呼び出すため、同関数の排他制御およびファイルシステム権限の制約に従う。

アルゴリズムまたは振る舞い

1. socketfile 引数に .lock を連結し、lockfile パスを生成する。 2. 生成した lockfile パスを引数として CreateLockFile を呼び出し、実際のロック作成処理を委譲する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/libpq/pqcomm.c:699
  • 呼び出し先: CreateLockFile (同ファイル内)

関連データ構造

  • MAXPGPATH: ファイルパスの最大長を規定する定数。

失敗モードと境界条件

  • CreateLockFile の動作に依存する。指定されたパスへの書き込み権限がない場合や、既にロックファイルが存在し排他ロックが取得できない場合に失敗する。

保守担当者向けメモ

  • 本関数は CreateLockFile のラッパーであり、ソケットファイル固有の命名規則（.lock の付与）を管理している。
  • ロックファイルの削除は proc_exit に登録された処理に依存するため、プロセスが異常終了した場合はロックファイルが残存する可能性がある。
