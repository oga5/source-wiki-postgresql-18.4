RemoveHead

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_parse.c:111

目的

LexizeData 構造体において、現在処理中のトークンリスト（towork）の先頭要素を削除し、処理済みリスト（waste）へ移動させるとともに、辞書位置インデックスをリセットします。

シグネチャまたは形状

static void RemoveHead(LexizeData *ld)

入力と出力

  • 入力: LexizeData *ld （辞書処理の状態を保持する構造体へのポインタ）
  • 出力: なし（副作用として ld の内部状態を更新）

前提条件と不変条件

  • ld->towork リストが空でないことが前提となります（LPLRemoveHead の呼び出しを伴うため）。

アルゴリズムまたは振る舞い

1. ld->towork リストから先頭要素を LPLRemoveHead で取り出します。 2. 取り出した要素を ld->waste リストの末尾に LPLAddTail で追加します。 3. ld->posDict を 0 に設定し、辞書チェーンの走査位置をリセットします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tsearch/ts_parse.c 内の複数箇所（153, 198, 239, 244行目）。
  • 呼び出し先: LPLAddTail および LPLRemoveHead。

関連データ構造

  • LexizeData: 辞書処理の現在の状態（辞書チェーン、作業中のトークンリスト、一時的な結果）を保持する構造体。

失敗モードと境界条件

  • ld->towork が空の状態で呼び出された場合の挙動については、本ソース抜粋からは判断できません。

保守担当者向けメモ

  • 本関数は LexizeExec における辞書チェーンの走査過程で、トークンの移動と状態リセットを担うユーティリティです。
  • posDict のリセットは、次の辞書適用がリストの先頭から行われることを保証するために重要です。
