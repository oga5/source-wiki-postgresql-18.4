tsvector_delete_str

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector_op.c:553

目的

指定された tsvector から特定の語彙（lexeme）を検索し、該当する要素を削除した新しい tsvector を生成する。

シグネチャまたは形状

Datum tsvector_delete_str(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TSVECTOR(0): 操作対象の tsvector。
  • PG_GETARG_TEXT_PP(1): 削除対象の語彙を表す text 型データ。
  • **出力**:
  • 削除後の tsvector を含む Datum。語彙が見つからない場合は元の tsvector をそのまま返す。

前提条件と不変条件

  • tsvector は可変長データ型であり、メモリ管理には palloc および PG_FREE_IF_COPY が適切に使用される必要がある。
  • 削除対象の語彙は tsvector 内の語彙リストと正確に一致する必要がある。

アルゴリズムまたは振る舞い

1. tsvector_bsearch を使用して、入力された tsvector 内から対象語彙のインデックスを検索する。 2. 語彙が見つからない場合（-1 が返された場合）、入力された tsvector をそのまま返す。 3. 語彙が見つかった場合、tsvector_delete_by_indices を呼び出し、該当インデックスの要素を除外した新しい tsvector を作成する。 4. 入力された引数に対して PG_FREE_IF_COPY を実行し、メモリを解放する。 5. 生成された新しい tsvector を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報には、本関数を呼び出す具体的な関数名は記載されていない。
  • **呼び出し先**:
  • tsvector_bsearch: 語彙の検索に使用。
  • tsvector_delete_by_indices: 語彙の削除処理に使用。

関連データ構造

  • TSVector: 全文検索用ベクトルの型定義。
  • text: PostgreSQL の可変長文字列型。

失敗モードと境界条件

  • 対象の語彙が tsvector 内に存在しない場合、エラーにはならず、元の tsvector がそのまま返される。
  • メモリ不足時には palloc 等の内部機構によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は「読み取り・フィルタリング・書き込み」のパターンに従っている。
  • tsvector の内部構造（WordEntry 等）を直接操作するのではなく、ユーティリティ関数（tsvector_bsearch, tsvector_delete_by_indices）を介して安全に操作している。
