MakeConfigurationMapping

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tsearchcmds.c:1287

目的

全文検索設定（pg_ts_config）におけるトークン型と辞書のマッピングを、システムカタログ pg_ts_config_map に対して追加、置換、または削除するために使用されます。

シグネチャまたは形状

static void MakeConfigurationMapping(AlterTSConfigurationStmt *stmt, HeapTuple tup, Relation relMap)

入力と出力

  • **入力**:
  • stmt: 実行する ALTER TEXT SEARCH CONFIGURATION 文の構文解析結果。
  • tup: 対象となる pg_ts_config カタログのタプル。
  • relMap: 操作対象の pg_ts_config_map リレーション。
  • **出力**: なし（副作用としてシステムカタログを更新）。

前提条件と不変条件

  • tup は pg_ts_config の有効なタプルであること。
  • relMap は pg_ts_config_map カタログを指す開かれたリレーションであること。
  • stmt 内の辞書名は get_ts_dict_oid によって解決可能であること。

アルゴリズムまたは振る舞い

1. **トークン解決**: getTokenTypes を使用し、パーサー ID に基づいて対象トークン型リストを取得します。 2. **削除処理**: stmt->override が真の場合、対象トークン型に関連する既存のマッピングを pg_ts_config_map から削除します。 3. **辞書解決**: stmt->dicts に含まれる辞書名を OID に変換します。 4. **置換処理**: stmt->replace が真の場合、指定されたトークン型に関連付けられた旧辞書を新辞書に更新します。 5. **新規追加**: stmt->replace が偽の場合、トークン型と辞書の組み合わせを pg_ts_config_map に挿入します。この際、MAX_CATALOG_MULTI_INSERT_BYTES に基づき、効率化のためバッチ挿入（CatalogTuplesMultiInsertWithInfo）を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/tsearchcmds.c 内の AlterTSConfiguration から呼び出されます。
  • **呼び出し先**: getTokenTypes, get_ts_dict_oid, systable_beginscan, CatalogTupleDelete, heap_modify_tuple, CatalogTupleUpdateWithInfo, CatalogTuplesMultiInsertWithInfo 等のカタログ操作関数。

関連データ構造

  • AlterTSConfigurationStmt: SQL 文の解析構造体。
  • Form_pg_ts_config: pg_ts_config カタログの構造。
  • Form_pg_ts_config_map: pg_ts_config_map カタログの構造。
  • TSTokenTypeItem: トークン型情報を保持する構造体。

失敗モードと境界条件

  • 指定された辞書名が存在しない場合、get_ts_dict_oid によりエラーが発生します。
  • stmt->replace 時に一致するトークン型や辞書が見つからない場合、更新は行われません。
  • 大量のエントリ追加時は、CatalogTuplesMultiInsertWithInfo によるバッチ処理でメモリ制限（MAX_CATALOG_MULTI_INSERT_BYTES）を考慮した分割が行われます。

保守担当者向けメモ

  • 本関数は ALTER TEXT SEARCH CONFIGURATION のマッピング変更ロジックを集中管理しています。
  • pg_ts_config_map のインデックス（TSConfigMapIndexId）を利用したスキャンを行っているため、カタログ定義変更時はインデックス定義との整合性に注意してください。
