outToken

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/outfuncs.c:154

目的

PostgreSQLのノード構造体に含まれる文字列データを、readfuncs.c による復元が可能な形式へエスケープして StringInfo バッファへ書き出します。

シグネチャまたは形状

void outToken(StringInfo str, const char *s)

入力と出力

  • **入力**:
  • str: 書き込み先の StringInfo バッファ。
  • s: エスケープ対象のヌル終端文字列。
  • **出力**: なし（str が指すバッファを更新）。

前提条件と不変条件

  • s が NULL の場合、<> を出力します。
  • s が空文字列（""）の場合、"" を出力します。
  • 復元時に特殊な意味を持つ文字（空白、改行、タブ、括弧、波括弧、バックスラッシュ）は、常にバックスラッシュでエスケープされます。
  • 文字列の先頭が <、"、数字、または符号付き数値（+ や - で始まる数値）である場合、先頭にバックスラッシュを付加して識別を容易にします。

アルゴリズムまたは振る舞い

1. s が NULL なら <> を書き込み終了。 2. s が空なら "" を書き込み終了。 3. 文字列の先頭文字が特定の予約文字（<, ", 数字, 符号付き数値）に該当する場合、\ を先行して書き込みます。 4. 文字列全体を走査し、各文字が (空白), \n, \t, (, ), {, }, \ のいずれかであれば、その前に \ を挿入してから文字を書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/outfuncs.c 内の各ノード出力関数（例: node->fldname の処理や node->sval の処理など）。
  • **呼び出し先**: appendStringInfoString および appendStringInfoChar（lib/stringinfo.h）。

関連データ構造

  • StringInfo: 文字列構築用バッファ。
  • nodes/nodes.h: ノード定義。

失敗モードと境界条件

  • s が NULL の場合、クラッシュせずに <> を出力して安全に処理します。
  • s が空文字列の場合、"" を出力して空であることを明示します。
  • メモリ不足時は appendStringInfo 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は readfuncs.c との整合性が極めて重要です。エスケープルールの変更は、必ず readfuncs.c 側のパースロジックと同期させてください。
  • readfuncs.c のコメントによると、outToken が出力する <> は復元時に空文字列として扱われる仕様です。
