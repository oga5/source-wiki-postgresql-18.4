find_str

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-armor.c:242

目的

指定されたメモリ領域（バイト配列）内から、特定の文字列を検索します。

シグネチャまたは形状

const uint8 *find_str(const uint8 *data, const uint8 *data_end, const char *str, int strlen)

入力と出力

  • **入力**:
  • data: 検索対象領域の開始ポインタ。
  • data_end: 検索対象領域の終了ポインタ（排他的）。
  • str: 検索する文字列。
  • strlen: 検索する文字列の長さ。
  • **出力**:
  • 見つかった場合はその位置を示すポインタを返します。
  • 見つからない場合、または入力が不正な場合は NULL を返します。

前提条件と不変条件

  • strlen が 0 の場合は常に NULL を返します。
  • 検索対象の領域サイズ（data_end - data）が strlen 未満の場合は NULL を返します。

アルゴリズムまたは振る舞い

1. memchr を使用して、検索対象領域から文字列の最初の 1 バイトを効率的に探します。 2. 最初の 1 バイトが見つかった位置から、残りの領域が strlen を満たしているか確認します。 3. memcmp を使用して、その位置から strlen 分のデータが str と一致するか検証します。 4. 一致すればそのポインタを返し、一致しなければ次の位置へ進めて検索を継続します。 5. 領域の最後まで一致が見つからなければ NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-armor.c 内のヘッダー解析処理（find_header 等）から呼び出されています。
  • **呼び出し先**: memchr および memcmp（標準 C ライブラリ関数）。

関連データ構造

  • uint8: PostgreSQL の c.h 等で定義される符号なし 8 ビット整数型。

失敗モードと境界条件

  • 検索対象領域の末尾付近で文字列長が収まらない場合、正しく NULL を返します。
  • strlen が負または 0 の場合、検索を行わず NULL を返します。

保守担当者向けメモ

  • 本関数はバイナリデータ内での文字列検索を目的としており、ヌル終端文字列を前提としないため、strlen 引数の正確な指定が必須です。
  • memchr を利用しているため、検索対象のデータ内に str[0] が頻出する場合でも比較的効率的に動作します。
