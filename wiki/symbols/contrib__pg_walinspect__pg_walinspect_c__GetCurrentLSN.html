GetCurrentLSN

定義

  • 種別: function
  • 定義箇所: contrib/pg_walinspect/pg_walinspect.c:71

目的

PostgreSQLサーバーの現在のWAL（Write-Ahead Log）位置（LSN）を取得します。主にWALの読み取り範囲を決定する際の終端位置として使用されます。

シグネチャまたは形状

static XLogRecPtr GetCurrentLSN(void)

入力と出力

  • 入力: なし
  • 出力: XLogRecPtr 型の現在のWAL位置。

前提条件と不変条件

  • 戻り値の curr_lsn は常に有効な値（XLogRecPtrIsInvalid が偽）であることがアサーションにより保証されています。

アルゴリズムまたは振る舞い

1. サーバーのリカバリ状態を確認します。 2. リカバリ中でない場合（!RecoveryInProgress()）、GetFlushRecPtr(NULL) を呼び出し、フラッシュされた最新のLSNを取得します。 3. リカバリ中の場合、GetXLogReplayRecPtr(NULL) を呼び出し、現在リプレイ中のLSNを取得します。 4. 取得したLSNを返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/pg_walinspect/pg_walinspect.c 内の複数の関数（pg_get_wal_records_info、pg_get_wal_stats、pg_get_wal_records_info_till_end_of_wal、pg_get_wal_stats_till_end_of_wal など）から、WAL読み取りの終端位置を特定するために呼び出されています。
  • 呼び出し先:
  • RecoveryInProgress()
  • GetFlushRecPtr(NULL)
  • GetXLogReplayRecPtr(NULL)

関連データ構造

  • XLogRecPtr: WALの論理位置を表すデータ型。

失敗モードと境界条件

  • サーバーの状態（リカバリ中か否か）に応じて異なる取得関数を選択するため、サーバーの現在の動作モードに依存します。
  • 取得したLSNが不正な値である場合はアサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は pg_waldump 等のツールと同様のロジックで現在のLSNを決定しています。変更が必要な場合は、PostgreSQLのWAL管理メカニズムとの整合性に注意してください。
