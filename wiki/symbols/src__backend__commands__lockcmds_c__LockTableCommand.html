LockTableCommand

定義

  • 種別: function
  • 定義箇所: src/backend/commands/lockcmds.c:40

目的

SQL の LOCK TABLE コマンドを実行し、指定されたリレーションおよび必要に応じてその依存関係（継承テーブルやビュー）に対して明示的なロックを獲得する。

シグネチャまたは形状

void LockTableCommand(LockStmt *lockstmt)

入力と出力

  • **入力**: LockStmt *lockstmt（実行対象のリレーションリスト、ロックモード、および NOWAIT オプションを含む構造体）。
  • **出力**: なし（成功時はロックを獲得し、失敗時はエラーをスローする）。

前提条件と不変条件

  • lockstmt->relations に含まれる各リレーションに対して、適切な権限チェック（RangeVarCallbackForLockTable 経由）が実行される。
  • ビューの場合は LockViewRecurse、継承テーブルの場合は LockTableRecurse が呼び出され、再帰的なロック適用が行われる。

アルゴリズムまたは振る舞い

1. lockstmt->relations リストを反復処理する。 2. RangeVarGetRelidExtended を呼び出し、リレーションの OID を取得する。この際、lockstmt->nowait に応じて RVR_NOWAIT フラグを制御する。 3. 取得したリレーションの型（relkind）を確認する。

  • **ビューの場合**: LockViewRecurse を呼び出し、ビュー定義が参照するリレーションを再帰的にロックする。
  • **継承テーブルの場合**: recurse フラグ（rv->inh）が真であれば、LockTableRecurse を呼び出し、子テーブルを探索してロックする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c:937
  • **呼び出し先**:
  • RangeVarGetRelidExtended
  • get_rel_relkind
  • LockViewRecurse
  • LockTableRecurse

関連データ構造

  • LockStmt: LOCK TABLE 文の解析結果を保持するノード。
  • RangeVar: リレーションの識別情報（名前、スキーマ、継承フラグなど）を保持。

失敗モードと境界条件

  • 指定されたリレーションが存在しない場合、エラーとなる。
  • NOWAIT 指定時にロックが即座に取得できない場合、エラーとなる。
  • 権限チェック（RangeVarCallbackForLockTable）で拒否された場合、エラーとなる。

保守担当者向けメモ

  • 本関数は LOCK TABLE のメインエントリーポイントであり、実際のロック獲得ロジックは storage/lmgr.h に依存している。
  • ビューの再帰ロックは循環参照を避ける仕組みが LockViewRecurse 内で管理されているため、変更時は注意が必要。
