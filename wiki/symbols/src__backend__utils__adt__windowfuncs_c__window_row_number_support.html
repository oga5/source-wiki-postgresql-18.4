window_row_number_support

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/windowfuncs.c:97

目的

row_number() ウィンドウ関数の最適化ヒントをオプティマイザに提供します。

シグネチャまたは形状

Datum window_row_number_support(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (単一の Node * 引数 rawreq)
  • **出力**: Datum (要求された最適化情報を含む Node *、または該当しない場合は NULL)

前提条件と不変条件

  • rawreq が SupportRequestWFuncMonotonic または SupportRequestOptimizeWindowClause のいずれかである場合にのみ、特定の最適化情報を返します。

アルゴリズムまたは振る舞い

1. **単調性の通知**: SupportRequestWFuncMonotonic が要求された場合、row_number() が単調増加関数であることを示し、req->monotonic に MONOTONICFUNC_INCREASING を設定します。 2. **フレーム最適化**: SupportRequestOptimizeWindowClause が要求された場合、フレーム指定を ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW に強制します。これにより、実行時にピア（同順位）行をチェックするコストを回避します。 3. **デフォルト**: 上記以外の要求に対しては NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: クエリ計画作成時にオプティマイザから呼び出されます（具体的な呼び出し元関数名はソースコードから特定不可）。
  • **呼び出し先**: IsA マクロを使用してノード型を判定します。

関連データ構造

  • SupportRequestWFuncMonotonic: 単調性チェック用ノード。
  • SupportRequestOptimizeWindowClause: ウィンドウ句の最適化用ノード。

失敗モードと境界条件

  • 未知のノード型が渡された場合は NULL を返し、最適化を行いません。

保守担当者向けメモ

  • 本関数は row_number() の実行効率を向上させるためのヒントを提供します。フレーム指定の変更（RANGE から ROWS への変換）は、実行エンジンにおけるピアチェックの省略を意図しています。変更を加える際は、実行エンジン側のロジックと整合性が取れているか確認してください。
