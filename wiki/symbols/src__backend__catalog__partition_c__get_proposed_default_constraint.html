get_proposed_default_constraint

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/partition.c:369

目的

パーティションテーブルのデフォルトパーティションに適用すべき制約式を生成します。既存のパーティション制約を否定（NOT）することで、デフォルトパーティションが受け入れるべきデータの範囲を導出します。

シグネチャまたは形状

List *get_proposed_default_constraint(List *new_part_constraints)

入力と出力

  • **入力**: new_part_constraints (List*) - 現在のパーティション制約のリスト。
  • **出力**: List* - デフォルトパーティション用の制約式（暗黙的なAND形式）。

前提条件と不変条件

  • 入力されるパーティション制約は、評価結果が NULL にならないことが前提です。
  • 否定操作は安全に行われます。

アルゴリズムまたは振る舞い

1. make_ands_explicit を使用して、入力された制約リストを明示的なAND式に変換します。 2. makeBoolExpr を使用して、生成された式全体を NOT で囲みます。 3. eval_const_expressions を呼び出し、定数式の評価と簡略化を行います。 4. canonicalize_qual を使用して、式を正規化します。 5. make_ands_implicit を使用して、最終的な制約式をリスト形式に戻して返します。

失敗モードと境界条件

  • 入力リストが空の場合の挙動については、ソースコード上での明示的なガード条件は記載されていません。

保守担当者向けメモ

  • 本関数は src/include/catalog/partition.h で宣言されており、外部モジュールからも利用可能です。
  • 式の正規化を行っているため、複雑な制約式に対しても一貫した形式でデフォルト制約を生成します。
