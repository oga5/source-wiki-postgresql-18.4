index_truncate_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/indextuple.c:575

目的

既存のインデックス・タプルから指定された属性数分だけを抽出し、新しいインデックス・タプルを作成する。

シグネチャまたは形状

IndexTuple index_truncate_tuple(TupleDesc sourceDescriptor, IndexTuple source, int leavenatts)

入力と出力

  • **入力**:
  • sourceDescriptor: 元のタプルの記述子。
  • source: 切り詰め対象の IndexTuple。
  • leavenatts: 残すべき属性の数。
  • **出力**:
  • 切り詰められた新しい IndexTuple を返す。

前提条件と不変条件

  • leavenatts <= sourceDescriptor->natts であること。
  • 生成されたタプルのサイズは、元のタプルのサイズ以下であること（Assert による検証）。

アルゴリズムまたは振る舞い

1. leavenatts が元の属性数と等しい場合、CopyIndexTuple を呼び出してコピーを返す。 2. CreateTupleDescTruncatedCopy を使用して、指定された属性数を持つ一時的なタプル記述子を作成する。 3. index_deform_tuple を使用して、元のタプルから属性値を展開する。 4. index_form_tuple を使用して、展開した値から新しいタプルを構築する。 5. 元のタプルの t_tid を新しいタプルにコピーする。 6. 一時的なタプル記述子を pfree で解放し、新しいタプルを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtutils.c 内の関数（例: pivot の生成時）。
  • **呼び出し先**: CopyIndexTuple, CreateTupleDescTruncatedCopy, index_deform_tuple, index_form_tuple, IndexTupleSize, pfree。

関連データ構造

  • IndexTuple: インデックス行のメモリ表現。
  • TupleDesc: タプルの構造記述子。

失敗モードと境界条件

  • leavenatts が元の属性数より大きい場合、アサーション違反が発生する。
  • メモリ不足時には index_form_tuple 等の内部関数がエラーを投げる可能性がある。

保守担当者向けメモ

  • 本関数は一時的な TupleDesc を作成するため、メモリリークを防ぐために pfree が適切に実行される必要がある。
  • t_tid は切り詰め後も維持されるため、インデックスの物理的な位置情報は保持される。
