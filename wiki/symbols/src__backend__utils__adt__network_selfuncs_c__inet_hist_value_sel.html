inet_hist_value_sel

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_selfuncs.c:618

目的

ヒストグラム統計情報を用いて、inet または cidr 型の定数に対する演算子（包含・オーバーラップ等）の選択率を推定します。

シグネチャまたは形状

static Selectivity inet_hist_value_sel(Datum *values, int nvalues, Datum constvalue, int opr_codenum)

入力と出力

  • **入力**:
  • values: ヒストグラムの境界値配列。
  • nvalues: ヒストグラムの要素数。
  • constvalue: 比較対象の定数（inet 型）。
  • opr_codenum: 演算子の種類を示す数値コード。
  • **出力**:
  • Selectivity (double): 推定された選択率（0.0〜1.0）。

前提条件と不変条件

  • nvalues が 1 以下の場合は 0.0 を返します。
  • MAX_CONSIDERED_ELEMS を超えるヒストグラム要素がある場合、計算コストを抑えるために間引き（デシメーション）が行われます。

アルゴリズムまたは振る舞い

1. ヒストグラムの各バケットに対し、inet_inclusion_cmp を用いて定数との包含関係を判定します。 2. バケットの両端が定数と完全に一致（包含関係が 0）する場合、そのバケットの選択率を 1.0 加算します。 3. バケットが部分的に一致する場合、inet_hist_match_divider を用いてビット単位の「一致度」を計算し、その結果に基づき確率を補正（$1/2^n$）して加算します。 4. 全バケットの累積値を考慮したバケット数で除算し、最終的な選択率を算出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network_selfuncs.c 内の networksel（制限句推定）および結合選択率推定ロジックから複数回呼び出されます。
  • **呼び出し先**:
  • inet_inclusion_cmp: 包含関係の比較。
  • inet_hist_match_divider: 部分一致時の確率補正値の算出。
  • DatumGetInetPP: Datum から inet 構造体への変換。
  • pow: 確率計算用。

関連データ構造

  • inet: ネットワークアドレスを保持する内部構造体。
  • Datum: PostgreSQL の汎用データ型表現。
  • MAX_CONSIDERED_ELEMS: 統計計算時に考慮する要素数の上限（1024）。

失敗モードと境界条件

  • nvalues <= 1: 統計情報が不十分な場合、0.0 を返して除算エラーを回避します。
  • ヒストグラムが空または単一値の場合、選択率は 0 とみなされます。

保守担当者向けメモ

  • 本関数はヒューリスティックな手法を用いています。推定精度が低い場合は、inet_hist_match_divider によるビット単位の補正ロジックが、対象のネットワークアドレスの分布と適合しているか確認してください。
  • 結合推定時に $O(N^2)$ の計算量にならないよう、MAX_CONSIDERED_ELEMS による間引きが適用されています。この定数の変更はプランナの計算コストに直結します。
