pgstat_beinit

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/backend_status.c:244

目的

現在のバックエンドプロセスを統計システムに登録し、共有メモリ上のステータス領域を初期化する。

シグネチャまたは形状

void pgstat_beinit(void)

入力と出力

  • 入力: なし（グローバル変数 MyProcNumber および BackendStatusArray を参照）
  • 出力: なし（副作用として MyBEEntry を設定し、プロセス終了フックを登録）

前提条件と不変条件

  • MyProcNumber が INVALID_PROC_NUMBER ではないこと。
  • MyProcNumber が 0 以上 NumBackendStatSlots 未満であること。

アルゴリズムまたは振る舞い

1. MyProcNumber を使用して、共有メモリ上の BackendStatusArray から当該プロセスのステータスエントリを特定し、グローバル変数 MyBEEntry に割り当てる。 2. プロセス終了時にステータスをクリーンアップするため、on_shmem_exit を用いて pgstat_beshutdown_hook を登録する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/postmaster/auxprocess.c
  • src/backend/utils/init/postinit.c
  • 呼び出し先:
  • on_shmem_exit (PostgreSQL内部の終了フック登録関数)
  • pgstat_beshutdown_hook (終了時に実行されるコールバック)

関連データ構造

  • BackendStatusArray: 全バックエンドのステータスを保持する共有メモリ上の配列。
  • MyBEEntry: 現在のプロセスが使用する PgBackendStatus 構造体へのポインタ。
  • MyProcNumber: 現在のプロセスの識別番号。

失敗モードと境界条件

  • MyProcNumber が範囲外の場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数はバックエンドの初期化プロセスにおいて必須であり、InitPostgres 等の初期化シーケンス内で呼び出される必要がある。
  • pgstat_beshutdown_hook はプロセス終了時のクリーンアップを担当するため、本関数で正しく登録されていることが重要である。
