GetSQLCurrentTimestamp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/timestamp.c:1662

目的

SQLの CURRENT_TIMESTAMP および CURRENT_TIMESTAMP(n) 構文を実装し、現在のトランザクション開始時刻を返します。

シグネチャまたは形状

TimestampTz GetSQLCurrentTimestamp(int32 typmod)

入力と出力

  • **入力**:
  • typmod (int32): タイムスタンプの精度（小数点以下の桁数）。負の値は精度制限なしを意味します。
  • **出力**:
  • TimestampTz: トランザクション開始時刻を表すタイムスタンプ値。

前提条件と不変条件

  • GetCurrentTransactionStartTimestamp() が有効なトランザクション開始時刻を返す環境で呼び出される必要があります。
  • typmod が 0 以上の場合、返されるタイムスタンプは指定された精度に丸められます。

アルゴリズムまたは振る舞い

1. GetCurrentTransactionStartTimestamp() を呼び出し、現在のトランザクション開始時刻を取得します。 2. typmod が 0 以上である場合、AdjustTimestampForTypmod を呼び出し、取得したタイムスタンプを指定された精度に丸めます。 3. 処理されたタイムスタンプを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c: 式評価時に CURRENT_TIMESTAMP の値を解決するために呼び出されます。
  • **呼び出し先**:
  • GetCurrentTransactionStartTimestamp()
  • AdjustTimestampForTypmod()

関連データ構造

  • TimestampTz: マイクロ秒単位の int64 で保持されるタイムスタンプ型。

失敗モードと境界条件

  • typmod が負の値の場合、精度調整はスキップされます。
  • トランザクション外で呼び出された場合の挙動は、GetCurrentTransactionStartTimestamp() の仕様に依存します。

保守担当者向けメモ

  • 本関数は CURRENT_TIMESTAMP 構文の直接的なバックエンド実装です。精度変更が必要な場合は AdjustTimestampForTypmod の仕様を確認してください。
