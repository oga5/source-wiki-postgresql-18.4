ReplicationStateCtl

定義

  • **種別**: struct
  • **定義箇所**: src/backend/replication/logical/origin.c:154

目的

論理レプリケーションにおける各起点の進捗状態（ReplicationState）を共有メモリ上で一元管理するための制御構造体です。

シグネチャまたは形状

typedef struct ReplicationStateCtl
{
    int tranche_id;
    ReplicationState states[FLEXIBLE_ARRAY_MEMBER];
} ReplicationStateCtl;

入力と出力

  • **入力**: 共有メモリ確保時に max_active_replication_origins に基づくサイズ計算が行われます。
  • **出力**: 共有メモリ上の ReplicationState 配列へのアクセスを提供します。

前提条件と不変条件

  • states 配列の長さは max_active_replication_origins に依存します。
  • tranche_id は、各起点に関連付けられた LWLock を管理するために使用されます。

アルゴリズムまたは振る舞い

  • 共有メモリの初期化時（ReplicationOriginShmemInit）に、この構造体全体が確保されます。
  • ReplicationOriginShmemSize 関数内で、offsetof(ReplicationStateCtl, states) を用いて必要な共有メモリサイズが計算されます。
  • 確保されたメモリ領域は、replication_states_ctl ポインタを通じてグローバルに参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/origin.c: ReplicationOriginShmemSize（サイズ計算）
  • src/backend/replication/logical/origin.c: ReplicationOriginShmemInit（初期化時のキャスト）
  • **呼び出し先**:
  • ReplicationState 構造体（配列要素として保持）

関連データ構造

  • ReplicationState: 各起点の進捗を保持する個別の構造体。
  • max_active_replication_origins: 本構造体のサイズを決定する GUC パラメータ。

失敗モードと境界条件

  • **共有メモリ不足**: max_active_replication_origins の設定値が大きすぎる場合、共有メモリの確保に失敗する可能性があります。
  • **インデックス範囲**: states 配列へのアクセスは、max_active_replication_origins の範囲内で行う必要があります。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を使用しているため、メモリ確保時には offsetof を使用した動的なサイズ計算が必須です。
  • 構造体内の tranche_id は、個々の ReplicationState が持つロックの管理に不可欠です。変更を加える際は、ロックの粒度と競合に注意してください。
