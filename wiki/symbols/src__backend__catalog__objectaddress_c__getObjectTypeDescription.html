getObjectTypeDescription

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaddress.c:4496

目的

ObjectAddress 構造体で指定されたデータベースオブジェクトの種類を特定し、人間が判読可能な文字列（例: "table", "role", "index" など）を返します。

シグネチャまたは形状

char *getObjectTypeDescription(const ObjectAddress *object, bool missing_ok)

入力と出力

  • **入力**:
  • object: 情報を取得したい対象オブジェクトを指す ObjectAddress へのポインタ。
  • missing_ok: 処理の挙動に関するフラグ（詳細は後述）。
  • **出力**:
  • オブジェクトの種類を表す文字列（char *）。

前提条件と不変条件

  • object->classId がサポートされているカタログOIDである必要があります。
  • 内部で StringInfoData を使用して文字列を構築するため、メモリ管理の制約を受けます。

アルゴリズムまたは振る舞い

1. initStringInfo で文字列バッファを初期化します。 2. object->classId に基づいて switch 文で分岐します。 3. リレーション、プロシージャ、制約など、詳細な分類が必要なオブジェクトについては、専用のサブ関数（getRelationTypeDescription 等）を呼び出してバッファに追記します。 4. それ以外の単純なオブジェクトについては、appendStringInfoString を用いて固定の文字列をバッファに書き込みます。 5. サポートされていない classId が渡された場合、elog(ERROR, ...) を発行して処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/label.c
  • src/backend/catalog/objectaddress.c (複数箇所)
  • src/backend/commands/event_trigger.c
  • **呼び出し先**:
  • getRelationTypeDescription
  • getProcedureTypeDescription
  • getConstraintTypeDescription
  • initStringInfo, appendStringInfoString, elog (PostgreSQL内部ユーティリティ)

関連データ構造

  • ObjectAddress: オブジェクトを特定するための基本構造体。
  • StringInfoData: 文字列構築用バッファ。

失敗モードと境界条件

  • classId が既知のカタログテーブルと一致しない場合、elog(ERROR) によりトランザクションがアボートされます。
  • missing_ok フラグは、サブ関数（getRelationTypeDescription 等）へ渡されますが、本関数自体は missing_ok の値に関わらず、未知の classId に対してはエラーを発生させます。

保守担当者向けメモ

  • 新しいオブジェクトタイプをシステムに追加した際は、本関数の switch 文に新しい case を追加する必要があります。
  • 既存の getRelationTypeDescription 等のサブ関数は、本関数の保守性を高めるために分離されています。変更時はこれらのサブ関数との整合性に注意してください。
