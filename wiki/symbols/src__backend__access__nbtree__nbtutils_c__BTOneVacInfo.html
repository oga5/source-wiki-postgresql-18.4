BTOneVacInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/access/nbtree/nbtutils.c:3486

目的

B-tree インデックスの VACUUM 処理において、個別のインデックスの識別情報と、現在実行中の VACUUM サイクルを追跡するために使用されます。

シグネチャまたは形状

typedef struct BTOneVacInfo
{
    LockRelId   relid;      /* インデックスのグローバル識別子 */
    BTCycleId   cycleid;    /* アクティブな VACUUM のサイクル ID */
} BTOneVacInfo;

入力と出力

本構造体はデータ保持を目的としており、関数としての入出力は持ちません。

前提条件と不変条件

  • relid はインデックスを一意に識別する LockRelId 型である必要があります。
  • cycleid は VACUUM の進行状況を管理するための BTCycleId 型の値です。

アルゴリズムまたは振る舞い

本構造体自体はデータコンテナであり、特定のアルゴリズムは実装していません。主に BTVacInfo 構造体内の配列要素として保持され、VACUUM 処理中のインデックス状態を管理するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtutils.c 内の VACUUM 関連処理において、BTVacInfo 構造体の初期化や参照時に使用されます。
  • **呼び出し先**:
  • 本構造体はデータ構造であるため、直接的な呼び出し先はありません。

関連データ構造

  • BTVacInfo: BTOneVacInfo を要素として含む、VACUUM 情報を管理する親構造体。
  • LockRelId: インデックスを識別するためのリレーション識別子。
  • BTCycleId: VACUUM サイクルを識別するための型。

失敗モードと境界条件

  • BTVacInfo 内で FLEXIBLE_ARRAY_MEMBER を用いて配列として管理されるため、メモリ割り当てサイズ（MaxBackends に基づく計算など）が不足すると境界外アクセスのリスクがあります。

保守担当者向けメモ

  • src/backend/access/nbtree/nbtutils.c:3646 にある通り、MaxBackends に基づいてメモリサイズが計算されます。この構造体のサイズ変更を行う際は、関連するメモリ割り当てロジックの整合性を確認してください。
