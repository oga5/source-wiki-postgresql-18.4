ReplicationSlotOnDisk

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/slot.c:65

目的

レプリケーションスロットの状態をディスク（$PGDATA/pg_replslot）へ永続化するためのデータ構造です。クラッシュリカバリ時にスロットの状態を復元するために使用されます。

シグネチャまたは形状

typedef struct ReplicationSlotOnDisk
{
    uint32      magic;
    pg_crc32c   checksum;
    uint32      version;
    uint32      length;
    ReplicationSlotPersistentData slotdata;
} ReplicationSlotOnDisk;

入力と出力

  • **入力**: スロットの永続的な状態（slotdata）およびメタデータ（magic, version 等）。
  • **出力**: ディスク書き込み用にシリアライズされた構造体データ。

前提条件と不変条件

  • 構造体の先頭部分はバージョン非依存である必要があります。
  • magic および checksum はチェックサム計算の対象外です。
  • version および length 以降のデータはチェックサムの対象となります。
  • slotdata の内容は version に応じて変化する可能性があります。

アルゴリズムまたは振る舞い

  • スロットの状態が更新されると、この構造体に値を詰め、チェックサムを計算した上でディスクへ書き出されます。
  • 構造体のサイズや各フィールドのオフセットは、offsetof や sizeof を用いて計算され、ディスクI/O時のデータ配置の整合性が保たれます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/slot.c 内の永続化処理（ReplicationSlotSave 等）から参照されます。
  • **呼び出し先**: 直接的な関数呼び出し関係はソースから特定できませんが、ReplicationSlotPersistentData を介してスロットの永続的属性を保持します。

関連データ構造

  • ReplicationSlotPersistentData: スロットの永続的な属性（LSNやxmin等）を保持する構造体。

失敗モードと境界条件

  • **チェックサム不一致**: ディスクから読み込んだ際に checksum が一致しない場合、スロットデータは破損しているとみなされます。
  • **バージョン不整合**: version フィールドが現在のPostgreSQLの期待値と異なる場合、読み込み処理が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体のレイアウトを変更する場合、既存のディスク上のデータとの互換性に注意してください。
  • slotdata のフィールド追加時は、version の更新とチェックサム計算ロジックへの影響を考慮する必要があります。
