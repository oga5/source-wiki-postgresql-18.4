line_starts_with

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/backup_label.c:223

目的

文字列バッファ内の特定の行が指定された文字列で始まるかを判定し、一致した場合はバッファポインタを一致箇所の直後に進めるための補助関数です。

シグネチャまたは形状

static bool line_starts_with(char *s, char *e, char *match, char **sout)

入力と出力

  • s (char *): 走査対象の開始ポインタ。
  • e (char *): 走査対象の終了ポインタ（境界）。
  • match (char *): 比較対象の文字列。
  • sout (char **): 一致した場合、一致した文字列の直後の位置を格納するポインタ（NULL指定可）。
  • 戻り値 (bool): match が s から始まる場合に true を返し、それ以外は false を返します。

前提条件と不変条件

  • s は e 以下の有効なメモリ領域を指している必要があります。
  • match はヌル終端文字列である必要があります。
  • sout が NULL でない場合、一致が確認された後にのみ更新されます。

アルゴリズムまたは振る舞い

1. s が e に達するか、match が終端に達するか、あるいは *s と *match が不一致になるまで、両方のポインタをインクリメントしながら比較します。 2. match が終端に達した場合（＝完全一致）、sout が NULL でなければ、現在の s の位置を sout に代入します。 3. match が終端に達したかどうかを判定し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_combinebackup/backup_label.c 内の parse_backup_label およびラベルファイル再構築ロジックから、START WAL LOCATION、START TIMELINE、INCREMENTAL 関連行の識別に使用されています。
  • **呼び出し先**:
  • 本関数は標準ライブラリや他の関数を呼び出しません。

関連データ構造

  • StringInfo (間接的): 本関数が操作するポインタは、StringInfo バッファ内のデータ領域を指すことを想定しています。

失敗モードと境界条件

  • s >= e: 比較を行わず false を返します。
  • match が空文字列の場合: 常に true を返し、sout には現在の s が代入されます。
  • match が s から e の範囲内に収まらない場合: false を返します。

保守担当者向けメモ

  • 本関数はバッファの破壊的変更を行いません。
  • sout に &s を渡すことで、呼び出し側で解析位置を効率的に進める設計になっています。
