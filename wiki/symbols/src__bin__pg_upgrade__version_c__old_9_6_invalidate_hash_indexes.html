old_9_6_invalidate_hash_indexes

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/version.c:54

目的

PostgreSQL 9.6 から 10 へのアップグレード時、内部形式が変更されたハッシュインデックスを検出し、無効化および再構築用スクリプトの生成を行う。

シグネチャまたは形状

void old_9_6_invalidate_hash_indexes(ClusterInfo *cluster, bool check_mode)

入力と出力

  • **入力**:
  • cluster: 対象クラスター情報 (ClusterInfo*)。
  • check_mode: 実行モード (bool)。true の場合は警告表示のみ行い、ファイル生成やカタログ更新は行わない。
  • **出力**:
  • なし（副作用として reindex_hash.sql の生成、およびカタログの更新を行う）。

前提条件と不変条件

  • cluster 引数は有効な ClusterInfo 構造体を指している必要がある。
  • check_mode が false の場合、ファイルシステムへの書き込み権限が必要。

アルゴリズムまたは振る舞い

1. 指定されたクラスター内の全データベースを走査する。 2. 各データベースに対し、pg_catalog をクエリして amname = 'hash' であるインデックスを特定する。 3. ハッシュインデックスが存在する場合:

  • check_mode が false の場合: reindex_hash.sql を作成し、各インデックスに対する REINDEX INDEX 文を書き込む。
  • check_mode が false の場合: カタログの pg_index テーブルを更新し、該当インデックスの indisvalid を false に設定する。

    • 4. 最後に、ハッシュインデックスが見つかった場合はユーザーに対して警告メッセージを表示する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/check.c:690 (チェックフェーズ)
  • src/bin/pg_upgrade/check.c:804 (アップグレードフェーズ)
  • **呼び出し先**:
  • connectToServer, executeQueryOrDie, fopen_priv, pg_fatal, initPQExpBuffer, appendPsqlMetaConnect, termPQExpBuffer, quote_identifier, PQntuples, PQfnumber, PQgetvalue, PQclear, PQfinish, fclose, report_status, pg_log, check_ok

関連データ構造

  • ClusterInfo: クラスターのメタデータおよびデータベースリストを保持。
  • DbInfo: 個別のデータベース情報を保持。

失敗モードと境界条件

  • reindex_hash.sql の作成に失敗した場合、pg_fatal によりプロセスが終了する。
  • データベース接続に失敗した場合、executeQueryOrDie によりプロセスが終了する。
  • ハッシュインデックスが存在しない場合、check_ok() が呼び出され、正常終了する。

保守担当者向けメモ

  • 本関数は 9.6 から 10 への移行に特化したロジックである。将来的にこのバージョン範囲のサポートが不要となった場合、削除を検討すること。
  • reindex_hash.sql の出力パスはハードコードされているため、変更が必要な場合は注意が必要。
