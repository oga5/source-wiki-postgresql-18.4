DumpId

定義

  • 種別: typedef
  • 定義箇所: src/bin/pg_dump/pg_backup.h:284

目的

pg_dump が実行時にダンプ対象の各オブジェクトを識別するために割り当てる論理的な連番 ID です。システムカタログ上の ID（CatalogId）とは異なり、依存関係の解決やダンプ順序の制御を行うための内部的な識別子として使用されます。

シグネチャまたは形状

typedef int DumpId;

入力と出力

  • 入力: 該当なし（プリミティブ型）
  • 出力: 整数値（オブジェクトの識別子）

前提条件と不変条件

  • 0 は InvalidDumpId として予約されており、有効なオブジェクト識別子としては使用されません。
  • pg_dump の実行セッション内でのみ有効な識別子です。

アルゴリズムまたは振る舞い

  • pg_dump は、ダンプ対象オブジェクトを管理する際にこの ID を付与します。
  • オブジェクト間の依存関係（addObjectDependency 等）を記録する際、参照先オブジェクトを特定するために使用されます。
  • findObjectByDumpId を通じて、ID から対応するオブジェクトを検索することが可能です。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/bin/pg_dump/common.c: オブジェクトの依存関係管理（addObjectDependency, removeObjectDependency）や検索（findObjectByDumpId）で使用。
  • src/bin/pg_dump/parallel.c: 並列処理におけるダンプ対象の識別に使用。
  • 呼び出し先:
  • 該当なし（プリミティブ型のため）。

関連データ構造

  • DumpableObject: DumpId を保持し、ダンプ対象のメタデータや依存関係を管理する構造体。

失敗モードと境界条件

  • 0 を指定した場合、無効な ID として扱われます。
  • 割り当て可能な最大値を超えた場合の挙動については、ソースコードの断片からは不明です。

保守担当者向けメモ

  • CatalogId と混同しないよう注意してください。DumpId は実行時の論理的な順序付けに特化した ID です。
  • common.c 内の lastDumpId 変数が、次に割り当てる ID の追跡に使用されています。
