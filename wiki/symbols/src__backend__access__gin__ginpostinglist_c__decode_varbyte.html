decode_varbyte

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginpostinglist.c:132

目的

可変長バイト（Varbyte）エンコーディングで圧縮された64ビット整数を復元します。

シグネチャまたは形状

static uint64 decode_varbyte(unsigned char **ptr)

入力と出力

  • **入力**: unsigned char **ptr - 圧縮されたバイト列を指すポインタのポインタ。
  • **出力**: uint64 - 復元された整数値。
  • **副作用**: *ptr は、読み取ったバイト数分だけ前方へ進められます。

前提条件と不変条件

  • 入力されるバイト列は、encode_varbyte によってエンコードされた形式である必要があります。
  • 7バイト目（最大値）には継続ビット（0x80）が含まれていないことが期待されます。

アルゴリズムまたは振る舞い

1. 各バイトの最下位7ビットをデータとして抽出し、継続ビット（最上位ビット: 0x80）を確認します。 2. 継続ビットが立っている場合、次のバイトを読み込み、シフト演算を用いて値を結合します。 3. 最大7バイトまで順次処理を行い、継続ビットが立っていないバイトに到達した時点で終了します。 4. 最終的に更新されたポインタ位置を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginpostinglist.c 内のTID展開処理（例: ginPostingListDecodeAllSegments 等の内部）。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません（標準的なビット演算のみを使用）。

関連データ構造

  • GinPostingList: GINインデックスでTIDリストを格納するために使用される構造体。

失敗モードと境界条件

  • **境界条件**: 7バイト目まで継続ビットが立っている場合、それ以降のバイトは読み込まれません。
  • **アサーション**: 7バイト目の読み込み時に継続ビットが立っている場合は Assert により検証されます。

保守担当者向けメモ

  • 本関数は encode_varbyte と対になる低レベル関数です。エンコーディング形式を変更する場合は、両方の関数を同期させる必要があります。
  • 7バイトを超えるデータは想定されていません。
