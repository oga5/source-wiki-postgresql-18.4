gistindex_keytest

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistget.c:124

目的

GiSTインデックスの個々のタプルが検索条件（ScanKey）を満たすか判定し、ORDER BY句がある場合には距離を計算する。

シグネチャまたは形状

bool gistindex_keytest(IndexScanDesc scan, IndexTuple tuple, Page page, OffsetNumber offset, bool *recheck_p, bool *recheck_distances_p)

入力と出力

  • **入力**:
  • scan: スキャン状態（IndexScanDesc）。
  • tuple: 評価対象のインデックスタプル。
  • page: タプルが格納されているページ。
  • offset: ページ内のタプルオフセット。
  • **出力**:
  • 戻り値: 条件に合致すれば true、そうでなければ false。
  • recheck_p: 検索条件の再評価が必要な場合に true が設定される。
  • recheck_distances_p: 距離の再計算が必要な場合に true が設定される。

前提条件と不変条件

  • 呼び出し元は、短命なメモリコンテキスト（tempCxt）で本関数を実行する必要がある。
  • GistTupleIsInvalid なタプルは、常に一致とみなされ、距離は負の無限大として扱われる。

アルゴリズムまたは振る舞い

1. **無効タプル判定**: 無効なタプルであれば、即座に true を返し、距離を最小値（負の無限大）に設定する。 2. **検索条件判定**: scan->keyData を順次走査し、Consistent 関数を呼び出して一致判定を行う。いずれかの条件で不一致となれば false を返す。 3. **距離計算**: scan->orderByData が存在する場合、Distance 関数を呼び出して各距離を計算し、so->distances に格納する。 4. **再評価フラグ**: 各関数呼び出しの結果に基づき、recheck フラグを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistget.c 内の gistScanPage（直接の呼び出しを確認）。
  • **呼び出し先**: 演算子クラスの Consistent および Distance 関数（FunctionCall5Coll 経由）、index_getattr、gistdentryinit。

関連データ構造

  • GISTScanOpaque: スキャン状態を保持。distances フィールドは本関数の出力先として使用される。
  • IndexTuple: 評価対象のデータ。
  • GISTENTRY: gistdentryinit で初期化される GiST エントリ構造。

失敗モードと境界条件

  • リーフページ上で無効なタプルが見つかった場合、elog(ERROR) を発生させる。
  • SK_ISNULL や SK_SEARCHNULL フラグの組み合わせにより、NULL 値の扱いが分岐する。
  • 距離計算において、NULL 値が含まれる場合は距離を 0.0、isnull を true として扱う。

保守担当者向けメモ

  • 本関数は giststate->tempCxt 等の適切なメモリコンテキストで実行されることが前提である。
  • recheck フラグは、インデックスの損失性（lossy）を上位層に伝えるために重要である。
  • 距離計算の recheck フラグはバージョン 9.5 で追加されたため、古い距離関数との互換性に注意すること。
