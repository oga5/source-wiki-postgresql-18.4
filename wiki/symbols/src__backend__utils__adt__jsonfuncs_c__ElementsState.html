ElementsState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonfuncs.c:122

目的

JSON配列の要素を抽出する関数（json_array_elements 等）の実行中における状態を保持するために使用されます。

シグネチャまたは形状

typedef struct ElementsState
{
	JsonLexContext *lex;
	const char *function_name;
	Tuplestorestate *tuple_store;
	TupleDesc	ret_tdesc;
	MemoryContext tmp_cxt;
	const char *result_start;
	bool		normalize_results;
	bool		next_scalar;
	char	   *normalized_scalar;
} ElementsState;

入力と出力

本構造体は関数内部で palloc0 を用いて初期化され、抽出処理の各ステップで状態の更新や参照が行われます。具体的な入力値や戻り値の詳細は、この構造体を利用する各SQL関数の実装に依存します。

前提条件と不変条件

  • lex は有効な JsonLexContext を指している必要があります。
  • tuple_store は結果セットを構築するための Tuplestorestate として初期化されている必要があります。

アルゴリズムまたは振る舞い

JSONの解析プロセスにおいて、解析器のコールバック関数が本構造体を参照・更新することで、現在の解析位置や抽出結果を保持します。normalize_results や next_scalar フラグにより、スカラー値の正規化や抽出の制御を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonfuncs.c 内の json_array_elements 関連関数（2314行目、2352行目、2374行目、2420行目、2435行目等）から参照・キャストされています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • JsonLexContext: JSONのトークン解析コンテキスト。
  • Tuplestorestate: 抽出された要素を格納する一時的なタプルストア。
  • TupleDesc: 返却されるタプルの記述子。

失敗モードと境界条件

  • メモリ不足時: palloc0 による割り当て失敗が発生する可能性があります。
  • 解析エラー時: JsonLexContext を介した解析処理中にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は json_array_elements 等のSRF（集合を返す関数）の実行状態を管理します。
  • 構造体の各フィールドは、解析器のコールバック関数とメインのSQL関数間で共有されるため、変更時は両方の整合性に注意してください。
