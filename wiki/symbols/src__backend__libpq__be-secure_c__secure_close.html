secure_close

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure.c:166

目的

クライアントとのセキュアな通信セッションを終了し、関連する暗号化リソースを解放する。

シグネチャまたは形状

void secure_close(Port *port)

入力と出力

  • **入力**: Port *port - 通信状態および暗号化フラグを保持する構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • port ポインタが有効であること。
  • USE_SSL が定義されている環境では、port->ssl_in_use フラグが現在の暗号化状態を正確に反映していること。

アルゴリズムまたは振る舞い

1. USE_SSL が定義されている場合、port->ssl_in_use を確認する。 2. 暗号化が有効であれば、be_tls_close(port) を呼び出してSSL/TLS接続を終了する。 3. 暗号化が無効な場合、または USE_SSL が定義されていない場合は何もしない。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/pqcomm.c:379 (secure_close(MyProcPort);)
  • **呼び出し先**:
  • be_tls_close (条件付き: USE_SSL 定義時)

関連データ構造

  • Port: src/include/libpq/libpq.h で定義される、接続状態を管理する構造体。

失敗モードと境界条件

  • port が NULL の場合の挙動は定義されていない。
  • be_tls_close 内部でのエラー発生時の振る舞いは、当該関数の実装に依存する。

保守担当者向けメモ

  • 本関数はラッパーであり、具体的な暗号化終了処理は be-secure-openssl.c 等の各実装ファイルに委譲される。
  • GSSAPI等の他の暗号化方式が追加された場合、本関数内で適切に分岐処理を行う必要がある可能性がある。
