PLy_trigger_build_args

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_exec.c:702

目的

PostgreSQL のトリガー実行コンテキスト（TriggerData）から、Python 側でトリガー関数に渡すための辞書オブジェクト（TD）を構築する。

シグネチャまたは形状

static PyObject *PLy_trigger_build_args(FunctionCallInfo fcinfo, PLyProcedure *proc, HeapTuple *rv);

入力と出力

  • **入力**:
  • fcinfo: PostgreSQL の関数呼び出し情報。context フィールドに TriggerData を保持。
  • proc: 現在実行中の PL/Python プロシージャ情報。
  • rv: 戻り値となる HeapTuple を格納するためのポインタ。
  • **出力**:
  • 成功時: トリガー情報を含む PyObject（辞書型）。
  • 失敗時: NULL。

前提条件と不変条件

  • PG_TRY ブロック外で PyDict_New 等の Python オブジェクト生成を行う必要がある（エラー時にスタックを巻き戻すため）。
  • tgnargs が存在する場合、pltargs リストが適切に初期化されること。
  • トリガーイベント（INSERT/UPDATE/DELETE/TRUNCATE）およびレベル（ROW/STATEMENT）が認識可能であること。

アルゴリズムまたは振る舞い

1. Python 辞書 pltdata を作成し、トリガー名、リレーション ID、テーブル名、スキーマ名、実行タイミング（BEFORE/AFTER/INSTEAD OF）を格納する。 2. PG_TRY ブロックを開始し、PostgreSQL のエラー発生時に備える。 3. トリガーレベルが ROW の場合:

  • INSERT/DELETE/UPDATE に応じて new および old タプルを PLy_input_from_tuple で Python オブジェクトに変換し、辞書にセットする。
  • 戻り値 rv に適切なタプル（tg_trigtuple または tg_newtuple）を設定する。

    • 4. トリガーレベルが STATEMENT の場合:

  • old および new を Py_None に設定し、rv を NULL に設定する。

    • 5. 各イベントタイプに応じた文字列（"INSERT" 等）を辞書に格納する。 6. 認識できないイベントやレベルの場合は elog(ERROR, ...) を発行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_exec.c 内の PLy_exec_trigger から呼び出される。
  • **呼び出し先**:
  • PLyUnicode_FromString, PyDict_New, PyDict_SetItemString, Py_DECREF 等の Python C API。
  • PLy_input_from_tuple（タプル変換用）。
  • SPI_getrelname, SPI_getnspname（SPI ユーティリティ）。
  • elog（エラー報告）。

関連データ構造

  • TriggerData: PostgreSQL のトリガー実行情報。
  • PLyProcedure: PL/Python プロシージャのメタデータ。
  • HeapTuple: PostgreSQL のタプル構造。

失敗モードと境界条件

  • PyDict_New や PyList_New がメモリ不足等で失敗した場合、NULL を返す。
  • 未知の tg_event が渡された場合、elog(ERROR) によりトランザクションが中断される。
  • PG_TRY ブロック内でのエラー発生時は、PostgreSQL のエラーハンドリング機構により制御が移る。

保守担当者向けメモ

  • PG_TRY ブロック内では return 文を使用できないため、エラー発生時は elog(ERROR) を経由して大域脱出させる必要がある。
  • Py_DECREF による参照カウント管理が重要であり、特に PG_TRY ブロック内でのメモリリークに注意すること。
  • BEFORE トリガーにおける生成列（generated columns）の扱いに留意すること。
