main

定義

  • 種別: function
  • 定義箇所: src/test/regress/pg_regress_main.c:110

目的

PostgreSQL 回帰テスト実行プログラムのエントリポイントとして、テスト実行フレームワークを初期化し、実行を制御します。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • 入力:
  • argc: コマンドライン引数の数。
  • argv: コマンドライン引数の文字列配列。
  • 出力:
  • int: プログラムの終了ステータス（regression_main の戻り値）。

前提条件と不変条件

  • regression_main が提供するテスト実行フレームワークが利用可能であること。
  • psql_init および psql_start_test が、テスト実行に必要な初期化およびプロセス起動のロジックとして正しく定義されていること。

アルゴリズムまたは振る舞い

1. regression_main を呼び出し、テスト実行の制御をフレームワークへ委譲します。 2. 引数として以下の関数ポインタを渡します。

  • psql_init: テスト実行前の環境初期化用。
  • psql_start_test: 個別のテストケース実行用。
  • NULL: 事後処理関数は不要として指定。

    • 3. regression_main から返された終了ステータスをそのまま呼び出し元（OS）へ返します。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）。
  • 呼び出し先: regression_main（src/test/regress/pg_regress.c 内で定義）。
  • 注記: ツールが提示した contrib/amcheck 関連の参照は、本シンボルとは無関係な「main」という単語の誤検知です。

関連データ構造

  • regression_main が内部で使用する各種設定構造体（pg_regress モジュール内の定義に依存）。

失敗モードと境界条件

  • regression_main の実行結果に依存します。引数の不正や環境設定の不備により、regression_main が非ゼロの終了ステータスを返す場合があります。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際のテスト実行ロジックは src/test/regress/pg_regress.c の regression_main に集約されています。
  • 動作変更が必要な場合は、psql_init や psql_start_test の実装を確認してください。
