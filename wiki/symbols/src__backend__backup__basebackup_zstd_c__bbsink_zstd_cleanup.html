bbsink_zstd_cleanup

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_zstd.c:312

目的

Zstandard圧縮シンクに関連付けられた圧縮コンテキスト（ZSTD_CCtx）を解放し、リソースをクリーンアップする。

シグネチャまたは形状

static void bbsink_zstd_cleanup(bbsink *sink)

入力と出力

  • **入力**: sink - bbsink 構造体へのポインタ。実際には bbsink_zstd 型としてキャストして使用される。
  • **出力**: なし。

前提条件と不変条件

  • sink は bbsink_zstd 型の構造体としてキャスト可能であること。
  • mysink->cctx が NULL の場合、本関数は何もしない（安全に呼び出し可能）。

アルゴリズムまたは振る舞い

1. 引数 sink を bbsink_zstd * 型にキャストする。 2. mysink->cctx が存在するか確認する。 3. 存在する場合、ZSTD_freeCCtx() を呼び出してコンテキストを解放し、mysink->cctx を NULL に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_zstd_ops 構造体内の関数ポインタとして登録されており、bbsink インターフェースを通じて呼び出される。
  • **呼び出し先**: ZSTD_freeCCtx (libzstd)。

関連データ構造

  • bbsink_zstd: Zstd圧縮用の状態を保持する構造体。cctx メンバを保持する。
  • ZSTD_CCtx: libzstd の圧縮コンテキスト。

失敗モードと境界条件

  • mysink->cctx が既に解放済み（NULL）である場合、二重解放を避けるためのガード条件により安全に処理される。

保守担当者向けメモ

  • 本関数は bbsink インターフェースのクリーンアップ処理の一部として機能する。
  • USE_ZSTD が定義されている環境でのみコンパイルされる。
