ReconnectToServer

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_db.c:72

目的

既存のデータベース接続を終了し、新しい接続を確立して ArchiveHandle を更新する。

シグネチャまたは形状

void ReconnectToServer(ArchiveHandle *AH, const char *dbname)

入力と出力

  • **入力**:
  • AH: 接続状態を保持する ArchiveHandle 構造体へのポインタ。
  • dbname: 新しい接続先データベース名（指定がある場合）。
  • **出力**: なし（副作用として AH 内の接続状態を更新）。

前提条件と不変条件

  • AH は有効な ArchiveHandle を指している必要がある。
  • dbname が指定された場合、ropt->cparams.override_dbname に格納され、以降の再接続試行にも影響する。

アルゴリズムまたは振る舞い

1. 現在の接続（AH->connection）を oldConn に退避する。 2. dbname が指定されていれば、ropt->cparams.override_dbname を更新する。 3. AH->connection を NULL に設定する（ConnectDatabaseAhx 内でのエラーチェックを回避するため）。 4. ConnectDatabaseAhx を呼び出し、新しい接続を確立する。この際、connCancel 等の内部状態が更新される。 5. PQfinish(oldConn) を呼び出し、古い接続をクローズする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_archiver.c:3493
  • **呼び出し先**:
  • ConnectDatabaseAhx
  • PQfinish (libpq)
  • pg_strdup

関連データ構造

  • ArchiveHandle: データベース接続およびアーカイブ状態を管理する構造体。
  • RestoreOptions: 接続パラメータ（cparams）を保持する構造体。
  • PGconn: libpq の接続ハンドル。

失敗モードと境界条件

  • ConnectDatabaseAhx が失敗した場合、内部で pg_fatal 等が呼び出され、プロセスが終了する可能性がある。
  • 接続切り替え中に SIGINT が発生した場合、AH->connection が一時的に NULL になることで、不正な接続へのアクセスを防止する設計となっている。

保守担当者向けメモ

  • 本関数は、古い接続を閉じる前に新しい接続を確立する順序を守ることで、シグナルハンドラによるデッドコネクションへのアクセスを防いでいる。この順序を変更しないこと。
