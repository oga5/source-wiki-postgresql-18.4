getRTEPermissionInfo

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_relation.c:4008

目的

指定された RangeTblEntry (RTE) に対応する RTEPermissionInfo 構造体を、パーサー状態やプランナーが保持するリストから取得・検証する。

シグネチャまたは形状

RTEPermissionInfo *getRTEPermissionInfo(List *rteperminfos, RangeTblEntry *rte)

入力と出力

  • **入力**:
  • rteperminfos: RTEPermissionInfo ノードを含むリスト。
  • rte: 権限情報を取得対象とする RangeTblEntry。
  • **出力**:
  • 成功時: 対応する RTEPermissionInfo 構造体へのポインタ。
  • 失敗時: 内部エラー（elog(ERROR)）が発生し、処理は中断される。

前提条件と不変条件

  • rte->perminfoindex は 1 から始まるインデックスであること。
  • rteperminfos リスト内の RTEPermissionInfo と、対象の rte が同一の relid を共有していること。

アルゴリズムまたは振る舞い

1. rte->perminfoindex が 0 であるか、リストの長さを超えている場合、エラーをスローする。 2. list_nth_node を使用し、perminfoindex - 1 をインデックスとしてリストから RTEPermissionInfo を取得する。 3. 取得した perminfo->relid と rte->relid を比較し、一致しない場合は整合性エラーとしてスローする。 4. 検証を通過した perminfo を返す。

  • **呼び出し元**:
  • src/backend/executor/execMain.c

関連データ構造

  • RangeTblEntry: クエリ内のリレーションを定義する構造体。perminfoindex を保持する。
  • RTEPermissionInfo: リレーションに対する権限情報を保持する構造体。

失敗モードと境界条件

  • perminfoindex が無効（0 またはリスト長以上）な場合、elog(ERROR) により処理が停止する。
  • relid の不一致（RTEと権限情報の間でリレーションが食い違っている場合）は、致命的な内部エラーとして処理される。

保守担当者向けメモ

  • 本関数は、クエリ解析から実行計画作成に至るまで、権限情報の整合性を保証するための重要なチェックポイントである。
  • relid の不一致は、パーサーまたはプランナーの構築過程における重大なバグを示唆するため、デバッグ時には perminfoindex の割り当てロジックを確認すること。
