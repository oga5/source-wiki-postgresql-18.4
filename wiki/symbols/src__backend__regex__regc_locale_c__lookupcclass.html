lookupcclass

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_locale.c:535

目的

正規表現内の文字クラス名（例: [:alpha:]）を、内部的に定義された列挙型 enum char_classes に変換する。

シグネチャまたは形状

enum char_classes lookupcclass(struct vars *v, const chr *startp, const chr *endp)

入力と出力

  • **入力**:
  • v: コンパイル状態を保持する vars 構造体（エラー報告用）。
  • startp: 文字クラス名の開始位置を指す chr 型ポインタ。
  • endp: 文字クラス名の終了位置（終端の直後）を指す chr 型ポインタ。
  • **出力**:
  • 成功時: 対応する enum char_classes の値。
  • 失敗時: REG_ECTYPE エラーをセットし、0 を返す。

前提条件と不変条件

  • classNames 配列が NULL で終端されていること。
  • startp から endp までの範囲が有効なメモリ領域であること。

アルゴリズムまたは振る舞い

1. startp と endp の差分から名前の長さ len を算出する。 2. classNames 配列を走査し、strlen が len と一致し、かつ pg_char_and_wchar_strncmp が 0 を返す（文字列が一致する）要素を探す。 3. 一致した要素のインデックスを enum char_classes にキャストして返す。 4. 一致する名前が見つからない場合、ERR(REG_ECTYPE) を呼び出してエラーを記録し、0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regcomp.c:1819（正規表現コンパイルプロセス内）。
  • **呼び出し先**: pg_char_and_wchar_strncmp（文字列比較）、ERR マクロ（エラー報告）。

関連データ構造

  • classNames: 文字クラス名のリスト。
  • enum char_classes: 文字クラスを識別する列挙型。
  • vars: コンパイル時のコンテキスト情報を保持する構造体。

失敗モードと境界条件

  • 指定された名前が classNames に存在しない場合、REG_ECTYPE エラーが発生する。
  • startp が endp より大きい場合、len は負の値となり、ループは即座に終了してエラーとなる。

保守担当者向けメモ

  • classNames 配列の内容を変更または拡張する場合は、regguts.h で定義されている enum char_classes との整合性を維持すること。
  • 本関数は regcomp.c から呼び出されるため、エラー発生時の vars 構造体の状態管理に注意が必要である。
