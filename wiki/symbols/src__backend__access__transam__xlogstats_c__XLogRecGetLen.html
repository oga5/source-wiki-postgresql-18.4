XLogRecGetLen

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogstats.c:21

目的

WALレコードの総サイズから、FPI（Full Page Image）データを除いた実データ長（rec_len）と、FPIデータの合計サイズ（fpi_len）を算出します。

シグネチャまたは形状

void XLogRecGetLen(XLogReaderState *record, uint32 *rec_len, uint32 *fpi_len)

入力と出力

  • **入力**:
  • record: 解析済みのWALレコード状態を保持する XLogReaderState 構造体へのポインタ。
  • **出力**:
  • rec_len: レコードの実データ長を格納する uint32 へのポインタ。
  • fpi_len: レコードに含まれる全FPIの合計長を格納する uint32 へのポインタ。

前提条件と不変条件

  • record は有効な XLogReaderState を指している必要があります。
  • XLogRecGetTotalLen(record) が呼び出し可能な状態である必要があります。

アルゴリズムまたは振る舞い

1. fpi_len を 0 に初期化します。 2. XLogRecMaxBlockId(record) までループし、各ブロックIDに対して以下を実行します。

  • XLogRecHasBlockRef でブロック参照の有無を確認し、存在しない場合はスキップします。
  • XLogRecHasBlockImage でFPIの有無を確認し、存在する場合は XLogRecGetBlock(record, block_id)->bimg_len を fpi_len に加算します。

    • 3. rec_len に XLogRecGetTotalLen(record) から fpi_len を減算した値を格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogstats.c: XLogRecStoreStats 関数内。
  • src/bin/pg_waldump/pg_waldump.c: WAL統計出力処理内。
  • **呼び出し先**:
  • XLogRecMaxBlockId
  • XLogRecHasBlockRef
  • XLogRecHasBlockImage
  • XLogRecGetBlock
  • XLogRecGetTotalLen

関連データ構造

  • XLogReaderState: WALレコードのデコード状態を保持する構造体。

失敗モードと境界条件

  • record が指す構造体が正しくデコードされていない場合、内部のブロック参照やサイズ計算で予期しない挙動を示す可能性があります。

保守担当者向けメモ

  • 本関数は xlogreader の内部構造（bimg_len）を直接参照する実装です。xlogreader のAPI変更時には本関数の更新が必要です。
