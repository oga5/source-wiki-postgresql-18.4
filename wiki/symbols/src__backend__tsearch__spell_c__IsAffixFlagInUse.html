IsAffixFlagInUse

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/spell.c:456

目的

指定された接辞（affix）データ内に、特定の接辞フラグ（affixflag）が含まれているかどうかを判定します。

シグネチャまたは形状

static bool IsAffixFlagInUse(IspellDict *Conf, int affix, const char *affixflag)

入力と出力

  • **入力**:
  • Conf: 辞書の状態を保持する IspellDict 構造体へのポインタ。
  • affix: 検索対象となる接辞データのインデックス。
  • affixflag: 検索する接辞フラグ文字列。
  • **出力**:
  • bool: フラグが見つかった場合は true、それ以外は false を返します。

前提条件と不変条件

  • affix は Conf->nAffixData 未満である必要があります（Assert により検証）。
  • affixflag が空文字列（*affixflag == 0）の場合、常に true を返します。

アルゴリズムまたは振る舞い

1. affixflag が空であれば直ちに true を返します。 2. Conf->AffixData[affix] から接辞フラグの文字列リストを取得します。 3. getNextFlagFromString を使用して、リストからフラグを一つずつ抽出します。 4. 抽出したフラグと affixflag を strcmp で比較し、一致すれば true を返します。 5. リストの終端まで一致するものが見つからなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/spell.c:643
  • src/backend/tsearch/spell.c:1978
  • **呼び出し先**:
  • getNextFlagFromString（内部で呼び出し）
  • strcmp（標準ライブラリ）

関連データ構造

  • IspellDict: 辞書全体の構成情報を保持する構造体。AffixData 配列を介して接辞情報にアクセスします。

失敗モードと境界条件

  • affix が Conf->nAffixData の範囲外である場合、アサーション違反により異常終了する可能性があります。
  • affixflag が非常に長い場合、内部の char flag[BUFSIZ] バッファでオーバーフローするリスクがあるため、呼び出し元で適切な長さの文字列が渡されることが前提となります。

保守担当者向けメモ

  • 本関数は辞書コンパイルおよび正規化処理において、特定の接辞ルールが適用可能かを判定するために頻繁に使用されます。
  • getNextFlagFromString の動作に依存しているため、接辞フラグのパース形式に変更があった場合は本関数のロジックも再確認してください。
