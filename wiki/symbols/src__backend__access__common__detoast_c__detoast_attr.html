detoast_attr

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/detoast.c:115

目的

TOAST（The Oversized-Attribute Storage Technique）されたデータや、圧縮・間接参照・展開済みオブジェクト、あるいはショートヘッダー形式の可変長データ（varlena）を、標準的なメモリ内形式へ復元（detoast）します。

シグネチャまたは形状

struct varlena *detoast_attr(struct varlena *attr);

入力と出力

  • **入力**: struct varlena *attr - 復元対象の可変長データポインタ。
  • **出力**: struct varlena * - 復元されたデータへのポインタ。入力が既に標準形式であればそのまま返される場合があり、変換が必要な場合は palloc された新しいメモリ領域が返されます。

前提条件と不変条件

  • 復元されたデータは palloc されるため、呼び出し元は必要に応じて pfree を行う責任があります。
  • 間接ポインタ（VARATT_IS_EXTERNAL_INDIRECT）の場合、ネストされた間接ポインタは許可されません。
  • 展開済みオブジェクト（VARATT_IS_EXTERNAL_EXPANDED）のフラット化処理は、圧縮やショートヘッダーを出力してはなりません。

アルゴリズムまたは振る舞い

入力データの形式を判定し、以下の順序で処理を行います： 1. **外部データ（オンディスク）**: toast_fetch_datum で取得し、圧縮されていれば toast_decompress_datum で解凍します。 2. **間接ポインタ**: VARATT_EXTERNAL_GET_POINTER で参照先を取得し、再帰的に detoast_attr を呼び出します。参照先が変更されていない場合はメモリコピーを作成します。 3. **展開済みオブジェクト**: detoast_external_attr を呼び出し、フラットな形式へ変換します。 4. **圧縮データ**: toast_decompress_datum で解凍します。 5. **ショートヘッダー**: 4バイトヘッダー形式へ変換し、メモリを再確保してコピーします。

呼び出し元と呼び出し先

  • **呼び出し元**: heaptoast.c、toast_helper.c、inv_api.c、fmgr.c 等から呼び出されます。
  • **呼び出し先**: toast_fetch_datum、toast_decompress_datum、detoast_external_attr、および再帰的に detoast_attr を呼び出します。

関連データ構造

  • struct varlena: PostgreSQLの可変長データ構造体。
  • struct varatt_indirect: 間接ポインタを保持する構造体。

失敗モードと境界条件

  • 圧縮データや外部データが破損している場合、呼び出し先の関数（toast_fetch_datum 等）でエラーが発生する可能性があります。
  • メモリ不足時には palloc がエラーをスローします。

保守担当者向けメモ

  • 本関数はメモリを割り当てる可能性があるため、呼び出し元でのメモリ管理（pfree）を徹底してください。
  • 圧縮アルゴリズムの特性上、スライス処理が必要な場合は detoast_attr_slice の使用を検討してください。
