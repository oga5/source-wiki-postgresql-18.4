logicalrep_write_begin

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/proto.c:48

目的

論理レプリケーションにおけるトランザクション開始メッセージ（BEGIN）をシリアライズし、出力バッファへ書き込む。

シグネチャまたは形状

void logicalrep_write_begin(StringInfo out, ReorderBufferTXN *txn)

入力と出力

  • out (入力/出力): メッセージを書き込むための StringInfo バッファ。
  • txn (入力): 書き込み対象のトランザクション情報を含む ReorderBufferTXN 構造体。
  • 戻り値: なし。

前提条件と不変条件

  • out は初期化済みであること。
  • txn は有効なトランザクションデータ（final_lsn, commit_time, xid）を保持していること。

アルゴリズムまたは振る舞い

1. LOGICAL_REP_MSG_BEGIN メッセージ識別子をバッファに書き込む。 2. txn から以下の固定長フィールドを順次バッファへ書き込む。

  • final_lsn (int64): トランザクションの最終 LSN。
  • commit_time (int64): トランザクションのコミット時刻。
  • xid (int32): トランザクション ID。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/pgoutput/pgoutput.c の pgoutput_begin（直接の証拠）。
  • 呼び出し先: pq_sendbyte, pq_sendint64, pq_sendint32（libpq/pqformat.h で定義されたバッファ操作関数）。

関連データ構造

  • StringInfo: 通信用の可変長バイトバッファ。
  • ReorderBufferTXN: 論理デコード中のトランザクション状態を保持する構造体。

失敗モードと境界条件

  • バッファのメモリ不足が発生した場合、pq_send* 関数群がエラーをスローする可能性がある。
  • txn が NULL の場合、セグメンテーション違反が発生する。

保守担当者向けメモ

  • 本関数は論理レプリケーションプロトコルの BEGIN メッセージを定義する。プロトコルバージョンを変更する場合、対応する logicalrep_read_begin も同様に更新し、logicalproto.h の定義と整合性を保つこと。
