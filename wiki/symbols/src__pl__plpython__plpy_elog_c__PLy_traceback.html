PLy_traceback

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_elog.c:165

目的

Python の例外情報（型、値、トレースバック）を PostgreSQL のエラー報告システムに適した文字列形式に変換し、エラーコンテキストを構築する。

シグネチャまたは形状

static void PLy_traceback(PyObject *e, PyObject *v, PyObject *tb,
                          char *volatile *xmsg, char *volatile *tbmsg, int *tb_depth)

入力と出力

  • **入力**:
  • e: Python 例外型オブジェクト
  • v: Python 例外値オブジェクト
  • tb: Python トレースバックオブジェクト
  • **出力**:
  • *xmsg: 例外メッセージ（char*）
  • *tbmsg: 整形されたトレースバック文字列（char*）
  • *tb_depth: トレースバックの深さ（int）

前提条件と不変条件

  • e が NULL の場合、出力ポインタは NULL に、深さは 0 に設定され即座に復帰する。
  • Python オブジェクトの参照カウント管理のため、PG_TRY/PG_FINALLY ブロック内で処理が行われる。
  • tb が NULL または Py_None になるまでスタックフレームを走査する。

アルゴリズムまたは振る舞い

1. **例外メッセージの生成**: e の __name__ と __module__ 属性、および v の文字列表現から xmsg を構築する。 2. **トレースバックの走査**: tb オブジェクトを辿り、各フレームから行番号、関数名、ファイル名を取得する。 3. **PL/Python コンテキストの適用**: 実行中のプロシージャ名（PLy_procedure_name）を特定し、Python の標準的なトレースバック形式を模倣した文字列を tbmsg に追加する。 4. **ソースコードの抽出**: ファイル名が <string> の場合、get_source_line を使用して該当行のソースコードをトレースバックに含める。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_elog.c 内の PLy_elog_impl（推測される主要な呼び出し元）。
  • **呼び出し先**: PyObject_GetAttrString, PLyUnicode_AsString, PLy_current_execution_context, PLy_procedure_name, get_source_line, elog（エラー発生時）。

関連データ構造

  • StringInfoData: 文字列構築用バッファ。
  • PLyExecutionContext: PL/Python の実行コンテキスト。
  • PyObject: Python の例外およびフレームオブジェクト。

失敗モードと境界条件

  • PyObject_GetAttrString 等の Python API が失敗した場合、elog(ERROR, ...) により PostgreSQL のエラーとして報告される。
  • トレースバックの走査中に tb_next の取得に失敗した場合も同様にエラーとなる。
  • 最初のフレームは <module> を指すため、*tb_depth > 0 の条件でスキップされる。

保守担当者向けメモ

  • Python の traceback モジュールの挙動を模倣するように設計されている。
  • メモリ管理は PostgreSQL の palloc/pfree と Python の参照カウントが混在するため、PG_FINALLY での Py_XDECREF 呼び出しを漏らさないこと。
  • tb_next の参照カウントは、呼び出し元が保持しているため、本関数内では明示的に解放しない設計となっている。
