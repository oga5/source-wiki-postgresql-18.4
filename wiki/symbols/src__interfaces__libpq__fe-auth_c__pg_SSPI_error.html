pg_SSPI_error

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth.c:206

目的

SSPI（Security Support Provider Interface）認証プロセス中に発生したエラーを診断し、その詳細を PGconn オブジェクトのエラーメッセージバッファに記録する。

シグネチャまたは形状

static void pg_SSPI_error(PGconn *conn, const char *mprefix, SECURITY_STATUS r);

入力と出力

  • conn: 接続状態を保持する PGconn 構造体へのポインタ。エラーメッセージの格納先として使用される。
  • mprefix: エラーメッセージの先頭に付与する接頭辞（コンテキスト情報）。
  • r: Windows SSPI API から返された SECURITY_STATUS 型のエラーコード。

前提条件と不変条件

  • Windows 環境での実行を前提とする（FormatMessage API を使用）。
  • conn は有効なポインタである必要がある。

アルゴリズムまたは振る舞い

1. FormatMessage API を呼び出し、引数 r に対応するシステムエラーメッセージを sysmsg バッファに取得する。 2. メッセージの取得に成功した場合、mprefix、システムメッセージ、および 16 進数表記のエラーコードを conn->errorMessage に追加する。 3. メッセージの取得に失敗した場合、mprefix と 16 進数表記のエラーコードのみを conn->errorMessage に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-auth.c 内の SSPI 認証処理（pg_SSPI_continue および pg_SSPI_startup 等）から呼び出される。
  • **呼び出し先**: Windows システム API FormatMessage および libpq 内部関数 appendPQExpBuffer。

関連データ構造

  • PGconn: libpq の接続状態を管理する構造体。
  • SECURITY_STATUS: Windows SSPI API が返すステータスコード型。

失敗モードと境界条件

  • FormatMessage がシステムメッセージを取得できない場合、エラーコードのみが記録される。
  • sysmsg バッファ（256 バイト）を超えるメッセージは切り捨てられる可能性がある。

保守担当者向けメモ

  • 本関数は Windows 特有の認証エラー処理を行うため、プラットフォーム依存のコードブロック内に配置されている。
  • src/backend/libpq/auth.c にも同名の関数が存在するが、本ページは src/interfaces/libpq/fe-auth.c（クライアント側）の定義を対象としている。
