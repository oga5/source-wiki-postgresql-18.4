copy_file

定義

  • **種別**: function
  • **定義箇所**: src/bin/pg_combinebackup/copy_file.c:51

目的

バックアップ再構築プロセスにおいて、指定された戦略（CopyMethod）に基づき、ソースファイルからデスティネーションへファイルをコピーまたは転送する抽象化レイヤーを提供します。

シグネチャまたは形状

void copy_file(const char *src, const char *dst, pg_checksum_context *checksum_ctx, CopyMethod copy_method, bool dry_run);

入力と出力

  • **入力**:
  • src: コピー元ファイルパス。
  • dst: コピー先ファイルパス。
  • checksum_ctx: チェックサム計算用のコンテキスト。
  • copy_method: 使用するコピー戦略（CLONE, COPY, LINK 等）。
  • dry_run: true の場合、実際のコピーを行わず存在確認とログ出力のみを行う。
  • **出力**: なし（void）。成功時はファイルをコピーし、失敗時は pg_fatal によりプロセスを終了します。

前提条件と不変条件

  • dry_run が true の場合、ソースファイルが読み取り可能（open 可能）である必要があります。
  • copy_method に応じた適切な実装関数（copy_file_clone 等）が内部で選択されます。
  • Windows 環境では、COPY_METHOD_COPY が指定された場合、自動的に COPY_METHOD_COPYFILE へ変換されます。

アルゴリズムまたは振る舞い

1. **ドライラン確認**: dry_run モードであれば、ソースファイルを開いて閉じ、存在を確認します。 2. **戦略選択**: copy_method に基づき、実行する関数ポインタ（strategy_implementation）とログ用名称を決定します。 3. **ログ出力**: 実行前（またはドライラン時）に、選択された戦略とチェックサムの有無をログ出力します。 4. **実行**: 選択された strategy_implementation を呼び出し、実際の I/O を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内には直接的な呼び出し元の記述はありません。ツールが提示した contrib/basic_archive/basic_archive.c 等の参照は、本シンボルと名前が一致する別の関数である可能性が高く、本モジュール内での直接的な関係性は確認できません。
  • **呼び出し先**: copy_file_clone, copy_file_blocks, copy_file_by_range, copy_file_copyfile, copy_file_link のいずれか。

関連データ構造

  • CopyMethod: コピー手法を定義する列挙型。
  • pg_checksum_context: チェックサム計算の状態を保持する構造体。

失敗モードと境界条件

  • **ファイルアクセス失敗**: open に失敗した場合、pg_fatal を発生させます。
  • **プラットフォーム非対応**: OS が選択されたコピー戦略をサポートしていない場合、呼び出された実装関数内で pg_fatal が発生する可能性があります。

保守担当者向けメモ

  • 本関数は pg_combinebackup のメインエントリーポイントの一つです。
  • strategy_implementation は関数ポインタとして動的に割り当てられるため、デバッグ時は選択された戦略が意図通りかを確認してください。
  • dry_run モードは単なるシミュレーションではなく、ファイルシステムへのアクセス権限を検証する重要なステップを含みます。
