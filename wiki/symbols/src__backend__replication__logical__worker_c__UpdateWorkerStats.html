UpdateWorkerStats

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/worker.c:3557

目的

論理レプリケーションワーカーの統計情報（最後に受信したLSNや時刻）を更新し、必要に応じて返信用のステータスを同期する。

シグネチャまたは形状

static void UpdateWorkerStats(XLogRecPtr last_lsn, TimestampTz send_time, bool reply)

入力と出力

  • **入力**:
  • last_lsn (XLogRecPtr): パブリッシャーから受信した最新のLSN。
  • send_time (TimestampTz): パブリッシャー側での送信時刻。
  • reply (bool): 返信を行うかどうかのフラグ。
  • **出力**: なし（void）。

前提条件と不変条件

  • グローバル変数 MyLogicalRepWorker が初期化され、有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. MyLogicalRepWorker の last_lsn を引数 last_lsn で更新する。 2. MyLogicalRepWorker の last_send_time を引数 send_time で更新する。 3. GetCurrentTimestamp() を呼び出し、MyLogicalRepWorker の last_recv_time を現在時刻で更新する。 4. reply が true の場合:

  • MyLogicalRepWorker の reply_lsn を last_lsn に設定する。
  • MyLogicalRepWorker の reply_time を send_time に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/worker.c:3681 (引数 reply=false で呼び出し)
  • src/backend/replication/logical/worker.c:3699 (引数 reply=true で呼び出し)
  • **呼び出し先**:
  • GetCurrentTimestamp() (PostgreSQL内部関数)

関連データ構造

  • MyLogicalRepWorker: 論理レプリケーションワーカーの状態を保持する構造体（replication/worker_internal.h で定義）。

失敗モードと境界条件

  • MyLogicalRepWorker が NULL の場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は、レプリケーションの進捗状況を追跡するための軽量な更新処理です。
  • reply フラグが true の場合は、パブリッシャーへのフィードバック送信に関連するステータスも更新されるため、フィードバック送信ロジックと整合性を保つ必要があります。
