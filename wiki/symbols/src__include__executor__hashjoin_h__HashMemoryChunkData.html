HashMemoryChunkData

定義

  • 種別: struct
  • 定義箇所: src/include/executor/hashjoin.h:128

目的

ハッシュ結合において、メモリ内にタプルを格納するためのチャンク単位の管理構造を提供します。

シグネチャまたは形状

typedef struct HashMemoryChunkData
{
    int         ntuples;
    size_t      maxlen;
    size_t      used;
    union
    {
        struct HashMemoryChunkData *unshared;
        dsa_pointer shared;
    }           next;
} HashMemoryChunkData;

入力と出力

  • 入力: 構造体内の各フィールド（ntuples, maxlen, used, next）により、チャンクの状態とリンクを保持します。
  • 出力: 構造体直後のメモリ領域にタプルデータが格納されます。

前提条件と不変条件

  • タプルバッファは、HashMemoryChunkData 構造体の直後、HASH_CHUNK_HEADER_SIZE（MAXALIGN 済み）のオフセットから開始されます。
  • maxlen および used には、ヘッダーサイズは含まれません。

アルゴリズムまたは振る舞い

  • チャンクを連結リストとして管理し、next 共用体を通じて非共有メモリ（unshared）または共有メモリ（dsa_pointer）上の次チャンクを指し示します。
  • タプル格納時に used を更新し、メモリ不足時には新しいチャンクを割り当ててリストを拡張します。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数情報は、提供されたソース抜粋には含まれていません。

関連データ構造

  • HashMemoryChunk: HashMemoryChunkData へのポインタ型。
  • HASH_CHUNK_HEADER_SIZE: 構造体サイズをアライメントしたヘッダーオフセット値。

失敗モードと境界条件

  • メモリ割り当て失敗時（dsa_area またはシステムメモリ不足）の挙動については、本構造体定義のみでは規定されていません。

保守担当者向けメモ

  • 本構造体は並列ハッシュ結合（Parallel Hash Join）と非並列の両方で利用されるため、next フィールドの共用体操作には注意が必要です。
  • タプルバッファのオフセット計算には必ず HASH_CHUNK_HEADER_SIZE を使用してください。
