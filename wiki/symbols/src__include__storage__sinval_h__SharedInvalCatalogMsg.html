SharedInvalCatalogMsg

定義

  • 種別: struct
  • 定義箇所: src/include/storage/sinval.h:70

目的

システムカタログの変更に伴い、他のバックエンドに対してキャッシュの無効化を通知するためのメッセージ構造体です。

シグネチャまたは形状

typedef struct {
    int8    id;         /* メッセージ型識別子（必須） */
    Oid     dbId;       /* データベースID（共有カタログの場合は0） */
    Oid     catId;      /* 無効化対象のカタログID */
} SharedInvalCatalogMsg;

入力と出力

  • **入力**: 構造体の各フィールド（id, dbId, catId）に値を設定することで、無効化対象を特定します。
  • **出力**: 共有メモリ上の無効化キューに書き込まれ、他のバックエンドによって読み取られます。

前提条件と不変条件

  • id フィールドは構造体の先頭に配置され、メッセージ型を識別するために使用されます。
  • dbId が 0 の場合、そのカタログは共有カタログとして扱われます。

アルゴリズムまたは振る舞い

1. バックエンドがシステムカタログを変更した際、この構造体に情報を格納します。 2. SendSharedInvalidMessages を通じて共有メモリ上の無効化キューへ送信されます。 3. 受信側のバックエンドは ReceiveSharedInvalidMessages を介してこのメッセージを取得し、id に基づいて対応するカタログキャッシュ（Catcache）の無効化処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: 具体的な関数呼び出しの証拠は提供されていませんが、src/include/storage/sinval.h 内の SharedInvalidationMessage 共用体の一部として参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。処理ロジックは src/backend/storage/ipc/sinval.c に実装されている無効化処理関数に依存します。

関連データ構造

  • SharedInvalidationMessage: 本構造体を含む、全無効化メッセージを包含する共用体。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • **ハッシュ衝突**: Catcacheの無効化はハッシュ値ベースで行われるため、稀に無関係なエントリが誤って無効化される可能性がありますが、これはキャッシュの再ロードによって整合性が保たれます。
  • **キューの溢れ**: 共有メモリ上の無効化キューが溢れた場合、PROCSIG_CATCHUP_INTERRUPT を用いた強制的な同期が必要となります。

保守担当者向けメモ

  • 本構造体はトランザクションに紐付いて送信されます。コミット時にのみ無効化が伝播される点に注意してください。
  • id フィールドの型定義は、メッセージの識別を確実にするため変更しないでください。
