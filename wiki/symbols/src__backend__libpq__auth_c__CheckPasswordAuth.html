CheckPasswordAuth

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth.c:787

目的

クライアントから平文パスワードを受け取り、データベースに保存されたロールのパスワードと照合することで認証を行う。

シグネチャまたは形状

static int CheckPasswordAuth(Port *port, const char **logdetail);

入力と出力

  • **入力**:
  • port: 現在の接続状態を保持する Port 構造体へのポインタ。
  • logdetail: 認証失敗時の詳細情報を格納するための文字列ポインタのポインタ。
  • **出力**:
  • 認証結果を示す整数値（STATUS_OK, STATUS_ERROR, STATUS_EOF）。

前提条件と不変条件

  • port は有効な接続状態であること。
  • 認証方式としてパスワード認証が選択されていること。

アルゴリズムまたは振る舞い

1. sendAuthRequest を呼び出し、クライアントに AUTH_REQ_PASSWORD を要求する。 2. recv_password_packet でクライアントからパスワードを受信する。受信失敗時は STATUS_EOF を返す。 3. get_role_password を使用して、対象ユーザーの保存済みパスワードを取得する。 4. 保存済みパスワードが存在する場合、plain_crypt_verify を呼び出し、受信したパスワードと照合する。 5. 照合成功時、set_authn_id を呼び出して認証済みIDを確定させる。 6. 取得したパスワードメモリを pfree で解放し、最終的な結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth.c 内の認証フロー（ClientAuthentication 等から呼び出される）。
  • **呼び出し先**:
  • sendAuthRequest
  • recv_password_packet
  • get_role_password
  • plain_crypt_verify
  • set_authn_id
  • pfree

関連データ構造

  • Port: 接続情報およびユーザー名を保持する構造体。

失敗モードと境界条件

  • クライアントがパスワードを送信せずに切断した場合: STATUS_EOF を返す。
  • ユーザーが存在しない、またはパスワードが設定されていない場合: STATUS_ERROR を返す。
  • パスワード照合に失敗した場合: plain_crypt_verify の結果に基づき STATUS_ERROR を返す。

保守担当者向けメモ

  • 本関数は平文パスワードを扱うため、メモリ上のパスワード文字列の取り扱いには注意すること。
  • 認証失敗時の詳細は logdetail を通じて呼び出し元に伝達される。
