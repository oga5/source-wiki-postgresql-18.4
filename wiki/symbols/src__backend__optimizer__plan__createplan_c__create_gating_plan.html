create_gating_plan

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/createplan.c:1021

目的

実行計画のノードに対して、実行時に評価される制約（gating_quals）を適用するための Result ノードを挿入する。

シグネチャまたは形状

Plan *create_gating_plan(PlannerInfo *root, Path *path, Plan *plan, List *gating_quals)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • path: 対応する Path ノード。
  • plan: ゲートを適用する対象の Plan ノード。
  • gating_quals: 実行時に評価される制約リスト。
  • **出力**:
  • ゲート機能を持つ Result ノード（または plan をラップした新しい Plan ツリー）。

前提条件と不変条件

  • gating_quals は空であってはならない（Assert(gating_quals)）。
  • ゲート適用後も、コストおよびサイズの見積もりは元の plan の値を維持する（保守的な見積もり）。

アルゴリズムまたは振る舞い

1. **既存ノードの最適化**: 入力 plan が既に Result ノードであり、かつ子ノードや定数制約を持たない場合は、そのノードを再利用（破棄）して最適化する。 2. **Resultノードの生成**: build_path_tlist を用いてターゲットリストを構築し、make_result を呼び出して gating_quals を持つ Result ノードを作成する。 3. **メタデータの継承**: copy_plan_costsize を使用して、元のプランのコストとサイズ情報を新しいノードにコピーする。 4. **安全性の伝播**: Path の parallel_safe フラグを新しいプランノードに設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/createplan.c 内の複数箇所（行813、1114など）。
  • **呼び出し先**: make_result, build_path_tlist, copy_plan_costsize 等のヘルパー関数。

関連データ構造

  • Plan: 実行計画の基本ノード。
  • Result: ゲートとして挿入されるプランノード型。
  • Path: 最適化段階のパス情報。
  • PlannerInfo: プランナのコンテキスト情報。

失敗モードと境界条件

  • gating_quals が定数 FALSE である場合、プランナの別箇所（clausesel.c）で処理されることが想定されている。
  • ゲート適用によるコストやサイズの変更は行わないため、実行時の実際の評価結果とは乖離する可能性がある。

保守担当者向けメモ

  • 本関数は、実行時に制約が偽となる場合にサブプランの実行をスキップさせる「ゲート」を挿入する役割を持つ。
  • ターゲットリストの生成には build_path_tlist を使用しており、CP_EXACT_TLIST の有無に関わらず、パスが要求するターゲットリストを常に返す設計となっている。
