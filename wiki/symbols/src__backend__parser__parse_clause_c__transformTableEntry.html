transformTableEntry

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_clause.c:394

目的

RangeVar（単純なリレーション参照）を解析し、クエリの FROM 句におけるテーブルエントリとして変換・登録します。

シグネチャまたは形状

static ParseNamespaceItem *transformTableEntry(ParseState *pstate, RangeVar *r)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態を保持する ParseState 構造体。
  • r: 変換対象の RangeVar 構造体。
  • **出力**:
  • 変換されたリレーションを指す ParseNamespaceItem 構造体へのポインタ。

前提条件と不変条件

  • pstate が有効な状態であること。
  • r が有効な RangeVar 構造体であること。

アルゴリズムまたは振る舞い

内部的に addRangeTableEntry を呼び出し、以下の処理を委譲します。 1. r に含まれるエイリアス情報 (r->alias) を使用。 2. 継承の有無 (r->inh) を適用。 3. リレーションを ParseState に追加し、対応する ParseNamespaceItem を生成して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_clause.c:1073 にて呼び出されていることが確認されています。
  • **呼び出し先**: addRangeTableEntry を呼び出します。

関連データ構造

  • ParseState: 解析中のクエリ状態を管理。
  • RangeVar: SQLの FROM 句等に現れるリレーション参照の構文解析結果。
  • ParseNamespaceItem: 名前空間に追加されたリレーションの管理用構造体。

失敗モードと境界条件

  • addRangeTableEntry が失敗した場合（リレーションが見つからない、権限がない等）、そのエラーがそのまま伝播します。

保守担当者向けメモ

  • 本関数は addRangeTableEntry の単なるラッパーです。リレーション解決のロジックを変更する場合は addRangeTableEntry を参照してください。
