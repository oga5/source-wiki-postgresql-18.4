plpgsql_validator

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_handler.c:441

目的

CREATE FUNCTION 実行時に、PL/pgSQL 関数の構文、型、およびコンパイルの妥当性を検証します。

シグネチャまたは形状

Datum plpgsql_validator(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して関数 OID を受け取ります。
  • **出力**: 戻り値なし (PG_RETURN_VOID())。検証失敗時はエラーを報告します。

前提条件と不変条件

  • CheckFunctionValidatorAccess により、呼び出し元に検証権限があることを確認します。
  • check_function_bodies が有効な場合にのみ、関数本体のコンパイルテストが実行されます。
  • 疑似型（pseudotype）の使用は、TRIGGER、EVENT_TRIGGER、RECORD、VOID、または多相型を除き禁止されています。

アルゴリズムまたは振る舞い

1. pg_proc システムカタログから対象関数の情報を取得します。 2. 戻り値および引数の型をチェックし、許可されていない疑似型が含まれていないか検証します。 3. check_function_bodies が真の場合:

  • SPI_connect() を呼び出し、SPI コンテキストを確立します。
  • 実行に必要な最小限の情報を保持する fake_fcinfo を構築します。
  • plpgsql_compile() を呼び出し、関数をコンパイルして構文エラー等を検出します。
  • SPI_finish() で接続を終了します。

    • 4. 最後に ReleaseSysCache() でシステムキャッシュを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（CREATE FUNCTION 実行時）。
  • **呼び出し先**: CheckFunctionValidatorAccess, SearchSysCache1, get_typtype, get_func_arg_info, SPI_connect, plpgsql_compile, SPI_finish, ReleaseSysCache。

関連データ構造

  • Form_pg_proc: 関数定義情報。
  • FunctionCallInfo (fcinfo): 関数呼び出し情報。
  • TriggerData / EventTriggerData: トリガー関数検証時に使用されるコンテキスト情報。

失敗モードと境界条件

  • 対象関数がシステムカタログに見つからない場合、elog(ERROR) で終了します。
  • 許可されない疑似型が使用されている場合、ereport(ERROR) で ERRCODE_FEATURE_NOT_SUPPORTED を送出します。
  • SPI_finish() が失敗した場合、elog(ERROR) で終了します。

保守担当者向けメモ

  • 本関数は check_function_bodies が無効な場合、本体のコンパイルテストをスキップします。そのため、バリデーションが成功しても実行時にエラーが発生する可能性があります。
  • fake_fcinfo の構築は plpgsql_compile を動作させるための最小限のハックであるため、コンパイルロジックに変更を加える際は注意してください。
