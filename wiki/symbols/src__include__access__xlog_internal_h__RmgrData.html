RmgrData

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlog_internal.h:349

目的

WAL（先行書き込みログ）レコードを処理するためのリソースマネージャ（Rmgr）のメソッドテーブルを定義します。各リソースマネージャが提供するRedo処理、デコード、識別などの機能を統合的に管理するために使用されます。

シグネチャまたは形状

typedef struct RmgrData
{
	const char *rm_name;
	void		(*rm_redo) (XLogReaderState *record);
	void		(*rm_desc) (StringInfo buf, XLogReaderState *record);
	const char *(*rm_identify) (uint8 info);
	void		(*rm_startup) (void);
	void		(*rm_cleanup) (void);
	void		(*rm_mask) (char *pagedata, BlockNumber blkno);
	void		(*rm_decode) (struct LogicalDecodingContext *ctx,
							  struct XLogRecordBuffer *buf);
} RmgrData;

入力と出力

  • **入力**: rm_redo, rm_desc は XLogReaderState を受け取ります。rm_identify は uint8 型の情報を入力とします。rm_mask はページデータとブロック番号を受け取ります。rm_decode は論理デコードコンテキストとレコードバッファを受け取ります。
  • **出力**: rm_name は文字列ポインタ、rm_identify は文字列ポインタを返します。その他は副作用（Redo実行、バッファへの記述、マスク処理等）を目的とします。

前提条件と不変条件

  • RmgrTable（RmgrData の配列）は、src/include/access/rmgrlist.h で定義されたリソースマネージャIDと同期している必要があります。
  • rm_redo 等の関数ポインタは、各リソースマネージャの実装において適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

WALリカバリ時や論理デコード時に、レコードの RmgrId に基づいて RmgrTable から対応する RmgrData を取得し、各メソッドを呼び出すことでレコードの再適用や解析を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogrecovery.c: GetRmgr() を介してリカバリ時に呼び出されます。
  • src/backend/replication/logical/decode.c: 論理デコード時に参照されます。
  • contrib/pg_walinspect/pg_walinspect.c: WALの内容を検査する際に rm_desc が利用されます。
  • **呼び出し先**:
  • RegisterCustomRmgr() を通じてカスタムリソースマネージャの登録に使用されます。

関連データ構造

  • RmgrTable: RmgrData のグローバル配列。
  • XLogReaderState: WALレコードの読み込み状態を保持する構造体。
  • LogicalDecodingContext: 論理デコードのコンテキスト。

失敗モードと境界条件

  • RmgrTable の範囲外の RmgrId を指定した場合の動作は未定義です（RM_MAX_ID による制限あり）。
  • 各関数ポインタが NULL の場合、呼び出し時にセグメンテーションフォールトが発生する可能性があります。

保守担当者向けメモ

  • 新しいリソースマネージャを追加する場合は、RmgrData の定義を維持しつつ src/include/access/rmgrlist.h を更新してください。
  • src/backend/access/transam/rmgr.c にある RmgrTable の定義と本構造体の定義は常に同期させる必要があります。
