keyGetItem

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginget.c:1004

目的

特定の GIN スキャンキーに対して、指定された位置 (advancePast) 以降で次に合致するアイテムポインタ（TID）を特定し、スキャン状態を更新する。

シグネチャまたは形状

static void keyGetItem(GinState *ginstate, MemoryContext tempCtx, GinScanKey key, ItemPointerData advancePast)

入力と出力

  • **入力**:
  • ginstate: GIN インデックスの操作状態。
  • tempCtx: 一時的なメモリコンテキスト。
  • key: 検索対象の GinScanKey 構造体。
  • advancePast: この位置より後のアイテムを検索するための基準点。
  • **出力**: なし（key->curItem を更新し、必要に応じて key->isFinished を設定する）。

前提条件と不変条件

  • key->isFinished が false であること。
  • key->curItem は、advancePast より大きい値に更新される。
  • 損失のあるページ（Lossy Page）ポインタは、同一ページ内の正確なポインタよりも後にソートされる。

アルゴリズムまたは振る舞い

1. **初期チェック**: key->curItem が既に advancePast より大きい場合は即座に終了する。 2. **必須エントリの走査**: key->requiredEntries 内の各エントリに対し、entryGetItem を呼び出して advancePast 以降の最小アイテム (minItem) を特定する。 3. **範囲の決定**: minItem に基づき、次に検索すべき範囲を advancePast に設定する。 4. **追加エントリの読み込み**: key->additionalEntries を走査し、必要に応じて entryGetItem で読み込み、minItem を更新する。 5. **一致判定**: 最終的な minItem を key->curItem に設定し、損失のあるページについては consistentFn を用いて三値論理（TRUE/FALSE/MAYBE）で判定を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: scanGetItem（src/backend/access/gin/ginget.c:1353 など）。
  • **呼び出し先**: entryGetItem、ginCompareItemPointers、ItemPointerIsLossyPage、consistentFn（ginlogic.c 経由）。

関連データ構造

  • GinScanKey: 検索キーの状態を保持する構造体。
  • GinScanEntry: 個別のエントリの走査状態。
  • ItemPointerData: ヒープタプルを指すポインタ。

失敗モードと境界条件

  • **全エントリ終了**: 全てのエントリが走査完了した場合、key->isFinished を true に設定する。
  • **損失のあるページ**: ページ全体が合致対象となる場合、ItemPointerSetLossyPage を使用してページ単位のポインタを返す。
  • **除外のみのキー**: excludeOnly フラグが立っている場合、必須エントリの制約を受けずに advancePast の次を検索する。

保守担当者向けメモ

  • keyGetItem は scanGetItem と密接に関連しており、ロジックの重複や不整合に注意すること。
  • 損失のあるページ（Lossy Page）の処理は、consistentFn の結果に依存するため、MAYBE 状態の扱いが性能に直結する。
  • entryGetItem の呼び出し順序や、minItem の更新ロジックは、インデックスの走査効率を左右する重要な最適化ポイントである。
