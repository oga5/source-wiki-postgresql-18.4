JsonPath

定義

  • 種別: struct
  • 定義箇所: src/include/utils/jsonpath.h:23

目的

SQL/JSON Path 言語の式をバイナリ形式（オンディスク形式）で保持するための可変長データ型（varlena）構造体です。

シグネチャまたは形状

typedef struct {
    int32    vl_len_;    /* varlena header */
    uint32   header;     /* バージョンおよびフラグ */
    char     data[FLEXIBLE_ARRAY_MEMBER];
} JsonPath;

入力と出力

  • **入力**: SQL 文字列が parsejsonpath 関数によって解析され、中間表現を経て本構造体にシリアライズされます。
  • **出力**: JsonPathItem 構造体へ変換されることで、JSONB データに対するクエリ実行エンジンがパスを走査・評価するために利用されます。

前提条件と不変条件

  • vl_len_ は varlena ヘッダーであるため、直接操作してはなりません。
  • header フィールドにはバージョン情報とフラグが含まれます。
  • 本構造体は可変長データ型として扱われるため、メモリ配置およびアライメントには PostgreSQL の varlena 規則が適用されます。

アルゴリズムまたは振る舞い

1. **シリアライズ**: 文字列から解析された中間表現（JsonPathParseResult）をバイナリ形式に変換して格納します。 2. **走査**: 実行時には JsonPathItem を介して data 配列内のノードを辿ります。各ノードは nextPos を持ち、リンク構造としてパスを評価します。 3. **評価**: JsonPathQuery 等の関数が本構造体を読み込み、JSONB データに対してパスを適用します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/executor/execExprInterp.c: JSONPath 評価時のエラーハンドリング等で使用。
  • src/backend/utils/adt/jsonb_gin.c: GIN インデックスの抽出処理で使用。
  • src/backend/utils/adt/jsonpath.c: 文字列変換や内部的なパス操作で使用。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。操作には jspInit や jspGetNext などのアクセサ関数が使用されます。

関連データ構造

  • JsonPathItem: 実行時にバイナリ形式を走査するための抽象化構造体。
  • JsonPathParseResult: 解析時の中間表現。
  • Jsonb: 本構造体を用いてクエリされる対象のデータ型。

失敗モードと境界条件

  • 不正なバイナリ形式が渡された場合、JsonPathItem による走査中にメモリ境界外アクセスや不正なポインタ参照が発生する可能性があります。
  • vl_len_ の不整合は、PostgreSQL の varlena 処理全体におけるメモリ破損を引き起こします。

保守担当者向けメモ

  • 本構造体はオンディスク形式であるため、フィールドの追加や変更は後方互換性を維持する必要があります。
  • 走査は「左オペランド」を起点とする設計になっているため、構造の変更時にはこの走査ロジックへの影響を考慮してください。
  • デバッグ時には jspOperationName を使用して、バイナリ内の演算子を可読形式で確認可能です。
