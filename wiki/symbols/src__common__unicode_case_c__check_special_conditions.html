check_special_conditions

定義

  • 種別: function
  • 定義箇所: src/common/unicode_case.c:369

目的

Unicode 文字のケース変換において、文脈依存の特殊な条件（例：ギリシャ語の最終シグマ）を満たしているかを判定します。

シグネチャまたは形状

static bool check_special_conditions(int conditions, const char *str, size_t len, size_t offset);

入力と出力

  • conditions: 判定対象の条件フラグ（PG_U_FINAL_SIGMA 等）。
  • str: 対象となる文字列の先頭ポインタ。
  • len: 文字列の全長。
  • offset: 現在の文字位置。
  • 戻り値: 条件を満たす場合は true、満たさない場合は false。

前提条件と不変条件

  • conditions が 0 の場合は常に true を返します。
  • サポートされていない条件が指定された場合、アサーションエラーが発生します。

アルゴリズムまたは振る舞い

1. conditions が 0 であれば true を返します。 2. conditions が PG_U_FINAL_SIGMA の場合、check_final_sigma を呼び出し、その結果を返します。 3. 上記以外の条件が渡された場合、Assert(false) により実行を停止します。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/unicode_case.c 内のケース変換ロジック（special_case テーブルを参照する箇所）。
  • 呼び出し先: check_final_sigma（conditions == PG_U_FINAL_SIGMA の場合）。

関連データ構造

  • PG_U_FINAL_SIGMA: ギリシャ語の最終シグマ判定用の定数。

失敗モードと境界条件

  • 未定義の conditions が渡された場合、デバッグビルドではアサーションにより停止します。
  • str が指すメモリ領域が len 未満である場合、check_final_sigma 内部での不正なメモリ参照のリスクがあります。

保守担当者向けメモ

  • 現在サポートされている特殊条件は PG_U_FINAL_SIGMA のみです。新しい文脈依存ルールを追加する場合は、本関数に条件分岐を追加してください。
