plsample_trigger_handler

定義

  • **種別**: function
  • **定義箇所**: src/test/modules/plsample/plsample.c:204

目的

PostgreSQL のトリガー呼び出しを処理するためのハンドラ実装テンプレート。トリガー実行時のメタデータ取得、SPI（Server Programming Interface）の接続、およびトリガーイベント情報のログ出力を行う。

シグネチャまたは形状

static HeapTuple plsample_trigger_handler(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（fcinfo を介して TriggerData 構造体へアクセス）。
  • **出力**: HeapTuple（処理対象のタプルを返す）。

前提条件と不変条件

  • CALLED_AS_TRIGGER(fcinfo) が真であること（トリガーマネージャから呼び出されていること）。
  • SPI_connect() を呼び出し、終了時に SPI_finish() を呼び出すこと。
  • PG_TRY/PG_CATCH ブロックにより、エラー発生時の安全なリソース解放を保証すること。

アルゴリズムまたは振る舞い

1. **初期化**: SPI_connect() を実行し、SPI_register_trigger_data() でトリガーデータを登録する。 2. **メタデータ取得**: SearchSysCache1 を使用して pg_proc から関数定義を取得し、ソースコードを NOTICE レベルでログ出力する。 3. **実行**: PG_TRY ブロック内で以下の処理を行う。

  • トリガー名、リレーション名、スキーマ名を出力。
  • tg_event を解析し、INSERT/UPDATE/DELETE/TRUNCATE の種別および実行タイミング（BEFORE/AFTER/INSTEAD OF）、実行レベル（ROW/STATEMENT）を判定してログ出力する。
  • トリガー引数を順次出力する。

    • 4. **終了**: SPI_finish() を呼び出し、処理対象のタプル（rettup）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: plsample_call_handler（src/test/modules/plsample/plsample.c 内）。
  • **呼び出し先**: SPI_connect, SPI_register_trigger_data, SearchSysCache1, ReleaseSysCache, SPI_getrelname, SPI_getnspname, SPI_finish 等の PostgreSQL SPI およびシステムキャッシュ関数。

関連データ構造

  • TriggerData: トリガー実行時のコンテキスト情報（fcinfo->context から取得）。
  • HeapTuple: 処理対象のタプル。
  • Form_pg_proc: 関数定義情報。

失敗モードと境界条件

  • CALLED_AS_TRIGGER ではない場合、ERROR で終了する。
  • pg_proc のキャッシュルックアップに失敗した場合、ERROR で終了する。
  • 関数ソースコードが見つからない場合、ERROR で終了する。
  • 未知のトリガーイベント、タイミング、またはレベルが検出された場合、ERROR で終了する。
  • SPI_finish() が SPI_OK_FINISH を返さない場合、ERROR で終了する。

保守担当者向けメモ

  • 本関数はサンプル実装であり、実際の言語処理系は含まれていない。
  • トリガーハンドラ内で SPI_connect() を使用しているため、必ず SPI_finish() が呼び出されるパスを通すこと。
  • PG_TRY ブロック内でのエラーハンドリングは、必要に応じてクリーンアップ処理を追加すること。
