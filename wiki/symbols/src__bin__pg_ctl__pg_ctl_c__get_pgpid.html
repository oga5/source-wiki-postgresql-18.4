get_pgpid

定義

  • 種別: function
  • 定義箇所: src/bin/pg_ctl/pg_ctl.c:245

目的

データディレクトリ内の PID ファイルを読み取り、現在動作している PostgreSQL サーバー（postmaster）のプロセス ID を取得する。

シグネチャまたは形状

static pid_t get_pgpid(bool is_status_request)

入力と出力

  • **入力**: is_status_request (bool) - ステータス確認要求かどうかを示すフラグ。真の場合、エラー発生時に終了コード 4 を返す。
  • **出力**: pid_t - 読み取ったプロセス ID。PID ファイルが存在しない場合は 0 を返す。

前提条件と不変条件

  • グローバル変数 pg_data が有効なデータディレクトリパスを指していること。
  • グローバル変数 pid_file が PID ファイルのパスを指していること。
  • グローバル変数 version_file がデータベースクラスターのバージョンファイルパスを指していること。

アルゴリズムまたは振る舞い

1. pg_data ディレクトリの存在を確認し、アクセス不能な場合はエラー終了する。 2. version_file の存在を確認し、データベースクラスターディレクトリでない場合はエラー終了する。 3. pid_file を読み取りモードで開く。ファイルが存在しない場合は 0 を返す。 4. ファイルから整数値を読み取る。読み取り失敗時は、ファイルが空か否かを判定し、エラーメッセージを出力して終了する。 5. 読み取った PID を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_ctl/pg_ctl.c 内の各コマンド実行関数（do_stop, do_restart, do_reload, do_status 等）。
  • **呼び出し先**: 標準ライブラリ関数 (stat, fopen, fscanf, fclose, ftell, feof) および write_stderr。

関連データ構造

  • pid_t (システム定義のプロセスID型)
  • FILE (標準入出力ストリーム)
  • struct stat (ファイル状態構造体)

失敗モードと境界条件

  • **ディレクトリ不在**: pg_data が存在しない場合、エラーを出力して終了する。
  • **クラスターディレクトリ判定失敗**: version_file が存在しない場合、エラーを出力して終了する。
  • **PID ファイル読み取り失敗**: ファイルが空、または整数値として解析できない場合、エラーを出力して終了する。
  • **終了コード**: is_status_request が真の場合、上記エラー時に 4 を返し、それ以外は 1 を返す。

保守担当者向けメモ

  • 本関数は pg_ctl の各操作においてサーバーの生存確認を行うための基盤である。
  • is_status_request フラグは、Linux Standard Base (LSB) 仕様に準拠した終了コードを返すために使用されている。
