heapam_tuple_tid_valid

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_handler.c:206

目的

指定されたタプル識別子（TID）が、現在のヒープスキャンにおいて論理的に有効な範囲内にあるかを判定します。

シグネチャまたは形状

static bool heapam_tuple_tid_valid(TableScanDesc scan, ItemPointer tid)

入力と出力

  • **入力**:
  • scan: 現在のスキャン状態を保持する TableScanDesc。
  • tid: 検査対象の ItemPointer。
  • **出力**:
  • bool: TIDが有効なブロック番号を指しており、かつスキャン対象のテーブル範囲内であれば true、それ以外は false を返します。

前提条件と不変条件

  • scan は HeapScanDesc にキャスト可能である必要があります。
  • tid が指すブロック番号は、スキャン対象テーブルの総ブロック数（hscan->rs_nblocks）未満である必要があります。

アルゴリズムまたは振る舞い

1. scan を HeapScanDesc 型にキャストし、スキャン対象の総ブロック数 rs_nblocks を取得します。 2. ItemPointerIsValid(tid) を呼び出し、TID自体が有効な形式かを確認します。 3. ItemPointerGetBlockNumber(tid) で取得したブロック番号が、rs_nblocks 未満であることを確認します。 4. 上記の条件を両方満たす場合にのみ true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_handler.c 内の heapam_methods 構造体初期化箇所（tuple_tid_valid フィールドへの代入）。
  • **呼び出し先**:
  • ItemPointerIsValid (マクロ)
  • ItemPointerGetBlockNumber (マクロ)
  • **補足**: 直接的な実行時呼び出し元については、提供されたソース excerpt には記載がありません。

関連データ構造

  • TableScanDesc: テーブルスキャンの汎用記述子。
  • HeapScanDesc: ヒープスキャン固有の記述子。rs_nblocks を保持します。
  • ItemPointer: タプル位置を特定するためのデータ構造。

失敗モードと境界条件

  • tid が無効な場合（ItemPointerIsValid が偽）、false を返します。
  • tid が指すブロック番号がテーブルの末尾を超えている場合、false を返します。

保守担当者向けメモ

  • 本関数は TableAmRoutine のコールバックとして実装されています。
  • rs_nblocks はスキャン開始時または更新時のテーブルサイズに基づいています。動的なテーブル拡張が発生した場合の整合性については、呼び出し側のスキャンロジックに依存します。
