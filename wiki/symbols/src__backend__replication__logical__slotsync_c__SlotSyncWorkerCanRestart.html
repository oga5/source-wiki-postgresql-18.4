SlotSyncWorkerCanRestart

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/slotsync.c:1762

目的

論理レプリケーションスロット同期ワーカーの再起動が可能かどうかを判定し、再起動が許可される場合は最後に起動した時刻を更新します。

シグネチャまたは形状

bool SlotSyncWorkerCanRestart(void)

入力と出力

  • **入力**: なし（共有メモリ上の SlotSyncCtx を参照）
  • **出力**: bool 型。再起動が許可される場合は true、前回の起動から SLOTSYNC_RESTART_INTERVAL_SEC 秒が経過していない場合は false を返します。

前提条件と不変条件

  • SlotSyncCtx が初期化されている必要があります。
  • SLOTSYNC_RESTART_INTERVAL_SEC で定義された間隔が、再起動の頻度を制限する不変条件として機能します。

アルゴリズムまたは振る舞い

1. 現在時刻 curtime を取得します。 2. curtime と SlotSyncCtx->last_start_time の差分を計算します。 3. 差分が SLOTSYNC_RESTART_INTERVAL_SEC 未満であれば false を返します。 4. 差分が間隔以上であれば、SlotSyncCtx->last_start_time を curtime で更新し、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/postmaster.c:3342 にて、ワーカーの再起動判定の一部として呼び出されています。
  • **呼び出し先**: time() 関数（標準ライブラリ）。

関連データ構造

  • SlotSyncCtx: 同期プロセスの状態を保持する共有メモリ構造体。last_start_time フィールドを使用して再起動間隔を管理します。

失敗モードと境界条件

  • **境界条件**: 前回の起動時刻が設定されていない場合や、システム時刻が変更された場合の挙動については、time_t の計算結果に依存します。
  • **失敗モード**: 共有メモリが正しくアタッチされていない場合、SlotSyncCtx へのアクセスで問題が発生する可能性があります。

保守担当者向けメモ

  • 本関数は postmaster プロセスから呼び出されるため、共有メモリへのアクセスが適切に同期されていることを前提としています。
  • 再起動間隔の定数 SLOTSYNC_RESTART_INTERVAL_SEC を変更する場合は、同期プロセスの負荷とリカバリ速度のトレードオフを考慮してください。
