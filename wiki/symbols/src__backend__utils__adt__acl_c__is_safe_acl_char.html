is_safe_acl_char

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/acl.c:145

目的

ACL（アクセス制御リスト）の文字列解析において、特定の文字が安全な識別子の一部として許容されるかを判定します。

シグネチャまたは形状

static inline bool is_safe_acl_char(unsigned char c, bool is_getid)

入力と出力

  • **入力**:
  • c: 判定対象の文字（unsigned char）。
  • is_getid: 識別子取得モードかどうかを示すフラグ。
  • **出力**:
  • bool: 文字が安全とみなされる場合は true、それ以外は false を返します。

前提条件と不変条件

  • IS_HIGHBIT_SET(c) が真（非ASCII文字）の場合、is_getid の値がそのまま結果となります。
  • IS_HIGHBIT_SET(c) が偽（ASCII文字）の場合、英数字またはアンダースコア（_）であれば安全とみなされます。

アルゴリズムまたは振る舞い

1. 文字 c の上位ビットが立っている（非ASCII）場合、is_getid の値を返します。 2. 上位ビットが立っていない場合、isalnum(c)（英数字）または c == '_' のいずれかを満たす場合に true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/acl.c:180
  • src/backend/utils/adt/acl.c:230
  • **呼び出し先**:
  • isalnum（標準ライブラリ関数）

関連データ構造

  • なし（プリミティブな型のみを使用）。

失敗モードと境界条件

  • is_getid が false の場合、非ASCII文字は常に安全ではない（false）と判定されます。
  • isalnum の挙動は現在のロケール設定に依存する可能性があります。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ACL 文字列のパース処理におけるホットパスで使用されることを想定しています。
  • 識別子の妥当性チェックルールを変更する場合は、is_getid のフラグ制御と併せて影響範囲を確認してください。
