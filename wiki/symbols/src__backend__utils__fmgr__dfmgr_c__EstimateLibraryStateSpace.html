EstimateLibraryStateSpace

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/dfmgr.c:701

目的

ロード済みの動的ライブラリリストをシリアライズするために必要な共有メモリ領域のサイズを算出します。

シグネチャまたは形状

Size EstimateLibraryStateSpace(void)

入力と出力

  • 入力: なし（グローバル変数 file_list を参照）
  • 出力: Size 型（必要なバイト数）

前提条件と不変条件

  • file_list が有効な DynamicFileList の連結リストを指していること。
  • add_size 関数により、サイズ計算時のオーバーフローが防止されていること。

アルゴリズムまたは振る舞い

1. 初期サイズを 1 に設定します（終端文字用）。 2. グローバルな file_list を先頭から順に走査します。 3. 各ノードの filename 文字列の長さに 1 を加えた値を、現在の合計サイズに加算します。 4. 最終的な合計サイズを返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/parallel.c:270
  • 呼び出し先: add_size（サイズ加算用ユーティリティ）

関連データ構造

  • DynamicFileList: ロード済みライブラリを管理する連結リスト構造体。

失敗モードと境界条件

  • file_list が空（NULL）の場合、初期値である 1 を返します。
  • 文字列長が極端に長い場合、add_size の内部ロジックに従い、オーバーフロー発生時にはエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数は、パラレルクエリ実行時にバックエンド間でロード済みライブラリの状態を共有するために使用されます。
  • file_list の構造を変更した場合は、本関数および対応する SerializeLibraryState / RestoreLibraryState の更新が必須です。
