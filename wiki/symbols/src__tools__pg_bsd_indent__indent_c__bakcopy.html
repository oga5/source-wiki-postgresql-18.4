bakcopy

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/indent.c:1202

目的

入力ファイルをバックアップ（.BAK ファイルを作成）し、元のファイル名で出力先を再オープンすることで、インプレース編集を可能にする。

シグネチャまたは形状

static void bakcopy(void)

入力と出力

  • **入力**: グローバル変数 in_name（ファイル名）および input（現在開いている入力ストリーム）。
  • **出力**: なし（副作用としてファイルシステム上に .BAK ファイルを作成し、グローバル変数 input および output を再設定する）。

前提条件と不変条件

  • in_name が有効なファイルパスを指していること。
  • input が読み取り可能な状態で開かれていること。

アルゴリズムまたは振る舞い

1. in_name からファイル名部分を抽出し、.BAK を付与したバックアップファイル名を生成する。 2. creat を使用してバックアップファイルをモード 0600 で作成する。 3. input ストリームの内容を 8KB 単位のバッファを用いてバックアップファイルへコピーする。 4. 元の input を閉じ、バックアップファイルを読み取り専用で再オープンして input に割り当てる。 5. 元のファイル名で output を書き込みモード（wb）でオープンする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/tools/pg_bsd_indent/indent.c:227（main 関数内）。
  • **呼び出し先**: creat, read, write, close, fclose, fopen, unlink, err（標準ライブラリおよびエラーハンドラ）。

関連データ構造

  • in_name: 入力ファイル名を保持するグローバル変数。
  • bakfile: バックアップファイル名を格納するグローバルバッファ。
  • input, output: ファイルストリームを保持するグローバル変数。

失敗モードと境界条件

  • バックアップファイルの作成（creat）に失敗した場合、err を呼び出しプログラムを終了する。
  • ファイルのコピー中に読み取りまたは書き込みエラーが発生した場合、err を呼び出し終了する。
  • バックアップファイルの再オープン（fopen）に失敗した場合、err を呼び出し終了する。
  • 出力ファイルのオープン（fopen）に失敗した場合、作成したバックアップファイルを unlink で削除し、err を呼び出し終了する。

保守担当者向けメモ

  • 本関数はファイルシステムへの破壊的な操作を含むため、エラー発生時のクリーンアップ処理（unlink）が適切に行われているか注意すること。
  • 0600 というパーミッションは、バックアップファイルが所有者のみ読み書き可能であることを保証する。
