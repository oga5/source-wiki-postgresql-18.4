InitResultRelInfo

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execMain.c:1246

目的

クエリ実行時に更新対象となるリレーションの管理構造体 ResultRelInfo を初期化し、実行に必要なトリガー情報や外部テーブル（FDW）ルーチンを設定する。

シグネチャまたは形状

void InitResultRelInfo(ResultRelInfo *resultRelInfo,
                       Relation resultRelationDesc,
                       Index resultRelationIndex,
                       ResultRelInfo *partition_root_rri,
                       int instrument_options)

入力と出力

  • **入力**:
  • resultRelInfo: 初期化対象の構造体ポインタ。
  • resultRelationDesc: 対象リレーションのディスクリプタ。
  • resultRelationIndex: レンジテーブル内のインデックス。
  • partition_root_rri: パーティションルートの ResultRelInfo（ルートでない場合は NULL）。
  • instrument_options: 実行統計計測のオプション。
  • **出力**: なし（resultRelInfo を直接書き換える）。

前提条件と不変条件

  • resultRelInfo は呼び出し元でメモリ確保されていること。
  • resultRelationDesc はオープンされていること。

アルゴリズムまたは振る舞い

1. MemSet で resultRelInfo をゼロクリアし、基本情報を設定する。 2. IsInplaceUpdateRelation に基づき ri_needLockTagTuple を設定する。 3. CopyTriggerDesc を使用してトリガー情報をコピーし、必要に応じて FmgrInfo や ExprState 用のメモリを割り当てる。 4. RELKIND_FOREIGN_TABLE の場合、GetFdwRoutineForRelation を呼び出し FDW ルーチンを取得する。 5. 実行時に必要となる各種スロットや式評価用フィールドを NULL または初期値で初期化する。 6. パーティション関連のルート情報（partition_root_rri）を格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c
  • src/backend/executor/execMain.c
  • src/backend/executor/execPartition.c
  • src/backend/executor/execUtils.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • MemSet, IsInplaceUpdateRelation, CopyTriggerDesc, palloc0, InstrAlloc, GetFdwRoutineForRelation

関連データ構造

  • ResultRelInfo: 更新対象リレーションの実行時状態を保持する主要構造体。
  • Relation: リレーションのキャッシュ情報。
  • FmgrInfo, ExprState: トリガー処理に使用される関数情報および式評価状態。

失敗モードと境界条件

  • メモリ不足時に palloc0 がエラーを発生させる可能性がある。
  • resultRelationDesc が無効な場合、CopyTriggerDesc や GetFdwRoutineForRelation で予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は ResultRelInfo の初期化のみを行い、実行に必要な一部のフィールド（ri_projectNew 等）は後続の処理で設定されるため、初期化直後の状態が「未設定」であることを前提とした設計になっている。
  • パーティション関連のフィールド（ri_RootResultRelInfo 等）の扱いは、パーティションルーティングのロジックと密接に関連している。
