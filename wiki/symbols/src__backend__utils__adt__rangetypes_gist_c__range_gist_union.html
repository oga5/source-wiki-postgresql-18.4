range_gist_union

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_gist.c:323

目的

GiSTインデックスにおいて、複数の範囲型（RangeType）を包含する最小の範囲を計算し、インデックスのノードを統合します。

シグネチャまたは形状

Datum range_gist_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GistEntryVector を受け取ります。これには統合対象となる複数の GISTENTRY が含まれます。
  • **出力**: 統合された結果を表す RangeType を Datum として返します。

前提条件と不変条件

  • 入力される GistEntryVector には少なくとも1つ以上のエントリが存在する必要があります。
  • 統合されるすべての範囲型は、同一の型キャッシュ（TypeCacheEntry）に関連付けられた同一の範囲型OIDを持つ必要があります。

アルゴリズムまたは振る舞い

1. entryvec の最初の要素から初期範囲を取得します。 2. range_get_typcache を使用して、対象範囲型の型キャッシュ情報を取得します。 3. 残りのすべてのエントリに対してループ処理を行い、range_super_union を呼び出して順次範囲を統合します。 4. range_super_union は、空範囲のフラグ管理を含め、GiSTインデックスの整合性を維持しながら範囲を結合します。 5. 最終的に統合された RangeType を PG_RETURN_RANGE_P マクロで返します。

呼び出し元と呼び出し先

  • **呼び出し元**: GiSTインデックスの構築および更新プロセス（PostgreSQLのGiSTコアエンジン）から呼び出されます。
  • **呼び出し先**:
  • range_get_typcache: 型情報の取得。
  • range_super_union: 範囲の結合処理。
  • DatumGetRangeTypeP / PG_RETURN_RANGE_P: 範囲型データの変換。

関連データ構造

  • GistEntryVector: 統合対象のインデックスエントリを保持する構造体。
  • GISTENTRY: 個別のインデックスエントリ。
  • RangeType: 範囲型の内部表現。
  • TypeCacheEntry: 範囲型のメタデータキャッシュ。

失敗モードと境界条件

  • 入力ベクトルが空の場合、entryvec->vector へのアクセスで未定義の動作を引き起こす可能性があります（GiSTの仕様上、通常は発生しません）。
  • 範囲型のOIDが不一致の場合、range_get_typcache または range_super_union 内でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は range_super_union に依存しており、空範囲のフラグ伝播ロジックはそちらで管理されています。
  • GiSTインデックスの整合性を保つため、範囲の結合時に空範囲のフラグが正しく処理されているかを確認してください。
