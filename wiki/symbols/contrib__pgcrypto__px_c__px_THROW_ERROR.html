px_THROW_ERROR

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/px.c:93

目的

pgcrypto モジュール内で発生した内部エラーコードを、PostgreSQL のトランザクション制御と統合された ereport メカニズムへ変換してスローします。

シグネチャまたは形状

void px_THROW_ERROR(int err);

入力と出力

  • **入力**: int err（px.h で定義された PXE_* エラーコード）。
  • **出力**: なし（PostgreSQL の ereport(ERROR, ...) を呼び出すため、呼び出し元には戻りません）。

前提条件と不変条件

  • 入力された err は、px_strerror が解釈可能な有効なエラーコードである必要があります。
  • 本関数は pg_noreturn として宣言されており、正常な制御フローへの復帰は想定されていません。

アルゴリズムまたは振る舞い

1. 入力された err が PXE_NO_RANDOM と一致するか判定します。 2. PXE_NO_RANDOM の場合、ERRCODE_INTERNAL_ERROR を指定して「乱数生成失敗」を示すエラーをスローします。 3. それ以外のエラーの場合、px_strerror(err) を呼び出してエラーコードに対応するメッセージを取得し、ERRCODE_EXTERNAL_ROUTINE_INVOCATION_EXCEPTION を指定してスローします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgcrypto.c:472
  • contrib/pgcrypto/pgp-pgsql.c (複数箇所: 363, 438, 525, 895, 949, 998行目)
  • **呼び出し先**:
  • px_strerror(int): エラーメッセージの取得に使用。
  • ereport(...): PostgreSQL のエラー報告マクロ。

関連データ構造

  • PXE_* エラーコード定数（px.h で定義）。

失敗モードと境界条件

  • 未定義のエラーコードが渡された場合、px_strerror の実装に依存したメッセージがスローされます。

保守担当者向けメモ

  • 本関数は PostgreSQL の ERROR レベルで ereport を発行するため、呼び出し元でリソースのクリーンアップが必要な場合は、本関数を呼び出す前に実行してください。
