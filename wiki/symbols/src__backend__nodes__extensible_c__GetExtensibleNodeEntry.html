GetExtensibleNodeEntry

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/extensible.c:99

目的

指定されたハッシュテーブルから、ノード名（extnodename）に対応する拡張ノードのメソッド構造体を取得します。

シグネチャまたは形状

static const void *GetExtensibleNodeEntry(HTAB *htable, const char *extnodename, bool missing_ok)

入力と出力

  • **入力**:
  • htable: 検索対象のハッシュテーブル（HTAB *）。
  • extnodename: 検索するノードの名前（const char *）。
  • missing_ok: 見つからない場合にエラーとせず NULL を返すかどうかのフラグ（bool）。
  • **出力**:
  • 成功時: 対応するメソッド構造体へのポインタ（const void *）。
  • 失敗時: missing_ok が true の場合は NULL。

前提条件と不変条件

  • htable が NULL の場合、検索は行われず entry は NULL となります。
  • extnodename は EXTNODENAME_MAX_LEN（64バイト）以内の文字列である必要があります。

アルゴリズムまたは振る舞い

1. htable が有効な場合、hash_search を使用して extnodename をキーに HASH_FIND モードで検索を実行します。 2. 検索結果が見つからない場合:

  • missing_ok が true ならば NULL を返します。
  • missing_ok が false ならば ereport(ERROR, ...) を呼び出し、処理を中断します。

    • 3. 検索結果が見つかった場合、そのエントリが保持する extnodemethods メンバを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/extensible.c 内の GetExtensibleNodeMethods および GetCustomScanMethods から呼び出されます。
  • **呼び出し先**:
  • hash_search（utils/hsearch.h）
  • ereport（postgres.h）

関連データ構造

  • ExtensibleNodeEntry: ハッシュテーブルに格納されるエントリ構造体。
  • HTAB: utils/hsearch.h で定義されるハッシュテーブル構造体。

失敗モードと境界条件

  • **失敗モード**:
  • 指定された extnodename がハッシュテーブルに存在せず、かつ missing_ok が false の場合、ERRCODE_UNDEFINED_OBJECT エラーが発生します。
  • **境界条件**:
  • htable が NULL の場合、検索は行われず、missing_ok の値に応じて NULL を返すかエラーとなります。

保守担当者向けメモ

  • 本関数は static であり、モジュール内部でのみ使用されます。
  • 拡張機能がノードを登録する際は、RegisterExtensibleNodeMethods を経由してハッシュテーブルが初期化されている必要があります。
