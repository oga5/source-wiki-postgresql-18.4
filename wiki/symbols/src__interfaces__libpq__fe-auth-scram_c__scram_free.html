scram_free

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth-scram.c:181

目的

SCRAM 認証セッションで使用された fe_scram_state 構造体およびその内部で動的に確保されたメモリ領域を解放する。

シグネチャまたは形状

void scram_free(void *opaq)

入力と出力

  • **入力**: opaq - fe_scram_state 構造体へのポインタ（void * 型）。
  • **出力**: なし。

前提条件と不変条件

  • opaq は fe_scram_state 型の有効なポインタであること。
  • 内部の各メンバ（password, sasl_mechanism, client_nonce 等）は、malloc 等で確保されているか、あるいは NULL である必要がある（free は NULL を許容するため）。

アルゴリズムまたは振る舞い

1. 引数 opaq を fe_scram_state * にキャストする。 2. 構造体内の以下のメンバに対して free() を実行する：

  • パスワードおよび SASL メカニズム文字列。
  • クライアント側で生成したメッセージ関連のバッファ（client_nonce, client_first_message_bare, client_final_message_without_proof）。
  • サーバーから受信したメッセージ関連のバッファ（server_first_message, salt, nonce, server_final_message）。

    • 3. 最後に state 構造体自体を free() する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し関係は提供されていない。
  • **呼び出し先**: 標準ライブラリの free()。

関連データ構造

  • fe_scram_state: SCRAM 認証の各ステップで保持される状態を管理する構造体。

失敗モードと境界条件

  • opaq が NULL の場合、free(state) でセグメンテーション違反が発生する可能性がある（実装上、呼び出し元が非 NULL を保証する必要がある）。

保守担当者向けメモ

  • 本関数は fe_scram_state の全メンバを網羅的に解放する設計となっている。構造体に新しいメンバを追加した場合は、必ず本関数で free を追加すること。
