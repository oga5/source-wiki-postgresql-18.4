pairingheap_node

定義

  • 種別: struct
  • 定義箇所: src/include/lib/pairingheap.h:30

目的

ペアリングヒープ（自己調整型優先度付きキュー）において、ヒープに格納される各要素を管理するための侵入型（Intrusive）ノード構造体です。

シグネチャまたは形状

typedef struct pairingheap_node
{
    struct pairingheap_node *first_child;
    struct pairingheap_node *next_sibling;
    struct pairingheap_node *prev_or_parent;
} pairingheap_node;

入力と出力

本構造体はデータ構造の構成要素であり、関数ではありません。ヒープ操作（挿入、削除、マージ）の過程で、ノード間のポインタ関係が更新されます。

前提条件と不変条件

  • **侵入型設計**: 管理対象のデータ構造に本構造体をメンバとして埋め込む必要があります。
  • **ポインタ関係**: first_child、next_sibling、prev_or_parent により、ノード間で双方向リストおよび親子関係を形成します。
  • **メモリ管理**: ヒープ構造体自体を解放しても、ノードのメモリは自動的に解放されません。ノードのライフサイクルは呼び出し側が管理する必要があります。

アルゴリズムまたは振る舞い

  • **マージ**: first_child や next_sibling を操作して、二つのヒープ（またはノード）を結合します。
  • **削除**: ルートノードが削除される際、first_child が指す子ノード群が再帰的にマージされ、新しいルートが構築されます。
  • **比較**: 優先度の判定には、pairingheap 構造体に登録された比較関数（pairingheap_comparator）が使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照例）**:
  • src/backend/access/gist/gistscan.c
  • src/backend/access/spgist/spgscan.c
  • src/backend/executor/nodeIndexscan.c
  • **呼び出し先**:
  • src/backend/lib/pairingheap.c 内の merge および merge_children 関数が、本構造体のポインタを操作してヒープの再構成を行います。

関連データ構造

  • pairingheap: ヒープ全体を管理する構造体。
  • pairingheap_container: pairingheap_node のポインタから、それを含む親構造体を取得するための型安全なマクロ。

失敗モードと境界条件

  • **空のヒープ**: ルートが NULL の場合、ノード操作は行われません。
  • **単一ノード**: 子を持たないノードは、first_child が NULL となります。

保守担当者向けメモ

  • 本構造体は侵入型であるため、利用側で構造体のオフセットを計算する pairingheap_container マクロの利用が必須です。
  • デバッグが必要な場合は、PAIRINGHEAP_DEBUG を定義することで pairingheap_dump を活用できます。
