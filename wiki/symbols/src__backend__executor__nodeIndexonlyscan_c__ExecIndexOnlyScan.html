ExecIndexOnlyScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexonlyscan.c:336

目的

インデックスのみスキャン（Index-Only Scan）を実行し、テーブルのヒープページにアクセスすることなくインデックスデータからクエリ結果を生成する。

シグネチャまたは形状

TupleTableSlot *ExecIndexOnlyScan(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（実行中のインデックスのみスキャンノードの状態）。
  • **出力**: TupleTableSlot *（スキャンによって取得されたタプル、またはスキャン終了時は空のスロット）。

前提条件と不変条件

  • pstate は IndexOnlyScanState 型にキャスト可能であること。
  • ランタイムキーが存在し、かつ未設定（ioss_RuntimeKeysReady が偽）の場合は、実行前に ExecReScan が呼び出される。

アルゴリズムまたは振る舞い

1. **ランタイムキーの確認**: ノードにランタイムキーが含まれており、かつ準備が完了していない場合、ExecReScan を呼び出してキーの再計算とリセットを行う。 2. **スキャンの実行**: ExecScan を呼び出し、以下のメソッドを介してタプルを取得する。

  • アクセスメソッド: IndexOnlyNext（インデックスからのタプル取得と可視性判定）。
  • 再チェックメソッド: IndexOnlyRecheck（インデックススキャンの再チェック）。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン内のノード処理フロー（ExecProcNode 経由で呼び出される）。
  • **呼び出し先**:
  • ExecReScan: ランタイムキーの初期化が必要な場合に呼び出される。
  • ExecScan: スキャンのメインループを実行するために呼び出される。
  • IndexOnlyNext / IndexOnlyRecheck: ExecScan の引数として渡される。

関連データ構造

  • IndexOnlyScanState: インデックスのみスキャンの実行状態を保持する構造体。
  • PlanState: 実行プランのノード状態。
  • TupleTableSlot: 取得したタプルを格納するスロット。

失敗モードと境界条件

  • **EvalPlanQual (EPQ)**: IndexOnlyRecheck は EPQ をサポートしておらず、呼び出された場合はエラーを投げる実装となっている。
  • **可視性判定**: 可視性マップの判定はロックを伴わないため、メモリバリアとインデックスのロック順序に依存する。

保守担当者向けメモ

  • 本関数は ExecProcNode に登録され、実行エンジンのメインループから呼び出される。
  • インデックスのみスキャンはヒープアクセスを回避する設計であるため、可視性マップの更新やインデックスの整合性に注意が必要である。
