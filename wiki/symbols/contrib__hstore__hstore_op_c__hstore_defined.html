hstore_defined

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_op.c:229

目的

指定された hstore 内に特定のキーが存在し、かつその値が NULL ではないかを判定する。

シグネチャまたは形状

Datum hstore_defined(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_HSTORE_P(0): 検索対象の HStore 構造体。
  • PG_GETARG_TEXT_PP(1): 検索するキーを表す text 型データ。
  • **出力**:
  • PG_RETURN_BOOL(res): キーが存在し、かつ値が非 NULL であれば true、それ以外は false を返す。

前提条件と不変条件

  • 入力された HStore は内部的にキーでソートされている必要がある（hstoreFindKey の前提条件）。
  • hstoreFindKey を使用してバイナリサーチを行うため、キーの検索効率は $O(\log N)$ である。

アルゴリズムまたは振る舞い

1. PG_GETARG_HSTORE_P および PG_GETARG_TEXT_PP を介して引数を取得する。 2. hstoreFindKey を呼び出し、対象キーのインデックスを検索する。 3. hstoreFindKey が返すインデックスが 0 以上（キーが見つかった）であり、かつ HSTORE_VALISNULL マクロによりその値が NULL ではないことを確認する。 4. 判定結果をブール値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_op.c 内で PG_FUNCTION_INFO_V1 および HSTORE_POLLUTE マクロにより登録されている。
  • **呼び出し先**:
  • hstoreFindKey (検索実行)
  • ARRPTR (内部データアクセス)
  • VARDATA_ANY / VARSIZE_ANY_EXHDR (入力テキストの解析)
  • HSTORE_VALISNULL (NULL 判定)

関連データ構造

  • HStore: 操作対象のデータ構造。
  • HEntry: hstore 内の各要素のメタデータ（キーの長さや値の有無など）を保持する構造体。

失敗モードと境界条件

  • キーが存在しない場合、hstoreFindKey は負の値を返し、結果は false となる。
  • キーは存在するが値が NULL である場合、HSTORE_VALISNULL が真となり、結果は false となる。

保守担当者向けメモ

  • 本関数は hstore の defined 演算子を実装している。
  • hstoreFindKey の第2引数に NULL を渡しており、検索範囲のキャッシュ（lowbound）は利用していない。
