PageGetFreeSpace

定義

  • 種別: function
  • 定義箇所: src/backend/storage/page/bufpage.c:905

目的

PostgreSQLのページ内において、新しくタプルを挿入するために利用可能な空き領域のサイズ（バイト単位）を算出します。

シグネチャまたは形状

Size PageGetFreeSpace(const PageData *page)

入力と出力

  • **入力**: page - 調査対象のページへのポインタ。
  • **出力**: Size - ページ内で利用可能な空き領域のバイト数。

前提条件と不変条件

  • 入力された page は PageHeaderData としてキャスト可能である必要があります。
  • ページヘッダー内の pd_lower（アイテムID配列の末尾）と pd_upper（タプルデータの先頭）の値を基に計算されます。

アルゴリズムまたは振る舞い

1. ページヘッダーの pd_upper から pd_lower を減算し、現在の物理的な空き領域を算出します。 2. 算出された値が ItemIdData（アイテムIDのサイズ）未満である場合は、実質的な空きがないと見なし 0 を返します。 3. それ以外の場合、新しいアイテムIDを格納するための領域（sizeof(ItemIdData)）を差し引いた残りの値を、利用可能な空き領域として返します。

関連データ構造

  • PageHeaderData: ページヘッダー構造体。pd_lower および pd_upper を保持します。
  • ItemIdData: ページ内のアイテムID配列の各要素サイズ。

失敗モードと境界条件

  • pd_lower > pd_upper のような異常な状態でも、符号付き演算を用いることで論理的に破綻しない値を返します。
  • 空き領域が ItemIdData のサイズに満たない場合、戻り値は 0 となります。

保守担当者向けメモ

  • 本関数は「概算」ではなく、現在のヘッダー状態に基づいた計算を行います。
  • ページ内の断片化（デフラグ）は考慮されません。断片化を含めた正確な空き容量が必要な場合は、PageRepairFragmentation 等による整理後に評価する必要があります。
