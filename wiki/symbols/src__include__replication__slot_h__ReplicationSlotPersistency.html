ReplicationSlotPersistency

定義

  • 種別: enum
  • 定義箇所: src/include/replication/slot.h:36

目的

レプリケーションスロットの永続性レベルを定義し、スロットがサーバーの再起動やセッション終了後にどのように扱われるかを規定します。

シグネチャまたは形状

typedef enum ReplicationSlotPersistency
{
    RS_PERSISTENT,
    RS_EPHEMERAL,
    RS_TEMPORARY,
} ReplicationSlotPersistency;

入力と出力

本シンボルは列挙型であり、関数ではありません。スロットの状態管理において、スロットの永続性属性を示す値として使用されます。

前提条件と不変条件

  • RS_PERSISTENT: サーバー再起動後も保持される永続的なスロット。
  • RS_EPHEMERAL: サーバー再起動時には保持されないが、明示的に削除されるまで存続するスロット。
  • RS_TEMPORARY: セッション終了時やエラー発生時に自動的に削除される一時的なスロット。

アルゴリズムまたは振る舞い

レプリケーションスロットの作成および管理プロセスにおいて、この列挙値に基づいてスロットのライフサイクルが制御されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/replication/slot.c: スロット作成および管理ロジック内で使用。
  • src/include/replication/slot.h: ReplicationSlot 構造体内の persistency フィールドの型として使用。
  • 呼び出し先: 本シンボルは列挙型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • ReplicationSlot: スロットのメモリ内表現。この構造体内の persistency フィールドで本型が使用されます。
  • ReplicationSlotPersistentData: ディスクに書き込まれるスロットの属性データ。

失敗モードと境界条件

  • 不適切な永続性レベルが設定された場合、サーバー再起動時にスロットが意図せず消失したり、逆に不要なスロットが残り続けたりする可能性があります。

保守担当者向けメモ

  • 新しい永続性レベルを追加する場合は、src/backend/replication/slot.c 内の関連する判定ロジック（スロットの削除やリカバリ処理）をすべて更新する必要があります。
