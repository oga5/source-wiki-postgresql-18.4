WalSndShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walsender.c:3680

目的

WAL 送信プロセス（WalSender）が使用する共有メモリ領域の割り当ておよび初期化を行う。

シグネチャまたは形状

void WalSndShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上の WalSndCtl が初期化される）

前提条件と不変条件

  • PostgreSQL の共有メモリ初期化フェーズにおいて呼び出される必要がある。
  • WalSndShmemSize() が返すサイズ分の共有メモリが確保可能であること。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を使用し、"Wal Sender Ctl" という名称で共有メモリ領域を確保する。 2. 既に領域が初期化済み（found が true）であれば処理を終了する。 3. 初回実行時のみ以下の初期化を行う：

  • WalSndCtl 構造体全体をゼロクリアする。
  • 同期レプリケーション用の待機キュー（SyncRepQueue）を初期化する。
  • max_wal_senders 分の各 WalSnd スロットに対してスピンロック（mutex）を初期化する。
  • WAL のフラッシュ、リプレイ、確認応答を待機するための条件変数（wal_flush_cv, wal_replay_cv, wal_confirm_rcv_cv）を初期化する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c 内の共有メモリ初期化処理から呼び出される。
  • 呼び出し先: ShmemInitStruct, MemSet, dlist_init, SpinLockInit, ConditionVariableInit 等の共有メモリおよび同期プリミティブ管理関数。

関連データ構造

  • WalSndCtlData: WAL 送信プロセス全体を管理する共有メモリ上の制御構造体。
  • WalSnd: 個別の WAL 送信スロット。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、ShmemInitStruct の内部実装によりプロセスが異常終了する可能性がある。
  • max_wal_senders の値が適切でない場合、共有メモリのサイズ計算やスロットの初期化範囲に影響を与える。

保守担当者向けメモ

  • 本関数はサーバー起動時の共有メモリ初期化プロセスの一部である。
  • WalSndCtl 構造体のレイアウトや WalSndShmemSize の計算ロジックを変更する場合は、本関数の初期化手順と整合性が取れているか確認すること。
