pgwin32_putenv

定義

  • 種別: function
  • 定義箇所: src/port/win32env.c:26

目的

Windows 環境において、プロセス全体の環境変数更新と、ロードされている複数の C ランタイムライブラリ（CRT）間での環境変数状態の同期を行う。

シグネチャまたは形状

int pgwin32_putenv(const char *envval);

入力と出力

  • **入力**: envval (環境変数設定文字列、形式は NAME=VALUE)
  • **出力**: 成功時に 0、失敗時に -1 を返す。

前提条件と不変条件

  • 引数 envval は strdup で複製して処理されるため、呼び出し元は元の文字列のメモリ管理を継続する。
  • modulenames 配列に定義された CRT モジュールのみが同期対象となる。
  • SetEnvironmentVariable は環境変数の追加時のみ呼び出される。

アルゴリズムまたは振る舞い

1. envval を複製し、= を境界として名前と値に分割する。 2. SetEnvironmentVariable を呼び出し、OS レベルで環境変数を更新する。 3. modulenames に列挙された既知の CRT モジュールを GetModuleHandleEx で探索する。 4. 発見された各モジュールから GetProcAddress で _putenv を取得し、値を同期させる。 5. 最後に、現在のプロセスがリンクしている CRT の _putenv を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/win32_port.h にて putenv マクロとして定義されており、PostgreSQL の各所から呼び出される可能性がある。
  • **呼び出し先**: SetEnvironmentVariable (Win32 API), GetModuleHandleEx (Win32 API), GetProcAddress (Win32 API), FreeLibrary (Win32 API), _putenv (CRT)。

関連データ構造

  • PUTENVPROC: CRT 内の _putenv を呼び出すための関数ポインタ型。
  • modulenames: 同期対象となる CRT モジュール名の静的配列。

失敗モードと境界条件

  • strdup によるメモリ確保失敗時に -1 を返す。
  • envval に = が含まれない場合、不正な形式として -1 を返す。
  • SetEnvironmentVariable が失敗した場合、-1 を返す。
  • modulenames に含まれない新しい CRT が導入された場合、その CRT の環境変数は同期されない。

保守担当者向けメモ

  • modulenames 配列は、将来的に新しい CRT が導入された際に更新が必要である。
  • SetEnvironmentVariable を環境変数の削除（値が空の場合）に呼び出すと一部の MinGW バージョンでクラッシュするため、追加時のみ呼び出すよう制御されている。
