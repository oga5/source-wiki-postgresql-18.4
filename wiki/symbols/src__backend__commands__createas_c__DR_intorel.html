DR_intorel

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/createas.c:51

目的

CREATE TABLE AS (CTAS) および CREATE MATERIALIZED VIEW コマンドにおいて、クエリ実行結果を新規作成したテーブルへ直接挿入するための DestReceiver 構造体です。

シグネチャまたは形状

typedef struct {
    DestReceiver pub;
    IntoClause *into;
    Relation   rel;
    ObjectAddress reladdr;
    CommandId  output_cid;
    int        ti_options;
    BulkInsertState bistate;
} DR_intorel;

入力と出力

  • **入力**: intorel_receive 等の関数を通じて受け取るタプルを、rel フィールドで指定されたリレーションへ挿入します。
  • **出力**: 挿入されたデータは物理テーブルとして永続化されます。

前提条件と不変条件

  • intorel_startup 関数によって、rel、reladdr、output_cid、ti_options、bistate が初期化されている必要があります。
  • intorel_shutdown を呼び出すことで、バルク挿入状態のクリーンアップとテーブルのクローズを確実に行う必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: CreateIntoRelDestReceiver により生成され、intorel_startup でターゲットテーブルの物理作成と挿入準備を行います。 2. **データ挿入**: intorel_receive が実行エンジンからタプルを受け取り、table_tuple_insert を用いてテーブルへ格納します。 3. **終了処理**: intorel_shutdown がバルク挿入の終了処理を行い、リソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecCreateTableAs が本構造体を作成・管理します。
  • **呼び出し先**: table_tuple_insert を呼び出し、テーブルアクセスメソッド層へデータを渡します。

関連データ構造

  • DestReceiver: 本構造体が継承するインターフェース。
  • IntoClause: CTAS のターゲット指定情報。
  • Relation: 挿入先となるテーブルリレーション。
  • BulkInsertState: バルク挿入の最適化状態を保持。

失敗モードと境界条件

  • 行レベルセキュリティ (RLS) が有効なテーブルに対しては、intorel_startup 内で明示的なエラーチェックが行われます。
  • WITH NO DATA オプション指定時は、プランナやエグゼキュータをバイパスするため、本構造体によるデータ挿入処理は実行されません。

保守担当者向けメモ

  • 本構造体は DestReceiver インターフェースの一部として実装されています。
  • intorel_receive 等の関数内で DR_intorel * へのキャストが行われており、各フィールドの整合性は intorel_startup の実装に依存しています。
