PQconnectdbParams

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-connect.c:764

目的

キーワードと値のペアを指定して、PostgreSQL データベースへの同期的な接続を開始し、確立する。

シグネチャまたは形状

PGconn *PQconnectdbParams(const char *const *keywords,
                          const char *const *values,
                          int expand_dbname);

入力と出力

  • **入力**:
  • keywords: 接続パラメータ名の配列（NULL 終端）。
  • values: 対応するパラメータ値の配列。
  • expand_dbname: データベース名に接続文字列が含まれる場合に展開を行うかどうかのフラグ。
  • **出力**:
  • 接続状態を保持する PGconn 構造体へのポインタ。接続失敗時も構造体は返される（status が CONNECTION_BAD となる）。

前提条件と不変条件

  • keywords および values 配列は、PQconnectStartParams が期待する形式に従う必要がある。
  • 戻り値の PGconn 構造体は、呼び出し元で PQfinish を使用して解放する必要がある。

アルゴリズムまたは振る舞い

1. PQconnectStartParams を呼び出し、接続プロセスを開始して PGconn オブジェクトを生成する。 2. 接続が正常に開始され、かつステータスが CONNECTION_BAD でない場合、pqConnectDBComplete を呼び出して接続が完了するまで待機する。 3. 確立された（あるいは失敗した）PGconn ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/oid2name/oid2name.c
  • contrib/postgres_fdw/connection.c

  • **呼び出し先**:
  • PQconnectStartParams
  • pqConnectDBComplete

関連データ構造

  • PGconn: 接続状態を管理する主要な構造体。

失敗モードと境界条件

  • 接続パラメータの解析失敗やネットワークエラーが発生した場合、PGconn 構造体が返されるが、その status フィールドは CONNECTION_BAD に設定される。
  • メモリ不足等の致命的なエラー時には NULL が返される可能性がある。

保守担当者向けメモ

  • 本関数は同期的な接続処理を行うラッパーである。非同期接続が必要な場合は PQconnectStartParams を直接使用すること。
  • pqConnectDBComplete はブロッキング待機を行うため、UI スレッド等からの呼び出しには注意が必要である。
