MemoizeHash_equal

定義

  • **種別**: function
  • **定義箇所**: src/backend/executor/nodeMemoize.c:220

目的

MemoizeState が管理するハッシュテーブルにおいて、2つの MemoizeKey が等価であるかを判定します。

シグネチャまたは形状

static bool MemoizeHash_equal(struct memoize_hash *tb, const MemoizeKey *key1, const MemoizeKey *key2)

入力と出力

  • **入力**:
  • tb: ハッシュテーブル構造体へのポインタ。
  • key1, key2: 比較対象の MemoizeKey 構造体。
  • **出力**:
  • bool: 両者が等価であれば true、そうでなければ false を返します。

前提条件と不変条件

  • tb->private_data は MemoizeState 構造体としてキャスト可能である必要があります。
  • mstate->probeslot は、本関数呼び出し前に prepare_probe_slot() 等によって適切に準備されている必要があります。

アルゴリズムまたは振る舞い

1. key1 のパラメータ情報を mstate->tableslot に格納します。 2. mstate->binary_mode が有効な場合:

  • econtext->ecxt_per_tuple_memory にメモリコンテキストを切り替えます。
  • tableslot と probeslot の全属性を取得し、各属性の NULL 状態および値をバイナリ比較（datum_image_eq）します。
  • 比較終了後、元のメモリコンテキストへ戻します。

    • 3. mstate->binary_mode が無効な場合:

  • econtext の内部タプルおよび外部タプルに各スロットをセットし、mstate->cache_eq_expr を評価して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: lib/simplehash.h で定義されたハッシュテーブル操作マクロ（SH_EQUAL）経由で呼び出されます。
  • **呼び出し先**: ExecStoreMinimalTuple, slot_getallattrs, TupleDescCompactAttr, datum_image_eq, MemoryContextSwitchTo, ExecQual。

関連データ構造

  • MemoizeState: ノードの実行状態とコンテキストを保持。
  • MemoizeKey: キャッシュのキー情報。
  • TupleTableSlot: 比較対象のタプルデータを保持。
  • ExprContext: 式評価のためのコンテキスト。

失敗モードと境界条件

  • binary_mode 時、属性数が一致しない場合やデータ型が不一致の場合の挙動については、本ソース内では明示的なエラーハンドリングは記述されていません。
  • メモリコンテキストの切り替え失敗や、式評価（ExecQual）中のエラーは、PostgreSQL の標準的なエラーメカニズムに依存します。

保守担当者向けメモ

  • 本関数は simplehash.h のハッシュテーブル実装と密結合しています。binary_mode の判定ロジックを変更する場合は、MemoizeState の初期化処理との整合性に注意してください。
