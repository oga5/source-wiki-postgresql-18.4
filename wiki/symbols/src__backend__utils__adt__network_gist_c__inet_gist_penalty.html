inet_gist_penalty

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/adt/network_gist.c:619

目的

GiSTインデックスの挿入時において、新しいキーを既存のインデックスノードに追加する際の「ペナルティ（コスト）」を計算します。このコストは、インデックスの木構造を最適に保つために使用されます。

シグネチャまたは形状

Datum inet_gist_penalty(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY *origent（既存のインデックスエントリ）
  • PG_GETARG_POINTER(1): GISTENTRY *newent（挿入対象の新しいエントリ）
  • PG_GETARG_POINTER(2): float *penalty（計算結果を格納するポインタ）
  • **出力**:
  • PG_RETURN_POINTER(penalty): 計算されたペナルティ値を格納したポインタを返します。

前提条件と不変条件

  • 入力される origent および newent のキーは GistInetKey 構造体として解釈可能である必要があります。
  • アドレスファミリー（IPv4/IPv6）が一致しない場合、ペナルティは最大値（4）となります。

アルゴリズムまたは振る舞い

1. **ファミリー判定**: gk_ip_family を比較し、一致しない場合はペナルティを 4 とします。 2. **包含関係の評価**:

  • 既存キーの最小ネットマスク長（minbits）が新しいキーのそれ以下である場合、共通ビット数（commonbits）を計算します。
  • 共通ビットが存在する場合、ペナルティは 1.0 / commonbits となり、共通部分が多いほどコストが低くなります。
  • 共通ビットがない場合はペナルティを 2 とします。

    • 3. **包含関係外**: 既存キーの minbits が新しいキーより大きい場合は、ペナルティを 3 とします。

呼び出し元と呼び出し先

  • **呼び出し元**: GiSTインデックスの挿入処理（gistinsert 等）から呼び出されます。
  • **呼び出し先**:
  • DatumGetInetKeyP: キーの型変換に使用。
  • gk_ip_family, gk_ip_minbits, gk_ip_addr, gk_ip_commonbits: GistInetKey の属性取得に使用。
  • bitncommon: 共通ビット数の計算に使用。

関連データ構造

  • GistInetKey: ネットワークアドレス、プレフィックス長、ネットマスク長を保持するインデックスキー構造体。
  • GISTENTRY: GiSTインデックスの各エントリを保持する構造体。

失敗モードと境界条件

  • GistInetKey として不正なデータが渡された場合の動作は未定義です。
  • 共通ビット数が 0 の場合、ペナルティは固定値 2 となります。

保守担当者向けメモ

  • 本関数は、インデックスのクラスタリング効率に直結します。ペナルティ値の重み付け（1, 2, 3, 4）を変更する場合は、インデックスの再構築が必要になる可能性があるため注意してください。
