heap_xlog_lock

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_xlog.c:996

目的

ヒープページ上のタプルに対するロック操作のWALリプレイを実行し、必要に応じて可視性マップ（Visibility Map）の更新を行う。

シグネチャまたは形状

static void heap_xlog_lock(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）。
  • **出力**: なし（副作用としてバッファおよび可視性マップを更新）。

前提条件と不変条件

  • record は xl_heap_lock 構造体を含む有効なWALレコードであること。
  • ページ上のタプルが offnum に存在し、かつ ItemIdIsNormal であること（そうでない場合は PANIC）。

アルゴリズムまたは振る舞い

1. **可視性マップの更新**: xlrec->flags に XLH_LOCK_ALL_FROZEN_CLEARED が設定されている場合、該当ブロックの可視性マップから VISIBILITYMAP_ALL_FROZEN ビットをクリアする。 2. **バッファの取得**: XLogReadBufferForRedo を呼び出し、ページが最新でない（BLK_NEEDS_REDO）場合にのみ以下の処理を行う。 3. **タプルヘッダーの更新**:

  • t_infomask からロック関連ビットおよび HEAP_MOVED をクリア。
  • t_infomask2 から HEAP_KEYS_UPDATED をクリア。
  • fix_infomask_from_infobits を用いて、WALレコード内の情報に基づきマスクを再設定。

    • 4. **ロック状態の反映**:

  • HEAP_XMAX_IS_LOCKED_ONLY が真の場合、HOT更新フラグをクリアし、t_ctid を自己参照（自身のブロックとオフセット）に設定。
  • xmax をレコードの値で更新し、cmax を FirstCommandId に設定。

    • 5. **コミット**: ページのLSNを更新し、バッファをダーティとしてマークする。 6. **終了処理**: バッファが有効であればロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: heap_redo（src/backend/access/heap/heapam_xlog.c 内のディスパッチャ）。
  • **呼び出し先**:
  • XLogRecGetBlockTag, XLogRecGetData（WAL解析）
  • CreateFakeRelcacheEntry, FreeFakeRelcacheEntry（リレーション管理）
  • visibilitymap_pin, visibilitymap_clear（可視性マップ操作）
  • XLogReadBufferForRedo, PageGetItemId, PageGetItem, PageSetLSN, MarkBufferDirty, UnlockReleaseBuffer（バッファ管理）
  • fix_infomask_from_infobits, HeapTupleHeaderClearHotUpdated, ItemPointerSet, HeapTupleHeaderSetXmax, HeapTupleHeaderSetCmax（タプル操作）

関連データ構造

  • xl_heap_lock: WALレコードのデータ構造。
  • XLogReaderState: WAL読み取り状態。
  • Buffer, Page, ItemId, HeapTupleHeader: ヒープページおよびタプルの物理構造。

失敗モードと境界条件

  • PageGetMaxOffsetNumber(page) < offnum または !ItemIdIsNormal(lp) の場合、PANIC を発生させてリカバリを停止する。

保守担当者向けメモ

  • 本関数は、ページが既に最新である場合でも、可視性マップの更新が必要な場合は処理を継続する設計となっている。
  • fix_infomask_from_infobits によるマスクの整合性維持が重要である。
