MdfdVec

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/smgr/md.c:81

目的

PostgreSQLの磁気ディスク（Magnetic Disk）ストレージマネージャにおいて、リレーションを構成する個々のセグメントファイルを管理するための構造体です。

シグネチャまたは形状

typedef struct _MdfdVec
{
	File		mdfd_vfd;		/* fd number in fd.c's pool */
	BlockNumber mdfd_segno;		/* segment number, from 0 */
} MdfdVec;

入力と出力

本構造体はデータ構造であり、関数ではありません。mdopenfork や _mdfd_getseg などの関数によって生成・操作され、ファイル記述子とセグメント番号のペアを保持します。

前提条件と不変条件

  • リレーションは RELSEG_SIZE ごとのセグメントに分割されます。
  • mdfd_vfd は storage/fd.c が管理する仮想ファイル記述子（VFD）プール内の有効なインデックスである必要があります。
  • mdfd_segno は 0 から始まるセグメント番号を示します。

アルゴリズムまたは振る舞い

  • MdCxt メモリコンテキスト上で配列として確保され、リレーションの各セグメントへのアクセスを仲介します。
  • _mdfd_getseg 等の内部関数により、要求されたブロック番号に対応するセグメントが特定され、必要に応じてファイルが開かれます。

呼び出し元と呼び出し先

  • **呼び出し元**: mdopenfork、_mdfd_openseg、_mdfd_getseg 等の md.c 内の関数が本構造体を生成・参照します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • MdCxt: MdfdVec オブジェクト全体を管理するためのメモリコンテキスト。
  • SMgrRelation: ストレージマネージャが管理するリレーションの抽象表現。

失敗モードと境界条件

  • mdfd_vfd が無効な場合、ファイル I/O 操作が失敗します。
  • セグメント番号の計算において、リレーションのサイズ制限を超えるアクセスが発生した場合の挙動は、呼び出し側のロジックに依存します。

保守担当者向けメモ

  • MdfdVec の配列は MdCxt 上で palloc されます。
  • セグメントの切り詰めや削除を行う際は、他のバックエンドが保持している可能性のあるファイル記述子との整合性に注意してください。
