r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_basque.c:906

目的

バスク語のステミング処理において、単語内の領域（RV, R1, R2）を特定し、SN_env 構造体の整数配列 I にその境界位置を格納する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z （ステミング環境へのポインタ）
  • **出力**: int （常に 1 を返す）
  • **副作用**: z->I[0] (R2), z->I[1] (R1), z->I[2] (RV) に境界位置（インデックス）を書き込む。

前提条件と不変条件

  • z は有効な SN_env 構造体を指していること。
  • g_v（母音グループ）が定義されていること。
  • 処理対象の文字列は ISO-8859-1 エンコーディングであること。

アルゴリズムまたは振る舞い

1. **初期化**: I[0], I[1], I[2] を単語の末尾位置（z->l）で初期化する。 2. **RVの特定**: 母音の出現パターンに基づき、単語内の RV 領域の開始位置を探索し z->I[2] に設定する。 3. **R1/R2の特定**: 母音と子音の遷移を追跡し、R1 領域の開始位置を z->I[1] に、R2 領域の開始位置を z->I[0] に設定する。 4. 探索は in_grouping および out_grouping を使用して、母音グループ g_v を基準に行われる。

呼び出し元と呼び出し先

  • **呼び出し元**: basque_ISO_8859_1_stem（src/backend/snowball/libstemmer/stem_ISO_8859_1_basque.c:1146）から呼び出される。
  • **呼び出し先**: in_grouping, out_grouping（Snowballランタイム関数）。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。I 配列（整数値）および c（現在のカーソル位置）、l（文字列長）を使用。

失敗モードと境界条件

  • 文字列が空、または母音が含まれない場合、領域は単語の末尾に設定される。
  • in_grouping / out_grouping の戻り値が負の場合、探索は中断され、現在の領域設定が維持される。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードであるため、直接編集しないこと。
  • 領域定義のロジックを変更する場合は、対応する .sbl ソースファイルを修正する必要がある。
  • g_v は母音を定義するグループであり、バスク語のステミング境界判定の基盤となる。
