IsThereOpClassInNamespace

定義

  • 種別: function
  • 定義箇所: src/backend/commands/opclasscmds.c:1829

目的

指定された名前空間（スキーマ）内において、特定のアクセスメソッドに対する演算子クラスが既に存在するかを確認し、重複を防止する。

シグネチャまたは形状

void IsThereOpClassInNamespace(const char *opcname, Oid opcmethod, Oid opcnamespace)

入力と出力

  • opcname: 演算子クラスの名前（文字列）。
  • opcmethod: アクセスメソッドのOID。
  • opcnamespace: 演算子クラスが属する名前空間のOID。
  • 出力: なし。重複が存在する場合はエラーをスローする。

前提条件と不変条件

  • SearchSysCacheExists3 を使用して CLAAMNAMENSP システムキャッシュを検索する。
  • 検索キーとして opcmethod、opcname、opcnamespace の組み合わせを使用する。

アルゴリズムまたは振る舞い

1. CLAAMNAMENSP システムキャッシュを検索し、指定された名前、アクセスメソッド、名前空間の組み合わせを持つ演算子クラスが存在するか確認する。 2. 存在する場合、ereport を呼び出し、ERRCODE_DUPLICATE_OBJECT エラーを発生させて処理を中断する。 3. エラーメッセージには、演算子クラス名、アクセスメソッド名（get_am_name で取得）、スキーマ名（get_namespace_name で取得）が含まれる。

呼び出し元と呼び出し先

  • src/backend/commands/alter.c:283

  • 呼び出し先:

関連データ構造

  • pg_opclass システムカタログ（CLAAMNAMENSP キャッシュの対象）。

失敗モードと境界条件

  • 指定された演算子クラスが既に存在する場合、ERROR レベルで処理が停止する。
  • 内部的に get_am_name や get_namespace_name が失敗した場合の挙動は、それら関数の実装に依存する。

保守担当者向けメモ

  • 本関数は主に DDL 実行時の名前重複チェックに使用される。
  • CLAAMNAMENSP キャッシュの定義変更や、演算子クラスの検索キーの仕様変更があった場合は本関数のロジックも追従する必要がある。
