tuplestore_clear

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplestore.c:429

目的

Tuplestorestate が保持するすべてのタプルを破棄し、状態を初期化して再利用可能な状態に戻す。

シグネチャまたは形状

void tuplestore_clear(Tuplestorestate *state)

入力と出力

  • **入力**: state (Tuplestorestate *) - クリア対象のタプルストア状態構造体。
  • **出力**: なし。

前提条件と不変条件

  • state は有効な Tuplestorestate 構造体を指している必要がある。
  • 実行後、メモリ使用量追跡 (availMem) は allowedMem にリセットされ、memtuples 配列が占有するメモリ量のみが再確保される。

アルゴリズムまたは振る舞い

1. tuplestore_updatemax(state) を呼び出し、最大メモリ使用量を更新する。 2. 一時ファイル (myfile) が存在する場合は BufFileClose でクローズし、ポインタを NULL にする。 3. MemoryContextReset(state->context) を実行し、メモリ内の全タプルを解放する。 4. availMem を allowedMem に戻し、memtuples 配列のメモリ消費分を USEMEM マクロで再計上する。 5. 状態を TSS_INMEM に設定し、カウンタ類（memtupdeleted, memtupcount, tuples）を 0 にリセットする。 6. すべての読み取りポインタ (readptrs) について、eof_reached を false に、current を 0 にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeCtescan.c
  • src/backend/executor/nodeRecursiveunion.c
  • src/backend/executor/nodeWindowAgg.c
  • **呼び出し先**:
  • tuplestore_updatemax
  • BufFileClose
  • MemoryContextReset
  • GetMemoryChunkSpace
  • USEMEM (マクロ)

関連データ構造

  • Tuplestorestate: タプルストアのメイン状態構造体。
  • TSReadPointer: 読み取り位置を管理する構造体。

失敗モードと境界条件

  • state が NULL の場合の挙動は定義されていない（セグメンテーション違反の可能性がある）。
  • メモリコンテキストが既に破棄されている場合、MemoryContextReset は失敗する。

保守担当者向けメモ

  • 本関数はメモリコンテキストをリセットするため、この関数を呼び出した後に以前のタプルへのポインタを参照することはできない。
  • USE_ASSERT_CHECKING が有効な場合、メモリ追跡の整合性が検証される。
