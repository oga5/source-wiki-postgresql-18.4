print_separator

定義

  • 種別: function
  • 定義箇所: src/fe_utils/print.c:378

目的

指定されたセパレータ設定に基づき、出力ストリームへ区切り文字を出力します。

シグネチャまたは形状

static void print_separator(struct separator sep, FILE *fout)

入力と出力

  • **入力**:
  • sep: 区切り文字の情報を保持する struct separator 型の構造体。
  • fout: 出力先の FILE ポインタ。
  • **出力**: なし（戻り値は void）。指定されたストリームへ直接書き込みを行います。

前提条件と不変条件

  • fout は書き込み可能な有効なファイルポインタである必要があります。
  • sep.separator_zero が真の場合、sep.separator の値に関わらずヌルバイト (\0) が優先して出力されます。

アルゴリズムまたは振る舞い

1. sep.separator_zero が真であれば、ヌルバイト (\000) を fout に書き込みます。 2. そうでなく、sep.separator が非 NULL であれば、その文字列を fout に書き込みます。 3. いずれの条件も満たさない場合、何も出力しません。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/print.c 内の複数の箇所から呼び出されています（例: printTable 関連の出力処理において、レコード区切りやフィールド区切りの出力に使用）。
  • **呼び出し先**: 標準ライブラリの fputc および fputs を呼び出します。

関連データ構造

  • struct separator: 区切り文字の定義を保持する構造体（separator_zero フラグと separator 文字列ポインタを含む）。

失敗モードと境界条件

  • fout が無効な場合、標準ライブラリの fputc/fputs の挙動に従いエラーが発生します。
  • sep.separator が NULL であり、かつ sep.separator_zero が偽の場合、何も出力されず正常終了します。

保守担当者向けメモ

  • 本関数は static であり、src/fe_utils/print.c 内でのみ使用されます。
  • psql 等の出力フォーマット（CSVやアラインメント形式など）において、レコードやフィールドの区切りを統一的に扱うためのヘルパー関数です。変更時は、各フォーマットの出力仕様に影響がないか確認してください。
