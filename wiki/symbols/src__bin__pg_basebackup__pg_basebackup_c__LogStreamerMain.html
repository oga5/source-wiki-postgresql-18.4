LogStreamerMain

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_basebackup.c:544

目的

WAL（Write-Ahead Log）ストリーミングのバックグラウンド処理を実行し、サーバーから受信した WAL データを指定されたメソッド（ディレクトリまたは tar）で保存する。

シグネチャまたは形状

static int LogStreamerMain(logstreamer_param *param)

入力と出力

  • **入力**: logstreamer_param *param（WAL ストリーミングに必要な開始位置、タイムライン、システム識別子、接続情報、圧縮設定を含む構造体）。
  • **出力**: 成功時は 0、失敗時は 1 を返す。

前提条件と不変条件

  • in_log_streamer グローバル変数が true に設定される。
  • param->bgconn は有効なデータベース接続である必要がある。
  • format グローバル変数に基づき、WAL の書き込みメソッドが適切に初期化される。

アルゴリズムまたは振る舞い

1. StreamCtl 構造体を初期化し、param からストリーミングパラメータを設定する。 2. format に応じて CreateWalDirectoryMethod または CreateWalTarMethod を呼び出し、WAL 書き込みメソッドを構築する。 3. ReceiveXlogStream を呼び出し、サーバーから WAL を受信する。 4. 受信完了後、walmethod->ops->finish を呼び出して書き込みを確定させる。 5. 接続を閉じ、walmethod->ops->free でリソースを解放する。 6. 失敗時には、Windows 環境であれば bgchild_exited フラグを立てて異常終了を通知する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/pg_basebackup.c 内のプロセスまたはスレッド生成箇所（_beginthreadex 等）。
  • **呼び出し先**: ReceiveXlogStream、CreateWalDirectoryMethod、CreateWalTarMethod、PQfinish。

関連データ構造

  • logstreamer_param: ストリーミングのパラメータを保持。
  • StreamCtl: receivelog.c で定義されるストリーミング制御構造体。
  • WalMethod: WAL 書き込み操作（finish, free 等）を抽象化したインターフェース。

失敗モードと境界条件

  • ReceiveXlogStream が失敗した場合、エラーを報告し 1 を返す。
  • WAL ファイルの書き込み終了処理（finish）が失敗した場合、エラーログを出力し 1 を返す。
  • Windows 環境では、異常終了時に bgchild_exited を設定してメインスレッドに通知する。

保守担当者向けメモ

  • 本関数は WAL ストリーミングのライフサイクルを管理する。stream.do_sync は false に設定されており、fsync は pg_basebackup の終了時にまとめて行われる設計である。
  • Windows と Unix で stop_socket の扱いが異なるため、プラットフォーム固有の終了シグナル処理に注意すること。
