dsa_free

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/dsa.c:825

目的

動的共有メモリ領域（DSA）から割り当てられたメモリブロックを解放し、管理プールまたは空きページマネージャへ返却する。

シグネチャまたは形状

void dsa_free(dsa_area *area, dsa_pointer dp)

入力と出力

  • area: 解放対象のメモリが属する dsa_area 構造体へのポインタ。
  • dp: 解放するメモリ領域を指す dsa_pointer。
  • 出力: なし（void）。

前提条件と不変条件

  • dp は有効な DSA ポインタであること。
  • check_for_freed_segments(area) により、解放済みのセグメントスロットが整理されていること。
  • CLOBBER_FREED_MEMORY が定義されている場合、解放されたメモリ領域は 0x7f で埋められる。
  • 大規模オブジェクト（DSA_SCLASS_SPAN_LARGE）の解放時は、DSA_AREA_LOCK および DSA_SCLASS_LOCK を排他モードで取得する。

アルゴリズムまたは振る舞い

1. **メタデータ特定**: dp からセグメント番号とオフセットを抽出し、対応する dsa_segment_map、dsa_area_span、および対象オブジェクトのアドレスを特定する。 2. **大規模オブジェクトの処理**: サイズクラスが DSA_SCLASS_SPAN_LARGE の場合、ページを FreePageManager に戻し、セグメントの再配置（rebin_segment）を行い、スパンをアンリンクして再帰的に dsa_free を呼び出す。 3. **小規模オブジェクトの処理**:

  • 対象スパンのフリーリストにオブジェクトを戻し、nallocatable をインクリメントする。
  • スパンの充填度（fullness class）に応じて、必要であれば unlink_span および add_span_to_fullness_class を実行してクラスを移動させる。
  • スパン全体が空になった場合（かつアクティブなスパンでない場合）、destroy_superblock を呼び出してメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c、src/backend/executor/nodeHash.c、src/backend/lib/dshash.c 等の各所から呼び出される。
  • **呼び出し先**: get_segment_by_index、dsa_get_address、FreePageManagerPut、rebin_segment、unlink_span、add_span_to_fullness_class、destroy_superblock、および再帰的に dsa_free。

関連データ構造

  • dsa_area: DSA 領域の管理構造体。
  • dsa_pointer: 共有メモリ内の相対位置を示すポインタ。
  • dsa_area_span: メモリブロックの集合（スパン）を管理する構造体。
  • dsa_segment_map: セグメント内のページマップ。

失敗モードと境界条件

  • dp が無効な場合や、既に解放済みの領域を二重に解放した場合の挙動は未定義（アサーションによるチェックは存在する）。
  • ロック取得順序（DSA_AREA_LOCK → DSA_SCLASS_LOCK）を遵守しない場合、デッドロックが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、大規模オブジェクトの解放時に自身を再帰呼び出しする設計となっている。
  • CLOBBER_FREED_MEMORY 定義時のメモリ破壊処理はデバッグ用であり、本番環境では無効化される。
  • スパンの充填度管理において、アクティブなスパンを即座に解放しないことで、頻繁な割り当て・解放によるヒステリシスを防止している。
