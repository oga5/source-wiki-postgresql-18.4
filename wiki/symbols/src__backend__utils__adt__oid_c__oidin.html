oidin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oid.c:36

目的

文字列形式の入力（C文字列）を、PostgreSQLの内部的な Oid（Object Identifier）型に変換します。

シグネチャまたは形状

Datum oidin(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される CSTRING 型の文字列。
  • **出力**: Oid 型の値を Datum 形式で返却します。

前提条件と不変条件

  • 入力文字列は有効な数値表現である必要があります。
  • 内部的に uint32in_subr を利用しており、型名は "oid" としてエラーメッセージ等に使用されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_CSTRING(0) を使用して、引数から入力文字列を取得します。 2. uint32in_subr 関数を呼び出し、文字列を uint32（Oid の実体）に変換します。この際、エラーコンテキストとして fcinfo->context が渡されます。 3. 変換結果を PG_RETURN_OID マクロを介して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/define.c:224
  • src/backend/utils/adt/regproc.c:1851
  • **呼び出し先**:
  • uint32in_subr (数値変換の基盤関数)

関連データ構造

  • Oid: システムカタログ等で使用されるオブジェクト識別子（uint32 の別名）。
  • Datum: PostgreSQLの汎用データ表現。

失敗モードと境界条件

  • 入力文字列が数値として解釈できない場合、uint32in_subr の仕様に従いエラーが発生します。
  • regproc.c 内の利用例では、ErrorSaveContext を用いたエラーハンドリングが考慮されています。

保守担当者向けメモ

  • 本関数は uint32in_subr に強く依存しています。数値変換ロジックの変更時は src/backend/utils/adt/int.c の該当箇所を確認してください。
  • oidin のエラー処理は regproc.c 等の呼び出し元で DirectInputFunctionCallSafe を通じて制御される場合があります。
