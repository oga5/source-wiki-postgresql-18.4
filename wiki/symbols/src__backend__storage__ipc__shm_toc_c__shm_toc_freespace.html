shm_toc_freespace

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shm_toc.c:130

目的

共有メモリ目次（TOC）セグメント内で、現在利用可能な空き容量（バイト数）を算出する。

シグネチャまたは形状

Size shm_toc_freespace(shm_toc *toc)

入力と出力

  • **入力**: toc (共有メモリ目次構造体へのポインタ)
  • **出力**: Size 型のバイト数（セグメント内で割り当て可能な残存領域）

前提条件と不変条件

  • toc は有効な shm_toc 構造体を指していること。
  • allocated_bytes + BUFFERALIGN(toc_bytes) <= total_bytes が常に成立すること（アサーションにより検証）。

アルゴリズムまたは振る舞い

1. toc->toc_mutex を取得し、セグメントの総容量 (toc_total_bytes)、割り当て済み容量 (toc_allocated_bytes)、エントリ数 (toc_nentry) を読み取る。 2. 読み取り後、直ちにスピンロックを解放する。 3. 現在の目次エントリ配列が占有するサイズ (toc_bytes) を計算する。これには shm_toc 構造体のヘッダサイズと、エントリ数分のサイズが含まれる。 4. total_bytes から、割り当て済み容量と目次エントリ領域（アライメント調整済み）の合計を減算し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は提供されていません。
  • **呼び出し先**: SpinLockAcquire および SpinLockRelease を使用して排他制御を行います。

関連データ構造

  • shm_toc: 目次を管理するメイン構造体。
  • shm_toc_entry: 目次内の各エントリを定義する構造体。

失敗モードと境界条件

  • toc が NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • 共有メモリの整合性が損なわれている場合、Assert によりプロセスが停止する可能性がある。

保守担当者向けメモ

  • 本関数はロックを取得して状態を読み取るため、頻繁な呼び出しはパフォーマンスに影響を与える可能性がある。
  • BUFFERALIGN マクロを使用してアライメントを考慮しているため、プラットフォーム固有の境界調整が適用される点に注意すること。
