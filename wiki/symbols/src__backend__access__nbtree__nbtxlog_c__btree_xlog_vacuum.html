btree_xlog_vacuum

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/nbtree/nbtxlog.c:597

目的

B-tree インデックスの VACUUM 操作に関連する WAL レコードをリプレイし、インデックスページ上の不要なタプルの削除および更新を再現する。

シグネチャまたは形状

static void btree_xlog_vacuum(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象の WAL レコード）
  • **出力**: なし（副作用としてバッファの内容を更新）

前提条件と不変条件

  • XLogReadBufferForRedoExtended を使用して対象ページを読み込み、BLK_NEEDS_REDO が返された場合にのみリプレイ処理を実行する。
  • ページに対してクリーンアップロック（cleanup lock）を取得する必要がある。

アルゴリズムまたは振る舞い

1. WAL レコードから xl_btree_vacuum 構造体を取得する。 2. 対象ページを読み込み、リプレイが必要な場合は以下を実行する：

  • nupdated > 0 の場合、btree_xlog_updates を呼び出し、指定されたオフセットのタプルを更新する。
  • ndeleted > 0 の場合、PageIndexMultiDelete を呼び出し、指定されたオフセットのタプルをページから削除する。
  • ページの btpo_cycleid を 0 にリセットし、BTP_HAS_GARBAGE フラグをクリアする。
  • ページの LSN を更新し、バッファをダーティとしてマークする。

    • 3. 最後にバッファのロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: btree_redo（src/backend/access/nbtree/nbtxlog.c:1048）
  • **呼び出し先**:
  • XLogReadBufferForRedoExtended
  • btree_xlog_updates
  • PageIndexMultiDelete
  • BTPageGetOpaque
  • PageSetLSN
  • MarkBufferDirty
  • UnlockReleaseBuffer

関連データ構造

  • xl_btree_vacuum: VACUUM 操作の WAL レコード構造体。
  • xl_btree_update: 更新対象のタプル情報。
  • BTPageOpaque: B-tree ページヘッダーの不透明領域。

失敗モードと境界条件

  • XLogReadBufferForRedoExtended が BLK_NEEDS_REDO を返さない場合、ページ更新はスキップされる。
  • xlrec->nupdated や xlrec->ndeleted が 0 の場合、それぞれの処理は実行されない。

保守担当者向けメモ

  • 本関数はリカバリ時の物理的なページ状態復元に特化している。
  • nbtree/README に記載されている通り、リカバリ時にはインデックス内のすべてのページに対して網羅的にクリーンアップロックを取得する必要はなく、削除対象アイテムを含むページのみで十分である。
