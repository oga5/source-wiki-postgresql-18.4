定義

目的

ECPG（Embedded SQL for C）の pgtypes ライブラリにおける数値型（numeric および decimal）の変換および演算機能の正確性を検証する回帰テストを実行する。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: グローバル変数 nums[]（テスト対象の数値文字列配列）。
  • 出力: 標準出力（stdout）へテスト結果のログを出力。エラー発生時は check_errno() を通じて標準エラー出力へ詳細が出力される。

前提条件と不変条件

  • nums[] 配列が有効な数値文字列で構成されていること。
  • ECPGdebug(1, stderr) により、実行時に詳細なデバッグログが有効化される。

アルゴリズムまたは振る舞い

1. **初期化**: nums 配列を走査し、各文字列を PGTYPESnumeric_from_asc で numeric 型に変換して動的配列 numarr に格納する。 2. **変換テスト**: 各 numeric 値に対し、以下の変換を行い結果を検証する。

  • 文字列への再変換（PGTYPESnumeric_to_asc）。
  • long 型および int 型への変換と、その逆変換後の値の比較（PGTYPESnumeric_cmp）。
  • double 型への変換（一部の例外を除く）。
  • decimal 型への変換と、その逆変換後の値の比較。

    • 3. **演算テスト**: numarr に格納された全ペアに対し、PGTYPESnumeric_add, sub, mul, div を実行し、結果を検証する。 4. **終了処理**: 確保したメモリ（numeric 型オブジェクトおよび decimal 型オブジェクト）を解放する。

呼び出し元と呼び出し先

  • 呼び出し元: 不明（テストスイートの実行プロセスにより起動される）。
  • 呼び出し先: PGTYPESnumeric_from_asc, PGTYPESnumeric_to_asc, PGTYPESnumeric_new, PGTYPESnumeric_to_long, PGTYPESnumeric_to_int, PGTYPESnumeric_to_double, PGTYPESnumeric_to_decimal, PGTYPESnumeric_add, PGTYPESnumeric_sub, PGTYPESnumeric_mul, PGTYPESnumeric_div, PGTYPESnumeric_cmp, PGTYPESnumeric_free, PGTYPESdecimal_new, PGTYPESdecimal_free, PGTYPESchar_free, check_errno, print_double, ECPGdebug。

関連データ構造

  • numeric: 数値型を保持する構造体。
  • decimal: 10進数型を保持する構造体。
  • numarr: numeric 型ポインタの動的配列。

失敗モードと境界条件

  • PGTYPESnumeric_from_asc 等の変換関数が失敗した場合、check_errno() が呼び出される。
  • i == 6 の場合、浮動小数点数のアンダーフローに関するアーキテクチャ依存の問題を避けるため、double への変換テストがスキップされる。
  • endptr が NULL でない場合、変換後のポインタ位置が検証される。

保守担当者向けメモ

  • 本関数は回帰テスト用であり、出力フォーマットは厳密に固定されている。変更時は期待値との差異に注意すること。
  • print_double 関数はプラットフォーム間の指数表記の差異を吸収するために使用されており、環境依存のテスト失敗を防ぐ役割を持つ。
