minmax_multi_init

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_minmax_multi.c:485

目的

BRIN インデックスの Multi Min/Max 演算クラスで使用される Ranges 構造体をメモリ上に初期化し、割り当てる。

シグネチャまたは形状

static Ranges *minmax_multi_init(int maxvalues)

入力と出力

  • **入力**: maxvalues (int) - 保持する値の最大数。
  • **出力**: 初期化された Ranges 構造体へのポインタ。

前提条件と不変条件

  • maxvalues > 0 であること（Assert により検証）。

アルゴリズムまたは振る舞い

1. Ranges 構造体の固定ヘッダーサイズと、maxvalues 分の Datum 配列に必要なサイズを計算する。 2. palloc0 を使用して、計算したサイズのメモリ領域をゼロクリアして確保する。 3. 確保した構造体の maxvalues フィールドに引数の値を設定する。 4. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_minmax_multi.c:742
  • src/backend/access/brin/brin_minmax_multi.c:2472
  • **呼び出し先**:
  • palloc0 (PostgreSQL メモリ管理関数)

関連データ構造

  • Ranges: 本関数が初期化する主要なデータ構造。

失敗モードと境界条件

  • maxvalues が 0 以下の場合、アサーション違反によりプロセスが停止する可能性がある。
  • メモリ不足時には palloc0 がエラーを発生させる。

保守担当者向けメモ

  • 本関数は Ranges 構造体のメモリレイアウト（ヘッダーと可変長配列の結合）に直接依存している。構造体の定義を変更する場合は、offsetof を使用したサイズ計算ロジックが正しいか確認すること。
