get_ENR

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/queryenvironment.c:95

目的

指定された名前を持つ一時的な名前付きリレーション（Ephemeral Named Relation: ENR）を、クエリ環境（QueryEnvironment）内から検索して取得します。

シグネチャまたは形状

EphemeralNamedRelation get_ENR(QueryEnvironment *queryEnv, const char *name)

入力と出力

  • **入力**:
  • queryEnv: 検索対象の QueryEnvironment 構造体へのポインタ。
  • name: 検索する ENR の名前（文字列）。
  • **出力**:
  • 見つかった場合は対応する EphemeralNamedRelation を返します。
  • 見つからない場合、または queryEnv が NULL の場合は NULL を返します。

前提条件と不変条件

  • name 引数は NULL であってはなりません（Assert により検証されます）。
  • queryEnv が NULL の場合、関数は安全に NULL を返します。

アルゴリズムまたは振る舞い

1. queryEnv が NULL であれば直ちに NULL を返します。 2. queryEnv->namedRelList に格納されているリストを線形探索します。 3. リスト内の各 EphemeralNamedRelation のメタデータ（md.name）と、引数の name を strcmp で比較します。 4. 一致する名前が見つかった場合、その EphemeralNamedRelation を返します。 5. リストの最後まで一致するものが見つからなければ NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeNamedtuplestorescan.c
  • src/backend/executor/spi.c
  • src/backend/utils/misc/queryenvironment.c（内部の登録・管理処理）
  • **呼び出し先**:
  • strcmp（標準ライブラリ）
  • lfirst（リスト操作マクロ）

関連データ構造

  • QueryEnvironment: ENR を保持するコンテナ。
  • EphemeralNamedRelation: ENR の実体。メタデータと実行時のデータポインタを保持。
  • List: namedRelList の実装に使用されるリスト構造。

失敗モードと境界条件

  • queryEnv が NULL の場合、エラーにはならず NULL を返します。
  • name が NULL の場合、アサーション違反によりプロセスが停止する可能性があります。
  • ENR が存在しない場合、正常な戻り値として NULL を返します。

保守担当者向けメモ

  • 現在の実装はリストの線形探索であり、ENR の数が少ないことを前提としています。パフォーマンス上の問題が発生した場合は、データ構造の変更を検討してください。
  • table_open 等のロック操作は本関数内では行われません。呼び出し側で適切なロックが保持されていることを前提としています。
