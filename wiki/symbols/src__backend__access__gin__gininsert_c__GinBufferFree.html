GinBufferFree

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/gininsert.c:1597

目的

GinBuffer 構造体に関連付けられたメモリリソースを解放し、構造体自体を破棄します。

シグネチャまたは形状

static void GinBufferFree(GinBuffer *buffer)

入力と出力

  • **入力**: buffer (GinBuffer *) - 解放対象のバッファ構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • buffer は有効なメモリ領域を指している必要があります。
  • buffer->items が割り当てられている場合、それは pfree で解放可能なメモリである必要があります。

アルゴリズムまたは振る舞い

1. buffer->items が存在する場合、pfree を呼び出して解放します。 2. バッファが空ではなく、かつカテゴリが GIN_CAT_NORM_KEY であり、型が値渡し（by-val）ではない場合、buffer->key が指すメモリ領域を pfree で解放します。 3. 最後に buffer 自体を pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gininsert.c:1790
  • src/backend/access/gin/gininsert.c:1978
  • **呼び出し先**:
  • pfree (PostgreSQL メモリ管理関数)
  • GinBufferIsEmpty (マクロまたは関数)
  • DatumGetPointer (マクロ)

関連データ構造

  • GinBuffer: GINインデックス構築中にキーとTIDリストを一時的に保持する構造体。

失敗モードと境界条件

  • buffer が NULL の場合、pfree の仕様に依存しますが、通常はセグメンテーション違反を引き起こす可能性があります。
  • buffer->items が NULL の場合はスキップされます。

保守担当者向けメモ

  • 本関数は GinBuffer のライフサイクルの終了時に呼び出されます。メモリリークを防ぐため、新しいフィールドを GinBuffer に追加した場合は、必ず本関数で解放処理を検討してください。
