assign_log_connections

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/backend_startup.c:1130

目的

GUC（Grand Unified Configuration）パラメータ log_connections の設定値が変更された際に、内部のグローバル変数 log_connections を更新するために使用されます。

シグネチャまたは形状

void assign_log_connections(const char *newval, void *extra)

入力と出力

  • newval: 設定される新しい値（文字列）。本関数内では使用されません。
  • extra: check_log_connections によって事前に計算・検証された、新しい設定値を含むメモリ領域へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • extra 引数は、check_log_connections 関数によって有効な int 型の値へのポインタとして準備されている必要があります。

アルゴリズムまたは振る舞い

1. extra 引数が指し示すメモリ領域から int 値をデリファレンスして取得します。 2. 取得した値をグローバル変数 log_connections に代入し、設定を反映させます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/misc/guc_tables.c 内のGUC設定テーブルから、設定変更時に呼び出されます。
  • 呼び出し先: 直接的な関数呼び出しはありません。

関連データ構造

  • log_connections: 接続ログの出力を制御するビットフラグを保持するグローバル変数。

失敗モードと境界条件

  • extra が NULL である場合、デリファレンス時にセグメンテーション違反が発生する可能性があります（ただし、GUCフレームワークの仕様上、check 関数が成功した後に assign が呼ばれるため、通常は発生しません）。

保守担当者向けメモ

  • 本関数は check_log_connections と対になる assign フックです。設定値の検証ロジックは check_log_connections 側に集約されているため、本関数は代入のみを行うシンプルな実装となっています。
