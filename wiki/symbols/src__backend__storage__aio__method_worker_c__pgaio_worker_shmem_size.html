pgaio_worker_shmem_size

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_worker.c:120

目的

PostgreSQLの非同期I/O（AIO）ワーカープロセス方式において、共有メモリ上に確保すべき合計サイズを算出する。

シグネチャまたは形状

size_t pgaio_worker_shmem_size(void)

入力と出力

  • 入力: なし
  • 出力: size_t 型の合計バイト数。

前提条件と不変条件

  • add_size 関数を用いて、各コンポーネントのサイズを安全に加算する。
  • 算出されるサイズは、キュー用メモリと制御構造体用メモリの合計値である。

アルゴリズムまたは振る舞い

1. pgaio_worker_queue_shmem_size を呼び出し、キューに必要なサイズを取得する。 2. pgaio_worker_control_shmem_size を呼び出し、制御構造体に必要なサイズを取得する。 3. 上記2つの値を add_size で加算し、その結果を返す。

呼び出し元と呼び出し先

  • 呼び出し元: 同一ファイル内の初期化処理（src/backend/storage/aio/method_worker.c:85 の構造体初期化など）。
  • 呼び出し先: pgaio_worker_queue_shmem_size および pgaio_worker_control_shmem_size。

関連データ構造

  • PgAioWorkerSubmissionQueue（キュー関連）
  • PgAioWorkerControl（制御構造体関連）

失敗モードと境界条件

  • add_size の内部仕様により、サイズ計算時のオーバーフローがチェックされる。

保守担当者向けメモ

  • 本関数は、ワーカー方式の共有メモリレイアウトが変更された際に、必ず更新する必要がある。
