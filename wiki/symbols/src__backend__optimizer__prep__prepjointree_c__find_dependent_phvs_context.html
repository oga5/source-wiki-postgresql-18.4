find_dependent_phvs_context

定義

  • **種別**: struct
  • **定義箇所**: src/backend/optimizer/prep/prepjointree.c:4021

目的

PlaceHolderVar (PHV) が特定のリレーション集合に依存しているかを探索する処理において、探索条件と状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct
{
	Relids		relids;       /* 探索対象のリレーションID集合 */
	int			sublevels_up; /* 現在のクエリレベルからの相対的な深さ */
} find_dependent_phvs_context;

入力と出力

本構造体はデータ保持用であり、関数ではありません。探索処理の過程で、この構造体のインスタンスが初期化され、探索対象の条件として利用されます。

前提条件と不変条件

  • relids は、探索対象となるリレーションの集合を指す必要があります。
  • sublevels_up は、現在のクエリレベルを基準とした階層の深さを表す非負の整数である必要があります。

アルゴリズムまたは振る舞い

この構造体は、ジョインツリーの走査中に PlaceHolderVar を検索する際の引数として渡されます。走査関数は、この構造体に格納された relids および sublevels_up を参照し、現在のノードが条件に合致するかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/prep/prepjointree.c 内の探索関数（4065行目および4087行目でローカル変数として宣言）。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数呼び出しは行いません。

関連データ構造

  • Relids: リレーションIDの集合を表現するビットマップセット。
  • PlaceHolderVar: 本構造体が探索対象とする最適化ノード。

失敗モードと境界条件

  • relids が空の場合、依存関係の判定結果は常に偽となります。
  • sublevels_up が現在のクエリの有効範囲を超えている場合、探索は正しく機能しません。

保守担当者向けメモ

  • 本構造体は prepjointree.c 内の特定の探索処理専用です。他のファイルからの参照は確認されていません。
  • 構造体のフィールドを変更する場合は、これを利用する探索関数側のロジックも併せて更新する必要があります。
