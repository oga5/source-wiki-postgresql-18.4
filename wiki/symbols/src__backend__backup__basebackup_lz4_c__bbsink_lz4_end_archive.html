bbsink_lz4_end_archive

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_lz4.c:227

目的

LZ4圧縮ストリームを終了させ、残りの圧縮データをフラッシュし、圧縮コンテキストを解放して、後続のシンクへ終了通知を転送する。

シグネチャまたは形状

static void bbsink_lz4_end_archive(bbsink *sink)

入力と出力

  • **入力**: sink (bbsink *) - 処理対象のLZ4シンク構造体。
  • **出力**: なし（副作用として後続シンクへのデータ転送およびリソース解放を行う）。

前提条件と不変条件

  • sink は bbsink_lz4 型としてキャスト可能であること。
  • mysink->base.bbs_next->bbs_buffer_length が LZ4F_compressBound(0, &mysink->prefs) 以上であること（アサーションにより検証）。

アルゴリズムまたは振る舞い

1. **バッファ確認**: 現在のバッファ残量がLZ4フッター書き込みに必要なサイズに満たない場合、bbsink_archive_contents を呼び出してバッファをフラッシュする。 2. **圧縮終了**: LZ4F_compressEnd を呼び出し、圧縮ストリームを完結させる。 3. **エラーチェック**: LZ4F_isError を用いて圧縮終了処理の成否を確認し、失敗時は elog(ERROR) で中断する。 4. **データ転送**: 圧縮されたフッターデータを後続のシンクへ送信し、bytes_written をリセットする。 5. **リソース解放**: LZ4F_freeCompressionContext で圧縮コンテキストを破棄し、ポインタを NULL に設定する。 6. **伝播**: bbsink_forward_end_archive を呼び出し、アーカイブ終了を後続のシンクへ通知する。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_lz4_ops 構造体を通じて、シンクパイプラインの終了処理時に呼び出される。
  • **呼び出し先**:
  • bbsink_archive_contents (後続シンクへのデータ転送)
  • LZ4F_compressEnd (LZ4ライブラリ)
  • LZ4F_freeCompressionContext (LZ4ライブラリ)
  • bbsink_forward_end_archive (パイプラインの次段へ通知)

関連データ構造

  • bbsink_lz4: 圧縮コンテキスト (ctx) や設定 (prefs) を保持する構造体。
  • bbsink: パイプラインの基底構造体。

失敗モードと境界条件

  • LZ4F_compressEnd がエラーを返した場合、elog(ERROR) により処理が中断される。
  • バッファサイズがフッター書き込みに不足している場合、自動的にフラッシュ処理が先行して実行される。

保守担当者向けメモ

  • 本関数は USE_LZ4 が定義されている環境でのみコンパイルされる。
  • bbsink_lz4 構造体の ctx は本関数で解放されるため、呼び出し後に再利用することはできない。
