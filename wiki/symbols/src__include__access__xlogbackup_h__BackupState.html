BackupState

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlogbackup.h:21

目的

PostgreSQL のベースバックアップ処理において、バックアップのライフサイクル全体（開始から終了まで）の状態を保持・管理するためのデータ構造です。

シグネチャまたは形状

typedef struct BackupState
{
    char        name[MAXPGPATH + 1];
    XLogRecPtr  startpoint;
    TimeLineID  starttli;
    XLogRecPtr  checkpointloc;
    pg_time_t   starttime;
    bool        started_in_recovery;
    XLogRecPtr  istartpoint;
    TimeLineID  istarttli;
    XLogRecPtr  stoppoint;
    TimeLineID  stoptli;
    pg_time_t   stoptime;
} BackupState;

入力と出力

本構造体自体はデータ保持用であり、build_backup_content() 関数に渡されることで、backup_label ファイルやバックアップ履歴ファイルへ書き出すための文字列形式に変換されます。

前提条件と不変条件

  • name フィールドは MAXPGPATH + 1 のサイズを持ち、null 終端が保証されます。
  • istartpoint および istarttli は増分バックアップの起点情報を保持します。

アルゴリズムまたは振る舞い

1. **初期化**: バックアップ開始時にインスタンス化され、開始時の WAL 位置（startpoint）、タイムライン（starttli）、チェックポイント位置、開始時刻などが記録されます。 2. **更新**: バックアップ終了時に、終了時の WAL 位置（stoppoint）、タイムライン（stoptli）、終了時刻が追記されます。 3. **変換**: build_backup_content() が本構造体を参照し、ishistoryfile フラグに応じて適切なフォーマットの文字列を生成します。

呼び出し元と呼び出し先

  • **呼び出し元（参照/操作）**:
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogfuncs.c
  • src/backend/backup/basebackup.c
  • src/backend/backup/basebackup_incremental.c
  • **呼び出し先**: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • XLogRecPtr: WAL 位置を保持する型。
  • TimeLineID: タイムライン識別子。
  • pg_time_t: 時刻管理用型。

失敗モードと境界条件

  • メモリ割り当て失敗時（palloc0 等の利用時）に処理が中断される可能性があります。
  • name フィールドのバッファサイズを超えるパス名は扱えません。

保守担当者向けメモ

  • 本構造体は src/backend/access/transam/xlogbackup.c のロジックと密接に結合しています。
  • 増分バックアップの仕様変更時には istartpoint および istarttli の扱いを確認してください。
  • 履歴ファイル生成時には終了時の情報も必要となるため、バックアップ終了処理の順序に注意してください。
