dlist_insert_after

定義

  • 種別: static inline function
  • 定義箇所: src/include/lib/ilist.h:380

目的

二重連結リスト（dlist）において、指定したノード（after）の直後に新しいノード（node）を挿入します。

シグネチャまたは形状

static inline void dlist_insert_after(dlist_node *after, dlist_node *node)

入力と出力

  • after: 挿入位置の基準となる既存のリストノード。
  • node: リストに挿入する新しいノード。
  • 戻り値: なし。

前提条件と不変条件

  • after は既にリストの一部であるか、有効なノードである必要があります。
  • node は挿入前にリストに含まれていない（または適切に初期化されている）ことが想定されます。
  • 挿入後、node の prev は after を、node の next は元の after->next を指すよう更新されます。

アルゴリズムまたは振る舞い

1. node->prev を after に設定します。 2. node->next を after->next に設定します。 3. after->next を node に更新します。 4. 元の after->next であったノードの prev を node に更新します。 これにより、リストのリンク構造が整合性を保ったまま挿入されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/gin/gindatapage.c:1638
  • src/backend/replication/syncrep.c:390
  • src/include/lib/ilist.h:732
  • 呼び出し先:
  • 本関数はポインタ操作のみを行うため、他の関数を呼び出しません。

関連データ構造

  • dlist_node: リストのリンクを保持する構造体。dlist_insert_after はこの構造体の next および prev ポインタを直接操作します。

失敗モードと境界条件

  • after が NULL の場合、デリファレンスによりクラッシュします。
  • after->next が存在しない（リストの末尾である）場合、node->next->prev へのアクセスが不正になる可能性があります。本関数はリストが循環していること、あるいは after->next が有効なノード（またはリストの終端を示すダミーノード）であることを前提としています。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結しています。
  • 侵入型リストの特性上、ノードのメモリ管理は呼び出し側に依存します。ノードがリストに含まれている間は、そのノードを含む構造体を解放しないでください。
