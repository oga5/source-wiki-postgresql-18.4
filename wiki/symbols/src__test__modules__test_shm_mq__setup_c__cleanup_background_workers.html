cleanup_background_workers

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_shm_mq/setup.c:244

目的

動的共有メモリ（DSM）セグメントの解放時に、関連付けられたバックグラウンドワーカーを強制終了させるためのクリーンアップ処理を実行します。

シグネチャまたは形状

static void cleanup_background_workers(dsm_segment *seg, Datum arg)

入力と出力

  • **入力**:
  • seg: 解放対象の dsm_segment へのポインタ。
  • arg: worker_state 構造体へのポインタを保持する Datum 型の値。
  • **出力**: なし（void）。

前提条件と不変条件

  • arg は有効な worker_state 構造体を指している必要があります。
  • wstate->nworkers は現在アクティブなワーカーの数と一致している必要があります。

アルゴリズムまたは振る舞い

1. arg から worker_state 構造体を取得します。 2. wstate->nworkers が 0 になるまでループを実行します。 3. ループ内で nworkers をデクリメントし、対応する wstate->handle を引数として TerminateBackgroundWorker を呼び出し、ワーカーを停止させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • on_dsm_detach フック（src/test/modules/test_shm_mq/setup.c:213）により、DSMセグメントがデタッチされる際に自動的に呼び出されます。
  • cancel_on_dsm_detach（src/test/modules/test_shm_mq/setup.c:78）の引数として参照されています。
  • **呼び出し先**:
  • TerminateBackgroundWorker（PostgreSQL内部関数）

関連データ構造

  • worker_state: 起動したバックグラウンドワーカーのハンドルと数を保持する構造体。
  • dsm_segment: 動的共有メモリセグメント。

失敗モードと境界条件

  • wstate->nworkers が 0 の場合、ループは実行されず何も行われません。
  • TerminateBackgroundWorker の呼び出し自体が失敗した場合の挙動は、PostgreSQLのバックグラウンドワーカー管理機構に依存します。

保守担当者向けメモ

  • 本関数は on_dsm_detach フックとして登録されるため、メモリコンテキストの生存期間に注意してください。setup.c 内では TopTransactionContext を使用してワーカーハンドルを管理することで、このフックが確実に動作するように設計されています。
