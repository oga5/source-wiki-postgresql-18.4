crc32_sz

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_gist.c:80

目的

指定されたバッファの内容に対して、PostgreSQL の伝統的な CRC-32 チェックサムを計算します。主に hstore 型のキーや値を GiST インデックス用のシグネチャ（ビットベクトル）にマッピングする際のハッシュ値生成に使用されます。

シグネチャまたは形状

static pg_crc32 crc32_sz(const char *buf, int size)

入力と出力

  • **入力**:
  • buf: ハッシュ対象のデータが格納されたバッファへのポインタ。
  • size: バッファのサイズ（バイト単位）。
  • **出力**:
  • pg_crc32: 計算された CRC-32 値。

前提条件と不変条件

  • utils/pg_crc.h で定義された INIT_TRADITIONAL_CRC32、COMP_TRADITIONAL_CRC32、FIN_TRADITIONAL_CRC32 マクロが利用可能であること。
  • 入力バッファ buf は、指定された size 分のメモリ領域として有効であること。

アルゴリズムまたは振る舞い

1. INIT_TRADITIONAL_CRC32 を使用して CRC 変数を初期化します。 2. COMP_TRADITIONAL_CRC32 を使用して、バッファ buf の内容を順次読み込み、CRC 値を更新します。 3. FIN_TRADITIONAL_CRC32 を使用して最終的なチェックサム値を確定させ、その値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_gist.c 内の以下の箇所から呼び出されています。
  • ghstore_compress（キーおよび値のハッシュ化）
  • ghstore_consistent（クエリ条件のハッシュ化）
  • **呼び出し先**:
  • utils/pg_crc.h で定義された CRC-32 計算用マクロ群。

関連データ構造

  • pg_crc32: PostgreSQL で定義された CRC-32 値を保持するための型。

失敗モードと境界条件

  • size が 0 の場合、初期化された CRC 値がそのまま返されます。
  • 本関数はハッシュ衝突を許容する設計であり、厳密な一致判定ではなく、インデックスにおけるフィルタリング（可能性の絞り込み）を目的としています。

保守担当者向けメモ

  • 本関数は hstore の GiST インデックスにおけるシグネチャ生成の基盤です。インデックスの衝突率を調整する際は、このハッシュ関数の特性を考慮してください。
  • hstore_gist.c 内の複数の箇所で、HSTORE_KEY や VARDATA 等のデータアクセス用マクロと組み合わせて使用されています。
