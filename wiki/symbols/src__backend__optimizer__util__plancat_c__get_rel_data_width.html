get_rel_data_width

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/plancat.c:1199

目的

リレーションの全属性の平均幅を合計し、タプルあたりの推定データ幅を算出します。

シグネチャまたは形状

int32 get_rel_data_width(Relation rel, int32 *attr_widths)

入力と出力

  • **入力**:
  • rel: 対象リレーションの Relation 構造体。
  • attr_widths: 各属性の幅を格納する配列（キャッシュ用）。NULL 指定可能。
  • **出力**:
  • 推定されたタプル幅（int32）。

前提条件と不変条件

  • rel は有効なリレーションであること。
  • attr_widths が提供される場合、各属性の幅が正の値であればそれが再利用されます。
  • get_typavgwidth が返す型平均幅は常に正の値である必要があります（Assert により検証）。

アルゴリズムまたは振る舞い

1. リレーションの全属性を走査します。 2. 削除された属性（attisdropped）はスキップします。 3. attr_widths 配列に既に値が存在する場合はそれを加算します。 4. キャッシュがない場合、get_attavgwidth で統計情報を取得し、取得できない場合は get_typavgwidth で型定義に基づく平均幅を取得します。 5. 取得した幅を attr_widths に保存し、合計値に加算します。 6. 最後に clamp_width_est を適用して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/table/tableam.c
  • src/backend/optimizer/util/plancat.c（get_relation_data_width 等から呼び出し）
  • **呼び出し先**:
  • RelationGetNumberOfAttributes
  • TupleDescAttr
  • get_attavgwidth
  • get_typavgwidth
  • clamp_width_est

関連データ構造

  • Relation: リレーションのメタデータ構造。
  • Form_pg_attribute: 属性定義情報。

失敗モードと境界条件

  • 属性が削除されている場合、その幅は計算に含まれません。
  • 統計情報（get_attavgwidth）が取得できない場合、型情報（get_typavgwidth）にフォールバックします。
  • attr_widths が NULL の場合、キャッシュは行われません。

保守担当者向けメモ

  • 本関数は costsize.c 内の set_rel_width() と整合性を保つ必要があります。
  • attr_widths 配列のインデックスは 1 始まりである点に注意してください。
