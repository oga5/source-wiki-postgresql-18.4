decide_s2k_iter

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-s2k.c:208

目的

OpenPGPのIterated-Salted S2Kモードにおいて、指定された反復回数（count）を満たすために必要なS2K反復回数パラメータ（0〜255の範囲）を決定します。

シグネチャまたは形状

static uint8 decide_s2k_iter(unsigned rand_byte, int count)

入力と出力

  • **入力**:
  • rand_byte: 反復回数のデフォルト値を決定するための乱数バイト。
  • count: ターゲットとなる最小の反復回数。-1 の場合はデフォルトの反復回数計算に使用されます。
  • **出力**:
  • uint8: 0〜255の範囲のS2K反復回数パラメータ。

前提条件と不変条件

  • count が -1 の場合、rand_byte を用いてデフォルトの反復回数範囲（96〜127）から値を算出します。
  • count が指定されている場合、s2k_decode_count(iter) が count 以上となる最小の iter を返します。

アルゴリズムまたは振る舞い

1. count が -1 の場合、96 + (rand_byte & 0x1F) を計算して返します。 2. count が正の値の場合、0から255まで順次 s2k_decode_count を呼び出し、結果が count 以上になる最初の値を返します。 3. 255までに条件を満たす値が見つからない場合は、最大値である 255 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: pgp_s2k_fill（contrib/pgcrypto/pgp-s2k.c:244）から呼び出されます。
  • **呼び出し先**: s2k_decode_count を呼び出します。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • count が非常に大きい場合、ループは255で終了し、結果として 255 が返されます。
  • rand_byte の下位5ビットのみがデフォルト値の計算に使用されます。

保守担当者向けメモ

  • 本関数はセキュリティとパフォーマンスのバランスを調整する役割を持ちます。反復回数のエンコード仕様については pgp.h の関連ヘルパー関数と併せて参照してください。
