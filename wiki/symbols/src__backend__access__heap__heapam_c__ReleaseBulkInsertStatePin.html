ReleaseBulkInsertStatePin

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam.c:2041

目的

BulkInsertState が保持しているバッファのピンを解放し、バルク挿入に関連するリレーション拡張状態をリセットする。

シグネチャまたは形状

void ReleaseBulkInsertStatePin(BulkInsertState bistate)

入力と出力

  • **入力**: BulkInsertState bistate（バルク挿入の状態を管理する構造体へのポインタ）
  • **出力**: なし（副作用として bistate の内容が更新される）

前提条件と不変条件

  • bistate は有効なポインタである必要がある。
  • bistate->current_buf が InvalidBuffer でない場合、そのバッファはピン留めされているものとみなされる。

アルゴリズムまたは振る舞い

1. bistate->current_buf が InvalidBuffer でない場合、ReleaseBuffer を呼び出してバッファを解放する。 2. bistate->current_buf を InvalidBuffer に設定する。 3. bistate->next_free および bistate->last_free を InvalidBlockNumber にリセットする。これにより、異なるパーティション間での領域検索の競合や誤ったオフセット参照を防ぐ。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfrom.c:1258
  • その他、heapam.c 内のバルク挿入処理フロー（ソースコード上のコメントより）。
  • **呼び出し先**:
  • ReleaseBuffer（バッファ管理サブシステム）

関連データ構造

  • BulkInsertState: バルク挿入時のバッファや空き領域情報を保持する構造体。

失敗モードと境界条件

  • bistate が NULL の場合の挙動は定義されていない（セグメンテーション違反の可能性がある）。
  • current_buf が既に解放済み、あるいは無効なバッファを指している場合の挙動は、ReleaseBuffer の仕様に依存する。

保守担当者向けメモ

  • 本関数は名前の通りバッファの解放を行うだけでなく、リレーション拡張状態（next_free, last_free）のリセットも行う。これは、異なるパーティションに対して同一の BulkInsertState を再利用する際に、古いパーティションの空き領域情報を引き継いでしまうことを防ぐために不可欠である。
