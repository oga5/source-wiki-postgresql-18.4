oauth_exchange

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-oauth.c:133

目的

SASL OAUTHBEARER 認証メカニズムにおけるサーバー側のプロトコル交換処理を実行し、クライアントから提供されたトークンの検証および状態遷移を管理する。

シグネチャまたは形状

static int oauth_exchange(void *opaq, const char *input, int inputlen,
                          char **output, int *outputlen, const char **logdetail);

入力と出力

  • **入力**:
  • opaq: oauth_ctx 構造体へのポインタ（セッション状態を保持）。
  • input: クライアントからのメッセージ文字列。
  • inputlen: 入力メッセージの長さ。
  • **出力**:
  • output: 認証失敗時のエラー応答メッセージ（JSON形式）を格納するバッファ。
  • outputlen: 出力メッセージの長さ。
  • **戻り値**: PG_SASL_EXCHANGE_SUCCESS（成功）、PG_SASL_EXCHANGE_CONTINUE（継続）、または PG_SASL_EXCHANGE_FAILURE（失敗）。

前提条件と不変条件

  • input が NULL の場合、ctx->state は OAUTH_STATE_INIT である必要がある。
  • inputlen は input の実際の文字列長と一致しなければならない。
  • 認証成功時には ctx->state が OAUTH_STATE_FINISHED に更新される。

アルゴリズムまたは振る舞い

1. **初期ハンドシェイク**: input が NULL の場合、空のチャレンジを返して継続する。 2. **状態チェック**: OAUTH_STATE_INIT または OAUTH_STATE_ERROR 状態を処理する。それ以外の状態はエラーとする。 3. **メッセージ解析**: クライアントからのメッセージを解析し、チャネルバインディングフラグ（'n' または 'y'）と認証属性を確認する。 4. **トークン抽出**: parse_kvpairs_for_auth を使用して auth キーの値を抽出する。 5. **検証**: validate 関数を呼び出し、トークンを検証する。

  • 成功時: OAUTH_STATE_FINISHED に遷移し SUCCESS を返す。
  • 失敗時: generate_error_response でエラー情報を生成し、OAUTH_STATE_ERROR に遷移して CONTINUE を返す。

    • 6. **クリーンアップ**: トークンを含むメモリ領域を explicit_bzero で消去する。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_be_oauth_mech 構造体を通じて認証フレームワークから呼び出される。
  • **呼び出し先**: parse_kvpairs_for_auth, validate, generate_error_response, sanitize_char, explicit_bzero。

関連データ構造

  • oauth_ctx: セッションごとの状態（state）とポート情報を保持する構造体。
  • oauth_state: 認証プロトコルの状態遷移を定義する列挙型。

失敗モードと境界条件

  • **プロトコル違反**: メッセージが空、長さ不一致、不正なチャネルバインディングフラグ、または必須属性の欠如が発生した場合、ERRCODE_PROTOCOL_VIOLATION でエラーを報告する。
  • **機能未サポート**: 認可ID（authzid）が含まれている場合、ERRCODE_FEATURE_NOT_SUPPORTED を報告する。
  • **検証失敗**: トークンが無効な場合、エラー応答を生成して認証を継続（再試行）させる。

保守担当者向けメモ

  • 本関数は RFC 7628 に準拠している。
  • input_copy を使用してトークンを処理し、終了時に explicit_bzero で確実に消去することで、メモリ上の機密情報漏洩を防いでいる。
  • 外部検証モジュールとのインターフェースは validate 関数にカプセル化されている。
