CreateTuplestoreDestReceiver

定義

  • 種別: function
  • 定義箇所: src/backend/executor/tstoreReceiver.c:237

目的

実行エンジンが生成したタプルを Tuplestore に格納するための DestReceiver オブジェクトを生成・初期化するコンストラクタです。

シグネチャまたは形状

DestReceiver *CreateTuplestoreDestReceiver(void);

入力と出力

  • **入力**: なし
  • **出力**: DestReceiver 構造体へのポインタ（実際には TStoreState 構造体として確保される）。

前提条件と不変条件

  • 戻り値の DestReceiver は、後続の SetTuplestoreDestReceiverParams 関数によってプライベートフィールドが設定されるまで、完全な状態ではありません。
  • 内部的に palloc0 を使用して TStoreState を確保するため、呼び出し元のメモリコンテキストが適切である必要があります。

アルゴリズムまたは振る舞い

1. TStoreState 構造体分のメモリをゼロクリアして確保します。 2. DestReceiver インターフェースの関数ポインタを以下の通り初期設定します。

  • receiveSlot: tstoreReceiveSlot_notoast（初期値）
  • rStartup: tstoreStartupReceiver
  • rShutdown: tstoreShutdownReceiver
  • rDestroy: tstoreDestroyReceiver

    • 3. mydest フィールドを DestTuplestore に設定します。 4. 確保した構造体を DestReceiver * にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/dest.c:139
  • src/backend/executor/tstoreReceiver.c:238（自己参照の可能性あり）
  • **呼び出し先**:
  • palloc0（メモリ確保）
  • tstoreReceiveSlot_notoast, tstoreStartupReceiver, tstoreShutdownReceiver, tstoreDestroyReceiver（関数ポインタの代入）

関連データ構造

  • TStoreState: 本関数が生成するプライベート構造体。
  • DestReceiver: 本関数が返す公開インターフェース構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 によりエラーが発生します。
  • 本関数自体はパラメータ検証を行いません。パラメータの不整合（detoast と tupmap の同時指定など）は、後続の SetTuplestoreDestReceiverParams でチェックされます。

保守担当者向けメモ

  • receiveSlot は初期値として tstoreReceiveSlot_notoast が設定されますが、rStartup 実行時に tstoreStartupReceiver によって状況に応じて最適な関数へ差し替えられます。
  • 本関数で設定される関数ポインタは tstoreReceiver.c 内の静的関数を指しており、インターフェースの契約に従っています。
