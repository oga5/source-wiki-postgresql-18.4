gimme_tour

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_erx.c:195

目的

エッジ組み換え交叉（ERX）アルゴリズムを用いて、2つの親個体から新しい子個体（ツアー）を生成する。

シグネチャまたは形状

int gimme_tour(PlannerInfo *root, Edge *edge_table, Gene *new_gene, int num_gene)

入力と出力

  • **入力**:
  • root: クエリ最適化のプランナ情報。
  • edge_table: 都市間の接続関係を保持するエッジテーブル。
  • new_gene: 生成されるツアーを格納する配列。
  • num_gene: ツアーに含まれる遺伝子（都市）の総数。
  • **出力**:
  • int: ツアー構築中に発生した「エッジ失敗（接続先候補の枯渇）」の総数。

前提条件と不変条件

  • edge_table は、gimme_edge_table によって適切に初期化されている必要がある。
  • new_gene は num_gene 分のメモリが確保されていること。
  • ツアー構築の各ステップで、選択された遺伝子は edge_table から削除され、unused_edges が -1 に更新される。

アルゴリズムまたは振る舞い

1. geqo_randint を使用して、ツアーの開始点となる遺伝子をランダムに選択する。 2. num_gene - 1 回のループを実行し、以下の処理を繰り返す：

  • remove_gene を呼び出し、直前に選択した遺伝子をエッジテーブルから除去する。
  • edge_table[...].unused_edges > 0 の場合、gimme_gene を呼び出して次に接続すべき遺伝子を決定する。
  • 接続先候補がない場合、edge_failure を呼び出してフォールバック処理を行い、失敗回数をカウントする。
  • 選択した遺伝子の unused_edges を -1 に設定し、使用済みとしてマークする。

    • 3. 最終的に発生したエッジ失敗の総数を返す。

呼び出し元と呼び出し先

関連データ構造

  • Edge: 都市間の接続関係を保持する構造体。
  • Gene: 遺伝子（都市）を表す型。
  • PlannerInfo: PostgreSQLのプランナ情報構造体。

失敗モードと境界条件

  • 接続可能なエッジが存在しない場合、edge_failure が呼び出され、残りの遺伝子からランダムに選択が行われる。この回数は戻り値として集計される。

保守担当者向けメモ

  • 本関数は ERX マクロが定義されている環境でのみ使用される。
  • edge_table の状態管理（特に unused_edges の更新）がツアーの妥当性に直結するため、remove_gene との整合性に注意すること。
