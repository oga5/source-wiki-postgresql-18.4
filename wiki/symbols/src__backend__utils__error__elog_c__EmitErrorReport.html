EmitErrorReport

定義

  • 種別: function
  • 定義箇所: src/backend/utils/error/elog.c:1691

目的

蓄積されたエラー情報（ErrorData）を、設定された出力先（サーバーログおよびクライアント）へ送信する。

シグネチャまたは形状

void EmitErrorReport(void)

入力と出力

  • **入力**: グローバルなエラーデータスタック errordata の現在の深さにある ErrorData 構造体。
  • **出力**: なし（副作用としてログ出力やクライアントへのメッセージ送信を行う）。

前提条件と不変条件

  • errordata_stack_depth が有効なエラーデータを指していること。
  • recursion_depth をインクリメントし、処理終了後にデクリメントする。
  • 処理中は edata->assoc_context を現在のメモリコンテキストとして使用する。

アルゴリズムまたは振る舞い

1. recursion_depth をインクリメントし、スタック深さをチェックする。 2. edata->assoc_context へメモリコンテキストを切り替える。 3. タイムスタンプ関連のフィールド（saved_timeval_set, formatted_log_time）をリセットする。 4. emit_log_hook が定義されている場合、サーバーログ送信前にフック関数を実行する（フックは output_to_server を無効化できる）。 5. edata->output_to_server が真であれば send_message_to_server_log を呼び出す。 6. edata->output_to_client が真であれば send_message_to_frontend を呼び出す。 7. メモリコンテキストを復元し、recursion_depth をデクリメントする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/bgworker.c
  • src/backend/postmaster/bgwriter.c
  • src/backend/postmaster/checkpointer.c
  • src/backend/postmaster/pgarch.c
  • src/backend/postmaster/walsummarizer.c
  • src/backend/postmaster/walwriter.c
  • src/backend/replication/logical/slotsync.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • send_message_to_server_log
  • send_message_to_frontend
  • emit_log_hook（存在する場合）

関連データ構造

  • ErrorData: エラーの詳細情報を保持する構造体。
  • errordata: ErrorData のスタック。

失敗モードと境界条件

  • emit_log_hook による output_to_server の無効化は許可されているが、有効化はサポートされていない。
  • フック関数内での ErrorData 内容の変更はサポートされていない。

保守担当者向けメモ

  • 本関数はエラー処理の最終段階で呼び出される。再帰的なエラー発生を防ぐため、recursion_depth の管理が重要である。
  • emit_log_hook はカスタムログフィルタリングや送信メカニズムのために提供されており、エラーメッセージの識別子として元の英語メッセージにアクセス可能である。
