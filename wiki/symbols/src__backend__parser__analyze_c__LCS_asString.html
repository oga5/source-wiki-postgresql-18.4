LCS_asString

定義

  • 種別: function
  • 定義箇所: src/backend/parser/analyze.c:3377

目的

LockClauseStrength 列挙型値を、対応する SQL の FOR ... 句の文字列形式に変換します。主にエラーメッセージやデバッグ出力で使用されます。

シグネチャまたは形状

const char *LCS_asString(LockClauseStrength strength)

入力と出力

  • **入力**: LockClauseStrength strength（ロック強度を示す列挙値）
  • **出力**: const char *（対応する SQL 文字列リテラル）

前提条件と不変条件

  • strength は有効な LockClauseStrength 値である必要があります。
  • LCS_NONE が渡された場合、Assert(false) によりデバッグビルドではアサーションエラーが発生します。

アルゴリズムまたは振る舞い

1. strength の値に基づき switch 文で分岐します。 2. 各列挙値（LCS_FORKEYSHARE, LCS_FORSHARE, LCS_FORNOKEYUPDATE, LCS_FORUPDATE）に対して、対応する文字列（例: "FOR UPDATE"）を返します。 3. LCS_NONE の場合はアサーション失敗となります。 4. 万が一、定義外の値が渡された場合は "FOR some" を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/initsplan.c
  • src/backend/optimizer/plan/planner.c
  • src/backend/parser/analyze.c（複数箇所）
  • **呼び出し先**:
  • Assert マクロ（LCS_NONE の場合）

関連データ構造

  • LockClauseStrength（列挙型）

失敗モードと境界条件

  • LCS_NONE を渡すとアサーション違反となります。
  • 未定義の列挙値が渡された場合、フォールバックとして "FOR some" を返しますが、これは想定外の動作です。

保守担当者向けメモ

  • 新しいロック強度が LockClauseStrength に追加された場合、本関数の switch 文にも必ずケースを追加してください。
