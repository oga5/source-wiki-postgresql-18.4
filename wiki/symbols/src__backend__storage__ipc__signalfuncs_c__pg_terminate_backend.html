pg_terminate_backend

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/signalfuncs.c:236

目的

指定されたプロセスID（PID）を持つバックエンドプロセスに対し、SIGTERM シグナルを送信して終了を要求するSQL関数です。

シグネチャまたは形状

Datum pg_terminate_backend(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • pid (int32): 終了対象のプロセスID。
  • timeout (int64): 終了を待機するミリ秒数。
  • **出力**:
  • bool: シグナル送信が成功し、かつ待機オプションが指定された場合は終了確認結果を、それ以外は送信の成否を返します。

前提条件と不変条件

  • timeout は負の値であってはなりません（負の場合はエラーを報告します）。
  • 実行には適切な権限が必要です（スーパーユーザー、または対象プロセスの所有者など）。

アルゴリズムまたは振る舞い

1. 引数から pid と timeout を取得します。 2. pg_signal_backend(pid, SIGTERM) を呼び出し、権限チェックとシグナル送信を行います。 3. 戻り値に基づき、権限不足の場合は適切なエラー（ERRCODE_INSUFFICIENT_PRIVILEGE）を報告します。 4. timeout > 0 かつシグナル送信が成功した場合、pg_wait_until_termination(pid, timeout) を呼び出し、プロセスが終了するまで待機してその結果を返します。 5. 上記以外の場合は、シグナル送信の成否（true/false）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます。src/backend/storage/ipc/procarray.c 内のコメントで本関数との権限チェックの差異について言及されています。
  • **呼び出し先**:
  • pg_signal_backend (シグナル送信と権限検証)
  • pg_wait_until_termination (終了待機)
  • ereport (エラー報告)

関連データ構造

  • PG_FUNCTION_ARGS: SQL関数呼び出しの引数コンテナ。

失敗モードと境界条件

  • **権限エラー**: スーパーユーザー権限の不足、オートバキュームワーカー終了権限の不足、または対象プロセスに対する操作権限がない場合にエラーが発生します。
  • **引数エラー**: timeout が負の場合にエラーが発生します。

保守担当者向けメモ

  • 本関数は SIGTERM を送信します。プロセスが即座に終了しない場合、timeout を指定することで待機処理が有効になります。
  • 権限チェックロジックは pg_signal_backend に依存しているため、権限要件の変更はそちらを確認してください。
