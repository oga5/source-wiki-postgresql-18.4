SearchSysCache1

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/syscache.c:220

目的

単一のキーを使用してシステムカタログキャッシュからタプルを検索し、取得します。

シグネチャまたは形状

HeapTuple SearchSysCache1(int cacheId, Datum key1)

入力と出力

  • **入力**:
  • cacheId: 検索対象のキャッシュを識別する整数ID。
  • key1: 検索に使用する単一のキー値（Datum）。
  • **出力**:
  • 成功時: 見つかったタプルへのポインタ（HeapTuple）。
  • 失敗時: NULL。

前提条件と不変条件

  • cacheId は有効な範囲内（0 以上 SysCacheSize 未満）であること。
  • 指定された cacheId に対応するキャッシュ構造体が初期化済みであること。
  • 指定されたキャッシュのキー数が 1 であること（cc_nkeys == 1）。

アルゴリズムまたは振る舞い

1. cacheId が有効範囲内であり、対応するキャッシュ構造体が初期化されていることをアサートで検証します。 2. 指定されたキャッシュのキー数が 1 であることをアサートで検証します。 3. SysCache 配列から該当するキャッシュ構造体を取得し、SearchCatCache1 を呼び出して検索を実行します。 4. SearchCatCache1 の戻り値をそのまま呼び出し元へ返します。

関連データ構造

  • SysCache[]: システムキャッシュを保持する CatCache 構造体の配列。
  • HeapTuple: 返されるタプルの型。

失敗モードと境界条件

  • cacheId が不正な場合、アサートにより異常終了します。
  • 指定されたキーに一致するタプルが存在しない場合、NULL を返します。
  • キャッシュのキー数が 1 でない場合、アサートにより異常終了します。

保守担当者向けメモ

  • 本関数が返すタプルはキャッシュ内部のポインタです。呼び出し側で直接変更してはなりません。変更が必要な場合は SearchSysCacheCopy を使用してください。
  • 新しいキャッシュを追加する場合は syscache_info.h を編集し、DECLARE_UNIQUE_INDEX が定義されていることを確認してください。
