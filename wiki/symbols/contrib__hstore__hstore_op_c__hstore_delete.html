hstore_delete

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_op.c:244

目的

hstore 値から指定されたキーを持つエントリを削除し、新しい hstore を生成して返します。

シグネチャまたは形状

Datum hstore_delete(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_HSTORE_P(0): 操作対象の HStore 構造体。
  • PG_GETARG_TEXT_PP(1): 削除対象のキーを表す text 型。
  • **出力**:
  • PG_RETURN_POINTER(out): 指定キーが除外された新しい HStore 構造体。

前提条件と不変条件

  • 入力 HStore はソート済みである必要があります。
  • 出力用のメモリは palloc により確保されます。
  • hstore の内部レイアウト（ヘッダー、エントリ配列、文字列領域）を操作するため、hstore.h で定義されたマクロ（ARRPTR, STRPTR, HS_COUNT 等）に依存します。

アルゴリズムまたは振る舞い

1. 入力 hstore のサイズに基づき、出力用のメモリを確保します。 2. 入力 hstore の全エントリを走査します。 3. 各エントリのキーを比較し、指定されたキーと一致しない場合のみ、HS_COPYITEM マクロを使用して出力用バッファにコピーします。 4. 最後に HS_FINALIZE マクロを呼び出し、最終的なエントリ数とデータサイズを反映させて hstore 構造を確定させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_op.c 内で HSTORE_POLLUTE(hstore_delete, delete) マクロを介して SQL 演算子として登録されています。
  • **呼び出し先**:
  • palloc (メモリ確保)
  • memcmp (キー比較)
  • HS_COPYITEM, HS_FINALIZE (hstore 構造操作マクロ)
  • PG_GETARG_*, PG_RETURN_* (PostgreSQL 関数インターフェース)

関連データ構造

  • HStore: 操作対象の主要データ構造。
  • HEntry: hstore 内の各要素（キーと値のメタデータ）を保持する構造体。

失敗モードと境界条件

  • 入力 hstore が空の場合、そのままの構造がコピーされます。
  • 指定されたキーが hstore 内に存在しない場合、元の hstore と同一の内容が返されます。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数は新しい hstore をメモリ上に構築するため、元のデータは変更されません。
  • HS_FINALIZE は outcount を使用してヘッダー情報を更新するため、ループ後のカウント管理が重要です。
