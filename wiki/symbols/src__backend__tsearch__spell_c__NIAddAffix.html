NIAddAffix

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/spell.c:679

目的

ISpell/Hunspell 辞書コンパイルプロセスにおいて、接辞（Affix）ルールを定義し、IspellDict 構造体の接辞配列に追加する。

シグネチャまたは形状

static void NIAddAffix(IspellDict *Conf, const char *flag, char flagflags, 
                       const char *mask, const char *find, const char *repl, int type)

入力と出力

  • **入力**:
  • Conf: 辞書状態を保持する IspellDict 構造体へのポインタ。
  • flag: 接辞フラグ文字列。
  • flagflags: 接辞の属性フラグ（FF_COMPOUNDONLY 等）。
  • mask: 接辞適用条件（正規表現または単純文字列）。
  • find: 検索対象文字列。
  • repl: 置換後文字列。
  • type: 接辞の種類（FF_SUFFIX または FF_PREFIX）。
  • **出力**: なし（Conf->Affix 配列を更新）。

前提条件と不変条件

  • Conf は初期化済みであること。
  • Conf->Affix 配列の容量が不足している場合、自動的に拡張される。
  • 複合語フラグ（FF_COMPOUNDONLY または FF_COMPOUNDPERMITFLAG）が設定されている場合、内部的に FF_COMPOUNDFLAG が付与される。

アルゴリズムまたは振る舞い

1. **メモリ確保**: Conf->naffixes が Conf->maffixes に達した場合、配列サイズを倍増させる。 2. **マスク処理**:

  • mask が . または空の場合: issimple を有効化。
  • RS_isRegis(mask) が真の場合: RS_compile を用いて regis 形式でコンパイル。
  • それ以外の場合: pg_regcomp を用いて POSIX 正規表現としてコンパイル。コンパイル失敗時は ERROR を発生させる。

    • 3. **データ格納**: フラグ、置換文字列、検索文字列を Conf のメモリコンテキスト内にコピーして格納。 4. **インクリメント**: Conf->naffixes を更新。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/spell.c 内の辞書読み込み処理（NIImportAffixes 等）から呼び出される。
  • **呼び出し先**:
  • repalloc, palloc, tmpalloc: メモリ確保。
  • RS_isRegis, RS_compile: regis 形式の処理。
  • pg_regcomp, pg_regerror: 正規表現コンパイル。
  • cpstrdup: 辞書メモリコンテキストへの文字列コピー。

関連データ構造

  • IspellDict: 辞書全体の状態を保持。
  • AFFIX: 接辞ルールを保持する構造体。

失敗モードと境界条件

  • **メモリ不足**: palloc/repalloc 失敗時に PostgreSQL の標準的なエラーハンドリングが動作する。
  • **正規表現エラー**: mask が不正な正規表現である場合、ERRCODE_INVALID_REGULAR_EXPRESSION で ereport(ERROR) を発行する。

保守担当者向けメモ

  • mask の処理において、regis 形式と正規表現形式の分岐がある。正規表現は DEFAULT_COLLATION_OID を使用してコンパイルされる点に注意。
  • 辞書構築用メモリコンテキストで管理されているため、個別の pfree は不要だが、Conf が破棄される際に一括解放される設計である。
