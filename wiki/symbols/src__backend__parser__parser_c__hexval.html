hexval

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parser.c:327

目的

16進数文字（'0'-'9', 'a'-'f', 'A'-'F'）を対応する整数値（0-15）に変換します。

シグネチャまたは形状

static unsigned int hexval(unsigned char c)

入力と出力

  • **入力**: unsigned char c （変換対象の文字）
  • **出力**: unsigned int （0から15の範囲の数値）

前提条件と不変条件

  • 入力文字 c は有効な16進数文字である必要があります。
  • 無効な文字が渡された場合、関数は値を返さずエラーを発生させます。

アルゴリズムまたは振る舞い

1. 入力文字が '0' から '9' の場合、c - '0' を計算して返します。 2. 入力文字が 'a' から 'f' の場合、c - 'a' + 0xA を計算して返します。 3. 入力文字が 'A' から 'F' の場合、c - 'A' + 0xA を計算して返します。 4. 上記いずれにも該当しない場合、elog(ERROR, ...) を呼び出し、処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parser.c: Unicodeエスケープシーケンスの解析処理において、複数の文字を組み合わせてUnicodeコードポイントを算出するために呼び出されています。
  • src/backend/utils/adt/varlena.c: 外部からの参照が確認されています。
  • **呼び出し先**:
  • elog: エラー発生時に呼び出されます。

関連データ構造

なし。

失敗モードと境界条件

  • **失敗モード**: 入力文字が16進数として不正な場合、elog(ERROR) によりトランザクションまたは解析処理が停止します。
  • **境界条件**: '0', '9', 'a', 'f', 'A', 'F' の各境界値は正しく処理されます。

保守担当者向けメモ

  • 本関数は static であり、主にUnicodeエスケープ処理の補助として使用されています。
  • エラー発生時の挙動は elog(ERROR) に依存しているため、呼び出し側でこの関数を呼ぶ際は、入力が16進数文字であることを事前に保証するか、エラーハンドリングの設計を考慮してください。
