bbsink_lz4_begin_archive

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_lz4.c:131

目的

LZ4圧縮を用いたバックアップアーカイブ処理を開始し、圧縮コンテキストの初期化とフレームヘッダーの書き込みを行います。

シグネチャまたは形状

static void bbsink_lz4_begin_archive(bbsink *sink, const char *archive_name)

入力と出力

  • **入力**:
  • sink: bbsink_lz4 構造体へのポインタ（bbsink としてキャスト）。
  • archive_name: アーカイブのファイル名。
  • **出力**: なし（副作用として圧縮コンテキストの生成と後続シンクへの開始通知を行う）。

前提条件と不変条件

  • sink は bbsink_lz4 型としてキャスト可能であること。
  • sink->bbs_next が存在すること（Assert により検証）。
  • LZ4ライブラリが利用可能であること。

アルゴリズムまたは振る舞い

1. LZ4F_createCompressionContext を呼び出し、圧縮コンテキストを生成する。失敗時はエラーを投げる。 2. LZ4F_compressBegin を使用し、後続シンクのバッファへLZ4フレームヘッダーを書き込む。失敗時はエラーを投げる。 3. 書き込んだヘッダーサイズ分、mysink->bytes_written を更新する。 4. archive_name に .lz4 を付与した文字列を生成する。 5. 後続のシンクに対して bbsink_begin_archive を呼び出し、処理を伝播させる。 6. 生成したファイル名文字列を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_lz4_ops 構造体を通じて、バックアップパイプラインから間接的に呼び出される。
  • **呼び出し先**:
  • LZ4F_createCompressionContext
  • LZ4F_compressBegin
  • bbsink_begin_archive（sink->bbs_next に対して）

関連データ構造

  • bbsink_lz4: 圧縮コンテキストや設定を保持する構造体。
  • bbsink: パイプラインの基底構造体。

失敗モードと境界条件

  • LZ4F_createCompressionContext が失敗した場合、elog(ERROR) により中断する。
  • LZ4F_compressBegin が失敗した場合、elog(ERROR) により中断する。
  • sink->bbs_next が NULL の場合、Assert によりデバッグビルドで停止する。

保守担当者向けメモ

  • 本関数は USE_LZ4 が定義されている環境でのみ有効です。
  • マニフェストデータは圧縮対象外ですが、本関数はアーカイブ開始時に呼び出されるため、圧縮ストリームの初期化に必須です。
