interval_cmp_upper

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistproc.c:326

目的

SplitInterval 構造体の upper フィールドに基づき、2つの間隔を比較してソート順序を決定します。

シグネチャまたは形状

static int interval_cmp_upper(const void *i1, const void *i2)

入力と出力

  • **入力**:
  • i1, i2: 比較対象となる SplitInterval 構造体へのポインタ（const void * として渡される）。
  • **出力**:
  • int: float8_cmp_internal の結果を返します。upper1 < upper2 なら負、upper1 > upper2 なら正、等しければ 0 を返します。

前提条件と不変条件

  • 入力ポインタ i1 および i2 は、有効な SplitInterval 構造体を指している必要があります。
  • SplitInterval 構造体には upper フィールド（float8 型）が存在することが前提です。

アルゴリズムまたは振る舞い

1. 引数 i1, i2 を const SplitInterval * にキャストします。 2. それぞれの構造体から upper フィールドの値を取得します。 3. float8_cmp_internal を呼び出し、取得した2つの値の比較結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistproc.c 内のソート処理（qsort 等の比較関数として使用）。
  • **呼び出し先**:
  • float8_cmp_internal
  • **注記**: src/backend/utils/adt/rangetypes_gist.c に同名の関数が存在しますが、シグネチャが異なるため、本シンボルとは別の実装です。

関連データ構造

  • SplitInterval: 本関数が比較対象とする構造体。

失敗モードと境界条件

  • i1 または i2 が NULL の場合、未定義の動作（セグメンテーション違反）を引き起こします。
  • upper 値が NaN である場合の挙動は float8_cmp_internal の実装に依存します。

保守担当者向けメモ

  • 本関数は GiST インデックスの picksplit アルゴリズムにおけるソート処理の一部として使用されます。
  • src/backend/utils/adt/rangetypes_gist.c に存在する同名の関数と混同しないよう注意してください。
