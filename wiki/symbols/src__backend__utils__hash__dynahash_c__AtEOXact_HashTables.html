AtEOXact_HashTables

定義

  • 種別: function
  • 定義箇所: src/backend/utils/hash/dynahash.c:1914

目的

トランザクション終了時（コミットまたはアボート）に、ハッシュテーブルのシーケンシャルスキャン状態をクリーンアップする。

シグネチャまたは形状

void AtEOXact_HashTables(bool isCommit)

入力と出力

  • **入力**:
  • isCommit (bool): トランザクションがコミットされる場合は true、アボートされる場合は false を指定する。
  • **出力**: なし。

前提条件と不変条件

  • num_seq_scans および seq_scan_tables 配列によって管理されるアクティブなスキャン状態が存在する。
  • コミット時にスキャンが残存している場合、それは hash_seq_term() の呼び出し忘れとみなされる。

アルゴリズムまたは振る舞い

1. isCommit が true の場合、現在アクティブなすべてのシーケンシャルスキャン（num_seq_scans 個）を走査し、WARNING レベルのログを出力してリークを警告する。 2. isCommit が false（アボート）の場合、警告を出さずにスキャン状態を破棄する。 3. 最後に、num_seq_scans を 0 にリセットし、スキャン状態をクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c (トランザクション管理処理)
  • 各種バックグラウンドプロセス（autovacuum.c, bgwriter.c, checkpointer.c, pgarch.c, walsummarizer.c, walwriter.c）の終了処理。
  • **呼び出し先**:
  • elog (警告出力時)

関連データ構造

  • num_seq_scans (グローバル変数: 現在のアクティブなスキャン数)
  • seq_scan_tables (グローバル変数: アクティブなスキャン対象のハッシュテーブルポインタ配列)

失敗モードと境界条件

  • コミット時にスキャンが残存している場合、メモリ安全性を考慮してハッシュテーブル名などの詳細情報は出力せず、ポインタアドレスのみを警告ログに出力する。

保守担当者向けメモ

  • 本関数は hash_seq_term() が呼び出されなかった場合の安全装置として機能する。
  • ログ出力時にハッシュテーブル名を参照すると、既に解放済みのメモリにアクセスするリスクがあるため、ポインタのみを表示する設計となっている。
