OpClassCacheLookup

定義

  • 種別: function
  • 定義箇所: src/backend/commands/opclasscmds.c:161

目的

指定されたアクセスメソッド（AM）と演算子クラス名に基づき、システムカタログから演算子クラスのタプルを検索・取得します。

シグネチャまたは形状

HeapTuple OpClassCacheLookup(Oid amID, List *opclassname, bool missing_ok)

入力と出力

  • **入力**:
  • amID: 検索対象のアクセスメソッドのOID。
  • opclassname: 演算子クラス名を指定するリスト（修飾名または非修飾名）。
  • missing_ok: true の場合、検索失敗時にエラーを発生させず NULL を返します。
  • **出力**:
  • 成功時: 該当する演算子クラスの HeapTuple。
  • 失敗時: missing_ok が true なら NULL、false ならエラーを発生させます。

前提条件と不変条件

  • opclassname は DeconstructQualifiedName で分解可能な形式である必要があります。
  • amID は有効なアクセスメソッドを指している必要があります。

アルゴリズムまたは振る舞い

1. DeconstructQualifiedName を使用して、opclassname をスキーマ名と演算子クラス名に分解します。 2. スキーマ名が指定されている場合:

  • LookupExplicitNamespace でスキーマOIDを特定し、SearchSysCache3（CLAAMNAMENSP）を用いて検索します。

    • 3. スキーマ名が指定されていない場合:

  • OpclassnameGetOpcid を使用して検索パスから演算子クラスOIDを特定し、SearchSysCache1（CLAOID）を用いて検索します。

    • 4. 検索結果が有効でなく、かつ missing_ok が false の場合:

  • SearchSysCache1（AMOID）でアクセスメソッドの存在を確認した上で、ereport(ERROR) を発行して処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/opclasscmds.c 内の演算子クラス操作処理（詳細はソースコード上の直接的な参照関係に基づく）。
  • **呼び出し先**: DeconstructQualifiedName, LookupExplicitNamespace, SearchSysCache3, OpclassnameGetOpcid, SearchSysCache1, HeapTupleIsValid, GETSTRUCT, elog, ereport, NameListToString, NameStr。

関連データ構造

  • HeapTuple: 検索結果として返されるシステムカタログの行データ。
  • Form_pg_am: アクセスメソッドの情報を保持する構造体（エラーメッセージ生成時に使用）。

失敗モードと境界条件

  • 指定された演算子クラスが存在しない場合、missing_ok が false であれば ERRCODE_UNDEFINED_OBJECT エラーが発生します。
  • 指定されたアクセスメソッド自体が存在しない場合、エラーメッセージ生成時に elog(ERROR) が発生します。

保守担当者向けメモ

  • 本関数はシステムキャッシュ（syscache）を利用して高速な検索を行います。新しいキャッシュキーを追加する際は syscache.h との整合性に注意してください。
  • エラーハンドリングにおいて、アクセスメソッドの存在確認を二重に行っているため、カタログの整合性が保たれていることが前提となります。
