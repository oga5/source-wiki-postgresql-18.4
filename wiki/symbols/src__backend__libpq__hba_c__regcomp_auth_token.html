regcomp_auth_token

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/hba.c:302

目的

認証設定ファイル（pg_hba.conf 等）内のトークンが / で始まる場合に、それを正規表現としてコンパイルし、AuthToken 構造体に格納する。

シグネチャまたは形状

static int regcomp_auth_token(AuthToken *token, char *filename, int line_num, char **err_msg, int elevel);

入力と出力

  • **入力**:
  • token: コンパイル対象の文字列を保持する AuthToken 構造体へのポインタ。
  • filename, line_num: エラー報告用のファイル名と行番号。
  • elevel: ereport で使用するエラーレベル。
  • **出力**:
  • 戻り値: 成功時は 0、失敗時は非ゼロの正規表現コンパイルエラーコード。
  • token->regex: コンパイルされた regex_t オブジェクト（成功時）。
  • err_msg: 失敗時にエラーメッセージ文字列が格納される。

前提条件と不変条件

  • token->regex は呼び出し時に NULL であること（Assert で検証）。
  • 文字列が / で始まらない場合、関数は何もせず 0 を返す。

アルゴリズムまたは振る舞い

1. token->string の先頭が / かを確認し、そうでなければ即座に終了する。 2. palloc0 で regex_t 構造体を割り当てる。 3. pg_mb2wchar_with_len を使用し、/ を除いた文字列をワイド文字形式に変換する。 4. pg_regcomp を呼び出し、REG_ADVANCED および C_COLLATION_OID を指定して正規表現をコンパイルする。 5. コンパイルに失敗した場合、pg_regerror でエラー文字列を取得し、ereport でログ出力を行うとともに err_msg を設定する。 6. 変換用の一時バッファ wstr を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/hba.c 内の以下の箇所から呼び出されることが確認されている。
  • 134, 303, 1454, 1479, 2795, 2802 行目。
  • **呼び出し先**: palloc0, palloc, strlen, pg_mb2wchar_with_len, pg_regcomp, pg_regerror, ereport, psprintf, pfree。

関連データ構造

  • AuthToken: トークン文字列とコンパイル済み正規表現を保持する構造体。
  • regex_t: 正規表現コンパイル結果を格納する標準的な構造体。

失敗モードと境界条件

  • 正規表現の構文が不正な場合、pg_regcomp が非ゼロを返し、ereport によるエラー報告と err_msg の設定が行われる。
  • メモリ不足時は palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は pg_hba.conf の正規表現解析において中心的な役割を果たす。
  • / で始まるトークンのみが正規表現として扱われる仕様である。
  • コンパイルされた正規表現は regexec_auth_token で実行される。
