pg_euc2wchar_with_len

定義

  • 種別: function
  • 定義箇所: src/common/wchar.c:107

目的

EUC（Extended Unix Code）エンコーディングのマルチバイト文字列を、PostgreSQL内部で使用されるワイド文字形式（pg_wchar）の配列へ変換します。

シグネチャまたは形状

int pg_euc2wchar_with_len(const unsigned char *from, pg_wchar *to, int len)

入力と出力

  • **入力**:
  • from: 変換元となるEUCエンコーディングの文字列（unsigned char *）。
  • to: 変換後のワイド文字を格納するバッファ（pg_wchar *）。
  • len: 入力文字列のバイト長。
  • **出力**:
  • 戻り値: 変換された文字数（int）。
  • to: 変換結果が格納され、末尾にヌル終端（0）が付与される。

前提条件と不変条件

  • to バッファは、入力文字列から変換される最大文字数分を格納するのに十分なサイズである必要があります。
  • 入力文字列はEUCエンコーディングとして妥当であると仮定されます。

アルゴリズムまたは振る舞い

入力バイト列を順次スキャンし、以下のルールでワイド文字へ変換します。 1. **SS2 (0x8E)**: JIS X 0201（1バイトカナ）として処理。後続の1バイトと合わせ2バイトを消費。 2. **SS3 (0x8F)**: JIS X 0212（漢字）として処理。後続の2バイトと合わせ3バイトを消費。 3. **高ビットセット (0x80以上)**: JIS X 0208（漢字）として処理。後続の1バイトと合わせ2バイトを消費。 4. **その他**: ASCII文字として処理。1バイトを消費。

  • 各ステップで MB2CHAR_NEED_AT_LEAST マクロを使用し、指定されたバイト長が残っているかを確認します。
  • 変換した文字数 cnt をカウントし、最後に to バッファをヌル終端して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/wchar.c 内の他の関数から参照されていることが確認されています（行108, 186, 217）。
  • **呼び出し先**: MB2CHAR_NEED_AT_LEAST マクロを使用しています。

関連データ構造

  • pg_wchar: PostgreSQL内部のワイド文字型。

失敗モードと境界条件

  • len が不足している場合、MB2CHAR_NEED_AT_LEAST マクロの挙動に依存します（通常はエラー処理または中断）。

保守担当者向けメモ

  • 本関数はEUCエンコーディングの構造に強く依存しています。エンコーディングの仕様変更や追加のEUC派生形式を扱う際は、SS2/SS3の判定ロジックを確認してください。
