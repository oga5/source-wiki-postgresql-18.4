JsonToIndex

定義

  • 種別: enum
  • 定義箇所: src/include/utils/jsonfuncs.h:25

目的

JSON または JSONB データの反復処理において、抽出対象とする要素の型を指定するためのビットフラグを提供します。

シグネチャまたは形状

typedef enum JsonToIndex
{
	jtiKey = 0x01,
	jtiString = 0x02,
	jtiNumeric = 0x04,
	jtiBool = 0x08,
	jtiAll = jtiKey | jtiString | jtiNumeric | jtiBool
} JsonToIndex;

入力と出力

  • **入力**: 反復処理関数（iterate_json_values や iterate_jsonb_values 等）の引数として使用されます。
  • **出力**: 指定されたビットフラグに対応する JSON 要素が抽出され、コールバック関数へ渡されます。

前提条件と不変条件

  • 各フラグはビット単位で定義されており、jtiAll はすべてのフラグの論理和として定義されています。
  • 複数の要素を対象とする場合は、ビット単位の論理和（|）を用いて組み合わせることが可能です。

アルゴリズムまたは振る舞い

  • iterate_json_values および iterate_jsonb_values は、この列挙型で指定されたフラグに基づき、JSON 構造内を走査します。
  • 指定された型に一致する要素が見つかった場合、登録されたコールバック関数が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは、具体的な呼び出し元関数は特定できません。
  • **呼び出し先**: 本シンボル自体はデータ構造（列挙型）であり、関数を呼び出すことはありません。

関連データ構造

  • JsonIterateStringValuesAction: 反復処理時に各要素に対して実行されるコールバック関数の型定義。
  • JsonSemAction: 解析時に使用されるセマンティックアクション構造体。

失敗モードと境界条件

  • フラグに 0 を指定した場合、反復処理においてどの要素も抽出対象とならない可能性があります。
  • 未定義のビット値を指定した場合の挙動は、反復処理関数の実装に依存します。

保守担当者向けメモ

  • 本シンボルは jsonfuncs.c で実装される反復処理関数の動作を制御する重要なインターフェースです。
  • 新しい JSON 型や抽出対象を追加する場合は、本列挙型に新しいビットフラグを追加し、対応する反復処理関数のロジックを更新する必要があります。
