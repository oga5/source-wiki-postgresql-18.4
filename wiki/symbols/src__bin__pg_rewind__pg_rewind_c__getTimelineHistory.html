getTimelineHistory

定義

  • **種別**: function
  • **定義箇所**: src/bin/pg_rewind/pg_rewind.c:857

目的

指定されたタイムラインID（TLI）に対応するタイムライン履歴情報を取得し、TimeLineHistoryEntry 配列として返します。

シグネチャまたは形状

TimeLineHistoryEntry *getTimelineHistory(TimeLineID tli, bool is_source, int *nentries)

入力と出力

  • **入力**:
  • tli: 取得対象のタイムラインID。
  • is_source: ソース（true）またはターゲット（false）のどちらから履歴を取得するかを指定するフラグ。
  • nentries: 取得した履歴エントリ数を格納するための出力用ポインタ。
  • **出力**:
  • 成功時: TimeLineHistoryEntry 構造体の配列へのポインタ。
  • 失敗時: 内部でメモリ確保やファイル読み込みに失敗した場合、呼び出し元へ例外が伝播する可能性があります。

前提条件と不変条件

  • tli が 1 の場合、履歴ファイルは存在しないため、開始・終了位置が InvalidXLogRecPtr である単一のエントリが生成されます。
  • tli が 1 以外の場合、TLHistoryFilePath を用いて履歴ファイルのパスが特定され、is_source に応じてソースまたはターゲットからファイル内容が読み込まれます。

アルゴリズムまたは振る舞い

1. tli == 1 の場合、ダミーの履歴エントリを作成して返します。 2. tli > 1 の場合:

  • TLHistoryFilePath でパスを生成します。
  • is_source が true なら source->fetch_file を、false なら slurpFile を使用してファイル内容をメモリに読み込みます。
  • rewind_parseTimeLineHistory を呼び出し、読み込んだ文字列を解析して TimeLineHistoryEntry 配列を構築します。
  • 読み込みに使用したバッファを解放します。

    • 3. debug モードが有効な場合、取得した履歴情報をログに出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_rewind/pg_rewind.c 内のメイン処理（ソースおよびターゲットの履歴取得時）。
  • **呼び出し先**:
  • TLHistoryFilePath（マクロまたは関数）
  • source->fetch_file（ソースがリモートの場合）
  • slurpFile（ターゲットまたはローカルソースの場合）
  • rewind_parseTimeLineHistory
  • pg_malloc / pg_free

関連データ構造

  • TimeLineHistoryEntry: タイムラインの開始・終了位置を保持する構造体。
  • TimeLineID: タイムラインを識別する数値型。

失敗モードと境界条件

  • 履歴ファイルが存在しない、または読み込み権限がない場合、ファイル操作関数（fetch_file または slurpFile）がエラーを発生させます。
  • rewind_parseTimeLineHistory が解析に失敗した場合、処理は中断されます。

保守担当者向けメモ

  • 本関数は pg_rewind の初期化フェーズにおいて、クラスタの分岐点を特定するために不可欠な履歴情報を供給します。
  • is_source フラグによるソースの抽象化（libpq 経由かローカルファイルか）に依存しているため、ソース取得ロジックを変更する際は注意が必要です。
