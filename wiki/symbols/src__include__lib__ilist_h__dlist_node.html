dlist_node

定義

  • 種別: struct
  • 定義箇所: src/include/lib/ilist.h:136

目的

PostgreSQL の「統合型（Integrated）」二重連結リストにおいて、リストのリンク情報（前後のノードへのポインタ）を保持するために使用されます。データ構造の内部に直接埋め込むことで、ノードごとの動的なメモリ割り当てを不要にします。

シグネチャまたは形状

typedef struct dlist_node dlist_node;

※構造体の内部フィールドは ilist.h 内で定義されており、双方向の走査を可能にするためのポインタを含みます。

入力と出力

本シンボルはデータ構造の一部として機能します。

  • **入力**: リスト操作関数（dlist_delete 等）に対して、ノードのポインタが渡されます。
  • **出力**: リスト操作の結果として、内部のリンクポインタが更新されます。

前提条件と不変条件

  • **メモリ管理**: dlist_node がリストに含まれている間、そのノードを含む構造体自体を free() してはいけません。
  • **整合性**: ILIST_DEBUG が定義されている場合、リストの整合性チェックが有効になります。

アルゴリズムまたは振る舞い

  • **埋め込み**: ユーザー定義の構造体にフィールドとして埋め込まれます。
  • **走査**: dlist_head から next ポインタを辿ることでリストを走査します。
  • **削除**: dlist_delete を呼び出すことで、ノードの前後ポインタを直接繋ぎ変え、定数時間でリストから取り外すことが可能です。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • 呼び出し先に関する具体的な情報は提供されていません。

関連データ構造

  • dlist_head: リストの起点となる構造体。
  • dlist_iter / dlist_mutable_iter: リスト走査および走査中の安全な削除を行うためのイテレータ。

失敗モードと境界条件

  • **空リスト**: dlist は NULL または「自身を指す（循環）」状態を空として扱います。循環リストの方が分岐が少なく高速に動作します。

保守担当者向けメモ

  • 本シンボルは ilist.h の基礎となる構造体です。変更を加える際は、dlist を利用するすべてのバックエンドコードへの影響を考慮してください。
  • 構造体内のポインタ操作はインライン化されていることが多いため、デバッグ時には src/backend/lib/ilist.c にある検証関数（dlist_check 等）を活用してください。
