yyscan_t

定義

  • 種別: typedef
  • 定義箇所: src/include/bootstrap/bootstrap.h:61

目的

PostgreSQLのブートストラップモードおよび各種拡張モジュール（cube, seg 等）で使用される、Flex/Bison生成パーサーのコンテキストを保持するための不透明ポインタ（opaque pointer）です。

シグネチャまたは形状

typedef void *yyscan_t;

入力と出力

  • **入力**: パーサーの初期化関数（例: cube_scanner_init）を通じて、スキャナの状態を保持するポインタとして渡されます。
  • **出力**: スキャナの終了関数（例: cube_scanner_finish）に渡され、リソースの解放に使用されます。

前提条件と不変条件

  • void * 型として定義されており、具体的な構造体定義はFlexが生成するコードに依存します。
  • パーサー関数（yyparse や yylex）を呼び出す際、初期化済みの有効な yyscan_t インスタンスを渡す必要があります。

アルゴリズムまたは振る舞い

  • スキャナの再入可能（reentrant）な状態を保持します。
  • 複数のパーサーインスタンスが並行して存在する場合でも、各スキャナの状態を分離して管理するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/cube/cube.c や contrib/seg/seg.c などの拡張モジュール、およびブートストラップ処理のパーサー（boot_yyparse）。
  • **呼び出し先**: ツールによる解析では、特定の関数呼び出しの直接的な対象ではなく、パーサー関連関数（yylex, yyparse, scanner_init, scanner_finish）の引数として受け渡されます。

関連データ構造

  • YYSTYPE: パーサーのセマンティック値型。
  • bootscanner.l / bootparse.y: ブートストラップ用パーサーのソースコード。

失敗モードと境界条件

  • NULL ポインタをパーサー関数に渡した場合の挙動は、各パーサーの実装に依存します。
  • 初期化されていない、または解放済みの yyscan_t を使用した場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本シンボルはFlexが生成するコードと密接に関連しています。bootstrap.h での定義は、ブートストラップ用パーサーと他の拡張モジュール間で共通の型インターフェースとして機能します。
  • 拡張モジュール（cube, seg）での利用例が確認されていますが、これらは本ヘッダーファイルで定義された型を共有していることを示唆しています。
