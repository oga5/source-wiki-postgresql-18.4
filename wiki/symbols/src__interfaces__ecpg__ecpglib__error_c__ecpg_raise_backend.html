ecpg_raise_backend

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/error.c:218

目的

PostgreSQLサーバー（バックエンド）から返された PGresult を解析し、エラー情報を sqlca（SQL Communication Area）構造体に反映させることで、ECPGアプリケーションへエラーを通知します。

シグネチャまたは形状

void ecpg_raise_backend(int line, PGresult *result, PGconn *conn, int compat)

入力と出力

  • **入力**:
  • line: エラー発生元のソースコード行番号。
  • result: libpq から返された PGresult オブジェクト。
  • conn: データベース接続オブジェクト PGconn。
  • compat: 互換モード（Informix等）を指定するフラグ。
  • **出力**: なし（void）。副作用として sqlca 構造体の更新およびメモリ解放が行われます。

前提条件と不変条件

  • sqlca 構造体が存在すること。取得できない場合はエラーログを出力して即座に終了します。
  • エラー処理の最後に ECPGfree_auto_mem() が呼び出され、一時的に確保されたメモリが解放されます。

アルゴリズムまたは振る舞い

1. ECPGget_sqlca() で sqlca 構造体を取得します。 2. PQresultErrorField を使用して、PGresult から SQLSTATE およびエラーメッセージを取得します。取得できない場合は PQerrorMessage にフォールバックします。 3. 内部エラーかつ接続が切断されている場合、SQLSTATE を 57P02 に、メッセージを「サーバーへの接続が失われました」に書き換えます。 4. sqlca.sqlerrm にエラーメッセージと行番号を格納し、sqlca.sqlstate を更新します。 5. sqlca.sqlcode を、互換モードを考慮した適切なエラーコードに設定します。 6. ecpg_log を通じてエラー詳細をログ出力します。 7. ECPGfree_auto_mem() を呼び出し、メモリをクリーンアップします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/error.c 内の各関数。
  • src/interfaces/ecpg/ecpglib/execute.c 内の ecpg_check_PQresult 等。
  • **呼び出し先**:
  • ECPGget_sqlca()
  • PQresultErrorField()
  • PQerrorMessage()
  • PQstatus()
  • ecpg_gettext()
  • ecpg_log()
  • ECPGfree_auto_mem()

関連データ構造

  • sqlca_t: エラー情報を保持する構造体。
  • PGresult, PGconn: libpq のデータ構造。

失敗モードと境界条件

  • sqlca が取得できない場合、ログを出力して処理を中断します。
  • result が NULL の場合、PQresultErrorField は NULL を返し、PQerrorMessage を使用する安全なフォールバック処理が行われます。

保守担当者向けメモ

  • 本関数は ecpglib のエラー報告の最終出口です。
  • メッセージの変更時は ecpg_gettext による国際化対応を維持してください。
  • エラー発生時に ECPGfree_auto_mem() が呼ばれるため、本関数の呼び出し後にメモリを参照し続ける実装は避ける必要があります。
