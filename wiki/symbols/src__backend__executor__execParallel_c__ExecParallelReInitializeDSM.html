ExecParallelReInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execParallel.c:964

目的

並列クエリ実行において、プランノードが使用する動的共有メモリ（DSM）の状態を再初期化し、クエリの再実行に備える。

シグネチャまたは形状

static bool ExecParallelReInitializeDSM(PlanState *planstate, ParallelContext *pcxt)

入力と出力

  • planstate: 再初期化対象のプランノード状態（PlanState）。
  • pcxt: 並列実行コンテキスト（ParallelContext）。
  • 戻り値: planstate_tree_walker の結果を返す。

前提条件と不変条件

  • planstate が NULL の場合は即座に false を返す。
  • parallel_aware フラグが真であるノードに対してのみ、各ノード固有の再初期化関数が呼び出される。

アルゴリズムまたは振る舞い

1. planstate が NULL であれば終了する。 2. nodeTag(planstate) を用いてノード種別を判定する。 3. T_SeqScanState, T_IndexScanState, T_IndexOnlyScanState, T_ForeignScanState, T_AppendState, T_CustomScanState, T_BitmapHeapScanState, T_HashJoinState のいずれかであり、かつ parallel_aware が真であれば、対応する Exec*ReInitializeDSM 関数を呼び出す。 4. T_BitmapIndexScanState, T_HashState, T_SortState, T_IncrementalSortState, T_MemoizeState はDSM状態を持つが、再初期化は不要であるため何もしない。 5. planstate_tree_walker を再帰的に呼び出し、プランツリー全体を走査する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/executor/execParallel.c 内の並列実行管理ロジックから呼び出される（ソース内の直接的な呼び出し箇所は 957 行目）。
  • 呼び出し先:
  • 各種ノード固有の Exec*ReInitializeDSM 関数。
  • planstate_tree_walker（再帰呼び出し）。

関連データ構造

  • PlanState: プランノードの実行状態。
  • ParallelContext: 並列実行の共有メモリコンテキスト。

失敗モードと境界条件

  • planstate が NULL の場合、安全に false を返す。
  • parallel_aware が偽のノードについては、再初期化処理をスキップする。

保守担当者向けメモ

  • 新しい並列対応ノードを追加する場合、本関数の switch 文に該当ノードのケースを追加し、適切な再初期化関数を呼び出す必要がある。
  • planstate_tree_walker を使用しているため、プランツリーの構造変更には注意すること。
