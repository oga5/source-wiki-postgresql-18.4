bbsink_zstd_archive_contents

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_zstd.c:192

目的

bbsink パイプラインを通じて渡されたデータを Zstandard (zstd) 形式で圧縮し、後続のシンクへ転送する。

シグネチャまたは形状

static void bbsink_zstd_archive_contents(bbsink *sink, size_t len)

入力と出力

  • **入力**:
  • sink: bbsink_zstd 構造体へのポインタ。
  • len: 圧縮対象となる sink->base.bbs_buffer 内のデータ長。
  • **出力**: なし（副作用として後続のシンクへ圧縮済みデータを転送）。

前提条件と不変条件

  • sink は bbsink_zstd 型としてキャスト可能であること。
  • mysink->cctx（ZSTD圧縮コンテキスト）が初期化済みであること。
  • mysink->zstd_outBuf が出力先として適切に設定されていること。

アルゴリズムまたは振る舞い

1. 入力バッファ inBuf を作成し、全データが処理されるまでループする。 2. ZSTD_compressBound を用いて、残りの入力データに対して必要な最大出力バッファサイズを計算する。 3. 現在の出力バッファ zstd_outBuf に十分な空きがない場合、bbsink_archive_contents を呼び出して後続のシンクへデータをフラッシュし、バッファをリセットする。 4. ZSTD_compressStream2 を呼び出し、ZSTD_e_continue モードでストリーム圧縮を実行する。 5. 圧縮処理がエラーを返した場合は elog(ERROR) で処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_zstd_ops 構造体を通じて、バックアップ処理のパイプラインから間接的に呼び出される。
  • **呼び出し先**:
  • bbsink_archive_contents（後続シンクへのデータ転送）
  • ZSTD_compressStream2（libzstd による圧縮）
  • ZSTD_isError / ZSTD_getErrorName（エラー判定）

関連データ構造

  • bbsink_zstd: 圧縮状態とバッファを保持する構造体。
  • ZSTD_inBuffer / ZSTD_outBuffer: libzstd が使用する入出力バッファ構造体。
  • bbsink: 基底となるシンクインターフェース。

失敗モードと境界条件

  • ZSTD_compressStream2 がエラーを返した場合、elog(ERROR) によりバックアップ処理全体が異常終了する。
  • 出力バッファの容量不足時には、自動的に後続シンクへのフラッシュが行われる。

保守担当者向けメモ

  • 本関数は USE_ZSTD が定義されている環境でのみ有効。
  • bbsink_zstd_ops の関数ポインタテーブルに登録されており、パイプラインのデータフローにおいて重要な中間処理を担う。
