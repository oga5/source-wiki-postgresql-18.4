ExecBitmapIndexScanEstimate

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapIndexscan.c:352

目的

並列クエリ実行時、BitmapIndexScan ノードの計測情報（Instrumentation）を格納するために必要な共有メモリ領域のサイズを見積もり、動的共有メモリ（DSM）の推定器に登録する。

シグネチャまたは形状

void ExecBitmapIndexScanEstimate(BitmapIndexScanState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: BitmapIndexScanState 構造体へのポインタ。計測情報（instrument）の有無を確認するために使用。
  • pcxt: ParallelContext 構造体へのポインタ。並列ワーカー数およびDSM推定器（estimator）へのアクセスに使用。
  • **出力**: なし（副作用として pcxt->estimator を更新）。

前提条件と不変条件

  • BitmapIndexScan 自体は並列スキャンをサポートしていない。
  • 本関数は、並列クエリ実行中に計測情報のみをDSM経由で共有するために呼び出される。
  • node->ss.ps.instrument が NULL であるか、または pcxt->nworkers が 0 の場合は、何も行わず即座に復帰する。

アルゴリズムまたは振る舞い

1. instrument フィールドの有無と並列ワーカー数を確認し、条件を満たさない場合は早期終了する。 2. SharedIndexScanInstrumentation 構造体のサイズを計算する。これには、固定部分（offsetof）と、ワーカー数に応じた IndexScanInstrumentation 配列のサイズが含まれる。 3. shm_toc_estimate_chunk を呼び出し、計算したサイズをDSMの推定器に登録する。 4. shm_toc_estimate_keys を呼び出し、共有メモリのキーを1つ確保する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:260 から呼び出されることが確認されている。
  • **呼び出し先**:
  • shm_toc_estimate_chunk
  • shm_toc_estimate_keys

関連データ構造

  • BitmapIndexScanState: スキャンノードの状態管理構造体。
  • ParallelContext: 並列クエリの実行コンテキスト。
  • SharedIndexScanInstrumentation: 並列ワーカー間で共有される計測情報の構造体。
  • IndexScanInstrumentation: 個別のワーカーの計測情報。

失敗モードと境界条件

  • pcxt->nworkers が 0 の場合、共有メモリの確保は行われない。
  • node->ss.ps.instrument が NULL の場合、計測情報の共有は行われない。

保守担当者向けメモ

  • 本関数は並列クエリにおける統計情報の集約を目的としており、スキャン自体の並列実行を有効にするものではない点に注意すること。
