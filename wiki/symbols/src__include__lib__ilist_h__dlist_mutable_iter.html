dlist_mutable_iter

定義

  • 種別: struct
  • 定義箇所: src/include/lib/ilist.h:198

目的

二重連結リスト（dlist）の走査中に、現在のノードを安全に削除することを可能にするためのイテレータ構造体です。

シグネチャまたは形状

typedef struct dlist_mutable_iter
{
	dlist_node *cur;    /* 現在の要素 */
	dlist_node *next;   /* 次に走査するノード */
	dlist_node *end;    /* 走査を終了するノード */
} dlist_mutable_iter;

入力と出力

  • 入力: リスト走査の開始時に、リストのヘッド情報を用いて初期化されます。
  • 出力: 走査の各ステップで cur が現在のノードを指し、next が次のノードを保持します。

前提条件と不変条件

  • 走査中にノードを削除しても、next ポインタが保持されているため、イテレータはリストの整合性を維持したまま次の要素へ進むことができます。
  • リストノードは構造体の一部として埋め込まれているため、ノードがリストに含まれている間は、その構造体自体を free() してはなりません。

アルゴリズムまたは振る舞い

  • dlist_foreach_modify マクロと組み合わせて使用されます。
  • イテレータは、現在のノード（cur）を処理する前に、あらかじめ「次のノード（next）」をキャッシュします。これにより、cur がリストから削除された場合でも、走査を中断することなく継続可能です。

  • 呼び出し先: 構造体定義のみのため、直接的な関数呼び出し先はありません。

関連データ構造

  • dlist_node: リストのリンクを保持する構造体。
  • dlist_head: リストの起点。
  • dlist_iter: 削除を伴わない通常の走査用イテレータ。

失敗モードと境界条件

  • リストが空の場合、イテレータは適切に終了条件を満たす必要があります。
  • 走査中にリストの構造が外部から破壊的に変更された場合、イテレータの next ポインタが不正になる可能性があります。

保守担当者向けメモ

  • ILIST_DEBUG を定義することで、リストの整合性チェックを有効化できます。
  • 本構造体はスタック上に確保して使用することを想定しています。
