CreateEventTrigger

定義

  • 種別: function
  • 定義箇所: src/backend/commands/event_trigger.c:123

目的

新しいイベントトリガーをデータベースに登録する。

シグネチャまたは形状

Oid CreateEventTrigger(CreateEventTrigStmt *stmt)

入力と出力

  • **入力**: CreateEventTrigStmt *stmt (作成するイベントトリガーの定義情報を含む構造体)
  • **出力**: Oid (作成されたイベントトリガーのオブジェクトID)

前提条件と不変条件

  • 実行ユーザーはスーパーユーザーである必要がある。
  • 指定されたイベント名（ddl_command_start, ddl_command_end, sql_drop, login, table_rewrite のいずれか）が有効であること。
  • 同名のイベントトリガーが既に存在しないこと。
  • 指定されたトリガー関数が存在し、かつ戻り値の型が event_trigger であること。

アルゴリズムまたは振る舞い

1. **権限チェック**: superuser() を呼び出し、スーパーユーザーでない場合はエラーを報告する。 2. **イベント名検証**: stmt->eventname がサポートされているイベント名か確認する。 3. **フィルタ条件解析**: stmt->whenclause を走査し、tag フィルタを抽出する。 4. **タグ検証**: イベントの種類に応じて validate_ddl_tags または validate_table_rewrite_tags を呼び出し、タグリストを検証する。login イベントでのタグ指定はエラーとする。 5. **重複チェック**: EVENTTRIGGERNAME システムキャッシュを検索し、同名のトリガーが存在すればエラーを報告する。 6. **関数検証**: LookupFuncName でトリガー関数を特定し、戻り値の型が EVENT_TRIGGEROID であることを確認する。 7. **カタログ登録**: insert_event_trigger_tuple を呼び出し、システムカタログにエントリを追加してその OID を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c (DDLコマンド実行フローから呼び出される)
  • **呼び出し先**: superuser, error_duplicate_filter_variable, validate_ddl_tags, validate_table_rewrite_tags, SearchSysCache1, LookupFuncName, get_func_rettype, insert_event_trigger_tuple

関連データ構造

  • CreateEventTrigStmt: イベントトリガー作成文の構文解析結果を保持する構造体。
  • EVENTTRIGGERNAME: イベントトリガー名で検索するためのシステムキャッシュID。
  • EVENT_TRIGGEROID: イベントトリガー関数の戻り値として期待される型ID。

失敗モードと境界条件

  • **権限不足**: スーパーユーザー以外が実行した場合、ERRCODE_INSUFFICIENT_PRIVILEGE で失敗する。
  • **不正なイベント名**: 未定義のイベント名を指定した場合、ERRCODE_SYNTAX_ERROR で失敗する。
  • **重複定義**: 同名のトリガーが存在する場合、ERRCODE_DUPLICATE_OBJECT で失敗する。
  • **型不一致**: トリガー関数が event_trigger 型を返さない場合、ERRCODE_INVALID_OBJECT_DEFINITION で失敗する。
  • **サポート外のフィルタ**: login イベントでタグフィルタを指定した場合、ERRCODE_FEATURE_NOT_SUPPORTED で失敗する。

保守担当者向けメモ

  • 本関数は src/include/commands/event_trigger.h で宣言されている公開インターフェースである。
  • カタログ操作は insert_event_trigger_tuple に委譲されているため、カタログスキーマの変更時はそちらも確認すること。
