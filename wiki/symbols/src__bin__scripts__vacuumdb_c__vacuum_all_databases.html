vacuum_all_databases

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/vacuumdb.c:1068

目的

PostgreSQL インスタンス内のすべてのデータベースに対して、指定されたオプションで VACUUM または ANALYZE 操作を順次実行する。

シグネチャまたは形状

static void vacuum_all_databases(ConnParams *cparams,
                                 vacuumingOptions *vacopts,
                                 bool analyze_in_stages,
                                 SimpleStringList *objects,
                                 int concurrentCons,
                                 const char *progname, bool echo, bool quiet)

入力と出力

  • **入力**:
  • cparams: 接続パラメータ。
  • vacopts: 実行オプション。
  • analyze_in_stages: 統計情報の段階的更新を行うかどうかのフラグ。
  • objects: 処理対象のオブジェクトリスト。
  • concurrentCons: 並列接続数。
  • progname, echo, quiet: 実行時のログ出力制御用パラメータ。
  • **出力**: なし（副作用として各データベースへの接続とコマンド実行を行う）。

前提条件と不変条件

  • pg_database カタログへのアクセス権限が必要。
  • 接続可能なデータベース（datallowconn が真かつ datconnlimit <> -2）のみが対象となる。

アルゴリズムまたは振る舞い

1. connectMaintenanceDatabase を使用してメンテナンス用データベースに接続し、対象となる全データベース名を取得する。 2. analyze_in_stages が真の場合:

  • 統計情報の段階的更新（ANALYZE_NUM_STAGES 回のループ）を実行する。各ステージで全データベースを順次処理する。

    • 3. analyze_in_stages が偽の場合:

  • 各データベースに対して vacuum_one_database を一度だけ呼び出す。

    • 4. 各データベースの処理において、cparams->override_dbname を対象データベース名に書き換え、vacuum_one_database を呼び出す。 5. 最後に取得したデータベースリスト（PGresult）を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/scripts/vacuumdb.c 内の main 関数から呼び出される。
  • **呼び出し先**:
  • connectMaintenanceDatabase
  • executeQuery
  • PQfinish, PQntuples, PQgetvalue, PQclear
  • vacuum_one_database

関連データ構造

  • ConnParams: 接続設定。
  • vacuumingOptions: VACUUM/ANALYZE の実行設定。
  • SimpleStringList: 処理対象オブジェクトのリスト。
  • PGconn, PGresult: libpq の接続および結果保持構造体。

失敗モードと境界条件

  • データベースリストの取得に失敗した場合、プログラムは終了する可能性がある（executeQuery の挙動に依存）。
  • 接続制限や権限不足により特定のデータベースへの接続が失敗した場合、vacuum_one_database 内の処理に依存する。

保守担当者向けメモ

  • 本関数はデータベースのリストを取得するために一時的な接続を確立し、その後 vacuum_one_database で個別に再接続を行う設計となっている。
  • analyze_in_stages が有効な場合、データベース数 × ステージ数の接続が発生するため、接続数制限に注意が必要。
