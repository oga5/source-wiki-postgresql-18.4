date_test_fmt

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/compat_informix-rfmtdate.c:76

目的

内部日付表現を特定のフォーマット文字列に従って文字列へ変換し、その結果を標準出力へ表示する回帰テスト用関数。

シグネチャまたは形状

static void date_test_fmt(date d, const char *fmt)

入力と出力

  • **入力**:
  • d: 変換対象の date 型データ。
  • fmt: フォーマットを指定する文字列（例: "mmddyy", "yyyy/mm/dd" など）。
  • **出力**: なし（標準出力へ結果を表示）。

前提条件と不変条件

  • 内部的に静的変数 i を保持しており、呼び出されるたびにインクリメントされる。
  • rfmtdate が正常終了（戻り値 0）した場合のみ、変換後の文字列が i の値と共に表示される。

アルゴリズムまたは振る舞い

1. rfmtdate(d, fmt, buf) を呼び出し、日付 d を fmt に基づいて文字列 buf に変換する。 2. 変換結果の戻り値 r を確認する。 3. r != 0 の場合、check_return(r) を呼び出しエラー情報を出力する。 4. r == 0 の場合、printf を使用して現在のカウンタ i と変換結果 buf を標準出力へ出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内のテスト実行箇所（main 関数等から、様々なフォーマット文字列を指定して複数回呼び出される）。
  • **呼び出し先**:
  • rfmtdate: 日付変換を実行する Informix 互換ライブラリ関数。
  • check_return: エラー発生時に呼び出されるヘルパー関数。
  • printf: 標準出力用関数。

関連データ構造

  • date: ECPG で使用される日付型。

失敗モードと境界条件

  • rfmtdate が 0 以外の値を返した場合、check_return を介してエラーコードが処理される。
  • フォーマット文字列 fmt が不正な場合や、変換対象の日付 d が無効な場合に rfmtdate が失敗する可能性がある。

保守担当者向けメモ

  • 本関数は ECPG の Informix 互換モードの回帰テストの一部です。テストケースを追加する際は、main 関数内の呼び出しリストに新しいフォーマットパターンを追加してください。
