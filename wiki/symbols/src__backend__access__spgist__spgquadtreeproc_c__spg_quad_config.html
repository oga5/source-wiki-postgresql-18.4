spg_quad_config

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/spgist/spgquadtreeproc.c:26

目的

SP-GiSTインデックスにおける四分木（Quadtree）の実装において、インデックスのメタデータ設定を行う。

シグネチャまたは形状

Datum spg_quad_config(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（spgConfigIn 構造体へのポインタを含む）
  • **出力**: void（spgConfigOut 構造体の各フィールドを更新することで設定を反映）

前提条件と不変条件

  • PG_GETARG_POINTER(1) を介して spgConfigOut 構造体への有効なポインタが取得可能であること。

アルゴリズムまたは振る舞い

1. spgConfigOut 構造体の prefixType を POINTOID に設定し、点データ用インデックスであることを宣言する。 2. labelType を VOIDOID に設定し、ノードラベルが不要であることを指定する。 3. canReturnData を true に設定し、インデックスから元のデータを取得可能にする。 4. longValuesOK を false に設定し、長い値を許容しない構成とする。 5. PG_RETURN_VOID() を呼び出し終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiSTのインデックス構築または初期化プロセス（具体的な呼び出し元関数名はソースコードの文脈からは特定不可）。
  • **呼び出し先**: なし。

関連データ構造

  • spgConfigOut: インデックスの構成情報を保持する構造体。
  • POINTOID, VOIDOID: PostgreSQLの型識別子。

失敗モードと境界条件

  • 引数として渡されるポインタが不正な場合、メモリ保護違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数はSP-GiSTのプラグインインターフェースの一部であり、インデックスの特性を定義する重要な役割を持つ。設定変更時は access/spgist.h の定義と整合性を取ること。
