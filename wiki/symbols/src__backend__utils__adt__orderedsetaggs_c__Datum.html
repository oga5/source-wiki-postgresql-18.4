Datum

定義

  • 種別: typedef
  • 定義箇所: src/backend/utils/adt/orderedsetaggs.c:500（および postgres.h:69）

目的

PostgreSQLのバックエンドにおいて、任意のデータ型を汎用的に扱うための内部表現型です。

シグネチャまたは形状

typedef uintptr_t Datum; （※ postgres.h での定義に基づく）

入力と出力

  • 入力: 任意のデータ型の値（またはポインタ）。
  • 出力: 内部表現としての uintptr_t 値。

前提条件と不変条件

  • Datum は、値が「値渡し（by-value）」可能な型の場合はその値そのものを保持し、「参照渡し（by-reference）」可能な型の場合はデータへのポインタを保持します。
  • 参照渡しの場合、データはメモリ上で適切にアライメントされている必要があります。

アルゴリズムまたは振る舞い

  • 実行エンジンにおいて、型を問わず統一的に値を保持・受け渡しするために使用されます。
  • orderedsetaggs.c 内では、LerpFunc（線形補間関数）の引数や戻り値として、ソートされたデータセットから抽出された値の受け渡しに利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/amcheck 配下の各検証ツールや、PostgreSQLの集約関数エンジン全般。
  • 呼び出し先: 特定のデータ型に対する操作関数（fmgr を介した型固有の関数）。
  • 注記: orderedsetaggs.c 内での具体的な呼び出し関係は、本ソース抜粋からは特定できません。

関連データ構造

  • LerpFunc: Datum を引数に取る線形補間関数の関数ポインタ型。
  • OSAPerGroupState: 集約処理中に Datum を含むタプルを保持する tuplesort を管理します。

失敗モードと境界条件

  • 参照渡し型の Datum を扱う際、対象のメモリ領域が解放された後にアクセスすると不正なメモリ参照が発生します。
  • Datum が保持する値が NULL かどうかは、別途 bool 型のフラグで管理する必要があります。

保守担当者向けメモ

  • orderedsetaggs.c における Datum は、主に tuplesort を介したソート対象の値として扱われます。
  • postgres.h で定義されるグローバルな型であるため、本ファイル固有の定義ではなく、システム共通の規約に従って取り扱うこと。
