TimeLineHistoryCmd

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/replnodes.h:117

目的

レプリケーションプロトコルにおける TIMELINE_HISTORY コマンドのパラメータを保持するためのパースノードです。

シグネチャまたは形状

typedef struct TimeLineHistoryCmd
{
	NodeTag		type;
	TimeLineID	timeline;
} TimeLineHistoryCmd;

入力と出力

  • **入力**: 構文解析器がレプリケーションコマンドを解釈する際、対象となるタイムラインIDが timeline フィールドに格納されます。
  • **出力**: 本構造体は実行エンジンへコマンド情報を引き渡すためのデータコンテナとして機能します。

前提条件と不変条件

  • type フィールドには、PostgreSQLのノードシステムにおける適切な NodeTag が設定されている必要があります。
  • timeline は access/xlogdefs.h で定義された TimeLineID 型である必要があります。

アルゴリズムまたは振る舞い

本構造体はデータ保持専用のノードです。構文解析器によってインスタンス化され、その後、実行エンジンがこの構造体を参照してタイムライン履歴の送信処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walsender.c 内の処理において、コマンドノードとしてキャストまたは参照されます。
  • **呼び出し先**: src/backend/replication/walsender.c の SendTimeLineHistory 関数が、本構造体を引数として受け取り処理を実行します。

関連データ構造

  • NodeTag: ノードの種類を識別するためのタグ。
  • TimeLineID: access/xlogdefs.h で定義された、タイムラインを識別するための型。

失敗モードと境界条件

  • 構造体内の timeline が無効な値である場合、後続の SendTimeLineHistory 実行時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 新しいレプリケーションコマンドを追加する際は、本構造体と同様に NodeTag を先頭に配置してください。
  • 本構造体をコピーまたはデバッグ出力する必要がある場合は、src/backend/nodes/copyfuncs.c および outfuncs.c への対応する実装が必要です。
