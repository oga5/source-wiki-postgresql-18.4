cust_ind

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-array_of_struct.c:43

目的

ECPG（Embedded SQL for C）において、データベース操作時に構造体メンバの NULL 値やデータ状態を管理するためのインジケータ変数用構造体。

シグネチャまたは形状

typedef struct ind {
    short name_ind;
    short phone_ind;
} cust_ind;

入力と出力

  • 入力: SQL 操作（SELECT 等）の結果として得られる各列の状態（NULL か否か等）。
  • 出力: C 言語側で参照可能な short 型のインジケータ値。

前提条件と不変条件

  • 構造体内の各メンバは short 型であること。
  • ECPG プリプロセッサが生成するコードにおいて、対応するホスト変数構造体と対になって使用されること。

アルゴリズムまたは振る舞い

データベースからフェッチした値が NULL である場合、対応するインジケータ変数（name_ind や phone_ind）に負の値が設定されることで、アプリケーション側で NULL 判定を可能にする。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/test/expected/preproc-array_of_struct.c 内で cust_ind inds[10] として配列宣言され、ECPGdo 呼び出し時に使用される。
  • 呼び出し先: 情報なし。

関連データ構造

  • customer: 本構造体と対になるデータ保持用構造体。
  • sqlca: SQL 通信領域。エラーハンドリングに使用される。

失敗モードと境界条件

  • 構造体のメンバ順序と、SQL 文で指定される列の順序が一致していない場合、インジケータ変数が誤った列の状態を反映する可能性がある。

保守担当者向けメモ

  • 本構造体は ECPG のテスト用生成コードの一部であるため、直接編集しないこと。
  • preproc-pointer_to_struct.c でも同名の構造体が定義されており、テストスイート間での重複定義に注意が必要。
