CopyFromStateData

定義

  • **種別**: struct
  • **定義箇所**: src/include/commands/copyfrom_internal.h:59

目的

COPY FROM コマンドの実行中における全状態を保持・管理する中心的なデータ構造です。データソースからの読み込み、解析、変換、およびテーブルへの挿入処理に必要なメタデータとバッファをカプセル化します。

シグネチャまたは形状

typedef struct CopyFromStateData {
    /* フォーマットルーチンおよび低レベル状態 */
    const struct CopyFromRoutine *routine;
    CopySource copy_src;
    FILE *copy_file;
    StringInfo fe_msgbuf;
    EolType eol_type;
    int file_encoding;
    bool need_transcoding;
    Oid conversion_proc;

    /* COPYコマンドパラメータ */
    Relation rel;
    List *attnumlist;
    char *filename;
    bool is_program;
    copy_data_source_cb data_source_cb;
    CopyFormatOptions opts;
    bool *convert_select_flags;
    Node *whereClause;

    /* エラーメッセージ用メタデータ */
    const char *cur_relname;
    uint64 cur_lineno;
    const char *cur_attname;
    const char *cur_attval;
    bool relname_only;

    /* 実行状態および作業用バッファ */
    MemoryContext copycontext;
    /* ... (型変換情報、デフォルト値式、バッファ管理用メンバ) ... */
    StringInfoData attribute_buf;
    StringInfoData line_buf;
    char *input_buf;
    char *raw_buf;
    uint64 bytes_processed;
} CopyFromStateData;

入力と出力

  • **入力**: COPY コマンドで指定されたソース（ファイル、プログラム、またはフロントエンド）から raw_buf を介して生データを受け取ります。
  • **出力**: 解析されたデータは attribute_buf に格納され、最終的にテーブルへ挿入されます。エラー発生時は cur_* フィールドを使用してコンテキスト情報が出力されます。

前提条件と不変条件

  • input_buf および raw_buf は、常にバッファ末尾に \0 が存在することが保証されます。
  • attribute_buf および line_buf は、処理サイクル間で再利用され、メモリ割り当てのオーバーヘッドを削減します。
  • 構造体が巨大であるため、関数間での受け渡しには必ずポインタ型である CopyFromState を使用する必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: ソースタイプやフォーマットオプションに基づき状態を初期化します。 2. **読み込み**: raw_buf に生データを読み込み、必要に応じてエンコーディング変換を経て input_buf に格納します。 3. **解析**: line_buf に行単位でデータを切り出し、attribute_buf を用いて各フィールドを抽出します。 4. **変換・挿入**: in_functions 等を用いて Datum を生成し、テーブルへ挿入します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfrom.c 内で palloc0 を用いてインスタンス化されます。
  • **呼び出し先**: 構造体内の関数ポインタ（routine）や data_source_cb を介して、各フォーマット固有の解析ロジックやデータ読み込み関数が呼び出されます。

関連データ構造

  • CopyFromState (CopyFromStateData* の別名)
  • CopySource (読み込み元識別)
  • CopyFormatOptions (フォーマット設定)
  • StringInfoData (バッファ管理)
  • CopyFromRoutine (フォーマット別処理ルーチン)

失敗モードと境界条件

  • input_buf や raw_buf のサイズ（各64KB）を超えるデータ読み込み時のバッファ管理。
  • エンコーディング変換エラー発生時の input_reached_error フラグによる制御。
  • COPY 中のソフトエラー発生時、escontext を介したエラーのトラップと num_errors のカウント。

保守担当者向けメモ

  • attribute_buf や line_buf の再利用はパフォーマンスの要です。直接操作する場合はバッファ境界条件に注意してください。
  • CopyFromStateData は非常に巨大な構造体です。スタックオーバーフローを避けるため、関数引数には必ず CopyFromState を使用してください。
