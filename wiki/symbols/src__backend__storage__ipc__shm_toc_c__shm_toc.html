shm_toc

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/ipc/shm_toc.c:26

目的

動的共有メモリ（DSM）セグメントを論理的な領域に分割し、一意のキーを用いて共有メモリ内のデータ構造を安全に参照するための「目次（Table of Contents）」を保持する。

シグネチャまたは形状

struct shm_toc
{
    uint64      toc_magic;              /* Magic number identifying this TOC */
    slock_t     toc_mutex;              /* Spinlock for mutual exclusion */
    Size        toc_total_bytes;        /* Bytes managed by this TOC */
    Size        toc_allocated_bytes;    /* Bytes allocated of those managed */
    uint32      toc_nentry;             /* Number of entries in TOC */
    shm_toc_entry toc_entry[FLEXIBLE_ARRAY_MEMBER];
};

入力と出力

  • 本構造体はセグメントの先頭に配置され、共有メモリ領域の管理状態を保持する。
  • toc_entry 配列を介して、キーとセグメント先頭からの相対オフセットを紐付ける。

前提条件と不変条件

  • 共有メモリはプロセスごとに異なる仮想アドレスにマッピングされるため、TOC には絶対アドレスではなく、セグメント先頭からの相対オフセットが格納される。
  • toc_allocated_bytes はセグメントの末尾から前方へ向かって割り当てられ、先頭から成長する toc_entry 配列と衝突しないように管理される。

アルゴリズムまたは振る舞い

  • **初期化**: shm_toc_create によりスピンロックが初期化され、セグメント先頭に配置される。
  • **割り当て**: shm_toc_allocate は、管理領域の末尾からメモリを切り出す。
  • **登録**: shm_toc_insert は書き込みバリア（pg_write_barrier）を使用し、エントリ数更新前の整合性を保証する。
  • **検索**: shm_toc_lookup は読み込みバリア（pg_read_barrier）を用い、ロックを取得せずにエントリを走査する。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • _brin_parallel_build_main (src/backend/access/brin/brin.c)
  • _gin_parallel_build_main (src/backend/access/gin/gininsert.c)
  • _bt_parallel_build_main (src/backend/access/nbtree/nbtsort.c)
  • parallel_vacuum_main (src/backend/commands/vacuumparallel.c)
  • ExecParallelGetReceiver / ExecParallelGetQueryDesc (src/backend/executor/execParallel.c)
  • **呼び出し先**: 構造体内部で shm_toc_entry を使用。

関連データ構造

  • shm_toc_entry: TOC 内の各エントリを保持する構造体。

失敗モードと境界条件

  • **スケーラビリティ**: 多数のキーを扱うようには設計されていない。多数のポインタが必要な場合は、TOC には単一のデータ構造へのポインタのみを登録し、その構造体内部で管理することが推奨される。

保守担当者向けメモ

  • 本構造体はロックフリー検索を前提としているため、変更時にはメモリバリアの整合性に十分注意すること。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズ計算には注意が必要。
