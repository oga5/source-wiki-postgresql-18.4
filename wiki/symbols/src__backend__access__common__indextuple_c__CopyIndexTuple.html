CopyIndexTuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/indextuple.c:546

目的

既存の IndexTuple をメモリ上の新しい領域に複製し、独立したインスタンスを作成する。

シグネチャまたは形状

IndexTuple CopyIndexTuple(IndexTuple source)

入力と出力

  • **入力**: source (複製元の IndexTuple)
  • **出力**: 複製された IndexTuple へのポインタ。

前提条件と不変条件

  • source は有効な IndexTuple であること。
  • palloc を使用するため、呼び出し元は適切なメモリコンテキストがアクティブであることを保証する必要がある。

アルゴリズムまたは振る舞い

1. IndexTupleSize(source) を呼び出し、ソースタプルのサイズを算出する。 2. palloc を使用して、算出したサイズ分のメモリを確保する。 3. memcpy を使用して、ソースタプルの内容を確保したメモリ領域へコピーする。 4. 複製されたタプルへのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c
  • src/backend/access/gist/gist.c
  • src/backend/access/hash/hashovfl.c
  • src/backend/access/hash/hashpage.c
  • src/backend/access/nbtree/nbtdedup.c
  • src/backend/access/nbtree/nbtinsert.c
  • src/backend/access/common/indextuple.c (再帰的または内部的な参照)
  • **呼び出し先**:
  • IndexTupleSize (マクロ)
  • palloc
  • memcpy

関連データ構造

  • IndexTuple: インデックス行のメモリ表現。

失敗モードと境界条件

  • palloc がメモリ不足により失敗した場合、PostgreSQLの標準的なエラーハンドリング（elog(ERROR, ...)）により処理が中断される。

保守担当者向けメモ

  • 本関数はメモリの動的確保を行うため、呼び出し側で不要になった際に pfree を行うか、適切なメモリコンテキストの解放を通じてメモリリークを防ぐ必要がある。
