reduce_outer_joins_pass1_state

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/prep/prepjointree.c:86

目的

外結合（Outer Join）の削減処理における第1パスにおいて、ジョインツリーの各サブツリーの状態を保持するために使用されます。

シグネチャまたは形状

typedef struct reduce_outer_joins_pass1_state
{
    Relids      relids;          /* base relids within this subtree */
    bool        contains_outer;  /* does subtree contain outer join(s)? */
    List       *sub_states;      /* List of states for subtree components */
} reduce_outer_joins_pass1_state;

入力と出力

本構造体はデータ保持用であり、関数としての入出力は持ちません。主に reduce_outer_joins_pass1 関数によってインスタンス化され、ツリーの再帰的な走査を通じて状態が伝播されます。

前提条件と不変条件

  • relids は、対応するサブツリー内に含まれるすべてのベースリレーションの識別子セットを保持します。
  • contains_outer は、サブツリー内に外結合が存在するかどうかを示すフラグです。
  • sub_states は、子ノードの reduce_outer_joins_pass1_state を格納するリストです。

アルゴリズムまたは振る舞い

ジョインツリーを再帰的に走査し、各ノードにおけるリレーションの包含関係と外結合の有無を収集します。この情報は、後続のパスで外結合を内結合へ変換可能か判断するための基礎データとなります。

呼び出し元と呼び出し先

  • **呼び出し元**: reduce_outer_joins_pass1 関数が本構造体を生成し、再帰的に処理を行います。
  • **呼び出し先**: 本構造体自体はデータ構造であり、呼び出し先はありません。

関連データ構造

  • reduce_outer_joins_pass2_state: 第2パスで使用される関連構造体。
  • reduce_outer_joins_partial_state: 部分的な状態管理に使用される構造体。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性があります。
  • 深いジョインツリーに対しては、再帰処理に伴うスタックオーバーフローのリスクがあります（check_stack_depth() による保護が前提）。

保守担当者向けメモ

  • 本構造体は reduce_outer_joins の最適化プロセスにおいて不可欠です。構造を変更する場合は、第1パスおよび第2パスの走査ロジックとの整合性に注意してください。
