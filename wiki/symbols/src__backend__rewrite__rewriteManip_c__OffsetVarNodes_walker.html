OffsetVarNodes_walker

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteManip.c:387

目的

クエリツリーまたは式ツリーを走査し、指定されたネストレベル（sublevels_up）に一致するノードの範囲テーブルインデックス（varno）や関連するリレーションIDを、指定されたオフセット値で一括シフトします。

シグネチャまたは形状

static bool OffsetVarNodes_walker(Node *node, OffsetVarNodes_context *context)

入力と出力

  • **入力**:
  • node: 走査対象のノード（Node*）。
  • context: 走査状態とオフセット情報を保持する OffsetVarNodes_context 構造体。
  • **出力**:
  • 戻り値: 走査を継続する場合は false、中断する場合は true を返します。

前提条件と不変条件

  • context->sublevels_up は、現在走査中のクエリレベルを追跡するために使用されます。
  • インプレースでノードを直接書き換えるため、呼び出し元は必要に応じて copyObject() を使用してツリーを複製しておく必要があります。
  • Query ノードに遭遇するたびに sublevels_up をインクリメントし、再帰後にデクリメントするルールが適用されます。

アルゴリズムまたは振る舞い

1. **ノード種別ごとの処理**:

  • Var: varlevelsup が context->sublevels_up と一致する場合、varno、varnullingrels、varnosyn をオフセット分加算します。
  • CurrentOfExpr, RangeTblRef, JoinExpr, AppendRelInfo: sublevels_up == 0 の場合に限り、対応するインデックスフィールドをオフセット分加算します。
  • PlaceHolderVar: phlevelsup が一致する場合、phrels および phnullingrels をオフセット分加算します。

    • 2. **再帰走査**:

  • Query ノードの場合: sublevels_up をインクリメントし、query_tree_walker を介してサブクエリ内を再帰的に走査します。
  • その他のノードの場合: expression_tree_walker を使用してツリーを走査します。

    • 3. **アサーション**: PlanRowMark、SpecialJoinInfo、PlaceHolderInfo、MinMaxAggInfo は本関数では処理対象外としてアサートされます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/rewrite/rewriteManip.c 内の OffsetVarNodes 関数から呼び出されます。
  • **呼び出し先**:
  • query_tree_walker（Query ノード走査用）
  • expression_tree_walker（汎用的な式ツリー走査用）
  • offset_relid_set（リレーションIDセットのオフセット計算用）

関連データ構造

  • OffsetVarNodes_context: 走査のコンテキスト情報（オフセット値と現在のネストレベル）を保持します。
  • Var, CurrentOfExpr, RangeTblRef, JoinExpr, PlaceHolderVar, AppendRelInfo, Query: 操作対象となるノード群。

失敗モードと境界条件

  • node が NULL の場合は即座に false を返します。
  • sublevels_up の管理が不適切な場合、誤ったネストレベルの変数が書き換えられる可能性があります。

保守担当者向けメモ

  • 本関数はインプレース修正を行うため、副作用に注意してください。
  • sublevels_up の管理は、サブクエリへの潜り込みと復帰のタイミングで正確に行う必要があります。
  • 新しいノードタイプが追加された場合、必要に応じて本関数での処理を追加してください。
