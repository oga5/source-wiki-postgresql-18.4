AggStatePerPhaseData

定義

  • 種別: struct
  • 定義箇所: src/include/executor/nodeAgg.h:280

目的

Grouping Sets 等の複数フェーズにわたる集約処理において、各段階（フェーズ）固有の状態およびメタデータを管理する。

シグネチャまたは形状

typedef struct AggStatePerPhaseData
{
    AggStrategy aggstrategy;
    int         numsets;
    int        *gset_lengths;
    Bitmapset **grouped_cols;
    ExprState **eqfunctions;
    Agg        *aggnode;
    Sort       *sortnode;
    ExprState  *evaltrans;
    ExprState  *evaltrans_cache[2][2];
} AggStatePerPhaseData;

入力と出力

  • **入力**: Agg ノードおよび Sort ノードの構成情報、遷移関数評価用の ExprState。
  • **出力**: 各フェーズの実行に必要なキャッシュ済み式評価状態（evaltrans_cache）。

前提条件と不変条件

  • evaltrans_cache は、outerops（インデックス0）と TTSOpsMinimalTuple（インデックス1）の組み合わせ、および NULL チェックの有無（インデックス0: なし, 1: あり）で構成される。

アルゴリズムまたは振る舞い

  • 集約実行の各フェーズにおいて、グループ化の戦略やソート順序を保持する。
  • evaltrans を通じて遷移関数の評価を行い、最適化のために evaltrans_cache にコンパイル済み式のバリエーションを保持する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeAgg.c 内で palloc0 を用いてフェーズ数分確保される。
  • **呼び出し先**: ExecBuildAggTrans（src/include/executor/executor.h）が本構造体を引数として受け取り、遷移関数の構築を行う。

関連データ構造

  • Agg: プランノード。
  • Sort: 入力順序を制御するノード。
  • ExprState: 式評価の状態管理。
  • AggStatePerPhase: 本構造体へのポインタ型（src/include/nodes/execnodes.h で定義）。

失敗モードと境界条件

  • numsets が 0 の場合、Grouping Sets を伴わない集約として扱われる。

保守担当者向けメモ

  • 本構造体は AggState の一部として管理される。
  • evaltrans_cache のインデックス構造は、実行時のタプル形式や NULL チェックの有無に応じた最適化パスを指すため、式評価エンジンの変更時には整合性に注意すること。
