format_type_extended

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/format_type.c:111

目的

データ型OIDおよび型修飾子（typmod）を、人間が読みやすい標準的なSQL形式の文字列に変換します。

シグネチャまたは形状

char *format_type_extended(Oid type_oid, int32 typemod, bits16 flags)

入力と出力

  • **入力**:
  • type_oid: 変換対象のデータ型OID。
  • typemod: 型修飾子（例: varchar(n) の n）。
  • flags: 動作を制御するビットフラグ（FORMAT_TYPE_INVALID_AS_NULL, FORMAT_TYPE_ALLOW_INVALID, FORMAT_TYPE_TYPEMOD_GIVEN など）。
  • **出力**:
  • 変換された型名の文字列（palloc 済み）。
  • 無効なOIDやキャッシュミス時にフラグ設定に応じて NULL、"-"、"???" を返却。

前提条件と不変条件

  • pg_type システムカタログへのアクセスが必要です。
  • typemod が -1 の場合、型修飾子は指定されていないものとして扱われます。
  • 組み込み型（bool, int4 等）は、SQL標準に準拠した名称に変換されます。

アルゴリズムまたは振る舞い

1. **無効OIDチェック**: type_oid が InvalidOid の場合、フラグに従い NULL または "-" を返します。 2. **キャッシュ検索**: SearchSysCache1 で pg_type を検索し、型情報を取得します。失敗時はフラグに従い NULL、"???" を返すかエラーを投げます。 3. **配列判定**: IsTrueArrayType かつ TYPSTORAGE_PLAIN でない場合、配列の基底型を特定し、再帰的に処理を切り替えます。 4. **型名変換**:

  • 組み込み型（BIT, BOOL, BPCHAR, FLOAT4, FLOAT8, INT2, INT4, INT8, NUMERIC, INTERVAL, TIME, TIMETZ, TIMESTAMP, TIMESTAMPTZ, VARBIT）は、switch 文によりハードコードされた名称を選択します。
  • with_typemod が有効な場合、printTypmod を呼び出し、型固有の typmodout 関数で修飾子を付与します。

    • 5. **メモリ管理**: 結果文字列は palloc されます。

関連データ構造

  • Form_pg_type: pg_type カタログの行構造体。
  • HeapTuple: システムキャッシュから取得される型定義のタプル。

失敗モードと境界条件

  • **キャッシュミス**: pg_type に該当OIDが存在しない場合、flags に FORMAT_TYPE_ALLOW_INVALID が含まれていれば "???" を返し、そうでなければ elog(ERROR) を発生させます。
  • **typmodの扱い**: typmod が -1 の場合、型固有の出力関数を呼び出さず、型名のみを返却するロジックが含まれています。

保守担当者向けメモ

  • 新しい組み込み型を追加する場合、switch 文へのケース追加と、必要に応じた typmodout の処理を検討してください。
  • format_type は本関数のラッパーであり、引数のフラグを省略したインターフェースを提供しています。
  • type_maximum_size と型定義の知識を共有しているため、型定義の変更時は整合性に注意してください。
