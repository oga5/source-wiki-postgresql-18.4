RT_NODE_16

定義

  • 種別: struct
  • 定義箇所: src/include/lib/radixtree.h:528

目的

適応型基数木（ART）において、中程度の数の子ノード（最大16個）を保持するためのノード構造を提供します。

シグネチャまたは形状

typedef struct RT_NODE_16
{
    RT_NODE     base;
    uint8       chunks[RT_FANOUT_16_MAX];
    RT_PTR_ALLOC children[FLEXIBLE_ARRAY_MEMBER];
} RT_NODE_16;

入力と出力

  • **入力**: 検索や挿入の際、uint8 型のキーチャンク（chunk）を受け取ります。
  • **出力**: RT_NODE_16_SEARCH_EQ 等の関数を通じて、対応する子ノードへのポインタ（RT_PTR_ALLOC）を返します。

前提条件と不変条件

  • base フィールドとして RT_NODE を保持し、基数木のノードとしての共通属性を継承します。
  • children は可変長配列として定義されており、ノードのサイズクラスに応じて動的に確保されます。

アルゴリズムまたは振る舞い

  • **検索**: chunks 配列内のキーチャンクを走査し、一致するインデックスに対応する children を特定します。
  • **挿入位置の特定**: RT_NODE_16_GET_INSERTPOS を使用して、新しいキーチャンクを挿入すべき位置を決定します。
  • **メモリ管理**: RT_FANOUT_16_LO および RT_FANOUT_16_HI マクロに基づき、ノードのサイズクラスに応じたメモリ領域が確保されます。

呼び出し元と呼び出し先

  • **呼び出し元**: 基数木の検索、挿入、およびノードの成長を管理する関数群（RT_NODE_16_SEARCH_EQ 等）。
  • **呼び出し先**: RT_PTR_ALLOC を介した子ノードへのアクセス。
  • ※具体的な関数呼び出しの連鎖については、ソースコード内のマクロ展開に依存するため、静的解析のみでは完全なコールグラフを特定できません。

関連データ構造

  • RT_NODE: 基底ノード構造体。
  • RT_PTR_ALLOC: 子ノードを指すポインタ型（ローカルメモリまたは共有メモリのDSAポインタ）。

失敗モードと境界条件

  • **メモリ不足**: children 配列の動的確保時にメモリ不足が発生する可能性があります。
  • **容量超過**: RT_FANOUT_16_MAX を超える子ノードが必要となった場合、より大きなノード種別（RT_NODE_48 等）への変換が必要です。

保守担当者向けメモ

  • 本構造体はマクロテンプレートの一部であり、RT_NODE_16 という名前はコンパイル時に生成されます。
  • offsetof(RT_NODE_16, children) を用いたメモリ計算が行われているため、構造体のメンバ順序を変更する際は注意が必要です。
