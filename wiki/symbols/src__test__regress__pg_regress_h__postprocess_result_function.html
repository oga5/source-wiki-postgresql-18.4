postprocess_result_function

定義

  • 種別: typedef
  • 定義箇所: src/test/regress/pg_regress.h:42

目的

テスト実行のライフサイクルにおいて、個別のテスト結果に対する後処理（検証やログ出力など）を行うためのコールバック関数型を定義します。

シグネチャまたは形状

typedef void (*postprocess_result_function) (const char *filename);

入力と出力

  • 入力: const char *filename - 後処理対象となる結果ファイルのパス。
  • 出力: なし（void）。

前提条件と不変条件

  • 本型を使用して定義された関数は、テスト実行エンジンが指定する結果ファイルパスを引数として受け取ります。
  • 戻り値を持たないため、副作用（ファイル検証、ステータス更新など）によって結果を反映させる必要があります。

アルゴリズムまたは振る舞い

pg_regress のテスト実行フローにおいて、テストプロセス終了後に呼び出されるフックとして機能します。具体的な処理内容は、この型を実装する関数に委ねられます。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/regress/pg_regress.c 内の各箇所（行1644, 1850, 2072）で、postfunc という引数名でこの型が参照されています。
  • 呼び出し先: 本シンボル自体は型定義であるため、具体的な呼び出し先は実装関数に依存します。

関連データ構造

  • pg_regress.h 内で定義される他のコールバック型（init_function, test_start_function）と併用され、テスト駆動プログラムのカスタマイズに使用されます。

失敗モードと境界条件

  • 引数 filename が NULL または無効なパスである場合、実装側の関数で適切にエラーハンドリングを行う必要があります。

保守担当者向けメモ

  • 本型を実装する関数を追加する際は、pg_regress.c のテスト実行ロジックとの整合性に注意してください。
  • pg_regress.c 内の各所で使用されている postfunc 引数の文脈を追うことで、具体的な利用パターンを確認可能です。
