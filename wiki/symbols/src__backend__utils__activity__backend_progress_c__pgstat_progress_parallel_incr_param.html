pgstat_progress_parallel_incr_param

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/activity/backend_progress.c:90

目的

並列クエリ実行時において、ワーカープロセスからリーダープロセスへ進捗パラメータの加算を通知、またはリーダープロセス自身が直接進捗パラメータを加算するためのインターフェースを提供します。

シグネチャまたは形状

void pgstat_progress_parallel_incr_param(int index, int64 incr)
  • index: 更新対象の進捗パラメータのインデックス（0〜19）。
  • incr: 加算する値。

入力と出力

  • **入力**: index（対象パラメータ番号）、incr（加算値）。
  • **出力**: なし（戻り値なし）。
  • **副作用**:
  • 現在のプロセスがパラレルワーカーである場合、PqMsg_Progress メッセージを送信します。
  • 現在のプロセスがリーダーである場合、共有メモリ上の進捗パラメータを直接更新します。

前提条件と不変条件

  • index は有効な進捗パラメータの範囲内である必要があります。
  • pgstat_track_activities が有効でない場合、本関数は実質的に何もしません。

アルゴリズムまたは振る舞い

1. IsParallelWorker() を呼び出し、現在のプロセスがパラレルワーカーかどうかを判定します。 2. **ワーカーの場合**:

  • StringInfoData を初期化します。
  • PqMsg_Progress メッセージタイプを指定してメッセージを開始します。
  • index と incr をメッセージに書き込みます。
  • メッセージを送信（pq_endmessage）します。

    • 3. **リーダーの場合**:

  • pgstat_progress_incr_param(index, incr) を直接呼び出し、共有メモリ上の値を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/vacuum.c
  • src/backend/commands/vacuumparallel.c
  • **呼び出し先**:
  • IsParallelWorker()
  • initStringInfo()
  • pq_beginmessage()
  • pq_sendint32()
  • pq_sendint64()
  • pq_endmessage()
  • pgstat_progress_incr_param()

関連データ構造

  • StringInfoData: メッセージ構築用バッファ。
  • PgBackendStatus: 共有メモリ上の進捗情報保持構造体（リーダーが更新する対象）。

失敗モードと境界条件

  • メッセージ送信時に通信エラーが発生した場合の挙動は、libpq のメッセージング層に依存します。
  • index が範囲外の場合の動作は、内部で呼び出される pgstat_progress_incr_param の実装に依存します。

保守担当者向けメモ

  • 本関数は、並列処理における進捗更新の「通知」と「直接更新」を抽象化しています。
  • PqMsg_Progress メッセージのフォーマットを変更する場合は、リーダー側の受信処理（本ソースファイル外）との整合性に注意してください。
