DatumGetExpandedArrayX

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_expanded.c:371

目的

入力された Datum から、書き込み可能な展開済み配列（ExpandedArrayHeader）を取得します。既存の展開済み配列であればそれを再利用し、そうでなければ expand_array を用いて変換を行います。

シグネチャまたは形状

ExpandedArrayHeader *DatumGetExpandedArrayX(Datum d, ArrayMetaState *metacache)

入力と出力

  • **入力**:
  • d: 処理対象の Datum（配列型）。
  • metacache: 型情報キャッシュ（ArrayMetaState）。指定された場合、取得した配列の型情報で更新されます。
  • **出力**:
  • 展開済み配列のヘッダー構造体へのポインタ（ExpandedArrayHeader *）。

前提条件と不変条件

  • 入力が既に展開済み配列である場合、eah->ea_magic == EA_MAGIC が真であることを保証します。
  • metacache が提供されている場合、その内容は eah の element_type, typlen, typbyval, typalign で上書きされます。

アルゴリズムまたは振る舞い

1. 入力 d が既に書き込み可能な展開済み配列（VARATT_IS_EXTERNAL_EXPANDED_RW）であるかを確認します。 2. 該当する場合、そのヘッダーを返し、必要に応じて metacache を更新します。 3. 該当しない場合、expand_array を呼び出して d を展開済み形式に変換し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/utils/array.h:270
  • src/include/utils/array.h:476
  • **呼び出し先**:
  • expand_array（変換が必要な場合）
  • DatumGetPointer / DatumGetEOHP（マクロ）

関連データ構造

  • ExpandedArrayHeader: 展開済み配列のヘッダー。
  • ArrayMetaState: 配列の型情報を保持するキャッシュ構造体。

失敗モードと境界条件

  • 入力が有効な配列データでない場合、内部で呼び出される expand_array 等の挙動に依存します。
  • metacache が NULL の場合、キャッシュの更新はスキップされます。

保守担当者向けメモ

  • 本関数は、既存の展開済み配列を効率的に再利用するためのパスを提供します。
  • metacache を介した型情報の同期は、配列操作のパフォーマンス最適化において重要です。
