VacAttrStats

定義

  • **種別**: struct
  • **定義箇所**: src/include/commands/vacuum.h:116

目的

ANALYZE コマンド実行時、各列（またはインデックス式）の統計情報を保持・計算するためのデータ構造です。型固有の統計収集ロジックと、メインの ANALYZE 処理との間のインターフェースとして機能します。

シグネチャまたは形状

typedef struct VacAttrStats {
    /* 初期化フィールド */
    int attstattarget;
    Oid attrtypid;
    int32 attrtypmod;
    Form_pg_type attrtype;
    Oid attrcollid;
    MemoryContext anl_context;

    /* 統計計算ルーチン用 */
    AnalyzeAttrComputeStatsFunc compute_stats;
    int minrows;
    void *extra_data;

    /* 計算結果格納フィールド */
    bool stats_valid;
    float4 stanullfrac;
    int32 stawidth;
    float4 stadistinct;
    int16 stakind[STATISTIC_NUM_SLOTS];
    Oid staop[STATISTIC_NUM_SLOTS];
    Oid stacoll[STATISTIC_NUM_SLOTS];
    int numnumbers[STATISTIC_NUM_SLOTS];
    float4 *stanumbers[STATISTIC_NUM_SLOTS];
    int numvalues[STATISTIC_NUM_SLOTS];
    Datum *stavalues[STATISTIC_NUM_SLOTS];

    /* 型情報フィールド */
    Oid statypid[STATISTIC_NUM_SLOTS];
    int16 statyplen[STATISTIC_NUM_SLOTS];
    bool statypbyval[STATISTIC_NUM_SLOTS];
    char statypalign[STATISTIC_NUM_SLOTS];

    /* 内部管理フィールド */
    int tupattnum;
    HeapTuple *rows;
    TupleDesc tupDesc;
    Datum *exprvals;
    bool *exprnulls;
    int rowstride;
} VacAttrStats;

入力と出力

  • **入力**: ANALYZE 処理の開始時に、対象列のメタデータ（型、照合順序、ターゲット等）が設定されます。
  • **出力**: compute_stats 関数が実行されることで、stats_valid フラグと共に統計値（NULL率、平均幅、個別値数、ヒストグラム等）が格納されます。

前提条件と不変条件

  • compute_stats 関数が呼び出される前に、メインの ANALYZE コードによって初期化フィールドが設定されている必要があります。
  • stats_valid が true の場合、統計値フィールドには有効なデータが格納されていると見なされます。
  • 内部管理フィールド（tupattnum 等）は ANALYZE のメイン処理専用であり、型固有の関数から直接参照してはなりません。

アルゴリズムまたは振る舞い

1. **初期化**: ANALYZE が対象列ごとに本構造体を割り当て、基本情報を設定します。 2. **型固有処理**: 型に応じた typanalyze 関数が呼び出され、compute_stats 関数ポインタと extra_data が設定されます。 3. **統計計算**: compute_stats 関数が実行され、stanullfrac や stavalues 等の統計データが埋められます。 4. **型情報更新**: 必要に応じて statypid 等の配列要素型情報を更新し、統計情報の保存形式を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/analyze.c 内の examine_attribute 関数などで生成・参照されます。
  • **呼び出し先**: compute_stats フィールドに格納された関数ポインタを介して、型固有の統計計算関数が呼び出されます。

関連データ構造

  • AnalyzeAttrComputeStatsFunc: 統計計算を行う関数ポインタ型。
  • Form_pg_type: pg_type システムカタログの行構造。
  • STATISTIC_NUM_SLOTS: 統計情報スロットの最大数。

失敗モードと境界条件

  • typanalyze 関数が false を返した場合、統計計算はスキップされます。
  • statypid 等の配列フィールドは初期状態で attrtypid に基づいて初期化されますが、カスタム関数による上書きが可能です。

保守担当者向けメモ

  • 新しいデータ型に対する統計収集ロジックを追加する場合、compute_stats 関数の実装と、本構造体の各フィールドの役割を理解する必要があります。
  • 内部管理フィールドは ANALYZE のコアロジックに依存しているため、変更時には analyze.c への影響を考慮してください。
