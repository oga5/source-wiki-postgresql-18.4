btree_xlog_updates

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtxlog.c:556

目的

B-tree インデックスページ上の既存タプルに対して、WAL リプレイ時に削除対象の TID（Tuple Identifier）を反映させる更新処理を行う。

シグネチャまたは形状

static void btree_xlog_updates(Page page, OffsetNumber *updatedoffsets, xl_btree_update *updates, int nupdated)

入力と出力

  • **入力**:
  • page: 更新対象のインデックスページ。
  • updatedoffsets: 更新対象タプルのオフセット番号配列。
  • updates: 適用する更新情報（xl_btree_update 構造体）の先頭ポインタ。
  • nupdated: 更新対象の数。
  • **出力**: なし（副作用として page を直接変更する）。

前提条件と不変条件

  • page は適切にロックされた状態であること。
  • updates は nupdated 分のデータを含む連続したメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. nupdated 回のループ処理を実行する。 2. 各イテレーションにおいて、updatedoffsets[i] から対象タプルを取得する。 3. BTVacuumPosting 構造体を一時的に確保し、WAL レコードから削除対象の TID 情報をコピーする。 4. _bt_update_posting を呼び出し、タプル内の TID を削除済みとしてマークする更新を実行する。 5. PageIndexTupleOverwrite を使用して、更新後のタプルをページ上の元の位置に書き戻す。 6. 更新に使用した一時メモリを解放する。 7. updates ポインタを次のレコード位置へ進める（SizeOfBtreeUpdate と削除 TID 数に基づくオフセット加算）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtxlog.c 内の WAL リプレイ処理（btree_redo からの分岐先）。
  • **呼び出し先**:
  • _bt_update_posting: 実際のタプル更新ロジックを実行。
  • PageIndexTupleOverwrite: ページ上のタプルを置換。
  • palloc / pfree: メモリ管理。

関連データ構造

  • xl_btree_update: WAL に記録された更新情報。
  • BTVacuumPostingData: 更新処理中に使用される一時的なデータ構造。
  • IndexTuple: インデックス内のタプル形式。

失敗モードと境界条件

  • PageIndexTupleOverwrite が失敗した場合、PANIC レベルでエラーを発生させ、リカバリを停止する。

保守担当者向けメモ

  • 本関数は、重複排除（deduplication）されたタプルに対する部分的な削除処理をリカバリ時に再現するために使用される。
  • updates ポインタの算術演算は、可変長データ構造を扱うため、xl_btree_update のレイアウト変更時には注意が必要である。
