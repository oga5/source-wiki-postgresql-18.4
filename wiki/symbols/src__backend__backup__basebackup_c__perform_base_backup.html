perform_base_backup

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup.c:233

目的

PostgreSQL の物理ベースバックアップにおけるメインループを制御し、データディレクトリおよびテーブルスペースのファイルを bbsink インターフェースを通じてストリーミング送信する。

シグネチャまたは形状

static void perform_base_backup(basebackup_options *opt, bbsink *sink, IncrementalBackupInfo *ib);

入力と出力

  • **入力**:
  • opt: バックアップオプション（ラベル、チェックポイント設定、マニフェスト設定など）。
  • sink: データ送信先を抽象化した bbsink オブジェクト。
  • ib: 増分バックアップ用情報（NULL の場合はフルバックアップ）。
  • **出力**: なし（副作用として sink へのデータ書き出しおよびバックアップ状態の遷移を行う）。

前提条件と不変条件

  • AuxProcessResourceOwner が初期化されていること。
  • do_pg_backup_start から do_pg_backup_stop までの処理は、エラー発生時に do_pg_abort_backup が確実に呼び出されるよう PG_ENSURE_ERROR_CLEANUP ブロック内で実行されること。

アルゴリズムまたは振る舞い

1. **初期化**: バックアップ状態の初期化、マニフェストの準備、および do_pg_backup_start によるバックアップモードへの移行。 2. **準備**: 増分バックアップの場合は PrepareForIncrementalBackup を実行。 3. **サイズ見積もり**: opt->progress が有効な場合、全テーブルスペースのサイズを計算。 4. **送信ループ**:

  • 各テーブルスペースを順次処理。
  • メインデータディレクトリの場合、backup_label、tablespace_map、データファイル群、pg_control を順に送信。
  • 外部テーブルスペースの場合、対応する tar アーカイブを作成して送信。
  • 各アーカイブの終了時に tar 終端ブロックを書き込み。

    • 5. **終了処理**: do_pg_backup_stop を呼び出し、バックアップモードを解除。 6. **WAL 追跡**: opt->includewal が有効な場合、バックアップ終了後に WAL ファイルを追記する準備を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup.c 内の関数（SendBaseBackup 等から間接的に呼び出される）。
  • **呼び出し先**: do_pg_backup_start, do_pg_backup_stop, do_pg_abort_backup, sendDir, sendTablespace, bbsink インターフェース関数群（bbsink_begin_backup, bbsink_begin_archive 等）。

関連データ構造

  • basebackup_options: バックアップ設定。
  • bbsink: データ送信パイプライン。
  • BackupState: バックアップの進行状況や LSN を保持。
  • tablespaceinfo: テーブルスペースのパスやサイズ情報。
  • backup_manifest_info: バックアップ整合性検証用マニフェスト。

失敗モードと境界条件

  • do_pg_backup_start 以降の処理中にエラーが発生した場合、do_pg_abort_backup が実行され、バックアップカウンターのリークを防ぐ。
  • opt->includewal が有効な場合、メインデータディレクトリの送信後に WAL ファイルを追記するため、メインデータディレクトリは常に最後に送信される必要がある。

保守担当者向けメモ

  • bbsink は単なる出力先ではなく、圧縮やフィルタリングを行うパイプラインとして機能する。
  • WAL ファイルはバックアップの最後にまとめて送信される設計である。
  • PG_ENSURE_ERROR_CLEANUP の使用により、エラー時のクリーンアップが保証されている。
