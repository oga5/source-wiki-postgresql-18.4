gin_cmp_tslexeme

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsginidx.c:23

目的

GINインデックスのB-tree構造内において、2つのtsvector語彙（lexeme）を比較し、その順序関係を決定する。

シグネチャまたは形状

Datum gin_cmp_tslexeme(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される2つの text 型データ（tsvectorの語彙）。
  • **出力**: int32 型の比較結果。
  • 0: 両者が等しい
  • 0未満: 第1引数が第2引数より小さい
  • 0より大きい: 第1引数が第2引数より大きい

前提条件と不変条件

  • 入力される text 型データは、tsvectorの語彙として有効な形式であること。
  • PG_GETARG_TEXT_PP を使用しているため、必要に応じてTOAST展開が行われる。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を用いて、入力された2つの text ポインタ a と b を取得する。 2. tsCompareString を呼び出し、語彙の文字列データを比較する。この際、前方一致フラグは false として完全一致比較を行う。 3. PG_FREE_IF_COPY を用いて、必要に応じて一時的なコピーメモリを解放する。 4. 比較結果を PG_RETURN_INT32 で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内には、本関数を直接呼び出している具体的な呼び出し元の記述はありません。
  • **呼び出し先**: tsCompareString（比較ロジックの実行）。

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • tsvector: 全文検索用の語彙リストを保持するデータ型。

失敗モードと境界条件

  • 入力がNULLの場合、PG_GETARG_TEXT_PP の仕様に従いエラーが発生する可能性がある。
  • 文字列の長さが異なる場合や、内容が異なる場合は tsCompareString の仕様に従い辞書順で比較される。

保守担当者向けメモ

  • 本関数はGINインデックスの比較関数として登録されることを想定している。
  • tsCompareString の第5引数が false に固定されているため、この関数は完全一致比較専用である。前方一致が必要な場合は別の関数（gin_cmp_prefix等）を参照すること。
