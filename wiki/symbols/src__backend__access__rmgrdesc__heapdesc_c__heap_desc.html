heap_desc

定義

  • void heap_desc(StringInfo buf, XLogReaderState *record)
  • src/backend/access/rmgrdesc/heapdesc.c:183

目的

ヒープアクセスメソッドに関連するWALレコードの内容をデコードし、人間が読める形式の文字列として StringInfo バッファに出力する。主に pg_waldump 等のツールで利用される。

シグネチャまたは形状

void heap_desc(StringInfo buf, XLogReaderState *record);

入力と出力

  • **入力**:
  • buf: 出力先となる StringInfo 構造体へのポインタ。
  • record: 解析対象のWALレコード状態を保持する XLogReaderState へのポインタ。
  • **出力**: なし（buf に解析結果が追記される）。

前提条件と不変条件

  • record は有効なWALレコードを指している必要がある。
  • XLogRecGetData を通じて取得されるデータは、XLOG_HEAP_OPMASK で特定されるヒープ操作タイプと一致する構造体（xl_heap_insert 等）としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. XLogRecGetInfo で取得したレコード情報から XLR_INFO_MASK を除外し、XLOG_HEAP_OPMASK を適用して操作タイプを特定する。 2. 操作タイプ（INSERT, DELETE, UPDATE, HOT_UPDATE, TRUNCATE, CONFIRM, LOCK, INPLACE）に応じて分岐する。 3. 各操作に対応する構造体にデータをキャストし、オフセットやフラグ、トランザクションID等の情報を appendStringInfo を用いてバッファに書き込む。 4. 必要に応じて infobits_desc（ビットフラグ用）や truncate_flags_desc、array_desc（配列用）などのヘルパー関数を呼び出し、詳細を整形する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h で定義されるリソースマネージャインターフェース（RM_HEAP_ID）を通じて、WAL解析ツール（pg_waldump 等）から呼び出される。
  • **呼び出し先**:
  • infobits_desc
  • truncate_flags_desc
  • array_desc
  • standby_desc_invalidations
  • XLogRecGetData, XLogRecGetInfo（マクロ）

関連データ構造

  • StringInfo: 文字列構築用バッファ。
  • XLogReaderState: WALレコードの読み取り状態。
  • xl_heap_insert, xl_heap_delete, xl_heap_update, xl_heap_truncate, xl_heap_confirm, xl_heap_lock, xl_heap_inplace: 各ヒープ操作に対応するWALレコード構造体。

失敗モードと境界条件

  • 未知の操作タイプが渡された場合、本関数内では何も出力されない（デフォルトの動作）。
  • record が不正なデータを含む場合、キャスト後の構造体アクセスにより予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • 新しいヒープWALレコードタイプを追加した際は、本関数の分岐条件にケースを追加すること。
  • heap2_desc との役割分担（XLOG_HEAP_* か XLOG_HEAP2_* か）を維持すること。
