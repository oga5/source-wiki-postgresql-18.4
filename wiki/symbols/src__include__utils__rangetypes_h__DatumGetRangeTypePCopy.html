DatumGetRangeTypePCopy

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/rangetypes.h:79

目的

Datum 型として渡された範囲型データを、メモリ上でコピーし、そのポインタを RangeType* 型として取得します。

シグネチャまたは形状

static inline RangeType *DatumGetRangeTypePCopy(Datum X)

入力と出力

  • **入力**: Datum X （範囲型データ）
  • **出力**: RangeType* （コピーされた範囲型データへのポインタ）

前提条件と不変条件

  • 入力 Datum X は、有効な範囲型データである必要があります。
  • PG_DETOAST_DATUM_COPY マクロを使用するため、入力が圧縮またはアウトライン化されている場合は、メモリ上に展開（デトースト）されたコピーが作成されます。

アルゴリズムまたは振る舞い

1. 入力された Datum X に対して PG_DETOAST_DATUM_COPY を実行します。 2. これにより、必要に応じてデータの展開とメモリコピーが行われ、結果として RangeType 構造体へのポインタが返されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/utils/rangetypes.h:92 にてマクロ PG_GETARG_RANGE_P_COPY(n) の定義内で使用されています。
  • **呼び出し先**:
  • PG_DETOAST_DATUM_COPY マクロ（PostgreSQLのコアメモリ管理機能）。

関連データ構造

  • RangeType: 範囲型のメモリ表現（可変長データ）。

失敗モードと境界条件

  • メモリ不足時には PG_DETOAST_DATUM_COPY の内部実装によりエラーが発生する可能性があります。
  • 入力が NULL の場合、PG_DETOAST_DATUM_COPY の振る舞いに依存します。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、パフォーマンスを考慮して設計されています。
  • RangeType は可変長構造体であるため、直接フィールドを操作せず、必ず提供されているマクロや関数経由でアクセスしてください。
