xlhp_freeze_plan

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/heapam_xlog.h:342

目的

VACUUM処理におけるタプルの凍結（Freezing）計画をWAL（Write-Ahead Logging）に記録するためのデータ構造です。ヒープページのタプルに対する状態変更をリカバリ時に再現するために使用されます。

シグネチャまたは形状

typedef struct xlhp_freeze_plan
{
    TransactionId xmax;
    uint16        t_infomask2;
    uint16        t_infomask;
    uint8         frzflags;
    uint16        ntuples;
} xlhp_freeze_plan;

入力と出力

  • **入力**: src/backend/access/heap/pruneheap.c 等で生成される凍結情報。
  • **出力**: WALレコードの一部としてシリアライズされ、リカバリ時に heap_redo 等で読み取られます。

前提条件と不変条件

  • ntuples は、この計画に関連付けられたページオフセット番号の配列の長さを表します。
  • 本構造体は、xl_heap_prune 構造体に関連する可変長データの一部として扱われます。

アルゴリズムまたは振る舞い

1. **記録**: heap_log_freeze_new_plan 等により、タプルの凍結状態（xmax, infomask 等）が本構造体に格納されます。 2. **再生**: リカバリ時、heapdesc.c の array_desc 等を通じてWALからデコードされ、対象ページのタプルに対して凍結処理が適用されます。

呼び出し元と呼び出し先

  • **呼び出し元（記録側）**: src/backend/access/heap/pruneheap.c（heap_log_freeze_eq, heap_log_freeze_new_plan）
  • **呼び出し元（解析側）**: src/backend/access/rmgrdesc/heapdesc.c（WAL内容のデコード処理）
  • **呼び出し先**: 直接的な関数呼び出し関係はありません（データ構造体のため）。

関連データ構造

  • xl_heap_prune: 本構造体を含む親となるWALレコード構造体。
  • xlhp_prune_items: ページ内のオフセット情報を保持する構造体。
  • HeapTupleFreeze: 凍結処理の実行時に使用されるメモリ上の構造体。

失敗モードと境界条件

  • WALレコードのデコード時に nplans や ntuples の値が不正な場合、メモリの読み取り境界を逸脱する可能性があります。
  • 可変長データとして扱われるため、シリアライズ時のオフセット計算（cursor の加算）が正確である必要があります。

保守担当者向けメモ

  • 本構造体は xl_heap_prune の一部として可変長で続くため、単体での解析ではなく heap_xlog_deserialize_prune_and_freeze の実装と併せて確認してください。
  • src/backend/access/rmgrdesc/heapdesc.c でのデコードロジックが、本構造体のメモリレイアウトに直接依存している点に注意してください。
