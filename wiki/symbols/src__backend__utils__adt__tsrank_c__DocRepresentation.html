DocRepresentation

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsrank.c:504

目的

全文検索のランキング処理（特に近接度ランキング ts_rank_cd）において、文書内の単語位置情報とクエリの適合状態を保持するための内部表現構造体です。

シグネチャまたは形状

typedef struct {
    union {
        struct {
            QueryItem **items;
            int16       nitem;
        } query;
        struct {
            QueryItem  *item;
            WordEntry  *entry;
        } map;
    } data;
    WordEntryPos pos;
} DocRepresentation;

入力と出力

  • 本構造体は、get_docrep 関数等によって文書から生成され、Cover 関数等のランキング計算ロジックへ渡されます。
  • data 共用体は、コンパイル済みの文書表現（query）または文書構築用の中間マッピング（map）のいずれかとして機能します。

前提条件と不変条件

  • Cover 関数等のランキングアルゴリズムにおいて、配列として確保された DocRepresentation の各要素が、文書内の単語位置情報（pos）を保持している必要があります。

アルゴリズムまたは振る舞い

  • 文書をクエリに適合する形式に変換する際、単語の出現位置（WordEntryPos）と、それに対応するクエリ項目（QueryItem）を紐付けます。
  • data 共用体により、コンパイルフェーズとマッピングフェーズでメモリを効率的に再利用します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • tsrank.c 内の Cover 関数（ランキング計算時）
  • tsrank.c 内の fillQueryRepresentationData 関数
  • tsrank.c 内の get_docrep 関数（文書表現の構築時）
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • QueryItem: クエリの構成要素。
  • WordEntry: 文書内の単語エントリ。
  • WordEntryPos: 単語の出現位置情報。
  • QueryRepresentation: クエリの解析済み表現。
  • CoverExt: 近接度計算時の拡張情報。

失敗モードと境界条件

  • palloc によるメモリ確保失敗時にエラーが発生する可能性があります。
  • 文書が非常に長い場合、DocRepresentation の配列サイズが大きくなるため、スタックやヒープの消費量に注意が必要です。

保守担当者向けメモ

  • data 共用体は、用途に応じて query 構造体と map 構造体を切り替えて使用します。コード修正時は、現在どちらのモードで利用されているかを tsrank.c 内の文脈から正確に判断してください。
