g_int_consistent

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_gist.c:46

目的

PostgreSQL の intarray モジュールにおける GiST インデックスの検索処理（consistent 関数）を実装し、クエリ条件とインデックスキーを照合します。

シグネチャまたは形状

Datum g_int_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY *entry（インデックスエントリ）
  • PG_GETARG_ARRAYTYPE_P_COPY(1): ArrayType *query（検索条件）
  • PG_GETARG_UINT16(2): StrategyNumber strategy（演算子戦略）
  • PG_GETARG_POINTER(4): bool *recheck（再チェックフラグの出力先）
  • **出力**:
  • Datum: 検索結果（bool）を PG_RETURN_BOOL で返す。
  • *recheck: RTSameStrategyNumber の場合に true が設定される。

前提条件と不変条件

  • strategy が RTSameStrategyNumber の場合、インデックスの再チェックが必要であることを示すため *recheck に true が設定されます。
  • BooleanSearchStrategy 以外の場合、query に対して CHECKARRVALID および PREPAREARR マクロによる検証と準備が行われます。

アルゴリズムまたは振る舞い

1. **戦略の判定**:

  • BooleanSearchStrategy: execconsistent を呼び出し、ブール検索を実行して終了します。
  • その他の戦略: query をソートし、strategy に応じて以下のヘルパー関数を呼び出します。
  • RTOverlapStrategyNumber: inner_int_overlap
  • RTSameStrategyNumber: リーフノードなら g_int_same、それ以外なら inner_int_contains
  • RTContainsStrategyNumber / RTOldContainsStrategyNumber: inner_int_contains
  • RTContainedByStrategyNumber / RTOldContainedByStrategyNumber: リーフノードなら inner_int_contains、それ以外なら true を返して全探索を強制します。

    • 2. **メモリ管理**: 処理の最後に pfree(query) を実行し、コピーされたクエリ配列を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックスエンジン（contrib/intarray/_int_gist.c:30 で PG_FUNCTION_INFO_V1 により登録）。
  • **呼び出し先**: execconsistent, inner_int_overlap, inner_int_contains, g_int_same。

関連データ構造

  • GISTENTRY: GiST インデックスのエントリ構造。
  • ArrayType: 整数配列の内部表現。
  • StrategyNumber: 演算子クラスの戦略番号。

失敗モードと境界条件

  • RTContainedByStrategyNumber 関連のコードは、intarray 1.4 以降では演算子クラスから <@ 演算子が削除されたため到達不能ですが、旧バージョンとの互換性のために保持されています。
  • strategy が定義外の場合は false を返します。

保守担当者向けメモ

  • 本関数は標準的な GiST インデックス用です。データがスパースな場合は gist__intbig_ops の使用が推奨されます。
  • RTContainedByStrategyNumber の非リーフノード処理では、空配列の存在を考慮して全探索（true）を行っています。
