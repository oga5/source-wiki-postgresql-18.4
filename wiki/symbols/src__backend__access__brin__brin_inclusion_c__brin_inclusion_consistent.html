brin_inclusion_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_inclusion.c:249

目的

BRIN（Block Range Index）において、特定のブロック範囲の集約値とスキャンキーを比較し、その範囲内に検索条件を満たすデータが存在する可能性があるかを判定する。

シグネチャまたは形状

Datum brin_inclusion_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): BrinDesc *（インデックス記述子）
  • PG_GETARG_POINTER(1): BrinValues *（ブロック範囲の集約値）
  • PG_GETARG_POINTER(2): ScanKey（検索条件）
  • **出力**:
  • Datum（bool 型として評価される。一致の可能性がある場合は true、ない場合は false）

前提条件と不変条件

  • PG_NARGS() == 3 であること。
  • column->bv_allnulls が false であること（全NULLの範囲は対象外）。
  • INCLUSION_UNMERGEABLE フラグが立っている場合、常に true を返す。

アルゴリズムまたは振る舞い

1. **初期チェック**: INCLUSION_UNMERGEABLE が真であれば、結合不可能な値が含まれているため、安全側に倒して true を返す。 2. **戦略別の評価**: key->sk_strategy に基づき処理を分岐する。

  • **配置戦略（Left, Right等）**: 対となる演算子の結果を論理否定して判定する。
  • **包含・重なり戦略**: 対応する演算子関数を直接呼び出し、その結果を返す。
  • **被包含戦略**: Overlap 演算子で判定し、真であれば true を返す。偽の場合は INCLUSION_CONTAINS_EMPTY フラグを確認する。
  • **隣接戦略**: Overlap 演算子で判定し、偽であればさらに Adjacent 演算子で判定する。

    • 3. **関数実行**: inclusion_get_strategy_procinfo を介して取得した関数を FunctionCall2Coll で実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での明示的な呼び出し元は提供されていません。
  • **呼び出し先**:
  • inclusion_get_strategy_procinfo: 演算子に対応する関数情報の取得。
  • FunctionCall2Coll: 取得した関数情報の実行。

関連データ構造

  • BrinDesc: インデックスのメタデータ。
  • BrinValues: ブロック範囲の集約値（INCLUSION_UNION 等のインデックスを含む）。
  • ScanKey: 検索条件。

失敗モードと境界条件

  • inclusion_get_strategy_procinfo が必要な戦略に対応する関数を見つけられない場合、エラーがスローされる。
  • 幾何データ型において、面積比較を用いる戦略は一部のケースで制限がある。
  • 空要素（Empty）を含む場合、戦略に応じて INCLUSION_CONTAINS_EMPTY フラグによる個別の判定が行われる。

保守担当者向けメモ

  • 本関数は古いシグネチャ（3引数）を使用している。
  • inclusion_get_strategy_procinfo は brin_minmax.c の実装と設計が共通しているため、修正時は整合性に注意すること。
  • 演算子の論理否定による実装は、対となる演算子が演算子クラスに定義されていることを前提としている。
