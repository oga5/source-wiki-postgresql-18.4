sepgsql_dml_privileges

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/dml.c:281

目的

SQL文の実行時（Executor）において、各テーブルおよびカラムに対するDML操作（SELECT, INSERT, UPDATE, DELETE）がSELinuxポリシーによって許可されているかを検証するメインエントリーポイントです。

シグネチャまたは形状

bool sepgsql_dml_privileges(List *rangeTbls, List *rteperminfos, bool abort_on_violation)

入力と出力

  • **入力**:
  • rangeTbls: 範囲テーブルのリスト。
  • rteperminfos: 権限チェック対象となる RTEPermissionInfo のリスト。
  • abort_on_violation: 権限違反時にエラーを発生させるかを示すフラグ。
  • **出力**:
  • bool: すべての権限チェックが成功した場合は true、失敗した場合は false。

前提条件と不変条件

  • rteperminfos リスト内の各エントリに対して、必要なSELinux権限（SEPG_DB_TABLE__*）が算出されます。
  • 継承テーブルの場合、親テーブルの属性番号は子テーブルの属性番号へ補正される必要があります。
  • 権限チェック対象が空（required == 0）の場合、そのエントリはスキップされます。

アルゴリズムまたは振る舞い

1. rteperminfos を反復し、各エントリの requiredPerms から必要なSELinux権限ビットを特定します。 2. 継承が有効な場合、find_all_inheritors を用いて対象テーブルおよび全子テーブルのOIDを取得します。 3. 各テーブルに対し、fixup_inherited_columns を呼び出してカラムレベルの権限ビットマップを子テーブルの属性番号に適合させます。 4. check_relation_privileges を呼び出し、テーブルおよびカラム単位の権限を検証します。 5. 権限違反が発生し、かつ abort_on_violation が真の場合、処理は中断されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/hooks.c:303 (DMLフック経由)
  • **呼び出し先**:
  • find_all_inheritors
  • fixup_inherited_columns
  • check_relation_privileges

関連データ構造

  • RTEPermissionInfo: 実行時の権限情報。
  • Bitmapset: カラムレベルの権限を管理するために使用。

失敗モードと境界条件

  • 権限チェックが失敗した場合、abort_on_violation の設定に応じて false を返すか、内部でエラーが発生します。
  • 継承テーブルの解決において、属性番号の不一致が発生する可能性があるため、fixup_inherited_columns による補正が必須です。

保守担当者向けメモ

  • 本関数は hooks.c から呼び出される重要なセキュリティフックです。変更時は sepgsql_avc_check_perms を含む権限キャッシュの整合性に注意してください。
  • 継承テーブルの処理において、大規模な階層構造を持つテーブルでは fixup_inherited_columns の呼び出し回数が増加し、パフォーマンスに影響を与える可能性があります。
