StoreSingleInheritance

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_inherits.c:507

目的

システムカタログ pg_inherits に、テーブルまたはインデックスの継承関係を示す単一のレコードを挿入します。

シグネチャまたは形状

void StoreSingleInheritance(Oid relationId, Oid parentOid, int32 seqNumber)

入力と出力

  • **入力**:
  • relationId: 子リレーションの OID。
  • parentOid: 親リレーションの OID。
  • seqNumber: 継承順序を示すシーケンス番号。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • 呼び出し元は、対象のリレーションに対して適切なロックを保持している必要があります。
  • pg_inherits カタログに対する RowExclusiveLock を内部で取得・解放します。

アルゴリズムまたは振る舞い

1. pg_inherits システムカタログを RowExclusiveLock モードで開きます。 2. 以下の値を設定したタプルを構築します。

  • inhrelid: relationId
  • inhparent: parentOid
  • inhseqno: seqNumber
  • inhdetachpending: false（デフォルトでデタッチ保留状態ではないとして挿入）

    • 3. CatalogTupleInsert を使用してカタログにレコードを書き込みます。 4. タプルを解放し、カタログを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/index.c (インデックス作成時)
  • src/backend/commands/indexcmds.c (パーティションインデックス作成時)
  • src/backend/commands/tablecmds.c (テーブル継承設定時)
  • **呼び出し先**:
  • table_open / table_close
  • heap_form_tuple / heap_freetuple
  • CatalogTupleInsert

関連データ構造

  • pg_inherits: 継承関係を保持するシステムカタログ。
  • Relation: pg_inherits カタログのアクセスに使用。

失敗モードと境界条件

  • カタログへの書き込み権限がない場合や、ロック取得に失敗した場合はエラーが発生します。
  • 既に同一の継承関係が存在する場合、カタログの制約（ユニークインデックス等）により挿入が失敗する可能性があります。

保守担当者向けメモ

  • 本関数は低レベルなカタログ操作を行うため、整合性チェックは呼び出し元（tablecmds.c 等）で行う必要があります。
  • inhdetachpending は常に false で初期化されるため、デタッチ処理中のレコードを挿入する用途には使用できません。
