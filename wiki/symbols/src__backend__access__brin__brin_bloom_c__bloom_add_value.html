bloom_add_value

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_bloom.c:368

目的

ブルームフィルタに対して指定された値をハッシュ化して追加し、フィルタ内の対応するビットを立てることで、値の存在可能性を記録します。

シグネチャまたは形状

static BloomFilter *bloom_add_value(BloomFilter *filter, uint32 value, bool *updated)

入力と出力

  • **入力**:
  • filter: 操作対象の BloomFilter 構造体へのポインタ。
  • value: 追加する uint32 型の値。
  • updated: フィルタの状態が変更された場合に true を格納するフラグ用ポインタ（NULL可）。
  • **出力**:
  • 操作後の BloomFilter へのポインタを返します。

前提条件と不変条件

  • filter は初期化済みである必要があります。
  • filter->nbits および filter->nhashes が正しく設定されている必要があります。
  • 「Less Hashing, Same Performance」論文に基づく二重ハッシュ手法（h1 + i * h2）を利用します。

アルゴリズムまたは振る舞い

1. BLOOM_SEED_1 および BLOOM_SEED_2 を用いて、入力値から2つのハッシュ値 h1, h2 を計算します。 2. filter->nhashes 回のループを実行し、各回で (h1 + i * h2) % filter->nbits を計算してビット位置を特定します。 3. 特定したビットが未設定（0）であれば、ビットを立て（1に設定）、filter->nbits_set をインクリメントします。 4. updated が指定されている場合、ビット操作が行われた際に true を設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin_bloom.c:580 にて brin_bloom_add_value からの呼び出しが確認されています。
  • **呼び出し先**: hash_bytes_uint32_extended を利用してハッシュ値を計算します。

関連データ構造

  • BloomFilter: フィルタのビットマップデータ、ビット数、ハッシュ関数数などを保持する構造体。

失敗モードと境界条件

  • filter->nbits が0の場合、剰余演算で除算エラーが発生する可能性があります。
  • filter->data 配列の範囲外アクセスを防ぐため、nbits は適切に割り当てられている必要があります。

保守担当者向けメモ

  • 本関数は brin_bloom_add_value の内部ロジックとして機能します。
  • ビット操作は 0x01 << bit を用いてバイト単位で行われるため、filter->data のメモリレイアウトに依存します。
