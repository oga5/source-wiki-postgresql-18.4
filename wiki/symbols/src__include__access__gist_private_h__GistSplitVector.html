GistSplitVector

定義

  • 種別: struct
  • 定義箇所: src/include/access/gist_private.h:235

目的

GiSTインデックスのページ分割処理において、分割後の左右のページに配置されるタプルの要約（Union）情報および、どちらの側にも配置可能なタプルを管理するために使用されます。

シグネチャまたは形状

typedef struct GistSplitVector
{
    GIST_SPLITVEC splitVector;
    Datum         spl_lattr[INDEX_MAX_KEYS];
    bool          spl_lisnull[INDEX_MAX_KEYS];
    Datum         spl_rattr[INDEX_MAX_KEYS];
    bool          spl_risnull[INDEX_MAX_KEYS];
    bool         *spl_dontcare;
} GistSplitVector;

入力と出力

  • **入力**: GISTSTATE や IndexTuple 配列から生成され、PickSplit メソッドとの間でやり取りされます。
  • **出力**: 分割後の左右のページにおけるキーの包含範囲（Union）および、配置の柔軟性を示すフラグ。

前提条件と不変条件

  • spl_lattr および spl_rattr は、それぞれ分割後の左側および右側のページに含まれる全サブキーの Union を保持する必要があります。
  • spl_dontcare は、ペナルティなしで左右どちらのページにも配置可能なタプルを識別するために使用されます。

アルゴリズムまたは振る舞い

1. gistunionsubkey を介して、分割後の各サイドのキーの Union を計算します。 2. placeOne 等の関数により、各タプルを左右どちらのページに配置するかを決定する際に本構造体が参照されます。 3. gistUserPicksplit を通じて、ユーザー定義の PickSplit メソッドの結果と統合されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gist.c および src/backend/access/gist/gistsplit.c 内の各関数（gistunionsubkey, placeOne, gistUserPicksplit など）。
  • **呼び出し先**: ユーザー定義の PickSplit メソッド（gistUserPicksplit 経由）。

関連データ構造

  • GIST_SPLITVEC: splitVector フィールドとして保持され、ユーザー定義の分割ロジックとのインターフェースとなります。
  • GISTSTATE: インデックスのサポート関数情報を提供します。

失敗モードと境界条件

  • INDEX_MAX_KEYS を超えるキー数を持つインデックスには対応していません。
  • spl_dontcare 配列のメモリ割り当てが適切に行われていない場合、メモリ破壊の可能性があります。

保守担当者向けメモ

  • 本構造体は gist.c および gistsplit.c におけるページ分割ロジックの中核です。変更を加える際は、PickSplit メソッドとの互換性に注意してください。
  • spl_dontcare のフラグがどのように活用されているかについては、placeOne 関数の実装を参照してください。
