xl_brin_insert

定義

  • 種別: struct
  • 定義箇所: src/include/access/brin_xlog.h:63

目的

BRIN (Block Range Index) インデックスへのタプル挿入操作を WAL (Write-Ahead Logging) に記録するためのデータ構造です。リカバリ時にインデックスページを再構築するために必要なメタデータを保持します。

シグネチャまたは形状

typedef struct xl_brin_insert
{
    BlockNumber heapBlk;
    BlockNumber pagesPerRange;
    OffsetNumber offnum;
} xl_brin_insert;

入力と出力

  • **入力**: 挿入対象のヒープブロック番号 (heapBlk)、範囲あたりのページ数 (pagesPerRange)、およびインデックスページ内の挿入位置 (offnum)。
  • **出力**: なし（本構造体は WAL レコードのペイロードとして書き込まれます）。

前提条件と不変条件

  • SizeOfBrinInsert マクロ（offsetof(xl_brin_insert, offnum) + sizeof(OffsetNumber)）を使用して、WAL レコードのサイズを正しく計算する必要があります。

アルゴリズムまたは振る舞い

1. **記録**: インデックス挿入時、この構造体に情報を格納し、XLogInsert を介して WAL に書き出します。 2. **再生**: リカバリ時、brin_redo が WAL レコードから本構造体を読み取ります。その後、brin_xlog_insert 等の関数がこのメタデータを使用して対象のインデックスページを特定し、挿入操作を再現します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin_pageops.c におけるインデックス挿入処理。
  • **呼び出し先**: src/backend/access/brin/brin_xlog.c の Redo ロジックおよび src/backend/access/rmgrdesc/brindesc.c のログ記述子。

関連データ構造

  • XLOG_BRIN_INSERT: 本構造体を使用する WAL 操作種別。
  • brin_redo: 本構造体を利用してリカバリを実行する関数。

失敗モードと境界条件

  • 構造体内の各フィールドが不正なブロック番号やオフセットを指している場合、リカバリ時にページ操作エラーが発生する可能性があります。

保守担当者向けメモ

  • 新しいフィールドを追加する場合は、SizeOfBrinInsert マクロの定義を適切に更新してください。
  • 本構造体は brin_xlog.c の Redo ロジックと密接に結合しているため、変更時は必ず対となる実装を確認してください。
