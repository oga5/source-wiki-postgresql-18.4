ExtendBufferedRelLocal

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/localbuf.c:344

目的

一時テーブル（ローカルリレーション）の物理的なサイズを拡張し、新しいブロックに対応するローカルバッファを割り当てて初期化する。

シグネチャまたは形状

BlockNumber ExtendBufferedRelLocal(BufferManagerRelation bmr,
                                   ForkNumber fork,
                                   uint32 flags,
                                   uint32 extend_by,
                                   BlockNumber extend_upto,
                                   Buffer *buffers,
                                   uint32 *extended_by)

入力と出力

  • **入力**:
  • bmr: 対象リレーションのストレージマネージャ情報。
  • fork: 拡張対象のフォーク番号。
  • extend_by: 拡張するブロック数。
  • extend_upto: 拡張の上限ブロック番号（指定時）。
  • buffers: 割り当てられたバッファIDを格納する配列。
  • **出力**:
  • 戻り値: 拡張開始位置のブロック番号。
  • extended_by: 実際に拡張されたブロック数。

前提条件と不変条件

  • LocalBufHash が初期化済みであること（未初期化時は内部で InitLocalBuffers() を呼び出す）。
  • 拡張後のブロック数が MaxBlockNumber を超えないこと。
  • extend_upto が指定されている場合、現在のリレーションサイズがその値を超えないこと。

アルゴリズムまたは振る舞い

1. **バッファ確保**: GetLocalVictimBuffer() を使用して必要な数のローカルバッファを確保し、メモリをゼロクリアする。 2. **タグ登録**: 各バッファに新しいブロックタグを割り当て、LocalBufHash に登録する。既にタグが存在する場合は既存バッファを再利用する。 3. **I/O 準備**: StartLocalBufferIO() を呼び出し、バッファの状態を更新する。 4. **物理拡張**: smgrzeroextend() を呼び出し、ストレージ層で物理的にファイルを拡張する。 5. **状態確定**: 拡張したバッファの BM_VALID ビットを立て、統計情報を更新する。

呼び出し元と呼び出し先

  • **呼び出し先**:

関連データ構造

  • BufferManagerRelation: リレーションの管理構造体。
  • LocalBufferLookupEnt: ハッシュテーブルの検索エントリ。
  • BufferDesc: バッファ記述子。

失敗モードと境界条件

  • リレーションが MaxBlockNumber に達している場合、ERROR を発生させる。
  • メモリ不足やストレージ層でのエラー発生時は、PostgreSQL の標準的なエラー処理機構により中断される。

保守担当者向けメモ

  • 本関数はプロセスローカルなバッファのみを対象とするため、共有バッファマネージャのような複雑なロックは不要だが、アトミックな状態更新（pg_atomic_...）は必須である。
  • buffers 配列のサイズが extend_by 以上であることを呼び出し元が保証する必要がある。
