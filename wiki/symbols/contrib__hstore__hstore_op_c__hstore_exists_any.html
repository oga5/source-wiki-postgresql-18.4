hstore_exists_any

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_op.c:161

目的

hstore 値の中に、指定されたキー配列のいずれかが存在するかを判定する。

シグネチャまたは形状

Datum hstore_exists_any(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_HSTORE_P(0): 検索対象の HStore 構造体。
  • PG_GETARG_ARRAYTYPE_P(1): 検索するキーの集合を含む ArrayType。
  • **出力**:
  • PG_RETURN_BOOL(res): いずれかのキーが見つかった場合は true、それ以外は false を返す。

前提条件と不変条件

  • hstoreArrayToPairs によって変換されたキー配列は、内部的にソート済みである必要がある。
  • hstoreFindKey の検索において、lowbound 変数により検索範囲が最適化される。

アルゴリズムまたは振る舞い

1. 入力された ArrayType を hstoreArrayToPairs を用いて Pairs 構造体の配列に変換する。 2. ソート済みのキー配列を順次走査する。 3. 各キーに対し hstoreFindKey を呼び出し、HStore 内を検索する。 4. lowbound を更新しながら検索を行うことで、前回の検索結果に基づいた範囲絞り込みを行い、効率的に探索する。 5. いずれかのキーが発見された時点で検索を打ち切り、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ経由で呼び出される（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • hstoreArrayToPairs: キー配列の変換とソート。
  • hstoreFindKey: HStore 内のバイナリサーチ。

関連データ構造

  • HStore: 検索対象となる hstore 内部表現。
  • Pairs: hstoreArrayToPairs が生成するキーと長さのペア構造体。
  • ArrayType: 入力されたキーの配列。

失敗モードと境界条件

  • 入力された hstore が空の場合、検索は成功せず false を返す。
  • 指定されたキー配列が空の場合、ループは実行されず false を返す。

保守担当者向けメモ

  • 本関数は hstore のソート特性を利用して検索を高速化している。hstoreFindKey に渡す lowbound の更新ロジックは、この最適化の核心であるため変更時には注意が必要。
  • hstoreArrayToPairs が重複排除を行うため、入力配列に重複があっても検索結果には影響しない。
