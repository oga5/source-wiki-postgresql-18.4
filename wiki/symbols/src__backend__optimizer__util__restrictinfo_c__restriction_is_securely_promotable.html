restriction_is_securely_promotable

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/restrictinfo.c:421

目的

指定された RestrictInfo が、リレーションのセキュリティ制約を侵害することなく、そのリレーションのベース制約として昇格（適用）可能かどうかを判定します。

シグネチャまたは形状

bool restriction_is_securely_promotable(RestrictInfo *restrictinfo, RelOptInfo *rel)

入力と出力

  • **入力**:
  • restrictinfo: 評価対象の RestrictInfo 構造体。
  • rel: 比較対象となるリレーションの RelOptInfo 構造体。
  • **出力**:
  • bool: 昇格が安全であれば true、そうでなければ false を返します。

前提条件と不変条件

  • rel->baserestrict_min_security が、当該リレーションに対して適用されるべきベース制約の最小セキュリティレベルを保持している必要があります。

アルゴリズムまたは振る舞い

以下のいずれかの条件を満たす場合に true を返します。 1. restrictinfo の security_level が、リレーションの baserestrict_min_security 以下である場合。 2. restrictinfo が leakproof（情報漏洩耐性がある）とマークされている場合。 それ以外の場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/indxpath.c:2607
  • src/backend/optimizer/path/tidpath.c:248
  • src/backend/optimizer/path/tidpath.c:451
  • **呼び出し先**:
  • 直接的な関数呼び出しは含まれていません。

関連データ構造

  • RestrictInfo: 制約条件のメタデータ（セキュリティレベルや漏洩耐性フラグ）を保持します。
  • RelOptInfo: リレーションの最適化に関する情報（ベース制約の最小セキュリティレベルなど）を保持します。

失敗モードと境界条件

  • restrictinfo または rel が NULL の場合、ポインタ参照エラーが発生する可能性があります（ソースコード上での明示的なチェックは存在しません）。

保守担当者向けメモ

  • 本関数は、セキュリティレベルに基づく制約のプッシュダウンや昇格の可否を判断する重要なセキュリティ境界です。leakproof 属性の変更やセキュリティレベルの定義変更時には、本関数のロジックが意図通りか再確認してください。
