ExpandedRecordFieldInfo

定義

  • 種別: struct
  • 定義箇所: src/include/utils/expandedrecord.h:168

目的

レコード型（複合型）の特定のフィールドに関するメタデータ（属性番号、型、照合順序）を保持し、展開されたレコード（Expanded Record）内でのフィールド操作を支援するために使用されます。

シグネチャまたは形状

typedef struct ExpandedRecordFieldInfo
{
    int         fnumber;        /* レコード内の属性番号 */
    Oid         ftypeid;        /* フィールドの型ID */
    int32       ftypmod;        /* フィールドの型修飾子 */
    Oid         fcollation;     /* フィールドの照合順序（存在する場合） */
} ExpandedRecordFieldInfo;

入力と出力

本構造体自体はデータ保持用であり、関数ではありません。フィールドの属性情報を一括して管理・伝達するためのコンテナとして機能します。

前提条件と不変条件

  • fnumber は、対象となるレコード型の有効な属性番号である必要があります。
  • ftypeid および ftypmod は、PostgreSQLの型システムにおいて整合性が取れている必要があります。

アルゴリズムまたは振る舞い

展開されたレコードのフィールドアクセスや更新処理において、対象フィールドの型情報を特定するために参照されます。特に、動的な型チェックや、フィールド値の変換が必要な場面で利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/plpgsql.h で構造体メンバとして利用されています。また、src/backend/utils/adt/expandedrecord.c および src/include/utils/expandedrecord.h 内の関数引数として参照されています。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ExpandedRecordHeader: 展開されたレコードのメイン構造体。
  • TupleDesc: 複合型の記述子。本構造体の情報は TupleDesc から抽出されることが一般的です。

失敗モードと境界条件

  • 無効な fnumber が指定された場合、後続のフィールドアクセス処理でエラーが発生する可能性があります。
  • ftypeid が不正な場合、型変換やデータ操作時に予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は、レコードのフィールド情報を一時的に保持する軽量な構造体です。
  • src/pl/plpgsql 等の外部モジュールからも参照されているため、構造体の変更は広範囲に影響を及ぼす可能性があります。
