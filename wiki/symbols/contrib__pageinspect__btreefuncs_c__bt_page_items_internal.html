bt_page_items_internal

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/btreefuncs.c:622

目的

B-treeインデックスの指定されたページ内の各タプルを走査し、その内容をSRF（Set Returning Function）として出力する。

シグネチャまたは形状

static Datum bt_page_items_internal(PG_FUNCTION_ARGS, enum pageinspect_version ext_version)

入力と出力

  • **入力**:
  • PG_FUNCTION_ARGS: SQL関数呼び出しの引数（リレーション名、ブロック番号）。
  • ext_version: APIのバージョン（PAGEINSPECT_V1_8 または PAGEINSPECT_V1_9）。
  • **出力**:
  • Datum: ページ内の各タプルの詳細情報を含む行データ。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要。
  • 対象リレーションはB-treeインデックスである必要がある（bt_index_block_validate による検証）。
  • 読み取り専用の AccessShareLock を使用してインデックスを保護する。

アルゴリズムまたは振る舞い

1. **初期化**: SRF_IS_FIRSTCALL の際、対象インデックスをオープンし、指定ブロックをメモリに読み込む。 2. **データコピー**: バッファのピンを保持し続けることを避けるため、ページ内容をローカルメモリにコピーし、即座にバッファとリレーションを解放する。 3. **状態管理**: FuncCallContext を使用して、現在のオフセット番号やページ属性（リーフページか、最右端か）を保持する。 4. **反復処理**: fctx->call_cntr が max_calls に達するまで、bt_page_print_tuples を呼び出してタプルを変換し、結果を順次返す。 5. **終了**: ページが削除されている場合はタプルを解釈せず、処理を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pageinspect/btreefuncs.c 内のSQLラッパー関数（bt_page_items_1_9 および bt_page_items_1_8）。
  • **呼び出し先**:
  • bt_index_block_validate
  • bt_page_print_tuples
  • SRF 関連API（SRF_FIRSTCALL_INIT, SRF_RETURN_NEXT 等）

関連データ構造

  • ua_page_items: ページ内容、オフセット、ページ属性を保持する状態構造体。
  • BTPageOpaqueData: B-treeページヘッダーの不透明領域。

失敗モードと境界条件

  • **権限エラー**: スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー終了。
  • **検証エラー**: bt_index_block_validate により、不正なブロック番号や非B-treeインデックスに対してエラーが発生する。
  • **削除済みページ**: ページが削除されている場合、NOTICEを出力し、タプルを返さずに終了する。
  • **型不一致**: 戻り値の型が複合型でない場合、エラー終了する。

保守担当者向けメモ

  • 本関数は pageinspect バージョン間の互換性を維持するため、ext_version 引数を通じてブロック番号の型（uint32 vs int64）を切り替えている。
  • ページ内容をローカルメモリにコピーする設計のため、大規模なインデックスページを扱う際はメモリ消費に注意すること。
