PLy_exception_set

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_elog.c:489

目的

Python の例外オブジェクトに対して、フォーマットされたメッセージ文字列をセットします。

シグネチャまたは形状

void PLy_exception_set(PyObject *exc, const char *fmt, ...)

入力と出力

  • **入力**:
  • exc: セット対象の Python 例外クラスまたはインスタンス (PyObject*)。
  • fmt: メッセージのフォーマット文字列 (const char*)。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**: なし（void）。内部で PyErr_SetString を呼び出し、Python インタプリタの例外状態を更新します。

前提条件と不変条件

  • fmt は dgettext による翻訳対象となります。
  • 内部バッファサイズは 1024 バイトに固定されており、これを超えるメッセージは切り詰められます。

アルゴリズムまたは振る舞い

1. 可変長引数リストを va_start で初期化します。 2. dgettext を用いてフォーマット文字列を翻訳し、vsnprintf で固定長バッファ buf にメッセージを書き込みます。 3. va_end で引数リストを終了します。 4. PyErr_SetString(exc, buf) を呼び出し、指定された例外型と構築したメッセージを Python のエラー状態として設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_cursorobject.c (カーソル操作時のバリデーションエラー)
  • src/pl/plpython/plpy_plpymodule.c (引数チェック等のモジュールレベルエラー)
  • **呼び出し先**:
  • dgettext (国際化対応)
  • vsnprintf (文字列構築)
  • PyErr_SetString (Python C API)

関連データ構造

  • PyObject: Python のオブジェクト表現。

失敗モードと境界条件

  • メッセージが 1024 バイトを超える場合、vsnprintf の仕様に従い切り詰められます。
  • PyErr_SetString が失敗した場合の戻り値チェックは行われません。

保守担当者向けメモ

  • 本関数は pg_attribute_printf(2, 3) 属性が付与されており、コンパイル時にフォーマット文字列の整合性がチェックされます。
  • Python の例外状態をセットするのみであり、PostgreSQL 側の ereport を直接発行するわけではありません。エラーを PostgreSQL に伝播させる場合は、この後に Python の例外をキャッチして PLy_elog_impl 等へ渡す必要があります。
