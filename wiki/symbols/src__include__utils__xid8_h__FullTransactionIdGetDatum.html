FullTransactionIdGetDatum

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/xid8.h:23

目的

64ビットトランザクションID (FullTransactionId) を、PostgreSQL の関数呼び出し規約で使用される汎用データ型 Datum に変換する。

シグネチャまたは形状

static inline Datum FullTransactionIdGetDatum(FullTransactionId X)

入力と出力

  • **入力**: FullTransactionId 型のトランザクションID。
  • **出力**: Datum 型に変換された値。

前提条件と不変条件

  • 入力される FullTransactionId は有効な値であること。
  • 内部的に U64FromFullTransactionId を介して 64 ビット整数へ変換されるため、FullTransactionId 型の内部表現と UInt64 の互換性が維持されている必要がある。

アルゴリズムまたは振る舞い

1. 引数 X (FullTransactionId) を U64FromFullTransactionId 関数に渡し、64ビット整数値に変換する。 2. 得られた 64 ビット整数を UInt64GetDatum マクロに渡し、Datum 型へ変換して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/xid8funcs.c:621 (SRF_RETURN_NEXT 内)
  • src/include/utils/xid8.h:30 (PG_RETURN_FULLTRANSACTIONID マクロ定義内)
  • **呼び出し先**:
  • U64FromFullTransactionId (型変換用)
  • UInt64GetDatum (Datum 変換用)

関連データ構造

  • FullTransactionId: 64ビットトランザクションIDを保持する型。
  • Datum: PostgreSQL の汎用データ表現型。

失敗モードと境界条件

  • 特筆すべき失敗モードはない。ただし、FullTransactionId が無効な値である場合、変換後の Datum もその無効な値を保持したままとなる。

保守担当者向けメモ

  • 本関数は非常に軽量なラッパーであり、主に SQL 関数から FullTransactionId を戻り値として返す際に使用される。
  • 変更を加える際は、access/transam.h で定義されている FullTransactionId の構造定義との整合性に注意すること。
