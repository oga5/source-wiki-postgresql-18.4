ARRAY_TYPE

定義

  • 種別: enum
  • 定義箇所: src/interfaces/ecpg/ecpglib/ecpglib_extern.h:27

目的

ECPGライブラリにおいて、ホスト変数が配列であるか、またその配列の種類（標準配列かベクトルか）を識別するために使用されます。

シグネチャまたは形状

enum ARRAY_TYPE {
    ECPG_ARRAY_ERROR,
    ECPG_ARRAY_NOT_SET,
    ECPG_ARRAY_ARRAY,
    ECPG_ARRAY_VECTOR,
    ECPG_ARRAY_NONE
};

入力と出力

  • 入力: 該当なし（列挙型定義）
  • 出力: 該当なし（列挙型定義）

前提条件と不変条件

  • ECPG_ARRAY_NOT_SET は、初期化状態や型情報が未確定であることを示すデフォルト値として使用されます。

アルゴリズムまたは振る舞い

  • data.c において、配列の区切り文字の判定（array_delimiter）や境界チェック（array_boundary）の制御フラグとして機能します。
  • execute.c において、型情報キャッシュ（ECPGtype_information_cache）の構築や、SQL実行時の型判定に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/interfaces/ecpg/ecpglib/data.c
  • src/interfaces/ecpg/ecpglib/execute.c
  • src/interfaces/ecpg/ecpglib/ecpglib_extern.h（構造体メンバとしての定義）
  • 呼び出し先: 該当なし（列挙型のため）

関連データ構造

  • struct variable: メンバとして enum ARRAY_TYPE isarray を保持します。
  • struct ECPGtype_information_cache: 型情報のキャッシュ管理に使用されます。

失敗モードと境界条件

  • ECPG_ARRAY_ERROR: 配列処理中にエラーが発生したことを示す状態として使用されます。

保守担当者向けメモ

  • 本列挙型は、ECPGのランタイムライブラリがホスト変数の型を動的に解決する際の重要な判断基準となります。新しい配列型や特殊なデータ構造をサポートする際は、この列挙型に値を追加し、data.c 内の判定ロジックを更新する必要があります。
