validate_relation_kind

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/table.c:137

目的

指定されたリレーションがテーブルとして有効であるかを検証し、不適切な種別（インデックスや複合型など）であればエラーを発生させるガードレール関数です。

シグネチャまたは形状

static inline void validate_relation_kind(Relation r)

入力と出力

  • **入力**: Relation r（検証対象のリレーション構造体）
  • **出力**: なし（正常時は何も返さず終了）
  • **副作用**: 不適切な種別の場合、ereport(ERROR) によりトランザクションを中断するエラーを発行します。

前提条件と不変条件

  • 引数 r が有効な Relation 構造体を指していること。
  • r->rd_rel が初期化済みであること。

アルゴリズムまたは振る舞い

1. r->rd_rel->relkind を参照し、以下のいずれかに該当するかを確認します。

  • RELKIND_INDEX
  • RELKIND_PARTITIONED_INDEX
  • RELKIND_COMPOSITE_TYPE

    • 2. 上記のいずれかであれば、ERRCODE_WRONG_OBJECT_TYPE を伴うエラーを発生させます。 3. エラーメッセージには、対象リレーション名と、サポートされていない種別であることを示す詳細情報が含まれます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/table/table.c 内のテーブルオープン処理
  • src/backend/access/index/indexam.c
  • src/backend/access/sequence/sequence.c

    • (注: ツールによる静的解析結果に基づく)

  • **呼び出し先**:
  • RelationGetRelationName (マクロ)
  • errdetail_relkind_not_supported (関数)
  • ereport (マクロ)

関連データ構造

  • Relation: リレーションのメタデータを保持する構造体。
  • Form_pg_class (rd_rel): リレーションの種別（relkind）を保持する構造体。

失敗モードと境界条件

  • **失敗モード**: 対象リレーションがテーブル以外の種別（インデックス、複合型など）である場合、ereport(ERROR) により処理が停止します。
  • **境界条件**: r が NULL の場合、あるいは r->rd_rel が未初期化の場合は未定義の動作となります。

保守担当者向けメモ

  • 本関数は static inline で定義されており、複数のソースファイル（table.c, indexam.c, sequence.c）で個別に定義・利用されています。変更時は各ファイルでの定義の一貫性に注意してください。
  • 新しいリレーション種別が追加された際、テーブルとして許可すべきでない場合は本関数の条件式に追加する必要があります。
