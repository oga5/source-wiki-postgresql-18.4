pg_tolower

定義

  • 種別: function
  • 定義箇所: src/port/pgstrcasecmp.c:121

目的

ロケールを考慮しつつ、プラットフォーム間で一貫した小文字変換を行う。特に7ビットASCII範囲外の文字に対しては環境依存のロケール設定を反映させるハイブリッドな変換機能を提供する。

シグネチャまたは形状

unsigned char pg_tolower(unsigned char ch);

入力と出力

  • **入力**: unsigned char ch （変換対象の文字）
  • **出力**: unsigned char （変換後の小文字、または変換不要な場合は元の文字）

前提条件と不変条件

  • 入力値は unsigned char 型であること。
  • 7ビットASCII範囲（'A'-'Z'）の文字は、ロケール設定に関わらず常に小文字に変換される。
  • ASCII範囲外の文字については、IS_HIGHBIT_SET(ch) が真である場合にのみ、標準ライブラリの isupper および tolower を使用して変換を試みる。

アルゴリズムまたは振る舞い

1. 入力文字 ch が 'A' から 'Z' の範囲内にある場合、ASCIIコードの差分を加算して小文字に変換する。 2. 上記以外で、IS_HIGHBIT_SET(ch) が真（8ビット目がセットされている）かつ isupper(ch) が真である場合、標準ライブラリの tolower(ch) を呼び出して変換する。 3. いずれの条件にも該当しない場合、入力された ch をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/datetime.c
  • src/backend/utils/adt/like.c
  • src/backend/utils/adt/pg_locale_libc.c
  • src/backend/utils/misc/tzparser.c
  • src/bin/psql/tab-complete.in.c
  • **呼び出し先**:
  • 標準ライブラリ関数: isupper, tolower

関連データ構造

  • なし

失敗モードと境界条件

  • **マルチバイト文字**: 本実装はマルチバイト文字セットに対しては不完全であり、適切な変換が行われない可能性がある。
  • **ロケール依存**: ASCII範囲外の文字変換は実行時のロケール設定に依存するため、環境によって結果が異なる可能性がある。

保守担当者向けメモ

  • トルコ語ロケール等の特殊なケースでの不整合を避けるため、ASCII範囲外の文字に対してのみロケール関数を呼び出す設計となっている。
  • 厳密なASCII変換が必要な場合は、本関数ではなく pg_ascii_tolower を使用すること。
  • src/backend/parser/scansup.c の downcase_truncate_identifier() とロジックの一貫性を保つ必要がある。
