unicodeStyleFormat

定義

  • 種別: struct
  • 定義箇所: src/fe_utils/print.c:126

目的

Unicode罫線を使用した表形式出力において、行、列、境界線のスタイル定義および、折り返しや改行時の装飾文字を保持するための構造体です。

シグネチャまたは形状

typedef struct unicodeStyleFormat
{
	unicodeStyleRowFormat row_style[2];
	unicodeStyleColumnFormat column_style[2];
	unicodeStyleBorderFormat border_style[2];
	const char *header_nl_left;
	const char *header_nl_right;
	const char *nl_left;
	const char *nl_right;
	const char *wrap_left;
	const char *wrap_right;
	bool		wrap_right_border;
} unicodeStyleFormat;

入力と出力

本構造体はデータ保持用であり、関数ではありません。各フィールドには、Unicode罫線描画に必要な文字列定数や、行・列・境界線のスタイル定義が格納されます。

前提条件と不変条件

  • row_style、column_style、border_style は、それぞれ2つの要素を持つ配列として定義されており、異なる状態（例：通常行と強調行など）を保持することを想定しています。
  • 各ポインタフィールド（header_nl_left 等）は、有効な文字列リテラルを指す必要があります。

アルゴリズムまたは振る舞い

本構造体は、psql 等のフロントエンドにおける出力整形処理において、Unicode罫線を用いた表の描画ルールを定義するために使用されます。描画ロジックは、この構造体の各フィールドを参照して、適切な罫線文字や装飾文字を選択・出力します。

呼び出し元と呼び出し先

  • 呼び出し元: src/fe_utils/print.c 内の unicode_style 定数定義（140行目）で初期化に使用されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • unicodeStyleRowFormat: 行のスタイル定義。
  • unicodeStyleColumnFormat: 列のスタイル定義。
  • unicodeStyleBorderFormat: 境界線のスタイル定義。

失敗モードと境界条件

  • 構造体内のポインタフィールドに NULL が設定された場合、描画ロジック側で適切に処理されないとセグメンテーション違反を引き起こす可能性があります。
  • wrap_right_border はブール値であり、折り返し時の右端境界線の描画有無を制御します。

保守担当者向けメモ

  • 本構造体は src/fe_utils/print.c 内の静的定数 unicode_style を構成するために使用されています。
  • スタイル定義を追加・変更する場合は、関連する unicodeStyleRowFormat 等の定義も併せて確認してください。
