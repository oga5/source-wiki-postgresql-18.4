ExprSetupInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/execExpr.c:56

目的

式評価の初期化フェーズにおいて、タプルスロットのデフォーム（属性抽出）に必要な範囲情報と、式に含まれるサブプラン情報を保持するために使用されます。

シグネチャまたは形状

typedef struct ExprSetupInfo
{
    AttrNumber  last_inner;
    AttrNumber  last_outer;
    AttrNumber  last_scan;
    AttrNumber  last_old;
    AttrNumber  last_new;
    List       *multiexpr_subplans;
} ExprSetupInfo;

入力と出力

  • **入力**: 式ツリーの走査過程で、各スロット（inner, outer, scan, old, new）から参照される最大属性番号および MULTIEXPR 型のサブプランノードが本構造体に蓄積されます。
  • **出力**: 蓄積された情報は ExecPushExprSetupSteps に渡され、式評価の先頭にタプルデフォーム命令（EEOP_FETCHSOME）を挿入するために利用されます。

前提条件と不変条件

  • 各 last_ フィールドは、対応するタプルスロットから読み取られるべき属性の最大番号を保持します。
  • multiexpr_subplans は、式ツリー内に存在する MULTIEXPR サブプランのリストを保持します。

アルゴリズムまたは振る舞い

1. 式のコンパイル開始時に ExprSetupInfo が初期化されます。 2. expr_setup_walker が式ツリーを再帰的に走査し、各ノードが参照する属性番号を特定して、本構造体の対応する last_ フィールドを更新します。 3. 走査完了後、ExecPushExprSetupSteps が本構造体の情報を参照し、必要な属性をデフォームするための命令を ExprState に追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecInitExpr 等の式初期化関数から、ローカル変数としてインスタンス化されます。
  • **呼び出し先**: expr_setup_walker（走査用）、ExecPushExprSetupSteps（命令生成用）に引数として渡されます。

関連データ構造

  • ExprState: 式評価の実行状態を保持する構造体。
  • ExprEvalStep: 式評価の各ステップを定義する命令構造体。
  • List: multiexpr_subplans を保持するために使用される PostgreSQL のリスト構造。

失敗モードと境界条件

  • 属性番号が 0 以下の場合はデフォーム不要とみなされます。
  • 式ツリーが非常に深い場合、expr_setup_walker の再帰呼び出しによりスタックオーバーフローが発生する可能性があります（check_stack_depth() による保護が前提）。

保守担当者向けメモ

  • 本構造体は式評価の「コンパイル」フェーズでのみ使用されます。
  • last_ フィールドの更新ロジックを変更する場合は、expr_setup_walker 内のノード走査ロジックと整合性を取る必要があります。
