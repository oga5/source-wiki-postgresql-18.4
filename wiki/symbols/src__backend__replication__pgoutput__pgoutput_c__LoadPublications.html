LoadPublications

定義

  • 種別: function
  • 定義箇所: src/backend/replication/pgoutput/pgoutput.c:1799

目的

指定されたパブリケーション名のリストから、システムカタログ上の Publication オブジェクトのリストを取得します。

シグネチャまたは形状

static List *LoadPublications(List *pubnames)

入力と出力

  • **入力**: pubnames (パブリケーション名の文字列リスト)
  • **出力**: 見つかったパブリケーションオブジェクト (Publication) のリスト。存在しないパブリケーションはスキップされます。

前提条件と不変条件

  • 入力された pubnames は文字列のリストである必要があります。
  • 実行時にシステムカタログからパブリケーション情報を検索するため、データベース接続およびカタログへのアクセス権限が必要です。

アルゴリズムまたは振る舞い

1. pubnames リストを走査し、各要素に対して GetPublicationByName を呼び出します。 2. パブリケーションが存在する場合、そのオブジェクトを結果リストに追加します。 3. パブリケーションが存在しない場合、WARNING レベルのログを出力し、処理を継続します。 4. 最終的に収集されたパブリケーションのリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/pgoutput/pgoutput.c:2110 において、data->publications の初期化時に呼び出されています。
  • **呼び出し先**: GetPublicationByName (システムカタログからパブリケーションを取得する関数)。

関連データ構造

  • List: PostgreSQLの汎用リスト構造。
  • Publication: パブリケーションの定義情報を保持する構造体。

失敗モードと境界条件

  • 指定されたパブリケーション名がシステムカタログに存在しない場合、エラーではなく WARNING が発生し、該当するパブリケーションはリストから除外されます。
  • pubnames が空のリストである場合、空のリストを返します。

保守担当者向けメモ

  • 本関数は pgoutput プラグインの初期化プロセスの一部として使用されます。
  • パブリケーションが存在しない場合の挙動は WARNING ですが、これは WAL デコード時点でのカタログ状態に依存するため、運用環境でのパブリケーション作成タイミングに注意が必要です。
