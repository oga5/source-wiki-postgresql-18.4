calculate_relation_size

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/dbsize.c:325

目的

指定されたリレーション（テーブル、インデックス等）の特定のフォーク（データ、FSM、VMなど）について、物理的なディスク上の合計サイズをバイト単位で計算します。

シグネチャまたは形状

int64 calculate_relation_size(RelFileLocator *rfn, ProcNumber backend, ForkNumber forknum)

入力と出力

  • **入力**:
  • rfn: 対象リレーションの RelFileLocator 構造体へのポインタ。
  • backend: バックエンドプロセス番号（通常のリレーションは InvalidBackendId）。
  • forknum: 計測対象のフォーク番号。
  • **出力**:
  • int64: 合計ファイルサイズ（バイト単位）。

前提条件と不変条件

  • rfn が指すリレーションのパスが relpathbackend によって正しく解決可能であること。
  • 実行ユーザーが対象オブジェクトへの適切なアクセス権限を有していること（呼び出し元でチェック済みである前提）。

アルゴリズムまたは振る舞い

1. relpathbackend を使用して、指定されたリレーションフォークのベースパスを取得します。 2. segcount を 0 から開始し、path、path.1、path.2 ... と順次ファイルパスを生成します。 3. stat() を用いて各セグメントファイルのサイズを取得し、totalsize に加算します。 4. CHECK_FOR_INTERRUPTS() をループ内で呼び出し、クエリのキャンセル要求を監視します。 5. stat() が ENOENT（ファイルが存在しない）を返した時点でセグメントの走査を終了します。 6. 最終的な totalsize を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/dbsize.c 内の各関数（calculate_table_size, calculate_toast_table_size, calculate_total_relation_size など）。
  • **呼び出し先**:
  • relpathbackend: パス生成。
  • stat(): ファイル属性取得。
  • CHECK_FOR_INTERRUPTS(): 割り込みチェック。
  • ereport(): ファイルアクセスエラー時の例外処理。

関連データ構造

  • RelFileLocator: リレーションを特定するための識別子。
  • RelPathStr: ファイルパス文字列を保持する構造体。
  • struct stat: ファイルシステムから取得したファイル情報を保持する標準構造体。

失敗モードと境界条件

  • **ファイルアクセスエラー**: stat() が ENOENT 以外のエラーを返した場合、ereport(ERROR, ...) によりトランザクションが中断されます。
  • **割り込み**: CHECK_FOR_INTERRUPTS() により、ユーザーによるクエリキャンセルが発生した場合は処理が中断されます。

保守担当者向けメモ

  • 本関数はリレーションの物理ファイルを直接走査するため、巨大なリレーションに対しては I/O 負荷がかかります。
  • セグメントファイル（relname.1 等）の命名規則に依存しているため、ストレージ管理側の命名規則が変更された場合は本関数のロジックも追従する必要があります。
