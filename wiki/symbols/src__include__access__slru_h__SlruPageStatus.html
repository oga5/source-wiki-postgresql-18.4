SlruPageStatus

定義

  • 種別: enum
  • 定義箇所: src/include/access/slru.h:47

目的

SLRU（Simple LRU）バッファ管理機構において、各ページバッファの現在の状態を識別するために使用されます。

シグネチャまたは形状

typedef enum {
    SLRU_PAGE_EMPTY,
    SLRU_PAGE_READ_IN_PROGRESS,
    SLRU_PAGE_VALID,
    SLRU_PAGE_WRITE_IN_PROGRESS,
} SlruPageStatus;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし

前提条件と不変条件

  • 各バッファは常にこれら4つの状態のいずれか一つを保持します。
  • SLRU_PAGE_VALID は、ページが有効であり、かつ書き込み中ではない状態を示します。

アルゴリズムまたは振る舞い

  • SLRU_PAGE_EMPTY: バッファが使用されていない状態。
  • SLRU_PAGE_READ_IN_PROGRESS: ディスクからメモリへページを読み込み中の状態。
  • SLRU_PAGE_VALID: ページがメモリ上に存在し、読み書き可能な状態。
  • SLRU_PAGE_WRITE_IN_PROGRESS: メモリ上のページをディスクへ書き出し中の状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/slru.c 内のバッファ管理ロジックが、共有メモリ上の page_status 配列を操作する際に参照します。
  • 呼び出し先: 情報なし。

関連データ構造

  • SlruSharedData: SLRU の共有状態を管理する構造体。この中で SlruPageStatus 型の配列 page_status が保持されます。

失敗モードと境界条件

  • 状態遷移の整合性は、SLRU のバンク単位またはバッファ単位のロックによって保護される必要があります。本シンボル自体に境界条件はありません。

保守担当者向けメモ

  • src/backend/access/transam/slru.c において、共有メモリ確保時に nslots * sizeof(SlruPageStatus) としてメモリサイズが計算されています。状態を追加・変更する場合は、この計算式への影響を確認してください。
