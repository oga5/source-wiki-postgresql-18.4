pqinitmask

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqsignal.c:40

目的

PostgreSQLバックエンドの動作状態に応じた3種類のシグナルマスク（UnBlockSig, BlockSig, StartupBlockSig）を初期化します。

シグネチャまたは形状

void pqinitmask(void)

入力と出力

  • 入力: なし
  • 出力: なし（グローバル変数 UnBlockSig, BlockSig, StartupBlockSig を副作用として更新）

前提条件と不変条件

  • UnBlockSig は本関数で空に初期化されますが、実行時に InitializeWaitEventSupport() によって変更される可能性があるため、読み取り専用として扱わないこと。
  • プラットフォームごとに利用可能なシグナルが異なるため、#ifdef による条件付きコンパイルで制御されています。

アルゴリズムまたは振る舞い

1. sigemptyset() を使用して UnBlockSig を空に初期化します。 2. sigfillset() を使用して BlockSig および StartupBlockSig を全シグナルセット状態にします。 3. ハードウェア例外や制御シグナル（SIGTRAP, SIGSEGV, SIGCONT 等）など、ブロックすべきでないシグナルを sigdelset() で両方のセットから除外します。 4. 起動時専用のセットである StartupBlockSig に対しては、さらに SIGQUIT, SIGTERM, SIGALRM を除外し、起動プロセスが停止要求を即座に受け取れるように調整します。

関連データ構造

  • sigset_t: シグナルセットを保持する型。Windows環境では pqsignal.h にてエミュレーションされます。
  • UnBlockSig, BlockSig, StartupBlockSig: 本関数で管理されるグローバルなシグナルセット変数。

失敗モードと境界条件

  • プラットフォームによって特定のシグナル（例: SIGSYS）が定義されていない場合、#ifdef により該当する sigdelset 処理はスキップされます。

保守担当者向けメモ

  • 本関数はバックエンド起動時に呼び出されます。
  • 新たなシグナルをマスクから除外する必要がある場合は、プラットフォーム依存性を考慮し、既存の #ifdef ブロックに従って追加してください。
