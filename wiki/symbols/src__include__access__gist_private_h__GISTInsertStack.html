GISTInsertStack

定義

  • 種別: struct
  • 定義箇所: src/include/access/gist_private.h:207

目的

GiSTインデックスへの挿入処理において、ツリーを降下する際のパスを記録し、ページ分割発生時の再試行制御を行うためのスタック構造体です。

シグネチャまたは形状

typedef struct GISTInsertStack
{
    BlockNumber blkno;
    Buffer      buffer;
    Page        page;
    GistNSN     lsn;
    bool        retry_from_parent;
    OffsetNumber downlinkoffnum;
    struct GISTInsertStack *parent;
} GISTInsertStack;

入力と出力

  • **入力**: 挿入対象のページ情報（ブロック番号、バッファ、ページポインタ）、親ノードへのポインタ、および親ページ内でのオフセット番号。
  • **出力**: 挿入パスを保持するスタックノード。ページ分割が必要な場合は retry_from_parent フラグが設定され、親ノードへの再帰的な再試行を促します。

前提条件と不変条件

  • parent ポインタは、ツリーのルート方向へ向かうパスを形成します。
  • lsn はページ更新の認識およびページ分割の検知（nsn との比較）に使用されます。
  • retry_from_parent が true の場合、現在のページへのダウンリンクが新しいキーをカバーしなくなったことを示唆します。

アルゴリズムまたは振る舞い

1. 挿入処理（gistdoinsert）の過程で、ツリーを降下するたびに本構造体が生成・連結されます。 2. ページ分割が発生した場合、このスタックを遡ることで、親ノードの更新や再試行に必要な情報を取得します。 3. lsn を用いて、ツリー降下中にページが更新されたか、あるいは分割されたかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gist.c 内の gistinserttuple、gistinserttuples、gistfinishsplit 等の挿入関連関数。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • GISTInsertState: 挿入処理全体のコンテキストを保持する構造体。
  • GISTInsertStack (再帰的): 親ノードへのポインタとして自身を参照します。

失敗モードと境界条件

  • **メモリ不足**: palloc0 によるスタックノードの割り当て失敗。
  • **再試行のループ**: ページ分割が連続して発生した場合、スタックを遡って再試行するロジックが正しく機能しないと無限ループの可能性があります。

保守担当者向けメモ

  • 本構造体は GiST インデックスの挿入アルゴリズムの核心部分です。ページ分割時の再試行ロジックを変更する場合は、lsn と nsn の比較条件を慎重に確認してください。
  • retry_from_parent フラグの取り扱いは、インデックスの整合性に直結するため注意が必要です。
