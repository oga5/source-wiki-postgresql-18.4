MemoizeHash_hash

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMemoize.c:157

目的

MemoizeState が保持するキャッシュハッシュテーブルにおいて、現在のプローブ用タプル（probeslot）からハッシュ値を計算する。

シグネチャまたは形状

uint32 MemoizeHash_hash(struct memoize_hash *tb, const MemoizeKey *key)

入力と出力

  • **入力**:
  • tb: ハッシュテーブル構造体へのポインタ。
  • key: ハッシュ計算対象のキー（本関数内では mstate->probeslot を参照）。
  • **出力**:
  • uint32: 計算されたハッシュ値。

前提条件と不変条件

  • mstate->probeslot が有効なタプルを保持していること。
  • mstate->nkeys が定義されており、キーの数と一致していること。
  • econtext->ecxt_per_tuple_memory が一時的なメモリ割り当てのために使用される。

アルゴリズムまたは振る舞い

1. econtext->ecxt_per_tuple_memory にメモリコンテキストを切り替える。 2. mstate->binary_mode の値に応じてハッシュ計算手法を分岐する。

  • **バイナリモード**: datum_image_hash を使用して、各属性のバイナリイメージから直接ハッシュを算出する。
  • **通常モード**: mstate->hashfunctions に格納された関数を FunctionCall1Coll で呼び出し、各属性のハッシュを算出する。

    • 3. 各属性のハッシュ値を pg_rotate_left32 で回転させながら XOR で結合する。 4. NULL値はハッシュ値 0 として扱う。 5. 最後に murmurhash32 を適用して最終的なハッシュ値を返す。 6. 元のメモリコンテキストに復帰する。

呼び出し元と呼び出し先

  • **呼び出し元**: lib/simplehash.h のマクロ SH_HASH_KEY を通じて、ハッシュテーブル操作時に呼び出される（直接の呼び出し元は nodeMemoize.c 内のハッシュテーブル実装）。
  • **呼び出し先**: pg_rotate_left32, datum_image_hash, FunctionCall1Coll, murmurhash32。

関連データ構造

  • MemoizeState: 実行状態とハッシュ関数情報を保持。
  • MemoizeKey: キャッシュキーの定義。
  • TupleTableSlot: ハッシュ計算の元となるデータを含む。

失敗モードと境界条件

  • pslot->tts_isnull が真の場合、その属性はハッシュ計算において 0 として扱われる。
  • メモリコンテキストの切り替えに失敗した場合、PostgreSQLの標準的なエラーハンドリング（elog/ereport）により中断される可能性がある。

保守担当者向けメモ

  • 本関数は simplehash.h のハッシュテーブル実装と密結合している。
  • binary_mode の切り替えは、キャッシュの効率と正確性に直結するため、属性の型定義変更時には注意が必要。
