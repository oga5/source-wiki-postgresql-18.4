IsPromoteSignaled

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/startup.c:287

目的

スタンバイからプライマリへの昇格要求（SIGUSR2）がシグナルハンドラによって受信されているかどうかを確認する。

シグネチャまたは形状

bool IsPromoteSignaled(void)

入力と出力

  • 入力: なし
  • 出力: promote_signaled 変数の現在の真偽値を返す。

前提条件と不変条件

  • 本関数は、startup.c 内で管理される静的変数 promote_signaled の状態を読み取る。
  • 昇格要求が処理された後、状態をリセットするには ResetPromoteSignaled() を呼び出す必要がある。

アルゴリズムまたは振る舞い

1. 内部変数 promote_signaled の値を参照する。 2. その値をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlogrecovery.c:4480 （リカバリ中の昇格チェックに使用）
  • 呼び出し先: なし

関連データ構造

  • promote_signaled: src/backend/postmaster/startup.c 内で定義された静的な bool 型変数。

失敗モードと境界条件

  • 本関数自体は単純な値の読み取りであるため、失敗モードは存在しない。
  • 呼び出し元が本関数の戻り値を確認した直後にシグナルが到着した場合の競合については、呼び出し側のロジック（CheckPromoteSignal() 等との併用）に依存する。

保守担当者向けメモ

  • 本関数は src/include/postmaster/startup.h で外部公開されている。
  • 昇格処理のフローを追う際は、StartupProcTriggerHandler（シグナル受信側）および ResetPromoteSignaled（状態クリア側）とセットで参照すること。
