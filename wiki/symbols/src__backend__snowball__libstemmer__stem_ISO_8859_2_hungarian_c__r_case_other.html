r_case_other

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_2_hungarian.c:624

目的

ハンガリー語のステミング処理において、特定の接尾辞パターン（a_6 配列で定義）を単語末尾から検索し、条件に応じて削除または置換を行う。

シグネチャまたは形状

static int r_case_other(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（ステミング環境を保持する構造体）
  • **出力**: 成功時に 1、失敗またはマッチしない場合に 0、エラー発生時に負の値を返す。

前提条件と不変条件

  • z->c（現在のカーソル位置）が、単語の末尾から少なくとも 4 文字以上離れている必要がある。
  • 単語の末尾（z->c - 1）が文字コード 108（'l'）であること。
  • r_R1 関数により、マッチした接尾辞が R1 領域内にあることが検証される。

アルゴリズムまたは振る舞い

1. z->ket を現在のカーソル位置に設定する。 2. 単語末尾が 'l' でない場合、または十分な長さがない場合は終了する。 3. find_among_b を使用し、a_6 配列に基づき末尾から接尾辞を検索する。 4. マッチした場合、z->bra を設定し、r_R1 を呼び出して対象領域を確認する。 5. マッチした接尾辞の種類（among_var）に応じて以下の処理を行う：

  • 1: slice_del で接尾辞を削除。
  • 2: slice_from_s で接尾辞を s_4 に置換。
  • 3: slice_from_s で接尾辞を s_5 に置換。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/libstemmer/stem_ISO_8859_2_hungarian.c および src/backend/snowball/libstemmer/stem_UTF_8_hungarian.c 内のメイン処理フローから呼び出される。
  • **呼び出し先**: find_among_b, r_R1, slice_del, slice_from_s（いずれも Snowball ランタイム関数）。

関連データ構造

  • SN_env: ステミング環境を管理する構造体。
  • a_6: 検索対象となる接尾辞パターンを保持する among 構造体の配列。

失敗モードと境界条件

  • 単語が短すぎる場合や、末尾が 'l' でない場合は即座に 0 を返す。
  • r_R1 が失敗した場合、その戻り値をそのまま返す。
  • slice_del や slice_from_s がメモリ不足等の理由で負の値を返した場合、そのエラーコードを伝播させる。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードである。ロジックの変更が必要な場合は、直接編集せず元の Snowball スクリプト（.sbl）を修正すること。
  • a_6 配列の定義や r_R1 の領域判定ロジックに変更を加える際は、本関数の動作に直接影響するため注意が必要である。
