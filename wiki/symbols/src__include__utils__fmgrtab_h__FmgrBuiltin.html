FmgrBuiltin

定義

  • 種別: struct
  • 定義箇所: src/include/utils/fmgrtab.h:25

目的

PostgreSQLの実行ファイルに静的にコンパイルされた「組み込み関数」のメタデータを保持し、関数マネージャ（fmgr）による高速な関数ルックアップと実行を可能にするための構造体です。

シグネチャまたは形状

typedef struct {
    Oid         foid;         /* 関数のOID */
    short       nargs;        /* 引数の数 (0..FUNC_MAX_ARGS)、可変長の場合は -1 */
    bool        strict;       /* 厳密性 (strict) かどうか */
    bool        retset;       /* 集合を返す関数かどうか */
    const char *funcName;     /* C言語での関数名 */
    PGFunction  func;         /* コンパイル済み関数へのポインタ */
} FmgrBuiltin;

入力と出力

  • 入力: 本構造体は、fmgr_builtins[] 配列の要素として静的に定義されます。
  • 出力: 関数マネージャがOIDをキーとして検索した際、該当する FmgrBuiltin 構造体が返され、そこから関数ポインタ（func）が抽出されます。

前提条件と不変条件

  • システム起動時にロードされる静的なメタデータテーブルの一部です。
  • foid は一意である必要があります。
  • func は PGFunction 型（fmgr.h で定義）に適合する関数ポインタである必要があります。

アルゴリズムまたは振る舞い

1. 関数呼び出し時、fmgrはOIDをキーとして fmgr_builtin_oid_index を参照し、fmgr_builtins 配列内の対応するインデックスを特定します。 2. 特定された FmgrBuiltin 構造体から func ポインタを取得します。 3. 取得したポインタを介して、システムカタログを検索することなく直接C関数を実行します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/fmgr/fmgr.c 内の関数ルックアップロジック（fmgr_lookupByName 等）から参照されます。
  • 呼び出し先: 本構造体自体はデータ構造であり、関数を呼び出す側です。func メンバが指す先が実際の実行関数となります。

関連データ構造

  • fmgr_builtins[]: 全組み込み関数を保持するグローバル配列。
  • fmgr_builtin_oid_index[]: OIDから配列インデックスを解決するためのルックアップテーブル。
  • Oid: access/transam.h で定義される識別子型。
  • PGFunction: fmgr.h で定義される関数ポインタ型。

失敗モードと境界条件

  • OIDが fmgr_builtin_oid_index に存在しない場合、マッピングは InvalidOidBuiltinMapping となります。
  • nargs が -1 の場合、引数の数は可変長として扱われます。

保守担当者向けメモ

  • 本構造体は src/include/catalog/pg_proc.dat からビルドプロセスを通じて自動生成されるテーブルの一部です。手動での直接編集は避けてください。
  • 新しい組み込み関数の追加が必要な場合は、カタログ定義ファイルを更新してください。
