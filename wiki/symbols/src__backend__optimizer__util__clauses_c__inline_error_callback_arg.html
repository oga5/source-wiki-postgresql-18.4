inline_error_callback_arg

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/util/clauses.c:84

目的

SQL関数やセットを返す関数（SRF）のインライン化処理中に発生したエラーを報告する際、エラーコンテキスト情報として関数名やソースコード情報を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	char	   *proname;
	char	   *prosrc;
} inline_error_callback_arg;

入力と出力

  • 入力: proname（関数名）、prosrc（関数ソース）の各文字列ポインタ。
  • 出力: なし（構造体として状態を保持）。

前提条件と不変条件

  • エラー発生時に error_context_stack に登録されるコールバック関数へ渡される引数として利用されます。
  • 保持される文字列ポインタは、エラー発生時に有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

  • インライン化処理の開始時に本構造体を初期化し、エラー発生時のコンテキストとして設定します。
  • エラー発生時、コールバック関数内で arg から本構造体にキャストされ、proname や prosrc を参照してエラーメッセージを構築します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/util/clauses.c 内のインライン化関連関数（inline_function 等）にて、エラーコールバックの引数としてインスタンス化されます。
  • 呼び出し先: エラー発生時に呼び出されるエラー報告用コールバック関数（callback_arg として参照）。

関連データ構造

  • src/backend/optimizer/util/clauses.c 内のインライン化処理に関連する関数群。

失敗モードと境界条件

  • 構造体内のポインタが指す文字列が解放済みである場合、エラー報告時に不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体はエラー報告専用の軽量なコンテナです。インライン化処理のロジック変更に伴い、エラーメッセージの具体性を高めるためにフィールドを追加する場合は、対応するコールバック関数側の参照ロジックも更新してください。
