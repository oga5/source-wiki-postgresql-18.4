CloseArchive

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_archiver.c:260

目的

アーカイブ処理を終了し、関連するリソースおよび出力ファイルを適切にクローズする。

シグネチャまたは形状

void CloseArchive(Archive *AHX)

入力と出力

  • **入力**: Archive *AHX （アーカイブハンドルへのポインタ）
  • **出力**: なし（副作用としてリソースの解放とファイルクローズを行う）

前提条件と不変条件

  • AHX は有効な Archive 構造体である必要がある。
  • 内部的に ArchiveHandle へキャストして処理される。

アルゴリズムまたは振る舞い

1. AHX を ArchiveHandle *AH にキャストする。 2. AH->ClosePtr コールバック関数を呼び出し、アーカイブ形式固有の終了処理を実行する。 3. errno を 0 にリセットする。 4. EndCompressFileHandle(AH->OF) を呼び出し、出力ファイルハンドルをクローズする。 5. ファイルクローズに失敗した場合は pg_fatal を呼び出し、エラーメッセージを出力して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c
  • src/bin/pg_dump/pg_restore.c
  • **呼び出し先**:
  • AH->ClosePtr (アーカイブ形式固有の関数)
  • EndCompressFileHandle

関連データ構造

  • Archive / ArchiveHandle: アーカイブの状態を保持する構造体。
  • CompressFileHandle: 出力ファイルの管理に使用される構造体。

失敗モードと境界条件

  • EndCompressFileHandle が失敗した場合、pg_fatal によりプロセスが異常終了する。

保守担当者向けメモ

  • 本関数は pg_dump および pg_restore のライフサイクルの最終段階で呼び出される。
  • AH->ClosePtr はアーカイブ形式（カスタム、tar、ディレクトリ等）に応じて動的に割り当てられるため、デバッグ時は対象の形式の実装を確認すること。
  • pg_restore.c のコメントにある通り、本関数の呼び出し後に AH が解放される可能性がある点に注意が必要である。
