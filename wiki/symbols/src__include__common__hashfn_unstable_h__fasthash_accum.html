fasthash_accum

定義

  • 種別: function
  • 定義箇所: src/include/common/hashfn_unstable.h:135

目的

ハッシュ計算の進行状況を保持する fasthash_state に対して、入力データ（バイト列）を蓄積・混合し、ハッシュ計算を段階的に進めるために使用されます。

シグネチャまたは形状

static inline void fasthash_accum(fasthash_state *hs, const char *k, size_t len)

入力と出力

  • hs: ハッシュ状態を保持する fasthash_state 構造体へのポインタ。
  • k: ハッシュ対象となる入力データの先頭ポインタ。
  • len: 入力データの長さ（バイト単位）。
  • 出力: なし（hs の内部状態が更新されます）。

前提条件と不変条件

  • len <= FH_SIZEOF_ACCUM であること（Assert により検証）。
  • プラットフォームのエンディアン（WORDS_BIGENDIAN の有無）に応じて、バイト単位のロードが正しく行われること。

アルゴリズムまたは振る舞い

1. hs->accum を 0 に初期化します。 2. 入力長 len に応じて、k から hs->accum へデータをロードします。

  • エンディアンに応じて、バイト単位のシフト演算または memcpy を使用して、hs->accum にデータを配置します。
  • len が 0 の場合は即座にリターンします。

    • 3. 最後に fasthash_combine(hs) を呼び出し、蓄積されたデータをハッシュ状態に混合します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/include/common/hashfn_unstable.h 内の各箇所（fasthash_accum_cstring 等の内部処理や、特定のデータ長に対するハッシュ計算）。
  • 呼び出し先:
  • fasthash_combine（状態の混合処理）。

関連データ構造

  • fasthash_state: ハッシュ計算の進行状況を保持する構造体。

失敗モードと境界条件

  • len が FH_SIZEOF_ACCUM を超える場合、Assert が有効なビルドではアボートします。
  • len が 0 の場合、何も処理を行わず正常終了します。

保守担当者向けメモ

  • 本関数は「不安定」なハッシュ実装の一部であり、プラットフォーム間で結果が一致しません。永続化が必要なデータには使用しないでください。
  • エンディアン依存の処理が含まれているため、修正時は WORDS_BIGENDIAN の両方のケースを考慮してください。
