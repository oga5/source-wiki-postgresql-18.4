set_plan_references

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/setrefs.c:287

目的

プランナが生成したプランツリーに対し、実行エンジン（Executor）が利用可能な物理的構造への変換および最終的な後処理を行う。

シグネチャまたは形状

Plan *set_plan_references(PlannerInfo *root, Plan *plan)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報およびクエリ情報を保持する PlannerInfo 構造体。
  • plan: 処理対象のプランツリーのルートノード。
  • **出力**:
  • 参照解決および調整が完了した Plan ツリーのルートノードを返す。

前提条件と不変条件

  • root->glob->finalrtable が初期化されていること。
  • プランツリー内の各ノードは、createplan.c 等によって論理的に構築済みであること。
  • rtoffset は、現在のレンジテーブルの長さに基づいて算出される。

アルゴリズムまたは振る舞い

1. **レンジテーブルの統合**: add_rtes_to_flat_rtable を呼び出し、クエリのレンジテーブルをフラット化する。 2. **メタデータの調整**:

  • root->rowMarks を走査し、レンジテーブルインデックスを rtoffset 分だけオフセットして glob->finalrowmarks に格納する。
  • root->append_rel_list を走査し、AppendRelInfo のリレーションIDをオフセットし、不要な translated_vars を削除する。

    • 3. **サブプランの準備**: AlternativeSubPlan が存在する場合、処理用のワークスペースを確保する。 4. **プランツリーの走査**: set_plan_refs を呼び出し、プランツリー全体を再帰的に走査して変数参照を解決する。 5. **サブプランの最適化**: AlternativeSubPlan の解決後、使用されなかったサブプランを glob->subplans リストから削除（NULL化）する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planner.c: プランナのメイン処理フローから呼び出される。
  • src/backend/optimizer/plan/setrefs.c: SubqueryScan ノードの処理などで再帰的に呼び出される。
  • **呼び出し先**:
  • add_rtes_to_flat_rtable
  • set_plan_refs (内部的な再帰関数)

関連データ構造

  • PlannerInfo: プラン作成中の状態を保持。
  • PlannerGlobal: クエリ全体で共有されるプラン情報（レンジテーブル、サブプラン等）。
  • PlanRowMark, AppendRelInfo: 実行時に必要なメタデータ。

失敗モードと境界条件

  • AlternativeSubPlan が含まれる場合、複数のコピーが異なる解決結果を持つ可能性があるため、走査完了後にまとめて不要なサブプランを削除する。
  • AppendRelInfo の translated_vars は実行時に不要なため、この段階で破棄される。

保守担当者向けメモ

  • 本関数はプランツリーのインプレース修正を行う。
  • varno が OUTER_VAR 等に書き換わった後は、元のリレーションとの対応関係が失われるため、デバッグ時には注意が必要である。
  • SubqueryScan の削除などの最適化は、参照解決が完了した後に実行される。
