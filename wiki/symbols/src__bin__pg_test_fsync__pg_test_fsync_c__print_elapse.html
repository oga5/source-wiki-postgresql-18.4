print_elapse

定義

  • 種別: function
  • 定義箇所: src/bin/pg_test_fsync/pg_test_fsync.c:627

目的

ベンチマークテストの計測結果（経過時間と操作回数）に基づき、1秒あたりの操作回数（ops/sec）および1操作あたりの平均時間（マイクロ秒）を計算し、標準出力へ整形表示する。

シグネチャまたは形状

static void print_elapse(struct timeval start_t, struct timeval stop_t, int ops)

入力と出力

  • **入力**:
  • start_t: 計測開始時刻 (struct timeval)
  • stop_t: 計測終了時刻 (struct timeval)
  • ops: 実行された操作の総回数 (int)
  • **出力**:
  • なし（戻り値は void）。標準出力へ OPS_FORMAT に基づくフォーマット済み文字列を出力する。

前提条件と不変条件

  • ops は 0 より大きい値である必要がある（0 の場合、除算により avg_op_time_us が無限大または不正な値となる）。
  • stop_t は start_t 以降の時刻であること。

アルゴリズムまたは振る舞い

1. stop_t と start_t の差分から、秒単位の経過時間 total_time を算出する。 2. ops を total_time で除算し、1秒あたりの操作回数 per_second を算出する。 3. total_time を ops で除算し、さらに USECS_SEC を乗じて1操作あたりの平均時間 avg_op_time_us を算出する。 4. printf を使用し、OPS_FORMAT 定数に従って結果を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_test_fsync/pg_test_fsync.c 内の各テスト関数（test_sync 等）から呼び出される。
  • **呼び出し先**: 標準ライブラリの printf を呼び出す。

関連データ構造

  • struct timeval: POSIX で定義される時刻構造体（sys/time.h）。

失敗モードと境界条件

  • ops が 0 の場合、total_time / ops でゼロ除算が発生する可能性がある。
  • total_time が 0 に極めて近い場合、per_second の計算で精度上の問題やオーバーフローが発生する可能性がある。

保守担当者向けメモ

  • 本関数は pg_test_fsync における共通の出力ロジックである。出力フォーマットを変更する場合は OPS_FORMAT マクロの定義を確認すること。
