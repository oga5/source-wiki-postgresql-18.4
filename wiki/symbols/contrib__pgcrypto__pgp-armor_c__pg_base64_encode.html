pg_base64_encode

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-armor.c:44

目的

バイナリデータを Base64 エンコードされた ASCII 文字列に変換し、76 文字ごとに改行を挿入して出力します。

シグネチャまたは形状

int pg_base64_encode(const uint8 *src, unsigned len, uint8 *dst)

入力と出力

  • **入力**:
  • src: エンコード対象のバイナリデータへのポインタ。
  • len: 入力データのバイト長。
  • dst: エンコード結果を格納するバッファへのポインタ。
  • **出力**:
  • 戻り値: エンコード後のデータ長（バイト単位）。

前提条件と不変条件

  • dst バッファは、エンコード後のデータ（改行を含む）を格納するために十分なサイズが確保されている必要があります。
  • 内部で使用される _base64 テーブルは、Base64 変換用の標準的な文字セットを保持している必要があります。

アルゴリズムまたは振る舞い

1. 入力データを 3 バイトずつ読み込み、4 つの 6 ビット値に分割して Base64 文字に変換します。 2. 76 文字出力するごとに改行コード (\n) を挿入します。 3. 入力データが 3 バイトの倍数でない場合、パディング文字 = を使用して 4 文字単位に調整します。 4. 最終的な出力バイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-armor.c 内のアーマー処理ロジックから呼び出されます。また、src/backend/utils/adt/encode.c にも同名の関数が存在しますが、本関数は pgp-armor.c 内で定義された独自実装です。
  • **呼び出し先**: 外部関数への明示的な呼び出しはありません。

関連データ構造

  • _base64: Base64 エンコードに使用される静的な文字配列（ソースコード内で定義）。

失敗モードと境界条件

  • **バッファオーバーフロー**: dst バッファが不足している場合、メモリ破壊が発生する可能性があります。呼び出し側で十分な領域を確保する必要があります。
  • **パディング**: 入力長が 3 の倍数でない場合、適切に = でパディングされます。

保守担当者向けメモ

  • 本関数は contrib/pgcrypto 内の独自実装であり、PostgreSQL の他のモジュール（src/backend/utils/adt/encode.c など）にある Base64 実装と重複しています。修正時はこの重複に留意してください。
