PQpingParams

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-connect.c:782

目的

指定された接続パラメータを使用してデータベースサーバーへの接続を試行し、サーバーが応答可能かどうか（ping）を確認する。

シグネチャまたは形状

PGPing PQpingParams(const char *const *keywords, const char *const *values, int expand_dbname);

入力と出力

  • **入力**:
  • keywords: 接続パラメータ名の配列（NULL 終端）。
  • values: 接続パラメータ値の配列（NULL 終端）。
  • expand_dbname: データベース名を展開するかどうかを指定するフラグ。
  • **出力**:
  • PGPing 型の列挙値（サーバーの状態を示す）。

前提条件と不変条件

  • keywords および values は、PQconnectStartParams が受け入れ可能な形式である必要がある。
  • 内部で PQconnectStartParams を呼び出すため、呼び出し側は接続パラメータのメモリ管理を適切に行う必要がある。

アルゴリズムまたは振る舞い

1. PQconnectStartParams を呼び出し、接続試行を開始するための PGconn オブジェクトを生成する。 2. internal_ping を呼び出し、接続状態の確認を行う。 3. PQfinish を呼び出し、生成した PGconn オブジェクトを解放する。 4. internal_ping の結果を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/scripts/pg_isready.c:192
  • src/test/regress/pg_regress.c:2460
  • src/test/regress/pg_regress.c:2523
  • **呼び出し先**:
  • PQconnectStartParams
  • internal_ping
  • PQfinish

関連データ構造

  • PGconn: 接続状態を保持する構造体。
  • PGPing: 接続結果を表す列挙型。

失敗モードと境界条件

  • PQconnectStartParams が NULL を返した場合、internal_ping の挙動に依存する（internal_ping が NULL ポインタを適切に処理する必要がある）。
  • ネットワークエラーや認証失敗が発生した場合、internal_ping はそれに応じた PGPing の値を返す。

保守担当者向けメモ

  • 本関数は接続の確立から終了までを単一の処理として完結させるラッパーである。
  • 接続の成否だけでなく、サーバーが受け入れ可能な状態にあるかを判定する目的で使用される。
