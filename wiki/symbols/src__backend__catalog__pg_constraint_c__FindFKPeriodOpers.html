FindFKPeriodOpers

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_constraint.c:1665

目的

外部キー制約の期間（PERIOD）処理において、RI（参照整合性）チェックを最適化するために必要な演算子（ContainedBy 演算子および交差演算子）の OID を取得する。

シグネチャまたは形状

void FindFKPeriodOpers(Oid opclass,
                       Oid *containedbyoperoid,
                       Oid *aggedcontainedbyoperoid,
                       Oid *intersectoperoid)

入力と出力

  • **入力**:
  • opclass: 検索対象の演算子クラスの OID。
  • **出力**:
  • containedbyoperoid: 取得された ContainedBy 演算子の OID が格納される。
  • aggedcontainedbyoperoid: 取得された集約用 ContainedBy 演算子の OID が格納される。
  • intersectoperoid: 取得された交差演算子の OID が格納される。

前提条件と不変条件

  • opclass は range または multirange 型に関連付けられた演算子クラスである必要がある。それ以外の場合はエラーとなる。
  • 内部で呼び出される GetOperatorFromCompareType が成功し、適切な演算子が見つかることが前提となる。

アルゴリズムまたは振る舞い

1. get_opclass_opfamily_and_input_type を使用して、指定された opclass が range または multirange 型であることを検証する。 2. GetOperatorFromCompareType を呼び出し、以下の2種類の ContainedBy 演算子を検索・取得する。

  • 左右の引数が opclass の型と一致するもの。
  • 左引数が opclass の型、右引数が ANYMULTIRANGEOID と一致するもの。

    • 3. opcintype（入力型）に応じて、適切な交差演算子（OID_RANGE_INTERSECT_RANGE_OP または OID_MULTIRANGE_INTERSECT_MULTIRANGE_OP）を intersectoperoid に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_constraint.c
  • src/backend/commands/tablecmds.c
  • src/backend/utils/adt/ri_triggers.c
  • **呼び出し先**:
  • get_opclass_opfamily_and_input_type
  • GetOperatorFromCompareType
  • ereport / elog (エラーハンドリング)

関連データ構造

  • Oid: PostgreSQL のオブジェクト識別子。
  • ANYRANGEOID, ANYMULTIRANGEOID: 範囲型および多重範囲型の型 OID。

失敗モードと境界条件

  • opclass が range または multirange 型でない場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラーとなる。
  • opclass のキャッシュルックアップに失敗した場合、elog(ERROR) が発生する。
  • 期待される演算子が見つからない場合や、予期しない opcintype が渡された場合はエラーとなる。

保守担当者向けメモ

  • 本関数は RI トリガーの最適化ロジックに深く依存している。演算子の検索ロジックを変更する場合は、ri_triggers.c 側の動作に影響がないか確認すること。
