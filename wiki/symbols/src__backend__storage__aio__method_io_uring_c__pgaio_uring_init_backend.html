pgaio_uring_init_backend

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_io_uring.c:396

目的

現在のバックエンドプロセスに対して、io_uring 関連のコンテキストを初期化し、プロセス固有のコンテキストポインタを割り当てる。

シグネチャまたは形状

static void pgaio_uring_init_backend(void)

入力と出力

  • 入力: なし（グローバルな MyProcNumber を使用）
  • 出力: なし（副作用として pgaio_my_uring_context を設定）

前提条件と不変条件

  • MyProcNumber が pgaio_uring_procs() が返す有効な範囲内にあること。

アルゴリズムまたは振る舞い

1. MyProcNumber が有効な範囲内にあるか Assert で検証する。 2. pgaio_uring_contexts 配列から、現在のプロセス番号に対応するインデックスの要素を pgaio_my_uring_context に代入する。

呼び出し元と呼び出し先

  • 呼び出し元: pgaio_uring_ops 構造体の .init_backend フィールドを通じて、AIO サブシステムから呼び出される。
  • 呼び出し先: なし。

関連データ構造

  • pgaio_uring_contexts: バックエンドごとの PgAioUringContext を保持する共有メモリ上の配列。
  • pgaio_my_uring_context: 現在のバックエンドが使用する PgAioUringContext へのポインタ。

失敗モードと境界条件

  • MyProcNumber が pgaio_uring_procs() の上限を超えている場合、アサーション失敗により異常終了する。

保守担当者向けメモ

  • 本関数はバックエンドの初期化フェーズで実行される。
  • pgaio_uring_shmem_init によって共有メモリが確保された後に呼び出されることが前提となっている。
