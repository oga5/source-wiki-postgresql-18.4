printtup_shutdown

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/printtup.c:388

目的

DestReceiver インターフェースを実装する DR_printtup オブジェクトの終了処理を行い、関連するメモリリソースを解放する。

シグネチャまたは形状

static void printtup_shutdown(DestReceiver *self)

入力と出力

  • **入力**: self (DestReceiver *) - 終了対象のレシーバーインスタンス。
  • **出力**: なし。

前提条件と不変条件

  • self は DR_printtup 構造体としてキャスト可能であること。
  • 実行後、myState->myinfo、myState->attrinfo、myState->buf.data、myState->tmpcontext はすべて NULL に設定される。

アルゴリズムまたは振る舞い

1. self を DR_printtup * 型の myState にキャストする。 2. myState->myinfo が存在する場合、pfree でメモリを解放し、ポインタを NULL にする。 3. myState->attrinfo ポインタを NULL にクリアする。 4. myState->buf.data が存在する場合、pfree でメモリを解放し、ポインタを NULL にする。 5. myState->tmpcontext が存在する場合、MemoryContextDelete でメモリコンテキスト全体を破棄し、ポインタを NULL にする。

呼び出し元と呼び出し先

  • **呼び出し元**: printtup_create_DR 関数内で DestReceiver の rShutdown フィールドに本関数が登録されている。
  • **呼び出し先**: pfree および MemoryContextDelete を呼び出す。

関連データ構造

  • DR_printtup: 本関数が操作するプライベート状態構造体。
  • DestReceiver: 本関数が実装を提供するインターフェース構造体。

失敗モードと境界条件

  • myState の各メンバが既に NULL である場合、各ポインタのチェックにより安全にスキップされる。

保守担当者向けメモ

  • 本関数は DestReceiver のライフサイクル終了時に呼び出される。メモリリークを防ぐため、DR_printtup 構造体に追加の動的メモリ割り当てを行った場合は、必ず本関数内で解放処理を追加すること。
