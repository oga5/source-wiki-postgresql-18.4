pgstat_report_autovac

定義

  • src/backend/utils/activity/pgstat_database.c:54

目的

指定されたデータベースにおけるオートバキュームの開始時刻を統計情報として即時記録する。

シグネチャまたは形状

void pgstat_report_autovac(Oid dboid)

入力と出力

  • **入力**: dboid (対象データベースのOID)
  • **出力**: なし（共有メモリ上の統計情報を直接更新）

前提条件と不変条件

  • IsUnderPostmaster が真であること（シングルユーザーモードでは実行不可）。
  • 統計情報の更新は即時反映される。

アルゴリズムまたは振る舞い

1. pgstat_get_entry_ref_locked を呼び出し、指定されたデータベースの統計エントリをロックして取得する。 2. 取得したエントリの last_autovac_time フィールドに GetCurrentTimestamp() の値を書き込む。 3. pgstat_unlock_entry を呼び出し、ロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/autovacuum.c:1566
  • **呼び出し先**:
  • pgstat_get_entry_ref_locked
  • GetCurrentTimestamp
  • pgstat_unlock_entry

関連データ構造

  • PgStat_EntryRef: 統計エントリへの参照およびロック管理用構造体。
  • PgStatShared_Database: 共有メモリ上に保持されるデータベース統計構造体。

失敗モードと境界条件

  • IsUnderPostmaster が偽の場合、Assert により異常終了する。
  • 統計エントリの取得に失敗した場合の挙動は、pgstat_get_entry_ref_locked の実装に依存する。

保守担当者向けメモ

  • 本関数はオートバキュームの開始を記録する。終了時の報告と一貫性を保つため、即時反映を行う設計となっている。
  • 処理中にロックを取得するため、長時間ブロックされる可能性があるが、オートバキュームの性質上許容される。
