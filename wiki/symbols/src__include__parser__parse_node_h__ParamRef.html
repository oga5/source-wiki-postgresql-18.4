ParamRef

定義

  • 種別: typedef
  • 定義箇所: src/include/parser/parse_node.h:93
  • 構造体定義: nodes/parsenodes.h:317

目的

SQL 文中のパラメータ参照（例: $1, $2）を表現するための構文木ノードです。パーサーが解析過程でパラメータを識別し、後続の変換処理で適切な値や型情報と結びつけるために使用されます。

シグネチャまたは形状

ParamRef は Node 構造体の一種であり、ソーステキスト内の位置情報（location）を保持します。

入力と出力

  • 入力: SQL 文中のパラメータ構文（$n）。
  • 出力: 解析処理（transformParamRef 等）を経て、実行時に解決可能な Param ノードやその他の式ノードに変換されます。

前提条件と不変条件

  • ParamRef は解析の初期段階で生成される構文木ノードです。
  • 実行時には、ParseParamRefHook を介して解決される必要があります。

アルゴリズムまたは振る舞い

1. パーサーが SQL をスキャンし、$n 形式のトークンを検出すると ParamRef ノードを作成します。 2. 解析フェーズにおいて、transformParamRef 関数が呼び出され、ParseState を通じてパラメータの型や値の解決を試みます。 3. ParseParamRefHook が定義されている場合、そのフック関数（sql_fn_param_ref や paramlist_param_ref など）が呼び出され、パラメータの解決ロジックが実行されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/parser/parse_expr.c: transformParamRef 関数内で変換処理の対象となります。
  • src/backend/nodes/nodeFuncs.c: ノードの走査時に location フィールドが参照されます。
  • **呼び出し先（フック実装）**:
  • src/backend/executor/functions.c: sql_fn_param_ref
  • src/backend/nodes/params.c: paramlist_param_ref
  • src/backend/parser/parse_param.c: fixed_paramref_hook, variable_paramref_hook

関連データ構造

  • ParseState: 解析状態を保持し、パラメータ解決のコンテキストを提供します。
  • ParseParamRefHook: ParamRef を解決するための関数ポインタ型。
  • Node: 基底となるノード構造体。

失敗モードと境界条件

  • パラメータ番号が範囲外である場合や、解決に必要なコンテキスト（ParamListInfo 等）が提供されていない場合、解析エラーが発生します。
  • ParseParamRefHook が設定されていない状態でパラメータが参照された場合、解決不能としてエラーとなる可能性があります。

保守担当者向けメモ

  • ParamRef は構文木上のノードであり、実行可能なクエリ木（Param ノード）とは区別してください。
  • パラメータ解決のロジックを変更する場合は、src/backend/parser/parse_param.c のフック関数を確認してください。
