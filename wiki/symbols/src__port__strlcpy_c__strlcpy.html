strlcpy

定義

  • 種別: function
  • 定義箇所: src/port/strlcpy.c:44

目的

strncpy に代わる安全な文字列コピー関数。バッファサイズを超えないコピーと、常にヌル終端を保証することで、バッファオーバーフローを防止する。

シグネチャまたは形状

size_t strlcpy(char *dst, const char *src, size_t siz)

入力と出力

  • **入力**:
  • dst: コピー先のバッファへのポインタ。
  • src: コピー元のヌル終端文字列へのポインタ。
  • siz: dst バッファのサイズ（バイト単位）。
  • **出力**:
  • 戻り値: コピーしようとした元の文字列の全長（ヌル文字を含まない）。戻り値が siz 以上の場合、切り捨てが発生したことを示す。

前提条件と不変条件

  • siz が 0 の場合、コピーは行われず、src の長さのみが返される。
  • dst が指すバッファは、少なくとも siz バイトの書き込み可能領域を持つ必要がある。
  • コピー完了後、dst は必ずヌル終端される（siz > 0 の場合）。

アルゴリズムまたは振る舞い

1. siz が 0 でない場合、最大 siz-1 バイトまで src から dst へコピーを行う。 2. コピー中に src の終端に達した場合はループを終了する。 3. siz バイト分コピーしても終端に達しない場合、dst の末尾（dst[siz-1]）にヌル文字を書き込む。 4. src の残りの長さを走査し、src の全文字数（ヌル文字を含まない）を計算して返す。

関連データ構造

  • 特になし（標準の size_t 型を使用）。

失敗モードと境界条件

  • siz が 0 の場合: コピーは行われず、src の長さのみが返される。
  • src が dst より長い場合: dst は siz-1 バイトまでコピーされ、末尾にヌル文字が付与される。戻り値は siz 以上となり、切り捨てが発生したことが通知される。

保守担当者向けメモ

  • 本関数は OpenBSD の実装に基づいている。
  • 戻り値が siz 以上であるかどうかを確認することで、呼び出し元はデータ損失を検知できる。
