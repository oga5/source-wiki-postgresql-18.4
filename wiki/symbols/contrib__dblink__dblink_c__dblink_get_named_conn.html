dblink_get_named_conn

定義

  • 種別: function
  • 定義箇所: contrib/dblink/dblink.c:255

目的

指定された名前を持つリモートデータベース接続（PGconn）を取得します。

シグネチャまたは形状

static PGconn *dblink_get_named_conn(const char *conname)

入力と出力

  • **入力**: conname (接続名を示す文字列)
  • **出力**: 成功した場合は対応する PGconn 構造体へのポインタ。失敗した場合は NULL を返しますが、内部でエラーを報告します。

前提条件と不変条件

  • getConnectionByName を介してハッシュテーブルから接続情報を検索します。
  • 接続が見つからない場合、dblink_conn_not_avail を呼び出してエラーを発生させます。

アルゴリズムまたは振る舞い

1. getConnectionByName(conname) を呼び出し、名前付き接続のハッシュテーブルから remoteConn 構造体を取得します。 2. 接続が存在する場合、その remoteConn 構造体内の conn フィールド（PGconn）を返します。 3. 接続が存在しない場合、dblink_conn_not_avail を呼び出してエラー処理を行い、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: getConnectionByName, dblink_conn_not_avail
  • **呼び出し元**:
  • contrib/dblink/dblink.c 内の複数の関数（PG_GETARG_TEXT_PP を使用して引数から接続名を取得する箇所など、計8箇所で確認）。

関連データ構造

  • remoteConn: 接続状態とカーソル管理情報を保持する構造体。

失敗モードと境界条件

  • 指定された名前の接続がハッシュテーブルに存在しない場合、dblink_conn_not_avail が呼び出され、エラーが報告されます。

保守担当者向けメモ

  • 本関数は静的関数であり、dblink モジュール内での接続取得の標準的な手段として利用されています。
  • 接続名が NULL または無効な場合の挙動は、呼び出し元の getConnectionByName の実装に依存します。
