getSubscriptingRoutines

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/lsyscache.c:3307

目的

指定されたデータ型の添字操作（subscripting）をサポートする関数ポインタ構造体を取得します。

シグネチャまたは形状

const struct SubscriptRoutines *getSubscriptingRoutines(Oid typid, Oid *typelemp)

入力と出力

  • **入力**:
  • typid: 調査対象のデータ型のOID。
  • typelemp: 添字付けされた要素の型OIDを格納するためのポインタ（出力用）。
  • **出力**:
  • 成功時: SubscriptRoutines 構造体へのポインタ。
  • 失敗時: NULL（型が添字操作をサポートしていない場合）。

前提条件と不変条件

  • typid は有効な型OIDである必要があります。
  • 戻り値の構造体は、型定義で指定された添字操作関数を呼び出すことで取得されます。

アルゴリズムまたは振る舞い

1. get_typsubscript(typid, typelemp) を呼び出し、当該型の添字操作関数（typsubscript）のOIDを取得します。 2. 取得したOIDが有効でない場合、NULL を返します。 3. 有効な場合、その関数を引数なしで実行（OidFunctionCall0）し、戻り値を SubscriptRoutines 構造体へのポインタとしてキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExpr.c
  • src/backend/optimizer/util/clauses.c
  • src/backend/parser/parse_node.c
  • **呼び出し先**:
  • get_typsubscript（内部関数）
  • OidFunctionCall0（関数実行エンジン）

関連データ構造

  • SubscriptRoutines: 添字操作に必要な関数群を保持する構造体。

失敗モードと境界条件

  • 指定された型が添字操作を定義していない場合、get_typsubscript が無効なOIDを返し、本関数は NULL を返します。

保守担当者向けメモ

  • 本関数は get_typsubscript の結果をキャッシュするものではなく、毎回関数呼び出しを介してルーチンを取得します。パフォーマンスが重要なパスで呼び出される場合は注意してください。
