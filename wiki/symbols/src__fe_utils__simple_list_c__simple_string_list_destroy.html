simple_string_list_destroy

定義

  • 種別: function
  • 定義箇所: src/fe_utils/simple_list.c:124

目的

SimpleStringList 構造体によって管理されている単方向連結リストの全セルをメモリから解放する。

シグネチャまたは形状

void simple_string_list_destroy(SimpleStringList *list)

入力と出力

  • **入力**: SimpleStringList *list（破棄対象のリストへのポインタ）
  • **出力**: なし（void）

前提条件と不変条件

  • list は有効な SimpleStringList 構造体を指している必要がある。
  • リスト内の各セルは pg_malloc 等で確保されたメモリ領域であること。

アルゴリズムまたは振る舞い

1. リストの先頭要素 (list->head) を取得する。 2. リストが空になるまで以下の処理を繰り返す。

  • 現在のセルの next ポインタを一時的に保持する。
  • 現在のセルを pg_free で解放する。
  • 保持していた next ポインタを現在のセルとして更新し、走査を継続する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/scripts/reindexdb.c:477
  • src/bin/scripts/reindexdb.c:796
  • **呼び出し先**:
  • pg_free（メモリ解放用）

関連データ構造

  • SimpleStringList: リストの先頭および末尾を保持する構造体。
  • SimpleStringListCell: リストの各要素を構成する構造体。

失敗モードと境界条件

  • list が NULL の場合、あるいはリストが空（head が NULL）の場合、ループは実行されず何も行われない。
  • 本関数はリストのセルを解放するのみであり、リスト構造体自体（list ポインタが指すメモリ）の解放は呼び出し元で行う必要がある。

保守担当者向けメモ

  • 文字列リストは FLEXIBLE_ARRAY_MEMBER を利用して文字列をセル内にコピーして保持しているため、本関数による pg_free で文字列領域も併せて解放される。
  • ポインタリストの破棄を行う simple_ptr_list_destroy とは異なり、文字列リスト専用のメモリ管理を行う点に注意すること。
