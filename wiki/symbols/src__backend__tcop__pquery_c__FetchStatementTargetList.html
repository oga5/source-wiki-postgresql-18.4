FetchStatementTargetList

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/pquery.c:348

目的

与えられたステートメント（Node）から、その実行結果として返されるターゲットリスト（List *）を抽出する。

シグネチャまたは形状

List *FetchStatementTargetList(Node *stmt)

入力と出力

  • **入力**: stmt (解析済みクエリまたは計画済みステートメントを表す Node ポインタ)
  • **出力**: 抽出されたターゲットリスト（List *）。該当するものがない場合や不明な型の場合は NIL を返す。

前提条件と不変条件

  • stmt が NULL の場合は NIL を返す。
  • FetchStmt の場合、ismove が false であることを前提とする（Assert による検証あり）。
  • Portal や PreparedStatement を参照する場合、それらが有効であることを前提とする。

アルゴリズムまたは振る舞い

stmt の型に応じて以下の処理を行う： 1. **Query**: CMD_SELECT なら targetList を、returningList があればそれを返す。CMD_UTILITY の場合は内部の utilityStmt を参照する。 2. **PlannedStmt**: CMD_SELECT なら planTree->targetlist を、hasReturning が真なら planTree->targetlist を返す。CMD_UTILITY の場合は内部の utilityStmt を参照する。 3. **FetchStmt**: 指定されたポータル名から FetchPortalTargetList を呼び出し、結果を取得する。 4. **ExecuteStmt**: 指定された準備済みステートメントを検索し、FetchPreparedStatementTargetList を呼び出して結果を取得する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/pquery.c (内部参照)
  • src/backend/utils/cache/plancache.c:1777
  • **呼び出し先**:
  • FetchPortalTargetList
  • FetchPreparedStatementTargetList
  • GetPortalByName
  • FetchPreparedStatement

関連データ構造

  • Query, PlannedStmt, FetchStmt, ExecuteStmt
  • Portal, PreparedStatement

失敗モードと境界条件

  • FetchStmt で指定されたポータル名が存在しない場合、GetPortalByName の挙動に依存する。
  • ExecuteStmt で指定された準備済みステートメントが存在しない場合、FetchPreparedStatement のエラーハンドリング（missing_ok=true 指定時は NULL 戻り）に従う。
  • 認識できないノード型が渡された場合は NIL を返す。

保守担当者向けメモ

  • 本関数はクエリの実行計画やポータルの状態から動的にターゲットリストを特定する。新しいステートメント型が追加された場合は、本関数の分岐条件を更新する必要がある。
