OperatorValidateParams

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_operator.c:555

目的

演算子の定義パラメータが、その演算子の型構成（単項/二項、戻り値型）と矛盾していないかを検証する。

シグネチャまたは形状

void OperatorValidateParams(Oid leftTypeId,
                            Oid rightTypeId,
                            Oid operResultType,
                            bool hasCommutator,
                            bool hasNegator,
                            bool hasRestrictionSelectivity,
                            bool hasJoinSelectivity,
                            bool canMerge,
                            bool canHash)

入力と出力

  • **入力**: 演算子の左辺型 (leftTypeId)、右辺型 (rightTypeId)、戻り値型 (operResultType)、および各属性の有無を示すフラグ群。
  • **出力**: なし（戻り値 void）。検証に失敗した場合は ereport(ERROR, ...) によりトランザクションを中断する。

前提条件と不変条件

  • leftTypeId または rightTypeId のいずれかが無効（!OidIsValid）な場合、単項演算子とみなされる。
  • operResultType が BOOLOID 以外の場合、ブール演算子特有の属性は設定できない。

アルゴリズムまたは振る舞い

1. **二項演算子の検証**: leftTypeId と rightTypeId の双方が有効でない場合、以下の属性が設定されているとエラーを発生させる。

  • hasCommutator (交換子)
  • hasJoinSelectivity (結合選択率)
  • canMerge (マージ結合)
  • canHash (ハッシュ結合)

    • 2. **ブール演算子の検証**: operResultType が BOOLOID でない場合、以下の属性が設定されているとエラーを発生させる。

  • hasNegator (否定子)
  • hasRestrictionSelectivity (制限選択率)
  • hasJoinSelectivity (結合選択率)
  • canMerge (マージ結合)
  • canHash (ハッシュ結合)

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_operator.c (演算子作成処理内)
  • src/backend/commands/operatorcmds.c (DefineOperator 等のコマンド実行時)
  • **呼び出し先**:
  • ereport (エラー報告用マクロ)

関連データ構造

  • pg_operator カタログ（本関数は、このカタログに挿入されるパラメータの整合性を保証する）

失敗モードと境界条件

  • ERRCODE_INVALID_FUNCTION_DEFINITION を伴うエラーが発生する。
  • 単項演算子に対して二項演算子専用の属性（Commutator等）を指定した場合に失敗する。
  • 非ブール演算子に対してブール演算子専用の属性（Negator等）を指定した場合に失敗する。

保守担当者向けメモ

  • 本関数はカタログへの書き込み前に呼び出されるガードレールである。新しい演算子属性が追加された場合、その属性が単項/二項や戻り値型に対して妥当かどうかを本関数に追加する必要がある。
