check_primary_key

定義

  • 種別: function
  • 定義箇所: contrib/spi/refint.c:49

目的

トリガーベースの参照整合性チェックを行い、挿入または更新されるタプルのキー値が、参照先テーブルに存在することを検証する。

シグネチャまたは形状

Datum check_primary_key(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (トリガーマネージャから渡される TriggerData を含む)。
  • **出力**: Datum (挿入または更新対象のタプル)。
  • **副作用**: 整合性違反時に ereport(ERROR) を発行しトランザクションを中断する。

前提条件と不変条件

  • AFTER トリガーとして起動されること。
  • 行レベル（ROW）トリガーとして起動されること。
  • DELETE イベントでは使用不可。
  • CREATE TRIGGER の引数として、キーカラム名と参照先テーブル名を交互に指定する必要がある（引数の数は奇数であること）。

アルゴリズムまたは振る舞い

1. **検証**: トリガーマネージャからの呼び出しであるか、イベント種別が適切かを確認する。 2. **計画の取得**: トリガー名とリレーションIDから find_plan を呼び出し、キャッシュされた実行計画を検索する。 3. **準備**: 計画が存在しない場合、SPI_prepare を用いて SELECT 1 FROM ... WHERE ... クエリを構築し、TopMemoryContext に保持する。 4. **実行**: SPI_execp を使用して、対象タプルのキー値で参照先テーブルを検索する。 5. **判定**: 検索結果が 0 件の場合、整合性違反としてエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のトリガーマネージャ（CREATE TRIGGER 文経由）。
  • **呼び出し先**: SPI_connect, find_plan, SPI_fnumber, SPI_getbinval, SPI_gettypeid, SPI_prepare, SPI_keepplan, SPI_execp, SPI_finish。

関連データ構造

  • TriggerData: トリガーの実行コンテキスト。
  • EPlan: 実行計画を管理する構造体。
  • SPIPlanPtr: SPI の実行計画ポインタ。

失敗モードと境界条件

  • **エラー**:
  • トリガーマネージャ以外からの呼び出し。
  • DELETE イベントでの呼び出し。
  • 引数の数が偶数（不正な構成）。
  • 存在しないカラムの指定。
  • 参照先テーブルにキーが存在しない場合（ERRCODE_TRIGGERED_ACTION_EXCEPTION）。
  • **境界条件**: キー値が NULL の場合はチェックをスキップし、タプルをそのまま返す。

保守担当者向けメモ

  • 本関数は古いスタイルのトリガー実装である。現代の PostgreSQL では標準の FOREIGN KEY 制約の使用が推奨される。
  • SPI_keepplan を使用して計画を TopMemoryContext に保持しているため、メモリリークに注意すること。
  • DEBUG_QUERY マクロを有効にすると、実行される SQL がログに出力される。
