create_and_test_bloom

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_bloomfilter/test_bloomfilter.c:71

目的

ブルームフィルタの構築、データ投入、および偽陽性率の測定を統括し、統計的な妥当性を検証する。

シグネチャまたは形状

static void create_and_test_bloom(int power, int64 nelements, int callerseed)

入力と出力

  • **入力**:
  • power: ビット配列サイズを決定する指数（23〜32）。
  • nelements: フィルタに投入する要素数。
  • callerseed: 乱数生成器のシード値（負の場合はグローバルな乱数を使用）。
  • **出力**: なし（ereport を介してログまたは警告を出力）。

前提条件と不変条件

  • power はビット配列サイズを算出するために使用され、メモリ確保の基盤となる。
  • seed は PG_INT32_MAX 以下である必要がある。
  • 偽陽性率が FPOSITIVE_THRESHOLD (0.01) を超える場合、WARNING レベルで報告される。

アルゴリズムまたは振る舞い

1. power から bloom_work_mem を算出する。 2. callerseed に基づき、または乱数生成器から seed を決定する。 3. bloom_create でフィルタを生成する。 4. populate_with_dummy_strings で要素を投入する。 5. nfalsepos_for_missing_strings で偽陽性数をカウントする。 6. 偽陽性率を計算し、結果をログ出力する。 7. bloom_free でフィルタのメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_bloomfilter/test_bloomfilter.c:134（test_bloomfilter 関数内）
  • **呼び出し先**:
  • bloom_create
  • populate_with_dummy_strings

  • bloom_free
  • pg_prng_int32p
  • elog / ereport

関連データ構造

  • bloom_filter: ブルームフィルタの内部状態を保持する構造体。

失敗モードと境界条件

  • 偽陽性率が閾値を超えた場合、WARNING が発行される。
  • power の値が不適切な場合、メモリ確保やフィルタの動作に影響を与える可能性がある。

保守担当者向けメモ

  • 本関数はテスト用モジュールの一部であり、確率的な事象を扱うため、テスト結果が閾値付近で変動する可能性がある。
  • CHECK_FOR_INTERRUPTS() は呼び出し元のループ内に配置されているため、本関数内では明示的に呼び出されていない。
