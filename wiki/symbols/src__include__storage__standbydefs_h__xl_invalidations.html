xl_invalidations

定義

  • 種別: struct
  • 定義箇所: src/include/storage/standbydefs.h:63

目的

ホットスタンバイ環境において、プライマリサーバーからスタンバイサーバーへキャッシュ無効化メッセージを伝播させるためのWALレコード構造体です。

シグネチャまたは形状

typedef struct xl_invalidations
{
    Oid            dbId;
    Oid            tsId;
    bool           relcacheInitFileInval;
    int            nmsgs;
    SharedInvalidationMessage msgs[FLEXIBLE_ARRAY_MEMBER];
} xl_invalidations;

入力と出力

  • **入力**: プライマリ側で生成されたキャッシュ無効化メッセージの集合。
  • **出力**: スタンバイ側でリカバリ処理中に読み込まれ、共有メモリ上のキャッシュ状態を更新するために使用されます。

前提条件と不変条件

  • FLEXIBLE_ARRAY_MEMBER を使用した可変長構造体です。
  • 構造体のサイズ計算には、定義されているマクロ MinSizeOfInvalidations (offsetof(xl_invalidations, msgs)) を使用する必要があります。

アルゴリズムまたは振る舞い

1. **記録**: プライマリ側で発生した無効化イベントを msgs 配列に格納し、WALレコードとして書き出します。 2. **再生**: スタンバイ側で XLogReader を介してレコードを読み込みます。standby_redo 関数がこの構造体を参照し、含まれるメッセージを適用してスタンバイ側のキャッシュ整合性を維持します。

呼び出し元と呼び出し先

  • **呼び出し元 (参照)**:
  • src/backend/access/rmgrdesc/standbydesc.c: レコードのデバッグ出力に使用。
  • src/backend/storage/ipc/standby.c: XLogRecGetData を介してレコードを読み込み、適用処理を実行。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • SharedInvalidationMessage: msgs 配列の各要素の型。
  • xl_standby_locks, xl_running_xacts: 同様にスタンバイの同期に使用される他のWALレコード構造体。

失敗モードと境界条件

  • 可変長配列のため、メモリレイアウトを直接扱う際はオフセット計算を誤るとバッファオーバーランや不正なメモリ参照を引き起こす可能性があります。必ず MinSizeOfInvalidations を基準とした計算を行ってください。

保守担当者向けメモ

  • 本構造体は src/include/storage/standbydefs.h で定義されています。
  • 構造体の変更時は、standby_redo 側の適用ロジックおよび standby_desc 側のデバッグ出力ロジックとの整合性を確認してください。
