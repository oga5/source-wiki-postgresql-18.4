hash_get_shared_size

定義

  • 種別: function
  • 定義箇所: src/backend/utils/hash/dynahash.c:854

目的

共有メモリ上に配置されるハッシュテーブルの制御ヘッダーおよびディレクトリ構造に必要な合計サイズを計算します。

シグネチャまたは形状

Size hash_get_shared_size(HASHCTL *info, int flags)

入力と出力

  • **入力**:
  • info: ハッシュテーブルの構成情報を含む HASHCTL 構造体へのポインタ。
  • flags: ビットフラグ。HASH_DIRSIZE が設定されている必要があります。
  • **出力**: 必要なメモリサイズ（Size 型）。

前提条件と不変条件

  • flags に HASH_DIRSIZE が含まれている必要があります（Assert により検証）。
  • info->dsize（現在のディレクトリサイズ）と info->max_dsize（最大ディレクトリサイズ）が一致している必要があります（Assert により検証）。

アルゴリズムまたは振る舞い

1. HASH_DIRSIZE フラグの有無を確認します。 2. dsize と max_dsize が等しいことを確認します。 3. sizeof(HASHHDR)（ハッシュテーブルヘッダー）と、info->dsize 分の HASHSEGMENT（ディレクトリセグメント）の合計サイズを計算して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/shmem.c:354 からの呼び出しが確認されています。
  • **呼び出し先**:
  • 本関数内での関数呼び出しはありません。

関連データ構造

  • HASHCTL: ハッシュテーブルの初期化パラメータを保持する構造体。
  • HASHHDR: 共有メモリ上のハッシュテーブルヘッダー。
  • HASHSEGMENT: ハッシュテーブルのディレクトリを構成するセグメント。

失敗モードと境界条件

  • flags に HASH_DIRSIZE が指定されていない場合、または dsize と max_dsize が不一致の場合、デバッグビルドではアサーション失敗によりプログラムが停止します。

保守担当者向けメモ

  • 本関数は共有メモリハッシュテーブルの初期化時に必要な領域を算出するために使用されます。
  • src/include/utils/hsearch.h で外部公開されています。
