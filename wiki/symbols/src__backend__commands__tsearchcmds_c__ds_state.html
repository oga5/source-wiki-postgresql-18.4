ds_state

定義

  • 種別: enum
  • 定義箇所: src/backend/commands/tsearchcmds.c:1631

目的

全文検索（Tsearch）に関連する辞書オプション等の設定文字列を解析する際の、ステートマシン状態を定義します。

シグネチャまたは形状

typedef enum {
    CS_WAITKEY,
    CS_INKEY,
    CS_INQKEY,
    CS_WAITEQ,
    CS_WAITVALUE,
    CS_INSQVALUE,
    CS_INDQVALUE,
    CS_INWVALUE
} ds_state;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし

前提条件と不変条件

  • 辞書オプションの解析ロジックにおいて、現在の解析位置の状態を保持するために使用されます。

アルゴリズムまたは振る舞い

  • CS_WAITKEY: キーの開始を待機する状態。
  • CS_INKEY: キーの読み取り中。
  • CS_INQKEY: クォートされたキーの読み取り中。
  • CS_WAITEQ: キーと値の間の等号（=）を待機する状態。
  • CS_WAITVALUE: 値の開始を待機する状態。
  • CS_INSQVALUE: シングルクォートで囲まれた値の読み取り中。
  • CS_INDQVALUE: ダブルクォートで囲まれた値の読み取り中。
  • CS_INWVALUE: クォートされていない値の読み取り中。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/tsearchcmds.c 内の辞書オプション解析処理。
  • 呼び出し先: 情報なし。

関連データ構造

  • なし

失敗モードと境界条件

  • 状態遷移の定義のみであり、このシンボル自体に失敗モードはありません。

保守担当者向けメモ

  • 本列挙型は辞書定義のオプション文字列をパースする際の内部状態管理用です。解析ロジックの変更時には、この状態遷移図が網羅的であるか確認してください。
