plperl_init_shared_libs

定義

  • 種別: function
  • 定義箇所: src/pl/plperl/plperl.c:2168

目的

Perl インタプリタの初期化プロセスにおいて、PL/Perl 拡張に必要な共有ライブラリ（XS モジュール）を Perl 空間に登録する。

シグネチャまたは形状

static void plperl_init_shared_libs(pTHX)
  • pTHX: Perl インタプリタのコンテキスト（Perl のスレッドローカルストレージ）。

入力と出力

  • 入力: なし（pTHX は暗黙的なコンテキスト）。
  • 出力: なし（副作用として Perl インタプリタ内部のシンボルテーブルを更新）。

前提条件と不変条件

  • Perl インタプリタが初期化中であること。
  • boot_DynaLoader および boot_PostgreSQL__InServer__Util がリンク可能であること。

アルゴリズムまたは振る舞い

1. DynaLoader::boot_DynaLoader を newXS を用いて Perl 空間に登録し、動的ロード機能を有効化する。 2. PostgreSQL::InServer::Util::bootstrap を newXS を用いて登録し、PL/Perl ユーティリティ関数群を Perl 空間から利用可能にする。 3. __FILE__ をファイル名として登録し、デバッグ情報として保持する。

呼び出し元と呼び出し先

  • 呼び出し元: perl_parse（src/pl/plperl/plperl.c:848）の引数として渡され、Perl インタプリタの起動時にコールバックとして実行される。
  • 呼び出し先:
  • newXS (Perl API)
  • boot_DynaLoader (外部シンボル)
  • boot_PostgreSQL__InServer__Util (外部シンボル)

関連データ構造

  • pTHX: Perl インタプリタの実行コンテキスト。

失敗モードと境界条件

  • Perl インタプリタの起動に失敗している場合、本関数は到達しない。
  • newXS が失敗した場合、Perl インタプリタの初期化プロセス自体が異常終了する可能性がある。

保守担当者向けメモ

  • SPI 関連のブートストラップ（...::SPI::bootstrap）は本関数ではなく select_perl_context() 内で行われるため、混同しないこと。
  • 本関数は perl_parse のコールバックとして設計されているため、実行タイミングはインタプリタの初期化フェーズに限定される。
