LargeObjectExistsWithSnapshot

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_largeobject.c:175

目的

指定された OID を持つラージオブジェクトが、特定の Snapshot の時点においてシステムカタログ pg_largeobject_metadata に存在するかを検証します。

シグネチャまたは形状

bool LargeObjectExistsWithSnapshot(Oid loid, Snapshot snapshot)

入力と出力

  • **入力**:
  • loid (Oid): 存在を確認するラージオブジェクトの識別子。
  • snapshot (Snapshot): 検索に使用するスナップショット。
  • **出力**:
  • bool: オブジェクトが存在する場合は true、存在しない場合は false を返します。

前提条件と不変条件

  • システムキャッシュを介さず、カタログテーブルを直接走査します。
  • 呼び出し元のトランザクション境界内で実行されます。

アルゴリズムまたは振る舞い

1. pg_largeobject_metadata テーブルに対するスキャンキーを loid で初期化します。 2. table_open を使用して pg_largeobject_metadata を AccessShareLock で開きます。 3. systable_beginscan を呼び出し、指定された snapshot を用いてインデックス（LargeObjectMetadataOidIndexId）経由で検索を開始します。 4. systable_getnext でタプルを取得し、有効なタプルが存在すれば true を返します。 5. systable_endscan でスキャンを終了し、table_close でリレーションを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_largeobject.c:169
  • src/backend/storage/large_object/inv_api.c:244
  • src/backend/utils/adt/acl.c:4714
  • **呼び出し先**:
  • table_open
  • systable_beginscan
  • systable_getnext
  • systable_endscan
  • table_close

関連データ構造

  • pg_largeobject_metadata: 検索対象のシステムカタログ。
  • LargeObjectMetadataOidIndexId: 検索に使用するインデックス。
  • ScanKeyData: 検索条件を保持する構造体。

失敗モードと境界条件

  • snapshot が無効な場合、検索結果は不定となります。
  • pg_largeobject_metadata テーブルへのアクセス権限がない場合、table_open でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はシステムキャッシュを使用しないため、頻繁な呼び出しはカタログへの直接アクセスを伴い、パフォーマンスに影響を与える可能性があります。
  • 削除処理（LargeObjectDrop）との整合性を保つため、メタデータの存在確認には常にこの関数を使用してください。
