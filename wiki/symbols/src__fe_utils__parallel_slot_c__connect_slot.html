connect_slot

定義

  • 種別: function
  • 定義箇所: src/fe_utils/parallel_slot.c:286

目的

指定された ParallelSlotArray 内の特定スロットに対してデータベース接続を確立し、必要に応じて初期化コマンドを実行します。

シグネチャまたは形状

static void connect_slot(ParallelSlotArray *sa, int slotno, const char *dbname);

入力と出力

  • **入力**:
  • sa: 接続パラメータやスロット配列を保持する ParallelSlotArray 構造体へのポインタ。
  • slotno: 接続対象となるスロットのインデックス。
  • dbname: 接続先データベース名（NULL の場合は既存のパラメータを使用）。
  • **出力**: なし（副作用として sa->slots[slotno].connection を更新）。

前提条件と不変条件

  • sa->cparams が有効な接続パラメータを保持していること。
  • select() を使用するプラットフォーム上の制約として、ファイルディスクリプタ番号が FD_SETSIZE 未満である必要があります。

アルゴリズムまたは振る舞い

1. sa->cparams->override_dbname を一時的に退避し、指定された dbname で上書きします。 2. connectDatabase を呼び出して接続を確立し、スロットに格納します。 3. 退避していた override_dbname を復元します。 4. プラットフォームに応じたファイルディスクリプタの検証を行います。

  • Windows環境: slotno が FD_SETSIZE 以上の場合、エラーを出力して終了します。
  • その他: PQsocket で取得したディスクリプタが FD_SETSIZE 以上の場合、エラーを出力して終了します。

    • 5. sa->initcmd が設定されている場合、確立した接続に対して executeCommand を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/fe_utils/parallel_slot.c 内の内部関数（ParallelSlotsGetIdle 等から呼び出される可能性あり）。
  • **呼び出し先**:
  • connectDatabase
  • PQsocket
  • pg_log_error / pg_log_error_hint
  • exit
  • executeCommand

関連データ構造

  • ParallelSlotArray: 接続パラメータおよびスロット配列を管理する構造体。
  • ParallelSlot: 個別の接続状態を保持する構造体。

失敗モードと境界条件

  • **ディスクリプタ制限**: FD_SETSIZE（通常1024）を超えるファイルディスクリプタが割り当てられた場合、プログラムはエラーメッセージを出力して即座に終了します。
  • **接続失敗**: connectDatabase が失敗した場合の挙動は、同関数の実装に依存します。

保守担当者向けメモ

  • 本関数は select() の制限に強く依存しており、Windows環境とPOSIX環境で検証ロジックが異なります。
  • pgbench 等のツールから参照される重要なインフラストラクチャの一部です。
