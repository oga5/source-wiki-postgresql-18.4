GetCurrentTimeUsec

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/datetime.c:396

目的

現在のトランザクション開始時刻を、指定されたタイムゾーン設定に基づいて pg_tm 構造体および小数秒（fsec_t）へ変換して取得する。

シグネチャまたは形状

void GetCurrentTimeUsec(struct pg_tm *tm, fsec_t *fsec, int *tzp)

入力と出力

  • **入力**:
  • tm: 結果を格納する struct pg_tm へのポインタ。
  • fsec: 小数秒を格納する fsec_t へのポインタ。
  • tzp: タイムゾーンオフセット（秒）を格納する int へのポインタ（NULL 指定可能）。
  • **出力**: なし（引数経由で値を書き戻す）。

前提条件と不変条件

  • GetCurrentTransactionStartTimestamp() が有効なトランザクション開始時刻を返すこと。
  • session_timezone が現在のセッションで設定されていること。
  • 内部キャッシュ（cache_ts, cache_timezone 等）は、トランザクション開始時刻またはセッションのタイムゾーンが変更された場合にのみ更新される。

アルゴリズムまたは振る舞い

1. 現在のトランザクション開始時刻（cur_ts）を取得する。 2. キャッシュされた時刻およびタイムゾーンと現在の値を比較する。 3. 値が異なる場合、timestamp2tm() を呼び出して時刻を分解し、結果を静的変数（キャッシュ）に保存する。 4. timestamp2tm() が失敗した場合は ERRCODE_DATETIME_VALUE_OUT_OF_RANGE でエラーを報告する。 5. キャッシュから tm および fsec に値をコピーし、tzp が指定されていればオフセット値を書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/date.c
  • src/backend/utils/adt/datetime.c（内部のラッパー関数等）
  • **呼び出し先**:
  • GetCurrentTransactionStartTimestamp()
  • timestamp2tm()
  • ereport()

関連データ構造

  • struct pg_tm: 日時要素（年、月、日、時、分、秒）を保持する構造体。
  • fsec_t: 小数秒を表現する型。
  • TimestampTz: タイムゾーン付きのタイムスタンプ型。
  • pg_tz: タイムゾーン情報を保持する構造体。

失敗モードと境界条件

  • timestamp2tm() の計算結果が範囲外となった場合、ERROR を発生させトランザクションを中断する。
  • タイムゾーンポインタの比較によりキャッシュの有効性を判定しているため、pg_tzset() 内のハッシュテーブルでエントリが再利用されるような状況下では、キャッシュの整合性に注意が必要である（現状の設計では安全と見なされている）。

保守担当者向けメモ

  • 本関数は静的変数を用いたキャッシュ機構を備えており、頻繁な呼び出しに対する最適化が行われている。
  • timestamp2tm の呼び出し結果をキャッシュするため、セッション設定の変更やトランザクションの進行に伴うキャッシュ無効化ロジックが正しく機能していることを確認すること。
