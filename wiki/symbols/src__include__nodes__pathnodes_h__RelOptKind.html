RelOptKind

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/pathnodes.h:849

目的

クエリプランナが扱うリレーション（RelOptInfo）の種類を識別し、最適化の各フェーズにおけるリレーションの役割を区別するために使用されます。

シグネチャまたは形状

typedef enum RelOptKind
{
	RELOPT_BASEREL,
	RELOPT_JOINREL,
	RELOPT_OTHER_MEMBER_REL,
	RELOPT_OTHER_JOINREL,
	RELOPT_UPPER_REL,
	RELOPT_OTHER_UPPER_REL,
} RelOptKind;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: RelOptInfo 構造体の reloptkind フィールドとして保持されます。

前提条件と不変条件

  • RelOptInfo 構造体は必ずいずれか一つの RelOptKind を保持します。
  • 特定の最適化フェーズ（ベースリレーションの生成、結合探索、上位リレーションの処理など）において、適切な種類が割り当てられる必要があります。

アルゴリズムまたは振る舞い

プランナは、この列挙値に基づいて RelOptInfo を分類し、以下の処理を制御します。

  • RELOPT_BASEREL: 基本的なテーブルやサブクエリのスキャン対象。
  • RELOPT_JOINREL: 複数のリレーションを結合した結果。
  • RELOPT_UPPER_REL: 集約、ソート、ウィンドウ関数などの上位処理の結果。
  • OTHER_ 接頭辞が付く値: パーティションテーブルのメンバや、特定の最適化コンテキストにおける特殊なリレーションを識別します。

呼び出し元と呼び出し先

  • 呼び出し元: RelOptInfo 構造体を初期化するプランナの各関数。
  • 呼び出し先: 本シンボル自体はデータ型であり、関数を呼び出しません。
  • 補足: 具体的な関数レベルの呼び出し関係は、提供されたソース情報からは特定できません。

関連データ構造

  • RelOptInfo: 本列挙型をメンバとして保持する、リレーションの最適化情報を格納する主要構造体。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 新しいリレーションの種類を追加する必要がある場合は、この列挙型を拡張し、プランナ内の RelOptInfo を扱う各ロジック（特に RelOptInfo の生成および判定箇所）を更新してください。
  • RELOPT_UPPER_REL は、通常の結合やスキャンとは異なる上位処理（集約等）を扱うために使用されます。
