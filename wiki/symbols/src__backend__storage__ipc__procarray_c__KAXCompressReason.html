KAXCompressReason

定義

  • 種別: enum
  • 定義箇所: src/backend/storage/ipc/procarray.c:260

目的

ホットスタンバイ時における KnownAssignedXids 配列の圧縮処理を実行する際、そのトリガーとなった理由を識別するために使用されます。

シグネチャまたは形状

typedef enum KAXCompressReason
{
    KAX_NO_SPACE,               /* 配列末尾の空き領域確保が必要 */
    KAX_PRUNE,                  /* 古いエントリの削除（prune）直後 */
    KAX_TRANSACTION_END,        /* トランザクションのコミットまたは削除直後 */
    KAX_STARTUP_PROCESS_IDLE,   /* スタートアッププロセスが待機状態に入る直前 */
} KAXCompressReason;

入力と出力

  • 入力: KnownAssignedXidsCompress 関数の引数として渡されます。
  • 出力: なし（列挙型定数）。

前提条件と不変条件

  • KnownAssignedXidsCompress 関数内でのみ使用されることを想定しています。
  • 各列挙値は、配列の圧縮が必要な特定のシステム状態に対応しています。

アルゴリズムまたは振る舞い

KnownAssignedXidsCompress 関数において、この列挙値に基づいて圧縮処理の実行判断やログ出力、あるいは処理の最適化が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/procarray.c 内の KnownAssignedXidsCompress 関数がこの型を引数として受け取ります。
  • 呼び出し先: 情報なし。

関連データ構造

  • KnownAssignedXids: ホットスタンバイ時に実行中のトランザクションを追跡する配列。

失敗モードと境界条件

  • 列挙型に含まれない値が渡された場合の挙動は定義されていません。

保守担当者向けメモ

  • 本シンボルは KnownAssignedXidsCompress の動作を制御するためのフラグとして機能します。新しい圧縮トリガーを追加する場合は、この列挙型を拡張してください。
