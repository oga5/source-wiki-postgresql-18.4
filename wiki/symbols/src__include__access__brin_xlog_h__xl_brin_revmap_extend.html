xl_brin_revmap_extend

定義

  • 種別: struct
  • 定義箇所: src/include/access/brin_xlog.h:115

目的

BRIN（Block Range Index）インデックスの Revmap ページが拡張された際の WAL（Write-Ahead Logging）レコード構造を定義します。

シグネチャまたは形状

typedef struct xl_brin_revmap_extend
{
    BlockNumber targetBlk;
} xl_brin_revmap_extend;

入力と出力

  • **入力**: targetBlk（拡張対象のブロック番号）。
  • **出力**: なし（本構造体は WAL レコードのデータ部として書き込まれます）。

前提条件と不変条件

  • targetBlk は、バックアップブロック（backup block 1）の一部として格納されているため、本構造体内の targetBlk は冗長な情報である可能性があります。

アルゴリズムまたは振る舞い

1. BRIN インデックスの Revmap ページが拡張される際、この構造体に情報を格納して WAL レコードとして記録します。 2. リカバリ（Redo）時には、XLogRecGetData を通じてレコードから本構造体を読み出し、対象のインデックスページに対する操作を再実行します。

呼び出し元と呼び出し先

  • **呼び出し元（記録側）**: src/backend/access/brin/brin_revmap.c がこの構造体を使用して WAL レコードを生成します。
  • **呼び出し先（再生側）**: src/backend/access/brin/brin_xlog.c がリカバリ時にこの構造体を読み取り、インデックスの整合性を復元します。
  • **その他**: src/backend/access/rmgrdesc/brindesc.c が WAL レコードのデバッグ出力（説明）のために本構造体を参照します。

関連データ構造

  • SizeOfBrinRevmapExtend: 本構造体のサイズを計算するためのマクロが src/include/access/brin_xlog.h に定義されています。

失敗モードと境界条件

  • 構造体内の targetBlk とバックアップブロックの情報が不整合を起こす可能性については、ソースコード上のコメントで冗長性が指摘されています。

保守担当者向けメモ

  • 本構造体は brin_xlog.c での Redo 処理と密接に関連しています。構造を変更する場合は、brindesc.c のデバッグ出力関数および SizeOfBrinRevmapExtend マクロの更新を忘れないでください。
