ReceiveCopyBinaryHeader

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfromparse.c:189

目的

COPY FROM コマンドのバイナリ形式入力において、ファイルヘッダーの検証と初期化を行う。

シグネチャまたは形状

void ReceiveCopyBinaryHeader(CopyFromState cstate)

入力と出力

  • **入力**: cstate (CopyFromState) - COPY 操作の状態を保持する構造体。
  • **出力**: なし（成功時はヘッダーを消費し、失敗時は ereport(ERROR, ...) を通じて例外を発生させる）。

前提条件と不変条件

  • 入力ストリームがバイナリ形式の開始位置にあること。
  • BinarySignature が定義されていること。

アルゴリズムまたは振る舞い

1. **シグネチャ検証**: 入力から 11 バイトを読み込み、BinarySignature と一致するか確認する。 2. **フラグ検証**: 4 バイトのフラグフィールドを読み込む。WITH OIDS フラグ（ビット 16）が立っている場合、または未定義のクリティカルフラグが存在する場合はエラーとする。 3. **ヘッダー拡張長の処理**: 4 バイトの拡張ヘッダー長を読み込む。値が負の場合はエラーとする。 4. **拡張ヘッダーのスキップ**: 読み込んだ長さ分だけ入力ストリームを読み飛ばす。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfrom.c:224
  • **呼び出し先**: CopyReadBinaryData, CopyGetInt32

関連データ構造

  • CopyFromState: COPY FROM の実行状態を管理する構造体（src/backend/commands/copyfrom_internal.h で定義）。

失敗モードと境界条件

  • **ERRCODE_BAD_COPY_FILE_FORMAT**:
  • シグネチャが一致しない場合。
  • フラグフィールドが読み込めない、または不正なフラグ（WITH OIDS 等）が含まれる場合。
  • 拡張ヘッダー長が読み込めない、または負の値である場合。
  • 拡張ヘッダーのスキップ中にデータが不足した場合。

保守担当者向けメモ

  • 本関数はバイナリ形式の COPY プロトコルの先頭部分を厳密に検証する。プロトコル仕様の変更時には、フラグのビット定義やシグネチャの長さを確認すること。
