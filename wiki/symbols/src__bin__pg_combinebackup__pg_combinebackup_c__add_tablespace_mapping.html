add_tablespace_mapping

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/pg_combinebackup.c:445

目的

コマンドライン引数からテーブルスペースのパス置換ルール（OLDDIR=NEWDIR）を解析し、cb_options 構造体のマッピングリストに追加する。

シグネチャまたは形状

static void add_tablespace_mapping(cb_options *opt, char *arg)

入力と出力

  • opt: 実行オプションを保持する cb_options 構造体へのポインタ（入力および更新対象）。
  • arg: "OLDDIR=NEWDIR" 形式の文字列（入力）。
  • 出力: なし（副作用として opt->tsmappings リストを更新）。

前提条件と不変条件

  • arg には = がちょうど1つ含まれている必要がある。
  • OLDDIR および NEWDIR は絶対パスでなければならない。
  • \ を用いて = をエスケープ可能（\= はリテラルの = として扱われる）。
  • パスは正規化（canonicalize_path）される。

アルゴリズムまたは振る舞い

1. cb_tablespace_mapping 構造体をメモリ確保する。 2. arg を走査し、= を境界として old_dir と new_dir に分割する。 3. 複数の = が存在する場合や、パスが空の場合は pg_fatal で終了する。 4. is_absolute_path を用いて両パスが絶対パスであることを検証する。 5. canonicalize_path でパスを正規化する。 6. 新しいマッピングを opt->tsmappings リストの先頭に挿入する。

呼び出し元と呼び出し先

  • 呼び出し元: main 関数（src/bin/pg_combinebackup/pg_combinebackup.c:199）にて、コマンドライン引数の解析中に呼び出される。
  • 呼び出し先: pg_malloc0, pg_fatal, is_absolute_path, canonicalize_path（いずれも PostgreSQL 共通ユーティリティ）。

関連データ構造

  • cb_options: 実行設定を保持する構造体。
  • cb_tablespace_mapping: テーブルスペースのパス置換情報を保持する連結リスト構造体。

失敗モードと境界条件

  • MAXPGPATH を超えるディレクトリ名は pg_fatal を引き起こす。
  • = が含まれない、または複数含まれる場合は pg_fatal を引き起こす。
  • 相対パスが指定された場合は pg_fatal を引き起こす。

保守担当者向けメモ

  • 本関数は pg_basebackup のテーブルスペースマッピング処理と類似しているが、ローカル環境でのパス操作に特化している点に注意すること。
  • tsmappings リストはスタックのように先頭に追加されるため、リストの順序は引数指定の逆順となる。
