ExecIncrementalSortEstimate

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIncrementalSort.c:1172

目的

パラレルクエリ実行時、インクリメンタルソートノードの統計情報を共有メモリに格納するために必要な領域サイズを見積もります。

シグネチャまたは形状

void ExecIncrementalSortEstimate(IncrementalSortState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: インクリメンタルソートの状態を保持する IncrementalSortState 構造体。
  • pcxt: パラレルクエリのコンテキスト情報。
  • **出力**: なし（副作用として pcxt->estimator に見積もりサイズとキーが登録されます）。

前提条件と不変条件

  • node->ss.ps.instrument が有効であり、かつ pcxt->nworkers > 0 である場合にのみ処理が行われます。

アルゴリズムまたは振る舞い

1. 実行統計の計測が不要、またはワーカーが存在しない場合は即座に終了します。 2. ワーカー数に基づき、IncrementalSortInfo 配列を格納するための合計サイズを計算します。 3. SharedIncrementalSortInfo 構造体のヘッダーサイズを加算します。 4. shm_toc_estimate_chunk を呼び出し、計算したサイズを共有メモリの見積もりに追加します。 5. shm_toc_estimate_keys を呼び出し、共有メモリ内のキー領域を 1 つ確保します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:298 において、パラレル実行計画の初期化プロセスから呼び出されます。
  • **呼び出し先**: mul_size, add_size, shm_toc_estimate_chunk, shm_toc_estimate_keys（いずれも共有メモリ管理用のユーティリティ関数）。

関連データ構造

  • IncrementalSortState: ノードの実行状態。
  • ParallelContext: パラレルクエリの実行コンテキスト。
  • IncrementalSortInfo: 各ワーカーのソート統計情報。
  • SharedIncrementalSortInfo: 共有メモリ上に配置される統計情報のコンテナ。

失敗モードと境界条件

  • pcxt->nworkers が 0 の場合、見積もり処理はスキップされます。
  • node->ss.ps.instrument が NULL の場合、見積もり処理はスキップされます。

保守担当者向けメモ

  • 本関数は、ExecIncrementalSortInitializeDSM と対になる見積もり処理です。共有メモリのレイアウトを変更する場合は、両方の関数を整合させる必要があります。
