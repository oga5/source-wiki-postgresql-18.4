serializeAnalyzeStartup

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_dr.c:207

目的

EXPLAIN (SERIALIZE) オプション実行時における、シリアライズ計測用 DestReceiver の初期化を行う。

シグネチャまたは形状

static void serializeAnalyzeStartup(DestReceiver *self, int operation, TupleDesc typeinfo);

入力と出力

  • **入力**:
  • self: SerializeDestReceiver 構造体へのポインタ。
  • operation: 実行される操作の種類。
  • typeinfo: タプルの記述情報。
  • **出力**: なし（副作用として self が指す構造体の内部状態を初期化する）。

前提条件と不変条件

  • self が指す SerializeDestReceiver の es（ExplainState）が NULL でないこと。
  • receiver->es->serialize が EXPLAIN_SERIALIZE_NONE 以外であること。

アルゴリズムまたは振る舞い

1. receiver->es->serialize の値に基づき、receiver->format を設定する（TEXT なら 0、BINARY なら 1）。 2. 行ごとのメモリ管理用に AllocSetContextCreate を呼び出し、tmpcontext を作成する。 3. initStringInfo を呼び出し、シリアライズ用バッファ buf を初期化する。 4. receiver->metrics をゼロクリアし、計測用タイマーをリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていないが、DestReceiver インターフェースの rStartup フィールドに本関数が割り当てられている（src/backend/commands/explain_dr.c:281）。
  • **呼び出し先**: AllocSetContextCreate, initStringInfo, memset, INSTR_TIME_SET_ZERO。

関連データ構造

  • SerializeDestReceiver: 計測状態を保持する構造体。
  • SerializeMetrics: 計測結果を格納する構造体。
  • ExplainState: EXPLAIN コマンドの状態管理用構造体。

失敗モードと境界条件

  • EXPLAIN_SERIALIZE_NONE が指定されている場合、Assert(false) によりアサーション失敗となる。

保守担当者向けメモ

  • 本関数は printtup.c の初期化ロジックを模倣している。
  • バッファは行間で再利用されるため、メモリ管理のライフサイクルに注意すること。
