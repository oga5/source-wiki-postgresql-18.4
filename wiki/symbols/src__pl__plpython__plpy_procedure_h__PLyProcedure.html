PLyProcedure

定義

  • 種別: struct
  • 定義箇所: src/pl/plpython/plpy_procedure.h:25

目的

PL/Pythonにおける関数およびトリガーの実行に必要なメタデータとコンパイル済みコードを保持し、システムカタログへの再アクセスを回避するためのキャッシュ用構造体です。

シグネチャまたは形状

typedef struct PLyProcedure {
    MemoryContext mcxt;
    char *proname;
    char *pyname;
    TransactionId fn_xmin;
    ItemPointerData fn_tid;
    bool fn_readonly;
    bool is_setof;
    bool is_procedure;
    bool is_trigger;
    PLyObToDatum result;
    PLyDatumToOb result_in;
    char *src;
    char **argnames;
    PLyDatumToOb *args;
    int nargs;
    Oid langid;
    List *trftypes;
    PyObject *code;
    PyObject *statics;
    PyObject *globals;
    long calldepth;
    PLySavedArgs *argstack;
} PLyProcedure;

入力と出力

  • **入力**: キャッシュ生成時にシステムカタログから読み取られた関数定義情報。
  • **出力**: 実行時にPython環境へ渡されるコンパイル済みコード (code)、引数変換情報 (args)、および戻り値変換情報 (result)。

前提条件と不変条件

  • mcxt に、構造体本体および付随するデータが保持されます。
  • キャッシュエントリの破棄時には、保持している MemoryContext を適切に解放する必要があります。
  • プロセスごとのメモリ空間で動作し、マルチスレッド環境には対応していません。

アルゴリズムまたは振る舞い

1. **キャッシュ管理**: PLy_procedure_get を通じてハッシュテーブルから取得されます。 2. **初期化**: キャッシュミス時にはシステムカタログから情報を読み取り、Pythonコードのコンパイルおよびデータ変換情報の初期化が行われます。 3. **実行**: calldepth を用いて再帰呼び出しを追跡し、argstack で外側の呼び出し引数を管理します。 4. **状態保持**: statics および globals により、関数呼び出し間でのデータ保持が可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_exec.c 内の各関数（PLy_exec_function, PLy_exec_trigger, PLy_procedure_call 等）。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接関数を呼び出すことはありませんが、PLy_function_build_args 等の実行系関数が本構造体のメンバを参照します。

関連データ構造

  • PLyProcedureKey: キャッシュ検索用のキー。
  • PLyProcedureEntry: キャッシュハッシュテーブルのエントリ。
  • PLySavedArgs: 再帰呼び出し時の引数スタック。
  • PLyObToDatum / PLyDatumToOb: データ型変換情報。

失敗モードと境界条件

  • **キャッシュ無効化**: fn_xmin および fn_tid を用いて、システムカタログ上の定義変更を検知します。
  • **メモリ管理**: 独自の MemoryContext を持つため、不適切な解放はメモリリークや不正なメモリ参照を引き起こす可能性があります。

保守担当者向けメモ

  • トリガーとして呼び出される場合、is_trigger が真となり、リレーションに依存したキャッシュ管理が行われます。
  • src メンバには、マングリング後のテキスト形式のプロシージャコードが格納されます。
