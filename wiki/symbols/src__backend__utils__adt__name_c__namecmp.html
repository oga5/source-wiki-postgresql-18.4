namecmp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/name.c:134

目的

PostgreSQLの name 型（固定長文字列）同士を比較し、その順序関係を判定します。

シグネチャまたは形状

static int namecmp(Name arg1, Name arg2, Oid collid)

入力と出力

  • **入力**:
  • arg1, arg2: 比較対象の Name 型ポインタ。
  • collid: 比較に使用する照合順序（Collation）の OID。
  • **出力**:
  • 比較結果を整数で返します（arg1 < arg2 なら負、arg1 == arg2 なら 0、arg1 > arg2 なら正）。

前提条件と不変条件

  • name 型は NAMEDATALEN バイトの固定長として扱われます。
  • C ロケール（collid == C_COLLATION_OID）の場合、NAMEDATALEN を超える部分は比較されません。

アルゴリズムまたは振る舞い

1. **高速パス**: collid が C_COLLATION_OID と一致する場合、strncmp を用いて NAMEDATALEN バイトまでを直接比較します。 2. **汎用パス**: それ以外の場合、varstr_cmp を呼び出し、指定された照合順序に基づいた文字列比較ロジックへ委譲します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • nameeq, namene, namelt, namelt, namelt, namelt, namelt などの name 型比較演算子関数（src/backend/utils/adt/name.c 内）。
  • **呼び出し先**:
  • strncmp (標準Cライブラリ)
  • varstr_cmp (PostgreSQL 汎用文字列比較関数)

関連データ構造

  • Name: システムカタログ等で使用される固定長文字列型。
  • NameStr(): Name 型から内部の文字列バッファを取り出すマクロ。

失敗モードと境界条件

  • NAMEDATALEN を超える文字列が入力された場合、比較は NAMEDATALEN バイトで打ち切られます。

保守担当者向けメモ

  • 本関数はシステムカタログの比較で頻繁に呼び出されるため、C ロケール時の高速パスが維持されています。照合順序の変更や NAMEDATALEN の定義変更時には影響範囲を確認してください。
