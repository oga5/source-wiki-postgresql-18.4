astreamer_tar_header

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_tar.c:266

目的

tarアーカイブのストリームからヘッダーブロックを解析し、メンバー情報を抽出して後続のストリーマーへ転送する。

シグネチャまたは形状

static bool astreamer_tar_header(astreamer_tar_parser *mystreamer)

入力と出力

  • **入力**: mystreamer（astreamer_tar_parser 型）。bbs_buffer には TAR_BLOCK_SIZE 分のデータが格納されている必要がある。
  • **出力**: bool 型。アーカイブの終端（全バイトがNUL）に達した場合は false、それ以外の場合は true を返す。

前提条件と不変条件

  • mystreamer->base.bbs_buffer.len が TAR_BLOCK_SIZE と等しいこと。
  • 抽出されたパス名が空でないこと、および path_is_safe_for_extraction を満たすこと。

アルゴリズムまたは振る舞い

1. **終端判定**: バッファ全体がNULバイトであれば false を返し、アーカイブの終了を通知する。 2. **検証**: isValidTarHeader を使用してヘッダーの妥当性を確認し、不正な場合は pg_fatal で終了する。 3. **情報抽出**: パス名、サイズ、モード、UID、GIDをヘッダーから読み取る。 4. **型判定**: TAR_OFFSET_TYPEFLAG に基づき、ファイル、ディレクトリ、シンボリックリンクのフラグを設定する。PAX拡張フォーマットが検出された場合は pg_fatal で終了する。 5. **パディング計算**: tarPaddingBytesRequired を用いて、後続のデータに必要なパディングバイト数を算出する。 6. **転送**: astreamer_content を呼び出し、抽出した member 情報とヘッダーブロックを次段のストリーマーへ渡す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/astreamer_tar.c 内のパーサー処理（astreamer_tar_parser_content 等から呼び出される）。
  • **呼び出し先**: isValidTarHeader, read_tar_number, path_is_safe_for_extraction, tarPaddingBytesRequired, astreamer_content。

関連データ構造

  • astreamer_tar_parser: パーサーの状態を保持する構造体。
  • astreamer_member: 解析されたメンバーのメタデータを保持する構造体。

失敗モードと境界条件

  • **致命的エラー (pg_fatal)**:
  • ヘッダーが不正な場合。
  • パス名が空の場合。
  • パス名が安全でない場合。
  • PAX拡張フォーマットが検出された場合。
  • **境界条件**: 全バイトがNULのブロックは、アーカイブの正常な終端として処理される。

保守担当者向けメモ

  • 本関数は astreamer インターフェースの一部として動作します。
  • pax 拡張フォーマットは現在サポート対象外です。
  • ヘッダーの検証には pgtar.h で定義された定数と関数が使用されています。
