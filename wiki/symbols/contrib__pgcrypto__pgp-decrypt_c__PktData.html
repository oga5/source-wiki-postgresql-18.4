PktData

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/pgp-decrypt.c:168

目的

OpenPGP パケットのメタデータ（タイプおよび長さ）を保持し、復号処理中のパケット解析状態を管理するために使用されます。

シグネチャまたは形状

struct PktData
{
    int type;
    int len;
};

入力と出力

  • 本構造体は、pgp_create_pkt_reader 等のフィルタ処理において、パケットの属性を保持するコンテキストとして機能します。
  • palloc を介してメモリが割り当てられ、パケット解析の結果が各フィールドに格納されます。

前提条件と不変条件

  • type および len は、PGP パケットヘッダーの解析結果に基づいて設定されます。
  • 構造体のインスタンスは、パケット読み取りフィルタの priv 領域として保持されることが想定されています。

アルゴリズムまたは振る舞い

  • パケット解析処理において、ヘッダーから読み取ったパケットタグ（type）とデータ長（len）を保持します。
  • 復号パイプライン内でのパケット走査時に、現在のパケットの属性を特定するために参照されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/pgp-decrypt.c 内のパケット読み取りフィルタ関連関数（pktreader_pull 等）から参照されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • PullFilter: 本構造体が priv フィールドを通じて関連付けられる、ストリーム処理用のフィルタ構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時: palloc が失敗した場合、PostgreSQL のメモリ管理機構によりエラーがスローされます。
  • 不正なパケット長: パケットヘッダー解析時に不正な長さが検出された場合、本構造体に格納される値も無効なものとなります。

保守担当者向けメモ

  • 本構造体は pgp-decrypt.c 内のローカルなパケット解析状態を保持するシンプルなコンテナです。
  • 構造体の拡張が必要な場合は、パケット解析ロジック（pgp_parse_pkt_hdr 等）との整合性に注意してください。
