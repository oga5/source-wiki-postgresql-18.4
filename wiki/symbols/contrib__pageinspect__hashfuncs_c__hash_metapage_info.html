hash_metapage_info

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/hashfuncs.c:514

目的

ハッシュインデックスのメタページの内容を解析し、その内部パラメータ（マジックナンバー、バージョン、バケット数、ビットマップ情報等）を SQL で読み取り可能な形式で抽出する。

シグネチャまたは形状

Datum hash_metapage_info(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: bytea 型の生ページデータ（PG_GETARG_BYTEA_P(0)）。
  • **出力**: メタページ内の各フィールドを格納した複合型（HeapTuple）。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要（superuser() チェック）。
  • 入力されたページは verify_hash_page によって LH_META_PAGE（メタページ）であることが検証されなければならない。
  • 戻り値の型は複合型（TYPEFUNC_COMPOSITE）である必要がある。

アルゴリズムまたは振る舞い

1. スーパーユーザー権限を確認し、失敗した場合はエラーを投げる。 2. verify_hash_page を呼び出し、入力ページが有効なハッシュメタページであることを確認する。 3. get_call_result_type を使用して戻り値のタプル記述子を取得・準備する。 4. HashPageGetMeta を介してページから HashMetaPageData 構造体を取得する。 5. 構造体内の各フィールド（hashm_magic, hashm_version, hashm_ntuples 等）を Datum 配列に格納する。 6. hashm_spares および hashm_mapp 配列を construct_array_builtin を用いて PostgreSQL の配列型に変換する。 7. heap_form_tuple を用いて結果をタプル化し、返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（SELECT * FROM hash_metapage_info(...)）。
  • **呼び出し先**: verify_hash_page（検証用）、HashPageGetMeta（メタデータ取得用）、construct_array_builtin（配列構築用）、heap_form_tuple（タプル生成用）。

関連データ構造

  • HashMetaPageData: ハッシュインデックスのメタページ構造体。
  • HashPageStat: ページ統計情報（本関数と関連する解析用構造体）。

失敗モードと境界条件

  • スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE エラーが発生する。
  • ページがメタページでない場合、verify_hash_page によりエラーが発生する。
  • 戻り値の型が複合型でない場合、elog(ERROR) が発生する。

保守担当者向けメモ

  • 本関数は pageinspect モジュールの一部であり、物理的なインデックス構造に直接依存している。access/hash.h で定義されている HashMetaPageData 構造体の変更時には、本関数のフィールドマッピングも更新が必要となる。
