inet_gist_compress

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_gist.c:541

目的

GiSTインデックスへの挿入時に、INET 型の値をインデックス格納用の GistInetKey 構造体に変換します。

シグネチャまたは形状

Datum inet_gist_compress(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体を受け取ります。
  • **出力**: 変換後の GISTENTRY を保持する Datum を返します。

前提条件と不変条件

  • entry->leafkey が真の場合のみ、INET から GistInetKey への変換処理が行われます。
  • entry->key が NULL の場合は、空の GISTENTRY が初期化されます。
  • GistInetKey は varlena 形式に従い、SET_GK_VARSIZE(r) によってサイズが設定されます。

アルゴリズムまたは振る舞い

1. **リーフノード判定**: entry->leafkey が真であれば変換処理を実行し、偽であれば元の entry をそのまま返します。 2. **データ変換**:

  • DatumGetInetPP を用いて入力値を inet 型として取得します。
  • GistInetKey 構造体をメモリ確保し、以下の値をコピーします。
  • アドレスファミリー (ip_family)
  • 最小ネットマスク長 (ip_bits)
  • 共通プレフィックス長 (gk_ip_maxbits を使用)
  • アドレス本体 (ip_addr)

    • 3. **初期化**: gistentryinit を使用して、変換後のキーを含む新しい GISTENTRY を作成します。

呼び出し元と呼び出し先

  • **呼び出し元**: GiSTインデックスの挿入処理（PostgreSQLのGiSTフレームワークから呼び出されます）。
  • **呼び出し先**: palloc, palloc0, DatumGetInetPP, ip_family, ip_bits, gk_ip_maxbits, ip_addr, gk_ip_addrsize, SET_GK_VARSIZE, gistentryinit。

関連データ構造

  • GistInetKey: ネットワークアドレスのインデックス表現。
  • GISTENTRY: GiSTインデックスの各エントリを保持する構造体。
  • inet: PostgreSQLのネットワーク型。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • entry->key が NULL の場合、変換を行わず空の GISTENTRY を返します。

保守担当者向けメモ

  • 本関数は INET 型をインデックス専用の GistInetKey に変換する役割を担います。
  • GistInetKey は varlena 形式ですが、通常の INET 型とは内部構造が異なるため、インデックスから値を取り出す際は inet_gist_fetch を経由する必要があります。
