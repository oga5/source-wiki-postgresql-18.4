get_decomposed_size

定義

  • 種別: function
  • 定義箇所: src/common/unicode_norm.c:159

目的

Unicode 文字を正規化する際に、分解後の文字数（バッファサイズ）を算出する。

シグネチャまたは形状

static int get_decomposed_size(pg_wchar code, bool compat)

入力と出力

  • **入力**:
  • code: 対象の Unicode コードポイント（pg_wchar）。
  • compat: 互換分解（Compatibility Decomposition）を含めるかどうかを示すフラグ。
  • **出力**:
  • 分解後のコードポイント数（int）。

前提条件と不変条件

  • ハングル文字（SBASE から SBASE + SCOUNT の範囲）は、テーブルを参照せず計算によってサイズが決定される。
  • entry が NULL の場合、または分解情報を持たない場合はサイズ 1 を返す。

アルゴリズムまたは振る舞い

1. **ハングル文字の判定**: 対象がハングル文字の場合、計算により 2 または 3 を返す。 2. **テーブル検索**: get_code_entry を使用して文字の分解情報を取得する。 3. **分解の再帰的カウント**:

  • 分解情報が存在する場合、get_code_decomposition で分解コードのリストを取得する。
  • 各分解コードに対して再帰的に get_decomposed_size を呼び出し、合計サイズを算出する。
  • 互換分解が不要な場合で、かつ対象が互換分解のみを持つ場合は 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/unicode_norm.c 内の正規化処理フロー（unicode_normalize 等の内部処理）。
  • **呼び出し先**:
  • get_code_entry
  • get_code_decomposition
  • get_decomposed_size（再帰呼び出し）

関連データ構造

  • pg_unicode_decomposition: 文字の分解情報を保持する構造体。

失敗モードと境界条件

  • entry が NULL の場合は、分解不可としてサイズ 1 を返す安全な設計となっている。
  • 互換分解フラグの指定により、正規分解のみか互換分解を含むかの挙動が分岐する。

保守担当者向けメモ

  • 本関数は再帰的に呼び出されるため、分解の深さに応じたスタック消費に注意すること。
  • ハングル文字の分解ロジックは Unicode TR15 に準拠している。
