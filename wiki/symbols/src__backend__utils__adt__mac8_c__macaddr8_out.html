macaddr8_out

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mac8.c:233

目的

macaddr8 型の内部表現を、標準的なコロン区切りの16進数文字列（EUI-64形式）に変換して出力します。

シグネチャまたは形状

Datum macaddr8_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_MACADDR8_P(0) を介して取得される macaddr8 構造体へのポインタ。
  • **出力**: 8バイトのMACアドレスを "%02x:%02x:%02x:%02x:%02x:%02x:%02x:%02x" 形式でフォーマットした cstring（char *）。

前提条件と不変条件

  • 入力される macaddr8 構造体は、有効な8バイトのデータを含んでいる必要があります。
  • palloc(32) によりメモリが確保されるため、PostgreSQLのメモリコンテキスト内で実行される必要があります。

アルゴリズムまたは振る舞い

1. palloc(32) を呼び出し、出力文字列用のメモリ領域を確保します。 2. snprintf を使用し、構造体内の8つのメンバ（a から h）を16進数2桁ずつコロンで連結して書き込みます。 3. PG_RETURN_CSTRING マクロを介して、生成された文字列を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報には、本関数を直接呼び出している具体的な関数名は記載されていません。
  • **呼び出し先**:
  • PG_GETARG_MACADDR8_P: 引数取得用マクロ。
  • palloc: メモリ確保用関数。
  • snprintf: 文字列フォーマット用標準ライブラリ関数。
  • PG_RETURN_CSTRING: 戻り値変換用マクロ。

関連データ構造

  • macaddr8: 8バイトのMACアドレスを保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • snprintf のバッファサイズ（32バイト）は、8バイトのMACアドレスをコロン区切りで表現するのに十分な長さです。

保守担当者向けメモ

  • 本関数は macaddr8 型の出力用関数であり、SQL層でのデータ表示に使用されます。
  • 構造体のメンバレイアウト（a〜h）に変更を加える場合は、フォーマット文字列との整合性に注意してください。
