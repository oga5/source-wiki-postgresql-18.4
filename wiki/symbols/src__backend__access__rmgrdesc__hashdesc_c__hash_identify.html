hash_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/hashdesc.c:125

目的

WAL（Write-Ahead Log）レコードの info フィールドに基づき、ハッシュインデックス操作の種類を人間が判読可能な文字列として識別します。

シグネチャまたは形状

const char *hash_identify(uint8 info);

入力と出力

  • **入力**: info (uint8型) - WALレコードの info フィールド。
  • **出力**: 操作名を表す文字列定数（例: "INSERT"）。該当する操作が見つからない場合は NULL を返します。

前提条件と不変条件

  • 入力された info は、XLR_INFO_MASK を用いてフラグ部分が除外された状態で評価されます。
  • 本関数は表示用の記述子ルーチンであり、リカバリ等のロジック実行には関与しません。

アルゴリズムまたは振る舞い

1. 引数 info に対して ~XLR_INFO_MASK を適用し、操作タイプを抽出します。 2. switch 文を用いて、抽出された値に対応するハッシュインデックス操作（XLOG_HASH_INIT_META_PAGE から XLOG_HASH_VACUUM_ONE_PAGE まで）を判定します。 3. 一致した操作名に対応する文字列リテラルを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h で定義されている RmgrData 構造体の rm_identify フィールドとして登録されており、WAL解析ツール（pg_waldump 等）から間接的に呼び出されます。
  • **呼び出し先**: 外部関数は呼び出しません。

関連データ構造

  • XLOG_HASH_* 定数: access/hash_xlog.h で定義されるハッシュインデックスのWALレコードタイプ。

失敗モードと境界条件

  • 定義されていない info 値が渡された場合、id は初期値の NULL のまま返されます。

保守担当者向けメモ

  • 新しいハッシュインデックスのWAL操作を追加した際は、必ず本関数の switch 文に新しい case を追加してください。
  • info フィールドのマスク処理（& ~XLR_INFO_MASK）は、操作タイプを正しく特定するために必須です。
