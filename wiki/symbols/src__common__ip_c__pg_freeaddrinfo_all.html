pg_freeaddrinfo_all

定義

  • 種別: function
  • 定義箇所: src/common/ip.c:81

目的

pg_getaddrinfo_all によって生成された addrinfo 構造体のリストを、その生成元（標準 API または Unix ドメインソケット用エミュレーション）に応じて適切に解放する。

シグネチャまたは形状

void pg_freeaddrinfo_all(int hint_ai_family, struct addrinfo *ai)

入力と出力

  • hint_ai_family: addrinfo の生成時に使用されたアドレスファミリー（AF_UNIX か否かの判定に使用）。
  • ai: 解放対象の addrinfo 構造体へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • hint_ai_family が AF_UNIX の場合、ai は getaddrinfo_unix によって確保されたメモリ構造である必要がある。
  • hint_ai_family が AF_UNIX 以外の場合、ai は標準の getaddrinfo によって確保されたメモリ構造である必要がある。

アルゴリズムまたは振る舞い

1. hint_ai_family が AF_UNIX であるかを確認する。 2. AF_UNIX の場合:

  • リスト内の各 addrinfo 要素を反復処理する。
  • 各要素の ai_addr（sockaddr_un）を free する。
  • addrinfo 構造体自体を free する。

    • 3. AF_UNIX 以外の場合:

  • ai が NULL でなければ、標準ライブラリの freeaddrinfo(ai) を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth.c (複数箇所)
  • src/backend/libpq/hba.c (複数箇所)
  • **呼び出し先**:
  • free (標準ライブラリ)
  • freeaddrinfo (標準ライブラリ)

関連データ構造

  • struct addrinfo: ネットワークアドレス情報を保持する標準構造体。

失敗モードと境界条件

  • ai が NULL の場合、関数は何もせず安全に終了する。
  • hint_ai_family に誤った値を渡した場合、メモリリークまたは不正なメモリ解放が発生する可能性がある。

保守担当者向けメモ

  • addrinfo 構造体自体からは、それが標準 API 由来か getaddrinfo_unix 由来かを判別できないため、呼び出し側で hint_ai_family を正しく管理・伝達することが必須である。
