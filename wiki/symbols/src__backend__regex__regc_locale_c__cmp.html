cmp

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_locale.c:742

目的

2つの chr 型配列を、指定された長さ分だけメモリレベルで比較し、一致するかどうかを判定します。

シグネチャまたは形状

static int cmp(const chr *x, const chr *y, size_t len)

入力と出力

  • **入力**:
  • x: 比較対象の文字列（chr 型ポインタ）。
  • y: 比較対象の文字列（chr 型ポインタ）。
  • len: 比較する要素数（size_t）。
  • **出力**:
  • int: 両者が完全に一致する場合は 0 を返し、不一致の場合は非ゼロの値を返します。

前提条件と不変条件

  • x および y は、少なくとも len 個の chr 要素を保持する有効なメモリ領域を指している必要があります。
  • len は比較する chr 型の個数であり、バイト数ではない点に注意が必要です。

アルゴリズムまたは振る舞い

1. VS() マクロ（void * へのキャスト）を使用して、x と y を汎用ポインタに変換します。 2. memcmp を呼び出し、len * sizeof(chr) バイト分のメモリ領域を直接比較します。 3. memcmp の結果をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは、具体的な呼び出し元は特定できません。
  • **呼び出し先**: memcmp（標準ライブラリ関数）。

関連データ構造

  • chr: 正規表現エンジン内で使用される文字型。

失敗モードと境界条件

  • len が 0 の場合、memcmp の仕様に従い、常に 0（一致）が返されます。
  • x または y が NULL である場合、memcmp の動作に依存して未定義の挙動（セグメンテーション違反等）を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数はバックリファレンスの照合時に使用される比較ルーチンです。
  • memcmp を使用しているため、ロケール依存の照合ではなく、バイナリレベルでの完全一致判定を行います。大文字小文字を区別しない比較が必要な場合は、代わりに casecmp を使用してください。
