SnapBuildOnDisk

定義

  • **種別**: struct
  • **定義箇所**: src/include/replication/snapbuild_internal.h:175

目的

論理デコーディングにおけるスナップショット構築状態（SnapBuild）をディスクへ永続化（シリアライズ）するためのデータ構造です。チェックサムによる整合性保護とバージョン管理機能を提供します。

シグネチャまたは形状

typedef struct SnapBuildOnDisk
{
    uint32      magic;
    pg_crc32c   checksum;
    uint32      version;
    uint32      length;
    SnapBuild   builder;
    /* 可変長の TransactionId 配列が続く */
} SnapBuildOnDisk;

入力と出力

  • **入力**: SnapBuild 構造体の内容および動的なトランザクションIDリスト。
  • **出力**: ディスク書き込み可能なシリアライズ形式のデータブロック。

前提条件と不変条件

  • 構造体の先頭部分（magic, checksum）はバージョン非依存である必要があります。
  • length フィールドは、固定サイズのヘッダー部分を除いたディスク上のデータサイズを保持します。
  • 構造体末尾には、可変量の TransactionId が続くメモリレイアウトを前提としています。

アルゴリズムまたは振る舞い

1. **シリアライズ**: SnapBuild 構造体と関連するトランザクションIDをメモリ上に配置し、pg_crc32c を用いてチェックサムを計算・格納します。 2. **整合性チェック**: 復元時に magic 番号と checksum を検証し、データ破損がないことを確認します。 3. **復元**: SnapBuildRestoreSnapshot() 関数により、ディスク上のデータからメモリ上の SnapBuild 状態が再構築されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/snapbuild.c 内のシリアライズおよび復元ロジック。
  • **呼び出し先**: contrib/pg_logicalinspect/pg_logicalinspect.c（検査用ツール）。
  • **補足**: 呼び出し元・呼び出し先の詳細なコールグラフは、提供されたソースの範囲内では限定的な参照のみ確認可能です。

関連データ構造

  • SnapBuild: 本構造体内部に保持されるメインの状態管理構造体。
  • TransactionId: 構造体末尾に可変長配列として格納される識別子。
  • pg_crc32c: 整合性チェックに使用される型。

失敗モードと境界条件

  • **データ破損**: checksum の不一致により、復元プロセスが失敗する可能性があります。
  • **バージョン不整合**: version フィールドが現在のシステムと互換性がない場合、復元が拒否される可能性があります。
  • **メモリレイアウト**: SnapBuildOnDisk 構造体自体のサイズ変更やフィールド順序の変更は、既存のディスク上のデータとの互換性を損なうため注意が必要です。

保守担当者向けメモ

  • 本構造体を変更する場合は、src/include/replication/snapbuild_internal.h 内の関連マクロも更新する必要があります。
  • SnapBuild のシリアライズ形式は論理デコーディングの正確性に直結するため、変更時には SnapBuildRestoreSnapshot のロジックと整合性を確認してください。
