NumericLong

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/numeric.c:142

目的

PostgreSQL の NUMERIC 型において、4バイトのヘッダーを持つ「ロング形式」の数値を格納するために使用されます。

シグネチャまたは形状

struct NumericLong
{
	uint16		n_sign_dscale;	/* 符号および表示スケール */
	int16		n_weight;		/* 最初の桁の重み */
	NumericDigit n_data[FLEXIBLE_ARRAY_MEMBER]; /* 数値データ本体 */
};

入力と出力

  • 本構造体は、Numeric 型のオンディスク表現の一部として機能します。
  • n_sign_dscale には符号情報と表示スケールがパックされ、n_weight には数値の重みが格納されます。
  • n_data は可変長配列であり、実際の数値データ（NumericDigit）を保持します。

前提条件と不変条件

  • NumericLong は、4バイトのヘッダーを持つ形式として定義されています。
  • NumericShort 形式とは異なり、より大きな数値や複雑な数値を表現するために使用されます。

アルゴリズムまたは振る舞い

  • NUMERIC 型の算術演算や格納において、数値の規模に応じて NumericShort と使い分けられます。
  • 内部的には、NumericVar 構造体へ展開して算術演算を行い、結果を再び NumericLong 形式にパックして保存するフローが一般的です。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/numeric.c 内の数値処理関数群（numeric_in や算術演算関数など）。
  • 呼び出し先: 具体的な関数呼び出し関係は、提供されたソース excerpt には含まれていません。

関連データ構造

  • NumericData: NUMERIC 型の全体的なオンディスク表現。
  • NumericShort: 小さな数値を格納するための代替構造体。
  • NumericVar: 算術演算時に使用されるメモリ上の作業用構造体。
  • NumericDigit: 数値の各桁を保持する型。

失敗モードと境界条件

  • n_data は可変長配列であるため、メモリ割り当ておよびアクセス時には適切なサイズ計算が必要です。
  • 4バイトヘッダーの制約（残りの14ビットの利用など）に関する境界条件が存在します。

保守担当者向けメモ

  • NumericLong は NumericShort と対になる構造体です。修正を行う際は、src/include/utils/numeric.h で定義されている関連マクロや、numeric.c 内のパック・アンパック処理との整合性に注意してください。
