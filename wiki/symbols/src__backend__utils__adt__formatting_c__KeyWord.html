KeyWord

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/formatting.c:144

目的

TO_CHAR や TO_DATE 等の関数で使用されるフォーマット指定子（例: YYYY, MM, 9, 0）の属性を定義し、解析および変換処理を制御するために使用されます。

シグネチャまたは形状

typedef struct
{
	const char *name;
	int			len;
	int			id;
	bool		is_digit;
	FromCharDateMode date_mode;
} KeyWord;

入力と出力

本構造体はデータ型ではなく定義体であるため、直接的な入力・出力はありません。各フィールドは以下の役割を持ちます。

  • name: 指定子の文字列（例: "YYYY"）。
  • len: 指定子の長さ。
  • id: 内部識別子。
  • is_digit: 数値指定子かどうかのフラグ。
  • date_mode: 日時変換時のモード（FromCharDateMode 型）。

前提条件と不変条件

  • DCH_keywords および NUM_keywords 配列内で使用されます。
  • フォーマット文字列の解析時、これらの配列はアルファベット順にソートされている必要があります。

アルゴリズムまたは振る舞い

  • parse_format() 関数がフォーマット文字列を走査する際、この構造体の配列を参照してトークンを識別します。
  • index_seq_search() 関数により、特定の文字列に対応する KeyWord が検索されます。

呼び出し元と呼び出し先

  • **呼び出し元**: parse_format() がフォーマット文字列の解析時に参照します。また、dump_index() がデバッグや構造のダンプ時に参照します。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • FormatNode: フォーマット文字列の解析結果を保持するツリー構造。
  • FromCharDateMode: 日時変換時のモードを定義する列挙型。
  • DCH_keywords / NUM_keywords: KeyWord 型の静的配列。

失敗モードと境界条件

  • 指定された文字列が KeyWord 配列内に存在しない場合、検索関数は一致するエントリを見つけられません。
  • フォーマット文字列がキャッシュサイズを超える場合、解析処理の効率が低下する可能性があります。

保守担当者向けメモ

  • 新しいフォーマット指定子を追加する場合、DCH_keywords または NUM_keywords テーブルと、対応する DCH_poz / NUM_poz 列挙型を更新する必要があります。
  • KeyWord フィールドのソート順序（アルファベット順）は、検索アルゴリズムの正当性に直結するため厳守してください。
