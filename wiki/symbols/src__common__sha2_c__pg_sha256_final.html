pg_sha256_final

定義

  • 種別: function
  • 定義箇所: src/common/sha2.c:576

目的

SHA-256 ハッシュ計算の最終化を行い、パディング処理の完了とダイジェストの出力、およびコンテキストの安全な破棄を行う。

シグネチャまたは形状

void pg_sha256_final(pg_sha256_ctx *context, uint8 *digest)

入力と出力

  • context: SHA-256 計算状態を保持する pg_sha256_ctx 構造体へのポインタ。
  • digest: 計算結果を格納するバッファへのポインタ。NULL の場合、ダイジェストの書き出しはスキップされる。

前提条件と不変条件

  • context は初期化済みであり、データが入力されている必要がある。
  • digest が NULL でない場合、PG_SHA256_DIGEST_LENGTH 分の書き込み領域が確保されていること。
  • 実行終了後、context は memset によりゼロクリアされる。

アルゴリズムまたは振る舞い

1. digest が NULL でない場合:

  • SHA256_Last(context) を呼び出し、パディングと最終ブロックの変換を実行する。
  • WORDS_BIGENDIAN が未定義（リトルエンディアン環境）の場合、REVERSE32 マクロを用いて状態変数のバイト順序をホスト順に変換する。
  • memcpy を使用して context->state から digest へ結果をコピーする。

    • 2. 最後に memset を実行し、context 全体をゼロクリアして機密情報を消去する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/cryptohash.c:209 で呼び出されていることが確認されている。
  • **呼び出し先**: SHA256_Last を呼び出す。

関連データ構造

  • pg_sha256_ctx: SHA-256 の計算状態を保持する構造体（sha2_int.h で定義）。

失敗モードと境界条件

  • digest が NULL の場合、ハッシュ計算の最終化（パディングと変換）は行われるが、結果のコピーはスキップされる。
  • いずれの場合も、関数終了時には context がゼロクリアされるため、再利用は不可となる。

保守担当者向けメモ

  • 本関数は src/common/sha2.c 内の他の SHA-2 ファミリー関数と同様、Aaron D. Gifford 氏の実装に基づいている。
  • エンディアン変換は WORDS_BIGENDIAN の定義に依存するため、移植時には注意が必要である。
