appendBinaryStringInfo

定義

  • 種別: function
  • 定義箇所: src/common/stringinfo.c:280

目的

StringInfo バッファに対して、ヌル文字を含む可能性のある任意のバイナリデータを追加する。

シグネチャまたは形状

void appendBinaryStringInfo(StringInfo str, const void *data, int datalen)

入力と出力

  • str: 操作対象の StringInfo 構造体へのポインタ。
  • data: 追加するバイナリデータの先頭アドレス。
  • datalen: 追加するデータのバイト数。
  • 戻り値: なし。

前提条件と不変条件

  • str は NULL であってはならない（Assert により検証）。
  • str が指すバッファは、追加後のデータ長を保持するのに十分なメモリを確保するか、enlargeStringInfo によって自動拡張される必要がある。
  • 実行後、str->data の末尾には常にヌル文字（\0）が配置される。

アルゴリズムまたは振る舞い

1. str が NULL でないことを確認する。 2. enlargeStringInfo を呼び出し、datalen 分のデータを追加するために必要なバッファ容量を確保する。 3. memcpy を使用して、現在のバッファ末尾（str->data + str->len）に data をコピーする。 4. str->len を datalen 分増加させる。 5. バッファの新しい末尾（str->data[str->len]）にヌル文字を書き込み、C 言語の文字列としての安全性を維持する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/hstore/hstore_io.c
  • contrib/postgres_fdw/deparse.c
  • contrib/test_decoding/test_decoding.c
  • contrib/unaccent/unaccent.c
  • src/backend/access/transam/parallel.c
  • 呼び出し先:
  • enlargeStringInfo (内部でバッファ拡張を実行)
  • memcpy (標準ライブラリ)

関連データ構造

  • StringInfoData: data（バッファ）、len（現在長）、maxlen（容量）を保持する構造体。

失敗モードと境界条件

  • str が NULL の場合、アサーション違反により停止する。
  • バッファの拡張（enlargeStringInfo）がメモリ不足で失敗した場合、palloc または malloc の挙動に従いエラーが発生する。

保守担当者向けメモ

  • 本関数はヌル終端を強制的に維持する。バイナリデータ専用でヌル終端が不要な場合は、appendBinaryStringInfoNT の使用を検討すること。
  • StringInfo が読み取り専用（maxlen == 0）の場合、enlargeStringInfo 内で Assert が発生するため注意が必要。
