ReplaceVarsNoMatchOption

定義

  • 種別: enum
  • 定義箇所: src/include/rewrite/rewriteManip.h:37

目的

クエリ書き換え処理において、変数（Var ノード）の置換対象が見つからなかった場合の挙動を制御するためのオプションを定義します。

シグネチャまたは形状

typedef enum ReplaceVarsNoMatchOption
{
	REPLACEVARS_REPORT_ERROR,	/* エラーをスローする */
	REPLACEVARS_CHANGE_VARNO,	/* Var の varno のみを変更する */
	REPLACEVARS_SUBSTITUTE_NULL,	/* NULL 定数に置換する */
} ReplaceVarsNoMatchOption;

入力と出力

  • 本シンボル自体は列挙型であり、関数への引数として渡されることで、置換ロジックの分岐条件として機能します。

前提条件と不変条件

  • REPLACEVARS_REPORT_ERROR が選択された場合、一致するものが見つからないと処理はエラーで中断されます。
  • REPLACEVARS_SUBSTITUTE_NULL が選択された場合、該当する Var は NULL を表す Const ノードに置き換えられます。

アルゴリズムまたは振る舞い

  • rewriteManip.c 内の置換関数において、この列挙値に基づき以下のいずれかの処理が実行されます。
  • REPLACEVARS_REPORT_ERROR: 一致する RTE がない場合にエラーを発生させる。
  • REPLACEVARS_CHANGE_VARNO: Var ノードの varno 属性のみを更新する。
  • REPLACEVARS_SUBSTITUTE_NULL: 一致しない Var を NULL 値を持つ Const ノードに置換する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/rewrite/rewriteManip.c 内の変数置換関連関数（replace_rte_variables 等）。
  • 呼び出し先: 本シンボル自体はデータ型であり、関数を呼び出すことはありません。

関連データ構造

  • replace_rte_variables_context: 変数置換のコンテキストを保持する構造体であり、本オプションをメンバとして保持します。

失敗モードと境界条件

  • REPLACEVARS_REPORT_ERROR を指定した場合、一致しない Var が存在するとプログラムはエラーをスローして終了します。

保守担当者向けメモ

  • 新しい置換戦略が必要になった場合は、この列挙型に値を追加し、rewriteManip.c 内の該当する switch 文を更新してください。
