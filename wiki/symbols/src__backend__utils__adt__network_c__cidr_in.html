cidr_in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network.c:128

目的

文字列形式のネットワークアドレスを cidr 型の内部表現（inet 構造体）に変換します。

シグネチャまたは形状

Datum cidr_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る cstring 型の文字列（IP アドレスおよびオプションのマスク長）。
  • **出力**: cidr 型の内部表現を保持する inet 構造体へのポインタを Datum として返します。

前提条件と不変条件

  • 入力文字列は有効な IP アドレス形式である必要があります。
  • cidr 型の不変条件として、マスク長より右側のビットは常に 0 である必要があります。この検証は内部で呼び出される network_in 内の addressOK によって保証されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_CSTRING(0) を使用して入力文字列を取得します。 2. network_in 関数を呼び出し、第 2 引数に true を渡すことで cidr モードとして処理を実行します。 3. network_in が返す inet 構造体を PG_RETURN_INET_P マクロでラップして返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: network_in（src/backend/utils/adt/network.c 内）。
  • **呼び出し元**: ソースコード内での直接的な呼び出し元は確認されていませんが、PostgreSQL の型システムを通じて SQL レベルの入力変換時に呼び出されます。

関連データ構造

  • inet: ネットワーク型を保持する内部構造体。
  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約に従う引数コンテナ。

失敗モードと境界条件

  • 不正な IP アドレス形式や、マスク長が範囲外である場合、network_in 内でエラーが発生し、トランザクションがアボートされます。
  • cidr 型の制約（マスク長より右側のビットが 0 でない）に違反する場合、addressOK による検証でエラーとなります。

保守担当者向けメモ

  • 本関数は cidr 型専用のラッパーであり、実質的なパース処理は network_in に依存しています。ロジックの変更が必要な場合は network_in を参照してください。
