FreeTupleDesc

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tupdesc.c:501

目的

TupleDesc 構造体およびその内部で保持されている制約情報（TupleConstr）に関連するメモリを解放します。

シグネチャまたは形状

void FreeTupleDesc(TupleDesc tupdesc)

入力と出力

  • **入力**: tupdesc (解放対象の TupleDesc へのポインタ)
  • **出力**: なし

前提条件と不変条件

  • tupdesc->tdrefcount が 0 以下であることを前提とします（Assert による検証）。
  • 参照カウントが正の値を保持している場合、この関数を呼び出すことは不適切です。

アルゴリズムまたは振る舞い

1. tupdesc->tdrefcount が 0 以下であることを確認します。 2. tupdesc->constr が存在する場合、以下の順序で内部メモリを解放します。

  • defval（デフォルト値）: 各 adbin 文字列および配列自体を解放します。
  • missing（欠損値）: am_present が真かつ attbyval が偽である場合、各 am_value を解放し、配列自体を解放します。
  • check（チェック制約）: 各 ccname および ccbin 文字列、配列自体を解放します。
  • 最後に tupdesc->constr 構造体自体を解放します。

    • 3. 最後に tupdesc 構造体自体を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/tupdesc.c 内の複数箇所
  • src/backend/access/spgist/spgscan.c
  • src/backend/catalog/heap.c
  • src/backend/executor/execSRF.c
  • src/backend/replication/pgoutput/pgoutput.c
  • src/backend/utils/adt/expandedrecord.c
  • src/backend/utils/adt/jsonfuncs.c
  • **呼び出し先**: pfree（メモリ解放関数）

関連データ構造

  • TupleDesc: 解放対象の主要構造体。
  • TupleConstr: tupdesc->constr として保持される制約情報。

失敗モードと境界条件

  • tupdesc が NULL の場合、あるいは tdrefcount が 0 より大きい場合に Assert が有効であればプログラムが停止します。
  • 内部の各ポインタが NULL の場合はスキップされます。

保守担当者向けメモ

  • 本関数は TupleDesc の所有権が完全に放棄されたことを前提としています。参照カウント管理については IncrTupleDescRefCount / DecrTupleDescRefCount を参照してください。
  • TupleConstr 内のメモリレイアウトを変更した場合は、本関数の解放ロジックも同期して更新する必要があります。
