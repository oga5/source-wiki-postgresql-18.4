drandom

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pseudorandomfuncs.c:83

目的

SQL関数 random() の実体として、[0.0, 1.0) の範囲で一様分布する疑似乱数を生成し、PostgreSQLの Datum 型として返却します。

シグネチャまたは形状

Datum drandom(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL層からの引数）。
  • **出力**: float8 型の乱数値（Datum 形式）。

前提条件と不変条件

  • 内部状態 prng_state がプロセス単位で保持されており、初回呼び出し時に initialize_prng() によって初期化される必要があります。
  • 戻り値は常に 0.0 以上 1.0 未満の範囲に収まります。

アルゴリズムまたは振る舞い

1. initialize_prng() を呼び出し、乱数生成器の状態を初期化（または既存の状態を維持）します。 2. pg_prng_double(&prng_state) を呼び出し、現在の状態に基づいて乱数を生成します。 3. 生成された float8 値を PG_RETURN_FLOAT8 マクロを介して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（random() 関数経由）。
  • **呼び出し先**:
  • initialize_prng()（状態の初期化）
  • pg_prng_double()（乱数生成のコアロジック）

関連データ構造

  • prng_state: プロセス内で共有される乱数生成器の状態を保持する構造体。

失敗モードと境界条件

  • initialize_prng() が失敗した場合、またはシード設定が不適切な場合、乱数生成器の初期化が影響を受ける可能性があります。
  • 戻り値の範囲は [0.0, 1.0) であり、1.0 は含まれません。

保守担当者向けメモ

  • 本関数はプロセス単位の静的状態に依存しています。並列クエリ実行時やセッション間での乱数生成器の同期は行われない点に注意してください。
  • 乱数生成アルゴリズムの変更が必要な場合は、common/pg_prng.h の定義を確認してください。
