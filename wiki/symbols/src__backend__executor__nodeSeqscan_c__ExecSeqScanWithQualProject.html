ExecSeqScanWithQualProject

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSeqscan.c:170

目的

シーケンシャルスキャンにおいて、フィルタ条件（Qual）とプロジェクション（射影）の両方が必要な場合に、タプルの取得・評価・変換を一括して実行する。

シグネチャまたは形状

TupleTableSlot *ExecSeqScanWithQualProject(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（実行中のシーケンシャルスキャンノードの状態）。
  • **出力**: フィルタ条件を通過し、プロジェクションが適用された後の TupleTableSlot *。

前提条件と不変条件

  • pstate->state->es_epq_active が NULL であること（EvalPlanQualがアクティブでないこと）。
  • pstate->qual が NULL でないこと。
  • pstate->ps_ProjInfo が NULL でないこと。

アルゴリズムまたは振る舞い

1. pstate を SeqScanState にキャストする。 2. ExecScanExtended を呼び出し、以下のメソッドを渡してスキャンを実行する。

  • SeqNext: テーブルアクセスメソッドから次のタプルを取得する。
  • SeqRecheck: 必要に応じてタプルの再評価を行う。
  • pstate->qual: フィルタ条件の評価。
  • pstate->ps_ProjInfo: 取得したタプルに対するプロジェクション（射影）の適用。

    • 3. ExecScanExtended が返した TupleTableSlot をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeSeqscan.c:276 において、SeqScanState の ExecProcNode 関数ポインタとして設定される。
  • **呼び出し先**:
  • SeqNext（スキャン実行）
  • SeqRecheck（再評価）
  • ExecScanExtended（共通実行基盤）

関連データ構造

  • SeqScanState: シーケンシャルスキャンの実行状態を保持する構造体。
  • PlanState: 実行計画ノードの共通状態。
  • TupleTableSlot: タプルデータを保持するスロット。

失敗モードと境界条件

  • Assert 文により、条件（Qual）やプロジェクション情報が欠如している場合はアボートする。
  • テーブルアクセスメソッド（SeqNext）がタプルを返さない場合、ExecScanExtended の挙動に従い NULL または空のスロットが返される。

保守担当者向けメモ

  • 本関数は、最適化パスの一つとして ExecInitSeqScan で動的に割り当てられる。
  • ExecScanExtended を利用しているため、スキャンノード共通の実行ロジック（フィルタリングや射影の適用順序など）に依存している。
