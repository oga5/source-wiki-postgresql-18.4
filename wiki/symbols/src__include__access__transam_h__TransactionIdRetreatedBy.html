TransactionIdRetreatedBy

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/access/transam.h:321

目的

指定されたトランザクションID（XID）から指定された量を減算し、結果が有効な通常トランザクションIDの範囲を下回る場合に、循環を考慮して調整された値を返す。

シグネチャまたは形状

static inline TransactionId TransactionIdRetreatedBy(TransactionId xid, uint32 amount)

入力と出力

  • **入力**:
  • xid: 減算の起点となる TransactionId。
  • amount: 減算する量（uint32）。
  • **出力**:
  • 減算後の TransactionId。

前提条件と不変条件

  • FirstNormalTransactionId 未満の値は、予約済みID（InvalidTransactionId 等）として扱われる。
  • 本関数は、減算結果が FirstNormalTransactionId を下回った場合、その値をさらにデクリメントし続けることで、有効なXID範囲外への逸脱を補正する。

アルゴリズムまたは振る舞い

1. 引数 xid から amount を減算する。 2. 減算後の xid が FirstNormalTransactionId 未満である間、xid をデクリメントし続ける。 3. 最終的な xid を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • **呼び出し先**: 内部で FirstNormalTransactionId 定数を使用しています。

関連データ構造

  • TransactionId: 32ビットのトランザクション識別子。

失敗モードと境界条件

  • amount が非常に大きい場合、while ループによるデクリメント処理が繰り返される。
  • FirstNormalTransactionId を下回る値に対する挙動は、本関数のループ処理により、予約済みID領域内での調整が行われる。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結している。
  • 32ビットXIDの循環を考慮したロジックであるため、64ビットの FullTransactionId を扱う関数群（FullTransactionIdRetreat 等）との混同に注意すること。
