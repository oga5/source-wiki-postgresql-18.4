brinRevmapExtend

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_revmap.c:111

目的

BRINインデックスの成長に合わせてリバースレンジマップ（revmap）を物理的に拡張し、指定されたヒープブロックに対応する領域を確保します。

シグネチャまたは形状

void brinRevmapExtend(BrinRevmap *revmap, BlockNumber heapBlk)

入力と出力

  • **入力**:
  • revmap: 操作対象の BrinRevmap 構造体へのポインタ。
  • heapBlk: 拡張が必要なヒープブロック番号。
  • **出力**: なし（void型）。

前提条件と不変条件

  • revmap は初期化済みである必要があります。
  • 拡張処理後、取得されたマップブロック番号は InvalidBlockNumber ではなく、かつメタページ（BRIN_METAPAGE_BLKNO）ではなく、現在の revmap->rm_lastRevmapPage 以下である必要があります（アサーションによる検証）。

アルゴリズムまたは振る舞い

1. revmap_extend_and_get_blkno を呼び出し、ヒープブロックに対応するrevmapページを物理的に拡張してそのブロック番号を取得します。 2. 取得したブロック番号が有効な範囲内にあることをアサーションで検証します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_pageops.c (2箇所)
  • **呼び出し先**:
  • revmap_extend_and_get_blkno

関連データ構造

  • BrinRevmap: revmapの状態を保持する構造体。
  • BlockNumber: ヒープおよびインデックスのブロック位置を特定するための型。

失敗モードと境界条件

  • インデックスの拡張に伴い、既存のページと競合が発生した場合は brin_evacuate_page が内部的に呼び出され、ページ退避が行われます。
  • 物理的な拡張に失敗した場合、PostgreSQLの標準的なエラー処理メカニズムにより例外が発生します。

保守担当者向けメモ

  • 本関数はrevmapの物理的な拡張を担いますが、実際のページ操作は revmap_physical_extend を経由して行われます。
  • revmap_extend_and_get_blkno が返すブロック番号が rm_lastRevmapPage を超えないことを保証する設計となっています。
