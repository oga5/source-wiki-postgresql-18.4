transformDeleteStmt

定義

  • 種別: function
  • 定義箇所: src/backend/parser/analyze.c:552

目的

SQLの DELETE 文の抽象構文木（DeleteStmt）を、オプティマイザーが処理可能なクエリツリー（Query）へ変換する。

シグネチャまたは形状

Query *transformDeleteStmt(ParseState *pstate, DeleteStmt *stmt)

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する ParseState 構造体。
  • stmt: 構文解析器が生成した DeleteStmt 構造体。
  • **出力**: 変換済みの Query 構造体へのポインタ。

前提条件と不変条件

  • pstate は適切に初期化されている必要がある。
  • stmt->relation に対して適切なテーブルロック（ACL_DELETE）が取得される。
  • 変換過程で ParseState の p_rtable や p_joinlist が更新される。

アルゴリズムまたは振る舞い

1. Query ノードを生成し、commandType を CMD_DELETE に設定する。 2. WITH 句が存在する場合、transformWithClause を呼び出し cteList を構築する。 3. setTargetTable を呼び出し、削除対象のリレーションを特定して resultRelation に設定する。 4. WHERE CURRENT OF 句がビューに対して使用されている場合、エラーを発生させる。 5. USING 句が存在する場合、transformFromClause を呼び出し、削除対象リレーションを lateral 参照から除外した状態で解析する。 6. WHERE 句を transformWhereClause で変換し、RETURNING 句を transformReturningClause で変換する。 7. ParseState から rtable、jointree、各種フラグ（hasSubLinks 等）を Query ノードにコピーする。 8. assign_query_collations で照合順序を確定させ、必要に応じて parseCheckAggregates で集約関数の検証を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/analyze.c 内の transformStmt（推論される主要なディスパッチャー）。
  • **呼び出し先**: transformWithClause, setTargetTable, transformFromClause, transformWhereClause, transformReturningClause, assign_query_collations, parseCheckAggregates 等。

関連データ構造

  • Query: 変換後のクエリツリー。
  • DeleteStmt: 変換前の構文木。
  • ParseState: 解析中の状態管理用構造体。

失敗モードと境界条件

  • WHERE CURRENT OF をビューに対して使用した場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラー終了する。
  • 権限チェックや型解決の失敗により、下位の transform* 関数からエラーが伝播する場合がある。

保守担当者向けメモ

  • USING 句の解析前後で nsitem->p_lateral_only を切り替えることで、USING 句内のサブクエリが削除対象テーブルを直接参照できないように制御している点に注意すること。
  • assign_query_collations は必ず集約チェックの前に実行する必要がある。
