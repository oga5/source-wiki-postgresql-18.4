pltcl_interp_desc

定義

  • 種別: struct
  • 定義箇所: src/pl/tcl/pltcl.c:119

目的

ユーザーごとの Tcl インタプリタ状態を保持し、PostgreSQL 内で Tcl スクリプトを実行するためのコンテキストを管理する。

シグネチャまたは形状

typedef struct pltcl_interp_desc
{
    Oid         user_id;        /* ハッシュキー（先頭である必要がある） */
    Tcl_Interp *interp;         /* Tcl インタプリタ本体 */
    Tcl_HashTable query_hash;   /* pltcl_query_desc 構造体のハッシュテーブル */
} pltcl_interp_desc;

入力と出力

  • 入力: user_id（Oid）をキーとしてハッシュテーブルで管理される。
  • 出力: 保持する Tcl_Interp ポインタを通じて、Tcl 実行環境へのアクセスを提供する。

前提条件と不変条件

  • user_id はハッシュテーブルのキーとして機能するため、構造体の先頭メンバである必要がある。
  • ユーザーごとにインタプリタが分離されることでセキュリティが確保される。

アルゴリズムまたは振る舞い

  • pltcl_fetch_interp により、適切なインタプリタが取得または生成される。
  • pltcl_init_interp を介してインタプリタの初期化が行われる。
  • 内部の query_hash を使用して、実行されるクエリの記述子を管理する。

呼び出し元と呼び出し先

  • 呼び出し元: pltcl_fetch_interp（取得）、pltcl_init_interp（初期化）、および各ハンドラ関数内でインタプリタ状態を参照するために使用される。
  • 呼び出し先: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行わない。

関連データ構造

  • Tcl_Interp: Tcl 言語のインタプリタ状態。
  • Tcl_HashTable: query_hash として使用され、pltcl_query_desc を格納する。

失敗モードと境界条件

  • 構造体生成時のメモリ不足や、Tcl インタプリタの初期化失敗が想定される。
  • ユーザー ID ごとの分離が不完全な場合、セキュリティ上のリスクが生じる可能性がある。

保守担当者向けメモ

  • 本構造体は PostgreSQL のハッシュテーブルで管理されるため、メモリレイアウトの変更には注意が必要。
  • user_id が構造体の先頭にあるという不変条件は、ハッシュテーブルの検索ロジックに依存しているため厳守すること。
