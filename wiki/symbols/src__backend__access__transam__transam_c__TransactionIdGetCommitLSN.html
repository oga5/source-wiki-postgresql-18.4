TransactionIdGetCommitLSN

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/transam.c:381

目的

指定されたトランザクションID（XID）がコミットされた際のログシーケンス番号（LSN）を取得します。

シグネチャまたは形状

XLogRecPtr TransactionIdGetCommitLSN(TransactionId xid)

入力と出力

  • **入力**: xid (対象のトランザクションID)
  • **出力**: XLogRecPtr (コミット時のLSN。コミットされていない場合や特殊なXIDの場合は InvalidXLogRecPtr を返す)

前提条件と不変条件

  • cachedFetchXid と cachedCommitLSN は、直近の TransactionLogFetch 呼び出しによって更新されたキャッシュ状態を指します。
  • 特殊なXID（TransactionIdIsNormal が偽となるもの）は、常にコミット済みとみなされます。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: 引数 xid がグローバルな cachedFetchXid と一致する場合、共有メモリへのアクセスを回避してキャッシュされた cachedCommitLSN を即座に返します。 2. **特殊XID判定**: xid が正規のトランザクションIDでない場合、InvalidXLogRecPtr を返します。 3. **ステータス取得**: 上記以外の場合、TransactionIdGetStatus を呼び出し、トランザクションのステータスとともにLSNを取得して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_visibility.c:120
  • **呼び出し先**: TransactionIdEquals (マクロ), TransactionIdIsNormal (マクロ), TransactionIdGetStatus (内部関数)

関連データ構造

  • XLogRecPtr: WAL上の位置を示す64ビット整数。
  • TransactionId: トランザクションを識別する32ビット整数。

失敗モードと境界条件

  • xid がまだコミットされていない場合、TransactionIdGetStatus の挙動に依存してLSNが取得できない可能性があります。
  • キャッシュミス時は共有メモリへのアクセスが発生するため、高負荷時にはパフォーマンスに影響を与える可能性があります。

保守担当者向けメモ

  • 本関数は TransactionLogFetch のキャッシュを前提として最適化されています。キャッシュの有効性は TransactionLogFetch の実装に依存するため、キャッシュロジックを変更する際は注意してください。
