CheckSubscriptionRelkind

定義

  • **種別**: function
  • **定義箇所**: src/backend/executor/execReplication.c:876

目的

論理レプリケーションのターゲットとして指定されたリレーションが、サポートされている種類（通常のテーブルまたはパーティションテーブル）であるかを検証します。

シグネチャまたは形状

void CheckSubscriptionRelkind(char relkind, const char *nspname, const char *relname)

入力と出力

  • **入力**:
  • relkind: 検証対象のリレーション種別（pg_class.relkind）。
  • nspname: リレーションが属するスキーマ名。
  • relname: リレーション名。
  • **出力**: なし。
  • **副作用**: 指定されたリレーションがサポート対象外の場合、ERROR レベルで処理を中断し、エラーメッセージを報告します。

前提条件と不変条件

  • relkind は pg_class カタログから取得された有効な文字コードである必要があります。

アルゴリズムまたは振る舞い

1. 引数 relkind が RELKIND_RELATION（通常のテーブル）または RELKIND_PARTITIONED_TABLE（パーティションテーブル）のいずれかであるかを確認します。 2. 上記のいずれにも該当しない場合、ERRCODE_WRONG_OBJECT_TYPE エラーを発生させます。 3. エラー時には、対象のスキーマ名とリレーション名を含むメッセージと、サポートされていない relkind に応じた詳細情報を出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/subscriptioncmds.c
  • src/backend/replication/logical/relation.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • ereport（エラー報告用マクロ）
  • errdetail_relkind_not_supported（エラー詳細生成用）

関連データ構造

  • pg_class（relkind フィールド）

失敗モードと境界条件

  • **失敗モード**: relkind がビュー、シーケンス、外部テーブルなどのサポート外のオブジェクトタイプである場合、常にエラーとなります。

保守担当者向けメモ

  • 本関数は論理レプリケーションのセットアップおよび適用プロセスにおける型チェックの共通インターフェースです。新しいリレーションタイプをレプリケーション対象として追加する場合は、本関数の条件分岐を更新する必要があります。
