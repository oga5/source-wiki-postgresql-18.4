DSMRegistryShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/dsm_registry.c:62

目的

共有メモリ上に配置されるレジストリ管理構造体（DSMRegistryCtxStruct）を格納するために必要なバイト数を算出します。

シグネチャまたは形状

Size DSMRegistryShmemSize(void)

入力と出力

  • 入力: なし
  • 出力: Size 型（DSMRegistryCtxStruct のサイズを MAXALIGN で調整した値）

前提条件と不変条件

  • 戻り値は、PostgreSQLの共有メモリ初期化プロセスにおいて、レジストリ用の領域を確保するために使用されます。

アルゴリズムまたは振る舞い

1. DSMRegistryCtxStruct のサイズを取得します。 2. MAXALIGN マクロを適用し、プラットフォームのメモリ配置要件に合わせてアライメント調整された値を返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/ipc/ipci.c:115 (ShmemCreate 等の初期化フロー内)
  • 呼び出し先:
  • MAXALIGN マクロ（プラットフォーム依存のメモリ配置調整）

関連データ構造

  • DSMRegistryCtxStruct: 本関数がサイズを算出する対象の構造体。

失敗モードと境界条件

  • 本関数自体は計算のみを行うため、失敗モードはありません。
  • 戻り値は常に MAXALIGN された正の整数となります。

保守担当者向けメモ

  • DSMRegistryCtxStruct の定義が変更された場合、本関数が返すサイズが適切か確認してください。
  • src/backend/storage/ipc/ipci.c での呼び出しは、システム全体の共有メモリ確保量に影響するため、変更時は注意が必要です。
