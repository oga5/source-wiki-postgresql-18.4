astreamer_plain_writer_finalize

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_file.c:133

目的

astreamer_plain_writer コンテキストに関連付けられたファイルストリームを適切にクローズし、リソースを解放する。

シグネチャまたは形状

static void astreamer_plain_writer_finalize(astreamer *streamer)

入力と出力

  • **入力**: astreamer *streamer （astreamer_plain_writer 構造体としてキャストして使用）
  • **出力**: なし（副作用としてファイルクローズを実行）

前提条件と不変条件

  • 引数 streamer は astreamer_plain_writer 型の構造体として安全にキャスト可能であること。
  • mystreamer->file が有効なファイルポインタであるか、あるいは NULL であること。

アルゴリズムまたは振る舞い

1. 引数 streamer を astreamer_plain_writer 型にキャストする。 2. should_close_file フラグが真である場合、fclose を呼び出してファイルストリームを閉じる。 3. fclose が失敗した場合は pg_fatal を呼び出し、エラーメッセージを出力してプロセスを終了する。 4. 終了処理後、mystreamer->file を NULL に設定し、mystreamer->should_close_file を false に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/astreamer_file.c 内の astreamer_ops 構造体（finalize フィールド）を通じて呼び出される。
  • **呼び出し先**: fclose（標準ライブラリ）、pg_fatal（common/logging.h）。

関連データ構造

  • astreamer: 基底構造体。
  • astreamer_plain_writer: ファイル書き込み用のコンテキスト構造体。

失敗モードと境界条件

  • fclose が 0 以外の値を返した場合、pg_fatal により致命的エラーとして処理される。

保守担当者向けメモ

  • 本関数は astreamer インターフェースの終了処理の一部として設計されている。
  • ファイルクローズ後の状態リセット（file = NULL 等）は、二重解放や不正なポインタ参照を防ぐために重要である。
