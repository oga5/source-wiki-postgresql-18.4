window_rank_support

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/windowfuncs.c:157

目的

RANK() ウィンドウ関数に対するオプティマイザ向けのサポート関数です。単調性の通知およびウィンドウフレームの最適化設定を行います。

シグネチャまたは形状

Datum window_rank_support(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して Node *rawreq を受け取ります。
  • **出力**: SupportRequestWFuncMonotonic または SupportRequestOptimizeWindowClause の要求に応じて、更新されたノードポインタを Datum として返却します。要求に合致しない場合は NULL を返します。

前提条件と不変条件

  • rawreq は SupportRequestWFuncMonotonic または SupportRequestOptimizeWindowClause のいずれかである必要があります。
  • RANK() 関数は単調増加する性質を持つものとして扱われます。

アルゴリズムまたは振る舞い

1. **単調性の通知**: SupportRequestWFuncMonotonic が要求された場合、req->monotonic を MONOTONICFUNC_INCREASING に設定します。 2. **フレーム最適化**: SupportRequestOptimizeWindowClause が要求された場合、フレームオプションを ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW に強制変更します。

  • これにより、実行時に RANGE モードで必要となるピア行（同順位行）のチェックを回避し、計算コストを削減します。

呼び出し元と呼び出し先

  • **呼び出し元**: クエリ計画作成時にオプティマイザから呼び出されます（具体的な呼び出し元関数名はソースコード上では明示されていません）。
  • **呼び出し先**: IsA マクロおよび PG_RETURN_POINTER マクロを使用します。

関連データ構造

  • SupportRequestWFuncMonotonic: 単調性情報を伝達するための構造体。
  • SupportRequestOptimizeWindowClause: ウィンドウ句の最適化情報を伝達するための構造体。

失敗モードと境界条件

  • 未知の要求タイプが渡された場合、何も変更せず NULL を返します。

保守担当者向けメモ

  • 本関数は window_row_number_support() と同様のフレーム最適化戦略をとっています。フレーム定義を変更する際は、RANK() の計算ロジックが ROWS モードでも正しく動作することを維持してください。
