non_negative

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistproc.c:338

目的

浮動小数点数の値が負である場合に、それを 0.0 に正規化して返すためのユーティリティ関数です。

シグネチャまたは形状

static inline float non_negative(float val)

入力と出力

  • **入力**: float val（評価対象の浮動小数点数）
  • **出力**: float（入力値が 0 以上であればその値、負であれば 0.0）

前提条件と不変条件

  • 入力値が負の無限大や NaN（非数）である場合の挙動は、標準的な比較演算子 val >= 0.0f の仕様に依存します。

アルゴリズムまたは振る舞い

1. 引数 val が 0.0 以上であるか判定します。 2. 0.0 以上であれば val をそのまま返します。 3. 0.0 未満であれば 0.0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistproc.c:436
  • src/backend/access/gist/gistproc.c:438
  • （その他、同ファイル内の分割評価ロジックにて使用）
  • **呼び出し先**: なし（標準的な比較演算のみを実行）

関連データ構造

  • なし

失敗モードと境界条件

  • 入力値が負の数の場合、常に 0.0 が返されます。
  • 浮動小数点数の精度限界に近い値については、標準的な float 型の比較規則に従います。

保守担当者向けメモ

  • static inline で定義されているため、コンパイル時に呼び出し元へ展開されることが期待されます。
  • 主に GiST インデックスの分割アルゴリズム（picksplit）における面積やオーバーラップの計算結果を正規化するために使用されています。
