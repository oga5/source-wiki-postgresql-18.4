GinBuildState

定義

  • **種別**: struct
  • **定義箇所**: src/backend/access/gin/gininsert.c:139

目的

GINインデックス構築プロセスにおける状態管理を行う構造体です。メモリコンテキスト、ソート状態、統計情報、および並列構築時の同期情報を保持します。

シグネチャまたは形状

typedef struct {
    GinState        ginstate;
    double          indtuples;
    GinStatsData    buildStats;
    MemoryContext   tmpCtx;
    MemoryContext   funcCtx;
    BuildAccumulator accum;
    ItemPointerData tid;
    int             work_mem;
    GinLeader      *bs_leader;
    int             bs_num_workers;
    double          bs_numtuples;
    double          bs_reltuples;
    Tuplesortstate *bs_sortstate;
    Tuplesortstate *bs_worker_sort;
} GinBuildState;

入力と出力

  • **入力**: インデックス構築開始時に初期化され、ヒープスキャンやキー抽出の過程で各フィールドが更新されます。
  • **出力**: 構築完了後、GinStatsData や蓄積されたデータがインデックスのメタデータや物理ページへ反映されます。

前提条件と不変条件

  • bs_leader は並列インデックス構築時のみ、かつリーダープロセスにおいてのみ有効です。
  • bs_sortstate はリーダーを含む全ワーカーで利用されます。
  • bs_worker_sort は各ワーカー内での最初のマージパスに限定して使用されます。

アルゴリズムまたは振る舞い

  • ヒープスキャンから抽出されたキーとTIDを BuildAccumulator に蓄積します。
  • Tuplesortstate を用いてキー順に整列を行い、インデックスページへの書き込みを管理します。
  • 並列構築時には、ワーカーが生成した結果をリーダーが統合する際のコンテキストとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: ginbuild（インデックス構築のメイン関数）にてインスタンス化されます。
  • **呼び出し先**: ginHeapTupleBulkInsert、ginFlushBuildState、_gin_parallel_merge、_gin_parallel_heapscan 等の関数に引数として渡され、状態の読み書きが行われます。

関連データ構造

  • GinState: GINインデックスの基本状態。
  • BuildAccumulator: キーとTIDの蓄積用構造体。
  • GinLeader: 並列構築時のリーダー管理用構造体。
  • Tuplesortstate: ソート処理用構造体。

失敗モードと境界条件

  • work_mem の制限により、BuildAccumulator やソート処理がメモリ不足に陥る可能性があります。
  • 並列構築時、ワーカー間の同期やリーダーによるマージ処理において、共有メモリの競合や不整合が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は直列および並列の両方の構築パスで使用されるため、変更時は両方の動作への影響を確認してください。
  • bs_leader や bs_sortstate のライフサイクルは、並列構築の開始（_gin_begin_parallel）および終了（_gin_end_parallel）と密接に関連しています。
