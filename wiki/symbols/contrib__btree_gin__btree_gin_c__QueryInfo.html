QueryInfo

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gin/btree_gin.c:22

目的

GINインデックスにおいて、B-tree形式の範囲検索（<, <=, =, >=, >）を実行する際に、検索条件の戦略（StrategyNumber）と比較対象の値を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct QueryInfo
{
	StrategyNumber strategy;
	Datum		datum;
	bool		is_varlena;
	Datum		(*typecmp) (FunctionCallInfo);
} QueryInfo;

入力と出力

  • **入力**: 構造体の各フィールド（戦略番号、比較対象データ、可変長フラグ、比較関数ポインタ）を保持します。
  • **出力**: 検索処理の過程で、この構造体から strategy や datum が読み出され、インデックススキャン範囲の決定や比較判定に利用されます。

前提条件と不変条件

  • typecmp フィールドには、対象データ型に対応する比較関数（FunctionCallInfo を引数に取るもの）が格納されている必要があります。
  • datum は、インデックス内の値と比較可能な形式である必要があります。

アルゴリズムまたは振る舞い

  • gin_extract_query_[type] 関数内でインスタンス化され、検索条件をカプセル化します。
  • 検索実行時、GINエンジンはこの構造体を参照して、インデックス内の値が検索条件を満たすか、あるいはスキャンを終了すべきかを判定します。
  • BTLess 系戦略では、インデックスの先頭からクエリ値までをスキャンするため、この構造体に保持された値がスキャン範囲の境界として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: gin_btree_extract_query および各データ型ごとの gin_extract_query_[type] 関数が、この構造体を割り当てて初期化します。
  • **呼び出し先**: gin_btree_compare_prefix などの関数が、この構造体から datum や strategy を取得して比較ロジックを実行します。

関連データ構造

  • StrategyNumber: access/stratnum.h で定義されるB-tree比較戦略定数。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • leftmostvalue を使用する型（numeric や enum 等）では、絶対的な最小値が存在しないため、NULL ポインタや InvalidOid をダミー値として使用するケースがあります。この構造体を利用する際は、対象データ型の特性に応じた適切なダミー値の取り扱いが必要です。

保守担当者向けメモ

  • 本構造体は contrib/btree_gin/btree_gin.c 内の複数の関数で共有されています。構造体のフィールドを変更する場合は、関連するすべての gin_extract_query および比較関数への影響を確認してください。
