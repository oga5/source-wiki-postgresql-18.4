目的

ECPG（Embedded SQL for C）プリプロセッサの回帰テストとして、データベースオブジェクト（テーブル、関数、トリガー）の作成・実行・削除の一連のライフサイクルを検証する。

シグネチャまたは形状

int main()

入力と出力

  • 入力: なし（プログラム引数を使用しない）。
  • 出力: stdout にトリガーの起動結果を出力。stderr にデバッグ情報およびエラーメッセージを出力。終了コード 0 を返す。

前提条件と不変条件

  • ECPGランタイムライブラリが利用可能であること。
  • データベース ecpg1_regression への接続が可能であること。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、ECPGconnect でデータベースに接続。ECPGsetcommit でコミットモードを有効にする。 2. **セットアップ**: ECPGdo を使用して My_Table テーブル、Log テーブル、および My_Table_Check 関数とそれに関連するトリガーを作成する。 3. **実行**: My_Table に対して INSERT 文を2回実行し、トリガーを介して Log テーブルにデータが挿入されることを確認する。 4. **検証**: SELECT 文で Log テーブルからデータを取得し、text 変数に格納して printf で結果を表示する。 5. **クリーンアップ**: 作成したトリガー、関数、テーブルを順次 DROP し、ECPGdisconnect で接続を終了する。 6. **エラーハンドリング**: 各SQL実行後に sqlca.sqlwarn および sqlca.sqlcode を確認し、異常があれば sqlprint() を呼び出す。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内に情報なし。
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGsetcommit, ECPGdo, sqlprint, printf, ECPGdisconnect。

関連データ構造

  • sqlca: SQL通信領域。SQL文実行後の状態確認に使用。
  • text: char[25] 型のバッファ。SELECT 結果の格納に使用。

失敗モードと境界条件

  • SQL実行時にエラー（sqlca.sqlcode < 0）または警告（sqlca.sqlwarn[0] == 'W'）が発生した場合、sqlprint() が呼び出される。
  • データベース接続失敗やテーブル作成失敗などのSQLエラー発生時も、プログラムは終了せず後続の処理を継続する構造となっている。

保守担当者向けメモ

  • 本関数はプリプロセッサによって自動生成されたコードであるため、直接編集しないこと。
  • 各SQL文の直後に定型的なエラーチェックブロックが挿入されている。
  • テスト実行時の詳細ログは ECPGdebug(1, stderr) により制御されている。
