inet_spg_leaf_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_spgist.c:322

目的

SP-GiSTインデックスのリーフノードにおいて、格納された値がスキャンキーの条件に適合するかを判定する。

シグネチャまたは形状

Datum inet_spg_leaf_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgLeafConsistentIn（スキャンキー情報等）および spgLeafConsistentOut（結果格納用）を受け取る。
  • **出力**: bool 型の判定結果を PG_RETURN_BOOL で返す。また、out->recheck に false を設定し、out->leafValue にリーフの値を格納する。

前提条件と不変条件

  • すべてのテストは「完全（exact）」であると見なされ、再チェック（recheck）は不要である。

アルゴリズムまたは振る舞い

1. in->leafDatum から inet 型の値を抽出する。 2. out->recheck を false に設定する。 3. out->leafValue に抽出した値を設定する。 4. inet_spg_consistent_bitmap を呼び出し、スキャンキーとの適合性を判定してその結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiSTインデックスの検索プロセスから呼び出される（直接的な呼び出し元関数の詳細は提供されていない）。
  • **呼び出し先**: inet_spg_consistent_bitmap を呼び出す。

関連データ構造

  • spgLeafConsistentIn: SP-GiSTのリーフ整合性チェック用入力構造体。
  • spgLeafConsistentOut: SP-GiSTのリーフ整合性チェック用出力構造体。
  • inet: ネットワーク型（inet）の内部表現。

失敗モードと境界条件

  • 特筆すべき失敗モードは定義されていない。

保守担当者向けメモ

  • 本関数は inet_spg_consistent_bitmap に処理を委譲するラッパーとして機能している。
  • 適合判定ロジックを変更する場合は、inet_spg_consistent_bitmap の実装を確認すること。
