pgstat_init_snapshot_fixed

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat.c:1081

目的

カスタム統計種別（PGSTAT_KIND_CUSTOM_MIN から PGSTAT_KIND_CUSTOM_MAX）に関連する固定数の統計データ領域を、バックエンドローカルのスナップショット内に初期化します。

シグネチャまたは形状

static void pgstat_init_snapshot_fixed(void)

入力と出力

  • 入力: なし（グローバル変数 pgStatLocal および登録済みの統計種別情報を参照）
  • 出力: なし（副作用として pgStatLocal.snapshot.custom_data を更新）

前提条件と不変条件

  • TopMemoryContext が初期化されている必要があります。
  • 統計種別情報が pgstat_get_kind_info を通じて取得可能である必要があります。

アルゴリズムまたは振る舞い

1. PGSTAT_KIND_CUSTOM_MIN から PGSTAT_KIND_CUSTOM_MAX までの範囲でループを実行します。 2. 各統計種別について pgstat_get_kind_info を呼び出し、情報が存在し、かつ fixed_amount が設定されているかを確認します。 3. 条件を満たす場合、kind_info->shared_data_len に基づいて TopMemoryContext 上にメモリを割り当て、pgStatLocal.snapshot.custom_data の該当インデックスに格納します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/activity/pgstat.c:186 (プロトタイプ宣言)
  • src/backend/utils/activity/pgstat.c:646 (呼び出し箇所)
  • 呼び出し先:
  • pgstat_get_kind_info
  • MemoryContextAlloc

関連データ構造

  • PgStat_KindInfo: 統計種別のメタデータ（fixed_amount や shared_data_len を保持）。
  • pgStatLocal: バックエンドローカルな統計状態を保持するグローバル構造体。

失敗モードと境界条件

  • pgstat_get_kind_info が NULL を返す場合、その種別はスキップされます。
  • fixed_amount が 0 の場合、メモリ割り当ては行われません。
  • メモリ割り当て失敗時は MemoryContextAlloc の標準的な挙動（エラー発生）に従います。

保守担当者向けメモ

  • 本関数はカスタム統計種別の初期化に特化しています。組み込みの統計種別は対象外です。
  • 割り当てられたメモリは TopMemoryContext に属するため、バックエンドの生存期間中保持されます。
