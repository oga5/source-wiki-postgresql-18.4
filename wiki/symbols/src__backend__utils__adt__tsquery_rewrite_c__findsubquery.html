findsubquery

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_rewrite.c:266

目的

tsquery の木構造（QTNode）を走査し、指定された部分木パターン ex を別の部分木 subs に置換する起点となる関数です。

シグネチャまたは形状

QTNode *findsubquery(QTNode *root, QTNode *ex, QTNode *subs, bool *isfind)

入力と出力

  • **入力**:
  • root: 走査対象となる tsquery の木構造のルートノード。
  • ex: 置換対象となる部分木パターン。
  • subs: 置換後の新しい部分木。
  • isfind: 置換が発生したかどうかを格納するブール値へのポインタ（NULL指定可能）。
  • **出力**:
  • 置換処理後の新しい tsquery 木構造のルートノードを返します。

前提条件と不変条件

  • root および ex は、事前に QTNSort 等で正規化された QTNode 構造体である必要があります。
  • 内部で再帰処理を行うため、スタック深さの制限に注意が必要です。

アルゴリズムまたは振る舞い

1. ローカル変数 DidFind を false で初期化します。 2. 内部関数 dofindsubquery を呼び出し、木構造の再帰的な探索と置換を実行します。 3. dofindsubquery の結果を root に再代入します。 4. isfind が非NULLの場合、置換の成否（DidFind）を書き込みます。 5. 最終的な root を返します。

関連データ構造

  • QTNode: tsquery を表現する木構造体。

失敗モードと境界条件

  • root が NULL の場合、そのまま NULL を返します。
  • 置換対象が見つからない場合、木構造は変更されず、isfind には false が設定されます。
  • 非常に深い木構造に対しては、スタックオーバーフローのリスクがあります。

保守担当者向けメモ

  • 本関数は置換のラッパーであり、実際の探索ロジックは dofindsubquery に依存しています。
  • 置換後の木構造は、dofindsubquery によって不要な演算子ノードの削除や簡略化が行われます。
