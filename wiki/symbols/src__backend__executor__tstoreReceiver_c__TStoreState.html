TStoreState

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/tstoreReceiver.c:30

目的

DestReceiver インターフェースを継承し、実行エンジンから受け取ったタプルを Tuplestore に格納するための状態管理を行うプライベート構造体です。

シグネチャまたは形状

typedef struct {
    DestReceiver pub;
    Tuplestorestate *tstore;
    MemoryContext cxt;
    bool detoast;
    TupleDesc target_tupdesc;
    const char *map_failure_msg;
    Datum *outvalues;
    Datum *tofree;
    TupleConversionMap *tupmap;
    TupleTableSlot *mapslot;
} TStoreState;

入力と出力

  • **入力**: CreateTuplestoreDestReceiver により生成され、SetTuplestoreDestReceiverParams を通じて Tuplestorestate や変換設定が注入されます。
  • **出力**: 実行エンジンから渡されたタプルを、設定された変換ルール（detoast または TupleConversionMap）に従って処理し、tstore へ格納します。

前提条件と不変条件

  • tstore への書き込みは、指定された MemoryContext (cxt) の下で行われます。
  • detoast フラグと target_tupdesc（タプル変換）を同時に有効にすることはできません（SetTuplestoreDestReceiverParams 内の Assert により保護）。

アルゴリズムまたは振る舞い

1. **初期化**: tstoreStartupReceiver がタプルの属性を評価し、適切な受信関数（notoast, detoast, tupmap）を選択します。 2. **受信**: 実行エンジンがタプルを渡すと、選択された関数が outvalues や tupmap を使用してタプルを変換し、tuplestore_putvalues または tuplestore_puttupleslot を呼び出します。 3. **終了**: tstoreShutdownReceiver により、変換マップやワークスペース用メモリが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/tstoreReceiver.c 内の各関数（tstoreStartupReceiver, tstoreReceiveSlot_*, tstoreShutdownReceiver, CreateTuplestoreDestReceiver など）がこの構造体をキャストして操作します。
  • **呼び出し先**: tuplestore.h の API（tuplestore_putvalues 等）および tupconvert.h の変換関数を呼び出します。

関連データ構造

  • DestReceiver: 基底となるインターフェース構造体。
  • Tuplestorestate: データの格納先となる実体。
  • TupleConversionMap: タプル構造変換が必要な場合に使用。
  • TupleTableSlot: 変換後のタプルを保持するためのスロット。

失敗モードと境界条件

  • target_tupdesc が指定されている場合、変換失敗時に map_failure_msg を用いたエラーハンドリングが行われます。
  • メモリコンテキストの不一致により、Tuplestore への書き込み時にメモリリークや不正アクセスが発生する可能性があるため、呼び出し元は適切なコンテキストを維持する必要があります。

保守担当者向けメモ

  • 新しい変換ロジックを追加する場合は、tstoreStartupReceiver の条件分岐を拡張し、対応する tstoreReceiveSlot_* 関数を実装してください。
  • detoast と target_tupdesc の排他制約は現在の設計上の不変条件です。
