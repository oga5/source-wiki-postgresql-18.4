btree_xlog_delete

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtxlog.c:651

目的

B-tree インデックスの削除操作に関する WAL レコードをリプレイし、対象ページからタプルの削除および更新を適用して物理的な状態を復元する。

シグネチャまたは形状

static void btree_xlog_delete(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象の WAL レコード）。
  • **出力**: なし（副作用としてバッファの内容を更新）。

前提条件と不変条件

  • ホットスタンバイ環境の場合、ページ更新前に ResolveRecoveryConflictWithSnapshot を呼び出し、リカバリ競合を解決する必要がある。
  • ページ更新時にクリーンアップロックは不要。
  • ページ内の BTP_HAS_GARBAGE フラグは、削除処理後に適切にクリアされる必要がある。

アルゴリズムまたは振る舞い

1. **競合解決**: ホットスタンバイ環境であれば、xl_btree_delete 内の snapshotConflictHorizon を用いてリカバリ競合を解決する。 2. **バッファ読み込み**: XLogReadBufferForRedo を呼び出し、対象ブロックが BLK_NEEDS_REDO を返す場合のみ処理を続行する。 3. **更新適用**: xlrec->nupdated > 0 の場合、btree_xlog_updates を呼び出してタプルの更新を反映する。 4. **削除適用**: xlrec->ndeleted > 0 の場合、PageIndexMultiDelete を使用して指定されたオフセットのタプルを削除する。 5. **フラグ更新**: ページ内の btpo_flags から BTP_HAS_GARBAGE ビットをクリアする。 6. **コミット**: ページの LSN を更新し、バッファをダーティとしてマークする。 7. **解放**: バッファが有効であればロックを解除し解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: btree_redo（src/backend/access/nbtree/nbtxlog.c:1051 付近でディスパッチされる）。
  • **呼び出し先**:
  • ResolveRecoveryConflictWithSnapshot
  • XLogRecGetBlockTag
  • XLogReadBufferForRedo
  • XLogRecGetBlockData
  • btree_xlog_updates
  • PageIndexMultiDelete
  • BTPageGetOpaque
  • PageSetLSN
  • MarkBufferDirty
  • UnlockReleaseBuffer

関連データ構造

  • xl_btree_delete: 削除操作の WAL レコード構造体。
  • xl_btree_update: 更新操作の WAL レコード構造体。
  • BTPageOpaque: B-tree ページヘッダーの不透明部分（btpo_flags を含む）。

失敗モードと境界条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO 以外を返した場合、ページ更新はスキップされる。
  • nupdated または ndeleted が 0 の場合、それぞれの処理は実行されない。

保守担当者向けメモ

  • 本関数はリカバリ時の物理的整合性を保証する。論理的な削除操作（nbtinsert.c 等）とは異なり、ロックの取得は簡略化されている。
  • BTP_HAS_GARBAGE フラグのクリアは必須であり、削除後のページ状態を正しく反映させるために重要である。
