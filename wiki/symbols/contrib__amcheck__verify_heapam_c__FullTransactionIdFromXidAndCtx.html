FullTransactionIdFromXidAndCtx

定義

  • 種別: function
  • 定義箇所: contrib/amcheck/verify_heapam.c:1976

目的

32ビットのトランザクションID（TransactionId）を、HeapCheckContext が保持する現在のトランザクション状態に基づいて、64ビットのフルトランザクションID（FullTransactionId）へ変換します。

シグネチャまたは形状

static FullTransactionId FullTransactionIdFromXidAndCtx(TransactionId xid, const HeapCheckContext *ctx)

入力と出力

  • **入力**:
  • xid: 変換対象の32ビットトランザクションID。
  • ctx: 検証コンテキスト（HeapCheckContext）。next_xid および next_fxid を参照するために使用。
  • **出力**:
  • 変換された FullTransactionId。

前提条件と不変条件

  • ctx->next_xid および ctx->next_fxid は正常な値であること。
  • ctx->next_fxid から抽出される32ビットXIDと ctx->next_xid が一致していること。
  • 戻り値の FullTransactionId は常に正常な値（FullTransactionIdIsNormal）であること。

アルゴリズムまたは振る舞い

1. xid が正常なトランザクションIDでない場合、エポック0の FullTransactionId を返します。 2. ctx->next_xid と xid の差分（32ビットモジュロ演算）を計算します。 3. 破損により xid がエポック0より前を指すような異常なケースでは、FirstNormalFullTransactionId を返します。 4. 通常時は、ctx->next_fxid から計算した64ビット値から差分を減算し、対応する FullTransactionId を算出します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_heapam.c 内の複数箇所（ctx.relfrozenfxid の初期化、ctx->oldest_fxid の計算、およびタプル検証時の fxid 算出など）。
  • **呼び出し先**: TransactionIdIsNormal, FullTransactionIdFromEpochAndXid, U64FromFullTransactionId, XidFromFullTransactionId, EpochFromFullTransactionId, FullTransactionIdFromU64, FirstNormalFullTransactionId 等のアクセスマクロ/関数。

関連データ構造

  • HeapCheckContext: 検証中のトランザクション状態を保持する構造体。
  • FullTransactionId: 64ビットのトランザクションID表現。
  • TransactionId: 32ビットのトランザクションID表現。

失敗モードと境界条件

  • xid が正常でない場合、エポック0として処理されます。
  • 32ビットのモジュロ演算の結果、エポックの境界を跨ぐような破損が検出された場合、FirstNormalFullTransactionId にフォールバックします。

保守担当者向けメモ

  • 本関数は amcheck によるヒープ検証中に、古いXIDを現在のコンテキストに合わせて64ビット化するために不可欠です。
  • 破損データに対する安全策が含まれていますが、論理的な整合性が取れない極端な破損データに対しては、FirstNormalFullTransactionId を返すことでクラッシュを回避しています。
