add_one_float8

定義

  • 種別: function
  • 定義箇所: src/tutorial/funcs.c:36

目的

PostgreSQLのユーザー定義関数として、入力された float8 型の値に 1.0 を加算して返すサンプル実装です。

シグネチャまたは形状

Datum add_one_float8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される Datum 型の引数。PG_GETARG_FLOAT8(0) により第1引数を float8 型として取得します。
  • **出力**: PG_RETURN_FLOAT8 マクロを通じて、計算結果（arg + 1.0）を Datum 型に変換して返します。

前提条件と不変条件

  • PostgreSQLのV1呼び出し規約に従う必要があります。
  • PG_FUNCTION_INFO_V1(add_one_float8) マクロによる関数情報の登録が必須です。

アルゴリズムまたは振る舞い

1. PG_GETARG_FLOAT8(0) を呼び出し、入力値を float8 型として抽出します。 2. 抽出した値に 1.0 を加算します。 3. PG_RETURN_FLOAT8 を使用して、加算結果を呼び出し元へ返却します。 ※ float8 型は内部的に参照渡しですが、マクロを使用することでその複雑さを隠蔽しています。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（SQLクエリから呼び出されることを想定）。
  • **呼び出し先**:
  • PG_GETARG_FLOAT8（マクロ）
  • PG_RETURN_FLOAT8（マクロ）
  • 補足: ソースコード内の PG_FUNCTION_INFO_V1(add_one_float8) は、本関数をV1規約の関数として登録するための宣言です。

関連データ構造

  • Datum: PostgreSQLの汎用データ型表現。
  • float8: C言語の double に相当するPostgreSQLの浮動小数点型。

失敗モードと境界条件

  • 引数が提供されない場合や、型変換が不可能な場合は、PostgreSQLの関数マネージャによってエラーがスローされます。

保守担当者向けメモ

  • 本関数は src/tutorial/funcs.c に含まれるV1規約のサンプルです。
  • メモリ管理（palloc等）は本関数内では発生しませんが、将来的な拡張で可変長データを扱う場合は palloc の使用とメモリコンテキストの考慮が必要です。
