r_residual_suffix

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_spanish.c:944

目的

スペイン語のステミング処理の最終段階において、単語末尾に残存する特定の接尾辞を特定し、条件に応じて削除する。

シグネチャまたは形状

static int r_residual_suffix(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z（Snowballのランタイム環境構造体）
  • 出力: int（処理が成功した場合は1、失敗または該当なしの場合は0、エラー時は負の値を返す）

前提条件と不変条件

  • z は有効な SN_env 構造体を指していること。
  • a_9 配列（8要素）が定義されており、find_among_b による後方検索が可能であること。
  • 削除操作を行う場合、r_RV 関数により対象領域がRV（語幹の開始位置）内にあることが検証される。

アルゴリズムまたは振る舞い

1. z->ket を現在のカーソル位置に設定し、a_9 テーブルを用いて後方から接尾辞を検索する。 2. 該当する接尾辞が見つからない場合は終了する。 3. among_var の値に応じて分岐する:

  • **case 1**: r_RV を呼び出し、RV領域内であれば slice_del で接尾辞を削除する。
  • **case 2**: r_RV を呼び出し、RV領域内であれば削除を実行。その後、削除した直前の文字が 'u' であり、さらにその前が 'g' である場合に限り、追加の削除処理を試みる。

    • 4. 処理結果（成功・失敗・エラー）を呼び出し元に返す。

呼び出し元と呼び出し先

  • 呼び出し元: spanish_UTF_8_stem（直接の言及はないが、モジュール内の制御フローに基づく）。
  • 呼び出し先: find_among_b, r_RV, slice_del。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_9: among 構造体の配列。残余接尾辞のパターンを定義。

失敗モードと境界条件

  • r_RV が0を返した場合、領域外とみなされ削除は行われない。
  • slice_del が負の値を返した場合、メモリ操作エラーとしてそのまま伝播する。
  • case 2 において、'u' や 'g' の確認時にバッファ境界（z->lb）に達した場合は処理を中断する。

保守担当者向けメモ

  • 本関数は自動生成コードの一部である。ロジックの変更が必要な場合は、直接編集せずSnowballソース（.sbl）を修正すること。
  • case 2 の 'gu' 削除ロジックは、スペイン語特有の綴りルールに基づいている。
