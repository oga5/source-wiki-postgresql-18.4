execTuplesHashPrepare

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execGrouping.c:96

目的

ハッシュ集約や重複排除処理において、各カラムの比較演算子に対応する等価比較関数およびハッシュ関数を準備する。

シグネチャまたは形状

void execTuplesHashPrepare(int numCols, const Oid *eqOperators, Oid **eqFuncOids, FmgrInfo **hashFunctions)

入力と出力

  • **入力**:
  • numCols: 処理対象のカラム数。
  • eqOperators: 各カラムに対応する等価演算子（Oid）の配列。
  • **出力**:
  • *eqFuncOids: 演算子から導出された等価比較関数の Oid 配列（呼び出し側でメモリ確保される）。
  • *hashFunctions: 演算子から導出されたハッシュ関数の FmgrInfo 配列（呼び出し側でメモリ確保される）。

前提条件と不変条件

  • eqOperators 配列は numCols 分の要素を持つこと。
  • 指定された等価演算子に対して、対応するハッシュ関数がシステムカタログから取得可能であること。
  • 取得された左右のハッシュ関数が一致していること（クロスタイプ演算子はサポート外）。

アルゴリズムまたは振る舞い

1. numCols 分のメモリを palloc で確保する。 2. 各カラムの等価演算子について以下を実行する:

  • get_opcode で等価比較関数を取得する。
  • get_op_hash_functions で左右のハッシュ関数を取得する。
  • 左右のハッシュ関数が同一であることを Assert で検証する。
  • fmgr_info を呼び出し、ハッシュ関数を FmgrInfo 構造体にキャッシュする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • nodeAgg.c
  • nodeRecursiveunion.c
  • nodeSetOp.c
  • **呼び出し先**:
  • get_opcode
  • get_op_hash_functions
  • fmgr_info

関連データ構造

  • FmgrInfo: 関数呼び出しのための情報構造体。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • 指定された演算子に対応するハッシュ関数が見つからない場合、elog(ERROR) により処理を中断する。

保守担当者向けメモ

  • 本関数はメモリを動的に確保するため、呼び出し側で適切に管理する必要がある。
  • クロスタイプハッシュ演算子はサポートしていないため、将来的な拡張時には Assert の制約を見直す必要がある。
