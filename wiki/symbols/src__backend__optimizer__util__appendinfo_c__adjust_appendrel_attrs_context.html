adjust_appendrel_attrs_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/util/appendinfo.c:31

目的

式ツリー内の属性（Var ノード）を親リレーションから子リレーションへ変換する際、再帰的な走査処理に必要な状態情報を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	PlannerInfo *root;
	int			nappinfos;
	AppendRelInfo **appinfos;
} adjust_appendrel_attrs_context;

入力と出力

  • **入力**:
  • root: 現在のプランナのグローバル情報。
  • nappinfos: appinfos 配列内の要素数。
  • appinfos: 変換対象となる AppendRelInfo 構造体の配列。
  • **出力**: 本構造体自体は状態保持用であり、変換処理の結果は adjust_appendrel_attrs_mutator 等の関数を通じて式ツリーへ反映されます。

前提条件と不変条件

  • appinfos は、変換対象となる親リレーションと子リレーションの対応関係を正しく保持している必要があります。
  • nappinfos は appinfos 配列の有効な長さを正確に示している必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は expression_tree_mutator を利用した再帰的な式変換処理において、コンテキストオブジェクトとして渡されます。
  • adjust_appendrel_attrs_mutator が式ツリーを走査する際、このコンテキストを参照して Var ノードの属性番号を子テーブルの対応する番号へ書き換えます。

呼び出し元と呼び出し先

  • **呼び出し元**: adjust_appendrel_attrs および adjust_appendrel_attrs_mutator がこの構造体をインスタンス化または参照します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • PlannerInfo: プランナのグローバル状態。
  • AppendRelInfo: 親子リレーション間の変換メタデータ。

失敗モードと境界条件

  • 構造体内のポインタが NULL である場合、または nappinfos が不適切な値である場合、式変換処理中にセグメンテーションフォールトや不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は adjust_appendrel_attrs 関連の変換ルーチンにおいて不可欠な状態保持コンテナです。変更を加える際は、adjust_appendrel_attrs_mutator での参照方法と整合性を取る必要があります。
