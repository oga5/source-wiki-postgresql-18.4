plpgsql_ns_find_nearest_loop

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_funcs.c:213

目的

名前空間チェーンを遡り、最も内側にあるループラベル（PLPGSQL_LABEL_LOOP）を検索します。

シグネチャまたは形状

PLpgSQL_nsitem *plpgsql_ns_find_nearest_loop(PLpgSQL_nsitem *ns_cur)

入力と出力

  • **入力**: ns_cur (検索を開始する名前空間アイテムへのポインタ)
  • **出力**: 見つかった場合は該当する PLpgSQL_nsitem へのポインタ、見つからない場合は NULL を返します。

前提条件と不変条件

  • ns_cur は NULL または有効な名前空間アイテムのリンクリストのノードである必要があります。
  • 名前空間チェーンは prev ポインタを辿ることで構成されています。

アルゴリズムまたは振る舞い

1. ns_cur が NULL になるまでループを繰り返します。 2. 現在のノード ns_cur の itemtype が PLPGSQL_NSTYPE_LABEL であり、かつ itemno が PLPGSQL_LABEL_LOOP であるかを確認します。 3. 条件に合致した場合、そのノードを即座に返します。 4. 合致しない場合、ns_cur を ns_cur->prev に更新して探索を続行します。 5. リストの末尾に到達しても見つからない場合、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは、特定の呼び出し元を特定できません。
  • **呼び出し先**: 外部関数への呼び出しはありません。

関連データ構造

  • PLpgSQL_nsitem: 名前空間アイテムを保持する構造体。itemtype および itemno フィールドを使用してラベルの種類を判定します。

失敗モードと境界条件

  • ns_cur が NULL の場合、ループに入らず直ちに NULL を返します。
  • 名前空間チェーン内にループラベルが存在しない場合、探索はリストの終端まで行われ、NULL が返されます。

保守担当者向けメモ

  • 本関数は plpgsql.h で宣言されており、コンパイル時または実行時のスコープ解析において、CONTINUE や EXIT ステートメントが対象とするループを特定する際に利用されることを想定しています。
