InlineWorkListItem

定義

  • 種別: struct
  • 定義箇所: src/backend/jit/llvm/llvmjit_inline.cpp:72

目的

JIT コンパイルにおけるモジュール境界を越えた関数インライン展開の計画において、インライン化対象となるシンボルと、その検索対象となるモジュールインデックスの集合を保持するために使用されます。

シグネチャまたは形状

typedef struct InlineWorkListItem
{
	llvm::StringRef symbolName;
	llvm::SmallVector<llvm::ModuleSummaryIndex *, 2> searchpath;
} InlineWorkListItem;

入力と出力

  • **入力**: インライン化対象の関数名（symbolName）および、その定義を探索するための LLVM モジュールサマリーインデックスのリスト（searchpath）。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • symbolName は、LLVM モジュール内で解決されるべき関数名を指す必要があります。
  • searchpath は、インライン化の計画構築時に llvm_build_inline_plan によって設定される探索パスを保持します。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持用ですが、インライン化計画の構築プロセスにおいて以下の役割を果たします。 1. InlineWorkList（InlineWorkListItem のリスト）に格納され、インライン化すべき関数のキューとして機能します。 2. llvm_build_inline_plan 内で、pop_back_val() を用いて順次取り出され、対象関数の定義検索と再帰的な依存関係解析に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/jit/llvm/llvmjit_inline.cpp 内の llvm_build_inline_plan 関数が、この構造体を生成および操作します。
  • **呼び出し先**: 本構造体はデータ構造であるため、直接的な関数呼び出しは行いません。

関連データ構造

  • InlineWorkList: InlineWorkListItem を要素とする llvm::SmallVector 型のエイリアス。
  • llvm::ModuleSummaryIndex: LLVM のモジュールサマリー情報。
  • llvm::StringRef: シンボル名を保持するための LLVM 文字列参照型。

失敗モードと境界条件

  • searchpath が空の場合、対象シンボルの定義を検索できず、インライン化の計画が失敗する可能性があります。
  • symbolName が無効な場合、LLVM のインデックス検索処理において不整合が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は llvmjit_inline.cpp 内のインライン化計画フェーズでのみ使用されます。
  • searchpath の要素数（デフォルト 2）は、インライン化の探索深さや複雑さに応じて調整される可能性があります。
