VariableHasHook

定義

  • 種別: function
  • 定義箇所: src/bin/psql/variables.c:436

目的

指定された変数名が VariableSpace 内に存在し、かつその変数に代入フック（assign_hook）または置換フック（substitute_hook）のいずれかが登録されているかを判定する。

シグネチャまたは形状

bool VariableHasHook(VariableSpace space, const char *name)

入力と出力

  • **入力**:
  • space: 変数リストを保持する VariableSpace 構造体。
  • name: 検索対象の変数名。
  • **出力**:
  • bool: 指定された変数が存在し、かつフックが登録されていれば true、それ以外（変数が見つからない、またはフックが未登録）であれば false を返す。

前提条件と不変条件

  • space および name は NULL であってはならない（Assert により検証）。
  • 変数リストは名前順（strcmp 順）にソートされている必要がある。

アルゴリズムまたは振る舞い

1. space 内の変数リストを先頭から順に走査する。 2. strcmp を用いて変数名を比較する。 3. 名前が一致した場合、その変数の substitute_hook または assign_hook が NULL でないかを確認し、いずれかが存在すれば true を返す。 4. 比較結果が正（cmp > 0）となった場合、リストはソートされているため、対象の変数は存在しないと判断してループを終了する。 5. リストの末尾まで一致が見つからなければ false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/common.c:824
  • **呼び出し先**:
  • strcmp（標準ライブラリ）

関連データ構造

  • VariableSpace: 変数リストのヘッダー構造体。
  • struct _variable: 変数名、値、およびフック関数ポインタを保持する内部構造体。

失敗モードと境界条件

  • space または name が NULL の場合、アサーション違反によりプログラムが停止する可能性がある。
  • 変数が存在しない場合、または変数は存在するがフックが両方とも NULL の場合は false を返す。

保守担当者向けメモ

  • 本関数は、変数の値そのものの有無ではなく、フックの有無を判定することに特化している。
  • リストのソート順序に依存した検索を行っているため、変数リストの挿入・削除ロジックがソート順を維持していることが前提となる。
