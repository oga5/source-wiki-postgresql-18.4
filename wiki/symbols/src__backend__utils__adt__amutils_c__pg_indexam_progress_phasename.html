pg_indexam_progress_phasename

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/amutils.c:450

目的

指定されたアクセスメソッド（AM）のOIDとフェーズ番号に基づき、インデックス構築プロセスの現在のフェーズ名を取得する。

シグネチャまたは形状

Datum pg_indexam_progress_phasename(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): 対象とするアクセスメソッドのOID (amoid)。
  • PG_GETARG_INT32(1): 取得対象のフェーズ番号 (phasenum)。
  • **出力**:
  • フェーズ名を表す text 型のデータ。
  • AMが指定されていない、または該当するフェーズ名定義が存在しない場合は NULL を返す。

前提条件と不変条件

  • GetIndexAmRoutineByAmId が正常に IndexAmRoutine 構造体を返却できること。
  • 対象のAMが ambuildphasename ルーチンを実装していること。

アルゴリズムまたは振る舞い

1. GetIndexAmRoutineByAmId を呼び出し、指定された amoid に対応する IndexAmRoutine を取得する。 2. ルーチンが存在しない、または ambuildphasename 関数ポインタが未定義の場合は NULL を返す。 3. ambuildphasename(phasenum) を呼び出し、フェーズ名（文字列）を取得する。 4. 取得した文字列が NULL であれば NULL を返す。 5. 取得した文字列を CStringGetTextDatum を用いて text 型の Datum に変換し、返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（直接的なC言語レベルの呼び出し元情報は提供されていない）。
  • **呼び出し先**:
  • GetIndexAmRoutineByAmId
  • routine->ambuildphasename (AM固有の実装)

関連データ構造

  • IndexAmRoutine: アクセスメソッドの機能定義を保持する構造体。ambuildphasename 関数ポインタを含む。

失敗モードと境界条件

  • amoid が無効な場合、GetIndexAmRoutineByAmId の挙動に従う（通常はエラーまたはNULL）。
  • ambuildphasename が指定された phasenum に対して NULL を返す場合、本関数も NULL を返す。

保守担当者向けメモ

  • 本関数はインデックス構築の進捗報告をサポートするためのラッパーである。
  • 新しいAMを追加し、進捗報告をサポートする場合は、IndexAmRoutine の ambuildphasename を適切に実装する必要がある。
