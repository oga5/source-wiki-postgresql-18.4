PQresultAlloc

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-exec.c:543

目的

PGresult オブジェクトに関連付けられたメモリ領域を効率的に割り当てるためのインターフェースです。

シグネチャまたは形状

void *PQresultAlloc(PGresult *res, size_t nBytes);

入力と出力

  • **入力**:
  • res: メモリ割り当て対象となる PGresult オブジェクトへのポインタ。
  • nBytes: 割り当てるバイト数。
  • **出力**:
  • 成功時: 割り当てられたメモリ領域へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • res が NULL である場合、または res が静的なエラー用オブジェクト OOM_result を指している場合、割り当ては行われず NULL を返します。

アルゴリズムまたは振る舞い

1. 引数 res が有効な PGresult オブジェクトであるかを確認します。 2. 無効な場合（NULL または OOM_result）は直ちに NULL を返します。 3. 有効な場合、内部関数 pqResultAlloc(res, nBytes, true) を呼び出し、メモリ割り当てを実行します。この内部関数は、メモリ断片化を抑えるために大きなブロック単位でメモリを管理する仕組みを利用します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-exec.c 内のクエリ結果構築処理（例: 属性記述子の割り当てなど）。
  • **呼び出し先**: pqResultAlloc（内部関数）。

関連データ構造

  • PGresult: クエリ結果を保持する主要な構造体。
  • OOM_result: メモリ不足時に使用される静的なエラー結果オブジェクト。

失敗モードと境界条件

  • res が NULL の場合、安全に NULL を返します。
  • res が OOM_result を指している場合、メモリ割り当てを試みずに NULL を返します。
  • 内部の pqResultAlloc がメモリ確保に失敗した場合の挙動は、当該関数の実装に依存します。

保守担当者向けメモ

  • 本関数は PGresult のメモリ管理における公開 API です。個別のフィールドごとに malloc するのではなく、PGresult に紐付いたメモリブロックを再利用する設計となっているため、メモリ解放時には PQclear を使用してください。
