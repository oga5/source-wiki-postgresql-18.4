run_reindex_command

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/reindexdb.c:581

目的

構築済みの REINDEX SQL 文をデータベースに対して非同期に発行し、実行失敗時にはエラーログを出力する。

シグネチャまたは形状

static void run_reindex_command(PGconn *conn, ReindexType type, const char *name, bool echo, PQExpBufferData *sql)

入力と出力

  • **入力**:
  • conn: データベース接続ハンドル (PGconn *)。
  • type: 再構築対象の種別 (ReindexType)。
  • name: 対象オブジェクト名（DB名、テーブル名等）。
  • echo: SQL 文を標準出力に表示するかどうかのフラグ。
  • sql: 実行する SQL 文が格納されたバッファ (PQExpBufferData *)。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • sql バッファには、有効な REINDEX SQL 文が格納されている必要がある。
  • conn は接続済みであること。

アルゴリズムまたは振る舞い

1. echo が真の場合、sql->data を標準出力に表示する。 2. PQsendQuery を使用して、SQL 文をデータベースサーバーへ非同期に送信する。 3. 送信が失敗した場合（PQsendQuery が 1 を返さない場合）、type に応じて適切なエラーメッセージを pg_log_error で出力する。エラーメッセージには対象オブジェクト名とサーバーからのエラー詳細が含まれる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/scripts/reindexdb.c 内の並列実行処理ループ（reindex_one_database 等から呼び出される箇所）。
  • **呼び出し先**:
  • printf (標準出力)
  • PQsendQuery (libpq)
  • pg_log_error (ログ出力)
  • PQdb, PQerrorMessage (libpq)

関連データ構造

  • ReindexType: 再構築対象（DATABASE, INDEX, SCHEMA, SYSTEM, TABLE）を定義する列挙型。
  • PGconn: libpq のデータベース接続構造体。
  • PQExpBufferData: SQL 文を保持する動的文字列バッファ。

失敗モードと境界条件

  • PQsendQuery が失敗した場合、処理は中断されず、エラーログが出力されるのみである。
  • type ごとにエラーメッセージのフォーマットが固定されており、name が NULL の場合（REINDEX_DATABASE や REINDEX_SYSTEM の場合など）の挙動は各ケース文に依存する。

保守担当者向けメモ

  • 本関数は SQL の送信のみを担当し、実行結果の待機や確認は行わない。並列実行時の同期制御は呼び出し元で行う必要がある。
  • 新しい ReindexType を追加した場合は、エラーログ出力用の switch 文にもケースを追加すること。
