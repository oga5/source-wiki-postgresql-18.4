RelationGetIndexScan

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/genam.c:79

目的

インデックススキャンを管理するためのデータ構造 IndexScanDesc をメモリ上に割り当て、初期化する。

シグネチャまたは形状

IndexScanDesc RelationGetIndexScan(Relation indexRelation, int nkeys, int norderbys)

入力と出力

  • **入力**:
  • indexRelation: スキャン対象のインデックスリレーション。
  • nkeys: スキャンキーの数。
  • norderbys: オーダーバイ句の数。
  • **出力**:
  • 初期化済みの IndexScanDesc 構造体へのポインタ。

前提条件と不変条件

  • nkeys または norderbys が 0 より大きい場合、対応するワークスペース（keyData または orderByData）がメモリ確保される。
  • xs_snapshot は InvalidSnapshot で初期化されるため、呼び出し元で別途設定する必要がある。
  • リカバリ実行中に開始されたトランザクションであるかどうかが xactStartedInRecovery に記録され、それに基づき ignore_killed_tuples が設定される。

アルゴリズムまたは振る舞い

1. IndexScanDescData サイズのメモリを palloc で確保する。 2. インデックスリレーション、キー数、オーダーバイ数を設定する。 3. nkeys および norderbys の数に応じて、スキャンキー用のメモリ領域を確保する（0以下の場合は NULL）。 4. リカバリ状態を確認し、xactStartedInRecovery を設定する。リカバリ中でない場合は ignore_killed_tuples を true に設定する。 5. その他のポインタメンバ（heapRelation, opaque, instrument 等）を NULL で初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blscan.c
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/ginscan.c
  • src/backend/access/gist/gistscan.c
  • src/backend/access/hash/hash.c
  • **呼び出し先**:
  • palloc (メモリ確保)
  • TransactionStartedDuringRecovery (リカバリ状態判定)

関連データ構造

  • IndexScanDescData: 本関数が生成するスキャン記述子の実体。
  • Relation: インデックスリレーションの定義。
  • ScanKeyData: スキャンキーのデータ構造。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • nkeys や norderbys に負の値を指定した場合の挙動は定義されていない。

保守担当者向けメモ

  • 本関数は IndexScanDesc のメモリ確保と基本設定のみを行う。AM固有の初期化（amrescan 等）は呼び出し元で行う必要がある。
  • ignore_killed_tuples のロジックはリカバリ中の MVCC 整合性を維持するために重要であるため、変更時は注意が必要。
