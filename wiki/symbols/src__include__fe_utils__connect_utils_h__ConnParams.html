ConnParams

定義

  • 種別: struct
  • 定義箇所: src/include/fe_utils/connect_utils.h:25

目的

PostgreSQLのフロントエンドツールがデータベース接続を確立する際に必要なパラメータを一元管理し、接続ロジックの共通化と簡素化を図るための構造体です。

シグネチャまたは形状

typedef struct _connParams
{
    const char *dbname;
    const char *pghost;
    const char *pgport;
    const char *pguser;
    enum trivalue prompt_password;
    const char *override_dbname;
} ConnParams;

入力と出力

  • **入力**: 接続先情報（ホスト、ポート、ユーザー、DB名）およびパスワードプロンプトの制御フラグ。
  • **出力**: 保持されたパラメータは connectDatabase() 等の関数に渡され、libpq の接続処理に使用されます。

前提条件と不変条件

  • dbname フィールドには、単なるデータベース名だけでなく、接続文字列（URI形式など）を指定可能です。
  • override_dbname が NULL でない場合、コマンドライン引数等で指定された dbname よりも優先して使用されます（ただし、接続文字列内の他のパラメータは上書きされません）。

アルゴリズムまたは振る舞い

1. 呼び出し元が各フィールドを設定し、connectDatabase() に渡します。 2. prompt_password が TRI_YES の場合、必要に応じて対話的なパスワード入力処理が実行されます。 3. 接続試行時には PQconnectdbParams が呼び出されます。 4. 接続失敗時には、パスワード再入力等の再試行ロジックが適用される場合があります。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_amcheck, pg_dump, clusterdb, createdb 等のフロントエンドツール。
  • **呼び出し先**: connectDatabase() および connectMaintenanceDatabase() 関数が本構造体を引数として受け取ります。

関連データ構造

  • trivalue: パスワードプロンプトの挙動（デフォルト、強制無効、強制有効）を制御する列挙型。

失敗モードと境界条件

  • dbname に接続文字列が含まれる場合、その解析は libpq の仕様に依存します。
  • 接続後のリソース解放（PQfinish 等）は本構造体ではなく、呼び出し側の責任で行う必要があります。

保守担当者向けメモ

  • allow_password_reuse フラグ（構造体外の関連ロジック）を扱う際は、セキュリティ上のリスクを考慮してください。
  • dbname が接続文字列である可能性を常に考慮し、文字列操作を行う際は注意が必要です。
