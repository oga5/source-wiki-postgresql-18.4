SplitDirectoriesString

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varlena.c:3707

目的

区切り文字で分割されたディレクトリパスのリストを解析し、各パスを正規化してリストに格納する。

シグネチャまたは形状

bool SplitDirectoriesString(char *rawstring, char separator, List **namelist)

入力と出力

  • **入力**:
  • rawstring: 解析対象の文字列（破壊的に変更される）。
  • separator: パスを区切る文字。
  • namelist: 解析結果のパスリストを格納するポインタ。
  • **出力**:
  • 成功時は true を返し、namelist に pstrdup されたパスのリストが格納される。
  • 失敗時は false を返す。

前提条件と不変条件

  • rawstring は呼び出し元で変更可能なメモリ領域である必要がある。
  • namelist は初期化済み（NIL）のリストポインタを指すこと。
  • 引用符（"）で囲まれたパス内の連続する引用符は、単一の引用符として扱われる。
  • パスが MAXPGPATH を超える場合は切り詰められる。
  • 各パスは canonicalize_path によって正規化される。

アルゴリズムまたは振る舞い

1. 先頭の空白をスキップする。空文字列の場合は即座に true を返す。 2. 文字列をループで走査し、各パスを抽出する。

  • 引用符で囲まれている場合、終了引用符までをパスとして認識し、内部の引用符ペアをエスケープ解除する。
  • 引用符がない場合、区切り文字または文字列終端までをパスとして認識する。

    • 3. 抽出したパスの末尾の空白を除去し、\0 で終端する。 4. パス長が MAXPGPATH 以上であれば切り詰める。 5. pstrdup でメモリを確保し、canonicalize_path を適用後、namelist に追加する。 6. 区切り文字が見つかった場合は次のパスの解析へ進み、終端に達するまで繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth-oauth.c
  • src/backend/postmaster/postmaster.c
  • src/backend/utils/init/miscinit.c
  • **呼び出し先**:
  • pstrdup
  • canonicalize_path
  • lappend
  • scanner_isspace
  • strchr
  • memmove

関連データ構造

  • List: PostgreSQLのリスト構造（namelist に使用）。

失敗モードと境界条件

  • 引用符が閉じられていない場合（endp == NULL）は false を返す。
  • 引用符なしのパスが空である場合（curname == endp）は false を返す。
  • 期待される区切り文字の後に不正な文字がある場合は false を返す。

保守担当者向けメモ

  • 本関数は rawstring を直接書き換えるため、呼び出し元でコピーを渡す必要がある（auth-oauth.c 等での実装例を参照）。
  • MAXPGPATH による切り詰めは、パスの整合性を損なう可能性があるため注意が必要。
