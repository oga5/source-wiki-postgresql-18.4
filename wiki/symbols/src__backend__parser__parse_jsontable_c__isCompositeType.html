isCompositeType

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_jsontable.c:374

目的

与えられた型OIDが、JSONデータ型、レコード型、配列型、複合型、またはそれらを基底とするドメイン型であるかを判定します。主に JSON_TABLE 構文解析において、対象の型を JSON_VALUE で扱うか JSON_QUERY で扱うかを決定するために使用されます。

シグネチャまたは形状

static bool isCompositeType(Oid typid)

入力と出力

  • **入力**: typid (Oid) - 判定対象の型OID。
  • **出力**: bool - 判定対象が複合的な構造を持つ型であれば true、それ以外は false。

前提条件と不変条件

  • get_typtype および type_is_array、getBaseType といったシステムカタログ操作関数が利用可能であること。
  • ドメイン型に対しては再帰的に基底型を評価するため、型定義の循環参照がないことが前提となります。

アルゴリズムまたは振る舞い

1. get_typtype(typid) を呼び出し、型の分類を取得します。 2. 以下のいずれかに該当する場合に true を返します。

  • JSONOID または JSONBOID である。
  • RECORDOID である。
  • type_is_array(typid) が真である。
  • typtype が TYPTYPE_COMPOSITE である。
  • typtype が TYPTYPE_DOMAIN であり、かつその基底型に対して isCompositeType を再帰的に呼び出した結果が真である。

    • 3. 上記のいずれにも該当しない場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_jsontable.c 内の解析ロジック（JSON_TABLE カラム処理時）。
  • src/backend/commands/typecmds.c（型定義コマンド関連のチェック）。
  • **呼び出し先**:
  • get_typtype
  • type_is_array
  • getBaseType
  • isCompositeType（自身を再帰呼び出し）

関連データ構造

  • Oid: PostgreSQLの型識別子。
  • TYPTYPE_COMPOSITE, TYPTYPE_DOMAIN: 型の分類を示す定数。

失敗モードと境界条件

  • 無効な typid が渡された場合、内部で呼び出される get_typtype 等のカタログアクセス関数がエラーを発生させる可能性があります。
  • ドメイン型の深いネスト構造に対しては、スタックオーバーフローのリスクが理論上存在します。

保守担当者向けメモ

  • 本関数は JSON_TABLE の型判定ロジックの核となるため、新しいJSON関連型や複合型が追加された場合は判定条件の更新が必要です。
  • src/backend/commands/typecmds.c からも参照されているため、変更時は影響範囲に注意してください。
