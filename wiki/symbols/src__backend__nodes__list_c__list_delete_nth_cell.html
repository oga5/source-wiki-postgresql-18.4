list_delete_nth_cell

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/list.c:766

目的

リストの指定されたインデックス（n）にある要素を削除し、リストの整合性を維持した状態でリストを返します。

シグネチャまたは形状

List *list_delete_nth_cell(List *list, int n)

入力と出力

  • **入力**:
  • list: 操作対象の List 構造体へのポインタ。
  • n: 削除対象の要素のインデックス（0から始まる）。
  • **出力**:
  • 削除後の List 構造体へのポインタ。リストが空になった場合は NIL を返します。

前提条件と不変条件

  • n は 0 以上かつ list->length 未満である必要があります（Assert により検証）。
  • 操作の前後で check_list_invariants(list) が呼び出され、リストの整合性が検証されます。

アルゴリズムまたは振る舞い

1. **初期チェック**: list->length が 1 の場合、list_free(list) を呼び出してメモリを解放し、NIL を返します。 2. **要素削除**:

  • 通常時（DEBUG_LIST_MEMORY_USAGE 未定義）: memmove を使用して、削除対象インデックス以降の要素を前方にシフトし、list->length をデクリメントします。
  • デバッグ時: 新しいメモリ領域を確保し、削除対象を除いた要素をコピーした後、古い領域を解放（または無効化）します。

    • 3. **戻り値**: 変更後のリストポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c
  • src/backend/commands/tablecmds.c
  • src/backend/nodes/list.c (内部的な再帰・補助呼び出し)
  • src/backend/optimizer/path/equivclass.c
  • src/backend/optimizer/path/joinpath.c
  • src/include/nodes/pg_list.h (マクロ経由)
  • **呼び出し先**:
  • check_list_invariants
  • list_free
  • memmove / memcpy
  • MemoryContextAlloc / pfree (デバッグ時)

関連データ構造

  • List: リストのヘッダ構造体。
  • ListCell: リストの要素を保持する配列の型。

失敗モードと境界条件

  • n が範囲外の場合、Assert によりプログラムが停止します。
  • リストの最後の要素を削除した場合、リスト自体が解放され NIL が返されます。

保守担当者向けメモ

  • 本関数は配列ベースのリスト操作を行うため、要素の削除に伴いメモリ上の要素シフトが発生します。
  • DEBUG_LIST_MEMORY_USAGE が有効な場合、メモリ再配置が強制されるため、ポインタの安定性に依存するコードは避けてください。
