parse_slash_copy

定義

  • 種別: function
  • 定義箇所: src/bin/psql/copy.c:88

目的

psql の \copy メタコマンドの引数文字列を解析し、実行に必要な設定情報を copy_options 構造体に格納する。

シグネチャまたは形状

static struct copy_options *parse_slash_copy(const char *args);

入力と出力

  • **入力**: args (解析対象の \copy コマンド引数文字列)
  • **出力**: 解析結果を保持する copy_options 構造体へのポインタ。解析失敗時は NULL を返す。

前提条件と不変条件

  • args が NULL の場合、エラーログを出力して NULL を返す。
  • copy_options 構造体は pg_malloc0 でゼロ初期化される。
  • PROGRAM オプションを使用する場合、コマンド文字列はシングルクォートで囲まれている必要がある。

アルゴリズムまたは振る舞い

1. strtokx を用いてトークン単位で引数を走査する。 2. binary キーワード、括弧で囲まれたクエリ、スキーマ・テーブル名、カラムリストを順次解析し、before_tofrom バッファに構築する。 3. FROM または TO キーワードを識別し、result->from フラグを設定する。 4. ファイル名、PROGRAM、STDIN/STDOUT、PSTDIN/PSTDOUT のいずれかを判定し、result->file 等に格納する。 5. 残りの文字列を after_tofrom として取得する。 6. 解析中にトークンが不足した場合は error ラベルへ遷移し、リソースを解放して NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: do_copy (src/bin/psql/copy.c:276)
  • **呼び出し先**: strtokx, pg_malloc0, pg_strdup, xstrcat, strip_quotes, expand_tilde, pg_strcasecmp, free_copy_options, pg_log_error

関連データ構造

  • copy_options: 解析結果を保持する構造体。

失敗モードと境界条件

  • 引数が空、または構文が不正な場合は pg_log_error でメッセージを出力し、free_copy_options を呼び出してメモリリークを防ぐ。
  • PROGRAM 指定時にクォートが正しくない場合、解析エラーとして処理される。

保守担当者向けメモ

  • strtokx の呼び出し順序は、\copy の構文規則に厳密に従っているため、構文拡張を行う際はトークン解析ロジックの整合性に注意すること。
  • before_tofrom への文字列追加には xstrcat を使用しており、メモリ管理がこの関数内で完結している。
