bbsink_zstd_begin_archive

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/backup/basebackup_zstd.c:157

目的

Zstandard圧縮を用いたバックアップアーカイブの開始処理を行います。圧縮セッションのリセットと、後続のシンクに対するアーカイブ開始の通知を担います。

シグネチャまたは形状

static void bbsink_zstd_begin_archive(bbsink *sink, const char *archive_name)

入力と出力

  • **入力**:
  • sink: bbsink 構造体へのポインタ。実際には bbsink_zstd 型として扱われます。
  • archive_name: アーカイブのファイル名。
  • **出力**: なし（副作用として圧縮状態の初期化と後続シンクへの通知を行います）。

前提条件と不変条件

  • sink は bbsink_zstd 型のインスタンスである必要があります。
  • sink->bbs_next が存在し、後続のシンクが有効である必要があります。

アルゴリズムまたは振る舞い

1. ZSTD_CCtx_reset を呼び出し、ZSTD_reset_session_only オプションを用いて圧縮コンテキストをリセットします（既存の圧縮パラメータは保持されます）。 2. mysink->zstd_outBuf を、後続シンクのバッファ (bbs_next->bbs_buffer) を参照するように設定します。 3. archive_name に .zst 拡張子を付与した文字列を生成します。 4. 後続シンクの begin_archive を呼び出し、圧縮後のファイル名を渡してアーカイブ処理を開始させます。 5. 生成した一時的なファイル名文字列を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_zstd_ops 構造体を通じて、バックアップパイプラインから間接的に呼び出されます。
  • **呼び出し先**:
  • ZSTD_CCtx_reset (libzstd)
  • bbsink_begin_archive (後続のシンク)

関連データ構造

  • bbsink_zstd: 圧縮状態を保持する構造体。
  • ZSTD_CCtx: libzstd の圧縮コンテキスト。
  • bbsink: ベースとなるシンクインターフェース。

失敗モードと境界条件

  • sink->bbs_next が NULL の場合、Assert により停止します。
  • メモリ不足により psprintf が失敗した場合、PostgreSQLの標準的なエラーハンドリングによりアボートします。

保守担当者向けメモ

  • 本関数は ZSTD_reset_session_only を使用しているため、圧縮レベルやスレッド数などの設定は bbsink_zstd_new で設定されたものが維持されます。
  • アーカイブ名の変更（.zst の付与）は、このレイヤーで行われるため、後続のシンクは圧縮済みファイルとして処理を行う前提となります。
