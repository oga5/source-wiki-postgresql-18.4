SetPipelineVariables

定義

  • 種別: function
  • 定義箇所: src/bin/psql/common.c:533

目的

現在のパイプライン処理状態（同期回数、コマンド数、利用可能な結果数）を psql の内部変数として設定し、ユーザーがスクリプトや表示で参照可能にする。

シグネチャまたは形状

static void SetPipelineVariables(void)

入力と出力

  • **入力**: グローバル変数 pset（pset.piped_syncs, pset.piped_commands, pset.available_results）。
  • **出力**: なし（副作用として pset.vars が更新される）。

前提条件と不変条件

  • pset.vars が初期化されていること。
  • パイプライン処理に関連するカウンタ値が pset 構造体に正しく保持されていること。

アルゴリズムまたは振る舞い

1. pset.piped_syncs を文字列に変換し、変数 PIPELINE_SYNC_COUNT に格納する。 2. pset.piped_commands を文字列に変換し、変数 PIPELINE_COMMAND_COUNT に格納する。 3. pset.available_results を文字列に変換し、変数 PIPELINE_RESULT_COUNT に格納する。 4. 各変数の設定には SetVariable を使用する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/common.c 内の複数箇所（行 1709, 1722, 2184）。
  • **呼び出し先**: SetVariable（src/bin/psql/variables.c で定義）。

関連データ構造

  • pset（struct _psqlSettings）: パイプライン状態を保持するグローバル設定構造体。

失敗モードと境界条件

  • snprintf によるバッファサイズ（32バイト）を超える数値がカウンタに格納されている場合、文字列変換が切り詰められる可能性がある。

保守担当者向けメモ

  • 本関数はパイプラインモードのステータスを可視化するために使用されます。新しいパイプライン関連の統計情報を追加する場合は、本関数で変数を追加してください。
