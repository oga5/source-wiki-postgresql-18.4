page_checksum_internal

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/rawpage.c:341

目的

指定されたページデータ（bytea）とブロック番号に基づき、ページチェックサムを計算して返す。

シグネチャまたは形状

static Datum page_checksum_internal(PG_FUNCTION_ARGS, enum pageinspect_version ext_version)

入力と出力

  • **入力**:
  • PG_FUNCTION_ARGS: SQL関数呼び出しの引数コンテキスト。
  • ext_version: pageinspect 拡張機能のバージョン（PAGEINSPECT_V1_8 または PAGEINSPECT_V1_9）。
  • **出力**:
  • Datum: 計算されたチェックサム値（int16）。ページが新規（PageIsNew）の場合は NULL を返す。

前提条件と不変条件

  • 呼び出し元はスーパーユーザーである必要がある。
  • ブロック番号（blkno）は 0 以上 MaxBlockNumber 以下である必要がある。

アルゴリズムまたは振る舞い

1. **権限確認**: superuser() を呼び出し、権限がない場合はエラーを報告する。 2. **引数取得**: ext_version に応じてブロック番号を解析する（V1.8なら uint32、それ以外は int64）。 3. **ページ変換**: get_page_from_raw を呼び出し、入力された bytea をメモリ上でアライメントされた Page 構造体に変換する。 4. **新規ページ判定**: PageIsNew(page) が真であれば NULL を返す。 5. **計算**: pg_checksum_page(page, blkno) を呼び出し、結果を int16 として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pageinspect/rawpage.c 内の page_checksum_1_9 および page_checksum 関数。
  • **呼び出し先**:
  • superuser()
  • get_page_from_raw()
  • PageIsNew()
  • pg_checksum_page()

関連データ構造

  • Page: ページデータのメモリ表現。
  • bytea: 入力される生のページデータ。
  • pageinspect_version: 拡張機能のバージョンを識別する列挙型。

失敗モードと境界条件

  • **権限不足**: スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー。
  • **無効なブロック番号**: blkno < 0 または blkno > MaxBlockNumber の場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー。
  • **新規ページ**: ページが初期化されていない場合、チェックサム計算を行わず NULL を返す。

保守担当者向けメモ

  • 本関数は pageinspect 拡張機能のバージョン差異を吸収する役割を持つ。新しいバージョンが追加された際は、ext_version の条件分岐を更新する必要がある。
