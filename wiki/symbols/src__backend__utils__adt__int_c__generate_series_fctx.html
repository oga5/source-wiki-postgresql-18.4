generate_series_fctx

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/int.c:47

目的

generate_series 関数が呼び出し間で保持すべき状態（コンテキスト）を格納するための構造体です。

シグネチャまたは形状

typedef struct
{
    int32       current;
    int32       finish;
    int32       step;
} generate_series_fctx;

入力と出力

本構造体は関数呼び出し間の状態保持に使用されます。

  • current: 現在生成中の値。
  • finish: 生成を終了する上限値。
  • step: 各ステップでの増分値。

前提条件と不変条件

  • FuncCallContext を介して永続化されることを前提としています。
  • int32 型の範囲内での演算を想定しています。

アルゴリズムまたは振る舞い

  • generate_series 関数の実行中、この構造体は palloc によってメモリ確保され、各イテレーションで current が更新されます。
  • 呼び出しごとに current が finish を超えたかどうかが判定され、集合生成の継続または終了が決定されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/int.c および src/backend/utils/adt/int8.c 内の generate_series 実装関数。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • FuncCallContext: PostgreSQL のセットリターニング関数（SRF）において、呼び出し間で状態を保持するために本構造体を格納するコンテナ。

失敗モードと境界条件

  • メモリ不足: palloc 失敗時にエラーが発生します。
  • オーバーフロー: current に step を加算する際、int32 の範囲を超える可能性があります（算術演算時のオーバーフローチェックについては int.c のモジュール概要を参照）。

保守担当者向けメモ

  • int8.c でも同名の構造体が定義されています。修正を行う際は、int.c と int8.c の両方の実装に影響がないか確認してください。
  • 本構造体は int32 専用です。int8 型の範囲を扱う場合は int8.c 側の定義が使用されます。
