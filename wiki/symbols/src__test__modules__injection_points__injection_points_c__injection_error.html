injection_error

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_points.c:244

目的

テスト実行中に特定のコードパスで意図的にエラーを発生させ、エラーハンドリングの動作を検証するためのコールバック関数です。

シグネチャまたは形状

void injection_error(const char *name, const void *private_data, void *arg)

入力と出力

  • name: インジェクションポイントの識別名。
  • private_data: InjectionPointCondition 構造体へのポインタ。実行条件の判定に使用されます。
  • arg: エラーメッセージに含める追加情報（文字列）を指すポインタ。
  • 出力: なし（エラー発生時は elog(ERROR, ...) によりプロセスが中断されます）。

前提条件と不変条件

  • private_data は InjectionPointCondition 型としてキャスト可能である必要があります。
  • injection_point_allowed による条件判定が真（true）である場合にのみ、エラーがトリガーされます。

アルゴリズムまたは振る舞い

1. injection_point_allowed(condition) を呼び出し、現在の実行コンテキストでエラーを発生させるべきか判定します。偽の場合は即座に終了します。 2. pgstat_report_inj(name) を呼び出し、インジェクションポイントの実行統計を記録します。 3. arg が提供されている場合、その内容を付加したエラーメッセージと共に elog(ERROR, ...) を発行します。 4. arg が提供されていない場合、標準的なエラーメッセージと共に elog(ERROR, ...) を発行します。

呼び出し元と呼び出し先

  • 呼び出し元: INJECTION_POINT() マクロが展開された箇所から、インジェクションポイントの実行エンジンを介して呼び出されます。
  • 呼び出し先:
  • injection_point_allowed (条件判定)
  • pgstat_report_inj (統計記録)
  • elog (エラー送出)

関連データ構造

  • InjectionPointCondition: 実行条件を保持する構造体。

失敗モードと境界条件

  • private_data が無効なポインタである場合、セグメンテーション違反が発生する可能性があります。
  • arg が文字列として適切に終端されていない場合、elog の出力時にメモリ読み取りエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はテスト専用です。本番環境のコードパスに影響を与えないよう注意してください。
  • elog(ERROR, ...) を使用しているため、呼び出し元のトランザクションはアボートされます。テストケース側で EXCEPTION 句などを用いた適切なキャッチが必要です。
