LruDelete

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/fd.c:1288

目的

LRU（Least Recently Used）リングから指定された仮想ファイル記述子（VFD）を削除し、対応するOSのファイル記述子（FD）をクローズしてリソースを解放する。

シグネチャまたは形状

static void LruDelete(File file)

入力と出力

  • **入力**: file (型: File) - 削除対象のVFDインデックス。
  • **出力**: なし（副作用としてファイルクローズとLRUリングの更新を行う）。

前提条件と不変条件

  • file != 0 であること（VfdCache[0] はLRUリングのアンカーであるため）。
  • 指定された file に対応するVFDが有効なFDを保持していること。

アルゴリズムまたは振る舞い

1. pgaio_closing_fd を呼び出し、非同期I/Oに関連するクローズ処理を行う。 2. OSの close(2) を呼び出し、物理的なファイル記述子を解放する。 3. vfdP->fd を VFD_CLOSED に更新し、現在開いているファイル数を示すグローバル変数 nfile をデクリメントする。 4. Delete(file) を呼び出し、内部のLRUリング構造から当該VFDレコードを切り離す。

  • close が失敗した場合、エラーログを出力するが、内部状態の整合性を優先して処理を継続する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/file/fd.c 内のキャッシュ管理ロジック（ReleaseLruFiles 等）から、キャッシュ溢れ時に呼び出される。
  • その他、ファイル削除やリソース解放の過程で呼び出される。
  • **呼び出し先**:
  • pgaio_closing_fd
  • close (システムコール)
  • Delete (内部関数)
  • elog (エラーログ出力)

関連データ構造

  • Vfd: ファイルの状態を保持する構造体。
  • VfdCache: VFDを管理するグローバル配列。
  • nfile: 現在開いているファイル数のカウンタ。

失敗モードと境界条件

  • file == 0 の場合、Assert により異常終了する。
  • close システムコールが失敗した場合、ログを出力するが、ファイル記述子のリークを許容して処理を続行する。

保守担当者向けメモ

  • 本関数はLRUリングの物理的なクローズを担当する低レベル関数です。
  • VfdCache[0] はリングの番兵であるため、決して削除対象にしてはなりません。
  • nfile の管理と Delete() によるリング操作の順序は、キャッシュの整合性に直結するため注意してください。
