relopt_value

定義

  • 種別: struct
  • 定義箇所: src/include/access/reloptions.h:76

目的

ユーザーが指定したリレーションオプション（pg_class.reloptions 等）を解析し、その値とメタデータを保持するために使用される構造体です。

シグネチャまたは形状

typedef struct relopt_value
{
	relopt_gen *gen;
	bool		isset;
	union
	{
		bool		bool_val;
		int			int_val;
		double		real_val;
		int			enum_val;
		char	   *string_val; /* 個別に割り当てられる */
	}			values;
} relopt_value;

入力と出力

  • **入力**: build_reloptions 等の関数において、解析対象のオプション定義（relopt_gen）と、ユーザーから提供された値がこの構造体に格納されます。
  • **出力**: 解析済みの値は、各アクセスメソッドが動作を動的に変更するためのパラメータとして利用されます。

前提条件と不変条件

  • gen フィールドは、対応するオプションの定義（型やデフォルト値など）を指す必要があります。
  • string_val を使用する場合、そのメモリは個別に割り当てられるため、ライフサイクル管理に注意が必要です。
  • isset は、オプションが明示的に設定された場合に true となります。

アルゴリズムまたは振る舞い

1. **解析**: parse_one_reloption を通じて、文字列形式のオプションがこの構造体に変換されます。 2. **格納**: オプションの型に応じて、共用体 values の適切なフィールドに値が格納されます。 3. **変換**: build_reloptions は、この構造体の配列を生成し、最終的に C 構造体（StdRdOptions 等）へマッピングするための基盤となります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/reloptions.c: parse_one_reloption, build_reloptions, allocateReloptStruct 等で頻繁に操作されます。
  • contrib/ltree/ltree_gist.c: ltree_gist_relopts_validator で参照されます。
  • **呼び出し先**:
  • この構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • relopt_gen: オプションの共通ヘッダー。
  • relopt_bool, relopt_int, relopt_real, relopt_enum, relopt_string: 各型に対応する定義構造体。

失敗モードと境界条件

  • string_val のメモリ割り当て失敗時、または不正な型変換が発生した場合の挙動は、呼び出し元の解析関数（parse_one_reloption 等）の検証ロジックに依存します。
  • 構造体配列のサイズ（numoptions）が正しく管理されていない場合、バッファオーバーフローのリスクがあります。

保守担当者向けメモ

  • string_val は個別にメモリが割り当てられるため、この構造体を含む配列を解放する際は、文字列のメモリリークが発生しないよう注意してください。
  • 新しいオプション型を追加する場合は、共用体 values にフィールドを追加し、解析ロジックを更新する必要があります。
