xl_invalid_page_key

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/xlogutils.c:65

目的

WALリプレイ中に発生した「存在しないページへの参照」を追跡するためのハッシュテーブル用キー構造体です。

シグネチャまたは形状

typedef struct xl_invalid_page_key
{
    RelFileLocator locator;     /* 対象リレーション */
    ForkNumber   forkno;        /* フォーク番号 */
    BlockNumber  blkno;         /* ページ番号 */
} xl_invalid_page_key;

入力と出力

本構造体はデータ構造のキーとして機能します。

  • **入力**: リレーションの識別子（locator）、フォーク番号、ブロック番号。
  • **出力**: ハッシュテーブルの検索や挿入に使用される一意なキー値。

前提条件と不変条件

  • 構造体内にパディングが含まれないことが前提とされています（xlogutils.c:145 のコメントによる）。
  • リカバリプロセス中のみ使用されることを想定しています。

アルゴリズムまたは振る舞い

  • invalid_page_tab ハッシュテーブルにおいて、無効なページ参照を特定するための識別子として使用されます。
  • リカバリ中にページ読み込みが失敗した場合、このキーを用いてハッシュテーブルにエントリが追加されます。
  • リカバリ終了時に、このキーで管理されるエントリが残存していないか検証されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlogutils.c 内の無効ページ管理関数群。
  • **呼び出し先**: 直接的な関数呼び出しはありませんが、hash_create 等のハッシュテーブル操作関数において sizeof(xl_invalid_page_key) がキーサイズとして渡されます。

関連データ構造

  • xl_invalid_page: このキーに対応する値を保持する構造体。
  • invalid_page_tab: 本構造体をキーとして使用するハッシュテーブル。

失敗モードと境界条件

  • 構造体内にパディングが含まれる場合、ハッシュテーブルの検索や比較処理で予期せぬ不整合が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は invalid_page_tab のキーとしてのみ利用されます。フィールドの追加や変更を行う際は、ハッシュテーブルのキーサイズ計算（ctl.keysize）に影響がないか確認してください。
