ExecTidRangeScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTidrangescan.c:305

目的

TID（物理タプル位置）の範囲指定に基づいたテーブルスキャンを実行する。

シグネチャまたは形状

static TupleTableSlot *ExecTidRangeScan(PlanState *pstate)

入力と出力

  • **入力**: pstate (PlanState *) - 実行中のTID範囲スキャンノードの状態。
  • **出力**: TupleTableSlot * - スキャンによって取得された次のタプル、またはスキャン終了時は NULL。

前提条件と不変条件

  • pstate は TidRangeScanState 型にキャスト可能であること。
  • スキャン対象のリレーションは初期化済みであること。

アルゴリズムまたは振る舞い

1. pstate を TidRangeScanState にキャストする。 2. ExecScan を呼び出し、スキャン処理を委譲する。

  • スキャン実行メソッドとして TidRangeNext を使用する。
  • 再チェックメソッドとして TidRangeRecheck を使用する。

    • 3. ExecScan が返した TupleTableSlot をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（ExecProcNode 経由で呼び出される）。
  • **呼び出し先**:
  • ExecScan（共通基盤）
  • TidRangeNext（スキャン実行用）
  • TidRangeRecheck（再チェック用）

関連データ構造

  • TidRangeScanState: 本ノードの実行状態を保持する構造体。
  • PlanState: 実行計画ノードの共通状態。
  • TupleTableSlot: タプルを保持するスロット。

失敗モードと境界条件

  • 範囲が空と判定された場合、TidRangeNext を通じて即座にスキャン終了（NULL）が返される。
  • テーブルアクセス層（Table AM）が提供する table_scan_getnextslot_tidrange が失敗または終了した場合、それに準ずる。

保守担当者向けメモ

  • 本関数は ExecScan を呼び出すラッパーである。詳細なスキャンロジックは TidRangeNext に依存している。
  • TidRangeScanState の ss.ps.ExecProcNode に本関数が割り当てられることで、実行エンジンから呼び出される。
