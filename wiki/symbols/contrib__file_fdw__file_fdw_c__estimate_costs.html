estimate_costs

定義

  • 種別: function
  • 定義箇所: contrib/file_fdw/file_fdw.c:1143

目的

外部テーブルのプランニングにおいて、シーケンシャルスキャンを想定したコスト（起動コストおよび総コスト）を算出する。

シグネチャまたは形状

static void estimate_costs(PlannerInfo *root, RelOptInfo *baserel,
                           FileFdwPlanState *fdw_private,
                           Cost *startup_cost, Cost *total_cost)

入力と出力

  • root: プランナのグローバル情報。
  • baserel: 対象となる外部リレーションの情報。
  • fdw_private: 推定されたページ数およびタプル数を含む状態構造体。
  • startup_cost (出力): 算出された起動コスト。
  • total_cost (出力): 算出された総コスト。

前提条件と不変条件

  • fdw_private には、事前に計算された pages および ntuples が格納されている必要がある。
  • 外部ソースがファイルかプログラムかにかかわらず、本関数はシーケンシャルスキャンに基づくコスト計算を行う。

アルゴリズムまたは振る舞い

1. **I/Oコストの算出**: seq_page_cost に fdw_private->pages を乗じて加算する。 2. **起動コストの設定**: baserel->baserestrictcost.startup をそのまま startup_cost に代入する。 3. **CPUコストの算出**: 通常のシーケンシャルスキャンの10倍の cpu_tuple_cost を適用し、レコード解析コストを考慮する。これに baserel->baserestrictcost.per_tuple を加えた値をタプル数分乗算する。 4. **総コストの算出**: 起動コストと実行コストの合計を total_cost に設定する。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/file_fdw/file_fdw.c 内のプランニング処理（fileGetForeignRelSize 等の文脈）。
  • 呼び出し先: 直接的な関数呼び出しは含まないが、baserel 構造体のメンバを参照する。

関連データ構造

  • PlannerInfo: プランナのコンテキスト。
  • RelOptInfo: リレーションの最適化情報。
  • FileFdwPlanState: file_fdw 特有のプランニング状態（ページ数、タプル数）。

失敗モードと境界条件

  • 外部ソースがプログラムの場合、実際のコストと乖離する可能性があるが、本関数は代替手段がないため一律の計算式を適用する。
  • 外部リレーションに対してアクセスパスが単一であるため、本関数によるコスト計算の精度がプラン選択に与える影響は限定的である。

保守担当者向けメモ

  • 本関数は cost_seqscan() のロジックを模倣している。COPY コマンドの解析コストを考慮して CPU コストを 10 倍に設定している点に注意すること。
  • 統計情報の精度は fdw_private の内容に依存するため、コスト計算の改善が必要な場合は fdw_private を生成する箇所を確認すること。
