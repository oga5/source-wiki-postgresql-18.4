process_secret_key

定義

  • contrib/pgcrypto/pgp-pubkey.c:336

目的

OpenPGP 形式の秘密鍵パケットを解析し、メモリ上の PGP_PubKey 構造体へ読み込みます。必要に応じて、提供されたパスワードを用いて S2K（String-to-Key）変換および CFB モードによる復号を行います。

シグネチャまたは形状

static int process_secret_key(PullFilter *pkt, PGP_PubKey **pk_p, const uint8 *key, int key_len);

入力と出力

  • pkt: 読み込み対象の PullFilter。
  • pk_p: 成功時に構築された PGP_PubKey 構造体へのポインタが格納される出力先。
  • key: 復号用パスワード（バイト列）。
  • key_len: パスワードの長さ。
  • 戻り値: 成功時は 0、失敗時は負の PXE_* エラーコードを返します。

前提条件と不変条件

  • pkt は有効な PGP パケットストリームを指している必要があります。
  • 秘密鍵が暗号化されている場合（HIDE_SHA1 または HIDE_CKSUM）、key は NULL であってはなりません。
  • 処理終了後、エラーが発生した場合は内部で pgp_key_free が呼び出され、メモリが解放されます。

アルゴリズムまたは振る舞い

1. _pgp_read_public_key を呼び出し、公開鍵部分を先に解析します。 2. 秘密鍵の隠蔽タイプ（hide_type）を確認します。

  • 暗号化されている場合: pgp_s2k_read で S2K パラメータを読み込み、pgp_s2k_process で鍵を導出します。その後、pgp_cfb_create で復号フィルタを作成し、ストリームに適用します。
  • 平文の場合: そのまま読み込みます。

    • 3. 鍵のアルゴリズム（RSA, ELG, DSA）に応じて、MPI（多倍長整数）を読み込みます。 4. 最後に整合性チェック（SHA1 またはチェックサム）を行い、パケットの終端を確認します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/pgp-pubkey.c 内の関数（internal_read_key 等からの参照が示唆されています）。
  • 呼び出し先: _pgp_read_public_key, pgp_s2k_read, pgp_s2k_process, pgp_get_cipher_block_size, pullf_read_fixed, pgp_cfb_create, pullf_create, pgp_mpi_read, check_key_sha1, check_key_cksum, pgp_expect_packet_end, pullf_free, pgp_cfb_free, pgp_key_free。

関連データ構造

  • PGP_PubKey: 鍵情報を保持する主要構造体。
  • PGP_S2K: S2K 変換設定。
  • PGP_CFB: CFB モード復号用コンテキスト。
  • PullFilter: 入力ストリーム制御用。

失敗モードと境界条件

  • PXE_PGP_NEED_SECRET_PSW: 暗号化された鍵に対してパスワードが提供されなかった場合。
  • PXE_PGP_UNSUPPORTED_CIPHER: 未知の暗号アルゴリズムが指定された場合。
  • PXE_PGP_KEYPKT_CORRUPT: 隠蔽タイプが不明、またはアルゴリズムごとの MPI 読み込みや整合性チェックに失敗した場合。

保守担当者向けメモ

  • 本関数は PGP バージョン 4 パケット形式に依存しています。
  • 復号フィルタ（pf_decrypt）の解放は、関数終了時に適切に行われるよう設計されています。
  • MPI の読み込み順序は PGP 仕様に厳密に従っているため、構造体定義の変更時には注意が必要です。
