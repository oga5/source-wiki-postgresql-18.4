coerce_function_result_tuple

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_exec.c:823

目的

PL/pgSQL 関数が返す複合型（レコード）の結果を、呼び出し元が期待する型定義（TupleDesc）に合わせて変換・適合させる。

シグネチャまたは形状

static void coerce_function_result_tuple(PLpgSQL_execstate *estate, TupleDesc tupdesc)

入力と出力

  • **入力**:
  • estate: 実行状態を保持する PLpgSQL_execstate 構造体。estate->retval に変換対象の Datum が格納されている。
  • tupdesc: 呼び出し元が期待する結果の型定義。
  • **出力**:
  • estate->retval: 変換後の Datum で上書きされる。

前提条件と不変条件

  • estate->rettype が複合型（rowtype）であることを前提とする（exec_stmt_return による検証済み）。
  • 展開済みレコード（Expanded Record）の場合、er_magic が ER_MAGIC であること。

アルゴリズムまたは振る舞い

1. **展開済みレコードの処理**:

  • estate->retval が展開済みレコードの場合、その TupleDesc を取得し、期待される tupdesc との互換性をチェックする。
  • 変換が必要な場合（tupmap が存在する場合）、属性マッピングを行い、SPI_returntuple を介して上位コンテキストへコピーする。
  • 型IDが一致しない場合、レコードをフラット化し、期待される型IDと型修飾子（typmod）でヘッダーを上書きする。
  • 変換不要な場合、SPI_datumTransfer を使用して所有権を上位コンテキストへ移譲する。

    • 2. **通常の複合型データの処理**:

  • 展開済みでない場合、deconstruct_composite_datum で HeapTuple に分解する。
  • 属性マッピングが必要であれば実行し、SPI_returntuple を介して結果をコピーする。
  • 最後に ReleaseTupleDesc で一時的なディスクリプタを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/pl_exec.c 内の関数（exec_stmt_return 等の文脈で呼び出される）。
  • **呼び出し先**:
  • expanded_record_get_tupdesc, convert_tuples_by_position, execute_attr_map_tuple, SPI_returntuple, SPI_datumTransfer, deconstruct_composite_datum, ReleaseTupleDesc 等。

関連データ構造

  • PLpgSQL_execstate: 実行状態と戻り値を保持。
  • TupleDesc: 戻り値の型定義。
  • ExpandedRecordHeader: 展開済みレコードの内部表現。
  • TupleConversionMap: 属性変換用のマップ。

失敗モードと境界条件

  • 期待される型と返されるレコードの型が互換性を持たない場合、convert_tuples_by_position がエラーを発生させる可能性がある。
  • 展開済みレコードの型IDが一致しない場合、手動でのフラット化とヘッダー書き換えが行われる。

保守担当者向けメモ

  • 本関数は estate->retval を直接書き換えるため、呼び出し後のメモリコンテキストの状態に注意すること。
  • 展開済みレコードのフラット化処理は datumCopy の実装を模倣しているため、将来的なデータ構造の変更時には整合性に注意が必要。
