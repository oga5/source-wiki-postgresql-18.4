DictISpell

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/dict_ispell.c:24

目的

PostgreSQLの全文検索におけるIspell辞書の状態を保持するための構造体です。ストップワードリストと、Ispell形式の辞書オブジェクトを統合管理します。

シグネチャまたは形状

typedef struct
{
	StopList	stoplist;
	IspellDict	obj;
} DictISpell;

入力と出力

本構造体はデータ保持用であり、直接的な入力・出力関数ではありません。初期化時（dispell_init）にメモリが割り当てられ、検索時（dispell_lexize）に参照されます。

前提条件と不変条件

  • dispell_init によって適切に初期化されている必要があります。
  • obj フィールドは、Ispell辞書ファイルおよび接尾辞ファイルから構築された有効な辞書データを含んでいる必要があります。

アルゴリズムまたは振る舞い

  • stoplist: 正規化後の単語から除外すべき単語のリストを保持します。
  • obj: 形態素解析や語幹抽出を行うためのIspell辞書の実体（IspellDict）を保持します。

呼び出し元と呼び出し先

  • 呼び出し元: dispell_init（メモリ確保）、dispell_lexize（辞書データの参照）。
  • 呼び出し先: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • StopList: ストップワードを管理する構造体。
  • IspellDict: tsearch/dicts/spell.h で定義される、Ispell辞書の低レベルなデータ構造。

失敗モードと境界条件

  • メモリ不足: palloc0 によるメモリ確保失敗時にエラーが発生する可能性があります。
  • 辞書データ不整合: dispell_init において、辞書ファイルや接尾辞ファイルが読み込めない場合、本構造体は正常に構築されません。

保守担当者向けメモ

  • 本構造体は src/backend/tsearch/spell.c で定義された低レベルな API のラッパーとして機能します。
  • dispell_lexize で使用される際、メモリ管理は呼び出し側で行われるため、構造体内の各フィールドのライフサイクルに注意してください。
