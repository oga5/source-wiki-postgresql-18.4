lo_close

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-lobj.c:95

目的

データベース内のラージオブジェクトをクローズし、関連するリソースを解放します。

シグネチャまたは形状

int lo_close(PGconn *conn, int fd);

入力と出力

  • **入力**:
  • conn: データベース接続を管理する PGconn 構造体へのポインタ。
  • fd: クローズ対象のラージオブジェクト記述子（ファイルディスクリプタ）。
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時は -1 を返します。

前提条件と不変条件

  • 呼び出し前に lo_initialize(conn) が内部で実行され、サーバー側の関数OIDがキャッシュされている必要があります。
  • ラージオブジェクト操作は現在のトランザクション内で実行されます。

アルゴリズムまたは振る舞い

1. lo_initialize を呼び出し、必要なサーバー側関数のOIDがキャッシュされていることを確認します。失敗した場合は -1 を返します。 2. PQArgBlock を使用して、対象の fd を引数として設定します。 3. PQfn（Fastpath関数呼び出し）を介して、サーバー側の lo_close 関数を実行します。 4. サーバーからの応答が PGRES_COMMAND_OK であれば、PQclear で結果を解放し、サーバー側の戻り値を返します。 5. エラーが発生した場合は、PQclear で結果を解放し、-1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_dump ユーティリティ（pg_backup_archiver.c, pg_dump.c）や、fe-lobj.c 内の他のラージオブジェクト関連関数から呼び出されます。
  • **呼び出し先**: lo_initialize（初期化用）、PQfn（サーバー通信用）、PQresultStatus、PQclear（libpqの実行結果処理用）。

関連データ構造

  • PGconn: 接続状態およびラージオブジェクト関数のOIDキャッシュを保持します。
  • PQArgBlock: サーバーへ送信する引数構造体。

失敗モードと境界条件

  • lo_initialize が失敗した場合、-1 を返します。
  • サーバー側でクローズ処理が失敗した場合（無効な fd など）、PQresultStatus がエラーを示し、-1 を返します。

保守担当者向けメモ

  • 本関数は PQfn を使用してサーバー側の lo_close を呼び出します。
  • 非常に大きなサイズを扱う場合や、トランザクションのアボート状態については fe-lobj.c の全体的な設計方針に従ってください。
