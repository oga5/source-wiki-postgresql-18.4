dependency_is_fully_matched

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/dependencies.c:594

目的

指定された関数従属性（MVDependency）に含まれるすべての属性が、クエリ句等で参照されている属性集合（Bitmapset）によって完全にカバーされているかを判定します。

シグネチャまたは形状

static bool dependency_is_fully_matched(MVDependency *dependency, Bitmapset *attnums)

入力と出力

  • **入力**:
  • dependency: 検査対象の関数従属性構造体。
  • attnums: 比較対象となる属性番号のビットマップ集合。
  • **出力**:
  • bool: 従属性内の全属性が attnums に含まれていれば true、一つでも欠けていれば false を返します。

前提条件と不変条件

  • dependency は有効な MVDependency 構造体を指している必要があります。
  • attnums は bms_is_member で検査可能な有効なビットマップ集合である必要があります。

アルゴリズムまたは振る舞い

1. dependency->nattributes で定義された属性の数だけループを回します。 2. 各属性番号（dependency->attributes[j]）を取り出します。 3. bms_is_member を使用して、その属性番号が attnums に存在するかを確認します。 4. 存在しない属性が一つでも見つかった時点で直ちに false を返し、すべての属性が確認できれば true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/dependencies.c:974（ソース内の直接的な参照として確認）。
  • **呼び出し先**: bms_is_member（nodes/bitmapset.h で定義されるマクロ/関数）。

関連データ構造

  • MVDependency: 関数従属性を保持する構造体。属性のリストと従属性の度合いを格納します。
  • Bitmapset: 属性番号の集合を表現するために使用されます。

失敗モードと境界条件

  • dependency が空（nattributes == 0）の場合、ループは実行されず、関数は true を返します。
  • attnums が空のビットマップセットである場合、dependency が属性を持つ限り false を返します。

保守担当者向けメモ

  • 本関数は、プランナが統計情報を適用する際に、クエリの条件句が従属性を完全にカバーしているかを検証するために使用されます。
  • 属性番号の比較は Bitmapset を介して行われるため、attnums の構築元と dependency 内の属性番号の定義が一致していることが前提となります。
