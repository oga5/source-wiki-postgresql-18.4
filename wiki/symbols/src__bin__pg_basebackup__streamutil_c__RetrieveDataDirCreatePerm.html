RetrieveDataDirCreatePerm

定義

  • **種別**: function
  • **定義箇所**: src/bin/pg_basebackup/streamutil.c:354

目的

サーバーから data_directory_mode 設定値を取得し、クライアント側のディレクトリ作成権限設定を同期させる。

シグネチャまたは形状

static bool RetrieveDataDirCreatePerm(PGconn *conn);

入力と出力

  • **入力**: PGconn *conn（アクティブなデータベース接続へのポインタ）
  • **出力**: bool（成功した場合は true、失敗した場合は false）

前提条件と不変条件

  • conn は NULL であってはならない（Assert により検証）。
  • サーバーバージョンが MINIMUM_VERSION_FOR_GROUP_ACCESS 未満の場合、本関数は何もせず true を返す。

アルゴリズムまたは振る舞い

1. サーバーバージョンを確認し、古いバージョンの場合はデフォルトのグループアクセス権限を維持して終了する。 2. SHOW data_directory_mode コマンドをサーバーに対して実行する。 3. 取得した結果（8進数表記の文字列）を解析し、整数値に変換する。 4. 変換した値を SetDataDirectoryCreatePerm() に渡し、クライアント側の状態を更新する。 5. 実行結果に応じて PGresult を解放し、成否を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_basebackup/pg_receivewal.c
  • src/bin/pg_basebackup/pg_recvlogical.c
  • src/bin/pg_basebackup/streamutil.c（GetConnection 内）
  • **呼び出し先**:
  • PQexec, PQresultStatus, PQntuples, PQnfields, PQgetvalue, PQclear, PQserverVersion, PQerrorMessage (libpq)
  • SetDataDirectoryCreatePerm (外部関数)
  • pg_log_error (共通ログ出力)

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • PGresult: クエリ実行結果を保持する構造体。

失敗モードと境界条件

  • **接続エラー**: PQexec が PGRES_TUPLES_OK 以外を返した場合、エラーログを出力して false を返す。
  • **データ形式エラー**: 期待される行数・列数が得られない場合、または sscanf による8進数解析に失敗した場合、エラーログを出力して false を返す。

保守担当者向けメモ

  • 本関数はサーバーの data_directory_mode に依存しているため、サーバー側の設定変更がクライアントのディレクトリ作成権限に直接影響する。
  • サーバーバージョンによる条件分岐（MINIMUM_VERSION_FOR_GROUP_ACCESS）は、互換性維持のために必須である。
