GetRecordedFreeSpace

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/freespace.c:253

目的

指定されたヒープページ（heapBlk）について、Free Space Map (FSM) に記録されている現在の空き容量をバイト単位で取得する。

シグネチャまたは形状

Size GetRecordedFreeSpace(Relation rel, BlockNumber heapBlk)

入力と出力

  • **入力**:
  • rel: 対象リレーションの Relation 構造体。
  • heapBlk: 空き容量を確認したいヒープのブロック番号。
  • **出力**:
  • Size: 該当ページに記録されている空き容量（バイト数）。記録が存在しない場合は 0 を返す。

前提条件と不変条件

  • FSM は「ヒント」として機能するため、返される値は最新の物理的な空き容量と完全に一致するとは限らない。
  • fsm_readbuf が有効なバッファを返さない場合、空き容量は 0 とみなされる。

アルゴリズムまたは振る舞い

1. fsm_get_location を呼び出し、指定された heapBlk に対応する FSM 上の論理アドレス（FSMAddress）とスロット番号を特定する。 2. fsm_readbuf を使用して、該当する FSM ページを読み込む。 3. バッファが有効であれば、fsm_get_avail を用いて指定スロットのカテゴリ値（uint8）を取得する。 4. ReleaseBuffer でバッファを解放する。 5. fsm_space_cat_to_avail を使用して、取得したカテゴリ値を実際のバイト数に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_freespacemap/pg_freespacemap.c
  • contrib/pgstattuple/pgstatapprox.c
  • src/backend/access/heap/vacuumlazy.c
  • **呼び出し先**:
  • fsm_get_location
  • fsm_readbuf
  • fsm_get_avail
  • ReleaseBuffer
  • fsm_space_cat_to_avail

関連データ構造

  • Relation: リレーション情報を保持する構造体。
  • FSMAddress: FSM 内の論理的な位置を示す構造体。
  • Buffer: バッファマネージャが管理するページバッファ。

失敗モードと境界条件

  • fsm_readbuf が無効なバッファを返した場合（例: FSM がまだ初期化されていない、または該当ページが存在しない場合）、関数は 0 を返す。

保守担当者向けメモ

  • 本関数は FSM の読み取り専用アクセスを行う。
  • 戻り値は Size 型であり、内部的な 256 段階のカテゴリ値を実際のバイト数に変換したものである点に注意すること。
