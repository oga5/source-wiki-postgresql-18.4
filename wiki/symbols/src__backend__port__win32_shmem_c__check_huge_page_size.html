check_huge_page_size

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32_shmem.c:641

目的

GUC（Grand Unified Configuration）パラメータ huge_page_size の設定値が、Windows プラットフォームにおいて有効であるかを検証する。

シグネチャまたは形状

bool check_huge_page_size(int *newval, void **extra, GucSource source)

入力と出力

  • **入力**:
  • newval: 設定しようとしている huge_page_size の値へのポインタ。
  • extra: GUC フック用の追加データ（本関数では未使用）。
  • source: 設定のソース（本関数では未使用）。
  • **出力**:
  • true: 設定値が 0 である場合。
  • false: 設定値が 0 以外である場合。

前提条件と不変条件

  • Windows プラットフォームにおいて、huge_page_size は 0 でなければならないという制約がある。

アルゴリズムまたは振る舞い

1. newval が指す値が 0 以外であるかを確認する。 2. 0 以外の場合、GUC_check_errdetail を使用してエラー詳細を設定し、false を返す。 3. 0 の場合、検証を通過し true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/guc_tables.c: GUC テーブル定義内で参照されている。
  • src/backend/port/sysv_shmem.c: 参照の可能性あり（ツールによるヒューリスティックな検出）。
  • **呼び出し先**:
  • GUC_check_errdetail: エラーメッセージの生成に使用。

関連データ構造

  • GucSource: GUC 設定のソースを示す列挙型。

失敗モードと境界条件

  • newval が 0 以外の値を持つ場合、検証は失敗し、PostgreSQL の設定変更が拒否される。

保守担当者向けメモ

  • 本関数は Windows 環境専用の制約を強制するフックである。他のプラットフォーム（sysv_shmem.c 等）との整合性に注意すること。
