hex_decode

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/data.c:160

目的

16進数文字列（bytea 形式）をバイナリデータにデコードします。

シグネチャまたは形状

static unsigned hex_decode(const char *src, unsigned len, char *dst)

入力と出力

  • **入力**:
  • src: デコード対象の16進数文字列へのポインタ。
  • len: src の長さ。
  • dst: デコード結果を格納するバッファへのポインタ。
  • **出力**:
  • 成功時: デコードされたバイト数（unsigned）。
  • 失敗時: -1（入力文字列が不完全な場合）。

前提条件と不変条件

  • dst は、少なくとも len / 2 バイト以上の書き込み可能な領域を確保している必要があります。
  • 入力文字列内の空白文字（スペース、改行、タブ、復帰）は無視されます。

アルゴリズムまたは振る舞い

1. src から srcend（終端）までを順次走査します。 2. 空白文字（' ', '\n', '\t', '\r'）に遭遇した場合はスキップします。 3. それ以外の文字は get_hex を用いて 4 ビットの値に変換します。 4. 2 文字（v1, v2）を読み取り、v1 << 4 | v2 の演算で 1 バイトを生成し、dst に書き込みます。 6. 処理したバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/data.c:532 にて、variable->len の計算およびデータ変換のために呼び出されています。
  • **呼び出し先**: get_hex を呼び出します。
  • **注記**: src/backend/utils/adt/encode.c に同名の関数が存在しますが、本シンボルとはシグネチャおよび定義箇所が異なります。

関連データ構造

  • なし。

失敗モードと境界条件

  • **不完全なペア**: 16進数文字列の長さが奇数などで、ペアが完結しない場合は -1 を返します。
  • **空白文字**: 文字列中に含まれる空白は無視され、デコード処理には影響しません。

保守担当者向けメモ

  • 本関数は ecpglib 内の bytea 型処理に特化しています。
  • get_hex の実装に依存しているため、変換ロジックを変更する場合は get_hex の仕様を確認してください。
