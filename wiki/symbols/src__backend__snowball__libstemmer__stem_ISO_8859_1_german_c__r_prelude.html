r_prelude

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_german.c:155

目的

ドイツ語のステミング処理における前処理を実行し、単語内の特定の文字パターン（母音に挟まれた 'u' や 'y'）の置換および、a_0 に定義された接頭辞・文字の正規化を行う。

シグネチャまたは形状

static int r_prelude(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z （Snowball ステミング環境へのポインタ）
  • 出力: 成功時に 1 を返し、slice_from_s 等の操作でエラーが発生した場合は負の値を返す。

前提条件と不変条件

  • z が有効な SN_env 構造体を指していること。
  • g_v（母音グループ）が定義されていること。
  • a_0（置換対象の文字パターン群）が定義されていること。

アルゴリズムまたは振る舞い

1. **母音間文字の置換**: 単語全体を走査し、母音（g_v）に挟まれた 'u' を 'U' に、'y' を 'Y' に置換する。 2. **文字の正規化**: find_among を使用して a_0 に定義されたパターンを検索し、一致した場合は対応する文字列（s_2〜s_5 等）に置換する。 3. 一致しない文字はそのままスキップして処理を継続する。

呼び出し元と呼び出し先

  • 呼び出し元: german_ISO_8859_1_stem （src/backend/snowball/libstemmer/stem_ISO_8859_1_german.c）
  • 呼び出し先: in_grouping, slice_from_s, find_among （Snowball ランタイム関数）

関連データ構造

  • struct SN_env: ステミング対象の文字列バッファやカーソル位置（c, l, bra, ket）を保持する。
  • a_0: find_among で使用される置換ルールのトライ木構造。

失敗モードと境界条件

  • slice_from_s がメモリ不足やバッファ操作エラーで負の値を返した場合、その値をそのまま呼び出し元へ伝播させる。
  • 文字列の末尾に達した場合はループを終了する。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードであるため、直接編集せず元の Snowball スクリプトを修正すること。
  • g_v や a_0 の定義変更は、本関数の挙動に直接影響を与える。
