upsert_pg_statistic

定義

  • **種別**: function
  • **定義箇所**: src/backend/statistics/attribute_stats.c:846

目的

システムカタログ pg_statistic に対して、統計情報の行を挿入または更新（Upsert）します。

シグネチャまたは形状

static void upsert_pg_statistic(Relation starel, HeapTuple oldtup, Datum *values, bool *nulls, bool *replaces)

入力と出力

  • **入力**:
  • starel: pg_statistic リレーションのハンドル。
  • oldtup: 更新対象の既存タプル（存在しない場合は NULL）。
  • values, nulls, replaces: 更新に使用するデータ配列、NULLフラグ配列、および更新対象列を示すフラグ配列。
  • **出力**: なし（副作用としてシステムカタログを更新）。

前提条件と不変条件

  • starel は pg_statistic を開いた有効なリレーションである必要があります。
  • oldtup が有効な場合、そのタプルは starel に対応している必要があります。

アルゴリズムまたは振る舞い

1. oldtup が有効な場合、heap_modify_tuple を使用して既存タプルを更新し、CatalogTupleUpdate を呼び出します。 2. oldtup が無効な場合、heap_form_tuple を使用して新規タプルを構築し、CatalogTupleInsert を呼び出します。 3. 処理したタプルを heap_freetuple で解放します。 4. CommandCounterIncrement() を呼び出し、変更を可視化します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/attribute_stats.c 内の統計更新処理（例: 540行目付近）。
  • **呼び出し先**: heap_modify_tuple, CatalogTupleUpdate, heap_form_tuple, CatalogTupleInsert, heap_freetuple, CommandCounterIncrement。

関連データ構造

  • Relation: リレーション記述子。
  • HeapTuple: pg_statistic の行データ。

失敗モードと境界条件

  • pg_statistic への書き込み権限やカタログ更新時の制約違反によりエラーが発生する可能性があります。
  • oldtup が無効な場合に新規挿入を行いますが、その際のタプル構築失敗は基盤の heap_form_tuple に依存します。

保守担当者向けメモ

  • 本関数は pg_statistic への直接的な書き込みを行う低レベル関数です。
  • CommandCounterIncrement() を呼び出しているため、本関数をループ内で大量に呼び出す場合はパフォーマンスへの影響を考慮してください。
