AtSubAbort_smgr

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/storage.c:974

目的

サブトランザクションのアボート時に、そのサブトランザクション内で予約された物理リレーションの削除処理を適切に実行する。

シグネチャまたは形状

void AtSubAbort_smgr(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として物理ファイルの削除を実行）

前提条件と不変条件

  • 本関数はサブトランザクションのアボート処理中に呼び出されることを前提とする。
  • smgrDoPendingDeletes(false) を呼び出すことで、現在のサブトランザクションに関連付けられた削除予約のみを処理する。

アルゴリズムまたは振る舞い

1. smgrDoPendingDeletes(false) を呼び出す。 2. 引数 false により、コミット時ではなくアボート時の処理として、保留中の物理ファイル削除を実行する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c:5351
  • 呼び出し先: smgrDoPendingDeletes

関連データ構造

  • pendingDeletes: トランザクション内で削除待ちのリレーションを保持するリスト（smgrDoPendingDeletes が参照）。

失敗モードと境界条件

  • 物理ファイルの削除に失敗した場合の挙動は smgrDoPendingDeletes の実装に依存する。

保守担当者向けメモ

  • 本関数はサブトランザクション専用のクリーンアップ処理である。メインのトランザクション終了時の処理には AtCommit_smgr や AtAbort_smgr 等の別関数が使用される点に注意すること。
