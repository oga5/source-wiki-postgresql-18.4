vacuum_xid_failsafe_check

定義

  • src/backend/commands/vacuum.c:1283

目的

VACUUM 処理中にトランザクション ID（XID）またはマルチトランザクション ID（MultiXactId）の周回によるデータ損失を防ぐためのフェイルセーフ条件を判定する。

シグネチャまたは形状

bool vacuum_xid_failsafe_check(const struct VacuumCutoffs *cutoffs)

入力と出力

  • **入力**: cutoffs (対象リレーションの relfrozenxid および relminmxid を含む VacuumCutoffs 構造体へのポインタ)
  • **出力**: bool (フェイルセーフをトリガーすべき場合は true、それ以外は false)

前提条件と不変条件

  • cutoffs->relfrozenxid は正規のトランザクション ID であること。
  • cutoffs->relminmxid は有効なマルチトランザクション ID であること。

アルゴリズムまたは振る舞い

1. **XID フェイルセーフ判定**:

  • vacuum_failsafe_age と autovacuum_freeze_max_age * 1.05 の大きい方をスキップ閾値とする。
  • 現在の ReadNextTransactionId() から閾値を引いた値（xid_skip_limit）を算出する。
  • 対象の relfrozenxid が xid_skip_limit より古い場合、true を返す。

    • 2. **MultiXactId フェイルセーフ判定**:

  • vacuum_multixact_failsafe_age と autovacuum_multixact_freeze_max_age * 1.05 の大きい方をスキップ閾値とする。
  • 現在の ReadNextMultiXactId() から閾値を引いた値（multi_skip_limit）を算出する。
  • 対象の relminmxid が multi_skip_limit より古い場合、true を返す。

    • 3. 上記いずれにも該当しない場合、false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/vacuumlazy.c:2956 (VACUUM 処理ループ内)
  • **呼び出し先**: ReadNextTransactionId(), ReadNextMultiXactId(), TransactionIdPrecedes(), MultiXactIdPrecedes() 等のアクセサ関数。

関連データ構造

  • VacuumCutoffs: VACUUM 処理における凍結閾値を保持する構造体。

失敗モードと境界条件

  • xid_skip_limit が正規のトランザクション ID 範囲を下回る場合、FirstNormalTransactionId にクランプされる。
  • multi_skip_limit が FirstMultiXactId を下回る場合、FirstMultiXactId にクランプされる。

保守担当者向けメモ

  • 本関数は VACUUM の実行中に、インデックスのスキップや処理の緊急停止を判断するために使用される。
  • 閾値計算に使用される autovacuum_freeze_max_age 等の GUC パラメータ変更の影響を受ける。
