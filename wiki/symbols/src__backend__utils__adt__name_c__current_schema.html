current_schema

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/name.c:278

目的

現在のセッションにおける検索パス（search path）の先頭にあるスキーマ名を name 型として取得します。

シグネチャまたは形状

Datum current_schema(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL 関数呼び出し規約に基づく引数）。
  • **出力**: Datum 型。検索パスが空の場合、またはスキーマ名が取得できない場合は NULL を返します。それ以外の場合は、スキーマ名を表す name 型の値を返します。

前提条件と不変条件

  • fetch_search_path(false) を使用して検索パスを取得します。
  • 取得した検索パスのリストは、使用後に list_free で解放されます。
  • 戻り値は namein 関数を介して name 型に変換されます。

アルゴリズムまたは振る舞い

1. fetch_search_path(false) を呼び出し、現在の検索パスリストを取得します。 2. リストが空（NIL）であれば NULL を返します。 3. リストの先頭要素（linitial_oid）からスキーマの OID を取得し、get_namespace_name でスキーマ名（nspname）を解決します。 4. 取得したリストを list_free で解放します。 5. スキーマ名が解決できなかった場合（削除済みスキーマ等）は NULL を返します。 6. 解決できた場合、DirectFunctionCall1 を使用して namein 関数を呼び出し、文字列を name 型の Datum に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c:3254（実行エンジンによる評価）
  • **呼び出し先**:
  • fetch_search_path
  • linitial_oid
  • get_namespace_name
  • list_free
  • namein

関連データ構造

  • List: 検索パスを保持するために使用。
  • name: 戻り値の型。
  • Datum: PostgreSQL の汎用データ表現。

失敗モードと境界条件

  • 検索パスが設定されていない場合、NULL を返します。
  • 検索パスの先頭にあるスキーマが既に削除されている場合、NULL を返します。

保守担当者向けメモ

  • 本関数は安定（stable）な関数として扱われます。
  • namein を直接呼び出しているため、name 型の固定長制限（NAMEDATALEN）が適用されます。
