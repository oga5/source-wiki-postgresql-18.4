checkViewColumns

定義

  • 種別: function
  • 定義箇所: src/backend/commands/view.c:266

目的

CREATE OR REPLACE VIEW 実行時に、既存のビュー定義と新しい定義の間で列構成の互換性を検証し、破壊的な変更を阻止する。

シグネチャまたは形状

static void checkViewColumns(TupleDesc newdesc, TupleDesc olddesc)

入力と出力

  • **入力**:
  • newdesc: 新しいビュー定義のタプル記述子。
  • olddesc: 既存のビュー定義のタプル記述子。
  • **出力**: なし（戻り値は void）。
  • **副作用**: 互換性がない場合、ereport(ERROR, ...) を発行してトランザクションを中断する。

前提条件と不変条件

  • ビューの列数、列名、データ型、照合順序（collation）の変更は、他のビューやルールとの依存関係を壊す可能性があるため禁止されている。
  • olddesc は既存のビュー定義を保持している必要がある。

アルゴリズムまたは振る舞い

1. **列数チェック**: newdesc の列数が olddesc より少ない場合、列の削除とみなしてエラーを投げる。 2. **列ごとの比較**: 各列について以下の属性を順次検証する。

  • attisdropped: 削除済みフラグが一致しない場合はエラー。
  • attname: 列名が異なる場合はエラー（ALTER VIEW ... RENAME COLUMN の使用を推奨）。
  • atttypid / atttypmod: データ型や型修飾子が異なる場合はエラー。
  • attcollation: 照合順序が異なる場合はエラー。

    • 3. **制約の無視**: 制約フィールドは検証対象外とする（既存ビューのデフォルト値は維持されるため）。

呼び出し元と呼び出し先

  • **呼び出し元**: DefineVirtualRelation（src/backend/commands/view.c 内）。replace モードでビューを作成する際に呼び出される。
  • **呼び出し先**: ereport、TupleDescAttr、NameStr、format_type_with_typemod、get_collation_name 等のユーティリティ関数。

関連データ構造

  • TupleDesc: タプルの構造を記述するデータ構造。
  • Form_pg_attribute: システムカタログ pg_attribute の行構造。

失敗モードと境界条件

  • **列の削除**: newdesc->natts < olddesc->natts の場合に ERRCODE_INVALID_TABLE_DEFINITION で失敗。
  • **属性の不一致**: 列名、型、照合順序のいずれかが変更された場合に ERRCODE_INVALID_TABLE_DEFINITION で失敗。

保守担当者向けメモ

  • 本関数はビューの列定義の厳格な整合性を保証する。将来的にビューの列属性の変更を許可する仕様変更を行う場合は、依存関係（ルールシステム等）への影響を慎重に評価する必要がある。
