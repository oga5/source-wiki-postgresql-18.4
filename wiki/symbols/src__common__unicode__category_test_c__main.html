定義

目的

PostgreSQL の Unicode カテゴリおよび文字プロパティの実装と、ICU ライブラリの定義との整合性を検証するためのテストプログラムのエントリポイントです。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**: argc（引数の数）、argv（引数の配列）。
  • **出力**: 正常終了時は 0 を返します。テスト失敗時は icu_test() 内でエラー詳細を出力し、プログラムが終了します。

前提条件と不変条件

  • USE_ICU が定義されている環境で実行されることを想定しています。
  • PG_UNICODE_VERSION および U_UNICODE_VERSION（USE_ICU 定義時）が定義されている必要があります。

アルゴリズムまたは振る舞い

1. parse_unicode_version を呼び出し、PostgreSQL 側の Unicode バージョンを解析して pg_unicode_version に格納します。 2. バージョン情報を標準出力に表示します。 3. USE_ICU が定義されている場合:

  • parse_unicode_version を呼び出し、ICU 側の Unicode バージョンを解析して icu_unicode_version に格納します。
  • ICU のバージョン情報を標準出力に表示します。
  • icu_test() を呼び出し、全コードポイントの走査と検証を実行します。

    • 4. USE_ICU が定義されていない場合:

  • ICU が利用不可である旨を標準出力に表示し、テストをスキップします。

呼び出し元と呼び出し先

  • **呼び出し元**: OS の実行環境から直接呼び出されるエントリポイントです。
  • **呼び出し先**:
  • parse_unicode_version
  • icu_test (条件付き)
  • **注記**: ツールが提示した contrib/amcheck 等の main への言及は、本シンボルとは無関係な別プログラムのコンテキストであり、本関数の呼び出し元ではありません。

関連データ構造

  • pg_unicode_version (グローバル変数)
  • icu_unicode_version (グローバル変数)

失敗モードと境界条件

  • USE_ICU が定義されていない場合、テストは実行されずスキップされます。
  • icu_test() 内で不一致が検出された場合、詳細なエラーメッセージが出力され、プログラムは異常終了します。

保守担当者向けメモ

  • 本関数は category_test プログラムの制御フローを管理します。
  • テストの失敗は、PostgreSQL 側の Unicode テーブル生成スクリプトと ICU の定義の不整合を示唆している可能性が高いため、icu_test の検証ロジックと併せて確認してください。
