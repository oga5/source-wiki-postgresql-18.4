r_Step_2

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_english.c:652

目的

Porterステミングアルゴリズムの第2ステップを実行し、特定の英語接尾辞を置換または削除する。

シグネチャまたは形状

static int r_Step_2(struct SN_env * z)

入力と出力

  • 入力: SN_env 構造体（ステミング対象の文字列と現在のカーソル位置を含む）。
  • 出力: 成功時に 1、失敗または処理対象外の場合に 0、メモリ操作等のエラー時に負の値を返す。

前提条件と不変条件

  • z->c（現在のカーソル位置）が有効な範囲内であること。
  • r_R1 関数によって、対象の接尾辞が単語のR1領域内にあることが検証される。

アルゴリズムまたは振る舞い

1. find_among_b を使用して、単語末尾から a_5 テーブルに定義された接尾辞を検索する。 2. 接尾辞が見つかった場合、r_R1 を呼び出し、その接尾辞がR1領域内にあるか確認する。 3. R1領域内であれば、switch 文で among_var（検索結果のID）に基づき、対応する接尾辞を slice_from_s で置換する。 4. among_var が 15 の場合、g_valid_LI グループに含まれないことを確認した上で slice_del を実行する。 5. among_var が 13 の場合、直前の文字が 'l' であることを確認してから置換を行う。

呼び出し元と呼び出し先

  • 呼び出し元: stem_UTF_8_english.c および関連するステミング実装ファイル（stem_ISO_8859_1_english.c 等）のメイン処理フローから呼び出される。
  • 呼び出し先: find_among_b, r_R1, slice_from_s, slice_del, in_grouping_b_U。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • a_5: ステップ2で使用される接尾辞検索テーブル。
  • g_valid_LI: ステップ2の例外処理で使用される文字グループ。

失敗モードと境界条件

  • 接尾辞が検索テーブルに存在しない場合、またはR1領域外にある場合は 0 を返す。
  • slice_from_s や slice_del が内部エラー（メモリ不足等）を返した場合、その負の値をそのまま伝播させる。
  • 文字列の先頭や境界付近で、z->lb を超えるアクセスが発生しないようガード条件が設定されている。

保守担当者向けメモ

  • 本関数は自動生成コードであるため、直接編集せず、Snowballのソース定義（.sbl）を修正すること。
  • among_var の各ケースは、Porterアルゴリズムの特定の置換ルールに対応している。ルール変更時は a_5 テーブルとの整合性に注意が必要。
