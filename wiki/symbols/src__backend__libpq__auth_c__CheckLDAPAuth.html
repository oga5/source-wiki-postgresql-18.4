CheckLDAPAuth

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth.c:2435

目的

LDAP認証方式を用いたPostgreSQLクライアントの認証処理を実行する。

シグネチャまたは形状

static int CheckLDAPAuth(Port *port);

入力と出力

  • **入力**: Port *port（接続情報および pg_hba.conf から読み込まれたLDAP設定を含む）。
  • **出力**: int 型のステータスコード（STATUS_OK, STATUS_ERROR, STATUS_EOF）。

前提条件と不変条件

  • port->hba が初期化されており、LDAP関連の設定（ldapserver, ldapbasedn 等）が参照可能であること。
  • LDAPライブラリ（OpenLDAPまたはWinLDAP）が利用可能であること。

アルゴリズムまたは振る舞い

1. **設定検証**: LDAPサーバーまたはベースDNが指定されているか確認する。 2. **ポート設定**: ldaps スキームの場合はデフォルトポートを LDAPS_PORT に設定する。 3. **パスワード要求**: クライアントへ AUTH_REQ_PASSWORD を送信し、パスワードを受信する。 4. **接続確立**: InitializeLDAPConnection を呼び出し、LDAPサーバーへの接続を初期化する。 5. **DN検索（ベースDN指定時）**:

  • ユーザー名に不正な文字（*, (, ), \, /）が含まれていないか検証する。
  • ldap_simple_bind_s を用いて検索用バインドを行う。
  • FormatSearchFilter または属性設定に基づき検索フィルタを構築し、ldap_search_s でユーザーDNを特定する。
  • 検索結果が単一であることを確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth.c 内の認証ディスパッチャー（ClientAuthentication 関数）。
  • **呼び出し先**: sendAuthRequest, recv_password_packet, InitializeLDAPConnection, FormatSearchFilter, および各LDAPライブラリ関数（ldap_simple_bind_s, ldap_search_s 等）。

関連データ構造

  • Port: 接続状態と認証設定を保持する構造体。
  • LDAP: LDAP接続ハンドル。
  • LDAPMessage: LDAP検索結果を保持する構造体。

失敗モードと境界条件

  • **設定不備**: サーバー名やベースDNが未指定の場合、LOG レベルでエラーを出力し STATUS_ERROR を返す。
  • **認証失敗**: パスワードが送信されない場合、STATUS_EOF を返す。
  • **検索失敗**: ユーザーが特定できない、または複数存在する場合、LOG レベルで詳細を出力し STATUS_ERROR を返す。
  • **LDAPエラー**: バインドや検索が失敗した場合、ldap_err2string を用いたエラーログを出力し終了する。

保守担当者向けメモ

  • ユーザー名に含まれる特殊文字のフィルタリングは、LDAPインジェクションを防ぐための重要なセキュリティ対策である。
  • 認証失敗時のエラーメッセージは、セキュリティ上の理由からクライアントには詳細を返さない設計となっている。ログ出力には errdetail_for_ldap を活用すること。
