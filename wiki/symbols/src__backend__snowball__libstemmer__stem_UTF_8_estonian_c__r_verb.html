目的

エストニア語のステミング処理において、単語の末尾から動詞特有の接尾辞を検索・削除または置換し、語幹を正規化する。

シグネチャまたは形状

static int r_verb(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（Snowball ランタイム環境構造体）。現在の単語データおよび処理領域（RV領域）の情報を含む。
  • **出力**: int。処理が成功した場合は 1、失敗または適用対象外の場合は 0、ランタイムエラーが発生した場合は負の値を返す。

前提条件と不変条件

  • z->I[0] に格納された領域境界（RV領域の開始位置）より現在のカーソル位置 z->c が後ろにあること。
  • a_1 配列に定義された動詞接尾辞のパターンマッチングに依存する。

アルゴリズムまたは振る舞い

1. **領域制限**: z->I[0] を基に処理範囲を制限し、カーソルを末尾に設定する。 2. **接尾辞検索**: find_among_b を使用し、a_1 配列（21個のパターン）に基づき末尾から前方へ接尾辞を検索する。 3. **条件分岐**:

  • among_var == 1: 該当する接尾辞を削除する。
  • among_var == 2: 該当する接尾辞を s_0 で定義された文字列に置換する。
  • among_var == 3: g_V1 グルーピング（母音）の条件を満たす場合のみ、接尾辞を削除する。

    • 4. **終了**: 処理結果を SN_env に反映し、成功または失敗を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/libstemmer/stem_UTF_8_estonian.c 内のステミングパイプラインから呼び出される。
  • **呼び出し先**:
  • find_among_b: 接尾辞検索に使用。
  • slice_del: 接尾辞削除に使用。
  • slice_from_s: 接尾辞置換に使用。
  • in_grouping_b_U: 母音判定に使用。
  • 注: 他の言語ファイル（ロシア語、アルメニア語等）にも同名のシンボルが存在するが、本ページはエストニア語版の定義に基づく。

関連データ構造

  • SN_env: Snowball ランタイム環境。
  • a_1: 動詞接尾辞を格納した among 構造体の配列。
  • g_V1: 母音判定用の文字グループ。

失敗モードと境界条件

  • z->c < z->I[0]: 処理領域が空または無効な場合、0 を返す。
  • find_among_b が一致しない場合、0 を返す。
  • slice_del や slice_from_s がランタイムエラー（負の値）を返した場合、その値をそのまま伝播させる。
