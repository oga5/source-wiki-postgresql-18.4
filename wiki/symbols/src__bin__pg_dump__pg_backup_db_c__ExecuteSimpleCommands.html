ExecuteSimpleCommands

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_db.c:318

目的

バッファリングされた SQL 文字列を解析し、セミコロン（;）で区切られた個別の SQL コマンドを抽出して実行する。

シグネチャまたは形状

static void ExecuteSimpleCommands(ArchiveHandle *AH, const char *buf, size_t bufLen)

入力と出力

  • **入力**:
  • AH: ArchiveHandle 構造体へのポインタ。状態管理（sqlparse）および実行に使用。
  • buf: 実行対象の SQL コマンドを含む文字列バッファ。
  • bufLen: buf の長さ。
  • **出力**: なし（副作用としてデータベースへのクエリ実行を行う）。

前提条件と不変条件

  • AH->sqlparse.curCmd が初期化されていない場合、内部で createPQExpBuffer() を用いて初期化される。
  • SQL 解析状態（SQL_SCAN, SQL_IN_SINGLE_QUOTE, SQL_IN_DOUBLE_QUOTE）は AH->sqlparse に保持され、関数呼び出しを跨いで維持される。

アルゴリズムまたは振る舞い

1. AH->sqlparse.curCmd が未初期化であれば作成する。 2. 入力バッファ buf を一文字ずつ走査する。 3. 改行文字は、コマンドバッファが空の場合に限りスキップする。それ以外はバッファに追加する。 4. 状態遷移マシンを用いて解析を行う:

  • SQL_SCAN: セミコロン ; を検出すると ExecuteSqlCommand を呼び出してコマンドを実行し、バッファをリセットする。引用符 ' または " を検出すると、それぞれのクォート内状態へ遷移する。
  • SQL_IN_SINGLE_QUOTE: ' を検出すると SQL_SCAN に戻る。バックスラッシュによるエスケープ処理を考慮する。
  • SQL_IN_DOUBLE_QUOTE: " を検出すると SQL_SCAN に戻る。

    • 5. 処理終了時、バッファの内容は次の呼び出しまで保持される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_archiver.c:976 (BLOB COMMENTS 処理)
  • src/bin/pg_dump/pg_backup_db.c:408 (ExecuteSqlCommandBuf 内)
  • **呼び出し先**:
  • ExecuteSqlCommand (クエリ実行)
  • appendPQExpBufferChar / resetPQExpBuffer (バッファ操作)

関連データ構造

  • ArchiveHandle: 状態保持に使用。特に sqlparse メンバ（curCmd, state, backSlash 等）が重要。

失敗モードと境界条件

  • ExecuteSqlCommand が失敗した場合、内部で die_on_query_failure を通じてプロセスが終了する可能性がある。
  • 特殊な SQL 構文（コメントなど）の解析には対応していない。

保守担当者向けメモ

  • 本関数は簡易的なステートマシンであり、複雑な SQL 解析は行わない。
  • AH->public.std_strings の値によって、シングルクォート内のバックスラッシュの扱いが変化する点に注意すること。
