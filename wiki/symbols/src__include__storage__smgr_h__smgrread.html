smgrread

定義

  • 種別: function
  • 定義箇所: src/include/storage/smgr.h:123

目的

指定されたリレーションの特定のブロックを読み込み、提供されたバッファに格納する。

シグネチャまたは形状

static inline void
smgrread(SMgrRelation reln, ForkNumber forknum, BlockNumber blocknum, void *buffer)

入力と出力

  • reln (SMgrRelation): 読み込み対象のリレーションハンドル。
  • forknum (ForkNumber): 読み込むフォーク（データ、FSM、VMなど）。
  • blocknum (BlockNumber): 読み込むブロック番号。
  • buffer (void *): 読み込んだデータを格納するメモリ領域。
  • 戻り値: なし（void）。

前提条件と不変条件

  • smgrread は単一ブロックの読み込み操作を行う。
  • 内部的に smgrreadv を呼び出すラッパーとして実装されている。

アルゴリズムまたは振る舞い

1. 引数として受け取った buffer ポインタを配列として扱い、要素数 1 のベクトルとして smgrreadv を呼び出す。 2. smgrreadv を介して、適切なストレージマネージャ（md.c 等）へ処理を委譲する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_prewarm/pg_prewarm.c
  • src/backend/catalog/storage.c
  • src/backend/storage/buffer/bufmgr.c
  • **呼び出し先**:
  • smgrreadv

関連データ構造

  • SMgrRelation: リレーションの物理的な I/O を管理するハンドル。
  • ForkNumber: リレーション内のフォークを識別する列挙型。
  • BlockNumber: ブロックを識別する整数型。

失敗モードと境界条件

  • 存在しないブロックや無効なフォークを指定した場合の挙動は、下位のストレージマネージャ（md.c 等）の実装に依存する。
  • エラー発生時は elog(ERROR) 等によりトランザクションがアボートされる可能性がある。

保守担当者向けメモ

  • 本関数は smgrreadv の単一ブロック用ラッパーである。新規実装やパフォーマンスが重視される箇所では、ベクトル I/O インターフェースである smgrreadv の直接利用が推奨される。
  • smgr.c の関数群はリエントラントではないため、割り込み時の動作に注意すること。
