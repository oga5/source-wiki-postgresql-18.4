normalize_libc_locale_name

定義

  • 種別: function
  • 定義箇所: src/backend/commands/collationcmds.c:595

目的

libcロケール名からエンコーディング指定（例: .utf8）を除去し、正規化されたロケール名を生成する。

シグネチャまたは形状

static bool normalize_libc_locale_name(char *new, const char *old)

入力と出力

  • old: 変換元のロケール名文字列（入力）。
  • new: 変換後のロケール名を格納するバッファ（出力）。
  • 戻り値: ロケール名が変更された（エンコーディング指定が除去された）場合は true、変更がなかった場合は false を返す。

前提条件と不変条件

  • new は old と同等以上の長さを保持できる十分なメモリ領域を確保している必要がある。
  • old はヌル終端文字列であること。

アルゴリズムまたは振る舞い

1. old 文字列を走査し、.（ドット）が現れるまで new にコピーする。 2. . が現れた場合、それ以降のエンコーディングタグ（英数字および - で構成される部分）をスキップする。 3. 走査終了後、new にヌル終端文字を付与する。 4. エンコーディングタグの除去が行われた場合、changed フラグを true に設定して返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/collationcmds.c 内のロケール処理箇所（pg_import_system_collations 等の文脈で利用）。
  • 呼び出し先: 特になし（標準的なC言語のポインタ操作のみ）。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • old に . が含まれない場合、文字列はそのままコピーされ、戻り値は false となる。
  • エンコーディングタグが空、または不正な形式の場合の挙動は、走査ロジックに依存する。

保守担当者向けメモ

  • 本関数は collationcmds.c 内のロケール名正規化ユーティリティとして機能する。
  • エンコーディングタグの判定ロジック（英数字と - のみ）は、libcのロケール名規則に依存しているため、プラットフォーム固有のロケール名形式に変更が生じた場合は注意が必要である。
