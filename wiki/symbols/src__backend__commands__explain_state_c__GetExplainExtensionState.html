GetExplainExtensionState

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_state.c:258

目的

EXPLAIN コマンド実行中に拡張機能が保持する状態（opaque pointer）を、指定された ID に基づいて ExplainState 構造体から取得します。

シグネチャまたは形状

void *GetExplainExtensionState(ExplainState *es, int extension_id)

入力と出力

  • **入力**:
  • es: ExplainState 構造体へのポインタ。
  • extension_id: 取得対象の拡張機能に割り当てられた一意の整数 ID。
  • **出力**:
  • 成功時: 拡張機能が保持する状態データへの void * ポインタ。
  • 失敗時: NULL（ID が範囲外の場合）。

前提条件と不変条件

  • extension_id は 0 以上である必要があります（Assert により検証）。
  • es は有効な ExplainState 構造体を指している必要があります。

アルゴリズムまたは振る舞い

1. extension_id が 0 未満でないことをアサートします。 2. extension_id が es->extension_state_allocated（現在確保されている配列サイズ）以上である場合、NULL を返します。 3. 上記以外の場合、es->extension_state 配列の指定インデックスにあるポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_overexplain/pg_overexplain.c 内の複数箇所で、拡張機能の状態取得のために呼び出されています。
  • **呼び出し先**: 本関数自体は他の関数を呼び出しません。

関連データ構造

  • ExplainState: EXPLAIN の実行状態を管理する構造体。extension_state 配列を保持します。

失敗モードと境界条件

  • extension_id が確保済みの配列サイズを超えている場合、安全に NULL を返します。
  • extension_id が負の値の場合、アサーション違反によりプロセスが停止する可能性があります。

保守担当者向けメモ

  • 本関数で返されるポインタは、ExplainState の生存期間中のみ有効です。
  • extension_state 配列は動的に拡張される可能性があるため、ポインタの長期保持には注意が必要です。
