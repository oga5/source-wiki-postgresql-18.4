be_gssapi_write

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-gssapi.c:102

目的

GSSAPIを使用して、クライアントへの送信データを暗号化し、トランスポート層へ書き出す。

シグネチャまたは形状

ssize_t be_gssapi_write(Port *port, const void *ptr, size_t len);

入力と出力

  • **入力**:
  • port: 接続情報を保持する Port 構造体。
  • ptr: 送信対象のデータバッファ。
  • len: 送信データのバイト数。
  • **出力**:
  • 成功時: 送信完了したバイト数（len と一致）。
  • 失敗時: -1 を返し、errno を設定する。

前提条件と不変条件

  • GSSAPI認証が完了し、port->gss->ctx が有効であること。
  • 呼び出し元は、前回の呼び出しで送信しきれなかったデータ（PqGSSSendConsumed）を再送する必要がある。
  • 内部状態 PqGSSSendConsumed は、送信完了時に 0 にリセットされる。

アルゴリズムまたは振る舞い

1. **再送チェック**: len < PqGSSSendConsumed の場合、エラーとして処理する。 2. **バッファ送信**: PqGSSSendLength にデータが残っている場合、secure_raw_write を呼び出し、送信を試みる。送信が完了するまでループする。 3. **暗号化**: 未送信データがある場合、gss_wrap を使用してデータを暗号化する。

  • PQ_GSS_MAX_PACKET_SIZE を超えないよう分割して処理する。
  • 暗号化されたデータに4バイトのネットワークバイトオーダーの長さヘッダーを付与し、PqGSSSendBuffer に格納する。

    • 4. **ループ**: 全データが暗号化・送信されるまで繰り返す。 5. **完了**: PqGSSSendConsumed をリセットし、送信バイト数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/be-secure.c (直接確認済み)
  • **呼び出し先**:
  • secure_raw_write (トランスポート層への書き出し)
  • gss_wrap (GSSAPI暗号化)
  • gss_release_buffer (GSSAPIバッファ解放)
  • pg_hton32 (バイトオーダー変換)
  • pg_GSS_error / ereport (エラーハンドリング)

関連データ構造

  • Port: 接続状態を保持。
  • PqGSSSendBuffer: 暗号化済みデータの一時保存用静的バッファ。
  • PqGSSSendLength, PqGSSSendNext, PqGSSSendConsumed: 送信状態を管理する静的変数。
  • PQ_GSS_MAX_PACKET_SIZE: 最大パケットサイズ定数。

失敗モードと境界条件

  • **GSSAPIエラー**: gss_wrap が GSS_S_COMPLETE 以外を返した場合、-1 を返し ECONNRESET を設定。
  • **暗号化不備**: conf_state が 0（機密性なし）の場合、エラーとして処理。
  • **パケットサイズ超過**: 暗号化後のサイズが PQ_GSS_MAX_PACKET_SIZE を超える場合、エラーとして処理。
  • **再送不整合**: 呼び出し元が期待される再送データを提供しない場合、COMMERROR をログ出力し、ECONNRESET を設定。

保守担当者向けメモ

  • 本関数は静的変数を用いて状態を保持するため、単一のGSSAPI接続を前提としている。
  • エラー発生時は elog(FATAL) を避け、elog(COMMERROR) を使用して無限再帰を防止すること。
  • パケットサイズはプロトコル仕様の一部であるため、変更時はクライアントとの互換性に注意が必要。
