TrackItem

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/analyze.c:2074

目的

統計情報の計算過程において、特定のデータ値とその出現回数をペアで保持し、集計やソートを行うための汎用的なデータ構造です。

シグネチャまたは形状

typedef struct
{
    Datum       value;  /* 追跡対象の値 */
    int         count;  /* 出現回数 */
} TrackItem;

入力と出力

  • 入力: Datum 型の値と、それに対応する出現回数（整数）。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • value は Datum 型であり、比較可能なデータである必要があります。
  • count は非負の整数として扱われることが想定されます。

アルゴリズムまたは振る舞い

  • analyze.c 内では、統計計算時に値を追跡するための配列要素として使用されます。
  • src/backend/tsearch/ts_typanalyze.c では、ハッシュテーブルの要素として使用され、全文検索用の語彙（lexemes）の出現頻度をカウントするために利用されます。
  • ハッシュテーブルでの使用時には、hash_ctl.entrysize に sizeof(TrackItem) が指定されます。
  • ソートが必要な場合、qsort_interruptible を用いて TrackItem のポインタ配列を並べ替える処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/analyze.c: palloc によるメモリ確保時に使用。
  • src/backend/tsearch/ts_typanalyze.c: ハッシュテーブルの検索およびソート処理で使用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • Datum: PostgreSQL の汎用データ型。
  • hash_ctl: ts_typanalyze.c でハッシュテーブルを構築する際に使用される制御構造体。

失敗モードと境界条件

  • メモリ不足: palloc によるメモリ確保時に失敗する可能性があります。
  • 型の不整合: Datum として格納される値が、比較関数やハッシュ関数と互換性がない場合、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は analyze.c と ts_typanalyze.c の両方で定義・利用されています。変更を加える際は、両方のモジュールにおける整合性に注意してください。
  • Datum 型を扱うため、メモリ管理や型のライフサイクルについては PostgreSQL のメモリコンテキストのルールに従う必要があります。
