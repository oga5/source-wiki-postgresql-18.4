FPgt

定義

  • 種別: function
  • 定義箇所: src/include/utils/geo_decls.h:70

目的

浮動小数点数 A が B より大きいかを、許容誤差 EPSILON を考慮して判定する。

シグネチャまたは形状

static inline bool FPgt(double A, double B)

入力と出力

  • **入力**: 比較対象の浮動小数点数 A および B。
  • **出力**: A > B + EPSILON が真であれば true、そうでなければ false を返す。

前提条件と不変条件

  • EPSILON を用いた曖昧な比較を行う。
  • NaN（非数）が含まれる場合、常に false を返す。
  • 数値解析上の厳密な推移律を満たさない可能性がある。

アルゴリズムまたは振る舞い

A が B に EPSILON を加えた値よりも大きい場合にのみ真を返す。これにより、浮動小数点演算に伴う微小な誤差を無視した「より大きい」判定を実現する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistproc.c
  • src/backend/access/spgist/spgkdtreeproc.c
  • src/backend/utils/adt/geo_ops.c
  • **呼び出し先**:
  • math.h（EPSILON の定義や浮動小数点演算に関連）

関連データ構造

  • Point, BOX などの幾何データ型（これらの座標比較に使用される）。

失敗モードと境界条件

  • NaN が入力された場合、比較結果は常に false となる。
  • 非常に大きな値や極端に小さな値の比較において、EPSILON の影響により期待される順序関係と異なる結果を返す可能性がある。

保守担当者向けメモ

  • 本関数は数値解析の専門家によって設計されたものではないため、厳密な計算が求められる箇所での使用には注意が必要である。
  • FPeq, FPne, FPlt, FPle と同様の比較ロジック群の一部である。
