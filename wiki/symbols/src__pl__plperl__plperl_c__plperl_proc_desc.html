plperl_proc_desc

定義

  • 種別: struct
  • 定義箇所: src/pl/plperl/plperl.c:102

目的

PL/Perl 関数およびトリガーのコンパイル済みメタデータと、Perl インタプリタ内での実行に必要な情報を保持する構造体です。

シグネチャまたは形状

typedef struct plperl_proc_desc
{
    char       *proname;
    MemoryContext fn_cxt;
    unsigned long fn_refcount;
    TransactionId fn_xmin;
    ItemPointerData fn_tid;
    SV         *reference;
    plperl_interp_desc *interp;
    bool        fn_readonly;
    Oid         lang_oid;
    List       *trftypes;
    bool        lanpltrusted;
    bool        fn_retistuple;
    bool        fn_retisset;
    bool        fn_retisarray;
    Oid         result_oid;
    FmgrInfo    result_in_func;
    Oid         result_typioparam;
    int         nargs;
    FmgrInfo   *arg_out_func;
    bool       *arg_is_rowtype;
    Oid        *arg_arraytype;
} plperl_proc_desc;

入力と出力

本構造体自体はデータ保持用ですが、以下の情報を管理します。

  • **入力**: pg_proc カタログから読み込まれた関数定義、引数情報、および Perl サブルーチンへの参照。
  • **出力**: 実行時に PostgreSQL の Datum と Perl の SV 間で変換を行うための関数情報（FmgrInfo）および型情報。

前提条件と不変条件

  • 各 plperl_proc_desc は専用のメモリコンテキスト fn_cxt を持ちます。
  • fn_refcount が 0 になると、free_plperl_function により構造体および関連する Perl サブルーチン参照が解放されます。
  • 信頼された関数（lanpltrusted が true）の場合、実行ユーザーID ごとに独立したインタプリタで管理されます。

アルゴリズムまたは振る舞い

  • compile_plperl_function によって生成され、plperl_proc_hash でキャッシュされます。
  • 関数定義が変更された場合、fn_xmin と fn_tid を用いてキャッシュの有効性が検証されます。
  • plperl_create_sub を通じて、Perl 空間での無名サブルーチンコンパイル結果（reference）が格納されます。

呼び出し元と呼び出し先

  • **呼び出し元**: compile_plperl_function（生成）、plperl_call_handler（実行時の参照）、free_plperl_function（解放）。
  • **呼び出し先**: 本構造体はデータ保持が主目的であり、関数ポインタを直接保持して実行するのではなく、FmgrInfo を介して PostgreSQL の型変換関数を呼び出します。

関連データ構造

  • plperl_interp_desc: 関数が作成された Perl インタプリタの記述子。
  • FmgrInfo: PostgreSQL の関数マネージャ情報。
  • SV: Perl のスカラー値（サブルーチン参照を保持）。

失敗モードと境界条件

  • MULTIPLICITY をサポートしない Perl ビルド環境では、複数のインタプリタを割り当てることができず、実行時にエラーが発生する可能性があります。
  • キャッシュ検証失敗時やメモリ不足時には、fn_cxt の解放と再コンパイルが発生します。

保守担当者向けメモ

  • 本構造体は plperl_proc_hash に直接格納されるのではなく、ポインタを介して管理されます。
  • trftypes は SQL/JSON 変換（Transform）の適用状況を追跡するために使用されます。
  • 参照カウント管理が正しく行われない場合、メモリリークやインタプリタの早期解放を招くため注意が必要です。
