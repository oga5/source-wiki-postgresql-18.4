SignalHandlerForConfigReload

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/interrupt.c:60

目的

PostgreSQLのバックグラウンドプロセスにおいて、設定ファイルの再読み込み要求（SIGHUP）を受信した際に、メインループでの処理を予約し、プロセスを待機状態から起床させるために使用されます。

シグネチャまたは形状

void SignalHandlerForConfigReload(SIGNAL_ARGS)

入力と出力

  • **入力**: SIGNAL_ARGS（シグナルハンドラとして呼び出されるための標準的な引数）。
  • **出力**: なし（void）。副作用としてグローバルフラグおよびラッチの状態を更新します。

前提条件と不変条件

  • 本関数はシグナルハンドラとして実行されるため、非同期安全（async-signal-safe）である必要があります。
  • MyLatch が現在のプロセスに対して適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. ConfigReloadPending フラグを true に設定し、設定再読み込みが保留中であることを記録します。 2. SetLatch(MyLatch) を呼び出し、当該プロセスが待機状態にある場合は即座に起床させます。これにより、メインループが次回の反復で ProcessMainLoopInterrupts() を通じて設定再読み込み処理を実行できるようになります。

呼び出し元と呼び出し先

  • **呼び出し元**: OSのシグナル配送機構。以下の各モジュールで SIGHUP シグナルのハンドラとして登録されています。
  • contrib/pg_prewarm/autoprewarm.c
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/bgwriter.c
  • src/backend/postmaster/checkpointer.c
  • src/backend/postmaster/pgarch.c
  • src/backend/postmaster/syslogger.c
  • src/backend/postmaster/walsummarizer.c
  • src/backend/postmaster/walwriter.c
  • src/backend/replication/logical/applyparallelworker.c
  • src/backend/replication/logical/launcher.c
  • **呼び出し先**: SetLatch（src/backend/storage/ipc/latch.c）。

関連データ構造

  • ConfigReloadPending: 設定再読み込み要求の有無を保持する sig_atomic_t 型のフラグ。
  • MyLatch: 現在のプロセスに関連付けられたラッチ構造体。

失敗モードと境界条件

  • シグナルハンドラ内での実行であるため、複雑な処理やメモリ確保は行いません。
  • SetLatch はシグナルハンドラ内での呼び出しを想定して設計されています。

保守担当者向けメモ

  • 本関数はシグナルハンドラ内での非同期安全性を維持するため、フラグ設定とラッチ通知のみを行う最小限の設計となっています。
  • 実際の再読み込み処理は、メインループ内で呼び出される ProcessMainLoopInterrupts() が ConfigReloadPending フラグを検知することで実行されます。
