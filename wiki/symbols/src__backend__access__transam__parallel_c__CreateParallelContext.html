CreateParallelContext

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/parallel.c:172

目的

並列クエリ実行のための ParallelContext 構造体を初期化し、ワーカープロセスがリーダーのトランザクション環境を継承するための準備を行う。

シグネチャまたは形状

ParallelContext *CreateParallelContext(const char *library_name, const char *function_name, int nworkers)

入力と出力

  • **入力**:
  • library_name: ワーカーのエントリポイントを含む共有ライブラリ名。
  • function_name: ワーカーのメイン関数名。
  • nworkers: 起動するワーカーの数。
  • **出力**:
  • 初期化された ParallelContext 構造体へのポインタ。

前提条件と不変条件

  • IsInParallelMode() が真であること（並列モード外での作成は安全ではないためアサートされる）。
  • nworkers は 0 以上の整数であること。

アルゴリズムまたは振る舞い

1. TopTransactionContext にメモリコンテキストを切り替え、ParallelContext 構造体をゼロ初期化して割り当てる。 2. 現在のサブトランザクション ID を取得し、構造体に格納する。 3. 指定されたライブラリ名と関数名をコピーして保持する。 4. 現在の error_context_stack を保存する。 5. 共有メモリのレイアウトを計算するための shm_toc_estimator を初期化する。 6. 作成したコンテキストをグローバルな pcxt_list に追加する。 7. 元のメモリコンテキストに復帰する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/nbtree/nbtsort.c
  • src/backend/commands/vacuumparallel.c
  • src/backend/executor/execParallel.c
  • **呼び出し先**:
  • MemoryContextSwitchTo, palloc0, GetCurrentSubTransactionId, pstrdup, shm_toc_initialize_estimator, dlist_push_head

関連データ構造

  • ParallelContext: 並列実行の単位を管理する構造体。
  • shm_toc_estimator: 共有メモリのサイズ見積もり用構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる。
  • 並列モード外で呼び出された場合、アサート失敗によりプロセスが停止する。

保守担当者向けメモ

  • 本関数は TopTransactionContext 上でメモリを確保するため、トランザクション終了時に適切に破棄される必要がある。
  • library_name と function_name は pstrdup されているため、呼び出し側で解放する必要はない。
