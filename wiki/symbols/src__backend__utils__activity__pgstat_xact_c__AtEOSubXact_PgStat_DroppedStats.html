AtEOSubXact_PgStat_DroppedStats

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_xact.c:135

目的

サブトランザクション終了時（コミットまたはアボート）に、保留中の統計情報の作成・削除要求を処理し、親トランザクションの状態へ統合または破棄を行う。

シグネチャまたは形状

static void AtEOSubXact_PgStat_DroppedStats(PgStat_SubXactStatus *xact_state, bool isCommit, int nestDepth)

入力と出力

  • xact_state: 処理対象のサブトランザクションの統計状態。
  • isCommit: サブトランザクションがコミットされた場合は true、アボートされた場合は false。
  • nestDepth: 現在のネスト深さ。
  • 出力: なし（副作用として統計状態の更新およびメモリ解放を行う）。

前提条件と不変条件

  • xact_state->pending_drops リスト内の各アイテムは、トランザクションのライフサイクルに従って適切に管理されている必要がある。
  • 関数終了時に xact_state->pending_drops が空であること（Assert により検証）。

アルゴリズムまたは振る舞い

1. pending_drops リストが空であれば即座に終了する。 2. pgstat_get_xact_stack_level(nestDepth - 1) を呼び出し、親トランザクションの状態を取得する。 3. リスト内の各 PgStat_PendingDroppedStatsItem を走査する。

  • **アボート時 (!isCommit) かつ作成要求 (pending->is_create)**: 統計オブジェクトを物理的に削除し、メモリを解放する。削除失敗時は not_freed_count をインクリメントする。
  • **コミット時 (isCommit)**: 統計オブジェクトの削除要求を親トランザクションの pending_drops リストへ移動する（親がアボートした際に備えるため）。
  • **その他**: 不要なアイテムとしてメモリを解放する。

    • 4. 削除失敗が発生した場合は pgstat_request_entry_refs_gc() を呼び出し、ガベージコレクションを要求する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat_xact.c 内の AtEOSubXact_PgStat から呼び出される。
  • **呼び出し先**:
  • pgstat_get_xact_stack_level: 親トランザクション状態の取得。
  • pgstat_drop_entry: 統計オブジェクトの物理削除。
  • pgstat_request_entry_refs_gc: 参照カウントのガベージコレクション要求。
  • pfree: メモリ解放。

関連データ構造

  • PgStat_SubXactStatus: サブトランザクションの統計状態を保持。
  • PgStat_PendingDroppedStatsItem: トランザクション内で保留された統計変更アイテム。
  • dclist: pending_drops を管理する双方向リスト。

失敗モードと境界条件

  • pgstat_drop_entry が失敗した場合、not_freed_count がカウントされ、関数終了時に pgstat_request_entry_refs_gc() が実行される。
  • サブトランザクションがネストの最上位（トップレベル）である場合、nestDepth - 1 の扱いには注意が必要（本関数はサブトランザクション用として設計されている）。

保守担当者向けメモ

  • 本関数は、統計情報の「作成」と「削除」がトランザクションの成否と逆の挙動をとる点に注意すること。
  • コミット時に親へ移動させる処理は、統計オブジェクトの削除がトランザクションの最終的なコミットまで確定できないために必要である。
