gbt_uuidgt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_uuid.c:36

目的

2つの pg_uuid_t 値を比較し、第1引数が第2引数より大きいかどうかを判定します。

シグネチャまたは形状

static bool gbt_uuidgt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の pg_uuid_t 型へのポインタ（void * として渡される）。
  • flinfo: 関数呼び出し情報（本関数内では未使用）。
  • **出力**:
  • bool: a > b であれば true、それ以外は false を返します。

前提条件と不変条件

  • a および b は有効な pg_uuid_t 構造体を指している必要があります。
  • 比較は uuid_internal_cmp を介して実行され、UUID のバイト列比較に基づいた順序付けが行われます。

アルゴリズムまたは振る舞い

1. a と b を pg_uuid_t 型にキャストします。 2. uuid_internal_cmp を呼び出し、その戻り値が 0 より大きい（a > b）かどうかを評価します。 3. 評価結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_uuid.c 内の静的配列（GiST 演算子クラスのサポート関数テーブル）から参照されています。
  • **呼び出し先**:
  • uuid_internal_cmp

関連データ構造

  • pg_uuid_t: PostgreSQL の UUID 型を表現する構造体。

失敗モードと境界条件

  • 引数が NULL の場合の動作は定義されていません。
  • UUID のバイト列比較はエンディアンに依存せず、一貫した順序を維持するように設計されています。

保守担当者向けメモ

  • 本関数は btree_gist 拡張における UUID 型の比較演算子（>）のバックエンド実装です。
  • ロジックの詳細は uuid_internal_cmp の実装に依存します。
