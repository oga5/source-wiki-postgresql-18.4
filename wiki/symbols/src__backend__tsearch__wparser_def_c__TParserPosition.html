TParserPosition

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/wparser_def.c:229

目的

全文検索パーサーにおいて、複雑な構造（URLやXMLタグなど）を解析する際の再帰的な状態遷移を管理するためのスタック要素として機能します。

シグネチャまたは形状

typedef struct TParserPosition
{
    int            posbyte;        /* バイト単位の現在位置 */
    int            poschar;        /* 文字単位の現在位置 */
    int            charlen;        /* 現在の文字の長さ */
    int            lenbytetoken;   /* トークン累積のバイト長 */
    int            lenchartoken;   /* トークン累積の文字長 */
    TParserState   state;          /* 現在のパーサー状態 */
    struct TParserPosition *prev;  /* スタックの前の要素へのポインタ */
    const TParserStateActionItem *pushedAtAction; /* 遷移をトリガーしたアクション */
} TParserPosition;

入力と出力

  • 入力: newTParserPosition 関数により、既存の TParserPosition を引数として新しいインスタンスが生成されます。
  • 出力: パーサーのスタック構造の一部として保持され、状態の復元や追跡に使用されます。

前提条件と不変条件

  • prev ポインタにより、解析の履歴をスタックとして保持します。
  • palloc を使用してメモリが割り当てられます。

アルゴリズムまたは振る舞い

  • newTParserPosition 関数は、現在の状態をコピーして新しいスタック要素を作成します。
  • パーサーが特定のトークン構造（URL等）に入り込む際、現在の状態を TParserPosition としてスタックにプッシュし、解析終了後に prev を参照して状態を復元します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tsearch/wparser_def.c 内の解析ロジック（prsd_nexttoken 等の内部処理）から、状態遷移のスタック操作として参照されます。
  • 呼び出し先: palloc（メモリ確保）、memcpy/memset（構造体の初期化）。

関連データ構造

  • TParserState: TParserPosition 内で保持される現在の解析状態。
  • TParserStateActionItem: 状態遷移を定義するテーブル要素。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • スタックの深さが過度になった場合の制限については、ソースコードの記述からは明示されていません。

保守担当者向けメモ

  • 本構造体は wparser_def.c 内の複雑な状態遷移を支える重要なスタック要素です。
  • newTParserPosition 関数で memcpy を使用しているため、構造体にポインタ以外の動的メモリ管理が必要なメンバを追加する場合は注意してください。
