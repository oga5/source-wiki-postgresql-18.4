read_text_file

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/genfile.c:210

目的

サーバー上の指定されたファイルを読み込み、テキストデータ（text *）として返却します。

シグネチャまたは形状

static text *read_text_file(const char *filename, int64 seek_offset, int64 bytes_to_read, bool missing_ok)

入力と出力

  • **入力**:
  • filename: 読み込むファイルのパス。
  • seek_offset: 読み込みを開始するオフセット位置。
  • bytes_to_read: 読み込むバイト数。
  • missing_ok: ファイルが存在しない場合にエラーとせず NULL を返すかどうかのフラグ。
  • **出力**:
  • 成功時: ファイル内容を含む text * 型のポインタ。
  • 失敗時（missing_ok が真でファイル不在の場合）: NULL。

前提条件と不変条件

  • 呼び出し前に convert_and_check_filename 等でパスの検証と権限チェックが行われていることが前提です。
  • 返却されるデータは、PostgreSQL の有効な文字列エンコーディングに従っている必要があります。

アルゴリズムまたは振る舞い

1. read_binary_file を呼び出し、指定された範囲のデータを bytea 型として取得します。 2. read_binary_file が NULL を返した場合は、そのまま NULL を返します。 3. データが取得できた場合、pg_verifymbstr を使用して、そのバイト列が現在のデータベースエンコーディングとして妥当な文字列であるか検証します。 4. 検証を通過した後、bytea * を text * にキャストして返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/genfile.c 内の関数（pg_read_file_common 等のラッパー関数から呼び出されることが示唆されています）。
  • **呼び出し先**:
  • read_binary_file: ファイルのバイナリ読み込みを実行。
  • pg_verifymbstr: 文字列の妥当性検証を実行。

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • bytea: PostgreSQL の可変長バイナリ型。

失敗モードと境界条件

  • pg_verifymbstr による検証で不正なマルチバイト文字列が検出された場合、エラーが発生します。
  • read_binary_file が内部でファイルアクセスエラー（権限不足やファイルロック等）を発生させた場合、そのエラーが伝播します。

保守担当者向けメモ

  • 本関数は read_binary_file の結果をテキストとして解釈するラッパーです。エンコーディングの検証が必須であるため、バイナリデータそのものを扱う場合は read_binary_file を直接使用してください。
  • pg_verifymbstr の第3引数は false であり、不正な文字列に対してはエラーを投げる挙動となります。
