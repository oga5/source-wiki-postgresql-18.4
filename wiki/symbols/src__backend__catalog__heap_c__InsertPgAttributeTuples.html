InsertPgAttributeTuples

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/heap.c:730

目的

システムカタログ pg_attribute に対して、指定されたタプル記述子（TupleDesc）に基づく属性情報を一括挿入する。

シグネチャまたは形状

void InsertPgAttributeTuples(Relation pg_attribute_rel,
                             TupleDesc tupdesc,
                             Oid new_rel_oid,
                             const FormExtraData_pg_attribute tupdesc_extra[],
                             CatalogIndexState indstate)

入力と出力

  • pg_attribute_rel: 挿入対象の pg_attribute リレーション。
  • tupdesc: 挿入する属性定義を含むタプル記述子。
  • new_rel_oid: 新規リレーションの OID（指定しない場合は InvalidOid）。
  • tupdesc_extra: 統計情報やオプションを含む追加データ（任意）。
  • indstate: インデックス更新用の状態管理構造体（NULL の場合、内部で生成）。

前提条件と不変条件

  • 本関数はトランザクション内で呼び出される必要がある。
  • pg_attribute_rel は pg_attribute カタログとして正しくオープンされていること。

アルゴリズムまたは振る舞い

1. MAX_CATALOG_MULTI_INSERT_BYTES に基づき、メモリ効率を考慮したバッチサイズで TupleTableSlot を確保する。 2. tupdesc から各属性の情報を抽出し、slot に値を設定する。new_rel_oid が有効な場合は attrelid を上書きする。 3. attstattarget や attoptions は tupdesc_extra が提供されればその値を、なければ NULL として扱う。 4. attacl, attfdwoptions, attmissingval は常に NULL として設定される。 5. バッチサイズに達するか、全属性の処理が完了した時点で CatalogTuplesMultiInsertWithInfo を呼び出し、カタログおよび関連インデックスを一括更新する。 6. 最後に確保したスロットを解放し、必要に応じてインデックス状態をクローズする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/heap.c 内のテーブル作成処理、src/backend/catalog/index.c、および src/backend/commands/tablecmds.c から参照されている。
  • **呼び出し先**: CatalogOpenIndexes, CatalogTuplesMultiInsertWithInfo, CatalogCloseIndexes 等のカタログ操作関数を呼び出す。

関連データ構造

  • Relation: リレーション記述子。
  • TupleDesc: タプル記述子。
  • FormExtraData_pg_attribute: 属性の追加情報構造体。
  • TupleTableSlot: タプル挿入用の中間バッファ。

失敗モードと境界条件

  • tupdesc が空の場合、挿入は行われない。
  • メモリ不足時には palloc がエラーを発生させる。
  • カタログのインデックス更新失敗時は、PostgreSQL の標準的なエラーハンドリングによりトランザクションがアボートされる。

保守担当者向けメモ

  • 本関数は pg_attribute への挿入を最適化するために一括挿入（Multi-Insert）を利用している。
  • attacl 等の特定の列がハードコードで NULL に設定されている点に注意。
  • new_rel_oid の扱いは、既存の属性定義を流用して新しいテーブルを作成する際の OID 付け替えを考慮している。
