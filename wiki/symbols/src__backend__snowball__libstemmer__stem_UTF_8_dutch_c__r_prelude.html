r_prelude

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_dutch.c:147

目的

オランダ語のステミング処理における前処理を実行する。具体的には、アクセント記号の正規化、'y' の変換、および母音に挟まれた 'i' や 'y' の処理を行う。

シグネチャまたは形状

static int r_prelude(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z （Snowball ステミング環境へのポインタ）
  • 出力: 成功時に 1 を返し、失敗時（メモリ不足や処理エラー）には負の値を返す。

前提条件と不変条件

  • z は dutch_UTF_8_create_env によって初期化されている必要がある。
  • z->p には UTF-8 エンコードされた文字列が格納されている。

アルゴリズムまたは振る舞い

1. **アクセント記号の正規化**: a_0 テーブルに基づき、特定の文字パターンを検索して置換する。 2. **'y' の変換**: 単語内の 'y' を特定の形式（s_5）に置換する。 3. **母音間処理**: 母音（g_v）に挟まれた 'i' を 'I' に、'y' を 'Y' に置換するループ処理を行う。 4. 処理中、z->bra（開始位置）と z->ket（終了位置）を更新しながら、slice_from_s を用いて文字列の置換を行う。

呼び出し元と呼び出し先

  • 呼び出し元: dutch_UTF_8_stem（メインエントリポイント）から呼び出されることが想定される。
  • 呼び出し先: find_among, slice_from_s, skip_utf8, in_grouping_U などの Snowball ランタイム関数。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • a_0: r_prelude で使用される接尾辞や文字パターンの検索テーブル。
  • g_v: 母音グループを定義するデータ構造。

失敗モードと境界条件

  • slice_from_s が負の値を返した場合、そのエラーコードをそのまま伝播させる。
  • 文字列の末尾に達した場合や、マッチング対象が見つからない場合は、現在の位置を維持して処理を継続または終了する。

  • g_v や a_0 などのテーブル定義は、オランダ語の言語ルールに依存している。
