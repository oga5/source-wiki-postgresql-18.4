unicode_strlower

定義

  • 種別: function
  • 定義箇所: src/common/unicode_case.c:100

目的

UTF-8 文字列を小文字に変換します。PostgreSQL のフロントエンドおよびバックエンドの両方から利用可能な Unicode 規格準拠のユーティリティです。

シグネチャまたは形状

size_t unicode_strlower(char *dst, size_t dstsize, const char *src, ssize_t srclen, bool full);

入力と出力

  • dst: 変換後の文字列を格納するバッファ。
  • dstsize: dst バッファのサイズ（バイト単位）。0 の場合、必要なバッファサイズを計算して返します。
  • src: 変換元の UTF-8 文字列。
  • srclen: src の長さ（バイト単位）。-1 の場合はヌル終端文字列として扱われます。
  • full: true の場合、ギリシャ語の最終シグマ処理などの文脈依存の特殊マッピングを適用します。
  • 戻り値: 変換後の文字列に必要なバッファサイズ（バイト単位）。

前提条件と不変条件

  • src は有効な UTF-8 文字列である必要があります。
  • dstsize が 0 の場合、変換は行われず、必要なサイズのみが返されます。

アルゴリズムまたは振る舞い

内部関数 convert_case を呼び出し、以下の手順で処理を行います。 1. 入力文字列を UTF-8 として走査します。 2. 各文字に対して casemap 関数を適用し、変換ルール（単純変換または特殊変換）を決定します。 3. full フラグが有効な場合、check_special_conditions を介して文脈依存の特殊マッピングを適用します。 4. 変換された文字を UTF-8 にエンコードし、出力バッファに書き込みます。

関連データ構造

失敗モードと境界条件

  • dstsize が変換後の文字列を格納するのに不十分な場合、バッファオーバーフローを防ぐための切り詰めやエラーハンドリングが convert_case の実装に依存します。
  • ASCII 文字（0x80 未満）に対しては、テーブル参照による高速パスが適用されます。

保守担当者向けメモ

  • ギリシャ語の最終シグマ（Final Sigma）などの文脈依存処理は check_final_sigma で前後関係を走査して判定されます。
  • タイトルケース変換とは異なり、単語境界の判定は行いません。
