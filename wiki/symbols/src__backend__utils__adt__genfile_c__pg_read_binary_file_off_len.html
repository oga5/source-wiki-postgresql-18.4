pg_read_binary_file_off_len

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/genfile.c:347

目的

サーバー上の指定されたファイルから、オフセットと長さを指定してバイナリデータ（bytea）を読み込みます。

シグネチャまたは形状

Datum pg_read_binary_file_off_len(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • filename_t (text): 読み込むファイルのパス。
  • seek_offset (int64): 読み込みを開始するオフセット位置。
  • bytes_to_read (int64): 読み込むバイト数。
  • **出力**:
  • bytea: 読み込まれたファイルの内容。ファイルが存在しない場合や読み込みに失敗した場合は NULL を返します。

前提条件と不変条件

  • 呼び出し元は PostgreSQL の関数呼び出し規約に従う必要があります。
  • 内部で pg_read_binary_file_common を呼び出すため、実行には pg_read_server_files ロールの権限が必要です。

アルゴリズムまたは振る舞い

1. 引数からファイルパス、オフセット、読み込みバイト数を取得します。 2. pg_read_binary_file_common を呼び出し、以下の引数を渡します：

  • missing_ok = false: ファイルが存在しない場合にエラーを発生させます。
  • include_error = false: エラー発生時に例外を投げず、呼び出し元へ制御を戻します。

    • 3. 戻り値が NULL であれば、そのまま NULL を返します。 4. 戻り値が有効なデータであれば、PG_RETURN_BYTEA_P マクロを使用して bytea 型として結果を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: pg_read_binary_file_common
  • **呼び出し元**: 索引化された情報には、本関数を直接呼び出している特定のSQLインターフェースや内部関数は含まれていません。

関連データ構造

  • Datum: PostgreSQL の汎用データ型表現。
  • bytea: PostgreSQL の可変長バイナリデータ型。

失敗モードと境界条件

  • 指定されたパスがデータディレクトリやログディレクトリの範囲外である場合、convert_and_check_filename によりエラーが発生します。
  • ファイルが存在しない場合、missing_ok が false であるためエラーが発生します。
  • 権限がない場合、アクセス拒否エラーが発生します。

保守担当者向けメモ

  • 本関数は pg_read_binary_file_off_len_missing と対になる存在であり、missing_ok フラグの扱いが異なります。
  • 修正を行う際は、src/backend/utils/adt/genfile.c 内の他のファイル読み込み関数と一貫性を保つようにしてください。
