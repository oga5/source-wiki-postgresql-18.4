pq_verify_peer_name_matches_certificate_ip

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/libpq/fe-secure-common.c:156

目的

SSL/TLSサーバー証明書に含まれるバイナリ形式のIPアドレスと、接続先ホストのIPアドレスを照合し、一致を確認する。

シグネチャまたは形状

int pq_verify_peer_name_matches_certificate_ip(PGconn *conn,
                                               const unsigned char *ipdata,
                                               size_t iplen,
                                               char **store_name);

入力と出力

  • **入力**:
  • conn: 接続状態を保持する PGconn 構造体。
  • ipdata: 証明書から抽出されたネットワークバイトオーダーのIPアドレスデータ。
  • iplen: ipdata のバイト長（IPv4なら4、IPv6なら16）。
  • store_name: 証明書のIPアドレスを人間が読める形式に変換した文字列を格納するためのポインタ。
  • **出力**:
  • 戻り値: 一致した場合は 1、一致しない場合は 0、エラー発生時は -1 を返す。
  • *store_name: 成功時に変換後の文字列（呼び出し元で解放が必要）、失敗時は NULL が設定される。

前提条件と不変条件

  • conn->connhost[conn->whichhost].host に有効なホスト名またはIPアドレス文字列が格納されていること。
  • ipdata はネットワークバイトオーダーであること。
  • iplen が4または16以外の場合、エラーとして扱われる。

アルゴリズムまたは振る舞い

1. conn から接続先ホスト文字列を取得し、空でないことを検証する。 2. iplen に基づき、IPv4 (inet_aton) または IPv6 (inet_pton) を用いてホスト文字列をバイナリ形式に変換する。 3. 変換後のバイナリデータと ipdata を memcmp で比較し、一致すれば match を 1 に設定する。 4. pg_inet_net_ntop を使用して、証明書のIPアドレスを人間が読める形式に変換し、store_name に格納する。 5. 変換失敗や不正な長さの場合は libpq_append_conn_error を呼び出し、エラー情報を PGconn に記録して -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-secure-openssl.c 内の証明書検証ロジックから呼び出されることが確認されている。
  • **呼び出し先**: inet_aton, inet_pton (プラットフォーム依存), memcmp, pg_inet_net_ntop, libpq_append_conn_error, strdup。

関連データ構造

  • PGconn: 接続情報およびエラーメッセージ格納用。
  • in_addr / in6_addr: IPアドレスのバイナリ表現用構造体。

失敗モードと境界条件

  • **ホスト名未指定**: host が空の場合、エラーとして -1 を返す。
  • **不正なIP長**: iplen が4でも16でもない場合、エラーとして -1 を返す。
  • **変換失敗**: pg_inet_net_ntop による文字列変換に失敗した場合、エラーとして -1 を返す。
  • **不一致**: IPアドレスが一致しない場合、match は 0 のまま正常終了する。

保守担当者向けメモ

  • inet_aton の使用は、inet_pton では許容されない代替的なIPv4表記をサポートするための意図的な選択である。
  • IPv6のサポートは HAVE_INET_PTON マクロに依存している。
  • store_name は strdup で確保されるため、呼び出し元で適切にメモリ管理を行う必要がある。
