CalculateShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/ipci.c:88

目的

PostgreSQL の起動時に必要な共有メモリの総サイズを算出します。各サブシステムが必要とするメモリ量を合算し、システム全体で確保すべき共有メモリブロックの容量を決定します。

シグネチャまたは形状

Size CalculateShmemSize(int *num_semaphores)

入力と出力

  • **入力**: num_semaphores (int へのポインタ)。NULL でない場合、算出された必要なセマフォの総数が格納されます。
  • **出力**: Size 型の戻り値。計算された共有メモリの総バイト数（8192 バイト単位に切り上げ済み）。

前提条件と不変条件

  • 合算処理において add_size 関数を使用し、size_t のオーバーフローを防止しています。
  • 算出されるサイズは、各サブシステムの ShmemSize 関数群の戻り値と、ロードされた拡張モジュールによる要求 (total_addin_request) の合計です。

アルゴリズムまたは振る舞い

1. ProcGlobalSemas() を呼び出し、必要なセマフォ数を取得します。 2. 初期値として 100KB を設定し、各サブシステム（バッファマネージャ、ロックマネージャ、WAL、DSM 等）の ShmemSize 関数を順次呼び出して size に加算します。 3. 拡張モジュールから要求された共有メモリ量 (total_addin_request) を加算します。 4. 最終的なサイズを 8192 バイトの倍数に切り上げます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/ipci.c 内の CreateSharedMemoryAndSemaphores 等から呼び出されます。
  • **呼び出し先**:
  • 各サブシステムの ShmemSize 関数群（BufferManagerShmemSize, LockManagerShmemSize, XLOGShmemSize 等多数）。
  • add_size（サイズ加算用ユーティリティ）。
  • ProcGlobalSemas（セマフォ数算出）。

関連データ構造

  • ShmemIndexEnt: 共有メモリインデックスの各エントリサイズ算出に使用。
  • total_addin_request: 拡張モジュールによって予約された共有メモリ量。

失敗モードと境界条件

  • add_size を使用しているため、計算過程で size_t の最大値を超えた場合はエラーが発生する可能性があります。
  • 戻り値は常に 8192 バイトの倍数となるよう調整されます。

保守担当者向けメモ

  • 新しいサブシステムを追加して共有メモリを使用する場合、必ず本関数内でそのサブシステムの ShmemSize 関数を呼び出すように修正してください。
  • EXEC_BACKEND 環境（Windows 等）とそれ以外で初期化パスが異なる場合がありますが、本関数は両方のパスで共有メモリサイズを確定させるために使用されます。
