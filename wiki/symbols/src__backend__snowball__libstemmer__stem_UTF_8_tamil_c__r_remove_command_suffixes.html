r_remove_command_suffixes

目的

タミル語のステミング処理において、単語末尾の命令形接尾辞を除去し、語幹を正規化する。

シグネチャまたは形状

static int r_remove_command_suffixes(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境構造体）
  • 出力: int（処理成功時に 1、失敗または条件不適合時に 0、エラー時に負の値を返す）

前提条件と不変条件

  • r_has_min_length(z) が正の値を返すこと（最小長要件を満たすこと）。
  • 処理対象の文字列末尾が特定のバイト値（191）で終わっていること。

アルゴリズムまたは振る舞い

1. r_has_min_length を呼び出し、最小長チェックを行う。 2. 環境の検索範囲（lb, c）を調整し、末尾から検索を開始する。 3. 文字列末尾のバイト値が 191 であるかを確認し、かつ a_15 配列に定義されたパターンと find_among_b で照合する。 4. マッチした場合、slice_del を用いて該当する接尾辞を削除する。 5. 検索ポインタを初期位置に戻し、成功を示す 1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_tamil.c 内のメイン処理フロー（1399行目付近）。
  • 呼び出し先: r_has_min_length, find_among_b, slice_del。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_15: 命令形接尾辞のパターンを定義した among 構造体の配列。

失敗モードと境界条件

  • r_has_min_length が 0 または負の値を返した場合、処理を中断する。
  • 文字列長が不足している場合や、末尾バイトが 191 でない場合、あるいは a_15 にマッチしない場合は 0 を返す。
  • slice_del がエラー（負の値）を返した場合は、その値をそのまま伝播させる。
