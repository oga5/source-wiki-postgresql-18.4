gbt_oid_compress

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_oid.c:119

目的

GiSTインデックスに格納される Oid 型の値を、インデックス構造に適した形式に圧縮・変換します。

シグネチャまたは形状

Datum gbt_oid_compress(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体へのポインタを受け取ります。
  • **出力**: Datum 型として、圧縮されたインデックスキー（oidKEY 構造体）を返します。

前提条件と不変条件

  • tinfo 構造体（gbtree_ninfo 型）が、Oid 型の比較ロジックを保持している必要があります。
  • 入力された GISTENTRY は有効な Oid 値を指している必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から GISTENTRY 構造体を取得します。 2. btree_utils_num.h で定義された汎用関数 gbt_num_compress を呼び出します。 3. その際、Oid 型固有の定義を含む tinfo 構造体を引数として渡し、数値型の圧縮ロジックを委譲します。 4. PG_RETURN_POINTER を通じて、変換された結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのGiSTインデックス構築プロセス（直接の呼び出し元詳細はソースコード上では明示されていません）。
  • **呼び出し先**: gbt_num_compress（btree_utils_num.h 定義）。

関連データ構造

  • oidKEY: インデックスノードに格納される Oid の範囲を保持する構造体。
  • tinfo: Oid 型の比較関数やサイズ情報を保持する gbtree_ninfo 構造体。
  • GISTENTRY: GiSTインデックスの各エントリを保持する構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合や、tinfo が正しく初期化されていない場合は予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数はアダプターとして機能しており、実際の圧縮ロジックは btree_utils_num.c に集約されています。ロジックの変更が必要な場合は、まず btree_utils_num.c を確認してください。
