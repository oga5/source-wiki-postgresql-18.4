initcap_wbnext

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_builtin.c:52

目的

Unicode文字列のタイトルケース変換において、単語の境界を判定するためのイテレータ関数です。pg_u_isalnum を用いて文字の属性変化を検出し、単語の開始位置を特定します。

シグネチャまたは形状

static size_t initcap_wbnext(void *state);

入力と出力

  • **入力**: void *state（実際には WordBoundaryState 構造体へのポインタ）。
  • **出力**: 次の単語境界のオフセット（size_t）。文字列の末尾に達した場合は wbstate->len を返します。

前提条件と不変条件

  • state は WordBoundaryState 構造体として初期化されている必要があります。
  • wbstate->str は有効なUTF-8文字列である必要があります。
  • wbstate->offset は常に文字列の有効な文字境界を指している必要があります。

アルゴリズムまたは振る舞い

1. wbstate->offset が文字列の長さを超えるか、終端文字に達するまでループします。 2. 現在のオフセット位置の文字を utf8_to_unicode で取得し、pg_u_isalnum で英数字かどうかを判定します。 3. 初回呼び出し時、または前回の文字と英数字属性（curr_alnum vs prev_alnum）が変化した際に、そのオフセットを単語境界として記録し、現在のオフセットを返します。 4. 属性が変化しない場合は、次の文字へオフセットを進めてループを継続します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale_builtin.c 内の strtitle_builtin
  • src/common/unicode/case_test.c 内のテスト関数群
  • **呼び出し先**:
  • utf8_to_unicode
  • pg_u_isalnum
  • unicode_utf8len

関連データ構造

  • WordBoundaryState: 走査中の文字列、現在のオフセット、直前の文字の属性（英数字か否か）、および初期化状態を保持する構造体。

失敗モードと境界条件

  • 文字列が空の場合、または終端に達した場合は wbstate->len を返します。
  • 不正なUTF-8シーケンスが含まれる場合の挙動は、依存する utf8_to_unicode の実装に準拠します。

保守担当者向けメモ

  • 本関数は複雑な言語ルールを考慮せず、英数字属性の切り替わりのみを単語境界とみなすシンプルな実装です。
  • pg_u_isalnum の判定ロジックは common/unicode_category.h に依存しています。
