TimeTzADT

定義

  • 種別: struct
  • 定義箇所: src/include/utils/date.h:27

目的

PostgreSQLにおける TIME WITH TIME ZONE 型の内部表現を保持するための構造体です。

シグネチャまたは形状

typedef struct
{
    TimeADT     time;   /* 月・年以外のすべての時間単位 */
    int32       zone;   /* 数値形式のタイムゾーン（秒単位） */
} TimeTzADT;

入力と出力

  • 本構造体は、SQL関数インターフェース（fmgr）を介して Datum 型との相互変換が行われます。
  • 変換には DatumGetTimeTzADTP や TimeTzADTPGetDatum などのマクロが利用されます。

前提条件と不変条件

  • time フィールドは TimeADT（int64）型であり、時刻情報を保持します。
  • zone フィールドは、UTCからのオフセットを秒単位の整数値として保持します。

アルゴリズムまたは振る舞い

  • src/backend/utils/adt/date.c 内の関数（例: tm2timetz）において、時刻構造体（pg_tm）から本構造体への変換が行われます。
  • brin_minmax_multi.c 等では、本構造体内の TimeADT 値を用いて減算等の演算が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_time.c
  • src/backend/access/brin/brin_minmax_multi.c
  • src/backend/utils/adt/date.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。メモリ確保には palloc が使用されます。

関連データ構造

  • TimeADT: 時刻の内部表現（int64）。
  • pg_tm: 時刻分解用の構造体。

失敗モードと境界条件

  • メモリ割り当てが必要なコンテキスト（palloc）で使用される場合、メモリ不足によるエラーが発生する可能性があります。
  • タイムゾーンオフセットの範囲外の値が zone に設定された場合の挙動については、本ソース excerpt からは判断できません。

保守担当者向けメモ

  • 本構造体は TimeADT と int32 のペアで構成されており、アライメントやパディングに注意してください。
  • 比較演算や算術演算を実装する際は、time フィールドの単位（int64）と zone フィールドの単位（秒）を適切に扱う必要があります。
