scram_verify_plain_password

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-scram.c:522

目的

提供された平文パスワードが、保存されているSCRAMシークレットと一致するかを検証します。

シグネチャまたは形状

bool scram_verify_plain_password(const char *username, const char *password, const char *secret)

入力と出力

  • **入力**:
  • username: 検証対象のユーザー名（ログ出力用）。
  • password: クライアントから提供された平文パスワード。
  • secret: pg_authid に格納されているSCRAM形式のシークレット文字列。
  • **出力**:
  • bool: パスワードが一致すれば true、一致しない、または解析不能な場合は false を返します。

前提条件と不変条件

  • secret は parse_scram_secret で解析可能な形式である必要があります。
  • pg_saslprep によるパスワードの正規化が試行されます。成功した場合は正規化後のパスワードが使用されます。

アルゴリズムまたは振る舞い

1. parse_scram_secret を呼び出し、secret から反復回数、ハッシュアルゴリズム、ソルト、StoredKey、ServerKey を抽出します。 2. Base64デコードによりソルトを取得します。 3. pg_saslprep を用いて入力パスワードを正規化します。 4. scram_SaltedPassword および scram_ServerKey を使用して、入力パスワードから ServerKey を計算します。 5. timingsafe_bcmp を使用し、計算された ServerKey と保存されていた ServerKey を比較して一致を確認します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/crypt.c:272
  • **呼び出し先**:
  • parse_scram_secret
  • pg_b64_dec_len / pg_b64_decode
  • pg_saslprep
  • scram_SaltedPassword
  • scram_ServerKey
  • timingsafe_bcmp

関連データ構造

  • pg_cryptohash_type: 使用するハッシュアルゴリズムの指定。
  • SCRAM_MAX_KEY_LEN: キー計算用バッファの最大長。

失敗モードと境界条件

  • parse_scram_secret が失敗した場合、ログにエラーを出力し false を返します。
  • ソルトのBase64デコードに失敗した場合、ログにエラーを出力し false を返します。
  • ServerKey の計算中にエラーが発生した場合、elog(ERROR) により処理を中断します。

保守担当者向けメモ

  • 本関数はパスワード検証の最終段階で ServerKey を比較します。これはクライアントが提示した証明（Proof）の検証とは異なるプロセスであることに注意してください。
  • timingsafe_bcmp を使用しているため、比較処理におけるタイミング攻撃耐性が考慮されています。
