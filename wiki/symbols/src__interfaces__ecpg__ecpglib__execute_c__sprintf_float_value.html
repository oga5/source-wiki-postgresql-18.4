sprintf_float_value

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/execute.c:471

目的

C 言語の float 型の値を文字列に変換し、指定された区切り文字を付与してバッファへ書き込みます。

シグネチャまたは形状

void sprintf_float_value(char *ptr, float value, const char *delim)

入力と出力

  • ptr (char*): 書き込み先のバッファ。十分なサイズが確保されている必要があります。
  • value (float): 文字列化する浮動小数点値。
  • delim (const char*): 値の末尾に付与する文字列（区切り文字など）。
  • 出力: なし（ptr が指すメモリ領域を更新）。

前提条件と不変条件

  • ptr は sprintf が書き込み可能な有効なメモリ領域を指していること。
  • value が NaN または Infinity の場合、それぞれ "NaN", "Infinity", "-Infinity" という文字列が生成されます。
  • 通常の数値の場合、%.15g フォーマットで精度を保持して変換されます。

アルゴリズムまたは振る舞い

1. isnan(value) を判定し、真であれば "NaN" と delim を ptr に書き込む。 2. isinf(value) を判定し、真であれば符号に応じて "-Infinity" または "Infinity" と delim を ptr に書き込む。 3. 上記以外の場合、%.15g フォーマット指定子を用いて value を文字列化し、delim を連結して ptr に書き込む。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/interfaces/ecpg/ecpglib/execute.c:727 (配列要素の処理)
  • src/interfaces/ecpg/ecpglib/execute.c:732 (単一値の処理)
  • 呼び出し先:
  • sprintf (標準ライブラリ)
  • isnan, isinf (math.h)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • ptr が指すバッファサイズが不足している場合、バッファオーバーフローが発生します。
  • delim が NULL の場合、sprintf 内部で未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は float 型専用です。double 型を扱う場合は sprintf_double_value を使用してください。
  • %.15g は浮動小数点の精度を維持するための選択ですが、特定のロケール環境下で小数点記号がカンマになる可能性があるため、呼び出し元の環境設定に注意してください。
