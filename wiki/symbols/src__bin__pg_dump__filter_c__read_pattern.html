read_pattern

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/filter.c:301

目的

フィルタファイルからオブジェクト名パターンを読み込み、引用符や修飾子を含む複雑な識別子を解析して正規化された文字列を構築する。

シグネチャまたは形状

static const char *read_pattern(FilterStateData *fstate, const char *str, PQExpBuffer pattern);

入力と出力

  • **入力**:
  • fstate: フィルタ処理の状態管理データ。
  • str: 解析対象の文字列ポインタ。
  • pattern: 解析結果を格納する動的文字列バッファ（PQExpBuffer）。
  • **出力**:
  • 戻り値: 解析が終了した位置の文字列ポインタを返す。

前提条件と不変条件

  • 入力文字列 str は、フィルタ設定ファイルの行の一部であること。
  • fstate->exit_nicely が有効な終了処理関数として定義されていること。

アルゴリズムまたは振る舞い

1. 先頭の空白をスキップする。 2. 文字列が空の場合は pg_log_filter_error を呼び出し、fstate->exit_nicely で終了する。 3. #（コメント開始）または終端文字に達するまでループし、以下の処理を行う：

  • 通常の識別子文字を pattern に追加する。
  • "（引用符）を検出した場合、read_quoted_string を呼び出して引用符内の文字列を処理する。
  • ,（カンマ）を検出した場合、, を追加する。
  • ., (, ) を検出した場合、その文字をそのまま追加する。
  • 処理中に空白を検出した場合、後続の文字追加時にスペースを挿入するかを制御する。

    • 4. 解析した位置のポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/filter.c 内の filter_read_item から呼び出される。
  • **呼び出し先**:
  • read_quoted_string
  • pg_log_filter_error
  • fstate->exit_nicely
  • appendPQExpBufferChar / appendPQExpBufferStr

関連データ構造

  • FilterStateData: フィルタ処理の状態を保持する構造体。
  • PQExpBuffer: 文字列を動的に構築するためのバッファ構造体。

失敗モードと境界条件

  • オブジェクト名パターンが指定されていない場合、エラーログを出力してプログラムを終了する。
  • # 文字以降はコメントとして無視される。

保守担当者向けメモ

  • 本関数は pg_get_line_buf 等で読み込まれた行をパースする。引用符の扱いや修飾子の連結ロジックは pg_dump のオブジェクト名形式に依存しているため、変更時は注意が必要。
