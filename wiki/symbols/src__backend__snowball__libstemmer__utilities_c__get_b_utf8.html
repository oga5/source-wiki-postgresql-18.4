get_b_utf8

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/utilities.c:93

目的

UTF-8 エンコードされた文字列バッファから、現在のカーソル位置より前の文字を後方へデコードし、その文字コードとバイト数を取得します。

シグネチャまたは形状

static int get_b_utf8(const symbol * p, int c, int lb, int * slot)

入力と出力

  • **入力**:
  • p: 文字列バッファへのポインタ。
  • c: 現在のカーソル位置（デコード開始位置の直後）。
  • lb: 下限境界（これ以上前には戻らない）。
  • slot: デコードされた文字コードを格納する整数へのポインタ。
  • **出力**:
  • 戻り値: デコードされた文字のバイト数（1〜4）。境界に達した場合は 0 を返す。
  • *slot: デコードされた文字の Unicode コードポイント。

前提条件と不変条件

  • c は p 内の有効なインデックスであること。
  • lb はバッファの有効な開始位置または境界を示すこと。

アルゴリズムまたは振る舞い

1. c <= lb の場合、処理を中断し 0 を返す。 2. c をデクリメントし、バッファからバイトを取得する。 3. 取得したバイトの先頭ビットパターンに基づき、UTF-8 のマルチバイトシーケンスを逆方向に解析する。

  • 1バイト文字（0xxxxxxx）: そのまま slot に格納し 1 を返す。
  • 2〜4バイト文字: 後続バイトを順次読み込み、ビット演算でコードポイントを再構築して、読み込んだバイト数を返す。

    • 4. 解析中に lb に到達した場合は、その時点までの結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/libstemmer/utilities.c 内の 132 行目および 156 行目から呼び出されています。
  • **呼び出し先**: 直接的な関数呼び出しはありません。

関連データ構造

  • symbol: 文字列を保持する型（header.h で定義）。

失敗モードと境界条件

  • c <= lb: 境界条件に達したことを示し、0 を返して終了します。
  • 不正な UTF-8 シーケンス: 本関数はシーケンスの妥当性検証よりも、ビットパターンに基づく抽出を優先します。

保守担当者向けメモ

  • 本関数は後方検索用です。前方検索には get_utf8 を使用してください。
  • slot に書き込まれる値は、UTF-8 のバイト列から復元された Unicode コードポイントです。
