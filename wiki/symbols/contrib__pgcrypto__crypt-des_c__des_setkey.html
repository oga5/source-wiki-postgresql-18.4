des_setkey

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/crypt-des.c:395

目的

入力された 8 バイトのキーを DES アルゴリズムで使用可能な内部形式（16 ラウンド分のサブキー）へ変換・展開します。

シグネチャまたは形状

static int des_setkey(const char *key)

入力と出力

  • **入力**: key (8 バイトの文字列ポインタ)
  • **出力**: 成功時は 0 を返します。

前提条件と不変条件

  • des_initialised が偽の場合、内部で des_init() を呼び出し、マスクテーブルを初期化します。
  • 入力キーが前回と同じである場合、再計算をスキップする最適化が行われます（ゼロキーを除く）。

アルゴリズムまたは振る舞い

1. **キーの読み込み**: pg_ntoh32 を使用して、入力キーをビッグエンディアンとして 2 つの 32 ビット値 (rawkey0, rawkey1) に変換します。 2. **キー置換**: key_perm_maskl および key_perm_maskr を用いて、初期置換と 28 ビットのサブキーへの分割を行います。 3. **サブキー生成**: 16 ラウンドにわたり、以下の処理を繰り返します。

  • key_shifts に基づいてサブキーを回転させます。
  • comp_maskl および comp_maskr を使用して圧縮置換を行い、暗号化用 (en_keys) および復号用 (de_keys) のサブキーを生成・格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/crypt-des.c 内の関数（crypt-des.c:679, 720 付近）。
  • **呼び出し先**: des_init()。

関連データ構造

  • en_keysl, en_keysr: 暗号化用サブキー格納配列。
  • de_keysl, de_keysr: 復号用サブキー格納配列。
  • key_perm_maskl, key_perm_maskr, comp_maskl, comp_maskr: 置換テーブル。

失敗モードと境界条件

  • ゼロキー（すべてのビットが 0）の場合、最適化ロジックがスキップされ、毎回再計算が行われます。
  • 入力ポインタ key が 8 バイト未満のメモリ領域を指している場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は DES アルゴリズムの脆弱性のため、既存のパスワードハッシュとの互換性維持のみを目的としています。
  • 8 バイトの配列を uint32 として扱うため、CPU のアライメント制限に依存しない設計となっています。
