ItemIdData

定義

  • 種別: struct
  • 定義箇所: src/include/storage/itemid.h:25

目的

PostgreSQLのバッファページ内における「ラインポインタ（Line Pointer）」として、ページ内のタプル（データ）の位置、長さ、および状態を管理するメタデータ構造体です。

シグネチャまたは形状

typedef struct ItemIdData
{
	unsigned	lp_off:15,		/* ページ先頭からのオフセット */
				lp_flags:2,		/* ラインポインタの状態 */
				lp_len:15;		/* タプルのバイト長 */
} ItemIdData;

入力と出力

  • 本構造体はデータ保持を目的としており、関数としての入力・出力はありません。
  • バッファページ上の ItemIdData 配列に対し、アクセサマクロを通じてオフセットや長さを読み取り、ページ内の実データへアクセスします。

前提条件と不変条件

  • 32ビットのビットフィールド構造体です。
  • lp_flags が LP_REDIRECT の場合、lp_off はタプルのオフセットではなく、次のラインポインタへのインデックスとして解釈されます。
  • メモリレイアウトはコンパイラやアーキテクチャに依存する可能性がありますが、PostgreSQLのページ構造として固定されています。

アルゴリズムまたは振る舞い

  • ページ内のアイテム状態（未使用、使用中、リダイレクト、削除済み）を管理します。
  • ItemIdSet* 系のマクロを通じて状態更新が行われます。
  • 実行時の制御フローは持たず、データ構造とマクロによる状態管理に特化しています。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/access/brin/brin_bloom.c
  • src/backend/access/brin/brin_pageops.c
  • src/backend/access/gin/ginentrypage.c
  • src/backend/access/gin/ginfast.c
  • src/backend/access/gist/gistbuild.c
  • 呼び出し先:
  • 構造体定義のため、特定の関数呼び出し先はありません。

関連データ構造

  • src/include/storage/bufpage.h: ページヘッダーとラインポインタ配列の配置を定義。
  • src/include/storage/item.h: ページ内のアイテムそのものの定義。

失敗モードと境界条件

  • ItemIdSet* 系のマクロは引数 itemId を複数回評価する可能性があるため、副作用のある式（例: ptr++）を渡すと予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体はビットフィールドを使用しているため、メモリレイアウトの変更には注意が必要です。
  • LP_REDIRECT（HOT更新の追跡）など、フラグの意味論は itemid.h 内の定数定義と密接に結びついています。
