pgstat_prepare_report_checksum_failure

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_database.c:144

目的

チェックサムエラー発生時に、統計情報を報告するための準備を行う。後続の報告処理において、共有メモリのDSMセグメントへのマッピング等の重い処理を回避し、即時報告を可能にすることを目的とする。

シグネチャまたは形状

void pgstat_prepare_report_checksum_failure(Oid dboid)

入力と出力

  • **入力**: dboid (Oid) - 統計情報を準備する対象データベースのOID。
  • **出力**: なし。

前提条件と不変条件

  • CritSectionCount が 0 であること（クリティカルセクション内ではないこと）。

アルゴリズムまたは振る舞い

1. Assert(!CritSectionCount) により、クリティカルセクション外であることを検証する。 2. pgstat_get_entry_ref を呼び出し、指定されたデータベース (dboid) に対応する統計エントリの参照をローカルに確保する。この際、create 引数に true を指定し、必要に応じてエントリを作成する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c
  • src/backend/catalog/storage.c
  • src/backend/storage/buffer/bufmgr.c
  • src/test/modules/test_aio/test_aio.c
  • **呼び出し先**:
  • pgstat_get_entry_ref (統計サブシステム内部関数)

関連データ構造

  • PgStat_StatDBEntry: データベース統計を保持する構造体（pgstat_get_entry_ref を通じて操作される）。

失敗モードと境界条件

  • 本関数はメモリ確保を伴う可能性があるため、クリティカルセクション内での呼び出しはアサーションにより禁止されている。

保守担当者向けメモ

  • 本関数は、チェックサムエラー発生時に pgstat_report_checksum_failures_in_db を呼び出す前の事前準備として必須である。
  • 統計情報の更新は pgstat_track_counts フラグの状態に依存する設計となっているため、呼び出し側で考慮が必要な場合がある。
