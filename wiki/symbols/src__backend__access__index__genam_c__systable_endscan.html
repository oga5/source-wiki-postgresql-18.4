systable_endscan

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/genam.c:602

目的

システムカタログスキャン（systable_*）で使用したリソースを解放し、スキャン状態を終了させる。

シグネチャまたは形状

void systable_endscan(SysScanDesc sysscan)

入力と出力

  • **入力**: sysscan (SysScanDesc) - 終了対象となるシステムスキャン記述子。
  • **出力**: なし。

前提条件と不変条件

  • sysscan は systable_beginscan によって初期化されていること。
  • 本関数を呼び出した後、sysscan ポインタは無効となるため参照してはならない。

アルゴリズムまたは振る舞い

1. **スロット解放**: sysscan->slot が存在する場合、ExecDropSingleTupleTableSlot を呼び出して解放し、ポインタを NULL に設定する。 2. **スキャン終了**:

  • インデックススキャンが使用されていた場合 (sysscan->irel が非 NULL): index_endscan でインデックススキャンを終了し、index_close でインデックスリレーションを閉じる（AccessShareLock を解放）。
  • ヒープスキャンが使用されていた場合: table_endscan を呼び出す。

    • 3. **スナップショット解放**: sysscan->snapshot が存在する場合、UnregisterSnapshot を呼び出す。 4. **フラグのリセット**: CheckXidAlive が有効な場合、グローバル変数 bsysscan を false にリセットする。 5. **メモリ解放**: pfree(sysscan) を実行し、記述子自体をメモリから解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • contrib/sepgsql/ 配下の各ファイル（database.c, label.c, proc.c, relation.c, schema.c）
  • src/backend/access/common/toast_internals.c
  • **呼び出し先**:
  • ExecDropSingleTupleTableSlot
  • index_endscan
  • index_close
  • table_endscan
  • UnregisterSnapshot
  • pfree

関連データ構造

  • SysScanDesc: システムカタログスキャンを管理する構造体。

失敗モードと境界条件

  • sysscan が NULL の場合、あるいは既に解放済みのポインタを渡した場合の動作は未定義。

保守担当者向けメモ

  • bsysscan フラグのリセットは、トランザクション処理におけるシステムカタログスキャン状態の整合性を保つために重要である。詳細は xact.c のコメントを参照のこと。
  • インデックススキャンの終了時には AccessShareLock を明示的に指定してクローズしている。
