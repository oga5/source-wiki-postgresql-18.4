hash_any_extended

定義

  • 種別: function
  • 定義箇所: src/include/common/hashfn.h:36

目的

任意のバイト列に対して、指定されたシード値を用いたハッシュ値を計算します。ハッシュ衝突攻撃を回避するため、実行ごとに異なるシード値を使用してハッシュ値を変化させる用途に適しています。

シグネチャまたは形状

static inline Datum hash_any_extended(const unsigned char *k, int keylen, uint64 seed)

入力と出力

  • **入力**:
  • k: ハッシュ対象のバイト列へのポインタ。
  • keylen: バイト列の長さ。
  • seed: ハッシュ計算の初期値として使用する uint64 値。
  • **出力**:
  • Datum: 計算されたハッシュ値。

前提条件と不変条件

  • 本関数は、結果の32ビットすべてが均等にランダムであることを前提としています。
  • src/common/hashfn.c で定義されている hash_bytes_extended を内部で呼び出します。

アルゴリズムまたは振る舞い

1. 引数として受け取ったバイト列 k、長さ keylen、およびシード値 seed を hash_bytes_extended 関数に渡します。 2. hash_bytes_extended が計算した結果を UInt64GetDatum マクロで Datum 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し先**: hash_bytes_extended (定義は src/common/hashfn.c)
  • **呼び出し元**:
  • contrib/citext/citext.c
  • contrib/hstore/hstore_op.c
  • contrib/ltree/ltree_op.c
  • src/backend/access/hash/hashfunc.c
  • src/backend/lib/bloomfilter.c
  • src/backend/nodes/queryjumblefuncs.c

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • uint64: シード値に使用される符号なし64ビット整数。

失敗モードと境界条件

  • バイト列の長さ keylen が負の値である場合の挙動については、本ソース抜粋からは特定できません。

保守担当者向けメモ

  • 本関数はインライン関数として定義されています。
  • 衝突回避が必要なハッシュテーブルの実装において、標準的な手段として利用されます。
  • hash_bytes_extended の実装詳細については src/common/hashfn.c を参照してください。
