StrategySyncStart

定義

  • **種別**: function
  • **定義箇所**: src/backend/storage/buffer/freelist.c:393

目的

バックグラウンドライター（BgBufferSync）がバッファの同期処理を開始すべき位置を特定し、現在のバッファ割り当て統計情報を取得します。

シグネチャまたは形状

int StrategySyncStart(uint32 *complete_passes, uint32 *num_buf_alloc)

入力と出力

  • **入力**:
  • complete_passes: クロック掃引の完了回数を格納するポインタ（NULL可）。
  • num_buf_alloc: 前回の呼び出し以降のバッファ割り当て回数を格納するポインタ（NULL可）。
  • **出力**:
  • 戻り値: 次に同期対象とすべきバッファのインデックス（nextVictimBuffer % NBuffers）。

前提条件と不変条件

  • StrategyControl->buffer_strategy_lock を使用して共有状態を保護します。
  • numBufferAllocs は取得時にアトミックに 0 にリセットされます。

アルゴリズムまたは振る舞い

1. buffer_strategy_lock を取得します。 2. nextVictimBuffer の現在値を読み取り、バッファ配列内のインデックスを計算します。 3. complete_passes が指定されている場合、StrategyControl->completePasses に、nextVictimBuffer のラップアラウンド分を加算して返します。 4. num_buf_alloc が指定されている場合、numBufferAllocs の現在値をアトミックに読み取り、同時に 0 にリセットして返します。 5. ロックを解放し、計算したインデックスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/bufmgr.c:3667
  • **呼び出し先**:
  • SpinLockAcquire / SpinLockRelease
  • pg_atomic_read_u32
  • pg_atomic_exchange_u32

関連データ構造

  • BufferStrategyControl: 共有メモリ上のバッファ戦略管理構造体。
  • NBuffers: システム全体の共有バッファ数。

失敗モードと境界条件

  • complete_passes や num_buf_alloc が NULL の場合、それらの値の書き込みはスキップされます。
  • nextVictimBuffer が NBuffers を超える場合、ラップアラウンド分が complete_passes に適切に加算されます。

保守担当者向けメモ

  • 本関数は BgBufferSync が同期の開始点を決定するために使用されます。
  • numBufferAllocs のリセットはアトミック操作で行われるため、統計情報の取りこぼしを防ぐ設計となっています。
