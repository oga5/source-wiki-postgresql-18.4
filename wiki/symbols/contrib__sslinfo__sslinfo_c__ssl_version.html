ssl_version

定義

  • 種別: function
  • 定義箇所: contrib/sslinfo/sslinfo.c:56

目的

現在のPostgreSQLセッションで使用されているSSL/TLSプロトコルのバージョン文字列を取得し、SQL関数として返します。

シグネチャまたは形状

Datum ssl_version(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出しの引数コンテキスト）。
  • **出力**: Datum 型（SSLバージョンを示す text 型のデータ）。SSLが使用されていない場合やバージョンが取得できない場合は NULL を返します。

前提条件と不変条件

  • MyProcPort が有効であり、現在のセッション情報にアクセス可能であること。
  • SSL接続が確立されていること（MyProcPort->ssl_in_use が真であること）。

アルゴリズムまたは振る舞い

1. MyProcPort->ssl_in_use を確認し、SSLが使用されていない場合は PG_RETURN_NULL() を実行します。 2. be_tls_get_version(MyProcPort) を呼び出し、SSLバージョン文字列を取得します。 3. 取得した文字列が NULL の場合は PG_RETURN_NULL() を実行します。 4. 取得した文字列を cstring_to_text でPostgreSQLの text 型に変換し、PG_RETURN_TEXT_P を通じて返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます（PG_FUNCTION_INFO_V1(ssl_version) により登録）。
  • **呼び出し先**: be_tls_get_version（src/backend/libpq/be-secure-openssl.c 等で定義される低レイヤー関数）。

関連データ構造

  • Port（MyProcPort として参照されるセッション情報構造体）。

失敗モードと境界条件

  • SSLが有効でない場合、または be_tls_get_version が情報を返せない場合は NULL を返します。
  • OpenSSLの内部エラー等で be_tls_get_version が失敗した場合の挙動は、当該関数の実装に依存します。

保守担当者向けメモ

  • 本関数は MyProcPort を直接参照するため、セッション状態の整合性に依存します。
