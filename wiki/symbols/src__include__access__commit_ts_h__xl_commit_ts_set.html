xl_commit_ts_set

定義

  • 種別: struct
  • 定義箇所: src/include/access/commit_ts.h:49

目的

トランザクションのコミット時刻およびレプリケーション元情報をWAL（Write-Ahead Logging）に記録するためのデータ構造です。

シグネチャまたは形状

typedef struct xl_commit_ts_set
{
    TimestampTz timestamp;
    RepOriginId nodeid;
    TransactionId mainxid;
    /* subxact Xids follow */
} xl_commit_ts_set;

入力と出力

  • 入力: トランザクションのコミット時に、コミット時刻 (timestamp)、レプリケーション元ID (nodeid)、およびメイントランザクションID (mainxid) が設定されます。
  • 出力: WALレコードの一部として永続化されます。構造体直後にサブトランザクションのXIDが可変長データとして続く設計です。

前提条件と不変条件

  • track_commit_timestamp GUC変数が有効である必要があります。
  • 構造体内の mainxid 以降にサブトランザクションのXIDが連続して配置される前提でメモリレイアウトが構成されています。

アルゴリズムまたは振る舞い

トランザクション終了時に、この構造体を用いてコミット情報をWALに書き込みます。リカバリ時には、このWALレコードを読み取ることで、共有メモリ上のコミット時刻管理用SLRUキャッシュを再構築します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報には具体的な呼び出し元関数は記載されていません。
  • 呼び出し先: 提供されたソース情報には具体的な呼び出し先は記載されていません。

関連データ構造

  • SizeOfCommitTsSet: offsetof(xl_commit_ts_set, mainxid) を用いて計算される、構造体の固定部分のサイズ定義。

失敗モードと境界条件

  • track_commit_timestamp が無効な状態で本構造体を利用しようとした場合、システムはエラーを発生させます。
  • SLRUキャッシュの切り捨て（Truncate）処理により、古いトランザクションのデータは削除されるため、存在しないXIDに対するアクセスは境界条件となります。

保守担当者向けメモ

  • 本構造体は可変長データ（サブトランザクションXID）を伴うため、サイズ計算やメモリコピーの際は SizeOfCommitTsSet マクロを適切に使用してください。
  • 構造体の変更はWALの互換性に直結するため、慎重に行う必要があります。
