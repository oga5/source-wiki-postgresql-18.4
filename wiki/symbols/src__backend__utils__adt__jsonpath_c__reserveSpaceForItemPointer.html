reserveSpaceForItemPointer

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonpath.c:506

目的

jsonpath のバイナリ表現構築において、後からノード間のリンク（オフセット）を書き込むための領域をバッファ上に確保します。

シグネチャまたは形状

int32 reserveSpaceForItemPointer(StringInfo buf)

入力と出力

  • **入力**: buf (StringInfo) - バイナリデータを構築中のバッファ。
  • **出力**: int32 - 確保した領域のバッファ先頭からのオフセット値。

前提条件と不変条件

  • buf は有効な StringInfo 構造体である必要があります。
  • 確保される領域は sizeof(int32)（4バイト）です。

アルゴリズムまたは振る舞い

1. 現在のバッファ長 buf->len を pos として記録します。 2. 初期値 0 を持つ int32 型の変数 ptr を用意します。 3. appendBinaryStringInfo を使用し、ptr の内容（4バイト）をバッファの末尾に追加します。 4. 記録しておいた pos を返却します。これにより、呼び出し元は後続の処理でこの位置を特定し、適切なオフセット値で上書きすることが可能になります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonpath.c 内の以下の箇所から呼び出されています。
  • flattenJsonPathParseItem（複数箇所: 265, 307, 308, 336, 366行目）
  • **呼び出し先**: appendBinaryStringInfo（lib/stringinfo.h で定義）

関連データ構造

  • StringInfo: PostgreSQL の可変長文字列/バイナリバッファ管理構造体。

失敗モードと境界条件

  • appendBinaryStringInfo が内部的にメモリ不足（OOM）を検知した場合、PostgreSQL のエラーハンドリング機構によりアボートされます。

保守担当者向けメモ

  • 本関数は、jsonpath のバイナリ形式が「4バイトアライメントされた可変長アイテムの連結リスト」であるという特性に基づいています。
  • 返却されるオフセットは、バッファの先頭からの相対位置です。後続の処理でこの位置を更新する際は、バッファの整合性を損なわないよう注意してください。
