hk_depth_search

定義

  • 種別: function
  • 定義箇所: src/backend/lib/bipartite_match.c:145

目的

Hopcroft-Karp アルゴリズムにおいて、深さ優先探索（DFS）を用いて増大路（Augmenting Path）を特定し、二部グラフのマッチングを更新する。

シグネチャまたは形状

static bool hk_depth_search(BipartiteMatchState *state, int u)

入力と出力

  • **入力**:
  • state: マッチング状態を保持する BipartiteMatchState 構造体へのポインタ。
  • u: 現在探索中の左側ノードのインデックス。
  • **出力**:
  • bool: 増大路が見つかりマッチングが更新された場合は true、それ以外は false を返す。

前提条件と不変条件

  • state->distance 配列には、事前に hk_breadth_search によって計算された各ノードの距離情報が格納されている必要がある。
  • ノード数は SHRT_MAX 未満であること。
  • check_stack_depth() により、再帰呼び出し時のスタックオーバーフローが防止される。

アルゴリズムまたは振る舞い

1. ノード u が 0（ダミーノード）であれば、増大路の終端に到達したとみなし true を返す。 2. distance[u] が HK_INFINITY の場合、このノードからは増大路を構成できないため false を返す。 3. u に隣接する各ノード v を走査する。 4. distance[pair_vu[v]] が distance[u] + 1 と一致する場合、再帰的に hk_depth_search を呼び出す。 5. 再帰呼び出しが成功した場合、マッチングペア pair_vu[v] および pair_uv[u] を更新し true を返す。 6. 探索の結果、増大路が見つからなかった場合は distance[u] を HK_INFINITY に設定し、再探索を避ける（枝刈り）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/lib/bipartite_match.c 内のメインループ（BipartiteMatch 関数内）。
  • hk_depth_search 自身（再帰呼び出し）。
  • **呼び出し先**:
  • check_stack_depth()（スタック安全性の確認）。
  • hk_depth_search（再帰）。

関連データ構造

  • BipartiteMatchState: マッチング計算に必要な distance（距離）、pair_uv（左側ノードのマッチング先）、pair_vu（右側ノードのマッチング先）、adjacency（隣接リスト）を保持する。

失敗モードと境界条件

  • u が 0 の場合は即座に成功とみなす。
  • distance[u] が HK_INFINITY の場合は探索を打ち切る。
  • 隣接リストが空、または有効な増大路が存在しない場合は false を返す。

保守担当者向けメモ

  • 本関数は再帰的に呼び出されるため、グラフの深さに応じたスタック消費に注意すること。
  • distance 配列の更新は、探索効率を維持するための重要な枝刈り処理である。
