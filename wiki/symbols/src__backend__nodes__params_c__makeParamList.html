makeParamList

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/params.c:43

目的

クエリ実行時に使用されるパラメータ保持構造体 ParamListInfo をメモリ上に確保し、初期化を行うコンストラクタ関数。

シグネチャまたは形状

ParamListInfo makeParamList(int numParams)

入力と出力

  • **入力**: numParams (確保するパラメータの個数)
  • **出力**: 初期化済みの ParamListInfo 構造体へのポインタ

前提条件と不変条件

  • numParams に応じたメモリ領域が palloc によって確保される。
  • parserSetup フックにはデフォルトで paramlist_parser_setup が設定され、その引数 parserSetupArg には生成された構造体自身がセットされる。
  • paramFetch 等の動的フックは初期状態では NULL に設定される。

アルゴリズムまたは振る舞い

1. ParamListInfoData のヘッダーサイズと、指定された numParams 分の ParamExternData 配列を格納するために必要な合計サイズを計算する。 2. palloc を使用してメモリを割り当てる。 3. 構造体の各フィールドを初期化する。

  • paramFetch, paramFetchArg, paramCompile, paramCompileArg, paramValuesStr を NULL に設定。
  • parserSetup に paramlist_parser_setup を割り当て、parserSetupArg に自身を指すポインタを格納。
  • numParams に引数の値を設定。

    • 4. 確保した構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/prepare.c
  • src/backend/executor/functions.c
  • src/backend/executor/spi.c
  • src/backend/nodes/params.c (内部的な複製処理など)
  • src/backend/tcop/postgres.c
  • src/pl/plpgsql/src/pl_exec.c
  • **呼び出し先**:
  • palloc (メモリ確保)
  • paramlist_parser_setup (関数ポインタとして代入)

関連データ構造

  • ParamListInfo: パラメータリストを管理する主要構造体。
  • ParamExternData: 個々のパラメータ情報を保持する配列要素。

失敗モードと境界条件

  • palloc がメモリ不足により失敗した場合、PostgreSQL の標準的なエラー処理メカニズムによりアボートされる。

保守担当者向けメモ

  • 本関数は ParamListInfo のメモリレイアウトを決定する。構造体の定義を変更する場合は、サイズ計算ロジック (offsetof を含む箇所) を必ず更新すること。
  • parserSetup フックがデフォルトで設定されるため、呼び出し側で個別に設定しない限り、標準的なパラメータ解析ロジックが適用される。
