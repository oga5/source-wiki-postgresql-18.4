InitLocalBuffers

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/localbuf.c:718

目的

一時テーブル用バッファマネージャの初期化を行い、バッファ記述子、参照カウント配列、および検索用ハッシュテーブルをメモリ上に確保する。

シグネチャまたは形状

static void InitLocalBuffers(void)

入力と出力

  • 入力: なし（グローバル変数 num_temp_buffers を参照）
  • 出力: なし（副作用としてグローバルなバッファ管理構造を初期化）

前提条件と不変条件

  • 並列ワーカー（IsParallelWorker()）からの呼び出しは禁止されており、実行された場合はエラーとなる。
  • num_temp_buffers に基づいてメモリを確保する。
  • 確保したメモリ領域が不足した場合は FATAL エラーを発生させる。

アルゴリズムまたは振る舞い

1. **並列チェック**: 現在のプロセスが並列ワーカーであるかを確認し、該当する場合はエラーを投げる。 2. **メモリ確保**: LocalBufferDescriptors、LocalBufferBlockPointers、LocalRefCount の各配列を calloc で確保する。 3. **初期化**: 各バッファ記述子に対し、buf_id を負の値（-i - 2）で設定し、I/O 参照をクリアする。 4. **ハッシュテーブル作成**: LocalBufHash を hash_create で生成する。 5. **完了**: NLocBuffer にバッファ数を設定し、初期化完了とする。

呼び出し元と呼び出し先

  • 呼び出し元: ソースコード内の複数の箇所から呼び出される（localbuf.c 内の LocalBufferAlloc 等）。
  • 呼び出し先: IsParallelWorker, calloc, GetLocalBufferDescriptor, pgaio_wref_clear, hash_create, ereport, elog。

関連データ構造

  • LocalBufferDescriptors: バッファ記述子（BufferDesc）の配列。
  • LocalBufferBlockPointers: バッファブロックへのポインタ配列。
  • LocalRefCount: 各バッファの参照カウント配列。
  • LocalBufHash: バッファ検索用のハッシュテーブル。

失敗モードと境界条件

  • メモリ不足: calloc が失敗した場合、FATAL エラーで終了する。
  • ハッシュテーブル作成失敗: hash_create が失敗した場合、ERROR で終了する。
  • 並列実行: 並列ワーカーから呼び出された場合、ERRCODE_INVALID_TRANSACTION_STATE でエラーとなる。

保守担当者向けメモ

  • 本関数は、一時テーブルへの初回アクセス時に実行される遅延初期化の役割を果たす。
  • buf_id が負の値であることは、共有バッファ（0以上）と区別するための重要な不変条件である。
  • LocalBufferDescriptors などのグローバル変数は、本関数によって初めて有効な状態になる。
