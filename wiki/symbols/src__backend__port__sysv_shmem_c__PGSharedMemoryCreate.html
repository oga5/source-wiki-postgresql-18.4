PGSharedMemoryCreate

定義

  • 種別: function
  • 定義箇所: src/backend/port/sysv_shmem.c:699

目的

PostgreSQLの共有メモリセグメントを初期化および作成します。System V共有メモリを基盤とし、データディレクトリに関連付けられた一意なキーを用いてセグメントを確保・アタッチします。

シグネチャまたは形状

PGShmemHeader *PGSharedMemoryCreate(Size size, PGShmemHeader **shim)

入力と出力

  • **入力**:
  • size: 確保する共有メモリのサイズ（バイト単位）。
  • shim: 共有メモリヘッダーへのポインタを格納する出力用引数。
  • **出力**:
  • 成功時: 共有メモリ領域の先頭アドレス（PGShmemHeader 構造体）を返す。
  • 失敗時: ereport(FATAL, ...) によりプロセスを終了する。

前提条件と不変条件

  • DataDir（データディレクトリ）が有効であること。
  • size は PGShmemHeader のサイズを格納するのに十分な大きさであること。
  • shared_memory_type が SHMEM_TYPE_MMAP の場合、匿名共有メモリが使用され、System V領域は最小限のヘッダーのみとなる。

アルゴリズムまたは振る舞い

1. **初期化**: データディレクトリの inode/device 番号を取得し、共有メモリキーのシードとする。 2. **メモリ確保**:

  • SHMEM_TYPE_MMAP の場合: CreateAnonymousSegment で実領域を確保し、on_shmem_exit にデタッチ処理を登録。
  • それ以外: 指定サイズで System V 共有メモリを確保。

    • 3. **セグメント探索**: InternalIpcMemoryCreate を呼び出し、利用可能な IPC キーが見つかるまでループする。

  • 既存セグメントが存在する場合、PGSharedMemoryAttach で状態を確認。
  • SHMSTATE_UNATTACHED（使用済みだが解放可能）であれば、shmctl(IPC_RMID) で削除して再試行。
  • SHMSTATE_ATTACHED であれば、他プロセスが使用中と判断し FATAL エラーとする。

    • 4. **ヘッダー設定**: 確保した領域に PGShmemHeader（PID、マジックナンバー、ディレクトリID等）を書き込む。 5. **戻り値**: 匿名メモリ使用時はそのアドレスを、それ以外は System V 領域のアドレスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/ipci.c
  • **呼び出し先**: InternalIpcMemoryCreate, CreateAnonymousSegment, PGSharedMemoryAttach, shmget, shmctl, shmdt, dsm_cleanup_using_control_segment

関連データ構造

  • PGShmemHeader: 共有メモリの制御情報を保持する構造体。
  • IpcMemoryKey, IpcMemoryId: System V IPC の識別子。
  • IpcMemoryState: セグメントの状態（列挙型）。

失敗モードと境界条件

  • **Huge Pages**: プラットフォームが非対応、または shared_memory_type が MMAP 以外の場合にエラー。
  • **セグメント競合**: 既存の共有メモリが他プロセスによって使用中の場合、FATAL エラーで停止。
  • **リソース枯渇**: shmget が失敗し続ける場合、ループによりキーをインクリメントして探索を継続する。

保守担当者向けメモ

  • EXEC_BACKEND 環境下では、PGSharedMemoryReAttach による再アタッチ処理が別途必要になる点に注意。
  • InternalIpcMemoryCreate の失敗は必ずしも致命的ではなく、既存セグメントの再利用や削除のトリガーとして機能する。
