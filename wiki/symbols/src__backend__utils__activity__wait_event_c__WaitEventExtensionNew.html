WaitEventExtensionNew

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/wait_event.c:162

目的

拡張モジュールが独自の待機イベントを動的に登録するための公開APIを提供します。

シグネチャまたは形状

uint32 WaitEventExtensionNew(const char *wait_event_name);

入力と出力

  • **入力**: wait_event_name (const char *) - 登録する待機イベントの名前。
  • **出力**: uint32 - 割り当てられた一意な待機イベントID。

前提条件と不変条件

  • 共有メモリ上のハッシュテーブルに空きがあること（上限 WAIT_EVENT_CUSTOM_HASH_MAX_SIZE = 128）。
  • 内部的に WaitEventCustomLock による排他制御が行われるため、呼び出しはスレッドセーフ（プロセスセーフ）です。

アルゴリズムまたは振る舞い

1. 呼び出し元から受け取った wait_event_name を引数として WaitEventCustomNew を呼び出します。 2. その際、待機クラスとして PG_WAIT_EXTENSION を指定します。 3. WaitEventCustomNew 内で共有メモリ上のカウンタから一意なIDを取得し、名前とIDをハッシュテーブルに登録してそのIDを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • contrib/postgres_fdw/connection.c
  • src/test/modules/test_shm_mq/setup.c
  • src/test/modules/test_shm_mq/test.c
  • src/test/modules/worker_spi/worker_spi.c
  • **呼び出し先**:
  • WaitEventCustomNew (同ファイル内)

関連データ構造

  • WaitEventCustomHashByInfo / WaitEventCustomHashByName (共有メモリ上のハッシュテーブル)

失敗モードと境界条件

  • 登録数が上限（128）を超えた場合、WaitEventCustomNew の内部実装に依存したエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は PG_WAIT_EXTENSION クラス専用のラッパーです。他のクラスのイベントを登録する場合は WaitEventInjectionPointNew 等の適切なAPIを使用してください。
