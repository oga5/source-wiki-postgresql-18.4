lineptr

定義

  • 種別: struct
  • 定義箇所: src/include/fe_utils/mbprint.h:16

目的

文字列の断片（ポインタ）と、その断片が占める表示幅を保持し、マルチバイト文字列を端末の表示幅に合わせて整形・管理するために使用されます。

シグネチャまたは形状

struct lineptr
{
	unsigned char *ptr;
	int			width;
};

入力と出力

  • ptr: 文字列の断片を指すポインタ。
  • width: その断片が画面上で占める表示幅（カラム数）。

前提条件と不変条件

  • pg_wcsformat 関数において、文字列を分割した結果を格納する配列として利用されます。
  • 表示幅の計算は、対象となるエンコーディングに基づいた適切なロジックで行われることが前提です。

アルゴリズムまたは振る舞い

文字列を端末の表示幅に合わせて分割する際、各行の開始位置を ptr に、その行の表示幅を width に格納することで、後続の出力処理が効率的に行えるようにします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/fe_utils/mbprint.h 内の pg_wcsformat 関数が本構造体を利用します。
  • **注意**: ツールが検出した src/backend/libpq/hba.c 内の lineptr は、変数名が一致していますが、本構造体型とは異なる用途（文字列トークン解析用のポインタ操作）で使用されており、直接的な関連性は確認できません。

関連データ構造

  • pg_wcsformat: 本構造体の配列を生成・操作する関数。
  • pg_wcssize: 整形に必要なサイズを算出する関数。

失敗モードと境界条件

  • 文字列が空の場合や、マルチバイト文字の境界が不正な場合の挙動は、本構造体自体ではなく、これを利用する pg_wcsformat 等の関数側の実装に依存します。

保守担当者向けメモ

  • 本構造体は fe_utils モジュールにおける表示制御専用です。src/backend/libpq/hba.c 等で同名の変数が見られますが、混同しないよう注意してください。
  • 構造体の定義は非常にシンプルですが、マルチバイト文字の表示幅計算ロジックと密結合しているため、変更時は mbprint.c の実装と整合性を確認してください。
