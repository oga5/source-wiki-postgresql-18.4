printTableContent

定義

  • 種別: struct
  • 定義箇所: src/include/fe_utils/print.h:163

目的

クエリ結果の整形出力において、ヘッダー、セルデータ、フッター、および出力設定を保持し、テーブル構築プロセスを抽象化するためのメイン構造体です。

シグネチャまたは形状

typedef struct printTableContent
{
	const printTableOpt *opt;
	const char *title;
	int			ncolumns;
	int			nrows;
	const char **headers;
	const char **header;
	const char **cells;
	const char **cell;
	uint64		cellsadded;
	bool	   *cellmustfree;
	printTableFooter *footers;
	printTableFooter *footer;
	char	   *aligns;
	char	   *align;
} printTableContent;

入力と出力

  • **入力**: printTableInit を介して初期化され、printTableAddHeader や printTableAddCell を通じてデータが投入されます。
  • **出力**: printTable 関数が本構造体を参照し、設定されたフォーマットに従ってストリームへ整形出力を行います。

前提条件と不変条件

  • ncolumns および nrows は初期化時に指定される必要があります。
  • cellmustfree 配列は、対応するセル文字列が free() を必要とする場合に true に設定されます。
  • headers および cells は NULL 終端の配列である必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: printTableInit で構造体の各フィールドが準備されます。 2. **構築**: header や cell ポインタを更新しながら、動的にデータを追加します。 3. **出力処理**: printTable が opt（printTableOpt）を参照し、ページャの要否判定やフォーマット変換を実行します。 4. **解放**: printTableCleanup が cellmustfree を参照し、動的に確保されたメモリを適切に解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: psql の command.c、crosstabview.c、describe.c などから参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、fe_utils/print.c 内の IsPagerNeeded、print_aligned_vertical、footers_with_default、print_unaligned_text 等の関数によって読み取られます。

関連データ構造

  • printTableOpt: 出力設定（罫線、ページャ等）を保持。
  • printTableFooter: フッター情報を保持する構造体。
  • PGresult: libpq 由来のクエリ結果データ。

失敗モードと境界条件

  • メモリ不足: セルやヘッダーの追加時にメモリ確保が失敗する可能性があります。
  • 不正なポインタ: cellmustfree の管理が不適切な場合、二重解放やメモリリークが発生します。

保守担当者向けメモ

  • 本構造体に追加する文字列は、所有権を明確にするため cellmustfree フラグの管理を徹底してください。
  • printTableCleanup を呼び出さない場合、メモリリークが発生するため注意が必要です。
  • ページャ制御が必要な場合は、opt フィールドの設定と ClosePager の呼び出しを呼び出し側で保証してください。
