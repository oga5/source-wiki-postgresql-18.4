create_ctas_nodata

定義

  • 種別: function
  • 定義箇所: src/backend/commands/createas.c:154

目的

CREATE TABLE AS または CREATE MATERIALIZED VIEW コマンドにおいて、データを含めずにテーブル定義のみを作成する処理を実行します。

シグネチャまたは形状

static ObjectAddress create_ctas_nodata(List *tlist, IntoClause *into)

入力と出力

  • **入力**:
  • tlist: クエリのターゲットリスト（List *）。
  • into: INTO 句の情報を含む構造体（IntoClause *）。
  • **出力**:
  • 作成されたリレーションの ObjectAddress を返します。

前提条件と不変条件

  • into->colNames が指定されている場合、その数は tlist 内の非ジャンク（!resjunk）カラム数以下である必要があります。
  • カラムの型が照合順序（collation）をサポートしている場合、照合順序が解決可能である必要があります。

アルゴリズムまたは振る舞い

1. tlist を走査し、resjunk でない要素から ColumnDef リスト（attrList）を構築します。 2. into->colNames が提供されている場合、それを使用してカラム名を上書きします。 3. 各カラムの型、型修飾子、照合順序を抽出します。 4. 照合順序が未解決かつ型が照合可能である場合、エラーを報告します。 5. 構築した attrList を用いて create_ctas_internal を呼び出し、リレーション定義を完了させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/createas.c 内のメイン実行フロー（ExecCreateTableAs 等から参照される）。
  • **呼び出し先**: create_ctas_internal を呼び出します。

関連データ構造

  • IntoClause: INTO 句の構文解析結果を保持。
  • TargetEntry: クエリのターゲットリスト要素。
  • ColumnDef: テーブル定義を構成するカラム情報。
  • ObjectAddress: 作成されたオブジェクトの識別子。

失敗モードと境界条件

  • into->colNames の数がターゲットリストのカラム数より多い場合、ERRCODE_SYNTAX_ERROR で失敗します。
  • 照合順序が決定できず、かつ型が照合可能である場合、ERRCODE_INDETERMINATE_COLLATION で失敗します。

保守担当者向けメモ

  • 本関数はプランナやエグゼキュータを介さず、定義のみを生成するパスです。ダンプ/リストア時の依存関係解決において重要な役割を果たします。
  • DefineRelation ではなく create_ctas_internal を経由することで、TOAST テーブル作成やマテリアライズドビューのメタデータ保存を統合しています。
