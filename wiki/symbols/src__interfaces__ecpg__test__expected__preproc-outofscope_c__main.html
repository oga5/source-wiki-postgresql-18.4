main

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-outofscope.c:261

目的

ECPG（Embedded SQL for C）プリプロセッサの回帰テストとして、異なる関数スコープ間でのカーソル操作（DECLARE/OPEN/FETCH/CLOSE）の正当性を検証する。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（コマンドライン引数は受け取らない）
  • 出力: 正常終了時に 0 を返す。SQL操作でエラーが発生した場合は exit(1) により異常終了する。

前提条件と不変条件

  • データベース ecpg1_regression への接続が成功していること。
  • テスト用テーブル a1 が作成され、データが挿入されていること。
  • get_var1 によって確保されたメモリ領域が、ループ処理の各反復で memset により初期化されること。

アルゴリズムまたは振る舞い

1. **環境構築**: ECPGconnect でデータベースに接続し、set datestyle、create table、insert を実行してテストデータを準備する。 2. **スコープ外準備**: get_var1 を呼び出し、カーソル操作に必要な構造体ポインタを初期化する。 3. **カーソル操作**: open_cur1 でカーソルを開き、ループ内で get_record1 を呼び出してデータを順次取得する。 4. **データ出力**: 取得した各行のフィールド値とNULLインジケータを標準出力に表示する。 5. **終了処理**: close_cur1 でカーソルを閉じ、確保したメモリを free し、テーブル削除と切断処理を経て終了する。

呼び出し元と呼び出し先

  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGdo, ECPGtrans, get_var1, open_cur1, get_record1, close_cur1, ECPGdisconnect, printf, free, memset, strcpy
  • 呼び出し元: 不明（テスト実行時のエントリポイント）

関連データ構造

  • MYTYPE: データベース行データを保持する構造体。
  • MYNULLTYPE: 各フィールドのNULL状態を追跡するインジケータ構造体。
  • sqlca: SQL通信領域（エラーチェックに使用）。

失敗モードと境界条件

  • 各SQL操作の直後に sqlca.sqlcode < 0 をチェックしており、エラー発生時は即座に exit(1) する。
  • get_record1 実行時に ECPG_NOT_FOUND が返された場合、ループを終了する。

保守担当者向けメモ

  • 本関数はプリプロセッサによって生成されたコードである。ECPGdo マクロ内の引数は、SQL文がCの関数呼び出しに変換されたものである。
  • ECPGget_var を介した関数間での変数参照が正しく行われているかを確認するテストケースであるため、構造体ポインタの受け渡しロジックを変更する際は注意が必要である。
