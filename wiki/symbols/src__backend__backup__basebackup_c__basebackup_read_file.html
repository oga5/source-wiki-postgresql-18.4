basebackup_read_file

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup.c:2115

目的

ベースバックアップ処理において、指定されたファイル記述子からデータを安全に読み込み、読み込み待機状態の統計情報を記録する。

シグネチャまたは形状

static ssize_t basebackup_read_file(int fd, char *buf, size_t nbytes, off_t offset, const char *filename, bool partial_read_ok)

入力と出力

  • **入力**:
  • fd: 読み込み対象のファイル記述子。
  • buf: 読み込んだデータを格納するバッファ。
  • nbytes: 読み込むバイト数。
  • offset: 読み込みを開始するファイル内のオフセット。
  • filename: エラー報告時に使用するファイル名。
  • partial_read_ok: true の場合、要求したバイト数より少ない読み込みを許容する。
  • **出力**:
  • 読み込んだバイト数を ssize_t 型で返す。

前提条件と不変条件

  • pg_pread を使用するため、ファイル記述子はシーク可能な状態である必要がある。
  • partial_read_ok が false の場合、読み込み結果が 0 より大きく、かつ nbytes と一致しない場合はエラーとして処理される。

アルゴリズムまたは振る舞い

1. pgstat_report_wait_start(WAIT_EVENT_BASEBACKUP_READ) を呼び出し、待機イベントを開始する。 2. pg_pread を用いて、指定されたオフセットからデータを読み込む。 3. pgstat_report_wait_end() を呼び出し、待機イベントを終了する。 4. 読み込み結果が負（エラー）の場合、ereport でエラーを発生させる。 5. partial_read_ok が false で、かつ読み込みバイト数が要求と異なる場合、ereport でエラーを発生させる。 6. 読み込んだバイト数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c 内の複数箇所（主にバックアップデータのストリーミング処理ループ内）。
  • **呼び出し先**:
  • pgstat_report_wait_start / pgstat_report_wait_end
  • pg_pread
  • ereport

関連データ構造

  • 特になし。

失敗モードと境界条件

  • **ファイル読み込み失敗**: pg_pread が負の値を返した場合、errcode_for_file_access() を用いてエラーを報告する。
  • **不完全な読み込み**: partial_read_ok が false の際、要求したバイト数と実際の読み込みバイト数が一致しない場合にエラーとなる。

保守担当者向けメモ

  • 本関数は basebackup.c 内のファイル読み込み処理を抽象化し、統計情報の記録と一貫したエラーハンドリングを強制するために使用されている。
  • 修正を行う際は、pgstat の待機イベント定義が適切か確認すること。
