conv_compare

定義

  • 種別: function
  • 定義箇所: src/common/unicode_norm.c:52

目的

Unicodeコードポイントの値を比較し、ソートや検索のための順序付けを行う。

シグネチャまたは形状

static int conv_compare(const void *p1, const void *p2)

入力と出力

  • p1: 比較対象の uint32 型コードポイントへのポインタ。
  • p2: pg_unicode_decomposition 構造体へのポインタ。
  • 戻り値: p1 が指す値が p2 の codepoint より大きければ 1、小さければ -1、等しければ 0 を返す。

前提条件と不変条件

  • p1 は uint32 型の値を指している必要がある。
  • p2 は codepoint メンバを持つ pg_unicode_decomposition 型の構造体を指している必要がある。

アルゴリズムまたは振る舞い

1. p1 が指すメモリ領域から uint32 値を読み取る。 2. p2 が指す構造体から codepoint メンバを読み取る。 3. 両者を比較し、標準的な比較結果（1, 0, -1）を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/unicode_norm.c 内のバイナリサーチ処理（bsearch 等の比較関数として利用）。
  • 呼び出し先: なし。

関連データ構造

  • pg_unicode_decomposition: p2 が指す構造体として使用される。

失敗モードと境界条件

  • ポインタが NULL の場合や、指し示すメモリ領域が不正な場合は未定義の動作となる。

保守担当者向けメモ

  • 本関数は bsearch 等の標準ライブラリ関数から呼び出されることを想定した比較関数である。
  • pg_unicode_decomposition 構造体の定義変更時には、本関数のメンバアクセスが整合しているか確認すること。
