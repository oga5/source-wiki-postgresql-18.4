PruneStepResult

定義

  • 種別: struct
  • 定義箇所: src/backend/partitioning/partprune.c:128

目的

パーティションプルーニングの各ステップを実行した結果を保持するための構造体です。特定のプルーニングステップによって選択されたパーティションを特定し、特殊なパーティション（デフォルトやNULL用）をスキャンすべきかを伝達します。

シグネチャまたは形状

typedef struct PruneStepResult
{
    Bitmapset  *bound_offsets;
    bool        scan_default;
    bool        scan_null;
} PruneStepResult;

入力と出力

本構造体は、プルーニングステップ（perform_pruning_base_step や perform_pruning_combine_step など）の戻り値として生成されます。

前提条件と不変条件

  • bound_offsets は、テーブルの boundinfo 内におけるパーティションのオフセットを示します。
  • scan_default および scan_null は、それぞれデフォルトパーティションおよびNULL値用パーティションをスキャン対象に含めるかどうかのフラグです。

アルゴリズムまたは振る舞い

プルーニング処理において、各ステップの評価結果をこの構造体に格納します。複数のステップ結果は PruneStepResult ** 型の配列として管理され、最終的にどのパーティションをスキャンすべきかの判断材料となります。

呼び出し元と呼び出し先

  • **呼び出し元**: get_matching_hash_bounds, get_matching_list_bounds, get_matching_range_bounds, perform_pruning_base_step, perform_pruning_combine_step 等がこの構造体を生成します。
  • **呼び出し先**: 実行時にプルーニング情報を評価する関数群が、この構造体のフィールドを参照してスキャン対象を決定します。

関連データ構造

  • PartitionPruneContext: プルーニング実行時のコンテキスト情報。
  • Bitmapset: 選択されたパーティションのオフセットを保持するために使用されます。

失敗モードと境界条件

  • bound_offsets が NULL の場合、そのステップでは特定のパーティションが選択されなかったことを意味します。
  • メモリ割り当て失敗時は palloc0 によりエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は、プランニング時および実行時の両方のプルーニングロジックで共通して使用されます。
  • PruneStepResult **results のようにポインタの配列として扱われることが多いため、メモリ管理（palloc0 等）には注意してください。
