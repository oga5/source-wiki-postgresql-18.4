CreatePartitionPruneState

定義

  • **種別**: function
  • **定義箇所**: src/backend/executor/execPartition.c:1973

目的

クエリ実行時におけるパーティションプルーニングのための状態管理構造体（PartitionPruneState）を初期化し、実行計画に含まれるプルーニング情報を実行環境へ適合させる。

シグネチャまたは形状

static PartitionPruneState *CreatePartitionPruneState(
    EState *estate, 
    PartitionPruneInfo *pruneinfo,
    Bitmapset **all_leafpart_rtis
)

入力と出力

  • **入力**:
  • estate: 実行状態（EState）。
  • pruneinfo: プランナーが生成したパーティションプルーニング情報。
  • all_leafpart_rtis: リーフパーティションの範囲テーブルインデックス（RTI）を格納するビットマップセットへのポインタ。
  • **出力**:
  • 初期化済みの PartitionPruneState 構造体へのポインタ。

前提条件と不変条件

  • pruneinfo->prune_infos は空であってはならない（n_part_hierarchies > 0）。
  • estate->es_partition_directory が未設定の場合、本関数内で CreatePartitionDirectory を用いて初期化される。
  • パーティションのリレーションキャッシュエントリは、実行期間中オープンかつロックされた状態であることが前提となる。

アルゴリズムまたは振る舞い

1. ExprContext を作成し、式評価の準備を行う。 2. PartitionPruneState 構造体をメモリ確保し、pruneinfo から情報をコピーする。 3. パーティションプルーニング関数呼び出し用の専用メモリコンテキスト（prune_context）を作成し、メモリリークを防止する。 4. 各パーティション階層について、現在のリレーションキャッシュと pruneinfo の情報を比較・照合する。

  • プラン作成時と現在でパーティション構成が一致する場合、マップ情報を直接コピーする。
  • 構成が異なる場合（例: 並行処理によるデタッチ等）、relid_map を走査して不一致箇所を特定し、存在しないパーティションをプルーニング対象としてマークする。

    • 5. 最終的に、実行時に使用可能なプルーニング状態を構築して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execPartition.c 内の実行初期化処理から呼び出される（直接的な呼び出し元関数の詳細はソース抜粋からは特定不可）。
  • **呼び出し先**: CreateExprContext, CreatePartitionDirectory, palloc, AllocSetContextCreate, ExecGetRangeTableRelation, RelationGetPartitionKey, PartitionDirectoryLookup 等。

関連データ構造

  • PartitionPruneState: プルーニング実行時のメイン状態保持構造体。
  • PartitionPruningData: 各階層のプルーニング情報を保持。
  • PartitionedRelPruningData: 個別のパーティションリレーションに関するプルーニング情報。
  • PartitionPruneInfo: プランナーが提供する元データ。

失敗モードと境界条件

  • パーティション構成がプラン作成時と現在で不一致の場合、relid_map を走査して動的に対応する。
  • 存在しないパーティションやデタッチされたパーティションは、マップ上で適切に除外（-1 等）される。

保守担当者向けメモ

  • 本関数は InitExecPartitionPruneContexts 等の初期化処理と密接に関連している。
  • subplan_map や subpart_map の構築ロジックは、プラン作成時と実行時のパーティション構成の乖離を吸収するための重要な箇所である。
