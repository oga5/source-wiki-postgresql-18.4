RT_NODE_4

定義

  • 種別: struct
  • 定義箇所: src/include/lib/radixtree.h:518

目的

適応型基数木（ART）において、最大4つの子ノードを保持するための最小サイズのノード構造を提供します。

シグネチャまたは形状

typedef struct RT_NODE_4
{
    RT_NODE        base;
    uint8          chunks[RT_FANOUT_4_MAX];
    RT_PTR_ALLOC   children[FLEXIBLE_ARRAY_MEMBER];
} RT_NODE_4;

入力と出力

  • **入力**: 挿入操作時に chunk（キーの断片）と子ノードへのポインタを受け取ります。
  • **出力**: 検索操作時に chunks 配列内のキーと一致するインデックスを特定し、対応する children を返します。

前提条件と不変条件

  • base メンバとして RT_NODE を保持し、基数木のノードとしての共通属性を継承します。
  • children はフレキシブル配列メンバであり、ノードのサイズクラスに応じたメモリ確保が必要です。
  • chunks 配列は、子ノードを識別するためのキー断片を格納します。

アルゴリズムまたは振る舞い

  • 子ノード数が少ない場合にメモリ効率を最適化するために使用されます。
  • 子ノードの追加により容量を超過した場合、より大きなノード種別（RT_NODE_16 等）へ昇格（成長）させる必要があります。
  • 検索時は chunks を走査して対象のキー断片を特定し、そのインデックスに対応する children を参照します。

呼び出し元と呼び出し先

  • **呼び出し元**: 基数木の挿入・検索・削除ロジック（RT_SET, RT_FIND 等の内部実装）から参照されます。
  • **呼び出し先**: RT_NODE（基底構造体）および RT_PTR_ALLOC（ポインタ管理型）を利用します。

関連データ構造

  • RT_NODE: 基底ノード構造。
  • RT_NODE_16, RT_NODE_48, RT_NODE_256: 他のサイズクラスのノード構造。
  • RT_RADIX_TREE: 本ノードを保持するメインの基数木構造体。

失敗モードと境界条件

  • **メモリ不足**: children 配列の動的確保時に失敗する可能性があります。
  • **容量超過**: RT_FANOUT_4_MAX を超える子ノードの追加は、ノードの置換（成長）処理を伴わない場合、境界外アクセスを引き起こします。

保守担当者向けメモ

  • 本構造体はマクロテンプレートの一部として定義されており、RT_FANOUT_4 などの定数はコンパイル時に決定されます。
  • children 配列のサイズは sizeof(RT_NODE_4) + RT_FANOUT_4 * sizeof(RT_PTR_ALLOC) として計算されるため、メモリ確保時にはこの計算式を遵守してください。
