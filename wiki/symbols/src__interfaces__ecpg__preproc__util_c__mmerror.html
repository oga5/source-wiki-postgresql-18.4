mmerror

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/util.c:49

目的

ECPGプリプロセッサにおけるエラーおよび警告メッセージの報告を行います。

シグネチャまたは形状

void mmerror(int error_code, enum errortype type, const char *error, ...);

入力と出力

  • **入力**:
  • error_code: エラーの種類を識別する整数値。
  • type: エラーの深刻度（enum errortype）。
  • error: メッセージのフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**: なし（戻り値は void）。標準エラー出力へ整形されたメッセージを出力します。

前提条件と不変条件

  • 可変長引数リスト（va_list）が正しく初期化されている必要があります。
  • 内部で呼び出される vmmerror が標準エラー出力への書き込みを処理します。

アルゴリズムまたは振る舞い

1. va_start を使用して可変長引数リスト ap を初期化します。 2. 内部関数 vmmerror を呼び出し、引数 error_code、type、error、および ap を渡してメッセージの整形と出力を行います。 3. va_end を呼び出して引数リストの処理を終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/descriptor.c (複数箇所)
  • src/interfaces/ecpg/preproc/ecpg.c
  • src/interfaces/ecpg/preproc/parser.c
  • **呼び出し先**:
  • vmmerror (内部関数)

関連データ構造

  • enum errortype: エラーの深刻度を定義する列挙型。
  • va_list: 可変長引数を扱うための標準ライブラリ型。

失敗モードと境界条件

  • フォーマット文字列と引数の不整合が発生した場合、標準的な printf 系関数と同様の未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の出力ロジックは vmmerror に依存しています。
  • 致命的なエラーを扱う場合は、ファイルを削除して終了する mmfatal が別途用意されているため、用途に応じて使い分けてください。
