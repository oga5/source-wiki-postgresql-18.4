XLogRecPtr

定義

  • 種別: typedef（struct へのポインタまたは型定義）
  • 定義箇所: replication/walreceiver.h:365（src/backend/replication/walreceiver.c:109 にも構造体定義あり）

目的

PostgreSQL の WAL（Write Ahead Log）ストリームにおけるログ位置（LSN: Log Sequence Number）を表現するための型です。

シグネチャまたは形状

ソースコード上では、WAL ストリームの進捗管理を行う LogstreamResult 構造体内で以下のように使用されています。

static struct
{
    XLogRecPtr  Write;  /* スタンバイで書き込まれた最後のバイト + 1 */
    XLogRecPtr  Flush;  /* スタンバイでフラッシュされた最後のバイト + 1 */
} LogstreamResult;

入力と出力

  • **入力**: WAL のオフセット位置を示す 64 ビット整数値として扱われます。
  • **出力**: WAL ストリームの現在位置や、ディスクへの書き込み・フラッシュ完了位置を指し示します。

前提条件と不変条件

  • Write は、スタンバイサーバーにおいてディスクへ書き込まれた WAL データの末尾位置（+1）を保持します。
  • Flush は、ディスクへ fsync 等により永続化された WAL データの末尾位置（+1）を保持します。
  • 不変条件として Flush <= Write が成立する必要があります。

アルゴリズムまたは振る舞い

walreceiver プロセスにおいて、プライマリから受信した WAL データの進捗を追跡するために使用されます。XLogWalRcvWrite で Write が更新され、XLogWalRcvFlush で Flush が更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**: walreceiver プロセス内の書き込み・フラッシュ処理関数。
  • **呼び出し先**: 共有メモリを介してリカバリプロセスがこの値を読み取り、リカバリの進捗を判断します。
  • **その他**: contrib 以下のモジュール（amcheck, pg_walinspect 等）で LSN の参照や検証に広く利用されています。

関連データ構造

  • LogstreamResult: walreceiver 内で受信進捗を保持する構造体。
  • WalRcv: 共有メモリ上のレプリケーション状態管理構造体。

失敗モードと境界条件

  • WAL の最大値を超えた場合のオーバーフローや、無効な LSN 値（InvalidXLogRecPtr）の取り扱いについては、本ソース抜粋範囲外のため詳細な記述なし。

保守担当者向けメモ

  • 本シンボルは replication/walreceiver.h で定義されており、walreceiver.c はそれを参照しています。
  • 構造体 LogstreamResult は walreceiver.c 内の静的変数であり、プロセス内での進捗管理に使用されます。
