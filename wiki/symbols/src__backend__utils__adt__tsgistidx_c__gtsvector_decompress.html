gtsvector_decompress

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsgistidx.c:242

目的

GiST インデックスに格納された tsvector データを、他のサポート関数が処理可能な形式へ展開（detoast）します。

シグネチャまたは形状

Datum gtsvector_decompress(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体を受け取ります。
  • **出力**: 展開済みの GISTENTRY へのポインタを Datum として返します。

前提条件と不変条件

  • 入力された GISTENTRY の key フィールドは、TOAST 圧縮されている可能性があります。
  • 他の gtsvector サポート関数は TOAST された値を処理できないため、本関数による展開が必須です。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) で GISTENTRY を取得します。 2. PG_DETOAST_DATUM を使用して、インデックスキーを SignTSVector 形式に展開します。 3. 展開が必要だった場合（元のデータと展開後のデータが異なる場合）：

  • 新しい GISTENTRY をメモリ上に確保（palloc）します。
  • gistentryinit を呼び出し、展開済みのキーをセットして返却します。

    • 4. 展開が不要だった場合、元の entry をそのまま返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: GiST インデックスのアクセスメソッド（具体的な呼び出し元はソースコードの文脈からは特定できません）。
  • **呼び出し先**: PG_DETOAST_DATUM, palloc, gistentryinit。

関連データ構造

  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。
  • SignTSVector: tsvector のインデックス表現。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • 入力データが不正な形式である場合、PG_DETOAST_DATUM が失敗する可能性があります。

保守担当者向けメモ

  • 本関数は「不正確（inexact）」な GiST インデックスのライフサイクルの一部です。
  • palloc で確保されたメモリは、GiST インデックスのメモリコンテキスト管理に依存します。
