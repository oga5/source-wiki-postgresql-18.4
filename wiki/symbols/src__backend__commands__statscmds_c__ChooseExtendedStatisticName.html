ChooseExtendedStatisticName

定義

  • 種別: function
  • 定義箇所: src/backend/commands/statscmds.c:847

目的

拡張統計情報オブジェクトに対して、指定された名前の構成要素に基づき、名前空間内で一意となる名前を生成する。

シグネチャまたは形状

char *ChooseExtendedStatisticName(const char *name1, const char *name2, const char *label, Oid namespaceid)

入力と出力

  • **入力**:
  • name1, name2: 名前生成のベースとなる文字列（通常はテーブル名や列名）。
  • label: 統計情報のラベル。
  • namespaceid: 統計情報を作成する名前空間のOID。
  • **出力**:
  • 生成された一意な統計情報名（呼び出し元で pfree する必要がある）。

前提条件と不変条件

  • 指定された namespaceid において、pg_statistic_ext カタログ内に重複する名前が存在しないことを保証する。
  • 名前が衝突した場合は、label に連番を付与して再試行する。

アルゴリズムまたは振る舞い

1. makeObjectName を使用して、name1, name2, label から初期候補名を生成する。 2. GetSysCacheOid2 を呼び出し、STATEXTNAMENSP キャッシュを用いて当該名前空間内に同名の統計情報が存在するか確認する。 3. 存在しない場合（!OidIsValid）は、その名前を確定して返す。 4. 存在する場合は、label に連番（pass）を付与した文字列を生成し、衝突がなくなるまでループを繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/statscmds.c 内の統計情報作成処理（CREATE STATISTICS 等）。
  • **呼び出し先**:
  • makeObjectName（名前の構築）
  • GetSysCacheOid2（カタログ検索）
  • pfree（メモリ解放）

関連データ構造

  • pg_statistic_ext: 拡張統計情報を保持するシステムカタログ。
  • STATEXTNAMENSP: 統計情報名と名前空間による検索用システムキャッシュ。

失敗モードと境界条件

  • 名前空間内に無数の衝突が発生した場合、ループが継続する可能性がある。
  • メモリ不足時には makeObjectName 等でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は名前の自動生成を行うため、同時実行環境下で CREATE STATISTICS が並行して実行される場合、名前の衝突を避けるために適切なロックや CommandCounterIncrement の呼び出しが呼び出し元で必要となる場合がある。
