IndexAmTranslateStrategy

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/amapi.c:117

目的

インデックスのアクセスメソッド（AM）固有の戦略番号を、コアシステムが扱う汎用的な比較型（CompareType）へ変換します。

シグネチャまたは形状

CompareType IndexAmTranslateStrategy(StrategyNumber strategy, Oid amoid, Oid opfamily, bool missing_ok)

入力と出力

  • **入力**:
  • strategy: 変換対象の戦略番号。
  • amoid: 対象アクセスメソッドのOID。
  • opfamily: 演算子族のOID。
  • missing_ok: 変換失敗時にエラーを投げるか（false）、COMPARE_INVALID を返すか（true）のフラグ。
  • **出力**: 変換後の CompareType 値。

前提条件と不変条件

  • B-tree AM（BTREE_AM_OID）の場合、有効な戦略番号であれば変換をスキップしてそのまま返します。
  • それ以外のAMでは、GetIndexAmRoutineByAmId を介して取得した IndexAmRoutine 内の amtranslatestrategy 関数ポインタを使用します。

アルゴリズムまたは振る舞い

1. **最適化パス**: amoid が BTREE_AM_OID かつ strategy が有効な範囲内であれば、即座に strategy を CompareType にキャストして返します。 2. **ルーチン取得**: GetIndexAmRoutineByAmId を呼び出し、対象AMのルーチン構造体を取得します。 3. **変換実行**: ルーチン内の amtranslatestrategy 関数が存在すればそれを実行し、存在しなければ COMPARE_INVALID を設定します。 4. **エラーハンドリング**: missing_ok が false かつ結果が COMPARE_INVALID の場合、elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeMergejoin.c
  • src/backend/utils/adt/selfuncs.c
  • src/backend/utils/cache/lsyscache.c
  • **呼び出し先**:
  • GetIndexAmRoutineByAmId
  • amroutine->amtranslatestrategy（各AMの実装関数）

関連データ構造

  • IndexAmRoutine: AMの機能を提供する関数ポインタ群を保持する構造体。
  • CompareType: 比較操作の種類を示す列挙型。
  • StrategyNumber: インデックスの戦略番号。

失敗モードと境界条件

  • amoid に対応するAMが存在しない場合、GetIndexAmRoutineByAmId がエラーを発生させる可能性があります。
  • amtranslatestrategy が定義されていないAMに対して missing_ok=false で呼び出した場合、エラーが発生します。

保守担当者向けメモ

  • B-treeの最適化パスはカタログ検索を回避するための重要なショートカットです。新しいAMを追加する際は、amtranslatestrategy が適切に実装されているか確認してください。
  • 戻り値の COMPARE_INVALID は、変換不能であることを示す境界値として使用されます。
