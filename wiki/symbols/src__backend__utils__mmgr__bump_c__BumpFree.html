BumpFree

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/bump.c:616

目的

バンプアロケータ（BumpContext）において、個別のメモリチャンクに対する解放操作がサポートされていないことを明示し、誤った呼び出しに対してエラーを発生させるために存在します。

シグネチャまたは形状

void BumpFree(void *pointer)

入力と出力

  • **入力**: pointer (解放対象として渡されるメモリ領域のポインタ)
  • **出力**: なし（成功時は戻り値を返さず、常にエラーをスローします）

前提条件と不変条件

  • バンプアロケータは個別の pfree を許可しません。
  • 本関数が呼び出された場合、常に ERROR が発生します。

アルゴリズムまたは振る舞い

1. elog(ERROR, ...) を呼び出し、pfree がバンプアロケータでサポートされていない旨を通知して処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c: MCTX_BUMP_ID の free_p 関数ポインタとして登録されています。
  • **呼び出し先**:
  • elog: エラー報告のために呼び出されます。

関連データ構造

  • BumpContext: 本関数が管理対象とするメモリコンテキストの構造体。

失敗モードと境界条件

  • 本関数は呼び出された時点で常に失敗（ERROR）するように設計されています。

保守担当者向けメモ

  • この関数は、MemoryContext インターフェースの要件を満たすために実装されたプレースホルダーです。バンプアロケータの設計上、個別のメモリ解放は意図的に禁止されているため、この動作を変更しないでください。
