parse_params

定義

  • 種別: function
  • 定義箇所: contrib/xml2/xslt_proc.c:203

目的

SQL から渡された key=value,key=value 形式の文字列（text 型）を、libxslt が処理可能な const char ** 型の配列（NULL 終端）に変換します。

シグネチャまたは形状

static const char **parse_params(text *paramstr)

入力と出力

  • **入力**: text *paramstr（key=value のペアがカンマ区切りで並んだ文字列）
  • **出力**: const char **（key と value が交互に格納された NULL 終端の配列）

前提条件と不変条件

  • 入力文字列は text_to_cstring によってメモリ上にコピーされ、関数内で破壊的に変更されます。
  • 返される配列は palloc によって割り当てられるため、呼び出し元で適切に管理する必要があります。
  • max_params は常に偶数で維持されます。

アルゴリズムまたは振る舞い

1. 入力文字列を C 文字列に変換し、初期サイズ 20 の配列を確保します。 2. 文字列を =（キーと値の区切り）および ,（ペアの区切り）で走査します。 3. = が見つからないペアは無視されます。 4. 配列の容量が不足した場合は repalloc でサイズを倍増させます。 5. 最後に配列の末尾に NULL を格納して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/xml2/xslt_proc.c 内の xslt_process 関数（67行目）。
  • **呼び出し先**: PostgreSQL のメモリ管理関数 palloc, repalloc および文字列変換関数 text_to_cstring。標準ライブラリの strstr を使用します。

関連データ構造

  • text: PostgreSQL の可変長文字列型。

失敗モードと境界条件

  • = を含まない文字列が渡された場合、その要素は無視されます。
  • メモリ不足時には palloc/repalloc が PostgreSQL のエラーメカニズムを通じて例外を発生させます。

保守担当者向けメモ

  • 本関数は入力文字列を破壊的に書き換えるため、呼び出し元で元の text オブジェクトを保持していても、この関数に渡した後の文字列の内容は保証されません。
  • max_params の初期値は 20 ですが、必要に応じて動的に拡張される設計になっています。
