TidRangeQualFromRestrictInfoList

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/tidpath.c:397

目的

与えられた RestrictInfo リストから、TID範囲スキャン（TidRangePath）に使用可能な条件式を抽出する。

シグネチャまたは形状

static List *TidRangeQualFromRestrictInfoList(List *rlist, RelOptInfo *rel)

入力と出力

  • **入力**:
  • rlist: 評価対象となる RestrictInfo のリスト。
  • rel: 対象となるリレーションの RelOptInfo 構造体。
  • **出力**:
  • TID範囲スキャンに使用可能な RestrictInfo を含むリスト。条件に合致するものがない場合は NIL を返す。

前提条件と不変条件

  • 対象のリレーション（rel）が AMFLAG_HAS_TID_RANGE フラグを保持している必要がある。保持していない場合、即座に NIL を返す。

アルゴリズムまたは振る舞い

1. rel->amflags を確認し、AMFLAG_HAS_TID_RANGE が設定されていない場合は NIL を返して終了する。 2. rlist 内の各 RestrictInfo 要素を走査する。 3. IsTidRangeClause を呼び出し、各要素がTID範囲条件として有効かを判定する。 4. 有効と判定された RestrictInfo を新しいリストに追加し、最終的にそのリストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/tidpath.c:543（create_tidscan_paths 内）
  • **呼び出し先**:
  • IsTidRangeClause（各 RestrictInfo の判定に使用）

関連データ構造

  • RestrictInfo: クエリの制約条件を保持する構造体。
  • RelOptInfo: リレーションの最適化情報を保持する構造体。

失敗モードと境界条件

  • rel が AMFLAG_HAS_TID_RANGE をサポートしていない場合、条件の有無に関わらず NIL を返す。
  • 入力リスト rlist が空の場合、結果として NIL を返す。

保守担当者向けメモ

  • 本関数は TidRangePath の生成に特化している。TIDの等価比較（=) を扱う TidQualFromRestrictInfoList とは別個のロジックである点に注意すること。
