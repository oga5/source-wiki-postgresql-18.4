am_parallel_apply_worker

定義

  • 種別: static inline function
  • 定義箇所: src/include/replication/worker_internal.h:347

目的

現在のプロセスが論理レプリケーションにおける「並列適用ワーカー（Parallel Apply Worker）」として動作しているかどうかを判定する。

シグネチャまたは形状

bool am_parallel_apply_worker(void)

入力と出力

  • 入力: なし（グローバル変数 MyLogicalRepWorker を参照）
  • 出力: 現在のプロセスが並列適用ワーカーであれば true、そうでなければ false を返す。

前提条件と不変条件

  • MyLogicalRepWorker が指すワーカー情報が有効（in_use が真）である必要がある。この条件は関数内の Assert によって強制される。

アルゴリズムまたは振る舞い

1. MyLogicalRepWorker->in_use が真であることを検証する。 2. isParallelApplyWorker(MyLogicalRepWorker) を呼び出し、ワーカーの型が並列適用ワーカーであるかを判定してその結果を返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/replication/logical/applyparallelworker.c
  • src/backend/replication/logical/worker.c
  • 呼び出し先:
  • isParallelApplyWorker（マクロまたはインライン関数と推測されるが、本ソース excerpt に定義は含まれない）

関連データ構造

  • LogicalRepWorker: ワーカーの状態を保持する構造体。MyLogicalRepWorker はこの構造体を指すグローバル変数。

失敗モードと境界条件

  • MyLogicalRepWorker が初期化されていない、または in_use が偽の状態（ワーカーが割り当てられていない状態）で呼び出された場合、Assert によりプロセスが異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル経由で各バックエンドプロセスに組み込まれる。
  • 判定ロジックは MyLogicalRepWorker の状態に完全に依存しているため、ワーカーのライフサイクル管理に変更を加える際は注意が必要である。
