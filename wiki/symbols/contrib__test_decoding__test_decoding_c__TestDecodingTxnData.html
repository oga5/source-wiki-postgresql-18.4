TestDecodingTxnData

定義

  • **種別**: struct
  • **定義箇所**: contrib/test_decoding/test_decoding.c:48

目的

論理デコーディングの出力プラグインにおいて、個々のトランザクション実行中の状態（変更の発生有無）を追跡・保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	bool		xact_wrote_changes;
	bool		stream_wrote_changes;
} TestDecodingTxnData;

入力と出力

本構造体はデータ保持用であり、直接的な関数入出力は持ちません。各フィールドは論理デコーディングのコールバック関数内で、トランザクションの進行状況に応じて更新されます。

前提条件と不変条件

  • MemoryContextAllocZero を使用してゼロ初期化された状態で割り当てられることが想定されています。
  • トランザクションのライフサイクル（開始からコミット/ロールバックまで）を通じて、txn->output_plugin_private に格納されて管理されます。

アルゴリズムまたは振る舞い

  • **状態管理**:
  • xact_wrote_changes: トランザクション内で変更イベントが書き込まれたかどうかを記録します。
  • stream_wrote_changes: ストリーミング処理中に変更イベントが書き込まれたかどうかを記録します。
  • これらは、空のトランザクションの出力を制御する際や、ストリーミング時の状態判断に利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_decode_begin_txn や pg_decode_begin_prepare_txn 等のコールバック関数内で、MemoryContextAllocZero を介してインスタンス化されます。
  • **呼び出し先**: txn->output_plugin_private を通じて、トランザクションに関連する各デコーディング処理関数から参照・更新されます。

関連データ構造

  • TestDecodingData: プラグイン全体の設定を保持する構造体。
  • LogicalDecodingContext: PostgreSQLコアが提供する論理デコーディングのコンテキスト。

失敗モードと境界条件

  • メモリ割り当て失敗時、MemoryContextAllocZero がエラーを発生させる可能性があります。
  • トランザクションがアボートした場合、本構造体で保持していた状態は pg_decode_shutdown 等の終了処理に伴い破棄されます。

保守担当者向けメモ

  • 本構造体はトランザクションごとの一時的な状態保持を目的としています。ストリーミングモードでの動作を含め、トランザクションのコミットまで状態が維持される必要がある点に留意してください。
