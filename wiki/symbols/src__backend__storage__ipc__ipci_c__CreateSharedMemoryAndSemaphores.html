CreateSharedMemoryAndSemaphores

定義

  • **種別**: function
  • **定義箇所**: src/backend/storage/ipc/ipci.c:199

目的

PostgreSQL の起動プロセスにおいて、共有メモリ領域の確保、セマフォの予約、および各サブシステムの共有メモリデータ構造の初期化を統括する。

シグネチャまたは形状

void CreateSharedMemoryAndSemaphores(void)

入力と出力

  • **入力**: なし（グローバルな設定および各サブシステムの ShmemSize 関数を参照）
  • **出力**: なし（副作用として共有メモリセグメントの作成と初期化を行う）

前提条件と不変条件

  • IsUnderPostmaster が false であること（Postmaster 起動時のみ実行される）。
  • huge_pages_status が "unknown" ではないこと。

アルゴリズムまたは振る舞い

1. CalculateShmemSize を呼び出し、必要な共有メモリ総量とセマフォ数を算出する。 2. PGSharedMemoryCreate により OS から共有メモリセグメントを確保する。 3. InitShmemAccess で共有メモリへのアクセスを初期化する。 4. PGReserveSemaphores で必要な数のセマフォを予約する。 5. InitShmemAllocation で共有メモリ内のメモリ管理機構を有効化する。 6. CreateOrAttachShmemStructs を呼び出し、各サブシステムのデータ構造を初期化する。 7. dsm_postmaster_startup で動的共有メモリ（DSM）機能を初期化する。 8. shmem_startup_hook が設定されていれば実行し、拡張モジュールによる共有メモリ割り当てを許可する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/postmaster.c
  • src/backend/bootstrap/bootstrap.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • CalculateShmemSize, PGSharedMemoryCreate, InitShmemAccess, PGReserveSemaphores, InitShmemAllocation, CreateOrAttachShmemStructs, dsm_postmaster_startup, shmem_startup_hook

関連データ構造

  • PGShmemHeader: 共有メモリセグメントのヘッダー情報。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、PGSharedMemoryCreate 内でエラーが発生しプロセスが終了する。
  • huge_pages_status が "unknown" の場合、アサーション違反により停止する。

保守担当者向けメモ

  • 本関数は PostgreSQL の起動シーケンスの根幹です。新しいサブシステムで共有メモリが必要な場合は、CalculateShmemSize および CreateOrAttachShmemStructs への登録が必須です。
  • EXEC_BACKEND 環境（Windows 等）では、子プロセス側での再接続処理との整合性に注意してください。
