WordBoundaryState

定義

  • 種別: struct
  • 定義箇所: src/common/unicode/case_test.c:39

目的

タイトルケース変換（initcap）処理において、文字列内の単語境界を判定するための状態を保持する構造体です。

シグネチャまたは形状

struct WordBoundaryState
{
	const char *str;        /* 対象文字列 */
	size_t		len;        /* 文字列長 */
	size_t		offset;     /* 現在の走査位置 */
	bool		posix;      /* POSIX 準拠モードフラグ */
	bool		init;       /* 初期化済みフラグ */
	bool		prev_alnum; /* 直前の文字が英数字かどうかのフラグ */
};

入力と出力

本構造体は関数間で状態を共有するために使用されます。入力として文字列ポインタと長さを保持し、走査の進捗（offset）や直前の文字の状態（prev_alnum）を出力・更新することで、単語の区切りを追跡します。

前提条件と不変条件

  • init フラグが false の状態から開始し、適切な初期化関数によって true に設定されることが想定されます。
  • str は len で指定された範囲内で有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

単語境界の判定ロジックにおいて、現在の文字が単語の開始位置にあるかどうかを判断するために使用されます。prev_alnum を用いて、直前の文字が英数字であるか否かを記録し、現在の文字との組み合わせで単語の区切りを特定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/unicode/case_test.c 内のテスト関数群。
  • src/backend/utils/adt/pg_locale_builtin.c 内のロケール関連処理。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。この構造体を利用する関数（initcap_wbnext 等）がロジックを担います。

関連データ構造

  • initcap_wbnext: 本構造体を引数として受け取り、単語境界を特定するイテレータ関数。

失敗モードと境界条件

  • len が実際の文字列長と一致しない場合、範囲外アクセスが発生する可能性があります。
  • str が NULL の状態で走査を開始した場合の動作は定義されていません。
