ShutdownAuxiliaryProcess

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/auxprocess.c:97

目的

補助プロセス（Auxiliary Process）の終了時に、保持しているリソースを安全に解放し、クリーンアップを行う。

シグネチャまたは形状

static void ShutdownAuxiliaryProcess(int code, Datum arg)

入力と出力

  • code: 終了コード（before_shmem_exit の仕様に準拠）。
  • arg: 補助的な引数（現在は使用されていない）。
  • 戻り値: なし。

前提条件と不変条件

  • before_shmem_exit を介して登録されるコールバック関数であること。
  • 補助プロセスが終了処理のフェーズに入っていること。

アルゴリズムまたは振る舞い

1. LWLockReleaseAll() を呼び出し、プロセスが保持しているすべての軽量ロック（LWLock）を強制的に解放する。 2. ConditionVariableCancelSleep() を呼び出し、待機中の条件変数があればキャンセルする。 3. pgstat_report_wait_end() を呼び出し、統計情報システムに対して待機状態の終了を報告する。

呼び出し元と呼び出し先

  • 呼び出し元: before_shmem_exit（共有メモリ終了処理の登録メカニズムにより呼び出される）。
  • 呼び出し先: LWLockReleaseAll, ConditionVariableCancelSleep, pgstat_report_wait_end。

関連データ構造

  • LWLock: 補助プロセスが保持するロック状態。
  • ConditionVariable: 補助プロセスが待機に使用する同期プリミティブ。

失敗モードと境界条件

  • 共有メモリへのアクセスが既に無効化されている状況下での実行は想定されていない。
  • ロックの解放漏れはシステム全体のデッドロックや停止を招くため、本関数による確実なクリーンアップが必須である。

保守担当者向けメモ

  • 本関数はトランザクションを扱わない補助プロセス専用のクリーンアップルーチンである。
  • 新たなリソースを補助プロセスが保持するようになった場合は、本関数内で適切に解放処理を追加すること。
