GinPageDeletePostingItem

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/gindatapage.c:416

目的

GINポスティングツリーの内部ノード（非リーフページ）から、指定されたオフセット位置にある PostingItem を削除し、ページ内のアイテム配列を詰め直します。

シグネチャまたは形状

void GinPageDeletePostingItem(Page page, OffsetNumber offset)

入力と出力

  • **入力**:
  • page: 操作対象のGINデータページ（Page 型）。
  • offset: 削除対象のアイテム位置（OffsetNumber 型）。
  • **出力**: なし（副作用としてページ内容を直接変更）。

前提条件と不変条件

  • 対象ページはリーフページであってはならない（!GinPageIsLeaf(page)）。
  • offset は有効な範囲内（FirstOffsetNumber 以上、かつページ内の最大オフセット maxoff 以下）であること。

アルゴリズムまたは振る舞い

1. ページ内の現在の最大オフセット maxoff を取得します。 2. 削除対象の offset が maxoff でない場合、memmove を使用して、offset + 1 以降の全 PostingItem を前方へシフトし、空いた領域を埋めます。 3. maxoff をデクリメントし、ページヘッダーの maxoff を更新します。 4. GinDataPageSetDataSize を呼び出し、削除後のアイテム数に基づいてページ内のデータサイズを再設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginvacuum.c: バキューム処理中の親ページ更新。
  • src/backend/access/gin/ginxlog.c: WALリプレイ時のページ復元処理。
  • **呼び出し先**:
  • GinPageGetOpaque: ページヘッダーの不透明領域取得。
  • GinPageIsLeaf: ページ種別の判定。
  • GinDataPageGetPostingItem: アイテム配列へのポインタ取得。
  • GinDataPageSetDataSize: ページ内データサイズの更新。

関連データ構造

  • PostingItem: ポスティングツリーの内部ノードに格納される、子ページへのポインタと最小TIDを含む構造体。

失敗モードと境界条件

  • Assert により、リーフページに対して呼び出された場合や、無効なオフセットが指定された場合は異常終了します。
  • ページ内にアイテムが1つしかない状態で削除を行う場合、memmove はスキップされ、maxoff が0に更新されます。

保守担当者向けメモ

  • 本関数は内部ノード専用です。リーフページのアイテム削除には ginVacuumPostingTreeLeaf 等の専用ロジックを使用してください。
  • memmove を使用しているため、アイテムの移動コストは削除位置以降のアイテム数に比例します。
