parse_test_flags

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_regex/test_regex.c:248

目的

Tcl の正規表現テストスイートと互換性のある文字列フラグを解析し、PostgreSQL の正規表現エンジンが使用する内部フラグ（cflags, eflags）およびテスト用メタ情報へ変換する。

シグネチャまたは形状

static void parse_test_flags(test_re_flags *flags, text *opts)

入力と出力

  • **入力**:
  • flags: 解析結果を格納する test_re_flags 構造体へのポインタ。
  • opts: テストオプションを含む text 型の文字列。
  • **出力**: なし（flags 引数を介して副作用として値を書き込む）。

前提条件と不変条件

  • flags は呼び出し元で確保された有効なメモリ領域であること。
  • opts が NULL の場合、デフォルト値（REG_ADVANCED 等）が適用される。
  • opts に含まれる各文字は、定義されたスイッチケースのいずれかに合致する必要がある。

アルゴリズムまたは振る舞い

1. flags 構造体のメンバをデフォルト値（glob=false, indices=false, partial=false）で初期化する。 2. opts 文字列を先頭から順に走査し、各文字を switch 文で評価する。 3. 文字に応じて以下の処理を行う：

  • g, i, n, x 等の正規表現オプションを cflags に反映。
  • a, b, c, e 等の Tcl 互換フラグを cflags または eflags に反映。
  • A〜U の文字を info ビットマスクに累積。
  • c フラグが指定された場合、REG_EXPECT の設定に加え、partial および indices を強制的に有効化する。

    • 4. 認識できない文字が入力された場合、ereport(ERROR, ...) を呼び出し処理を中断する。 5. 最終的な cflags, eflags, info を flags 構造体に格納して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_regex/test_regex.c 内の test_regex 関数（SRF_IS_FIRSTCALL ブロック内）。
  • **呼び出し先**: ereport（エラー発生時）、pg_mblen_range（エラーメッセージ生成時）。

関連データ構造

  • test_re_flags: 解析結果を保持する構造体。

失敗モードと境界条件

  • **エラー**: 未定義のフラグ文字が opts に含まれる場合、ERRCODE_INVALID_PARAMETER_VALUE を伴うエラーを発生させる。
  • **境界**: opts が空文字列の場合、デフォルト値のみが設定される。

保守担当者向けメモ

  • 本関数内のフラグ定義は Tcl の正規表現テストスイートと互換性を持つ必要がある。変更時は src/backend/regex/ のフラグ定義との整合性に注意すること。
  • c フラグの処理において、partial と indices を強制的に有効化しているのは、テストハーネス側で期待されるデータを出力させるための設計上の要件である。
