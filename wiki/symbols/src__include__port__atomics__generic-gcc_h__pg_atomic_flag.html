pg_atomic_flag

定義

  • 種別: struct
  • 定義箇所: src/include/port/atomics/generic-gcc.h:69

目的

PostgreSQLにおける原子的なフラグ操作（Test-and-Set）のためのデータ構造を提供します。

シグネチャまたは形状

typedef struct pg_atomic_flag
{
    volatile [int|char] value;
} pg_atomic_flag;

入力と出力

  • 入力: なし（データ構造定義）
  • 出力: なし

前提条件と不変条件

  • HAVE_GCC__SYNC_INT32_TAS マクロの定義状況に応じて、内部の value 型が int または char に決定されます。
  • volatile 修飾子により、コンパイラによる最適化での読み飛ばしを防止します。
  • x86/x86_64環境では arch-x86.h で定義されるため、本定義は使用されません。

アルゴリズムまたは振る舞い

本構造体は、pg_atomic_test_set_flag_impl 等の原子操作関数において、メモリ上のフラグ状態を保持するために使用されます。int幅のTAS（Test-and-Set）が利用可能なプラットフォームでは効率性を優先して int が選択されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/include/port/atomics.h（各原子操作関数のインターフェース）
  • src/include/port/atomics/generic-gcc.h（各実装関数: pg_atomic_test_set_flag_impl 等）
  • src/backend/postmaster/autovacuum.c（wi_dobalance 変数として利用）
  • 呼び出し先: なし（データ構造のため）

関連データ構造

  • pg_atomic_uint32
  • pg_atomic_uint64

失敗モードと境界条件

  • 64bit環境における原子操作の無効化設定（PG_DISABLE_64_BIT_ATOMICS）の影響は受けませんが、プラットフォーム固有のTAS実装の成否に依存します。

保守担当者向けメモ

  • 本構造体は atomics.h を経由して使用されることを前提としています。
  • x86以外のプラットフォームにおける移植性を考慮し、int と char の切り替えロジックが組み込まれています。
