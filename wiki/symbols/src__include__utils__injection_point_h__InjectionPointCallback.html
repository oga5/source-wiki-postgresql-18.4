InjectionPointCallback

定義

  • 種別: typedef
  • 定義箇所: src/include/utils/injection_point.h:32

目的

インジェクションポイント（実行パスへの動的な割り込み）において実行されるコールバック関数の型を定義します。テストやデバッグ目的で、特定のコード実行箇所に外部ライブラリの関数を挿入するために使用されます。

シグネチャまたは形状

typedef void (*InjectionPointCallback) (const char *name,
                                        const void *private_data,
                                        void *arg);

入力と出力

  • name (const char *): インジェクションポイントの識別名。
  • private_data (const void *): インジェクションポイントに関連付けられた不変のデータ。
  • arg (void *): 呼び出し元から渡される任意の引数。
  • 戻り値: なし (void)。

前提条件と不変条件

  • 本機能は、ビルド時に --enable-injection-points が指定されている場合にのみ有効です。
  • 共有メモリ上の状態管理には、ロックフリーな読み取りを可能にするためのジェネレーションカウンタプロトコルが使用されます。

アルゴリズムまたは振る舞い

1. INJECTION_POINT または INJECTION_POINT_CACHED マクロが実行されると、本型で定義されたコールバックが呼び出されます。 2. INJECTION_POINT_CACHED を使用する場合、一度ロードされたコールバックはバックエンドローカルなハッシュテーブルにキャッシュされ、以降の呼び出しコストが削減されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/misc/injection_point.c 内のインジェクションポイント実行ロジック（callback 変数として参照）。
  • 呼び出し先: ユーザーが InjectionPointAttach を介して動的に紐付けた外部ライブラリ内の関数。

関連データ構造

  • InjectionPointsCtl: 共有メモリ上のインジェクションポイント状態管理構造体。
  • ActiveInjectionPoints: 共有メモリ上で管理されるアクティブなインジェクションポイントのリスト。

失敗モードと境界条件

  • ビルドオプションが有効でない場合、マクロは NOP（何もしない）として展開されます。
  • 共有メモリ上のエントリを操作する際は、ジェネレーションカウンタの更新順序に注意が必要です。

保守担当者向けメモ

  • 本機能は本番環境のパフォーマンスや安全性に影響を与えないよう設計されています。
  • 実装の詳細については src/backend/utils/misc/injection_point.c を参照してください。
