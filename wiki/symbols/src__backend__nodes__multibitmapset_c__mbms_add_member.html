mbms_add_member

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/multibitmapset.c:43

目的

List 内の指定されたインデックス（listidx）にある Bitmapset に対して、特定のビット（bitidx）を立てる。必要に応じてリストを動的に拡張する。

シグネチャまたは形状

List *mbms_add_member(List *a, int listidx, int bitidx)

入力と出力

  • **入力**:
  • a: 操作対象の List（Bitmapset を要素として保持）。
  • listidx: 対象となるリストのインデックス。
  • bitidx: Bitmapset 内で立てるビットのインデックス。
  • **出力**:
  • 更新された List ポインタを返す。

前提条件と不変条件

  • listidx および bitidx は負の値であってはならない（負の場合はエラーを投げる）。
  • リスト a は Bitmapset を要素として保持している必要がある。

アルゴリズムまたは振る舞い

1. 引数のインデックスが負でないことを検証する。 2. listidx が現在のリスト長以上である場合、lappend(a, NULL) を繰り返してリストを必要な長さまで拡張する。 3. list_nth_cell を使用して対象のリストセルを取得する。 4. lfirst_node(Bitmapset, lc) で対象の Bitmapset を取り出し、bms_add_member を適用する。 5. 更新された Bitmapset をリストセルに格納し、リストの先頭ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/clauses.c 内の複数箇所。
  • **呼び出し先**:
  • list_length, lappend, list_nth_cell, lfirst_node, bms_add_member, lfirst, elog。

関連データ構造

  • List: Bitmapset を格納するコンテナ。
  • Bitmapset: 個別のビット集合を管理する構造体。

失敗モードと境界条件

  • listidx < 0 または bitidx < 0: ERROR を発生させ処理を中断する。
  • メモリ不足: lappend 内部でメモリ不足が発生した場合、PostgreSQL の標準的なメモリ管理に従いエラーとなる。

保守担当者向けメモ

  • 本関数はインプレース更新を行うため、呼び出し側は必ず戻り値でリストポインタを更新すること。
  • リストの拡張時に NULL を追加するため、リスト内の要素にアクセスする際は lfirst_node 等で型安全性を確保する必要がある。
