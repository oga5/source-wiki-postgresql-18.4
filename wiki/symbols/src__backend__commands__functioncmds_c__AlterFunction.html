AlterFunction

定義

  • 種別: function
  • 定義箇所: src/backend/commands/functioncmds.c:1360

目的

SQLの ALTER FUNCTION 文を処理し、システムカタログ pg_proc 上の関数またはプロシージャの属性を変更する。

シグネチャまたは形状

ObjectAddress AlterFunction(ParseState *pstate, AlterFunctionStmt *stmt)

入力と出力

  • **入力**:
  • pstate: パーサー状態。
  • stmt: 変更対象の関数名、引数、および適用するアクション（DefElem リスト）を含む AlterFunctionStmt 構造体。
  • **出力**: 変更された関数の ObjectAddress を返す。

前提条件と不変条件

  • 対象の関数は pg_proc に存在する必要がある。
  • 呼び出し元は対象関数の所有者である必要がある。
  • 集約関数（PROKIND_AGGREGATE）の変更は許可されない。
  • leakproof 属性の変更はスーパーユーザーのみに許可される。

アルゴリズムまたは振る舞い

1. pg_proc を RowExclusiveLock でオープンし、対象関数を LookupFuncWithArgs で特定する。 2. SearchSysCacheCopy1 で対象関数のタプルを取得し、所有権限を検証する。 3. compute_common_attribute を用いて stmt->actions を解析し、各属性（volatility, cost, rows, parallel等）の更新値を抽出する。 4. 抽出された属性値を pg_proc のタプルに反映させる。

  • support 属性の場合は、依存関係（pg_depend）の追加または置換を行う。
  • set_items（設定パラメータ）がある場合は、update_proconfig_value を経由して proconfig 配列を更新する。

    • 5. CatalogTupleUpdate でシステムカタログを更新し、InvokeObjectPostAlterHook を呼び出して変更を通知する。 6. テーブルをクローズし、メモリ上のタプルを解放して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c (SQL解析後の実行フェーズ)
  • **呼び出し先**:
  • compute_common_attribute, LookupFuncWithArgs, object_ownercheck, interpret_func_volatility, interpret_func_support, update_proconfig_value, CatalogTupleUpdate 等。

関連データ構造

  • AlterFunctionStmt: SQL文の構文木。
  • Form_pg_proc: pg_proc カタログの行構造。
  • ObjectAddress: オブジェクトの識別子。

失敗モードと境界条件

  • 対象関数が見つからない場合、LookupFuncWithArgs がエラーを発生させる。
  • 所有者でない場合、aclcheck_error が発生する。
  • 集約関数に対して実行した場合、ERRCODE_WRONG_OBJECT_TYPE でエラーとなる。
  • COST や ROWS に非正の値を指定した場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーとなる。
  • proretset が偽の関数に対して ROWS を指定した場合、エラーとなる。

保守担当者向けメモ

  • procForm を介した直接的な値の更新は、CatalogTupleUpdate を呼び出す前までに行うこと。更新後の procForm はダングリングポインタとなるため注意が必要。
  • prokind によるプロシージャと関数の分岐処理が含まれているため、将来的な属性追加時は compute_common_attribute との整合性に留意すること。
