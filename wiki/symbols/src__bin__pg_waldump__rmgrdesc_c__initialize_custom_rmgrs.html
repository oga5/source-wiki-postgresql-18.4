initialize_custom_rmgrs

定義

  • 種別: function
  • 定義箇所: src/bin/pg_waldump/rmgrdesc.c:72

目的

カスタムリソースマネージャ用の記述子テーブル（CustomRmgrDesc）を初期化し、デフォルトのデコード処理を割り当てる。

シグネチャまたは形状

static void initialize_custom_rmgrs(void)

入力と出力

  • 入力: なし（グローバルなカスタムリソースマネージャ関連の静的配列を操作する）
  • 出力: なし（副作用として CustomRmgrDesc 配列および CustomRmgrDescInitialized フラグを更新する）

前提条件と不変条件

  • RM_N_CUSTOM_IDS 個のカスタムリソースマネージャが存在することを前提とする。
  • 実行後、CustomRmgrDescInitialized が true に設定される。

アルゴリズムまたは振る舞い

1. RM_N_CUSTOM_IDS 回ループし、各カスタムリソースマネージャに対して以下の処理を行う。

  • custom### 形式の識別子文字列を CustomNumericNames に生成する（### は RM_MIN_CUSTOM_ID からのオフセット）。
  • CustomRmgrDesc 配列の各要素に対し、生成した名前、および default_desc と default_identify 関数ポインタを割り当てる。

    • 2. 最後に CustomRmgrDescInitialized を true に設定し、初期化完了を示す。

呼び出し元と呼び出し先

  • 呼び出し元: GetRmgrDesc（src/bin/pg_waldump/rmgrdesc.c 内）から、カスタムリソースマネージャの初回アクセス時に呼び出される。
  • 呼び出し先: default_desc および default_identify（いずれも rmgrdesc.c 内で定義された関数）。

関連データ構造

  • CustomRmgrDesc: カスタムリソースマネージャの記述子を保持する配列。
  • CustomNumericNames: カスタムリソースマネージャの名前を格納するバッファ。
  • CustomRmgrDescInitialized: 初期化状態を管理するフラグ。

失敗モードと境界条件

  • snprintf による文字列生成において、CUSTOM_NUMERIC_NAME_LEN を超える名前は切り詰められる可能性がある。

保守担当者向けメモ

  • 本関数は GetRmgrDesc からのみ呼び出されることを想定している。
  • カスタムリソースマネージャの識別子範囲（RM_MIN_CUSTOM_ID）や個数（RM_N_CUSTOM_IDS）に変更がある場合は、本関数のループ範囲と名前生成ロジックを再確認すること。
