ItemPointerIsValid

定義

  • 種別: function
  • 定義箇所: src/include/storage/itemptr.h:82

目的

指定された ItemPointerData が有効なタプル位置を指しているかどうかを判定する。

シグネチャまたは形状

static inline bool ItemPointerIsValid(const ItemPointerData *pointer)

入力と出力

  • **入力**: pointer (const ItemPointerData *) - 検査対象のアイテムポインタ。
  • **出力**: bool - ポインタが非 NULL であり、かつオフセット番号 (ip_posid) が 0 でない場合に true を返す。

前提条件と不変条件

  • pointer は PointerIsValid マクロ（通常は NULL チェック）によって有効である必要がある。
  • ip_posid == 0 は無効な状態を示す。

アルゴリズムまたは振る舞い

1. PointerIsValid(pointer) を呼び出し、ポインタ自体が NULL でないことを確認する。 2. pointer->ip_posid != 0 を評価し、オフセット番号が有効範囲にあることを確認する。 3. 両方の条件が真である場合にのみ true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c
  • contrib/amcheck/verify_nbtree.c
  • src/backend/access/brin/brin_revmap.c
  • src/backend/access/gin/ginbulk.c
  • **呼び出し先**:
  • PointerIsValid (マクロ)

関連データ構造

  • ItemPointerData: 物理的なタプル位置を保持する 6 バイトの構造体。

失敗モードと境界条件

  • pointer が NULL の場合、PointerIsValid の定義に依存するが、一般的には false を返す。
  • ip_posid が 0 の場合、たとえポインタが非 NULL であっても false を返す。

保守担当者向けメモ

  • 本関数はインライン関数であり、パフォーマンスが重要なパスで頻繁に使用される。
  • ip_posid が MaxOffsetNumber を超える値を持つ場合は、特殊な状態（移動やトークン）として扱われるが、本関数は単に「0 ではないこと」のみを検証する。
