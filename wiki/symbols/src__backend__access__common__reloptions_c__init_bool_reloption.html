init_bool_reloption

定義

  • src/backend/access/common/reloptions.c:842

目的

ブール型のストレージパラメータ（リレーションオプション）を定義するためのメモリ領域を確保し、初期化を行う。

シグネチャまたは形状

static relopt_bool *init_bool_reloption(bits32 kinds, const char *name, const char *desc, bool default_val, LOCKMODE lockmode);

入力と出力

  • **入力**:
  • kinds: オプションが適用されるコンテキスト（ビットマスク）。
  • name: オプション名。
  • desc: オプションの説明文。
  • default_val: オプションのデフォルト値。
  • lockmode: オプション変更時に必要なロックレベル。
  • **出力**:
  • 初期化済みの relopt_bool 構造体へのポインタ。

前提条件と不変条件

  • allocate_reloption が正常にメモリを確保できること。
  • name および desc は有効な文字列ポインタであること。

アルゴリズムまたは振る舞い

1. allocate_reloption を呼び出し、RELOPT_TYPE_BOOL 型としてメモリを確保し、基本メタデータ（kinds, name, desc, lockmode）を設定する。 2. 確保された relopt_bool 構造体の default_val フィールドに、引数 default_val を代入する。 3. 初期化された構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/reloptions.c 内の add_bool_reloption および add_local_bool_reloption から呼び出される。
  • **呼び出し先**:
  • allocate_reloption

関連データ構造

  • relopt_bool: ブール型オプションのメタデータを保持する構造体。
  • RELOPT_TYPE_BOOL: オプションの型識別子。

失敗モードと境界条件

  • メモリ不足時には allocate_reloption 経由でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は静的関数であり、主に reloptions.c 内のオプション登録用ラッパー関数から利用される。
  • 新しいブール型オプションを追加する際は、直接本関数を呼ぶのではなく、公開されている add_bool_reloption 等のインターフェース経由で行うこと。
