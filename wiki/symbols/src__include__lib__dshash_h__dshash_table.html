dshash_table

定義

  • **種別**: struct
  • **定義箇所**: src/include/lib/dshash.h:21

目的

動的共有メモリ（DSM）領域上に構築される、プロセス間で共有可能な並行ハッシュテーブルを表現する不透明な構造体です。

シグネチャまたは形状

typedef struct dshash_table dshash_table;

※不透明型（opaque type）として定義されており、内部メンバは公開されていません。

入力と出力

本構造体自体はデータ構造のハンドルとして機能します。操作には dshash_parameters で定義されたハッシュ関数や比較関数が入力として必要となります。

前提条件と不変条件

  • **共有メモリ**: dsa（Dynamic Shared Area）を基盤として利用します。
  • **アクセス制御**: 内部的にパーティション単位の LWLock を使用して並行性を管理します。
  • **関数ポインタ**: プロセス間で共有できないため、各プロセスが dshash_attach する際に再指定する必要があります。

アルゴリズムまたは振る舞い

  • **並行アクセス**: パーティション分割により競合を最小化します。
  • **動的拡張**: テーブルの負荷が高まると、全パーティションのロックを取得してバケットの再配置（リサイズ）を行います。
  • **操作**: キーによる検索、挿入、削除、およびシーケンシャルスキャンをサポートします。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**: src/backend/lib/dshash.c 内の内部関数群（delete_item, resize, ensure_valid_bucket_pointers, find_in_bucket, insert_item_into_bucket, insert_into_bucket, delete_key_from_bucket, delete_item_from_bucket, hash_key, equal_keys, copy_key 等）。
  • **呼び出し先**: 外部からの直接的な呼び出し先情報は提供されていません。

関連データ構造

  • dshash_table_handle: 他プロセスと共有するための識別子。
  • dshash_parameters: ハッシュ関数や比較関数などの設定構造体。
  • dshash_seq_status: シーケンシャルスキャン時の状態保持用構造体。
  • dsa_pointer: 基盤となる共有メモリ領域管理ポインタ。

失敗モードと境界条件

  • **リサイズ時の負荷**: 全パーティションのロックが必要となるため、頻繁なリサイズはパフォーマンス低下を招きます。
  • **内部アクセス**: 不透明型であるため、内部メンバへの直接アクセスは禁止されています。

保守担当者向けメモ

  • 内部構造体（dshash_table_control 等）の詳細は src/backend/lib/dshash.c に定義されています。
  • 内部メンバへの直接アクセスは避け、必ず提供されている API を通じて操作してください。
