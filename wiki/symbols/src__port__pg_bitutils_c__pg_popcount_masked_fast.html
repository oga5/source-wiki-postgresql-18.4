pg_popcount_masked_fast

定義

  • 種別: function
  • 定義箇所: src/port/pg_bitutils.c:299

目的

バッファ内の各バイトに対して指定されたマスクを適用し、ビットがセットされている（1である）箇所の総数を高速にカウントする。

シグネチャまたは形状

static uint64 pg_popcount_masked_fast(const char *buf, int bytes, bits8 mask)

入力と出力

  • **入力**:
  • buf: カウント対象のデータバッファへのポインタ。
  • bytes: バッファのサイズ（バイト単位）。
  • mask: 各バイトに適用するビットマスク。
  • **出力**:
  • uint64: マスク適用後の全バイトにおける1の総数。

前提条件と不変条件

  • buf が指すメモリ領域は少なくとも bytes 分の有効なサイズを持つこと。
  • SIZEOF_VOID_P が 8 以上の場合、8バイト境界にアラインメントされていると効率的に処理される。
  • SIZEOF_VOID_P が 8 未満の場合、4バイト境界にアラインメントされていると効率的に処理される。

アルゴリズムまたは振る舞い

1. **マスク生成**: ポインタサイズに応じて、64ビットまたは32ビットのワード全体に mask を展開した maskv を作成する。 2. **ワード単位処理**: バッファがワード境界にアラインメントされている場合、ワード単位で読み込み、maskv とのビットANDをとった結果に対して pg_popcount64_fast または pg_popcount32_fast を呼び出して加算する。 3. **残余処理**: ワード単位処理で扱いきれなかった端数バイト、またはアラインメントされていないバッファに対しては、pg_number_of_ones ルックアップテーブルを使用してバイト単位でカウントを行う。 4. **戻り値**: 合計したカウント値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/pg_bitutils.c 内で pg_popcount_masked_optimized 関数ポインタに代入されている。直接の呼び出し元はソースコードから特定不可。
  • **呼び出し先**:
  • pg_popcount64_fast または pg_popcount32_fast（ハードウェア最適化実装）。
  • pg_number_of_ones（ルックアップテーブル）。

関連データ構造

  • pg_number_of_ones: 0-255の各値に対するビット数を保持するルックアップテーブル。

失敗モードと境界条件

  • bytes が 0 の場合、ループに入らず 0 を返す。
  • buf が NULL の場合、未定義動作となる。
  • アラインメントされていないバッファに対しては、ワード単位処理をスキップし、すべてバイト単位のルックアップ処理へフォールバックする。

保守担当者向けメモ

  • 本関数は pg_popcount_masked_choose によって実行時に選択される最適化実装の一つである。
  • コンパイル時の SIZEOF_VOID_P 定義により、64ビットパスと32ビットパスのいずれかが選択される。
  • pg_popcount64_fast 等の依存関数は、CPUの POPCNT 命令サポート状況に依存するため、実行環境のCPU特性を考慮する必要がある。
