parse_compress_options

定義

  • 種別: function
  • 定義箇所: src/common/compression.c:425

目的

圧縮設定文字列を「アルゴリズム名」と「詳細オプション」に分割し、後続の処理で利用可能な形式に変換する。

シグネチャまたは形状

void parse_compress_options(const char *option, char **algorithm, char **detail)

入力と出力

  • **入力**:
  • option: 解析対象の圧縮設定文字列（例: "gzip", "zstd:level=5"）。
  • **出力**:
  • algorithm: 抽出されたアルゴリズム名（呼び出し側で pfree が必要）。
  • detail: 抽出された詳細オプション文字列（オプションがない場合は NULL、呼び出し側で pfree が必要）。

前提条件と不変条件

  • algorithm および detail は、呼び出し側で確保されたポインタのアドレスであること。
  • 内部で pstrdup および palloc を使用しているため、呼び出し側は結果として返された文字列のメモリ管理（解放）に責任を持つこと。

アルゴリズムまたは振る舞い

1. **数値のみの入力判定**: strtol を使用し、入力が数値のみか確認する。

  • 0 の場合: algorithm に "none" を設定し、detail は NULL とする。
  • 0 以外の場合: 後方互換性のため algorithm に "gzip" を設定し、detail に入力文字列全体をコピーする。

    • 2. **アルゴリズムと詳細の分離**: 文字列に : が含まれるか確認する。

  • : がない場合: 入力全体を algorithm とし、detail は NULL とする。
  • : がある場合: : より前の部分を algorithm に、後ろの部分を detail に分割してコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c:1007
  • src/bin/pg_basebackup/pg_receivewal.c:728
  • src/bin/pg_dump/pg_dump.c:683
  • **呼び出し先**:
  • pstrdup (PostgreSQL メモリ管理関数)
  • palloc (PostgreSQL メモリ管理関数)
  • strtol, strchr, memcpy (標準 C ライブラリ)

関連データ構造

  • pg_compress_specification: 本関数で分割された文字列を元に、後続の parse_compress_specification() が初期化する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc/pstrdup 経由でエラーが発生する可能性がある。
  • 空文字列が渡された場合の挙動は、strtol の結果および : の有無に依存する。

保守担当者向けメモ

  • 本関数はフロントエンドツール（pg_dump 等）での利用を想定している。
  • 数値のみの指定を gzip として扱うレガシーな挙動は、将来的なアルゴリズム追加時に競合しないよう注意が必要である。
