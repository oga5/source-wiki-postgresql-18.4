domain_in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/domains.c:226

目的

ドメイン型に対する標準的な入力関数として、文字列形式の入力を受け取り、基底型への変換および定義されたドメイン制約の検証を行う。

シグネチャまたは形状

Datum domain_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_CSTRING(0): 入力値（文字列）。
  • PG_GETARG_OID(1): 対象となるドメイン型の OID。
  • **出力**:
  • Datum: 変換および検証済みの値。
  • NULL: 入力が NULL の場合、または検証に失敗した場合。

前提条件と不変条件

  • domain_in は非厳密（non-strict）な関数であり、NULL 入力を許容する。
  • fcinfo->flinfo->fn_extra を使用して、カタログ検索を最小化するための DomainIOData をキャッシュする。
  • ドメイン型が変更されない限り、キャッシュされた DomainIOData を再利用する。

アルゴリズムまたは振る舞い

1. **入力チェック**: 第1引数が NULL の場合は string を NULL として扱い、第2引数（型 OID）が NULL の場合は即座に NULL を返す。 2. **キャッシュ管理**: fn_extra を確認し、未初期化または型が一致しない場合は domain_state_setup を呼び出して DomainIOData を構築・保存する。 3. **基底型変換**: InputFunctionCallSafe を呼び出し、基底型の入力関数を実行して文字列を Datum に変換する。 4. **制約検証**: domain_check_input を呼び出し、変換後の値に対して NOT NULL や CHECK 制約を評価する。 5. **戻り値**: 検証成功時は変換後の Datum を返し、入力が NULL であった場合は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL レベルでの型変換処理（直接的な呼び出し元はソースコード上では明示されていないが、SQL インターフェース経由で呼び出される）。
  • **呼び出し先**:
  • domain_state_setup: キャッシュ構築用。
  • InputFunctionCallSafe: 基底型の入力処理用。
  • domain_check_input: 制約検証用。

関連データ構造

  • DomainIOData: ドメインの I/O 状態、基底型の関数情報、制約リスト、評価コンテキストを保持する構造体。

失敗モードと境界条件

  • **NULL 入力**: 入力が NULL の場合、制約チェックをスキップして NULL を返す。
  • **制約違反**: domain_check_input 内で制約違反が検出された場合、エラーが報告される（escontext を介したエラーハンドリング）。
  • **型不一致**: キャッシュされた domain_type と現在の domainType が異なる場合、再初期化が行われる。

保守担当者向けメモ

  • 本関数は FmgrInfo の fn_extra を利用したキャッシュ機構に依存している。
  • CHECK 制約の評価には ExprContext を使用するため、メモリ管理（ReScanExprContext 等）が domain_check_input 側で適切に行われていることを前提としている。
