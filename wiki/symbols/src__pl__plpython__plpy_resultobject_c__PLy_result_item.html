PLy_result_item

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_resultobject.c:243

目的

PLyResultObject が保持するクエリ結果の行リストから、指定されたインデックスの行（Pythonオブジェクト）を取得します。

シグネチャまたは形状

static PyObject *PLy_result_item(PyObject *arg, Py_ssize_t idx)

入力と出力

  • **入力**:
  • arg: PLyResultObject へのポインタ（PyObject としてキャスト）。
  • idx: 取得対象の行インデックス。
  • **出力**:
  • 指定されたインデックスの PyObject（行データ）。取得に成功した場合は参照カウントがインクリメントされて返されます。失敗した場合は NULL を返します。

前提条件と不変条件

  • arg は PLyResultObject 型の有効なインスタンスである必要があります。
  • 内部の rows メンバは PyList であることが前提です。

アルゴリズムまたは振る舞い

1. arg を PLyResultObject 構造体にキャストします。 2. PyList_GetItem を使用して、内部の rows リストから idx 番目の要素を取得します。 3. 取得した要素が NULL でない場合、Py_INCREF を呼び出して参照カウントを増やします。 4. 取得したオブジェクト（または NULL）を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Py_sq_item スロットを通じてPythonインタープリタから呼び出されます（src/pl/plpython/plpy_resultobject.c:45）。
  • **呼び出し先**: PyList_GetItem（Python C API）、Py_INCREF（Python C API）。

関連データ構造

  • PLyResultObject: クエリ結果を保持するC構造体。内部に rows（PyObject *）を保持します。

失敗モードと境界条件

  • idx がリストの範囲外である場合、PyList_GetItem は NULL を返し、本関数もそのまま NULL を返します。
  • arg が不正なポインタである場合の挙動は未定義です。

保守担当者向けメモ

  • 本関数はPythonのシーケンスプロトコルの一部として実装されています。
  • 参照カウントの管理（Py_INCREF）は、Pythonのリストから要素を取得する際の標準的な作法に従っています。
