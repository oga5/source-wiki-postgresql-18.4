latin1_to_mic

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/latin_and_mic/latin_and_mic.c:45

目的

ISO-8859-1 (LATIN1) エンコーディングの文字列を MULE_INTERNAL (MIC) エンコーディングへ変換する PostgreSQL の動的ロード可能な変換プロシージャです。

シグネチャまたは形状

Datum latin1_to_mic(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の引数を受け取ります。
  • src (CSTRING): 変換元の文字列バッファ。
  • dest (CSTRING): 変換後の文字列を格納するバッファ。
  • len (INT32): 変換対象のバイト長。
  • noError (BOOL): 変換エラー発生時の挙動フラグ。
  • **出力**: 変換されたバイト数を Datum 型（INT32）として返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_LATIN1, PG_MULE_INTERNAL) により、呼び出し時にエンコーディングIDが LATIN1 から MULE_INTERNAL への変換として正しいことが検証されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得します。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングの整合性を確認します。 3. latin2mic ヘルパー関数を呼び出し、LC_ISO8859_1 を指定して実際の変換処理を実行します。 4. 変換されたバイト数を PG_RETURN_INT32 で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の変換フレームワーク（直接的な呼び出し元情報はソースコード上では明示されていませんが、PG_FUNCTION_INFO_V1(latin1_to_mic) マクロにより関数として登録されています）。
  • **呼び出し先**: latin2mic（mb/pg_wchar.h で定義された変換ヘルパー関数）。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQL の関数マネージャ（fmgr）が使用する引数構造体。
  • PG_LATIN1, PG_MULE_INTERNAL: エンコーディング識別子。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、PostgreSQL のエラーハンドリング機構により処理が中断されます。
  • noError 引数の値に応じて、変換中の不正な文字に対する挙動が latin2mic 内で制御されます。

保守担当者向けメモ

  • 本関数はボイラープレート的なラッパーです。変換ロジックの変更が必要な場合は、src/backend/utils/mb/wchar.c にある latin2mic を参照してください。
