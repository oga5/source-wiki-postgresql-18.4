GetHugePageSize

定義

  • 種別: function
  • 定義箇所: src/backend/port/sysv_shmem.c:478

目的

mmap で共有メモリを確保する際に使用する Huge Page のサイズと、対応する mmap フラグを取得する。

シグネチャまたは形状

void GetHugePageSize(Size *hugepagesize, int *mmap_flags)

入力と出力

  • **入力**:
  • hugepagesize: Huge Page サイズを格納するための Size 型ポインタ。
  • mmap_flags: mmap に渡すフラグを格納するための int 型ポインタ。
  • **出力**:
  • 引数ポインタ経由で値が書き込まれる（MAP_HUGETLB が未定義の場合は 0 が書き込まれる）。

前提条件と不変条件

  • MAP_HUGETLB が定義されている環境でのみ、具体的なサイズとフラグが計算される。
  • Linux 環境では /proc/meminfo からシステムデフォルトの Huge Page サイズを読み取る。
  • huge_page_size（GUC変数）が設定されている場合は、システムデフォルトよりも優先される。

アルゴリズムまたは振る舞い

1. MAP_HUGETLB が未定義の場合、引数に 0 を設定して即座に終了する。 2. Linux 環境であれば /proc/meminfo を解析し、Hugepagesize を取得する。 3. 以下の優先順位で hugepagesize_local を決定する:

  • 明示的な設定 (huge_page_size)
  • システムデフォルト (default_hugepagesize)
  • 上記が不明な場合のフォールバック値: 2MB

    • 4. mmap_flags_local に MAP_HUGETLB を設定する。 5. MAP_HUGE_MASK および MAP_HUGE_SHIFT が定義されている場合、ページサイズに応じたシフト値をフラグに加算する。 6. 最後に、引数として渡されたポインタが NULL でない場合に結果を代入する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/port/sysv_shmem.c
  • src/backend/port/win32_shmem.c
  • src/backend/storage/ipc/ipci.c
  • src/backend/storage/ipc/shmem.c
  • **呼び出し先**:
  • AllocateFile, fgets, sscanf, FreeFile (Linux環境でのファイル読み込み)
  • pg_ceil_log2_64 (フラグ計算時)

関連データ構造

  • Size (PostgreSQL のメモリサイズ型)
  • huge_page_size (GUC 変数)

失敗モードと境界条件

  • /proc/meminfo の読み込みや解析に失敗した場合、デフォルト値（2MB）が使用される。
  • MAP_HUGETLB が定義されていないプラットフォームでは、常に 0 が返される。
  • 実際のシステム設定が 2MB より大きい場合、mmap や munmap がアライメントエラーで失敗する可能性がある。

保守担当者向けメモ

  • MAP_HUGETLB が定義されている環境でのみ動作する。
  • Linux 以外のプラットフォームにおける Huge Page サイズの取得ロジックは現状存在せず、2MB 固定のフォールバックに依存している。
