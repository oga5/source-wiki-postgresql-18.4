ossl_cipher

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/openssl.c:239

目的

各暗号アルゴリズムに関する静的なメタデータおよび初期化関数を保持し、OpenSSL の EVP API を pgcrypto の抽象インターフェースへマッピングするための構造体です。

シグネチャまたは形状

struct ossl_cipher
{
    int (*init) (PX_Cipher *c, const uint8 *key, unsigned klen, const uint8 *iv);
    ossl_EVP_cipher_func cipher_func;
    int block_size;
    int max_key_size;
};

入力と出力

  • init: 暗号コンテキスト PX_Cipher、鍵データ、鍵長、初期化ベクトル（IV）を入力として受け取り、成功時に 0、失敗時にエラーコードを返します。
  • cipher_func: OpenSSL の EVP_CIPHER を取得するための関数ポインタです。
  • block_size, max_key_size: それぞれ暗号アルゴリズムのブロックサイズおよび最大鍵長を整数で保持します。

前提条件と不変条件

  • 本構造体のインスタンスは、contrib/pgcrypto/openssl.c 内で static const として定義される必要があります。
  • init 関数は、PX_Cipher が適切に割り当てられた状態で呼び出されることを前提とします。

アルゴリズムまたは振る舞い

  • 各暗号アルゴリズム（Blowfish, DES, AES 等）の特性を静的に定義します。
  • init フィールドを通じて、特定の暗号アルゴリズムに対する初期化処理をカプセル化します。
  • cipher_func を介して、実行時に OpenSSL の具体的な暗号実装へアクセスします。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/openssl.c 内の ossl_cipher_lookup 等で参照されます。
  • 呼び出し先: ossl_bf_cbc, ossl_aes_ecb 等の静的インスタンス定義において、具体的な初期化関数や EVP 関連関数が割り当てられます。

関連データ構造

  • PX_Cipher: pgcrypto モジュール共通の暗号化インターフェース構造体。
  • ossl_EVP_cipher_func: OpenSSL の EVP_CIPHER を返す関数型。

失敗モードと境界条件

  • init 関数が提供する初期化処理において、鍵長が max_key_size を超える場合や、OpenSSL 側の初期化に失敗した場合にエラーを返します。

保守担当者向けメモ

  • 新しい暗号アルゴリズムを追加する際は、本構造体のインスタンスを static const で定義し、ossl_cipher_lookup に登録してください。
  • init 関数内でのリソース確保は、ResourceOwner を介した管理が必須です。
