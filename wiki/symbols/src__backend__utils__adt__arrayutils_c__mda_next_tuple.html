mda_next_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arrayutils.c:207

目的

多次元配列の要素を走査する際、現在のインデックス位置から次のインデックス位置へ更新します。

シグネチャまたは形状

int mda_next_tuple(int n, int *curr, const int *span);

入力と出力

  • n: 配列の次元数。
  • curr: 現在のインデックス位置を保持する配列（入力および更新対象）。
  • span: 各次元のサイズを保持する配列。
  • 戻り値:
  • 0以上: 更新された次元のインデックス。
  • -1: 全ての要素の走査が完了したことを示す。

前提条件と不変条件

  • n は正の整数である必要があります。
  • curr および span は少なくとも n 個の要素を持つ配列である必要があります。
  • curr の各要素は 0 <= curr[i] < span[i] を満たしている必要があります。

アルゴリズムまたは振る舞い

1. 最下位次元（n-1）のインデックスをインクリメントし、span[n-1] で剰余をとります。 2. インクリメントの結果、値が 0 に戻った場合（桁上がりが発生した場合）、上位の次元に対して同様のインクリメントと剰余計算を繰り返します。 3. 桁上がりが止まった次元のインデックスを返します。 4. 全ての次元が一周して 0 に戻った場合（走査終了）、-1 を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/arrayfuncs.c 内の複数の箇所で、多次元配列の反復処理ループの条件式として使用されています。
  • 呼び出し先: 本関数は直接的な関数呼び出しを行いません。

関連データ構造

  • ArrayType: PostgreSQL の配列内部表現（utils/array.h で定義）。

失敗モードと境界条件

  • n <= 0 の場合、直ちに -1 を返します。
  • 配列の全要素を走査し終えた場合、戻り値は -1 となります。

保守担当者向けメモ

  • 本関数は「Multi-Dimensional Array」の略称である mda_ 系ユーティリティの一部です。
  • 呼び出し元は、本関数が返す戻り値が -1 になるまでループを継続する設計となっています。
