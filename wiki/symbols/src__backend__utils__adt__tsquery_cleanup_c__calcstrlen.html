calcstrlen

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_cleanup.c:362

目的

NODE 構造体で表現された tsquery ツリーを走査し、最終的な TSQuery オブジェクトを生成するために必要な文字列バッファの合計サイズを算出します。

シグネチャまたは形状

static int32 calcstrlen(NODE *node)

入力と出力

  • **入力**: NODE *node（クエリツリーのノードへのポインタ）
  • **出力**: int32（ノードおよびその子孫に含まれるオペランドの文字列長と終端文字の合計サイズ）

前提条件と不変条件

  • node は有効な NODE 構造体を指している必要があります。
  • node->valnode が QI_VAL の場合、qoperand.length が有効である必要があります。
  • node->valnode が QI_OPR の場合、node->right および node->left が適切にリンクされている必要があります。

アルゴリズムまたは振る舞い

1. ノードの型が QI_VAL（オペランド）の場合、そのオペランドの長さ（length）に終端文字分（+1）を加算して返します。 2. ノードの型が QI_OPR（演算子）の場合:

  • 右ノード（node->right）に対して再帰的に calcstrlen を呼び出し、サイズを加算します。
  • 演算子が OP_NOT 以外の場合、左ノード（node->left）に対しても再帰的に calcstrlen を呼び出し、サイズを加算します。

    • 3. 最終的な合計サイズを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsquery_cleanup.c 内の cleanup_tsquery_stopwords 関数（lenstr = calcstrlen(root); として呼び出し）。
  • **呼び出し先**: 自身（再帰呼び出し）。

関連データ構造

  • NODE: クエリツリーを構成するノード構造体。
  • QueryItem: tsquery の内部表現。

失敗モードと境界条件

  • **スタックオーバーフロー**: 再帰的なツリー走査を行うため、非常に深いクエリツリーに対してはスタックを消費します（ファイル内の他関数と同様、check_stack_depth() による保護が前提となります）。
  • **不正なポインタ**: node が NULL の場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は tsquery の再構築時に必要なメモリ量を決定する重要なステップです。
  • OP_NOT が単項演算子として扱われ、左ノードを無視するロジックが含まれている点に注意してください。
