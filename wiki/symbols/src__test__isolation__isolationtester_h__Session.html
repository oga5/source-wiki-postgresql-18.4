Session

定義

  • 種別: struct
  • 定義箇所: src/test/isolation/isolationtester.h:24

目的

PostgreSQLの分離レベルテストにおいて、独立したデータベース接続をシミュレートするための単位を定義します。テスト仕様（TestSpec）の一部として、特定の接続に関連付けられたSQL実行環境を保持します。

シグネチャまたは形状

typedef struct
{
	char	   *name;
	char	   *setupsql;
	char	   *teardownsql;
	Step	  **steps;
	int			nsteps;
} Session;

入力と出力

  • **入力**: specparse.y による解析結果に基づき、セッション名、セットアップ/ティアダウン用のSQL文字列、および実行可能なステップの配列が格納されます。
  • **出力**: 実行エンジン（isolationtester.c）が本構造体を参照し、各セッションに対して順次SQLを発行します。

前提条件と不変条件

  • steps は Step 構造体へのポインタ配列です。
  • nsteps は steps 配列の要素数を保持します。
  • isolationtester.c 内の check_testspec() が実行されるまで、内部のポインタ参照は未解決である可能性があります。

アルゴリズムまたは振る舞い

本構造体はデータ保持に特化しており、直接的なロジックは持ちません。テスト実行エンジンは、TestSpec を走査する過程でこの Session を参照し、定義された setupsql を実行した後に steps に含まれる各ステップを順次処理します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/isolation/isolationtester.c（実行エンジンがテスト実行時に参照）。
  • **呼び出し先**: Step 構造体（steps メンバを通じて参照）。
  • 注記: ツールが検出した src/backend/access/common/session.c 等の他の Session 構造体は、本テスト用構造体とは無関係です。

関連データ構造

  • TestSpec: セッションを管理するルート構造体。
  • Step: セッション内で実行される個別のSQLコマンドを定義する構造体。

失敗モードと境界条件

  • steps が NULL または nsteps が 0 の場合、そのセッションでは実行すべきステップが存在しないものとして扱われます。
  • setupsql や teardownsql が NULL の場合、対応するSQL実行はスキップされます。

保守担当者向けメモ

  • 本構造体はテスト定義用であり、バックエンドのセッション管理機能とは別物です。
  • Step 構造体とのリンクは check_testspec() で構築されるため、構造体定義の変更時にはこのリンク解決ロジックへの影響を確認してください。
