ExecProcNodeFirst

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execProcnode.c:447

目的

プランノードの初回実行時に、スタック深さのチェックと、以降の実行効率化のための関数ポインタ（ExecProcNode）の切り替えを行う。

シグネチャまたは形状

static TupleTableSlot *ExecProcNodeFirst(PlanState *node)

入力と出力

  • **入力**: PlanState *node（実行対象のプランノード状態）
  • **出力**: TupleTableSlot *（ノードの実行結果として取得されたタプル）

前提条件と不変条件

  • 本関数は、対象プランノードの初回実行時にのみ呼び出されることを前提としている。
  • ExecProcNode の呼び出しは、同一プランノードに対して常にほぼ同じスタック深さで行われると仮定している。

アルゴリズムまたは振る舞い

1. check_stack_depth() を呼び出し、スタックの深さを検証する。 2. node->instrument の有無を確認し、実行統計情報の計測が必要か判断する。

  • 計測が必要な場合: node->ExecProcNode を ExecProcNodeInstr に設定する。
  • 不要な場合: node->ExecProcNode を node->ExecProcNodeReal（直接実行用関数）に設定する。

    • 3. 切り替え後の node->ExecProcNode を呼び出し、その結果を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内の ExecProcNode 呼び出し箇所（初期化直後の初回実行時）。
  • **呼び出し先**:
  • check_stack_depth()
  • node->ExecProcNode（切り替え後の関数ポインタ）

関連データ構造

  • PlanState: 実行中のプランノードの状態を保持する構造体。instrument フィールドや関数ポインタ ExecProcNode を含む。
  • TupleTableSlot: 実行結果として返されるタプルを保持する構造体。

失敗モードと境界条件

  • スタック深さが許容範囲を超えている場合、check_stack_depth() によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、頻繁に呼び出される ExecProcNode のオーバーヘッドを削減するための動的ディスパッチ機構の一部である。
  • 新しいノードタイプを追加する際、本関数のロジック自体を変更する必要は通常ないが、ExecProcNodeReal が正しく設定されていることを確認すること。
