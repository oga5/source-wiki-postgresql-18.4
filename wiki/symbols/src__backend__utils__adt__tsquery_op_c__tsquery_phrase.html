tsquery_phrase

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_op.c:149

目的

2つの tsquery 型の引数を受け取り、それらが隣接（距離1）して出現するフレーズ検索条件を構築します。

シグネチャまたは形状

Datum tsquery_phrase(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの tsquery 型の引数を受け取ります。
  • **出力**: Datum 型として、距離1のフレーズ検索条件を表す新しい tsquery を返します。

前提条件と不変条件

  • 入力された tsquery は、内部的に QTNode ツリー構造へ変換可能な有効な形式である必要があります。
  • 本関数は tsquery_phrase_distance を呼び出すラッパーであり、距離引数として固定値 1 を渡します。

アルゴリズムまたは振る舞い

1. PG_GETARG_DATUM を使用して、2つの入力引数を取得します。 2. DirectFunctionCall3 を介して tsquery_phrase_distance 関数を呼び出します。 3. その際、第3引数（距離）として Int32GetDatum(1) を明示的に渡します。 4. tsquery_phrase_distance の戻り値をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: tsquery_phrase_distance
  • **呼び出し元**: ソースコード内での直接的な呼び出し元は明示されていませんが、SQL インターフェースを通じて呼び出されることを想定しています。

関連データ構造

  • tsquery: 全文検索用のデータ型。
  • QTNode: tsquery を操作するために内部的に使用されるツリー構造。

失敗モードと境界条件

  • 入力引数が無効な tsquery 形式である場合、内部で呼び出される tsquery_phrase_distance に依存してエラーが発生する可能性があります。
  • メモリ不足時には、tsquery の変換処理に伴いシステムエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は単なる距離指定のショートカットです。フレーズ検索のロジック変更は tsquery_phrase_distance を参照してください。
  • メモリ管理は tsquery_phrase_distance 内で行われるため、本関数側での追加の解放処理は不要です。
