decompress_init

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-compress.c:203

目的

PGP伸長フィルタ（PullFilter）の初期化を行い、zlib ストリームの状態をセットアップします。

シグネチャまたは形状

static int decompress_init(void **priv_p, void *arg, PullFilter *src);

入力と出力

  • **入力**:
  • priv_p: 伸長用データ構造（DecomprData）を格納するためのポインタ。
  • arg: PGP_Context 構造体へのポインタ。
  • src: 読み込み元の PullFilter。
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時は PXE_PGP_UNSUPPORTED_COMPR または PXE_PGP_COMPRESSION_ERROR を返します。

前提条件と不変条件

  • ctx->compress_algo が PGP_COMPR_ZLIB または PGP_COMPR_ZIP である必要があります。
  • z_alloc および z_free 関数が zlib のメモリ管理に使用されます。

アルゴリズムまたは振る舞い

1. 圧縮アルゴリズムがサポート対象外であればエラーを返します。 2. DecomprData 構造体をメモリ確保し、priv_p に割り当てます。 3. zlib のメモリ管理関数として z_alloc と z_free を設定します。 4. アルゴリズムに応じて zlib の初期化を行います。

  • PGP_COMPR_ZIP の場合: inflateInit2(&dec->stream, -15) を呼び出します。
  • PGP_COMPR_ZLIB の場合: inflateInit(&dec->stream) を呼び出します。

    • 5. 初期化に失敗した場合はメモリを解放し、エラーコードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: pgp_decompress_filter 関数から参照されています。
  • **呼び出し先**: palloc0, pfree, inflateInit, inflateInit2, px_debug。

関連データ構造

  • DecomprData: 伸長処理の状態を保持する構造体。
  • PGP_Context: PGP処理のコンテキスト情報。
  • PullFilter: フィルタパイプラインの抽象インターフェース。

失敗モードと境界条件

  • サポートされていない圧縮アルゴリズムが指定された場合、PXE_PGP_UNSUPPORTED_COMPR を返します。
  • zlib の初期化（inflateInit / inflateInit2）が Z_OK を返さない場合、メモリを解放した上で PXE_PGP_COMPRESSION_ERROR を返します。

保守担当者向けメモ

  • inflateInit2 に渡される -15 は、zlib のヘッダーなし（raw deflate）形式を扱うための設定です。
  • z_alloc および z_free は PostgreSQL のメモリ管理関数をラップしており、メモリリークを防止する設計となっています。
