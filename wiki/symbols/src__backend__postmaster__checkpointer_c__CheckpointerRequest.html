CheckpointerRequest

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/checkpointer.c:107

目的

バックエンドプロセスからチェックポインタープロセスへ、特定のファイルの同期（fsync）要求を伝達するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	SyncRequestType type;		/* 要求の種類 */
	FileTag		ftag;			/* ファイル識別子 */
} CheckpointerRequest;

入力と出力

  • **入力**: SyncRequestType（要求種別）および FileTag（対象ファイル情報）。
  • **出力**: なし（構造体定義のため）。

前提条件と不変条件

  • CheckpointerShmemStruct 内の共有メモリキューに格納される要素として使用されます。
  • ftag は対象ファイルを一意に特定できる情報である必要があります。

アルゴリズムまたは振る舞い

本構造体はデータ保持のみを目的としています。バックエンドが ForwardSyncRequest を通じてこの構造体を共有メモリのキューに書き込み、チェックポインターが AbsorbSyncRequests を介してこれを読み取り、順次処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/checkpointer.c 内の各関数（ForwardSyncRequest 等）が、この構造体をインスタンス化し、共有メモリへ書き込みます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数ではないため呼び出し先はありません。

関連データ構造

  • SyncRequestType: 要求の種類を定義する列挙型。
  • FileTag: 同期対象ファイルを識別するための構造体。
  • CheckpointerShmemStruct: 本構造体を配列として保持する共有メモリ上の管理構造体。

失敗モードと境界条件

  • 共有メモリ上のキューが満杯の場合、要求の受け入れが制限される可能性があります。
  • ftag が無効なファイルを参照している場合、チェックポインター側での同期処理時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は CheckpointerShmemStruct の一部として FLEXIBLE_ARRAY_MEMBER を用いた配列で管理されています。サイズ変更やフィールド追加を行う際は、共有メモリのレイアウトおよび sizeof(CheckpointerRequest) を使用している箇所（memcpy や palloc 等）への影響を考慮してください。
