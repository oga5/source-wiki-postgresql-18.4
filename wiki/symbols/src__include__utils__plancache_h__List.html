List

定義

  • 種別: typedef
  • 定義箇所: src/include/utils/plancache.h:42

目的

PostgreSQL のクエリ計画キャッシュ管理において、ノードやオブジェクトの集合を保持するための汎用的なリスト構造を定義します。

シグネチャまたは形状

typedef struct List List;

（注：src/include/utils/plancache.h においては前方宣言または型定義として扱われています）

入力と出力

  • 入力: 関連する関数（例: PostRewriteHook）において、クエリツリーのリスト（List *querytree_list）として引数に渡されます。
  • 出力: リスト操作関数を通じて、ノードの集合や特定の要素が返されます。

前提条件と不変条件

  • plancache.h 内の文脈では、クエリの解析結果や計画に関連するオブジェクトを格納するために使用されます。
  • 侵入型リスト（ilist.h）とは異なる、PostgreSQL の標準的なノードリスト構造として利用されます。

アルゴリズムまたは振る舞い

  • 複数のクエリツリーやオプション、属性情報を順序立てて保持します。
  • PostRewriteHook のようなコールバック関数において、書き換え後のクエリツリーリストを管理するために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • contrib/dblink/dblink.c: 接続オプションのリスト管理に使用。
  • contrib/file_fdw/file_fdw.c: COPY オプションやスキャン句のリスト管理に使用。
  • contrib/amcheck/verify_heapam.c: 属性情報のリスト管理に使用。
  • 呼び出し先:
  • 提供されたソース excerpt からは、特定の関数呼び出し先は特定できません。

関連データ構造

  • CachedPlanSource: クエリのソース情報を管理する構造体。
  • PostRewriteHook: List を引数に取るフック関数の型定義。

失敗モードと境界条件

  • メモリ不足: リストへの要素追加時にメモリ割り当てが失敗する可能性があります。
  • 空リスト: リストが空（NIL）の場合の処理は、各操作関数に依存します。

保守担当者向けメモ

  • 本シンボルは PostgreSQL 全体で多用される汎用的なリスト構造です。plancache.h においては、主にクエリの解析・書き換えプロセスで生成されるノードの保持に使用されます。
  • lib/ilist.h が直接インクルードされていますが、List 自体は nodes/pg_list.h で定義される標準的なリスト構造を指すことが一般的です。実装の詳細を確認する場合は nodes/pg_list.h を参照してください。
