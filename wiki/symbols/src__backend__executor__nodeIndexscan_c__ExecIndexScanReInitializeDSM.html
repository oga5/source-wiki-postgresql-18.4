ExecIndexScanReInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexscan.c:1746

目的

並列クエリ実行において、インデックススキャンの共有メモリ（DSM）状態を再初期化し、新たなスキャンを開始可能な状態にする。

シグネチャまたは形状

void ExecIndexScanReInitializeDSM(IndexScanState *node, ParallelContext *pcxt);

入力と出力

  • **入力**:
  • node: IndexScanState 構造体へのポインタ。スキャンノードの状態を保持する。
  • pcxt: ParallelContext 構造体へのポインタ。並列実行コンテキスト。
  • **出力**: なし（void）。

前提条件と不変条件

  • node->ss.ps.plan->parallel_aware が真である必要がある（Assert により検証）。

アルゴリズムまたは振る舞い

1. ノードが並列実行可能であることを確認する。 2. index_parallelrescan を呼び出し、node->iss_ScanDesc が保持するインデックススキャン記述子の並列スキャン状態をリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execParallel.c:983
  • **呼び出し先**:
  • index_parallelrescan（インデックスアクセスメソッドのインターフェース）

関連データ構造

  • IndexScanState: インデックススキャンノードの実行時状態を管理する構造体。
  • ParallelContext: 並列クエリ実行のコンテキスト。

失敗モードと境界条件

  • parallel_aware フラグが設定されていないノードに対して呼び出された場合、アサーション違反により異常終了する。

保守担当者向けメモ

  • 本関数は、並列ワーカーがスキャンを再開する際に呼び出される。インデックス固有の並列スキャン状態のリセットのみを担うため、他の実行状態のリセットが必要な場合は別途処理を行う必要がある。
