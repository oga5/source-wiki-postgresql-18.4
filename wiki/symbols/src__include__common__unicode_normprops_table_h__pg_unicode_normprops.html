pg_unicode_normprops

目的

Unicode 正規化（Normalization）における「クイックチェック（Quick Check）」処理において、特定のコードポイントが正規化済みであるか、あるいは正規化が必要かを高速に判定するためのデータ構造を提供します。

シグネチャまたは形状

typedef struct
{
	unsigned int codepoint:21;
	signed int	quickcheck:4;
} pg_unicode_normprops;

入力と出力

  • **入力**: 構造体内の codepoint（21ビット）および quickcheck（4ビット）フィールド。
  • **出力**: 正規化状態（YES/NO/MAYBE）を示す quickcheck 値。

前提条件と不変条件

  • メモリ効率化のため、ビットフィールドを使用してサイズを最小化しています。
  • 本構造体は、生成スクリプト generate-unicode_normprops_table.pl によって自動生成される静的テーブルの一部として使用されます。

アルゴリズムまたは振る舞い

1. 正規化処理の際、対象のコードポイントに対して完全ハッシュ関数（NFC_QC_hash_func 等）を用いてインデックスを算出します。 2. 算出されたインデックスを用いて、本構造体の配列から該当するエントリを取得します。 3. 取得した quickcheck 値に基づき、正規化の要否を判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/unicode_norm.c 内の正規化処理ロジックが、本構造体の配列（UnicodeNormProps_NFC_QC 等）を参照します。
  • **呼び出し先**: 情報なし。

関連データ構造

  • pg_unicode_norminfo: 本構造体を含むルックアップテーブル、ハッシュ関数、要素数を管理する構造体。

失敗モードと境界条件

  • 本構造体は完全ハッシュ関数と組み合わせて使用されるため、ハッシュ衝突が発生しない設計となっています。
  • 手動編集は禁止されており、変更が必要な場合は生成スクリプトを修正する必要があります。
