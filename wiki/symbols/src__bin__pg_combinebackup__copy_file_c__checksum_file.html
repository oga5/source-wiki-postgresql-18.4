checksum_file

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/copy_file.c:140

目的

指定されたファイルの内容を読み込み、提供されたコンテキストを使用してチェックサムを計算・更新する。主に、コピー操作自体がチェックサム計算を伴わない手法（CLONE、LINK 等）の後に検証を行うために使用される。

シグネチャまたは形状

static void checksum_file(const char *src, pg_checksum_context *checksum_ctx)

入力と出力

  • **入力**:
  • src: チェックサム対象ファイルのパス（const char *）。
  • checksum_ctx: チェックサム計算の状態を保持するコンテキスト（pg_checksum_context *）。
  • **出力**: なし（void）。計算結果は checksum_ctx に反映される。

前提条件と不変条件

  • checksum_ctx->type が CHECKSUM_TYPE_NONE の場合、関数は即座にリターンし、ファイル操作は行われない。
  • ファイルの読み込みには 50 * BLCKSZ バイトのバッファが動的に割り当てられる。

アルゴリズムまたは振る舞い

1. checksum_ctx のタイプを確認し、不要であれば終了する。 2. src で指定されたファイルを読み取り専用モード（O_RDONLY | PG_BINARY）で開く。 3. ファイルの終端に達するまで、50 * BLCKSZ バイト単位でデータを読み込む。 4. 読み込んだ各ブロックを pg_checksum_update に渡し、チェックサムを更新する。 5. 読み込み完了後、バッファを解放し、ファイルディスクリプタを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_combinebackup/copy_file.c 内の各コピー処理関数（copy_file_clone、copy_file_by_range、copy_file_copyfile、copy_file_link 等）から、コピー完了後の検証ステップとして呼び出される。
  • **呼び出し先**: open, read, close（システムコール）、pg_malloc, pg_free（メモリ管理）、pg_checksum_update（チェックサム計算）、pg_fatal（エラー報告）。

関連データ構造

  • pg_checksum_context: チェックサム計算のアルゴリズムや現在の状態を保持する構造体。

失敗モードと境界条件

  • **ファイルオープン失敗**: open が失敗した場合、pg_fatal によりプロセスが終了する。
  • **読み込みエラー**: read 中にエラーが発生した場合、pg_fatal によりプロセスが終了する。
  • **チェックサム更新失敗**: pg_checksum_update が負の値を返した場合、pg_fatal によりプロセスが終了する。

保守担当者向けメモ

  • 本関数は同期的に動作し、ファイル全体を読み込むため、対象ファイルが大きい場合は I/O 負荷が発生する。
  • BLCKSZ は PostgreSQL のブロックサイズ定義に依存する。
  • エラー発生時は pg_fatal を使用して即座に終了するため、呼び出し元で個別のエラーハンドリングを行う必要はない。
