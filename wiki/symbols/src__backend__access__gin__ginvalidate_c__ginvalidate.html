ginvalidate

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginvalidate.c:30

目的

GIN（Generalized Inverted Index）演算子クラス（opclass）および演算子族（opfamily）の定義が、PostgreSQLのシステムカタログ上で正当であるかを検証する。

シグネチャまたは形状

bool ginvalidate(Oid opclassoid)

入力と出力

  • **入力**: opclassoid (検証対象の演算子クラスのOID)
  • **出力**: bool (検証に合格した場合は true、失敗した場合は false)

前提条件と不変条件

  • opclassoid に対応する演算子クラスがシステムカタログ pg_opclass に存在すること。
  • 演算子族内のサポート関数および演算子は、GINが要求する特定のシグネチャおよび戦略番号の制約に従うこと。

アルゴリズムまたは振る舞い

1. pg_opclass から演算子クラス情報を取得し、関連する演算子族を特定する。 2. pg_amproc および pg_amop から、当該演算子族に属する全サポート関数と演算子を検索する。 3. **サポート関数の検証**:

  • 左型と右型が一致しているかを確認する。
  • GIN_COMPARE_PROC 等の各サポート関数について、期待される引数・戻り値の型と個数（check_amproc_signature）を検証する。

    • 4. **演算子の検証**:

  • 戦略番号が 1 から 63 の範囲内であることを確認する。
  • GINは ORDER BY 演算子をサポートしないため、amoppurpose が AMOP_SEARCH であること、およびソート族が指定されていないことを確認する。
  • check_amop_signature を用いて演算子のシグネチャを検証する。

    • 5. 検証失敗時は INFO レベルでエラーメッセージを報告し、result を false に設定して処理を継続する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginutil.c において、amroutine->amvalidate に割り当てられている。
  • **呼び出し先**: SearchSysCache1, SearchSysCacheList1, check_amproc_signature, check_amop_signature, identify_opfamily_groups 等のユーティリティ関数。

関連データ構造

  • Form_pg_opclass, Form_pg_amproc, Form_pg_amop: システムカタログの行構造。
  • CatCList: システムカタログのキャッシュリスト。
  • OpFamilyOpFuncGroup: 演算子族のグループ化情報。

失敗モードと境界条件

  • 指定された opclassoid が存在しない場合、elog(ERROR) により処理が中断される。
  • サポート関数の型不一致や、不正な戦略番号、サポートされない ORDER BY 演算子が含まれる場合、result が false となり検証失敗として扱われる。

保守担当者向けメモ

  • サポート関数には必須とオプションが存在する。ループ内で continue されている関数はオプション扱いである。
