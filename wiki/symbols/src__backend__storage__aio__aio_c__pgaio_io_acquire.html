pgaio_io_acquire

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio.c:161

目的

非同期I/O（AIO）操作を実行するための PgAioHandle を取得します。利用可能なハンドルがない場合は、ハンドルが解放されるまで待機するブロッキング関数です。

シグネチャまたは形状

PgAioHandle *pgaio_io_acquire(struct ResourceOwnerData *resowner, PgAioReturn *ret)

入力と出力

  • **入力**:
  • resowner: I/O操作に関連付けられるリソースオーナー。
  • ret: 操作の結果や状態を格納するための PgAioReturn 構造体へのポインタ。
  • **出力**:
  • 成功時: 確保された PgAioHandle へのポインタ。
  • 失敗時: 戻り値として NULL を返すことはなく、ハンドルが確保できるまでループ内で待機します。

前提条件と不変条件

  • 呼び出し元は、I/O操作を開始する前に本関数を呼び出す必要があります。
  • 本関数は、バックエンドが利用可能なすべてのハンドルを使い果たしている場合、pgaio_io_wait_for_free() を呼び出して空きができるまで待機します。

アルゴリズムまたは振る舞い

1. pgaio_io_acquire_nb（非ブロッキング版）を呼び出し、ハンドル取得を試みます。 2. ハンドルが取得できた場合、そのハンドルを返します。 3. ハンドルが取得できなかった場合、pgaio_io_wait_for_free() を呼び出してI/O完了を待ち、再度ステップ1から試行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c: ReadBuffersCanStartIO() の後続処理として呼び出されます。
  • **呼び出し先**:
  • pgaio_io_acquire_nb: ハンドルの即時取得を試みる内部関数。
  • pgaio_io_wait_for_free: ハンドルが解放されるまで待機する内部関数。

関連データ構造

  • PgAioHandle: I/O操作のライフサイクルを管理するハンドル構造体。
  • ResourceOwnerData: リソースの所有権を管理する構造体。
  • PgAioReturn: I/O操作の戻り値や状態を保持する構造体。

失敗モードと境界条件

  • 本関数はループ構造により、ハンドルが枯渇している状況下でも成功するまで待機を継続します。
  • 割り込みやエラー発生時のクリーンアップについては、ResourceOwner を介した管理に依存します。

保守担当者向けメモ

  • 本関数はブロッキング動作を行うため、デッドロックを避ける設計が必要です。
  • 状態遷移を伴うため、呼び出し元は HOLD_INTERRUPTS() 等の割り込み制御に関するガイドラインを遵守してください。
