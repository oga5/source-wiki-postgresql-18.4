checkWellFormedRecursionWalker

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/parser/parse_cte.c:1026

目的

再帰的CTE（共通テーブル式）の構造がSQL標準に適合しているかを検証し、再帰参照が許可されていない場所でCTEが参照されていないかをチェックする。

シグネチャまたは形状

static bool checkWellFormedRecursionWalker(Node *node, CteState *cstate)

入力と出力

  • **入力**:
  • node: 走査対象の構文木ノード。
  • cstate: 再帰検証の状態を保持する CteState 構造体。
  • **出力**:
  • bool: raw_expression_tree_walker の規約に従い、走査を継続する場合は false を返す。

前提条件と不変条件

  • cstate->context は現在の再帰コンテキスト（RECURSION_OK 等）を保持しており、ノードの走査に応じて適切に更新・復元される必要がある。
  • cstate->innerwiths は、ネストされた WITH 句による名前の隠蔽を追跡するために使用される。

アルゴリズムまたは振る舞い

1. **RangeVar (テーブル参照)**:

  • innerwiths を確認し、内側の WITH 句で隠蔽されている場合は無視する。
  • 現在のCTE名と一致する場合、cstate->context が RECURSION_OK でなければエラーを報告する。
  • 再帰参照が2回以上出現した場合はエラーを報告する。

    • 2. **SelectStmt**:

  • WITH 句が存在する場合、再帰の有無に応じて innerwiths を更新し、CTE内のクエリを再帰的に走査する。

    • 3. **JoinExpr**:

  • 結合タイプに応じてコンテキストを RECURSION_OUTERJOIN に変更し、再帰参照の可否を制御する。

    • 4. **SubLink**:

  • サブクエリは独立しているため、コンテキストを RECURSION_SUBLINK に強制変更して走査する。

    • 5. **その他**:

  • raw_expression_tree_walker を呼び出し、ツリーの残りの部分を走査する。

呼び出し元と呼び出し先

  • **呼び出し元**: checkWellFormedRecursion および関数自身による再帰呼び出し。
  • **呼び出し先**: checkWellFormedSelectStmt、raw_expression_tree_walker、および自身。

関連データ構造

  • CteState: 再帰検証の全体状態（現在のコンテキスト、再帰参照回数、内側のCTEリストなど）を保持。
  • RecursionContext: 再帰参照の可否を定義する列挙型。

失敗モードと境界条件

  • **ERRCODE_INVALID_RECURSION**:
  • 再帰参照が許可されていないコンテキスト（例: OUTER JOIN の右側や SubLink 内）でCTEが参照された場合。
  • 再帰的CTEが2回以上参照された場合。
  • **エラー**: 認識できない JoinExpr 型が渡された場合に elog で停止する。

保守担当者向けメモ

  • JOIN や SubLink を通過する際に cstate->context を一時的に変更するロジックが重要です。状態の復元（save_context）が正しく行われているか注意してください。
  • innerwiths の操作は、ネストされた WITH 句による名前の可視性制御に不可欠です。
