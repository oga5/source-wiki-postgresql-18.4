write_stderr

定義

  • 種別: function
  • 定義箇所: src/bin/pg_ctl/pg_ctl.c:201

目的

標準エラー出力または Windows イベントログへ、フォーマット済みのエラーメッセージを出力する。

シグネチャまたは形状

static void write_stderr(const char *fmt, ...);

入力と出力

  • 入力: printf 形式のフォーマット文字列 fmt および可変長引数。
  • 出力: なし（戻り値は void）。副作用として、コンソールまたはイベントログへ文字列を書き出す。

前提条件と不変条件

  • va_list を使用するため、呼び出し元は標準的な可変長引数の規約に従う必要がある。
  • Windows 環境においてサービスとして実行されている場合、メッセージは write_eventlog を介してイベントログへ送られる。

アルゴリズムまたは振る舞い

1. va_start により可変長引数リストを初期化する。 2. プラットフォーム判定を行う：

  • **Unix 系**: vfprintf を使用して stderr へ直接出力する。
  • **Windows**: pgwin32_is_service() を確認する。
  • サービスとして実行中の場合: vsnprintf でバッファ（最大2048バイト）に書き出し、write_eventlog を呼び出す。
  • コンソール実行中の場合: vfprintf を使用して stderr へ出力する。

    • 3. va_end を呼び出し、引数リストを終了する。

呼び出し元と呼び出し先

  • **呼び出し先**: vfprintf (標準ライブラリ), vsnprintf (標準ライブラリ), pgwin32_is_service (内部), write_eventlog (内部)。
  • **呼び出し元**: ツールによる解析では、src/backend/bootstrap/bootstrap.c、src/backend/main/main.c、src/backend/port/win32/crashdump.c、src/backend/port/win32/signal.c 等からの参照が確認されている。これらは直接的な呼び出しの可能性が高い。

関連データ構造

  • va_list: 可変長引数を処理するための標準型。

失敗モードと境界条件

  • Windows 環境でサービスとして実行中にメッセージが 2048 バイトを超える場合、vsnprintf の切り詰めが発生する可能性がある。
  • write_eventlog が失敗した場合の挙動については、本ソース内では明示されていない。

保守担当者向けメモ

  • Windows サービス実行時のバッファサイズ（2048バイト）は固定値であるため、長大なログを出力する可能性がある場合は注意が必要。
  • Unix 環境では stderr への書き込みのみに依存しているため、実行環境の標準エラー出力がリダイレクトされているかどうかに依存する。
