build_path_tlist

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/createplan.c:824

目的

Path ノードの pathtarget に基づき、実行計画で使用されるターゲットリスト（List of TargetEntry）を構築します。

シグネチャまたは形状

List *build_path_tlist(PlannerInfo *root, Path *path)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • path: ターゲットリストの元となる Path ノード。
  • **出力**:
  • List *: 構築された TargetEntry のリスト。

前提条件と不変条件

  • path->pathtarget が有効であること。
  • パラメータ化されたパスの場合、path->param_info が設定されていること。

アルゴリズムまたは振る舞い

1. path->pathtarget->exprs を反復処理します。 2. パスがパラメータ化されている場合（path->param_info が存在）、replace_nestloop_params を呼び出し、式内の横方向参照（lateral references）を Param ノードに置換します。 3. 各式に対して makeTargetEntry を呼び出し、TargetEntry を生成します。 4. path->pathtarget->sortgrouprefs が存在する場合、対応する ressortgroupref を TargetEntry に割り当てます。 5. 生成された TargetEntry をリストに追加し、最終的なリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: createplan.c 内の複数の箇所（create_plan 関連のノード生成処理など）から呼び出されています。
  • **呼び出し先**:
  • replace_nestloop_params（条件付き）
  • makeTargetEntry
  • lappend

関連データ構造

  • Path: 入力となるパス構造体。
  • TargetEntry: 出力リストの要素。
  • PlannerInfo: プランナのコンテキスト。

失敗モードと境界条件

  • path->pathtarget が空の場合、空のリストを返します。
  • sortgrouprefs が NULL の場合、ressortgroupref への割り当てはスキップされます。

保守担当者向けメモ

  • 本関数は TargetEntry ノードの再作成を行わず、既存の式を再利用または変換してリストを構築します。
  • パラメータ化されたパスの処理において、式内の参照解決が正しく行われているかを確認してください。
