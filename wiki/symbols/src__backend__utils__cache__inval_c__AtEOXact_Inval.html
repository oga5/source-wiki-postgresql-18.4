AtEOXact_Inval

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/cache/inval.c:1198

目的

トランザクション終了時（コミットまたはアボート）に、蓄積されたキャッシュ無効化メッセージを処理し、トランザクション状態をクリーンアップする。

シグネチャまたは形状

void AtEOXact_Inval(bool isCommit)

入力と出力

  • **入力**: isCommit (bool) - トランザクションがコミットされる場合は true、アボートされる場合は false。
  • **出力**: なし（副作用として共有メモリへのメッセージ送出またはローカルキャッシュのフラッシュを行う）。

前提条件と不変条件

  • transInvalInfo が指す構造体がトップレベル（my_level == 1 かつ parent == NULL）であること。
  • inplaceInvalInfo は本関数の実行時に NULL にリセットされる。

アルゴリズムまたは振る舞い

1. transInvalInfo が NULL の場合、即座に終了する。 2. isCommit が true の場合:

  • RelcacheInitFileInval が設定されていれば、RelationCacheInitFilePreInvalidate() を実行。
  • CurrentCmdInvalidMsgs を PriorCmdInvalidMsgs に統合。
  • ProcessInvalidationMessagesMulti を呼び出し、SendSharedInvalidMessages を介して共有メモリの SI メッセージキューへ書き出す。
  • RelcacheInitFileInval が設定されていれば、RelationCacheInitFilePostInvalidate() を実行。

    • 3. isCommit が false の場合:

  • ProcessInvalidationMessages を呼び出し、LocalExecuteInvalidationMessage を用いてローカルキャッシュのみをフラッシュする。

    • 4. 最後に transInvalInfo を NULL に設定し、トランザクションの無効化情報を破棄する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c (コミットおよびアボート処理)
  • src/backend/access/transam/twophase.c
  • src/backend/utils/cache/inval.c (内部的なエラーリカバリ等)
  • **呼び出し先**:
  • RelationCacheInitFilePreInvalidate / RelationCacheInitFilePostInvalidate
  • AppendInvalidationMessages
  • ProcessInvalidationMessagesMulti / ProcessInvalidationMessages
  • SendSharedInvalidMessages / LocalExecuteInvalidationMessage

関連データ構造

  • TransInvalidationInfo: トランザクションごとの無効化メッセージを保持する構造体。
  • InvalidationMsgsGroup: メッセージのグループ化に使用。

失敗モードと境界条件

  • transInvalInfo が NULL の場合は何もしない（正常な早期終了）。
  • トランザクションレベルがトップレベルでない場合、Assert により停止する。

保守担当者向けメモ

  • 本関数はトランザクション終了時のキャッシュ整合性の最終防衛線である。
  • isCommit=true の場合のみ共有メモリへの伝播が行われる点に注意すること。
  • AtInplace_Inval との組み合わせでトランザクション全体の無効化処理が完結する設計となっている。
