BitmapOr

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/plannodes.h:456

目的

複数のビットマップスキャン結果を論理和（OR）で結合するための実行計画ノードです。

シグネチャまたは形状

typedef struct BitmapOr
{
	Plan		plan;
	bool		isshared;
	List	   *bitmapplans;
} BitmapOr;

入力と出力

  • **入力**: bitmapplans リストに含まれる複数の子プランノード。
  • **出力**: 子プランから生成されたビットマップの論理和結果。

前提条件と不変条件

  • plan メンバを構造体の先頭に保持し、Plan 構造体としてキャスト可能です。
  • BitmapOr ノードは、通常の Plan ノードが持つ lefttree および righttree を使用しません。

アルゴリズムまたは振る舞い

  • エグゼキュータは ExecInitBitmapOr を介して初期化を行い、bitmapplans に含まれるすべてのサブスキャンを開始します。
  • ExecProcNode 呼び出し規約には対応しておらず、専用の実行ロジック（nodeBitmapOr.c 内の関数群）によって処理されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execProcnode.c: ExecInitBitmapOr を呼び出し。
  • src/backend/commands/explain.c: 実行計画の表示処理で参照。
  • **呼び出し先**:
  • src/backend/executor/nodeBitmapOr.c: ExecInitBitmapOr, ExecEndBitmapOr, ExecReScanBitmapOr 等の実装。

関連データ構造

  • Plan: 基底構造体。
  • List: 子プランノードを保持するリスト。

失敗モードと境界条件

  • ExecProcNode 経由で呼び出された場合、エラー（elog(ERROR, ...)）が発生します。

保守担当者向けメモ

  • 本ノードは BitmapAnd と同様に、ビットマップインデックススキャンを統合する役割を持ちます。
  • 実行時の状態管理については src/include/nodes/execnodes.h で定義される対応する実行時ノードを参照してください。
