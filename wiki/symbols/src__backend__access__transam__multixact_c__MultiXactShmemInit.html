MultiXactShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/multixact.c:2124

目的

MultiXact 管理に必要な共有メモリ領域の初期化および SLRU（Simple Least Recently Used）キャッシュのセットアップを行う。

シグネチャまたは形状

void MultiXactShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（共有メモリ上のグローバル変数および SLRU 制御構造体を初期化する副作用を持つ）

前提条件と不変条件

  • Postmaster プロセスまたはバックエンドプロセスが共有メモリを確保した後に呼び出される必要がある。
  • MultiXactOffsetCtl および MultiXactMemberCtl が有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. MultiXactOffsetCtl および MultiXactMemberCtl の PagePrecedes コールバック関数を設定する。 2. SimpleLruInit を呼び出し、multixact_offset および multixact_member 用の SLRU バッファとディレクトリを初期化する。 3. ShmemInitStruct を使用して MultiXactState（共有状態構造体）を確保する。 4. Postmaster 起動時（!IsUnderPostmaster）であれば、確保した MultiXactState をゼロクリアする。 5. OldestMemberMXactId および OldestVisibleMXactId のポインタを、確保した共有メモリ内のオフセットに基づいて設定する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c 内の CreateSharedMemoryAndSemaphores から呼び出される。
  • 呼び出し先: SimpleLruInit, ShmemInitStruct, SlruPagePrecedesUnitTests, MemSet 等。

関連データ構造

  • MultiXactOffsetCtl, MultiXactMemberCtl: SLRU 管理用構造体。
  • MultiXactState: 共有メモリ上に配置される MultiXact の状態管理構造体。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、ShmemInitStruct がエラーを発生させる。
  • IsUnderPostmaster が真の場合に found フラグが立っていない場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は PostgreSQL の起動シーケンスにおいて重要な役割を果たすため、変更時は ipci.c での初期化順序に注意すること。
  • MultiXactMemberCtl は SimpleLruTruncate を呼び出さない設計となっている点に留意。
