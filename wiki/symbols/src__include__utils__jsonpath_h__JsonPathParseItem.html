JsonPathParseItem

定義

  • 種別: struct
  • 定義箇所: src/include/utils/jsonpath.h:215

目的

SQL/JSON Path 式を解析する過程で生成される、中間表現（ASTノード）を保持するための構造体です。

シグネチャまたは形状

typedef struct JsonPathParseItem JsonPathParseItem;

※構造体の内部フィールドは、next（パス内の次ノード）、left/right（演算子の左右オペランド）、arg（引数）、from/to（範囲指定）、expr（式）などのポインタを含みます。

入力と出力

  • **入力**: SQL 文字列としての JSON Path 式。
  • **出力**: parsejsonpath 関数によって生成される JsonPathParseResult の構成要素として、バイナリ形式の JsonPath にシリアライズされる前の中間表現を提供します。

前提条件と不変条件

  • 本構造体は解析時の中間表現であり、実行時のバイナリ形式（JsonPathItem）とは異なります。
  • 構造体内のポインタ（next, left, right 等）は、解析ツリーの階層構造を形成します。

アルゴリズムまたは振る舞い

1. SQL 文字列の解析時に、各パス要素（演算子、メソッド、リテラル等）に対応する JsonPathParseItem が生成されます。 2. 解析ツリーとして連結され、最終的にバイナリ形式の JsonPath 構造へシリアライズされます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonpath.c 内の解析ルーチンが本構造体を操作します。
  • **呼び出し先**: 構造体内の各ポインタフィールドを通じて、他の JsonPathParseItem インスタンスを参照します。

関連データ構造

  • JsonPathParseResult: 解析結果全体を保持するコンテナ。
  • JsonPathItem: 実行時に使用されるバイナリ形式のパス走査用構造体。

失敗モードと境界条件

  • 解析対象の JSON Path 式が文法的に不正な場合、解析プロセスでエラーが発生します。
  • 複雑なネスト構造を持つパスの場合、メモリ割り当て（palloc）の制限を受ける可能性があります。

保守担当者向けメモ

  • 本構造体は解析用の中間表現です。バイナリ形式の JsonPathItem と混同しないよう注意してください。
  • 構造体のフィールド定義は src/include/utils/jsonpath.h を参照してください。
