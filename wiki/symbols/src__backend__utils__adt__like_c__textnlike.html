textnlike

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like.c:304

目的

SQLの NOT LIKE 演算子を実装し、指定された文字列がパターンに一致しない場合に真（true）を返します。

シグネチャまたは形状

Datum textnlike(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの text 型引数（対象文字列 str とパターン pat）を受け取ります。
  • **出力**: Datum 型として、一致しない場合に true、一致する場合に false を返します。

前提条件と不変条件

  • PG_GETARG_TEXT_PP を使用して引数を取得するため、入力は text 型である必要があります。
  • データベースの照合順序（Collation）が PG_GET_COLLATION() を通じて適切に取得できる必要があります。

アルゴリズムまたは振る舞い

1. VARDATA_ANY および VARSIZE_ANY_EXHDR を使用して、入力文字列とパターンのデータポインタおよび長さを取得します。 2. GenericMatchText 関数を呼び出し、パターンマッチングを実行します。 3. GenericMatchText の戻り値が LIKE_TRUE（一致）ではないことを確認します。 4. 一致しない場合（LIKE_TRUE 以外）に true を返し、一致した場合には false を返します。 5. PG_RETURN_BOOL を介して結果を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジンから直接呼び出されます（ソースコード内での明示的な呼び出し元は確認されていません）。
  • **呼び出し先**: GenericMatchText を呼び出します。

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • GenericMatchText の内部ロジックに依存します。
  • 入力引数が NULL の場合、PG_GETARG_TEXT_PP の仕様に従い、呼び出し元で適切に処理される必要があります。

保守担当者向けメモ

  • 本関数は NOT LIKE の否定論理を担うラッパーです。マッチングの詳細は GenericMatchText に委譲されているため、アルゴリズムの変更は like_match.c 側の修正を検討してください。
