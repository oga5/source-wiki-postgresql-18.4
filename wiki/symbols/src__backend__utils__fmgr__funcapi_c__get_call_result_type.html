get_call_result_type

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/funcapi.c:275

目的

関数呼び出し情報（fcinfo）から、その関数の戻り値の型（resultTypeId）およびタプル記述子（resultTupleDesc）を動的に解決し、戻り値の型クラス（TypeFuncClass）を返します。

シグネチャまたは形状

TypeFuncClass get_call_result_type(FunctionCallInfo fcinfo, Oid *resultTypeId, TupleDesc *resultTupleDesc);

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報。flinfo（関数情報）および resultinfo（SRF用情報）を含みます。
  • resultTypeId: 戻り値の型OIDを格納するためのポインタ（NULL可）。
  • resultTupleDesc: 戻り値のタプル記述子を格納するためのポインタ（NULL可）。
  • **出力**:
  • 戻り値: TypeFuncClass（TYPEFUNC_COMPOSITE, TYPEFUNC_SCALAR, TYPEFUNC_RECORD 等）。

前提条件と不変条件

  • fcinfo が有効な関数呼び出しコンテキストを指している必要があります。
  • 内部で internal_get_result_type を呼び出すため、同関数の制約（pg_proc カタログや式ツリーの整合性など）に依存します。

アルゴリズムまたは振る舞い

1. fcinfo->flinfo->fn_oid（関数OID）を取得します。 2. fcinfo->flinfo->fn_expr（式ツリー）を取得します。 3. fcinfo->resultinfo を ReturnSetInfo 型にキャストして取得します。 4. 上記の情報を引数として internal_get_result_type を呼び出し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • contrib/hstore/hstore_op.c
  • contrib/pageinspect/brinfuncs.c
  • contrib/pageinspect/btreefuncs.c
  • contrib/pageinspect/ginfuncs.c
  • **呼び出し先**:
  • internal_get_result_type（同一ファイル内）

関連データ構造

  • FunctionCallInfo: 関数呼び出しの実行時情報。
  • ReturnSetInfo: 集合を返す関数（SRF）の実行時情報。
  • TypeFuncClass: 戻り値の型分類を示す列挙型。
  • TupleDesc: 複合型の構造定義。

失敗モードと境界条件

  • TYPEFUNC_RECORD が返された場合、戻り値の構造がその時点では確定していないことを示します。呼び出し側で適切な処理が必要です。
  • 呼び出し元で resultTupleDesc が NULL の場合、記述子の取得はスキップされます。

保守担当者向けメモ

  • 本関数はコストが高いため、SRF内で使用する場合は初回呼び出し時に結果を FuncCallContext 等にキャッシュし、以降の呼び出しでは再利用することを推奨します。
  • 多相型（anyelement 等）の解決は internal_get_result_type 経由で行われます。
