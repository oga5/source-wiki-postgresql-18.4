setup_parse_fixed_parameters

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_param.c:67

目的

クエリ解析において、型が事前に確定しているパラメータ（$1, $2 等）を処理するためのフックを ParseState に設定します。

シグネチャまたは形状

void setup_parse_fixed_parameters(ParseState *pstate, const Oid *paramTypes, int numParams)

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する ParseState 構造体へのポインタ。
  • paramTypes: パラメータの型（Oid）を格納した配列。
  • numParams: パラメータの総数。
  • **出力**: なし（副作用として pstate を更新）。

前提条件と不変条件

  • pstate は有効なメモリ領域を指している必要があります。
  • paramTypes は少なくとも numParams 分の要素を持つ配列である必要があります。

アルゴリズムまたは振る舞い

1. FixedParamState 構造体用のメモリを割り当てます。 2. 引数で受け取った paramTypes と numParams を FixedParamState に格納します。 3. pstate->p_ref_hook_state に作成した FixedParamState を設定します。 4. pstate->p_paramref_hook に fixed_paramref_hook 関数を登録します。 5. 固定パラメータ処理のため、p_coerce_param_hook は設定しません。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/analyze.c:118 にて呼び出されています。
  • **呼び出し先**: palloc（メモリ割り当て）。fixed_paramref_hook をフックとして登録します。

関連データ構造

  • FixedParamState: 固定パラメータの型情報と個数を保持する内部構造体。
  • ParseState: クエリ解析の文脈を保持する構造体。

失敗モードと境界条件

  • palloc がメモリ不足により失敗した場合、PostgreSQL の標準的なエラー処理メカニズムによりアボートします。

保守担当者向けメモ

  • 本関数は、解析対象のクエリに含まれるパラメータの型が外部から提供される場合にのみ使用されます。
  • p_coerce_param_hook を使用しない設計であるため、型変換の動的な解決が必要な場合は setup_parse_variable_parameters を検討してください。
