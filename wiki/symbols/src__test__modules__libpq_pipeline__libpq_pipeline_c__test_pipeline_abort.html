test_pipeline_abort

定義

  • 種別: function
  • 定義箇所: src/test/modules/libpq_pipeline/libpq_pipeline.c:704

目的

libpq のパイプラインモードにおいて、クエリ実行中にエラーが発生した際の PGRES_PIPELINE_ABORTED 状態への遷移、および PQpipelineSync による正常状態への復旧プロセスを検証する。

シグネチャまたは形状

static void test_pipeline_abort(PGconn *conn)

入力と出力

  • 入力: PGconn *conn（確立済みのデータベース接続）
  • 出力: なし（エラー発生時は pg_fatal を呼び出しプログラムを終了する）

前提条件と不変条件

  • 呼び出し時点で conn は有効であること。
  • テスト実行前にテーブルの初期化（DROP/CREATE）が行われる。
  • パイプラインモードへの移行（PQenterPipelineMode）が成功していること。

アルゴリズムまたは振る舞い

1. **セットアップ**: データベーステーブルを準備し、パイプラインモードに移行する。 2. **クエリ送信**: 正常なクエリ、意図的にエラーを発生させるクエリ、その後の正常なクエリを順次送信し、PQpipelineSync で区切る。 3. **結果検証**: PQgetResult をループし、以下の順序でステータスを検証する。

  • 正常クエリの PGRES_COMMAND_OK。
  • エラークエリの PGRES_FATAL_ERROR。
  • 後続クエリの PGRES_PIPELINE_ABORTED。
  • PQpipelineSync による PGRES_PIPELINE_SYNC の受信と、PQpipelineStatus が PQ_PIPELINE_ABORTED から復旧することの確認。

    • 4. **最終確認**: 2つ目のパイプラインの正常終了を確認し、最後に不正なクエリ（セミコロン区切り）を送信してエラーハンドリングの挙動を検証する。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/libpq_pipeline/libpq_pipeline.c 内の main 関数（またはテストスイート実行部）。
  • 呼び出し先: PQexec, PQenterPipelineMode, PQsendQueryParams, PQpipelineSync, PQgetResult, PQpipelineStatus, PQclear, pg_fatal 等の libpq API。

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • PGresult: クエリ実行結果を保持する構造体。

失敗モードと境界条件

  • PQenterPipelineMode や PQsendQueryParams 等の API が失敗した場合、pg_fatal によりテストが中断される。
  • PQgetResult が期待するステータス（PGRES_PIPELINE_ABORTED や PGRES_PIPELINE_SYNC）を返さない場合、テストは失敗とみなされる。
  • パイプライン同期後に PQpipelineStatus が期待通りに更新されない場合、エラーとして報告される。

保守担当者向けメモ

  • 本関数は libpq のパイプラインエラー復旧ロジックの網羅的なテストケースとして機能する。
  • PQpipelineSync が PGRES_PIPELINE_SYNC を返すことで、クライアントがエラー状態から脱却し、通常のパイプライン処理を再開できることを確認している点に注意すること。
