FullTransactionIdAdvance

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/transam.h:127

目的

64ビットの FullTransactionId をインクリメントし、予約済みトランザクションID（FirstNormalTransactionId 未満）をスキップして、有効な通常のトランザクションIDへ進める。

シグネチャまたは形状

static inline void FullTransactionIdAdvance(FullTransactionId *dest)

入力と出力

  • **入力**: dest - 更新対象の FullTransactionId 構造体へのポインタ。
  • **出力**: なし（ポインタ経由で dest の値を直接更新）。

前提条件と不変条件

  • dest が指すメモリ領域は有効であること。
  • FullTransactionId の下位32ビットが予約済みID（FirstNormalTransactionId 未満）である場合、それらをスキップして通常のIDに到達するまでインクリメントを繰り返す。

アルゴリズムまたは振る舞い

1. dest->value を 1 インクリメントする。 2. FullTransactionIdPrecedes を使用し、更新後の値が FirstNormalFullTransactionId より前であれば処理を終了する。 3. そうでない場合、XidFromFullTransactionId で抽出した下位32ビットが FirstNormalTransactionId に達するまで、dest->value を繰り返しインクリメントする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/varsup.c:214
  • src/backend/storage/ipc/procarray.c:2474
  • **呼び出し先**:
  • FullTransactionIdPrecedes
  • XidFromFullTransactionId

関連データ構造

  • FullTransactionId: 64ビットのトランザクション識別子。
  • FirstNormalTransactionId: 通常のトランザクションIDの開始値。
  • FirstNormalFullTransactionId: 通常の FullTransactionId の開始値。

失敗モードと境界条件

  • 予約済みID（InvalidTransactionId 等）をスキップするロジックにより、常に有効な通常のトランザクションIDへ遷移する。
  • 64ビット値のオーバーフローについては、PostgreSQLのトランザクションID管理の仕様に依存する。

保守担当者向けメモ

  • 本関数はインライン定義されており、頻繁に呼び出されるため、パフォーマンスへの影響を考慮すること。
  • FullTransactionId の構造変更や、予約済みIDの範囲定義に変更があった場合は、本関数のスキップロジックも整合性を確認する必要がある。
