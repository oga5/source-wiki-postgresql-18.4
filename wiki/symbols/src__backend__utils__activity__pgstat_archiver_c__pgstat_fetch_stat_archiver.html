pgstat_fetch_stat_archiver

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_archiver.c:57

目的

WAL アーカイバの統計情報を取得し、呼び出し元へ提供するためのスナップショットを生成します。

シグネチャまたは形状

PgStat_ArchiverStats *pgstat_fetch_stat_archiver(void)

入力と出力

  • 入力: なし
  • 出力: PgStat_ArchiverStats 構造体へのポインタ。

前提条件と不変条件

  • 統計システムが初期化されている必要があります。
  • 呼び出しにより、内部的な統計スナップショットが最新の状態に更新されます。

アルゴリズムまたは振る舞い

1. pgstat_snapshot_fixed(PGSTAT_KIND_ARCHIVER) を呼び出し、アーカイバ統計の最新スナップショットを生成します。この際、必要に応じてリセットオフセットを考慮した差分計算が行われます。 2. 内部的なローカル統計構造体 pgStatLocal.snapshot.archiver へのポインタを返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/pgstatfuncs.c:2074 (SQL 関数から統計情報を取得する際に使用)
  • 呼び出し先: pgstat_snapshot_fixed

関連データ構造

  • PgStat_ArchiverStats: アーカイバの統計情報（成功・失敗カウンタ、最終実行時刻など）を保持する構造体。
  • pgStatLocal: 統計システムが保持するローカルスナップショットを格納するグローバル変数。

失敗モードと境界条件

  • 統計システムが未初期化の場合や、共有メモリへのアクセスに失敗した場合は、基盤となる pgstat_snapshot_fixed の挙動に依存します。

保守担当者向けメモ

  • 本関数は統計の「記録」ではなく「公開」を担います。
  • 統計リセット処理は、累積値の直接的なゼロクリアではなく、reset_offset を用いたデルタ計算によって実現されているため、本関数が返す値はリセット後の差分として算出されます。
