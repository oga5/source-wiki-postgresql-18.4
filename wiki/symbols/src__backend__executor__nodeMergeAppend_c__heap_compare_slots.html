heap_compare_slots

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMergeAppend.c:287

目的

MergeAppend ノードにおいて、複数のソート済み入力ストリームから取得したタプルを比較し、binaryheap 内での順序を決定するために使用されます。

シグネチャまたは形状

int32 heap_compare_slots(Datum a, Datum b, void *arg)

入力と出力

  • **入力**:
  • a, b: 比較対象のタプルを指す SlotNumber（Datum 型として渡される）。
  • arg: MergeAppendState 構造体へのポインタ。
  • **出力**:
  • 比較結果を示す int32 値。a が b より小さい場合は負、等しい場合は 0、大きい場合は正の値を返します。

前提条件と不変条件

  • arg は MergeAppendState 型にキャスト可能であること。
  • a および b が指すスロット（ms_slots）は、TupIsNull でないこと（Assert により検証）。
  • node->ms_sortkeys に基づき、すべてのソートキーについて順次比較が行われること。

アルゴリズムまたは振る舞い

1. arg から MergeAppendState を取得し、a と b を SlotNumber に変換して対応する TupleTableSlot を特定します。 2. 設定されたソートキーの数（ms_nkeys）分だけループし、各属性に対して ApplySortComparator を呼び出します。 3. 比較結果が 0 でない場合、その結果を反転（INVERT_COMPARE_RESULT）して即座に返します。 4. すべてのキーで比較結果が 0 であれば、最終的に 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: binaryheap の操作関数（binaryheap_allocate 等）から間接的に呼び出されます。src/backend/executor/nodeMergeAppend.c および src/backend/executor/nodeGatherMerge.c での参照が確認されています。
  • **呼び出し先**: slot_getattr および ApplySortComparator を呼び出します。

関連データ構造

  • MergeAppendState: 実行状態とソートキーを保持。
  • TupleTableSlot: 比較対象のタプルを保持。
  • SortSupport: 比較ロジックをカプセル化した構造体。

失敗モードと境界条件

  • s1 または s2 が空（TupIsNull）の場合、Assert により異常終了します。
  • ソートキーが空の場合、常に 0 を返します。

保守担当者向けメモ

  • abbreviated key（省略キー）による最適化は、タプルがヒープに順次投入される特性上、変換コストがメリットを上回るため意図的に無効化されています。
  • 比較結果の反転は、ヒープが最小値を取り出す構造であるのに対し、ソート順の維持に必要な順序を調整するために行われています。
