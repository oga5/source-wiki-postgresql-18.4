pg_ascii_dsplen

定義

  • 種別: function
  • 定義箇所: src/common/wchar.c:93

目的

ASCIIエンコーディングにおける単一文字の表示幅（カラム数）を判定します。

シグネチャまたは形状

static int pg_ascii_dsplen(const unsigned char *s)

入力と出力

  • **入力**: s - 評価対象の文字を指す unsigned char 型のポインタ。
  • **出力**:
  • 0: 文字列終端（\0）の場合。
  • -1: 制御文字（0x20 未満または 0x7f）の場合。
  • 1: 上記以外（印字可能なASCII文字）の場合。

前提条件と不変条件

  • 入力ポインタ s が指す先は、有効なメモリ領域である必要があります。
  • ASCIIエンコーディングの仕様に基づき、各文字は1バイトとして扱われます。

アルゴリズムまたは振る舞い

1. *s が \0 であれば 0 を返します。 2. *s が 0x20（スペース）未満、または 0x7f（DEL）であれば、表示不能な制御文字とみなし -1 を返します。 3. それ以外の場合は、標準的なASCII文字として表示幅 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/wchar.c 内の複数箇所（pg_ascii_dsplen を直接呼び出すコードや、pg_wchar_table を介したディスパッチ）。
  • PG_SQL_ASCII エンコーディング定義の関数テーブル（pg_wchar_table）に登録されています。
  • **呼び出し先**:
  • 本関数は他の関数を呼び出しません。

関連データ構造

  • pg_wchar_table: エンコーディングごとの操作関数を保持するグローバルテーブル。本関数はこのテーブルの dsplen フィールドに格納されます。

失敗モードと境界条件

  • s が NULL の場合、未定義の動作（セグメンテーション違反）を引き起こす可能性があります。
  • 制御文字の判定は 0x20 未満および 0x7f に限定されており、拡張ASCII文字（0x80 以上）は考慮されていません。

保守担当者向けメモ

  • 本関数は PG_SQL_ASCII エンコーディング専用です。他のマルチバイトエンコーディングには適用しないでください。
  • 表示幅の計算ロジックを変更する場合は、pg_wchar_table を介して利用している他のモジュールへの影響を確認してください。
