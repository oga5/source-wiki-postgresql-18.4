set_dump_section

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_utils.c:41

目的

コマンドライン引数で指定されたセクション名（pre-data, data, post-data）を解析し、対応するビットマスクを dumpSections 変数に反映させる。

シグネチャまたは形状

void set_dump_section(const char *arg, int *dumpSections)

入力と出力

  • arg (入力): 解析対象のセクション名文字列。
  • dumpSections (入出力): セクション状態を保持するビットマスクへのポインタ。

前提条件と不変条件

  • dumpSections が指す値が DUMP_UNSECTIONED である場合、最初の呼び出しとみなされ、ビットマスクは 0 に初期化される。
  • 引数 arg は pre-data, data, post-data のいずれかである必要がある。

アルゴリズムまたは振る舞い

1. *dumpSections が DUMP_UNSECTIONED ならば、値を 0 にリセットする。 2. arg を文字列比較し、一致するセクションに応じてビットマスクを更新する。

  • pre-data: DUMP_PRE_DATA をセット。
  • data: DUMP_DATA をセット。
  • post-data: DUMP_POST_DATA をセット。

    • 3. 一致しない場合はエラーメッセージを出力し、exit_nicely(1) を呼び出してプロセスを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c
  • src/bin/pg_dump/pg_restore.c
  • **呼び出し先**:
  • pg_log_error / pg_log_error_hint (エラー報告)
  • exit_nicely (エラー時の終了処理)

関連データ構造

  • dumpSections: セクションの選択状態を管理するビットマスク（pg_backup_utils.h で定義）。

失敗モードと境界条件

  • 未知のセクション名が渡された場合、pg_log_error を経由して exit_nicely(1) が実行され、プログラムが異常終了する。

保守担当者向けメモ

  • 本関数は pg_dump と pg_restore の両方から利用される共通ユーティリティである。
  • 新しいセクションタイプを追加する場合は、本関数の条件分岐と pg_backup_utils.h のビット定義の両方を更新する必要がある。
