TempNamespaceStatus

定義

  • 種別: enum
  • 定義箇所: src/include/catalog/namespace.h:45

目的

一時テーブル用名前空間の状態（使用状況）を識別するために使用されます。

シグネチャまたは形状

typedef enum TempNamespaceStatus
{
    TEMP_NAMESPACE_NOT_TEMP,    /* 非一時名前空間、または存在しない */
    TEMP_NAMESPACE_IDLE,        /* 存在し、アクティブなセッションに属していない */
    TEMP_NAMESPACE_IN_USE,      /* アクティブなセッションに属している */
} TempNamespaceStatus;

入力と出力

  • 入力: checkTempNamespaceStatus(Oid namespaceId) 関数において、対象の名前空間OIDが入力として渡されます。
  • 出力: 上記の列挙値のいずれかを返します。

前提条件と不変条件

  • 一時名前空間の管理ロジックにおいて、セッションの生存期間やアクティブ状態と連動して状態が遷移します。

アルゴリズムまたは振る舞い

  • TEMP_NAMESPACE_NOT_TEMP: 対象が一時テーブル用ではない、あるいは存在しない場合に返されます。
  • TEMP_NAMESPACE_IDLE: 一時名前空間は存在するが、現在どのセッションにも割り当てられていない状態を示します。
  • TEMP_NAMESPACE_IN_USE: 現在アクティブなセッションによって使用中であることを示します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/namespace.c:3731 での参照が確認されています。
  • 呼び出し先: 本シンボル自体は列挙型であり、関数ではありません。checkTempNamespaceStatus 関数がこの型を戻り値として使用します。

関連データ構造

  • Oid: 名前空間の識別子として使用されます。

失敗モードと境界条件

  • 存在しないOIDが渡された場合、TEMP_NAMESPACE_NOT_TEMP が返されます。

保守担当者向けメモ

  • 本列挙型は、src/backend/catalog/namespace.c における一時名前空間のクリーンアップやセッション管理のロジックと密接に関係しています。状態遷移の整合性は namespace.c 内の管理関数に依存します。
