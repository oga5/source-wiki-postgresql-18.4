RelationCacheInitFileRemoveInDir

定義

  • 種別: static void 関数
  • 定義箇所: src/backend/utils/cache/relcache.c:6932

目的

指定されたテーブルスペースディレクトリ内にある各データベースディレクトリを走査し、リレーションキャッシュの初期化ファイル（pg_internal.init）を削除する。

シグネチャまたは形状

static void RelationCacheInitFileRemoveInDir(const char *tblspcpath)

入力と出力

  • 入力: tblspcpath (const char *) - 走査対象となるテーブルスペースのパス。
  • 出力: なし（副作用としてファイルシステム上のファイルを削除する）。

前提条件と不変条件

  • tblspcpath は有効なディレクトリパスであること。
  • RELCACHE_INIT_FILENAME で定義されるファイル名が、各データベースディレクトリ内に存在し得る。
  • データベースディレクトリは数値のみで構成される名前を持つものと想定している。

アルゴリズムまたは振る舞い

1. AllocateDir を使用して tblspcpath を開く。 2. ReadDirExtended を用いてディレクトリ内のエントリを順次読み込む。 3. 各エントリ名が数値のみで構成されているかを確認する（データベースOIDのディレクトリを特定するため）。 4. 数値ディレクトリである場合、そのパス内に RELCACHE_INIT_FILENAME を構築する。 5. unlink_initfile を呼び出し、該当する初期化ファイルを削除する。 6. 最後に FreeDir を呼び出し、ディレクトリハンドルを解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/cache/relcache.c 内の関数（"base" パスやその他のパスを指定して呼び出される）。
  • 呼び出し先:
  • AllocateDir, ReadDirExtended, FreeDir（ディレクトリ操作用）。
  • unlink_initfile（ファイル削除用）。

関連データ構造

  • DIR: ディレクトリ操作用ハンドル。
  • struct dirent: ディレクトリ内のエントリ情報。

失敗モードと境界条件

  • ディレクトリが開けない場合や読み込めない場合、ReadDirExtended の引数 LOG によりエラーが記録される可能性がある。
  • 数値以外の名前を持つディレクトリやファイルはスキップされる。
  • unlink_initfile が失敗した場合の挙動は、当該関数の実装に依存する。

保守担当者向けメモ

  • 本関数はリレーションキャッシュの整合性を保つためのクリーンアップ処理の一部である。
  • ディレクトリ走査のロジックは、PostgreSQLのデータベースディレクトリ構造（数値OID）に強く依存しているため、ディレクトリ構成の変更時には注意が必要である。
