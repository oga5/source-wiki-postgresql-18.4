ExecStatusType

定義

  • 種別: enum
  • 定義箇所: src/interfaces/libpq/libpq-fe.h:122

目的

PostgreSQL サーバーに対して実行したクエリやコマンドの結果状態を識別するために使用されます。

シグネチャまたは形状

typedef enum {
    PGRES_EMPTY_QUERY,
    PGRES_COMMAND_OK,
    PGRES_TUPLES_OK,
    PGRES_COPY_OUT,
    PGRES_COPY_IN,
    PGRES_BAD_RESPONSE,
    PGRES_NONFATAL_ERROR,
    PGRES_FATAL_ERROR,
    PGRES_COPY_BOTH,
    PGRES_SINGLE_TUPLE,
    PGRES_PIPELINE_SYNC,
    PGRES_PIPELINE_ABORTED,
    PGRES_TUPLES_CHUNK
} ExecStatusType;

入力と出力

  • **入力**: PQresultStatus() 関数等の戻り値として取得されます。
  • **出力**: クエリ実行結果のステータスを表現する列挙値。

前提条件と不変条件

  • libpq の内部実装（fe-exec.c）と列挙順序が一致している必要があります。
  • 列挙値は追加される可能性がありますが、既存の定数の変更や削除は行われません。

アルゴリズムまたは振る舞い

クエリ実行後の PGresult オブジェクトの状態を判定します。

  • PGRES_TUPLES_OK はタプルを返すクエリの成功を示します。
  • PGRES_COMMAND_OK はタプルを返さないコマンドの成功を示します。
  • PGRES_COPY_* 系はデータ転送中であることを示します。
  • PGRES_PIPELINE_* 系はパイプラインモードでの同期や中断状態を示します。

呼び出し元と呼び出し先

  • **呼び出し元**: psql/common.c（PQresultStatus を介して利用）、pg_dump/pg_backup_db.c、pg_upgrade/server.c 等。
  • **呼び出し先**: 情報なし。

関連データ構造

  • PGresult: クエリ実行結果を保持する構造体。この構造体のメンバとしてステータスが保持されます。

失敗モードと境界条件

  • PGRES_BAD_RESPONSE: サーバーから予期しない応答を受信した場合。
  • PGRES_FATAL_ERROR: クエリ実行が失敗した場合。
  • PGRES_NONFATAL_ERROR: 通知や警告メッセージが発生した場合。
  • PGRES_PIPELINE_ABORTED: パイプライン内の先行するコマンドの失敗により、実行されなかったコマンドに対して設定されます。

保守担当者向けメモ

  • fe-exec.c 内の PQmakeEmptyPGresult 等でこの型が使用されています。
  • 新しいステータスを追加する場合は、libpq-fe.h の定義と fe-exec.c の処理ロジックの整合性を確認してください。
