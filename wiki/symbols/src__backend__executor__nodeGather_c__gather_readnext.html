gather_readnext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGather.c:310

目的

並列クエリ実行において、複数のバックグラウンドワーカーが管理する TupleQueueReader からタプルを非ブロックで取得し、統合する。

シグネチャまたは形状

static MinimalTuple gather_readnext(GatherState *gatherstate)

入力と出力

  • **入力**: GatherState *gatherstate（並列実行の状態を保持する構造体）
  • **出力**: 取得した MinimalTuple。すべてのワーカーからの読み取りが完了した場合は NULL を返す。

前提条件と不変条件

  • gatherstate->nextreader は常に gatherstate->nreaders 未満であること。
  • ワーカーが初期化に失敗した場合、TupleQueueReaderNext は NULL を返すが、本関数はそれをタプル未生成として扱い、後続の WaitForParallelWorkersToFinish でエラーを検出する設計となっている。

アルゴリズムまたは振る舞い

1. **割り込みチェック**: CHECK_FOR_INTERRUPTS() を呼び出し、非同期イベントを処理する。 2. **非ブロック読み取り**: 現在の nextreader が指すキューから TupleQueueReaderNext を用いてタプルを取得する。 3. **完了処理**: 読み取りが完了したキューは gatherstate->reader 配列から削除し、すべてのキューが空になれば ExecShutdownGatherWorkers を呼び出して終了する。 4. **ラウンドロビン**: タプルが取得できなかった場合、nextreader を進めて次のキューを走査する。 5. **待機**: すべての生存するキューを走査してもタプルが得られない場合、need_to_scan_locally が真であれば NULL を返し、偽であれば WaitLatch を用いてタプルの到着を待機する。

呼び出し元と呼び出し先

  • **呼び出し元**: gather_getnext（ソース内の記述より推測）
  • **呼び出し先**: TupleQueueReaderNext, ExecShutdownGatherWorkers, WaitLatch, ResetLatch

関連データ構造

  • GatherState: 並列実行の進捗とキューの状態を管理する。
  • TupleQueueReader: ワーカーからタプルを読み取るためのキューインターフェース。

失敗モードと境界条件

  • **ワーカーの終了**: すべてのワーカーがタプル生成を終えた場合、nreaders が 0 となり処理を終了する。
  • **ローカル実行**: need_to_scan_locally が有効な場合、ワーカーからタプルが得られなくても待機せず NULL を返すことで、リーダープロセスによるローカル実行へ制御を戻す。

保守担当者向けメモ

  • 本関数は非ブロック読み取りを前提としているため、ワーカーがタプルを生成するまで待機しない。
  • nextreader の更新はラウンドロビン方式で行われるが、効率化のため、ブロックが必要になるまでは同一のキューを優先的に読み取る実装となっている。
