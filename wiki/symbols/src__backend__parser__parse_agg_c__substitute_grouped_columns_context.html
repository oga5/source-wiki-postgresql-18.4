substitute_grouped_columns_context

定義

  • 種別: struct
  • 定義箇所: src/backend/parser/parse_agg.c:46

目的

クエリ内の式を走査し、グループ化された列（grouped columns）を RTE_GROUP への参照に置換する処理において、状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    ParseState *pstate;
    Query      *qry;
    bool        hasJoinRTEs;
    List       *groupClauses;
    List       *groupClauseCommonVars;
    List       *gset_common;
    bool        have_non_var_grouping;
    List      **func_grouped_rels;
    int         sublevels_up;
    bool        in_agg_direct_args;
} substitute_grouped_columns_context;

入力と出力

本構造体は関数引数として渡され、substitute_grouped_columns やその内部で使用されるミューテーター関数（substitute_grouped_columns_mutator）間で共有されます。

前提条件と不変条件

  • pstate および qry は有効なポインタである必要があります。
  • sublevels_up は現在のクエリレベルからの相対的な深さを表し、再帰的な走査において適切に管理される必要があります。

アルゴリズムまたは振る舞い

  • expression_tree_mutator を用いた式ツリーの書き換え処理において、現在のグループ化状態やクエリレベルの情報を保持するために使用されます。
  • have_non_var_grouping や in_agg_direct_args などのフラグにより、置換対象の判定ロジックを制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_agg.c 内の substitute_grouped_columns および関連する変換関数からインスタンス化されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、substitute_grouped_columns_mutator 等の関数に引数として渡されます。

関連データ構造

  • ParseState: パーサーの状態管理。
  • Query: クエリツリーのルート。
  • List: グループ化句や変数リストの保持に使用。

失敗モードと境界条件

  • sublevels_up の管理が不適切な場合、誤ったクエリレベルの変数を参照する可能性があります。
  • GROUPING SETS の展開に関連する複雑なネスト構造において、スタックオーバーフローや処理制限（4096セットの制限など）に抵触する可能性があります。

保守担当者向けメモ

  • 本構造体は substitute_grouped_columns の再帰処理における状態保持の要です。フィールドを追加・変更する際は、substitute_grouped_columns_mutator での参照整合性に注意してください。
  • sublevels_up のカウンタが正しく更新されているか、デバッグ時には特に注意を払ってください。
