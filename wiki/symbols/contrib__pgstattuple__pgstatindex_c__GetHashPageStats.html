GetHashPageStats

定義

  • 種別: function
  • 定義箇所: contrib/pgstattuple/pgstatindex.c:741

目的

Hashインデックスの個別のページを走査し、生存タプル数、削除済みタプル数、および空き容量を計算して統計情報構造体に加算する。

シグネチャまたは形状

static void GetHashPageStats(Page page, HashIndexStat *stats)

入力と出力

  • **入力**:
  • page: 解析対象のHashインデックスページ（Page型）。
  • stats: 統計情報を格納する HashIndexStat 構造体へのポインタ。
  • **出力**: なし（stats が指す構造体のメンバを直接更新する）。

前提条件と不変条件

  • page は有効なHashインデックスページであること。
  • stats は呼び出し元によって適切に初期化されていること。

アルゴリズムまたは振る舞い

1. PageGetMaxOffsetNumber を使用して、ページ内の最大オフセット番号を取得する。 2. FirstOffsetNumber から最大オフセットまでループし、各アイテムIDを PageGetItemId で取得する。 3. ItemIdIsDead を判定し、生存タプルであれば stats->live_items を、削除済みであれば stats->dead_items をインクリメントする。 4. ループ終了後、PageGetExactFreeSpace を呼び出し、その結果を stats->free_space に加算する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgstattuple/pgstatindex.c 内のHashインデックス走査ロジック（672行目、677行目付近）。
  • **呼び出し先**:
  • PageGetMaxOffsetNumber
  • PageGetItemId
  • ItemIdIsDead
  • PageGetExactFreeSpace

関連データ構造

  • HashIndexStat: Hashインデックスの統計情報を保持する構造体。
  • Page: PostgreSQLのページデータ構造。
  • ItemId: ページ内のアイテム識別子。

失敗モードと境界条件

  • ページが破損している場合、PageGetMaxOffsetNumber や PageGetItemId の挙動は未定義となる可能性がある。
  • 呼び出し元で適切なロックが取得されていない場合、競合状態が発生する可能性がある。

保守担当者向けメモ

  • 本関数はHashインデックスのページ単位の統計収集に特化している。
  • 統計情報の精度は、呼び出し元がページを走査する際のロック状態に依存する。
