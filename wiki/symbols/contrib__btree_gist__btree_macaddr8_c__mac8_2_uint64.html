mac8_2_uint64

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_macaddr8.c:93

目的

macaddr8 型の値を uint64 型に変換し、GiST インデックスのペナルティ計算（木構造のバランス調整）を可能にするための補助関数です。

シグネチャまたは形状

static uint64 mac8_2_uint64(macaddr8 *m)

入力と出力

  • **入力**: macaddr8 *m （変換対象の MAC アドレスへのポインタ）
  • **出力**: uint64 （8バイトの MAC アドレスをビッグエンディアン形式の数値として解釈した値）

前提条件と不変条件

  • 入力ポインタ m は有効な macaddr8 構造体を指している必要があります。
  • macaddr8 は 8 バイトのデータ型として扱われます。

アルゴリズムまたは振る舞い

1. macaddr8 の内部表現を unsigned char の配列として読み込みます。 2. 8 バイトの各バイトを、上位バイトから順に uint64 の対応するビット位置へシフトして加算します。 3. 最終的な 64 ビット整数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_macaddr8.c 内の gbt_macad8_penalty 関数（推測される文脈）において、origentry および newentry の下限・上限値を変換するために呼び出されます。
  • **呼び出し先**: 特になし（標準的な C の算術演算のみを使用）。

関連データ構造

  • macaddr8: PostgreSQL の MAC アドレス型。
  • mac8KEY: macaddr8 の範囲を保持する構造体。

失敗モードと境界条件

  • 入力が NULL の場合の挙動は定義されていません。
  • 8 バイトを超えるデータや不正なメモリ領域へのアクセスに対する保護は行われていません。

保守担当者向けメモ

  • 本関数は macaddr8 を数値として扱うための基盤です。インデックスのペナルティ計算において、キー間の距離を算出する際の正規化手段として機能します。
  • 類似の macaddr (6バイト) 用の実装と比較する際は、バイト数の違いに注意してください。
