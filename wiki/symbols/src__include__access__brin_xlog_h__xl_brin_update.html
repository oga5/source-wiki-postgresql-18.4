xl_brin_update

定義

  • 種別: struct
  • 定義箇所: src/include/access/brin_xlog.h:87

目的

BRIN インデックスにおける更新操作（XLOG_BRIN_UPDATE）を WAL に記録するためのデータ構造です。インデックスページ上の既存タプルの更新情報を保持し、リカバリ時の再実行を可能にします。

シグネチャまたは形状

typedef struct xl_brin_update
{
    OffsetNumber oldOffnum;
    xl_brin_insert insert;
} xl_brin_update;

入力と出力

  • **入力**: 更新対象の古いタプルのオフセット番号 (oldOffnum) および、新しいタプルの挿入に必要なメタデータ (insert)。
  • **出力**: なし（本構造体は WAL レコードのペイロードとして書き込まれます）。

前提条件と不変条件

  • insert フィールドには、xl_brin_insert 構造体で定義される挿入操作のメタデータが含まれている必要があります。
  • SizeOfBrinUpdate マクロ（offsetof(xl_brin_update, insert) + SizeOfBrinInsert）を使用して、WAL レコードのサイズを正しく計算する必要があります。

アルゴリズムまたは振る舞い

1. **記録**: インデックスの更新操作時に、対象となる古いタプルの位置 (oldOffnum) と、新しいタプルの情報を xl_brin_insert 構造体に格納して WAL レコードを生成します。 2. **再生**: リカバリプロセスにおいて、brin_redo 関数がこのレコードを読み取ります。XLogRecGetData を介して本構造体を取得し、oldOffnum を用いて古いタプルを特定・処理した後、insert フィールドの情報に基づき新しいタプルの挿入処理を再実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_pageops.c: インデックス更新処理の過程で本構造体がインスタンス化されます。
  • **呼び出し先**:
  • src/backend/access/brin/brin_xlog.c: brin_redo 関数内で、WAL レコードから本構造体をキャストして読み取り、ページ操作を実行します。
  • src/backend/access/rmgrdesc/brindesc.c: WAL レコードのデバッグ出力（brin_desc）において、内容の解析に使用されます。

関連データ構造

  • xl_brin_insert: 本構造体の一部として埋め込まれ、挿入操作のメタデータを定義します。
  • OffsetNumber: ページ内のタプル位置を指す型。

失敗モードと境界条件

  • WAL レコードの読み取り時にデータが破損している場合、XLogRecGetData によるキャスト後のフィールドアクセスで不正な値を取得する可能性があります。
  • oldOffnum が対象ページ内の有効なオフセット範囲外である場合、再実行時にページ操作エラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は xl_brin_insert を包含しているため、xl_brin_insert の定義に変更を加える際は、SizeOfBrinUpdate マクロの整合性が保たれているか確認してください。
  • リカバリ時の挙動については、src/backend/access/brin/brin_xlog.c の実装を参照してください。
