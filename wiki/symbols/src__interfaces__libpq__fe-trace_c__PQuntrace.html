PQuntrace

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-trace.c:48

目的

指定されたデータベース接続 (PGconn) におけるデバッグ用のトレース出力を無効化し、関連するリソースをクリーンアップする。

シグネチャまたは形状

void PQuntrace(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn - トレースを停止する対象の接続オブジェクト。
  • **出力**: なし（void）。

前提条件と不変条件

  • conn が NULL の場合、関数は何もせず直ちに終了する。
  • 実行後、conn->Pfdebug は NULL に設定され、conn->traceFlags は 0 にリセットされる。

アルゴリズムまたは振る舞い

1. conn が NULL でないことを確認する。 2. conn->Pfdebug（トレース出力先ファイルポインタ）が設定されている場合、以下の処理を行う：

  • fflush(conn->Pfdebug) を呼び出し、バッファ内のデータをフラッシュする。
  • conn->Pfdebug を NULL に設定する。

    • 3. conn->traceFlags を 0 に設定し、トレースフラグを無効化する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は src/interfaces/libpq/fe-trace.c:39 に存在することが確認されている。
  • **呼び出し先**: 標準ライブラリの fflush を呼び出す。

関連データ構造

  • PGconn: libpq の接続状態を保持する構造体。Pfdebug（出力先）および traceFlags（設定フラグ）をメンバとして含む。

失敗モードと境界条件

  • conn が NULL の場合は安全に無視される。
  • Pfdebug が既に NULL の場合は、フラッシュ処理はスキップされる。

保守担当者向けメモ

  • 本関数は PQtrace によって開始されたトレースセッションを終了させるための対となる関数である。
  • トレース出力先（Pfdebug）のクローズ処理自体は本関数内では行われないため、ファイルポインタの管理には注意すること。
