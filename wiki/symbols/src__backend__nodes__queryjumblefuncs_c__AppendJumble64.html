AppendJumble64

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/queryjumblefuncs.c:361

目的

クエリのフィンガープリント生成プロセスにおいて、8バイト（64ビット）のデータをハッシュ計算用のバッファに追加する。

シグネチャまたは形状

static pg_noinline void AppendJumble64(JumbleState *jstate, const unsigned char *value)

入力と出力

  • **入力**:
  • jstate: ハッシュ計算の状態を保持する JumbleState 構造体へのポインタ。
  • value: ハッシュバッファに追加する8バイトのデータへのポインタ。
  • **出力**: なし（副作用として jstate 内のバッファを更新）。

前提条件と不変条件

  • jstate は初期化済みであること。
  • value は少なくとも8バイトの有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. jstate->pending_nulls が 0 より大きい場合、FlushPendingNulls(jstate) を呼び出して保留中の NULL 情報をバッファにフラッシュする。 2. AppendJumbleInternal(jstate, value, 8) を呼び出し、指定された8バイトのデータをハッシュバッファに書き込む。 3. pg_noinline 指定により、インライン展開を抑制し、関数呼び出しのオーバーヘッドを許容しつつコードサイズを制御する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/queryjumblefuncs.c 内のノード走査処理（例: expr->item のハッシュ化など）。
  • **呼び出し先**:
  • FlushPendingNulls
  • AppendJumbleInternal

関連データ構造

  • JumbleState: クエリのハッシュ化状態、バッファ、および保留中の NULL 情報を管理する構造体。

失敗モードと境界条件

  • バッファのオーバーフロー: AppendJumbleInternal 内部でバッファが満杯になった場合、既存の内容をハッシュ化してバッファを再利用する処理が行われる。

保守担当者向けメモ

  • 本関数は queryId 生成の基盤となるシリアライズ関数の一つである。
  • 新しいノード型を追加する際、64ビットのフィールドをハッシュに含める場合は本関数を使用すること。
  • pending_nulls の処理順序は、ハッシュの整合性を保つために重要である。
