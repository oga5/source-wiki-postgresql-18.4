compute_common_attribute

定義

  • 種別: function
  • 定義箇所: src/backend/commands/functioncmds.c:514

目的

関数またはプロシージャの定義時に指定される属性（volatility, strict, security, leakproof, set, cost, rows, support, parallel）を解析し、対応する DefElem ポインタへ割り当てる。

シグネチャまたは形状

bool compute_common_attribute(ParseState *pstate,
                              bool is_procedure,
                              DefElem *defel,
                              DefElem **volatility_item,
                              DefElem **strict_item,
                              DefElem **security_item,
                              DefElem **leakproof_item,
                              List **set_items,
                              DefElem **cost_item,
                              DefElem **rows_item,
                              DefElem **support_item,
                              DefElem **parallel_item)

入力と出力

  • **入力**:
  • pstate: パーサー状態。
  • is_procedure: 現在の定義対象がプロシージャであるかを示すフラグ。
  • defel: 解析対象の属性定義要素。
  • 各種 DefElem ** 引数: 属性の格納先。
  • **出力**:
  • 戻り値: 属性が認識された場合は true、それ以外は false。
  • 各種 DefElem ** 引数: 属性が指定されていた場合、そのポインタが更新される。
  • set_items: set 属性の場合、リストに値が追加される。

前提条件と不変条件

  • is_procedure が true の場合、プロシージャ定義で許可されていない属性（volatility, strict, leakproof, cost, rows, support, parallel）が指定されるとエラーが発生する。
  • 各属性（set を除く）は、既に値が設定されている状態で再度指定されると、errorConflictingDefElem を呼び出しエラーとなる。

アルゴリズムまたは振る舞い

1. defel->defname を比較し、対応する属性を特定する。 2. プロシージャ定義で許可されない属性が指定された場合、procedure_error ラベルへジャンプし ERROR を報告する。 3. 既に値が設定されている属性が重複指定された場合、エラーを報告する。 4. 正常に認識された場合、対応する引数に defel を格納し true を返す。 5. 認識できない属性名の場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/functioncmds.c 内の関数定義解析処理（CREATE FUNCTION 等の構文解析ループ内）。
  • **呼び出し先**:
  • errorConflictingDefElem: 属性の重複指定時に呼び出される。
  • lappend: set 属性のリスト追加に使用。
  • ereport: エラー報告に使用。

関連データ構造

  • ParseState: パーサーの状態を保持する構造体。
  • DefElem: SQL定義要素を表すノード。

失敗モードと境界条件

  • **エラー**:
  • プロシージャ定義で関数専用の属性が指定された場合（ERRCODE_INVALID_FUNCTION_DEFINITION）。
  • 同一属性が複数回指定された場合。
  • **境界条件**: 認識できない属性名が渡された場合は、エラーにせず false を返す。

保守担当者向けメモ

  • 本関数は関数とプロシージャの属性解析を共通化しているため、新しい属性を追加する際は is_procedure フラグによる制約チェックが適切か確認すること。
