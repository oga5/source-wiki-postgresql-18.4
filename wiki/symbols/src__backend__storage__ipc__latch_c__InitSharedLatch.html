InitSharedLatch

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/latch.c:92

目的

共有メモリ上で使用されるラッチ（Latch）構造体を初期化し、プロセス間での同期を可能にするための準備を行う。

シグネチャまたは形状

void InitSharedLatch(Latch *latch)

入力と出力

  • **入力**: latch (初期化対象となる Latch 構造体へのポインタ)
  • **出力**: なし（副作用として latch が指すメモリ領域が更新される）

前提条件と不変条件

  • latch は有効なメモリ領域を指している必要がある。
  • 共有メモリ上のラッチとして使用するため、ポストマスター起動時など、子プロセスがフォークされる前に呼び出される必要がある。

アルゴリズムまたは振る舞い

1. **Windows 環境の場合**: CreateEvent を呼び出し、継承可能なイベントハンドルを作成して latch->event に格納する。失敗した場合は elog(ERROR) を発生させる。 2. **共通処理**:

  • is_set を false に設定。
  • maybe_sleeping を false に設定。
  • owner_pid を 0 に設定。
  • is_shared を true に設定。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogrecovery.c: XLogRecoveryCtl->recoveryWakeupLatch の初期化。
  • src/backend/storage/lmgr/proc.c: proc->procLatch の初期化。
  • **呼び出し先**:
  • Windows 環境下では CreateEvent および ZeroMemory を使用。

関連データ構造

  • Latch: ラッチの状態（セット状態、所有者PID、共有フラグ、OS固有のイベントハンドル）を保持する構造体。

失敗モードと境界条件

  • Windows 環境において CreateEvent が失敗した場合、elog(ERROR) によりプロセスが停止する。

保守担当者向けメモ

  • 本関数は共有メモリ上のラッチを対象とするため、ローカルラッチ用の InitLatch と混同しないこと。
  • is_shared を true に設定することで、共有メモリ内での利用を明示している。
