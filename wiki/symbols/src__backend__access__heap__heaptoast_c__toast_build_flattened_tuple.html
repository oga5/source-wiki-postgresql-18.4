toast_build_flattened_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heaptoast.c:562

目的

外部ストレージ（TOAST）に格納されている属性を含むタプルを、すべての属性がインライン化された（フラット化された）新しいタプルとして再構築します。

シグネチャまたは形状

HeapTuple toast_build_flattened_tuple(TupleDesc tupleDesc, Datum *values, bool *isnull)

入力と出力

  • **入力**:
  • tupleDesc: タプルの記述子。
  • values: 属性値の配列。
  • isnull: 各属性がNULLかどうかを示す配列。
  • **出力**:
  • 外部参照がすべて展開された新しい HeapTuple。

前提条件と不変条件

  • tupleDesc->natts が MaxTupleAttributeNumber 以下であること。
  • 処理対象の属性が可変長（attlen == -1）かつ非NULLであること。
  • 外部ストレージにある属性は detoast_external_attr によってメモリ上に展開される。

アルゴリズムまたは振る舞い

1. 入力された values 配列をローカルの new_values にコピーします。 2. 各属性を走査し、外部ストレージ（VARATT_IS_EXTERNAL）にある可変長属性を特定します。 3. 外部属性が見つかった場合、detoast_external_attr を呼び出してインラインデータに変換し、new_values を更新します。 4. 変換されたデータは一時的に freeable_values に保持されます。 5. heap_form_tuple を使用して、展開済みの値から新しいタプルを生成します。 6. 最後に、展開のために確保した一時メモリを pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c
  • **呼び出し先**:
  • detoast_external_attr
  • heap_form_tuple
  • pfree

関連データ構造

  • HeapTuple: 再構築されたタプルの型。
  • TupleDesc: タプルの構造定義。
  • varlena: 可変長データの内部表現。

失敗モードと境界条件

  • メモリ不足時に detoast_external_attr や heap_form_tuple がエラーを発生させる可能性があります。
  • 属性数が MaxTupleAttributeNumber を超える場合、Assert により停止します。

保守担当者向けメモ

  • 本関数は元のタプルを破壊せず、新しいメモリ領域にタプルを生成して返します。
  • 外部属性の展開に伴うメモリ確保は、関数終了時に確実に解放される設計となっています。
