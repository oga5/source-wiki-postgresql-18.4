ShippableCacheEntry

定義

  • 種別: struct
  • 定義箇所: contrib/postgres_fdw/shippable.c:49

目的

postgres_fdw において、特定のデータベースオブジェクトがリモートサーバーへプッシュダウン（実行委譲）可能かどうかを判定した結果をキャッシュするためのエントリ構造体です。

シグネチャまたは形状

typedef struct {
    ShippableCacheKey key;    /* ハッシュキー（構造体の先頭である必要がある） */
    bool        shippable;    /* 判定結果（true: 出荷可能, false: 不可） */
} ShippableCacheEntry;

入力と出力

  • 入力: ShippableCacheKey をキーとしてハッシュテーブルに格納されます。
  • 出力: shippable フィールドにより、対象オブジェクトのプッシュダウン可否を保持します。

前提条件と不変条件

  • key フィールドは構造体の先頭に配置する必要があります（ハッシュテーブル操作の制約）。
  • キャッシュはバックエンドの生存期間中保持されます。

アルゴリズムまたは振る舞い

  • ShippableCacheHash ハッシュテーブル内で、オブジェクトID、カタログID、サーバーIDの組み合わせに対する判定結果を保持します。
  • キャッシュミスが発生した場合、lookup_shippable が呼び出され、その結果が本構造体に格納されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/postgres_fdw/shippable.c 内のキャッシュ検索および更新ロジック（hash_seq_search、hash_search 等を使用する関数群）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ShippableCacheKey: キャッシュの検索キー。
  • ShippableCacheHash: 本構造体を要素として管理するハッシュテーブル。

失敗モードと境界条件

  • pg_foreign_server が更新された場合、InvalidateShippableCacheCallback によってキャッシュ全体がフラッシュされるため、本構造体のインスタンスも破棄されます。

保守担当者向けメモ

  • key フィールドが構造体の先頭にあることは、ハッシュテーブルのメモリレイアウト上の不変条件です。変更時は注意してください。
  • 現在の設計では、拡張機能のメンバーシップに基づいて shippable が決定されます。
