TailMatchesImpl

定義

  • 種別: function
  • 定義箇所: src/bin/psql/tab-complete.in.c:1648

目的

入力された単語列の末尾が、指定されたパターン（可変長引数）と一致するかを判定し、タブ補完の文脈条件を評価します。

シグネチャまたは形状

static bool TailMatchesImpl(bool case_sensitive, int previous_words_count, char **previous_words, int narg, ...);

入力と出力

  • **入力**:
  • case_sensitive: 大文字小文字を区別するかどうかのフラグ。
  • previous_words_count: previous_words 配列内の単語数。
  • previous_words: 入力行を分解した単語の配列。
  • narg: 比較対象となるパターンの数。
  • ...: 比較対象の文字列パターン（可変長引数）。
  • **出力**:
  • bool: パターンが末尾と一致すれば true、そうでなければ false。

前提条件と不変条件

  • narg は 64 以下である必要があります（内部バッファ argarray のサイズ制限）。
  • previous_words_count が narg 未満の場合、一致は不可能とみなされ直ちに false を返します。

アルゴリズムまたは振る舞い

1. narg が 64 を超えていないか Assert で検証します。 2. previous_words_count と narg を比較し、入力単語数が不足している場合は false を返します。 3. 可変長引数リスト（va_list）から narg 個の文字列パターンを取り出し、ローカル配列 argarray に格納します。 4. TailMatchesArray を呼び出し、実際のパターンマッチング処理を委譲してその結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/tab-complete.in.c 内の補完ロジック（match_previous_words 等の文脈）。
  • **呼び出し先**: TailMatchesArray。

関連データ構造

  • argarray: 可変長引数を受け取るための最大 64 要素の const char * 配列。

失敗モードと境界条件

  • narg > 64: Assert によりプログラムが停止します。
  • previous_words_count < narg: 比較を行わずに false を返します。

保守担当者向けメモ

  • 本関数は TailMatchesArray へのラッパーです。可変長引数インターフェースを固定長配列インターフェースに変換する役割を担っています。
  • gen_tabcomplete.pl による自動生成コードの一部として利用されることを想定しています。
