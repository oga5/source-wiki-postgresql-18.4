ExecSampleScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSamplescan.c:78

目的

SQLの TABLESAMPLE 句を実行するためのスキャンノードのメイン処理を実行し、サンプリングされたタプルを返します。

シグネチャまたは形状

TupleTableSlot *ExecSampleScan(PlanState *pstate)

入力と出力

  • **入力**: pstate (PlanState *) - 実行中の SampleScan ノードのプラン状態。
  • **出力**: TupleTableSlot * - サンプリング手法によって抽出された次のタプル、またはスキャン終了時は空のスロット。

前提条件と不変条件

  • pstate は SampleScanState 型にキャスト可能である必要があります。
  • ExecScan を介して呼び出されるため、スキャン状態の初期化が完了している必要があります。

アルゴリズムまたは振る舞い

1. pstate を SampleScanState にキャストします。 2. ExecScan 関数を呼び出し、サンプリングスキャンを実行します。 3. ExecScan の引数として、タプル取得用の SampleNext 関数と、再チェック用の SampleRecheck 関数を渡します。 4. ExecScan が返した TupleTableSlot をそのまま呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（scanstate->ss.ps.ExecProcNode に本関数が割り当てられている）。
  • **呼び出し先**:
  • ExecScan (共通スキャン実行ロジック)
  • SampleNext (サンプリング手法に基づくタプル取得)
  • SampleRecheck (フィルタ条件の再チェック)

関連データ構造

  • SampleScanState: スキャンノードの状態を保持する構造体。
  • PlanState: プラン実行時の状態を保持する基底構造体。
  • TupleTableSlot: タプルを格納するスロット。

失敗モードと境界条件

  • ExecScan の内部動作に依存します。
  • サンプリング手法（TsmRoutine）が提供する SampleNext がタプルを供給できなくなった場合、ExecScan はスキャン終了を通知します。

保守担当者向けメモ

  • 本関数は ExecScan を呼び出すためのラッパーであり、具体的なサンプリングロジックは SampleNext に委譲されています。
  • SampleRecheck が常に true を返すのは、サンプリングスキャンにおいて再チェックが必要なフィルタ条件が存在しないためです。
