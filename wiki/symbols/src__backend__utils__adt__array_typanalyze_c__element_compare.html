element_compare

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_typanalyze.c:739

目的

配列要素の比較を行うためのラッパー関数です。ANALYZE 処理中に配列要素のソートや検索を行う際、型固有の比較関数を呼び出すために使用されます。

シグネチャまたは形状

static int element_compare(const void *key1, const void *key2)

入力と出力

  • **入力**:
  • key1, key2: 比較対象となる Datum 型へのポインタ。
  • **出力**:
  • int: key1 が key2 より小さい場合は負の値、等しい場合は 0、大きい場合は正の値を返します。

前提条件と不変条件

  • グローバル変数または静的変数 array_extra_data が初期化されており、その中の cmp（比較関数）および coll_id（照合順序）が有効である必要があります。
  • key1 および key2 は Datum 型の値を指している必要があります。

アルゴリズムまたは振る舞い

1. key1 および key2 が指す Datum 値を取得します。 2. array_extra_data に格納された比較関数を FunctionCall2Coll を介して呼び出します。 3. 比較関数の戻り値を DatumGetInt32 で int 型に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • 同一ファイル内の trackitem_compare_element（qsort 用の比較関数として使用）。
  • 同一ファイル内の他の箇所（element_compare を直接呼び出すロジックが存在）。
  • **呼び出し先**:
  • FunctionCall2Coll（PostgreSQL の関数呼び出しインターフェース）。
  • array_extra_data->cmp が指す型固有の比較関数。

関連データ構造

  • ArrayAnalyzeExtraData: 配列の型情報や比較関数を保持する構造体。本関数はこの構造体を通じて比較ロジックを決定します。

失敗モードと境界条件

  • array_extra_data が初期化されていない状態で呼び出された場合、セグメンテーション違反が発生する可能性があります。
  • 比較対象の Datum が不正な値である場合、呼び出し先の比較関数に依存した動作となります。

保守担当者向けメモ

  • 本関数は qsort 等の標準ライブラリ関数から呼び出されることを想定したシグネチャになっています。
  • src/backend/utils/adt/array_selfuncs.c にも同名の関数が存在しますが、引数構成（void *arg の有無）が異なるため、混同しないよう注意してください。
