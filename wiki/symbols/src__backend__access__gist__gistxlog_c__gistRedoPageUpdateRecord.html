gistRedoPageUpdateRecord

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistxlog.c:69

目的

GiSTインデックスのページ更新操作をWAL（Write-Ahead Log）からリプレイし、インデックスページの整合性を復元する。

シグネチャまたは形状

static void gistRedoPageUpdateRecord(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）。
  • **出力**: なし（副作用としてバッファへの変更適用およびロック解除を行う）。

前提条件と不変条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返す場合のみ、ページ更新処理を実行する。
  • 1対1のタプル置換時は PageIndexTupleOverwrite を使用し、それ以外は PageIndexMultiDelete および PageAddItem を使用して整合性を維持する。
  • 処理完了後、対象ページに最新のLSNをセットし、バッファをダーティ状態にする必要がある。

アルゴリズムまたは振る舞い

1. XLogReadBufferForRedo を呼び出し、対象ページを読み込む。 2. xldata->ntodelete と xldata->ntoinsert の値に基づき、以下のいずれかの処理を行う：

  • **1対1置換**: PageIndexTupleOverwrite を使用して既存タプルを上書きする。
  • **削除**: PageIndexMultiDelete を使用して古いタプルを削除する。リーフページの場合は GistMarkTuplesDeleted を呼び出す。

    • 3. 新しいタプルが存在する場合、PageAddItem を使用してページに追加する。 4. 挿入されたタプル数がWALレコードの期待値と一致することを確認する。 5. ページにLSNを書き込み、MarkBufferDirty を実行する。 6. WALレコードにブロック参照（block_id 1）が含まれる場合、gistRedoClearFollowRight を呼び出して子ページのFollow-Rightフラグをクリアする。 7. 最後にバッファのロックを解除し、解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: gist_redo（src/backend/access/gist/gistxlog.c 内のディスパッチャ）。
  • **呼び出し先**: XLogReadBufferForRedo, XLogRecGetBlockData, BufferGetPage, PageIndexTupleOverwrite, PageIndexMultiDelete, GistPageIsLeaf, GistMarkTuplesDeleted, PageAddItem, PageSetLSN, MarkBufferDirty, XLogRecHasBlockRef, gistRedoClearFollowRight, UnlockReleaseBuffer。

関連データ構造

  • XLogReaderState: WALレコードの読み取り状態を保持。
  • gistxlogPageUpdate: 更新対象のタプル数やオフセット情報を含む構造体。
  • Buffer, Page: PostgreSQLのバッファ管理およびページ構造。

失敗モードと境界条件

  • PageIndexTupleOverwrite または PageAddItem が失敗した場合、elog(ERROR) によりリカバリ処理が中断される。
  • 挿入されたタプル数がWALレコードのメタデータと一致しない場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数はGiSTインデックスのページ更新の原子性を保証する。
  • gistRedoClearFollowRight の呼び出しは、ターゲットページが削除された後でも実行される必要があるため、バッファの有効性確認と独立して行われる。
