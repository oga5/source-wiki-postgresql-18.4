examine_expression

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/extended_stats.c:603

目的

ANALYZE 処理の一環として、統計対象となる式（Expression）のメタデータを準備し、型固有の解析関数を呼び出して統計計算の準備を行う。

シグネチャまたは形状

VacAttrStats *examine_expression(Node *expr, int stattarget)

入力と出力

  • **入力**:
  • expr: 解析対象の式（Node）。
  • stattarget: 統計の精度を指定するターゲット値。
  • **出力**:
  • 成功時: 初期化済みの VacAttrStats 構造体へのポインタ。
  • 失敗時: NULL（解析が不要、または統計計算が不可能と判断された場合）。

前提条件と不変条件

  • expr は NULL であってはならない。
  • pg_type システムカタログから対象式の型情報が取得可能である必要がある。
  • 統計計算に必要な typanalyze 関数（またはデフォルトの std_typanalyze）が正常に完了すること。

アルゴリズムまたは振る舞い

1. VacAttrStats 構造体をゼロクリアして割り当てる。 2. 式の型（exprType）、型修飾子（exprTypmod）、照合順序（exprCollation）を抽出して構造体に設定する。 3. TYPEOID システムキャッシュから型情報を取得し、構造体の型属性（attrtype）を初期化する。 4. STATISTIC_NUM_SLOTS 分の統計スロットを、対象式の型情報に基づいて初期化する。 5. 型固有の typanalyze 関数を呼び出す。指定がない場合は std_typanalyze を使用する。 6. 解析結果（ok）、計算関数（compute_stats）、最小行数（minrows）を検証し、条件を満たさない場合はメモリを解放して NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/extended_stats.c 内の複数箇所（統計構築プロセスにおいて、式ベースの統計を処理する際に呼び出される）。
  • **呼び出し先**:
  • exprType, exprTypmod, exprCollation（式情報の抽出）
  • SearchSysCacheCopy1（型情報の取得）
  • OidFunctionCall1（型固有の typanalyze 関数の実行）
  • std_typanalyze（フォールバック用解析関数）

関連データ構造

  • VacAttrStats: 統計計算の入力および結果を保持する構造体。
  • Node: 解析対象の式ツリー。
  • Form_pg_type: 型情報を保持するカタログ構造体。

失敗モードと境界条件

  • 型情報のキャッシュルックアップに失敗した場合、elog(ERROR) により処理が中断される。
  • typanalyze 関数が false を返した場合、または compute_stats が設定されない場合、あるいは minrows が 0 以下の場合、解析は失敗とみなされ NULL が返される。

保守担当者向けメモ

  • stats->anl_context に CurrentMemoryContext を割り当てているが、これが最適かどうかは検討の余地がある（ソース内の XXX コメント参照）。
  • 統計対象の式が複雑な場合や、型固有の解析がサポートされていない場合の挙動については、std_typanalyze の実装に依存する。
