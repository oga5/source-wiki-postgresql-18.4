PLy_global_args_push

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_exec.c:610

目的

PL/Python プロシージャの再帰呼び出し時において、現在の引数状態をスタックに退避し、グローバル辞書が汚染されるのを防ぐ。

シグネチャまたは形状

static void PLy_global_args_push(PLyProcedure *proc)

入力と出力

  • **入力**: proc (PLyProcedure *) - 現在実行中のプロシージャ構造体。
  • **出力**: なし（副作用として proc 内の argstack および calldepth を更新）。

前提条件と不変条件

  • proc->calldepth が 0 より大きい場合、現在の引数値を保持する PLySavedArgs 構造体が生成される。
  • proc->argstack および proc->calldepth の更新後は、エラーが発生しない状態で制御を戻す必要がある。

アルゴリズムまたは振る舞い

1. proc->calldepth を確認し、再帰呼び出し中（> 0）であれば以下の処理を行う。

  • PLy_function_save_args(proc) を呼び出し、現在の引数値を格納した PLySavedArgs ノードを生成する。
  • 生成したノードを proc->argstack の先頭に連結する。

    • 2. 最後に proc->calldepth をインクリメントする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_exec.c 内の関数実行およびトリガー実行の初期化フェーズ（行 68, 357）。
  • **呼び出し先**:
  • PLy_function_save_args

関連データ構造

  • PLyProcedure: プロシージャの実行状態やスタックを保持する構造体。
  • PLySavedArgs: 退避された引数値を保持するスタックノード。

失敗モードと境界条件

  • proc->calldepth が 0 の場合、スタックへのプッシュは行われず、深さのカウントのみが行われる。

保守担当者向けメモ

  • 本関数で proc->argstack または proc->calldepth を変更した後は、エラーが発生しないように設計されている。
  • 対となる PLy_global_args_pop が存在し、呼び出し終了時にスタックの復元を行う。
