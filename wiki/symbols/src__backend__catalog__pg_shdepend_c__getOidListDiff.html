getOidListDiff

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_shdepend.c:420

目的

2つのソート済み Oid 配列を比較し、それぞれの配列に固有の要素（差分）を抽出して、元の配列をその差分のみで上書きする。

シグネチャまたは形状

static void getOidListDiff(Oid *list1, int *nlist1, int *nlist2, Oid *list2, int *nlist2)

入力と出力

  • **入力**:
  • list1, list2: 比較対象となる Oid の配列。
  • nlist1, nlist2: 各配列の要素数へのポインタ。
  • **出力**:
  • list1, list2: 関数実行後、それぞれ相手側に存在しない要素のみが格納される。
  • nlist1, nlist2: 差分抽出後の要素数に更新される。

前提条件と不変条件

  • 入力される list1 および list2 は、あらかじめソートされている必要がある。
  • 関数終了時、list1 には「list1 にのみ存在する要素」が、list2 には「list2 にのみ存在する要素」が格納される。

アルゴリズムまたは振る舞い

1. 2つのポインタ in1, in2 を用いて配列を走査し、共通する要素をスキップする。 2. 共通しない要素が見つかった場合、小さい方の値を対応する出力用インデックス (out1 または out2) に書き込む。 3. 一方の配列の走査が完了した後、残りの要素をそれぞれの出力用配列にコピーする。 4. 最後に、各配列の要素数ポインタ (nlist1, nlist2) を、書き込まれた要素数 (out1, out2) で更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/pg_shdepend.c 内の処理（例: 539 行目付近で oldmembers と newmembers の比較に使用）。
  • **呼び出し先**: 外部関数への依存なし（純粋な配列操作関数）。

関連データ構造

  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • 配列が空の場合、ループは実行されず、要素数は 0 に更新される。
  • 入力配列がソートされていない場合、正しく差分を抽出できない。

保守担当者向けメモ

  • 本関数は pg_shdepend.c 内の内部的なユーティリティであり、共有依存関係のメンバーシップ変更時などに、変更前後のリストの差分を特定するために使用される。
  • 破壊的な更新を行うため、呼び出し側で元の配列の内容が不要であることを確認すること。
