bbsink_zstd

定義

  • **種別**: struct
  • **定義箇所**: src/backend/backup/basebackup_zstd.c:23

目的

PostgreSQLのベースバックアップ処理において、Zstandard (zstd) 圧縮を適用するための状態管理構造体です。bbsink インターフェースを継承し、圧縮パイプラインの中間層として機能します。

シグネチャまたは形状

typedef struct bbsink_zstd
{
    bbsink      base;
    pg_compress_specification *compress;
    ZSTD_CCtx  *cctx;
    ZSTD_outBuffer zstd_outBuf;
} bbsink_zstd;

入力と出力

  • **入力**: bbsink インターフェースを介して渡されるデータストリーム。
  • **出力**: 圧縮されたデータストリームを後続の bbsink へ転送。

前提条件と不変条件

  • USE_ZSTD が定義され、libzstd がリンクされている環境でのみ有効です。
  • base メンバは bbsink として初期化されている必要があります。
  • cctx は begin_backup 時に初期化され、end_backup 時に解放される必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: bbsink_zstd_new によりメモリが確保され、圧縮パラメータが保持されます。 2. **圧縮処理**: archive_contents が呼び出されるたびに、ZSTD_compressStream2 を用いて入力データを逐次圧縮します。 3. **バッファ管理**: 圧縮後のデータは zstd_outBuf を経由して後続のシンクへ渡されます。 4. **終了処理**: end_archive で ZSTD_e_end を使用して圧縮ストリームをフラッシュし、end_backup でコンテキストを破棄します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup_zstd.c 内の各関数（bbsink_zstd_new, bbsink_zstd_begin_backup, bbsink_zstd_archive_contents 等）が、bbsink ポインタを本構造体にキャストして操作します。
  • **呼び出し先**: libzstd ライブラリの関数（ZSTD_compressStream2 等）を呼び出します。

関連データ構造

  • bbsink: 基底となるシンクインターフェース。
  • pg_compress_specification: 圧縮レベルやスレッド数などの設定情報。
  • ZSTD_CCtx: libzstd の圧縮コンテキスト。
  • ZSTD_outBuffer: 圧縮後の出力バッファ。

失敗モードと境界条件

  • ZSTD_compressBound を使用して、圧縮後のデータが後続のバッファを溢れないように管理する必要があります。
  • マニフェストファイルは圧縮対象外であり、bbsink_zstd_manifest_contents を通じて透過的に転送されます。

保守担当者向けメモ

  • 本構造体は bbsink パイプラインの一部であるため、メモリ管理（palloc0 等）は PostgreSQL のメモリコンテキストに従います。
  • 圧縮レベルやスレッド数の変更は pg_compress_specification を通じて行われます。
