pg_vprintf

定義

  • 種別: function
  • 定義箇所: src/port/snprintf.c:275

目的

PostgreSQL 独自の printf ファミリー実装の一部として、標準出力 (stdout) へのフォーマット済み出力機能を提供します。

シグネチャまたは形状

int pg_vprintf(const char *fmt, va_list args);

入力と出力

  • **入力**:
  • fmt: フォーマット文字列。
  • args: 可変長引数リスト (va_list)。
  • **出力**:
  • 戻り値: 書き込まれたバイト数（エラー発生時は負の値）。

前提条件と不変条件

  • stdout が有効なストリームとして存在している必要があります。
  • PostgreSQL の移植性レイヤーの一部であるため、プラットフォーム固有の printf 実装の差異を吸収した動作が保証されます。

アルゴリズムまたは振る舞い

内部的に pg_vfprintf を呼び出し、第一引数に stdout を渡すことで処理を委譲します。これにより、pg_vfprintf が持つフォーマット解析および出力ロジック（%n$ による引数順序制御や %m によるエラーメッセージ展開など）をそのまま利用します。

呼び出し元と呼び出し先

  • **呼び出し先**: pg_vfprintf
  • **呼び出し元**:
  • src/include/port.h にて vprintf マクロとして定義されており、PostgreSQL コードベース全体から標準の vprintf の代替として参照される可能性があります。
  • src/pl/plperl/plperl_system.h にて vprintf として再定義されています。

関連データ構造

  • va_list: 可変長引数を扱うための標準型。
  • PrintfTarget: pg_vfprintf 内部で出力先管理に使用される構造体。

失敗モードと境界条件

  • pg_vfprintf の動作に依存します。バッファオーバーランやフォーマットエラーが発生した場合、標準の snprintf 準拠の挙動（書き込まれるべきだったバイト数を返す等）に従います。

保守担当者向けメモ

  • 本関数は pg_vfprintf への単なるラッパーです。ロジックの修正が必要な場合は pg_vfprintf またはそのバックエンドである dopr() を確認してください。
  • src/include/port.h で vprintf が本関数にマクロ置換されているため、このファイル内での標準関数呼び出しには注意が必要です。
