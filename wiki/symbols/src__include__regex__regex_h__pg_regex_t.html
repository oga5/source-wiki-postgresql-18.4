pg_regex_t

定義

  • 種別: struct
  • 定義箇所: src/include/regex/regex.h:133

目的

PostgreSQL の正規表現エンジンにおいて、コンパイル済みの正規表現の状態を保持するために使用される主要な構造体です。

シグネチャまたは形状

typedef struct {
    int         re_magic;
    size_t      re_nsub;
    long        re_info;
    int         re_csize;
    char       *re_endp;
    Oid         re_collation;
    char       *re_guts;
    char       *re_fns;
} pg_regex_t;

入力と出力

  • **入力**: pg_regcomp 関数に未初期化の pg_regex_t 構造体へのポインタを渡し、コンパイル設定を格納します。
  • **出力**: コンパイル成功後、正規表現の解析結果やメタデータが構造体内に保持されます。

前提条件と不変条件

  • 内部構造は隠蔽されており、直接の操作は推奨されません。
  • re_guts は移植性を考慮し void * ではなく char * として定義されています。
  • pg_regcomp で初期化し、使用後は必ず pg_regfree でメモリを解放する必要があります。

アルゴリズムまたは振る舞い

  • re_info フィールドは、正規表現に含まれる特性（後方参照、境界量指定子、ロケール依存性など）をビットマスクで保持します。
  • re_collation により、LC_CTYPE の挙動を決定する照合順序を管理します。
  • コンパイルされた正規表現の実行時、pg_regexec はこの構造体を参照してマッチング処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_regcomp、pg_regexec、pg_regprefix、pg_regfree、pg_regerror などの関数が引数として受け取ります。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • pg_regmatch_t: マッチ結果の範囲を格納するために併用されます。
  • rm_detail_t: 補足的な制御や報告に使用されます。
  • regex_t: pg_regex_t へのエイリアスとして定義されています。

失敗モードと境界条件

  • pg_regcomp 実行時に REG_ESPACE（メモリ不足）や REG_BADPAT（不正なパターン）などのエラーが発生した場合、構造体は適切に初期化されない可能性があります。
  • re_magic フィールドによる内部的な整合性チェックが行われることが想定されます。

保守担当者向けメモ

  • 本構造体は POSIX regex_t との互換性を保ちつつ、PostgreSQL 固有の拡張（マルチバイト対応や照合順序）を統合しています。
  • 内部構造（re_guts 等）を変更する場合は、src/backend/regex/ 配下のエンジン実装との整合性に注意してください。
