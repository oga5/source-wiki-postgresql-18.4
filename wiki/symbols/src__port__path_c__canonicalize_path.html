canonicalize_path

定義

  • 種別: function
  • 定義箇所: src/port/path.c:336

目的

ファイルシステムパスを正規化し、冗長な区切り文字や .、.. を解決して一貫した形式に変換する。

シグネチャまたは形状

void canonicalize_path(char *path);

入力と出力

  • **入力**: path (正規化対象の文字列へのポインタ。インプレースで書き換えられる)
  • **出力**: なし（戻り値は void）

前提条件と不変条件

  • 入力された path は書き込み可能である必要がある。
  • この関数はパスの長さを拡大させないため、呼び出し側はバッファオーバーフローを懸念せずに既存のバッファをそのまま利用可能。

アルゴリズムまたは振る舞い

1. 内部的に canonicalize_path_enc(path, PG_SQL_ASCII) を呼び出す。 2. Windows 環境では \ を / に置換し、ドライブレターの処理を行う。 3. 状態マシンを用いてパスを走査し、冗長なパス要素を解決する。

呼び出し元と呼び出し先

  • **呼び出し先**: canonicalize_path_enc
  • **呼び出し元**:
  • src/backend/commands/extension.c
  • src/backend/commands/tablespace.c
  • src/backend/commands/variable.c
  • src/backend/utils/adt/genfile.c
  • src/backend/utils/adt/varlena.c
  • src/backend/utils/fmgr/dfmgr.c
  • src/backend/utils/misc/conffiles.c

関連データ構造

  • canonicalize_state: パス走査時の状態管理に使用される構造体。

失敗モードと境界条件

  • 入力文字列が NULL の場合の挙動についてはソースコード内に明示的なガード条件の記述なし。
  • パスの長さが変更される可能性がある（短縮される）。

保守担当者向けメモ

  • 本関数は PG_SQL_ASCII エンコーディングを指定して内部関数を呼び出している。マルチバイト文字を含むパスを扱う際は、エンコーディングの整合性に注意すること。
  • Windows 特有の \ と / の混在問題に対処しているため、プラットフォーム依存のパス操作を行う際は本関数を通すことが推奨される。
