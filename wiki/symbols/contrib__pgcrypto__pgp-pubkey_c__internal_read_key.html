internal_read_key

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pubkey.c:457

目的

OpenPGP 形式のデータストリームから、暗号化に使用可能な公開鍵または秘密鍵を検索・抽出します。

シグネチャまたは形状

static int internal_read_key(PullFilter *src, PGP_PubKey **pk_p, const uint8 *psw, int psw_len, int pubtype);

入力と出力

  • **入力**:
  • src: 読み込み元の PullFilter。
  • psw, psw_len: 秘密鍵復号用のパスワードとその長さ。
  • pubtype: 期待する鍵の種類（0: 公開鍵, 1: 秘密鍵）。
  • **出力**:
  • pk_p: 抽出された PGP_PubKey 構造体へのポインタを格納。
  • 戻り値: 成功時は 0、失敗時は負の PXE_* エラーコード。

前提条件と不変条件

  • 入力ストリームは OpenPGP パケット形式である必要があります。
  • 鍵の抽出は、can_encrypt フラグが真である鍵に対してのみ行われます。
  • 複数のメイン鍵パケットが含まれている場合はエラーとなります。

アルゴリズムまたは振る舞い

1. pgp_parse_pkt_hdr を用いてストリーム内のパケットを順次走査します。 2. パケットタグに応じて以下の処理を行います：

  • 公開鍵/秘密鍵パケット: 既にメイン鍵を読み込み済みであればエラーとし、そうでなければスキップします。
  • 公開サブ鍵: pubtype が 0 の場合、_pgp_read_public_key で解析します。
  • 秘密サブ鍵: pubtype が 1 の場合、process_secret_key で復号・解析します。
  • その他（署名、ユーザーID等）: pgp_skip_packet で無視します。

    • 3. 抽出した鍵が暗号化可能（can_encrypt）であれば enc_key に保持します。既に enc_key が存在する場合はエラー（PXE_PGP_MULTIPLE_SUBKEYS）となります。 4. ストリーム終了時に enc_key が存在すれば pk_p にセットして正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: pgp_set_pubkey（contrib/pgcrypto/pgp-pubkey.c:576）
  • **呼び出し先**:
  • pgp_parse_pkt_hdr, pgp_create_pkt_reader, pgp_skip_packet, pullf_free, pgp_key_free
  • _pgp_read_public_key
  • process_secret_key

関連データ構造

  • PGP_PubKey: 抽出された鍵情報を保持する構造体。
  • PullFilter: 入力データストリームの抽象化。

失敗モードと境界条件

  • PXE_PGP_MULTIPLE_KEYS: 複数のメイン鍵パケットが存在する場合。
  • PXE_PGP_MULTIPLE_SUBKEYS: 暗号化可能なサブ鍵が複数存在する場合。
  • PXE_PGP_EXPECT_SECRET_KEY / PXE_PGP_EXPECT_PUBLIC_KEY: pubtype とパケット種別が不一致の場合。
  • PXE_PGP_NO_USABLE_KEY: 暗号化可能な鍵が見つからなかった場合。
  • PXE_PGP_UNEXPECTED_PKT: 未知または予期しないパケットタグが検出された場合。

保守担当者向けメモ

  • 本関数は「バージョン 4」パケット形式のみを想定しています。
  • 鍵のメモリ管理は pgp_key_free に依存しているため、構造体定義の変更時には解放処理の整合性に注意してください。
