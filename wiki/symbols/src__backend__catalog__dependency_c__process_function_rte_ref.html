process_function_rte_ref

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/dependency.c:2335

目的

クエリ内の関数呼び出し（RangeTblEntry）において、特定の列番号（attnum）が複合型を返す場合に、その型への依存関係を記録する。

シグネチャまたは形状

static void process_function_rte_ref(RangeTblEntry *rte, AttrNumber attnum, find_expr_references_context *context)

入力と出力

  • rte: 処理対象の範囲テーブルエントリ。
  • attnum: 参照されている列番号。
  • context: 依存関係のリスト（addrs）を保持するコンテキスト構造体。
  • 出力: なし（副作用として context->addrs にオブジェクトアドレスを追加）。

前提条件と不変条件

  • rte は関数呼び出しを含む範囲テーブルエントリであること。
  • attnum は rte が生成する列の範囲内であること。

アルゴリズムまたは振る舞い

1. rte->functions リストを走査し、指定された attnum を生成する RangeTblFunction を特定する。 2. 対象関数の戻り値型（tupdesc）を取得する。 3. 戻り値が RECORD 型ではなく、名前付き複合型である場合、その型に対応するリレーションを特定し、add_object_address を呼び出して依存関係を context->addrs に追加する。 4. 戻り値が RECORD 型の場合や、関数が序数（ordinality）列を生成する場合、依存関係の追加は行わない。 5. 該当する列が見つからない場合は ERRCODE_UNDEFINED_COLUMN でエラーを報告する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/dependency.c 内の式参照探索ロジック（find_expr_references_walker 等から間接的に参照される）。
  • 呼び出し先: get_expr_result_tupdesc, get_typ_typrelid, add_object_address, ereport。

関連データ構造

  • RangeTblEntry: クエリ内のソース（関数等）を記述する構造体。
  • find_expr_references_context: 依存関係収集のコンテキスト。
  • RangeTblFunction: 範囲テーブル内の関数定義。

失敗モードと境界条件

  • 指定された attnum がどの関数にも属さない場合、ereport によりエラーが発生する。
  • 戻り値型が RECORD 型の場合、個別の列に対する依存関係は追跡されない。

保守担当者向けメモ

  • 本関数は、関数が返す複合型の列が削除された際に、依存関係を正しく追跡するために重要である。
  • tupdesc が NULL の場合（funccolnames が存在する場合など）、型情報は解決されないため、依存関係の追加はスキップされる。
