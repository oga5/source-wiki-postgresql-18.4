sepgsql_get_permissive

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/hooks.c:65

目的

SE-PostgreSQL の動作モードである「permissive（許容）」設定の状態を取得します。

シグネチャまたは形状

bool sepgsql_get_permissive(void)

入力と出力

  • 入力: なし
  • 出力: bool 型。true の場合、SE-PostgreSQL は強制モードではなく許容モードで動作しています。

前提条件と不変条件

  • モジュール内のグローバル変数 sepgsql_permissive の値を返します。
  • この関数は、SE-PostgreSQL の初期化完了後に呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. モジュール内で保持されている sepgsql_permissive 変数の現在の値を参照します。 2. その値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/sepgsql/label.c:254 (直接の参照を確認)
  • 呼び出し先:
  • なし（純粋なゲッター関数です）

関連データ構造

  • sepgsql_permissive: contrib/sepgsql/hooks.c 内で定義されている GUC（Grand Unified Configuration）変数。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。常に現在の設定値を返します。

保守担当者向けメモ

  • 本関数は sepgsql.h で extern 宣言されており、モジュール内の他ファイルから sepgsql_permissive の状態を確認するために使用されます。
  • GUC 変数の値が変更された場合、即座にその新しい値が反映されます。
