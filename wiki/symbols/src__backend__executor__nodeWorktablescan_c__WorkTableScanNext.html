WorkTableScanNext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeWorktablescan.c:29

目的

再帰的クエリ（WITH RECURSIVE）の実行中、親ノードである RecursiveUnion が保持する作業テーブル（tuplestore）から、次のタプルを順次取得してスキャンノードへ提供する。

シグネチャまたは形状

static TupleTableSlot *WorkTableScanNext(WorkTableScanState *node)

入力と出力

  • **入力**: WorkTableScanState *node（スキャン状態を保持する構造体）
  • **出力**: TupleTableSlot *（取得したタプルを格納したスロット。これ以上タプルがない場合は NULL）

前提条件と不変条件

  • node->ss.ps.state->es_direction が前方スキャン（ForwardScanDirection）であることを要求する（Assert による検証）。
  • 本ノードは作業テーブルの唯一の読み手であると仮定する。
  • 後方スキャン（Backward Scan）はサポートされない。

アルゴリズムまたは振る舞い

1. node->rustate->working_table から対象の Tuplestorestate を取得する。 2. node->ss.ss_ScanTupleSlot をターゲットスロットとして指定する。 3. tuplestore_gettupleslot を呼び出し、tuplestore から次のタプルをスロットに読み込む。 4. 取得したスロットを呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeWorktablescan.c 内の ExecScan インターフェース経由で呼び出される（ExecScanAccessMtd として登録）。
  • **呼び出し先**: tuplestore_gettupleslot（tuplestore からのデータ取得）。

関連データ構造

  • WorkTableScanState: スキャンノードの状態を管理する構造体。
  • Tuplestorestate: 作業テーブルの実体。
  • TupleTableSlot: タプルを保持するためのコンテナ。

失敗モードと境界条件

  • **後方スキャン**: ScanDirectionIsForward 以外の方向が指定された場合、アサーション違反により停止する。
  • **データ枯渇**: tuplestore にこれ以上タプルが存在しない場合、NULL を返す。

保守担当者向けメモ

  • 本関数はパフォーマンス上の理由から、tuplestore の読み取りポインタを共有し、コピーを避ける設計となっている。このため、他のノードが同時に同じ作業テーブルを読み取ることは想定されていない。
  • 後方スキャンをサポートする場合、nodeRecursiveunion.c 側での tuplestore 生成設定の変更が必要となるが、現状ではその必要性がないため実装されていない。
