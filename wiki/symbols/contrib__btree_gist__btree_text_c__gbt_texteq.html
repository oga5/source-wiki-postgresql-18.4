gbt_texteq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_text.c:45

目的

PostgreSQLのGiSTインデックスにおいて、text型データ同士の等価性判定を行うためのサポート関数です。

シグネチャまたは形状

static bool gbt_texteq(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の text 型データへのポインタ。
  • collation: 照合順序（Collation）のOID。
  • flinfo: 関数マネージャ情報（使用されません）。
  • **出力**:
  • bool: a と b が等しい場合に true、それ以外の場合に false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は有効な text 型データである必要があります。
  • 内部で texteq 関数を呼び出すため、PostgreSQLの標準的なテキスト比較ルールに従います。

アルゴリズムまたは振る舞い

1. DirectFunctionCall2Coll を介して、PostgreSQLの標準関数 texteq を呼び出します。 2. a と b を Datum 型に変換し、指定された collation を適用して比較を実行します。 3. 比較結果を DatumGetBool で論理値に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_text.c 内の tinfo 構造体（gbtree_vinfo）の初期化時に参照されています。
  • **呼び出し先**: texteq（PostgreSQL標準のテキスト等価比較関数）。

関連データ構造

  • gbtree_vinfo: text 型のインデックス操作に必要な比較関数群を保持する構造体。

失敗モードと境界条件

  • a または b が NULL の場合、標準の texteq 関数の挙動に従います。
  • 照合順序（collation）が不適切な場合、比較結果が期待と異なる可能性があります。

保守担当者向けメモ

  • 本関数は text 型専用です。bpchar 型には別の関数（gbt_bpchareq）が用意されています。
  • GiSTインデックスの consistent 関数等からコールバックされることを想定した設計になっています。
