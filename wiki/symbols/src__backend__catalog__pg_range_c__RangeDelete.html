RangeDelete

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_range.c:112

目的

指定された範囲型（Range Type）に対応する pg_range システムカタログ上のエントリを削除します。

シグネチャまたは形状

void RangeDelete(Oid rangeTypeOid)

入力と出力

  • **入力**: rangeTypeOid (Oid) - 削除対象となる範囲型のOID。
  • **出力**: なし。

前提条件と不変条件

  • pg_range テーブルに対する RowExclusiveLock を獲得可能であること。
  • 呼び出し元は、対象の範囲型に関連する依存関係（pg_depend 等）の削除を別途管理している必要があります（本関数は pg_range テーブルの行削除のみを行います）。

アルゴリズムまたは振る舞い

1. table_open を使用して pg_range を RowExclusiveLock モードで開きます。 2. ScanKeyInit を用いて、rngtypid 列が引数の rangeTypeOid と一致するタプルを検索するためのスキャンキーを初期化します。 3. systable_beginscan を呼び出し、RangeTypidIndexId インデックスを使用してシステムスキャンを開始します。 4. systable_getnext で該当するタプルを反復取得し、CatalogTupleDelete を呼び出して各タプルを削除します。 5. systable_endscan でスキャンを終了し、table_close でテーブルを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/typecmds.c:684
  • **呼び出し先**: table_open, ScanKeyInit, systable_beginscan, systable_getnext, CatalogTupleDelete, systable_endscan, table_close

関連データ構造

  • pg_range: 範囲型のメタデータを保持するシステムカタログ。
  • RangeTypidIndexId: pg_range の rngtypid 列に対するインデックス。

失敗モードと境界条件

  • 対象の rangeTypeOid が存在しない場合、スキャンは何もヒットせず、削除処理は行われません。
  • システムカタログのロックやアクセス権限に関連するエラーが発生した場合、PostgreSQLの標準的なエラーハンドリングによりトランザクションがアボートされる可能性があります。

保守担当者向けメモ

  • 本関数は pg_range テーブルの物理的な行削除のみを担います。型システム全体の整合性を保つため、依存関係の削除処理が適切に先行または後続しているか確認してください。
