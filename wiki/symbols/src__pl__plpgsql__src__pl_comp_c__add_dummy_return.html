add_dummy_return

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_comp.c:939

目的

PL/pgSQL 関数の実行ブロックの末尾に、明示的な RETURN 文がない場合に自動的に補完する。これにより、関数が正しく終了することを保証する。

シグネチャまたは形状

static void add_dummy_return(PLpgSQL_function *function)

入力と出力

  • **入力**: PLpgSQL_function *function（コンパイル対象の関数構造体）
  • **出力**: なし（引数の function 構造体内部の action を直接書き換える）

前提条件と不変条件

  • function->action が初期化されていること。
  • function->nstatements が現在のステートメント数を保持しており、新しいステートメント追加時にインクリメントされること。

アルゴリズムまたは振る舞い

1. **ブロックのラップ**: 関数本体が EXCEPTION 句を持つか、ラベルが付与されている場合、既存の action を新しい PLpgSQL_stmt_block でラップする。これにより、追加される RETURN が EXCEPTION 句の影響を受けたり、EXIT 文でスキップされたりするのを防ぐ。 2. **RETURN 文の追加**: 関数本体の末尾が PLPGSQL_STMT_RETURN でない場合、新しい PLpgSQL_stmt_return ノードを作成し、関数本体の末尾に追加する。 3. **戻り値の指定**: 追加される RETURN 文は、関数の出力パラメータ番号 (function->out_param_varno) を参照する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/pl_comp.c 内の plpgsql_compile および plpgsql_compile_inline から呼び出される。
  • **呼び出し先**: palloc0（メモリ確保）、list_make1、llast、lappend（リスト操作）。

関連データ構造

  • PLpgSQL_function: コンパイルされた関数全体を保持する構造体。
  • PLpgSQL_stmt_block: ブロック構造を表現するノード。
  • PLpgSQL_stmt_return: RETURN 文を表現するノード。

失敗モードと境界条件

  • palloc0 によるメモリ不足が発生した場合、PostgreSQL のエラーハンドリング機構によりアボートされる。
  • 関数が既に RETURN で終了している場合は、何も追加されない。

保守担当者向けメモ

  • この関数は、コンパイルの最終段階で呼び出されることを想定している。
  • ラップ処理を行う際、stmtid を適切にインクリメントして一意性を維持している点に注意すること。
