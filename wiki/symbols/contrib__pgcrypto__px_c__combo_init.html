combo_init

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/px.c:181

目的

PX_Combo オブジェクト内の暗号アルゴリズム（PX_Cipher）に対し、指定された鍵と初期化ベクトル（IV）を用いて初期化を行う。

シグネチャまたは形状

int combo_init(PX_Combo *cx, const uint8 *key, unsigned klen, const uint8 *iv, unsigned ivlen);

入力と出力

  • **入力**:
  • cx: 初期化対象の PX_Combo 構造体ポインタ。
  • key: 暗号化鍵データ。
  • klen: 鍵の長さ。
  • iv: 初期化ベクトルデータ。
  • ivlen: 初期化ベクトルの長さ。
  • **出力**:
  • int: 成功時に 0、失敗時にエラーコードを返す。

前提条件と不変条件

  • cx->cipher が有効な PX_Cipher オブジェクトを指していること。
  • 鍵および IV のバッファは、内部で palloc0 を用いて一時的にコピーされるため、呼び出し元でメモリ管理を継続する必要はない。

アルゴリズムまたは振る舞い

1. px_cipher_key_size および px_cipher_iv_size を呼び出し、対象アルゴリズムが必要とする鍵と IV のサイズを取得する。 2. IV が必要な場合、ivbuf を確保し、指定された iv をコピーする（ivlen が ivs を超える場合は切り詰め、不足する場合はゼロ埋めされる）。 3. 鍵を keybuf にコピーする（klen が ks を超える場合は切り詰められる）。 4. px_cipher_init を呼び出し、準備した keybuf と ivbuf で暗号アルゴリズムを初期化する。 5. 一時確保した ivbuf と keybuf を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/px.c 内で cx->init = combo_init; として関数ポインタが代入されている。直接的な呼び出し元はソースコード上では明示されていない。
  • **呼び出し先**: px_cipher_key_size, px_cipher_iv_size, px_cipher_init（いずれも px.h で定義されたインターフェース）。

関連データ構造

  • PX_Combo: 暗号化とパディングを抽象化する構造体。
  • PX_Cipher: 個別の暗号アルゴリズムの実装を保持する構造体。

失敗モードと境界条件

  • palloc0 によるメモリ確保失敗時は PostgreSQL のエラーハンドリングにより中断される。
  • px_cipher_init がエラーを返した場合、その値をそのまま戻り値として返す。
  • ivlen や klen がアルゴリズムの期待値と異なる場合、切り詰め処理が行われる。

保守担当者向けメモ

  • 本関数は PX_Combo の初期化フェーズを担う。新しい暗号アルゴリズムを追加する際は、PX_Cipher インターフェースが本関数の期待する key_size および iv_size を正しく返すことを確認すること。
