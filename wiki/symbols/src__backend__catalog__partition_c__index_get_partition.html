index_get_partition

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/partition.c:175

目的

指定されたパーティションリレーションに属するインデックスのうち、特定の親インデックス（indexId）に対応するパーティションインデックスのOIDを検索して返します。

シグネチャまたは形状

Oid index_get_partition(Relation partition, Oid indexId)

入力と出力

  • **入力**:
  • partition: 対象となるパーティションリレーション（Relation）。
  • indexId: 検索対象の親インデックスのOID。
  • **出力**:
  • 見つかった場合は対応するパーティションインデックスのOID。
  • 見つからない場合は InvalidOid。

前提条件と不変条件

  • partition は有効なリレーションである必要があります。
  • 内部で RelationGetIndexList を呼び出すため、呼び出し元は適切なロックを保持している必要があります。

アルゴリズムまたは振る舞い

1. RelationGetIndexList を使用して、指定された partition に紐づくインデックスのリストを取得します。 2. リスト内の各インデックスOIDについて、SearchSysCache1（RELOID）を用いて pg_class カタログを検索します。 3. relispartition フラグを確認し、パーティションであるインデックスのみを対象とします。 4. get_partition_parent を呼び出し、そのインデックスの親が indexId と一致するかを判定します。 5. 一致した場合、リストを解放して該当するOIDを返します。 6. 全てのインデックスを確認しても見つからない場合は InvalidOid を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c（複数箇所）
  • **呼び出し先**:
  • RelationGetIndexList
  • SearchSysCache1 / ReleaseSysCache
  • get_partition_parent
  • list_free

関連データ構造

  • Relation: リレーション情報を保持する構造体。
  • Form_pg_class: pg_class カタログのタプル構造。

失敗モードと境界条件

  • SearchSysCache1 でタプルが見つからない場合、elog(ERROR) により処理が中断されます。
  • パーティションにインデックスが存在しない場合、または一致する親インデックスがない場合は InvalidOid を返します。

保守担当者向けメモ

  • 本関数は pg_class.relispartition フラグに依存しています。
  • 呼び出し元で list_free を行う必要はありません（関数内で実行されます）。
