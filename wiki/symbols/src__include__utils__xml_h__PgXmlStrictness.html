PgXmlStrictness

定義

  • 種別: enum
  • 定義箇所: src/include/utils/xml.h:39

目的

XML処理時におけるエラー報告および通知の厳密さ（レベル）を制御するために使用される列挙型です。

シグネチャまたは形状

typedef enum
{
    PG_XML_STRICTNESS_LEGACY,
    PG_XML_STRICTNESS_WELLFORMED,
    PG_XML_STRICTNESS_ALL,
} PgXmlStrictness;

入力と出力

  • 入力: XML処理関数の初期化時（pg_xml_init 等）に引数として渡されます。
  • 出力: 該当する列挙値が、XMLパーサーの動作モードを決定します。

前提条件と不変条件

  • libxml2 が有効な環境で利用されることを前提としています。
  • 本列挙型は、XML処理セッションの初期化時に指定され、そのセッション中のエラーハンドリング方針を固定します。

アルゴリズムまたは振る舞い

  • PG_XML_STRICTNESS_LEGACY: 関数結果がエラー条件を示す場合を除き、エラーを無視します。
  • PG_XML_STRICTNESS_WELLFORMED: パーサー以外のメッセージを無視します。
  • PG_XML_STRICTNESS_ALL: すべての通知、警告、およびエラーを報告します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/adt/xml.c: pg_xml_init の引数として使用。
  • contrib/xml2/xpath.c: pgxml_parser_init の引数として使用。
  • contrib/xml2/xslt_proc.c: pgxml_parser_init の引数として使用。
  • 呼び出し先: 本シンボルは列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • PgXmlErrorContext: XML処理の初期化関数が返すコンテキスト構造体。
  • pg_xml_init: 本列挙型を引数に取る初期化関数。

失敗モードと境界条件

  • libxml2 が無効なビルド環境では、本列挙型を利用する関数はエラーを返すか、期待通りに動作しない可能性があります。

保守担当者向けメモ

  • 本列挙型は src/include/utils/xml.h で定義されており、XML処理の挙動を調整する重要なインターフェースです。変更時は src/backend/utils/adt/xml.c におけるエラーハンドリングロジックへの影響を確認してください。
