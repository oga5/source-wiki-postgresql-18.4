logicalmsg_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/logicalmsgdesc.c:45

目的

WALレコードの識別子ビットに基づき、論理メッセージ（Logical Message）に関連するレコードであるかを判定し、人間が読める形式の識別子文字列を返す。

シグネチャまたは形状

const char *logicalmsg_identify(uint8 info)

入力と出力

  • **入力**: uint8 info - WALレコードの情報を保持するビットマスク。
  • **出力**:
  • info が XLOG_LOGICAL_MESSAGE と一致する場合: "MESSAGE" という文字列定数。
  • 一致しない場合: NULL。

前提条件と不変条件

  • info ビットマスクから XLR_INFO_MASK を除外した値が XLOG_LOGICAL_MESSAGE と比較される。

アルゴリズムまたは振る舞い

1. 引数 info に対して ~XLR_INFO_MASK を適用し、補助的なフラグを除去する。 2. 除去後の値が XLOG_LOGICAL_MESSAGE と等しいかを確認する。 3. 等しければ "MESSAGE" を返し、そうでなければ NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h 内の PG_RMGR マクロ定義において、リソースマネージャの識別関数として登録されている。
  • **呼び出し先**: 該当なし。

関連データ構造

  • XLOG_LOGICAL_MESSAGE: 論理メッセージを示すWALレコードタイプ定数。

失敗モードと境界条件

  • info に論理メッセージ以外の値が渡された場合、本関数は NULL を返す。これは呼び出し側（WALダンプツール等）が未知のレコードタイプとして処理するための正常な挙動である。

保守担当者向けメモ

  • 本関数は pg_waldump 等のツールがWALレコードを識別するために使用される。
  • 新しい論理メッセージ関連のサブタイプが追加された場合は、本関数の条件分岐を拡張する必要がある。
