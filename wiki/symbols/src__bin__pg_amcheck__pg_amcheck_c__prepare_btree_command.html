prepare_btree_command

定義

  • 種別: function
  • 定義箇所: src/bin/pg_amcheck/pg_amcheck.c:882

目的

B-tree 索引の整合性チェックを実行するための SQL クエリを構築し、指定されたバッファに格納する。

シグネチャまたは形状

static void prepare_btree_command(PQExpBuffer sql, RelationInfo *rel, PGconn *conn)

入力と出力

  • **入力**:
  • sql: クエリ文字列を格納する PQExpBuffer。
  • rel: チェック対象のリレーション情報（OID、スキーマ名、一意性チェックフラグ等を含む）。
  • conn: データベース接続（本関数内では直接使用されていない）。
  • **出力**:
  • sql バッファの内容が上書きされ、構築された SQL クエリが格納される。

前提条件と不変条件

  • rel が指す RelationInfo 構造体は有効であること。
  • opts（グローバル変数）が適切に初期化されていること。
  • 構築される SQL は、contrib/amcheck 拡張機能がインストールされているデータベースに対して実行されることを前提としている。

アルゴリズムまたは振る舞い

1. resetPQExpBuffer(sql) を呼び出し、バッファを初期化する。 2. opts.parent_check の値に応じて、以下のいずれかの関数を呼び出す SQL を生成する：

  • true の場合: bt_index_parent_check を使用。
  • false の場合: bt_index_check を使用。

    • 3. opts.heapallindexed および opts.rootdescend の設定に基づき、SQL の引数を動的に構築する。 4. pg_catalog.pg_class および pg_catalog.pg_index を参照し、対象リレーションが一時テーブルではなく、かつ有効（indisready, indisvalid, indislive）であることを確認する条件句を付与する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_amcheck/pg_amcheck.c:792 にて呼び出されている。
  • **呼び出し先**: resetPQExpBuffer, appendPQExpBuffer（いずれも libpq 関連ユーティリティ）。

関連データ構造

  • PQExpBuffer: SQL 文字列構築用バッファ。
  • RelationInfo: 対象リレーションのメタデータ。
  • AmcheckOptions: ユーザー指定の実行オプション。

失敗モードと境界条件

  • rel->reloid が無効な場合、SQL は結果を返さない可能性がある。
  • 一時テーブル（RELPERSISTENCE_TEMP）はクエリの対象から明示的に除外される。
  • contrib/amcheck がデータベースに存在しない場合、実行時に SQL エラーが発生する。

保守担当者向けメモ

  • 本関数は contrib/amcheck のインターフェース変更に依存している。検証関数の引数やシグネチャが変更された場合は、本関数の SQL 構築ロジックも更新が必要である。
  • conn 引数は現在の実装では使用されていないが、将来的な拡張のために保持されている可能性がある。
