toast_datum_size

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/detoast.c:600

目的

与えられた Datum が指す可変長データ（varlena）の論理的なサイズ（復元後のサイズ）を算出します。

シグネチャまたは形状

Size toast_datum_size(Datum value)

入力と出力

  • **入力**: Datum value（対象となる可変長データへのポインタを含む Datum）
  • **出力**: Size（データが展開・復元された状態での論理的なバイトサイズ）

前提条件と不変条件

  • 入力 value は有効な varlena 構造体を指している必要があります。
  • 間接参照（EXTERNAL_INDIRECT）の場合、再帰的に解決されますが、ネストされた間接参照は許可されません（Assert により検証されます）。

アルゴリズムまたは振る舞い

入力データの形式に応じて以下の通りサイズを算出します。 1. **外部ストレージ（On-disk）**: TOASTポインタから外部データのサイズ（extsize）を取得します。 2. **間接参照（Indirect）**: ポインタを辿り、参照先のデータに対して再帰的に toast_datum_size を呼び出します。 3. **展開済みオブジェクト（Expanded）**: EOH_get_flat_size を使用してフラット化されたサイズを取得します。 4. **ショートヘッダー**: VARSIZE_SHORT を使用してサイズを取得します。 5. **その他（インライン）**: VARSIZE を使用して、圧縮の有無に関わらず格納されているサイズを取得します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/detoast.c (内部再帰)
  • src/backend/utils/adt/varlena.c:5301
  • **呼び出し先**:
  • toast_datum_size (自身を再帰呼び出し)
  • EOH_get_flat_size (展開済みオブジェクト用)
  • 各種 VARATT_* マクロおよび VARSIZE マクロ

関連データ構造

  • varlena: 可変長データの基本構造体。
  • varatt_external: 外部ストレージ用ポインタ構造体。
  • varatt_indirect: 間接参照用ポインタ構造体。

失敗モードと境界条件

  • ネストされた間接参照が検出された場合、アサーション違反により停止します。
  • 無効な Datum ポインタが渡された場合の動作は未定義です。

保守担当者向けメモ

  • 本関数はデータの物理的な格納サイズではなく、論理的なサイズを返すことに注意してください。
  • 外部ストレージのサイズ算出において、TOASTポインタ自体のサイズはカウントに含まれません。
