startup_progress_timeout_handler

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/startup.c:302

目的

起動処理の進捗監視用タイマーが期限切れとなったことを記録し、後続の処理で進捗ログの出力や監視を行うためのフラグを立てる。

シグネチャまたは形状

void startup_progress_timeout_handler(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数を更新）

前提条件と不変条件

  • startup_progress_timer_expired というグローバル変数が定義されていること。
  • 本関数はタイマー割り込みハンドラとして呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. グローバル変数 startup_progress_timer_expired を true に設定する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlog.c:5571 にて参照されている。
  • 呼び出し先: なし。

関連データ構造

  • startup_progress_timer_expired: 起動進捗タイマーの期限切れ状態を保持するブール値のフラグ。

失敗モードと境界条件

  • 特筆すべき失敗モードはない。タイマー割り込みのタイミングで確実にフラグを更新する。

保守担当者向けメモ

  • 本関数は utils/timeout.h で管理されるタイマーのコールバックとして使用される。
  • src/backend/access/transam/xlogrecovery.c:493 に本関数に関する言及があるが、直接的な呼び出し関係についてはさらなる調査が必要である。
