ValuesNext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeValuesscan.c:46

目的

VALUES 句で指定された定数リストを順次評価し、仮想タプルとして提供するスキャン処理を実行します。

シグネチャまたは形状

TupleTableSlot *ValuesNext(ValuesScanState *node)

入力と出力

  • **入力**: ValuesScanState *node（スキャン状態を保持するノード構造体）
  • **出力**: 次の行を格納した TupleTableSlot *。全行を処理し終えた場合は NULL を返します。

前提条件と不変条件

  • node->ss.ps.state から取得した es_direction に基づき、前方または後方スキャンを行います。
  • node->curr_idx は現在のスキャン位置を保持します。
  • 評価結果の Datum は、MakeExpandedObjectReadOnly により読み取り専用状態に強制されます。

アルゴリズムまたは振る舞い

1. **位置更新**: ScanDirection に応じて node->curr_idx を増減させます。 2. **スロット初期化**: ExecClearTuple を呼び出し、結果格納用スロットをクリアします。 3. **式評価**:

  • ReScanExprContext で前回の評価コンテキストをリセットします。
  • exprstatelist が未初期化の場合、ExecInitExprList で式状態を構築します（JITコンパイルは無効化されます）。
  • ExecEvalExpr を用いて各列の式を評価し、結果をスロットに格納します。

    • 4. **タプル格納**: ExecStoreVirtualTuple を呼び出し、仮想タプルとして確定させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeValuesscan.c 内の ExecScan インターフェース（ExecScanAccessMtd 経由）。
  • **呼び出し先**: ExecClearTuple, ReScanExprContext, MemoryContextSwitchTo, ExecInitExprList, ExecEvalExpr, MakeExpandedObjectReadOnly, ExecStoreVirtualTuple。

関連データ構造

  • ValuesScanState: スキャンノードの状態管理構造体。
  • TupleTableSlot: 仮想タプルを保持するコンテナ。
  • ExprContext: 式評価のためのメモリコンテキスト。

失敗モードと境界条件

  • **範囲外**: curr_idx が有効範囲外（0未満または array_len 以上）の場合、タプルを生成せず NULL を返します。
  • **メモリ管理**: 長大な VALUES リストに対してメモリ消費を抑えるため、行ごとの評価状態を動的に管理しています。

保守担当者向けメモ

  • 本関数は VALUES 句の式を一度だけ評価することを前提としており、JITコンパイルを意図的に無効化しています。
  • 評価結果の ExpandedObject は、上位ノードでの多重参照に備えて必ず読み取り専用化する必要があります。
