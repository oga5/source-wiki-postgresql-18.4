index_getattr

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/itup.h:131

目的

インデックスタプル（IndexTuple）から指定された属性番号の値を Datum 形式で抽出する。

シグネチャまたは形状

static inline Datum index_getattr(IndexTuple tup, int attnum, TupleDesc tupleDesc, bool *isnull)

入力と出力

  • **入力**:
  • tup: 対象のインデックスタプル。
  • attnum: 抽出する属性のインデックス（1始まり）。
  • tupleDesc: タプルの記述子。
  • isnull: NULL判定結果を格納するポインタ（出力用）。
  • **出力**:
  • 戻り値: 抽出された属性値（Datum）。
  • *isnull: 属性がNULLの場合は true、それ以外は false が設定される。

前提条件と不変条件

  • isnull は有効なポインタであること。
  • attnum は 1 以上であること。

アルゴリズムまたは振る舞い

1. isnull を false に初期化する。 2. IndexTupleHasNulls(tup) を確認し、NULL値が含まれない場合:

  • TupleDescCompactAttr を用いて属性情報を取得する。
  • attcacheoff が有効（0以上）であれば、キャッシュされたオフセットを使用して fetchatt で値を抽出する。
  • キャッシュが無効であれば、nocache_index_getattr を呼び出して値を抽出する。

    • 3. NULL値が含まれる可能性がある場合:

  • att_isnull を用いて当該属性がNULLか判定する。
  • NULLであれば *isnull を true にし、NULL を返す。
  • NULLでなければ nocache_index_getattr を呼び出して値を抽出する。

関連データ構造

  • IndexTuple: インデックスタプルの構造体。
  • TupleDesc: タプルの型情報を含む記述子。
  • Datum: 汎用的なデータ表現型。

失敗モードと境界条件

  • attnum が 1 未満の場合、アサーションエラーが発生する。
  • isnull に NULL ポインタを渡した場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、パフォーマンス向上のためにインライン化されている。
  • 複雑な属性位置計算は nocache_index_getattr に委譲されている。
  • IndexTuple のメモリレイアウト（ヘッダー、NULLビットマップ、データ部）に強く依存しているため、構造変更時は注意が必要。
