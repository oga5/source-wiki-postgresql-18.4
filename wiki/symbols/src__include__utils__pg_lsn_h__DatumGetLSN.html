DatumGetLSN

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/pg_lsn.h:21

目的

PostgreSQLの汎用データ型である Datum を、WAL（Write-Ahead Log）の位置を示す XLogRecPtr 型へ変換します。

シグネチャまたは形状

static inline XLogRecPtr DatumGetLSN(Datum X)

入力と出力

  • **入力**: Datum X（LSNとして解釈される値）
  • **出力**: XLogRecPtr（64ビットのログシーケンス番号）

前提条件と不変条件

  • 入力 Datum X は、内部的に int64 として格納された有効なLSN値であることを前提とします。
  • XLogRecPtr は uint64 と同等であり、本関数は DatumGetInt64 を介してビット列をキャストします。

アルゴリズムまたは振る舞い

1. DatumGetInt64(X) を呼び出し、Datum 型から int64 型へ変換します。 2. 得られた値を XLogRecPtr 型へキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_subscription.c
  • src/backend/commands/subscriptioncmds.c
  • src/backend/replication/libpqwalreceiver/libpqwalreceiver.c
  • src/backend/replication/logical/slotsync.c
  • src/include/utils/pg_lsn.h (マクロ PG_GETARG_LSN 内)
  • **呼び出し先**:
  • DatumGetInt64 (PostgreSQLの fmgr インターフェース)

関連データ構造

  • XLogRecPtr: LSNを保持する基底型（access/xlogdefs.h で定義）。
  • Datum: PostgreSQLの汎用データ表現型。

失敗モードと境界条件

  • 本関数自体は型変換のみを行うため、エラーチェックは行いません。
  • Datum が不正な値である場合や、期待される型と異なる場合は、呼び出し元での整合性確保が必要です。

保守担当者向けメモ

  • 本関数はインライン化されており、SQL関数内で引数を取得する際の標準的な手段として PG_GETARG_LSN マクロ経由で多用されます。変更時は fmgr のデータ変換規約に影響がないか確認してください。
