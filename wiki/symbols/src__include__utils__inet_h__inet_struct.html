inet_struct

定義

  • 種別: struct
  • 定義箇所: src/include/utils/inet.h:23

目的

PostgreSQLにおけるネットワークアドレス（inet型およびcidr型）の内部メモリ表現を定義する構造体です。

シグネチャまたは形状

typedef struct
{
	unsigned char family;		/* PGSQL_AF_INET or PGSQL_AF_INET6 */
	unsigned char bits;			/* number of bits in netmask */
	unsigned char ipaddr[16];	/* up to 128 bits of address */
} inet_struct;

入力と出力

  • 本構造体は直接の関数引数ではなく、inet型（varlena形式）のデータ領域としてメモリ上に配置されます。
  • VARDATA_ANY マクロを介して、inet型データのペイロード部分を inet_struct* にキャストしてアクセスします。

前提条件と不変条件

  • family フィールドには PGSQL_AF_INET または PGSQL_AF_INET6 が設定されている必要があります。
  • ipaddr は最大128ビット（16バイト）を保持可能です。
  • inet型として扱う場合、メモリ確保時にゼロ初期化を行うことで、VARDATA_ANY がヘッダーを正しく認識できるようにする必要があります。

アルゴリズムまたは振る舞い

  • inet型データの各属性（ファミリー、ビット数、アドレス）へアクセスするための抽象化マクロ（ip_family, ip_bits, ip_addr）の基盤となります。
  • 可変長データ（varlena）のヘッダーサイズ計算において、offsetof(inet_struct, ipaddr) を用いてアドレス部分のオフセットを算出します。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/utils/inet.h 内の各種アクセスマクロ（ip_family等）および SET_VARSIZE を含むサイズ計算マクロから参照されます。
  • 呼び出し先: 本構造体はデータ定義であり、特定の関数を呼び出すものではありません。

関連データ構造

  • inet: 本構造体をラップした varlena 形式のデータ型。
  • macaddr, macaddr8: ネットワーク関連の別データ型（本構造体とは独立した固定長構造体）。

失敗モードと境界条件

  • family フィールドが不正な値である場合、後続のネットワーク処理で予期せぬ動作を引き起こす可能性があります。
  • ipaddr のサイズ（16バイト）を超えるアドレス情報を格納することはできません。

保守担当者向けメモ

  • inet型は1バイトヘッダーの varlena 形式を優先しますが、4バイトヘッダーとの互換性も考慮されています。
  • ip_addrsize() や SET_INET_VARSIZE() を使用する際は、family フィールドの整合性を必ず確認してください。
