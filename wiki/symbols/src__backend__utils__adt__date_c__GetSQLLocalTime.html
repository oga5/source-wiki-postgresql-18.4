GetSQLLocalTime

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/date.c:369

目的

SQLの LOCALTIME および LOCALTIME(n) 式を実装し、現在のローカル時刻を TimeADT 型として取得する。

シグネチャまたは形状

TimeADT GetSQLLocalTime(int32 typmod)

入力と出力

  • **入力**: typmod (int32) - 時刻の精度（小数点以下の桁数）を指定する修飾子。
  • **出力**: TimeADT - 現在のローカル時刻を表すマイクロ秒単位の整数値。

前提条件と不変条件

  • typmod は TIME 型の有効な精度範囲内である必要がある。
  • GetCurrentTimeUsec が現在の時刻情報を正しく取得できる環境であること。

アルゴリズムまたは振る舞い

1. GetCurrentTimeUsec を呼び出し、現在の時刻（tm）、小数秒（fsec）、およびタイムゾーン（tz）を取得する。 2. tm2time を使用して、取得した時刻情報を TimeADT 形式（マイクロ秒単位）に変換する。 3. AdjustTimeForTypmod を呼び出し、指定された typmod に基づいて時刻の精度を調整する。 4. 調整済みの TimeADT 値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c:3229 (式評価時の実行)
  • **呼び出し先**:
  • GetCurrentTimeUsec (時刻取得)
  • tm2time (時刻構造体から内部表現への変換)
  • AdjustTimeForTypmod (精度調整)

関連データ構造

  • TimeADT: 時刻を保持する int64 型の内部表現。
  • pg_tm: 日付・時刻の各フィールドを保持する構造体。

失敗モードと境界条件

  • typmod が不正な場合、AdjustTimeForTypmod 内でエラーが発生する可能性がある。
  • システム時刻の取得に失敗した場合の挙動は GetCurrentTimeUsec に依存する。

保守担当者向けメモ

  • 本関数は LOCALTIME の実装であるため、タイムゾーンオフセットを考慮しないローカル時刻の取得に特化している。
  • TimeADT は int64 であり、マイクロ秒単位で管理されている点に留意すること。
