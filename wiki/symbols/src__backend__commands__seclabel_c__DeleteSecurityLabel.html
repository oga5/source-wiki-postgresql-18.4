DeleteSecurityLabel

定義

  • 種別: function
  • 定義箇所: src/backend/commands/seclabel.c:522

目的

指定されたデータベースオブジェクトに関連付けられているすべてのセキュリティラベルをシステムカタログから削除します。

シグネチャまたは形状

void DeleteSecurityLabel(const ObjectAddress *object)

入力と出力

  • **入力**: object (削除対象のオブジェクトを特定する ObjectAddress 構造体へのポインタ)
  • **出力**: なし（副作用としてシステムカタログを更新）

前提条件と不変条件

  • object が共有リレーション（IsSharedRelation）である場合、objectSubId は 0 である必要があります。
  • pg_seclabel カタログに対する RowExclusiveLock を取得して操作を行います。

アルゴリズムまたは振る舞い

1. **共有オブジェクトの判定**: IsSharedRelation を使用し、共有オブジェクトであれば DeleteSharedSecurityLabel を呼び出して終了します。 2. **スキャンキーの初期化**: 対象オブジェクトの objectId および classId をキーとして設定します。objectSubId が 0 でない場合は、それも検索条件に含めます。 3. **カタログ削除**: pg_seclabel テーブルをスキャンし、一致するすべてのタプルを CatalogTupleDelete を用いて削除します。 4. **リソース解放**: スキャンを終了し、テーブルをクローズします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/dependency.c:1334
  • **呼び出し先**:
  • DeleteSharedSecurityLabel (共有オブジェクトの場合)
  • table_open / table_close
  • systable_beginscan / systable_getnext / systable_endscan
  • CatalogTupleDelete

関連データ構造

  • ObjectAddress: 削除対象を特定するための構造体。
  • pg_seclabel: セキュリティラベルを保持するシステムカタログ。

失敗モードと境界条件

  • 対象オブジェクトにラベルが存在しない場合、削除処理は何も行わず正常終了します。
  • 共有オブジェクトと通常オブジェクトで、アクセスするカタログ（pg_shseclabel vs pg_seclabel）が自動的に切り替わります。

保守担当者向けメモ

  • 本関数はラベルの「削除」のみを担当します。ラベルの検証や権限チェックは呼び出し元で行われることを前提としています。
  • objectSubId が 0 の場合はオブジェクト全体、0 以外の場合は特定の列や属性を指すものとして処理されます。
