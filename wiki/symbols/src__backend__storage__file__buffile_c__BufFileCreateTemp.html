BufFileCreateTemp

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/buffile.c:192

目的

一時的なデータ保持に使用する新しい BufFile オブジェクトを作成し、初期化します。

シグネチャまたは形状

BufFile *BufFileCreateTemp(bool interXact)

入力と出力

  • **入力**:
  • interXact (bool): トランザクションを跨いでファイルを保持するかどうかを指定します。
  • **出力**:
  • 戻り値: 初期化された BufFile 構造体へのポインタ。

前提条件と不変条件

  • PrepareTempTablespaces() が呼び出され、一時テーブルスペースが設定されている必要があります。
  • OpenTemporaryFile() が成功し、有効なファイル記述子（pfile >= 0）が取得できる必要があります。

アルゴリズムまたは振る舞い

1. PrepareTempTablespaces() を呼び出し、一時ファイル用のテーブルスペース設定を保証します。 2. OpenTemporaryFile(interXact) を呼び出し、一時ファイルを作成して物理的なファイル記述子を取得します。 3. makeBufFile(pfile) を呼び出し、取得したファイル記述子をラップする BufFile 構造体を割り当てます。 4. file->isInterXact に引数 interXact の値を設定します。 5. 初期化済みの BufFile ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistbuildbuffers.c
  • src/backend/backup/backup_manifest.c
  • src/backend/executor/nodeHashjoin.c
  • src/backend/utils/sort/logtape.c
  • src/backend/utils/sort/tuplestore.c
  • **呼び出し先**:
  • PrepareTempTablespaces()
  • OpenTemporaryFile()
  • makeBufFile()

関連データ構造

  • BufFile: 本関数が生成するバッファ付きファイル管理構造体。

失敗モードと境界条件

  • OpenTemporaryFile() が失敗した場合、内部の Assert(pfile >= 0) によりアボートします。
  • メモリ不足時には makeBufFile() 内で palloc がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数は一時ファイル作成の標準的なエントリポイントです。
  • interXact フラグは、リソース所有権やファイル削除のタイミングを制御するために重要です。
