splitTableColumnsSpec

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/common.c:32

目的

TABLE[(COLUMNS)] 形式の文字列を解析し、テーブル名部分とカラム指定部分に分離します。

シグネチャまたは形状

void splitTableColumnsSpec(const char *spec, int encoding, char **table, const char **columns)

入力と出力

  • spec: 解析対象の文字列（TABLE または TABLE(COLUMNS)）。
  • encoding: 文字列のエンコーディング（PQmblenBounded で使用）。
  • table: 出力用。分離されたテーブル名が格納される（呼び出し元で解放が必要）。
  • columns: 出力用。spec 内の ( 以降の文字列を指すポインタ。

前提条件と不変条件

  • spec はヌル終端文字列であること。
  • 識別子内の引用符（"）は PostgreSQL のルールに従って処理されます。

アルゴリズムまたは振る舞い

1. 文字列を先頭から走査し、引用符で囲まれた範囲をスキップしながら、最初の ( を探します。 2. 引用符内の ""（エスケープされた引用符）は、引用状態を反転させずにスキップします。 3. ( が見つかった場合、その直前までを table として pnstrdup で複製します。 4. columns には、見つかった ( 以降の文字列へのポインタを格納します。( が存在しない場合は、文字列の終端を指します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/scripts/common.c
  • src/bin/scripts/vacuumdb.c
  • **呼び出し先**:
  • PQmblenBounded (PostgreSQL フロントエンドライブラリ)
  • pnstrdup (メモリ割り当て関数)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • ( が含まれない場合、table には入力文字列全体がコピーされ、columns は終端（\0）を指します。
  • 引用符が正しく閉じられていない場合、文字列の最後まで走査されます。

保守担当者向けメモ

  • 本関数は PostgreSQL の識別子引用符ルールに基づいています。引用符のネストやマルチバイト文字の処理については、fe_utils/string_utils.h 等の関連ユーティリティとの整合性に注意してください。
