pg_read_file_all

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/genfile.c:318

目的

サーバー上の指定されたファイル全体を読み込み、テキストデータとして返します。

シグネチャまたは形状

Datum pg_read_file_all(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介してファイルパス（text 型）を受け取ります。
  • **出力**: 読み込んだファイルの内容を text 型として返します。ファイルが存在しない場合は NULL を返します。

前提条件と不変条件

  • 呼び出し元は PostgreSQL の関数呼び出し規約に従う必要があります。
  • 内部で pg_read_file_common を使用しており、実行には pg_read_server_files ロールの権限が必要です。

アルゴリズムまたは振る舞い

1. 引数からファイルパスを取得します。 2. pg_read_file_common を呼び出し、以下の引数を渡します。

  • offset: 0（ファイルの先頭から開始）
  • length: -1（ファイル全体を読み込む）
  • missing_ok: true（ファイルが存在しなくてもエラーにしない）
  • include_no_newline: false

    • 3. pg_read_file_common の戻り値が NULL の場合、PG_RETURN_NULL() を実行します。 4. 読み込みに成功した場合、その内容を PG_RETURN_TEXT_P(ret) で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェースから呼び出されることを想定しています（具体的な SQL 呼び出し元はソースコード上では特定されていません）。
  • **呼び出し先**: pg_read_file_common を呼び出します。

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • PG_FUNCTION_ARGS: 関数呼び出しの引数コンテキスト。

失敗モードと境界条件

  • 指定されたファイルが存在しない場合、missing_ok=true の指定によりエラーにはならず、NULL が返されます。
  • 権限がない場合や、パスがデータディレクトリ外を指している場合は、内部の convert_and_check_filename によりエラーが発生します。

保守担当者向けメモ

  • 本関数は pg_read_file_all_missing と同様のロジックを共有しています。機能変更時は pg_read_file_common の仕様変更が影響しないか確認してください。
  • missing_ok フラグが true に設定されているため、ファイル不在時の挙動は呼び出し元で適切に処理される前提となっています。
