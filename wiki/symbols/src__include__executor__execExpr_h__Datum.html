Datum

定義

  • 種別: struct（src/include/executor/execExpr.h:378 における constval 構造体の一部として定義）
  • 補足: nodes/execnodes.h:230 に typedef が存在します。

目的

PostgreSQLの実行エンジンにおいて、SQLの定数値を保持・表現するために使用されます。

シグネチャまたは形状

struct
{
    Datum value;
    bool  isnull;
} constval;

入力と出力

  • 入力: 式評価の初期化時、定数ノードから抽出された値が value に格納されます。
  • 出力: 式評価の実行時、この構造体から value および isnull フラグが読み取られ、後続の評価ステップへ渡されます。

前提条件と不変条件

  • value は、対応するデータ型の内部表現を保持します。
  • isnull が true の場合、value の内容は無視されるか、特定の無効値として扱われます。

アルゴリズムまたは振る舞い

式評価の実行中、定数評価ステップ（EEOP_CONST）において、この構造体に格納された value と isnull が評価結果として出力スロットにコピーされます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execExpr.c 内の式コンパイル処理（ExecInitExpr 等）により初期化されます。
  • 呼び出し先: src/backend/executor/execExprInterp.c 内のインタープリタ実行ループから参照されます。
  • 備考: contrib/amcheck 配下の各ファイル（verify_gin.c, verify_heapam.c, verify_nbtree.c）において、データ値の検証や比較のために Datum 型が広範に参照されています。

関連データ構造

  • ExprEvalStep: 式評価の各ステップを定義する構造体。constval はこの構造体内の union d の一部として保持されます。

失敗モードと境界条件

  • Datum 型のサイズはプラットフォームに依存します（通常はポインタサイズ）。
  • isnull フラグの管理を誤ると、NULL許容の式評価において誤った結果を返す可能性があります。

保守担当者向けメモ

  • Datum はPostgreSQL全体で多用される汎用型です。execExpr.h 内の constval 構造体は、式評価における定数保持に特化したコンテキストで使用されています。
  • 新しい命令を追加する際は、ExprEvalStep のサイズ制約（キャッシュライン最適化）を考慮し、constval の配置がメモリ効率に影響しないか確認してください。
