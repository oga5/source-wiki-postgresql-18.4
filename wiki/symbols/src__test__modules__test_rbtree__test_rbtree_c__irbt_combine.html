irbt_combine

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_rbtree/test_rbtree.c:51

目的

PostgreSQL の赤黒木ライブラリにおいて、挿入しようとしたキーが既に木に存在する場合の重複処理（結合）を定義するコールバック関数です。

シグネチャまたは形状

static void irbt_combine(RBTNode *existing, const RBTNode *newdata, void *arg);

入力と出力

  • existing: 既に木に格納されている既存のノード（RBTNode）。
  • newdata: 新たに挿入しようとしたノード（RBTNode）。
  • arg: 呼び出し元から渡される補助引数（本実装では未使用）。
  • 戻り値: なし。

前提条件と不変条件

  • existing および newdata は、IntRBTreeNode 構造体としてキャスト可能である必要があります。
  • 本関数は、赤黒木ライブラリが重複キーを検出した際に呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. existing と newdata を IntRBTreeNode 型にキャストします。 2. 両者の key 値を比較します。 3. キーが一致しない場合、elog(ERROR, ...) を呼び出し、テスト失敗として処理を中断します。キーが一致する場合は何も行わず正常終了します。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/test_rbtree/test_rbtree.c 内の赤黒木構築処理（rbt_populate 等）から、rbt_create を介して赤黒木ライブラリへ渡されるコールバックとして参照されます。
  • 呼び出し先: elog（PostgreSQL のエラー報告関数）。

関連データ構造

  • IntRBTreeNode: 本テストモジュールで定義された、RBTNode を継承する整数キー保持用のノード構造体。

失敗モードと境界条件

  • existing->key != newdata->key: 異なるキーを持つノードが重複として扱われた場合、エラーが発生します。これはテストの整合性チェックとして機能します。

保守担当者向けメモ

  • 本関数はテスト専用のコールバックです。実運用コードにおける赤黒木の結合ロジックとは異なり、キーの不一致を致命的なエラーとして報告することで、ライブラリの重複判定ロジックの正当性を検証しています。
