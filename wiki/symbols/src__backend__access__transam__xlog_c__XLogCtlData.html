XLogCtlData

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/xlog.c:451

目的

PostgreSQL の WAL（Write-Ahead Log）管理における共有メモリ上の制御状態を保持する主要なデータ構造です。WAL の挿入、フラッシュ、リカバリ状態、およびチェックポイント情報を一元管理します。

シグネチャまたは形状

typedef struct XLogCtlData {
    XLogCtlInsert Insert;
    XLogwrtRqst   LogwrtRqst;
    XLogRecPtr    RedoRecPtr;
    FullTransactionId ckptFullXid;
    XLogRecPtr    asyncXactLSN;
    XLogRecPtr    replicationSlotMinLSN;
    XLogSegNo     lastRemovedSegNo;
    pg_atomic_uint64 unloggedLSN;
    pg_time_t     lastSegSwitchTime;
    XLogRecPtr    lastSegSwitchLSN;
    pg_atomic_uint64 logInsertResult;
    pg_atomic_uint64 logWriteResult;
    pg_atomic_uint64 logFlushResult;
    XLogRecPtr    InitializedUpTo;
    char         *pages;
    pg_atomic_uint64 *xlblocks;
    int           XLogCacheBlck;
    TimeLineID    InsertTimeLineID;
    TimeLineID    PrevTimeLineID;
    RecoveryState SharedRecoveryState;
    bool          InstallXLogFileSegmentActive;
    bool          WalWriterSleeping;
    XLogRecPtr    lastCheckPointRecPtr;
    XLogRecPtr    lastCheckPointEndPtr;
    CheckPoint    lastCheckPoint;
    XLogRecPtr    lastFpwDisableRecPtr;
    slock_t       info_lck;
} XLogCtlData;

入力と出力

  • 本構造体自体はデータ保持用であり、関数ではありません。
  • 共有メモリ上のグローバル変数 XLogCtl を通じて、WAL 関連の各プロセス（バックエンド、チェックポインター、WAL ライター等）が状態を読み書きします。

前提条件と不変条件

  • info_lck (スピンロック) が、構造体内の多くの共有変数（LogwrtRqst, RedoRecPtr, SharedRecoveryState 等）の保護に使用されます。
  • logInsertResult, logWriteResult, logFlushResult はアトミック操作でアクセスされ、info_lck を必要としません。
  • xlblocks および InitializedUpTo の変更には WALBufMappingLock が必要です。
  • InsertTimeLineID 等のタイムライン情報は、リカバリ終了後は不変となります。

アルゴリズムまたは振る舞い

  • **WAL バッファ管理**: pages 配列と xlblocks を用いて、WAL ページをキャッシュします。
  • **同期**: info_lck による排他制御と、アトミック変数による高頻度な LSN 更新を組み合わせて並列性を確保します。
  • **リカバリ状態管理**: SharedRecoveryState を通じて、システムが現在クラッシュリカバリ中かアーカイブリカバリ中かを追跡します。

呼び出し元と呼び出し先

  • 呼び出し元: XLOGShmemInit で初期化され、XLogInsertRecord, XLogFlush, AdvanceXLInsertBuffer 等の WAL 管理関数がこの構造体を参照・更新します。
  • 呼び出し先: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • XLogCtlInsert: WAL 挿入の制御情報。
  • XLogwrtRqst, XLogwrtResult: WAL の書き込み要求と結果。
  • CheckPoint: チェックポイントのメタデータ。
  • RecoveryState: リカバリ状態の列挙型。

失敗モードと境界条件

  • 共有メモリの初期化失敗時、XLogCtl は NULL のままとなる可能性があります。
  • XLogCacheBlck を超えるバッファアクセスは発生しないよう、AdvanceXLInsertBuffer 等で制御されています。
  • 競合するロック（WALWriteLock 等）の取得順序を誤るとデッドロックのリスクがあります。

保守担当者向けメモ

  • XLogCtlData は PostgreSQL の WAL 処理の心臓部です。フィールドを追加・変更する際は、どのロック（info_lck、WALBufMappingLock、ControlFileLock 等）で保護されているかを厳密に確認してください。
  • sizeof(XLogCtlData) は共有メモリ割り当て時に使用されるため、構造体のサイズ変更は共有メモリレイアウトに影響を与えます。
