binary_decode

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/encode.c:95

目的

SQLの decode() 関数を実装し、指定されたエンコーディング方式（HEX, Base64, Escape等）に基づいてテキストデータをバイナリ（bytea）へ変換します。

シグネチャまたは形状

Datum binary_decode(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_PP(0): デコード対象のテキストデータ。
  • PG_GETARG_DATUM(1): エンコーディング方式を指定する文字列（例: 'hex', 'base64'）。
  • **出力**:
  • PG_RETURN_BYTEA_P(result): デコードされたバイナリデータ（bytea）。

前提条件と不変条件

  • 指定されたエンコーディング名が pg_find_encoding で解決可能であること。
  • デコード後のデータサイズが MaxAllocSize - VARHDRSZ を超えないこと。
  • デコード処理後の実際のデータ長が、事前に計算した見積もりサイズ（resultlen）以下であること。

アルゴリズムまたは振る舞い

1. TextDatumGetCString でエンコーディング名を取得し、pg_find_encoding で対応する関数テーブルを検索します。 2. enc->decode_len を呼び出し、デコード後のバッファサイズを算出します。 3. MaxAllocSize を基準にメモリ不足をチェックし、palloc で結果用メモリを確保します。 4. enc->decode を呼び出し、入力データをデコードして確保したメモリ領域に書き込みます。 5. 実際の書き込みサイズが見積もりを超えていないか検証し、SET_VARSIZE で最終的なサイズを設定して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます（直接のC言語レベルの呼び出し元はソースコード上では明示されていません）。
  • **呼び出し先**:
  • pg_find_encoding: エンコーディング方式の検索。
  • enc->decode_len: サイズ計算用関数ポインタ。
  • enc->decode: デコード実行用関数ポインタ。

関連データ構造

  • pg_encoding: エンコーディング方式ごとの関数ポインタ（decode, decode_len 等）を保持する構造体。
  • bytea: PostgreSQLの可変長バイナリデータ型。

失敗モードと境界条件

  • **認識できないエンコーディング**: pg_find_encoding が NULL を返した場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告します。
  • **メモリ制限超過**: デコード結果が MaxAllocSize を超える場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラーを報告します。
  • **バッファオーバーフロー**: デコード処理後のサイズが事前の見積もりを超えた場合、FATAL エラーでプロセスを停止します。

保守担当者向けメモ

  • 新しいエンコーディング方式を追加する場合は、enclist 配列に定義を追加してください。
  • 32bit環境でのメモリ安全性を確保するため、サイズ計算には常に uint64 を使用し、MaxAllocSize との比較を厳密に行う必要があります。
