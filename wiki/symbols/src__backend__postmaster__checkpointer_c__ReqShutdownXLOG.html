ReqShutdownXLOG

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/checkpointer.c:920

目的

チェックポインタープロセスに対して、データベースのシャットダウン処理（XLOGの終了処理）を要求するシグナルハンドラです。

シグネチャまたは形状

static void ReqShutdownXLOG(SIGNAL_ARGS)

入力と出力

  • **入力**: SIGNAL_ARGS（シグナルハンドラの標準引数）
  • **出力**: なし（副作用としてグローバル変数とラッチを更新）

前提条件と不変条件

  • 本関数はシグナルハンドラとして呼び出されることを前提としています。
  • MyLatch が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. グローバル変数 ShutdownXLOGPending を true に設定し、シャットダウン要求が保留中であることを記録します。 2. SetLatch(MyLatch) を呼び出し、チェックポインターのメインループが待機状態から即座に復帰して要求を処理できるようにします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/checkpointer.c 内で SIGINT シグナルのハンドラとして登録されています。
  • **呼び出し先**: SetLatch（PostgreSQLのラッチ管理関数）。

関連データ構造

  • ShutdownXLOGPending（グローバルなフラグ変数）
  • MyLatch（現在のプロセスのラッチ）

失敗モードと境界条件

  • シグナルハンドラ内であるため、非同期安全な操作のみを行う必要があります。本関数はフラグ設定とラッチ操作のみを行うため、この制約を満たしています。

保守担当者向けメモ

  • 本関数は SIGINT を受信した際に実行されます。チェックポインターがこのシグナルを受け取った場合、通常のチェックポイント処理ではなく、シャットダウンに向けたXLOG処理へ移行するトリガーとなります。
