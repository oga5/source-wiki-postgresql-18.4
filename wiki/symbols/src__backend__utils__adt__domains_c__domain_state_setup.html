domain_state_setup

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/domains.c:75

目的

ドメイン型の入出力処理および制約検証に必要な情報を初期化し、DomainIOData 構造体を構築する。

シグネチャまたは形状

DomainIOData *domain_state_setup(Oid domainType, bool binary, MemoryContext mcxt)

入力と出力

  • **入力**:
  • domainType: 対象となるドメイン型の OID。
  • binary: バイナリ形式の入出力を行う場合は true、テキスト形式の場合は false。
  • mcxt: 構造体の割り当てに使用するメモリコンテキスト。
  • **出力**:
  • 初期化済みの DomainIOData 構造体へのポインタ。

前提条件と不変条件

  • domainType は有効なドメイン型である必要がある。
  • mcxt は DomainIOData を保持するのに十分な生存期間を持つ必要がある。

アルゴリズムまたは振る舞い

1. 指定されたメモリコンテキストに DomainIOData を割り当てる。 2. lookup_type_cache を使用してドメインの基底型情報を取得し、対象がドメイン型であることを検証する（不一致時はエラー）。 3. 基底型の入出力関数情報を取得し、fmgr_info_cxt を用いて関数情報をキャッシュする。 4. InitDomainConstraintRef を呼び出し、ドメインに関連付けられた制約情報を初期化する。 5. 式評価用の econtext を NULL に初期化し、キャッシュを有効として構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/domains.c 内の domain_in および domain_recv 関数から呼び出される。
  • **呼び出し先**:
  • lookup_type_cache
  • getTypeBinaryInputInfo / getTypeInputInfo
  • fmgr_info_cxt
  • InitDomainConstraintRef

関連データ構造

  • DomainIOData: ドメインのI/O状態、基底型情報、制約参照、実行コンテキストを保持する。
  • TypeCacheEntry: 型のキャッシュ情報を保持する。

失敗モードと境界条件

  • domainType がドメイン型ではない場合、ERRCODE_DATATYPE_MISMATCH でエラーを報告する。
  • lookup_type_cache が失敗した場合、システムカタログの整合性エラーとして例外が発生する可能性がある。

保守担当者向けメモ

  • 本関数は FmgrInfo の fn_extra にキャッシュされる DomainIOData の生成元である。
  • ExprContext は遅延初期化されるため、本関数内では NULL に設定されている。
  • メモリ管理は mcxt に依存するため、呼び出し側で適切なコンテキストが渡されていることを確認すること。
