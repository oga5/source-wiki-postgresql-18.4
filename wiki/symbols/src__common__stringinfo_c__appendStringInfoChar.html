appendStringInfoChar

定義

  • 種別: function
  • 定義箇所: src/common/stringinfo.c:241

目的

StringInfo バッファの末尾に単一の文字を追加し、バッファのヌル終端を維持する。

シグネチャまたは形状

void appendStringInfoChar(StringInfo str, char ch)

入力と出力

  • **入力**:
  • str: 操作対象の StringInfo 構造体へのポインタ。
  • ch: 追加する文字。
  • **出力**: なし（副作用として str が指すバッファの内容とメタデータが更新される）。

前提条件と不変条件

  • str は有効な StringInfo 構造体を指していること。
  • str->maxlen > 0 であること（読み取り専用バッファに対して呼び出した場合、内部の enlargeStringInfo がアサーションエラーを引き起こす可能性がある）。
  • 実行後、str->data は常にヌル終端されていること。

アルゴリズムまたは振る舞い

1. 現在のバッファ容量（maxlen）に対して、新しい文字とヌル終端文字を格納する十分な空きがあるか確認する。 2. 空きが不足している場合、enlargeStringInfo を呼び出してバッファを拡張する。 3. str->data の現在の末尾位置（len）に文字 ch を書き込む。 4. str->len をインクリメントする。 5. 新しい末尾位置にヌル終端文字 '\0' を書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/cube/cube.c
  • contrib/dblink/dblink.c
  • contrib/hstore/hstore_io.c
  • **呼び出し先**:
  • enlargeStringInfo（バッファ拡張が必要な場合）

関連データ構造

  • StringInfoData（StringInfo はこの構造体へのポインタ）: 文字列バッファ、現在の長さ、最大容量、カーソル位置を保持する。

失敗モードと境界条件

  • **メモリ不足**: enlargeStringInfo 経由で palloc/repalloc が失敗した場合、メモリ不足エラーが発生する。
  • **読み取り専用バッファ**: maxlen が 0 のバッファに対して呼び出すと、enlargeStringInfo 内でアサーション失敗または不正なメモリ操作が発生する。

保守担当者向けメモ

  • 本関数は StringInfo の基本的な追加操作であり、頻繁に呼び出される。
  • ヌル終端を保証するため、バイナリデータ用関数（appendBinaryStringInfoNT 等）とは異なり、常に len + 1 バイトの空きを確保する設計となっている。
