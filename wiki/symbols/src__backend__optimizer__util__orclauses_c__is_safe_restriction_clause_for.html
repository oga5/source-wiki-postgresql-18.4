is_safe_restriction_clause_for

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/orclauses.c:125

目的

指定された RestrictInfo が、特定の RelOptInfo に対して制限条件として安全に使用できるかを判定します。

シグネチャまたは形状

static bool is_safe_restriction_clause_for(RestrictInfo *rinfo, RelOptInfo *rel)

入力と出力

  • **入力**:
  • rinfo: 評価対象の RestrictInfo 構造体。
  • rel: 制限条件を適用する対象の RelOptInfo 構造体。
  • **出力**:
  • bool: 条件が安全であれば true、そうでなければ false を返します。

前提条件と不変条件

  • rinfo および rel は有効なポインタである必要があります。
  • 判定対象の条件式が、対象リレーション以外の変数を含まないこと、および揮発性関数を含まないことが求められます。

アルゴリズムまたは振る舞い

以下の条件を順に確認し、すべてを満たす場合にのみ true を返します。 1. **擬似定数の除外**: rinfo->pseudoconstant が真であれば false を返します。 2. **リレーションの照合**: rinfo->clause_relids（条件が参照するリレーションの集合）が、対象 rel->relids と完全に一致することを確認します。一致しない場合は false を返します。 3. **揮発性の確認**: contain_volatile_functions を使用し、条件式内に揮発性関数が含まれていないかを確認します。含まれている場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/orclauses.c 内の extract_or_clause 関数から呼び出されます。
  • **呼び出し先**:
  • bms_equal (ビットマップセットの比較)
  • contain_volatile_functions (式ツリー内の揮発性関数検出)

関連データ構造

  • RestrictInfo: 制限条件を保持する構造体。
  • RelOptInfo: プランナが扱うリレーションの情報を保持する構造体。

失敗モードと境界条件

  • rinfo が擬似定数である場合、または対象リレーション以外のテーブルを参照している場合は、安全ではないとみなされ false が返されます。
  • 揮発性関数を含む条件は、予期せぬ副作用を避けるため除外されます。

保守担当者向けメモ

  • 本関数は、OR条件の分解過程において、抽出されたサブ条件が単一リレーションに対して安全に適用可能かを保証する重要なフィルタです。
  • contain_volatile_functions の呼び出しは、クエリ実行時の副作用を防止するために不可欠です。
