BTWriteState

定義

  • 種別: struct
  • 定義箇所: src/backend/access/nbtree/nbtsort.c:246

目的

B-Treeインデックスのバルクロード（一括構築）処理において、書き込み対象のインデックスリレーション、ヒープリレーション、およびバッファ管理状態を保持するために使用されます。

シグネチャまたは形状

typedef struct BTWriteState
{
	Relation	heap;
	Relation	index;
	BulkWriteState *bulkstate;
	BTScanInsert inskey;
	BlockNumber btws_pages_alloced;
} BTWriteState;

入力と出力

本構造体は関数引数として渡され、以下の情報を管理します。

  • **入力**: heap（ソース）、index（ターゲット）、bulkstate（書き込み用バッファ状態）。
  • **出力**: btws_pages_alloced（構築中に割り当てられたページ数の追跡）。

前提条件と不変条件

  • bulkstate は storage/bulk_write.h で定義された効率的な書き込みメカニズムと連携している必要があります。
  • inskey はインデックス挿入時のスキャンキーとして機能します。

アルゴリズムまたは振る舞い

  • インデックス構築のメインループにおいて、_bt_load 等の関数に渡され、ページ割り当て（_bt_blnewpage）やページ書き出し（_bt_blwritepage）のコンテキストとして機能します。
  • 構築プロセス全体を通じて、割り当てられたページ数をインクリメントし、インデックスの物理的な構築状態を追跡します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtsort.c 内のインデックス構築プロセス（btbuild 等）。
  • **呼び出し先**: 以下の関数が BTWriteState を引数として受け取ります。
  • _bt_blnewpage
  • _bt_blwritepage
  • _bt_pagestate
  • _bt_buildadd
  • _bt_sort_dedup_finish_pending
  • _bt_uppershutdown
  • _bt_load

関連データ構造

  • Relation: ヒープおよびインデックスのメタデータ。
  • BulkWriteState: バッファキャッシュをバイパスした書き込み状態。
  • BTScanInsert: インデックス挿入用のスキャンキー。
  • BTPageState: 構築中の各レベルの状態管理。

失敗モードと境界条件

  • bulkstate の初期化失敗や、メモリ不足による割り当て失敗が想定されます。
  • ページ割り当て数（btws_pages_alloced）がインデックスの最大サイズ制限を超過する場合の挙動は、呼び出し側のロジックに依存します。

保守担当者向けメモ

  • 本構造体は nbtsort.c 内のバルクロード専用です。通常の nbtinsert.c による挿入処理とは異なるパスである点に注意してください。
  • btws_pages_alloced は統計情報やデバッグ目的で利用されることが多いため、変更時は関連するログ出力や統計集計への影響を確認してください。
