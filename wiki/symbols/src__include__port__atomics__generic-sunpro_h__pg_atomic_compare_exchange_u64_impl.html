pg_atomic_compare_exchange_u64_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/generic-sunpro.h:92

目的

Solaris 環境において、64ビット整数のアトミックな比較交換（CAS）操作を提供します。

シグネチャまたは形状

static inline bool pg_atomic_compare_exchange_u64_impl(volatile pg_atomic_uint64 *ptr, uint64 *expected, uint64 newval)

入力と出力

  • **入力**:
  • ptr: 操作対象となる pg_atomic_uint64 構造体へのポインタ。
  • expected: 期待値を保持する変数へのポインタ。操作失敗時には現在の値で更新されます。
  • newval: 交換対象となる新しい値。
  • **出力**:
  • bool: 比較が成功（*ptr == *expected）した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • expected ポインタは 8 バイト境界にアライメントされている必要があります（AssertPointerAlignment による検証）。
  • ptr は volatile 修飾されており、コンパイラによる最適化でメモリ操作が省略されないことが保証されます。

アルゴリズムまたは振る舞い

1. Solaris のシステム関数 atomic_cas_64 を呼び出し、ptr->value の現在の値と *expected を比較します。 2. 一致する場合、ptr->value を newval に更新します。 3. atomic_cas_64 は、操作直前の ptr->value の値を返します。 4. 返された値と *expected を比較し、結果を ret に格納します。 5. *expected を atomic_cas_64 から返された現在の値で更新します。 6. ret を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: Solaris システム関数 atomic_cas_64。
  • **呼び出し元**: src/include/port/atomics/generic.h 内の汎用アトミック関数群（pg_atomic_compare_exchange_u64 等）から呼び出されます。

関連データ構造

  • pg_atomic_uint64: アトミック操作対象の 64 ビット値を保持する構造体。

失敗モードと境界条件

  • expected が 8 バイト境界にない場合、アサーションエラーが発生します。
  • atomic_cas_64 の仕様により、比較が失敗した場合は *expected が現在の値で上書きされるため、呼び出し側はループ内で再試行が可能です。

保守担当者向けメモ

  • 本関数は Solaris 環境専用です。
  • atomic_cas_64 の戻り値が「現在の値」であるという仕様を利用して、PostgreSQL の期待する CAS インターフェースを実装しています。
