DeleteAttributeTuples

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/heap.c:1618

目的

指定されたリレーション（OID）に関連付けられた pg_attribute カタログ内のすべてのタプルを削除します。

シグネチャまたは形状

void DeleteAttributeTuples(Oid relid)

入力と出力

  • **入力**: relid (Oid) - 属性を削除する対象リレーションのOID。
  • **出力**: なし。

前提条件と不変条件

  • 本関数はトランザクション内で呼び出される必要があります。
  • pg_attribute システムカタログに対する RowExclusiveLock を取得します。

アルゴリズムまたは振る舞い

1. table_open を使用して pg_attribute を RowExclusiveLock モードで開きます。 2. ScanKeyInit を用いて、対象リレーションの OID (Anum_pg_attribute_attrelid) に一致するタプルを検索するためのスキャンキーを設定します。 3. systable_beginscan を使用し、AttributeRelidNumIndexId インデックス経由でスキャンを開始します。 4. systable_getnext で該当するタプルを順次取得し、CatalogTupleDelete を呼び出して物理的に削除します。 5. systable_endscan でスキャンを終了し、table_close で pg_attribute を閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c (リレーション削除処理内)
  • src/backend/catalog/index.c (インデックス削除処理内)
  • **呼び出し先**:
  • table_open / table_close
  • ScanKeyInit
  • systable_beginscan / systable_getnext / systable_endscan
  • CatalogTupleDelete

関連データ構造

  • pg_attribute: 属性情報を保持するシステムカタログ。
  • Relation, SysScanDesc, ScanKeyData, HeapTuple: PostgreSQL のテーブルアクセスおよびスキャン用データ構造。

失敗モードと境界条件

  • 対象の relid が存在しない場合、スキャンは何もヒットせず、削除処理は行われません。
  • ロック取得やカタログアクセス時にエラーが発生した場合、トランザクションの異常終了に伴いロールバックされます。

保守担当者向けメモ

  • 本関数は pg_attribute を直接操作するため、カタログの整合性に直結します。変更時は src/backend/catalog/indexing.c 等のインデックス保守ロジックとの整合性に注意してください。
