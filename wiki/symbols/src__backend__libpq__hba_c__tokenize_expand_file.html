tokenize_expand_file

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/hba.c:494

目的

認証設定ファイル（pg_hba.conf 等）内のインクルード指定（@filename）を再帰的に展開し、読み込まれたトークンを現在のリストに追加する。

シグネチャまたは形状

List *tokenize_expand_file(List *tokens, const char *outer_filename, const char *inc_filename, int elevel, int depth, char **err_msg)

入力と出力

  • **入力**:
  • tokens: 現在のトークンリスト。
  • outer_filename: インクルード元ファイル名。
  • inc_filename: インクルード対象ファイル名。
  • elevel: エラーレベル。
  • depth: 再帰の深さ。
  • err_msg: エラー発生時にメッセージを格納するポインタ。
  • **出力**:
  • 展開されたトークンを含む更新済みの List *。エラー時は元のリストを返す。

前提条件と不変条件

  • tokenize_context メモリコンテキストが有効であること。
  • インクルードされるファイルは open_auth_file で開ける状態であること。

アルゴリズムまたは振る舞い

1. AbsoluteConfigLocation を使用してインクルードファイルのフルパスを解決する。 2. open_auth_file でファイルを開く。失敗時はエラーを記録し、元のリストを返す。 3. tokenize_auth_file を再帰的に呼び出し、インクルードファイルの内容をトークン化する。 4. 取得したトークンリストを走査し、各行にエラーがないか確認する。 5. エラーがない場合、tokenize_context メモリコンテキストへ切り替えた上で、各トークンを tokens リストに追加する。 6. free_auth_file でファイルを閉じ、メモリを解放して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/hba.c 内のトークン解析処理（@ 記号を含む行の処理時）。
  • **呼び出し先**: AbsoluteConfigLocation, open_auth_file, tokenize_auth_file, pfree, pstrdup, MemoryContextSwitchTo, lappend, free_auth_file。

関連データ構造

  • List: PostgreSQLのリスト構造。
  • TokenizedAuthLine: トークン化された行データ。
  • AuthToken: 個別のトークンデータ。
  • tokenize_context: 解析用メモリコンテキスト。

失敗モードと境界条件

  • ファイルが開けない場合、open_auth_file がエラーを処理し、関数は即座に終了する。
  • インクルードされたファイル内でエラーが発生した場合、tok_line->err_msg を通じてエラーが伝播し、処理が中断される。
  • 再帰的なインクルードが深すぎる場合、open_auth_file の制限により失敗する可能性がある。

保守担当者向けメモ

  • 本関数は tokenize_context を明示的に切り替えてメモリ管理を行っているため、リスト操作時のコンテキスト変更を維持すること。
  • 再帰的なインクルードが発生するため、depth パラメータによる制限が適切に機能していることを確認すること。
