FreeSnapshotBuilder

定義

  • **種別**: function
  • **定義箇所**: src/backend/replication/logical/snapbuild.c:232

目的

SnapBuild オブジェクトに関連付けられたリソースを解放し、メモリコンテキストを破棄する。

シグネチャまたは形状

void FreeSnapshotBuilder(SnapBuild *builder)

入力と出力

  • **入力**: builder (SnapBuild*) - 解放対象のスナップショットビルダー構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • builder は有効な SnapBuild 構造体を指している必要がある。
  • builder->context は、このビルダーに関連するリソースが割り当てられているメモリコンテキストである必要がある。

アルゴリズムまたは振る舞い

1. builder->snapshot が NULL でない場合、SnapBuildSnapDecRefcount を呼び出してスナップショットの参照カウントを減らし、builder->snapshot を NULL に設定する。 2. builder->context に保持されているメモリコンテキストを MemoryContextDelete により破棄し、関連するすべてのリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/logical.c:685
  • **呼び出し先**:
  • SnapBuildSnapDecRefcount
  • MemoryContextDelete

関連データ構造

  • SnapBuild: スナップショット構築の状態を保持するメイン構造体。

失敗モードと境界条件

  • builder が NULL の場合、builder->context へのアクセスでクラッシュする可能性がある。
  • builder->snapshot が既に解放済み、あるいは無効なポインタである場合、SnapBuildSnapDecRefcount 内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はメモリコンテキスト全体を破棄するため、呼び出し後に builder ポインタを参照することはできない。
  • スナップショットの参照カウント管理とメモリコンテキストのライフサイクル管理が分離されている点に注意すること。
