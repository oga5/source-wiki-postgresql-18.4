gtsquery_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_gist.c:52

目的

GiST インデックス検索において、インデックス内のシグネチャと検索クエリの包含関係を判定し、検索対象の候補を絞り込む。

シグネチャまたは形状

Datum gtsquery_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY *entry (インデックスエントリ)
  • PG_GETARG_TSQUERY(1): TSQuery query (検索条件)
  • PG_GETARG_UINT16(2): StrategyNumber strategy (検索戦略)
  • PG_GETARG_POINTER(4): bool *recheck (再検証フラグの出力先)
  • **出力**:
  • Datum: bool 型の判定結果（PG_RETURN_BOOL）
  • *recheck: 常に true が設定される

前提条件と不変条件

  • 本関数は常に recheck = true を設定する。これは、シグネチャによる判定が偽陽性を含む可能性のある「不完全（inexact）」なインデックス検索であることを前提としているためである。

アルゴリズムまたは振る舞い

1. 入力された entry から TSQuerySign（ビットマップ）を抽出し、検索クエリ query からも同様に TSQuerySign を生成する。 2. recheck を true に設定する。 3. strategy に基づき以下の判定を行う：

  • RTContainsStrategyNumber (包含):
  • リーフノードの場合: (key & sq) == sq
  • 非リーフノードの場合: (key & sq) != 0
  • RTContainedByStrategyNumber (被包含):
  • リーフノードの場合: (key & sq) == key
  • 非リーフノードの場合: (key & sq) != 0
  • その他: false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の明示的な呼び出し元として src/backend/utils/adt/tsquery_gist.c:275 が確認できる。
  • **呼び出し先**: makeTSQuerySign を呼び出してクエリのシグネチャを生成する。

関連データ構造

  • GISTENTRY: GiST インデックスのエントリ構造体。
  • TSQuerySign: tsquery を圧縮したビットマップ表現。
  • TSQuery: 全文検索クエリの型。

失敗モードと境界条件

  • 未定義の strategy が渡された場合は false を返す。
  • シグネチャの比較はビット演算に依存しており、ハミング距離等のヒューリスティックな性質上、必ず再検証（recheck）が必要となる。

保守担当者向けメモ

  • 本関数は gtsquery_consistent_oldsig とは異なり、現在の PostgreSQL のシグネチャ形式に対応している。
  • インデックスの品質は hemdist 等のシグネチャ生成ロジックに依存するため、検索効率の改善が必要な場合はそちらのアルゴリズムを検討すること。
