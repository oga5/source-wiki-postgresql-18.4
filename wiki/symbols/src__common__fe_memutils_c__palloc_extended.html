palloc_extended

定義

  • 種別: function
  • 定義箇所: src/common/fe_memutils.c:132

目的

フロントエンド環境において、フラグ指定による柔軟なメモリ確保機能を提供します。バックエンドのメモリ管理インターフェースとの互換性を維持し、移植性を高めるために使用されます。

シグネチャまたは形状

void *palloc_extended(Size size, int flags)

入力と出力

  • **入力**:
  • size (Size): 確保するメモリのバイト数。
  • flags (int): メモリ確保の挙動を制御するフラグ（例: MCXT_ALLOC_ZERO, MCXT_ALLOC_NO_OOM, MCXT_ALLOC_HUGE）。
  • **出力**:
  • 確保されたメモリ領域へのポインタを返します。

前提条件と不変条件

  • FRONTEND マクロが定義されている環境でのみ使用可能です。
  • 確保サイズが SIZE_MAX / 2 を超える場合、エラーとして扱われます。
  • メモリ確保に失敗した場合、MCXT_ALLOC_NO_OOM フラグが指定されていない限り、プロセスは exit(EXIT_FAILURE) により終了します。

アルゴリズムまたは振る舞い

内部的に pg_malloc_internal(size, flags) を呼び出し、以下の処理を委譲します。 1. 指定されたサイズが妥当か検証します。 2. malloc を実行し、必要に応じてエラーハンドリングを行います。 3. MCXT_ALLOC_ZERO フラグが指定されている場合、確保したメモリをゼロクリアします。

呼び出し元と呼び出し先

  • **呼び出し先**: pg_malloc_internal
  • **呼び出し元**:
  • contrib/pg_prewarm/autoprewarm.c
  • contrib/pg_stat_statements/pg_stat_statements.c
  • src/backend/access/transam/xlogreader.c
  • src/backend/postmaster/pmchild.c
  • src/backend/utils/mmgr/mcxt.c

関連データ構造

  • Size 型（メモリサイズ指定用）

失敗モードと境界条件

  • size が 0 の場合、移植性の観点から malloc(1) として扱われます。
  • メモリ不足時、MCXT_ALLOC_NO_OOM フラグがない場合はプロセスが終了します。
  • SIZE_MAX / 2 を超える要求は、物理メモリの有無に関わらずエラーとなります。

保守担当者向けメモ

  • 本関数はバックエンドの palloc_extended と同一のシグネチャを持ち、フロントエンドコードがバックエンドのメモリ管理ロジックを再利用できるように設計されています。
  • 修正を行う際は、src/common/fe_memutils.c 内の他の確保関数（pg_malloc 等）との整合性に注意してください。
