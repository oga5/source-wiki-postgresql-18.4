pg_atomic_init_u32

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics.h:220

目的

pg_atomic_uint32 型の変数を指定された値で初期化します。

シグネチャまたは形状

static inline void pg_atomic_init_u32(volatile pg_atomic_uint32 *ptr, uint32 val)

入力と出力

  • ptr: 初期化対象となる pg_atomic_uint32 型変数へのポインタ（volatile 修飾あり）。
  • val: 設定する 32bit の初期値。
  • 戻り値: なし。

前提条件と不変条件

  • ptr は 4 バイト境界に配置されている必要があります（AssertPointerAlignment(ptr, 4) により検証されます）。

アルゴリズムまたは振る舞い

1. ポインタのアライメントを検証します。 2. プラットフォーム固有の実装である pg_atomic_init_u32_impl(ptr, val) を呼び出し、メモリへの書き込みを行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/vacuumparallel.c
  • src/backend/executor/nodeHashjoin.c
  • src/backend/nodes/tidbitmap.c
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/pgarch.c
  • src/backend/replication/logical/applyparallelworker.c
  • src/backend/storage/buffer/buf_init.c
  • src/backend/storage/buffer/freelist.c
  • **呼び出し先**:
  • pg_atomic_init_u32_impl（プラットフォームおよびコンパイラ固有の実装へ委譲）。

関連データ構造

  • pg_atomic_uint32: アトミック操作対象の 32bit 整数型。

失敗モードと境界条件

  • アライメントが 4 バイトでない場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、実際の書き込み処理は pg_atomic_init_u32_impl に依存します。
  • 呼び出し元で共有メモリ上の変数を初期化する際によく利用されます。
