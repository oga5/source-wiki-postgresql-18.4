get_command_tag

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_ddl_deparse/test_ddl_deparse.c:70

目的

CollectedCommand 構造体に含まれるパースツリーから、SQL コマンドタグ（例: CREATE TABLE）を生成し、テキスト形式で返します。

シグネチャまたは形状

Datum get_command_tag(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_POINTER(0) を介して CollectedCommand 構造体へのポインタを受け取ります。
  • **出力**: コマンドタグを表す text 型の Datum を返します。パースツリーが存在しない場合は NULL を返します。

前提条件と不変条件

  • 入力される CollectedCommand 構造体は有効である必要があります。
  • cmd->parsetree が NULL の場合、関数は安全に NULL を返します。

アルゴリズムまたは振る舞い

1. 引数から CollectedCommand 構造体を取得します。 2. parsetree フィールドが空であれば PG_RETURN_NULL() を実行します。 3. parsetree が存在する場合、CreateCommandName() を呼び出してコマンド名を取得します。 4. 取得したコマンド名を cstring_to_text() で PostgreSQL の text 型に変換し、PG_RETURN_TEXT_P() で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数呼び出しインターフェース（PG_FUNCTION_INFO_V1(get_command_tag) により登録）。
  • **呼び出し先**: CreateCommandName（tcop/utility.h で定義）、cstring_to_text。

関連データ構造

  • CollectedCommand: DDL 操作の内容を保持する構造体。

失敗モードと境界条件

  • cmd->parsetree が NULL の場合、エラーにはならず NULL を返します。
  • CreateCommandName が未知のノード型を受け取った場合の挙動は、当該関数の実装に依存します。

保守担当者向けメモ

  • 本関数はテストモジュール用であり、プロダクションコードでの使用は想定されていません。
  • CreateCommandName の仕様変更や、CollectedCommand 構造体の定義変更に伴い、本関数のロジックも影響を受ける可能性があります。
