StartLogStreamer

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_basebackup.c:615

目的

WAL（Write-Ahead Log）ストリーミングを実行するためのバックグラウンドプロセスまたはスレッドを初期化および起動する。

シグネチャまたは形状

static void StartLogStreamer(char *startpos, uint32 timeline, char *sysidentifier,
                             pg_compress_algorithm wal_compress_algorithm,
                             int wal_compress_level)

入力と出力

  • **入力**:
  • startpos: WAL ストリーミング開始位置（%X/%X 形式の文字列）。
  • timeline: タイムライン ID。
  • sysidentifier: システム識別子。
  • wal_compress_algorithm / wal_compress_level: WAL 圧縮設定。
  • **出力**: なし（副作用としてバックグラウンドプロセス/スレッドを生成）。

前提条件と不変条件

  • conn（グローバル接続）が確立されていること。
  • basedir が設定されていること。
  • startpos は sscanf で解析可能な形式であること。

アルゴリズムまたは振る舞い

1. logstreamer_param 構造体を割り当て、引数と開始位置（セグメント境界に調整済み）を格納する。 2. 非 Windows 環境では pipe を作成する。 3. GetConnection() を呼び出し、ストリーミング専用の接続を確立する。 4. サーバーバージョンに応じて pg_wal または pg_xlog ディレクトリパスを決定する。 5. レプリケーションスロットが必要な場合、CreateReplicationSlot を実行する。 6. format == 'p'（プレーンモード）の場合、archive_status ディレクトリおよび（必要に応じて）summaries ディレクトリを作成する。 7. OS に応じてプロセス（fork）またはスレッド（_beginthreadex）を生成し、LogStreamerMain を実行する。 8. 親プロセスでは、終了時に子プロセスを停止させるための atexit ハンドラを登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/pg_basebackup.c:2120
  • **呼び出し先**: GetConnection, CreateReplicationSlot, LogStreamerMain, fork (Unix), _beginthreadex (Windows)

関連データ構造

  • logstreamer_param: ストリーミングに必要なパラメータを保持する構造体。

失敗モードと境界条件

  • sscanf による startpos 解析失敗時: pg_fatal で終了。
  • pipe 作成失敗時: pg_fatal で終了。
  • GetConnection 失敗時: exit(1) で終了。
  • ディレクトリ作成失敗時: pg_fatal で終了。
  • プロセス/スレッド生成失敗時: pg_fatal で終了。

保守担当者向けメモ

  • 本関数は Unix と Windows で並行処理の実装が分岐しているため、修正時は両方のパスを考慮すること。
  • pg_wal と pg_xlog の名称変更に関するバージョン判定ロジックが含まれているため、将来的なディレクトリ構造変更時には注意が必要。
