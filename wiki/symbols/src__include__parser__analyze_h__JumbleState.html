JumbleState

定義

  • 種別: typedef
  • 定義箇所: src/include/parser/analyze.h:22
  • 実体定義: nodes/queryjumble.h:38

目的

クエリのフィンガープリント（Jumble）生成プロセスにおいて、解析中の状態を保持するための構造体です。

シグネチャまたは形状

構造体の詳細なメンバ構成は nodes/queryjumble.h に定義されています。analyze.h では主に post_parse_analyze_hook_type の引数として参照されます。

入力と出力

  • 入力: クエリ解析の過程で生成される ParseState や Query 構造体と組み合わせて使用されます。
  • 出力: クエリの正規化や定数情報の抽出など、クエリの識別子生成に必要な情報を保持します。

前提条件と不変条件

  • post_parse_analyze_hook を介して解析処理に渡される際、有効な JumbleState インスタンスが提供される必要があります。

アルゴリズムまたは振る舞い

クエリの解析完了後に呼び出されるフック関数において、クエリの構造をトラバースし、フィンガープリントを計算するためのコンテキストとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照）:
  • contrib/pg_stat_statements/pg_stat_statements.c: クエリ統計情報の収集に使用。
  • src/backend/commands/createas.c および src/backend/commands/explain.c: 解析処理の初期化時に参照。
  • 呼び出し先: 構造体自体の操作関数は nodes/queryjumble.h で定義されています。

関連データ構造

  • ParseState: 解析状態を保持するコンテキスト。
  • Query: 解析済みのクエリ構造体。
  • post_parse_analyze_hook_type: 本構造体を利用するフック関数の型定義。

失敗モードと境界条件

  • JumbleState が NULL の状態でフック関数が呼び出された場合、または初期化されていない状態で参照された場合の挙動は、各実装（pg_stat_statements 等）に依存します。

保守担当者向けメモ
