WalSummariesAreComplete

定義

  • 種別: function
  • 定義箇所: src/backend/backup/walsummary.c:137

目的

指定されたLSN範囲が、提供されたWAL要約ファイルのリストによって隙間なくカバーされているかを判定します。

シグネチャまたは形状

bool WalSummariesAreComplete(List *wslist, XLogRecPtr start_lsn, XLogRecPtr end_lsn, XLogRecPtr *missing_lsn)

入力と出力

  • **入力**:
  • wslist: WalSummaryFile 構造体のリスト。
  • start_lsn: 検証を開始するLSN。
  • end_lsn: 検証を終了するLSN。
  • **出力**:
  • 戻り値: 範囲が完全にカバーされていれば true、そうでなければ false。
  • missing_lsn: 範囲に隙間がある場合、最初に欠落が検出されたLSNが格納されます。

前提条件と不変条件

  • タイムラインの整合性は考慮されません。呼び出し側で事前に FilterWalSummaries 等を用いてタイムラインを一致させる必要があります。
  • リスト内の要約ファイルは重複していても正しく処理されます。

アルゴリズムまたは振る舞い

1. 入力リストが空の場合、missing_lsn に InvalidXLogRecPtr を設定し false を返します。 2. リストのコピーを作成し、ListComparatorForWalSummaryFiles を用いて start_lsn 順にソートします。 3. ソートされたリストを順次走査し、current_lsn（初期値は start_lsn）を更新します。

  • 各要約ファイルの start_lsn が current_lsn を超えている場合、隙間とみなしてループを終了します。
  • 要約ファイルの end_lsn が current_lsn を超える場合、current_lsn をその end_lsn に更新します。
  • 更新後の current_lsn が end_lsn 以上に達した時点で true を返します。

    • 4. ループ終了時に end_lsn に到達していなければ、最後に到達した current_lsn を missing_lsn に設定し false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup_incremental.c:519
  • **呼び出し先**: list_copy, list_sort, ListComparatorForWalSummaryFiles

関連データ構造

  • WalSummaryFile: 各要約ファイルのメタデータ（start_lsn, end_lsn 等）を保持する構造体。
  • List: PostgreSQLのリスト構造。

失敗モードと境界条件

  • リストが空の場合、即座に false を返します。
  • タイムラインが混在している場合、本関数はそれを区別せずに評価するため、誤った判定結果を返す可能性があります。

保守担当者向けメモ

  • 本関数はタイムラインを考慮しないため、呼び出し側でのフィルタリングが必須です。
  • リストのソートを行うため、リストの要素数が多い場合は計算量に注意してください。
