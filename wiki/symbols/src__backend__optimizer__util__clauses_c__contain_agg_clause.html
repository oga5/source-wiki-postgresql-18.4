contain_agg_clause

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/clauses.c:178

目的

与えられた式ツリー内に集約関数（Aggref ノード）が含まれているかどうかを判定します。

シグネチャまたは形状

bool contain_agg_clause(Node *clause)

入力と出力

  • **入力**: clause (走査対象となる式ツリーのルートノード)
  • **出力**: 集約関数が見つかった場合は true、それ以外の場合は false を返します。

前提条件と不変条件

  • clause は有効な Node ツリーである必要があります。
  • 内部的に contain_agg_clause_walker を呼び出し、ツリーを再帰的に探索します。

アルゴリズムまたは振る舞い

1. contain_agg_clause_walker を呼び出し、式ツリーの走査を開始します。 2. 走査中に Aggref ノードが検出された場合、walker は即座に true を返し、探索を中断します。 3. ツリー全体を走査しても Aggref ノードが見つからない場合、false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/equivclass.c
  • src/backend/optimizer/plan/planner.c
  • src/backend/optimizer/util/var.c
  • **呼び出し先**:
  • contain_agg_clause_walker (同一ファイル内)

関連データ構造

  • Node: PostgreSQLの式ツリーを構成する基本構造体。
  • Aggref: 集約関数呼び出しを表すノード。

失敗モードと境界条件

  • clause が NULL の場合、contain_agg_clause_walker の実装に依存しますが、一般的に false を返します。
  • サブクエリ内の集約関数については、walker の探索範囲（サブクエリを跨ぐか否か）に依存します。

保守担当者向けメモ

  • 本関数は単なるラッパーです。ロジックの変更が必要な場合は、contain_agg_clause_walker を修正してください。
  • クエリ最適化において、特定の式が集約を含むかどうかの判定は、インデックススキャンや並列実行の可否を決定する重要なガード条件となります。
