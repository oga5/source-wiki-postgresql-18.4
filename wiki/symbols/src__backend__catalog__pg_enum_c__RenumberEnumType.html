RenumberEnumType

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_enum.c:760

目的

列挙型（ENUM）の enumsortorder 値が精度限界に達した際、既存の全ラベルに対して順序値を 1 から順に再割り当てし、カタログの整合性を維持する。

シグネチャまたは形状

static void RenumberEnumType(Relation pg_enum, HeapTuple *existing, int nelems)

入力と出力

  • **入力**:
  • pg_enum: pg_enum システムカタログの開かれたリレーション。
  • existing: 既存の列挙型ラベルのタプル配列。
  • nelems: 配列内の要素数。
  • **出力**: なし（副作用としてシステムカタログを更新）。

前提条件と不変条件

  • pg_enum リレーションは既に排他ロックされている必要がある。
  • existing 配列は、ソート済みの状態であること。
  • enumsortorder の値は float4 型であり、本関数により 1 から nelems までの整数値に再設定される。

アルゴリズムまたは振る舞い

1. 配列の末尾から先頭に向かって（nelems - 1 から 0 まで）ループ処理を行う。 2. 各タプルを heap_copytuple で複製し、enumsortorder を現在のインデックス値 + 1 に更新する。 3. CatalogTupleUpdate を呼び出し、システムカタログを更新する。 4. 全ての更新完了後、CommandCounterIncrement() を呼び出し、変更を可視化する。

  • 逆順に処理することで、更新中のユニーク制約違反を回避する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/pg_enum.c 内の AddEnumLabel から呼び出される（ラベル追加時に順序値の空きがなくなった場合）。
  • **呼び出し先**: heap_copytuple, GETSTRUCT, CatalogTupleUpdate, heap_freetuple, CommandCounterIncrement（いずれも PostgreSQL 内部 API）。

関連データ構造

  • Form_pg_enum: pg_enum カタログのタプル構造体。
  • HeapTuple: PostgreSQL のヒープタプル表現。

失敗モードと境界条件

  • pg_enum リレーションの更新権限がない場合や、メモリ不足時にはエラーが発生する可能性がある。
  • nelems が 0 の場合、ループは実行されず何も行われない。

保守担当者向けメモ

  • 本関数は float4 の精度限界を解消するためのコストの高い操作であるため、頻繁な呼び出しは避けるべきである。
  • システムカタログの直接更新を行うため、トランザクションの整合性に注意すること。
