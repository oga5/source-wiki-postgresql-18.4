mdexists

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/md.c:192

目的

指定されたリレーションの特定のフォーク（ForkNumber）に対応する物理ファイルがディスク上に存在するかを確認する。

シグネチャまたは形状

bool mdexists(SMgrRelation reln, ForkNumber forknum)

入力と出力

  • **入力**:
  • reln: 対象のリレーションを表す SMgrRelation 構造体。
  • forknum: 確認対象のフォーク番号（ForkNumber）。
  • **出力**:
  • bool: ファイルが存在する場合は true、存在しない場合は false を返す。

前提条件と不変条件

  • リカバリ中（InRecovery が真）でない場合、関数呼び出し時に既存のファイル記述子を閉じる処理が行われる。

アルゴリズムまたは振る舞い

1. **クローズ処理**: リカバリ中でない場合、mdclose(reln, forknum) を呼び出し、現在開いているファイル記述子を閉じる。これにより、前回のチェック以降にファイルが削除（unlink）された場合でも確実に検知できるようにする。 2. **存在確認**: mdopenfork(reln, forknum, EXTENSION_RETURN_NULL) を呼び出す。

  • EXTENSION_RETURN_NULL フラグを指定することで、ファイルが存在しない場合にエラーを発生させず、NULL を返すように制御する。

    • 3. **結果判定**: mdopenfork の戻り値が NULL でなければ true を、NULL であれば false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/smgr/smgr.c: smgr_exists 関数ポインタ経由で呼び出される。
  • **呼び出し先**:
  • mdclose: ファイル記述子の解放に使用。
  • mdopenfork: ファイルのオープンと存在確認に使用。

関連データ構造

  • SMgrRelation: ストレージマネージャが管理するリレーションのハンドル。
  • ForkNumber: リレーションのフォーク（メイン、FSM、VMなど）を識別する列挙型。

失敗モードと境界条件

  • **リカバリ中の挙動**: リカバリ中は mdclose をスキップする最適化が行われる。これはリカバリプロセスがリレーション削除時に既にファイルを閉じていることを前提としている。
  • **ファイル不在**: ファイルが存在しない場合、mdopenfork が NULL を返すことで正しく false が返される。

保守担当者向けメモ

  • 本関数は smgr.c を介してストレージマネージャのインターフェースとして公開されている。
  • mdclose の呼び出しは、物理的なファイル状態とメモリ上のキャッシュ状態を同期させるための重要なステップである。
