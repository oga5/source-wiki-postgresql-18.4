smgr_bulk_start_rel

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/bulk_write.c:86

目的

リレーションの新規作成や初期ロードを行うためのバルク書き込み操作を開始し、状態管理オブジェクト（BulkWriteState）を初期化する。

シグネチャまたは形状

BulkWriteState *smgr_bulk_start_rel(Relation rel, ForkNumber forknum)

入力と出力

  • **入力**:
  • rel: 対象となるリレーションの記述子（Relation）。
  • forknum: 書き込み対象のフォーク番号（ForkNumber）。
  • **出力**:
  • BulkWriteState *: バルク書き込み操作の状態を保持する構造体へのポインタ。

前提条件と不変条件

  • 対象リレーションに対して、他のバックエンドが同時にアクセスしないことが前提である。
  • RelationGetSmgr(rel) により、リレーションからストレージマネージャ記述子が取得可能であること。

アルゴリズムまたは振る舞い

1. RelationGetSmgr(rel) を呼び出し、リレーションからストレージマネージャ記述子を取得する。 2. リレーションが WAL を必要とするか、または対象が INIT_FORKNUM であるかを判定する。 3. 上記の情報を引数として smgr_bulk_start_smgr を呼び出し、その戻り値をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistbuild.c
  • src/backend/access/heap/rewriteheap.c
  • src/backend/access/nbtree/nbtree.c
  • src/backend/access/nbtree/nbtsort.c
  • src/backend/access/spgist/spginsert.c
  • **呼び出し先**:
  • smgr_bulk_start_smgr (同一ファイル内)

関連データ構造

  • BulkWriteState: 本関数が生成する状態管理構造体。
  • Relation: リレーション記述子。
  • ForkNumber: フォーク識別子。

失敗モードと境界条件

  • メモリ不足により BulkWriteState の確保に失敗した場合、エラーが発生する可能性がある。
  • 無効な Relation や ForkNumber が渡された場合の挙動は、内部で呼び出される smgr_bulk_start_smgr の実装に依存する。

保守担当者向けメモ

  • 本関数は smgr_bulk_start_smgr のラッパーであり、リレーションキャッシュエントリ（Relation）が存在する場合のインターフェースを提供する。
  • WAL の要否判定ロジック（RelationNeedsWAL(rel) || forknum == INIT_FORKNUM）は、初期ロード時の整合性保証において重要である。
