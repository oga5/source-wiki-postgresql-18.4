CallerFInfoFunctionCall1

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/fmgr.c:1064

目的

FmgrInfo 構造体を使用して、引数を1つ取る関数を直接呼び出すためのラッパー関数です。

シグネチャまたは形状

Datum CallerFInfoFunctionCall1(PGFunction func, FmgrInfo *flinfo, Oid collation, Datum arg1)

入力と出力

  • **入力**:
  • func: 呼び出す関数ポインタ。
  • flinfo: 関数に関連付けられた FmgrInfo 構造体。
  • collation: 照合順序（Collation）のOID。
  • arg1: 関数に渡す Datum 型の引数。
  • **出力**:
  • 関数の実行結果を Datum 型で返す。

前提条件と不変条件

  • flinfo は適切に初期化されている必要があります。
  • 呼び出される関数は NULL を返さないことが期待されています。

アルゴリズムまたは振る舞い

1. LOCAL_FCINFO マクロを使用して、引数1つ用の FunctionCallInfoBaseData をスタック上に確保します。 2. InitFunctionCallInfoData を呼び出し、fcinfo を初期化します。 3. arg1 を fcinfo->args[0] に設定し、isnull を false に設定します。 4. 関数ポインタ func を fcinfo を引数として実行します。 5. 実行後、fcinfo->isnull を確認し、関数が NULL を返した場合は elog(ERROR) を発生させます。 6. 正常終了時に result を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/fmgr.h に宣言があり、PostgreSQL内部の各所から呼び出されます。
  • **呼び出し先**: 引数として渡された PGFunction 型の関数ポインタ。

関連データ構造

  • FmgrInfo: 関数呼び出しのメタデータ保持用構造体。
  • FunctionCallInfoBaseData: 実行時の引数やコンテキストを保持する構造体。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • 呼び出された関数が NULL を返した場合、elog(ERROR) によりエラーが発生します。

保守担当者向けメモ

  • 本関数は、呼び出し側が結果として NULL を期待していないケースでの利用を想定しています。
  • LOCAL_FCINFO を使用しているため、スタック領域を消費します。再帰的な呼び出しが深い場合は注意してください。
