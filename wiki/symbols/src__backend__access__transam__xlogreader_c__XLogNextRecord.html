XLogNextRecord

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogreader.c:325

目的

デコード済みWALレコードのキューから次のレコードを取得し、リーダーの状態を更新します。

シグネチャまたは形状

DecodedXLogRecord *XLogNextRecord(XLogReaderState *state, char **errormsg)

入力と出力

  • **入力**:
  • state: XLogReaderState 構造体へのポインタ。
  • errormsg: エラーメッセージを格納するためのポインタのポインタ。
  • **出力**:
  • 戻り値: 取得した DecodedXLogRecord 構造体へのポインタ。キューが空の場合は NULL を返します。
  • errormsg: エラーが発生した場合はエラーメッセージ文字列へのポインタが設定されます。

前提条件と不変条件

  • state->EndRecPtr は、直前の XLogBeginRead() または XLogNextRecord() の呼び出しによって設定されている必要があります。
  • 呼び出しごとに、以前に返されたレコードは XLogReleasePreviousRecord() によって解放されます。

アルゴリズムまたは振る舞い

1. XLogReleasePreviousRecord(state) を呼び出し、前回のレコードを解放します。 2. state->decode_queue_head が空の場合:

  • 遅延されたエラーメッセージがあれば errormsg に設定します。
  • state->EndRecPtr が有効であることを確認（Assert）し、NULL を返します。

    • 3. キューにレコードが存在する場合:

  • state->record を state->decode_queue_head に更新します。
  • state->ReadRecPtr を現在のレコードの lsn に、state->EndRecPtr を next_lsn に更新します。
  • 取得したレコードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogprefetcher.c
  • src/backend/access/transam/xlogreader.c (内部的な状態遷移やテストコード等)
  • **呼び出し先**:
  • XLogReleasePreviousRecord (直接呼び出し)

関連データ構造

  • XLogReaderState: リーダーの現在の状態、デコードキュー、および読み取りポインタを保持します。
  • DecodedXLogRecord: 解析済みのWALレコード情報を保持します。

失敗モードと境界条件

  • **キューが空**: レコードがこれ以上存在しない場合、NULL を返します。
  • **エラーの遅延報告**: 以前の処理で発生したエラーが errormsg_deferred として保持されている場合、この関数がエラーを報告する役割を担います。
  • **無効なポインタ**: state->EndRecPtr が無効な場合、アサーション違反が発生します。

保守担当者向けメモ

  • 本関数は、歴史的な理由から XLogRecXXX マクロがリーダーの状態（ReadRecPtr 等）を参照できるように、リーダーの内部ポインタを更新する副作用を持ちます。
  • フロントエンド（pg_waldump等）で使用される場合、ereport が利用できないため、エラー報告は errormsg 引数を介した制御に依存しています。
