bit_in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varbit.c:146

目的

SQLの文字列リテラル（バイナリまたは16進数形式）を、PostgreSQLの内部的なビット列表現である VarBit 構造体に変換します。

シグネチャまたは形状

Datum bit_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_CSTRING(0): 変換対象の文字列（'b'/'B'プレフィックス付きのバイナリ、'x'/'X'プレフィックス付きの16進数、またはプレフィックスなしのバイナリ）。
  • PG_GETARG_INT32(2): atttypmod（ビット長制限）。
  • **出力**:
  • Datum: 変換された VarBit 構造体へのポインタ。

前提条件と不変条件

  • VarBit 構造体の最後のバイトの未使用ビットは、palloc0 によるゼロ初期化およびパディング処理により、常にゼロでなければなりません。
  • atttypmod が指定されている場合、入力文字列から算出されるビット長と一致する必要があります。一致しない場合はエラーとなります。

アルゴリズムまたは振る舞い

1. **形式判定**: 文字列の先頭文字を確認し、バイナリ形式か16進数形式かを判定します。 2. **長さ計算**: 入力文字列からビット長を算出します。16進数の場合は slen * 4 となります。 3. **メモリ確保**: VARBITTOTALLEN マクロで必要なサイズを計算し、palloc0 でメモリを確保します。 4. **データ変換**:

  • **バイナリ形式**: 文字 '0'/'1' を走査し、ビット単位で VarBit のデータ領域に詰め込みます。
  • **16進数形式**: 文字 '0'-'9', 'A'-'F', 'a'-'f' を走査し、4ビットずつバイトデータに変換して格納します。

    • 5. **返却**: 構築した VarBit 構造体を Datum として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gin/btree_gin.c
  • src/backend/parser/parse_node.c
  • src/backend/utils/adt/varbit.c (内部的な再利用)
  • **呼び出し先**:
  • palloc0: メモリ確保。
  • ereturn: エラー発生時の制御。
  • pg_mblen_cstr: 文字列長計算。

関連データ構造

  • VarBit: ビット列を保持する構造体。
  • bits8: ビット操作に使用される符号なし8ビット整数型。

失敗モードと境界条件

  • **長さ不一致**: atttypmod と入力文字列のビット長が異なる場合、ERRCODE_STRING_DATA_LENGTH_MISMATCH を発生させます。
  • **不正な文字**: バイナリ形式で '0'/'1' 以外、または16進数形式で有効な16進数文字以外が含まれる場合、ERRCODE_INVALID_TEXT_REPRESENTATION を発生させます。
  • **サイズ超過**: 16進数入力が VARBITMAXLEN を超える場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED を発生させます。

保守担当者向けメモ

  • 本関数は varbit_in と類似のロジックを共有しています。
  • ビット順序は最上位ビット（MSB）が先頭となるため、シフト演算の方向には注意してください。
  • VARBIT_PAD マクロによるパディングの不変条件が維持されていることを確認してください。
