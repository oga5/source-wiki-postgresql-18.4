build_hash_table

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSetOp.c:83

目的

SetOp ノードにおいてハッシュ戦略（SETOP_HASHED）が選択された際、タプルのグループ化と出現回数の集計を行うためのハッシュテーブルを初期化する。

シグネチャまたは形状

static void build_hash_table(SetOpState *setopstate)

入力と出力

  • **入力**: SetOpState *setopstate（実行状態を保持する構造体）。
  • **出力**: なし（副作用として setopstate->hashtable を構築・設定する）。

前提条件と不変条件

  • node->strategy が SETOP_HASHED であること。
  • node->numGroups が 0 より大きいこと。

アルゴリズムまたは振る舞い

1. ExecGetResultType を用いて外部プランのタプル記述子を取得する。 2. BuildTupleHashTable を呼び出し、以下の情報を渡してハッシュテーブルを生成する。

  • 比較列のインデックス、等価関数（eqfuncoids）、ハッシュ関数（hashfunctions）、照合順序（cmpCollations）。
  • グループ数（numGroups）および各グループのデータサイズ（sizeof(SetOpStatePerGroupData)）。
  • メモリコンテキスト（es_query_cxt、tableContext、ecxt_per_tuple_memory）。

    • 3. 構築されたハッシュテーブルを setopstate->hashtable に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeSetOp.c 内の実行ロジック（ExecSetOp 等から間接的に呼び出される）。
  • **呼び出し先**: BuildTupleHashTable（ハッシュテーブル構築の基盤関数）。

関連データ構造

  • SetOpState: 実行ノードの状態を保持。
  • SetOpStatePerGroupData: ハッシュテーブルの各エントリに格納される、グループごとの出現回数データ。
  • TupleHashTable: BuildTupleHashTable が返すハッシュテーブル構造体。

失敗モードと境界条件

  • node->numGroups が 0 以下の場合、アサーションエラーが発生する。
  • メモリ不足時には BuildTupleHashTable 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は SetOp ノード専用の初期化ロジックである。nodeAgg.c や nodeMemoize.c にも同名の関数が存在するが、それらとは異なる実装であることに注意すること。
  • ハッシュテーブルのメモリは setopstate->tableContext で管理されており、再スキャン時にリセットされる設計となっている。
