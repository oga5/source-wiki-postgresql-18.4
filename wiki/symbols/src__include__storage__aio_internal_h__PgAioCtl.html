PgAioCtl

定義

  • **種別**: struct
  • **定義箇所**: src/include/storage/aio_internal.h:229

目的

PostgreSQL の非同期 I/O (AIO) サブシステム全体で共有される、グローバルな制御構造体です。各バックエンドの I/O 状態、共有 I/O ベクトル、および I/O ハンドルの管理を一元化します。

シグネチャまたは形状

typedef struct PgAioCtl
{
    int          backend_state_count;
    PgAioBackend *backend_state;
    uint32       iovec_count;
    struct iovec *iovecs;
    uint64       *handle_data;
    uint32       io_handle_count;
    PgAioHandle  *io_handles;
} PgAioCtl;

入力と出力

  • **入力**: PGC_POSTMASTER GUC 設定に基づき、最大 I/O 数やバッファ数に応じたメモリ領域が割り当てられます。
  • **出力**: システム内の各バックエンドが参照する共有メモリ上の制御情報を提供します。

前提条件と不変条件

  • iovecs および handle_data は、個別の I/O 操作が複数のバッファにまたがる場合に、バッファ ID の追跡や状態変更のために使用されます。
  • iovecs の割り当ては PgAioCtl 内で行われ、構成可能な最大数に依存します。

アルゴリズムまたは振る舞い

  • 本構造体は、AIO サブシステムの初期化時に pgaio_ctl としてグローバルに配置されます。
  • 各バックエンドは backend_state を通じて自身の I/O 状態を管理し、io_handles を介して個別の I/O 操作を追跡します。
  • 完了処理時には、handle_data を参照して関連する BufferDesc の状態を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio_init.c: 初期化時にメモリサイズ計算およびポインタの割り当てに使用。
  • src/backend/storage/aio/aio.c: グローバル変数 pgaio_ctl として参照。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • PgAioBackend: 各バックエンドの I/O 状態を保持。
  • PgAioHandle: 個別の I/O 操作を追跡。
  • struct iovec: I/O 操作のデータ配置を定義。

失敗モードと境界条件

  • iovec_count や io_handle_count が不足する場合、I/O の投入が制限される可能性があります。
  • 共有メモリの割り当て失敗は、サブシステムの初期化失敗に直結します。

保守担当者向けメモ

  • PgAioCtl の各フィールドは、メモリ効率を考慮して設計されています。フィールドの追加や変更を行う際は、共有メモリのレイアウトとアライメントに注意してください。
  • handle_data は、隣接するページを結合した際に実際の iovec 数と不一致が生じる可能性があるため、インデックス管理には注意が必要です。
