scanGetItem

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginget.c:1299

目的

複数のスキャンキーをロックステップで走査し、すべてのキー条件を満たす単一のタプル（TID）または損失のあるページ（Lossy Page）を特定する。

シグネチャまたは形状

static bool scanGetItem(IndexScanDesc scan, ItemPointerData advancePast, ItemPointerData *item, bool *recheck)

入力と出力

  • **入力**:
  • scan: インデックススキャン記述子。
  • advancePast: この位置より後のアイテムを検索するための基準点。
  • **出力**:
  • item: 条件に合致した ItemPointerData。
  • recheck: 損失のあるページが含まれる場合など、再チェックが必要かを示すフラグ。
  • **戻り値**: 検索が成功した場合は true、すべてのキーの走査が完了した場合は false。

前提条件と不変条件

  • keyGetItem ストリームは、同一ページに対して「完全な TID」と「損失のあるポインタ」を混在させないこと（混在すると走査の不整合が生じるため）。
  • ginNewScanKey は、最初のキーを excludeOnly としてマークしてはならない。

アルゴリズムまたは振る舞い

1. advancePast を超えるアイテムを各キーに対して keyGetItem で順次取得する。 2. いずれかのキーで isFinished が真となった場合、走査を終了する。 3. 各キーの curItemMatches を確認し、一致しない場合は advancePast を更新してループを継続する。 4. 一致した場合は、そのアイテムがすべてのキーで共通しているかを確認する。

  • 損失のあるページ（Lossy Page）の場合、ブロック番号の一致を確認する。
  • 完全な TID の場合、ginCompareItemPointers で一致を確認する。

    • 5. すべてのキーで一致が確認されるまでループを繰り返す。 6. 最後に、いずれかのキーで recheckCurItem が設定されていれば、recheck を true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginget.c 内の検索処理（例: gingetbitmap のループ内）。
  • **呼び出し先**: keyGetItem（各キーのアイテム取得）、ginCompareItemPointers（TID の比較）。

関連データ構造

  • IndexScanDesc: インデックススキャンの状態を保持。
  • GinScanOpaque: GIN 固有のスキャン状態（nkeys, keys 等を保持）。
  • GinScanKey: 個別の検索キーの状態。
  • ItemPointerData: タプル位置情報。

失敗モードと境界条件

  • **中断**: CHECK_FOR_INTERRUPTS() により、クエリ実行中の割り込みを処理する。
  • **終了条件**: いずれかのキーの走査が完了（isFinished）した時点で false を返す。
  • **損失のあるページ**: ItemPointerIsLossyPage が真の場合、特定のオフセットではなくページ全体が対象となるため、比較ロジックがブロック番号ベースに切り替わる。

保守担当者向けメモ

  • 本関数は GIN 検索の AND 条件統合の核心部である。
  • 損失のあるポインタと完全な TID が混在する際の比較ロジックは非常に繊細であり、変更時は keyGetItem との整合性に注意すること。
  • i == 0 のキーを基準に他のキーを追従させる設計となっている。
