InjectionPointCacheEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/misc/injection_point.c:95

目的

各バックエンドプロセス内において、インジェクションポイントの情報をハッシュテーブル形式で保持し、共有メモリへのアクセス頻度を低減するためのローカルキャッシュエントリとして機能します。

シグネチャまたは形状

typedef struct InjectionPointCacheEntry
{
    char        name[INJ_NAME_MAXLEN];
    char        private_data[INJ_PRIVATE_MAXLEN];
    InjectionPointCallback callback;
    int         slot_idx;
    uint64      generation;
} InjectionPointCacheEntry;

入力と出力

本構造体はデータ保持用であり、関数ではありません。キャッシュの更新時には共有メモリ上の InjectionPointEntry から値がコピーされ、実行時には callback フィールドが関数ポインタとして利用されます。

前提条件と不変条件

  • slot_idx と generation は、キャッシュ作成時の共有メモリ上の状態を保持します。
  • キャッシュの有効性は、現在の共有メモリ上の世代カウンタと generation を比較することで検証されます。

アルゴリズムまたは振る舞い

  • InjectionPointRun 実行時に InjectionPointCacheRefresh を介して参照されます。
  • キャッシュミスまたは世代不一致が発生した場合、共有メモリから情報を再取得し、本構造体の内容が更新されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/misc/injection_point.c 内のキャッシュ管理関数群（InjectionPointCacheRefresh 等）およびハッシュテーブル操作関数。
  • 呼び出し先: 本構造体自体はデータ構造であり、直接的な呼び出し先はありません。保持する callback フィールドが実行時に呼び出されます。

関連データ構造

  • InjectionPointEntry: 共有メモリ上に存在するマスターデータ。
  • InjectionPointsCtl: 共有メモリ上の管理構造体。

失敗モードと境界条件

  • INJ_NAME_MAXLEN および INJ_PRIVATE_MAXLEN を超えるデータは格納できません。
  • 共有メモリ上の世代カウンタが更新された場合、本キャッシュは無効とみなされ、再同期が必要です。

保守担当者向けメモ

  • 本構造体はバックエンドローカルなメモリ上に存在します。
  • 共有メモリとの同期には generation カウンタによる楽観的ロックの仕組みが利用されているため、キャッシュの整合性維持にはこのカウンタの比較が不可欠です。
