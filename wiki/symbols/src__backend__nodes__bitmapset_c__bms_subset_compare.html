bms_subset_compare

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/bitmapset.c:444

目的

2つの Bitmapset 間における包含関係（等価、部分集合、または包含関係なし）を判定する。

シグネチャまたは形状

BMS_Comparison bms_subset_compare(const Bitmapset *a, const Bitmapset *b)

入力と出力

  • **入力**:
  • a: 比較対象の Bitmapset ポインタ（NULL 許容）。
  • b: 比較対象の Bitmapset ポインタ（NULL 許容）。
  • **出力**:
  • BMS_Comparison 型の列挙値（BMS_EQUAL, BMS_SUBSET1 (a ⊆ b), BMS_SUBSET2 (b ⊆ a), BMS_DIFFERENT のいずれか）。

前提条件と不変条件

  • a および b は bms_is_valid_set を満たす有効なビットマップセットであること。
  • 空集合は NULL ポインタとして表現される。

アルゴリズムまたは振る舞い

1. **NULLチェック**: 両方が NULL なら BMS_EQUAL、一方が NULL なら空集合でない方を包含側として判定する。 2. **共通ワードの走査**: 両方の words 配列の重なり部分を do-while ループで比較する。

  • (aword & ~bword) != 0 であれば a は b の部分集合ではない。
  • (bword & ~aword) != 0 であれば b は a の部分集合ではない。
  • 既に判定済みの包含関係と矛盾するビット差分が見つかった場合は BMS_DIFFERENT を即座に返す。

    • 3. **ワード数の比較**: 共通部分の走査後、ワード数に差がある場合は、長い方の集合が短い方の集合を包含していないことを確認し、結果を確定させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/indxpath.c
  • src/backend/optimizer/plan/initsplan.c
  • src/backend/optimizer/util/pathnode.c
  • **呼び出し先**:
  • bms_is_valid_set（アサーション内）

関連データ構造

  • Bitmapset: 集合を表現する構造体。
  • BMS_Comparison: 比較結果を保持する列挙型。

失敗モードと境界条件

  • 入力が NULL の場合、正しく空集合として処理される。
  • 集合のワード数が異なる場合、長い方の集合にのみ存在するビットの有無によって包含関係が判定される。

保守担当者向けメモ

  • bms_subset_compare は、包含関係の判定において BMS_DIFFERENT を返す条件（両方の集合が互いに相手の部分集合ではない状態）を効率的に検出するように設計されている。
  • 戻り値の BMS_Comparison 定義については src/include/nodes/bitmapset.h を参照のこと。
