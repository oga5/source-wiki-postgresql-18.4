leaps_thru_end_of_nonneg

定義

  • 種別: function
  • 定義箇所: src/timezone/localtime.c:1399

目的

指定された年 y までの（0年を起点とした）閏日の総数を計算します。

シグネチャまたは形状

static int leaps_thru_end_of_nonneg(int y)

入力と出力

  • **入力**: int y （対象となる年）
  • **出力**: int （0年から y 年の末日までに含まれる閏日の合計数）

前提条件と不変条件

  • 入力 y は非負の整数であることを前提としています。
  • グレゴリオ暦のルール（4で割り切れる年は閏年、100で割り切れる年は平年、400で割り切れる年は閏年）に基づいた計算を行います。

アルゴリズムまたは振る舞い

以下の式を用いて閏日の数を算出します。 y / 4 - y / 100 + y / 400

  • y / 4: 4年ごとの閏日を加算。
  • y / 100: 100年ごとの平年化分を減算。
  • y / 400: 400年ごとの閏日を再加算。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/timezone/localtime.c 内の leaps_thru_end_of 関数から呼び出されています。
  • **呼び出し先**:
  • 特になし（算術演算のみ）。

関連データ構造

  • なし

失敗モードと境界条件

  • 入力 y が負の場合、この関数は意図した通りに動作しません（負の入力に対しては leaps_thru_end_of が別のロジックを提供しています）。
  • 整数オーバーフローの可能性については、y の値が int 型の範囲内である必要があります。

保守担当者向けメモ

  • 本関数は leaps_thru_end_of の内部ロジックの一部です。グレゴリオ暦の計算ロジックを変更する場合を除き、修正の必要性は低いです。
