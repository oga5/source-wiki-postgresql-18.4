geqo_copy

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_copy.c:44

目的

遺伝的アルゴリズム（GEQO）における個体（染色体）データを、複製元から複製先へコピーする。

シグネチャまたは形状

void geqo_copy(PlannerInfo *root, Chromosome *chromo1, Chromosome *chromo2, int string_length)

入力と出力

  • **入力**:
  • root: PlannerInfo 構造体へのポインタ（本関数内では未使用）。
  • chromo1: コピー先となる Chromosome 構造体へのポインタ。
  • chromo2: コピー元となる Chromosome 構造体へのポインタ。
  • string_length: コピーする遺伝子配列の長さ。
  • **出力**: なし（chromo1 が指すメモリ領域を直接更新する）。

前提条件と不変条件

  • chromo1 および chromo2 は有効な Chromosome 構造体を指している必要がある。
  • string_length は chromo1->string および chromo2->string が確保している配列サイズを超えてはならない。

アルゴリズムまたは振る舞い

1. 0 から string_length - 1 までのインデックスに対し、chromo2->string[i] の値を chromo1->string[i] へ逐次代入する。 2. 最後に、chromo2->worth（適応度）の値を chromo1->worth へコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/geqo/geqo_pool.c
  • src/backend/optimizer/geqo/geqo_selection.c
  • **呼び出し先**:
  • 本関数内から明示的に呼び出している関数はない。

関連データ構造

  • Chromosome: 遺伝子配列（string）と適応度（worth）を保持する構造体（src/include/optimizer/geqo.h で定義）。

失敗モードと境界条件

  • string_length が実際の配列サイズより大きい場合、バッファオーバーフローが発生する可能性がある。
  • ポインタが NULL の場合、セグメンテーション違反が発生する。

保守担当者向けメモ

  • 非常に軽量なメモリコピー操作であるため、パフォーマンス上のボトルネックにはなりにくい。
  • 遺伝的アルゴリズムの反復処理で頻繁に呼び出されるため、インライン化の検討対象となる。
  • PlannerInfo *root 引数は現在の実装では使用されていない。
