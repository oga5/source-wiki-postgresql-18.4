pgwin32_install_crashdump_handler

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/crashdump.c:177

目的

Windows環境において、プロセス内で未処理の例外が発生した際にミニダンプを生成するための例外ハンドラをOSに登録する。

シグネチャまたは形状

void pgwin32_install_crashdump_handler(void)

入力と出力

  • 入力: なし
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • Windows APIの SetUnhandledExceptionFilter が利用可能であること。
  • 本関数はプロセス起動時に呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. Windows APIの SetUnhandledExceptionFilter を呼び出す。 2. 引数として、同一ファイル内で定義されている crashDumpHandler 関数を渡す。 3. これにより、以降プロセス内で未処理の例外が発生した際、OSは自動的に crashDumpHandler を実行するようになる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/main/main.c:83
  • 呼び出し先: Windows API SetUnhandledExceptionFilter

関連データ構造

  • crashDumpHandler（コールバック関数として使用）

失敗モードと境界条件

  • 本関数自体は例外ハンドラの登録を行うのみであり、失敗時のエラーハンドリングは含まれていない。
  • 実際のダンプ生成処理（crashDumpHandler）は、スタックオーバーフローや深刻なメモリ不足時には正常に動作しない可能性がある。

保守担当者向けメモ

  • 本関数は src/include/port/win32_port.h にて外部公開されている。
  • crashDumpHandler はクラッシュコンテキストで実行されるため、本関数が登録するハンドラ内ではメモリ割り当てやログ出力などの複雑な処理を避け、write_stderr のみを使用すること。
