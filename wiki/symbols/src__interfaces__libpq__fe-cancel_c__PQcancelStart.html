PQcancelStart

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-cancel.c:203

目的

キャンセル要求を送信するための非同期接続プロセスを開始します。

シグネチャまたは形状

int PQcancelStart(PGcancelConn *cancelConn);

入力と出力

  • **入力**: PGcancelConn *cancelConn（キャンセル用接続オブジェクトへのポインタ）
  • **出力**: 成功した場合は非ゼロ（pqConnectDBStart の戻り値に依存）、失敗した場合は 0 を返します。

前提条件と不変条件

  • cancelConn が有効なポインタである必要があります。
  • 接続状態が CONNECTION_BAD である場合、処理は直ちに失敗します。
  • 接続状態が CONNECTION_ALLOCATED（初期状態）である必要があります。既に送信が開始されている場合、エラーとして扱われます。

アルゴリズムまたは振る舞い

1. cancelConn が NULL または接続状態が CONNECTION_BAD であれば 0 を返します。 2. 接続状態が CONNECTION_ALLOCATED でない場合、エラーメッセージを記録し、状態を CONNECTION_BAD に更新して 0 を返します。 3. 上記の条件を満たす場合、pqConnectDBStart を呼び出して接続処理を開始し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/libpq/libpq-be-fe-helpers.h:399

  • **呼び出し先**: pqConnectDBStart（fe-connect.c 内の関数と推測される）

関連データ構造

  • PGcancelConn: キャンセル要求に必要な情報を保持する構造体。

失敗モードと境界条件

  • cancelConn が NULL の場合、0 を返します。
  • 既にキャンセル要求が進行中（CONNECTION_ALLOCATED 以外）の場合、エラーメッセージが設定され、0 を返します。
  • pqConnectDBStart が失敗した場合、その戻り値（0）がそのまま返されます。

保守担当者向けメモ

  • 本関数は非同期APIの一部です。後続の処理には PQcancelPoll を使用して接続の完了を待機する必要があります。
  • libpq_append_conn_error を使用してエラー情報を記録するため、呼び出し元は必要に応じてエラーメッセージを確認してください。
