pgstat_checkpointer_init_shmem_cb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_checkpointer.c:88

目的

チェックポイント統計用共有メモリ領域のロックを初期化する。

シグネチャまたは形状

void pgstat_checkpointer_init_shmem_cb(void *stats)

入力と出力

  • **入力**: stats (共有メモリ上の PgStatShared_Checkpointer 構造体へのポインタ)
  • **出力**: なし（副作用として共有メモリ内のロックが初期化される）

前提条件と不変条件

  • 引数 stats は、PgStatShared_Checkpointer 型としてキャスト可能な有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. 引数 stats を PgStatShared_Checkpointer * 型にキャストする。 2. LWLockInitialize を呼び出し、構造体内の lock メンバを LWTRANCHE_PGSTATS_DATA トランシェで初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c:427 において、統計初期化時のコールバックとして登録されている。
  • **呼び出し先**: LWLockInitialize (PostgreSQLのロック管理サブシステム)

関連データ構造

  • PgStatShared_Checkpointer: チェックポイント統計を保持する共有メモリ構造体。
  • LWLock: 軽量ロック（Lightweight Lock）。

失敗モードと境界条件

  • 引数 stats が NULL の場合、デリファレンスによりクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数は pgstat サブシステムの共有メモリ初期化プロセスの一部として実行される。
  • ロックのトランシェとして LWTRANCHE_PGSTATS_DATA が指定されているため、統計データへのアクセス制御において他の統計データと一貫したロック階層が維持される。
