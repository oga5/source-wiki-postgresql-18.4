ginint4_queryextract

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_gin.c:12

目的

GIN インデックスの extractQuery メソッドとして、検索クエリ（QUERYTYPE または ArrayType）を解析し、インデックス検索に使用するキー配列と検索モードを抽出します。

シグネチャまたは形状

Datum ginint4_queryextract(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_QUERYTYPE_P(0) または PG_GETARG_ARRAYTYPE_P(0): 検索条件。
  • PG_GETARG_POINTER(1): 抽出されたキーの数を格納する int32 *nentries。
  • PG_GETARG_UINT16(2): 演算子戦略番号（StrategyNumber）。
  • PG_GETARG_POINTER(6): 検索モードを格納する int32 *searchMode。
  • **出力**:
  • Datum *: 抽出されたキーの配列。
  • *nentries および *searchMode を介してメタデータを返却。

前提条件と不変条件

  • strategy が BooleanSearchStrategy の場合、クエリサイズが 0 より大きい必要があります。
  • BooleanSearchStrategy 以外の場合、CHECKARRVALID マクロにより入力配列の妥当性が検証されます。

アルゴリズムまたは振る舞い

1. **ブール検索の場合**:

  • クエリ内の VAL 型アイテムを抽出し、インデックス検索キーとして返します。
  • 必須値が含まれないクエリ（例: ! 42）の場合、*searchMode を GIN_SEARCH_MODE_ALL に設定し、全スキャンを指示します。それ以外は GIN_SEARCH_MODE_DEFAULT とします。

    • 2. **配列検索の場合**:

  • 入力配列の各要素をキーとして抽出します。
  • 演算子戦略に応じて *searchMode を決定します。
  • RTOverlapStrategyNumber: GIN_SEARCH_MODE_DEFAULT
  • RTContainedByStrategyNumber / RTOldContainedByStrategyNumber: GIN_SEARCH_MODE_INCLUDE_EMPTY
  • RTSameStrategyNumber: 配列が空なら GIN_SEARCH_MODE_INCLUDE_EMPTY、それ以外は GIN_SEARCH_MODE_DEFAULT
  • RTContainsStrategyNumber / RTOldContainsStrategyNumber: 配列が空なら GIN_SEARCH_MODE_ALL、それ以外は GIN_SEARCH_MODE_DEFAULT

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GIN インデックスエンジン（直接的な呼び出し元コードはソース内に明示されていませんが、PG_FUNCTION_INFO_V1 により登録されています）。
  • **呼び出し先**: query_has_required_values（ブール検索時）、palloc、elog 等の PostgreSQL 内部関数。

関連データ構造

  • QUERYTYPE, ITEM: ブール検索用のデータ構造。
  • ArrayType: 配列検索用のデータ構造。
  • StrategyNumber: 演算子戦略を識別する定数。

失敗モードと境界条件

  • 未知の strategy が渡された場合、elog(ERROR) により処理を中断します。
  • 空のブールクエリが渡された場合、NULL を返して失敗させます。
  • 配列検索において要素数が 0 の場合、演算子戦略に応じて GIN_SEARCH_MODE_INCLUDE_EMPTY または GIN_SEARCH_MODE_ALL を選択し、適切な検索挙動を維持します。

保守担当者向けメモ

  • 本関数は GIN インデックスのインターフェースの一部です。recheck が必要な演算子（ContainedBy 等）については、この関数で searchMode を適切に設定し、後続の ginint4_consistent で整合性チェックを行う設計になっています。
