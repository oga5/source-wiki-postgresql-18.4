ReplicationSlotInitialize

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slot.c:238

目的

バックエンドプロセス終了時にレプリケーションスロット関連のクリーンアップ処理が確実に実行されるよう、終了ハンドラを登録します。

シグネチャまたは形状

void ReplicationSlotInitialize(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として終了ハンドラの登録を行う）

前提条件と不変条件

  • 本関数は、プロセスが共有メモリにアクセスする準備が整った段階で呼び出される必要があります。

アルゴリズムまたは振る舞い

1. before_shmem_exit を呼び出し、プロセス終了時に ReplicationSlotShmemExit 関数が実行されるよう登録します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/init/postinit.c:665
  • 呼び出し先: ReplicationSlotShmemExit

関連データ構造

  • ReplicationSlotCtl (間接的: ReplicationSlotShmemExit が参照する共有メモリ制御構造体)

失敗モードと境界条件

  • 終了ハンドラの登録に失敗した場合、PostgreSQLの標準的なエラー処理メカニズムに従いプロセスが異常終了する可能性があります。

保守担当者向けメモ

  • 本関数はプロセス初期化時に一度だけ呼び出されることを想定しています。
  • ReplicationSlotShmemExit のロジックを変更する際は、本関数による登録が適切に機能しているか確認してください。
