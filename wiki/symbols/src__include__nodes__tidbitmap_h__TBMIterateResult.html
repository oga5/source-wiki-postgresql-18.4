TBMIterateResult

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/tidbitmap.h:62

目的

TIDビットマップの走査結果を保持し、特定のブロックに含まれるタプル情報や、そのページに対する再チェックの必要性を呼び出し元へ伝達するために使用されます。

シグネチャまたは形状

typedef struct TBMIterateResult
{
    BlockNumber blockno;
    bool        lossy;
    bool        recheck;
    void       *internal_page;
} TBMIterateResult;

入力と出力

本構造体は関数への出力引数として渡され、値が書き込まれます。

  • **入力**: 呼び出し元が確保した TBMIterateResult 構造体へのポインタ。
  • **出力**: 走査されたブロック番号 (blockno)、損失フラグ (lossy)、再チェック要否 (recheck)、および内部的なページ参照 (internal_page) が格納されます。

前提条件と不変条件

  • internal_page は void * 型であり、API利用者はその内部構造（PagetableEntry 等）に直接依存してはなりません。
  • recheck は、ページが損失付き（lossy）である場合に常に true となります。また、損失付きでなくともクエリの要件に応じて true になる可能性があります。

アルゴリズムまたは振る舞い

tbm_iterate などの走査関数において、ビットマップから次の対象ブロックが特定されるたびに、その情報が本構造体に書き込まれます。呼び出し元はこの構造体を参照して、ヒープ上のどのページを読み込み、どのタプルを検証すべきかを判断します。

呼び出し元と呼び出し先

  • **主な呼び出し元**:
  • src/backend/nodes/tidbitmap.c: tbm_private_iterate, tbm_shared_iterate, tbm_iterate
  • src/backend/access/heap/heapam.c および heapam_handler.c: ビットマップヒープスキャンの実行時
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • TIDBitmap: 走査対象となるビットマップ本体。
  • TBMIterator: 走査状態を管理するイテレータ構造体。
  • BlockNumber: storage/itemptr.h で定義されるブロック識別子。

失敗モードと境界条件

  • 構造体自体に失敗モードはありませんが、tbm_iterate が走査終了を通知した際、本構造体の内容は未定義または無効となります。
  • 呼び出し元が適切なメモリ領域を確保していない場合、書き込み時にメモリ破壊が発生します。

保守担当者向けメモ

  • internal_page は実装詳細を隠蔽するためのポインタです。将来的に内部構造を変更する場合でも、このポインタの型を変更することでAPIの互換性を維持可能です。
  • recheck フラグの判定ロジックは、ビットマップの損失状態とクエリの実行計画に依存するため、変更時には tidbitmap.c の走査ロジックとの整合性に注意してください。
