WriteTarState

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_basebackup/pg_basebackup.c:67

目的

バックアップ処理において、tar形式で出力されるストリームの書き込み状態を管理するための構造体です。

シグネチャまたは形状

typedef struct WriteTarState
{
	int			tablespacenum;
	astreamer  *streamer;
} WriteTarState;

入力と出力

  • tablespacenum: 書き込み対象のテーブルスペース番号を保持します。
  • streamer: データ処理パイプラインを構成する astreamer 構造体へのポインタを保持します。

前提条件と不変条件

  • astreamer は、バックアップデータの受信および処理を行うパイプラインの一部として初期化されている必要があります。

アルゴリズムまたは振る舞い

本構造体は、pg_basebackup におけるデータ受信処理において、特定のテーブルスペースに対するtarストリームの書き込みコンテキストを保持するために使用されます。callback_data として渡された際に、この構造体を通じて現在のテーブルスペース番号やストリーマーの参照にアクセスします。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_basebackup/pg_basebackup.c 内の関数（main 等のバックアップ実行フロー）でインスタンス化され、コールバック関数へ渡されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、保持している streamer メンバを通じて astreamer インターフェースの関数が呼び出されます。

関連データ構造

  • astreamer: データ処理パイプラインの抽象化レイヤー。

失敗モードと境界条件

  • streamer が NULL の状態で操作が行われた場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • pg_basebackup.c 内の 1603 行目でインスタンス化され、1665 行目でコールバックデータとしてキャストされて使用されています。この構造体を利用するコールバック関数の実装を確認する際は、astreamer のパイプライン構成と併せて参照してください。
