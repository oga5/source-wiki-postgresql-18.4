multi_sort_init

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/extended_stats.c:838

目的

複数次元のソート操作をサポートするための MultiSortSupport 構造体を初期化し、メモリを割り当てる。

シグネチャまたは形状

MultiSortSupport multi_sort_init(int ndims)

入力と出力

  • **入力**: ndims (int) - ソート対象の次元数。
  • **出力**: MultiSortSupport 型のポインタ。MultiSortSupportData 構造体と、それに続く ndims 個分の SortSupportData 配列を保持するメモリ領域を指す。

前提条件と不変条件

  • ndims >= 2 であること（アサーションにより強制）。

アルゴリズムまたは振る舞い

1. offsetof(MultiSortSupportData, ssup) と sizeof(SortSupportData) * ndims の合計サイズを計算する。 2. palloc0 を使用して、計算したサイズ分のメモリをゼロ初期化して割り当てる。 3. 割り当てた構造体の ndims フィールドに引数の値を設定する。 4. 確保したメモリ領域へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/dependencies.c
  • src/backend/statistics/mcv.c
  • src/backend/statistics/mvdistinct.c
  • **呼び出し先**:
  • palloc0 (メモリ割り当て)

関連データ構造

  • MultiSortSupportData: 複数次元ソートの制御情報を保持する構造体。
  • SortSupportData: 個別の次元に対するソートサポート情報。

失敗モードと境界条件

  • ndims < 2 の場合、アサーション違反によりプロセスが停止する可能性がある。
  • メモリ不足時には palloc0 がエラーを発生させる。

保守担当者向けメモ

  • 本関数は extended_stats_internal.h で宣言されており、拡張統計の各計算ロジック（依存関係、MCV、多変量NDistinct）で共通して利用される。
  • 構造体の末尾に可変長の配列を配置する設計となっているため、メモリレイアウトの変更時には offsetof の計算に注意すること。
