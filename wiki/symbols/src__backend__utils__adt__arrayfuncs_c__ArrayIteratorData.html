ArrayIteratorData

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/adt/arrayfuncs.c:68

目的

配列要素を効率的に走査するためのイテレータ状態を保持する構造体です。

シグネチャまたは形状

typedef struct ArrayIteratorData
{
    ArrayType  *arr;
    bits8      *nullbitmap;
    int         nitems;
    int16       typlen;
    bool        typbyval;
    char        typalign;
    int         slice_ndim;
    int         slice_len;
    int        *slice_dims;
    int        *slice_lbound;
    Datum      *slice_values;
    bool       *slice_nulls;
    char       *data_ptr;
    int         current_item;
} ArrayIteratorData;

入力と出力

  • **入力**: array_create_iterator を通じて初期化されます。
  • **出力**: 走査中の現在位置 (data_ptr) および現在の要素インデックス (current_item) を更新し、スライス操作時には slice_values および slice_nulls に結果を格納します。

前提条件と不変条件

  • arr は有効な ArrayType を指している必要があります。
  • typlen, typbyval, typalign は、配列要素の型情報と一致している必要があります。
  • slice_len が 0 より大きい場合、slice_values と slice_nulls はそのサイズ分確保されている必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: array_create_iterator により、配列のメタデータ（型情報、次元、NULLビットマップ）が構造体にロードされます。 2. **走査**: data_ptr を用いて配列のデータ領域を順次移動し、current_item をインクリメントします。 3. **スライス処理**: スライスが要求されている場合、現在の走査位置に基づいて slice_values および slice_nulls を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/arrayfuncs.c 内の array_create_iterator にて palloc0 を介してインスタンス化されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、具体的な走査ロジックは arrayfuncs.c 内のイテレータ関連関数によって操作されます。

関連データ構造

  • ArrayType: 配列の内部表現。
  • ArrayIterator: ArrayIteratorData へのポインタ型（src/include/utils/array.h で定義）。

失敗モードと境界条件

  • 配列が空の場合、nitems は 0 となり、走査は即座に終了します。
  • メモリ不足時には palloc0 がエラーを発生させます。

保守担当者向けメモ

  • 本構造体は arrayfuncs.c 内でプライベートに使用されることを意図しています。
  • ARR_DATA_PTR を直接操作するよりも、このイテレータを使用する方が安全かつ効率的です。
  • 構造体の各フィールドは、配列のメモリレイアウト（ヘッダー、NULLビットマップ、データ本体）に強く依存しているため、レイアウト変更時には本構造体の更新が必要です。
