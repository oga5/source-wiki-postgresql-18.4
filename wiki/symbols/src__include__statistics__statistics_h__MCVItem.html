MCVItem

定義

  • 種別: struct
  • 定義箇所: src/include/statistics/statistics.h:78

目的

拡張統計情報における「最頻値（Most Common Values）」リストの各要素を保持するためのデータ構造です。列の組み合わせごとの出現頻度を格納し、クエリプランナによる選択率推定の精度向上に寄与します。

シグネチャまたは形状

typedef struct MCVItem
{
    double      frequency;      /* この組み合わせの出現頻度 */
    double      base_frequency; /* 独立と仮定した場合の出現頻度 */
    bool       *isnull;         /* NULLフラグの配列 */
    Datum      *values;         /* 値の配列 */
} MCVItem;

入力と出力

本構造体は単体で関数入出力に用いられることは少なく、主に MCVList 構造体の一部としてメモリ上に配置されます。values および isnull フィールドは、統計対象となる列数に応じた配列を指します。

前提条件と不変条件

  • values および isnull が指す配列の長さは、統計対象となる列数と一致している必要があります。
  • frequency は、観測された実際の出現頻度を保持します。

アルゴリズムまたは振る舞い

プランナは MCVList 内の MCVItem を走査し、クエリ条件と一致する組み合わせを検索します。一致が見つかった場合、frequency を用いて選択率を推定します。base_frequency は、列間に相関がないと仮定した場合の期待値であり、実際の頻度との比較に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/mcv.c 内の複数の関数（mcv_get_item 等）が、MCVList から個別の MCVItem を取得して参照しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • MCVList: MCVItem の配列を保持する上位の構造体。
  • Datum: PostgreSQLの汎用データ型。values 配列の各要素として使用されます。

失敗モードと境界条件

  • MCVList 内の items 配列を操作する際、MCVItem のサイズ計算（sizeof(MCVItem)）を誤るとメモリ破壊を引き起こす可能性があります。
  • values や isnull が指すメモリ領域が適切に確保されていない場合、不正なメモリアクセスが発生します。

保守担当者向けメモ

  • 本構造体は MCVList 内で可変長配列の一部として扱われることが多いため、メモリレイアウトの変更には細心の注意を払ってください。
  • values および isnull はポインタであるため、MCVList のシリアライズ・デシリアライズ時には、これらの指す先のデータも適切に処理する必要があります。
