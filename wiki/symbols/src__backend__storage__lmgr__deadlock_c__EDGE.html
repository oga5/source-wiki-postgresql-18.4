EDGE

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/lmgr/deadlock.c:47

目的

ロック待ちグラフ（Waits-for Graph）における、プロセス間の依存関係（エッジ）を表現するための構造体です。

シグネチャまたは形状

typedef struct {
    PGPROC *waiter;    /* 待ちグループのリーダー */
    PGPROC *blocker;   /* 待機対象のグループリーダー */
    LOCK   *lock;      /* 待機対象のロック */
    int     pred;      /* TopoSort用ワークスペース */
    int     link;      /* TopoSort用ワークスペース */
} EDGE;

入力と出力

本構造体はデータ保持を目的としており、関数としての入出力はありません。ただし、TopoSort や ExpandConstraints などの関数において、制約条件のリストとして渡されます。

前提条件と不変条件

  • waiter および blocker は PGPROC 構造体への有効なポインタである必要があります。
  • lock は現在競合が発生している LOCK 構造体を指す必要があります。

アルゴリズムまたは振る舞い

  • ロック待ちグラフ上のノード間関係を定義します。
  • pred および link フィールドは、トポロジカルソート（TopoSort）アルゴリズム実行時の内部的なグラフ探索および順序付けの作業領域として使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: deadlock.c 内の DeadLockCheck、TestConfiguration、FindLockCycle 等が本構造体の配列を確保・操作します。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。
  • 注: src/backend/optimizer/geqo/geqo_main.c に同名のコメントが存在しますが、本構造体との直接的な関連性は確認されていません。

関連データ構造

  • PGPROC: ロック待ちを行うプロセスを特定します。
  • LOCK: 競合対象となるロックオブジェクトを特定します。

失敗モードと境界条件

  • メモリ確保時（palloc）に失敗した場合、システムはエラーを発生させます。
  • グラフ探索時に循環参照が検出された場合、本構造体を用いてデッドロックの解消（待ち行列の再構成）が試行されます。

保守担当者向けメモ

  • pred と link は TopoSort 実行時のみ使用される一時的な領域です。これらを他の目的で再利用する場合は、アルゴリズムの整合性に注意してください。
  • softEdges として扱われる場合、ロック待ち行列の順序入れ替えによるデッドロック解消の対象となります。
