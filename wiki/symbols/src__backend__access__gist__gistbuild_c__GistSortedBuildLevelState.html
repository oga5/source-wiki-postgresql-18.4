GistSortedBuildLevelState

定義

  • **種別**: struct
  • **定義箇所**: src/backend/access/gist/gistbuild.c:123

目的

GiSTインデックスの「ソート済み構築（Sorted Build）」において、インデックスの各階層における構築状態を保持するために使用されます。

シグネチャまたは形状

typedef struct GistSortedBuildLevelState
{
    int            current_page;
    BlockNumber    last_blkno;
    struct GistSortedBuildLevelState *parent;
    Page           pages[GIST_SORTED_BUILD_PAGE_NUM];
} GistSortedBuildLevelState;

入力と出力

  • **入力**: 構築中のインデックス階層情報、ページバッファ、および親階層へのポインタ。
  • **出力**: 構築プロセスを通じて更新される current_page（現在のページインデックス）および last_blkno（最後に割り当てられたブロック番号）。

前提条件と不変条件

  • parent ポインタは、最上位階層（ルート）の場合は NULL となります。
  • pages 配列は、GIST_SORTED_BUILD_PAGE_NUM で定義された固定数のページを保持します。

アルゴリズムまたは振る舞い

  • ソート済み構築プロセスにおいて、ボトムアップでインデックスページを生成する際の作業領域として機能します。
  • gist_indexsortbuild 関数により、インデックスの各レベルごとにこの構造体が割り当てられ、ページが順次埋められます。
  • ページが一杯になると、gist_indexsortbuild_levelstate_flush を通じてディスクへ書き出され、必要に応じて親階層への挿入が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: gist_indexsortbuild 関数がこの構造体を palloc0 で初期化し、構築プロセス全体で管理します。
  • **呼び出し先**: gist_indexsortbuild_levelstate_add および gist_indexsortbuild_levelstate_flush 関数が、この構造体を引数として受け取り、ページ操作を実行します。

関連データ構造

  • GISTBuildState: インデックス構築全体のコンテキストを保持する構造体。
  • Page: PostgreSQLの標準的なページ構造体。

失敗モードと境界条件

  • メモリ不足時: palloc0 による構造体割り当て時にエラーが発生する可能性があります。
  • 階層の深さ: インデックスの階層が深くなるにつれ、parent ポインタを介して再帰的に構造体が連鎖します。

保守担当者向けメモ

  • 本構造体はソート済み構築専用です。バッファリング構築モードでは使用されません。
  • GIST_SORTED_BUILD_PAGE_NUM の値がページ構築の効率に直結するため、変更時はメモリ使用量とI/O効率のトレードオフを考慮してください。
