hashoid

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hashfunc.c:116

目的

PostgreSQLのOID（Object Identifier）型に対するハッシュ値を計算し、ハッシュインデックスやハッシュ結合のキーとして利用可能にする。

シグネチャまたは形状

Datum hashoid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される、OID型のSQL引数。
  • **出力**: Datum 型として返される32ビットのハッシュ値。

前提条件と不変条件

  • 入力されたOIDは PG_GETARG_OID(0) により取得される。
  • 等価なOID入力に対しては、常に同一のハッシュ値を返す必要がある。

アルゴリズムまたは振る舞い

1. PG_GETARG_OID(0) を呼び出し、第1引数のOIDを取得する。 2. 取得したOIDを uint32 型にキャストする。 3. hash_uint32 関数を呼び出し、その結果を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのハッシュアクセスメソッド（pg_amproc システムカタログ経由で呼び出される）。
  • **呼び出し先**: hash_uint32（common/hashfn.h で定義された整数用ハッシュ関数）。

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • Oid: オブジェクト識別子型。

失敗モードと境界条件

  • PG_GETARG_OID マクロの仕様に従い、引数が存在しない場合や型が不適切な場合はエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はOID専用のハッシュ計算を行う。ハッシュ結合のパフォーマンス向上のためにシード値を受け取る必要がある場合は、対となる hashoidextended を使用すること。
