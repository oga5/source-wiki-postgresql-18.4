pg_get_viewdef_wrap

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ruleutils.c:715

目的

指定されたビューの OID に基づき、指定された折り返し幅（wrap）設定を適用して、人間が読み取り可能な SQL 定義文字列を生成する。

シグネチャまたは形状

Datum pg_get_viewdef_wrap(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): 対象ビューの OID。
  • PG_GETARG_INT32(1): SQL 出力の折り返し幅（wrap）。
  • **出力**:
  • Datum: ビュー定義を表す text 型。定義が見つからない場合は NULL を返す。

前提条件と不変条件

  • 本関数は pretty printing を有効にする前提で動作する。
  • 内部で pg_get_viewdef_worker を呼び出し、その結果を text 型に変換して返却する。

アルゴリズムまたは振る舞い

1. 引数からビューの OID と折り返し幅を取得する。 2. GET_PRETTY_FLAGS(true) を使用して整形フラグを生成する。 3. pg_get_viewdef_worker を呼び出し、ビュー定義の文字列を取得する。 4. 結果が NULL の場合は PG_RETURN_NULL() を実行する。 5. 結果が文字列の場合、string_to_text を介して text 型に変換し、PG_RETURN_TEXT_P で返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: 外部からの SQL 関数呼び出し（直接的な呼び出し元情報はソースコード内に明示されていない）。
  • **呼び出し先**:
  • pg_get_viewdef_worker (定義の逆コンパイルを実行)
  • string_to_text (C 文字列を PostgreSQL の text 型へ変換)

関連データ構造

  • Datum: PostgreSQL の汎用データ表現。
  • PG_FUNCTION_ARGS: 関数呼び出しの引数コンテキスト。

失敗モードと境界条件

  • 指定された OID に対応するビューが存在しない場合、pg_get_viewdef_worker が NULL を返し、本関数も NULL を返す。

保守担当者向けメモ

  • 本関数は pg_get_viewdef 系のラッパーであり、主に折り返し幅の制御を目的としている。
  • 逆コンパイルのロジック自体は pg_get_viewdef_worker に依存しているため、出力形式の変更が必要な場合はそちらを参照すること。
