main

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.c:87

目的

PostgreSQL のメジャーバージョンアップグレード処理のメインエントリーポイントとして、旧クラスタから新クラスタへのデータ移行プロセス全体をオーケストレーションする。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**: コマンドライン引数 (argc, argv)。
  • **出力**: 正常終了時に 0 を返し、異常終了時には pg_fatal 等を通じてエラーメッセージを出力し終了する。

前提条件と不変条件

  • 実行環境において、新旧クラスタのデータディレクトリへのアクセス権限が適切であること。
  • pg_upgrade の実行前に、環境変数や引数を通じて新旧クラスタのパスが正しく指定されていること。
  • umask が PG_MODE_MASK_OWNER に設定され、ディレクトリ作成時の権限が制御されていること。

アルゴリズムまたは振る舞い

1. **初期化**: ログ出力の初期化、ロケール設定、コマンドライン引数の解析、データディレクトリの調整を行う。 2. **検証**: 新旧クラスタの互換性チェック、バージョン確認、ソケットディレクトリの取得を行う。 3. **準備**: 新クラスタのディレクトリ作成、pg_resetwal による初期化、ロケールとエンコーディングの設定を行う。 4. **移行**:

  • 旧クラスタのダンプと新クラスタへのスキーマ復元。
  • トランザクションログ（XID/WAL）の引き継ぎ。
  • テーブルスペースの転送（リンクまたはコピー）。
  • pg_resetwal を用いた次期 OID の設定。
  • 論理レプリケーションスロットの移行。

    • 5. **最終化**: 必要に応じたデータ同期、旧クラスタ削除用スクリプトの生成、警告の出力、一時ディレクトリのクリーンアップを行う。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（エントリーポイントのため）。
  • **呼び出し先**: pg_logging_init, parseCommandLine, make_outputdirs, setup, check_cluster_compatibility, start_postmaster, stop_postmaster, prepare_new_cluster, copy_xact_xlog_xid, create_new_objects, transfer_all_new_tablespaces, exec_prog 等、pg_upgrade モジュール内の各ユーティリティ関数。
  • **注記**: ツールが提示した contrib/amcheck 等の参照は、本関数とは無関係な「main」という名称の偶然の一致であり、本関数の呼び出し元ではない。

関連データ構造

  • old_cluster, new_cluster: アップグレード対象のクラスタ情報を保持するグローバル構造体。
  • user_opts: ユーザーが指定したアップグレードオプションを保持する構造体。

失敗モードと境界条件

  • **権限エラー**: 新旧クラスタのディレクトリ権限が読み取れない場合、pg_fatal により停止する。
  • **互換性エラー**: check_cluster_compatibility 等で不整合が検出された場合、アップグレードは中断される。
  • **転送モード**: TRANSFER_MODE_LINK または TRANSFER_MODE_SWAP を選択した場合、旧クラスタの無効化処理が実行される。

保守担当者向けメモ

  • 本関数はアップグレードの全工程を記述しているため、処理順序（特に start_postmaster / stop_postmaster のタイミング）の変更には注意が必要。
  • WIN32 固有の処理が含まれる場合があるため、プラットフォーム依存の挙動を確認すること。
  • pg_resetwal の呼び出しは、スキーマ復元後かつ論理スロット移行前に行う必要がある。
