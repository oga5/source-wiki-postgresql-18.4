pg_popcount_fast

定義

  • 種別: function
  • 定義箇所: src/port/pg_bitutils.c:253

目的

指定されたメモリバッファ内の全バイトに含まれる「1」のビット数を合計（ポピュレーションカウント）します。

シグネチャまたは形状

static uint64 pg_popcount_fast(const char *buf, int bytes)

入力と出力

  • **入力**:
  • buf: カウント対象のデータが格納されたメモリ領域へのポインタ。
  • bytes: バッファのサイズ（バイト単位）。
  • **出力**:
  • uint64: バッファ全体に含まれる「1」のビット数の総和。

前提条件と不変条件

  • bytes が負の値である場合、ループは実行されず 0 を返します。
  • バッファが 8 バイト（64ビット環境）または 4 バイト（32ビット環境）でアライメントされている場合、ワード単位での高速なカウント処理が適用されます。

アルゴリズムまたは振る舞い

1. **ワード単位の処理**:

  • SIZEOF_VOID_P >= 8 の環境では、バッファが 8 バイト境界にアライメントされていれば、pg_popcount64_fast を用いて 8 バイトずつ処理します。
  • それ以外の環境では、バッファが 4 バイト境界にアライメントされていれば、pg_popcount32_fast を用いて 4 バイトずつ処理します。

    • 2. **残余バイトの処理**:

  • ワード単位の処理で扱いきれなかった残りのバイト（またはアライメントされていないバッファ）に対して、pg_number_of_ones ルックアップテーブルを使用して 1 バイトずつビット数を加算します。

    • 3. **戻り値**: 累積された合計値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/pg_bitutils.c 内の初期化処理において、pg_popcount_optimized 関数ポインタへ代入されます。
  • **呼び出し先**:
  • pg_popcount64_fast または pg_popcount32_fast（環境依存）。
  • pg_number_of_ones（ルックアップテーブル）。

関連データ構造

  • pg_number_of_ones: 0-255 の各値に対するビット数を保持するルックアップテーブル。

失敗モードと境界条件

  • buf が NULL の場合、未定義の動作を引き起こす可能性があります。
  • bytes が 0 の場合、ループは実行されず 0 を返します。

保守担当者向けメモ

  • 本関数は pg_popcount_optimized を介して利用されることを想定しています。
  • アライメント判定には TYPEALIGN マクロが使用されています。
  • コンパイル時の環境（SIZEOF_VOID_P）によって、使用されるワードサイズと呼び出される高速化関数が切り替わります。
