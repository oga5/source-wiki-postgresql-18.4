compare4

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conv.c:338

目的

bsearch 関数で使用するための比較関数であり、uint32 型のコードポイントと pg_local_to_utf_combined 構造体の配列要素を比較して、結合文字の検索をサポートします。

シグネチャまたは形状

static int compare4(const void *p1, const void *p2)

入力と出力

  • **入力**:
  • p1: 比較対象の uint32 値へのポインタ。
  • p2: 比較対象の pg_local_to_utf_combined 構造体へのポインタ。
  • **出力**:
  • 1: p1 の値が p2 の code フィールドより大きい場合。
  • 0: 両者が等しい場合。
  • -1: p1 の値が p2 の code フィールドより小さい場合。

前提条件と不変条件

  • p2 が指すメモリ領域は pg_local_to_utf_combined 構造体として正しくキャスト可能である必要があります。
  • p1 は uint32 型の値を指している必要があります。

アルゴリズムまたは振る舞い

1. p1 が指す値を uint32 型の変数 v1 に読み込みます。 2. p2 を pg_local_to_utf_combined 型にキャストし、その code フィールドを v2 に読み込みます。 3. v1 と v2 を比較し、標準的な比較結果（1, 0, -1）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mb/conv.c 内の bsearch を呼び出す箇所（例: 800行目付近）。
  • **呼び出し先**: 特になし（標準的な比較ロジックのみを実行）。

関連データ構造

  • pg_local_to_utf_combined: p2 が指す構造体型。

失敗モードと境界条件

  • p1 または p2 が NULL の場合、ポインタ参照時に未定義の動作が発生します。
  • p2 が指す構造体のメモリレイアウトが code フィールドを先頭に持たない場合、意図しない値が読み込まれる可能性があります。

保守担当者向けメモ

  • 本関数は bsearch 用のコールバックであるため、シグネチャを変更する際は bsearch の仕様に適合させる必要があります。
  • 比較対象のデータ構造 pg_local_to_utf_combined の定義が変更された場合は、本関数のキャストおよびメンバアクセスが正しいか確認してください。
