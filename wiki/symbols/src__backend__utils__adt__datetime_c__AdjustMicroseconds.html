AdjustMicroseconds

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/datetime.c:627

目的

指定された値（整数部および浮動小数点部）とスケール係数を用いて、pg_itm_in 構造体のマイクロ秒フィールドを更新します。

シグネチャまたは形状

static bool AdjustMicroseconds(int64 val, double fval, int64 scale, struct pg_itm_in *itm_in)

入力と出力

  • **入力**:
  • val: 加算対象の整数値。
  • fval: 加算対象の浮動小数点値。
  • scale: 値をスケーリングするための係数。
  • itm_in: 更新対象の pg_itm_in 構造体へのポインタ。
  • **出力**:
  • 成功した場合は true、オーバーフロー等により失敗した場合は false を返します。
  • itm_in->tm_usec が更新されます。

前提条件と不変条件

  • itm_in は有効な pg_itm_in 構造体を指している必要があります。
  • int64_multiply_add が内部で使用されており、計算結果が int64 の範囲内に収まる必要があります。

アルゴリズムまたは振る舞い

1. val に scale を乗算し、その結果を itm_in->tm_usec に加算します。この際 int64_multiply_add を使用してオーバーフローチェックを行います。 2. 続いて AdjustFractMicroseconds を呼び出し、fval と scale を用いて残りの端数処理を行います。 3. 両方の処理が成功した場合にのみ true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/datetime.c 内の複数の箇所（行 3688, 3694, 3700, 3714, 3720, 4092, 4096, 4100, 4106, 4107）から呼び出されています。
  • **呼び出し先**:
  • int64_multiply_add
  • AdjustFractMicroseconds

関連データ構造

  • pg_itm_in: 日時計算用の中間構造体。

失敗モードと境界条件

  • int64_multiply_add がオーバーフローを検出した場合、false を返して処理を中断します。
  • AdjustFractMicroseconds が失敗した場合、その結果を伝播して false を返します。

保守担当者向けメモ

  • 本関数は日時計算の低レベルな算術処理を担います。計算ロジックを変更する際は、呼び出し元で指定される scale（USECS_PER_SEC 等）との整合性に注意してください。
