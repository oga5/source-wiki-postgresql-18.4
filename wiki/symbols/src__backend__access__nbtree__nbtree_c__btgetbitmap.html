btgetbitmap

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtree.c:287

目的

B-tree インデックスを走査し、条件に一致するすべてのタプルを TIDBitmap に追加します。

シグネチャまたは形状

int64 btgetbitmap(IndexScanDesc scan, TIDBitmap *tbm)

入力と出力

  • **入力**:
  • scan: インデックススキャンの状態を保持する IndexScanDesc。
  • tbm: 抽出したタプル ID を格納する TIDBitmap。
  • **出力**:
  • int64: ビットマップに追加されたタプルの総数。

前提条件と不変条件

  • scan->heapRelation が NULL であること（アサーションにより検証）。
  • so->currPos を通じて現在のスキャン位置が管理されていること。

アルゴリズムまたは振る舞い

1. _bt_first を呼び出し、スキャンの初期位置を特定します。 2. ページ内のタプルを順次走査し、tbm_add_tuples を用いてビットマップにタプル ID を追加します。 3. ページ内の全タプルを処理した後、_bt_next を呼び出して次のページへ移動します。 4. so->numArrayKeys が設定されている場合、_bt_start_prim_scan を用いて複数のプリミティブスキャンを繰り返し実行します。 5. 最終的に追加したタプルの総数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtree.c 内の IndexAmRoutine 構造体（amgetbitmap フィールド）を通じて呼び出されます。
  • **呼び出し先**:
  • _bt_first: スキャンの開始。
  • _bt_next: 次のタプルへの移動。
  • _bt_start_prim_scan: プリミティブスキャンの開始。
  • tbm_add_tuples: ビットマップへのタプル追加。

関連データ構造

  • IndexScanDesc: スキャン全体の制御構造。
  • BTScanOpaque: B-tree 特有のスキャン状態（scan->opaque）。
  • TIDBitmap: タプル ID の集合を保持するビットマップ構造。

失敗モードと境界条件

  • インデックスが空の場合、_bt_first が false を返し、ビットマップには何も追加されず 0 が返されます。
  • プリミティブスキャンが複数必要な場合、すべてのスキャンが完了するまでループが継続されます。

保守担当者向けメモ

  • 本関数は IndexAmRoutine の一部として実装されています。
  • ページ内の移動ロジックは _bt_next に依存しているため、スキャン順序やページ境界の処理に変更を加える際は nbtsearch.c の実装と整合性を確認してください。
