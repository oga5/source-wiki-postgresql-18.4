cfb_process

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-cfb.c:196

目的

PGP CFB（Cipher Feedback）モードにおける暗号化および復号処理のメインループを実行し、入力データに対してブロック暗号を用いた XOR 演算とフィードバックレジスタの更新を行う。

シグネチャまたは形状

static int cfb_process(PGP_CFB *ctx, const uint8 *data, int len, uint8 *dst, mix_data_t mix_data)

入力と出力

  • **入力**:
  • ctx: 暗号化コンテキスト（PGP_CFB）。
  • data: 処理対象の入力データバッファ。
  • len: 入力データの長さ。
  • dst: 処理結果を格納する出力バッファ。
  • mix_data: データの XOR 処理を行う関数ポインタ（mix_encrypt_normal 等）。
  • **出力**:
  • 戻り値: 常に 0 を返す。
  • dst: 処理済みのデータが書き込まれる。
  • ctx: 内部状態（フィードバックレジスタ fr、ブロック位置 pos、ブロック番号 block_no）が更新される。

前提条件と不変条件

  • ctx は pgp_cfb_create によって適切に初期化されていること。
  • mix_data は、指定された暗号化/復号モード（通常または再同期）に従って XOR 演算を行う関数であること。
  • ctx->block_size は使用する暗号アルゴリズムのブロックサイズと一致していること。

アルゴリズムまたは振る舞い

1. **部分ブロック処理**: ctx->pos > 0 の場合、前回の残りのブロックに対して mix_data を適用する。ブロックが満たされたら ctx->encbuf を ctx->fr にコピーし、ctx->pos をリセットする。 2. **メインループ**: len > 0 の間、以下の処理を繰り返す。

  • px_cipher_encrypt を呼び出し、現在のフィードバックレジスタ ctx->fr を暗号化して ctx->fre を生成する。
  • ctx->block_no が 5 未満であればインクリメントする。
  • mix_data を呼び出し、入力データと ctx->fre の XOR を取って出力を作成する。
  • ブロック境界に達した場合、ctx->encbuf を ctx->fr にコピーし、ctx->pos をリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-cfb.c 内の pgp_cfb_encrypt および pgp_cfb_decrypt から呼び出される。
  • **呼び出し先**:
  • mix_data（引数として渡された関数ポインタ）
  • px_cipher_encrypt（px.h で定義された暗号化関数）

関連データ構造

  • PGP_CFB: 暗号化コンテキスト。フィードバックレジスタ、暗号器、現在のブロック位置、バッファ等を保持する。

失敗モードと境界条件

  • len が 0 の場合、ループは実行されず即座に終了する。
  • ctx->block_no が 5 に達すると、それ以上のインクリメントは停止する。
  • mix_data の内部実装に依存するが、本関数自体は戻り値としてエラーコードを返さない設計となっている。

保守担当者向けメモ

  • 本関数は mix_data を介して暗号化と復号、および通常モードと再同期モードの差異を抽象化している。
  • resync モード時の特殊なレジスタ更新ロジックは mix_data 側にカプセル化されているため、ロジック変更時は mix_encrypt_resync 等の関連関数を確認すること。
