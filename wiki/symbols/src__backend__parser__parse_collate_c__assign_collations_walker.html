assign_collations_walker

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_collate.c:254

目的

構文木を再帰的に走査し、各ノードに対して適切な照合順序（Collation）を決定・付与する。式解析後の後処理として、照合順序の伝播や競合検出を行う。

シグネチャまたは形状

static bool assign_collations_walker(Node *node, assign_collations_context *context)

入力と出力

  • **入力**:
  • node: 走査対象の構文木ノード。
  • context: 照合順序の状態（pstate, collation, strength 等）を保持するコンテキスト構造体。
  • **出力**:
  • bool: 走査を継続する場合は false、ノードの処理が完了し子ノードの走査が不要な場合は true（または特定のケースで false を返して終了）。

前提条件と不変条件

  • node が NULL の場合は何もしない。
  • 各ノードの処理において、再帰呼び出しの前にローカルな assign_collations_context を初期化し、状態を保護する必要がある。
  • 照合順序が不要なノード（RowExpr 等）は、子ノードの走査後に早期リターンする。

アルゴリズムまたは振る舞い

1. **初期化**: ノードごとにローカルコンテキストを作成し、照合順序情報を初期化する。 2. **再帰走査**: expression_tree_walker を使用して、ノードの種類に応じた適切なタイミングで子ノードを再帰的に処理する。 3. **ノード別処理**:

  • T_CollateExpr: 明示的な照合順序を適用し、COLLATE_EXPLICIT を設定する。
  • T_FieldSelect: フィールドの照合順序を継承し、COLLATE_IMPLICIT を設定する。
  • T_RowExpr: 子ノードの照合順序を個別に割り当て、自身は照合順序を持たないため終了する。
  • T_RowCompareExpr: 各列ペアの共通照合順序を特定し、自身は照合順序を持たないため終了する。
  • T_CoerceToDomain: ドメイン定義の照合順序を優先し、デフォルトの場合は子ノードの照合順序をバブルアップさせる。

呼び出し元と呼び出し先

  • **呼び出し元**: assign_query_collations を起点として、expression_tree_walker を介して再帰的に呼び出される。
  • **呼び出し先**: expression_tree_walker（再帰用）、select_common_collation（RowCompareExpr 用）、assign_list_collations（RowExpr 用）、get_typcollation（CoerceToDomain 用）。

関連データ構造

  • assign_collations_context: 走査中の状態保持用。
  • CollateStrength: 照合順序の派生強度（NONE, IMPLICIT, CONFLICT, EXPLICIT）。

失敗モードと境界条件

  • 照合順序の競合が発生した場合、COLLATE_CONFLICT が設定される。
  • 照合順序を持たない型（複合型など）に対しては InvalidOid を扱い、伝播を停止する。

保守担当者向けメモ

  • Aggref や Hypothetical-set 集約関数などの特殊ノードの処理は複雑なため、修正時は parse_collate.c 内の関連関数との整合性に注意すること。
  • expression_tree_walker の再帰順序が照合順序の伝播に影響するため、ノードごとの再帰タイミング（前処理か後処理か）を慎重に扱うこと。
