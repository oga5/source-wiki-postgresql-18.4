lclTocEntry

定義

  • **種別**: struct
  • **定義箇所**: src/bin/pg_dump/pg_backup_directory.c:61

目的

ディレクトリ形式アーカイブにおいて、各TOC（Table of Contents）エントリに対応するデータファイルのファイル名を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
    char *filename; /* ディレクトリを除いたベース名 */
} lclTocEntry;

入力と出力

  • **入力**: pg_malloc0 等によりメモリが割り当てられ、filename メンバに文字列が格納されます。
  • **出力**: te->formatData を通じて、アーカイブ内の各エントリに関連付けられたデータとして参照されます。

前提条件と不変条件

  • filename はディレクトリパスを含まないベース名である必要があります。
  • te->formatData にキャストして格納されるため、メモリ管理は呼び出し側のライフサイクルに依存します。

アルゴリズムまたは振る舞い

ディレクトリ形式のアーカイブ処理において、特定のTOCエントリがどのファイルに書き込まれるか、あるいはどのファイルから読み込まれるかを識別するメタデータとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_custom.c 内の複数の箇所で、te->formatData を介してこの構造体へのポインタが取得・操作されています。
  • **呼び出し先**: 提供されたソース情報からは、特定の関数呼び出し先は特定できません。

関連データ構造

  • ArchiveHandle: アーカイブ全体のハンドル。
  • TocEntry (te): lclTocEntry が formatData メンバとして格納される対象の構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時（pg_malloc0 呼び出し時）の挙動は、PostgreSQLの標準的なエラーハンドリングに従います。
  • filename が適切に初期化されていない場合、ファイル操作時にパス解決エラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は pg_backup_directory.c で定義されていますが、pg_backup_custom.c からも参照されています。修正を行う際は、両形式間でのデータ構造の整合性に注意してください。
