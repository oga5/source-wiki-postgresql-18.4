astreamer_gzip_writer_content

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_gzip.c:149

目的

astreamer インターフェースを介して渡されたデータを gzip 圧縮し、関連付けられたファイルへ書き出す。

シグネチャまたは形状

static void astreamer_gzip_writer_content(
    astreamer *streamer,
    astreamer_member *member,
    const char *data,
    int len,
    astreamer_archive_context context
)

入力と出力

  • **入力**:
  • streamer: astreamer_gzip_writer 型にキャストされるストリーマー構造体。
  • data: 書き込み対象のデータバッファ。
  • len: 書き込むデータの長さ。
  • **出力**: なし（戻り値は void）。成功時は gzwrite を通じてファイルへ出力される。

前提条件と不変条件

  • streamer は astreamer_gzip_writer 構造体として正しく初期化されている必要がある。
  • len が 0 の場合、本関数は何もせず即座に復帰する。

アルゴリズムまたは振る舞い

1. streamer を astreamer_gzip_writer 型にキャストする。 2. len が 0 であれば処理を終了する。 3. gzwrite を呼び出し、data を mystreamer->gzfile へ書き込む。 4. 書き込み結果が len と一致しない場合、エラー処理を行う。

  • errno が 0 の場合はディスク容量不足（ENOSPC）とみなす。
  • pg_fatal を呼び出し、エラーメッセージと共にプロセスを終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer インターフェースの content メソッドとして登録されている（src/fe_utils/astreamer_gzip.c:62）。具体的な呼び出し元はソースコード上では明示されていない。
  • **呼び出し先**:
  • gzwrite (zlib)
  • get_gz_error (本モジュール内)
  • pg_fatal (PostgreSQL ログ出力)

関連データ構造

  • astreamer_gzip_writer: gzFile を保持する構造体。

失敗モードと境界条件

  • **書き込み失敗**: gzwrite が要求したバイト数を書き込めなかった場合、pg_fatal により異常終了する。
  • **ディスク容量不足**: 書き込み失敗時に errno が設定されていない場合、ENOSPC として扱われる。

保守担当者向けメモ

  • 本関数は gzFile を直接操作するため、他の圧縮モジュールと異なり、圧縮結果を別の astreamer に渡すパイプライン構成には対応していない。
  • pg_fatal を使用しているため、エラー発生時は呼び出し元の制御フローに関わらずプロセスが終了する点に注意すること。
