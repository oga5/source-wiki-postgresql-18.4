gin_tsquery_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsginidx.c:213

目的

GINインデックスを用いた全文検索において、インデックスから抽出された候補行が、指定された tsquery の条件を実際に満たしているかを判定します。

シグネチャまたは形状

Datum gin_tsquery_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): bool *check（クエリ内の各オペランドの存在状態）
  • PG_GETARG_TSQUERY(2): TSQuery query（検索条件）
  • PG_GETARG_POINTER(4): Pointer *extra_data（追加のメタデータ）
  • PG_GETARG_POINTER(5): bool *recheck（再評価が必要かを示すフラグ）
  • **出力**:
  • Datum（bool 型の真偽値）: クエリ条件に合致するかを返却。
  • *recheck: インデックスのみでは判定不能な場合に true が設定されます。

前提条件と不変条件

  • query->size > 0 の場合にのみ判定処理が実行されます。
  • recheck は初期状態で false に設定されます。

アルゴリズムまたは振る舞い

1. GinChkVal 構造体を初期化し、クエリの各オペランドとインデックスの照合状態（check）をマッピングします。 2. TS_execute_ternary を呼び出し、checkcondition_gin をコールバックとして使用して論理評価を行います。 3. 評価結果に基づき以下の通り動作します：

  • TS_NO: res = false
  • TS_YES: res = true
  • TS_MAYBE: res = true かつ *recheck = true（インデックスだけでは判定不可のため、ヒープでの再評価を要求）。

    • 4. 最終的な res を PG_RETURN_BOOL で返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: GINインデックスの検索プロセスから呼び出されます（ソースコード内の直接的な呼び出し元は、互換性維持のためのスタブ関数 src/backend/utils/adt/tsginidx.c:332, 352 など）。
  • **呼び出し先**: TS_execute_ternary およびコールバック関数 checkcondition_gin を呼び出します。

関連データ構造

  • TSQuery: 全文検索クエリの構造体。
  • GinChkVal: TS_execute_ternary に渡すための検索状態保持用構造体。
  • GinTernaryValue: インデックス内のキー状態（真、偽、不明）。

失敗モードと境界条件

  • query->size が 0 の場合、判定は行われず false が返されます。
  • 位置情報や重み付けなど、インデックスのキー情報だけではクエリの完全な条件を満たせない場合、TS_MAYBE が返され、recheck フラグが有効になります。

保守担当者向けメモ

  • 本関数は TS_execute_ternary を介して論理演算を行うため、クエリの評価ロジックを変更する場合は checkcondition_gin の実装を確認してください。
  • recheck フラグの挙動は、インデックスの精度と検索パフォーマンスに直結するため、変更時は慎重に検討してください。
