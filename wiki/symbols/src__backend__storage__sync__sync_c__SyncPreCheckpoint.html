SyncPreCheckpoint

定義

  • 種別: function
  • 定義箇所: src/backend/storage/sync/sync.c:176

目的

チェックポイント開始直前の準備処理として、転送済みの削除要求を確定させ、サイクルカウンタを更新することで、チェックポイント処理の境界を明確にする。

シグネチャまたは形状

void SyncPreCheckpoint(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として内部状態の更新を行う）

前提条件と不変条件

  • AbsorbSyncRequests() が呼び出されることで、チェックポイント開始前に転送されたすべての unlink 要求が現在のチェックポイント処理対象として吸収される。
  • checkpoint_cycle_ctr は、チェックポイントの境界を管理するために使用される。

アルゴリズムまたは振る舞い

1. AbsorbSyncRequests() を呼び出し、保留中の同期および削除要求を現在のキューに取り込む。これにより、DROP TABLESPACE 等の操作で発行された要求が現在のチェックポイントで確実に処理されることを保証する。 2. checkpoint_cycle_ctr をインクリメントする。この時点以降に到着した削除要求は、次のサイクルカウンタが割り当てられ、次回のチェックポイントまで処理が持ち越される。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlog.c
  • 呼び出し先: AbsorbSyncRequests()（内部関数）

関連データ構造

  • checkpoint_cycle_ctr: チェックポイントのサイクルを識別するためのカウンタ。

失敗モードと境界条件

  • 境界条件: 本関数呼び出し後に発生した要求は、現在のチェックポイント処理の対象外となる。

保守担当者向けメモ

  • 本関数はチェックポイントの開始シーケンスにおいて重要な役割を果たす。checkpoint_cycle_ctr の更新タイミングを変更すると、チェックポイントの処理範囲に影響が出るため注意が必要である。
