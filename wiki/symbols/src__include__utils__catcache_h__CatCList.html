CatCList

定義

  • 種別: struct
  • 定義箇所: src/include/utils/catcache.h:159

目的

カタログキャッシュにおいて、部分一致検索や複数タプルを返す検索の結果を保持するためのリスト構造体です。

シグネチャまたは形状

typedef struct catclist {
    int         cl_magic;
    uint32      hash_value;
    dlist_node  cache_elem;
    Datum       keys[CATCACHE_MAXKEYS];
    int         refcount;
    bool        dead;
    bool        ordered;
    short       nkeys;
    int         n_members;
    CatCache   *my_cache;
    CatCTup    *members[FLEXIBLE_ARRAY_MEMBER];
} CatCList;

入力と出力

  • **入力**: キャッシュ検索時に生成される検索キー（keys）およびハッシュ値（hash_value）。
  • **出力**: 検索条件に合致する CatCTup の配列（members）。

前提条件と不変条件

  • cl_magic は CL_MAGIC (0x52765103) で初期化され、エントリの識別に使用されます。
  • refcount が 0 になった時点で、dead フラグが立っていれば物理的なメモリ解放が可能です。
  • keys 配列のうち、nkeys で指定された数までが有効な検索キーとして扱われます。

アルゴリズムまたは振る舞い

  • カタログキャッシュの検索結果をキャッシュし、同一条件の再検索を高速化します。
  • dlist_node を介して、所属する CatCache 内のリスト構造に連結されます。
  • members はフレキシブル配列メンバとして定義されており、検索結果のタプル数に応じて動的にメモリが確保されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/index/amvalidate.c (identify_opfamily_groups), src/backend/catalog/namespace.c, src/backend/access/spgist/spgutils.c, および各種 validate 関数（ginvalidate.c, nbtvalidate.c 等）から参照されています。
  • **呼び出し先**: 構造体定義のため、特定の関数呼び出しは含みません。

関連データ構造

  • CatCache: このリストを所有する親キャッシュ構造体。
  • CatCTup: リストの各要素として保持される個別のキャッシュタプル。
  • dlist_node: キャッシュ内のリスト管理に使用される双方向リストノード。

失敗モードと境界条件

  • refcount の管理不備はメモリリークを引き起こします。
  • dead フラグが設定されたエントリは、参照カウントがゼロになるまでメモリ上に残存します。

保守担当者向けメモ

  • members はフレキシブル配列であるため、構造体自体のサイズ計算やメモリ割り当て時には sizeof(CatCList) + n_members * sizeof(CatCTup *) のような計算が必要です。
  • ordered フラグは、リスト内のメンバがインデックス順に並んでいるかどうかを示します。
