r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_portuguese.c:507

目的

ポルトガル語のステミング処理において、単語内の領域（RV, R1, R2）の境界位置を特定し、SN_env 構造体のインデックス I[0] から I[2] に格納する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z（Snowball ランタイム環境）
  • 出力: 常に 1 を返す。
  • 副作用: z->I[0]（R2の開始位置）、z->I[1]（R1の開始位置）、z->I[2]（RVの開始位置）を更新する。

前提条件と不変条件

  • z が有効な SN_env 構造体を指していること。
  • g_v（母音グループ）が定義されていること。
  • 領域の境界は単語の長さ z->l を超えない。

アルゴリズムまたは振る舞い

1. **初期化**: I[0], I[1], I[2] を単語の末尾位置 z->l に初期化する。 2. **RV 領域の特定**:

  • 母音と子音の並びに基づき、RV の開始位置を検索する。
  • 特定の母音・子音の組み合わせパターン（in_grouping, out_grouping）に従い、条件に合致する位置を z->I[2] に設定する。

    • 3. **R1, R2 領域の特定**:

  • 母音と子音の遷移を順次スキャンし、R1 および R2 の境界を計算して z->I[1] および z->I[0] に格納する。

    • 4. **復元**: 処理中のカーソル位置 z->c を開始時の位置にリセットして終了する。

呼び出し元と呼び出し先

  • 呼び出し元: portuguese_ISO_8859_1_stem（直接の呼び出し関係はソースコードの制御フローから推論されるが、本抜粋には明示的な呼び出し元コードは含まれていない）。
  • 呼び出し先: in_grouping, out_grouping（Snowball ランタイムのユーティリティ関数）。

関連データ構造

  • struct SN_env: ステミング処理の環境状態を保持する構造体。I[] 配列に領域インデックスを格納する。

失敗モードと境界条件

  • 単語が短すぎる場合や、母音・子音のパターンが定義に合致しない場合、領域インデックスは初期値（単語末尾）のまま維持される。
  • in_grouping / out_grouping が失敗した場合は、各ステップでラベル（lab0～lab7）へジャンプし、部分的な処理をスキップする。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードである。ロジックの変更が必要な場合は、直接編集せず元の .sbl スクリプトを修正すること。
  • z->I[0..2] の値は、後続の接尾辞削除処理（r_standard_suffix 等）の条件判定において不可欠なインデックスである。
