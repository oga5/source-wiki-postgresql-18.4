conditional_stack_get_query_len

定義

  • 種別: function
  • 定義箇所: src/fe_utils/conditional.c:161

目的

ConditionalStack のスタックトップに記録されているクエリ長（query_len）を取得します。これは、非アクティブな \if ブロック内で蓄積されたバッファを復元または破棄する際に使用されます。

シグネチャまたは形状

int conditional_stack_get_query_len(ConditionalStack cstack)

入力と出力

  • **入力**: cstack (ConditionalStack 型) - 操作対象のスタック構造体。
  • **出力**: int 型。スタックトップの query_len 値、またはスタックが空の場合は -1 を返します。

前提条件と不変条件

  • cstack は有効な ConditionalStack オブジェクトである必要があります。
  • スタック内の各要素は、conditional_stack_set_query_len によって適切な値が設定されていることが想定されます。

アルゴリズムまたは振る舞い

1. conditional_stack_empty(cstack) を呼び出し、スタックが空かどうかを確認します。 2. スタックが空であれば -1 を返します。 3. スタックが空でなければ、スタックの先頭要素（cstack->head）が保持する query_len フィールドの値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c:3825 にて呼び出されています。
  • **呼び出し先**: conditional_stack_empty を内部で呼び出しています。

関連データ構造

  • ConditionalStack: スタック全体を管理する構造体。
  • query_len: ConditionalStack の各要素が保持するメタデータで、ブロック開始時のクエリバッファの長さを記録します。

失敗モードと境界条件

  • スタックが空の場合、エラーや例外を発生させる代わりに -1 を返します。

保守担当者向けメモ

  • 本関数は純粋なデータアクセス関数であり、状態の変更は行いません。
  • query_len の値は、\if ブロックのネスト状態に応じてクエリバッファを正しく管理するために不可欠です。
