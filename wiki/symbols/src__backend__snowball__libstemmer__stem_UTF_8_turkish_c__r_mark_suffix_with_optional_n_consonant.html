目的

トルコ語のステミング処理において、単語末尾の「n」子音の有無を考慮しながら特定の接尾辞パターンを照合・処理する。

シグネチャまたは形状

static int r_mark_suffix_with_optional_n_consonant(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（Snowball ステミング環境構造体）
  • 出力: int（成功時に 1、失敗時に 0 を返す）

前提条件と不変条件

  • z->p（文字列バッファ）、z->c（カーソル）、z->l（長さ）、z->lb（境界）が有効な状態であること。
  • g_vowel グループが定義されていること。

アルゴリズムまたは振る舞い

1. **パターンマッチング**:

  • 末尾が 'n' であるかを確認し、その直前に母音が存在するかを in_grouping_b_U で検証する。
  • 'n' が存在しない場合、あるいは 'n' の直前が母音でない場合は、別の条件分岐（lab1）へ遷移する。

    • 2. **条件分岐**:

  • lab1 では、'n' の有無を再確認し、さらに直前の文字が母音であるかを確認する処理を行う。
  • 最終的に条件を満たせば 1 を、満たさなければ 0 を返す。

    • 3. **状態復元**: 各ステップでカーソル位置（z->c）を保存・復元し、マッチング失敗時に元の状態へ戻る。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_turkish.c 内の行番号 707 および 793。
  • 呼び出し先: in_grouping_b_U（母音判定）、skip_b_utf8（UTF-8 文字のスキップ）。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • g_vowel: 母音を定義するグループ（in_grouping_b_U で使用）。

失敗モードと境界条件

  • z->c <= z->lb: カーソルが境界に達している場合はマッチング失敗。
  • skip_b_utf8 が負の値を返した場合、処理を中断し 0 を返す。
