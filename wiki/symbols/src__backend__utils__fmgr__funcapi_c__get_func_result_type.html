get_func_result_type

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/funcapi.c:409

目的

関数ID（Oid）に基づいて、その関数が返す結果の型クラス、型ID、およびタプル記述子（TupleDesc）を取得します。

シグネチャまたは形状

TypeFuncClass get_func_result_type(Oid functionId, Oid *resultTypeId, TupleDesc *resultTupleDesc)

入力と出力

  • **入力**:
  • functionId: 調査対象の関数の Oid。
  • resultTypeId: 戻り値の型IDを格納するためのポインタ（出力用）。
  • resultTupleDesc: 戻り値が複合型の場合、その構造を格納するためのポインタ（出力用）。
  • **出力**:
  • 戻り値: TypeFuncClass 型（TYPEFUNC_SCALAR, TYPEFUNC_COMPOSITE, TYPEFUNC_RECORD 等）。

前提条件と不変条件

  • functionId は有効な pg_proc カタログ内の関数を参照している必要があります。
  • 出力引数 resultTypeId および resultTupleDesc は、呼び出し元で確保された有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

内部関数 internal_get_result_type を呼び出し、式ツリー（call_expr）や引数情報（call_expr）を NULL として渡すことで、関数定義のみに基づいた静的な型解決を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_proc.c:979
  • **呼び出し先**:
  • internal_get_result_type（同一ファイル内）

関連データ構造

  • TypeFuncClass: 関数の戻り値の性質を示す列挙型。
  • TupleDesc: 複合型やレコードの構造を記述するデータ構造。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • 指定された functionId がシステムカタログに存在しない場合、エラーが発生する可能性があります。
  • 戻り値が TYPEFUNC_RECORD の場合、resultTupleDesc には具体的な構造が設定されない可能性があるため、呼び出し側での確認が必要です。

保守担当者向けメモ

  • 本関数は internal_get_result_type のラッパーです。より詳細なコンテキスト（式ツリー等）が必要な場合は、get_call_result_type の使用を検討してください。
