CachedFunctionHashKey

定義

  • 種別: struct
  • 定義箇所: src/include/utils/funccache.h:52

目的

関数実行時のメタデータやコンパイル済み情報をキャッシュする際、特定の関数呼び出しを一意に識別するためのハッシュキーとして機能します。

シグネチャまたは形状

typedef struct CachedFunctionHashKey
{
    Oid         funcOid;
    bool        isTrigger;
    bool        isEventTrigger;
    Size        cacheEntrySize;
    Oid         trigOid;
    Oid         inputCollation;
    int         nargs;
    TupleDesc   callResultType;
    Oid         argtypes[FUNC_MAX_ARGS];
} CachedFunctionHashKey;

入力と出力

  • **入力**: 関数呼び出し時の OID、トリガーフラグ、照合順序、引数型、戻り値の型記述子など。
  • **出力**: ハッシュテーブル検索および比較に使用される固定サイズの構造体。

前提条件と不変条件

  • 構造体内のパディングバイトはゼロクリアされている必要があります。
  • argtypes 配列は FUNC_MAX_ARGS のサイズを持ち、有効な引数は nargs 分のみです。
  • 構造体の後半部分（callResultType 以降）を変更する場合、funccache.c 内のハッシュ関数および比較ロジックの修正が必須です。

アルゴリズムまたは振る舞い

  • 関数実行エンジンがキャッシュを検索する際、現在の FunctionCallInfo から本構造体を生成し、ハッシュテーブルのキーとして使用します。
  • cacheEntrySize をキーに含めることで、言語実装の変更等によるキャッシュエントリサイズの不一致を検知します。
  • トリガー関数の場合、trigOid を含めることで、トリガーごとに個別のコンパイル結果を保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/functions.c および src/backend/utils/cache/funccache.c 内のキャッシュ管理ロジック。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接的な関数呼び出しは行いません。

関連データ構造

  • CachedFunction: キャッシュエントリの基底構造体。
  • TupleDesc: 複合型を返す関数の結果記述子。
  • FunctionCallInfo: 関数呼び出し情報。

失敗モードと境界条件

  • nargs が FUNC_MAX_ARGS を超える場合、またはハッシュキーの比較ロジックと構造体定義が同期していない場合に、キャッシュの誤ヒットや不整合が発生します。

保守担当者向けメモ

  • 本構造体は funccache.c のハッシュテーブル管理と密接に結合しています。フィールドの追加・削除時は、必ず funccache.c の memcmp 範囲やハッシュ計算ロジックを確認してください。
  • 構造体内のパディングバイトの扱いに注意してください。
