pg_copy_logical_replication_slot_b

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slotfuncs.c:866

目的

論理レプリケーションスロットをコピーするためのSQL関数インターフェースを提供します。

シグネチャまたは形状

Datum pg_copy_logical_replication_slot_b(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL経由で渡される引数。ソーススロット名、ターゲットスロット名などを含む）。
  • **出力**: Datum（コピーされたスロットの情報を含むタプル）。

前提条件と不変条件

  • 呼び出し元が適切な権限を持っている必要があります。
  • ソーススロットが有効であり、コピー可能な状態である必要があります。

アルゴリズムまたは振る舞い

1. copy_replication_slot 関数を呼び出します。 2. その際、第2引数に true を渡し、論理スロットとしてのコピー処理を実行します。 3. copy_replication_slot から返された結果をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（SQLコマンド経由での実行）。
  • **呼び出し先**: copy_replication_slot（内部ヘルパー関数）。

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • PG_FUNCTION_ARGS: 関数引数を受け取るためのマクロ。

失敗モードと境界条件

  • ソーススロットが存在しない場合、エラーが発生します。
  • ターゲットスロット名が既に存在する場合、エラーが発生します。
  • ソーススロットが無効化されている場合や、WAL予約が完了していない場合にはコピーが拒否されます。

保守担当者向けメモ

  • 本関数は copy_replication_slot への単なるラッパーです。ロジックの変更が必要な場合は、src/backend/replication/slotfuncs.c 内の copy_replication_slot を参照してください。
