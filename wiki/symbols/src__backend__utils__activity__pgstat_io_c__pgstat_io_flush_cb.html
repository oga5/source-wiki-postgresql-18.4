pgstat_io_flush_cb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_io.c:188

目的

バックエンドプロセス内で一時的に蓄積されたI/O統計（PendingIOStats）を、共有メモリ上のグローバルな統計領域へ同期（フラッシュ）します。

シグネチャまたは形状

bool pgstat_io_flush_cb(bool nowait)

入力と出力

  • **入力**:
  • nowait (bool): true の場合、共有メモリのロック取得に失敗した際に待機せず即座に制御を戻します。
  • **出力**:
  • bool: ロック取得に失敗してフラッシュをスキップした場合は true を返し、それ以外（フラッシュ完了または統計が空）の場合は false を返します。

前提条件と不変条件

  • have_iostats が false の場合、処理は即座に終了します。
  • 共有メモリへの書き込み時には、MyBackendType に対応する LWLock を排他モードで取得する必要があります。
  • 処理完了後、ローカルの PendingIOStats はゼロクリアされ、have_iostats は false に設定されます。

アルゴリズムまたは振る舞い

1. have_iostats を確認し、統計が存在しない場合は false を返します。 2. MyBackendType に基づき、共有メモリ上の対象統計領域とロックを取得します。 3. nowait が true でロック取得に失敗した場合は true を返します。それ以外は LWLockAcquire でロックを確保します。 4. 全てのI/Oオブジェクト、コンテキスト、操作の組み合わせについて、ローカルの PendingIOStats の値を共有メモリ上の bktype_shstats に加算します。 5. pgstat_bktype_io_stats_valid を用いて、更新後の統計が妥当であることを検証します。 6. ロックを解放し、ローカルの PendingIOStats を memset で初期化し、have_iostats を false に更新して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/pgstat.c:443 (統計サブシステムのコールバック登録)
  • src/backend/utils/activity/pgstat_io.c:177 (ラッパー関数経由)
  • **呼び出し先**:
  • LWLockAcquire / LWLockConditionalAcquire / LWLockRelease
  • pgstat_bktype_io_stats_valid
  • INSTR_TIME_GET_MICROSEC (マクロ)

関連データ構造

  • PendingIOStats: ローカルの統計蓄積用変数。
  • PgStat_BktypeIO: 共有メモリ上のバックエンドタイプ別I/O統計構造体。
  • pgStatLocal.shmem: 統計サブシステムの共有メモリ領域。

失敗モードと境界条件

  • nowait が true の場合、ロック競合が発生すると統計のフラッシュがスキップされます。
  • pgstat_bktype_io_stats_valid による検証で不正な状態が検出された場合、Assert により停止する可能性があります。

保守担当者向けメモ

  • 本関数は pgstat サブシステムのコールバックとして登録されており、統計の永続化において重要な役割を果たします。
  • PendingIOStats のリセット処理はロック解放後に行われるため、この間は統計の不整合が発生しないよう注意が必要です。
