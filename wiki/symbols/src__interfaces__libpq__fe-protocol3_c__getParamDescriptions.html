getParamDescriptions

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-protocol3.c:688

目的

PostgreSQL プロトコル 3.0 における ParameterDescription メッセージ（'t'）を解析し、クエリのパラメータ型情報を PGresult オブジェクトに格納します。

シグネチャまたは形状

static int getParamDescriptions(PGconn *conn, int msgLength)

入力と出力

  • **入力**:
  • conn: 現在の接続状態を保持する PGconn 構造体。
  • msgLength: 受信したメッセージの長さ。
  • **出力**:
  • 成功時は 0 を返し、conn->result にパラメータ情報を含む PGresult を設定します。
  • 失敗時（メモリ不足やデータ不足）も 0 を返しますが、conn->errorMessage にエラー内容を記録し、conn->result にエラー状態の PGresult を設定します。

前提条件と不変条件

  • メッセージタイプ（'t'）とメッセージ長は、呼び出し元である pqParseInput3 によって既に読み取り済みであること。
  • conn->inCursor は、メッセージのパラメータ数フィールドの先頭を指していること。

アルゴリズムまたは振る舞い

1. PQmakeEmptyPGresult で空の PGresult を作成します。 2. メッセージからパラメータ数（2バイト）を読み取ります。 3. パラメータ数が 0 より大きい場合、PGresParamDesc 配列を result 内に割り当て、ゼロクリアします。 4. パラメータ数分だけループし、各パラメータの型 ID（4バイト）を読み取って result->paramDescs に格納します。 5. 処理が完了すると、構築した result を conn->result に割り当てます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-protocol3.c 内のメッセージ解析ループ（pqParseInput3 からのディスパッチ）。
  • **呼び出し先**: PQmakeEmptyPGresult, pqGetInt, pqResultAlloc, MemSet, libpq_gettext, pqClearAsyncResult, appendPQExpBuffer, pqSaveErrorResult, PQclear（エラー時）。

関連データ構造

  • PGconn: 接続状態および入力バッファを管理。
  • PGresult: 解析結果を保持する構造体。
  • PGresParamDesc: パラメータの型情報（typid）を保持する構造体。

失敗モードと境界条件

  • **データ不足**: メッセージ長が期待されるパラメータ数や型 ID の読み取りに足りない場合、エラーとして処理されます。
  • **メモリ不足**: PGresult やパラメータ記述配列の割り当てに失敗した場合、エラーメッセージを記録し、既存の result を解放してメモリを回収します。
  • **メッセージ消費**: 失敗時であっても、conn->inCursor をメッセージの末尾まで進めることで、後続のメッセージ解析への影響を防ぎます。

保守担当者向けメモ

  • 本関数はプロトコル 3.0 専用です。
  • エラー発生時に gettext() を呼び出す前に古い result を解放することで、メモリ不足下での翻訳メッセージ生成成功率を高める工夫がなされています。
