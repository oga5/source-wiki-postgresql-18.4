PLy_procedure_create

定義

  • src/pl/plpython/plpy_procedure.c:129

目的

PL/Python関数またはトリガーの実行に必要なメタデータとメモリコンテキストを初期化し、PLyProcedure 構造体を生成する。

シグネチャまたは形状

static PLyProcedure *PLy_procedure_create(HeapTuple procTup, Oid fn_oid, bool is_trigger)

入力と出力

  • **入力**:
  • procTup: pg_proc システムカタログのタプル。
  • fn_oid: 関数のOID。
  • is_trigger: トリガー関数かどうかを示すフラグ。
  • **出力**:
  • 初期化済みの PLyProcedure 構造体へのポインタ。

前提条件と不変条件

  • TopMemoryContext が存在すること。
  • procTup は有効な pg_proc タプルであること。
  • 戻り値および引数の型が擬似型（pseudotype）である場合、特定の例外（void, record 等）を除きエラーとなる。

アルゴリズムまたは振る舞い

1. **名前生成**: __plpython_procedure_<name>_<oid> 形式のPython用関数名を生成し、非英数字を _ に置換する。 2. **メモリ確保**: 関数専用の長寿命な MemoryContext を作成し、PLyProcedure 構造体を割り当てる。 3. **メタデータ設定**: pg_proc から関数名、xmin、tid、揮発性、戻り値のセット属性、言語ID等をコピーする。 4. **型変換設定**:

  • 非トリガー関数の場合、戻り値の型変換関数を PLy_output_setup_func で設定する。
  • 引数がある場合、get_func_arg_info で引数情報を取得し、入力変換用の PLyDatumToOb 配列を構築する。

    • 5. **エラーハンドリング**: PG_TRY ブロック内で実行され、キャッシュルックアップ失敗時などはエラーを投げる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_procedure.c 内の PLy_procedure_get（直接の証拠あり）。
  • **呼び出し先**: GETSTRUCT, snprintf, AllocSetContextCreate, MemoryContextSwitchTo, palloc0, SysCacheGetAttr, SearchSysCache1, PLy_output_setup_func, get_func_arg_info, format_type_be 等。

関連データ構造

  • PLyProcedure: 関数実行状態を保持する主要構造体。
  • HeapTuple: pg_proc カタログ情報。
  • MemoryContext: 関数ごとのリソース管理用コンテキスト。

失敗モードと境界条件

  • **バッファオーバーラン**: 生成された関数名が NAMEDATALEN + 256 を超える場合、elog(ERROR) を発生させる。
  • **キャッシュルックアップ失敗**: 型情報の取得に失敗した場合、elog(ERROR) を発生させる。
  • **サポート外の型**: 擬似型（pseudotype）が引数や戻り値に使用されている場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラーとなる。

保守担当者向けメモ

  • 本関数は PLyProcedure の初期化のみを担当し、実際のソースコードのコンパイルは PLy_procedure_compile が担う。
  • is_trigger が真の場合、戻り値の型変換設定はスキップされ、実行時に PLy_exec_trigger が処理を行う設計となっている。
