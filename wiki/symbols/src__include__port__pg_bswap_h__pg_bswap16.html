pg_bswap16

定義

  • 種別: static inline function
  • 定義箇所: src/include/port/pg_bswap.h:41

目的

16ビット符号なし整数のバイト順序を反転（エンディアン変換）します。

シグネチャまたは形状

static inline uint16 pg_bswap16(uint16 x)

入力と出力

  • **入力**: uint16 x （変換対象の16ビット符号なし整数）
  • **出力**: uint16 （バイト順序が反転された値）

前提条件と不変条件

  • 入力値は uint16 型である必要があります。
  • 本関数は、コンパイラ組み込み関数（__builtin_bswap16 や _byteswap_ushort）が利用できない環境におけるフォールバック実装として提供されます。

アルゴリズムまたは振る舞い

入力値 x に対して以下のビット演算を行い、バイト順序を入れ替えます。 1. x を左に8ビットシフトし、下位8ビットをマスク（0xff00）で抽出。 2. x を右に8ビットシフトし、上位8ビットをマスク（0x00ff）で抽出。 3. 上記2つの結果を論理和（|）で結合して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • pg_hton16 マクロ（src/include/port/pg_bswap.h:120）
  • pg_ntoh16 マクロ（src/include/port/pg_bswap.h:124）
  • **呼び出し先**: なし（純粋なビット演算のみを実行）

関連データ構造

  • uint16（c.h 等で定義される標準的な16ビット符号なし整数型）

失敗モードと境界条件

  • 符号付き整数を渡した場合、予期せぬ動作が発生する可能性があるため、事前にキャストが必要です。
  • コンパイル環境によっては、本関数ではなくプリプロセッサマクロによって __builtin_bswap16 等の組み込み関数に置換されるため、本実装が実行されない場合があります。

保守担当者向けメモ

  • 本関数はフォールバック用です。パフォーマンス向上のため、可能な限りコンパイラ組み込み関数が優先される設計になっています。
  • 修正を行う際は、pg_bswap32 や pg_bswap64 と同様の設計方針に従ってください。
