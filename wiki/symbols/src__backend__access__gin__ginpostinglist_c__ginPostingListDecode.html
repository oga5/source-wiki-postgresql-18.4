ginPostingListDecode

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginpostinglist.c:283

目的

圧縮された GinPostingList 構造体から、元の ItemPointer（TID）配列を復元します。

シグネチャまたは形状

ItemPointer ginPostingListDecode(GinPostingList *plist, int *ndecoded_out);

入力と出力

  • **入力**:
  • plist: 展開対象となる圧縮済みポスティングリストへのポインタ。
  • ndecoded_out: 復元されたアイテム数を格納するための出力用整数ポインタ。
  • **出力**:
  • 復元された ItemPointer の配列を返します。

前提条件と不変条件

  • plist は有効な GinPostingList 構造体を指している必要があります。
  • 内部的に SizeOfGinPostingList(plist) を使用してリスト全体のサイズを算出するため、構造体ヘッダが整合している必要があります。

アルゴリズムまたは振る舞い

1. SizeOfGinPostingList(plist) を呼び出し、対象リストのサイズを特定します。 2. ginPostingListDecodeAllSegments を呼び出し、圧縮された全セグメントを走査・展開します。 3. 展開された結果（ItemPointer 配列）と、復元されたアイテム数（ndecoded_out）を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c
  • contrib/pageinspect/ginfuncs.c
  • src/backend/access/gin/gindatapage.c
  • src/backend/access/gin/ginentrypage.c
  • src/backend/access/gin/ginpostinglist.c (自己参照)
  • src/backend/access/gin/ginvacuum.c
  • **呼び出し先**:
  • ginPostingListDecodeAllSegments

関連データ構造

  • GinPostingList: GINインデックスでTIDを格納するために使用される圧縮済みデータ構造。
  • ItemPointer: PostgreSQLのタプル識別子（TID）。

失敗モードと境界条件

  • plist が不正なメモリ領域を指している場合、セグメンテーション違反が発生する可能性があります。
  • 圧縮データが破損している場合、ginPostingListDecodeAllSegments 経由で不正な値が復元されるか、デコード処理中にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は ginPostingListDecodeAllSegments への単なるラッパーです。
  • 圧縮アルゴリズムの変更や GinPostingList の構造変更を行う際は、本関数が依存する SizeOfGinPostingList マクロおよび ginPostingListDecodeAllSegments の実装との整合性に注意してください。
