ECPGdeallocate_desc

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/ecpg/ecpglib/descriptor.c:755

目的

スレッドローカルストレージに保持されている SQL 記述子（Descriptor）を名前で検索し、メモリから解放する。

シグネチャまたは形状

bool ECPGdeallocate_desc(int line, const char *name)
  • line: 呼び出し元のソースコード行番号（エラー報告用）。
  • name: 解放対象の記述子名。
  • 戻り値: 成功時に true、失敗時に false を返す。

入力と出力

  • **入力**: name で指定された記述子名。
  • **出力**: 成功時は該当する記述子をリストから削除しメモリを解放する。失敗時は sqlca にエラー情報を設定する。

前提条件と不変条件

  • ECPGget_sqlca() が有効な sqlca_t 構造体を返すこと。
  • スレッドローカルな記述子リストが get_descriptors() を通じてアクセス可能であること。

アルゴリズムまたは振る舞い

1. ECPGget_sqlca() で現在のスレッドの sqlca を取得し、初期化する。 2. get_descriptors() を呼び出し、スレッドローカルな記述子リストを走査する。 3. name と一致する記述子を検索する。 4. 一致した記述子が見つかった場合：

  • リストのリンクを繋ぎ変えて対象をリストから除外する。
  • descriptor_free(desc) を呼び出し、メモリを解放する。
  • true を返して終了する。

    • 5. 一致するものが見つからない場合：

  • ECPG_UNKNOWN_DESCRIPTOR エラーを発生させ、false を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPG プリプロセッサが生成した C コード（例: compat_informix-describe.c, preproc-describe.c 等のテストコード内）。
  • **呼び出し先**:
  • ECPGget_sqlca()
  • ecpg_init_sqlca()
  • get_descriptors()
  • set_descriptors()
  • descriptor_free()
  • ecpg_raise()

関連データ構造

  • struct descriptor: 記述子の実体。リスト構造を成す next ポインタを持つ。
  • struct sqlca_t: SQL 通信領域。

失敗モードと境界条件

  • ECPGget_sqlca() が NULL を返した場合、ECPG_OUT_OF_MEMORY エラーとなる。
  • 指定した name の記述子がリスト内に存在しない場合、ECPG_UNKNOWN_DESCRIPTOR エラーとなる。

保守担当者向けメモ

  • 本関数はスレッドセーフであり、descriptor_key を介したスレッドローカルな管理下で動作する。
  • 記述子の削除後にリストの整合性が保たれるよう、prev ポインタを用いたリンクの繋ぎ変え処理が適切に行われているか注意すること。
