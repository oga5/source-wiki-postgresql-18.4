FSMAddress

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/freespace/freespace.c:84

目的

Free Space Map (FSM) の木構造における論理的な位置（階層とページ番号）を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	int			level;			/* level */
	int			logpageno;		/* page number within the level */
} FSMAddress;

入力と出力

  • **入力**: 構造体の各メンバ（level および logpageno）は整数値として定義されます。
  • **出力**: 本構造体は、FSM の木構造内での特定のノードを指し示す論理アドレスとして機能します。

前提条件と不変条件

  • level は FSM の木構造における階層を示します。
  • logpageno は指定された level 内でのページ番号を示します。

アルゴリズムまたは振る舞い

本構造体は、fsm_logical_to_physical 関数等を通じて、論理的なアドレスから物理的なブロック番号への変換を行うための基点となります。木構造のナビゲーション（親ノードや子ノードの特定）において、位置情報を一意に識別するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: fsm_get_child, fsm_get_parent, fsm_get_location, fsm_get_heap_blk, fsm_logical_to_physical, fsm_readbuf, fsm_set_and_search, fsm_vacuum_page 等の内部関数で引数または戻り値として使用されています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • FSM_ROOT_ADDRESS: ルートノードを指す静的な FSMAddress 定数。

失敗モードと境界条件

  • 構造体自体に失敗モードはありませんが、無効な level や logpageno を保持した場合、fsm_logical_to_physical 等の変換関数が物理的に存在しないブロックを参照する可能性があります。

保守担当者向けメモ

  • 本構造体は FSM の論理構造を抽象化するものであり、物理的なストレージレイアウトとの対応関係は fsm_logical_to_physical 等の変換ロジックに依存しています。変更時は関連するナビゲーション関数への影響を確認してください。
