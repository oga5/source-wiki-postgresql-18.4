cb_cleanup_dir

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_combinebackup/pg_combinebackup.c:51

目的

処理失敗時に削除すべきディレクトリのパスと、その削除範囲を管理するためのリンクリスト構造体です。

シグネチャまたは形状

typedef struct cb_cleanup_dir
{
    char       *target_path;
    bool        rmtopdir;
    struct cb_cleanup_dir *next;
} cb_cleanup_dir;

入力と出力

  • 入力: target_path（削除対象のパス文字列）、rmtopdir（最上位ディレクトリ自体を削除するかどうかのフラグ）。
  • 出力: なし（構造体定義）。

前提条件と不変条件

  • next ポインタにより、複数の削除対象ディレクトリを連結して管理可能です。
  • cleanup_directories_atexit 等のクリーンアップ処理において、リストを走査して順次削除を行うために使用されます。

アルゴリズムまたは振る舞い

  • 本構造体は、プログラム実行中に作成された一時的なディレクトリや出力先ディレクトリを追跡するために使用されます。
  • pg_malloc を用いて動的に割り当てられ、cleanup_dir_list 静的変数に保持されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_combinebackup/pg_combinebackup.c 内のディレクトリ作成処理（pg_malloc によるインスタンス化）およびクリーンアップ処理（リストの走査）。
  • 呼び出し先: なし（データ構造のため）。

関連データ構造

  • cleanup_dir_list: 本構造体を要素とする静的なリンクリストのヘッドポインタ。

失敗モードと境界条件

  • メモリ不足時に pg_malloc が失敗する可能性があります。
  • target_path が指すパスが既に削除されている、あるいは権限不足で削除できない場合、クリーンアップ処理が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は pg_combinebackup の終了時やエラー発生時の安全なディレクトリ削除を保証するためのものです。リストの追加順序と削除順序が適切であるか（再帰的な削除を考慮しているか）を確認してください。
