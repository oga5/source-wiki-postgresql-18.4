WaitForOlderSnapshots

定義

  • 種別: function
  • 定義箇所: src/backend/commands/indexcmds.c:434

目的

指定されたトランザクションID（limitXmin）よりも古いスナップショットを保持しているアクティブなトランザクションの終了を待機します。主に並行インデックス構築において、整合性を保証するために使用されます。

シグネチャまたは形状

void WaitForOlderSnapshots(TransactionId limitXmin, bool progress)

入力と出力

  • **入力**:
  • limitXmin: 待機対象の基準となるトランザクションID。
  • progress: 進捗状況を報告するかどうかを示すフラグ。
  • **出力**: なし（副作用として待機および進捗報告を行う）。

前提条件と不変条件

  • 呼び出し元は、limitXmin が現在のシステム状態において適切な基準値であることを保証する必要があります。
  • 待機対象のプロセスは、PROC_IS_AUTOVACUUM、PROC_IN_VACUUM、または PROC_IN_SAFE_IC フラグを持つものに限定されます。

アルゴリズムまたは振る舞い

1. GetCurrentVirtualXIDs を使用して、limitXmin より古いスナップショットを持つ仮想トランザクションIDのリストを取得します。 2. progress が真の場合、待機対象の総数を進捗モニターに報告します。 3. 取得したリストを反復処理し、各トランザクションに対して以下を実行します：

  • 最初の反復以降は、再度 GetCurrentVirtualXIDs を呼び出し、待機対象リストの更新（終了したトランザクションの除外）を行います。
  • 依然として有効なトランザクションに対しては、VirtualXactLock を呼び出して終了を待機します。
  • progress が真の場合、現在待機中のPIDを報告し、完了したステップ数を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/indexcmds.c (複数箇所)
  • src/backend/commands/tablecmds.c:21447
  • **呼び出し先**:
  • GetCurrentVirtualXIDs
  • pgstat_progress_update_param
  • VirtualTransactionIdIsValid
  • VirtualTransactionIdEquals
  • SetInvalidVirtualTransactionId
  • pfree
  • ProcNumberGetProc
  • VirtualXactLock

関連データ構造

  • VirtualTransactionId: 仮想トランザクションID。
  • PGPROC: プロセス情報構造体。

失敗モードと境界条件

  • 待機対象のトランザクションが既に終了している場合、VirtualTransactionIdIsValid のチェックによりスキップされます。
  • progress が偽の場合、進捗報告に関連する処理はすべてバイパスされます。

保守担当者向けメモ

  • 本関数は並行処理の整合性維持に直結するため、待機対象フラグ（PROC_IN_SAFE_IC 等）の変更には注意が必要です。
  • 進捗報告のパラメータ（PROGRESS_WAITFOR_TOTAL 等）は、pgstat モジュールと同期している必要があります。
