plpgsql_exec_event_trigger

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_exec.c:1174

目的

イベントトリガーとして定義された PL/pgSQL 関数の実行を制御します。実行状態の初期化、エラーハンドリングのセットアップ、ステートメントの実行、および終了処理を行います。

シグネチャまたは形状

void plpgsql_exec_event_trigger(PLpgSQL_function *func, EventTriggerData *trigdata)

入力と出力

  • **入力**:
  • func: 実行対象のコンパイル済み PL/pgSQL 関数構造体。
  • trigdata: イベントトリガー実行時に渡されるデータ構造。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • plpgsql_estate_setup により実行状態（PLpgSQL_execstate）が初期化されている必要があります。
  • 実行終了時に plpgsql_destroy_econtext および exec_eval_cleanup を呼び出し、メモリリソースを解放する必要があります。

アルゴリズムまたは振る舞い

1. plpgsql_estate_setup で実行状態を初期化し、trigdata を設定します。 2. error_context_stack にエラーコールバックを登録し、スタックトレースを有効化します。 3. copy_plpgsql_datums を呼び出し、関数内の全データムのローカル実行用コピーを作成します。 4. インストルメンテーションプラグインが有効な場合、func_beg フックを呼び出します。 5. exec_toplevel_block を実行し、関数本体のステートメントを処理します。 6. 実行結果が PLPGSQL_RC_RETURN でない場合、トリガープロシージャが値を返さなかったものとしてエラーを報告します。 7. インストルメンテーションプラグインが有効な場合、func_end フックを呼び出します。 8. メモリコンテキストのクリーンアップを行い、エラーコンテキストスタックを復元します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/pl_handler.c 内のイベントトリガーハンドラから呼び出されます。
  • **呼び出し先**: plpgsql_estate_setup, copy_plpgsql_datums, exec_toplevel_block, plpgsql_destroy_econtext, exec_eval_cleanup 等の内部関数。

関連データ構造

  • PLpgSQL_function: 実行される関数の定義。
  • PLpgSQL_execstate: 実行中の関数の状態を保持する構造体。
  • EventTriggerData: イベントトリガー固有のデータ。

失敗モードと境界条件

  • 関数が RETURN 文に到達せずに終了した場合、ERRCODE_S_R_E_FUNCTION_EXECUTED_NO_RETURN_STATEMENT エラーが発生します。
  • 実行中に発生したエラーは plpgsql_exec_error_callback を通じて報告されます。

保守担当者向けメモ

  • 本関数はイベントトリガー専用の実行パスです。通常の関数実行（plpgsql_exec_function）やトリガー実行（plpgsql_exec_trigger）とは異なるエントリポイントである点に注意してください。
  • プラグインによるフック（func_beg, func_end）が挿入されているため、計測やデバッグを行う際はこれらのフックの挙動を確認してください。
