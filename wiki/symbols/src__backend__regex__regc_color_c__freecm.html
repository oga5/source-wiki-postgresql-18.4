freecm

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_color.c:102

目的

カラーマップ（colormap）構造体に関連する動的に割り当てられたメモリ領域を解放する。

シグネチャまたは形状

static void freecm(struct colormap *cm)

入力と出力

  • 入力: cm（解放対象の colormap 構造体へのポインタ）
  • 出力: なし

前提条件と不変条件

  • cm は有効な colormap 構造体を指している必要がある。
  • 解放処理後、cm->magic が 0 に設定される。

アルゴリズムまたは振る舞い

1. cm->magic を 0 に設定し、構造体の無効化を示す。 2. cm->cd が cm->cdspace（静的領域）と異なる場合、cm->cd を解放する。 3. cm->locolormap が NULL でない場合、これを解放する。 4. cm->cmranges が NULL でない場合、これを解放する。 5. cm->hicolormap が NULL でない場合、これを解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/regex/regcomp.c 内の freecm(&g->cmap) から呼び出される。
  • 呼び出し先: FREE マクロ（メモリ解放関数）。

関連データ構造

  • struct colormap: 正規表現エンジンにおける文字の彩色情報を保持する構造体。

失敗モードと境界条件

  • cm が NULL の場合、ポインタ参照によりクラッシュする可能性がある。
  • 各メンバが NULL の場合は解放処理をスキップするため、二重解放や不正なメモリ解放は発生しない設計となっている。

保守担当者向けメモ

  • 本関数は regcomp.c からも参照されるため、colormap 構造体の定義変更時には本関数の解放対象メンバと整合性を取る必要がある。
