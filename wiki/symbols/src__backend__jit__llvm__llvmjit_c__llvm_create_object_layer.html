llvm_create_object_layer

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit.c:1174

目的

LLVM ORC JIT 用のオブジェクトリンクレイヤー（LLVMOrcObjectLayerRef）を生成し、デバッグおよびプロファイリング用のイベントリスナーを登録する。

シグネチャまたは形状

static LLVMOrcObjectLayerRef llvm_create_object_layer(void *Ctx, LLVMOrcExecutionSessionRef ES, const char *Triple)

入力と出力

  • **入力**:
  • Ctx: コンテキストポインタ（未使用）。
  • ES: LLVMOrcExecutionSessionRef 型の実行セッション。
  • Triple: ターゲットトリプル文字列（未使用）。
  • **出力**:
  • 生成された LLVMOrcObjectLayerRef オブジェクト。

前提条件と不変条件

  • LLVM のバージョンやビルド設定に応じて、適切なメモリマネージャー（SectionMemoryManager）が選択される必要がある。
  • jit_debugging_support および jit_profiling_support のグローバル設定に基づき、リスナーの登録可否が決定される。

アルゴリズムまたは振る舞い

1. **レイヤー生成**: LLVM バージョンおよびビルドオプションに基づき、LLVMOrcCreateRTDyldObjectLinkingLayer... 系関数を呼び出し、メモリマネージャーを付与したオブジェクトレイヤーを生成する。 2. **デバッグリスナー登録**: jit_debugging_support が有効な場合、LLVMCreateGDBRegistrationListener を作成し、レイヤーに登録する。 3. **プロファイリングリスナー登録**: jit_profiling_support が有効な場合、LLVMCreatePerfJITEventListener を作成し、レイヤーに登録する。 4. **戻り値**: 設定済みのレイヤーオブジェクトを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の記述から、llvm_create_jit_instance 等の初期化プロセスから呼び出されることが示唆されるが、具体的な呼び出し元関数名はソース抜粋からは特定できない。
  • **呼び出し先**:
  • LLVMOrcCreateRTDyldObjectLinkingLayerWithSectionMemoryManager (またはそのバリエーション)
  • LLVMCreateGDBRegistrationListener
  • LLVMOrcRTDyldObjectLinkingLayerRegisterJITEventListener
  • LLVMCreatePerfJITEventListener

関連データ構造

  • LLVMOrcObjectLayerRef: 生成されるオブジェクトレイヤーのハンドル。
  • LLVMOrcExecutionSessionRef: JIT 実行セッション。
  • LLVMJITEventListenerRef: デバッグ/プロファイリング用リスナー。

失敗モードと境界条件

  • LLVM のバージョンが古い、または特定のバックポート設定が欠如している場合、コンパイル時に適切なメモリマネージャー関数が選択されない可能性がある。
  • LLVMCreatePerfJITEventListener が NULL を返す場合、プロファイリングリスナーの登録はスキップされる。

保守担当者向けメモ

  • 本関数は LLVM のバージョン依存の条件分岐（#if）を多く含んでいる。LLVM のメジャーバージョンアップ時には、メモリマネージャー生成関数の API 変更を注視すること。
  • Ctx および Triple 引数は現在の実装では使用されていない。将来的な拡張で利用される可能性がある。
