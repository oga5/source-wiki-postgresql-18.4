isValidTarHeader

定義

  • 種別: function
  • 定義箇所: src/port/tar.c:111

目的

提供されたバッファが有効な tar ヘッダーであるかを、チェックサムおよびマジックナンバーの照合を通じて検証します。

シグネチャまたは形状

bool isValidTarHeader(const char *header)

入力と出力

  • **入力**: header (tar ヘッダーブロックを指す const char * 型のポインタ)
  • **出力**: 検証に成功した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • header は少なくとも 512 バイトの有効なメモリ領域を指している必要があります。
  • チェックサムフィールドは、計算時には「8 つのスペース」として扱われる必要があります（tarChecksum の仕様に依存）。

アルゴリズムまたは振る舞い

1. tarChecksum(header) を呼び出し、ヘッダー全体のチェックサムを算出します。 2. read_tar_number を使用して、ヘッダー内のチェックサムフィールド（TAR_OFFSET_CHECKSUM）から格納されている値を読み取ります。 3. 算出したチェックサムと読み取った値が一致しない場合は false を返します。 4. 一致する場合、以下のいずれかのマジックナンバーとバージョン情報を照合し、合致すれば true を返します。

  • POSIX tar: magic が "ustar\0" かつ version が "00"
  • GNU tar: magic が "ustar \0"
  • 旧 pg_dump 形式: magic が "ustar00\0"

    • 5. 上記のいずれにも該当しない場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_archiver.c:2352
  • src/fe_utils/astreamer_tar.c:299
  • **呼び出し先**:
  • tarChecksum (同ファイル内)
  • read_tar_number (同ファイル内)

関連データ構造

  • pgtar.h: TAR_OFFSET_CHECKSUM, TAR_OFFSET_MAGIC, TAR_OFFSET_VERSION などのオフセット定数が定義されています。

失敗モードと境界条件

  • チェックサムの不一致: ヘッダーが破損しているか、tar 形式ではない場合に発生します。
  • マジックナンバーの不一致: サポートされていない tar 形式や、無効なデータブロックである場合に発生します。

保守担当者向けメモ

  • 本関数は pg_dump 等のバックアップ処理において、ストリーム内のヘッダーを識別する重要な役割を担っています。
  • マジックナンバーの判定ロジックは、過去の pg_dump バージョンとの互換性を維持するために複数の形式を許容しています。
