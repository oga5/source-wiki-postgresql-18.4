libpqrcv_create_slot

定義

  • 種別: function
  • 定義箇所: src/backend/replication/libpqwalreceiver/libpqwalreceiver.c:909

目的

プライマリサーバーに対して CREATE_REPLICATION_SLOT コマンドを発行し、物理または論理レプリケーションスロットを作成する。

シグネチャまたは形状

char *libpqrcv_create_slot(WalReceiverConn *conn, const char *slotname,
                           bool temporary, bool two_phase, bool failover,
                           CRSSnapshotAction snapshot_action, XLogRecPtr *lsn)

入力と出力

  • **入力**:
  • conn: 接続状態を保持する WalReceiverConn 構造体。
  • slotname: 作成するスロット名。
  • temporary: 一時スロットとして作成するかどうか。
  • two_phase: 論理スロットの場合、2フェーズコミットを有効にするか。
  • failover: 論理スロットの場合、フェイルオーバーを有効にするか。
  • snapshot_action: スナップショットの扱い（CRS_EXPORT_SNAPSHOT 等）。
  • lsn: スロット作成時のLSNを格納するポインタ（出力用）。
  • **出力**:
  • 成功時: スナップショット名（文字列、呼び出し元で解放が必要）または NULL。
  • 失敗時: ereport(ERROR) により処理を中断。

前提条件と不変条件

  • conn->streamConn が有効な接続状態であること。
  • サーバーバージョンが15以上の場合、新しいオプション構文（括弧付き）が使用される。

アルゴリズムまたは振る舞い

1. サーバーバージョンに基づき、CREATE_REPLICATION_SLOT コマンド文字列を構築する。 2. 論理スロットの場合は LOGICAL pgoutput を指定し、two_phase、failover、snapshot_action に応じたオプションを付与する。 3. 物理スロットの場合は PHYSICAL RESERVE_WAL を指定する。 4. libpqsrv_exec を介してコマンドを実行する。 5. 実行結果が PGRES_TUPLES_OK でない場合はエラーを報告する。 6. 成功時、結果セットからLSNを抽出し、スナップショット名があればコピーして返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/libpqwalreceiver/libpqwalreceiver.c 内の PQWalReceiverFunctions 構造体（.walrcv_create_slot フィールド）を通じて間接的に呼び出される。
  • **呼び出し先**: initStringInfo, appendStringInfo, libpqsrv_exec, PQresultStatus, PQclear, ereport, DatumGetLSN, PQgetvalue, pstrdup 等。

関連データ構造

  • WalReceiverConn: 接続情報およびストリーミング接続を保持。
  • PGresult: libpq からの実行結果。
  • StringInfoData: コマンド文字列構築用バッファ。

失敗モードと境界条件

  • コマンド実行が PGRES_TUPLES_OK 以外を返した場合、ERRCODE_PROTOCOL_VIOLATION でエラー終了する。
  • サーバーバージョンによる構文差異（15未満と15以上）を適切に切り替える。

保守担当者向けメモ

  • 本関数は libpq の COPY モードではなく、通常のクエリ実行インターフェース (libpqsrv_exec) を使用している。
  • 戻り値の文字列は pstrdup されているため、呼び出し元でメモリリークが発生しないよう注意すること。
