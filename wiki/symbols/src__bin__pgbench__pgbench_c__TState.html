TState

定義

  • **種別**: struct
  • **定義箇所**: src/bin/pgbench/pgbench.c:647

目的

pgbench における各ワーカースレッドの状態を保持・管理するための構造体です。スレッド固有の実行コンテキスト、統計情報、乱数生成器、およびクライアント状態の配列を保持します。

シグネチャまたは形状

typedef struct {
    int tid;
    THREAD_T thread;
    CState *state;
    int nstate;
    pg_prng_state ts_choose_rs;
    pg_prng_state ts_throttle_rs;
    pg_prng_state ts_sample_rs;
    int64 throttle_trigger;
    FILE *logfile;
    pg_time_usec_t create_time;
    pg_time_usec_t started_time;
    pg_time_usec_t bench_start;
    pg_time_usec_t conn_duration;
    StatsData stats;
    int64 latency_late;
} TState;

入力と出力

  • **入力**: スレッド初期化時に tid や nstate が設定され、state 配列が割り当てられます。
  • **出力**: 実行終了後、stats フィールドに集計された統計データが格納され、logfile にログが書き出されます。

前提条件と不変条件

  • state は nstate 個の CState 構造体からなる配列を指す必要があります。
  • 各スレッドは独立した pg_prng_state を保持し、スレッド間での乱数生成の決定論的独立性を保証します。

アルゴリズムまたは振る舞い

  • **スレッド管理**: threadRun 内で各スレッドのメインループを制御します。
  • **状態遷移**: advanceConnectionState を通じて、保持する CState 配列内の各クライアントのステートマシンを駆動します。
  • **統計収集**: トランザクションのレイテンシや実行時間を stats に蓄積し、必要に応じて processXactStats で更新します。
  • **スロットリング**: throttle_trigger を使用して、Poisson プロセスに基づくトランザクション開始タイミングの制御を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（スレッド生成時）、threadRun（スレッド内ループ）、printProgressReport（進捗表示時）。
  • **呼び出し先**: doLog、processXactStats、chooseScript、advanceConnectionState などの関数に引数として渡され、操作されます。

関連データ構造

  • CState: スレッドが管理する個々のクライアント状態。
  • StatsData: スレッドごとのトランザクション統計の集計先。
  • pg_prng_state: 乱数生成器の状態。

失敗モードと境界条件

  • logfile が NULL の場合、ログ出力は行われません。
  • nstate が 0 の場合、クライアントの処理は行われません。
  • メモリ割り当て失敗時、pg_malloc によりプログラムが終了する可能性があります。

保守担当者向けメモ

  • 本構造体は pgbench のマルチスレッド実行の基盤です。スレッド固有のデータを追加する場合は、競合を避けるためこの構造体内に配置してください。
  • ts_ で始まる乱数生成器は、スレッド間の独立性を維持するために重要です。変更時は注意してください。
