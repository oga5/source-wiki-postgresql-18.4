get_insert_query_def

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ruleutils.c:6938

目的

Query 構造体から INSERT 文の SQL 文字列を再構築（デパース）します。

シグネチャまたは形状

static void get_insert_query_def(Query *query, deparse_context *context)

入力と出力

  • **入力**:
  • query: デパース対象の INSERT クエリツリー。
  • context: デパース処理の状態を保持する deparse_context 構造体。
  • **出力**: なし（context->buf に SQL 文字列が追記されます）。

前提条件と不変条件

  • query->resultRelation が指す RTE が RTE_RELATION である必要があります。
  • INSERT ... SELECT と INSERT ... VALUES が同時に存在することは許容されません。
  • targetList の列名は、tle->resname ではなくシステムカタログ（get_attname）から取得し、スキーマ変更（RENAME）に追従する必要があります。

アルゴリズムまたは振る舞い

1. get_with_clause を呼び出し、WITH 句を出力します。 2. クエリ内の RTE を走査し、RTE_SUBQUERY または RTE_VALUES の存在を確認します。 3. INSERT INTO <relname> を出力し、必要に応じてエイリアスを付与します。 4. targetList を走査し、列名リスト (col1, col2, ...) を生成します。この際、resjunk なエントリは無視されます。 5. query->override が設定されている場合、OVERRIDING SYSTEM/USER VALUE を出力します。 6. クエリのタイプに応じて以下のいずれかを出力します：

  • select_rte がある場合: SELECT クエリを再帰的にデパース。
  • values_rte がある場合: VALUES リストをデパース。
  • strippedexprs がある場合: VALUES (...) をデパース。
  • いずれでもない場合: DEFAULT VALUES を出力。

    • 7. ON CONFLICT 句が存在する場合、競合解決条件（arbiter）および DO NOTHING / DO UPDATE SET をデパースします。 8. RETURNING 句が存在する場合、get_returning_clause を呼び出して出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/ruleutils.c 内のクエリデパース処理（get_query_def 等からディスパッチ）。
  • **呼び出し先**: get_with_clause, rt_fetch, generate_relation_name, get_rte_alias, get_attname, quote_identifier, processIndirection, get_query_def, get_values_def, get_rule_list_toplevel, appendContextKeyword, get_rule_expr, get_constraint_name, get_update_query_targetlist_def, get_returning_clause。

関連データ構造

  • Query: デパース対象のクエリツリー。
  • deparse_context: 出力バッファや名前空間を管理する状態オブジェクト。
  • RangeTblEntry: テーブルやサブクエリの情報を保持。
  • TargetEntry: INSERT 対象の列情報を保持。

失敗モードと境界条件

  • select_rte と values_rte が両方存在する場合、elog(ERROR) を発生させます。
  • select_rte または values_rte が複数存在する場合、elog(ERROR) を発生させます。
  • ON CONFLICT で指定された制約のキャッシュルックアップに失敗した場合、elog(ERROR) を発生させます。

保守担当者向けメモ

  • 本関数は INSERT 文の構文再構築に特化しています。ON CONFLICT の WHERE 句デパース時には、パーサの挙動に合わせて一時的に context->varprefix を無効化する処理が含まれているため、修正時は注意してください。
  • tle->resname は RENAME 後の列名と一致しない可能性があるため、必ず get_attname を使用してカタログから最新の列名を取得する仕様を維持してください。
