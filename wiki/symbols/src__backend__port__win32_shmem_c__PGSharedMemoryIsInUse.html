PGSharedMemoryIsInUse

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32_shmem.c:112

目的

Windows 環境において、特定の共有メモリセグメントが現在システム上で使用中であるかどうかを判定します。

シグネチャまたは形状

bool PGSharedMemoryIsInUse(unsigned long id1, unsigned long id2)

入力と出力

  • **入力**: id1, id2（現在この実装では使用されていません）
  • **出力**: 共有メモリセグメントが存在し、オープン可能であれば true、それ以外は false を返します。

前提条件と不変条件

  • GetSharedMemName() がデータディレクトリに基づいた有効な共有メモリ名を生成できる必要があります。
  • 呼び出し元は、戻り値が現在の共有メモリの状態を反映した一時的なスナップショットであることを考慮する必要があります。

アルゴリズムまたは振る舞い

1. GetSharedMemName() を呼び出し、共有メモリ名を生成します。 2. OpenFileMapping を使用して、指定された名前の共有メモリセグメントを読み取り専用モードで開こうと試みます。 3. 生成された名前文字列を free() で解放します。 4. OpenFileMapping が NULL を返した場合は false を返します。 5. セグメントが正常に開けた場合は、CloseHandle でハンドルをクローズし、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/init/miscinit.c:1404
  • src/backend/port/sysv_shmem.c（ツールによる参照の可能性あり）
  • **呼び出し先**:
  • GetSharedMemName
  • Windows API: OpenFileMapping, CloseHandle
  • 標準ライブラリ: free

関連データ構造

  • PGShmemHeader（storage/pg_shmem.h で定義）

失敗モードと境界条件

  • GetSharedMemName() がメモリ不足等で失敗した場合の挙動は、当該関数の実装に依存します。
  • OpenFileMapping が失敗した場合（セグメントが存在しない等）、関数は安全に false を返します。

保守担当者向けメモ

  • 引数 id1 および id2 は現在の Windows 実装では使用されていません。これは POSIX 実装とのインターフェース互換性のために維持されています。
  • この関数は共有メモリの存在確認のみを行い、その内容や整合性については関与しません。
