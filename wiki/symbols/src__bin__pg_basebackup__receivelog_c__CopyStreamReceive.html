CopyStreamReceive

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/receivelog.c:938

目的

PostgreSQLのストリーミングレプリケーションプロトコルにおいて、COPY モードでサーバーから送信されるデータメッセージを非ブロッキングで受信し、呼び出し元へバッファを返却します。

シグネチャまたは形状

int CopyStreamReceive(PGconn *conn, long timeout, pgsocket stop_socket, char **buffer);

入力と出力

  • **入力**:
  • conn: サーバーとの接続を保持する PGconn オブジェクト。
  • timeout: データ受信を待機する最大ミリ秒数。
  • stop_socket: 待機を中断するためのソケット。
  • buffer: 受信したデータメッセージを格納するための char* へのポインタ。
  • **出力**:
  • > 0: 受信したメッセージのバイト数（buffer にデータが格納される）。
  • 0: データが利用不可（タイムアウトまたは待機中）。
  • -1: エラー発生。
  • -2: ストリーミング終了。

前提条件と不変条件

  • *buffer は呼び出し時に NULL である必要があります（Assert により検証）。
  • 成功時、呼び出し元は受け取った buffer のメモリを解放する責任を負います。

アルゴリズムまたは振る舞い

1. PQgetCopyData を呼び出し、即座に利用可能なデータがあるか確認します。 2. データがない場合、CopyStreamPoll を使用して指定された timeout まで待機します。 3. CopyStreamPoll が成功した場合、PQconsumeInput で入力を消費し、再度 PQgetCopyData を試行します。 4. 取得したデータは buffer 引数を通じて呼び出し元に渡されます。 5. ストリーミング終了（-1）やエラー（-2）が PQgetCopyData から返された場合、適切な戻り値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: HandleCopyStream（src/bin/pg_basebackup/receivelog.c 内の810行目および859行目）。
  • **呼び出し先**:
  • PQgetCopyData (libpq)
  • CopyStreamPoll
  • PQconsumeInput (libpq)
  • pg_log_error

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • pgsocket: ソケット記述子。

失敗モードと境界条件

  • **タイムアウト**: CopyStreamPoll がデータを検知できない場合、0 を返します。
  • **通信エラー**: PQconsumeInput または PQgetCopyData が失敗した場合、pg_log_error でログを出力し -1 を返します。
  • **ストリーミング終了**: サーバーから終了通知を受けた場合、-2 を返します。

保守担当者向けメモ

  • 本関数は COPY プロトコルの低レベルな受信処理をラップしています。
  • buffer のメモリ管理は libpq の PQgetCopyData に依存しているため、呼び出し元での解放漏れに注意してください。
  • CopyStreamPoll を経由することで、長時間ブロックせずに外部からの停止要求（stop_socket）に応答可能な設計となっています。
