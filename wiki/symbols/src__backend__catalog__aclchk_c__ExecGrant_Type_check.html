ExecGrant_Type_check

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/aclchk.c:2403

目的

型オブジェクトに対する GRANT / REVOKE 操作の妥当性を検証し、特定の依存関係を持つ型への権限変更を禁止する。

シグネチャまたは形状

static void ExecGrant_Type_check(InternalGrant *istmt, HeapTuple tuple)

入力と出力

  • **入力**:
  • istmt: 実行される権限操作の詳細を含む InternalGrant 構造体。
  • tuple: 対象となる pg_type システムカタログのタプル。
  • **出力**: なし（正常時は復帰、異常時はエラーをスロー）。

前提条件と不変条件

  • tuple は pg_type システムカタログの有効なタプルであること。

アルゴリズムまたは振る舞い

1. tuple から Form_pg_type 構造体を取得する。 2. IsTrueArrayType を用いて、対象が配列型であるかを確認する。配列型の場合はエラーをスローし、要素型に対して権限を設定するよう促す。 3. typtype が TYPTYPE_MULTIRANGE であるかを確認する。マルチレンジ型の場合はエラーをスローし、対応するレンジ型に対して権限を設定するよう促す。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecGrant_common（src/backend/catalog/aclchk.c:615）から、型に対する権限変更の検証用コールバックとして呼び出される。
  • **呼び出し先**:
  • GETSTRUCT（マクロ）
  • IsTrueArrayType
  • ereport

関連データ構造

  • InternalGrant: 権限操作のコンテキストを保持する構造体。
  • Form_pg_type: pg_type カタログの行構造。

失敗モードと境界条件

  • **配列型**: IsTrueArrayType が真を返す場合、ERRCODE_INVALID_GRANT_OPERATION で失敗する。
  • **マルチレンジ型**: typtype が TYPTYPE_MULTIRANGE の場合、ERRCODE_INVALID_GRANT_OPERATION で失敗する。

保守担当者向けメモ

  • 本関数は ExecGrant_common を介して呼び出されるため、直接的な呼び出し元は aclchk.c 内の汎用的な権限付与ロジックに限定されている。
  • 新しい型の種類が追加され、同様の権限制限が必要になった場合は、本関数にチェックロジックを追加すること。
