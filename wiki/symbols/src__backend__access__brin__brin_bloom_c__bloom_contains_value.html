bloom_contains_value

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_bloom.c:405

目的

指定された値がブルームフィルタ内に存在するかを判定します。BRINインデックスにおいて、特定のページ範囲をスキャン対象とするか否かの判断に使用されます。

シグネチャまたは形状

static bool bloom_contains_value(BloomFilter *filter, uint32 value)

入力と出力

  • **入力**:
  • filter: 判定対象となる BloomFilter 構造体へのポインタ。
  • value: 判定対象の uint32 型の値。
  • **出力**:
  • bool: 値がフィルタ内に存在する可能性がある場合は true、確実に存在しない場合は false を返します。

前提条件と不変条件

  • filter は初期化済みである必要があります。
  • 「Less Hashing, Same Performance」論文に基づき、2つのハッシュ値（h1, h2）から filter->nhashes 個のハッシュ値を生成するアルゴリズムが前提となります。

アルゴリズムまたは振る舞い

1. hash_bytes_uint32_extended を使用し、BLOOM_SEED_1 および BLOOM_SEED_2 で value をハッシュ化し、h1 と h2 を算出します。 2. i を 0 から filter->nhashes - 1 までループさせます。 3. 各反復で h = (h1 + i * h2) % filter->nbits を計算し、対応するビット位置を特定します。 4. 該当するビットが filter->data 内で立っていない場合、即座に false を返します。 5. すべてのハッシュ値に対応するビットが立っている場合、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin_bloom.c 内の brin_bloom_consistent 関数から呼び出されます。
  • **呼び出し先**: hash_bytes_uint32_extended（common/hashfn.h）を呼び出します。

関連データ構造

  • BloomFilter: ブルームフィルタのビットマップデータ、ビット数（nbits）、ハッシュ関数数（nhashes）を保持する構造体。

失敗モードと境界条件

  • フィルタ内のすべてのビットがセットされている場合、値が存在しなくても true を返す誤判定（偽陽性）が発生します。
  • filter->nbits が 0 の場合、剰余演算で除算エラーが発生する可能性があります（構造体の初期化状態に依存）。

保守担当者向けメモ

  • 本関数は、BRINインデックスの検索効率を左右するクリティカルパスに位置します。
  • ハッシュ生成アルゴリズムを変更する場合は、brin_bloom_add_value との整合性を維持してください。
