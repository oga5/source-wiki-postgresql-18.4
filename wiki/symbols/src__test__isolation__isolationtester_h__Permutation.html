Permutation

定義

  • 種別: struct
  • 定義箇所: src/test/isolation/isolationtester.h:69

目的

PostgreSQLの分離レベルテストにおいて、テストステップの特定の実行順序（順列）を定義するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	int			nsteps;
	PermutationStep **steps;
} Permutation;

入力と出力

  • 入力: nsteps（ステップ数）、steps（PermutationStep 構造体へのポインタ配列）。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • steps は nsteps 個の要素を持つ配列を指す必要があります。
  • TestSpec 構造体の一部として管理され、テスト実行エンジンによって参照されます。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のみを行います。テスト実行エンジン（isolationtester.c）がこの構造体を走査し、保持されている PermutationStep のリストを順次実行することで、定義された順列を再現します。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/isolation/isolationtester.c 内のテスト実行ロジック（testspec->permutations を介してアクセス）。
  • 呼び出し先: なし（本構造体は PermutationStep を保持するコンテナです）。

関連データ構造

  • PermutationStep: 順列内で実行される個別のステップを定義します。
  • TestSpec: Permutation のリストを保持するルート構造体です。

失敗モードと境界条件

  • nsteps が 0 の場合、順列は空として扱われます。
  • steps ポインタが適切に初期化されていない場合、実行時にセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • Permutation は TestSpec の一部としてパーサー（specparse.y）により生成されます。
  • 実行時のリンク解決（PermutationStep 内のポインタ解決）は isolationtester.c の check_testspec() で行われるため、それ以前の段階で構造体にアクセスする際は注意が必要です。
