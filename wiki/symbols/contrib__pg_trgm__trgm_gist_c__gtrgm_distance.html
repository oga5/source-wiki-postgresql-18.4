gtrgm_distance

定義

  • 種別: function
  • 定義箇所: contrib/pg_trgm/trgm_gist.c:451

目的

GiST インデックスを用いた類似度検索（k-NN）において、インデックスキーとクエリ間の距離を計算します。

シグネチャまたは形状

Datum gtrgm_distance(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • entry: GISTENTRY 構造体（インデックスノード）。
  • query: 検索対象の text 型データ。
  • strategy: 演算子戦略番号。
  • recheck: 検索結果の再評価が必要かを示すフラグへのポインタ。
  • **出力**:
  • float8: 計算された距離（類似度に基づく）。

前提条件と不変条件

  • fcinfo->flinfo->fn_extra を使用して、同一クエリに対するトリグラム抽出結果をキャッシュします。
  • DistanceStrategyNumber 以外の戦略では recheck が true に設定されます。
  • リーフノードは元のトリグラムを保持し、非リーフノードはシグネチャ（ビットベクトル）を保持します。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: fn_extra を確認し、クエリが変更されていれば generate_trgm で再生成してキャッシュを更新します。 2. **戦略分岐**:

  • DistanceStrategyNumber 等の戦略に基づき処理を行います。
  • **リーフノード**: cnt_sml を用いて直接類似度を計算し、1.0 - sml を距離として返します。
  • **非リーフノード**: ISALLTRUE の場合は距離 0.0 を返します。それ以外の場合は cnt_sml_sign_common でシグネチャから共通トリグラム数を推定し、距離を算出します。

    • 3. **結果返却**: 計算された float8 値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックスエンジン（直接的な呼び出し元関数の詳細はソース内に明示されていません）。
  • **呼び出し先**: generate_trgm, cnt_sml, cnt_sml_sign_common, MemoryContextAlloc, pfree。

関連データ構造

  • GISTENTRY: GiST インデックスのノード情報。
  • TRGM: トリグラムデータ構造。
  • fcinfo->flinfo->fn_extra: クエリのトリグラムキャッシュを保持する領域。

失敗モードと境界条件

  • 未知の strategy が渡された場合、elog(ERROR) を発生させます。
  • len == 0 の場合、距離として -1.0 を返します。

保守担当者向けメモ

  • fn_extra を使用したキャッシュは、インデックスの再スキャン時にリークする可能性があるため注意が必要です。
  • リーフノードでの sml 計算時に volatile キーワードを使用しており、コンパイラの最適化による計算結果の不一致を防いでいます。
