RelnameGetRelid

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/namespace.c:884

目的

現在のセッションの検索パス（search_path）に基づき、指定されたリレーション名に対応する OID を解決します。

シグネチャまたは形状

Oid RelnameGetRelid(const char *relname)

入力と出力

  • **入力**: relname (解決対象のリレーション名)
  • **出力**: 見つかった場合はそのリレーションの Oid、見つからない場合は InvalidOid

前提条件と不変条件

  • 検索パスが最新であることを保証するため、内部で recomputeNamespacePath() を呼び出します。
  • activeSearchPath が現在のセッションの検索順序を保持している必要があります。

アルゴリズムまたは振る舞い

1. recomputeNamespacePath() を呼び出し、検索パスを最新状態に更新します。 2. activeSearchPath リストを先頭から順に走査します。 3. 各名前空間（namespaceId）に対して get_relname_relid(relname, namespaceId) を実行し、リレーションを検索します。 4. 有効な OID が見つかった時点でその値を返し、リストの最後まで見つからなかった場合は InvalidOid を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/namespace.c (複数箇所)
  • src/pl/plpgsql/src/pl_comp.c:1679
  • **呼び出し先**:
  • recomputeNamespacePath()
  • get_relname_relid()

関連データ構造

  • activeSearchPath: 検索対象となる名前空間 OID のリスト。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • 指定された名前のリレーションが検索パス内のどの名前空間にも存在しない場合、InvalidOid を返します。
  • 検索パスが空の場合、検索は行われず InvalidOid を返します。

保守担当者向けメモ

  • 本関数は非修飾名のリレーション解決に使用されます。修飾名（スキーマ指定あり）の解決には適していません。
  • get_relname_relid の結果に依存しているため、カタログキャッシュの整合性が保たれていることが前提となります。
