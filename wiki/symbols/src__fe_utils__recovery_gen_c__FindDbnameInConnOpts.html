FindDbnameInConnOpts

定義

  • 種別: function
  • 定義箇所: src/fe_utils/recovery_gen.c:180

目的

接続オプションの配列から、有効な値を持つ dbname パラメータを検索し、その値を複製して取得する。

シグネチャまたは形状

static char *FindDbnameInConnOpts(PQconninfoOption *conn_opts)

入力と出力

  • **入力**: PQconninfoOption *conn_opts（libpqの接続オプション配列へのポインタ）
  • **出力**: char *（見つかった場合は dbname の値のコピーを返す。見つからない場合や値が空の場合は NULL を返す）

前提条件と不変条件

  • conn_opts は keyword が NULL で終わる配列である必要がある。
  • 戻り値は pg_strdup によってメモリが確保されているため、呼び出し元で適切に解放する必要がある。

アルゴリズムまたは振る舞い

1. conn_opts 配列を順次走査する。 2. keyword が "dbname" と一致する要素を探す。 3. 一致した要素の val が NULL ではなく、かつ空文字列（'\0'）でないことを確認する。 4. 条件を満たす場合、その値を pg_strdup で複製して返す。 5. 配列の終端に達しても見つからない場合は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/fe_utils/recovery_gen.c 内の関数（217行目および232行目付近で呼び出されている）。
  • **呼び出し先**:
  • strcmp (標準ライブラリ)
  • pg_strdup (PostgreSQLフロントエンド用メモリ複製関数)

関連データ構造

  • PQconninfoOption: libpq で定義される接続オプションを保持する構造体。

失敗モードと境界条件

  • conn_opts が空、または dbname オプションが含まれていない場合は NULL を返す。
  • dbname オプションが存在しても値が空文字列の場合は NULL を返す。
  • メモリ不足時には pg_strdup 内部でエラーが発生し、プロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は static であり、src/fe_utils/recovery_gen.c 内でのみ使用されるヘルパー関数である。
  • dbname の検索ロジックは単純な文字列比較に基づいているため、接続オプションの仕様変更時には注意が必要である。
