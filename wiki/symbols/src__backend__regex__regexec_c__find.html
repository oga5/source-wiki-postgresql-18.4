find

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexec.c:420

目的

正規表現の検索を実行し、マッチング範囲の特定およびサブマッチの解析を行う。

シグネチャまたは形状

static int find(struct vars *v, struct cnfa *cnfa, struct colormap *cm)

入力と出力

  • **入力**:
  • v: マッチング状態を保持する vars 構造体。
  • cnfa: 検索対象の NFA。
  • cm: 文字クラスのカラーマッピング。
  • **出力**:
  • int: 成功時は REG_OKAY、失敗時は REG_NOMATCH またはエラーコードを返す。

前提条件と不変条件

  • v->g->tree が有効であること。
  • v->nmatch > 0 の場合、v->pmatch 配列に結果が書き込まれる。
  • REG_EXPECT フラグが設定されている場合、v->details が初期化されていること。

アルゴリズムまたは振る舞い

1. **検索**: newdfa で検索用 DFA を生成し、shortest を用いてマッチングの可能性のある範囲（open から close）を特定する。 2. **マッチング**: 特定された範囲内で、newdfa で生成した本番用 DFA を使用し、shortest（SHORTER フラグ時）または longest を用いてマッチの開始位置と終了位置を決定する。 3. **詳細化**: v->pmatch[0] にマッチ範囲を格納する。 4. **サブマッチ**: v->nmatch > 1 の場合、cdissect を呼び出して再帰的にサブマッチを特定する。 5. **クリーンアップ**: 使用した DFA 領域を freedfa で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は提供されていません。
  • **呼び出し先**: newdfa, shortest, longest, freedfa, cdissect, MDEBUG, LOFF, OFF, ISERR, NOERR。

関連データ構造

  • vars: マッチングのコンテキスト。
  • cnfa: コンパイル済み NFA。
  • dfa: 状態管理構造体。
  • pmatch: マッチ結果の開始・終了位置を保持する配列。

失敗モードと境界条件

  • newdfa が NULL を返した場合、v->err を返す。
  • マッチが見つからない場合、REG_NOMATCH を返す。
  • cdissect 実行中にエラーが発生した場合、そのエラーコードを伝播する。

保守担当者向けメモ

  • 本関数はバックリファレンスを含まない、あるいは単純なケースのマッチングを想定しています。
  • REG_EXPECT フラグが有効な場合、rm_extend の更新処理が含まれます。
  • shorter 変数は v->g->tree->flags に基づいて動的に決定されます。
