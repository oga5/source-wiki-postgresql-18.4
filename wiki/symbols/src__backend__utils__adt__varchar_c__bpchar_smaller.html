bpchar_smaller

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varchar.c:964

目的

2つの char(n) (BpChar) 型の値を比較し、小さい方の値を返します。

シグネチャまたは形状

Datum bpchar_smaller(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの BpChar 型のデータを受け取ります。
  • **出力**: Datum 型として、比較結果が小さい方の BpChar 値を返します。

前提条件と不変条件

  • 入力値は BpChar 型として正当である必要があります。
  • 比較には現在のセッションの照合順序（Collation）が使用されます。

アルゴリズムまたは振る舞い

1. bcTruelen を使用して、各入力値から末尾の空白を除いた「真の長さ」を計算します。 2. varstr_cmp を呼び出し、計算した長さと現在の照合順序に基づいて2つの値を比較します。 3. 比較結果が 0 以下（arg1 <= arg2）であれば arg1 を、それ以外であれば arg2 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 外部からの直接的な呼び出し元情報は提供されていません。
  • **呼び出し先**:
  • bcTruelen: 真の長さの計算に使用。
  • varstr_cmp: ロケールを考慮した文字列比較に使用。
  • PG_GET_COLLATION: 現在の照合順序の取得に使用。
  • PG_GETARG_BPCHAR_PP / PG_RETURN_BPCHAR_P: 引数の取得および戻り値の返却に使用。

関連データ構造

  • BpChar: char(n) 型の内部表現。
  • varlena: BpChar が内部的に利用する可変長データ構造。

失敗モードと境界条件

  • 入力値が NULL の場合、PG_GETARG_BPCHAR_PP のマクロ定義に従い、呼び出し元で適切に処理される必要があります。
  • 照合順序の設定によっては、比較結果が予期しない順序になる可能性があります。

保守担当者向けメモ

  • 本関数は bpchar 型の比較演算子の一部として機能します。
  • 末尾の空白を無視する仕様は SQL 標準に準拠しており、bcTruelen による事前計算が比較の正確性を担保しています。
