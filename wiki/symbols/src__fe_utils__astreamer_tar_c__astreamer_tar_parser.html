astreamer_tar_parser

定義

  • 種別: struct
  • 定義箇所: src/fe_utils/astreamer_tar.c:30

目的

tarアーカイブのバイトストリームを解析し、ヘッダー、コンテンツ、トレーラーの各セグメントへ分解するための状態保持構造体です。

シグネチャまたは形状

typedef struct astreamer_tar_parser
{
    astreamer           base;
    astreamer_archive_context next_context;
    astreamer_member    member;
    size_t              file_bytes_sent;
    size_t              pad_bytes_expected;
} astreamer_tar_parser;

入力と出力

  • **入力**: astreamer インターフェースを介して受け取る未加工のバイトストリーム。
  • **出力**: 解析されたtar構造（ヘッダー情報、ファイルコンテンツ、パディング情報）を後続のストリーマーへ転送。

前提条件と不変条件

  • astreamer 基底構造体を先頭に保持し、キャストによる相互変換を前提としています。
  • astreamer_tar_header 関数による検証を経て、有効なtarヘッダーであることが期待されます。

アルゴリズムまたは振る舞い

  • astreamer_tar_parser_content を通じてデータを受け取り、next_context（ヘッダー、コンテンツ、トレーラー）の状態遷移に基づいてストリームを切り出します。
  • astreamer_buffer_until を利用し、ブロック単位での処理を保証します。
  • 不正なデータやサポート外の pax 拡張フォーマットが検出された場合、pg_fatal により処理を終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer_tar_parser_new（インスタンス生成時）、および astreamer_tar_parser_content 等のストリーマー操作関数。
  • **呼び出し先**: astreamer_tar_header（ヘッダー解析用内部関数）。

関連データ構造

  • astreamer: 基底インターフェース。
  • astreamer_archive_context: 解析状態（ヘッダー、コンテンツ、トレーラー）を管理する列挙型。
  • astreamer_member: メンバー情報を保持する構造体。

失敗モードと境界条件

  • **パディング**: tarファイルの末尾に不足しがちなNULバイトブロックの欠落を補完する設計となっています。
  • **検証エラー**: isValidTarHeader による検証失敗時、および pax フォーマット検出時には致命的エラーとして処理を停止します。

保守担当者向けメモ

  • 本構造体は astreamer パイプラインの一部として動作します。
  • メンバー変数 file_bytes_sent および pad_bytes_expected は、現在のファイルコンテンツの進捗と、tar仕様上のパディング量を追跡するために使用されます。
