bbsink_gzip_begin_backup

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_gzip.c:93

目的

gzip圧縮シンクの初期化処理を行い、圧縮データ送出のためのバッファ確保および後続シンクへの開始通知を行う。

シグネチャまたは形状

static void bbsink_gzip_begin_backup(bbsink *sink)

入力と出力

  • **入力**: sink (bbsink *) - 圧縮処理を行うシンク構造体へのポインタ。
  • **出力**: なし（副作用として sink->bbs_buffer の割り当てと後続シンクの初期化を実行）。

前提条件と不変条件

  • sink は有効な bbsink 構造体を指していること。
  • sink->bbs_buffer_length が適切に設定されていること。
  • HAVE_LIBZ が定義されている環境であること。

アルゴリズムまたは振る舞い

1. sink->bbs_buffer_length のサイズでメモリを確保し、sink->bbs_buffer に割り当てる。これは、入力データと圧縮後のデータが異なるため、独自のバッファが必要となるためである。 2. bbsink_begin_backup を呼び出し、後続のシンク（sink->bbs_next）に対してバックアップ開始を通知する。その際、現在のシンクのバッファ長を引数として渡す。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_gzip 構造体のメソッドテーブル（.begin_backup）を通じて呼び出される。
  • **呼び出し先**: palloc（メモリ確保）、bbsink_begin_backup（後続シンクの初期化）。

関連データ構造

  • bbsink: バックアップ処理のパイプラインを構成する基本構造体。
  • bbsink_gzip: 本関数が操作対象とする圧縮シンクのコンテキスト。

失敗モードと境界条件

  • メモリ不足時: palloc がエラーを発生させる可能性がある。
  • sink->bbs_next が未設定の場合: bbsink_begin_backup 呼び出し時にクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数は bbsink インターフェースの一部として実装されている。
  • 確保したバッファは、後続のシンクへ渡すデータの一時保管場所として利用される。
  • 圧縮後のデータバッファサイズは、入力バッファサイズと同一に設定されている。
