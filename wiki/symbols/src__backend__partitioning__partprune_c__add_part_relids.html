add_part_relids

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partprune.c:399

目的

パーティションのリレーションID集合（Bitmapset）を、既存の階層リスト（List）に統合または追加する。

シグネチャまたは形状

List *add_part_relids(List *allpartrelids, Bitmapset *partrelids)

入力と出力

  • **入力**:
  • allpartrelids: 既存のパーティション階層リスト。各要素は Bitmapset。
  • partrelids: 追加対象のパーティションリレーションID集合。
  • **出力**:
  • 更新済み、または新規追加された allpartrelids リスト。

前提条件と不変条件

  • partrelids は空であってはならない（bms_next_member が 0 より大きい値を返すことを前提としているため）。

アルゴリズムまたは振る舞い

1. partrelids 内の最初のメンバー（最上位の親パーティションのインデックス）を取得する。 2. allpartrelids リストを走査し、同じ最上位親を持つ既存の Bitmapset を探す。 3. 一致するものが見つかった場合、bms_add_members を用いて partrelids の内容を既存の集合にマージし、リストを更新して終了する。 4. 一致するものが見つからなかった場合、partrelids を新しい要素としてリストの末尾に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/partitioning/partprune.c 内の関数（行295など）。
  • **呼び出し先**: bms_next_member, bms_add_members, lfirst, lappend 等の Bitmapset および List 操作関数。

関連データ構造

  • Bitmapset: パーティションのリレーションIDを保持するために使用。
  • List: パーティション階層のリストを管理するために使用。

失敗モードと境界条件

  • partrelids が空の場合、Assert(targetpart > 0) によりアサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は、複数のパーティション階層が混在する状況下で、リレーションIDの重複を排除しつつ階層構造を整理するために使用される。
  • Bitmapset の操作を伴うため、メモリ管理は呼び出し側のコンテキストに依存する。
