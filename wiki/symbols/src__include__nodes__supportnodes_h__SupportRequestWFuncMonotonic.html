SupportRequestWFuncMonotonic

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/supportnodes.h:290

目的

ウィンドウ関数が特定のウィンドウ句（WindowClause）のコンテキストにおいて単調性を持つかどうかを判定するためのリクエスト構造体です。

シグネチャまたは形状

typedef struct SupportRequestWFuncMonotonic
{
    NodeTag        type;
    WindowFunc    *window_func;
    WindowClause  *window_clause;
    MonotonicFunction monotonic;
} SupportRequestWFuncMonotonic;

入力と出力

  • **入力**:
  • window_func: 評価対象のウィンドウ関数データへのポインタ。
  • window_clause: 評価対象のウィンドウ句データへのポインタ。
  • **出力**:
  • monotonic: 判定結果を格納する MonotonicFunction 型のフィールド。

前提条件と不変条件

  • 本構造体はプランナーサポート関数（Planner Support Functions）のプロトコルの一部として使用されます。
  • サポート関数は、リクエストを処理できない場合は NULL を返します。

アルゴリズムまたは振る舞い

1. プランナーまたは実行エンジンがウィンドウ関数の単調性を確認する必要がある際に、本構造体を生成します。 2. 登録されたサポート関数がこの構造体を受け取り、window_func と window_clause を解析します。 3. 判定結果を monotonic フィールドに設定して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/allpaths.c
  • src/backend/utils/adt/int8.c
  • src/backend/utils/adt/windowfuncs.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、特定の関数を直接呼び出すものではありません。サポート関数側でロジックが実行されます。

関連データ構造

  • WindowFunc: src/include/nodes/primnodes.h で定義されるウィンドウ関数ノード。
  • WindowClause: src/include/nodes/parsenodes.h で定義されるウィンドウ句ノード。
  • MonotonicFunction: 単調性の状態を保持する列挙型（定義詳細は本ソース外）。

失敗モードと境界条件

  • サポート関数が単調性を判定できない場合、構造体へのポインタではなく NULL を返却することで、最適化をスキップするプロトコルとなっています。

保守担当者向けメモ

  • 本構造体は一時的なリクエスト用であり、サポート関数内で永続的なメモリ確保は不要です。
  • 「プランナーサポート関数」という名称ですが、実行時にも呼び出される可能性があるため、プランナー専用のロジックと限定しないよう注意してください。
