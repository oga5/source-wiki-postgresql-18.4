simple_oid_list_member

定義

  • 種別: function
  • 定義箇所: src/fe_utils/simple_list.c:44

目的

指定された SimpleOidList 内に特定の Oid 値が存在するかを判定します。

シグネチャまたは形状

bool simple_oid_list_member(SimpleOidList *list, Oid val)

入力と出力

  • **入力**:
  • list: 検索対象の SimpleOidList 構造体へのポインタ。
  • val: 検索する Oid 値。
  • **出力**:
  • リスト内に val と一致する要素が存在する場合は true、存在しない場合は false を返します。

前提条件と不変条件

  • list は初期化済みである必要があります。
  • リストは単方向連結リストであり、本関数はリストの先頭から順次走査を行います。

アルゴリズムまたは振る舞い

1. リストの先頭 (list->head) から順に各セルを走査します。 2. 各セルの val フィールドと引数の val を比較します。 3. 一致する値が見つかった時点で直ちに true を返します。 4. リストの末尾まで走査しても一致するものが見つからない場合、false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c 内の複数の箇所（スキーマ、テーブル、拡張機能、外部サーバー等の包含・除外リストの判定に使用）。
  • **呼び出し先**: 本関数はリストの走査のみを行い、他の関数を呼び出しません。

関連データ構造

  • SimpleOidList: リストのヘッドを保持する構造体。
  • SimpleOidListCell: リストの各要素（値と次のセルへのポインタ）を保持する構造体。

失敗モードと境界条件

  • リストが空（list->head が NULL）の場合、ループは実行されず、正しく false を返します。
  • 要素数 $N$ に対して $O(N)$ の計算量を要するため、大規模なリストの検索には適していません。

保守担当者向けメモ

  • 本関数は単純な線形探索です。パフォーマンス上の懸念がある場合は、リストのデータ構造自体を見直す必要があります。
  • SimpleOidList のメモリ管理（破棄など）については、関連する simple_oid_list_destroy を参照してください。
