print_key

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-info.c:89

目的

8バイトのバイナリ形式PGP Key IDを、可読な16進数文字列に変換します。

シグネチャまたは形状

static int print_key(uint8 *keyid, char *dst)

入力と出力

  • **入力**:
  • keyid: 8バイトのバイナリデータ（Key ID）へのポインタ。
  • dst: 変換後の文字列を格納するバッファへのポインタ。
  • **出力**:
  • 戻り値: 書き込まれた文字数（16）。
  • dst: 16進数表現された文字列（ヌル終端付き）が書き込まれる。

前提条件と不変条件

  • dst 引数は、16文字の16進数文字列と終端文字を格納するために、少なくとも17バイト以上のメモリ領域を確保している必要があります。
  • hextbl がスコープ内で定義されている必要があります。

アルゴリズムまたは振る舞い

1. 8バイトの keyid 配列をループで走査します。 2. 各バイトを上位4ビットと下位4ビットに分割します。 3. hextbl を参照して、各4ビットを対応する16進数文字（'0'-'9', 'a'-'f'等）に変換します。 4. 変換した2文字を dst バッファに順次書き込みます。 5. 最後に dst の末尾にヌル文字（0）を付与します。 6. 書き込んだ文字数である16を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-info.c 内の pgp_get_keyid から呼び出されます。
  • **呼び出し先**: 外部関数は呼び出しません。

関連データ構造

  • uint8: PGP Key IDのバイナリ表現に使用される型。
  • hextbl: 16進数変換用のルックアップテーブル。

失敗モードと境界条件

  • dst バッファが17バイト未満の場合、バッファオーバーフローが発生します。
  • 入力 keyid が8バイト未満のメモリ領域を指している場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本関数は pgp-info.c 内の内部ヘルパーであり、Key IDのフォーマット変換のみを担います。
  • 変換後の文字列長は常に固定（16文字）であるため、戻り値は常に16となります。
