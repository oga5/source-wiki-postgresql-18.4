PosixSemaphoreCreate

定義

  • 種別: function
  • 定義箇所: src/backend/port/posix_sema.c:85

目的

名前付きPOSIXセマフォを生成し、初期値1で初期化してプロセス間同期用のセマフォハンドルを取得する。

シグネチャまたは形状

static sem_t *PosixSemaphoreCreate(void)

入力と出力

  • **入力**: なし
  • **出力**: 成功時は初期化された sem_t * を返し、失敗時は elog(FATAL, ...) によりプロセスを終了する。

前提条件と不変条件

  • nextSemKey がグローバルなカウンタとしてセマフォ名の生成に使用される。
  • IPCProtection がセマフォのアクセス権限として使用される。
  • 作成されたセマフォは、作成直後に sem_unlink される。

アルゴリズムまたは振る舞い

1. nextSemKey をインクリメントし、/pgsql-%d という形式で一意なセマフォ名を生成する。 2. sem_open を呼び出し、O_CREAT | O_EXCL フラグを用いてセマフォを作成する。 3. EEXIST（名前の衝突）、EACCES、EINTR が発生した場合はループして再試行する。その他のエラーは FATAL で終了する。 4. sem_unlink を呼び出し、名前空間からセマフォを削除する（プロセス終了時の自動クリーンアップを保証するため）。 5. 取得した sem_t * を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/port/posix_sema.c 内のセマフォ初期化処理（PGSemaphoreCreate 等）から呼び出される。
  • **呼び出し先**: sem_open（POSIX API）、sem_unlink（POSIX API）、snprintf、elog。

関連データ構造

  • sem_t: POSIXセマフォの型。

失敗モードと境界条件

  • **名前の衝突**: EEXIST が発生した場合、ループにより別のキーで再試行する。
  • **権限エラー**: EACCES が発生した場合、ループにより再試行する。
  • **システムエラー**: sem_open が上記以外のエラーを返した場合、elog(FATAL) によりプロセスが強制終了される。

保守担当者向けメモ

  • 本関数は名前付きセマフォを使用する環境でのみ有効である。
  • sem_unlink を直後に呼ぶことで、プロセスがクラッシュしてもセマフォがシステム上に残留しないように設計されている。
  • EXEC_BACKEND 環境下では、名前付きセマフォの共有メモリ外への割り当て制約により使用できない場合がある。
