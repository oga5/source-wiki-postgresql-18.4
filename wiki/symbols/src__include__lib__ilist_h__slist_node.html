slist_node

定義

  • 種別: struct
  • 定義箇所: src/include/lib/ilist.h:223

目的

PostgreSQL の統合型単一連結リスト（slist）において、リストの各要素を繋ぐためのリンクポインタを保持する構造体です。ユーザー定義のデータ構造に埋め込んで使用します。

シグネチャまたは形状

struct slist_node {
    struct slist_node *next;
};

入力と出力

  • **入力**: リスト操作関数（slist_delete 等）に対して、ノードのポインタが渡されます。
  • **出力**: リスト内の次のノードへのポインタ（next）を保持します。

前提条件と不変条件

  • リストに含まれている間、この構造体を含む親構造体を free() してはなりません。
  • next ポインタは、リストの終端では NULL またはリストの終端を示す適切な値に設定されている必要があります。

アルゴリズムまたは振る舞い

  • リストのリンクをデータ構造内部に直接保持する「統合型」設計を採用しており、ノードごとの動的なメモリ割り当てを回避します。
  • slist_head と組み合わせて使用され、一方向の走査をサポートします。

呼び出し元と呼び出し先

  • **呼び出し元（参照例）**:
  • src/backend/commands/event_trigger.c
  • src/backend/storage/ipc/dsm.c
  • src/include/executor/spi.h
  • **呼び出し先**:
  • src/backend/lib/ilist.c 内のリスト操作関数（slist_delete 等）が、この構造体の next ポインタを操作します。

関連データ構造

  • slist_head: リストの起点となる構造体。
  • slist_iter: リスト走査用のイテレータ。
  • slist_mutable_iter: 走査中に安全にノードを削除するためのイテレータ。

失敗モードと境界条件

  • リストが空の場合、slist_head の head は終端状態を指します。
  • 循環参照や不正なポインタが next に設定された場合、走査時に無限ループやセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • ILIST_DEBUG を定義することで、リストの整合性チェックを有効化できます。
  • slist_container マクロを使用することで、slist_node のポインタから、それが埋め込まれている親構造体のポインタを復元可能です。
