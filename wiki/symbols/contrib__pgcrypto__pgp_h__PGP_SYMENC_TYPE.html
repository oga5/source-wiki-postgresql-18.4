PGP_SYMENC_TYPE

定義

  • 種別: enum
  • 定義箇所: contrib/pgcrypto/pgp.h:75

目的

OpenPGP (RFC 4880) 規格で定義される対称暗号アルゴリズムを識別するための列挙型です。

シグネチャまたは形状

enum PGP_SYMENC_TYPE
{
    PGP_SYM_PLAIN = 0,
    PGP_SYM_IDEA = 1,
    PGP_SYM_DES3 = 2,
    PGP_SYM_CAST5 = 3,
    PGP_SYM_BLOWFISH = 4,
    PGP_SYM_SAFER_SK128 = 5,
    PGP_SYM_DES_SK = 6,
    PGP_SYM_AES_128 = 7,
    PGP_SYM_AES_192 = 8,
    PGP_SYM_AES_256 = 9,
    PGP_SYM_TWOFISH = 10
};

入力と出力

  • 入力: なし（定数定義）
  • 出力: なし（定数定義）

前提条件と不変条件

  • 各値は RFC 4880 に準拠した対称暗号アルゴリズムの識別子に対応しています。
  • PGP_SYM_IDEA、PGP_SYM_SAFER_SK128、PGP_SYM_DES_SK は廃止（obsolete）としてマークされています。

アルゴリズムまたは振る舞い

本シンボルは、PGP 暗号化セッションにおいて使用する対称暗号アルゴリズムを選択するための識別子として機能します。具体的な暗号化処理の実装（pgp-encrypt.c 等）において、この列挙値に基づいて適切な暗号化バックエンドが選択されます。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース抜粋には、本列挙型を参照している具体的な関数呼び出し元は記載されていません。
  • 呼び出し先: 提供されたソース抜粋には、本列挙型が呼び出している関数は記載されていません。

関連データ構造

  • PGP_Context: 暗号化/復号セッションの状態を保持する構造体であり、この中で使用する対称暗号アルゴリズムを本列挙型を用いて指定します。

失敗モードと境界条件

  • 定義外の整数値をキャストして使用した場合の挙動は未定義です。
  • 廃止されたアルゴリズム（PGP_SYM_IDEA 等）を選択した場合、バックエンドの実装状況によってはエラーが発生する可能性があります。

保守担当者向けメモ

  • 本列挙型は RFC 4880 に強く依存しています。新しいアルゴリズムの追加や仕様変更の際は、RFC の最新の暗号アルゴリズム登録状況を確認してください。
  • PGP_SYM_PLAIN (0) は暗号化を行わないモードを示唆していますが、利用にあたっては pgcrypto モジュール全体での取り扱いを確認してください。
