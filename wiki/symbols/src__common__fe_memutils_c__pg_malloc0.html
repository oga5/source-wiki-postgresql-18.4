pg_malloc0

定義

  • 種別: function
  • 定義箇所: src/common/fe_memutils.c:58

目的

指定されたサイズのメモリを確保し、その領域をゼロクリアして返却する。メモリ確保失敗時にはプロセスを終了させることで、呼び出し側のエラーチェックを簡略化する。

シグネチャまたは形状

void *pg_malloc0(size_t size);

入力と出力

  • **入力**: size_t size（確保するメモリのバイト数）
  • **出力**: 確保されたメモリ領域へのポインタ。確保失敗時はプロセスが終了するため、戻り値は常に有効なメモリを指す。

前提条件と不変条件

  • SIZE_MAX / 2 を超える確保要求はエラーとして扱われる。
  • 確保に失敗した場合、標準エラー出力にメッセージを出力し exit(EXIT_FAILURE) を呼び出す。

アルゴリズムまたは振る舞い

1. 内部関数 pg_malloc_internal を呼び出す。 2. その際、MCXT_ALLOC_ZERO フラグを渡し、確保後のメモリ領域をゼロクリアするよう指示する。 3. 確保されたメモリ領域を呼び出し元に返却する。

呼び出し元と呼び出し先

  • **呼び出し先**: pg_malloc_internal
  • **呼び出し元**:
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_basebackup/pg_receivewal.c
  • src/bin/pg_basebackup/streamutil.c
  • src/bin/pg_basebackup/walmethods.c

関連データ構造

  • MCXT_ALLOC_ZERO: メモリ確保時にゼロクリアを指示するためのフラグ。

失敗モードと境界条件

  • size が 0 の場合、移植性の観点から malloc(1) 相当の挙動となる。
  • メモリ不足（OOM）発生時は、プロセス全体が終了する。

保守担当者向けメモ

  • 本関数は FRONTEND マクロが定義されている環境でのみ利用可能。
  • バックエンドの palloc0 と同等のセマンティクスを提供することを目的としている。
