timestamptz_to_str

定義

  • 種別: function
  • 定義箇所: src/bin/pg_waldump/compat.c:48

目的

PostgreSQLの内部タイムスタンプ値（TimestampTz）を、人間が判読可能な文字列（YYYY-MM-DD HH:MM:SS.uuuuuu Z）に変換する。

シグネチャまたは形状

const char *timestamptz_to_str(TimestampTz t);

入力と出力

  • **入力**: TimestampTz t（エポックからのマイクロ秒数）
  • **出力**: const char *（静的バッファへのポインタ。日付、時刻、マイクロ秒、タイムゾーンを含む文字列）

前提条件と不変条件

  • 内部で静的バッファを使用しているため、同一スコープ内で複数回呼び出すと以前の結果が上書きされる。
  • TimestampTz 型は utils/datetime.h で定義されている必要がある。

アルゴリズムまたは振る舞い

1. timestamptz_to_time_t(t) を呼び出し、内部表現を time_t に変換する。 2. localtime() を使用してローカル時刻構造体を取得する。 3. strftime() を用いて日付・時刻部分（%Y-%m-%d %H:%M:%S）とタイムゾーン部分（%Z）を生成する。 4. snprintf() を使用し、元のタイムスタンプから抽出したマイクロ秒部分（t % USECS_PER_SEC）を結合して静的バッファに書き込む。 5. バッファの先頭アドレスを返す。

呼び出し元と呼び出し先

  • **呼び出し先**: timestamptz_to_time_t（内部変換用）、localtime、strftime、snprintf（標準ライブラリ）。
  • **呼び出し元**: ツール内の記述子関数（rmgrdesc.c 等）や、contrib/test_decoding/test_decoding.c 等のバックエンドコードから参照されている。

関連データ構造

  • TimestampTz: PostgreSQLのタイムスタンプ型。
  • struct tm: 標準ライブラリの時刻構造体。
  • MAXDATELEN: 文字列バッファの最大長を規定する定数。

失敗モードと境界条件

  • 静的バッファを使用しているため、スレッドセーフではない。
  • strftime の結果が MAXDATELEN を超える場合、切り詰めが発生する可能性がある。

保守担当者向けメモ

  • 本関数はバックエンドの時刻処理ロジックをクライアントツールで再利用するための暫定的な互換レイヤーである。
  • 本来は src/common 等への共通化が望ましいが、現状は compat.c に実装が閉じ込められている。
