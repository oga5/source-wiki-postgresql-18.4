ginCombineData

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginbulk.c:29

目的

GINインデックス構築時、既存のキー（GinEntryAccumulator）に新しいTID（タプル識別子）を追加・マージする。

シグネチャまたは形状

static void ginCombineData(RBTNode *existing, const RBTNode *newdata, void *arg)

入力と出力

  • **入力**:
  • existing: 更新対象の既存ノード（GinEntryAccumulator型）。
  • newdata: 追加するデータを持つノード（GinEntryAccumulator型）。単一のTIDを含むことが前提。
  • arg: BuildAccumulator 構造体へのポインタ。メモリ管理に使用。
  • **出力**: なし（existingの内容を直接更新）。

前提条件と不変条件

  • newdata は単一のTIDのみを含むこと。
  • BuildAccumulator の allocatedMemory は、eo->list のメモリ確保量と同期していること。

アルゴリズムまたは振る舞い

1. **メモリ拡張**: eo->count が eo->maxcount に達した場合、repalloc_huge を用いてTID配列を倍増させる。 2. **エラーチェック**: maxcount が INT_MAX を超える場合は、maintenance_work_mem の削減を促すエラーを発生させる。 3. **順序判定**: 追加されるTIDが既存リストの末尾より小さい場合、shouldSort フラグを true に設定する。 4. **データ追加**: 新しいTIDを eo->list に格納し、eo->count をインクリメントする。

呼び出し元と呼び出し先

  • **呼び出し元**: 赤黒木（RBT）の挿入処理において、キーが重複した際に呼び出される（src/backend/access/gin/ginbulk.c 内のRBT操作に関連）。
  • **呼び出し先**: ginCompareItemPointers（順序判定用）、repalloc_huge（メモリ拡張用）、GetMemoryChunkSpace（メモリ管理用）。

関連データ構造

  • GinEntryAccumulator: キーとTIDリストを保持するRBTノード。
  • BuildAccumulator: インデックス構築状態とメモリ使用量を管理する構造体。
  • ItemPointerData: TIDを格納するデータ型。

失敗モードと境界条件

  • **メモリ不足**: maxcount が INT_MAX を超える場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED で停止する。
  • **順序不整合**: 挿入されたTIDが既存リストの末尾より小さい場合、shouldSort フラグが立ち、後続のソート処理が強制される。

保守担当者向けメモ

  • 本関数は RBT（赤黒木）のコールバックとして設計されている。
  • newdata が単一のTIDしか持たないという前提は、GINのバルク構築ロジックに強く依存しているため、変更時は注意が必要。
