int_custom_lt

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_custom_types/test_custom_types.c:134

目的

カスタム型に対する「より小さい」比較演算（<）を実装し、PostgreSQLの演算子インターフェースとして機能させる。

シグネチャまたは形状

Datum int_custom_lt(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS を介して受け取る2つの int32 値。
  • 出力: PG_RETURN_BOOL を介して、第1引数が第2引数より小さい場合に true、それ以外の場合に false を返す。

前提条件と不変条件

  • 入力値は int32 型として解釈可能であること。
  • PostgreSQLの関数マネージャ（fmgr）規約に従い、PG_GETARG_INT32 マクロで引数を取得する必要がある。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT32(0) および PG_GETARG_INT32(1) を呼び出し、比較対象の2つの整数を取得する。 2. C言語の比較演算子 < を用いて大小比較を行う。 3. 比較結果を PG_RETURN_BOOL マクロで PostgreSQL の boolean 型として返す。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLの演算子実行エンジン（SQLレベルで < 演算子が呼び出された際）。
  • 呼び出し先: PG_GETARG_INT32 および PG_RETURN_BOOL（PostgreSQLの関数マネージャAPI）。
  • その他: PG_FUNCTION_INFO_V1(int_custom_lt) により、PostgreSQLの関数情報として登録されている。

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • PG_FUNCTION_ARGS: 関数呼び出し時の引数コンテキスト。

失敗モードと境界条件

  • 整数値の範囲外の入力については、PG_GETARG_INT32 の仕様に依存する。
  • 比較対象が NULL の場合、PostgreSQLの呼び出し規約により本関数は呼び出されない（strict関数として定義されている場合）。

保守担当者向けメモ

  • 本関数はテストモジュールの一部であり、単純な int32 の比較を行う。
  • 複雑なロジックは含まれていないため、変更時は比較演算の論理が正しいかのみを確認すること。
