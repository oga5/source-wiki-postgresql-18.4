sqlca_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/thread-alloc.c:47

目的

ECPG（Embedded SQL for C）におけるSQL通信領域（SQL Communication Area）を定義し、データベース操作時のエラー情報や警告、実行結果のステータスを保持するために使用されます。

シグネチャまたは形状

struct sqlca_t {
    char    sqlcaid[8];
    long    sqlabc;
    long    sqlcode;
    struct {
        int     sqlerrml;
        char    sqlerrmc[SQLERRMC_LEN];
    } sqlerrm;
    char    sqlerrp[8];
    long    sqlerrd[6];
    char    sqlwarn[8];
    char    sqlstate[5];
};

入力と出力

  • **入力**: ECPGランタイムライブラリがSQL文の実行結果に基づいて本構造体の各フィールドを更新します。
  • **出力**: アプリケーションコードが本構造体を参照することで、直前のSQL実行の成功・失敗や、処理された行数、エラーメッセージなどを取得します。

前提条件と不変条件

  • sqlerrd[2] には、INSERT、UPDATE、DELETE 文の実行後に処理された行数が格納されます。
  • sqlwarn[0] は、他の警告フラグが一つでもセットされている場合に 'W' に設定されます。
  • sqlstate[5] は、SQL標準準拠のエラーコードを保持します。

アルゴリズムまたは振る舞い

ECPGライブラリの各関数（ECPGdo 等）が実行される際、内部的にこの構造体が更新されます。アプリケーションはこれを確認することで、エラーハンドリングや制御フローの分岐を行います。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/ecpglib/connect.c
  • src/interfaces/ecpg/ecpglib/data.c
  • src/interfaces/ecpg/ecpglib/descriptor.c
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • sqlda_t: 動的SQL記述子領域。
  • sqlvar_t: sqlda_t 内で使用される列定義構造体。

失敗モードと境界条件

  • sqlerrmc のサイズは SQLERRMC_LEN に制限されており、これを超えるエラーメッセージは切り詰められる可能性があります。
  • スレッド環境下では、ECPGget_sqlca() を介してスレッド固有の sqlca インスタンスを取得する必要があります。

保守担当者向けメモ

  • 本構造体はECPGのバイナリ互換性に直結するため、フィールドの順序やサイズを変更する際は注意が必要です。
  • thread-alloc.c はテストコードであり、この構造体がマルチスレッド環境で正しく機能することを検証する役割を担っています。
