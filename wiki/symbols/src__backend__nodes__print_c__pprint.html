pprint

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/print.c:53

目的

PostgreSQL の内部データ構造（Node）を整形（pretty-print）し、標準出力へダンプするデバッグ用ユーティリティです。

シグネチャまたは形状

void pprint(const void *obj)

入力と出力

  • **入力**: obj (ダンプ対象のノード構造体へのポインタ)
  • **出力**: なし（標準出力への書き出しのみ）

前提条件と不変条件

  • obj は nodeToStringWithLocations が処理可能な有効なノード構造体である必要があります。
  • 内部で pfree を使用するため、メモリコンテキストが適切に管理されている環境で呼び出される必要があります。

アルゴリズムまたは振る舞い

1. nodeToStringWithLocations(obj) を呼び出し、ノードをシリアライズされた文字列に変換します。 2. pretty_format_node_dump(s) を呼び出し、インデントや改行を挿入した整形済み文字列を生成します。 3. 生成された文字列を printf で標準出力に書き出し、fflush(stdout) でバッファをフラッシュします。 4. 使用した一時的な文字列メモリを pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/allpaths.c
  • src/backend/optimizer/plan/planner.c
  • src/include/nodes/print.h（nodeDisplay マクロ経由）
  • **呼び出し先**:
  • nodeToStringWithLocations
  • pretty_format_node_dump
  • pfree
  • printf
  • fflush

関連データ構造

  • Node: PostgreSQL のノード構造体全般。

失敗モードと境界条件

  • obj が NULL または不正なポインタである場合、nodeToStringWithLocations の内部実装に依存してクラッシュする可能性があります。
  • 非常に巨大なノード構造を渡した場合、文字列変換および整形処理によりメモリ消費量が増大します。

保守担当者向けメモ

  • 本関数はデバッグ専用です。本番環境のコードには含めないでください。
  • nodeDisplay マクロのエイリアスとして利用されています。
  • 整形ロジックはヒューリスティックに基づいているため、複雑なノード構造では表示が崩れる場合があります。
