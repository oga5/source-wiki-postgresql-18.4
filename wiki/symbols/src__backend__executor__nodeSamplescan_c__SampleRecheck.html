SampleRecheck

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSamplescan.c:59

目的

EvalPlanQual（EPQ）実行時に、サンプリングスキャンにおいてタプルの再チェックを行うためのルーチンです。

シグネチャまたは形状

static bool SampleRecheck(SampleScanState *node, TupleTableSlot *slot)

入力と出力

  • **入力**:
  • node: SampleScanState 構造体へのポインタ。
  • slot: 再チェック対象の TupleTableSlot。
  • **出力**:
  • 常に true を返します。

前提条件と不変条件

  • サンプリングスキャンでは、heap_beginscan にチェック可能なキーを渡さないため、再チェックの必要性がありません。

アルゴリズムまたは振る舞い

  • 呼び出されると、追加の検証処理を行わずに直ちに true を返します。これは SeqScan と同様の振る舞いです。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeSamplescan.c:85 において、ExecScanRecheckMtd 型の関数ポインタとして登録されています。
  • **呼び出し先**: 該当なし。

関連データ構造

  • SampleScanState: サンプリングスキャンの実行状態を保持する構造体。
  • TupleTableSlot: タプルを保持するためのスロット構造体。

失敗モードと境界条件

  • 特になし。常に成功（true）を返します。

保守担当者向けメモ

  • 本関数は SampleScan における再チェックのプレースホルダーとして機能しています。将来的にサンプリング手法がフィルタ条件を伴うようになった場合でも、現在の設計では再チェックは不要であるという前提に基づいています。
