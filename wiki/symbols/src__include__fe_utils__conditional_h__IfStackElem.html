IfStackElem

定義

  • 種別: struct
  • 定義箇所: src/include/fe_utils/conditional.h:58

目的

ネストされた \if ブロックの実行状態および解析コンテキストを管理するためのスタック要素です。

シグネチャまたは形状

typedef struct IfStackElem
{
    ifState             if_state;       /* 現在の状態 */
    int                 query_len;      /* 最後の分岐開始時のクエリバッファ長 */
    int                 paren_depth;    /* 最後の分岐開始時の括弧の深さ */
    struct IfStackElem *next;           /* 次のスタック要素（親ブロック）へのポインタ */
} IfStackElem;

入力と出力

  • **入力**: \if 文の解析時に生成され、現在の ifState および解析器のバッファ状態（長さ、括弧の深さ）を保持します。
  • **出力**: スタックのトップとして参照され、conditional_active() 関数が現在のブロックを実行すべきか否かを判定するための情報を提供します。

前提条件と不変条件

  • next ポインタは、より外側の \if ブロックを指すか、スタックの終端であれば NULL となります。
  • query_len および paren_depth は、非アクティブなブロックにおいてバッファを適切に巻き戻すための基準点として機能します。

アルゴリズムまたは振る舞い

  • インタプリタが \if に遭遇すると、新しい IfStackElem が生成されスタックにプッシュされます。
  • 非アクティブなブロック（ifState が IFSTATE_FALSE や IFSTATE_IGNORED）に入った際、解析器は本構造体に保存されたメタデータを用いて、不要なテキストの破棄や解析状態の制御を行います。
  • \endif に遭遇すると、スタックからポップされ破棄されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/conditional.c 内のスタック操作関数（プッシュ、ポップ、参照処理）からインスタンス化および操作されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ifState: 条件分岐の状態を定義する列挙型。
  • ConditionalStack: IfStackElem の連結リストを管理する構造体。

失敗モードと境界条件

  • メモリ不足時: pg_malloc による割り当て失敗が発生する可能性があります。
  • スタックアンダーフロー: \endif が \if よりも多く呼び出された場合の挙動は、呼び出し側の制御ロジックに依存します。

保守担当者向けメモ

  • 本構造体は psql や pgbench のスクリプト実行制御において、条件分岐のネストを追跡する唯一の手段です。
  • query_len と paren_depth の値は、非アクティブなブロック内の構文解析をスキップする際に重要となるため、変更時は psqlscan との整合性に注意してください。
