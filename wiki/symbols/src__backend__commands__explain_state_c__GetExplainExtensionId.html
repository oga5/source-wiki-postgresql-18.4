GetExplainExtensionId

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_state.c:220

目的

拡張機能（Extension）に対して一意の整数 ID を割り当て、ExplainState 内の拡張機能用状態管理配列へのインデックスを提供します。

シグネチャまたは形状

int GetExplainExtensionId(const char *extension_name);

入力と出力

  • **入力**: extension_name (拡張機能名を識別する文字列)
  • **出力**: 割り当てられた整数 ID（0 以上のインデックス）

前提条件と不変条件

  • 返される ID は、現在のバックエンドプロセスの生存期間中のみ有効です。
  • 拡張機能名が既に登録されている場合、既存の ID が返されます。

アルゴリズムまたは振る舞い

1. ExplainExtensionNameArray を走査し、指定された extension_name が既に存在するか確認します。見つかった場合はそのインデックスを返します。 2. 配列が未初期化の場合、TopMemoryContext 上に初期サイズ 16 で配列を確保します。 3. 配列が満杯の場合、pg_nextpower2_32 を用いて容量を拡張し、repalloc でメモリを再確保します。 4. 新しい拡張機能名を配列の末尾に追加し、そのインデックスを返すと同時に ExplainExtensionNamesAssigned をインクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_overexplain/pg_overexplain.c 等の拡張機能モジュール。
  • **呼び出し先**: MemoryContextAlloc, repalloc, pg_nextpower2_32 (メモリ管理およびサイズ計算用)。

関連データ構造

  • ExplainExtensionNameArray: 拡張機能名を保持する静的配列。
  • ExplainExtensionNamesAssigned: 現在割り当て済みの拡張機能数。
  • ExplainExtensionNamesAllocated: 配列の現在の確保済みサイズ。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAlloc または repalloc がエラーを発生させます。
  • 拡張機能名が NULL の場合の挙動は定義されていません。

保守担当者向けメモ

  • 本関数で取得した ID は、SetExplainExtensionState / GetExplainExtensionState を使用して ExplainState 内の extension_state 配列にアクセスする際のキーとして使用されます。
  • 配列の拡張に伴いポインタが再配置される可能性があるため、拡張機能側で ID をキャッシュする際は注意してください。
