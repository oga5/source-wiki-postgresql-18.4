GetAggInitVal

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeWindowAgg.c:3134

目的

集約関数の初期値（initval）を、テキスト形式のデータから指定された遷移型（transtype）の Datum 形式へ変換して取得する。

シグネチャまたは形状

Datum GetAggInitVal(Datum textInitVal, Oid transtype)

入力と出力

  • **入力**:
  • textInitVal: 初期値を表すテキスト形式の Datum。
  • transtype: 初期値の変換先となるデータ型の Oid。
  • **出力**:
  • 変換後の Datum 型の初期値。

前提条件と不変条件

  • transtype に対応する入力関数がシステムカタログから取得可能であること。
  • textInitVal は TextDatumGetCString で処理可能なテキストデータであること。

アルゴリズムまたは振る舞い

1. getTypeInputInfo を呼び出し、transtype に対応する入力関数（typinput）とパラメータ（typioparam）を取得する。 2. TextDatumGetCString を用いて、textInitVal を C 文字列（strInitVal）に変換する。 3. OidInputFunctionCall を使用し、取得した入力関数を介して文字列を Datum 型へ変換する。 4. 変換に使用した一時的な C 文字列を pfree で解放する。 5. 変換された Datum を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAgg.c
  • src/backend/executor/nodeWindowAgg.c
  • src/backend/optimizer/prep/prepagg.c
  • **呼び出し先**:
  • getTypeInputInfo
  • TextDatumGetCString
  • OidInputFunctionCall
  • pfree

関連データ構造

  • Datum: PostgreSQL の汎用データ表現型。
  • Oid: オブジェクト識別子。

失敗モードと境界条件

  • 指定された transtype に有効な入力関数が存在しない場合、getTypeInputInfo がエラーを発生させる可能性がある。
  • textInitVal が不正な形式である場合、OidInputFunctionCall 内で変換エラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は nodeAgg.c および nodeWindowAgg.c、prepagg.c で共通して利用されているユーティリティ関数である。修正を行う際は、これらすべてのコンテキストで動作に影響がないか確認すること。
