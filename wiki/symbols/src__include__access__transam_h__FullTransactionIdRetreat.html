FullTransactionIdRetreat

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/transam.h:102

目的

64ビットの FullTransactionId を1つ前の値へ戻す（デクリメントする）。その際、32ビットのトランザクションID（XID）体系において特殊な意味を持つ予約済みIDをスキップする。

シグネチャまたは形状

static inline void FullTransactionIdRetreat(FullTransactionId *dest)

入力と出力

  • **入力**: FullTransactionId *dest（デクリメント対象の64ビットトランザクションIDへのポインタ）
  • **出力**: なし（ポインタ経由で dest の値を直接更新）

前提条件と不変条件

  • 64ビットの FullTransactionId は、32ビットXIDのようなラップアラウンドによる特殊IDの到達可能性を持たない。
  • FirstNormalFullTransactionId より前の値（特殊ID領域）に到達した場合は、それ以上のデクリメントを行わない。

アルゴリズムまたは振る舞い

1. dest->value を1減算する。 2. 更新後の値が FirstNormalFullTransactionId より小さい場合、処理を終了する。 3. 更新後の値に対応する32ビットXIDが FirstNormalTransactionId 未満である限り、dest->value を繰り返し1ずつ減算し、特殊なXIDをスキップする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c:6123
  • src/backend/storage/ipc/procarray.c:4508
  • **呼び出し先**:
  • FullTransactionIdPrecedes (マクロ/インライン関数)
  • XidFromFullTransactionId (マクロ/インライン関数)

関連データ構造

  • FullTransactionId: 64ビットのトランザクション識別子。
  • FirstNormalFullTransactionId: 通常のトランザクションIDの開始値。
  • FirstNormalTransactionId: 32ビットXIDにおける通常IDの開始値。

失敗モードと境界条件

  • FirstNormalFullTransactionId に達した後は、それ以上デクリメントされない。
  • 32ビットXIDの特殊領域（FirstNormalTransactionId 未満）に該当する値は、ループ処理によってスキップされる。

保守担当者向けメモ

  • 本関数は32ビットXIDの互換性を維持するために特殊IDをスキップするロジックを含んでいる。64ビットID自体はラップアラウンドしないが、32ビットXIDへの変換時に特殊な意味を持つ値を避ける必要がある点に注意すること。
