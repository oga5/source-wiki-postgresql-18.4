FreeExprContext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execUtils.c:415

目的

式評価コンテキスト（ExprContext）を破棄し、関連するメモリおよび登録済みコールバックをクリーンアップする。

シグネチャまたは形状

void FreeExprContext(ExprContext *econtext, bool isCommit)

入力と出力

  • econtext: 破棄対象の ExprContext 構造体へのポインタ。
  • isCommit: トランザクション終了時に呼び出されるコールバックへ渡されるフラグ。
  • 戻り値: なし。

前提条件と不変条件

  • econtext は有効な ExprContext を指している必要がある。
  • econtext が EState に所有されている場合、その EState の es_exprcontexts リストから適切にリンクが解除される。

アルゴリズムまたは振る舞い

1. ShutdownExprContext(econtext, isCommit) を呼び出し、登録されたすべてのクリーンアップコールバックを実行する。 2. econtext->ecxt_per_tuple_memory が保持するメモリコンテキストを MemoryContextDelete で解放する。 3. econtext が EState に関連付けられている場合、estate->es_exprcontexts リストから当該 econtext を削除する。 4. 最後に pfree(econtext) を実行し、ExprContext 構造体自体のメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execUtils.c (内部的な管理処理)
  • src/pl/plpgsql/src/pl_exec.c (PL/pgSQL の実行終了時など)
  • **呼び出し先**:
  • ShutdownExprContext
  • MemoryContextDelete
  • list_delete_ptr
  • pfree

関連データ構造

  • ExprContext: 式評価の状態とメモリコンテキストを保持する構造体。
  • EState: クエリ実行全体の状態を保持する構造体。

失敗モードと境界条件

  • econtext が NULL の場合の挙動は定義されていない（ソースコード上はポインタ参照が発生するためクラッシュの可能性がある）。
  • econtext->ecxt_estate が NULL の場合、EState からのリンク解除処理はスキップされる。

保守担当者向けメモ

  • 本関数は ExprContext のライフサイクル管理の終端である。コールバックの実行順序やメモリ解放のタイミングは、ExprContext を利用する他のプランノードのクリーンアップ処理と密接に関係しているため、変更時は注意が必要である。
