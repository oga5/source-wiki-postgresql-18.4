XLogPrefetcherAllocate

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogprefetcher.c:361

目的

WALリカバリ中の先読み処理を管理するための XLogPrefetcher 構造体を初期化し、メモリを割り当てる。

シグネチャまたは形状

XLogPrefetcher *XLogPrefetcherAllocate(XLogReaderState *reader)

入力と出力

  • **入力**: reader (WALレコードを読み取るための XLogReaderState へのポインタ)
  • **出力**: 初期化済みの XLogPrefetcher 構造体へのポインタ

前提条件と不変条件

  • palloc0 を使用するため、呼び出し時に有効なメモリコンテキストが存在すること。
  • SharedStats が初期化済みであること。

アルゴリズムまたは振る舞い

1. XLogPrefetcher 構造体用にメモリをゼロクリアして割り当てる。 2. 引数の reader を構造体に保持する。 3. RelFileLocator をキーとするハッシュテーブル filter_table を作成し、フィルタリング用の双方向リスト filter_queue を初期化する。 4. 共有メモリ上の統計情報（SharedStats）の wal_distance、block_distance、io_depth を 0 にリセットする。 5. reconfigure_count を XLogPrefetchReconfigureCount - 1 に設定し、初回利用時にストリーミング読み取り設定が割り当てられるようにする。 6. 構築した prefetcher を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlogrecovery.c 内のリカバリ初期化処理から呼び出される。
  • **呼び出し先**: palloc0 (メモリ確保)、hash_create (ハッシュテーブル作成)、dlist_init (リスト初期化)。

関連データ構造

  • XLogPrefetcher: 先読み状態を保持するメイン構造体。
  • XLogReaderState: WAL読み取り状態。
  • XLogPrefetcherFilter: フィルタテーブルの各エントリ。
  • SharedStats: 共有メモリ上の統計管理構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる。
  • hash_create が失敗した場合、PostgreSQLの標準的なエラーハンドリングによりプロセスが停止する可能性がある。

保守担当者向けメモ

  • 本関数はリカバリ開始時に一度だけ呼び出されることを想定している。
  • reconfigure_count の初期値設定は、将来的なGUC変更による再構成ロジックと密接に関連しているため、変更時は XLogPrefetchReconfigureCount の定義を確認すること。
