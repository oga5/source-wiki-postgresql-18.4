tarChecksum

定義

  • 種別: function
  • 定義箇所: src/port/tar.c:89

目的

POSIX 準拠の tar ヘッダーブロックのチェックサムを算出します。

シグネチャまたは形状

int tarChecksum(const char *header);

入力と出力

  • **入力**: header - 512 バイトの tar ヘッダーブロックを指すポインタ。
  • **出力**: 計算されたチェックサム値を int 型で返します。

前提条件と不変条件

  • header は少なくとも 512 バイトの有効なメモリ領域を指している必要があります。
  • チェックサムフィールド（TAR_OFFSET_CHECKSUM から 8 バイト分）は、計算時に「8 つのスペース（ASCII 0x20）」として扱われます。

アルゴリズムまたは振る舞い

1. 初期値として 8 * ' '（スペース 8 個分の合計値）を sum に設定します。 2. ヘッダーの先頭から 512 バイト（TAR_BLOCK_SIZE）までループ処理を行います。 3. 現在のインデックスがチェックサムフィールドの範囲外である場合のみ、そのバイト値を符号なし整数として sum に加算します。 4. 最終的な sum を返します。

呼び出し元と呼び出し先

  • **呼び出し先**:

関連データ構造

  • TAR_BLOCK_SIZE（ヘッダーサイズ）
  • TAR_OFFSET_CHECKSUM（ヘッダー内でのチェックサムフィールドのオフセット）

失敗モードと境界条件

  • 入力ポインタが NULL の場合の挙動は定義されていません。
  • チェックサムフィールドの範囲外のバイトのみを合計対象とするため、ヘッダーの構造が pgtar.h の定義と一致している必要があります。

保守担当者向けメモ

  • 本関数は POSIX 規格に従い、チェックサムフィールドをスペースとして扱うことで、ヘッダー生成時および検証時の整合性を確保しています。
  • GNU 拡張などの特殊なフォーマットを扱う場合でも、チェックサム計算の基本ロジックは本関数に依存します。
