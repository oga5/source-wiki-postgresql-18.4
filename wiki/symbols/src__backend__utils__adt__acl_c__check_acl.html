check_acl

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/acl.c:590

目的

PostgreSQL のアクセス制御リスト（ACL）として渡された配列が、有効な AclItem の配列であるかを検証する。

シグネチャまたは形状

static void check_acl(const Acl *acl)

入力と出力

  • **入力**: const Acl *acl - 検証対象の ACL 配列へのポインタ。
  • **出力**: なし。検証に失敗した場合は ereport(ERROR, ...) により処理を中断する。

前提条件と不変条件

  • 入力された acl は PostgreSQL の配列型として正しく初期化されている必要がある。

アルゴリズムまたは振る舞い

以下の順序で配列の妥当性をチェックする。 1. **型チェック**: 配列の要素型が ACLITEMOID であることを確認する。 2. **次元チェック**: 配列が 1 次元であることを確認する。 3. **NULL チェック**: 配列内に NULL 値が含まれていないことを確認する。 いずれかの条件を満たさない場合、適切なエラーコード（ERRCODE_INVALID_PARAMETER_VALUE または ERRCODE_NULL_VALUE_NOT_ALLOWED）でエラーを発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/acl.c 内の複数の関数（aclupdate 等の ACL 操作関数）から、ACL 配列を処理する直前のガードとして呼び出されている。
  • **呼び出し先**: PostgreSQL の配列操作マクロ（ARR_ELEMTYPE, ARR_NDIM, ARR_HASNULL）およびエラー報告関数 ereport を呼び出す。

関連データ構造

  • Acl: acl.h で定義される ACL 配列型。
  • AclItem: ACL 配列の各要素。

失敗モードと境界条件

  • **型不一致**: 要素型が ACLITEMOID でない場合にエラー。
  • **次元不一致**: 多次元配列が渡された場合にエラー。
  • **NULL 含有**: 配列要素に NULL が含まれる場合にエラー。

保守担当者向けメモ

  • 本関数は ACL 関連の関数における入力バリデーションの標準的な入り口である。ACL 構造の定義に変更があった場合は、本関数のチェックロジックも追従する必要がある。
