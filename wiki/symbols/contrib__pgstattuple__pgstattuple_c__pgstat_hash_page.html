pgstat_hash_page

定義

  • 種別: function
  • 定義箇所: contrib/pgstattuple/pgstattuple.c:454

目的

ハッシュインデックスの特定のページを走査し、統計情報（空き容量やタプル数）を収集する。

シグネチャまたは形状

static void pgstat_hash_page(pgstattuple_type *stat, Relation rel, BlockNumber blkno, BufferAccessStrategy bstrategy)

入力と出力

  • **入力**:
  • stat: 統計情報を格納する pgstattuple_type 構造体へのポインタ。
  • rel: 対象のハッシュインデックスリレーション。
  • blkno: 走査対象のブロック番号。
  • bstrategy: バッファアクセスのための戦略オブジェクト。
  • **出力**: なし（stat 構造体のフィールドを直接更新する）。

前提条件と不変条件

  • rel はハッシュインデックスである必要がある。
  • 呼び出し側で適切なロック（AccessShareLock 等）が保持されていること。
  • 内部で ReadBufferExtended を使用するため、バッファマネージャが利用可能であること。

アルゴリズムまたは振る舞い

1. ReadBufferExtended を使用して指定されたブロックを読み込み、HASH_READ モードでロックする。 2. PageIsNew の場合、ページは完全に空とみなし、stat->free_space に BLCKSZ を加算する。 3. ページがハッシュインデックスの形式（HashPageOpaqueData）であることを確認する。 4. ページのタイプ（hasho_flag）に応じて分岐する:

  • LH_UNUSED_PAGE: 空きページとして stat->free_space に BLCKSZ を加算。
  • LH_BUCKET_PAGE または LH_OVERFLOW_PAGE: pgstat_index_page を呼び出し、ページ内のタプルを走査して統計を更新。
  • その他（LH_BITMAP_PAGE, LH_META_PAGE 等）: 何も行わない。

    • 5. 最後に _hash_relbuf を呼び出し、バッファのロック解除と解放を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgstattuple/pgstattuple.c 内のインデックス走査ロジックから参照されている。
  • **呼び出し先**:
  • ReadBufferExtended, LockBuffer, BufferGetPage, PageIsNew, PageGetSpecialSize, HashPageGetOpaque, PageGetMaxOffsetNumber, _hash_relbuf
  • pgstat_index_page（タプル走査時）

関連データ構造

  • pgstattuple_type: 統計情報の集計先。
  • HashPageOpaqueData: ハッシュインデックスページのメタデータ構造。
  • BufferAccessStrategy: バッファ読み込みの最適化戦略。

失敗モードと境界条件

  • ページが破損している場合、PageGetSpecialSize のチェックで弾かれ、何も処理されない（コメントに「maybe corrupted」と記載）。
  • PageIsNew の判定により、初期化されていないページは空き容量として正しく計上される。

保守担当者向けメモ

  • 本関数はハッシュインデックス特有のページレイアウトに依存している。ハッシュインデックスの内部構造に変更があった場合は、hasho_flag の判定ロジックを更新する必要がある。
  • _hash_relbuf は access/hash.h で定義されている内部関数であるため、依存関係の変更に注意すること。
