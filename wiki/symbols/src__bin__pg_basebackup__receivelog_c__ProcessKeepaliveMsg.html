ProcessKeepaliveMsg

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/receivelog.c:992

目的

ストリーミングレプリケーション中にサーバーから送信される Keepalive メッセージ（型 'k'）を解析し、サーバーから要求された場合にフィードバックを返送する。

シグネチャまたは形状

static bool ProcessKeepaliveMsg(PGconn *conn, StreamCtl *stream, char *copybuf, int len, XLogRecPtr blockpos, TimestampTz *last_status)

入力と出力

  • **入力**:
  • conn: サーバー接続用 PGconn オブジェクト。
  • stream: ストリーミング設定を保持する StreamCtl 構造体。
  • copybuf: 受信した CopyData メッセージのバッファ。
  • len: メッセージの長さ。
  • blockpos: 現在のWAL位置（XLogRecPtr）。
  • last_status: 最後にステータスを更新した時刻へのポインタ。
  • **出力**:
  • 成功時に true、エラー発生時またはフィードバック送信失敗時に false を返す。

前提条件と不変条件

  • copybuf は 'k' 型のメッセージを含む CopyData メッセージであること。
  • still_sending が真である場合にのみフィードバック送信処理が行われる。

アルゴリズムまたは振る舞い

1. メッセージヘッダー（型、WAL終了位置、送信時刻）をスキップし、サーバーからの応答要求フラグを読み取る。 2. メッセージ長が不正な場合はエラーを記録して false を返す。 3. サーバーが応答を要求しており、かつ still_sending が真の場合:

  • reportFlushPosition が有効かつ lastFlushPosition < blockpos であれば、現在のWALファイルを sync（fsync）し、lastFlushPosition を更新する。
  • feGetCurrentTimestamp() で現在時刻を取得し、sendFeedback を呼び出してサーバーへ応答を送信する。
  • last_status を現在時刻で更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/receivelog.c 内の HandleCopyStream 関数（828行目付近）。
  • **呼び出し先**:
  • stream->walmethod->ops->sync: WALファイルの同期書き込み。
  • feGetCurrentTimestamp(): 現在時刻の取得。
  • sendFeedback(): サーバーへのフィードバック送信。
  • pg_log_error / pg_fatal: エラーログ出力。

関連データ構造

  • StreamCtl: ストリーミングの制御パラメータを保持。
  • PGconn: libpq の接続ハンドル。
  • WalWriteMethod: WAL書き込みメソッドの抽象化インターフェース。

失敗モードと境界条件

  • メッセージ長が期待値より短い場合、エラーを返して処理を中断する。
  • sync 呼び出しが失敗した場合、pg_fatal によりプロセスが終了する。
  • sendFeedback が失敗した場合、関数は false を返す。

保守担当者向けメモ

  • 本関数は CopyData メッセージ内の Keepalive メッセージのみを処理する。
  • sync 処理は同期レプリケーションの正常終了時に重要となるため、変更時は注意が必要である。
  • walfile はモジュール内の静的変数として参照されている。
