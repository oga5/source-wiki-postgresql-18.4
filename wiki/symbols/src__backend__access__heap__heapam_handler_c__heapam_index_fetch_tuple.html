heapam_index_fetch_tuple

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/heap/heapam_handler.c:114

目的

インデックススキャンにおいて、指定されたタプルID（TID）に対応するヒープタプルを検索し、可視性判定を経てスロットへ格納する。HOT（Heap Only Tuple）チェーンの追跡をサポートする。

シグネチャまたは形状

bool heapam_index_fetch_tuple(
    struct IndexFetchTableData *scan,
    ItemPointer tid,
    Snapshot snapshot,
    TupleTableSlot *slot,
    bool *call_again,
    bool *all_dead
)

入力と出力

  • **入力**:
  • scan: スキャン状態を保持するデータ構造。
  • tid: 検索対象のタプルID。
  • snapshot: 可視性判定に使用するスナップショット。
  • slot: 取得したタプルを格納するバッファ付きスロット。
  • call_again: HOTチェーンの継続確認用フラグ。
  • **出力**:
  • 戻り値: タプルが見つかり、かつ可視である場合に true、それ以外は false。
  • slot: 成功時にタプルデータが格納される。
  • call_again: HOTチェーンの続きがある場合に true が設定される。
  • all_dead: 走査対象がすべて無効である場合に設定される。

前提条件と不変条件

  • slot は TTS_IS_BUFFERTUPLE を満たす必要がある（Assert による検証）。
  • scan は IndexFetchHeapData 型としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. **バッファ管理**: call_again が false の場合、tid が指すブロックへバッファを切り替え、必要に応じてページ内の不要領域排除（heap_page_prune_opt）を行う。 2. **可視性判定**: バッファを共有ロックし、heap_hot_search_buffer を呼び出してHOTチェーンを検索する。 3. **スロットへの格納**: タプルが見つかった場合、ExecStoreBufferHeapTuple を使用してスロットに格納する。 4. **HOTチェーン継続判定**: MVCCスナップショットでない場合は、複数のチェーン要素が可視である可能性があるため、call_again を true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_handler.c 内の heapam_methods 構造体（index_fetch_tuple フィールド）を通じて呼び出される。
  • **呼び出し先**:
  • ReleaseAndReadBuffer, LockBuffer (storage/bufmgr.h)
  • heap_page_prune_opt (src/backend/access/heap/pruneheap.c)
  • heap_hot_search_buffer (src/backend/access/heap/heapam.c)
  • ExecStoreBufferHeapTuple (executor/executor.h)

関連データ構造

  • IndexFetchHeapData: ヒープスキャン用の拡張データ構造。
  • BufferHeapTupleTableSlot: バッファ付きタプルスロット。
  • TableAmRoutine: 本関数が登録されるテーブルAMのルーチン構造体。

失敗モードと境界条件

  • 指定されたTIDが物理的に存在しない、あるいはスナップショットにより可視でない場合は false を返す。
  • HOTチェーンの末尾に到達した場合は call_again を false に設定して終了する。

保守担当者向けメモ

  • 本関数は heapam.c が提供する低レベルなヒープ操作関数をラップするブリッジである。
  • 複雑な可視性ロジックやページ操作の詳細は heapam.c および heapam_visibility.c を参照すること。
