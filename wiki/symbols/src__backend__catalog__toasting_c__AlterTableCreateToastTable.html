AlterTableCreateToastTable

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/toasting.c:57

目的

既存のテーブルに対して、TOAST（The Oversized-Attribute Storage Technique）テーブルを後から追加するために使用されます。

シグネチャまたは形状

void AlterTableCreateToastTable(Oid relOid, Datum reloptions, LOCKMODE lockmode)

入力と出力

  • relOid: TOASTテーブルを追加する対象テーブルのOID。
  • reloptions: TOASTテーブルの作成時に適用するオプション（Datum形式）。
  • lockmode: テーブル操作時に取得するロックモード。
  • 出力: なし（void）。

前提条件と不変条件

  • 対象テーブルがTOASTテーブルを必要とする状態である必要があります。
  • 内部的に CheckAndCreateToastTable を呼び出すため、同関数が要求するカタログ操作の整合性条件に依存します。

アルゴリズムまたは振る舞い

1. 指定された relOid、reloptions、lockmode を引数として CheckAndCreateToastTable を呼び出します。 2. CheckAndCreateToastTable の第4引数に true を渡し、既存テーブルへのTOAST追加であることを明示します。 3. 第5引数には InvalidOid を渡し、新規作成プロセスを実行します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/tablecmds.c:5368（ALTER TABLE コマンド実行時）。
  • 呼び出し先: CheckAndCreateToastTable（src/backend/catalog/toasting.c 内）。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。
  • Datum: PostgreSQLの汎用データ表現。
  • LOCKMODE: ロック制御用列挙型。

失敗モードと境界条件

  • 対象テーブルが既にTOASTテーブルを持っている場合や、TOASTをサポートしないシステムカタログ等の場合は、CheckAndCreateToastTable 内の判定ロジックにより処理がスキップまたはエラーとなります。

保守担当者向けメモ

  • 本関数は ALTER TABLE コマンドのラッパーとして機能します。TOASTテーブルの物理的な作成ロジックは create_toast_table に集約されているため、作成手順の変更が必要な場合はそちらを参照してください。
