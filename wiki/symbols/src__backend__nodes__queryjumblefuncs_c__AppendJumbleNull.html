AppendJumbleNull

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/queryjumblefuncs.c:309

目的

クエリのフィンガープリント生成プロセスにおいて、NULL 値の出現を記録し、ハッシュ計算の効率化と正規化を補助する。

シグネチャまたは形状

static pg_attribute_always_inline void AppendJumbleNull(JumbleState *jstate)

入力と出力

  • **入力**: JumbleState *jstate（ハッシュ計算の状態を保持する構造体へのポインタ）
  • **出力**: なし（副作用として jstate を更新）

前提条件と不変条件

  • jstate は有効な JumbleState 構造体を指している必要がある。
  • pg_attribute_always_inline が指定されており、呼び出し元でインライン展開されることが前提となっている。

アルゴリズムまたは振る舞い

1. jstate->pending_nulls カウンタをインクリメントする。 2. 連続する NULL 値を直接ハッシュバッファに書き込まず、カウンタで集約することで、ハッシュ計算のオーバーヘッドを削減する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/queryjumblefuncs.c:552
  • src/backend/nodes/queryjumblefuncs.c:570
  • **呼び出し先**: なし（インライン関数であり、他の関数を呼び出さない）

関連データ構造

  • JumbleState: クエリのハッシュ計算用バッファや pending_nulls カウンタを保持する構造体。

失敗モードと境界条件

  • jstate が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • pending_nulls カウンタのオーバーフローについては、本関数単体では制御していない。

保守担当者向けメモ

  • 本関数は FlushPendingNulls と対で使用される。NULL が連続する箇所で呼び出され、最終的に FlushPendingNulls が呼ばれた際にカウンタ値がハッシュバッファに反映される仕組みである。
  • パフォーマンス向上のため、インライン化が必須である。変更を加える際は、コンパイラによるインライン展開が阻害されないよう注意すること。
