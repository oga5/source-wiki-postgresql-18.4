ResourceOwnerForgetPlanCacheRef

定義

  • 種別: static inline function
  • 定義箇所: src/backend/utils/cache/plancache.c:130

目的

指定された ResourceOwner が保持している CachedPlan への参照を解除し、リソース管理対象から除外する。

シグネチャまたは形状

void ResourceOwnerForgetPlanCacheRef(ResourceOwner owner, CachedPlan *plan)

入力と出力

  • owner: 参照を解除する対象の ResourceOwner。
  • plan: 参照を解除する対象の CachedPlan 構造体へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • ResourceOwnerRememberPlanCacheRef を通じて、事前に当該 plan が owner に登録されている必要がある。
  • plan は有効な CachedPlan 構造体を指している必要がある。

アルゴリズムまたは振る舞い

1. plan ポインタを Datum 型に変換する。 2. planref_resowner_desc をリソース記述子として使用し、ResourceOwnerForget を呼び出して owner から当該リソースの追跡を削除する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/plancache.c 内の ResourceOwnerForgetPlanCacheRef 呼び出し箇所（例: 1409 行目付近）。
  • 呼び出し先: ResourceOwnerForget（utils/resowner.h で定義）。

関連データ構造

  • CachedPlan: キャッシュされた実行計画を保持する構造体。
  • ResourceOwner: リソースのライフサイクルを管理するオブジェクト。
  • planref_resowner_desc: CachedPlan 参照を管理するためのリソース記述子。

失敗モードと境界条件

  • owner が無効な場合や、plan が当該 owner に登録されていない場合に ResourceOwnerForget がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は ResourceOwner を介したメモリ管理の整合性を保つために不可欠である。CachedPlan を解放する前に必ず呼び出す必要がある。
  • planref_resowner_desc の定義および ResourceOwner の詳細については src/include/utils/resowner.h を参照のこと。
