lexeme_compare

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_typanalyze.c:499

目的

2つの LexemeHashKey 構造体を比較し、辞書順序を決定する。

シグネチャまたは形状

static int lexeme_compare(const void *key1, const void *key2)

入力と出力

  • **入力**: key1, key2 (比較対象となる LexemeHashKey へのポインタをキャストした void *)
  • **出力**: 比較結果を示す整数。
  • 1: key1 が key2 より大きい場合
  • -1: key1 が key2 より小さい場合
  • 0: 両者が等しい場合（strncmp の結果に依存）

前提条件と不変条件

  • 入力ポインタは LexemeHashKey 構造体としてキャスト可能であること。
  • LexemeHashKey 内の length フィールドと lexeme 文字列が正しく初期化されていること。

アルゴリズムまたは振る舞い

1. まず length フィールドを比較し、長さが異なる場合はその大小関係を返す。 2. 長さが等しい場合、strncmp を使用して lexeme 文字列をバイト単位で比較し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/ts_typanalyze.c 内のハッシュテーブル操作やソート処理から呼び出される（直接的な呼び出し箇所として 493 行目および 535 行目が確認されている）。
  • **呼び出し先**:
  • strncmp (標準ライブラリ関数)

関連データ構造

  • LexemeHashKey: 語彙の文字列ポインタと長さを保持する構造体。

失敗モードと境界条件

  • length が 0 の場合、strncmp は 0 を返し、等価とみなされる。
  • 入力ポインタが NULL の場合の挙動は定義されていない。

保守担当者向けメモ

  • 本関数は LexemeHashKey のバイナリ比較を行うため、ロケール依存のソート順序ではなく、バイト単位の厳密な比較を行う点に注意すること。
