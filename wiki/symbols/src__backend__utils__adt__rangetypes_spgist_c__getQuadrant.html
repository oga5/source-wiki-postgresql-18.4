getQuadrant

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_spgist.c:94

目的

範囲型データ（RangeType）を、指定された重心（centroid）を基準とした4つの象限（1〜4）または空範囲用の象限（5）に分類する。

シグネチャまたは形状

int16 getQuadrant(TypeCacheEntry *typcache, const RangeType *centroid, const RangeType *tst)

入力と出力

  • **入力**:
  • typcache: 範囲型の型キャッシュ情報。
  • centroid: 分割の基準となる重心範囲。
  • tst: 分類対象の範囲。
  • **出力**:
  • int16: 象限番号（1〜5）。

前提条件と不変条件

  • typcache は有効な範囲型キャッシュであること。
  • centroid および tst は range_deserialize で解析可能な有効な範囲データであること。

アルゴリズムまたは振る舞い

1. range_deserialize を用いて centroid と tst の境界値（下限・上限）および空フラグを抽出する。 2. tst が空範囲であれば 5 を返す。 3. tst の下限と centroid の下限を比較し、tst の上限と centroid の上限を比較する。

  • 下限 >= 重心下限 かつ 上限 >= 重心上限: 1
  • 下限 >= 重心下限 かつ 上限 < 重心上限: 2
  • 下限 < 重心下限 かつ 上限 < 重心上限: 3
  • 下限 < 重心下限 かつ 上限 >= 重心上限: 4

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgquadtreeproc.c 内の複数の箇所（SP-GiSTインデックスの構築および検索時）。
  • src/backend/utils/adt/geo_spgist.c（幾何学型のSP-GiST実装）。
  • **呼び出し先**:
  • range_deserialize
  • range_cmp_bounds

関連データ構造

  • RangeType: 範囲型の内部表現。
  • RangeBound: 範囲の境界値（下限・上限）。
  • TypeCacheEntry: 型のメタデータキャッシュ。

失敗モードと境界条件

  • tst が空範囲の場合、常に 5 を返す。
  • 境界値の比較において range_cmp_bounds が使用されており、型定義に応じた比較ロジックに依存する。

保守担当者向けメモ

  • 本関数は範囲型専用の実装であるが、geo_spgist.c でも同名の関数が定義されており、型ごとに空間分割のロジックが独立している点に注意すること。
  • 象限の番号付け（1〜4）は、空間分割の論理的な配置に対応している。
