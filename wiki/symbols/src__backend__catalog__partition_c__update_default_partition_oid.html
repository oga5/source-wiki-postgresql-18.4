update_default_partition_oid

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/partition.c:339

目的

指定されたパーティションテーブル（parentId）のデフォルトパーティションのOIDを、システムカタログ pg_partitioned_table 上で更新します。

シグネチャまたは形状

void update_default_partition_oid(Oid parentId, Oid defaultPartId)

入力と出力

  • **入力**:
  • parentId (Oid): 更新対象のパーティションテーブルのOID。
  • defaultPartId (Oid): 新たに設定するデフォルトパーティションのOID（解除時は InvalidOid を指定）。
  • **出力**: なし（副作用としてシステムカタログを更新）。

前提条件と不変条件

  • parentId に対応するエントリが pg_partitioned_table に存在すること。
  • 呼び出し側で適切なロック（RowExclusiveLock 等）が確保されていることが期待されます。

アルゴリズムまたは振る舞い

1. pg_partitioned_table を RowExclusiveLock モードで開きます。 2. SearchSysCacheCopy1 を使用して、PARTRELID キャッシュから parentId に該当するタプルのコピーを取得します。 3. キャッシュルックアップが失敗した場合は elog(ERROR) を発生させます。 4. 取得したタプルの partdefid フィールドを defaultPartId に書き換えます。 5. CatalogTupleUpdate を呼び出し、変更をシステムカタログに永続化します。 6. 最後にタプルを解放し、テーブルを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • table_open / table_close
  • SearchSysCacheCopy1
  • CatalogTupleUpdate
  • heap_freetuple

関連データ構造

  • pg_partitioned_table (システムカタログ)
  • Form_pg_partitioned_table (カタログタプルの構造体)

失敗モードと境界条件

  • parentId が pg_partitioned_table に存在しない場合、キャッシュルックアップ失敗としてエラー終了します。

保守担当者向けメモ

  • 本関数は pg_partitioned_table の partdefid を直接更新します。デフォルトパーティションの追加や削除に伴う整合性チェックは、呼び出し側のコマンド処理（tablecmds.c 等）で行う必要があります。
