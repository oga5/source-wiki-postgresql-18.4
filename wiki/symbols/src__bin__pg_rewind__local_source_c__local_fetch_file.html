local_fetch_file

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/local_source.c:63

目的

ローカルデータディレクトリ内の指定されたファイルを読み込み、その内容をメモリ上に取得します。

シグネチャまたは形状

static char *local_fetch_file(rewind_source *source, const char *path, size_t *filesize);

入力と出力

  • **入力**:
  • source: rewind_source 構造体へのポインタ（local_source としてキャストして使用）。
  • path: 読み込むファイルの相対パス。
  • filesize: 読み込んだファイルのサイズを格納するための出力用ポインタ。
  • **出力**:
  • 読み込んだファイル内容を保持するメモリ領域へのポインタ（char *）。

前提条件と不変条件

  • source 引数は local_source 型の構造体として初期化されている必要があります。
  • path は source が保持するデータディレクトリ (datadir) からの相対パスとして有効である必要があります。

アルゴリズムまたは振る舞い

1. source を local_source 型にキャストし、保持されているデータディレクトリパスを取得します。 2. slurpFile 関数を呼び出し、データディレクトリ内の指定パスからファイル全体を読み込みます。 3. 読み込んだファイルサイズを filesize に格納し、ファイル内容のバッファを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: init_local_source 関数内で、rewind_source インターフェースの関数ポインタとして登録されます。
  • **呼び出し先**: slurpFile 関数を呼び出します。

関連データ構造

  • rewind_source: 抽象化されたソースインターフェース。
  • local_source: ローカルファイルシステム上のデータディレクトリパスを保持する構造体。

失敗モードと境界条件

  • slurpFile が失敗した場合（ファイルが存在しない、読み込み権限がない等）の挙動は slurpFile の実装に依存します。

保守担当者向けメモ

  • 本関数は rewind_source インターフェースの一部として実装されています。
  • local_source.c 内の他の関数（local_queue_fetch_file 等）とは異なり、ファイル全体をメモリに読み込む同期的な処理を行います。
