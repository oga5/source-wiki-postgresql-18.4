DirectFunctionCall8Coll

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/fmgr.c:980

目的

8つの引数を取る関数を、FmgrInfo 構造体を介さずに直接呼び出すためのオーバーヘッドを最小化したラッパー関数です。

シグネチャまたは形状

Datum DirectFunctionCall8Coll(
    PGFunction func, 
    Oid collation, 
    Datum arg1, Datum arg2, Datum arg3, Datum arg4, 
    Datum arg5, Datum arg6, Datum arg7, Datum arg8
)

入力と出力

  • **入力**:
  • func: 呼び出す関数のポインタ。
  • collation: 照合順序（Collation）のOID。
  • arg1〜arg8: 関数に渡す8つの Datum 型引数。
  • **出力**:
  • 関数の実行結果を Datum 型で返す。

前提条件と不変条件

  • 呼び出される関数は、NULL値を返さないことが期待されます。
  • 引数はすべて非NULLとして扱われます（isnull = false が設定されます）。

アルゴリズムまたは振る舞い

1. LOCAL_FCINFO(fcinfo, 8) マクロを使用して、スタック上に8引数用の FunctionCallInfoBaseData を確保します。 2. InitFunctionCallInfoData を呼び出し、照合順序と引数情報を初期化します。 3. 8つの引数すべてを fcinfo->args 配列に格納し、isnull フラグを false に設定します。 4. 関数ポインタ func を fcinfo を引数として実行します。 5. 実行後、fcinfo->isnull が真であれば elog(ERROR) を発生させます。 6. 正常終了時に result を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/fmgr.h:697 にてマクロ経由で呼び出されています。
  • **呼び出し先**: 引数として渡された PGFunction 型の関数ポインタを直接実行します。

関連データ構造

  • FunctionCallInfoBaseData: 実行時の引数やコンテキストを保持する構造体。
  • Datum: PostgreSQLの汎用データ型。
  • PGFunction: 関数ポインタの型定義。

失敗モードと境界条件

  • 呼び出された関数が内部でNULLを返した場合、elog(ERROR) によりエラーが発生します。

保守担当者向けメモ

  • 本関数はパフォーマンスが重要なパスで使用されます。
  • ポリモーフィック関数など、FmgrInfo の初期化が必要な複雑な関数には使用できません。
  • 引数の数が固定されているため、引数が増減する場合は対応する別の DirectFunctionCall*Coll を使用してください。
