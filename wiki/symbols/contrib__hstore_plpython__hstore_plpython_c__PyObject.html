PyObject

定義

  • 種別: typedef (Python C API の不透明型)
  • 定義箇所: contrib/hstore_plpython/hstore_plpython.c:14

目的

Python のあらゆるオブジェクトを表現するための汎用的なデータ構造です。hstore_plpython モジュール内では、PostgreSQL の hstore 型と Python の dict 型の間でデータを変換する際の仲介役として使用されます。

シグネチャまたは形状

Python C API によって定義される不透明な構造体です。本モジュール内では、主にポインタ型 (PyObject *) として扱われます。

入力と出力

  • **入力**: Python 側のデータ（辞書、文字列、タプルなど）を指すポインタとして関数に渡されます。
  • **出力**: Python C API 関数（PyDict_SetItem や PyMapping_Items など）の戻り値として生成され、PostgreSQL 側の処理へ渡されます。

前提条件と不変条件

  • Python インタプリタが初期化されている環境で利用される必要があります。
  • 参照カウント（Reference Counting）の管理が必須であり、不要になったオブジェクトには Py_DECREF を呼び出す必要があります。

アルゴリズムまたは振る舞い

  • hstore_to_plpython においては、hstore の各要素を Python の文字列オブジェクトに変換し、PyDict_SetItem を用いて辞書に格納する際の要素として振る舞います。
  • plpython_to_hstore においては、PyMapping_Items を介して Python 辞書のキーと値のペアを抽出するためのコンテナとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: hstore_to_plpython および plpython_to_hstore 関数内で直接的に参照・操作されます。
  • **呼び出し先**: Python C API の各種関数（PyDict_SetItem, PyMapping_Check, PyMapping_Items 等）が、この型を引数として受け取ります。

関連データ構造

  • Py_ssize_t: Python オブジェクトのサイズやインデックスを扱うために併用されます。
  • Datum: PostgreSQL 側のデータ表現であり、PyObject との間で相互変換が行われます。

失敗モードと境界条件

  • **メモリ不足**: Python オブジェクトの生成時にメモリが不足した場合、Python C API は NULL を返します。これを確認せずにポインタを逆参照するとクラッシュします。
  • **型不一致**: PyMapping_Check 等で確認せずに非マッピングオブジェクトを操作した場合、Python 側の例外が発生します。

保守担当者向けメモ

  • 本シンボルは Python C API の一部であり、hstore_plpython.c 内で再定義されているわけではありません。
  • PG_TRY/PG_FINALLY ブロック内での Py_DECREF の呼び出しを徹底し、エラー発生時のメモリリークを防止してください。
