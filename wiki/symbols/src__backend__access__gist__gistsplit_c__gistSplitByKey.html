gistSplitByKey

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistsplit.c:622

目的

GiSTインデックスのページ分割において、マルチカラム（複数列）対応の最適化を行うメインエントリーポイントです。ユーザー定義の picksplit 関数が単一列のみを考慮する場合に、後続の列を使用して「ペナルティなしで移動可能なタプル（don't-care）」を再分配し、インデックスの品質を向上させます。

シグネチャまたは形状

void gistSplitByKey(Relation r, Page page, IndexTuple *itup, int len,
                    GISTSTATE *giststate, GistSplitVector *v, int attno)

入力と出力

  • **入力**:
  • r: 対象のリレーション。
  • page: 分割対象のページ。
  • itup: 分割対象のインデックスタプル配列。
  • len: タプル数。
  • giststate: GiSTの内部状態。
  • v: 分割結果を格納する構造体。
  • attno: 現在処理中の列インデックス（0から開始）。
  • **出力**: なし（v を通じて分割結果を更新）。

前提条件と不変条件

  • attno は0から始まり、再帰的にインクリメントされます。
  • attno == 0 の場合、最終的なユニオンキーの再計算が保証されます。
  • ユーザー定義の picksplit が不適切な分割を行った場合、v->spl_dontcare を用いて再帰的な再分割が行われます。

アルゴリズムまたは振る舞い

1. **初期化**: 指定された attno 列の全タプルから GistEntryVector を構築し、NULL値を特定します。 2. **NULL値の処理**: NULL値のみで構成される場合、次の列へ再帰するか、列がない場合は gistSplitHalf で強制分割します。 3. **NULLと非NULLの分離**: NULL値と非NULL値が混在する場合、これらを別々のページへ分離します。 4. **ユーザー定義分割**: 非NULL値に対して gistUserPicksplit を呼び出します。 5. **再帰的最適化**: 分割が不十分（don't-careタプルが存在）な場合、それらのタプルのみを抽出し、attno + 1 列に対して再帰的に gistSplitByKey を呼び出します。 6. **マージ**: 再帰呼び出しの結果と元の分割結果をマージし、最終的な分割状態を確定します。 7. **最終化**: attno == 0 かつ複数列インデックスの場合、全列のユニオンキーを再計算します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gist.c:1483 および自身（再帰呼び出し）。
  • **呼び出し先**: gistUserPicksplit, gistSplitHalf, gistunionsubkey, gistdentryinit, index_getattr。

関連データ構造

  • GistSplitVector: 分割結果を保持する構造体。
  • GistEntryVector: 各タプルのキー情報を保持する配列。
  • GISTSTATE: GiSTインデックスの操作状態。

失敗モードと境界条件

  • **全列NULL**: attno が最終列に達していない場合は次の列へ、達している場合は gistSplitHalf へフォールバックします。
  • **退化分割**: gistUserPicksplit が全タプルを片側に寄せるような分割を行った場合、v->spl_dontcare がNULLとなり、その分割を破棄して次の列での分割を試みます。

保守担当者向けメモ

  • 本関数は「ペナルティ関数が100%正確ではない」という前提で設計されています。
  • 再帰呼び出し（attno > 0）の際は、呼び出し元がユニオンキーの整合性を管理するため、内部でのユニオンキー計算は省略されます。
  • v->spl_dontcare の再帰処理において、元の分割結果を backupSplit に退避させることで、再帰後のマージを安全に行っています。
