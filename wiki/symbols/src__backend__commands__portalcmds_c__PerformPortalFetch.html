PerformPortalFetch

定義

  • 種別: function
  • 定義箇所: src/backend/commands/portalcmds.c:176

目的

SQLの FETCH および MOVE コマンドを処理し、指定されたポータルからタプルを取得またはカーソル位置を移動させる。

シグネチャまたは形状

void PerformPortalFetch(FetchStmt *stmt, DestReceiver *dest, QueryCompletion *qc)

入力と出力

  • **入力**:
  • stmt: 実行する FETCH または MOVE コマンドの構文解析ツリー。
  • dest: タプルの出力先（DestReceiver）。
  • qc: コマンド完了ステータスを格納する構造体（NULL可）。
  • **出力**: なし（副作用としてポータルの状態更新および qc への書き込みを行う）。

前提条件と不変条件

  • stmt->portalname が空文字列であってはならない（プロトコルレベルの無名ポータルと競合するため）。
  • 指定された名前のポータルが既に存在し、有効であること。

アルゴリズムまたは振る舞い

1. stmt->portalname の妥当性を検証し、空の場合はエラーを報告する。 2. GetPortalByName を使用してポータルを取得し、存在しない場合はエラーを報告する。 3. stmt->ismove が真の場合、出力先 dest を None_Receiver に強制変更する。 4. PortalRunFetch を呼び出し、指定された方向と行数でポータルの実行を進める。 5. qc が提供されている場合、SetQueryCompletion を使用して処理された行数とコマンドタグ（CMDTAG_MOVE または CMDTAG_FETCH）を記録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c:706（直接の呼び出しを確認）
  • **呼び出し先**:
  • GetPortalByName
  • PortalRunFetch
  • SetQueryCompletion
  • ereport（エラー時）

関連データ構造

  • FetchStmt: FETCH/MOVE コマンドの情報を保持。
  • Portal: クエリ実行状態を保持する抽象オブジェクト。
  • DestReceiver: クエリ結果の出力先インターフェース。
  • QueryCompletion: コマンド実行結果のメタデータ。

失敗モードと境界条件

  • **無効なカーソル名**: portalname が空の場合、ERRCODE_INVALID_CURSOR_NAME でエラー。
  • **カーソル不在**: 指定した名前のポータルが見つからない場合、ERRCODE_UNDEFINED_CURSOR でエラー。

保守担当者向けメモ

  • 本関数は MOVE コマンドと FETCH コマンドの両方を処理するため、ismove フラグによる DestReceiver の切り替えロジックが重要です。
  • ポータルのメモリ管理や実行状態の詳細は PortalRunFetch に依存しています。
