out_gistxlogPageDelete

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/gistdesc.c:51

目的

GiSTインデックスのページ削除操作に関連するWALレコードの内容を、人間が読める形式に変換してバッファへ出力します。

シグネチャまたは形状

static void out_gistxlogPageDelete(StringInfo buf, gistxlogPageDelete *xlrec)

入力と出力

  • **入力**:
  • buf: 出力先の StringInfo バッファ。
  • xlrec: 解析対象の gistxlogPageDelete 構造体へのポインタ。
  • **出力**: なし（buf に文字列が追記されます）。

前提条件と不変条件

  • xlrec は有効な gistxlogPageDelete 構造体を指している必要があります。
  • buf は初期化済みの StringInfo である必要があります。

アルゴリズムまたは振る舞い

1. xlrec->deleteXid からエポックとトランザクションIDを抽出します。 2. 抽出した情報と xlrec->downlinkOffset を整形し、以下の形式で buf に追加します。 deleteXid <epoch>:<xid>; downlink <downlinkOffset>

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/rmgrdesc/gistdesc.c 内のメインのデコード処理（gist_desc 関数等）から呼び出されます。
  • **呼び出し先**: appendStringInfo（lib/stringinfo.h）、EpochFromFullTransactionId、XidFromFullTransactionId。

関連データ構造

  • gistxlogPageDelete: GiSTのページ削除WALレコードを保持する構造体。
  • StringInfo: PostgreSQLの可変長文字列バッファ。

失敗モードと境界条件

  • buf の容量が不足した場合、appendStringInfo の内部仕様に従いメモリ拡張が行われます。
  • xlrec が NULL の場合、プログラムはクラッシュします。

保守担当者向けメモ

  • 本関数はGiSTのWALレコード表示用ルーチンの一部です。新しいフィールドが gistxlogPageDelete 構造体に追加された場合は、本関数の出力フォーマットも更新してください。
