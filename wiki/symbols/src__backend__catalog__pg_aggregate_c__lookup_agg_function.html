lookup_agg_function

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_aggregate.c:826

目的

集約関数のサポート関数（遷移関数、最終関数、結合関数等）をシステムカタログから検索し、そのシグネチャと型の一貫性を検証して Oid を取得する。

シグネチャまたは形状

Oid lookup_agg_function(List *fnName, int nargs, Oid *input_types, Oid variadicArgType, Oid *rettype)

入力と出力

  • **入力**:
  • fnName: 関数名（リスト形式）。
  • nargs: 引数の数。
  • input_types: 引数の型OID配列。
  • variadicArgType: 可変長引数の型（ANYOID 等）。
  • rettype: 戻り値の型（出力用ポインタ）。
  • **出力**:
  • 成功時: 検索された関数の Oid を返す。
  • 失敗時: エラーをスローする。

前提条件と不変条件

  • 検索対象の関数は、集合を返す関数（retset）であってはならない。
  • 集約が VARIADIC ANY を要求する場合、対象関数も同様の宣言である必要がある。
  • 実行時に型変換（coercion）を必要とする関数は許可されない。
  • 呼び出し元（集約作成者）は、対象関数に対する ACL_EXECUTE 権限を保持していなければならない。

アルゴリズムまたは振る舞い

1. func_get_detail を呼び出し、カタログから関数情報を取得する。 2. 関数が存在しない、または集合を返す場合はエラーを報告する。 3. variadicArgType が ANYOID の場合、対象関数の可変長引数型が一致するか検証する。 4. enforce_generic_type_consistency を呼び出し、多相型の一貫性を強制し、戻り値の型を確定させる。 5. 引数ごとに IsBinaryCoercible を確認し、実行時の型変換が必要な場合はエラーを報告する。 6. object_aclcheck を使用して、実行権限を確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/pg_aggregate.c 内の AggregateCreate 処理において、transfn, mtransfn, minvtransfn, finalfn, combinefn, serialfn, deserialfn, mfinalfn の各検索時に呼び出される。
  • **呼び出し先**: func_get_detail, ereport, func_signature_string, enforce_generic_type_consistency, IsBinaryCoercible, object_aclcheck, get_func_name。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。
  • List: 関数名を保持するリスト構造。

失敗モードと境界条件

  • ERRCODE_UNDEFINED_FUNCTION: 関数が見つからない場合。
  • ERRCODE_DATATYPE_MISMATCH: 集合を返す関数が指定された場合、または可変長引数の型が不一致の場合。
  • ERRCODE_DATATYPE_MISMATCH: 実行時の型変換が必要な場合。
  • 権限不足: aclcheck_error によるエラー報告。

保守担当者向けメモ

  • 本関数は nodeAgg.c が実行時に型変換を処理できないという制約に基づき、厳格な型チェックを行っている。
  • 多相型の解決には enforce_generic_type_consistency を利用しているため、型解決ロジックの変更時はこの挙動に注意すること。
