CheckLogrotateSignal

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/syslogger.c:1573

目的

ログローテーションを要求するシグナルファイル（LOGROTATE_SIGNAL_FILE）の存在を確認し、手動ローテーションのトリガーを判定する。

シグネチャまたは形状

bool CheckLogrotateSignal(void)

入力と出力

  • **入力**: なし（内部で LOGROTATE_SIGNAL_FILE を参照）
  • **出力**:
  • true: シグナルファイルが存在する場合
  • false: シグナルファイルが存在しない場合

前提条件と不変条件

  • LOGROTATE_SIGNAL_FILE が定義されている環境であること。
  • 呼び出し元は、この関数の戻り値に基づいてログローテーション処理を実行する責任を持つ。

アルゴリズムまたは振る舞い

1. stat() システムコールを使用して、LOGROTATE_SIGNAL_FILE の情報を取得する。 2. stat() が成功（戻り値が 0）した場合、ファイルが存在すると判断して true を返す。 3. それ以外の場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/postmaster.c:3753
  • **呼び出し先**:
  • stat() (標準ライブラリ関数)

関連データ構造

  • struct stat: ファイルの状態情報を格納するために使用。

失敗モードと境界条件

  • stat() がファイル非存在以外のエラー（権限不足など）を返した場合、ファイルが存在しないものとして false を返す。

保守担当者向けメモ

  • 本関数はファイルシステムの存在確認のみを行う。シグナルファイルの削除は RemoveLogrotateSignalFiles が担当する。
  • LOGROTATE_SIGNAL_FILE の定義場所については、syslogger.h および関連するヘッダーを参照すること。
