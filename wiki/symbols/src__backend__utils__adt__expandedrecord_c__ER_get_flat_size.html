ER_get_flat_size

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/expandedrecord.c:651

目的

展開されたレコード（ExpandedRecordHeader）を物理的なタプル形式へ変換する際に必要なバイトサイズを計算し、結果をキャッシュする。

シグネチャまたは形状

static Size ER_get_flat_size(ExpandedObjectHeader *eohptr)

入力と出力

  • **入力**: ExpandedObjectHeader *eohptr（展開されたレコードのヘッダーへのポインタ）。
  • **出力**: Size（フラット化されたタプルに必要な合計バイト数）。

前提条件と不変条件

  • erh->er_magic が ER_MAGIC と一致していること。
  • 匿名レコード型の場合、er_typmod が適切に設定されていること。
  • 複合データ型は外部ストレージ（TOAST）にある値を含んではならない（関数内でインライン化される）。

アルゴリズムまたは振る舞い

1. **型情報の補完**: 匿名レコード型の場合、タプル記述子から typmod を取得・設定する。 2. **キャッシュ確認**: フラット化された値が有効かつ外部データを含まない場合、または既に flat_size が計算済みの場合はその値を返す。 3. **分解**: フィールド値が展開されていない場合、deconstruct_expanded_record を呼び出して分解する。 4. **TOAST処理**: 外部ストレージにあるフィールドが存在する場合、expanded_record_set_field_internal を用いてインライン化（detoast）を行う。 5. **サイズ計算**:

  • ヘッダーサイズ（HeapTupleHeaderData）を算出。
  • NULLビットマップが必要な場合は加算。
  • heap_compute_data_size を用いてデータ本体のサイズを算出。

    • 6. **キャッシュ更新**: 計算結果を erh->flat_size, erh->data_len, erh->hoff, erh->hasnull に保存し、値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 展開オブジェクトプロトコルの一部として、ER_methods テーブル経由で呼び出される（直接の呼び出し元詳細はソースコードの静的解析からは特定不可）。
  • **呼び出し先**:
  • expanded_record_get_tupdesc
  • assign_record_type_typmod
  • deconstruct_expanded_record
  • expanded_record_set_field_internal
  • heap_compute_data_size

関連データ構造

  • ExpandedRecordHeader: レコードのメタデータとフィールド値を保持するメイン構造体。
  • TupleDesc: タプルの構造定義。
  • HeapTupleHeaderData: 物理タプルのヘッダー構造。

失敗モードと境界条件

  • er_magic の不一致により Assert が失敗する可能性がある。
  • 外部ストレージ（TOAST）のインライン化処理において、メモリ不足やデータ変換エラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は ER_methods を通じて呼び出されるプロトコル関数である。
  • ER_FLAG_HAVE_EXTERNAL フラグのクリアは、ER_flatten_into が高速パスを誤って選択しないように制御されている。
  • 計算結果は erh 構造体にキャッシュされるため、レコードの内容が変更された場合はこれらのキャッシュフラグが適切にリセットされている必要がある。
