testrightleft

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_rbtree/test_rbtree.c:203

目的

PostgreSQL の赤黒木ライブラリにおける、降順（右から左）の走査機能が正しく動作することを検証する。

シグネチャまたは形状

static void testrightleft(int size)

入力と出力

  • **入力**: size (int) - 構築する赤黒木のノード数。
  • **出力**: なし。検証失敗時には elog(ERROR, ...) を通じてエラーを報告する。

前提条件と不変条件

  • create_int_rbtree() により、有効な赤黒木が初期化されていること。
  • rbt_populate により、木に size 個のノードが挿入されていること。
  • 走査結果は、キー値が降順（大きい順）に並んでいる必要がある。

アルゴリズムまたは振る舞い

1. 空の木に対して rbt_begin_iterate を呼び出し、要素が取得されないことを確認する。 2. rbt_populate を使用して、1 から size までの整数キーを持つノードを木に挿入する。 3. rbt_begin_iterate に RightLeftWalk を指定し、イテレータを初期化する。 4. rbt_iterate をループで呼び出し、以下の不変条件を検証する。

  • 各ノードのキーが直前のノードのキーより小さいこと（降順の維持）。

    • 5. 走査終了後、最後のキーが 0 であること、および走査した要素数が size と一致することを確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_rbtree/test_rbtree.c:510（test_rb_tree 関数内から呼び出される）。
  • **呼び出し先**:
  • create_int_rbtree
  • rbt_begin_iterate
  • rbt_populate
  • rbt_iterate
  • elog

関連データ構造

  • RBTree: 検証対象の赤黒木構造体。
  • RBTreeIterator: 走査に使用するイテレータ。
  • IntRBTreeNode: テスト用の整数キーを持つノード構造体。

失敗モードと境界条件

  • **空の木**: 走査時に要素が返されるとエラーとなる。
  • **順序違反**: 走査結果が降順でない場合、エラーとなる。
  • **走査不完全**: 最後のキーが 0 でない、または走査した要素数が size と一致しない場合、エラーとなる。

保守担当者向けメモ

  • 本関数は RightLeftWalk の正当性のみを検証する。昇順の検証については testleftright を参照のこと。
  • 乱数生成器の状態に依存せず、常に 1 から size までの連続した値でテストを行う。
