ExecEndResult

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeResult.c:239

目的

Result ノードの実行終了処理を行い、関連するリソースを解放する。

シグネチャまたは形状

void ExecEndResult(ResultState *node)

入力と出力

  • **入力**: ResultState *node （終了対象となる Result ノードの実行時状態）
  • **出力**: なし（void）

前提条件と不変条件

  • node は ExecInitResult によって初期化済みであること。
  • outerPlanState(node) を介して子ノードの状態にアクセス可能であること。

アルゴリズムまたは振る舞い

1. outerPlanState(node) を呼び出し、Result ノードが保持する子プランの状態を取得する。 2. ExecEndNode を呼び出し、子ノードの終了処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:589
  • **呼び出し先**: ExecEndNode（子ノードの終了処理）

関連データ構造

  • ResultState: Result ノードの実行時状態を保持する構造体。

失敗モードと境界条件

  • 子ノードが NULL の場合、ExecEndNode の仕様に依存する（通常は安全に処理される）。

保守担当者向けメモ

  • 本関数は Result ノード固有の終了処理を担うが、現状では子ノードの終了処理を委譲するのみである。
  • Result ノード自体が追加のリソース（メモリコンテキストや一時的なバッファ等）を保持するように拡張された場合、本関数内でそれらの解放処理を追加する必要がある。
