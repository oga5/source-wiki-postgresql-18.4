Walfile

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_basebackup/walmethods.h:17

目的

WAL（Write-Ahead Log）データの書き込み操作を抽象化するための基底構造体。異なる書き込み手法（ファイルシステム、ディレクトリ等）を統一的に扱うためのインターフェースを提供します。

シグネチャまたは形状

typedef struct {
    WalWriteMethod *wwmethod;
    pgoff_t        currpos;
    char           *pathname;
    /* 構造体の末尾にメソッド固有のフィールドが続く */
} Walfile;

入力と出力

  • 入力: WalWriteMethod を介して初期化されます。
  • 出力: currpos により現在の書き込み位置を保持し、pathname により対象ファイルパスを管理します。

前提条件と不変条件

  • 各 WalWriteMethod は、本構造体を自身の拡張構造体の最初のメンバーとして埋め込む必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、具体的な書き込み処理（write, sync, close 等）を WalWriteMethod が提供する関数ポインタへ委譲するためのヘッダーとして機能します。
  • currpos は書き込み操作に伴い更新されることが想定されています。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_basebackup/receivelog.c および src/bin/pg_basebackup/walmethods.c 内の各関数。
  • 呼び出し先: WalWriteMethod で定義されたメソッド群（dir_write, dir_sync, dir_close 等）。

関連データ構造

  • WalWriteMethod: 書き込み手法を定義する構造体。
  • WalCloseMethod: dir_close 等で使用される終了処理の列挙型。

失敗モードと境界条件

  • 構造体のメモリレイアウトが正しく継承されていない場合、メソッド固有フィールドへのアクセス時に未定義の動作が発生します。

保守担当者向けメモ

  • 本構造体は継承を模した設計となっているため、メンバーの追加や順序変更を行う際は、派生構造体との整合性に注意してください。
