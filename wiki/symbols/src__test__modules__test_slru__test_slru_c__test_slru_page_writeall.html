test_slru_page_writeall

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_slru/test_slru.c:84

目的

テスト用SLRU（Simple Least Recently Used）キャッシュ内の全ダーティページをディスクへ書き出すためのSQLインターフェース関数です。

シグネチャまたは形状

Datum test_slru_page_writeall(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL経由で呼び出される標準的なPostgreSQL関数引数）。
  • **出力**: void（PG_RETURN_VOID() を介して戻り値を返さない）。

前提条件と不変条件

  • 本関数は shared_preload_libraries でロードされた環境下で実行される必要があります。
  • TestSlruCtl が SimpleLruInit によって初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. TestSlruCtl（テスト用SLRU制御構造体）を引数として SimpleLruWriteAll を呼び出します。 2. SimpleLruWriteAll の第2引数に true を渡し、ダーティなバッファをすべてディスクへフラッシュします。 3. 処理完了後、PG_RETURN_VOID() を実行して制御を戻します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: SimpleLruWriteAll（access/slru.h で定義されたPostgreSQLコアのSLRU API）。

関連データ構造

  • TestSlruCtl: テストモジュール内で定義された SlruCtlData 型のグローバル変数。SLRUの共有メモリ状態を管理します。

失敗モードと境界条件

  • 共有メモリが初期化されていない状態で呼び出された場合、PostgreSQLのバックエンドプロセスが異常終了する可能性があります。
  • ディスク書き込み時にI/Oエラーが発生した場合、SimpleLruWriteAll 内部でエラーがスローされます。

保守担当者向けメモ

  • 本関数は SimpleLruWriteAll のラッパーであり、テスト目的で強制的に全ページを同期させるために使用されます。
  • SimpleLruWriteAll の第2引数（do_fsync）を true に固定しているため、呼び出し時にはディスクへの物理的な同期が保証されます。
