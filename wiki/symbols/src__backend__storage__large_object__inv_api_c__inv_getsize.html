inv_getsize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/large_object/inv_api.c:339

目的

指定されたラージオブジェクト（LO）の現在の総バイト数を取得します。

シグネチャまたは形状

static uint64 inv_getsize(LargeObjectDesc *obj_desc)

入力と出力

  • **入力**: obj_desc (LargeObjectDesc型へのポインタ) - 対象となるラージオブジェクトの記述子。
  • **出力**: uint64 - ラージオブジェクトの総バイト数。データが存在しない場合は 0 を返します。

前提条件と不変条件

  • obj_desc は有効なポインタである必要があります（Assertによる検証あり）。
  • open_lo_relation() が呼び出され、pg_largeobject テーブルおよびインデックスが利用可能な状態である必要があります。

アルゴリズムまたは振る舞い

1. pg_largeobject テーブルに対して、対象の loid をキーとした順序付きスキャン（systable_beginscan_ordered）を開始します。 2. インデックスが loid と pageno の両方で構成されていることを利用し、BackwardScanDirection（逆方向スキャン）を実行します。 3. 最初に取得されたタプル（＝ページ番号が最大であるページ）を特定します。 4. 該当タプルの pageno と、そのページ内のデータ長（len）から、以下の式で総バイト数を算出します。 lastbyte = (pageno * LOBLKSIZE) + len 5. タプルが一つも存在しない場合、サイズは 0 となります。 6. スキャンを終了し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/large_object/inv_api.c 内の inv_seek 等で参照されている可能性があります（ソース内の直接的な言及あり）。
  • **呼び出し先**:
  • open_lo_relation()
  • systable_beginscan_ordered()
  • systable_getnext_ordered()
  • getdatafield()
  • systable_endscan_ordered()

関連データ構造

  • LargeObjectDesc: ラージオブジェクトの状態を保持する構造体。
  • pg_largeobject: ラージオブジェクトのデータを格納するシステムカタログ。

失敗モードと境界条件

  • pg_largeobject 内のフィールドに NULL が含まれている場合、elog(ERROR) により異常終了します。
  • ラージオブジェクトが空の場合、サイズ 0 が正しく返されます。

保守担当者向けメモ

  • 本関数は pg_largeobject のインデックス構造に強く依存しています。インデックス定義が変更された場合、スキャンロジックの妥当性を再確認してください。
  • getdatafield によって返される datafield が動的に割り当てられた場合、pfree によるメモリ解放が確実に行われるよう設計されています。
