socket_close

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqcomm.c:348

目的

バックエンド終了時に libpq の通信状態をクリーンアップし、セキュアな通信層（SSL/GSSAPI）を適切に終了させる。

シグネチャまたは形状

static void socket_close(int code, Datum arg)

入力と出力

  • **入力**:
  • code: on_proc_exit から渡される終了コード。
  • arg: on_proc_exit から渡される引数。
  • **出力**: なし（void）。

前提条件と不変条件

  • MyProcPort が NULL の場合、スタンドアロンバックエンドとみなされ、何も実行されない。
  • MyProcPort が有効な場合、secure_close を呼び出して SSL 層を終了させる。

アルゴリズムまたは振る舞い

1. MyProcPort が存在するか確認する。 2. ENABLE_GSS が定義されている場合、GSSAPI コンテキスト (ctx) と認証情報 (cred) を解放する。 3. secure_close(MyProcPort) を呼び出し、SSL 層を安全に終了させる。 4. ソケットの明示的な close() は行わず、プロセス終了までソケットを維持する（クライアント側での同期的なクローズ検知を可能にするため）。 5. MyProcPort->sock を PGINVALID_SOCKET に設定し、以降の I/O を禁止する。

呼び出し元と呼び出し先

  • **呼び出し元**: on_proc_exit(socket_close, 0) を通じて、バックエンドプロセスの終了時に呼び出される。
  • **呼び出し先**:
  • gss_delete_sec_context (GSSAPI 有効時)
  • gss_release_cred (GSSAPI 有効時)
  • secure_close

関連データ構造

  • Port (グローバル変数 MyProcPort を介して参照)
  • PGINVALID_SOCKET

失敗モードと境界条件

  • MyProcPort が NULL の場合は安全に無視される。
  • GSSAPI や SSL の終了処理において、コンテキストや認証情報が初期化されていない場合は、それぞれの解放関数が適切にガードされている（GSS_C_NO_CONTEXT 等のチェック）。

保守担当者向けメモ

  • 本関数は on_proc_exit に登録されるため、バックエンド終了時のクリーンアップ順序に依存する。
  • ソケットを明示的に close() しない設計は、クライアントが接続終了を確実に検知できるようにするための意図的な仕様である。
