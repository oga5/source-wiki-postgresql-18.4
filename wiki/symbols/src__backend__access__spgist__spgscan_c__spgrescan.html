spgrescan

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgscan.c:379

目的

SP-GiST インデックススキャンを再初期化し、新しい検索キーおよび順序付け条件を適用します。

シグネチャまたは形状

void spgrescan(IndexScanDesc scan, ScanKey scankey, int nscankeys, ScanKey orderbys, int norderbys)

入力と出力

  • **入力**:
  • scan: インデックススキャンの状態記述子。
  • scankey: 検索条件の配列。
  • nscankeys: 検索条件の数。
  • orderbys: 順序付け（Order-by）条件の配列。
  • norderbys: 順序付け条件の数。
  • **出力**: なし（副作用として scan 構造体内部の状態を更新）。

前提条件と不変条件

  • scan->opaque は SpGistScanOpaque 型としてキャスト可能であること。
  • scan->keyData および scan->orderByData は、それぞれ nscankeys および norderbys を保持するのに十分なサイズが確保されていること。

アルゴリズムまたは振る舞い

1. **検索キーのコピー**: scankey が提供されている場合、scan->keyData にコピーします。 2. **順序付け条件の初期化**: orderbys が提供されている場合、scan->orderByData にコピーし、各演算子の戻り値型を get_func_rettype で取得して so->orderByTypes に格納します。 3. **キーの事前処理**: spgPrepareScanKeys(scan) を呼び出し、スキャン用の内部表現を構築します。 4. **キューの初期化**: resetSpGistScanOpaque(so) を呼び出し、走査用のキューをリセットします。 5. **統計情報の更新**: pgstat_count_index_scan を呼び出し、実行統計を記録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgutils.c: amroutine->amrescan に登録。
  • **呼び出し先**:
  • spgPrepareScanKeys
  • resetSpGistScanOpaque
  • get_func_rettype
  • pgstat_count_index_scan

関連データ構造

  • IndexScanDesc: スキャン全体の制御構造体。
  • SpGistScanOpaque: SP-GiST 固有のスキャン状態（scan->opaque）。
  • ScanKey: 検索条件を保持するデータ構造。

失敗モードと境界条件

  • scankey や orderbys が NULL の場合、それぞれのコピー処理はスキップされます。
  • 順序付け演算子の戻り値型が不明な場合、get_func_rettype の挙動に依存します。

保守担当者向けメモ

  • 本関数は amrescan インターフェースとして呼び出されます。
  • orderByTypes の設定は、将来的な距離計算の型判定に利用されます。
  • spgPrepareScanKeys および resetSpGistScanOpaque がスキャン状態の整合性を維持する中心的な役割を担っています。
