base_yylex_location

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/parser.c:238

目的

スキャナ（base_yylex）をラップし、取得したトークンに対して適切な位置情報（base_yylloc）を生成・補完する補助関数です。

シグネチャまたは形状

static int base_yylex_location(void)

入力と出力

  • **入力**: なし（内部で base_yylex() を呼び出し、グローバル変数 base_yylval および base_yytext を参照）。
  • **出力**: int 型のトークン識別子。

前提条件と不変条件

  • base_yylex() が呼び出し可能であること。
  • base_yylloc が位置情報を格納するために利用可能であること。
  • 文字列リテラル以外のトークンについては、base_yylloc に格納される文字列が小文字に正規化されること。

アルゴリズムまたは振る舞い

1. base_yylex() を呼び出し、トークンを取得する。 2. トークンの種類を判定する。

  • Op, IDENT, SCONST 等の特定のトークンの場合、base_yylval.str を複製して base_yylloc に設定する。
  • それ以外のトークンの場合、base_yytext を複製して base_yylloc に設定し、ASCII文字範囲（'A'-'Z'）を小文字に変換する。

    • 3. 取得したトークンを戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/parser.c 内の filtered_base_yylex 等から呼び出される。
  • **呼び出し先**:
  • base_yylex()（スキャナ）
  • loc_strdup()（メモリ確保と文字列複製）

関連データ構造

  • base_yylloc: トークンの位置情報を保持するグローバル変数。
  • base_yylval: トークンの値を保持する共用体。
  • base_yytext: 現在のトークンのテキスト表現。

失敗モードと境界条件

  • loc_strdup() がメモリ不足により失敗した場合の挙動については、本ソース内では明示されていません。
  • ASCII範囲外の文字が含まれる場合、小文字変換の対象外となります。

保守担当者向けメモ

  • 本関数は yylloc の生成を担っています。トークンの種類によって base_yylval.str を使うか base_yytext を使うかが分岐しているため、新しいトークンタイプを追加する際は switch 文のケース追加を検討してください。
  • 小文字変換は ASCII 限定である点に注意してください。
