CheckEnableRlsResult

定義

  • 種別: enum
  • 定義箇所: src/include/utils/rls.h:41

目的

特定のテーブルに対して行レベルセキュリティ（RLS）を適用すべきかどうか、およびその判定結果がプランキャッシュの無効化を必要とするかを表現するために使用されます。

シグネチャまたは形状

enum CheckEnableRlsResult
{
    RLS_NONE,
    RLS_NONE_ENV,
    RLS_ENABLED,
};

入力と出力

本シンボルは列挙型であり、関数 check_enable_rls の戻り値として使用されます。

前提条件と不変条件

  • RLS_NONE_ENV および RLS_ENABLED が返された場合、プランキャッシュの無効化が必要です。
  • RLS_NONE が返された場合、プランキャッシュの無効化は不要です。

アルゴリズムまたは振る舞い

  • RLS_NONE: RLS は適用されません。
  • RLS_NONE_ENV: 現時点では RLS は適用されませんが、ユーザーの変更や環境変数の変化により適用対象となる可能性がある状態です。
  • RLS_ENABLED: RLS が適用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/misc/rls.c 内の check_enable_rls 関数がこの列挙値を生成します。
  • 呼び出し先: 本シンボル自体はデータ型であるため、呼び出し先はありません。

関連データ構造

  • check_enable_rls(Oid relid, Oid checkAsUser, bool noError): この列挙型を返す主要な関数です。

失敗モードと境界条件

  • check_enable_rls の noError 引数が false の場合、エラー発生時に例外が投げられる可能性があります。true の場合はエラー時でも RLS_ENABLED を返す仕様となっています。

保守担当者向けメモ

  • RLS_NONE_ENV と RLS_ENABLED の両方がプランキャッシュの無効化をトリガーする点に注意してください。これは、環境の変化が RLS の適用有無を左右するためです。
