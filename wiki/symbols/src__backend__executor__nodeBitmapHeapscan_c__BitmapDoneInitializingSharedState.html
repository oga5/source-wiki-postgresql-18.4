BitmapDoneInitializingSharedState

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapHeapscan.c:180

目的

並列ビットマップヒープスキャンにおいて、共有状態の初期化が完了したことを通知し、待機中のワーカープロセスを解放する。

シグネチャまたは形状

static inline void BitmapDoneInitializingSharedState(ParallelBitmapHeapState *pstate)

入力と出力

  • **入力**: ParallelBitmapHeapState *pstate - 並列実行時の共有状態を保持する構造体へのポインタ。
  • **出力**: なし（副作用として共有メモリの状態を更新）。

前提条件と不変条件

  • pstate は有効な共有メモリ領域を指している必要がある。
  • 本関数呼び出しにより、pstate->state が BM_FINISHED に設定される。

アルゴリズムまたは振る舞い

1. pstate->mutex を使用してスピンロックを獲得する。 2. pstate->state を BM_FINISHED に更新する。 3. スピンロックを解放する。 4. pstate->cv（条件変数）に対して ConditionVariableBroadcast を実行し、初期化完了を待機している全ワーカープロセスを起床させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeBitmapHeapscan.c 内の並列処理初期化ロジック（詳細はソースコード上の直接的な呼び出し箇所を参照）。
  • **呼び出し先**: SpinLockAcquire, SpinLockRelease, ConditionVariableBroadcast（いずれも PostgreSQL 内部の同期プリミティブ）。

関連データ構造

  • ParallelBitmapHeapState: 並列ビットマップスキャンの共有状態を管理する構造体。

失敗モードと境界条件

  • pstate が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • スピンロックの獲得に失敗した場合や、共有メモリが破損している場合は動作が保証されない。

保守担当者向けメモ

  • 本関数は static inline で定義されており、並列実行時の同期制御のクリティカルパスに含まれる。
  • BM_FINISHED 状態への遷移は、ワーカープロセスがビットマップの生成完了を待機するループを終了させるための重要なトリガーである。
