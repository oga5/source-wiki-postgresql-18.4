injection_points_cleanup

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_points.c:222

目的

プロセス終了時に、ローカルに保持されているインジェクションポイントをすべてデタッチし、関連する統計情報を削除してクリーンアップを行う。

シグネチャまたは形状

static void injection_points_cleanup(int code, Datum arg);

入力と出力

  • 入力:
  • code: on_proc_exit コールバックの終了コード。
  • arg: Datum 型の引数（本関数では使用されない）。
  • 出力: なし（void）。

前提条件と不変条件

  • injection_point_local が真である場合、inj_list_local に登録されたすべてのインジェクションポイントが有効な状態であること。
  • 本関数は on_proc_exit 登録を介して呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. injection_point_local が偽であれば、何もせず即座に終了する。 2. inj_list_local リストを走査する。 3. 各要素（インジェクションポイント名）に対して InjectionPointDetach を呼び出し、ポイントを無効化する。 4. pgstat_drop_inj を呼び出し、対応する統計情報を削除する。

呼び出し元と呼び出し先

  • 呼び出し元: before_shmem_exit（src/test/modules/injection_points/injection_points.c:514 にて登録）。
  • 呼び出し先:
  • InjectionPointDetach
  • pgstat_drop_inj

関連データ構造

  • inj_list_local: ローカルに保持されたインジェクションポイント名のリスト。
  • injection_point_local: ローカルのインジェクションポイントの有無を示すフラグ。

失敗モードと境界条件

  • injection_point_local が初期化されていない、または空の場合、関数は安全に早期リターンする。
  • InjectionPointDetach または pgstat_drop_inj が内部でエラーを発生させた場合、PostgreSQLの標準的なエラーハンドリングに従いプロセスが中断される可能性がある。

保守担当者向けメモ

  • 本関数は on_proc_exit コールバックとして登録されるため、直接呼び出すのではなく、プロセス終了時の自動クリーンアップ機構に依存させること。
  • 並列テスト実行時にインジェクションポイントの競合や残留を防ぐための重要なライフサイクル管理機能である。
