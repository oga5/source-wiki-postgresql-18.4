pairingheap_remove

定義

  • 種別: function
  • 定義箇所: src/backend/lib/pairingheap.c:169

目的

ペアリングヒープから指定されたノードを削除し、ヒープ構造を再構築して整合性を維持します。

シグネチャまたは形状

void pairingheap_remove(pairingheap *heap, pairingheap_node *node)

入力と出力

  • **入力**:
  • heap: 操作対象のペアリングヒープへのポインタ。
  • node: 削除対象のノードへのポインタ。
  • **出力**: なし（副作用としてヒープ構造が更新されます）。

前提条件と不変条件

  • node は heap 内に存在している必要があります。
  • 削除対象のノードがルートである場合、内部的に pairingheap_remove_first が呼び出されます。
  • 削除後もヒープのプロパティ（比較関数に基づく順序）が維持される必要があります。

アルゴリズムまたは振る舞い

1. **ルート判定**: 削除対象がルートノードであれば pairingheap_remove_first を実行して終了します。 2. **ポインタの特定**: 削除対象ノードの親または直前の兄弟ノードから、当該ノードを指すポインタ（prev_ptr）を特定します。 3. **子ノードの処理**:

  • 削除対象に子ノードが存在する場合、merge_children を呼び出して子ノード群を単一のサブヒープに統合し、それを削除箇所の後継としてリンクします。
  • 子ノードが存在しない場合、単に兄弟ノードをリンクし直して削除対象を切り離します。

    • 4. **リンクの更新**: 削除対象の前後関係を調整し、ヒープの連結リスト構造を修復します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/reorderbuffer.c
  • src/backend/utils/time/snapmgr.c
  • **呼び出し先**:
  • pairingheap_remove_first
  • merge_children

関連データ構造

  • pairingheap: ヒープ全体を管理する構造体。
  • pairingheap_node: ヒープ内の各要素。first_child, next_sibling, prev_or_parent ポインタを保持します。

失敗モードと境界条件

  • **ルート削除**: 削除対象がルートの場合、pairingheap_remove_first の境界条件に従います。
  • **孤立ノード**: 子ノードを持たないノードの削除は、単なるポインタの付け替えとして処理されます。
  • **アサーション**: *prev_ptr == node の不変条件が満たされない場合、プログラムは異常終了します。

保守担当者向けメモ

  • 本関数はノードのメモリ解放を行いません。ノードのライフサイクル管理は呼び出し側に依存します。
  • merge_children による再構築は「2パス法」で行われ、ヒープの償却計算量を維持する重要なステップです。
