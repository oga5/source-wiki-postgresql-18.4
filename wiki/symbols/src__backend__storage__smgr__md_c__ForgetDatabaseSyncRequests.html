ForgetDatabaseSyncRequests

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/md.c:1582

目的

指定されたデータベースに関連する保留中の fsync および削除（unlink）要求を、同期管理システムから破棄（忘却）させるために呼び出されます。

シグネチャまたは形状

void ForgetDatabaseSyncRequests(Oid dbid)

入力と出力

  • **入力**: dbid (Oid) - 対象とするデータベースの識別子。
  • **出力**: なし（void）。

前提条件と不変条件

  • rlocator の spcOid および relNumber は 0 に設定され、データベース全体を対象とするフィルタとして初期化されます。
  • INIT_MD_FILETAG マクロにより、指定された dbid を持つデータベースレベルの FileTag が構築されます。

アルゴリズムまたは振る舞い

1. RelFileLocator 構造体を初期化し、対象データベースの OID を設定します。 2. INIT_MD_FILETAG を使用して、InvalidForkNumber および InvalidBlockNumber を含むデータベース単位の FileTag を生成します。 3. RegisterSyncRequest を呼び出し、SYNC_FILTER_REQUEST フラグを指定して、当該データベースに関連する保留中の同期要求をフィルタリング・削除するよう依頼します。この際、エラー発生時の再試行フラグは true に設定されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/dbcommands.c (データベース削除やリカバリ処理に関連する箇所)
  • **呼び出し先**:
  • RegisterSyncRequest (同期要求管理システム)

関連データ構造

  • Oid: データベース識別子。
  • RelFileLocator: リレーションの物理的な場所を特定するための構造体。
  • FileTag: 同期要求を識別するためのタグ構造体。

失敗モードと境界条件

  • RegisterSyncRequest 内部でエラーが発生した場合、retryOnError が true であるため、システムは再試行を試みます。

保守担当者向けメモ

  • 本関数はデータベース削除時などに、不要になった保留中の I/O 同期要求をクリーンアップするために不可欠です。
  • SYNC_FILTER_REQUEST を使用しているため、個別のファイルではなくデータベース単位での一括削除が行われる点に注意してください。
