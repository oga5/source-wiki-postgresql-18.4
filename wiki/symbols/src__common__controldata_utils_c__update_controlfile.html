update_controlfile

定義

  • 種別: function
  • 定義箇所: src/common/controldata_utils.c:188

目的

データベースの制御ファイル（global/pg_control）を更新し、整合性を保つためにタイムスタンプの更新、CRCの再計算、およびディスクへの書き込みと同期を行う。

シグネチャまたは形状

void update_controlfile(const char *DataDir, ControlFileData *ControlFile, bool do_sync)

入力と出力

  • DataDir: 制御ファイルが存在するデータディレクトリのパス。
  • ControlFile: 更新対象の制御データ構造体へのポインタ。
  • do_sync: true の場合、書き込み後に fsync を実行して物理的な同期を行う。
  • 出力: なし（成功時は制御ファイルを更新、失敗時はプロセスを終了）。

前提条件と不変条件

  • ControlFile 構造体は、書き込み前に有効なデータで初期化されている必要がある。
  • バックエンド（FRONTEND 未定義）で実行する場合、呼び出し元は適切なロック（ControlFileLock 等）を保持している必要がある。
  • ファイルサイズは PG_CONTROL_FILE_SIZE に固定され、不足分はゼロパディングされる。

アルゴリズムまたは振る舞い

1. ControlFile->time を現在時刻で更新する。 2. ControlFile の内容に基づき、CRCを再計算して ControlFile->crc に格納する。 3. PG_CONTROL_FILE_SIZE バイトのバッファを作成し、構造体データをコピーして残りをゼロで埋める。 4. DataDir 内の global/pg_control を開く。 5. バッファの内容をファイルへ書き込む。 6. do_sync が true の場合、fsync を実行してディスクへフラッシュする。 7. ファイルを閉じる。

  • エラー発生時、バックエンドでは ereport(PANIC, ...)、フロントエンドでは pg_fatal(...) を呼び出し、プロセスを終了させる。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/xlog.c
  • src/bin/pg_basebackup/pg_createsubscriber.c
  • src/bin/pg_checksums/pg_checksums.c
  • src/bin/pg_resetwal/pg_resetwal.c
  • src/bin/pg_rewind/pg_rewind.c
  • 呼び出し先:
  • time(), memset(), memcpy(), snprintf(), open(), write(), fsync(), close()
  • INIT_CRC32C, COMP_CRC32C, FIN_CRC32C (CRC計算)
  • BasicOpenFile (バックエンド時)
  • pgstat_report_wait_start/end (バックエンド時)

関連データ構造

  • ControlFileData: 制御ファイルのデータ構造（catalog/pg_control.h で定義）。

失敗モードと境界条件

  • ファイルオープン失敗、書き込み失敗、fsync 失敗、またはクローズ失敗時に致命的エラーとしてプロセスが終了する。
  • 書き込み時にディスク容量不足が発生した場合、errno が設定されていない場合は ENOSPC とみなして処理される。

保守担当者向けメモ

  • FRONTEND マクロの定義有無により、エラーハンドリングの挙動（ereport vs pg_fatal）が完全に分岐するため、修正時は両方のパスを考慮すること。
  • バックエンド実行時の待機イベント（WAIT_EVENT_CONTROL_FILE_WRITE_UPDATE 等）の報告が適切に行われているか確認すること。
