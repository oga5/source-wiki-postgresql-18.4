Gzip_getc

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_gzip.c:312

目的

GZIP 圧縮されたファイルストリームから 1 バイトを読み込み、呼び出し元に返します。

シグネチャまたは形状

int Gzip_getc(CompressFileHandle *CFH)

入力と出力

  • **入力**: CompressFileHandle *CFH（GZIP 圧縮ファイルの状態を保持する構造体）。
  • **出力**: 読み込んだ 1 バイト（int 型）。エラーまたは EOF の場合は EOF を返します。

前提条件と不変条件

  • CFH->private_data に有効な gzFile ハンドルが格納されている必要があります。
  • 呼び出し前に errno がリセットされます。

アルゴリズムまたは振る舞い

1. CFH->private_data から gzFile ハンドルを取得します。 2. gzgetc() を呼び出し、ストリームから 1 バイトを読み取ります。 3. 戻り値が EOF の場合、gzeof() を使用して真の EOF かエラーかを判定します。

  • gzeof() が偽（エラー）の場合: pg_fatal を呼び出し、エラーメッセージを出力してプロセスを終了します。
  • gzeof() が真（EOF）の場合: pg_fatal を呼び出し、EOF 到達を報告してプロセスを終了します。

    • 4. 正常時は読み取ったバイト値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_gzip.c 内で CFH->getc_func に割り当てられています。
  • **呼び出し先**: zlib ライブラリの gzgetc および gzeof を呼び出します。また、エラー時には pg_backup_utils.h で定義される pg_fatal を呼び出します。

関連データ構造

  • CompressFileHandle: 圧縮ファイルの操作を抽象化するための構造体。private_data メンバを介して gzFile を保持します。

失敗モードと境界条件

  • **EOF 到達**: gzeof が真を返す場合、pg_fatal により異常終了します。
  • **読み込みエラー**: gzgetc が EOF を返し、かつ gzeof が偽である場合、ファイル読み込みエラーとして pg_fatal により異常終了します。

保守担当者向けメモ

  • 本関数は zlib の gzgetc マクロを直接使用せず、関数としてラップすることでライブラリとヘッダー間の構造体レイアウト不一致を回避しています。
  • エラー発生時に pg_fatal を使用して即座に終了するため、呼び出し側でエラーを個別にハンドリングする必要はありません。
