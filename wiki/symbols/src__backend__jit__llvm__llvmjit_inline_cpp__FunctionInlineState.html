FunctionInlineState

定義

  • 種別: struct
  • 定義箇所: src/backend/jit/llvm/llvmjit_inline.cpp:83

目的

JIT コンパイルにおける関数インライン展開の最適化プロセスにおいて、各関数のインライン化コスト、処理状況、および判定結果を管理するために使用されます。

シグネチャまたは形状

typedef struct FunctionInlineState
{
	int costLimit;
	bool processed;
	bool inlined;
	bool allowReconsidering;
} FunctionInlineState;

入力と出力

本構造体はデータ保持用であり、直接的な関数呼び出しの入力・出力は持ちません。各フィールドはインライン化計画の構築過程で更新されます。

前提条件と不変条件

  • costLimit は、インライン化の可否を判断するためのコスト閾値を保持します。
  • processed が true の場合、その関数に対するインライン化の判定が完了していることを示します。
  • inlined は、実際にインライン化が実行されたか否かを示します。

アルゴリズムまたは振る舞い

インライン化の計画構築（llvm_build_inline_plan）において、対象関数の状態を追跡するために利用されます。再帰的なインライン化の過程で、コスト制限の減衰や依存関係の解決状況を記録し、重複した解析を避けるために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/jit/llvm/llvmjit_inline.cpp 内のインライン化計画構築ロジック（llvm_build_inline_plan 等）から参照・更新されます。
  • **呼び出し先**: 本構造体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • FunctionInlineStates: llvm::StringMap<FunctionInlineState> として定義されており、関数名をキーとして複数の FunctionInlineState を管理します。

失敗モードと境界条件

  • 構造体の初期化が不十分な場合（{} によるゼロ初期化漏れなど）、processed や inlined のフラグが意図しない値となり、インライン化の判定ロジックが誤動作する可能性があります。

保守担当者向けメモ

  • 本構造体は llvmjit_inline.cpp 内のローカルな最適化状態を保持するものです。
  • allowReconsidering フラグの挙動については、再帰的なインライン化コスト計算の文脈で利用されているため、変更時はコスト制限の減衰ロジックへの影響を考慮してください。
