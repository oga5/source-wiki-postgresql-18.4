TM_IndexDeleteOp

定義

  • 種別: struct
  • 定義箇所: src/include/access/tableam.h:243

目的

インデックス削除操作の状態を管理し、テーブルからインデックスへの削除要求を追跡するための構造体です。

シグネチャまたは形状

typedef struct TM_IndexDeleteOp
{
    Relation    irel;               /* 対象インデックスリレーション */
    BlockNumber iblknum;            /* インデックスブロック番号（エラー報告用） */
    bool        bottomup;           /* ボトムアップ削除か否か */
    int         bottomupfreespace;  /* ボトムアップ削除の空き容量ターゲット */
    int         ndeltids;           /* 現在の削除対象TID数 */
    TM_IndexDelete *deltids;        /* 削除対象TIDの配列 */
    TM_IndexStatus *status;         /* インデックス状態の配列 */
} TM_IndexDeleteOp;

入力と出力

  • **入力**: 削除対象のインデックスリレーション（irel）および削除方式（bottomup）を指定して初期化されます。
  • **出力**: インデックスAMによって deltids および status 配列が更新され、削除の進捗と結果が記録されます。

前提条件と不変条件

  • irel は有効なインデックスリレーションである必要があります。
  • deltids および status 配列は、インデックスAMによって適切に初期化・管理される必要があります。

アルゴリズムまたは振る舞い

1. 削除操作の初期化時に本構造体が生成されます。 2. heap_index_delete_tuples 等の関数を通じて、テーブルからインデックスへの削除要求が処理されます。 3. インデックスAMは、deltids に格納されたTIDに基づきインデックスエントリを削除し、その結果を status に反映させます。 4. ボトムアップ削除（bottomup が true）の場合、インデックスの空き容量を最適化するための追加処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam.c（heap_index_delete_tuples 等）、src/backend/access/nbtree/nbtdedup.c、src/backend/access/nbtree/nbtinsert.c 等から参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、具体的な削除ロジックはインデックスAM（nbtree 等）の関数へ渡されます。

関連データ構造

  • TM_IndexDelete: 削除対象のTID情報を保持。
  • TM_IndexStatus: 各TIDに対する削除結果の状態を保持。
  • Relation: インデックスリレーションの定義。

失敗モードと境界条件

  • ndeltids が 0 の場合、削除対象が存在しないため処理はスキップされます。
  • インデックスブロック番号（iblknum）はエラー報告に使用されるため、無効なブロック番号が指定された場合、エラーハンドリング時に不正確な情報が出力される可能性があります。

保守担当者向けメモ

  • 本構造体は heapam.c とインデックスAM間のインターフェースとして機能します。フィールドの追加や変更を行う際は、heap_index_delete_tuples および各インデックスAMの削除実装への影響を確認してください。
