citext_cmp

定義

  • 種別: function
  • 定義箇所: contrib/citext/citext.c:109

目的

PostgreSQLの citext 型において、照合順序（Collation）を考慮した大文字・小文字を区別しない大小比較を行うための関数です。

シグネチャまたは形状

Datum citext_cmp(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される2つの text 型引数（left, right）。
  • **出力**: int32 型の比較結果（left < right なら負、left == right なら 0、left > right なら正）。

前提条件と不変条件

  • 入力された text 型データは、必要に応じてTOAST解除されます。
  • 比較処理には PG_GET_COLLATION() で取得される現在の照合順序が使用されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を使用して、入力引数を取得します。 2. 内部関数 citextcmp を呼び出し、正規化（小文字変換）と照合順序に基づいた比較を実行します。 3. PG_FREE_IF_COPY を使用して、コピーされた入力引数のメモリを適切に解放します。 4. PG_RETURN_INT32 を使用して、比較結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(citext_cmp) マクロによる登録（直接的な関数呼び出しの証拠はソース内に限定的です）。
  • **呼び出し先**: citextcmp（内部比較関数）、PG_GETARG_TEXT_PP、PG_GET_COLLATION、PG_FREE_IF_COPY、PG_RETURN_INT32。

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • 入力がNULLの場合、PG_GETARG_TEXT_PP の挙動に従いエラーとなる可能性があります。
  • メモリ不足時には、PG_GETARG_TEXT_PP が例外をスローする可能性があります。

保守担当者向けメモ

  • 本関数は citext 型のB-treeインデックス構築やソート処理の基盤となります。
  • citextcmp 内部で str_tolower が呼び出されるため、パフォーマンスは文字列の長さに依存します。
  • 照合順序の扱いは citextcmp の実装に依存しているため、比較ロジックの変更時は citextcmp を参照してください。
