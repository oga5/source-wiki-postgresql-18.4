ComputeExtStatisticsRows

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/extended_stats.c:261

目的

テーブルに関連付けられたすべての拡張統計オブジェクトを走査し、それらの統計ターゲット（stattarget）に基づいて、ANALYZE 処理で必要となる最適なサンプル行数を計算する。

シグネチャまたは形状

int ComputeExtStatisticsRows(Relation onerel, int natts, VacAttrStats **vacattrstats)

入力と出力

  • **入力**:
  • onerel: 対象テーブルのリレーション。
  • natts: 解析対象の属性数。
  • vacattrstats: 各属性の統計情報配列。
  • **出力**:
  • 戻り値: 計算されたサンプル行数（300 * 最大の統計ターゲット）。

前提条件と不変条件

  • natts が 0 の場合、直ちに 0 を返す。
  • 統計オブジェクトの構築に必要な属性情報が lookup_var_attr_stats で取得できない場合、そのオブジェクトは計算対象から除外される。
  • 内部的に AllocSetContextCreate を使用してメモリコンテキストを生成し、処理終了後に解放する。

アルゴリズムまたは振る舞い

1. 統計オブジェクトの格納用カタログ（pg_statistic_ext）をオープンし、対象リレーションに関連する全エントリを取得する。 2. 各統計オブジェクトについて以下を実行する:

  • lookup_var_attr_stats を呼び出し、統計に必要な属性情報が揃っているか確認する。
  • statext_compute_stattarget を呼び出し、オブジェクト設定および属性設定から統計ターゲットを決定する。
  • 取得したターゲット値のうち、最大値を保持する。

    • 3. 最後に、最大ターゲット値に 300 を乗じた値をサンプル行数として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/analyze.c:518
  • **呼び出し先**:
  • table_open / table_close
  • fetch_statentries_for_relation
  • lookup_var_attr_stats
  • statext_compute_stattarget
  • MemoryContextSwitchTo 等のメモリ管理関数

関連データ構造

  • StatExtEntry: 統計オブジェクトの定義を保持する構造体。
  • VacAttrStats: 属性ごとの統計情報を保持する構造体。
  • Relation: リレーション情報を保持する構造体。

失敗モードと境界条件

  • 対象テーブルに拡張統計オブジェクトが存在しない場合、result は初期値の 0 のままとなり、戻り値は 0 となる。
  • lookup_var_attr_stats が NULL を返す場合、その統計オブジェクトは無視される。

保守担当者向けメモ

  • 本関数は ANALYZE の初期段階で呼び出されるため、計算コストを抑える必要がある。
  • 統計ターゲットの計算ロジック（statext_compute_stattarget）に変更を加える場合は、本関数が返すサンプル行数への影響を考慮すること。
