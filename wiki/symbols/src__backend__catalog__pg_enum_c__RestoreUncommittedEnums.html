RestoreUncommittedEnums

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_enum.c:872

目的

シリアライズされた未コミットの列挙型および列挙値の情報を復元し、現在のトランザクションにおける追跡用ハッシュテーブルを再構築します。

シグネチャまたは形状

void RestoreUncommittedEnums(void *space)

入力と出力

  • **入力**: space (シリアライズされた Oid 配列へのポインタ)
  • **出力**: なし（副作用としてグローバルなハッシュテーブル uncommitted_enum_types および uncommitted_enum_values を初期化・構築する）

前提条件と不変条件

  • uncommitted_enum_types および uncommitted_enum_values が NULL であること（Assert により検証）。
  • space は EstimateUncommittedEnumsSpace および SerializeUncommittedEnums によって生成された有効なデータ構造であること。

アルゴリズムまたは振る舞い

1. space から Oid 配列を読み取ります。 2. 列挙型（types）のリストが空でない場合、init_uncommitted_enum_types() を呼び出し、ハッシュテーブルを初期化して各 Oid を挿入します。 3. 列挙値（values）のリストが空でない場合、init_uncommitted_enum_values() を呼び出し、ハッシュテーブルを初期化して各 Oid を挿入します。 4. 各リストは OidIsValid が偽となるまで順次処理されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/parallel.c:1542
  • **呼び出し先**: init_uncommitted_enum_types, init_uncommitted_enum_values, hash_search

関連データ構造

  • uncommitted_enum_types (ハッシュテーブル)
  • uncommitted_enum_values (ハッシュテーブル)

失敗モードと境界条件

  • space が指すデータが不正な場合、メモリ破壊や予期せぬ動作を引き起こす可能性があります。
  • 既にハッシュテーブルが初期化されている状態で呼び出された場合、Assert によりプロセスが停止します。

保守担当者向けメモ

  • 本関数は並列クエリ実行時など、トランザクション状態をワーカープロセスへ引き継ぐ際に使用されます。
  • space のレイアウトは SerializeUncommittedEnums の実装と密接に結合しているため、シリアライズ形式を変更する際は両方の関数を更新する必要があります。
