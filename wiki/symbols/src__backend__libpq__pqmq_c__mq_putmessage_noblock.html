mq_putmessage_noblock

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqmq.c:198

目的

共有メモリメッセージキュー（shm_mq）を介した非ブロッキングなメッセージ送信を試行するためのインターフェースです。

シグネチャまたは形状

static void mq_putmessage_noblock(char msgtype, const char *s, size_t len)

入力と出力

  • msgtype (char): 送信するメッセージのタイプ識別子。
  • s (const char *): 送信データ本体。
  • len (size_t): 送信データの長さ。
  • 戻り値: なし（常にエラーを発生させるため）。

前提条件と不変条件

  • 本関数は PqCommMethods 構造体の一部として登録されていますが、現時点では呼び出しをサポートしていません。

アルゴリズムまたは振る舞い

1. 呼び出された際、即座に elog(ERROR, ...) を発行し、処理を中断します。 2. 共有メモリキューの基盤である shm_mq は非ブロッキング送信をサポートしていますが、本関数は送信開始から完了までを保証する仕組みが欠如しているため、意図的にエラーを返す設計となっています。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/libpq/pqmq.c 内の PqCommMqMethods 構造体定義（関数ポインタとして参照）。
  • 呼び出し先: elog（PostgreSQLのエラー報告関数）。

関連データ構造

  • PqCommMethods: 通信メソッドを定義する構造体。本関数はこの構造体の putmessage_noblock フィールドに割り当てられています。

失敗モードと境界条件

  • 呼び出し時に必ず ERROR レベルのログを発行し、トランザクションまたはプロセスを異常終了させます。

保守担当者向けメモ

  • 本関数は現在利用できません。将来的に非ブロッキング送信の要件が発生した場合は、送信開始から完了までの状態管理を実装する必要があります。
