pg_SASL_init

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/libpq/fe-auth.c:434

目的

サーバーから送信された AuthenticationSASL メッセージを解析し、クライアントがサポートする最適な SASL 認証メカニズムを選択・初期化します。

シグネチャまたは形状

static int pg_SASL_init(PGconn *conn, int payloadlen, bool *async);

入力と出力

  • **入力**:
  • conn: 接続状態を保持する PGconn 構造体。
  • payloadlen: 認証メッセージのペイロード長。
  • async: 非同期認証の可否を示すフラグへのポインタ。
  • **出力**:
  • 成功時は STATUS_OK（または内部的な成功値）、失敗時はエラーを conn->errorMessage に設定しエラーコードを返します。

前提条件と不変条件

  • 接続が確立されており、サーバーから SASL 認証要求が送信されていること。
  • conn->channel_binding の設定が SSL 状態と整合していること（SSL 未使用時に require が指定されている場合はエラー）。
  • 認証メカニズムの選択後、必要に応じてパスワードが取得可能であること。

アルゴリズムまたは振る舞い

1. **初期チェック**: チャネルバインディングの要件と SSL 状態の整合性を検証し、重複した SASL 要求を排除します。 2. **メカニズムのネゴシエーション**: サーバーが提示するメカニズムリストを順次読み込み、以下の優先順位で選択します。

  • SCRAM-SHA-256-PLUS (チャネルバインディング対応時)
  • SCRAM-SHA-256
  • OAUTHBEARER

    • 3. **ポリシー検証**: require_auth 設定がある場合、選択されたメカニズムが許可されているか確認します。 4. **パスワード準備**: 選択されたメカニズムがパスワードを必要とする場合、conn->connhost または conn->pgpass からパスワードを取得します。 5. **状態遷移**: 選択したメカニズムの構造体（pg_scram_mech 等）を conn->sasl に割り当てます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-auth.c 内の認証ディスパッチャー（pg_fe_sendauth 等から呼び出される）。
  • **呼び出し先**: pqGets（メッセージ読み込み）、libpq_append_conn_error（エラー報告）、initPQExpBuffer 等のユーティリティ関数。

関連データ構造

  • PGconn: 接続状態および認証設定を保持。
  • pg_fe_sasl_mech: 選択された SASL メカニズムの定義。
  • PQExpBufferData: メカニズム名などの文字列バッファ。

失敗モードと境界条件

  • **サポート外**: サーバーが提示したメカニズムにサポート対象が含まれない場合。
  • **設定不整合**: SSL 未使用時にチャネルバインディングが要求されている場合。
  • **認証情報不足**: パスワードが必要なメカニズムでパスワードが提供されていない場合。
  • **メモリ不足**: PQExpBuffer の操作失敗時。

保守担当者向けメモ

  • 本関数は認証メカニズムの「選択」と「初期化」を担当します。実際の認証プロトコル実行（SCRAM 等）は、選択後に conn->sasl に設定されたハンドラへ委譲されます。
  • 新しい SASL メカニズムを追加する場合は、ループ内の条件分岐を拡張してください。
