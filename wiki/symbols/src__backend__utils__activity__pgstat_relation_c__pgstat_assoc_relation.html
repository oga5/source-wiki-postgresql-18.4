pgstat_assoc_relation

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_relation.c:130

目的

リレーション（テーブルやインデックス）のキャッシュエントリと、対応する統計情報保持構造体（PgStat_TableStatus）を紐付けます。

シグネチャまたは形状

void pgstat_assoc_relation(Relation rel)

入力と出力

  • **入力**: Relation rel - 統計情報を紐付ける対象のリレーションキャッシュエントリ。
  • **出力**: なし（副作用として rel->pgstat_info が更新されます）。

前提条件と不変条件

  • rel->pgstat_enabled が真であること。
  • rel->pgstat_info が NULL であること（未紐付け状態であること）。
  • 取得した PgStat_TableStatus エントリの relation フィールドが NULL であること（他のリレーションと重複して紐付いていないこと）。

アルゴリズムまたは振る舞い

1. rel->pgstat_enabled および rel->pgstat_info の状態をアサートします。 2. pgstat_prep_relation_pending を呼び出し、対象リレーションの OID と共有フラグに基づいて統計エントリを取得または作成します。 3. 取得した統計エントリの relation フィールドが空であることを確認します。 4. 統計エントリの relation フィールドに現在の rel を代入し、相互参照を確立します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat_relation.c 内の他関数および src/include/pgstat.h で定義されたマクロ経由で呼び出されます。
  • **呼び出し先**: pgstat_prep_relation_pending。

関連データ構造

  • Relation: リレーションキャッシュエントリ。
  • PgStat_TableStatus: リレーションの統計情報を保持する構造体。

失敗モードと境界条件

  • rel->pgstat_enabled が偽の場合、アサート失敗により異常終了します。
  • すでに pgstat_info が設定されている場合、アサート失敗により異常終了します。
  • 統計エントリが既に別リレーションに紐付いている場合、アサート失敗により異常終了します。

保守担当者向けメモ

  • 本関数はリレーションと統計情報のライフサイクルを同期させる重要な接点です。
  • 統計エントリの重複紐付けを防ぐためのアサートが厳格に行われているため、リレーションのクローズ処理との整合性に注意してください。
