loc_chunk

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/preproc/util.c:118

目的

ECPGプリプロセッサにおいて、構文解析中に使用される一時的なメモリ領域を管理するためのチャンク構造体です。解析サイクルごとのメモリ一括解放を効率化するために使用されます。

シグネチャまたは形状

typedef struct loc_chunk
{
    struct loc_chunk *next;        /* リストの次要素へのポインタ */
    unsigned int chunk_used;       /* data[] 内の未使用領域の開始インデックス */
    unsigned int chunk_avail;      /* data[] 内の残り利用可能バイト数 */
    char         data[FLEXIBLE_ARRAY_MEMBER]; /* 実際のデータ格納領域 */
} loc_chunk;

入力と出力

  • 本構造体はデータ保持用であり、関数ではありません。
  • loc_alloc 関数によって本構造体のインスタンスが生成・管理され、reclaim_local_storage 関数によってリスト全体が解放されます。

前提条件と不変条件

  • loc_chunk は 8192 バイト単位で確保されることが想定されています。
  • 個別の free は行わず、reclaim_local_storage による一括解放を前提とします。
  • data メンバは FLEXIBLE_ARRAY_MEMBER を使用しており、構造体末尾に可変長の領域を保持します。

アルゴリズムまたは振る舞い

  • loc_alloc は loc_chunks リストを走査し、空き容量のあるチャンクを探します。
  • 空きがない場合、新規に loc_chunk を確保してリストの先頭に追加します。
  • reclaim_local_storage はリストを辿り、すべてのチャンクを free してリストを初期化します。

呼び出し元と呼び出し先

  • 呼び出し元: loc_alloc 関数が本構造体のメモリ確保およびリスト管理を行います。
  • 呼び出し先: 本構造体自体はデータ構造であり、関数を呼び出しません。

関連データ構造

  • loc_chunks: loc_chunk 型の静的リストポインタ。
  • LOC_CHUNK_OVERHEAD: offsetof を用いて計算される、data 領域を除いた構造体のヘッダサイズ。

失敗モードと境界条件

  • 非常に大きな文字列を扱う場合、8192 バイトのチャンクサイズ制限により、メモリの断片化や効率低下を招く可能性があります。
  • reclaim_local_storage を呼び出し忘れるとメモリリークが発生します。

保守担当者向けメモ

  • loc_alloc で確保されたメモリは、解析の節目で必ず reclaim_local_storage() が呼ばれていることを確認してください。
  • data メンバへのアクセス時は、chunk_used および chunk_avail の整合性を維持してください。
