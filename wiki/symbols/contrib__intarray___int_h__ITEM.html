ITEM

定義

  • 種別: struct
  • 定義箇所: contrib/intarray/_int.h:140

目的

ブール検索クエリ（QUERYTYPE）を構成する個々の要素（演算子またはオペランド）を保持するための構造体です。

シグネチャまたは形状

typedef struct ITEM
{
	int16		type;
	int16		left;
	int32		val;
} ITEM;

入力と出力

  • **入力**: 構造体の各フィールド（type, left, val）は、クエリの構文解析結果に基づいて設定されます。
  • **出力**: QUERYTYPE 構造体の一部として、ブール検索の評価ロジック（execute 関数など）に渡されます。

前提条件と不変条件

  • QUERYTYPE 構造体内で可変長配列の一部として使用されるため、メモリ確保時には COMPUTESIZE マクロを用いて適切なサイズを計算する必要があります。
  • type フィールドには、クエリの演算子やオペランドを識別するためのコードが格納されます。

アルゴリズムまたは振る舞い

  • QUERYTYPE に格納されたポーランド記法形式のクエリを走査する際、各 ITEM が順次参照されます。
  • _int_bool.c 内の execute 関数や checkcondition_* 系関数において、現在の ITEM が演算子かオペランドかを type で判別し、条件判定を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/intarray/_int_bool.c 内の execute 関数、checkcondition_arr、checkcondition_bit、checkcondition_gin などが、クエリ評価のために ITEM を参照します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • QUERYTYPE: ITEM の配列を保持する可変長構造体。
  • COMPUTESIZE: ITEM の個数から QUERYTYPE のサイズを算出するマクロ。

失敗モードと境界条件

  • QUERYTYPEMAXITEMS マクロにより、メモリ割り当て可能な ITEM の最大数が制限されています。
  • QUERYTYPE のメモリ確保時に COMPUTESIZE を使用しなかった場合、バッファオーバーフローや不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • ITEM は QUERYTYPE の一部としてメモリ上に連続して配置される前提で設計されています。
  • type フィールドに割り当てられるコード値の定義については、_int.h 内の関連コメントを参照してください。
