coerce_type

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_coerce.c:156

目的

式（Node）のデータ型をターゲット型へ変換し、必要に応じて変換ノードを挿入して式ツリーを修飾する。

シグネチャまたは形状

Node *coerce_type(ParseState *pstate, Node *node,
                  Oid inputTypeId, Oid targetTypeId, int32 targetTypeMod,
                  CoercionContext ccontext, CoercionForm cformat, int location)

入力と出力

  • **入力**:
  • pstate: パーサー状態（エラー位置報告に使用）。
  • node: 変換対象の式ノード。
  • inputTypeId, targetTypeId: 変換元および変換先の型OID。
  • targetTypeMod: ターゲット型の型修飾子。
  • ccontext, cformat: 変換の文脈と形式。
  • location: エラー報告用の構文位置。
  • **出力**: 変換後の Node*。変換が不要な場合は元のノードをそのまま返す。

前提条件と不変条件

  • can_coerce_type による変換可否の事前検証が完了していることを前提とする。
  • UNKNOWN 型の定数は、ターゲット型が多相型（ANYELEMENT 等）の場合、そのまま返される。
  • UNKNOWN 型の定数でターゲットが配列や範囲型等の場合、入力関数を呼び出して変換を試みる。

アルゴリズムまたは振る舞い

1. **変換不要判定**: ターゲット型と入力型が同一、またはノードが NULL の場合はそのまま返す。 2. **多相型処理**: ターゲットが ANYOID 等の多相型である場合、変換を行わずノードをそのまま返す。 3. **配列/範囲/列挙型処理**: ターゲットが ANYARRAYOID 等の場合、ドメイン型であればベース型へ RelabelType を適用し、それ以外はそのまま返す。 4. **定数変換**: 入力が UNKNOWN 型の定数である場合、ターゲット型の入力関数（typinput）を使用して新しい Const ノードを生成する。

  • INTERVALOID の場合は targetTypeMod を保持し、それ以外は -1 を使用する。
  • 生成された定数は、必要に応じてデトースト（PG_DETOAST_DATUM）される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/analyze.c
  • src/backend/parser/parse_clause.c
  • src/backend/parser/parse_coerce.c（内部再帰的呼び出しの可能性あり）
  • **呼び出し先**:
  • makeRelabelType, makeNode, getBaseType, getBaseTypeAndTypmod, typeidType, typeTypeCollation, typeLen, typeByVal, setup_parser_errposition_callback, stringTypeDatum 等。

関連データ構造

  • Node, ParseState, Const, RelabelType, Oid

失敗モードと境界条件

  • UNKNOWN 型の定数に対し、ターゲット型の入力関数が不正な値を渡された場合、エラーが発生する。
  • INTERVAL 型以外の型で typmod を渡すと、入力関数が期待通りに動作しない可能性があるため、内部で -1 に調整される。

保守担当者向けメモ

  • 本関数は元の式ツリーを破壊せず、新しいノードをラップして返す設計である。
  • UNKNOWN 定数の変換において、typinput 関数が非イミュータブルな場合の評価タイミングについては将来的な検討課題（ソース内の XXX コメント参照）。
