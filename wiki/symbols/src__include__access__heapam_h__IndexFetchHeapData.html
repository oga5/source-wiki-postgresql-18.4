IndexFetchHeapData

定義

  • 種別: struct
  • 定義箇所: src/include/access/heapam.h:113

目的

インデックススキャンを通じてヒープテーブルからデータを取得する際の、ヒープ固有の状態を保持するための構造体です。

シグネチャまたは形状

typedef struct IndexFetchHeapData
{
    IndexFetchTableData xs_base;
    Buffer              xs_cbuf;
} IndexFetchHeapData;

入力と出力

  • **入力**: IndexFetchTableData（AM 非依存の基底部分）を継承します。
  • **出力**: xs_cbuf を通じて、現在スキャン中のヒープバッファを保持します。

前提条件と不変条件

  • xs_cbuf が InvalidBuffer でない場合、そのバッファに対するピン（pin）が保持されている必要があります。

アルゴリズムまたは振る舞い

  • インデックススキャン実行時に、ヒープ側のバッファ管理状態を保持します。
  • src/backend/access/heap/heapam_handler.c において、スキャン状態の初期化や参照に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_handler.c 内の関数（palloc0 によるメモリ確保および型キャストによる参照）。
  • **呼び出し先**: 構造体定義のため、特定の関数呼び出しは含まれません。

関連データ構造

  • IndexFetchTableData: 本構造体の基底となる AM 非依存のデータ構造。
  • Buffer: storage/buf.h で定義されるバッファ識別子。

失敗モードと境界条件

  • xs_cbuf が InvalidBuffer の場合、バッファはピンされていません。

保守担当者向けメモ

  • 本構造体は heapam_handler.c で動的に確保・キャストされるため、メモリ管理のライフサイクルに注意してください。
  • xs_base とのメモリレイアウトの整合性は IndexFetchTableData の定義に依存します。
