pg_log_filter_error

定義

  • src/bin/pg_dump/filter.c:153

目的

フィルタファイルの解析中に発生した構文エラーや不正な設定を、標準エラー出力へログとして記録する。

シグネチャまたは形状

void pg_log_filter_error(FilterStateData *fstate, const char *fmt, ...)

入力と出力

  • **入力**:
  • fstate: フィルタ処理の状態を保持する FilterStateData 構造体へのポインタ。現在の行番号 (lineno)、ファイル名 (filename)、または標準入力 (fp) を特定するために使用される。
  • fmt: printf 形式のフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**: なし（戻り値は void）。副作用として pg_log_error を介したログ出力を行う。

前提条件と不変条件

  • fstate は有効な FilterStateData 構造体を指している必要がある。
  • fstate->lineno は現在解析中の行番号を正しく保持している必要がある。

アルゴリズムまたは振る舞い

1. 可変長引数を vsnprintf を用いて最大256バイトのバッファ buf に展開する。 2. fstate->fp が stdin と等しい場合、「標準入力の何行目でエラーが発生したか」を示すメッセージをログ出力する。 3. それ以外の場合、「指定されたファイル名の何行目でエラーが発生したか」を示すメッセージをログ出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/filter.c 内の各解析関数（filter_read_item 等）、および pg_dump.c や pg_dumpall.c のフィルタ適用ロジックから呼び出される。
  • **呼び出し先**: vsnprintf（標準Cライブラリ）、pg_log_error（common/logging.h）。

関連データ構造

  • FilterStateData: フィルタの解析状態（ファイルポインタ、ファイル名、行番号など）を管理する構造体。

失敗モードと境界条件

  • エラーメッセージが256バイトを超える場合、vsnprintf の制限によりメッセージが切り詰められる可能性がある。
  • fstate が NULL の場合、デリファレンスによりクラッシュする可能性がある（ソースコード上でのガードは存在しない）。

保守担当者向けメモ

  • 本関数は pg_dump および pg_restore のフィルタ機能におけるエラー報告の標準インターフェースである。エラーメッセージの形式を変更する場合は、本関数を修正することで一貫性を保つことができる。
