member_verify_control_data

定義

  • 種別: function
  • 定義箇所: src/bin/pg_verifybackup/astreamer_verify.c:366

目的

バックアップアーカイブ内の制御ファイル（global/pg_control）の整合性を検証します。

シグネチャまたは形状

static void member_verify_control_data(astreamer *streamer)

入力と出力

  • **入力**: astreamer *streamer（検証ストリーマーのコンテキスト）
  • **出力**: なし（検証失敗時は report_fatal_error によりプロセスを終了）

前提条件と不変条件

  • 対象ファイルが XLOG_CONTROL_FILE であること。
  • mystreamer->verify_control_data フラグが真であること。
  • 制御ファイルの内容が mystreamer->control_file にバッファリングされていること。

アルゴリズムまたは振る舞い

1. **サイズ検証**: mystreamer->control_file_bytes が PG_CONTROL_FILE_SIZE と一致するか確認する。 2. **CRC検証**: ControlFileData 構造体の CRC フィールドを除いたデータから CRC32C を計算し、格納されている値と比較する。 3. **バージョン検証**: pg_control_version が PG_CONTROL_VERSION と一致するか確認する。 4. **識別子検証**: マニフェストに記録された system_identifier と、制御ファイル内の system_identifier が一致するか確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_verifybackup/astreamer_verify.c 内のトレーラー処理フェーズから呼び出される（直接的な関数名はソースコードの文脈上、astreamer_verify_content 等のトレーラー処理ロジックから参照される）。
  • **呼び出し先**: report_fatal_error（エラー発生時）、INIT_CRC32C, COMP_CRC32C, FIN_CRC32C, EQ_CRC32C（CRC計算・比較用マクロ）。

関連データ構造

  • astreamer_verify: 検証状態を保持する構造体。
  • ControlFileData: 制御ファイルの構造体定義。
  • manifest_data: マニフェスト情報を保持する構造体。

失敗モードと境界条件

  • **サイズ不一致**: 制御ファイルのサイズが期待値と異なる場合に致命的エラーを報告。
  • **CRC不一致**: 計算された CRC がファイル内の値と異なる場合に致命的エラーを報告。
  • **バージョン不一致**: 制御ファイルのバージョンが現在の PostgreSQL のバージョンと合致しない場合に致命的エラーを報告。
  • **システム識別子不一致**: マニフェストと制御ファイル間でシステム識別子が異なる場合に致命的エラーを報告。

保守担当者向けメモ

  • 制御ファイルのサイズ比較において int へのキャストが行われており、理論上のオーバーフローリスクが存在する。
  • 本関数はマニフェストのバージョンが1より大きい場合にのみ実行される前提で設計されている。
