mic_to_euc_kr

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_kr_and_mic/euc_kr_and_mic.c:59

目的

MULE_INTERNAL (MIC) エンコーディングから EUC_KR エンコーディングへ文字列を変換する、PostgreSQLの動的ロード可能な変換プロシージャです。

シグネチャまたは形状

Datum mic_to_euc_kr(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を受け取ります。
  • src (CSTRING): 変換元のMIC文字列。
  • dest (CSTRING): 変換結果を格納するバッファ。
  • len (INT32): 入力文字列の長さ。
  • noError (BOOL): 変換エラー時の挙動フラグ。
  • **出力**: PG_RETURN_INT32 を介して、変換されたバイト数を返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_MULE_INTERNAL, PG_EUC_KR) マクロにより、エンコーディングIDの整合性が検証されます。

アルゴリズムまたは振る舞い

1. fmgr インターフェースを通じて引数を取得します。 2. エンコーディング変換の引数チェックを実行します。 3. 内部関数 mic2euc_kr を呼び出し、実際のバイト変換処理（MICの LC_KS5601 プレフィックスの除去と EUC_KR 形式への復元）を実行します。 4. 変換されたバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(mic_to_euc_kr) マクロにより、PostgreSQLの関数マネージャから呼び出されることが登録されています。
  • **呼び出し先**: 内部関数 mic2euc_kr を呼び出します。

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • PG_FUNCTION_ARGS: 関数マネージャの引数コンテキスト。

失敗モードと境界条件

  • noError が true の場合、変換エラーが発生しても例外を投げず、処理を中断してそれまでに変換できたバイト数を返します。
  • 不正なエンコーディングシーケンスに対する検証は mic2euc_kr 内で行われます。

保守担当者向けメモ

  • 本関数は fmgr インターフェースのラッパーです。変換ロジックの変更が必要な場合は、内部関数 mic2euc_kr を参照してください。
