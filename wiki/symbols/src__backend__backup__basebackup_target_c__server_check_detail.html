server_check_detail

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_target.c:231

目的

server バックアップターゲットに対して、指定された詳細情報（target_detail）が提供されているかを検証します。

シグネチャまたは形状

static void *server_check_detail(char *target, char *target_detail)

入力と出力

  • **入力**:
  • target: ターゲット名（文字列）。エラーメッセージ生成に使用。
  • target_detail: 検証対象の詳細情報文字列。
  • **出力**:
  • 成功時: target_detail ポインタをそのまま返します。
  • 失敗時: ereport(ERROR, ...) により処理を中断します。

前提条件と不変条件

  • target_detail が NULL であってはなりません。

アルゴリズムまたは振る舞い

1. target_detail が NULL かどうかを確認します。 2. NULL の場合、ERRCODE_SYNTAX_ERROR を発生させ、ターゲット名を含むエラーメッセージを出力してバックアップ処理を停止します。 3. NULL でない場合、入力された target_detail をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup_target.c 内の静的配列 builtin_backup_targets の初期化時に、server ターゲットの検証関数として登録されています。
  • **呼び出し先**: PostgreSQL のエラー報告関数 ereport を呼び出します。

関連データ構造

  • BaseBackupTargetType: 本関数はこの構造体の check_detail フィールドに設定されます。

失敗モードと境界条件

  • target_detail が指定されていない場合、バックアップ開始の初期段階でエラーとなります。

保守担当者向けメモ

  • 本関数は server ターゲット専用の検証ロジックです。将来的に server ターゲットが詳細情報を必要としない仕様に変更される場合は、本関数のロジックを更新する必要があります。
