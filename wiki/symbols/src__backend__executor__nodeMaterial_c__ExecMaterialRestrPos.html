ExecMaterialRestrPos

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMaterial.c:289

目的

Material ノードにおいて、以前に ExecMaterialMarkPos で保存した Tuplestore の読み取り位置を復元し、アクティブな読み取りポインタをその位置に更新する。

シグネチャまたは形状

void ExecMaterialRestrPos(MaterialState *node)

入力と出力

  • **入力**: MaterialState *node （マテリアライズノードの状態構造体）
  • **出力**: なし（副作用として node->tuplestorestate 内の読み取りポインタを更新）

前提条件と不変条件

  • node->eflags に EXEC_FLAG_MARK フラグが設定されている必要がある（Assert により検証）。

アルゴリズムまたは振る舞い

1. node->tuplestorestate が初期化されていない（まだマテリアライズされていない）場合は、何もせず終了する。 2. tuplestore_copy_read_pointer を呼び出し、保存済みのマーク（インデックス 1）からアクティブな読み取りポインタ（インデックス 0）へ位置情報をコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:393 （ExecRestrPos 経由での呼び出しが確認されている）
  • **呼び出し先**: tuplestore_copy_read_pointer （src/backend/utils/sort/tuplestore.c で定義）

関連データ構造

  • MaterialState: マテリアライズノードの実行状態を保持する構造体。tuplestorestate を介して Tuplestore を管理する。

失敗モードと境界条件

  • tuplestorestate が NULL の場合、関数は安全に早期リターンする。
  • EXEC_FLAG_MARK フラグが欠如している場合、デバッグビルドではアサーション失敗により停止する。

保守担当者向けメモ

  • 本関数は Tuplestore の読み取り位置管理に依存している。Tuplestore の内部実装やポインタ管理に変更がある場合は、インデックス指定（1 から 0 へのコピー）の整合性を確認すること。
