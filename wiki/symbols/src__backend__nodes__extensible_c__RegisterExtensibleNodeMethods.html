RegisterExtensibleNodeMethods

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/extensible.c:75

目的

拡張機能が定義したカスタムノードのメソッド構造体を、コアシステムのレジストリに登録する。

シグネチャまたは形状

void RegisterExtensibleNodeMethods(const ExtensibleNodeMethods *methods)

入力と出力

  • **入力**: methods - 登録対象の ExtensibleNodeMethods 構造体へのポインタ。extnodename フィールドが含まれている必要がある。
  • **出力**: なし。

前提条件と不変条件

  • methods 引数は NULL であってはならない。
  • methods->extnodename は EXTNODENAME_MAX_LEN（64バイト）以内である必要がある。
  • 内部ハッシュテーブル extensible_node_methods が初期化済みであること（初回呼び出し時に自動的に初期化される）。

アルゴリズムまたは振る舞い

1. 内部関数 RegisterExtensibleNodeEntry を呼び出す。 2. その際、ハッシュテーブルとして extensible_node_methods を指定する。 3. 登録用ラベルとして "Extensible Node Methods" を使用する。 4. methods->extnodename をキーとして、methods 構造体をハッシュテーブルに格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: 拡張機能（外部モジュール）がカスタムノードを定義する際に呼び出す。
  • **呼び出し先**: RegisterExtensibleNodeEntry（内部関数）。

関連データ構造

  • ExtensibleNodeMethods: 拡張ノードの操作メソッドを定義する構造体。
  • extensible_node_methods: 登録されたメソッドを保持する内部ハッシュテーブル。

失敗モードと境界条件

  • 同一の extnodename で既に登録が存在する場合、RegisterExtensibleNodeEntry 内でエラーが発生する可能性がある。
  • メモリ不足等のシステムエラーが発生した場合、RegisterExtensibleNodeEntry がエラーを報告する。

保守担当者向けメモ

  • 本関数は公開APIであり、src/include/nodes/extensible.h で宣言されている。
  • 登録されたメソッドは、GetExtensibleNodeMethods を通じてコアシステム（copyfuncs.c や outfuncs.c など）から参照される。
