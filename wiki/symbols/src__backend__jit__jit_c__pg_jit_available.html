pg_jit_available

定義

  • 種別: function
  • 定義箇所: src/backend/jit/jit.c:55

目的

JIT（Just-In-Time）コンパイル機能が現在利用可能かどうかを判定し、その結果を SQL レベルで返却する。

シグネチャまたは形状

Datum pg_jit_available(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（PostgreSQL の関数呼び出し規約に基づく引数）。
  • **出力**: Datum 型として、JIT が利用可能な場合は true、そうでない場合は false を返す。

前提条件と不変条件

  • 内部的に provider_init() を呼び出し、JIT プロバイダーの初期化を試みる。
  • provider_init() が正常に完了するか、既に初期化済みである場合に true を返す。

アルゴリズムまたは振る舞い

1. provider_init() を呼び出す。 2. provider_init() の戻り値（プロバイダーのロード成功可否）を PG_RETURN_BOOL マクロを通じて呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェースから呼び出されることを想定している（詳細は提供されたソース範囲外のため不明）。
  • **呼び出し先**: provider_init() を呼び出す。

関連データ構造

  • provider（jit.c 内で管理される JIT プロバイダーの関数ポインタ構造体）。

失敗モードと境界条件

  • JIT プロバイダーのロードに失敗した場合、provider_init() は false を返し、本関数も false を返す。
  • 一度ロードに失敗したプロバイダーは、provider_failed_loading フラグにより再試行が抑止される設計となっている。

保守担当者向けメモ

  • 本関数は SQL からの問い合わせに応答するためのラッパーである。
  • 実際の JIT プロバイダーのロードロジックは provider_init() に依存しているため、動作の詳細については provider_init() の実装を参照すること。
