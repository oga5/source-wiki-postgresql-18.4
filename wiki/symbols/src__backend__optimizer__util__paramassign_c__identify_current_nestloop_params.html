identify_current_nestloop_params

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/paramassign.c:621

目的

ネストループ結合の左側（外側）のパスで評価可能なパラメータ（Var または PlaceHolderVar）を root->curOuterParams リストから特定し、抽出する。

シグネチャまたは形状

List *identify_current_nestloop_params(PlannerInfo *root, Relids leftrelids, Relids outerrelids)

入力と出力

  • **入力**:
  • root: プランナーのグローバル情報。
  • leftrelids: 左側パスで利用可能なリレーションIDの集合。
  • outerrelids: 結合に必要な外部リレーションIDの集合。
  • **出力**:
  • 抽出された NestLoopParam 構造体のリスト。

前提条件と不変条件

  • root->curOuterParams は、本関数によって抽出された要素が削除され、更新される。
  • Var の場合、その varno が leftrelids に含まれている必要がある。
  • PlaceHolderVar の場合、その評価位置（ph_eval_at）が leftrelids と outerrelids の和集合のサブセットであり、かつ leftrelids と重複している必要がある。

アルゴリズムまたは振る舞い

1. leftrelids と outerrelids を結合し、評価可能な全リレーション集合 allleftrelids を作成する。 2. root->curOuterParams を反復処理し、各 NestLoopParam を検査する。 3. **Var の場合**: varno が leftrelids に含まれるか確認し、該当すれば curOuterParams から削除。varnullingrels を leftrelids との積集合で更新し、結果リストに追加する。 4. **PlaceHolderVar の場合**: 評価位置 ph_eval_at が allleftrelids のサブセットかつ leftrelids と重複するか確認。

  • 条件を満たす場合、curOuterParams から削除。
  • hasSubLinks が真の場合、ph_var をコピーして nlp->paramval を更新。
  • phnullingrels を get_placeholder_nulling_relids と leftrelids の積集合で更新し、結果リストに追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/createplan.c:4418
  • **呼び出し先**:
  • bms_union, bms_is_member, bms_intersect, bms_is_subset, bms_overlap (Bitmapset操作)
  • find_placeholder_info, get_placeholder_nulling_relids
  • foreach_delete_current, lappend, copyObject

関連データ構造

  • PlannerInfo: プランナーのコンテキスト。
  • NestLoopParam: ネストループ結合で渡されるパラメータ情報。
  • Relids: リレーションIDの集合（Bitmapset）。
  • PlaceHolderVar: プレースホルダ変数。

失敗モードと境界条件

  • outerrelids が空の場合、allleftrelids は leftrelids と等しくなる。
  • PlaceHolderVar がサブクエリから引き上げられた場合、SubLink を含む可能性があるため、copyObject による再構築が行われる。

保守担当者向けメモ

  • nullingrels の更新は外結合の最適化に関連する繊細な処理である。
  • root->curOuterParams の破壊的な変更（foreach_delete_current）を伴うため、呼び出しタイミングに注意すること。
