SlabContextCreate

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/slab.c:321

目的

SLAB アロケータ用のメモリコンテキストを作成し、指定されたブロックサイズとチャンクサイズに基づいて管理構造体を初期化する。

シグネチャまたは形状

MemoryContext SlabContextCreate(MemoryContext parent,
                                const char *name,
                                Size blockSize,
                                Size chunkSize)

入力と出力

  • **入力**:
  • parent: 親メモリコンテキスト。
  • name: コンテキスト名。
  • blockSize: 割り当てブロックのサイズ。
  • chunkSize: 管理するオブジェクトの固定サイズ。
  • **出力**:
  • 初期化された MemoryContext へのポインタ。

前提条件と不変条件

  • blockSize は MEMORYCHUNK_MAX_BLOCKOFFSET 以下であること。
  • fullChunkSize（ヘッダー込みのチャンクサイズ）は MEMORYCHUNK_MAX_VALUE 以下であること。
  • chunksPerBlock（1ブロックあたりのチャンク数）が 1 以上であること。

アルゴリズムまたは振る舞い

1. **サイズ計算**: chunkSize がポインタサイズ未満の場合は調整し、ヘッダーサイズを加算して fullChunkSize を算出する。 2. **ブロック容量計算**: blockSize からブロックヘッダーを除いた領域に収まるチャンク数を算出する。 3. **メモリ確保**: malloc を使用して SlabContext 構造体（および可変長の管理領域）を確保する。 4. **初期化**:

  • チャンクサイズ、ブロックサイズ、シフト量（blocklist_shift）を計算・設定する。
  • 空きブロックリスト (emptyblocks) および各ブロックリスト (blocklist) を初期化する。

    • 5. **コンテキスト登録**: MemoryContextCreate を呼び出し、PostgreSQL のメモリ管理階層に登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/reorderbuffer.c
  • src/include/lib/radixtree.h
  • src/test/modules/test_radixtree/test_radixtree.c
  • **呼び出し先**:
  • malloc (標準ライブラリ)
  • dclist_init / dlist_init (lib/ilist.h)
  • MemoryContextCreate (src/backend/utils/mmgr/mcxt.c)
  • elog / ereport (エラー報告)

関連データ構造

  • SlabContext: 本関数で初期化されるメイン構造体。
  • MemoryContext: 戻り値の型。
  • SlabBlock: ブロック管理用構造体。

失敗モードと境界条件

  • **メモリ不足**: malloc が失敗した場合、ereport(ERROR) を発行し、メモリコンテキストの統計情報を出力して終了する。
  • **サイズ不整合**: chunksPerBlock が 0 になる場合（blockSize が小さすぎる場合）、elog(ERROR) で異常終了する。

保守担当者向けメモ

  • SlabRealloc はサポートされていないため、本関数で設定した chunkSize は固定される。
  • MEMORY_CONTEXT_CHECKING が有効な場合、デバッグ用のセンチネルバイト領域が fullChunkSize に追加される。
  • MemoryContextCreate 呼び出し前は ereport を避ける必要がある（ヘッダーリーク防止のため）。
