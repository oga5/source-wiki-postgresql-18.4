gistvacuumcleanup

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistvacuum.c:74

目的

GiST インデックスの VACUUM 処理の終了時に呼び出され、統計情報の確定やインデックス全体の走査による統計収集を行う。

シグネチャまたは形状

IndexBulkDeleteResult *gistvacuumcleanup(IndexVacuumInfo *info, IndexBulkDeleteResult *stats)

入力と出力

  • **入力**:
  • info: IndexVacuumInfo 構造体（VACUUM の実行オプションやヒープのタプル数情報を含む）。
  • stats: IndexBulkDeleteResult 構造体（gistbulkdelete による前処理の結果。未実行の場合は NULL）。
  • **出力**: 更新された IndexBulkDeleteResult 構造体。

前提条件と不変条件

  • info->analyze_only が真の場合、何もせず入力された stats をそのまま返す。
  • インデックスのタプル数がヒープのタプル数を超過している場合、ヒープの正確なカウントが既知であれば、統計情報をヒープの数に補正する。

アルゴリズムまたは振る舞い

1. analyze_only モードであれば即座に終了する。 2. stats が NULL（gistbulkdelete が未実行）の場合、palloc0 で IndexBulkDeleteResult を初期化し、gistvacuumscan を呼び出してインデックス全体を走査し統計情報を収集する。 3. info->estimated_count が偽（正確なヒープタプル数が判明している）の場合、stats->num_index_tuples が info->num_heap_tuples を超えていれば、後者を上限として値を補正する。 4. 最終的な stats を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gist.c: amroutine->amvacuumcleanup として登録され、PostgreSQL の VACUUM フレームワークから呼び出される。
  • **呼び出し先**:
  • gistvacuumscan: stats が NULL の場合にインデックスの統計情報を収集するために呼び出される。

関連データ構造

  • IndexVacuumInfo: VACUUM の実行パラメータを保持。
  • IndexBulkDeleteResult: インデックスのタプル数やページ数などの統計情報を保持。

失敗モードと境界条件

  • 並行して発生するページ分割により、インデックスのタプル数が一時的に過大評価される可能性がある。本関数はヒープのタプル数情報を利用してこの不整合を緩和する。

保守担当者向けメモ

  • 本関数は VACUUM の最終フェーズを担当する。gistbulkdelete が実行済みであればその結果を再利用し、未実行であれば gistvacuumscan を介して全走査を行うという二段構えの設計になっている。
