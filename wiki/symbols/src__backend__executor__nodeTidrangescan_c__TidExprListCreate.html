TidExprListCreate

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTidrangescan.c:105

目的

TID範囲スキャン実行ノードにおいて、クエリ内の ctid 比較条件（tidrangequals）を解析し、実行時に評価可能な TidOpExpr のリストを生成・保持する。

シグネチャまたは形状

static void TidExprListCreate(TidRangeScanState *tidrangestate)

入力と出力

  • **入力**: TidRangeScanState *tidrangestate（スキャンノードの状態構造体）。この構造体内の ss.ps.plan から TidRangeScan ノードを取得する。
  • **出力**: なし（副作用として tidrangestate->trss_tidexprs に TidOpExpr のリストを格納する）。

前提条件と不変条件

  • tidrangestate->ss.ps.plan が TidRangeScan 型のプランノードを指していること。
  • node->tidrangequals に格納されている各要素が OpExpr 型であること。

アルゴリズムまたは振る舞い

1. tidrangestate からプランノード node を取得する。 2. node->tidrangequals リストを走査する。 3. 各要素が OpExpr であることを確認し、そうでない場合は elog(ERROR) を発生させる。 4. MakeTidOpExpr を呼び出し、各 OpExpr を TidOpExpr に変換する。 5. 生成された TidOpExpr をリスト tidexprs に追加する。 6. 最終的なリストを tidrangestate->trss_tidexprs に代入する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeTidrangescan.c 内の初期化処理（ExecInitTidRangeScan 等）から呼び出されることが示唆されている。
  • **呼び出し先**: MakeTidOpExpr を呼び出す。

関連データ構造

  • TidRangeScanState: スキャンノードの実行時状態を保持する構造体。
  • TidRangeScan: プランノード。tidrangequals（ctid 比較式のリスト）を保持する。
  • TidOpExpr: ctid 比較条件を保持する内部構造体。

失敗モードと境界条件

  • node->tidrangequals 内の要素が OpExpr ではない場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は nodeTidscan.c にも同名の関数が存在するが、本ページは nodeTidrangescan.c の実装を対象としている。
  • TidOpExpr の生成ロジックに変更を加える場合は、MakeTidOpExpr の仕様と併せて確認すること。
