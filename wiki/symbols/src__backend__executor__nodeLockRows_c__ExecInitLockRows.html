ExecInitLockRows

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeLockRows.c:290

目的

LockRows ノードの実行時状態を初期化し、クエリ実行に必要な行ロックおよび同時実行制御（EPQ: EvalPlanQual）のためのデータ構造を構築する。

シグネチャまたは形状

LockRowsState *ExecInitLockRows(LockRows *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の LockRows プランノード。
  • estate: クエリ実行状態を保持する EState。
  • eflags: 実行フラグ（EXEC_FLAG_MARK はサポート外）。
  • **出力**:
  • 初期化済みの LockRowsState 構造体へのポインタ。

前提条件と不変条件

  • InitPlan フェーズにおいて、グローバルな ExecRowMark リストが既に構築されている必要がある。
  • LockRows ノードは ExecQual や ExecProject を呼び出さないため、ExprContext を必要としない。
  • node->rowMarks に含まれる各 PlanRowMark を走査し、ロック対象とEPQ対象に分類する。

アルゴリズムまたは振る舞い

1. LockRowsState 構造体を割り当て、ExecLockRows を実行関数として設定する。 2. ExecInitResultTypeTL を呼び出し、結果の型を初期化する。 3. ExecInitNode を呼び出し、下位プラン（outerPlan）を再帰的に初期化する。 4. node->rowMarks を反復処理し、以下の条件で分類する:

  • isParent フラグが立っている、またはプルーニング済みのリレーションは無視する。
  • ExecBuildAuxRowMark を使用して補助情報を構築する。
  • RowMarkRequiresRowShareLock が真であれば lr_arowMarks（ロック対象）へ追加する。
  • それ以外は epq_arowmarks（EPQ再評価用）へ追加する。

    • 5. EvalPlanQualInit を呼び出し、EPQの状態管理を初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execProcnode.c: ExecInitNode からの呼び出し。
  • src/backend/executor/execMain.c: プラン初期化時。
  • **呼び出し先**:
  • ExecInitNode (下位ノード初期化)
  • ExecFindRowMark / ExecBuildAuxRowMark (行マーク情報の構築)
  • EvalPlanQualInit (EPQ状態の初期化)

関連データ構造

  • LockRowsState: ノードの実行時状態を保持。
  • LockRows: プランノード定義。
  • ExecRowMark / ExecAuxRowMark: 行ロックおよびEPQのためのメタデータ。
  • EState: 実行状態全体。

失敗モードと境界条件

  • eflags に EXEC_FLAG_MARK が含まれている場合、Assert により停止する。
  • プルーニングされたリレーションに対する行マークは適切にスキップされる。

保守担当者向けメモ

  • 本関数は射影（Projection）を行わないため、ps_ProjInfo は NULL に設定される。
  • lr_arowMarks にはロックが必要なもののみが格納され、それ以外はEPQ再評価用に分離されるという設計意図を維持すること。
