RoleNameItem

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_dump.c:74

目的

データベース内のロール（Role）の識別子（OID）と名称をペアとして保持し、ダンプ処理中にロール名を効率的に検索・管理するために使用されます。

シグネチャまたは形状

typedef struct
{
	Oid			roleoid;		/* role's OID */
	const char *rolename;		/* role's name */
} RoleNameItem;

入力と出力

  • 入力: ロール情報を格納する配列の要素として使用されます。
  • 出力: 検索時に roleoid に基づいて対応する rolename を提供します。

前提条件と不変条件

  • rolenames 配列としてメモリ上に確保され、検索のためにソートされた状態で保持されることが想定されます。

アルゴリズムまたは振る舞い

  • pg_dump 実行時に、システムカタログから取得したロール情報をこの構造体の配列に格納します。
  • 検索処理（pg_dump.c 内の二分探索ロジックなど）において、roleoid をキーとして RoleNameItem の配列を走査し、対応するロール名を取得するために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/pg_dump.c 内のロール情報管理ロジック。
  • 呼び出し先: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • rolenames: RoleNameItem 型の静的配列（src/bin/pg_dump/pg_dump.c:195）。

失敗モードと境界条件

  • メモリ不足: pg_malloc による配列確保時に失敗する可能性があります。
  • 検索失敗: 配列がソートされていない場合や、対象の roleoid が存在しない場合に検索が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は pg_dump.c 内の静的配列 rolenames を介して利用されます。配列のサイズ変更やソート順序の変更を行う際は、関連する二分探索ロジック（src/bin/pg_dump/pg_dump.c:10515-10520 付近）との整合性に注意してください。
