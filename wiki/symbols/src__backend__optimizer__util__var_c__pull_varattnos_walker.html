pull_varattnos_walker

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/var.c:308

目的

式ツリーを再帰的に走査し、指定されたリレーション（varno）かつ現在のクエリレベル（varlevelsup == 0）に該当する Var ノードの属性番号（varattno）を収集します。

シグネチャまたは形状

static bool pull_varattnos_walker(Node *node, pull_varattnos_context *context);

入力と出力

  • **入力**:
  • node: 走査対象のノード（Node *）。
  • context: 収集した属性番号を保持するビットマップセット（varattnos）と、対象の varno を含むコンテキスト構造体。
  • **出力**:
  • 走査を継続する場合は false、中断する場合は true を返します。

前提条件と不変条件

  • context は有効な pull_varattnos_context 構造体を指している必要があります。
  • 走査対象のツリー内にプラン化されていない Query ノードが含まれていないことを前提とします（Assert により検証）。

アルゴリズムまたは振る舞い

1. node が NULL の場合は false を返して終了します。 2. node が Var 型の場合:

  • varno がコンテキストの指定と一致し、かつ varlevelsup が 0（現在のクエリレベル）であれば、varattno を FirstLowInvalidHeapAttributeNumber で補正した値を context->varattnos に追加します。

    • 3. node が Query 型の場合はアサーションエラーとなります。 4. 上記以外の場合、expression_tree_walker を呼び出し、子ノードに対して再帰的に本関数を適用します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/var.c 内の pull_varattnos から呼び出されます。
  • **呼び出し先**: expression_tree_walker（nodes/nodeFuncs.h）を呼び出し、ツリーの走査を委譲します。

関連データ構造

  • Var: 走査対象となる変数ノード。
  • pull_varattnos_context: 収集結果を格納するコンテキスト構造体。
  • FirstLowInvalidHeapAttributeNumber: 属性番号のオフセット計算に使用される定数。

失敗モードと境界条件

  • Query ノードがツリー内に存在する場合、アサーションにより処理が停止します。
  • node が NULL の場合は安全に無視されます。

保守担当者向けメモ

  • 本関数は expression_tree_walker のコールバックとして設計されています。
  • varlevelsup == 0 のチェックにより、サブクエリ内の変数を誤って収集しないよう制御されています。
