system_rows_samplescangetsamplesize

定義

  • 種別: function
  • 定義箇所: contrib/tsm_system_rows/tsm_system_rows.c:104

目的

SYSTEM_ROWS テーブルサンプルメソッドにおいて、プランナがクエリ実行コストを見積もるために必要な、サンプリング対象のページ数とタプル数を提供します。

シグネチャまたは形状

static void system_rows_samplescangetsamplesize(
    PlannerInfo *root,
    RelOptInfo *baserel,
    List *paramexprs,
    BlockNumber *pages,
    double *tuples
)

入力と出力

  • **入力**:
  • root: プランナのコンテキスト情報。
  • baserel: 対象リレーションの統計情報（タプル数、ページ数など）。
  • paramexprs: TABLESAMPLE SYSTEM_ROWS(n) で指定された引数リスト。
  • **出力**:
  • pages: 見積もられたサンプリング対象ページ数（BlockNumber へのポインタ経由）。
  • tuples: 見積もられたサンプリング対象タプル数（double へのポインタ経由）。

前提条件と不変条件

  • paramexprs の先頭要素は、サンプリングする行数を指定する式である必要があります。
  • baserel->tuples および baserel->pages が有効な値であることを前提として計算を行います。

アルゴリズムまたは振る舞い

1. **行数見積もり**: paramexprs から行数（ntuples）を抽出します。定数でない場合や無効な値の場合はデフォルト値 1000 を使用します。 2. **制約適用**: ntuples をリレーションの総タプル数でクランプ（上限設定）します。 3. **ページ数見積もり**:

  • リレーションのタプル密度（tuples / pages）を計算し、それに基づいて必要なページ数（npages）を算出します。
  • 統計情報が不足している場合は、1ページあたり1タプルと仮定します。

    • 4. **最終調整**: npages をリレーションの総ページ数でクランプし、結果を pages および tuples に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: tsm_system_rows_handler 内で TsmRoutine 構造体の SampleScanGetSampleSize フィールドに登録され、プランナから呼び出されます。
  • **呼び出し先**: estimate_expression_value（式評価）、clamp_row_est（値の範囲制限）を呼び出します。

関連データ構造

  • PlannerInfo
  • RelOptInfo
  • TsmRoutine

失敗モードと境界条件

  • ntuples が負の値の場合、デフォルト値 1000 に置き換えられます。
  • baserel->tuples や baserel->pages が 0 の場合、ページ数は ntuples と同等とみなされます。

保守担当者向けメモ

  • 本関数はプランナによるコスト見積もりのためのヒューリスティックな計算を行うものであり、実際の実行時のサンプリング結果と完全に一致するとは限りません。
  • clamp_row_est を使用して、見積もり値が妥当な範囲に収まるように制御されています。
