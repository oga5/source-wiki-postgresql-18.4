xlog_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/xlogdesc.c:57

目的

WAL（Write-Ahead Log）の「XLOG」リソースマネージャに関連するレコード内容を解析し、人間が読める形式の文字列として StringInfo バッファに書き出す。

シグネチャまたは形状

void xlog_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 出力文字列を格納する StringInfo 構造体へのポインタ。
  • record: 解析対象のWALレコード状態を保持する XLogReaderState 構造体へのポインタ。
  • **出力**: なし（buf を通じて文字列が更新される）。

前提条件と不変条件

  • record は有効な XLogReaderState を指していること。
  • XLogRecGetInfo(record) で取得される info フィールドが、XLOGリソースマネージャで定義された有効なタイプであること。

アルゴリズムまたは振る舞い

1. XLogRecGetData でレコードのペイロードを取得し、XLogRecGetInfo でレコードタイプ（info）を特定する。 2. info ビットから XLR_INFO_MASK を除去し、以下のタイプに応じて分岐処理を行う。

  • XLOG_CHECKPOINT_SHUTDOWN / ONLINE: CheckPoint 構造体にキャストし、チェックポイントの各フィールド（redo LSN, TLI, xid, oid等）を整形して出力。
  • XLOG_NEXTOID: OID値を抽出して出力。
  • XLOG_RESTORE_POINT: リストアポイント名を文字列として出力。
  • XLOG_FPI / XLOG_FPI_FOR_HINT: 処理なし。
  • XLOG_BACKUP_END: バックアップ開始時のLSNを出力。
  • XLOG_PARAMETER_CHANGE: xl_parameter_change 構造体からパラメータ設定値（max_connections等）を読み取り、文字列化して出力。
  • XLOG_FPW_CHANGE: Full Page Writesの有効/無効状態を出力。
  • XLOG_END_OF_RECOVERY: リカバリ終了時のタイムラインや時刻を出力。
  • XLOG_OVERWRITE_CONTRECORD: 上書きされたLSNと時刻を出力。
  • XLOG_CHECKPOINT_REDO: wal_level を出力。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h で定義されるリソースマネージャリストを通じて、WAL解析ツール（pg_waldump 等）から間接的に呼び出される。
  • **呼び出し先**: XLogRecGetData, XLogRecGetInfo, appendStringInfo, get_wal_level_string, timestamptz_to_str, memcpy 等。

関連データ構造

  • StringInfo: 文字列バッファ。
  • XLogReaderState: WALレコード読み取り状態。
  • CheckPoint, xl_parameter_change, xl_end_of_recovery, xl_restore_point, xl_overwrite_contrecord: 各WALレコードタイプに対応する構造体。

失敗モードと境界条件

  • info が未知のタイプである場合、何も出力されず終了する。
  • memcpy を使用しているため、レコードのペイロードサイズが期待される構造体サイズと一致しない場合、メモリ破壊や不正な読み取りが発生する可能性がある。

保守担当者向けメモ

  • 新しいXLOGレコードタイプを追加した際は、必ず xlog_identify と本関数の switch 文（または if-else 分岐）を更新すること。
  • wal_level などの列挙型は get_wal_level_string を介して変換することで、一貫性を維持すること。
