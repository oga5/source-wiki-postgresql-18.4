debackslash

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/read.c:213

目的

シリアライズされた文字列からノードを復元する際、エスケープ文字（バックスラッシュ）を除去して元の文字列データを取得する。

シグネチャまたは形状

char *debackslash(const char *token, int length)

入力と出力

  • **入力**:
  • token: エスケープされた文字列へのポインタ。
  • length: 対象文字列の長さ。
  • **出力**: エスケープが解除された新しい文字列（palloc によりメモリ確保された領域）。

前提条件と不変条件

  • length は token が指す文字列の有効な長さであること。
  • 戻り値は呼び出し元で解放（pfree）する必要がある。

アルゴリズムまたは振る舞い

1. length + 1 バイトのメモリを確保する。 2. 入力文字列を走査し、バックスラッシュ（\）が出現した場合、その直後の文字をスキップして次の文字をコピーする。 3. バックスラッシュ以外の文字はそのままコピーする。 4. 最後にヌル終端文字を付与して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/read.c: stringToNode 関連の処理内で makeString や makeBitString の引数として呼び出される。
  • src/backend/nodes/readfuncs.c: 特定のノード型の読み込み処理内で呼び出される。
  • **呼び出し先**:
  • palloc（PostgreSQLのメモリ割り当て関数）。

関連データ構造

  • Node: 本関数が処理する文字列データは、主に Node ツリーの復元過程で使用される。

失敗モードと境界条件

  • length が 1 の場合、バックスラッシュのスキップ処理は行われない。
  • palloc が失敗した場合はメモリ不足エラー（elog(ERROR)）が発生する。

保守担当者向けメモ

  • pg_strtok はバックスラッシュを保持したままトークンを切り出すため、値の復元時には必ず本関数を通す必要がある。
  • readfuncs.c 内のコメントによれば、単一文字の場合は本関数の呼び出しオーバーヘッドを避ける最適化が検討されることがある。
