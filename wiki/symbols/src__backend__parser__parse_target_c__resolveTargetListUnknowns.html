resolveTargetListUnknowns

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_target.c:287

目的

ターゲットリスト内の型が不明な式（UNKNOWN型）を、デフォルトのTEXT型へ強制変換（coerce）するために使用されます。

シグネチャまたは形状

void resolveTargetListUnknowns(ParseState *pstate, List *targetlist)

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する ParseState 構造体へのポインタ。
  • targetlist: 変換対象となる TargetEntry ノードのリスト。
  • **出力**: なし（リスト内の TargetEntry の expr フィールドを直接更新します）。

前提条件と不変条件

  • targetlist に含まれる要素は TargetEntry 型である必要があります。
  • pstate は有効な解析状態を指している必要があります。

アルゴリズムまたは振る舞い

1. targetlist 内の各 TargetEntry を順次走査します。 2. 各エントリの式（tle->expr）の型を exprType で取得します。 3. 型が UNKNOWNOID である場合、coerce_type を呼び出し、その式を TEXTOID（TEXT型）へ暗黙的にキャストします。 4. 変換後の式を元の TargetEntry に再代入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/analyze.c: クエリ解析の最終段階で呼び出されます（qry->targetList および qry->returningList に対して）。
  • **呼び出し先**:
  • exprType: 式の型を取得。
  • coerce_type: 式を TEXT 型へ変換。

関連データ構造

  • TargetEntry: ターゲットリストの各要素を構成するノード。
  • ParseState: 解析中のクエリ状態を管理する構造体。
  • Oid: PostgreSQLの型識別子（UNKNOWNOID, TEXTOID）。

失敗モードと境界条件

  • リストが空の場合、何も行わず正常終了します。
  • UNKNOWN 型以外の式が含まれている場合、その式は変更されません。

保守担当者向けメモ

  • 本関数はクエリ解析の最終段階で実行されるため、型解決が不完全なまま残った定数などを安全に TEXT 型へ確定させる役割を持ちます。
  • 新しいノード型を追加した際、exprType が正しく型を返せることを確認してください。
