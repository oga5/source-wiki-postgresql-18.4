add_eq_member

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/equivclass.c:627

目的

新しい非子（non-child）の EquivalenceMember を構築し、指定された EquivalenceClass (EC) に追加します。

シグネチャまたは形状

static EquivalenceMember *add_eq_member(EquivalenceClass *ec, Expr *expr, Relids relids, JoinDomain *jdomain, Oid datatype)

入力と出力

  • **入力**:
  • ec: 追加先の EquivalenceClass 構造体。
  • expr: 追加する式。
  • relids: 式が参照するリレーションの集合。
  • jdomain: 結合ドメイン（存在する場合）。
  • datatype: 式のデータ型 OID。
  • **出力**:
  • 生成され、EC に登録された EquivalenceMember 構造体へのポインタ。

前提条件と不変条件

  • make_eq_member を呼び出してメンバーを初期化します。
  • ec->ec_members リストに新しいメンバーが追加されます。
  • ec->ec_relids に、追加されたメンバーの relids がビットマップとしてマージされます。

アルゴリズムまたは振る舞い

1. make_eq_member を呼び出し、引数に基づいて EquivalenceMember を作成します。 2. lappend を使用して、作成したメンバーを ec->ec_members リストの末尾に追加します。 3. bms_add_members を使用して、ec->ec_relids に新しいメンバーの relids を追加・更新します。 4. 作成した EquivalenceMember を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/equivclass.c 内の複数箇所（行446, 463, 498, 500, 845など）。
  • **呼び出し先**: make_eq_member（メンバー生成）、lappend（リスト操作）、bms_add_members（ビットマップ操作）。

関連データ構造

  • EquivalenceClass: 等価クラスを管理するメイン構造体。
  • EquivalenceMember: EC 内の個別の式を保持する構造体。
  • Relids: リレーション識別子のビットマップセット。

失敗モードと境界条件

  • 本関数自体は揮発性関数や集合を返す関数のチェックを行いません（ソースコード内のコメントによる）。
  • メモリ不足時は lappend や make_eq_member 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は非子メンバーの追加専用です。子メンバー（継承テーブル等）の追加には add_child_eq_member を使用してください。
  • ec_relids の更新は、この EC がどのリレーションにまたがっているかを追跡するために不可欠です。
