RelationCopyStorage

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/storage.c:477

目的

あるリレーションの物理ストレージ（フォーク）の内容を、別のリレーションへ物理的にコピーする。

シグネチャまたは形状

void RelationCopyStorage(SMgrRelation src, SMgrRelation dst, ForkNumber forkNum, char relpersistence)

入力と出力

  • **入力**:
  • src: コピー元のストレージマネージャリレーション。
  • dst: コピー先のストレージマネージャリレーション。
  • forkNum: コピー対象のフォーク番号（MAIN_FORKNUM 等）。
  • relpersistence: リレーションの永続性（RELPERSISTENCE_PERMANENT, RELPERSISTENCE_UNLOGGED 等）。
  • **出力**: なし（副作用として dst にデータが書き込まれる）。

前提条件と不変条件

  • dst は既に作成済みであること。
  • コピー処理中に割り込み（CHECK_FOR_INTERRUPTS）が発生した場合、処理は中断される。
  • ページ読み込み時にチェックサムエラーが検出された場合、統計情報が更新され、エラーが発生する。

アルゴリズムまたは振る舞い

1. **WAL 記録判定**: XLogIsNeeded() が真かつ、永続的なリレーションまたはアンログドの初期フォークである場合に WAL 記録を有効化する。 2. **初期化**: smgr_bulk_start_smgr を用いて一括書き込み状態を開始する。 3. **データ転送**:

  • ソースの全ブロック数（smgrnblocks）を取得し、各ブロックを順次読み込む。
  • PageIsVerified を用いて読み込んだページの整合性を検証する。
  • 検証失敗時は ERROR を発生させる。
  • smgr_bulk_write を介して、ページをコピー先へ書き出す（WAL 記録が必要な場合はフルページイメージとして記録）。

    • 4. **終了処理**: smgr_bulk_finish を呼び出し、一括書き込みを完了させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • smgr_bulk_start_smgr, smgrnblocks, smgrread, smgr_bulk_get_buf, smgr_bulk_write, smgr_bulk_finish
  • PageIsVerified, pgstat_prepare_report_checksum_failure, relpathbackend, ereport

関連データ構造

  • SMgrRelation: ストレージマネージャレベルのリレーション表現。
  • BulkWriteState: 一括書き込み操作の状態を保持する構造体。
  • ForkNumber: 操作対象のフォークを識別する列挙型。

失敗モードと境界条件

  • **データ破損**: 読み込んだページが PageIsVerified を通過しない場合、ERRCODE_DATA_CORRUPTED でエラー終了する。
  • **割り込み**: 処理中にシグナルを受信すると中断される。
  • **チェックサム失敗**: ignore_checksum_failure が設定されていない場合、チェックサムエラーは致命的なエラーとして扱われる。

保守担当者向けメモ

  • 本関数はページの内容を解釈せず、フルページイメージとしてコピーするため、WAL ログのサイズが大きくなる可能性がある。
  • relpersistence の判定ロジックは、アンログドテーブルの初期フォークが永続的なリレーションと同様の同期・WAL 記録を必要とする仕様に基づいている。
