GetTsmRoutine

定義

  • 種別: function
  • 定義箇所: src/backend/access/tablesample/tablesample.c:26

目的

指定されたサンプリングメソッドのハンドラ OID を受け取り、対応する TsmRoutine 構造体を取得・検証する。

シグネチャまたは形状

TsmRoutine *GetTsmRoutine(Oid tsmhandler)

入力と出力

  • **入力**: tsmhandler (サンプリングメソッドを実装する関数の OID)
  • **出力**: TsmRoutine * (検証済みのサンプリングルーチン構造体へのポインタ)

前提条件と不変条件

  • 引数 tsmhandler は、PG_FUNCTION_INFO_V1 マクロで定義された標準的な PostgreSQL 関数を指している必要がある。
  • 返される構造体は TsmRoutine 型であり、IsA マクロによる型チェックを通過しなければならない。

アルゴリズムまたは振る舞い

1. OidFunctionCall1 を使用し、引数 NULL を渡して tsmhandler 関数を呼び出す。 2. 返された Datum を TsmRoutine 構造体ポインタにキャストする。 3. 構造体が NULL でないこと、および IsA(routine, TsmRoutine) が真であることを確認する。 4. 検証成功時は構造体ポインタを返し、失敗時は elog(ERROR, ...) を発生させて処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeSamplescan.c
  • src/backend/optimizer/path/allpaths.c
  • src/backend/optimizer/path/costsize.c
  • src/backend/parser/parse_clause.c
  • **呼び出し先**:
  • OidFunctionCall1 (PostgreSQL 内部関数)

関連データ構造

  • TsmRoutine (src/include/access/tsmapi.h で定義)

失敗モードと境界条件

  • ハンドラ関数が NULL を返した場合、または TsmRoutine 構造体として正しく認識できない場合、ERROR レベルのログを出力して処理を停止する。

保守担当者向けメモ

  • 本関数は TABLESAMPLE 句の抽象化層として機能する。新しいサンプリングメソッドを追加する際は、本関数を修正する必要はなく、各メソッド側で TsmRoutine を正しく実装・登録すればよい。
