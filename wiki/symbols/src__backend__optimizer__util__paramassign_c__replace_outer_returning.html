replace_outer_returning

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/paramassign.c:367

目的

上位クエリレベルで定義された ReturningExpr を、現在のクエリレベルで参照可能な PARAM_EXEC パラメータに変換します。

シグネチャまたは形状

Param *replace_outer_returning(PlannerInfo *root, ReturningExpr *rexpr)

入力と出力

  • **入力**:
  • root: 現在のクエリレベルの PlannerInfo 構造体。
  • rexpr: 変換対象の ReturningExpr ノード。
  • **出力**:
  • 変換後の Param ノード（PARAM_EXEC 型）。

前提条件と不変条件

  • rexpr->retlevelsup が 0 より大きく、かつ現在の root->query_level 未満であること。
  • 変換後の rexpr は retlevelsup が 0 に調整されること。

アルゴリズムまたは振る舞い

1. rexpr->retlevelsup を辿り、対象の ReturningExpr が属するクエリレベルの PlannerInfo を特定します。 2. rexpr をコピーし、IncrementVarSublevelsUp を用いてレベルを現在のコンテキストに調整します。 3. 重複排除は行わず、常に新しい PlannerParamItem を作成します。 4. root->glob->paramExecTypes に式の型（ptype）を追加し、そのインデックスを paramId とします。 5. root->plan_params に PlannerParamItem を追加します。 6. 最終的に、対応する PARAM_EXEC を持つ Param ノードを生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/subselect.c:2012
  • **呼び出し先**:
  • copyObject
  • IncrementVarSublevelsUp
  • makeNode
  • lappend_oid
  • lappend
  • exprType, exprTypmod, exprCollation, exprLocation

関連データ構造

  • PlannerInfo: クエリのプランニング状態を保持。
  • ReturningExpr: INSERT ... RETURNING 等の式。
  • PlannerParamItem: plan_params リストに格納されるパラメータ定義。
  • Param: 実行時に値を参照するためのノード。

失敗モードと境界条件

  • rexpr->retlevelsup が範囲外の場合、Assert により停止します。
  • メモリ不足時は copyObject や lappend 等の内部関数でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は ReturningExpr の重複排除を行いません。これは、複数の参照があっても毎回新しいスロットを割り当てる設計意図によるものです。
  • root->glob->paramExecTypes はプラン全体で共有されるため、ID の割り当て順序はプラン生成の整合性に直結します。
