RevokeRoleGrantAction

定義

  • 種別: enum
  • 定義箇所: src/backend/commands/user.c:60

目的

ロールのメンバーシップ権限（GRANT）を剥奪する際、実行すべき具体的なアクションを定義します。

シグネチャまたは形状

typedef enum
{
	RRG_NOOP,
	RRG_REMOVE_ADMIN_OPTION,
	RRG_REMOVE_INHERIT_OPTION,
	RRG_REMOVE_SET_OPTION,
	RRG_DELETE_GRANT,
} RevokeRoleGrantAction;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし

前提条件と不変条件

  • plan_single_revoke 関数等において、メンバーシップの更新計画を立てるために使用されます。
  • 各列挙値は、pg_auth_members カタログに対する特定の更新操作（オプションの削除またはレコード自体の削除）に対応します。

アルゴリズムまたは振る舞い

  • RRG_NOOP: 何も行わない。
  • RRG_REMOVE_ADMIN_OPTION: admin_option を無効化する。
  • RRG_REMOVE_INHERIT_OPTION: inherit_option を無効化する。
  • RRG_REMOVE_SET_OPTION: set_option を無効化する。
  • RRG_DELETE_GRANT: pg_auth_members から該当するメンバーシップレコードを削除する。

呼び出し元と呼び出し先

  • 呼び出し元: initialize_revoke_actions および plan_single_revoke がこの型を配列として使用しています。
  • 呼び出し先: 情報なし。

関連データ構造

  • CatCList: メンバーシップリストの管理に使用されます。
  • GrantRoleOptions: GRANT 時に指定されるオプションを保持する構造体。

失敗モードと境界条件

  • 該当するアクションが定義されていない場合、または無効なアクションが渡された場合の挙動については、本ソース excerpt からは特定できません。

保守担当者向けメモ

  • 本列挙型は src/backend/commands/user.c 内の再帰的な権限剥奪ロジックにおいて、どの属性を剥奪するかを決定するために不可欠です。
  • メンバーシップのオプション（ADMIN, INHERIT, SET）が増減した場合は、この列挙型を更新する必要があります。
