setup_simple_rel_arrays

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/optimizer/util/relnode.c:93

目的

プランナの初期化フェーズにおいて、ベースリレーション（RelOptInfo）および関連情報へ高速にアクセスするための配列を確保・初期化する。

シグネチャまたは形状

void setup_simple_rel_arrays(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root（プランナのグローバル状態を保持する構造体）
  • **出力**: なし（root 構造体内の配列フィールドを直接更新する）

前提条件と不変条件

  • root->parse->rtable が既に存在していること。
  • root->simple_rel_array は初期化時にすべて NULL で埋められる。
  • root->simple_rte_array は rtable のインデックス（1始まり）と一致するようにマッピングされる。

アルゴリズムまたは振る舞い

1. rtable のリスト長に基づき、必要な配列サイズ（size = list_length + 1）を算出する。 2. root->simple_rel_array を palloc0 で確保し、NULL で初期化する。 3. root->simple_rte_array を確保し、rtable 内の各 RangeTblEntry をインデックス順に格納する。 4. root->append_rel_list が空であれば終了する。 5. root->append_rel_list が存在する場合、root->append_rel_array を確保し、各 AppendRelInfo を child_relid をインデックスとして格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planmain.c
  • src/backend/optimizer/plan/planner.c
  • src/backend/optimizer/prep/prepunion.c
  • **呼び出し先**:
  • palloc0（メモリ確保）
  • elog（エラー報告）

関連データ構造

  • PlannerInfo: プランナのメイン状態保持構造体。
  • RelOptInfo: ベースリレーション情報。
  • RangeTblEntry: レンジテーブルエントリ。
  • AppendRelInfo: 継承やUNION ALLに関連する情報。

失敗モードと境界条件

  • root->append_rel_array への格納時に、同一の child_relid に対して既にエントリが存在する場合、elog(ERROR) により処理が中断される。
  • child_relid が確保した配列サイズを超過する場合、Assert により検証される。

保守担当者向けメモ

  • 本関数はプランナの初期化フローにおいて重要な役割を果たす。
  • append_rel_array は継承展開の過程で更新される可能性があるため、拡張コード側での適切な管理が求められる。
