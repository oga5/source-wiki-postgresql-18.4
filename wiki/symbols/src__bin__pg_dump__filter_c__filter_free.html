filter_free

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/filter.c:59

目的

FilterStateData 構造体に関連付けられたリソース（動的バッファおよびファイルポインタ）を解放し、状態をクリーンアップします。

シグネチャまたは形状

void filter_free(FilterStateData *fstate)

入力と出力

  • **入力**: FilterStateData *fstate - 解放対象のフィルタ状態データへのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • fstate が NULL の場合、関数は何もせず正常に終了します。
  • fstate->linebuff.data は free() 可能なメモリ領域を指している必要があります。
  • fstate->fp が stdin 以外の有効なファイルポインタである場合、そのクローズが試行されます。

アルゴリズムまたは振る舞い

1. fstate が NULL であれば直ちに復帰します。 2. fstate->linebuff.data を解放し、ポインタを NULL に設定します。 3. fstate->fp が存在し、かつ stdin ではない場合、fclose() を実行します。

  • クローズに失敗した場合は pg_log_error を使用してエラーメッセージを出力します。

    • 4. fstate->fp を NULL に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c:20426
  • src/bin/pg_dump/pg_dumpall.c:1886
  • src/bin/pg_dump/pg_restore.c:708
  • **呼び出し先**:
  • free (標準ライブラリ)
  • fclose (標準ライブラリ)
  • pg_log_error (共通ログ出力)

関連データ構造

  • FilterStateData (src/bin/pg_dump/filter.h で定義)

失敗モードと境界条件

  • fstate が NULL の場合: 安全に無視されます。
  • fclose の失敗: pg_log_error を介してエラーが報告されますが、プログラムの実行は継続されます。
  • stdin の扱い: fstate->fp が stdin である場合、この関数はファイルをクローズしません。

保守担当者向けメモ

  • 本関数は FilterStateData の所有権を解放する役割を担います。filter_init で確保されたリソースと対になるように設計されています。
  • pg_log_error を使用しているため、呼び出し元で適切なログ設定が行われていることを前提としています。
