ArchiveEntry

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_archiver.c:1239

目的

ダンプ対象のオブジェクト（テーブル、関数等）を表す TocEntry 構造体を新規作成し、アーカイブの目次（TOC）リストに登録する。

シグネチャまたは形状

TocEntry *ArchiveEntry(Archive *AHX, CatalogId catalogId, DumpId dumpId, ArchiveOpts *opts)

入力と出力

  • **入力**:
  • AHX: アーカイブハンドル（ArchiveHandle へのポインタ）。
  • catalogId: オブジェクトのカタログ ID。
  • dumpId: オブジェクトのダンプ ID。
  • opts: オブジェクトのメタデータ（タグ、名前空間、SQL 文、依存関係等）を含むオプション構造体。
  • **出力**:
  • 新規作成され、TOC リストにリンクされた TocEntry 構造体へのポインタ。

前提条件と不変条件

  • AHX は有効な ArchiveHandle である必要がある。
  • opts 内の文字列フィールドは、pg_strdup によりコピーされるため、呼び出し元で解放可能である。
  • AH->toc は循環リストのヘッドとして初期化されている必要がある。

アルゴリズムまたは振る舞い

1. pg_malloc0 を使用して TocEntry をメモリ確保する。 2. AH->tocCount をインクリメントし、AH->maxDumpId を更新する。 3. 新規エントリを AH->toc が指す循環リストの末尾に追加する。 4. opts から catalogId、dumpId、section、各種文字列（tag, namespace, tablespace 等）をコピーする。 5. opts->nDeps > 0 の場合、依存関係リストをメモリ確保し、内容をコピーする。 6. ダンプ用関数（dataDumper）および定義用関数（defnDumper）のポインタと引数を設定する。 7. AH->ArchiveEntryPtr が設定されている場合、その関数を呼び出してフォーマット固有の処理を実行する。 8. 構築した newToc を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c 内の各ダンプ処理関数（テーブル、マテリアライズドビューのダンプ時など）、および src/bin/pg_dump/common.c。
  • **呼び出し先**: AH->ArchiveEntryPtr（アーカイブ形式に応じたコールバック関数）。

関連データ構造

  • TocEntry: 本関数が生成する主要な構造体。
  • ArchiveHandle: アーカイブの状態と TOC リストを保持する。
  • ArchiveOpts: オブジェクト生成に必要なメタデータ群。

失敗モードと境界条件

  • メモリ不足時: pg_malloc0 および pg_malloc が失敗した場合、プロセスは終了する可能性がある。
  • opts->nDeps が 0 の場合: 依存関係リストは NULL に初期化される。

保守担当者向けメモ

  • 本関数は pg_dump のオブジェクト登録の要所である。TOC の順序や依存関係の不整合をデバッグする際は、ここでの dumpId の割り当てとリストへの挿入ロジックを確認すること。
  • AH->ArchiveEntryPtr は _allocAH 等で初期化されるフォーマット固有の関数であるため、特定のアーカイブ形式（tar, custom 等）で挙動が異なる場合は、その実装を確認すること。
