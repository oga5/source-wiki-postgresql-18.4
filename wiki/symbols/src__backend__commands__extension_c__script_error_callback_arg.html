script_error_callback_arg

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/extension.c:123

目的

拡張機能のインストールまたは更新スクリプト実行中にエラーが発生した際、エラーコンテキスト情報（ファイル名や位置）を報告するためのコールバック引数として使用されます。

シグネチャまたは形状

typedef struct
{
	const char *sql;			/* スクリプトファイル全体の内容 */
	const char *filename;		/* スクリプトファイルのパス */
	ParseLoc	stmt_location;	/* 現在の文の開始位置（不明な場合は -1） */
	ParseLoc	stmt_len;		/* バイト単位の長さ（0は文字列の末尾までを意味する） */
} script_error_callback_arg;

入力と出力

  • **入力**: 実行中のスクリプト内容、ファイルパス、および現在の解析位置情報。
  • **出力**: エラー発生時に error_context_stack を通じて、ユーザーに対して詳細なエラー位置情報を提供します。

前提条件と不変条件

  • sql フィールドは、スクリプト実行中、有効なメモリ領域を指している必要があります。
  • stmt_location が有効な場合、エラー報告時にその位置が特定されます。

アルゴリズムまたは振る舞い

1. 拡張機能のスクリプト実行処理において、この構造体が初期化されます。 2. エラーが発生すると、登録されたエラーコールバック関数がこの構造体を参照し、エラーメッセージにファイル名と位置情報を付加します。 3. stmt_len が 0 の場合、解析エンジンは文字列の終端までを対象として処理します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/extension.c 内のスクリプト実行ロジック（execute_extension_script 等）において、エラーハンドリングの準備としてインスタンス化されます。
  • **呼び出し先**: PostgreSQL のエラー報告メカニズム（errcontext 等）から参照されます。

関連データ構造

  • ParseLoc: SQL文の解析位置を保持する型。

失敗モードと境界条件

  • stmt_location が -1 の場合、エラー報告において正確な位置情報が表示されない可能性があります。
  • sql ポインタが指すメモリが解放された後にエラーが発生した場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は extension.c 内のスクリプト実行時のエラーコンテキストを管理する専用の構造体です。
  • 拡張機能のスクリプト実行処理を修正する際は、エラー発生時にこの構造体の各フィールドが正しく更新されていることを確認してください。
