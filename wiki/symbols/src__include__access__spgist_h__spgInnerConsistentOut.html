spgInnerConsistentOut

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgist.h:154

目的

SP-GiSTインデックスの内部ノード走査時、検索条件に合致する子ノードの特定結果を保持するために使用される出力構造体です。

シグネチャまたは形状

typedef struct spgInnerConsistentOut
{
	int			nNodes;			/* 訪問する子ノードの数 */
	int		   *nodeNumbers;	/* ノード配列内のインデックス */
	int		   *levelAdds;		/* 各ノードのレベル増分 */
	Datum	   *reconstructedValues;	/* 関連する再構築値 */
	void	  **traversalValues;	/* 演算子クラス固有の走査値 */
	double	  **distances;		/* 関連する距離情報 */
} spgInnerConsistentOut;

入力と出力

本構造体は、spgInnerConsistent サポート関数が検索結果を呼び出し元へ返すための出力コンテナとして機能します。

前提条件と不変条件

  • nNodes は、走査対象として特定された子ノードの数を表します。
  • nodeNumbers、levelAdds、reconstructedValues、traversalValues、distances は、少なくとも nNodes 分の要素を保持する配列である必要があります。

アルゴリズムまたは振る舞い

SP-GiSTの検索処理において、spgInnerConsistent 関数は現在のノードを評価し、次に探索すべき子ノードを本構造体に格納します。呼び出し元は、この構造体を参照してインデックスツリーの降下を継続します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgscan.c 内の検索処理。
  • **呼び出し先**: 演算子クラスの実装（spgkdtreeproc.c, spgquadtreeproc.c, spgtextproc.c, geo_spgist.c, network_spgist.c, rangetypes_spgist.c, spgist_name_ops.c など）。

関連データ構造

  • spgInnerConsistentIn: 検索時の入力構造体。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • nNodes が 0 の場合、検索条件に合致する子ノードが存在しないことを示します。
  • メモリ割り当ての失敗や、配列ポインタの初期化不足は、セグメンテーション違反を引き起こす可能性があります。

保守担当者向けメモ

  • traversalValues は、検索中にノード間で状態を引き継ぐために使用されます。複雑な検索アルゴリズムを実装する際は、このポインタの管理が重要です。
  • 演算子クラスを新規追加する際は、spgvalidate を通じて本構造体を含むサポート関数の実装が検証されます。
