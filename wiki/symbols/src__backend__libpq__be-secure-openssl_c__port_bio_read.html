port_bio_read

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-openssl.c:911

目的

OpenSSL の BIO インターフェースを PostgreSQL の Port 構造体（ソケット）に適合させ、TLS 通信におけるデータ読み込み処理を抽象化する。

シグネチャまたは形状

static int port_bio_read(BIO *h, char *buf, int size);

入力と出力

  • **入力**:
  • h: OpenSSL の BIO オブジェクト。内部データとして Port 構造体を保持。
  • buf: 読み込み先バッファ。
  • size: 読み込み要求サイズ。
  • **出力**:
  • 読み込んだバイト数、またはエラーを示す整数値。

前提条件と不変条件

  • BIO_get_data(h) により、有効な Port 構造体が取得できること。
  • buf が NULL でない場合にのみ読み込み処理が実行される。

アルゴリズムまたは振る舞い

1. BIO オブジェクトから Port 構造体を取得する。 2. buf が非 NULL の場合、secure_raw_read を呼び出してソケットからデータを読み込む。 3. BIO_clear_retry_flags(h) を呼び出し、既存の再試行フラグをクリアする。 4. 読み込み結果が 0 の場合、port->last_read_was_eof を true に設定する。 5. 読み込み結果が 0 以下の場合、errno が EINTR、EWOULDBLOCK、または EAGAIN であれば、BIO_set_retry_read(h) を呼び出し、OpenSSL に対して再試行を要求する。 6. 最終的な読み込みバイト数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: OpenSSL ライブラリ内部（BIO_read 等を介して間接的に呼び出される）。
  • **呼び出し先**: secure_raw_read（PostgreSQL のソケット読み込み関数）。
  • **注記**: ソースコード内の直接的な呼び出し元は src/backend/libpq/be-secure-openssl.c:999 にて BIO_meth_set_read に登録される箇所が確認できる。

関連データ構造

  • BIO: OpenSSL の I/O 抽象化構造体。
  • Port: PostgreSQL の接続状態を保持する構造体。

失敗モードと境界条件

  • buf が NULL の場合、読み込み処理はスキップされ 0 が返される。
  • ソケットが非ブロッキング状態や割り込み発生時に errno が設定された場合、BIO_set_retry_read が呼び出され、OpenSSL の再試行メカニズムが有効になる。

保守担当者向けメモ

  • 本関数は port_bio_method_ptr の設定を通じて OpenSSL に渡される。
  • secure_raw_read の挙動に依存するため、ソケット I/O の変更時は本関数のエラーハンドリングと整合性を確認すること。
