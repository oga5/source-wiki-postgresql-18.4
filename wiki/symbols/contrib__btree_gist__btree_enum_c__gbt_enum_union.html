gbt_enum_union

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_enum.c:145

目的

GiST インデックス構築時において、複数の列挙型（ENUM）キーを包含する最小の範囲（oidKEY）を計算します。

シグネチャまたは形状

Datum gbt_enum_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GistEntryVector（インデックスエントリの集合）および internal ポインタ（サイズ格納用）を受け取ります。
  • **出力**: 複数の入力キーをカバーする新しい oidKEY 構造体へのポインタを Datum として返します。

前提条件と不変条件

  • ENUM 型は内部的に OID として扱われるため、oidKEY 構造体（lower, upper）を使用して範囲を表現します。
  • tinfo（gbtree_ninfo 構造体）が ENUM 型の比較ロジックを保持している必要があります。

アルゴリズムまたは振る舞い

1. palloc を使用して、結果を格納するための oidKEY 領域を確保します。 2. 第2引数に oidKEY のサイズを書き込みます。 3. btree_utils_num.c で定義された汎用関数 gbt_num_union を呼び出し、tinfo を用いてエントリベクトルの和集合を計算します。 4. 計算結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス構築プロセス（直接の呼び出し元はソースコード上では明示されていません）。
  • **呼び出し先**: gbt_num_union（btree_utils_num.c 内の汎用関数）。

関連データ構造

  • oidKEY: ENUM の範囲（下限・上限）を保持する構造体。
  • tinfo: ENUM 型の比較関数やサイズ情報を保持する gbtree_ninfo 構造体。
  • GistEntryVector: GiST インデックスの複数のエントリを保持する構造体。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性があります。
  • 入力された GistEntryVector が空の場合の挙動は、依存先の gbt_num_union の実装に依存します。

保守担当者向けメモ

  • 本関数は oidKEY を用いた OID 範囲計算のラッパーです。ロジックの変更が必要な場合は、contrib/btree_gist/btree_utils_num.c の gbt_num_union を参照してください。
  • tinfo 構造体は btree_enum.c 内で定義されており、本関数が正しく動作するための比較関数ポインタを提供しています。
