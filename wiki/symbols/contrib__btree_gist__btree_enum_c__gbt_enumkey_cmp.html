gbt_enumkey_cmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_enum.c:66

目的

GiST インデックスの内部ノードにおける oidKEY（ENUM 型の範囲）のソート順を決定し、インデックス構築時の効率的な並び替えをサポートします。

シグネチャまたは形状

static int gbt_enumkey_cmp(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の Nsrt 構造体へのポインタ。内部に oidKEY を保持します。
  • flinfo: PostgreSQL の関数マネージャー情報。
  • **出力**:
  • int: 比較結果（a < b なら負、a == b なら 0、a > b なら正）。

前提条件と不変条件

  • 入力 a および b は Nsrt 構造体へのポインタとしてキャスト可能であること。
  • flinfo は enum_cmp 関数を呼び出すために有効であること。
  • ENUM 型は内部的に OID として扱われ、順序付けが可能であること。

アルゴリズムまたは振る舞い

1. a と b からそれぞれ oidKEY（lower, upper）を取り出します。 2. lower 値を比較します。値が異なる場合は、enum_cmp を呼び出してその結果を返します。 3. lower 値が等しい場合、upper 値を比較します。

  • upper 値も等しければ 0 を返します。
  • upper 値が異なれば、enum_cmp を呼び出してその結果を返します。

    • 4. 比較には CallerFInfoFunctionCall2 を介して PostgreSQL 標準の enum_cmp 関数を使用します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_enum.c 内の静的初期化子（tinfo 構造体の定義箇所）。
  • **呼び出し先**: enum_cmp（PostgreSQL 標準関数）。

関連データ構造

  • oidKEY: ENUM の範囲（lower, upper）を保持する構造体。
  • Nsrt: ソート処理用のラッパー構造体。

失敗モードと境界条件

  • flinfo が無効な場合、CallerFInfoFunctionCall2 の実行により予期せぬ動作が発生する可能性があります。
  • ENUM 型の順序付けが定義されていない場合、enum_cmp の挙動に依存します。

保守担当者向けメモ

  • 本関数は GiST インデックスの内部ノードのソート順を決定する重要な役割を担います。
  • tinfo 構造体を通じて gbt_enum_sortsupport 等から間接的に利用されます。
