ColumnIOData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/rowtypes.c:34

目的

複合型やレコードの各列（フィールド）に対する入出力処理に必要なメタデータおよび関数情報を保持し、実行時のキャッシュとして機能します。

シグネチャまたは形状

typedef struct ColumnIOData
{
	Oid			column_type;    /* 列のデータ型OID */
	Oid			typiofunc;      /* 入出力関数のOID */
	Oid			typioparam;     /* 入出力関数に渡すパラメータ */
	bool		typisvarlena;   /* 可変長型かどうか */
	FmgrInfo	proc;           /* 関数マネージャ情報 */
} ColumnIOData;

入力と出力

本構造体は関数呼び出しのライフサイクル内で fn_extra にキャッシュされるデータの一部として利用されます。直接的な入力・出力関数ではなく、レコード内の各フィールドを処理するための設定情報を提供します。

前提条件と不変条件

  • proc フィールドは、fmgr_info 等を用いて適切に初期化されている必要があります。
  • 複合型の構造（TupleDesc）が変更された場合、キャッシュされた本構造体は無効化される必要があります。

アルゴリズムまたは振る舞い

  • レコードの入出力処理において、各フィールドの型に応じたI/O関数をルックアップし、その結果を本構造体に格納します。
  • 以降の処理では、このキャッシュされた FmgrInfo を使用して、各フィールドの値を効率的に変換します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/rowtypes.c 内のレコード入出力関数（record_in, record_out 等）が、フィールド処理の初期化時に本構造体を利用します。
  • **その他**: contrib/hstore/hstore_io.c や src/backend/utils/adt/jsonfuncs.c においても、同様の目的で本構造体が参照または定義されています。

関連データ構造

  • RecordIOData: 複合型全体の入出力情報を管理する構造体であり、ColumnIOData を配列として保持することがあります。
  • TupleDesc: レコードの構造定義。

失敗モードと境界条件

  • 削除された列（attisdropped）が含まれる場合、本構造体を用いた処理ではスキップされる必要があります。
  • 型キャッシュのルックアップに失敗した場合、システムエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は rowtypes.c 以外でも利用されているため、変更を加える際は contrib モジュールや jsonfuncs.c への影響を考慮してください。
  • FmgrInfo を含むため、メモリ管理（特に MemoryContext の生存期間）には注意が必要です。
