explain_ExecutorStart

定義

  • 種別: function
  • 定義箇所: contrib/auto_explain/auto_explain.c:264

目的

PostgreSQL のクエリ実行開始時にフックし、実行計画の統計情報収集の準備およびサンプリング判定を行う。

シグネチャまたは形状

static void explain_ExecutorStart(QueryDesc *queryDesc, int eflags)

入力と出力

  • **入力**:
  • queryDesc: 実行対象のクエリ情報。
  • eflags: 実行フラグ（EXEC_FLAG_EXPLAIN_ONLY 等）。
  • **出力**: なし（副作用として queryDesc のインストゥルメンテーション設定を更新）。

前提条件と不変条件

  • nesting_level が 0 の場合のみ、サンプリング判定が行われる。
  • パラレルワーカー内ではサンプリング判定をスキップし、ログ出力対象外とする。
  • queryDesc->totaltime が未割り当ての場合、クエリコンテキスト上に計測用メモリを確保する。

アルゴリズムまたは振る舞い

1. **サンプリング判定**: トップレベルのクエリであれば、auto_explain_sample_rate に基づき current_query_sampled を更新する。 2. **インストゥルメンテーション設定**: auto_explain_enabled() が真かつ log_analyze が有効な場合、queryDesc->instrument_options に計測フラグ（タイマー、行数、バッファ、WAL）を追加する。 3. **フックの連鎖**: 既存の prev_ExecutorStart があればそれを呼び出し、なければ standard_ExecutorStart を呼び出す。 4. **計測準備**: ログ出力対象であれば、queryDesc->totaltime を初期化し、実行時間の計測準備を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の実行エンジン（ExecutorStart_hook を介して呼び出される）。
  • **呼び出し先**:
  • prev_ExecutorStart または standard_ExecutorStart
  • InstrAlloc（計測用メモリ確保）
  • MemoryContextSwitchTo（メモリコンテキスト切り替え）

関連データ構造

  • QueryDesc: 実行中のクエリ情報と実行計画を保持。
  • Instrumentation: 実行統計情報を格納する構造体。

失敗モードと境界条件

  • nesting_level > 0 の場合、サンプリング判定は行われず、親クエリの判定結果を継承する。
  • IsParallelWorker() が真の場合、サンプリング判定を強制的に false とし、二重ログ出力を防止する。

保守担当者向けメモ

  • 本関数は ExecutorStart_hook に登録される。他の拡張モジュールとの競合を避けるため、必ず prev_ExecutorStart を呼び出すこと。
  • queryDesc->totaltime のメモリ確保は queryDesc->estate->es_query_cxt 上で行うことで、クエリ終了時の自動解放を保証している。
