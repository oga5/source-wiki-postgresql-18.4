SerialControlData

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/lmgr/predicate.c:345

目的

直列化可能スナップショット分離（SSI）において、SLRU（Simple Least Recently Used）キャッシュで管理されるトランザクション情報の制御状態を保持するために使用されます。

シグネチャまたは形状

typedef struct SerialControlData
{
    int64       headPage;       /* 最新の初期化済みページ */
    TransactionId headXid;      /* SLRU 内の最新の有効な Xid */
    TransactionId tailXid;      /* 追跡対象となる最も古い xmin */
} SerialControlData;

入力と出力

  • 本構造体は共有メモリ上に配置され、SSI 関連のトランザクション管理プロセスによって読み書きされます。

前提条件と不変条件

  • 共有メモリ上の構造体であるため、アクセス時には適切なロック（predicate.c 内のコメントによれば、特定のロックによる保護が想定される）が必要です。

アルゴリズムまたは振る舞い

  • headPage を通じて SLRU の最新位置を追跡します。
  • headXid および tailXid を用いて、SLRU 内で保持すべきトランザクション ID の範囲を定義し、古いトランザクション情報の管理を行います。

呼び出し元と呼び出し先

  • 呼び出し元: ShmemInitStruct（共有メモリ初期化時）および add_size（メモリサイズ計算時）で参照されています。
  • 呼び出し先: 直接的な関数呼び出し関係はソース抜粋からは特定できません。

関連データ構造

  • SerialControl: SerialControlData へのポインタ型として定義されています。

失敗モードと境界条件

  • 共有メモリの初期化失敗や、SLRU のページ管理範囲を超えたトランザクション ID の参照が発生した場合の挙動については、本ソース抜粋からは情報が得られません。

保守担当者向けメモ

  • 本構造体は src/backend/storage/lmgr/predicate.c 内の静的データ管理に使用されています。
  • 変更を加える際は、SLRU のページングロジックおよびトランザクション ID の有効範囲管理への影響を考慮してください。
