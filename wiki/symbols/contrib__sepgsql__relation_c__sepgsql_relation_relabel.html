sepgsql_relation_relabel

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/relation.c:563

目的

リレーション（テーブル、シーケンス、ビュー）に対する SECURITY LABEL コマンド実行時に、SELinuxのアクセス制御権限を検証する。

シグネチャまたは形状

void sepgsql_relation_relabel(Oid relOid, const char *seclabel)

入力と出力

  • relOid (Oid): 対象リレーションのオブジェクト識別子。
  • seclabel (const char *): 設定しようとしている新しいセキュリティラベル文字列。
  • 戻り値: なし。権限チェックに失敗した場合はエラーをスローする。

前提条件と不変条件

  • 対象オブジェクトはテーブル、パーティションテーブル、シーケンス、またはビューである必要がある。それ以外の場合は ERRCODE_WRONG_OBJECT_TYPE でエラーとなる。
  • 呼び出し元は、対象オブジェクトに対する SETATTR および RELABELFROM 権限、および新しいラベルに対する RELABELTO 権限を保持している必要がある。

アルゴリズムまたは振る舞い

1. get_rel_relkind を使用してリレーションの種類を判定し、対応するSELinuxクラス（db_table, db_sequence, db_view）を決定する。 2. getObjectIdentity を呼び出し、監査ログ用のオブジェクト名を取得する。 3. sepgsql_avc_check_perms を呼び出し、現在のクライアントが対象オブジェクトに対して SETATTR および RELABELFROM 権限を持つか検証する。 4. sepgsql_avc_check_perms_label を呼び出し、新しいラベルに対して RELABELTO 権限を持つか検証する。 5. 最後に pfree で監査用文字列のメモリを解放する。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/sepgsql/label.c:509 からの呼び出しが確認されている。
  • 呼び出し先: get_rel_relkind, getObjectIdentity, sepgsql_avc_check_perms, sepgsql_avc_check_perms_label, pfree。

関連データ構造

  • ObjectAddress: 対象リレーションを特定するために使用。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • サポート外のリレーション種別（インデックス等）が渡された場合、エラーが発生する。
  • SELinuxポリシーによる権限チェックで拒否された場合、sepgsql_avc_check_perms または sepgsql_avc_check_perms_label 内でエラーが発生する。

保守担当者向けメモ

  • 本関数は SECURITY LABEL コマンドのフックとして機能する。
  • インデックスは独立したオブジェクトとして扱われず、親テーブルの属性として処理されるため、本関数はインデックスに対しては直接使用されない。
