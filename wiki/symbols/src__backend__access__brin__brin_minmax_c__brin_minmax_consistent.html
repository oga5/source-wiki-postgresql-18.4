brin_minmax_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_minmax.c:136

目的

BRINインデックスのページ範囲（BrinValues）が、指定されたスキャンキー（ScanKey）の条件を満たす可能性があるかを判定する。

シグネチャまたは形状

Datum brin_minmax_consistent(PG_FUNCTION_ARGS)
  • 引数: PG_FUNCTION_ARGS (内部的に BrinDesc*, BrinValues*, ScanKey を取得)
  • 戻り値: Datum (bool値として解釈される一致判定結果)

入力と出力

  • 入力:
  • bdesc: インデックス記述子。
  • column: 評価対象のページ範囲要約情報。
  • key: 検索条件（演算子、属性番号、比較値）。
  • 出力:
  • bool: 範囲内に条件を満たすデータが存在し得る場合は true、そうでない場合は false。

前提条件と不変条件

  • PG_NARGS() == 3 であること。
  • column->bv_allnulls が false であること（NULLのみの範囲は対象外）。
  • minmax_get_strategy_procinfo により、適切な比較関数が取得可能であること。

アルゴリズムまたは振る舞い

スキャンキーの戦略番号（sk_strategy）に基づき、以下の通り比較を行う： 1. **< または <=**: 範囲の最小値（bv_values[0]）とキー値を比較。 2. **> または >=**: 範囲の最大値（bv_values[1]）とキー値を比較。 3. **=**: 最小値がキー値以下、かつ最大値がキー値以上であるかを確認する（2段階の比較）。 4. 無効な戦略番号が指定された場合は elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内には明示的な呼び出し元の記述なし。
  • 呼び出し先:
  • minmax_get_strategy_procinfo: 比較関数の取得。
  • FunctionCall2Coll: 取得した比較関数の実行。

関連データ構造

  • BrinValues: ページ範囲の要約情報（最小値・最大値を保持）。
  • ScanKey: 検索条件の定義。
  • FmgrInfo: 比較演算子の実行情報。

失敗モードと境界条件

  • bv_allnulls が true の場合、アサーション違反により停止する。
  • 未定義の戦略番号が渡された場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は古い3引数シグネチャを使用している。
  • minmax_get_strategy_procinfo は inclusion_get_strategy_procinfo と設計が共通化されているため、修正時は両方のファイルを確認すること。
