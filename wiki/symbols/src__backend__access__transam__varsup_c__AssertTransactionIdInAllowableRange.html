AssertTransactionIdInAllowableRange

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/varsup.c:672

目的

開発ビルドにおいて、指定されたトランザクションID（XID）がシステム上の有効な範囲（oldestXid から nextXid の間）にあるかを検証するデバッグ用関数です。

シグネチャまたは形状

void AssertTransactionIdInAllowableRange(TransactionId xid)

入力と出力

  • **入力**: xid (検証対象の TransactionId)
  • **出力**: なし（範囲外の場合は Assert によりプログラムが停止）

前提条件と不変条件

  • xid は有効なトランザクションIDである必要があります。
  • xid が BootstrapTransactionId や FrozenTransactionId などの非正規なIDである場合、検証はスキップされます。
  • 共有メモリ上の TransamVariables にアクセスするため、メモリバリア（pg_memory_barrier）を介して最新の oldestXid および nextXid を取得します。

アルゴリズムまたは振る舞い

1. TransactionIdIsValid(xid) を確認し、無効な場合はアサート失敗。 2. TransactionIdIsNormal(xid) を確認し、正規のXIDでなければ即座にリターン。 3. pg_memory_barrier() を実行し、共有メモリ上の最新状態を読み込む。 4. TransamVariables から oldestXid と nextXid を取得。 5. xid が oldest_xid 以上かつ next_xid 以下であることを検証。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/varsup.c:673
  • src/backend/storage/ipc/procarray.c:4329
  • **呼び出し先**:
  • TransactionIdIsValid (マクロ)
  • TransactionIdIsNormal (マクロ)
  • XidFromFullTransactionId (マクロ)
  • TransactionIdFollowsOrEquals (マクロ)
  • TransactionIdPrecedesOrEquals (マクロ)

関連データ構造

  • TransamVariables: 共有メモリ上のトランザクション状態管理構造体。
  • TransactionId: 32bitのトランザクション識別子。

失敗モードと境界条件

  • **ロックの競合**: 本関数は XidGenLock を取得しません。これは、呼び出し元が既にロックを保持している可能性があるためです。そのため、nextXid の読み取りはアトミックな32bit読み取りに依存しています。
  • **範囲の判定**: xid が next_xid と等しい場合も許容されます。これは、ProcArray から取得したXIDが、nextXid が更新される直前の値である可能性があるためです。

保守担当者向けメモ

  • 本関数は開発ビルドでのみ有効なデバッグツールです。
  • nextXid は64bit（FullTransactionId）ですが、本関数では下位32bitのみを比較対象としています。
  • 呼び出し元でロックを保持している場合でも安全に動作するように設計されています。
