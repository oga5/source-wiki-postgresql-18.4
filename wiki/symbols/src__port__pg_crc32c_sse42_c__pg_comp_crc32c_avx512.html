pg_comp_crc32c_avx512

定義

  • 種別: function
  • 定義箇所: src/port/pg_crc32c_sse42.c:94

目的

AVX-512 命令セット（vpclmulqdq 等）を活用し、大規模データに対する CRC-32C チェックサム計算を高速化する。

シグネチャまたは形状

pg_crc32c pg_comp_crc32c_avx512(pg_crc32c crc, const void *data, size_t len)

入力と出力

  • **入力**:
  • crc: 初期 CRC 値。
  • data: 計算対象のデータバッファ。
  • len: データ長（バイト単位）。
  • **出力**:
  • 計算された pg_crc32c 型のチェックサム値。

前提条件と不変条件

  • 実行環境の CPU が AVX-512 命令セットをサポートしている必要がある。
  • 不正命令例外を避けるため、呼び出し前に CPU 機能フラグによる実行時判定が必須である。

アルゴリズムまたは振る舞い

1. **アライメント調整**: len > 256 の場合、キャッシュライン境界に合わせるため、先頭の端数バイトを _mm_crc32_u8 および _mm_crc32_u64 で処理する。 2. **ベクトル演算**: len >= 64 の場合、64 バイト単位で vpclmulqdq を含む AVX-512 命令を用いて並列計算を行う。 3. **削減処理**: 512 ビットの計算結果を 128 ビットへ、さらに 32 ビットへと段階的に削減する。 4. **残余処理**: 最後に残った端数バイトおよび削減後の値を pg_comp_crc32c_sse42 に渡し、最終的な CRC 値を算出する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/pg_crc32c_sse42_choose.c（CPU 機能判定に基づく動的選択）。
  • **呼び出し先**: pg_comp_crc32c_sse42（残余処理用）。

関連データ構造

  • pg_crc32c: チェックサム値を保持する型。
  • __m512i, __m128i: AVX-512/SSE 用のベクトル型。

失敗モードと境界条件

  • AVX-512 非対応の CPU で実行された場合、不正命令例外が発生する。
  • データ長が 64 バイト未満の場合、ベクトル演算をスキップし、直接 pg_comp_crc32c_sse42 へ処理が委譲される。

保守担当者向けメモ

  • 本関数は USE_AVX512_CRC32C_WITH_RUNTIME_CHECK 定義時のみ有効となる。
  • AVX-512 実装は外部生成コードを含んでおり、特定の最適化（vpclmulqdq）に強く依存している。
  • アライメント処理は、明示的なメモリ配置よりも命令パイプラインの効率を優先している。
