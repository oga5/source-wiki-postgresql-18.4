pg_checksum_context

定義

  • 種別: struct
  • 定義箇所: src/include/common/checksum_helper.h:52

目的

PostgreSQL全体で利用される、多様なチェックサムアルゴリズムを統一的に扱うための抽象化レイヤーにおける、計算状態の保持構造体です。

シグネチャまたは形状

typedef struct pg_checksum_context
{
	pg_checksum_type type;
	pg_checksum_raw_context raw_context;
} pg_checksum_context;

入力と出力

  • **入力**: pg_checksum_init 関数により、特定の pg_checksum_type で初期化されます。
  • **出力**: pg_checksum_final 関数を介して、計算結果がバッファへ書き出されます。

前提条件と不変条件

  • pg_checksum_final を呼び出すことで、内部的に確保されたリソース（pg_cryptohash_ctx 等）が解放される設計です。呼び出し忘れはリソースリークの原因となります。
  • 本構造体は、初期化、更新、最終化という一貫したライフサイクル管理下で使用される必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: pg_checksum_init でアルゴリズムを指定し、内部状態を準備します。 2. **更新**: pg_checksum_update を繰り返し呼び出し、計算対象のデータを逐次処理します。 3. **最終化**: pg_checksum_final により計算を完了し、内部リソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • src/backend/backup/backup_manifest.c
  • src/backend/backup/basebackup.c
  • src/bin/pg_combinebackup/backup_label.c
  • src/bin/pg_combinebackup/copy_file.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、具体的な計算処理は src/common/checksum_helper.c 内の関数群（pg_checksum_init 等）が raw_context を操作することで行われます。

関連データ構造

  • pg_checksum_type: 使用するアルゴリズムを識別する列挙型。
  • pg_checksum_raw_context: アルゴリズムごとの具体的な内部状態を保持する共用体または構造体。

失敗モードと境界条件

  • MD5は本モジュールではサポートされていません。
  • CRC-32Cは高速ですが暗号学的な安全性はありません。用途に応じたアルゴリズム選択が必要です。
  • pg_checksum_final の呼び出しを怠ると、内部リソースが解放されません。

保守担当者向けメモ

  • 本構造体は src/include/common/checksum_helper.h で定義されており、実装は src/common/checksum_helper.c に存在します。
  • 暗号学的に強力なチェックサムが必要な場合は、SHA-2ファミリーを選択してください。
