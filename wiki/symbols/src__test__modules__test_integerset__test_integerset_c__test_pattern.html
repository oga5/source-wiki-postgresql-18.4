test_pattern

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_integerset/test_integerset.c:130

目的

test_spec 構造体で定義されたパターンに基づき、IntegerSet の構築、メンバ判定、およびイテレーションの正確性を検証する。

シグネチャまたは形状

static void test_pattern(const test_spec *spec)

入力と出力

  • 入力: spec (テストパターンを定義する test_spec 構造体へのポインタ)
  • 出力: なし（検証失敗時は elog(ERROR) により異常終了）

前提条件と不変条件

  • spec->pattern_str は '0' または '1' からなる文字列であること。
  • intset_test_stats が有効な場合、標準エラー出力に統計情報が出力される。
  • 構築された IntegerSet は、指定された spacing と pattern_str に従って生成された値の集合と一致しなければならない。

アルゴリズムまたは振る舞い

1. **パターン解析**: spec->pattern_str を解析し、'1' の位置を pattern_values 配列に格納する。 2. **メモリ確保**: AllocSetContextCreate を用いて専用のメモリコンテキストを作成し、その中で intset_create() を呼び出す。 3. **データ構築**: spec->num_values に達するまで、spacing を加算しながら intset_add_member を繰り返し呼び出す。 4. **検証**:

  • intset_num_entries で総数を検証。
  • pg_prng_uint64_range を用いたランダムな値に対し、intset_is_member で存在確認を行う。
  • intset_begin_iterate および intset_iterate_next を用いて、全要素が期待通りの順序で取得できるかを確認する。

    • 5. **クリーンアップ**: テスト終了後、確保したメモリコンテキストを解放する（ソースコード上では MemoryContextDelete への言及あり）。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/test_integerset/test_integerset.c 内のメインループ（test_specs 配列の走査時）。
  • 呼び出し先: intset_create, intset_add_member, intset_num_entries, intset_is_member, intset_begin_iterate, intset_iterate_next, intset_memory_usage 等の lib/integerset.h API。

関連データ構造

  • IntegerSet: テスト対象のデータ構造。
  • test_spec: テストのパターン定義（pattern_str, spacing, num_values を保持）。

失敗モードと境界条件

  • intset_num_entries の結果が期待値と異なる場合、elog(ERROR) を送出。
  • intset_is_member の判定結果が期待値と不一致の場合、elog(ERROR) を送出。
  • イテレータが期待する値と異なる値を返した場合、または要素数が一致しない場合、elog(ERROR) を送出。

保守担当者向けメモ

  • intset_test_stats を有効にすると、詳細なメモリ使用量や処理時間が標準エラー出力に表示される。
  • Simple-8b エンコーディングの制約（2^60）に関連する境界条件が含まれているため、実装変更時は test_huge_distances 等のテストケースとの整合性に注意すること。
