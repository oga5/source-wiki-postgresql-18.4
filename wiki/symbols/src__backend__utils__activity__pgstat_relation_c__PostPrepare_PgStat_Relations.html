PostPrepare_PgStat_Relations

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_relation.c:727

目的

2フェーズコミット（2PC）の準備フェーズ完了後、トランザクションに関連付けられたリレーション統計情報の追跡状態をクリーンアップします。

シグネチャまたは形状

void PostPrepare_PgStat_Relations(PgStat_SubXactStatus *xact_state)

入力と出力

  • **入力**: xact_state (PgStat_SubXactStatus *) - 準備されたトランザクションに関連する統計状態のリスト。
  • **出力**: なし（副作用としてメモリ上の状態を更新）。

前提条件と不変条件

  • xact_state は有効なポインタである必要があります。
  • 呼び出し時点で、対象となるリレーション統計エントリのトランザクション状態（trans）は、準備済み状態へ移行済みである必要があります。

アルゴリズムまたは振る舞い

1. xact_state->first から始まる PgStat_TableXactStatus のリストを順次走査します。 2. 各エントリが保持する親リレーション統計構造体（tabstat）を取得します。 3. tabstat->trans ポインタを NULL に設定し、当該リレーションと現在のトランザクション状態との直接的なリンクを解除します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat_xact.c:225
  • **呼び出し先**: 外部関数呼び出しなし。

関連データ構造

  • PgStat_SubXactStatus: トランザクション内の統計状態を管理する構造体。
  • PgStat_TableXactStatus: 特定リレーションのトランザクション内統計状態。
  • PgStat_TableStatus: リレーションの統計情報を保持するメイン構造体。

失敗モードと境界条件

  • xact_state が空（first が NULL）の場合、ループは実行されず何も行われません。
  • tabstat が NULL であるような不正なリスト構造が渡された場合の挙動は未定義です。

保守担当者向けメモ

  • 本関数は、トランザクションが「準備済み（PREPARED）」状態になった後に、ローカルの統計追跡ポインタをクリアするために使用されます。
  • tabstat->trans を NULL に設定することで、以降の統計更新が現在のトランザクションコンテキストから切り離されることを保証します。
