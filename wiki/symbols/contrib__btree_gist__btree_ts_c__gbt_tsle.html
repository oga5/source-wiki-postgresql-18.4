gbt_tsle

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_ts.c:78

目的

2つの Timestamp 値を比較し、第一引数が第二引数以下（less than or equal to）であるかを判定する。GiSTインデックスにおける比較操作の一部として使用される。

シグネチャまたは形状

static bool gbt_tsle(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の Timestamp へのポインタ。
  • b: 比較対象の Timestamp へのポインタ。
  • flinfo: 関数呼び出し情報（本関数内では直接使用されない）。
  • **出力**:
  • bool: *a <= *b であれば true、そうでなければ false を返す。

前提条件と不変条件

  • a および b は有効な Timestamp 型のメモリ領域を指している必要がある。
  • 内部的に timestamp_le 関数を呼び出すため、PostgreSQLのタイムスタンプ比較規則に従う。

アルゴリズムまたは振る舞い

1. 引数 a と b を Timestamp 型のポインタとしてキャストする。 2. TimestampGetDatumFast マクロを使用して、各タイムスタンプを Datum 形式に変換する。 3. DirectFunctionCall2 を介して、PostgreSQL標準の timestamp_le 関数を実行する。 4. 結果を DatumGetBool で論理値に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_ts.c 内の静的構造体（tinfo 等）から参照されている可能性がある（直接的な関数呼び出しの詳細はソース抜粋からは不明）。
  • **呼び出し先**:
  • timestamp_le (PostgreSQL内部関数)

関連データ構造

  • Timestamp: 比較対象となるタイムスタンプ型。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、セグメンテーション違反が発生する可能性がある。
  • timestamp_le の仕様に依存し、無効なタイムスタンプ値が渡された場合の挙動は当該関数に準ずる。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で定義される共通ロジックから呼び出されることを想定したラッパーである。
  • timestamptz 型を扱う場合、呼び出し元で事前にGMTへの正規化が行われている必要がある。
