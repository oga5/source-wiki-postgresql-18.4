process_slot

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/task.c:235

目的

pg_upgrade におけるデータベースタスクの並列実行において、各スロットの状態遷移を管理し、接続の確立からクエリの実行、結果の処理までを非同期に実行する。

シグネチャまたは形状

static void process_slot(const ClusterInfo *cluster, UpgradeTaskSlot *slot, const UpgradeTask *task);

入力と出力

  • **入力**:
  • cluster: 対象クラスタ情報。
  • slot: 処理対象の UpgradeTaskSlot 構造体。
  • task: 実行すべきタスク定義。
  • **出力**: なし（副作用として slot の状態および接続状態を更新）。

前提条件と不変条件

  • slot->ready が真であること。偽の場合は即座に復帰する。
  • dbs_processing および dbs_complete グローバル変数が、タスクの進捗管理のために適切に更新されること。

アルゴリズムまたは振る舞い

slot->state に基づくステートマシンとして動作する。 1. **FREE**: 未処理のデータベースがあれば dbs_processing をインクリメントし、CONNECTING 状態へ移行して start_conn() を呼び出す。 2. **CONNECTING**: PQconnectPoll() を呼び出し、接続完了を待機する。完了時に RUNNING_QUERIES へ移行し、PQsendQuery() でクエリを開始する。 3. **RUNNING_QUERIES**: PQconsumeInput() で入力を消費し、PQisBusy() が偽になるまで process_query_result() を呼び出して各ステップを処理する。全ステップ完了後、接続を解放し、スロットを初期化して再帰的に process_slot() を呼び出し、次のタスクへ移行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/task.c 内のメインループおよび wait_on_slots() から呼び出される。
  • process_slot 自身（再帰呼び出し）。
  • **呼び出し先**:
  • start_conn()
  • process_query_result()
  • libpq 関数群 (PQconnectPoll, PQsendQuery, PQconsumeInput, PQisBusy, PQfinish, PQerrorMessage)
  • pg_fatal()

関連データ構造

  • UpgradeTaskSlot: スロットの状態（FREE, CONNECTING, RUNNING_QUERIES）および接続情報を保持。
  • UpgradeTask: 実行すべきクエリステップの集合。
  • ClusterInfo: データベース配列情報。

失敗モードと境界条件

  • PQconnectPoll が PGRES_POLLING_FAILED を返した場合、pg_fatal により異常終了する。
  • PQsendQuery が失敗した場合、pg_fatal により異常終了する。
  • PQconsumeInput が失敗した場合、pg_fatal により異常終了する。
  • 全データベースの処理が完了している場合、FREE 状態で即座に復帰する。

保守担当者向けメモ

  • 本関数は非ブロッキング I/O を前提としているため、select_mode の設定が wait_on_slots の動作に直結する点に注意すること。
  • 再帰呼び出しは、スロットの解放直後に次のデータベース処理を即座に開始するために使用されている。
