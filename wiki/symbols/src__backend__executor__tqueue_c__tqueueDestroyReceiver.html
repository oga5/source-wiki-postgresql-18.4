tqueueDestroyReceiver

定義

  • 種別: function
  • 定義箇所: src/backend/executor/tqueue.c:104

目的

並列クエリ実行において使用された TQueueDestReceiver オブジェクトを破棄し、関連する共有メモリキューのデタッチおよびメモリ解放を行います。

シグネチャまたは形状

static void tqueueDestroyReceiver(DestReceiver *self)

入力と出力

  • **入力**: DestReceiver *self - 破棄対象のレシーバー構造体（実際には TQueueDestReceiver 型として扱われる）。
  • **出力**: なし（void）。

前提条件と不変条件

  • self は TQueueDestReceiver 構造体としてキャスト可能であること。
  • self が指すメモリ領域は、本関数呼び出し後に解放されるため、以降のアクセスは無効となる。

アルゴリズムまたは振る舞い

1. 引数 self を TQueueDestReceiver 型にキャストします。 2. tqueue->queue が NULL でない場合、shm_mq_detach を呼び出して共有メモリキューからデタッチします。 3. pfree(self) を呼び出し、レシーバー構造体自体が占有していたメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/tqueue.c 内で DestReceiver インターフェースの rDestroy フィールドに本関数が割り当てられています。直接的な呼び出し元関数の特定には、より広範な実行トレースが必要です。
  • **呼び出し先**:
  • shm_mq_detach (共有メモリキューのデタッチ)
  • pfree (メモリ解放)

関連データ構造

  • TQueueDestReceiver: 本関数が操作する主要な構造体。
  • DestReceiver: tqueue.c で実装される汎用的なタプル受信インターフェース。

失敗モードと境界条件

  • tqueue->queue が既に NULL である場合、shm_mq_detach はスキップされ、安全にメモリ解放のみが行われます。

保守担当者向けメモ

  • 本関数は DestReceiver インターフェースの一部として呼び出されることを前提としています。
  • 共有メモリのライフサイクル管理において、shm_mq_detach の呼び出しは必須であり、二重解放を防ぐための NULL チェックが含まれています。
