pgstat_bestart_initial

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/backend_status.c:269

目的

バックエンドプロセスの起動時に、共有メモリ上の PgBackendStatus エントリを初期化し、プロセスの初期状態（STATE_STARTING）を設定する。

シグネチャまたは形状

void pgstat_bestart_initial(void)

入力と出力

  • **入力**: なし（グローバル変数 MyBEEntry, MyProcPid, MyBackendType, MyStartTimestamp, MyProcPort を参照）。
  • **出力**: なし（共有メモリ上の MyBEEntry が指す領域を更新）。

前提条件と不変条件

  • pgstat_beinit() が事前に呼び出され、MyBEEntry が初期化されていること。
  • MyBEEntry が NULL でないこと（Assert で検証）。

アルゴリズムまたは振る舞い

1. 共有メモリ上の PgBackendStatus 構造体をローカル変数 lbeentry にコピーする。 2. lbeentry の各フィールド（PID、バックエンドタイプ、開始時刻、クライアント情報等）を初期値で埋める。 3. PGSTAT_BEGIN_WRITE_ACTIVITY マクロを使用して、共有メモリへの書き込みプロトコルを開始する。 4. lbeentry を共有メモリへ書き戻す。 5. アプリケーション名、クライアントホスト名、クエリ文字列を空文字で初期化し、必要に応じて SSL/GSS ステータスをゼロクリアする。 6. PGSTAT_END_WRITE_ACTIVITY マクロを使用して書き込みプロトコルを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/auxprocess.c
  • src/backend/utils/init/postinit.c
  • **呼び出し先**:
  • memcpy
  • MemSet
  • strlcpy
  • memset
  • PGSTAT_BEGIN_WRITE_ACTIVITY / PGSTAT_END_WRITE_ACTIVITY（マクロ）

関連データ構造

  • PgBackendStatus: バックエンドの活動状況を保持する共有メモリ上の構造体。
  • MyBEEntry: 現在のプロセスが使用する共有メモリ上の PgBackendStatus へのポインタ。

失敗モードと境界条件

  • MyProcPort が NULL の場合（例: autovacuum プロセス）、クライアントアドレスやホスト名はゼロまたは空文字として扱われる。
  • 共有メモリへの書き込みは st_changecount を用いた非ブロッキングプロトコルに従うため、読み取り側との競合は発生しないが、読み取り側はリトライが必要となる。

保守担当者向けメモ

  • 本関数は pgstat_beinit() の直後に呼び出されることを前提としている。
  • st_changecount の更新プロトコルを遵守するため、共有メモリへの直接書き込みは必ず PGSTAT_BEGIN_WRITE_ACTIVITY と PGSTAT_END_WRITE_ACTIVITY の間で行うこと。
