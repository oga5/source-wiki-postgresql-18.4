PGTYPESdate_from_asc

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/ecpg/pgtypeslib/datetime.c:46

目的

文字列形式の日付を解析し、ECPGライブラリ内部で使用される date 型（整数値）へ変換します。

シグネチャまたは形状

date PGTYPESdate_from_asc(char *str, char **endptr)

入力と出力

  • **入力**:
  • str: 解析対象の文字列。
  • endptr: 解析終了位置を受け取るポインタ（NULL指定可能）。
  • **出力**:
  • 成功時: 2000年1月1日を基準としたオフセット値（date型）。
  • 失敗時: INT_MIN を返し、グローバル変数 errno に PGTYPES_DATE_BAD_DATE を設定。

前提条件と不変条件

  • 入力文字列 str の長さは MAXDATELEN 以下である必要があります。
  • 内部計算において、2000年1月1日を基準日（date2j(2000, 1, 1)）として扱います。
  • EuroDates フラグは固定で false として処理されます。

アルゴリズムまたは振る舞い

1. 文字列長が MAXDATELEN を超える場合、エラーを返します。 2. ParseDateTime および DecodeDateTime を呼び出し、文字列をトークン化して tm 構造体に分解します。 3. 解析された型が DTK_DATE または DTK_EPOCH であることを確認します（DTK_EPOCH の場合は GetEpochTime で変換）。 4. date2j 関数を用いて、解析された年月日からユリウス日を算出し、基準日（2000-01-01）との差分を計算して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/data.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-dt_test.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-dt_test2.c
  • src/interfaces/ecpg/test/expected/sql-array.c
  • **呼び出し先**:
  • ParseDateTime, DecodeDateTime, GetEpochTime, date2j

関連データ構造

  • date: ECPG内部の日付表現（整数）。
  • struct tm: 日付要素を保持する標準構造体。

失敗モードと境界条件

  • 文字列が長すぎる場合、または解析不能な形式である場合に PGTYPES_DATE_BAD_DATE エラーが発生します。
  • DTK_DATE または DTK_EPOCH 以外の型として解析された場合、エラーとなります。

保守担当者向けメモ

  • 本関数は pgtypeslib モジュールにおける日付変換の主要なエントリポイントです。
  • 内部的な日付計算はユリウス日変換に依存しているため、修正時は dt_common.c のロジックとの整合性に注意してください。
