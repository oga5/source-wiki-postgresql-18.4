get_controlfile_by_exact_path

定義

  • 種別: function
  • 定義箇所: src/common/controldata_utils.c:67

目的

指定されたファイルパスからPostgreSQLの制御ファイル（ControlFileData）を読み込み、CRC検証およびバイトオーダーの整合性チェックを行う低レベル関数。

シグネチャまたは形状

ControlFileData *get_controlfile_by_exact_path(const char *ControlFilePath, bool *crc_ok_p)

入力と出力

  • **入力**:
  • ControlFilePath: 読み込む制御ファイルのパス。
  • crc_ok_p: CRC検証結果を格納するためのブール型ポインタ（必須）。
  • **出力**:
  • 読み込まれた ControlFileData 構造体へのポインタ。

前提条件と不変条件

  • crc_ok_p は非NULLであること。
  • FRONTEND マクロの定義有無により、エラー発生時の挙動（ereport または pg_fatal）が切り替わる。

アルゴリズムまたは振る舞い

1. ControlFileData 構造体用のメモリを確保する。 2. 指定されたパスを読み取り専用モードで開く。 3. ファイル内容を構造体へ読み込む。読み込み失敗時はエラー終了する。 4. CRCを計算し、ファイル内の値と比較して *crc_ok_p に結果を格納する。 5. **フロントエンド実行時のみ**: CRCが不一致の場合、最大10回までリトライを行う（並行書き込みによる不整合対策）。 6. バイトオーダーの不一致をチェックし、警告またはエラーを出力する。 7. 読み込んだ構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_combinebackup/pg_combinebackup.c
  • src/bin/pg_verifybackup/pg_verifybackup.c
  • src/common/controldata_utils.c（get_controlfile 関数内）
  • **呼び出し先**:
  • open, read, close (標準システムコール)
  • OpenTransientFile, CloseTransientFile (バックエンド時)
  • INIT_CRC32C, COMP_CRC32C, FIN_CRC32C, EQ_CRC32C (CRC計算用マクロ)
  • pg_usleep (フロントエンド時)

関連データ構造

  • ControlFileData: 制御ファイルのデータ構造定義（catalog/pg_control.h）。

失敗モードと境界条件

  • **ファイルオープン失敗**: 指定パスが存在しない、または権限がない場合にエラー終了。
  • **読み込みサイズ不一致**: ファイルサイズが sizeof(ControlFileData) と異なる場合にエラー終了。
  • **CRC不一致**: フロントエンドにおいて、10回のリトライ後もCRCが一致しない場合はそのまま処理を継続する（*crc_ok_p が false となる）。
  • **バイトオーダー不一致**: バイトオーダーが不適合と判断された場合、警告またはエラーが発生する。

保守担当者向けメモ

  • 本関数は FRONTEND マクロの定義により、エラーハンドリングの戦略が大きく異なる。修正時は両方のパスを考慮すること。
  • フロントエンドでのリトライロジックは、サーバー稼働中の制御ファイル読み取り時に発生しうる一時的な不整合を許容するためのものである。
