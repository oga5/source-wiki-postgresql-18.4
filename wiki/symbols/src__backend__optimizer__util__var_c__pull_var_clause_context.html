pull_var_clause_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/util/var.c:58

目的

式ツリーの走査中に、抽出された変数ノードのリストと走査の制御フラグを保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct
{
	List	   *varlist;
	int			flags;
} pull_var_clause_context;

入力と出力

  • varlist: 抽出された Var ノード（または PlaceHolderVar 等）を格納するリスト。
  • flags: 走査の挙動を制御するためのビットフラグ。

前提条件と不変条件

  • varlist は走査開始前に初期化されている必要があります。
  • pull_var_clause_walker 関数によって、ツリー走査中にこの構造体へノードが蓄積されます。

アルゴリズムまたは振る舞い

expression_tree_walker を利用した再帰的な走査において、特定の条件に合致するノードを varlist に追加する役割を担います。flags を用いて、どの種類のノードを抽出対象とするかの制御が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/util/var.c 内の pull_var_clause 関数などでインスタンス化されます。
  • 呼び出し先: pull_var_clause_walker 関数がこの構造体を引数として受け取り、走査状態を更新します。

関連データ構造

  • List: 抽出されたノードを保持するために使用されます。
  • pull_var_clause_walker: 本構造体を操作する主要な走査関数です。

失敗モードと境界条件

  • メモリ不足により varlist への追加（lappend 等）が失敗する可能性があります。
  • 走査対象のツリーが循環参照を持つ場合、スタックオーバーフローを引き起こす可能性があります。

保守担当者向けメモ

  • flags の具体的なビット定義については、pull_var_clause_walker の実装を参照してください。
  • 本構造体は expression_tree_walker パターンに従っているため、走査ロジックを変更する際は walker 関数側の整合性に注意してください。
