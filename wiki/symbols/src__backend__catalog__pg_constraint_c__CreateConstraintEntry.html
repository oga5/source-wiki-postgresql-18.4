CreateConstraintEntry

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_constraint.c:50

目的

PostgreSQLのシステムカタログ pg_constraint に新しい制約エントリを挿入し、制約のメタデータを永続化する。

シグネチャまたは形状

Oid CreateConstraintEntry(const char *constraintName, Oid constraintNamespace, char constraintType, bool isDeferrable, bool isDeferred, bool isEnforced, bool isValidated, Oid parentConstrId, Oid relId, const int16 *constraintKey, int constraintNKeys, int constraintNTotalKeys, Oid domainId, Oid indexRelId, Oid foreignRelId, const int16 *foreignKey, const Oid *pfEqOp, const Oid *ppEqOp, const Oid *ffEqOp, int foreignNKeys, char foreignUpdateType, char foreignDeleteType, const int16 *fkDeleteSetCols, int numFkDeleteSetCols, char foreignMatchType, const Oid *exclOp, Node *conExpr, const char *conBin, bool conIsLocal, int16 conInhCount, bool conNoInherit, bool conPeriod, bool is_internal)

入力と出力

  • **入力**: 制約名、名前空間、型、遅延可能性、強制フラグ、検証フラグ、親制約OID、対象リレーションOID、キー配列、外部キー情報、式（conExpr, conBin）、継承関連フラグ等。
  • **出力**: 新規作成された制約の Oid を返す。

前提条件と不変条件

  • isEnforced が false の場合、制約型は CONSTRAINT_CHECK または CONSTRAINT_FOREIGN でなければならない。
  • isEnforced が false の場合、isValidated は false でなければならない。
  • constraintName は非NULLであること。
  • カタログ操作のため、適切なロック（RowExclusiveLock）が保持されていること。

アルゴリズムまたは振る舞い

1. pg_constraint テーブルを RowExclusiveLock で開く。 2. C言語の配列（キーや演算子）をPostgreSQLの ArrayType に変換する。 3. GetNewOidWithIndex を使用して新しいOIDを取得する。 4. pg_constraint の各属性に対応する values 配列と nulls 配列を構築する。 5. 構築したデータを用いてカタログ行を挿入する（ソースコードの続きで CatalogTupleInsert が行われる）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/heap.c, src/backend/catalog/index.c, src/backend/commands/tablecmds.c, src/backend/commands/trigger.c, src/backend/commands/typecmds.c などから参照されている。
  • **呼び出し先**: table_open, namestrcpy, palloc, construct_array_builtin, GetNewOidWithIndex 等のユーティリティ関数。

関連データ構造

  • pg_constraint (カタログ定義)
  • ArrayType (PostgreSQL内部配列表現)
  • ObjectAddresses (依存関係管理用)

失敗モードと境界条件

  • constraintNKeys や foreignNKeys が0以下の場合は、対応する配列フィールドはNULLとして扱われる。
  • 必須の制約属性が欠落している場合や、カタログへの挿入時に一意性制約違反が発生した場合はエラーとなる。

保守担当者向けメモ

  • 本関数は制約のカタログ登録のみを担当する。依存関係の登録（pg_depend への書き込み）は、この関数の呼び出し元または後続の処理で行われる必要がある。
  • NOT NULL 制約など、特定の制約型に特化した処理が必要な場合は、本関数を呼び出す前に適切な前処理を行うこと。
