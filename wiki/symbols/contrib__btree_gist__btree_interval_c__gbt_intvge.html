gbt_intvge

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_interval.c:37

目的

interval 型の2つの値に対して「以上（greater than or equal to）」の比較を行い、その結果をブール値で返します。

シグネチャまたは形状

static bool gbt_intvge(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の interval 型データへのポインタ。
  • flinfo: 関数呼び出し情報（PostgreSQLの内部関数呼び出し用）。
  • **出力**:
  • bool: a >= b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な interval 型のデータ構造を指している必要があります。
  • 内部的に interval_ge 関数を呼び出すため、PostgreSQLの interval 型比較ロジックに準拠します。

アルゴリズムまたは振る舞い

1. IntervalPGetDatum マクロを使用して、入力ポインタ a および b を Datum 型に変換します。 2. DirectFunctionCall2 を介して、PostgreSQLのコア関数 interval_ge を呼び出します。 3. interval_ge の戻り値を DatumGetBool でブール値に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_interval.c 内の静的テーブル（gbtree_ninfo 構造体の一部として参照されている可能性が高い）。
  • **呼び出し先**: interval_ge（PostgreSQLコアの interval 比較関数）。

関連データ構造

  • interval: PostgreSQLの標準的な時間間隔データ型。

失敗モードと境界条件

  • 入力が NULL の場合、PostgreSQLの関数呼び出し規約に従い、予期せぬ動作やエラーが発生する可能性があります。
  • interval 型の比較ロジック（interval_ge）がエラーを発生させた場合、本関数もそれに追従します。

保守担当者向けメモ

  • 本関数は btree_gist における interval 型の比較演算を GiST フレームワークに適合させるためのラッパーです。
  • ロジックの変更が必要な場合は、interval_ge の仕様を確認してください。
