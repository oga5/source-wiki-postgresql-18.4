clause_selectivity

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/clausesel.c:666

目的

クエリプランナがコスト見積もりのために使用する、特定の条件式（clause）に対する選択率（Selectivity）を算出します。

シグネチャまたは形状

Selectivity clause_selectivity(PlannerInfo *root,
                               Node *clause,
                               int varRelid,
                               JoinType jointype,
                               SpecialJoinInfo *sjinfo);

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • clause: 選択率を評価する対象の式（Node）。
  • varRelid: 評価対象のリレーションID。
  • jointype: 結合の種類（JOIN_INNER 等）。
  • sjinfo: 特殊な結合に関する情報。
  • **出力**:
  • Selectivity（double型）: 0.0 から 1.0 の範囲で、条件に適合する行の割合を返します。

前提条件と不変条件

  • clause_selectivity_ext のラッパーとして機能し、拡張統計の使用フラグを true に固定して呼び出します。
  • 呼び出し元は、有効な PlannerInfo および Node 構造体を提供する必要があります。

アルゴリズムまたは振る舞い

1. clause_selectivity_ext を呼び出し、拡張統計（Extended Statistics）を有効にした状態で選択率の計算を委譲します。 2. 内部的にキャッシュ機構を利用し、同一条件に対する再計算を回避します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/costsize.c: コスト見積もり時に呼び出されます。
  • src/backend/optimizer/plan/setrefs.c: 参照解決時に呼び出されます。
  • src/backend/optimizer/util/orclauses.c: OR 条件の選択率計算時に呼び出されます。
  • **呼び出し先**:
  • clause_selectivity_ext: 実際の計算ロジックを担う関数。

関連データ構造

  • PlannerInfo: プランナのコンテキスト情報。
  • RestrictInfo: 選択率の結果がキャッシュされる構造体。
  • SpecialJoinInfo: 結合の制約情報。

失敗モードと境界条件

  • 統計情報が不足している場合、デフォルトの選択率推定値が返されます。
  • clause が評価不能な形式である場合、計算ロジックがフォールバック処理を行う可能性があります。

保守担当者向けメモ

  • 本関数は clause_selectivity_ext への単なるパススルーです。ロジックの変更が必要な場合は clause_selectivity_ext を参照してください。
  • RestrictInfo へのキャッシュ結果が再利用されるため、本関数を直接呼び出すことでプランナの計算コストを削減できます。
