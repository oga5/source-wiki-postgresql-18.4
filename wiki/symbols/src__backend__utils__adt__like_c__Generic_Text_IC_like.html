Generic_Text_IC_like

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like.c:176

目的

SQLの ILIKE 演算子を実装するため、大文字小文字を区別せずに文字列パターンマッチングを行う。

シグネチャまたは形状

static inline int Generic_Text_IC_like(text *str, text *pat, Oid collation)

入力と出力

  • **入力**:
  • str: 検索対象の文字列 (text*)
  • pat: パターン文字列 (text*)
  • collation: 照合順序 (Oid)
  • **出力**:
  • マッチした場合は LIKE_TRUE (1)、マッチしない場合は LIKE_FALSE (0) を返す。

前提条件と不変条件

  • collation が有効な Oid であること。無効な場合はエラーとなる。
  • 指定された照合順序が非決定論的（nondeterministic）である場合、サポートされていないためエラーとなる。

アルゴリズムまたは振る舞い

1. **照合順序の検証**: pg_newlocale_from_collation を使用してロケール情報を取得し、決定論的であることを確認する。 2. **エンコーディング判定**: データベースの最大文字長が1バイトを超える場合、またはICUプロバイダを使用している場合は、以下の手順を実行する：

  • lower() 関数を呼び出し、パターンと対象文字列の両方を小文字に変換する。
  • エンコーディングに応じて UTF8_MatchText または MB_MatchText を呼び出す。

    • 3. **シングルバイトの場合**: 効率化のため lower() を介さず、SB_IMatchText を使用して文字単位で大文字小文字を無視した比較を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/like.c 内のSQL関数（texticlike 等の内部実装箇所）から呼び出される。
  • **呼び出し先**:
  • lower (SQL関数)
  • UTF8_MatchText, MB_MatchText, SB_IMatchText (マッチングアルゴリズム)
  • pg_newlocale_from_collation

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • pg_locale_t: ロケール情報を保持する構造体。

失敗モードと境界条件

  • **ERRCODE_INDETERMINATE_COLLATION**: 照合順序が解決できない場合に発生。
  • **ERRCODE_FEATURE_NOT_SUPPORTED**: 非決定論的な照合順序が指定された場合に発生。

保守担当者向けメモ

  • 本関数は like_match.c で定義されたアルゴリズムを呼び出すフロントエンドである。
  • ICUプロバイダ使用時は単一文字のケースフォールディングがサポートされないため、必ず lower() を使用するパスを通る。
  • シングルバイト環境での最適化（SB_IMatchText）とマルチバイト環境での挙動の違いに注意すること。
