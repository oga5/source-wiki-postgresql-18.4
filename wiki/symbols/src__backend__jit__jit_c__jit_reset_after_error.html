jit_reset_after_error

定義

  • 種別: function
  • 定義箇所: src/backend/jit/jit.c:126

目的

JIT コンパイル処理中にエラーが発生した際、JIT プロバイダー側の内部状態をリセットし、クリーンアップを行う。

シグネチャまたは形状

void jit_reset_after_error(void)

入力と出力

  • 入力: なし
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • provider_successfully_loaded が真である場合にのみ、プロバイダーの reset_after_error 関数が呼び出される。

アルゴリズムまたは振る舞い

1. provider_successfully_loaded フラグを確認する。 2. フラグが真であれば、ロード済みの JIT プロバイダーが保持する reset_after_error コールバック関数を実行する。 3. フラグが偽であれば、何も行わずに終了する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/tcop/postgres.c:4473
  • 呼び出し先:
  • provider.reset_after_error()（プロバイダーが提供する関数ポインタ）

関連データ構造

  • provider: ロードされた JIT プロバイダーの関数ポインタを保持する構造体。
  • provider_successfully_loaded: プロバイダーのロード状態を管理するグローバル変数。

失敗モードと境界条件

  • プロバイダーがロードされていない場合、本関数は安全に何もしない（エラーにはならない）。

保守担当者向けメモ

  • 本関数は jit.c 内の抽象化レイヤーであり、具体的なリセットロジックは各プロバイダー（LLVM など）の実装に依存する。
  • エラー発生時のセッション状態の整合性を保つために重要であるため、プロバイダーの追加・変更時には reset_after_error が適切に実装されていることを確認すること。
