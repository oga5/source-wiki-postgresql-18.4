verifybackup_system_identifier

定義

  • 種別: function
  • 定義箇所: src/bin/pg_verifybackup/pg_verifybackup.c:536

目的

JSON マニフェストから読み取ったシステム識別子（system identifier）を、検証コンテキスト内の manifest_data 構造体に格納する。

シグネチャまたは形状

static void verifybackup_system_identifier(JsonManifestParseContext *context, uint64 manifest_system_identifier)

入力と出力

  • **入力**:
  • context: マニフェスト解析コンテキスト。private_data メンバを介して manifest_data 構造体を参照する。
  • manifest_system_identifier: マニフェストから抽出された 64 ビットのシステム識別子。
  • **出力**: なし（副作用として context->private_data が指す構造体を更新）。

前提条件と不変条件

  • context->private_data が manifest_data 型の有効なポインタとして初期化されていること。

アルゴリズムまたは振る舞い

1. context->private_data を manifest_data 型としてキャストし、ローカル変数 manifest に保持する。 2. 引数 manifest_system_identifier を manifest->system_identifier に代入する。 3. 検証処理自体は本関数内では行わず、後のステージで実行される。

呼び出し元と呼び出し先

  • **呼び出し元**: マニフェスト解析処理（parse_manifest_file 等）のコールバックとして呼び出される。
  • **呼び出し先**: なし。

関連データ構造

  • JsonManifestParseContext: マニフェスト解析のコンテキスト。
  • manifest_data: バックアップマニフェストの情報を保持する構造体。

失敗モードと境界条件

  • 特筆すべき失敗モードはない。入力された識別子をそのまま構造体に格納する。

保守担当者向けメモ

  • 本関数は JsonManifestParseContext のコールバックとして登録される。
  • 実際のシステム識別子の検証（バックアップディレクトリの制御ファイルとの照合など）は、この関数が値を格納した後の検証フェーズで行われる。
