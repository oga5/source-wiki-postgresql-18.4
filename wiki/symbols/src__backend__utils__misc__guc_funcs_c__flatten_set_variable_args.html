flatten_set_variable_args

定義

  • src/backend/utils/misc/guc_funcs.c:191

目的

SQLの構文解析結果である List 形式の引数を、GUC（Grand Unified Configuration）システムが処理可能な文字列形式に変換・正規化します。

シグネチャまたは形状

static char *flatten_set_variable_args(const char *name, List *args);

入力と出力

  • **入力**:
  • name: 設定対象のGUCパラメータ名。
  • args: A_Const ノード（または TypeCast でラップされたノード）のリスト。
  • **出力**:
  • 変換された設定値の文字列。args が NIL の場合は NULL を返します。

前提条件と不変条件

  • args 内の各要素は A_Const ノードである必要があります。
  • GUC_LIST_INPUT フラグが設定されていないパラメータに対して複数の引数が渡された場合、エラーを発生させます。
  • TypeCast を伴う引数は、INTERVAL 型（TIME ZONE 設定用）のみサポートされます。

アルゴリズムまたは振る舞い

1. args が空であれば NULL を返す。 2. find_option を用いて対象パラメータのフラグを取得する。 3. リスト内の各ノードを走査し、StringInfo バッファに順次追加する。

  • T_Integer: 数値を文字列化して追加。
  • T_Float: 文字列としてそのまま追加。
  • T_String:
  • TypeCast がある場合、interval_in/interval_out を経由して値を正規化し、INTERVAL '...' 形式で出力。
  • GUC_LIST_QUOTE フラグがある場合、quote_identifier で識別子をクォートして追加。
  • それ以外はそのまま追加。

    • 4. 複数の引数がある場合はカンマ区切りで連結する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/guc_funcs.c 内の ExecSetVariableStmt 等から呼び出されます。
  • **呼び出し先**:
  • find_option, initStringInfo, appendStringInfo, list_length, list_head, lfirst, nodeTag, intVal, castNode, strVal, typenameTypeIdAndMod, DirectFunctionCall1/3, quote_identifier, ereport, elog 等。

関連データ構造

  • List, ListCell: 引数リストの保持に使用。
  • StringInfoData: 変換後の文字列構築用バッファ。
  • config_generic: GUCパラメータの属性（フラグ等）を参照するために使用。
  • A_Const, TypeCast: パーサーが生成する構文木ノード。

失敗モードと境界条件

  • args 内に A_Const 以外のノードが含まれる場合、elog(ERROR) で停止します。
  • GUC_LIST_INPUT が未設定のパラメータに対して引数が2つ以上ある場合、ereport(ERROR) で停止します。
  • INTERVAL 型以外のキャストが指定された場合、Assert により停止します。

保守担当者向けメモ

  • 本関数はSQLインターフェース層の変換ロジックであり、実際の値の適用は guc.c 側の関数に委ねられます。
  • INTERVAL の正規化に interval_in/interval_out を使用しているため、型定義の変更時には注意が必要です。
