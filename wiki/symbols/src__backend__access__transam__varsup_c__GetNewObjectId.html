GetNewObjectId

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/varsup.c:554

目的

システム全体で一意なオブジェクトID（OID）を払い出します。OIDの枯渇を防ぐため、一定数（VAR_OID_PREFETCH）をまとめてWALに記録し、共有メモリ上で管理するOIDカウンタを更新します。

シグネチャまたは形状

Oid GetNewObjectId(void)

入力と出力

  • **入力**: なし
  • **出力**: 新規に割り当てられた Oid 値

前提条件と不変条件

  • リカバリ中（RecoveryInProgress() が真）ではないこと。
  • OidGenLock を排他モードで取得して実行されること。
  • 戻り値は InvalidOid (0) ではなく、かつ FirstNormalObjectId 以上であることが保証されます（ブートストラップモードを除く）。

アルゴリズムまたは振る舞い

1. **リカバリチェック**: リカバリ中の場合はエラーを発生させます。 2. **ロック取得**: OidGenLock を排他モードで取得します。 3. **Wraparound保護**:

  • nextOid が FirstNormalObjectId 未満の場合、通常モードであれば FirstNormalObjectId にリセットし、予約済みOIDとの衝突を回避します。
  • ブートストラップモード等の特殊なケースでは、FirstGenbkiObjectId を基準に判定を行います。

    • 4. **WALへのプリフェッチ**: oidCount が0の場合、XLogPutNextOid を呼び出して nextOid + VAR_OID_PREFETCH までの範囲をWALに記録し、oidCount を VAR_OID_PREFETCH にリセットします。 5. **ID払い出し**: 現在の nextOid を結果として保持し、nextOid をインクリメント、oidCount をデクリメントします。 6. **ロック解放**: OidGenLock を解放して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/catalog.c（カタログオブジェクトのOID生成時に利用）
  • **呼び出し先**:
  • RecoveryInProgress()
  • LWLockAcquire() / LWLockRelease()
  • XLogPutNextOid()

関連データ構造

  • TransamVariables: 共有メモリ上の構造体。nextOid（次に割り当てるOID）と oidCount（プリフェッチ済みの残りOID数）を保持します。

失敗モードと境界条件

  • **リカバリ中**: elog(ERROR) により処理が中断されます。
  • **OID枯渇**: nextOid が上限に達した場合、Wraparound処理によりリセットされます。
  • **ロック競合**: OidGenLock の取得待ちが発生する可能性があります。

保守担当者向けメモ

  • 本関数は OidGenLock を保持するため、処理は最小限に抑えられています。
  • VAR_OID_PREFETCH による一括予約は、WAL書き込み回数を削減するための最適化です。
  • IsPostmasterEnvironment の状態によって、OIDの範囲制限（FirstNormalObjectId 等）の適用ルールが異なります。
