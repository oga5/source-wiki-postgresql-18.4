PLAINTREE

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsquery_cleanup.c:54

目的

tsquery のクエリツリー（NODE）をフラットな QueryItem 配列へ変換するプロセスにおいて、変換状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct
{
	QueryItem  *ptr;    /* QueryItem 配列へのポインタ */
	int			len;    /* ptr に割り当てられたサイズ */
	int			cur;    /* ptr に格納されている要素数 */
} PLAINTREE;

入力と出力

本構造体は、plaintree 関数および plainnode 関数において、変換中の QueryItem 配列のバッファ管理および現在の書き込み位置を追跡するために使用されます。

前提条件と不変条件

  • ptr は、変換対象のクエリツリーから生成される QueryItem を格納するために十分なメモリが割り当てられている必要があります。
  • cur は常に 0 以上 len 以下の値を保持し、現在の書き込み位置を示します。

アルゴリズムまたは振る舞い

  • plaintree 関数内で初期化され、plainnode 関数がツリーを再帰的に走査する際、この構造体を通じて QueryItem 配列への書き込みと要素数のカウントが行われます。
  • 変換プロセスにおいて、cur は新しいノードが追加されるたびにインクリメントされます。

呼び出し元と呼び出し先

  • 呼び出し元: plaintree 関数（src/backend/utils/adt/tsquery_cleanup.c:99 でローカル変数として宣言）。
  • 呼び出し先: plainnode 関数（src/backend/utils/adt/tsquery_cleanup.c:62 の引数として渡される）。

関連データ構造

  • QueryItem: tsquery の各要素を表現する構造体。
  • NODE: tsquery のツリー構造を表現する内部構造体。

失敗モードと境界条件

  • len を超えて ptr に書き込もうとした場合、メモリ破壊が発生する可能性があります。この構造体を利用する関数側で、事前に必要なバッファサイズを計算（calcstrlen 等を使用）しておく必要があります。

保守担当者向けメモ

  • 本構造体は tsquery の再構築プロセスに特化した一時的な状態保持用です。
  • ptr のメモリ管理は呼び出し側の関数に依存しているため、ライフサイクル管理には注意してください。
