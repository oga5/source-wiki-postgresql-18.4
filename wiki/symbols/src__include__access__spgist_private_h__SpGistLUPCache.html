SpGistLUPCache

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgist_private.h:108

目的

SP-GiSTインデックス操作において、各バックエンドが直近に使用したページ情報をキャッシュし、メタページへのアクセス競合を低減するために使用されます。

シグネチャまたは形状

typedef struct SpGistLUPCache
{
    SpGistLastUsedPage cachedPage[SPGIST_CACHED_PAGES];
} SpGistLUPCache;

入力と出力

  • 入力: SPGIST_CACHED_PAGES 個の SpGistLastUsedPage 構造体を保持する配列。
  • 出力: なし（データ構造体）。

前提条件と不変条件

  • SPGIST_CACHED_PAGES で定義された固定数のキャッシュエントリを保持します。

アルゴリズムまたは振る舞い

  • インデックスの挿入や検索処理において、最後に使用したページ情報を保持することで、メタページへの頻繁なアクセスを回避する仕組みを提供します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/include/access/spgist_private.h:119 (共有ストレージとしての利用)
  • src/include/access/spgist_private.h:260 (ローカルストレージとしての利用)
  • 呼び出し先:
  • SpGistLastUsedPage (構成要素として使用)

関連データ構造

  • SpGistLastUsedPage: キャッシュされる個々のページ情報を保持する構造体。

失敗モードと境界条件

  • キャッシュサイズ（SPGIST_CACHED_PAGES）を超えるページアクセスが発生した場合の挙動については、本構造体定義のみでは不明です。

保守担当者向けメモ

  • 本構造体は、インデックスの物理的なページ管理における最適化レイヤーです。変更を加える際は、同時実行制御およびメタページへのアクセス頻度への影響を考慮してください。
