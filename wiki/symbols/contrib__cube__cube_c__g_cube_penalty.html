g_cube_penalty

定義

  • 種別: function
  • 定義箇所: contrib/cube/cube.c:489

目的

GiST 索引において、新しいデータ（newentry）を既存のツリーノード（origentry）に追加する際に生じる「ペナルティ」を計算します。このペナルティは、ノードのボックスを拡張した際の面積（または体積）の増加分として定義されます。

シグネチャまたは形状

Datum g_cube_penalty(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY *origentry（既存の索引エントリ）
  • PG_GETARG_POINTER(1): GISTENTRY *newentry（追加対象の索引エントリ）
  • PG_GETARG_POINTER(2): float *result（計算結果を格納するポインタ）
  • **出力**:
  • float8: 拡張によるサイズの増加分を返します。

前提条件と不変条件

  • origentry および newentry のキーは NDBOX 型として解釈可能である必要があります。
  • DatumGetNDBOXP マクロを使用して、GISTENTRY から NDBOX 構造体への変換が行われます。

アルゴリズムまたは振る舞い

1. cube_union_v0 を呼び出し、既存のボックスと新しいボックスを包含する最小のボックス（ud）を計算します。 2. rt_cube_size を使用して、拡張後のボックス（ud）のサイズ tmp1 を計算します。 3. 同様に、既存のボックスのサイズ tmp2 を計算します。 4. 拡張によるサイズ増加分（tmp1 - tmp2）を result に格納し、その値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST 索引構築・挿入プロセスから呼び出されます（直接的な関数呼び出しの記述はソース内にありませんが、PG_FUNCTION_INFO_V1 マクロにより GiST インターフェースとして登録されています）。
  • **呼び出し先**:
  • cube_union_v0: 2つのボックスの和集合を計算。
  • rt_cube_size: ボックスのサイズを計算。

関連データ構造

  • NDBOX: 多次元ボックスを表現するデータ構造。
  • GISTENTRY: GiST 索引の各ノードを表現する PostgreSQL の標準構造体。

失敗モードと境界条件

  • NDBOX の次元数が一致しない場合や、不正なメモリ領域を指している場合の挙動は、基盤となる cube_union_v0 および rt_cube_size の実装に依存します。

保守担当者向けメモ

  • 本関数は GiST 索引の分割判断（picksplit）や挿入時のパス選択に影響を与える重要な関数です。
  • 面積計算のロジックを変更する場合は、rt_cube_size の定義と整合性が取れていることを確認してください。
