int_list_to_array

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mcxtfuncs.c:47

目的

整数値のリスト（List）を、SQLで利用可能なPostgreSQLの整数配列（ArrayType）に変換します。

シグネチャまたは形状

Datum int_list_to_array(const List *list)

入力と出力

  • **入力**: list (整数を含む List 構造体へのポインタ)
  • **出力**: Datum 型（INT4OID 型の ArrayType をラップしたもの）

前提条件と不変条件

  • 入力される list は整数値のみを含んでいる必要があります。
  • palloc を使用してメモリを確保するため、呼び出し元は適切なメモリコンテキストにいる必要があります。

アルゴリズムまたは振る舞い

1. list_length(list) でリストの要素数を取得します。 2. 要素数分の Datum 配列を palloc で確保します。 3. foreach_int マクロを使用してリストを走査し、各整数値を Int32GetDatum で Datum に変換して配列に格納します。 4. construct_array_builtin を呼び出し、INT4OID を指定して ArrayType を構築します。 5. 構築した配列を PointerGetDatum でラップして返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/mcxtfuncs.c:167（values[4] = int_list_to_array(path); の箇所で確認）
  • **呼び出し先**:
  • list_length
  • palloc
  • foreach_int / foreach_current_index
  • Int32GetDatum
  • construct_array_builtin
  • PointerGetDatum

関連データ構造

  • List: PostgreSQLの標準的なリスト構造。
  • ArrayType: PostgreSQLの配列型内部表現。
  • Datum: PostgreSQLの汎用データ表現。

失敗モードと境界条件

  • リストが空の場合、length が 0 となり、空の配列が返されます。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数は mcxtfuncs.c 内でメモリコンテキストの階層パス（path）をSQL結果セットに変換するために使用されています。
  • INT4OID を固定で使用しているため、リスト内の要素が32ビット整数であることを前提としています。
