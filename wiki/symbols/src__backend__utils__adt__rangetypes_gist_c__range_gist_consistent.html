range_gist_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_gist.c:190

目的

GiSTインデックスにおける範囲型（Range Type）の検索条件判定を行うメインエントリポイントです。インデックスのノード（リーフまたは内部）とクエリの型（範囲、マルチ範囲、要素）に応じて、適切な整合性チェック関数へ処理を振り分けます。

シグネチャまたは形状

Datum range_gist_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY *entry（インデックスエントリ）
  • PG_GETARG_DATUM(1): Datum query（検索条件）
  • PG_GETARG_UINT16(2): StrategyNumber strategy（演算子戦略）
  • PG_GETARG_OID(3): Oid subtype（クエリのサブタイプ）
  • PG_GETARG_POINTER(4): bool *recheck（再チェックフラグ）
  • **出力**:
  • Datum: bool 型の判定結果（PG_RETURN_BOOL）
  • *recheck: 常に false が設定されます（範囲型のGiST演算はすべて正確であるため）。

前提条件と不変条件

  • *recheck は常に false に設定されます。
  • range_get_typcache を使用して、キーの型に対応する TypeCacheEntry を取得する必要があります。

アルゴリズムまたは振る舞い

1. recheck を false に初期化します。 2. entry->key から RangeType を取得し、型キャッシュをロードします。 3. GIST_LEAF(entry) を判定し、リーフノードか内部ノードかを分岐します。 4. subtype に基づき、以下のいずれかの関数へディスパッチします：

  • **範囲型クエリ**: range_gist_consistent_leaf_range / range_gist_consistent_int_range
  • **マルチ範囲型クエリ**: range_gist_consistent_leaf_multirange / range_gist_consistent_int_multirange
  • **要素型クエリ**: range_gist_consistent_leaf_element / range_gist_consistent_int_element

    • 5. 判定結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: GiSTインデックスの検索エンジン（直接の呼び出し元情報はソースコード上では明示されていません）。
  • **呼び出し先**:
  • range_get_typcache
  • range_gist_consistent_leaf_range / range_gist_consistent_int_range
  • range_gist_consistent_leaf_multirange / range_gist_consistent_int_multirange
  • range_gist_consistent_leaf_element / range_gist_consistent_int_element

関連データ構造

  • GISTENTRY: GiSTインデックスのノードエントリ。
  • RangeType: 範囲型の内部表現。
  • TypeCacheEntry: 型情報のキャッシュ。

失敗モードと境界条件

  • subtype が無効（!OidIsValid）または ANYRANGEOID の場合は、通常の範囲型クエリとして処理されます。
  • 内部ノードとリーフノードで異なる判定ロジックが適用されるため、インデックス構造の整合性が重要です。

保守担当者向けメモ

  • 本関数はすべての範囲型演算を「正確（exact）」とみなすため、再チェックは不要です。
  • 新しい範囲型やマルチ範囲型のサポートを追加する際は、ディスパッチ先の関数群に適切なロジックが実装されていることを確認してください。
