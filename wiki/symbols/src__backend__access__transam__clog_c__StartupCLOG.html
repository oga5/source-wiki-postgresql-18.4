StartupCLOG

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/clog.c:876

目的

CLOG（Commit Log）管理システムにおける最新ページ番号の初期化を行う。

シグネチャまたは形状

void StartupCLOG(void)

入力と出力

  • **入力**: なし（内部的に TransamVariables->nextXid を参照）。
  • **出力**: なし。副作用として共有メモリ上の latest_page_number を更新する。

前提条件と不変条件

  • TransamVariables が初期化済みであること。
  • システム起動時またはリカバリ処理の過程で呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. TransamVariables->nextXid から現在のトランザクションID（xid）を取得する。 2. TransactionIdToPage(xid) を呼び出し、その xid が属するページ番号を算出する。 3. 算出されたページ番号を XactCtl->shared->latest_page_number にアトミック書き込み（pg_atomic_write_u64）し、システムが認識する最新の CLOG ページとして設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c:5651
  • src/backend/storage/ipc/procarray.c:4398（コメント内での言及）
  • **呼び出し先**:
  • XidFromFullTransactionId
  • TransactionIdToPage
  • pg_atomic_write_u64

関連データ構造

  • XactCtl: CLOG の共有メモリ状態を管理する構造体。
  • TransamVariables: トランザクション管理のグローバル変数群。

失敗モードと境界条件

  • nextXid が不正な値である場合、誤ったページ番号が初期値として設定される可能性がある。

保守担当者向けメモ

  • 本関数はシステム起動時の初期化シーケンスの一部である。
  • latest_page_number は pg_atomic_write_u64 を用いて更新されており、並行して動作する他のプロセスからの参照に対して整合性が保たれる設計となっている。
