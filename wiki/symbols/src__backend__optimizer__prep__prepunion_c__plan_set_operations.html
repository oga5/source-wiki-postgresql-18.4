plan_set_operations

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepunion.c:92

目的

SQL の集合演算（UNION, INTERSECT, EXCEPT）を含むクエリのプランニングを開始し、論理的な集合演算ツリーを物理的な実行パスへと変換するメインエントリポイントです。

シグネチャまたは形状

RelOptInfo *plan_set_operations(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root（現在のクエリのプランニング状態を保持する構造体）
  • **出力**: RelOptInfo *（集合演算の結果を表すリレーション情報）

前提条件と不変条件

  • root->parse->setOperations が存在すること。
  • クエリが集合演算専用の構造であり、jointree に fromlist や quals、groupClause 等が含まれていないこと。
  • root->eq_classes が空であり、等価クラスの統合が完了していること（root->ec_merging_done = true に設定）。

アルゴリズムまたは振る舞い

1. setup_simple_rel_arrays を呼び出し、リーフとなるサブクエリ用のリレーション配列を準備します。 2. 集合演算ツリーの左端（leftmost）にあるサブクエリを特定し、そのターゲットリストを列名の整合性維持のために取得します。 3. root->hasRecursion が真の場合、generate_recursion_path を呼び出して再帰的 UNION を処理します。 4. それ以外の場合、recurse_set_operations を再帰的に呼び出し、集合演算ツリー全体を走査してパスを生成します。 5. 最終的に構築されたターゲットリストを root->processed_tlist に格納し、生成された RelOptInfo を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planner.c（planner 関数内から呼び出されます）
  • **呼び出し先**: setup_simple_rel_arrays, generate_recursion_path, recurse_set_operations

関連データ構造

  • PlannerInfo: プランニングの全体状態。
  • SetOperationStmt: 集合演算の論理ツリー構造。
  • RelOptInfo: 集合演算の結果を保持するリレーション情報。
  • RangeTblEntry: サブクエリの参照情報。

失敗モードと境界条件

  • topop（SetOperationStmt）が空の場合、アサーションエラーが発生します。
  • 集合演算ツリーがサポートされていない構造（groupClause 等の存在）である場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は root->processed_tlist を設定する責務があります。
  • leftmostQuery のターゲットリストは、SELECT INTO 等の正しい動作のために全生成ターゲットリストの基準として使用されます。
  • 再帰的 UNION と非再帰的集合演算で処理経路が分岐するため、修正時は両方のパスを確認してください。
