dlopen

定義

  • 種別: function
  • 定義箇所: src/port/win32dlopen.c:75

目的

Windows環境において、POSIX標準の dlopen 関数をエミュレートし、共有ライブラリ（DLL）をプロセス空間にロードします。

シグネチャまたは形状

void *dlopen(const char *file, int mode);

入力と出力

  • **入力**:
  • file: ロード対象のDLLファイルパス（文字列）。
  • mode: ロードモード（POSIX互換のフラグ）。
  • **出力**:
  • 成功時: ロードされたDLLのハンドル（void *）。
  • 失敗時: NULL。

前提条件と不変条件

  • Windows APIの LoadLibrary を利用して実装されています。
  • DLLロード時のシステムエラーダイアログ（ポップアップ）は抑制されます。
  • エラー発生時の状態保持に使用される静的変数 last_dyn_error はスレッドセーフではありません。

アルゴリズムまたは振る舞い

1. SetErrorMode を呼び出し、SEM_FAILCRITICALERRORS および SEM_NOOPENFILEERRORBOX を設定して、DLLロード時のシステムエラーダイアログを無効化します。 2. LoadLibrary を実行して指定されたファイルをロードします。 3. SetErrorMode を元の状態に戻します。 4. ロードに失敗した場合、set_dl_error() を呼び出してエラー情報を記録し、NULL を返します。 5. ロードに成功した場合、エラーバッファをクリアし、取得したハンドルを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/fmgr/dfmgr.c
  • src/interfaces/libpq/fe-auth-oauth.c
  • **呼び出し先**:
  • SetErrorMode (Windows API)
  • LoadLibrary (Windows API)
  • set_dl_error (内部ヘルパー関数)

関連データ構造

  • HMODULE: Windows APIで使用されるモジュールハンドル。
  • last_dyn_error: エラーメッセージを保持する静的バッファ。

失敗モードと境界条件

  • 指定されたファイルが見つからない、またはロードできない場合、LoadLibrary が NULL を返し、本関数は NULL を返します。
  • エラー発生時は set_dl_error() によりエラー詳細が記録されます。

保守担当者向けメモ

  • 本関数はWindowsビルド時のみコンパイルされます。
  • last_dyn_error が静的変数であるため、マルチスレッド環境での同時呼び出しには注意が必要です。
