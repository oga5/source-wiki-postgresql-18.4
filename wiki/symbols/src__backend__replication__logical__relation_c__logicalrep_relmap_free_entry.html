logicalrep_relmap_free_entry

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/relation.c:131

目的

論理レプリケーションのリレーションマッピングキャッシュエントリ（LogicalRepRelMapEntry）が保持するメモリリソースを解放する。

シグネチャまたは形状

static void logicalrep_relmap_free_entry(LogicalRepRelMapEntry *entry)

入力と出力

  • **入力**: entry - 解放対象となる LogicalRepRelMapEntry 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • entry は有効なメモリ領域を指している必要がある。
  • entry->remoterel 内の各ポインタ（nspname, relname, attnames 等）は、palloc によって割り当てられたメモリである必要がある。

アルゴリズムまたは振る舞い

1. remoterel 構造体内の文字列（名前空間名、リレーション名）を pfree で解放する。 2. リモート属性が存在する場合（natts > 0）、各属性名および属性名配列、属性型配列を順次 pfree で解放する。 3. リモートキーのビットマップ（attkeys）を bms_free で解放する。 4. entry->attrmap が存在する場合、free_attrmap を呼び出して属性マッピング情報を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/relation.c:181
  • src/backend/replication/logical/relation.c:588
  • **呼び出し先**:
  • pfree (PostgreSQL メモリ管理)
  • bms_free (ビットマップセット解放)
  • free_attrmap (属性マップ解放)

関連データ構造

  • LogicalRepRelMapEntry: 本関数が解放対象とするメインのデータ構造。
  • LogicalRepRelation: entry 内に含まれるリモートリレーション情報。

失敗モードと境界条件

  • entry が NULL の場合、あるいは entry 内の各ポインタが既に解放済みである場合はセグメンテーション違反を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数はキャッシュエントリの破棄時にのみ使用される。新しいフィールドを LogicalRepRelMapEntry に追加した場合は、必ず本関数で解放処理を更新すること。
