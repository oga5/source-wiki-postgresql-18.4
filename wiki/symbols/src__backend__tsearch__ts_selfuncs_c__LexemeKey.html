LexemeKey

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/ts_selfuncs.c:42

目的

全文検索の選択率推定処理において、語彙（Lexeme）をキーとして統計情報を検索・照合するための識別子として使用されます。

シグネチャまたは形状

typedef struct {
    char   *lexeme;
    int     length;
} LexemeKey;

入力と出力

  • 入力: 語彙を表す文字列ポインタと、その長さ。
  • 出力: 構造体インスタンスとして、検索キーや比較対象のキーを保持します。

前提条件と不変条件

  • lexeme は有効な文字列を指している必要があります。
  • length は lexeme の長さを正しく反映している必要があります。

アルゴリズムまたは振る舞い

bsearch 等の検索関数において、特定の語彙を特定するための比較キーとして機能します。統計情報内の語彙と照合する際のデータ構造として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照）:
  • src/backend/tsearch/ts_selfuncs.c:291: 検索キーとしてインスタンス化。
  • src/backend/tsearch/ts_selfuncs.c:438: 比較関数内で型キャストして参照。
  • 呼び出し先: 情報なし。

関連データ構造

  • TextFreq: 統計情報から読み取った語彙と頻度を保持する構造体であり、LexemeKey と組み合わせて検索対象となります。

失敗モードと境界条件

  • lexeme が NULL の場合や、length が実際の文字列長と一致しない場合に、比較関数での誤動作やメモリ不正アクセスのリスクがあります。

保守担当者向けメモ

  • 本構造体は ts_selfuncs.c 内の局所的な検索処理に特化しています。比較関数 compare_lexeme_textfreq 等と併用されるため、変更時はこれらの関数との整合性に注意してください。
