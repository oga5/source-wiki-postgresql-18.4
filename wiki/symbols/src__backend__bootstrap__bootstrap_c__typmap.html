typmap

定義

  • 種別: struct
  • 定義箇所: src/backend/bootstrap/bootstrap.c:142

目的

ブートストラップモードにおいて、システムカタログ pg_type が構築されるまでの過渡期に、データ型の OID とその属性情報を一時的に保持するためのハック的なデータ構造です。

シグネチャまたは形状

struct typmap
{
    Oid            am_oid;
    FormData_pg_type am_typ;
};

入力と出力

  • 入力: am_oid（型識別子）、am_typ（pg_type の行データ構造）。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • pg_type カタログがまだ存在しない、あるいは完全に利用可能ではない初期化フェーズで使用されます。
  • Typ リスト（List *）の要素として管理され、型情報のキャッシュとして機能します。

アルゴリズムまたは振る舞い

  • ブートストラップ処理中に、新しい型情報が定義されるたびに palloc でメモリを確保し、この構造体に値を格納して Typ リストに追加します。
  • 既存の型情報を検索する際、この構造体を通じて am_oid や型属性（サイズ、アライメント等）を参照します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/bootstrap/bootstrap.c 内の型定義・検索ロジック（gettype 等の文脈）。
  • 呼び出し先: なし（データ構造のため）。

関連データ構造

  • FormData_pg_type: am_typ フィールドの型。
  • Typ: struct typmap を要素として保持する静的な List。
  • Ap: struct typmap を指す静的なポインタ変数。

失敗モードと境界条件

  • メモリ不足: palloc 失敗時にシステム全体が停止する可能性があります。
  • 整合性: pg_type が構築された後のフェーズで、このキャッシュと実際のカタログ内容の間に不整合が生じないよう管理する必要があります。

保守担当者向けメモ

  • ソースコード内で「a hack」と明記されている通り、ブートストラップ特有の暫定的な仕組みです。
  • Typ リストを走査する際は、lfirst マクロを使用して struct typmap * にキャストしてアクセスします。
