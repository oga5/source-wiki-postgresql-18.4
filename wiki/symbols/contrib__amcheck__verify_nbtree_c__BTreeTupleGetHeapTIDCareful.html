BTreeTupleGetHeapTIDCareful

定義

  • 種別: function
  • 定義箇所: contrib/amcheck/verify_nbtree.c:3533

目的

B-Tree インデックス内のタプルからヒープ TID を安全に取得し、そのタプルがピボット（pivot）か非ピボット（non-pivot）かという期待される属性と、実際のタプルメタデータが一致しているかを検証する。

シグネチャまたは形状

static inline ItemPointer BTreeTupleGetHeapTIDCareful(BtreeCheckState *state, IndexTuple itup, bool nonpivot)

入力と出力

  • **入力**:
  • state: 検証状態を保持する BtreeCheckState 構造体。
  • itup: 対象の IndexTuple。
  • nonpivot: 呼び出し元が期待するタプル種別（true なら非ピボット、false ならピボット）。
  • **出力**:
  • 成功時: タプルに関連付けられた ItemPointer（ヒープ TID）。
  • 失敗時: ERRCODE_INDEX_CORRUPTED を伴うエラーを発生させ、処理を中断する。

前提条件と不変条件

  • state->heapkeyspace が真であること。
  • インデックスの構造が破損していない限り、非ピボットタプルには有効なヒープ TID が存在すること。

アルゴリズムまたは振る舞い

1. BTreeTupleIsPivot(itup) を用いてタプルの実態を確認する。 2. 呼び出し元が指定した nonpivot フラグと、実際のタプル属性（ピボットか否か）を比較する。

  • 期待と実態が一致しない場合、ERRCODE_INDEX_CORRUPTED でエラーを報告する。

    • 3. BTreeTupleGetHeapTID(itup) を呼び出し、ヒープ TID を取得する。 4. nonpivot が真であるにもかかわらず、取得した TID が ItemPointerIsValid で無効と判定された場合、破損としてエラーを報告する。 5. 検証を通過した TID を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_nbtree.c:3150
  • contrib/amcheck/verify_nbtree.c:3265
  • **呼び出し先**:
  • BTreeTupleIsPivot (マクロ)
  • BTreeTupleGetHeapTID (マクロ)
  • ItemPointerIsValid (マクロ)
  • ereport (エラー報告)

関連データ構造

  • BtreeCheckState: インデックス検証のコンテキストとターゲットブロック情報を保持。
  • IndexTuple: 検証対象の B-Tree タプル。

失敗モードと境界条件

  • **不整合エラー**: タプル種別（ピボット/非ピボット）の不一致、または非ピボットタプルにおけるヒープ TID の欠落が発生した場合、インデックス破損とみなして ERROR をスローする。
  • **境界条件**: ページ内のアイテムオフセットやページタイプに基づく期待値が正しく渡されない場合、誤ったエラー報告が発生する可能性がある。

保守担当者向けメモ

  • 本関数は amcheck の防御的検証の一部であり、インデックスの論理的整合性を厳格にチェックする。
  • nonpivot 引数は呼び出し側の文脈（ページタイプやオフセット）に依存するため、呼び出し側での判定ロジックに変更を加える際は注意が必要である。
