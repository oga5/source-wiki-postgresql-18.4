gbt_int8_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int8.c:135

目的

GiSTインデックス検索において、特定のインデックスノードが指定されたクエリ条件（int8型）を満たすかどうかを判定します。

シグネチャまたは形状

Datum gbt_int8_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY 構造体（インデックスキーを含む）。
  • PG_GETARG_INT64(1): 比較対象のクエリ値。
  • PG_GETARG_UINT16(2): 比較戦略（StrategyNumber）。
  • PG_GETARG_POINTER(4): recheck フラグへのポインタ。
  • **出力**:
  • Datum (bool): 条件を満たす場合は true、それ以外は false。
  • *recheck: 常に false が設定されます（本関数による判定は完全であるため）。

前提条件と不変条件

  • entry->key は int64KEY 構造体として解釈可能である必要があります。
  • リーフノードにおいては lower == upper であるという不変条件が前提となります。

アルゴリズムまたは振る舞い

1. recheck フラグを false に設定します。 2. entry->key から int64KEY を取得し、GBT_NUMKEY_R 構造体に範囲（lower, upper）をマッピングします。 3. gbt_num_consistent 関数を呼び出し、範囲の包含関係と戦略に基づいた判定結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのGiSTインデックスエンジン（直接の呼び出し元情報はソースコード内に明示されていません）。
  • **呼び出し先**: gbt_num_consistent（btree_utils_num.c で定義）。

関連データ構造

  • int64KEY: インデックスノードが保持する int8 型の範囲（lower, upper）を定義する構造体。
  • GBT_NUMKEY_R: 数値範囲を扱うための共通構造体。
  • tinfo: int8 型の比較ロジックを保持する静的メタデータ変数。

失敗モードと境界条件

  • 本関数は recheck を false に設定するため、インデックス検索結果に対する再評価は不要です。
  • 内部で呼び出される gbt_num_consistent が数値比較および範囲判定の境界条件を処理します。

保守担当者向けメモ

  • 本関数は int8 型専用のラッパーであり、実質的なロジックは btree_utils_num.c の gbt_num_consistent に委譲されています。
  • 新しい数値型を追加する際は、本関数をテンプレートとして利用してください。
