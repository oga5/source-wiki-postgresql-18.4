BackgroundWorkerArray

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/bgworker.c:94

目的

共有メモリ上に配置され、Postmasterおよび各バックエンドプロセス間でバックグラウンドワーカーの状態を共有・管理するためのデータ構造です。

シグネチャまたは形状

typedef struct BackgroundWorkerArray
{
    int            total_slots;
    uint32         parallel_register_count;
    uint32         parallel_terminate_count;
    BackgroundWorkerSlot slot[FLEXIBLE_ARRAY_MEMBER];
} BackgroundWorkerArray;

入力と出力

  • **入力**: 共有メモリ領域として確保され、BackgroundWorkerSlot 配列を通じてワーカーの登録情報や状態を保持します。
  • **出力**: Postmasterがこの構造体をスキャンすることで、ワーカーの起動、終了、再起動のライフサイクル管理を行います。

前提条件と不変条件

  • Postmasterは共有メモリのロックを取得できないため、本構造体へのアクセスには in_use フラグとメモリバリア（pg_read_barrier/pg_write_barrier）を用いたロックレスなプロトコルが必須です。
  • 共有メモリ上のデータであるため、Postmasterが読み取る際はデータ破損を想定したパラノイア的なチェックが推奨されます。

アルゴリズムまたは振る舞い

  • total_slots に基づき、固定数のワーカー用スロットを管理します。
  • parallel_register_count および parallel_terminate_count は、並列クエリ用ワーカーの登録・終了状況を追跡するために使用されます。
  • slot 配列は FLEXIBLE_ARRAY_MEMBER を使用しており、共有メモリ初期化時に必要なサイズ分だけ動的に確保されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/bgworker.c 内の各関数（BackgroundWorkerShmemInit 等）がこの構造体を初期化・参照します。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接的な関数呼び出しは行いません。

関連データ構造

  • BackgroundWorkerSlot: 各ワーカーの状態を保持する個別のスロット構造体。
  • BackgroundWorkerHandle: ワーカーを操作するためのハンドル。

失敗モードと境界条件

  • 共有メモリのサイズ計算時、offsetof(BackgroundWorkerArray, slot) を用いてヘッダーサイズを算出する必要があります。
  • スロット数が上限に達した場合、新規の動的ワーカー登録は失敗します。

保守担当者向けメモ

  • 本構造体は src/backend/postmaster/bgworker.c 内の静的変数 BackgroundWorkerData を通じてアクセスされます。
  • 並列ワーカー（BGWORKER_CLASS_PARALLEL）に関連するカウンタが含まれていますが、一般的な拡張機能での利用には適さないため注意してください。
