GetState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonfuncs.c:86

目的

JSONデータの解析およびパス指定による要素抽出を行う際、実行時の状態を保持するために使用されます。

シグネチャまたは形状

typedef struct GetState
{
	JsonLexContext *lex;
	text	   *tresult;
	const char *result_start;
	bool		normalize_results;
	bool		next_scalar;
	int			npath;
	char	  **path_names;
	int		   *path_indexes;
	bool	   *pathok;
	int		   *array_cur_index;
} GetState;

入力と出力

  • **入力**: palloc0 を介して初期化され、JSON解析のコンテキストやパス情報が格納されます。
  • **出力**: 解析結果（tresult）や、現在の走査状態（array_cur_index 等）が更新されます。

前提条件と不変条件

  • npath は、パス関連配列（path_names, path_indexes, pathok, array_cur_index）の長さを保持します。
  • pathok は、現在の解析深度が指定されたパスと一致しているかを示します。

アルゴリズムまたは振る舞い

JSONの解析プロセスにおいて、JsonLexContext を用いてトークンを走査し、path_names や path_indexes に基づいて目的のJSON断片を特定します。解析の進行に伴い、array_cur_index が各階層の現在のインデックスを追跡します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonfuncs.c 内の複数の関数（1110, 1162, 1181, 1199, 1246, 1297, 1337, 1355, 1403, 1447 行目など）で、状態管理用としてインスタンス化およびキャストされて利用されています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • JsonLexContext: JSON解析の基盤となるコンテキスト構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時（palloc0）にはPostgreSQLの標準的なエラーハンドリングが適用されます。
  • パスがJSON構造と一致しない場合、pathok が適切に更新され、抽出処理がスキップされる挙動をとります。

保守担当者向けメモ

  • 本構造体は jsonfuncs.c 内の抽出系関数における状態保持の要です。
  • 構造体の各フィールドは、JSONパスの走査深度と密接に関連しているため、パス処理ロジックを変更する際は npath と関連配列の整合性に注意してください。
