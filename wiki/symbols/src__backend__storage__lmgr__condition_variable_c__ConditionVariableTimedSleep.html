ConditionVariableTimedSleep

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/condition_variable.c:111

目的

指定された条件変数（ConditionVariable）に対して、タイムアウト付きでプロセスを待機させる。条件変数の待機リストへの登録と、ラッチ（MyLatch）を用いた待機状態への遷移を管理する。

シグネチャまたは形状

bool ConditionVariableTimedSleep(ConditionVariable *cv, long timeout, uint32 wait_event_info)

入力と出力

  • **入力**:
  • cv: 待機対象の条件変数。
  • timeout: タイムアウト時間（ミリ秒）。負の値はタイムアウトなしを意味する。
  • wait_event_info: 待機イベントの識別子。
  • **出力**:
  • bool: タイムアウトが発生した場合は true、シグナル等により起床した場合は false を返す。

前提条件と不変条件

  • 呼び出し前に ConditionVariablePrepareToSleep(cv) が実行されていることが推奨される。未準備の場合は本関数内で準備が行われ、即座に false が返される。
  • 共有メモリ内のデータ構造整合性を保つため、内部でスピンロック（cv->mutex）を使用する。
  • 呼び出し側は、Spurious Wakeup（偽の起床）を考慮し、必ず while ループ内で条件を再評価する必要がある。

アルゴリズムまたは振る舞い

1. **準備確認**: cv_sleep_target が対象の cv と一致するか確認し、不一致なら ConditionVariablePrepareToSleep を呼び出して終了する。 2. **待機設定**: タイムアウトが指定されている場合、開始時刻を記録し、WaitLatch 用のイベントフラグを設定する。 3. **待機ループ**:

  • WaitLatch を呼び出し、ラッチがセットされるまで待機する。
  • ResetLatch でラッチをリセットする。
  • スピンロックを取得し、自身が待機リストから外れている（シグナルを受けた）ことを確認する。外れている場合はリストに再挿入する。
  • CHECK_FOR_INTERRUPTS() を実行し、割り込み要求を処理する。
  • タイムアウトが指定されている場合、経過時間を計算し、期限を超過していれば true を返す。

関連データ構造

  • ConditionVariable: 待機リストとスピンロックを保持する構造体。
  • cv_sleep_target: 現在のプロセスが待機準備中の条件変数を指すスレッドローカル変数。
  • MyLatch: 現在のプロセスのラッチ。
  • MyProcNumber: 現在のプロセスの識別子。

失敗モードと境界条件

  • **タイムアウト**: 待機時間が経過した場合、true を返して終了する。
  • **割り込み**: CHECK_FOR_INTERRUPTS() により、クエリキャンセルやシャットダウン要求が処理される。
  • **偽の起床**: シグナル以外の理由でラッチがセットされた場合、ループを継続して再待機する。

保守担当者向けメモ

  • 本関数は ConditionVariableSleep と異なり、経過時間を追跡してタイムアウトを管理する。
  • 待機リストの操作は cv->mutex で保護されているため、リスト操作時は必ずロックを取得すること。
  • 呼び出し側が ConditionVariableCancelSleep を呼び出すまで、プロセスは待機リストに留まる必要がある。
