datetkn

定義

  • 種別: struct
  • 定義箇所: src/include/utils/datetime.h:207

目的

日付・時刻の解析処理において、キーワード（例: NOW, EPOCH）やフィールド型（例: YEAR, MONTH）を保持し、検索や識別を行うためのデータ構造です。

シグネチャまたは形状

typedef struct
{
    char        token[TOKMAXLEN + 1]; /* NUL終端されたトークン文字列 */
    char        type;                 /* フィールド型コード */
    int32       value;                /* 型に応じた値 */
} datetkn;

入力と出力

  • 本構造体は、解析器が入力文字列をトークン化する際の照合対象として使用されます。
  • type フィールドは DTK_* マクロ群に対応するビットマスクや識別子を保持し、後続の変換処理へ渡されます。

前提条件と不変条件

  • token は常に NUL 終端されている必要があります。
  • datetktbl 等のテーブル内で使用される場合、検索アルゴリズムの都合上、アルファベット順にソートされている必要があります。

アルゴリズムまたは振る舞い

  • src/backend/utils/adt/datetime.c 内の datebsearch 関数により、バイナリサーチの対象として利用されます。
  • 解析関数は入力文字列を datetkn の token と照合し、一致した場合は対応する type と value を用いて内部表現（pg_tm や fsec_t）への変換を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/datetime.c 内の解析ロジック（ParseDateTime 等）が、トークン検索のために本構造体を参照します。
  • **呼び出し先**: 直接的な関数呼び出し関係はありませんが、datebsearch 関数が本構造体の配列を引数として受け取ります。

関連データ構造

  • datetktbl / deltatktbl: datetkn 型の静的配列。
  • datecache / deltacache: datetkn へのポインタを保持するキャッシュ配列。
  • TimeZoneAbbrevTable: タイムゾーン短縮名の解析に関連する構造体。

失敗モードと境界条件

  • token の長さが TOKMAXLEN を超える場合、切り詰めや解析エラーが発生する可能性があります。
  • テーブルがソートされていない場合、datebsearch による検索が正しく機能しません。

保守担当者向けメモ

  • type フィールドのコード値（DTK_*）を変更する際は、INTERVAL 型の typmod 等への影響を考慮し、initdb が必要となる可能性がある点に注意してください。
  • 新しいキーワードを追加する際は、必ず datetktbl のソート順序を維持してください。
