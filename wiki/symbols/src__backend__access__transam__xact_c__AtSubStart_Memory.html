AtSubStart_Memory

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xact.c:1253

目的

サブトランザクション開始時に、そのサブトランザクション専用のメモリコンテキストを作成し、アクティブなメモリコンテキストを切り替える。

シグネチャまたは形状

static void AtSubStart_Memory(void)

入力と出力

  • 入力: なし（グローバルな CurrentTransactionState および CurrentMemoryContext を参照）
  • 出力: なし（副作用としてメモリコンテキストの状態を更新）

前提条件と不変条件

  • CurTransactionContext が NULL でないこと（Assert により検証）。
  • 現在のトランザクション状態（CurrentTransactionState）が有効であること。

アルゴリズムまたは振る舞い

1. 現在のトランザクション状態（s）を取得する。 2. サブトランザクション開始前のメモリコンテキストを s->priorContext に保存する。 3. AllocSetContextCreate を使用して、親の CurTransactionContext を親とする新しい CurTransactionContext を作成する。このコンテキストは、サブトランザクションのコミット時には生存し、アボート時には破棄されるデータを保持する。 4. 新しく作成したコンテキストを s->curTransactionContext に格納する。 5. MemoryContextSwitchTo を呼び出し、新しい CurTransactionContext をアクティブなメモリコンテキストに設定する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c 内のサブトランザクション開始処理（AtSubStart_Memory への直接的な呼び出し箇所はソースコードの行番号 366 および 5082 に存在）。
  • 呼び出し先: AllocSetContextCreate, MemoryContextSwitchTo（いずれも utils/memutils.h で定義）。

関連データ構造

  • TransactionStateData: トランザクション状態を保持する構造体。priorContext および curTransactionContext フィールドが更新される。
  • MemoryContext: PostgreSQLのメモリ管理単位。

失敗モードと境界条件

  • メモリ不足時には AllocSetContextCreate がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数はサブトランザクションのメモリ階層を構築する重要な役割を担う。サブトランザクションのアボート時にこのコンテキストが正しく解放されるよう、対応するクリーンアップ処理（AtSubAbort_Memory 等）との整合性に注意すること。
