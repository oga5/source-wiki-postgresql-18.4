ProcessCancelRequestPacket

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/backend_startup.c:893

目的

クライアントから送信されたキャンセルリクエストパケットを解析し、対象のバックエンドプロセスに対してキャンセル処理を要求する。

シグネチャまたは形状

static void ProcessCancelRequestPacket(Port *port, void *pkt, int pktlen)

入力と出力

  • **入力**:
  • port: 接続情報を保持する Port 構造体へのポインタ。
  • pkt: 受信したパケットデータへのポインタ。
  • pktlen: パケットのバイト長。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • パケット長が CancelRequestPacket 構造体の最小サイズ（cancelAuthCode フィールドのオフセット）以上であること。
  • キャンセルキーの長さが 1 バイト以上、かつ 256 バイト以下であること。

アルゴリズムまたは振る舞い

1. パケット長が最小サイズを満たしているか検証し、満たさない場合は COMMERROR を報告して終了する。 2. キャンセルキーの長さが有効範囲内（1〜256バイト）にあるか検証し、範囲外の場合は COMMERROR を報告して終了する。 3. パケットを CancelRequestPacket 構造体にキャストする。 4. pg_ntoh32 を用いてネットワークバイトオーダーの PID をホストバイトオーダーに変換する。 5. SendCancelRequest を呼び出し、PID、認証コード、キー長を渡してキャンセル処理を依頼する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/backend_startup.c 内の接続初期化処理（570行目付近）から呼び出される。
  • **呼び出し先**: SendCancelRequest（外部関数）。

関連データ構造

  • Port: 接続状態を管理する構造体。
  • CancelRequestPacket: キャンセルリクエストのプロトコル定義。

失敗モードと境界条件

  • **プロトコル違反**: パケット長が不正な場合、ERRCODE_PROTOCOL_VIOLATION を伴う COMMERROR が発生する。
  • **キー長不正**: キャンセルキーの長さが 0 または 256 バイトを超える場合、同様にプロトコル違反として処理される。

保守担当者向けメモ

  • 本関数は backend_startup.c 内で静的関数として定義されており、主に接続確立フェーズでのリクエスト処理に使用される。
  • SendCancelRequest への引き渡し時に PID のエンディアン変換を行っている点に注意すること。
