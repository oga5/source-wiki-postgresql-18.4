WalRcvRunning

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walreceiverfuncs.c:75

目的

WALレシーバープロセスが現在稼働中であるか、あるいは起動中であるかを判定し、起動プロセスがタイムアウトした場合には強制的に停止状態へ遷移させる。

シグネチャまたは形状

bool WalRcvRunning(void)

入力と出力

  • **入力**: なし（共有メモリ上のグローバル構造体 WalRcv を参照）。
  • **出力**: bool 型。WALレシーバーが稼働中または起動中であれば true、停止状態であれば false を返す。

前提条件と不変条件

  • 共有メモリ上の WalRcv 構造体に対して、スピンロック（mutex）を用いた排他制御が行われる。
  • 起動プロセスが WALRCV_STARTUP_TIMEOUT を超えて WALRCV_STARTING 状態にある場合、本関数は自動的に状態を WALRCV_STOPPED に更新する。

アルゴリズムまたは振る舞い

1. 共有メモリから現在の walRcvState と startTime を取得する。 2. 状態が WALRCV_STARTING であり、かつ現在時刻との差が WALRCV_STARTUP_TIMEOUT を超過している場合：

  • 再度スピンロックを取得し、状態を WALRCV_STOPPED に更新する。
  • 状態遷移を通知するため ConditionVariableBroadcast を実行する。

    • 3. 最終的な状態が WALRCV_STOPPED でなければ true を、そうでなければ false を返す。

関連データ構造

  • WalRcvData: WALレシーバーの状態を保持する共有メモリ構造体。
  • WalRcvState: プロセスの状態（WALRCV_STARTING, WALRCV_STOPPED 等）。

失敗モードと境界条件

  • **タイムアウト**: 起動処理が WALRCV_STARTUP_TIMEOUT を超えると、WALレシーバーが後に起動しても即座に終了するよう強制停止状態に遷移させる。
  • **競合**: スピンロックにより、状態確認と更新の間の競合を防ぐ。

保守担当者向けメモ

  • 本関数はスタートアッププロセスがリカバリの進捗を判断するために頻繁に呼び出す。
  • タイムアウト処理は、WALレシーバーが何らかの理由で起動に失敗した際に、リカバリプロセスが永久に待機状態になるのを防ぐ重要な安全装置である。
