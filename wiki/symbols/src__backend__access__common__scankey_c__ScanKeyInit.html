ScanKeyInit

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/scankey.c:75

目的

インデックススキャンやテーブルスキャンで使用する ScanKeyData 構造体を、標準的な設定値で初期化します。

シグネチャまたは形状

void ScanKeyInit(ScanKey entry, AttrNumber attributeNumber, StrategyNumber strategy, RegProcedure procedure, Datum argument)

入力と出力

  • **入力**:
  • entry: 初期化対象の ScanKey 構造体へのポインタ。
  • attributeNumber: 検索対象の属性番号。
  • strategy: 比較演算子の戦略番号。
  • procedure: 比較関数の OID。
  • argument: 比較に使用する値（Datum）。
  • **出力**: なし（entry が指す構造体が更新される）。

前提条件と不変条件

  • entry は有効なメモリ領域を指している必要があります。
  • fmgr_info が内部でメモリを割り当てる可能性があるため、ScanKey の生存期間中、呼び出し元の CurrentMemoryContext が有効である必要があります。

アルゴリズムまたは振る舞い

1. entry->sk_flags を 0 に初期化します。 2. attributeNumber、strategy、argument を構造体の各フィールドに代入します。 3. sk_subtype を InvalidOid に設定します。 4. sk_collation を C_COLLATION_OID に固定設定します。 5. fmgr_info を呼び出し、指定された procedure (OID) に基づいて関数メタデータを entry->sk_func に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_heapam.c
  • contrib/dblink/dblink.c
  • contrib/sepgsql/ 配下の各ファイル（database.c, proc.c, relation.c, schema.c）
  • **呼び出し先**:
  • fmgr_info (関数ルックアップ)

関連データ構造

  • ScanKeyData: 初期化対象となる構造体。
  • FmgrInfo: sk_func フィールドに格納される関数情報構造体。

失敗モードと境界条件

  • NULL 検索や特殊な照合順序が必要なケースには対応していません（これらが必要な場合は ScanKeyEntryInitialize を使用してください）。
  • procedure に無効な OID が指定された場合、fmgr_info の挙動に従いエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は簡略版の初期化インターフェースです。デフォルトで C 照合順序が適用される点に注意してください。
  • 頻繁に呼び出されるスキャン処理において、fmgr_info のルックアップコストを削減したい場合は、ScanKeyEntryInitializeWithInfo の利用を検討してください。
