MemoryContextData

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/memnodes.h:117

目的

PostgreSQLのメモリ管理サブシステムにおいて、メモリコンテキストの基底となるデータ構造を提供します。各メモリコンテキスト実装の先頭に配置されることで、階層管理やメタデータ保持のための共通インターフェースとして機能します。

シグネチャまたは形状

typedef struct MemoryContextData {
    pg_node_attr(abstract)
    NodeTag     type;
    bool        isReset;
    bool        allowInCritSection;
    Size        mem_allocated;
    const MemoryContextMethods *methods;
    MemoryContext parent;
    MemoryContext firstchild;
    MemoryContext prevchild;
    MemoryContext nextchild;
    const char *name;
    const char *ident;
    MemoryContextCallback *reset_cbs;
} MemoryContextData;

入力と出力

  • 本構造体は直接的な関数ではありませんが、MemoryContextMethods（仮想関数テーブル）を介して、メモリの確保・解放・リセットといった操作の入力として利用されます。

前提条件と不変条件

  • **継承**: 新しいメモリコンテキスト型を定義する場合、必ず本構造体を先頭メンバとして配置する必要があります。
  • **階層構造**: parent、firstchild、prevchild、nextchild ポインタにより、メモリコンテキストは木構造を形成します。
  • **抽象性**: pg_node_attr(abstract) が示す通り、本構造体そのもののインスタンスは作成されません。

アルゴリズムまたは振る舞い

  • **仮想関数呼び出し**: methods メンバに格納された関数ポインタを通じて、具体的なメモリ管理実装（AllocSet や Slab 等）の処理が実行されます。
  • **階層管理**: 親コンテキストがリセットまたは削除される際、firstchild から始まる子コンテキストのリストを辿り、再帰的に処理が行われます。
  • **状態追跡**: isReset は前回の初期化以降にメモリ確保が行われていないかを示し、mem_allocated は当該コンテキストで確保されたメモリ量を追跡します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mmgr/mcxt.c 等のメモリ管理API、および elog.c によるエラーデータ処理などで参照されます。
  • **呼び出し先**: methods メンバに設定された関数ポインタを通じて、各実装（aset.c, slab.c, bump.c, generation.c）の関数が呼び出されます。

関連データ構造

  • MemoryContextMethods: 仮想関数テーブル。
  • MemoryContext: 本構造体へのポインタ型。
  • MemoryContextCallback: リセット/削除時に実行されるコールバックリスト。

失敗モードと境界条件

  • parent が NULL の場合、そのコンテキストはトップレベルコンテキストとして扱われます。
  • methods が正しく初期化されていない状態でメモリ操作を行うと、未定義の動作を引き起こします。

保守担当者向けメモ

  • 新しいメモリコンテキスト型を追加した際は、MemoryContextIsValid マクロを更新してください。
  • 構造体の先頭メンバとしての配置が必須であるため、メンバの順序変更には細心の注意を払ってください。
