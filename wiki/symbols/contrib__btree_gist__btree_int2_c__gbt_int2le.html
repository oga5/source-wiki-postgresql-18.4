gbt_int2le

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int2.c:44

目的

int2（2バイト整数）型の値同士を比較し、左辺が右辺以下であるか（<=）を判定する。

シグネチャまたは形状

bool gbt_int2le(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の int16 値へのポインタ。
  • b: 比較対象の int16 値へのポインタ。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • bool: *a <= *b が真であれば true、そうでなければ false を返す。

前提条件と不変条件

  • a および b は、有効な int16 型の値を指している必要がある。
  • flinfo は NULL でも動作する（関数内で参照されていないため）。

アルゴリズムまたは振る舞い

1. ポインタ a および b を const int16 * にキャストする。 2. それぞれのポインタが指す値をデリファレンスし、C 言語の <= 演算子を用いて比較を行う。 3. 比較結果をそのまま戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_int2.c:87 にて、静的構造体 tinfo の初期化時に参照されている。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • int16: PostgreSQL の 2 バイト整数型。

失敗モードと境界条件

  • ポインタ a または b が NULL の場合、デリファレンスによりセグメンテーション違反が発生する可能性がある。
  • 整数値の範囲外（オーバーフロー等）については、C 言語の標準的な整数比較の挙動に従う。

保守担当者向けメモ

  • 本関数は btree_gist 拡張における int2 型の比較ロジックの一部であり、btree_utils_num が提供する汎用的な GiST インデックス操作から利用されることを想定している。
  • 類似の関数として gbt_int2lt, gbt_int2gt, gbt_int2ge が同ファイル内に存在する。
