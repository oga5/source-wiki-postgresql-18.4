PgStat_EntryRef

定義

  • **種別**: struct
  • **定義箇所**: src/include/utils/pgstat_internal.h:135

目的

バックエンドが共有メモリ上の統計エントリへ効率的にアクセスし、未フラッシュの統計更新（pending）を一時的に保持するためのローカルキャッシュ構造体です。

シグネチャまたは形状

typedef struct PgStat_EntryRef
{
    PgStatShared_HashEntry *shared_entry;
    PgStatShared_Common    *shared_stats;
    uint32                  generation;
    void                   *pending;
    dlist_node              pending_node;
} PgStat_EntryRef;

入力と出力

  • **入力**: 共有メモリ上の PgStatShared_HashEntry へのポインタおよび統計データ。
  • **出力**: pending メンバを介して、統計種別ごとの flush_pending_cb コールバックへ渡される更新データ。

前提条件と不変条件

  • shared_stats は shared_entry->body を指し、dsa_get_address() の呼び出しを避けるためのローカルキャッシュとして機能する。
  • generation は共有エントリの再利用回数を追跡し、一貫性を維持する。
  • pending_node は、未フラッシュの統計を管理するバックエンドローカルなリスト pgStatPending に所属する。

アルゴリズムまたは振る舞い

1. **アクセス**: バックエンドは本構造体を通じて共有統計へアクセスし、競合を避けるために更新を pending 領域に蓄積する。 2. **フラッシュ**: pgstat_report_stat() 等の契機で、PgStat_KindInfo に定義された flush_pending_cb が呼び出され、pending データが共有メモリへマージされる。 3. **生存期間**: 統計エントリの削除は参照カウントがゼロになるまで遅延される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c 内の統計管理関数群（pgstat_get_entry_ref 等）。
  • **呼び出し先**: PgStat_KindInfo 内の flush_pending_cb（統計種別ごとのフラッシュ処理）。

関連データ構造

  • PgStatShared_HashEntry: 共有ハッシュテーブル上の実体。
  • PgStatShared_Common: 統計データ本体。
  • PgStat_KindInfo: 統計種別ごとの振る舞いを定義するメタデータ。
  • dlist_node: 未フラッシュ統計のリスト管理に使用。

失敗モードと境界条件

  • 共有メモリ上の統計エントリが削除された場合、generation の不一致により古い参照が無効化される可能性がある。
  • 統計種別ごとに pending データのフォーマットが異なるため、flush_pending_cb の実装が不適切であるとデータマージに失敗する。

保守担当者向けメモ

  • 本構造体は共有メモリではなくバックエンドローカルなメモリ上に配置される。
  • shared_stats へのアクセスは、共有メモリの競合を最小限にするための最適化である。
  • 統計エントリのガベージコレクション（gc_request_count）との相互作用に注意すること。
