inet_cidr_pton_ipv4

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/inet_net_pton.c:95

目的

IPv4 アドレスまたは CIDR 表記の文字列を、ネットワークバイトオーダーのバイナリ形式に変換し、プレフィックス長（ビット長）を返します。

シグネチャまたは形状

static int inet_cidr_pton_ipv4(const char *src, u_char *dst, size_t size);

入力と出力

  • src: 変換対象の IPv4 アドレス文字列（ドット区切り10進数または16進数）。
  • dst: 変換後のバイナリデータを格納するバッファ。
  • size: dst のバッファサイズ。
  • 戻り値: 成功時はプレフィックス長（0〜32）、失敗時は -1 を返す。

前提条件と不変条件

  • dst は少なくとも 4 バイト以上の領域を指している必要があります。
  • 変換後のデータはネットワークバイトオーダーで格納されます。

アルゴリズムまたは振る舞い

1. **パース**:

  • 0x で始まる場合は16進数として、それ以外はドット区切りの10進数としてオクテットを解析します。

    • 2. **CIDR 処理**:

  • 文字列中に / が存在する場合、その後の数値をプレフィックス長として読み取ります。
  • / がない場合、アドレスのオクテット数に基づき、クラスフルルール（Class A〜E）に従ってデフォルトのプレフィックス長を推論します。

    • 3. **補完**:

  • プレフィックス長が指定されたオクテット数より大きい場合、不足分をゼロで埋めてネットワークアドレスを拡張します。

    • 4. **検証**:

  • 不正なフォーマットや範囲外の数値、バッファサイズ不足が発生した場合はエラーを返します。

呼び出し元と呼び出し先

  • 呼び出し元: 同一ファイル内の pg_inet_net_pton から呼び出されます。
  • 呼び出し先: 標準ライブラリの strchr, isdigit, isxdigit, tolower, assert 等を使用します。

関連データ構造

  • u_char: ネットワークアドレスの各オクテットを保持するために使用されます。

失敗モードと境界条件

  • ENOENT: 文字列が不正な形式である、またはアドレスが検出できない場合に設定されます。
  • EMSGSIZE: バッファサイズが不足している、またはプレフィックス長が 32 を超える場合に設定されます。

保守担当者向けメモ

  • 本関数は ISC ライセンスに基づく外部由来のコードです。
  • size 引数に -1 を渡すことで、バッファサイズ制限を回避する特殊な挙動が設計に含まれています。
  • IPv4 クラスフルルールの推論ロジックは、*odst の値に基づいて固定的に行われます。
