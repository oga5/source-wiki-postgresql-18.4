bbsink_archive_contents

定義

  • 種別: function
  • 定義箇所: src/include/backup/basebackup_sink.h:199

目的

バックアップのアーカイブデータを、シンクチェーン内の現在のシンクから次段のシンクへ転送・処理するために呼び出されます。

シグネチャまたは形状

static inline void bbsink_archive_contents(bbsink *sink, size_t len)

入力と出力

  • **入力**:
  • sink: 処理対象の bbsink 構造体へのポインタ。
  • len: 転送するデータのバイト数。
  • **出力**: なし（副作用として bbs_ops->archive_contents が実行される）。

前提条件と不変条件

  • sink は NULL であってはならない。
  • len は 0 より大きく、かつ sink->bbs_buffer_length（バッファ容量）以下であること。

アルゴリズムまたは振る舞い

1. sink が NULL でないことを確認する（Assert）。 2. len が有効な範囲内（0 < len <= bbs_buffer_length）であることを確認する（Assert）。 3. sink->bbs_ops->archive_contents を呼び出し、具体的なシンクの処理ロジックを実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c (複数箇所)
  • src/backend/backup/basebackup_gzip.c (圧縮シンクの実装内)
  • **呼び出し先**:
  • sink->bbs_ops->archive_contents (各シンク固有の実装関数)

関連データ構造

  • bbsink: シンクの基本構造体。
  • bbsink_ops: 各種バックアップイベントに対するコールバック関数テーブル。

失敗モードと境界条件

  • sink が NULL の場合、アサーション失敗によりプロセスが停止する可能性がある。
  • len が 0 またはバッファ容量を超過している場合、アサーション失敗によりプロセスが停止する。

保守担当者向けメモ

  • 本関数はインライン関数であり、各シンクの archive_contents コールバックを呼び出すためのラッパーとして機能する。
  • 呼び出し側は、この関数を呼ぶ前にバッファ（bbs_buffer）を適切に満たしておく必要がある。
