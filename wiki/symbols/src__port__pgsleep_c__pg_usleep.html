pg_usleep

定義

  • **種別**: function
  • **定義箇所**: src/port/pgsleep.c:40

目的

プラットフォーム非依存な方法で、呼び出し元プロセスをマイクロ秒単位で一時停止させる。

シグネチャまたは形状

void pg_usleep(long microsec)

入力と出力

  • **入力**: microsec (long型) - 停止させるマイクロ秒数。
  • **出力**: なし（void型）。

前提条件と不変条件

  • microsec が 0 以下の場合は何も行わずに即座に復帰する。
  • OS のカーネルティック周期に依存するため、指定した時間よりも長くスリープする可能性がある（最大 20ms 程度の誤差が生じることがある）。

アルゴリズムまたは振る舞い

1. microsec > 0 の場合のみ処理を実行する。 2. **Unix系環境**:

  • microsec を秒（tv_sec）とナノ秒（tv_nsec）に分割し、struct timespec を構築する。
  • nanosleep を呼び出してプロセスを停止させる。

    • 3. **Windows環境**:

  • microsec をミリ秒に換算（500μs未満は1msに切り上げ、それ以上は (microsec + 500) / 1000 で四捨五入）する。
  • SleepEx を呼び出してプロセスを停止させる。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下のソースコードから呼び出されていることが確認されている。
  • contrib/auth_delay/auth_delay.c
  • src/backend/access/nbtree/nbtpage.c
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogutils.c
  • src/backend/commands/vacuum.c
  • src/backend/libpq/pqcomm.c
  • src/backend/port/win32/socket.c
  • **呼び出し先**:
  • Unix系: nanosleep
  • Windows: SleepEx

関連データ構造

  • struct timespec (Unix系でのスリープ時間指定に使用)

失敗モードと境界条件

  • **精度**: OS のスケジューリング精度に依存し、指定時間より長い待機が発生し得る。
  • **バックエンドでの利用**: バックエンドプロセスで長時間スリープするとシグナルやラッチへの応答性が低下するため、可能な限り WaitLatch() の使用が推奨される。
  • **Windows環境**: WIN32 が定義されていない環境でのみコンパイルされる。Windows バックエンドでは本関数ではなく src/backend/port/win32/signal.c の実装が使用される。

保守担当者向けメモ

  • 本関数はフロントエンドツールやバックエンドプロセスでの簡易的な遅延処理を目的としている。
  • 割り込みや即時応答が必要なバックエンド処理には適さないため、代替手段の検討が必要な場合は WaitLatch() を参照すること。
