ExecReScanLockRows

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeLockRows.c:402

目的

LockRows ノードの実行状態をリセットし、プランツリーの再スキャンを準備します。

シグネチャまたは形状

void ExecReScanLockRows(LockRowsState *node)

入力と出力

  • **入力**: LockRowsState *node（対象となる LockRows ノードの実行状態構造体）
  • **出力**: なし（副作用としてプラン状態を更新）

前提条件と不変条件

  • node は有効な LockRowsState を指している必要があります。
  • outerPlanState(node) によって取得される下位プラン状態が初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. outerPlanState(node) を介して下位ノードのプラン状態を取得します。 2. 下位ノードの chgParam が NULL である（パラメータ変更による再スキャンが不要な）場合のみ、ExecReScan を呼び出して下位ノードをリセットします。 3. chgParam が NULL でない場合、下位ノードは最初の ExecProcNode 呼び出し時に自動的に再スキャンされるため、本関数では何もしません。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:294
  • **呼び出し先**: ExecReScan（実行エンジン内の汎用再スキャン関数）

関連データ構造

  • LockRowsState: LockRows ノードの実行時状態を保持する構造体。
  • PlanState: プランツリーの実行状態を保持する基底構造体。

失敗モードと境界条件

  • node が NULL の場合、セグメンテーション違反が発生する可能性があります。
  • 下位ノードが NULL の場合、outerPlanState の呼び出しで失敗する可能性があります。

保守担当者向けメモ

  • 本関数は LockRows ノード固有の再スキャンロジックを実装しています。EPQ（EvalPlanQual）の状態管理（lr_epqstate）のリセットが必要な場合、将来的に本関数内での明示的な処理追加が検討される可能性がありますが、現時点では下位ノードの再スキャンのみが責務です。
