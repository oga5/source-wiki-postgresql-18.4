spgist_name_inner_consistent

定義

  • 種別: function
  • 定義箇所: src/test/modules/spgist_name_ops/spgist_name_ops.c:265

目的

SP-GiSTインデックスの内部ノードにおいて、検索条件（scankeys）に基づき、どの下位ノードを走査すべきかを判定する。

シグネチャまたは形状

Datum spgist_name_inner_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgInnerConsistentIn 構造体を受け取る。これには、現在のレベル、再構築された値、ノードラベル、および検索キーが含まれる。
  • **出力**: spgInnerConsistentOut 構造体に、一致したノード番号、レベルの増分、および次のレベルへ引き継ぐ再構築された値を格納する。戻り値は void。

前提条件と不変条件

  • in->reconstructedValue は、以前の呼び出しで生成されたロング形式の varlena ヘッダーを持つ text 型データであると仮定する。
  • 再構築された値は name 型ではなく text 型として扱われる。
  • in->level は、以前に再構築された値の長さを表す。

アルゴリズムまたは振る舞い

1. **値の再構築**: 親ノードから引き継いだ値、現在のタプルのプレフィックス、およびノードラベルを連結し、現在のノードが表す文字列を生成する。 2. **ノード走査**: 各子ノードに対して、ノードラベルを連結して完全な文字列を構成する。 3. **条件判定**: 各子ノードの文字列と scankeys を比較する。

  • BTLess / BTLessEqual: 文字列が検索キーより大きい場合は不一致。
  • BTEqual: 文字列が検索キーと一致しない、または長さが不足している場合は不一致。
  • BTGreater / BTGreaterEqual: 文字列が検索キーより小さい場合は不一致。

    • 4. **結果の格納**: 条件を満たすノードのみを out 構造体に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのSP-GiSTインデックス走査エンジン（直接的な呼び出し元コードはソース内に明示されていないが、PG_FUNCTION_INFO_V1 マクロにより登録されている）。
  • **呼び出し先**: palloc, DatumGetTextPP, memcpy, datumCopy, elog などのPostgreSQL内部関数。

関連データ構造

  • spgInnerConsistentIn: 検索入力パラメータ。
  • spgInnerConsistentOut: 検索結果出力パラメータ。
  • text: 内部的な文字列保持に使用される型。

失敗モードと境界条件

  • 未知の strategy が指定された場合、elog(ERROR) により処理を中断する。
  • nodeChar <= 0 の場合、ダミーノードとして扱い、データ連結を行わない。

保守担当者向けメモ

  • 本関数は照合順序（Collation）を考慮せず、memcmp によるバイナリ比較を行う。
  • reconstructedValue の再構築ロジックは、コアの spgtextproc.c と同様のパターンに従っているため、変更時は同ファイルとの整合性を確認すること。
