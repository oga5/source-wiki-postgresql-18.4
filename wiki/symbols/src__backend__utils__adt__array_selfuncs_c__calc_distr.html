calc_distr

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_selfuncs.c:1009

目的

配列要素の出現確率分布を計算し、包含関係の選択率推定における確率モデルを構築します。

シグネチャまたは形状

static float *calc_distr(const float *p, int n, int m, float rest)

入力と出力

  • **入力**:
  • p: 各要素の出現確率を格納した配列。
  • n: p の要素数。
  • m: 計算対象の最大要素数（行列の列数）。
  • rest: 統計情報に含まれない「その他」の要素が占める確率の合計。
  • **出力**:
  • float *: 計算された確率分布（サイズ m + 1 の配列）。呼び出し元で pfree する必要があります。

前提条件と不変条件

  • p は n 個の確率値を持つ有効なポインタであること。
  • m は計算に必要な最大要素数として正の整数であること。
  • 内部で palloc を使用しているため、メモリ不足時はエラーが発生します。

アルゴリズムまたは振る舞い

1. **確率分布の計算**: 各要素の出現を独立事象と仮定し、動的計画法を用いて要素数ごとの確率分布を算出します。 2. **行列の最適化**: 計算に必要なメモリを2行分（現在と前回）に限定し、反復的に更新します。 3. **ポアソン分布による補正**: rest が DEFAULT_CONTAIN_SEL を超える場合、統計情報にない要素の出現をポアソン分布でモデル化し、既存の分布と畳み込み演算を行って補正します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/array_selfuncs.c 内の統計情報処理ロジック（878行目、879行目）。
  • **呼び出し先**:
  • palloc / pfree (PostgreSQL メモリ管理)
  • exp (math.h)

関連データ構造

  • float *: 確率分布を保持する動的配列。

失敗モードと境界条件

  • rest が小さい場合、ポアソン分布による補正はスキップされます。
  • n や m が 0 の場合、初期状態の分布が返されます。
  • palloc が失敗した場合、PostgreSQL のエラーハンドリングにより処理が中断されます。

保守担当者向けメモ

  • 本関数は array_selfuncs.c 内の選択率推定における中核的な統計計算ロジックです。
  • 畳み込み演算の計算量は O(m^2) です。m が非常に大きい場合、プランナのコスト計算時間に影響を与える可能性があるため注意してください。
