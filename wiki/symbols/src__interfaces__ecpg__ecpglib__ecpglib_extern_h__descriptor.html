descriptor

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/ecpglib/ecpglib_extern.h:112

目的

ECPG（Embedded SQL for C）において、SQL記述子（Descriptor）の情報を保持し、ランタイムライブラリ内で管理するための構造体です。

シグネチャまたは形状

struct descriptor
{
	char	   *name;
	PGresult   *result;
	struct descriptor *next;
	int			count;
	struct descriptor_item *items;
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。各フィールドは以下の通りです。

  • name: 記述子を識別するための名前。
  • result: libpq の PGresult オブジェクトへのポインタ。
  • next: 記述子のリストを構成するための次要素へのポインタ。
  • count: 記述子内の項目数。
  • items: descriptor_item 構造体のリストへのポインタ。

前提条件と不変条件

  • ecpglib 内のメモリ管理ポリシーに従い、動的に割り当てられることが想定されます。
  • next ポインタにより、複数の記述子が連結されたリスト構造を形成します。

アルゴリズムまたは振る舞い

  • ECPGランタイムにおいて、SQL実行結果のメタデータやホスト変数とのマッピング情報を保持するために使用されます。
  • descriptor.c 等のモジュール内で、記述子の検索、追加、削除といったリスト操作の対象となります。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な参照関係は、提供された情報からは特定できません。ツールが検出した contrib/ 配下の descriptor への言及は、PostgreSQLのコア機能や他の拡張モジュールにおける「タプル記述子」を指すものであり、本ECPGの struct descriptor とは別物である可能性が高いです。
  • **呼び出し先**: PGresult（libpq）および descriptor_item 構造体を参照します。

関連データ構造

  • struct descriptor_item: 記述子内の個々の項目詳細を保持します。
  • PGresult: libpq が提供するクエリ結果構造体。

失敗モードと境界条件

  • メモリ不足時に name や items の割り当てが失敗する可能性があります。
  • 記述子名が重複した場合のリスト管理における整合性維持が重要です。

保守担当者向けメモ

  • 本構造体はECPGライブラリ内部専用です。
  • contrib/ 配下のコードで見られる「tuple descriptor」という用語と混同しないよう注意してください。
  • 記述子のメモリ解放を行う際は、ecpg_free を使用してメモリ管理ポリシーを遵守してください。
