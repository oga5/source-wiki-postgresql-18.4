getNextGISTSearchItem

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistget.c:537

目的

GiSTインデックスのスキャンにおいて、優先度付きキュー（pairingheap）から次に探索すべき検索アイテムを取得します。

シグネチャまたは形状

static GISTSearchItem *getNextGISTSearchItem(GISTScanOpaque so)

入力と出力

  • **入力**: so (GISTScanOpaque) - 現在のスキャン状態を保持する構造体。内部の queue を参照します。
  • **出力**: GISTSearchItem * - キューから取り出されたアイテム。キューが空の場合は NULL を返します。

前提条件と不変条件

  • so->queue が初期化されている必要があります。
  • 返却された GISTSearchItem ポインタのメモリ解放（pfree）の責任は呼び出し元にあります。

アルゴリズムまたは振る舞い

1. so->queue が空でないかを確認します。 2. 空でない場合、pairingheap_remove_first を呼び出し、ヒープから最小要素を取り出して返却します。 3. 空である場合、NULL を返して探索の終了を示します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistget.c 内の複数箇所（gistgettuple 等の検索ロジック内で、次の探索対象を取得するために使用）。
  • **呼び出し先**:
  • pairingheap_is_empty
  • pairingheap_remove_first

関連データ構造

  • GISTScanOpaque: スキャン状態を管理し、pairingheap を保持する構造体。
  • GISTSearchItem: 探索対象のページやタプル情報を保持する構造体。

失敗モードと境界条件

  • キューが空の状態で呼び出された場合、エラーにはならず NULL を返します。呼び出し元はこの戻り値が NULL かどうかを判定してループを終了させる必要があります。

保守担当者向けメモ

  • 本関数は単なるキューからの取り出し処理であり、メモリ管理の責任が呼び出し元にある点に注意してください。
  • 呼び出し元で pfree を忘れるとメモリリークの原因となります。
