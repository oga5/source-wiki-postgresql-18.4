slist_check

定義

  • 種別: function
  • 定義箇所: src/backend/lib/ilist.c:113

目的

単方向連結リストの構造的な整合性を検証します。主にリスト内に循環参照が存在しないかを確認し、リストが有限であることを保証するために使用されます。

シグネチャまたは形状

void slist_check(const slist_head *head)

入力と出力

  • **入力**: head - 検証対象の単方向連結リストのヘッドポインタ（const slist_head *）。
  • **出力**: なし。正常時は終了し、異常時は elog(ERROR, ...) を通じてエラーを報告します。

前提条件と不変条件

  • head は NULL であってはなりません。NULL の場合はエラーとなります。
  • リストは有限である必要があります。

アルゴリズムまたは振る舞い

1. head が NULL かどうかをチェックし、該当する場合はエラーを発生させます。 2. リストの先頭から順に next ポインタを辿り、リストの終端（NULL）に到達するまで走査を繰り返します。 3. 走査中に循環参照が存在する場合、このループは終了しません。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/lib/ilist.c:51
  • src/include/lib/ilist.h:997, 1011, 1035, 1045
  • **呼び出し先**:
  • elog (PostgreSQLのエラー報告関数)

関連データ構造

  • slist_head: 単方向連結リストのヘッド構造体。
  • slist_node: 単方向連結リストのノード構造体。

失敗モードと境界条件

  • **NULLポインタ**: head が NULL の場合、elog によりエラーがスローされます。
  • **循環リスト**: リスト内に循環が含まれている場合、この関数は無限ループに陥ります。

保守担当者向けメモ

  • 本関数はデバッグ用途を主眼としており、ILIST_DEBUG が有効な環境での使用を想定しています。
  • 単方向リストの性質上、リストの終端到達確認以外の高度な整合性チェック（二重連結リストにおける prev ポインタの検証など）は行えません。
