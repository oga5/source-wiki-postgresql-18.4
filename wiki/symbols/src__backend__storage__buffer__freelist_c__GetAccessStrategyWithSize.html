GetAccessStrategyWithSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/freelist.c:625

目的

指定されたサイズ（KB）に基づき、特定の操作（スキャンやバルク処理）で使用する BufferAccessStrategy オブジェクトを動的に割り当てて初期化します。

シグネチャまたは形状

BufferAccessStrategy GetAccessStrategyWithSize(BufferAccessStrategyType btype, int ring_size_kb);

入力と出力

  • **入力**:
  • btype: BufferAccessStrategyType 型。割り当てる戦略のタイプを指定。
  • ring_size_kb: int 型。リングバッファのサイズをKB単位で指定。
  • **出力**:
  • 成功時: 初期化された BufferAccessStrategy 構造体へのポインタ。
  • 失敗時（または ring_size_kb が 0 の場合）: NULL。

前提条件と不変条件

  • ring_size_kb は 0 以上であること（Assert により検証）。
  • NBuffers（共有バッファ総数）は 16 以上であることを前提としています。
  • 割り当てられるリングバッファ数は、共有バッファ総数の 1/8 を上限とします。

アルゴリズムまたは振る舞い

1. **サイズ変換**: ring_size_kb をブロックサイズ（BLCKSZ）に基づき、バッファ数（ring_buffers）に換算します。 2. **早期リターン**: ring_buffers が 0 の場合は NULL を返します。 3. **上限設定**: ring_buffers を NBuffers / 8 と比較し、小さい方の値を採用します。 4. **メモリ割り当て**: palloc0 を使用し、BufferAccessStrategyData 構造体と、それに続く ring_buffers 分の Buffer 配列を確保します。 5. **初期化**: 構造体の btype および nbuffers フィールドを設定し、オブジェクトを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/vacuum.c
  • src/backend/commands/vacuumparallel.c
  • src/backend/postmaster/autovacuum.c
  • src/backend/storage/buffer/freelist.c（GetAccessStrategy からの呼び出し）
  • **呼び出し先**:
  • palloc0（メモリ確保）

関連データ構造

  • BufferAccessStrategyData: リングバッファ管理用のメイン構造体。
  • BufferAccessStrategyType: 戦略の種類を定義する列挙型。

失敗モードと境界条件

  • ring_size_kb が 0 の場合、戦略は不要とみなされ NULL が返されます。
  • メモリ不足時には palloc0 がエラーを発生させます。

保守担当者向けメモ

  • 本関数は BufferAccessStrategy のメモリレイアウト（構造体の末尾に可変長のバッファ配列を持つ）に依存しています。構造体定義を変更する際は、offsetof を使用した割り当てサイズ計算に注意してください。
