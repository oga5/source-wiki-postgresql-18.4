PLyObject_FromJsonbContainer

定義

  • 種別: function
  • 定義箇所: contrib/jsonb_plpython/jsonb_plpython.c:136

目的

PostgreSQL の JsonbContainer（JSONB の配列またはオブジェクト）を、対応する Python のリストまたは辞書オブジェクトへ再帰的に変換する。

シグネチャまたは形状

static PyObject *PLyObject_FromJsonbContainer(JsonbContainer *jsonb)

入力と出力

  • **入力**: JsonbContainer *jsonb - 変換対象の JSONB コンテナ構造体。
  • **出力**: PyObject * - 変換後の Python オブジェクト（list または dict）。失敗時は NULL を返すか、PostgreSQL のエラーとして elog(ERROR) を発行する。

前提条件と不変条件

  • plpython 拡張がロードされ、必要な関数ポインタが初期化されていること。
  • 変換中に発生する Python のメモリ管理（Py_DECREF）と PostgreSQL のエラーハンドリング（PG_TRY/PG_CATCH）の整合性が保たれていること。

アルゴリズムまたは振る舞い

1. JsonbIteratorInit を用いてイテレータを初期化し、最初のトークンを取得する。 2. トークンの種別に応じて分岐する:

  • WJB_BEGIN_ARRAY:
  • rawScalar の場合は単一要素として処理する。
  • 通常の配列の場合は PyList_New でリストを作成し、WJB_ELEM トークンを走査して PLyObject_FromJsonbValue で各要素を変換し、リストに追加する。
  • WJB_BEGIN_OBJECT:
  • PyDict_New で辞書を作成し、WJB_KEY と WJB_VALUE のペアを交互に取得する。
  • PLyUnicode_FromJsonbValue でキーを、PLyObject_FromJsonbValue で値を変換し、PyDict_SetItem で辞書に格納する。

    • 3. 変換処理は PG_TRY ブロックで囲まれ、エラー発生時には PG_CATCH ブロックにて確保済みの Python オブジェクトの参照カウントを適切に解放（Py_XDECREF）する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/jsonb_plpython/jsonb_plpython.c 内の複数箇所（jsonb_to_plpython 等の変換ロジックから参照されている）。
  • **呼び出し先**:
  • JsonbIteratorInit, JsonbIteratorNext (PostgreSQL JSONB API)
  • PLyObject_FromJsonbValue, PLyUnicode_FromJsonbValue (再帰的な変換用)
  • PyList_New, PyDict_New, PyList_Append, PyDict_SetItem, Py_XDECREF (Python C API)

関連データ構造

  • JsonbContainer, JsonbIterator, JsonbValue: PostgreSQL 側の JSONB 表現。
  • PyObject: Python 側のオブジェクト表現。

失敗モードと境界条件

  • PyList_New や PyDict_New がメモリ不足等で NULL を返した場合、関数は直ちに NULL を返す。
  • 予期しない JSONB トークンが検出された場合、elog(ERROR) により PostgreSQL のトランザクションが中断される。
  • 変換中に例外が発生した場合、PG_CATCH ブロックによりメモリリークを防ぎつつ PG_RE_THROW でエラーを伝播させる。

保守担当者向けメモ

  • 本関数は volatile 修飾子を付けた Python オブジェクトポインタを使用しており、PG_TRY ブロック内でのスタック巻き戻しによるメモリリークを防止している。修正を加える際は、Py_XDECREF の呼び出し漏れがないか注意すること。
