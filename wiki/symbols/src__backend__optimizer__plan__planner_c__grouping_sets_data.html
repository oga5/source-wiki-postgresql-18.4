grouping_sets_data

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/planner.c:99

目的

GROUP BY 句における GROUPING SETS、ROLLUP、CUBE などの複雑な集約処理に必要なメタデータを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    List       *rollups;
    List       *hash_sets_idx;
    double      dNumHashGroups;
    bool        any_hashable;
    Bitmapset  *unsortable_refs;
    Bitmapset  *unhashable_refs;
    List       *unsortable_sets;
    int        *tleref_to_colnum_map;
} grouping_sets_data;

入力と出力

本構造体はデータ保持用であり、関数としての入出力は持ちません。主に preprocess_grouping_sets 関数によって生成され、プランニングの各フェーズで参照されます。

前提条件と不変条件

  • any_hashable が false の場合、ハッシュベースの集約は選択されません。
  • unsortable_refs および unhashable_refs は、それぞれソートまたはハッシュが不可能な参照を特定するためのビットマップセットとして機能します。

アルゴリズムまたは振る舞い

集約処理の最適化において、どのグループ化セットをハッシュ化し、どのセットをソートで処理するかを決定するための情報を管理します。tleref_to_colnum_map は、ターゲットリストの参照と列番号の対応付けに使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: preprocess_grouping_sets 関数でインスタンス化され、grouping_planner 内で集約プランの構築時に参照されます。
  • **呼び出し先**: 本構造体自体はデータコンテナであり、直接関数を呼び出すことはありません。

関連データ構造

  • PlannerInfo: プランニングの全体的な状態を保持する構造体。
  • List, Bitmapset: PostgreSQL の標準的なデータ構造。

失敗モードと境界条件

  • grouping_sets_data が NULL の場合、GROUPING SETS 構文がクエリに含まれていないことを示します（planner.c:1785 等の初期化ロジックを参照）。

保守担当者向けメモ

  • 本構造体は grouping_planner における集約処理の複雑さを管理する中心的な役割を担います。
  • 構造体のフィールド追加や変更を行う際は、preprocess_grouping_sets での初期化ロジックと、grouping_planner 内での利用箇所（ソート/ハッシュの判定ロジック）の両方に影響が及ぶ点に注意してください。
