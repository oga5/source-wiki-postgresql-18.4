gin_cmp_prefix

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsginidx.c:39

目的

GINインデックスにおける前方一致検索（prefix search）のために、2つの文字列（text型）を比較します。

シグネチャまたは形状

Datum gin_cmp_prefix(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_PP(0): 比較対象の文字列 a
  • PG_GETARG_TEXT_PP(1): 比較対象の文字列 b
  • **出力**:
  • int32: 比較結果。a が b で始まる場合は 0、それ以外の場合は tsCompareString の結果に基づいた値。

前提条件と不変条件

  • 入力引数は text 型のポインタとして扱われます。
  • PG_FREE_IF_COPY マクロにより、必要に応じてメモリの解放が行われます。

アルゴリズムまたは振る舞い

1. tsCompareString を呼び出し、a と b を比較します（前方一致判定のため、第5引数に true を指定）。 2. 比較結果 cmp が 0 未満（a < b）の場合、cmp を 1 に設定します。これは、GINインデックスのスキャンにおいて、前方一致しない場合にスキャンを継続させないための制御です。 3. 最終的な cmp 値を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に直接的な呼び出し元の記述はありません。
  • **呼び出し先**: tsCompareString を呼び出します。

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約で使用される引数構造体。

失敗モードと境界条件

  • tsCompareString の仕様に依存します。
  • cmp < 0 の場合に強制的に 1 を返すことで、前方一致検索の範囲外となった際のインデックススキャンを停止させる設計となっています。

保守担当者向けメモ

  • 本関数は tsvector の前方一致検索用です。
  • NOT_USED マクロで囲まれた引数（StrategyNumber 等）は、現在の実装では使用されていません。将来的な拡張やAPI互換性のために残されています。
