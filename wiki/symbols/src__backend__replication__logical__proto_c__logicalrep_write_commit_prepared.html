logicalrep_write_commit_prepared

定義

  • src/backend/replication/logical/proto.c:236

目的

論理レプリケーションにおける2フェーズコミット（2PC）の完了メッセージをシリアライズし、ネットワーク転送用のバッファへ書き込む。

シグネチャまたは形状

void logicalrep_write_commit_prepared(StringInfo out, ReorderBufferTXN *txn, XLogRecPtr commit_lsn)

入力と出力

  • out (入出力): メッセージを書き込む StringInfo バッファ。
  • txn (入力): コミット対象のトランザクション情報（ReorderBufferTXN）。GIDおよびトランザクションメタデータを含む。
  • commit_lsn (入力): コミット時のログシーケンス番号（LSN）。

前提条件と不変条件

  • txn->gid が NULL であってはならない（2フェーズコミットトランザクションであることが必須）。

アルゴリズムまたは振る舞い

1. メッセージタイプ識別子 LOGICAL_REP_MSG_COMMIT_PREPARED を書き込む。 2. フラグフィールド（現在は 0 固定）を書き込む。 3. 以下の順序でデータをバッファへ書き込む：

  • commit_lsn (int64)
  • txn->end_lsn (int64)
  • txn->xact_time.commit_time (int64)
  • txn->xid (int32)
  • txn->gid (文字列)

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/pgoutput/pgoutput.c:699
  • **呼び出し先**: pq_sendbyte, pq_sendint64, pq_sendint32, pq_sendstring（いずれも libpq/pqformat.h 経由）

関連データ構造

  • StringInfo: 通信バッファ。
  • ReorderBufferTXN: トランザクションの状態とメタデータを保持する構造体。

失敗モードと境界条件

  • txn->gid が NULL の場合、Assert により異常終了する。
  • バッファのメモリ不足時は pq_send* 関数群の内部挙動に従いエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はプロトコル定義の一部である。メッセージ形式を変更する場合は、対応するデシリアライズ関数（logicalrep_read_commit_prepared）および logicalproto.h の定義と同期させること。
