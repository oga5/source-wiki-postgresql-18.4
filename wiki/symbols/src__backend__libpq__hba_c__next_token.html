next_token

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/hba.c:186

目的

設定ファイル（pg_hba.conf 等）の文字列から、次のトークンを抽出してバッファに格納します。引用符やカンマ、コメントを適切に処理し、構文解析を支援します。

シグネチャまたは形状

static bool next_token(char **lineptr, StringInfo buf, bool *initial_quote, bool *terminating_comma)

入力と出力

  • **入力**:
  • lineptr: 解析対象文字列へのポインタ。解析が進むと更新されます。
  • buf: 抽出したトークンを格納する StringInfo。
  • **出力**:
  • buf: 抽出されたトークン文字列。
  • initial_quote: トークンが引用符で始まっていた場合に true。
  • terminating_comma: トークンの直後にカンマが存在した場合に true。
  • **戻り値**: トークンが抽出された場合（saw_quote または buf->len > 0）に true を返します。

前提条件と不変条件

  • lineptr は有効な文字列を指している必要があります。
  • buf は resetStringInfo により初期化されます。
  • 引用符内の二重引用符（""）はリテラルとして扱われます。

アルゴリズムまたは振る舞い

1. **初期化**: buf をリセットし、出力フラグを false に設定します。 2. **スキップ**: 空白文字およびカンマを読み飛ばします。 3. **トークン構築**: 以下の条件まで文字を buf に追加します。

  • 空白文字またはカンマ（引用符外の場合）。
  • 行末（\0）。
  • #（コメント開始、引用符外の場合）。

    • 4. **引用符処理**: 引用符で囲まれた範囲内では、空白やカンマをトークンの一部として保持します。 5. **後処理**: トークン終了直後の文字を lineptr から1つ戻し、次回の読み込み位置を調整します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/hba.c 内の解析処理（tokenize_auth_file 等）から呼び出されることが示唆されていますが、具体的な関数名はソース抜粋からは特定できません。
  • **呼び出し先**: resetStringInfo, pg_isblank, appendStringInfoChar を呼び出します。
  • 注意: src/backend/parser/parser.c に存在する next_token という名前のシンボルは、本関数とは別物であり、混同しないよう注意してください。

関連データ構造

  • StringInfo: 文字列バッファを管理する構造体。

失敗モードと境界条件

  • 文字列の終端（\0）に達した場合、適切に処理を終了します。
  • トークンが空（buf->len == 0 かつ引用符なし）の場合、false を返します。

保守担当者向けメモ

  • 本関数は pg_hba.conf の字句解析における基盤です。引用符のネストやエスケープ処理に変更を加える際は、in_quote および was_quote の状態遷移を慎重に確認してください。
