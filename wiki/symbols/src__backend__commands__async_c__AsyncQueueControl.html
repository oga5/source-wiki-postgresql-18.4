AsyncQueueControl

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/async.c:281

目的

PostgreSQLの非同期通知機能（NOTIFY等）において、共有メモリ上で通知キューの状態とリスナーの管理を行うための制御構造体です。

シグネチャまたは形状

typedef struct AsyncQueueControl
{
	QueuePosition head;
	QueuePosition tail;
	int64		stopPage;
	ProcNumber	firstListener;
	TimestampTz lastQueueFillWarn;
	QueueBackendStatus backend[FLEXIBLE_ARRAY_MEMBER];
} AsyncQueueControl;

入力と出力

  • **入力**: 共有メモリ領域として確保され、バックエンドプロセスが LISTEN を実行した際や通知を送信する際に更新されます。
  • **出力**: キューの読み取り位置（tail）や書き込み位置（head）、および各バックエンドのリスナー状態を保持します。

前提条件と不変条件

  • tail は、すべてのリスニング中のバックエンドが読み取ったキュー位置以下である必要があります。
  • stopPage は、最も古い再利用されていないページを指し、tail.page 以下である必要があります。
  • NotifyQueueLock によって排他制御が行われます。

アルゴリズムまたは振る舞い

  • 通知送信時、head を進めて新しい通知をキューに追加します。
  • LISTEN 実行時、自身のPIDを backend 配列に登録します。
  • 通知の到着時、firstListener から始まるリストを辿り、該当するバックエンドへ PROCSIG_NOTIFY_INTERRUPT を送信します。
  • asyncQueueAdvanceTail により、各バックエンドの読み取り状況に応じて tail を進め、不要なSLRUページを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: AsyncShmemInit（初期化時）、および通知の送信・受信・管理を行う各関数（Async_Notify 等）。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接関数を呼び出すことはありませんが、NotifyQueueLock を介して排他制御が行われます。

関連データ構造

  • QueuePosition: キュー内の位置を特定する構造体。
  • QueueBackendStatus: 各バックエンドのリスナー状態を保持する構造体。
  • AsyncQueueEntry: キューに格納される個々の通知メッセージ。

失敗モードと境界条件

  • キューが満杯の場合、通知の送信がブロックされるか、警告が記録されます（lastQueueFillWarn で制御）。
  • 読み取りが遅いバックエンドが存在すると、tail が進まずディスク領域（SLRU）を過剰に消費する可能性があります。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を含んでいるため、共有メモリ確保時には offsetof(AsyncQueueControl, backend) を使用してサイズを計算する必要があります。
  • ロック取得順序（NotifyQueueTailLock -> NotifyQueueLock）の厳守がデッドロック回避に不可欠です。
