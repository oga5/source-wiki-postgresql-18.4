hmac_init

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/px-hmac.c:51

目的

HMAC 計算のための内部状態（ipad および opad）を初期化し、ハッシュコンテキストを準備します。

シグネチャまたは形状

static void hmac_init(PX_HMAC *h, const uint8 *key, unsigned klen)

入力と出力

  • h: 初期化対象の PX_HMAC 構造体へのポインタ。
  • key: HMAC 計算に使用する秘密鍵。
  • klen: 鍵の長さ（バイト単位）。
  • 出力: なし（h の内部状態が更新されます）。

前提条件と不変条件

  • h->md が有効な PX_MD コンテキストを指している必要があります。
  • RFC 2104 に準拠した HMAC 初期化処理を行います。

アルゴリズムまたは振る舞い

1. ハッシュアルゴリズムのブロックサイズ（bs）を取得します。 2. 鍵の長さが bs を超える場合、鍵をハッシュ化して短縮します。それ以外の場合はそのまま使用します。 3. 鍵と HMAC_IPAD / HMAC_OPAD を XOR し、PX_HMAC 構造体内の ipad および opad バッファに格納します。 4. 一時的な鍵バッファを px_memset でゼロクリアし、メモリを解放します。 5. 内部のダイジェストコンテキストに対して ipad を適用（px_md_update）し、後続のデータ入力に備えます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/px-hmac.c 内で PX_HMAC 構造体の init 関数ポインタとして設定されます。
  • 呼び出し先: px_md_block_size, palloc0, px_md_update, px_md_finish, px_md_reset, memcpy, px_memset, pfree。

関連データ構造

  • PX_HMAC: HMAC の状態を保持する構造体。
  • PX_MD: 基盤となるメッセージダイジェストコンテキスト。

失敗モードと境界条件

  • palloc0 がメモリ不足時にエラーを発生させる可能性があります。
  • 鍵の長さがブロックサイズより大きい場合、内部ダイジェスト関数が正常に動作することが前提となります。

保守担当者向けメモ

  • 本関数は機密情報（鍵）を扱うため、処理終了後に keybuf を確実にゼロクリアしています。
  • px_find_hmac を通じて呼び出されることを想定した内部関数です。
