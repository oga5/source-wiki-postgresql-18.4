LockMethodData

定義

  • 種別: struct
  • 定義箇所: src/include/storage/lock.h:109

目的

ロックマネージャにおける、特定のロック手法（Lock Method）の競合ルールおよびメタデータを定義する構造体です。

シグネチャまたは形状

typedef struct LockMethodData
{
	int			numLockModes;
	const LOCKMASK *conflictTab;
	const char *const *lockModeNames;
	const bool *trace_flag;
} LockMethodData;

入力と出力

本構造体は関数ではなくデータ定義です。ロックマネージャの初期化時やロック競合判定時に、定義された競合テーブル（conflictTab）やモード名が参照されます。

前提条件と不変条件

  • numLockModes は、そのロック手法でサポートされるロックモードの総数を示します。
  • conflictTab は、各ロックモード間の競合関係を保持するビットマスク配列へのポインタである必要があります。

アルゴリズムまたは振る舞い

ロックマネージャは、この構造体を参照して LockCheckConflicts を実行します。具体的には、要求されたロックモードと既存のロック状態を conflictTab を用いて照合し、競合が発生するかどうかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/lmgr/lock.c 内で default_lockmethod および user_lockmethod として静的に定義・初期化されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • LOCKMASK: 競合判定に使用されるビットマスク型。
  • LockMethod: LockMethodData へのポインタ型（typedef const LockMethodData *LockMethod;）。

失敗モードと境界条件

  • numLockModes が実際の conflictTab のサイズと一致しない場合、メモリ破壊や不正な競合判定を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体はロックマネージャの挙動を決定する重要な設定データです。新しいロック手法を追加する際は、conflictTab の整合性に十分注意してください。
  • src/backend/storage/lmgr/lock.c にある既存のインスタンス（default_lockmethod 等）が参照の起点となります。
