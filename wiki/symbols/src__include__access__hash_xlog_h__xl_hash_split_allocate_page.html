xl_hash_split_allocate_page

定義

  • 種別: struct
  • 定義箇所: src/include/access/hash_xlog.h:98

目的

ハッシュインデックスのバケット分割時における、新しいページ割り当て操作をWAL（Write-Ahead Logging）に記録するためのデータ構造です。

シグネチャまたは形状

typedef struct xl_hash_split_allocate_page
{
	uint32		new_bucket;
	uint16		old_bucket_flag;
	uint16		new_bucket_flag;
	uint8		flags;
} xl_hash_split_allocate_page;

入力と出力

本構造体はWALレコードのペイロードとして書き込まれ、リカバリ時に読み取られます。

  • **入力**: 分割操作時に _hash_doinsert 等から設定されるバケット情報およびフラグ。
  • **出力**: リカバリ処理（hash_redo）において、インデックスページの再構築および状態復元に使用されます。

前提条件と不変条件

  • flags フィールドには8ビットが利用可能です。
  • 本構造体のサイズおよびオフセットは、offsetof(xl_hash_split_allocate_page, flags) + sizeof(uint8) により計算可能です。

アルゴリズムまたは振る舞い

ハッシュインデックスの分割処理において、新しいバケットの割り当てが発生した際にこの構造体がWALレコードに格納されます。リカバリ時には、この構造体をキャストして読み込み、new_bucket や各フラグ情報に基づいて、対象となるインデックスページの状態を整合性のある状態へ復元します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/hash/hashpage.c:902: 構造体のインスタンス化が行われています。
  • **呼び出し先（参照元）**:
  • src/backend/access/hash/hash_xlog.c:314: XLogRecGetData を通じてレコードから本構造体へキャストされます。
  • src/backend/access/rmgrdesc/hashdesc.c:59: ログ出力（デバッグ・解析用）のために参照されます。

関連データ構造

  • src/include/access/hash_xlog.h: 本構造体が定義されているヘッダーファイル。
  • src/include/access/hash.h: ハッシュインデックスのメタページ等の基本構造を定義。

失敗モードと境界条件

  • WALレコードの読み込み時にデータが破損している場合、キャスト後のフィールド値が不正になる可能性があります。
  • flags のビット定義が変更された場合、古いWALレコードとの互換性が失われる可能性があります。

保守担当者向けメモ

  • 本構造体は hash_redo におけるリカバリ処理の核となるため、フィールドの追加や変更を行う際は、hash_xlog.c 内のリカバリロジックおよび hashdesc.c のログ出力ロジックを必ず更新してください。
