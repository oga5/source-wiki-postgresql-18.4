DIR

定義

  • 種別: struct
  • 定義箇所: src/include/port/win32_msvc/dirent.h:18

目的

Windows環境において、POSIX準拠のディレクトリ走査を行うための状態保持用不透明構造体（opaque structure）です。

シグネチャまたは形状

typedef struct DIR DIR;

入力と出力

  • 本構造体自体は型定義であり、具体的なメンバは src/port/dirent.c で定義されます。
  • opendir 関数によって初期化され、readdir および closedir 関数に渡されます。

前提条件と不変条件

  • Windows環境（MSVC）でのみ使用されます。
  • opendir で正常に初期化された後にのみ、他のディレクトリ操作関数で使用可能です。

アルゴリズムまたは振る舞い

  • ディレクトリ走査のライフサイクルを管理します。
  • opendir 呼び出し時に、指定されたパスをWindowsの FindFirstFile 形式に変換して保持します。
  • readdir 呼び出し時に、内部で FindFirstFile または FindNextFile を実行し、結果を struct dirent にマッピングします。
  • closedir 呼び出し時に、保持しているWindowsハンドルを FindClose で解放し、メモリをクリーンアップします。

呼び出し元と呼び出し先

  • 呼び出し元（例）:
  • src/backend/access/heap/rewriteheap.c
  • src/backend/access/transam/slru.c
  • src/backend/access/transam/twophase.c
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/backup/basebackup.c
  • 呼び出し先:
  • 実装の詳細（src/port/dirent.c）において、Windows APIの FindFirstFile, FindNextFile, FindClose を呼び出します。

関連データ構造

  • struct dirent: ディレクトリ内の各エントリ情報を保持する構造体。

失敗モードと境界条件

  • Windowsの MAX_PATH 制限に依存します。
  • d_ino はサポートされていないため、常に0が返されます。
  • d_type の判定にはWindows固有の属性（FILE_ATTRIBUTE_REPARSE_POINT 等）が使用されます。

保守担当者向けメモ

  • 本構造体は src/port/dirent.c で定義される不透明な型です。構造体の内部メンバを変更する場合は、必ず src/port/dirent.c の実装と整合性を取ってください。
  • POSIX標準の完全な模倣ではなく、PostgreSQLのビルドに必要な最小限の機能セットであることを留意してください。
