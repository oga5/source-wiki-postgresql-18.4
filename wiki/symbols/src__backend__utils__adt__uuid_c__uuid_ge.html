uuid_ge

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/uuid.c:236

目的

2つの uuid 型の値が「以上（greater than or equal to）」の関係にあるかを判定します。

シグネチャまたは形状

Datum uuid_ge(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの pg_uuid_t 型へのポインタを受け取ります。
  • **出力**: 第1引数が第2引数以上であれば true、そうでなければ false を返します（PG_RETURN_BOOL）。

前提条件と不変条件

  • 入力される pg_uuid_t ポインタは有効である必要があります。
  • 内部比較には uuid_internal_cmp が使用され、16バイトのバイナリ比較が行われます。

アルゴリズムまたは振る舞い

1. PG_GETARG_UUID_P マクロを使用して、引数から2つの pg_uuid_t 構造体を取得します。 2. uuid_internal_cmp を呼び出し、2つの UUID を比較します。 3. 比較結果が 0 以上（>= 0）であるかを判定し、その結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し先**: uuid_internal_cmp
  • **呼び出し元**: ソースコード内に直接的な呼び出し元の記述はありません。SQL インターフェースを介して呼び出されることが想定されます。

関連データ構造

  • pg_uuid_t: UUID を保持する 16 バイトの構造体。

失敗モードと境界条件

  • 引数が NULL の場合、PG_GETARG_UUID_P の挙動に従い、呼び出し元で適切に処理される必要があります。
  • 比較はバイナリレベルで行われるため、UUID の各バイトの符号なし比較結果に依存します。

保守担当者向けメモ

  • 本関数は uuid_internal_cmp に依存しています。比較ロジックを変更する場合は、uuid_internal_cmp の実装を確認してください。
