addToArray

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/tzparser.c:188

目的

タイムゾーン定義エントリをソート済み配列に挿入、または既存エントリの更新を行う。

シグネチャまたは形状

static int addToArray(tzEntry **base, int *arraysize, int n, tzEntry *entry, bool override)

入力と出力

  • **入力**:
  • base: tzEntry 配列へのポインタ。配列の再確保時に更新される。
  • arraysize: 現在の配列容量。必要に応じて倍増される。
  • n: 現在の有効なエントリ数。
  • entry: 挿入または比較対象の tzEntry。
  • override: 重複時に上書きを許可するフラグ。
  • **出力**:
  • 成功時: 更新後のエントリ数（n または n+1）。
  • 失敗時: -1（重複かつ override が偽の場合）。

前提条件と不変条件

  • 配列は abbrev フィールドの strcmp 順で常にソートされている必要がある。
  • datetime.c 側の検索ロジックとソート順が一致していること。

アルゴリズムまたは振る舞い

1. 二分探索により、entry->abbrev に一致する既存エントリを検索する。 2. **重複発見時**:

  • 内容が同一であれば変更せず n を返す。
  • override が真であれば、既存エントリを新しい値で更新して n を返す。
  • override が偽であれば、GUC_check_errmsg を通じてエラーを報告し -1 を返す。

    • 3. **新規挿入時**:

  • 配列容量が不足している場合、repalloc で容量を倍増させる。
  • memmove を使用して挿入位置以降の要素をシフトし、memcpy で新しいエントリを格納する。
  • n + 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/tzparser.c:431（ParseTzFile 内のループ処理）。
  • **呼び出し先**: repalloc（メモリ管理）、strcmp、memmove、memcpy、GUC_check_errmsg、GUC_check_errdetail。

関連データ構造

  • tzEntry: タイムゾーンの略称、オフセット、DST フラグ、ファイル名、行番号を保持する構造体。

失敗モードと境界条件

  • **重複エラー**: 略称が重複し、かつ override が指定されていない場合、エラーメッセージを生成して -1 を返す。
  • **メモリ不足**: repalloc が失敗した場合、PostgreSQL の標準的なメモリ管理ルールに従いエラー（elog(ERROR) 等）が発生する可能性がある。

保守担当者向けメモ

  • 本関数は GUC フックから呼ばれるため、直接的な elog(ERROR) を避け、GUC_check_errmsg を使用してエラーを蓄積する設計となっている。
  • strcmp によるソート順は datetime.c の検索ロジックと密結合しているため、変更時は注意すること。
