xl_brin_createidx

定義

  • 種別: struct
  • 定義箇所: src/include/access/brin_xlog.h:50

目的

BRIN（Block Range Index）インデックスの作成操作をWAL（Write-Ahead Logging）に記録するための構造体です。リカバリ時にインデックスの初期化状態を再構築するために使用されます。

シグネチャまたは形状

typedef struct xl_brin_createidx
{
    BlockNumber pagesPerRange;
    uint16      version;
} xl_brin_createidx;

入力と出力

  • **入力**: インデックス作成時に設定される pagesPerRange（範囲あたりのブロック数）および version（インデックスのバージョン番号）。
  • **出力**: WALレコードの一部として書き込まれるバイナリデータ。

前提条件と不変条件

  • SizeOfBrinCreateIdx マクロ（offsetof(xl_brin_createidx, version) + sizeof(uint16)）を使用して、WALレコードのサイズを正確に計算する必要があります。

アルゴリズムまたは振る舞い

1. **記録**: インデックス作成時、本構造体に値を格納し、WALレコードとして書き込みます。 2. **再生**: リカバリプロセスにおいて、WALレコードから本構造体を読み出し、対象のBRINインデックスのメタデータとして pagesPerRange と version を適用します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c（インデックス作成時のWAL記録）
  • **呼び出し先**:
  • src/backend/access/brin/brin_xlog.c（Redo処理でのデータ読み取り）
  • src/backend/access/rmgrdesc/brindesc.c（WALログのデコード・表示）

関連データ構造

  • BlockNumber: PostgreSQLのブロック番号型。
  • XLogRecGetData: WALレコードから本構造体を取り出すために使用される関数。

失敗モードと境界条件

  • WALレコードの読み取り時に、データサイズが SizeOfBrinCreateIdx と一致しない場合、リカバリ処理が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体にフィールドを追加する場合は、必ず SizeOfBrinCreateIdx マクロを更新してください。
  • version フィールドは、インデックスの互換性維持のために重要です。変更時には既存のRedoロジックへの影響を考慮してください。
