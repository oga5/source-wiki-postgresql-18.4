ExecuteGrantStmt

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/aclchk.c:391

目的

SQLの GRANT および REVOKE 文を実行し、対象オブジェクトのアクセス制御リスト（ACL）を更新するためのメインエントリーポイントです。

シグネチャまたは形状

void ExecuteGrantStmt(GrantStmt *stmt)

入力と出力

  • **入力**: GrantStmt *stmt（解析済みの GRANT / REVOKE 文の構文木）
  • **出力**: なし（副作用としてシステムカタログを更新）

前提条件と不変条件

  • stmt->grantor が指定されている場合、現在のユーザーと一致しなければならない（SQL互換性のための制約）。
  • stmt->targtype は ACL_TARGET_OBJECT または ACL_TARGET_ALL_IN_SCHEMA である必要がある。
  • stmt->objtype はサポートされているオブジェクト型（テーブル、関数、スキーマ等）である必要がある。

アルゴリズムまたは振る舞い

1. **権限者検証**: grantor が指定されている場合、現在のユーザーと一致するか確認し、不一致ならエラーを投げる。 2. **内部形式への変換**: GrantStmt を内部構造体 InternalGrant に変換する。 3. **オブジェクト特定**: targtype に応じて objectNamesToOids または objectsInSchemaToOids を呼び出し、対象オブジェクトの OID を取得する。 4. **被付与者の解決**: grantees リストを走査し、PUBLIC を ACL_ID_PUBLIC に変換するなどして OID リストを作成する。 5. **権限ビットマスクの生成**: stmt->privileges をオブジェクト型に応じた AclMode ビットマスクに変換する。 6. **処理の継続**: 変換された InternalGrant 構造体を用いて、後続の権限更新処理（ソース抜粋範囲外）へ引き継ぐ。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c（ProcessUtility 等の解析ツリー処理経由）
  • **呼び出し先**:
  • get_rolespec_oid
  • objectNamesToOids / objectsInSchemaToOids
  • lappend_oid
  • gettext_noop
  • elog / ereport
  • (ソース抜粋範囲外の内部処理関数へ続く)

関連データ構造

  • GrantStmt: SQL解析結果を保持する構造体。
  • InternalGrant: ExecuteGrantStmt 内で構築される、権限操作のための内部用構造体。
  • AclMode: 権限ビットマスク。
  • RoleSpec: ロール指定。

失敗モードと境界条件

  • **エラー**:
  • grantor が現在のユーザーと異なる場合。
  • 未知の targtype または objtype が指定された場合。
  • get_rolespec_oid が失敗した場合。
  • **境界条件**:
  • privileges が NIL の場合、all_privs フラグを立てて全権限を対象とする。

保守担当者向けメモ

  • 本関数は GRANT / REVOKE の前処理を担当する。実際のカタログ更新ロジックは、この後に呼び出される内部ルーチン（ExecGrant_Relation 等）に依存する。
  • 列レベルの権限指定がある場合、col_privs に格納されるが、本抜粋範囲ではその後の処理は記述されていない。
