ExplainSerializeOption

定義

  • 種別: enum
  • 定義箇所: src/include/commands/explain_state.h:20

目的

EXPLAIN コマンド実行時におけるクエリ出力のシリアライズ形式を指定するために使用されます。

シグネチャまたは形状

typedef enum ExplainSerializeOption
{
    EXPLAIN_SERIALIZE_NONE,    /* シリアライズなし */
    EXPLAIN_SERIALIZE_TEXT,    /* テキスト形式でのシリアライズ */
    EXPLAIN_SERIALIZE_BINARY,  /* バイナリ形式でのシリアライズ */
} ExplainSerializeOption;

入力と出力

  • 入力: EXPLAIN コマンドのオプション設定プロセスにおいて、この列挙型の値が選択されます。
  • 出力: ExplainState 構造体内の serialize フィールドに格納され、後続の実行エンジンによる出力処理の挙動を制御します。

前提条件と不変条件

  • ExplainState 構造体のメンバーとして保持されます。
  • 値は EXPLAIN_SERIALIZE_NONE, EXPLAIN_SERIALIZE_TEXT, EXPLAIN_SERIALIZE_BINARY のいずれか一つに限定されます。

アルゴリズムまたは振る舞い

  • EXPLAIN_SERIALIZE_NONE: デフォルト値。クエリ結果のシリアライズを行いません。
  • EXPLAIN_SERIALIZE_TEXT / EXPLAIN_SERIALIZE_BINARY: 指定された形式に基づき、クエリの出力データをシリアライズする処理をトリガーします。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/commands/explain_state.h 内の ExplainState 構造体定義でフィールドとして参照されています。
  • 呼び出し先: 提供されたソース情報には、本シンボルを直接操作する具体的な関数呼び出しの記述はありません。

関連データ構造

  • ExplainState: 本列挙型を保持するメインの管理構造体。

失敗モードと境界条件

  • 未定義の列挙値を設定した場合の挙動については、本ソース excerpt からは判断できません。

保守担当者向けメモ

  • 本シンボルは EXPLAIN の出力形式を拡張する際に重要となります。新しいシリアライズ形式を追加する場合は、この列挙型を更新し、対応する処理を src/backend/commands/explain_state.c 等に実装する必要があります。
