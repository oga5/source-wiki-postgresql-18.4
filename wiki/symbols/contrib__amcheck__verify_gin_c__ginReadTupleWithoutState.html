ginReadTupleWithoutState

定義

  • 種別: function
  • 定義箇所: contrib/amcheck/verify_gin.c:98

目的

GINインデックスのタプルから、圧縮または非圧縮の形式に応じてアイテムポインタ（ItemPointer）の配列を抽出・復元します。

シグネチャまたは形状

ItemPointer ginReadTupleWithoutState(IndexTuple itup, int *nitems)

入力と出力

  • **入力**:
  • itup: 読み取り対象の IndexTuple。
  • nitems: 抽出されたアイテムポインタの個数を格納するための出力用ポインタ。
  • **出力**:
  • 復元された ItemPointer 配列の先頭アドレス。

前提条件と不変条件

  • itup は有効なGINエントリタプルである必要があります。
  • 圧縮されている場合、タプルヘッダー内のアイテム数（nipd）とデコード後のアイテム数（ndecoded）が一致しなければなりません。

アルゴリズムまたは振る舞い

1. タプルからポスティングリストのポインタとアイテム数を取得します。 2. GinItupIsCompressed を使用して圧縮状態を判定します。

  • **圧縮されている場合**: ginPostingListDecode を呼び出し、圧縮されたデータを展開します。展開後の個数がタプルヘッダーの記録と一致するか検証し、不一致の場合はエラーを発生させます。
  • **圧縮されていない場合**: palloc でメモリを確保し、memcpy を使用してデータをコピーします。

    • 3. 抽出したアイテム数を nitems に書き込み、配列のポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_gin.c:628 にて呼び出されています。
  • **呼び出し先**:
  • ginPostingListDecode（圧縮時）
  • palloc（メモリ確保）
  • memcpy（非圧縮時）
  • elog（エラー発生時）

関連データ構造

  • IndexTuple: GINインデックスのタプル形式。
  • ItemPointer: 抽出されるアイテムポインタの型。
  • GinPostingList: 圧縮されたポスティングリストの構造体。

失敗モードと境界条件

  • **不整合エラー**: 圧縮されたタプルにおいて、ヘッダーに記録されたアイテム数とデコード後のアイテム数が一致しない場合、ERROR レベルでログ出力し処理を中断します。
  • **メモリ不足**: palloc が失敗した場合は、PostgreSQLの標準的なメモリ管理機構によりエラーが発生します。

保守担当者向けメモ

  • 本関数は amcheck による検証プロセスの一部として使用されます。
  • 圧縮形式の変更や新しいGINタプルフォーマットの導入時には、ginPostingListDecode との整合性を確認してください。
