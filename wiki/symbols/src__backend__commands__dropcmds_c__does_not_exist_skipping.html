does_not_exist_skipping

定義

  • 種別: function
  • 定義箇所: src/backend/commands/dropcmds.c:242

目的

SQLの DROP 文において IF EXISTS が指定されている際、対象オブジェクトが存在しない場合に発行する適切な警告メッセージを構築し、処理をスキップするためのヘルパー関数です。

シグネチャまたは形状

static void does_not_exist_skipping(ObjectType objtype, Node *object)

入力と出力

  • **入力**:
  • objtype: 削除対象のオブジェクト型（ObjectType 列挙型）。
  • object: 削除対象を特定するためのノード情報（TypeName, List, ObjectWithArgs など）。
  • **出力**: なし（副作用として ereport による NOTICE メッセージの出力を行う）。

前提条件と不変条件

  • 本関数は、呼び出し元で対象オブジェクトが存在しないことが確認された後に呼び出されることを前提としています。
  • objtype に応じて、object ノードを適切な型へキャストする必要があります。

アルゴリズムまたは振る舞い

1. objtype に基づき switch 文で分岐します。 2. オブジェクトの種類に応じて、以下のいずれかの補助関数を呼び出し、スキーマや型、関連リレーションの不在を確認します。

  • schema_does_not_exist_skipping
  • type_in_list_does_not_exist_skipping
  • owningrel_does_not_exist_skipping

    • 3. 補助関数が不在を特定した場合は、その関数内でメッセージが構築されます。 4. 補助関数が不在を特定しなかった（または補助関数が存在しない）場合、本関数内でオブジェクト型に応じたデフォルトの警告メッセージ（例: type "..." does not exist, skipping）を定義し、ereport(NOTICE, ...) を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: RemoveObjects（src/backend/commands/dropcmds.c 内）。
  • **呼び出し先**: schema_does_not_exist_skipping, type_in_list_does_not_exist_skipping, owningrel_does_not_exist_skipping などの各ヘルパー関数。

関連データ構造

  • ObjectType: 削除対象の型を識別する列挙型。
  • Node: 汎用的な構文解析ノード。
  • TypeName, ObjectWithArgs: オブジェクトを特定するための具体的なノード構造体。

失敗モードと境界条件

  • 未サポートの ObjectType が渡された場合、メッセージが構築されず、警告も出力されない可能性があります。
  • object ノードの型が objtype と一致しない場合、castNode マクロによりエラーが発生する可能性があります。

保守担当者向けメモ

  • 新しいオブジェクト型を DROP 文に追加する際は、本関数の switch 文に該当するケースを追加し、必要に応じて専用の不在判定ヘルパーを実装してください。
  • IF EXISTS メッセージの統一感を保つため、既存のメッセージフォーマット（gettext_noop を使用）に従うことが推奨されます。
