ResolveOpClass

定義

  • 種別: function
  • 定義箇所: src/backend/commands/indexcmds.c:2261

目的

指定されたインデックス演算子クラス（OpClass）を解決し、その OID を返します。演算子クラスが指定されていない場合は、データ型とアクセスメソッドに基づいたデフォルトの演算子クラスを特定します。

シグネチャまたは形状

Oid ResolveOpClass(const List *opclass, Oid attrType, const char *accessMethodName, Oid accessMethodId)

入力と出力

  • **入力**:
  • opclass: 演算子クラス名のリスト（NIL の場合はデフォルトを検索）。
  • attrType: インデックス対象のデータ型 OID。
  • accessMethodName: エラーメッセージ表示用のアクセスメソッド名。
  • accessMethodId: 検索対象のアクセスメソッド OID。
  • **出力**: 解決された演算子クラスの OID を返します。

前提条件と不変条件

  • 演算子クラスが見つからない場合、または指定されたデータ型と互換性がない場合はエラーを発生させます。
  • 演算子クラスの入力型（opcintype）と attrType は、バイナリ互換性（IsBinaryCoercible）を満たす必要があります。

アルゴリズムまたは振る舞い

1. **デフォルト検索**: opclass が NIL の場合、GetDefaultOpClass を呼び出し、見つからなければエラーを報告します。 2. **名前解決**:

  • スキーマ修飾名がある場合、指定されたスキーマ内で CLAAMNAMENSP キャッシュを検索します。
  • 修飾名がない場合、検索パスを使用して OpclassnameGetOpcid で OID を取得し、CLAOID キャッシュを検索します。

    • 3. **検証**: 取得した演算子クラスの opcintype が attrType とバイナリ互換であることを確認します。 4. **終了**: キャッシュを解放し、OID を返します。

関連データ構造

  • pg_opclass システムカタログ（Form_pg_opclass 構造体としてアクセス）。

失敗モードと境界条件

  • 指定された演算子クラスが存在しない場合、ERRCODE_UNDEFINED_OBJECT でエラーとなります。
  • データ型と演算子クラスの型に互換性がない場合、ERRCODE_DATATYPE_MISMATCH でエラーとなります。
  • デフォルトの演算子クラスが存在しない場合、エラーとなります。
