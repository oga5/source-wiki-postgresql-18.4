combo_encrypt

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/px.c:218

目的

PX_Combo オブジェクトを使用して、指定されたデータに対して暗号化処理を実行します。

シグネチャまたは形状

static int combo_encrypt(PX_Combo *cx, const uint8 *data, unsigned dlen, uint8 *res, unsigned *rlen);

入力と出力

  • cx: 暗号化コンテキストを保持する PX_Combo 構造体へのポインタ。
  • data: 暗号化対象の入力データバッファ。
  • dlen: 入力データの長さ。
  • res: 暗号化結果を格納する出力バッファ。
  • rlen: 出力データの実際の長さを格納するポインタ。
  • 戻り値: 成功時に 0 を返し、失敗時にはエラーコードを返します。

前提条件と不変条件

  • cx は combo_init によって適切に初期化されている必要があります。
  • res バッファは、暗号化およびパディング後のデータを受け取るのに十分なサイズである必要があります。

アルゴリズムまたは振る舞い

内部で保持している cx->cipher（暗号アルゴリズム）と cx->padding（パディング方式）を使用して、px_cipher_encrypt を呼び出し、暗号化処理を委譲します。

呼び出し元と呼び出し先

  • 呼び出し先: px_cipher_encrypt
  • 呼び出し元: contrib/pgcrypto/px.c 内の PX_Combo 構造体の関数ポインタ encrypt に割り当てられています。その他の直接的な呼び出し元については、提供されたソース範囲内では特定できません。

関連データ構造

  • PX_Combo: 暗号化アルゴリズムとパディング設定を保持する構造体。

失敗モードと境界条件

  • px_cipher_encrypt の戻り値に依存します。エラーが発生した場合、そのエラーコードがそのまま返されます。

保守担当者向けメモ

  • 本関数は PX_Combo インターフェースの一部です。新しい暗号化アルゴリズムやパディング方式を追加する際は、px_cipher_encrypt が正しく動作することを確認してください。
