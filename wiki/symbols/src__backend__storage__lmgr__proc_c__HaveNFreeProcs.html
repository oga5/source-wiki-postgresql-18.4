HaveNFreeProcs

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/proc.c:781

目的

共有メモリ上のフリープロセスリスト（ProcGlobal->freeProcs）に、少なくとも指定された数（n）の空き PGPROC 構造体が存在するかを確認する。

シグネチャまたは形状

bool HaveNFreeProcs(int n, int *nfree)

入力と出力

  • **入力**:
  • n: 確認したい空きプロセスの最小数（n > 0 である必要がある）。
  • nfree: 現在の空きプロセス数を格納するための出力用ポインタ。
  • **出力**:
  • 戻り値: n 個以上の空きプロセスが存在する場合は true、それ以外は false。
  • nfree: 実際にカウントされた空きプロセス数（最大で n までカウント）。

前提条件と不変条件

  • n > 0 であること。
  • nfree が有効なメモリ領域を指していること。
  • 内部で ProcStructLock スピンロックを取得するため、呼び出し元はスピンロックの再帰的取得によるデッドロックを考慮する必要がある。

アルゴリズムまたは振る舞い

1. ProcStructLock スピンロックを取得し、共有メモリ上のプロセス管理構造を保護する。 2. ProcGlobal->freeProcs リストを走査し、空きプロセスをカウントする。 3. カウントが n に達した時点で走査を打ち切る。 4. ProcStructLock を解放する。 5. カウント値が n と一致するかを判定し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/init/postinit.c:939（接続制限チェック時に使用）
  • **呼び出し先**:
  • SpinLockAcquire / SpinLockRelease（storage/spin.h）
  • dlist_foreach（リスト走査用マクロ）

関連データ構造

  • PGPROC: プロセス状態を管理する構造体。
  • ProcGlobal: 全プロセスの状態を保持する共有メモリ上のグローバルヘッダー。
  • ProcStructLock: フリーリストを保護するスピンロック。

失敗モードと境界条件

  • n に 0 以下の値を指定した場合、Assert により異常終了する。
  • nfree に NULL を渡した場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数は、新しいバックエンドプロセスを起動する前に、十分なリソースが確保可能かを判断するために使用される。
  • ProcStructLock を保持する時間はリストの走査数に依存するため、n が極端に大きい場合は注意が必要だが、通常はプロセス数の上限に依存する。
