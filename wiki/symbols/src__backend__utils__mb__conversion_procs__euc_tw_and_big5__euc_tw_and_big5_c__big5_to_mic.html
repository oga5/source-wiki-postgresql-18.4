big5_to_mic

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_tw_and_big5/euc_tw_and_big5.c:115

目的

PostgreSQLのSQLインターフェースを介して、BIG5 エンコーディングから MULE_INTERNAL (MIC) エンコーディングへ文字列を変換する。

シグネチャまたは形状

Datum big5_to_mic(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_CSTRING(2): 変換元文字列 (src)
  • PG_GETARG_CSTRING(3): 変換先バッファ (dest)
  • PG_GETARG_INT32(4): 入力文字列の長さ (len)
  • PG_GETARG_BOOL(5): エラー発生時の挙動フラグ (noError)
  • **出力**:
  • Datum (int32): 変換されたバイト数。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_BIG5, PG_MULE_INTERNAL) により、エンコーディングIDの妥当性が検証される。
  • noError が true の場合、変換エラーが発生しても例外を投げず、処理を中断して変換済みのバイト数を返す。

アルゴリズムまたは振る舞い

1. 引数から変換元バッファ、変換先バッファ、長さ、エラー制御フラグを取得する。 2. エンコーディング変換の引数チェックを実行する。 3. 内部関数 big52mic を呼び出し、実際のバイト変換ロジックを実行する。 4. 変換されたバイト数を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: big52mic (同ファイル内の内部関数)。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約に基づく引数コンテナ。

失敗モードと境界条件

  • 入力文字列が不正な BIG5 シーケンスを含む場合、noError フラグの設定に応じて、エラーを発生させるか、変換を中断して処理済みのバイト数を返す。

保守担当者向けメモ

  • 本関数は変換のインターフェース層であり、具体的な変換ロジックは big52mic に依存している。
  • noError フラグの扱いは COPY コマンド等の柔軟なデータロードをサポートするためのものである。
