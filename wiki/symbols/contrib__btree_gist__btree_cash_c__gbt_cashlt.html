gbt_cashlt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_cash.c:49

目的

money 型の値に対して「より小さい（less than）」という比較演算を提供し、GiST インデックス内での順序判定を可能にします。

シグネチャまたは形状

static bool gbt_cashlt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象となる Cash 型へのポインタ（void * として渡される）。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • bool: *a < *b が真であれば true、それ以外は false を返します。

前提条件と不変条件

  • a および b は有効な Cash 型の値を指している必要があります。
  • money 型は内部的に 64bit 整数として扱われます。

アルゴリズムまたは振る舞い

1. 引数 a と b を const Cash * 型にキャストします。 2. それぞれが指す値を直接比較演算子 < で比較します。 3. 比較結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_cash.c 内の tinfo 構造体（gbtree_ninfo）で比較関数として登録されており、gbt_num_consistent などの汎用 GiST 関数から間接的に呼び出されます。
  • **呼び出し先**: 直接的な関数呼び出しはありません（C言語の組み込み比較演算子を使用）。

関連データ構造

  • Cash: money 型の内部表現。
  • gbtree_ninfo: money 型の比較ロジックを GiST 汎用関数に提供するための構造体。

失敗モードと境界条件

  • ポインタが NULL の場合、セグメンテーション違反が発生する可能性があります。
  • money 型の範囲外の値や不正なメモリ領域を指している場合の動作は未定義です。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で定義された数値型 GiST インデックスのロジックと密接に連携しています。
  • 比較ロジックを変更する場合は、gbt_cashgt や gbt_casheq などの他の比較関数と整合性が取れていることを確認してください。
