gbt_bitgt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bit.c:25

目的

BIT 型および VARBIT 型のデータに対して「より大きい（greater than）」比較を行い、GiST インデックスの構築や検索をサポートする。

シグネチャまたは形状

static bool gbt_bitgt(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象のビット列データへのポインタ。
  • collation: 照合順序（本関数では未使用）。
  • flinfo: 関数呼び出し情報（本関数では未使用）。
  • **出力**:
  • bool: a > b であれば true、そうでなければ false を返す。

前提条件と不変条件

  • 入力ポインタ a および b は、PostgreSQL の BIT/VARBIT 型として有効なデータ構造を指している必要がある。
  • 内部的に DirectFunctionCall2 を使用するため、PostgreSQL の関数マネージャが初期化されている環境で実行される必要がある。

アルゴリズムまたは振る舞い

1. a と b を PointerGetDatum を介して Datum 型に変換する。 2. PostgreSQL 標準の bitgt 関数を DirectFunctionCall2 で呼び出し、ビット列としての大小比較を実行する。 3. 比較結果を DatumGetBool で論理値に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し先**: bitgt（PostgreSQL 標準のビット列比較関数）。
  • **呼び出し元**: contrib/btree_gist/btree_bit.c 内の tinfo 構造体（gbtree_vinfo）において、比較関数として参照されている。

関連データ構造

  • gbtree_vinfo: btree_gist フレームワークが型固有の操作を呼び出すために使用する設定テーブル。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、DirectFunctionCall2 の仕様に従い未定義の動作を引き起こす可能性がある。
  • bitgt 関数がエラーをスローした場合、本関数もそのエラーを伝播させる。

保守担当者向けメモ

  • 本関数は PostgreSQL 標準の bitgt をラップしたアダプターである。ロジックの変更が必要な場合は、標準の bitgt の仕様を確認すること。
  • collation および flinfo 引数は現在の実装では使用されていないが、btree_gist のインターフェース要件により保持されている。
