LockTableAclCheck

定義

  • 種別: function
  • 定義箇所: src/backend/commands/lockcmds.c:279

目的

指定されたリレーションに対して、要求されたロックモードに基づき、ユーザーが適切な権限を有しているかを検証する。

シグネチャまたは形状

AclResult LockTableAclCheck(Oid reloid, LOCKMODE lockmode, Oid userid)

入力と出力

  • **入力**:
  • reloid: 対象リレーションの OID。
  • lockmode: 取得しようとしているロックのモード（LOCKMODE 型）。
  • userid: 権限チェックを行う対象のユーザー OID。
  • **出力**:
  • AclResult: 権限チェックの結果（ACLCHECK_OK またはエラーコード）。

前提条件と不変条件

  • ACL_MAINTAIN, ACL_UPDATE, ACL_DELETE, ACL_TRUNCATE のいずれかの権限があれば、すべてのロックモードが許可される。
  • lockmode が AccessShareLock 以下の場合、ACL_SELECT 権限も許可条件に含まれる。
  • lockmode が RowExclusiveLock 以下の場合、ACL_INSERT 権限も許可条件に含まれる。

アルゴリズムまたは振る舞い

1. ロックモードに応じた必要な権限ビットマスク（aclmask）を構築する。 2. pg_class_aclcheck を呼び出し、指定されたユーザーが対象リレーションに対して aclmask で定義された権限を持っているかを確認する。 3. チェック結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/lockcmds.c 内の LOCK TABLE コマンド実行フローにおいて、リレーションのロック前検証として呼び出される。
  • **呼び出し先**:
  • pg_class_aclcheck（utils/acl.h で定義された関数）。

関連データ構造

  • LOCKMODE: ロックの強度を定義する列挙型。
  • AclResult: 権限チェックの結果を示す列挙型。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • pg_class_aclcheck が権限不足を検知した場合、呼び出し元で適切なエラー処理が行われることを前提としている。
  • 存在しない reloid を指定した場合の挙動は pg_class_aclcheck の仕様に依存する。

保守担当者向けメモ

  • 本関数は LOCK TABLE コマンドの権限検証ロジックを集中管理している。新しいロックモードや権限要件を追加する場合は、aclmask の構築ロジックを更新すること。
