StringInfoData

定義

  • 種別: struct
  • 定義箇所: src/include/lib/stringinfo.h:46

目的

PostgreSQL における可変長文字列バッファを管理するためのデータ構造です。動的なクエリ構築やバイナリデータのバッファリングを効率的に行うために使用されます。

シグネチャまたは形状

typedef struct StringInfoData
{
    char       *data;   /* 文字列バッファへのポインタ */
    int         len;    /* 現在の文字列長（null終端を含まない） */
    int         maxlen; /* バッファの総容量 */
    int         cursor; /* 走査用カーソル位置 */
} StringInfoData;

入力と出力

本構造体はデータ保持用であり、関数呼び出しの入力として渡される際は、StringInfo（StringInfoData*）として扱われます。出力として、操作後のバッファ状態（data の内容や len の更新）が保持されます。

前提条件と不変条件

  • maxlen が 0 の場合、読み取り専用モードとして扱われます。この状態で append 系関数を呼び出すとアサーションエラーが発生します。
  • data は palloc 等で確保されたメモリ領域を指す必要があります。

アルゴリズムまたは振る舞い

  • **動的拡張**: appendStringInfo 等の API を通じてデータが追加される際、必要に応じて repalloc が呼び出され、バッファが自動的に拡張されます。
  • **抽象化**: C 文字列（null 終端）とバイナリデータの両方を同一のインターフェースで管理します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な使用例）**:
  • contrib/cube/cube.c
  • contrib/dblink/dblink.c
  • contrib/hstore/hstore_io.c
  • **呼び出し先**: 構造体自体はデータ保持用であり、操作には appendStringInfo や initStringInfo などの API 関数が使用されます。

関連データ構造

  • StringInfo: StringInfoData へのポインタ型。

失敗モードと境界条件

  • **メモリ不足**: バッファ拡張時に repalloc が失敗した場合、メモリ管理システムによりエラーが報告されます。
  • **読み取り専用モード**: maxlen=0 の状態で変更操作を行うとアサーションエラーとなります。

保守担当者向けメモ

  • スタック上に確保した StringInfoData を破棄する際は、data フィールドのみを pfree する必要があります。
  • appendStringInfoCharMacro は引数を複数回評価するため、副作用のある式を渡さないよう注意してください。
