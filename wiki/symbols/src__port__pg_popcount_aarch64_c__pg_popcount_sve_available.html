pg_popcount_sve_available

定義

  • 種別: function
  • 定義箇所: src/port/pg_popcount_aarch64.c:59

目的

実行時のCPU環境において、AArch64のSVE（Scalable Vector Extension）命令セットが利用可能かどうかを判定する。

シグネチャまたは形状

static inline bool pg_popcount_sve_available(void)

入力と出力

  • 入力: なし
  • 出力: SVEが利用可能な場合は true、それ以外の場合は false を返す。

前提条件と不変条件

  • POPCNT_AARCH64 が定義された環境でコンパイルされていること。
  • USE_SVE_POPCNT_WITH_RUNTIME_CHECK が定義されている場合にのみ、実行時の判定ロジックが有効に機能する。

アルゴリズムまたは振る舞い

1. HAVE_ELF_AUX_INFO が定義されている場合、elf_aux_info を用いて AT_HWCAP フラグを取得し、HWCAP_SVE ビットを確認する。 2. HAVE_GETAUXVAL が定義されている場合、getauxval(AT_HWCAP) を呼び出し、同様に HWCAP_SVE ビットを確認する。 3. 上記いずれも利用できない環境では、常に false を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/pg_popcount_aarch64.c 内の初期化処理（pg_popcount_choose 等の文脈）から、SVE実装を選択すべきかの判断基準として呼び出される。
  • 呼び出し先: elf_aux_info または getauxval（プラットフォーム依存のシステムコール）。

関連データ構造

  • AT_HWCAP: CPU機能フラグを取得するための補助ベクトル型定数。
  • HWCAP_SVE: SVEサポートを示すビットマスク。

失敗モードと境界条件

  • システムコール（elf_aux_info や getauxval）が利用できない環境では、SVEが物理的に存在していても false を返す。
  • AT_HWCAP の取得に失敗した場合、false を返す。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、コンパイル時の環境変数に応じて適切なシステムコールを選択する。
  • SVEサポートの判定は、実行時の動的ディスパッチにおいて重要な役割を果たす。
