update_node

定義

  • 種別: function
  • 定義箇所: contrib/fuzzystrmatch/daitch_mokotoff.c:261

目的

Daitch-Mokotoff Soundex アルゴリズムにおいて、音韻コードのツリー構造を更新し、新しい音韻単位の追加や分岐を管理します。

シグネチャまたは形状

static void update_node(dm_node *first_node[2], dm_node *last_node[2],
                        dm_node *node, int ix_node,
                        int letter_no, int prev_code_index, int next_code_index,
                        const char *next_code_digits, int digit_no,
                        ArrayBuildState *soundex)

入力と出力

  • **入力**:
  • first_node, last_node: リーフノードのリンクリスト管理用ポインタ配列。
  • node: 更新対象の現在のノード。
  • next_code_digits: 追加対象の音韻コード文字列。
  • digit_no: 現在処理中のコード桁位置。
  • soundex: 結果を蓄積する ArrayBuildState。
  • **出力**: なし（副作用として node の状態更新および soundex への蓄積を行う）。

前提条件と不変条件

  • node は有効な dm_node 構造体を指している必要があります。
  • next_code_digits はヌル終端された文字列である必要があります。

アルゴリズムまたは振る舞い

1. initialize_node を呼び出し、現在のノードを初期化します。 2. prev_code_index を用いて、現在の文字エンコーディングが直前の音韻状態と矛盾しないか検証し、矛盾する場合は処理を中断します。 3. next_code_digit が 'X'（無視される文字）であるか、既存のコードと重複する場合は、現在のノードを「ダーティ」としてマークします。 4. 新しいコード桁が必要な場合は find_or_create_child_node を呼び出し、子ノードを生成または取得します。 5. ダーティな各ノードに対し add_next_code_digit を実行します。 6. next_code_digits に後続の桁がある場合は再帰的に update_node を呼び出し、ない場合は set_leaf を呼び出してリーフノードとしてリストに登録します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/fuzzystrmatch/daitch_mokotoff.c 内の update_node（再帰呼び出し）および他の関数（ソースコード上の直接的な呼び出し元は daitch_mokotoff_coding 等の文脈が示唆されますが、明示的なリストは提供されていません）。
  • **呼び出し先**: initialize_node, find_or_create_child_node, add_next_code_digit, set_leaf, update_node（再帰）。

関連データ構造

  • dm_node: 音韻コードのツリー構造を保持するノード。
  • ArrayBuildState: PostgreSQL の配列構築用状態管理構造体。

失敗モードと境界条件

  • prev_code_index による検証で不一致が発生した場合、そのパスの処理は即座に終了します。
  • find_or_create_child_node がメモリ不足等で NULL を返した場合、その分岐の処理は行われません。

保守担当者向けメモ

  • 本関数は再帰的に呼び出されるため、スタック深さに注意してください。
  • next_code_digits の処理において、'X' はコードのスキップを意味する特殊な値として扱われています。
  • 複数の音韻コードが生成される性質上、入力文字列の長さに対して計算量が指数的に増加する可能性がある点に留意してください。
