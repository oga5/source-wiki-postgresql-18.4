GetErrorContextStack

定義

  • 種別: function
  • 定義箇所: src/backend/utils/error/elog.c:2063

目的

現在登録されているエラーコンテキストコールバックを順次実行し、その結果として生成されたコンテキスト情報の文字列を返します。

シグネチャまたは形状

char *GetErrorContextStack(void)

入力と出力

  • **入力**: なし（グローバルな error_context_stack を参照）
  • **出力**: コンテキスト情報が格納された文字列（char *）を返します。

前提条件と不変条件

  • 呼び出し元のメモリコンテキスト（CurrentMemoryContext）が、戻り値の文字列の割り当てに使用されます。
  • recursion_depth を一時的にインクリメントし、再帰的なエラー発生を監視します。

アルゴリズムまたは振る舞い

1. recursion_depth をインクリメントします。 2. get_error_stack_entry() を呼び出し、作業用の ErrorData エントリを確保します。 3. edata->assoc_context に呼び出し元の CurrentMemoryContext を設定します。 4. error_context_stack を辿り、各コールバック関数（econtext->callback）を実行して edata->context に情報を蓄積します。 5. errordata_stack_depth と recursion_depth をデクリメントしてスタックを解放します。 6. 蓄積された edata->context を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpgsql/src/pl_exec.c:2501 (確認済み)
  • **呼び出し先**:
  • get_error_stack_entry() (内部関数)
  • error_context_stack に登録された各コールバック関数

関連データ構造

  • ErrorData: エラー情報を保持する構造体。
  • ErrorContextCallback: コンテキスト情報を収集するためのコールバック構造体。
  • error_context_stack: コンテキストコールバックの連結リスト。

失敗モードと境界条件

  • コールバック関数内でエラーが発生した場合、通常の PostgreSQL エラー処理メカニズムを通じて再帰的なエラーとして扱われます。
  • 戻り値の文字列は呼び出し元のメモリコンテキストで割り当てられるため、呼び出し元がメモリ管理に責任を持ちます。

保守担当者向けメモ

  • 本関数は recursion_depth を操作することで、エラー処理中の再帰的なエラー発生を制御しています。変更を加える際は、無限ループが発生しないよう注意してください。
