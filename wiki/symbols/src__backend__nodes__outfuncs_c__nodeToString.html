nodeToString

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/outfuncs.c:804

目的

PostgreSQLの内部ノード構造を、デバッグやシリアライズに適したASCII文字列形式に変換する公開APIです。

シグネチャまたは形状

char *nodeToString(const void *obj)

入力と出力

  • **入力**: obj (変換対象のノード構造体へのポインタ)
  • **出力**: ノードの文字列表現を保持する動的に割り当てられた char * 型の文字列。

前提条件と不変条件

  • obj は有効なPostgreSQLノード構造体である必要があります。
  • 内部で check_stack_depth() を呼び出しており、再帰的なノード走査においてスタックオーバーフローを防止する不変条件が適用されます。

アルゴリズムまたは振る舞い

1. nodeToStringInternal(obj, false) を呼び出します。 2. 第2引数に false を渡すことで、パース時の位置情報（location）を除外した文字列を生成します。 3. 内部的に StringInfo バッファを使用して効率的に文字列を構築し、結果を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c
  • src/backend/catalog/index.c
  • src/backend/catalog/pg_attrdef.c
  • src/backend/catalog/pg_proc.c
  • src/backend/catalog/pg_publication.c
  • src/backend/commands/policy.c
  • **呼び出し先**: nodeToStringInternal

関連データ構造

  • StringInfo: 文字列構築用バッファ。
  • Node: 変換対象となるノードの基底構造体。

失敗モードと境界条件

  • obj が NULL の場合、または不正なノードタグを持つ場合、内部の outNode 処理においてエラーが発生する可能性があります。
  • 非常に深いノードツリーを処理する場合、スタック制限に達するとエラーが発生します。

保守担当者向けメモ

  • 本関数は nodeToStringInternal のラッパーです。位置情報を含めた文字列が必要な場合は nodeToStringWithLocations を使用してください。
  • readfuncs.c による復元を想定しているため、出力形式の変更には注意が必要です。
