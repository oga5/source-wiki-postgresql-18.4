GISTBuildState

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gist/gistbuild.c:82

目的

GiSTインデックスの構築プロセス全体を通じて、インデックス関係、メモリコンテキスト、および選択された構築戦略（ソート済み構築またはバッファリング構築）に関連する状態を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    Relation    indexrel;
    Relation    heaprel;
    GISTSTATE  *giststate;
    Size        freespace;
    GistBuildMode buildMode;
    int64       indtuples;
    int64       indtuplesSize;
    GISTBuildBuffers *gfbb;
    HTAB       *parentMap;
    Tuplesortstate *sortstate;
    BlockNumber pages_allocated;
    BulkWriteState *bulkstate;
} GISTBuildState;

入力と出力

  • **入力:** インデックス構築開始時に初期化され、ヒープリレーションやインデックスリレーションへの参照を保持します。
  • **出力:** 構築プロセス終了時、BulkWriteState を通じたインデックスの書き込みや、バッファリング構築におけるバッファのフラッシュ処理に使用されます。

前提条件と不変条件

  • buildMode に応じて、必要なデータ構造（gfbb/parentMap または sortstate）が適切に初期化されている必要があります。
  • indexrel および heaprel は構築期間中有効である必要があります。

アルゴリズムまたは振る舞い

  • **戦略管理:** GistBuildMode に基づき、ソート済み構築（tuplesort を利用）またはバッファリング構築（GISTBuildBuffers を利用）のいずれかのパスを制御します。
  • **統計収集:** indtuples および indtuplesSize を通じて、構築中のインデックスの進捗とサイズを追跡します。
  • **リソース管理:** BulkWriteState を保持し、大量のインデックスページ書き込みを効率化します。

呼び出し元と呼び出し先

  • **呼び出し元:** gistbuild 関数（インデックス構築のメインエントリーポイント）により初期化されます。
  • **呼び出し先:** 以下の関数群が GISTBuildState を引数として受け取り、状態を更新または参照します。
  • gist_indexsortbuild, gist_indexsortbuild_levelstate_add, gist_indexsortbuild_levelstate_flush
  • gistInitBuffering, calculatePagesPerBuffer
  • gistBufferingBuildInsert, gistProcessItup, gistbufferinginserttuples
  • gistBufferingFindCorrectParent, gistProcessEmptyingQueue, gistEmptyAllBuffers

関連データ構造

  • GistBuildMode: 構築戦略を定義する列挙型。
  • GISTBuildBuffers: バッファリング構築時に使用されるバッファ管理構造体。
  • Tuplesortstate: ソート済み構築時に使用されるソート状態。
  • BulkWriteState: 大量書き込みを管理する構造体。
  • HTAB: parentMap として使用されるハッシュテーブル。

失敗モードと境界条件

  • メモリ不足: tuplesort や GISTBuildBuffers が大量のメモリを消費する場合、システムのリソース制限に達する可能性があります。
  • バッファリング構築の切り替え: インデックスサイズが effective_cache_size を超える場合にバッファリングモードへ移行するロジックが正常に機能しない場合、I/O効率が低下します。

保守担当者向けメモ

  • 本構造体は「ソート済み構築」と「バッファリング構築」という2つの異なるパスのデータを混在させて保持しています。関数内でアクセスする際は、現在の buildMode がどちらのパスであるかを必ず確認してください。
  • parentMap はバッファリング構築時にのみ使用されるため、ソート済み構築時には NULL であることが期待されます。
