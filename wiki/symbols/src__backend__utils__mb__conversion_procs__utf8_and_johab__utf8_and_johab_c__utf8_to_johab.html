utf8_to_johab

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_johab/utf8_and_johab.c:62

目的

UTF-8エンコーディングの文字列をJOHABエンコーディングへ変換するPostgreSQLの関数です。SQLレベルでの文字コード変換処理を担います。

シグネチャまたは形状

Datum utf8_to_johab(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の引数を受け取ります。
  • src (第3引数): 変換元のUTF-8文字列。
  • dest (第4引数): 変換結果を格納するバッファ。
  • len (第5引数): 変換対象のバイト長。
  • noError (第6引数): 変換エラー時にエラーを投げるか（false）、不正な文字をスキップするか（true）を制御するフラグ。
  • **出力**: 変換されたバイト数を Datum 型（int32）として返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_UTF8, PG_JOHAB) マクロにより、変換元がUTF-8、変換先がJOHABであることを検証します。
  • 変換には johab_from_unicode_tree を使用します。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得します。 2. エンコーディングの整合性を検証します。 3. UtfToLocal 関数を呼び出し、johab_from_unicode_tree を用いてUTF-8からJOHABへの変換を実行します。 4. 変換されたバイト数を戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの変換フレームワーク（pg_conversion システムカタログ経由でSQLエンジンから呼び出される）。
  • **呼び出し先**: UtfToLocal（src/backend/utils/mb/wchar.c で定義される汎用変換関数）。

関連データ構造

  • johab_from_unicode_tree: ../../Unicode/utf8_to_johab.map で定義される変換用木構造データ。

失敗モードと境界条件

  • noError が false の場合、変換不可能な文字が含まれるとエラーが発生します。
  • noError が true の場合、不正な文字はスキップされます。

保守担当者向けメモ

  • 本関数は変換ロジックの「接着剤」であり、実際の変換アルゴリズムは mb/wchar.c の汎用関数とマップファイルに依存しています。
  • マップファイルは自動生成されることが多いため、編集時には注意が必要です。
