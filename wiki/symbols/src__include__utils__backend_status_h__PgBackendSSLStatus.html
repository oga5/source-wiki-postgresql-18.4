PgBackendSSLStatus

定義

  • 種別: struct
  • 定義箇所: src/include/utils/backend_status.h:50

目的

バックエンドプロセスのSSL接続に関する詳細な統計情報を保持し、共有メモリ上で管理するための構造体です。

シグネチャまたは形状

typedef struct PgBackendSSLStatus
{
    int         ssl_bits;
    char        ssl_version[NAMEDATALEN];
    char        ssl_cipher[NAMEDATALEN];
    char        ssl_client_dn[NAMEDATALEN];
    char        ssl_client_serial[NAMEDATALEN];
    char        ssl_issuer_dn[NAMEDATALEN];
} PgBackendSSLStatus;

入力と出力

  • **入力**: SSL接続確立時に取得される暗号化強度、プロトコルバージョン、暗号スイート、および証明書情報（DN、シリアル番号、発行者DN）。
  • **出力**: 外部からの監視要求に対し、共有メモリ上の当該構造体を通じて現在のSSL接続状態を提供します。

前提条件と不変条件

  • ssl_client_serial は RFC 5280 に基づき最大20オクテットであるため、NAMEDATALEN サイズで十分な領域が確保されています。
  • 共有メモリ上のデータであるため、読み書きには backend_status.h で定義されたチェンジカウンタ方式のプロトコル（PGSTAT_BEGIN_WRITE_ACTIVITY 等）による保護が必要です。

アルゴリズムまたは振る舞い

  • バックエンドのSSL状態が変化した際、対応するメモリ領域を更新します。
  • 読み取り側は、チェンジカウンタの整合性を確認することで、更新中の不整合なデータを取得しないよう制御されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/backend_status.c 内の関数群が、共有メモリの初期化、クリア（memset）、および状態のコピー（memcpy）に使用しています。
  • **呼び出し先**: 提供されたソース情報からは、本構造体が直接呼び出す関数や、本構造体を直接呼び出す外部モジュールの詳細は特定できません。

関連データ構造

  • PgBackendStatus: バックエンドの全体的な状態を保持する構造体であり、本構造体をメンバとして含む可能性があります。

失敗モードと境界条件

  • NAMEDATALEN を超える文字列データが渡された場合、バッファオーバーフローの危険性があるため、格納前に切り詰めが必要です。
  • 共有メモリの初期化前にアクセスした場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本構造体は統計情報収集システム（pgstat）とは独立したリアルタイム監視用です。累積統計と混同しないよう注意してください。
  • 構造体のメンバを変更する場合は、共有メモリのレイアウトに影響するため、関連する memcpy や memset のサイズ指定箇所をすべて更新してください。
