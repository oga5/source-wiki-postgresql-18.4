bbsink_copystream_new

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_copy.c:107

目的

バックアップデータをクライアントへ転送するための bbsink_copystream インスタンスを生成・初期化するコンストラクタ。

シグネチャまたは形状

bbsink *bbsink_copystream_new(bool send_to_client)

入力と出力

  • **入力**: send_to_client (bool) - クライアントへデータを送信するかどうかを指定するフラグ。
  • **出力**: bbsink * - 初期化された bbsink 構造体へのポインタ。

前提条件と不変条件

  • palloc0 によりメモリがゼロクリアされた状態で割り当てられる。
  • bbs_ops には bbsink_copystream_ops が静的に割り当てられる。
  • 進捗報告用のタイマー（last_progress_report_time）は GetCurrentTimestamp() で初期化され、累積バイト数（bytes_done_at_last_time_check）は 0 に設定される。

アルゴリズムまたは振る舞い

1. bbsink_copystream 構造体分のメモリを確保する。 2. base.bbs_ops に bbsink_copystream_ops を設定し、インターフェースをバインドする。 3. 引数 send_to_client を構造体に保存する。 4. 進捗報告の追跡用フィールドを初期化する。 5. bbsink 基底構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup.c:1034 にて呼び出されている。
  • **呼び出し先**: palloc0 (メモリ確保), GetCurrentTimestamp (時刻取得)。

関連データ構造

  • bbsink_copystream: 本関数で生成されるメイン構造体。
  • bbsink_copystream_ops: 関数ポインタテーブル。
  • bbsink: 継承元の基底構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる。

保守担当者向けメモ

  • 本関数は bbsink インターフェースのコンストラクタであり、basebackup サブシステムにおける COPY OUT プロトコル実装の起点となる。
  • bbsink_copystream_ops の定義と整合性を保つ必要がある。
