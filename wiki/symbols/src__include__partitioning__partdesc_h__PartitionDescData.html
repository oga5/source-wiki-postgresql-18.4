PartitionDescData

定義

  • 種別: struct
  • 定義箇所: src/include/partitioning/partdesc.h:29

目的

パーティション化されたテーブルの構成情報（OID、リーフフラグ、境界情報）を保持し、get_partition_for_tuple() における検索処理を高速化するためのキャッシュフィールドを提供する。

シグネチャまたは形状

typedef struct PartitionDescData
{
    int             nparts;
    bool            detached_exist;
    Oid            *oids;
    bool           *is_leaf;
    PartitionBoundInfo boundinfo;
    int             last_found_datum_index;
    int             last_found_part_index;
    int             last_found_count;
} PartitionDescData;

入力と出力

  • **入力**: 構造体生成時にシステムカタログから構築されたパーティション情報。
  • **出力**: 検索処理のヒントとして利用される last_found_* フィールドの更新。

前提条件と不変条件

  • oids および is_leaf 配列は nparts 個の要素を持つ。
  • PartitionDesc（本構造体へのポインタ）は、リレーションキャッシュ（relcache）の生存期間に依存する。リレーションのロックを保持している間のみ安全に利用可能。

アルゴリズムまたは振る舞い

  • **検索キャッシュ**: get_partition_for_tuple() 実行時、直前の検索結果を last_found_datum_index および last_found_part_index に保持する。
  • **最適化**: last_found_count を用いて、LIST/RANGE パーティションにおける連続した検索のヒット率を追跡し、検索効率を向上させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/partitioning/partdesc.c 内でメモリ確保および初期化が行われる。
  • **呼び出し先**:
  • 提供されたソース情報には、本構造体内の関数ポインタやメソッド呼び出しに関する情報は含まれていない。

関連データ構造

  • PartitionBoundInfo: パーティション境界情報のコレクション。
  • PartitionDesc: PartitionDescData へのポインタ型（partdefs.h で定義）。

失敗モードと境界条件

  • last_found_* フィールドは、検索が一度も行われていない場合 -1 で初期化される。
  • omit_detached フラグが有効な場合、スナップショットの状態に応じてキャッシュの再利用可否が動的に変化する。

保守担当者向けメモ

  • 本構造体は relcache と密接に統合されている。キャッシュの無効化処理については src/backend/utils/cache/partcache.c を参照すること。
  • detached_exist フラグは、デタッチされたパーティションが存在するかを判定するために使用される。
