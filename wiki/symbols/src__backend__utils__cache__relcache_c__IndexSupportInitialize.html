IndexSupportInitialize

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/relcache.c:1615

目的

インデックスの演算子クラス（opclass）情報をキャッシュから取得し、リレーション記述子のサポートプロシージャおよび関連情報を初期化する。

シグネチャまたは形状

static void IndexSupportInitialize(oidvector *indclass,
                                   RegProcedure *indexSupport,
                                   Oid *opFamily,
                                   Oid *opcInType,
                                   StrategyNumber maxSupportNumber,
                                   AttrNumber maxAttributeNumber)

入力と出力

  • **入力**:
  • indclass: インデックスの演算子クラス OID の配列。
  • maxSupportNumber: サポートプロシージャの最大数。
  • maxAttributeNumber: インデックス対象の属性数。
  • **出力**:
  • indexSupport: サポートプロシージャの OID を格納するバッファ。
  • opFamily: 演算子ファミリーの OID を格納するバッファ。
  • opcInType: 演算子クラスの入力型 OID を格納するバッファ。

前提条件と不変条件

  • indclass 内の各 OID は有効である必要がある（無効な場合は ERROR を発生させる）。
  • indexSupport、opFamily、opcInType は、それぞれ必要なサイズ分が事前に割り当てられている必要がある。

アルゴリズムまたは振る舞い

1. maxAttributeNumber 回ループし、各属性の演算子クラスを処理する。 2. LookupOpclassInfo を呼び出し、キャッシュから当該演算子クラスの情報を取得する。 3. 取得した OpClassCacheEnt から opFamily および opcInType を対応する出力バッファへコピーする。 4. maxSupportNumber > 0 の場合、memcpy を用いてサポートプロシージャのリストを indexSupport バッファへコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/relcache.c 内の RelationInitIndexAccessInfo から呼び出される。
  • **呼び出し先**: LookupOpclassInfo（演算子クラス情報の取得）。

関連データ構造

  • OpClassCacheEnt: 演算子クラスのキャッシュエントリ。
  • oidvector: 演算子クラス OID の配列。
  • RegProcedure: プロシージャ OID。

失敗モードと境界条件

  • indclass->values[attIndex] が無効な OID である場合、elog(ERROR, "bogus pg_index tuple") により処理が中断される。
  • maxSupportNumber が 0 の場合、サポートプロシージャのコピーはスキップされる。

保守担当者向けメモ

  • 本関数はリレーションキャッシュ構築の一部として実行される。
  • LookupOpclassInfo を介したキャッシュ利用により、システムカタログへの直接アクセスを回避している。
  • indexSupport 配列への書き込みは attIndex * maxSupportNumber のオフセット計算に基づいているため、バッファサイズとインデックス計算の整合性に注意すること。
