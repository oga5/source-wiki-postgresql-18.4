pgstat_io_init_shmem_cb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_io.c:277

目的

I/O統計用の共有メモリ領域を初期化する際、各バックエンドタイプに対応する軽量ロック（LWLock）を初期化します。

シグネチャまたは形状

void pgstat_io_init_shmem_cb(void *stats)

入力と出力

  • **入力**: stats (共有メモリ上の PgStatShared_IO 構造体へのポインタ)
  • **出力**: なし（副作用として stats 内のロックが初期化される）

前提条件と不変条件

  • stats 引数は PgStatShared_IO 型としてキャスト可能である必要があります。
  • 共有メモリの割り当てが完了した直後の初期化フェーズで呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. 引数 stats を PgStatShared_IO * 型にキャストします。 2. BACKEND_NUM_TYPES で定義されたバックエンドタイプの数だけループを実行します。 3. 各バックエンドタイプに対応する stat_shmem->locks[i] に対して、LWTRANCHE_PGSTATS_DATA を指定して LWLockInitialize を実行し、ロックを有効化します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c:444 において、統計サブシステムの初期化コールバックとして登録されています。
  • **呼び出し先**: LWLockInitialize（PostgreSQLのストレージ管理ライブラリ関数）。

関連データ構造

  • PgStatShared_IO: I/O統計を保持する共有メモリ構造体。
  • LWLock: 統計データへのアクセスを制御するための軽量ロック。

失敗モードと境界条件

  • stats が NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • BACKEND_NUM_TYPES の範囲外へのアクセスは発生しない設計ですが、共有メモリのサイズが不整合な場合はメモリ破壊の恐れがあります。

保守担当者向けメモ

  • 本関数は共有メモリの初期化時に一度だけ実行されることを想定しています。
  • 新しいバックエンドタイプが追加された場合、BACKEND_NUM_TYPES の定義が適切に更新されていることを確認してください。
