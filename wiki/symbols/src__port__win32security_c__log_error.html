log_error

定義

  • 種別: function
  • 定義箇所: src/port/win32security.c:27

目的

PostgreSQL の初期化フェーズなど、標準的な ereport() が利用できない状況下において、標準エラー出力（stderr）へ簡易的にエラーメッセージを出力する。

シグネチャまたは形状

static void log_error(const char *fmt, ...);

入力と出力

  • 入力: fmt（書式指定文字列）、および可変長引数（...）。
  • 出力: なし（戻り値は void）。標準エラー出力へフォーマットされた文字列を書き出す。

前提条件と不変条件

  • stdarg.h で定義される va_list を使用するため、呼び出し側は適切な引数を提供する必要がある。
  • FRONTEND マクロの定義有無により、出力先を制御する。

アルゴリズムまたは振る舞い

1. va_start を呼び出し、可変長引数リストを初期化する。 2. FRONTEND マクロが定義されていない場合（バックエンド）、vwrite_stderr を使用して出力する。 3. FRONTEND マクロが定義されている場合（フロントエンド）、vfprintf(stderr, ...) を使用して標準エラー出力へ書き出す。 4. va_end を呼び出し、引数リストを解放する。

呼び出し元と呼び出し先

  • 呼び出し先: vwrite_stderr（バックエンド時）、vfprintf（フロントエンド時）。
  • 呼び出し元: src/common/exec.c 内の各所（log_error マクロ経由での呼び出しが確認されている）。

関連データ構造

  • va_list: 可変長引数を処理するための標準型。

失敗モードと境界条件

  • stderr がクローズされている場合や、書き込み権限がない環境では出力に失敗する可能性がある。
  • 可変長引数の型が fmt の指定と一致しない場合、未定義動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は ereport() が利用できない極めて早い段階での使用を想定している。
  • src/common/exec.c にてマクロ定義されており、呼び出し時には errcode() 等の関数を引数として渡す設計となっている。
