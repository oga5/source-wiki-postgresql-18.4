ParamExecData

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/params.h:146

目的

クエリ実行エンジンにおいて、サブプランの実行結果など、エグゼキュータ内部で一時的に生成・保持されるパラメータ値を管理するために使用されます。

シグネチャまたは形状

typedef struct ParamExecData
{
	void	   *execPlan;		/* should be "SubPlanState *" */
	Datum		value;
	bool		isnull;
} ParamExecData;

入力と出力

  • **入力**: execPlan（サブプランの状態へのポインタ）、value（パラメータ値）、isnull（値がNULLかどうかのフラグ）。
  • **出力**: エグゼキュータ内部で参照されるパラメータ値の保持。

前提条件と不変条件

  • execPlan は SubPlanState * として扱われることが想定されています。
  • isnull が true の場合、value の内容は無視されるか、未定義として扱われます。

アルゴリズムまたは振る舞い

エグゼキュータは、クエリ実行時に ParamExecData の配列を確保し、各パラメータの実行状態や計算結果を保持します。サブプランの評価が必要な場合、execPlan を通じて関連する状態にアクセスし、計算された value を格納します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • src/backend/executor/execMain.c: es_param_exec_vals として配列のメモリ確保に使用。
  • src/backend/executor/execExprInterp.c, src/backend/executor/execParallel.c, src/backend/executor/nodeCtescan.c, src/backend/executor/nodeNestloop.c, src/backend/executor/nodeRecursiveunion.c: パラメータ値の取得や操作のために参照。
  • **呼び出し先**: 構造体定義のため、特定の関数呼び出しは含みません。

関連データ構造

  • SubPlanState: execPlan フィールドが指し示す対象。
  • Datum: value フィールドの型。

失敗モードと境界条件

  • メモリ確保時（palloc0）に失敗した場合、システム全体でメモリ不足エラーが発生します。
  • execPlan が正しく初期化されていない状態で参照された場合、未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は PARAM_EXEC 型のパラメータ専用です。外部から注入される PARAM_EXTERN パラメータとは管理機構が異なります。
  • execPlan フィールドは void * で定義されていますが、実態は SubPlanState * であるため、キャストの整合性に注意してください。
