transfer_all_new_tablespaces

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/relfilenumber.c:107

目的

pg_upgrade におけるデータベースファイル移行処理の起点であり、設定された転送モードに基づき、全テーブルスペースにわたるリレーションファイルの移行（コピー、リンク、クローン、またはスワップ）を制御します。

シグネチャまたは形状

void transfer_all_new_tablespaces(DbInfoArr *old_db_arr, DbInfoArr *new_db_arr, char *old_pgdata, char *new_pgdata)

入力と出力

  • **入力**:
  • old_db_arr, new_db_arr: 旧および新クラスタのデータベース情報配列。
  • old_pgdata, new_pgdata: 旧および新クラスタのデータディレクトリパス。
  • **出力**: なし（副作用としてファイルシステム操作を実行）。

前提条件と不変条件

  • user_opts.transfer_mode が有効な転送モード（CLONE, COPY, COPY_FILE_RANGE, LINK, SWAP）に設定されていること。
  • os_info.num_old_tablespaces および os_info.old_tablespaces が適切に初期化されていること。

アルゴリズムまたは振る舞い

1. user_opts.transfer_mode に応じて、進捗状況を表示するメッセージ（prep_status_progress）を選択します。 2. 並列実行数（user_opts.jobs）を確認します。

  • **非並列モード (jobs <= 1)**: parallel_transfer_all_new_dbs を呼び出し、テーブルスペースパスに NULL を渡して全テーブルスペースを対象に処理を実行します。
  • **並列モード (jobs > 1)**:
  • デフォルトのテーブルスペース（old_pgdata）に対して parallel_transfer_all_new_dbs を実行します。
  • 登録されている全ユーザーテーブルスペースに対してループし、それぞれ個別に parallel_transfer_all_new_dbs を実行します。
  • reap_child(true) を呼び出し、全子プロセスの終了を待機します。

    • 3. 最後に end_progress_output() および check_ok() を呼び出し、処理の完了を通知します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/pg_upgrade.c:185
  • **呼び出し先**:
  • parallel_transfer_all_new_dbs
  • reap_child
  • prep_status_progress, end_progress_output, check_ok（ユーティリティ関数）

関連データ構造

  • DbInfoArr: データベース情報の配列。
  • user_opts: ユーザー指定のオプション（転送モード、ジョブ数など）。
  • os_info: OSレベルのクラスタ情報（テーブルスペースリストなど）。

失敗モードと境界条件

  • 並列実行時に子プロセスの生成や管理でエラーが発生した場合、reap_child のループ内で適切に処理される必要があります。
  • 転送モードが未定義の場合の挙動は定義されていません。

保守担当者向けメモ

  • 本関数は並列実行の制御ロジックを含みます。デバッグ時は user_opts.jobs の値によって実行パスが大きく分岐することに留意してください。
  • swap モード時はディレクトリ構造を直接操作するため、他のモードよりも慎重な検証が必要です。
