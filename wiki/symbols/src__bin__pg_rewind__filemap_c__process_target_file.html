process_target_file

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/filemap.c:315

目的

同期先（ターゲット）データディレクトリ内のファイルをスキャンし、そのメタデータをハッシュテーブル（filehash）に登録して追跡する。

シグネチャまたは形状

void process_target_file(const char *path, file_type_t type, size_t size, const char *link_target)

入力と出力

  • **入力**:
  • path: 対象ファイルのパス。
  • type: ファイルの種別（file_type_t）。
  • size: ファイルサイズ。
  • link_target: シンボリックリンクの場合のリンク先パス（それ以外は NULL）。
  • **出力**: なし（副作用としてハッシュテーブルを更新）。

前提条件と不変条件

  • pg_wal パスがシンボリックリンクとして渡された場合、内部的に FILE_TYPE_DIRECTORY として扱われる。
  • 除外フィルタは適用されない（ソース側で除外されたファイルをターゲットから削除可能にするため）。

アルゴリズムまたは振る舞い

1. insert_filehash_entry(path) を呼び出し、該当パスの file_entry_t を取得または作成する。 2. 既に target_exists が true であれば、重複ファイルとして pg_fatal を発生させる。 3. 取得したエントリに対し、target_exists を true に設定し、target_type、target_size、target_link_target（存在する場合）を格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/pg_rewind.c 内の traverse_datadir からコールバックとして呼び出される。
  • **呼び出し先**:
  • insert_filehash_entry
  • pg_fatal
  • pg_strdup

関連データ構造

  • file_entry_t: ファイルのメタデータと同期状態を保持する構造体。
  • filehash: ファイルパスをキーとするハッシュテーブル。

失敗モードと境界条件

  • 同一パスに対して本関数が二度呼び出された場合、pg_fatal によりプロセスが終了する。

保守担当者向けメモ

  • 本関数は除外リストを適用しない設計であるため、ターゲットディレクトリのクリーンアップ（削除対象の特定）に不可欠な役割を果たす。修正時は file_entry_t の構造変更に注意すること。
