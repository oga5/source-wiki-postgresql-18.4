isbn_in

定義

  • 種別: function
  • 定義箇所: contrib/isn/isn.c:1002

目的

入力された文字列形式の ISBN を PostgreSQL 内部の ean13 型へ変換する。

シグネチャまたは形状

Datum isbn_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して取得される ISBN 文字列。
  • **出力**: 成功時は ean13 型の値を PG_RETURN_EAN13 マクロで返し、失敗時は PG_RETURN_NULL() を返す。

前提条件と不変条件

  • 入力文字列は ISBN として妥当な形式である必要がある。
  • string2ean 関数が内部でチェックデジットの検証を行う。

アルゴリズムまたは振る舞い

1. PG_GETARG_CSTRING(0) で入力文字列を取得する。 2. string2ean 関数を呼び出し、文字列を ean13 型へ変換する。この際、識別子として ISBN を指定する。 3. string2ean が false を返した場合は変換失敗とみなし、NULL を返す。 4. 成功した場合は、変換された ean13 値を返す。

呼び出し元と呼び出し先

  • **呼び出し先**: string2ean (文字列解析および検証用)。
  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1(isbn_in) により登録）。具体的な SQL クエリからの呼び出しについては、提供されたソースからは直接的な呼び出し元を特定できません。

関連データ構造

  • ean13: 内部ストレージ形式（uint64）。
  • ISBN: string2ean に渡される識別子定数。

失敗モードと境界条件

  • 入力文字列が ISBN の形式に適合しない場合、またはチェックデジットが不正な場合（g_weak 設定に依存）、string2ean が失敗し、本関数は NULL を返す。

保守担当者向けメモ

  • 本関数は contrib/isn モジュールの型変換ロジックの一部であり、ISBN 特有の検証ルールは string2ean および ISBN.h で定義された範囲テーブルに依存している。
