bloom_filter_size

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_bloom.c:269

目的

ブルームフィルタの構築に必要なビット数、バイト数、および最適なハッシュ関数数を、推定要素数と許容誤判定率に基づいて算出する。

シグネチャまたは形状

static void bloom_filter_size(int ndistinct, double false_positive_rate,
                              int *nbytesp, int *nbitsp, int *nhashesp)

入力と出力

  • **入力**:
  • ndistinct: ページ範囲内の推定要素数。
  • false_positive_rate: 許容する誤判定率。
  • nbytesp, nbitsp, nhashesp: 結果を格納するためのポインタ（NULL許容）。
  • **出力**:
  • なし（ポインタ引数経由で値を返却）。

前提条件と不変条件

  • nbytesp, nbitsp, nhashesp が NULL でない場合、それぞれ計算結果が書き込まれる。
  • 算出されたビット数は、バイト単位に切り上げられる。

アルゴリズムまたは振る舞い

1. **ビット数算出**: -(n * ln(p)) / (ln(2))^2 の公式に基づき、必要なビット数 nbits を算出。 2. **バイト数調整**: nbits を 8 の倍数に切り上げ、nbytes を決定。 3. **ハッシュ関数数算出**: ln(2) * m / n を計算し、四捨五入して最適なハッシュ関数数 k を決定。 4. **結果格納**: 各ポインタが非NULLであれば、算出した値を格納。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin_bloom.c 内の関数から呼び出されていることが確認されている（具体的な関数名はソース文脈からは特定不可）。
  • **呼び出し先**: math.h の ceil, log, pow, floor を使用。

関連データ構造

  • BloomFilter: 本関数で算出されたパラメータを用いて初期化される。

失敗モードと境界条件

  • ndistinct や false_positive_rate が極端な値の場合の挙動については、本ソース excerpt 内に明示的なエラーハンドリングの記述はない。

保守担当者向けメモ

  • 本関数は「Less Hashing, Same Performance」論文の手法に基づいている。
  • Windows環境での round() 関数の互換性を考慮し、floor と ceil を組み合わせた四捨五入ロジックを採用している。
