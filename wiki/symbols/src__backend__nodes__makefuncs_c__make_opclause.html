make_opclause

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/makefuncs.c:700

目的

演算子式を表す OpExpr ノードを生成し、初期化する。

シグネチャまたは形状

Expr *make_opclause(Oid opno, Oid opresulttype, bool opretset,
                    Expr *leftop, Expr *rightop,
                    Oid opcollid, Oid inputcollid)

入力と出力

  • **入力**:
  • opno: 演算子のOID。
  • opresulttype: 演算結果の型OID。
  • opretset: 演算子が集合を返す場合は true。
  • leftop: 左辺の式ノード。
  • rightop: 右辺の式ノード（単項演算子の場合は NULL）。
  • opcollid: 演算子の照合順序（Collation）OID。
  • inputcollid: 入力値の照合順序OID。
  • **出力**: 構築された OpExpr ノードへのポインタ（Expr * 型として返却）。

前提条件と不変条件

  • makeNode(OpExpr) によりメモリが確保される。
  • opfuncid は常に InvalidOid で初期化される。
  • location フィールドは常に -1（不明）に設定される。

アルゴリズムまたは振る舞い

1. makeNode(OpExpr) を呼び出し、ノード領域を確保する。 2. 引数 opno, opresulttype, opretset, opcollid, inputcollid を構造体の対応するフィールドに代入する。 3. opfuncid に InvalidOid を設定する。 4. rightop が存在する場合、list_make2 を使用して leftop と rightop を引数リストに格納する。存在しない場合は list_make1 で leftop のみを格納する。 5. location を -1 に設定し、構築したノードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/indxpath.c
  • src/backend/optimizer/plan/initsplan.c
  • src/backend/optimizer/plan/planner.c
  • src/backend/optimizer/plan/subselect.c
  • src/backend/optimizer/util/predtest.c
  • **呼び出し先**:
  • makeNode (マクロ)
  • list_make1 / list_make2

関連データ構造

  • OpExpr: 本関数が生成する主要な構造体。
  • Expr: 戻り値の型。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • rightop が NULL の場合、単項演算子として扱われ、args リストは要素数1で構築される。
  • メモリ不足時は makeNode 経由でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は OpExpr の基本的な初期化のみを行う。opfuncid の解決など、より詳細な設定が必要な場合は呼び出し側で別途処理すること。
