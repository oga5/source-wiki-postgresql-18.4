load_validator_library

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-oauth.c:737

目的

OAuth認証用の外部検証モジュールを動的にロードし、コールバック関数群（ValidatorCallbacks）および内部状態（validator_module_state）を初期化する。

シグネチャまたは形状

static void load_validator_library(const char *libname)

入力と出力

  • **入力**: libname (ロード対象の共有ライブラリパスを示す文字列)
  • **出力**: なし（成功時）。失敗時は ereport(ERROR, ...) によりプロセスを中断する。

前提条件と不変条件

  • libname は NULL であってはならず、空文字列でも不可（Assert で検証済み）。
  • 指定されたライブラリは _PG_oauth_validator_module_init シンボルをエクスポートしている必要がある。
  • check_oauth_validator により、ライブラリの存在と妥当性は事前に確認されているものとする。

アルゴリズムまたは振る舞い

1. load_external_function を使用して、指定ライブラリから _PG_oauth_validator_module_init 関数をロードする。 2. ロードした初期化関数を実行し、ValidatorCallbacks 構造体を取得する。 3. 取得したコールバック構造体の magic 番号が PG_OAUTH_VALIDATOR_MAGIC と一致するか検証する。 4. 必須コールバックである validate_cb が存在するか確認する。 5. ValidatorModuleState をメモリ確保し、startup_cb が定義されていれば実行する。 6. MemoryContextRegisterResetCallback を使用し、メモリコンテキストリセット時に shutdown_validator_library が呼ばれるよう登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-oauth.c:120 (oauth_init 内)
  • **呼び出し先**:
  • load_external_function (外部ライブラリロード)
  • palloc0 (メモリ確保)
  • MemoryContextRegisterResetCallback (終了時処理の登録)
  • shutdown_validator_library (リセットコールバックとして登録)
  • ValidatorCallbacks 内の startup_cb (存在する場合)

関連データ構造

  • ValidatorCallbacks: 検証モジュールが提供するコールバック関数群。
  • ValidatorModuleState: 検証モジュールのプライベート状態を保持する構造体。

失敗モードと境界条件

  • _PG_oauth_validator_module_init シンボルが見つからない場合、エラーを報告。
  • magic 番号が不一致の場合、エラーを報告。
  • 必須の validate_cb が NULL の場合、エラーを報告。

保守担当者向けメモ

  • 本関数は check_oauth_validator による事前チェックを前提としているため、ライブラリのパス検証は最小限に留めている。
  • ValidatorCallbacks の ABI 互換性は magic 番号で管理されており、メジャーバージョン内での変更に対応している。
