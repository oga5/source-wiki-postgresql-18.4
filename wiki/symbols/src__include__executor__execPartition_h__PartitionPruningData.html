PartitionPruningData

定義

  • 種別: struct
  • 定義箇所: src/include/executor/execPartition.h:84

目的

クエリ実行時におけるパーティションプルーニングに必要な情報を保持するデータ構造です。

シグネチャまたは形状

typedef struct PartitionPruningData
{
    int         num_partrelprunedata;
    PartitionedRelPruningData partrelprunedata[FLEXIBLE_ARRAY_MEMBER];
} PartitionPruningData;

入力と出力

  • 入力: 実行計画の初期化時に構築されるパーティション階層情報。
  • 出力: 実行時に評価される PartitionedRelPruningData の配列。

前提条件と不変条件

  • partrelprunedata は可変長配列として定義されており、num_partrelprunedata に基づいてメモリ確保が行われる必要があります。

アルゴリズムまたは振る舞い

  • パーティション階層ごとに保持され、実行時に不要なパーティション（サブプラン）を特定するために使用されます。
  • find_matching_subplans_recurse 等の関数を通じて、階層構造を再帰的に走査し、プルーニング対象を決定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execPartition.c 内のプルーニング初期化および実行ロジック。
  • 呼び出し先: PartitionedRelPruningData を参照し、プルーニング処理を実行します。

関連データ構造

  • PartitionedRelPruningData: 本構造体内の配列要素として保持される、個別のパーティション関係のプルーニング情報。
  • PartitionPruneState: 本構造体を保持し、実行時の状態管理を行う上位構造体。

失敗モードと境界条件

  • メモリ確保時、offsetof を用いた適切なサイズ計算が必須です（src/backend/executor/execPartition.c:2033 参照）。
  • 階層構造が深い場合、再帰的な処理によるスタック消費に注意が必要です。

保守担当者向けメモ

  • 本構造体は実行計画の初期化時と実行時の両方で参照されるため、変更時は PartitionPruneState との整合性に注意してください。
