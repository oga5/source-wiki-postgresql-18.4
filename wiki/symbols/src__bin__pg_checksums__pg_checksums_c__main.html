定義

目的

PostgreSQL データディレクトリに対するページレベルのチェックサム検証、有効化、または無効化処理の実行。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**: コマンドライン引数（argc, argv）。データディレクトリ指定（-D）、操作モード（-c, -d, -e）、進捗表示オプション（-P）等を受け取る。
  • **出力**: 処理結果を標準出力に表示し、成功時は 0、エラー時は 1 または pg_fatal による異常終了を返す。

前提条件と不変条件

  • 対象の PostgreSQL クラスタが停止状態（DB_SHUTDOWNED または DB_SHUTDOWNED_IN_RECOVERY）であること。
  • クラスタのバージョンおよびブロックサイズ（BLCKSZ）が実行バイナリと互換性を持つこと。
  • pg_control ファイルの CRC が整合していること。

アルゴリズムまたは振る舞い

1. **初期化**: ログ設定、ロケール設定を行い、getopt_long で引数を解析する。 2. **検証**: get_controlfile() でクラスタ状態を読み込み、互換性と停止状態を確認する。 3. **モード判定**: 指定されたモード（CHECK, ENABLE, DISABLE）に基づき、現在のチェックサム設定との矛盾がないかチェックする。 4. **走査**:

  • --progress 指定時は、scan_directory() を sizeonly=true で呼び出し、全データサイズを事前計算する。
  • global, base, pg_tblspc の各ディレクトリに対し、scan_directory() を実行してチェックサム処理を行う。

    • 5. **完了報告**: 処理したファイル数、ブロック数、および検出された不正チェックサム数を表示する。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（エントリポイント）。
  • **呼び出し先**: usage(), getopt_long(), get_controlfile(), scan_directory(), progress_report(), pg_fatal() 等。
  • **注記**: ツールによる contrib/amcheck 等からの参照は、本シンボル main とは異なる文脈（同名の別関数）である可能性が高く、直接の呼び出し関係を示す証拠はありません。

関連データ構造

  • ControlFileData (クラスタメタデータ)
  • PgChecksumMode (実行モード)
  • option (コマンドライン引数定義)

失敗モードと境界条件

  • **引数エラー**: 不正なオプション指定やデータディレクトリの欠如時にエラー終了。
  • **互換性エラー**: クラスタのバージョン不一致やブロックサイズ不一致時に pg_fatal または終了。
  • **状態エラー**: クラスタが稼働中の場合、チェックサム不整合を招くため実行を拒否する。
  • **モード矛盾**: チェックサムが既に有効な状態での ENABLE 指定や、無効な状態での DISABLE 指定はエラーとなる。

保守担当者向けメモ

  • 進捗表示（--progress）を有効にするとディレクトリ走査が2回行われるため、大規模クラスタでは処理時間に注意が必要。
  • skip[] 配列で定義された除外ファイル（pg_control 等）の管理は本関数の外部で行われるため、新しいシステムファイル追加時は注意すること。
