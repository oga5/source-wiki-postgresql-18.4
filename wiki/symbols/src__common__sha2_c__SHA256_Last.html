SHA256_Last

定義

  • 種別: function
  • 定義箇所: src/common/sha2.c:528

目的

SHA-256 ハッシュ計算の最終段階において、メッセージのパディング処理とビット長情報の埋め込みを行い、最後の変換処理を実行する。

シグネチャまたは形状

static void SHA256_Last(pg_sha256_ctx *context)

入力と出力

  • 入力: pg_sha256_ctx *context（ハッシュ計算のコンテキスト構造体）
  • 出力: なし（副作用として context 内のバッファおよび内部状態を更新）

前提条件と不変条件

  • context は初期化およびデータ更新済みの有効な状態であること。
  • WORDS_BIGENDIAN が未定義の場合、内部で REVERSE64 マクロによりビットカウントのバイト順序がホスト順序から変換される。

アルゴリズムまたは振る舞い

1. 現在のバッファ使用量（usedspace）を計算する。 2. パディングの開始として、usedspace 位置に 0x80 を付与する。 3. バッファの残り領域をゼロで埋める。

  • 残り領域がビット長情報（8バイト）を保持できない場合は、一度 SHA256_Transform を呼び出してブロックを処理し、バッファをクリアしてから再度パディングを行う。

    • 4. バッファの末尾に 64 ビットのビット長情報を書き込む。 5. 最後に SHA256_Transform を呼び出し、最終的なダイジェストを確定させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/sha2.c 内の pg_sha256_final 等から呼び出される（ソース内の直接的な呼び出し箇所として src/common/sha2.c:582 および 999 が確認されている）。
  • 呼び出し先: SHA256_Transform（ブロック圧縮関数）。

関連データ構造

  • pg_sha256_ctx: ハッシュ計算の状態を保持する構造体。

失敗モードと境界条件

  • usedspace がブロック境界に近い場合、パディングのために追加の SHA256_Transform が実行される。
  • メモリ管理は context 構造体内部のバッファに対して行われるため、外部メモリの不正アクセスリスクは低い。

保守担当者向けメモ

  • 本関数は SHA256_Transform と密接に結合している。
  • エンディアン変換は WORDS_BIGENDIAN の定義に依存するため、移植時には注意が必要である。
