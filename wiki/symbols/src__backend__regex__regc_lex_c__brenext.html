brenext

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_lex.c:860

目的

BRE（Basic Regular Expressions）モードにおける次のトークンを特定し、字句解析を行う。

シグネチャまたは形状

static int brenext(struct vars *v, chr c)

入力と出力

  • **入力**:
  • v: 正規表現コンパイル状態を保持する vars 構造体へのポインタ。
  • c: 現在処理対象の文字。
  • **出力**:
  • 成功時は 1 を返し、v->nexttype および v->nextvalue にトークン情報を格納する。
  • 失敗時は 0 を返す。

前提条件と不変条件

  • 本関数は regc_lex.c 内で定義されており、vars 構造体のポインタ v が有効であること。
  • c が \ の場合、入力文字列の終端（EOS）に達していないこと。

アルゴリズムまたは振る舞い

1. 入力文字 c に基づき分岐する。 2. *, [, ., ^, $ などの特殊文字を判定し、文脈に応じてトークンをセットする。 3. [ の場合、特定のPOSIX文字クラス（[[:<:]] や [[:>:]]）を検出し、特殊トークンとして処理する。 4. \ が入力された場合、後続の文字を読み取り、エスケープシーケンス（\{, \(, \), <（単語開始）, >（単語終了）, 後方参照 \1〜\9）を解析する。 5. 上記以外は PLAIN トークンとして処理する。 6. RETV または RET マクロを使用してトークンタイプを確定し、解析を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regc_lex.c 内の字句解析メインループから呼び出される。
  • **呼び出し先**: NOTE, RETV, RET, INTOCON, FAILW, ATEOS, NEXT1, NEXT2, LASTTYPE, CHR, DIGITVAL, iscalnum などのマクロおよび関数。

関連データ構造

  • struct vars: 正規表現コンパイルのコンテキストを保持する構造体。
  • chr: 正規表現内の文字型。

失敗モードと境界条件

  • 入力文字列が \ で終わっている場合、REG_EESCAPE エラーを発生させる。
  • REG_EXPANDED フラグが有効な場合、$ の処理において skip() を呼び出し、空白やコメントをスキップする。

保守担当者向けメモ

  • 本関数は regcomp.c にインクルードされるため、v は暗黙的に共有される。
  • L_BRACK や L_BBND へのコンテキスト遷移（INTOCON）は、後続の字句解析に影響を与えるため注意が必要。
  • assert(NOTREACHED) に到達しないよう、すべてのエスケープシーケンスのケースが網羅されていることを確認すること。
