g_intbig_penalty

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_intbig_gist.c:292

目的

GiST インデックスへの新しいエントリ挿入時、既存のインデックスノードに対してどの程度「ペナルティ」が発生するかを計算し、最適な挿入位置を決定するためのコストを算出する。

シグネチャまたは形状

Datum g_intbig_penalty(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY *origentry（既存のインデックスエントリ）
  • PG_GETARG_POINTER(1): GISTENTRY *newentry（挿入対象のエントリ）
  • PG_GETARG_POINTER(2): float *penalty（コスト計算結果を格納するポインタ）
  • **出力**:
  • penalty ポインタ経由で計算されたコスト（float）を返す。

前提条件と不変条件

  • origentry は ISSIGNKEY である必要がある。
  • GET_SIGLEN() マクロにより、現在のインデックス設定に基づいたシグネチャ長が取得可能であること。

アルゴリズムまたは振る舞い

1. origentry および newentry から GISTTYPE 形式のキーを抽出する。 2. hemdist 関数を呼び出し、既存のキーと新しいキーの間のハミング距離を計算する。 3. 計算されたハミング距離を penalty に代入し、そのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス管理メカニズム（直接的な呼び出し元コードはソース内に明示されていない）。
  • **呼び出し先**: hemdist（ハミング距離計算）、GET_SIGLEN（シグネチャ長取得）。

関連データ構造

  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。
  • GISTTYPE: intbig 型のインデックスキー表現。

失敗モードと境界条件

  • siglen が正しく設定されていない場合、hemdist の計算結果に影響を及ぼす可能性がある。
  • 入力ポインタが NULL の場合の挙動は、PostgreSQL の標準的な関数呼び出し規約に依存する。

保守担当者向けメモ

  • 本関数は hemdist に依存しているため、コスト計算のロジックを変更する場合は hemdist の実装を確認すること。
  • PG_FUNCTION_INFO_V1(g_intbig_penalty) により、PostgreSQL の関数インターフェースとして登録されている。
