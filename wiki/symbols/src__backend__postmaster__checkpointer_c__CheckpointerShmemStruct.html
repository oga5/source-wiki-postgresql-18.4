CheckpointerShmemStruct

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/checkpointer.c:113

目的

チェックポインタープロセスとバックエンドプロセス間で共有されるメモリ領域を定義し、チェックポイントの実行状態管理およびファイル同期（fsync）要求のキューイングを行うための構造体です。

シグネチャまたは形状

typedef struct {
    pid_t       checkpointer_pid;
    slock_t     ckpt_lck;
    int         ckpt_started;
    int         ckpt_done;
    int         ckpt_failed;
    int         ckpt_flags;
    ConditionVariable start_cv;
    ConditionVariable done_cv;
    int         num_requests;
    int         max_requests;
    CheckpointerRequest requests[FLEXIBLE_ARRAY_MEMBER];
} CheckpointerShmemStruct;

入力と出力

  • **入力**: バックエンドプロセスが ckpt_flags を設定し、requests 配列に fsync 要求を追加します。
  • **出力**: チェックポインタープロセスが ckpt_started、ckpt_done、ckpt_failed を更新し、状態変化を ConditionVariable を通じて待機中のバックエンドに通知します。

前提条件と不変条件

  • ckpt_lck スピンロックが、ckpt_ で始まるすべてのフィールドおよび要求キューの整合性を保護します。
  • requests 配列は FLEXIBLE_ARRAY_MEMBER を使用しており、共有メモリ確保時に動的にサイズが決定されます。

アルゴリズムまたは振る舞い

1. **状態管理**: チェックポイントの開始・完了・失敗をカウンタのインクリメントで追跡します。 2. **同期**: start_cv および done_cv を使用して、チェックポイントの進捗を待機しているバックエンドプロセスを起床させます。 3. **要求集約**: バックエンドから委譲された fsync 要求を requests キューに蓄積し、チェックポインターが定期的に回収・処理します。

呼び出し元と呼び出し先

  • **呼び出し元**: CheckpointerShmemInit（初期化時）、CheckpointerMain（メインループ内）、および RequestCheckpoint や ForwardSyncRequest を呼び出すバックエンドプロセス。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • CheckpointerRequest: requests 配列の各要素として格納される fsync 要求データ。

失敗モードと境界条件

  • **キューの溢れ**: num_requests が max_requests に達した場合、要求の追加が制限される可能性があります。
  • **プロセス未起動**: checkpointer_pid が 0 の場合、チェックポインターが未起動であることを示します。

保守担当者向けメモ

  • CheckpointerShmem ポインタを介してアクセスされる際は、volatile 修飾子を伴うキャストが使用されることが一般的です。
  • 共有メモリサイズを計算する際は offsetof(CheckpointerShmemStruct, requests) を使用してベースサイズを算出してください。
