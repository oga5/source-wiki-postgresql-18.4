decrypt_init

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-decrypt.c:286

目的

復号処理におけるフィルタ初期化を行い、復号ストリームのバッファサイズを要求する。

シグネチャまたは形状

static int decrypt_init(void **priv_p, void *arg, PullFilter *src);

入力と出力

  • **入力**:
  • priv_p: フィルタのプライベートデータを格納するためのポインタ。
  • arg: PGP_CFB 構造体へのポインタ（復号コンテキスト）。
  • src: 読み取り元の PullFilter。
  • **出力**:
  • 戻り値: 復号処理に必要なバッファサイズとして 4096 を返す。
  • priv_p: arg で渡された PGP_CFB コンテキストが設定される。

前提条件と不変条件

  • arg は有効な PGP_CFB 構造体を指している必要がある。
  • 呼び出し側は、本関数が返すバッファサイズ（4096バイト）を確保する責任を負う。

アルゴリズムまたは振る舞い

1. 引数 arg（PGP_CFB コンテキスト）を priv_p に代入し、フィルタのプライベート状態として保持する。 2. 復号処理に必要なバッファサイズとして 4096 を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-decrypt.c 内のフィルタ構築処理から参照されている。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • PGP_CFB: CFB モードの復号状態を保持する構造体。
  • PullFilter: データを逐次処理するための抽象化された読み取りフィルタ構造体。

失敗モードと境界条件

  • 提供されたソースコード上では、エラーを返すパスは定義されていない。

保守担当者向けメモ

  • 本関数は PullFilter の初期化コールバックとして機能する。
  • 戻り値の 4096 は固定値であり、将来的にバッファサイズの動的調整が必要な場合はこの箇所を修正すること。
