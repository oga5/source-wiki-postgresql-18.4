RelationGetPartitionKey

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/partcache.c:50

目的

指定されたリレーション（Relation）に関連付けられたパーティションキー情報を取得します。

シグネチャまたは形状

PartitionKey RelationGetPartitionKey(Relation rel)

入力と出力

  • **入力**: rel (対象のリレーション構造体へのポインタ)
  • **出力**: PartitionKey (パーティションキー構造体へのポインタ、または NULL)

前提条件と不変条件

  • パーティションキーは一度作成されると変更されない前提で設計されています。
  • リレーションがパーティションテーブル（RELKIND_PARTITIONED_TABLE）でない場合、NULL を返します。

アルゴリズムまたは振る舞い

1. rel->rd_rel->relkind を確認し、パーティションテーブルでなければ即座に NULL を返します。 2. リレーションのキャッシュ内にパーティションキー（rd_partkey）が存在するか確認します。 3. rd_partkey が NULL の場合、RelationBuildPartitionKey(rel) を呼び出してキャッシュを構築します。 4. 最終的に rel->rd_partkey を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/partition.c
  • src/backend/commands/indexcmds.c
  • src/backend/executor/execPartition.c
  • src/backend/optimizer/util/plancat.c
  • src/backend/parser/parse_utilcmd.c
  • src/backend/partitioning/partbounds.c
  • **呼び出し先**:
  • RelationBuildPartitionKey (内部で条件付き呼び出し)

関連データ構造

  • Relation: リレーションキャッシュ構造体。
  • PartitionKey: パーティション戦略、キー列、演算子クラス等を保持する構造体。

失敗モードと境界条件

  • パーティションテーブル以外のリレーションが渡された場合、エラーにはならず NULL を返します。
  • rd_partkey が未構築の場合、RelationBuildPartitionKey を通じてシステムカタログへのアクセスが発生します。

保守担当者向けメモ

  • 本関数は relcache の一部として動作します。メモリ管理は CacheMemoryContext に依存しており、リレーションの破棄時に適切に解放される設計です。
