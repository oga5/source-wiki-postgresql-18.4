InitControlFile

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlog.c:4199

目的

データベースの制御ファイル（pg_control）のメモリ内イメージである ControlFile を初期化し、システム識別子や運用パラメータを設定する。

シグネチャまたは形状

static void InitControlFile(uint64 sysidentifier, uint32 data_checksum_version)

入力と出力

  • **入力**:
  • sysidentifier (uint64): データベースクラスタを一意に識別するシステム識別子。
  • data_checksum_version (uint32): データチェックサムのバージョン。
  • **出力**: なし（ControlFile 構造体への副作用のみ）。

前提条件と不変条件

  • ControlFile が指すメモリ領域が有効であること。
  • pg_strong_random が成功し、認証用のモックノンスが生成可能であること。

アルゴリズムまたは振る舞い

1. pg_strong_random を使用して認証用のランダムなモックノンスを生成する。生成に失敗した場合は PANIC で終了する。 2. ControlFile 構造体全体をゼロクリアする。 3. sysidentifier を設定する。 4. 生成したモックノンスを ControlFile->mock_authentication_nonce にコピーする。 5. データベースの状態を DB_SHUTDOWNED に設定し、unloggedLSN を FirstNormalUnloggedLSN で初期化する。 6. 現在のグローバル設定値（MaxConnections, max_worker_processes, max_wal_senders, max_prepared_xacts, max_locks_per_xact, wal_level, wal_log_hints, track_commit_timestamp）を ControlFile に書き込む。 7. data_checksum_version を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c:5213 (初期化プロセスの一部として呼び出される)
  • **呼び出し先**:
  • pg_strong_random
  • memset
  • memcpy
  • ereport

関連データ構造

  • ControlFileData: pg_control ファイルの構造体。
  • ControlFile: ControlFileData 型のグローバルなメモリ内イメージ。

失敗モードと境界条件

  • pg_strong_random が乱数を生成できない場合、PANIC レベルのエラーが発生し、システムは即座に停止する。

保守担当者向けメモ

  • 本関数はデータベース初期化時（initdb 相当の処理）にのみ使用される。
  • ControlFile に追加のパラメータを保存する必要が生じた場合は、本関数内で適切に値を代入すること。
