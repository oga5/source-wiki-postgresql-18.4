ResetPlanCache

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/plancache.c:2299

目的

キャッシュされているすべての実行計画（CachedPlanSource）および式（CachedExpression）を無効化する。

シグネチャまたは形状

void ResetPlanCache(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルなキャッシュリストの状態を更新）

前提条件と不変条件

  • saved_plan_list および cached_expression_list に格納されている各要素は、それぞれ CACHEDPLANSOURCE_MAGIC または CACHEDEXPR_MAGIC で検証可能であること。

アルゴリズムまたは振る舞い

1. saved_plan_list を走査し、各 CachedPlanSource に対して以下を実行する:

  • すでに無効な場合はスキップする。
  • StmtPlanRequiresRevalidation を呼び出し、再検証が必要なステートメントか判定する。不要な場合はスキップする（トランザクション制御文など、再解析しても結果が変わらないものを保護するため）。
  • is_valid フラグを false に設定し、関連する汎用計画（gplan）が存在する場合は同様に無効化する。

    • 2. cached_expression_list を走査し、各 CachedExpression の is_valid フラグを false に設定する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/commands/discard.c (DISCARD ALL 実行時)
  • src/backend/commands/trigger.c
  • src/backend/utils/cache/plancache.c (内部的なキャッシュリセット処理)
  • 呼び出し先:
  • StmtPlanRequiresRevalidation (判定ロジック)
  • dlist 操作関数群 (dlist_foreach, dlist_container)

関連データ構造

  • CachedPlanSource: キャッシュされたクエリソース。
  • CachedExpression: キャッシュされた式。
  • saved_plan_list: キャッシュされた計画ソースのリスト。
  • cached_expression_list: キャッシュされた式のリスト。

失敗モードと境界条件

  • トランザクション制御文（ROLLBACK 等）は、異常終了時の実行を保証するため、本関数による無効化の対象から除外される。
  • 既に無効化されている要素に対しては、何ら操作を行わず安全にスキップする。

保守担当者向けメモ

  • 本関数はキャッシュの強制リセットを行うため、呼び出し時にはシステム全体の計画キャッシュが再構築されるコストが発生することに留意すること。
  • StmtPlanRequiresRevalidation の判定ロジックを変更する場合は、本関数による無効化対象が意図通りか確認が必要である。
