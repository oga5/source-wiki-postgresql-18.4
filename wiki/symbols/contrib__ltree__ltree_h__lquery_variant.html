lquery_variant

定義

  • 種別: struct
  • 定義箇所: contrib/ltree/ltree.h:58

目的

lquery 内で OR 条件として並ぶ複数の代替パスのうち、個々のラベル条件（バリアント）を表現するために使用されます。

シグネチャまたは形状

typedef struct {
    int32   val;    /* ラベル文字列の CRC */
    uint16  len;    /* ラベル文字列のバイト長 */
    uint8   flag;   /* LVAR_xxx フラグ */
    char    name[FLEXIBLE_ARRAY_MEMBER];
} lquery_variant;

入力と出力

  • 入力: lquery 構造体の一部として格納され、クエリ実行時にパスとの照合に使用されます。
  • 出力: 照合結果に基づき、マッチングの成否を判定するロジックへ情報を提供します。

前提条件と不変条件

  • name フィールドは可変長配列として定義されているため、構造体のサイズ計算には LVAR_HDRSIZE マクロを使用する必要があります。
  • name の後続データは MAXALIGN で整列されている必要があります。

アルゴリズムまたは振る舞い

  • val フィールドにはラベル文字列の CRC 値が格納され、高速な比較に使用されます。
  • flag フィールドにより、ワイルドカードや特定の検索条件（LVAR_ANYEND 等）を制御します。
  • LVAR_NEXT マクロを用いることで、メモリ上の次の lquery_variant へ安全にポインタを進めることができます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • contrib/ltree/_ltree_gist.c
  • contrib/ltree/lquery_op.c
  • contrib/ltree/ltree_gist.c
  • contrib/ltree/ltree_io.c
  • 呼び出し先:
  • 提供されたソース excerpt には、本構造体が呼び出す関数や外部シンボルの情報は含まれていません。

関連データ構造

  • lquery: 本構造体を保持する上位構造体。
  • lquery_level: lquery を構成するレベル構造。

失敗モードと境界条件

  • sizeof を直接使用して構造体サイズを計算すると、FLEXIBLE_ARRAY_MEMBER の影響でアライメント違反が発生します。
  • LVAR_NEXT マクロを使用せずにポインタ演算を行うと、メモリ上の配置不整合により不正なメモリ参照を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体はディスクフォーマットの一部であるため、フィールドの順序やサイズを変更する際は後方互換性に十分注意してください。
  • LVAR_HDRSIZE および LVAR_NEXT マクロの定義が、構造体のメモリレイアウトと一致していることを常に確認してください。
