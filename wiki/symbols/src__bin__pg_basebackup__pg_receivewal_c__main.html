目的

pg_receivewal クライアントツールのエントリポイントであり、コマンドライン引数の解析、設定の検証、および WAL ストリーミング処理の初期化を行う。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。接続先情報、ディレクトリパス、レプリケーションスロット設定、圧縮オプション等を受け取る。
  • **出力**:
  • 正常終了時は 0 を返す。
  • 引数エラーや実行時エラーが発生した場合は、エラーメッセージを出力し 1 を返す。

前提条件と不変条件

  • pg_logging_init および set_pglocale_pgservice が呼び出され、ロギングとロケールが初期化されていること。
  • --create-slot と --drop-slot は同時に指定できない。
  • --synchronous と --no-sync は同時に指定できない。
  • スロット操作（作成・削除）を行う場合は、--slot オプションが必須である。

アルゴリズムまたは振る舞い

1. **初期化**: プログラム名とロケールの設定を行う。 2. **引数解析**: getopt_long を使用し、接続文字列、ディレクトリ、終了位置（LSN）、圧縮設定、スロット管理フラグなどを解析する。 3. **検証**: 互いに排他的なオプションの組み合わせや、必須引数の欠落をチェックする。 4. **圧縮設定**: 指定された圧縮アルゴリズムを解析し、内部状態を更新する。 5. **終了処理**: 引数解析後にエラーがある場合は pg_log_error を通じて通知し終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: OS（プログラム実行時）。
  • **呼び出し先**:
  • getopt_long: 引数解析。
  • usage: ヘルプ表示。
  • pg_fatal, pg_log_error: エラー報告。
  • parse_compress_options, parse_compress_algorithm: 圧縮設定の解析。
  • （注: ソース抜粋以降の処理として、FindStreamingStart や StreamLog 等が続くことが想定されるが、本抜粋内では明示的な呼び出しは確認できない。）

関連データ構造

  • struct option long_options[]: コマンドライン引数の定義。
  • pg_compress_specification: 圧縮設定を保持する構造体。

失敗モードと境界条件

  • **引数エラー**: 不正なオプション、必須引数の欠落、排他オプションの同時指定時にエラーメッセージを出力して終了する。
  • **解析エラー**: endpos の形式が XX/XX に適合しない場合、または不明な圧縮アルゴリズムが指定された場合に pg_fatal を呼び出し終了する。

保守担当者向けメモ

  • 本関数は引数解析の大部分を担っている。新しいコマンドラインオプションを追加する場合は long_options と switch 文を更新すること。
  • 抜粋以降のコードで、解析された設定値を用いて実際にストリーミングを開始する処理（StreamLog 等）が続くため、設定値の整合性チェックは本関数内で完結させる必要がある。
