gin_tsquery_triconsistent

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/tsginidx.c:262

目的

GINインデックスの検索において、クエリ条件（tsquery）がインデックス内のキー状態と合致するかを、3値論理（真・偽・不明）を用いて判定します。

シグネチャまたは形状

Datum gin_tsquery_triconsistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GinTernaryValue *check（各クエリオペランドの状態配列）
  • PG_GETARG_TSQUERY(2): TSQuery query（検索対象のクエリ）
  • PG_GETARG_POINTER(4): Pointer *extra_data（オペランドとインデックスキーのマップ情報）
  • **出力**:
  • PG_RETURN_GIN_TERNARY_VALUE(res): GinTernaryValue 型の判定結果（GIN_TRUE, GIN_FALSE, GIN_MAYBE）

前提条件と不変条件

  • query->size > 0 の場合にのみ判定処理が実行されます。
  • extra_data[0] には、クエリオペランドとインデックスキーを対応付けるためのマップ情報が格納されている必要があります。

アルゴリズムまたは振る舞い

1. GinChkVal 構造体を初期化し、クエリのアイテム配列、チェック状態配列、およびマップ情報を設定します。 2. TS_execute_ternary 関数を呼び出し、checkcondition_gin をコールバックとして使用してクエリの論理評価を行います。 3. 評価には TS_EXEC_PHRASE_NO_POS フラグが渡され、位置情報を考慮しない論理判定が行われます。 4. 評価結果（GinTernaryValue）を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: GINインデックスの検索エンジン（直接的な呼び出し元情報はソースコード内に明示されていません）。
  • **呼び出し先**:
  • TS_execute_ternary
  • checkcondition_gin

関連データ構造

  • GinTernaryValue: GINインデックスの3値論理状態。
  • TSQuery: 全文検索クエリの内部表現。
  • GinChkVal: 整合性チェック時に使用されるコンテキスト構造体。

失敗モードと境界条件

  • query->size が 0 の場合、判定処理はスキップされ、初期値である GIN_FALSE が返されます。

保守担当者向けメモ

  • 本関数は TS_execute_ternary を利用して論理評価を行うため、クエリの論理構造に変更がある場合は checkcondition_gin のロジックと併せて確認してください。
  • extra_data のインデックスアクセス（extra_data[0]）は、gin_extract_tsquery が生成するデータ構造と密接に結合しています。
