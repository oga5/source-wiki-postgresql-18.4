MergeJoinClauseData

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeMergejoin.c:120

目的

マージ結合において、結合条件となる左右の入力式を評価し、その結果を比較するために必要な情報を保持する。

シグネチャまたは形状

typedef struct MergeJoinClauseData
{
    ExprState  *lexpr;          /* 左側（outer）入力式 */
    ExprState  *rexpr;          /* 右側（inner）入力式 */
    Datum       ldatum;         /* 現在の左側値 */
    Datum       rdatum;         /* 現在の右側値 */
    bool        lisnull;        /* 左側値のNULLフラグ */
    bool        risnull;        /* 右側値のNULLフラグ */
    SortSupportData ssup;       /* 比較に必要なソートサポートデータ */
} MergeJoinClauseData;

入力と出力

  • **入力**: 実行時に ExprState を評価して得られる Datum 型の結合キー値およびそのNULL状態。
  • **出力**: 保持された SortSupportData を用いて、MJCompare 等の関数が左右の値を比較し、結合の進行方向を決定する。

前提条件と不変条件

  • 結合対象となる入力リレーションは、本構造体に含まれる ssup（ソートサポートデータ）が規定する順序でソート済みである必要がある。
  • lexpr および rexpr は、結合実行前に初期化されている必要がある。

アルゴリズムまたは振る舞い

  • 結合処理の各ステップにおいて、lexpr と rexpr を評価し、その結果を ldatum/rdatum に格納する。
  • ssup を介して比較ロジックを呼び出し、タプルのスキップや結合の判定を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeMergejoin.c 内で、結合句の配列としてメモリ確保（palloc0）が行われる。
  • **呼び出し先**: 比較ロジックにおいて SortSupportData の関数ポインタが利用される。
  • ※具体的な関数呼び出しの連鎖については、提供されたソース情報からは詳細な追跡が困難です。

関連データ構造

  • ExprState: 結合条件式の評価状態を保持。
  • SortSupportData: 値の比較ルール（照合順序や型固有の比較関数）を保持。
  • MergeJoinClause: MergeJoinClauseData へのポインタ型（src/include/nodes/execnodes.h で定義）。

失敗モードと境界条件

  • NULL値の扱い: マージ結合においてNULLは「一致しない」とみなされるため、lisnull または risnull が真の場合の比較ロジックに依存する。
  • 結合条件が定数でない場合、右外部結合や完全外部結合ではサポートされない制約がある。

保守担当者向けメモ

  • 本構造体は nodeMergejoin.c のメインループにおける比較の基盤となる。デバッグ時には MJ_dump マクロを使用して、保持されている ldatum/rdatum の状態を確認することが推奨される。
