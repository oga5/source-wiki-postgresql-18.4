GistEntryVector

定義

  • 種別: struct
  • 定義箇所: src/include/access/gist.h:236

目的

GiSTインデックスの構築や分割処理において、複数の GISTENTRY を一括して保持・管理するためのコンテナ構造体です。

シグネチャまたは形状

typedef struct
{
    int32       n;              /* 要素数 */
    GISTENTRY   vector[FLEXIBLE_ARRAY_MEMBER];
} GistEntryVector;

入力と出力

  • **入力**: GiSTインデックスの PickSplit メソッド等において、処理対象となるエントリの配列として渡されます。
  • **出力**: 構造体内の vector 配列を通じて、各エントリのキー値、ページ番号、オフセット番号にアクセス可能です。

前提条件と不変条件

  • n は vector 配列に格納されている有効な GISTENTRY の数を示します。
  • vector は可変長配列（FLEXIBLE_ARRAY_MEMBER）として定義されており、メモリ割り当て時には n に応じた適切なサイズが確保されている必要があります。

アルゴリズムまたは振る舞い

GiSTインデックスのページ分割（Split）処理において、既存のタプル群を左右のページへ振り分けるための入力データとして機能します。各エントリは GISTENTRY 型であり、インデックスのキー情報と物理的な位置情報を保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist 配下の各モジュール（btree_bit.c, btree_bool.c, btree_bytea.c, btree_cash.c, btree_date.c, btree_enum.c 等）において、PG_GETARG_POINTER(0) を介して取得される形で頻繁に使用されています。
  • **呼び出し先**: gbt_num_picksplit などの GiST サポート関数へ引数として渡されるケースが確認されています。

関連データ構造

  • GISTENTRY: 本構造体の要素として格納される、個別のインデックスエントリを表現する構造体。
  • GIST_SPLITVEC: ページ分割の結果を格納する構造体。

失敗モードと境界条件

  • n が 0 の場合、空のベクトルとして扱われます。
  • 可変長配列を使用しているため、メモリ確保時に n の値と実際のメモリサイズが不整合を起こすと、バッファオーバーランのリスクがあります。

保守担当者向けメモ

  • 本構造体は contrib モジュール内の GiST 実装で広く利用されています。変更を加える際は、これら外部モジュールへの影響を考慮してください。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズ計算には sizeof 演算子だけでなく、n を考慮した動的な計算が必要です。
