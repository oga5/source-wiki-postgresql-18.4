tsmatchsel

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_selfuncs.c:66

目的

PostgreSQLのクエリプランナに対し、tsvector @@ tsquery 演算子を用いた検索条件の選択率（Selectivity）を推定します。

シグネチャまたは形状

Datum tsmatchsel(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): PlannerInfo *root（プランナ情報）
  • PG_GETARG_POINTER(2): List *args（演算子の引数リスト）
  • PG_GETARG_INT32(3): int varRelid（リレーションID）
  • **出力**:
  • float8: 推定された選択率（0.0 から 1.0 の範囲）。

前提条件と不変条件

  • 入力引数が Var と Const の比較形式である必要があります。
  • Const 側の値が TSQUERYOID 型であり、かつ Var 側の型が TSVECTOROID である場合にのみ詳細な計算が行われます。
  • 統計情報が不足している場合や型が適合しない場合は、デフォルト値 DEFAULT_TS_MATCH_SEL (0.005) が返されます。

アルゴリズムまたは振る舞い

1. get_restriction_variable を使用して、式が Var と Const の比較であることを確認します。 2. Const が NULL の場合、選択率 0.0 を返します。 3. Const が TSQUERYOID 型かつ Var が TSVECTOROID 型であることを検証します。 4. 条件を満たす場合、tsquerysel を呼び出して統計情報に基づいた選択率を算出します。 5. 算出された選択率に対し CLAMP_PROBABILITY マクロを適用し、範囲を 0.0〜1.0 に制限して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: プランナ（詳細な呼び出し元情報はソースコード上では明示されていません）。
  • **呼び出し先**: get_restriction_variable, tsquerysel, ReleaseVariableStats。

関連データ構造

  • VariableStatData: 統計情報を保持する構造体。
  • PlannerInfo: クエリプランナの情報を保持する構造体。
  • TSQUERYOID, TSVECTOROID: 型識別子。

失敗モードと境界条件

  • **統計情報不足**: get_restriction_variable が失敗した場合や、型が一致しない場合はデフォルト値 0.005 を返します。
  • **NULL定数**: Const が NULL の場合、選択率 0.0 を返します。
  • **型不一致**: tsvector 以外の列に対する演算子として誤って使用された場合、デフォルト値 0.005 を返します。

保守担当者向けメモ

  • 本関数は @@ 演算子の選択率推定の入り口です。詳細な確率計算は tsquerysel に委譲されています。
  • 統計情報が MCELEM（最頻要素）として格納されている前提で動作するため、統計情報の収集ロジック（ts_typanalyze.c）に変更があった場合は注意が必要です。
