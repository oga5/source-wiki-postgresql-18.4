pgstat_create_transactional

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_xact.c:360

目的

統計オブジェクトの作成をトランザクション管理下で行い、トランザクションのコミットまたはアボートと統計情報のライフサイクルを同期させる。

シグネチャまたは形状

void pgstat_create_transactional(PgStat_Kind kind, Oid dboid, uint64 objid)

入力と出力

  • **入力**:
  • kind: 統計対象の種別（PgStat_Kind）。
  • dboid: データベースのOID。
  • objid: 対象オブジェクトの識別子。
  • **出力**: なし（void）。

前提条件と不変条件

  • 呼び出し時に TopTransactionContext が有効である必要がある。
  • 同一の統計エントリが既に存在する場合、警告を発行した上で pgstat_reset を実行し、既存の統計をリセットする。

アルゴリズムまたは振る舞い

1. pgstat_get_entry_ref を呼び出し、指定された統計エントリが既に存在するか確認する。 2. 存在する場合は ereport(WARNING, ...) で警告をログし、pgstat_reset を実行して統計を初期化する。 3. create_drop_transactional_internal を呼び出し、create フラグを true に設定して、現在のトランザクションスタックに統計作成の予約を追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/pgstat_function.c
  • src/backend/utils/activity/pgstat_relation.c
  • src/backend/utils/activity/pgstat_subscription.c
  • **呼び出し先**:
  • pgstat_get_entry_ref
  • pgstat_get_kind_info
  • pgstat_reset
  • create_drop_transactional_internal

関連データ構造

  • PgStat_Kind: 統計の種類を定義する列挙型。
  • PgStat_PendingDroppedStatsItem: トランザクション内で作成・削除を追跡するための内部構造体。

失敗モードと境界条件

  • 統計エントリが既に存在する場合、警告が発生するが処理は継続される。
  • メモリ割り当て失敗時は TopTransactionContext のメモリ管理ルールに従い、エラーがスローされる可能性がある。

保守担当者向けメモ

  • 本関数は統計の「作成」をトランザクションに登録する。トランザクションがアボートされた場合、この作成処理は AtEOXact_PgStat を通じて取り消される。
  • 2相コミット（PREPARE）時には AtPrepare_PgStat が関与するため、通常の終了処理とは異なるパスを辿る点に注意すること。
