目的

ECPG（Embedded SQL for C）の Informix 互換ライブラリが提供する rfmtlong 関数の動作を検証するための回帰テスト用エントリーポイントです。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（引数を受け取らない）
  • 出力: 正常終了時に 0 を返します。

前提条件と不変条件

  • ECPG のデバッグ出力が有効化されていること（ECPGdebug(1, stderr)）。
  • テスト対象の rfmtlong 関数が利用可能であること。

アルゴリズムまたは振る舞い

1. ECPGdebug(1, stderr) を呼び出し、デバッグログを標準エラー出力に設定します。 2. 固定値 -8494493 を対象として、多様なフォーマット文字列（#, $, *, +, &, < 等の修飾子を含む）を引数に fmtlong() を繰り返し呼び出します。 3. fmtlong() 内で rfmtlong() が実行され、変換結果が標準出力へ書き出されます。 4. 全てのテストケース実行後、0 を返して終了します。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムの実行時）
  • 呼び出し先:
  • ECPGdebug: ECPG のデバッグ設定関数。
  • fmtlong: rfmtlong を呼び出し、結果を整形して出力するラッパー関数。

関連データ構造

  • sqlca_t: ECPG のエラーハンドリングに使用される構造体（sqlca.h 経由）。

失敗モードと境界条件

  • rfmtlong の変換処理が失敗した場合、fmtlong 内部で check_return が呼び出され、エラーコードが標準出力へ出力されます。

保守担当者向けメモ

  • 本関数はテストの期待値生成用コードです。ロジックの変更は src/interfaces/ecpg/pgtypeslib/ 側の実装に影響を与えるため、テスト結果の差異を確認する際は注意してください。
  • 呼び出し元および呼び出し先の情報は、提供されたソースコードの範囲内でのみ記述しています。
