BufferManagerShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/buf_init.c:67

目的

PostgreSQLの起動時に、共有メモリ上のバッファ管理用データ構造（バッファ記述子、データブロック、I/O待機用変数、チェックポイント用配列）を確保および初期化します。

シグネチャまたは形状

void BufferManagerShmemInit(void)

入力と出力

  • 入力: なし（グローバル変数 NBuffers および BLCKSZ を参照）
  • 出力: なし（共有メモリ領域の確保およびグローバルポインタ変数への割り当てを行う）

前提条件と不変条件

  • ShmemInitStruct を通じて共有メモリを確保する。
  • 共有メモリが既に初期化されている場合（EXEC_BACKEND環境等）、再初期化は行われない。
  • 全てのバッファ記述子、ブロック、CV、チェックポイント用配列は、一括して確保または一括して再利用される必要がある。

アルゴリズムまたは振る舞い

1. **メモリ確保**: ShmemInitStruct を呼び出し、以下の領域を共有メモリ上に確保する。

  • BufferDescriptors: バッファ記述子配列（キャッシュライン境界にアライメント）。
  • BufferBlocks: データブロック領域（PG_IO_ALIGN_SIZE にアライメント）。
  • BufferIOCVArray: I/O待機用条件変数配列（キャッシュライン境界にアライメント）。
  • CkptBufferIds: チェックポイント用ソート配列。

    • 2. **初期化**: 共有メモリが新規作成された場合のみ、以下の処理を実行する。

  • 全バッファ記述子をループで走査し、タグのクリア、状態の初期化、ロックおよび条件変数の初期化を行う。
  • 各バッファを freeNext ポインタで連結し、空きリストを構築する。
  • StrategyInitialize を呼び出し、バッファ置換戦略を初期化する。
  • WritebackContextInit を呼び出し、バックエンドごとのフラッシュコンテキストを初期化する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c（システム起動時の共有メモリ初期化プロセス）
  • 呼び出し先: ShmemInitStruct, GetBufferDescriptor, ClearBufferTag, pg_atomic_init_u32, pgaio_wref_clear, LWLockInitialize, ConditionVariableInit, StrategyInitialize, WritebackContextInit

関連データ構造

  • BufferDesc: バッファのメタデータを保持する構造体。
  • BufferDescPadded: キャッシュライン境界に合わせたパディング付き記述子。
  • CkptSortItem: チェックポイント時に使用されるソート用構造体。

失敗モードと境界条件

  • 共有メモリ確保時に一部のみが成功し、他が失敗する状況は Assert により不正として扱われる。
  • NBuffers が0の場合の挙動については、本ソースコード内では明示的なガードは記述されていない。

保守担当者向けメモ

  • 本関数はPostmaster起動時に一度だけ実行されることを前提としている。
  • EXEC_BACKEND 環境では、子プロセスが共有メモリにアタッチする際に呼び出されるが、初期化ロジックは found フラグによりスキップされる。
  • メモリレイアウト（アライメント）の変更は、性能に直結するため慎重に行うこと。
