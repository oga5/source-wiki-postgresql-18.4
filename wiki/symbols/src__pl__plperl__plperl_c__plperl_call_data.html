plperl_call_data

定義

  • 種別: struct
  • 定義箇所: src/pl/plperl/plperl.c:173

目的

PL/Perl 関数の実行中におけるコールスタック状態およびコンテキスト情報を保持し、PostgreSQL の実行エンジンと Perl インタプリタ間の橋渡しを行うための構造体です。

シグネチャまたは形状

typedef struct plperl_call_data
{
    plperl_proc_desc *prodesc;
    FunctionCallInfo fcinfo;
    Tuplestorestate *tuple_store;
    TupleDesc    ret_tdesc;
    Oid          cdomain_oid;
    void        *cdomain_info;
    MemoryContext tmp_cxt;
} plperl_call_data;

入力と出力

  • **入力**: 関数実行時に prodesc（関数メタデータ）および fcinfo（PostgreSQL の関数呼び出し情報）が設定されます。
  • **出力**: セットを返す関数（SRF）の場合、tuple_store を介して結果セットが蓄積・出力されます。

前提条件と不変条件

  • cdomain_oid は、複合型上のドメインを返す場合を除き 0 に初期化されます。
  • セットを返す関数では、tmp_cxt が行ごとの変換処理におけるメモリ管理に使用されます。

アルゴリズムまたは振る舞い

  • 関数実行の開始時に current_call_data としてスタック上に配置され、実行終了時に復元されます。
  • セットを返す関数（SRF）の実行中、tmp_cxt は行出力ごとに MemoryContextReset され、メモリリークを防止します。

呼び出し元と呼び出し先

  • **呼び出し元**: plperl_call_handler 等のハンドラ関数内で、実行コンテキストを保持するためにインスタンス化されます（src/pl/plperl/plperl.c:1859, 1904）。
  • **呼び出し先**: 直接的な関数呼び出し先はありませんが、内部の prodesc や fcinfo はデータ変換関数や SPI 実行関数によって参照されます。

関連データ構造

  • plperl_proc_desc: コンパイルされた関数のメタデータ。
  • FunctionCallInfo: PostgreSQL 側の関数呼び出し引数情報。
  • Tuplestorestate: セットを返す関数における結果保持用構造体。
  • MemoryContext: メモリ管理用コンテキスト。

失敗モードと境界条件

  • tmp_cxt のリセットを怠ると、大量の行を返す関数においてメモリ消費が肥大化する可能性があります。
  • 呼び出し元で current_call_data の保存・復元が正しく行われない場合、再帰的な関数呼び出しやエラーハンドリング時にコンテキストが破壊されるリスクがあります。

保守担当者向けメモ

  • 本構造体は current_call_data という静的変数を通じてグローバルに追跡されています。
  • セットを返す関数（SRF）の処理ロジックを変更する際は、tmp_cxt のライフサイクルと tuple_store への書き込みタイミングに注意してください。
