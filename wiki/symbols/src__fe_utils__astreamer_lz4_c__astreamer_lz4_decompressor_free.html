astreamer_lz4_decompressor_free

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_lz4.c:412

目的

LZ4伸長ストリーマーに関連するリソース（コンテキスト、バッファ、ストリーマー構造体自体）を解放し、メモリリークを防ぐ。

シグネチャまたは形状

static void astreamer_lz4_decompressor_free(astreamer *streamer)

入力と出力

  • **入力**: astreamer *streamer - 解放対象のストリーマーインスタンス。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 streamer は astreamer_lz4_frame 構造体としてキャスト可能であること。
  • streamer->bbs_next が有効なストリーマーを指しているか、NULLであること。

アルゴリズムまたは振る舞い

1. 引数の streamer を astreamer_lz4_frame 型にキャストする。 2. 後続のストリーマー（streamer->bbs_next）に対して astreamer_free を呼び出し、パイプラインの残りを解放する。 3. LZ4F_freeDecompressionContext を呼び出し、LZ4伸長コンテキスト（dctx）を破棄する。 4. pfree を使用して、内部バッファ（bbs_buffer.data）を解放する。 5. 最後に pfree を使用して、ストリーマー構造体自体を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: 直接的な呼び出し元はソースコード上では明示されていませんが、astreamer フレームワークの終了処理プロセスから呼び出されることが想定されます。
  • **呼び出し先**:
  • astreamer_free (後続ストリーマーの解放)
  • LZ4F_freeDecompressionContext (LZ4ライブラリ関数)
  • pfree (メモリ解放)

関連データ構造

  • astreamer_lz4_frame: 伸長処理の内部状態を保持する構造体。
  • astreamer: ストリーマーの基底構造体。

失敗モードと境界条件

  • streamer が NULL の場合、ポインタ参照エラーが発生する可能性がある（本関数内での明示的なNULLチェックは存在しない）。

保守担当者向けメモ

  • 本関数は USE_LZ4 が定義されている環境でのみ有効です。
  • pfree を使用しているため、PostgreSQLのメモリコンテキスト内で実行される必要があります。
