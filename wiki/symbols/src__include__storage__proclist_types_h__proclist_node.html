proclist_node

定義

  • 種別: struct
  • 定義箇所: src/include/storage/proclist_types.h:28

目的

PostgreSQLのプロセス管理において、プロセス番号（ProcNumber）を用いた双方向連結リストの各要素を構成するためのリンク情報を提供します。

シグネチャまたは形状

typedef struct proclist_node
{
    ProcNumber  next; /* 次のPGPROCのpgprocno */
    ProcNumber  prev; /* 前のPGPROCのpgprocno */
} proclist_node;

入力と出力

  • 入力: 本構造体は PGPROC 構造体の一部として埋め込まれます。
  • 出力: next および prev メンバには、リスト内の隣接するプロセスのインデックス（ProcNumber）が格納されます。

前提条件と不変条件

  • リストに含まれていないノードは、next == prev == 0 である必要があります。
  • リスト内での循環参照は許可されていません。
  • 共有メモリ内での配置を考慮し、ポインタではなく ProcNumber（整数インデックス）を使用します。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のみを行い、リストの挿入・削除・走査といった操作は src/include/storage/proclist.h で定義されたインライン関数やマクロによって実行されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/storage/proc.h 内の PGPROC 構造体において、lwWaitLink および cvWaitLink として利用されています。
  • 呼び出し先: src/include/storage/proclist.h 内の関数（proclist_node_get 等）が、本構造体へのポインタを取得・操作するために参照しています。

関連データ構造

  • proclist_head: リストの先頭と末尾を管理する構造体。
  • PGPROC: 本ノードが埋め込まれるプロセス管理用構造体。
  • ProcNumber: storage/procnumber.h で定義されるプロセス識別用インデックス。

失敗モードと境界条件

  • 空リストは INVALID_PROC_NUMBER で表現されます。
  • リストの整合性を維持するため、構造体のメンバを直接操作せず、必ず proclist.h の API を経由する必要があります。

保守担当者向けメモ

  • 本構造体は型定義のみを提供します。リスト操作のロジックは proclist.h を参照してください。
  • 複数のリストに同一の proclist_node を同時に含めることはできません。
