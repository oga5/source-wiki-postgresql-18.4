SyncScanShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/syncscan.c:125

目的

同期スキャン機能のために共有メモリ上で必要となる合計サイズを算出する。

シグネチャまたは形状

Size SyncScanShmemSize(void)

入力と出力

  • 入力: なし
  • 出力: Size 型（必要な共有メモリのバイト数）

前提条件と不変条件

  • SYNC_SCAN_NELEM 定数に基づいたサイズ計算が行われる。

アルゴリズムまたは振る舞い

SizeOfScanLocations(SYNC_SCAN_NELEM) を呼び出し、同期スキャン用のデータ構造（ss_scan_locations_t）を格納するために必要なメモリサイズを返却する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:146（共有メモリ初期化プロセスの一部として呼び出される）
  • 呼び出し先: SizeOfScanLocations（マクロまたはインライン関数と推測されるが、本ソース excerpt に定義は含まれない）

関連データ構造

  • ss_scan_locations_t: 同期スキャンの位置情報を保持するメイン構造体。

失敗モードと境界条件

  • 特になし。計算は定数 SYNC_SCAN_NELEM に依存する。

保守担当者向けメモ

  • 本関数は PostgreSQL の共有メモリ初期化シーケンスにおいて、必要な領域を確保するために不可欠である。
  • SYNC_SCAN_NELEM の値を変更する場合は、本関数が返すサイズも自動的に調整される。
