cmpEntryAccumulator

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginbulk.c:71

目的

GINインデックス構築時に使用される赤黒木（Red-Black Tree）において、GinEntryAccumulator ノード同士を比較し、順序を決定する。

シグネチャまたは形状

static int cmpEntryAccumulator(const RBTNode *a, const RBTNode *b, void *arg)

入力と出力

  • **入力**:
  • a, b: 比較対象の RBTNode（実際には GinEntryAccumulator 型へキャストされる）。
  • arg: BuildAccumulator 構造体へのポインタ。比較に必要な ginstate を保持する。
  • **出力**:
  • 比較結果を示す整数（a < b なら負、a == b なら 0、a > b なら正）。

前提条件と不変条件

  • a および b は GinEntryAccumulator 型としてキャスト可能であること。
  • arg は有効な BuildAccumulator 構造体を指していること。

アルゴリズムまたは振る舞い

1. 引数の RBTNode ポインタを GinEntryAccumulator 型にキャストする。 2. arg から BuildAccumulator を取得する。 3. ginCompareAttEntries を呼び出し、属性番号（attnum）、キー値（key）、およびカテゴリ（category）に基づいて比較を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginbulk.c 内の赤黒木操作関数（rbt_insert 等のコールバックとして使用）。
  • **呼び出し先**:
  • ginCompareAttEntries（access/gin_private.h で定義される比較関数）。

関連データ構造

  • GinEntryAccumulator: 比較対象となるノード構造体。
  • BuildAccumulator: 比較に必要なコンテキスト（ginstate）を保持する構造体。
  • RBTNode: 赤黒木の汎用ノード構造体。

失敗モードと境界条件

  • ginCompareAttEntries がエラーを発生させた場合、その例外が伝播する。
  • arg が NULL の場合、BuildAccumulator へのアクセス時にセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は赤黒木の比較コールバックとしてのみ使用される。
  • 比較ロジックを変更する場合は、ginCompareAttEntries の仕様と整合性を保つ必要がある。
