AppendJumble

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/queryjumblefuncs.c:296

目的

クエリのフィンガープリント生成プロセスにおいて、指定されたバイト列をハッシュ計算用のバッファに追加します。

シグネチャまたは形状

static pg_noinline void AppendJumble(JumbleState *jstate, const unsigned char *value, Size size)

入力と出力

  • **入力**:
  • jstate: ハッシュ計算の状態とバッファを保持する JumbleState 構造体へのポインタ。
  • value: ハッシュ対象となるデータのバイト列。
  • size: value のバイトサイズ。
  • **出力**: なし（副作用として jstate 内のバッファを更新）。

前提条件と不変条件

  • jstate は有効な JumbleState 構造体を指している必要があります。
  • pg_noinline 指定により、インライン展開は抑制されます。

アルゴリズムまたは振る舞い

1. jstate->pending_nulls が 0 より大きい場合、保留中の NULL 値を FlushPendingNulls を呼び出してバッファに書き出します。 2. AppendJumbleInternal を呼び出し、指定された value と size をハッシュバッファに書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/queryjumblefuncs.c 内の各ノード走査処理（マクロ JUMBLE_NODE や JUMBLE_FIELD を経由する箇所を含む）。
  • **呼び出し先**:
  • FlushPendingNulls
  • AppendJumbleInternal

関連データ構造

  • JumbleState: クエリのフィンガープリント生成に必要な状態（バッファ、保留中のNULL数など）を保持する構造体。

失敗モードと境界条件

  • バッファが溢れた場合、AppendJumbleInternal 内部でハッシュ化処理が実行され、バッファが再利用されます。

保守担当者向けメモ

  • 新しいノード型を追加する際は、本関数を直接呼び出すか、提供されているマクロを使用してノードのフィールドをシリアライズしてください。
  • pg_noinline が指定されているため、スタックフレームの管理に注意してください。
