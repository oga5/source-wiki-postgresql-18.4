HeapPageFreeze

定義

  • 種別: struct
  • 定義箇所: src/include/access/heapam.h:177

目的

VACUUM 処理において、ヒープページ内のタプルを凍結（Freeze）する際の計画と、その結果として更新されるトランザクション ID（XID）およびマルチトランザクション ID（MXID）の追跡状態を保持するために使用されます。

シグネチャまたは形状

typedef struct HeapPageFreeze
{
    bool            freeze_required;
    TransactionId   FreezePageRelfrozenXid;
    MultiXactId     FreezePageRelminMxid;
    TransactionId   NoFreezePageRelfrozenXid;
    MultiXactId     NoFreezePageRelminMxid;
} HeapPageFreeze;

入力と出力

本構造体は関数引数として渡され、VACUUM の進行に伴い各フィールドが更新されます。

  • freeze_required: ページ凍結の必要性を示すフラグ。
  • FreezePage... / NoFreezePage...: 凍結実行時および非実行時の、リレーションの relfrozenxid および relminmxid を更新するための追跡値。

前提条件と不変条件

  • vacuumlazy.c は、タプルの有無に関わらず、定義上あらゆるページを凍結可能です。
  • ページを凍結する際は、原則として OldestXmin より小さいすべての XID を凍結対象とします。
  • NoFreeze 系のトラッカーは、クリーンアップロックを取得せずにスキャンを行う場合と同様のロジックで維持されます。

アルゴリズムまたは振る舞い

  • heap_prepare_freeze_tuple から得られる「凍結計画」に基づき、ページ単位の凍結状態を管理します。
  • マルチトランザクションの処理において、即時削除（eager）と遅延処理（lazy）の柔軟な切り替えを可能にするための状態保持を行います。
  • ページが「完全に凍結された」と見なされる場合、可視性マップ（Visibility Map）の更新判断に利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/heapam.c および src/backend/access/heap/pruneheap.c 内の VACUUM 関連処理から参照されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数呼び出しを直接行うものではありません。

関連データ構造

  • HeapTupleFreeze: 個別のタプルに対する凍結計画を保持する構造体。
  • PruneFreezeResult: ページ剪定および凍結の結果をまとめる構造体。

失敗モードと境界条件

  • ページ内に凍結対象の XID が存在しない場合でも、トラッカー値は維持される必要があります。
  • 凍結処理の柔軟性を維持するため、MultiXact の影響範囲を過度に限定しない設計となっています。

保守担当者向けメモ

  • 本構造体は、トランザクション ID 周回問題を防ぐための VACUUM ロジックの中核を担います。
  • FreezePage と NoFreezePage のトラッカーが、どのような条件下で relfrozenxid の更新に寄与するかを理解することが重要です。
