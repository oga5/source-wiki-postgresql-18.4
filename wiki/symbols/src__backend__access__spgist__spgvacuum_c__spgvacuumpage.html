spgvacuumpage

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgvacuum.c:621

目的

SP-GiST インデックスの各ページに対して VACUUM 処理を実行し、不要なタプルの削除、リダイレクト情報の整理、および空き領域の管理（FSM への登録）を行う。

シグネチャまたは形状

static void spgvacuumpage(spgBulkDeleteState *bds, Buffer buffer)

入力と出力

  • **入力**:
  • bds: spgBulkDeleteState 構造体へのポインタ。VACUUM 実行中の状態や統計情報を保持。
  • buffer: 処理対象のインデックスページを保持するバッファ。
  • **出力**: なし（副作用としてバッファの更新および FSM への反映を行う）。

前提条件と不変条件

  • ルートページは削除対象外であり、FSM にも登録されない。
  • ページロックは関数内で排他モード（BUFFER_LOCK_EXCLUSIVE）で取得される。
  • 処理終了時には必ずバッファのロック解除と解放が行われる。

アルゴリズムまたは振る舞い

1. 対象ページのロックを取得し、ページの内容を確認する。 2. ページの状態に応じた処理を実行:

  • **新規ページ/空ページ**: 特段の処理を行わない。
  • **リーフページ**:
  • ルートページであれば vacuumLeafRoot を呼び出す。
  • それ以外であれば vacuumLeafPage でタプルを削除し、vacuumRedirectAndPlaceholder でリダイレクト情報を整理する。
  • **内部ページ**: vacuumRedirectAndPlaceholder を呼び出し、リダイレクト情報を整理する。

    • 3. ルートページ以外の場合:

  • ページが空または新規であれば RecordFreeIndexPage を呼び出し、削除ページとして統計を更新する。
  • それ以外であれば SpGistSetLastUsedPage を呼び出し、最終使用ブロックとして記録する。

    • 4. バッファのロックを解除し、解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: spgvacuumscan（src/backend/access/spgist/spgvacuum.c 内で確認）。
  • **呼び出し先**:
  • vacuumLeafRoot
  • vacuumLeafPage
  • vacuumRedirectAndPlaceholder
  • RecordFreeIndexPage
  • SpGistSetLastUsedPage

関連データ構造

  • spgBulkDeleteState: VACUUM 処理の全体状態を保持。
  • Buffer: PostgreSQL のバッファ管理構造体。
  • Page: インデックスページデータ。

失敗モードと境界条件

  • データベースクラッシュ直後に発生しうる「全ゼロページ」は、新規ページとして扱われリサイクルされる。
  • ルートページに対する FSM 登録や削除は、インデックス構造の整合性を保つため明示的に回避されている。

保守担当者向けメモ

  • 本関数は spgvacuumscan による物理走査の過程で呼び出される。
  • vacuumLeafPage や vacuumRedirectAndPlaceholder といった下位関数がページ内の複雑な構造変更を担っているため、修正時はそれらの仕様と整合させること。
