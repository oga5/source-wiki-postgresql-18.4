init_libpq_source

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/libpq_source.c:80

目的

リモートの PostgreSQL サーバーからデータを取得するための libpq_source 構造体を初期化し、rewind_source インターフェースを実装するコンストラクタとして機能します。

シグネチャまたは形状

rewind_source *init_libpq_source(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn（リモートサーバーへの既存の libpq 接続）。
  • **出力**: rewind_source *（初期化されたインターフェース構造体へのポインタ）。

前提条件と不変条件

  • 引数 conn は有効な PGconn オブジェクトである必要があります。
  • 内部で init_libpq_conn(conn) を呼び出し、接続の初期設定（タイムアウト無効化、読み取り専用モード設定等）を行います。

アルゴリズムまたは振る舞い

1. init_libpq_conn(conn) を呼び出し、接続環境を整えます。 2. libpq_source 構造体メモリを確保します。 3. rewind_source インターフェースの各関数ポインタ（traverse_files, fetch_file, queue_fetch_file, queue_fetch_range, finish_fetch, get_current_wal_insert_lsn, destroy）を、対応する libpq_ 接頭辞を持つ関数に割り当てます。 4. StringInfo バッファ（paths, offsets, lengths）を初期化し、リクエストキューイングの準備を行います。 5. 構築した rewind_source インターフェースを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/pg_rewind.c:316
  • **呼び出し先**: init_libpq_conn, pg_malloc0, initStringInfo

関連データ構造

  • libpq_source: rewind_source を継承し、接続情報とリクエストキューを保持する構造体。
  • rewind_source: pg_rewind がリモートデータソースを抽象化するために使用するインターフェース構造体。

失敗モードと境界条件

  • pg_malloc0 によるメモリ確保が失敗した場合、プロセスは終了します。
  • init_libpq_conn 内での接続設定が失敗した場合の挙動は、当該関数の実装に依存します。

保守担当者向けメモ

  • 本関数は接続の所有権を管理しません。libpq_destroy を呼び出しても PGconn は閉じられないため、呼び出し元で適切に管理する必要があります。
  • StringInfo バッファは、process_queued_fetch_requests でのリクエストバッチ処理に使用されます。
