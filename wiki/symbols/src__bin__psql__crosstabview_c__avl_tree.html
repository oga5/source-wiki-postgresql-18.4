avl_tree

定義

  • 種別: struct
  • 定義箇所: src/bin/psql/crosstabview.c:73

目的

psql の \crosstabview コマンド実行時、クエリ結果から抽出された行および列ヘッダーの重複排除とソート済み配列生成を行うための平衡二分探索木を管理する。

シグネチャまたは形状

typedef struct _avl_tree
{
    int         count;          /* 全ノード数 */
    avl_node   *root;           /* 木のルート */
    avl_node   *end;            /* 参照可能な空ノード（番兵） */
} avl_tree;

入力と出力

  • 入力: avlInit による初期化後、avlMergeValue を通じて文字列データ（ヘッダー名およびソート値）が挿入される。
  • 出力: avlCollectFields を介して、木に格納されたデータがソート済み配列として抽出される。

前提条件と不変条件

  • end メンバは、子ノードが存在しない場合の終端を示す不変の空ノードとして機能する。
  • 挿入専用の構造であり、削除機能は提供されない。
  • 1つのセルに複数の値がマッピングされる重複発生時には、処理が中断される。

アルゴリズムまたは振る舞い

  • 挿入時に AVL 木の平衡を維持する。
  • avlInit で初期化され、avlFree でメモリが解放される。
  • avlAdjustBalance などの補助関数により、挿入後の回転操作（avlRotate）を行い、平衡状態を保つ。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/crosstabview.c 内の piv_columns および piv_rows として宣言・使用されている。
  • 呼び出し先: avlInit, avlFree, avlAdjustBalance, avlMergeValue, avlCollectFields 等の avl 接頭辞を持つ関数群が本構造体を操作する。

関連データ構造

  • avl_node: 木の各ノードを構成する構造体。
  • pivot_field: ヘッダー値、ソート値、ランクを保持する構造体。

失敗モードと境界条件

  • CROSSTABVIEW_MAX_COLUMNS (1600) を超える列が生成される場合、メモリ保護のためエラーを返す。
  • ソート列が指定された際、数値として解釈できない場合はランク 0 として扱われる。

保守担当者向けメモ

  • 本構造体は挿入専用の最小限の実装である。検索や削除の機能は含まれていない点に注意すること。
