ECPGdescribe

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/descriptor.c:854

目的

プリペアドステートメントのメタデータを取得し、指定されたSQL記述子（Descriptor）またはSQLDA構造体に格納します。

シグネチャまたは形状

bool ECPGdescribe(int line, int compat, bool input, const char *connection_name, const char *stmt_name, ...);

可変長引数（...）により、対象となる記述子名またはSQLDAポインタのリストを受け取ります。

入力と出力

  • **入力**:
  • line: 呼び出し元のソースコード行番号（エラー報告用）。
  • compat: 互換性モード（Informixモード等）。
  • input: 入力パラメータの記述を要求するかどうか（現在は未サポート）。
  • connection_name: 接続名。
  • stmt_name: 記述対象のプリペアドステートメント名。
  • ...: ECPGttype 型の識別子と、それに対応する記述子名またはSQLDAポインタのペア。
  • **出力**:
  • 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • input 引数が true の場合、ECPG_UNSUPPORTED エラーを発生させます。
  • 指定された connection_name および stmt_name が有効である必要があります。
  • 可変長引数は ECPGt_EORT で終了する必要があります。

アルゴリズムまたは振る舞い

1. 接続およびプリペアドステートメントの存在を確認します。 2. 可変長引数を走査し、各要素（ECPGt_descriptor または ECPGt_sqlda）を処理します。 3. PQdescribePrepared を呼び出し、サーバーからメタデータを取得します。 4. ECPGt_descriptor の場合: 既存の desc->result を解放し、新しい PGresult を格納します。 5. ECPGt_sqlda の場合: 互換性モードに応じて ecpg_build_compat_sqlda または ecpg_build_native_sqlda を使用してSQLDAを構築し、古いメモリを解放して更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPGプリプロセッサが生成したコードから呼び出されます（例: compat_informix-describe.c 等のテストコードで確認）。
  • **呼び出し先**: ecpg_get_connection, ecpg_find_prepared_statement, ecpg_find_desc, PQdescribePrepared, ecpg_build_compat_sqlda, ecpg_build_native_sqlda 等。

関連データ構造

  • struct descriptor: 記述子情報を保持。
  • struct sqlda_compat / struct sqlda_struct: SQLDA構造体。
  • PGresult: libpqのクエリ結果構造体。

失敗モードと境界条件

  • 接続が見つからない場合、またはステートメントが準備されていない場合は false を返し、エラーを発生させます。
  • PQdescribePrepared が失敗した場合、処理を中断します。
  • メモリ割り当て失敗や不正な引数指定時の挙動については、ソースコード上での明示的な例外処理は限定的です。

保守担当者向けメモ

  • 可変長引数の解析ロジックは、ECPGプリプロセッサが生成するコードの構造と密接に結合しているため、変更時はプリプロセッサ側の生成ロジックとの整合性に注意してください。
