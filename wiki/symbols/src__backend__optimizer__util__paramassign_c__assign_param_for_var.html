assign_param_for_var

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/paramassign.c:65

目的

クエリ内の特定の Var ノードに対し、実行時に値を渡すための PARAM_EXEC ID を割り当てる、または既存の ID を再利用する。

シグネチャまたは形状

int assign_param_for_var(PlannerInfo *root, Var *var)

入力と出力

  • **入力**:
  • root: 現在のクエリレベルの PlannerInfo 構造体。
  • var: パラメータ化対象の Var ノード。
  • **出力**:
  • 割り当てられた PARAM_EXEC ID（整数）。

前提条件と不変条件

  • var->varlevelsup を用いて、変数が定義されているクエリレベルまで root を遡る必要がある。
  • 割り当てられた ID は root->glob->paramExecTypes に型情報として登録され、プラン全体で一意に管理される。

アルゴリズムまたは振る舞い

1. **レベルの特定**: var->varlevelsup に基づき、変数が属するクエリレベルの PlannerInfo を特定する。 2. **重複排除**: 対象レベルの root->plan_params を走査し、属性（varno, varattno, vartype 等）および varnullingrels が一致する既存の PlannerParamItem があれば、その paramId を即座に返す。 3. **新規割り当て**:

  • 一致するものがない場合、var をコピーし varlevelsup を 0 に設定する。
  • 新規の PlannerParamItem を作成し、root->glob->paramExecTypes の現在の長さを新しい paramId として割り当てる。
  • root->glob->paramExecTypes に変数の型情報を追加し、root->plan_params に新しい項目を登録する。

    • 4. **戻り値**: 確定した paramId を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/paramassign.c 内の他の関数（replace_outer_var 等から呼び出されることが示唆されている）。
  • **呼び出し先**:
  • copyObject: 変数ノードの複製。
  • makeNode: PlannerParamItem の生成。
  • lappend, lappend_oid: リストへの追加。
  • bms_equal: varnullingrels の比較。

関連データ構造

  • PlannerInfo: クエリのプランニング情報を保持する構造体。
  • PlannerParamItem: パラメータ化された項目を保持するリスト要素。
  • Var: 参照される変数ノード。
  • ParamExecTypes: root->glob 内で管理される実行時パラメータの型リスト。

失敗モードと境界条件

  • varlevelsup が不正な場合、root->parent_root の参照でクラッシュする可能性がある。
  • 比較ロジックは _equalVar に準拠するが、varlevelsup は無視されるため、異なるレベルの変数が同一の paramId を共有することはない。

保守担当者向けメモ

  • 本関数は PlaceHolderVar 用の assign_param_for_placeholdervar と類似のロジックを持つ。修正時は両者の整合性に注意すること。
  • varnullingrels の比較は外結合の最適化において重要であり、ここでの不一致は実行時の誤った結果を招く。
