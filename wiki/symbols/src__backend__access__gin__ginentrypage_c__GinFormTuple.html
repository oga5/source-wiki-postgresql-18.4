GinFormTuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginentrypage.c:43

目的

GINインデックスのエントリーツリーで使用される、キーとポスティングリストを格納したインデックスタプルを構築する。

シグネチャまたは形状

IndexTuple GinFormTuple(GinState *ginstate,
                        OffsetNumber attnum, Datum key, GinNullCategory category,
                        Pointer data, Size dataSize, int nipd,
                        bool errorTooBig)

入力と出力

  • **入力**:
  • ginstate: GINインデックスの状態管理構造体。
  • attnum, key, category: タプルに含める属性番号、キー値、およびNULLカテゴリ。
  • data, dataSize: オプションのポスティングリストデータとそのサイズ。
  • nipd: ポスティングリスト内の項目数。
  • errorTooBig: サイズ超過時にエラーを発生させるかのフラグ。
  • **出力**: 構築された IndexTuple。サイズ超過かつ errorTooBig が真の場合はエラー終了、偽の場合は NULL を返す。

前提条件と不変条件

  • index_form_tuple により生成されたタプルは MAXALIGN 調整される。
  • category が GIN_CAT_NORM_KEY 以外の場合、タプルはNULL値を持つ必要があり、GinNullCategory を格納するための領域が確保される。
  • 最終的なタプルサイズは GinMaxItemSize を超えてはならない。

アルゴリズムまたは振る舞い

1. index_form_tuple を使用して、属性番号とキーから基本タプルを生成する。 2. IndexTupleSize を取得し、必要に応じて GinNullCategory 用の領域を確保する。 3. ポスティングリストのオフセットと項目数をタプルヘッダーに記録する。 4. dataSize 分の領域を確保し、必要に応じて repalloc でタプルを拡張する。拡張時は新規領域をゼロクリアする。 5. data が提供されている場合、GinGetPosting で取得した位置にデータをコピーする。 6. category が GIN_CAT_NORM_KEY 以外の場合、GinSetNullCategory を呼び出してカテゴリバイトを書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginfast.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/gin/ginvacuum.c
  • **呼び出し先**:
  • index_form_tuple (PostgreSQLコア関数)
  • repalloc (PostgreSQLメモリ管理)
  • GinSetPostingOffset, GinSetNPosting, GinGetPosting, GinSetNullCategory (GIN内部マクロ/関数)

関連データ構造

  • GinState: GINインデックスの操作状態を保持する構造体。
  • IndexTuple: PostgreSQLのインデックス行構造。
  • GinNullCategory: GINインデックスにおけるNULL値の分類。

失敗モードと境界条件

  • **サイズ超過**: 構築後のタプルサイズが GinMaxItemSize を超える場合、errorTooBig が真であれば ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラー終了し、偽であれば pfree を実行して NULL を返す。
  • **メモリ不足**: repalloc 失敗時はPostgreSQLのメモリ管理機構によりエラー終了する。

保守担当者向けメモ

  • 本関数は index_form_tuple の後にサイズ調整を行うため、パディングによる領域の無駄が発生する可能性がある。
  • WALリカバリ時にも使用されるため、ginstate の整合性に注意すること。
  • 以前のバージョンでは拡張領域のゼロクリアが行われていなかったため、古いデータとの互換性には留意が必要。
