file_fdw_validator

定義

  • 種別: function
  • 定義箇所: contrib/file_fdw/file_fdw.c:206

目的

外部テーブル定義時に指定されたオプションの妥当性を検証し、セキュリティ権限チェックを行う。

シグネチャまたは形状

Datum file_fdw_validator(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_DATUM(0): オプションリスト（List）。
  • PG_GETARG_OID(1): オブジェクトのカタログOID。
  • **出力**: PG_RETURN_VOID() により値を返さない。検証失敗時は ereport(ERROR, ...) を発行する。

前提条件と不変条件

  • filename オプションの設定には pg_read_server_files ロールの権限が必要。
  • program オプションの設定には pg_execute_server_program ロールの権限が必要。
  • ForeignTableRelationId の場合、filename または program のいずれかが必須。

アルゴリズムまたは振る舞い

1. untransformRelOptions でオプションリストを展開する。 2. 各オプションを is_valid_option で検証し、不明な場合は類似の有効なオプションをヒントとして提示する。 3. filename または program が指定されている場合、現在のユーザーの権限を has_privs_of_role で確認する。 4. force_not_null および force_null オプションの重複チェックと型検証を行う。 5. 残りのオプションを ProcessCopyOptions に渡し、COPY コマンドの標準的な検証ロジックを実行する。 6. 外部テーブル定義において filename と program の双方が欠如している場合はエラーとする。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の FDW インフラストラクチャ（SQL レベルの CREATE FOREIGN TABLE 等から間接的に呼び出される）。
  • **呼び出し先**: is_valid_option, initClosestMatch, updateClosestMatch, getClosestMatch, has_privs_of_role, defGetString, defGetBoolean, ProcessCopyOptions。

関連データ構造

  • FileFdwOption: 有効なオプション定義の配列。
  • DefElem: オプションのキーと値のペアを保持するノード。

失敗モードと境界条件

  • **権限不足**: 必要なロールを持たないユーザーが filename や program を設定しようとした場合、ERRCODE_INSUFFICIENT_PRIVILEGE で失敗する。
  • **重複設定**: filename と program の両方が指定された場合、または force_not_null 等が複数回指定された場合、ERRCODE_SYNTAX_ERROR で失敗する。
  • **不正なオプション**: file_fdw がサポートしないオプションが指定された場合、ERRCODE_FDW_INVALID_OPTION_NAME で失敗する。

保守担当者向けメモ

  • ProcessCopyOptions は COPY コマンドのコアロジックに依存しているため、src/backend/commands/copyfrom.c の変更が影響する可能性がある。
  • filename および program の権限チェックは、この関数内で行うことが現在の実装上の制約となっている。
