GrantRole

定義

  • 種別: function
  • 定義箇所: src/backend/commands/user.c:1479

目的

ロールメンバーシップの付与（GRANT）または剥奪（REVOKE）を実行し、pg_auth_members カタログを更新する。

シグネチャまたは形状

void GrantRole(ParseState *pstate, GrantRoleStmt *stmt)

入力と出力

  • **入力**:
  • pstate: パーサー状態。
  • stmt: GRANT / REVOKE 文の構文解析ノード。opt（オプション）、grantor（付与者）、grantee_roles（被付与者）、granted_roles（対象ロール）、is_grant（操作種別）、behavior（削除時の挙動）を含む。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • pg_authid に対する AccessShareLock を保持する。
  • 操作実行前に check_role_membership_authorization を呼び出し、現在のユーザーが対象ロールに対して操作権限を持つことを検証する。

アルゴリズムまたは振る舞い

1. InitGrantRoleOptions でオプション構造体を初期化し、stmt->opt を解析して admin, inherit, set の各フラグを設定する。 2. stmt->grantor が指定されていればその OID を取得する。 3. stmt->grantee_roles を OID リストに変換する。 4. pg_authid をオープンする。 5. stmt->granted_roles を反復処理し、各ロールに対して以下を実行する:

  • 列指定（priv->cols）が含まれる場合はエラーとする。
  • is_grant が真なら AddRoleMems を呼び出し、偽なら DelRoleMems を呼び出す。

    • 6. pg_authid をクローズする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c:766
  • **呼び出し先**:
  • InitGrantRoleOptions, get_rolespec_oid, roleSpecsToIds, table_open, get_role_oid, check_role_membership_authorization, AddRoleMems, DelRoleMems, table_close

関連データ構造

  • GrantRoleStmt: 構文解析ノード。
  • GrantRoleOptions: GRANT オプション（admin, inherit, set）を保持する構造体。
  • pg_authid, pg_auth_members: 操作対象のシステムカタログ。

失敗モードと境界条件

  • 未知のロールオプションが指定された場合、ERRCODE_SYNTAX_ERROR でエラー。
  • オプション値が不正な場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー。
  • GRANT/REVOKE ROLE に列指定が含まれる場合、ERRCODE_INVALID_GRANT_OPERATION でエラー。
  • 権限チェック失敗時は check_role_membership_authorization 内でエラーが発生する。

保守担当者向けメモ

  • 本関数は pg_auth_members の更新を AddRoleMems および DelRoleMems に委譲している。
  • pg_authid のロックは table_close 後もトランザクション終了まで保持される点に注意。
