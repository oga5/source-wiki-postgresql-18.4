buildDefaultACLCommands

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/dumputils.c:365

目的

データベースオブジェクトのデフォルト権限（Default ACL）を操作するための ALTER DEFAULT PRIVILEGES SQL コマンドを構築します。

シグネチャまたは形状

bool buildDefaultACLCommands(const char *type, const char *nspname,
                             const char *acls, const char *acldefault,
                             const char *owner,
                             int remoteVersion,
                             PQExpBuffer sql)

入力と出力

  • **入力**:
  • type: オブジェクトタイプ（例: "TABLES"）。
  • nspname: スキーマ名（NULL の場合はスキーマ指定なし）。
  • acls: 現在の ACL 文字列。
  • acldefault: デフォルトの ACL 文字列。
  • owner: ロール名。
  • remoteVersion: 接続先データベースのバージョン。
  • sql: 生成されたコマンドを格納する PQExpBuffer。
  • **出力**:
  • 成功時に true、失敗時に false を返します。sql バッファに SQL 文が追記されます。

前提条件と不変条件

  • owner は fmtId でエスケープされるため、有効な識別子である必要があります。
  • デフォルト ACL には initprivs の概念が存在しないため、内部的に空文字列が buildACLCommands へ渡されます。

アルゴリズムまたは振る舞い

1. ALTER DEFAULT PRIVILEGES FOR ROLE <owner> を含むプレフィックス文字列を構築します。 2. nspname が指定されている場合、IN SCHEMA <nspname> をプレフィックスに追加します。 3. buildACLCommands を呼び出し、実際の GRANT/REVOKE 差分を生成して sql バッファに連結します。 4. 処理の過程で一時的に使用した prefix バッファを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c:16187 にて呼び出されています。
  • **呼び出し先**: buildACLCommands を内部で呼び出します。

関連データ構造

  • PQExpBuffer: SQL 文の構築に使用される動的文字列バッファ。

失敗モードと境界条件

  • buildACLCommands が失敗した場合、本関数も false を返し、処理を中断します。
  • owner や nspname が NULL または不正な値の場合、fmtId の挙動に依存します。

保守担当者向けメモ

  • 本関数は SET ROLE 等のセッション操作を行わず、コマンド内に直接ロール名を埋め込むことで、権限エラー発生時に意図しないユーザーのデフォルト権限が変更されるリスクを回避しています。
  • PostgreSQL のバージョン間で ALTER DEFAULT PRIVILEGES の構文に変更があった場合は、本関数内のプレフィックス構築ロジックを確認してください。
