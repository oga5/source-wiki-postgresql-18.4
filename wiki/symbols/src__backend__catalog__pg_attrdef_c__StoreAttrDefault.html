StoreAttrDefault

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_attrdef.c:34

目的

テーブルの列に対するデフォルト値を pg_attrdef カタログに永続化し、関連する属性情報の更新および依存関係の記録を行う。

シグネチャまたは形状

Oid StoreAttrDefault(Relation rel, AttrNumber attnum, Node *expr, bool is_internal)

入力と出力

  • **入力**:
  • rel: 対象テーブルのリレーション記述子。
  • attnum: デフォルト値を設定する列番号。
  • expr: デフォルト値の式（Node ツリー）。
  • is_internal: 内部的なデフォルト値（生成列など）かどうかを示すフラグ。
  • **出力**:
  • 新規作成された pg_attrdef エントリの Oid を返す。

前提条件と不変条件

  • rel は RowExclusiveLock でオープンされている必要がある。
  • pg_attribute システムキャッシュが利用可能であること。

アルゴリズムまたは振る舞い

1. **シリアライズ**: nodeToString を用いて expr を文字列形式に変換する。 2. **カタログ挿入**: pg_attrdef に新しいエントリを挿入し、生成された Oid を取得する。 3. **属性更新**: pg_attribute を検索し、対象列の atthasdef フラグを true に更新する。 4. **依存関係の記録**:

  • pg_attrdef エントリと列の間に依存関係を作成する（生成列の場合は DEPENDENCY_INTERNAL、通常は DEPENDENCY_AUTO）。
  • 式内で使用されているオブジェクトに対する依存関係を記録する。

    • 5. **フック実行**: InvokeObjectPostCreateHookArg を呼び出し、後処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • table_open, table_close
  • nodeToString
  • GetNewOidWithIndex
  • heap_form_tuple, CatalogTupleInsert, CatalogTupleUpdate
  • SearchSysCacheCopy2
  • recordDependencyOn, recordDependencyOnSingleRelExpr
  • InvokeObjectPostCreateHookArg

関連データ構造

  • pg_attrdef: デフォルト値を保持するシステムカタログ。
  • pg_attribute: 列定義を保持するシステムカタログ。

失敗モードと境界条件

  • SearchSysCacheCopy2 が対象属性を見つけられない場合、elog(ERROR) により処理を中断する。

保守担当者向けメモ

  • ALTER TABLE によるデフォルト値の変更は「削除→再作成」のシーケンスで行われるため、フック処理の実装時には古いエントリの存在を確認する必要がある。
  • 生成列のデフォルト値は DEPENDENCY_INTERNAL として扱われるため、通常のデフォルト値とは削除時の挙動が異なる点に注意すること。
