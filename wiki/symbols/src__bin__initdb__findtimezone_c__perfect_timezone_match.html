perfect_timezone_match

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/findtimezone.c:319

目的

指定されたタイムゾーン名が、テスト用データセット（tztry）に対して完全に一致するかを判定します。

シグネチャまたは形状

static bool perfect_timezone_match(const char *tzname, struct tztry *tt);

入力と出力

  • **入力**:
  • tzname: 照合対象のタイムゾーン名（文字列）。
  • tt: 比較基準となるテストデータ構造体（struct tztry）。
  • **出力**:
  • bool: すべてのテストケースで一致した場合は true、それ以外は false を返します。

前提条件と不変条件

  • tt は有効な tztry 構造体へのポインタである必要があります。
  • score_timezone 関数が、テストデータセット内の全テスト回数（tt->n_test_times）を正しく算出できる状態である必要があります。

アルゴリズムまたは振る舞い

1. score_timezone(tzname, tt) を呼び出し、指定されたタイムゾーン名がテストデータセットと何回一致するかを計算します。 2. そのスコアが tt->n_test_times（全テスト回数）と等しいかどうかを比較します。 3. スコアが全回数と一致する場合にのみ true を返し、完全一致とみなします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/initdb/findtimezone.c:593（自動検出プロセスにおけるタイムゾーン探索ループ内）
  • **呼び出し先**:
  • score_timezone

関連データ構造

  • struct tztry: タイムゾーンの妥当性を検証するためのサンプリングデータおよびテスト回数を保持する構造体。

失敗モードと境界条件

  • tzname が無効なタイムゾーン名である場合、score_timezone は低いスコアを返し、本関数は false を返します。
  • tt->n_test_times が 0 の場合、ロジック上は常に true を返す可能性があります（データセットの初期化状態に依存）。

保守担当者向けメモ

  • 本関数は、システム自動検出時に候補となるタイムゾーンを絞り込むための厳密なフィルタとして機能します。
  • score_timezone の実装に変更があった場合、この「完全一致」の定義が維持されるか確認してください。
