astreamer_recovery_injector_new

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/astreamer_inject.c:64

目的

アーカイブストリーム処理パイプラインに組み込むための、リカバリ設定注入用ストリーマー（astreamer_recovery_injector）を初期化し、コンストラクタとして機能する。

シグネチャまたは形状

astreamer *astreamer_recovery_injector_new(astreamer *next,
                                           bool is_recovery_guc_supported,
                                           PQExpBuffer recoveryconfcontents)

入力と出力

  • **入力**:
  • next: パイプライン内の後続ストリーマーへのポインタ。
  • is_recovery_guc_supported: PostgreSQL のバージョンに応じたリカバリ設定方式（recovery.conf か postgresql.auto.conf か）を判定するためのフラグ。
  • recoveryconfcontents: 注入するリカバリ設定内容を保持するバッファ。
  • **出力**:
  • 初期化された astreamer 構造体へのポインタを返す。

前提条件と不変条件

  • palloc0 によりメモリがゼロクリアされた状態で astreamer_recovery_injector が割り当てられる。
  • bbs_ops には静的定数 astreamer_recovery_injector_ops が割り当てられる。

アルゴリズムまたは振る舞い

1. astreamer_recovery_injector 構造体分のメモリを動的に確保する。 2. 構造体の基底部分（base）の bbs_ops に操作関数テーブルを設定する。 3. 引数として受け取った後続ストリーマー（next）、GUC サポートフラグ、設定内容バッファを構造体の各メンバに格納する。 4. 初期化済みの astreamer 基底ポインタを呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/pg_basebackup.c:1245 にて直接呼び出されている。
  • **呼び出し先**: 内部で palloc0 を呼び出す。

関連データ構造

  • astreamer_recovery_injector: 本関数で初期化される状態保持用構造体。
  • astreamer: パイプラインの基底構造体。
  • astreamer_ops: ストリーマーの操作関数テーブル。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は astreamer フレームワークのコンストラクタであり、設定ファイルの注入ロジックは後続の content 処理関数へ委譲される。
  • is_recovery_guc_supported フラグは、PostgreSQL 12 以降の挙動変更を制御する重要なパラメータである。
