identify_opfamily_groups

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/amvalidate.c:42

目的

演算子ファミリーに属する演算子およびサポート関数のリストを、データ型ペア（左型・右型）ごとに集約し、戦略番号および関数番号をビットマスクとして整理したグループリストを生成します。

シグネチャまたは形状

List *identify_opfamily_groups(CatCList *oprlist, CatCList *proclist)

入力と出力

  • **入力**:
  • oprlist: pg_amop カタログのメンバーリスト（CatCList）。
  • proclist: pg_amproc カタログのメンバーリスト（CatCList）。
  • **出力**:
  • List *: OpFamilyOpFuncGroup 構造体のリスト。

前提条件と不変条件

  • 入力される oprlist および proclist は、カタログのキー順にソートされている必要があります。ソートされていない場合、elog(ERROR) が発生します。
  • 演算子戦略番号およびサポート関数番号は 1 から 63 までの範囲のみがサポートされます。これ以外の値は無視されます。

アルゴリズムまたは振る舞い

1. 入力された2つのリストを並行して走査します。 2. 現在のデータ型ペア（lefttype, righttype）と一致する演算子または関数を、既存の OpFamilyOpFuncGroup に追加します。 3. 一致しない場合、新しい OpFamilyOpFuncGroup を割り当て、リストに追加します。 4. 演算子および関数は、それぞれのビットマスク（operatorset, functionset）に戦略番号/関数番号を 1 << number の形式で格納します。 5. 全てのメンバーを走査し終えるまでこれを繰り返します。

関連データ構造

  • OpFamilyOpFuncGroup: 演算子ファミリーのグループ情報を保持する構造体（src/include/access/amvalidate.h で定義）。
  • CatCList: システムカタログのキャッシュリスト。
  • Form_pg_amop / Form_pg_amproc: カタログの行データ構造。

失敗モードと境界条件

  • oprlist または proclist がソートされていない場合、実行時エラーとなります。
  • 64以上の戦略番号や関数番号は無視されます。
  • 入力リストが空の場合、空のリストが返されます。

保守担当者向けメモ

  • 本関数はインデックスの整合性検証における前処理として使用されます。
  • 演算子と関数のリストをマージする際、データ型ペアの順序に基づいて新しいグループを作成するロジックが含まれています。この比較ロジックに変更を加える際は、カタログのソート順序との整合性に注意してください。
