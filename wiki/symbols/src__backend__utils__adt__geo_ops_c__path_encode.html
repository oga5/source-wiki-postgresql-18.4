path_encode

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_ops.c:339

目的

幾何データ型（パス、ポリゴン、ボックス、線分など）の座標配列を、PostgreSQLの幾何学的なテキスト表現形式に変換する。

シグネチャまたは形状

char *path_encode(enum path_delim path_delim, int npts, Point *pt)

入力と出力

  • **入力**:
  • path_delim: パスの境界形式（PATH_CLOSED, PATH_OPEN, PATH_NONE）。
  • npts: 処理する点の数。
  • pt: 座標データ（Point構造体）の配列へのポインタ。
  • **出力**:
  • 変換された文字列（char *）。StringInfo を介して動的に構築される。

前提条件と不変条件

  • pt は少なくとも npts 個の Point 構造体を含む有効なメモリ領域を指している必要がある。
  • pair_encode が各点の座標変換に使用される。

アルゴリズムまたは振る舞い

1. initStringInfo で文字列バッファを初期化。 2. path_delim に応じて、開始デリミタ（LDELIM または LDELIM_EP）をバッファに追加。 3. npts 分のループを実行し、各点に対して以下の処理を行う：

  • 2点目以降であれば、点同士の区切り文字（DELIM）を追加。
  • 各点を LDELIM と RDELIM で囲み、pair_encode を呼び出して座標値（x, y）を文字列化。

    • 4. 最後に path_delim に応じて、終了デリミタ（RDELIM または RDELIM_EP）を追加。 5. 構築された文字列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/geo_ops.c 内の各幾何型出力関数（box_out, path_out, point_out, lseg_out, poly_out 等）。
  • **呼び出し先**:
  • initStringInfo, appendStringInfoChar (PostgreSQL StringInfo API)
  • pair_encode (各点の座標文字列化)

関連データ構造

  • Point: 座標（x, y）を保持する構造体。
  • enum path_delim: パスの境界スタイルを定義する列挙型。
  • StringInfoData: 文字列構築用バッファ。

失敗モードと境界条件

  • npts が 0 の場合、デリミタのみを含む空のパス表現が生成される可能性がある。
  • メモリ不足時には initStringInfo 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は geo_ops.c 内の複数の幾何型で共通利用されている。デリミタの変更や出力形式の修正を行う際は、影響範囲が広いため注意すること。
  • pair_encode への依存が強いため、座標の精度やフォーマットを変更する場合は両方の関数を確認すること。
