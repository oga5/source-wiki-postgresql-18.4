simple_table_tuple_delete

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/tableam.c:290

目的

テーブル内のタプルを削除するための簡易ラッパー関数です。複雑な引数やエラーハンドリングを抽象化し、呼び出し元がトランザクション整合性を保った状態で安全に削除を実行できるようにします。

シグネチャまたは形状

void simple_table_tuple_delete(Relation rel, ItemPointer tid, Snapshot snapshot)

入力と出力

  • **入力**:
  • rel: 削除対象のテーブルリレーション。
  • tid: 削除対象タプルの物理位置（ItemPointer）。
  • snapshot: 削除操作の可視性を判断するためのスナップショット。
  • **出力**: なし（成功時は正常終了、失敗時は elog(ERROR) によりトランザクションを中断）。

前提条件と不変条件

  • 本関数は、トランザクションの整合性が保証されている環境で使用する必要があります。
  • 内部で GetCurrentCommandId(true) を使用するため、アクティブなトランザクション内である必要があります。

アルゴリズムまたは振る舞い

1. table_tuple_delete を呼び出し、現在のコマンドIDと指定されたスナップショットを用いて削除を試行します。 2. 削除結果（TM_Result）を評価します。

  • TM_Ok: 正常終了。
  • TM_SelfModified: 現在のコマンド内で既に更新済みであるためエラー。
  • TM_Updated: 並行更新が発生しているためエラー。
  • TM_Deleted: 並行削除が発生しているためエラー。
  • その他: 未定義のステータスとしてエラー。

    • 3. 成功時以外は elog(ERROR) を発行し、処理を停止します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execReplication.c:755
  • **呼び出し先**:
  • table_tuple_delete (内部で呼び出し)
  • GetCurrentCommandId (内部で呼び出し)
  • elog (エラー発生時に呼び出し)

関連データ構造

  • Relation: テーブルのメタデータ構造体。
  • ItemPointer: タプルの物理位置を示す構造体。
  • Snapshot: 可視性制御のための構造体。
  • TM_Result: 削除操作の結果コード。

失敗モードと境界条件

  • タプルが既に削除または更新されている場合、elog(ERROR) によりトランザクションがアボートされます。
  • table_tuple_delete が予期しない結果コードを返した場合、エラーとして処理されます。

保守担当者向けメモ

  • 本関数はエラーハンドリングを簡略化しているため、呼び出し側で個別のエラー復旧が必要な場合には使用しないでください。
  • table_tuple_delete の引数である wait は true に固定されており、コミット完了を待機する仕様です。
