start_conn

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/task.c:173

目的

指定されたデータベース接続スロットに対して、libpq の非同期接続処理を開始する。

シグネチャまたは形状

static void start_conn(const ClusterInfo *cluster, UpgradeTaskSlot *slot);

入力と出力

  • **入力**:
  • cluster: 接続先クラスタ情報（ポート、ソケットディレクトリ等）。
  • slot: 接続状態を保持する UpgradeTaskSlot 構造体（対象データベースのインデックスを含む）。
  • **出力**: なし（副作用として slot->conn に PGconn オブジェクトを格納）。

前提条件と不変条件

  • slot->db_idx が cluster->dbarr.dbs の有効な範囲内であること。
  • os_info.user が適切に設定されていること。

アルゴリズムまたは振る舞い

1. PQExpBuffer を使用して接続文字列を構築する。 2. データベース名、ユーザー名、ポート、および必要に応じてソケットディレクトリを文字列に追加する。 3. プロトコルネゴシエーションがサポートされていない場合は、max_protocol_version=3.0 を付与する。 4. PQconnectStart を呼び出し、非同期接続を開始する。 5. 接続オブジェクトの生成に失敗した場合は pg_fatal を呼び出し、プロセスを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/task.c 内の process_slot 関数から呼び出される。
  • **呼び出し先**: libpq API（initPQExpBuffer, appendPQExpBufferStr, appendConnStrVal, appendPQExpBuffer, PQconnectStart, termPQExpBuffer）。

関連データ構造

  • ClusterInfo: クラスタの接続設定を保持。
  • UpgradeTaskSlot: 接続状態および対象データベースのインデックスを保持。
  • DbInfo: データベース名などの情報を保持。

失敗モードと境界条件

  • メモリ不足により PQconnectStart が NULL を返した場合、pg_fatal により異常終了する。
  • 接続文字列の構築中にエラーが発生した場合の挙動は libpq のバッファ管理に依存する。

保守担当者向けメモ

  • 本関数は非同期接続の開始のみを担当する。接続の完了待機は PQconnectPoll を用いて後続のステートマシン（process_slot）で行われる。
  • 接続文字列の構築には appendConnStrVal を使用しており、識別子のクォーティングが適切に行われるよう設計されている。
