BrinOptions

定義

  • 種別: struct
  • 定義箇所: src/include/access/brin.h:21

目的

BRIN (Block Range Index) インデックスの動作を制御するための設定パラメータを保持する構造体です。

シグネチャまたは形状

typedef struct BrinOptions
{
    int32       vl_len_;        /* varlena header */
    BlockNumber pagesPerRange;
    bool        autosummarize;
} BrinOptions;

入力と出力

  • **入力**: インデックス作成時やリレーションのオプション設定時に、pages_per_range および autosummarize パラメータとして値を受け取ります。
  • **出力**: BrinGetPagesPerRange および BrinGetAutoSummarize マクロを通じて、インデックススキャンや構築処理の制御値として提供されます。

前提条件と不変条件

  • vl_len_ は varlena ヘッダーであり、直接操作してはなりません。
  • pagesPerRange は、BRIN インデックスが対象とするブロック範囲のサイズを決定します。

アルゴリズムまたは振る舞い

本構造体は、リレーションの rd_options に格納され、BRIN インデックスの挙動を動的に決定するために参照されます。src/backend/access/brin/brin.c において、pages_per_range および autosummarize の各オプションとマッピングされます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin.c 内のインデックスオプション解析処理（pages_per_range, autosummarize の設定）。
  • **呼び出し先**: src/include/access/brin.h 内の BrinGetPagesPerRange および BrinGetAutoSummarize マクロが、Relation 構造体から本構造体のメンバを抽出するために使用します。

関連データ構造

  • Relation: インデックスのオプション（rd_options）を保持する構造体。

失敗モードと境界条件

  • BrinGetPagesPerRange マクロは、リレーションが正しく BRIN インデックスであることを検証する Assert を含みます。
  • 設定値が提供されていない場合、BrinGetPagesPerRange はデフォルト値（BRIN_DEFAULT_PAGES_PER_RANGE = 128）を返します。

保守担当者向けメモ

  • 本構造体は varlena 型として扱われるため、メモリレイアウトの変更には注意が必要です。
  • インデックスの挙動を変更する新しいパラメータを追加する場合は、brin.c のオプション定義と整合性を取る必要があります。
