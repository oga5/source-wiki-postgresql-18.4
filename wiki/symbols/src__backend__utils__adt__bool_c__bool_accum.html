bool_accum

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/bool.c:340

目的

集約関数（EVERYやANYなど）の計算過程において、現在の集約状態を更新し、蓄積されたブール値の統計情報を管理します。

シグネチャまたは形状

Datum bool_accum(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • 第1引数: BoolAggState へのポインタ（集約状態、NULL許容）。
  • 第2引数: bool 型の値（入力値、NULL許容）。
  • **出力**: 更新された BoolAggState 構造体へのポインタを Datum として返します。

前提条件と不変条件

  • 第1引数がNULLの場合、makeBoolAggState を呼び出して新しい状態を初期化します。
  • 第2引数がNULLでない場合のみ、状態の更新が行われます。

アルゴリズムまたは振る舞い

1. PG_ARGISNULL(0) を確認し、状態が未初期化であれば makeBoolAggState を呼び出して新規作成します。 2. 第2引数（入力値）がNULLでない場合：

  • state->aggcount（処理した非NULL値の総数）をインクリメントします。
  • 入力値が true であれば、state->aggtrue（真の個数）をインクリメントします。

    • 3. 更新された state ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（集約関数実行時）。
  • **呼び出し先**:
  • makeBoolAggState（状態の初期化時）。
  • PG_ARGISNULL, PG_GETARG_POINTER, PG_GETARG_BOOL, PG_RETURN_POINTER（PostgreSQL関数インターフェースマクロ）。

関連データ構造

  • BoolAggState: 集約計算中に保持される内部状態構造体。

失敗モードと境界条件

  • 入力値がNULLの場合、状態の更新はスキップされます。
  • 最初の呼び出し時に状態がNULLであれば、内部的にメモリ確保が行われます。

保守担当者向けメモ

  • 本関数は bool_accum_inv とペアで動作し、ウィンドウ関数やインクリメンタルな集約をサポートする設計となっています。
  • BoolAggState の定義および初期化ロジックに変更を加える際は、本関数と bool_accum_inv の両方に影響がないか確認してください。
