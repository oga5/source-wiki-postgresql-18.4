setNewTmpRes

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_parse.c:157

目的

LexizeData 構造体内の辞書処理結果（一時的な語彙リスト）を更新し、メモリリークを防ぐために既存の古い結果を解放する。

シグネチャまたは形状

static void setNewTmpRes(LexizeData *ld, ParsedLex *lex, TSLexeme *res)

入力と出力

  • ld (LexizeData*): 辞書処理の状態を保持する構造体。
  • lex (ParsedLex*): 新しく関連付ける解析済みトークン。
  • res (TSLexeme*): 新しく設定する語彙の配列。
  • 戻り値: なし。

前提条件と不変条件

  • ld->tmpRes が指すメモリは、palloc によって確保されている必要がある。
  • res は NULL または TSLexeme 配列の先頭ポインタであること。

アルゴリズムまたは振る舞い

1. ld->tmpRes が既に存在する場合、その配列内の各 lexeme 文字列を pfree で解放し、その後 ld->tmpRes 自体を pfree で解放する。 2. 新しい res を ld->tmpRes に代入する。 3. 新しい lex を ld->lastRes に代入する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tsearch/ts_parse.c 内の LexizeExec 関数（225行目および306行目）。
  • 呼び出し先: pfree（PostgreSQL のメモリ管理関数）。

関連データ構造

  • LexizeData: 辞書処理の現在の状態を保持する構造体。
  • ParsedLex: 解析されたトークン情報。
  • TSLexeme: 辞書によって生成された語彙の配列。

失敗モードと境界条件

  • ld->tmpRes が NULL の場合、解放処理はスキップされ、新しい値が代入される。
  • res が NULL の場合、ld->tmpRes は NULL に更新される。

保守担当者向けメモ

  • 本関数は LexizeExec における辞書適用後の状態更新を担う。メモリ管理の責任が LexizeData に集約されているため、tmpRes を更新する際は必ず本関数を経由すること。
