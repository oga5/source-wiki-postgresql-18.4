find_variable

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/variable.c:192

目的

指定された名前のホスト変数を検索し、その変数情報を保持する variable 構造体を返します。構造体メンバや配列アクセスを含む複雑な変数名にも対応します。

シグネチャまたは形状

struct variable *find_variable(const char *name);

入力と出力

  • **入力**: name (検索対象の変数名文字列)
  • **出力**: 見つかった変数の struct variable *。見つからない場合や不正な場合は mmfatal によりプロセスが終了します。

前提条件と不変条件

  • 変数名は allvariables リストに登録されている必要があります。
  • 構造体メンバや配列アクセスが含まれる場合、再帰的な解決が行われます。

アルゴリズムまたは振る舞い

1. strpbrk を使用して、変数名に .（構造体）、[（配列）、-（ポインタ）が含まれるか判定します。 2. 特殊文字が含まれない場合、find_simple を呼び出して単純な変数を検索します。 3. [ が含まれる場合、配列のインデックス部分をスキップして解析します。

  • その後に . が続く場合は find_struct を呼び出します。
  • それ以外の場合は find_simple で基底変数を取得し、型に応じて新しい変数エントリを生成して返します。

    • 4. その他の特殊文字が含まれる場合は find_struct を呼び出します。 5. 最終的に変数が特定できない場合は mmfatal を発行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/descriptor.c
  • src/interfaces/ecpg/preproc/type.c
  • src/interfaces/ecpg/preproc/variable.c
  • **呼び出し先**:
  • find_simple
  • find_struct
  • new_variable
  • mmfatal

関連データ構造

  • struct variable: 変数情報を保持する構造体。
  • allvariables: 宣言された全変数を保持するリンクリスト。

失敗モードと境界条件

  • 変数が未宣言の場合: mmfatal によりエラー終了。
  • 配列の括弧が閉じられていない場合: mmfatal によりエラー終了。
  • 配列アクセスにおいて対象がポインタでない場合: mmfatal によりエラー終了。

保守担当者向けメモ

  • find_struct や find_simple と密接に連携しています。
  • 構造体メンバの再帰的な解決や、配列のインデックス処理が複雑であるため、変更時は brace_level の整合性に注意してください。
