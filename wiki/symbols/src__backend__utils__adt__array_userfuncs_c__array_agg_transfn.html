array_agg_transfn

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_userfuncs.c:555

目的

集約関数 array_agg の遷移関数として、入力された要素を順次蓄積し、配列を構築する。

シグネチャまたは形状

Datum array_agg_transfn(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報。
  • 第1引数: 蓄積状態（internal 型、ArrayBuildState へのポインタ）。
  • 第2引数: 追加対象の要素。
  • **出力**:
  • 蓄積状態を保持する ArrayBuildState へのポインタを Datum として返却。

前提条件と不変条件

  • 集約コンテキスト（aggcontext）内で呼び出される必要がある。
  • 第1引数が NULL の場合、新しい ArrayBuildState を初期化する。
  • 遷移型は internal として宣言されており、ポインタサイズとして安全に受け渡し可能である。

アルゴリズムまたは振る舞い

1. get_fn_expr_argtype を用いて入力データの型（arg1_typeid）を特定する。 2. AggCheckCallContext により、集約コンテキスト外からの呼び出しでないことを検証する。 3. 第1引数が NULL であれば initArrayResult で状態を初期化し、既存であればポインタを取得する。 4. accumArrayResult を呼び出し、現在の要素を状態に追加する。 5. 更新された ArrayBuildState ポインタを PG_RETURN_POINTER で返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の集約実行エンジン（nodeAgg.c 等）から呼び出される（直接のソースコード上の呼び出し元は明示されていない）。
  • **呼び出し先**:
  • get_fn_expr_argtype
  • AggCheckCallContext
  • initArrayResult
  • accumArrayResult
  • ereport / elog

関連データ構造

  • ArrayBuildState: 配列構築中の状態を保持する構造体。
  • PG_FUNCTION_ARGS: 関数呼び出し引数セット。

失敗モードと境界条件

  • arg1_typeid が InvalidOid の場合、エラーを報告する。
  • 集約コンテキスト外で呼び出された場合、elog(ERROR) により処理を中断する。

保守担当者向けメモ

  • 本関数は array_agg の状態遷移を担うため、並列クエリ実行時のシリアライズ/デシリアライズ関数との整合性に注意すること。
  • fn_extra を介したキャッシュ戦略は本関数内では明示されていないが、集約実行エンジン側での管理に依存している。
