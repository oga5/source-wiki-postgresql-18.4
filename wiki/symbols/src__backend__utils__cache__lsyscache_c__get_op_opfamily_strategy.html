get_op_opfamily_strategy

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/lsyscache.c:83

目的

指定された演算子（opno）が特定の演算子ファミリー（opfamily）内で持つ戦略番号（strategy number）を取得します。

シグネチャまたは形状

int get_op_opfamily_strategy(Oid opno, Oid opfamily)

入力と出力

  • **入力**:
  • opno (Oid): 検索対象の演算子OID。
  • opfamily (Oid): 検索対象の演算子ファミリーOID。
  • **出力**:
  • 成功時: 演算子ファミリー内での戦略番号（正の整数）。
  • 失敗時: 0（該当するエントリが見つからない場合）。

前提条件と不変条件

  • AMOPOPID システムキャッシュを使用して検索を行います。
  • 検索には AMOP_SEARCH フラグが使用されます。

アルゴリズムまたは振る舞い

1. SearchSysCache3 を呼び出し、AMOPOPID キャッシュから指定された演算子とファミリーに対応するタプルを検索します。 2. タプルが見つからない場合、0 を返します。 3. タプルが見つかった場合、GETSTRUCT マクロを使用して Form_pg_amop 構造体にキャストします。 4. 構造体内の amopstrategy フィールドから戦略番号を抽出します。 5. ReleaseSysCache を呼び出してタプルの参照カウントを解放します。 6. 抽出した戦略番号を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/indexcmds.c
  • src/backend/optimizer/path/indxpath.c
  • src/backend/utils/adt/selfuncs.c
  • src/backend/utils/cache/relcache.c
  • **呼び出し先**:
  • SearchSysCache3 (システムキャッシュ検索)
  • ReleaseSysCache (キャッシュ解放)
  • GETSTRUCT (タプル構造体変換)

関連データ構造

  • Form_pg_amop: pg_amop システムカタログの行構造体。amopstrategy フィールドを保持します。

失敗モードと境界条件

  • 指定された opno と opfamily の組み合わせが pg_amop に存在しない場合、関数は 0 を返します。これはエラーではなく、該当なしを示す正常な戻り値として扱われます。

保守担当者向けメモ

  • 本関数は pg_amop カタログへのアクセスを抽象化するラッパーです。
  • 頻繁に呼び出されるため、キャッシュミスが発生しないよう注意が必要です。
  • 戻り値 0 は「戦略番号が存在しない」ことを意味するため、呼び出し側でこの値を適切に処理する必要があります。
