InteractiveBackend

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/tcop/postgres.c:236

目的

対話型接続において、ユーザーからの入力をプロンプト表示とともに取得し、クエリ文字列としてバッファへ格納する。

シグネチャまたは形状

static int InteractiveBackend(StringInfo inBuf)

入力と出力

  • **入力**: StringInfo inBuf（入力文字列を蓄積するためのバッファ）
  • **出力**:
  • PqMsg_Query: 正常にクエリが入力された場合。
  • EOF: 入力がない状態で EOF を検知した場合。

前提条件と不変条件

  • inBuf は呼び出し前に初期化またはリセットされている必要がある。
  • interactive_getc() を通じて標準入力から文字を読み取る。

アルゴリズムまたは振る舞い

1. backend> プロンプトを表示し、inBuf をリセットする。 2. interactive_getc() を使用して、EOF またはコマンド終了条件を満たすまで文字を読み取る。

  • **-j モード (UseSemiNewlineNewline)**: セミコロンの後に2つの改行が続くまで入力を継続する。
  • **通常モード**: 改行文字でコマンドを終了する。ただし、改行の直前にバックスラッシュがある場合は継続文字として扱い、改行を破棄する。

    • 3. 入力された文字列の末尾に \0 を付与する。 4. EchoQuery が真の場合、標準出力にクエリ内容をエコーバックする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/postgres.c 内のメインループ（PostgresMain 相当の処理フロー内）。
  • **呼び出し先**: interactive_getc（文字取得）、resetStringInfo / appendStringInfoChar（バッファ操作）。

関連データ構造

  • StringInfo: PostgreSQL の可変長文字列バッファ構造体。

失敗モードと境界条件

  • **EOF**: 入力開始時に EOF を検知し、バッファが空の場合は EOF を返して終了を通知する。
  • **バックスラッシュ継続**: 改行の直前に \ がある場合、改行をコマンド終了とみなさず入力を継続する。

保守担当者向けメモ

  • 本関数は対話型セッション専用である。
  • UseSemiNewlineNewline フラグの状態により、コマンド終了判定ロジックが分岐するため、変更時は両モードの挙動を確認すること。
