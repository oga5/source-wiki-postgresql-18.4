customer

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c:34

目的

ECPG（Embedded SQL for C）プリプロセッサの回帰テストにおいて、構造体ポインタをホスト変数として扱う際のデータマッピングおよびメモリレイアウトの正当性を検証するために定義されています。

シグネチャまたは形状

typedef struct {
  struct varchar_1 { int len; char arr[50]; } name;
  int phone;
} customer;

入力と出力

  • **入力**: SQLクエリの結果セットを格納するためのホスト変数として使用されます。
  • **出力**: ECPGdo 関数を介してフェッチされたデータが、この構造体のメンバ（name および phone）に書き込まれます。

前提条件と不変条件

  • 構造体メンバ name は、ECPGが期待する len（長さ）と arr（データ配列）を持つ varchar 型の形式に従う必要があります。
  • 構造体へのポインタとしてメモリ確保（malloc）を行う際、sizeof(customer) を用いて適切なメモリサイズが指定されている必要があります。

アルゴリズムまたは振る舞い

1. malloc により customer 構造体の配列領域をヒープ上に確保します。 2. ECPGdo 関数実行時、この構造体へのポインタを介してデータベースから取得した値が各メンバにマッピングされます。 3. 取得されたデータは、標準出力（printf）を通じて期待値との比較検証に供されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c 内のメイン処理フロー。
  • **呼び出し先**: ECPGdo 関数（SQL実行およびデータフェッチ用）。
  • **参照元**: src/interfaces/ecpg/test/expected/compat_informix-test_informix2.c（printf 内での参照）。

関連データ構造

  • varchar_1: customer 構造体内部で定義されているネストされた構造体。
  • cust_ind: SQLのNULL値判定に使用されるインジケータ構造体。

失敗モードと境界条件

  • malloc によるメモリ確保失敗時、プログラムは異常終了する可能性があります。
  • varchar 型の len メンバが 50 を超えるデータを受信した場合、バッファオーバーフローやデータ切り捨てのリスクがあります。

保守担当者向けメモ

  • 本構造体はプリプロセッサのテスト用生成コードの一部です。直接編集せず、プリプロセッサの挙動変更が必要な場合は .pgc ソースを確認してください。
  • 構造体のメンバ構成を変更する場合、ECPGdo 呼び出し時の型指定（ECPGt_varchar 等）との整合性に注意してください。
