astreamer_archive_context

定義

  • 種別: enum
  • 定義箇所: src/include/fe_utils/astreamer.h:62

目的

アーカイブ処理（tar等）のストリームパイプラインにおいて、現在処理中のデータがアーカイブ構造のどの部分に該当するかを識別するために使用されます。

シグネチャまたは形状

typedef enum
{
	ASTREAMER_UNKNOWN,
	ASTREAMER_MEMBER_HEADER,
	ASTREAMER_MEMBER_CONTENTS,
	ASTREAMER_MEMBER_TRAILER,
	ASTREAMER_ARCHIVE_TRAILER,
} astreamer_archive_context;

入力と出力

  • **入力**: アーカイブストリームの解析状態。
  • **出力**: 以下のいずれかの状態値:
  • ASTREAMER_UNKNOWN: 未定義または初期状態。
  • ASTREAMER_MEMBER_HEADER: アーカイブメンバーのヘッダー部分。
  • ASTREAMER_MEMBER_CONTENTS: アーカイブメンバーのデータ本体。
  • ASTREAMER_MEMBER_TRAILER: アーカイブメンバーのトレーラー部分。
  • ASTREAMER_ARCHIVE_TRAILER: アーカイブ全体の終了部分。

前提条件と不変条件

  • ストリーム処理の各コンポーネントは、このコンテキスト値に基づいてデータの解釈や処理の切り替えを行う必要があります。

アルゴリズムまたは振る舞い

  • ストリーム処理のパイプラインにおいて、データが流れる際に現在のコンテキストを伝播させます。各コンポーネント（astreamer）は、この値を受け取り、現在のデータチャンクがアーカイブのどのセクションに属するかを判断して処理を分岐させます。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下のファイル内の関数がこの型を引数として受け取っています。
  • src/bin/pg_basebackup/astreamer_inject.c
  • src/bin/pg_verifybackup/astreamer_verify.c
  • src/fe_utils/astreamer_file.c
  • src/fe_utils/astreamer_gzip.c
  • **呼び出し先**: 列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • astreamer: このコンテキストを処理する基底構造体。
  • astreamer_member: アーカイブ内のファイルメタデータを保持する構造体。

失敗モードと境界条件

  • ASTREAMER_UNKNOWN が予期せず継続する場合、パイプラインの解析ロジックに不整合が生じている可能性があります。

保守担当者向けメモ

  • 本列挙型は、astreamer パイプラインの各ステージ間で状態を共有するための重要な指標です。新しいストリーム処理コンポーネントを追加する際は、このコンテキスト値に応じた適切な処理分岐を実装してください。
