DatumGetInetPP

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/inet.h:122

目的

PostgreSQLの Datum 型として渡されたネットワークアドレス（inet 型）を、メモリ上の inet 構造体ポインタへ変換（デトースト）します。

シグネチャまたは形状

static inline inet *DatumGetInetPP(Datum X)

入力と出力

  • **入力**: Datum X （inet 型のデータ）。
  • **出力**: inet * （展開済みまたは直接参照可能な inet 構造体へのポインタ）。

前提条件と不変条件

  • 入力 Datum X は、有効な inet 型のデータである必要があります。
  • PG_DETOAST_DATUM_PACKED マクロを使用しているため、1バイトヘッダーまたは4バイトヘッダーのいずれの形式であっても適切に処理されます。

アルゴリズムまたは振る舞い

1. 引数 X に対して PG_DETOAST_DATUM_PACKED を実行します。 2. 必要に応じてデータの展開（detoast）を行い、結果を inet * 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network.c
  • src/backend/utils/adt/network_gist.c
  • src/backend/utils/adt/network_selfuncs.c
  • src/backend/utils/adt/network_spgist.c
  • **呼び出し先**: PG_DETOAST_DATUM_PACKED マクロ（fmgrインターフェースの一部）。

関連データ構造

  • inet: ネットワークアドレスを保持する構造体。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • 入力が inet 型として不正な場合、PG_DETOAST_DATUM_PACKED の内部でエラーが発生する可能性があります。
  • メモリ不足時には、展開処理に伴い palloc が失敗する可能性があります。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、パフォーマンスを考慮して inet 型の属性アクセス前に頻繁に使用されます。
  • inet 型の内部表現（1バイト/4バイトヘッダー）の変更があった場合は、PG_DETOAST_DATUM_PACKED の挙動と整合性が取れているか確認してください。
