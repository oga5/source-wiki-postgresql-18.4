internal_yylex

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_scanner.c:338

目的

PL/pgSQL スキャナの低レベルなトークン取得関数。コアの字句解析器（core_yylex）をラップし、PL/pgSQL 特有のトークン変換やプッシュバック処理を行う。

シグネチャまたは形状

static int internal_yylex(TokenAuxData *auxdata, yyscan_t yyscanner)

入力と出力

  • **入力**:
  • auxdata: トークンの値や位置情報を格納する TokenAuxData 構造体へのポインタ。
  • yyscanner: PostgreSQL コアの字句解析器の状態を保持するスキャナハンドル。
  • **出力**:
  • 戻り値: 取得したトークンコード（int）。

前提条件と不変条件

  • yyextra（スキャナの拡張状態）が初期化されていること。
  • auxdata は呼び出し元によってメモリが確保されていること。

アルゴリズムまたは振る舞い

1. **プッシュバック確認**: yyextra->num_pushbacks が 0 より大きい場合、スタックから以前にプッシュバックされたトークンと auxdata を復元して返す。 2. **コア解析**: スタックが空の場合、core_yylex を呼び出してコアの字句解析器からトークンを取得する。 3. **トークン変換**:

  • Op（演算子）として認識された場合、<< を LESS_LESS、>> を GREATER_GREATER、# を # に変換する。
  • PARAM として認識された場合、yytext を pstrdup で複製して auxdata->lval.str に格納する。

    • 4. **メタデータ計算**: auxdata->lloc を基に auxdata->leng（トークン長）を算出する。

呼び出し元と呼び出し先

  • **呼び出し元**: plpgsql_yylex および pl_scanner.c 内のトークン先読み処理（plpgsql_peek 等）。
  • **呼び出し先**: core_yylex（PostgreSQL コアの字句解析関数）。

関連データ構造

  • TokenAuxData: トークンの値、位置、長さを保持する構造体。
  • yyscan_t: コアの字句解析器の状態。
  • yyextra: plpgsql_yy_extra_type 型の構造体。プッシュバックスタックを保持する。

失敗モードと境界条件

  • **プッシュバック上限**: MAX_PUSHBACKS（4）を超えるプッシュバックはサポートされていない。
  • **メモリ**: PARAM トークン処理時の pstrdup 失敗時にメモリ不足エラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は core_yylex の結果を直接加工するため、コア側のトークン定義変更が影響する可能性がある。
  • yytext の長さを計算する際、scanbuf のオフセット計算が正しく行われていることを前提としている。
