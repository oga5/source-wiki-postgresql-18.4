WalSnd

定義

  • **種別**: struct
  • **定義箇所**: src/include/replication/walsender_private.h:41

目的

PostgreSQLのレプリケーションシステムにおいて、個々のWAL Senderプロセスの実行状態、スタンバイからのフィードバック情報、および同期レプリケーションの優先順位を共有メモリ上で管理する。

シグネチャまたは形状

typedef struct WalSnd {
    pid_t           pid;
    WalSndState     state;
    XLogRecPtr      sentPtr;
    bool            needreload;
    XLogRecPtr      write;
    XLogRecPtr      flush;
    XLogRecPtr      apply;
    TimeOffset      writeLag;
    TimeOffset      flushLag;
    TimeOffset      applyLag;
    int             sync_standby_priority;
    slock_t         mutex;
    TimestampTz     replyTime;
    ReplicationKind kind;
} WalSnd;

入力と出力

  • **入力**: スタンバイサーバーからのフィードバックメッセージ（LSN進捗、ラグ情報、受信時刻）。
  • **出力**: 同期レプリケーションの待機キュー管理や、レプリケーション状態の監視に使用される共有メモリ上の状態値。

前提条件と不変条件

  • pid が 0 の場合、そのWAL Senderは非アクティブとみなされる。
  • 構造体内の共有変数へのアクセスは、原則として mutex（スピンロック）による保護が必要。ただし、プロセス自身が書き込むフィールドについては、特定の条件下でロックなしの読み取りが許容される設計となっている。

アルゴリズムまたは振る舞い

1. **状態追跡**: WalSndState を通じてプロセスのライフサイクル（STARTUP, STREAMING等）を保持する。 2. **フィードバック更新**: スタンバイから受信した write/flush/apply LSNおよびラグ情報を更新する。 3. **同期管理**: sync_standby_priority を用いて同期スタンバイの優先順位を管理する。 4. **排他制御**: mutex を用いて、複数のプロセス間での共有変数への同時アクセスを制御する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/walsender.c: MyWalSnd を介して自身の状態を更新・参照。
  • src/backend/replication/syncrep.c: 同期レプリケーションの待機キュー管理のために参照。
  • **呼び出し先**:
  • 本構造体はデータ構造であり、関数を直接呼び出すことはない。操作は walsender.c 等の実装コードによって行われる。

関連データ構造

  • WalSndState: プロセスの状態を定義する列挙型。
  • WalSndCtlData: 全WAL Senderを統括するグローバルな共有メモリ構造体。
  • SyncRepQueue: 同期レプリケーションの待機キュー。

失敗モードと境界条件

  • write/flush/apply LSNは、スタンバイから値が提供されていない場合、無効な値となる可能性がある。
  • writeLag/flushLag/applyLag は、計測不能な場合に -1 が設定される。
  • sync_standby_priority が 0 の場合、そのスタンバイは同期レプリケーションのリストに含まれていないことを示す。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されるため、メンバーの追加や変更は共有メモリレイアウトに影響を与える。
  • pid や needreload などのフィールドはスピンロックが必須である一方、プロセス自身が更新するフィールドのロック要否については、src/include/replication/walsender_private.h のコメントおよび walsender.c の実装を十分に確認すること。
