InstrEndLoop

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/executor/instrument.c:139

目的

実行計画ノードの1実行サイクル（ループ）における統計情報を累積し、次回のループ実行に向けて計測状態をリセットします。

シグネチャまたは形状

void InstrEndLoop(Instrumentation *instr)

入力と出力

  • **入力**: Instrumentation *instr（計測対象の統計構造体へのポインタ）
  • **出力**: なし（副作用として instr が指す構造体の値を更新）

前提条件と不変条件

  • instr->running が true であること（false の場合は何もせず終了）。
  • instr->starttime がゼロであること（ノードが実行中であるとみなされる場合はエラー elog(ERROR) を発生させる）。

アルゴリズムまたは振る舞い

1. **実行状態の確認**: instr->running が偽であれば即座に終了。 2. **実行中チェック**: instr->starttime がゼロでない場合、ノードが実行中であるとしてエラーを投げる。 3. **統計の累積**:

  • instr->counter から経過時間を取得し、instr->total に加算。
  • instr->firsttuple を instr->startup に加算。
  • instr->tuplecount を instr->ntuples に加算。
  • instr->nloops を 1 インクリメント。

    • 4. **リセット**:

  • instr->running を false に設定。
  • instr->starttime および instr->counter をゼロに初期化。
  • instr->firsttuple および instr->tuplecount を 0 にリセット。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/auto_explain/auto_explain.c

関連データ構造

  • Instrumentation: 実行計画ノードの統計情報を保持する構造体。

失敗モードと境界条件

  • InstrEndLoop が実行中のノードに対して呼び出された場合、elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数は、ノードが複数回実行されるループ構造を持つ場合に、各サイクルの統計を親ノードや最終結果へ集約するために不可欠です。
  • 統計の累積は Instrumentation 構造体内のフィールドに対して行われるため、並列クエリ実行時には InstrAccumParallelQuery 等による別途集約が必要となる点に注意してください。
