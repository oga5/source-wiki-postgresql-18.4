BtreeLastVisibleEntry

定義

  • 種別: struct
  • 定義箇所: contrib/amcheck/verify_nbtree.c:156

目的

B-Tree インデックスの検証プロセスにおいて、最後に確認された可視なタプル（またはポストイングリスタの要素）の情報を保持するために使用されます。

シグネチャまたは形状

typedef struct BtreeLastVisibleEntry
{
	BlockNumber blkno;			/* インデックスブロック番号 */
	OffsetNumber offset;		/* インデックスブロック内のオフセット */
	int			postingIndex;	/* ポスティングリスト内の番号 (-1 は非重複排除タプル) */
	ItemPointer tid;			/* ヒープタプル ID */
} BtreeLastVisibleEntry;

入力と出力

本構造体は関数間で状態を渡すためのデータコンテナとして機能します。初期化時には InvalidBlockNumber や InvalidOffsetNumber を用いて無効値で初期化されます（contrib/amcheck/verify_nbtree.c:1246）。

前提条件と不変条件

  • postingIndex が -1 の場合、そのタプルは重複排除（deduplication）されていないことを示します。
  • tid はヒープ内のタプル位置を指します。

アルゴリズムまたは振る舞い

インデックスの走査中に、直前に処理した可視タプルの情報を記録し、後続のタプルとの比較や整合性チェックの基準として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_nbtree.c 内の検証ロジック（行 1246 で初期化、行 194, 872, 913 で引数として参照）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • BtreeCheckState: インデックス検証の全体状態を管理する構造体。

失敗モードと境界条件

  • tid が NULL の状態で参照される場合、呼び出し側のロジックに依存します。
  • インデックスが破損している場合、保持される blkno や offset が無効な領域を指す可能性があります。

保守担当者向けメモ

  • 本構造体は verify_nbtree.c 内の特定の検証関数間で状態を共有するために使用されています。変更を加える際は、この構造体を参照しているすべての関数（行 194, 872, 913 等）の整合性を確認してください。
