RBTOrderControl

定義

  • 種別: enum
  • 定義箇所: src/include/lib/rbtree.h:35

目的

赤黒木（Red-Black Tree）の走査順序を制御するための列挙型です。

シグネチャまたは形状

typedef enum RBTOrderControl
{
    LeftRightWalk,  /* inorder: left child, node, right child */
    RightLeftWalk   /* reverse inorder: right, node, left */
} RBTOrderControl;

入力と出力

  • 入力: rbt_begin_iterate 関数の引数として使用されます。
  • 出力: 走査の方向（昇順または降順）を決定します。

前提条件と不変条件

  • LeftRightWalk は左部分木、ノード、右部分木の順（昇順）で走査します。
  • RightLeftWalk は右部分木、ノード、左部分木の順（降順）で走査します。

アルゴリズムまたは振る舞い

rbt_begin_iterate に渡されることで、イテレータが木を辿る際の再帰的またはスタックベースの走査順序を切り替えます。

呼び出し元と呼び出し先

  • 呼び出し元: rbt_begin_iterate を呼び出すコード（src/backend/lib/rbtree.c:802 等）。
  • 呼び出し先: 本シンボル自体は列挙型であり、関数ではないため呼び出し先はありません。

関連データ構造

  • RBTreeIterator: 走査状態を保持する構造体。
  • RBTree: 走査対象となる赤黒木本体。

失敗モードと境界条件

  • 定義された値（LeftRightWalk, RightLeftWalk）以外の値を渡した場合の動作は未定義です。

保守担当者向けメモ

  • 本列挙型は rbt_begin_iterate のインターフェースの一部として固定されています。走査順序の追加が必要な場合は、この列挙型と rbt_iterate の内部ロジックの両方を更新する必要があります。
