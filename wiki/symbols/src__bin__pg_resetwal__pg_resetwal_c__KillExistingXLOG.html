KillExistingXLOG

定義

  • 種別: function
  • 定義箇所: src/bin/pg_resetwal/pg_resetwal.c:1005

目的

PostgreSQL データディレクトリ内の pg_wal（または pg_xlog）ディレクトリから、既存の WAL セグメントファイルおよび部分的な WAL ファイルをすべて削除し、WAL 領域をクリーンアップする。

シグネチャまたは形状

static void KillExistingXLOG(void)

入力と出力

  • 入力: なし（グローバルな XLOGDIR を対象とする）
  • 出力: なし（成功時はディレクトリ内の対象ファイルを削除。失敗時は pg_fatal によりプロセスを終了）

前提条件と不変条件

  • XLOGDIR が存在するディレクトリであること。
  • 実行ユーザーが対象ファイルに対する削除権限を持っていること。

アルゴリズムまたは振る舞い

1. opendir(XLOGDIR) を呼び出し、WAL ディレクトリを開く。失敗した場合は致命的エラーとする。 2. readdir を使用してディレクトリ内の全エントリを走査する。 3. 各エントリのファイル名が IsXLogFileName または IsPartialXLogFileName に合致する場合、フルパスを構築して unlink を実行する。 4. 削除に失敗した場合は pg_fatal を呼び出し、処理を中断する。 5. ディレクトリの走査完了後、closedir を実行してディレクトリを閉じる。

呼び出し元と呼び出し先

  • 呼び出し元: main 関数 (src/bin/pg_resetwal/pg_resetwal.c:522)
  • 呼び出し先: opendir, readdir, closedir, unlink, pg_fatal, IsXLogFileName, IsPartialXLogFileName

関連データ構造

  • DIR: ディレクトリ操作用構造体
  • struct dirent: ディレクトリ内のファイルエントリ情報

失敗モードと境界条件

  • ディレクトリが開けない場合、または閉じられない場合に pg_fatal で終了する。
  • ファイルの削除（unlink）に失敗した場合、pg_fatal で終了する。
  • readdir 中にエラーが発生した場合、pg_fatal で終了する。

保守担当者向けメモ

  • 本関数は破壊的な操作を行うため、呼び出し前に必ずデータディレクトリのバックアップが取得されていることを前提とする。
  • XLOGDIR の定義は access/xlog_internal.h 等に依存しているため、WAL ディレクトリの構成変更時には注意が必要である。
