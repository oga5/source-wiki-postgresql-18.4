GenerationReset

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/generation.c:282

目的

GenerationContext 内の全メモリブロックを走査し、KeeperBlock 以外のブロックを解放または初期化することで、コンテキストを初期状態に戻す。

シグネチャまたは形状

void GenerationReset(MemoryContext context)

入力と出力

  • **入力**: MemoryContext context（GenerationContext としてキャストして使用）
  • **出力**: なし（副作用としてコンテキスト内のメモリが解放・リセットされる）

前提条件と不変条件

  • context は GenerationIsValid を満たす有効な GenerationContext であること。
  • 実行後、blocks リストには KeeperBlock のみが残り、リストの要素数が 1 であることが保証される。

アルゴリズムまたは振る舞い

1. MEMORY_CONTEXT_CHECKING が定義されている場合、GenerationCheck を呼び出しメモリの整合性を検証する。 2. set->freeblock を NULL にリセットする。 3. set->blocks リストを走査する。

  • KeeperBlock の場合は GenerationBlockMarkEmpty を呼び出し、空状態にリセットする。
  • それ以外のブロックは GenerationBlockFree を呼び出して解放する。

    • 4. 現在の作業ブロック (set->block) を KeeperBlock に設定する。 5. 次回のブロック割り当てサイズ (nextBlockSize) を初期値 (initBlockSize) に戻す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c（MemoryContextMethods の reset フィールドとして登録）
  • **呼び出し先**:
  • GenerationCheck（条件付き）
  • GenerationBlockMarkEmpty
  • GenerationBlockFree
  • KeeperBlock（マクロ）

関連データ構造

  • GenerationContext: メモリ管理のメイン構造体。
  • GenerationBlock: 管理単位となるメモリブロック。
  • dlist: ブロック管理用の双方向リスト。

失敗モードと境界条件

  • GenerationIsValid による検証失敗時にアサーションエラーが発生する。
  • KeeperBlock 以外のブロックが存在しない場合、リストの不変条件を維持したまま正常に終了する。

保守担当者向けメモ

  • 本関数は GenerationContext のリセット処理を担う。freeblock を先に NULL 化するのは、GenerationBlockFree が freeblock を解放対象と見なさないようにするための設計である。
  • KeeperBlock はコンテキストの寿命と同期するため、リセット時も解放されず再利用される点に注意すること。
