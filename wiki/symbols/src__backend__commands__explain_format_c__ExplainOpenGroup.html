ExplainOpenGroup

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_format.c:278

目的

EXPLAIN コマンドの出力において、階層構造（オブジェクトや配列）の開始を定義し、フォーマットに応じた適切な構文（タグ、括弧、インデント）を生成する。

シグネチャまたは形状

void ExplainOpenGroup(const char *objtype, const char *labelname, bool labeled, ExplainState *es)

入力と出力

  • objtype: グループの型名（XML出力時のタグ名などに使用）。
  • labelname: JSON/YAML出力時のキー名。NULL の場合、ラベルなしの要素として扱われる。
  • labeled: true の場合、JSONでは { を、false の場合は [ を使用する。
  • es: ExplainState 構造体。出力バッファ、現在のフォーマット、インデントレベル、グループ状態を保持する。

前提条件と不変条件

  • es->format に基づいて適切なフォーマット処理が選択される。
  • ExplainOpenGroup を呼び出した後は、対応する ExplainCloseGroup を呼び出す必要がある。
  • JSON/YAML形式では、es->grouping_stack が階層管理のために更新される。

アルゴリズムまたは振る舞い

es->format の値に応じて以下の処理を行う：

1. **TEXT**: 何も行わない。 2. **XML**: ExplainXMLTag を呼び出し、インデントレベルを増やす。 3. **JSON**:

  • ExplainJSONLineEnding で行末処理を行う。
  • labelname があれば出力し、labeled に応じて { または [ を書き込む。
  • grouping_stack に新しい階層を追加し、インデントを増やす。

    • 4. **YAML**:

  • ExplainYAMLLineStarting を呼び出す。
  • labelname があればキーとして出力し、なければ - を出力する。
  • grouping_stack を更新し、インデントを増やす。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/explain.c（Query, Planning, Settings, Triggers, JIT, Options, Timing, Generation, Serialization 等の出力時）および contrib/pg_overexplain/pg_overexplain.c。
  • **呼び出し先**: ExplainXMLTag, ExplainJSONLineEnding, ExplainYAMLLineStarting, appendStringInfo 系関数、lcons_int。

関連データ構造

  • ExplainState: 出力状態を管理するメイン構造体。
  • grouping_stack: JSON/YAMLのネスト状態を追跡する整数リスト。

失敗モードと境界条件

  • labelname が NULL の場合、JSON/YAMLではラベルなしの要素として処理される。
  • labeled フラグの指定が誤っていると、JSONの構造（オブジェクトか配列か）が不正になる可能性がある。

保守担当者向けメモ

  • 新しいフォーマットを追加する場合は、switch 文に新しい case を追加する必要がある。
  • grouping_stack の操作は、ExplainCloseGroup と対になるように厳密に管理すること。
