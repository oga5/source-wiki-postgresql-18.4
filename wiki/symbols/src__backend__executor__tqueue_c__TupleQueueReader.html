TupleQueueReader

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/tqueue.c:43

目的

並列クエリ実行において、共有メモリキュー（shm_mq）からタプルを非同期または同期的に読み出すための状態を保持する構造体です。

シグネチャまたは形状

struct TupleQueueReader
{
    shm_mq_handle *queue; /* 受信元の共有メモリキューハンドル */
};

入力と出力

  • **入力**: 共有メモリ上の shm_mq（queue メンバ経由）。
  • **出力**: TupleQueueReaderNext 関数を通じて、共有メモリからデシリアライズされたタプルを提供します。

前提条件と不変条件

  • 読み取ったタプルは共有メモリ上のポインタを直接指すため、次の読み出し呼び出しまでの間のみ有効です。
  • shm_mq のライフサイクル管理（接続・切断）と同期して使用される必要があります。

アルゴリズムまたは振る舞い

  • TupleQueueReaderNext を介して shm_mq_receive を呼び出し、共有メモリからタプルを抽出します。
  • nowait 引数を用いたノンブロッキング読み出しをサポートしており、並列実行時の待機制御に使用されます。
  • DestroyTupleQueueReader により、関連する共有メモリキューのデタッチ処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execParallel.c
  • src/backend/executor/nodeGather.c
  • src/backend/executor/nodeGatherMerge.c
  • **呼び出し先**:
  • shm_mq インターフェース（shm_mq_receive 等）。

関連データ構造

  • shm_mq_handle: 共有メモリキューの通信ハンドル。
  • TQueueDestReceiver: 送信側の対応構造体。

失敗モードと境界条件

  • 共有メモリキューが空の場合、nowait 指定時は即座に制御を戻します。
  • 読み出し時に共有メモリ上のタプル形式（MinimalTuple）が破損している場合、デシリアライズ処理でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は shm_mq のラッパーです。タプルのシリアライズ/デシリアライズは MinimalTuple 形式に依存しています。
  • 構造体のインスタンスは CreateTupleQueueReader で生成されます。
