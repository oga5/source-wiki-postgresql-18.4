PGP_PKT_TYPE

定義

  • 種別: enum
  • 定義箇所: contrib/pgcrypto/pgp.h:44

目的

OpenPGP (RFC 4880) 規格で定義されるパケットタイプを識別するための列挙型です。PGP データの解析および生成プロセスにおいて、各パケットの役割を特定するために使用されます。

シグネチャまたは形状

enum PGP_PKT_TYPE {
    PGP_PKT_RESERVED = 0,
    PGP_PKT_PUBENCRYPTED_SESSKEY = 1,
    PGP_PKT_SIGNATURE = 2,
    PGP_PKT_SYMENCRYPTED_SESSKEY = 3,
    PGP_PKT_SECRET_KEY = 5,
    PGP_PKT_PUBLIC_KEY = 6,
    PGP_PKT_SECRET_SUBKEY = 7,
    PGP_PKT_COMPRESSED_DATA = 8,
    PGP_PKT_SYMENCRYPTED_DATA = 9,
    PGP_PKT_MARKER = 10,
    PGP_PKT_LITERAL_DATA = 11,
    PGP_PKT_TRUST = 12,
    PGP_PKT_USER_ID = 13,
    PGP_PKT_PUBLIC_SUBKEY = 14,
    PGP_PKT_USER_ATTR = 17,
    PGP_PKT_SYMENCRYPTED_DATA_MDC = 18,
    PGP_PKT_MDC = 19,
    PGP_PKT_PRIV_61 = 61
};

入力と出力

  • 入力: なし（定数定義）
  • 出力: なし（定数定義）

前提条件と不変条件

  • RFC 4880 に準拠したパケット構造を前提としています。
  • PGP_PKT_PRIV_61 は GnuPG の秘密鍵リング (secring) で使用される特殊な値です。

アルゴリズムまたは振る舞い

本シンボル自体はデータ構造の識別子であり、ロジックは持ちません。PGP データのパース処理において、ストリームから読み取られたパケットタグ値と本列挙値を比較することで、後続の処理（復号、署名検証、データ展開など）を分岐させるために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは、具体的な関数レベルの呼び出し元は特定できません。
  • 呼び出し先: なし。

関連データ構造

  • PGP_Context: PGP 処理のセッション状態を保持する構造体であり、パケット処理の過程で本列挙値が参照されます。

失敗モードと境界条件

  • 未定義のパケットタイプが入力された場合、パース処理側でエラーハンドリングを行う必要があります。
  • PGP_PKT_RESERVED (0) は RFC 上予約されており、通常のデータストリームには現れません。

保守担当者向けメモ

  • 本列挙型は RFC 4880 のパケットタグ定義と密接に関連しています。新しいパケットタイプのサポートを追加する際は、RFC を参照し、既存の列挙値と重複しないように注意してください。
