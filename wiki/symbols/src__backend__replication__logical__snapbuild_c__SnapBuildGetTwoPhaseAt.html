SnapBuildGetTwoPhaseAt

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/snapbuild.c:285

目的

SnapBuild 構造体が保持する、二相コミット（Two-Phase Commit）に関連するトランザクション状態が有効となった WAL 位置（two_phase_at）を取得します。

シグネチャまたは形状

XLogRecPtr SnapBuildGetTwoPhaseAt(SnapBuild *builder)

入力と出力

  • **入力**: builder (SnapBuild *) - スナップショット構築の状態を管理する構造体へのポインタ。
  • **出力**: XLogRecPtr - 二相コミットが有効化された時点を示す WAL のログシーケンス番号（LSN）。

前提条件と不変条件

  • builder は有効な SnapBuild 構造体を指している必要があります。

アルゴリズムまたは振る舞い

builder 構造体内の two_phase_at メンバの値を直接返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/decode.c:728 にて、ctx->snapshot_builder を引数として呼び出されています。
  • **呼び出し先**: 本関数は直接的な関数呼び出しを行わず、構造体のメンバを参照して値を返します。

関連データ構造

  • SnapBuild: 論理デコーディングにおけるスナップショット構築状態を保持するメイン構造体。two_phase_at メンバを保持します。

失敗モードと境界条件

  • builder が NULL の場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は単なるゲッターであり、状態の変更は行いません。
  • 二相コミットのサポートに関連する値であり、論理デコーディングにおけるトランザクションの追跡精度に影響します。
