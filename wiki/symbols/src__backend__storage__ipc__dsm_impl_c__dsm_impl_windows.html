dsm_impl_windows

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/dsm_impl.c:609

目的

Windows環境における動的共有メモリ（DSM）セグメントの作成、アタッチ、デタッチ、および破棄をOSネイティブAPIを用いて実行する。

シグネチャまたは形状

static bool dsm_impl_windows(dsm_op op, dsm_handle handle, Size request_size,
                             void **impl_private, void **mapped_address,
                             Size *mapped_size, int elevel);

入力と出力

  • op: 実行する操作（DSM_OP_CREATE, DSM_OP_ATTACH, DSM_OP_DETACH, DSM_OP_DESTROY）。
  • handle: セグメントを一意に識別するハンドル。
  • request_size: 作成時に要求するメモリサイズ。
  • impl_private: 実装固有のデータ（Windowsでは HANDLE）を格納するポインタ。
  • mapped_address: マッピングされたメモリの開始アドレス。
  • mapped_size: マッピングされたメモリの実際のサイズ。
  • elevel: エラー発生時のログレベル。
  • 戻り値: 成功時に true、失敗時に false を返す。

前提条件と不変条件

  • SEGMENT_NAME_PREFIX を用いた名前付き共有メモリセグメントとして管理される。
  • DSM_OP_DETACH または DSM_OP_DESTROY 時、Windowsの仕様により参照がなくなると自動的にオブジェクトが破棄されるため、両操作は同等に扱われる。
  • VirtualQuery を使用して、マッピングされた領域のサイズを検証する。

アルゴリズムまたは振る舞い

1. **名前解決**: handle からセグメント名を生成する。 2. **終了処理**: DETACH または DESTROY の場合、UnmapViewOfFile でメモリを解放し、CloseHandle でハンドルを閉じる。 3. **作成/アタッチ**:

  • CREATE の場合: CreateFileMapping を呼び出し、ページファイル上にセグメントを作成する。既に存在する場合はエラーを返す。
  • ATTACH の場合: OpenFileMapping を呼び出し、既存のセグメントを開く。

    • 4. **マッピング**: MapViewOfFile を使用してプロセスのアドレス空間にセグメントをマッピングする。 5. **サイズ取得**: VirtualQuery を実行し、mapped_size を更新する。 6. **状態保存**: impl_private に hmap ハンドルを格納する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/dsm_impl.c 内の dsm_impl_op 関数から呼び出される。
  • 呼び出し先: Windows API (CreateFileMapping, OpenFileMapping, MapViewOfFile, UnmapViewOfFile, CloseHandle, VirtualQuery) を直接呼び出す。

関連データ構造

  • dsm_op: 操作種別を定義する列挙型。
  • dsm_handle: セグメント識別子。
  • HANDLE: Windowsのカーネルオブジェクトハンドル（impl_private に格納）。

失敗モードと境界条件

  • CreateFileMapping が ERROR_ALREADY_EXISTS または ERROR_ACCESS_DENIED を返した場合、作成は失敗する。
  • マッピングやハンドル操作に失敗した場合、_dosmaperr を通じてシステムエラーを errno に変換し、ereport で通知する。
  • 失敗時は、既に確保したリソース（ハンドルやマッピング）を適切に解放するロールバック処理を行う。

保守担当者向けメモ

  • Windowsのカーネルオブジェクトの生存期間管理は特殊であるため、pin/unpin 関数との連携に注意すること。
  • GetSharedMemName に関連する既知の問題が解決された場合、セグメントの命名規則を見直す可能性がある。
