ObjectsInPublicationToOids

定義

  • 種別: function
  • 定義箇所: src/backend/commands/publicationcmds.c:183

目的

パブリケーション定義に含まれるオブジェクト指定リスト（PublicationObjSpec）を解析し、テーブルのリストとスキーマのOIDリストに分離・変換します。

シグネチャまたは形状

static void ObjectsInPublicationToOids(List *pubobjspec_list, ParseState *pstate, List **rels, List **schemas)

入力と出力

  • **入力**:
  • pubobjspec_list: パブリケーションに追加するオブジェクトのリスト。
  • pstate: 構文解析状態（現在は未使用）。
  • **出力**:
  • rels: 抽出されたテーブルオブジェクトのリスト（*rels に追加）。
  • schemas: 抽出されたスキーマのOIDリスト（*schemas に追加）。

前提条件と不変条件

  • pubobjspec_list が NIL の場合、何も行わずに即座に復帰します。
  • スキーマ指定時に重複がある場合、list_append_unique_oid により重複が排除されます。

アルゴリズムまたは振る舞い

1. pubobjspec_list を走査し、各要素の pubobjtype に基づいて分岐します。 2. PUBLICATIONOBJ_TABLE: テーブル情報を rels リストに追加します。 3. PUBLICATIONOBJ_TABLES_IN_SCHEMA: 指定されたスキーマ名を get_namespace_oid でOIDに変換し、schemas リストに追加します。 4. PUBLICATIONOBJ_TABLES_IN_CUR_SCHEMA: fetch_search_path を使用して現在のスキーマを特定し、そのOIDを schemas リストに追加します。検索パスが空の場合はエラーを報告します。 5. 未知のオブジェクトタイプが指定された場合は elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/publicationcmds.c 内の複数箇所（パブリケーション作成・変更処理時）。
  • **呼び出し先**:
  • get_namespace_oid
  • list_append_unique_oid
  • fetch_search_path
  • linitial_oid
  • list_free
  • ereport / elog

関連データ構造

  • PublicationObjSpec: パブリケーション対象オブジェクトの定義構造体。
  • ParseState: 構文解析状態を保持する構造体。

失敗モードと境界条件

  • PUBLICATIONOBJ_TABLES_IN_CUR_SCHEMA 指定時に検索パスが空の場合、ERRCODE_UNDEFINED_SCHEMA でエラー終了します。
  • 未定義のオブジェクトタイプが渡された場合、elog により異常終了します。

保守担当者向けメモ

  • 本関数はパブリケーションの定義変更時に呼び出されるユーティリティです。スキーマの重複排除を行っているため、呼び出し側でリストを初期化してから渡す必要があります。
