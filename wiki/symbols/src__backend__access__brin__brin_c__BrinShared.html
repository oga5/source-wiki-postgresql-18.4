BrinShared

定義

  • 種別: struct
  • 定義箇所: src/backend/access/brin/brin.c:57

目的

BRIN インデックスの並列構築時において、リーダープロセスとワーカープロセス間で共有される状態を保持します。インデックス構築のメタデータ、進捗管理、および構築結果の集約に使用されます。

シグネチャまたは形状

typedef struct BrinShared {
    Oid         heaprelid;
    Oid         indexrelid;
    bool        isconcurrent;
    BlockNumber pagesPerRange;
    int         scantuplesortstates;
    int64       queryid;
    ConditionVariable workersdonecv;
    slock_t     mutex;
    int         nparticipantsdone;
    double      reltuples;
    double      indtuples;
} BrinShared;

入力と出力

  • **入力**: 並列構築開始時にリーダープロセスによって共有メモリ上に割り当てられ、初期化されます。
  • **出力**: 構築終了時に、ワーカープロセスによって更新された統計情報（reltuples, indtuples 等）がリーダーによって読み取られます。

前提条件と不変条件

  • mutex は、heapdesc より前のフィールドを除く、並列構築中に変更される状態（nparticipantsdone, reltuples, indtuples）を保護します。
  • workersdonecv は、すべてのワーカーが終了したことをリーダーが確認するために使用されます。
  • BrinShared の直後には ParallelTableScanDescData が配置されるメモリレイアウトが想定されています。

アルゴリズムまたは振る舞い

1. **初期化**: リーダープロセスが shm_toc_allocate を用いて共有メモリを確保し、インデックス構築の基本設定を書き込みます。 2. **並列処理**: ワーカープロセスは共有された BrinShared を参照してスキャンを実行します。 3. **同期**: 各ワーカーは終了時に nparticipantsdone を更新し、workersdonecv を通知します。 4. **集約**: リーダープロセスはすべてのワーカーが完了するのを待ち、共有された統計情報を基にインデックス構築を完了させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin.c 内の並列構築関連関数（_brin_parallel_merge 等）から参照されます。
  • **呼び出し先**: 直接的な関数呼び出し先はありませんが、ConditionVariable を介してプロセス間の同期を行います。

関連データ構造

  • ParallelTableScanDescData: 本構造体の直後に配置される並列スキャン記述子。
  • BrinLeader: 本構造体を保持し、並列構築を統括する構造体。

失敗モードと境界条件

  • BUFFERALIGN によるアライメント調整が必須であり、構造体サイズ計算時に考慮する必要があります。
  • 共有メモリの割り当てに失敗した場合、インデックス構築は開始できません。

保守担当者向けメモ

  • 本構造体は並列構築専用です。フィールドを追加する際は、mutex による保護が必要な「可変状態」か、構築中に不変な「設定値」かを明確に区別してください。
  • ParallelTableScanDescData を直後に配置するメモリレイアウトに依存しているため、構造体のサイズ変更や配置変更には注意が必要です。
