size_box

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistproc.c:67

目的

与えられた BOX 型の面積を計算します。GiSTインデックスにおけるペナルティ計算（ノード追加時のコスト評価）などで使用されます。

シグネチャまたは形状

static float8 size_box(const BOX *box)

入力と出力

  • **入力**: const BOX *box （計算対象の矩形）
  • **出力**: float8 （矩形の面積。無効な矩形の場合は 0.0、NaNが含まれる場合は無限大を返す）

前提条件と不変条件

  • 入力 box は有効なポインタである必要があります。
  • 矩形の幅または高さが負またはゼロの場合、面積は 0.0 とみなされます。

アルゴリズムまたは振る舞い

1. **ゼロ幅チェック**: box->high.x <= box->low.x または box->high.y <= box->low.y の場合、0.0 を返します。 2. **NaNチェック**: high 座標に NaN が含まれる場合、get_float8_infinity() を返します。 3. **面積計算**: 上記以外の場合、(high.x - low.x) * (high.y - low.y) を計算して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistproc.c 内のペナルティ計算ロジック（例: box_penalty 等の内部処理）から呼び出されます。
  • **呼び出し先**:
  • float8_le, float8_mi, float8_mul, isnan, get_float8_infinity などの浮動小数点演算ユーティリティ。

関連データ構造

  • BOX: src/backend/utils/geo_decls.h で定義される矩形構造体。

失敗モードと境界条件

  • **ゼロまたは負のサイズ**: 幅や高さが負またはゼロの矩形は、面積 0.0 として扱われます。
  • **NaN**: high 座標に NaN が含まれる場合、無限大として処理されます。
  • **オーバーフロー**: 計算結果が float8 の範囲を超える可能性があります。

保守担当者向けメモ

  • 本関数は gistproc.c 内の幾何学的計算の基礎となります。
  • 浮動小数点演算の特性上、NaN や無限大の扱いに注意してください。特に low 座標が NaN である可能性は、先行するチェックにより排除されています。
