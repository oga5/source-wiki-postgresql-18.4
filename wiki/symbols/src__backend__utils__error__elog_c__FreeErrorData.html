FreeErrorData

定義

  • 種別: function
  • 定義箇所: src/backend/utils/error/elog.c:1822

目的

CopyErrorData によって生成された ErrorData 構造体およびその内部で保持されている動的メモリを解放します。

シグネチャまたは形状

void FreeErrorData(ErrorData *edata)

入力と出力

  • **入力**: edata (ErrorData *) - 解放対象となるエラーデータ構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • edata は CopyErrorData 等によって適切に割り当てられた有効なポインタである必要があります。
  • edata が NULL の場合の挙動については、本ソース抜粋からは明示されていません。

アルゴリズムまたは振る舞い

1. FreeErrorDataContents(edata) を呼び出し、ErrorData 構造体内部の文字列フィールド（メッセージ、詳細、ヒントなど）が保持するメモリを解放します。 2. pfree(edata) を呼び出し、ErrorData 構造体自体のメモリ領域を解放します。

呼び出し元と呼び出し先

  • **呼び出し先**: FreeErrorDataContents、pfree
  • **呼び出し元**:
  • contrib/postgres_fdw/connection.c
  • src/backend/replication/logical/reorderbuffer.c
  • src/pl/plpython/plpy_plpymodule.c
  • src/pl/plpython/plpy_spi.c
  • src/pl/tcl/pltcl.c

関連データ構造

  • ErrorData: エラーの詳細情報を保持する構造体。

失敗モードと境界条件

  • edata が指すメモリが既に解放されている場合や、不正なポインタである場合はメモリ破壊を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は CopyErrorData でコピーされたデータのライフサイクルを終了させるために使用されます。エラー処理のメモリ管理において、ErrorContext 内のメモリリークを防ぐために重要です。
