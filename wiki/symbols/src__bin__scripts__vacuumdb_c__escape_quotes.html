escape_quotes

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/vacuumdb.c:473

目的

入力文字列内のシングルクォートをエスケープし、SQL文中で安全に使用可能な形式の文字列を生成します。

シグネチャまたは形状

static char *escape_quotes(const char *src);

入力と出力

  • **入力**: src (エスケープ対象の文字列へのポインタ)
  • **出力**: エスケープ処理済みの文字列を保持する動的に割り当てられたメモリ領域へのポインタ。

前提条件と不変条件

  • 入力 src は有効なヌル終端文字列である必要があります。
  • 戻り値のメモリ領域は、呼び出し元で適切に解放（free）される必要があります。

アルゴリズムまたは振る舞い

1. escape_single_quotes_ascii(src) を呼び出し、エスケープ処理を実行します。 2. 処理結果が NULL（メモリ不足）の場合、pg_fatal を呼び出してプログラムを異常終了させます。 3. 成功した場合は、エスケープ済みの文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: escape_single_quotes_ascii（fe_utils/string_utils.h 由来と推測される）、pg_fatal
  • **呼び出し元**: src/bin/scripts/vacuumdb.c 内の main 関数（vacopts.buffer_usage_limit の設定時）。
  • **注記**: ツールにより src/bin/initdb/initdb.c や src/fe_utils/recovery_gen.c にも同名の関数定義および参照が存在することが示唆されていますが、本ファイル（vacuumdb.c）内での直接的な呼び出し関係は上記のみが確認されています。

関連データ構造

  • なし

失敗モードと境界条件

  • メモリ不足が発生した場合、pg_fatal によりエラーメッセージを出力してプロセスが終了します。

保守担当者向けメモ

  • 本関数は vacuumdb.c 内で static として定義されています。
  • 他のソースファイル（initdb.c 等）にも同名の関数が存在するため、リンク時の衝突を避けるためにファイルスコープに限定されています。変更を加える際は、このファイル内での利用に影響がないか確認してください。
