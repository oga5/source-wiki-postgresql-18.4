CreateOptsFile

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/postmaster.c:4062

目的

Postmasterの起動時に使用されたコマンドライン引数を記録したファイル（postmaster.opts）を作成し、サーバーの起動設定を永続化する。

シグネチャまたは形状

static bool CreateOptsFile(int argc, char *argv[], char *fullprogname)

入力と出力

  • **入力**:
  • argc: コマンドライン引数の数。
  • argv[]: コマンドライン引数の文字列配列。
  • fullprogname: 実行されたプログラムのフルパス。
  • **出力**:
  • bool: ファイル作成および書き込みが成功した場合は true、失敗した場合は false を返す。

前提条件と不変条件

  • カレントディレクトリに postmaster.opts という名前のファイルを作成する権限が必要である。
  • ファイル書き込み中にエラーが発生した場合、ereport(LOG, ...) を通じてログが出力される。

アルゴリズムまたは振る舞い

1. postmaster.opts を書き込みモード（"w"）で開く。 2. 開けない場合はエラーをログ出力し、false を返す。 3. fullprogname をファイルに書き込む。 4. argv に含まれる各引数をスペース区切りで引用符付きでファイルに追記する。 5. ファイルを閉じる。閉じる際にエラーが発生した場合はログ出力し、false を返す。 6. 全工程が正常に完了すれば true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/postmaster.c 内の PostmasterMain 関数（1288行目付近）から呼び出される。
  • **呼び出し先**: 標準ライブラリの fopen, fprintf, fputs, fclose および PostgreSQL の ereport マクロを使用する。

関連データ構造

  • FILE: 標準 C ライブラリのファイルポインタ。

失敗モードと境界条件

  • ファイル作成失敗: ディレクトリの書き込み権限不足やディスクフルにより fopen が失敗する場合。
  • 書き込み失敗: fclose 時にディスク容量不足等でフラッシュが失敗する場合。
  • いずれの失敗時も、PostgreSQL のログシステムを通じてエラーが報告される。

保守担当者向けメモ

  • 本関数は postmaster.opts という固定のファイル名を使用する。
  • 引数の引用符付けは単純な文字列連結で行われるため、特殊文字を含む引数の扱いには注意が必要である。
