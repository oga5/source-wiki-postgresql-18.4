error

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-dyntest.c:146

目的

ECPGテストプログラムにおいて、SQL操作の失敗時にエラー情報を標準出力へ表示し、プロセスを異常終了させるためのユーティリティ関数です。

シグネチャまたは形状

static void error(void);

入力と出力

  • 入力: なし（グローバルな sqlca 構造体を参照）
  • 出力: なし（標準出力への書き出しとプロセス終了）

前提条件と不変条件

  • sqlca 構造体が有効なSQL通信領域として初期化されていること。
  • 本関数が呼び出された時点で、直前のSQL操作が失敗していることが想定されます。

アルゴリズムまたは振る舞い

1. sqlca.sqlcode（エラーコード）および sqlca.sqlerrm.sqlerrmc（エラーメッセージ）を標準出力へフォーマット出力します。 2. exit(1) を呼び出し、ステータスコード1でプログラムを終了します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース抜粋内では特定できません。
  • 呼び出し先: printf（標準ライブラリ）、exit（標準ライブラリ）。

関連データ構造

  • sqlca_t: SQL通信領域。エラーコードやエラーメッセージの取得元として使用されます。

失敗モードと境界条件

  • sqlca.sqlerrm.sqlerrmc が空文字列または不正なメモリを指している場合、printf の出力結果が意図しないものになる可能性があります。
  • プロセス終了を伴うため、本関数呼び出し以降のクリーンアップ処理（記述子の解放など）は実行されません。

保守担当者向けメモ

  • 本関数はテスト用プログラム専用です。実運用コードでのエラーハンドリングには適していません。
  • sqlca の内容に依存しているため、ECPGのプリプロセッサ設定や環境によって出力内容が変化する点に注意してください。
