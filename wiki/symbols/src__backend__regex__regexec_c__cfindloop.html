cfindloop

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexec.c:550

目的

バックリファレンス等を含む複雑な正規表現において、対象文字列内のマッチング候補を探索し、cdissect を用いてサブマッチの妥当性を検証するマッチングエンジンの核心部。

シグネチャまたは形状

static int cfindloop(struct vars *v, struct cnfa *cnfa, struct colormap *cm, struct dfa *d, struct dfa *s, chr **coldp)

入力と出力

  • **入力**:
  • v: マッチング状態を保持する vars 構造体。
  • cnfa, cm: 正規表現のNFAおよびカラーマップ。
  • d: 詳細なマッチングを行うためのDFA。
  • s: 検索用のDFA。
  • coldp: マッチング失敗時の「冷たい開始点（coldstart）」を格納するポインタ。
  • **出力**:
  • int: 成功時は REG_OKAY、失敗時は REG_NOMATCH、エラー時はエラーコードを返す。

前提条件と不変条件

  • d および s は NULL であってはならない。
  • マッチング成功時、v->nmatch > 0 であれば v->pmatch[0] にマッチ範囲が格納される。

アルゴリズムまたは振る舞い

1. **検索範囲の特定**: shortest を使用し、検索用DFA s でマッチ可能な範囲を特定する。 2. **開始位置の反復**: 特定された範囲内で begin 位置を順次移動させる。 3. **終了位置の探索**: shortest または longest を使用し、正規表現のフラグに応じて end 位置を決定する。 4. **サブマッチ検証**: cdissect を呼び出し、ツリー構造に基づいた再帰的な検証を行う。 5. **結果の確定**: cdissect が REG_OKAY を返せばマッチ成功として終了する。失敗した場合は、次の候補位置へ進む。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regexec.c 内の cfind 関数から呼び出される。
  • **呼び出し先**: shortest, longest, cdissect を呼び出す。

関連データ構造

  • vars: マッチングのコンテキスト。
  • dfa: 遅延評価DFAの状態管理。
  • cnfa: コンパイル済みNFA。

失敗モードと境界条件

  • shortest/longest がエラーを返した場合、そのエラーコードを伝播する。
  • マッチする範囲が見つからない場合、REG_NOMATCH を返す。
  • cdissect が REG_NOMATCH 以外を返した場合、そのエラーコードを返す。

保守担当者向けメモ

  • 本関数はバックリファレンスが存在する場合の再帰的探索の起点となる。
  • shorter フラグにより、最短マッチと最長マッチの挙動が切り替わる。
  • coldp はマッチングの効率化（失敗時の再開位置の最適化）のために使用される。
