CreateTransientRelDestReceiver

定義

  • 種別: function
  • 定義箇所: src/backend/commands/matview.c:464

目的

マテリアライズド・ビュー（MV）の更新処理において、クエリ結果を一時テーブルへ流し込むための DestReceiver オブジェクトを生成・初期化する。

シグネチャまたは形状

DestReceiver *CreateTransientRelDestReceiver(Oid transientoid)

入力と出力

  • **入力**: transientoid (Oid) - データを挿入する対象の一時テーブルのOID。
  • **出力**: DestReceiver * - 初期化された DR_transientrel 構造体へのポインタ。

前提条件と不変条件

  • palloc0 を使用してメモリを確保するため、メモリ不足時はエラーが発生する。
  • 返される DestReceiver は、transientrel_receive、transientrel_startup、transientrel_shutdown、transientrel_destroy の各関数ポインタで構成される。

アルゴリズムまたは振る舞い

1. DR_transientrel 構造体分のメモリをゼロクリアして確保する。 2. DestReceiver の各コールバック関数（receiveSlot, rStartup, rShutdown, rDestroy）を、対応する transientrel_* 関数に割り当てる。 3. mydest フィールドを DestTransientRel に設定する。 4. 引数で受け取った transientoid を構造体に格納し、呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/matview.c:329
  • src/backend/tcop/dest.c:151
  • **呼び出し先**:
  • palloc0 (メモリ確保)
  • transientrel_receive, transientrel_startup, transientrel_shutdown, transientrel_destroy (関数ポインタの代入先)

関連データ構造

  • DR_transientrel: DestReceiver を継承した、一時テーブル操作用の拡張構造体。
  • DestReceiver: クエリ結果の出力先を抽象化した構造体。

失敗モードと境界条件

  • transientoid が無効なOIDである場合、後続の DestReceiver コールバック実行時にエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は DestReceiver のインターフェースを介して、MV更新時のデータ挿入ロジックをカプセル化している。
  • transientrel_destroy が明示的に代入されているが、ソース抜粋内には定義が含まれていないため、関連する破棄処理の変更時には注意が必要。
