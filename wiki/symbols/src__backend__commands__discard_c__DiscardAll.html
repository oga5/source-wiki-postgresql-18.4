DiscardAll

定義

  • 種別: function
  • 定義箇所: src/backend/commands/discard.c:56

目的

SQLコマンド DISCARD ALL を実行し、現在のセッション状態を初期状態へリセットする。

シグネチャまたは形状

static void DiscardAll(bool isTopLevel)

入力と出力

  • **入力**: isTopLevel (bool型) - コマンドがトップレベルで実行されているかを示すフラグ。
  • **出力**: なし（副作用としてセッション状態を更新）。

前提条件と不変条件

  • トランザクションブロック内での実行は禁止されており、PreventInTransactionBlock によって検証される。

アルゴリズムまたは振る舞い

以下の順序でセッションリソースを破棄・リセットする： 1. PortalHashTableDeleteAll(): ポータルをクローズする（ユーザー定義コードが実行される可能性があるため最初に行う）。 2. SetPGVariable("session_authorization", NIL, false): セッション認証状態をリセット。 3. ResetAllOptions(): GUC設定を初期化。 4. DropAllPreparedStatements(): 準備済み文を破棄。 5. Async_UnlistenAll(): 非同期通知の購読を解除。 6. LockReleaseAll(USER_LOCKMETHOD, true): ユーザーロックを解放。 7. ResetPlanCache(): プランキャッシュをクリア。 8. ResetTempTableNamespace(): 一時テーブルを削除。 9. ResetSequenceCaches(): シーケンスキャッシュをリセット。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の DiscardCommand から呼び出されることが確認されている。
  • **呼び出し先**: PreventInTransactionBlock, PortalHashTableDeleteAll, SetPGVariable, ResetAllOptions, DropAllPreparedStatements, Async_UnlistenAll, LockReleaseAll, ResetPlanCache, ResetTempTableNamespace, ResetSequenceCaches を呼び出す。

関連データ構造

  • DiscardStmt (構文解析結果として間接的に関連)

失敗モードと境界条件

  • トランザクションブロック内で呼び出された場合、エラーが発生する。

保守担当者向けメモ

  • DISCARD ALL はセッション内の全状態を破棄する強力なコマンドであるため、処理順序は副作用を考慮して慎重に設計されている。
  • ユーザー定義コードが実行される可能性のある処理（ポータルのクローズなど）を優先的に配置している。
