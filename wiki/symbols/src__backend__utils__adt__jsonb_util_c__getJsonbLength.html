getJsonbLength

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_util.c:158

目的

JsonbContainer 内の指定されたインデックスにある要素のバイト長を取得します。

シグネチャまたは形状

uint32 getJsonbLength(const JsonbContainer *jc, int index)

入力と出力

  • **入力**:
  • jc: 対象となる JsonbContainer 構造体へのポインタ。
  • index: コンテナ内の要素インデックス。
  • **出力**:
  • uint32: 指定された要素のバイト長。

前提条件と不変条件

  • index は jc->children 配列の有効な範囲内である必要があります。
  • jc->children[index] は JEntry 型のデータであり、マクロ JBE_HAS_OFF および JBE_OFFLENFLD を介して解釈されます。

アルゴリズムまたは振る舞い

1. JBE_HAS_OFF マクロを用いて、対象の JEntry がオフセット情報を持っているか判定します。 2. **オフセットを持つ場合**: getJsonbOffset を呼び出して要素の開始位置を取得し、JBE_OFFLENFLD で取得した終了位置（+1）から開始位置を減算して長さを算出します。 3. **オフセットを持たない場合**: JBE_OFFLENFLD の値を直接長さとして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonb_util.c 内の複数箇所（要素の長さ計算時）。
  • **呼び出し先**:
  • getJsonbOffset（オフセットが必要な場合のみ）。
  • **注記**: 呼び出し元および呼び出し先の詳細は、提供されたソースコードの断片およびツールによる検索結果に基づいています。

関連データ構造

  • JsonbContainer: jsonb のバイナリ表現を保持する構造体。
  • JEntry: JsonbContainer 内の各要素のメタデータ（型、オフセット、長さ）を格納する型。

失敗モードと境界条件

  • index が範囲外の場合、メモリ破壊や不正なメモリアクセスが発生する可能性があります。
  • JEntry のフラグが破損している場合、誤った長さが返される可能性があります。

保守担当者向けメモ

  • 本関数は jsonb のバイナリ構造に強く依存しています。JEntry のビットレイアウトに変更を加える際は、本関数のロジックも併せて更新する必要があります。
  • JBE_HAS_OFF などのマクロ定義が utils/jsonb.h に存在するため、構造変更時はそちらの定義も確認してください。
