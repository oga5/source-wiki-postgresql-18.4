hashint2extended

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hashfunc.c:65

目的

int16 型の値を、指定されたシード値を用いてハッシュ化し、ハッシュ結合等の処理で利用可能な Datum 型のハッシュ値を生成します。

シグネチャまたは形状

Datum hashint2extended(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて以下の値を受け取ります。
  • 第1引数: int16 型の値。
  • 第2引数: int64 型のシード値。
  • **出力**: Datum 型のハッシュ値。

前提条件と不変条件

  • 入力された int16 値は内部的に int32 にキャストされて処理されます。
  • ハッシュ関数としての性質上、同一の入力値とシード値に対しては常に同一のハッシュ値を返却する必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT16(0) により第1引数を取得し、int32 型へキャストします。 2. PG_GETARG_INT64(1) により第2引数（シード値）を取得します。 3. hash_uint32_extended 関数を呼び出し、キャストした値とシード値を渡して計算結果を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に明示的な呼び出し元情報は存在しませんが、PostgreSQLのシステムカタログ pg_amproc を介してハッシュアクセスメソッドから呼び出される設計です。
  • **呼び出し先**: hash_uint32_extended (PostgreSQL内部のハッシュ計算関数)。

関連データ構造

  • Datum: PostgreSQLにおける汎用的なデータ表現型。
  • PG_FUNCTION_ARGS: SQL関数呼び出し時の引数コンテキスト。

失敗モードと境界条件

  • 入力引数が不足している場合や、PG_FUNCTION_ARGS が正しく初期化されていない場合は、マクロの仕様に従いエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は int16 型専用の拡張ハッシュ関数です。新しい型を追加する際は、同様に ...extended バージョンを実装し、シード値を受け取れるようにしてください。
  • hash_uint32_extended への依存があるため、ハッシュアルゴリズムの変更時は影響範囲を確認してください。
