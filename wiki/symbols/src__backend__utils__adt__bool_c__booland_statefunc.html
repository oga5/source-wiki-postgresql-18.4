booland_statefunc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/bool.c:299

目的

SQLの集約関数 bool_and の状態遷移を計算します。現在の集約状態と次の入力値を論理積（AND）で結合し、更新された状態を返します。

シグネチャまたは形状

Datum booland_statefunc(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの bool 値を受け取ります。
  • 第1引数: 現在の集約状態（bool）
  • 第2引数: 次の入力値（bool）
  • **出力**: Datum 型として、論理積の結果（bool）を返します。

前提条件と不変条件

  • PostgreSQLの関数呼び出し規約（fmgr）に従う必要があります。
  • 入力値は PG_GETARG_BOOL マクロで取得可能である必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_BOOL(0) で現在の状態を取得します。 2. PG_GETARG_BOOL(1) で次の入力値を取得します。 3. 両者の論理積（&&）を計算します。 4. PG_RETURN_BOOL を使用して、計算結果を Datum 型として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（集約関数の実行時）。
  • **呼び出し先**: PostgreSQLの関数マネージャマクロ（PG_GETARG_BOOL, PG_RETURN_BOOL）。
  • 注: ソースコード内での直接的な関数呼び出し関係は、提供された情報からは特定されていません。

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • bool: C言語の論理型。

失敗モードと境界条件

  • 引数がNULLの場合の挙動は、PostgreSQLの関数マネージャの呼び出し規約に依存します。本関数自体は単純な論理積演算を行うため、入力が有効な bool 値であることを前提としています。

保守担当者向けメモ

  • 本関数は bool_and 集約のロジックを担う単純なラッパーです。論理演算の仕様変更が必要な場合はここを修正します。
  • 関連する boolor_statefunc と同様の設計パターンに従っています。
