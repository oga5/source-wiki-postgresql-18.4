preprocess_aggref

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepagg.c:115

目的

クエリ内の個々の Aggref ノードを解析し、集約関数のメタデータ解決、遷移状態の共有判定、およびプランニングに必要な情報（AggInfo, AggTransInfo）の生成を行う。

シグネチャまたは形状

static void preprocess_aggref(Aggref *aggref, PlannerInfo *root)

入力と出力

  • **入力**:
  • aggref: 解析対象の集約関数ノード。
  • root: プランニング全体の情報を保持する PlannerInfo 構造体。
  • **出力**: なし（副作用として root 内の agginfos および aggtransinfos リストを更新し、aggref ノードの属性を直接書き換える）。

前提条件と不変条件

  • aggref->agglevelsup == 0 であること（サブクエリ内の集約ではないこと）。
  • pg_aggregate システムカタログから該当する集約関数の情報を取得できること。

アルゴリズムまたは振る舞い

1. **メタデータ取得**: pg_aggregate から遷移関数、最終関数、結合関数等の OID を取得する。 2. **型解決**: 多相型（Polymorphic）の遷移型を get_aggregate_argtypes および resolve_aggregate_transtype で解決し、aggref->aggtranstype に設定する。 3. **共有可能性判定**: aggfinalmodify が AGGMODIFY_READ_WRITE でない場合、遷移状態を共有可能（shareable）とみなす。 4. **集約の統合**: find_compatible_agg を呼び出し、同一の集約呼び出しが存在すれば aggno を再利用する。存在しない場合は新規 AggInfo を作成する。 5. **遷移状態の共有**: find_compatible_trans を呼び出し、遷移状態を共有可能な既存の AggTransInfo を検索する。見つからない場合は新規 AggTransInfo を作成し、root->aggtransinfos に追加する。 6. **制約チェック**: aggorder や aggdistinct が存在する場合、部分集約（Partial Aggregation）を不可とするフラグを立てる。

呼び出し元と呼び出し先

  • **呼び出し元**: preprocess_aggrefs（クエリツリー走査中に呼び出される）。
  • **呼び出し先**: SearchSysCache1, get_aggregate_argtypes, resolve_aggregate_transtype, find_compatible_agg, find_compatible_trans, GetAggInitVal 等。

関連データ構造

  • Aggref: 解析対象の集約ノード。
  • PlannerInfo: プランニング状態を保持する構造体。
  • AggInfo: 集約関数の最終計算情報を保持。
  • AggTransInfo: 遷移状態の計算情報を保持。

失敗モードと境界条件

  • pg_aggregate のキャッシュルックアップに失敗した場合、elog(ERROR) で処理を中断する。
  • 遷移型が解決できない場合や、集約関数定義が不正な場合はシステムカタログの整合性に依存する。

保守担当者向けメモ

  • 本関数は入力された Aggref ノードをインプレースで更新する。
  • aggtranstypmod は、遷移型が入力型と同一の場合にのみ入力の typmod を継承する簡易的なヒューリスティックを用いている。
  • 部分集約の可否判定は root->hasNonPartialAggs フラグを通じて行われるため、このフラグの更新ロジックには注意が必要。
