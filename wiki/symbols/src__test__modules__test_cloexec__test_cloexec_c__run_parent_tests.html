run_parent_tests

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_cloexec/test_cloexec.c:76

目的

Windows環境において、O_CLOEXEC フラグが付与されたファイル記述子が子プロセスに継承されないことを検証するための親プロセス側の処理を実行する。

シグネチャまたは形状

static void run_parent_tests(const char *testfile1, const char *testfile2)

入力と出力

  • **入力**:
  • testfile1: O_CLOEXEC を付与して開くテスト用ファイルパス。
  • testfile2: O_CLOEXEC なしで開くテスト用ファイルパス。
  • **出力**: なし（成功時は標準出力に結果を表示し、失敗時は exit(1) によりプロセスを終了する）。

前提条件と不変条件

  • Windows環境で実行されること。
  • CreateProcess の bInheritHandles 引数が TRUE に設定されていること（これが本テストの検証の前提）。

アルゴリズムまたは振る舞い

1. testfile1 を O_CLOEXEC 付きで、testfile2 をなしで開く。 2. _get_osfhandle() を使用して、各ファイル記述子からWindowsネイティブハンドルを取得する。 3. 取得したハンドルを引数として、自身を子プロセスとして CreateProcess で起動する。この際、bInheritHandles を TRUE に設定する。 4. WaitForSingleObject を用いて子プロセスの終了を待機する。 5. 子プロセスの終了コードを取得し、0 であれば検証成功、それ以外であれば失敗と判定する。 6. 最後に開いたファイル記述子をクローズする。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数 (src/test/modules/test_cloexec/test_cloexec.c:59)
  • **呼び出し先**:
  • open, close, _get_osfhandle (標準/移植性ライブラリ)
  • GetModuleFileName, CreateProcess, WaitForSingleObject, GetExitCodeProcess, CloseHandle (Windows API)

関連データ構造

  • STARTUPINFO: プロセス起動時の設定情報。
  • PROCESS_INFORMATION: 生成されたプロセスのハンドルおよびID情報。

失敗モードと境界条件

  • ファイルのオープン失敗: open が負の値を返した場合、エラーメッセージを出力して終了する。
  • ハンドル取得失敗: _get_osfhandle が INVALID_HANDLE_VALUE を返した場合、エラーメッセージを出力して終了する。
  • プロセス生成失敗: CreateProcess が失敗した場合、GetLastError を出力して終了する。
  • 検証失敗: 子プロセスの終了コードが 0 以外の場合、O_CLOEXEC の動作が正しくないものとして終了する。

保守担当者向けメモ

  • 本関数はWindows専用のテストロジックである。
  • CreateProcess の bInheritHandles が TRUE であることが、O_CLOEXEC の有効性を確認する唯一の手段であるため、この設定を変更しないこと。
