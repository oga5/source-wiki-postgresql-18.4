BaseInit

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/postinit.c:611

目的

PostgreSQL バックエンドプロセスにおいて、データベース接続や認証が行われる前の、ストレージ、バッファ、統計情報などの低レイヤーなサブシステムを初期化する。

シグネチャまたは形状

void BaseInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として各サブシステムの初期化を実行）

前提条件と不変条件

  • MyProc が NULL でないこと（Assert により検証）。

アルゴリズムまたは振る舞い

以下の順序で各サブシステムの初期化を実行する： 1. DebugFileOpen: デバッグ用ファイル記述子の初期化。 2. InitFileAccess: ファイルアクセス機能の初期化。 3. pgstat_initialize: 統計情報報告機能の初期化。 4. pgaio_init_backend: 非同期I/O（AIO）の初期化。 5. InitSync, smgrinit, InitBufferManagerAccess: ストレージおよびバッファマネージャの初期化。 6. InitTemporaryFileAccess: 一時ファイルアクセス機能の初期化。 7. InitXLogInsert: WALレコード構築用バッファの初期化。 8. InitLockManagerAccess: ロックマネージャのローカル構造体の初期化。 9. ReplicationSlotInitialize: レプリケーションスロットの初期化。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/bootstrap/bootstrap.c
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/auxprocess.c
  • src/backend/postmaster/bgworker.c
  • src/backend/replication/logical/slotsync.c
  • src/backend/tcop/postgres.c
  • 呼び出し先:
  • DebugFileOpen, InitFileAccess, pgstat_initialize, pgaio_init_backend, InitSync, smgrinit, InitBufferManagerAccess, InitTemporaryFileAccess, InitXLogInsert, InitLockManagerAccess, ReplicationSlotInitialize

関連データ構造

  • MyProc (プロセス制御ブロック)

失敗モードと境界条件

  • MyProc が初期化されていない場合、Assert によりプロセスが停止する。
  • 各サブシステムの初期化関数内でリソース確保に失敗した場合、PostgreSQL の標準的なエラーハンドリングに従いプロセスが終了する可能性がある。

保守担当者向けメモ

  • 初期化順序は、依存関係（例：pgstat が他のサブシステムより先に初期化される必要がある等）に強く依存しているため、順序の変更には注意が必要である。
  • InitTemporaryFileAccess は pgstat の後に配置されており、終了フックの実行順序を考慮した設計となっている。
