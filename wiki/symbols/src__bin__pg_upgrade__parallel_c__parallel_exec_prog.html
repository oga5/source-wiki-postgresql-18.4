parallel_exec_prog

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/parallel.c:61

目的

外部コマンドを並列実行するための抽象化レイヤーを提供します。pg_upgrade において、指定されたジョブ数制限内でコマンドを非同期実行し、必要に応じて完了待ちを行います。

シグネチャまたは形状

void parallel_exec_prog(const char *log_file, const char *opt_log_file, const char *fmt, ...);

入力と出力

  • log_file: 実行結果のログ出力先ファイルパス。
  • opt_log_file: オプションのログ出力先（存在する場合）。
  • fmt, ...: printf 形式のコマンド文字列。
  • 出力: なし（void）。異常終了時は pg_fatal を呼び出しプロセスを停止します。

前提条件と不変条件

  • user_opts.jobs が設定されていること。
  • Windows 環境では、thread_handles および exec_thread_args が初期化されていること。
  • parallel_jobs 変数が現在の実行中ジョブ数を正確に保持していること。

アルゴリズムまたは振る舞い

1. **フォーマット**: fmt を基にコマンド文字列を生成します。 2. **直列実行**: user_opts.jobs <= 1 の場合、exec_prog を直接呼び出し同期実行します。 3. **並列実行**:

  • reap_child(false) を呼び出し、完了済みのジョブを回収します。
  • parallel_jobs >= user_opts.jobs の場合、reap_child(true) を呼び出し、ジョブが空くまでブロックします。
  • parallel_jobs をインクリメントします。
  • fflush(NULL) により stdio をフラッシュします。
  • **Unix**: fork() を行い、子プロセスで exec_prog を実行します。
  • **Windows**: exec_thread_args に引数を設定し、_beginthreadex で win32_exec_prog を起動します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/dump.c:58
  • src/bin/pg_upgrade/pg_upgrade.c:673
  • **呼び出し先**:
  • reap_child
  • exec_prog
  • _beginthreadex (Windows)
  • fork (Unix)
  • pg_fatal

関連データ構造

  • exec_thread_arg: Windows でスレッドに引数を渡すための構造体。
  • thread_handles: Windows でのスレッドハンドル配列。
  • exec_thread_args: Windows でのスレッド引数配列。

失敗モードと境界条件

  • fork() または _beginthreadex() の失敗時に pg_fatal を呼び出し、プログラムを終了します。
  • Windows では MAXIMUM_WAIT_OBJECTS（64）の制限を受ける可能性があります。
  • 異常終了した子プロセス/スレッドがある場合、reap_child を通じて検知され、全体が停止します。

保守担当者向けメモ

  • Windows 実装では、スレッド引数のメモリ管理をプロセス生存期間中維持する設計になっています。
  • Unix 系と Windows 系でプロセス/スレッド生成の抽象化が大きく異なるため、修正時は #ifdef WIN32 ブロックの両方を確認してください。
