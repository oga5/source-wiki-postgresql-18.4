list_free

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/list.c:1545

目的

指定されたリスト構造体自体をメモリから解放します。リスト内の各要素が指すメモリ領域の解放は行いません（浅い解放）。

シグネチャまたは形状

void list_free(List *list)

入力と出力

  • **入力**: list (解放対象の List 構造体へのポインタ)
  • **出力**: なし

前提条件と不変条件

  • list が NULL の場合、本関数は安全に何もしません（list_free_private の仕様に基づく）。
  • 解放後の list ポインタへのアクセスは未定義動作となります。

アルゴリズムまたは振る舞い

内部的に list_free_private(list, false) を呼び出します。第2引数に false を渡すことで、リストが保持する要素（ポインタ等）を再帰的に解放せず、リストのヘッダおよび要素配列のみを解放します。

関連データ構造

  • List: リストのヘッダ構造体。本関数によりこの構造体と内部の elements 配列が解放されます。

失敗モードと境界条件

  • list が NULL の場合は正常に終了します。
  • リスト内の要素が指すメモリ領域を解放する必要がある場合は、本関数ではなく list_free_deep を使用する必要があります。

保守担当者向けメモ

  • リストの要素が動的に確保されたメモリを指している場合、本関数を使用するとメモリリークが発生します。要素の解放が必要な場合は list_free_deep を検討してください。
