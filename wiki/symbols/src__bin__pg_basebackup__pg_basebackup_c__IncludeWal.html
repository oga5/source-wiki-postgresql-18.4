IncludeWal

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_basebackup/pg_basebackup.c:112

目的

pg_basebackup における WAL（Write-Ahead Log）の取得方法を定義する列挙型です。

シグネチャまたは形状

typedef enum
{
	NO_WAL,
	FETCH_WAL,
	STREAM_WAL,
} IncludeWal;

入力と出力

  • NO_WAL: WAL を含めない。
  • FETCH_WAL: バックアップ終了後に WAL を取得する。
  • STREAM_WAL: バックアップ中に WAL をストリーミングする。

前提条件と不変条件

  • pg_basebackup.c 内の静的変数 includewal の型として使用されます。
  • デフォルト値は STREAM_WAL です。

アルゴリズムまたは振る舞い

pg_basebackup のメインロジックにおいて、この値が STREAM_WAL である場合、StartLogStreamer 関数を呼び出して WAL ストリーミング用の並行プロセス（またはスレッド）を起動する制御フローの分岐条件として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_basebackup/pg_basebackup.c 内の静的変数定義（static IncludeWal includewal = STREAM_WAL;）にて参照されています。
  • 呼び出し先: 本シンボル自体は列挙型定義であり、関数呼び出しは行いません。

関連データ構造

  • logstreamer_param: WAL ストリーミングのパラメータ管理に使用される構造体。

失敗モードと境界条件

  • 列挙型以外の値が代入された場合の挙動については、ソースコード上での明示的な制約は確認できません。

保守担当者向けメモ

  • pg_basebackup のコマンドライン引数解析において、ユーザー指定に基づいてこの列挙値が設定されます。
  • 新しい WAL 取得モードを追加する場合は、この列挙型と、それを利用する BaseBackup 関数内の条件分岐を更新する必要があります。
