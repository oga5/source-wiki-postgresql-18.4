ValidateRestrictionEstimator

定義

  • 種別: function
  • 定義箇所: src/backend/commands/operatorcmds.c:274

目的

演算子定義時に指定された制限（Restriction）推定関数の妥当性を検証し、その OID を取得する。

シグネチャまたは形状

Oid ValidateRestrictionEstimator(List *restrictionName)

入力と出力

  • **入力**: restrictionName (推定関数名を表すリスト)
  • **出力**: 検証済みの推定関数の Oid
  • **副作用**: 検証失敗時に ereport(ERROR) を発行する。

前提条件と不変条件

  • 推定関数は、引数として (PlannerInfo *, Oid, List *, int4) を受け取るシグネチャを持つ必要がある。
  • 推定関数は戻り値として float8 型を返す必要がある。

アルゴリズムまたは振る舞い

1. LookupFuncName を使用し、指定された名前から 4 引数の関数を検索する。 2. get_func_rettype を呼び出し、戻り値が FLOAT8OID であることを確認する。 3. 関数が組み込み（FirstGenbkiObjectId 以上）でない場合、呼び出し元がスーパーユーザーであることを要求する。 4. 組み込み関数の場合は、現在のユーザーが当該関数に対する ACL_EXECUTE 権限を持っているかを確認する。 5. 検証を通過した場合、関数の Oid を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/operatorcmds.c 内の演算子定義処理（DefineOperator 等）から呼び出される。
  • **呼び出し先**: LookupFuncName, get_func_rettype, superuser, object_aclcheck, aclcheck_error 等のシステムカタログ操作関数。

関連データ構造

  • Oid: 関数や型を識別するオブジェクト ID。
  • List: 関数名を保持するリスト構造。

失敗モードと境界条件

  • **戻り値不一致**: 戻り値が float8 でない場合、ERRCODE_INVALID_OBJECT_DEFINITION でエラー終了する。
  • **権限不足**: 非組み込み関数でスーパーユーザーでない場合、または組み込み関数で実行権限がない場合、それぞれ ERRCODE_INSUFFICIENT_PRIVILEGE または権限エラーで終了する。
  • **関数未定義**: LookupFuncName が関数を見つけられない場合、エラー終了する。

保守担当者向けメモ

  • 本関数は ValidateJoinEstimator と同様の権限チェックロジックを共有している。
  • 組み込み関数と拡張関数の間でセキュリティ要件が異なるため、変更時は restrictionOid >= FirstGenbkiObjectId の判定条件に注意すること。
