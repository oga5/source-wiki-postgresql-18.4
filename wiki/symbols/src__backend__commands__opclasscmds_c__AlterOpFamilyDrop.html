AlterOpFamilyDrop

定義

  • 種別: function
  • 定義箇所: src/backend/commands/opclasscmds.c:1029

目的

ALTER OPERATOR FAMILY ... DROP コマンドの実行において、指定された演算子またはサポート関数を演算子ファミリーから削除する。

シグネチャまたは形状

static void AlterOpFamilyDrop(AlterOpFamilyStmt *stmt, Oid amoid, Oid opfamilyoid,
                              int maxOpNumber, int maxProcNumber, List *items)

入力と出力

  • **入力**:
  • stmt: ALTER OPERATOR FAMILY 文の構文解析ツリー。
  • amoid: 対象アクセスメソッドのOID。
  • opfamilyoid: 対象演算子ファミリーのOID。
  • maxOpNumber, maxProcNumber: 各AMで許容される演算子およびサポート関数の最大番号。
  • items: 削除対象となる演算子や関数の定義リスト（CreateOpClassItem のリスト）。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • 削除対象の演算子番号および関数番号は、1から maxOpNumber（または maxProcNumber）の範囲内である必要がある。範囲外の場合はエラーとなる。
  • items 内の各要素は、processTypesSpec により型情報が解決されている必要がある。

アルゴリズムまたは振る舞い

1. items リストを走査し、各要素の itemtype に応じて OpFamilyMember 構造体を生成する。

  • OPCLASS_ITEM_OPERATOR: 演算子としてリスト operators に追加。
  • OPCLASS_ITEM_FUNCTION: サポート関数としてリスト procedures に追加。

    • 2. dropOperators を呼び出し、pg_amop カタログから該当する演算子タプルを削除する。 3. dropProcedures を呼び出し、pg_amproc カタログから該当するサポート関数タプルを削除する。 4. EventTriggerCollectAlterOpFam を呼び出し、イベントトリガー用に削除情報を収集する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/opclasscmds.c 内の AlterOpFamily 関数から呼び出される。
  • **呼び出し先**: processTypesSpec, addFamilyMember, dropOperators, dropProcedures, EventTriggerCollectAlterOpFam。

関連データ構造

  • AlterOpFamilyStmt: SQL文の解析結果を保持。
  • OpFamilyMember: 演算子ファミリーのメンバー（演算子または関数）の情報を保持する内部構造体。
  • CreateOpClassItem: 構文解析された個別のアイテム定義。

失敗モードと境界条件

  • 指定された番号が許容範囲外の場合、ERRCODE_INVALID_OBJECT_DEFINITION でエラー終了する。
  • 未知の itemtype が渡された場合、elog(ERROR) により異常終了する。

保守担当者向けメモ

  • 本関数は ALTER OPERATOR FAMILY の削除処理のみを担う。追加処理については AlterOpFamilyAdd を参照すること。
  • 演算子や関数の型解決には processTypesSpec を利用しており、新しいAMを追加する際は、この型解決ロジックが対象AMの要件を満たしているか確認が必要である。
