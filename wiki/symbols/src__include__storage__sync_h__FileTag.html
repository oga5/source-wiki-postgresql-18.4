FileTag

定義

  • 種別: struct
  • 定義箇所: src/include/storage/sync.h:50

目的

PostgreSQLのストレージサブシステムにおいて、同期（fsync）や削除などの操作対象となるファイルを一意に識別するための構造体です。

シグネチャまたは形状

typedef struct FileTag
{
	int16		handler;		/* SyncRequestHandler value */
	int16		forknum;		/* ForkNumber */
	RelFileLocator rlocator;
	uint64		segno;
} FileTag;

入力と出力

  • **入力**: 構造体の各フィールド（ハンドラ識別子、フォーク番号、リレーショナルファイルロケータ、セグメント番号）により構成されます。
  • **出力**: 同期要求のキューイングやハッシュテーブルでの重複排除のキーとして利用されます。

前提条件と不変条件

  • handler は SyncRequestHandler の値として定義されている必要があります。
  • rlocator（RelFileLocator）により、対象ファイルの物理的な位置が特定可能である必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、バックエンドプロセスから発行された同期要求を sync.c 内のハッシュテーブル（pendingOps）で管理する際のキーとして使用されます。
  • チェックポイント処理時、このタグに基づいて適切なハンドラ（md.c 等）が呼び出され、実際のI/O操作が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/access/transam/clog.c: clogsyncfiletag
  • src/backend/access/transam/commit_ts.c: committssyncfiletag
  • src/backend/access/transam/multixact.c: multixactoffsetssyncfiletag, multixactmemberssyncfiletag
  • src/backend/access/transam/slru.c: SlruSyncFileTag
  • src/backend/postmaster/checkpointer.c: ForwardSyncRequest
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • RelFileLocator: ファイルの物理位置を特定するために内部で保持。
  • SyncRequestHandler: 同期処理を委譲するためのハンドラ識別子。

失敗モードと境界条件

  • 構造体の初期化漏れ（memset 等によるゼロクリアの欠如）は、ハッシュテーブルでの誤ったマッチングや未定義の動作を引き起こす可能性があります。
  • segno が正しく設定されていない場合、マルチセグメントファイルの同期漏れが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は sync.c の pendingOps ハッシュテーブルのキーとして機能するため、フィールドの追加や変更を行う際はハッシュ関数の更新が必要になる場合があります。
  • 現在は md.c の要件が強く反映されていますが、将来的に新しいストレージハンドラを追加する際は handler フィールドの拡張性を考慮してください。
