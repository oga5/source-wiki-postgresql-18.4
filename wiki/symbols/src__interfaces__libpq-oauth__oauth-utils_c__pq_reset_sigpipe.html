pq_reset_sigpipe

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq-oauth/oauth-utils.c:207

目的

ネットワーク通信操作の完了後、pq_block_sigpipe によって一時的にブロックされていた SIGPIPE シグナルを復元し、シグナルマスクを元の状態に戻します。

シグネチャまたは形状

void pq_reset_sigpipe(sigset_t *osigset, bool sigpipe_pending, bool got_epipe)

入力と出力

  • osigset (入力): pthread_sigmask で保存された以前のシグナルマスク。
  • sigpipe_pending (入力): pq_block_sigpipe 呼び出し時点で既に SIGPIPE が保留されていたかを示すフラグ。
  • got_epipe (入力): ネットワーク操作中に EPIPE エラーが発生したかを示すフラグ。
  • 戻り値: なし。

前提条件と不変条件

  • osigset は、事前に pq_block_sigpipe 等で取得された有効なシグナルマスクである必要があります。
  • 本関数実行前後で、ソケットエラー状態（SOCK_ERRNO）が保持されることを保証します。

アルゴリズムまたは振る舞い

1. 現在のソケットエラー番号を save_errno に退避します。 2. got_epipe が真かつ sigpipe_pending が偽の場合、保留中の SIGPIPE が存在するかを確認します。存在する場合は sigwait を使用してそのシグナルを消費（クリア）します。 3. pthread_sigmask を用いて、osigset に保存されていた元のシグナルマスクを復元します。 4. 退避していた save_errno をソケットエラー変数に書き戻します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/interfaces/libpq-oauth/oauth-curl.c:3092
  • src/interfaces/libpq/fe-print.c:333
  • src/interfaces/libpq/fe-secure.c:86
  • src/interfaces/libpq/fe-secure.c:554
  • 呼び出し先:
  • sigpending, sigismember, sigemptyset, sigaddset, sigwait (シグナル操作)
  • pthread_sigmask (シグナルマスク復元)

関連データ構造

  • sigset_t: シグナルセットを保持する標準型。

失敗モードと境界条件

  • sigpending や sigwait が失敗した場合の明示的なエラーハンドリングは実装されていません。
  • got_epipe が偽の場合、保留中の SIGPIPE のクリア処理はスキップされます。

保守担当者向けメモ

  • 本関数は libpq の内部ユーティリティであり、libpq-oauth モジュールと libpq 本体で共有されています。変更時は両方の動作に影響を与える点に注意してください。
  • SOCK_ERRNO および SOCK_ERRNO_SET マクロを介してプラットフォーム固有のソケットエラー処理を抽象化しています。
