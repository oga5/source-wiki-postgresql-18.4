astreamer_plain_writer

定義

  • 種別: struct
  • 定義箇所: src/fe_utils/astreamer_file.c:24

目的

アーカイブストリーミングフレームワークにおいて、ストリームデータを物理的なファイルシステム上の単一ファイルへ書き出すためのコンテキストを保持します。

シグネチャまたは形状

typedef struct astreamer_plain_writer
{
    astreamer    base;              /* 基底ストリーマー構造体 */
    char        *pathname;          /* 書き込み先ファイルパス */
    FILE        *file;              /* 開いているファイルポインタ */
    bool         should_close_file; /* 終了時にファイルを閉じるべきかのフラグ */
} astreamer_plain_writer;

入力と出力

  • **入力**: astreamer_plain_writer_new() を介して初期化され、astreamer 基底構造体を通じてデータを受け取ります。
  • **出力**: 指定された pathname に対応するファイルシステム上のファイルへデータが書き込まれます。

前提条件と不変条件

  • base メンバは astreamer 型として初期化されている必要があります。
  • file が有効な場合、それは pathname に対応する書き込み可能なストリームである必要があります。

アルゴリズムまたは振る舞い

  • astreamer_plain_writer_new() でインスタンス化されます。
  • ストリーム処理中、astreamer_plain_writer_content() 等の関数によって、file メンバを介して逐次的な書き込みが行われます。
  • astreamer_plain_writer_finalize() および astreamer_plain_writer_free() を通じて、リソースの解放とファイルのクローズ処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/astreamer_file.c 内の astreamer_plain_writer_new、astreamer_plain_writer_content、astreamer_plain_writer_finalize、astreamer_plain_writer_free がこの構造体を操作します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、具体的な関数呼び出しの詳細は各操作関数（_content, _finalize 等）に依存します。

関連データ構造

  • astreamer: 本構造体が継承する基底構造体。
  • FILE: 標準Cライブラリのファイルストリーム。

失敗モードと境界条件

  • pathname が無効な場合や、ファイルシステムへの書き込み権限がない場合に失敗する可能性があります。
  • should_close_file フラグの管理が不適切な場合、リソースリークが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は astreamer_plain_writer_new で palloc0 を使用してメモリ確保されています。
  • src/fe_utils/astreamer_gzip.c 等の他のストリーマー実装と設計パターンを共有しています。
