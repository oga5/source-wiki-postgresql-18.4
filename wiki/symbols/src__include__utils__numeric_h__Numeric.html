Numeric

定義

  • 種別: typedef struct NumericData *Numeric;
  • 定義箇所: src/include/utils/numeric.h:54

目的

PostgreSQLのNUMERIC型（正確な数値）を扱うための不透明ポインタ型です。内部構造を隠蔽し、外部モジュールに対して抽象化されたインターフェースを提供します。

シグネチャまたは形状

struct NumericData へのポインタとして定義されています。

入力と出力

  • 入力: Datum 型から PG_GETARG_NUMERIC マクロ等を通じて取得されます。
  • 出力: NumericGetDatum マクロを通じて Datum 型へ変換され、システムへ返されます。

前提条件と不変条件

  • Numeric の内部メンバへ直接アクセスすることは禁止されています。
  • 演算や操作は、src/backend/utils/adt/numeric.c で提供される関数および本ヘッダーで定義されたマクロを介して行う必要があります。

アルゴリズムまたは振る舞い

本シンボル自体は型定義であり、具体的な演算ロジックは持ちません。データフローとしては、Datum として渡された値をデトースト（必要に応じてメモリ展開）し、Numeric ポインタとして演算関数へ受け渡すための基盤となります。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • contrib/btree_gin/btree_gin.c
  • contrib/btree_gist/btree_numeric.c
  • contrib/jsonb_plpython/jsonb_plpython.c
  • src/backend/utils/adt/dbsize.c
  • 呼び出し先:
  • 内部構造の実体は src/backend/utils/adt/numeric.c に存在します。具体的な関数呼び出し関係については、本ソース excerpt には記載がありません。

関連データ構造

  • NumericData: Numeric が指し示す実際の構造体（numeric.c で定義）。
  • Datum: PostgreSQLの汎用データ表現。

失敗モードと境界条件

  • Numeric 型の操作において、精度（NUMERIC_MAX_PRECISION）やスケールの制約を超過した場合の挙動は、各演算関数（numeric_add_opt_error 等）の仕様に依存します。
  • _opt_error サフィックスを持つ関数を使用する場合、エラーは例外ではなく bool *have_error を通じて通知されます。

保守担当者向けメモ

  • 本シンボルは不透明ポインタであるため、メモリレイアウトの変更は numeric.c 側でのみ完結します。
  • 外部モジュールからの利用時は、必ず fmgr.h 経由の変換マクロを使用してください。
