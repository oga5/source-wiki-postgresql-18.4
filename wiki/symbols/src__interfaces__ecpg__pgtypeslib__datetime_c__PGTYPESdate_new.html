PGTYPESdate_new

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/datetime.c:14

目的

ECPGアプリケーションにおいて、日付型（date）を格納するためのメモリを動的に確保します。

シグネチャまたは形状

date *PGTYPESdate_new(void)

入力と出力

  • 入力: なし
  • 出力: 確保された date 型へのポインタ。メモリ不足時には NULL を返します。

前提条件と不変条件

  • 戻り値が NULL でない場合、呼び出し元は PGTYPESdate_free を使用してメモリを解放する責任を負います。
  • 内部的には pgtypes_alloc を使用してメモリを割り当てます。

アルゴリズムまたは振る舞い

1. pgtypes_alloc を呼び出し、sizeof(date) 分のメモリ領域を確保します。 2. 確保した領域へのポインタを返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/test/expected/pgtypeslib-dt_test2.c での使用例が確認されています。
  • 呼び出し先: pgtypes_alloc（メモリ確保用ライブラリ関数）。

関連データ構造

  • date: ECPG内で使用される日付型（内部的には整数値）。

失敗モードと境界条件

  • システムのメモリが不足している場合、pgtypes_alloc が NULL を返すため、本関数も NULL を返します。

保守担当者向けメモ

  • 本関数で確保したメモリは、必ず対応する解放関数で管理してください。
  • date 型の内部表現は2000年1月1日を基準としたオフセット値として扱われます。
