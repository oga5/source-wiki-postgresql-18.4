InitializeClientEncoding

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/mbutils.c:289

目的

バックエンド起動時にクライアントエンコーディング設定を確定させ、必要な変換関数を初期化する。

シグネチャまたは形状

void InitializeClientEncoding(void)

入力と出力

  • 入力: なし（グローバル変数 pending_client_encoding を参照）
  • 出力: なし（成功時）
  • 副作用: クライアントエンコーディングのセットアップ、Utf8ToServerConvProc への変換関数キャッシュの格納、backend_startup_complete フラグの更新。

前提条件と不変条件

  • backend_startup_complete が false であること。
  • データベースエンコーディングが確定していること。

アルゴリズムまたは振る舞い

1. backend_startup_complete を true に設定する。 2. PrepareClientEncoding および SetClientEncoding を呼び出し、保留中のクライアントエンコーディングを適用する。失敗した場合は FATAL エラーを発生させる。 3. サーバーエンコーディングが PG_UTF8 および PG_SQL_ASCII 以外の場合、UTF8 からサーバーエンコーディングへの変換関数を検索する。 4. 変換関数が見つかった場合、TopMemoryContext に FmgrInfo を割り当て、Utf8ToServerConvProc にキャッシュする。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/init/postinit.c
  • 呼び出し先:
  • PrepareClientEncoding
  • SetClientEncoding
  • GetDatabaseEncoding
  • FindDefaultConversionProc
  • MemoryContextAlloc
  • fmgr_info_cxt
  • ereport (FATAL)

関連データ構造

  • Utf8ToServerConvProc (グローバル変数)
  • FmgrInfo
  • TopMemoryContext

失敗モードと境界条件

  • 指定されたエンコーディング間の変換がサポートされていない場合、FATAL エラーでプロセスが終了する。
  • 変換関数が見つからない場合、Utf8ToServerConvProc は NULL のまま維持される。

保守担当者向けメモ

  • 本関数はバックエンド起動時に一度だけ実行されることを想定している。
  • Utf8ToServerConvProc は TopMemoryContext に割り当てられるため、セッション終了までメモリが保持される。
