jsonb_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb.c:88

目的

クライアントから送信されたバイナリ形式の JSONB データを受信し、PostgreSQL 内部の Jsonb 型（Datum）へ変換する。

シグネチャまたは形状

Datum jsonb_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して StringInfo バッファを受け取る。バッファの先頭にはバージョン番号（1バイト）が含まれる。
  • **出力**: 変換された Jsonb 構造体へのポインタを Datum として返す。

前提条件と不変条件

  • 入力バッファのバージョン番号が 1 である必要がある。それ以外の値はエラーとなる。
  • jsonb_from_cstring が内部で呼び出されるため、入力データは有効な JSON 文字列として解釈可能である必要がある。

アルゴリズムまたは振る舞い

1. pq_getmsgint を使用して、入力バッファから1バイトのバージョン番号を読み取る。 2. バージョンが 1 であることを確認する。それ以外の場合は elog(ERROR) を発生させる。 3. pq_getmsgtext を使用して、バッファの残りの部分から JSON 文字列を抽出する。 4. 抽出した文字列を jsonb_from_cstring に渡し、Jsonb 形式の Datum に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（fmgr）経由で呼び出される（直接の呼び出し元コードは提供されていない）。
  • **呼び出し先**:
  • pq_getmsgint / pq_getmsgtext (libpq/pqformat.h)
  • jsonb_from_cstring (src/backend/utils/adt/jsonb.c)

関連データ構造

  • StringInfo: バイナリデータの送受信に使用されるバッファ構造体。
  • Jsonb: PostgreSQL 内部の JSONB バイナリ表現。

失敗モードと境界条件

  • バージョン番号が 1 以外の場合、elog(ERROR) によりトランザクションが中断される。
  • jsonb_from_cstring が失敗した場合（不正な JSON 構文など）、その内部でエラーが発生し処理が中断される。

保守担当者向けメモ

  • 本関数はバイナリプロトコル経由の入力専用である。テキスト形式の入力については jsonb_in が担当する。
  • バージョン管理は pq_getmsgint で行われており、将来的なフォーマット変更時にはこの分岐ロジックを拡張する必要がある。
