libpqsrv_connect_params

定義

  • 種別: function
  • 定義箇所: src/include/libpq/libpq-be-fe-helpers.h:83

目的

PostgreSQL バックエンド内で、キーワードと値のペアを使用して外部データベースへの接続を開始するためのラッパー関数です。バックエンドのファイル記述子（FD）管理ルールに従い、接続確立までの待機処理を安全に行います。

シグネチャまたは形状

PGconn *libpqsrv_connect_params(const char *const *keywords,
                                const char *const *values,
                                int expand_dbname,
                                uint32 wait_event_info)

入力と出力

  • **入力**:
  • keywords: 接続パラメータ名の配列（libpq の PQconnectStartParams に準拠）。
  • values: 接続パラメータ値の配列。
  • expand_dbname: データベース名展開の可否を指定するフラグ。
  • wait_event_info: 待機イベントの識別子（WaitLatchOrSocket で使用）。
  • **出力**:
  • 接続状態を保持する PGconn * オブジェクトを返します。

前提条件と不変条件

  • バックエンドの FD 制限内での動作が保証されます（libpqsrv_connect_prepare による事前準備のため）。
  • 接続処理中の割り込み（シグナル）は libpqsrv_connect_internal 内で適切に処理されます。

アルゴリズムまたは振る舞い

1. libpqsrv_connect_prepare() を呼び出し、バックエンドの FD 予約を行います。 2. PQconnectStartParams() を呼び出し、非ブロッキング接続を開始します。 3. libpqsrv_connect_internal() を呼び出し、接続が完了するまでラッチ待機と割り込みチェックを繰り返します。 4. 確立された PGconn オブジェクトを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/connection.c:617
  • src/backend/replication/libpqwalreceiver/libpqwalreceiver.c:226
  • **呼び出し先**:
  • libpqsrv_connect_prepare
  • PQconnectStartParams
  • libpqsrv_connect_internal

関連データ構造

  • PGconn: libpq の接続ハンドル。

失敗モードと境界条件

  • 接続失敗時やリソース不足時には libpq の仕様に従い、エラー状態を含む PGconn が返されます。
  • 割り込みが発生した場合、libpqsrv_connect_internal の内部ロジックにより、バックエンドの割り込み処理が優先されます。

保守担当者向けメモ

  • 本関数は libpq-be-fe-helpers.h で定義されたインライン関数です。
  • バックエンドの FD 管理と統合されているため、直接 PQconnectdb を使用せず、必ず本ヘルパー関数を経由してください。
  • 接続処理は完全な非ブロッキングモードではないため、将来的な改善の余地があります。
