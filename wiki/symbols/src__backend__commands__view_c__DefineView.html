DefineView

定義

  • 種別: function
  • 定義箇所: src/backend/commands/view.c:355

目的

CREATE VIEW または CREATE OR REPLACE VIEW コマンドのメインエントリーポイントとして、ビューの解析、検証、およびリレーションの作成・更新を行う。

シグネチャまたは形状

ObjectAddress DefineView(ViewStmt *stmt, const char *queryString, int stmt_location, int stmt_len)

入力と出力

  • **入力**:
  • stmt: 解析された ViewStmt 構造体。
  • queryString: 元のSQLクエリ文字列。
  • stmt_location, stmt_len: クエリ内でのコマンド位置と長さ。
  • **出力**:
  • ObjectAddress: 作成または更新されたビューのオブジェクト識別子。

前提条件と不変条件

  • stmt->query は単一の SELECT クエリである必要がある。
  • SELECT INTO 構文はサポートされない。
  • WITH 句内にデータ変更文（INSERT/UPDATE/DELETE）を含むことはできない。
  • UNLOGGED ビューは作成できない。
  • WITH CHECK OPTION が指定された場合、そのビューは自動更新可能（auto-updatable）である必要がある。

アルゴリズムまたは振る舞い

1. **解析**: parse_analyze_fixedparams を呼び出し、生の解析ツリーを Query 構造体に変換する。 2. **検証**: クエリが SELECT であること、SELECT INTO や不正な WITH 句が含まれていないことを確認する。 3. **オプション処理**: WITH CHECK OPTION が指定されている場合、ビューが自動更新可能か検証する。 4. **エイリアス適用**: stmt->aliases が存在する場合、クエリのターゲットリストに列名を割り当てる。 5. **永続性決定**: 一時テーブルを使用するクエリの場合、ビューを一時ビュー（RELPERSISTENCE_TEMP）として扱う。 6. **リレーション作成**: DefineVirtualRelation を呼び出し、カタログへの登録および既存ビューの置換処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c:1648
  • **呼び出し先**:
  • parse_analyze_fixedparams
  • view_query_is_auto_updatable
  • DefineVirtualRelation

関連データ構造

  • ViewStmt: CREATE VIEW コマンドの構文解析結果を保持する。
  • Query: 解析後のクエリツリー。
  • ObjectAddress: 作成されたオブジェクトの識別情報。

失敗モードと境界条件

  • SELECT INTO を含むクエリの場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラー終了する。
  • WITH 句にデータ変更文が含まれる場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラー終了する。
  • WITH CHECK OPTION が自動更新不可能なビューに指定された場合、エラー終了する。
  • 列名が多すぎる場合、ERRCODE_SYNTAX_ERROR でエラー終了する。
  • UNLOGGED ビューを作成しようとした場合、ERRCODE_SYNTAX_ERROR でエラー終了する。

保守担当者向けメモ

  • 本関数はビュー作成のオーケストレーターであり、実際のカタログ操作は DefineVirtualRelation に委譲されている。
  • CREATE OR REPLACE VIEW のロジックは DefineVirtualRelation 内で処理されるため、置換時の整合性チェックについてはそちらを参照すること。
