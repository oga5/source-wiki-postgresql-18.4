is_bounding_box_test_exact

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_spgist.c:507

目的

SP-GiSTインデックスを用いた空間検索において、境界ボックス（Bounding Box）による近似判定が正確（exact）であるか、あるいは再チェック（recheck）が必要かを判定する。

シグネチャまたは形状

static bool is_bounding_box_test_exact(StrategyNumber strategy)

入力と出力

  • **入力**: StrategyNumber strategy（実行中の空間演算子の戦略番号）
  • **出力**: bool（true の場合、境界ボックスによる判定が正確であることを示す。false の場合、再チェックが必要であることを示す）

前提条件と不変条件

  • 入力される strategy は、PostgreSQLの幾何演算子に対応する有効な戦略番号であること。

アルゴリズムまたは振る舞い

指定された戦略番号に基づき、以下のいずれかに該当する場合は true を返し、それ以外の場合は false を返す。

  • RTLeftStrategyNumber
  • RTOverLeftStrategyNumber
  • RTOverRightStrategyNumber
  • RTRightStrategyNumber
  • RTOverBelowStrategyNumber
  • RTBelowStrategyNumber
  • RTAboveStrategyNumber
  • RTOverAboveStrategyNumber

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/geo_spgist.c 内の spg_box_quad_leaf_consistent 関数において、recheck フラグの決定に使用されている。
  • **呼び出し先**: なし。

関連データ構造

  • StrategyNumber: 演算子の戦略を識別するための整数型。

失敗モードと境界条件

  • switch 文の default ケースにより、リストに含まれない戦略番号が渡された場合は一律で false を返す。

保守担当者向けメモ

  • 本関数は、多角形などの境界ボックスによる近似検索を行う際に、インデックスの結果をそのまま信頼できるか（exact）を制御する重要なロジックである。新しい空間演算子を追加する際は、その演算子が境界ボックスのみで正確に判定可能かを確認し、必要に応じて本関数のリストを更新すること。
