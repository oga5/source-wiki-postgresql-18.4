SharedRecordTypmodRegistry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/typcache.c:181

目的

並列クエリ環境において、レコード型の TupleDesc と typmod を共有メモリ上で管理するためのレジストリ構造体です。

シグネチャまたは形状

struct SharedRecordTypmodRegistry
{
    dshash_table_handle record_table_handle;
    dshash_table_handle typmod_table_handle;
    pg_atomic_uint32 next_typmod;
};

入力と出力

本構造体自体はデータ保持用であり、関数ではありません。共有メモリ上の dshash テーブルを介して、TupleDesc の検索や新しい typmod の割り当てを行います。

前提条件と不変条件

  • 共有メモリセグメント上に配置される必要があります。
  • next_typmod はアトミック操作によって管理され、一意な typmod 番号を生成します。

アルゴリズムまたは振る舞い

1. **レコード検索**: record_table_handle を使用して、特定の TupleDesc に一致するエントリを検索します。 2. **typmod 検索**: typmod_table_handle を使用して、typmod から対応する TupleDesc を逆引きします。 3. **typmod 生成**: 新しいレコード型が登録される際、next_typmod をインクリメントして新しい識別子を割り当てます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/session.c: セッション初期化およびアタッチ処理で利用。
  • src/backend/utils/cache/typcache.c: レコード型のキャッシュ検索および共有メモリサイズの計算で利用。
  • **呼び出し先**:
  • SharedRecordTypmodRegistryInit: 本構造体の初期化に使用。
  • dshash API: 内部のハッシュテーブル操作に使用。

関連データ構造

  • dshash_table_handle: 共有メモリ上のハッシュテーブルへのハンドル。
  • SharedRecordTableEntry: record_table_handle に格納されるエントリ。
  • SharedTypmodTableEntry: typmod_table_handle に格納されるエントリ。

失敗モードと境界条件

  • 共有メモリが不足している場合、dshash の操作が失敗する可能性があります。
  • next_typmod が上限に達した場合の挙動については、ソースコードの断片からは明示されていません。

保守担当者向けメモ

  • 本構造体は並列クエリのライフサイクルと密接に関連しています。SharedRecordTypmodRegistryInit を通じた初期化と、セッション終了時のデタッチ処理が正しく行われることを確認してください。
  • dshash のキー定義（SharedRecordTableKey 等）と本構造体の整合性に注意してください。
