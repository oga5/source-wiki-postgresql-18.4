bit_out

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varbit.c:279

目的

BIT 型のデータを SQL クライアントへ送信するための文字列表現に変換します。

シグネチャまたは形状

Datum bit_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される BIT 型の Datum。
  • **出力**: cstring 型（C 文字列）として、ビット列を '0' と '1' の並びで表現した文字列を返します。

前提条件と不変条件

  • 入力データは VarBit 構造体として適切に初期化されている必要があります。
  • VARBIT_CORRECTLY_PADDED(s) マクロにより、最後のバイトの未使用ビットがゼロで埋められていることが保証されている必要があります。

アルゴリズムまたは振る舞い

現在の実装では、varbit_out(fcinfo) を直接呼び出すことで、BIT VARYING 型と共通の出力ロジックを実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（fmgr）経由で呼び出されます。
  • **呼び出し先**: varbit_out を呼び出します。

関連データ構造

  • VarBit: ビット列を保持する構造体。vl_len_（ヘッダ）、bit_len（ビット長）、bit_dat（データ）で構成されます。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性があります。
  • 入力データが不正なパディング状態である場合、出力される文字列が期待と異なる可能性があります。

保守担当者向けメモ

  • ソースコード内に、16進数形式で出力するための代替実装が #if 0 ブロックとして残されていますが、現在は使用されていません。
  • BIT 型と BIT VARYING 型の出力ロジックは同一であるため、変更時は varbit_out との整合性に注意してください。
