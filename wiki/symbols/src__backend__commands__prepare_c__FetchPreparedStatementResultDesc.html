FetchPreparedStatementResultDesc

定義

  • 種別: function
  • 定義箇所: src/backend/commands/prepare.c:465

目的

プリペアドステートメントに関連付けられた結果のタプル記述子（TupleDesc）を取得します。

シグネチャまたは形状

TupleDesc FetchPreparedStatementResultDesc(PreparedStatement *stmt)

入力と出力

  • **入力**: PreparedStatement *stmt - 対象のプリペアドステートメントへのポインタ。
  • **出力**: TupleDesc - ステートメントの結果構造を記述するタプル記述子のコピー。ステートメントに結果がない場合は NULL を返します。

前提条件と不変条件

  • stmt->plansource->fixed_result が真であること（アサーションにより検証）。プリペアドステートメントの結果のタプル記述子は変更されないことが前提です。

アルゴリズムまたは振る舞い

1. stmt->plansource->fixed_result を検証します。 2. stmt->plansource->resultDesc が存在する場合、CreateTupleDescCopy を呼び出してそのコピーを返します。 3. resultDesc が存在しない場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c:2112
  • **呼び出し先**:
  • CreateTupleDescCopy（PostgreSQL内部のメモリ管理関数）

関連データ構造

  • PreparedStatement: プリペアドステートメントの情報を保持する構造体。
  • CachedPlanSource: PreparedStatement 内で計画のソースを管理する構造体。
  • TupleDesc: タプルの構造を定義する記述子。

失敗モードと境界条件

  • stmt が無効なポインタである場合、または stmt->plansource が初期化されていない場合は未定義の動作となります。
  • fixed_result が偽の場合、アサーション違反によりプロセスが停止する可能性があります。

保守担当者向けメモ

  • 本関数は結果のコピーを返すため、呼び出し側で不要になった際に FreeTupleDesc 等で適切にメモリを解放する必要があります。
  • プリペアドステートメントの再検証が不要な設計に基づいているため、キャッシュの整合性チェックは含まれていません。
