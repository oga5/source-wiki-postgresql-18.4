DefineAggregate

定義

  • 種別: function
  • 定義箇所: src/backend/commands/aggregatecmds.c:52

目的

SQL の CREATE AGGREGATE コマンドを処理し、集約関数の定義に必要な属性の解析、権限チェック、およびシステムカタログへの登録準備を行う。

シグネチャまたは形状

ObjectAddress DefineAggregate(ParseState *pstate, List *name, List *args, bool oldstyle, List *parameters, bool replace);

入力と出力

  • **入力**:
  • pstate: パーサー状態。
  • name: 集約関数の名前（修飾名）。
  • args: 引数リスト。
  • oldstyle: 8.2 以前の古い構文形式かどうかのフラグ。
  • parameters: CREATE AGGREGATE で指定された属性（sfunc, stype 等）のリスト。
  • replace: 既存の定義を置換するかどうかのフラグ。
  • **出力**:
  • ObjectAddress: 作成された集約関数のオブジェクトアドレス。

前提条件と不変条件

  • 対象スキーマに対する CREATE 権限が必要。
  • stype（遷移データ型）および sfunc（遷移関数）の指定が必須。
  • mstype（移動集約の遷移型）を指定する場合、msfunc および minvfunc の指定が必須。

アルゴリズムまたは振る舞い

1. **名前解決と権限確認**: QualifiedNameGetCreationNamespace でスキーマを特定し、object_aclcheck で作成権限を検証する。 2. **構文解析**: oldstyle フラグに基づき引数を解釈し、parameters リストを走査して各属性（sfunc, finalfunc, initcond 等）を抽出する。 3. **検証**: 必須属性の有無を確認し、hypothetical 属性が指定された場合は AGGKIND_HYPOTHETICAL として扱う。 4. **カタログ登録準備**: 抽出した情報を基に、後続のカタログ操作（AggregateCreate）へ引き渡すための引数群を構築する。

失敗モードと境界条件

  • 必須属性（stype, sfunc）が欠落している場合、ERRCODE_INVALID_FUNCTION_DEFINITION でエラー終了する。
  • 権限がない場合、aclcheck_error を呼び出す。
  • 不明な属性が指定された場合、警告（WARNING）を発行する。
  • hypothetical 属性が通常の集約関数に対して指定された場合、エラー終了する。

保守担当者向けメモ

  • initcond は text 型として保存されるため、実行時の型入力関数で検証される。
  • 新しい集約属性を追加する際は、parameters を走査するループ内のロジックを更新する必要がある。
  • oldstyle フラグによる互換性維持のため、古い構文（sfunc1, stype1 等）の処理が混在している点に注意すること。
