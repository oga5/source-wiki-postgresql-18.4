pgaio_worker_control_shmem_size

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_worker.c:113

目的

PostgreSQLの非同期I/O（AIO）ワーカープロセス方式において、共有メモリ上に配置される制御構造体 PgAioWorkerControl のために必要なメモリサイズを算出します。

シグネチャまたは形状

size_t pgaio_worker_control_shmem_size(void)

入力と出力

  • 入力: なし
  • 出力: size_t 型のバイト数。PgAioWorkerControl 構造体のヘッダー部分と、最大ワーカー数（MAX_IO_WORKERS）分の PgAioWorkerSlot 配列を格納するために必要な合計サイズを返します。

前提条件と不変条件

  • MAX_IO_WORKERS が定義されている環境で実行される必要があります。
  • 共有メモリの初期化プロセスにおいて、ShmemInitStruct 等へ渡すサイズ計算の一部として利用されます。

アルゴリズムまたは振る舞い

1. offsetof(PgAioWorkerControl, workers) により、構造体内の workers 配列メンバまでのオフセットを取得します。 2. sizeof(PgAioWorkerSlot) * MAX_IO_WORKERS を加算し、全スロットを保持するための領域を算出します。 3. 計算結果を合計サイズとして返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/aio/method_worker.c:127（pgaio_worker_shmem_size 内）
  • src/backend/storage/aio/method_worker.c:151（pgaio_worker_shmem_init 内）
  • 呼び出し先:
  • なし（標準的なC言語の演算および offsetof マクロのみを使用）

関連データ構造

  • PgAioWorkerControl: ワーカーの稼働状況を管理するメインの制御構造体。
  • PgAioWorkerSlot: 個々のワーカープロセスの状態を保持する構造体。

失敗モードと境界条件

  • MAX_IO_WORKERS が極端に大きい場合、計算結果が size_t の範囲を超える可能性がありますが、通常のPostgreSQLの構成では想定されていません。

保守担当者向けメモ

  • 本関数は、共有メモリのレイアウト変更（PgAioWorkerControl 構造体の定義変更）に合わせて更新する必要があります。
  • 構造体のパディングやアライメントについては、コンパイラの offsetof および sizeof の挙動に依存します。
