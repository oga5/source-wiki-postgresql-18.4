pg_rusage_show

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/pg_rusage.c:39

目的

初期スナップショット（PGRUsage）からの経過時間およびCPU使用時間（ユーザー時間・システム時間）を計算し、人間が読みやすいフォーマットの文字列として出力する。

シグネチャまたは形状

const char *pg_rusage_show(const PGRUsage *ru0)

入力と出力

  • **入力**: ru0 - 比較対象となる開始時点のリソース使用状況を保持した PGRUsage 構造体へのポインタ。
  • **出力**: フォーマット済みの文字列を格納した静的バッファへのポインタ。

前提条件と不変条件

  • ru0 は pg_rusage_init() によって適切に初期化されている必要がある。
  • 本関数は静的バッファ（static char result[100]）を再利用して結果を返すため、スレッドセーフではない。同一プロセス内での同時呼び出しは避けること。

アルゴリズムまたは振る舞い

1. pg_rusage_init(&ru1) を呼び出し、現在時刻とリソース使用状況を取得する。 2. 経過時間、ユーザーCPU時間、システムCPU時間のそれぞれについて、マイクロ秒単位の繰り下げ処理（tv_usec が負になる場合の補正）を行う。 3. snprintf を使用し、ru1 と ru0 の差分を計算して文字列に変換する。 4. 変換後の静的バッファ result を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/vacuumlazy.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/catalog/index.c
  • src/backend/commands/analyze.c
  • src/backend/commands/cluster.c
  • src/backend/commands/indexcmds.c
  • src/backend/utils/sort/tuplesort.c
  • **呼び出し先**:
  • pg_rusage_init

関連データ構造

  • PGRUsage: timeval（壁時計時間）と rusage（CPU使用統計）を保持する構造体。

失敗モードと境界条件

  • **静的バッファの競合**: 前述の通り、静的バッファを使用しているため、呼び出し後に結果文字列を保持したい場合は呼び出し側でコピーが必要である。
  • **精度**: マイクロ秒単位の計算を行っているが、表示は小数点以下2桁（10ミリ秒単位）に丸められる。

保守担当者向けメモ

  • 本関数はデバッグやログ出力用である。高頻度な計測や厳密なパフォーマンス分析には適さない。
  • 戻り値のバッファは関数内で静的に確保されているため、マルチスレッド環境での使用は不可。
