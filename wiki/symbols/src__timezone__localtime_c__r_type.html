r_type

定義

  • 種別: enum
  • 定義箇所: src/timezone/localtime.c:65

目的

POSIX 形式のタイムゾーン文字列における、夏時間（DST）の切り替わりルールを指定するための型定義です。

シグネチャまたは形状

enum r_type
{
    JULIAN_DAY,             /* Jn = Julian day */
    DAY_OF_YEAR,            /* n = day of year */
    MONTH_NTH_DAY_OF_WEEK,  /* Mm.n.d = month, week, day of week */
};

入力と出力

  • 本シンボル自体は列挙型であり、入力や出力を持つ関数ではありません。
  • rule 構造体のメンバとして保持され、後続の計算ロジックで参照されます。

前提条件と不変条件

  • rule 構造体内の r_type フィールドに格納され、switch 文による条件分岐の制御に使用されます。

アルゴリズムまたは振る舞い

  • JULIAN_DAY: 閏年を考慮したユリウス日（Jn形式）による指定。
  • DAY_OF_YEAR: 年内の通算日（n形式）による指定。
  • MONTH_NTH_DAY_OF_WEEK: 「第 m 月の第 n 週の d 曜日」（Mm.n.d形式）による指定。

呼び出し元と呼び出し先

  • 呼び出し元（設定箇所）: src/timezone/localtime.c 内の tzparse 関数周辺で、POSIX 文字列の解析結果に基づき各列挙値が代入されます。
  • 呼び出し先（参照箇所）: src/timezone/localtime.c 内の switch (rulep->r_type) 文において、ルールに応じた時刻計算ロジックへ分岐するために使用されます。

関連データ構造

  • rule: r_type をメンバとして保持する構造体。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 新たなルール形式を追加する場合は、本列挙型に値を追加し、switch 文の網羅性を維持する必要があります。
