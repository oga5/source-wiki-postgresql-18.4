ExecEndLockRows

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeLockRows.c:393

目的

LockRows 実行ノードの終了処理を行い、関連するリソースを解放する。

シグネチャまたは形状

void ExecEndLockRows(LockRowsState *node)

入力と出力

  • **入力**: LockRowsState *node - 終了対象となる LockRows ノードの実行時状態構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 node は有効な LockRowsState を指している必要がある。
  • EvalPlanQualEnd は複数回呼び出されても安全である。

アルゴリズムまたは振る舞い

1. node->lr_epqstate に対して EvalPlanQualEnd を呼び出し、同時実行制御（EPQ）に関連するリソースを解放する。 2. outerPlanState(node) を介して下位ノードを取得し、ExecEndNode を呼び出してサブプランツリーの終了処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:746（ExecEndNode 経由の呼び出しが示唆される）。
  • **呼び出し先**:
  • EvalPlanQualEnd
  • ExecEndNode
  • outerPlanState（マクロ）

関連データ構造

  • LockRowsState: LockRows ノードの実行時状態を保持する構造体。lr_epqstate を含む。

失敗モードと境界条件

  • 特記事項なし。

保守担当者向けメモ

  • 本関数は LockRows ノードのクリーンアップを担当する。EPQ（EvalPlanQual）の状態管理と、下位プランノードの終了処理を順次実行する。
  • EvalPlanQualEnd は冪等性を持つため、二重呼び出しによる副作用はない。
