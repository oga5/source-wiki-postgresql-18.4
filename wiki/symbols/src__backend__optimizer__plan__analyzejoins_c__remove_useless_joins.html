remove_useless_joins

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/analyzejoins.c:105

目的

クエリの最適化過程において、結合結果が左側の入力と重複することが証明できる不要な左結合（Left Join）を特定し、クエリから削除・簡略化します。

シグネチャまたは形状

List *remove_useless_joins(PlannerInfo *root, List *joinlist)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報を保持する PlannerInfo 構造体。
  • joinlist: 現在の結合ツリーを構成するリスト。
  • **出力**:
  • 不要な結合が削除され、更新された joinlist。

前提条件と不変条件

  • root->join_info_list に格納された SpecialJoinInfo を走査対象とします。
  • 結合の削除は、右側のリレーションが単一のベースリレーション（baserel）である場合にのみ実行可能です。
  • 結合削除に伴い、PlannerInfo 内の関連データ構造（join_info_list、joinlist 等）の整合性が維持される必要があります。

アルゴリズムまたは振る舞い

1. root->join_info_list を走査し、join_is_removable を用いて削除可能な結合を判定します。 2. 削除可能な結合が見つかった場合、以下の処理を行います。

  • innerrelid を特定し、remove_leftjoinrel_from_query を呼び出してクエリから当該リレーションを排除します。
  • remove_rel_from_joinlist を呼び出し、joinlist から該当リレーションを削除します（削除数が1でない場合はエラーを発生させます）。
  • 処理済みの SpecialJoinInfo を root->join_info_list から削除します。

    • 3. 結合削除により他の結合が新たに削除可能になる可能性があるため、リストの走査を最初からやり直します（restart ラベルへのジャンプ）。 4. 削除可能な結合がなくなるまで繰り返し、最終的な joinlist を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planmain.c:228
  • **呼び出し先**:
  • join_is_removable
  • remove_leftjoinrel_from_query
  • remove_rel_from_joinlist
  • list_delete_cell

関連データ構造

  • PlannerInfo: プランナの全体状態を管理。
  • SpecialJoinInfo: 結合の特性（左結合、セミジョイン等）を保持。
  • List: 結合ツリーの構造を保持。

失敗モードと境界条件

  • remove_rel_from_joinlist が指定されたリレーションを joinlist 内で見つけられない場合、elog(ERROR) により処理を中断します。
  • join_is_removable が右側リレーションが単一の baserel ではないと判定した場合、その結合はスキップされます。

保守担当者向けメモ

  • 本関数は、結合削除後に attr_needed ビットが変化することで、以前は削除不可だった結合が削除可能になるケースを考慮し、再帰的な走査（restart）を行っています。
  • 結合削除に伴う PlaceHolderVar の評価位置（ph_eval_at）の整合性については、呼び出し先の remove_leftjoinrel_from_query 等が責任を負います。
