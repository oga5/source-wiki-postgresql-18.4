bbsink_server_new

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_server.c:59

目的

PostgreSQL サーバー上のローカルファイルシステムにベースバックアップを保存するための bbsink インスタンスを生成・初期化する。

シグネチャまたは形状

bbsink *bbsink_server_new(bbsink *next, char *pathname)

入力と出力

  • **入力**:
  • next: 次のシンクへのポインタ。
  • pathname: バックアップ保存先の絶対パス。
  • **出力**:
  • 初期化された bbsink 構造体へのポインタを返す。

前提条件と不変条件

  • 呼び出し元は ROLE_PG_WRITE_SERVER_FILES 権限を保持している必要がある（関数内で検証）。
  • pathname は絶対パスでなければならない。
  • 指定されたディレクトリは、存在しないか、存在する場合は空でなければならない。

アルゴリズムまたは振る舞い

1. bbsink_server 構造体をメモリ確保し、bbsink_server_ops を割り当てる。 2. ROLE_PG_WRITE_SERVER_FILES 権限をチェックし、不足している場合は ERROR をスローする。 3. pathname が絶対パスであることを確認し、相対パスの場合は ERROR をスローする。 4. pg_check_dir を使用してディレクトリの状態を確認する。

  • 存在しない場合: MakePGDirectory で作成する。
  • 空の場合: 正常終了。
  • 空でない場合: ERROR をスローする。
  • アクセス不可の場合: ERROR をスローする。

    • 5. 初期化済みの bbsink ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup_target.c
  • **呼び出し先**:
  • palloc0, StartTransactionCommand, CommitTransactionCommand, has_privs_of_role, GetUserId, ereport, is_absolute_path, pg_check_dir, MakePGDirectory

関連データ構造

  • bbsink: バックアップストリームの基底インターフェース。
  • bbsink_server: ローカルファイル保存用の状態を保持する構造体。
  • bbsink_server_ops: bbsink インターフェースの関数ポインタテーブル。

失敗モードと境界条件

  • 権限不足: ERRCODE_INSUFFICIENT_PRIVILEGE を発生させる。
  • 相対パス指定: ERRCODE_INVALID_NAME を発生させる。
  • ディレクトリが空でない: ERRCODE_DUPLICATE_FILE を発生させる。
  • ディレクトリ作成失敗やアクセス権限エラー: errcode_for_file_access() に基づくエラーを発生させる。

保守担当者向けメモ

  • 本関数は durable_rename を前提としたディレクトリ管理を行うため、バックアップ先の整合性を厳格にチェックする設計となっている。
  • fsync 失敗時の挙動については、リカバリ不能な状態を避けるため PANIC ではなく ERROR を使用する方針がとられている。
