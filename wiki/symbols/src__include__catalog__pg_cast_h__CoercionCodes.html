CoercionCodes

定義

  • 種別: enum
  • 定義箇所: src/include/catalog/pg_cast.h:75

目的

データ型変換（キャスト）が許可されるコンテキストを定義し、SQL クエリ実行時の型変換の妥当性を制御します。

シグネチャまたは形状

typedef enum CoercionCodes
{
    COERCION_CODE_IMPLICIT = 'i',
    COERCION_CODE_ASSIGNMENT = 'a',
    COERCION_CODE_EXPLICIT = 'e',
} CoercionCodes;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: char 型の値（'i', 'a', 'e'）としてシステムカタログ pg_cast に格納されます。

前提条件と不変条件

  • pg_cast テーブルの castcontext 列に保存される値と対応します。
  • データベース内では char 型として保持されますが、クライアントコードでは本列挙型として扱われます。

アルゴリズムまたは振る舞い

  • COERCION_CODE_IMPLICIT ('i'): 式の評価時など、暗黙的な型変換が許可されるコンテキスト。
  • COERCION_CODE_ASSIGNMENT ('a'): 代入操作時など、代入による型変換が許可されるコンテキスト。
  • COERCION_CODE_EXPLICIT ('e'): CAST 構文や :: 演算子による明示的な型変換が許可されるコンテキスト。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数情報は、提供されたソース抜粋からは特定できません。

関連データ構造

  • FormData_pg_cast: pg_cast システムカタログの行構造。
  • CoercionMethod: キャストの変換手法を定義する列挙型。
  • CoercionContext: src/include/nodes/primnodes.h で定義される、内部的に対応する列挙型。

失敗モードと境界条件

  • 定義された文字（'i', 'a', 'e'）以外の値が castcontext に設定された場合、型変換ロジックが予期しない動作をする可能性があります。

保守担当者向けメモ

  • Catalog.pm がこのファイルを解析してカタログ初期化スクリプトを生成するため、列挙子の追加や変更には注意が必要です。
  • pg_cast の castcontext 列は char 型であるため、本列挙型との整合性を維持してください。
