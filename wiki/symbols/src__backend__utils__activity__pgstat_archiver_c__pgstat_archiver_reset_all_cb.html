pgstat_archiver_reset_all_cb

定義

  • src/backend/utils/activity/pgstat_archiver.c:73

目的

WAL アーカイバの統計情報をリセットします。累積値をゼロにするのではなく、現在の統計値を reset_offset として保存することで、以降のスナップショット取得時に差分計算を行うための基準点を設定します。

シグネチャまたは形状

void pgstat_archiver_reset_all_cb(TimestampTz ts)

入力と出力

  • **入力**: ts (リセット時刻を示す TimestampTz)
  • **出力**: なし（共有メモリ上の統計構造体 PgStatShared_Archiver を更新）

前提条件と不変条件

  • 共有メモリ上のアーカイバ統計構造体 pgStatLocal.shmem->archiver が初期化されている必要があります。
  • 統計情報の更新プロトコルに従い、LWLock による排他制御が行われます。

アルゴリズムまたは振る舞い

1. 共有メモリ上のアーカイバ統計構造体へのポインタを取得します。 2. LWLockAcquire を呼び出し、stats_shmem->lock を排他モードで獲得します。 3. pgstat_copy_changecounted_stats を使用し、現在の統計値 (stats_shmem->stats) を reset_offset にコピーします。この際、changecount を更新して整合性を保証します。 4. stats_shmem->stats.stat_reset_timestamp に引数 ts を代入し、リセット時刻を記録します。 5. LWLockRelease を呼び出し、ロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c:396 において、統計リセット用コールバックテーブルの要素として参照されています。
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease
  • pgstat_copy_changecounted_stats

関連データ構造

  • PgStatShared_Archiver: アーカイバ統計を保持する共有メモリ構造体。
  • pgStatLocal: 統計システムのローカル状態を保持するグローバル変数。

失敗モードと境界条件

  • 共有メモリが未初期化の場合、ポインタ参照によりクラッシュする可能性があります。
  • ロック獲得時にデッドロックが発生する可能性は、統計システムのロック順序規則に依存します。

保守担当者向けメモ

  • 本関数は累積値を直接リセットするのではなく、オフセットを保存するデルタ計算方式を採用しています。統計の連続性を維持するため、リセット後の統計値取得時には必ず pgstat_archiver_snapshot_cb を通じてオフセットの減算が行われる必要があります。
