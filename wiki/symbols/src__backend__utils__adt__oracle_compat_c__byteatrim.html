byteatrim

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oracle_compat.c:637

目的

bytea 型のデータに対して、指定された文字セットに含まれる文字を文字列の両端から除去（トリミング）します。

シグネチャまたは形状

Datum byteatrim(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_BYTEA_PP(0): トリミング対象の bytea 型データ。
  • PG_GETARG_BYTEA_PP(1): 除去対象の文字セットを含む bytea 型データ。
  • **出力**:
  • PG_RETURN_BYTEA_P(ret): トリミング後の新しい bytea 型データ。

前提条件と不変条件

  • 入力引数は PostgreSQL の関数呼び出し規約（PG_FUNCTION_ARGS）に従う必要があります。
  • dobyteatrim 関数が内部的に呼び出され、両端（左側および右側）のトリミングが有効化されます。

アルゴリズムまたは振る舞い

1. 引数から対象文字列と除去対象文字セットを取得します。 2. dobyteatrim 関数を呼び出し、左側と右側の両方をトリミングする設定（true, true）で処理を実行します。 3. 処理結果を bytea 型として返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: dobyteatrim
  • **呼び出し元**: ソースコード内の記述から、SQL インターフェース経由で呼び出されることが示唆されていますが、具体的な呼び出し元関数名はソース excerpt に含まれていません。

関連データ構造

  • bytea: PostgreSQL の可変長バイナリデータ型。

失敗モードと境界条件

  • 入力データが NULL の場合、PG_GETARG_BYTEA_PP マクロの仕様に従い、呼び出し元で適切に処理される必要があります。
  • メモリ不足時には dobyteatrim 内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は dobyteatrim のラッパーです。トリミングロジックの変更が必要な場合は dobyteatrim を参照してください。
  • bytealtrim（左側のみ）や他のトリミング関数と実装パターンを共有しています。
