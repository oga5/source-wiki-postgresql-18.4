gist_qe

定義

  • 種別: function
  • 定義箇所: contrib/ltree/_ltree_gist.c:441

目的

GiSTインデックスにおいて、ltreeのキー（シグネチャ）が指定されたlqueryの条件を満たす可能性があるかを判定する。

シグネチャまたは形状

static bool gist_qe(ltree_gist *key, lquery *query, int siglen)

入力と出力

  • **入力**:
  • key: 判定対象のltree_gist構造体。
  • query: 検索条件となるlquery構造体。
  • siglen: シグネチャの長さ（バイト単位）。
  • **出力**:
  • bool: 条件を満たす可能性がある場合はtrue、明らかに一致しない場合はfalseを返す。

前提条件と不変条件

  • keyがLTG_ISALLTRUEフラグを持つ場合、無条件でtrueを返す。
  • lqueryの各レベルにおいて、シグネチャによる判定が可能な場合（LQL_CANLOOKSIGN）のみ、ビットベクトルを用いたフィルタリングを行う。

アルゴリズムまたは振る舞い

1. keyが全ビットセット状態（LTG_ISALLTRUE）であればtrueを返す。 2. queryの各レベル（lquery_level）を順次走査する。 3. 各レベルでシグネチャ判定が可能な場合、そのレベルに含まれる全バリアント（lquery_variant）を走査する。 4. バリアントのハッシュ値がkeyのビットベクトル（LTG_SIGN）に含まれているかを確認する。 5. レベル内のいずれのバリアントもビットベクトルに含まれていない場合、そのパスは一致しないと判断しfalseを返す。 6. 全レベルのチェックを通過すればtrueを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/_ltree_gist.c 内の関数
  • contrib/ltree/ltree_gist.c 内の関数
  • **呼び出し先**:
  • LQUERY_FIRST, LTG_SIGN, LTG_ISALLTRUE, LQL_CANLOOKSIGN, LQL_FIRST, GETBIT, AHASHVAL, LVAR_NEXT, LQL_NEXT（これらはマクロまたはインライン関数）

関連データ構造

  • ltree_gist: GiSTインデックスに格納されるltreeのシグネチャ情報。
  • lquery: 検索条件を保持する構造体。
  • lquery_level, lquery_variant: lqueryを構成する階層および値の構造体。

失敗モードと境界条件

  • queryのレベル数が0の場合、ループをスキップしてtrueを返す。
  • siglenが不適切な場合、GETBITおよびAHASHVALの計算結果が範囲外となる可能性がある。

保守担当者向けメモ

  • 本関数はGiSTインデックスの検索効率を左右するフィルタリング処理である。
  • LQL_CANLOOKSIGNが偽となる条件（シグネチャで判定できないクエリ）については、この関数ではスキップされるため、呼び出し側で別途詳細な照合が必要となる。
