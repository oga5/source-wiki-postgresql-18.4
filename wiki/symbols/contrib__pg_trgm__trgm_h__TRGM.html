TRGM

定義

  • 種別: struct
  • 定義箇所: contrib/pg_trgm/trgm.h:58

目的

PostgreSQL の pg_trgm モジュールにおいて、トライグラム（3文字の連続）の集合を保持するための可変長データ構造を提供します。インデックス（GiST/GIN）のキーとして使用されます。

シグネチャまたは形状

typedef struct
{
	int32		vl_len_;
	uint8		flag;
	char		data[FLEXIBLE_ARRAY_MEMBER];
} TRGM;

入力と出力

  • **入力**: generate_trgm や generate_wildcard_trgm などの関数により、文字列からトライグラムが抽出され、本構造体に格納されます。
  • **出力**: cnt_sml 等の関数が本構造体を受け取り、類似度計算や包含判定の結果を返します。

前提条件と不変条件

  • vl_len_ は PostgreSQL の可変長データ（varlena）ヘッダーであり、直接操作してはなりません。
  • flag フィールドにより、data 部が配列形式（ARRKEY）かビットマップ形式（SIGNKEY）かを識別します。

アルゴリズムまたは振る舞い

  • flag の値に基づき、マクロ（ISARRKEY, ISSIGNKEY, ISALLTRUE）を用いて内部データの解釈を切り替えます。
  • インデックス検索時には、この構造体内のフラグを参照して、効率的な比較アルゴリズムを選択します。

呼び出し元と呼び出し先

  • **呼び出し元**: trgm_gin.c および trgm_gist.c 内の関数が、インデックス操作の過程で本構造体を生成・参照します。
  • **呼び出し先**: generate_trgm, cnt_sml, trgm_contained_by 等の関数が、引数として TRGM 構造体を受け取ります。

関連データ構造

  • trgm: 基本単位となる char[3] 型。
  • BITVECP: ビットマップ形式のデータにアクセスするためのポインタ型。

失敗モードと境界条件

  • vl_len_ の不適切な操作はメモリ破壊を引き起こすため、PostgreSQL の標準的な varlena 操作関数を介する必要があります。
  • data 部のサイズは vl_len_ に依存するため、構造体のコピーやメモリ確保時には適切なサイズ計算が必須です。

保守担当者向けメモ

  • flag フィールドのビット定義（ARRKEY, SIGNKEY 等）を変更する場合は、関連するすべてのマクロおよび trgm_op.c 等のロジックとの整合性を確認してください。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズ計算には VARSIZE() マクロを使用してください。
