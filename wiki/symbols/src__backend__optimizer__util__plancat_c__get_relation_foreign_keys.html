get_relation_foreign_keys

定義

  • **種別**: function
  • **定義箇所**: src/backend/optimizer/util/plancat.c:588

目的

リレーションの外部キー（FK）情報を取得し、クエリ内の他のリレーションとの結合条件として利用可能な ForeignKeyOptInfo 構造体を生成して PlannerInfo に格納する。

シグネチャまたは形状

static void get_relation_foreign_keys(PlannerInfo *root, RelOptInfo *rel, Relation relation, bool inhparent)

入力と出力

  • **入力**:
  • root: 現在のクエリのプランナ情報。
  • rel: 対象リレーションの最適化情報（RelOptInfo）。
  • relation: 対象リレーションのキャッシュされた記述子。
  • inhparent: 対象が継承ツリーの親であるかを示すフラグ。
  • **出力**: なし（副作用として root->fkey_list に ForeignKeyOptInfo を追加）。

前提条件と不変条件

  • rel->reloptkind が RELOPT_BASEREL であること。
  • クエリ内に少なくとも2つ以上のリレーションが存在すること。
  • inhparent が false であること（継承親のFKは無視される）。
  • RelationGetRelid(relation) と cachedfk->conrelid が一致すること。

アルゴリズムまたは振る舞い

1. **ガード条件**: ベースリレーションでない場合、またはクエリ内のリレーション数が2未満の場合は早期リターンする。継承親の場合も同様にリターンする。 2. **FK取得**: RelationGetFKeyList を呼び出し、リレーションキャッシュからFKリストを取得する。 3. **フィルタリングと生成**:

  • 強制されていない（conenforced が偽）FKをスキップする。
  • クエリ内の rtable を走査し、FKの参照先（confrelid）と一致する RangeTblEntry を探す。
  • 継承テーブルや自己参照FKを除外し、条件を満たすものに対して ForeignKeyOptInfo を作成する。
  • 作成した情報を root->fkey_list に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: get_relation_info（src/backend/optimizer/util/plancat.c:552 付近）。
  • **呼び出し先**: RelationGetFKeyList, RelationGetRelid, makeNode, lappend。

関連データ構造

  • ForeignKeyOptInfo: 結合最適化のために生成されるFK情報保持構造体。
  • ForeignKeyCacheInfo: リレーションキャッシュから取得されるFKの生データ。
  • PlannerInfo: クエリ全体のプランニング状態を保持。

失敗モードと境界条件

  • クエリ内に単一のリレーションしかない場合、FKの探索は行われない。
  • 継承親リレーションに対してはFK処理がスキップされる。
  • 参照先がクエリ内に存在しない場合、ForeignKeyOptInfo は生成されない。

保守担当者向けメモ

  • 本関数はリレーションキャッシュから取得したリストを直接参照するため、関数内でカタログアクセスを行ってはならない（キャッシュフラッシュによる不整合を防ぐため）。
  • match_foreign_keys_to_quals() が後続処理で不要な ForeignKeyOptInfo を削除することを前提としている。
