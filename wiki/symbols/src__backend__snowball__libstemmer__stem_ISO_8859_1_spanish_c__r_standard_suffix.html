r_standard_suffix

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/snowball/libstemmer/stem_ISO_8859_1_spanish.c:711

目的

スペイン語のステミング処理において、名詞や形容詞に関連する標準的な接尾辞を単語末尾から検索し、条件を満たす場合に削除または置換を行う。

シグネチャまたは形状

static int r_standard_suffix(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（Snowballステミング環境構造体）
  • **出力**: int（成功時は1、失敗時は0、エラー時は負の値を返す）

前提条件と不変条件

  • z->c（現在のカーソル位置）が有効な範囲にあること。
  • r_mark_regions によって、単語内の領域（R1, R2）が事前に計算・設定されていること。
  • among 構造体配列 a_6 が接尾辞の検索対象として定義されていること。

アルゴリズムまたは振る舞い

1. **初期チェック**: 単語末尾の文字が特定のビットマスク条件を満たすか確認し、効率的なマッチングを行う。 2. **接尾辞検索**: find_among_b を使用して、単語末尾から a_6 に定義された接尾辞を後方検索する。 3. **条件分岐**: マッチした接尾辞の種類（among_var）に応じて以下の処理を行う：

  • **削除**: r_R2（または r_R1）で指定領域内であることを確認後、slice_del で接尾辞を削除。
  • **置換**: r_R2 で領域確認後、slice_from_s で別の文字列に置換。
  • **複合処理**: 一部のケースでは、削除後にさらに別の接尾辞（a_3, a_4, a_5 など）を再帰的または連続的に検索・削除する。

    • 4. **終了**: 処理が完了すれば1を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: spanish_ISO_8859_1_stem（メインエントリーポイント）から呼び出されることが想定される。
  • **呼び出し先**: find_among_b, r_R1, r_R2, slice_del, slice_from_s, eq_s_b。
  • **注記**: ツールによる解析では、他の言語（Catalan, Dutch, French等）のソースファイルからも同名の関数が参照されていることが示唆されているが、本ファイル内での直接的な呼び出し関係はソースコードの文脈に依存する。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • among: 接尾辞マッチングのためのトライ木構造をフラット化した配列。

失敗モードと境界条件

  • find_among_b がマッチしない場合、0を返して終了する。
  • r_R1 や r_R2 の領域チェックで条件を満たさない場合、処理を中断して0を返す。
  • メモリ操作関数（slice_del 等）が負の値を返した場合、エラーとしてそのまま伝播させる。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによって自動生成されている。ロジックの変更が必要な場合は、直接Cファイルを編集せず、対応する .sbl ソースファイルを修正すること。
  • among_var の各ケースは、スペイン語の文法規則に基づく接尾辞の優先順位を反映している。
