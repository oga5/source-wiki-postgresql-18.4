astreamer_inject_file

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/astreamer_inject.c:218

目的

アーカイブストリーム内に新しいメンバー（ファイル）を注入するための汎用ヘルパー関数です。

シグネチャまたは形状

void astreamer_inject_file(astreamer *streamer, char *pathname, char *data, int len)

入力と出力

  • **入力**:
  • streamer: 処理対象の astreamer オブジェクト。
  • pathname: 注入するファイルのパス名。
  • data: 注入するファイルの内容。
  • len: data の長さ。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • astreamer フレームワークのパイプライン上で動作することを前提とします。
  • 注入されるファイルは通常ファイルとして扱われます（is_regular = true）。
  • UID/GID は歴史的な慣習に基づき、それぞれ 04000 および 02000 に固定されます。

アルゴリズムまたは振る舞い

1. astreamer_member 構造体を初期化し、指定されたパス名、サイズ、および固定の権限・属性を設定します。 2. astreamer_content を呼び出し、以下の順序でストリームにデータを送出します。

  • ASTREAMER_MEMBER_HEADER: ヘッダー情報の生成を後続のストリーマーに委譲。
  • ASTREAMER_MEMBER_CONTENTS: ファイルの実データを転送。
  • ASTREAMER_MEMBER_TRAILER: トレーラー情報の生成を後続のストリーマーに委譲。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/astreamer_inject.c 内のリカバリ設定注入処理。
  • src/bin/pg_basebackup/pg_basebackup.c 内の backup_manifest 生成処理。
  • **呼び出し先**:
  • astreamer_content（astreamer フレームワークの基底関数）。

関連データ構造

  • astreamer: ストリーミング処理の基底構造体。
  • astreamer_member: 注入するファイルのメタデータ（パス、サイズ、権限等）を保持する構造体。

失敗モードと境界条件

  • 本関数自体はエラーハンドリングを行いません。後続の astreamer_content 呼び出し先で発生するエラーに依存します。
  • アーカイブ形式に応じたヘッダーやトレーラーの生成は本関数内では行われず、後続のストリーマーが適切に処理する必要があります。

保守担当者向けメモ

  • 本関数で設定される UID/GID は歴史的な慣習によるものであり、特定の意味を持ちません。
  • 注入されたデータは、後続のストリーマーによってアーカイブ形式（tar等）に再パッケージ化される必要があります。
