pg_u_prop_alphabetic

定義

  • 種別: function
  • 定義箇所: src/common/unicode_category.c:110

目的

指定された Unicode コードポイントが「アルファベット（Alphabetic）」プロパティを持つかどうかを判定します。

シグネチャまたは形状

bool pg_u_prop_alphabetic(pg_wchar code)

入力と出力

  • **入力**: pg_wchar code（判定対象の Unicode コードポイント）
  • **出力**: bool（該当する場合は true、それ以外は false）

前提条件と不変条件

  • 入力されるコードポイントは UTF-8 エンコーディングに基づいた pg_wchar 型である必要があります。
  • Unicode 規格の安定性ポリシーに基づき、カテゴリおよびプロパティ値は固定されているものとして扱われます。

アルゴリズムまたは振る舞い

1. **ASCII 範囲の判定**: code が 0x80 未満の場合、unicode_opt_ascii テーブルを参照し、PG_U_PROP_ALPHABETIC ビットマスクを用いて定数時間で判定結果を返します。 2. **範囲外の判定**: 0x80 以上のコードポイントに対しては、unicode_alphabetic テーブルに対して range_search を実行し、該当する範囲内にコードポイントが含まれるかを二分探索で特定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/unicode/category_test.c:68
  • src/common/unicode_category.c:222
  • **呼び出し先**:
  • range_search（src/common/unicode_category.c 内のユーティリティ関数）

関連データ構造

  • unicode_opt_ascii: ASCII 文字のプロパティを保持する静的テーブル。
  • unicode_alphabetic: 非 ASCII 文字のアルファベット範囲を保持する静的テーブル。

失敗モードと境界条件

  • 0x80 未満の入力に対してはテーブル参照による高速な判定が行われます。
  • 0x80 以上の入力に対しては range_search による二分探索が行われるため、テーブルの定義範囲外のコードポイントに対する挙動は range_search の実装に依存します。

保守担当者向けメモ
