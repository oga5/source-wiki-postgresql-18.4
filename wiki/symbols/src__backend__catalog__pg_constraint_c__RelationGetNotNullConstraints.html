RelationGetNotNullConstraints

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_constraint.c:833

目的

指定されたリレーション（テーブル）に関連付けられた NOT NULL 制約のリストを取得する。

シグネチャまたは形状

List *RelationGetNotNullConstraints(Oid relid, bool cooked, bool include_noinh)

入力と出力

  • **入力**:
  • relid (Oid): 対象テーブルの OID。
  • cooked (bool): true の場合、内部処理用の CookedConstraint 構造体リストを返す。false の場合、構文解析後の Constraint ノードリストを返す。
  • include_noinh (bool): true の場合、継承されない（connoinherit）制約も結果に含める。
  • **出力**:
  • List *: NOT NULL 制約のリスト。

前提条件と不変条件

  • pg_constraint システムカタログに対する AccessShareLock を取得してスキャンを行う。
  • relid に対応するテーブルが存在し、アクセス可能であること。

アルゴリズムまたは振る舞い

1. pg_constraint を conrelid インデックスを用いてスキャンする。 2. 各行について以下を判定する:

  • contype が CONSTRAINT_NOTNULL であること。
  • connoinherit が設定されている場合、include_noinh が true でなければスキップする。

    • 3. extractNotNullColumn を呼び出し、制約対象の列番号を取得する。 4. cooked フラグに応じて以下のいずれかを生成し、リストに追加する:

  • cooked == true: CookedConstraint 構造体を割り当て、制約のメタデータを格納する。
  • cooked == false: Constraint ノードを作成し、列名や検証状態などの情報を設定する。

    • 5. スキャン終了後、リレーションをクローズしてリストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c
  • src/backend/parser/parse_utilcmd.c
  • **呼び出し先**:
  • table_open, table_close
  • systable_beginscan, systable_getnext, systable_endscan
  • extractNotNullColumn
  • palloc, pstrdup, lappend
  • makeNode (cooked=false の場合)

関連データ構造

  • pg_constraint (システムカタログ)
  • CookedConstraint (cooked=true の場合)
  • Constraint (cooked=false の場合)

失敗モードと境界条件

  • 対象テーブルに NOT NULL 制約が存在しない場合、空のリスト（NIL）を返す。
  • relid が無効な場合、table_open 等でエラーが発生する可能性がある。

保守担当者向けメモ

  • NOT NULL 制約は pg_attribute ではなく pg_constraint で管理されているため、列の制約情報を取得する際は本関数が必須となる。
  • cooked モードと通常モードで生成されるデータ構造が異なるため、呼び出し側の用途に合わせて適切にフラグを指定すること。
