SharedTuplestoreChunk

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/sort/sharedtuplestore.c:42

目的

並列クエリ実行時、一時ファイルに書き込まれるタプルを管理するためのチャンクヘッダー構造体です。データを STS_CHUNK_PAGES 単位で管理し、巨大なタプルのオーバーフロー処理を透過的に行うために使用されます。

シグネチャまたは形状

typedef struct SharedTuplestoreChunk
{
    int         ntuples;        /* このチャンク内のタプル数 */
    int         overflow;       /* オーバーフロー時、これを含む総チャンク数 */
    char        data[FLEXIBLE_ARRAY_MEMBER]; /* 可変長データ領域 */
} SharedTuplestoreChunk;

入力と出力

  • **入力**: sts_puttuple 実行時に、タプルデータが data 領域に書き込まれます。
  • **出力**: sts_parallel_scan_next 実行時に、読み取り側がこの構造体を介してタプルを復元します。

前提条件と不変条件

  • STS_CHUNK_HEADER_SIZE は offsetof(SharedTuplestoreChunk, data) として定義されており、ヘッダーサイズとして使用されます。
  • チャンクサイズを超える巨大なタプルは、複数のオーバーフローチャンクに分割されます。

アルゴリズムまたは振る舞い

  • 書き込み時、各バックエンドは自身の担当ファイルに対して SharedTuplestoreChunk 形式でデータを書き込みます。
  • 読み取り時、chunk_header.overflow を参照することで、分割されたチャンクをスキップまたは連続して読み込む制御が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/sharedtuplestore.c 内の sts_puttuple（書き込み時）、sts_parallel_scan_next（読み取り時）にて参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • SharedTuplestore: 共有メモリ上の制御構造体。
  • SharedTuplestoreAccessor: 各バックエンドのローカル操作用構造体。write_chunk フィールドで本構造体を保持します。

失敗モードと境界条件

  • チャンクサイズを超えるタプルは overflow フィールドで管理されますが、この際、読み取り側が正しくヘッダーを解析しない場合、データ読み取りの不整合が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を使用しているため、メモリ割り当て時には STS_CHUNK_HEADER_SIZE を考慮したサイズ計算が必要です。
  • 並列スキャン時の読み取り順序は保証されないため、この構造体を用いたデータ復元は、あくまで並列処理の分配用として設計されています。
