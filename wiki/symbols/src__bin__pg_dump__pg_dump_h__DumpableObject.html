DumpableObject

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_dump.h:147

目的

pg_dump において、データベースから抽出されたあらゆるダンプ対象オブジェクトをメモリ上で統一的に表現するための基底構造体です。オブジェクトの識別、依存関係の追跡、およびダンプ処理の制御を担います。

シグネチャまたは形状

typedef struct _dumpableObject
{
	DumpableObjectType objType;
	CatalogId	catId;
	DumpId		dumpId;
	char	   *name;
	struct _namespaceInfo *namespace;
	DumpComponents dump;
	DumpComponents dump_contains;
	DumpComponents components;
	bool		ext_member;
	bool		depends_on_ext;
	DumpId	   *dependencies;
	int			nDeps;
	int			allocDeps;
} DumpableObject;

入力と出力

  • **入力**: システムカタログから読み取られた各オブジェクトの属性情報。
  • **出力**: pg_dump の各処理フェーズ（ソート、アーカイブ生成）で利用される、依存関係とダンプ制御情報を含む構造体インスタンス。

前提条件と不変条件

  • name フィールドは NULL であってはなりません。
  • catId はカタログ化されていないオブジェクトの場合、ゼロとなります。
  • dumpId は AssignDumpId() 関数によって割り当てられる必要があります。
  • dependencies 配列は nDeps 個の有効な DumpId を保持し、allocDeps はその割り当て済みサイズを管理します。

アルゴリズムまたは振る舞い

  • **依存関係管理**: dependencies 配列により、オブジェクト間の順序制約をグラフとして保持します。これは sortDumpableObjects によるダンプ順序決定の基礎となります。
  • **ダンプ制御**: DumpComponents 型のビットマスク（dump, dump_contains, components）を使用して、定義・データ・ACL 等のどの要素をダンプ対象とするかを個別に制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/common.c 内の AssignDumpId() や recordAdditionalCatalogID() が本構造体を操作・参照します。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは含みません。

関連データ構造

  • DumpableObjectType: オブジェクトの種類を識別する列挙型。
  • CatalogId: システムカタログ上のオブジェクトを一意に識別する構造体。
  • DumpComponents: ダンプ対象要素を指定するビットマスク。
  • NamespaceInfo: 所属する名前空間情報。

失敗モードと境界条件

  • メモリ不足時: dependencies 配列の拡張（pg_realloc_array 等）に失敗する可能性があります。
  • 依存関係の循環: nDeps の管理やソートアルゴリズムにおいて、循環参照が発生した場合の挙動は pg_dump_sort.c の実装に依存します。

保守担当者向けメモ

  • 本構造体は pg_dump の実行期間中生存するため、メモリ管理には注意が必要です。
  • 新しいオブジェクト型を追加する場合は、DumpableObjectType の更新と併せて、本構造体を用いた依存関係構築ロジックが適切に機能することを確認してください。
