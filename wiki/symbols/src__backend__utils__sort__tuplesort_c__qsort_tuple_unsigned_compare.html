qsort_tuple_unsigned_compare

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplesort.c:494

目的

ソート対象のタプル（SortTuple）を、符号なし比較ロジックに基づいて比較する。

シグネチャまたは形状

static pg_attribute_always_inline int
qsort_tuple_unsigned_compare(SortTuple *a, SortTuple *b, Tuplesortstate *state)

入力と出力

  • **入力**:
  • a, b: 比較対象の SortTuple 構造体へのポインタ。
  • state: ソート状態を保持する Tuplesortstate 構造体へのポインタ。
  • **出力**:
  • 比較結果を返す。a < b なら負、a > b なら正、a == b なら 0 を返す。

前提条件と不変条件

  • state->base.sortKeys[0] に有効なソートキーが設定されていること。
  • ApplyUnsignedSortComparator が datum1 および isnull1 を正しく処理できること。

アルゴリズムまたは振る舞い

1. ApplyUnsignedSortComparator を呼び出し、第1ソートキーに基づいて比較を行う。 2. 比較結果が 0 以外であれば、その値を直ちに返す。 3. 比較結果が 0（同値）の場合:

  • state->base.onlyKey が存在する場合（単一キーソートの場合）、追加の比較を行わず 0 を返す。
  • それ以外の場合、state->base.comparetup_tiebreak を呼び出し、タイブレーク（順序決定）を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/sort/tuplesort.c 内のソートテンプレート（ST_COMPARE マクロ経由）。
  • **呼び出し先**:
  • ApplyUnsignedSortComparator
  • state->base.comparetup_tiebreak（関数ポインタ経由）

関連データ構造

  • SortTuple: ソート対象のタプルを保持する構造体。
  • Tuplesortstate: ソート操作の全体状態を保持する構造体。

失敗モードと境界条件

  • state が NULL の場合や、sortKeys が初期化されていない場合は未定義の動作となる。
  • タイブレーク関数が定義されていない状態で同値が発生した場合の挙動は、comparetup_tiebreak の実装に依存する。

保守担当者向けメモ

  • 本関数は pg_attribute_always_inline が指定されており、パフォーマンス向上のためインライン展開を意図している。
  • ApplyUnsignedSortComparator は符号なし比較を行うため、ソート対象のデータ型が符号なしとして扱われることを前提としている点に注意すること。
