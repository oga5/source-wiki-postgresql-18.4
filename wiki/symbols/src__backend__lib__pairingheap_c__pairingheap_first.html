pairingheap_first

定義

  • 種別: function
  • 定義箇所: src/backend/lib/pairingheap.c:129

目的

ペアリングヒープ内の最小（または最大）要素であるルートノードを返します。

シグネチャまたは形状

pairingheap_node *pairingheap_first(pairingheap *heap)

入力と出力

  • **入力**: heap (pairingheap *) - 対象のペアリングヒープ構造体へのポインタ。
  • **出力**: pairingheap_node * - ヒープのルートノード。

前提条件と不変条件

  • ヒープが空であってはなりません（pairingheap_is_empty(heap) が真である場合、アサーションエラーが発生します）。

アルゴリズムまたは振る舞い

1. pairingheap_is_empty を使用して、ヒープが空でないことを検証します。 2. ヒープ構造体 heap が保持するルートノードへのポインタ ph_root を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeIndexscan.c
  • src/backend/replication/logical/reorderbuffer.c
  • src/backend/utils/time/snapmgr.c
  • **呼び出し先**:
  • pairingheap_is_empty (マクロまたはインライン関数と推測されるが、本ソース excerpt に定義は含まれない)

関連データ構造

  • pairingheap: ヒープ全体を管理する構造体。
  • pairingheap_node: ヒープ内の各要素を表すノード構造体。

失敗モードと境界条件

  • ヒープが空の状態で呼び出された場合、Assert によりプログラムが停止します。

保守担当者向けメモ

  • 本関数はヒープのルートを返すのみであり、ノードの削除やメモリ解放は行いません。
  • 呼び出し側は、返されたノードがヒープの生存期間中にのみ有効であることを前提とする必要があります。
