pg_cryptohash_type

定義

  • 種別: enum
  • 定義箇所: src/include/common/cryptohash.h:19

目的

PostgreSQLの暗号学的ハッシュ関数（MD5, SHA1, SHA2系）を抽象化し、統一されたインターフェースで利用するための識別子として機能します。

シグネチャまたは形状

typedef enum
{
    PG_MD5 = 0,
    PG_SHA1,
    PG_SHA224,
    PG_SHA256,
    PG_SHA384,
    PG_SHA512,
} pg_cryptohash_type;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: pg_cryptohash_create() 等の関数へ渡す引数として使用されます。

前提条件と不変条件

  • 各列挙値は、pg_cryptohash_create() 関数において、対応するハッシュアルゴリズムのコンテキストを生成するためのキーとして使用されます。
  • 内部的にOpenSSLが利用できない環境では、src/common/cryptohash.c 内のフォールバック実装がこの型に基づいて処理を分岐します。

アルゴリズムまたは振る舞い

  • 指定されたハッシュアルゴリズムを識別し、計算コンテキストの初期化や処理の切り替えを制御します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の例）:
  • src/backend/libpq/auth-scram.c
  • src/backend/libpq/crypt.c
  • src/backend/utils/adt/cryptohashfuncs.c
  • src/common/cryptohash.c
  • src/common/cryptohash_openssl.c
  • 呼び出し先: 情報なし。

関連データ構造

  • pg_cryptohash_ctx: この列挙型で指定されたアルゴリズムの状態を保持する不透明（opaque）な構造体。

失敗モードと境界条件

  • 未定義の列挙値が渡された場合の挙動は、各実装関数（pg_cryptohash_create 等）の内部ロジックに依存します。

保守担当者向けメモ

  • 新しいハッシュアルゴリズムを追加する場合は、この列挙型に値を追加し、src/common/cryptohash.c および関連する実装ファイルで対応する処理を実装する必要があります。
