RemoveObjects

定義

  • 種別: function
  • 定義箇所: src/backend/commands/dropcmds.c:52

目的

SQLの DROP 文（DROP FUNCTION, DROP TYPE 等）の実行を統括し、指定されたオブジェクトの解決、権限チェック、および依存関係を考慮した一括削除処理を行う。

シグネチャまたは形状

void RemoveObjects(DropStmt *stmt)

入力と出力

  • **入力**: DropStmt *stmt（削除対象オブジェクトのリスト、削除タイプ、IF EXISTS フラグ、削除動作 behavior を含む構造体）
  • **出力**: なし（成功時はオブジェクトを削除し、失敗時はエラーをスローする）

前提条件と不変条件

  • stmt->objects 内の各オブジェクトに対して get_object_address が呼び出される。
  • stmt->missing_ok が偽の場合、オブジェクトが存在しないと get_object_address がエラーをスローする。
  • 削除対象が OBJECT_FUNCTION であり、かつ集約関数（Aggregate）である場合はエラーとなる。
  • 削除処理の完了まで、対象オブジェクトには AccessExclusiveLock が保持される。

アルゴリズムまたは振る舞い

1. new_object_addresses() で空のオブジェクトリストを作成する。 2. stmt->objects を走査し、各オブジェクトの ObjectAddress を取得する。 3. オブジェクトが存在しない場合、missing_ok が真であれば does_not_exist_skipping を呼び出し、警告を出して次のオブジェクトへ進む。 4. 存在する場合は、所有権チェックを行い、一時名前空間へのアクセスがあればトランザクションフラグ XACT_FLAGS_ACCESSEDTEMPNAMESPACE を更新する。 5. 取得したアドレスを add_exact_object_address でリストに追加する。 6. ループ終了後、performMultipleDeletions を呼び出し、収集した全オブジェクトを stmt->behavior に基づいて一括削除する。 7. 最後に free_object_addresses でメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（SQL解析後の実行フェーズ）
  • **呼び出し先**:
  • get_object_address（オブジェクトの特定）
  • does_not_exist_skipping（存在しない場合の警告生成）
  • object_ownercheck / check_object_ownership（権限検証）
  • performMultipleDeletions（実際の削除実行）
  • table_close（リレーションのクローズ）

関連データ構造

  • DropStmt: 削除対象の定義を保持する構造体。
  • ObjectAddresses: 削除対象オブジェクトのリストを管理する構造体。
  • ObjectAddress: 個別のオブジェクトを識別するOIDとクラスIDのペア。

失敗モードと境界条件

  • 指定されたオブジェクトが存在しない場合、missing_ok が偽であればエラーとなる。
  • DROP FUNCTION で集約関数を指定した場合、ERRCODE_WRONG_OBJECT_TYPE エラーが発生する。
  • 所有権チェックに失敗した場合、エラーが発生する。

保守担当者向けメモ

  • 本関数はすべての DROP コマンドを扱うわけではない。サポートされない型については elog(ERROR, ...) が発生する。
  • IF EXISTS の警告メッセージ生成は does_not_exist_skipping に委譲されており、詳細なエラーハンドリングが必要な場合は同関数を修正すること。
  • 削除処理は performMultipleDeletions に依存しているため、依存関係の解決ロジックは src/backend/catalog/dependency.c を参照すること。
