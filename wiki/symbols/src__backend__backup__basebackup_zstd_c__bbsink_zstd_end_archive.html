bbsink_zstd_end_archive

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_zstd.c:234

目的

Zstandard圧縮を用いたアーカイブ処理を終了し、圧縮コンテキストに残存するデータをフラッシュして後続のシンクへ転送する。

シグネチャまたは形状

static void bbsink_zstd_end_archive(bbsink *sink)

入力と出力

  • **入力**: bbsink *sink （bbsink_zstd 構造体としてキャストして使用）
  • **出力**: なし（副作用として後続のシンクへデータを転送）

前提条件と不変条件

  • sink は bbsink_zstd 型のインスタンスであること。
  • mysink->cctx が初期化済みであること。
  • mysink->base.bbs_next が有効な後続シンクを指していること。

アルゴリズムまたは振る舞い

1. ZSTD_e_end モードで ZSTD_compressStream2 を呼び出し、圧縮ストリームの終了処理を行う。 2. 出力バッファ（zstd_outBuf）の空き容量が不足する場合、bbsink_archive_contents を呼び出して現在のバッファ内容を後続シンクへフラッシュし、バッファをリセットする。 3. 圧縮データが残存しなくなるまで（yet_to_flush > 0）上記処理をループする。 4. 圧縮処理完了後、バッファに未転送のデータがあれば再度 bbsink_archive_contents を呼び出す。 5. 最後に bbsink_forward_end_archive を呼び出し、アーカイブ終了の通知を後続シンクへ伝播させる。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_zstd_ops 構造体を通じて、バックアップパイプラインの終了処理時に呼び出される。
  • **呼び出し先**:
  • bbsink_archive_contents（後続シンクへのデータ転送）
  • ZSTD_compressStream2（libzstd による圧縮終了処理）
  • bbsink_forward_end_archive（終了通知の転送）

関連データ構造

  • bbsink_zstd: 圧縮状態とバッファを保持する構造体。
  • ZSTD_CCtx: libzstd の圧縮コンテキスト。
  • bbsink: 基底となるシンクインターフェース。

失敗モードと境界条件

  • ZSTD_compressStream2 がエラーを返した場合、elog(ERROR) により処理を中断する。
  • バッファのフラッシュ時に後続シンクのバッファ容量が不足する場合、動的にバッファを再利用して転送を行う。

保守担当者向けメモ

  • 本関数は USE_ZSTD が定義されている環境でのみコンパイルされる。
  • ZSTD_compressBound(0) を使用して、フラッシュ時に必要な最小バッファサイズを確保している。
  • マニフェストファイル等の非圧縮データは本関数を通さず透過的に処理されるため、本関数はあくまで圧縮対象のアーカイブ終了処理に特化している。
