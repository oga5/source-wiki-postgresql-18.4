scan_directory

定義

  • 種別: function
  • 定義箇所: src/bin/pg_checksums/pg_checksums.c:298

目的

PostgreSQLデータディレクトリを再帰的に走査し、各データファイルのチェックサム処理（検証または書き込み）を行う、あるいは処理対象の総サイズを計算する。

シグネチャまたは形状

int64 scan_directory(const char *basedir, const char *subdir, bool sizeonly)

入力と出力

  • **入力**:
  • basedir: 走査対象の親ディレクトリパス。
  • subdir: 走査対象のサブディレクトリ名。
  • sizeonly: true の場合、ファイル処理を行わずサイズ合計のみを計算する。
  • **出力**:
  • 走査した全ファイルの合計サイズ（int64）。

前提条件と不変条件

  • 対象ディレクトリは opendir で開ける必要がある。
  • pg_tblspc ディレクトリ内の走査時は、TABLESPACE_VERSION_DIRECTORY の存在が前提となる。
  • .、..、一時ファイル、一時ディレクトリ、および .DS_Store は走査対象から除外される。

アルゴリズムまたは振る舞い

1. 指定されたパスを結合し、ディレクトリを開く。 2. ディレクトリ内の各エントリをループ処理する。 3. 通常ファイルの場合:

  • skipfile() で除外対象か判定する。
  • ファイル名からセグメント番号とフォーク情報を抽出する。
  • only_filenode が指定されている場合、一致しないファイルはスキップする。
  • sizeonly が false の場合、scan_file() を呼び出してチェックサム処理を実行する。

    • 4. ディレクトリまたはシンボリックリンクの場合:

  • pg_tblspc 内であれば、リンク先を解決して再帰的に scan_directory を呼び出す。
  • それ以外は、現在のパスを基準に再帰的に scan_directory を呼び出す。

    • 5. 最後にディレクトリを閉じ、累積したサイズを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • main 関数（src/bin/pg_checksums/pg_checksums.c:593-600）から、global、base、pg_tblspc に対して呼び出される。
  • 自身（再帰呼び出し）: src/bin/pg_checksums/pg_checksums.c:417, 423。
  • **呼び出し先**:
  • scan_file(): 通常ファイルの処理時に呼び出される。
  • skipfile(): ファイルの除外判定に使用。
  • opendir, readdir, closedir, lstat, snprintf, pg_fatal 等の標準/PostgreSQLライブラリ関数。

関連データ構造

  • struct dirent: ディレクトリ内のエントリ情報。
  • struct stat: ファイル属性情報。

失敗モードと境界条件

  • ディレクトリが開けない場合、pg_fatal によりプログラムが終了する。
  • lstat に失敗した場合、pg_fatal によりプログラムが終了する。
  • ファイル名から抽出したセグメント番号が不正（atoi の結果が0でファイル名が数値でない場合など）な場合、pg_fatal により終了する。

保守担当者向けメモ

  • 進捗表示（--progress）を利用する場合、sizeonly=true で一度全走査が行われるため、大規模なデータディレクトリでは処理時間が倍増する点に留意すること。
  • 新しいシステムファイルを除外対象に追加する場合は skipfile() を更新する必要がある。
