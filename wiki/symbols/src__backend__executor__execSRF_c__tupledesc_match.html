tupledesc_match

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execSRF.c:942

目的

集合を返す関数（SRF）が生成する行の型定義（TupleDesc）と、クエリ側が期待する行の型定義が互換性を持つか検証する。

シグネチャまたは形状

static void tupledesc_match(TupleDesc dst_tupdesc, TupleDesc src_tupdesc)

入力と出力

  • dst_tupdesc (入力): クエリが期待する行の型定義。
  • src_tupdesc (入力): 関数が実際に返す行の型定義。
  • 戻り値: なし。型が一致しない場合は ereport(ERROR, ...) により処理を中断する。

前提条件と不変条件

  • 両方の TupleDesc は有効なポインタである必要がある。
  • 属性数（natts）が一致していること。
  • 各属性において、型がバイナリ互換（IsBinaryCoercible）であるか、または物理的なストレージ属性（長さ・アライメント）が一致していること。

アルゴリズムまたは振る舞い

1. **属性数の比較**: dst_tupdesc->natts と src_tupdesc->natts を比較し、不一致であればエラーを報告する。 2. **属性ごとの検証**: 各属性について以下の順でチェックを行う。

  • IsBinaryCoercible を用いて、関数側の型からクエリ側の型へバイナリ変換可能か確認する。可能であれば次の属性へ進む。
  • 変換不可の場合、attisdropped（削除済み属性）でない限りエラーを報告する。
  • 削除済み属性の場合、attlen（長さ）と attalign（アライメント）を比較し、物理ストレージが一致しない場合はエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execSRF.c 内の以下の箇所から呼び出されることが確認されている。
  • src/backend/executor/execSRF.c:420
  • src/backend/executor/execSRF.c:911
  • **呼び出し先**:
  • TupleDescAttr (マクロ)
  • IsBinaryCoercible
  • format_type_be
  • ereport

関連データ構造

  • TupleDesc: 行の記述子。
  • Form_pg_attribute: 属性のメタデータ構造体。

失敗モードと境界条件

  • **ERRCODE_DATATYPE_MISMATCH**: 属性数、型互換性、または削除済み属性の物理ストレージ定義が一致しない場合に発生する。
  • 属性数が 0 の場合、ループは実行されず正常終了する。

保守担当者向けメモ

  • 本関数は SRF の実行時チェックにおいて重要な役割を果たす。動的な型（RECORD 型など）を扱う際の厳密な検証ポイントであるため、型変換ロジックの変更時には本関数の検証条件を再確認すること。
