normal_rand_fctx

定義

  • 種別: struct
  • 定義箇所: contrib/tablefunc/tablefunc.c:89

目的

正規分布に従う乱数生成関数 normal_rand において、関数呼び出し間（SRFの複数回呼び出し）で状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    float8      mean;       /* 分布の平均 */
    float8      stddev;     /* 分布の標準偏差 */
    float8      carry_val;  /* 次回使用する生成済みの値 */
    bool        use_carry;  /* carry_val を使用するかどうかのフラグ */
} normal_rand_fctx;

入力と出力

本構造体はデータ保持用であり、直接的な入出力関数はありません。normal_rand 関数内でメモリ確保および値の読み書きが行われます。

前提条件と不変条件

  • normal_rand 関数が SRF（Set Returning Function）として動作する際、初回呼び出し時に palloc を介してメモリが確保される必要があります。
  • use_carry が true の場合、carry_val には Box-Muller 法で生成された有効な正規乱数が格納されている必要があります。

アルゴリズムまたは振る舞い

Box-Muller 法は一度の計算で2つの正規乱数を生成するため、本構造体は生成された片方の値を carry_val に保持し、次回の呼び出し時に再計算なしでその値を提供することで効率化を図ります。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/tablefunc/tablefunc.c 内の normal_rand 関数。
  • 呼び出し先: 情報なし。

関連データ構造

  • normal_rand: 本構造体を利用する正規分布乱数生成関数。

失敗モードと境界条件

  • メモリ不足時: palloc 呼び出し時にエラーが発生する可能性があります。
  • 状態の不整合: use_carry フラグと carry_val の値が同期していない場合、不正な乱数が出力される可能性があります。

保守担当者向けメモ

  • 本構造体は normal_rand 関数の実行コンテキストとしてのみ使用されます。
  • carry_val の管理は normal_rand 内のロジックに依存しているため、乱数生成アルゴリズムを変更する際は本構造体のフィールド定義と整合性を取る必要があります。
