initHyperLogLog

定義

  • 種別: function
  • 定義箇所: src/backend/lib/hyperloglog.c:65

目的

HyperLogLog (HLL) アルゴリズムで使用する状態構造体 hyperLogLogState を初期化し、指定された精度に基づいてレジスタ配列の確保および補正係数 alphaMM の計算を行う。

シグネチャまたは形状

void initHyperLogLog(hyperLogLogState *cState, uint8 bwidth)

入力と出力

  • **入力**:
  • cState: 初期化対象となる hyperLogLogState 構造体へのポインタ。
  • bwidth: レジスタ数を決定するためのビット幅（4〜16の範囲）。
  • **出力**: なし（cState が指す構造体の各フィールドが更新される）。

前提条件と不変条件

  • bwidth は 4 以上 16 以下である必要がある。
  • cState は有効なメモリ領域を指している必要がある。
  • cState->hashesArr は palloc0 によってゼロ初期化される。

アルゴリズムまたは振る舞い

1. bwidth の範囲チェックを行い、範囲外の場合は elog(ERROR) を発生させる。 2. bwidth からレジスタ数 nRegisters ($2^{bwidth}$) を算出する。 3. palloc0 を使用して、レジスタ配列 hashesArr に必要なメモリを確保する。 4. レジスタ数に応じた補正係数 alpha を決定する（16, 32, 64 の場合は固定値、それ以外は計算式を使用）。 5. 最終的な推定値算出に使用する alphaMM（alpha * nRegisters^2）を計算し、構造体に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAgg.c
  • src/backend/lib/hyperloglog.c (内部再帰的呼び出し)
  • src/backend/utils/adt/mac.c
  • src/backend/utils/adt/network.c
  • src/backend/utils/adt/numeric.c
  • src/backend/utils/adt/uuid.c
  • src/backend/utils/adt/varlena.c
  • **呼び出し先**:
  • palloc0 (PostgreSQL メモリ管理関数)
  • elog (エラー報告関数)

関連データ構造

  • hyperLogLogState: HLL の状態を保持する構造体。レジスタ幅、レジスタ数、配列サイズ、ハッシュ配列、および補正係数 alphaMM を含む。

失敗モードと境界条件

  • bwidth が 4 未満または 16 を超える場合、elog(ERROR) により処理が中断される。
  • メモリ不足時には palloc0 がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は Hideaki Ohno 氏の C++ 実装に基づいている。
  • alpha の計算ロジックは、HLL アルゴリズムのバイアス補正に直結するため、変更時は関連論文の定数定義を確認すること。
  • hashesArr は負の無限大ではなくゼロで初期化される点に注意（Coupon Collector Problem の議論に基づく）。
