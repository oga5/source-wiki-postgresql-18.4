CreateDBStrategy

定義

  • 種別: enum
  • 定義箇所: src/backend/commands/dbcommands.c:82

目的

データベース作成（CREATE DATABASE）時における、テンプレートデータベースからの複製手法を定義する。

シグネチャまたは形状

typedef enum CreateDBStrategy
{
	CREATEDB_WAL_LOG,
	CREATEDB_FILE_COPY,
} CreateDBStrategy;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: CREATEDB_WAL_LOG または CREATEDB_FILE_COPY のいずれかの値。

前提条件と不変条件

  • データベース作成処理において、いずれかの戦略が選択される必要がある。
  • CREATEDB_WAL_LOG は pg_class をスキャンしてブロック単位でコピーし、WALに書き込む手法。
  • CREATEDB_FILE_COPY はファイルシステムレベルでディレクトリをコピーし、チェックポイントを強制する手法。

アルゴリズムまたは振る舞い

  • CREATEDB_WAL_LOG: データベースの物理構造をWALログ経由で再構築する。
  • CREATEDB_FILE_COPY: 物理ディレクトリの直接コピーによりデータベースを複製する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/dbcommands.c 内のデータベース作成ロジック（createdb() 等）で使用される。
  • 呼び出し先: 情報なし。

関連データ構造

  • CreateDatabaseUsingWalLog(): CREATEDB_WAL_LOG 戦略の実装関数。
  • CreateDatabaseUsingFileCopy(): CREATEDB_FILE_COPY 戦略の実装関数。

失敗モードと境界条件

  • 適切な戦略が選択されない場合、データベース作成処理が正常に進行しない可能性がある。
  • 物理的なファイル操作を伴うため、権限やディスク容量不足による失敗が想定される。

保守担当者向けメモ

  • 本列挙型は src/backend/commands/dbcommands.c 内のデータベース複製ロジックの分岐に使用される。
  • 新たな複製戦略を追加する場合は、本列挙型を拡張し、対応する処理関数を実装する必要がある。
