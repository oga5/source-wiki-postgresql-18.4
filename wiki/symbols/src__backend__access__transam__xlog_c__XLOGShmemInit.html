XLOGShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlog.c:4955

目的

PostgreSQL の起動時または共有メモリ初期化時に、WAL 管理に必要な共有メモリ領域（XLogCtl）および制御ファイル（ControlFile）のメモリイメージを確保・初期化する。

シグネチャまたは形状

void XLOGShmemInit(void)

入力と出力

  • **入力**: なし（グローバルな共有メモリ状態および ControlFile ポインタを参照）。
  • **出力**: なし（副作用として共有メモリ上の XLogCtl および ControlFile が初期化される）。

前提条件と不変条件

  • ShmemInitStruct を介して共有メモリを割り当てるため、本関数は共有メモリの初期化フェーズで呼び出される必要がある。
  • XLogCtlData のサイズは XLogRecPtr フィールドの配置により、適切なアライメントが維持されているものと想定される。
  • WALInsertLocks は XLOG_BLCKSZ 境界および WALInsertLockPadded のサイズに合わせて適切にアライメントされる。

アルゴリズムまたは振る舞い

1. **メモリ確保**: ShmemInitStruct を使用し、XLogCtl および ControlFile 用の共有メモリ領域を確保する。 2. **再初期化の判定**: 既に共有メモリが確保されている場合（foundCFile または foundXLog が真）、既存の WALInsertLocks を設定して終了する。 3. **初期化処理**:

  • XLogCtl 構造体をゼロクリアする。
  • ローカルに読み込まれていた ControlFile の内容を共有メモリへコピーする。
  • XLogCtl 内の xlblocks（各バッファの LSN）、WALInsertLocks（挿入ロック）、および pages（WAL バッファページ）の領域を計算・割り当て、初期化する。
  • XLogCtl の基本フィールド（SharedRecoveryState 等）を設定する。
  • 各種スピンロック（insertpos_lck, info_lck）およびアトミック変数（logInsertResult 等）を初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/ipci.c:288 (共有メモリ初期化シーケンス)
  • **呼び出し先**:
  • ShmemInitStruct, AllocSetContextCreate, MemoryContextAllowInCriticalSection (WAL_DEBUG時), pfree, memset, memcpy, pg_atomic_init_u64, LWLockInitialize, TYPEALIGN, SpinLockInit

関連データ構造

  • XLogCtlData: WAL 管理の全状態を保持する共有メモリ上の構造体。
  • ControlFileData: pg_control ファイルのメモリ内イメージ。
  • WALInsertLockPadded: WAL 挿入ロックのパディング付き構造体。
  • pg_atomic_uint64: LSN 等の更新に使用されるアトミック型。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、ShmemInitStruct 経由でエラーが発生する。
  • WAL_DEBUG が有効な場合、メモリ割り当て失敗時に PANIC を引き起こす可能性がある。
  • foundCFile と foundXLog の一方が真で他方が偽の場合、Assert により停止する。

保守担当者向けメモ

  • XLogCtl のレイアウト変更時は、allocptr を用いたポインタ計算が各フィールドのサイズとアライメントを正しく反映しているか確認すること。
  • XLOGbuffers の値に基づいてメモリ領域が動的に確保されるため、バッファ数変更時の影響範囲に注意が必要。
