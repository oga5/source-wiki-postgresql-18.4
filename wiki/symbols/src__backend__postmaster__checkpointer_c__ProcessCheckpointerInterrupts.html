ProcessCheckpointerInterrupts

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/checkpointer.c:641

目的

チェックポインタープロセスにおいて、保留中のシグナルや設定変更要求、メモリコンテキストのログ出力要求を処理する。

シグネチャまたは形状

void ProcessCheckpointerInterrupts(void)

入力と出力

  • 入力: なし（グローバルなフラグ変数を参照）
  • 出力: なし（副作用として設定の再読み込みやメモリログ出力を行う）

前提条件と不変条件

  • チェックポインタープロセスのメインループ内で呼び出されることを想定している。
  • グローバルなフラグ（ProcSignalBarrierPending, ConfigReloadPending, LogMemoryContextPending）の状態に基づいて動作する。

アルゴリズムまたは振る舞い

1. **プロセスシグナルバリア**: ProcSignalBarrierPending が真の場合、ProcessProcSignalBarrier() を呼び出す。 2. **設定の再読み込み**: ConfigReloadPending が真の場合、以下の処理を行う。

  • フラグを false にリセットする。
  • ProcessConfigFile(PGC_SIGHUP) を呼び出し、設定ファイルを再読み込みする。
  • UpdateSharedMemoryConfig() を呼び出し、変更された設定値を共有メモリに反映させる。

    • 3. **メモリコンテキストのログ出力**: LogMemoryContextPending が真の場合、ProcessLogMemoryContextInterrupt() を呼び出す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/checkpointer.c 内のメインループおよび各処理ポイント（行164, 367, 544, 623）。
  • 呼び出し先: ProcessProcSignalBarrier, ProcessConfigFile, UpdateSharedMemoryConfig, ProcessLogMemoryContextInterrupt。

関連データ構造

  • ProcSignalBarrierPending (グローバルフラグ)
  • ConfigReloadPending (グローバルフラグ)
  • LogMemoryContextPending (グローバルフラグ)

失敗モードと境界条件

  • 設定ファイルの読み込みや共有メモリの更新中にエラーが発生した場合、PostgreSQLの標準的なエラーハンドリング（sigsetjmp 等）に依存する。

保守担当者向けメモ

  • チェックポインターはシャットダウン時に最後に終了するプロセスであるため、設定変更の共有メモリへの反映など、他のプロセスに影響する重要なタスクを担っている。
  • 新たな割り込み処理を追加する場合は、本関数内で適切にフラグチェックを行うこと。
