AnlIndexData

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/analyze.c:61

目的

テーブルのインデックスに対する統計情報収集プロセスにおいて、対象インデックスのメタデータおよび収集対象の属性情報を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct AnlIndexData
{
	IndexInfo  *indexInfo;		/* BuildIndexInfo result */
	double		tupleFract;		/* fraction of rows for partial index */
	VacAttrStats **vacattrstats;	/* index attrs to analyze */
	int			attr_cnt;
} AnlIndexData;

入力と出力

  • **入力**: palloc0 を用いてメモリ確保され、indexInfo や vacattrstats が設定されます。
  • **出力**: 収集された統計情報は vacattrstats を通じて後続の統計更新処理へ渡されます。

前提条件と不変条件

  • vacattrstats は attr_cnt 個の要素を持つ配列である必要があります。
  • indexInfo は有効な IndexInfo 構造体を指している必要があります。

アルゴリズムまたは振る舞い

1. analyze.c 内の統計収集処理において、対象となるインデックスごとに AnlIndexData の配列が割り当てられます。 2. 各インデックスの属性情報が vacattrstats に格納され、部分インデックスの場合は tupleFract に行の割合が設定されます。 3. 統計計算の過程で、この構造体を通じてインデックス属性のサンプリングや解析が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/analyze.c 内の関数（do_analyze_rel 等）が、インデックス解析の準備段階でこの構造体をインスタンス化し、管理します。
  • **呼び出し先**: 具体的な呼び出し先（関数）の詳細は、提供されたソース抜粋からは特定できません。

関連データ構造

  • IndexInfo: インデックスのメタデータ。
  • VacAttrStats: 列ごとの統計情報を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: palloc0 によるメモリ確保が失敗した場合、システムはエラーを発生させます。
  • attr_cnt が 0 の場合、インデックス属性の解析はスキップされます。

保守担当者向けメモ

  • 本構造体は analyze.c 内でのみ使用される一時的な管理用データです。
  • インデックスの統計計算ロジックを変更する際は、vacattrstats のライフサイクルとメモリ管理（anl_context）に注意してください。
