pgstat_report_inj

定義

  • **種別**: 関数
  • **定義箇所**: src/test/modules/injection_points/injection_stats.c:163

目的

指定された名前のインジェクションポイントが実行された回数を、PostgreSQLの統計サブシステム（pgstat）を通じて記録・集計します。

シグネチャまたは形状

void pgstat_report_inj(const char *name)

入力と出力

  • **入力**: name (const char *) - 実行されたインジェクションポイントの識別名。
  • **出力**: なし（副作用として統計データが更新されます）。

前提条件と不変条件

  • inj_stats_loaded および inj_stats_enabled が真である場合にのみ処理が実行されます。
  • PGSTAT_KIND_INJECTION 統計種別が事前に登録されている必要があります。

アルゴリズムまたは振る舞い

1. 統計機能が無効な場合は直ちに終了します。 2. PGSTAT_INJ_IDX(name) マクロを使用して、インジェクションポイント名からハッシュキーを生成します。 3. pgstat_prep_pending_entry を呼び出し、ローカルの pending 領域から対象エントリの参照を取得します。 4. 取得したエントリの numcalls カウンタをインクリメントします。 5. 共有メモリへの直接書き込みは行わず、ローカルの pending 領域を更新することで高頻度な呼び出し時のオーバーヘッドを抑制します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/injection_points/injection_points.c (複数箇所)
  • **呼び出し先**:
  • pgstat_prep_pending_entry (PostgreSQL統計サブシステム)
  • PGSTAT_INJ_IDX (マクロ)

関連データ構造

  • PgStat_StatInjEntry: 統計データを保持する構造体。
  • PgStat_EntryRef: pgstat サブシステムが管理するエントリ参照。

失敗モードと境界条件

  • 統計機能が無効な場合、呼び出しは無視されます。
  • name に対応するエントリが未作成の場合、pgstat_prep_pending_entry の仕様に従い、必要に応じてエントリの準備が行われます。

保守担当者向けメモ

  • 本関数は injection_points.c から頻繁に呼び出されるため、処理は最小限に抑えられています。
  • PGSTAT_KIND_INJECTION は 25 とハードコードされているため、他の統計種別と競合しないよう注意してください。
