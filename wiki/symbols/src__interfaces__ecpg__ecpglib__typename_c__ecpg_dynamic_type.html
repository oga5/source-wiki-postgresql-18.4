ecpg_dynamic_type

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/typename.c:72

目的

PostgreSQLの内部型OIDを、SQL標準の型識別子（SQL3_*）へ変換します。

シグネチャまたは形状

int ecpg_dynamic_type(Oid type)

入力と出力

  • **入力**: Oid type（PostgreSQLのシステムカタログにおける型OID）
  • **出力**: int（対応するSQL標準の型識別子。該当しない場合は 0）

前提条件と不変条件

  • 本関数は状態を持たない純粋な変換関数です。
  • 入力されたOIDが既知の型（BOOLOID, INT2OID, TEXTOID 等）である場合、対応する SQL3_* 定数を返します。

アルゴリズムまたは振る舞い

switch-case 文を用いたルックアップテーブルとして動作します。 1. 入力された Oid を評価します。 2. 一致するケースがあれば、対応する SQL3 定数（例: BOOLOID なら SQL3_BOOLEAN）を返します。 3. 一致するケースがない場合、デフォルト値として 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/descriptor.c: get_int_item 内およびログ出力にて使用。
  • src/interfaces/ecpg/ecpglib/execute.c: SQL実行時の型判定に使用。
  • **呼び出し先**:
  • 本関数は特定の関数を呼び出しません。

関連データ構造

  • catalog/pg_type_d.h: 入力値となる Oid 定数（INT4OID 等）の定義元。
  • sql3types.h: 出力値となる SQL3_* 定数の定義元。

失敗モードと境界条件

  • 未知の Oid が渡された場合、エラーを発生させるのではなく 0 を返します。呼び出し側でこの戻り値を適切に処理する必要があります。

保守担当者向けメモ

  • 新しいPostgreSQLの型をECPGでサポートする必要が生じた場合は、switch 文に新しい case を追加してください。
  • 本関数は ecpglib の型解決の基盤であるため、変更時は descriptor.c や execute.c への影響を確認してください。
