CreateForeignDataWrapper

定義

  • 種別: function
  • 定義箇所: src/backend/commands/foreigncmds.c:568

目的

SQLの CREATE FOREIGN DATA WRAPPER コマンドを処理し、新しい外部データラッパー（FDW）をシステムカタログ pg_foreign_data_wrapper に登録する。

シグネチャまたは形状

ObjectAddress CreateForeignDataWrapper(ParseState *pstate, CreateFdwStmt *stmt)

入力と出力

  • **入力**:
  • pstate: パーサーの状態情報。
  • stmt: CreateFdwStmt 構造体（FDW名、ハンドラ関数、バリデーター関数、オプションを含む）。
  • **出力**:
  • ObjectAddress: 作成されたFDWオブジェクトのクラスIDとオブジェクトID。

前提条件と不変条件

  • 実行ユーザーはスーパーユーザーである必要がある。
  • 同名のFDWが既に存在してはならない。
  • pg_foreign_data_wrapper テーブルに対する RowExclusiveLock が保持される。

アルゴリズムまたは振る舞い

1. **権限チェック**: superuser() を呼び出し、スーパーユーザー権限を確認する。 2. **重複確認**: GetForeignDataWrapperByName を使用し、同名のFDWが存在しないことを検証する。 3. **カタログ準備**: GetNewOidWithIndex で新しいOIDを取得し、FDW名、所有者（現在のユーザー）、ハンドラ関数、バリデーター関数をタプル値として設定する。 4. **オプション変換**: parse_func_options で関数オプションを解析し、transformGenericOptions を通じてバリデーターによる検証済みのオプション配列を生成する。 5. **カタログ挿入**: heap_form_tuple と CatalogTupleInsert を使用してシステムカタログへ書き込む。 6. **依存関係記録**: ハンドラ関数、バリデーター関数、所有者、および現在の拡張機能に対する依存関係を pg_depend に登録する。 7. **フック実行**: InvokeObjectPostCreateHook を呼び出し、作成後の通知を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（SQL解析後のユーティリティ実行フロー）。
  • **呼び出し先**:
  • table_open / table_close
  • superuser
  • GetForeignDataWrapperByName
  • GetNewOidWithIndex
  • parse_func_options
  • transformGenericOptions
  • CatalogTupleInsert
  • recordDependencyOn / recordDependencyOnOwner / recordDependencyOnCurrentExtension
  • InvokeObjectPostCreateHook

関連データ構造

  • CreateFdwStmt: SQLパーサーが生成するFDW作成用ステートメント構造体。
  • pg_foreign_data_wrapper: FDW定義を保持するシステムカタログ。
  • ObjectAddress: 作成されたオブジェクトを特定するための識別子。

失敗モードと境界条件

  • **権限エラー**: スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー終了する。
  • **重複エラー**: 同名のFDWが既に存在する場合、ERRCODE_DUPLICATE_OBJECT でエラー終了する。

保守担当者向けメモ

  • 本関数はDDL処理の一部であり、実行時のFDW動作（fdwapi.c 等）とは分離されている。
  • オプションのバリデーションは transformGenericOptions に依存しているため、バリデーションロジックの変更時は同関数を確認すること。
