having_collation_ctx

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/planner.c:139

目的

HAVING 句の照合順序（collation）の競合を検出する処理において、走査中のコンテキスト情報を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	Index		group_rtindex;
	List	   *ancestor_collids;
} having_collation_ctx;

入力と出力

本構造体は関数間でコンテキストを渡すためのデータ保持用であり、直接的な入力・出力関数ではありません。

前提条件と不変条件

  • group_rtindex は、現在処理中のグループ化対象の範囲テーブルインデックスを指します。
  • ancestor_collids は、照合順序の競合チェックに必要な親階層の照合順序 ID リストを保持します。

アルゴリズムまたは振る舞い

find_having_collation_conflicts 関数および having_collation_conflict_walker 関数において、クエリツリーを走査する際の現在の状態（どのグループ範囲を対象としているか、どの照合順序が有効か）を追跡するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planner.c 内の find_having_collation_conflicts 関数でインスタンス化されます。
  • **呼び出し先**: having_collation_conflict_walker 関数に引数として渡され、ツリー走査中に参照・更新されます。

関連データ構造

  • List: ancestor_collids フィールドで使用。
  • Index: group_rtindex フィールドで使用。

失敗モードと境界条件

  • ancestor_collids が空リストの場合、照合順序の競合チェックは行われません。
  • 走査対象のノードが group_rtindex と一致しない場合、照合順序の検証はスキップされます。

保守担当者向けメモ

  • 本構造体は planner.c 内の局所的な解析用コンテキストです。
  • 照合順序の競合検出ロジックを変更する際は、having_collation_conflict_walker 内での ancestor_collids の更新ルールを確認してください。
