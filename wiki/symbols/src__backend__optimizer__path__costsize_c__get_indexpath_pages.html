get_indexpath_pages

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/costsize.c:972

目的

ビットマップインデックスパス（BitmapAndPath または BitmapOrPath）または単一のインデックスパス（IndexPath）が参照するインデックスの総ページ数を再帰的に算出します。

シグネチャまたは形状

static double get_indexpath_pages(Path *bitmapqual)

入力と出力

  • **入力**: Path *bitmapqual - 評価対象のパス構造体。BitmapAndPath、BitmapOrPath、または IndexPath のいずれかである必要があります。
  • **出力**: double - 該当するインデックスが占有する総ページ数。

前提条件と不変条件

  • 入力される Path は、BitmapAndPath、BitmapOrPath、または IndexPath のいずれかのノードタイプである必要があります。
  • IndexPath の場合、indexinfo フィールドが有効であり、その中の pages が参照可能である必要があります。

アルゴリズムまたは振る舞い

1. **再帰的走査**:

  • BitmapAndPath または BitmapOrPath の場合、保持している bitmapquals リストを走査し、各要素に対して get_indexpath_pages を再帰的に呼び出してページ数を加算します。

    • 2. **終端条件**:

  • IndexPath に到達した場合、そのインデックスが持つ pages フィールドの値を返します。

    • 3. **エラーハンドリング**:

  • 上記以外のノードタイプが渡された場合、elog(ERROR) を発行して処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/costsize.c 内の関数から呼び出されています。
  • **呼び出し先**:
  • 再帰的に自身（get_indexpath_pages）を呼び出します。

関連データ構造

  • Path: 汎用的なパス構造体。
  • BitmapAndPath, BitmapOrPath: ビットマップインデックスの論理演算を表現する構造体。
  • IndexPath: 単一のインデックススキャンを表現する構造体。

失敗モードと境界条件

  • **未対応のノードタイプ**: BitmapAndPath, BitmapOrPath, IndexPath 以外のノードが渡された場合、エラーが発生します。
  • **再帰の深さ**: 入力されるパスツリーの深さに依存しますが、通常はクエリプランの構造に制限されます。

保守担当者向けメモ

  • 本関数はビットマップインデックスのコスト計算において、対象となるインデックスの物理的なサイズを把握するために使用されます。
  • 新しいインデックス関連のパスタイプが追加された場合は、本関数の条件分岐に追加が必要です。
