reset_variables

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/variable.c:377

目的

SQL文の処理に関連する入力引数リスト（argsinsert）および出力引数リスト（argsresult）を解放し、初期状態にリセットします。

シグネチャまたは形状

void reset_variables(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルな引数リストが解放・初期化される）

前提条件と不変条件

  • argsinsert および argsresult が指すリスト構造が有効であること。
  • 呼び出し後、argsinsert および argsresult は NULL に設定される。

アルゴリズムまたは振る舞い

1. argsinsert リストを先頭から順に走査し、各要素を free() で解放する。 2. argsinsert を NULL に設定する。 3. argsresult リストを先頭から順に走査し、各要素を free() で解放する。 4. argsresult を NULL に設定する。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは特定の呼び出し箇所は特定できません。
  • 呼び出し先: free()（標準ライブラリ関数）

関連データ構造

  • struct arguments: SQL文の引数情報を保持するリンクリスト構造体。
  • argsinsert: 入力引数を保持するグローバルなリンクリストのヘッド。
  • argsresult: 出力引数を保持するグローバルなリンクリストのヘッド。

失敗モードと境界条件

  • リストが既に空（NULL）の場合、ループは実行されず安全に終了します。

保守担当者向けメモ

  • 本関数はメモリリークを防ぐためのクリーンアップ処理です。新しい引数リストを追加する際は、必ず本関数によるリセットが適切に行われるライフサイクルであることを確認してください。
