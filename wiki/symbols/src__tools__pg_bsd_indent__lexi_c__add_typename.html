add_typename

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/lexi.c:686

目的

動的配列 typenames に新しい型名文字列を追加し、型名リストをアルファベット順に維持する。

シグネチャまたは形状

void add_typename(const char *key)

入力と出力

  • 入力: key (追加する型名文字列)
  • 出力: なし（副作用として typenames 配列を更新）

前提条件と不変条件

  • typenames 配列は常にアルファベット順にソートされている必要がある。
  • typename_count は現在の配列容量を保持し、typename_top は末尾のインデックスを示す。

アルゴリズムまたは振る舞い

1. **容量確保**: typename_top + 1 が typename_count に達した場合、realloc で容量を倍増させる。メモリ確保失敗時は err(1, NULL) で終了する。 2. **挿入位置の特定**:

  • リストが空、または key が末尾要素より大きい場合は末尾に追加する。
  • それ以外の場合は、リスト内を走査して挿入位置 p を特定する。

    • 3. **重複排除**: key が既にリスト内に存在する場合は何もせず終了する。 4. **挿入**: memmove を使用して既存の要素をシフトし、strdup で複製した key を挿入する。 5. **エラー処理**: strdup の結果が NULL の場合は err(1, NULL) で終了する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/tools/pg_bsd_indent/args.c:296
  • src/tools/pg_bsd_indent/args.c:347
  • 呼び出し先:
  • realloc, strdup, strcmp, memmove, err (標準ライブラリ関数)

関連データ構造

  • typenames: 型名を保持する動的配列。
  • typename_count: typenames の現在の最大容量。
  • typename_top: typenames 内の最後の有効な要素のインデックス。

失敗モードと境界条件

  • メモリ不足: realloc または strdup が失敗した場合、err を呼び出しプログラムを終了する。
  • 重複登録: 既に存在する型名が渡された場合、何も変更せずに処理を終了する。

保守担当者向けメモ

  • typenames 配列はソート済みであることが前提であるため、挿入ロジックを変更する際は順序が維持されることを確認すること。
  • typename_count の倍増ロジックは typenames の初期化状態に依存するため、初期化関数との整合性に注意すること。
