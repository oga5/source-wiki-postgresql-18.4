entryIsMoveRight

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginentrypage.c:242

目的

GINエントリーツリーの探索において、現在のページよりも右側のページに目的のキーが存在する可能性があるかを判定します。

シグネチャまたは形状

static bool entryIsMoveRight(GinBtree btree, Page page)

入力と出力

  • **入力**:
  • btree: GINインデックスのB-tree操作状態を保持する構造体。
  • page: 判定対象のページ。
  • **出力**:
  • bool: 右側のページへ移動すべき場合は true、そうでない場合は false を返します。

前提条件と不変条件

  • btree および page は有効なポインタである必要があります。
  • ページが右端（RightMost）である場合、移動の必要はないとみなされます。

アルゴリズムまたは振る舞い

1. GinPageRightMost(page) を確認し、右端ページであれば false を返します。 2. getRightMostTuple(page) を呼び出し、ページ内の右端タプルを取得します。 3. 取得したタプルから属性番号、キー値、カテゴリを抽出します。 4. ginCompareAttEntries を使用し、検索対象のキー（btree->entryKey 等）とページ内の右端キーを比較します。 5. 検索対象キーがページ内の右端キーよりも大きい場合、より右側のページにキーが存在すると判断し true を返します。それ以外は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginentrypage.c 内の btree->isMoveRight フィールドに本関数が代入されています（src/backend/access/gin/ginentrypage.c:759）。
  • **呼び出し先**:
  • getRightMostTuple
  • gintuple_get_attrnum
  • gintuple_get_key
  • ginCompareAttEntries

関連データ構造

  • GinBtree: インデックス操作のコンテキストを保持。
  • Page: GINインデックスのページ構造。
  • IndexTuple: ページ内に格納されたキー情報。

失敗モードと境界条件

  • ページが空の場合、getRightMostTuple の挙動に依存します。
  • 比較対象のキーがNULLカテゴリを持つ場合、ginCompareAttEntries の仕様に従って比較が行われます。

保守担当者向けメモ

  • 本関数は GinBtree のコールバックとして使用されるため、シグネチャを変更する際は ginPrepareEntryScan 等の初期化箇所との整合性に注意してください。
  • エントリーツリーは静的な性質を持つため、この判定ロジックはツリーの構造維持において重要な役割を果たします。
