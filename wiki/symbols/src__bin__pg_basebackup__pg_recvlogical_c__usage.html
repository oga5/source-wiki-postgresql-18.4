usage

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_recvlogical.c:80

目的

pg_recvlogical コマンドの利用方法、サポートされているオプション、および接続設定に関するヘルプメッセージを標準出力に表示する。

シグネチャまたは形状

static void usage(void);

入力と出力

  • 入力: なし（グローバル変数 progname, fsync_interval, plugin, standby_message_timeout を参照）。
  • 出力: なし（標準出力へヘルプテキストを出力）。

前提条件と不変条件

  • progname が適切に初期化されていること。
  • fsync_interval および standby_message_timeout が数値として有効であること。

アルゴリズムまたは振る舞い

1. プログラム名 (progname) を含む概要を表示する。 2. 実行可能なアクション（--create-slot, --drop-slot, --start）を列挙する。 3. 利用可能なオプション（ファイル出力、LSN 指定、タイムアウト設定等）をカテゴリ別に整形して出力する。 4. 接続オプション（ホスト、ポート、ユーザー名等）を表示する。 5. バグ報告先およびプロジェクトのホームページ URL を表示して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内には直接の呼び出し元に関する記述はありません。
  • 呼び出し先: printf (標準ライブラリ) および _() (国際化対応マクロ)。

関連データ構造

  • なし。

失敗モードと境界条件

  • 標準出力への書き込みが失敗した場合の挙動は printf の仕様に依存する。

保守担当者向けメモ

  • 新しいコマンドラインオプションを追加した際は、必ず本関数にも対応する説明文を追加すること。
  • fsync_interval や standby_message_timeout のデフォルト値は、グローバル変数の値がそのまま表示されるため、定数変更時には自動的に追従する。
