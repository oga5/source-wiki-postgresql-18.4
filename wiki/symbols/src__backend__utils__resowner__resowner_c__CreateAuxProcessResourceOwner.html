CreateAuxProcessResourceOwner

定義

  • 種別: function
  • 定義箇所: src/backend/utils/resowner/resowner.c:998

目的

補助プロセス（Auxiliary Process）専用のリソースオーナーを初期化し、プロセス終了時にリソースが確実に解放されるよう登録を行う。

シグネチャまたは形状

void CreateAuxProcessResourceOwner(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 AuxProcessResourceOwner および CurrentResourceOwner を設定）

前提条件と不変条件

  • AuxProcessResourceOwner が NULL であること。
  • CurrentResourceOwner が NULL であること。

アルゴリズムまたは振る舞い

1. ResourceOwnerCreate を呼び出し、名前 "AuxiliaryProcess" でリソースオーナーを作成し、AuxProcessResourceOwner に代入する。 2. CurrentResourceOwner を作成した AuxProcessResourceOwner に設定する。 3. on_shmem_exit を使用して、プロセス終了時にリソースを解放するためのコールバック関数 ReleaseAuxProcessResourcesCallback を登録する。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • src/backend/postmaster/auxprocess.c:76
  • src/backend/replication/walsender.c:305
  • src/backend/utils/init/postinit.c:785
  • 呼び出し先:
  • ResourceOwnerCreate
  • on_shmem_exit

関連データ構造

  • ResourceOwnerData（ResourceOwnerCreate により生成される）

失敗モードと境界条件

  • AuxProcessResourceOwner または CurrentResourceOwner が既に初期化されている場合、Assert により停止する。

保守担当者向けメモ

  • 本関数は補助プロセスの初期化フェーズで一度だけ呼び出されることを想定している。
  • 登録される ReleaseAuxProcessResourcesCallback は、共有メモリの終了処理（ShutdownXLOG など）の後に実行される必要がある。
