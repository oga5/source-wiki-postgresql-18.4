目的

ECPG（Embedded SQL for C）におけるインジケータ変数の挙動を検証する回帰テスト用プログラム。SQLのNULL値や値の切り捨てがC言語側でどのように扱われるかを実証する。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし
  • 出力: 標準出力にテスト結果（intvar および nullind の値）を出力。標準エラー出力にECPGのデバッグログを出力。終了コードとして 0 を返す。

前提条件と不変条件

  • データベース ecpg1_regression への接続が可能であること。
  • テスト実行中にテーブル indicator_test を作成・削除する権限があること。

アルゴリズムまたは振る舞い

1. **接続と初期化**: ECPGconnect でデータベースに接続し、コミットモードをオフに設定する。 2. **テーブル作成**: indicator_test テーブルを作成し、トランザクションをコミットする。 3. **データ挿入**:

  • 通常の値を挿入。
  • インジケータ変数 nullind を -1 に設定し、NULL値を挿入。
  • nullind を 0 に戻し、値を挿入。

    • 4. **データ取得と検証**: SELECT 文でデータを取得し、インジケータ変数の状態を printf で出力して検証する。 5. **更新テスト**: UPDATE 文でインジケータ変数を用いてNULL値を設定し、その結果を再取得して検証する。 6. **終了処理**: テーブルを削除し、接続を解除する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGsetcommit, ECPGdo, ECPGtrans, ECPGdisconnect, printf

関連データ構造

  • intvar: ホスト変数（整数値）。
  • nullind: インジケータ変数（-1はNULLを意味する）。

失敗モードと境界条件

  • データベース接続失敗時やSQL実行エラー時に、ECPGライブラリのデフォルトの挙動（デバッグログ出力等）が発生する。
  • インジケータ変数が正しく設定されていない場合、NULL値の挿入や取得が期待通りに行われない可能性がある。

保守担当者向けメモ

  • 本関数は ecpg プリプロセッサが生成したCコードの期待値である。直接編集せず、テストの比較対象として扱うこと。
  • ECPGdebug(1, stderr) が有効なため、実行時の詳細な挙動は標準エラー出力から確認可能。
