string_hash

定義

  • 種別: function
  • 定義箇所: src/common/hashfn.c:659

目的

NUL終端文字列をハッシュ化するためのラッパー関数。ハッシュテーブルへの格納時に切り詰められる長さを考慮し、文字列のハッシュ値を計算する。

シグネチャまたは形状

uint32 string_hash(const void *key, Size keysize)

入力と出力

  • **入力**:
  • key: ハッシュ対象の文字列（const void *）。
  • keysize: ハッシュテーブル内で許容される最大長（Size）。
  • **出力**:
  • uint32: 計算されたハッシュ値。

前提条件と不変条件

  • key は NUL 終端文字列であること。
  • ハッシュ計算対象の長さは、strlen(key) と keysize - 1 の小さい方に制限される。

アルゴリズムまたは振る舞い

1. strlen を用いて入力文字列の長さを取得する。 2. Min(s_len, keysize - 1) により、ハッシュテーブル格納時に切り詰められる長さを算出する。 3. 算出した長さに基づき、hash_bytes を呼び出してハッシュ値を生成する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/lib/dshash.c: ハッシュ関数として本関数を転送利用。
  • src/backend/utils/hash/dynahash.c: ハッシュテーブルのハッシュメソッドとして設定（非推奨の利用例を含む）。
  • **呼び出し先**:
  • hash_bytes (同ファイル内): 実際のハッシュ計算を実行。
  • strlen (標準ライブラリ): 文字列長取得。

関連データ構造

  • dynahash: PostgreSQL のハッシュテーブル実装。本関数はキー計算用関数として利用される。

失敗モードと境界条件

  • keysize が 0 の場合、keysize - 1 の計算でアンダーフローが発生する可能性があるため注意が必要。
  • hash_bytes は elog(ERROR) を投げない設計であるため、本関数も同様に安全なパスで動作する。

保守担当者向けメモ

  • 本関数は dynahash において明示的に指定することが非推奨となっている。これは、将来的な実装変更や trampoline 経由の呼び出しによるオーバーヘッドを避けるためである。
  • 文字列の切り詰めロジックは、ハッシュテーブルのキー格納仕様と一致させる必要がある。変更時は dynahash.c 側の格納ロジックと整合性を確認すること。
