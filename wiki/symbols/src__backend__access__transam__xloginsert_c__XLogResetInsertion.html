XLogResetInsertion

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xloginsert.c:221

目的

WALレコード構築用の作業領域をリセットし、次のレコード構築に備えるための内部関数です。

シグネチャまたは形状

void XLogResetInsertion(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルな作業領域の状態を初期化する）

前提条件と不変条件

  • 本関数は、WALレコードの構築プロセスが終了した後に呼び出されることを想定しています。
  • 実行後、begininsert_called フラグが false に設定され、レコード構築の開始状態が解除されます。

アルゴリズムまたは振る舞い

1. registered_buffers 配列内の全エントリについて、in_use フラグを false に設定します。 2. レコード構築に関連する以下のグローバル変数を初期化します。

  • num_rdatas = 0
  • max_registered_block_id = 0
  • mainrdata_len = 0
  • mainrdata_last を mainrdata_head を指すようにリセット
  • curinsert_flags = 0
  • begininsert_called = false

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/xact.c: トランザクション終了処理等で呼び出し。
  • src/backend/access/transam/xloginsert.c: XLogInsert 後のクリーンアップやエラー発生時のリセット処理で呼び出し。
  • 呼び出し先:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • registered_buffers: 登録されたブロックのメタデータを保持する配列。
  • XLogRecData: WALレコードのデータチャンクを連結する構造体。
  • mainrdata_head: メインデータ領域の先頭構造体。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。本関数は常に成功し、作業領域をクリーンな状態へ戻します。
