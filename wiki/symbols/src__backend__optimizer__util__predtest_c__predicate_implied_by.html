predicate_implied_by

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/predtest.c:151

目的

与えられた条件リスト（clause_list）が、対象となる述語リスト（predicate_list）を論理的に含意するか（すなわち、clause_list が真ならば predicate_list も必ず真となるか）を判定します。

シグネチャまたは形状

bool predicate_implied_by(List *predicate_list, List *clause_list, bool weak)

入力と出力

  • **入力**:
  • predicate_list: 含意されるべき述語のリスト。
  • clause_list: 前提となる条件のリスト。
  • weak: 判定の厳密さを制御するフラグ。
  • **出力**:
  • bool: 含意関係が成立する場合は true、そうでない場合は false。

前提条件と不変条件

  • predicate_list の各要素は IMMUTABLE である必要があります。
  • predicate_list が空（NIL）の場合は、空虚な真として true を返します。
  • clause_list が空（NIL）の場合は、制約がないため含意は成立せず false を返します。

アルゴリズムまたは振る舞い

1. 引数が空リストである場合の早期リターン処理を行います。 2. リストの長さが 1 の場合、リストの先頭要素を直接抽出して AND 再帰のオーバーヘッドを回避します。 3. 抽出したノードを predicate_implied_by_recurse に渡し、再帰的な論理包含判定を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c
  • src/backend/optimizer/path/indxpath.c
  • src/backend/optimizer/plan/createplan.c
  • **呼び出し先**:
  • predicate_implied_by_recurse (直接の再帰呼び出し先)

関連データ構造

  • List (PostgreSQL のリスト構造)
  • Node (式ツリーのノード)

失敗モードと境界条件

  • **スタックオーバーフロー**: 非常に深い AND/OR 構造を持つ式ツリーを渡した場合、再帰処理によりスタックを消費します。
  • **不変性の欠如**: predicate_list に IMMUTABLE でない関数や演算子が含まれる場合、プラン作成時と実行時で結果が乖離する可能性があります。

保守担当者向けメモ

  • 本関数はトップレベルのラッパーであり、実際の論理判定は predicate_implied_by_recurse に委譲されます。
  • 呼び出し元（特に indxpath.c や createplan.c）では、本関数が第一引数（predicate_list）の不変性を前提としている点に注意して使用されています。
