pg_tablespace_size_name

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/dbsize.c:303

目的

指定されたテーブルスペース名の物理ディスクサイズ（バイト単位）を計算し、SQLインターフェースへ返却します。

シグネチャまたは形状

Datum pg_tablespace_size_name(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_NAME(0) を介してテーブルスペース名（Name型）を受け取ります。
  • **出力**: テーブルスペースの合計サイズを int64 型で返します。対象が存在しない場合や計算不能な場合は NULL を返します。

前提条件と不変条件

  • 入力されたテーブルスペース名から get_tablespace_oid を用いて OID を取得します。
  • 内部で calculate_tablespace_size を呼び出し、ファイルシステムレベルでの集計を行います。

アルゴリズムまたは振る舞い

1. 引数からテーブルスペース名を取得し、対応する OID を検索します。 2. calculate_tablespace_size(tblspcOid) を呼び出し、対象テーブルスペースの全物理サイズを集計します。 3. 集計結果が負の値（エラー等）であれば PG_RETURN_NULL() を実行します。 4. 正常な場合は PG_RETURN_INT64(size) によりバイト数を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースから直接呼び出されます（直接的なコード上の呼び出し元はソース内に明示されていません）。
  • **呼び出し先**:
  • get_tablespace_oid
  • calculate_tablespace_size

関連データ構造

  • Name: テーブルスペース名を保持する型。
  • Oid: テーブルスペースを一意に識別する識別子。
  • int64: サイズ計算結果を保持する型。

失敗モードと境界条件

  • 指定した名前のテーブルスペースが存在しない場合、get_tablespace_oid がエラーを発生させる可能性があります。
  • calculate_tablespace_size が負の値を返した場合、本関数は NULL を返します。

保守担当者向けメモ

  • 本関数は pg_tablespace_size_oid と同様のロジックを共有していますが、入力が名前である点が異なります。
  • ファイル走査を伴うため、実行には適切な権限（pg_read_all_stats ロール等）が必要です。
