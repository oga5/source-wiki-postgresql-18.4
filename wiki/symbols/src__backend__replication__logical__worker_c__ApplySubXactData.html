ApplySubXactData

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/worker.c:343

目的

論理レプリケーションにおけるサブトランザクションの情報を管理し、ストリーミングされた変更データ内でのオフセットを追跡するために使用されます。

シグネチャまたは形状

typedef struct ApplySubXactData
{
	uint32		nsubxacts;		/* サブトランザクション数 */
	uint32		nsubxacts_max;	/* サブトランザクション配列の現在の容量 */
	TransactionId subxact_last; /* 最後に処理されたサブトランザクションのXID */
	SubXactInfo *subxacts;		/* 変更ファイル内でのサブトランザクションオフセット */
} ApplySubXactData;

入力と出力

  • 入力: 本構造体は、論理レプリケーションワーカーの実行状態の一部として保持されます。
  • 出力: 構造体内の各フィールドは、サブトランザクションの追跡および変更ファイルへのアクセスインデックスとして機能します。

前提条件と不変条件

  • subxacts フィールドは、動的に割り当てられた SubXactInfo 配列を指す必要があります。
  • nsubxacts は nsubxacts_max を超えてはなりません。

アルゴリズムまたは振る舞い

  • 本構造体は、ストリーミングトランザクションの処理中にサブトランザクションの階層と位置を記録します。
  • subxact_last を更新することで、直近のサブトランザクション状態を保持します。
  • 変更ファイル（BufFile）からデータを読み込む際、subxacts 配列を使用して特定のサブトランザクションに関連する変更箇所を特定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/logical/worker.c 内の静的変数 subxact_data として初期化および参照されています。
  • 呼び出し先: 具体的な関数呼び出しの詳細は提供されていませんが、ストリーミングトランザクションのシリアライズおよび適用処理の過程で参照されることが示唆されています。

関連データ構造

  • SubXactInfo: サブトランザクションのオフセット情報を保持する構造体。
  • TransactionId: PostgreSQLのトランザクション識別子。

失敗モードと境界条件

  • nsubxacts_max に達した場合、配列の再割り当て（リサイズ）が必要です。
  • subxacts が NULL の場合、サブトランザクション情報が未初期化であることを示します。

保守担当者向けメモ

  • 本構造体は worker.c 内の静的変数 subxact_data を通じて管理されています。
  • ストリーミングトランザクションの複雑な状態遷移に関与しているため、変更を加える際は BufFile を介した読み書きの整合性に注意してください。
