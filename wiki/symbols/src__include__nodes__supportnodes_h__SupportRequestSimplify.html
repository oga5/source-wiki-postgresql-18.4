SupportRequestSimplify

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/supportnodes.h:64

目的

プランナーによる定数畳み込み等の最適化フェーズにおいて、特定の関数呼び出しをより単純な形式に書き換えるためのリクエストを表現する構造体です。

シグネチャまたは形状

typedef struct SupportRequestSimplify
{
	NodeTag		type;
	struct PlannerInfo *root;
	FuncExpr   *fcall;
} SupportRequestSimplify;

入力と出力

  • **入力**:
  • root: プランナーのインフラストラクチャ情報（PlannerInfo）。
  • fcall: 簡略化対象となる関数呼び出しノード（FuncExpr）。
  • **出力**:
  • サポート関数が簡略化可能と判断した場合、書き換え後のノードが返されます。処理できない場合は NULL が返されます。

前提条件と不変条件

  • SupportRequestSimplify 構造体は一時的な利用を想定しており、永続的なメモリ確保は行われません。
  • サポート関数がリクエストを処理できない場合、エラーではなく NULL を返すことがプロトコルとして規定されています。

アルゴリズムまたは振る舞い

1. プランナーが関数呼び出しの最適化が必要と判断した際、本構造体を生成します。 2. 対象関数に紐付けられたサポート関数が、本構造体を引数として呼び出されます。 3. サポート関数は fcall を評価し、簡略化が可能であれば結果を返し、不可能であれば NULL を返します。 4. 呼び出し元は返された結果に基づき、プランの書き換えを行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/clauses.c にて、サポート関数を呼び出すためのリクエスト生成が行われています。
  • **呼び出し先**: src/backend/utils/adt/ 配下の各ファイル（date.c, numeric.c, rangetypes.c, timestamp.c 等）にて、IsA(rawreq, SupportRequestSimplify) による型チェックを経て、各データ型のサポート関数内で処理されます。

関連データ構造

  • PlannerInfo: プランナーのインフラストラクチャ情報。
  • FuncExpr: 簡略化対象となる関数呼び出しノード。

失敗モードと境界条件

  • サポート関数が簡略化をサポートしていない場合、NULL を返すことで後方互換性を維持します。

保守担当者向けメモ

  • 本構造体は「プランナーサポート関数」の一部ですが、実行時（Executor）にも呼び出される可能性があるため、プランナー専用のロジックと限定しないよう注意してください。
