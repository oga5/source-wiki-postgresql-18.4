PGLZ_HistEntry

定義

  • 種別: struct
  • 定義箇所: src/common/pg_lzcompress.c:210

目的

LZ77 圧縮アルゴリズムにおいて、入力データ内の重複パターンを高速に検索するためのハッシュテーブル用リンクリスト要素として機能します。

シグネチャまたは形状

typedef struct PGLZ_HistEntry
{
    struct PGLZ_HistEntry *next; /* ハッシュキーのリスト用リンク */
    struct PGLZ_HistEntry *prev;
    int            hindex;       /* 現在のハッシュキー */
    const char    *pos;          /* 入力データ内の位置 */
} PGLZ_HistEntry;

入力と出力

本構造体はデータ構造であり、関数ではありません。圧縮処理中に hist_entries 配列の一部として管理され、入力データの各位置（pos）をハッシュテーブルに登録するために使用されます。

前提条件と不変条件

  • pglz_compress 実行時に、PGLZ_HISTORY_SIZE + 1 個の要素を持つ配列 hist_entries が確保されます。
  • 各エントリは、特定のハッシュキーに対応するリンクリストの一部として連結されます。

アルゴリズムまたは振る舞い

  • 圧縮プロセスにおいて、入力データの各バイト位置をハッシュ化し、その結果を hindex に保持します。
  • next および prev ポインタを用いて、同じハッシュ値を持つ過去のデータ位置をリンクリスト形式で追跡します。これにより、圧縮時に過去のデータとの一致（マッチ）を効率的に検索します。

呼び出し元と呼び出し先

  • 呼び出し元: pglz_compress 関数内で、履歴管理のために本構造体の配列が初期化・操作されます。
  • 呼び出し先: 本構造体自体はデータ構造であり、直接関数を呼び出すことはありません。

関連データ構造

  • PGLZ_Strategy: 圧縮戦略を定義する構造体。
  • hist_entries: PGLZ_HistEntry 型の静的配列（src/common/pg_lzcompress.c:256）。

失敗モードと境界条件

  • 履歴テーブルのサイズ（PGLZ_HISTORY_SIZE）を超えるデータ検索は行われません。
  • マクロ pglz_hist_add 等を使用する際、引数の評価順序や副作用に注意が必要です（複数回評価される可能性があるため）。

保守担当者向けメモ

  • 本構造体は pglz_compress の内部的なハッシュ検索専用です。
  • 伸長処理（pglz_decompress）では使用されません。
