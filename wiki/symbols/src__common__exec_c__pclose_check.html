pclose_check

定義

  • 種別: function
  • 定義箇所: src/common/exec.c:390

目的

パイプ経由で実行したプロセスの終了状態を監視し、エラー発生時に標準的なログ出力を行うためのラッパー関数です。

シグネチャまたは形状

int pclose_check(FILE *stream);

入力と出力

  • **入力**: stream (パイプを開いた FILE * ポインタ)
  • **出力**: int (プロセスの終了ステータス。成功時は 0、失敗時は pclose の戻り値)

前提条件と不変条件

  • stream は popen 等で開かれた有効なパイプストリームである必要があります。
  • 呼び出し後、stream はクローズされます。

アルゴリズムまたは振る舞い

1. pclose(stream) を呼び出し、プロセスの終了ステータスを取得します。 2. 終了ステータスが 0 の場合、正常終了として 0 を返します。 3. 終了ステータスが -1 の場合、pclose 自体の失敗として log_error を用いてシステムエラーを記録します。 4. それ以外（ゼロ以外）の場合、wait_result_to_str を使用して終了理由を文字列化し、log_error で記録した後にその文字列を pfree で解放します。 5. 最終的に取得した終了ステータスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/initdb/initdb.c
  • src/common/exec.c
  • **呼び出し先**:
  • pclose(3)
  • log_error
  • wait_result_to_str
  • pfree

関連データ構造

  • FILE (標準ライブラリのストリーム)

失敗モードと境界条件

  • pclose が -1 を返した場合、システムエラーとして記録されます。
  • プロセスが異常終了した場合、wait_result_to_str を介して詳細な終了理由がログ出力されます。

保守担当者向けメモ

  • 本関数はエラー発生時に log_error を通じてメッセージを出力するため、呼び出し側で再度エラーメッセージを表示する必要はありません。
  • wait_result_to_str が返す文字列は動的に割り当てられるため、pfree による解放が必須です。
