scram_ClientKey

定義

  • 種別: function
  • 定義箇所: src/common/scram-common.c:141

目的

SCRAM認証プロセスにおいて、ソルト付きパスワード（SaltedPassword）から ClientKey を導出する。

シグネチャまたは形状

int scram_ClientKey(const uint8 *salted_password,
                    pg_cryptohash_type hash_type, int key_length,
                    uint8 *result, const char **errstr);

入力と出力

  • **入力**:
  • salted_password: PBKDF2等で生成されたソルト付きパスワード。
  • hash_type: 使用するハッシュアルゴリズム（pg_cryptohash_type）。
  • key_length: キーの長さ。
  • **出力**:
  • result: 導出された ClientKey を格納するバッファ。
  • errstr: 失敗時にエラーメッセージが設定される。
  • 戻り値: 成功時は 0、失敗時は -1。

前提条件と不変条件

  • salted_password は、事前にSASLprepによる正規化が完了していること。
  • result バッファは、指定された key_length を保持するのに十分なサイズであること。

アルゴリズムまたは振る舞い

1. 指定された hash_type でHMACコンテキストを生成する。 2. salted_password をキーとしてHMACを初期化する。 3. 文字列 "Client Key" を入力としてHMAC計算を実行する。 4. 計算結果を result に格納し、コンテキストを解放して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq/fe-auth-scram.c
  • src/backend/libpq/auth-scram.c（state->ClientKey の生成時）
  • **呼び出し先**:
  • pg_hmac_create, pg_hmac_init, pg_hmac_update, pg_hmac_final, pg_hmac_free, pg_hmac_error

関連データ構造

  • pg_hmac_ctx: HMAC計算用コンテキスト。
  • pg_cryptohash_type: ハッシュアルゴリズムの指定用列挙型。

失敗モードと境界条件

  • pg_hmac_create がメモリ不足等で失敗した場合、-1 を返し errstr にエラー情報を設定する。
  • HMACの初期化、更新、または最終化のいずれかのステップでエラーが発生した場合、コンテキストを解放し -1 を返す。

保守担当者向けメモ

  • 本関数はフロントエンドとバックエンドの両方から利用される共有コードである。
  • 戻り値が -1 の場合、呼び出し元は errstr を参照してエラーハンドリングを行う必要がある。
