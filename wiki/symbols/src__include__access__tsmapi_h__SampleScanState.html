SampleScanState

定義

目的

TABLESAMPLE 句を用いたテーブルスキャン実行時において、サンプリングアルゴリズム固有の状態情報を保持するための構造体です。

シグネチャまたは形状

typedef struct SampleScanState SampleScanState; （詳細は nodes/execnodes.h に定義）

入力と出力

本シンボル自体は型定義ですが、これを利用するコールバック関数（InitSampleScan_function 等）は、SampleScanState へのポインタを入力として受け取り、状態の初期化や更新を行います。

前提条件と不変条件

  • TsmRoutine を介して呼び出される各サンプリングメソッドの関数において、SampleScanState ポインタが有効であることが前提です。
  • エグゼキュータによるスキャン開始前に、適切な初期化関数（InitSampleScan）が呼び出されている必要があります。

アルゴリズムまたは振る舞い

  • TABLESAMPLE の実行中、サンプリングメソッド（BERNOULLI や SYSTEM など）が、選択したブロックやタプルの追跡、乱数生成器の状態保持などの目的で使用します。
  • エグゼキュータは、この構造体を通じてサンプリングの進捗を管理します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • contrib/tsm_system_rows/tsm_system_rows.c
  • contrib/tsm_system_time/tsm_system_time.c
  • 上記の各ファイル内で、InitSampleScan、BeginSampleScan、NextSampleBlock、NextSampleTuple の引数として参照されています。
  • **呼び出し先**:
  • 本シンボルはデータ構造であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • TsmRoutine: サンプリングメソッドのインターフェースを定義する構造体。
  • nodes/execnodes.h: SampleScanState の実体が定義されているヘッダー。
  • nodes/pathnodes.h: プランナ関連の依存関係。

失敗モードと境界条件

  • 構造体の初期化が不完全な状態でサンプリング関数が呼び出された場合、未定義の動作を引き起こす可能性があります。
  • NextSampleBlock が NULL を返す場合、システムはサンプリングを終了またはデフォルトのシーケンシャルスキャンへ移行します。

保守担当者向けメモ

  • SampleScanState のフィールド構成を変更する場合は、nodes/execnodes.h の定義を確認してください。
  • 新しいサンプリングメソッドを追加する際は、既存の contrib/ 配下の実装を参考に、SampleScanState を適切に初期化・利用するようにしてください。
