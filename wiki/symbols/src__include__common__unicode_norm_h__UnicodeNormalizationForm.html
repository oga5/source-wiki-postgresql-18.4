UnicodeNormalizationForm

定義

  • 種別: enum
  • 定義箇所: src/include/common/unicode_norm.h:19

目的

Unicode 標準（UAX #15）に基づく文字列の正規化形式を指定するために使用されます。

シグネチャまたは形状

typedef enum
{
	UNICODE_NFC = 0,
	UNICODE_NFD = 1,
	UNICODE_NFKC = 2,
	UNICODE_NFKD = 3,
} UnicodeNormalizationForm;

入力と出力

  • 入力: なし（列挙型の定義）
  • 出力: UNICODE_NFC, UNICODE_NFD, UNICODE_NFKC, UNICODE_NFKD のいずれかの値。

前提条件と不変条件

  • 各値は 0 から 3 までの整数値に対応します。
  • 本列挙型は、正規化処理を行う関数群の引数として使用されます。

アルゴリズムまたは振る舞い

  • UNICODE_NFC: 正規化形式 C（正準等価による合成）を指定します。
  • UNICODE_NFD: 正規化形式 D（正準等価による分解）を指定します。
  • UNICODE_NFKC: 正規化形式 KC（互換等価による合成）を指定します。
  • UNICODE_NFKD: 正規化形式 KD（互換等価による分解）を指定します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/utils/adt/varlena.c
  • src/common/unicode_norm.c（unicode_normalize, qc_is_allowed, unicode_is_normalized_quickcheck）
  • 呼び出し先: なし（本シンボルは定義体であり、関数ではないため）。

関連データ構造

  • UnicodeNormalizationQC: 正規化状態（YES, NO, MAYBE）を保持する列挙型。

失敗モードと境界条件

  • 範囲外の整数値をキャストして代入した場合の動作は未定義です。

保守担当者向けメモ

  • 本列挙型はフロントエンドとバックエンドの両方で利用されるため、変更時は両方の互換性に注意してください。
  • 実装の詳細については src/common/unicode_norm.c および UAX #15 の仕様を参照してください。
