SharedInvalBackendInit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/sinvaladt.c:271

目的

共有キャッシュ無効化（Shared Cache Invalidation）システムにおいて、現在のバックエンドプロセスを共有メモリ上の管理構造に登録し、初期化を行う。

シグネチャまたは形状

void SharedInvalBackendInit(bool sendOnly)

入力と出力

  • **入力**:
  • sendOnly (bool): true の場合、このバックエンドはメッセージの送信のみを行い、受信（キャッシュ更新）を行わないことを示す。
  • **出力**: なし。

前提条件と不変条件

  • MyProcNumber が有効な範囲（0 以上 NumProcStateSlots 未満）で設定されていること。
  • 呼び出し時に SInvalWriteLock を保持していないこと（内部で排他ロックを取得するため）。
  • 該当する ProcState スロットが現在未使用（procPid == 0）であること。

アルゴリズムまたは振る舞い

1. MyProcNumber を用いて共有メモリ上の ProcState スロットを特定する。 2. SInvalWriteLock を排他モードで取得する。 3. スロットが既に使用中であればエラーを投げる。 4. pgprocnos 配列に自身の MyProcNumber を追加する。 5. 共有メモリからローカルのトランザクションID状態を読み込む。 6. 自身の ProcState を初期化する（PIDの設定、nextMsgNum を現在の maxMsgNum に同期、各フラグを false に設定）。 7. SInvalWriteLock を解放する。 8. プロセス終了時に CleanupInvalidationState が自動実行されるよう on_shmem_exit に登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/standby.c: SharedInvalBackendInit(true);
  • src/backend/utils/init/postinit.c: SharedInvalBackendInit(false);
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease (SInvalWriteLock)
  • on_shmem_exit (CleanupInvalidationState)
  • elog (エラーハンドリング)

関連データ構造

  • SISeg: 共有メモリ上のメイン構造体。
  • ProcState: 各バックエンドの進捗状態を保持する構造体。
  • MyProcNumber: 現在のプロセスのインデックス。

失敗モードと境界条件

  • MyProcNumber が未設定の場合、ERROR を発生させる。
  • MyProcNumber が範囲外の場合、PANIC を発生させる。
  • スロットが既に他のプロセスによって使用されている場合、ERROR を発生させる。

保守担当者向けメモ

  • 本関数は SInvalWriteLock を排他モードで取得するため、頻繁な呼び出しはパフォーマンスに影響を与える可能性があるが、通常はプロセス初期化時に一度のみ実行される。
  • on_shmem_exit によるクリーンアップ登録は、プロセスが異常終了した場合でも共有メモリ上の状態を整合させるために不可欠である。
