dsm_unpin_mapping

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/dsm.c:933

目的

指定された動的共有メモリ（DSM）セグメントの管理を、現在のリソースオーナー（CurrentResourceOwner）に紐付けることで、トランザクション終了時の自動クリーンアップ対象として登録します。

シグネチャまたは形状

void dsm_unpin_mapping(dsm_segment *seg)

入力と出力

  • **入力**: seg - 管理対象となる dsm_segment 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • seg->resowner が NULL であること（既にリソースオーナーに紐付いていないこと）。
  • CurrentResourceOwner が有効であること。

アルゴリズムまたは振る舞い

1. seg->resowner が NULL であることをアサートで確認します。 2. CurrentResourceOwner がリソースを保持できるよう、ResourceOwnerEnlarge を呼び出して容量を確保します。 3. seg->resowner に CurrentResourceOwner を代入します。 4. ResourceOwnerRememberDSM を呼び出し、現在のリソースオーナーの管理下にある DSM セグメントリストに当該セグメントを追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • **呼び出し先**: ResourceOwnerEnlarge および ResourceOwnerRememberDSM を呼び出します。

関連データ構造

  • dsm_segment: DSM セグメントを管理する構造体。
  • ResourceOwner: トランザクション等のリソースライフサイクルを管理する構造体。

失敗モードと境界条件

  • CurrentResourceOwner が NULL の場合、ResourceOwnerEnlarge 等の内部でエラーが発生する可能性があります。
  • seg が無効なポインタである場合、アサート失敗またはセグメンテーション違反が発生します。

保守担当者向けメモ

  • 関数名に "unpin" とありますが、実際にはリソースオーナーへの登録（紐付け）を行う関数です。
  • dsm_pin_segment 等の「ピン留め」機能とは異なり、本関数はリソースオーナーによる自動解放の管理下に置くための処理です。
