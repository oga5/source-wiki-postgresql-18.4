LWLockShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lwlock.c:431

目的

PostgreSQL の共有メモリ領域において、軽量ロック（LWLock）システム全体を保持するために必要な合計バイト数を算出します。

シグネチャまたは形状

Size LWLockShmemSize(void);

入力と出力

  • **入力**: なし（グローバルな設定変数 NumLWLocksForNamedTranches および NamedLWLockTrancheRequestArray を参照）。
  • **出力**: 必要な共有メモリサイズ（Size 型）。

前提条件と不変条件

  • 本関数は、共有メモリの初期化プロセスにおいて、必要なメモリ量を決定するために呼び出されます。
  • mul_size および add_size 関数を使用し、計算過程でのオーバーフローを防止しています。

アルゴリズムまたは振る舞い

以下の要素の合計サイズを算出します。

1. **LWLock 配列**: 固定数（NUM_FIXED_LWLOCKS）と名前付きトランチ（NumLWLocksForNamedTranches()）の合計数分の LWLockPadded 構造体領域。 2. **動的割り当てカウンタ**: 整数型サイズとアライメント用のパディング領域。 3. **名前付きトランチ構造体**: NamedLWLockTrancheRequests 分の NamedLWLockTranche 構造体領域。 4. **トランチ名**: 各トランチ名の文字列長（終端文字含む）の合計。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/ipci.c:130（共有メモリサイズ計算時）
  • src/backend/storage/lmgr/lwlock.c:466（CreateLWLocks 内）
  • **呼び出し先**:
  • NumLWLocksForNamedTranches()
  • mul_size(), add_size()
  • strlen()

関連データ構造

  • LWLockPadded: LWLock のパディング付き構造体。
  • NamedLWLockTranche: 名前付きトランチの定義構造体。
  • NamedLWLockTrancheRequestArray: 名前付きトランチ要求の配列。

失敗モードと境界条件

  • NamedLWLockTrancheRequests が 0 の場合、トランチ関連のメモリは加算されません。
  • メモリ計算中にオーバーフローが発生した場合、mul_size や add_size がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数で算出されるサイズは、CreateLWLocks で実際に共有メモリを確保する際の計算と完全に一致している必要があります。新しい LWLock 関連のデータ構造を共有メモリに追加する場合は、必ず本関数を更新してください。
