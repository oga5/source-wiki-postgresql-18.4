TidStoreDetach

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tidstore.c:268

目的

共有メモリ上に構築された TidStore 構造体に関連付けられたリソースを解放し、現在のバックエンドからのデタッチを行います。

シグネチャまたは形状

void TidStoreDetach(TidStore *ts)

入力と出力

  • **入力**: ts (TidStore *) - デタッチ対象の TidStore 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • ts は共有メモリ上に存在する TidStore である必要があります（内部で TidStoreIsShared(ts) によるアサーションチェックが行われます）。

アルゴリズムまたは振る舞い

1. TidStoreIsShared(ts) を検証し、共有メモリ用であることを確認します。 2. shared_ts_detach を呼び出し、内部のラディックツリー構造をデタッチします。 3. dsa_detach を呼び出し、DSA（Dynamic Shared Area）領域をデタッチします。 4. pfree(ts) を呼び出し、バックエンドローカルメモリ上の TidStore 制御構造体を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/vacuumparallel.c: dead_items の解放時に呼び出されます。
  • **呼び出し先**:
  • shared_ts_detach (内部関数)
  • dsa_detach
  • pfree

関連データ構造

  • TidStore: 本関数が管理・解放するメイン構造体。
  • DSA (Dynamic Shared Area): 共有メモリ領域の管理に使用されます。

失敗モードと境界条件

  • ts がローカルメモリ専用の TidStore である場合、Assert により異常終了します。
  • 共有メモリのデタッチ処理において、DSA 領域が既に無効化されている場合や、不正なポインタが渡された場合は未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は共有メモリ版の TidStore 専用です。ローカルメモリ版の TidStore を破棄する場合は、別の関数（TidStoreDestroy 等）を使用する必要があります。
  • 呼び出し側で、この関数を呼び出す前に適切な同期処理が行われていることを確認してください。
