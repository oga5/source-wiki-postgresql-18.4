entryIsEnoughSpace

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginentrypage.c:458

目的

GINインデックスのエントリーツリーページにおいて、新しいインデックスタプルを挿入するための十分な空き容量があるかを判定します。

シグネチャまたは形状

static bool entryIsEnoughSpace(GinBtree btree, Buffer buf, OffsetNumber off, GinBtreeEntryInsertData *insertData)

入力と出力

  • **入力**:
  • btree: GIN B-treeの制御構造。
  • buf: 対象となるページを含むバッファ。
  • off: 挿入位置を示すオフセット番号。
  • insertData: 挿入対象のデータ（entryを含む）および削除フラグ（isDelete）。
  • **出力**:
  • bool: 挿入可能であれば true、不足していれば false を返します。

前提条件と不変条件

  • insertData->entry が存在すること。
  • 対象ページがデータページではなく、エントリーツリーのページであること（!GinPageIsData(page)）。

アルゴリズムまたは振る舞い

1. 挿入対象のタプルサイズ（addedsz）を計算します。これにはタプル本体のサイズと ItemIdData のサイズが含まれます。 2. 削除操作を伴う場合（insertData->isDelete が真）、指定されたオフセット off にある既存タプルのサイズ（releasedsz）を計算し、解放される領域として加算します。 3. 現在のページの空き容量（PageGetFreeSpace）に releasedsz を加えた値が、addedsz 以上であるかを比較します。 4. 条件を満たせば true、そうでなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginentrypage.c 内の entryBeginPlaceToPage から呼び出されます。
  • **呼び出し先**:
  • BufferGetPage
  • PageGetItem
  • PageGetItemId
  • IndexTupleSize
  • PageGetFreeSpace

関連データ構造

  • GinBtree: GIN B-tree操作のコンテキスト。
  • GinBtreeEntryInsertData: 挿入対象のインデックスタプルおよび操作フラグを保持する構造体。
  • Buffer: ページを保持するバッファ。

失敗モードと境界条件

  • ページがデータページ（GinPageIsData）である場合、アサーションエラーが発生します。
  • 挿入対象のタプルがページサイズに対して極端に大きい場合、常に false を返します。

保守担当者向けメモ

  • 本関数は、エントリーツリーのページ分割が必要かどうかを判断するための重要な判定ロジックです。
  • 削除を伴う挿入（更新に近い操作）の場合、既存タプルの削除分を考慮した正確な空き容量計算を行う必要があります。
