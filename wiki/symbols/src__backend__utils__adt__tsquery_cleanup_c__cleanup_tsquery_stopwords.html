cleanup_tsquery_stopwords

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_cleanup.c:386

目的

入力された TSQuery からストップワード（QI_VALSTOP）を除去し、最適化された新しい TSQuery を生成する。

シグネチャまたは形状

TSQuery cleanup_tsquery_stopwords(TSQuery in, bool noisy)

入力と出力

  • **入力**:
  • in: 処理対象の TSQuery オブジェクト。
  • noisy: ストップワードのみで構成されるクエリの場合に通知（NOTICE）を出すかどうかを指定するフラグ。
  • **出力**:
  • ストップワード除去後の新しい TSQuery オブジェクト。クエリが空になった場合は size が 0 のオブジェクトを返す。

前提条件と不変条件

  • in->size == 0 の場合、処理を行わず入力された in をそのまま返す。
  • 内部的に maketree を使用してフラットな配列をツリー構造に変換し、clean_stopword_intree で再帰的に処理を行う。
  • ストップワード除去後、plaintree を用いてツリーを再びフラットな QueryItem 配列へ再構築する。

アルゴリズムまたは振る舞い

1. maketree で TSQuery をツリー構造に変換し、clean_stopword_intree を呼び出してストップワードを除去する。 2. ストップワード除去によりツリーが空になった場合、noisy が真であれば通知を発行し、サイズ 0 の TSQuery を生成して返す。 3. ツリーが残っている場合、calcstrlen で必要な文字列バッファサイズを算出し、plaintree でツリーをフラットな配列へ変換する。 4. 新しい TSQuery オブジェクトをメモリ確保し、QueryItem 配列をコピーする。 5. オペランド（検索語）を新しいメモリ領域へ再配置し、各 QueryOperand の distance を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery.c:936
  • **呼び出し先**:

関連データ構造

  • TSQuery: 全文検索クエリの内部表現。
  • NODE: クエリツリーを表現するための内部構造体。
  • QueryItem, QueryOperand: TSQuery 内の各ノードやオペランドを定義する構造体。

失敗モードと境界条件

  • **ストップワードのみ**: クエリがストップワードのみで構成される場合、root が NULL となり、サイズ 0 の TSQuery が返される。
  • **メモリ不足**: palloc によるメモリ確保が失敗した場合、PostgreSQL のエラーハンドリングによりアボートされる。

保守担当者向けメモ

  • 本関数はツリーの再帰処理を行うため、深いクエリ構造に対してはスタックオーバーフローのリスクがある。必要に応じて check_stack_depth() の呼び出しが適切に行われているか確認すること。
  • ストップワード除去に伴うフレーズ演算子の距離（ladd, radd）の再計算ロジックは clean_stopword_intree に依存しているため、距離計算の不具合はそちらを確認すること。
