gistnospace

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistutil.c:58

目的

GiSTインデックスページに指定されたタプル群を挿入する際、ページ分割が必要かどうかを判定します。

シグネチャまたは形状

bool gistnospace(Page page, IndexTuple *itvec, int len, OffsetNumber todelete, Size freespace)

入力と出力

  • **入力**:
  • page: 対象のGiSTインデックスページ。
  • itvec: 挿入予定のインデックスタプルの配列。
  • len: itvec 内のタプル数。
  • todelete: 削除予定のタプルオフセット（削除がない場合は InvalidOffsetNumber）。
  • freespace: 挿入時に考慮すべき追加の空き容量。
  • **出力**:
  • bool: ページ内に収まらない（分割が必要）場合は true、収まる場合は false を返します。

前提条件と不変条件

  • itvec に含まれる各タプルおよび todelete で指定されたタプルは、IndexTupleSize を用いてサイズ計算が可能である必要があります。

アルゴリズムまたは振る舞い

1. freespace を初期値として、挿入予定の全タプル（itvec）のサイズと、それらに必要な ItemIdData の合計サイズを算出します。 2. todelete が有効なオフセットであれば、削除対象タプルのサイズと ItemIdData 分を deleted 変数に加算します。 3. 現在のページの空き容量 (PageGetFreeSpace) に deleted 分を加えた値が、ステップ1で算出した必要サイズを下回る場合、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gist.c: インデックス挿入処理において、ページ分割の要否判定に使用。
  • **呼び出し先**:
  • PageGetItem, PageGetItemId, PageGetFreeSpace, IndexTupleSize: PostgreSQLのページ管理およびインデックスタプル操作用マクロ。

関連データ構造

  • Page: インデックスページ構造体。
  • IndexTuple: GiSTインデックスに格納されるタプル。
  • OffsetNumber: タプルのオフセット識別子。

失敗モードと境界条件

  • todelete が InvalidOffsetNumber の場合、削除による空き容量の増加は考慮されません。
  • ページが既に満杯に近い状態で、かつ len が大きい場合、常に true を返します。

保守担当者向けメモ

  • 本関数は、GiSTの挿入処理における「ページ分割のトリガー」を決定する重要なロジックです。
  • freespace 引数は、インデックス構築時や更新時の動的な空き容量要件を柔軟に扱うために使用されます。
