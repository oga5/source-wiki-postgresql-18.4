GetAggInitVal

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAgg.c:4383

目的

集約関数の初期値（initval）を、テキスト形式から指定されたデータ型の内部表現（Datum）へ変換して取得する。

シグネチャまたは形状

Datum GetAggInitVal(Datum textInitVal, Oid transtype)

入力と出力

  • **入力**:
  • textInitVal: 初期値を表すテキスト形式の Datum。
  • transtype: 初期値の変換先となるデータ型の Oid。
  • **出力**:
  • 変換後の Datum 型の初期値。

前提条件と不変条件

  • transtype に対応する入力関数がシステムカタログから取得可能であること。
  • textInitVal は TextDatumGetCString で変換可能な有効なテキストデータであること。

アルゴリズムまたは振る舞い

1. getTypeInputInfo を呼び出し、transtype に対応する入力関数（typinput）とパラメータ（typioparam）を取得する。 2. TextDatumGetCString を使用して、textInitVal を C 文字列（strInitVal）に変換する。 3. OidInputFunctionCall を呼び出し、取得した入力関数を用いて strInitVal を Datum に変換する。 4. pfree を呼び出し、一時的に確保した strInitVal を解放する。 5. 変換された initVal を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAgg.c
  • src/backend/executor/nodeWindowAgg.c
  • src/backend/optimizer/prep/prepagg.c
  • **呼び出し先**:
  • getTypeInputInfo
  • TextDatumGetCString
  • OidInputFunctionCall
  • pfree

関連データ構造

  • Datum: PostgreSQL の汎用データ表現型。
  • Oid: オブジェクト識別子。

失敗モードと境界条件

  • transtype が無効な型である場合、getTypeInputInfo がエラーを発生させる可能性がある。
  • 入力関数が不正な文字列を処理できない場合、OidInputFunctionCall がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は nodeAgg.c だけでなく nodeWindowAgg.c や prepagg.c からも参照されており、集約処理における共通の初期化ロジックとして機能している。変更時はこれらのモジュールへの影響を考慮すること。
