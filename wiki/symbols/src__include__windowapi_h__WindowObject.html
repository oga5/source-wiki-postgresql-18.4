WindowObject

定義

  • 種別: typedef struct WindowObjectData *WindowObject;
  • 定義箇所: src/include/windowapi.h:37

目的

ウィンドウ関数が実行時にパーティション内のデータへアクセスし、状態を管理するための不透明な実行コンテキストを提供します。

シグネチャまたは形状

WindowObjectData 構造体へのポインタ型です。内部構造は隠蔽されており、直接メンバにアクセスすることはできません。

入力と出力

  • **入力**: PG_WINDOW_OBJECT() マクロを使用して、関数呼び出し情報（fcinfo->context）から取得します。
  • **出力**: ウィンドウ関数がパーティション内の行データ、フレーム情報、またはローカルメモリを取得するためのインターフェースとして機能します。

前提条件と不変条件

  • PostgreSQL の V1 関数呼び出し規約に従う関数内でのみ使用可能です。
  • WindowObjectIsValid() マクロにより、使用前に有効性を検証することが推奨されます。
  • 構造体メンバへの直接アクセスは禁止されており、提供された Win* 系 API 関数を経由して操作する必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: PG_WINDOW_OBJECT() を介してコンテキストを取得します。 2. **データアクセス**: WinGetFuncArgInPartition や WinGetFuncArgInFrame を使用し、相対位置（seektype）を指定してデータをフェッチします。 3. **状態管理**: WinGetPartitionLocalMemory を使用して、パーティション処理期間中に有効なローカルメモリを確保・保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeWindowAgg.c 内のウィンドウ関数実行エンジンが、このオブジェクトを生成・管理し、各ウィンドウ関数へ渡します。
  • **呼び出し先**: 本シンボル自体はポインタ型であり、関数ではありません。関連する操作関数（WinGetFuncArgInPartition 等）が本オブジェクトを引数として受け取ります。

関連データ構造

  • WindowObjectData: WindowObject が指し示す実体（nodeWindowAgg.c で定義）。
  • FunctionCallInfo (fcinfo): WindowObject を保持するコンテキストの起点。

失敗モードと境界条件

  • 無効な WindowObject を使用した場合の挙動は未定義です。
  • パーティションの境界を超えたデータアクセスを試みた場合の挙動は、各 Win* API の仕様に依存します。

保守担当者向けメモ

  • 本構造体は nodeWindowAgg.c で隠蔽されているため、API の変更は windowapi.h と nodeWindowAgg.c の両方に影響を与える可能性があります。
  • メモリ管理については、WinGetPartitionLocalMemory で確保した領域がパーティション終了時に適切に解放されることを前提としています。
