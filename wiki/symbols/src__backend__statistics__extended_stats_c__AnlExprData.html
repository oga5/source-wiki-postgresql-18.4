AnlExprData

定義

  • 種別: struct
  • 定義箇所: src/backend/statistics/extended_stats.c:86

目的

拡張統計情報（Extended Statistics）の構築プロセスにおいて、分析対象となる式（Expression）とその統計情報を保持するために使用されます。

シグネチャまたは形状

typedef struct AnlExprData
{
    Node         *expr;         /* 分析対象の式 */
    VacAttrStats *vacattrstat;  /* 分析対象の統計属性 */
} AnlExprData;

入力と出力

  • **入力**: build_expr_data 関数により、式のリストと統計ターゲット（stattarget）に基づいて初期化されます。
  • **出力**: compute_expr_stats および serialize_expr_stats に渡され、統計計算結果の算出およびカタログへのシリアライズに使用されます。

前提条件と不変条件

  • expr は有効な Node ポインタである必要があります。
  • vacattrstat は、分析対象の属性統計情報を保持する構造体へのポインタです。

アルゴリズムまたは振る舞い

  • ANALYZE 実行時、build_expr_data がこの構造体の配列をメモリ上に確保し、各要素を初期化します。
  • compute_expr_stats がこの構造体を参照して、式に対する統計情報を計算します。
  • serialize_expr_stats が計算済みのデータをカタログ格納用に変換します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/extended_stats.c 内の統計構築ロジック（BuildRelationExtStatistics 等）から動的に生成・参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • VacAttrStats: 属性統計情報を保持する構造体。
  • Node: PostgreSQLの構文木ノード。

失敗モードと境界条件

  • palloc0 によるメモリ確保失敗時にエラーが発生する可能性があります。
  • nexprs（式の数）が0の場合、構造体の配列確保は行われません。

保守担当者向けメモ

  • 本構造体は extended_stats.c 内の統計構築パイプラインにおいて、式ベースの統計を扱うための主要なデータコンテナです。
  • 新しい統計種別を追加する際、式評価のデータフローを変更する場合は本構造体のメンバ構成に影響が出る可能性があります。
