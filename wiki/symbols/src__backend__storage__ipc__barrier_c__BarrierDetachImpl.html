BarrierDetachImpl

定義

  • 種別: static inline function
  • 定義箇所: src/backend/storage/ipc/barrier.c:299

目的

動的バリアからプロセスを離脱させ、必要に応じてバリアのフェーズ更新や待機中の他プロセスの起床を行う。

シグネチャまたは形状

static inline bool BarrierDetachImpl(Barrier *barrier, bool arrive)

入力と出力

  • **入力**:
  • barrier: 操作対象の Barrier 構造体へのポインタ。
  • arrive: 現在のプロセスが離脱前にバリア到達済みとして扱うかを示すフラグ。
  • **出力**:
  • bool: 離脱の結果、バリアの参加者が 0 名になった場合に true を返す。

前提条件と不変条件

  • barrier->static_party が false であること（動的バリアであること）。
  • barrier->participants > 0 であること。
  • 内部操作は barrier->mutex スピンロックによって保護される。

アルゴリズムまたは振る舞い

1. barrier->mutex を取得し、barrier->participants をデクリメントする。 2. 以下の条件を満たす場合、バリアのフェーズを進行させ、待機中の全プロセスを起床させる準備を行う。

  • arrive が真、または離脱後も参加者が残っている。
  • かつ、現在の到達数 barrier->arrived が更新後の barrier->participants と一致する。

    • 3. 上記条件でフェーズ進行が決定した場合、barrier->arrived を 0 にリセットし、barrier->phase をインクリメントする。 4. barrier->participants が 0 になったかを判定し、スピンロックを解放する。 5. フェーズ進行が決定していた場合、ConditionVariableBroadcast を呼び出し、待機中の全プロセスを起床させる。 6. 最後に、参加者が 0 になったかどうかを示すフラグを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • BarrierArriveAndDetach (直接の呼び出し)
  • BarrierArriveAndDetachExceptLast (直接の呼び出し)
  • **呼び出し先**:
  • SpinLockAcquire / SpinLockRelease
  • ConditionVariableBroadcast

関連データ構造

  • Barrier: 同期状態を保持する構造体。mutex、participants、arrived、phase、condition_variable を含む。

失敗モードと境界条件

  • participants が 0 の状態で呼び出された場合、Assert により異常終了する。
  • 静的バリア（static_party が真）に対して呼び出された場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数は BarrierArriveAndDetach および BarrierArriveAndDetachExceptLast の共通実装である。
  • arrive 引数は、離脱するプロセスがそのフェーズの完了に寄与するかどうかを制御する。
  • 離脱によって参加者が 0 になった場合、呼び出し側でバリアの破棄や後処理を行う必要がある。
