pg_abs_s64

定義

  • 種別: function
  • 定義箇所: src/include/common/int.h:351

目的

符号付き64ビット整数（int64）の絶対値を、オーバーフローを回避して符号なし64ビット整数（uint64）として取得する。

シグネチャまたは形状

static inline uint64 pg_abs_s64(int64 a)

入力と出力

  • **入力**: int64 a （絶対値を求める対象の整数）
  • **出力**: uint64 （a の絶対値）

前提条件と不変条件

  • a が PG_INT64_MIN の場合、通常の符号反転では正の範囲に収まらないため、特別な処理を行う。
  • 戻り値は常に非負の uint64 型として表現される。

アルゴリズムまたは振る舞い

1. 入力値 a が PG_INT64_MIN と等しいか判定する。 2. PG_INT64_MIN の場合、PG_INT64_MAX + 1 を計算して返す（これは 2^63 に相当する）。 3. それ以外の場合、i64abs(a) を呼び出し、その結果を uint64 にキャストして返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/cash.c:448
  • src/backend/utils/adt/numeric.c:8235
  • **呼び出し先**:
  • i64abs() （プラットフォーム依存の絶対値計算関数）

関連データ構造

  • int64 / uint64: PostgreSQL で定義された標準的な64ビット整数型。

失敗モードと境界条件

  • a = PG_INT64_MIN: 負の最小値は、通常の符号反転ではオーバーフローを引き起こすが、本関数は PG_INT64_MAX + 1 を返すことで安全に処理する。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、ヘッダーファイル内で完結している。
  • i64abs の実装がプラットフォームによって異なる可能性があるため、移植性を考慮する場合は src/include/common/int.h の全体的な設計方針に従うこと。
