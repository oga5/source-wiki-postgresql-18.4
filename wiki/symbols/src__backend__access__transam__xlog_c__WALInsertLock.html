WALInsertLock

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/xlog.c:368

目的

複数のバックエンドプロセスが WAL（Write-Ahead Log）バッファへレコードを挿入する際の競合を制御し、並列性を確保するための軽量ロック構造体です。

シグネチャまたは形状

typedef struct
{
	LWLock		lock;
	pg_atomic_uint64 insertingAt;
	XLogRecPtr	lastImportantAt;
} WALInsertLock;

入力と出力

  • **入力**: 構造体内の lock（LWLock）を介して、WAL 挿入領域への排他アクセスを制御します。
  • **出力**: insertingAt（アトミックな 64 ビット整数）により、現在そのロックを使用して挿入が行われている WAL の位置を追跡します。

前提条件と不変条件

  • 本構造体は共有メモリ上に配置されます。
  • WAL 挿入を行うプロセスは、まずこの構造体の lock を取得する必要があります。
  • insertingAt は、WAL 挿入の進行状況を追跡するために使用されます。

アルゴリズムまたは振る舞い

1. **ロック取得**: WAL レコードを挿入しようとするプロセスは、固定数の WALInsertLock のいずれかを取得します。 2. **位置の予約**: ロック取得後、insertingAt を更新して WAL バッファ内の書き込み位置を予約します。 3. **データコピー**: 予約した領域に対して WAL レコードのデータをコピーします。 4. **解放**: 挿入完了後、ロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlog.c 内の WAL 挿入処理（XLogInsertRecord 等）から参照されます。
  • **呼び出し先**: 構造体内部で LWLock（storage/lwlock.h）および pg_atomic_uint64（port/atomics.h）を利用しています。
  • 注: 具体的な関数レベルの呼び出し関係は、提供されたソース excerpt の範囲では限定的です。

関連データ構造

  • WALInsertLockPadded: WALInsertLock をキャッシュライン境界に配置するためのパディング付き構造体。
  • XLogCtlData: WAL 管理の全体状態を保持する構造体。本ロックはこの内部で管理されます。

失敗モードと境界条件

  • **競合**: 複数のバックエンドが同時に WAL を挿入しようとする場合、ロックの競合が発生し、待機状態となります。
  • **不整合**: ロック取得後に値を再確認しない場合、WAL の挿入位置に不整合が生じる可能性があります。

保守担当者向けメモ

  • 本構造体は src/backend/access/transam/xlog.c の 368 行目で定義されています。
  • WAL のアクティビティを直接追跡する役割を持つため、この構造体の変更は WAL 挿入のパフォーマンスに直結します。
  • 構造体のサイズや配置（パディング）がキャッシュラインの競合に影響を与えるため、変更時は注意が必要です。
