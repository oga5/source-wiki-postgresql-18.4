remove_target_dir

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/file_ops.c:255

目的

ターゲットデータディレクトリ内の指定されたディレクトリを安全に削除します。

シグネチャまたは形状

static void remove_target_dir(const char *path)

入力と出力

  • **入力**: path (削除対象のディレクトリパス。ターゲットデータディレクトリからの相対パス)
  • **出力**: なし（成功時は正常終了、失敗時は pg_fatal によりプロセスを終了）

前提条件と不変条件

  • path_is_safe_for_extraction(path) が真であること（ターゲットディレクトリ外へのアクセスを防ぐため）。
  • グローバル変数 datadir_target がターゲットディレクトリのパスとして設定されていること。

アルゴリズムまたは振る舞い

1. path_is_safe_for_extraction を呼び出し、パスの安全性を検証する。安全でない場合は pg_fatal で終了する。 2. グローバル変数 dry_run が真であれば、何もせず終了する。 3. datadir_target と path を結合し、完全なパスを生成する。 4. rmdir() を実行してディレクトリを削除する。 5. rmdir() が失敗した場合は、エラーメッセージと共に pg_fatal を呼び出し、プロセスを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/file_ops.c 内の remove_target 関数から呼び出されることが確認されています。
  • **呼び出し先**: path_is_safe_for_extraction、snprintf、rmdir、pg_fatal。

関連データ構造

  • datadir_target (グローバル変数: ターゲットデータディレクトリのパス)
  • dry_run (グローバル変数: ドライランモードの有効/無効)

失敗モードと境界条件

  • **パスの安全性違反**: path_is_safe_for_extraction が偽を返すと pg_fatal により終了する。
  • **削除失敗**: rmdir が失敗した場合（ディレクトリが空でない、権限不足など）、pg_fatal によりエラー詳細を出力して終了する。

保守担当者向けメモ

  • 本関数は dry_run フラグを尊重します。新しい削除ロジックを追加する際は、このパターンを維持してください。
  • rmdir を使用しているため、削除対象のディレクトリが空でない場合は失敗します。ディレクトリツリーの再帰的な削除が必要な場合は、別の処理フローを検討する必要があります。
