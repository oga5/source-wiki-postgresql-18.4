gin_consistent_hstore

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_gin.c:150

目的

GIN 索引を用いた検索において、索引から得られた候補がクエリ条件（@> や ? 等）を実際に満たしているかを判定し、再確認（recheck）が必要かどうかを決定する。

シグネチャまたは形状

Datum gin_consistent_hstore(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • check (bool *): 索引キーがクエリ条件に合致しているかを示すフラグ配列。
  • strategy (StrategyNumber): 実行する検索演算子の戦略番号。
  • nkeys (int32): 検索対象のキー数。
  • recheck (bool *): 出力用。再確認が必要な場合に true を設定する。
  • **出力**:
  • Datum (bool): 検索条件に合致した場合は true、それ以外は false。

前提条件と不変条件

  • strategy には HStoreContainsStrategyNumber、HStoreExistsStrategyNumber、HStoreExistsAnyStrategyNumber、HStoreExistsAllStrategyNumber のいずれかが渡される必要がある。それ以外の場合はエラーとなる。

アルゴリズムまたは振る舞い

1. **HStoreContainsStrategyNumber (@>)**:

  • recheck を true に設定する（キーと値のペアの結合情報が索引にないため）。
  • check 配列内のすべてのキーが true であれば true を返し、一つでも false があれば false を返す。

    • 2. **HStoreExistsStrategyNumber (?) / HStoreExistsAnyStrategyNumber (?|)**:

  • recheck を false に設定する。
  • 常に true を返す（デフォルトの検索モードでキーの存在が保証されているため）。

    • 3. **HStoreExistsAllStrategyNumber (?&)**:

  • recheck を false に設定する。
  • check 配列内のすべてのキーが true であれば true を返し、一つでも false があれば false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GIN アクセスメソッド（直接的な呼び出し元コードはソース内に明示されていない）。
  • **呼び出し先**: elog（エラー発生時）。

関連データ構造

  • StrategyNumber: 演算子の種類を識別するために使用。
  • hstore 関連の戦略定数（HStoreContainsStrategyNumber 等）。

失敗モードと境界条件

  • 未定義の strategy が渡された場合、elog(ERROR, ...) により処理が中断される。
  • nkeys が 0 の場合、ループ処理はスキップされ、戦略に応じた初期値（res = true）が返される。

保守担当者向けメモ

  • 本関数は GIN 索引の consistent インターフェースを実装している。
  • recheck が true になるのは、hstore の GIN 索引がキーと値のペアとしての結合情報を保持していないためである。
  • 索引項目に付与されるフラグ（K, V, N）との整合性を維持すること。
