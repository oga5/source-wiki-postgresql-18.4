BTreeShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtutils.c:3640

目的

B-tree インデックスの管理に必要な共有メモリ領域のサイズを算出する。

シグネチャまたは形状

Size BTreeShmemSize(void)

入力と出力

  • 入力: なし
  • 出力: Size 型（必要な共有メモリのバイト数）

前提条件と不変条件

  • MaxBackends（最大バックエンド数）が定義されている環境で実行されること。
  • BTVacInfo 構造体のサイズおよび BTOneVacInfo 構造体のサイズが計算の基礎となる。

アルゴリズムまたは振る舞い

1. BTVacInfo 構造体の vacuums フィールドまでのオフセットサイズを取得する。 2. MaxBackends に BTOneVacInfo のサイズを乗算し、共有メモリの総サイズに加算する。 3. 計算結果のサイズを返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/ipc/ipci.c:145 (共有メモリ初期化プロセスの一部として呼び出される)
  • 呼び出し先:
  • add_size (PostgreSQL ユーティリティ関数)
  • mul_size (PostgreSQL ユーティリティ関数)
  • offsetof (標準マクロ)

関連データ構造

  • BTVacInfo: B-tree のバキューム情報を保持する構造体。
  • BTOneVacInfo: 個別のバキューム情報を保持する構造体。

失敗モードと境界条件

  • add_size および mul_size は、計算結果がオーバーフローする場合に適切なエラー処理を行う（PostgreSQL のメモリ管理規約に基づく）。

保守担当者向けメモ

  • 本関数は BTVacInfo 構造体のレイアウト変更に伴い、計算ロジックを更新する必要がある。
  • src/include/access/nbtree.h に外部宣言が存在する。
