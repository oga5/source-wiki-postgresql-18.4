vm_extend

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/visibilitymap.c:633

目的

可視性マップ（Visibility Map）の物理ファイルを指定されたブロック数まで拡張し、他のバックエンドに対してリレーションのストレージ管理情報（smgr）の無効化を通知する。

シグネチャまたは形状

static Buffer vm_extend(Relation rel, BlockNumber vm_nblocks)

入力と出力

  • **入力**:
  • rel: 対象のリレーション。
  • vm_nblocks: 拡張後の目標ブロック数。
  • **出力**:
  • 拡張されたマップのバッファ（Buffer）。

前提条件と不変条件

  • VISIBILITYMAP_FORKNUM を対象として操作を行う。
  • 拡張操作は ExtendBufferedRelTo を介して実行される。

アルゴリズムまたは振る舞い

1. ExtendBufferedRelTo を呼び出し、可視性マップのフォークを vm_nblocks まで拡張する。この際、EB_CREATE_FORK_IF_NEEDED および EB_CLEAR_SIZE_CACHE フラグを使用し、エラー時には RBM_ZERO_ON_ERROR を適用する。 2. CacheInvalidateSmgr を呼び出し、共有無効化メッセージを送信する。これにより、他のバックエンドが保持している当該リレーションの古い smgr 参照を強制的にクローズさせる。 3. 拡張されたバッファを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/visibilitymap.c 内の関数（vm_extend を呼び出す箇所が確認されている）。
  • **呼び出し先**:
  • ExtendBufferedRelTo
  • CacheInvalidateSmgr
  • RelationGetSmgr

関連データ構造

  • Relation: 操作対象のリレーション情報。
  • Buffer: 拡張されたマップのバッファハンドル。
  • VISIBILITYMAP_FORKNUM: 可視性マップが格納されるフォーク番号。

失敗モードと境界条件

  • ExtendBufferedRelTo の仕様に基づき、ファイルシステムの制限やディスク容量不足により拡張が失敗する可能性がある。
  • RBM_ZERO_ON_ERROR が指定されているため、エラー発生時にはゼロ初期化されたバッファが返される挙動をとる。

保守担当者向けメモ

  • 本関数は、可視性マップの物理的なサイズ変更を伴うため、並行して実行される他のバックエンドとの整合性を CacheInvalidateSmgr によって維持している。
  • 呼び出し元で vm_extend を使用する際は、同時実行による拡張の競合が発生し得ることを考慮する必要がある。
