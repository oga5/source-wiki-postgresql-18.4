encrypt_password

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/crypt.c:116

目的

指定されたアルゴリズムに基づいてパスワードをハッシュ化し、認証用の文字列を生成します。

シグネチャまたは形状

char *encrypt_password(PasswordType target_type, const char *role, const char *password);

入力と出力

  • **入力**:
  • target_type: 生成するハッシュ形式（PasswordType 列挙型）。
  • role: ロール名（MD5ハッシュのソルトとして使用）。
  • password: ハッシュ化対象の平文パスワード。
  • **出力**:
  • ハッシュ化されたパスワード文字列（palloc 済み）。

前提条件と不変条件

  • 入力されたパスワードが既に暗号化済み（PASSWORD_TYPE_PLAINTEXT 以外）である場合、変換は行われず元の文字列が複製されて返されます。
  • 生成されるハッシュ文字列は MAX_ENCRYPTED_PASSWORD_LEN (512バイト) 以下である必要があります。これを超える場合はエラーとなります。

アルゴリズムまたは振る舞い

1. get_password_type で入力パスワードの形式を判定します。 2. 既に暗号化されている場合は、そのまま複製して返します。 3. 平文の場合は target_type に応じて処理を分岐します：

  • PASSWORD_TYPE_MD5: pg_md5_encrypt を使用してハッシュ化します。
  • PASSWORD_TYPE_SCRAM_SHA_256: pg_be_scram_build_secret を使用してハッシュ化します。
  • PASSWORD_TYPE_PLAINTEXT: エラーを発生させます。

    • 4. MD5使用時は、設定に応じて非推奨警告（md5_password_warnings）を出力します。

関連データ構造

  • PasswordType: パスワードのハッシュアルゴリズムを定義する列挙型。

失敗モードと境界条件

  • **エラー**:
  • target_type が PASSWORD_TYPE_PLAINTEXT の場合。
  • ハッシュ化後の文字列長が MAX_ENCRYPTED_PASSWORD_LEN を超える場合。
  • pg_md5_encrypt が失敗した場合。
  • **境界条件**: 認証時はデータベース選択前であるため、TOASTテーブルへのアクセスを伴う処理は制限されます。

保守担当者向けメモ

  • MD5サポートは将来的に廃止予定であるため、MD5関連の警告ロジックが含まれています。
  • セキュリティ上の理由から、認証失敗時の詳細はクライアントへ返さずサーバーログに出力する設計を維持してください。
