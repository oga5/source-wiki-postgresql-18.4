LogStmtLevel

定義

  • 種別: enum
  • 定義箇所: src/include/tcop/tcopprot.h:31

目的

SQLステートメントのログ出力レベルを制御するための列挙型です。

シグネチャまたは形状

typedef enum
{
    LOGSTMT_NONE,
    LOGSTMT_DDL,
    LOGSTMT_MOD,
    LOGSTMT_ALL,
} LogStmtLevel;

入力と出力

  • 本シンボル自体は列挙型であり、入力や出力は持ちません。
  • GetCommandLogLevel(Node *parsetree) 関数が、この列挙型の値を戻り値として返すことが src/include/tcop/utility.h で宣言されています。

前提条件と不変条件

  • LOGSTMT_NONE から LOGSTMT_ALL までの順序は、ログ出力の包含関係（なし < DDLのみ < DDL+更新系 < すべて）を暗黙的に示唆しています。

アルゴリズムまたは振る舞い

  • LOGSTMT_NONE: ステートメントをログ出力しません。
  • LOGSTMT_DDL: データ定義言語（DDL）ステートメントのみをログ出力します。
  • LOGSTMT_MOD: データ変更ステートメントおよびDDLをログ出力します。
  • LOGSTMT_ALL: すべてのステートメントをログ出力します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tcop/utility.c 内で、ログレベルの判定や変数定義に使用されています。
  • 呼び出し先: 本シンボルは列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • Node: GetCommandLogLevel の引数として使用される構文解析ツリーのノード。

失敗モードと境界条件

  • 該当なし。

保守担当者向けメモ

  • src/include/tcop/utility.h にある GetCommandLogLevel と密接に関連しています。ログ出力の要件が変更された場合は、この列挙型の定義と、それを利用する utility.c 側の判定ロジックを併せて確認してください。
