r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_danish.c:136

目的

単語内の処理領域（R1, R2相当）を特定し、ステミングアルゴリズムが接尾辞を適用するための境界位置を計算する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z （ステミング環境の状態を保持する構造体）
  • 出力: 成功時に 1、失敗時に 0 を返す。内部的に z->I[0] および z->I[1] を更新する。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • g_v（母音グループ）が定義されている環境で使用される。

アルゴリズムまたは振る舞い

1. z->I[1] を文字列の末尾 z->l で初期化する。 2. 文字列の先頭から3文字スキップした位置を z->I[0] に設定する。 3. out_grouping_U を使用して、母音以外の文字を検索する。 4. 続いて in_grouping_U を使用して、母音の直後の位置を特定し、その位置を z->I[1] に更新する。 5. z->I[1] が z->I[0] より小さい場合、z->I[1] を z->I[0] に設定する（領域の最小境界を保証）。

呼び出し元と呼び出し先

  • 呼び出し元: danish_UTF_8_stem（ステミング処理の初期化フェーズで呼び出される）。
  • 呼び出し先: skip_utf8, out_grouping_U, in_grouping_U（いずれも libstemmer の共通ユーティリティ）。

関連データ構造

  • struct SN_env: ステミング環境を管理する構造体。z->I 配列に領域境界を格納する。

失敗モードと境界条件

  • skip_utf8 またはグループ検索関数が負の値を返した場合、処理を中断し 0 を返す。
  • 文字列が短く、領域を特定できない場合は境界値が調整される。

保守担当者向けメモ

  • 本関数は自動生成コードの一部である。アルゴリズムの変更が必要な場合は、直接編集せず元のSnowballソース（.sbl）を修正すること。
  • z->I[0] と z->I[1] は、後続の接尾辞除去ルーチンで参照される重要なインデックスである。
