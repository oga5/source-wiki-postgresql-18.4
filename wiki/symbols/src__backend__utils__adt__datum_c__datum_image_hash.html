datum_image_hash

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/datum.c:357

目的

Datum 値の論理的な意味ではなく、その物理的なバイナリイメージに基づいてハッシュ値を計算します。

シグネチャまたは形状

uint32 datum_image_hash(Datum value, bool typByVal, int typLen)

入力と出力

  • **入力**:
  • value: ハッシュ対象の Datum。
  • typByVal: 値渡し（true）か参照渡し（false）かを示すフラグ。
  • typLen: 型の長さ（バイト数）。1, 2, 4（値渡し用）、-1（可変長）、-2（C文字列）のいずれか。
  • **出力**:
  • 計算された uint32 型のハッシュ値。

前提条件と不変条件

  • typByVal および typLen は、対象となるデータ型の正しいメタデータである必要があります。不適切な値を渡すとメモリ破壊や不正なポインタ参照を引き起こします。
  • typLen が -1 の場合、TOAST されたデータは適切に展開（detoast）されます。

アルゴリズムまたは振る舞い

1. **値渡し (typByVal == true)**: typLen に応じて Datum を正規化し、sizeof(Datum) 分のメモリ領域に対して hash_bytes を実行します。 2. **固定長参照渡し (typLen > 0)**: Datum が指すポインタから typLen バイト分を hash_bytes で処理します。 3. **可変長 (typLen == -1)**: toast_raw_datum_size でサイズを算出し、PG_DETOAST_DATUM_PACKED で展開後、データ本体に対して hash_bytes を実行します。必要に応じて一時的なメモリを pfree します。 4. **C文字列 (typLen == -2)**: DatumGetCString で文字列を取得し、終端文字を含む長さで hash_bytes を実行します。 5. **エラー**: 上記以外の typLen が指定された場合、elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeMemoize.c:183 (確認済み)
  • **呼び出し先**:
  • hash_bytes (common/hashfn.h)
  • toast_raw_datum_size (access/detoast.h)
  • PG_DETOAST_DATUM_PACKED (access/detoast.h)
  • pfree (postgres.h)
  • elog (postgres.h)

関連データ構造

  • Datum: PostgreSQL の汎用データ型表現。
  • varlena: 可変長データ型の内部構造体。

失敗モードと境界条件

  • typLen が -1 以外の予期しない値の場合、elog(ERROR) により処理が中断されます。
  • TOAST されたデータがメモリ不足等で展開できない場合、PG_DETOAST_DATUM_PACKED 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はバイナリ比較を行うため、同じ論理値であっても内部表現が異なる場合は異なるハッシュ値が生成されます。
  • typLen == -1 の処理において、PG_DETOAST_DATUM_PACKED がコピーを作成した場合のみ pfree を行う安全なメモリ管理が行われています。
