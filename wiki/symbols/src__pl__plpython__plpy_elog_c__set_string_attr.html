set_string_attr

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_elog.c:595

目的

Python オブジェクトに対して、指定された属性名で文字列値（または None）を設定するユーティリティ関数です。主に PostgreSQL のエラー情報を Python 例外オブジェクトへマッピングする際に使用されます。

シグネチャまたは形状

static bool set_string_attr(PyObject *obj, char *attrname, char *str)

入力と出力

  • **入力**:
  • obj: 属性を設定する対象の PyObject。
  • attrname: 設定する属性名を示す文字列。
  • str: 設定する値。NULL の場合は Python の None が設定されます。
  • **出力**:
  • bool: 属性の設定に成功した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • obj は有効な PyObject である必要があります。
  • str が NULL でない場合、PLyUnicode_FromString を介して Python 文字列オブジェクトへ変換されます。
  • 内部で Py_INCREF および Py_DECREF を使用し、一時的に生成した属性値の参照カウントを適切に管理します。

アルゴリズムまたは振る舞い

1. str が NULL であれば Py_None を取得し、参照カウントをインクリメントします。 2. str が NULL でなければ PLyUnicode_FromString(str) で Python 文字列を生成します。生成失敗時は false を返します。 3. PyObject_SetAttrString を呼び出し、obj に attrname で指定された属性をセットします。 4. 生成した一時オブジェクトの参照カウントを Py_DECREF で減らします。 5. PyObject_SetAttrString の戻り値が -1 でない（成功した）場合に true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_elog.c 内の複数箇所（sqlstate, detail, hint, query 等の属性を Python 例外オブジェクトにセットする処理）。
  • **呼び出し先**: PLyUnicode_FromString（文字列変換）、PyObject_SetAttrString（属性設定）、Py_INCREF/Py_DECREF（参照カウント操作）。

関連データ構造

  • PyObject: Python の汎用オブジェクト型。

失敗モードと境界条件

  • PLyUnicode_FromString がメモリ不足等で NULL を返した場合、関数は直ちに false を返します。
  • PyObject_SetAttrString が失敗した場合（戻り値が -1）、関数は false を返します。

保守担当者向けメモ

  • 本関数は PLy_exception_set_with_details 等から呼び出され、PostgreSQL のエラー情報を Python 例外オブジェクトへ転送する際の定型処理を抽象化しています。
  • 属性設定の成否を呼び出し元に伝える設計となっているため、呼び出し側で戻り値のチェックを省略しないよう注意してください。
