bt_index_check

定義

  • 種別: function
  • 定義箇所: contrib/amcheck/verify_nbtree.c:251

目的

SQLインターフェースを介してB-Treeインデックスの論理的な整合性を検証するためのエントリポイントです。

シグネチャまたは形状

Datum bt_index_check(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • indrelid (OID): 検証対象のインデックスのOID（第1引数）。
  • heapallindexed (bool, オプション): ヒープ整合性チェックの有効化（第2引数）。
  • checkunique (bool, オプション): 一意性制約の検証（第3引数）。
  • **出力**: void（検証失敗時はエラーをスロー）。

前提条件と不変条件

  • 対象インデックスに対して AccessShareLock を取得します。
  • heapallindexed が true の場合、インデックス内の全タプルがヒープに存在するかを検証します。
  • checkunique が true の場合、インデックスの一意性制約を検証します。

アルゴリズムまたは振る舞い

1. BTCallbackState を初期化し、引数からオプション設定を読み込みます。 2. amcheck_lock_relation_and_check を呼び出し、対象インデックスをロックした上で検証用コールバック bt_index_check_callback を実行します。 3. インデックスの構造的健全性（キー順序、兄弟リンク等）および指定されたオプションに基づく検証を行い、完了後に制御を戻します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • SQLクエリ（例: SELECT bt_index_check(...)）
  • src/bin/pg_amcheck/pg_amcheck.c
  • **呼び出し先**:
  • amcheck_lock_relation_and_check
  • bt_index_check_callback

関連データ構造

  • BTCallbackState: 検証オプションを保持する構造体。

失敗モードと境界条件

  • インデックスが破損している場合、検証プロセス内でエラーが発生しトランザクションが中断されます。
  • AccessShareLock を取得できない場合、ロック待ちまたはエラーが発生します。

保守担当者向けメモ

  • 本関数はSQLから直接呼び出されるため、引数の数（PG_NARGS()）に基づいた動的なオプション処理を行っています。
  • インデックスの構造検証の詳細は bt_index_check_callback およびその先の bt_check_every_level 等に委譲されています。
