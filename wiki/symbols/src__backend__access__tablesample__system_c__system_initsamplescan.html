system_initsamplescan

定義

  • 種別: function
  • 定義箇所: src/backend/access/tablesample/system.c:129

目的

TABLESAMPLE SYSTEM メソッドによるスキャン実行のための状態管理構造体（SystemSamplerData）を初期化し、メモリを割り当てる。

シグネチャまたは形状

static void system_initsamplescan(SampleScanState *node, int eflags)

入力と出力

  • **入力**:
  • node: SampleScanState 構造体へのポインタ。スキャン状態を保持する。
  • eflags: スキャン実行時のフラグ（本関数内では未使用）。
  • **出力**: なし（副作用として node->tsm_state にメモリが割り当てられる）。

前提条件と不変条件

  • node は有効な SampleScanState 構造体を指している必要がある。
  • palloc0 により、SystemSamplerData の全メンバはゼロ初期化される。

アルゴリズムまたは振る舞い

1. palloc0 を使用して SystemSamplerData サイズのメモリ領域を確保する。 2. 確保した領域を node->tsm_state に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: tsm_system_handler 構造体を通じて、TABLESAMPLE の初期化フェーズで呼び出される。
  • **呼び出し先**: palloc0（PostgreSQL のメモリ管理関数）。

関連データ構造

  • SampleScanState: スキャン実行時の状態を保持する構造体。
  • SystemSamplerData: 本関数で初期化される、system メソッド専用のプライベート状態構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる。

保守担当者向けメモ

  • SystemSamplerData は palloc0 で確保されるため、スキャン終了時にメモリリークが発生しないよう、適切なクリーンアップ処理が呼び出されていることを確認すること。
