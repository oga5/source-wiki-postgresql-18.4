Result

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/plannodes.h:248

目的

クエリ実行計画ツリーにおいて、定数のみで構成されるフィルタ条件（resconstantqual）を評価する、あるいは単一の行を生成するためのプランノードを定義します。

シグネチャまたは形状

typedef struct Result
{
	Plan		plan;
	Node	   *resconstantqual;
} Result;

入力と出力

  • **入力**: plan メンバを介して子ノード（lefttree）からのタプルを受け取ることが可能です。
  • **出力**: resconstantqual の評価結果に基づき、タプルを親ノードへ渡すか、あるいはフィルタリングによって破棄します。

前提条件と不変条件

  • Plan 構造体を最初のメンバとして保持しており、Plan* へのキャストが可能です。
  • resconstantqual は、実行時に評価される定数式（または定数のみを含む式）を保持します。

アルゴリズムまたは振る舞い

1. エグゼキュータは、このノードの実行時に resconstantqual を評価します。 2. resconstantqual が偽（または NULL）と評価された場合、このノードはタプルを返しません。 3. resconstantqual が真と評価された場合、子ノードから取得したタプル（または子ノードがない場合は単一の計算結果）をそのまま親ノードへ渡します。

呼び出し元と呼び出し先

  • **呼び出し元**: プランナ（src/backend/optimizer/）がクエリ最適化の過程でこのノードを生成します。
  • **呼び出し先**: エグゼキュータ（src/backend/executor/）がこのノードを走査し、実行します。
  • **注記**: ツールによる参照情報には、contrib/postgres_fdw や src/backend/access/ 等のコードが含まれていますが、これらは主にこの構造体への言及や、プランツリー内での存在確認を行う箇所であり、直接的な関数呼び出し関係はソース本文からは特定できません。

関連データ構造

  • Plan: 基底構造体。コストやターゲットリストなどの共通情報を保持します。

失敗モードと境界条件

  • resconstantqual が評価不能な式である場合、実行時にエラーが発生する可能性があります。
  • 子ノードが存在しない場合、Result ノードは定数のみの評価結果に基づいて単一の行を生成する役割を果たします。

保守担当者向けメモ

  • Result ノードは、Append ノードの上位に配置され計算を補助するケースや、postgres_fdw 等の外部データラッパーでプランツリーの先頭に配置されるケースが確認されています。
  • メンバへのアクセスには、Plan 構造体に対する標準的なマクロを使用してください。
