GlobalVisHorizonKind

定義

  • 種別: enum
  • 定義箇所: src/backend/storage/ipc/procarray.c:249

目的

PostgreSQLにおける可視性判定（Visibility Test）の対象となるデータ種別を識別するための列挙型です。各種別は、トランザクションの可視性境界（Horizon）を計算する際のカテゴリとして使用されます。

シグネチャまたは形状

typedef enum GlobalVisHorizonKind
{
    VISHORIZON_SHARED,
    VISHORIZON_CATALOG,
    VISHORIZON_DATA,
    VISHORIZON_TEMP,
} GlobalVisHorizonKind;

入力と出力

  • 本シンボル自体は列挙型であり、入力や出力は持ちません。
  • ComputeXidHorizons 等の関数において、どのカテゴリの境界値を計算するかを指定する引数として利用されます。

前提条件と不変条件

  • VISHORIZON_SHARED から VISHORIZON_TEMP までの4つの値が定義されており、これらは GlobalVisState 構造体内で各カテゴリの境界値を管理するために使用されます。

アルゴリズムまたは振る舞い

  • 各列挙子は、以下のテーブル種別に対応する可視性境界を表します。
  • VISHORIZON_SHARED: 共有カタログ用
  • VISHORIZON_CATALOG: カタログ用
  • VISHORIZON_DATA: 一般データ用
  • VISHORIZON_TEMP: 一時テーブル用

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/procarray.c:1970 付近で、この型を戻り値として使用する関数が定義されています。
  • 呼び出し先: 本シンボルは列挙型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • GlobalVisState: 各 GlobalVisHorizonKind に対応する境界値を保持する構造体。

失敗モードと境界条件

  • 列挙型であるため、定義外の値が渡された場合の動作は未定義です。

保守担当者向けメモ

  • 新しいテーブル種別や可視性判定のカテゴリを追加する必要がある場合は、この列挙型を拡張し、関連する GlobalVisState の計算ロジックを更新してください。
