getTypeBinaryInputInfo

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/lsyscache.c:3117

目的

指定されたデータ型のバイナリ入力関数（receive関数）のOIDと、その関数に渡すための補助的な型情報（IOParam）を取得します。

シグネチャまたは形状

void getTypeBinaryInputInfo(Oid type, Oid *typReceive, Oid *typIOParam)

入力と出力

  • **入力**:
  • type (Oid): 情報を取得する対象のデータ型OID。
  • typReceive (Oid *): バイナリ入力関数のOIDを格納するポインタ。
  • typIOParam (Oid *): 入力関数に渡す補助的な型情報（通常は要素型OIDなど）を格納するポインタ。
  • **出力**:
  • なし（引数のポインタ経由で値を返却）。

前提条件と不変条件

  • type に対応するシステムカタログ pg_type のエントリが存在すること。
  • 対象の型が「シェル型（未定義型）」ではないこと。
  • 対象の型にバイナリ入力関数（typreceive）が定義されていること。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用して TYPEOID キャッシュから対象型の HeapTuple を検索します。 2. キャッシュミス時はエラーを発生させます。 3. 取得したタプルから Form_pg_type 構造体へキャストし、以下の検証を行います。

  • typisdefined が偽であればエラー（シェル型へのアクセス）。
  • typreceive が無効なOIDであればエラー（バイナリ入力関数未定義）。

    • 4. pt->typreceive を typReceive に代入します。 5. getTypeIOParam を呼び出し、その戻り値を typIOParam に代入します。 6. ReleaseSysCache を呼び出し、参照カウントを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfrom.c
  • src/backend/replication/logical/worker.c
  • src/backend/tcop/fastpath.c
  • src/backend/tcop/postgres.c
  • src/backend/utils/adt/array_userfuncs.c
  • src/backend/utils/adt/domains.c
  • src/backend/utils/adt/rowtypes.c
  • **呼び出し先**:

  • ReleaseSysCache
  • elog / ereport

関連データ構造

  • Form_pg_type: pg_type システムカタログの行構造体。
  • HeapTuple: システムカタログの行データ。

失敗モードと境界条件

  • **キャッシュ検索失敗**: elog(ERROR) によりトランザクションを中断します。
  • **シェル型へのアクセス**: ERRCODE_UNDEFINED_OBJECT でエラーを報告します。
  • **バイナリ入力関数未定義**: ERRCODE_UNDEFINED_FUNCTION でエラーを報告します。

保守担当者向けメモ

  • 本関数は palloc されたメモリを直接返却するわけではないため、呼び出し元でのメモリ解放は不要です。
  • getTypeIOParam の内部ロジックに依存しているため、IOParamの決定ルールに変更がある場合はそちらも確認してください。
