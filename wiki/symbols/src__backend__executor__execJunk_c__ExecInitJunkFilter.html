ExecInitJunkFilter

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execJunk.c:59

目的

クエリ実行結果のタプルから、不要な「ジャンク属性」（ctid やソートキーなど）を除去するための JunkFilter 構造体を初期化します。

シグネチャまたは形状

JunkFilter *ExecInitJunkFilter(List *targetList, TupleTableSlot *slot)

入力と出力

  • **入力**:
  • targetList: プランノードのターゲットリスト（TargetEntry のリスト）。
  • slot: フィルタリング後のタプルを格納する TupleTableSlot。NULL の場合は新規作成されます。
  • **出力**:
  • 初期化済みの JunkFilter 構造体へのポインタ。

前提条件と不変条件

  • targetList 内の各 TargetEntry は、resjunk フラグによってジャンク属性かどうかが識別されている必要があります。
  • cleanMap は、クリーンなタプルの属性番号から元のタプルの属性番号へのマッピングを保持します。
  • 成功時、junkfilter->jf_cleanMap は cleanLength（ジャンクでない属性の数）分の要素を持つ配列となります。

アルゴリズムまたは振る舞い

1. ExecCleanTypeFromTL(targetList) を呼び出し、ジャンク属性を除外したタプル記述子（cleanTupType）を生成します。 2. slot が提供されていればその記述子を更新し、なければ MakeSingleTupleTableSlot で新規作成します。 3. targetList を走査し、resjunk が false である属性のみを抽出し、その属性番号を cleanMap に格納します。 4. JunkFilter 構造体を割り当て、ターゲットリスト、クリーンなタプル記述子、cleanMap、結果用スロットをセットして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExpr.c:3233
  • src/backend/executor/execMain.c:1024
  • src/backend/executor/functions.c:833
  • **呼び出し先**:
  • ExecCleanTypeFromTL

  • palloc
  • makeNode

関連データ構造

  • JunkFilter: フィルタリングに必要な情報を保持する構造体。
  • TargetEntry: ターゲットリストの各要素。resjunk フラグを持つ。
  • TupleTableSlot: タプルを保持するスロット。

失敗モードと境界条件

  • cleanLength が 0 の場合（ジャンク属性のみ、あるいは空のターゲットリスト）、cleanMap は NULL に設定されます。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 現在の実装では cleanMap を使用して手動で属性をマッピングしていますが、将来的に ProjectionInfo を活用したより効率的な手法への置き換えが検討されています。
  • cleanMap の属性番号は 1-based です。
