shm_mq_handle

定義

  • 種別: struct
  • 定義箇所: src/include/storage/shm_mq.h:26

目的

各バックエンドプロセスが共有メモリメッセージキュー（shm_mq）を操作するために保持する、プライベートな状態管理構造体です。

シグネチャまたは形状

不透明な構造体（opaque struct）として定義されています。詳細は src/backend/storage/ipc/shm_mq.c で管理されます。

入力と出力

本構造体自体はデータ保持用であり、入出力は shm_mq_send や shm_mq_receive などの API を通じて行われます。

前提条件と不変条件

  • 本構造体は、shm_mq_attach を呼び出すことで生成されます。
  • 単一リーダー・単一ライターの制約に従い、特定のハンドルは一方向の通信（送信または受信）に使用されます。

アルゴリズムまたは振る舞い

  • 共有メモリ上の shm_mq 構造体と、操作を行うローカルプロセスを橋渡しします。
  • プロセス間通信において、キューの接続状態や同期のためのラッチ情報を保持します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/transam/parallel.c
  • src/backend/executor/execParallel.c
  • src/backend/executor/tqueue.c
  • src/backend/libpq/pqmq.c
  • **呼び出し先**:
  • 構造体の内部実装は src/backend/storage/ipc/shm_mq.c に依存しています。

関連データ構造

  • shm_mq: 共有メモリ上に配置されるキュー本体。
  • shm_mq_result: 操作結果を示す列挙型。
  • dsm_segment: メッセージキューが配置される動的共有メモリセグメント。

失敗モードと境界条件

  • 相手側プロセスが終了した場合、shm_mq_receive 等の API を通じて SHM_MQ_DETACHED が返されます。この場合、ハンドルを介した通信は継続できません。

保守担当者向けメモ

  • 本構造体は不透明型として定義されているため、構造体のメンバに直接アクセスせず、shm_mq.h で提供される公開 API を使用してください。
  • 複数のプロセスから同一のハンドルを同時に操作することはできません。
