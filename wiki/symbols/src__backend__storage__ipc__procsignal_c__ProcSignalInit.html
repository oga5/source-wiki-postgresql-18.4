ProcSignalInit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/procsignal.c:166

目的

現在のプロセスに対して共有メモリ上の ProcSignalSlot を割り当て、シグナル受信状態およびバリア状態を初期化する。

シグネチャまたは形状

void ProcSignalInit(const uint8 *cancel_key, int cancel_key_len)

入力と出力

  • **入力**:
  • cancel_key: プロセス固有のキャンセルキー（バイト列）。
  • cancel_key_len: キーの長さ（0 から MAX_CANCEL_KEY_LENGTH の範囲）。
  • **出力**: なし（副作用として共有メモリおよびプロセスローカル変数を更新）。

前提条件と不変条件

  • MyProcNumber が正しく設定されていること。
  • MyProcNumber が 0 以上 NumProcSignalSlots 未満であること。
  • プロセス起動時の初期化フェーズで呼び出されること（状態の不整合を防ぐため）。

アルゴリズムまたは振る舞い

1. MyProcNumber に対応する共有メモリ上の ProcSignalSlot を取得する。 2. スピンロック (pss_mutex) を取得し、排他制御下で以下を実行する。

  • 既存のシグナルフラグをすべてクリアする。
  • バリアチェックマスクを 0 にリセットする。
  • 現在のグローバルなバリア生成番号 (psh_barrierGeneration) を取得し、自身のスロットに書き込む。
  • キャンセルキーをスロットにコピーし、自身のPIDを記録する。

    • 3. スピンロックを解放する。 4. 以前のPIDが 0 でなかった場合、スロットが再利用されている旨をログ出力する。 5. MyProcSignalSlot グローバル変数に自身のスロットアドレスを保持する。 6. プロセス終了時に CleanupProcSignalState が実行されるよう on_shmem_exit に登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/auxprocess.c:69
  • src/backend/utils/init/postinit.c:752
  • **呼び出し先**:
  • SpinLockAcquire / SpinLockRelease
  • pg_atomic_read_u32 / pg_atomic_write_u32 / pg_atomic_read_u64 / pg_atomic_write_u64
  • MemSet
  • memcpy
  • on_shmem_exit (登録先: CleanupProcSignalState)
  • elog

関連データ構造

  • ProcSignalSlot: プロセスごとのシグナル状態を保持する構造体。
  • ProcSignalHeader: 共有メモリ上の管理ヘッダー。

失敗モードと境界条件

  • MyProcNumber が未設定または範囲外の場合、ERROR でプロセスを終了させる。
  • cancel_key_len が MAX_CANCEL_KEY_LENGTH を超える場合、アサーション違反となる。
  • スロットが空でない（以前のPIDが残っている）場合、LOG レベルで警告を出力する。

保守担当者向けメモ

  • 本関数はプロセス起動の早い段階で実行される必要がある。これは、初期化後にキャッシュされた状態が古いバリア生成番号と混在することを防ぐためである。
  • pss_mutex を用いた排他制御を行っているが、PIDの読み取りなど一部はアトミック操作で最適化されている。
