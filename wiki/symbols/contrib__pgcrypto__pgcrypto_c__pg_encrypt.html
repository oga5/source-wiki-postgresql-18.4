pg_encrypt

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgcrypto.c:251

目的

SQL インターフェースから提供されたデータと鍵を用い、指定されたアルゴリズムで対称鍵暗号化を実行する。

シグネチャまたは形状

Datum pg_encrypt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_BYTEA_PP(0): 暗号化対象のデータ (bytea)
  • PG_GETARG_BYTEA_PP(1): 暗号化に使用する鍵 (bytea)
  • PG_GETARG_TEXT_PP(2): アルゴリズム名 (text)
  • **出力**: 暗号化されたデータ (bytea) を Datum として返す。

前提条件と不変条件

  • find_provider により、指定されたアルゴリズム名に対応する PX_Combo プロバイダーが取得可能であること。
  • PostgreSQL のメモリ管理ルールに従い、入力引数は PG_FREE_IF_COPY で適切に解放される。

アルゴリズムまたは振る舞い

1. find_provider を呼び出し、暗号化アルゴリズムを特定して PX_Combo コンテキストを取得する。 2. px_combo_encrypt_len で必要な出力バッファサイズを計算し、palloc でメモリを確保する。 3. px_combo_init で鍵を設定する（IV は指定されないため NULL）。 4. px_combo_encrypt を実行してデータを暗号化する。 5. px_combo_free でコンテキストを解放する。 6. 処理成功時は結果を返し、失敗時は ereport(ERROR, ...) を発行してトランザクションを中断する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1(pg_encrypt) により登録）。
  • **呼び出し先**: find_provider, px_combo_encrypt_len, px_combo_init, px_combo_encrypt, px_combo_free, px_strerror。

関連データ構造

  • PX_Combo: 暗号化処理の抽象化コンテキスト。
  • bytea, text: PostgreSQL のデータ型。

失敗モードと境界条件

  • find_provider がアルゴリズムを見つけられない場合、エラーが発生する。
  • px_combo_init または px_combo_encrypt がエラーを返した場合、確保したメモリを pfree し、ERRCODE_EXTERNAL_ROUTINE_INVOCATION_EXCEPTION を伴うエラーを報告する。

保守担当者向けメモ

  • 本関数は SQL と内部の px レイヤーを繋ぐブリッジである。暗号化アルゴリズムの詳細は px レイヤー（px.c 等）を参照すること。
  • PG_FREE_IF_COPY の使用により、入力引数のメモリ管理が適切に行われていることを確認すること。
