LocalPgBackendStatus

定義

  • 種別: struct
  • 定義箇所: src/include/utils/backend_status.h:249

目的

バックエンドプロセスの状態情報をローカルメモリ上で保持するための構造体です。共有メモリ上の PgBackendStatus を補完し、プロセス番号やトランザクションIDなどの詳細なセッション情報を管理します。

シグネチャまたは形状

typedef struct LocalPgBackendStatus
{
    PgBackendStatus backendStatus;
    ProcNumber      proc_number;
    TransactionId   backend_xid;
    TransactionId   backend_xmin;
    int             backend_subxact_count;
    bool            backend_subxact_overflowed;
} LocalPgBackendStatus;

入力と出力

  • **入力**: 共有メモリ上の PgBackendStatus エントリおよび現在のセッションのトランザクション状態。
  • **出力**: 統計情報収集や監視プロセスが参照可能な、ローカルにキャッシュされたバックエンド状態の集合。

前提条件と不変条件

  • backend_xid および backend_xmin は、有効なトランザクションが存在しない場合 InvalidTransactionId となります。
  • backend_subxact_count は現在のセッションにおけるキャッシュされたサブトランザクション数を表します。

アルゴリズムまたは振る舞い

  • 本構造体は、共有メモリ上の状態をローカルにコピーし、検索や統計処理を効率化するために使用されます。
  • src/backend/utils/activity/backend_status.c 内で、bsearch() を用いた配列検索の対象として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/backend_status.c 内の統計情報取得関数群（pgstat_get_local_beentry_by_proc_number 等）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • PgBackendStatus: 共有メモリ上で管理されるバックエンドの基底状態構造体。
  • ProcNumber: プロセス識別子。
  • TransactionId: トランザクション識別子。

失敗モードと境界条件

  • backend_subxact_overflowed が true の場合、サブトランザクション数がキャッシュ制限を超過していることを示します。
  • 検索操作（bsearch）を行う際は、比較関数 cmp_lbestatus が正しく実装されている必要があります。

保守担当者向けメモ

  • 本構造体は共有メモリ上の PgBackendStatus を内包しているため、共有メモリ側の定義が変更された場合は本構造体のメモリレイアウトにも影響が出る可能性があります。
  • 統計情報の整合性を保つため、ローカルコピーの更新タイミングと共有メモリの同期プロトコルに注意してください。
