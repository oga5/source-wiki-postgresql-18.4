keepwal_entry

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_rewind/filemap.c:68

目的

pg_rewind において、同期対象から除外すべき、あるいは保持すべき WAL 関連ファイルのパスと状態を管理するためのデータ構造です。

シグネチャまたは形状

typedef struct keepwal_entry
{
    const char *path;
    uint32      status;
} keepwal_entry;

入力と出力

本構造体はデータ保持用であり、関数ではありません。path にはファイルパスの文字列ポインタを、status には状態を示すビットフラグ等を保持します。

前提条件と不変条件

  • path は有効な文字列を指している必要があります。
  • simplehash.h を用いたハッシュテーブルの要素として利用されることを前提としています。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持に特化しており、filemap.c 内で SH_ELEMENT_TYPE として定義され、ハッシュテーブルの要素として管理されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_rewind/filemap.c 内のハッシュテーブル操作関数（keepwal_entry_exists 等）から参照されます。
  • 呼び出し先: 構造体であるため、直接的な呼び出し先はありません。

関連データ構造

  • filemap.c 内のハッシュテーブル（simplehash.h を使用）。

失敗モードと境界条件

  • path が NULL の場合、またはハッシュテーブルへの挿入時にメモリ不足が発生した場合の挙動は、利用側の実装に依存します。

保守担当者向けメモ

  • src/bin/pg_rewind/filemap.c:75 にて SH_ELEMENT_TYPE として定義されており、simplehash の型として利用されています。変更時はハッシュテーブルの整合性に注意してください。
