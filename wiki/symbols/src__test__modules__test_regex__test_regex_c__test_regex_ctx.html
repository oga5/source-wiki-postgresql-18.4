test_regex_ctx

定義

  • 種別: struct
  • 定義箇所: src/test/modules/test_regex/test_regex.c:37

目的

PostgreSQL の正規表現エンジンに対するテスト実行中、複数回の呼び出し（SRF: Set Returning Function）にわたってマッチング状態、位置情報、およびコンパイル済みフラグを保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct test_regex_ctx
{
    test_re_flags re_flags;
    rm_detail_t   details;
    text         *orig_str;
    int           nmatches;
    int           npatterns;
    int          *match_locs;
    int           next_match;
    Datum        *elems;
    bool         *nulls;
    pg_wchar     *wide_str;
    char         *conv_buf;
    int           conv_bufsiz;
} test_regex_ctx;

入力と出力

  • **入力**: setup_test_matches 関数によって初期化され、palloc0 を介してメモリが割り当てられます。
  • **出力**: build_test_match_result 関数へ渡され、マッチング結果を SQL 配列形式に変換するためのワークスペースとして利用されます。

前提条件と不変条件

  • match_locs 配列のサイズは nmatches * npatterns * 2 です。
  • elems および nulls 配列は npatterns + 1 個の要素を保持します。
  • next_match は処理すべき次のマッチングの 0 始まりのインデックスを指します。

アルゴリズムまたは振る舞い

1. **初期化**: setup_test_matches が正規表現のコンパイルと対象文字列全体のマッチングを実行し、その結果を本構造体に格納します。 2. **反復**: test_regex 関数が SRF として呼び出されるたびに、funcctx->user_fctx から本構造体を取得し、next_match を進めながらマッチング結果を逐次構築します。 3. **終了**: 全てのマッチングが完了すると、呼び出し元は SRF_RETURN_DONE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: setup_test_matches（初期化）、test_regex（状態保持）、build_test_match_result（結果構築）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • test_re_flags: 正規表現のコンパイルおよび実行オプションを保持。
  • rm_detail_t: 正規表現実行時の詳細情報。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • nmatches が 0 の場合、反復処理は即座に終了します。
  • マルチバイト文字を含む場合、wide_str への変換および conv_buf の管理が適切に行われないと、インデックスの不整合が発生する可能性があります。

保守担当者向けメモ

  • match_locs は 0 始まりの文字インデックスを格納します。
  • elems および nulls は build_test_match_result の実行時のみ使用されるワークスペースです。
  • 本構造体は SRF のコンテキストとして user_fctx に格納されるため、メモリリークを防ぐために MemoryContext のライフサイクルに注意してください。
