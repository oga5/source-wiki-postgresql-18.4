TrieChar

定義

  • 種別: struct
  • 定義箇所: contrib/unaccent/unaccent.c:43

目的

アクセント記号付き文字を置換するための Trie 木構造の各ノードを表現し、置換ルールを効率的に保持・検索するために使用されます。

シグネチャまたは形状

typedef struct TrieChar
{
	struct TrieChar *nextChar;
	char	   *replaceTo;
	int			replacelen;
} TrieChar;

入力と出力

  • **入力**: 構造体内の各フィールドは、Trie 木の階層構造（nextChar）および置換後の文字列データ（replaceTo, replacelen）を保持します。
  • **出力**: 該当なし（データ構造体のため）。

前提条件と不変条件

  • Trie 木の各ノードは、256 個の TrieChar 構造体からなる配列として確保されることが想定されています（contrib/unaccent/unaccent.c:33 の記述に基づく）。
  • nextChar は、次の文字に対応するノード配列を指します。

アルゴリズムまたは振る舞い

  • 本構造体は、placeChar 関数によって Trie 木へのルール登録時に利用され、findReplaceTo 関数によって入力文字列の走査および置換ルールの検索に使用されます。
  • 文字列のバイト単位での走査をサポートし、最長一致検索を実現するためのノードとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: placeChar（ノードの確保と初期化）、findReplaceTo（ノードの走査）、および unaccent_init 関連の初期化処理。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • rootTrie: Trie 木の起点となる TrieChar へのポインタ。

失敗モードと境界条件

  • メモリ確保失敗時: palloc0 を使用しているため、PostgreSQL のメモリ管理ポリシーに従い、確保失敗時はエラーが発生します。
  • 境界条件: マルチバイト文字はバイト単位で処理されるため、エンコーディングが不整合な場合、Trie 木の走査結果が意図しない挙動を示す可能性があります。

保守担当者向けメモ

  • 本構造体は unaccent モジュールの置換ロジックの根幹です。Trie 木のノード配列（256要素）のメモリレイアウトを変更する場合は、placeChar および findReplaceTo の走査ロジックへの影響を考慮してください。
