PLy_function_restore_args

定義

  • **種別**: 関数
  • **定義箇所**: src/pl/plpython/plpy_exec.c:541

目的

再帰的な関数呼び出し等で退避されていた PL/Python プロシージャの引数および実行コンテキスト（args, TD）を、Python のグローバル辞書（proc->globals）に復元し、一時的な保存用構造体を解放する。

シグネチャまたは形状

static void PLy_function_restore_args(PLyProcedure *proc, PLySavedArgs *savedargs)

入力と出力

  • **入力**:
  • proc: 対象となる PLyProcedure 構造体へのポインタ。
  • savedargs: 復元対象の引数状態を保持する PLySavedArgs 構造体へのポインタ。
  • **出力**: なし（副作用として proc->globals を更新し、メモリを解放する）。

前提条件と不変条件

  • savedargs は、事前に PLy_function_save_args 等によって適切に初期化・設定されている必要がある。
  • proc->globals は有効な Python 辞書オブジェクトであること。

アルゴリズムまたは振る舞い

1. **名前付き引数の復元**: proc->argnames が存在する場合、savedargs->namedargs に保持された各引数を proc->globals に再設定し、一時保持していたオブジェクトの参照カウントを Py_DECREF で減らす。 2. **args オブジェクトの復元**: savedargs->args が存在する場合、proc->globals に "args" として再設定し、参照カウントを減らす。 3. **TD オブジェクトの復元**: savedargs->td が存在する場合、proc->globals に "TD" として再設定し、参照カウントを減らす。 4. **メモリ解放**: pfree(savedargs) を呼び出し、保存用構造体自体を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_exec.c:116 (SRF 状態のクリーンアップ時)
  • src/pl/plpython/plpy_exec.c:655 (関数実行終了時)
  • **呼び出し先**:
  • PyDict_SetItemString (Python C API)
  • Py_DECREF (Python C API)
  • pfree (PostgreSQL メモリ管理)

関連データ構造

  • PLyProcedure: プロシージャの定義および実行環境（グローバル辞書等）を保持。
  • PLySavedArgs: 退避された引数、args、TD オブジェクトを保持する構造体。

失敗モードと境界条件

  • savedargs が NULL の場合、あるいは proc->globals が無効な場合、セグメンテーションフォールトが発生する可能性がある。
  • Python C API の関数呼び出しが失敗した場合の挙動については、本関数内では明示的なエラーハンドリングを行っていない。

保守担当者向けメモ

  • 本関数は PLy_function_save_args と対になる操作を行う。参照カウントの管理（Py_DECREF）が重要であり、復元後に Python 辞書へ所有権を移譲する設計となっている。
