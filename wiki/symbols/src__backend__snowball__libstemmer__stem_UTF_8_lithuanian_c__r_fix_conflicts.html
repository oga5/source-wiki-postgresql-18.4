r_fix_conflicts

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_lithuanian.c:667

目的

リトアニア語の語幹抽出過程において、特定の綴りパターンに起因する不整合や衝突を修正する。

シグネチャまたは形状

static int r_fix_conflicts(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（Snowball ステミング環境構造体）
  • **出力**: int 型。成功時は 1、修正対象外または処理不要な場合は 0、エラー発生時は負の値を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • 内部で find_among_b を使用しており、a_2 テーブルが定義されていることが前提となる。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置 z->c を z->ket に設定する。 2. 文字列の末尾付近の条件チェック（境界条件および特定の文字コード判定）を行い、条件を満たさない場合は 0 を返す。 3. find_among_b を用いて、後方から a_2 テーブルに基づいた接尾辞マッチングを行う。 4. マッチした場合、z->bra を現在のカーソル位置に設定し、マッチしたインデックス（among_var）に応じて slice_from_s を呼び出し、該当部分を置換する。 5. 置換処理が正常終了すれば 1 を返し、エラー時はそのエラーコードを伝播させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/libstemmer/stem_UTF_8_lithuanian.c:788（lithuanian_UTF_8_stem 内のメイン処理フローから呼び出される）。
  • **呼び出し先**: find_among_b（Snowball ランタイム）、slice_from_s（Snowball ランタイム）。

関連データ構造

  • struct SN_env: ステミング処理の状態を保持する構造体。
  • a_2: 修正対象の接尾辞パターンを保持する among テーブル。

失敗モードと境界条件

  • 文字列長が不足している場合（z->c - 3 <= z->lb）、処理を中断し 0 を返す。
  • find_among_b がマッチしない場合、0 を返す。
  • slice_from_s がメモリ不足やその他の理由で失敗した場合、負の戻り値を返す。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードである。直接編集せず、元の .sbl スクリプトを修正すること。
  • among_var の各ケース（1〜8）は、特定の綴り修正ルールに対応している。修正が必要な場合は、対応する Snowball スクリプトの fix_conflicts 定義を確認すること。
