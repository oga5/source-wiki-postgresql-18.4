pg_md5_init

定義

  • 種別: function
  • 定義箇所: src/common/md5.c:381

目的

MD5 メッセージダイジェスト計算のためのコンテキスト構造体 (pg_md5_ctx) を初期化し、ハッシュ計算を開始可能な状態にする。

シグネチャまたは形状

void pg_md5_init(pg_md5_ctx *ctx)

入力と出力

  • **入力**: pg_md5_ctx *ctx - 初期化対象となるコンテキスト構造体へのポインタ。
  • **出力**: なし（副作用として ctx の各メンバが初期値で上書きされる）。

前提条件と不変条件

  • ctx は有効なメモリ領域を指している必要がある。
  • 初期化後、ctx は MD5 アルゴリズムの標準的な初期状態（定数 MD5_A0 ～ MD5_D0）を保持する。

アルゴリズムまたは振る舞い

1. メッセージ長カウンタ (md5_n) とバッファ内インデックス (md5_i) を 0 にリセットする。 2. MD5 アルゴリズムの初期定数（MD5_A0, MD5_B0, MD5_C0, MD5_D0）をコンテキストのステータス変数に代入する。 3. 内部バッファ (md5_buf) を memset により 0 で埋める。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/cryptohash.c:108 で確認されている。
  • **呼び出し先**: memset（標準ライブラリ関数）。

関連データ構造

  • pg_md5_ctx: ハッシュ計算の状態を保持する構造体（md5_int.h で定義）。

失敗モードと境界条件

  • ctx が NULL の場合、未定義の動作（セグメンテーション違反等）を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は RFC 1321 に準拠した純粋な実装の一部である。
  • パスワードハッシュ等の用途には適しているが、現代のセキュリティ要件（衝突耐性など）が求められる用途には SHA-256 等のより強力なアルゴリズムの使用を検討すること。
