XLogReaderFree

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogreader.c:161

目的

XLogReaderState 構造体によって確保されたメモリおよび関連リソースを解放し、リーダーの状態を破棄する。

シグネチャまたは形状

void XLogReaderFree(XLogReaderState *state)

入力と出力

  • **入力**: state (XLogReaderState *) - 解放対象のリーダー状態構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • state は XLogReaderAllocate 等で適切に初期化された有効なポインタであること。
  • 関数呼び出し後、state が指すメモリ領域にはアクセスしてはならない。

アルゴリズムまたは振る舞い

1. state->seg.ws_file が -1 でない場合、state->routine.segment_close コールバックを呼び出し、開いているセグメントファイルを閉じる。 2. decode_buffer が存在し、かつ free_decode_buffer が真であれば、decode_buffer を解放する。 3. エラーメッセージバッファ errormsg_buf を解放する。 4. readRecordBuf が存在する場合は解放する。 5. readBuf を解放する。 6. 最後に state 構造体自体を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_walinspect/pg_walinspect.c
  • src/backend/access/transam/twophase.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/postmaster/walsummarizer.c
  • src/backend/replication/logical/logical.c
  • src/bin/pg_rewind/parsexlog.c
  • **呼び出し先**:
  • state->routine.segment_close (コールバック関数)
  • pfree (メモリ解放関数)

関連データ構造

  • XLogReaderState: リーダーのライフサイクル管理に使用されるメイン構造体。

失敗モードと境界条件

  • state が NULL の場合、pfree の挙動に依存する（通常はセグメンテーション違反を引き起こす可能性がある）。
  • 各バッファポインタが NULL の場合、そのポインタに対する pfree はスキップされるため安全である。

保守担当者向けメモ

  • 本関数は pfree を使用するため、フロントエンドツールで利用する際は、適切なメモリコンテキストがアクティブであることを確認すること。
  • segment_close コールバックは、ws_file が開かれている場合にのみ実行される。
