manifest_file

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_combinebackup/load_manifest.h:22

目的

バックアップマニフェストに含まれる個別のファイルメタデータ（パス、サイズ、チェックサム等）をメモリ上で保持するために使用されます。

シグネチャまたは形状

typedef struct manifest_file
{
	uint32		status;			/* hash status */
	const char *pathname;
	uint64		size;
	pg_checksum_type checksum_type;
	int			checksum_length;
	uint8	   *checksum_payload;
} manifest_file;

入力と出力

  • 本構造体は、load_backup_manifest() を通じてマニフェストファイルから読み込まれたデータで初期化されます。
  • simplehash を用いたハッシュテーブルの要素として格納されます。

前提条件と不変条件

  • simplehash の要素として利用されるため、load_manifest.c 内で SH_ELEMENT_TYPE として定義されています。
  • checksum_payload は checksum_length に応じたメモリ領域を指す必要があります。

アルゴリズムまたは振る舞い

  • マニフェスト内のファイル情報を構造化し、pg_combinebackup におけるファイル検索や統合処理を容易にします。
  • ハッシュテーブル内での管理状態を status フィールドで保持します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_combinebackup/load_manifest.c 内のハッシュテーブル操作関数およびマニフェスト解析ロジック。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。
  • 注意: src/bin/pg_basebackup/pg_basebackup.c に manifest_file という名前の変数が見られますが、これは FILE * 型であり、本構造体とは無関係です。

関連データ構造

  • manifest_data: 本構造体を要素として含むハッシュテーブルを保持するルート構造体。
  • pg_checksum_type: common/checksum_helper.h で定義されるチェックサムアルゴリズムの識別子。

失敗モードと境界条件

  • マニフェストの解析中にメモリ不足が発生した場合、構造体の割り当てや checksum_payload の確保に失敗する可能性があります。
  • 不正なマニフェスト形式の場合、各フィールドに不整合な値が設定される可能性があります。

保守担当者向けメモ

  • simplehash を使用しているため、ハッシュテーブルの操作関数（manifest_files_insert 等）は load_manifest.c 内でマクロ展開により生成されています。
  • pathname は const char * ですが、マニフェスト読み込み時に確保されたメモリ領域を指すため、ライフサイクル管理に注意してください。
