XLogWalRcvProcessMsg

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walreceiver.c:818

目的

プライマリサーバーから受信したレプリケーションメッセージ（WALデータまたはKeepalive）を解析し、適切な処理へ振り分ける。

シグネチャまたは形状

static void XLogWalRcvProcessMsg(unsigned char type, char *buf, Size len, TimeLineID tli)

入力と出力

  • type: メッセージ種別（'w': WALデータ, 'k': Keepalive）。
  • buf: メッセージ本体のデータバッファ。
  • len: バッファの長さ。
  • tli: 現在のタイムラインID。
  • 出力: なし（副作用としてWALの書き込みやプライマリへの応答送信を行う）。

前提条件と不変条件

  • buf は len バイト以上の有効なメモリ領域を指していること。
  • メッセージ種別が 'w' または 'k' 以外の場合、プロトコル違反としてエラーを報告する。

アルゴリズムまたは振る舞い

1. **メッセージ種別の判定**:

  • **'w' (WALデータ)**:
  • ヘッダー（dataStart, walEnd, sendTime）を読み取り、ProcessWalSndrMessage を呼び出して受信状況を更新する。
  • 残りのバッファを XLogWalRcvWrite に渡し、ローカルディスクへの書き込みを実行する。
  • **'k' (Keepalive)**:
  • ヘッダー（walEnd, sendTime, replyRequested）を読み取り、ProcessWalSndrMessage を呼び出す。
  • replyRequested が真であれば、XLogWalRcvSendReply を呼び出し、即座にプライマリへ応答を返す。

    • 2. **エラーハンドリング**:

  • ヘッダー長が期待値に満たない場合、または未知のメッセージタイプを受信した場合は ERRCODE_PROTOCOL_VIOLATION でエラーを投げる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walreceiver.c 内のストリーミングループ（walrcv_receive の直後）。
  • **呼び出し先**:
  • ProcessWalSndrMessage
  • XLogWalRcvWrite
  • XLogWalRcvSendReply
  • ereport (エラー時)

関連データ構造

  • StringInfoData: 受信メッセージの解析に使用。
  • XLogRecPtr: WALの論理位置を保持。

失敗モードと境界条件

  • メッセージ長がヘッダーサイズ未満の場合、プロトコル違反としてプロセスが停止する。
  • 未定義のメッセージタイプを受信した場合、プロトコル違反としてプロセスが停止する。

保守担当者向けメモ

  • 本関数は libpqwalreceiver からの受信データを処理する中継点です。プロトコル仕様の変更（ヘッダー構造の変更など）は、本関数の hdrlen 計算および pq_getmsgint64 等の呼び出し箇所に直接影響します。
