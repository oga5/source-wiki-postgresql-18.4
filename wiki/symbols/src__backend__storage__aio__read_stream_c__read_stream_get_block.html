read_stream_get_block

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/read_stream.c:178

目的

ReadStream から次に読み込むべきブロック番号を取得します。バッファリングされたブロックがあればそれを返し、なければコールバックを介して新しいブロック番号を要求します。

シグネチャまたは形状

static inline BlockNumber read_stream_get_block(ReadStream *stream, void *per_buffer_data)

入力と出力

  • **入力**:
  • stream: 操作対象の ReadStream 構造体へのポインタ。
  • per_buffer_data: コールバックに渡されるバッファごとのデータ領域へのポインタ。
  • **出力**:
  • 取得した BlockNumber。

前提条件と不変条件

  • stream は有効な ReadStream インスタンスである必要があります。
  • per_buffer_data が提供される場合、そのサイズは stream->per_buffer_data_size と一致している必要があります。

アルゴリズムまたは振る舞い

1. stream->buffered_blocknum を確認し、InvalidBlockNumber でなければその値を返し、該当フィールドを InvalidBlockNumber にリセットします。 2. バッファリングされたブロックがない場合:

  • VALGRIND_MAKE_MEM_UNDEFINED を呼び出し、per_buffer_data 領域を未定義状態としてマークします。
  • stream->callback を実行し、新しいブロック番号を取得します。この際、stream->callback_private_data と per_buffer_data が引数として渡されます。

    • 3. 取得したブロック番号を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/read_stream.c 内の複数箇所（行 463, 824, 1033）。
  • **呼び出し先**:
  • stream->callback（ReadStream に登録された関数ポインタ）。

関連データ構造

  • ReadStream: ストリームの状態を保持するメイン構造体。
  • BlockNumber: PostgreSQL におけるブロック識別子。

失敗モードと境界条件

  • コールバックが InvalidBlockNumber を返す場合、そのまま呼び出し元に伝播されます。
  • per_buffer_data が NULL の場合、Valgrind のメモリチェックはスキップされるか、呼び出し側の実装に依存します（ソース内の呼び出し例では NULL を渡すケースも存在します）。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスを重視して設計されています。
  • Valgrind のマクロ呼び出しは、バッファ消費者がデータを正しく初期化しているかを検証するための重要なデバッグ支援です。変更時はこの整合性に注意してください。
