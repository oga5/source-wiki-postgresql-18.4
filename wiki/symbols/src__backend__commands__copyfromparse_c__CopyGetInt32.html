CopyGetInt32

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfromparse.c:361

目的

ネットワークバイトオーダーでエンコードされた 32 ビット整数を、入力ストリームから読み込み、ホストのバイトオーダーに変換して取得する。

シグネチャまたは形状

static inline bool CopyGetInt32(CopyFromState cstate, int32 *val)

入力と出力

  • **入力**:
  • cstate: CopyFromState 型のコピー操作の状態管理構造体。
  • val: 読み込んだ値を格納するための int32 型ポインタ。
  • **出力**:
  • 戻り値: 読み込みが成功した場合は true、失敗した場合は false。
  • val: 成功時にネットワークバイトオーダーから変換された値が書き込まれる。

前提条件と不変条件

  • cstate は有効なコピー操作の状態を指している必要がある。
  • 入力データはネットワークバイトオーダー（ビッグエンディアン）で提供されることを前提とする。

アルゴリズムまたは振る舞い

1. CopyReadBinaryData を呼び出し、入力ストリームから sizeof(uint32) 分のデータを読み込む。 2. 読み込みバイト数が sizeof(uint32) と一致しない場合、val を 0 に初期化し false を返す。 3. 読み込みに成功した場合、pg_ntoh32 を使用してネットワークバイトオーダーからホストのバイトオーダーへ変換し、val に格納して true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfromparse.c 内のバイナリデータ解析処理（行 164, 202, 216, 2020 付近）。
  • **呼び出し先**:
  • CopyReadBinaryData (直接の依存先)
  • pg_ntoh32 (バイトオーダー変換用)

関連データ構造

  • CopyFromState: COPY FROM 操作の実行状態を保持する構造体。

失敗モードと境界条件

  • 入力ストリームから 4 バイト未満でデータが終了した場合、読み込み失敗として false を返す。

保守担当者向けメモ

  • 本関数は static inline で定義されており、COPY のバイナリモードにおける低レベルなデータ読み込みを担う。
  • 呼び出し元で false が返された場合の挙動（エラーハンドリング等）は、各呼び出し箇所に依存する。
