AllocSetAlloc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/aset.c:966

目的

AllocSet メモリコンテキストにおいて、指定されたサイズのメモリチャンクを確保する。フリーリストの再利用、既存ブロックの空き領域利用、または新規ブロックの割り当てを動的に選択する。

シグネチャまたは形状

void *AllocSetAlloc(MemoryContext context, Size size, int flags)

入力と出力

  • **入力**:
  • context: 割り当て対象の MemoryContext（AllocSet としてキャストされる）。
  • size: 要求するメモリサイズ（バイト単位）。
  • flags: 割り当て動作を制御するフラグ。
  • **出力**:
  • 確保されたメモリ領域へのポインタ。

前提条件と不変条件

  • context は AllocSetIsValid を満たす必要がある。
  • set->blocks は「キーパーブロック」の存在により NULL にならない。
  • 要求サイズが allocChunkLimit を超える場合は、AllocSetAllocLarge へ委譲される。

アルゴリズムまたは振る舞い

1. **巨大割り当ての判定**: 要求サイズが allocChunkLimit を超える場合、AllocSetAllocLarge を呼び出して終了する。 2. **フリーリストの検索**: 要求サイズに対応するフリーリストを AllocSetFreeIndex で特定し、空きチャンクがあればそれを再利用する。

  • 再利用時、VALGRIND マクロを用いてメモリのアクセス権限を適切に制御する。
  • MEMORY_CONTEXT_CHECKING が有効な場合、センチネルバイトの配置やサイズ検証を行う。

    • 3. **既存ブロックの利用**: フリーリストに空きがない場合、現在のブロック (set->blocks) の空き領域を確認する。

  • 十分な空きがあれば AllocSetAllocChunkFromBlock を呼び出す。
  • 不足している場合は AllocSetAllocFromNewBlock を呼び出し、新しいブロックを確保して割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c: MemoryContext インターフェースの alloc 関数ポインタとして登録。
  • src/backend/utils/mmgr/aset.c: 内部的な再帰的割り当てやテストコードから参照。
  • **呼び出し先**:
  • AllocSetAllocLarge: 巨大な割り当て要求時。
  • AllocSetFreeIndex: サイズに応じたインデックス計算。
  • AllocSetAllocFromNewBlock: ブロックの空きが不足している場合。
  • AllocSetAllocChunkFromBlock: 既存ブロックから切り出す場合。

関連データ構造

  • AllocSet: MemoryContext を継承した主要構造体。
  • AllocBlock: メモリブロックを管理するヘッダー。
  • MemoryChunk: チャンクのメタデータを保持する構造体。
  • AllocFreeListLink: フリーリスト内のチャンクを繋ぐためのリンク構造。

失敗モードと境界条件

  • 要求サイズが allocChunkLimit を超える場合、AllocSetAllocLarge に依存する。
  • 現在のブロックに空きがない場合、新しいブロックの確保（malloc 等）が失敗するとメモリ不足エラーとなる。

保守担当者向けメモ

  • 本関数は AllocSet におけるメモリ割り当てのホットパスである。
  • VALGRIND マクロによるメモリ保護が厳密に行われているため、変更時はアクセス権限の整合性に注意すること。
  • MEMORY_CONTEXT_CHECKING 有効時のセンチネルバイト配置は、デバッグにおいてメモリ破壊を早期検知するために重要である。
