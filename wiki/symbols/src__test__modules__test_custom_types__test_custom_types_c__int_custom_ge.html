int_custom_ge

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_custom_types/test_custom_types.c:161

目的

カスタム型に対する「以上（greater than or equal to）」比較演算を実装するPostgreSQLのバックエンド関数です。

シグネチャまたは形状

Datum int_custom_ge(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される2つの int32 型の値。
  • **出力**: PG_RETURN_BOOL を通じて、第1引数が第2引数以上である場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力値は int32 として解釈可能である必要があります。
  • PostgreSQLの関数マネージャインターフェース（fmgr.h）に準拠した呼び出し規約に従う必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT32(0) および PG_GETARG_INT32(1) を使用して、渡された2つの引数を int32 型として取得します。 2. C言語の比較演算子 >= を用いて、第1引数と第2引数の大小関係を評価します。 3. 評価結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの演算子実行エンジンから呼び出されます（直接的なコード上の呼び出し元はソース内に明示されていません）。
  • **呼び出し先**: PG_GETARG_INT32 および PG_RETURN_BOOL マクロ（PostgreSQLの関数マネージャAPI）。
  • **その他**: PG_FUNCTION_INFO_V1(int_custom_ge) が同ファイル内の行36で定義されており、PostgreSQLの関数登録に使用されています。

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • PG_FUNCTION_ARGS: 関数引数を受け取るためのマクロ。

失敗モードと境界条件

  • 引数が int32 の範囲外である場合や、不正なメモリ領域を指している場合は、基盤となるマクロの挙動に依存します。
  • int32 同士の比較であるため、オーバーフロー等の境界条件は標準的な整数比較の挙動に従います。

保守担当者向けメモ

  • 本関数はテスト用モジュールの一部であり、標準的な比較ロジックを実装しています。
  • 比較ロジックを変更する場合は、関連する他の比較関数（int_custom_gt, int_custom_le 等）との整合性に注意してください。
