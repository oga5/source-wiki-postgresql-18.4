ginFinishSplit

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginbtree.c:671

目的

GINインデックスのページ分割処理において、分割されたページへのダウンリンクを親ページへ挿入し、必要に応じて親ページの分割を再帰的に伝播させることで、ツリーの整合性を完了させる。

シグネチャまたは形状

static void ginFinishSplit(GinBtree btree, GinBtreeStack *stack, bool freestack, GinStatsData *buildStats)

入力と出力

  • **入力**:
  • btree: GIN B-tree操作用のコンテキスト構造体。
  • stack: 分割されたページとその親へのパスを保持するスタック。
  • freestack: 処理終了後にスタックメモリを解放するかどうかを示すフラグ。
  • buildStats: インデックス構築統計情報（更新用）。
  • **出力**: なし（副作用としてツリー構造を更新）。

前提条件と不変条件

  • stack は分割された子ページを指している必要がある。
  • 処理中に遭遇した GIN_INCOMPLETE_SPLIT フラグを持つページは、ginFinishOldSplit によって修復される必要がある。
  • 親ページへのダウンリンク挿入が完了するまで、再帰的に処理が継続される。

アルゴリズムまたは振る舞い

1. **ループ開始**: 親ページをロックし、不完全な分割状態であれば ginFinishOldSplit で修復する。 2. **親の探索**: btree->findChildPtr を使用して、現在のページを指すダウンリンクを親ページ内で検索する。 3. **右兄弟への移動**: 親ページ内に該当するダウンリンクが見つからない場合、ginStepRight で右兄弟へ移動し、必要に応じて ginFindParents で親スタックを再構築する。 4. **ダウンリンク挿入**: btree->prepareDownlink で作成したデータを ginPlaceToPage を用いて親ページに挿入する。 5. **再帰的伝播**: ginPlaceToPage が分割を必要とした場合（done == false）、現在の親を新たな子としてループを継続する。 6. **終了処理**: 挿入が完了するまでスタックを遡り、ロックを解放する。freestack が真の場合はスタックメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginbtree.c 内の ginPlaceToPage（分割発生時）および ginFinishOldSplit の呼び出し元。
  • **呼び出し先**:
  • ginFinishOldSplit
  • btree->findChildPtr
  • ginStepRight
  • ginFindParents
  • btree->prepareDownlink
  • ginPlaceToPage
  • freeGinBtreeStack

関連データ構造

  • GinBtree: GIN B-tree操作の制御構造体。
  • GinBtreeStack: ツリーの階層構造を保持するスタック。
  • GinStatsData: インデックス構築の統計情報。

失敗モードと境界条件

  • **右端ページ**: 親ページが右端であるにもかかわらずダウンリンクが見つからない場合、ginFindParents を呼び出してスタックを再構築する。
  • **不完全な分割**: 処理中に GIN_INCOMPLETE_SPLIT フラグに遭遇した場合、その分割を先に完了させないと、ダウンリンク挿入先が正しく特定できず失敗する可能性がある。

保守担当者向けメモ

  • 本関数は再帰的なツリー更新を行うため、ロックの順序（下位から上位へ）と解放タイミングに注意すること。
  • USE_INJECTION_POINTS が有効な場合、リーフおよび内部ノードの分割完了前にインジェクションポイントが挿入されており、テスト時に利用可能である。
