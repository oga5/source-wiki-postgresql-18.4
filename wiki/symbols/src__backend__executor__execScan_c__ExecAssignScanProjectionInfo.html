ExecAssignScanProjectionInfo

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execScan.c:80

目的

スキャンノードにおける射影（Projection）処理を初期化します。プランナーが生成したターゲットリストと物理タプルの構造が一致する場合に射影処理をスキップする最適化を含みます。

シグネチャまたは形状

void ExecAssignScanProjectionInfo(ScanState *node)

入力と出力

  • **入力**: ScanState *node（スキャンノードの状態構造体）
  • **出力**: なし（副作用として node->ps 内の射影情報が更新されます）

前提条件と不変条件

  • node->ss_ScanTupleSlot が初期化され、有効な TupleDesc を保持している必要があります。
  • node->ps.plan が Scan 構造体としてキャスト可能である必要があります。

アルゴリズムまたは振る舞い

1. node->ps.plan から Scan 構造体を取得します。 2. node->ss_ScanTupleSlot からタプル記述子（TupleDesc）を取得します。 3. ExecConditionalAssignProjectionInfo を呼び出し、スキャン対象のリレーション ID（scanrelid）とタプル記述子を用いて、必要な射影情報を設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • nodeBitmapHeapscan.c
  • nodeCtescan.c
  • nodeFunctionscan.c
  • nodeIndexscan.c
  • nodeNamedtuplestorescan.c
  • nodeSamplescan.c
  • nodeSeqscan.c
  • nodeSubqueryscan.c
  • **呼び出し先**:
  • ExecConditionalAssignProjectionInfo（src/backend/executor/execTuples.c で定義）

関連データ構造

  • ScanState: スキャンノードの実行状態を保持する構造体。
  • Scan: プランツリーにおけるスキャンノードの定義。
  • TupleDesc: タプルの構造定義。

失敗モードと境界条件

  • node が NULL の場合、または node->ss_ScanTupleSlot が未初期化の場合はクラッシュする可能性があります。

保守担当者向けメモ

  • 本関数は ExecConditionalAssignProjectionInfo へのラッパーです。射影の最適化ロジックに変更を加える場合は、呼び出し先の ExecConditionalAssignProjectionInfo を参照してください。
  • ExecAssignScanProjectionInfoWithVarno という類似関数が存在するため、用途に応じて使い分けてください。
