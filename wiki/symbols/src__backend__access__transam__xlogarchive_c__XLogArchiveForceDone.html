XLogArchiveForceDone

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogarchive.c:509

目的

指定されたWALセグメントがアーカイブ済みであることを示す状態ファイル（.done）を強制的に作成し、アーカイブ処理の対象から除外する。

シグネチャまたは形状

void XLogArchiveForceDone(const char *xlog)

入力と出力

  • **入力**: xlog (WALセグメント名を示す文字列)
  • **出力**: なし（戻り値なし）。副作用として pg_wal/archive_status/ ディレクトリ内に .done ファイルを作成する。

前提条件と不変条件

  • xlog 引数は有効なWALセグメント名であること。
  • 呼び出し元は、対象のWALファイルが既にアーカイブ済みであるか、あるいはアーカイブをスキップすべきであると判断している必要がある。

アルゴリズムまたは振る舞い

1. 対象の .done ファイルが既に存在するか stat() で確認し、存在する場合は即座に終了する。 2. 対象の .ready ファイルが存在する場合、それを durable_rename() を用いて .done ファイルへリネームし、終了する。 3. 上記いずれも存在しない場合、空の .done ファイルを AllocateFile() で新規作成する。 4. ファイル作成または書き込みに失敗した場合は ereport(LOG, ...) でエラーを記録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogarchive.c (内部ロジック)
  • src/backend/replication/walreceiver.c (WAL受信プロセス)
  • src/backend/backup/basebackup.c (ベースバックアップ処理)
  • src/bin/pg_basebackup/receivelog.c (外部ツール)
  • **呼び出し先**:
  • StatusFilePath (パス構築)
  • stat (ファイル存在確認)
  • durable_rename (ファイルリネーム)
  • AllocateFile / FreeFile (ファイルI/O)
  • ereport (エラー通知)

関連データ構造

  • MAXPGPATH: ファイルパスの最大長を定義する定数。
  • struct stat: ファイル状態取得用構造体。

失敗モードと境界条件

  • .done ファイル作成時のディスクフルや権限不足が発生した場合、ereport によりログが出力されるが、関数自体は呼び出し元へ制御を戻す。
  • リネーム操作が失敗した場合、durable_rename の内部挙動に従い警告が記録される。

保守担当者向けメモ

  • 本関数は、アーカイブプロセスが処理を開始する前に、特定のWALファイルを「完了」と見なす必要がある場合に利用される。
  • Windows環境等でのファイルロック競合を避けるため、可能な限り durable_rename を優先する設計となっている。
