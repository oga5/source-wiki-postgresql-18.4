bbsink_begin_manifest

定義

  • 種別: function
  • 定義箇所: src/include/backup/basebackup_sink.h:224

目的

バックアップマニフェストの生成開始を、シンクチェーン内の現在のシンクに通知する。

シグネチャまたは形状

static inline void bbsink_begin_manifest(bbsink *sink)

入力と出力

  • **入力**: sink (bbsink *) - 操作対象となるシンク構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • sink は NULL であってはならない（Assert により検証）。
  • sink->bbs_ops が初期化されており、begin_manifest コールバック関数が実装されている必要がある。

アルゴリズムまたは振る舞い

1. 引数として渡された sink が NULL でないことを確認する。 2. sink の bbs_ops テーブルから begin_manifest コールバックを呼び出し、当該シンク固有のマニフェスト開始処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/backup_manifest.c:363
  • src/backend/backup/basebackup_sink.c:79（再帰的な転送処理として）
  • **呼び出し先**:
  • sink->bbs_ops->begin_manifest(sink)

関連データ構造

  • bbsink: シンクの基本構造体。
  • bbsink_ops: シンクの操作テーブル。begin_manifest 関数ポインタを保持する。

失敗モードと境界条件

  • sink が NULL の場合、アサーション違反によりプロセスが異常終了する。
  • sink->bbs_ops または begin_manifest 関数ポインタが NULL の場合、セグメンテーション違反が発生する。

保守担当者向けメモ

  • 本関数は static inline で定義されており、シンクチェーンの各段階で呼び出されることを想定している。
  • チェーンの次段へ処理を委譲する場合は、呼び出し元が sink->bbs_next を指定して本関数を再帰的に呼び出す必要がある。
