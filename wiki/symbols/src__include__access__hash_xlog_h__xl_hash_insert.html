xl_hash_insert

定義

  • 種別: struct
  • 定義箇所: src/include/access/hash_xlog.h:62

目的

ハッシュインデックスへのタプル挿入操作をWAL（Write-Ahead Logging）に記録するためのデータ構造です。リカバリ時に、どのオフセット位置に挿入が行われたかを再現するために使用されます。

シグネチャまたは形状

typedef struct xl_hash_insert
{
    OffsetNumber offnum;
} xl_hash_insert;

入力と出力

  • **入力**: 挿入対象のページ内オフセット番号（OffsetNumber）。
  • **出力**: WALレコードのペイロードとして書き込まれる構造体データ。

前提条件と不変条件

  • 本構造体は、ハッシュインデックスの挿入処理において、WALレコードのデータ部として格納される必要があります。
  • SizeOfHashInsert マクロ（offsetof(xl_hash_insert, offnum) + sizeof(OffsetNumber)）により、レコードサイズの計算が定義されています。

アルゴリズムまたは振る舞い

1. **書き込み**: _hash_doinsert 等の関数が、挿入先のオフセットを本構造体に設定し、XLogInsert を通じてWALへ書き込みます。 2. **リカバリ**: hash_redo 関数がWALレコードから本構造体を読み出し、指定されたオフセット情報を基にインデックスページの再構築を行います。

呼び出し元と呼び出し先

  • **呼び出し元（書き込み側）**: src/backend/access/hash/hashinsert.c 内の挿入処理。
  • **呼び出し先（リカバリ側）**: src/backend/access/hash/hash_xlog.c 内の hash_redo 関数。
  • **その他**: src/backend/access/rmgrdesc/hashdesc.c にて、ログ出力（デバッグ用）のために参照されます。

関連データ構造

  • OffsetNumber: インデックスページ内のタプル位置を示す型。
  • XLogRecord: 本構造体がペイロードとして格納されるWALレコードの基底構造。

失敗モードと境界条件

  • 構造体のサイズやオフセットの整合性が取れない場合、リカバリ時のデータ読み込み（XLogRecGetData）で不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は非常に単純ですが、ハッシュインデックスのリカバリロジックの根幹を成します。構造体メンバを変更する場合は、SizeOfHashInsert マクロおよび hash_redo でのキャスト処理が正しく追従していることを確認してください。
