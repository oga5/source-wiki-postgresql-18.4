FunctionRecheck

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeFunctionscan.c:248

目的

EvalPlanQual（EPQ）実行時にタプルの再チェックを行うためのアクセスメソッドルーチンです。

シグネチャまたは形状

static bool FunctionRecheck(FunctionScanState *node, TupleTableSlot *slot)

入力と出力

  • 入力:
  • node: FunctionScanState 構造体へのポインタ。
  • slot: 再チェック対象の TupleTableSlot。
  • 出力:
  • bool: 常に true を返します。

前提条件と不変条件

  • 本関数は FunctionScan ノードにおいて、再チェックの必要がないことを前提としています。

アルゴリズムまたは振る舞い

  • 内部処理は行わず、即座に true を返します。これは、FunctionScan が生成するタプルに対して追加の再検証が不要であることを示しています。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/nodeFunctionscan.c 内の ExecScanRecheckMtd を介して参照されています。
  • 呼び出し先: なし。

関連データ構造

  • FunctionScanState: FunctionScan ノードの実行時状態を保持する構造体。
  • TupleTableSlot: タプルを保持するためのスロット構造体。

失敗モードと境界条件

  • 特になし。常に成功（true）を返します。

保守担当者向けメモ

  • 本関数は ExecScanRecheckMtd インターフェースを満たすためのスタブとして機能しています。将来的に FunctionScan で再チェックが必要なロジックが追加されない限り、この実装を変更する必要はありません。
