print_lo_result

定義

  • 種別: function
  • 定義箇所: src/bin/psql/large_obj.c:18

目的

ラージオブジェクト操作の結果を、psql の設定に基づき標準出力（またはクエリ出力先）およびログファイルへ整形して出力する。

シグネチャまたは形状

static void print_lo_result(const char *fmt, ...);

入力と出力

  • **入力**:
  • fmt: 出力フォーマットを指定する書式文字列。
  • ...: 可変長引数。
  • **出力**: なし（void）。副作用として pset.queryFout および pset.logfile への書き込みを行う。

前提条件と不変条件

  • pset 構造体が初期化されていること。
  • pset.queryFout が有効な出力ストリームを指していること。

アルゴリズムまたは振る舞い

1. **標準出力への書き込み**: pset.quiet が偽の場合に実行される。

  • 出力形式が PRINT_HTML の場合、<p> タグで囲んで出力する。
  • vfprintf を使用して書式付き文字列を出力する。
  • 出力形式が PRINT_HTML なら </p>\n を、それ以外なら \n を末尾に付与する。

    • 2. **ログファイルへの書き込み**: pset.logfile が設定されている場合に実行される。

  • vfprintf を使用して書式付き文字列をログファイルへ出力し、末尾に改行を付与する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • do_lo_export (src/bin/psql/large_obj.c:164)
  • do_lo_import (src/bin/psql/large_obj.c:224)
  • do_lo_unlink (src/bin/psql/large_obj.c:261)
  • **呼び出し先**:
  • 標準ライブラリ: vfprintf, fputs, va_start, va_end。

関連データ構造

  • pset (グローバル設定構造体): 出力先 (queryFout)、ログファイル (logfile)、出力設定 (popt) を保持。

失敗モードと境界条件

  • pset.queryFout や pset.logfile が無効な場合、標準ライブラリの fputs/vfprintf の挙動に依存する。
  • pset.quiet が真の場合、標準出力への書き込みはスキップされる。

保守担当者向けメモ

  • 本関数は pg_attribute_printf(1, 2) 属性を持ち、コンパイル時に書式文字列のチェックが行われる（src/bin/psql/large_obj.c:16）。
  • HTML 出力モードの判定は pset.popt.topt.format に依存している。
