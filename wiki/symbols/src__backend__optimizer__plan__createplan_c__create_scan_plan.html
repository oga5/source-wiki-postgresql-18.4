create_scan_plan

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/createplan.c:558

目的

プランナが選択した Path ノードを、実行エンジンが処理可能な Plan ノード（スキャン系）へ変換する。

シグネチャまたは形状

Plan *create_scan_plan(PlannerInfo *root, Path *best_path, int flags)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • best_path: 変換対象の Path 構造体。
  • flags: ターゲットリスト生成を制御するビットフラグ（CP_IGNORE_TLIST, CP_LABEL_TLIST 等）。
  • **出力**:
  • 変換された Plan 構造体へのポインタ。

前提条件と不変条件

  • best_path は有効な Path 構造体であること。
  • 結合リレーション（IS_JOIN_REL）の場合、best_path は T_ForeignScan または T_CustomScan である必要がある。
  • baserestrictinfo リスト自体は変更してはならない。

アルゴリズムまたは振る舞い

1. **制約の抽出**: best_path->pathtype に応じて、適用すべき制限句（scan_clauses）を決定する。インデックススキャンの場合はインデックス固有の制約を使用する。 2. **パラメータ化の処理**: パラメータ化されたスキャンの場合、ppi_clauses を追加する。 3. **ゲートノードの判定**: get_gating_quals を呼び出し、定数評価が必要な制約（gating_clauses）があるか確認する。存在する場合、flags をリセットする。 4. **ターゲットリスト（tlist）の構築**:

  • CP_IGNORE_TLIST が指定されていれば tlist は NULL。
  • use_physical_tlist が真であれば物理的なタプル構造を優先し、必要に応じて build_physical_tlist を使用。
  • それ以外は build_path_tlist を使用。

    • 5. **ノード生成**: pathtype に基づく switch 文により、create_seqscan_plan や create_indexscan_plan などの専用関数を呼び出し、Plan ノードを生成する。

呼び出し元と呼び出し先

  • **呼び出し元**: create_plan_recurse（src/backend/optimizer/plan/createplan.c 内の推論される呼び出し元）。
  • **呼び出し先**:
  • get_gating_quals, use_physical_tlist, build_physical_tlist, build_path_tlist
  • 各種 create_xxxscan_plan 関数（create_seqscan_plan, create_indexscan_plan 等）。

関連データ構造

  • PlannerInfo: プランナの全体状態。
  • Path: 変換元の最適化パス。
  • Plan: 変換後の実行計画ノード。
  • RelOptInfo: リレーションの最適化情報。

失敗モードと境界条件

  • build_physical_tlist が NIL を返す場合（削除された列が存在する場合など）、通常の build_path_tlist にフォールバックする。
  • gating_clauses が存在する場合、flags が強制的にクリアされる。

保守担当者向けメモ

  • 本関数はスキャン系ノードのディスパッチャとして機能する。新しいスキャンタイプを追加する場合は、switch 文にケースを追加する必要がある。
  • ターゲットリストの最適化（use_physical_tlist）は、実行時の射影コストを削減するための重要な最適化である。
