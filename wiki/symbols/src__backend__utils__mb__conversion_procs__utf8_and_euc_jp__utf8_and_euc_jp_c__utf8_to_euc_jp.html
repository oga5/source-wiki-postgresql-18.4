utf8_to_euc_jp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_euc_jp/utf8_and_euc_jp.c:62

目的

UTF-8エンコーディングからEUC_JPエンコーディングへの文字セット変換を実行するPostgreSQLのロード可能な変換プロシージャです。

シグネチャまたは形状

Datum utf8_to_euc_jp(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を受け取ります。
  • src (第3引数): 変換元UTF-8文字列。
  • dest (第4引数): 変換先EUC_JP文字列の格納バッファ。
  • len (第5引数): 変換対象のバイト長。
  • noError (第6引数): 変換エラー時の挙動を制御するフラグ。
  • **出力**: 変換されたバイト数を Datum 型（int32）として返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_UTF8, PG_EUC_JP) マクロにより、変換元がUTF-8、変換先がEUC_JPであることが検証されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得します。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングIDの整合性を確認します。 3. UtfToLocal 関数を呼び出し、変換を実行します。この際、euc_jp_from_unicode_tree を変換テーブルとして使用します。 4. 変換結果のバイト数を PG_RETURN_INT32 で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（PostgreSQLの関数管理システム経由）。
  • **呼び出し先**: UtfToLocal（変換ユーティリティ関数）。

関連データ構造

  • euc_jp_from_unicode_tree: UTF-8からEUC_JPへの変換に使用される変換テーブル構造体。

失敗モードと境界条件

  • noError 引数が false の場合、変換不可能な文字が含まれるとエラーを発生させます。
  • 変換元・変換先のエンコーディングIDが不一致の場合、CHECK_ENCODING_CONVERSION_ARGS により処理が中断されます。

保守担当者向けメモ

  • 本関数は変換ロジックの「接着剤」であり、実際の変換テーブルは ../../Unicode/utf8_to_euc_jp.map から生成されます。
  • 変換ロジックの詳細は src/backend/utils/mb/wchar.c の UtfToLocal を参照してください。
