WalSummarizerData

定義

  • **種別**: struct
  • **定義箇所**: src/backend/postmaster/walsummarizer.c:57

目的

WAL Summarizer プロセスの状態を共有メモリ上で管理し、要約の進捗状況やプロセス情報を他のプロセスへ公開するために使用される制御構造体です。

シグネチャまたは形状

typedef struct {
    bool           initialized;
    TimeLineID     summarized_tli;
    XLogRecPtr     summarized_lsn;
    bool           lsn_is_exact;
    ProcNumber     summarizer_pgprocno;
    XLogRecPtr     pending_lsn;
    ConditionVariable summary_file_cv;
} WalSummarizerData;

入力と出力

  • **入力**: プロセス実行中に summarized_lsn や pending_lsn が更新され、共有メモリを通じて状態が伝播されます。
  • **出力**: summary_file_cv を通じて、要約の完了を待機しているプロセスへ通知が行われます。

前提条件と不変条件

  • initialized が false の間、他のフィールドは無効な値を含みます。
  • WALSummarizerLock により、summary_file_cv を除く全フィールドの同期が保護されます。
  • pending_lsn は常に summarized_lsn 以上である必要があります。

アルゴリズムまたは振る舞い

  • **初期化**: ディスク上の既存サマリファイルを走査し、情報を共有メモリに格納した後に initialized が true に設定されます。
  • **要約位置の管理**: summarized_lsn と summarized_tli は、次に作成されるサマリファイルの開始位置を示します。lsn_is_exact は、その位置が正確な境界か、推定値（WAL ファイルの先頭など）かを示します。
  • **進捗広告**: pending_lsn は、要約プロセスが読み込んだ最新の WAL レコードの終了位置を外部に通知するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: WalSummarizerShmemSize（サイズ計算）、WalSummarizerShmemInit（初期化）、および walsummarizer.c 内の各プロセス制御関数から参照されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ConditionVariable: summary_file_cv フィールドで使用され、同期処理を担います。
  • ProcNumber: プロセス識別に使用されます。
  • XLogRecPtr: WAL の位置指定に使用されます。

失敗モードと境界条件

保守担当者向けメモ

  • summary_file_cv は他のフィールドと異なり、独自の同期メカニズムで管理されています。
  • summarizer_pgprocno が INVALID_PROC_NUMBER の場合、プロセスが実行されていないことを示します。
