having_collation_conflict_walker

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/planner.c:1507

目的

式ツリーを走査し、特定の照合順序（Collation）の競合が発生していないかを検出する。特に、決定論的でない照合順序を持つ Var ノードが、期待される照合順序と一致しない文脈で使用されているかを判定する。

シグネチャまたは形状

static bool having_collation_conflict_walker(Node *node, having_collation_ctx *ctx)

入力と出力

  • **入力**:
  • node: 走査対象の式ツリーノード。
  • ctx: 照合順序の文脈を保持する having_collation_ctx 構造体。
  • **出力**:
  • 競合が検出された場合は true、それ以外は false を返す。

前提条件と不変条件

  • node が Var 型の場合、varlevelsup == 0 であること（上位レベルの変数は対象外）。
  • ctx->ancestor_collids は、走査中のパスにおける照合順序のスタックとして機能する。

アルゴリズムまたは振る舞い

1. **Var ノード**: ctx->group_rtindex と一致し、かつ非決定論的な照合順序を持つ場合、ctx->ancestor_collids 内の既存の照合順序と比較し、不一致があれば競合とみなす。 2. **RowCompareExpr**: 各列の照合順序を ctx->ancestor_collids に一時的にプッシュし、再帰的に走査する。 3. **CaseExpr**: arg の走査前に各 WHEN 句の照合順序をスタックにプッシュし、走査後にリストアする。その後、各 WHEN 句の式と結果、および defresult を走査する。 4. **その他**: exprInputCollation で得られる照合順序をスタックにプッシュし、expression_tree_walker を用いて子ノードを再帰的に走査する。

呼び出し元と呼び出し先

  • **呼び出し元**: find_having_collation_conflicts および自身（再帰呼び出し）。
  • **呼び出し先**: expression_tree_walker、および自身（再帰呼び出し）。

関連データ構造

  • having_collation_ctx: 照合順序の競合判定に必要な状態（group_rtindex や ancestor_collids）を保持する。

失敗モードと境界条件

  • node が NULL の場合は直ちに false を返す。
  • OidIsValid で照合順序の有効性を確認し、無効な場合はスタック操作を行わない。

保守担当者向けメモ

  • 本関数は expression_tree_walker を介して式ツリー全体を網羅的に走査する。新しいノードタイプを追加する際は、照合順序の伝播が適切に処理されるか確認が必要。
