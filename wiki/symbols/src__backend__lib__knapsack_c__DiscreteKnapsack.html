DiscreteKnapsack

定義

  • 種別: function
  • 定義箇所: src/backend/lib/knapsack.c:51

目的

0/1 ナップサック問題を解き、指定された重量制限内で価値の合計が最大となるアイテムの組み合わせを算出する。

シグネチャまたは形状

Bitmapset *DiscreteKnapsack(int max_weight, int num_items, int *item_weights, double *item_values)

入力と出力

  • **入力**:
  • max_weight: 許容される最大重量（非負）。
  • num_items: アイテムの総数。
  • item_weights: 各アイテムの重量を格納した配列。
  • item_values: 各アイテムの価値を格納した配列（NULL の場合は全アイテムの価値を 1 とみなす）。
  • **出力**:
  • 選択されたアイテムのインデックスを保持する Bitmapset へのポインタ。

前提条件と不変条件

  • max_weight >= 0 であること。
  • num_items > 0 かつ item_weights が有効なポインタであること。
  • 内部で一時的な MemoryContext を作成するため、呼び出し元は返却された Bitmapset が現在のメモリコンテキストにコピーされたものであることを前提とする。

アルゴリズムまたは振る舞い

1. 計算専用の MemoryContext を作成し、動的計画法用の配列 values（最大価値）と sets（選択済みアイテム集合）を確保する。 2. 各アイテムについて、max_weight から item_weights[i] まで逆順にループし、価値の合計が最大となる組み合わせを更新する。 3. 価値の更新条件 values[j] <= values[ow] + iv を満たす場合、sets[j] を更新し、アイテム i を追加する。 4. 計算終了後、結果の Bitmapset から計算用ダミービットを除去し、呼び出し元のコンテキストへコピーして返す。 5. 一時的な MemoryContext を削除し、計算用メモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planner.c:4687 での呼び出しが確認されている。
  • **呼び出し先**: bms_make_singleton, bms_replace_members, bms_add_member, bms_copy, bms_del_member などの Bitmapset 操作関数、および MemoryContext 関連関数。

関連データ構造

  • Bitmapset: 選択されたアイテムのインデックス管理に使用。
  • MemoryContext: 計算中のメモリ管理に使用。

失敗モードと境界条件

  • max_weight が非常に大きい場合、計算量 $O(nW)$ およびメモリ消費が増大する。
  • 重量が 0 のアイテムは、常に結果の Bitmapset に含まれる。

保守担当者向けメモ

  • 本関数は内部で MemoryContext を切り替えるため、呼び出し側は結果の Bitmapset が palloc されたメモリ領域にあることを意識する必要がある。
  • 非整数値や大きな重みを扱う場合は、事前に値をスケーリングして整数範囲に収めることが推奨される。
