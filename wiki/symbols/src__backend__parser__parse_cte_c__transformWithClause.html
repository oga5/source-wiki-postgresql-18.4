transformWithClause

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_cte.c:109

目的

WITH 句（CTE）の構文解析および意味解析を行い、各CTEを Query ノードへ変換して名前空間（p_ctenamespace）を構築します。

シグネチャまたは形状

List *transformWithClause(ParseState *pstate, WithClause *withClause)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態を保持する ParseState 構造体。
  • withClause: 解析対象の WITH 句構造体。
  • **出力**:
  • List *: 解析済みのCTEを含む名前空間リスト。

前提条件と不変条件

  • pstate->p_ctenamespace および pstate->p_future_ctes は、呼び出し時に NIL である必要があります。
  • WITH 句内の各CTE名は一意である必要があります。

アルゴリズムまたは振る舞い

1. **初期検証**: CTE名の重複チェックを行い、各CTEの初期状態（cterecursive = false, cterefcount = 0）を設定します。データ変更ステートメントが含まれる場合は pstate->p_hasModifyingCTE をセットします。 2. **再帰的CTE (WITH RECURSIVE) の処理**:

  • CteState を初期化し、依存関係グラフを構築します (makeDependencyGraph)。
  • 再帰構造の妥当性を検証します (checkWellFormedRecursion)。
  • 依存関係に基づきトポロジカルソートされた順序で、名前空間への追加と analyzeCTE による解析を実行します。

    • 3. **非再帰的CTEの処理**:

  • p_future_ctes にCTEリストを保持し、順次 analyzeCTE で解析して名前空間に追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/analyze.c (複数箇所)
  • src/backend/parser/parse_merge.c:131
  • **呼び出し先**:
  • makeDependencyGraph
  • checkWellFormedRecursion
  • analyzeCTE

関連データ構造

  • WithClause: 解析対象の構文木。
  • CommonTableExpr: CTEの定義情報。
  • CteState: 再帰解析時の依存関係や順序を保持する一時構造体。
  • ParseState: 解析状態を管理するメイン構造体。

失敗モードと境界条件

  • **重複名**: 同一の WITH 句内でCTE名が重複している場合、ERRCODE_DUPLICATE_ALIAS エラーを発生させます。
  • **再帰の不正**: checkWellFormedRecursion により、SQL標準に違反する再帰構造が検出された場合にエラーとなります。

保守担当者向けメモ

  • 本関数は WITH 句の再帰・非再帰の分岐点です。再帰的CTEの解析順序は makeDependencyGraph に依存するため、依存関係解決ロジックを変更する際は注意してください。
  • p_ctenamespace の構築順序は、プランナがクエリを処理する順序に直結します。
