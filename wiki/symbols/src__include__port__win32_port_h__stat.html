stat

定義

  • 種別: struct
  • 定義箇所: src/include/port/win32_port.h:254

目的

Windows 環境（MSVC 等）における標準的な stat 構造体の 32 ビット制限を回避し、64 ビット対応の __stat64 と互換性のあるファイル属性情報を保持するために定義されています。

シグネチャまたは形状

struct stat
{
    _dev_t      st_dev;
    _ino_t      st_ino;
    unsigned short st_mode;
    short       st_nlink;
    short       st_uid;
    short       st_gid;
    _dev_t      st_rdev;
    __int64     st_size;
    __time64_t  st_atime;
    __time64_t  st_mtime;
    __time64_t  st_ctime;
};

入力と出力

  • 本シンボルはデータ構造であり、関数ではありません。
  • stat() 関数（src/port/win32stat.c に実装）の呼び出しを通じて、ファイルシステムから取得された情報がこの構造体に格納されます。

前提条件と不変条件

  • struct __stat64 とメモリレイアウトが一致している必要があります。
  • Windows ネイティブビルドまたは MinGW ビルド環境でのみ使用されます（Cygwin では使用されません）。

アルゴリズムまたは振る舞い

  • 本構造体は、POSIX 準拠の stat 構造体を Windows 上でエミュレートするための型定義として機能します。
  • コンパイル時に sys/stat.h の標準定義を上書きまたは補完することで、PostgreSQL のコアコードがプラットフォームの差異を意識せずにファイル情報を扱えるようにします。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • contrib/basic_archive/basic_archive.c
  • contrib/file_fdw/file_fdw.c
  • その他、ファイルシステム情報を取得する PostgreSQL の各モジュール。
  • **呼び出し先**:
  • 本構造体はデータ型であるため、直接的な呼び出し先はありません。ただし、この構造体を利用する stat() 関数は src/port/win32stat.c で実装されています。

関連データ構造

  • _dev_t, _ino_t, __int64, __time64_t: Windows 環境で定義される基底型。

失敗モードと境界条件

  • 32 ビット環境の古いコンパイラやライブラリとの混在により、構造体のサイズが期待と異なる場合にデータ破損の可能性があります。
  • stat() 関数が失敗した場合、構造体の中身は不定となります。

保守担当者向けメモ

  • windows.h や sys/stat.h とのインクルード順序が厳格です。修正時は win32_port.h 内の既存の定義順序を維持してください。
  • struct __stat64 との整合性が崩れると、ファイルサイズの取得などで深刻なバグを引き起こすため、構造体のメンバ変更には注意が必要です。
