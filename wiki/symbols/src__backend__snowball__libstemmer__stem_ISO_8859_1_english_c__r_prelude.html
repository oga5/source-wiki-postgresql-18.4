r_prelude

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_english.c:361

目的

ステミング処理の開始時に、入力文字列に対して特定の文字置換や削除を行い、アルゴリズムの正規化を行う。

シグネチャまたは形状

static int r_prelude(struct SN_env * z)

入力と出力

  • 入力: SN_env 構造体（ステミング環境）。
  • 出力: 成功時に 1、エラー発生時に負の値を返す。
  • 副作用: z->I[2]（内部フラグ）の更新、および z->p が指す文字列の書き換え。

前提条件と不変条件

  • SN_env 構造体が初期化済みであること。
  • header.h で定義された SN_env の各フィールド（c, l, bra, ket 等）が有効な状態であること。

アルゴリズムまたは振る舞い

1. **アポストロフィの削除**: 文字列の先頭が ' であれば削除する。 2. **'y' の正規化**:

  • 先頭の 'y' を 'Y' に置換し、フラグ z->I[2] を 1 に設定する。
  • 母音（g_v グループ）の直後にある 'y' を 'Y' に置換し、フラグ z->I[2] を 1 に設定する。

    • 3. 処理完了後、成功値 1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: english_ISO_8859_1_stem（src/backend/snowball/libstemmer/stem_ISO_8859_1_english.c 内）から呼び出される。
  • 呼び出し先: slice_del, slice_from_s, in_grouping（いずれも libstemmer のランタイム関数）。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • g_v: 母音を定義するグループ（in_grouping で使用）。

失敗モードと境界条件

  • slice_del または slice_from_s が失敗した場合、負の戻り値をそのまま伝播させる。
  • 文字列が空の場合や、検索対象の文字が見つからない場合は、該当する処理をスキップして正常終了する。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードである。ロジックの変更が必要な場合は、直接編集せず元の .sbl スクリプトを修正すること。
  • z->I[2] は、後続の r_postlude で 'Y' を 'y' に戻すかどうかの判定に使用される。
