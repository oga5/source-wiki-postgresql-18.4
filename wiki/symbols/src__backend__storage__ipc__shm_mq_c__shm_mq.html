shm_mq

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/ipc/shm_mq.c:71

目的

PostgreSQL のバックエンドプロセス間でメッセージをやり取りするための、単一リーダー・単一ライターの共有メモリメッセージキューを定義します。

シグネチャまたは形状

struct shm_mq
{
    slock_t        mq_mutex;
    PGPROC        *mq_receiver;
    PGPROC        *mq_sender;
    pg_atomic_uint64 mq_bytes_read;
    pg_atomic_uint64 mq_bytes_written;
    Size           mq_ring_size;
    bool           mq_detached;
    uint8          mq_ring_offset;
    char           mq_ring[FLEXIBLE_ARRAY_MEMBER];
};

入力と出力

  • **入力**: 共有メモリ上に配置されるデータ構造であり、mq_ring を通じてメッセージの送受信が行われます。
  • **出力**: mq_bytes_read および mq_bytes_written のアトミックな更新により、キューの状態が同期されます。

前提条件と不変条件

  • 単一リーダー・単一ライターの構成であること。
  • mq_bytes_read と mq_bytes_written はアトミック操作とメモリバリアによって保護されており、スピンロック（mq_mutex）の使用を最小限に抑える設計であること。
  • 相手プロセスが終了した場合は mq_detached フラグにより検知可能であること。

アルゴリズムまたは振る舞い

  • **リングバッファ管理**: mq_ring を使用したリングバッファ構造を共有メモリ上で管理します。
  • **同期**: PGPROC を介して Latch を利用し、バッファが満杯または空の際にプロセスをブロッキング/ウェイクアップします。
  • **効率化**: 頻繁な通知を避けるため、一定量（リングサイズの1/4）のデータが溜まるまでポインタ更新を遅延させる仕組みを持ちます。
  • **終了処理**: mq_detached フラグを立てることで、相手側に切断を通知します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/transam/parallel.c
  • src/backend/executor/execParallel.c
  • src/backend/executor/tqueue.c
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、操作には shm_mq_send や shm_mq_receive 等の関数が使用されます。

関連データ構造

  • shm_mq_handle: 各バックエンドが保持するキューへのアクセス用プライベートハンドル。
  • PGPROC: プロセス間同期のためのプロセス情報。

失敗モードと境界条件

  • **バッファフル/エンプティ**: 送信時にバッファが満杯、または受信時に空の場合、WaitLatch による待機が発生します（nowait = true の場合は SHM_MQ_WOULD_BLOCK を返します）。
  • **予期せぬ終了**: 相手プロセスが終了した場合、BackgroundWorkerHandle を介して検知し、デッドロックを回避します。

保守担当者向けメモ

  • mq_bytes_read や mq_bytes_written の操作にはアトミック操作が必須です。
  • mq_ring は FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズは動的に決定されます。
