RI_FKey_check_upd

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ri_triggers.c:489

目的

外部キー制約に関連する UPDATE 操作が発生した際、制約違反がないかを検証するトリガープロシージャとして機能します。

シグネチャまたは形状

Datum RI_FKey_check_upd(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（トリガー実行コンテキストを含む）。
  • **出力**: Datum 型（通常はトリガーの戻り値として PointerGetDatum(NULL) を返します）。

前提条件と不変条件

  • 本関数は、UPDATE イベントに対して設定されたトリガーから呼び出される必要があります。
  • 内部で ri_CheckTrigger を呼び出し、トリガーが正しいタイミングとイベント（RI_TRIGTYPE_UPDATE）で実行されているかを検証します。

アルゴリズムまたは振る舞い

1. ri_CheckTrigger を呼び出し、トリガーの実行コンテキストが適切かを確認します。 2. INSERT 操作用の検証ロジックである RI_FKey_check を呼び出し、更新後の値が外部キー制約を満たしているかを検証します。 3. RI_FKey_check の戻り値をそのまま返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c: 外部キー制約のトリガー定義時に参照されます。
  • **呼び出し先**:
  • ri_CheckTrigger: トリガーの実行環境の妥当性チェック。
  • RI_FKey_check: 実際の制約検証ロジック。

関連データ構造

  • TriggerData: fcinfo->context からキャストして取得される、トリガー実行時のメタデータ構造体。

失敗モードと境界条件

  • ri_CheckTrigger による検証で、トリガーの呼び出しタイミングやイベントが不適切と判断された場合、エラーが発生します。
  • RI_FKey_check 内部で制約違反が検出された場合、エラーが報告されます。

保守担当者向けメモ

  • 本関数は RI_FKey_check のラッパーとして実装されており、ロジックの大部分を共有しています。修正を行う際は、INSERT と UPDATE の両方のケースに影響が出る点に留意してください。
