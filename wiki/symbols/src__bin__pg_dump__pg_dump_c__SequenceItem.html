SequenceItem

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_dump.c:126

目的

データベースのシーケンスオブジェクトに関するメタデータおよび状態情報を保持し、pg_dump 実行中にシーケンスの定義と現在の値をエクスポートするために使用される。

シグネチャまたは形状

typedef struct {
    Oid         oid;            /* シーケンスのOID */
    SeqType     seqtype;        /* シーケンスのデータ型 */
    bool        cycled;         /* サイクルするかどうか */
    int64       minv;           /* 最小値 */
    int64       maxv;           /* 最大値 */
    int64       startv;         /* 開始値 */
    int64       incby;          /* 増分値 */
    int64       cache;          /* キャッシュサイズ */
    int64       last_value;     /* シーケンスの最終値 */
    bool        is_called;      /* nextvalが値を返す前に進むかどうか */
    bool        null_seqtuple;  /* pg_get_sequence_dataがNULLを返したか */
} SequenceItem;

入力と出力

本構造体は、システムカタログから取得したシーケンス情報を格納するコンテナとして機能する。出力として、ダンプ処理時にシーケンスの再構築に必要なパラメータを提供する。

前提条件と不変条件

  • pg_dump の実行中、シーケンスの整合性を保つために last_value や is_called などの状態が適切に保持される必要がある。
  • null_seqtuple は、シーケンスデータ取得時の異常系を判定するフラグとして機能する。

アルゴリズムまたは振る舞い

  • pg_dump 内でシーケンス情報の配列として管理され、bsearch を用いた検索や、メモリ確保（pg_malloc / pg_malloc0）による動的なリスト管理が行われる。
  • SequenceItemCmp（比較関数）を介して、シーケンスのソートや検索に使用される。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/pg_dump.c 内のシーケンス収集およびソート処理。
  • 呼び出し先: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行わない。

関連データ構造

  • SeqType: シーケンスのデータ型を定義する列挙型。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • pg_get_sequence_data が NULL を返した場合、null_seqtuple フラグによりその状態が記録される。
  • メモリ不足時には pg_malloc 等のメモリ確保関数が失敗する可能性がある。

保守担当者向けメモ

  • 本構造体は pg_dump.c 内で静的配列 sequences として管理されている。
  • 構造体のフィールド追加や変更を行う際は、SequenceItemCmp などの比較ロジックや、pg_malloc を使用している箇所でのサイズ計算に影響がないか確認すること。
