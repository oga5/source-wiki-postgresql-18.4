get_relation_statistics_worker

定義

  • **種別**: function
  • **定義箇所**: src/backend/optimizer/util/plancat.c:1420

目的

指定された拡張統計情報（pg_statistic_ext_data）に基づき、構築済みの統計種別を判定し、対応する StatisticExtInfo ノードを生成してリストに追加する。

シグネチャまたは形状

static void get_relation_statistics_worker(List **stainfos, RelOptInfo *rel, Oid statOid, bool inh, Bitmapset *keys, List *exprs)

入力と出力

  • **入力**:
  • stainfos: StatisticExtInfo ノードを格納するリストへのポインタ。
  • rel: 関連するリレーションの RelOptInfo。
  • statOid: 統計情報の OID。
  • inh: 継承された統計情報を含めるかどうかを示すフラグ。
  • keys: 統計対象の属性キーセット。
  • exprs: 統計対象の式リスト。
  • **出力**: なし（stainfos リストを直接更新）。

前提条件と不変条件

  • statOid に対応する pg_statistic_ext_data エントリがシステムキャッシュに存在すること。
  • RelOptInfo は有効な状態であること。

アルゴリズムまたは振る舞い

1. STATEXTDATASTXOID システムキャッシュから statOid と inh をキーに pg_statistic_ext_data タプルを検索する。 2. タプルが見つからない場合は即座に終了する。 3. statext_is_kind_built を使用して、以下の統計種別が構築済みかを確認する。

  • STATS_EXT_NDISTINCT (NDistinct)
  • STATS_EXT_DEPENDENCIES (関数従属性)
  • STATS_EXT_MCV (最頻値)
  • STATS_EXT_EXPRESSIONS (式統計)

    • 4. 構築済みの各種別に対し、StatisticExtInfo ノードを新規作成し、入力された rel, keys, exprs 等の情報をコピーして stainfos リストに追加する。 5. 最後に ReleaseSysCache を呼び出し、取得したタプルを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/plancat.c 内の get_relation_statistics 関数から、継承の有無（true/false）を切り替えて2回呼び出される。
  • **呼び出し先**: SearchSysCache2, GETSTRUCT, statext_is_kind_built, makeNode, bms_copy, lappend, ReleaseSysCache。

関連データ構造

  • StatisticExtInfo: 拡張統計情報を保持するプランナ用ノード。
  • Form_pg_statistic_ext_data: システムカタログ pg_statistic_ext_data の構造体。
  • RelOptInfo: リレーションの最適化情報を保持する構造体。

失敗モードと境界条件

  • 指定された statOid に対応するタプルがシステムキャッシュに存在しない場合、何も行わずに正常終了する。
  • 構築済みの統計種別が一つも存在しない場合、リストへの追加は行われない。

保守担当者向けメモ

  • 本関数は get_relation_statistics の内部的なヘルパーとして機能する。
  • 新しい統計種別が PostgreSQL に追加された場合、本関数の判定ロジックにも追加が必要となる可能性がある。
