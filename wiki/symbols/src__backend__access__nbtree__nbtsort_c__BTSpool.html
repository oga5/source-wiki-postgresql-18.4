BTSpool

定義

  • 種別: struct
  • 定義箇所: src/backend/access/nbtree/nbtsort.c:79

目的

B-Treeインデックス構築プロセスにおいて、ヒープから読み込まれたタプルをソートし、一時的に保持するための状態管理構造体です。

シグネチャまたは形状

typedef struct BTSpool
{
    Tuplesortstate *sortstate;    /* tuplesort.c用状態データ */
    Relation    heap;             /* ソースとなるヒープリレーション */
    Relation    index;            /* 構築対象のインデックスリレーション */
    bool        isunique;         /* 一意性制約の有無 */
    bool        nulls_not_distinct; /* NULL値の区別に関するフラグ */
} BTSpool;

入力と出力

  • **入力**: palloc0 を介してメモリ確保され、各フィールドが初期化されます。
  • **出力**: _bt_spooldestroy を通じて、保持している Tuplesortstate などのリソースが解放されます。

前提条件と不変条件

  • sortstate は tuplesort.c の機能を利用するため、適切に初期化されている必要があります。
  • インデックス構築のライフサイクルにおいて、BTSpool はソートフェーズからリーフページ構築フェーズまでの間、有効な状態を維持しなければなりません。

アルゴリズムまたは振る舞い

1. **初期化**: palloc0 によりメモリが割り当てられ、インデックス構築に必要なリレーション情報や制約フラグが設定されます。 2. **ソート処理**: sortstate を通じてヒープから読み込まれたタプルがソートされます。 3. **構築への供給**: ソート済みタプルは _bt_leafbuild 等の関数に渡され、インデックスのリーフページへ書き出されます。 4. **破棄**: 構築完了後、_bt_spooldestroy により関連リソースがクリーンアップされます。

呼び出し元と呼び出し先

  • **呼び出し元**: nbtsort.c 内のインデックス構築ロジック（btbuild 等）から動的に生成されます。
  • **呼び出し先**: tuplesort.c の各関数（sortstate 経由）、および _bt_spooldestroy、_bt_spool、_bt_leafbuild、_bt_parallel_scan_and_sort などの nbtsort.c 内の関数群。

関連データ構造

  • Tuplesortstate: ソートアルゴリズムの実行状態を保持。
  • Relation: PostgreSQLのテーブルおよびインデックスのメタデータ構造。

失敗モードと境界条件

  • メモリ不足: palloc0 によるメモリ確保失敗時にエラーが発生する可能性があります。
  • ソート失敗: tuplesort 内部でのエラー（ディスク容量不足など）が sortstate を通じて伝播する可能性があります。

保守担当者向けメモ

  • 本構造体は並列インデックス構築時にも使用されます。並列ワーカープロセスが必要とする状態を保持するため、並列実行時の整合性に注意してください。
  • isunique や nulls_not_distinct はインデックスの制約チェックに直結するため、変更時は nbtree の挿入ロジックとの整合性を確認してください。
