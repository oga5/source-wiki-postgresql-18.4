gbt_int2gt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int2.c:29

目的

int2（2バイト整数）型の値同士を比較し、第一引数が第二引数より大きいかどうかを判定する。

シグネチャまたは形状

bool gbt_int2gt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の int16 値へのポインタ。
  • b: 比較対象の int16 値へのポインタ。
  • flinfo: 関数呼び出し情報（本関数内では未使用）。
  • **出力**:
  • bool: *a > *b であれば true、そうでなければ false を返す。

前提条件と不変条件

  • a および b は、有効な int16 型の値を指している必要がある。
  • flinfo は NULL でも動作する（関数内で参照されないため）。

アルゴリズムまたは振る舞い

1. a が指すメモリ領域を int16 型としてキャストし、値を読み取る。 2. b が指すメモリ領域を int16 型としてキャストし、値を読み取る。 3. C 言語の標準的な比較演算子 > を用いて大小比較を行い、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_int2.c 内の静的構造体 tinfo の初期化において参照されている。
  • **呼び出し先**: 本関数は直接的な関数呼び出しを行わない。

関連データ構造

  • int16: PostgreSQL で定義される 2 バイト整数型。

失敗モードと境界条件

  • a または b が NULL ポインタである場合、セグメンテーション違反が発生する可能性がある。
  • 整数値の範囲外（オーバーフロー等）については、C 言語の int16 型の仕様に従う。

保守担当者向けメモ

  • 本関数は btree_utils_num.c が提供する汎用的な GiST 数値処理ロジックから呼び出されるためのラッパーである。
  • 比較ロジックを変更する場合は、同様のパターンで実装されている gbt_int2lt や gbt_int2eq との整合性に注意すること。
