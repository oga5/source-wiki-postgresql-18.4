ExecDeleteEpilogue

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeModifyTable.c:1513

目的

DELETE 操作の完了処理（エピローグ）を実行します。主に AFTER ROW DELETE トリガーの起動と、パーティション更新に伴う遷移状態の管理を担います。

シグネチャまたは形状

static void ExecDeleteEpilogue(ModifyTableContext *context, ResultRelInfo *resultRelInfo, ItemPointer tupleid, HeapTuple oldtuple, bool changingPart);

入力と出力

  • context: 実行コンテキスト（ModifyTableState を含む）。
  • resultRelInfo: ターゲットリレーションの情報。
  • tupleid: 削除されたタプルの物理位置（ItemPointer）。
  • oldtuple: 削除されたタプルのデータ。
  • changingPart: パーティション移動を伴う操作かどうかを示すフラグ。

前提条件と不変条件

  • context および resultRelInfo は有効なポインタである必要があります。
  • 呼び出し時点で、対象タプルの物理的な削除処理は完了している必要があります。

アルゴリズムまたは振る舞い

1. **パーティション更新の判定**: 操作が CMD_UPDATE であり、かつ mt_transition_capture が有効な場合、パーティション移動に伴う「OLD TABLE」への遷移情報を処理します。この場合、ExecARUpdateTriggers を呼び出し、後続の DELETE トリガーが重複して遷移情報をキャプチャしないようフラグを調整します。 2. **トリガー実行**: ExecARDeleteTriggers を呼び出し、AFTER ROW DELETE トリガーを起動します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeModifyTable.c 内の ExecDelete 関数から呼び出されます。
  • **呼び出し先**:
  • ExecARUpdateTriggers（条件付き）
  • ExecARDeleteTriggers

関連データ構造

  • ModifyTableContext: 実行状態を保持する構造体。
  • ResultRelInfo: ターゲットテーブルのメタデータ。
  • TransitionCaptureState: トリガー遷移状態を管理する構造体。

失敗モードと境界条件

  • トリガー実行中にエラーが発生した場合、PostgreSQL の標準的なエラー処理メカニズムによりトランザクションがアボートされます。
  • mt_transition_capture が NULL の場合、遷移情報のキャプチャはスキップされます。

保守担当者向けメモ

  • 本関数は ExecDelete の末尾で実行されるクリーンアップ処理です。トリガーの実行順序や、パーティション更新時の遷移テーブルへの書き込みロジックを変更する場合は、ExecARUpdateTriggers との整合性に注意してください。
