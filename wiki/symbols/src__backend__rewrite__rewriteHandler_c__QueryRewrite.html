QueryRewrite

定義

  • **種別**: function
  • **定義箇所**: src/backend/rewrite/rewriteHandler.c:4634

目的

パーサが生成したトップレベルのクエリツリーに対し、ルール適用、RIR（Retrieve-Instead-Retrieve）ルールの展開、およびコマンド完了タグの決定を行い、実行可能なクエリツリーのリストを生成する。

シグネチャまたは形状

List *QueryRewrite(Query *parsetree)

入力と出力

  • **入力**: parsetree (解析済みの Query 構造体へのポインタ)
  • **出力**: 書き換えられた Query 構造体のリスト (List *)

前提条件と不変条件

  • 入力される parsetree はトップレベルのオリジナルクエリであること (parsetree->querySource == QSRC_ORIGINAL)。
  • parsetree->canSetTag が真であること。
  • 処理後のリストにおいて、canSetTag が真であるクエリは最大で1つであること。

アルゴリズムまたは振る舞い

1. **非SELECTルールの適用**: RewriteQuery を呼び出し、INSERT/UPDATE/DELETE等のルールを適用してクエリリストを生成する。 2. **RIRルールの展開**: 生成された各クエリに対し fireRIRrules を適用し、ビューの展開を行う。この際、元のクエリIDを各クエリに伝播させる。 3. **コマンドタグの決定**:

  • 元のクエリがリスト内に存在する場合は、そのクエリがタグを設定する。
  • 存在しない場合は、元のクエリと同じコマンド種別を持つ最後の INSTEAD ルール適用済みクエリがタグを設定するよう canSetTag を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/createas.c
  • src/backend/commands/explain.c
  • src/backend/commands/matview.c
  • src/backend/commands/portalcmds.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • RewriteQuery
  • fireRIRrules

関連データ構造

  • Query: クエリツリーを保持する主要な構造体。
  • List: 書き換え結果のクエリリスト。

失敗モードと境界条件

  • ルール適用によりクエリが0件になる場合がある。
  • canSetTag を持つクエリが結果リスト内に存在しない場合、呼び出し元（tcop）側でデフォルトのタグ設定を行う必要がある。

保守担当者向けメモ

  • 本関数はトップレベルのクエリに対してのみ適用される。
  • fireRIRrules によるビュー展開時に無限再帰が発生しないよう、システム側で activeRIRs 等を用いた追跡が行われている。
  • USE_ASSERT_CHECKING が無効な場合、QSRC_ORIGINAL を見つけた時点でループを早期終了する最適化が行われる。
