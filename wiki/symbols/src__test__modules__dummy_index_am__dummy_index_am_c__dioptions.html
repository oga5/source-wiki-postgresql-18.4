dioptions

定義

  • 種別: function
  • 定義箇所: src/test/modules/dummy_index_am/dummy_index_am.c:221

目的

インデックス作成時に指定されたリレーションオプション（reloptions）を解析・検証し、DummyIndexOptions 構造体として構築するためのハンドラ関数です。

シグネチャまたは形状

static bytea *dioptions(Datum reloptions, bool validate)

入力と出力

  • **入力**:
  • reloptions: 解析対象のオプションデータ（Datum 型）。
  • validate: オプションの妥当性検証を行うかどうかを示すフラグ。
  • **出力**:
  • 解析結果を格納した bytea * 型のポインタ。

前提条件と不変条件

  • di_relopt_tab が定義されており、有効なリレーションオプションのテーブルとして参照可能であること。
  • di_relopt_kind が適切に初期化されていること。

アルゴリズムまたは振る舞い

1. build_reloptions 関数を呼び出し、以下の情報を渡してオプションの解析を実行します。

  • reloptions および validate フラグ。
  • di_relopt_kind（オプションの種類）。
  • sizeof(DummyIndexOptions)（結果を格納する構造体のサイズ）。
  • di_relopt_tab およびその要素数（lengthof(di_relopt_tab)）。

    • 2. 解析された結果を bytea * にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/dummy_index_am/dummy_index_am.c:313: IndexAmRoutine 構造体の amoptions フィールドに本関数が登録されています。
  • **呼び出し先**:
  • build_reloptions（PostgreSQLのコア関数）。

関連データ構造

  • DummyIndexOptions: オプション値を保持する構造体。
  • di_relopt_tab: オプションの定義テーブル。

失敗モードと境界条件

  • validate が true の場合、不正なオプションが指定されると build_reloptions 内部でエラーが発生し、処理が中断される可能性があります。

保守担当者向けメモ

  • 本関数は IndexAmRoutine の一部として使用されます。新しいオプションを di_relopt_tab に追加した場合は、本関数を変更することなく自動的に解析対象に含まれます。
