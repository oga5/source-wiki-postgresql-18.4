backup_manifest_option

定義

  • 種別: enum
  • 定義箇所: src/include/backup/backup_manifest.h:20

目的

バックアップ生成プロセスにおいて、バックアップマニフェストの作成要否およびエンコーディングの挙動を制御するために使用されます。

シグネチャまたは形状

typedef enum manifest_option
{
	MANIFEST_OPTION_YES,
	MANIFEST_OPTION_NO,
	MANIFEST_OPTION_FORCE_ENCODE,
} backup_manifest_option;

入力と出力

  • 本シンボルは列挙型であり、関数ではありません。
  • バックアップ処理の構成設定として、マニフェスト生成の動作モードを決定する入力値として機能します。

前提条件と不変条件

  • マニフェスト生成の有効/無効の切り替えに使用されます。
  • MANIFEST_OPTION_FORCE_ENCODE は、マニフェストのエンコーディング制御に関連する特殊な挙動を指示します。

アルゴリズムまたは振る舞い

  • MANIFEST_OPTION_YES: マニフェストの生成を有効にします。
  • MANIFEST_OPTION_NO: マニフェストの生成を無効にします。
  • MANIFEST_OPTION_FORCE_ENCODE: 強制的なエンコーディング処理を伴う生成を指示します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/backup/backup_manifest.c:58
  • src/backend/backup/basebackup.c:75
  • src/include/backup/backup_manifest.h:39
  • 呼び出し先:
  • 本シンボルは列挙型であり、直接的な呼び出し先はありません。

関連データ構造

  • backup_manifest_info: マニフェスト生成のコンテキストを保持する構造体。本列挙型の値に基づいて動作が分岐します。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • マニフェスト生成のロジックを変更する際は、この列挙型の各値が backup_manifest_info の初期化および InitializeBackupManifest 等の関数で適切にハンドリングされているか確認してください。
