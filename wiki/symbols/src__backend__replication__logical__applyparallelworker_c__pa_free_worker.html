pa_free_worker

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/applyparallelworker.c:555

目的

並列適用ワーカー（PA）の解放および終了処理を行います。ワーカーをプールへ戻すか、必要に応じてプロセスを停止させます。

シグネチャまたは形状

static void pa_free_worker(ParallelApplyWorkerInfo *winfo)

入力と出力

  • **入力**: ParallelApplyWorkerInfo *winfo - 解放対象のワーカー情報構造体。
  • **出力**: なし。

前提条件と不変条件

  • 呼び出し元が並列適用ワーカー自身ではないこと（!am_parallel_apply_worker()）。
  • 対象のワーカーが使用中であること（winfo->in_use）。
  • トランザクション状態が完了（PARALLEL_TRANS_FINISHED）であること。

アルゴリズムまたは振る舞い

1. ParallelApplyTxnHash から対象のトランザクションID（xid）を削除します。削除に失敗した場合はエラーを発生させます。 2. 以下のいずれかの条件を満たす場合、ワーカーを停止・破棄します。

  • serialize_changes が真である（データ転送のタイムアウト等によりシリアライズが発生した）。
  • プール内のワーカー数が max_parallel_apply_workers_per_subscription の半分を超えている。
  • 停止時は logicalrep_pa_worker_stop(winfo) および pa_free_worker_info(winfo) を実行します。

    • 3. 上記条件を満たさない場合、ワーカーをプールへ戻すため in_use および serialize_changes フラグを false に設定します。

関連データ構造

  • ParallelApplyWorkerInfo: ワーカーの管理状態を保持。
  • ParallelApplyTxnHash: トランザクションIDとワーカーの対応を管理するハッシュテーブル。
  • ParallelApplyWorkerPool: 再利用可能なワーカーのリスト。

失敗モードと境界条件

  • ParallelApplyTxnHash からの削除失敗時に elog(ERROR) を発生させます。
  • メッセージキューの不整合を避けるため、シリアライズが発生している場合はワーカーを再利用せず停止させる設計となっています。

保守担当者向けメモ

  • 本関数はワーカーのライフサイクル管理の終端を担います。プールサイズ制限のロジックは、システム全体のリソース消費とワーカー起動コストのトレードオフに基づいています。
