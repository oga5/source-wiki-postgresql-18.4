match_inj_entries

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_stats.c:200

目的

統計サブシステム内の共有ハッシュエントリが、インジェクションポイントに関連する統計種別（PGSTAT_KIND_INJECTION）であるかを判定する述語関数です。

シグネチャまたは形状

static bool match_inj_entries(PgStatShared_HashEntry *entry, Datum match_data)

入力と出力

  • **入力**:
  • entry: 検査対象となる PgStatShared_HashEntry 構造体へのポインタ。
  • match_data: 比較に使用する Datum 型の値（本関数内では未使用）。
  • **出力**:
  • bool: entry の種別が PGSTAT_KIND_INJECTION と一致する場合は true、それ以外は false を返します。

前提条件と不変条件

  • entry は有効な PgStatShared_HashEntry 構造体を指している必要があります。
  • PGSTAT_KIND_INJECTION は、PostgreSQLの統計サブシステムにおいてインジェクションポイント統計を一意に識別する定数として定義されています。

アルゴリズムまたは振る舞い

1. 引数として渡された entry の key.kind フィールドにアクセスします。 2. その値が PGSTAT_KIND_INJECTION と等しいかどうかを比較します。 3. 比較結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: pgstat_drop_matching_entries 関数からコールバックとして呼び出されます（src/test/modules/injection_points/injection_stats.c:213）。
  • **呼び出し先**: 外部関数への呼び出しはありません。

関連データ構造

  • PgStatShared_HashEntry: 統計サブシステムの共有ハッシュテーブルにおけるエントリの基底構造体。

失敗モードと境界条件

  • entry が NULL の場合、デリファレンスによりクラッシュする可能性があります。

保守担当者向けメモ

  • 本関数は pgstat_drop_matching_entries のフィルタリングロジックとしてのみ使用されます。
  • match_data 引数は現在の実装では無視されていますが、将来的に特定のインジェクションポイントのみを対象とするような拡張が必要になった場合に利用可能です。
