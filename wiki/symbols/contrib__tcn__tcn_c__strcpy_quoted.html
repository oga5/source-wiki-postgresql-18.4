strcpy_quoted

定義

  • 種別: function
  • 定義箇所: contrib/tcn/tcn.c:35

目的

文字列を特定の引用符で囲み、文字列内に含まれる同一の引用符をエスケープして StringInfo バッファに追加するユーティリティ関数です。

シグネチャまたは形状

static void strcpy_quoted(StringInfo r, const char *s, const char q)

入力と出力

  • r (入力/出力): データを追加する StringInfo バッファ。
  • s (入力): 引用符で囲む対象のソース文字列。
  • q (入力): 引用符として使用する文字（例: " や '）。
  • 出力: なし（r が指すバッファが更新されます）。

前提条件と不変条件

  • r は有効な StringInfo オブジェクトである必要があります。
  • s はヌル終端文字列である必要があります。

アルゴリズムまたは振る舞い

1. q をバッファ r に追加します。 2. s の各文字を走査します。

  • 現在の文字が q と一致する場合、エスケープとしてもう一つ q をバッファに追加します。
  • 現在の文字をバッファに追加します。

    • 3. 最後に閉じ引用符として q をバッファに追加します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/tcn/tcn.c 内の triggered_change_notification 関数から複数回呼び出されます。
  • 呼び出し先: lib/stringinfo.h で定義されている appendStringInfoCharMacro を使用します。

関連データ構造

  • StringInfo: PostgreSQL の動的文字列バッファ構造体。

失敗モードと境界条件

  • s が空文字列の場合、q が2つ連続して出力されます（例: ""）。
  • s に q が含まれる場合、その箇所が二重に記述されます（例: q='"' で s="a"b" の場合、"a""b" となる）。

保守担当者向けメモ

  • 本関数は単純な引用符エスケープを行うため、複雑なデータ型や特殊文字を含む文字列を扱う際は、クライアント側でのパース時に注意が必要です。
