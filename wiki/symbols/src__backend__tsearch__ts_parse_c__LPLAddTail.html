LPLAddTail

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_parse.c:72

目的

ListParsedLex 構造体で管理される連結リストの末尾に、新しい ParsedLex 要素を追加します。

シグネチャまたは形状

static void LPLAddTail(ListParsedLex *list, ParsedLex *newpl)

入力と出力

  • **入力**:
  • list: 操作対象の ListParsedLex 構造体へのポインタ。
  • newpl: リストに追加する ParsedLex 構造体へのポインタ。
  • **出力**: なし（副作用としてリストの状態を更新）。

前提条件と不変条件

  • list は有効な ListParsedLex 構造体を指している必要があります。
  • 追加される newpl は、リストに追加される準備ができている必要があります。
  • リストの末尾に追加された後、newpl->next は必ず NULL に設定されます。

アルゴリズムまたは振る舞い

1. list->tail が存在する場合（リストが空でない場合）：

  • 現在の末尾要素の next ポインタを newpl に設定します。
  • list->tail を newpl に更新します。

    • 2. list->tail が存在しない場合（リストが空の場合）：

  • list->head と list->tail の両方に newpl を設定します。

    • 3. 最後に newpl->next を NULL に設定し、リストの終端であることを保証します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/ts_parse.c 内の LexizeExec 関数から呼び出されます。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • ListParsedLex: トークンリストを管理する構造体（head および tail ポインタを保持）。
  • ParsedLex: リストの各要素となる構造体（next ポインタを保持）。

失敗モードと境界条件

  • リストが空の状態から要素を追加する場合、head と tail の両方が正しく更新されます。
  • 既存のリストに要素を追加する場合、末尾のポインタのみが更新されます。

保守担当者向けメモ

  • 本関数は LexizeData 内の towork（作業中リスト）や waste（処理済みリスト）の操作に使用されます。
  • ParsedLex 構造体のメモリ管理は呼び出し元に依存するため、本関数内でメモリの割り当てや解放は行われません。
