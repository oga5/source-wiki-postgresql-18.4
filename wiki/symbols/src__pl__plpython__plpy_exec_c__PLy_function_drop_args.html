PLy_function_drop_args

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_exec.c:581

目的

PL/Python 関数実行時に一時的に保持された引数構造体（PLySavedArgs）を解放し、関連する Python オブジェクトの参照カウントを適切に減少させる。

シグネチャまたは形状

static void PLy_function_drop_args(PLySavedArgs *savedargs)

入力と出力

  • **入力**: PLySavedArgs *savedargs - 解放対象の引数状態を保持する構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • savedargs は有効なメモリ領域を指している必要がある。
  • 内部の Python オブジェクト（namedargs, args, td）は、Python インタプリタの参照カウント管理下にある必要がある。

アルゴリズムまたは振る舞い

1. savedargs->namedargs 配列内の各 Python オブジェクトに対して Py_XDECREF を実行し、参照カウントを減少させる。 2. savedargs->args および savedargs->td オブジェクトに対して Py_XDECREF を実行する。 3. 最後に pfree(savedargs) を呼び出し、構造体自体が使用していたメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_exec.c:274
  • src/pl/plpython/plpy_exec.c:688

    • （いずれも srfstate->savedargs を引数として呼び出している）

  • **呼び出し先**:
  • Py_XDECREF (Python C API)
  • pfree (PostgreSQL メモリ管理)

関連データ構造

  • PLySavedArgs: 引数、名前付き引数、およびトリガーデータ（td）を保持する構造体。

失敗モードと境界条件

  • savedargs が NULL の場合、pfree の挙動に依存する（通常はクラッシュする可能性があるため、呼び出し側で非 NULL を保証する必要がある）。
  • Py_XDECREF は NULL ポインタを許容するため、構造体内の各メンバが NULL であっても安全に処理される。

保守担当者向けメモ

  • 本関数は PLySRFState のクリーンアッププロセスの一部として使用される。
  • Python オブジェクトの参照カウント管理と PostgreSQL のメモリ管理（pfree）が混在しているため、構造体のメンバ構成に変更を加える際は、必ず Py_XDECREF の対象を更新すること。
