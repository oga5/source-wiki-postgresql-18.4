buffer_call_start_io

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_aio/test_aio.c:528

目的

PostgreSQL のバッファ I/O 開始処理を SQL インターフェースから直接呼び出し、テスト目的でバッファの I/O 状態を制御する。

シグネチャまたは形状

Datum buffer_call_start_io(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • buf (int32): 対象のバッファ ID。
  • for_input (bool): 読み込み用か書き込み用かのフラグ。
  • nowait (bool): ロック待機を行わないかどうかのフラグ。
  • **出力**:
  • bool: I/O の開始に成功した場合は true、失敗した場合は false を返す。

前提条件と不変条件

  • 指定された buf が有効なバッファであることを前提とする。
  • 共有バッファの場合、成功時に ResourceOwner から当該バッファの I/O 追跡を解除する。

アルゴリズムまたは振る舞い

1. BufferIsLocal(buf) を判定し、ローカルバッファであれば StartLocalBufferIO を、共有バッファであれば StartBufferIO を呼び出す。 2. 共有バッファかつ I/O 開始に成功した場合、ResourceOwnerForgetBufferIO を呼び出し、リソースオーナーによる自動的な I/O 解放を抑制する。 3. 処理後のバッファ参照カウントを DebugPrintBufferRefcount を用いて LOG レベルで出力する。 4. 最終的に can_start の値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出される（PG_FUNCTION_INFO_V1 マクロによる登録あり）。
  • **呼び出し先**:
  • StartLocalBufferIO / StartBufferIO
  • GetLocalBufferDescriptor / GetBufferDescriptor
  • ResourceOwnerForgetBufferIO
  • DebugPrintBufferRefcount
  • ereport

関連データ構造

  • Buffer (バッファ識別子)
  • ResourceOwner (現在のリソースオーナー)

失敗モードと境界条件

  • StartBufferIO または StartLocalBufferIO が失敗した場合、can_start は false となり、リソースオーナーの操作はスキップされる。
  • 共有バッファ以外（ローカルバッファ）に対しては、リソースオーナーの操作は行われない。

保守担当者向けメモ

  • 本関数はテスト専用であり、共有バッファのロック状態を強制的に操作する。本番環境のコードには流用しないこと。
  • ResourceOwnerForgetBufferIO の呼び出しは、テスト中に意図しない I/O 解放を防ぐためのハックである。
