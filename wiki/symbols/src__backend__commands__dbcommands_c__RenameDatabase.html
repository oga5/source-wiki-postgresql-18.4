RenameDatabase

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/commands/dbcommands.c:1901

目的

既存のデータベースの名前を変更する。

シグネチャまたは形状

ObjectAddress RenameDatabase(const char *oldname, const char *newname)

入力と出力

  • **入力**:
  • oldname: 変更対象のデータベース名。
  • newname: 新しいデータベース名。
  • **出力**:
  • 変更されたデータベースの ObjectAddress を返す。

前提条件と不変条件

  • 呼び出し元は、対象データベースに対する所有者権限（object_ownercheck）および CREATEDB 権限を保持している必要がある。
  • 変更対象のデータベースが現在接続中のデータベース（MyDatabaseId）であってはならない。
  • 変更対象のデータベースに対して、他のセッションや準備されたトランザクションが存在してはならない。
  • newname で指定された名前のデータベースが既に存在してはならない。

アルゴリズムまたは振る舞い

1. pg_database システムカタログに対して RowExclusiveLock を取得する。 2. get_db_info を使用して対象データベースの OID を取得し、排他ロックをかける。 3. 権限チェックおよび競合チェック（既存名確認、接続確認）を行う。 4. SearchSysCacheLockedCopy1 で pg_database のタプルを取得する。 5. namestrcpy を用いて datname フィールドを newname に更新し、CatalogTupleUpdate でカタログを書き換える。 6. InvokeObjectPostAlterHook を呼び出し、変更を通知する。 7. table_close でロックを解放し、ObjectAddress を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/alter.c:382（ALTER DATABASE ... RENAME TO コマンド実行時）。
  • **呼び出し先**: table_open, get_db_info, object_ownercheck, have_createdb_privilege, get_database_oid, CountOtherDBBackends, SearchSysCacheLockedCopy1, CatalogTupleUpdate, UnlockTuple, InvokeObjectPostAlterHook, ObjectAddressSet, table_close。

関連データ構造

  • ObjectAddress: 操作対象のオブジェクトを識別する構造体。
  • pg_database: データベース情報を保持するシステムカタログ。

失敗モードと境界条件

  • **ERRCODE_UNDEFINED_DATABASE**: oldname が存在しない場合。
  • **ERRCODE_INSUFFICIENT_PRIVILEGE**: 権限が不足している場合。
  • **ERRCODE_DUPLICATE_DATABASE**: newname が既に存在する場合。
  • **ERRCODE_FEATURE_NOT_SUPPORTED**: 現在接続中のデータベース名を変更しようとした場合。
  • **ERRCODE_OBJECT_IN_USE**: 対象データベースにアクティブなセッションが存在する場合。

保守担当者向けメモ

  • 本関数は非トランザクション的な操作を含むため、エラー発生時の整合性に注意が必要。
  • ENFORCE_REGRESSION_TEST_NAME_RESTRICTIONS が定義されている場合、回帰テスト用データベース名に関する警告がログ出力される。
