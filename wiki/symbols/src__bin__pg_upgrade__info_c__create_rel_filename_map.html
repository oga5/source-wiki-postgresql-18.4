create_rel_filename_map

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/info.c:164

目的

新旧クラスタ間でのリレーション（テーブルやインデックス）のファイルパスを照合し、移行に必要な情報を FileNameMap 構造体に格納する。

シグネチャまたは形状

static void create_rel_filename_map(
    const char *old_data, const char *new_data,
    const DbInfo *old_db, const DbInfo *new_db,
    const RelInfo *old_rel, const RelInfo *new_rel,
    FileNameMap *map
)

入力と出力

  • **入力**:
  • old_data, new_data: 新旧クラスタのデータディレクトリパス。
  • old_db, new_db: データベース情報。
  • old_rel, new_rel: 移行対象のリレーション情報。
  • **出力**:
  • map: 渡された FileNameMap 構造体の各フィールド（テーブルスペースパス、サフィックス、OID、リレーションファイル番号、名前空間、リレーション名）が更新される。

前提条件と不変条件

  • old_rel と new_rel は、OID が一致するペアとして既に特定されている必要がある。
  • データベースの OID およびリレーションのファイル番号は、新旧クラスタ間で不変であると仮定される。

アルゴリズムまたは振る舞い

1. **テーブルスペースの判定**:

  • old_rel および new_rel の tablespace 文字列が空であれば、デフォルトの /base ディレクトリを使用する。
  • 空でなければ、指定されたテーブルスペースのパスと、グローバル変数 old_cluster/new_cluster が保持するテーブルスペースサフィックスを使用する。

    • 2. **マッピングの割り当て**:

  • データベース OID とリレーションファイル番号を map にコピーする。
  • ログおよびエラー報告用に、名前空間名 (nspname) とリレーション名 (relname) を old_rel からコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/info.c 内の gen_db_file_maps から呼び出されることが確認されている。
  • **呼び出し先**: 直接的な関数呼び出しは行わず、構造体のフィールド設定のみを行う。

関連データ構造

  • DbInfo: データベースのメタデータを保持。
  • RelInfo: リレーションのメタデータを保持。
  • FileNameMap: ファイル移行の定義を保持する構造体。

失敗モードと境界条件

  • tablespace 文字列が空の場合のパス生成ロジックに依存している。
  • 呼び出し元で old_rel と new_rel が正しくペアリングされていない場合、不整合なマッピングが生成される可能性がある。

保守担当者向けメモ

  • 本関数は gen_db_file_maps による線形走査の一部として機能する。
  • テーブルスペースのパス解決ロジックは、pg_upgrade のクラスタ構成に強く依存しているため、変更時は old_cluster / new_cluster グローバル変数の整合性に注意すること。
