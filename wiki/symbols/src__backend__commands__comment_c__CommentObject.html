CommentObject

定義

  • 種別: function
  • 定義箇所: src/backend/commands/comment.c:39

目的

SQLの COMMENT ON 文を処理し、データベースオブジェクトに対するコメントの作成、更新、または削除を行う。

シグネチャまたは形状

ObjectAddress CommentObject(CommentStmt *stmt)

入力と出力

  • **入力**: CommentStmt *stmt（解析済みの COMMENT ON 文の構造体）
  • **出力**: ObjectAddress（操作対象となったオブジェクトの識別情報）

前提条件と不変条件

  • 実行ユーザーは対象オブジェクトの所有者である必要がある（check_object_ownership で検証）。
  • get_object_address により、対象オブジェクトに対する ShareUpdateExclusiveLock が取得される。
  • データベース、テーブルスペース、ロールは共有オブジェクトとして扱われ、それ以外は通常オブジェクトとして扱われる。

アルゴリズムまたは振る舞い

1. **データベース名の検証**: OBJECT_DATABASE の場合、存在確認を行い、存在しない場合は警告を出して終了する。 2. **オブジェクト特定**: get_object_address を呼び出し、対象のOIDとクラスIDを特定し、ロックを取得する。 3. **権限確認**: check_object_ownership で所有権を検証する。 4. **型固有の制約**: OBJECT_COLUMN の場合、特定のリレーション種別（テーブル、ビュー等）以外へのコメント付与をエラーとする。 5. **カタログ操作**:

  • 共有オブジェクト（DB, ロール, テーブルスペース）の場合: CreateSharedComments を呼び出す。
  • その他: CreateComments を呼び出す。

    • 6. **後処理**: 取得したリレーションをクローズする（ロックはトランザクション終了まで保持）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c
  • src/backend/tcop/utility.c
  • **呼び出し先**:
  • get_object_address
  • check_object_ownership
  • CreateSharedComments / CreateComments
  • relation_close

関連データ構造

  • CommentStmt: SQL文の解析結果を保持する構造体。
  • ObjectAddress: オブジェクトを識別するための構造体。
  • pg_description / pg_shdescription: コメントを格納するシステムカタログ。

失敗モードと境界条件

  • 対象オブジェクトが存在しない場合、get_object_address がエラーを投げる。
  • 所有者でない場合、check_object_ownership がエラーを投げる。
  • サポートされていないリレーション種別へのカラムコメント付与はエラーとなる。
  • 空文字列のコメントは、内部的にコメントの削除として処理される。

保守担当者向けメモ

  • pg_dump との整合性のため、インデックスカラムへのコメントは許可されていない。
  • 共有オブジェクトと通常オブジェクトで操作するカタログが異なるため、修正時は両方の関数（CreateComments / CreateSharedComments）への影響を考慮すること。
