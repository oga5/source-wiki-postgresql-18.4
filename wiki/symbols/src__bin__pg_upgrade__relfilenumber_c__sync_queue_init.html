sync_queue_init

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/relfilenumber.c:48

目的

swap モードにおけるファイル同期処理のためのバッファリング機構（同期キュー）を初期化します。

シグネチャまたは形状

static inline void sync_queue_init(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルな同期キューの状態を更新）

前提条件と不変条件

  • sync_queue_inited フラグにより、複数回呼び出されても初期化処理は一度のみ実行されます。
  • SYNC_QUEUE_MAX_LEN 分のメモリ領域が palloc を介して確保されます。

アルゴリズムまたは振る舞い

1. sync_queue_inited が true であれば即座に終了します。 2. sync_queue_inited を true に設定します。 3. SYNC_QUEUE_MAX_LEN 回ループし、各キュー要素に対して MAXPGPATH サイズのメモリを割り当てます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_upgrade/relfilenumber.c 内の同期処理が必要な箇所から呼び出されます。
  • 呼び出し先: palloc（メモリ割り当て関数）。

関連データ構造

  • sync_queue: 同期対象のファイルパスを保持するグローバルな配列。
  • sync_queue_inited: 初期化状態を管理するブール値。
  • SYNC_QUEUE_MAX_LEN: キューの最大長を定義する定数。

失敗モードと境界条件

  • メモリ不足時: palloc が失敗した場合、PostgreSQL の標準的なエラーハンドリングに従いプロセスが終了します。

保守担当者向けメモ

  • 本関数は inline で定義されており、swap モードの実行パスにおいて同期キューが確実に準備されていることを保証します。
  • sync_queue_push や sync_queue_sync_all と組み合わせて使用される設計です。
