spgPickSplitIn

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgist.h:110

目的

SP-GiSTインデックスにおいて、リーフタプルの分割戦略を決定するための入力情報を保持する構造体です。

シグネチャまたは形状

typedef struct spgPickSplitIn
{
    int         nTuples;        /* リーフタプルの数 */
    Datum      *datums;         /* リーフタプルのデータ配列 (要素数: nTuples) */
    int         level;          /* 現在のツリーレベル (0から開始) */
} spgPickSplitIn;

入力と出力

  • **入力**: インデックス挿入処理中に、ノードの分割が必要と判断された際に本構造体が生成されます。
  • **出力**: 本構造体は spgPickSplitOut 構造体と対で使用され、演算子クラスの picksplit サポート関数へ渡されます。

前提条件と不変条件

  • datums 配列は nTuples の長さを持つ必要があります。
  • level は 0 以上の整数です。

アルゴリズムまたは振る舞い

SP-GiSTの挿入処理において、ノードが一杯になった際、演算子クラスの picksplit 関数が呼び出されます。この関数は spgPickSplitIn を受け取り、データをどのように分割して新しい内部ノードを構成するかを決定し、その結果を spgPickSplitOut に格納して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgdoinsert.c
  • 各種演算子クラスの実装（spgkdtreeproc.c, spgquadtreeproc.c, spgtextproc.c, geo_spgist.c, network_spgist.c, rangetypes_spgist.c など）で、PG_GETARG_POINTER(0) を介して受け取られます。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • spgPickSplitOut: 分割結果を格納する対となる構造体。
  • Datum: インデックス対象のデータ型。

失敗モードと境界条件

  • nTuples が 0 の場合や、メモリ割り当てに失敗した場合の挙動は、呼び出し側の実装に依存します。

保守担当者向けメモ

  • 本構造体はSP-GiSTの演算子クラス定義において必須のインターフェースの一部です。変更を加える際は、すべての既存演算子クラス実装（spgkdtreeproc.c 等）との整合性を確認してください。
