moveArrayTypeName

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_type.c:904

目的

指定された型に関連付けられた配列型の名前を、新しいベース型名に基づいて再生成し、カタログ上で更新する。

シグネチャまたは形状

bool moveArrayTypeName(Oid typeOid, const char *typeName, Oid typeNamespace)

入力と出力

  • **入力**:
  • typeOid: 対象となる配列型のOID。
  • typeName: 新しいベース型名。
  • typeNamespace: 型が存在する名前空間のOID。
  • **出力**:
  • bool: 処理が成功した場合は true、対象が配列型でない等の理由で変更不要または変更不可の場合は false を返す。

前提条件と不変条件

  • 対象の型がシェル型（未定義）である場合、何もせず true を返す。
  • 対象の型が自動生成された配列型である必要がある（get_element_type で取得した要素型の配列型が typeOid と一致すること）。

アルゴリズムまたは振る舞い

1. get_typisdefined を確認し、シェル型であれば即座に終了する。 2. get_element_type を使用して要素型を特定し、その要素型の配列型が typeOid と一致するか検証する。 3. makeArrayTypeName を呼び出し、新しいベース型名に基づいた一意な配列型名を生成する。 4. RenameTypeInternal を呼び出し、カタログ上の型名を更新する。 5. CommandCounterIncrement を呼び出し、後続の処理で更新後の名前が正しく認識されるようにコマンドカウンタを更新する。 6. 生成された名前文字列を解放し、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/heap.c、src/backend/catalog/pg_type.c、src/backend/commands/typecmds.c の各所から呼び出される。
  • **呼び出し先**: get_typisdefined, get_element_type, get_array_type, makeArrayTypeName, RenameTypeInternal, CommandCounterIncrement, pfree。

関連データ構造

  • pg_type: 型情報を保持するシステムカタログ。

失敗モードと境界条件

  • 対象の型が自動生成された配列型ではない場合、false を返す。
  • makeArrayTypeName が生成する名前が既存の型名と衝突する場合、連番が付与された名前が生成される。

保守担当者向けメモ

  • 本関数は ALTER TYPE 等でベース型の名前が変更された際に、連動して配列型の名前を追従させるために使用される。
  • CommandCounterIncrement の呼び出しは、同一トランザクション内で後続の makeArrayTypeName が古い名前を拾わないようにするために必須である。
