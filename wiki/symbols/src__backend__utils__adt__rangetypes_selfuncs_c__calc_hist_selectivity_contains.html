calc_hist_selectivity_contains

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_selfuncs.c:1138

目的

範囲型データにおいて、特定の範囲が別の範囲を包含する（@> 演算子）確率を、境界ヒストグラムと長さヒストグラムを用いて推定する。

シグネチャまたは形状

static double calc_hist_selectivity_contains(
    TypeCacheEntry *typcache,
    const RangeBound *lower, const RangeBound *upper,
    const RangeBound *hist_lower, int hist_nvalues,
    Datum *length_hist_values, int length_hist_nvalues
)

入力と出力

  • **入力**:
  • typcache: 型固有の比較関数やサブ差分関数を含むキャッシュ。
  • lower, upper: クエリ対象範囲の境界。
  • hist_lower: 統計情報として保持されている範囲の下限値ヒストグラム。
  • hist_nvalues: ヒストグラムの要素数。
  • length_hist_values: 範囲の長さのヒストグラム。
  • length_hist_nvalues: 長さヒストグラムの要素数。
  • **出力**:
  • 包含関係が成立する確率（0.0 から 1.0 の範囲）。

前提条件と不変条件

  • 範囲の「下限値」と「長さ」は独立していると仮定して計算される。
  • hist_lower は昇順にソートされている必要がある。
  • lower_index が負の場合、クエリ範囲の下限がヒストグラムの範囲外にあると見なし、直ちに 0.0 を返す。

アルゴリズムまたは振る舞い

1. rbound_bsearch を使用し、クエリの下限値が含まれるヒストグラムのビンを特定する。 2. ヒストグラムの範囲外であれば 0.0 を返す。 3. get_position を用いて、特定したビン内でのクエリ下限値の相対位置（lower_bin_width）を線形補間により算出する。 4. クエリ下限値からヒストグラムの開始位置まで逆方向にループし、各ビンにおける「クエリ範囲を包含する確率」を累積する。

  • 各ステップで get_distance を用いて、ヒストグラムの境界とクエリ上限との距離を計算する。
  • calc_length_hist_frac を呼び出し、その距離以上の長さを持つ範囲の割合を算出する。
  • 算出した割合とビンの幅を掛け合わせ、sum_frac に加算する。

    • 5. 最終的な累積値 sum_frac を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/rangetypes_selfuncs.c 内の関数（calc_hist_selectivity 等から参照される）。
  • src/backend/utils/adt/multirangetypes_selfuncs.c 内の関数。
  • **呼び出し先**:
  • rbound_bsearch: ヒストグラム内の位置特定。
  • get_position: ビン内での相対位置計算。
  • get_distance: 境界間の距離計算。
  • calc_length_hist_frac: 長さヒストグラムに基づく確率計算。

関連データ構造

  • RangeBound: 範囲の境界（値と包含フラグ）。
  • TypeCacheEntry: 型固有の操作関数を保持するキャッシュ構造体。

失敗モードと境界条件

  • lower_index < 0: クエリ範囲の下限がヒストグラムの最小値より小さい場合、選択率は 0.0 となる。
  • lower_index >= hist_nvalues - 1: クエリ範囲の下限がヒストグラムの最大値を超える場合、最後のビンを基準に計算を行う。

保守担当者向けメモ

  • 本関数は、ヒストグラムのビン間を線形補間する幾何学的なアプローチを採用している。
  • 累積分布関数の積分を近似するロジックが含まれており、データ分布が仮定（下限と長さの独立性）から大きく逸脱する場合、推定精度が低下する。
