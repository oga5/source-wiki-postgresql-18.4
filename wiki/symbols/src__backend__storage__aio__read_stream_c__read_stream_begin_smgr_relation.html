read_stream_begin_smgr_relation

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/read_stream.c:759

目的

SMgrRelation を対象としたデータ読み込みストリームを開始します。

シグネチャまたは形状

ReadStream *read_stream_begin_smgr_relation(int flags,
                                            BufferAccessStrategy strategy,
                                            SMgrRelation smgr,
                                            char smgr_persistence,
                                            ForkNumber forknum,
                                            ReadStreamBlockNumberCB callback,
                                            void *callback_private_data,
                                            size_t per_buffer_data_size)

入力と出力

  • **入力**:
  • flags: ストリームの動作を制御するフラグ。
  • strategy: バッファアクセスの戦略。
  • smgr: 対象のリレーションストレージマネージャ。
  • smgr_persistence: リレーションの永続性設定。
  • forknum: 読み込み対象のフォーク番号。
  • callback: 次のブロック番号を取得するためのコールバック関数。
  • callback_private_data: コールバックに渡す任意のデータ。
  • per_buffer_data_size: 各バッファに関連付ける追加データのサイズ。
  • **出力**:
  • 初期化された ReadStream 構造体へのポインタを返します。

前提条件と不変条件

  • read_stream_begin_impl を呼び出すためのラッパーとして機能します。
  • 渡される smgr は有効なリレーションストレージマネージャである必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取ったパラメータをそのまま read_stream_begin_impl に転送します。 2. read_stream_begin_impl の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c:5180
  • **呼び出し先**:
  • read_stream_begin_impl (同一ファイル内)

関連データ構造

  • ReadStream: ストリームの状態を保持するメイン構造体。
  • SMgrRelation: ストレージマネージャのリレーション情報。

失敗モードと境界条件

  • 内部で呼び出される read_stream_begin_impl の失敗モードに依存します（メモリ不足や無効な引数など）。

保守担当者向けメモ

  • 本関数は read_stream_begin_impl への直接的なパススルーです。ロジックの変更が必要な場合は、read_stream_begin_impl を参照してください。
