warn

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-whenever.c:39

目的

ECPG（Embedded SQL for C）の回帰テストにおいて、SQL実行時にカラムの切り詰め（truncation）が発生した際の警告メッセージを標準エラー出力へ出力する。

シグネチャまたは形状

static void warn(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として stderr へ文字列を出力）

前提条件と不変条件

  • 本関数は preproc-whenever.c 内のテスト実行中に呼び出されることを想定している。
  • 呼び出し元は、SQL実行後の sqlca 構造体の状態に基づき、警告条件が満たされたと判断している必要がある。

アルゴリズムまたは振る舞い

1. fprintf を使用し、"Warning: At least one column was truncated\n" という固定メッセージを標準エラー出力（stderr）に書き出す。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内には、本関数を呼び出している具体的な箇所は明示されていない。
  • 呼び出し先: 標準ライブラリ関数 fprintf を呼び出す。

関連データ構造

  • sqlca: 本関数が直接参照することはないが、本関数が定義されているファイルの文脈上、SQL実行結果を保持する sqlca 構造体の状態に応じたハンドラとして機能する。

失敗モードと境界条件

  • stderr への書き込みが失敗した場合の挙動は、標準ライブラリの fprintf に依存する。

保守担当者向けメモ

  • 本関数は手書きのコードではなく、ECPGプリプロセッサが生成するCコードの期待値として定義されている。
  • 警告メッセージの内容を変更する場合は、プリプロセッサの生成ロジックと、このテストファイルの期待値の両方を整合させる必要がある。
