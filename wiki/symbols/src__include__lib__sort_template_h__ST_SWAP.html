ST_SWAP

定義

  • 種別: function
  • 定義箇所: src/include/lib/sort_template.h:272

目的

sort_template.h で生成されるソート関数内において、指定された型 ST_POINTER_TYPE の2つの要素を入れ替えるための汎用的なスワップ処理を提供します。

シグネチャまたは形状

static inline void ST_SWAP(ST_POINTER_TYPE * a, ST_POINTER_TYPE * b)

入力と出力

  • **入力**:
  • a: 入れ替え対象となる要素へのポインタ。
  • b: 入れ替え対象となるもう一方の要素へのポインタ。
  • **出力**: なし（引数として渡されたポインタ先の値を直接入れ替えます）。

前提条件と不変条件

  • ST_POINTER_TYPE がコンパイル時に適切に定義されている必要があります。
  • a および b は有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. a が指す値を一時変数 tmp にコピーします。 2. b が指す値を a が指す位置にコピーします。 3. tmp に保持していた値を b が指す位置にコピーします。

  • static inline 関数として定義されており、コンパイル時に展開されることで関数呼び出しのオーバーヘッドを排除します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/lib/sort_template.h 内のソートアルゴリズム実装（DO_SWAP マクロ経由での利用を含む）。
  • **呼び出し先**:
  • なし（言語組み込みの代入演算のみを使用）。

関連データ構造

  • ST_POINTER_TYPE: ソート対象の要素型を指すマクロ定義。

失敗モードと境界条件

  • a または b が NULL の場合、未定義動作を引き起こします。
  • メモリ領域が重なっている場合や、型サイズが極端に大きい場合の挙動については、呼び出し側のメモリ管理に依存します。

保守担当者向けメモ

  • 本関数はマクロによるメタプログラミングの一部として生成されます。デバッグ時は、展開後のコードにおける ST_SORT 接頭辞が付与された関数名を確認してください。
  • ST_SWAP は ST_SWAPN とは異なり、単一の ST_POINTER_TYPE 要素の入れ替えに特化しています。
