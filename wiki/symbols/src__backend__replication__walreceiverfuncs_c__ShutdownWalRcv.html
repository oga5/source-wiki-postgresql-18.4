ShutdownWalRcv

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walreceiverfuncs.c:178

目的

WALレシーバープロセスを安全に停止させ、その終了を待機する。

シグネチャまたは形状

void ShutdownWalRcv(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてWALレシーバーの停止と共有メモリ状態の更新を行う）

前提条件と不変条件

  • 共有メモリ上の WalRcv 構造体へのアクセスは、内部で mutex スピンロックにより保護される。
  • 停止処理中、WALレシーバーの状態が WALRCV_STOPPED に遷移するまで、呼び出し元は条件変数 walRcvStoppedCV を用いて待機する。

アルゴリズムまたは振る舞い

1. WalRcv->mutex を取得し、現在の walRcvState に応じて以下の処理を行う。

  • WALRCV_STARTING の場合: 状態を WALRCV_STOPPED に変更し、停止フラグを立てる。
  • WALRCV_STREAMING, WAITING, RESTARTING の場合: 状態を WALRCV_STOPPING に変更し、対象のプロセスID（pid）を保持する。

    • 2. 停止フラグが立っている場合、walRcvStoppedCV をブロードキャストする。 3. プロセスIDが有効な場合、kill(pid, SIGTERM) を発行してWALレシーバーに終了を促す。 4. ConditionVariablePrepareToSleep を呼び出し、WalRcvRunning() が偽（状態が WALRCV_STOPPED）になるまで ConditionVariableSleep で待機する。 5. 終了確認後、ConditionVariableCancelSleep を呼び出して待機を解除する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlog.c（ShutdownWalRcv のラッパー経由）
  • 呼び出し先: kill（システムコール）、ConditionVariable 関連関数、WalRcvRunning（状態確認）

関連データ構造

  • WalRcvData: 共有メモリ上のWALレシーバー制御構造体。
  • walRcvState: レシーバーの現在の状態（WALRCV_STOPPED, WALRCV_STOPPING 等）。
  • walRcvStoppedCV: 停止完了を通知するための条件変数。

失敗モードと境界条件

  • WALレシーバーが既に停止している場合（WALRCV_STOPPED）、本関数は即座に復帰する。
  • kill 送信後にWALレシーバーが正常に終了しない場合、ConditionVariableSleep により待機状態が継続する可能性がある。

保守担当者向けメモ

  • 本関数はスタートアッププロセスからの呼び出しを想定している。
  • 共有メモリの競合を防ぐため、状態遷移には必ず mutex を介すること。
  • WalRcvRunning() による待機ループは、WALレシーバーが確実に終了したことを保証するための重要な同期ポイントである。
