FindColsContext

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeAgg.c:361

目的

式ツリーを走査する際、集約関数（Aggref）の内部にある列参照と、それ以外の場所にある列参照を分類・追跡するために使用されるコンテキスト構造体です。

シグネチャまたは形状

typedef struct FindColsContext
{
    bool        is_aggref;      /* 集約関数内かどうかのフラグ */
    Bitmapset  *aggregated;     /* 集約関数内の列参照セット */
    Bitmapset  *unaggregated;   /* 集約関数外の列参照セット */
} FindColsContext;

入力と出力

  • 入力: find_cols_walker 関数において、走査対象のノードと本構造体へのポインタが渡されます。
  • 出力: 走査結果として、aggregated および unaggregated ビットマップセットに列参照情報が格納されます。

前提条件と不変条件

  • aggregated および unaggregated は、走査開始前に適切に初期化（または空の Bitmapset として準備）されている必要があります。

アルゴリズムまたは振る舞い

  • find_cols_walker 関数が式ツリーを再帰的に走査する際、この構造体を参照して現在のノードが Aggref の下にあるかを判断します。
  • is_aggref フラグの状態に応じて、検出された列参照を aggregated または unaggregated のいずれかのビットマップセットへ追加します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/nodeAgg.c 内の関数（find_cols_walker を呼び出す箇所）。
  • 呼び出し先: find_cols_walker 関数が本構造体を利用して走査を行います。

関連データ構造

  • Bitmapset: 列参照のインデックスを保持するために使用されます。
  • Aggref: 集約関数の呼び出しを表すノード。

失敗モードと境界条件

  • メモリ不足: Bitmapset への追加時にメモリ不足が発生する可能性があります。
  • 複雑な式: ネストされた集約関数や複雑な式ツリーにおいて、is_aggref フラグの管理が正しく行われない場合、列の分類が不正確になる可能性があります。

保守担当者向けメモ

  • 本構造体は find_cols_walker と密結合しています。走査ロジックを変更する場合は、is_aggref の状態遷移が正しく維持されているかを確認してください。
