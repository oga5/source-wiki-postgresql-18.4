ToastTupleContext

定義

  • 種別: struct
  • 定義箇所: src/include/access/toast_helper.h:41

目的

テーブルアクセスメソッド（AM）がタプルの挿入や更新を行う際、TOAST（圧縮や外部ストレージへの移動）処理の状態を保持・管理するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    Relation        ttc_rel;        /* 対象リレーション */
    Datum          *ttc_values;     /* カラム値 */
    bool           *ttc_isnull;     /* NULLフラグ */
    Datum          *ttc_oldvalues;  /* 更新前のカラム値 */
    bool           *ttc_oldisnull;  /* 更新前のNULLフラグ */
    uint8           ttc_flags;      /* 処理フラグ */
    ToastAttrInfo  *ttc_attr;       /* 各カラムのTOAST状態 */
} ToastTupleContext;

入力と出力

  • **入力**: 呼び出し側が ttc_rel、ttc_values、ttc_isnull を設定し、ttc_attr に natts 分のメモリを確保して渡す必要があります。挿入時は ttc_oldvalues と ttc_oldisnull を NULL に設定します。
  • **出力**: toast_tuple_init 等の関数によって ttc_attr 内の各属性の状態が更新され、TOAST 処理の判断材料として利用されます。

前提条件と不変条件

  • ttc_attr が指す配列の長さは、ttc_rel->rd_att->natts と一致している必要があります。
  • toast_tuple_init を呼び出す前に、上記入力フィールドを適切に初期化する必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: toast_tuple_init により、タプル内の各属性をスキャンし、TOAST 処理の要否を判定します。 2. **処理**: toast_tuple_try_compression や toast_tuple_externalize を通じて、個別の属性に対する圧縮や外部化を実行します。 3. **選定**: toast_tuple_find_biggest_attribute を用いて、TOAST 処理の優先順位を決定します。 4. **終了**: toast_tuple_cleanup を呼び出し、一時的なリソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heaptoast.c 等のテーブル AM 実装。
  • **呼び出し先**: toast_tuple_init, toast_tuple_find_biggest_attribute, toast_tuple_try_compression, toast_tuple_externalize, toast_tuple_cleanup などの toast_helper.c で定義された関数群。

関連データ構造

  • ToastAttrInfo: 各カラムの TOAST 状態（フラグ、サイズ等）を保持します。
  • Relation: 対象テーブルのメタデータ。

失敗モードと境界条件

  • ttc_attr 配列のメモリ確保が不十分な場合、メモリ破壊が発生します。
  • 挿入時に ttc_oldvalues 等を適切に NULL に設定しない場合、誤った古いデータの参照が発生する可能性があります。

保守担当者向けメモ

  • TOASTCOL_ 系フラグと TOAST_ 系フラグはビットレベルで互換性があるため、フラグ操作時は定義を確認してください。
  • 本構造体は主にテーブル AM 実装者が利用するものであり、一般的なユーザーインターフェースではありません。
