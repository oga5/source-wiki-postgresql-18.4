SlruScanDirCbDeleteCutoff

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/slru.c:1727

目的

SLRU ディレクトリのスキャン中に、指定されたカットオフページより古いセグメントファイルを削除するためのコールバック関数です。

シグネチャまたは形状

static bool SlruScanDirCbDeleteCutoff(SlruCtl ctl, char *filename, int64 segpage, void *data);

入力と出力

  • **入力**:
  • ctl: SLRU 制御構造体。
  • filename: スキャン対象のファイル名。
  • segpage: 現在のセグメントの開始ページ番号。
  • data: 削除判定の基準となるカットオフページ番号（int64 へのポインタ）。
  • **出力**:
  • 戻り値: 常に false を返し、スキャンを継続します。

前提条件と不変条件

  • data 引数は int64 型の値を指している必要があります。
  • SlruMayDeleteSegment が提供する削除条件を満たす場合にのみ、物理削除が実行されます。

アルゴリズムまたは振る舞い

1. data からカットオフページ番号を取得します。 2. SlruMayDeleteSegment を呼び出し、現在のセグメント（segpage）が削除可能か判定します。 3. 削除可能であれば、SlruInternalDeleteSegment を呼び出し、セグメント番号（segpage / SLRU_PAGES_PER_SEGMENT）に基づいてファイルを削除します。 4. 常に false を返し、ディレクトリ内の全ファイルを走査し続けます。

呼び出し元と呼び出し先

  • **呼び出し元**: SlruScanDirectory（src/backend/access/transam/slru.c:1493）からコールバックとして呼び出されます。
  • **呼び出し先**:
  • SlruMayDeleteSegment
  • SlruInternalDeleteSegment

関連データ構造

  • SlruCtl: SLRU インスタンスの制御構造体。

失敗モードと境界条件

  • data が無効なポインタである場合、未定義の動作を引き起こします。
  • 削除対象のセグメントが既に存在しない場合や、ファイルシステム権限により削除が失敗する場合の挙動は、内部で呼び出される SlruInternalDeleteSegment の実装に依存します。

保守担当者向けメモ

  • 本関数は SlruScanDirectory のスキャンロジックの一部として機能します。削除の可否判定ロジックを変更する場合は SlruMayDeleteSegment を参照してください。
