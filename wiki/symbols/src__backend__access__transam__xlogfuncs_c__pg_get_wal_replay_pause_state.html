pg_get_wal_replay_pause_state

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/transam/xlogfuncs.c:592

目的

現在のWALリカバリ処理のポーズ状態（一時停止状態）を取得し、SQL層へ文字列として返却します。

シグネチャまたは形状

Datum pg_get_wal_replay_pause_state(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出し規約に基づく引数）
  • **出力**: text 型（以下のいずれかの文字列）
  • "not paused": リカバリは一時停止していません。
  • "pause requested": 一時停止が要求されています。
  • "paused": リカバリは一時停止中です。

前提条件と不変条件

  • サーバーがリカバリ中である必要があります。リカバリ中でない場合に呼び出された場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE エラーを発生させます。

アルゴリズムまたは振る舞い

1. RecoveryInProgress() を呼び出し、リカバリ中であることを確認します。 2. GetRecoveryPauseState() を呼び出し、内部的なリカバリポーズ状態を取得します。 3. 取得した状態（RECOVERY_NOT_PAUSED, RECOVERY_PAUSE_REQUESTED, RECOVERY_PAUSED）に応じて、対応する文字列を statestr に代入します。 4. statestr が NULL でないことを Assert で検証します。 5. cstring_to_text を用いて文字列を変換し、PG_RETURN_TEXT_P マクロで結果を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます。具体的なSQLクエリの呼び出し元情報はソースコードからは特定できません。
  • **呼び出し先**:
  • RecoveryInProgress()
  • GetRecoveryPauseState()
  • cstring_to_text()

関連データ構造

  • RecoveryPauseState（列挙型）: GetRecoveryPauseState() が返す状態値。

失敗モードと境界条件

  • リカバリ中でない場合、エラーをスローして処理を中断します。

保守担当者向けメモ

  • 本関数は xlogfuncs.c 内の他の関数と同様、リカバリ状態のチェックが必須です。
  • GetRecoveryPauseState() が返す列挙型に新しい状態が追加された場合、本関数の switch 文にも対応するケースを追加する必要があります。
