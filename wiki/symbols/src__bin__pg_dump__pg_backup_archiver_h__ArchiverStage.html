ArchiverStage

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_dump/pg_backup_archiver.h:167

目的

アーカイブ処理（ダンプまたは復元）における現在の実行フェーズを追跡・管理するために使用されます。

シグネチャまたは形状

typedef enum
{
	STAGE_NONE = 0,
	STAGE_INITIALIZING,
	STAGE_PROCESSING,
	STAGE_FINALIZING,
} ArchiverStage;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: ArchiveHandle 構造体内の状態保持用メンバーとして利用されます。

前提条件と不変条件

  • ArchiveHandle 構造体において、現在の処理段階を示す stage およびエラー発生時の段階を示す lastErrorStage として保持されます。
  • 処理の進行に伴い、STAGE_NONE から順次遷移することが想定されます。

アルゴリズムまたは振る舞い

  • STAGE_NONE: 初期状態。
  • STAGE_INITIALIZING: アーカイブのオープンや設定の読み込みなど、準備段階。
  • STAGE_PROCESSING: メインのデータダンプまたは復元処理中。
  • STAGE_FINALIZING: 終了処理やクリーンアップ中。

呼び出し元と呼び出し先

  • 呼び出し元: ArchiveHandle 構造体（src/bin/pg_dump/pg_backup_archiver.h:337, 338）で参照されています。
  • 呼び出し先: 具体的な関数レベルの呼び出し元・呼び出し先情報は、提供されたソース excerpt には含まれていません。

関連データ構造

  • ArchiveHandle: この列挙型を状態管理に使用するメイン構造体。

失敗モードと境界条件

  • 失敗モード: 提供されたソース excerpt には記載がありません。
  • 境界条件: STAGE_NONE（0）から始まり、STAGE_FINALIZING で終了する一連のライフサイクルを前提としています。

保守担当者向けメモ

  • 本列挙型は ArchiveHandle の状態遷移を監視するために重要です。エラーハンドリング時に lastErrorStage と比較することで、どのフェーズで問題が発生したかを特定する用途で使用されます。
