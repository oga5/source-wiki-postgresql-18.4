HandleParallelApplyMessageInterrupt

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/applyparallelworker.c:1004

目的

並列適用ワーカーに対して、メッセージの受信割り込みが発生したことを通知し、処理を促すためのシグナルハンドラとして機能します。

シグネチャまたは形状

void HandleParallelApplyMessageInterrupt(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数およびラッチの状態を更新）

前提条件と不変条件

  • 本関数は、シグナルハンドラまたは割り込みコンテキストから呼び出されることを想定しています。
  • MyLatch が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. InterruptPending を true に設定し、割り込みが発生したことをシステム全体に通知します。 2. ParallelApplyMessagePending を true に設定し、並列適用ワーカー固有のメッセージ処理が必要であることをマークします。 3. SetLatch(MyLatch) を呼び出し、待機中のプロセス（ワーカー）を起床させます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/ipc/procsignal.c:696（直接的な呼び出しの証拠あり）
  • 呼び出し先:
  • SetLatch（PostgreSQLのラッチ管理機能）

関連データ構造

  • InterruptPending (グローバル変数)
  • ParallelApplyMessagePending (グローバル変数)
  • MyLatch (現在のプロセスのラッチ)

失敗モードと境界条件

  • MyLatch が設定されていない状態で呼び出された場合、PostgreSQLのラッチ管理機構によりエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は、論理レプリケーションの並列適用ワーカーがリーダーからのメッセージを非同期に受け取るための重要なトリガーです。
  • ParallelApplyMessagePending フラグは、後続の ProcessParallelApplyInterrupts 等でチェックされることを前提としています。
