HashMetaPageData

定義

  • 種別: struct
  • 定義箇所: src/include/access/hash.h:244

目的

PostgreSQL ハッシュインデックスのルート（ブロック 0）に配置されるメタデータ構造体です。インデックスの物理的な状態、バケットの分割状況、およびオーバーフローページの管理情報を保持します。

シグネチャまたは形状

typedef struct HashMetaPageData
{
    uint32      hashm_magic;
    uint32      hashm_version;
    double      hashm_ntuples;
    uint16      hashm_ffactor;
    uint16      hashm_bsize;
    uint16      hashm_bmsize;
    uint16      hashm_bmshift;
    uint32      hashm_maxbucket;
    uint32      hashm_highmask;
    uint32      hashm_lowmask;
    uint32      hashm_ovflpoint;
    uint32      hashm_firstfree;
    uint32      hashm_nmaps;
    RegProcedure hashm_procid;
    uint32      hashm_spares[HASH_MAX_SPLITPOINTS];
    BlockNumber hashm_mapp[HASH_MAX_BITMAPS];
} HashMetaPageData;

入力と出力

本構造体は関数ではなくデータ型です。インデックス操作時にバッファマネージャを介して読み込まれ、インデックスの構造変更（分割など）に伴い内容が更新されます。

前提条件と不変条件

  • インデックスのブロック 0 は常にメタページとして使用されます。
  • hashm_bmsize は 2 のべき乗である必要があります。
  • インデックスの構造変更（バケット分割等）を行う際は、メタページを排他ロックする必要があります。

アルゴリズムまたは振る舞い

  • **動的拡張**: hashm_spares 配列と hashm_maxbucket を使用して、ハッシュテーブルの動的なバケット分割を管理します。
  • **オーバーフロー管理**: hashm_firstfree および hashm_mapp を通じて、オーバーフローページの再利用をビットマップ形式で制御します。
  • **マッピング**: hashm_highmask および hashm_lowmask は、ハッシュ値をバケット番号へ変換する際のモジュロ演算に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pageinspect/hashfuncs.c および src/backend/access/hash/hashpage.c 内の各関数が、インデックスの検証やメタデータ更新のために本構造体を参照しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • HashMetaPage: HashMetaPageData へのポインタ型。
  • HashPageOpaqueData: 各インデックスページの末尾に配置されるメタデータ。
  • Bucket: バケット番号を表す uint32 型。

失敗モードと境界条件

  • メタページが破損している場合、インデックスの全スキャンおよび検索が失敗します。
  • HASH_MAX_SPLITPOINTS や HASH_MAX_BITMAPS の制限を超えるインデックス拡張は、構造上の限界に達します。

保守担当者向けメモ

  • 実装の詳細（特にバケット分割ロジック）については src/backend/access/hash/hashpage.c を参照してください。
  • WAL ログ（src/include/access/hash_xlog.h）において、バックアップブロックとしてメタページが記録されるため、変更時はログ出力の整合性に注意してください。
