InstrAccumParallelQuery

定義

  • 種別: function
  • 定義箇所: src/backend/executor/instrument.c:217

目的

並列クエリ実行において、各ワーカープロセスから報告されたバッファ使用量および WAL 生成量の統計を、リーダープロセスのグローバルな統計カウンタへ集約する。

シグネチャまたは形状

void InstrAccumParallelQuery(BufferUsage *bufusage, WalUsage *walusage)

入力と出力

  • **入力**:
  • bufusage: ワーカーから収集された BufferUsage 構造体へのポインタ。
  • walusage: ワーカーから収集された WalUsage 構造体へのポインタ。
  • **出力**: なし（副作用としてグローバル変数を更新）。

前提条件と不変条件

  • bufusage および walusage は、並列ワーカーによって適切に計算された有効な統計データであること。
  • 本関数は、並列クエリの終了処理フェーズにおいて、リーダープロセス側で呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. BufferUsageAdd を呼び出し、引数 bufusage の内容をグローバル変数 pgBufferUsage に加算する。 2. WalUsageAdd を呼び出し、引数 walusage の内容をグローバル変数 pgWalUsage に加算する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/nbtree/nbtsort.c
  • src/backend/commands/vacuumparallel.c
  • src/backend/executor/execParallel.c
  • **呼び出し先**:
  • BufferUsageAdd
  • WalUsageAdd

関連データ構造

  • BufferUsage: バッファアクセス統計を保持する構造体。
  • WalUsage: WAL 生成量統計を保持する構造体。
  • pgBufferUsage: プロセス全体で累積されるグローバルなバッファ統計カウンタ。
  • pgWalUsage: プロセス全体で累積されるグローバルな WAL 統計カウンタ。

失敗モードと境界条件

  • ポインタ引数が NULL の場合の挙動については、本ソース excerpt 内では定義されていない。

保守担当者向けメモ

  • 本関数は統計の「累積」を行うものであり、リセット機能は持たない。
  • 並列クエリ実行時の統計統合において、リーダープロセスがワーカーの統計を正しく収集・集約するための重要な接点である。
