hmac_free

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/px-hmac.c:123

目的

PX_HMAC 構造体に関連付けられたリソースを解放し、機密情報を含むメモリ領域を安全に消去します。

シグネチャまたは形状

static void hmac_free(PX_HMAC *h)

入力と出力

  • 入力: PX_HMAC *h （解放対象の HMAC コンテキストへのポインタ）
  • 出力: なし

前提条件と不変条件

  • 引数 h は有効な PX_HMAC 構造体を指している必要があります。
  • 内部のダイジェストコンテキスト h->md およびパディング領域 h->p.ipad, h->p.opad は、palloc 等で確保されている必要があります。

アルゴリズムまたは振る舞い

1. px_md_block_size(h->md) を呼び出し、使用しているハッシュアルゴリズムのブロックサイズを取得します。 2. px_md_free(h->md) を呼び出し、内部のダイジェストコンテキストを解放します。 3. px_memset を使用して、h->p.ipad および h->p.opad のメモリ領域をブロックサイズ分だけゼロクリアします。 4. pfree を使用して、ipad、opad、および PX_HMAC 構造体自体のメモリを解放します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/px-hmac.c 内の px_find_hmac 関数において、PX_HMAC 構造体の free メンバにこの関数ポインタが代入されています。
  • 呼び出し先: px_md_block_size, px_md_free, px_memset, pfree。

関連データ構造

  • PX_HMAC: HMAC 操作の状態を保持する構造体。

失敗モードと境界条件

  • h が NULL の場合、pfree やメンバアクセスによりセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は機密情報（キーや中間ハッシュ）のメモリ漏洩を防ぐため、解放前に px_memset によるゼロクリアを確実に行う必要があります。
  • px_find_hmac で割り当てられたリソースのライフサイクル管理の終端として機能します。
