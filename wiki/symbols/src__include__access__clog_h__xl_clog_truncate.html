xl_clog_truncate

定義

  • 種別: struct
  • 定義箇所: src/include/access/clog.h:32

目的

Commit Log (CLOG) の切り詰め（Truncate）処理が発生した際に、その操作を WAL（Write Ahead Log）に記録するための構造体です。

シグネチャまたは形状

typedef struct xl_clog_truncate
{
    int64       pageno;
    TransactionId oldestXact;
    Oid         oldestXactDb;
} xl_clog_truncate;

入力と出力

  • **入力**: CLOG 切り詰め時に、削除対象となるページ番号（pageno）、および切り詰め後の最も古いトランザクションID（oldestXact）とそのデータベースOID（oldestXactDb）が格納されます。
  • **出力**: WAL レコードの一部としてディスクへ書き込まれます。

前提条件と不変条件

  • pageno は切り詰め対象となる CLOG ページのインデックスです。
  • クラッシュリカバリ時には、この構造体を用いて clog_redo が CLOG ページの再構築を行います。

アルゴリズムまたは振る舞い

1. **WAL 記録**: src/backend/access/transam/clog.c 内の切り詰め処理において、この構造体に値を設定し XLogRegisterData を通じて WAL に書き出します。 2. **リカバリ**: src/backend/access/transam/xlog.c のリドゥ処理において、この構造体を読み取り、初期化以降に変更があった場合に CLOG の状態を同期させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/clog.c: 切り詰め実行時にこの構造体をインスタンス化し、WAL 登録を行います。
  • **呼び出し先**:
  • src/backend/access/rmgrdesc/clogdesc.c: ログのデバッグ出力（記述）のためにこの構造体を参照します。
  • src/backend/access/transam/xlog.c: クラッシュリカバリ時のリドゥ処理でこの構造体を参照します。

関連データ構造

  • TransactionId: トランザクションを識別する型。
  • Oid: データベースを識別する型。
  • XidStatus: CLOG が管理するトランザクション状態の列挙型。

失敗モードと境界条件

  • 構造体のサイズやフィールドの整合性が WAL レコードの読み取り時に一致しない場合、リカバリ処理が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は WAL レコードのバイナリレイアウトに直結するため、フィールドの追加や変更を行う際は、clog_redo および clog_desc（rmgrdesc/clogdesc.c）の更新が必須です。
