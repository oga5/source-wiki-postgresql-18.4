tsvectorin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector.c:174

目的

文字列形式の全文検索データ（tsvector）を解析し、PostgreSQL 内部で利用可能なバイナリ形式の TSVector 構造体へ変換する。

シグネチャまたは形状

Datum tsvectorin(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) により取得される、全文検索用の文字列（lexeme とオプションの position/weight を含む）。
  • **出力**: TSVector 型の Datum。解析失敗時は NULL を返す。

前提条件と不変条件

  • 入力文字列は init_tsvector_parser で解析可能であること。
  • 語彙（lexeme）の長さは MAXSTRLEN 未満であること。
  • 構築される tsvector 全体のサイズは MAXSTRPOS を超えないこと。
  • 内部の語彙は uniqueentry により重複排除およびソート済みであること。

アルゴリズムまたは振る舞い

1. **解析**: init_tsvector_parser を初期化し、gettoken_tsvector を用いてトークン、位置情報、重みを順次抽出する。 2. **一時バッファ構築**: 抽出したトークンを tmpbuf に格納し、WordEntryIN 配列にメタデータを記録する。必要に応じて repalloc でバッファを拡張する。 3. **正規化**: uniqueentry を呼び出し、語彙のソートと重複排除を行う。 4. **バイナリ構築**: TSVector 構造体を割り当て、ヘッダー、語彙文字列、位置情報配列を順次配置する。位置情報が存在する場合は SHORTALIGN でアライメントを調整する。 5. **終了処理**: close_tsvector_parser でパーサーを解放し、構築した TSVector を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（tsvector 型の入力関数として登録）。
  • **呼び出し先**: init_tsvector_parser, gettoken_tsvector, uniqueentry, close_tsvector_parser, palloc, repalloc, pfree。

関連データ構造

  • TSVector: 最終的な内部バイナリ表現。
  • WordEntryIN: 入力処理中の一時的な語彙管理構造体。
  • WordEntry: TSVector 内の各語彙のメタデータ。
  • WordEntryPos: 位置情報と重みを保持する構造体。

失敗モードと境界条件

  • **ERRCODE_PROGRAM_LIMIT_EXCEEDED**: 語彙が MAXSTRLEN を超える場合、または文字列全体が MAXSTRPOS を超える場合に発生。
  • **メモリ不足**: palloc/repalloc 失敗時にエラーが発生する。
  • **解析エラー**: gettoken_tsvector がエラーを検知した場合、SOFT_ERROR_OCCURRED により NULL を返す。

保守担当者向けメモ

  • WordEntryPos のビット操作を行う際は tsearch/ts_type.h の定義を参照すること。
  • 大規模な入力に対しては repalloc が頻発するため、メモリ断片化に注意が必要。
  • uniqueentry によるソート後のレイアウト変更が TSVector の整合性に直結するため、変更時は uniqueentry の仕様を再確認すること。
