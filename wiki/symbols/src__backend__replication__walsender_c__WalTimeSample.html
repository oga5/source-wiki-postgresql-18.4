WalTimeSample

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/walsender.c:216

目的

WAL（Write-Ahead Log）の特定の時点における LSN（Log Sequence Number）と、その時点のタイムスタンプをペアで保持し、レプリケーションのラグ追跡を行うためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	XLogRecPtr	lsn;
	TimestampTz time;
} WalTimeSample;

入力と出力

  • 入力: XLogRecPtr 型のログ位置情報と TimestampTz 型の時刻情報。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • LagTracker 構造体の一部として使用され、レプリケーションの遅延計算に利用されます。
  • lsn は WAL ストリーム内の有効な位置を指す必要があります。

アルゴリズムまたは振る舞い

  • 本構造体自体はデータ保持に特化しており、LagTracker 内のバッファや last_read、overflowed 配列の要素として、時系列のログ位置と時刻の履歴を記録するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/walsender.c 内のラグ追跡ロジック（lag_tracker 関連の処理）から参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • LagTracker: WalTimeSample を配列として保持し、ラグ計算を管理する構造体。
  • XLogRecPtr: WAL 上の論理位置を示す型。
  • TimestampTz: PostgreSQL のタイムスタンプ型。

失敗モードと境界条件

  • LAG_TRACKER_BUFFER_SIZE を超えるデータが投入された場合、バッファの循環やオーバーフロー管理が必要となります。

保守担当者向けメモ

  • LagTracker の実装において、WalTimeSample の配列がどのように更新・参照されているかを確認してください。特に head や read_heads を用いたインデックス操作が、ラグ計算の精度に直結します。
