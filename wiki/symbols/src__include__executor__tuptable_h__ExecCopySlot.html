ExecCopySlot

定義

  • 種別: function
  • 定義箇所: src/include/executor/tuptable.h:524

目的

ソーススロット (srcslot) の内容を宛先スロット (dstslot) にコピーします。

シグネチャまたは形状

static inline TupleTableSlot *ExecCopySlot(TupleTableSlot *dstslot, TupleTableSlot *srcslot)

入力と出力

  • **入力**:
  • dstslot: コピー先の TupleTableSlot。
  • srcslot: コピー元の TupleTableSlot。
  • **出力**: コピー後の dstslot を返します。

前提条件と不変条件

  • srcslot は空であってはなりません (!TTS_EMPTY(srcslot))。
  • srcslot と dstslot は同一のオブジェクトであってはなりません。
  • 両スロットの TupleDesc が保持する属性数 (natts) は一致している必要があります。

アルゴリズムまたは振る舞い

1. 入力スロットの有効性および属性数の一致をアサーションで検証します。 2. dstslot の tts_ops（スロット操作関数テーブル）に定義された copyslot 関数を呼び出し、スロットの実装型に応じたコピー処理を実行します。 3. コピー先の dstslot を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/commands/copyfrom.c
  • src/backend/commands/trigger.c
  • src/backend/executor/execExprInterp.c
  • src/backend/executor/execMain.c
  • src/backend/executor/execReplication.c
  • src/backend/executor/nodeCtescan.c
  • src/backend/executor/nodeGroup.c
  • **呼び出し先**: dstslot->tts_ops->copyslot（スロット型に応じた実装関数）。

関連データ構造

  • TupleTableSlot: タプルを保持する基本構造体。
  • TupleTableSlotOps: スロットの挙動を定義する関数ポインタテーブル。

失敗モードと境界条件

  • 前提条件を満たさない場合（空のスロットのコピー、属性数不一致など）、アサーションによりプログラムが停止する可能性があります。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、スロットの具体的なコピーロジックは tts_ops を介して動的に決定されます。特定のタプル形式（HeapTuple, MinimalTuple等）に依存しない汎用的なコピーインターフェースです。
