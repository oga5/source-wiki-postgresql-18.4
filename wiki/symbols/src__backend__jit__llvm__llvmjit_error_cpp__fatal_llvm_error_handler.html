fatal_llvm_error_handler

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_error.cpp:133

目的

LLVM内部で発生した致命的なエラーを捕捉し、PostgreSQLのプロセス管理と整合する形で安全に終了させるためのエラーハンドラです。

シグネチャまたは形状

static void fatal_llvm_error_handler(void *user_data, const char *reason, bool gen_crash_diag);

入力と出力

  • user_data: LLVMから渡されるユーザー定義データ（本実装では使用されません）。
  • reason: エラー内容を示す文字列。
  • gen_crash_diag: クラッシュ診断情報を生成するかどうかを示すフラグ。
  • 出力: なし（ereport(FATAL, ...) によりプロセスが終了するため、呼び出し元には戻りません）。

前提条件と不変条件

  • 本関数は、LLVMの install_fatal_error_handler を介して登録されることを前提としています。
  • LLVMの内部状態が回復不能な状態にある際に呼び出されます。

アルゴリズムまたは振る舞い

1. 受け取った reason を含むメッセージで ereport(FATAL, ...) を発行します。 2. エラーコードとして ERRCODE_OUT_OF_MEMORY を指定し、PostgreSQLのFATALエラーとして処理を中断します。 3. プロセスを終了させることで、LLVMが直接 exit() や abort() を呼び出すことを防ぎます。

呼び出し元と呼び出し先

  • 呼び出し元: LLVMライブラリ内部の致命的エラー発生時に、登録されたハンドラとして呼び出されます。
  • 呼び出し先: PostgreSQLの ereport マクロ（FATALレベル）。

関連データ構造

  • なし。

失敗モードと境界条件

  • 本関数自体が失敗することはありませんが、ereport(FATAL, ...) が呼び出されるため、呼び出し元の処理は即座に停止します。

保守担当者向けメモ

  • 本関数はC++で記述されていますが、PostgreSQLのC言語ベースのエラーシステムと連携します。
  • LLVMの内部状態を破壊する可能性があるため、ERROR ではなく FATAL を使用してプロセスを終了させる設計になっています。
