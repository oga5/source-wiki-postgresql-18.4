gbt_enumgt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_enum.c:32

目的

PostgreSQL の列挙型（ENUM）に対する GiST インデックスの「より大きい（greater than）」比較操作をサポートします。

シグネチャまたは形状

static bool gbt_enumgt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の ENUM 値（内部的に Oid として扱われるポインタ）。
  • flinfo: 関数呼び出し情報を保持する FmgrInfo 構造体。
  • **出力**:
  • a > b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は有効な Oid を指している必要があります。
  • flinfo は enum_gt 関数を呼び出すために適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. a と b が指す値を Oid として取得します。 2. CallerFInfoFunctionCall2 を介して、PostgreSQL 標準の enum_gt 関数を呼び出します。 3. enum_gt の結果（Datum）を DatumGetBool で論理値に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_enum.c 内の tinfo 構造体（gbtree_ninfo）の初期化時に参照されます。
  • **呼び出し先**: PostgreSQL 標準の enum_gt 関数。

関連データ構造

  • tinfo: gbtree_ninfo 構造体。本関数を比較用コールバックとして保持します。
  • Oid: ENUM 型の内部表現。

失敗モードと境界条件

  • a または b が NULL ポインタである場合、メモリ保護違反が発生する可能性があります。
  • flinfo が無効な場合、CallerFInfoFunctionCall2 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で定義された汎用 GiST ユーティリティからコールバックとして呼び出されることを前提としています。
  • ENUM 型の順序は内部的な OID 値に依存するため、OID の比較ロジックと密接に関連しています。
