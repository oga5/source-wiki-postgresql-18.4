Regis

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/dicts/regis.h:32

目的

PostgreSQL の全文検索（tsearch）において、ISpell 辞書等で使用される高速な正規表現サブセット（文字集合およびその否定）を保持するための構造体です。

シグネチャまたは形状

typedef struct Regis
{
	RegisNode  *node;
	uint32
				issuffix:1,
				nchar:16,
				unused:15;
} Regis;

入力と出力

  • node: パターンの各要素を表現する RegisNode リンクリストへのポインタ。
  • issuffix: サフィックス判定を行うかを示すフラグ。
  • nchar: パターンに含まれる文字数。

前提条件と不変条件

  • RS_compile によって初期化される必要があります。
  • 動的に割り当てられた RegisNode リストを含むため、使用後は RS_free を呼び出してメモリを解放する必要があります。

アルゴリズムまたは振る舞い

  • RegisNode のリンクリストを介して、文字列の包含（RSF_ONEOF）または除外（RSF_NONEOF）のパターンマッチングを管理します。
  • RS_execute は、この構造体に格納されたパターンに基づき、対象文字列が適合するかを判定します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/tsearch/regis.c: RS_compile, RS_free, RS_execute 等で操作されます。
  • src/include/tsearch/dicts/spell.h: SPELL 構造体等のメンバとして保持されます。
  • 呼び出し先:
  • RegisNode 構造体（node メンバを通じて参照）。

関連データ構造

  • RegisNode: パターンの各要素を定義するリンクリスト構造体。

失敗モードと境界条件

  • パターンが長すぎる場合、リンクリストの走査によりマッチングコストが増大します。
  • 汎用的な正規表現エンジンではないため、サポート対象外の構文を含む文字列を渡した際の挙動は RS_compile の実装に依存します。

保守担当者向けメモ

  • 本構造体は RS_compile で生成された後、RS_free で明示的に解放される必要があります。
  • issuffix フラグや nchar はビットフィールドとして定義されており、メモリ効率が考慮されています。
