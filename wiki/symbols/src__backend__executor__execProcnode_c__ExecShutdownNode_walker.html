ExecShutdownNode_walker

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execProcnode.c:777

目的

プランツリーを再帰的に走査し、特定の実行ノード（Gather、ForeignScan、CustomScan、GatherMerge、Hash、HashJoin）に対してシャットダウン処理を適用する。

シグネチャまたは形状

static bool ExecShutdownNode_walker(PlanState *node, void *context)

入力と出力

  • **入力**:
  • node: 走査対象の PlanState ノード。
  • context: 走査コンテキスト（現在は使用されていない）。
  • **出力**:
  • bool: 常に false を返し、走査を継続する。

前提条件と不変条件

  • node が NULL の場合は即座に終了する。
  • check_stack_depth() によりスタック深さのチェックを行う。
  • ノードが一度でも実行された場合（node->instrument->running が真の場合）、シャットダウン処理中に InstrStartNode および InstrStopNode を呼び出し、実行統計情報を適切に記録する。

アルゴリズムまたは振る舞い

1. node が NULL であれば false を返す。 2. スタック深さを確認する。 3. ノードが実行中であれば、統計計測を開始する。 4. planstate_tree_walker を呼び出し、子ノードに対して再帰的に本関数を適用する。 5. nodeTag(node) に基づき、以下のノードタイプに対して専用のシャットダウン関数を呼び出す。

  • T_GatherState -> ExecShutdownGather
  • T_ForeignScanState -> ExecShutdownForeignScan
  • T_CustomScanState -> ExecShutdownCustomScan
  • T_GatherMergeState -> ExecShutdownGatherMerge
  • T_HashState -> ExecShutdownHash
  • T_HashJoinState -> ExecShutdownHashJoin

    • 6. ノードが実行中であった場合、InstrStopNode を呼び出し、0タプルとして計測を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内の ExecShutdownNode から呼び出される。
  • **呼び出し先**:
  • planstate_tree_walker（再帰呼び出し）
  • ExecShutdownGather、ExecShutdownForeignScan、ExecShutdownCustomScan、ExecShutdownGatherMerge、ExecShutdownHash、ExecShutdownHashJoin

関連データ構造

  • PlanState: 実行ノードの状態を保持する構造体。
  • Instrumentation: ノードの実行統計情報を保持する構造体。

失敗モードと境界条件

  • ノードが一度も実行されていない場合、統計情報の誤った更新を避けるため、InstrStartNode / InstrStopNode の呼び出しはスキップされる。

保守担当者向けメモ

  • 新しいノードタイプでシャットダウン時にリソース解放が必要な場合は、switch 文にケースを追加すること。
  • planstate_tree_walker を使用しているため、ツリー構造の変更には注意が必要である。
