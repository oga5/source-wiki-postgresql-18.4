setitimer

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/timer.c:85

目的

Windows環境においてPOSIX準拠の setitimer インターフェースをエミュレートし、ITIMER_REAL タイマー機能を提供します。

シグネチャまたは形状

int setitimer(int which, const struct itimerval *value, struct itimerval *ovalue);

入力と出力

  • **入力**:
  • which: タイマーの種類（ITIMER_REAL のみサポート）。
  • value: 設定するタイマー値。
  • ovalue: 前回のタイマー設定を格納するポインタ（NULL可）。
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時は ereport(FATAL, ...) によりプロセスが終了します。

前提条件と不変条件

  • value は NULL であってはなりません。
  • value->it_interval（インターバル値）は 0 である必要があります（インターバルタイマーは未サポート）。
  • which は ITIMER_REAL である必要があります。
  • 初回呼び出し時にタイマー管理用スレッドとイベントオブジェクトが生成されます。

アルゴリズムまたは振る舞い

1. **初期化**: 初回呼び出し時、CreateEvent でイベントを作成し、CreateThread で pg_timer_thread を起動します。CRITICAL_SECTION を初期化します。 2. **設定更新**: CRITICAL_SECTION を用いて排他制御を行い、timerCommArea.value を更新します。ovalue が指定されている場合は、更新前の値をコピーします。 3. **通知**: SetEvent を呼び出し、待機中のタイマー管理スレッドを起床させて設定変更を伝達します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/fork_process.c
  • src/backend/utils/misc/timeout.c
  • （注: 上記はツールによる参照候補であり、直接的な呼び出し関係の全容は不明です）
  • **呼び出し先**:
  • CreateEvent, CreateThread, InitializeCriticalSection, EnterCriticalSection, LeaveCriticalSection, SetEvent (Win32 API)
  • pg_timer_thread (管理スレッド関数)
  • ereport (PostgreSQLエラー報告)

関連データ構造

  • timerCommArea: タイマー設定と同期用オブジェクトを保持する共有構造体。
  • itimerval: POSIXタイマー設定構造体。

失敗モードと境界条件

  • **FATALエラー**: イベント作成またはスレッド作成に失敗した場合、ereport(FATAL, ...) が発行され、バックエンドプロセスが停止します。
  • **制限事項**: ITIMER_REAL 以外のタイマー種別や、インターバルタイマー（it_interval）の使用はアサーションにより禁止されています。

保守担当者向けメモ

  • 本実装はミリ秒単位の精度を持つ WaitForSingleObjectEx に依存しています。
  • timerCommArea へのアクセスは CRITICAL_SECTION で保護されていますが、デッドロックやイベントの取りこぼしには注意が必要です。
