AggStatePerAggData

定義

  • 種別: struct
  • 定義箇所: src/include/executor/nodeAgg.h:187

目的

集約関数（Aggregate）の最終計算（Finalization）フェーズに必要なメタデータを保持する構造体です。中間状態から最終的な集約結果を生成するための情報を管理します。

シグネチャまたは形状

typedef struct AggStatePerAggData {
    Aggref     *aggref;
    int         transno;
    Oid         finalfn_oid;
    FmgrInfo    finalfn;
    int         numFinalArgs;
    List       *aggdirectargs;
    int16       resulttypeLen;
    bool        resulttypeByVal;
    bool        shareable;
} AggStatePerAggData;

入力と出力

  • **入力**: 遷移状態（transno で指定されるインデックス）および必要に応じて aggdirectargs（順序付き集合集約の直接引数）。
  • **出力**: finalfn を適用した結果。resulttypeLen および resulttypeByVal に基づき、メモリ上のコピーや削除が制御されます。

前提条件と不変条件

  • finalfn（FmgrInfo）は、finalfn_oid が InvalidOid でない場合にのみ有効です。
  • numFinalArgs は常に 1 以上（遷移状態値）であり、必要に応じて集約入力列に対応する NULL 値が追加されます。
  • shareable が false の場合、最終関数が読み書き両用であるため、他の集約と状態値を共有できません。

アルゴリズムまたは振る舞い

1. 集約の最終化段階において、finalfn_oid が指定されていれば finalfn を呼び出します。 2. transno を参照して、対応する遷移状態を取得します。 3. resulttypeLen および resulttypeByVal を用いて、計算結果のデータ型に応じたメモリ管理（コピーや解放）を行います。 4. 複数の同一 Aggref が存在する場合、この構造体は最初の Aggref を指し、共有されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeAgg.c 内の集約実行ロジック（palloc0 を用いてメモリ確保される箇所など）。
  • **呼び出し先**: FmgrInfo を介した集約関数（finalfn）。
  • 注: 呼び出し元・呼び出し先の詳細な関数呼び出しグラフは、提供されたソース excerpt には含まれていません。

関連データ構造

  • Aggref: 集約式そのものを表すノード。
  • AggStatePerTransData: 遷移関数（transfn）実行に必要な情報を保持する対となる構造体。
  • FmgrInfo: 関数呼び出しのための管理情報。

失敗モードと境界条件

  • finalfn_oid が InvalidOid の場合、最終関数は呼び出されません。
  • shareable が false の場合、状態の共有が制限されるため、メモリ消費量が増加する可能性があります。

保守担当者向けメモ

  • AggStatePerTransData と分離されているのは、複数の集約関数が同一の遷移状態を共有できるようにするためです。この設計によりメモリ効率が最適化されています。
  • 実際のロジック（transfn や finalfn の呼び出し）については、nodeAgg.c の ExecAgg 関数を参照してください。
