get_collation

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_utilcmd.c:2176

目的

指定された照合順序（collation）のスキーマ修飾名を取得する。

シグネチャまたは形状

List *get_collation(Oid collation, Oid actual_datatype)

入力と出力

  • **入力**:
  • collation: 取得対象の照合順序の OID。
  • actual_datatype: 照合順序が適用されるデータ型の OID。
  • **出力**:
  • スキーマ名と照合順序名を要素として持つ List（makeString で生成された文字列のリスト）。照合順序が指定されていない場合や、データ型のデフォルトと一致する場合は NIL を返す。

前提条件と不変条件

  • collation が有効な OID である必要がある。
  • pg_collation システムカタログへのアクセス権限が必要。

アルゴリズムまたは振る舞い

1. collation が無効（OidIsValid が偽）であれば NIL を返す。 2. get_typcollation(actual_datatype) を呼び出し、対象データ型のデフォルト照合順序と一致するか確認する。一致すれば NIL を返す。 3. SearchSysCache1 を使用して pg_collation システムカタログから該当するタプルを検索する。 4. 検索に失敗した場合は elog(ERROR) で異常終了する。 5. 取得したタプルから名前空間（スキーマ）の OID を抽出し、get_namespace_name でスキーマ名を取得する。 6. 照合順序名を取得し、スキーマ名と照合順序名をリストに格納して返す。 7. 最後に ReleaseSysCache でキャッシュタプルを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_utilcmd.c:1930（インデックス定義の解析処理内）
  • **呼び出し先**:
  • get_typcollation

関連データ構造

  • Form_pg_collation: pg_collation カタログの行構造体。
  • List: PostgreSQL のリスト構造。

失敗モードと境界条件

  • collation が無効な場合: NIL を返す。
  • actual_datatype のデフォルト照合順序と一致する場合: NIL を返す。
  • システムキャッシュで照合順序が見つからない場合: elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は、DDL 変換時に照合順序を明示的に指定する必要がある場合に呼び出される。
  • 常にスキーマ修飾された名前を返す設計となっている。
