output_type

定義

  • 種別: struct
  • 定義箇所: contrib/pgstattuple/pgstatapprox.c:32

目的

ヒープテーブルの統計情報の推定結果を保持するためのデータ構造です。pgstattuple_approx 関連の処理において、スキャンおよび外挿によって得られた各種メトリクスを格納するために使用されます。

シグネチャまたは形状

typedef struct output_type
{
	uint64		table_len;
	double		scanned_percent;
	uint64		tuple_count;
	uint64		tuple_len;
	double		tuple_percent;
	uint64		dead_tuple_count;
	uint64		dead_tuple_len;
	double		dead_tuple_percent;
	uint64		free_space;
	double		free_percent;
} output_type;

入力と出力

本構造体は関数への出力引数として渡され、統計値が書き込まれます。

  • **入力**: statapprox_heap 関数等において、初期化された output_type 構造体へのポインタが渡されます。
  • **出力**: 構造体の各フィールドに、テーブルの物理サイズ、タプル数、デッドタプル数、空き容量などの推定値が格納されます。

前提条件と不変条件

  • statapprox_heap に渡される前に、呼び出し元で {0} 等によるゼロ初期化が行われることが一般的です。
  • 構造体内の各フィールドは、ヒープスキャンおよび vac_estimate_reltuples による外挿計算の結果を反映します。

アルゴリズムまたは振る舞い

本構造体は、statapprox_heap 内で以下のプロセスを経て値が更新されます。 1. 可視性マップ（Visibility Map）に基づき、ページ単位でスキャンまたはスキップを判定。 2. スキャンしたページについては実測値を加算。 3. スキップしたページについてはフリースペースマップ（FSM）および外挿ロジックを用いて推定値を加算。 4. 最終的に、全ページに対する統計の合計値が本構造体に保持されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgstattuple/pgstatapprox.c 内の statapprox_heap 関数が引数として受け取り、値を設定します。また、同ファイル内の関数でローカル変数として宣言されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。
  • **注意**: contrib/test_decoding や src/backend/replication 等で言及されている output_type は、名前が同一ですが本構造体とは無関係な別定義です。

関連データ構造

  • Relation: ヒープスキャン対象のリレーション情報。
  • HeapTuple: 統計算出の過程で個々のタプルを評価する際に参照されます。

失敗モードと境界条件

  • 構造体内の各フィールドは uint64 または double であり、極端に巨大なテーブルや統計値に対してはオーバーフローの可能性がありますが、PostgreSQLの標準的な統計処理の範囲内で設計されています。
  • 一時テーブル（RELATION_IS_OTHER_TEMP）に対しては、本構造体を用いた統計取得処理自体が拒否されます。

保守担当者向けメモ

  • 本構造体は pgstatapprox.c 専用の内部構造体です。他のモジュールにある同名のシンボルと混同しないよう注意してください。
  • 統計項目を追加・変更する場合は、statapprox_heap 内の集計ロジックおよび外挿計算ロジックとの整合性を確認してください。
