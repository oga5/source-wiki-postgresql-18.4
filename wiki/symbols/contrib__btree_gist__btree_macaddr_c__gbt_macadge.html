gbt_macadge

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_macaddr.c:35

目的

macaddr 型の「以上（greater than or equal to）」比較をGiSTインデックスの検索処理で利用可能にするためのラッパー関数です。

シグネチャまたは形状

bool gbt_macadge(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**: 比較対象となる2つの macaddr 型へのポインタ（a, b）および関数マネージャ情報（flinfo）。
  • **出力**: a >= b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は有効な macaddr 型のデータである必要があります。
  • 内部で macaddr_ge を呼び出すため、PostgreSQLの標準的な macaddr 比較ロジックに依存します。

アルゴリズムまたは振る舞い

1. PointerGetDatum を使用して、入力ポインタをPostgreSQLの Datum 形式に変換します。 2. DirectFunctionCall2 を介して、標準の macaddr_ge 関数を呼び出します。 3. 戻り値を DatumGetBool で論理値に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: macaddr_ge（PostgreSQL標準の比較関数）。
  • **呼び出し元**: 同一ファイル内の静的構造体 tinfo（gbtree_ninfo 型）から参照されています。具体的な実行時の呼び出し元は、btree_utils_num.c 内の汎用GiSTユーティリティ関数です。

関連データ構造

  • tinfo: macaddr 型の比較関数を保持する設定テーブル。
  • macaddr: PostgreSQLのネットワークアドレス型。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、PostgreSQLの基盤関数に依存した動作となります。
  • macaddr_ge の実行時にエラーが発生した場合、その例外がそのまま伝播します。

保守担当者向けメモ

  • 本関数は btree_utils_num が期待するコールバックシグネチャに合わせて実装されています。
  • macaddr8 型など、他の型に対する同様の関数と一貫した設計になっています。
