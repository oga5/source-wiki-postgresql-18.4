mbms_overlap_sets

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/multibitmapset.c:145

目的

2つのマルチビットマップセット（List の各要素が Bitmapset である構造）を比較し、共通部分を持つリストインデックスの集合を Bitmapset として抽出する。

シグネチャまたは形状

Bitmapset *mbms_overlap_sets(const List *a, const List *b)

入力と出力

  • **入力**:
  • a: 比較対象のマルチビットマップセット（List *）
  • b: 比較対象のマルチビットマップセット（List *）
  • **出力**:
  • 共通部分が存在するリストインデックスをビットとして保持する Bitmapset *。共通部分がない場合は NULL を返す。

前提条件と不変条件

  • 引数 a および b は、List の各要素が Bitmapset であることを前提とする。
  • リストの長さが異なる場合、短い方のリストの長さに合わせて処理が終了する。

アルゴリズムまたは振る舞い

1. result を NULL で初期化する。 2. forboth マクロを使用して、リスト a と b を同時に走査する。 3. 各ステップで、対応する Bitmapset (bmsa, bmsb) を取得する。 4. bms_overlap(bmsa, bmsb) を呼び出し、両者に共通ビットが存在するか判定する。 5. 共通部分がある場合、現在のリストインデックスを bms_add_member を用いて result に追加する。 6. 走査終了後、result を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/prep/prepjointree.c:3420 にて mbms_overlap_sets(nonnullable_vars, forced_null_vars) として呼び出されている。
  • **呼び出し先**:
  • lfirst_node (マクロ)
  • bms_overlap
  • foreach_current_index (マクロ)
  • bms_add_member

関連データ構造

  • List: マルチビットマップセットのコンテナ。
  • Bitmapset: 各リスト要素および戻り値として使用されるビット集合。

失敗モードと境界条件

  • 両方のリストが空（NIL）の場合、result は NULL のまま返される。
  • リストの長さが異なる場合、短い方のリストの終端で処理が停止する。
  • リスト要素が NULL である場合、lfirst_node の呼び出しによりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はインプレース更新を行わず、新しい Bitmapset を生成して返す。
  • forboth の性質上、リストの長さが不一致の場合の挙動は「短い方に合わせる」仕様である点に注意すること。
