fsm_get_heap_blk

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/freespace.c:515

目的

FSM（Free Space Map）の最下層（リーフ）レベルにおける論理アドレスとスロット番号から、対応するヒープページの物理ブロック番号を算出する。

シグネチャまたは形状

static BlockNumber fsm_get_heap_blk(FSMAddress addr, uint16 slot)

入力と出力

  • **入力**:
  • addr: FSMAddress 型。対象となるFSMページのアドレス。
  • slot: uint16 型。FSMページ内のスロット番号。
  • **出力**:
  • BlockNumber: 計算されたヒープ内の物理ブロック番号。

前提条件と不変条件

  • addr.level が FSM_BOTTOM_LEVEL であること（アサーションにより検証される）。

アルゴリズムまたは振る舞い

1. addr.level が最下層であることを確認する。 2. addr.logpageno（論理ページ番号）を unsigned int にキャストし、SlotsPerFSMPage（1ページあたりのスロット数）を乗算する。 3. その結果に slot を加算して、対応するヒープブロック番号を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/freespace/freespace.c 内の関数（行174および行731での呼び出しを確認）。
  • **呼び出し先**:
  • 特になし（算術演算のみを実行）。

関連データ構造

  • FSMAddress: FSM木構造上の位置を示す論理アドレス構造体。
  • SlotsPerFSMPage: FSMページ内に格納されるスロットの定数（storage/fsm_internals.h で定義）。

失敗モードと境界条件

  • addr.level が FSM_BOTTOM_LEVEL 以外の場合、アサーション違反によりプロセスが停止する可能性がある。

保守担当者向けメモ

  • 本関数はFSMの論理構造から物理的なヒープブロックへのマッピングを行う低レベルな変換関数である。
  • SlotsPerFSMPage の値が変更された場合、計算結果に直接影響するため注意が必要である。
