searchChar

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgtextproc.c:157

目的

SP-GiST インデックスのノードラベル配列から、指定された文字（int16 型）を二分探索で検索します。

シグネチャまたは形状

static bool searchChar(Datum *nodeLabels, int nNodes, int16 c, int *i)

入力と出力

  • **入力**:
  • nodeLabels: 検索対象となる Datum 型のノードラベル配列。
  • nNodes: 配列内の要素数。
  • c: 検索対象の文字コード。
  • i: 検索結果のインデックスを格納するポインタ。
  • **出力**:
  • 戻り値: 指定した文字が見つかった場合は true、見つからなかった場合は false。
  • *i: 文字が見つかった場合はその位置、見つからなかった場合は挿入すべき位置（StopHigh）が設定されます。

前提条件と不変条件

  • nodeLabels は昇順にソートされている必要があります。
  • nNodes は配列の有効な要素数である必要があります。

アルゴリズムまたは振る舞い

1. StopLow（0）と StopHigh（nNodes）を境界として二分探索を実行します。 2. 中間値 middle を計算し、検索対象 c と比較します。

  • c < middle の場合、探索範囲を前半に絞ります。
  • c > middle の場合、探索範囲を後半に絞ります。
  • 一致した場合、*i に位置を格納し true を返します。

    • 3. ループ終了後、一致しなかった場合は *i に StopHigh を格納し false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgtextproc.c 内の関数（spg_text_choose 等でのノード選択時）。
  • src/test/modules/spgist_name_ops/spgist_name_ops.c（テストモジュール）。
  • **呼び出し先**:
  • DatumGetInt16 マクロ（Datum から int16 への変換）。

関連データ構造

  • Datum: PostgreSQL の汎用データ型。ここではノードラベルを保持するために使用されます。

失敗モードと境界条件

  • nNodes が 0 の場合、ループに入らず *i に 0 が設定され false が返されます。
  • 配列内に重複するラベルが存在する場合の挙動は、二分探索の標準的な仕様に従います。

保守担当者向けメモ

  • 本関数は spgtextproc.c 内の文字列処理における基本的な探索ロジックです。
  • int16 型の比較を行っているため、ラベルとして扱える文字コードの範囲に注意してください。
