ExecBitmapOr

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapOr.c:42

目的

BitmapOr ノードに対する ExecProcNode インターフェースの呼び出しを拒否し、エラーを発生させるためのスタブ関数です。

シグネチャまたは形状

static TupleTableSlot *ExecBitmapOr(PlanState *pstate);

入力と出力

  • **入力**: pstate (PlanState *) - 実行対象のプラン状態。
  • **出力**: なし（常にエラーを投げるため戻り値は到達しない）。

前提条件と不変条件

  • BitmapOr ノードはタプル単位の取得（ExecProcNode）をサポートしていません。
  • 本関数は、BitmapOr ノードの ExecProcNode フィールドに割り当てられることで、誤った呼び出しを検知するために使用されます。

アルゴリズムまたは振る舞い

1. elog(ERROR, ...) を呼び出し、BitmapOr ノードが ExecProcNode 呼び出し規約をサポートしていない旨のメッセージを出力して処理を中断します。 2. 戻り値として NULL を返しますが、実際にはエラーにより制御が戻ることはありません。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecInitBitmapOr 内で、bitmaporstate->ps.ExecProcNode に本関数が代入されています。
  • **呼び出し先**: elog（エラー報告用）。

関連データ構造

  • PlanState: 実行ノードの共通状態構造体。
  • BitmapOrState: BitmapOr ノードの実行状態を保持する構造体。

失敗モードと境界条件

  • 本関数が呼び出された場合、常にエラーが発生します。これは設計上の意図的な失敗モードです。

保守担当者向けメモ

  • BitmapOr はタプルを直接生成するのではなく、MultiExecBitmapOr を通じてビットマップを統合する仕組みであるため、このノードに対して ExecProcNode を呼び出すことは誤りです。この設計に変更がない限り、本関数の実装を修正する必要はありません。
