resetSpGistScanOpaque

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/spgist/spgscan.c:153

目的

SP-GiST インデックススキャンの状態（SpGistScanOpaque）をリセットし、次回のスキャン実行に向けてメモリとキューを再初期化します。

シグネチャまたは形状

static void resetSpGistScanOpaque(SpGistScanOpaque so)

入力と出力

  • **入力**: so (SpGistScanOpaque) - リセット対象のスキャン状態構造体。
  • **出力**: なし（副作用として so の内部状態が更新されます）。

前提条件と不変条件

  • so->traversalCxt が有効なメモリコンテキストとして存在していること。
  • so が保持するポインタ配列（distances, reconTups）は、メモリリークを防ぐために適切に解放される必要がある。

アルゴリズムまたは振る舞い

1. **メモリリセット**: so->traversalCxt をリセットし、関連するメモリを解放します。 2. **キューの再構築**: so->traversalCxt コンテキスト内で pairingheap を再割り当てします。 3. **探索開始項目の追加**: searchNulls および searchNonNulls フラグに基づき、spgAddStartItem を呼び出して初期探索項目をキューに追加します。 4. **リソース解放**:

  • numberOfOrderBys > 0 の場合、distances 配列内の各要素を pfree します。
  • want_itup が真の場合、reconTups 配列内の各要素を pfree します。

    • 5. **ポインタの初期化**: so->iPtr および so->nPtrs を 0 にリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgscan.c:420 (推測される文脈: spgrescan 内での再初期化)
  • **呼び出し先**:
  • MemoryContextReset
  • MemoryContextSwitchTo
  • pairingheap_allocate
  • spgAddStartItem
  • pfree

関連データ構造

  • SpGistScanOpaque: スキャン状態を保持する主要な構造体。
  • pairingheap: KNN 検索時の優先度付きキューとして使用。

失敗モードと境界条件

  • so が NULL の場合、メモリ参照エラーが発生します。
  • traversalCxt が既に破棄されている場合、MemoryContextReset でエラーが発生します。

保守担当者向けメモ

  • 本関数は spgrescan 等から呼び出され、スキャン状態をクリーンな状態に戻します。
  • distances や reconTups の解放はメモリリーク防止に不可欠です。新しいフィールドを SpGistScanOpaque に追加した際は、本関数での解放処理が必要か検討してください。
