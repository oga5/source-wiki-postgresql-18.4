is_shippable

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/shippable.c:161

目的

データベースオブジェクト（関数、演算子、データ型など）が、リモートサーバー上で実行可能（shippable）であるかを判定します。

シグネチャまたは形状

bool is_shippable(Oid objectId, Oid classId, PgFdwRelationInfo *fpinfo)

入力と出力

  • **入力**:
  • objectId: 判定対象オブジェクトのOID。
  • classId: オブジェクトが属するカタログのOID。
  • fpinfo: 外部テーブル情報を含む PgFdwRelationInfo 構造体。
  • **出力**:
  • bool: リモート実行可能であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 組み込みオブジェクト（is_builtin が真を返すもの）は常に true とみなされます。
  • fpinfo->shippable_extensions が空の場合、組み込み以外のオブジェクトはすべて false となります。
  • キャッシュの初期化は初回呼び出し時に行われます。

アルゴリズムまたは振る舞い

1. is_builtin(objectId) を呼び出し、組み込みオブジェクトであれば即座に true を返します。 2. fpinfo->shippable_extensions が NIL であれば false を返します。 3. ShippableCacheHash が未初期化であれば InitializeShippableCache() を実行します。 4. ShippableCacheKey を生成し、ShippableCacheHash を検索します。 5. キャッシュヒット時はその結果を返します。 6. キャッシュミス時は lookup_shippable を呼び出して判定し、その結果をキャッシュに格納してから返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/deparse.c 内の複数の箇所（定数や関数、演算子のプッシュダウン判定時）。
  • **呼び出し先**:
  • is_builtin
  • InitializeShippableCache
  • hash_search
  • lookup_shippable

関連データ構造

  • PgFdwRelationInfo: 外部テーブルのメタデータとサーバー情報。
  • ShippableCacheKey: キャッシュ検索用のキー（objid, classid, serverid）。
  • ShippableCacheEntry: キャッシュに格納される判定結果。
  • ShippableCacheHash: 判定結果を保持するハッシュテーブル。

失敗モードと境界条件

  • fpinfo が無効な場合、ポインタ参照エラーが発生する可能性があります。
  • カタログ検索（lookup_shippable 内）が失敗またはエラーを発生させた場合、呼び出し元に伝播します。
  • キャッシュ無効化コールバックにより、サーバー設定変更時にキャッシュがフラッシュされるため、再判定が行われます。

保守担当者向けメモ

  • 本関数はクエリ最適化のパフォーマンスに直結するため、キャッシュの効率が重要です。
  • lookup_shippable を呼び出す際は、カタログ検索がキャッシュ無効化をトリガーする可能性があるため、キャッシュエントリの作成は必ず判定後に行う必要があります。
