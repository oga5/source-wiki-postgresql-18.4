LookupTypeName

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_type.c:37

目的

SQL文中の型名（TypeName）をシステムカタログ上の型情報（pg_type）へ解決し、対応する Type（HeapTuple）を取得する主要なエントリポイントです。

シグネチャまたは形状

Type LookupTypeName(ParseState *pstate, const TypeName *typeName, int32 *typmod_p, bool missing_ok)

入力と出力

  • **入力**:
  • pstate: 構文解析状態（エラー位置報告に使用）。
  • typeName: 解決対象の型名構造体。
  • typmod_p: 型修飾子（typmod）を格納するポインタ。
  • missing_ok: 型が見つからない場合にエラーを投げるか（false）、NULLを返すか（true）のフラグ。
  • **出力**:
  • 成功時: pg_type システムカタログの HeapTuple（Type 型）。
  • 失敗時: missing_ok が true の場合は NULL。

前提条件と不変条件

  • 呼び出し元は、返された Type（HeapTuple）に対して必ず ReleaseSysCache を実行しなければなりません。
  • typisdefined フラグを確認せずに型情報を利用すると、未定義のシェル型を誤って処理する可能性があるため注意が必要です。

アルゴリズムまたは振る舞い

内部的に LookupTypeNameExtended を呼び出し、以下の処理を委譲します。 1. TypeName の名前リストを解析し、pg_type システムキャッシュを検索します。 2. 型が見つかった場合、typenameTypeMod を介して型修飾子の検証・変換を行います。 3. missing_ok に応じて、型が見つからない場合の挙動（エラー送出またはNULL返却）を制御します。

呼び出し元と呼び出し先

  • **呼び出し先**: LookupTypeNameExtended
  • **呼び出し元（直接的な証拠）**:
  • src/backend/catalog/objectaddress.c
  • src/backend/commands/functioncmds.c
  • src/backend/commands/typecmds.c
  • src/backend/parser/parse_type.c（内部の再帰的または代替的な呼び出し）

関連データ構造

  • TypeName: 構文解析中の型名表現。
  • Type: HeapTuple へのエイリアス。
  • ParseState: 構文解析のコンテキスト。

失敗モードと境界条件

  • missing_ok が false の場合、型が見つからないとエラーが発生します。
  • キャッシュリーク: 呼び出し側で ReleaseSysCache を忘れると、システムキャッシュのメモリリークを引き起こします。

保守担当者向けメモ

  • 本関数は LookupTypeNameExtended のラッパーです。詳細なロジック変更は LookupTypeNameExtended を参照してください。
  • エラー発生時には pstate を通じて parser_errposition が利用されるため、デバッグ時には SQL 文中の位置情報が正確に報告されます。
