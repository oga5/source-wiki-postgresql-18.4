FullTransactionId

定義

  • 種別: struct
  • 定義箇所: src/include/access/transam.h:65

目的

32ビットのトランザクションID（TransactionId）にエポック（上位32ビット）を付与し、64ビットの完全なトランザクション識別子として表現することで、XIDのラップアラウンド問題を内部的に緩和する。

シグネチャまたは形状

typedef struct FullTransactionId
{
	uint64		value;
} FullTransactionId;

入力と出力

  • 入力: 64ビットの符号なし整数 value を保持する構造体。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • value フィールドは、システム全体で一意かつ単調増加するトランザクション識別子として扱われる。
  • 32ビットの TransactionId と比較して、エポック情報を含むため、循環を考慮せずに比較可能である。

アルゴリズムまたは振る舞い

  • 本構造体自体はデータ保持のみを行う。
  • 実際の操作（比較や変換）は、src/include/access/transam.h で提供されるインライン関数やマクロ（FullTransactionIdFromEpochAndXid や FullTransactionIdNewer など）を介して行われる。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • contrib/amcheck/verify_heapam.c: next_fxid, oldest_fxid, relfrozenfxid として使用。また、FullTransactionIdFromXidAndCtx 関数内で利用。
  • contrib/pageinspect/btreefuncs.c: BTPageGetDeleteXid の戻り値として使用。
  • 呼び出し先:
  • 提供されたソース情報からは、本構造体が直接呼び出す関数やシンボルは特定できません。

関連データ構造

  • TransactionId: 32ビットのトランザクション識別子。
  • TransamVariablesData: サーバー全体で共有されるトランザクション生成状態を保持する構造体。

失敗モードと境界条件

  • 64ビットの最大値に達した場合の挙動については、本構造体の定義からは直接読み取れません。
  • 32ビットXIDから変換する際、エポック情報の不整合がある場合は不正なIDが生成される可能性があります。

保守担当者向けメモ

  • 32ビットの古いインターフェースと64ビットの新しいインターフェースが混在しているため、コード修正時は型の一貫性に注意してください。
  • FRONTEND マクロで囲まれた関数群からは参照できない場合があるため、クライアントツール側での利用には制限があります。
