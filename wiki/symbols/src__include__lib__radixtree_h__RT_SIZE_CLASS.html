RT_SIZE_CLASS

定義

  • 種別: enum
  • 定義箇所: src/include/lib/radixtree.h:632

目的

適応型基数木（ART）において、ノードが保持する子ノードの数に応じたメモリレイアウトのサイズクラスを識別するために使用されます。

シグネチャまたは形状

typedef enum RT_SIZE_CLASS
{
    RT_CLASS_4 = 0,
    RT_CLASS_16_LO,
    RT_CLASS_16_HI,
    RT_CLASS_48,
    RT_CLASS_256
} RT_SIZE_CLASS;

入力と出力

  • 入力: なし（列挙型定数）
  • 出力: なし

前提条件と不変条件

  • RT_SIZE_CLASS は、基数木のノードが動的に成長する際のメモリ割り当てサイズを決定する指標として機能します。
  • 各列挙値は、対応するノード構造（RT_NODE_4, RT_NODE_16, RT_NODE_48, RT_NODE_256）と一対一で関連付けられることを前提としています。

アルゴリズムまたは振る舞い

  • 基数木のノードが満杯になった際、より大きな RT_SIZE_CLASS へ移行するための識別子として利用されます。
  • RT_ALLOC_NODE 関数において、ノードの種別（kind）と共に引数として渡され、適切なメモリ領域の確保を制御します。

呼び出し元と呼び出し先

  • 呼び出し元: RT_ALLOC_NODE 関数（src/include/lib/radixtree.h:831）がこの型を引数として受け取ります。
  • 呼び出し先: 情報なし。

関連データ構造

  • RT_NODE_4, RT_NODE_16, RT_NODE_48, RT_NODE_256: 各サイズクラスに対応するノード構造体。
  • RT_RADIX_TREE: この列挙型を利用してノード管理を行うメイン構造体。

失敗モードと境界条件

  • 該当なし。

保守担当者向けメモ

  • 本シンボルはマクロ展開によって生成されるテンプレートの一部であり、RT_MAKE_NAME マクロを通じて名前が修飾されます。
  • RT_CLASS_16_LO と RT_CLASS_16_HI は、16スロットノードの内部表現の差異を吸収するために存在します。
