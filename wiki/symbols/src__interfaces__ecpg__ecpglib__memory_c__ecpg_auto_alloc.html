ecpg_auto_alloc

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/memory.c:100

目的

メモリを動的に確保し、そのポインタをスレッドローカルな自動解放リストに登録することで、ECPGライブラリ内でのメモリ管理を簡素化・自動化します。

シグネチャまたは形状

char *ecpg_auto_alloc(long size, int lineno);

入力と出力

  • **入力**:
  • size: 確保するメモリサイズ（バイト単位）。
  • lineno: エラー発生時に報告されるソースコード上の行番号。
  • **出力**:
  • 成功時: 確保されたメモリ領域へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • 内部で ecpg_alloc を使用しており、メモリ確保失敗時にはECPG固有のエラー処理が実行されます。
  • 確保されたメモリは、後続の ECPGfree_auto_mem 呼び出しによって一括解放される対象となります。

アルゴリズムまたは振る舞い

1. ecpg_alloc(size, lineno) を呼び出し、メモリを確保します。 2. 確保に失敗した場合は NULL を返します。 3. 確保に成功した場合、ecpg_add_mem(ptr, lineno) を呼び出し、スレッドローカルな追跡リストにポインタを追加します。 4. リストへの追加に失敗した場合は、確保したメモリを ecpg_free(ptr) で解放し、NULL を返します。 5. 正常終了時は、確保したメモリへのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/descriptor.c
  • src/interfaces/ecpg/ecpglib/execute.c
  • **呼び出し先**:
  • ecpg_alloc
  • ecpg_add_mem
  • ecpg_free

関連データ構造

  • auto_mem: スレッドローカルなメモリ追跡リストを構成する構造体。

失敗モードと境界条件

  • ecpg_alloc がメモリ不足等で失敗した場合、本関数も NULL を返します。
  • ecpg_add_mem によるリストへの登録が失敗した場合、確保したメモリは即座に解放され、本関数は NULL を返します。

保守担当者向けメモ

  • 本関数は、ECPGが生成するCコードがメモリのライフサイクルを個別に管理しなくて済むように設計されています。
  • 確保したメモリの所有権を移譲したい場合は、ecpg_clear_auto_mem を使用して追跡リストから除外する必要があります。
