encode_varbyte

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginpostinglist.c:114

目的

64ビット整数を可変長バイト（Varbyte）形式にエンコードし、バッファへ書き込みます。GINインデックスにおけるTID（ItemPointer）の差分圧縮を効率化するために使用されます。

シグネチャまたは形状

static void encode_varbyte(uint64 val, unsigned char **ptr)

入力と出力

  • **入力**:
  • val: エンコード対象の64ビット整数。
  • ptr: 書き込み先バッファの現在位置を指すポインタのポインタ。
  • **出力**:
  • ptr: エンコードされたバイト数分だけ前進したバッファ位置を指すように更新されます。

前提条件と不変条件

  • ptr が指すバッファには、エンコード結果を格納するために十分な領域が確保されている必要があります。
  • 差分エンコーディングの性質上、入力される val は非負である必要があります。

アルゴリズムまたは振る舞い

1. val が 0x7F（127）より大きい間、以下の処理を繰り返します。

  • val の下位7ビットを取り出し、最上位ビット（MSB）を1にセットしてバッファに書き込みます（継続フラグ）。
  • val を右に7ビットシフトします。

    • 2. 最後に残った val（0x7F以下）を、MSBを0にしてバッファに書き込みます。 3. 書き込み完了後、ptr を書き込み後の次の位置へ更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginpostinglist.c 内の ginCompressPostingList 等から呼び出されます。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません（低レベルなメモリ書き込み操作のみ）。

関連データ構造

  • GinPostingList: 本関数でエンコードされたデータが格納される構造体。

失敗モードと境界条件

  • **バッファオーバーフロー**: バッファの残り容量が不足している場合、メモリ破壊を引き起こす可能性があります。呼び出し側で十分な領域を確保する必要があります。
  • **入力値**: 負の値を渡した場合の挙動は定義されていません。

保守担当者向けメモ

  • 本関数は7ビット単位の可変長エンコーディングを実装しています。ディスクフォーマットに直結するため、エンコードロジックを変更する場合は decode_varbyte との整合性を維持してください。
