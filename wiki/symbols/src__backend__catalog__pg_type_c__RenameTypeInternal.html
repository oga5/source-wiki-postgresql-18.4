RenameTypeInternal

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_type.c:764

目的

指定されたデータ型の名前を変更し、必要に応じて関連する配列型の名前を再帰的に更新する。

シグネチャまたは形状

void RenameTypeInternal(Oid typeOid, const char *newTypeName, Oid typeNamespace)

入力と出力

  • typeOid: 変更対象の型のOID。
  • newTypeName: 新しい型名。
  • typeNamespace: 型が属する名前空間のOID。
  • 出力: なし（副作用として pg_type カタログを更新）。

前提条件と不変条件

  • typeNamespace は、対象の型の現在の名前空間と一致しなければならない（スキーマ変更は許可されない）。
  • pg_type カタログへの書き込み権限が必要。

アルゴリズムまたは振る舞い

1. pg_type を RowExclusiveLock でオープンし、対象の型情報をキャッシュから取得する。 2. 新しい型名が既に存在するか確認する。存在する場合、それが自動生成された配列型であれば moveArrayTypeName で退避を試みる。それ以外の場合はエラーを報告する。 3. namestrcpy を使用して型名を更新し、CatalogTupleUpdate でカタログを永続化する。 4. InvokeObjectPostAlterHook を呼び出し、変更を通知する。 5. 対象の型が配列型（typarray）を持つ場合、新しい型名に基づいて配列型の名前を生成し、自身を再帰的に呼び出して配列型の名前も更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_type.c (内部再帰呼び出し)
  • src/backend/commands/tablecmds.c
  • src/backend/commands/typecmds.c
  • **呼び出し先**:
  • table_open, table_close
  • SearchSysCacheCopy1, GetSysCacheOid2
  • moveArrayTypeName
  • CatalogTupleUpdate
  • InvokeObjectPostAlterHook
  • makeArrayTypeName (再帰時)

関連データ構造

  • pg_type: 型情報を保持するシステムカタログ。
  • Form_pg_type: pg_type の行構造体。

失敗モードと境界条件

  • typeOid が無効な場合、elog によりエラー終了する。
  • 新しい型名が既に存在し、かつそれが配列型として退避できない場合、ERRCODE_DUPLICATE_OBJECT でエラーを報告する。

保守担当者向けメモ

  • 本関数は pg_type の行を直接更新するため、カタログの整合性に注意すること。
  • 配列型の名前変更は再帰的に行われるため、無限ループや不整合な名前生成が発生しないよう makeArrayTypeName のロジックと連動している点に留意すること。
