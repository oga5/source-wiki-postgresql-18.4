read_pubkey_keyid

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-info.c:37

目的

PGP 公開鍵パケットから鍵識別子（Key ID）を抽出し、それが暗号化に使用可能な鍵であるかを判定する。

シグネチャまたは形状

static int read_pubkey_keyid(PullFilter *pkt, uint8 *keyid_buf);

入力と出力

  • **入力**:
  • pkt: 読み込み対象の PGP パケットストリーム（PullFilter）。
  • keyid_buf: 抽出した 8 バイトの Key ID を格納するためのバッファ。
  • **出力**:
  • int: 成功時は 1（暗号化鍵として特定）、対象外の鍵の場合は 0、エラー発生時は負の値を返す。

前提条件と不変条件

  • keyid_buf は、少なくとも 8 バイトの書き込み先として有効であること。
  • pkt は _pgp_read_public_key および pgp_skip_packet が正常に処理可能な状態であること。

アルゴリズムまたは振る舞い

1. _pgp_read_public_key を呼び出し、パケットから公開鍵情報を解析・抽出する。 2. pgp_skip_packet を呼び出し、秘密鍵部分が存在する場合はそれをスキップする。 3. 抽出した鍵のアルゴリズム（pk->algo）を確認する。

  • ELG_ENCRYPT、RSA_ENCRYPT、RSA_ENCRYPT_SIGN のいずれかであれば、pk->key_id から 8 バイトを keyid_buf にコピーし、1 を返す。
  • それ以外の場合は 0 を返す。

    • 4. 終了時に pgp_key_free を呼び出し、一時的に確保した鍵構造体メモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-info.c 内のパケット解析ループから呼び出される（直接の呼び出し元関数名はソース抜粋からは特定不可）。
  • **呼び出し先**:
  • _pgp_read_public_key
  • pgp_skip_packet
  • pgp_key_free

関連データ構造

  • PullFilter: PGP パケットの読み込みストリーム。
  • PGP_PubKey: 公開鍵の属性（アルゴリズムや Key ID）を保持する構造体。

失敗モードと境界条件

  • _pgp_read_public_key または pgp_skip_packet がエラー（負の値）を返した場合、そのエラーコードをそのまま戻り値として終了する。
  • 鍵構造体のメモリ確保や解析に失敗した場合、適切にクリーンアップが行われる。

保守担当者向けメモ

  • 本関数は暗号化鍵の特定に特化しており、署名専用鍵などは対象外（0 を返す）として扱われる点に注意すること。
  • keyid_buf のバッファオーバーランを防ぐため、呼び出し側で 8 バイト以上の領域が確保されていることを保証する必要がある。
