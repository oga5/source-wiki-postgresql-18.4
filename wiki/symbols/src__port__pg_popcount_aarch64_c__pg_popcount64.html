pg_popcount64

定義

  • 種別: function
  • 定義箇所: src/port/pg_popcount_aarch64.c:308

目的

64ビット整数に含まれるセットビット（1のビット）の数をカウントします。AArch64アーキテクチャにおいて、Neon命令セットを活用して効率的に計算を行います。

シグネチャまたは形状

int pg_popcount64(uint64 word)

入力と出力

  • **入力**: uint64 word（カウント対象の64ビット整数）
  • **出力**: int（入力値に含まれる1のビット数）

前提条件と不変条件

  • 本関数は POPCNT_AARCH64 が定義されている環境でのみ有効です。
  • 入力値は uint64 型として扱われます。

アルゴリズムまたは振る舞い

1. 入力された word を vld1_u8 を用いてNeonレジスタにロードします。 2. vcnt_u8 命令により、各バイト内のビット数をカウントします。 3. vaddv_u8 命令により、カウントされた各バイトの合計を算出し、結果を返します。 4. コンパイラが __builtin_popcountl() をサポートしている場合、同等のNeon命令列に最適化されることが期待されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/acl.c
  • src/include/nodes/bitmapset.h (マクロ bmw_popcount 経由)
  • src/port/pg_bitutils.c (関数ポインタ経由)
  • **呼び出し先**:
  • vld1_u8, vcnt_u8, vaddv_u8 (ARM Neon組み込み関数)

関連データ構造

  • なし

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。入力されたすべての64ビット値に対して、正確なビットカウントを返します。

保守担当者向けメモ

  • 本関数は src/port/pg_popcount_aarch64.c 内で定義されていますが、src/port/pg_bitutils.c で定義される関数ポインタ pg_popcount64 を通じて動的に呼び出される設計になっています。
  • コンパイル環境やCPU機能に応じて、より高度な最適化（SVEなど）が適用される可能性があるため、変更時は pg_popcount_aarch64.c 全体のディスパッチロジックと整合性を確認してください。
