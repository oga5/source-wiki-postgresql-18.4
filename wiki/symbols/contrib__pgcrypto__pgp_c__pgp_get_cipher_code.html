pgp_get_cipher_code

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp.c:114

目的

OpenPGP 暗号アルゴリズムの名称（文字列）を、内部で使用される数値コードへ変換します。

シグネチャまたは形状

int pgp_get_cipher_code(const char *name)

入力と出力

  • **入力**: name (暗号アルゴリズム名を表す文字列)
  • **出力**: 成功時は対応する内部コード（int）、該当するアルゴリズムが見つからない場合は PXE_PGP_UNSUPPORTED_CIPHER を返します。

前提条件と不変条件

  • cipher_list が有効な cipher_info 構造体の配列として定義されている必要があります。
  • 比較は pg_strcasecmp を用いて大文字小文字を区別せずに行われます。

アルゴリズムまたは振る舞い

1. 静的に定義された cipher_list 配列を走査します。 2. 各要素の name フィールドと引数の name を pg_strcasecmp で比較します。 3. 一致するものが見つかった場合、その要素の code を返します。 4. 配列の終端（name が NULL）に達するまで一致が見つからなかった場合、エラーコード PXE_PGP_UNSUPPORTED_CIPHER を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-pgsql.c: ユーザー指定の暗号アルゴリズム設定時に呼び出されます。
  • contrib/pgcrypto/pgp.c: 内部的なアルゴリズム解決処理（309行目、320行目）で呼び出されます。
  • **呼び出し先**:
  • pg_strcasecmp（PostgreSQL の文字列比較関数）

関連データ構造

  • struct cipher_info: 暗号アルゴリズムの名称と内部コードを保持する構造体。

失敗モードと境界条件

  • 引数 name に一致するアルゴリズムが定義されていない場合、PXE_PGP_UNSUPPORTED_CIPHER が返されます。
  • cipher_list が空、または終端が正しく設定されていない場合、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • サポートする暗号アルゴリズムを追加する場合は、cipher_list 配列を更新してください。
  • 本関数は文字列ベースの検索を行うため、パフォーマンスが重要なループ内での頻繁な呼び出しには注意が必要です。
