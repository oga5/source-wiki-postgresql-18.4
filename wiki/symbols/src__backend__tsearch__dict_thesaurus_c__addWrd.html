addWrd

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/dict_thesaurus.c:105

目的

シソーラス辞書の置換ルールにおける置換後の単語（lexeme）を、内部構造体 TheSubstitute に追加・登録します。

シグネチャまたは形状

static void addWrd(DictThesaurus *d, char *b, char *e, uint32 idsubst, uint16 nwrd, uint16 posinsubst, bool useasis)

入力と出力

  • **入力**:
  • d: 対象の DictThesaurus 構造体。
  • b, e: 追加する単語の開始および終了ポインタ。
  • idsubst: 置換ルールのインデックス。
  • nwrd: 現在の置換フレーズ内での単語位置（0でリセット）。
  • posinsubst: 置換フレーズ内の位置情報。
  • useasis: DT_USEASIS フラグを立てるかどうかの真偽値。
  • **出力**: なし（副作用として d の内部メモリを更新）。

前提条件と不変条件

  • nwrd == 0 の場合、内部の静的カウンタ nres および ntres がリセットされ、必要に応じて d->subst 配列が拡張されます。
  • ptr->res 配列は常に NULL で終端されます。

アルゴリズムまたは振る舞い

1. nwrd == 0 の場合、静的カウンタを初期化し、idsubst が現在の容量を超えていれば d->subst を palloc/repalloc で拡張します。 2. 指定された idsubst に対応する TheSubstitute 構造体を取得し、lastlexeme を更新します。 3. ptr->res 配列の容量が不足している場合、倍増させて再割り当てします。 4. 新しい単語を palloc で確保したメモリにコピーし、ptr->res[nres] に格納します。 5. useasis フラグを設定し、ptr->res[++nres].lexeme を NULL に設定して終端を維持します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/dict_thesaurus.c 内のコンパイル処理（263行目、279行目付近）。
  • **呼び出し先**: palloc, repalloc, memcpy（PostgreSQL メモリ管理および標準ライブラリ関数）。

関連データ構造

  • DictThesaurus: シソーラス辞書のメイン構造体。
  • TheSubstitute: 置換ルールを保持する構造体。
  • TSLexeme: 置換後の単語情報を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc/repalloc がエラーを発生させます。
  • idsubst が範囲外の場合、配列の再割り当てが発生します。

保守担当者向けメモ

  • nres および ntres が static 変数として定義されているため、本関数は単一スレッドまたは特定のコンパイルコンテキスト内での逐次的な呼び出しを前提としています。並列処理での利用には注意が必要です。
