sql_check

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/compat_informix-test_informix2.c:100

目的

ECPG（Embedded SQL for C）の回帰テストにおいて、SQL 実行後のエラー状態を監視し、異常発生時に診断メッセージの出力とトランザクションのロールバックを行うユーティリティ関数です。

シグネチャまたは形状

static void sql_check(const char *fn, const char *caller, int ignore)

入力と出力

  • **入力**:
  • fn: 呼び出し元の関数名（文字列）。
  • caller: 実行した SQL 操作の名称（文字列）。
  • ignore: 無視すべき SQLCODE の値（例: データが存在しない場合の 100）。
  • **出力**: なし（戻り値は void）。エラー発生時は exit(1) によりプロセスを終了します。

前提条件と不変条件

  • グローバル変数 SQLCODE および sqlca 構造体が、直前の SQL 操作の結果を保持している必要があります。
  • ignore 引数に指定された値と SQLCODE が一致する場合、関数は何もせず即座に復帰します。

アルゴリズムまたは振る舞い

1. SQLCODE が ignore と一致するか確認し、一致すれば終了。 2. SQLCODE が 0 でない場合、エラーと判断して以下の処理を実行：

  • sqlca.sqlerrm.sqlerrmc を用いてエラー詳細を含む文字列を生成。
  • 生成したメッセージを stderr および stdout に出力。
  • ECPGtrans を呼び出し、トランザクションのロールバックを試行。
  • ロールバックの結果を判定し、その結果を再度出力。
  • exit(1) を呼び出し、プログラムを異常終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数内の各 SQL 操作（connect, create, insert, select max, select, update, drop, disconnect）の直後に呼び出されます。
  • **呼び出し先**: ECPGtrans（ECPG ライブラリ関数）。

関連データ構造

  • sqlca: SQL 通信領域（SQLCA）。エラーコードやエラーメッセージを保持します。

失敗モードと境界条件

  • SQLCODE が 0 の場合は正常系として処理を継続します。
  • SQLCODE が ignore と一致する場合、エラーとして扱われません。
  • ロールバック処理自体が失敗した場合、その旨がメッセージとして出力された後に終了します。

保守担当者向けメモ

  • 本関数は自動生成されたテストコードの一部です。直接編集は推奨されません。
  • ignore 引数は、特定の SQL ステートメントで期待される戻り値（例：NOT FOUND を示す 100）を許容するために使用されています。
