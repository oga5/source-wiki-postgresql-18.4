bound_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_spgist.c:185

目的

RangeBound 型の2つの値を比較し、ソートや検索のための順序関係を決定する。

シグネチャまたは形状

static int bound_cmp(const void *a, const void *b, void *arg)

入力と出力

  • **入力**:
  • a, b: 比較対象の RangeBound 構造体へのポインタ。
  • arg: TypeCacheEntry 構造体へのポインタ（比較に必要な型情報を含む）。
  • **出力**:
  • int: a が b より小さい場合は負の値、等しい場合は 0、大きい場合は正の値を返す。

前提条件と不変条件

  • arg は有効な TypeCacheEntry を指している必要がある。
  • a および b は RangeBound 型としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. 引数 a と b を RangeBound 型にキャストする。 2. arg を TypeCacheEntry 型にキャストする。 3. range_cmp_bounds 関数を呼び出し、型キャッシュ情報を用いて境界値の比較結果を算出・返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/rangetypes_spgist.c 内の spg_range_quad_picksplit 関数（262行目、264行目付近）から、ソート処理等のために呼び出されている。
  • **呼び出し先**:
  • range_cmp_bounds（utils/rangetypes.h で定義されている範囲境界比較関数）。

関連データ構造

  • RangeBound: 範囲型の境界値を保持する構造体。
  • TypeCacheEntry: 範囲型の比較に必要な型固有のキャッシュ情報。

失敗モードと境界条件

  • a または b が NULL の場合、あるいは arg が不正なポインタである場合、セグメンテーションフォールトが発生する可能性がある。

保守担当者向けメモ

  • 本関数は qsort 等の比較関数インターフェースに適合するように設計されている。
  • 比較ロジックの詳細は range_cmp_bounds に依存しているため、比較基準を変更する場合はそちらを確認すること。
