ExecGrant_common

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/aclchk.c:2115

目的

SQLの GRANT / REVOKE 文の実行に伴い、指定されたオブジェクト群のアクセス制御リスト（ACL）を更新し、関連するカタログ情報および依存関係を同期する汎用的な処理を行う。

シグネチャまたは形状

static void ExecGrant_common(
    InternalGrant *istmt, 
    Oid classid, 
    AclMode default_privs,
    void (*object_check) (InternalGrant *istmt, HeapTuple tuple)
)

入力と出力

  • **入力**:
  • istmt: 実行対象の権限操作情報（対象オブジェクト、権限、付与者など）。
  • classid: 対象オブジェクトが属するシステムカタログのOID。
  • default_privs: ACLが未設定の場合に使用されるデフォルト権限。
  • object_check: オブジェクト型固有の検証を行うコールバック関数（任意）。
  • **出力**: なし（副作用としてシステムカタログを更新）。

前提条件と不変条件

  • istmt が指すオブジェクトリストは有効であること。
  • classid に対応するシステムカタログが table_open 可能であること。
  • 処理対象のオブジェクトがシステムカタログ上に存在すること。

アルゴリズムまたは振る舞い

1. istmt->privileges が未設定の場合、default_privs で初期化する。 2. 対象カタログを RowExclusiveLock でオープンする。 3. istmt->objects 内の各オブジェクトについて以下を繰り返す：

  • システムキャッシュから対象タプルを検索し、存在しない場合はエラーを投げる。
  • object_check が提供されていれば実行する。
  • 現在のACLを取得し、存在しない場合は acldefault を適用する。
  • select_best_grantor で適切な付与者と権限オプションを決定する。
  • restrict_and_check_grant で権限の妥当性を検証し、警告やエラーを生成する。
  • merge_acl_with_grant で新しいACLを生成する。
  • CatalogTupleUpdate でカタログを更新し、updateAclDependencies で共有依存関係を同期する。
  • recordExtensionInitPriv で拡張機能の初期権限を記録する。
  • CommandCounterIncrement を呼び出し、重複オブジェクト処理時のエラーを回避する。

    • 4. 最後にカタログをクローズする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/aclchk.c 内の各オブジェクト型専用の権限更新関数（データベース、型、外部データラッパー、外部サーバー、関数、言語、スキーマ、テーブルスペースの各操作）。
  • **呼び出し先**: table_open, SearchSysCacheLocked1, select_best_grantor, restrict_and_check_grant, merge_acl_with_grant, CatalogTupleUpdate, updateAclDependencies, recordExtensionInitPriv 等。

関連データ構造

  • InternalGrant: 権限操作のパラメータを保持する構造体。
  • Acl: アクセス制御リストの構造体。
  • HeapTuple: システムカタログの行データ。

失敗モードと境界条件

  • 対象オブジェクトがシステムカタログに見つからない場合、elog(ERROR) によりトランザクションが中断される。
  • 権限付与の妥当性チェック（restrict_and_check_grant）に失敗した場合、エラーが発生する。
  • RowExclusiveLock の競合により待機が発生する場合がある。

保守担当者向けメモ

  • 本関数は、オブジェクト型に依存しない共通ロジックをカプセル化している。型固有のロジックが必要な場合は object_check コールバックを活用すること。
  • CommandCounterIncrement は、同一トランザクション内で同じオブジェクトが複数回指定された場合の整合性を保つために必須である。
