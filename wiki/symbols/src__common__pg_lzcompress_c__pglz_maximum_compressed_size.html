pglz_maximum_compressed_size

定義

  • 種別: function
  • 定義箇所: src/common/pg_lzcompress.c:856

目的

LZ77 圧縮アルゴリズムにおいて、指定された入力サイズ（rawsize）から圧縮後の最大サイズを見積もり、バッファオーバーランを防止するための上限値を算出します。

シグネチャまたは形状

int32 pglz_maximum_compressed_size(int32 rawsize, int32 total_compressed_size)

入力と出力

  • **入力**:
  • rawsize: 圧縮対象となるデータのバイト数。
  • total_compressed_size: 許容される圧縮後の最大サイズ（上限）。
  • **出力**:
  • 算出された圧縮後の最大サイズ（int32）。

前提条件と不変条件

  • 内部計算には int64 を使用し、計算過程でのオーバーフローを防止しています。
  • 戻り値は常に total_compressed_size を超えないことが保証されます。

アルゴリズムまたは振る舞い

1. **基本見積もり**: 各バイトに対して 1 ビットの制御ビットが必要であると仮定し、(rawsize * 9 + 7) / 8 バイトを基本サイズとして算出します。 2. **コーナーケースの補正**: 圧縮データの先頭付近で、リテラルバイトの後に 2〜3 バイトのマッチタグが続く場合を考慮し、安全のために 2 バイトを加算します。 3. **上限適用**: 算出された値と total_compressed_size を比較し、小さい方の値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/detoast.c:256 にて呼び出しが確認されています。
  • **呼び出し先**: PostgreSQL の Min マクロを使用しています。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • rawsize が非常に大きい場合でも、int64 で計算を行うためオーバーフローは発生しません。
  • 最終的な戻り値は int32 にキャストされるため、total_compressed_size が int32 の範囲内であることを前提としています。

保守担当者向けメモ

  • 本関数は pglz_compress が使用する出力バッファのサイズを決定する際に重要です。変更を加える際は、LZ77 の制御ビットとタグのオーバーヘッド計算に影響がないか確認してください。
