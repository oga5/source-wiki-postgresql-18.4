ThrowErrorData

定義

  • 種別: function
  • 定義箇所: src/backend/utils/error/elog.c:1899

目的

ErrorData 構造体で記述されたエラー情報を、PostgreSQL のエラー報告メカニズムを通じて報告する。主にソフトエラーの処理や、既存のエラー情報を再報告する際に使用される。

シグネチャまたは形状

void ThrowErrorData(ErrorData *edata)

入力と出力

  • **入力**: edata (報告対象のエラー情報を含む ErrorData 構造体へのポインタ)
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • edata は有効な ErrorData 構造体を指している必要がある。
  • errstart() がエラー報告を許可した場合にのみ処理が進行する。
  • メモリ割り当てには newedata->assoc_context が使用される。

アルゴリズムまたは振る舞い

1. errstart() を呼び出し、エラーレベルとドメインに基づいて報告の可否を判定する。報告不要な場合は即座に終了する。 2. エラースタックの現在のエントリ (newedata) を取得し、再帰深度をインクリメントする。 3. newedata->assoc_context にメモリコンテキストを切り替え、edata の各フィールド（メッセージ、詳細、ヒント、スタックトレース、SQL関連情報など）を pstrdup() を用いてコピーする。 4. メモリコンテキストを元に戻し、再帰深度をデクリメントする。 5. errfinish() を呼び出し、ログ出力やクライアントへの送信を含む最終的なエラー処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c:1197
  • src/backend/statistics/attribute_stats.c:746
  • **呼び出し先**:
  • errstart()
  • MemoryContextSwitchTo()
  • pstrdup()

関連データ構造

  • ErrorData: エラーの詳細情報を保持する構造体。

失敗モードと境界条件

  • errstart() が false を返した場合、エラーは報告されず処理が中断される。
  • pstrdup() によるメモリ確保が失敗した場合、メモリ不足エラーが発生する可能性がある（ErrorContext の設計により、エラー処理中のメモリ不足には耐性を持つ）。

保守担当者向けメモ

  • 本関数は ErrorData の内容をスタック上のエントリにコピーするため、呼び出し元の edata が解放されても報告内容は保持される。
  • recursion_depth を操作することで、エラー処理中の再帰的なエラー発生を監視している。
