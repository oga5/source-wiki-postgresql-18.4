RangeVarGetCreationNamespace

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/namespace.c:653

目的

指定された RangeVar に基づき、新しいリレーションを作成すべき名前空間（スキーマ）の OID を決定する。

シグネチャまたは形状

Oid RangeVarGetCreationNamespace(const RangeVar *newRelation)

入力と出力

  • **入力**: newRelation (RangeVar*) - 作成対象のリレーション情報。
  • **出力**: Oid - 決定された名前空間の OID。

前提条件と不変条件

  • newRelation->catalogname が指定されている場合、現在のデータベース名と一致しなければならない。
  • 一時テーブル（RELPERSISTENCE_TEMP）や pg_temp スキーマが指定された場合、AccessTempTableNamespace を介して一時名前空間の初期化が行われる。
  • 本関数内では USAGE 権限のチェックは行われない（呼び出し元で実施される）。

アルゴリズムまたは振る舞い

1. **カタログ名の検証**: catalogname が存在する場合、現在のデータベース名と比較し、不一致ならエラーを投げる。 2. **スキーマの特定**:

  • schemaname が "pg_temp" の場合、一時名前空間を初期化しその OID を返す。
  • schemaname が明示されている場合、get_namespace_oid で OID を取得する。
  • relpersistence が RELPERSISTENCE_TEMP の場合、一時名前空間を初期化しその OID を返す。
  • 上記以外の場合、recomputeNamespacePath を呼び出し、activeTempCreationPending が真なら一時名前空間を初期化し、そうでなければ activeCreationNamespace を返す。

    • 3. **エラー処理**: 最終的な namespaceId が無効な場合はエラーを投げる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/namespace.c
  • src/backend/commands/createas.c
  • src/backend/parser/parse_utilcmd.c
  • **呼び出し先**:
  • get_database_name
  • AccessTempTableNamespace
  • get_namespace_oid
  • recomputeNamespacePath

関連データ構造

  • RangeVar: リレーションのカタログ名、スキーマ名、リレーション名、永続性フラグを保持する構造体。
  • myTempNamespace: セッションの一時名前空間 OID。
  • activeCreationNamespace: 現在の検索パスに基づくデフォルトの作成先名前空間。

失敗モードと境界条件

  • **クロスデータベース参照**: catalogname が現在のデータベースと異なる場合は ERRCODE_FEATURE_NOT_SUPPORTED で失敗する。
  • **スキーマ未定義**: activeCreationNamespace が無効な状態でデフォルトの作成先が特定できない場合、ERRCODE_UNDEFINED_SCHEMA で失敗する。

保守担当者向けメモ

  • 本関数は作成先を決定するのみであり、実際の作成権限（CREATE 権限）の確認は呼び出し元に委ねられている点に注意すること。
