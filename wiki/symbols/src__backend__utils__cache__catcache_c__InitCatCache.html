InitCatCache

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/cache/catcache.c:888

目的

システムカタログのタプルをキャッシュするための CatCache 構造体を初期化し、グローバルなキャッシュ管理リストに登録する。

シグネチャまたは形状

CatCache *InitCatCache(int id, Oid reloid, Oid indexoid, int nkeys, const int *key, int nbuckets)

入力と出力

  • **入力**:
  • id: キャッシュ識別子。
  • reloid: 対象カタログのOID。
  • indexoid: 使用するインデックスのOID。
  • nkeys: キーの数。
  • key: 属性番号の配列。
  • nbuckets: 初期ハッシュバケット数（2のべき乗である必要がある）。
  • **出力**: 初期化された CatCache 構造体へのポインタ。

前提条件と不変条件

  • nbuckets は 2 のべき乗でなければならない（Assert で検証）。
  • key 配列内の各属性番号は有効である必要がある。
  • CacheMemoryContext が存在しない場合は自動的に作成される。

アルゴリズムまたは振る舞い

1. CacheMemoryContext へメモリコンテキストを切り替える。 2. グローバルな CacheHdr が未初期化であれば初期化する。 3. CatCache 構造体をアラインメントを考慮して確保し、nbuckets 分のハッシュバケットを割り当てる。 4. キャッシュのメタデータ（ID、OID、キー定義など）を設定する。 5. キャッシュを CacheHdr->ch_caches リストの先頭にプッシュする。 6. 元のメモリコンテキストに戻し、作成した構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/syscache.c の SysCache 初期化処理から呼び出されることが確認されている。
  • **呼び出し先**: CreateCacheMemoryContext, MemoryContextSwitchTo, palloc, palloc_aligned, palloc0, slist_init, slist_push_head 等のメモリ管理およびリスト操作関数。

関連データ構造

  • CatCache: キャッシュ管理のメイン構造体。
  • CatCacheHeader: キャッシュ全体のグループヘッダー。
  • CacheMemoryContext: キャッシュエントリを保持するメモリコンテキスト。

失敗モードと境界条件

  • nbuckets が 2 のべき乗でない場合、アサーション違反により停止する。
  • メモリ不足時は palloc 等がエラーを発生させる。

保守担当者向けメモ

  • 本関数はキャッシュの構造的な初期化のみを行い、カタログのリレーション自体は開かない。
  • cc_lbucket はリスト検索が要求されるまで遅延割り当てされる。
  • CATCACHE_STATS が定義されている場合、終了時に統計を出力するためのフックが登録される。
