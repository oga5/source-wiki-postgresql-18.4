trueTriConsistentFn

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginlogic.c:55

目的

GINインデックススキャンにおいて、検索条件が常に合致すると判定される場合に、三値論理（GinTernaryValue）として GIN_TRUE を返すための定数関数です。

シグネチャまたは形状

static GinTernaryValue trueTriConsistentFn(GinScanKey key)

入力と出力

  • **入力**: GinScanKey key（スキャンキーの状態を保持する構造体）
  • **出力**: GinTernaryValue 型の GIN_TRUE

前提条件と不変条件

  • 入力された key の内容は参照されず、常に GIN_TRUE を返します。

アルゴリズムまたは振る舞い

  • 呼び出されると、無条件で GIN_TRUE を返します。これは主に GIN_SEARCH_MODE_EVERYTHING のような、インデックスエントリをすべて合致とみなすケースで使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginlogic.c 内の ginInitConsistentFunction において、key->triConsistentFn に関数ポインタとして代入されます。
  • **呼び出し先**: 本関数は他の関数を呼び出しません。

関連データ構造

  • GinScanKey: GINスキャン中の状態を管理する構造体。本関数はこの構造体のメンバ triConsistentFn に割り当てられます。
  • GinTernaryValue: GINの三値論理（TRUE, FALSE, MAYBE）を表現する列挙型。

失敗モードと境界条件

  • 本関数自体は常に成功し、失敗モードはありません。

保守担当者向けメモ

  • 本関数は GIN_SEARCH_MODE_EVERYTHING モード時のデフォルト動作として利用されます。ロジックの変更は不要ですが、スキャンキーの初期化ロジックに変更を加える際は、この関数ポインタの割り当て箇所を確認してください。
