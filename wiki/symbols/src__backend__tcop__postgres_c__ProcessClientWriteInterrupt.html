ProcessClientWriteInterrupt

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/postgres.c:547

目的

クライアントへの書き込み処理中に発生する割り込みを処理し、プロセス終了要求（ProcDiePending）がある場合に、書き込みのブロックによる応答遅延やデッドロックを防ぐ。

シグネチャまたは形状

void ProcessClientWriteInterrupt(bool blocked)

入力と出力

  • **入力**:
  • blocked (bool): 現在の書き込み操作がブロックされているかどうかを示すフラグ。
  • **出力**: なし（副作用としてプロセスの状態や割り込みフラグを更新）。

前提条件と不変条件

  • errno の値は関数呼び出し前後で保存・復元され、外部への影響を最小限に抑える。
  • ProcDiePending が真の場合、プロセス終了処理が優先される。

アルゴリズムまたは振る舞い

1. 現在の errno を保存する。 2. ProcDiePending が真（終了要求あり）の場合:

  • blocked が真（書き込みがブロック中）の場合:
  • 割り込み保留中（InterruptHoldoffCount == 0）かつクリティカルセクション外（CritSectionCount == 0）であれば、whereToSendOutput を DestNone に変更してエラーメッセージ送信を回避し、CHECK_FOR_INTERRUPTS() を呼び出して即座に終了処理へ移行する。
  • blocked が偽の場合:
  • SetLatch(MyLatch) を呼び出し、プロセスラッチをセットして終了処理を促す。

    • 3. 保存した errno を復元して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-secure.c (複数の箇所で true または false を引数として呼び出し)
  • **呼び出し先**:
  • CHECK_FOR_INTERRUPTS()
  • SetLatch()

関連データ構造

  • ProcDiePending (グローバル変数: プロセス終了要求フラグ)
  • MyLatch (プロセスラッチ)
  • InterruptHoldoffCount / CritSectionCount (割り込み制御用カウンタ)
  • whereToSendOutput (出力先制御変数)

失敗モードと境界条件

  • InterruptHoldoffCount や CritSectionCount が 0 でない場合、ProcDiePending が真であっても CHECK_FOR_INTERRUPTS() は実行されず、終了処理が遅延する可能性がある。
  • 書き込みがブロックされていない場合、単にラッチをセットするのみで、即時の終了は保証されない。

保守担当者向けメモ

  • 本関数は、クライアントとの通信プロトコル同期を維持しつつ、終了要求を迅速に処理するためのガードレールとして機能する。
  • whereToSendOutput を DestNone に変更するロジックは、部分的なプロトコルメッセージ送信による同期喪失を防ぐための重要な措置である。
