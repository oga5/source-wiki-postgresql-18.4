file_type_t

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_rewind/filemap.h:30

目的

pg_rewind において、ファイルシステム上の各オブジェクトがどのような種類（通常ファイル、ディレクトリ、シンボリックリンク）であるかを識別するために使用されます。

シグネチャまたは形状

typedef enum
{
	FILE_TYPE_UNDEFINED = 0,
	FILE_TYPE_REGULAR,
	FILE_TYPE_DIRECTORY,
	FILE_TYPE_SYMLINK,
} file_type_t;

入力と出力

  • 入力: 該当なし（列挙型定義）
  • 出力: file_entry_t 構造体の source_type および target_type フィールドとして保持されます。

前提条件と不変条件

  • FILE_TYPE_UNDEFINED は初期状態または未設定状態を示します。
  • pg_rewind の同期処理において、ソースおよびターゲットのファイル属性を比較する際の型判定基準となります。

アルゴリズムまたは振る舞い

  • ファイルシステムから取得した情報を元に、process_source_file および process_target_file 関数を通じて、各ファイルパスに対応する型が割り当てられます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/bin/pg_rewind/filemap.c: process_source_file および process_target_file の引数として使用。
  • src/bin/pg_rewind/libpq_source.c: リモートソースからのファイル情報取得時に使用。
  • 呼び出し先: 該当なし（列挙型のため）。

関連データ構造

  • file_entry_t: 同期対象ファイルのメタデータを保持する構造体であり、本型をメンバとして含みます。

失敗モードと境界条件

  • ファイルシステムから型を特定できない場合や、未初期化状態では FILE_TYPE_UNDEFINED が使用されます。

保守担当者向けメモ

  • 新しいファイルタイプをサポートする必要が生じた場合は、本列挙型に値を追加し、filemap.c 内の判定ロジックを更新してください。
