ClearOrSaveAllResults

定義

  • 種別: function
  • 定義箇所: src/bin/psql/common.c:581

目的

データベース接続から残存するすべてのクエリ結果を読み出し、それぞれに対して保存または破棄の処理を行う。

シグネチャまたは形状

static void ClearOrSaveAllResults(void)

入力と出力

  • 入力: グローバル変数 pset.db（データベース接続ハンドル）。
  • 出力: なし（戻り値なし）。

前提条件と不変条件

  • pset.db が有効なデータベース接続を保持していること。
  • 本関数は、PQgetResult が NULL を返すまで（すべての結果が消費されるまで）ループを実行する。

アルゴリズムまたは振る舞い

1. PQgetResult(pset.db) を呼び出し、結果オブジェクトを取得する。 2. 結果が NULL でない限り、取得した PGresult を引数として ClearOrSaveResult を呼び出す。 3. PQgetResult が NULL を返すまで、上記の手順を繰り返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/psql/common.c:1758
  • src/bin/psql/common.c:2166
  • 呼び出し先:
  • PQgetResult (libpq)
  • ClearOrSaveResult (同一ファイル内)

関連データ構造

  • PGresult: libpq が提供するクエリ結果構造体。
  • pset: psql のグローバル設定を保持する構造体（pset.db を参照）。

失敗モードと境界条件

  • 接続が既に切断されている場合、PQgetResult は NULL を返すか、libpq の仕様に従った挙動を示す。
  • 本関数自体はエラーコードを返さないため、呼び出し側で接続状態を監視する必要がある。

保守担当者向けメモ

  • 本関数は、クエリ実行後のクリーンアップ処理において、パイプラインモードや非同期処理の残存結果を確実に処理するために使用される。
  • ClearOrSaveResult の実装変更が本関数の挙動に直接影響を与える点に留意すること。
