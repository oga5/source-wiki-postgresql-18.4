tuplestore_begin_common

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplestore.c:255

目的

Tuplestorestate 構造体を初期化し、メモリ内でのタプル格納に必要なリソース（メモリコンテキスト、タプル格納用配列、読み取りポインタ管理領域）を割り当てる。

シグネチャまたは形状

static Tuplestorestate *tuplestore_begin_common(int eflags, bool interXact, int maxKBytes)

入力と出力

  • **入力**:
  • eflags: 読み取りポインタの動作フラグ。
  • interXact: トランザクションを跨ぐかどうかを示すフラグ。
  • maxKBytes: 使用可能な最大メモリ容量（KB単位）。
  • **出力**:
  • 初期化済みの Tuplestorestate 構造体へのポインタ。

前提条件と不変条件

  • CurrentMemoryContext が有効であること。
  • CurrentResourceOwner が適切に設定されていること。
  • maxKBytes に基づき、availMem が初期化される。
  • memtuples 配列の初期サイズは ALLOCSET_SEPARATE_THRESHOLD を考慮して決定される。

アルゴリズムまたは振る舞い

1. palloc0 を使用して Tuplestorestate を確保し、状態を TSS_INMEM に設定する。 2. GenerationContextCreate を呼び出し、タプル格納用のメモリコンテキストを作成する。 3. memtuples 配列を確保し、GetMemoryChunkSpace を用いて使用メモリ量を追跡する。 4. 読み取りポインタ管理用の readptrs 配列を確保し、インデックス 0 のポインタを初期化する。 5. 成功時に初期化済み状態構造体を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: tuplestore_begin_heap（src/backend/utils/sort/tuplestore.c:343）から呼び出される。
  • **呼び出し先**: palloc0, GenerationContextCreate, palloc, GetMemoryChunkSpace 等のメモリ管理関数。

関連データ構造

  • Tuplestorestate: 本関数で初期化されるメインの状態構造体。
  • TSReadPointer: 読み取り位置を管理する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc 等がエラーを発生させる。
  • maxKBytes が 0 の場合、availMem は 0 となる。

保守担当者向けメモ

  • memtuples の初期サイズ計算には ALLOCSET_SEPARATE_THRESHOLD が関与しており、メモリ割り当ての最適化が図られている。
  • readptrs は初期状態で 8 個分のスロットが確保されるが、必要に応じて拡張される設計となっている。
