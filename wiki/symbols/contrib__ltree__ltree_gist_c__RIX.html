RIX

定義

  • 種別: struct
  • 定義箇所: contrib/ltree/ltree_gist.c:279

目的

GiST インデックスの分割処理（ltree_picksplit）において、ltree データのソートおよび管理を行うための補助的な構造体です。

シグネチャまたは形状

typedef struct rix
{
	int			index;
	ltree	   *r;
} RIX;

入力と出力

  • 本構造体はデータ保持用であり、関数としての入出力は持ちません。
  • ltree_picksplit 内で palloc を用いて配列として確保され、ソート処理の対象となります。

前提条件と不変条件

  • r メンバは有効な ltree へのポインタである必要があります。
  • index は元のデータセットにおける位置を保持するために使用されます。

アルゴリズムまたは振る舞い

  • ltree_picksplit において、インデックス対象のデータを RIX 配列に格納し、treekey_cmp 関数を用いて ltree 値に基づいたソートを行うために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/ltree/ltree_gist.c 内の ltree_picksplit 関数。
  • **呼び出し先**: treekey_cmp 関数（qsort の比較関数として使用）。

関連データ構造

  • ltree: RIX 構造体が保持するデータ型。
  • treekey_cmp: RIX 配列のソートに使用される比較関数。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性があります。
  • maxoff に基づいて動的に配列サイズが決定されるため、インデックスの要素数に依存します。

保守担当者向けメモ

  • 本構造体は ltree_picksplit 内の局所的なソート処理専用です。
  • treekey_cmp を介して ltree_compare を呼び出すため、ltree 型の比較ロジックに変更があった場合は本構造体を利用するソート処理も影響を受ける可能性があります。
