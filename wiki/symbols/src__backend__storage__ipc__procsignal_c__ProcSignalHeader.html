ProcSignalHeader

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/ipc/procsignal.c:84

目的

PostgreSQLの全バックエンドプロセスにおけるシグナル状態を管理するための、共有メモリ上のヘッダー構造体です。

シグネチャまたは形状

struct ProcSignalHeader
{
    pg_atomic_uint64 psh_barrierGeneration;
    ProcSignalSlot psh_slot[FLEXIBLE_ARRAY_MEMBER];
};

入力と出力

  • **入力**: 共有メモリの初期化時に割り当てられ、各プロセスのシグナル状態（ProcSignalSlot）を保持します。
  • **出力**: グローバルバリアの生成番号（psh_barrierGeneration）および各スロットのシグナルフラグを通じて、プロセス間通信の基盤を提供します。

前提条件と不変条件

  • 共有メモリ上に配置される必要があります。
  • psh_slot は可変長配列として定義されており、システム内の最大プロセス数に応じてメモリが確保されます。
  • psh_barrierGeneration はアトミック操作（pg_atomic_uint64）によって保護される必要があります。

アルゴリズムまたは振る舞い

  • プロセス間シグナル通信のルートとして機能します。
  • psh_barrierGeneration は、システム全体で同期が必要なバリアイベントの生成番号を追跡します。
  • 各プロセスは psh_slot 配列内の自身のスロットを介して、受信したシグナルの理由を確認します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/procsignal.c 内の初期化関数およびシグナル管理関数。
  • src/backend/postmaster/launch_backend.c にて外部参照（ProcSignal ポインタ経由）。
  • **呼び出し先**:
  • 構造体内部で ProcSignalSlot を保持しています。

関連データ構造

  • ProcSignalSlot: 各プロセスのシグナル状態を保持する構造体。
  • ProcSignal: ProcSignalHeader 型のグローバルポインタ（procsignal.c で定義）。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、システムは起動できません。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、メモリ確保時に正しいサイズ計算（offsetof を使用）を行わないとバッファオーバーフローのリスクがあります。

保守担当者向けメモ

  • 本構造体は不透明な構造体（opaque struct）として扱われるべきであり、詳細は procsignal.c 内に限定してください。
  • psh_slot へのアクセスは、プロセスIDに基づくインデックス計算を伴うため、インデックスの範囲外アクセスに注意してください。
