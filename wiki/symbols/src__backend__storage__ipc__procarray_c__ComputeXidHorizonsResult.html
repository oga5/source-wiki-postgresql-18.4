ComputeXidHorizonsResult

定義

  • **種別**: struct
  • **定義箇所**: src/backend/storage/ipc/procarray.c:179

目的

PostgreSQLのMVCC（多版同時実行制御）において、トランザクションの可視性判定や不要領域の削除（VACUUM）に必要な、各種トランザクションID（XID）の境界値を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct ComputeXidHorizonsResult
{
    FullTransactionId latest_completed;
    TransactionId slot_xmin;
    TransactionId slot_catalog_xmin;
    TransactionId oldest_considered_running;
    TransactionId shared_oldest_nonremovable;
    TransactionId shared_oldest_nonremovable_raw;
    TransactionId catalog_oldest_nonremovable;
    TransactionId data_oldest_nonremovable;
    TransactionId temp_oldest_nonremovable;
} ComputeXidHorizonsResult;

入力と出力

本構造体は ComputeXidHorizons() 関数によって値が設定（出力）され、主に GlobalVisUpdateApply() 関数へ渡されて可視性状態の更新に使用されます。

前提条件と不変条件

  • ComputeXidHorizons() が ProcArrayLock を保持している間に計算された値である必要があります。
  • shared_oldest_nonremovable_raw は、レプリケーションスロットの catalog_xmin の影響を受けない点を除き、shared_oldest_nonremovable と同等の意味を持ちます。

アルゴリズムまたは振る舞い

  • 各フィールドは、テーブル種別（共有、カタログ、ユーザー定義、一時）やレプリケーションスロットの状態に基づき、削除不可な最小のXIDを保持します。
  • oldest_considered_running は、VACUUMが pg_subtrans のルックアップを行うために必要な最小XIDを特定するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: ComputeXidHorizons() が本構造体のインスタンスを生成・設定します。また、GlobalVisUpdateApply() が引数として受け取ります。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • ProcArrayStruct: 本構造体の計算元となるプロセス配列を管理するメイン構造体。
  • GlobalVisState: 本構造体の計算結果を反映する状態管理構造体。

失敗モードと境界条件

  • レプリケーションスロットがアクティブな場合、slot_xmin や slot_catalog_xmin が境界値に影響を与え、不要なタプルの削除が抑制される可能性があります。
  • shared_oldest_nonremovable_raw は、ホットスタンバイフィードバックを通じてアップストリームへ情報を送る際に、スロットの影響を排除するために使用されます。

保守担当者向けメモ

  • oldest_considered_running は歴史的な経緯からレプリケーションスロットの影響を含んでいますが、将来的に変更される可能性があります。
  • pg_subtrans の切り詰め判定に利用されるため、この構造体の計算ロジックを変更する際は、サブトランザクションの可視性への影響を十分に考慮してください。
