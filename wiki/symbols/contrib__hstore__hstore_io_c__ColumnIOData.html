ColumnIOData

定義

  • 種別: struct
  • 定義箇所: contrib/hstore/hstore_io.c:815

目的

レコード型や複合型と hstore 間の変換処理において、各カラムの入出力に必要なメタデータと関数ポインタをキャッシュするために使用されます。

シグネチャまたは形状

typedef struct ColumnIOData
{
	Oid			column_type;   /* カラムのデータ型OID */
	Oid			typiofunc;     /* 入出力関数のOID */
	Oid			typioparam;    /* 入出力関数のパラメータOID */
	FmgrInfo	proc;          /* 関数呼び出し用情報 */
} ColumnIOData;

入力と出力

本構造体は関数への引数ではなく、変換処理中の状態保持（キャッシュ）として利用されます。各フィールドは Oid および FmgrInfo 型であり、PostgreSQL の関数マネージャが型変換を実行するために必要な情報を保持します。

前提条件と不変条件

  • FmgrInfo proc フィールドは、PostgreSQL の関数呼び出しメカニズム（fmgr）で利用可能な状態である必要があります。
  • 変換対象となるカラム数に応じて、ColumnIOData の配列としてメモリ確保されることが一般的です。

アルゴリズムまたは振る舞い

  • 変換処理の初期化フェーズで、各カラムの型情報に基づき typiofunc や typioparam がルックアップされ、fmgr_info を用いて proc が初期化されます。
  • 以降の変換ループでは、このキャッシュされた proc を使用して、各カラムの値を効率的に変換します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/hstore/hstore_io.c 内の hstore_from_record や hstore_populate_record 等の変換関数が、カラム情報の初期化時に本構造体を参照・構築します。
  • **呼び出し先**: FmgrInfo を介して、各カラムの型に対応する PostgreSQL の標準的な入出力関数（InputFunctionCall 等）が呼び出されます。
  • **外部参照**: src/backend/utils/adt/jsonfuncs.c においても、JSON 変換時のメタデータキャッシュとして本構造体が参照されている痕跡があります。

関連データ構造

  • RecordIOData: レコード変換全体の状態を管理する構造体であり、ColumnIOData の配列を保持します。
  • FmgrInfo: PostgreSQL の関数呼び出し情報を保持する標準構造体。

失敗モードと境界条件

  • カラム数が動的に決定されるため、メモリ確保時に ncolumns * sizeof(ColumnIOData) の計算でオーバーフローや不足が発生しないことが前提となります。
  • typiofunc が無効な場合、proc の初期化時にエラーが発生する可能性があります。

保守担当者向けメモ

  • hstore_io.c 内の変換関数で ncolumns を用いて配列を確保する箇所が複数存在します。変更時はこれらの確保ロジックの整合性に注意してください。
  • jsonfuncs.c からの参照があるため、本構造体の定義を変更する場合は、hstore モジュール外への影響範囲を調査する必要があります。
