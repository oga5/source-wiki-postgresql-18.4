get_partition_col_attnum

定義

  • 種別: function
  • 定義箇所: src/include/utils/partcache.h:79

目的

パーティションキーの定義構造体から、指定されたインデックスに対応するパーティション列の属性番号（AttrNumber）を取得します。

シグネチャまたは形状

static inline int16 get_partition_col_attnum(PartitionKey key, int col)

入力と出力

  • **入力**:
  • key: パーティションキー情報を保持する PartitionKey 構造体へのポインタ。
  • col: 取得対象の列インデックス（0から始まる整数）。
  • **出力**:
  • int16: 指定された列に対応する属性番号。

前提条件と不変条件

  • key は有効な PartitionKey 構造体を指している必要があります。
  • col は 0 以上、かつ key->partnatts 未満である必要があります（境界チェックは関数内で行われません）。

アルゴリズムまたは振る舞い

1. key が保持する partattrs 配列にアクセスします。 2. 指定されたインデックス col に格納されている属性番号を直接返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/partition.c:273
  • src/backend/executor/execPartition.c:1644
  • **呼び出し先**:
  • なし（インライン関数として定義されており、直接的な関数呼び出しは発生しません）。

関連データ構造

  • PartitionKey: パーティションキーのメタデータを保持する構造体。partattrs フィールドを通じて属性番号の配列を管理します。

失敗モードと境界条件

  • col が不正な値（負数やパーティション列数以上）である場合、メモリ破壊や未定義の動作を引き起こす可能性があります。本関数自体には境界チェックが含まれていません。

保守担当者向けメモ

  • 本関数はインライン関数であり、パフォーマンスを重視して設計されています。構造変更を行う際は、PartitionKeyData の partattrs 配列の定義と整合性を保つ必要があります。
