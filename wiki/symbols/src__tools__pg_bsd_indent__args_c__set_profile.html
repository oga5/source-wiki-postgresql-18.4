set_profile

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/args.c:175

目的

プログラムの設定プロファイル（.indent.pro）を読み込み、整形ルールをグローバル変数に反映させる。

シグネチャまたは形状

void set_profile(const char *profile_name)

入力と出力

  • **入力**:
  • profile_name: 読み込むプロファイルファイルのパス（NULL の場合はデフォルトのホームディレクトリパスが使用される）。
  • **出力**: なし（副作用としてグローバル変数を更新）。

前提条件と不変条件

  • getenv("HOME") が有効なパスを返すこと。
  • scan_profile が呼び出される前に、対象ファイルが正常にオープン可能であること。

アルゴリズムまたは振る舞い

1. profile_name が NULL の場合、$HOME/.indent.pro を対象とする。それ以外の場合は、指定されたパス（先頭2文字を除外）を対象とする。 2. 対象ファイルをオープンし、scan_profile を呼び出して内容を解析・適用した後、ファイルを閉じる。 3. カレントディレクトリの .indent.pro をオープンし、同様に scan_profile を呼び出して解析・適用した後、ファイルを閉じる。 4. 最後に option_source を "Command line" に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/tools/pg_bsd_indent/indent.c:189
  • **呼び出し先**: scan_profile

関連データ構造

  • option_source: 現在解析中のソースを示すグローバル変数。

失敗モードと境界条件

  • ファイルが存在しない場合、そのファイルに対する解析はスキップされる（エラー終了はしない）。
  • getenv("HOME") が NULL を返す場合の挙動については、本ソースコード内では明示的なガード処理が確認できない。

保守担当者向けメモ

  • 本関数は、ユーザー指定のプロファイルとデフォルトのプロファイルを順次読み込む。後から読み込まれた設定が優先される順序となっている。
  • option_source の更新は、エラー発生時の通知先を追跡するために行われている。
