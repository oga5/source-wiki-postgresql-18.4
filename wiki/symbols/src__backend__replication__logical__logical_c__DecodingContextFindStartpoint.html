DecodingContextFindStartpoint

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/logical.c:634

目的

論理デコーディングを開始するために、レプリケーションスロットの restart_lsn からWALを読み進め、整合性のある開始点（Consistent Point）を特定する。

シグネチャまたは形状

void DecodingContextFindStartpoint(LogicalDecodingContext *ctx)

入力と出力

  • **入力**: LogicalDecodingContext *ctx（論理デコーディングの状態を保持するコンテキスト構造体）
  • **出力**: なし（副作用として ctx 内のリーダー状態更新およびスロット情報の更新を行う）

前提条件と不変条件

  • ctx->slot が有効であること。
  • ctx->reader が初期化済みであること。
  • 呼び出し前に slot->data.restart_lsn が開始位置として設定されていること。

アルゴリズムまたは振る舞い

1. XLogBeginRead を呼び出し、スロットの restart_lsn からWALの読み取りを開始する。 2. DecodingContextReady(ctx) が真を返すまで、以下のループを繰り返す。

  • XLogReadRecord でWALレコードを読み込む。
  • LogicalDecodingProcessRecord を呼び出し、レコードを処理してコンテキストの状態を更新する。
  • CHECK_FOR_INTERRUPTS() を実行し、割り込み要求をチェックする。

    • 3. 整合性のある地点に到達後、スロットの confirmed_flush を最後に読み込んだレコードの終了位置（ctx->reader->EndRecPtr）に更新する。 4. two_phase が有効な場合、two_phase_at も同様に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/slotfuncs.c:159
  • src/backend/replication/walsender.c:1314
  • **呼び出し先**:
  • XLogBeginRead
  • XLogReadRecord
  • LogicalDecodingProcessRecord
  • DecodingContextReady
  • CHECK_FOR_INTERRUPTS
  • SpinLockAcquire / SpinLockRelease

関連データ構造

  • LogicalDecodingContext: デコーディングの主要状態管理構造体。
  • ReplicationSlot: レプリケーションスロットのメタデータ（restart_lsn, confirmed_flush 等）。
  • XLogReaderState: WAL読み取り用構造体。

失敗モードと境界条件

  • XLogReadRecord がエラーを返した場合、elog(ERROR) により処理を中断する。
  • レコードが読み込めない場合（!record）、整合性のある開始点が見つからないものとして elog(ERROR) を発生させる。

保守担当者向けメモ

  • 本関数は整合性のある地点に到達するまでループするため、WALの量によっては長時間実行される可能性がある。
  • two_phase 関連の更新はスピンロックで保護されているため、スロットの排他制御に注意すること。
