accessMethodNameCompare

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dump_sort.c:527

目的

2つのアクセスメソッド（Access Method）を、その名前（dobj.name）に基づいて辞書順で比較する。

シグネチャまたは形状

static int accessMethodNameCompare(Oid am1, Oid am2)

入力と出力

  • **入力**:
  • am1 (Oid): 比較対象のアクセスメソッドのOID。
  • am2 (Oid): 比較対象のアクセスメソッドのOID。
  • **出力**:
  • int: strcmp の結果を返す。am1 と am2 が等しい場合は 0。

前提条件と不変条件

  • findAccessMethodByOid が有効な AccessMethodInfo 構造体を返すことを前提とする。
  • am1 と am2 が同一のOIDである場合、比較処理を行わず即座に 0 を返す。

アルゴリズムまたは振る舞い

1. 引数 am1 と am2 が等しいか確認し、等しければ 0 を返す。 2. findAccessMethodByOid を使用して、それぞれのOIDに対応する AccessMethodInfo 構造体を取得する。 3. いずれかの構造体が取得できなかった場合（カタログ破損とみなす）、Assert(false) を実行し 0 を返す。 4. 取得した構造体内の dobj.name を strcmp で比較し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump_sort.c:307 (推測: opcobj1->opcmethod の比較)
  • src/bin/pg_dump/pg_dump_sort.c:318 (推測: opfobj1->opfmethod の比較)
  • **呼び出し先**:
  • findAccessMethodByOid (外部関数)
  • strcmp (標準ライブラリ)

関連データ構造

  • AccessMethodInfo: アクセスメソッドの情報を保持する構造体。dobj.name メンバを含む。

失敗モードと境界条件

  • **カタログ破損**: findAccessMethodByOid が NULL を返した場合、アサーションエラーが発生し、関数は 0 を返す。

保守担当者向けメモ

  • 本関数は pgTypeNameCompare と同様のカタログ破損時のエラーハンドリング方針を採用している。
  • 比較対象のOIDがアクセスメソッドとして存在しない場合、ソート順序が不定になる可能性があるため注意が必要。
