spg_text_inner_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgtextproc.c:425

目的

SP-GiST 索引の検索時、現在の内部タプルからどの枝（子ノード）を探索すべきかを判定し、枝刈りを行う。

シグネチャまたは形状

Datum spg_text_inner_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgInnerConsistentIn（検索キー、現在のパス、ノードラベル等）および spgInnerConsistentOut（探索対象ノードのリスト）を受け取る。
  • **出力**: void。結果は out 引数を通じて返される。

前提条件と不変条件

  • in->reconstructedValue は NULL または in->level と一致する長さを持つ必要がある。
  • in->reconstructedValue は toasted 状態ではなく、ロングヘッダー形式であると仮定される。
  • C ロケール以外では、照合順序依存の演算子に対して木全体を走査する必要がある。

アルゴリズムまたは振る舞い

1. **値の再構築**: 親ノードまでのパス、現在のタプルの接頭辞、およびノードラベルを連結し、現在のパスまでの文字列を復元する。 2. **ノードの走査**: 各子ノードに対し、ラベルを付与して完全な文字列を形成する。 3. **枝刈り判定**: 各検索キー（scankeys）に対し、復元した文字列と照合する。

  • 照合順序依存の演算子かつ C ロケールでない場合は、判定をスキップ（探索対象とする）。
  • memcmp を用いて文字列比較を行い、演算子（<, <=, =, >=, >, prefix）に基づいて一致しない枝を排除する。

    • 4. **結果の格納**: 条件を満たしたノードのインデックス、レベルの増分、および再構築された値を out 構造体に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は明示されていないが、SP-GiST のインデックススキャン処理（spgscan.c 等）から間接的に呼び出される。
  • **呼び出し先**: palloc, DatumGetPointer, DatumGetTextPP, memcpy, memcmp, datumCopy, elog 等の PostgreSQL 内部関数。

関連データ構造

  • spgInnerConsistentIn: 検索条件と現在の木の状態を保持。
  • spgInnerConsistentOut: 探索すべきノードと、次のレベルへ引き継ぐ再構築済みデータ。

失敗モードと境界条件

  • nodeChar <= 0: ダミーラベルとして扱い、文字列の復元には含めない。
  • strategy が不明な場合: elog(ERROR) により異常終了する。
  • collate_is_c が偽の場合: 照合順序依存の演算子では枝刈りを行わず、全ノードを探索対象とする。

保守担当者向けメモ

  • 本関数は text 型の基数木構造における検索の要である。
  • SPG_IS_COLLATION_AWARE_STRATEGY マクロによるロケール判定がパフォーマンスに直結するため、変更時は注意が必要。
  • reconstructedValue のメモリ管理は palloc に依存しており、大規模なインデックススキャン時にはメモリ消費量に留意すること。
