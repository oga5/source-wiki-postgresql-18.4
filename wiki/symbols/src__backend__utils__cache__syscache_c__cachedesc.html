cachedesc

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/syscache.c:69

目的

個々のシステムキャッシュ（syscache）の構成情報を保持するための構造体です。システムカタログのルックアップを最適化するために必要なリレーション、インデックス、およびキー属性の定義を格納します。

シグネチャまたは形状

struct cachedesc
{
    Oid         reloid;         /* キャッシュ対象リレーションのOID */
    Oid         indoid;         /* キャッシュ用インデックスリレーションのOID */
    int         nkeys;          /* キャッシュルックアップに必要なキー数 */
    int         key[4];         /* キー属性の属性番号 */
    int         nbuckets;       /* キャッシュ用ハッシュバケット数 */
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。システム初期化時に syscache_info.h の定義に基づき、CatCache 構造体を構築するためのメタデータとして入力されます。

前提条件と不変条件

  • key 配列は最大4つまでの属性番号を保持します。
  • nkeys は実際に使用されるキーの数と一致している必要があります。

アルゴリズムまたは振る舞い

本構造体自体はロジックを持ちません。InitCatalogCache 関数がこの構造体の配列を参照し、各キャッシュのハッシュテーブル管理機構（CatCache）を初期化するために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/syscache.c 内の初期化処理（InitCatalogCache）から参照されます。
  • 呼び出し先: 情報なし。

関連データ構造

  • CatCache: cachedesc の情報をもとに生成される、実際のキャッシュ管理用構造体。
  • syscache_info.h: cachedesc のインスタンスが定義されているメタデータヘッダー。

失敗モードと境界条件

  • key 配列のサイズ（4）を超えるキー数を持つキャッシュ定義は、この構造体では表現できません。
  • reloid や indoid に無効なOIDが指定された場合、キャッシュの初期化時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 新しいシステムキャッシュを追加する際は、syscache_info.h に定義を追加し、この構造体で正しく表現できることを確認してください。
  • key 配列のサイズ制限（4）は、現在のシステムキャッシュの設計上の制約です。これを超えるキーが必要な場合は構造体の拡張が必要です。
