config_var_value

定義

  • 種別: struct
  • 定義箇所: src/include/utils/guc_tables.h:45

目的

PostgreSQLのGUC（Grand Unified Configuration）システムにおいて、設定変数の値を保持するための汎用的なコンテナ構造体です。

シグネチャまたは形状

typedef struct config_var_value
{
	union config_var_val val;
	void	   *extra;
} config_var_value;

入力と出力

  • **入力**: val（設定値そのもの）および extra（check_hook 等で生成された付加情報へのポインタ）。
  • **出力**: 設定変更時やスタック操作時に、変数値を一括して保持・転送するために使用されます。

前提条件と不変条件

  • val は union config_var_val 型であり、設定変数の型（bool, int, real, string, enum）に応じた値が格納されます。
  • extra フィールドは、複雑な設定値の検証や割り当て処理において、フック関数間で状態を共有するために使用されます。

アルゴリズムまたは振る舞い

  • GUC変数の値が変更される際、新しい値を保持する一時的なコンテナとして機能します。
  • SET LOCAL 等によるトランザクション内の一時的な設定変更において、以前の値（prior）やマスクされた値（masked）を保持するスタック構造の一部として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/utils/misc/guc.c: set_stack_value および discard_stack_value 関数内で引数として使用。
  • src/backend/utils/misc/guc.c: 設定変更時のローカル変数 newvalue として使用。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • union config_var_val: config_var_value 内で実際の値を保持する共用体。
  • config_generic: GUC変数の基底構造体。prior や masked フィールドの型として config_var_value を使用します。

失敗モードと境界条件

  • extra ポインタが指すメモリ領域の管理は、各変数の assign_hook および check_hook の実装に依存します。不適切なメモリ管理はメモリリークや不正な参照を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体はGUCシステムの「値の受け渡し」における標準的なインターフェースです。新しいGUC変数を追加する際や、既存のフック処理を修正する際は、val と extra の整合性が保たれていることを確認してください。
