PLy_spi_subtransaction_begin

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_spi.c:564

目的

SPI 操作を安全に実行するための内部サブトランザクションを開始し、メモリコンテキストを関数実行時のコンテキストへ切り替える。

シグネチャまたは形状

void PLy_spi_subtransaction_begin(MemoryContext oldcontext, ResourceOwner oldowner)

入力と出力

  • **入力**:
  • oldcontext: サブトランザクション開始前のメモリコンテキスト。
  • oldowner: サブトランザクション開始前のリソースオーナー（本関数内では未使用）。
  • **出力**: なし（void）。

前提条件と不変条件

  • 本関数は PG_TRY ブロック内での呼び出しを想定している。
  • 呼び出し後、メモリコンテキストは oldcontext に復帰する。

アルゴリズムまたは振る舞い

1. BeginInternalSubTransaction(NULL) を呼び出し、新しい内部サブトランザクションを開始する。 2. MemoryContextSwitchTo(oldcontext) を呼び出し、メモリコンテキストを呼び出し元のコンテキストへ切り替える。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_cursorobject.c (複数箇所)
  • src/pl/plpython/plpy_spi.c (複数箇所)
  • **呼び出し先**:
  • BeginInternalSubTransaction (PostgreSQL SPI API)
  • MemoryContextSwitchTo (PostgreSQL Memory Management API)

関連データ構造

  • MemoryContext
  • ResourceOwner

失敗モードと境界条件

  • BeginInternalSubTransaction が失敗した場合、PostgreSQL のエラーハンドリング機構により longjmp が発生し、制御が呼び出し元の PG_CATCH ブロックへ移る。

保守担当者向けメモ

  • 本関数はサブトランザクションの開始のみを担当する。対応する終了処理（コミットまたはアボート）については、それぞれ PLy_spi_subtransaction_commit および PLy_spi_subtransaction_abort を参照すること。
  • oldowner 引数は現在の実装では使用されていない。
