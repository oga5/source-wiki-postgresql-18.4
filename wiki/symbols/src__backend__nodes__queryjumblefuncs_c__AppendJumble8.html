AppendJumble8

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/queryjumblefuncs.c:319

目的

クエリのフィンガープリント（queryId）生成プロセスにおいて、8ビット（1バイト）のデータをハッシュ計算用のバッファに追加します。

シグネチャまたは形状

static pg_noinline void AppendJumble8(JumbleState *jstate, const unsigned char *value)

入力と出力

  • **入力**:
  • jstate: ハッシュ計算の状態とバッファを保持する JumbleState 構造体へのポインタ。
  • value: 追加する 1 バイトのデータ。
  • **出力**: なし（jstate 内のバッファが更新されます）。

前提条件と不変条件

  • jstate は有効な JumbleState 構造体を指している必要があります。
  • AppendJumbleInternal を呼び出す前に、保留中の NULL 値がある場合は FlushPendingNulls によってフラッシュされる必要があります。

アルゴリズムまたは振る舞い

1. jstate->pending_nulls が 0 より大きい場合、FlushPendingNulls(jstate) を呼び出して保留中の NULL 情報をバッファに書き出します。 2. AppendJumbleInternal(jstate, value, 1) を呼び出し、指定された 1 バイトのデータをハッシュバッファに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/queryjumblefuncs.c:543（ソースコード内の直接的な参照）
  • **呼び出し先**:
  • FlushPendingNulls
  • AppendJumbleInternal

関連データ構造

  • JumbleState: クエリのハッシュ計算状態を管理する構造体。

失敗モードと境界条件

  • バッファが満杯になった場合、AppendJumbleInternal の内部処理によりバッファがフラッシュされ、再利用されます。
  • pg_noinline 指定により、インライン展開が抑制されています。

保守担当者向けメモ

  • 本関数は、ノードのシリアライズにおいて 1 バイトのフィールドをハッシュ化する際に使用されます。
  • 新しいノード型を追加する際は、queryjumblefuncs.switch.c 等で本関数が適切に利用されているか確認してください。
