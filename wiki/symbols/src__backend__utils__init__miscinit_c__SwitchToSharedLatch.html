SwitchToSharedLatch

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/miscinit.c:214

目的

現在のプロセスが使用するラッチを、ローカルの LocalLatchData から共有メモリ上の MyProc->procLatch へ切り替えます。

シグネチャまたは形状

void SwitchToSharedLatch(void)

入力と出力

  • 入力: なし（グローバル変数 MyLatch、MyProc、FeBeWaitSet を参照）
  • 出力: なし（副作用としてグローバル状態を更新）

前提条件と不変条件

  • MyLatch が &LocalLatchData を指していること。
  • MyProc が NULL ではないこと。

アルゴリズムまたは振る舞い

1. MyLatch ポインタを MyProc->procLatch に更新します。 2. FeBeWaitSet が存在する場合、ModifyWaitEvent を呼び出し、待機イベントの対象ラッチを新しい共有ラッチに更新します。 3. SetLatch を呼び出し、切り替え後の共有ラッチをセット状態にします（ローカルラッチが既にセットされていた場合に備えた安全策）。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/lmgr/proc.c (2箇所)
  • 呼び出し先:
  • ModifyWaitEvent
  • SetLatch

関連データ構造

  • MyLatch: プロセスが現在監視しているラッチへのポインタ。
  • LocalLatchData: プロセスローカルなラッチ構造体。
  • MyProc: 現在のバックエンドプロセスに対応する PGPROC 構造体。
  • FeBeWaitSet: フロントエンド/バックエンド通信用の待機セット。

失敗モードと境界条件

  • MyProc が初期化されていない状態で呼び出された場合、Assert により異常終了します。
  • MyLatch が既に共有ラッチを指している状態で呼び出された場合、Assert により異常終了します。

保守担当者向けメモ

  • 本関数は、プロセスが初期化フェーズから共有メモリを利用可能な状態へ移行する際に使用されます。
  • SetLatch の呼び出しは、切り替え前のローカルラッチで発生していた可能性のあるイベントを共有ラッチへ引き継ぐための重要な処理です。
