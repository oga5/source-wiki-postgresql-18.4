CreateBlockRefTableReader

定義

  • 種別: function
  • 定義箇所: src/common/blkreftable.c:576

目的

ブロック参照テーブルを読み込むための BlockRefTableReader 構造体を初期化し、対象データのマジックナンバーを検証する。

シグネチャまたは形状

BlockRefTableReader *CreateBlockRefTableReader(
    io_callback_fn read_callback,
    void *read_callback_arg,
    char *error_filename,
    report_error_fn error_callback,
    void *error_callback_arg
)

入力と出力

  • **入力**:
  • read_callback: データを読み込むためのコールバック関数。
  • read_callback_arg: コールバックに渡す引数。
  • error_filename: エラー発生時に報告するファイル名。
  • error_callback: エラー発生時に呼び出す報告用関数。
  • error_callback_arg: エラー報告関数に渡す引数。
  • **出力**: 初期化済みの BlockRefTableReader 構造体へのポインタを返す。

前提条件と不変条件

  • palloc0 によりメモリがゼロクリアされた状態で構造体が割り当てられる。
  • 読み込まれるデータの先頭 4 バイトが BLOCKREFTABLE_MAGIC と一致しなければならない。
  • CRC32C の計算状態が INIT_CRC32C により初期化される。

アルゴリズムまたは振る舞い

1. palloc0 を使用して BlockRefTableReader 構造体をメモリ上に確保する。 2. 引数で渡されたコールバック関数およびエラー報告用の情報を構造体に格納する。 3. CRC32C の計算状態を初期化する。 4. BlockRead を介してデータの先頭からマジックナンバーを読み込む。 5. 読み込んだ値が BLOCKREFTABLE_MAGIC と一致するか確認し、不一致の場合は error_callback を実行する。 6. 正常であれば初期化済みの構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_incremental.c
  • src/backend/backup/walsummary.c
  • src/backend/backup/walsummaryfuncs.c
  • src/bin/pg_walsummary/pg_walsummary.c
  • **呼び出し先**:
  • palloc0 (メモリ確保)
  • INIT_CRC32C (CRC 初期化)
  • BlockRefTableRead (データ読み込み)
  • error_callback (エラー報告)

関連データ構造

  • BlockRefTableReader: 読み込み状態を保持するメイン構造体。
  • io_callback_fn: I/O 抽象化のための関数ポインタ型。
  • report_error_fn: エラー報告のための関数ポインタ型。

失敗モードと境界条件

  • マジックナンバーが期待値と異なる場合、error_callback が呼び出され、処理が中断される。
  • read_callback が失敗した場合の挙動は、当該コールバックの実装に依存する。

保守担当者向けメモ

  • 本関数は BlockRefTable のデシリアライズ処理の入り口である。
  • io_callback_fn を介した抽象化により、ファイルシステム以外のソースからの読み込みも可能である。
