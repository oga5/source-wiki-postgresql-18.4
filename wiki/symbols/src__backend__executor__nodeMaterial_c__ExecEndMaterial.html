ExecEndMaterial

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMaterial.c:239

目的

Material ノードの終了処理を行い、保持しているリソースの解放と子ノードのシャットダウンを実行する。

シグネチャまたは形状

void ExecEndMaterial(MaterialState *node)

入力と出力

  • **入力**: MaterialState *node（終了対象となる Material ノードの状態構造体）
  • **出力**: なし（void）

前提条件と不変条件

  • node は有効な MaterialState 構造体を指している必要がある。
  • node->tuplestorestate が NULL でない場合、それは有効な Tuplestore 状態を指している必要がある。

アルゴリズムまたは振る舞い

1. node->tuplestorestate が NULL でない場合、tuplestore_end を呼び出して Tuplestore に関連付けられたメモリやディスクリソースを解放する。 2. node->tuplestorestate を NULL に設定する。 3. outerPlanState(node) を介して子ノード（サブプラン）を取得し、ExecEndNode を呼び出して子ノードの終了処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:706（実行エンジンによるノード終了処理の一部として呼び出される）。
  • **呼び出し先**:
  • tuplestore_end（Tuplestore の解放）
  • ExecEndNode（子ノードの終了処理）

関連データ構造

  • MaterialState: Material ノードの実行状態を保持する構造体。tuplestorestate メンバを介して Tuplestore を管理する。

失敗モードと境界条件

  • node が NULL の場合、あるいは node 内のポインタが不正な場合は未定義動作となる。
  • tuplestorestate が既に NULL の場合、tuplestore_end は呼び出されず、安全にスキップされる。

保守担当者向けメモ

  • 本関数は Material ノードのクリーンアップの責任を負う。新しいリソースを MaterialState に追加した場合は、必ず本関数で解放処理を追加すること。
  • 子ノードの終了処理（ExecEndNode）は必ず最後に行う必要がある。
