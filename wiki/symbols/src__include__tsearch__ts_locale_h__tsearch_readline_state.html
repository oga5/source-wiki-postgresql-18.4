tsearch_readline_state

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/ts_locale.h:24

目的

全文検索（tsearch）関連の設定ファイル読み込みにおいて、現在の読み込み進捗、行番号、およびエラー発生時のコンテキスト情報を保持するために使用される状態管理構造体です。

シグネチャまたは形状

typedef struct
{
	FILE	   *fp;
	const char *filename;
	int			lineno;
	StringInfoData buf;
	char	   *curline;
	ErrorContextCallback cb;
} tsearch_readline_state;

入力と出力

  • **入力**: fp（ファイルポインタ）、filename（ファイル名）を保持し、読み込み処理の基点となります。
  • **出力**: buf（UTF-8形式の入力行）および curline（データベースエンコーディングに変換された現在の行）を介して、読み込まれたデータを提供します。

前提条件と不変条件

  • tsearch_readline_begin で初期化し、tsearch_readline_end でリソースを解放するペアでの使用が必須です。
  • curline は NULL、buf.data と同一、または palloc された文字列のいずれかとなります。
  • lineno は読み込みの進行に伴い、エラー報告のために逐次更新されます。

アルゴリズムまたは振る舞い

1. **初期化**: tsearch_readline_begin によりファイルを開き、エラーハンドリング用のコールバックをスタックに登録します。 2. **読み込み**: tsearch_readline が呼び出されるたびに、ファイルから1行ずつ読み込み、lineno をインクリメントします。 3. **変換**: 必要に応じて、読み込んだデータをデータベースのエンコーディングへ変換します。 4. **終了**: tsearch_readline_end により、ファイルクローズとエラーコールバックの復元が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/dict_xsyn/dict_xsyn.c、contrib/unaccent/unaccent.c、src/backend/tsearch/dict_synonym.c、src/backend/tsearch/dict_thesaurus.c、src/backend/tsearch/spell.c など。
  • **呼び出し先**: src/backend/tsearch/ts_locale.c 内の tsearch_readline_begin、tsearch_readline、tsearch_readline_end によって操作されます。

関連データ構造

  • StringInfoData: buf フィールドで使用される動的文字列バッファ。
  • ErrorContextCallback: エラー発生時に詳細なコンテキスト（ファイル名や行番号）を報告するためのコールバック構造体。

失敗モードと境界条件

  • ファイルのオープン失敗や読み込みエラーが発生した場合、ErrorContextCallback を通じてエラー情報が報告されます。
  • curline のメモリ管理（palloc された場合の解放）は、tsearch_readline_end の呼び出しによって適切に行われる必要があります。

保守担当者向けメモ

  • 本構造体は ts_locale.c の関数群と密接に結合しています。修正を行う際は、必ず begin/end のライフサイクルが維持されているか確認してください。
  • curline のポインタが指すメモリ領域の所有権が状況によって異なるため、メモリリークには特に注意が必要です。
