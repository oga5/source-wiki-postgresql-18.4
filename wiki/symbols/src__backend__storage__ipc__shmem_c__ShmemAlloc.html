ShmemAlloc

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shmem.c:151

目的

共有メモリ領域から指定されたサイズのメモリブロックを割り当てます。

シグネチャまたは形状

void *ShmemAlloc(Size size)

入力と出力

  • **入力**: size (割り当てるバイト数)
  • **出力**: 割り当てられたメモリ領域へのポインタ。
  • **戻り値**: 成功時は割り当てられたメモリ領域の先頭アドレスを返し、失敗時はエラーを発生させます。

前提条件と不変条件

  • 共有メモリ管理システムが初期化済みである必要があります。
  • 割り当てられる領域はキャッシュライン境界（CACHELINEALIGN）にアライメントされます。
  • 共有メモリは一度割り当てられると解放できません。

アルゴリズムまたは振る舞い

1. ShmemAllocRaw(size, &allocated_size) を呼び出し、共有メモリから領域の確保を試みます。 2. 確保に失敗した場合（newSpace が NULL の場合）、ERRCODE_OUT_OF_MEMORY エラーを発生させ、処理を中断します。 3. 成功した場合は、確保されたメモリ領域のポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し先**: ShmemAllocRaw
  • **呼び出し元（直接的な証拠）**:
  • src/backend/storage/ipc/shmem.c:419
  • src/backend/storage/lmgr/lwlock.c:471
  • **注記**: 共有メモリ初期化の初期段階では ShmemAlloc は使用できず、代わりに ShmemAllocUnlocked が使用されるべきである旨の言及がソース内に存在します。

関連データ構造

  • ShmemIndex: 共有メモリ上のデータ構造を管理するハッシュテーブル。
  • CACHELINEALIGN: アライメント計算に使用される定数。

失敗モードと境界条件

  • 共有メモリの空き容量が不足している場合、ereport(ERROR, ...) によりプロセスが終了します。

保守担当者向けメモ

  • 本関数はエラー発生時に ereport を発行します。エラーを発生させずに NULL を受け取りたい場合は ShmemAllocNoError を使用してください。
  • 共有メモリの動的解放機能は存在しないため、メモリリークを避ける設計が必要です。
