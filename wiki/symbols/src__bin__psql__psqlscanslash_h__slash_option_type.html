slash_option_type

定義

  • 種別: enum
  • 定義箇所: src/bin/psql/psqlscanslash.h:15

目的

psql のメタコマンド引数を解析する際、スキャナがトークンをどのように解釈・処理すべきかを指定するための列挙型です。

シグネチャまたは形状

enum slash_option_type
{
    OT_NORMAL,
    OT_SQLID,
    OT_SQLIDHACK,
    OT_FILEPIPE,
    OT_WHOLE_LINE
};

入力と出力

  • 入力: psql_scan_slash_option 関数へ引数として渡されます。
  • 出力: スキャナの解析戦略を決定するフラグとして機能します。

前提条件と不変条件

  • psql_scan_slash_option を呼び出す際に、解析対象の引数に適した列挙値を選択する必要があります。

アルゴリズムまたは振る舞い

各列挙値は以下の解析戦略に対応します：

  • OT_NORMAL: 標準的な引数として処理します。
  • OT_SQLID: SQL識別子として処理します。
  • OT_SQLIDHACK: SQL識別子として処理しますが、小文字化（downcase）を行いません。
  • OT_FILEPIPE: ファイル名またはパイプとして処理します。
  • OT_WHOLE_LINE: 行の残りの部分すべてを引数として取得します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/psqlscanslash.h 内の psql_scan_slash_option 関数宣言で利用されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • PsqlScanState: psql_scan_slash_option が状態管理に使用する構造体。

失敗モードと境界条件

  • 不適切な slash_option_type を指定した場合、メタコマンドの引数が誤って解釈される可能性があります。

保守担当者向けメモ

  • 新しいメタコマンドを追加する際は、引数の性質（識別子か、ファイルパスか、行全体か）に応じて適切なタイプを選択してください。
  • 実装の詳細は src/fe_utils/psqlscan.c を参照してください。
