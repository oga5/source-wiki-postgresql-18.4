exclude_list_item

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_checksums/pg_checksums.c:96

目的

チェックサム処理の対象外とするファイルやディレクトリを定義するための構造体です。

シグネチャまたは形状

struct exclude_list_item
{
	const char *name;
	bool		match_prefix;
};

入力と出力

  • name: 除外対象のファイル名またはパスの接頭辞（文字列）。
  • match_prefix: true の場合、name を接頭辞として前方一致で判定します。false の場合は完全一致で判定します。

前提条件と不変条件

  • name は NULL であってはなりません。
  • この構造体の配列（skip[] など）は、通常 static const として定義され、実行中に変更されることはありません。

アルゴリズムまたは振る舞い

  • ファイル走査時に、対象ファイル名がこの構造体で定義されたルールに合致するかを判定するために使用されます。
  • match_prefix が true の場合、対象ファイル名が name で始まるかを確認します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_checksums/pg_checksums.c 内の skip[] 配列定義にて使用されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。判定ロジック側（skipfile 関数など）から参照されます。
  • その他: src/backend/backup/basebackup.c および src/bin/pg_rewind/filemap.c でも同名の構造体が定義・使用されています。

関連データ構造

  • skip[]: exclude_list_item 型の静的配列。pg_checksums において除外対象リストを保持します。

失敗モードと境界条件

  • 境界条件: match_prefix を誤って設定すると、意図しないファイルまで除外対象となる可能性があります。
  • 失敗モード: 本構造体自体に起因する失敗モードはありません。

保守担当者向けメモ

  • 新しいシステムファイルや一時ファイルをチェックサム検証から除外する必要がある場合は、src/bin/pg_checksums/pg_checksums.c 内の skip[] 配列に本構造体のインスタンスを追加してください。
  • 他のツール（pg_basebackup や pg_rewind）でも同名の構造体が使用されていますが、各ツールで除外対象の定義は独立しています。
