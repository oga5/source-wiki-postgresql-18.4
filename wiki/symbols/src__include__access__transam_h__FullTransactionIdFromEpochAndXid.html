FullTransactionIdFromEpochAndXid

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/access/transam.h:70

目的

32ビットのトランザクションID（TransactionId）とエポック（epoch）を組み合わせて、64ビットの完全なトランザクションID（FullTransactionId）を構築します。

シグネチャまたは形状

static inline FullTransactionId FullTransactionIdFromEpochAndXid(uint32 epoch, TransactionId xid)

入力と出力

  • **入力**:
  • epoch (uint32): トランザクションIDのエポック値。
  • xid (TransactionId): 32ビットのトランザクションID。
  • **出力**:
  • FullTransactionId: 構築された64ビットのトランザクションID構造体。

前提条件と不変条件

  • FullTransactionId 構造体は、内部的に64ビットの value フィールドを持つものとして扱われます。
  • 入力された epoch は上位32ビットへ、xid は下位32ビットへ配置されます。

アルゴリズムまたは振る舞い

1. epoch を uint64 にキャストし、32ビット左シフトします。 2. シフトした epoch と xid のビット論理和（OR）をとり、64ビットの値を生成します。 3. 生成した値を FullTransactionId 構造体の value フィールドに格納して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_heapam.c
  • src/backend/access/transam/varsup.c
  • src/backend/access/transam/xlog.c
  • src/bin/pg_resetwal/pg_resetwal.c
  • src/include/access/gist.h
  • src/include/access/transam.h (マクロ InvalidFullTransactionId, FirstNormalFullTransactionId 等の定義内)
  • **呼び出し先**:
  • 本関数は算術演算のみを行うため、他の関数を呼び出しません。

関連データ構造

  • FullTransactionId: 64ビットのトランザクションIDを保持する構造体。
  • TransactionId: 32ビットのトランザクション識別子。

失敗モードと境界条件

  • 特段の失敗モードはありません。入力値の組み合わせがそのまま64ビット値として合成されます。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で直接利用されることを想定しています。
  • 32ビットXIDのラップアラウンド対策として、エポック管理を行う際に必須となるユーティリティです。変更時は FullTransactionId 構造体のメモリレイアウトとの整合性に注意してください。
