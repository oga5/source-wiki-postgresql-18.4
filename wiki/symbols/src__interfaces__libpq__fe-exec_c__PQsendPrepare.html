PQsendPrepare

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-exec.c:1552

目的

PostgreSQLサーバーに対して、指定されたSQLクエリをプリペアドステートメントとして準備（Parse）するリクエストを非同期に送信します。

シグネチャまたは形状

int PQsendPrepare(PGconn *conn, const char *stmtName, const char *query, int nParams, const Oid *paramTypes);

入力と出力

  • **入力**:
  • conn: データベース接続オブジェクト。
  • stmtName: 準備するステートメントの名前（NULL不可）。
  • query: 実行するSQLクエリ文字列（NULL不可）。
  • nParams: パラメータ数（0から PQ_QUERY_PARAM_MAX_LIMIT の範囲内であること）。
  • paramTypes: パラメータのOID配列（nParams > 0 の場合）。
  • **出力**:
  • 成功時は 1 を返します。
  • 失敗時は 0 を返し、接続オブジェクトにエラーメッセージが設定されます。

前提条件と不変条件

  • PQsendQueryStart が成功し、接続がクエリ送信可能な状態である必要があります。
  • パイプラインモードでない場合、この関数は Parse メッセージの後に Sync メッセージを送信します。

アルゴリズムまたは振る舞い

1. 引数のバリデーションを行い、不正な場合はエラーを記録して終了します。 2. pqAllocCmdQueueEntry を使用してコマンドキューエントリを確保します。 3. PqMsg_Parse メッセージを構築し、ステートメント名、クエリ文字列、パラメータ型情報を順次送信します。 4. パイプラインモード（PQ_PIPELINE_OFF）でない場合は、PqMsg_Sync を送信します。 5. エントリの queryclass を PGQUERY_PREPARE に設定し、クエリ文字列を保持します。 6. pqPipelineFlush を呼び出してバッファをフラッシュし、成功すればキューにエントリを追加して 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/postgres_fdw.c、src/bin/psql/common.c、src/test/modules/libpq_pipeline/libpq_pipeline.c 等から参照されています。
  • **呼び出し先**: PQsendQueryStart, pqAllocCmdQueueEntry, pqPutMsgStart, pqPuts, pqPutInt, pqPutMsgEnd, pqPipelineFlush, pqAppendCmdQueueEntry, pqRecycleCmdQueueEntry 等の内部関数を呼び出します。

関連データ構造

  • PGconn: 接続状態を保持する構造体。
  • PGcmdQueueEntry: 送信待ちのコマンドを管理するキューエントリ。

失敗モードと境界条件

  • stmtName または query が NULL の場合、エラーを返します。
  • nParams が範囲外の場合、エラーを返します。
  • メモリ不足等で pqAllocCmdQueueEntry が失敗した場合、0 を返します。
  • メッセージ送信中にネットワークエラー等が発生した場合、sendFailed ラベルへ遷移し、リソースを解放して 0 を返します。

保守担当者向けメモ

  • 本関数は非同期インターフェースです。結果の確認には PQgetResult を使用してください。
  • entry->query の保持は strdup に依存しており、メモリ不足時は NULL になる可能性があります。
  • パイプラインモード時の挙動（Sync メッセージの省略）については、conn->pipelineStatus の状態遷移と整合性を保つ必要があります。
