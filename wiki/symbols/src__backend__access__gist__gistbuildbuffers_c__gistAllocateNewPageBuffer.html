gistAllocateNewPageBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistbuildbuffers.c:180

目的

GiST インデックスのバッファリングビルドにおいて、ノードバッファ用の新しいメモリページを確保し、初期化する。

シグネチャまたは形状

static GISTNodeBufferPage *gistAllocateNewPageBuffer(GISTBuildBuffers *gfbb)

入力と出力

  • **入力**: gfbb (GISTBuildBuffers*) - ビルドバッファ全体の管理構造体。
  • **出力**: GISTNodeBufferPage* - 新規確保・初期化されたページバッファへのポインタ。

前提条件と不変条件

  • gfbb->context が有効なメモリコンテキストであること。
  • 戻り値の pageBuffer->prev は常に InvalidBlockNumber に設定される。
  • ページ内の空き領域（PAGE_FREE_SPACE）は、ブロックサイズからデータオフセットを差し引いた値で初期化される。

アルゴリズムまたは振る舞い

1. gfbb->context を使用して、BLCKSZ（ブロックサイズ）分のメモリをゼロクリア状態で確保する。 2. 確保した GISTNodeBufferPage の prev フィールドを InvalidBlockNumber に設定する。 3. マクロ PAGE_FREE_SPACE を用いて、ページ内の利用可能な空き領域を計算し、初期値を設定する。 4. 初期化済みのページポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistbuildbuffers.c 内の gistGetNodeBuffer（2箇所で呼び出し）
  • **呼び出し先**:
  • MemoryContextAllocZero（メモリ確保用）

関連データ構造

  • GISTBuildBuffers: ビルド状態を保持する構造体。
  • GISTNodeBufferPage: ノードバッファのページ構造体。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAllocZero がエラーを発生させ、プロセスが停止する可能性がある。

保守担当者向けメモ

  • 本関数は gistbuildbuffers.c 内の静的関数であり、外部からは直接呼び出されない。
  • ページ内のデータオフセット計算は PAGE_FREE_SPACE マクロに依存しているため、GISTNodeBufferPage の構造変更時には当該マクロの定義を確認すること。
