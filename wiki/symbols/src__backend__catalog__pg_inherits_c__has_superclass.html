has_superclass

定義

  • **種別**: function
  • **定義箇所**: src/backend/catalog/pg_inherits.c:376

目的

指定されたリレーション（テーブル）が、他のテーブルを継承しているかどうかを判定します。

シグネチャまたは形状

bool has_superclass(Oid relationId)

入力と出力

  • **入力**: relationId (Oid) - 検査対象のリレーション識別子。
  • **出力**: bool - 継承関係が存在する場合は true、存在しない場合は false を返します。

前提条件と不変条件

  • 呼び出し元は、対象のリレーションに対して適切なロックを保持している必要があります。
  • pg_inherits システムカタログに対する AccessShareLock を取得してスキャンを行います。

アルゴリズムまたは振る舞い

1. pg_inherits システムカタログを AccessShareLock で開きます。 2. InheritsRelidSeqnoIndexId インデックスを使用して、inhrelid が指定された relationId と一致する行を検索します。 3. systable_getnext を呼び出し、該当するタプルが1件でも存在すれば true を返します。 4. スキャンを終了し、カタログを閉じて結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/indexcmds.c:1424
  • src/backend/commands/tablecmds.c:21321
  • src/backend/commands/trigger.c:459
  • **呼び出し先**:
  • table_open / table_close
  • systable_beginscan / systable_getnext / systable_endscan
  • ScanKeyInit

関連データ構造

  • pg_inherits: 継承関係を保持するシステムカタログ。
  • InheritsRelationId: pg_inherits の OID。
  • InheritsRelidSeqnoIndexId: inhrelid をキーとするインデックス。

失敗モードと境界条件

  • 指定された relationId が存在しない場合や、継承関係がない場合は false を返します。
  • カタログスキャン中にエラーが発生した場合、PostgreSQL のエラーハンドリング機構により処理が中断される可能性があります。

保守担当者向けメモ

  • 本関数はカタログ上の直接的なエントリをチェックするため、正確な継承状態を返します。
  • 類似の has_subclass は pg_class のフラグを参照する高速なチェックですが、本関数は pg_inherits を直接参照するため、より厳密な判定が必要な場合に使用されます。
