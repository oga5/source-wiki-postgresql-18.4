ECPGdisconnect

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/ecpg/ecpglib/connect.c:680

目的

指定されたデータベース接続、またはすべてのデータベース接続を終了し、関連するリソースを解放します。

シグネチャまたは形状

bool ECPGdisconnect(int lineno, const char *connection_name)

入力と出力

  • **入力**:
  • lineno: 呼び出し元の行番号（エラー報告用）。
  • connection_name: 切断対象の接続名。"ALL" を指定すると全接続を終了します。
  • **出力**:
  • 成功時は true、失敗時は false を返します。

前提条件と不変条件

  • ECPGget_sqlca() が有効な sqlca_t 構造体を返す必要があります。
  • 接続リストの操作は connections_mutex によって排他制御されます。

アルゴリズムまたは振る舞い

1. ECPGget_sqlca() を呼び出し、SQLCA 構造体を取得します。取得失敗時は ECPG_OUT_OF_MEMORY エラーを発生させ false を返します。 2. connections_mutex をロックします。 3. connection_name が "ALL" の場合:

  • ecpg_init_sqlca() で SQLCA を初期化します。
  • all_connections リストを走査し、各接続に対して ecpg_finish() を呼び出して破棄します。

    • 4. それ以外の場合:

  • ecpg_get_connection_nr() で指定された接続を取得します。
  • ecpg_init() で接続の妥当性を確認し、成功すれば ecpg_finish() で破棄します。

    • 5. connections_mutex をアンロックし、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPGで生成されたアプリケーションコード（テストスイート等で ECPGdisconnect(__LINE__, "name") として頻繁に使用）。
  • **呼び出し先**:
  • ECPGget_sqlca()
  • ecpg_raise()
  • ecpg_init_sqlca()
  • ecpg_finish()
  • ecpg_get_connection_nr()
  • ecpg_init()
  • pthread_mutex_lock / pthread_mutex_unlock

関連データ構造

  • sqlca_t: SQL通信領域。
  • connection: 接続状態を保持する構造体。
  • all_connections: 全接続を管理する連結リストのヘッド。
  • connections_mutex: 接続リスト保護用のミューテックス。

失敗モードと境界条件

  • sqlca の取得に失敗した場合、メモリ不足エラーとして false を返します。
  • 指定された connection_name が存在しない場合、ecpg_init() が失敗し、ミューテックスを解放した上で false を返します。

保守担当者向けメモ

  • 本関数は libpq の PQfinish をラップする ecpg_finish() を呼び出すことで、物理的な接続終了とメモリ解放を行います。
  • マルチスレッド環境下では、connections_mutex によるロックが必須です。修正時はデッドロックに注意してください。
