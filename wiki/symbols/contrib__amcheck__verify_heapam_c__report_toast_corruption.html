report_toast_corruption

定義

  • 種別: function
  • 定義箇所: contrib/amcheck/verify_heapam.c:966

目的

TOAST データの検証中に検出された破損情報を、検証コンテキストの tuplestore に記録し、検証状態を破損あり（is_corrupt）に更新する。

シグネチャまたは形状

static void report_toast_corruption(HeapCheckContext *ctx, ToastedAttribute *ta, char *msg);

入力と出力

  • ctx (HeapCheckContext*): 検証中のリレーション情報や tuplestore を保持するコンテキスト。
  • ta (ToastedAttribute*): 破損が検出された TOAST 属性の情報（ブロック番号、オフセット番号、属性番号を含む）。
  • msg (char*): 検出された破損内容を示すエラーメッセージ。
  • 戻り値: なし。

前提条件と不変条件

  • ctx および ta は有効なポインタであること。
  • ctx->tupstore が初期化されており、書き込み可能であること。

アルゴリズムまたは振る舞い

1. report_corruption_internal を呼び出し、ta が保持するブロック番号 (blkno)、オフセット番号 (offnum)、属性番号 (attnum) およびエラーメッセージ msg を ctx->tupstore に格納する。 2. ctx->is_corrupt フラグを true に設定し、当該リレーションに破損が存在することを記録する。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/amcheck/verify_heapam.c 内の複数の箇所（行 1570, 1578, 1590, 1610, 1622, 1633, 1900, 1904 など）。
  • 呼び出し先: report_corruption_internal（破損情報を tuplestore に書き込むための内部ヘルパー）。

関連データ構造

  • HeapCheckContext: 検証の全体状態を管理する構造体。
  • ToastedAttribute: TOAST データの位置情報（ブロック、オフセット、属性）を保持する構造体。

失敗モードと境界条件

  • ctx->tupstore が NULL の場合や、メモリ不足により tuplestore への書き込みが失敗した場合は、基盤となる report_corruption_internal の挙動に従いエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は TOAST 属性の検証に特化した報告用関数です。ヒープタプル自体の破損報告には report_corruption を使用してください。
  • ctx->is_corrupt は一度 true になるとリセットされないため、検証プロセス全体を通じて破損の有無を追跡する不変条件として機能します。
