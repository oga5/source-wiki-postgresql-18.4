ShmemAllocUnlocked

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shmem.c:237

目的

共有メモリ領域から、ロックを取得せずに指定されたサイズのメモリブロックを割り当てます。主に共有メモリの初期化段階など、排他制御用ロック自体がまだ利用できない状況で使用されます。

シグネチャまたは形状

void *ShmemAllocUnlocked(Size size)

入力と出力

  • **入力**: size (割り当てるバイト数)
  • **出力**: 割り当てられたメモリ領域の先頭アドレス（void *）。失敗時はエラーを報告し、呼び出し元には戻りません。

前提条件と不変条件

  • ShmemSegHdr が初期化済みであること（NULL でないこと）。
  • 割り当てられる領域は MAXALIGN に従ってアライメントされます。
  • 本関数は排他制御を行わないため、呼び出し側で適切な同期が確保されている必要があります。

アルゴリズムまたは振る舞い

1. 要求された size を MAXALIGN でアライメントします。 2. ShmemSegHdr->freeoffset を現在の開始位置として取得します。 3. 新しい終了位置（newFree）を計算し、共有メモリの総容量（totalsize）を超えていないか確認します。 4. 超過している場合は ERRCODE_OUT_OF_MEMORY でエラーを報告します。 5. ShmemSegHdr->freeoffset を newFree に更新します。 6. ShmemBase にオフセットを加算したアドレスを計算し、アライメントを確認した上で返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/port/posix_sema.c
  • src/backend/port/sysv_sema.c
  • src/backend/storage/ipc/shmem.c (初期化処理内)
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行わず、マクロ MAXALIGN およびグローバル変数 ShmemSegHdr, ShmemBase を使用します。

関連データ構造

  • ShmemSegHdr: 共有メモリセグメントのヘッダー情報（空き領域管理用）。
  • ShmemBase: 共有メモリのベースアドレス。

失敗モードと境界条件

  • **メモリ不足**: 要求サイズが残りの共有メモリ容量を超える場合、ereport(ERROR, ...) によりプロセスが終了します。

保守担当者向けメモ

  • 本関数はロックを取得しないため、マルチスレッド環境や並行アクセスが発生する状況での呼び出しは厳禁です。
  • ShmemAlloc と異なり、初期化フェーズのブートストラップ用として設計されています。
