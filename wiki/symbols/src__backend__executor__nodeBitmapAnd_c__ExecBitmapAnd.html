ExecBitmapAnd

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapAnd.c:41

目的

BitmapAnd ノードに対する ExecProcNode 呼び出し規約の適用を明示的に拒否し、エラーを発生させるための関数です。

シグネチャまたは形状

static TupleTableSlot *ExecBitmapAnd(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（実行中のノードの状態）
  • **出力**: なし（常にエラーをスローするため戻り値は到達しません）

前提条件と不変条件

  • 本関数は BitmapAnd ノードの ExecProcNode フィールドに割り当てられます。
  • BitmapAnd ノードはタプルを直接返さない設計であるため、本関数が呼び出されることは想定されていません。

アルゴリズムまたは振る舞い

1. elog(ERROR, ...) を呼び出し、BitmapAnd ノードが ExecProcNode 呼び出し規約をサポートしていない旨のメッセージを出力して実行を停止します。 2. コンパイラの警告を避けるため NULL を返しますが、このコードパスには到達しません。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジンが BitmapAnd ノードに対して誤って ExecProcNode を呼び出した場合。
  • **呼び出し先**: elog（エラー報告用）。

関連データ構造

  • PlanState: 実行ノードの状態を保持する構造体。
  • BitmapAndState: BitmapAnd ノードの実行状態を管理する構造体。

失敗モードと境界条件

  • 本関数自体が、呼び出された時点で「失敗」として定義されています。

保守担当者向けメモ

  • BitmapAnd ノードの実行には、必ず MultiExecProcNode を使用してください。ExecProcNode を介した実行はサポート外です。
