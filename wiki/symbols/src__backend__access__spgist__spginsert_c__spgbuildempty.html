spgbuildempty

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spginsert.c:153

目的

SP-GiSTインデックスの初期化を行い、メタページ、ルートページ、およびNULL値用ルートページを空の状態で作成する。

シグネチャまたは形状

void spgbuildempty(Relation index)

入力と出力

  • **入力**: index (初期化対象のインデックスリレーション)
  • **出力**: なし（副作用としてストレージ上に初期化されたページが書き込まれる）

前提条件と不変条件

  • index は有効なリレーションとして渡される必要がある。
  • smgr_bulk_start_rel を使用して一括書き込みセッションが開始される。

アルゴリズムまたは振る舞い

1. smgr_bulk_start_rel を呼び出し、インデックスの初期化用書き込みセッションを開始する。 2. SpGistInitMetapage を用いてメタページを初期化し、SPGIST_METAPAGE_BLKNO に書き込む。 3. SpGistInitPage を用いてルートページ（SPGIST_LEAF）を初期化し、SPGIST_ROOT_BLKNO に書き込む。 4. SpGistInitPage を用いてNULL値用ルートページ（SPGIST_LEAF | SPGIST_NULLS）を初期化し、SPGIST_NULL_BLKNO に書き込む。 5. smgr_bulk_finish を呼び出し、一括書き込みを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgutils.c 内で ambuildempty フィールドに割り当てられている。
  • **呼び出し先**:
  • smgr_bulk_start_rel, smgr_bulk_get_buf, smgr_bulk_write, smgr_bulk_finish (ストレージ管理)
  • SpGistInitMetapage, SpGistInitPage (SP-GiST ページ初期化)

関連データ構造

  • Relation: インデックスリレーション情報。
  • BulkWriteState, BulkWriteBuffer: 一括書き込み処理に使用される構造体。

失敗モードと境界条件

  • ストレージへの書き込み失敗時にエラーが発生する可能性がある。
  • ページ番号（SPGIST_METAPAGE_BLKNO 等）の定義がストレージレイアウトと一致している必要がある。

保守担当者向けメモ

  • 本関数はインデックスの新規作成時にのみ呼び出される。
  • ページ初期化のフラグ（SPGIST_LEAF, SPGIST_NULLS）は SP-GiST のインデックス構造の根幹に関わるため、変更時は関連するスキャン処理への影響を考慮すること。
