HashJoinTableData

定義

  • 種別: struct
  • 定義箇所: src/include/executor/hashjoin.h:298

目的

PostgreSQLのハッシュ結合実行時における、メモリ内ハッシュテーブル、バッチ処理（ディスクへのスピル）、および並列ハッシュ結合の同期状態を統合管理するメイン構造体です。

シグネチャまたは形状

typedef struct HashJoinTableData {
    int nbuckets;
    int log2_nbuckets;
    int nbuckets_original;
    int nbuckets_optimal;
    int log2_nbuckets_optimal;
    union { struct HashJoinTupleData **unshared; dsa_pointer_atomic *shared; } buckets;
    bool skewEnabled;
    HashSkewBucket **skewBucket;
    int skewBucketLen;
    int nSkewBuckets;
    int *skewBucketNums;
    int nbatch;
    int curbatch;
    int nbatch_original;
    int nbatch_outstart;
    bool growEnabled;
    double totalTuples;
    double partialTuples;
    double skewTuples;
    BufFile **innerBatchFile;
    BufFile **outerBatchFile;
    Size spaceUsed;
    Size spaceAllowed;
    Size spacePeak;
    Size spaceUsedSkew;
    Size spaceAllowedSkew;
    MemoryContext hashCxt;
    MemoryContext batchCxt;
    MemoryContext spillCxt;
    HashMemoryChunk chunks;
    HashMemoryChunk current_chunk;
    dsa_area *area;
    ParallelHashJoinState *parallel_state;
    ParallelHashJoinBatchAccessor *batches;
    dsa_pointer current_chunk_shared;
} HashJoinTableData;

入力と出力

  • **入力**: 内側（Inner）リレーションのタプル群。
  • **出力**: ハッシュテーブル構築後のバケット状態、およびメモリ制限超過時に生成される一時ファイル（innerBatchFile, outerBatchFile）。

前提条件と不変条件

  • nbuckets は2のべき乗である必要があります。
  • skewBucketLen は2のべき乗である必要があります。
  • nbatch > 1 の場合のみ、innerBatchFile および outerBatchFile 配列が割り当てられます。
  • バッチ0のタプルは一時ファイルへ書き出されず、メモリ上で処理されます。

アルゴリズムまたは振る舞い

1. **構築**: 内側リレーションをスキャンし、ハッシュ値に基づきバケットへ格納。メモリ制限（spaceAllowed）を超過すると nbatch を増やし、バッチを一時ファイルへスピルします。 2. **スキュー最適化**: 頻出値（MCV）がある場合、skewBucket を用いて効率的に格納します。 3. **メモリ管理**: hashCxt（全体）、batchCxt（バッチ毎）、spillCxt（ファイル用）の3つのメモリコンテキストを使い分けます。 4. **並列処理**: dsa_area を介して共有メモリを割り当て、parallel_state を用いて複数プロセス間で構築・プローブを同期します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHash.c（palloc_object によるインスタンス生成が確認されています）。
  • **呼び出し先**: 提供されたソースコード内では、特定の関数呼び出し関係は明示されていません。

関連データ構造

  • HashJoinTupleData: タプルのリンク用ヘッダー。
  • HashSkewBucket: スキュー最適化用ハッシュテーブル。
  • ParallelHashJoinState: 並列実行時の同期制御構造体。
  • HashMemoryChunk: タプルの密な割り当て用リスト。

失敗モードと境界条件

  • メモリ不足時: nbatch を増加させ、一時ファイルへの書き出し（スピル）が発生します。
  • growEnabled が false の場合、nbatch の増加は停止します。
  • ファイル操作: BufFile はタプルが書き込まれるまで NULL のままです。

保守担当者向けメモ

  • 並列ハッシュ結合のデバッグには ParallelHashJoinState のフェーズ遷移の理解が不可欠です。
  • メモリコンテキストの使い分け（hashCxt, batchCxt, spillCxt）がメモリリーク防止の鍵となります。
