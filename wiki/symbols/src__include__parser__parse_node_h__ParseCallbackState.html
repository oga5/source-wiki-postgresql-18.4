ParseCallbackState

定義

  • 種別: struct
  • 定義箇所: src/include/parser/parse_node.h:342

目的

SQL 解析中にエラーが発生した際、エラーメッセージに適切なソースコード上の位置情報を付与するためのコンテキストを保持します。

シグネチャまたは形状

typedef struct ParseCallbackState
{
	ParseState *pstate;
	int			location;
	ErrorContextCallback errcallback;
} ParseCallbackState;

入力と出力

  • **入力**: ParseState へのポインタおよびエラー発生位置（location）。
  • **出力**: ErrorContextCallback を介して、エラー報告時に pstate が保持するソーステキストと location を用いた詳細なエラー位置情報を提供します。

前提条件と不変条件

  • pstate は有効な ParseState 構造体を指している必要があります。
  • location は解析対象のソース文字列における有効なバイトオフセットである必要があります。

アルゴリズムまたは振る舞い

1. setup_parser_errposition_callback を使用して、この構造体をエラーコールバックスタックに登録します。 2. 解析処理中にエラーが発生すると、登録されたコールバックが呼び出され、location に基づいてソーステキスト内の該当箇所が特定されます。 3. cancel_parser_errposition_callback を使用して、処理終了時にコールバックをスタックから解除します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/ 配下の各解析関数（analyze.c, parse_clause.c, parse_coerce.c, parse_func.c, parse_oper.c, parse_relation.c 等）が、局所変数としてインスタンス化し利用しています。
  • **呼び出し先**: src/backend/parser/parse_node.c 内の setup_parser_errposition_callback および cancel_parser_errposition_callback が、この構造体のフィールドを操作・参照します。

関連データ構造

  • ParseState: 解析状態を保持するメイン構造体。
  • ErrorContextCallback: PostgreSQL のエラー報告メカニズムで使用されるコールバック構造体。

失敗モードと境界条件

  • pstate が NULL の状態でコールバックが呼び出された場合、未定義の動作を引き起こす可能性があります。
  • location がソーステキストの範囲外である場合、エラー位置の特定が正しく行われない可能性があります。

保守担当者向けメモ

  • 本構造体はエラー報告の補助を目的としています。解析ロジックの変更に伴い、location の更新が適切に行われているか確認してください。
  • setup_ および cancel_ 関数と対で使用することが必須であり、スタックのアンバランスが発生しないよう注意してください。
