ua_page_stats

定義

  • 種別: struct
  • 定義箇所: contrib/pageinspect/btreefuncs.c:80

目的

B-treeインデックスのページ統計情報を取得する関数において、対象となるリレーション、ブロック番号、および処理範囲を指定するための引数構造体です。

シグネチャまたは形状

typedef struct ua_page_stats
{
	Oid			relid;
	int64		blkno;
	int64		blk_count;
	bool		allpages;
} ua_page_stats;

入力と出力

本構造体はデータ保持用であり、関数呼び出し時の引数として使用されます。

  • relid: 対象インデックスのOID。
  • blkno: 処理を開始するブロック番号。
  • blk_count: 処理対象のブロック数。
  • allpages: 全ページを対象とするかを示すフラグ。

前提条件と不変条件

  • palloc を使用してメモリ上に割り当てられることが想定されています。

アルゴリズムまたは振る舞い

本構造体は、bt_multi_page_stats 等の関数内で、ユーザーから渡された引数を内部的に保持するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pageinspect/btreefuncs.c 内の関数（bt_multi_page_stats 等）にて、palloc を介してインスタンス化されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • BTPageStat: ページ単位の統計情報を格納する構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時に palloc がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本構造体は contrib/pageinspect/btreefuncs.c 内でのみ使用される内部的なデータ構造です。変更時は同ファイル内の関連するSQL関数（bt_multi_page_stats 等）への影響を確認してください。
