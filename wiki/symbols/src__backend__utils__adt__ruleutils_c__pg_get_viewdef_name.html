pg_get_viewdef_name

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ruleutils.c:735

目的

指定されたビュー名（修飾名）からビューの定義を逆コンパイルし、人間が読み取り可能な SQL 文字列として取得します。

シグネチャまたは形状

Datum pg_get_viewdef_name(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して、ビュー名を指定する text 型の引数を受け取ります。
  • **出力**: ビューの定義を表す text 型の値を返します。ビューが見つからない場合は NULL を返します。

前提条件と不変条件

  • 入力されたビュー名は、textToQualifiedNameList を通じて修飾名リストとして解釈されます。
  • RangeVarGetRelid を使用してビューの OID を取得しますが、権限チェックを回避するため NoLock モードで検索を行います。

アルゴリズムまたは振る舞い

1. 入力された text 型のビュー名を RangeVar 構造体に変換します。 2. RangeVarGetRelid を呼び出し、対象ビューの OID を特定します。 3. PRETTYFLAG_INDENT を指定して pg_get_viewdef_worker を呼び出し、ビュー定義の逆コンパイルを実行します。 4. 逆コンパイル結果が NULL の場合は SQL の NULL を返します。 5. 結果が取得できた場合、string_to_text を介して text 型に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されます（具体的な呼び出し元コードの詳細は提供されていません）。
  • **呼び出し先**:
  • makeRangeVarFromNameList
  • textToQualifiedNameList
  • RangeVarGetRelid
  • pg_get_viewdef_worker
  • string_to_text

関連データ構造

  • RangeVar: ビュー名を特定するための構造体。
  • Datum: PostgreSQL の汎用データ型。
  • text: PostgreSQL の可変長文字列型。

失敗モードと境界条件

  • 指定されたビュー名が存在しない場合、RangeVarGetRelid の第3引数が false であるため、エラーが発生します。
  • pg_get_viewdef_worker が NULL を返した場合、関数は安全に SQL の NULL を返します。

保守担当者向けメモ

  • 本関数は pg_get_viewdef 系のバリエーションの一つであり、名前による検索に特化しています。
  • 権限チェックを行わずに OID を取得する設計となっているため、呼び出し時には注意が必要です。
