mdc_init

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-encrypt.c:91

目的

PGP暗号化パイプラインにおけるMDC（Modification Detection Code）計算用のSHA-1ハッシュコンテキストを初期化する。

シグネチャまたは形状

static int mdc_init(PushFilter *dst, void *init_arg, void **priv_p)

入力と出力

  • **入力**:
  • dst: 出力フィルタ（本関数内では未使用）。
  • init_arg: 初期化引数（本関数内では未使用）。
  • priv_p: 初期化されたハッシュコンテキストを格納するポインタ。
  • **出力**:
  • 成功時は 0 を返し、*priv_p に PX_MD 構造体へのポインタを格納する。
  • 失敗時は負の整数（エラーコード）を返す。

前提条件と不変条件

  • pgp_load_digest が PGP_DIGEST_SHA1 をサポートしている必要がある。

アルゴリズムまたは振る舞い

1. pgp_load_digest を呼び出し、SHA-1 アルゴリズムのハッシュコンテキストをロードする。 2. ロードに成功した場合、取得した PX_MD 構造体へのポインタを *priv_p に設定する。 3. 成功時に 0 を返し、処理を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-encrypt.c 内のフィルタチェーン構築処理から参照されている。
  • **呼び出し先**: pgp_load_digest（ハッシュコンテキストの生成）。
  • 注: pgp-decrypt.c にも同名のシンボルが存在するが、本ページは pgp-encrypt.c の定義を対象としている。

関連データ構造

  • PX_MD: 暗号プリミティブ（ハッシュ）のコンテキストを保持する構造体。
  • PushFilter: PGP暗号化パイプラインを構成するフィルタ構造体。

失敗モードと境界条件

  • pgp_load_digest が失敗した場合（例: メモリ不足やアルゴリズム未サポート）、そのエラーコードをそのまま呼び出し元へ伝播させる。

保守担当者向けメモ

  • 本関数は mdc_filter の初期化ルーチンとして機能する。
  • priv_p に格納されたコンテキストは、後続の mdc_write や mdc_free で使用されるため、ライフサイクル管理に注意すること。
