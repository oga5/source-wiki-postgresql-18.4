gbt_var_key_from_datum

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_utils_var.c:72

目的

可変長データ（varlena）を GiST インデックス用の GBT_VARKEY 形式へ変換し、メモリ上に複製します。

シグネチャまたは形状

static GBT_VARKEY *gbt_var_key_from_datum(const struct varlena *u)

入力と出力

  • **入力**: u - 変換対象の varlena 構造体ポインタ。
  • **出力**: 変換後のデータを格納した GBT_VARKEY 構造体へのポインタ。

前提条件と不変条件

  • 入力 u は有効な varlena 形式である必要があります。
  • 戻り値の GBT_VARKEY は palloc によってメモリ確保されるため、呼び出し側で適切に管理する必要があります。

アルゴリズムまたは振る舞い

1. 入力 u のサイズを VARSIZE(u) で取得します。 2. palloc を使用して、元のサイズにヘッダーサイズ（VARHDRSZ）を加えた領域を確保します。 3. memcpy を用いてデータを確保した領域にコピーします。 4. SET_VARSIZE マクロを使用して、新しい領域のサイズを正しく設定します。 5. 変換された GBT_VARKEY ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_utils_var.c 内の関数（例: 300行目付近）から呼び出されています。
  • **呼び出し先**: PostgreSQL のメモリ管理関数 palloc および varlena 関連マクロ（VARSIZE, VARDATA, SET_VARSIZE）。

関連データ構造

  • GBT_VARKEY: GiST インデックスで可変長データを扱うための内部構造体。
  • varlena: PostgreSQL の可変長データ型を表現する標準構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • 入力 u が NULL の場合、VARSIZE マクロの実行によりセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は static であり、btree_utils_var.c 内でのみ使用されます。
  • VARHDRSZ を加算してサイズを再定義しているため、インデックス格納時にヘッダー情報が保持される点に注意してください。
