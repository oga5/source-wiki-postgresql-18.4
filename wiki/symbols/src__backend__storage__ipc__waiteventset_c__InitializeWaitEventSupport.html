InitializeWaitEventSupport

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/waiteventset.c:240

目的

プロセス起動時に、OS 固有のイベント通知メカニズム（セルフパイプ、signalfd、または kqueue）を初期化し、シグナルハンドラによる起床を可能にするための基盤を構築します。

シグネチャまたは形状

void InitializeWaitEventSupport(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルなファイル記述子やシグナル設定を更新）

前提条件と不変条件

  • プロセス起動時に一度だけ呼び出されることを想定しています。
  • 子プロセスが親プロセスから継承した古いパイプや signalfd を保持している場合、それらを適切にクローズしてリソースを解放します。
  • fd.c によるファイル記述子の管理（ReserveExternalFD / ReleaseExternalFD）と整合性を保つ必要があります。

アルゴリズムまたは振る舞い

コンパイル時のフラグに応じて以下のいずれかの処理を行います。 1. **WAIT_USE_SELF_PIPE**:

  • pipe() でパイプを作成し、読み込み側と書き込み側の両方を非ブロッキングかつ FD_CLOEXEC に設定します。
  • latch_sigurg_handler を SIGURG シグナルに割り当てます。

    • 2. **WAIT_USE_SIGNALFD**:

  • signalfd() を使用して SIGURG を受信するように設定し、非ブロッキングかつ FD_CLOEXEC で開きます。
  • SIGURG をシグナルマスクから除外します。

    • 3. **WAIT_USE_KQUEUE**:

  • SIGURG を無視するように設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/postmaster.c
  • src/backend/utils/init/miscinit.c
  • **呼び出し先**:
  • pipe(), fcntl(), close(), signalfd(), pqsignal(), ReserveExternalFD(), ReleaseExternalFD() 等のシステムコールおよび内部関数。

関連データ構造

  • selfpipe_readfd, selfpipe_writefd, selfpipe_owner_pid (セルフパイプ使用時)
  • signal_fd (signalfd 使用時)
  • UnBlockSig (シグナルマスク)

失敗モードと境界条件

  • pipe() や signalfd() の呼び出しが失敗した場合、elog(FATAL, ...) を呼び出しプロセスを終了させます。
  • 子プロセスにおいて、親から継承した古いファイル記述子が適切にクローズされない場合、リソースリークが発生する可能性があります。

保守担当者向けメモ

  • 本関数はプラットフォーム依存のコードを多く含むため、OS ごとのイベント通知の仕組み（epoll vs kqueue vs poll）を理解した上で修正してください。
  • fd.c の管理下にあるファイル記述子を扱うため、ReserveExternalFD の呼び出し回数と close の整合性に注意してください。
