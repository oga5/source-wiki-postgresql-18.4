HandleEndOfCopyStream

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/receivelog.c:1177

目的

サーバー側から COPY ストリームが終了された際、ローカル側のストリーム終了処理および最終的なコマンド結果の取得を行う。

シグネチャまたは形状

static PGresult *HandleEndOfCopyStream(PGconn *conn, StreamCtl *stream, char *copybuf, XLogRecPtr blockpos, XLogRecPtr *stoppos)

入力と出力

  • **入力**:
  • conn: サーバーとの接続ハンドル。
  • stream: ストリーミング制御構造体。
  • copybuf: 受信バッファ（本関数内では未使用）。
  • blockpos: 現在のWAL書き込み位置。
  • stoppos: 終了位置を格納するためのポインタ。
  • **出力**:
  • 成功時: サーバーからの最終的な PGresult。
  • 失敗時: NULL。

前提条件と不変条件

  • サーバーが COPY ストリームを閉じた後に呼び出されることを想定している。
  • still_sending 静的変数が、ローカル側がまだ送信中であるかどうかの状態を保持している。

アルゴリズムまたは振る舞い

1. PQgetResult を呼び出し、サーバーからの最終結果を取得する。 2. still_sending が真の場合:

  • close_walfile を呼び出し、現在のWALファイルを閉じる。失敗した場合は NULL を返す。
  • サーバーからの結果が PGRES_COPY_IN であれば、PQputCopyEnd を使用して終了パケットを送信し、PQflush でフラッシュする。失敗した場合はエラーログを出力し NULL を返す。
  • still_sending を偽に更新する。

    • 3. stoppos に blockpos を代入する。 4. 取得した PGresult を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/receivelog.c:817
  • **呼び出し先**:
  • PQgetResult, PQresultStatus, PQputCopyEnd, PQflush, PQclear, PQerrorMessage (libpq)
  • close_walfile (同ファイル内)
  • pg_log_error (common/logging.h)

関連データ構造

  • PGconn: libpqの接続オブジェクト。
  • StreamCtl: ストリーミングの動作を定義する構造体。
  • XLogRecPtr: WALのログ位置を示す型。

失敗モードと境界条件

  • close_walfile が失敗した場合、エラーメッセージを出力し NULL を返す。
  • PQputCopyEnd または PQflush が失敗した場合、エラーログを出力し NULL を返す。
  • サーバーがエラーを投げた場合、still_sending が真であっても close_walfile や PQputCopyEnd はスキップされる。

保守担当者向けメモ

  • 本関数は静的変数 still_sending に依存しているため、並列実行や再入には注意が必要である。
  • copybuf 引数は現在の実装では使用されていないが、シグネチャに含まれている。
