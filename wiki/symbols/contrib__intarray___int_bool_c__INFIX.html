INFIX

定義

  • 種別: struct
  • 定義箇所: contrib/intarray/_int_bool.c:595

目的

クエリツリーを文字列形式（インフィックス記法）へ変換する際、変換処理の状態を保持するための構造体です。

シグネチャまたは形状

typedef struct {
    ITEM   *curpol;    /* 現在処理中のポーランド記法アイテム */
    char   *buf;       /* 出力バッファの先頭ポインタ */
    char   *cur;       /* 出力バッファの現在位置ポインタ */
    int32   buflen;    /* バッファの残り容量 */
} INFIX;

入力と出力

  • **入力**: ITEM 配列（ポーランド記法）を走査するためのポインタと、結果を格納するバッファ領域。
  • **出力**: buf に書き込まれたインフィックス形式の文字列。

前提条件と不変条件

  • curpol は有効な ITEM 配列を指している必要があります。
  • buf は十分な長さのメモリ領域を確保している必要があります。
  • cur は常に buf の範囲内を指し、buflen はバッファの境界を管理するために使用されます。

アルゴリズムまたは振る舞い

本構造体は、再帰的なクエリ変換処理において、現在の走査位置（curpol）と出力バッファの現在位置（cur）を追跡するために使用されます。infix() 関数などの変換ロジックが、この構造体を参照・更新しながらクエリツリーを走査し、演算子とオペランドを適切な順序でバッファに書き出します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/intarray/_int_bool.c 内の infix() 関数が本構造体を受け取り、変換処理を行います。また、同ファイル内の関数で INFIX 型の変数が宣言されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。
  • **注記**: contrib/ltree/ltxtquery_io.c および src/backend/utils/adt/tsquery.c にも同名の構造体定義が存在しますが、これらは別モジュールの定義であり、本シンボルとの直接的な依存関係はソースコード上からは確認できません。

関連データ構造

  • ITEM: クエリツリーの各ノード（値または演算子）を保持する構造体。

失敗モードと境界条件

  • buflen を超える書き込みが発生した場合、バッファオーバーフローのリスクがあります。
  • curpol が不正なメモリ領域を指している場合、セグメンテーション違反が発生します。

保守担当者向けメモ

  • 本構造体は ltree や tsquery モジュールでも類似の目的で使用されています。変更を加える際は、他のモジュールとの定義の競合や、共通化の可能性について考慮してください。
