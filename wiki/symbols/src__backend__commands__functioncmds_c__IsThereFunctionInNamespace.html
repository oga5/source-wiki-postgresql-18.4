IsThereFunctionInNamespace

定義

  • 種別: function
  • 定義箇所: src/backend/commands/functioncmds.c:2060

目的

指定された名前空間（スキーマ）内に、特定の名前と引数型を持つ関数が既に存在するかを確認し、存在する場合はエラーを発生させることで重複定義を防止する。

シグネチャまたは形状

void IsThereFunctionInNamespace(const char *proname, int pronargs, oidvector *proargtypes, Oid nspOid)

入力と出力

  • **入力**:
  • proname: 関数名（文字列）。
  • pronargs: 引数の数。
  • proargtypes: 引数の型を保持する oidvector。
  • nspOid: 検索対象のスキーマの OID。
  • **出力**: なし。関数が既に存在する場合は ereport(ERROR, ...) により処理が中断される。

前提条件と不変条件

  • proargtypes は有効な oidvector である必要がある。
  • nspOid は有効なスキーマ OID である必要がある。

アルゴリズムまたは振る舞い

1. PROCNAMEARGSNSP システムキャッシュを使用して、指定された名前、引数型、スキーマの組み合わせを持つエントリを検索する。 2. SearchSysCacheExists3 が真を返す場合、関数が既に存在すると判断する。 3. 重複が検出された場合、funcname_signature_string を用いてエラーメッセージを生成し、ERRCODE_DUPLICATE_FUNCTION エラーを投げる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/alter.c (関数名の変更や属性変更時の重複チェック)
  • **呼び出し先**:
  • SearchSysCacheExists3 (システムカタログ検索)
  • funcname_signature_string (エラーメッセージ用文字列生成)
  • get_namespace_name (スキーマ名取得)
  • ereport (エラー報告)

関連データ構造

  • oidvector: 関数の引数型リストを保持する構造体。
  • pg_proc: 関数定義を格納するシステムカタログ（キャッシュ検索対象）。

失敗モードと境界条件

  • 指定された関数が既に存在する場合、ERRCODE_DUPLICATE_FUNCTION でエラー終了する。
  • 検索対象のスキーマが存在しない場合、get_namespace_name が NULL を返す可能性がある（エラーメッセージのスキーマ名部分が NULL になる）。

保守担当者向けメモ

  • 本関数は、一意制約違反によるシステムカタログのインデックスエラーよりも、ユーザーにとって分かりやすいエラーメッセージを提供することを目的としている。
  • pg_proc の検索キー（名前、引数、スキーマ）に変更がある場合は、SearchSysCacheExists3 の引数も更新する必要がある。
