ltxtq_exec

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltxtquery_op.c:82

目的

PostgreSQL の ltree 型と ltxtquery 型を照合し、クエリ条件がパスに適合するかを判定する SQL 呼び出し可能な演算子関数（@ 演算子）。

シグネチャまたは形状

Datum ltxtq_exec(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の引数を受け取る。
  • 第1引数: ltree *（評価対象のパス）
  • 第2引数: ltxtquery *（評価するブール条件クエリ）
  • **出力**: bool 型の評価結果を Datum として返す。

前提条件と不変条件

  • ltxtquery はポーランド記法（後置記法）で格納されている必要がある。
  • ltree_execute 内部でスタック深さのチェックが行われるため、実行環境のスタック制限に依存する。

アルゴリズムまたは振る舞い

1. CHKVAL 構造体を初期化し、対象の ltree と ltxtquery 内のオペランド文字列を格納する。 2. ltree_execute を呼び出し、クエリの ITEM 配列を再帰的に走査する。 3. 走査中、VAL 型のノードに到達するたびにコールバック関数 checkcondition_str を実行し、個別の条件一致判定を行う。 4. 最終的な論理演算結果（true または false）を PG_RETURN_BOOL で返す。 5. 入力引数に対して PG_FREE_IF_COPY を実行し、メモリを適切に解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/_ltree_op.c（array_iterator 経由）
  • contrib/ltree/ltree_gist.c（DirectFunctionCall2 経由）
  • contrib/ltree/ltxtquery_op.c（DirectFunctionCall2 経由）
  • **呼び出し先**:
  • ltree_execute
  • checkcondition_str

関連データ構造

  • ltree: 評価対象のパスデータ。
  • ltxtquery: 検索条件を保持する構造体。
  • CHKVAL: 照合コンテキストを保持する構造体。

失敗モードと境界条件

  • ltree または ltxtquery が不正な形式の場合、ltree_execute 内部でエラーが発生する可能性がある。
  • 非常に深いネストを持つクエリの場合、スタックオーバーフローが発生する可能性がある（miscadmin.h の check_stack_depth() による保護あり）。

保守担当者向けメモ

  • 本関数は ltxtquery のポーランド記法構造に強く依存している。ロジックを変更する際は ltree_execute の走査仕様を確認すること。
  • PG_FREE_IF_COPY を使用しているため、メモリ管理の整合性に注意が必要。
