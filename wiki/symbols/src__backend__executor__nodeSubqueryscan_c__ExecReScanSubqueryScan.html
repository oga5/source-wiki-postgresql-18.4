ExecReScanSubqueryScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSubqueryscan.c:182

目的

SubqueryScan ノードの再スキャン（ReScan）を実行し、パラメータ変更に伴うサブプランの再初期化と再実行を制御する。

シグネチャまたは形状

void ExecReScanSubqueryScan(SubqueryScanState *node)

入力と出力

  • **入力**: SubqueryScanState *node（実行中のサブクエリスキャンノードの状態構造体）。
  • **出力**: なし（副作用としてノードの状態を更新する）。

前提条件と不変条件

  • node は初期化済みであること。
  • サブプラン（node->subplan）は、独自のメモリコンテキストで chgParam 状態を保持している。

アルゴリズムまたは振る舞い

1. ExecScanReScan(&node->ss) を呼び出し、スキャンノード共通の再スキャン処理を実行する。 2. node->ss.ps.chgParam が存在する場合、UpdateChangedParamSet を呼び出し、変更されたパラメータをサブプランに伝播させる。 3. サブプランの chgParam が NULL の場合、ExecReScan(node->subplan) を呼び出し、サブプランを強制的に再スキャンさせる。

  • chgParam が NULL でない場合、サブプランは最初の ExecProcNode 呼び出し時に自動的に再スキャンされる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:206（外部からの再スキャン要求）。
  • **呼び出し先**:
  • ExecScanReScan（共通基盤）
  • UpdateChangedParamSet（パラメータ伝播）
  • ExecReScan（サブプランの再実行）

関連データ構造

  • SubqueryScanState: サブクエリ実行時の状態を保持する構造体。
  • PlanState: node->ss.ps を通じてアクセスされる実行プランの状態。

失敗モードと境界条件

  • サブプランが NULL の場合や、node が正しく初期化されていない場合は未定義の動作となる。

保守担当者向けメモ

  • ExecReScan はサブプランのメモリコンテキストを考慮する必要があるため、本関数で明示的に chgParam のシグナリングを行っている。
  • SubPlan と SubqueryScan のコード統合は長年の課題であるが、現状では本関数が個別に再スキャンロジックを管理している。
