next_insert

定義

  • 種別: static int
  • 定義箇所: src/interfaces/ecpg/ecpglib/execute.c:110

目的

SQL 文の文字列内において、次に処理すべきホスト変数プレースホルダー（$n）または旧形式のプレースホルダー（?）の開始位置を検索します。

シグネチャまたは形状

static int next_insert(char *text, int pos, bool questionmarks, bool std_strings)

入力と出力

  • **入力**:
  • text: 検索対象の SQL 文文字列。
  • pos: 検索を開始するインデックス位置。
  • questionmarks: ? をプレースホルダーとして許可するかどうか。
  • std_strings: standard_conforming_strings 設定（エスケープ文字の扱いを決定）。
  • **出力**:
  • プレースホルダーが見つかった場合はその開始インデックス（int）。
  • 見つからなかった場合は -1。

前提条件と不変条件

  • 文字列内のクォート（'）で囲まれた範囲は、プレースホルダーの検索対象から除外されます。
  • std_strings が false の場合、バックスラッシュ（\）はエスケープ文字として扱われ、直後の文字をスキップします。

アルゴリズムまたは振る舞い

1. 指定された pos から文字列の終端まで走査します。 2. クォート内にある間はプレースホルダーを無視します。 3. クォート外において、以下のいずれかを検出した位置を返します。

  • $ で始まり、数字が続く形式（ただし、その後に英字やアンダースコアが続く場合はドル引用符とみなしてスキップ）。
  • questionmarks が有効な場合、? 文字。

    • 4. 終端に達した場合は -1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/execute.c 内の SQL 文解析処理（ECPGdo 関連の内部処理）から呼び出されます。
  • **呼び出し先**: 標準ライブラリの isdigit, isalpha, isascii を使用します。

関連データ構造

  • statement 構造体（stmt->command や stmt->questionmarks を通じて本関数へ情報が渡されます）。

失敗モードと境界条件

  • 文字列が空、またはプレースホルダーが存在しない場合は -1 を返します。
  • ドル記号（$）が数字を伴わない場合や、ドル引用符の構文と混同される可能性がある場合は、正しくプレースホルダーとして認識されないことがあります。

保守担当者向けメモ

  • 本関数は SQL 文の構文解析における字句スキャンの一部です。ドル引用符（$tag$...$tag$）の完全な解析ロジックは含まれていないため、複雑な SQL 文を扱う際は注意が必要です。
