searchChar

定義

  • 種別: function
  • 定義箇所: src/test/modules/spgist_name_ops/spgist_name_ops.c:96

目的

SP-GiSTインデックスのノードラベル配列から、指定された文字（int16）を二分探索し、一致する位置または挿入すべき位置を特定します。

シグネチャまたは形状

static bool searchChar(Datum *nodeLabels, int nNodes, int16 c, int *i)

入力と出力

  • **入力**:
  • nodeLabels: 検索対象となる Datum 型のノードラベル配列。
  • nNodes: 配列内の要素数。
  • c: 検索対象の文字（int16）。
  • i: 結果を格納するためのポインタ（出力用）。
  • **出力**:
  • 戻り値: 指定した文字 c が配列内に存在すれば true、存在しなければ false。
  • *i: 戻り値が true の場合は一致したインデックス、false の場合は c を挿入すべき位置（StopHigh）が格納されます。

前提条件と不変条件

  • nodeLabels は昇順にソートされている必要があります。
  • nNodes は配列の有効な要素数である必要があります。

アルゴリズムまたは振る舞い

1. StopLow（0）と StopHigh（nNodes）を境界として二分探索を実行します。 2. 中央値 StopMiddle を計算し、nodeLabels[StopMiddle] と c を比較します。 3. c が小さい場合は StopHigh を更新、大きい場合は StopLow を更新して範囲を絞り込みます。 4. 一致した場合はそのインデックスを *i に代入し true を返します。 5. 探索範囲がなくなった場合、*i に StopHigh を代入し false を返します。

関連データ構造

  • Datum: PostgreSQLの汎用データ型。ここでは int16 を保持するために使用されます。

失敗モードと境界条件

  • nNodes が 0 の場合、ループに入らず *i に 0 が代入され false が返されます。
  • 配列内に c が存在しない場合、挿入に適した適切な位置が *i に返されます。
