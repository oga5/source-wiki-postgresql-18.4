av_relation

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/autovacuum.c:190

目的

自動バキューム処理において、特定のテーブル（またはTOASTテーブル）に関連する設定オプションや識別情報を保持するための構造体です。

シグネチャまたは形状

typedef struct av_relation
{
    Oid         ar_toastrelid;  /* ハッシュキー - 先頭である必要がある */
    Oid         ar_relid;
    bool        ar_hasrelopts;
    AutoVacOpts ar_reloptions;  /* メインテーブルのreloptionsのコピー、またはNULL */
} av_relation;

入力と出力

  • 本構造体は、ハッシュテーブルのエントリとして使用されます。
  • ar_toastrelid をキーとして、テーブル固有のバキューム設定（AutoVacOpts）を管理します。

前提条件と不変条件

  • ar_toastrelid はハッシュテーブルのキーとして機能するため、構造体の先頭に配置される必要があります。

アルゴリズムまたは振る舞い

  • 自動バキュームのワーカープロセスが、処理対象テーブルのオプション情報をハッシュテーブルから検索・取得する際に利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/autovacuum.c 内のハッシュテーブル操作箇所（1967, 2056, 2121, 2778 行付近）で参照されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • AutoVacOpts: テーブルのバキューム設定オプションを保持する構造体。

失敗モードと境界条件

  • ハッシュテーブルのサイズ設定（ctl.entrysize）において、本構造体のサイズ（sizeof(av_relation)）と整合性が取れている必要があります。

保守担当者向けメモ

  • ar_toastrelid がハッシュキーとして機能するため、構造体のメンバ順序を変更する際は注意してください。
  • 本構造体は autovacuum.c 内のローカルなハッシュテーブル管理に依存しています。
