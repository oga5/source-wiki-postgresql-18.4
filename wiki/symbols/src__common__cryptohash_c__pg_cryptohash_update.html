pg_cryptohash_update

定義

  • 種別: function
  • 定義箇所: src/common/cryptohash.c:135

目的

pg_cryptohash_ctx に保持されたハッシュ計算コンテキストに対し、入力データを逐次的に追加してハッシュ値を更新します。

シグネチャまたは形状

int pg_cryptohash_update(pg_cryptohash_ctx *ctx, const uint8 *data, size_t len);

入力と出力

  • **入力**:
  • ctx: ハッシュ計算状態を保持するコンテキストへのポインタ。
  • data: ハッシュ計算対象となるデータのバッファ。
  • len: data のバイト長。
  • **出力**:
  • 成功時は 0 を返します。
  • ctx が NULL の場合は -1 を返します。

前提条件と不変条件

  • ctx は pg_cryptohash_create() によって適切に初期化されている必要があります。
  • ctx->type に基づき、対応するアルゴリズムの内部更新関数が呼び出されます。

アルゴリズムまたは振る舞い

1. ctx が NULL かどうかを検証し、該当する場合は即座にエラーを返します。 2. ctx->type の値を参照し、以下のいずれかのアルゴリズム固有の更新関数へ処理を委譲します。

  • PG_MD5: pg_md5_update
  • PG_SHA1: pg_sha1_update
  • PG_SHA224: pg_sha224_update
  • PG_SHA256: pg_sha256_update
  • PG_SHA384: pg_sha384_update
  • PG_SHA512: pg_sha512_update

    • 3. 処理完了後、0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/uuid-ossp/uuid-ossp.c
  • src/backend/backup/backup_manifest.c
  • src/backend/libpq/auth-scram.c
  • src/backend/utils/adt/cryptohashfuncs.c
  • src/common/checksum_helper.c
  • **呼び出し先**:
  • pg_md5_update, pg_sha1_update, pg_sha224_update, pg_sha256_update, pg_sha384_update, pg_sha512_update (各アルゴリズムの内部実装)

関連データ構造

  • pg_cryptohash_ctx: ハッシュアルゴリズムの種類と、各アルゴリズム固有のコンテキストデータを保持する不透明な構造体。

失敗モードと境界条件

  • ctx が NULL の場合、計算を行わず -1 を返します。
  • ctx->type が未知の値である場合、関数は何も行わず 0 を返します（スイッチ文のデフォルト動作）。

保守担当者向けメモ

  • 本関数は src/common/cryptohash.c における抽象化レイヤーの一部です。新しいハッシュアルゴリズムを追加する場合は、pg_cryptohash_ctx の定義更新と併せて、本関数の switch 文にケースを追加する必要があります。
