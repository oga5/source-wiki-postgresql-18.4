pg_open_tzfile

定義

  • 種別: function
  • 定義箇所: src/timezone/pgtz.c:75

目的

指定されたタイムゾーン名に対応するファイルシステム上のファイルを検索し、ファイル記述子を返します。大文字小文字を区別しない検索をサポートし、必要に応じて正規化された名前を取得します。

シグネチャまたは形状

int pg_open_tzfile(const char *name, char *canonname);

入力と出力

  • **入力**:
  • name: 検索対象のタイムゾーン名（例: "UTC", "Asia/Tokyo"）。
  • canonname: 正規化された名前を格納するためのバッファ（NULL指定可能）。
  • **出力**:
  • 成功時: ファイル記述子（int）。
  • 失敗時: -1。

前提条件と不変条件

  • pg_TZDIR() が有効なタイムゾーンディレクトリパスを返すこと。
  • name の長さが MAXPGPATH の制限内に収まること。
  • canonname が提供される場合、そのバッファサイズは TZ_STRLEN_MAX + 1 以上であること。

アルゴリズムまたは振る舞い

1. pg_TZDIR() を基点として検索パスを初期化します。 2. canonname が NULL の場合、まず指定された名前で直接 open() を試みます。 3. 直接のオープンに失敗した場合、または canonname が指定されている場合、パスをディレクトリレベルで分割し、scan_directory_ci() を用いて大文字小文字を区別せずに再帰的に検索します。 4. 検索成功時、canonname が指定されていれば、見つかったパスから正規化された名前をコピーします。 5. 最終的に open() を呼び出し、ファイル記述子を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/initdb/findtimezone.c
  • src/timezone/localtime.c
  • **呼び出し先**:
  • pg_TZDIR()
  • scan_directory_ci()
  • open() (システムコール)

関連データ構造

  • MAXPGPATH: パス名の最大長を規定する定数。
  • TZ_STRLEN_MAX: タイムゾーン名の最大長を規定する定数。

失敗モードと境界条件

  • 指定された名前が MAXPGPATH を超える場合、-1 を返します。
  • 該当するタイムゾーンファイルが見つからない場合、-1 を返します。
  • ディレクトリ走査中にエラーが発生した場合、-1 を返します。

保守担当者向けメモ

  • 本関数は pg_TZDIR() に依存しており、環境変数やビルド設定によるタイムゾーンディレクトリの変更が検索結果に影響します。
  • scan_directory_ci() を使用しているため、ファイルシステムが大文字小文字を区別しない環境であっても、一貫した検索結果が得られるよう設計されています。
