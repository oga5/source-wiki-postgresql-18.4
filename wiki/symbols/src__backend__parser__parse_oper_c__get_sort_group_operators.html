get_sort_group_operators

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_oper.c:179

目的

指定されたデータ型（argtype）に対して、ソートやグループ化操作に必要なデフォルトの演算子（<, =, >）およびハッシュ可能性をシステムカタログから取得する。

シグネチャまたは形状

void get_sort_group_operators(Oid argtype,
                              bool needLT, bool needEQ, bool needGT,
                              Oid *ltOpr, Oid *eqOpr, Oid *gtOpr,
                              bool *isHashable)

入力と出力

  • **入力**:
  • argtype: 対象のデータ型の OID。
  • needLT, needEQ, needGT: それぞれ小なり、等価、大なり演算子が必要かを示すフラグ。
  • **出力**:
  • ltOpr, eqOpr, gtOpr: 取得された演算子の OID を格納するポインタ。
  • isHashable: 型がハッシュ可能かどうかの結果を格納するポインタ（NULL でない場合のみ計算）。

前提条件と不変条件

  • typcache.c の検索アルゴリズムにより、返される演算子はすべて同一の演算子クラス（opclass）から取得されることが保証される。
  • 演算子が見つからない場合、関数はエラーを報告し、呼び出し元へは戻らない。

アルゴリズムまたは振る舞い

1. isHashable が要求されている場合、TYPECACHE_HASH_PROC を含むフラグを生成する。 2. lookup_type_cache を呼び出し、型キャッシュから演算子情報を取得する。 3. 取得した lt_opr, eq_opr, gt_opr が有効か確認する。 4. needLT, needEQ, needGT で指定された演算子が欠落している場合、ERRCODE_UNDEFINED_FUNCTION でエラーを発生させる。 5. 有効な演算子 OID を出力ポインタに書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/analyze.c
  • src/backend/parser/analyze.c
  • src/backend/parser/parse_clause.c
  • **呼び出し先**:
  • lookup_type_cache (utils/typcache.h)
  • ereport (postgres.h)

関連データ構造

  • TypeCacheEntry: 型に関連する演算子やプロシージャを保持するキャッシュ構造体。

失敗モードと境界条件

  • 指定された型に対して必要な演算子（needLT 等で指定）が定義されていない場合、ereport(ERROR, ...) により処理が中断される。
  • isHashable が NULL の場合、ハッシュ関連の処理はスキップされる。

保守担当者向けメモ

  • 本関数は pg_operator や pg_cast の変更時にキャッシュが無効化される仕組み（InvalidateOprCacheCallBack）に依存している。
  • 演算子の不一致や欠落に関するエラーメッセージは、ユーザーに対して明示的な演算子の指定を促す内容となっている。
