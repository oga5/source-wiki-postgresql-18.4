MemoryContextDelete

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/mcxt.c:459

目的

指定されたメモリコンテキストおよびそのすべての子孫コンテキストを破棄し、関連付けられたメモリを解放します。

シグネチャまたは形状

void MemoryContextDelete(MemoryContext context)

入力と出力

  • **入力**: context (破棄対象の MemoryContext へのポインタ)
  • **出力**: なし

前提条件と不変条件

  • context は MemoryContextIsValid を満たす有効なポインタである必要があります。
  • トランザクション中断時などのスタック枯渇リスクを考慮し、再帰呼び出しは行われません。

アルゴリズムまたは振る舞い

1. 指定された context を起点に、木構造の末端（子を持たないコンテキスト）まで非再帰的に降下します。 2. 末端のコンテキストを MemoryContextDeleteOnly で解放します。 3. 解放したコンテキストの親へ遡り、同様の処理を繰り返します。 4. 起点となった context 自体が解放された時点で処理を終了します。 5. 処理中に木構造を動的に変更するため、汎用的なトラバーサル関数は使用しません。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c
  • contrib/amcheck/verify_nbtree.c
  • contrib/bloom/blinsert.c
  • contrib/dblink/dblink.c
  • contrib/file_fdw/file_fdw.c
  • contrib/fuzzystrmatch/daitch_mokotoff.c
  • contrib/pg_trgm/trgm_regexp.c
  • **呼び出し先**:
  • MemoryContextDeleteOnly (内部関数)

関連データ構造

  • MemoryContext: メモリコンテキストの基底構造体。firstchild（最初の子）および parent（親）ポインタを保持し、階層構造を形成します。

失敗モードと境界条件

  • 無効なコンテキストポインタが渡された場合、Assert により停止します。
  • 階層が深い場合でも、非再帰的なループ構造によりスタックオーバーフローを回避します。

保守担当者向けメモ

  • 本関数はツリー構造を破壊しながら走査するため、MemoryContextTraverseNext 等の汎用トラバーサル関数は利用できません。
  • メモリ管理の整合性を維持するため、個別のコンテキスト解放には MemoryContextDeleteOnly を使用してください。
