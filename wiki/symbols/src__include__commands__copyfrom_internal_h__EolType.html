EolType

定義

  • 種別: enum
  • 定義箇所: src/include/commands/copyfrom_internal.h:35

目的

COPY FROM コマンドにおいて、入力データソースの行終端（End-of-Line）形式を識別および保持するために使用されます。

シグネチャまたは形状

typedef enum EolType
{
    EOL_UNKNOWN,
    EOL_NL,
    EOL_CR,
    EOL_CRNL,
} EolType;

入力と出力

  • 本シンボルは列挙型であり、入力や出力を持つ関数ではありません。
  • CopyFromStateData 構造体内の eol_type フィールドとして保持されます。

前提条件と不変条件

  • COPY 処理の初期化段階で、入力ソースの特性に基づき適切な値が設定される必要があります。
  • EOL_UNKNOWN は、行終端がまだ検出されていない初期状態を示します。

アルゴリズムまたは振る舞い

  • 入力データの解析過程において、行の区切りを判定する際のフラグとして機能します。
  • EOL_NL（\n）、EOL_CR（\r）、EOL_CRNL（\r\n）のいずれかに分類することで、プラットフォームやファイル形式に依存する改行コードの差異を吸収します。

呼び出し元と呼び出し先

  • 呼び出し元: CopyFromStateData 構造体（src/include/commands/copyfrom_internal.h:69）が本型を参照しています。
  • 呼び出し先: 本シンボルは列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • CopyFromStateData: COPY FROM の実行状態を管理する構造体であり、本型をメンバとして保持します。

失敗モードと境界条件

  • 入力データが空である場合や、行終端が定義されていない形式の場合、EOL_UNKNOWN が維持される可能性があります。

保守担当者向けメモ

  • COPY の解析ロジック（copyfromparse.c 等）において、改行コードの判定ロジックを変更する際は、本列挙型の定義と整合性が取れているか確認してください。
