partition_bounds_equal

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partbounds.c:895

目的

2つの PartitionBoundInfo 構造体が論理的に等価であるかを判定します。

シグネチャまたは形状

bool partition_bounds_equal(int partnatts, int16 *parttyplen, bool *parttypbyval,
                            PartitionBoundInfo b1, PartitionBoundInfo b2)

入力と出力

  • **入力**:
  • partnatts: パーティションキーの属性数。
  • parttyplen: 各属性の型長（int16 配列）。
  • parttypbyval: 各属性が値渡し可能かを示すフラグ（bool 配列）。
  • b1, b2: 比較対象の PartitionBoundInfo 構造体。
  • **出力**:
  • 両者が等価であれば true、そうでなければ false を返します。

前提条件と不変条件

  • b1 と b2 は有効な PartitionBoundInfo 構造体である必要があります。
  • 比較には datumIsEqual を使用するため、型定義情報（parttyplen, parttypbyval）が正確である必要があります。

アルゴリズムまたは振る舞い

1. **基本属性の比較**: 戦略（strategy）、データ数（ndatums）、インデックス数（nindexes）、null_index、default_index が一致するかを確認します。 2. **インデックス配列の比較**: indexes 配列の内容が完全に一致することを確認します。 3. **データ値の比較**:

  • **HASH戦略**: indexes 配列が一致すれば等価とみなします（USE_ASSERT_CHECKING 有効時は datums の内容も検証）。
  • **その他（RANGE/LIST等）**: 各データムについて、kind（境界の種類）が一致するかを確認し、有限値の場合は datumIsEqual を用いて値の比較を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/joinrels.c:1771
  • **呼び出し先**:
  • datumIsEqual（PostgreSQL 内部ユーティリティ）

関連データ構造

  • PartitionBoundInfo: パーティション境界情報を保持する構造体。

失敗モードと境界条件

  • 戦略が異なる場合は即座に false を返します。
  • kind が定義されている場合、非有限境界（無限大など）は値の比較を行わずにスキップされます。
  • 比較演算子ではなく datumIsEqual を使用するため、型固有の比較関数を呼び出さず、メモリ上のバイナリ表現が一致するかを判定します。

保守担当者向けメモ

  • 本関数は、リレーションキャッシュの更新判定など、パーティション境界の変更を検知するために使用されます。
  • datumIsEqual を使用する理由は、トランザクション中断時でも安全に動作させるためであり、カスタムの比較演算子を使用することは避けてください。
