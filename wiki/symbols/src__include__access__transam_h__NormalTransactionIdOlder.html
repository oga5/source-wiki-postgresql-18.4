NormalTransactionIdOlder

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/transam.h:348

目的

2つの正規トランザクションID（TransactionId）を比較し、より古い（先に発生した）方を返します。

シグネチャまたは形状

static inline TransactionId NormalTransactionIdOlder(TransactionId a, TransactionId b)

入力と出力

  • **入力**:
  • a: 比較対象の TransactionId
  • b: 比較対象の TransactionId
  • **出力**:
  • a または b のうち、NormalTransactionIdPrecedes の判定に基づき「より古い」と判断された方の TransactionId。

前提条件と不変条件

  • 入力される a および b は、TransactionIdIsNormal を満たす正規のトランザクションIDでなければなりません（Assert により検証されます）。

アルゴリズムまたは振る舞い

1. a と b が正規のIDであることを検証します。 2. NormalTransactionIdPrecedes(a, b) を呼び出し、a が b より先行しているか判定します。 3. 先行していれば a を返し、そうでなければ b を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • **呼び出し先**: TransactionIdIsNormal（マクロ/関数）、NormalTransactionIdPrecedes（マクロ/関数）。

関連データ構造

  • TransactionId: 32ビットのトランザクション識別子。

失敗モードと境界条件

  • TransactionId は循環するため、比較ロジックはラップアラウンドを考慮した NormalTransactionIdPrecedes の実装に依存します。
  • 正規範囲外のID（InvalidTransactionId や予約済みIDなど）が渡された場合、Assert が有効なビルドではプログラムが停止します。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、パフォーマンスを重視する箇所での利用を想定しています。
  • 64ビットの FullTransactionId ではなく、32ビットの TransactionId を扱うためのユーティリティです。
