ecpg_hex_encode

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/data.c:190

目的

バイナリデータ（bytea）を 16 進数表現の文字列にエンコードします。

シグネチャまたは形状

unsigned ecpg_hex_encode(const char *src, unsigned len, char *dst)

入力と出力

  • **入力**:
  • src: エンコード対象のバイナリデータへのポインタ。
  • len: src が指すデータのバイト長。
  • dst: エンコード結果を格納するバッファへのポインタ（十分な領域が確保されている必要がある）。
  • **出力**:
  • 戻り値: エンコード後の文字列長（len * 2）。

前提条件と不変条件

  • dst は、少なくとも len * 2 バイト以上の書き込み可能なメモリ領域を指している必要があります。
  • src は len バイト分読み取り可能である必要があります。

アルゴリズムまたは振る舞い

1. 16 進数変換用の静的テーブル hextbl ("0123456789abcdef") を使用します。 2. src の各バイトを上位 4 ビットと下位 4 ビットに分割します。 3. それぞれを hextbl のインデックスとして参照し、対応する 16 進文字を dst に順次書き込みます。 4. 全バイトの処理が完了するまで繰り返し、最後に len * 2 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/execute.c (複数箇所)
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • なし

失敗モードと境界条件

  • len が 0 の場合、ループは実行されず 0 を返します。
  • dst バッファが len * 2 未満のサイズである場合、バッファオーバーランが発生します。

保守担当者向けメモ

  • 本関数は bytea 型の変換処理の一部として使用されます。
  • ecpg_hex_enc_len と対になる設計となっており、エンコード後のサイズ計算にはそちらを参照してください。
