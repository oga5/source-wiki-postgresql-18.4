setRuleCheckAsUser_Query

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteDefine.c:650

目的

クエリツリー内のすべての RTEPermissionInfo に対して、権限チェックに使用するユーザーID（checkAsUser）を再帰的に設定します。

シグネチャまたは形状

static void setRuleCheckAsUser_Query(Query *qry, Oid userid)

入力と出力

  • **入力**:
  • qry: 処理対象の Query 構造体へのポインタ。
  • userid: 権限チェックに使用する Oid。
  • **出力**: なし（副作用として qry 構造体内の各ノードを直接更新）。

前提条件と不変条件

  • qry は有効な Query 構造体である必要があります。
  • qry->rteperminfos リスト内の各 RTEPermissionInfo は、この関数によって checkAsUser が指定された userid に上書きされます。

アルゴリズムまたは振る舞い

1. **RTEPermissionInfo の更新**: qry->rteperminfos リストを走査し、すべての RTEPermissionInfo の checkAsUser フィールドを userid に設定します。 2. **サブクエリの再帰処理**: qry->rtable を走査し、RTE_SUBQUERY 型の RangeTblEntry を見つけた場合、その subquery に対して自身を再帰的に呼び出します。 3. **CTE の再帰処理**: qry->cteList を走査し、各 CommonTableExpr が持つクエリに対して自身を再帰的に呼び出します。 4. **サブリンクの探索**: qry->hasSubLinks が真の場合、query_tree_walker を使用して setRuleCheckAsUser_walker を呼び出し、クエリツリー内のサブリンクを探索・処理します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/rewrite/rewriteDefine.c 内の setRuleCheckAsUser_walker（643行目付近）から呼び出されます。
  • **呼び出し先**:
  • setRuleCheckAsUser_Query（自身を再帰呼び出し）
  • query_tree_walker（サブリンク探索用）
  • setRuleCheckAsUser_walker（コールバック関数として）

関連データ構造

  • Query: クエリツリーのルート構造体。
  • RTEPermissionInfo: 権限チェック情報を保持する構造体。
  • RangeTblEntry: レンジテーブルエントリ。
  • CommonTableExpr: CTE（WITH句）の定義。

失敗モードと境界条件

  • qry が NULL の場合、ポインタ参照エラーが発生する可能性があります。
  • query_tree_walker に QTW_IGNORE_RC_SUBQUERIES フラグが渡されており、リレーションキャッシュに関連するサブクエリは探索対象から除外されます。

保守担当者向けメモ

  • 本関数はルール定義時の権限伝播を担います。サブクエリやCTEが含まれる複雑なクエリツリーを扱うため、再帰構造の変更時には query_tree_walker の設定と整合性を取る必要があります。
