run_child_tests

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_cloexec/test_cloexec.c:175

目的

Windows環境において、親プロセスから継承されたファイルハンドルが O_CLOEXEC の制約に従って正しく制御されているかを検証する。

シグネチャまたは形状

static void run_child_tests(const char *handle1_str, const char *handle2_str)

入力と出力

  • **入力**:
  • handle1_str: O_CLOEXEC を付与して開かれたファイルのハンドル文字列（16進数）。
  • handle2_str: O_CLOEXEC なしで開かれたファイルのハンドル文字列（16進数）。
  • **出力**: なし（標準出力へのテスト結果報告および exit() による終了コードの返却）。

前提条件と不変条件

  • 入力される文字列は、有効なWindowsハンドルを指す16進数形式であること。
  • 親プロセスが CreateProcess を呼び出す際、bInheritHandles を TRUE に設定していること。

アルゴリズムまたは振る舞い

1. 引数の文字列を sscanf で HANDLE 型に変換する。変換失敗時はエラーを出力し exit(1) する。 2. try_write_to_handle を呼び出し、各ハンドルへの書き込み可否を確認する。 3. O_CLOEXEC が付与された handle1 は書き込み不可（false）、付与されていない handle2 は書き込み可能（true）であることを期待する。 4. 期待通りであれば exit(0)（成功）、そうでなければ exit(1)（失敗）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/test/modules/test_cloexec/test_cloexec.c:50）。
  • **呼び出し先**: try_write_to_handle。

関連データ構造

  • HANDLE: Windows API で使用されるファイルハンドル型。

失敗モードと境界条件

  • **パース失敗**: 引数がハンドルとして解釈できない場合、exit(1) で異常終了する。
  • **検証失敗**: O_CLOEXEC の有無と書き込み可否の期待値が一致しない場合、exit(1) で異常終了する。

保守担当者向けメモ

  • 本関数はWindows環境でのみ動作するテストコードの一部である。
  • try_write_to_handle の戻り値が期待値と異なる場合、OSレベルでのハンドル継承制御が正しく機能していないことを示唆する。
