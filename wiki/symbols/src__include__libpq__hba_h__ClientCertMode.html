ClientCertMode

定義

  • 種別: enum
  • 定義箇所: src/include/libpq/hba.h:68

目的

PostgreSQL のホストベース認証（HBA）において、クライアント証明書の検証モードを定義するために使用されます。

シグネチャまたは形状

typedef enum ClientCertMode
{
	clientCertOff,
	clientCertCA,
	clientCertFull,
} ClientCertMode;

入力と出力

  • 入力: なし（列挙型定数）
  • 出力: clientCertOff（検証なし）、clientCertCA（CAによる検証）、clientCertFull（完全な検証）のいずれかの状態を表現します。

前提条件と不変条件

  • pg_hba.conf の解析結果を保持するデータ構造の一部として使用されます。
  • 認証方式の決定ロジックにおいて、接続時の証明書要件を規定します。

アルゴリズムまたは振る舞い

  • clientCertOff: クライアント証明書の検証を行いません。
  • clientCertCA: クライアント証明書が信頼されたCAによって署名されているかを確認します。
  • clientCertFull: clientCertCA の条件に加え、証明書の内容に基づいた検証を行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/libpq/hba.h 内の HbaLine 構造体において、メンバ変数 clientcert の型として参照されています。
  • 呼び出し先: 提供されたソース情報からは、具体的な関数呼び出し関係は特定できません。

関連データ構造

  • HbaLine: pg_hba.conf の各行を保持する構造体であり、この型をメンバとして含みます。

失敗モードと境界条件

  • 該当する設定値が pg_hba.conf 内で正しく指定されていない場合、認証ルールの解析時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本列挙型に値を追加または変更する場合は、src/backend/libpq/hba.c 内の関連するパース処理および認証ロジックとの整合性を確認してください。
