fetch_agg_sort_op

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/planagg.c:498

目的

指定された集約関数（aggfnoid）に対応するソート演算子（aggsortop）をシステムカタログ pg_aggregate から取得します。

シグネチャまたは形状

static Oid fetch_agg_sort_op(Oid aggfnoid)

入力と出力

  • **入力**: aggfnoid (Oid) - 検索対象の集約関数の OID。
  • **出力**: Oid - 見つかった場合は aggsortop、見つからない場合は InvalidOid。

前提条件と不変条件

  • SearchSysCache1 を使用して AGGFNOID キャッシュから情報を取得します。
  • 取得した HeapTuple は、関数終了前に ReleaseSysCache で解放される必要があります。

アルゴリズムまたは振る舞い

1. AGGFNOID システムキャッシュを aggfnoid で検索します。 2. タプルが無効な場合は InvalidOid を返します。 3. 有効なタプルから Form_pg_aggregate 構造体へキャストし、aggsortop フィールドを抽出します。 4. キャッシュを解放し、抽出した OID を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planagg.c 内の can_minmax_aggs 関数（282行目）から呼び出されます。
  • **呼び出し先**: SearchSysCache1、GETSTRUCT、ReleaseSysCache（いずれも PostgreSQL のシステムキャッシュ API）。

関連データ構造

  • Form_pg_aggregate: pg_aggregate システムカタログの行形式を保持する構造体。

失敗モードと境界条件

  • 指定された aggfnoid が pg_aggregate に存在しない場合、HeapTupleIsValid が偽となり InvalidOid を返します。

保守担当者向けメモ

  • 本関数は MIN/MAX 最適化の過程で、インデックススキャンに必要なソート順序を特定するために使用されます。
  • システムカタログの変更により aggsortop の定義が変更された場合、本関数のロジックも影響を受ける可能性があります。
