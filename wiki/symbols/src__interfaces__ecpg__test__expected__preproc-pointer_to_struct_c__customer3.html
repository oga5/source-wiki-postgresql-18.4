customer3

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c:100

目的

ECPGプリプロセッサの型解析能力を検証するためのテスト用構造体定義です。構造体へのポインタを介したデータフェッチの正当性を確認するために使用されます。

シグネチャまたは形状

struct customer3 {
    char name[50];
    int phone;
};

入力と出力

  • **入力**: malloc によりヒープ上に確保された customer3 型の配列（10要素分）。
  • **出力**: SQLクエリの結果が、この構造体のメンバ（name, phone）にマッピングされます。

前提条件と不変条件

  • malloc によるメモリ確保が成功していること。
  • 構造体のメンバ配置が、ECPGが生成するSQLデータマッピングの期待値と一致していること。

アルゴリズムまたは振る舞い

1. malloc を用いて customer3 型の配列を動的に確保し、ポインタ custs3 で参照します。 2. ECPGの ECPGdo 関数を通じて、データベースから取得したデータを custs3 が指すメモリ領域へフェッチします。 3. フェッチ時には sizeof(struct customer3) を用いてメモリレイアウトの整合性が維持されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c および src/interfaces/ecpg/test/expected/preproc-array_of_struct.c 内の ECPGdo 呼び出し。
  • **呼び出し先**: malloc（メモリ確保）、ECPGdo（SQL実行およびデータ格納）。

関連データ構造

  • custs3: customer3 構造体へのポインタ変数。
  • customer, customer2, customer4: 同様の目的で定義された他のテスト用構造体。

失敗モードと境界条件

  • malloc が失敗した場合、NULLポインタ参照によるクラッシュが発生する可能性があります。
  • データベースから取得したデータが name メンバのサイズ（50バイト）を超える場合、バッファオーバーフローのリスクがあります。

保守担当者向けメモ

  • 本構造体はプリプロセッサのテスト用生成コードの一部です。直接編集せず、テストロジックの変更が必要な場合は .pgc ソースファイルを確認してください。
  • ECPGdo 呼び出し時の long 型引数（サイズやカウント）が、構造体の定義と一致しているか注意してください。
