ReplicationStateOnDisk

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/origin.c:147

目的

論理レプリケーションの進捗状態をチェックポイント時にディスクへ永続化するためのデータ構造です。クラッシュリカバリ後にレプリケーションを正確な位置から再開するために使用されます。

シグネチャまたは形状

typedef struct ReplicationStateOnDisk
{
	RepOriginId roident;
	XLogRecPtr	remote_lsn;
} ReplicationStateOnDisk;

入力と出力

  • 本構造体は、共有メモリ上の状態をディスクへ書き出す際、および起動時にディスクから読み込む際のデータ形式として機能します。

前提条件と不変条件

  • roident はレプリケーション起点の識別子です。
  • remote_lsn はリモートノードから受信した最新のLSN（Log Sequence Number）を保持します。

アルゴリズムまたは振る舞い

  • チェックポイント処理において、共有メモリ上の各 ReplicationState から情報をコピーし、本構造体の形式でファイルへ書き出されます。
  • 起動時には、ファイルから本構造体を読み込み、レプリケーション起点の進捗状態を復元します。

呼び出し元と呼び出し先

  • 呼び出し元: CheckPointReplicationOrigin（書き出し時）、StartupReplicationOrigin（読み込み時）にてローカル変数 disk_state として利用されています。
  • 呼び出し先: 直接的な関数呼び出し関係はソースコード上では定義されていません。

関連データ構造

  • ReplicationState: 共有メモリ上で進捗を管理する構造体。
  • RepOriginId: 起点を識別するためのID型。
  • XLogRecPtr: WAL上の位置を示す型。

失敗モードと境界条件

  • 永続化ファイルが破損している場合、リカバリ時に読み込みエラーが発生する可能性があります。
  • 内部ID（RepOriginId）は2バイトに制限されているため、最大ノード数は65kに制限されます。

保守担当者向けメモ

  • 本構造体はディスクフォーマットの一部であるため、フィールドの追加や変更を行う際は、チェックポイントファイルの互換性に注意してください。
  • ファイルのレイアウトには、マジックナンバーやCRC32Cが含まれる設計となっています。
