transformRangeSubselect

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_clause.c:404

目的

SQLの FROM 句に現れるサブクエリ（RangeSubselect）を解析し、ParseNamespaceItem へ変換する。

シグネチャまたは形状

ParseNamespaceItem *transformRangeSubselect(ParseState *pstate, RangeSubselect *r)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態を保持する ParseState 構造体。
  • r: 変換対象の RangeSubselect ノード。
  • **出力**:
  • サブクエリに対応する ParseNamespaceItem 構造体へのポインタ。

前提条件と不変条件

  • pstate->p_expr_kind は EXPR_KIND_NONE であること。
  • pstate->p_lateral_active は false であること。
  • 変換対象のサブクエリは SELECT コマンドであること。

アルゴリズムまたは振る舞い

1. pstate->p_expr_kind を EXPR_KIND_FROM_SUBSELECT に設定する。 2. r->lateral が真の場合、pstate->p_lateral_active を有効化し、LATERAL 参照を可能にする。 3. parse_sub_analyze を呼び出し、サブクエリを再帰的に解析する。 4. 解析終了後、p_lateral_active を false に、p_expr_kind を EXPR_KIND_NONE に復元する。 5. 解析結果が SELECT コマンドであることを検証する。 6. addRangeTableEntryForSubquery を呼び出し、サブクエリ用の RangeTblEntry (RTE) と ParseNamespaceItem を生成して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_clause.c 内の transformFromClauseItem から呼び出される。
  • **呼び出し先**:
  • parse_sub_analyze (サブクエリの解析)
  • isLockedRefname (ロック対象の判定)
  • addRangeTableEntryForSubquery (RTEの作成)
  • elog (エラー発生時)

関連データ構造

  • ParseState: 解析状態を管理。
  • RangeSubselect: FROM 句のサブクエリノード。
  • ParseNamespaceItem: 名前空間のアイテム。
  • Query: 解析後のクエリ構造体。

失敗モードと境界条件

  • サブクエリが SELECT 以外のコマンド（INSERT, UPDATE 等）である場合、ERROR を発生させる。

保守担当者向けメモ

  • 本関数は LATERAL 参照のスコープ制御を行うため、p_lateral_active の操作順序を変更しないこと。
  • parse_sub_analyze への引数として渡されるロック判定ロジックは、エイリアスの有無に依存している点に注意。
