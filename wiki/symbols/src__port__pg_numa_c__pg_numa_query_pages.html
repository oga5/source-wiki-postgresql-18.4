pg_numa_query_pages

定義

  • 種別: function
  • 定義箇所: src/port/pg_numa.c:75

目的

指定されたメモリページ配列が配置されている NUMA ノード位置を問い合わせ、結果をステータス配列に格納します。

シグネチャまたは形状

int pg_numa_query_pages(int pid, unsigned long count, void **pages, int *status);

入力と出力

  • pid: 問い合わせ対象のプロセスID。
  • count: pages 配列に含まれるページ数。
  • pages: 問い合わせ対象のメモリページアドレスの配列。
  • status: 各ページに対応する NUMA ノード番号が格納される出力用配列。
  • 戻り値: 成功時は 0。エラー時は負の値を返します。

前提条件と不変条件

  • count 個の要素を持つ pages および status 配列が有効であること。
  • NUMA_QUERY_CHUNK_SIZE を超えるページ数を処理する場合、内部でループ処理が行われます。
  • バックエンドプロセス実行時は、チャンク処理の合間に CHECK_FOR_INTERRUPTS() が実行されます。

アルゴリズムまたは振る舞い

1. count が NUMA_QUERY_CHUNK_SIZE を超える場合、チャンク単位に分割して処理します（カーネルバグ do_pages_stat() 回避のため）。 2. 各チャンクに対して numa_move_pages を呼び出します。 3. チャンク処理中にエラー（戻り値が負）が発生した場合は即座に終了します。 4. 全てのチャンク処理完了後、next == count であることをアサートし、成功を返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/pg_buffercache/pg_buffercache_pages.c
  • src/backend/storage/ipc/shmem.c
  • 呼び出し先:
  • numa_move_pages (Linux libnuma ライブラリ)
  • CHECK_FOR_INTERRUPTS (バックエンド実行時)

関連データ構造

  • NUMA_QUERY_CHUNK_SIZE: 処理単位を規定する定数（32bit環境では16、それ以外は1024）。

失敗モードと境界条件

  • numa_move_pages が負の値を返した場合、そのエラーコードをそのまま返します。
  • count が 0 の場合、ループは実行されず 0 を返します。

保守担当者向けメモ

  • 本関数は Linux の libnuma に強く依存しています。
  • do_pages_stat() のバグ回避策としてチャンク分割を行っているため、この制限を変更する際はカーネル側の仕様を再確認してください。
  • FRONTEND 定義時は CHECK_FOR_INTERRUPTS() が無効化されます。
