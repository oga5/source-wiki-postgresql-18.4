index

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/preproc/type.h:95

目的

ECPGプリプロセッサにおいて、特定の識別子や文字列情報を保持するためのデータ構造です。

シグネチャまたは形状

struct index
{
	const char *index1;
	const char *index2;
	const char *str;
};

入力と出力

  • 入力: 構造体の各フィールド（index1, index2, str）に文字列ポインタを格納します。
  • 出力: 構造体インスタンスを通じて、保持された文字列情報が参照されます。

前提条件と不変条件

  • 各フィールドは const char * 型であり、指し示す文字列は変更不可として扱われます。
  • 構造体内のポインタが指すメモリ領域は、本構造体の生存期間中、有効である必要があります。

アルゴリズムまたは振る舞い

本構造体はデータの保持のみを目的としており、特定のアルゴリズムは実装されていません。プリプロセッサの解析過程で、関連する識別子や文字列をグループ化して管理するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは、本構造体を具体的にインスタンス化または参照している箇所は特定できません。
  • 呼び出し先: 本構造体はデータ保持用であり、関数を呼び出すことはありません。
  • 注意: ツールが検出した contrib/amcheck/verify_common.c 内の index への言及は、PostgreSQLのデータベースインデックスに関するものであり、本ヘッダーファイルで定義される struct index とは無関係です。

関連データ構造

  • src/interfaces/ecpg/preproc/type.h 内で定義される他のプリプロセッサ用構造体（ECPGtype, su_symbol 等）と並列して管理されます。

失敗モードと境界条件

  • 構造体内の各ポインタが NULL である場合、アクセス時にセグメンテーション違反が発生する可能性があります。
  • 保持される文字列のメモリ管理は本構造体の外部で行われるため、ダングリングポインタの発生に注意が必要です。

保守担当者向けメモ

  • 本構造体はデータベースのインデックス（catalog/index.h 等）とは全く異なる、ECPGプリプロセッサ内部の構文解析用データ構造です。混同しないよう注意してください。
  • 構造体の拡張が必要な場合は、type.c 内の関連する初期化・解放ロジックも併せて確認してください。
