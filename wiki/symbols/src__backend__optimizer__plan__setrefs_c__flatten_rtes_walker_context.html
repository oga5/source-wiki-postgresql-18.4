flatten_rtes_walker_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/setrefs.c:94

目的

レンジテーブルエントリ（RTE）を走査する際、グローバルなプランナ状態と対象のクエリ構造を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct
{
	PlannerGlobal *glob;
	Query	   *query;
} flatten_rtes_walker_context;

入力と出力

  • glob: プラン全体のグローバル情報（PlannerGlobal）へのポインタ。
  • query: 走査対象となるクエリツリー（Query）へのポインタ。

前提条件と不変条件

  • glob および query は、走査処理中に有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

flatten_rtes_walker 関数において、プランツリー内の RTE を再帰的に探索する際の引数として使用されます。この構造体を通じて、走査中のノードがどのクエリコンテキストに属しているか、およびグローバルなプランナ状態を共有します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/setrefs.c 内の flatten_unplanned_rtes 等の関数でインスタンス化されます。
  • **呼び出し先**: flatten_rtes_walker 関数に引数として渡され、走査処理の制御に使用されます。

関連データ構造

  • PlannerGlobal: プラン生成全体で共有されるグローバル状態。
  • Query: SQL クエリの解析結果を保持する構造体。
  • flatten_rtes_walker: 本構造体を引数として受け取り、RTE を走査する関数。

失敗モードと境界条件

  • 構造体内のポインタが NULL の場合、flatten_rtes_walker 内での参照時にクラッシュする可能性があります。

保守担当者向けメモ

  • 本構造体は setrefs.c 内のレンジテーブルフラット化処理専用です。他のコンテキスト構造体（fix_scan_expr_context 等）と混同しないよう注意してください。
