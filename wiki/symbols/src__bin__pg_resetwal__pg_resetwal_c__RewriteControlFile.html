RewriteControlFile

定義

  • 種別: function
  • 定義箇所: src/bin/pg_resetwal/pg_resetwal.c:893

目的

データベースの制御情報（pg_control）を、WALの初期化および強制的な「正常終了」状態へのリセットに合わせて更新し、ディスクへ書き戻す。

シグネチャまたは形状

static void RewriteControlFile(void)

入力と出力

  • **入力**: グローバル変数 ControlFile（修正済み）、newXlogSegNo、WalSegSz。
  • **出力**: なし（副作用としてディスク上の pg_control を更新）。

前提条件と不変条件

  • ControlFile 構造体には、リセットに必要なパラメータが既に設定されている必要がある。
  • update_controlfile 関数が呼び出され、ディスクへのフラッシュが保証される。

アルゴリズムまたは振る舞い

1. newXlogSegNo に基づき、ControlFile.checkPointCopy.redo を計算・設定する。 2. チェックポイント時刻を現在時刻に更新する。 3. データベースの状態を DB_SHUTDOWNED に設定し、リカバリポイントやバックアップ関連のフラグをリセットする。 4. wal_level を WAL_LEVEL_MINIMAL に設定し、各種 max_* パラメータをデフォルト値で上書きする（これらは起動時に postmaster によって再設定される）。 5. update_controlfile(".", &ControlFile, true) を呼び出し、制御ファイルをディスクへ書き出す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_resetwal/pg_resetwal.c 内の main 関数（521行目）。
  • **呼び出し先**: update_controlfile（common/controldata_utils.h 由来）。

関連データ構造

  • ControlFileData（ControlFile グローバル変数として使用）

失敗モードと境界条件

  • update_controlfile の内部でファイル書き込みエラーが発生した場合、ユーティリティは終了する可能性がある。

保守担当者向けメモ

  • 本関数は pg_resetwal の最終段階で実行される破壊的更新処理である。
  • 設定される max_* 値は暫定的なものであり、PostgreSQL サーバー起動時に上書きされる前提であるため、厳密な整合性よりも WAL_LEVEL_MINIMAL の維持が優先される。
