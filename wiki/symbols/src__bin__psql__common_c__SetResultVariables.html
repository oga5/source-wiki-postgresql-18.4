SetResultVariables

定義

  • 種別: function
  • 定義箇所: src/bin/psql/common.c:475

目的

クエリ実行結果に基づき、psql の内部変数（ERROR, SQLSTATE, ROW_COUNT 等）を更新する。

シグネチャまたは形状

static void SetResultVariables(PGresult *result, bool success)

入力と出力

  • **入力**:
  • result: PGresult オブジェクト（クエリ実行結果）。
  • success: クエリが成功したかどうかを示す真偽値。
  • **出力**: なし（副作用として pset.vars を更新）。

前提条件と不変条件

  • pset.vars が初期化されていること。
  • result は libpq の有効なポインタであること。

アルゴリズムまたは振る舞い

1. **成功時 (success == true)**:

  • ERROR を "false" に設定。
  • SQLSTATE を "00000" に設定。
  • ROW_COUNT を PQcmdTuples の結果（空の場合は "0"）に設定。

    • 2. **失敗時 (success == false)**:

  • ERROR を "true" に設定。
  • SQLSTATE を PQresultErrorField から取得（取得できない場合は空文字列）。
  • ROW_COUNT を "0" に設定。
  • LAST_ERROR_SQLSTATE を SQLSTATE と同様に設定。
  • LAST_ERROR_MESSAGE を PQresultErrorField から取得（取得できない場合は空文字列）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/common.c 内の複数箇所（1377, 1451, 1789, 2151 行目など）。
  • **呼び出し先**: SetVariable（変数設定用）、PQcmdTuples、PQresultErrorField（いずれも libpq 関数）。

関連データ構造

  • PGresult: libpq のクエリ結果構造体。
  • pset.vars: psql のグローバル変数管理構造体。

失敗モードと境界条件

  • PQresultErrorField が NULL を返す場合、SQLSTATE やエラーメッセージは空文字列として扱われる。
  • PQcmdTuples が空文字列を返す場合、ROW_COUNT は "0" となる。

保守担当者向けメモ

  • 本関数は psql の変数システムと libpq の結果を橋渡しする役割を持つ。
  • エラー発生時の LAST_ERROR_* 変数は、ユーザーが直前のエラー詳細を確認するために使用される。
