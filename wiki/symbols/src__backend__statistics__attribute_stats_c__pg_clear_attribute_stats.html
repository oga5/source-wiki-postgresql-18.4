pg_clear_attribute_stats

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/attribute_stats.c:940

目的

指定されたリレーションの特定の属性（カラム）に関連付けられた統計情報を pg_statistic システムカタログから削除します。

シグネチャまたは形状

Datum pg_clear_attribute_stats(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の引数を受け取ります。
  • スキーマ名 (C_ATTRELSCHEMA_ARG)
  • リレーション名 (C_ATTRELNAME_ARG)
  • 属性名 (C_ATTNAME_ARG)
  • 継承フラグ (C_INHERITED_ARG)
  • **出力**: なし（PG_RETURN_VOID() を返します）。

前提条件と不変条件

  • データベースのリカバリ中ではないこと。
  • 対象のリレーションに対して ShareUpdateExclusiveLock を取得できること。
  • 指定された属性がシステム列（attnum < 0）ではないこと。
  • 指定された属性がリレーション内に存在すること。

アルゴリズムまたは振る舞い

1. stats_check_required_arg を使用し、必要な引数がすべて提供されているか検証します。 2. RecoveryInProgress() を確認し、リカバリ中であればエラーを発生させます。 3. RangeVarGetRelidExtended を呼び出し、対象リレーションの OID を取得し、ロックを保持します。 4. get_attnum を使用して属性名から属性番号を特定します。 5. 属性番号が負（システム列）または無効な場合、エラーを発生させます。 6. delete_pg_statistic を呼び出し、該当する統計情報を削除します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は特定されていません（SQL インターフェース経由での呼び出しが想定されます）。
  • **呼び出し先**:
  • stats_check_required_arg
  • TextDatumGetCString
  • RecoveryInProgress
  • RangeVarGetRelidExtended
  • get_attnum
  • delete_pg_statistic

関連データ構造

  • cleararginfo: 引数の検証に使用される定義テーブル。

失敗モードと境界条件

  • **リカバリ中**: ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE を発生させます。
  • **システム列**: attnum < 0 の場合、ERRCODE_FEATURE_NOT_SUPPORTED を発生させます。
  • **属性不在**: 属性が存在しない場合、ERRCODE_UNDEFINED_COLUMN を発生させます。

保守担当者向けメモ

  • 本関数は単一属性の統計削除に特化しています。複数列の統計削除については extended_stats.c の関連関数を参照してください。
  • delete_pg_statistic が実際の削除ロジックをカプセル化しているため、削除条件の変更が必要な場合はそちらを確認してください。
