resetStringInfo

定義

  • 種別: function
  • 定義箇所: src/common/stringinfo.c:125

目的

既存の StringInfo バッファの内容を破棄し、空の状態にリセットします。

シグネチャまたは形状

void resetStringInfo(StringInfo str)

入力と出力

  • **入力**: str (StringInfo) - リセット対象のバッファ構造体へのポインタ。
  • **出力**: なし（副作用として str が指す構造体のメンバが更新されます）。

前提条件と不変条件

  • str は有効な StringInfoData 構造体を指している必要があります。
  • str->maxlen != 0 であること（読み取り専用バッファに対するリセットは Assert により禁止されています）。

アルゴリズムまたは振る舞い

1. str->maxlen が 0 でないことを検証します。 2. バッファの先頭 (str->data[0]) にヌル文字を書き込み、文字列を空にします。 3. 現在の文字列長 (str->len) を 0 に設定します。 4. 走査用カーソル (str->cursor) を 0 にリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_io.c
  • contrib/postgres_fdw/postgres_fdw.c
  • contrib/sepgsql/database.c および relation.c
  • contrib/tablefunc/tablefunc.c
  • src/backend/commands/copyfromparse.c
  • **呼び出し先**:
  • 本関数内での関数呼び出しはありません（Assert マクロを除く）。

関連データ構造

  • StringInfoData: data（バッファ）、len（長さ）、maxlen（容量）、cursor（カーソル位置）を保持する構造体。

失敗モードと境界条件

  • maxlen == 0 のバッファに対して呼び出した場合、アサーション失敗によりプログラムが停止します。

保守担当者向けメモ

  • 本関数はメモリの再確保を行わず、既存のバッファ領域を再利用します。
  • 読み取り専用バッファ（maxlen == 0）を誤ってリセットしようとしないよう、呼び出し側でバッファの初期化状態を確認してください。
