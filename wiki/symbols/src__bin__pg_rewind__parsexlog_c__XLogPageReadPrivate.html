XLogPageReadPrivate

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_rewind/parsexlog.c:47

目的

WAL（Write-Ahead Log）読み込み処理において、XLogReader が必要とするコンテキスト情報（アーカイブからの復元コマンドやタイムライン情報）を保持するためのプライベートデータ構造です。

シグネチャまたは形状

typedef struct XLogPageReadPrivate
{
	const char *restoreCommand;
	int			tliIndex;
} XLogPageReadPrivate;

入力と出力

  • restoreCommand: WAL セグメントをアーカイブから復元するためのコマンド文字列。
  • tliIndex: 現在のタイムラインインデックス。

前提条件と不変条件

  • XLogReader の private_data フィールドにキャストして格納されることを前提としています。
  • pg_rewind においては、WAL 解析時に SimpleXLogPageRead コールバック関数内でこの構造体へアクセスし、読み込み状態を制御します。

アルゴリズムまたは振る舞い

  • XLogReader が WAL ページを読み込む際、この構造体を参照して適切な WAL セグメントの取得元（ローカルまたはアーカイブ）を判断します。
  • tliIndex を用いて、複数のタイムラインにまたがる WAL 読み込みを管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/parsexlog.c 内の各関数（extractPageMap, findLastCheckpoint, readOneRecord 等）でインスタンス化され、XLogReader に渡されます。
  • **呼び出し先**: XLogReader のコールバック関数である SimpleXLogPageRead が、xlogreader->private_data を通じて本構造体を参照します。
  • 補足: src/backend/access/transam/xlogrecovery.c でも同名の構造体が定義・使用されていますが、本ページは pg_rewind のコンテキストに基づいています。

関連データ構造

  • XLogReader（access/xlogreader.h）: 本構造体を private_data として保持する WAL リーダー構造体。

失敗モードと境界条件

  • restoreCommand が設定されていない場合、アーカイブからの WAL 取得は行われません。
  • tliIndex が不正な場合、WAL 読み込み時にタイムライン不整合エラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は pg_rewind と PostgreSQL 本体（xlogrecovery.c）の両方で定義されていますが、それぞれ用途が異なります。変更を加える際は、pg_rewind 側の parsexlog.c での利用状況に注意してください。
