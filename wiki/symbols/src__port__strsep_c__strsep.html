strsep

定義

  • 種別: function
  • 定義箇所: src/port/strsep.c:48

目的

文字列を特定の区切り文字で分割（トークン化）します。標準ライブラリで strsep が提供されていないプラットフォームにおいて、PostgreSQL の移植性レイヤーとして機能します。

シグネチャまたは形状

char *strsep(char **stringp, const char *delim);

入力と出力

  • **入力**:
  • stringp: 分割対象文字列へのポインタのポインタ。
  • delim: 区切り文字として扱う文字の集合（文字列）。
  • **出力**:
  • トークンが見つかった場合、そのトークンの先頭ポインタを返します。
  • 文字列が空、または終端に達した場合は NULL を返します。

前提条件と不変条件

  • *stringp が NULL の場合、関数は即座に NULL を返します。
  • この関数は引数 *stringp が指す文字列を直接書き換える（破壊的変更を行う）ため、元の文字列を保持する必要がある場合は事前にコピーが必要です。

アルゴリズムまたは振る舞い

1. *stringp が NULL であれば終了します。 2. 文字列を 1 文字ずつ走査し、現在の文字が delim に含まれるかを確認します。 3. 区切り文字が見つかった場合、その位置を \0（ヌル文字）で上書きし、*stringp を次のトークンの開始位置へ更新してトークンを返します。 4. 文字列の終端（\0）に達した場合は、*stringp を NULL に設定して終了します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • 提供されたソースコード内では、特定の関数呼び出しは行われていません。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • *stringp が NULL の場合、安全に NULL を返します。
  • 文字列の末尾が区切り文字である場合や、連続する区切り文字がある場合、空のトークンを正しく生成します。

保守担当者向けメモ

  • 本関数は BSD 由来の標準的な挙動を模倣しています。
  • システム標準の strsep が存在しない環境でのみ使用されることを想定しています。
  • 破壊的変更を行うため、呼び出し側でのメモリ管理に注意してください。
