Size

定義

  • 種別: typedef
  • 定義箇所: src/include/c.h:584

目的

メモリサイズやデータ長を表現するための、PostgreSQL における標準的な型定義です。

シグネチャまたは形状

typedef size_t Size;

入力と出力

  • **入力**: size_t 型の値を保持します。
  • **出力**: メモリサイズやオフセット、配列の長さなどを表す整数値として利用されます。

前提条件と不変条件

  • C 標準ライブラリの size_t に依存します。
  • 符号なし整数型であり、オブジェクトのサイズを表現するのに十分な範囲を持ちます。

アルゴリズムまたは振る舞い

  • size_t のエイリアスとして機能し、PostgreSQL のコードベース全体でメモリサイズやデータ長を扱う際の型として統一的に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: 多数のモジュールで使用されています。例として以下が挙げられます。
  • contrib/amcheck/verify_nbtree.c (ブルームフィルタのサイズ計算等)
  • contrib/cube/cube.c (スキャンバッファ長)
  • contrib/pg_buffercache/pg_buffercache_pages.c (OS ページサイズ)
  • contrib/pg_stat_statements/pg_stat_statements.c (クエリテキストのオフセットや長さ)
  • **呼び出し先**: size_t を定義する標準ライブラリ（stddef.h 等）に依存しています。

関連データ構造

  • varlena 構造体（可変長データ）のサイズ計算や、メモリ割り当てに関連する関数群で頻繁に使用されます。

失敗モードと境界条件

  • size_t の最大値を超えるサイズを扱おうとした場合、オーバーフローが発生する可能性があります。
  • プラットフォームのアーキテクチャ（32bit/64bit）に依存して値の範囲が決定されます。

保守担当者向けメモ

  • 本シンボルは c.h で定義されており、PostgreSQL の全ソースファイルで利用可能です。
  • 型の互換性を維持するため、直接的な変更は避けてください。
  • size_t との完全な互換性が求められるため、キャストや演算を行う際は標準 C 言語のルールに従ってください。
