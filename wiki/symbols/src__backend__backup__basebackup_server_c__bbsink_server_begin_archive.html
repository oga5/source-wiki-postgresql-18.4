bbsink_server_begin_archive

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_server.c:133

目的

ベースバックアップのアーカイブ処理を開始し、指定されたパスに新しい出力ファイルを作成して開く。

シグネチャまたは形状

static void bbsink_server_begin_archive(bbsink *sink, const char *archive_name)

入力と出力

  • sink: bbsink_server 構造体へのポインタ。
  • archive_name: 作成するファイル名。
  • 出力: なし（副作用として mysink->file にファイル記述子が格納される）。

前提条件と不変条件

  • mysink->file が 0 であること（ファイルが未オープンであること）。
  • mysink->filepos が 0 であること。

アルゴリズムまたは振る舞い

1. mysink->pathname と archive_name を結合し、フルパスを生成する。 2. PathNameOpenFile を使用し、O_CREAT | O_EXCL | O_WRONLY | PG_BINARY モードでファイルを作成する。 3. ファイル作成に失敗した場合は ereport(ERROR, ...) を発行する。 4. 生成したパス文字列を解放する。 5. bbsink_forward_begin_archive を呼び出し、後続のシンクへ処理を転送する。

呼び出し元と呼び出し先

  • 呼び出し元: bbsink_server_ops 構造体（begin_archive 関数ポインタ経由）。
  • 呼び出し先: PathNameOpenFile（storage/fd.h）、bbsink_forward_begin_archive。

関連データ構造

  • bbsink_server: ローカルファイル保存用の状態を保持する構造体。
  • bbsink: basebackup_sink.h で定義されたインターフェース。

失敗モードと境界条件

  • 指定されたパスに同名のファイルが既に存在する場合、O_EXCL フラグによりファイル作成に失敗し、エラーが報告される。
  • ファイルシステム権限やディスク容量不足により PathNameOpenFile が失敗した場合、ERROR が発生する。

保守担当者向けメモ

  • 本関数は bbsink インターフェースの一部であり、バックアップストリームの開始時に呼び出される。
  • durable_rename 等の整合性保証は、後の end_archive やマニフェスト処理で行われるため、本関数は単純なファイル作成に特化している。
