pg_atomic_compare_exchange_u64_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/arch-ppc.h:168

目的

PowerPC アーキテクチャにおいて、64bit 値の比較交換（CAS）操作をアトミックに実行します。

シグネチャまたは形状

static inline bool pg_atomic_compare_exchange_u64_impl(volatile pg_atomic_uint64 *ptr, uint64 *expected, uint64 newval)

入力と出力

  • **入力**:
  • ptr: 操作対象のアトミック変数へのポインタ。
  • expected: 期待値が格納されたアドレス。成功時は更新されませんが、失敗時は現在のメモリ値で上書きされます。
  • newval: 比較が成功した場合に書き込まれる新しい値。
  • **出力**:
  • bool: 比較が成功し、値が更新された場合に true を返します。

前提条件と不変条件

  • expected ポインタは 8 バイト境界に整列されている必要があります（AssertPointerAlignment による検証）。
  • PowerPC の 64bit モード環境下での実行を前提とします。

アルゴリズムまたは振る舞い

1. **ロードリンク**: ldarx 命令を使用してメモリから値を読み込み、予約をセットします。 2. **比較**: 読み込んだ値と expected を比較します。 3. **条件分岐**:

  • 値が一致しない場合、lwsync を経て終了します。
  • 値が一致する場合、stdcx. 命令で newval の書き込みを試みます。

    • 4. **再試行**: stdcx. が失敗した場合（予約が解除された場合）、ステップ 1 に戻りループします。 5. **完了**: mfcr 命令で条件レジスタを取得し、成功・失敗を判定します。成功しなかった場合は、読み取った現在の値を expected に書き戻します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/atomics/generic.h 等の汎用アトミックインターフェースから呼び出されます。
  • **呼び出し先**: 直接的な関数呼び出しはありませんが、GCC インラインアセンブラを通じて PowerPC の CPU 命令（ldarx, stdcx., sync, lwsync 等）を実行します。

関連データ構造

  • pg_atomic_uint64: 操作対象となる 64bit アトミック型。

失敗モードと境界条件

  • **競合**: 他のプロセッサによるメモリ更新で予約が解除された場合、stdcx. が失敗し、ループによる再試行が発生します。
  • **定数最適化**: HAVE_I_CONSTRAINT__BUILTIN_CONSTANT_P が有効な場合、expected が定数であれば cmpdi 命令を用いた最適化が行われます。

保守担当者向けメモ

  • 本関数は ldarx/stdcx. のペアを使用しており、PowerPC のメモリモデルにおける標準的なアトミックパターンです。
  • インラインアセンブラ内の "memory" および "cc" 制約は、コンパイラの最適化によるメモリ順序の入れ替えを防ぐために不可欠です。修正時はこれらが維持されていることを確認してください。
