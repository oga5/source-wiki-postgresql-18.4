QuitSignalReason

定義

  • 種別: enum
  • 定義箇所: src/include/storage/pmsignal.h:52

目的

ポストマスター（親プロセス）が子プロセスに対して SIGQUIT シグナルを送信する際の理由を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
    PMQUIT_NOT_SENT = 0,    /* ポストマスターはSIGQUITを送信していない */
    PMQUIT_FOR_CRASH,       /* 他のバックエンドがクラッシュした */
    PMQUIT_FOR_STOP,        /* 即時停止（Immediate Stop）が命令された */
} QuitSignalReason;

入力と出力

  • 入力: SetQuitSignalReason(QuitSignalReason reason) を通じて状態が設定されます。
  • 出力: GetQuitSignalReason() を通じて現在の理由を取得可能です。

前提条件と不変条件

  • PMQUIT_NOT_SENT (0) は、シグナルが送信されていない初期状態を表します。
  • 共有メモリ上で管理されるため、プロセス間での参照が可能です。

アルゴリズムまたは振る舞い

ポストマスターは子プロセスを終了させる必要がある際、この列挙型を用いて理由を記録し、その後 SIGQUIT を送信します。子プロセス側は、この理由を参照することで、自身の終了処理（クラッシュ対応か、停止命令か）を適切に判断します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/postmaster.c の HandleFatalError がこの理由を扱うことが確認されています。
  • 呼び出し先: 直接的な関数呼び出し関係はソース抜粋からは不明ですが、SetQuitSignalReason および GetQuitSignalReason を介して src/backend/storage/ipc/pmsignal.c と相互作用します。

関連データ構造

  • src/backend/storage/ipc/pmsignal.c 内の sigquit_reason 変数で、この列挙型の値が保持されます。

失敗モードと境界条件

  • 理由が設定されていない場合、デフォルト値である PMQUIT_NOT_SENT が保持されます。

保守担当者向けメモ

  • 本シンボルは pmsignal モジュールにおけるプロセス終了制御の根幹です。理由の追加や変更を行う際は、postmaster.c 側の終了処理ロジックとの整合性に注意してください。
