SequenceChangePersistence

定義

  • 種別: function
  • 定義箇所: src/backend/commands/sequence.c:540

目的

シーケンスの永続性属性（relpersistence）を変更し、物理ファイルを再作成することで状態をリセットする。

シグネチャまたは形状

void SequenceChangePersistence(Oid relid, char newrelpersistence)

入力と出力

  • relid (Oid): 対象シーケンスのOID。
  • newrelpersistence (char): 新しい永続性属性（'p', 'u', 't' など）。
  • 出力: なし。

前提条件と不変条件

  • 対象シーケンスに対して AccessExclusiveLock を取得する必要がある。
  • WALが必要な永続性設定の場合、トップレベルのトランザクションIDが割り当てられている必要がある。

アルゴリズムまたは振る舞い

1. LockRelationOid を呼び出し、対象シーケンスを排他ロックする。 2. init_sequence でシーケンスのキャッシュ状態とリレーションを初期化する。 3. RelationNeedsWAL を確認し、必要な場合は GetTopTransactionId を呼び出す。 4. read_seq_tuple で現在のシーケンスデータを読み込む。 5. RelationSetNewRelfilenumber を呼び出し、新しい永続性属性で物理ファイルを再作成する。 6. fill_seq_with_data で新しい物理ファイルにデータを書き込む。 7. バッファを解放し、リレーションをクローズする。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/tablecmds.c 内の ALTER TABLE 関連処理から呼び出される。
  • 呼び出し先: LockRelationOid, init_sequence, RelationNeedsWAL, GetTopTransactionId, read_seq_tuple, RelationSetNewRelfilenumber, fill_seq_with_data, UnlockReleaseBuffer, sequence_close。

関連データ構造

  • SeqTable: シーケンスのキャッシュ状態を保持する構造体。
  • Relation: シーケンスのリレーション記述子。
  • Buffer: シーケンスのページを保持するバッファ。
  • HeapTupleData: シーケンスのタプルデータ。

失敗モードと境界条件

  • ロック取得失敗時や、物理ファイルの再作成プロセスでエラーが発生した場合、トランザクションはアボートされる。

保守担当者向けメモ

  • 本関数は ALTER SEQUENCE や ALTER TABLE によるシーケンスの永続性変更時に使用される。
  • RelationSetNewRelfilenumber による物理ファイルの再作成は、トランザクションの原子性を保証するための重要なステップである。
