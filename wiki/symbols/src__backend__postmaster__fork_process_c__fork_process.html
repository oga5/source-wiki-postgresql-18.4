fork_process

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/fork_process.c:32

目的

Unix系環境において、PostgreSQLのバックエンドプロセスを安全に生成するためのfork()システムコールのラッパー。シグナルの制御、標準入出力バッファのフラッシュ、および子プロセス固有の環境初期化を行う。

シグネチャまたは形状

pid_t fork_process(void);

入力と出力

  • **入力**: なし（環境変数 PG_OOM_ADJUST_FILE および PG_OOM_ADJUST_VALUE を参照）。
  • **出力**:
  • 親プロセス: 生成された子プロセスのPID。
  • 子プロセス: 0。

前提条件と不変条件

  • Windows環境（WIN32）では使用されない。
  • fork()実行直前にはfflush(NULL)が実行され、標準入出力バッファがフラッシュされる。
  • 子プロセスは、自身のシグナルハンドラを確立するまで、BlockSigに含まれるシグナルがブロックされた状態で開始される。

アルゴリズムまたは振る舞い

1. **前処理**: fflush(NULL)によりバッファをフラッシュする。 2. **シグナル制御**: sigprocmaskを用いてBlockSigをブロック状態にする。 3. **プロセス分岐**: fork()を実行する。

  • **子プロセス**:
  • MyProcPidを自身のPIDで更新する。
  • 環境変数 PG_OOM_ADJUST_FILE が存在する場合、そのファイルに PG_OOM_ADJUST_VALUE（デフォルトは "0"）を書き込み、LinuxのOOM Killerスコアを調整する。
  • pg_strong_random_init() を呼び出し、乱数生成器を初期化する。
  • **親プロセス**: sigprocmaskを復元し、子プロセスのPIDを返す。

    • 4. **プロファイリング**: LINUX_PROFILEが定義されている場合、fork()前後でプロファイリングタイマー（ITIMER_PROF）の保存と復元を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c 内の関数から呼び出されることが確認されている。
  • **呼び出し先**: fflush, sigprocmask, fork, getpid, getenv, open, write, close, pg_strong_random_init 等のシステムコールおよび内部関数。

関連データ構造

  • BlockSig: fork()中にブロックされるシグナルセット。
  • MyProcPid: 子プロセス内で更新されるグローバル変数。

失敗モードと境界条件

  • fork()自体が失敗した場合、その戻り値がそのまま返される。
  • OOMスコア調整用のファイルオープンや書き込みエラーは無視される。
  • LINUX_PROFILEが有効な場合、タイマー設定の競合や失敗がプロファイリング精度に影響を与える可能性がある。

保守担当者向けメモ

  • 本関数は EXEC_BACKEND（Windows等のプロセス生成方式）には対応していない。
  • Linux環境におけるOOM Killer対策は、Postmasterが継承した設定を子プロセス側で上書きするための回避策である。
  • 子プロセス開始時のシグナルブロックは、競合防止のための設計であり、将来的な緩和の余地がある。
