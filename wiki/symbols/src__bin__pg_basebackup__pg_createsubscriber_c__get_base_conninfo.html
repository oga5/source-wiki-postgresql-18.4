get_base_conninfo

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_createsubscriber.c:300

目的

接続文字列から dbname パラメータを抽出し、それ以外のパラメータのみで構成された新しい接続文字列を生成する。

シグネチャまたは形状

static char *get_base_conninfo(const char *conninfo, char **dbname)

入力と出力

  • **入力**:
  • conninfo: 解析対象の接続文字列。
  • dbname: 抽出したデータベース名を格納するためのポインタ（呼び出し元でメモリ解放が必要）。
  • **出力**:
  • 戻り値: dbname を除外した接続文字列（呼び出し元で free() が必要）。解析失敗時は NULL。

前提条件と不変条件

  • conninfo は libpq が解析可能な形式であること。
  • dbname が NULL でない場合、抽出された値が pg_strdup 経由で割り当てられる。

アルゴリズムまたは振る舞い

1. PQconninfoParse を使用して接続文字列を解析する。 2. 解析結果のオプションリストを走査する。 3. keyword が "dbname" である場合、その値を dbname 引数にコピーする。 4. それ以外の有効なパラメータは appendConnStrItem を用いてバッファに追加する。 5. バッファの内容を複製して返し、内部で使用したメモリ（PQconninfoOption 等）を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/pg_createsubscriber.c:2316 にて pub_base_conninfo の生成に使用。
  • **呼び出し先**: PQconninfoParse, createPQExpBuffer, appendConnStrItem, pg_strdup, destroyPQExpBuffer, PQconninfoFree, PQfreemem。

関連データ構造

  • PQconninfoOption: libpq の接続オプション構造体。
  • PQExpBuffer: 文字列構築用バッファ。

失敗モードと境界条件

  • PQconninfoParse が失敗した場合、エラーログを出力し NULL を返す。
  • dbname が接続文字列に含まれない場合、dbname 引数は更新されない。

保守担当者向けメモ

  • 本関数は dbname を分離することで、接続先サーバーの特定と、データベース名を指定しない接続文字列の再構築を同時に行うために使用される。
  • 戻り値の文字列は pg_strdup で確保されているため、呼び出し側でのメモリリークに注意すること。
