FunctionNext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeFunctionscan.c:58

目的

FunctionScan ノードにおいて、テーブル関数（SRF）の実行結果から次のタプルを取得し、スキャン結果スロットへ供給する。

シグネチャまたは形状

static TupleTableSlot *FunctionNext(FunctionScanState *node)

入力と出力

  • **入力**: FunctionScanState *node （実行時状態を保持する構造体）
  • **出力**: TupleTableSlot * （次のタプルを格納したスロット、またはタプルが尽きた場合は空のスロット）

前提条件と不変条件

  • node->funcstates には初期化済みの関数状態が含まれていること。
  • node->simple フラグが真の場合、関数戻り値型とスキャン結果型が一致していること。
  • node->ordinality が有効な場合、序数カラムは常にタプルの末尾に配置されること。

アルゴリズムまたは振る舞い

1. **高速パス (node->simple が真の場合)**:

  • 初回呼び出し時: ExecMakeTableFunctionResult で関数を実行し、結果を Tuplestore に格納する。
  • 2回目以降: Tuplestore から直接タプルを読み出し、スキャン結果スロットへ返す。

    • 2. **通常パス**:

  • node->ordinal をスキャン方向に合わせて更新する。
  • 各関数について、初回であれば Tuplestore を構築し、以降はそこからタプルを取得する。
  • 各関数の結果を scanslot にコピーする。結果が尽きた関数には NULL を埋める。
  • 全ての関数が終了するまで、結果を統合し続ける。
  • node->ordinality が有効な場合、最後に序数値を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecScan を介して呼び出される（src/backend/executor/nodeFunctionscan.c:270 に ExecScanAccessMtd として登録）。
  • **呼び出し先**: ExecMakeTableFunctionResult, tuplestore_rescan, tuplestore_gettupleslot, ExecClearTuple, slot_getallattrs 等。

関連データ構造

  • FunctionScanState: ノード全体の実行状態。
  • FunctionScanPerFuncState: 各関数ごとの Tuplestore やメタデータを管理。
  • TupleTableSlot: 実行結果を保持するスロット。

失敗モードと境界条件

  • **関数終了**: 各関数が結果を出し尽くした場合、該当カラムに NULL を設定して補完する。
  • **後方スキャン**: fs->rowcount を使用して、境界を超えた読み出しを制御する。
  • **初回実行**: tstore が NULL の場合にのみ ExecMakeTableFunctionResult が実行される。

保守担当者向けメモ

  • simple フラグによる最適化パスは、関数が1つかつ ORDINALITY がない場合にのみ有効。
  • fs->rowcount は後方スキャンを正しくサポートするために、初回順方向スキャン時に記録される。
  • 序数カラムの管理は、WITH ORDINALITY 指定の有無に関わらず、後方スキャン時の位置整合性のために内部的に維持される。
