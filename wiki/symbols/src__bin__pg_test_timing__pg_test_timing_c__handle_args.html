handle_args

定義

  • 種別: function
  • 定義箇所: src/bin/pg_test_timing/pg_test_timing.c:42

目的

コマンドライン引数を解析し、pg_test_timing ツールの実行期間（test_duration）を設定する。また、ヘルプやバージョン情報の表示要求を処理する。

シグネチャまたは形状

static void handle_args(int argc, char *argv[])

入力と出力

  • **入力**:
  • argc: 引数の数。
  • argv: 引数の文字列配列。
  • **出力**: なし（副作用としてグローバル変数 test_duration を更新し、標準出力または標準エラー出力へメッセージを出力する）。

前提条件と不変条件

  • progname が適切に初期化されていること。
  • test_duration がグローバル変数として定義されていること。
  • 引数 -d に指定される値は正の整数である必要がある。

アルゴリズムまたは振る舞い

1. --help または --version が指定されている場合、対応する情報を出力して終了する。 2. getopt_long を使用して -d (duration) オプションを解析する。 3. -d の値が数値として妥当か（strtoul の結果、範囲外でないか、空でないか）を検証する。 4. 検証成功時、test_duration に値を格納する。 5. 不明なオプションや余分な引数が存在する場合、エラーメッセージを表示して終了する。 6. 最後に、設定された期間を標準出力に通知する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_test_timing/pg_test_timing.c 内の main 関数（ソースコード上の直接的な呼び出しを確認）。
  • **呼び出し先**: getopt_long, strtoul, printf, fprintf, puts, exit 等の標準ライブラリ関数。
  • **注意**: src/bin/pg_test_fsync/pg_test_fsync.c にも同名の関数が存在するが、本ページは pg_test_timing.c 内の定義を対象としている。

関連データ構造

  • struct option long_options[]: getopt_long で使用されるオプション定義構造体。

失敗モードと境界条件

  • -d に数値以外や範囲外（0以下、または UINT_MAX を超える値）が指定された場合、エラーメッセージを表示し exit(1) で終了する。
  • 引数が多すぎる場合、エラーメッセージを表示し exit(1) で終了する。

保守担当者向けメモ

  • 本関数は getopt_long を利用しているため、新しいオプションを追加する場合は long_options 配列と switch 文を拡張すること。
  • test_duration の型は unsigned int であるため、入力値のオーバーフローチェックが適切に行われていることを確認済み。
