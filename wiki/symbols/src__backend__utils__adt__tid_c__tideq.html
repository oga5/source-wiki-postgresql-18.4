tideq

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tid.c:175

目的

2つの TID (Tuple Identifier) が物理的に同一のタプル位置を指しているかどうかを判定する。

シグネチャまたは形状

Datum tideq(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る2つの ItemPointer 型引数。
  • **出力**: PG_RETURN_BOOL を介して、両者が等しい場合は true、それ以外は false を返す。

前提条件と不変条件

  • 入力される ItemPointer は有効なタプル識別子であること。
  • 比較は ItemPointerCompare 関数の結果に基づき、ブロック番号とオフセット番号の両方が一致する場合にのみ等価とみなされる。

アルゴリズムまたは振る舞い

1. PG_GETARG_ITEMPOINTER マクロを使用して、第1引数および第2引数を取得する。 2. ItemPointerCompare 関数を呼び出し、その戻り値が 0（等価）であるかを評価する。 3. 評価結果をブール値として呼び出し元へ返却する。

呼び出し元と呼び出し先

  • **呼び出し先**: ItemPointerCompare
  • **呼び出し元**:
  • src/backend/optimizer/path/tidpath.c:184 (コードコメントにて演算子としての利用が言及されている)
  • その他、SQLレベルでの TID 型の等価比較演算子として利用される。

関連データ構造

  • ItemPointer: タプルの物理位置（ブロック番号とオフセット番号）を保持する構造体。

失敗モードと境界条件

  • 引数が NULL の場合、PG_GETARG_ITEMPOINTER の仕様に従い、関数マネージャの挙動に依存する。
  • ItemPointerCompare が内部的に行う比較ロジックに依存する。

保守担当者向けメモ

  • 本関数は TID 型の比較演算子として実装されている。
  • 比較ロジックの変更が必要な場合は、ItemPointerCompare の実装を確認すること。
