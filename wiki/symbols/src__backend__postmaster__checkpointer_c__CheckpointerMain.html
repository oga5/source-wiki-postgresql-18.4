CheckpointerMain

定義

  • **種別**: function
  • **定義箇所**: src/backend/postmaster/checkpointer.c:181

目的

PostgreSQLのチェックポインタープロセスのメインループを実行し、定期的なチェックポイントの実行、要求されたチェックポイントの処理、およびファイル同期（fsync）要求の集約を管理する。

シグネチャまたは形状

void CheckpointerMain(const void *startup_data, size_t startup_data_len)

入力と出力

  • **入力**:
  • startup_data: 起動データ（現在は使用されず、startup_data_len == 0 であることがアサートされる）。
  • startup_data_len: データの長さ。
  • **出力**: なし（プロセス終了までループ内で動作）。

前提条件と不変条件

  • startup_data_len は常に 0 であること。
  • AuxiliaryProcessMainCommon() を呼び出し、補助プロセスとしての初期化を完了していること。
  • sigsetjmp を使用したエラーハンドリング環境が構築されており、エラー発生時にはリソース解放と状態リセットを経て 1 秒間の待機後に再試行されること。

アルゴリズムまたは振る舞い

1. **初期化**: プロセスタイプを B_CHECKPOINTER に設定し、シグナルハンドラを登録する。SIGTERM は無視し、SIGUSR2 によるシャットダウン指示を待機する設計。 2. **メモリ管理**: 専用のメモリコンテキストを作成し、エラーリカバリ時のメモリリークを防ぐ。 3. **エラーハンドリング**: sigsetjmp を設定し、例外発生時にはロックの解放、バッファのアンロック、統計情報の更新などを行い、安全に復帰する。 4. **メインループ**:

  • ResetLatch(MyLatch) で待機状態をリセット。
  • 共有メモリから fsync 要求を吸い上げ、チェックポイントの必要性を判定。
  • 必要に応じて CreateCheckPoint を実行し、完了後に共有メモリを更新して待機中のバックエンドに通知。
  • 次のイベントまで待機。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c において、ポストマスターがプロセスを起動する際に参照される。
  • **呼び出し先**:
  • AuxiliaryProcessMainCommon()
  • pqsignal() (シグナルハンドラ設定)
  • before_shmem_exit()
  • AllocSetContextCreate()
  • sigsetjmp() / FlushErrorState()
  • LWLockReleaseAll() 等の各種リソース解放関数
  • UpdateSharedMemoryConfig()
  • ResetLatch()

関連データ構造

  • CheckpointerShmem: チェックポインターとバックエンド間で共有されるメモリ構造体。
  • sigjmp_buf: エラーリカバリ用のスタック状態保持。
  • MyLatch: プロセスの待機・起床を制御するラッチ。

失敗モードと境界条件

  • **チェックポイント失敗**: ckpt_active が真の状態でエラーが発生した場合、共有メモリの ckpt_failed をインクリメントし、待機中のバックエンドに通知して失敗を伝播させる。
  • **エラー時の再試行**: エラー発生後は pg_usleep(1000000L) により最低 1 秒間待機し、ログの溢れを防ぐ。

保守担当者向けメモ

  • SIGTERM を無視する設計は、シャットダウン時にすべてのバックエンドが終了した後に最終チェックポイントを実行するためである。
  • PG_TRY を使用せず sigsetjmp を直接使用しているのは、例外スタックの最下層でエラーリカバリを確実に行うためである。
  • チェックポイント処理中にエラーが発生してもプロセスは終了せず、リソースを解放して再試行可能な状態に戻る。
