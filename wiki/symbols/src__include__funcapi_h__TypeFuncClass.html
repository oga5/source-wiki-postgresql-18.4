TypeFuncClass

定義

  • 種別: enum
  • 定義箇所: src/include/funcapi.h:146

目的

関数が返す結果の型を分類し、実行時における型解決やタプル構築の処理方針を決定するために使用されます。

シグネチャまたは形状

typedef enum TypeFuncClass
{
    TYPEFUNC_SCALAR,            /* スカラー結果型 */
    TYPEFUNC_COMPOSITE,         /* 確定可能な行型結果 */
    TYPEFUNC_COMPOSITE_DOMAIN,  /* 確定可能な行型上のドメイン結果 */
    TYPEFUNC_RECORD,            /* 不確定な行型結果 */
    TYPEFUNC_OTHER,             /* 擬似型などの不正な型 */
} TypeFuncClass;

入力と出力

  • 本シンボル自体は列挙型であり、関数（例: get_type_func_class）の戻り値として型分類を返します。

前提条件と不変条件

  • 戻り値の型が複合型であるか、スカラーであるか、あるいは実行時まで確定しない（RECORD）かを識別するために利用されます。

アルゴリズムまたは振る舞い

  • TYPEFUNC_SCALAR: 単一の値を返す関数。
  • TYPEFUNC_COMPOSITE: 戻り値の行構造（TupleDesc）が静的に確定可能。
  • TYPEFUNC_COMPOSITE_DOMAIN: 複合型をベースとしたドメイン型。
  • TYPEFUNC_RECORD: 実行時に列定義が提供されるまで構造が不明な型。
  • TYPEFUNC_OTHER: 処理対象外の型。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照）**:
  • src/backend/executor/execSRF.c
  • src/backend/executor/nodeFunctionscan.c
  • src/backend/optimizer/prep/prepjointree.c
  • src/backend/optimizer/util/clauses.c
  • src/backend/parser/parse_relation.c
  • **呼び出し先**:
  • 本シンボルは列挙型であるため、呼び出し先はありません。

関連データ構造

  • TupleDesc: 複合型を扱う際に本列挙型と組み合わせて使用されます。

失敗モードと境界条件

  • TYPEFUNC_OTHER が返された場合、関数呼び出しの型解決においてエラーや予期せぬ動作が発生する可能性があります。

保守担当者向けメモ

  • 新しい型カテゴリを追加する場合は、本列挙型の定義と、それを利用する get_type_func_class 等の判定ロジックの両方を更新する必要があります。
