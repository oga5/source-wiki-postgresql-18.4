plpython3_validator

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_main.c:160

目的

PostgreSQL の SQL 関数作成時（CREATE FUNCTION）に、PL/Python 関数の構文や定義が妥当であるかを検証します。

シグネチャまたは形状

Datum plpython3_validator(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して検証対象の関数の OID を受け取ります。
  • **出力**: Datum 型（実質的に void）。検証成功時は正常終了し、失敗時はエラーをスローします。

前提条件と不変条件

  • CheckFunctionValidatorAccess により、呼び出し元が関数検証の権限を持っている必要があります。
  • グローバル変数 check_function_bodies が真である場合にのみ検証が実行されます。

アルゴリズムまたは振る舞い

1. 権限チェックを行い、不適格な場合は即座に終了します。 2. check_function_bodies が偽であれば検証をスキップして終了します。 3. PLy_initialize() を呼び出し、Python インタプリタの環境を準備します。 4. pg_proc システムカタログから対象関数の情報を取得します。 5. PLy_procedure_is_trigger() を用いて、対象がトリガー関数かどうかを判定します。 6. PLy_procedure_get() を呼び出し、関数のコンパイルと検証を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数定義処理（CREATE FUNCTION 等）から呼び出されます。
  • **呼び出し先**:
  • PLy_initialize
  • SearchSysCache1 / ReleaseSysCache
  • PLy_procedure_is_trigger
  • PLy_procedure_get

関連データ構造

  • Form_pg_proc: pg_proc システムカタログの行構造体。
  • HeapTuple: システムカタログから取得したタプル。

失敗モードと境界条件

  • SearchSysCache1 によるキャッシュルックアップが失敗した場合、elog(ERROR) によりトランザクションがアボートされます。
  • PLy_procedure_get 内で Python コードの構文エラーや型定義の不整合が検出された場合、エラーがスローされます。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_INFO_V1(plpython3_validator) マクロにより PostgreSQL の関数ハンドラとして登録されています。
  • トリガー関数の検証時には、特定のテーブルに関連付けられないため、PLy_procedure_get の第2引数には InvalidOid が渡されます。
