tlist_same_exprs

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/tlist.c:226

目的

2つのターゲットリスト（TargetEntry のリスト）が、保持する式（expr）の観点で同一であるかを判定する。

シグネチャまたは形状

bool tlist_same_exprs(List *tlist1, List *tlist2)

入力と出力

  • **入力**: 比較対象となる2つの List（tlist1, tlist2）。各要素は TargetEntry 構造体であると想定される。
  • **出力**: 両者のリスト長が等しく、かつ対応する位置にあるすべての TargetEntry の式が equal() 関数で等価と判定された場合に true を返す。それ以外は false を返す。

前提条件と不変条件

  • 入力リストの各要素は TargetEntry 型である必要がある。
  • リストの順序は重要であり、インデックスが一致する要素同士が比較される。

アルゴリズムまたは振る舞い

1. list_length() を用いて両リストの長さを比較し、異なる場合は直ちに false を返す。 2. forboth() マクロを使用して両リストを同時に走査する。 3. 各ステップで、対応する TargetEntry の expr フィールドを抽出し、equal() 関数を用いて比較する。 4. 一つでも式が一致しないペアがあれば false を返す。 5. 全ての要素が一致した場合に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/createplan.c: プランノード生成時のターゲットリスト整合性チェックに使用。
  • src/backend/optimizer/plan/planner.c: プランナの最適化プロセスにおけるターゲットリストの比較に使用。
  • **呼び出し先**:
  • equal(): 式の構造的等価性を判定するために呼び出される。

関連データ構造

  • TargetEntry: ターゲットリストの各要素を構成する構造体。
  • List: PostgreSQL の汎用リスト構造。

失敗モードと境界条件

  • リスト長が異なる場合、式の内容に関わらず false を返す。
  • 空リスト同士を比較した場合は、長さが等しいため true を返す。

保守担当者向けメモ

  • 本関数は式の構造的等価性（equal()）のみをチェックする。ターゲットリストのラベルや resno 等の他の属性は無視される。
  • 比較対象の式が複雑な場合、equal() の再帰的な走査によりコストが増大する可能性がある。
