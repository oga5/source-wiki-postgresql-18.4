SS_process_ctes

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/subselect.c:879

目的

クエリ内の WITH 句（CTE）を走査し、各 CTE をインライン展開するか、あるいは InitPlan として計画化するかを決定・処理する。

シグネチャまたは形状

void SS_process_ctes(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root（現在のクエリレベルの計画情報）。root->parse->cteList を参照する。
  • **出力**: なし（副作用として root 内の cte_plan_ids、init_plans、および root->glob のサブプラン関連リストを更新する）。

前提条件と不変条件

  • root->cte_plan_ids が空（NIL）であること。
  • 現在のクエリレベルでは root->plan_params が空であること。

アルゴリズムまたは振る舞い

1. root->parse->cteList を反復処理する。 2. 参照されていない SELECT 文の CTE はスキップし、cte_plan_ids にダミー値 -1 を追加する。 3. インライン化の条件（CTEMaterializeNever 指定、単一参照、再帰なし、DML 不含、揮発性関数不含など）を満たす場合、inline_cte を呼び出し、同様にダミー値を記録して次へ進む。 4. インライン化しない場合：

  • subquery_planner を呼び出し、CTE クエリの計画を生成する。
  • create_plan で Plan ノードを構築する。
  • SubPlan ノードを生成し、CTE_SUBLINK 型として設定する。
  • assign_special_exec_param で実行時パラメータ ID を割り当て、setParam に登録する。
  • 生成した Plan、Path、PlannerInfo を root->glob の各リストに追加する。
  • root->init_plans に SubPlan を追加し、cte_plan_ids に plan_id を記録する。
  • cost_subplan を呼び出し、コスト見積もりを完了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planner.c（planner 関数内から呼び出される）。
  • **呼び出し先**: inline_cte, subquery_planner, create_plan, fetch_upper_rel, assign_special_exec_param, cost_subplan 等。

関連データ構造

  • PlannerInfo: クエリ全体の計画状態を保持。
  • CommonTableExpr: CTE の定義情報。
  • SubPlan: CTE を実行するための計画ノード。
  • Query: CTE 内のクエリ定義。

失敗モードと境界条件

  • root->plan_params が空でない場合、予期せぬ外部参照として elog(ERROR) を発生させる。
  • 再帰的 CTE や複数の参照がある場合、インライン化の可否判定ロジックにより計画化の挙動が分岐する。

保守担当者向けメモ

  • SubPlan の setParam は、CTE の出力結果を保持するパラメータ ID を指すために使用される。
  • CTE の実行計画は InitPlan として扱われるため、parParam や args リストは空となる。
  • inline_cte との分岐条件は複雑であり、副作用（揮発性関数など）の有無がインライン化の可否に直結するため、変更時は注意が必要。
