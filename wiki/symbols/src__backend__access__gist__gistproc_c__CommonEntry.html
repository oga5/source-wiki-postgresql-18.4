CommonEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gist/gistproc.c:271

目的

GiSTインデックスのノード分割処理において、分割後のグループ割り当てを決定するために、各エントリのペナルティ差分と元のインデックスを保持する構造体です。

シグネチャまたは形状

typedef struct {
    int     index;  /* 初期配列におけるエントリのインデックス */
    float8  delta;  /* 異なるグループへ挿入した場合のペナルティの差分 */
} CommonEntry;

入力と出力

  • 本構造体は、gist_box_picksplit 等の分割ロジック内で、エントリの再分配コストを評価するために動的に生成されます。
  • delta フィールドは、分割候補の評価結果に基づき計算されます。

前提条件と不変条件

  • index は、処理対象の全エントリ配列内での位置を指す有効な値である必要があります。
  • delta は、2つのグループ間でのペナルティの差を表す float8 値です。

アルゴリズムまたは振る舞い

  • GiSTの分割アルゴリズムにおいて、ノード分割時に「どちらのグループに配置すべきか」を判断する際、各エントリの delta を計算します。
  • delta の値に基づいて CommonEntry の配列をソートし、ペナルティの大きい順にエントリを分配することで、分割の品質を最適化します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistproc.c 内の分割ロジック（gist_box_picksplit 等）が、palloc を用いて本構造体の配列を確保し、qsort でソートします。
  • **呼び出し先**: qsort に渡される比較関数（common_entry_cmp 等）が本構造体の delta フィールドを参照します。
  • **注記**: src/backend/utils/adt/rangetypes_gist.c にも同名の構造体定義が存在し、同様の目的で使用されています。

関連データ構造

  • GIST_SPLITVEC: 分割結果を格納する構造体。
  • ConsiderSplitContext: 分割の品質評価に使用されるコンテキスト情報。

失敗モードと境界条件

  • メモリ不足時: palloc によるメモリ確保が失敗する可能性があります。
  • エントリ数: nentries が0の場合、配列の確保やソート処理は行われません。

保守担当者向けメモ

  • 本構造体は gistproc.c と rangetypes_gist.c で重複定義されています。修正を行う際は両方の実装に影響がないか確認してください。
  • delta の計算ロジックは、GiSTの分割アルゴリズムの性能に直結するため、変更時にはインデックスの再構築コストと検索効率のトレードオフを考慮してください。
