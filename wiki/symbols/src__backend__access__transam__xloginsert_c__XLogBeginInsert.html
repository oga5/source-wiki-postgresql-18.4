XLogBeginInsert

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xloginsert.c:148

目的

WAL（Write-Ahead Logging）レコード構築プロセスの開始を宣言し、内部的な作業領域を初期化します。

シグネチャまたは形状

void XLogBeginInsert(void)

入力と出力

  • 入力: なし
  • 出力: なし（成功時）
  • 失敗時: エラー（elog(ERROR)）を発生させ、処理を中断します。

前提条件と不変条件

  • WAL 挿入が許可されている状態であること（XLogInsertAllowed() が真であること）。
  • 本関数が呼び出される前に、前回のレコード構築が完了（リセット）していること（max_registered_block_id == 0 等の内部状態が初期値であること）。
  • 二重呼び出しの禁止: begininsert_called フラグが偽であること。

アルゴリズムまたは振る舞い

1. 内部状態（max_registered_block_id, mainrdata_last, mainrdata_len）が初期状態であることをアサーションで確認します。 2. XLogInsertAllowed() を呼び出し、リカバリ中など WAL 挿入が許可されない状況でないか検証します。 3. begininsert_called フラグを確認し、既に開始されている場合はエラーを投げます。 4. begininsert_called を true に設定し、レコード構築フェーズを開始します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/pg_visibility/pg_visibility.c
  • src/backend/access/brin/brin.c
  • src/backend/access/brin/brin_pageops.c
  • src/backend/access/brin/brin_revmap.c
  • src/backend/access/gin/ginbtree.c
  • src/backend/access/gin/gindatapage.c
  • src/backend/access/gin/ginfast.c
  • 呼び出し先: XLogInsertAllowed（マクロまたは関数）

関連データ構造

  • begininsert_called: 構築フェーズの重複を防ぐための静的フラグ。
  • max_registered_block_id, mainrdata_head, mainrdata_len: レコード構築用の作業領域状態。

失敗モードと境界条件

  • リカバリ中に呼び出された場合: elog(ERROR) により中断。
  • XLogBeginInsert を連続して呼び出した場合: elog(ERROR) により中断。
  • 内部状態が初期化されていない（前回のレコード構築が正しく終了していない）場合: Assert 失敗によりデバッグビルドで停止。

保守担当者向けメモ

  • 本関数は、XLogRegister* 関数群によるデータ登録の前に必ず呼び出す必要があります。
  • 構築終了後は XLogInsert または XLogResetInsertion を通じて begininsert_called をリセットするフローが必須です。
