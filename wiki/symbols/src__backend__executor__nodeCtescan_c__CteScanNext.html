CteScanNext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeCtescan.c:30

目的

共通テーブル式（CTE）の結果を保持する Tuplestore からタプルを読み取る、または必要に応じて CTE サブプランを実行してタプルを生成・蓄積する。

シグネチャまたは形状

static TupleTableSlot *CteScanNext(CteScanState *node)

入力と出力

  • **入力**: CteScanState *node（CTE スキャンノードの実行時状態）
  • **出力**: TupleTableSlot *（取得したタプル、または EOF の場合は空のスロット）

前提条件と不変条件

  • node->leader が有効であり、共有の Tuplestorestate が保持されていること。
  • node->readptr が現在のスキャン位置として正しく設定されていること。
  • Tuplestore は他のノードと共有される可能性があるため、タプル取得時にはコピーが必要となる。

アルゴリズムまたは振る舞い

1. **状態取得**: ノードから実行状態、スキャン方向、および共有 Tuplestore を取得し、現在の読み取りポインタを選択する。 2. **逆方向スキャンの調整**: 逆方向スキャンかつ Tuplestore の EOF に達している場合、必要に応じて tuplestore_advance を呼び出し、末尾のタプルを適切に取得する。 3. **既存データの取得**: Tuplestore からタプルを試行的に取得する。成功すればそのスロットを返す。 4. **遅延評価（CTE サブプラン実行）**: Tuplestore が EOF であり、かつ CTE の実行が完了していない場合、ExecProcNode を呼び出してサブプランを実行する。

  • サブプランからタプルが得られた場合、それを Tuplestore に保存し、自身のスロットにコピーして返す。
  • サブプランが終了（NULL を返却）した場合は、leader->eof_cte を真に設定し、終了する。

    • 5. **終了処理**: 取得できるタプルがない場合は ExecClearTuple を呼び出し、空のスロットを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeCtescan.c 内の ExecScan インターフェース経由で呼び出される（直接的な呼び出し元関数の詳細はソースコード内の静的解析結果からは特定不可）。
  • **呼び出し先**:
  • tuplestore_select_read_pointer, tuplestore_ateof, tuplestore_advance, tuplestore_gettupleslot, tuplestore_puttupleslot
  • ExecProcNode (サブプラン実行用)
  • ExecCopySlot, ExecClearTuple

関連データ構造

  • CteScanState: CTE スキャンの実行状態を保持。
  • Tuplestorestate: CTE の結果を格納する共有ストレージ。
  • TupleTableSlot: タプルを保持するメモリ領域。

失敗モードと境界条件

  • **CTE の空**: サブプランが最初から NULL を返す場合、即座に EOF となる。
  • **共有状態**: 他のノードが Tuplestore に書き込みを行う可能性があるため、tuplestore_gettupleslot 呼び出し時にはコピーが必須。
  • **サブプランの非堅牢性**: 一度 NULL を返した後に再度呼び出されると不安定になるノードがあるため、eof_cte フラグによるショートサーキットが必須。

保守担当者向けメモ

  • tuplestore_select_read_pointer は、サブプラン実行後に読み取りポインタが変更される可能性があるため、再選択を行う必要がある。
  • forward 方向のスキャンのみがサブプランの実行をトリガーする。
