IntegerSet

定義

  • 種別: struct
  • 定義箇所: src/backend/lib/integerset.c:196

目的

大量の 64 ビット整数をメモリ効率よく保持するための B-tree ベースのデータ構造のルート構造体です。メモリコンテキスト、B-tree のルート、挿入バッファ、およびイテレータの状態を管理します。

シグネチャまたは形状

struct IntegerSet {
    MemoryContext context;
    uint64      mem_used;
    uint64      num_entries;
    uint64      highest_value;
    int         num_levels;
    intset_node *root;
    intset_node *rightmost_nodes[MAX_TREE_LEVELS];
    intset_leaf_node *leftmost_leaf;
    uint64      buffered_values[MAX_BUFFERED_VALUES];
    int         num_buffered_values;
    bool        iter_active;
    const uint64 *iter_values;
    int         iter_num_values;
    int         iter_valueno;
    intset_leaf_node *iter_node;
    int         iter_itemno;
    uint64      iter_values_buf[MAX_VALUES_PER_LEAF_ITEM];
};

入力と出力

  • 入力: intset_add_member を介して昇順で 64 ビット整数を受け取ります。
  • 出力: intset_is_member による存在判定、またはイテレータを介した全要素の走査結果を提供します。

前提条件と不変条件

  • 値は必ず昇順で追加する必要があります。
  • イテレーション実行中に intset_add_member を呼び出すことはできません。
  • IntegerSet 自体を解放する関数はなく、作成時に使用した MemoryContext を破棄することでメモリを解放します。

アルゴリズムまたは振る舞い

  • **挿入**: buffered_values に値を蓄積し、溢れた際に intset_flush_buffered_values を呼び出して Simple-8b アルゴリズムで圧縮し、B-tree のリーフノードへ書き込みます。
  • **検索**: バッファを検索後、B-tree を二分探索し、リーフノード内の圧縮データを simple8b_contains で走査します。
  • **イテレーション**: iter_values を用いてバッファおよび B-tree 内の値を順次返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/gist/gistvacuum.c などで利用されています。
  • 呼び出し先: intset_update_upper、intset_flush_buffered_values、intset_new_internal_node 等の内部関数が本構造体を操作します。

関連データ構造

  • intset_node (B-tree ノード)
  • intset_leaf_node (リーフノード)
  • intset_internal_node (内部ノード)
  • leaf_item (リーフ内の最小単位)

失敗モードと境界条件

  • 順序外の挿入はサポートされていません。
  • 圧縮効率はデータのクラスタリングに依存し、疎なデータではメモリ消費が増大します。

保守担当者向けメモ

  • rightmost_nodes は新しい値を追加する際に必要となる、各レベルの最右ノードへのポインタを保持します。
  • iter_values は、走査対象が B-tree のリーフかバッファかによって動的に切り替わります。
