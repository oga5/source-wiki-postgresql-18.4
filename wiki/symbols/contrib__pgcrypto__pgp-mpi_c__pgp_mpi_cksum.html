pgp_mpi_cksum

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-mpi.c:131

目的

OpenPGP 形式の多倍長整数（MPI）データに対するチェックサムを計算します。

シグネチャまたは形状

unsigned pgp_mpi_cksum(unsigned cksum, PGP_MPI *n)

入力と出力

  • **入力**:
  • cksum: 累積される初期チェックサム値。
  • n: チェックサム対象の PGP_MPI 構造体へのポインタ。
  • **出力**:
  • 16ビット（0〜65535）の範囲に収められた計算済みのチェックサム値を返します。

前提条件と不変条件

  • n は有効な PGP_MPI 構造体を指している必要があります。
  • n->data が指すバッファは、n->bytes で指定されたサイズ以上のメモリを保持している必要があります。

アルゴリズムまたは振る舞い

1. n->bits（MPIのビット長）を上位8ビットと下位8ビットに分割し、それぞれを cksum に加算します。 2. n->data に格納されているバイト列を順次 cksum に加算します。 3. 最終的な cksum に対して 0xFFFF とのビット論理積（AND）をとることで、下位16ビットのみを抽出して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-pubkey.c 内の複数の箇所で、公開鍵や秘密鍵のコンポーネント（elg.x, rsa.d, rsa.p, rsa.q, rsa.u, dsa.x）のチェックサム計算に使用されています。
  • **呼び出し先**:
  • 本関数は算術演算のみを行い、他の関数を呼び出しません。

関連データ構造

  • PGP_MPI: MPI データを保持する構造体。bits（ビット長）、bytes（バイト長）、data（データ本体へのポインタ）を保持します。

失敗モードと境界条件

  • n が NULL の場合、プログラムはセグメンテーション違反を引き起こす可能性があります。
  • cksum の初期値として大きな値が渡された場合でも、最終的な戻り値は 0xFFFF によるマスク処理により常に16ビット幅に制限されます。

保守担当者向けメモ

  • 本関数は、PGP パケット内の MPI データの整合性検証に使用されます。
  • 演算は単純な加算の累積であるため、計算順序が重要です。複数の MPI を連続してチェックサム計算する場合は、前の計算結果を次の cksum 引数に渡すことで一貫した結果が得られます。
