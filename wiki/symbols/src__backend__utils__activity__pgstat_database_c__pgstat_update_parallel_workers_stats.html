pgstat_update_parallel_workers_stats

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_database.c:304

目的

並列クエリ実行に関連する統計情報（起動しようとしたワーカー数および実際に起動したワーカー数）を、現在のデータベースの統計エントリに加算・記録します。

シグネチャまたは形状

void pgstat_update_parallel_workers_stats(PgStat_Counter workers_to_launch, PgStat_Counter workers_launched)

入力と出力

  • **入力**:
  • workers_to_launch (PgStat_Counter): 起動を試みた並列ワーカー数。
  • workers_launched (PgStat_Counter): 実際に起動に成功した並列ワーカー数。
  • **出力**: なし（副作用としてデータベース統計エントリを更新）。

前提条件と不変条件

  • MyDatabaseId が有効な OID である必要があります。無効な場合は何も行わずに戻ります。
  • 呼び出し元は、統計情報の更新が許可されているコンテキストであることを保証する必要があります。

アルゴリズムまたは振る舞い

1. MyDatabaseId が有効かを確認し、無効であれば即座に終了します。 2. pgstat_prep_database_pending(MyDatabaseId) を呼び出し、現在のデータベースに対応するローカルの統計エントリ（PgStat_StatDBEntry）を取得または作成します。 3. 取得したエントリの parallel_workers_to_launch に workers_to_launch を加算します。 4. 取得したエントリの parallel_workers_launched に workers_launched を加算します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execMain.c:488 (直接の呼び出しを確認)
  • **呼び出し先**:
  • pgstat_prep_database_pending (データベース統計エントリの準備)

関連データ構造

  • PgStat_StatDBEntry: データベース単位の統計情報を保持する構造体。本関数はこの構造体内の並列ワーカー関連フィールドを更新します。
  • PgStat_Counter: 統計カウンタに使用される型。

失敗モードと境界条件

  • MyDatabaseId が無効な場合、統計更新はスキップされます。
  • pgstat_prep_database_pending が内部的にメモリ確保等を行う場合、その失敗が本関数の実行に影響を与える可能性があります（詳細は pgstat_prep_database_pending の仕様に依存）。

保守担当者向けメモ

  • 本関数はデータベース単位の統計を更新します。テーブルやインデックス単位の並列処理統計とは別管理である点に注意してください。
  • 統計情報のフラッシュ（共有メモリへの同期）は、本関数ではなく、統計サブシステムのライフサイクル管理（pgstat_database_flush_cb 等）によって後続のタイミングで行われます。
