generate_series_timestamp_fctx

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/timestamp.c:59

目的

generate_series 関数において、timestamp 型の連続値を生成するためのイテレータ状態を保持する構造体です。

シグネチャまたは形状

typedef struct
{
    Timestamp   current;
    Timestamp   finish;
    Interval    step;
    int         step_sign;
} generate_series_timestamp_fctx;

入力と出力

本構造体は関数呼び出しのコンテキストとして使用されます。

  • 入力: current（現在値）、finish（終了値）、step（増分）、step_sign（増分の符号）。
  • 出力: なし（構造体メンバとして状態を保持）。

前提条件と不変条件

  • current および finish は Timestamp 型（マイクロ秒単位の int64）です。
  • step_sign は、step の符号を保持し、イテレーションの方向を決定するために使用されます。

アルゴリズムまたは振る舞い

generate_series の実行中、この構造体は以下の役割を果たします。 1. 初期化時に開始値、終了値、増分を保持します。 2. 各イテレーションステップで current を更新し、finish に到達したかを判定します。 3. step_sign を用いて、正方向および負方向のどちらの増分に対しても一貫した終了判定を行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/timestamp.c 内の generate_series 関連関数（6675行目付近でポインタとして宣言、6695-6696行目で palloc を使用してメモリ確保）。
  • 呼び出し先: 情報なし。

関連データ構造

  • Timestamp: 日時を保持する int64 型。
  • Interval: time、day、month を保持する構造体。

失敗モードと境界条件

  • メモリ確保失敗: palloc が失敗した場合、PostgreSQL のエラーハンドリングによりアボートされます。
  • 算術オーバーフロー: current に step を加算する際、int64 の範囲を超える可能性があります（timestamp.c の一般的な注意点として、オーバーフローチェックが必要です）。

保守担当者向けメモ

  • 本構造体は generate_series の実行状態を維持するためのものであり、関数呼び出し間で永続化されるものではありません。
  • step_sign の計算ロジックに変更を加える際は、timestamp 型の無限大（infinity）の扱いと整合性が取れているか確認してください。
