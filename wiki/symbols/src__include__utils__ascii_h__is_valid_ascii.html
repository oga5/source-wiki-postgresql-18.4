is_valid_ascii

定義

  • 種別: function
  • 定義箇所: src/include/utils/ascii.h:24

目的

指定されたバイト列が有効な 7 ビット ASCII 文字列（0x00〜0x7F）であるかを高速に検証します。

シグネチャまたは形状

static inline bool is_valid_ascii(const unsigned char *s, int len)

入力と出力

  • **入力**:
  • s: 検証対象のバイト列へのポインタ。
  • len: バイト列の長さ。
  • **出力**:
  • true: 全てのバイトが 7 ビット ASCII 範囲内であり、かつヌルバイトを含まない場合。
  • false: 0x80 以上の値を持つバイト、またはヌルバイトが含まれる場合。

前提条件と不変条件

  • len は sizeof(Vector8)（8 または 16 バイト）の倍数である必要があります。この条件は Assert によって検証されます。

アルゴリズムまたは振る舞い

1. 入力されたバイト列を Vector8 単位で順次読み込みます。 2. SIMD が利用可能な環境では、vector8_eq を使用してヌルバイトを検出し、vector8_or で累積します。 3. SIMD が利用できない環境（USE_NO_SIMD）では、各バイトに 0x7F を加算するビット演算を用いてヌルバイトを判定します。 4. 各チャンクに対して、高位ビット（0x80 以上）の有無を累積的に判定します。 5. 全てのチャンク処理後、累積されたフラグを確認し、非 ASCII 文字またはヌルバイトが検出された場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/wchar.c:1935 での参照が確認されています。
  • **呼び出し先**: vector8_load, vector8_broadcast, vector8_eq, vector8_or, vector8_is_highbit_set などの SIMD 抽象化関数（port/simd.h 経由）。

関連データ構造

  • Vector8: SIMD 処理で使用される 8 ビット単位のベクトル型。

失敗モードと境界条件

  • len が sizeof(Vector8) の倍数でない場合、Assert によりプログラムが停止する可能性があります。
  • ヌルバイト（0x00）は ASCII 範囲外として扱われ、false を返します。

保守担当者向けメモ

  • 本関数はパフォーマンス重視のため static inline で定義されています。
  • SIMD 命令セットの有無により内部ロジックが分岐するため、移植性を考慮した実装となっています。デバッグ時は USE_NO_SIMD 定義の有無を確認してください。
