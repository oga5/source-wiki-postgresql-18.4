目的

reindexdb コマンドラインユーティリティのエントリーポイントであり、引数の解析、実行モード（全データベース対象か単一データベース対象か）の決定、および再構築処理の開始を行う。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**: コマンドライン引数 (argc, argv)。接続オプション、対象オブジェクト（DB、テーブル、インデックス、スキーマ）、並列実行数、および各種フラグ（--concurrently 等）を受け取る。
  • **出力**: 正常終了時は 0、エラー発生時は 1 または pg_fatal による異常終了コードを返す。

前提条件と不変条件

  • getopt_long を使用して引数を解析する。
  • データベース名が指定されていない場合、環境変数 PGDATABASE、PGUSER、または現在のユーザー名からデフォルト値を決定する。
  • syscatalog（システムカタログ再構築）と並列実行（jobs > 1）を同時に指定することはできない。

アルゴリズムまたは振る舞い

1. **初期化**: ログ出力、ロケール、ヘルプ/バージョンオプションの処理を行う。 2. **引数解析**: getopt_long をループで回し、接続パラメータや再構築対象、オプションフラグを収集する。 3. **バリデーション**: 引数の競合（--all とデータベース名の同時指定など）をチェックし、エラーがあれば終了する。 4. **接続設定**: ConnParams 構造体に接続情報を格納する。 5. **実行モードの分岐**:

  • --all が指定されている場合: reindex_all_databases を呼び出す。
  • それ以外の場合: reindex_one_database を呼び出す（システムカタログ対象の場合は REINDEX_SYSTEM を指定）。

呼び出し元と呼び出し先

  • **呼び出し元**: OS（コマンド実行時）。
  • **呼び出し先**:
  • reindex_all_databases
  • reindex_one_database
  • getopt_long
  • pg_logging_init, get_progname, set_pglocale_pgservice
  • simple_string_list_append
  • option_parse_int
  • setup_cancel_handler
  • pg_fatal, pg_log_error

関連データ構造

  • ConnParams: データベース接続情報を保持する構造体。
  • SimpleStringList: テーブル、インデックス、スキーマのリストを管理する構造体。
  • option (struct): getopt_long 用のコマンドラインオプション定義。

失敗モードと境界条件

  • 引数が多すぎる場合や不正なオプションが指定された場合はエラーメッセージを表示して終了する。
  • REINDEX SYSTEM と並列実行を組み合わせた場合は pg_fatal により停止する。
  • データベース接続に失敗した場合や、対象オブジェクトが見つからない場合の挙動は、呼び出し先の関数に依存する。

保守担当者向けメモ

  • 新しい REINDEX オプションを追加する際は、long_options 配列と getopt_long の switch 文の両方を更新する必要がある。
  • reindex_one_database への引数引き渡し部分は、ソースコードの末尾で切り替わるため、追加の実行モードを実装する際は注意すること。
  • ツールによる呼び出し元情報の解析結果（amcheck 等）は、本シンボルとは無関係なコードベースの参照であり、本関数とは直接の呼び出し関係を持たない。
