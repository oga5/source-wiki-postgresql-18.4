list_delete_int

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/list.c:890

目的

整数型リスト（List）から、指定された値（datum）と一致する最初の要素を削除する。

シグネチャまたは形状

List *list_delete_int(List *list, int datum)

入力と出力

  • **入力**:
  • list: 操作対象の整数型リストへのポインタ。
  • datum: 削除対象の整数値。
  • **出力**:
  • 要素が削除された後のリスト、または一致する要素が見つからなかった場合は元のリスト。

前提条件と不変条件

  • list は整数型リストである必要がある（IsIntegerList(list) アサーションにより検証）。
  • 操作の前後で check_list_invariants(list) が呼び出され、リストの整合性が検証される。

アルゴリズムまたは振る舞い

1. リスト内の各要素を foreach マクロで順次走査する。 2. lfirst_int(cell) を使用して現在の要素の値と datum を比較する。 3. 一致する要素が見つかった場合、list_delete_cell を呼び出してそのセルを削除し、結果を返す。 4. リストの末尾まで一致するものが見つからなかった場合、リストを変更せずにそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planner.c:3399
  • **呼び出し先**:
  • list_delete_cell (リストからのセル削除を実行)
  • IsIntegerList (型チェック)
  • check_list_invariants (整合性検証)
  • lfirst_int (要素値の取得)

関連データ構造

  • List: PostgreSQLの配列ベースの動的リスト構造体。
  • ListCell: リスト内の各要素を指すためのポインタ型。

失敗モードと境界条件

  • リストが空の場合、ループは実行されず、入力されたリストがそのまま返される。
  • 指定した値がリスト内に存在しない場合、リストは変更されず、入力されたリストがそのまま返される。

保守担当者向けメモ

  • 本関数はリストの先頭から線形探索を行うため、リストの長さ $N$ に対して $O(N)$ の計算量を持つ。
  • list_delete_cell を呼び出すため、内部で配列のシフトが発生し、メモリ配置が変更される可能性がある。リスト操作中に ListCell へのポインタを保持し続けることは避けること。
