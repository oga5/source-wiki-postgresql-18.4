XLogStats

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlogstats.h:28

目的

WAL（Write-Ahead Log）の解析結果を保持し、リソースマネージャ（rmgr）およびレコードタイプごとの統計情報を集計するための構造体です。

シグネチャまたは形状

typedef struct XLogStats
{
    uint64      count;
#ifdef FRONTEND
    XLogRecPtr  startptr;
    XLogRecPtr  endptr;
#endif
    XLogRecStats rmgr_stats[RM_MAX_ID + 1];
    XLogRecStats record_stats[RM_MAX_ID + 1][MAX_XLINFO_TYPES];
} XLogStats;

入力と出力

  • 入力: XLogRecStoreStats() 関数を介して、XLogReaderState からデコードされた WAL レコード情報が加算されます。
  • 出力: 集計された統計データは、pg_waldump や pg_walinspect などのツールによって読み取られ、表示や出力に利用されます。

前提条件と不変条件

  • record_stats 配列は、xl_info の上位 4 ビットをインデックスとして使用します。
  • FRONTEND マクロが定義されている場合、startptr および endptr フィールドが構造体に含まれます。

アルゴリズムまたは振る舞い

1. XLogRecStoreStats() が WAL レコードを走査し、レコードのサイズ（FPI とそれ以外）を算出します。 2. 算出された値を rmgr_stats（リソースマネージャ単位）および record_stats（レコードタイプ単位）の該当するスロットに加算します。 3. XACT レコードなどの特殊なレコードタイプに対しては、xl_info のビットマスク調整を行い、適切に分類します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • contrib/pg_walinspect/pg_walinspect.c
  • src/bin/pg_waldump/pg_waldump.c
  • 呼び出し先:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。統計情報の更新には XLogRecStoreStats() が使用されます。

関連データ構造

  • XLogRecStats: 統計情報の基本単位（レコード数、本体サイズ、FPI サイズを保持）。
  • XLogReaderState: WAL レコードの読み取り元となる構造体。

失敗モードと境界条件

  • RM_MAX_ID を超えるリソースマネージャ ID が渡された場合、配列外参照の可能性があるため、呼び出し側でのバリデーションが必要です。
  • XACT レコードの統計処理において、例外的なビット操作が伴うため、実装上の条件分岐に依存します。

保守担当者向けメモ

  • FRONTEND マクロの有無により構造体のサイズが変化するため、バイナリ互換性が求められる環境では注意が必要です。
  • record_stats は二次元配列であり、インデックスの範囲が [RM_MAX_ID + 1][MAX_XLINFO_TYPES] であることを考慮してメモリを操作してください。
