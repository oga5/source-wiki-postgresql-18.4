scram_state

定義

  • **種別**: struct
  • **定義箇所**: src/backend/libpq/auth-scram.c:133

目的

PostgreSQLのSASL SCRAM-SHA-256認証セッションにおける状態保持。クライアントとの複数回にわたるメッセージ交換の進捗管理、検証用キーの保持、および認証失敗時のモック処理状態を管理する。

シグネチャまたは形状

typedef struct {
    scram_state_enum state;
    const char *username;
    Port *port;
    bool channel_binding_in_use;
    pg_cryptohash_type hash_type;
    int key_length;
    int iterations;
    char *salt;
    uint8 ClientKey[SCRAM_MAX_KEY_LEN];
    uint8 StoredKey[SCRAM_MAX_KEY_LEN];
    uint8 ServerKey[SCRAM_MAX_KEY_LEN];
    char cbind_flag;
    char *client_first_message_bare;
    char *client_username;
    char *client_nonce;
    char *client_final_message_without_proof;
    char *client_final_nonce;
    uint8 ClientProof[SCRAM_MAX_KEY_LEN];
    char *server_first_message;
    char *server_nonce;
    bool doomed;
    char *logdetail;
} scram_state;

入力と出力

  • **入力**: scram_init により初期化され、scram_exchange を通じてクライアントからのメッセージ（client_first_message, client_final_message）が解析・格納される。
  • **出力**: 認証成功時にセッション鍵が生成され、MyProcPort に反映される。失敗時は doomed フラグと logdetail が設定される。

前提条件と不変条件

  • scram_init によってメモリが割り当てられ、初期化されていること。
  • doomed フラグが設定された場合、認証は失敗とみなされ、以降の検証はモックとして処理される。

アルゴリズムまたは振る舞い

1. **初期化**: scram_init で構造体が作成される。 2. **メッセージ交換**: read_client_first_message および read_client_final_message により、クライアントからの情報を構造体に蓄積する。 3. **検証**: verify_client_proof 等の関数が構造体内の StoredKey や ClientProof を参照し、認証の正当性を判定する。 4. **モック認証**: ユーザーが存在しない場合、doomed フラグを立てて計算コストを維持し、タイミング攻撃を防止する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-scram.c 内の認証フロー関数（scram_init, scram_exchange 等）。
  • **呼び出し先**: 構造体自体はデータ保持用であり、read_client_first_message, read_client_final_message, verify_client_proof 等の関数がこの構造体を引数として受け取り操作する。

関連データ構造

  • scram_state_enum: 認証の現在のフェーズを定義。
  • Port: 接続情報を保持する構造体。
  • pg_cryptohash_type: 使用するハッシュアルゴリズムの指定。

失敗モードと境界条件

  • **認証失敗**: ユーザーが存在しない、または証明が一致しない場合、doomed フラグが true になり、logdetail に詳細が記録される。
  • **メモリ不足**: palloc0 によるメモリ確保失敗時はシステム全体に影響する。

保守担当者向けメモ

  • doomed フラグが設定されている場合、実際の認証処理は行われず、攻撃者に対する情報漏洩を防ぐためのモック処理が行われる点に注意すること。
  • チャネルバインディングは現在 tls-server-end-point のみに対応している。
