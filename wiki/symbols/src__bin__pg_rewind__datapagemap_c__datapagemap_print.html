datapagemap_print

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/datapagemap.c:116

目的

指定された datapagemap_t 構造体に記録されている変更済みブロック番号をデバッグログに出力する。

シグネチャまたは形状

void datapagemap_print(datapagemap_t *map)

入力と出力

  • **入力**: map (変更済みブロックを保持する datapagemap_t 構造体へのポインタ)
  • **出力**: なし（副作用としてデバッグログを出力）

前提条件と不変条件

  • map は有効な datapagemap_t 構造体を指している必要がある。
  • datapagemap_iterate が成功し、有効なイテレータを返すことが前提となる。

アルゴリズムまたは振る舞い

1. datapagemap_iterate(map) を呼び出し、イテレータを初期化する。 2. datapagemap_next を使用して、ビットマップ内のすべての変更済みブロック番号を順次取得する。 3. 取得した各ブロック番号を pg_log_debug を用いてログ出力する。 4. 最後に pg_free(iter) を呼び出し、イテレータに割り当てられたメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/filemap.c:556 にて entry->target_pages_to_overwrite を引数として呼び出されている。
  • **呼び出し先**:
  • datapagemap_iterate
  • datapagemap_next
  • pg_log_debug
  • pg_free

関連データ構造

  • datapagemap_t: 変更済みブロックを保持するビットマップ構造体。
  • datapagemap_iterator_t: 反復処理に使用されるイテレータ構造体。

失敗モードと境界条件

  • map が NULL の場合、datapagemap_iterate 内での動作に依存する。
  • 変更済みブロックが存在しない場合、ループは一度も実行されず、イテレータの解放のみが行われる。

保守担当者向けメモ

  • 本関数はデバッグ目的の出力を行うため、本番環境のパフォーマンスに影響を与える可能性がある点に留意すること。
  • datapagemap_iterate で確保されたメモリは本関数内で確実に解放される設計となっている。
