PortalRunSelect

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/pquery.c:863

目的

ポータル（カーソル）に対して指定された方向と件数でクエリ実行を行い、結果を DestReceiver へ配送する。

シグネチャまたは形状

uint64 PortalRunSelect(Portal portal, bool forward, long count, DestReceiver *dest)

入力と出力

  • **入力**:
  • portal: 実行対象のポータル構造体。
  • forward: 前方スキャンを行う場合は true、後方スキャンは false。
  • count: 取得するタプル数（FETCH_ALL は全件）。
  • dest: 結果の出力先インターフェース。
  • **出力**:
  • 実際に処理（取得）されたタプル数（uint64）。

前提条件と不変条件

  • portal->queryDesc または portal->holdStore のいずれかが有効であること。
  • 後方スキャンを行う場合、ポータルが CURSOR_OPT_NO_SCROLL オプションで作成されていないこと。

アルゴリズムまたは振る舞い

1. **出力先設定**: queryDesc が存在する場合、dest を現在の実行対象として更新する。 2. **スキャン方向の決定**:

  • forward 指定時、既に終端に達しているか count <= 0 なら NoMovementScanDirection を選択。
  • else（後方）指定時、既に始端に達しているか count <= 0 なら NoMovementScanDirection を選択。

    • 3. **実行**:

  • portal->holdStore があれば RunFromStore を呼び出す。
  • それ以外は PushActiveSnapshot でスナップショットを有効化し、ExecutorRun を実行後 PopActiveSnapshot を行う。

    • 4. **状態更新**: 処理件数に基づき、ポータルの atStart、atEnd、portalPos を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/pquery.c 内の PortalRun およびカーソル操作関連関数。
  • **呼び出し先**: RunFromStore、ExecutorRun、PushActiveSnapshot、PopActiveSnapshot。

関連データ構造

  • Portal: ポータルの状態を保持する構造体。
  • QueryDesc: 実行計画と実行状態を保持する構造体。
  • DestReceiver: クエリ結果の配送先を定義する構造体。

失敗モードと境界条件

  • **エラー**: CURSOR_OPT_NO_SCROLL が設定されたポータルに対して後方スキャンを試みた場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラーを報告する。
  • **境界条件**:
  • count が 0 の場合、実行エンジンには NoMovementScanDirection が渡され、セットアップとシャットダウンのみが行われる。
  • 取得件数が count 未満の場合、atStart または atEnd フラグが適切に更新される。

保守担当者向けメモ

  • queryDesc は保持されたカーソルや完了済みのユーティリティクエリでは NULL になるため、アクセス前に必ず holdStore の有無を確認すること。
  • ExecutorRun の呼び出し前後でスナップショットの管理（Push/PopActiveSnapshot）が必須である。
