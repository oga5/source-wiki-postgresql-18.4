pq_sendbytes

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqformat.c:125

目的

StringInfo バッファに対して、エンコーディング変換を行わない生のバイト列を直接追加します。

シグネチャまたは形状

void pq_sendbytes(StringInfo buf, const void *data, int datalen)

入力と出力

  • buf: データを追加する対象の StringInfo バッファ。
  • data: 追加するバイト列のポインタ。
  • datalen: 追加するデータの長さ（バイト単位）。
  • 戻り値: なし。

前提条件と不変条件

  • buf は初期化済みの StringInfo である必要があります。
  • 内部的に appendBinaryStringInfo を使用しており、バッファの末尾にヌル終端バイトを保持する特性を維持します。

アルゴリズムまたは振る舞い

1. 引数として受け取った data と datalen を、appendBinaryStringInfo 関数へ渡します。 2. StringInfo バッファの末尾に指定された長さのデータをコピーし、必要に応じてバッファを拡張します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/common/printtup.c
  • src/backend/commands/explain_dr.c
  • src/backend/libpq/auth.c
  • src/backend/replication/logical/proto.c
  • src/backend/replication/walsender.c
  • src/backend/tcop/fastpath.c
  • src/backend/tcop/postgres.c
  • 呼び出し先: appendBinaryStringInfo (直接の依存先として確認)

関連データ構造

  • StringInfo: メッセージ構築に使用される動的バッファ構造体。

失敗モードと境界条件

  • メモリ不足時: appendBinaryStringInfo の内部実装に従い、メモリ確保に失敗した場合はエラーが発生します。
  • 境界条件: datalen が 0 の場合、バッファの末尾にヌル終端バイトのみが維持されます。

保守担当者向けメモ

  • 本関数はエンコーディング変換を伴わないため、バイナリデータや既にエンコーディング済みのデータの送信に適しています。
  • 文字列を扱う際は、自動変換を行う pq_sendstring や pq_sendtext との使い分けに注意してください。
