clog_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/clogdesc.c:20

目的

CLOG（Commit Log）に関連するWALレコードの内容を解析し、人間が読み取り可能な形式に整形して StringInfo バッファに出力する。

シグネチャまたは形状

void clog_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 出力先となる StringInfo 構造体へのポインタ。
  • record: 解析対象の XLogReaderState 構造体へのポインタ。
  • **出力**: なし（buf に解析結果が追記される）。

前提条件と不変条件

  • record は有効なWALレコードを指していること。
  • XLogRecGetInfo で取得した情報ビットから XLR_INFO_MASK を除外した値が、CLOG_ZEROPAGE または CLOG_TRUNCATE のいずれかであること。

アルゴリズムまたは振る舞い

1. XLogRecGetInfo から補助フラグを除去した操作タイプ（info）を取得する。 2. info が CLOG_ZEROPAGE の場合:

  • レコードデータからページ番号（int64）を抽出し、buf に "page [pageno]" を追記する。

    • 3. info が CLOG_TRUNCATE の場合:

  • レコードデータから xl_clog_truncate 構造体を読み取り、ページ番号と最古のトランザクションIDを buf に "page [pageno]; oldestXact [oldestXact]" の形式で追記する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h にて RM_CLOG_ID リソースマネージャの記述関数として登録されており、WAL解析ツール（pg_waldump 等）から間接的に呼び出される。
  • **呼び出し先**: XLogRecGetData、XLogRecGetInfo、appendStringInfo、memcpy。

関連データ構造

  • StringInfo: 文字列バッファ。
  • XLogReaderState: WALレコード読み取り状態。
  • xl_clog_truncate: CLOG_TRUNCATE 操作時に使用される構造体。

失敗モードと境界条件

  • info が定義された操作タイプ以外の場合、何も出力されずに終了する。
  • レコードデータのサイズが期待される構造体サイズと一致しない場合、メモリコピー時に未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • 新しいCLOG操作が追加された場合は、本関数の条件分岐にケースを追加する必要がある。
  • XLR_INFO_MASK を用いたビットマスク処理は、WALレコードの補助フラグを無視するために必須である。
