ecpg_clear_auto_mem

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/memory.c:150

目的

スレッドローカルな自動メモリ追跡リスト（auto_mem 構造体のリスト）を破棄し、リスト管理用の構造体のみを解放します。

シグネチャまたは形状

void ecpg_clear_auto_mem(void)

入力と出力

  • 入力: なし（内部的に get_auto_allocs() を介してスレッドごとのリストを取得）
  • 出力: なし（副作用としてスレッドローカルなリストポインタを NULL にリセット）

前提条件と不変条件

  • get_auto_allocs() が返すリスト構造体自体は ecpg_free で解放可能であること。
  • 本関数はリスト構造体のみを解放し、リストが保持する実データ（確保されたメモリ領域）は解放しません。

アルゴリズムまたは振る舞い

1. get_auto_allocs() を呼び出し、現在のスレッドのメモリ追跡リストを取得します。 2. リストが存在する場合、do-while ループでリストの各ノードを辿ります。 3. 各ノード（struct auto_mem）に対して ecpg_free() を実行し、構造体メモリを解放します。 4. 全ノードの解放後、set_auto_allocs(NULL) を呼び出し、スレッドローカルなリストの参照をクリアします。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/interfaces/ecpg/ecpglib/connect.c:290
  • src/interfaces/ecpg/ecpglib/execute.c:2202
  • 呼び出し先:
  • get_auto_allocs()
  • ecpg_free()
  • set_auto_allocs()

関連データ構造

  • struct auto_mem: メモリ追跡リストを構成する構造体。

失敗モードと境界条件

  • リストが空（get_auto_allocs() が NULL を返す）の場合、何も行わず正常終了します。

保守担当者向けメモ

  • 本関数は、確保したメモリの所有権を外部へ移譲する際など、メモリの追跡のみを停止したい特殊なケースで使用されます。
  • 確保した実データを解放する必要がある場合は、代わりに ECPGfree_auto_mem を使用してください。
