BloomFillMetapage

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blutils.c:418

目的

Bloom インデックスのメタページ（BLOOM_METAPAGE_BLKNO）を初期化し、インデックスのライフサイクル全体で固定される設定オプションを書き込む。

シグネチャまたは形状

void BloomFillMetapage(Relation index, Page metaPage)

入力と出力

  • **入力**:
  • index: 対象インデックスの Relation 構造体。
  • metaPage: 初期化対象となるページバッファ。
  • **出力**: なし（metaPage の内容を直接更新する）。

前提条件と不変条件

  • index->rd_options が設定されているか、makeDefaultBloomOptions() が有効なデフォルト値を返す必要がある。
  • metaPage は BloomInitPage によってメタページとして初期化可能であること。
  • 処理終了後、PageHeader の pd_lower が pd_upper を超えないこと（アサーションにより検証）。

アルゴリズムまたは振る舞い

1. **オプションの決定**: index->rd_options が存在すれば使用し、なければ makeDefaultBloomOptions() を呼び出してデフォルト値を取得する。 2. **ページ初期化**: BloomInitPage(metaPage, BLOOM_META) を呼び出し、ページヘッダーをメタページとして設定する。 3. **メタデータ書き込み**:

  • BloomPageGetMeta でメタデータ領域を取得し、ゼロクリアする。
  • magickNumber に BLOOM_MAGICK_NUMBER を設定する。
  • 取得した opts をメタデータ構造体にコピーする。

    • 4. **ヘッダー更新**: PageHeader の pd_lower を BloomMetaPageData のサイズ分進める。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/bloom/blutils.c 内の別の関数（直接の呼び出し元はソースコードの文脈から推測可能だが、具体的な関数名は提供されていない）。
  • **呼び出し先**:
  • makeDefaultBloomOptions()
  • BloomInitPage()
  • BloomPageGetMeta()

関連データ構造

  • BloomOptions: インデックスの署名長や列数などの設定。
  • BloomMetaPageData: メタページに格納される永続的なメタデータ。
  • PageHeader: PostgreSQL のページレイアウト管理用ヘッダー。

失敗モードと境界条件

  • index->rd_options が不正な場合、makeDefaultBloomOptions() の挙動に依存する。
  • pd_lower が pd_upper を超える場合、アサーションエラーが発生する（FreeBlockNumberArray のサイズ計算ミス等が原因となり得る）。

保守担当者向けメモ

  • 本関数はインデックス作成時に一度だけ実行される。
  • BloomMetaPageData の構造を変更した場合は、pd_lower の更新ロジックとアサーション条件を再確認すること。
