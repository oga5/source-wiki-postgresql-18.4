MakeNewFileSetSegment

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/buffile.c:230

目的

FileSet 内に新しい物理ファイルセグメントを作成し、バッファ付きファイル（BufFile）の拡張をサポートする。

シグネチャまたは形状

static File MakeNewFileSetSegment(BufFile *buffile, int segment)

入力と出力

  • **入力**:
  • buffile: 対象となる BufFile 構造体へのポインタ。
  • segment: 作成するセグメントのインデックス番号。
  • **出力**:
  • 作成された物理ファイルの記述子（File 型）。

前提条件と不変条件

  • FileSetCreate が成功することを前提としており、失敗した場合はアサーションエラー（Assert(file > 0)）が発生する。
  • クラッシュ後の再起動時に残存する可能性のある、次のセグメント番号（segment + 1）のファイルは、混乱を避けるために事前に削除される。

アルゴリズムまたは振る舞い

1. **残存ファイルの削除**: FileSetSegmentName を使用して segment + 1 に該当するファイル名を生成し、FileSetDelete を呼び出して既存のファイルを削除する。 2. **新規セグメントの作成**: FileSetSegmentName を使用して segment に該当するファイル名を生成する。 3. **ファイル作成**: FileSetCreate を呼び出し、FileSet 内に新しい物理ファイルを作成する。 4. **検証**: 作成されたファイル記述子が有効（> 0）であることを確認する。 5. **返却**: 作成されたファイル記述子を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/file/buffile.c 内の BufFile 拡張処理（file->numFiles の更新時や、初期セグメント作成時など）。
  • **呼び出し先**:
  • FileSetSegmentName
  • FileSetDelete
  • FileSetCreate

関連データ構造

  • BufFile: 複数の物理ファイルセグメントを管理する論理ファイル構造体。
  • FileSet: 共有一時ファイルを管理するためのコンテナ。
  • File: fd.c が管理する仮想ファイル記述子。

失敗モードと境界条件

  • FileSetCreate が失敗した場合、アサーションによりプロセスが停止する。
  • segment + 1 の削除処理は、ファイルが存在しない場合は無視される（FileSetDelete の仕様に依存）。

保守担当者向けメモ

  • 本関数は static であり、buffile.c 内のバッファ管理ロジックからのみ呼び出される。
  • segment + 1 を削除するロジックは、クラッシュリカバリ後の整合性維持のために重要である。
