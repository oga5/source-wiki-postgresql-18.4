AdvanceOldestClogXid

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/varsup.c:354

目的

共有メモリ上の oldestClogXid を、指定された新しい oldest_datfrozenxid で更新します。これにより、CLOG（Commit Log）の切り捨て可能な範囲を管理します。

シグネチャまたは形状

void AdvanceOldestClogXid(TransactionId oldest_datfrozenxid)

入力と出力

  • **入力**: oldest_datfrozenxid (TransactionId型) - 更新対象となる新しい凍結トランザクションID。
  • **出力**: なし。

前提条件と不変条件

  • XactTruncationLock を排他モードで取得する必要があります。
  • oldestClogXid は、常に単調増加（あるいは維持）されることが期待されます。

アルゴリズムまたは振る舞い

1. XactTruncationLock を排他モードで取得します。 2. 現在の TransamVariables->oldestClogXid が、引数 oldest_datfrozenxid よりも古い（TransactionIdPrecedes が真）場合のみ、oldestClogXid を oldest_datfrozenxid に更新します。 3. XactTruncationLock を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/clog.c (2箇所)
  • src/backend/access/transam/xlog.c (2箇所)
  • **呼び出し先**:
  • LWLockAcquire
  • TransactionIdPrecedes
  • LWLockRelease

関連データ構造

  • TransamVariables: 共有メモリ上のグローバル変数構造体。oldestClogXid フィールドを保持します。
  • XactTruncationLock: 本関数が排他制御に使用する軽量ロック（LWLock）。

失敗モードと境界条件

  • 引数が現在の値よりも古い場合、更新は行われません。
  • ロック取得時に他のプロセスとの競合が発生する可能性があります。

保守担当者向けメモ

  • 本関数は XactTruncationLock を使用して共有状態を保護しています。このロックはトランザクションの切り捨てに関連する重要なロックであるため、呼び出し頻度や保持時間に注意してください。
