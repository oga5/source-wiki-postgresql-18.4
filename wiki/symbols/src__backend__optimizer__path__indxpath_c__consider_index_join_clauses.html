consider_index_join_clauses

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/indxpath.c:436

目的

結合条件（Join Clauses）を利用可能なインデックススキャンパスを生成するための候補を特定し、パラメータ化されたインデックスパスの構築を試みます。

シグネチャまたは形状

static void consider_index_join_clauses(
    PlannerInfo *root, 
    RelOptInfo *rel,
    IndexOptInfo *index,
    IndexClauseSet *rclauseset,
    IndexClauseSet *jclauseset,
    IndexClauseSet *eclauseset,
    List **bitindexpaths
)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel: 対象となるリレーション情報。
  • index: 評価対象のインデックス情報。
  • rclauseset, jclauseset, eclauseset: それぞれ制限句、結合句、等価クラス由来の結合句を保持する構造体。
  • **出力**:
  • bitindexpaths: 生成されたビットマップインデックスパスのリスト（ポインタ経由で更新）。

前提条件と不変条件

  • インデックスの各キー列に対して、利用可能な結合条件（jclauseset および eclauseset）が存在することが前提です。
  • 結合条件をインデックスの検索条件（indexqual）として適用することは、フィルタ（qpqual）として適用するよりも常に優先されるという方針に基づいています。

アルゴリズムまたは振る舞い

1. インデックスの各キー列（index->nkeycolumns）を順次走査します。 2. 各列において、jclauseset および eclauseset に含まれる結合条件を抽出します。 3. 抽出した結合条件を consider_index_join_outer_rels に渡し、パラメータ化されたパスの生成を試みます。 4. 探索効率を維持するため、ヒューリスティックな制限（考慮する外部リレーションセット数の制限）を適用しつつ、可能な限り多くの結合条件の組み合わせを試行します。 5. 既に試行済みの外部リレーションセットは considered_relids で管理し、重複を避けます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/indxpath.c 内の create_index_paths から呼び出されます。
  • **呼び出し先**: consider_index_join_outer_rels を呼び出し、実際のパス生成処理を委譲します。

関連データ構造

  • IndexOptInfo: インデックスのメタデータ。
  • IndexClauseSet: インデックス列ごとの適合条件を保持する構造体。
  • PlannerInfo: クエリ全体のプランニング状態。

失敗モードと境界条件

  • 結合条件が一つも存在しない場合、本関数は何も生成せず終了します。
  • 外部リレーションセットが膨大になる場合、ヒューリスティックな制限により一部の組み合わせがスキップされる可能性があります。

保守担当者向けメモ

  • 本関数はパラメータ化されたパス生成の制御層です。詳細なコスト計算やパス構築のロジックは consider_index_join_outer_rels に依存しています。
  • 結合条件の適用順序や最適化方針を変更する場合は、considered_relids による重複排除のロジックと、ヒューリスティックな制限値の妥当性を再確認してください。
