spgxlogAddNode

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgxlog.h:99

目的

SP-GiST インデックスにおいて、内部タプル（inner tuple）に新しいノードを追加する操作を記録するための WAL レコード構造体です。

シグネチャまたは形状

typedef struct spgxlogAddNode
{
    OffsetNumber offnum;
    OffsetNumber offnumNew;
    bool         newPage;
    int8         parentBlk;
    OffsetNumber offnumParent;
    uint16       nodeI;
    spgxlogState stateSrc;
    /* 更新された内部タプルが続く（アライメントなし） */
} spgxlogAddNode;

入力と出力

  • **入力**: インデックス操作時に本構造体に値を設定し、WAL に書き出します。
  • **出力**: リカバリ時に spg_redo 関数が本構造体を読み取り、対象ページの状態を復元します。

前提条件と不変条件

  • 構造体直後に続く内部タプルデータはアライメントされていません。アクセス時には memcpy 等を用いて適切にコピーする必要があります。
  • parentBlk は親ダウンリンクの所在を示し、0（オリジナルページ）、1（新規ページ）、2（別ページ）、-1（更新なし）のいずれかの値をとります。

アルゴリズムまたは振る舞い

1. **記録**: インデックスのノード追加操作時に、対象のオフセットや親ブロック情報を本構造体に格納し、XLogInsert を介して WAL に書き込みます。 2. **再生**: リカバリ時、spgxlogAddNode を読み込み、offnum（オリジナルページ）や offnumNew（新規ページ）の情報に基づき、ページ上のアイテムを更新します。 3. **データ配置**: 構造体末尾の可変長データ領域には、更新後の内部タプルが格納されています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgdoinsert.c: インデックス挿入処理中に本構造体を使用。
  • **呼び出し先**:
  • src/backend/access/rmgrdesc/spgdesc.c: WAL ログのデコード（説明出力）に使用。
  • src/backend/access/spgist/spgxlog.c: リカバリ時の Redo ロジック内で使用。

関連データ構造

  • spgxlogState: リカバリに必要なインデックス状態を保持する構造体。
  • OffsetNumber: ページ内のオフセット管理に使用。

失敗モードと境界条件

  • offnumNew が無効な場合、オリジナルページ内の古いタプルが上書きされたことを示します。
  • parentBlk が -1 の場合、親ダウンリンクの更新は行われません。

保守担当者向けメモ

  • 本構造体は可変長データを含むため、SizeOf... マクロ等を用いてサイズ計算を行う際は、構造体の固定部分と後続データの境界に注意してください。
  • バックアップブロックの番号（0, 1, 2）と parentBlk の対応関係がリカバリロジックの肝となります。
