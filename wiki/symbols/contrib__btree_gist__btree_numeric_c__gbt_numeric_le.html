gbt_numeric_le

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_numeric.c:52

目的

PostgreSQL の numeric 型に対する「以下（less than or equal to）」の比較演算を GiST インデックスのサポート関数として提供します。

シグネチャまたは形状

static bool gbt_numeric_le(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の numeric 型データへのポインタ。
  • collation: 照合順序（本関数では使用されません）。
  • flinfo: 関数呼び出し情報（本関数では使用されません）。
  • **出力**:
  • a <= b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な numeric 型のデータ構造を指している必要があります。
  • PostgreSQL のコア関数 numeric_le が正常に動作する環境である必要があります。

アルゴリズムまたは振る舞い

1. 入力された a と b を PointerGetDatum を用いて Datum 型に変換します。 2. DirectFunctionCall2 を介して PostgreSQL コアの numeric_le 関数を呼び出します。 3. numeric_le の戻り値を DatumGetBool で bool 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し先**: numeric_le（PostgreSQL コア関数）。
  • **呼び出し元**: contrib/btree_gist/btree_numeric.c 内の静的テーブル（tinfo 構造体初期化時など）から参照されています。具体的な実行時の呼び出し元はソースコードからは特定できません。

関連データ構造

  • numeric: PostgreSQL の可変長数値データ型。
  • tinfo (gbtree_vinfo): numeric 型の比較関数群を保持する設定構造体。

失敗モードと境界条件

  • numeric_le の仕様に依存します。入力が NULL の場合や、numeric 型として不正なデータが渡された場合の挙動はコア関数の実装に従います。

保守担当者向けメモ

  • 本関数は numeric_le への単なるラッパーです。比較ロジックに変更が必要な場合は、コアの numeric 型実装を確認してください。
  • btree_gist モジュールにおける他の比較関数（gbt_numeric_gt 等）と一貫した実装パターンに従っています。
