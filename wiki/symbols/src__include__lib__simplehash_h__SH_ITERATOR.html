SH_ITERATOR

定義

  • 種別: struct
  • 定義箇所: src/include/lib/simplehash.h:181

目的

simplehash.h で生成されるハッシュテーブル内を走査するためのイテレータ構造体です。

シグネチャまたは形状

typedef struct SH_ITERATOR
{
    uint32      cur;            /* 現在の要素 */
    uint32      end;            /* 終了位置 */
    bool        done;           /* 走査完了フラグ */
} SH_ITERATOR;

入力と出力

  • 入力: SH_START_ITERATE または SH_START_ITERATE_AT を介して初期化されます。
  • 出力: SH_ITERATE 関数により、ハッシュテーブル内の各要素へのポインタが順次返されます。

前提条件と不変条件

  • SH_ITERATOR は、対応するハッシュテーブル（SH_TYPE）の走査状態を保持します。
  • 走査中に要素の削除を行うことは安全ですが、挿入が行われた場合の挙動については注意が必要です。

アルゴリズムまたは振る舞い

  • SH_START_ITERATE で走査を開始し、SH_ITERATE を繰り返し呼び出すことで、ハッシュテーブル内の全要素を順次取得します。
  • done フラグは、すべてのバケットの走査が完了した時点で真となります。

呼び出し元と呼び出し先

  • 呼び出し元: SH_START_ITERATE、SH_START_ITERATE_AT、SH_ITERATE の各マクロ/関数が本構造体を利用します。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • SH_TYPE: 走査対象となるハッシュテーブルの型。
  • SH_ELEMENT_TYPE: ハッシュテーブルに格納されている要素の型。

失敗モードと境界条件

  • 走査中にハッシュテーブルがリサイズ（SH_GROW）された場合、イテレータの整合性が保たれるかは保証されません。

保守担当者向けメモ

  • 本構造体は simplehash.h のマクロ展開によって生成されるコードの一部として利用されます。
  • 走査中の要素削除はサポートされていますが、挿入操作が混在する場合の挙動については、利用側の実装で考慮が必要です。
