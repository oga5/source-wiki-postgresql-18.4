pgwin32_SharedMemoryDelete

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32_shmem.c:548

目的

プロセス終了時に共有メモリセグメントのデタッチ処理を実行し、リソースを解放する。

シグネチャまたは形状

static void pgwin32_SharedMemoryDelete(int status, Datum shmId)

入力と出力

  • status: 終了ステータス（on_shmem_exit のコールバック規約に従う）。
  • shmId: 削除対象の共有メモリセグメントを識別する Datum 型のハンドル。
  • 戻り値: なし。

前提条件と不変条件

  • 引数 shmId がグローバル変数 UsedShmemSegID と一致している必要がある（Assert による検証）。

アルゴリズムまたは振る舞い

1. Assert を用いて、渡された shmId が現在使用中の共有メモリセグメント ID (UsedShmemSegID) と一致することを確認する。 2. PGSharedMemoryDetach() を呼び出し、現在のプロセスから共有メモリをデタッチする。

呼び出し元と呼び出し先

  • 呼び出し元: on_shmem_exit に登録されたコールバックとして、プロセス終了時に呼び出される（src/backend/port/win32_shmem.c:401）。
  • 呼び出し先: PGSharedMemoryDetach()。

関連データ構造

  • UsedShmemSegID: 現在使用中の共有メモリハンドルを保持するグローバル変数。

失敗モードと境界条件

  • Assert が有効なビルドにおいて、shmId が UsedShmemSegID と一致しない場合、プログラムは異常終了する。

保守担当者向けメモ

  • 本関数は on_shmem_exit 経由で呼び出されることを前提としている。
  • Windows 環境における共有メモリのライフサイクル管理の一部であり、プロセス終了時のクリーンアップを担当する。
