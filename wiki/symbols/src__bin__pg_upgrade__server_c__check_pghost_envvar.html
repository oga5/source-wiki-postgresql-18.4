check_pghost_envvar

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/server.c:359

目的

pg_upgrade 実行時に、環境変数 PGHOST または PGHOSTADDR がリモートサーバーを指していないかを検証し、誤った環境への接続を防止する安全装置として機能します。

シグネチャまたは形状

void check_pghost_envvar(void);

入力と出力

  • **入力**: 環境変数 PGHOST および PGHOSTADDR（getenv を介して取得）。
  • **出力**: なし（正常終了時）。異常時は pg_fatal を呼び出し、プログラムを終了させます。

前提条件と不変条件

  • libpq のデフォルト設定情報（PQconndefaults）がメモリ上に確保可能であること。
  • 実行環境において libpq が正しく初期化されていること。

アルゴリズムまたは振る舞い

1. PQconndefaults() を呼び出し、libpq が認識する環境変数の一覧を取得します。 2. 一覧から PGHOST および PGHOSTADDR を特定します。 3. 各環境変数が設定されている場合、その値を確認します。 4. 値が以下のいずれにも該当しない場合、リモートサーバーへの接続とみなして pg_fatal でエラー終了します。

  • localhost
  • 127.0.0.1
  • ::1
  • Unix ソケットパス（is_unixsock_path による判定）

    • 5. 最後に PQconninfoFree() を呼び出し、確保したメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/pg_upgrade.c:344
  • **呼び出し先**:
  • PQconndefaults (libpq)
  • getenv (標準ライブラリ)
  • is_unixsock_path (内部ユーティリティ)
  • pg_fatal (内部ユーティリティ)
  • PQconninfoFree (libpq)

関連データ構造

  • PQconninfoOption: libpq が提供する接続オプション構造体。

失敗モードと境界条件

  • **メモリ不足**: PQconndefaults が NULL を返した場合、pg_fatal により終了します。
  • **リモート接続の検知**: ローカルホストや Unix ソケット以外が指定されている場合、安全のために処理を中断します。
  • **環境変数未設定**: PGHOST 等が設定されていない場合は何も行わず正常に終了します。

保守担当者向けメモ

  • 本関数は pg_upgrade の初期化フェーズで呼び出される重要なチェック機構です。
  • is_unixsock_path の判定ロジックに変更があった場合、本関数の挙動も影響を受ける可能性があります。
