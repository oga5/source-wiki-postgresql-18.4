EnableQueryId

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/queryjumblefuncs.c:168

目的

クエリID（queryId）の計算機能が有効である場合に、グローバルな有効フラグ query_id_enabled を true に設定します。

シグネチャまたは形状

void EnableQueryId(void)

入力と出力

  • 入力: なし（グローバル変数 compute_query_id を参照）
  • 出力: なし（グローバル変数 query_id_enabled を更新）

前提条件と不変条件

  • compute_query_id が COMPUTE_QUERY_ID_OFF 以外の値に設定されている場合にのみ、query_id_enabled が true に更新されます。

アルゴリズムまたは振る舞い

1. 現在の GUC パラメータ compute_query_id の値を確認します。 2. compute_query_id が COMPUTE_QUERY_ID_OFF でない場合、query_id_enabled を true に設定します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/pg_stat_statements/pg_stat_statements.c:402
  • 呼び出し先:
  • 提供されたソース excerpt に呼び出し先は含まれていません。

関連データ構造

  • compute_query_id (GUC パラメータ)
  • query_id_enabled (グローバルフラグ)

失敗モードと境界条件

  • compute_query_id が COMPUTE_QUERY_ID_OFF の場合、本関数を呼び出しても query_id_enabled は変更されません。

保守担当者向けメモ

  • 本関数は src/include/nodes/queryjumble.h で外部公開されています。
  • query_id_enabled の状態は、クエリのフィンガープリント処理（JumbleQuery 等）の実行可否を決定するために使用されます。
