PartitionedRelPruningData

定義

  • 種別: struct
  • 定義箇所: src/include/executor/execPartition.h:63

目的

パーティション化されたテーブルに対する実行時プルーニング（不要なパーティションの除外）に必要な情報を保持するデータ構造です。

シグネチャまたは形状

typedef struct PartitionedRelPruningData
{
	Relation	partrel;
	int			nparts;
	int		   *subplan_map;
	int		   *subpart_map;
	int		   *leafpart_rti_map;
	Bitmapset  *present_parts;
	List	   *initial_pruning_steps;
	List	   *exec_pruning_steps;
	PartitionPruneContext initial_context;
	PartitionPruneContext exec_context;
} PartitionedRelPruningData;

入力と出力

  • **入力**: 実行計画の初期化時および実行時に、パーティションプルーニングのコンテキスト（PartitionPruneContext）として利用されます。
  • **出力**: プルーニングの結果、どのサブプランやパーティションが有効であるかを特定するための情報源となります。

前提条件と不変条件

  • initial_pruning_steps および exec_pruning_steps は、それぞれ初期化時と実行時のプルーニング手順を定義します。
  • PartitionPruneContext は、プルーニングの評価に必要な状態を管理します。

アルゴリズムまたは振る舞い

  • 本構造体は、クエリ実行中にパラメータ値等に基づいて、スキャン対象から除外すべきパーティションを特定する仕組みを提供します。
  • 実行計画の初期化時（initial_prune）と実行時（exec_prune）の両方で参照され、動的なパーティション除外を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execPartition.c 内の関数群（198, 2034, 2042, 2398, 2453, 2527, 2572 行付近）で参照されています。
  • **呼び出し先**: 具体的な関数呼び出しの詳細はソース抜粋からは不明です。

関連データ構造

  • Relation: 対象となるパーティションテーブル。
  • PartitionPruneContext: プルーニングの評価コンテキスト。
  • Bitmapset: 有効なパーティションの集合管理に使用。
  • PartitionPruneState: 本構造体を利用して実行時の状態を管理する上位構造体。

失敗モードと境界条件

  • パーティションの階層構造が深い場合、PartitionDispatch を介した再帰的な処理が必要となるため、階層の深さに応じたメモリ管理やスタック使用量に注意が必要です。

保守担当者向けメモ

  • 本構造体は execPartition.c におけるプルーニング処理の核となります。
  • 実行計画の初期化と実行時の両方で再利用されるため、状態の整合性に注意してください。
  • 詳細は src/backend/executor/execPartition.c の実装を参照してください。
