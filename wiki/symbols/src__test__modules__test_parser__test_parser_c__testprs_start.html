testprs_start

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_parser/test_parser.c:46

目的

PostgreSQLの全文検索用カスタムパーサーの解析セッションを開始し、解析状態を保持するための ParserState 構造体を初期化します。

シグネチャまたは形状

Datum testprs_start(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): 解析対象となる入力文字列のバッファ。
  • PG_GETARG_INT32(1): 入力文字列の長さ。
  • **出力**:
  • PG_RETURN_POINTER(pst): 初期化された ParserState 構造体へのポインタを Datum 型として返します。

前提条件と不変条件

  • palloc0 により ParserState 構造体分のメモリがゼロクリアされた状態で確保されます。
  • pst->pos は解析開始位置として 0 に初期化されます。

アルゴリズムまたは振る舞い

1. palloc0 を使用して ParserState 構造体用のメモリを動的に確保します。 2. 第1引数から入力バッファのポインタを取得し、pst->buffer に格納します。 3. 第2引数から入力長を取得し、pst->len に格納します。 4. 解析位置 pst->pos を 0 に設定します。 5. 確保した ParserState 構造体へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの全文検索システム（ts_parser インターフェース経由で呼び出されることが想定されます）。
  • **呼び出し先**: palloc0（PostgreSQLのメモリ管理関数）。

関連データ構造

  • ParserState: 解析中のバッファ、長さ、現在位置を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させ、処理が中断されます。
  • 入力引数が正しく渡されない場合、PG_GETARG_* マクロの仕様に従い動作します。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_INFO_V1(testprs_start) により、PostgreSQLの関数マネージャに登録されています。
  • 解析終了時には、対応する testprs_end 関数で pfree を呼び出し、メモリを解放する必要があります。
