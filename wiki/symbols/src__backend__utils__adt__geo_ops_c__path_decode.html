path_decode

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_ops.c:265

目的

幾何データ型（パス、ポリゴン、ボックス、線分など）のテキスト表現を解析し、座標値の配列（Point）へ変換する共通ユーティリティ関数です。

シグネチャまたは形状

static bool path_decode(char *str, bool opentype, int npts, Point *p,
                        bool *isopen, char **endptr_p,
                        const char *type_name, const char *orig_string,
                        Node *escontext);

入力と出力

  • **入力**:
  • str: 解析対象の文字列。
  • opentype: 開いたパス（isopen）を許容するかどうか。
  • npts: 期待される点の数。
  • type_name, orig_string: エラー報告用の型名と元の文字列。
  • escontext: エラー処理用ノード。
  • **出力**:
  • p: 解析された座標が格納される Point 配列のポインタ。
  • isopen: パスがオープン形式であるかどうかのフラグ。
  • endptr_p: 解析終了位置（指定時）。
  • 戻り値: 成功時に true、失敗時に false。

前提条件と不変条件

  • p は npts 分の Point を格納できる十分なメモリ領域を指している必要があります。
  • 入力文字列は、PostgreSQLの幾何データ型で定義された区切り文字（LDELIM, RDELIM 等）に従った形式である必要があります。

アルゴリズムまたは振る舞い

1. 先頭の空白をスキップし、開始デリミタ（LDELIM_EP または LDELIM）を確認して isopen フラグを設定します。 2. npts 回ループし、pair_decode を呼び出して各点の座標（x, y）を順次解析します。 3. 各点の間に区切り文字（DELIM）がある場合はスキップします。 4. 終了デリミタ（RDELIM）を解析し、入れ子構造（depth）が解消されるまで処理を継続します。 5. endptr_p が指定されている場合は解析終了位置を返し、指定がない場合は文字列の末尾まで到達したことを検証します。 6. 構文エラー発生時は ereturn マクロを介してエラーを報告し false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • box_in (src/backend/utils/adt/geo_ops.c:431)
  • line_in (ソース内 1002)
  • path_in (ソース内 1445)
  • lseg_in (ソース内 2072)
  • poly_in (ソース内 3445)
  • **呼び出し先**:
  • pair_decode
  • ereturn (エラー処理用マクロ)

関連データ構造

  • Point: 座標を保持する構造体。
  • Node: エラーコンテキスト用。

失敗モードと境界条件

  • opentype が false なのにオープン形式のパスが入力された場合、エラーとなります。
  • 期待される点数（npts）と実際の入力数が一致しない場合、解析に失敗します。
  • 終了デリミタが不足している、または不正な文字が含まれている場合、ERRCODE_INVALID_TEXT_REPRESENTATION を伴うエラーとなります。

保守担当者向けメモ

  • 本関数は複数の幾何データ型で共通利用されているため、修正時は box, line, path, lseg, polygon の各型への影響を考慮してください。
  • depth 変数は入れ子構造の解析に使用されており、複雑な幾何学的表現のパースにおいて重要な役割を果たします。
