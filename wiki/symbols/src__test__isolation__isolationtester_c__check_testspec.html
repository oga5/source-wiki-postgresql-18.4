check_testspec

定義

  • 種別: function
  • 定義箇所: src/test/isolation/isolationtester.c:251

目的

TestSpec 構造体内のテスト仕様を検証し、ステップ名の一意性確認、セッション情報の紐付け、および順列（Permutation）におけるステップ間ブロック条件の解決を行う。

シグネチャまたは形状

static void check_testspec(TestSpec *testspec)

入力と出力

  • 入力: TestSpec *testspec （解析済みのテスト仕様データ構造）
  • 出力: なし（検証失敗時は stderr にエラーを出力し exit(1) を実行）

前提条件と不変条件

  • testspec は有効なポインタであること。
  • ステップ名はシステム全体で一意である必要がある。
  • ブロック条件（PermutationStepBlocker）において、同一セッション内のステップをブロック対象に指定することはできない。

アルゴリズムまたは振る舞い

1. **ステップの収集とソート**: 全セッションの全ステップを allsteps 配列に集約し、step_qsort_cmp を用いて名前順にソートする。 2. **一意性検証**: ソート済み配列を走査し、隣接するステップ名が重複していないか確認する。 3. **セッション紐付け**: 各ステップ構造体に所属するセッションのインデックスを記録する。 4. **順列の解決**:

  • 各順列（Permutation）内のステップ名を allsteps から検索し、PermutationStep 構造体に実体へのポインタを格納する。
  • 各ステップのブロック条件を走査し、対象となるステップを特定してリンクを構築する。

    • 5. **未使用ステップの警告**: 手動指定の順列が存在する場合、定義されたステップが一度も使用されていない場合に警告を出力する。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/isolation/isolationtester.c 内のメインロジック（main 関数付近）。
  • 呼び出し先: step_qsort_cmp, step_bsearch_cmp（比較関数）、pg_malloc, qsort, bsearch, strcmp, fprintf, exit。

関連データ構造

  • TestSpec: テスト全体の仕様を保持。
  • Step: 個別のテストステップ。
  • Permutation: 実行順序の定義。
  • PermutationStepBlocker: ステップ間の依存関係（ブロック条件）。

失敗モードと境界条件

  • **重複ステップ名**: 異なるセッション間で同じ名前のステップが存在する場合、エラー終了する。
  • **未定義ステップ**: 順列内で指定されたステップ名が定義されていない場合、エラー終了する。
  • **不正なブロック条件**: 存在しないステップをブロック対象に指定した場合、または同一セッション内のステップをブロック対象に指定した場合、エラー終了する。

保守担当者向けメモ

  • 本関数は isolationtester.h で定義される Step や PermutationStep の一部フィールド（session, step, used 等）を初期化する役割を持つ。
  • allsteps 配列は関数終了時に free される。
  • ブロック条件の解決は、順列ごとに独立して行われるため、他の順列のステップを参照することはできない。
