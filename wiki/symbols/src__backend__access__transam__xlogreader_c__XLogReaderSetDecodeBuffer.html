XLogReaderSetDecodeBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogreader.c:90

目的

XLogReaderState に対して、WALレコードのデコード処理で使用する外部バッファを割り当て、初期化します。

シグネチャまたは形状

void XLogReaderSetDecodeBuffer(XLogReaderState *state, void *buffer, size_t size)

入力と出力

  • **入力**:
  • state: 操作対象の XLogReaderState 構造体へのポインタ。
  • buffer: デコード用に使用するメモリ領域へのポインタ。
  • size: buffer のバイトサイズ。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • state->decode_buffer が NULL であること（既にバッファが割り当てられていないこと）を Assert で検証しています。

アルゴリズムまたは振る舞い

1. state->decode_buffer が NULL であることを確認します。 2. 引数で渡された buffer を state->decode_buffer に設定します。 3. state->decode_buffer_size に size を設定します。 4. state->decode_buffer_tail および state->decode_buffer_head を、提供された buffer の開始アドレスで初期化します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogrecovery.c:579 にて呼び出されています。
  • **呼び出し先**:
  • 本関数内から他の関数を呼び出している記述はありません。

関連データ構造

  • XLogReaderState: WALリーダーの状態を保持するメイン構造体。本関数はこの構造体内のデコード用バッファ関連フィールドを更新します。

失敗モードと境界条件

  • state が NULL の場合、あるいは state->decode_buffer が既に設定されている場合に Assert 違反が発生します。

保守担当者向けメモ

  • 本関数は XLogReaderState の初期化プロセスの一部として使用されます。
  • buffer に NULL を渡す呼び出し元（xlogrecovery.c）が存在するため、buffer が NULL である場合の挙動については、後続のデコード処理側でのガード条件を確認してください。
