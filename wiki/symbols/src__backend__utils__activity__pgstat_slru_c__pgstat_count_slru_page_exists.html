pgstat_count_slru_page_exists

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_slru.c:70

目的

指定されたSLRU（Simple Least Recently Used）キャッシュにおける「ページ存在確認」操作の発生回数を統計情報として記録する。

シグネチャまたは形状

void pgstat_count_slru_page_exists(int slru_idx)

入力と出力

  • **入力**: slru_idx (int) - 統計対象となるSLRUの識別インデックス。
  • **出力**: なし。副作用として、内部の統計カウンタが更新される。

前提条件と不変条件

  • slru_idx は有効なSLRUインデックスである必要がある。
  • 本関数は、各バックエンドが保持するローカルな統計カウンタ（pending_SLRUStats）を更新する。

アルゴリズムまたは振る舞い

1. get_slru_entry(slru_idx) を呼び出し、指定されたインデックスに対応する統計エントリを取得する。 2. 取得したエントリの blocks_exists メンバを 1 インクリメントする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/slru.c:757
  • **呼び出し先**: get_slru_entry（インライン化されている可能性あり）

関連データ構造

  • pending_SLRUStats: 各バックエンドが保持するローカルな統計カウンタ配列。
  • PgStatShared_SLRU: 共有メモリ上のSLRU統計構造体（フラッシュ時に本関数の結果が反映される）。

失敗モードと境界条件

  • 無効な slru_idx が渡された場合の動作については、get_slru_entry の実装に依存する。

保守担当者向けメモ

  • 本関数は統計収集のフックであり、高頻度で呼び出される可能性があるため、処理は最小限に抑えられている。
  • 共有メモリへの同期は、別途 pgstat_slru_flush_cb を通じて行われるため、本関数自体はロックを取得しない。
