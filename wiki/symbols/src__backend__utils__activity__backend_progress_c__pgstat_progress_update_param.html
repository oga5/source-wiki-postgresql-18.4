pgstat_progress_update_param

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/backend_progress.c:47

目的

現在実行中のバックエンドプロセスに関連付けられた進捗状況パラメータを更新します。

シグネチャまたは形状

void pgstat_progress_update_param(int index, int64 val)

入力と出力

  • index (int): 更新対象のパラメータインデックス（0〜19）。
  • val (int64): 設定する値。
  • 戻り値: なし。

前提条件と不変条件

  • index は 0 以上 PGSTAT_NUM_PROGRESS_PARAM 未満である必要があります（Assert により検証）。
  • pgstat_track_activities が有効である必要があります。無効な場合、関数は何も行わずにリターンします。

アルゴリズムまたは振る舞い

1. 現在のバックエンドの共有メモリ状態（MyBEEntry）を取得します。 2. beentry が NULL または pgstat_track_activities が無効な場合は即座に終了します。 3. PGSTAT_BEGIN_WRITE_ACTIVITY マクロを使用して、共有メモリへの書き込みロック（またはメモリバリア）を確保します。 4. 指定された index に対応する st_progress_param 配列の要素に val を代入します。 5. PGSTAT_END_WRITE_ACTIVITY マクロを使用して書き込みを完了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gininsert.c
  • src/backend/access/hash/hash.c
  • src/backend/access/hash/hashsort.c
  • src/backend/access/heap/heapam_handler.c
  • **呼び出し先**:
  • 本関数はマクロ PGSTAT_BEGIN_WRITE_ACTIVITY および PGSTAT_END_WRITE_ACTIVITY を使用します。これら以外の具体的な呼び出し先情報は提供されていません。

関連データ構造

  • PgBackendStatus: 共有メモリ上に存在するバックエンドの状態構造体。
  • MyBEEntry: 現在のバックエンドプロセスが参照する PgBackendStatus へのポインタ。

失敗モードと境界条件

  • index が範囲外の場合、アサーション違反が発生します。
  • pgstat_track_activities が無効な場合、更新は無視されます。
  • MyBEEntry が初期化されていない場合、更新は行われません。

保守担当者向けメモ

  • 本関数はパラメータの単一更新を行います。複数のパラメータをアトミックに更新する必要がある場合は、pgstat_progress_update_multi_param の使用を検討してください。
  • 各インデックスの具体的な意味論は、本関数ではなく各コマンドの実装側に依存します。
