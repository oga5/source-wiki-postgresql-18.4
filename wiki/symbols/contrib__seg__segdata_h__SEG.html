SEG

定義

  • 種別: struct
  • 定義箇所: contrib/seg/segdata.h:4

目的

数値範囲（Segment）を表現するためのデータ構造です。境界値および各境界の有効桁数や拡張フラグを保持し、contrib/seg モジュールにおける範囲演算の基盤となります。

シグネチャまたは形状

typedef struct SEG
{
	float4		lower;
	float4		upper;
	char		l_sigd;
	char		u_sigd;
	char		l_ext;
	char		u_ext;
} SEG;

入力と出力

  • 本構造体は、seg_yyparse 等の解析器によって生成された結果を保持するために使用されます。
  • seg.c 内の関数において、PG_GETARG_SEG_P マクロ等を通じて PostgreSQL の Datum から変換されたポインタとして入力されます。

前提条件と不変条件

  • float4 型の境界値と char 型のメタデータが混在しているため、メモリレイアウト上のアライメントに注意が必要です。
  • 解析器によって生成された後、seg.c 内の演算関数で利用されることが想定されています。

アルゴリズムまたは振る舞い

  • 数値範囲の「下限（lower）」と「上限（upper）」を保持します。
  • l_sigd / u_sigd は各境界の有効桁数を、l_ext / u_ext は拡張フラグを保持し、範囲の精度や特性を管理します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/seg/seg.c 内の各種関数（PG_GETARG_SEG_P を使用する関数群や、palloc を用いてメモリを確保する関数群）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • yyscan_t: 字句解析器の状態を保持する不透明ポインタ。
  • Datum: PostgreSQL の汎用データ型。DatumGetSegP マクロにより SEG * へ変換されます。

失敗モードと境界条件

  • メモリ確保時（palloc）の失敗が考えられます。
  • 構造体のパディングによるメモリレイアウトの差異が、プラットフォーム間で影響する可能性があります。

保守担当者向けメモ

  • 本構造体は seg.c、segscan.l、segparse.y 間で共有される「契約」です。変更時はこれらすべてのファイルへの影響を考慮してください。
  • float4 と char の混在によるアライメントの影響を考慮した設計となっています。
