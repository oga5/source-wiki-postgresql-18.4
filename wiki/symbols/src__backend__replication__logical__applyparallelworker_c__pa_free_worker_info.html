pa_free_worker_info

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/applyparallelworker.c:594

目的

並列適用ワーカーに関連するリソースを解放し、ワーカープールから当該ワーカー情報を削除する。

シグネチャまたは形状

static void pa_free_worker_info(ParallelApplyWorkerInfo *winfo)

入力と出力

  • **入力**: ParallelApplyWorkerInfo *winfo - 解放対象のワーカー情報構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • winfo は NULL であってはならない（Assert(winfo) により検証）。

アルゴリズムまたは振る舞い

1. **メッセージキューの切断**: mq_handle および error_mq_handle が存在する場合、shm_mq_detach を呼び出して切断する。 2. **ファイルクリーンアップ**: serialize_changes が有効な場合、stream_cleanup_files を呼び出し、シリアライズされた変更ファイルを削除する。 3. **DSMセグメントの解放**: dsm_seg が存在する場合、dsm_detach を呼び出して動的共有メモリセグメントを解放する。 4. **プールからの削除**: ParallelApplyWorkerPool リストから当該 winfo を削除する。 5. **メモリ解放**: pfree(winfo) を呼び出し、構造体自体のメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/applyparallelworker.c:452
  • src/backend/replication/logical/applyparallelworker.c:581
  • **呼び出し先**:
  • shm_mq_detach
  • stream_cleanup_files
  • dsm_detach
  • list_delete_ptr
  • pfree

関連データ構造

  • ParallelApplyWorkerInfo: ワーカーの状態（DSM、キュー、共有メモリ情報）を保持する構造体。
  • ParallelApplyWorkerPool: 再利用可能なワーカーを保持するリスト。

失敗モードと境界条件

  • winfo が NULL の場合、アサーションにより異常終了する。
  • 各ハンドルやセグメントが NULL の場合はスキップされるため、安全に処理される。

保守担当者向けメモ

  • 本関数はワーカーのライフサイクル管理の最終段階を担う。リソースリークを防ぐため、新しいリソース（共有メモリやファイル）を ParallelApplyWorkerInfo に追加した場合は、必ず本関数内で解放処理を行うこと。
