adjust_appendrel_attrs

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/appendinfo.c:199

目的

クエリ最適化プロセスにおいて、親リレーション（継承テーブルやパーティションテーブル）に対して定義された式ツリー内の Var ノードを、対応する子リレーションの属性へ変換するエントリポイントです。

シグネチャまたは形状

Node *adjust_appendrel_attrs(PlannerInfo *root, Node *node, int nappinfos, AppendRelInfo **appinfos)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • node: 変換対象の式ツリー（Query ノードは不可）。
  • nappinfos: appinfos 配列の要素数。
  • appinfos: 親子関係の変換情報を保持する AppendRelInfo 構造体の配列。
  • **出力**:
  • 変換済みの式ツリー（Node *）。

前提条件と不変条件

  • nappinfos >= 1 かつ appinfos が NULL でないこと。
  • 変換対象の node が Query ツリーではないこと（サブリンク変換後の式ツリーを対象とするため）。

アルゴリズムまたは振る舞い

1. adjust_appendrel_attrs_context 構造体を初期化し、root、nappinfos、appinfos を格納します。 2. adjust_appendrel_attrs_mutator を呼び出し、式ツリーを再帰的に走査します。 3. 走査中に Var ノードを検出した場合、AppendRelInfo に基づいて子リレーションの属性番号へ書き換えます。 4. RestrictInfo や PlaceHolderVar などのノードに対しては、関連する Relids を適切に更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/allpaths.c
  • src/backend/optimizer/path/equivclass.c
  • src/backend/optimizer/path/joinrels.c
  • src/backend/optimizer/plan/planner.c
  • **呼び出し先**:
  • adjust_appendrel_attrs_mutator（内部関数）

関連データ構造

  • AppendRelInfo: 親子間の列マッピング情報を保持する構造体。
  • adjust_appendrel_attrs_context: 変換処理中に状態を保持するコンテキスト構造体。
  • PlannerInfo: プランナのグローバル情報。

失敗モードと境界条件

  • 属性の型や照合順序が親と子で一致しない場合、内部で elog または ereport が発生し、処理が中断されます。
  • node が NULL の場合はそのまま NULL を返します。

保守担当者向けメモ

  • 本関数は式ツリーの変換を行うメインのエントリポイントです。Query ノードの変換はサポートされていません。
  • 変換処理は expression_tree_mutator を利用して再帰的に行われるため、新しいノードタイプを追加する際は adjust_appendrel_attrs_mutator の更新が必要です。
