UpperRelationKind

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/pathnodes.h:69

目的

クエリ最適化の最終段階（Upper Relation）において、実行される操作の種類を識別するために使用されます。

シグネチャまたは形状

typedef enum UpperRelationKind
{
    UPPERREL_SETOP,
    UPPERREL_PARTIAL_GROUP_AGG,
    UPPERREL_GROUP_AGG,
    UPPERREL_WINDOW,
    UPPERREL_PARTIAL_DISTINCT,
    UPPERREL_DISTINCT,
    UPPERREL_ORDERED,
    UPPERREL_FINAL
} UpperRelationKind;

入力と出力

  • 入力: 該当なし（列挙型定数）
  • 出力: 該当なし

前提条件と不変条件

  • UPPERREL_FINAL は列挙型の最後のエントリである必要があります。これは、この定数が関連する配列のサイズを決定するために使用されるためです。

アルゴリズムまたは振る舞い

プランナがクエリの最終的な処理ステップ（集合演算、集約、ウィンドウ関数、重複排除、ソートなど）を管理する際、各ステップをこの列挙値で分類します。これにより、特定の処理段階に対応する RelOptInfo を識別・取得することが可能になります。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/optimizer/util/relnode.c: fetch_upper_rel 関数内でリレーションの識別に使用。
  • contrib/postgres_fdw/postgres_fdw.c: 外部テーブルのパス生成ロジックで使用。
  • 呼び出し先: 該当なし（列挙型のため）

関連データ構造

  • RelOptInfo: UpperRelationKind と Relids の組み合わせによって特定される「アッパーリレーション」の情報を保持します。
  • PlannerInfo: クエリ全体の最適化状態を管理し、この列挙型を参照して各段階のパスを構築します。

失敗モードと境界条件

  • UPPERREL_FINAL を列挙型の最後以外に配置すると、配列サイズの計算を行うコードでメモリ破壊や不正なアクセスが発生する可能性があります。

保守担当者向けメモ

  • 新しい上位レベルの操作を追加する場合は、必ず UPPERREL_FINAL が最後になるように順序を維持してください。
  • fetch_upper_rel 関数がこの列挙型を使用してリレーションを検索する仕組みになっているため、変更時は関連する RelOptInfo の生成ロジックへの影響を確認してください。
