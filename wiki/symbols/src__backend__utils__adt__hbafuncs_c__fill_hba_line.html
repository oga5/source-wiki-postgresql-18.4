fill_hba_line

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/hbafuncs.c:201

目的

pg_hba.conf のパース結果（HbaLine 構造体）またはエラーメッセージを、SQLクエリ結果として出力するための Tuplestore 行へ変換・格納します。

シグネチャまたは形状

static void fill_hba_line(Tuplestorestate *tuple_store, TupleDesc tupdesc,
                          int rule_number, char *filename, int lineno, HbaLine *hba,
                          const char *err_msg)

入力と出力

  • **入力**:
  • tuple_store: 書き込み先の Tuplestorestate。
  • tupdesc: タプルの記述子。
  • rule_number: ルール番号。
  • filename: 設定ファイル名。
  • lineno: 行番号。
  • hba: パース済みの HbaLine 構造体（NULL の場合、エラー行として処理）。
  • err_msg: パースエラーが発生している場合のメッセージ文字列。
  • **出力**: なし（tuple_store への副作用としてタプルを追加）。

前提条件と不変条件

  • tupdesc->natts は NUM_PG_HBA_FILE_RULES_ATTS と一致する必要があります（Assert で検証）。
  • hba が NULL の場合、rule_number 以外のフィールドは NULL として設定されます。

アルゴリズムまたは振る舞い

1. values 配列と nulls 配列を初期化します。 2. err_msg が存在する場合、rule_number を NULL に設定し、エラーメッセージを最終カラムに格納します。 3. hba が有効な場合、以下のフィールドを順次変換して values に格納します。

  • conntype: 列挙型を文字列（"local", "host" 等）に変換。
  • databases / roles: AuthToken リストをテキスト配列に変換。
  • ip_cmp_method: IPアドレスとネットマスクを文字列化（pg_getnameinfo_all を使用）。
  • auth_method: hba_authname を使用して変換。
  • options: get_hba_options を呼び出して配列を取得。

    • 4. 最後に heap_form_tuple（直接の呼び出しは省略されているが、Tuplestore への格納処理）を経て行を確定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/hbafuncs.c 内の pg_hba_file_rules 関数から呼び出されます。
  • **呼び出し先**:
  • get_hba_options
  • hba_authname
  • pg_getnameinfo_all
  • clean_ipv6_addr
  • strlist_to_textarray

関連データ構造

  • HbaLine: 認証設定行の内部表現。
  • Tuplestorestate: SQL関数が結果を返すための一時的な行保持構造。
  • AuthToken: データベース名やロール名のトークンリスト。

失敗モードと境界条件

  • hba が NULL の場合、パース失敗とみなされ、エラーメッセージ以外のカラムが NULL に設定されます。
  • pg_getnameinfo_all が失敗した場合、IPアドレスは "???" となる可能性があります。
  • typestr や ip_cmp_method の各ケースで網羅性が欠けている場合、コンパイラの警告が発生します。

保守担当者向けメモ

  • 本関数は pg_hba_file_rules ビューのスキーマ定義と密接に結合しています。カラムの追加や変更を行う際は、NUM_PG_HBA_FILE_RULES_ATTS の定義と values/nulls 配列のインデックス操作を同期させる必要があります。
