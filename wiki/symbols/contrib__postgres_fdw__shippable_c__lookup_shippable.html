lookup_shippable

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/shippable.c:115

目的

特定のデータベースオブジェクトが、リモートサーバー上で実行可能（shippable）な拡張機能の一部であるかを判定します。

シグネチャまたは形状

static bool lookup_shippable(Oid objectId, Oid classId, PgFdwRelationInfo *fpinfo);

入力と出力

  • **入力**:
  • objectId: 判定対象オブジェクトの OID。
  • classId: オブジェクトが属するカタログの OID。
  • fpinfo: 外部テーブル情報を含む PgFdwRelationInfo 構造体。
  • **出力**:
  • bool: オブジェクトがリモート実行可能と判断された場合は true、それ以外は false。

前提条件と不変条件

  • fpinfo->shippable_extensions には、リモート実行が許可された拡張機能の OID リストが格納されている必要があります。
  • 本関数はキャッシュミス時にのみ呼び出されることを想定した、比較的コストの高い処理です。

アルゴリズムまたは振る舞い

1. getExtensionOfObject(classId, objectId) を呼び出し、対象オブジェクトが属する拡張機能の OID を取得します。 2. 取得した拡張機能 OID が有効であり、かつ fpinfo->shippable_extensions リストに含まれているかを確認します。 3. 両条件を満たす場合に true を返し、それ以外の場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/shippable.c 内のキャッシュ検索ロジックから呼び出されます。
  • **呼び出し先**: getExtensionOfObject（カタログ依存関係の解決）、list_member_oid（リスト内の存在確認）。

関連データ構造

  • PgFdwRelationInfo: 外部テーブルのメタデータと許可された拡張機能リストを保持します。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • getExtensionOfObject が無効な OID を返す場合、本関数は false を返します。
  • fpinfo が指すリストが空の場合、常に false を返します。

保守担当者向けメモ

  • 本関数はカタログ検索を伴うため、パフォーマンスへの影響を考慮し、呼び出し側で必ずキャッシュ（ShippableCacheHash）を介して利用してください。
  • 将来的に個別の関数単位での許可リスト機能が追加される場合、この判定ロジックの拡張が必要になる可能性があります。
