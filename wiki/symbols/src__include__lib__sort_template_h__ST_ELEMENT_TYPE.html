ST_ELEMENT_TYPE

定義

  • 種別: typedef
  • 定義箇所: src/include/lib/sort_template.h:188

目的

sort_template.h を用いてソート関数を生成する際、ソート対象となるデータ型を定義するために使用されます。

シグネチャまたは形状

マクロ定義により、任意の型（構造体、プリミティブ型、または void）として定義されます。

入力と出力

  • **入力**: ソート対象のデータ型。
  • **出力**: なし（型定義として機能）。

前提条件と不変条件

  • sort_template.h をインクルードする前に、利用側で ST_ELEMENT_TYPE を定義する必要があります。
  • void を指定する場合は、別途 ST_ELEMENT_TYPE_VOID の定義が必要となる場合があります。

アルゴリズムまたは振る舞い

本シンボルはコンパイル時の型指定子として機能します。ST_DECLARE および ST_DEFINE マクロが展開される際、この型に基づいてソート関数の引数や比較関数のシグネチャが生成されます。

呼び出し元と呼び出し先

  • **呼び出し元（定義側）**:
  • src/backend/storage/buffer/bufmgr.c:6442 (PendingWriteback)
  • src/backend/utils/sort/tuplesort.c:583, 593, 602, 612 (SortTuple)
  • src/include/lib/sort_template.h:138 (void)
  • **呼び出し先**:
  • 本シンボル自体は型定義であり、関数ではないため呼び出し先はありません。

関連データ構造

  • ST_COMPARATOR_TYPE_NAME: 本型を参照する比較関数ポインタ型。
  • ST_SORT: 本型を要素としてソートを行う関数。

失敗モードと境界条件

  • 型の不一致: ST_COMPARE マクロで定義された比較ロジックが、ST_ELEMENT_TYPE の構造と一致しない場合、コンパイルエラーが発生します。

保守担当者向けメモ

  • マクロによるメタプログラミングの一部であるため、デバッグ時にはプリプロセッサ展開後のコードを確認してください。
  • ST_ELEMENT_TYPE を変更する際は、必ず対応する ST_COMPARE マクロの定義も整合性を取る必要があります。
