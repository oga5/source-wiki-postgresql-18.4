pg_atomic_test_set_flag_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/generic.h:83

目的

フラグをアトミックにセットし、セット前の値が「未セット（0）」であったかどうかを判定する。

シグネチャまたは形状

static inline bool pg_atomic_test_set_flag_impl(volatile pg_atomic_flag *ptr)

入力と出力

  • **入力**: ptr - 操作対象となる pg_atomic_flag 型のポインタ。
  • **出力**: bool - フラグが以前に 0 であり、かつ今回の操作で 1 に変更できた場合に true を返す。既に 1 であった場合は false を返す。

前提条件と不変条件

  • ptr は volatile 修飾されており、アトミック操作の対象として適切に配置されている必要がある。
  • 本関数は atomics.h を経由して呼び出されることを前提としている。

アルゴリズムまたは振る舞い

内部で pg_atomic_exchange_u32_impl を呼び出し、対象のフラグに対して値 1 を書き込む。その戻り値（書き込み前の値）が 0 であるかを評価することで、フラグの獲得成否を判定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/atomics.h:185 からの呼び出しが確認されている。
  • **呼び出し先**: pg_atomic_exchange_u32_impl を内部で呼び出す。
  • **注記**: src/include/port/atomics/arch-x86.h や src/include/port/atomics/generic-gcc.h 等にも同名のシンボルが存在するが、これらは本関数の代替実装またはプラットフォーム固有の定義であり、本関数（generic.h 版）の直接の呼び出し先ではない。

関連データ構造

  • pg_atomic_flag: アトミック操作対象のフラグ構造体。

失敗モードと境界条件

  • 本関数は compare_exchange 等の基盤操作に依存しているため、基盤となるアトミック操作がハードウェアでサポートされていない環境では、ループによるエミュレーションが発生し、競合負荷が高い場合に性能低下を招く可能性がある。

保守担当者向けメモ

  • 本関数は generic.h 内のフォールバック実装である。アーキテクチャ固有の最適化が必要な場合は、arch-*.h にて同名の関数を定義し、マクロ等で本実装をオーバーライドすること。
  • インクルードガードが存在しないため、必ず atomics.h を経由して使用すること。
