AlterRole

定義

  • 種別: function
  • 定義箇所: src/backend/commands/user.c:618

目的

SQLの ALTER ROLE コマンドを実行し、データベースロールの属性（パスワード、接続制限、権限フラグ等）を変更する。

シグネチャまたは形状

Oid AlterRole(ParseState *pstate, AlterRoleStmt *stmt)

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する ParseState 構造体。
  • stmt: ALTER ROLE 文の構文解析結果を保持する AlterRoleStmt 構造体。
  • **出力**:
  • 変更対象となったロールの Oid を返す。

前提条件と不変条件

  • 予約済みロール（pg_ で始まるロール等）の変更は禁止されている。
  • ロール変更には、呼び出し元が CREATEROLE 権限を持ち、かつ対象ロールに対する ADMIN OPTION を保持している必要がある。
  • スーパーユーザー属性の変更や、スーパーユーザー自身に対する操作には、呼び出し元もスーパーユーザーである必要がある。

アルゴリズムまたは振る舞い

1. stmt->options を走査し、パスワード、接続制限、各種権限フラグ（SUPERUSER, INHERIT, CREATEDB 等）を抽出する。 2. pg_authid カタログを RowExclusiveLock で開き、対象ロールの存在を確認する。 3. 呼び出し元の権限を検証し、操作が許可されているか判定する。 4. 抽出されたオプションに基づき、pg_authid のタプルを更新する（ソースコード抜粋範囲外の処理を含む）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c 内のユーティリティ実行処理から呼び出される。
  • **呼び出し先**:
  • check_rolespec_name, get_rolespec_tuple, table_open, superuser, have_createrole_privilege, is_admin_of_role 等の内部関数およびカタログ操作関数。
  • check_password_hook（フックが設定されている場合）。

関連データ構造

  • AlterRoleStmt: ALTER ROLE 文の構文解析ツリー。
  • Form_pg_authid: pg_authid カタログの行構造。
  • GrantRoleOptions: ロールメンバーシップ関連のオプションを保持する構造体。

失敗モードと境界条件

  • **権限不足**: CREATEROLE 権限や ADMIN OPTION が不足している場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラーとなる。
  • **不正な値**: 接続制限（connectionlimit）に -1 未満の値を指定した場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーとなる。
  • **競合**: 同一のオプションを複数回指定した場合、errorConflictingDefElem によりエラーとなる。
  • **未定義オプション**: 認識できないオプションが指定された場合、elog(ERROR) を発生させる。

保守担当者向けメモ

  • 本関数は pg_authid を直接操作するため、カタログのスキーマ変更時には new_record 等の配列定義と整合性を取る必要がある。
  • セキュリティに関わる重要な関数であるため、権限チェックロジックの変更には細心の注意を払うこと。
  • パスワード変更処理は check_password_hook を介して外部モジュールから介入可能である点に留意すること。
