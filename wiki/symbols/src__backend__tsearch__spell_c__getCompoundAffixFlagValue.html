getCompoundAffixFlagValue

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/spell.c:1133

目的

文字列として与えられた複合語接辞フラグを解析し、対応するビットフラグ値の論理和を算出する。

シグネチャまたは形状

static int getCompoundAffixFlagValue(IspellDict *Conf, const char *s)

入力と出力

  • **入力**:
  • Conf: 辞書設定を保持する IspellDict 構造体へのポインタ。
  • s: 解析対象の接辞フラグ文字列。
  • **出力**:
  • int: 解析されたフラグのビットマスク値。

前提条件と不変条件

  • Conf->nCompoundAffixFlag が 0 の場合、常に 0 を返す。
  • Conf->CompoundAffixFlags は cmpcmdflag 関数を用いて二分探索可能な状態である必要がある。

アルゴリズムまたは振る舞い

1. Conf->nCompoundAffixFlag が 0 であれば即座に 0 を返す。 2. 入力文字列 s を getNextFlagFromString を用いて個別のフラグ文字列に分割する。 3. 各フラグ文字列に対し setCompoundAffixFlagValue を呼び出し、検索用キーを作成する。 4. bsearch を使用して Conf->CompoundAffixFlags 配列から該当するフラグ情報を検索する。 5. 見つかったフラグの value を結果のビットマスク（flag）に累積（論理和）する。 6. 全てのフラグを処理した後、最終的なビットマスクを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/spell.c:1397
  • src/backend/tsearch/spell.c:1638
  • **呼び出し先**:
  • getNextFlagFromString
  • setCompoundAffixFlagValue
  • bsearch (標準ライブラリ)
  • cmpcmdflag (比較関数)

関連データ構造

  • IspellDict: 辞書全体の構成と複合語フラグの配列を保持。
  • CompoundAffixFlag: 複合語フラグの定義と対応する値を保持する構造体。

失敗モードと境界条件

  • s が空文字列の場合、ループは実行されず 0 が返される。
  • bsearch でフラグが見つからない場合、そのフラグは無視され、エラーにはならず残りのフラグの処理が継続される。

保守担当者向けメモ

  • 本関数は bsearch を使用しているため、Conf->CompoundAffixFlags が事前にソートされていることが前提となる。
  • BUFSIZ を使用してフラグ文字列を一時保持しているため、フラグ文字列が BUFSIZ を超える場合は考慮が必要。
