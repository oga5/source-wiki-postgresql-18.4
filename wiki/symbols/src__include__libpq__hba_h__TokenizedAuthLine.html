TokenizedAuthLine

定義

  • 種別: struct
  • 定義箇所: src/include/libpq/hba.h:163

目的

pg_hba.conf や pg_ident.conf の各行を読み込み、解析可能な形式（トークン化された状態）で保持するためのデータ構造です。

シグネチャまたは形状

typedef struct TokenizedAuthLine
{
    List       *fields;         /* List of lists of AuthTokens */
    char       *file_name;      /* File name of origin */
    int         line_num;       /* Line number */
    char       *raw_line;       /* Raw line text */
    char       *err_msg;        /* Error message if any */
} TokenizedAuthLine;

入力と出力

  • **入力**: 設定ファイルの読み込み処理において、各行のテキストデータが本構造体の各フィールドに格納されます。
  • **出力**: parse_hba_line() や parse_ident_line() に渡され、最終的に HbaLine や IdentLine 構造体へ変換されるための入力データとなります。

前提条件と不変条件

  • fields は AuthToken のリストのリストとして構成されます。
  • err_msg が非 NULL の場合、その行の解析においてエラーが発生したことを示します。

アルゴリズムまたは振る舞い

1. tokenize_auth_file() によって設定ファイルが読み込まれ、本構造体のリストが生成されます。 2. 各 TokenizedAuthLine は、元のファイル名、行番号、および生のテキスト行を保持し、エラー発生時には err_msg に詳細が記録されます。 3. 後続のパース処理（parse_hba_line 等）がこの構造体を参照し、認証ルールとしての妥当性を検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/hba.c 内のファイル読み込みおよび解析ロジック（tokenize_auth_file 等）。
  • **呼び出し先**: parse_hba_line(TokenizedAuthLine *tok_line, int elevel) および parse_ident_line(TokenizedAuthLine *tok_line, int elevel) が本構造体を引数として受け取ります。

関連データ構造

  • AuthToken: fields リストを構成する個々のトークン要素。
  • HbaLine: 本構造体を解析して生成される、認証ルールを保持する構造体。
  • IdentLine: 本構造体を解析して生成される、ユーザーマッピングを保持する構造体。
  • List: nodes/pg_list.h で定義される PostgreSQL 標準のリスト構造。

失敗モードと境界条件

  • err_msg フィールドが設定されている場合、解析プロセスは当該行を無効なルールとして扱うか、エラーログを出力します。
  • メモリ不足時には palloc0 による割り当てが失敗する可能性があります。

保守担当者向けメモ

  • TokenizedAuthLine は設定ファイルのパースにおける中間表現です。構造を変更する場合は、hba.c 内のパース関数群との整合性に注意してください。
  • fields リストの構造（List of lists of AuthTokens）は、設定ファイルの行内のトークン構成を反映しています。
