rewriteSearchAndCycle

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteSearchCycle.c:202

目的

再帰的共通テーブル式（CTE）における SEARCH および CYCLE 句を、標準的な UNION クエリ構造へと書き換えます。

シグネチャまたは形状

CommonTableExpr *rewriteSearchAndCycle(CommonTableExpr *cte)

入力と出力

  • **入力**: CommonTableExpr *cte（search_clause または cycle_clause を含む CTE ノード）
  • **出力**: 書き換えられた CommonTableExpr 構造体へのポインタ。

前提条件と不変条件

  • cte->search_clause または cte->cycle_clause のいずれかが存在すること。
  • CTE のクエリ構造が UNION（SETOP_UNION）であること。
  • UNION の左右のサブクエリが RTE_SUBQUERY であること。

アルゴリズムまたは振る舞い

1. **初期化**: 入力 CTE をコピーし、UNION の左右のサブクエリ（rte1, rte2）を特定します。 2. **列の拡張**: SEARCH および CYCLE 句で指定された列を CTE の出力列リストに追加するための属性番号を算出します。 3. **左側サブクエリ（初期クエリ）の再構築**:

  • 元のサブクエリをコピーし、IncrementVarSublevelsUp を適用します。
  • SEARCH 句がある場合、深さ（*DEPTH*）や初期配列をターゲットリストに追加します。
  • CYCLE 句がある場合、デフォルトのマーク値と初期パス配列をターゲットリストに追加します。
  • rte1->subquery を書き換え後のクエリで更新します。

    • 4. **右側サブクエリ（再帰クエリ）の準備**:

  • 新しい RangeTblEntry を作成し、拡張された列名リスト（ewcl）を準備します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/rewrite/rewriteHandler.c:2044
  • **呼び出し先**:
  • copyObject, castNode, rt_fetch, makeNode, makeAlias, IncrementVarSublevelsUp, makeFromExpr, makeVar, makeTargetEntry, make_path_rowexpr, make_path_initial_array, lappend, lcons 等。

関連データ構造

  • CommonTableExpr: 書き換え対象の CTE ノード。
  • SetOperationStmt: UNION 構造を保持。
  • RangeTblEntry: サブクエリの参照。
  • RowExpr: パス記録用の行式。

失敗モードと境界条件

  • 再帰参照が UNION の右側サブクエリのトップレベルに存在しない場合、現在の実装ではエラーとなります。
  • search_clause と cycle_clause の両方が存在する場合、属性番号の計算において両方のオフセットが加算されます。

保守担当者向けメモ

  • 本関数は rewriteHandler.c から呼び出される書き換えパイプラインの一部です。
  • SEARCH 句の BREADTH FIRST では RECORDOID、DEPTH FIRST では RECORDARRAYOID が使用されます。
  • 現在の実装は UNION の右側サブクエリの構造に依存しており、将来的な拡張ポイントとして再帰参照の配置制限の緩和が挙げられます。
