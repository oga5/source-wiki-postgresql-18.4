pgp_armor_decode

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-armor.c:313

目的

PGP ASCII アーマー形式のテキストデータをデコードし、バイナリデータへ復元します。

シグネチャまたは形状

int pgp_armor_decode(const uint8 *src, int len, StringInfo dst);

入力と出力

  • **入力**:
  • src: デコード対象のアーマー形式文字列。
  • len: src の長さ。
  • dst: デコード結果を格納する StringInfo バッファ。
  • **出力**:
  • 成功時: デコードされたバイナリデータのバイト数（正の整数）。
  • 失敗時: PXE_PGP_CORRUPT_ARMOR などのエラーコード（負の整数）。

前提条件と不変条件

  • dst は enlargeStringInfo によって必要なサイズが確保されます。
  • デコードされたデータは、末尾に付与された CRC24 チェックサムと一致する必要があります。

アルゴリズムまたは振る舞い

1. find_header を使用してアーマーの開始および終了境界を特定します。 2. ヘッダーとデータ本体の間の空行をスキップします。 3. 末尾の = 以降にある 4 バイトの Base64 文字列をデコードし、CRC24 チェックサム値を取得します。 4. pg_base64_decode を使用して本体をデコードし、dst に格納します。 5. crc24 関数を用いてデコード済みデータのチェックサムを計算し、抽出した値と比較します。 6. 一致した場合のみ dst->len を更新し、成功を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-pgsql.c:893
  • **呼び出し先**:
  • find_header
  • memchr
  • pg_base64_decode
  • pg_base64_dec_len
  • enlargeStringInfo
  • crc24
  • elog

関連データ構造

  • StringInfo: PostgreSQL の可変長文字列バッファ。

失敗モードと境界条件

  • アーマーのヘッダーまたはフッターが見つからない場合、PXE_PGP_CORRUPT_ARMOR を返します。
  • CRC24 チェックサムが一致しない場合、PXE_PGP_CORRUPT_ARMOR を返します。
  • Base64 デコードが失敗した場合、エラーを返します。
  • デコード後のデータサイズがバッファサイズを超過する場合、FATAL エラーで停止します。

保守担当者向けメモ

  • 本関数は contrib/pgcrypto 内の独自 Base64 実装に依存しています。
  • pgp_extract_armor_headers とは異なり、本関数はデータ本体の復元と整合性検証に特化しています。
