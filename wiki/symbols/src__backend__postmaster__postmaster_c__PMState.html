PMState

定義

  • 種別: enum
  • 定義箇所: src/backend/postmaster/postmaster.c:335

目的

PostgreSQL のメインプロセスである Postmaster の現在の動作状態を管理・追跡するために使用されます。

シグネチャまたは形状

typedef enum {
    PM_INIT,
    PM_STARTUP,
    PM_RECOVERY,
    PM_HOT_STANDBY,
    PM_RUN,
    PM_STOP_BACKENDS,
    PM_WAIT_BACKENDS,
    PM_WAIT_XLOG_SHUTDOWN,
    PM_WAIT_XLOG_ARCHIVAL,
    PM_WAIT_IO_WORKERS,
    PM_WAIT_CHECKPOINTER,
    PM_WAIT_DEAD_END,
    PM_NO_CHILDREN
} PMState;

入力と出力

  • 本シンボル自体は状態値の定義です。
  • 状態の更新には UpdatePMState(PMState newState) 関数が使用され、現在の状態から新しい状態への遷移が記録されます。

前提条件と不変条件

  • Postmaster のライフサイクルにおいて、サーバーの状態は常にこの列挙型のいずれか一つに該当します。
  • 初期状態は PM_INIT です。

アルゴリズムまたは振る舞い

  • Postmaster は pmState の値に基づいて、次に起動すべきプロセスや実行すべきシャットダウン手順を決定します。
  • 状態遷移が発生するたびに UpdatePMState が呼ばれ、デバッグレベルが DEBUG1 の場合に遷移内容がログ出力されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/postmaster.c 内の各プロセス管理ロジック。
  • 呼び出し先: pmstate_name(PMState state) 関数が、列挙値を文字列に変換するために本シンボルを参照します。

関連データ構造

  • pmState: src/backend/postmaster/postmaster.c 内で定義される静的変数。現在の状態を保持します。

失敗モードと境界条件

  • 状態遷移の整合性は PostmasterStateMachine によって制御されます。不正な遷移が発生した場合の挙動については、本ソース抜粋からは判断できません。

保守担当者向けメモ

  • サーバーの挙動を追跡する際は、log_min_messages = debug1 を設定することで UpdatePMState による状態遷移ログを確認できます。
  • PM_RUN は通常の稼働状態を示し、それ以降の PM_WAIT_* 系はシャットダウンシーケンスにおける各サブプロセスの終了待機状態を表します。
