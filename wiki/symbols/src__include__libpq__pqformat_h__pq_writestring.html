pq_writestring

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/libpq/pqformat.h:107

目的

文字列をサーバーからクライアントのエンコーディングへ変換し、通信バッファ（StringInfoData）へヌル終端文字列として書き込みます。

シグネチャまたは形状

static inline void pq_writestring(StringInfoData *pg_restrict buf, const char *pg_restrict str)

入力と出力

  • **入力**:
  • buf: 書き込み先となる StringInfoData 構造体へのポインタ。
  • str: 書き込む対象のヌル終端文字列。
  • **出力**: なし（buf の内容が更新されます）。

前提条件と不変条件

  • buf が指す StringInfoData は、書き込む文字列の長さ（ヌル終端文字を含む）を格納するのに十分な容量（maxlen）を事前に確保している必要があります。
  • 実行時に Assert(buf->len + slen + 1 <= buf->maxlen) により、バッファオーバーフローが発生しないことが検証されます。

アルゴリズムまたは振る舞い

1. 入力文字列 str の長さを取得します。 2. pg_server_to_client を呼び出し、必要に応じてサーバーエンコーディングからクライアントエンコーディングへ変換します。 3. 変換が発生した場合、変換後の文字列の長さを再計算します。 4. memcpy を使用して、変換後の文字列（およびヌル終端文字）を buf->data の現在の末尾位置にコピーします。 5. buf->len を文字列の長さ分だけ進めます。 6. 変換のためにメモリが動的に割り当てられた場合（p != str）、pfree(p) を呼び出してメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/printtup.c:234
  • **呼び出し先**:
  • strlen
  • pg_server_to_client
  • memcpy
  • pfree

関連データ構造

  • StringInfoData: 通信バッファを管理する構造体（lib/stringinfo.h で定義）。

失敗モードと境界条件

  • バッファの容量不足が発生した場合、Assert が有効なビルドではプログラムが停止します。
  • pg_server_to_client が変換のためにメモリを割り当てた場合、関数終了時に必ず pfree が実行される設計となっています。

保守担当者向けメモ

  • 本関数は static inline であり、呼び出し前にバッファの容量が確保されていることを前提としています。動的なバッファ拡張が必要な場合は pq_sendstring を検討してください。
  • pg_restrict キーワードにより、ポインタのエイリアスが発生しないことがコンパイラに示唆されています。
