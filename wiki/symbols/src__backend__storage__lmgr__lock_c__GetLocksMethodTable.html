GetLocksMethodTable

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lock.c:525

目的

指定された LOCK 構造体に関連付けられたロックメソッドテーブル（LockMethod）を取得します。

シグネチャまたは形状

LockMethod GetLocksMethodTable(const LOCK *lock)

入力と出力

  • **入力**: lock (対象の LOCK 構造体へのポインタ)
  • **出力**: LockMethod (ロックの振る舞いを定義するメソッドテーブル)

前提条件と不変条件

  • lock ポインタは有効である必要があります。
  • LOCK_LOCKMETHOD(*lock) によって抽出された lockmethodid が、有効な LockMethods 配列の範囲内（0 < id < lengthof(LockMethods)）にあることを保証するアサーションが含まれています。

アルゴリズムまたは振る舞い

1. 引数 lock から LOCK_LOCKMETHOD マクロを用いて lockmethodid を抽出します。 2. lockmethodid が有効範囲内であることをアサートします。 3. LockMethods 配列から該当するインデックスの LockMethod を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/deadlock.c: ProcLockWakeup 呼び出し時および lockMethodTable の取得時に使用。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありませんが、LOCK_LOCKMETHOD マクロおよび LockMethods 配列に依存しています。

関連データ構造

  • LOCK: ロック対象のオブジェクト状態を保持する構造体。
  • LockMethod: ロックの競合ルールや振る舞いを定義する構造体。
  • LockMethods: ロックメソッドの定義を保持するグローバル配列。

失敗モードと境界条件

  • lockmethodid が範囲外の場合、アサーションエラーによりプロセスが異常終了します。

保守担当者向けメモ

  • 本関数はロックマネージャの内部的なルックアップ処理です。LockMethods 配列の構成に変更を加える際は、本関数のアサーション条件が維持されているか確認してください。
