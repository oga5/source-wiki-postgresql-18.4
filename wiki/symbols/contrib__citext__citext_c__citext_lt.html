citext_lt

定義

  • 種別: function
  • 定義箇所: contrib/citext/citext.c:248

目的

2つの citext 型の値を受け取り、左辺が右辺より小さいかどうか（"less than"）を判定する SQL 呼び出し可能な関数です。

シグネチャまたは形状

Datum citext_lt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの text 型（citext として扱われる）引数を受け取ります。
  • **出力**: PG_RETURN_BOOL を介して bool 型の値を返します。

前提条件と不変条件

  • 入力引数は PG_GETARG_TEXT_PP を使用して取得されるため、TOAST されたデータは必要に応じて展開されます。
  • 比較には現在の照合順序（PG_GET_COLLATION()）が使用されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を用いて2つの入力引数を取得します。 2. citextcmp 関数を呼び出し、現在の照合順序に基づいて比較を行います。 3. citextcmp の戻り値が 0 未満である場合に true を、それ以外の場合に false を result に格納します。 4. PG_FREE_IF_COPY を使用して、必要に応じて入力引数のメモリを解放します。 5. PG_RETURN_BOOL を使用して判定結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(citext_lt) マクロにより、PostgreSQL の関数マネージャから呼び出されることが示唆されています。
  • **呼び出し先**:
  • citextcmp（比較ロジックの実行）
  • PG_GETARG_TEXT_PP / PG_FREE_IF_COPY / PG_RETURN_BOOL（PostgreSQL 関数インターフェース）

関連データ構造

  • text: PostgreSQL の可変長文字列型。citext はこの型をベースに実装されています。

失敗モードと境界条件

  • 入力値が NULL の場合、PostgreSQL の関数呼び出し規約に従い、呼び出し元で NULL 処理が行われることが想定されます。
  • メモリ不足時には palloc 関連の例外が発生する可能性があります。

保守担当者向けメモ

  • 本関数は citext 型の < 演算子を実装しています。
  • 比較ロジックの変更が必要な場合は、内部で使用している citextcmp を確認してください。
