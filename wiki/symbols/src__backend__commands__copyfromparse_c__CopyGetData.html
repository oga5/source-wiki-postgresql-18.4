CopyGetData

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfromparse.c:244

目的

COPY FROM コマンド実行時、指定されたデータソース（ファイル、フロントエンド、またはコールバック）からバッファへデータを読み込みます。

シグネチャまたは形状

static int CopyGetData(CopyFromState cstate, void *databuf, int minread, int maxread)

入力と出力

  • **入力**:
  • cstate: CopyFromState 構造体（ソース種別や状態を保持）。
  • databuf: 読み込んだデータを格納するバッファへのポインタ。
  • minread: 読み込むべき最小バイト数。
  • maxread: 読み込み可能な最大バイト数。
  • **出力**:
  • 実際に読み込まれたバイト数を int 型で返します。

前提条件と不変条件

  • cstate->copy_src に基づいて適切な読み込み処理が選択されます。
  • フロントエンドからの読み込み時、cstate->fe_msgbuf が初期化されている必要があります。
  • 読み込み中にエラーが発生した場合、ereport(ERROR, ...) により処理が中断されます。

アルゴリズムまたは振る舞い

1. **ソース種別による分岐**:

  • COPY_FILE: fread を使用してファイルから直接読み込みます。読み込みバイト数が0の場合、cstate->raw_reached_eof を true に設定します。
  • COPY_FRONTEND: フロントエンドからのメッセージをループで受信します。PqMsg_CopyData メッセージをバッファへコピーし、PqMsg_CopyDone を受信すると EOF 状態へ移行します。Flush や Sync メッセージは無視されます。
  • COPY_CALLBACK: 登録された data_source_cb コールバック関数を呼び出してデータを取得します。

    • 2. **エラーハンドリング**: ファイル読み込み時の ferror や、フロントエンド通信時の予期せぬ EOF、不正なメッセージタイプに対してエラーを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfromparse.c 内の CopyLoadRawBuf および CopyReadBinaryData から呼び出されています。
  • **呼び出し先**:
  • fread (標準ライブラリ)
  • pq_startmsgread, pq_getbyte, pq_getmessage, pq_getmsgstring, pq_copymsgbytes (フロントエンド通信用 API)
  • cstate->data_source_cb (コールバック関数)

関連データ構造

  • CopyFromState: COPY FROM の実行状態を管理する構造体。
  • StringInfo (内部的に fe_msgbuf として使用): フロントエンドからのメッセージバッファ。

失敗モードと境界条件

  • **ファイルアクセスエラー**: fread 失敗時に ereport でエラー終了します。
  • **プロトコル違反**: フロントエンドから期待されないメッセージタイプを受信した場合にエラー終了します。
  • **接続切断**: トランザクション中に予期せぬ EOF を受信した場合にエラー終了します。
  • **キャンセル**: フロントエンド処理中に HOLD_CANCEL_INTERRUPTS / RESUME_CANCEL_INTERRUPTS を使用して割り込みを制御しています。

保守担当者向けメモ

  • COPY_FRONTEND モードにおける readmessage ラベルへの goto は、Flush や Sync メッセージをスキップして次のメッセージを待機するための制御フローです。
  • minread と maxread の指定により、バッファの充填量を細かく制御する設計となっています。
