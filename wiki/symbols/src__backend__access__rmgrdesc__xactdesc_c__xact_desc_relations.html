xact_desc_relations

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/xactdesc.c:281

目的

トランザクションに関連するリレーション（RelFileLocator）のリストを、人間が読める形式の文字列に変換してバッファへ追加する。

シグネチャまたは形状

static void xact_desc_relations(StringInfo buf, char *label, int nrels, RelFileLocator *xlocators)

入力と出力

  • buf (StringInfo): 出力先の文字列バッファ。
  • label (char *): リレーションリストの前に付与するラベル文字列。
  • nrels (int): リレーションの数。
  • xlocators (RelFileLocator *): リレーションを識別するロケータの配列。
  • 出力: なし（buf を直接更新する）。

前提条件と不変条件

  • nrels が 0 より大きい場合のみ、buf にラベルとリレーションパスが追記される。
  • xlocators は少なくとも nrels 個の要素を持つ有効な配列である必要がある。

アルゴリズムまたは振る舞い

1. nrels > 0 の場合、buf に ; と label、およびコロン（:）を書き込む。 2. nrels 回ループし、各 xlocators[i] に対して relpathperm 関数を呼び出し、メインフォーク（MAIN_FORKNUM）のパス文字列を取得する。 3. 取得したパスを buf に順次追記する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/rmgrdesc/xactdesc.c 内の各関数（xact_desc から呼び出される解析ロジック内）から、コミット、アボート、プリペア等のレコード処理時に呼び出される。
  • 呼び出し先:
  • appendStringInfo (PostgreSQL 内部ユーティリティ)
  • relpathperm (リレーションパス生成関数)

関連データ構造

  • StringInfo: 文字列構築用バッファ。
  • RelFileLocator: リレーションを特定するための識別子。

失敗モードと境界条件

  • nrels が 0 の場合、何も出力せず正常終了する。
  • relpathperm が返すパス文字列の長さが StringInfo の許容範囲を超える場合、バッファ拡張が発生する。

保守担当者向けメモ

  • 本関数は pg_waldump 等のログ出力で共通利用されるため、出力フォーマットを変更する際は影響範囲に注意すること。
  • MAIN_FORKNUM を固定で使用しているため、他のフォーク（FSMやVMなど）を対象とする場合は修正が必要となる。
