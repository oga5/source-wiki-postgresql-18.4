blgetbitmap

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blscan.c:74

目的

Bloom インデックス全体を走査し、指定された検索条件に合致するタプルの TID をビットマップへ追加する。

シグネチャまたは形状

int64 blgetbitmap(IndexScanDesc scan, TIDBitmap *tbm)

入力と出力

  • **入力**:
  • scan: インデックススキャン記述子。
  • tbm: 検索結果を格納するビットマップ。
  • **出力**:
  • int64: ビットマップに追加されたタプルの総数。

前提条件と不変条件

  • scan->opaque は BloomScanOpaque 型としてキャスト可能であること。
  • 初回実行時に検索シグネチャが計算され、so->sign にキャッシュされる。
  • 検索キーに NULL が含まれる場合、検索結果は 0 となる。
  • ページ走査中、CHECK_FOR_INTERRUPTS() により割り込みが許可される。

アルゴリズムまたは振る舞い

1. **シグネチャ生成**: so->sign が未初期化の場合、scan->keyData から Bloom フィルタの検索シグネチャを計算する。 2. **スキャン準備**: BAS_BULKREAD 戦略でバッファアクセス戦略を初期化し、インデックスの全ページを走査対象とする。 3. **ページ走査**: BLOOM_HEAD_BLKNO から末尾まで順次ページを読み込む。 4. **フィルタリング**: ページ内の各タプルに対し、タプルのシグネチャと検索シグネチャのビット単位 AND 演算を行い、包含関係を確認する。 5. **結果格納**: 条件に合致したタプルの heapPtr を tbm_add_tuples でビットマップに登録する。 6. **終了処理**: バッファアクセス戦略を解放し、カウントした TID 数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blutils.c: amroutine->amgetbitmap に設定され、PostgreSQL のインデックスアクセスメソッド経由で呼び出される。
  • **呼び出し先**:
  • signValue (blutils.c): 検索シグネチャの計算。
  • ReadBufferExtended, LockBuffer, UnlockReleaseBuffer: バッファ管理。
  • tbm_add_tuples: ビットマップへの結果追加。
  • CHECK_FOR_INTERRUPTS: 割り込みチェック。

関連データ構造

  • BloomScanOpaque: スキャン状態（検索シグネチャ sign を含む）を保持。
  • BloomTuple: インデックス内のタプル構造。
  • TIDBitmap: 検索結果を格納するビットマップ構造。

失敗モードと境界条件

  • 検索キーに SK_ISNULL フラグがある場合、即座に 0 を返す。
  • ページが新規作成直後（PageIsNew）または削除済み（BloomPageIsDeleted）の場合、そのページはスキップされる。

保守担当者向けメモ

  • 本関数は全ページ走査を行うため、インデックスサイズが大きい場合にコストが高くなる。
  • blrescan が呼ばれると so->sign が破棄されるため、次回の blgetbitmap 実行時に再計算が行われる。
  • BAS_BULKREAD を使用しているため、大規模なスキャンに適したバッファ管理が行われる。
