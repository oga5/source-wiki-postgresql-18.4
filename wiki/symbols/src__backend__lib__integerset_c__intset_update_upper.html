intset_update_upper

定義

  • 種別: function
  • 定義箇所: src/backend/lib/integerset.c:479

目的

B-tree 構造の整数集合において、ノード分割が発生した際に親ノードを更新し、必要に応じて木の高さを拡張する。

シグネチャまたは形状

static void intset_update_upper(IntegerSet *intset, int level, intset_node *child, uint64 child_key)

入力と出力

  • intset: 操作対象の IntegerSet 構造体。
  • level: 更新対象のノードレベル（0 はリーフ）。
  • child: 分割された、または新しく作成された子ノード。
  • child_key: 子ノードの最初の値（ダウンリンクキーとして使用）。

前提条件と不変条件

  • level > 0 であること。
  • IntegerSet の rightmost_nodes 配列が、各レベルの最新のノードを正しく指していること。
  • 木の高さが MAX_TREE_LEVELS を超えないこと。

アルゴリズムまたは振る舞い

1. **高さの拡張**: 現在の level が intset->num_levels 以上の場合、新しいルートノードを作成し、木全体の高さを 1 増加させる。 2. **親ノードへの挿入**: rightmost_nodes[level] が指す親ノードに空きがある場合、child_key と child へのポインタを追記する。 3. **再帰的分割**: 親ノードが満杯の場合、新しい内部ノードを作成し、child_key を格納した上で、intset_update_upper を level + 1 に対して再帰的に呼び出し、親の親（祖父母）を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/lib/integerset.c:455
  • src/backend/lib/integerset.c:545（再帰呼び出し）
  • **呼び出し先**:
  • intset_new_internal_node（ノード分割時）
  • intset_update_upper（再帰呼び出し）

関連データ構造

  • IntegerSet: 集合全体を管理するルート構造体。
  • intset_node: 汎用ノード型。
  • intset_internal_node: 内部ノード構造体。
  • intset_leaf_node: リーフノード構造体。

失敗モードと境界条件

  • **最大レベル超過**: intset->num_levels == MAX_TREE_LEVELS の場合、elog(ERROR) により処理を中断する。

保守担当者向けメモ

  • 本関数は IntegerSet の B-tree 構造を維持するための再帰的な更新処理です。
  • rightmost_nodes 配列の整合性が、挿入処理の正確性に直結します。
  • MAX_TREE_LEVELS に達した場合の挙動は致命的なエラーとして扱われます。
