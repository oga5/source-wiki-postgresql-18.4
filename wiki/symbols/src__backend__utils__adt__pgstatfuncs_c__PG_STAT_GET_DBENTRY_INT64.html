PG_STAT_GET_DBENTRY_INT64

定義

  • 種別: macro
  • 定義箇所: src/backend/utils/adt/pgstatfuncs.c:1044

目的

データベース統計情報（PgStat_StatDBEntry）から、特定の64ビット整数型の統計項目を抽出し、SQLの Datum 型として返却するためのテンプレートマクロです。

シグネチャまたは形状

PG_STAT_GET_DBENTRY_INT64(stat)
  • stat: 取得対象となる PgStat_StatDBEntry 構造体のフィールド名。

入力と出力

  • **入力**: SQL関数呼び出しの引数として Oid（データベースID）を受け取ります。
  • **出力**: 指定された統計項目の値を int64 型の Datum として返します。対象のデータベース統計情報が存在しない場合は NULL を返します。

前提条件と不変条件

  • 呼び出し元は PG_FUNCTION_ARGS を受け取るSQL関数である必要があります。
  • 内部で pgstat_fetch_stat_dbentry(dbid) を呼び出し、最新のスナップショットを取得します。
  • 統計情報の取得には、対象データベースに対する適切な権限が必要です。

アルゴリズムまたは振る舞い

1. PG_GETARG_OID(0) により、対象データベースのOIDを取得します。 2. pgstat_fetch_stat_dbentry を使用して、共有メモリまたはキャッシュから当該データベースの統計エントリを取得します。 3. エントリが存在しない場合は PG_RETURN_NULL() を実行します。 4. エントリが存在する場合、マクロ引数 stat で指定されたフィールド値を Int64GetDatum() を介して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_stat_get_db_blocks_hit, pg_stat_get_db_deadlocks, pg_stat_get_db_xact_commit 等の統計取得用SQL関数。
  • **呼び出し先**: pgstat_fetch_stat_dbentry（統計情報の取得）、Int64GetDatum（型変換）。

関連データ構造

  • PgStat_StatDBEntry: データベースごとの統計情報を保持する構造体。

失敗モードと境界条件

  • 指定された dbid に対応する統計情報が存在しない場合、関数は NULL を返します。
  • 統計情報システムが初期化されていない、またはアクセス権限がない場合、期待される値が取得できない可能性があります。

保守担当者向けメモ

  • 本マクロは統計項目追加の定型処理を簡素化するために使用されます。新しい統計項目を追加する際は、PgStat_StatDBEntry 構造体にフィールドを追加した上で、本マクロを用いて関数を定義してください。
  • 統計情報はスナップショットベースであるため、トランザクション内での一貫性は保証されますが、関数呼び出しのタイミングにより値が変動する点に留意してください。
