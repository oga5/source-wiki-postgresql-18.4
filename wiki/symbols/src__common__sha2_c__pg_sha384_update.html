pg_sha384_update

定義

  • 種別: function
  • 定義箇所: src/common/sha2.c:943

目的

SHA-384 ハッシュ計算において、入力データブロックを処理し、内部コンテキストを更新します。

シグネチャまたは形状

void pg_sha384_update(pg_sha384_ctx *context, const uint8 *data, size_t len)

入力と出力

  • context: 更新対象の pg_sha384_ctx 構造体へのポインタ。
  • data: ハッシュ計算対象となる入力データのバイト列。
  • len: 入力データの長さ（バイト単位）。
  • 戻り値: なし。

前提条件と不変条件

  • context は事前に pg_sha384_init によって初期化されている必要があります。
  • SHA-384 は SHA-512 のラッパーとして実装されているため、内部的に pg_sha512_ctx へのキャストが行われます。

アルゴリズムまたは振る舞い

内部的に pg_sha512_update を呼び出し、SHA-512 のアルゴリズムを用いてコンテキストを更新します。入力データはブロック単位でバッファリングされ、ブロック長に達するたびに変換処理が実行されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/cryptohash.c:156
  • 呼び出し先: pg_sha512_update

関連データ構造

  • pg_sha384_ctx: SHA-384 の状態を保持するコンテキスト構造体（sha2_int.h で定義）。

失敗モードと境界条件

  • 入力データが NULL または長さが 0 の場合の挙動については、pg_sha512_update の実装に依存します。

保守担当者向けメモ

  • 本関数は SHA-512 実装のラッパーです。SHA-384 固有のロジック変更が必要な場合は、pg_sha512_update への影響を考慮してください。
