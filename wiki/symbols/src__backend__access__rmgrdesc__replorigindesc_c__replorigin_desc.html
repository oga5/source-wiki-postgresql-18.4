replorigin_desc

定義

  • src/backend/access/rmgrdesc/replorigindesc.c:18

目的

論理レプリケーションにおけるレプリケーションオリジンのWALレコード（SETおよびDROP操作）を解析し、人間が読める形式の文字列としてバッファに出力する。

シグネチャまたは形状

void replorigin_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: ログ情報を追記するための StringInfo バッファ。
  • record: 解析対象のWALレコードを保持する XLogReaderState 構造体。
  • **出力**: なし（buf を通じて副作用として文字列が追加される）。

前提条件と不変条件

  • record は有効な XLogReaderState を指している必要がある。
  • XLogRecGetInfo(record) から取得される情報ビットは、XLR_INFO_MASK を除外した状態で XLOG_REPLORIGIN_SET または XLOG_REPLORIGIN_DROP のいずれかに合致する必要がある。

アルゴリズムまたは振る舞い

1. XLogRecGetData を用いてレコードのデータポインタを取得する。 2. XLogRecGetInfo に ~XLR_INFO_MASK を適用し、レコードのサブタイプを特定する。 3. switch 文により以下の処理を行う：

  • XLOG_REPLORIGIN_SET: xl_replorigin_set 構造体にキャストし、ノードID、リモートLSN、および強制フラグをバッファに書き出す。
  • XLOG_REPLORIGIN_DROP: xl_replorigin_drop 構造体にキャストし、ノードIDをバッファに書き出す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h において、RM_REPLORIGIN_ID リソースマネージャの記述子ルーチンとして登録されている。
  • **呼び出し先**:
  • XLogRecGetData / XLogRecGetInfo (WALリーダーAPI)
  • appendStringInfo (文字列操作API)
  • ※具体的な呼び出し元関数はソースコード上では明示されていない。

関連データ構造

  • StringInfo: 出力バッファ。
  • XLogReaderState: WALレコード読み取り状態。
  • xl_replorigin_set: SET操作のデータ構造。
  • xl_replorigin_drop: DROP操作のデータ構造。

失敗モードと境界条件

  • 未知の info ビットが渡された場合、何も出力せずに終了する（switch 文に該当ケースがないため）。
  • record が不正なデータポインタを指している場合、キャスト後のアクセスで未定義動作を引き起こす可能性がある。

保守担当者向けメモ

  • 新しいレプリケーションオリジンのWAL操作を追加した際は、必ず switch 文にケースを追加すること。
  • XLR_INFO_MASK によるビットマスク処理は、WALレコードのフラグを除去するために必須である。
