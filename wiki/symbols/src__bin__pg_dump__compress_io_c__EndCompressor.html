EndCompressor

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_io.c:147

目的

圧縮ストリームの処理を終了し、関連するリソースを解放します。

シグネチャまたは形状

void EndCompressor(ArchiveHandle *AH, CompressorState *cs);

入力と出力

  • **入力**:
  • AH: アーカイブ処理のハンドル（ArchiveHandle）。
  • cs: 終了対象となる圧縮コンテキスト（CompressorState）。
  • **出力**: なし（void）。

前提条件と不変条件

  • cs は AllocateCompressor によって初期化された有効なポインタである必要があります。
  • cs->end コールバック関数が適切に設定されている必要があります。

アルゴリズムまたは振る舞い

1. cs->end(AH, cs) を呼び出し、圧縮アルゴリズム固有の終了処理（バッファのフラッシュや終了コードの書き込み等）を実行します。 2. pg_free(cs) を呼び出し、CompressorState 構造体自体が占有していたメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_custom.c (複数箇所)
  • **呼び出し先**:
  • cs->end (関数ポインタ経由で各圧縮アルゴリズムの実装を呼び出す)
  • pg_free (メモリ解放)

関連データ構造

  • CompressorState: 圧縮処理の状態とコールバック関数を保持する構造体。
  • ArchiveHandle: アーカイブ全体の管理構造体。

失敗モードと境界条件

  • cs が NULL の場合、pg_free および関数ポインタの参照によりセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は圧縮ストリームのライフサイクルの終端を担います。新しい圧縮アルゴリズムを追加する際は、対応する CompressorState の end コールバックが正しく実装されていることを確認してください。
