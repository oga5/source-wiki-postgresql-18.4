is_foreign_pathkey

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/deparse.c:1120

目的

指定された PathKey がリモートサーバーでソート操作として実行可能（プッシュダウン可能）かどうかを判定する。

シグネチャまたは形状

bool is_foreign_pathkey(PlannerInfo *root, RelOptInfo *baserel, PathKey *pathkey)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • baserel: 対象となる外部リレーション情報。
  • pathkey: 評価対象のパスキー。
  • **出力**:
  • bool: リモート実行可能であれば true、そうでなければ false を返す。

前提条件と不変条件

  • baserel->fdw_private に PgFdwRelationInfo が格納されている必要がある。
  • pathkey が保持する EquivalenceClass および pk_opfamily が有効であること。

アルゴリズムまたは振る舞い

1. pathkey の EquivalenceClass (pathkey_ec) を取得し、揮発性（ec_has_volatile）を確認する。揮発性がある場合は false を返す。 2. is_shippable を呼び出し、pathkey の演算子族（pk_opfamily）がリモートサーバーで利用可能か検証する。利用不可の場合は false を返す。 3. find_em_for_rel を呼び出し、当該リレーションに対応する適切な等価クラスメンバーが存在するかを確認する。メンバーが存在すれば true を返し、存在しなければ false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c:938
  • **呼び出し先**:
  • is_shippable (外部モジュール shippable.c 由来)
  • find_em_for_rel (PostgreSQL コアのプランナ関数)

関連データ構造

  • PlannerInfo: プランニング中のクエリ情報。
  • RelOptInfo: リレーションの最適化情報。
  • PathKey: ソート順序を定義するノード。
  • PgFdwRelationInfo: postgres_fdw 固有のリレーション情報。
  • EquivalenceClass: 等価な式の集合。

失敗モードと境界条件

  • pathkey_ec->ec_has_volatile が真の場合、最適化の安全性確保のため即座に false を返す。
  • 演算子族がリモートサーバーで出荷可能（shippable）でない場合、リモートでのソート実行は許可されない。
  • 該当リレーションに対応する等価クラスメンバーが見つからない場合、ソート順序をリモートへ伝播できないため false となる。

保守担当者向けメモ

  • 本関数は postgres_fdw におけるソートのプッシュダウン可否を決定する重要なガードレールである。
  • is_shippable の判定ロジックに変更があった場合、本関数の挙動も影響を受ける可能性がある。
