AlterForeignDataWrapperOwner_oid

定義

  • 種別: function
  • 定義箇所: src/backend/commands/foreigncmds.c:323

目的

指定されたOIDを持つ外部データラッパー（Foreign Data Wrapper: FDW）の所有者を変更します。

シグネチャまたは形状

void AlterForeignDataWrapperOwner_oid(Oid fwdId, Oid newOwnerId)

入力と出力

  • **入力**:
  • fwdId (Oid): 変更対象のFDWのOID。
  • newOwnerId (Oid): 新しい所有者のロールOID。
  • **出力**: なし（成功時はカタログを更新し、失敗時はエラーをスロー）。

前提条件と不変条件

  • pg_foreign_data_wrapper システムカタログに対する RowExclusiveLock が取得されます。
  • 指定された fwdId に対応するFDWがシステムカタログに存在する必要があります。

アルゴリズムまたは振る舞い

1. pg_foreign_data_wrapper テーブルを RowExclusiveLock でオープンします。 2. FOREIGNDATAWRAPPEROID システムキャッシュを使用して、対象FDWのタプルをコピーとして取得します。 3. タプルが存在しない場合、ERRCODE_UNDEFINED_OBJECT エラーを発生させます。 4. AlterForeignDataWrapperOwner_internal を呼び出し、所有者変更の内部処理（カタログ更新および依存関係の処理）を実行します。 5. 取得したタプルを解放し、テーブルをクローズします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/pg_shdepend.c:1687（ロール削除時の依存関係処理などから呼び出される）。
  • **呼び出し先**: AlterForeignDataWrapperOwner_internal（実際の更新ロジック）。

関連データ構造

  • Relation: pg_foreign_data_wrapper システムカタログ。
  • HeapTuple: FDWの定義情報を含むタプル。

失敗モードと境界条件

  • 指定された fwdId が存在しない場合、ereport によりエラー終了します。

保守担当者向けメモ

  • 本関数はOIDベースの所有者変更インターフェースを提供します。名前ベースの変更が必要な場合は、別途用意されているラッパー関数を確認してください。
  • 実際のカタログ更新ロジックは AlterForeignDataWrapperOwner_internal に集約されています。
