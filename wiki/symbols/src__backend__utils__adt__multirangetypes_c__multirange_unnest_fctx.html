multirange_unnest_fctx

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/multirangetypes.c:2717

目的

マルチレンジ型（MultirangeType）の値を展開（unnest）する関数において、反復処理中の状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    MultirangeType *mr;       /* 展開対象のマルチレンジデータ */
    TypeCacheEntry *typcache; /* 型情報のキャッシュ */
    int             index;    /* 現在処理中のレンジのインデックス */
} multirange_unnest_fctx;

入力と出力

本構造体は関数内部で palloc を介してメモリ確保され、展開処理の各ステップで参照・更新されます。

前提条件と不変条件

  • mr は正規化された（ソート済みかつ非重複の）マルチレンジデータである必要があります。
  • index は、マルチレンジに含まれるレンジの総数を超えない範囲で管理される必要があります。

アルゴリズムまたは振る舞い

マルチレンジ型を個別のレンジに分解する際、この構造体を使用して現在の走査位置（index）を追跡します。これにより、セットリターン関数（SRF）としてマルチレンジを展開する際に、呼び出し間で状態を維持します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/multirangetypes.c 内のマルチレンジ展開処理関数（multirange_unnest 等）。
  • 呼び出し先: 構造体自体はデータ保持用であり、直接の関数呼び出しは行いません。

関連データ構造

  • MultirangeType: 展開対象となるマルチレンジのメモリ表現。
  • TypeCacheEntry: 型の属性情報（レンジの比較関数など）を保持するキャッシュ。

失敗モードと境界条件

  • mr が NULL の場合や、メモリ不足により palloc が失敗した場合は処理が継続できません。
  • index が範囲外に達した場合の挙動は、この構造体を利用する側の関数実装に依存します。

保守担当者向けメモ

  • 本構造体は multirange_unnest などの SRF 実装において、fcinfo->flinfo->fn_extra を介して状態を保持するために使用されることが想定されます。
  • 構造体のフィールド追加や変更を行う際は、関連する展開関数の初期化ロジックと整合性をとる必要があります。
