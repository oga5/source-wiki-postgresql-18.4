pg_regprefix

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regprefix.c:45

目的

コンパイル済みの正規表現オブジェクトから、マッチングの先頭に必ず現れる共通の接頭辞（prefix）を抽出します。この情報は、インデックススキャンやクエリ最適化におけるフィルタリングに利用されます。

シグネチャまたは形状

int pg_regprefix(regex_t *re, chr **string, size_t *slength);

入力と出力

  • **入力**:
  • re: コンパイル済みの正規表現オブジェクト。
  • string: 抽出された接頭辞を格納するバッファへのポインタ。
  • slength: 抽出された接頭辞の長さを格納する変数へのポインタ。
  • **出力**:
  • 戻り値: REG_PREFIX（接頭辞あり）、REG_EXACT（完全一致）、REG_NOMATCH（接頭辞なし）、またはエラーコード（REG_INVARG, REG_MIXED, REG_ESPACE）。
  • string: 成功時は接頭辞文字列（呼び出し元で解放が必要）、失敗時は NULL。

前提条件と不変条件

  • re は有効な正規表現オブジェクトであること。
  • string および slength は NULL でないこと。
  • re->re_csize が sizeof(chr) と一致すること。
  • 抽出された接頭辞の長さは、NFAの状態数（cnfa->nstates）を超えない。

アルゴリズムまたは振る舞い

1. 引数の妥当性チェックとロケールの初期化を行う。 2. 正規表現の内部構造（guts）からNFA（cnfa）を取得する。 3. MATCHALL フラグが立っている場合は REG_NOMATCH を返す。 4. NFAの状態数に基づき、接頭辞格納用のメモリを確保する。 5. findprefix を呼び出し、NFAの開始状態から遷移を辿って接頭辞を特定する。 6. 戻り値が REG_PREFIX または REG_EXACT でない場合は、確保したメモリを解放して結果をクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/regexp.c:2047
  • **呼び出し先**: findprefix（内部関数）

関連データ構造

  • regex_t: 正規表現オブジェクト。
  • struct guts: 正規表現の内部状態を保持する構造体。
  • struct cnfa: 解析対象となるコンパイル済みNFA。
  • chr: 正規表現エンジンが扱う文字型。

失敗モードと境界条件

  • REG_INVARG: 引数が不正、または正規表現オブジェクトが無効な場合。
  • REG_MIXED: 文字サイズが不一致の場合。
  • REG_NOMATCH: 接頭辞が特定できない、またはマッチ不可能と判断された場合。
  • REG_ESPACE: メモリ確保に失敗した場合。
  • バックリファレンスや複雑な先読み（lookaround）は完全には考慮されず、保守的な結果を返す。

保守担当者向けメモ

  • 本関数は正規表現のトップレベルのNFAのみを解析します。
  • 抽出された接頭辞は「マッチする文字列は必ずこの接頭辞を持つ」ことを保証しますが、逆は必ずしも真ではありません。
  • 確保された *string は、呼び出し元が責任を持って FREE する必要があります。
