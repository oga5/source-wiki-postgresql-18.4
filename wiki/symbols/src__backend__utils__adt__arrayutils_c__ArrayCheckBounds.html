ArrayCheckBounds

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arrayutils.c:116

目的

配列の次元数および各次元の境界値が、システム上の許容範囲内であるかを検証します。

シグネチャまたは形状

void ArrayCheckBounds(int ndim, const int *dims, const int *lb)

入力と出力

  • ndim: 配列の次元数。
  • dims: 各次元のサイズを格納した配列。
  • lb: 各次元の下限値（lower bound）を格納した配列。
  • 戻り値: なし。検証に失敗した場合は例外（エラー）を発生させます。

前提条件と不変条件

  • dims および lb は、少なくとも ndim 個の要素を持つ有効なメモリ領域を指している必要があります。
  • 内部的に ArrayCheckBoundsSafe を呼び出しており、計算結果がオーバーフローする場合や、配列サイズが MaxArraySize を超える場合にエラーを投げます。

アルゴリズムまたは振る舞い

1. 内部関数 ArrayCheckBoundsSafe を呼び出します。 2. 第4引数に NULL を渡すことで、エラー発生時に呼び出し元へ制御を戻さず、PostgreSQL のエラーメカニズムを通じて例外を発生させる動作を強制します。

呼び出し元と呼び出し先

  • 呼び出し先: ArrayCheckBoundsSafe
  • 呼び出し元（直接的な参照）:
  • src/backend/executor/execExprInterp.c
  • src/backend/utils/adt/array_userfuncs.c
  • src/backend/utils/adt/arrayfuncs.c (複数箇所)

関連データ構造

  • ArrayType: PostgreSQL の配列データ型の内部表現。本関数は、この構造体に含まれる次元情報や境界情報を検証するために使用されます。

失敗モードと境界条件

  • ndim が 0 未満の場合、または配列の総要素数が MaxArraySize を超える場合、エラーが発生します。
  • 各次元の dim[i] + lb[i] の計算において整数オーバーフローが発生する場合、エラーが発生します。

保守担当者向けメモ

  • 本関数は ArrayCheckBoundsSafe のラッパーです。エラーハンドリングの柔軟性が必要な場合は、直接 ArrayCheckBoundsSafe を使用してください。
  • arrayfuncs.c 内の多くの箇所で、配列操作の安全性を担保するために本関数が利用されています。変更を加える際は、配列のインデックス計算ロジックへの影響を考慮してください。
