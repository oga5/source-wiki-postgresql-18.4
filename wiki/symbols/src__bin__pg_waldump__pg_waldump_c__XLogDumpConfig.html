XLogDumpConfig

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_waldump/pg_waldump.c:55

目的

pg_waldump コマンドラインツールにおいて、ユーザーが指定した実行オプション（表示設定、フィルタリング条件、およびフルページイメージの保存先）を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct XLogDumpConfig
{
    /* display options */
    bool        quiet;
    bool        bkp_details;
    int         stop_after_records;
    int         already_displayed_records;
    bool        follow;
    bool        stats;
    bool        stats_per_record;

    /* filter options */
    bool        filter_by_rmgr[RM_MAX_ID + 1];
    bool        filter_by_rmgr_enabled;
    TransactionId filter_by_xid;
    bool        filter_by_xid_enabled;
    RelFileLocator filter_by_relation;
    bool        filter_by_extended;
    bool        filter_by_relation_enabled;
    BlockNumber filter_by_relation_block;
    bool        filter_by_relation_block_enabled;
    ForkNumber  filter_by_relation_forknum;
    bool        filter_by_fpw;

    /* save options */
    char       *save_fullpage_path;
} XLogDumpConfig;

入力と出力

本構造体は、main 関数でコマンドライン引数に基づいて初期化され、以降の処理において読み込み条件や出力形式を制御するための入力として各関数に渡されます。

前提条件と不変条件

  • main 関数内で memset を用いてゼロ初期化されることが想定されています。
  • 各フィルタリング条件は、対応する _enabled フラグが true の場合にのみ有効として扱われます。

アルゴリズムまたは振る舞い

  • pg_waldump のメインループにおいて、読み込んだ WAL レコードが本構造体のフィルタ条件に合致するかを判定するために参照されます。
  • 統計情報の表示やフルページイメージの保存といった後続処理の挙動を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（初期化および設定）、XLogDumpDisplayRecord、XLogDumpDisplayStats など。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • XLogDumpPrivate: WAL 読み込みの実行状態（タイムラインや LSN 範囲）を管理する構造体。
  • XLogReaderState: XLogReader インターフェースのメイン状態構造体。

失敗モードと境界条件

  • filter_by_rmgr 配列のサイズは RM_MAX_ID + 1 に依存しており、リソースマネージャの定義変更時に影響を受けます。
  • save_fullpage_path は文字列ポインタであるため、メモリ管理に注意が必要です。

保守担当者向けメモ

  • 本構造体へのフィールド追加は、main 関数内の引数解析ロジックおよび各出力関数への影響を考慮してください。
  • pg_walinspect と機能が重複する箇所があるため、変更時は両モジュールへの影響を評価してください。
