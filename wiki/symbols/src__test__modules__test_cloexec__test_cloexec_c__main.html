main

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_cloexec/test_cloexec.c:33

目的

Windows環境における O_CLOEXEC フラグの動作を検証するためのテストプログラムのエントリポイント。引数の有無により、親プロセス（テスト実行）と子プロセス（検証実行）の役割を切り替える。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**:
  • argc == 1: 親プロセスとして動作。
  • argc == 3: 子プロセスとして動作。argv[1] および argv[2] には、親から継承されたファイルハンドルの値が16進数文字列として渡される。
  • **出力**:
  • 正常終了時: 0 を返す。
  • 引数エラー時: 1 を返し、使用法を標準エラー出力に出力する。

前提条件と不変条件

  • 本関数は **Windows専用** である。WIN32 マクロが定義されていない環境では、エラーメッセージを出力して即座に終了する。
  • 子プロセスとして動作する場合、渡されるハンドルは有効なOSハンドルである必要がある。

アルゴリズムまたは振る舞い

1. **環境チェック**: WIN32 が定義されていない場合、エラーを出力して終了する。 2. **モード判定**:

  • 引数なし (argc == 1): 親プロセスとして動作。
  • 一時ファイル名を生成し、run_parent_tests を呼び出してテストを実行する。
  • 終了時に一時ファイルを削除する。
  • 引数あり (argc == 3): 子プロセスとして動作。
  • run_child_tests を呼び出し、渡されたハンドルに対して書き込みテストを実行する。
  • 上記以外: 使用法を表示して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: OSのプロセス起動により呼び出される。
  • **呼び出し先**:
  • run_parent_tests (親モード時)
  • run_child_tests (子モード時)
  • fprintf, snprintf, getpid, unlink (標準ライブラリ関数)
  • **注記**: ツールが提示した contrib/amcheck 等の参照は、本シンボルとは無関係な「main」という名称の一般的な用語に対する言及であり、本関数の呼び出し元ではない。

関連データ構造

  • MAXPGPATH: ファイルパスを格納するためのバッファサイズ定数。

失敗モードと境界条件

  • argc が 1 でも 3 でもない場合、不正な引数としてエラー終了する。
  • Windows以外のプラットフォームでは、テストを実行せずに終了する。

保守担当者向けメモ

  • 本関数は CreateProcess による自己再帰的なプロセス起動を前提としている。
  • run_parent_tests および run_child_tests の実装が、本テストの検証ロジックの核心である。
