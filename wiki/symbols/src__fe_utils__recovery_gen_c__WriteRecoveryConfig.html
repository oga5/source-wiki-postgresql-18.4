WriteRecoveryConfig

定義

  • 種別: function
  • 定義箇所: src/fe_utils/recovery_gen.c:124

目的

PostgreSQLのスタンバイサーバー構築において、接続情報に基づいたリカバリ設定ファイル（recovery.conf または postgresql.auto.conf）を生成し、必要に応じてシグナルファイルを作成する。

シグネチャまたは形状

void WriteRecoveryConfig(PGconn *pgconn, const char *target_dir, PQExpBuffer contents)

入力と出力

  • pgconn: 接続中のデータベース接続ハンドル。サーバーバージョンの判定に使用。
  • target_dir: 設定ファイルを出力するディレクトリパス。
  • contents: 書き込むリカバリ設定内容を保持する PQExpBuffer。
  • 出力: なし（ファイルシステムへの書き込みが副作用）。

前提条件と不変条件

  • pgconn は NULL であってはならない（Assert による検証）。
  • contents には有効なリカバリ設定文字列が格納されている必要がある。

アルゴリズムまたは振る舞い

1. PQserverVersion を用いてサーバーバージョンを確認し、MINIMUM_VERSION_FOR_RECOVERY_GUC 未満であれば古い形式（recovery.conf）、それ以降であれば新しい形式（postgresql.auto.conf）を使用すると判断する。 2. recovery.conf の場合は書き込みモード（"w"）で、postgresql.auto.conf の場合は追記モード（"a"）でファイルを開く。 3. contents の内容をファイルに書き出す。 4. 新しい形式（バージョン12以降）の場合、追加で standby.signal ファイルを空で作成する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_basebackup/pg_createsubscriber.c
  • src/bin/pg_rewind/pg_rewind.c
  • 呼び出し先:
  • fopen, fwrite, fclose (標準ライブラリ)
  • pg_fatal (エラーハンドリング)
  • PQserverVersion (libpq)

関連データ構造

  • PGconn: 接続情報およびサーバーバージョン保持用。
  • PQExpBuffer: 設定ファイルの内容を保持する動的バッファ。

失敗モードと境界条件

  • ファイルのオープンに失敗した場合、pg_fatal を呼び出しプログラムを終了する。
  • 書き込みに失敗した場合、pg_fatal を呼び出しプログラムを終了する。
  • バージョン判定により、出力先ファイル名とモードが動的に切り替わる。

保守担当者向けメモ

  • PostgreSQL 12を境にリカバリ設定の仕様が変更されているため、バージョン判定ロジックの変更には注意が必要。
  • メモリ不足等の致命的なエラー時は pg_fatal によりプロセスが終了するため、呼び出し側でリソースのクリーンアップを考慮する必要がある。
