TableFuncRoutine

定義

  • 種別: struct
  • 定義箇所: src/include/executor/tablefunc.h:52

目的

XMLTABLE のような、外部データソースから行セットを生成する「テーブル生成関数」のための抽象インターフェース（コールバックテーブル）を定義する。実行エンジンと具体的なデータ解析実装を疎結合に保つ役割を持つ。

シグネチャまたは形状

typedef struct TableFuncRoutine
{
    void (*InitOpaque) (struct TableFuncScanState *state, int natts);
    void (*SetDocument) (struct TableFuncScanState *state, Datum value);
    void (*SetNamespace) (struct TableFuncScanState *state, const char *name, const char *uri);
    void (*SetRowFilter) (struct TableFuncScanState *state, const char *path);
    void (*SetColumnFilter) (struct TableFuncScanState *state, const char *path, int colnum);
    bool (*FetchRow) (struct TableFuncScanState *state);
    Datum (*GetValue) (struct TableFuncScanState *state, int colnum, Oid typid, int32 typmod, bool *isnull);
    void (*DestroyOpaque) (struct TableFuncScanState *state);
} TableFuncRoutine;

入力と出力

  • 各関数ポインタは、実行状態を保持する TableFuncScanState 構造体へのポインタを第一引数に取る。
  • GetValue は、抽出した値を Datum 型で返し、値が NULL の場合は isnull を true に設定する。
  • FetchRow は、次の行が存在する場合は true を、存在しない場合は false を返す。

前提条件と不変条件

  • TableFuncScanState は前方宣言されており、具体的な構造は実行ノード側で管理される。
  • 各ルーチンは、実行エンジンが定義するライフサイクル（初期化→設定→反復→破棄）に従って呼び出される必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: InitOpaque で内部状態を構築する。 2. **設定**: SetDocument、SetNamespace、SetRowFilter、SetColumnFilter を用いて解析対象と抽出ルールを定義する。 3. **反復**: FetchRow を呼び出し、行を順次取得する。 4. **値抽出**: FetchRow が成功した各行に対して GetValue を呼び出し、列データを抽出する。 5. **終了**: DestroyOpaque でリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeTableFuncscan.c 内の実行ノードが、tstate->routine を介してこれらの関数を呼び出す。
  • **呼び出し先**: XmlTableRoutine（src/backend/utils/adt/xml.c）や JsonbTableRoutine（src/backend/utils/adt/jsonpath_exec.c）などの具体的な実装が、この構造体のインスタンスを提供している。

関連データ構造

  • TableFuncScanState: 実行状態を保持する構造体。
  • nodeTableFuncscan.h: このインターフェースを利用する実行ノードの定義。

失敗モードと境界条件

  • GetValue における型変換エラーや、FetchRow における解析エラーが発生する可能性がある。
  • 呼び出し順序が不正な場合（例：初期化前の呼び出し）の挙動は、各実装ルーチンに依存する。

保守担当者向けメモ

  • 本構造体は contrib/tablefunc とは無関係である。
  • 新しいテーブル生成関数を追加する場合は、本構造体の全関数ポインタを実装したルーチン群を定義する必要がある。
