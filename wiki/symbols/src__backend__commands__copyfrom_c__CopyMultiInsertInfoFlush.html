CopyMultiInsertInfoFlush

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfrom.c:661

目的

CopyMultiInsertInfo が保持するすべてのバルク挿入バッファをフラッシュ（書き出し）し、バッファリストのサイズが制限を超えた場合に古いバッファを解放してメモリを整理する。

シグネチャまたは形状

static inline void CopyMultiInsertInfoFlush(CopyMultiInsertInfo *miinfo, ResultRelInfo *curr_rri, int64 *processed)

入力と出力

  • miinfo: バルク挿入の状態を管理する構造体。
  • curr_rri: 現在処理中の ResultRelInfo。バッファの解放対象から除外するために使用される。
  • processed: 挿入されたタプル数のカウンタ。各バッファのフラッシュ時に更新される。

前提条件と不変条件

  • MAX_PARTITION_BUFFERS は 2 以上である必要がある（StaticAssertDecl により保証）。
  • miinfo->multiInsertBuffers 内のバッファは、CopyMultiInsertBufferFlush を通じて順次処理される。

アルゴリズムまたは振る舞い

1. miinfo->multiInsertBuffers リスト内の全バッファに対して CopyMultiInsertBufferFlush を実行し、データをテーブルへ書き出す。 2. miinfo の bufferedTuples および bufferedBytes を 0 にリセットする。 3. リスト長が MAX_PARTITION_BUFFERS を超えている場合、古いバッファから順に CopyMultiInsertBufferCleanup を呼び出して解放する。 4. 解放処理中、現在使用中のバッファ（curr_rri と一致するもの）に遭遇した場合は、リストの末尾に移動させて解放を回避する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfrom.c 内の複数箇所（CopyFrom メインループおよび終了処理に関連する箇所）。
  • **呼び出し先**: CopyMultiInsertBufferFlush（全バッファに対して）、CopyMultiInsertBufferCleanup（バッファ解放時）。

関連データ構造

  • CopyMultiInsertInfo: バルク挿入の全体状態を保持。
  • CopyMultiInsertBuffer: 個別のバッファ状態。
  • ResultRelInfo: ターゲットテーブルの情報。

失敗モードと境界条件

  • MAX_PARTITION_BUFFERS 未満のリストサイズであれば、解放処理はスキップされる。
  • 現在使用中のバッファがリストの先頭にある場合、リストの末尾へ移動させることで、誤った解放を防ぐロジックが組み込まれている。

保守担当者向けメモ

  • 本関数は static inline で定義されており、copyfrom.c 内のバルク挿入のライフサイクル管理において重要な役割を果たす。
  • バッファの削除順序は「作成が古いものから」というヒューリスティックに基づいている。
