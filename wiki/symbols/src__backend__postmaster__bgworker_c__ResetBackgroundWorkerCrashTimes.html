ResetBackgroundWorkerCrashTimes

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/bgworker.c:578

目的

登録されているすべてのバックグラウンドワーカーのクラッシュ状態をリセットし、再起動可能な状態へ移行させる。

シグネチャまたは形状

void ResetBackgroundWorkerCrashTimes(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として BackgroundWorkerList 内の各 RegisteredBgWorker の状態を更新する）

前提条件と不変条件

  • BackgroundWorkerList が初期化済みであること。
  • 並列ワーカー（BGWORKER_CLASS_PARALLEL）はクラッシュ後に生存・再起動できない設計であるため、本関数が処理する対象には含まれない（アサーションにより保証）。

アルゴリズムまたは振る舞い

1. BackgroundWorkerList を走査する。 2. BGW_NEVER_RESTART が設定されているワーカーは、ForgetBackgroundWorker を呼び出してリストから削除する。 3. それ以外のワーカーに対しては以下のリセット処理を行う：

  • rw_crashed_at を 0 に設定。
  • rw_pid を 0 に設定。
  • bgw_notify_pid を 0 に設定し、通知待ちを解除する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/postmaster.c:3194
  • 呼び出し先: ForgetBackgroundWorker

関連データ構造

  • RegisteredBgWorker: ワーカーの登録情報を保持する構造体。
  • BackgroundWorkerList: 登録済みワーカーを管理する dlist。

失敗モードと境界条件

  • BGW_NEVER_RESTART 指定のワーカーは、クラッシュ後に再起動させず即座に破棄される。
  • 並列ワーカーが本関数に到達した場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は Postmaster のプロセス管理ループから呼び出されることを想定している。
  • 並列ワーカーのカウント（parallel_register_count 等）との整合性を維持するため、クラッシュ後の処理順序には注意が必要である。
