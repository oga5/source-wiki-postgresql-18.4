spgNodePtr

定義

  • 種別: struct
  • 定義箇所: src/backend/access/spgist/spgtextproc.c:87

目的

SP-GiST の text 型インデックス構築時において、ノードのラベル（c）とそれに対応するインデックス情報（i）、およびデータ（d）を一時的に保持し、ソートや分配を行うためのデータ構造です。

シグネチャまたは形状

typedef struct spgNodePtr
{
    Datum       d;
    int         i;
    int16       c;
} spgNodePtr;

入力と出力

本構造体はデータ保持用であり、関数としての入出力は持ちません。主に spg_text_picksplit 関数内で、インデックス対象のタプルをノードへ分配する際の管理単位として使用されます。

前提条件と不変条件

  • c はノードラベルを表します。
  • i は対応するタプルのインデックスを指します。
  • d は Datum 型のデータを保持します。

アルゴリズムまたは振る舞い

spg_text_picksplit において、複数の値をソートする際に使用されます。具体的には、cmpNodePtr 関数によって c（ノードラベル）の値に基づいた昇順ソートが行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgtextproc.c 内の spg_text_picksplit 関数が、palloc を用いて本構造体の配列を確保し、利用します。
  • **呼び出し先**: qsort 関数が cmpNodePtr を介して本構造体の配列をソートします。

関連データ構造

  • Datum: PostgreSQL の汎用データ型。
  • spg_text_picksplit: 本構造体を利用してノードの分配ロジックを実行する関数。

失敗モードと境界条件

  • メモリ不足時: palloc によるメモリ確保が失敗した場合、PostgreSQL のエラーハンドリングによりアボートされます。
  • c の値: -1（終端）や -2（ダミーラベル）といった特殊な値が格納されることがあり、これらは文字列復元には寄与しません。

保守担当者向けメモ

  • 本構造体は spgtextproc.c 内の局所的な処理（主に picksplit）に特化しています。
  • c フィールドの比較ロジックを変更する場合は、cmpNodePtr の実装も併せて確認してください。
