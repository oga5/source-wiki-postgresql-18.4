CheckPubRelationColumnList

定義

  • **種別**: function
  • **定義箇所**: src/backend/commands/publicationcmds.c:780

目的

パブリケーションに含まれるリレーション（テーブル）に対して、カラムリスト（列の限定）が指定されている場合に、その構成が論理レプリケーションの制約に違反していないかを検証する。

シグネチャまたは形状

static void CheckPubRelationColumnList(char *pubname, List *tables, bool publish_schema, bool pubviaroot)

入力と出力

  • **入力**:
  • pubname: パブリケーション名。
  • tables: PublicationRelInfo 構造体のリスト。
  • publish_schema: スキーマ単位のパブリケーション設定が含まれるかを示すフラグ。
  • pubviaroot: パーティションルート経由で変更を公開するかを示すフラグ。
  • **出力**: なし（正常終了時）。エラー発生時は ereport(ERROR, ...) によりトランザクションを中断する。

前提条件と不変条件

  • tables リスト内の各要素は PublicationRelInfo 型であること。
  • カラムリストが指定されていないリレーション（pri->columns == NIL）は検証対象外としてスキップされる。

アルゴリズムまたは振る舞い

1. tables リストを走査し、カラムリストが指定されている各リレーションに対して以下のチェックを行う。 2. **スキーマ指定との競合**: publish_schema が true の場合、カラムリストの指定は許可されずエラーとなる。 3. **パーティションテーブルの制約**: pubviaroot が false かつ対象がパーティションテーブル（RELKIND_PARTITIONED_TABLE）である場合、カラムリストの指定は許可されずエラーとなる。これは、ルート経由の公開を行わない場合にパーティション固有のカラムリストが使用される仕様との整合性を保つためである。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/publicationcmds.c 内のパブリケーション作成・変更処理（CREATE PUBLICATION や ALTER PUBLICATION 関連のロジック）から複数箇所で呼び出されている。
  • **呼び出し先**:
  • get_namespace_name, RelationGetNamespace, RelationGetRelationName などのカタログアクセス関数。
  • ereport によるエラー報告。

関連データ構造

  • PublicationRelInfo: パブリケーションに関連付けられたリレーション情報。カラムリスト情報（columns）を保持する。

失敗モードと境界条件

  • **ERRCODE_INVALID_PARAMETER_VALUE**:
  • スキーマ単位のパブリケーション設定とカラムリストが同時に指定された場合。
  • publish_via_partition_root が無効な状態で、パーティションテーブルに対してカラムリストが指定された場合。

保守担当者向けメモ

  • 本関数は論理レプリケーションの定義整合性を維持するための重要なバリデーションである。
  • 将来的にカラムリストの制約が緩和される場合、この関数内の条件分岐を更新する必要がある。
  • pubviaroot の挙動変更時には、パーティションテーブルに対するチェックロジックの妥当性を再確認すること。
