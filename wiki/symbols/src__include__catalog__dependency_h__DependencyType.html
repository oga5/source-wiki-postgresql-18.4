DependencyType

定義

  • 種別: enum
  • 定義箇所: src/include/catalog/dependency.h:31

目的

データベースオブジェクト間の依存関係の性質を分類し、オブジェクト削除時の整合性維持や連鎖削除の制御を行うための識別子を提供します。

シグネチャまたは形状

typedef enum DependencyType
{
    DEPENDENCY_NORMAL = 'n',
    DEPENDENCY_AUTO = 'a',
    DEPENDENCY_INTERNAL = 'i',
    DEPENDENCY_PARTITION_PRI = 'P',
    DEPENDENCY_PARTITION_SEC = 'S',
    DEPENDENCY_EXTENSION = 'e',
    DEPENDENCY_AUTO_EXTENSION = 'x',
} DependencyType;

入力と出力

  • 本型は pg_depend カタログ内の deptype フィールド（char 型）に格納されます。
  • オブジェクト削除処理（performDeletion 等）において、この値が削除の可否や連鎖削除の挙動を決定する入力として利用されます。

前提条件と不変条件

  • 各列挙値は、対応する文字コード（'n', 'a', 'i' 等）と一対一で対応しています。
  • システムカタログ pg_depend に永続化される際、この値が依存関係のセマンティクスを規定します。

アルゴリズムまたは振る舞い

  • オブジェクトの作成時に recordDependencyOn を通じて記録されます。
  • 削除時には、この型に基づいて、依存先オブジェクトを自動的に削除するか、あるいは削除を禁止するかといった判断が下されます。

関連データ構造

  • pg_depend: 依存関係を保持するシステムカタログ。
  • ObjectAddress: 依存関係の対象となるオブジェクトを識別する構造体。

失敗モードと境界条件

  • 未定義の文字コードが pg_depend に存在する場合、削除処理のロジックで予期せぬ挙動を引き起こす可能性があります。

保守担当者向けメモ

  • SharedDependencyType と混同しないよう注意してください。本型は通常のオブジェクト依存関係を対象としています。
  • 新しい依存関係の種類を追加する場合は、pg_depend のスキーマ定義および削除処理ロジック（dependency.c）の両方を更新する必要があります。
