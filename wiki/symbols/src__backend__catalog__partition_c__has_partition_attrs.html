has_partition_attrs

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/partition.c:254

目的

指定されたリレーションのパーティションキー（列または式）に、与えられた属性集合（attnums）が含まれているかを判定します。

シグネチャまたは形状

bool has_partition_attrs(Relation rel, Bitmapset *attnums, bool *used_in_expr)

入力と出力

  • **入力**:
  • rel: 対象のリレーション（Relation）。
  • attnums: 検査対象の属性番号集合（Bitmapset）。
  • used_in_expr: 属性が式の一部として使用されている場合に情報を返すためのポインタ（bool *）。
  • **出力**:
  • 属性がパーティションキーに含まれる場合は true、それ以外は false を返します。

前提条件と不変条件

  • rel がパーティションテーブル（RELKIND_PARTITIONED_TABLE）である必要があります。
  • attnums が NULL の場合は常に false を返します。

アルゴリズムまたは振る舞い

1. リレーションのパーティションキー情報を取得します。 2. パーティションキーの各要素を順次走査します。 3. 要素が単純な列参照である場合、attnums に含まれるかを確認します。含まれる場合、used_in_expr を false に設定し true を返します。 4. 要素が式である場合、pull_varattnos を使用して式内で参照されている属性を抽出し、attnums との重複を確認します。重複がある場合、used_in_expr を true に設定し true を返します。 5. 全要素を走査しても一致しない場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c
  • src/backend/optimizer/util/inherit.c
  • **呼び出し先**:
  • RelationGetPartitionKey, get_partition_natts, get_partition_exprs, get_partition_col_attnum, pull_varattnos 等のパーティション管理用ユーティリティ関数。

関連データ構造

  • Relation: リレーションのメタデータ構造。
  • PartitionKey: パーティションキーの定義情報。
  • Bitmapset: 属性番号の集合を保持するビットマップ。

失敗モードと境界条件

  • rel がパーティションテーブルでない場合、または attnums が NULL の場合は即座に false を返します。
  • 属性番号の比較には FirstLowInvalidHeapAttributeNumber を用いたオフセット調整が行われます。

保守担当者向けメモ

  • 本関数は、DDL操作（tablecmds.c）や最適化フェーズ（inherit.c）において、特定の列に対する変更がパーティションキーに影響を与えるかを判断するために使用されます。
  • 式に含まれる属性の判定には pull_varattnos を使用しているため、複雑な式ツリーに対しても正確に属性の依存関係を追跡可能です。
