lstat_error_was_status_delete_pending

定義

  • 種別: function
  • 定義箇所: src/port/dirmod.c:103

目的

Windows 環境において、ファイルが削除保留（STATUS_DELETE_PENDING）状態にあるために発生したエラーを検出し、POSIX 互換の ENOENT エラーとして正しく解釈するための判定を行う。

シグネチャまたは形状

static bool lstat_error_was_status_delete_pending(void)

入力と出力

  • 入力: なし（グローバルな errno および Windows NT ステータスを参照）
  • 出力: bool 型。条件に合致する場合は true、それ以外は false を返す。

前提条件と不変条件

  • errno が ENOENT であることが、本関数が true を返すための必須条件である。
  • Windows 環境（WIN32 かつ !__CYGWIN__）でのみ、NT ステータスのチェックが有効となる。

アルゴリズムまたは振る舞い

1. 現在の errno が ENOENT でない場合、直ちに false を返す。 2. Windows 環境下において、pg_RtlGetLastNtStatus() を呼び出し、その戻り値が STATUS_DELETE_PENDING と一致するかを確認する。 3. 一致する場合は true を返し、それ以外の場合は false を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/dirmod.c 内の pgunlink 関数から参照されていることが確認されている。
  • 呼び出し先: pg_RtlGetLastNtStatus()（Windows 環境下のみ）。

関連データ構造

  • なし。

失敗モードと境界条件

  • Windows 以外のプラットフォームでは、errno == ENOENT のチェックのみが行われ、常に false を返す。
  • pg_RtlGetLastNtStatus() が利用できない環境や、NT ステータスが STATUS_DELETE_PENDING 以外の場合は、削除保留状態とは判定されない。

保守担当者向けメモ

  • 本関数は pgunlink におけるファイル削除処理の堅牢性を高めるためのハックである。Windows 特有のファイルシステム挙動を吸収する重要な箇所であるため、変更時は pgunlink への影響を十分に考慮すること。
