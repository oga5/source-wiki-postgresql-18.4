oauth_json_object_start

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/libpq/fe-auth-oauth.c:199

目的

JSON解析プロセスにおいて、オブジェクトの開始（{）を検出し、ネストレベルの管理およびバリデーションを行う。

シグネチャまたは形状

static JsonParseErrorType oauth_json_object_start(void *state);

入力と出力

  • **入力**: void *state（実際には struct json_ctx * 型のコンテキスト構造体へのポインタ）。
  • **出力**: JsonParseErrorType 型。エラーが発生した場合は JSON_SEM_ACTION_FAILED、正常時は JSON_SUCCESS を返す。

前提条件と不変条件

  • state は有効な json_ctx 構造体を指している必要がある。
  • ctx->nested は現在のJSONネスト深さを保持する。

アルゴリズムまたは振る舞い

1. **ターゲットフィールドの検証**: ctx->target_field が設定されている場合、オブジェクトの開始が許可されていない（文字列であるべき）と判断し、エラーを記録する。 2. **ネストレベルの更新**: ctx->nested をインクリメントする。 3. **深さ制限のチェック**: ctx->nested が MAX_SASL_NESTING_LEVEL を超えた場合、エラーを記録する。 4. **結果の返却**: oauth_json_has_error(ctx) を評価し、エラーがあれば失敗ステータスを、なければ成功ステータスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq-oauth/oauth-curl.c:505
  • src/interfaces/libpq/fe-auth-oauth.c:549（sem.object_start への代入を通じて間接的に使用）
  • **呼び出し先**:
  • oauth_json_set_error（エラー発生時）
  • oauth_json_has_error

関連データ構造

  • json_ctx: JSON解析の状態を保持する構造体。

失敗モードと境界条件

  • **ネスト超過**: MAX_SASL_NESTING_LEVEL を超える深いJSON構造に対して JSON_SEM_ACTION_FAILED を返す。
  • **構造違反**: 期待されるフィールド型と実際のJSON構造が一致しない場合、エラーを記録する。

保守担当者向けメモ

  • 本関数は common/jsonapi.h が提供するパーサーのコールバックとして機能する。
  • MAX_SASL_NESTING_LEVEL の定義は fe-auth-oauth.c 内の定数に依存する。
