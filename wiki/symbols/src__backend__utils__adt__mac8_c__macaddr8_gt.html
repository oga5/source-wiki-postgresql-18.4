macaddr8_gt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mac8.c:373

目的

2つの macaddr8 型の値を受け取り、第1引数が第2引数よりも大きいかどうかを判定する比較演算関数です。

シグネチャまたは形状

Datum macaddr8_gt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの macaddr8 型ポインタを受け取ります。
  • **出力**: PG_RETURN_BOOL を通じて、比較結果が真であれば true、偽であれば false を返します。

前提条件と不変条件

  • 入力される macaddr8 ポインタは有効である必要があります。
  • 内部的な比較ロジックは macaddr8_cmp_internal に依存しています。

アルゴリズムまたは振る舞い

1. PG_GETARG_MACADDR8_P マクロを使用して、2つの入力引数 a1 と a2 を取得します。 2. macaddr8_cmp_internal(a1, a2) を呼び出し、その戻り値が 0 より大きい（a1 > a2）かどうかを評価します。 3. 評価結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_macaddr8.c:32 において、DirectFunctionCall2 を介して呼び出されています。
  • **呼び出し先**:
  • macaddr8_cmp_internal（比較の基盤となる内部関数）。

関連データ構造

  • macaddr8: 8バイト（EUI-64）のMACアドレスを保持するデータ型。

失敗モードと境界条件

  • 引数が NULL の場合、PostgreSQLの関数呼び出し規約に従い、呼び出し元で適切に処理される必要があります（通常は strict 属性により自動的に NULL が返されます）。
  • macaddr8_cmp_internal が内部的に行う比較ロジック（hibits および lobits マクロによるビット演算）に依存しており、メモリレイアウトが不正な場合は予期せぬ結果を招く可能性があります。

保守担当者向けメモ

  • 本関数はSQL演算子 > の実装として機能します。
  • 比較ロジックを変更する場合は、macaddr8_cmp_internal を修正してください。
