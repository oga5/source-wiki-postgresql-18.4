bit_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varbit.c:817

目的

2つのビット列（VarBit）を比較し、その順序関係を判定します。

シグネチャまたは形状

int32 bit_cmp(VarBit *arg1, VarBit *arg2)

入力と出力

  • **入力**: 比較対象となる2つの VarBit 構造体へのポインタ arg1, arg2。
  • **出力**: arg1 < arg2 なら負の整数、arg1 == arg2 なら 0、arg1 > arg2 なら正の整数を返します。

前提条件と不変条件

  • VarBit 構造体のデータ領域（VARBITS）は、バイト単位で比較可能です。
  • 最後のバイトの未使用ビットは常にゼロクリアされている必要があります（VARBIT_PAD マクロによる不変条件）。この条件が満たされているため、比較時に末尾のマスク処理は不要です。

アルゴリズムまたは振る舞い

1. VARBITBYTES マクロを使用して、各ビット列のデータ長（バイト単位）を取得します。 2. memcmp を使用し、短い方の長さを上限としてバイト単位の比較を行います。 3. memcmp の結果が 0（等しい）の場合、VARBITLEN マクロでビット単位の長さを取得します。 4. ビット長が異なる場合は、ビット長が短い方を小さいと見なして結果を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/varbit.c 内の比較演算子実装関数群（=, !=, <, <=, >, >=）および比較関数から呼び出されます。
  • **呼び出し先**: memcmp（標準Cライブラリ）、VARBITBYTES、VARBITS、VARBITLEN（いずれも varbit.h で定義されたマクロ）。

関連データ構造

  • VarBit: ビット列を保持する構造体。vl_len_（ヘッダ）、bit_len（ビット長）、bit_dat（データ）で構成されます。

失敗モードと境界条件

  • ビット長が異なる場合、バイト単位の比較結果が等しくても、ビット長が短い方が小さいと判定されます。
  • memcmp はバイト単位で比較するため、ビット列のパディングが正しく行われていない場合、比較結果が不正確になる可能性があります。

保守担当者向けメモ

  • 本関数は B-tree インデックス等の比較処理の基盤となります。
  • varbit.h のコメントにある通り、パディングがゼロクリアされている前提に依存しているため、ビット列を操作する他の関数でパディングを破壊しないよう注意してください。
