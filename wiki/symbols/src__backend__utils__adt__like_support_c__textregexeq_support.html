textregexeq_support

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like_support.c:130

目的

正規表現演算子（~ 等）に対するプランナのサポート関数として機能し、インデックススキャンを利用可能な条件式への変換や、選択率の推定を仲介します。

シグネチャまたは形状

Datum textregexeq_support(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（プランナからの要求ノード rawreq を含む）。
  • **出力**: Datum 型。内部的には like_regex_support が生成した Node*（インデックス条件または選択率情報）を返します。

前提条件と不変条件

  • PG_GETARG_POINTER(0) により、有効な Node 構造体へのポインタが取得できる必要があります。
  • Pattern_Type_Regex を指定して like_regex_support を呼び出すため、正規表現特有の解析ロジックが適用されます。

アルゴリズムまたは振る舞い

1. 引数からプランナの要求ノード rawreq を抽出します。 2. like_regex_support 関数を呼び出し、第2引数に Pattern_Type_Regex を渡して処理を委譲します。 3. like_regex_support から返された結果をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: プランナ（インデックス適合性の判断時）。
  • **呼び出し先**: like_regex_support（実際のサポートロジックの実行）。

関連データ構造

  • Pattern_Type: パターンマッチングの種類を定義する列挙型（ここでは Pattern_Type_Regex を使用）。
  • Node: プランナの要求内容を保持する構造体。

失敗モードと境界条件

  • rawreq が不正なノードである場合、like_regex_support 内でエラーが発生する可能性があります。
  • 正規表現パターンがインデックス利用に適さない（固定接頭辞を持たない等）場合、like_regex_support はインデックス条件を生成せず、選択率の推定のみを行うか、デフォルト値を返します。

保守担当者向けメモ

  • 本関数は単なるラッパーです。正規表現のプランナサポートに関するロジック変更は、like_regex_support または regex_fixed_prefix を参照してください。
