InstrStopNode

定義

  • 種別: function
  • 定義箇所: src/backend/executor/instrument.c:83

目的

実行計画ノードの計測処理を終了し、経過時間、タプル数、バッファ使用量、および WAL 生成量の統計を累積値へ反映させます。

シグネチャまたは形状

void InstrStopNode(Instrumentation *instr, double nTuples)

入力と出力

  • **入力**:
  • instr: 更新対象の Instrumentation 構造体へのポインタ。
  • nTuples: 今回の処理で返されたタプル数。
  • **出力**: なし（instr が指す構造体の内部状態が更新されます）。

前提条件と不変条件

  • instr は初期化済みであること。
  • instr->need_timer が真の場合、事前に InstrStartNode が呼び出され、instr->starttime が設定されている必要があります（未設定の場合はエラーとなります）。

アルゴリズムまたは振る舞い

1. **タプル数の加算**: nTuples を instr->tuplecount に加算します。 2. **時間の計測**: need_timer が有効な場合、現在時刻を取得し、starttime からの差分を instr->counter に累積加算した後、starttime をゼロにリセットします。 3. **リソース使用量の集計**:

  • need_bufusage が有効な場合、グローバル変数 pgBufferUsage との差分を BufferUsageAccumDiff で計算し、instr->bufusage に加算します。
  • need_walusage が有効な場合、グローバル変数 pgWalUsage との差分を WalUsageAccumDiff で計算し、instr->walusage に加算します。

    • 4. **初回タプル時刻の記録**:

  • instr->running が偽（初回実行）の場合、現在の累積時間を firsttuple に記録し、running を真にします。
  • 非同期モード（async_mode）かつタプルが未出力の場合、同様に firsttuple を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execProcnode.c (各実行ノードの終了処理)
  • src/backend/executor/execMain.c (クエリ全体の実行終了時)
  • src/backend/executor/execAsync.c (非同期リクエストの完了時)
  • src/backend/commands/trigger.c (トリガー実行時)
  • **呼び出し先**:
  • BufferUsageAccumDiff
  • WalUsageAccumDiff
  • INSTR_TIME_* マクロ群

関連データ構造

  • Instrumentation: 計測データを保持する主要構造体。
  • BufferUsage, WalUsage: リソース使用量を追跡する構造体。

失敗モードと境界条件

  • InstrStartNode を呼び出さずに InstrStopNode を呼び出した場合、instr->need_timer が真であれば elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数はグローバルな統計カウンタ（pgBufferUsage 等）の差分を計算するため、計測対象のノードが実行されている間に他の処理がグローバルカウンタを更新しても、その影響を排除してノード固有の統計を抽出する設計になっています。
