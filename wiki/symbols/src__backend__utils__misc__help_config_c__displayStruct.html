displayStruct

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/help_config.c:73

目的

PostgreSQLの設定パラメータ（GUC）を表示対象とするか否かを、フラグに基づいて判定します。

シグネチャまたは形状

static bool displayStruct(mixedStruct *structToDisplay)

入力と出力

  • **入力**: mixedStruct *structToDisplay - 判定対象となるGUC変数の共用体ポインタ。
  • **出力**: bool - 表示対象であれば true、非表示対象であれば false を返します。

前提条件と不変条件

  • 引数 structToDisplay は有効なメモリ領域を指している必要があります。
  • structToDisplay が保持する generic メンバが、flags フィールドにアクセス可能であることを前提としています。

アルゴリズムまたは振る舞い

以下のいずれかのフラグが設定されている場合、非表示（false）と判定します。

  • GUC_NO_SHOW_ALL
  • GUC_NOT_IN_SAMPLE
  • GUC_DISALLOW_IN_FILE

これらがいずれも設定されていない場合にのみ、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/help_config.c 内のGUC変数リスト走査処理（61行目付近）。
  • **呼び出し先**: 本関数は直接的な関数呼び出しを行わず、構造体のメンバ参照のみを行います。

関連データ構造

  • mixedStruct: GUC変数を統一的に扱うための共用体。
  • config_generic: mixedStruct 内でフラグ情報を保持する共通構造体。

失敗モードと境界条件

  • structToDisplay が NULL の場合、未定義の動作（セグメンテーション違反）を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は --help-config 出力時のフィルタリングロジックを集中管理しています。新しい表示制御フラグを追加する場合は、本関数の条件式を更新してください。
