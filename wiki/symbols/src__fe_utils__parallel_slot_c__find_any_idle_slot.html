find_any_idle_slot

定義

  • 種別: function
  • 定義箇所: src/fe_utils/parallel_slot.c:178

目的

ParallelSlotArray 内で、現在使用されていない（inUse が偽である）最初のアイドルスロットのインデックスを検索します。

シグネチャまたは形状

static int find_any_idle_slot(const ParallelSlotArray *sa)

入力と出力

  • **入力**: sa (const ParallelSlotArray *) - 検索対象のスロット配列。
  • **出力**: int - 見つかったスロットのインデックス（0以上）。見つからない場合は -1 を返します。

前提条件と不変条件

  • sa は有効な ParallelSlotArray 構造体を指している必要があります。
  • スロット配列内の各要素は inUse フラグを持ち、これが false であることがアイドル状態の条件です。

アルゴリズムまたは振る舞い

1. 配列 sa->slots をインデックス 0 から sa->numslots - 1 まで順に走査します。 2. inUse が false である最初のスロットを見つけた時点で、そのインデックスを即座に返します。 3. 全てのスロットを走査してもアイドル状態のものが見つからない場合、-1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/parallel_slot.c:398（ParallelSlotsGetIdle 内での利用が確認されています）。
  • **呼び出し先**: 本関数は内部的な検索処理であり、他の関数を呼び出しません。

関連データ構造

  • ParallelSlotArray: スロットの配列と総数を保持する構造体。
  • ParallelSlot: 各接続状態と inUse フラグを保持する構造体。

失敗モードと境界条件

  • **境界条件**: sa->numslots が 0 の場合、ループは実行されず即座に -1 を返します。
  • **失敗モード**: 全てのスロットが使用中（inUse == true）の場合、期待通り -1 を返します。

保守担当者向けメモ

  • 本関数は単なる線形探索であり、計算量は O(N) です。
  • 呼び出し元である ParallelSlotsGetIdle は、この戻り値が -1 の場合に他のスロット確保戦略（接続の再利用や待機）へ移行する設計となっています。
