win32_read_locale

定義

  • 種別: function
  • 定義箇所: src/backend/commands/collationcmds.c:773

目的

Windows環境においてシステムロケールを列挙し、それらをPostgreSQLの照合順序（Collation）としてカタログへ登録するコールバック関数です。

シグネチャまたは形状

static BOOL CALLBACK win32_read_locale(LPWSTR pStr, DWORD dwFlags, LPARAM lparam)
  • pStr: ロケール名（ワイド文字列）。
  • dwFlags: 未使用。
  • lparam: CollParam 構造体へのポインタ。

入力と出力

  • 入力: lparam を介して渡される CollParam（名前空間IDやカウンタを含む）。
  • 出力: 常に TRUE を返し、列挙処理を継続します。成功時にカタログ（pg_collation）へレコードを挿入します。

前提条件と不変条件

  • EnumSystemLocalesEx からのコールバックとして呼び出されることを想定しています。
  • lparam は有効な CollParam 構造体を指している必要があります。

アルゴリズムまたは振る舞い

1. WideCharToMultiByte を使用して、ワイド文字列のロケール名をマルチバイト文字列に変換します。 2. create_collation_from_locale を呼び出し、ロケールをカタログへ登録します。 3. ロケール名にハイフン（-）が含まれる場合、それをアンダースコア（_）に置換した別名（エイリアス）を作成し、CollationCreate を用いて追加登録します。 4. 登録成功ごとに CommandCounterIncrement() を実行し、カタログの整合性を維持します。

呼び出し元と呼び出し先

  • 呼び出し元: EnumSystemLocalesEx（src/backend/commands/collationcmds.c:1042）から間接的に呼び出されます。
  • 呼び出し先: create_collation_from_locale, CollationCreate, get_collation_actual_version, CommandCounterIncrement, GetUserId, OidIsValid。

関連データ構造

  • CollParam: 登録先の名前空間や、作成された照合順序の数を追跡するための構造体。

失敗モードと境界条件

  • WideCharToMultiByte が失敗した場合、またはロケール名が空の場合は処理をスキップして TRUE を返します。
  • ロケール名が長すぎる場合、DEBUG1 ログを出力してスキップします。
  • create_collation_from_locale が負の値を返した場合、登録を中断して TRUE を返します。

保守担当者向けメモ

  • 本関数はWindows固有のロケール列挙ロジックです。POSIX環境との互換性を保つため、ハイフンをアンダースコアに置換する処理が含まれています。
  • CommandCounterIncrement の呼び出しは、大量のロケールを登録する際のパフォーマンスに影響を与える可能性があります。
