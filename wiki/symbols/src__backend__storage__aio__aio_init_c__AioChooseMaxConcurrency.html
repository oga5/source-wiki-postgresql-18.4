AioChooseMaxConcurrency

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_init.c:96

目的

PostgreSQL の非同期 I/O (AIO) サブシステムにおいて、システムリソース（バッファ数）に基づき、バックエンドあたりの最大同時実行 I/O 数を自動的に決定します。

シグネチャまたは形状

int AioChooseMaxConcurrency(void)

入力と出力

  • **入力**: なし（グローバル変数 NBuffers および MaxBackends を参照）
  • **出力**: int 型の同時実行数（1 以上 64 以下）

前提条件と不変条件

  • NBuffers がシステム内で適切に初期化されている必要があります。
  • MaxBackends および NUM_AUXILIARY_PROCS を用いて計算されるため、これらが確定した後に呼び出される必要があります。

アルゴリズムまたは振る舞い

1. 全バックエンド数（MaxBackends + NUM_AUXILIARY_PROCS）を算出します。 2. NBuffers を全バックエンド数で除算し、バックエンドあたりの比例配分値を求めます。 3. 最小値を 1 に制限します（Max(max_proportional_pins, 1)）。 4. 上限値を 64 に制限し、その値を返します（Min(max_proportional_pins, 64)）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio_init.c 内の初期化ロジック（設定値の検証および GUC 設定の生成時）。
  • **呼び出し先**:
  • Max() および Min() マクロ（PostgreSQL のユーティリティ関数）。

関連データ構造

  • NBuffers: バッファプール内のバッファ総数。
  • MaxBackends: 同時接続可能なバックエンドの最大数。

失敗モードと境界条件

  • NBuffers が非常に小さい場合でも、最小値 1 が保証されます。
  • システムリソースが豊富な場合でも、上限値 64 によってメモリ消費が抑制されます。

保守担当者向けメモ

  • 本関数は io_max_concurrency が明示的に設定されていない場合のデフォルト値を決定するために使用されます。
  • 64 という上限値は、AIO サブシステムのメモリレイアウト設計に直結するため、変更時には AioHandleShmemSize 等の計算式との整合性に注意してください。
