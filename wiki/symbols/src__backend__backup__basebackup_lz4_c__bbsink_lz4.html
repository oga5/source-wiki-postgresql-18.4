bbsink_lz4

定義

  • 種別: struct
  • 定義箇所: src/backend/backup/basebackup_lz4.c:23

目的

PostgreSQLのベースバックアップ処理において、LZ4圧縮を適用するための「シンク（sink）」状態を保持する構造体です。bbsink インターフェースの一部として、圧縮コンテキストやバッファ管理情報を保持します。

シグネチャまたは形状

typedef struct bbsink_lz4
{
    bbsink      base;
    int         compresslevel;
    LZ4F_compressionContext_t ctx;
    LZ4F_preferences_t prefs;
    size_t      bytes_written;
} bbsink_lz4;

入力と出力

  • **入力**: bbsink インターフェースを介して渡されるバックアップデータストリーム。
  • **出力**: LZ4圧縮されたデータストリーム。後続のシンクへ転送されます。

前提条件と不変条件

  • USE_LZ4 が定義されている環境でのみ有効です。
  • bbsink の共通基盤（base）が正しく初期化されている必要があります。
  • 圧縮処理はストリーミングで行われるため、LZ4F_compressBound を用いたバッファサイズの厳密な計算が前提となります。

アルゴリズムまたは振る舞い

1. **初期化**: bbsink_lz4_new によりコンテキストが生成され、圧縮レベルが設定されます。 2. **圧縮**: archive_contents 呼び出し時に LZ4F_compressUpdate を使用してデータを圧縮し、後続のシンクへ書き込みます。 3. **終了**: end_archive で LZ4F_compressEnd を呼び出し、残りのデータをフラッシュしてフレームを閉じます。 4. **クリーンアップ**: cleanup により、確保された圧縮コンテキストが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup_lz4.c 内の各関数（bbsink_lz4_new, bbsink_lz4_begin_backup, bbsink_lz4_begin_archive, bbsink_lz4_archive_contents, bbsink_lz4_end_archive, bbsink_lz4_manifest_contents, bbsink_lz4_cleanup）が、bbsink ポインタをキャストして本構造体を参照します。
  • **呼び出し先**: LZ4 Frameライブラリ（LZ4F_... 関数群）を呼び出します。

関連データ構造

  • bbsink: 基底となるシンク構造体。
  • LZ4F_compressionContext_t: LZ4圧縮コンテキスト。
  • LZ4F_preferences_t: LZ4圧縮設定。

失敗モードと境界条件

  • LZ4ライブラリの初期化や圧縮処理が失敗した場合、エラーが発生します。
  • マニフェストデータは圧縮対象外であり、そのまま後続へ転送される仕様です。
  • バッファが溢れる場合は、bbsink_archive_contents を呼び出してフラッシュする必要があります。

保守担当者向けメモ

  • 本構造体は basebackup_sink.h で定義されたパイプライン構造の一部です。
  • 圧縮処理はストリーミングで行われるため、入力データと出力データのサイズが一致しない点に注意してください。
  • ビルド時にLZ4ライブラリがリンクされていることが必須です。
