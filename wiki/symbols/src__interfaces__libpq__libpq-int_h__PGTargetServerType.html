PGTargetServerType

定義

  • 種別: enum
  • 定義箇所: src/interfaces/libpq/libpq-int.h:235

目的

データベース接続時に、接続先サーバーの役割（プライマリ、スタンバイ、読み取り専用など）を指定するための列挙型です。

シグネチャまたは形状

typedef enum
{
	SERVER_TYPE_ANY = 0,
	SERVER_TYPE_READ_WRITE,
	SERVER_TYPE_READ_ONLY,
	SERVER_TYPE_PRIMARY,
	SERVER_TYPE_STANDBY,
	SERVER_TYPE_PREFER_STANDBY,
	SERVER_TYPE_PREFER_STANDBY_PASS2
} PGTargetServerType;

入力と出力

  • **入力**: 接続パラメータとして指定されるターゲットサーバーの種別。
  • **出力**: 接続処理において、指定された条件を満たすサーバーを選択するための内部フラグとして使用されます。

前提条件と不変条件

  • SERVER_TYPE_ANY がデフォルト値として扱われます。
  • SERVER_TYPE_PREFER_STANDBY_PASS2 は内部的な状態遷移用であり、振る舞いは SERVER_TYPE_ANY と同等です。

アルゴリズムまたは振る舞い

接続確立プロセスにおいて、クライアントが要求するサーバーの特性を識別するために使用されます。接続先が要求されたタイプと一致するかを判定するロジックの基盤となります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/libpq-int.h:532 にて PGconn 構造体のメンバ target_server_type として参照されています。
  • **呼び出し先**: 提供されたソース情報からは、具体的な関数呼び出し関係は特定できません。

関連データ構造

  • PGconn: データベース接続状態を保持する構造体。本列挙型はこの構造体のメンバとして保持されます。

失敗モードと境界条件

  • 接続先サーバーが指定された PGTargetServerType の条件を満たさない場合、接続試行が失敗する可能性があります。

保守担当者向けメモ

  • 本シンボルは libpq 内部のプライベートな定義です。アプリケーション開発者は直接利用せず、接続文字列等の公開 API を通じて制御してください。
  • SERVER_TYPE_PREFER_STANDBY_PASS2 のような内部的な列挙値が追加されているため、ロジック変更時にはこの値が意図通りに ANY として扱われているか確認が必要です。
