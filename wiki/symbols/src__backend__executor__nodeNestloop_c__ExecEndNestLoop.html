ExecEndNestLoop

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeNestloop.c:360

目的

ネステッドループ結合ノードの実行を終了し、関連するリソースを解放する。

シグネチャまたは形状

void ExecEndNestLoop(NestLoopState *node)

入力と出力

  • **入力**: NestLoopState *node - 終了対象のネステッドループ結合ノードの状態構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 node は有効な NestLoopState 構造体を指している必要がある。
  • 内部および外部の子プランの状態が初期化済みであること。

アルゴリズムまたは振る舞い

1. デバッグログ（NL1_printf）を出力し、ノード処理の終了を開始する。 2. outerPlanState(node) を引数として ExecEndNode を呼び出し、外部子プランの終了処理を行う。 3. innerPlanState(node) を引数として ExecEndNode を呼び出し、内部子プランの終了処理を行う。 4. デバッグログを出力し、ノード処理の終了を完了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内の実行エンジン（ExecEndNode 経由の呼び出しが想定される）。
  • **呼び出し先**:
  • ExecEndNode（外部および内部の子プランに対して実行）
  • outerPlanState / innerPlanState（マクロ）

関連データ構造

  • NestLoopState: 実行時の状態を保持する構造体。
  • PlanState: 子プランの実行状態を管理する基底構造体。

失敗モードと境界条件

  • node が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • 子プランの状態が適切に初期化されていない場合、ExecEndNode 内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は ExecEndNode を介して子ノードのクリーンアップを委譲する役割を持つ。
  • デバッグ時には NL1_printf マクロにより、結合処理の終了フローをトレース可能である。
