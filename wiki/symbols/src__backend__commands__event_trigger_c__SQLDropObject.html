SQLDropObject

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/event_trigger.c:89

目的

sql_drop イベントトリガーにおいて、削除対象となるデータベースオブジェクトの情報を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct SQLDropObject
{
	ObjectAddress address;
	const char *schemaname;
	const char *objname;
	const char *objidentity;
	const char *objecttype;
	List	   *addrnames;
	List	   *addrargs;
	bool		original;
	bool		normal;
	bool		istemp;
	slist_node	next;
} SQLDropObject;

入力と出力

  • 本構造体はデータ保持用であり、関数としての入出力は持ちません。
  • palloc0 を用いてメモリ確保され、slist（単方向連結リスト）のノードとして管理されます。

前提条件と不変条件

  • slist_node next フィールドにより、イベントトリガー状態内のリスト構造に組み込まれます。
  • 削除対象オブジェクトの識別情報（address, objname 等）は、イベントトリガー実行時にカタログから取得される値と整合している必要があります。

アルゴリズムまたは振る舞い

  • DDL実行中に削除されるオブジェクトを追跡する際、各オブジェクトごとに本構造体が生成されます。
  • obtain_object_name_namespace 関数等を通じて、オブジェクトの識別名やスキーマ名が本構造体に格納されます。
  • 蓄積されたリストは、sql_drop イベントトリガーが起動された際に、トリガー関数から参照可能な情報として提供されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/event_trigger.c 内のオブジェクト削除追跡ロジック（palloc0 によるインスタンス化）。
  • 呼び出し先: obtain_object_name_namespace 関数が本構造体へのポインタを引数として受け取り、フィールドを埋めます。

関連データ構造

  • ObjectAddress: 削除対象オブジェクトのシステムカタログ上の位置を特定するために使用。
  • slist_node: SQLDropObject をリストとして連結するために使用。
  • EventTriggerQueryState: 本構造体のリストを保持する上位のコンテキスト構造体。

失敗モードと境界条件

  • メモリ不足時: palloc0 が失敗した場合、PostgreSQLの標準的なエラーハンドリングによりトランザクションがアボートされます。
  • 識別情報取得失敗: obtain_object_name_namespace が対象オブジェクトの情報を解決できない場合、フィールドが不完全な状態で保持される可能性があります。

保守担当者向けメモ

  • 本構造体は sql_drop イベントトリガー専用です。他のイベントトリガータイプとの互換性はありません。
  • slist を使用しているため、リスト操作時には slist_container マクロを使用して構造体ポインタへ変換してください。
