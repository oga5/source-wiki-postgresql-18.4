DirectFunctionCall7Coll

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/fmgr.c:946

目的

7つの引数を取る関数を、FmgrInfo構造体の初期化を介さずに直接呼び出すための軽量なラッパー関数です。

シグネチャまたは形状

Datum DirectFunctionCall7Coll(PGFunction func, Oid collation, Datum arg1, Datum arg2, Datum arg3, Datum arg4, Datum arg5, Datum arg6, Datum arg7)

入力と出力

  • **入力**:
  • func: 実行する関数ポインタ。
  • collation: 照合順序（Collation）のOID。
  • arg1〜arg7: 関数に渡す7つのDatum型引数。
  • **出力**:
  • 関数の実行結果をDatum型で返す。

前提条件と不変条件

  • 引数として渡される7つの値は、すべて非NULL（isnull = false）として扱われます。
  • 呼び出し先関数がNULLを返すことは想定されておらず、NULLが返された場合はエラーとなります。

アルゴリズムまたは振る舞い

1. LOCAL_FCINFOマクロを使用して、スタック上にFunctionCallInfoBaseData（fcinfo）を確保します。 2. InitFunctionCallInfoDataを呼び出し、引数数7、指定された照合順序でfcinfoを初期化します。 3. 7つの引数すべてをfcinfo->args配列に格納し、isnullフラグをfalseに設定します。 4. 関数ポインタfuncをfcinfoを引数として実行します。 5. 実行後、fcinfo->isnullを確認し、関数がNULLを返した場合はelog(ERROR)で異常終了します。 6. 正常終了時は結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/fmgr.h内のマクロDirectFunctionCall7から呼び出されます。
  • **呼び出し先**: 引数として渡されたPGFunction型の関数ポインタを実行します。

関連データ構造

  • FunctionCallInfoBaseData: 関数呼び出しのコンテキストを保持する構造体。
  • Datum: PostgreSQLの汎用データ型。
  • PGFunction: 関数ポインタの型定義。

失敗モードと境界条件

  • 呼び出し先関数がNULLを返した場合、elog(ERROR)によりトランザクションがアボートされます。
  • 引数にNULLを渡すことはできません（常にisnull = falseとして処理されます）。

保守担当者向けメモ

  • 本関数はパフォーマンス最適化のために提供されています。ポリモーフィック関数など、FmgrInfoの完全な初期化が必要な場合には使用できません。
  • src/include/fmgr.hで定義されているマクロ経由での利用が推奨されます。
