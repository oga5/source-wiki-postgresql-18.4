gbt_textge

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_text.c:36

目的

GiSTインデックスにおいて、text型の値が比較対象の値以上であるか（>=）を判定するサポート関数を提供します。

シグネチャまたは形状

static bool gbt_textge(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の text 型データへのポインタ。
  • collation: 照合順序（Collation）のOID。
  • flinfo: 関数マネージャ情報（本関数内では直接使用されません）。
  • **出力**:
  • bool: a >= b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は有効な text 型のデータ構造を指している必要があります。
  • 内部で text_ge 関数を呼び出すため、PostgreSQLの標準的なテキスト比較ルールに従います。

アルゴリズムまたは振る舞い

1. PointerGetDatum を使用して、入力された void * ポインタを Datum 型に変換します。 2. DirectFunctionCall2Coll を介して、PostgreSQLの標準関数 text_ge を呼び出します。 3. text_ge の戻り値（bool）を DatumGetBool で取得し、そのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_text.c 内の gbtree_vinfo 構造体（tinfo）の初期化時に参照されています。
  • **呼び出し先**:
  • text_ge（PostgreSQL標準のテキスト比較関数）

関連データ構造

  • gbtree_vinfo: text 型のインデックス操作に必要な比較関数を保持する構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、PostgreSQLの標準的な関数呼び出し規約に従い、予期せぬ動作やクラッシュを引き起こす可能性があります。
  • 照合順序（collation）が不正な場合、text_ge の実行時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は text 型専用の比較ロジックを抽象化するためのラッパーです。
  • bpchar 型の同様の処理については、別途 gbt_bpcharge が定義されています。
  • GiSTインデックスの再チェック（Lossy判定）は、本関数を呼び出す上位の consistent 関数側で制御されます。
