pg_leftmost_one_pos64

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/port/pg_bitutils.h:71

目的

64ビット符号なし整数において、最も上位にある「1」のビット位置（0〜63）を特定する。

シグネチャまたは形状

int pg_leftmost_one_pos64(uint64 word)

入力と出力

  • **入力**: uint64 word（対象となる64ビット整数）
  • **出力**: int（最上位ビットのインデックス。0が最下位ビット、63が最上位ビットに対応）

前提条件と不変条件

  • word != 0 であること。0を渡した場合の動作は未定義であり、デバッグビルドでは Assert により停止する。

アルゴリズムまたは振る舞い

実行環境に応じて以下のいずれかの手法を選択する： 1. **コンパイラ組み込み関数**: HAVE__BUILTIN_CLZ が定義されている場合、__builtin_clzl または __builtin_clzll を使用して先頭のゼロ数を計算し、63 - clz で位置を算出する。 2. **MSVC組み込み関数**: Windows環境（x64/ARM64）では _BitScanReverse64 を使用する。 3. **フォールバック**: 上記が利用できない場合、8ビットずつシフトしながら非ゼロのバイトを探索し、pg_leftmost_one_pos ルックアップテーブルを用いて位置を特定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/numutils.c
  • src/bin/pgbench/pgbench.c
  • src/common/pg_prng.c
  • src/include/lib/radixtree.h
  • src/include/nodes/bitmapset.h
  • src/include/port/pg_bitutils.h（pg_nextpower2_64 等の内部実装）
  • **呼び出し先**:
  • pg_leftmost_one_pos（フォールバック時のみ使用されるルックアップテーブル）

関連データ構造

  • pg_leftmost_one_pos: フォールバック時に使用される、8ビット値から最上位ビット位置を引くためのルックアップテーブル。

失敗モードと境界条件

  • **境界条件**: word が 0 の場合、アサーションエラーが発生する。
  • **境界条件**: word が 0x8000000000000000 の場合、戻り値は 63 となる。

保守担当者向けメモ

  • 本関数はインライン展開されるため、呼び出し側での word != 0 のチェックが重要である。
  • 新しいアーキテクチャで最適化を追加する場合は、HAVE__BUILTIN_CLZ の条件分岐を拡張するか、適切な組み込み関数を追加すること。
