gettoken_query

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltxtquery_io.c:60

目的

ltxtquery の入力文字列を解析し、トークン単位（オペランド、演算子、括弧、終了）に分解して順次返却する。

シグネチャまたは形状

int32 gettoken_query(QPRS_STATE *state, int32 *val, int32 *lenval, char **strval, uint16 *flag)

入力と出力

  • **入力**:
  • state: 解析状態を保持する QPRS_STATE 構造体（バッファポインタを含む）。
  • val: 演算子やトークン種別を格納する出力用ポインタ。
  • lenval: オペランドの長さを格納する出力用ポインタ。
  • strval: オペランドの開始位置を指すポインタ。
  • flag: オペランドの修飾子（LVAR_SUBLEXEME 等）を格納する出力用ポインタ。
  • **出力**:
  • 戻り値: トークンの種別（OPR, VAL, OPEN, CLOSE, END, ERR）。

前提条件と不変条件

  • state->buf は解析対象の文字列を指している必要がある。
  • state->state は WAITOPERAND, INOPERAND, WAITOPERATOR のいずれかの状態を維持する。
  • 括弧の対応関係（state->count）は、解析の進行に伴い正しく増減される。

アルゴリズムまたは振る舞い

1. state->buf を走査し、現在の状態に応じて以下の処理を行う。

  • WAITOPERAND: !（否定）、(（開始括弧）、またはラベル（オペランド）を待機する。
  • INOPERAND: ラベルの継続、または修飾子（%, @, *）を処理し、それ以外が来たら VAL を返す。
  • WAITOPERATOR: &, |（演算子）、)（終了括弧）、または終端文字を待機する。

    • 2. トークンを特定するたびに、対応する種別を返し、state->buf を進める。 3. 構文エラーが発生した場合は ERR を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/ltree/ltxtquery_io.c 内の makepol 関数から呼び出される。
  • **呼び出し先**: pg_mblen_cstr, t_iseq, ISLABEL, isspace, ereturn 等のユーティリティ関数。

関連データ構造

  • QPRS_STATE: 解析中のバッファ、状態、エラーコンテキスト、括弧の深さを保持する構造体。

失敗モードと境界条件

  • **構文エラー**: 予期しない文字や、括弧の不整合（count < 0）、修飾子の不正な配置が発生した場合に ERR を返す。
  • **終端**: 文字列の終端に達した際、括弧が閉じられていない場合は ERR を返す。

保守担当者向けメモ

  • 本関数は QPRS_STATE の状態遷移を介して動作するため、状態変数の変更には注意が必要。
  • pg_mblen_cstr を使用しており、マルチバイト文字に対応している。
  • ereturn マクロを使用しているため、エラー発生時は呼び出し元のコンテキストで適切に処理される。
