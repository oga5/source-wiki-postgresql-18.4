hash_combine

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/common/hashfn.h:67

目的

既存のハッシュ値に新しい値を組み合わせて、新しいハッシュ値を生成する。Boostライブラリの hash_combine と同様の目的を持つ。

シグネチャまたは形状

static inline uint32 hash_combine(uint32 a, uint32 b)

入力と出力

  • **入力**:
  • a (uint32): 既存のハッシュ値。
  • b (uint32): 結合する新しいハッシュ値。
  • **出力**:
  • uint32: 結合後の新しいハッシュ値。

前提条件と不変条件

  • 入力されるハッシュ値は、ビットが均等にランダムに分布していることが期待される。
  • 本関数はインライン展開されるため、呼び出し側のオーバーヘッドを最小化する。

アルゴリズムまたは振る舞い

以下のビット演算を用いてハッシュ値を混合する。 1. b に定数 0x9e3779b9 を加算する。 2. a を左に6ビットシフト、右に2ビットシフトした値と、加算済みの b を a と XOR 演算する。 3. 結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/tupdesc.c
  • src/backend/utils/cache/funccache.c
  • src/backend/utils/resowner/resowner.c
  • **呼び出し先**:
  • 外部関数呼び出しは行わない（純粋なビット演算のみ）。

関連データ構造

  • uint32 型（PostgreSQLの基本型定義に依存）

失敗モードと境界条件

  • 特筆すべき失敗モードはない。
  • 入力値が偏っている場合、結合後のハッシュ値の分布も偏る可能性がある。

保守担当者向けメモ

  • 本関数はハッシュテーブルのキー生成やインデックス計算において、複数の要素を単一のハッシュ値に集約するために使用される。
  • 変更を加える際は、ハッシュ値の衝突確率に影響を与えるため、ビットの混合特性を維持すること。
