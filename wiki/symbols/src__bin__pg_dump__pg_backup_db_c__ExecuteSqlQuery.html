ExecuteSqlQuery

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_db.c:228

目的

データベースに対して SQL クエリを実行し、期待される実行結果ステータスと照合する。

シグネチャまたは形状

PGresult *ExecuteSqlQuery(Archive *AHX, const char *query, ExecStatusType status);

入力と出力

  • **入力**:
  • AHX: Archive 構造体へのポインタ（内部で ArchiveHandle にキャストされる）。
  • query: 実行する SQL 文の文字列。
  • status: 期待される ExecStatusType（例: PGRES_TUPLES_OK）。
  • **出力**:
  • 成功時: クエリ結果を含む PGresult 構造体へのポインタ。
  • 失敗時: die_on_query_failure を呼び出し、プロセスを終了する。

前提条件と不変条件

  • AHX は有効な ArchiveHandle を指している必要がある。
  • AH->connection が確立されている必要がある。

アルゴリズムまたは振る舞い

1. Archive ポインタを ArchiveHandle 型にキャストする。 2. libpq の PQexec を使用して、指定された query をデータベースに対して実行する。 3. 実行結果のステータスを PQresultStatus で取得し、引数 status と比較する。 4. ステータスが一致しない場合、die_on_query_failure を呼び出してエラー処理を行い、プロセスを終了する。 5. ステータスが一致する場合、取得した PGresult を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_dump.c 内の各処理（テーブル定義取得、データ抽出、セキュリティラベル取得など）および pg_backup_db.c 内の内部処理。
  • **呼び出し先**: libpq の PQexec および PQresultStatus、エラー発生時の die_on_query_failure。

関連データ構造

  • Archive / ArchiveHandle: データベース接続情報を含むアーカイブ管理構造体。
  • PGresult: libpq が返すクエリ実行結果。
  • ExecStatusType: libpq で定義されるクエリ実行ステータスの列挙型。

失敗モードと境界条件

  • クエリ実行結果が期待した status と異なる場合、die_on_query_failure によりプログラムが異常終了する。
  • 接続が切断されている場合、PQexec の挙動に依存する（通常はエラーが返る）。

保守担当者向けメモ

  • 本関数はエラー発生時に呼び出し元へ制御を戻さず、プロセス全体を終了させる設計である。
  • 複雑な SQL 解析は行わないため、クエリ文字列の妥当性は呼び出し側で担保する必要がある。
