TSConfigCacheEntry

定義

  • **種別**: struct
  • **定義箇所**: src/include/tsearch/ts_cache.h:71

目的

全文検索設定（Text Search Configuration）のメタデータをメモリ上にキャッシュし、システムカタログ（pg_ts_config 等）への頻繁なアクセスを回避して検索処理を高速化するために使用されます。

シグネチャまたは形状

typedef struct {
    Oid            cfgId;      /* ハッシュキー（先頭必須） */
    bool           isvalid;    /* キャッシュ有効性フラグ */
    Oid            prsId;      /* 関連するパーサーのOID */
    int            lenmap;     /* マップの長さ */
    ListDictionary *map;       /* トークン型ごとの辞書リスト */
} TSConfigCacheEntry;

入力と出力

  • **入力**: ハッシュテーブル検索時のキーとして cfgId（Oid）を使用します。
  • **出力**: 設定に関連付けられたパーサーIDおよび、トークン型と辞書の対応関係を保持する ListDictionary 構造体を提供します。

前提条件と不変条件

  • cfgId は構造体の最初のメンバである必要があります（ハッシュテーブル検索の制約）。
  • isvalid が false の場合、キャッシュは無効とみなされ、カタログから再読み込みが必要です。

アルゴリズムまたは振る舞い

1. **ルックアップ**: hash_search を用いて TSConfigCacheHash から検索されます。 2. **キャッシュミス**: エントリが存在しない、または isvalid が false の場合、カタログから情報を取得して構造体を再構築します。 3. **無効化**: システムキャッシュの更新が検知されると、InvalidateTSCacheCallBack により isvalid が false に設定されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/ts_parse.c（LexizeInit 等で設定情報を参照）
  • src/backend/tsearch/wparser.c
  • src/backend/utils/cache/ts_cache.c（キャッシュ管理ロジック内）
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な関数呼び出し先は定義されていません。

関連データ構造

  • ListDictionary: トークン型ごとの辞書リストを保持するために使用されます。
  • TSAnyCacheEntry: キャッシュエントリの共通基盤構造体。

失敗モードと境界条件

  • **メモリ管理**: キャッシュ再構築時に古い map などのメモリが適切に解放されない場合、メモリリークが発生する可能性があります。
  • **無効化の遅延**: isvalid フラグの更新が適切に行われない場合、古い設定情報が使用される可能性があります。

保守担当者向けメモ

  • 構造体の先頭メンバ（cfgId）を変更すると、ハッシュテーブルの検索ロジックが破壊されるため注意してください。
  • isvalid フラグの管理は src/backend/utils/cache/ts_cache.c のコールバック関数に依存しています。変更時はキャッシュ無効化の整合性を確認してください。
