PLy_current_execution_context

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_main.c:367

目的

現在実行中のPL/Python関数の実行コンテキスト（PLyExecutionContext）を取得します。

シグネチャまたは形状

PLyExecutionContext *PLy_current_execution_context(void)

入力と出力

  • 入力: なし
  • 出力: 現在の PLyExecutionContext 構造体へのポインタを返します。

前提条件と不変条件

  • PLy_execution_contexts が NULL でないこと。
  • Python関数が実行中であることを前提としており、そうでない場合はエラーとなります。

アルゴリズムまたは振る舞い

1. グローバル変数 PLy_execution_contexts が NULL かどうかを確認します。 2. NULL の場合、elog(ERROR) を発行し、Python関数が実行中ではない旨を報告して処理を中断します。 3. NULL でない場合、スタックの先頭にある現在の実行コンテキストを返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/pl/plpython/plpy_cursorobject.c (複数箇所)
  • src/pl/plpython/plpy_elog.c:273
  • src/pl/plpython/plpy_exec.c (2箇所)
  • src/pl/plpython/plpy_spi.c (2箇所)
  • 呼び出し先:
  • なし（PostgreSQLの elog マクロを除く）

関連データ構造

  • PLyExecutionContext: 実行中の手続きやメモリコンテキストを保持するスタック構造体。

失敗モードと境界条件

  • Python関数が実行されていないコンテキストで本関数が呼び出された場合、elog(ERROR) によりトランザクションがアボートされます。

保守担当者向けメモ

  • 本関数は、PL/Pythonモジュール内の各コンポーネントが現在の実行状態（メモリコンテキストやSPI状態など）にアクセスするための標準的な手段です。
  • PLy_push_execution_context および PLy_pop_execution_context によるスタック管理と整合している必要があります。
