pg_get_viewdef_ext

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ruleutils.c:696

目的

指定されたビューの OID に基づき、その定義（SQL 文）を人間が読み取り可能な形式で取得します。整形オプションの指定が可能です。

シグネチャまたは形状

Datum pg_get_viewdef_ext(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): 対象となるビューの OID。
  • PG_GETARG_BOOL(1): 整形（pretty-print）を行うかどうかのフラグ。
  • **出力**:
  • text: ビューの定義を表す SQL 文字列。ビューが見つからない場合は NULL を返します。

前提条件と不変条件

  • GET_PRETTY_FLAGS マクロにより、入力されたブール値から内部的な整形フラグが生成されます。
  • 内部処理において WRAP_COLUMN_DEFAULT がデフォルトの折り返し列数として使用されます。

アルゴリズムまたは振る舞い

1. 引数からビューの OID と整形フラグを取得します。 2. pg_get_viewdef_worker を呼び出し、ビューの定義を文字列として生成します。 3. 生成された文字列が NULL の場合、PG_RETURN_NULL() で終了します。 4. 文字列が取得できた場合、string_to_text を介して text 型に変換し、PG_RETURN_TEXT_P で返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: pg_get_viewdef_worker
  • **呼び出し元**: 提供されたソース情報には、本関数を呼び出している具体的な関数名は記載されていません。

関連データ構造

  • Datum: PostgreSQL の汎用データ型。
  • PG_FUNCTION_ARGS: SQL 関数呼び出しの引数コンテキスト。

失敗モードと境界条件

  • 指定された OID に対応するビューが存在しない場合、pg_get_viewdef_worker が NULL を返し、本関数も NULL を返します。

保守担当者向けメモ

  • 本関数は pg_get_viewdef の拡張版であり、整形オプションを明示的に制御するインターフェースを提供します。
  • 逆コンパイルのロジックは pg_get_viewdef_worker に集約されているため、生成される SQL の構文変更が必要な場合はそちらを確認してください。
