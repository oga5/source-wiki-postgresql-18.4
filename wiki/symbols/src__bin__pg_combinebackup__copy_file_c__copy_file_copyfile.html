copy_file_copyfile

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/copy_file.c:307

目的

Windows環境において、CopyFile APIを使用してファイルをコピーし、必要に応じてチェックサム計算を行うための抽象化関数です。

シグネチャまたは形状

static void copy_file_copyfile(const char *src, const char *dst, pg_checksum_context *checksum_ctx);

入力と出力

  • src: コピー元ファイルのパス（文字列）。
  • dst: コピー先ファイルのパス（文字列）。
  • checksum_ctx: チェックサム計算用のコンテキスト構造体。
  • 戻り値: なし（成功時）。失敗時は pg_fatal によりプロセスを終了します。

前提条件と不変条件

  • Windows環境での実行を前提としています。
  • CopyFile APIが利用可能である必要があります。

アルゴリズムまたは振る舞い

1. CopyFile(src, dst, true) を呼び出し、ファイルをコピーします。 2. CopyFile が 0 を返した場合（失敗時）、GetLastError() を取得してエラー情報をマッピングし、pg_fatal でエラーメッセージを出力して終了します。 3. コピー成功後、checksum_file(src, checksum_ctx) を呼び出し、ソースファイルのチェックサム計算を行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_combinebackup/copy_file.c 内の strategy_implementation 関数ポインタへの代入を通じて間接的に呼び出されます。
  • 呼び出し先: CopyFile (Windows API), _dosmaperr, pg_fatal, checksum_file。

関連データ構造

  • pg_checksum_context: チェックサム計算の状態を保持する構造体。

失敗モードと境界条件

  • CopyFile が失敗した場合、OSのエラーコードに基づいた致命的なエラーとして処理されます。
  • ソースファイルが存在しない、またはアクセス権限がない場合に失敗します。

保守担当者向けメモ

  • 本関数はWindows固有の実装です。他のプラットフォームで同様の機能が必要な場合は、別の実装関数が選択される設計になっています。
  • checksum_file はコピー完了後に呼び出されるため、ファイルシステム上の読み込みが再度発生する点に留意してください。
