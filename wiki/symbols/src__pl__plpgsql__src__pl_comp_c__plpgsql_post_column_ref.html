plpgsql_post_column_ref

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_comp.c:1011

目的

SQL 式内の ColumnRef（列参照）を解析した後に呼び出されるパーサーフックとして、PL/pgSQL 変数とテーブル列の間の名前解決および競合判定を行う。

シグネチャまたは形状

static Node *plpgsql_post_column_ref(ParseState *pstate, ColumnRef *cref, Node *var)

入力と出力

  • **入力**:
  • pstate: パーサーの状態。
  • cref: 解析対象の ColumnRef ノード。
  • var: コアパーサーによって既に解決されたノード（テーブル列など）。
  • **出力**:
  • 解決された Node（PL/pgSQL 変数または var）、または解決不能な場合は NULL。

前提条件と不変条件

  • pstate->p_ref_hook_state に PLpgSQL_expr が格納されている必要がある。
  • 関数内の resolve_option 設定に従い、変数を優先するか列を優先するかの判定が行われる。

アルゴリズムまたは振る舞い

1. **解決オプションの確認**: resolve_option が PLPGSQL_RESOLVE_VARIABLE の場合、変数の不一致が確定しているため NULL を返す。 2. **列優先の確認**: resolve_option が PLPGSQL_RESOLVE_COLUMN であり、かつ var が存在する場合、テーブル列を優先するため NULL を返す。 3. **変数解決の試行**: resolve_column_ref を呼び出し、識別子を PL/pgSQL 変数として解決する。 4. **曖昧性の検出**: 変数とテーブル列の両方が見つかった場合、ERRCODE_AMBIGUOUS_COLUMN エラーを発生させる。 5. **結果の返却**: 解決されたノードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のパーサー（pstate->p_post_columnref_hook に設定される）。
  • **呼び出し先**: resolve_column_ref（内部で呼び出し）。

関連データ構造

  • ParseState: パーサーの状態管理。
  • ColumnRef: SQL 内の列参照ノード。
  • PLpgSQL_expr: PL/pgSQL 式のコンテキスト情報。

失敗モードと境界条件

  • **曖昧な参照**: 変数と列の両方に一致する場合、エラーを報告する。
  • **解決不能**: 変数としても列としても解決できない場合、NULL を返す。

保守担当者向けメモ

  • 本関数は plpgsql_parser_setup でフックとして登録される。
  • transformColumnRef の内部状態（crerr）にアクセスできないため、競合時のエラーメッセージは限定的である点に注意すること。
