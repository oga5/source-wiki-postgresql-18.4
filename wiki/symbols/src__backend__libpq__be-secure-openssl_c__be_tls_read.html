be_tls_read

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-openssl.c:764

目的

OpenSSL を用いた TLS 接続経由でデータを読み込み、PostgreSQL の通信レイヤーへセキュアな I/O を提供する。

シグネチャまたは形状

ssize_t be_tls_read(Port *port, void *ptr, size_t len, int *waitfor)

入力と出力

  • port: TLS 接続状態を保持する Port 構造体へのポインタ。
  • ptr: 読み込んだデータを格納するバッファ。
  • len: 読み込むバイト数。
  • waitfor: 非ブロッキング I/O が必要な場合に、待機すべき状態（WL_SOCKET_READABLE または WL_SOCKET_WRITEABLE）を格納する出力用ポインタ。
  • 戻り値: 読み込んだバイト数。エラーまたは非ブロッキング待機時は -1 を返す。

前提条件と不変条件

  • port->ssl が有効な OpenSSL セッションオブジェクトを指していること。
  • 呼び出し前に errno がクリアされていることが期待される。

アルゴリズムまたは振る舞い

1. ERR_clear_error() を呼び出し、OpenSSL のエラーキューを初期化する。 2. SSL_read() を実行し、TLS 経由でデータを取得する。 3. SSL_get_error() で結果を判定し、以下の通り処理する:

  • SSL_ERROR_NONE: 正常終了。
  • SSL_ERROR_WANT_READ / SSL_ERROR_WANT_WRITE: 非ブロッキング待機が必要と判断し、waitfor に対応するフラグを設定して errno を EWOULDBLOCK にセットする。
  • SSL_ERROR_SYSCALL: システムコールエラー。n が -1 でない場合は ECONNRESET を設定する。
  • SSL_ERROR_SSL: プロトコル違反として COMMERROR をログ出力し、ECONNRESET を設定する。
  • SSL_ERROR_ZERO_RETURN: ピアによる正常な接続終了。戻り値 0 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/libpq/be-secure.c (関数 be_tls_read を呼び出し)
  • 呼び出し先: OpenSSL ライブラリ (SSL_read, SSL_get_error, ERR_get_error, SSLerrmessage)

関連データ構造

  • Port: src/include/libpq/libpq-be.h で定義される接続状態管理構造体。

失敗モードと境界条件

  • 接続が切断された場合、SSL_ERROR_ZERO_RETURN により 0 が返される。
  • プロトコルエラーや予期せぬ SSL エラーが発生した場合、ereport(COMMERROR, ...) が実行され、errno が ECONNRESET に設定される。
  • 非ブロッキング I/O 発生時は errno が EWOULDBLOCK に設定される。

保守担当者向けメモ

  • OpenSSL のエラーキューはスレッドローカルであるため、エラー発生時の SSLerrmessage 呼び出しによるメッセージ取得は、エラー発生直後に行う必要がある。
  • SSL_ERROR_SYSCALL 発生時の errno は呼び出し元で処理されることを前提としている。
