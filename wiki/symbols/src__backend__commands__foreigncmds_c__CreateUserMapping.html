CreateUserMapping

定義

  • 種別: function
  • 定義箇所: src/backend/commands/foreigncmds.c:1115

目的

外部サーバーに対するユーザーマッピング（ユーザーと外部サーバー間の認証情報等の紐付け）をシステムカタログ pg_user_mapping に作成する。

シグネチャまたは形状

ObjectAddress CreateUserMapping(CreateUserMappingStmt *stmt)

入力と出力

  • **入力**: CreateUserMappingStmt *stmt（SQLの CREATE USER MAPPING 文の解析結果）
  • **出力**: 作成されたユーザーマッピングの ObjectAddress。IF NOT EXISTS 指定時に既に存在する場合は InvalidObjectAddress を返す。

前提条件と不変条件

  • 対象の外部サーバーが存在すること。
  • 呼び出し元ユーザーが対象サーバーに対する適切な権限（USAGE 等）を有していること。
  • サーバー内でユーザーマッピングが一意であること（重複時はエラーまたは NOTICE）。

アルゴリズムまたは振る舞い

1. pg_user_mapping を RowExclusiveLock でオープンする。 2. 指定されたロール（PUBLIC または特定のユーザー）の OID を取得する。 3. GetForeignServerByName でサーバーの存在を確認し、user_mapping_ddl_aclcheck で権限を検証する。 4. GetSysCacheOid2 を使用して、同一ユーザー・同一サーバーの組み合わせが既に存在しないか確認する。 5. transformGenericOptions を呼び出し、FDWのバリデーター関数を用いてオプションリストを検証・変換する。 6. heap_form_tuple で新しいタプルを作成し、CatalogTupleInsert でカタログに挿入する。 7. recordDependencyOn を使用して、サーバーに対する依存関係を記録する。 8. ユーザーが指定されている場合、recordDependencyOnOwner で所有者依存関係を記録する。 9. InvokeObjectPostCreateHook を呼び出し、作成後のフック処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c:1606（SQL解析後のユーティリティ実行フロー）
  • **呼び出し先**: table_open, get_rolespec_oid, GetForeignServerByName, user_mapping_ddl_aclcheck, GetSysCacheOid2, GetForeignDataWrapper, GetNewOidWithIndex, transformGenericOptions, heap_form_tuple, CatalogTupleInsert, recordDependencyOn, recordDependencyOnOwner, InvokeObjectPostCreateHook, table_close

関連データ構造

  • CreateUserMappingStmt: SQL文の構文解析木。
  • pg_user_mapping: ユーザーマッピング情報を保持するシステムカタログ。
  • ObjectAddress: 作成されたオブジェクトの識別子。

失敗モードと境界条件

  • **重複エラー**: 同一ユーザー・サーバーの組み合わせが既に存在する場合、IF NOT EXISTS がなければ ERRCODE_DUPLICATE_OBJECT でエラー終了する。
  • **権限エラー**: ユーザーマッピング作成に必要な権限がない場合、user_mapping_ddl_aclcheck がエラーを発生させる。
  • **バリデーションエラー**: transformGenericOptions 内でFDW固有のオプション検証に失敗した場合、エラー終了する。

保守担当者向けメモ

  • ユーザーマッピングは現在、拡張機能（Extension）のメンバーとしては扱われない仕様である。
  • オプションの変換ロジックは transformGenericOptions に依存しているため、バリデーションの挙動変更時は同関数を確認すること。
