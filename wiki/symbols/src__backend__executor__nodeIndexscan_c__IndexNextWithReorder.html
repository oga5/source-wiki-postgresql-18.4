IndexNextWithReorder

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexscan.c:168

目的

インデックススキャンにおいて、インデックスが提供する順序が不完全な場合や、ORDER BY 句の再評価が必要な場合に、pairingheap を用いてタプルの順序を正しく並べ替えて返却する。

シグネチャまたは形状

static TupleTableSlot *IndexNextWithReorder(IndexScanState *node);

入力と出力

  • **入力**: IndexScanState *node（スキャン状態を保持するノード構造体）
  • **出力**: TupleTableSlot *（順序が保証された次のタプル、またはスキャン終了時は空のスロット）

前提条件と不変条件

  • 前方スキャンのみサポートされる（indexorderdir および es_direction が前方であること）。
  • インデックススキャンが並列実行される場合、またはシリアル実行される場合の両方に対応する。
  • pairingheap 内のタプルは palloc されたコピーである必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: スキャン記述子（scandesc）が未初期化の場合、index_beginscan および index_rescan を実行する。 2. **再ソートキューの確認**: pairingheap が空でない場合、先頭要素の ORDER BY 値が最後に返した値以下であれば、キューからポップして返却する。 3. **インデックスからの取得**: index_getnext_slot でタプルを取得する。 4. **再評価**:

  • インデックスが損失性（xs_recheck）の場合、ExecQualAndReset でフィルタリングを行う。
  • ORDER BY の再チェックが必要な場合、EvalOrderByExpressions で値を再計算し、インデックスが返した値と照合する。

    • 5. **キューイング**: 順序が不正確な場合、またはキュー内に優先度の高いタプルが存在する場合は、タプルを pairingheap にプッシュし、次のループへ進む。 6. **返却**: 順序が確定したタプルを TupleTableSlot に格納して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecIndexScan（ExecScanAccessMtd 経由での呼び出しが確認されている）。
  • **呼び出し先**: index_beginscan, index_rescan, index_getnext_slot, pairingheap_first, reorderqueue_pop, reorderqueue_push, ExecQualAndReset, EvalOrderByExpressions, cmp_orderbyvals 等。

関連データ構造

  • IndexScanState: スキャンノードの状態管理。
  • ReorderTuple: 再ソート用キューに格納されるタプル構造体。
  • pairingheap: 順序保証のための優先度付きキュー。

失敗モードと境界条件

  • **順序エラー**: インデックスが返した順序が期待値より小さい場合、elog(ERROR, "index returned tuples in wrong order") を発生させる。
  • **スキャン終了**: インデックスからタプルが取得できず、かつキューも空になった場合、ExecClearTuple を返して終了する。
  • **割り込み**: CHECK_FOR_INTERRUPTS() により、処理中にクエリのキャンセル要求を検知する。

保守担当者向けメモ

  • 本関数は ORDER BY 句の再評価が必要な複雑なインデックススキャンを処理する。
  • pairingheap の操作は reorderqueue_pop / reorderqueue_push を介して行われるため、キューの整合性を維持すること。
  • インデックスが損失性である場合の再チェックロジックは、xs_recheck フラグに依存している。
