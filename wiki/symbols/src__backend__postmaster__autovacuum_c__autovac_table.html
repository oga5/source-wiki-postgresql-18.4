autovac_table

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/autovacuum.c:200

目的

自動バキューム（Autovacuum）ワーカーが特定のテーブルに対してバキュームまたはアナライズを実行する際に、対象テーブルの識別情報、実行パラメータ、およびコスト制御設定を保持するために使用されます。

シグネチャまたは形状

typedef struct autovac_table
{
    Oid            at_relid;
    VacuumParams   at_params;
    double         at_storage_param_vac_cost_delay;
    int            at_storage_param_vac_cost_limit;
    bool           at_dobalance;
    bool           at_sharedrel;
    char          *at_relname;
    char          *at_nspname;
    char          *at_datname;
} autovac_table;

入力と出力

本構造体は関数間で受け渡されるデータコンテナです。

  • **入力**: table_recheck_autovac 等の関数により、テーブルのOIDやストレージ設定から本構造体が構築されます。
  • **出力**: autovacuum_do_vac_analyze や autovac_report_activity に渡され、バキューム実行時の設定値やログ出力の参照先として利用されます。

前提条件と不変条件

  • at_relid は有効なテーブルのOIDである必要があります。
  • table_recheck_autovac によって生成される場合、名前フィールド（at_relname 等）が設定されない場合があります。

アルゴリズムまたは振る舞い

ワーカープロセスは、この構造体を参照して以下の処理を行います。 1. **実行制御**: at_params に基づき、バキュームの挙動（VACUUM/ANALYZEの別、オプション等）を決定します。 2. **コスト調整**: at_dobalance が真の場合、at_storage_param_vac_cost_delay および at_storage_param_vac_cost_limit を用いて、I/O負荷を動的に調整します。 3. **識別**: at_relname 等の文字列フィールドは、主にログ出力やプロセス状態の報告に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（生成）**: table_recheck_autovac が本構造体をメモリ上に割り当てて返します。
  • **呼び出し先（利用）**:
  • autovacuum_do_vac_analyze: バキューム実行のメイン処理。
  • autovac_report_activity: 現在の処理状況を報告する関数。

関連データ構造

  • VacuumParams: バキュームの実行オプションを保持する構造体。
  • HTAB: table_recheck_autovac 内でテーブル情報を検索する際に使用されるハッシュテーブル。

失敗モードと境界条件

  • メモリ不足時: palloc による割り当て失敗が発生する可能性があります。
  • 無効なOID: at_relid が存在しないテーブルを指している場合、後続のバキューム処理でエラーが発生します。

保守担当者向けメモ

  • 本構造体はワーカープロセス内での一時的な作業用データです。共有メモリ上の構造体（WorkerInfoData 等）とは異なり、プロセスローカルなメモリ領域で管理されます。
  • 名前フィールド（at_relname 等）は palloc されているため、不要になった際のメモリ解放に注意してください。
