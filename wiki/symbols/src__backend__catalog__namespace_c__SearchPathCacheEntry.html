SearchPathCacheEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/catalog/namespace.c:173

目的

SQLの search_path 設定に基づき、名前解決に使用されるスキーマリストと最終的な検索パスをキャッシュするためのデータ構造です。

シグネチャまたは形状

typedef struct SearchPathCacheEntry
{
    SearchPathCacheKey key;
    List       *oidlist;        /* ACLチェックを通過したスキーマOIDリスト */
    List       *finalPath;      /* 計算済みの最終的な検索パス */
    Oid         firstNS;        /* 明示的に指定された最初のスキーマ */
    bool        temp_missing;
    bool        forceRecompute; /* finalPathの再計算を強制するフラグ */
    char        status;         /* simplehash用ステータス */
} SearchPathCacheEntry;

入力と出力

  • 入力: SearchPathCacheKey をキーとしてハッシュテーブルに格納されます。
  • 出力: oidlist および finalPath を通じて、名前解決プロセスに有効なスキーマ順序を提供します。

前提条件と不変条件

  • oidlist に含まれるスキーマは、現在のセッションにおいてユーザーが USAGE 権限を持つものに限定されます。
  • simplehash ライブラリの要素として機能するため、status フィールドがハッシュテーブルの管理に使用されます。

アルゴリズムまたは振る舞い

  • namespace.c 内のキャッシュ機構において、検索パスの再計算コストを削減するために使用されます。
  • forceRecompute が設定されている場合、次回の検索パス解決時に finalPath の再計算がトリガーされます。
  • temp_missing フラグにより、一時テーブル用名前空間の欠落状態を追跡します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/namespace.c 内のキャッシュ検索および更新関数（SH_ELEMENT_TYPE として定義）。
  • 呼び出し先: 直接的な関数呼び出し関係はソースから特定できませんが、simplehash のマクロを通じてハッシュテーブル操作関数から参照されます。

関連データ構造

  • SearchPathCacheKey: キャッシュの検索キー。
  • List: PostgreSQLのリスト構造（oidlist, finalPath に使用）。
  • simplehash: 本構造体を要素として管理するハッシュテーブル実装。

失敗モードと境界条件

  • temp_missing が真の場合、一時テーブルの解決において特別な処理が必要となる可能性があります。
  • forceRecompute が適切に管理されない場合、古い検索パスが使用されるリスクがあります。

保守担当者向けメモ

  • 本構造体は simplehash を利用しているため、フィールドの追加や変更を行う際は lib/simplehash.h の要件に従う必要があります。
  • キャッシュの無効化ロジック（InvalidationCallback）との整合性に注意してください。
