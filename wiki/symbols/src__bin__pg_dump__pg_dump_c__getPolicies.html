getPolicies

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dump.c:4128

目的

データベース内の行レベルセキュリティ（RLS）ポリシー情報をシステムカタログから取得し、ダンプ対象の PolicyInfo オブジェクトを構築する。

シグネチャまたは形状

void getPolicies(Archive *fout, TableInfo tblinfo[], int numTables)

入力と出力

  • **入力**:
  • fout: アーカイブハンドル。
  • tblinfo[]: ダンプ対象テーブル情報の配列。
  • numTables: tblinfo 配列の要素数。
  • **出力**: なし（副作用として fout 内のダンプ対象オブジェクトリストに PolicyInfo を追加する）。

前提条件と不変条件

  • PostgreSQL 9.5 以降のバージョンであること。
  • ユーザーが --no-policies オプションを指定していないこと。
  • tblinfo に含まれるテーブルのうち、DUMP_COMPONENT_POLICY フラグが有効なもののみが処理対象となる。

アルゴリズムまたは振る舞い

1. **対象テーブルの特定**: tblinfo を走査し、RLS が有効なテーブルを特定して OID リストを作成する。 2. **RLS 有効化の表現**: RLS が有効なテーブルに対しては、polname が NULL の PolicyInfo オブジェクトを生成し、RLS が有効であることを記録する。 3. **ポリシー情報の取得**: 特定したテーブル OID を使用して pg_catalog.pg_policy をクエリし、ポリシー定義（権限、Qual、WithCheck 等）を取得する。 4. **オブジェクト構築**: 取得した各ポリシーに対して PolicyInfo 構造体を割り当て、ダンプ ID を割り当ててリストに保持する。 5. **クリーンアップ**: 使用したメモリバッファとクエリ結果を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/common.c:233
  • **呼び出し先**: ExecuteSqlQuery, pg_malloc, pg_strdup, findTableByOid, AssignDumpId, createPQExpBuffer 等のユーティリティ関数。

関連データ構造

  • PolicyInfo: ポリシー情報を保持する構造体。
  • TableInfo: テーブル情報を保持する構造体。
  • Archive: ダンプ処理のコンテキストを保持する構造体。

失敗モードと境界条件

  • remoteVersion < 90500 の場合、処理をスキップする。
  • dopt->no_policies が真の場合、処理をスキップする。
  • 対象テーブルが RELKIND_RELATION または RELKIND_PARTITIONED_TABLE 以外の場合、ポリシー取得の対象外とする。
  • サーバー側でポリシー定義を取得する際、ロックを保持していないテーブルに対して pg_get_expr を適用しないよう、サーバー側でのフィルタリングを行う。

保守担当者向けメモ

  • 本関数は pg_dump のスキーマ収集フェーズの一部である。
  • PolicyInfo オブジェクトの catId.tableoid は、RLS 有効化のみを示すオブジェクトの場合 0 に設定される。
  • バージョン 10 未満のサーバーに対しては polpermissive を 't' として扱う互換性処理が含まれている。
