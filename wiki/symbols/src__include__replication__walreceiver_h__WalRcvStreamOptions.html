WalRcvStreamOptions

定義

  • **種別**: struct
  • **定義箇所**: src/include/replication/walreceiver.h:167

目的

物理レプリケーションまたは論理レプリケーションのストリーミングを開始する際に必要なパラメータを保持する構造体です。

シグネチャまたは形状

typedef struct {
    bool        logical;        /* 物理/論理レプリケーションの識別 */
    char       *slotname;       /* レプリケーションスロット名 */
    XLogRecPtr  startpoint;     /* 開始位置 (LSN) */
    union {
        struct { TimeLineID startpointTLI; } physical;
        struct {
            uint32      proto_version;
            List       *publication_names;
            bool        binary;
            char       *streaming_str;
            bool        twophase;
            char       *origin;
        } logical;
    } proto;
} WalRcvStreamOptions;

入力と出力

本構造体は、レプリケーション開始時に設定値のコンテナとして機能します。

  • **入力**: 物理レプリケーションの場合は physical.startpointTLI、論理レプリケーションの場合はパブリケーション名やプロトコルバージョン等の詳細設定が入力されます。
  • **出力**: 構造体自体はデータ保持用であり、関数呼び出しを通じてストリーミング処理の挙動を制御します。

前提条件と不変条件

  • logical が true の場合は論理レプリケーション、false の場合は物理レプリケーションとして扱われます。
  • 共有メモリ上の構造体ではないため、プロセス間で共有される場合は適切な同期制御が必要です。

アルゴリズムまたは振る舞い

ストリーミング開始時に、この構造体の内容に基づいてプライマリサーバーへの接続およびストリーミング要求が構築されます。論理レプリケーションの場合、proto.logical 内の各フィールドがパブリッシャー側への要求パラメータとして使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/replication/logical/tablesync.c
  • src/backend/replication/logical/worker.c
  • src/backend/replication/walreceiver.c
  • **呼び出し先**:
  • src/backend/replication/libpqwalreceiver/libpqwalreceiver.c 内の関数（WalReceiverFunctions 実装）が、この構造体を引数として受け取り、通信プロトコルに応じた処理を実行します。

関連データ構造

  • XLogRecPtr: WALの開始位置を指す型。
  • TimeLineID: 物理レプリケーションで使用されるタイムライン識別子。
  • WalReceiverFunctionsType: 本構造体を利用して通信を行うインターフェース定義。

失敗モードと境界条件

  • slotname が指定されている場合、プライマリサーバー上に該当するスロットが存在しないとストリーミング開始時にエラーとなります。
  • startpoint が無効なLSNである場合、接続が拒否される可能性があります。

保守担当者向けメモ

  • 本構造体は libpqwalreceiver などの通信バックエンド実装と密結合しています。論理レプリケーションのパラメータ（twophase や origin など）を追加・変更する場合は、各通信バックエンド側の実装も更新する必要があります。
