dsm_detach

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/dsm.c:802

目的

動的共有メモリ（DSM）セグメントのデタッチ処理を行い、参照カウントがゼロに達した場合はセグメントの破棄を試行する。

シグネチャまたは形状

void dsm_detach(dsm_segment *seg)

入力と出力

  • **入力**: seg (デタッチ対象の dsm_segment 構造体へのポインタ)
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • seg は有効な dsm_segment を指している必要がある。
  • 割り込みを禁止した状態でコールバックを実行し、完了後に復帰させる。
  • 参照カウントが 1 の場合、そのセグメントはピン留め（pinned）されていない必要がある。

アルゴリズムまたは振る舞い

1. **コールバック実行**: seg->on_detach リストに登録された全コールバックを順次実行し、実行後にリストから削除する。 2. **マッピング解除**: seg->mapped_address が存在する場合、dsm_impl_op を用いてメモリマッピングを解除する。 3. **参照カウント更新**: DynamicSharedMemoryControlLock を取得し、制御セグメント内の参照カウントをデクリメントする。 4. **セグメント破棄**: 参照カウントが 1 になった場合、dsm_impl_op (DSM_OP_DESTROY) を呼び出してセグメントの破棄を試みる。成功時は制御セグメントのスロットを解放する。 5. **リソース解放**: ResourceOwner からの登録解除を行い、dsm_segment 構造体自体を pfree する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_prewarm/autoprewarm.c
  • src/backend/access/common/session.c
  • src/backend/access/transam/parallel.c
  • src/backend/replication/logical/applyparallelworker.c
  • src/backend/storage/ipc/dsm.c (内部のクリーンアップ処理)
  • **呼び出し先**:
  • dsm_impl_op (OS 依存のメモリ操作)
  • LWLockAcquire / LWLockRelease
  • ResourceOwnerForgetDSM
  • slist_pop_head_node / pfree 等のメモリ・リスト管理関数

関連データ構造

  • dsm_segment: バックエンドローカルなセグメント管理構造体。
  • dsm_control_header / dsm_control_item: 共有メモリ上の制御領域。
  • ResourceOwner: リソース管理用構造体。

失敗モードと境界条件

  • **マッピング解除の失敗**: マッピング解除に失敗しても、エラーを投げずに処理を続行する。
  • **破棄の失敗**: セグメントの破棄に失敗した場合、参照カウントは 1 のまま維持される。これは将来的な再試行（ポストマスター終了時など）に委ねられる。
  • **割り込み**: 割り込みによる中断を防ぐため、コールバック実行中は HOLD_INTERRUPTS() を使用する。

保守担当者向けメモ

  • 本関数はエラーを投げない設計となっている。これはトランザクションアボート時のクリーンアップ中にさらなるエラーが発生するのを防ぐためである。
  • is_main_region_dsm_handle で判定されるメイン領域の DSM は、OS 依存の操作ではなく FreePageManager を通じて管理される。
