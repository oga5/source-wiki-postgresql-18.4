save_ps_display_args

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/ps_status.c:118

目的

プロセス起動時に argv および環境変数領域を保護・確保し、後続のプロセス名（ps 表示用）更新処理でメモリを上書き可能にするための準備を行います。

シグネチャまたは形状

char **save_ps_display_args(int argc, char **argv);

入力と出力

  • **入力**: argc (引数の数), argv (引数文字列の配列)
  • **出力**: プロセス名更新用に安全にコピーされた、または再配置された argv 配列へのポインタを返します。

前提条件と不変条件

  • PS_USE_CLOBBER_ARGV が定義されている環境では、argv 領域を直接書き換えるため、環境変数の退避と argv のコピーが必須となります。
  • getopt() 等が argv へのポインタを保持している場合、本関数による argv の再配置が副作用を及ぼさないよう、呼び出しタイミングに注意が必要です。

アルゴリズムまたは振る舞い

1. **領域の特定**: PS_USE_CLOBBER_ARGV が有効な場合、argv とそれに続く環境変数領域がメモリ上で連続しているかを確認します。 2. **環境変数の退避**: argv 領域を拡張するために、既存の環境変数を別のメモリ領域へコピーし、environ ポインタを更新します。 3. **argv のコピー**: 引数解析時の整合性を保つため、argv 配列自体を新しいメモリ領域に複製します。 4. **プラットフォーム固有処理**: macOS (__darwin__) の場合、_NSGetArgv() を介して静的な argv ポインタを新しいコピーで更新します。 5. **バッファ設定**: ps_buffer に argv[0] の開始位置を設定し、書き込み可能な領域サイズを算出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/main/main.c (プロセス起動時の初期化フェーズ)
  • **呼び出し先**: malloc, strdup, exit, write_stderr (標準ライブラリおよびPostgreSQL内部関数)

関連データ構造

  • save_argc, save_argv: 呼び出し時の引数を保持する静的変数。
  • ps_buffer, ps_buffer_size: プロセス名更新に使用されるバッファとそのサイズ。
  • environ: プロセス環境変数配列。

失敗モードと境界条件

  • **メモリ不足**: malloc または strdup が失敗した場合、write_stderr にエラーを出力し、exit(1) でプロセスを終了します。
  • **連続性の欠如**: argv と環境変数がメモリ上で連続していない場合、ps_buffer を NULL に設定し、プロセス名の動的更新機能を無効化します。

保守担当者向けメモ

  • 本関数は OS 固有のメモリレイアウトに強く依存します。特に PS_USE_CLOBBER_ARGV が定義されている環境では、libc の実装（musl 等）による LD_LIBRARY_PATH の扱いなど、特殊な制約が含まれている点に注意してください。
  • プロセス名が更新されない場合は、まず update_process_title GUC 変数が有効かを確認し、次にプラットフォーム固有の PS_USE_... マクロが正しく選択されているかを確認してください。
