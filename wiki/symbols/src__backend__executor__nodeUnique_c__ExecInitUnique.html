ExecInitUnique

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeUnique.c:113

目的

Unique ノードの実行時状態である UniqueState 構造体を生成・初期化し、サブプランの初期化および重複排除に必要な比較関数の準備を行う。

シグネチャまたは形状

UniqueState *ExecInitUnique(Unique *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の Unique プランノード。
  • estate: クエリ実行状態を保持する EState。
  • eflags: 実行フラグ（EXEC_FLAG_BACKWARD および EXEC_FLAG_MARK はサポート外）。
  • **出力**:
  • 初期化済みの UniqueState 構造体へのポインタ。

前提条件と不変条件

  • 入力となるサブプランはソート済みである必要がある。
  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK が含まれていないこと（Assert により検証）。
  • Unique ノードは射影を行わないため、ps_ProjInfo は常に NULL に設定される。

アルゴリズムまたは振る舞い

1. UniqueState 構造体を割り当て、プランノードおよび実行状態をリンクする。 2. ExecProcNode 関数ポインタに ExecUnique を割り当てる。 3. ExecAssignExprContext を呼び出し、式評価用コンテキストを作成する。 4. ExecInitNode を呼び出し、サブプランを再帰的に初期化する。 5. ExecInitResultTupleSlotTL を用いて結果スロットを初期化する。 6. execTuplesMatchPrepare を呼び出し、node 内の列インデックス、演算子、照合順序に基づいた比較関数（eqfunction）を事前にコンパイルする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:351 （ExecInitNode 経由での呼び出しが示唆される）。
  • **呼び出し先**:
  • makeNode
  • ExecAssignExprContext
  • ExecInitNode
  • ExecInitResultTupleSlotTL
  • execTuplesMatchPrepare

関連データ構造

  • UniqueState: 実行時の状態を保持する主要構造体。
  • Unique: プランツリー上のノード定義。
  • EState: 実行状態。
  • TupleTableSlot: 結果格納用スロット。

失敗モードと境界条件

  • eflags にサポート外のフラグが指定された場合、デバッグビルドでは Assert により停止する。
  • サブプランの初期化に失敗した場合、そのエラーが伝播する。

保守担当者向けメモ

  • 本関数は Unique ノードのセットアップのみを担当する。実行時の重複排除ロジックについては ExecUnique を参照のこと。
  • eqfunction の準備には node->uniqColIdx 等のプラン情報が利用されるため、プランノードの整合性が重要である。
