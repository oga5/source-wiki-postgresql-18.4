load_backup_manifests

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/load_manifest.c:82

目的

複数のバックアップディレクトリからマニフェストファイルを読み込み、それぞれの内容を manifest_data 構造体の配列として取得する。

シグネチャまたは形状

manifest_data **load_backup_manifests(int n_backups, char **backup_directories)

入力と出力

  • **入力**:
  • n_backups (int): 処理対象のバックアップディレクトリ数。
  • backup_directories (char **): 各バックアップディレクトリのパスを格納した文字列配列。
  • **出力**:
  • manifest_data **: 読み込まれた各マニフェストデータを保持する manifest_data 構造体へのポインタ配列。

前提条件と不変条件

  • n_backups は 0 以上の整数であること。
  • backup_directories は少なくとも n_backups 個の有効なディレクトリパスを指していること。
  • 戻り値の配列は pg_malloc によって確保されるため、呼び出し元で適切に解放する必要がある。

アルゴリズムまたは振る舞い

1. n_backups の数に基づき、manifest_data * を格納するためのメモリ領域を pg_malloc で確保する。 2. 指定されたディレクトリ数分ループし、各ディレクトリに対して load_backup_manifest を呼び出す。 3. 各ディレクトリから取得した manifest_data へのポインタを結果配列に格納する。 4. 全ての処理完了後、構築された配列の先頭ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_combinebackup/pg_combinebackup.c:291
  • **呼び出し先**: load_backup_manifest (同ファイル内)

関連データ構造

  • manifest_data: 各バックアップのマニフェスト情報を保持する構造体。

失敗モードと境界条件

  • n_backups が 0 の場合、空の配列（または NULL）が返される。
  • load_backup_manifest が内部でエラー（ファイル読み込み失敗やパースエラー等）を発生させた場合、その挙動は load_backup_manifest の実装に依存する。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、個別のマニフェスト読み込みロジックは load_backup_manifest に集約されている。
  • メモリ管理は呼び出し元に委ねられているため、メモリリークを防ぐための解放処理が呼び出し側で正しく実装されているか確認すること。
