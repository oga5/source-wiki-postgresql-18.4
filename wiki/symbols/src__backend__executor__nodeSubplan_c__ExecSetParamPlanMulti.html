ExecSetParamPlanMulti

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSubplan.c:1259

目的

指定されたビットマップセットに含まれる複数のパラメータ（InitPlan）を順次評価し、実行コンテキスト内の値を更新する。

シグネチャまたは形状

void ExecSetParamPlanMulti(const Bitmapset *params, ExprContext *econtext)

入力と出力

  • params: 評価対象のパラメータIDを含む Bitmapset。
  • econtext: 実行コンテキスト（ExprContext）。パラメータ値の格納先である ecxt_param_exec_vals を保持する。
  • 出力: なし（副作用として econtext 内のパラメータ値が更新される）。

前提条件と不変条件

  • econtext->ecxt_param_exec_vals が有効であること。
  • params に含まれる各 paramid に対応する ParamExecData が存在すること。
  • ExecSetParamPlan の呼び出し後、対象の execPlan が NULL になることが期待される。

アルゴリズムまたは振る舞い

1. bms_next_member を使用して、params 内のすべての paramid を反復処理する。 2. 各 paramid に対応する ParamExecData を econtext から取得する。 3. execPlan が設定されている（未評価の）場合、ExecSetParamPlan を呼び出してサブプランを実行し、パラメータ値を確定させる。 4. 評価完了後、prm->execPlan が NULL になっていることを Assert で検証する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/executor/execMain.c
  • src/backend/executor/execParallel.c
  • 呼び出し先:
  • ExecSetParamPlan

関連データ構造

  • Bitmapset: 評価すべきパラメータIDの集合。
  • ExprContext: 実行時の式評価コンテキスト。
  • ParamExecData: パラメータの実行状態および値を保持する構造体。

失敗モードと境界条件

  • params が空の場合、何も処理を行わず正常終了する。
  • prm->execPlan が既に NULL の場合、そのパラメータはスキップされる。

保守担当者向けメモ

  • 本関数は、クエリ実行の初期化フェーズや並列クエリの実行準備において、依存するサブプランを一括で解決するために使用される。
  • ExecSetParamPlan の呼び出しにより、サブプランの実行結果が ParamExecData に格納されるため、呼び出し順序や依存関係の整合性に注意すること。
