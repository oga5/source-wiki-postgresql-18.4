pg_read_binary_file_common

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/genfile.c:259

目的

SQLインターフェースからサーバー上のバイナリファイルを読み込むための共通ラッパー関数です。パスの検証と権限チェックを経て、ファイル内容を bytea 型として取得します。

シグネチャまたは形状

static bytea *pg_read_binary_file_common(text *filename_t, int64 seek_offset, int64 bytes_to_read, bool read_to_eof, bool missing_ok)

入力と出力

  • **入力**:
  • filename_t: 読み込むファイルパス（text型）。
  • seek_offset: 読み込み開始位置（バイト単位）。
  • bytes_to_read: 読み込むバイト数。
  • read_to_eof: true の場合、オフセットから末尾まで全て読み込む。
  • missing_ok: true の場合、ファイル不在時にエラーとせず NULL を返す。
  • **出力**:
  • 読み込んだファイル内容を保持する bytea * 型のデータ。

前提条件と不変条件

  • read_to_eof が true の場合、bytes_to_read は -1 でなければならない（Assertによる検証）。
  • read_to_eof が false の場合、bytes_to_read は負の値であってはならない（エラー報告）。

アルゴリズムまたは振る舞い

1. 引数の整合性を検証します。 2. convert_and_check_filename を呼び出し、指定されたパスの正規化と pg_read_server_files ロールによる権限チェックを行います。 3. 検証済みのパスを使用して read_binary_file を呼び出し、ファイルシステムからデータを読み込みます。 4. 読み込まれたデータを bytea 型として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/genfile.c 内の各SQL関数ラッパー（pg_read_binary_file_off_len, pg_read_binary_file_off_len_missing, pg_read_binary_file_all 等）。
  • **呼び出し先**:
  • convert_and_check_filename（パス検証）
  • read_binary_file（低レベルファイル読み込み）

関連データ構造

  • bytea: PostgreSQLの可変長バイナリデータ型。
  • text: PostgreSQLの文字列型。

失敗モードと境界条件

  • **権限エラー**: convert_and_check_filename がパスのアクセス権限違反を検知した場合、エラーが発生します。
  • **パラメータエラー**: bytes_to_read が負の値（かつ read_to_eof が偽）の場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーとなります。
  • **ファイル不在**: missing_ok が false の場合、ファイルが存在しないとエラーになります。true の場合は NULL が返されます。

保守担当者向けメモ

  • 本関数はSQL関数から呼び出される複数のバリエーションの共通基盤です。修正時は、権限チェックのバイパスが発生しないよう convert_and_check_filename の呼び出しを維持してください。
  • read_to_eof フラグと bytes_to_read の関係性は、opr_sanity チェックをパスするために厳密に管理されています。
