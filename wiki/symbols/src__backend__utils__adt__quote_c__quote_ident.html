quote_ident

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/quote.c:24

目的

SQL識別子（テーブル名や列名など）を安全にクォートし、SQL文内で使用可能な形式に変換します。SQLインジェクションを防ぐために使用されます。

シグネチャまたは形状

Datum quote_ident(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る text 型の識別子。
  • **出力**: 二重引用符で囲まれ、必要に応じて内部の引用符がエスケープされた text 型の文字列。

前提条件と不変条件

  • 入力された text 型データは、text_to_cstring によって C 文字列に変換可能であること。
  • 内部的に呼び出される quote_identifier が、識別子としての安全性を保証する変換を行うこと。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP(0) を使用して入力引数を text 型として取得します。 2. text_to_cstring を呼び出し、入力データを C 文字列に変換します。 3. quote_identifier を呼び出し、識別子を二重引用符で囲むなどのエスケープ処理を実行します。 4. 変換された文字列を cstring_to_text で text 型に戻し、PG_RETURN_TEXT_P を介して返却します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • text_to_cstring
  • quote_identifier
  • cstring_to_text
  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • src/bin/pg_dump/pg_dump.c
  • src/bin/psql/describe.c
  • その他、SQLクエリを動的に構築するバックエンドおよびクライアントツール。

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • メモリ不足時: palloc 等の内部メモリ確保が失敗した場合、PostgreSQLの標準的なエラーハンドリングによりアボートされます。
  • 入力値がNULLの場合: PG_GETARG_TEXT_PP の仕様に基づき、NULL入力に対する挙動が決定されます。

保守担当者向けメモ

  • 本関数は quote_identifier へのラッパーとして機能しています。識別子のクォートルールを変更する場合は、quote_identifier の実装を確認してください。
  • quote_literal とはエスケープのルール（二重引用符 vs 単一引用符）が異なるため、混同しないよう注意してください。
