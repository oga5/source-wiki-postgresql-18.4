cluster_rel

定義

  • 種別: function
  • 定義箇所: src/backend/commands/cluster.c:310

目的

単一のリレーション（テーブルまたはマテリアライズドビュー）に対して、指定されたインデックスに基づく物理的なクラスタリング、または VACUUM FULL に相当する物理再構築を実行する。

シグネチャまたは形状

void cluster_rel(Relation OldHeap, Oid indexOid, ClusterParams *params)

入力と出力

  • **入力**:
  • OldHeap: 対象リレーションのオープン済み Relation 構造体。
  • indexOid: クラスタリングに使用するインデックスの OID（VACUUM FULL の場合は InvalidOid）。
  • params: 実行オプション（CLUOPT_VERBOSE, CLUOPT_RECHECK 等）を含む ClusterParams 構造体。
  • **出力**: なし（副作用としてリレーションの物理再構築を行う）。

前提条件と不変条件

  • OldHeap に対して AccessExclusiveLock が取得されている必要がある。
  • 呼び出し元でトランザクション管理が行われていること（本関数は単一テーブルの再構築を完結させる）。
  • 共有カタログのクラスタリングは禁止されている。

アルゴリズムまたは振る舞い

1. **初期化**: プログレスモニタの開始、実行ユーザー権限の切り替え、GUC 設定のネストレベル保存を行う。 2. **検証 (recheck 時)**: 権限確認、一時テーブルの除外、インデックスの存在確認および indisclustered フラグの整合性チェックを行う。 3. **制約チェック**: 共有カタログや他セッションの一時テーブルに対する操作をエラーとして拒否する。 4. **使用状況確認**: CheckTableNotInUse により、対象テーブルに対するアクティブなスキャンや保留中のトリガーイベントがないか確認する。 5. **ロック昇格**: 物理再構築に伴い、タプルやページに対する述語ロックをリレーション全体のロックへ昇格させる。 6. **再構築**: rebuild_relation を呼び出し、新規ヒープ作成、データコピー、ファイルスワップを実行する。 7. **終了処理**: GUC 設定のロールバック、ユーザー権限の復元、プログレスモニタの終了を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/cluster.c (複数リレーションのクラスタリング処理)
  • src/backend/commands/vacuum.c (VACUUM FULL 実行時)
  • **呼び出し先**:
  • rebuild_relation (再構築のメイン処理)
  • check_index_is_clusterable (インデックスの妥当性検証)
  • TransferPredicateLocksToHeapRelation (述語ロックの昇格)
  • index_open (インデックスのオープン)

関連データ構造

  • Relation: 対象テーブルのメタデータ。
  • ClusterParams: 実行オプションを保持。

失敗モードと境界条件

  • **エラー**: 共有カタログのクラスタリング、他セッションの一時テーブル操作、使用中のテーブルに対する操作は ereport(ERROR, ...) により中断される。
  • **スキップ**: マテリアライズドビューが未投入（populated）の場合、エラーを発生させずに処理を終了する。
  • **中断**: CHECK_FOR_INTERRUPTS() により、ユーザーによる中断要求を検知する。

保守担当者向けメモ

  • 本関数は OldHeap をクローズし、インデックスが指定されている場合はそれもクローズする。
  • recheck フラグが有効な場合、複数テーブル処理の合間にリレーションの状態が変化していないかを厳密に検証する。
  • ファイルスワップは rebuild_relation 内で行われるため、物理的な整合性に関する変更を行う際は rebuild_relation および finish_heap_swap の挙動を併せて確認すること。
