parse_scram_secret

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-scram.c:599

目的

PostgreSQLのSCRAM認証シークレット文字列を解析し、認証に必要なパラメータ（イテレーション数、ハッシュ型、ソルト、StoredKey、ServerKey）を抽出する。

シグネチャまたは形状

bool parse_scram_secret(const char *secret, int *iterations,
                        pg_cryptohash_type *hash_type, int *key_length,
                        char **salt, uint8 *stored_key, uint8 *server_key)

入力と出力

  • **入力**:
  • secret: SCRAM-SHA-256$<iterations>:<salt>$<storedkey>:<serverkey> 形式の文字列。
  • **出力**:
  • 成功時: true を返し、各ポインタ引数に解析結果を格納する。
  • 失敗時: false を返し、*salt を NULL に設定する。

前提条件と不変条件

  • secret 文字列は指定された区切り文字（$, :）を含む正しい形式である必要がある。
  • stored_key および server_key バッファは、SCRAM_SHA_256_KEY_LEN 分のメモリが確保されている必要がある。

アルゴリズムまたは振る舞い

1. 入力文字列を strsep を用いて各フィールドに分割する。 2. スキームが SCRAM-SHA-256 であることを確認し、ハッシュ型とキー長を設定する。 3. イテレーション数を整数に変換する。 4. ソルト、StoredKey、ServerKeyをBase64デコードする。 5. デコードされたキー長が期待値（SCRAM_SHA_256_KEY_LEN）と一致するか検証する。 6. 成功時に各値を引数経由で呼び出し元へ渡す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth-scram.c (複数箇所)
  • src/backend/libpq/crypt.c:103
  • **呼び出し先**:
  • strsep, pstrdup, palloc, pg_b64_decode, pg_b64_dec_len, strcmp, strtol, memcpy

関連データ構造

  • pg_cryptohash_type: ハッシュアルゴリズムの識別子。

失敗モードと境界条件

  • 形式が不正な場合（区切り文字の欠落、スキーム不一致、数値変換失敗、Base64デコード失敗、キー長不一致）は false を返す。
  • メモリ割り当て失敗時は palloc によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は SCRAM-SHA-256 専用の実装である。将来的に他のハッシュアルゴリズムをサポートする場合は、スキームチェックおよびキー長の処理を拡張する必要がある。
  • Base64デコード後のバッファは palloc されているため、呼び出し元でのメモリ管理に注意すること。
