IndexTupleHasNulls

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/itup.h:77

目的

インデックスタプル（IndexTupleData）内に NULL 値が含まれているかどうかを判定する。

シグネチャまたは形状

static inline bool IndexTupleHasNulls(const IndexTupleData *itup)

入力と出力

  • **入力**: itup - 検査対象の IndexTupleData 構造体へのポインタ。
  • **出力**: bool - NULL 値が存在する場合は true、存在しない場合は false を返す。

前提条件と不変条件

  • itup は有効な IndexTupleData 構造体を指している必要がある。
  • t_info フィールドが正しく初期化されていることを前提とする。

アルゴリズムまたは振る舞い

itup->t_info（タプルのメタ情報ビットマスク）に対して INDEX_NULL_MASK をビット単位の AND 演算で適用し、その結果が非ゼロであるかどうかを判定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pageinspect/btreefuncs.c
  • src/backend/access/common/indextuple.c
  • src/backend/access/gin/ginentrypage.c
  • src/backend/access/spgist/spgutils.c
  • src/include/access/itup.h
  • **呼び出し先**:
  • 本関数は単一のビット演算のみを行うため、他の関数を呼び出さない。

関連データ構造

  • IndexTupleData: インデックスタプルのヘッダー構造体。t_info フィールドを保持する。

失敗モードと境界条件

  • itup が NULL ポインタである場合、未定義の動作を引き起こす可能性がある。
  • INDEX_NULL_MASK が定義されていない環境ではコンパイルエラーとなる（itup.h 内で定義されている前提）。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、インライン関数として定義されている。ロジックを変更する場合は、インデックスの物理レイアウト（t_info のビット定義）との整合性に注意すること。
