load_file

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/dfmgr.c:148

目的

指定された共有ライブラリをプロセス空間にロードします。ライブラリ名の解決、セキュリティ制限の適用、およびロード済みリストへの登録を統合的に行います。

シグネチャまたは形状

void load_file(const char *filename, bool restricted)

入力と出力

  • **入力**:
  • filename: ロード対象のライブラリ名（省略形またはパス）。
  • restricted: セキュリティ制限を適用するかどうかを示すフラグ。
  • **出力**: なし（成功時はライブラリがロードされ、失敗時はエラーが発生します）。

前提条件と不変条件

  • restricted が true の場合、ライブラリ名は check_restricted_library_name による検証を通過する必要があります。
  • 内部的に expand_dynamic_library_name を使用して、ライブラリ名を絶対パスへ解決します。

アルゴリズムまたは振る舞い

1. **セキュリティチェック**: restricted が真であれば、check_restricted_library_name を呼び出し、不正なパス指定を拒否します。 2. **パス解決**: expand_dynamic_library_name を呼び出し、dynamic_library_path 設定に基づいてライブラリの絶対パスを取得します。 3. **ロード実行**: internal_load_library を呼び出し、ライブラリをロードします。既にロード済みの場合は、重複ロードを回避します。 4. **クリーンアップ**: 解決されたパス文字列（fullname）を pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/subscriptioncmds.c (複数箇所)
  • src/backend/replication/logical/slotsync.c
  • src/backend/replication/logical/worker.c
  • src/backend/replication/slotfuncs.c
  • src/backend/replication/walreceiver.c
  • src/backend/tcop/utility.c
  • **呼び出し先**:
  • check_restricted_library_name
  • expand_dynamic_library_name
  • internal_load_library
  • pfree

関連データ構造

  • DynamicFileList: internal_load_library 内でロード済みライブラリの追跡に使用されます。

失敗モードと境界条件

  • 指定されたライブラリが見つからない場合や、ABI 互換性（PG_MODULE_MAGIC）が確認できない場合、internal_load_library 経由でエラーが発生します。
  • セキュリティ制限に違反するパスが指定された場合、check_restricted_library_name により処理が中断されます。

保守担当者向けメモ

  • 本関数はライブラリのロードを管理する高レベルなインターフェースです。
  • ロードされたライブラリはプロセス終了までアンロードされない設計である点に注意してください。
