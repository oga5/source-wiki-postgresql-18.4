ECPGtype

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/preproc/type.h:17

目的

ECPG（Embedded SQL for C）プリプロセッサにおいて、ホスト変数の型情報を保持・管理するための主要なデータ構造です。C言語の型とSQLの型をマッピングし、プリプロセッサがソースコードを解析する際の型情報を抽象化します。

シグネチャまたは形状

struct ECPGtype {
    enum ECPGttype type;
    char *type_name;
    char *size;
    char *struct_sizeof;
    union {
        struct ECPGtype *element;
        struct ECPGstruct_member *members;
    } u;
    int counter;
};

入力と出力

  • **入力**: プリプロセッサの解析過程で ECPGmake_* 系関数（ECPGmake_simple_type, ECPGmake_array_type 等）を介して構築されます。
  • **出力**: ECPGdump_a_type 関数等により、解析された型情報がCコードとして出力されます。

前提条件と不変条件

  • type_name は構造体や共用体の場合にその名称を保持します。
  • size は配列の場合は要素数、varchar 型の場合は最大サイズを文字列として保持します。
  • struct_sizeof は構造体の場合、sizeof() の結果を文字列として保持します。
  • u 共用体は、配列の場合は element（要素の型）、構造体や共用体の場合は members（メンバーリスト）を指します。

アルゴリズムまたは振る舞い

  • **再帰的表現**: element や members を通じて再帰的な型構造（配列内の構造体など）を表現可能です。
  • **メモリ管理**: 動的に確保されるため、適切な解放処理が必要です。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/interfaces/ecpg/preproc/descriptor.c: descriptor_type の初期化および loc_alloc による動的確保。
  • src/interfaces/ecpg/preproc/preproc_extern.h: ecpg_no_indicator の外部宣言や check_indicator 関数の引数。
  • src/interfaces/ecpg/preproc/type.c: 型生成関数内での利用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • enum ECPGttype: type フィールドで使用される基本型定義。
  • struct ECPGstruct_member: 構造体や共用体のメンバーを管理する連結リスト。

失敗モードと境界条件

  • メモリ不足時に mm_alloc 等が失敗する可能性があります。
  • 再帰的な型定義を辿る際、循環参照や過度に深いネストが発生した場合の境界条件については、解析ロジック側の実装に依存します。

保守担当者向けメモ

  • 本構造体は mm_alloc で動的に確保されるため、メモリリークを防ぐために ECPGfree_* 関数による適切な解放を徹底してください。
  • u 共用体の使い分けは type フィールドの値に依存するため、アクセス時には必ず型チェックを行ってください。
