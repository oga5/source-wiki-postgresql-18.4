tar_free

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/walmethods.c:1335

目的

tar 形式の WAL 書き込みメソッドに関連するリソースを解放し、メモリリークを防ぐ。

シグネチャまたは形状

static void tar_free(WalWriteMethod *wwmethod)

入力と出力

  • 入力: WalWriteMethod *wwmethod - 解放対象の WalWriteMethod 構造体へのポインタ。
  • 出力: なし（void）

前提条件と不変条件

  • wwmethod は TarMethodData 構造体としてキャスト可能であること。
  • wwmethod が指すメモリ領域は、pg_free による解放が有効な状態であること。

アルゴリズムまたは振る舞い

1. wwmethod を TarMethodData 型にキャストする。 2. tarfilename メンバを pg_free で解放する。 3. HAVE_LIBZ が定義されており、かつ圧縮アルゴリズムが PG_COMPRESSION_GZIP である場合、zlibOut メンバを pg_free で解放する。 4. 最後に wwmethod 自体を pg_free で解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_basebackup/walmethods.c 内の WalWriteMethodOps 構造体（free コールバックとして登録）。
  • 呼び出し先: pg_free（メモリ解放用ユーティリティ）。

関連データ構造

  • WalWriteMethod: 抽象基底構造体。
  • TarMethodData: tar メソッド固有のデータを保持する構造体。

失敗モードと境界条件

  • wwmethod が NULL の場合、pg_free の仕様に依存する（通常は NULL チェックが行われる）。
  • 圧縮設定が PG_COMPRESSION_GZIP 以外の場合、zlibOut の解放はスキップされる。

保守担当者向けメモ

  • 本関数は WalWriteMethodOps の free ポインタ経由で呼び出されるため、直接呼び出すのではなく、インターフェースを介したクリーンアップフローの一部として維持すること。
