DirectFunctionCall5Coll

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/fmgr.c:885

目的

5つの引数を取る関数を、FmgrInfo構造体の初期化を介さずに直接呼び出すための高速なラッパー関数です。

シグネチャまたは形状

Datum DirectFunctionCall5Coll(PGFunction func, Oid collation, Datum arg1, Datum arg2, Datum arg3, Datum arg4, Datum arg5)

入力と出力

  • **入力**:
  • func: 呼び出す関数のポインタ (PGFunction)。
  • collation: 照合順序 (Oid)。
  • arg1 ～ arg5: 関数に渡す5つの引数 (Datum)。
  • **出力**:
  • 関数の実行結果を Datum 型で返します。

前提条件と不変条件

  • 呼び出される関数は、NULLを返さないことが期待されます。
  • 引数はすべて非NULLとして扱われます。

アルゴリズムまたは振る舞い

1. LOCAL_FCINFO マクロを使用して、スタック上に FunctionCallInfoBaseData を確保します。 2. InitFunctionCallInfoData を呼び出し、引数数5、指定された照合順序で初期化します。 3. 5つの引数 (arg1～arg5) を fcinfo->args 配列に格納し、各 isnull フラグを false に設定します。 4. 関数ポインタ func を実行します。 5. 実行結果の fcinfo->isnull を確認し、関数がNULLを返した場合は elog(ERROR) を発生させます。 6. 正常終了時に結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/selfuncs.c:2897
  • src/include/fmgr.h:691 (マクロ経由)
  • **呼び出し先**:
  • 引数として渡された PGFunction ポインタが指す関数。

関連データ構造

  • FunctionCallInfoBaseData: 実行時の引数やコンテキストを保持する構造体。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • **NULL戻り値**: 呼び出された関数がNULLを返した場合、elog(ERROR) によりエラー終了します。
  • **引数制限**: 5つを超える引数が必要な場合は、別の DirectFunctionCall バリアントを使用する必要があります。

保守担当者向けメモ

  • 本関数はパフォーマンス重視のパスで使用されます。ポリモーフィック関数など、FmgrInfo の完全な初期化が必要な関数には使用できません。
  • fcinfo の初期化において NULL を明示的に渡している箇所があるため、複雑な関数呼び出しには適さない点に注意してください。
