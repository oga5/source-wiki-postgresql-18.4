gettoken_query_websearch

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery.c:397

目的

Web検索スタイルのクエリ文字列をトークン化し、tsquery 構築のための演算子またはオペランドを順次抽出する。

シグネチャまたは形状

ts_tokentype gettoken_query_websearch(
    TSQueryParserState state, 
    int8 *operator, 
    int *lenval, 
    char **strval, 
    int16 *weight, 
    bool *prefix
)

入力と出力

  • **入力**:
  • state: 解析状態を保持する TSQueryParserState 構造体。
  • **出力**:
  • operator: 抽出された演算子（OP_NOT, OP_OR, OP_AND）。
  • lenval: トークン文字列の長さ。
  • strval: トークン文字列へのポインタ。
  • weight: 常に 0 に初期化される。
  • prefix: 常に false に初期化される。
  • **戻り値**: 抽出結果の種別（PT_OPR（演算子）, PT_VAL（値）, PT_END（終了）, PT_ERR（エラー））。

前提条件と不変条件

  • weight および prefix は常にデフォルト値（0, false）にリセットされる。
  • state->state に基づき、次に期待されるトークン（オペランドまたは演算子）を決定する。
  • 引用符（"）で囲まれた文字列は、内部のスペースを含めて単一のトークンとして扱われる。

アルゴリズムまたは振る舞い

1. **状態遷移**: state->state に応じて処理を分岐する。 2. **WAITOPERAND / WAITFIRSTOPERAND**:

  • - を検出した場合: OP_NOT を返し、状態を WAITOPERAND に更新。
  • " を検出した場合: 閉じ引用符までを単一のトークンとして抽出し、PT_VAL を返す。
  • 演算子を検出した場合: 無視してスキップ。
  • それ以外: tsvector パーサーを呼び出し、値を抽出。抽出失敗時は PT_END または PT_ERR を返す。

    • 3. **WAITOPERATOR**:

  • 文字列終端: PT_END を返す。
  • OR 演算子: parse_or_operator を呼び出し、成功すれば OP_OR を返す。
  • 空白以外: 暗黙の AND 演算子として OP_AND を返す。

    • 4. **ループ**: 処理が完了するまで、マルチバイト文字の長さを考慮してバッファポインタを進める。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery.c 内のパーサー初期化箇所（state.gettoken = gettoken_query_websearch;）。
  • **呼び出し先**:
  • t_iseq, ISOPERATOR, reset_tsvector_parser, gettoken_tsvector, parse_or_operator, pushStop, pg_mblen_cstr。

関連データ構造

  • TSQueryParserState: 解析中のトークン、バッファ、状態を保持する構造体。
  • ts_tokentype: 戻り値の型（PT_OPR, PT_VAL, PT_END, PT_ERR）。

失敗モードと境界条件

  • gettoken_tsvector がソフトエラーを報告した場合、PT_ERR を返す。
  • 引用符が閉じられていない場合、文字列の末尾までをトークンとして処理する。
  • 演算子の連続や不正な配置は、パーサーのロジックにより無視または暗黙の AND として処理される。

保守担当者向けメモ

  • 本関数は tsquery の Web 検索モード専用のトークナイザーである。
  • tsvector パーサーに依存しているため、tsvector 側の解析ロジック変更の影響を受ける可能性がある。
  • 演算子の優先順位や暗黙の AND 挿入ロジックは、この関数の状態遷移に強く依存している。
