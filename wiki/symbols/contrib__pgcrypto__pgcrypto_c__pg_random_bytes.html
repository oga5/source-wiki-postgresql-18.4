pg_random_bytes

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgcrypto.c:456

目的

指定されたバイト数の暗号論的疑似乱数を生成し、bytea 型として返却する SQL インターフェース関数。

シグネチャまたは形状

Datum pg_random_bytes(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る int4 型の長さ（バイト数）。
  • **出力**: bytea 型の乱数データ。

前提条件と不変条件

  • 入力長（len）は 1 以上 1024 以下であること。範囲外の場合はエラーをスローする。
  • 乱数生成器が利用可能であること。

アルゴリズムまたは振る舞い

1. 引数から生成するバイト数 len を取得する。 2. len が 1 未満または 1024 を超える場合、ERRCODE_EXTERNAL_ROUTINE_INVOCATION_EXCEPTION を伴うエラーを発生させる。 3. palloc を使用して VARHDRSZ + len 分のメモリを確保し、SET_VARSIZE でサイズを設定する。 4. pg_strong_random を呼び出し、確保したバッファに乱数を書き込む。 5. 乱数生成に失敗した場合は px_THROW_ERROR(PXE_NO_RANDOM) を実行する。 6. 成功時は PG_RETURN_BYTEA_P を介して結果を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1(pg_random_bytes) により登録）。
  • **呼び出し先**: pg_strong_random（乱数生成）、palloc（メモリ確保）、px_THROW_ERROR（エラーハンドリング）。

関連データ構造

  • bytea: PostgreSQL の可変長バイト列型。

失敗モードと境界条件

  • **範囲外エラー**: len < 1 または len > 1024 の場合に発生。
  • **乱数生成失敗**: システムの乱数ソースが利用できない場合に PXE_NO_RANDOM エラーが発生。

保守担当者向けメモ

  • 本関数は pgcrypto モジュールの SQL ラッパーであり、実際の乱数生成ロジックは pg_strong_random に依存している。
  • メモリ管理は PostgreSQL の palloc に従っており、呼び出し元で適切に解放されることが前提となっている。
