proclist_node_get

定義

  • 種別: function
  • 定義箇所: src/include/storage/proclist.h:47

目的

共有メモリ上の PGPROC 構造体内に埋め込まれた proclist_node へのポインタを、プロセス番号（procno）と構造体内のオフセット値から算出する。

シグネチャまたは形状

static inline proclist_node *proclist_node_get(int procno, size_t node_offset)

入力と出力

  • **入力**:
  • procno: 対象となるプロセスの識別番号。
  • node_offset: PGPROC 構造体の先頭から proclist_node メンバまでのバイトオフセット。
  • **出力**:
  • 指定されたプロセスに対応する proclist_node 構造体へのポインタ。

前提条件と不変条件

  • procno は有効なプロセスを指している必要がある（GetPGProcByNumber が有効なアドレスを返すこと）。
  • node_offset は、対象の PGPROC 構造体内で proclist_node が配置されている正しいオフセットであること。

アルゴリズムまたは振る舞い

1. GetPGProcByNumber(procno) を呼び出し、共有メモリ上の対象 PGPROC 構造体のベースアドレスを取得する。 2. 取得したベースアドレスに node_offset を加算し、proclist_node 型のポインタとしてキャストして返す。

呼び出し元と呼び出し先

  • **呼び出し先**: GetPGProcByNumber（storage/proc.h で定義）。
  • **呼び出し元**: src/include/storage/proclist.h 内のリスト操作関数（proclist_push_head_offset, proclist_delete_offset 等）およびイテレータマクロから頻繁に呼び出される。

関連データ構造

  • PGPROC: プロセス管理用の主要構造体。
  • proclist_node: 双方向リストの前後関係を保持する構造体。

失敗モードと境界条件

  • procno が無効な場合、GetPGProcByNumber の振る舞いに依存する（通常は無効なアドレスへのアクセスが発生する可能性がある）。
  • node_offset が不正な場合、メモリ上の誤った位置を指すことになる。

保守担当者向けメモ

  • 本関数はインライン関数であり、パフォーマンスを重視して設計されている。
  • 共有メモリ環境下で絶対アドレスがプロセスごとに異なることを考慮し、procno を基点とした相対的なアドレス解決を行う設計となっている。
