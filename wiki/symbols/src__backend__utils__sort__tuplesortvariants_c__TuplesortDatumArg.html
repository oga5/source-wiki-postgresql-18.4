TuplesortDatumArg

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/sort/tuplesortvariants.c:156

目的

Datum 型のソート処理において、対象となるデータ型の識別情報およびコピーに必要なメタデータを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    Oid    datumType;    /* ソート対象 Datum のデータ型 OID */
    int    datumTypeLen; /* Datum のコピーに必要な型長 */
} TuplesortDatumArg;

入力と出力

本構造体は tuplesort_begin_datum 等の初期化処理において、ソート状態を管理する Tuplesortstate の arg フィールドに格納される設定情報として機能します。

前提条件と不変条件

  • datumType には有効な PostgreSQL の型 OID が設定されている必要があります。
  • datumTypeLen は、メモリ上での Datum のコピーおよび管理を正しく行うために、対象型に応じた適切なバイト長である必要があります。

アルゴリズムまたは振る舞い

ソート実行時、本構造体を参照することで、Datum の型に応じた適切なメモリコピーや比較処理の準備が行われます。具体的には、tuplesortvariants.c 内の各関数が base->arg を本構造体にキャストして、型情報を取得します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/tuplesortvariants.c 内の tuplesort_begin_datum（推測される初期化関数）および、comparetup_datum、readtup_datum、writetup_datum 等の型固有の処理関数。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • Tuplesortstate: ソート処理の全状態を保持する構造体。本構造体はこの内部の arg フィールドを通じて参照されます。

失敗モードと境界条件

  • datumTypeLen が不適切な値である場合、Datum のコピー時にメモリ破壊や不正なデータ読み込みが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は Datum ソート専用のパラメータコンテナです。新しいソート対象型を追加する際は、同様の構成を持つ専用の Arg 構造体を定義し、Tuplesortstate に登録する設計パターンに従ってください。
