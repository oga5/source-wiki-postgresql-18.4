WorkerInfo

定義

  • 種別: typedef (ポインタ型)
  • 定義箇所: src/backend/postmaster/autovacuum.c:242
  • 実体: struct WorkerInfoData へのポインタ

目的

自動バキュームワーカープロセスの状態を追跡・管理するためのハンドルとして機能します。

シグネチャまたは形状

typedef struct WorkerInfoData *WorkerInfo;

入力と出力

  • 入力: なし（型定義）
  • 出力: なし（型定義）

前提条件と不変条件

  • AutoVacuumShmem 共有メモリ領域内で管理される WorkerInfoData インスタンスを指す必要があります。
  • ワーカープロセス内では、MyWorkerInfo 静的変数を通じて自身の状態を保持します。

アルゴリズムまたは振る舞い

  • WorkerInfo 型の変数は、ワーカーのフリーリスト（av_freeWorkers）や実行中キュー（av_runningWorkers）の操作に使用されます。
  • ワーカーの起動時には av_startingWorker を介して初期化され、終了時には FreeWorkerInfo によってフリーリストへ返却されます。

呼び出し元と呼び出し先

  • 呼び出し元: autovacuum.c 内のワーカー管理ロジック（AutoVacuumShmem を操作する各関数）。
  • 呼び出し先: dlist_container を用いて WorkerInfoData 構造体へアクセスするために使用されます。
  • 注: 具体的な関数呼び出しの連鎖については、ソースコードの文脈上、ワーカーのライフサイクル管理全般にわたって広範に使用されています。

関連データ構造

  • WorkerInfoData: このポインタが指す実体となる構造体。
  • AutoVacuumShmemStruct: ワーカーリスト（フリーリストおよび実行中キュー）を保持する共有メモリ構造体。

失敗モードと境界条件

  • NULL ポインタ: MyWorkerInfo が初期化されていない場合や、ワーカーが割り当てられていない場合に発生する可能性があります。
  • 共有メモリの競合: AutovacuumLock を取得せずにリスト操作を行うと、データ構造の不整合を招く恐れがあります。

保守担当者向けメモ

  • 本シンボルは WorkerInfoData 構造体への参照を抽象化するものです。
  • WorkerInfo を扱う際は、必ず AutovacuumLock による排他制御が適切に行われているか確認してください。
  • ワーカーの起動・終了フロー（av_startingWorker の更新や FreeWorkerInfo の呼び出し）を変更する際は、このポインタの整合性に注意してください。
