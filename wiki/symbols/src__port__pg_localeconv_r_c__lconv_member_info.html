lconv_member_info

定義

  • 種別: struct
  • 定義箇所: src/port/pg_localeconv_r.c:33

目的

struct lconv の各メンバをコピーおよび管理するためのメタデータテーブルを定義する構造体です。

シグネチャまたは形状

struct lconv_member_info
{
    bool        is_string;
    int         category;
    size_t      offset;
#ifdef TRANSLATE_FROM_LANGINFO
    nl_item     item;
#endif
};

入力と出力

本構造体はデータ定義用であり、関数ではないため直接の入出力はありません。ただし、pg_localeconv_copy_members() がこの構造体の配列を走査して lconv 構造体のメンバを初期化するために使用します。

前提条件と不変条件

  • is_string が true の場合、対象メンバは文字列として扱われ、strdup() 等によるメモリ確保の対象となります。
  • category は LC_NUMERIC または LC_MONETARY のいずれかである必要があります。

アルゴリズムまたは振る舞い

pg_localeconv_r.c 内の table[] 配列において、lconv 構造体の各フィールドに対するメタデータ（型、カテゴリ、構造体内のオフセット、および TRANSLATE_FROM_LANGINFO 定義時の nl_item）を保持します。これにより、ロケール情報を取得する際に、どのメンバをどのカテゴリから取得し、どのようにコピーすべきかを制御します。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/pg_localeconv_r.c 内の table[] 配列定義（63行目）にて使用されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • struct lconv: 標準 C ライブラリのロケール情報構造体。
  • table[]: lconv_member_info 型の静的配列。

失敗モードと境界条件

  • TRANSLATE_FROM_LANGINFO マクロの定義有無により、構造体のサイズとレイアウトが変化します。

保守担当者向けメモ

  • lconv 構造体のメンバ追加や変更を行う際は、table[] 配列内の対応する lconv_member_info 定義を更新する必要があります。
  • 文字列メンバのメモリ所有権は呼び出し元にあるため、pg_localeconv_free() による解放処理が is_string フラグと整合していることを確認してください。
