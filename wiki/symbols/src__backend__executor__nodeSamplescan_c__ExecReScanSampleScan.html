ExecReScanSampleScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSamplescan.c:201

目的

SampleScan ノードの状態をリセットし、次回のスキャン実行時にサンプリング処理を再初期化できるようにする。

シグネチャまたは形状

void ExecReScanSampleScan(SampleScanState *node)

入力と出力

  • **入力**: SampleScanState *node - リセット対象となるサンプリングスキャンノードの状態構造体。
  • **出力**: なし（副作用として node のメンバ変数を更新する）。

前提条件と不変条件

  • node は有効な SampleScanState 構造体を指している必要がある。
  • 本関数呼び出し後、node->begun が false に設定されるため、後続の実行時に tablesample_init を通じた BeginSampleScan の再実行が保証される。

アルゴリズムまたは振る舞い

2. node->done を false に設定し、スキャンが完了していない状態にする。 3. node->haveblock を false に設定し、現在処理中のブロックがない状態にする。 4. node->donetuples を 0 にリセットする。 5. ExecScanReScan(&node->ss) を呼び出し、共通のスキャン状態（ScanState）のリセット処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:170 からの呼び出しが確認されている。
  • **呼び出し先**: ExecScanReScan（src/backend/executor/execScan.c 内の関数と推測される）。

関連データ構造

  • SampleScanState: サンプリングスキャンの実行状態を保持する構造体。
  • ScanState: SampleScanState 内に含まれる共通スキャン状態。

失敗モードと境界条件

  • node が NULL の場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は TABLESAMPLE 句の再スキャン時に呼び出される。REPEATABLE 句の有無に関わらず、サンプリングの再試行を可能にするための必須処理である。
  • ExecScanReScan への委譲により、スキャンノード共通の基盤処理とサンプリング固有の状態リセットが分離されている。
