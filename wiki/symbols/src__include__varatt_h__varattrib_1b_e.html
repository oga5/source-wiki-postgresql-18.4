varattrib_1b_e

定義

  • 種別: struct
  • 定義箇所: src/include/varatt.h:134

目的

1バイトヘッダーを持つ可変長データのうち、拡張されたデータ型（TOASTポインタや展開済みオブジェクトなど）を表現するための構造体です。

シグネチャまたは形状

typedef struct {
    uint8 va_header;        /* 0x80 または 0x01 */
    uint8 va_tag;           /* データ型の識別子 */
    char  va_data[FLEXIBLE_ARRAY_MEMBER];
} varattrib_1b_e;

入力と出力

  • 入力: 任意のメモリ領域（varattrib_1b_e* にキャストして使用）。
  • 出力: va_tag による型識別および va_data を介した実データへのアクセス。

前提条件と不変条件

  • va_header は常に 0x80 または 0x01 である必要があります。
  • 構造体自体はアライメントが保証されないため、アクセス時には memcpy 等の適切な手段を用いるか、アライメントを考慮しないアクセスが許容される文脈で使用する必要があります。

アルゴリズムまたは振る舞い

  • va_header の値に基づき、特定のデータ形式（TOASTポインタ等）を識別します。
  • va_tag フィールドを使用して、格納されているデータの具体的な種類を判別します。
  • VARDATA_1B_E マクロを通じて、ヘッダー直後のデータ本体へアクセスします。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/utils/adt/expandeddatum.c:31
  • src/include/access/detoast.h:20, 24
  • src/include/varatt.h 内の各種マクロ定義（VARTAG_1B_E, SET_VARSIZE_1B_E 等）
  • 呼び出し先: 本構造体はデータレイアウトを定義するものであり、関数呼び出しは含みません。

関連データ構造

  • varattrib_1b: 標準的な1バイトヘッダー構造体。
  • vartag_external: va_tag で使用される識別子を定義する列挙型。

失敗モードと境界条件

  • アライメントが保証されないメモリ領域に対して直接キャストしてアクセスすると、CPUアーキテクチャによっては不正なメモリアクセスが発生する可能性があります。
  • va_header が期待値（0x80 / 0x01）以外の場合、データ構造の解釈が誤る可能性があります。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を使用しているため、サイズ計算時には sizeof(varattrib_1b_e) ではなく、実際のデータ長を考慮する必要があります。
  • ビッグエンディアン環境とリトルエンディアン環境でビット配置が異なる可能性があるため、直接的なビット操作は避け、提供されているマクロ経由で操作してください。
