connectToServer

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/server.c:27

目的

指定されたデータベースへの接続を確立し、接続が成功した場合にはセキュリティ設定（search_path）を適用した上で、確立された接続ハンドルを返します。

シグネチャまたは形状

PGconn *connectToServer(ClusterInfo *cluster, const char *db_name)

入力と出力

  • **入力**:
  • cluster: 接続先クラスター情報を含む ClusterInfo 構造体へのポインタ。
  • db_name: 接続対象のデータベース名。
  • **出力**:
  • 成功時: 確立された PGconn へのポインタ。
  • 失敗時: 内部で exit(1) を呼び出すため、呼び出し元へは戻りません。

前提条件と不変条件

  • get_db_conn 関数が正常に動作し、接続文字列を生成できること。
  • 接続確立後、ALWAYS_SECURE_SEARCH_PATH_SQL を実行してセキュアな検索パスを確保することが必須となります。

アルゴリズムまたは振る舞い

1. get_db_conn を呼び出し、対象クラスターへの接続を試みます。 2. 接続が NULL または PQstatus が CONNECTION_OK でない場合、エラーメッセージをログ出力し、PQfinish でリソースを解放した後、exit(1) でプロセスを終了します。 3. 接続成功後、executeQueryOrDie を使用して ALWAYS_SECURE_SEARCH_PATH_SQL を実行し、その結果を PQclear で破棄します。 4. 最終的に確立された PGconn ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/check.c
  • src/bin/pg_upgrade/function.c
  • src/bin/pg_upgrade/info.c
  • **呼び出し先**:
  • get_db_conn
  • PQstatus
  • pg_log
  • PQerrorMessage
  • PQfinish
  • executeQueryOrDie
  • PQclear

関連データ構造

  • ClusterInfo: 接続先クラスターの構成情報を保持する構造体。
  • PGconn: libpq の接続ハンドル。

失敗モードと境界条件

  • 接続失敗（conn == NULL または CONNECTION_OK 以外）が発生した場合、即座にプログラム全体が終了します。

保守担当者向けメモ

  • 本関数は pg_upgrade における接続の標準的な入り口です。接続失敗時にプログラムを終了させる設計となっているため、呼び出し側で接続の成否を個別にハンドリングする必要はありません。
  • executeQueryOrDie の戻り値は必ず PQclear されることが保証されています。
