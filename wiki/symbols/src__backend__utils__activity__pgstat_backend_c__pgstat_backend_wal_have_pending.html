pgstat_backend_wal_have_pending

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_backend.c:215

目的

現在のバックエンドプロセスにおいて、共有メモリへフラッシュすべき未処理の WAL 統計が存在するかどうかを判定する。

シグネチャまたは形状

static inline bool pgstat_backend_wal_have_pending(void)

入力と出力

  • **入力**: なし（グローバルな統計変数 pgWalUsage および prevBackendWalUsage を参照）。
  • **出力**: bool 型。未処理の WAL 統計がある場合は true、ない場合は false を返す。

前提条件と不変条件

  • 本関数は pgstat_backend モジュール内でのみ使用される。
  • pgWalUsage は現在のプロセスにおける累積 WAL 使用量、prevBackendWalUsage は前回のフラッシュ時点での WAL 使用量を保持している必要がある。

アルゴリズムまたは振る舞い

現在の WAL レコード数（pgWalUsage.wal_records）と、前回のフラッシュ時に記録された WAL レコード数（prevBackendWalUsage.wal_records）を比較する。両者が一致しない場合、統計の更新が必要であると判断して true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/pgstat_backend.c 内の統計フラッシュ処理（pgstat_flush_backend 等）において、更新の必要性を確認するために呼び出される。
  • **呼び出し先**:
  • なし。

関連データ構造

  • pgWalUsage: 現在のバックエンドの WAL 使用量を保持する構造体。
  • prevBackendWalUsage: 前回のフラッシュ完了時に更新される、比較基準となる WAL 使用量構造体。

失敗モードと境界条件

  • WAL 使用量が変化していない場合、常に false を返す。
  • 統計情報の初期化前や、プロセス開始直後でまだ WAL 操作が発生していない場合は false となる。

保守担当者向けメモ

  • 本関数は static inline で定義されており、頻繁に呼び出されることを想定した軽量なチェック関数である。
  • WAL 統計の差分計算ロジックに依存しているため、prevBackendWalUsage の更新タイミング（pgstat_flush_backend 内）に変更を加える際は注意すること。
