dsa_attach

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/dsa.c:509

目的

既存の動的共有メモリ領域（DSA）に現在のプロセスをアタッチし、当該領域を操作するための dsa_area 構造体を取得する。

シグネチャまたは形状

dsa_area *dsa_attach(dsa_handle handle)

入力と出力

  • **入力**: dsa_handle handle（アタッチ対象の DSA 領域を識別するハンドル）。
  • **出力**: 成功時は当該領域を管理する dsa_area 構造体へのポインタを返す。失敗時はエラーを報告し、呼び出し元へは戻らない。

前提条件と不変条件

  • 指定された handle は、有効な DSM（動的共有メモリ）セグメントのハンドルである必要がある。
  • dsm_attach が成功し、メモリマッピングが確立されていること。

アルゴリズムまたは振る舞い

1. dsm_attach(handle) を呼び出し、DSA 領域の先頭セグメントを現在のプロセスのアドレス空間にマッピングする。 2. マッピングに失敗した場合は ERROR を発生させる。 3. attach_internal を呼び出し、マッピングされたアドレスとセグメント情報から dsa_area 構造体を初期化する。 4. on_dsm_detach を登録し、制御セグメントがデタッチされる際に dsa_on_dsm_detach_release_in_place を実行してクリーンアップを行うよう予約する。 5. 初期化済みの dsa_area ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/tidstore.c
  • src/backend/replication/logical/launcher.c
  • src/backend/storage/ipc/dsm_registry.c
  • **呼び出し先**:
  • dsm_attach
  • attach_internal
  • dsm_segment_address
  • on_dsm_detach

関連データ構造

  • dsa_area: DSA 領域を管理するプロセスローカルな状態構造体。
  • dsa_handle: DSA 領域を識別するハンドル（実体は DSM ハンドル）。
  • dsm_segment: 基盤となる共有メモリセグメント。

失敗モードと境界条件

  • dsm_attach が NULL を返した場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE エラーを報告する。

保守担当者向けメモ

  • 本関数は dsa_attach_in_place と共通の内部処理（attach_internal）を利用している。
  • dsa_on_dsm_detach_release_in_place によるクリーンアップ登録は、セグメントのライフサイクル管理において重要である。
