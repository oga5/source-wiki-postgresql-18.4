parse_policy_command

定義

  • 種別: function
  • 定義箇所: src/backend/commands/policy.c:107

目的

ポリシーコマンド名（文字列）を、内部的に使用される単一文字のコマンド識別子（char）に変換します。

シグネチャまたは形状

static char parse_policy_command(const char *cmd_name)

入力と出力

  • **入力**: cmd_name (ポリシー操作の種類を示す文字列。例: "select", "insert" 等)
  • **出力**: 対応する char 型の識別子。
  • "all" → '*'
  • "select" → ACL_SELECT_CHR
  • "insert" → ACL_INSERT_CHR
  • "update" → ACL_UPDATE_CHR
  • "delete" → ACL_DELETE_CHR

前提条件と不変条件

  • 入力 cmd_name が NULL の場合、エラーが発生します。
  • 認識できないコマンド名が渡された場合、エラーが発生します。

アルゴリズムまたは振る舞い

1. cmd_name が NULL かどうかをチェックし、該当する場合は elog(ERROR) を送出します。 2. strcmp を用いて cmd_name を順次比較し、定義済みの文字列と一致した場合は対応する定数文字を返します。 3. いずれの条件にも一致しない場合、elog(ERROR) を送出し、処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/policy.c:594 にて、stmt->cmd_name を引数として呼び出されています。
  • **呼び出し先**: strcmp（標準Cライブラリ）、elog（PostgreSQLエラー報告マクロ）。

関連データ構造

  • ACL_SELECT_CHR 等の定数は utils/acl.h で定義されています。

失敗モードと境界条件

  • cmd_name が NULL の場合、またはサポートされていない文字列（例: "truncate" など）が渡された場合、elog(ERROR) によりトランザクションがアボートされます。

保守担当者向けメモ

  • 本関数は CREATE POLICY や ALTER POLICY 等の構文解析時に、ユーザー指定のコマンド名を内部表現へ正規化するために使用されます。新しいポリシーコマンドの種類を追加する場合は、本関数の条件分岐に追加が必要です。
