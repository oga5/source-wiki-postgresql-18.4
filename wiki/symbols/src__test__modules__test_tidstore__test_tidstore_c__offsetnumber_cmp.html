offsetnumber_cmp

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_tidstore/test_tidstore.c:78

目的

OffsetNumber 型の値を比較し、qsort 等のソート関数で使用可能な標準的な比較結果（-1, 0, 1）を返す。

シグネチャまたは形状

static int offsetnumber_cmp(const void *a, const void *b)

入力と出力

  • **入力**: 比較対象の OffsetNumber を指す const void * 型のポインタ a および b。
  • **出力**:
  • *a < *b の場合: -1
  • *a > *b の場合: 1
  • *a == *b の場合: 0

前提条件と不変条件

  • 入力ポインタ a および b は、有効な OffsetNumber 型の値を指している必要がある。

アルゴリズムまたは振る舞い

1. void * 型の引数を OffsetNumber * にキャストし、指し示す値をデリファレンスして取得する。 2. 取得した2つの値を比較し、昇順となるように整数値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_tidstore/test_tidstore.c 内の qsort 呼び出し（196行目）にて、比較関数として渡されている。
  • **呼び出し先**: なし（標準的な比較演算のみを実行）。

関連データ構造

  • OffsetNumber: PostgreSQL のブロック内オフセット番号を表す型。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、デリファレンスによりクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数は qsort のコールバックとして設計されている。比較ロジックを変更する場合は、qsort の要件である「第1引数が小さい場合に負、等しい場合に0、大きい場合に正」という契約を維持すること。
