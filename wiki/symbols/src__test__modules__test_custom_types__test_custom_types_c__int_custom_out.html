int_custom_out

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_custom_types/test_custom_types.c:57

目的

カスタム型 int_custom の内部表現（int32）を、SQL出力用の文字列（cstring）に変換する出力関数です。

シグネチャまたは形状

Datum int_custom_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_INT32(0) を介して int32 型のデータを受け取ります。
  • **出力**: PG_RETURN_CSTRING を介して、数値を文字列化した char * を返します。

前提条件と不変条件

  • palloc(12) により、符号、最大10桁の数値、および終端文字 \0 を格納するのに十分なメモリが確保されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT32(0) で入力値を取得します。 2. palloc(12) で出力文字列用のメモリを割り当てます。 3. pg_ltoa を使用して、整数値を文字列に変換し、割り当てたバッファに格納します。 4. 変換された文字列を PG_RETURN_CSTRING で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（PG_FUNCTION_INFO_V1(int_custom_out) マクロにより登録）。
  • **呼び出し先**: palloc（メモリ確保）、pg_ltoa（整数から文字列への変換）。

関連データ構造

  • int32: 本関数が扱う内部データ型。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • palloc がメモリ不足により失敗した場合、PostgreSQLの標準的なエラー処理によりアボートされます。
  • pg_ltoa は int32 の範囲内の数値を正しく処理します。

保守担当者向けメモ

  • 本関数はテストモジュール用であり、標準的な int4out と同様の挙動を模倣しています。
  • 戻り値のメモリは palloc で確保されているため、PostgreSQLのメモリ管理システムによって適切に解放されます。
