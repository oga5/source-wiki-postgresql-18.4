pq_recvbuf

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqcomm.c:897

目的

クライアントから送信されたデータをソケットから読み込み、内部バッファ（PqRecvBuffer）へ格納する。

シグネチャまたは形状

int pq_recvbuf(void)

入力と出力

  • **入力**: グローバル変数 PqRecvBuffer, PqRecvPointer, PqRecvLength, MyProcPort を参照。
  • **出力**:
  • 成功時: 0 を返し、PqRecvLength が更新される。
  • 失敗または EOF 時: EOF を返す。

前提条件と不変条件

  • 実行前に socket_set_nonblocking(false) が呼び出され、ブロッキングモードへ強制的に遷移する。
  • PqRecvBuffer は PQ_RECV_BUFFER_SIZE の容量を持つ。

アルゴリズムまたは振る舞い

1. **バッファ整理**: PqRecvPointer が 0 より大きい場合、未読データ（PqRecvLength - PqRecvPointer）をバッファの先頭へ memmove で詰め、ポインタをリセットする。 2. **読み込みループ**: secure_read を使用してソケットからデータを読み込む。

  • EINTR（割り込み）が発生した場合はループを継続する。
  • r < 0 かつ errno != 0 の場合、COMMERROR をログ出力し EOF を返す。
  • r == 0 の場合、EOF と判断し EOF を返す。

    • 3. **バッファ更新**: 読み込んだバイト数分 PqRecvLength を加算し、0 を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/pqcomm.c 内の各関数（pq_getbytes 等のバッファ確認ロジックから参照されている）。
  • **呼び出し先**: socket_set_nonblocking, secure_read, ereport。

関連データ構造

  • PqRecvBuffer: 受信データを保持する静的バッファ。
  • MyProcPort: 現在の接続状態を保持する Port 構造体。

失敗モードと境界条件

  • **EOF**: クライアントによる接続切断時は EOF を返す。
  • **エラー**: ソケット読み込みエラー時は COMMERROR を発行する。この際、再帰的な ereport を避けるため、Postmaster ログへの出力のみを行う。
  • **バッファ溢れ**: PQ_RECV_BUFFER_SIZE を超える読み込みは行わない。

保守担当者向けメモ

  • 本関数はブロッキングモードで動作するため、呼び出し元はタイムアウトやデッドロックのリスクを考慮する必要がある。
  • ereport 呼び出し時は、通信層の再帰的な呼び出しによるスタックオーバーフローを避けるため、エラーメッセージの出力先に注意すること。
