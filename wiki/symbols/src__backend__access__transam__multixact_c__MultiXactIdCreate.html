MultiXactIdCreate

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/multixact.c:478

目的

2つのトランザクションID（XID）とそれぞれのロック状態を組み合わせ、新しい MultiXactId を生成する。

シグネチャまたは形状

MultiXactId MultiXactIdCreate(TransactionId xid1, MultiXactStatus status1,
                              TransactionId xid2, MultiXactStatus status2)

入力と出力

  • **入力**:
  • xid1, xid2: 登録対象のトランザクションID。
  • status1, status2: 各トランザクションのロック状態（MultiXactStatus）。
  • **出力**:
  • 生成された新しい MultiXactId を返す。

前提条件と不変条件

  • xid1 および xid2 は有効なトランザクションIDであること。
  • xid1 と xid2 が同一である場合、status1 と status2 は異なる必要がある。
  • MultiXactIdSetOldestMember() が事前に呼び出され、最古のメンバーIDスロットが有効であること。

アルゴリズムまたは振る舞い

1. 入力された2つのXIDとステータスを MultiXactMember 配列に格納する。 2. MultiXactIdCreateFromMembers(2, members) を呼び出し、メンバーリストを永続化して新しい MultiXactId を取得する。 3. デバッグ用に mxid_to_string を使用してログを出力する。 4. 生成された MultiXactId を呼び出し元に返す。

  • 既存の MultiXactIdExpand と異なり、入力されたXIDが現在実行中であるかのチェックは行わない。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c (タプルロック更新時)
  • src/test/modules/test_slru/test_multixact.c (テストコード)
  • **呼び出し先**:
  • MultiXactIdCreateFromMembers
  • MyOldestMemberMXactIdSlot (マクロ)
  • TransactionIdIsValid (マクロ)
  • TransactionIdEquals (マクロ)
  • debug_elog3 (マクロ)

関連データ構造

  • MultiXactId: 32ビットの識別子。
  • MultiXactMember: トランザクションIDとロック状態を保持する構造体。
  • MultiXactStatus: ロックの性質を示す列挙型。

失敗モードと境界条件

  • 入力されたXIDが無効な場合、Assert により停止する。
  • MultiXactIdCreateFromMembers 内部で発生するSLRU領域の枯渇やI/Oエラーにより失敗する可能性がある。

保守担当者向けメモ

  • 本関数は2つのメンバーを束ねる専用のラッパーである。より多くのメンバーを扱う場合は MultiXactIdCreateFromMembers を直接使用すること。
  • MultiXactIdExpand と異なり、XIDの生存確認を省略しているため、呼び出し側で整合性を担保する必要がある。
