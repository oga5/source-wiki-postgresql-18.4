rbt_delete_node

定義

  • 種別: function
  • 定義箇所: src/backend/lib/rbtree.c:618

目的

赤黒木から指定されたノードを物理的に削除し、必要に応じて平衡状態を維持するための修正を行う。

シグネチャまたは形状

static void rbt_delete_node(RBTree *rbt, RBTNode *z);

入力と出力

  • **入力**:
  • rbt: 操作対象の赤黒木構造体へのポインタ。
  • z: 削除対象のノードへのポインタ。
  • **出力**: なし（副作用として木構造が更新される）。

前提条件と不変条件

  • z は有効なノードであること（NULL または RBTNIL の場合は即座に復帰する）。
  • 削除後、赤黒木の不変条件（各パスの黒ノード数が等しい、赤ノードの子供は黒である等）が維持される必要がある。

アルゴリズムまたは振る舞い

1. **削除対象の特定**: 削除対象 z が2つ未満の子を持つ場合は z を削除対象 y とする。それ以外の場合は z の後継ノード（右部分木の最小ノード）を y とする。 2. **ノードの切り離し**: y の唯一の子 x を特定し、y を木から取り外して x を y の親に接続する。 3. **データ移行**: y が z と異なる場合（後継ノードを削除した場合）、rbt_copy_data を使用して y のデータを z にコピーし、論理的に z を削除した状態にする。 4. **平衡維持**: 削除されたノード y が黒色であった場合、rbt_delete_fixup を呼び出して平衡を再調整する。 5. **メモリ解放**: rbt->freefunc が定義されている場合、削除したノード y のメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: rbt_delete（src/backend/lib/rbtree.c:697）から呼び出される。
  • **呼び出し先**:
  • rbt_copy_data
  • rbt_delete_fixup
  • rbt->freefunc（コールバック）

関連データ構造

  • RBTree: 木全体を管理する制御構造体。
  • RBTNode: ノードのヘッダー構造体。
  • RBTNIL: 葉ノードを表す静的な番兵。

失敗モードと境界条件

  • z が NULL または RBTNIL の場合、何も行わずに終了する。
  • 削除対象がルートノードである場合、rbt->root が正しく更新される。
  • 削除対象が黒ノードである場合、木全体の平衡が崩れる可能性があるため、必ず rbt_delete_fixup が実行される。

保守担当者向けメモ

  • 本関数は侵入型設計に基づいている。freefunc はノード自体のメモリ解放のみを担うため、ノード内のデータ構造に対する再帰的な解放処理は呼び出し側で考慮すること。
  • y != z の場合に発生するデータコピーは、rbt_copy_data コールバックに依存している。
