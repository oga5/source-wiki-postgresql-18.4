QueryCompletion

定義

  • 種別: typedef
  • 定義箇所: src/include/tcop/utility.h:71
  • 構造体定義: tcop/cmdtag.h:29

目的

SQLコマンドの実行結果に関する情報を保持し、呼び出し元へ伝達するためのデータ構造です。

シグネチャまたは形状

typedef として定義されており、実体は tcop/cmdtag.h で定義される構造体です。

入力と出力

  • **入力**: ユーティリティコマンドの実行関数（ProcessUtility 等）において、実行結果を記録するためにポインタとして渡されます。
  • **出力**: コマンド実行後、処理されたタプル数やコマンドタグなどの実行結果情報が格納されます。

前提条件と不変条件

  • ユーティリティコマンドの実行関数にポインタとして渡される際、呼び出し元でメモリが確保されている必要があります。
  • CREATE TABLE AS 等のコマンドでは、処理されたタプル数を正しく設定することが求められます。

アルゴリズムまたは振る舞い

コマンド実行中に実行結果（処理件数等）を QueryCompletion 構造体に書き込み、呼び出し元へ戻すことで、クライアントへの応答や統計情報の更新に利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照例）**:
  • src/backend/executor/spi.c
  • src/backend/replication/walsender.c
  • contrib/pg_stat_statements/pg_stat_statements.c
  • contrib/sepgsql/hooks.c
  • **呼び出し先**: ユーティリティコマンド実行関数（createas.c, matview.c, portalcmds.c, prepare.c 等）がこの構造体を受け取り、値を設定します。

関連データ構造

  • ProcessUtility_hook_type: フック関数の引数として使用されます。
  • CommandTag: コマンドの種類を識別するために併用されます。

失敗モードと境界条件

  • 構造体が初期化されていない状態で参照された場合、未定義の動作を引き起こす可能性があります。
  • 処理件数が不明なコマンドの場合、適切にゼロまたはデフォルト値で初期化されている必要があります。

保守担当者向けメモ

  • 本シンボルは tcop/cmdtag.h で定義される構造体へのエイリアスです。詳細なフィールド構成については tcop/cmdtag.h を参照してください。
  • ユーティリティコマンドを新規追加する際は、実行結果を適切に QueryCompletion に反映させる必要があります。
