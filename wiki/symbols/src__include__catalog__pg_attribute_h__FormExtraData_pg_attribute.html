FormExtraData_pg_attribute

定義

  • **種別**: struct
  • **定義箇所**: src/include/catalog/pg_attribute.h:212

目的

pg_attribute システムカタログの行構造において、CATALOG_VARLEN マクロにより通常のタプル記述子（TupleDesc）から除外されるフィールドを保持・伝達するための補助的な構造体です。

シグネチャまたは形状

typedef struct FormExtraData_pg_attribute
{
    NullableDatum attstattarget;
    NullableDatum attoptions;
} FormExtraData_pg_attribute;

入力と出力

  • **入力**: 統計ターゲット（attstattarget）および属性オプション（attoptions）を NullableDatum 形式で保持します。
  • **出力**: 構造体ポインタを通じて、列のメタデータ補完情報を提供します。

前提条件と不変条件

  • 本構造体に含まれるフィールドは、メモリ上の TupleDesc には含まれないため、別途管理する必要があります。
  • NullableDatum 型を使用しているため、値の有無（NULL許容性）を考慮したアクセスが必要です。

アルゴリズムまたは振る舞い

  • テーブル作成やインデックス定義時に、列の拡張属性情報を一時的に保持するために使用されます。
  • src/backend/catalog/heap.c や src/backend/catalog/index.c において、列数に応じた配列として確保され、メタデータの受け渡しに利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/catalog/heap.c: tupdesc_extra 引数として使用。
  • src/backend/catalog/index.c: インデックスリレーションの属性情報構築時に palloc0_array で確保。
  • src/include/catalog/heap.h: 関数宣言の引数として使用。
  • **呼び出し先**:
  • 特定の関数呼び出し関係はソース抜粋からは不明です。

関連データ構造

  • FormData_pg_attribute: pg_attribute カタログの物理的な行構造（固定長部分）。
  • TupleDesc: リレーションの列情報を保持する構造体。本構造体は、この TupleDesc を補完する役割を持ちます。

失敗モードと境界条件

  • palloc0_array によるメモリ確保失敗時のエラーハンドリングは、呼び出し側の実装に依存します。
  • natts（列数）が 0 の場合や、tupdesc_extra が NULL の場合のポインタ参照には注意が必要です。

保守担当者向けメモ

  • 本構造体のフィールドを変更した場合は、src/backend/catalog/heap.c 等の関連する初期化ロジックとの整合性を確認してください。
  • CATALOG_VARLEN で除外されるフィールドを扱うため、メモリレイアウトの変更時には TupleDesc との分離状態を維持する設計方針に従ってください。
