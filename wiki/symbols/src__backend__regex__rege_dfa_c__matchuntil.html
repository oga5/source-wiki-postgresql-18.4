matchuntil

定義

  • 種別: function
  • 定義箇所: src/backend/regex/rege_dfa.c:370

目的

指定された位置（probe）まで正規表現のマッチングを増分的に実行し、その地点でマッチが成立するかを判定する。

シグネチャまたは形状

int matchuntil(struct vars *v, struct dfa *d, chr *probe, struct sset **lastcss, chr **lastcp)

入力と出力

  • **入力**:
  • v: 正規表現実行時の状態を保持する vars 構造体。
  • d: DFA 実行状態を保持する dfa 構造体。
  • probe: マッチングを終了させる対象の文字位置。
  • lastcss: 前回の呼び出しからの状態セット（sset）のポインタ。
  • lastcp: 前回の呼び出しからの文字位置ポインタ。
  • **出力**:
  • 戻り値: マッチが成立した場合は 1、それ以外は 0。
  • lastcss, lastcp: 呼び出し後の状態と位置が更新される。

前提条件と不変条件

  • d->cnfa->maxmatchall は DUPINF であること。
  • lastcp が NULL または probe より進んでいる場合、初期化処理が再実行される。
  • ss が NULL になった場合、マッチは不可能とみなされる。

アルゴリズムまたは振る舞い

1. **高速パス**: MATCHALL フラグが有効な場合、文字数チェックのみを行い即座に判定する。 2. **初期化/再開**: lastcp が NULL または probe より先にある場合、initialize() と miss() を用いて開始状態を構築する。 3. **スキャンループ**: cp から probe に到達するまで、GETCOLOR で文字を色に変換し、DFA の状態遷移（outs）を辿る。キャッシュミス時は miss() を呼び出して遷移を計算する。 4. **終了判定**: probe 到達後、文字列終端（EOS）または次の文字を処理し、最終状態が POSTSTATE フラグを持っているかを確認する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/rege_dfa.c (内部の制約評価ロジック)
  • src/backend/regex/regexec.c
  • **呼び出し先**:
  • initialize()
  • miss()
  • GETCOLOR (マクロ)

関連データ構造

  • struct vars: 実行コンテキスト。
  • struct dfa: DFA の実行状態。
  • struct sset: DFA の状態セット（キャッシュエントリ）。
  • struct colormap: 文字色変換テーブル。

失敗モードと境界条件

  • miss() が NULL を返した場合、マッチングは中断され 0 を返す。
  • ss が NULL のままループを抜けた場合、内部エラーまたはマッチ不可能として 0 を返す。
  • POSTSTATE フラグを持たない状態に到達した場合、マッチ失敗として 0 を返す。

保守担当者向けメモ

  • 本関数は regexec.c からも呼び出される増分マッチングエンジンである。
  • REG_DEBUG が有効な場合、REG_FTRACE フラグにより詳細なトレースログが出力される。
  • lastcss と lastcp を介して状態を保持するため、呼び出し側でのポインタ管理が重要である。
