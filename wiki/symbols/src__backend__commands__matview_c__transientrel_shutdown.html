transientrel_shutdown

定義

  • 種別: function
  • 定義箇所: src/backend/commands/matview.c:536

目的

マテリアライズド・ビューの更新処理において、一時テーブルへのデータ挿入フェーズを終了し、リソースを解放する。

シグネチャまたは形状

static void transientrel_shutdown(DestReceiver *self)

入力と出力

  • **入力**: DestReceiver *self（DR_transientrel 構造体としてキャストして使用）
  • **出力**: なし（副作用としてリソース解放とテーブルクローズを行う）

前提条件と不変条件

  • self は DR_transientrel 型の構造体である必要がある。
  • myState->transientrel が有効なテーブルポインタを保持していること。

アルゴリズムまたは振る舞い

1. FreeBulkInsertState を呼び出し、バルク挿入状態を解放する。 2. table_finish_bulk_insert を呼び出し、一時テーブルへの挿入処理を完了させる。 3. table_close を呼び出し、一時テーブルを閉じる（ロックはコミットまで保持するため NoLock を指定）。 4. myState->transientrel を NULL に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: DestReceiver 構造体の rShutdown フィールドを通じて、エグゼキュータ終了時に呼び出される。
  • **呼び出し先**:
  • FreeBulkInsertState
  • table_finish_bulk_insert
  • table_close

関連データ構造

  • DR_transientrel: 一時テーブルへのデータ挿入を管理する構造体。bistate（バルク挿入状態）と transientrel（対象テーブル）を保持する。

失敗モードと境界条件

  • self が NULL または不正な型である場合、キャスト後のアクセスでクラッシュする可能性がある。
  • table_close 呼び出し後の transientrel への再アクセスは禁止されている。

保守担当者向けメモ

  • 本関数は DestReceiver インターフェースの一部として実装されている。
  • ロックをコミットまで保持する設計であるため、table_close には NoLock を渡している点に注意すること。
