ResourceOwnerRememberSnapshot

定義

  • 種別: function
  • 定義箇所: src/backend/utils/time/snapmgr.c:233

目的

指定されたスナップショット（Snapshot）をリソースオーナー（ResourceOwner）の管理下に登録し、トランザクション終了時などに自動的に解放されるよう追跡対象とする。

シグネチャまたは形状

static inline void ResourceOwnerRememberSnapshot(ResourceOwner owner, Snapshot snap)

入力と出力

  • **入力**:
  • owner: スナップショットを所有するリソースオーナー。
  • snap: 管理対象とするスナップショット構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • snap は有効な Snapshot 構造体であること。
  • owner は有効な ResourceOwner であること。

アルゴリズムまたは振る舞い

1. PointerGetDatum(snap) を使用して、スナップショットポインタを Datum 型に変換する。 2. ResourceOwnerRemember 関数を呼び出し、snapshot_resowner_desc（スナップショット用のリソース記述子）と共にリソースオーナーへ登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/time/snapmgr.c:848
  • **呼び出し先**:
  • ResourceOwnerRemember（リソースオーナーへの登録処理）

関連データ構造

  • Snapshot: 可視性判定に使用されるスナップショットデータ構造。
  • ResourceOwner: リソースのライフサイクルを管理するオブジェクト。
  • snapshot_resowner_desc: リソースオーナーがスナップショットを識別・管理するための記述子。

失敗モードと境界条件

  • リソースオーナーが既に破棄されている場合や、メモリ不足により登録に失敗した場合は、ResourceOwnerRemember 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、主に snapmgr.c 内でのスナップショット登録の抽象化を目的としている。
  • リソースオーナーによる管理は、トランザクション終了時のクリーンアップを確実にするために重要である。
