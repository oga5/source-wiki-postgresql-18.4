saveNodeLink

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgdoinsert.c:185

目的

SP-GiSTインデックスの親ページ内にある特定のインナータプルに対し、子ノードへのリンク先（ブロック番号およびオフセット番号）を更新し、変更をバッファに反映させる。

シグネチャまたは形状

static void saveNodeLink(Relation index, SPPageDesc *parent, BlockNumber blkno, OffsetNumber offnum)

入力と出力

  • **入力**:
  • index: 対象のインデックスリレーション。
  • parent: 親ページ情報および更新対象のノードインデックスを保持する SPPageDesc 構造体。
  • blkno: 新たに設定する子ノードのブロック番号。
  • offnum: 新たに設定する子ノードのオフセット番号。
  • **出力**: なし（副作用としてバッファを更新）。

前提条件と不変条件

  • parent->page および parent->buffer が有効であり、parent->offnum が指すアイテムが SpGistInnerTuple であること。

アルゴリズムまたは振る舞い

1. parent->page から parent->offnum を使用して SpGistInnerTuple を取得する。 2. spgUpdateNodeLink を呼び出し、当該インナータプルの parent->node 番目のノードのリンク先を blkno および offnum に書き換える。 3. MarkBufferDirty(parent->buffer) を呼び出し、変更されたページをダーティとしてマークする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgdoinsert.c 内の複数箇所（行 236, 514, 1280, 1317, 1640）。
  • **呼び出し先**: spgUpdateNodeLink、PageGetItem、PageGetItemId、MarkBufferDirty。

関連データ構造

  • SPPageDesc: 挿入中のページ状態を保持する構造体。
  • SpGistInnerTuple: SP-GiSTのインナータプル構造。

失敗モードと境界条件

  • parent が指すページやオフセットが不正な場合、PageGetItem 等でメモリ保護違反やアサーションエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際のリンク更新ロジックは spgUpdateNodeLink に依存している。
  • WALログの生成は本関数の呼び出し元、あるいは spgUpdateNodeLink の前後で適切に管理されている必要がある。
