ecpg_store_result

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/execute.c:302

目的

SQL クエリの実行結果（PGresult）から特定のフィールド値を取り出し、アプリケーション側のホスト変数へ格納（マッピング）する。

シグネチャまたは形状

bool ecpg_store_result(const PGresult *results, int act_field, const struct statement *stmt, struct variable *var)

入力と出力

  • **入力**:
  • results: libpq から取得したクエリ結果セット。
  • act_field: 格納対象のフィールドインデックス。
  • stmt: 現在実行中の SQL 文に関するメタデータ構造体。
  • var: 値を格納する先のホスト変数構造体。
  • **出力**:
  • 成功時は true、失敗時は false を返す。

前提条件と不変条件

  • results は有効な PGresult ポインタであること。
  • var が配列として定義されている場合、var->arrsize が 0 であってはならない。
  • メモリ割り当てが必要な場合、ecpg_auto_alloc を介して動的に確保される。

アルゴリズムまたは振る舞い

1. **配列判定**: ecpg_is_type_an_array を呼び出し、対象フィールドが配列かどうかを判定する。 2. **サイズ検証**: 配列でない場合、結果のタプル数がホスト変数の許容サイズを超えていないか確認する。 3. **メモリ確保**: ホスト変数のバッファが未割り当ての場合、結果セットのサイズに基づいて必要なメモリを計算し、ecpg_auto_alloc で確保する。 4. **データ転送**:

  • char** 型の特殊なケース（char**foo=0）では、ポインタ配列とデータ領域を個別に確保し、値をコピーする。
  • 通常のケースでは、ecpg_get_data を呼び出し、各タプルの値をホスト変数へ変換・コピーする。

    • 5. **終了処理**: 全タプルの処理完了後、成功状態を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:

  • **呼び出し先**:

関連データ構造

  • PGresult: libpq の結果セット構造体。
  • statement: SQL 文の実行状態を保持する構造体。
  • variable: ホスト変数の型、サイズ、値へのポインタを保持する構造体。

失敗モードと境界条件

  • **メモリ不足**: ecpg_auto_alloc が失敗した場合、false を返す。
  • **型不一致**: 配列として定義されていない変数に配列データが渡された場合、またはその逆の場合にエラーとなる。
  • **サイズ超過**: 結果セットのタプル数がホスト変数の配列サイズを超える場合、ECPG_TOO_MANY_MATCHES 等のエラーを発生させる。
  • **配列判定エラー**: ecpg_is_type_an_array が ECPG_ARRAY_ERROR を返した場合、メモリ不足エラーとして処理される。

保守担当者向けメモ

  • char** の特殊な処理は、メモリレイアウトが複雑なため変更時に注意が必要。
  • ecpg_get_data への引数渡しが多いため、型変換ロジックの修正時は data.c の実装と整合性を確認すること。
