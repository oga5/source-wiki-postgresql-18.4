logicalrep_relmap_update

定義

  • **種別**: function
  • **定義箇所**: src/backend/replication/logical/relation.c:163

目的

リモートリレーションの定義情報をキャッシュ（LogicalRepRelMap）に更新または新規登録します。

シグネチャまたは形状

void logicalrep_relmap_update(LogicalRepRelation *remoterel)

入力と出力

  • **入力**: remoterel (LogicalRepRelation *) - 更新対象となるリモートリレーションの定義情報。
  • **出力**: なし。副作用として LogicalRepRelMap ハッシュテーブルの内容が更新されます。

前提条件と不変条件

  • LogicalRepRelMap が初期化されていない場合、内部で logicalrep_relmap_init() が呼び出されます。
  • キャッシュへの書き込みは LogicalRepRelMapContext メモリコンテキスト上で行われます。

アルゴリズムまたは振る舞い

1. LogicalRepRelMap が未初期化であれば初期化します。 2. hash_search を使用して、指定された remoteid に対応するエントリをハッシュテーブルから検索または作成します。 3. 既存のエントリが見つかった場合、logicalrep_relmap_free_entry を呼び出して古いメモリ領域を解放します。 4. エントリをゼロクリアし、LogicalRepRelMapContext に切り替えた上で、remoterel の各フィールド（名前、属性名、型、レプリカアイデンティティ、属性キー）をコピーしてキャッシュを構築します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/tablesync.c:1162
  • src/backend/replication/logical/worker.c:2326
  • **呼び出し先**:
  • logicalrep_relmap_init
  • hash_search
  • logicalrep_relmap_free_entry
  • MemoryContextSwitchTo
  • pstrdup, palloc, bms_copy

関連データ構造

  • LogicalRepRelation: 入力として受け取るリモート側のリレーション定義。
  • LogicalRepRelMap: キャッシュを保持するハッシュテーブル。
  • LogicalRepRelMapEntry: ハッシュテーブルに格納されるキャッシュエントリ。

失敗モードと境界条件

  • メモリ不足時には palloc 等によりエラーが発生する可能性があります。
  • remoterel が NULL の場合の挙動は定義されていません。

保守担当者向けメモ

  • 本関数はキャッシュの「上書き」を行うため、既存エントリの解放とメモリコンテキストの管理が重要です。
  • 属性名や型情報のコピーは LogicalRepRelMapContext 上で行われるため、呼び出し元のメモリコンテキストに依存しません。
