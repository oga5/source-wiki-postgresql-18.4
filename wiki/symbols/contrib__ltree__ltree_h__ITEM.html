ITEM

定義

  • 種別: struct
  • 定義箇所: contrib/ltree/ltree.h:138

目的

ポーランド記法を用いたクエリ構造体（ltxtquery 等）において、演算子やオペランドなどの各要素を表現するための基本単位。

シグネチャまたは形状

typedef struct ITEM
{
	int16		type;
	int16		left;
	int32		val;
	uint8		flag;
	uint8		length;
	uint16		distance;
} ITEM;

入力と出力

  • 本構造体はデータ保持を目的としており、関数としての入出力は持ちません。
  • type フィールドにより、要素の種類（演算子またはオペランド）を識別します。

前提条件と不変条件

  • contrib/intarray モジュールにおいても同名の構造体が定義されており、メモリレイアウトの互換性が前提となっています。
  • 構造体内の各フィールドは固定長であり、パディングを含めたサイズ計算が必要です。

アルゴリズムまたは振る舞い

  • クエリ評価時、ITEM の配列を走査することで論理演算（AND, OR, NOT）や値の照合を行います。
  • left フィールドは、ツリー構造における左側の要素へのオフセットやインデックスとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/intarray/_int_bool.c 内の checkcondition_arr、checkcondition_bit、execute、checkcondition_gin 等が、ITEM 配列を走査または参照しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ltxtquery: ITEM の配列を保持する上位のクエリ構造体。
  • contrib/intarray/_int.h: 同名の ITEM 構造体が定義されており、マクロ COMPUTESIZE や QUERYTYPEMAXITEMS を通じてサイズ計算に使用されます。

失敗モードと境界条件

  • QUERYTYPEMAXITEMS を超える要素数を保持しようとした場合、メモリ割り当てやインデックス計算で境界違反が発生する可能性があります。
  • type フィールドに不正な値が設定された場合、評価ロジックが予期せぬ動作をする可能性があります。

保守担当者向けメモ

  • contrib/ltree と contrib/intarray の両方で定義されているため、変更を加える際は両モジュールへの影響を考慮してください。
  • FLEXIBLE_ARRAY_MEMBER を用いた配列操作が行われることが多いため、メモリ確保時には COMPUTESIZE マクロ等の適切な計算式を使用してください。
