finalizeMemory

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/SectionMemoryManager.cpp:273

目的

JITコンパイルされたコードおよびデータのメモリ領域に対し、適切なページ権限を適用し、命令キャッシュを同期させることで実行可能な状態に確定させる。

シグネチャまたは形状

bool finalizeMemory(std::string *ErrMsg)

入力と出力

  • **入力**: ErrMsg (エラー発生時にエラーメッセージを格納するためのポインタ、省略可能)
  • **出力**: 成功時に false、失敗時に true を返す。

前提条件と不変条件

  • CodeMem（コード領域）および RODataMem（読み取り専用データ領域）が既に確保されていること。
  • 読み書き可能データ領域は、本メソッド呼び出し時点で既に適切な権限が設定されていること。

アルゴリズムまたは振る舞い

1. applyMemoryGroupPermissions を呼び出し、CodeMem に読み取りおよび実行権限（MF_READ | MF_EXEC）を適用する。失敗時はエラーメッセージを格納し終了する。 2. applyMemoryGroupPermissions を呼び出し、RODataMem に読み取り専用権限（MF_READ）を適用する。失敗時はエラーメッセージを格納し終了する。 3. invalidateInstructionCache を呼び出し、命令キャッシュを無効化してデータキャッシュとの整合性を確保する。 4. 成功時は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/jit/SectionMemoryManager.h に記載があり、MCJITの finalizeObject 等から呼び出されることが示唆されている。
  • **呼び出し先**:
  • applyMemoryGroupPermissions
  • invalidateInstructionCache

関連データ構造

  • CodeMem
  • RODataMem

失敗モードと境界条件

  • applyMemoryGroupPermissions が失敗した場合、エラーコードを ErrMsg に書き込み、即座に true を返して終了する。

保守担当者向けメモ

  • 本メソッドはLLVMの RuntimeDyld と連携して動作する。
  • ARM等の命令キャッシュとデータキャッシュが分離しているアーキテクチャでは、invalidateInstructionCache の呼び出しが必須である。
  • 将来的なLLVMのバージョンアップやJITLinkへの移行に伴い、本メソッドの必要性や実装を見直す可能性がある。
