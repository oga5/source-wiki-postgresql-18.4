AioBackendShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_init.c:48

目的

AIOサブシステムにおいて、全バックエンドプロセス用の状態管理構造体（PgAioBackend）を格納するために必要な共有メモリサイズを算出します。

シグネチャまたは形状

Size AioBackendShmemSize(void)

入力と出力

  • 入力: なし
  • 出力: Size 型のバイト数。AioProcs() が返すバックエンド数と PgAioBackend 構造体のサイズの積を返します。

前提条件と不変条件

  • AioProcs() がシステム上の有効なバックエンド数を正しく返す状態であること。
  • mul_size 関数により、計算結果が Size 型の範囲内でオーバーフローしないことが保証されます。

アルゴリズムまたは振る舞い

1. AioProcs() を呼び出し、管理対象となるバックエンドの総数を取得します。 2. sizeof(PgAioBackend) を乗算し、必要なメモリ量を算出します。 3. 計算結果を返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/aio/aio_init.c:136 (AioShmemSize 内で共有メモリ総サイズ計算の一部として使用)
  • src/backend/storage/aio/aio_init.c:168 (AioShmemInit 内で共有メモリ確保時のサイズ指定に使用)
  • 呼び出し先:
  • AioProcs()
  • mul_size()

関連データ構造

  • PgAioBackend: 各バックエンドのAIO状態を保持する構造体。

失敗モードと境界条件

  • AioProcs() が 0 を返す場合、結果は 0 となります。
  • メモリ計算時にオーバーフローが発生する場合、mul_size の内部仕様に従いエラーが発生します。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウトを決定する重要な計算の一部です。PgAioBackend 構造体の定義が変更された場合は、必ず本関数が正しくサイズを反映しているか確認してください。
