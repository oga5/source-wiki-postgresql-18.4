pg_local_to_utf_combined

定義

  • 種別: struct
  • 定義箇所: src/include/mb/pg_wchar.h:503

目的

ローカルエンコーディングの文字コードと、それに対応するUTF-8コードポイント（最大2つ）をマッピングするためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	uint32		code;			/* local code */
	uint32		utf1;			/* UTF-8 code 1 */
	uint32		utf2;			/* UTF-8 code 2 */
} pg_local_to_utf_combined;

入力と出力

本構造体はデータ保持用であり、関数ではありません。変換処理の過程で、ローカルコードをキーとして検索され、対応するUTF-8コードが読み出されます。

前提条件と不変条件

  • code はローカルエンコーディングにおける文字コードを表します。
  • utf1 および utf2 は、対応するUnicodeコードポイントを保持します。

アルゴリズムまたは振る舞い

主に変換テーブルの要素として使用されます。src/backend/utils/mb/conv.c において、この構造体の配列（cmap）に対してバイナリサーチ（compare4 を使用）を行うことで、効率的なコード変換を実現します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/utils/mb/conv.c: 変換テーブルの検索および比較処理で使用。
  • src/include/mb/pg_wchar.h: 変換関数の引数定義で使用。
  • **呼び出し先**:
  • 本構造体はデータ構造であるため、直接的な関数呼び出しは行いません。

関連データ構造

  • pg_wchar: PostgreSQL内部で文字を扱うための符号なし整数型。

失敗モードと境界条件

  • 検索対象の code がテーブル内に存在しない場合、変換処理側でエラーハンドリングが必要です。
  • cmapsize（テーブルサイズ）の指定が誤っている場合、バイナリサーチ時にメモリ境界外アクセスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は変換テーブルの静的定義に使用されます。テーブルの順序はバイナリサーチの前提条件となるため、変更時は compare4 関数との整合性に注意してください。
