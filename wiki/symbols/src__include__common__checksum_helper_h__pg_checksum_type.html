pg_checksum_type

定義

  • 種別: enum
  • 定義箇所: src/include/common/checksum_helper.h:29

目的

PostgreSQL全体で利用されるチェックサムアルゴリズムを識別するための列挙型です。CRC-32Cのような高速なエラー検出から、SHA-2ファミリーによる暗号学的ハッシュまでを統一的に扱うための抽象化レイヤーとして機能します。

シグネチャまたは形状

typedef enum pg_checksum_type
{
	CHECKSUM_TYPE_NONE,
	CHECKSUM_TYPE_CRC32C,
	CHECKSUM_TYPE_SHA224,
	CHECKSUM_TYPE_SHA256,
	CHECKSUM_TYPE_SHA384,
	CHECKSUM_TYPE_SHA512,
} pg_checksum_type;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: チェックサムアルゴリズムの識別子

前提条件と不変条件

  • 本列挙型は pg_checksum_context 構造体と組み合わせて使用されます。
  • MD5は本モジュールではサポートされていません。

アルゴリズムまたは振る舞い

  • CHECKSUM_TYPE_NONE: チェックサム計算を行わない状態を示します。
  • CHECKSUM_TYPE_CRC32C: 高速なエラー検出用アルゴリズム。
  • CHECKSUM_TYPE_SHA224 ～ CHECKSUM_TYPE_SHA512: SHA-2ファミリーを用いた暗号学的ハッシュ計算。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/backup/backup_manifest.c
  • src/backend/backup/basebackup.c
  • src/backend/backup/basebackup_incremental.c
  • src/bin/pg_combinebackup/backup_label.c / .h
  • src/bin/pg_combinebackup/load_manifest.c / .h
  • src/bin/pg_combinebackup/pg_combinebackup.c
  • src/bin/pg_combinebackup/reconstruct.c
  • 呼び出し先: なし（本シンボルは列挙型定義であり、関数ではないため）

関連データ構造

  • pg_checksum_context: アルゴリズムの種類と内部状態を保持する構造体。
  • pg_checksum_raw_context: 内部的なハッシュコンテキスト。

失敗モードと境界条件

  • 適切なアルゴリズムを選択しない場合、計算結果が期待通りにならない可能性があります。
  • pg_checksum_final を呼び出さない場合、内部的に確保されたリソースが解放されない可能性があります。

保守担当者向けメモ

  • 新しいアルゴリズムを追加する場合は、本列挙型に定数を追加し、src/common/checksum_helper.c 内の対応する処理を更新してください。
  • 暗号学的な安全性が必要な場合は、SHA-2ファミリーを選択するよう設計されています。
