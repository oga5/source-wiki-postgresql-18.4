ginxlogInsertDataInternal

定義

  • 種別: struct
  • 定義箇所: src/include/access/ginxlog.h:97

目的

GINインデックスのデータページに対する挿入操作をWAL（Write-Ahead Log）に記録するための構造体です。

シグネチャまたは形状

typedef struct
{
	OffsetNumber offset;
	PostingItem newitem;
} ginxlogInsertDataInternal;

入力と出力

  • **入力**: GINデータページへの挿入位置（offset）および挿入対象のアイテム情報（newitem）。
  • **出力**: WALレコードの一部としてシリアライズされたバイナリデータ。

前提条件と不変条件

  • 本構造体はWALレコードのペイロードとして使用されるため、メモリレイアウトは固定されています。
  • 構造体内のフィールド順序およびサイズは、リカバリ時の memcpy による読み取りと互換性を保つ必要があります。

アルゴリズムまたは振る舞い

1. GINインデックスのデータページ更新時、この構造体に挿入先のオフセットとアイテム情報を格納します。 2. XLogInsert を通じてWALに書き出されます。 3. リカバリ時には、ginxlog.c 等のREDO処理において、この構造体をWALレコードからキャストして取得し、指定されたオフセットに newitem を挿入します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/gindatapage.c において、データページへの挿入操作時に本構造体が初期化・使用されます。
  • **呼び出し先**: src/backend/access/gin/ginxlog.c および src/backend/access/rmgrdesc/gindesc.c において、WALレコードのデコードおよびREDO処理のために参照されます。

関連データ構造

  • OffsetNumber: ページ内のアイテムオフセット。
  • PostingItem: GINインデックスのポスティングリストを構成するアイテム構造体。

失敗モードと境界条件

  • WALレコードのサイズが適切に計算されていない場合、memcpy による読み取り時にバッファオーバーフローやデータ破損が発生する可能性があります。
  • 構造体のアライメントやパディングがプラットフォーム間で不一致を起こすと、リカバリ時に不正な値を読み取るリスクがあります。

保守担当者向けメモ

  • 本構造体はWALのバイナリフォーマットの一部です。フィールドの追加や変更を行う際は、既存のWALログとの互換性を維持するため、パディングやアライメントに細心の注意を払ってください。
