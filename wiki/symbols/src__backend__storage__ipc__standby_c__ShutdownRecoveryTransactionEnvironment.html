ShutdownRecoveryTransactionEnvironment

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/standby.c:160

目的

ホットスタンバイ環境におけるトランザクション追跡状態を破棄し、関連するリソースを解放する。

シグネチャまたは形状

void ShutdownRecoveryTransactionEnvironment(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルなハッシュテーブルの状態を更新）

前提条件と不変条件

  • RecoveryLockHash が NULL の場合、本関数は何もしない（冪等性を保持）。
  • 呼び出し後、RecoveryLockHash および RecoveryLockXidHash は NULL にリセットされる。

アルゴリズムまたは振る舞い

1. RecoveryLockHash が NULL であれば早期リターンする。 2. ExpireAllKnownAssignedTransactionIds() を呼び出し、追跡中の進行中トランザクションをすべて終了済みとしてマークする。 3. StandbyReleaseAllLocks() を呼び出し、追跡対象のトランザクションが保持していたすべてのロックを解放する。 4. RecoveryLockHash および RecoveryLockXidHash を破棄し、ポインタを NULL に設定する。 5. VirtualXactLockTableCleanup() を呼び出し、仮想トランザクションのロックテーブルをクリーンアップする。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/xlog.c:6224
  • src/backend/postmaster/startup.c:207
  • 呼び出し先:
  • ExpireAllKnownAssignedTransactionIds()
  • StandbyReleaseAllLocks()
  • hash_destroy()
  • VirtualXactLockTableCleanup()

関連データ構造

  • RecoveryLockHash (グローバル変数)
  • RecoveryLockXidHash (グローバル変数)

失敗モードと境界条件

  • 本関数は RecoveryLockHash が既に NULL である場合を安全に処理するため、プロセス終了時に複数回呼び出されても問題ない設計となっている。

保守担当者向けメモ

  • 本関数はリカバリプロセスの終了処理の一部として実行される。
  • トランザクション追跡の初期化状態と同期しているため、InitRecoveryTransactionEnvironment() と対になる処理である。
