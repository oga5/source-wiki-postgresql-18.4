PGTYPESnumeric_mul

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/numeric.c:895

目的

2つの numeric 型変数の積を計算し、結果を第3の numeric 型変数に格納する。

シグネチャまたは形状

int PGTYPESnumeric_mul(numeric *var1, numeric *var2, numeric *result)

入力と出力

  • **入力**:
  • var1, var2: 乗算対象の numeric 構造体へのポインタ。
  • result: 計算結果を格納する numeric 構造体へのポインタ。
  • **出力**:
  • 成功時は 0 を返す。
  • メモリ確保に失敗した場合は -1 を返す。
  • result が指す構造体のメンバ（buf, digits, ndigits, weight, rscale, sign, dscale）が更新される。

前提条件と不変条件

  • result が指す既存のバッファは digitbuf_free により解放され、新しい計算結果のバッファで置き換えられる。
  • var1 と var2 の符号が一致すれば結果は正、異なれば負となる。

アルゴリズムまたは振る舞い

1. 結果の符号を決定し、必要な桁数を算出して digitbuf_alloc でメモリを確保する。 2. 筆算アルゴリズムに基づき、var1 と var2 の各桁を乗算して res_digits に加算する。 3. 計算後、必要に応じて四捨五入（res_digits[i] > 4 の場合）を行う。 4. 先頭および末尾の不要なゼロを除去し、weight と ndigits を調整する。 5. 結果がゼロの場合は符号を正に正規化する。 6. 最終的な値を result 構造体に反映する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test2.c
  • **呼び出し先**:
  • digitbuf_alloc（メモリ確保）
  • digitbuf_free（メモリ解放）
  • memset（標準ライブラリ）

関連データ構造

  • numeric: 数値の符号、小数点位置（weight）、桁数、および数値を保持する構造体。

失敗モードと境界条件

  • **メモリ不足**: digitbuf_alloc が NULL を返した場合、関数は -1 を返し終了する。
  • **ゼロの結果**: 計算結果がゼロの場合、ndigits は 0 となり、weight は 0、符号は正に設定される。

保守担当者向けメモ

  • 本関数は result の既存メモリを解放するため、呼び出し元で result が適切に初期化されているか、あるいは PGTYPESnumeric_new で確保されたものである必要がある。
  • rscale は var1->rscale + var2->rscale として算出される。
