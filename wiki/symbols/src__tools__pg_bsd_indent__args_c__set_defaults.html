set_defaults

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/args.c:245

目的

プログラム起動時に、グローバルな設定変数群を pro テーブルで定義されたデフォルト値で初期化する。

シグネチャまたは形状

void set_defaults(void)

入力と出力

  • 入力: なし（グローバル変数 pro および ps を参照）
  • 出力: なし（グローバル変数を直接更新）

前提条件と不変条件

  • pro テーブルが有効なオプション定義を保持していること。
  • ps（パーサー状態）構造体が初期化可能であること。

アルゴリズムまたは振る舞い

1. ps.case_indent を 0.0 に設定する（浮動小数点型のためテーブル初期化の対象外）。 2. pro 配列を反復処理し、各要素 p について以下を実行する。

  • p->p_type が PRO_SPECIAL でない場合、p->p_obj が指すグローバル変数に p->p_default の値を代入する。

呼び出し元と呼び出し先

  • 呼び出し元: src/tools/pg_bsd_indent/indent.c:187 の main 関数。
  • 呼び出し先: なし。

関連データ構造

  • struct pro: オプションの定義テーブル（名前、型、デフォルト値、対象変数のポインタを保持）。
  • ps (parser_state): 整形処理の状態を保持するグローバル構造体。

失敗モードと境界条件

  • pro テーブルの終端（p_name が NULL）に達するまでループが継続される。
  • PRO_SPECIAL 型のオプションは、この関数による自動代入の対象から除外される。

保守担当者向けメモ

  • オプションを追加する場合は pro[] 配列を編集するだけでよいが、浮動小数点数など特殊な型の場合はこの関数内に個別の初期化処理が必要となる。
  • pro[] テーブルの順序は eqin 関数による前方一致検索に依存するため、サブストリングの関係にあるオプション定義順序には注意が必要。
