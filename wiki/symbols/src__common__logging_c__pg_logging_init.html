pg_logging_init

定義

  • 種別: function
  • 定義箇所: src/common/logging.c:82

目的

PostgreSQLフロントエンドプログラムのログ出力システムを初期化し、プログラム名の設定、ログレベルの初期値設定、および環境変数に基づく出力設定（色付け等）を行う。

シグネチャまたは形状

void pg_logging_init(const char *argv0)

入力と出力

  • **入力**: argv0 (プログラムの実行パス。get_progname に渡され、ログ出力時のプログラム名として使用される)
  • **出力**: なし（副作用としてグローバルなログ設定状態を更新する）

前提条件と不変条件

  • stderr に対してログを出力する。
  • Windows環境では enable_vt_processing() を呼び出し、VT100エスケープシーケンスの処理を有効化する。
  • stderr はバッファリングされない設定（_IONBF）に変更される。

アルゴリズムまたは振る舞い

1. **端末判定**: isatty(fileno(stderr)) を用いて端末出力を判定し、Windowsの場合はVT100対応を確認する。 2. **初期化**: stderr を非バッファリングに設定し、progname を抽出、__pg_log_level を PG_LOG_INFO に初期化する。 3. **色設定**: 環境変数 PG_COLOR が always または auto（かつ端末が対応）の場合、色付けを有効化する。 4. **SGRパラメータ設定**: 環境変数 PG_COLORS が存在する場合、その値を解析して sgr_error, sgr_warning, sgr_note, sgr_locus に割り当てる。未設定の場合はデフォルト値を使用する。

呼び出し元と呼び出し先

  • **呼び出し元**: initdb, pg_dump, pg_basebackup, pg_checksums 等のPostgreSQLフロントエンドツール（main関数内での初期化時に呼び出される）。
  • **呼び出し先**: getenv, isatty, fileno, get_progname, setvbuf, strdup, strsep, strchr, strcmp, free。Windows環境では enable_vt_processing を呼び出す。

関連データ構造

  • __pg_log_level: 現在のログ出力閾値。
  • sgr_error, sgr_warning, sgr_note, sgr_locus: ログの色付けに使用されるSGR（Select Graphic Rendition）エスケープシーケンス文字列。

失敗モードと境界条件

  • strdup によるメモリ確保が失敗した場合、色設定の解析はスキップされる。
  • PG_COLORS の形式が不正な場合、解析処理は適切にトークンを処理するが、一部のSGR変数が初期化されない可能性がある。

保守担当者向けメモ

  • 本関数はフロントエンドプログラムの起動時に一度だけ呼び出されることを想定している。
  • Windows環境でのVT100対応は enable_vt_processing に依存しているため、プラットフォーム固有の挙動変更時には注意すること。
  • ログレベルの変更は本関数ではなく pg_logging_set_level を使用すること。
