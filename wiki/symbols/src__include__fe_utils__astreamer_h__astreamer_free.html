astreamer_free

定義

  • 種別: function
  • 定義箇所: src/include/fe_utils/astreamer.h:154

目的

astreamer オブジェクトおよびそのパイプラインに関連付けられたリソースを解放します。

シグネチャまたは形状

static inline void astreamer_free(astreamer *streamer)

入力と出力

  • **入力**: astreamer *streamer（解放対象のストリームオブジェクトへのポインタ）
  • **出力**: なし（void）

前提条件と不変条件

  • streamer は NULL であってはなりません（Assert により検証されます）。
  • streamer->bbs_ops が有効な astreamer_ops 構造体を指しており、その中の free 関数ポインタが実装されている必要があります。

アルゴリズムまたは振る舞い

1. streamer が NULL でないことを確認します。 2. streamer が保持する bbs_ops（操作関数群）の free コールバックを呼び出し、対象のストリームに関連するメモリやリソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_verifybackup/pg_verifybackup.c
  • src/fe_utils/astreamer_gzip.c
  • src/fe_utils/astreamer_lz4.c
  • src/fe_utils/astreamer_tar.c
  • src/fe_utils/astreamer_zstd.c
  • src/bin/pg_basebackup/astreamer_inject.c
  • **呼び出し先**: streamer->bbs_ops->free（各実装固有の解放関数）

関連データ構造

  • astreamer: ストリーム処理の基底構造体。
  • astreamer_ops: free コールバックを保持する操作定義構造体。

失敗モードと境界条件

  • streamer が NULL の場合、アサーションエラーが発生します。
  • bbs_ops が初期化されていない、または free 関数ポインタが NULL の場合、未定義の動作（セグメンテーション違反等）を引き起こします。

保守担当者向けメモ

  • 本関数はフロントエンド環境で動作するため、メモリコンテキストが存在しません。メモリリークを防ぐため、パイプラインの終了時には必ず本関数を呼び出す必要があります。
  • パイプラインが連結されている場合、呼び出し元で bbs_next を再帰的に解放する設計となっているケースが多いため、呼び出し順序に注意してください。
