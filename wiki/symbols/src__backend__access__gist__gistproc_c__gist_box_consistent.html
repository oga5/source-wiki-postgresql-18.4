gist_box_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistproc.c:112

目的

GiSTインデックスにおいて、矩形（box）データ型に対するクエリ条件とインデックスエントリの整合性を判定します。

シグネチャまたは形状

Datum gist_box_consistent(PG_FUNCTION_ARGS)
  • 引数: PG_FUNCTION_ARGS（GISTENTRY*, BOX*, StrategyNumber, Oid, bool*）
  • 戻り値: Datum（内部で bool に変換）

入力と出力

  • 入力:
  • entry: 評価対象のインデックスエントリ。
  • query: 比較対象の矩形。
  • strategy: 適用する演算子戦略番号。
  • recheck: 呼び出し元へ返す再チェックフラグ。
  • 出力:
  • 整合性判定結果（true または false）。
  • *recheck: 常に false を設定（本関数による判定は常に正確であるため）。

前提条件と不変条件

  • entry->key または query が NULL の場合、整合性は false と判定されます。
  • 本関数が処理するすべてのケースは「正確（exact）」であると見なされ、再チェックは不要です。

アルゴリズムまたは振る舞い

1. recheck フラグを false に初期化します。 2. 入力値のいずれかが NULL であれば false を返します。 3. GIST_LEAF(entry) を用いて、現在のノードがリーフか内部ノードかを判定します。

  • リーフノードの場合: gist_box_leaf_consistent を呼び出し、結果を返します。
  • 内部ノードの場合: rtree_internal_consistent を呼び出し、結果を返します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース excerpt には明示的な呼び出し元の記述はありません。
  • 呼び出し先:
  • gist_box_leaf_consistent
  • rtree_internal_consistent

関連データ構造

  • GISTENTRY: GiSTインデックスのエントリ構造体。
  • BOX: 2次元矩形を表すデータ構造。

失敗モードと境界条件

  • entry->key が NULL の場合、整合性は false となります。
  • query が NULL の場合、整合性は false となります。

保守担当者向けメモ

  • 本関数は box 型専用の整合性判定ロジックをカプセル化しています。
  • 内部ノードとリーフノードで処理が分岐するため、修正時は両方のパス（gist_box_leaf_consistent および rtree_internal_consistent）への影響を考慮してください。
