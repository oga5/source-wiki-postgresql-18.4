linkFile

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/file.c:189

目的

データディレクトリ間でファイルをハードリンクし、物理的なデータ移行を効率化する。

シグネチャまたは形状

void linkFile(const char *src, const char *dst, const char *schemaName, const char *relName);

入力と出力

  • **入力**:
  • src: 転送元ファイルパス。
  • dst: 転送先ファイルパス。
  • schemaName: 関連するスキーマ名（エラーログ用）。
  • relName: 関連するリレーション名（エラーログ用）。
  • **出力**: なし（成功時は復帰、失敗時は pg_fatal によりプロセス終了）。

前提条件と不変条件

  • src から dst へのハードリンク作成が OS レベルで許可されていること。
  • 呼び出し元は、この関数を呼び出す前に check_hard_link() 等でリンク可能かを確認することが推奨される。

アルゴリズムまたは振る舞い

1. POSIX システムコール link(src, dst) を実行する。 2. 戻り値が 0 未満（エラー）の場合、pg_fatal を呼び出し、スキーマ名、リレーション名、およびシステムエラーメッセージ（%m）を含む詳細なエラー情報を出力してプログラムを終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/relfilenumber.c:640
  • **呼び出し先**: OS システムコール link()

関連データ構造

  • なし（文字列ポインタのみを使用）。

失敗モードと境界条件

  • **失敗モード**: link() システムコールが失敗した場合（権限不足、ファイルシステムの境界を越えるリンク、ソースファイルが存在しない等）、pg_fatal により即座に移行処理が中断される。

保守担当者向けメモ

  • 本関数は pg_upgrade における物理ファイル転送層の一部である。
  • エラー発生時のログには %m を使用しているため、呼び出し時には errno が適切に設定されていることが前提となる。
