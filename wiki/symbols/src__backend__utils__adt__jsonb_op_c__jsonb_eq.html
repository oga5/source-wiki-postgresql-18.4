jsonb_eq

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_op.c:221

目的

2つの jsonb 型の値が等しいかどうかを判定し、SQLレベルの等価演算子（=）の結果を返します。

シグネチャまたは形状

Datum jsonb_eq(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの jsonb 型のポインタ（jba, jbb）を受け取ります。
  • **出力**: 2つの値が等しい場合は true、そうでない場合は false を Datum 型として返します。

前提条件と不変条件

  • 入力される jsonb ポインタは有効なメモリ領域を指している必要があります。
  • compareJsonbContainers 関数が、2つの jsonb コンテナの等価性を正しく判定できる状態である必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_JSONB_P マクロを使用して、引数から2つの Jsonb 構造体を取得します。 2. compareJsonbContainers 関数を呼び出し、両者の root 構造を比較します。 3. 比較結果が 0（等価）であれば res を true に設定します。 4. PG_FREE_IF_COPY を使用して、必要に応じて一時的なメモリ領域を解放します。 5. PG_RETURN_BOOL を使用して、判定結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（fmgr）を介してSQLから呼び出されます。
  • **呼び出し先**:
  • compareJsonbContainers（jsonb コンテナの比較を実行）
  • PG_GETARG_JSONB_P, PG_FREE_IF_COPY, PG_RETURN_BOOL（PostgreSQLの関数管理マクロ）

関連データ構造

  • Jsonb: jsonb 型の内部表現を保持する構造体。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • 入力が NULL の場合、PG_GETARG_JSONB_P の挙動に従い、呼び出し元で適切に処理される必要があります（本関数内での明示的な NULL チェックは記述されていません）。
  • compareJsonbContainers が内部的にエラーを発生させた場合、本関数も中断されます。

保守担当者向けメモ

  • 本関数は jsonb 型の比較演算の基本となるため、変更時は jsonb_cmp や他の比較演算子（jsonb_lt 等）との整合性に注意してください。
  • compareJsonbContainers の実装変更が本関数の挙動に直接影響します。
