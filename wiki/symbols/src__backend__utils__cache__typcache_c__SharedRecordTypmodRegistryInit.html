SharedRecordTypmodRegistryInit

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/typcache.c:2196

目的

並列クエリ実行時に使用する共有メモリ上のレコード型レジストリを初期化し、現在のバックエンドが保持するプライベートなレコード型キャッシュ情報を共有メモリへ移行する。

シグネチャまたは形状

void SharedRecordTypmodRegistryInit(SharedRecordTypmodRegistry *registry,
                                   dsm_segment *segment,
                                   dsa_area *area)

入力と出力

  • registry: 初期化対象となる共有レジストリ構造体へのポインタ。
  • segment: 共有メモリセグメント。
  • area: 共有メモリ割り当て領域（dsa_area）。
  • 出力: なし（副作用として registry の各フィールドおよび CurrentSession の共有レジストリ関連フィールドが設定される）。

前提条件と不変条件

  • 並列ワーカープロセス内ではないこと（!IsParallelWorker()）。
  • 現在のセッションにおいて、共有レジストリや共有テーブルが未設定であること。
  • TopMemoryContext を使用してハッシュテーブルを作成する。

アルゴリズムまたは振る舞い

1. dshash を用いて、タプル記述子を自身でインデックスする record_table と、typmod でインデックスする typmod_table を作成する。 2. registry 構造体に作成したハッシュテーブルのハンドルを格納し、next_typmod を初期化する。 3. 現在のバックエンドが保持するプライベートな RecordCacheArray を走査し、各エントリを共有メモリへコピーする。 4. コピーした各 TupleDesc を typmod_table および record_table に挿入する。 5. CurrentSession の共有レジストリ関連ポインタを更新し、セッション全体で共有状態を参照できるようにする。 6. on_dsm_detach を使用して、共有メモリデタッチ時のクリーンアップフック（shared_record_typmod_registry_detach）を登録する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/common/session.c:127
  • 呼び出し先: dshash_create, dshash_get_hash_table_handle, pg_atomic_init_u32, share_tupledesc, dshash_find_or_insert, dshash_release_lock, on_dsm_detach

関連データ構造

  • SharedRecordTypmodRegistry: 共有レジストリの管理構造体。
  • dshash_table: 共有ハッシュテーブル。
  • RecordCacheArray: バックエンドローカルなレコード型キャッシュ。
  • CurrentSession: セッション状態を保持するグローバル変数。

失敗モードと境界条件

  • dshash_find_or_insert で重複する typmod が検出された場合、elog(ERROR) により異常終了する。
  • IsParallelWorker() が真の場合、Assert により停止する。

保守担当者向けメモ

  • 本関数は、リーダープロセスが並列クエリを開始する際に一度だけ実行されることを想定している。
  • RecordCacheArray からのコピー処理において、TupleDesc の共有メモリへの配置（share_tupledesc）が必須である。
