ginhandler

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginutil.c:37

目的

GIN (Generalized Inverted Index) アクセスメソッドのルーチンテーブル（IndexAmRoutine）を構築し、PostgreSQL のインデックス管理システムに GIN の機能を提供するためのエントリポイントとして機能します。

シグネチャまたは形状

Datum ginhandler(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（標準的な PostgreSQL 関数呼び出し引数）。
  • **出力**: Datum 型として IndexAmRoutine 構造体へのポインタを返します。

前提条件と不変条件

  • 本関数は、GIN インデックスの作成時に呼び出される必要があります。
  • 返される IndexAmRoutine 構造体は、GIN がサポートする機能（挿入、スキャン、バキューム等）を正しくマッピングしている必要があります。

アルゴリズムまたは振る舞い

1. makeNode(IndexAmRoutine) を使用して、インデックスアクセスメソッドのルーチン構造体をメモリ上に確保します。 2. GIN の特性に合わせて、以下のフラグを設定します。

  • amcanmulticol = true（複数列インデックスをサポート）
  • amoptionalkey = true（オプションのキーをサポート）
  • amstorage = true（ストレージをサポート）
  • amcanbuildparallel = true（並列ビルドをサポート）
  • amusemaintenanceworkmem = true（メンテナンス用メモリ設定を尊重）
  • その他、順序付けやハッシュ検索などは false に設定。

    • 3. GIN 固有の関数ポインタ（ginbuild, gininsert, ginbulkdelete, ginvacuumcleanup, gincostestimate, ginoptions, ginbeginscan, gingetbitmap, ginendscan 等）を構造体に代入します。 4. PG_RETURN_POINTER を介して構築したルーチン構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のインデックス作成プロセス（具体的な呼び出し元はソースコードの文脈からは特定不可）。
  • **呼び出し先**: 構造体に登録された各 GIN 関連関数（ginbuild, gininsert, ginbulkdelete, ginvacuumcleanup, gincostestimate, ginoptions, ginbuildphasename, ginvalidate, ginadjustmembers, ginbeginscan, ginrescan, gingetbitmap, ginendscan）。

関連データ構造

  • IndexAmRoutine: GIN アクセスメソッドの機能定義を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には makeNode がエラーを発生させる可能性があります。
  • 登録された関数ポインタが NULL の場合、対応する操作（例: amgettuple）は実行時に利用不可となります。

保守担当者向けメモ

  • 本関数は GIN のインターフェース定義の要です。新しい GIN 機能を追加または廃止する際は、このルーチンテーブルの更新が必須となります。
  • amparallelvacuumoptions には VACUUM_OPTION_PARALLEL_BULKDEL および VACUUM_OPTION_PARALLEL_CLEANUP が設定されており、並列バキュームの挙動を制御しています。
