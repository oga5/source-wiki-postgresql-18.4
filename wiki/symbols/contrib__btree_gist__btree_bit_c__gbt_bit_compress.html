gbt_bit_compress

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bit.c:127

目的

GiST インデックス構築時に、BIT 型および VARBIT 型のインデックスエントリを圧縮形式に変換する。

シグネチャまたは形状

Datum gbt_bit_compress(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（GISTENTRY 構造体へのポインタを含む）
  • 出力: Datum（圧縮されたインデックスエントリ）

前提条件と不変条件

  • PG_GETARG_POINTER(0) を介して有効な GISTENTRY 構造体が取得できること。
  • tinfo（gbtree_vinfo 構造体）が、本関数内で gbt_var_compress に渡すための適切な設定値として定義されていること。

アルゴリズムまたは振る舞い

1. 引数から GISTENTRY 構造体を取得する。 2. btree_utils_var.c で定義されている汎用関数 gbt_var_compress を呼び出す。 3. その際、BIT/VARBIT 型固有の処理テーブルである tinfo を引数として渡し、圧縮処理を委譲する。 4. gbt_var_compress の戻り値をそのまま返す。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQL の GiST インデックス構築プロセス（直接の呼び出し元コードはソース抜粋に含まれていない）。
  • 呼び出し先: gbt_var_compress（btree_utils_var.c 内の関数）。

関連データ構造

  • GISTENTRY: GiST インデックスのエントリ情報を保持する構造体。
  • tinfo: gbtree_vinfo 型の静的構造体。BIT/VARBIT 型の比較や変換ロジックを保持する。

失敗モードと境界条件

  • 入力が NULL の場合や、GISTENTRY が不正な場合は gbt_var_compress の挙動に依存する。

保守担当者向けメモ

  • 本関数は gbt_var_compress への単なるラッパーである。型固有の圧縮ロジックを変更する場合は、tinfo の定義内容を確認すること。
  • PG_FUNCTION_INFO_V1(gbt_bit_compress) により、PostgreSQL の関数マネージャに登録されている。
