PQchangePassword

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/libpq/fe-auth.c:1530

目的

指定されたデータベース接続上で、特定のユーザーのパスワードを変更するための ALTER USER SQL コマンドを構築し、実行します。

シグネチャまたは形状

PGresult *PQchangePassword(PGconn *conn, const char *user, const char *passwd);

入力と出力

  • **入力**:
  • conn: データベース接続ハンドル (PGconn*)。
  • user: パスワードを変更する対象のユーザー名。
  • passwd: 新しいパスワード（平文）。
  • **出力**:
  • 成功時: SQL 実行結果を含む PGresult*。
  • 失敗時: NULL（内部的なエラー発生時）。

前提条件と不変条件

  • conn は有効な接続状態である必要があります。
  • 内部で PQencryptPasswordConn を呼び出すため、接続設定に基づいた適切な暗号化が行われます。
  • 実行される SQL は ALTER USER <user> PASSWORD <passwd> の形式です。

アルゴリズムまたは振る舞い

1. PQencryptPasswordConn を使用して、指定されたユーザー名とパスワードから暗号化済みパスワードを生成します。 2. 生成された暗号化パスワードを PQescapeLiteral でエスケープします。 3. 対象ユーザー名を PQescapeIdentifier でエスケープします。 4. PQExpBuffer を使用して ALTER USER 文を構築します。 5. PQexec を呼び出して SQL をサーバーへ送信し、結果を返します。 6. 構築に使用したメモリ（エスケープ済み文字列およびバッファ）を適切に解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c:2592 からの呼び出しが確認されています。
  • **呼び出し先**:
  • PQencryptPasswordConn
  • PQescapeLiteral
  • PQescapeIdentifier
  • PQexec
  • initPQExpBuffer / termPQExpBuffer / printfPQExpBuffer
  • PQfreemem

関連データ構造

  • PGconn: データベース接続状態を保持。
  • PGresult: SQL 実行結果を保持。
  • PQExpBufferData: SQL 文構築用の一時バッファ。

失敗モードと境界条件

  • PQencryptPasswordConn が失敗した場合、NULL を返します（エラーは内部で記録されます）。
  • PQescapeLiteral または PQescapeIdentifier が失敗した場合、それまでに確保したメモリを解放し NULL を返します。
  • PQexec が失敗した場合、その結果（エラー情報を含む PGresult）をそのまま返します。

保守担当者向けメモ

  • 本関数は libpq の公開 API です。変更時は libpq-fe.h の宣言との整合性に注意してください。
  • SQL インジェクションを防ぐため、ユーザー名とパスワードは必ずエスケープ処理を通す必要があります。
