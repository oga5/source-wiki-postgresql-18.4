test_resowner_remember_between_phases

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_resowner/test_resowner_basic.c:162

目的

ResourceOwner の解放フェーズ開始後に新しいリソースを登録しようとした際、期待通りエラーが発生することを確認するテスト関数です。

シグネチャまたは形状

Datum test_resowner_remember_between_phases(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS (PostgreSQLの関数呼び出し規約に基づく引数)
  • 出力: Datum (通常は void を返すが、本関数はエラーを発生させることを意図しているため到達しない)

前提条件と不変条件

  • ResourceOwnerRelease が呼び出された後の ResourceOwner に対して ResourceOwnerRemember を実行することは禁止されている。
  • 本関数は、この制約が正しく適用され、エラーがスローされることを不変条件として検証する。

アルゴリズムまたは振る舞い

1. ResourceOwnerCreate を使用して、現在のリソース所有者の子として新しい ResourceOwner を作成する。 2. ResourceOwnerRelease を呼び出し、RESOURCE_RELEASE_BEFORE_LOCKS フェーズで解放処理を開始する。 3. ResourceOwnerEnlarge を呼び出し、続いて ResourceOwnerRemember を実行する。 4. ResourceOwnerRemember の実行によりエラーが発生し、以降のコード（elog）には到達しないことを確認する。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLの関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • 呼び出し先:
  • ResourceOwnerCreate
  • ResourceOwnerRelease
  • ResourceOwnerEnlarge
  • ResourceOwnerRemember
  • elog

関連データ構造

  • ResourceOwner: リソースの所有権を管理する構造体。
  • string_desc: リソースの解放処理を定義する ResourceOwnerDesc。

失敗モードと境界条件

  • 本関数は、ResourceOwnerRemember が意図的に失敗（エラー発生）することを前提としており、エラーが発生しない場合は elog(ERROR, ...) によりテスト失敗として扱われる。

保守担当者向けメモ

  • 本関数は ResourceOwner の状態遷移制約をテストする目的で作成されています。ResourceOwnerRelease 呼び出し後の操作制限が変更された場合、本テストの期待値も更新する必要があります。
