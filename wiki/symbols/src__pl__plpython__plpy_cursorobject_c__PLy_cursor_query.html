PLy_cursor_query

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_cursorobject.c:96

目的

SQL クエリ文字列から PostgreSQL のカーソルを生成し、それをラップする Python オブジェクト（PLyCursorObject）を作成します。

シグネチャまたは形状

PyObject *PLy_cursor_query(const char *query)

入力と出力

  • **入力**: query (実行する SQL クエリ文字列)
  • **出力**: 成功時は PLyCursorObject をキャストした PyObject*、失敗時は NULL。

前提条件と不変条件

  • 呼び出し時に有効な PLyExecutionContext が存在すること。
  • PLyCursorType が初期化済みであること。
  • 戻り値の PLyCursorObject は、portalname が非 NULL であることが保証されます。

アルゴリズムまたは振る舞い

1. PLyCursorObject をメモリ上に確保し、専用のメモリコンテキスト (mcxt) を作成します。 2. PLy_input_setup_func を呼び出し、結果セットのタプル変換用設定を行います。 3. PLy_spi_subtransaction_begin により SPI サブトランザクションを開始します。 4. PG_TRY ブロック内で以下を実行します：

  • SPI_prepare でクエリをプリペアドプランに変換。
  • SPI_cursor_open でポータルを開き、PinPortal で固定。
  • portal->name をカーソルオブジェクトの portalname にコピー。
  • SPI_freeplan でプランを解放。
  • PLy_spi_subtransaction_commit でトランザクションを確定。

    • 5. PG_CATCH ブロックでエラー発生時に PLy_spi_subtransaction_abort を実行し、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は、同ファイル内の PLy_cursor 関数（行84）であることが確認されています。
  • **呼び出し先**: SPI_prepare, SPI_cursor_open, SPI_freeplan, PLy_spi_subtransaction_begin/commit/abort, AllocSetContextCreate, PinPortal 等の SPI およびメモリ管理関数。

関連データ構造

  • PLyCursorObject: カーソルの状態（ポータル名、クローズ状態、メモリコンテキスト）を保持。
  • PLyExecutionContext: 現在の実行コンテキスト。
  • Portal: PostgreSQL のカーソル実体。

失敗モードと境界条件

  • PyObject_New 失敗時: NULL を返します。
  • SPI_prepare または SPI_cursor_open 失敗時: elog(ERROR, ...) によりエラーを報告し、サブトランザクションをアボートして NULL を返します。
  • クエリ文字列が不正な場合: pg_verifymbstr により検証されます。

保守担当者向けメモ

  • 本関数は Python の C API を使用しており、Python 3.8 未満の環境では PLy_CursorType の参照カウント管理に注意が必要です。
  • カーソルに関連付けられたメモリコンテキストは PLyCursorObject の生存期間と同期しているため、メモリリークを防ぐために PLy_cursor_dealloc での適切な解放が必須です。
