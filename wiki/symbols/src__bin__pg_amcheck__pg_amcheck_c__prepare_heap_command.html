prepare_heap_command

定義

  • 種別: function
  • 定義箇所: src/bin/pg_amcheck/pg_amcheck.c:842

目的

ヒープリレーションの整合性を検証するための SQL クエリを構築し、指定されたバッファに格納する。

シグネチャまたは形状

static void prepare_heap_command(PQExpBuffer sql, RelationInfo *rel, PGconn *conn)

入力と出力

  • **入力**:
  • sql: クエリ文字列を格納する PQExpBuffer。
  • rel: 検証対象のリレーション情報 (RelationInfo)。
  • conn: データベース接続 (PGconn)。
  • **出力**: なし（sql バッファの内容が更新される）。

前提条件と不変条件

  • rel が有効な RelationInfo 構造体を指していること。
  • グローバル変数 opts (AmcheckOptions) が初期化済みであること。
  • rel->datinfo->amcheck_schema が検証関数を含むスキーマ名として設定されていること。

アルゴリズムまたは振る舞い

1. resetPQExpBuffer で sql バッファを初期化する。 2. verify_heapam 関数を呼び出す SQL を構築する。この際、opts の設定（on_error_stop, reconcile_toast, skip）を引数として埋め込む。 3. opts.startblock および opts.endblock が設定されている場合、対応する引数を SQL に追加する。 4. 最後に、対象リレーションの OID を指定し、一時リレーションを除外する WHERE 句を付与する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_amcheck/pg_amcheck.c 内の並列処理実行ロジック（776行目付近）から呼び出される。
  • **呼び出し先**: resetPQExpBuffer, appendPQExpBuffer（いずれも libpq 関連のユーティリティ）。

関連データ構造

  • PQExpBuffer: SQL 文字列構築用バッファ。
  • RelationInfo: 対象リレーションのメタデータ。
  • AmcheckOptions: ユーザー指定の検証オプション。

失敗モードと境界条件

  • opts.startblock や opts.endblock が負の値の場合、SQL には追加されない。
  • 一時リレーション（RELPERSISTENCE_TEMP）は検証対象から除外される。

保守担当者向けメモ

  • 本関数は verify_heapam を呼び出すための SQL を生成する。ヒープ検証は行を返す仕様であるため、結果の解析には verify_heap_slot_handler が使用される。
  • SQL 構築時に rel->datinfo->amcheck_schema を使用しているため、対象データベースごとのスキーマ設定が正しく反映される。
