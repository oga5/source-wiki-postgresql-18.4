be_tls_open_server

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-openssl.c:438

目的

PostgreSQL サーバーにおいて、クライアントとの TLS ハンドシェイクを実行し、セキュアな通信チャネルを確立する。

シグネチャまたは形状

int be_tls_open_server(Port *port)

入力と出力

  • **入力**: Port *port (接続中のソケット情報を含む構造体)
  • **出力**: 成功時は 0、失敗時は -1 を返す。

前提条件と不変条件

  • SSL_context が初期化済みであること。
  • port->ssl および port->peer が NULL であること。
  • 呼び出し元は、PostgreSQL の通信プロトコル層であること。

アルゴリズムまたは振る舞い

1. **初期設定**: SSL_CTX にデバッグ用コールバックと ALPN ネゴシエーション用コールバックを設定する。 2. **SSL オブジェクト生成**: SSL_new で新しい SSL オブジェクトを作成し、ssl_set_port_bio を用いて PostgreSQL のソケットと紐付ける。 3. **ハンドシェイク**: SSL_accept をループ内で実行する。

  • SSL_ERROR_WANT_READ または SSL_ERROR_WANT_WRITE が発生した場合、WaitLatchOrSocket を使用して I/O 準備ができるまで待機し、再試行する。
  • SSL_ERROR_SYSCALL や SSL_ERROR_SSL などのエラーが発生した場合は、適切なエラーメッセージをログ出力して終了する。

    • 4. **ALPN 確認**: ハンドシェイク完了後、SSL_get0_alpn_selected を呼び出し、ネゴシエーションされたプロトコルが PG_ALPN_PROTOCOL と一致するか検証する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/be-secure.c
  • **呼び出し先**: SSL_CTX_set_info_callback, SSL_CTX_set_alpn_select_cb, SSL_new, ssl_set_port_bio, SSL_accept, SSL_get_error, ERR_clear_error, ERR_get_error, WaitLatchOrSocket, SSL_get0_alpn_selected 等の OpenSSL API および PostgreSQL 内部関数。

関連データ構造

  • Port: 通信状態を保持する構造体。
  • SSL_context: サーバー全体で共有される SSL 設定状態。

失敗モードと境界条件

  • **SSL コンテキスト未設定**: SSL_context が NULL の場合、エラーを報告して -1 を返す。
  • **ハンドシェイク失敗**: プロトコル不一致や証明書エラーなど、OpenSSL がエラーを返した場合は COMMERROR としてログ出力し、-1 を返す。
  • **EOF 検出**: SSL_ERROR_ZERO_RETURN または SSL_ERROR_SYSCALL で EOF が検出された場合、エラーとして処理される。

保守担当者向けメモ

  • SSL_get_error() を呼び出す前に ERR_clear_error() を実行し、スレッドローカルなエラーキューをクリアすることが必須である。
  • WaitLatchOrSocket を使用する際、authentication_timeout が有効であることを前提としている。
  • ALPN の検証ロジックは、ハンドシェイク直後の SSL_get0_alpn_selected の結果に依存している。
