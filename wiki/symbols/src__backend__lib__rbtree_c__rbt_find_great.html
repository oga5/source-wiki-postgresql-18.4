rbt_find_great

定義

  • 種別: function
  • 定義箇所: src/backend/lib/rbtree.c:171

目的

赤黒木（RBTree）において、指定されたキー以上の値を持つノードを検索します。

シグネチャまたは形状

RBTNode *rbt_find_great(RBTree *rbt, const RBTNode *data, bool equal_match)

入力と出力

  • **入力**:
  • rbt: 検索対象の赤黒木構造体へのポインタ。
  • data: 検索キーとして使用するノードデータ。
  • equal_match: true の場合、キーと完全に一致するノードを優先的に返します。
  • **出力**:
  • 指定された条件を満たす最小のノードへのポインタ。該当するノードが存在しない場合は NULL を返します。

前提条件と不変条件

  • rbt は初期化済みであること。
  • rbt->comparator コールバック関数が適切に設定されていること。
  • RBTNIL が番兵として正しく機能していること。

アルゴリズムまたは振る舞い

1. 木のルートから走査を開始します。 2. comparator を用いて現在のノードと検索キーを比較します。 3. equal_match が true で、かつキーが一致（cmp == 0）した場合は、そのノードを直ちに返します。 4. キーが現在のノードより小さい場合、現在のノードを「候補（greater）」として保持し、左の子へ移動します。 5. キーが現在のノードより大きい場合、右の子へ移動します。 6. 走査終了後、保持していた「候補」ノードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_rbtree/test_rbtree.c 内のテスト関数群から、検索機能の検証のために呼び出されています。
  • **呼び出し先**: rbt->comparator（呼び出し側が提供する比較関数）。

関連データ構造

  • RBTree: 木全体を管理する制御構造体。
  • RBTNode: 木の各ノードのヘッダー。
  • RBTNIL: 葉ノードを表す静的な番兵。

失敗モードと境界条件

  • 木が空の場合、NULL を返します。
  • 検索キーより大きい値が木の中に存在しない場合、NULL を返します。
  • equal_match が false の場合、キーと完全に一致するノードはスキップされ、それより大きい最小のノードが返されます。

保守担当者向けメモ

  • 本関数は侵入型設計に基づいています。data 引数は RBTNode を含む構造体としてキャストして利用されることを前提としています。
  • 比較ロジックは rbt->comparator に依存しているため、比較関数の実装が不整合を起こすと検索結果が不正になります。
