parse_tli

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/backup_label.c:268

目的

文字列バッファからタイムラインID（TimeLineID）を抽出し、解析の成否を判定する。

シグネチャまたは形状

static bool parse_tli(char *s, char *e, TimeLineID *tli)

入力と出力

  • **入力**:
  • s: タイムラインIDの開始位置を示す文字列ポインタ。
  • e: タイムラインIDの終了位置（解析対象の末尾）を示すポインタ。
  • tli: 抽出した値を格納する TimeLineID 型のポインタ。
  • **出力**:
  • 成功した場合は true、失敗した場合は false を返す。

前提条件と不変条件

  • e は s が指す文字列内の有効な位置を指している必要がある。
  • 関数実行中、*e の位置にある文字を一時的に \0 に置換して解析を行い、終了時に元の値へ復元する。

アルゴリズムまたは振る舞い

1. *e の値を一時保存し、その位置を \0 で終端する。 2. sscanf を使用して s から符号なし整数を読み取り、tli に格納する。 3. 一時保存した値を *e に書き戻して復元する。 4. sscanf が正常に値を読み取った場合、読み取り終了位置（s[nchars]）が改行文字であることを確認する。 5. 読み取りに失敗した場合、または終了位置が改行でない場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_combinebackup/backup_label.c 内の parse_backup_label 関数（行75および94）。
  • **呼び出し先**:
  • 標準ライブラリ関数: sscanf。

関連データ構造

  • TimeLineID: PostgreSQLのタイムライン識別子。access/xlogdefs.h で定義。

失敗モードと境界条件

  • sscanf による数値変換が失敗した場合、false を返す。
  • 数値の直後に改行文字が存在しない場合、不正な形式とみなして false を返す。

保守担当者向けメモ

  • 本関数はバッファの内容を一時的に破壊（\0 の挿入）するため、呼び出し側はバッファが書き込み可能であることを保証する必要がある。
  • StringInfo バッファのカーソル位置管理と組み合わせて使用されることを想定している。
