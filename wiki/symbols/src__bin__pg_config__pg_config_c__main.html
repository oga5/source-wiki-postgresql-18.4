目的

PostgreSQL のビルド設定やディレクトリ構成情報を出力するコマンドラインツール pg_config のエントリポイントです。実行ファイルの場所を基準に設定情報を取得し、ユーザーの要求に応じて情報を表示します。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。--help や --bindir などのオプションを受け付けます。
  • **出力**:
  • 戻り値: 正常終了時は 0、エラー発生時は 1 を返します。
  • 標準出力: 設定項目名と値のペア（key = value 形式）。
  • 標準エラー出力: エラーメッセージや使用方法の助言。

前提条件と不変条件

  • 実行環境のパスが find_my_exec によって特定可能である必要があります。
  • info_items テーブルと help() 関数内の出力内容は同期している必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: ロケール設定を行い、find_my_exec で自身の実行パスを取得します。 2. **ヘルプ確認**: 引数に --help または -? が含まれる場合、help() を呼び出して終了します。 3. **データ取得**: get_configdata を使用して、インストールパスに基づいた設定情報のリストをメモリ上にロードします。 4. **出力処理**:

  • 引数がない場合: 全設定項目を列挙して終了します。
  • 引数がある場合: 各引数を info_items テーブルと照合し、一致する項目を show_item で出力します。

    • 5. **エラー処理**: 不明な引数が渡された場合はエラーメッセージを表示し、advice() を呼び出して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェルまたはプロセス起動元（ソースコード上の直接的な呼び出し元は提供されていません）。
  • **呼び出し先**:
  • set_pglocale_pgservice, get_progname, find_my_exec, get_configdata, show_item, help, advice, printf, fprintf, exit, strcmp。
  • *注記*: ツールが提示した contrib/amcheck 等の参照は、本関数の呼び出し元ではなく、同名の「main」という単語を含む別コンテキストのコードです。

関連データ構造

  • ConfigData: 設定情報の保持に使用される構造体。
  • InfoItem: コマンドライン引数と内部設定項目を紐付ける構造体。
  • info_items[]: サポートする全オプションを定義したテーブル。

失敗モードと境界条件

  • **実行パス特定失敗**: find_my_exec が失敗した場合、エラーメッセージを出力して終了します。
  • **引数不一致**: info_items に存在しない引数が指定された場合、エラーメッセージと advice() を出力して終了します。

保守担当者向けメモ

  • 本プログラムはリロケーション対応のため、パス情報はハードコードせず、実行時の実行ファイル位置から相対的に計算しています。
  • 新しいオプションを追加する際は、info_items テーブルと help() 関数の両方を更新してください。
