LogicalTapeSetCreate

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/logtape.c:555

目的

複数の論理テープを管理するための LogicalTapeSet 構造体を初期化し、物理的なストレージ（BufFile）を準備する。

シグネチャまたは形状

LogicalTapeSet *LogicalTapeSetCreate(bool preallocate, SharedFileSet *fileset, int worker)

入力と出力

  • **入力**:
  • preallocate (bool): ブロックの事前割り当てを有効にするか。
  • fileset (SharedFileSet *): 並列ソート用共有ファイルセット（指定がない場合は NULL）。
  • worker (int): ワーカーID（リーダープロセスの場合は -1）。
  • **出力**:
  • 初期化された LogicalTapeSet 構造体へのポインタ。

前提条件と不変条件

  • lts->freeBlocks は初期状態で 32 要素分のメモリが確保される。
  • lts->nBlocksAllocated, lts->nBlocksWritten, lts->nHoleBlocks は 0 に初期化される。

アルゴリズムまたは振る舞い

1. LogicalTapeSet 構造体をメモリ上に確保し、各カウンタを初期化する。 2. preallocate フラグを構造体に保存する。 3. 物理ストレージの決定:

  • fileset があり worker が -1（リーダー）の場合: pfile を NULL に設定（後続のインポートで連結するため）。
  • fileset がある場合: worker ID をファイル名として BufFileCreateFileSet を呼び出し、共有ファイルを作成。
  • それ以外（シリアルソート等）の場合: BufFileCreateTemp を使用して一時ファイルを作成。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAgg.c
  • src/backend/utils/sort/tuplesort.c
  • **呼び出し先**:
  • palloc (メモリ確保)
  • BufFileCreateFileSet (共有ファイル作成)
  • BufFileCreateTemp (一時ファイル作成)
  • pg_itoa (ファイル名生成)

関連データ構造

  • LogicalTapeSet: 本関数が初期化するメインの管理構造体。
  • SharedFileSet: 並列ソート時に使用される共有ファイル管理構造体。
  • BufFile: 物理的なファイルI/Oを抽象化する構造体。

失敗モードと境界条件

  • メモリ不足時に palloc がエラーを発生させる可能性がある。
  • BufFileCreateFileSet または BufFileCreateTemp が失敗した場合、基盤となる BufFile のエラーハンドリングに従う。

保守担当者向けメモ

  • リーダープロセス（worker == -1）において pfile が NULL になるのは、後続の LogicalTapeImport で既存のワーカー用ファイルを連結することを前提としているためである。
