目的

ECPG（Embedded SQL for C）プリプロセッサが生成する WHENEVER 指令の変換結果を検証するための回帰テスト用エントリーポイントです。SQL実行後のエラーや警告に対する制御フローの分岐が、期待通りにC言語の if 文や goto 文へ変換されているかを確認します。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし
  • 出力: 正常終了時に 0 を返します。

前提条件と不変条件

  • ECPGライブラリが初期化され、データベース接続が確立されていること。
  • sqlca 構造体がSQL実行後のステータスを保持していること。
  • 各SQL実行（ECPGdo）の直後に、sqlca.sqlcode や sqlca.sqlwarn を評価するコードが挿入されていること。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効にし、データベースへ接続します。 2. **SQL実行と検証**: create table や insert、select などのSQL操作を順次実行します。 3. **条件分岐のテスト**:

  • WHENEVER SQLWARNING: sqlca.sqlwarn[0] == 'W' の場合に warn() を呼び出すコードが生成されます。
  • WHENEVER SQLERROR: sqlca.sqlcode < 0 の場合に、指定されたアクション（sqlprint, print, print2, continue, goto）を実行するコードが生成されます。

    • 4. **制御フロー**: goto error; によるジャンプや、エラー発生時のハンドラ呼び出しを通じて、ECPGの仕様通りのエラーハンドリングが行われることを確認します。

呼び出し元と呼び出し先

  • 呼び出し元: テスト実行環境（OSまたはテストハーネス）
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGdo, ECPGtrans, sqlprint, warn, print, print2, printf

関連データ構造

  • sqlca: SQL通信領域。SQL実行後のステータス（sqlcode, sqlwarn）を保持します。

失敗モードと境界条件

  • 存在しないテーブルへのクエリ実行時に sqlca.sqlcode < 0 が発生し、定義されたエラーハンドラ（print, print2, goto 等）が適切にトリガーされることを境界条件として検証します。
  • WHENEVER ... STOP のような終了アクションについては、本ソースコード内では明示的な終了処理の検証は行われていません。

保守担当者向けメモ

  • 本ファイルは手書きのコードではなく、ECPGプリプロセッサの出力結果です。修正が必要な場合はプリプロセッサ側のロジックを確認してください。
  • goto error; はECPGの仕様に基づく標準的なエラーハンドリングパターンです。
  • 各SQL実行後に sqlca.sqlcode をチェックするコードが自動挿入されている点に注意してください。
