gist_redo

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistxlog.c:396

目的

GiSTインデックスのWAL（Write-Ahead Log）レコードを解析し、レコードの種類に応じて適切な復旧処理をディスパッチするメインエントリポイントです。

シグネチャまたは形状

void gist_redo(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）
  • **出力**: なし（副作用としてインデックスページの状態を更新）

前提条件と不変条件

  • opCtx が有効なメモリコンテキストとして存在している必要があります。
  • GiSTインデックスのリプレイ処理において、現時点では競合処理（Conflict processing）は不要とされています。

アルゴリズムまたは振る舞い

1. XLogRecGetInfo を使用してWALレコードの操作コード（info）を取得します。 2. opCtx にメモリコンテキストを切り替えます。 3. info に基づき、以下のいずれかの関数を呼び出します。

  • XLOG_GIST_PAGE_UPDATE: gistRedoPageUpdateRecord
  • XLOG_GIST_DELETE: gistRedoDeleteRecord
  • XLOG_GIST_PAGE_REUSE: gistRedoPageReuse
  • XLOG_GIST_PAGE_SPLIT: gistRedoPageSplitRecord
  • XLOG_GIST_PAGE_DELETE: gistRedoPageDelete
  • XLOG_GIST_ASSIGN_LSN: 何も行いません（gistGetFakeLSN 参照）。

    • 4. 未知の操作コードが渡された場合は elog(PANIC) を発生させます。 5. 最後にメモリコンテキストを元に戻し、opCtx をリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h でリソースマネージャ（RM_GIST_ID）として登録されており、WALリプレイプロセスから呼び出されます。
  • **呼び出し先**: gistRedoPageUpdateRecord, gistRedoDeleteRecord, gistRedoPageReuse, gistRedoPageSplitRecord, gistRedoPageDelete。

関連データ構造

  • XLogReaderState: WALレコードの読み取り状態を保持する構造体。
  • opCtx: リプレイ処理で使用されるメモリコンテキスト。

失敗モードと境界条件

  • 未知のWAL操作コードが渡された場合、PANIC レベルのログを出力してシステムを停止させます。

保守担当者向けメモ

  • 将来的にB-treeのような最適化（VACUUM外でのタプル削除など）を実装する場合、本関数内で競合処理を追加する必要があります。
  • XLOG_GIST_ASSIGN_LSN は現在NOP（何もしない）ですが、これは gistGetFakeLSN の仕様に関連しています。
