defGetString

定義

  • 種別: function
  • 定義箇所: src/backend/commands/define.c:34

目的

DefElem ノードの arg フィールドから値を抽出し、文字列形式へ変換して返します。SQL コマンドのオプション解析において、多様なデータ型を統一的に扱うためのアクセサです。

シグネチャまたは形状

char *defGetString(DefElem *def)

入力と出力

  • **入力**: DefElem *def （解析対象の定義要素ノード）
  • **出力**: 抽出された値の文字列（char *）。メモリは各変換関数内で確保または参照されます。

前提条件と不変条件

  • def->arg が NULL であってはなりません（NULL の場合はエラーを報告します）。
  • nodeTag(def->arg) がサポートされている型（Integer, Float, Boolean, String, TypeName, List, A_Star）のいずれかである必要があります。

アルゴリズムまたは振る舞い

1. def->arg が NULL の場合、ERRCODE_SYNTAX_ERROR を発生させます。 2. nodeTag(def->arg) に基づき、以下の通り型変換を行います。

  • T_Integer: long 型に変換し、psprintf で文字列化。
  • T_Float: Float ノードの fval をそのまま返却。
  • T_Boolean: true または false の文字列を返却。
  • T_String: strVal を使用して文字列を返却。
  • T_TypeName: TypeNameToString で文字列化。
  • T_List: NameListToString で文字列化。
  • T_A_Star: * を返却。

    • 3. 未知のノード型が渡された場合、elog(ERROR) で処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dict_int/dict_int.c
  • contrib/dict_xsyn/dict_xsyn.c
  • contrib/file_fdw/file_fdw.c
  • contrib/postgres_fdw/deparse.c
  • contrib/postgres_fdw/option.c
  • **呼び出し先**:
  • intVal, boolVal, strVal (マクロ)
  • castNode
  • psprintf, pstrdup
  • TypeNameToString, NameListToString
  • ereport, elog

関連データ構造

  • DefElem: src/include/nodes/parsenodes.h で定義される、SQL の WITH 句やオプション引数を保持する構造体。

失敗モードと境界条件

  • def->arg が NULL の場合: ereport により構文エラーとして処理されます。
  • 未サポートのノード型が渡された場合: elog により内部エラーとして処理されます。

保守担当者向けメモ

  • 本関数は純粋なデータ変換ユーティリティであり、データベースの状態変更は行いません。
  • 新しいノード型をサポートする必要がある場合は、switch 文にケースを追加してください。
  • 戻り値の文字列メモリ管理（pstrdup 等の利用有無）については、各変換関数の実装に依存するため注意してください。
