help

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/ecpg.c:34

目的

PostgreSQL 埋め込み SQL プリプロセッサ（ecpg）のコマンドラインオプションと使用方法を標準出力に表示する。

シグネチャまたは形状

static void help(const char *progname)

入力と出力

  • **入力**: progname（実行ファイル名を示す文字列）
  • **出力**: なし（void）。副作用として標準出力へヘルプメッセージを出力する。

前提条件と不変条件

  • progname は有効な文字列ポインタであること。
  • printf が標準出力に対して書き込み可能であること。

アルゴリズムまたは振る舞い

1. progname を用いてプログラムの概要と使用法（Usage）を表示する。 2. 利用可能なコマンドラインオプション（-c, -C, -D, -h, -i, -I, -o, -r, --regression, -t, -V, -?）の一覧を順次出力する。 3. YYDEBUG マクロが定義されている場合のみ、デバッグ用オプション -d を表示する。 4. 出力ファイル指定がない場合のデフォルトの命名規則を説明する。 5. バグ報告先およびプロジェクトのホームURLを表示して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内では、本関数を呼び出している箇所は特定されていません。
  • **呼び出し先**: 標準ライブラリの printf を使用しています。
  • 注意: ツールが提示した contrib/oid2name/oid2name.c 等の呼び出し元は、同名の別関数であり、本シンボルとは無関係です。

関連データ構造

  • なし。

失敗モードと境界条件

  • 標準出力がクローズされている場合、出力は失敗しますが、本関数自体はエラーコードを返しません。

保守担当者向けメモ

  • 新しいコマンドラインオプションを追加した際は、本関数内のメッセージも更新してください。
  • YYDEBUG 条件付きコンパイルブロックにより、ビルド構成によって出力内容が変化することに注意してください。
