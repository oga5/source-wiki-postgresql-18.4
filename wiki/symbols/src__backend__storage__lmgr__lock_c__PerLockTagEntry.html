PerLockTagEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/lmgr/lock.c:3360

目的

ロック対象オブジェクトの識別子（LOCKTAG）と、そのオブジェクトに対してセッションレベルまたはトランザクションレベルでロックが保持されているかを示すフラグを保持する構造体です。

シグネチャまたは形状

typedef struct {
    LOCKTAG lock;      /* ロック対象オブジェクトの識別子 */
    bool    sessLock;  /* セッションレベルでロックを保持しているか */
    bool    xactLock;  /* トランザクションレベルでロックを保持しているか */
} PerLockTagEntry;

入力と出力

  • 入力: 構造体内の各フィールド（lock, sessLock, xactLock）は、ロック状態の追跡に使用されます。
  • 出力: ハッシュテーブルの検索結果として、この構造体へのポインタが返されます。

前提条件と不変条件

  • 本構造体は、ロック管理用のハッシュテーブル（lockhtab）のエントリサイズとして使用されます。
  • sessLock および xactLock は、特定のロックモードが保持されているかどうかの状態フラグとして機能します。

アルゴリズムまたは振る舞い

  • hash_search を用いたハッシュテーブル操作において、エントリの型として利用されます。
  • ロックの獲得・解放処理において、特定の LOCKTAG に対応するエントリを検索し、その保持状態（セッション/トランザクション）を更新するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/lmgr/lock.c 内のハッシュテーブル初期化および検索処理。
  • 呼び出し先: 本構造体自体はデータ構造であり、関数呼び出しは行いません。

関連データ構造

  • LOCKTAG: ロック対象を識別するための構造体。
  • lockhtab: 本構造体をエントリとして格納するハッシュテーブル。

失敗モードと境界条件

  • ハッシュテーブルのメモリ不足や、検索対象の LOCKTAG が存在しない場合の hash_search の挙動に依存します。

保守担当者向けメモ

  • 本構造体は、ロックの保持レベル（セッション vs トランザクション）を区別するために重要です。変更を加える際は、ロックの解放ロジック（LockRelease 等）への影響を考慮してください。
