TableCommandResultHandler

定義

  • 種別: function
  • 定義箇所: src/fe_utils/parallel_slot.c:539

目的

テーブル操作（VACUUMやREINDEX等）の実行結果を評価し、エラー発生時に継続すべきか終了すべきかを判定する標準的なハンドラです。

シグネチャまたは形状

bool TableCommandResultHandler(PGresult *res, PGconn *conn, void *context);

入力と出力

  • **入力**:
  • res: 実行されたクエリの結果オブジェクト（PGresult）。
  • conn: クエリを実行したデータベース接続（PGconn）。
  • context: 呼び出し元から渡される任意のコンテキスト（本関数内では未使用）。
  • **出力**:
  • bool: 処理を継続する場合は true、致命的なエラーにより処理を中断すべき場合は false を返します。

前提条件と不変条件

  • res および conn は NULL であってはなりません（Assert により検証）。

アルゴリズムまたは振る舞い

1. PQresultStatus を確認し、PGRES_COMMAND_OK であれば正常終了として true を返します。 2. エラーが発生した場合、PQerrorMessage を用いてログを出力します。 3. エラーの SQLSTATE を取得し、それが ERRCODE_UNDEFINED_TABLE（テーブルが存在しない）である場合は、エラーを許容して true を返します。 4. それ以外のエラーの場合は、PQclear(res) を実行して結果を解放し、false を返して処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/scripts/reindexdb.c
  • src/bin/scripts/vacuumdb.c
  • いずれも ParallelSlotSetHandler を通じて本関数を登録しています。
  • **呼び出し先**:
  • libpq 関数群: PQresultStatus, PQresultErrorField, PQdb, PQerrorMessage, PQclear。
  • ログ出力: pg_log_error。

関連データ構造

  • PGresult: libpq のクエリ結果構造体。
  • PGconn: libpq のデータベース接続構造体。

失敗モードと境界条件

  • **テーブル不在**: ERRCODE_UNDEFINED_TABLE の場合はエラーとして扱わず、処理を継続します。
  • **その他のエラー**: データベース接続エラーや構文エラーなど、ERRCODE_UNDEFINED_TABLE 以外のすべてのエラーは致命的とみなされ、false を返します。

保守担当者向けメモ

  • 本関数は ParallelSlotResultHandler 型のシグネチャに従っています。
  • context 引数は現在使用されていませんが、将来的な拡張のためにシグネチャに含まれています。
  • エラー判定ロジックを変更する場合は、libpq のエラーコード定義との整合性に注意してください。
