dependency_degree

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/dependencies.c:220

目的

指定された属性の組み合わせにおける「関数従属性（Functional Dependency）」の度合い（0.0〜1.0）を算出します。

シグネチャまたは形状

static double dependency_degree(StatsBuildData *data, int k, AttrNumber *dependency)

入力と出力

  • **入力**:
  • data: 統計構築用データ構造。
  • k: 属性の数（2以上である必要がある）。
  • dependency: 評価対象の属性インデックスの配列。
  • **出力**:
  • double: 従属性の度合い。全行数に対する、従属性を支持する行数の割合。

前提条件と不変条件

  • k >= 2 であること（アサーションにより強制）。
  • data 内のすべての統計エントリが同一のタプル記述子を指していること。

アルゴリズムまたは振る舞い

1. **ソート準備**: 指定された k 個の属性に対し、各型のデフォルトソート演算子と照合順序を用いて MultiSortSupport を初期化します。 2. **データソート**: build_sorted_items を呼び出し、属性値に基づいてデータを辞書順にソートします。 3. **グループ化と検証**:

  • ソート済みデータを、最初の k-1 個の属性値が一致するグループに分割します。
  • 各グループ内で最後の属性（k-1 番目）の値が単一であれば、そのグループは従属性を「支持」しているとみなします。
  • 最後の属性値が複数存在する場合、そのグループは「矛盾」しているとみなします。

    • 4. **算出**: 支持された行数の合計を全行数で除算し、従属性の度合いを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/dependencies.c 内の統計構築処理（statext_dependencies_build 等）から呼び出されます。
  • **呼び出し先**: multi_sort_init, multi_sort_add_dimension, build_sorted_items, multi_sort_compare_dims, multi_sort_compare_dim 等のユーティリティ関数。

関連データ構造

  • StatsBuildData: 統計構築の入力データ。
  • MultiSortSupport: 複数列のソートを管理する構造体。
  • SortItem: ソート対象のデータ項目。

失敗モードと境界条件

  • k < 2 の場合、アサーション違反により停止します。
  • 型のソート演算子が見つからない場合、elog(ERROR) により処理が中断されます。
  • 属性数に対して計算コストが指数関数的に増加するため、大規模なデータセットでは ANALYZE の負荷が高まります。

保守担当者向けメモ

  • 本関数は、列間の相関を判定するための基礎的な計算ロジックです。
  • 現在の実装では列固有の照合順序（collation）を考慮していますが、将来的に特定の型に対する最適化が必要になる可能性があります。
  • build_sorted_items が同一のタプル記述子を前提としている点に注意してください。
