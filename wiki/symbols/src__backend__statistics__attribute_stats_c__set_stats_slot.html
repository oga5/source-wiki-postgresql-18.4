set_stats_slot

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/attribute_stats.c:778

目的

pg_statistic システムカタログの行データに対して、指定された統計種別（stakind）に対応するスロットを検索または割り当て、統計値（数値および値配列）を更新する。

シグネチャまたは形状

static void set_stats_slot(Datum *values, bool *nulls, bool *replaces,
                           int16 stakind, Oid staop, Oid stacoll,
                           Datum stanumbers, bool stanumbers_isnull,
                           Datum stavalues, bool stavalues_isnull)

入力と出力

  • **入力**:
  • values, nulls, replaces: pg_statistic 行の属性値を保持する配列（更新対象）。
  • stakind, staop, stacoll: 設定する統計種別、演算子、照合順序。
  • stanumbers, stavalues: 統計データ本体。
  • stanumbers_isnull, stavalues_isnull: データが NULL かどうかを示すフラグ。
  • **出力**: なし（values, nulls, replaces 配列を直接書き換える）。

前提条件と不変条件

  • values 配列は pg_statistic の行構造に対応している必要がある。
  • STATISTIC_NUM_SLOTS（統計スロットの最大数）を超えてスロットを割り当てることはできない。

アルゴリズムまたは振る舞い

1. **スロット検索**: STATISTIC_NUM_SLOTS の範囲内で、指定された stakind と一致する既存スロットを走査する。同時に、空きスロット（stakind が 0）があれば記録しておく。 2. **スロット決定**: 一致するスロットが見つかればそれを使用し、見つからなければ記録しておいた空きスロットを使用する。 3. **エラーチェック**: 既存スロットも空きスロットも存在しない場合、ERROR を発生させる。 4. **データ更新**: 決定したスロットの stakind, staop, stacoll を更新し、replaces フラグを立てる。 5. **値の適用**: stanumbers および stavalues が NULL でない場合、対応する配列位置に値を格納し、nulls を false に、replaces を true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/attribute_stats.c 内の attribute_statistics_update 関数から複数回呼び出される。
  • **呼び出し先**: ereport（エラー発生時）。

関連データ構造

  • pg_statistic システムカタログの行構造（Anum_pg_statistic_stakind1 等の定数で参照）。
  • Datum 型配列。

失敗モードと境界条件

  • **最大スロット超過**: STATISTIC_NUM_SLOTS を超える統計情報を書き込もうとした場合、ereport(ERROR, ...) により処理が中断される。

保守担当者向けメモ

  • 本関数は pg_statistic のスロット管理を抽象化するヘルパーである。スロットのインデックス計算には Anum_pg_statistic_stakind1 を基準としたオフセット計算が使用されているため、カタログ定義の変更時には注意が必要。
