pg_index_column_has_property

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/amutils.c:432

目的

指定されたインデックスの特定の列に対して、要求されたプロパティ（例: orderable 等）が有効かどうかを SQL レベルで照会します。

シグネチャまたは形状

Datum pg_index_column_has_property(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): 対象インデックスの OID (relid)。
  • PG_GETARG_INT32(1): 対象列の番号 (attno)。
  • PG_GETARG_TEXT_PP(2): 照会するプロパティ名 (propname)。
  • **出力**:
  • Datum: プロパティの有無を示す真偽値、または attno <= 0 の場合は NULL。

前提条件と不変条件

  • attno が 0 以下の場合、インデックス列のプロパティとしては無効とみなされ、即座に NULL を返します。

アルゴリズムまたは振る舞い

1. 入力された propname を C 文字列に変換します。 2. attno が 0 以下であれば NULL を返します。 3. それ以外の場合、内部関数 indexam_property を呼び出し、インデックスの OID と列番号を渡してプロパティ判定を委譲します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されます（具体的な SQL クエリ等の呼び出し元情報はソース内に記載なし）。
  • **呼び出し先**: indexam_property (内部関数)。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約で使用される引数コンテナ。
  • IndexAmRoutine: indexam_property 経由で間接的に参照される、アクセスメソッドの機能定義構造体。

失敗モードと境界条件

  • attno <= 0: 境界条件として定義されており、エラーではなく NULL を返します。
  • propname が不正な場合や、対象のインデックスが存在しない場合等の失敗モードは、呼び出し先の indexam_property の実装に依存します。

保守担当者向けメモ

  • 本関数は pg_index_column_has_property SQL 関数のエントリポイントです。
  • プロパティ判定ロジックの変更は indexam_property を修正してください。
  • attno の検証ロジックは pg_index カタログの列番号定義と整合している必要があります。
