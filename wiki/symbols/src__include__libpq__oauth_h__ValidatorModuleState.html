ValidatorModuleState

定義

  • 種別: struct
  • 定義箇所: src/include/libpq/oauth.h:21

目的

OAuthバリデータモジュールがセッション間やリクエスト間で状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct ValidatorModuleState
{
    int         sversion;      /* PG_VERSION_NUM */
    void       *private_data;  /* モジュール固有データへのポインタ */
} ValidatorModuleState;

入力と出力

  • **入力**: 構造体生成時に sversion が設定され、モジュールが必要に応じて private_data を割り当てます。
  • **出力**: 各コールバック関数（ValidatorStartupCB, ValidatorShutdownCB, ValidatorValidateCB）へ引数として渡されます。

前提条件と不変条件

  • sversion はサーバーの PG_VERSION_NUM を保持します。
  • private_data はモジュールが自由に利用可能な領域であり、各コールバック間で状態を共有するために使用されます。

アルゴリズムまたは振る舞い

1. **初期化**: auth-oauth.c 内で palloc0 を用いてメモリが確保されます。 2. **保持**: 認証フロー中、この構造体へのポインタが各バリデータコールバックに渡されます。 3. **利用**: モジュールは private_data を通じて、検証に必要なセッション固有の情報を保持・参照します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-oauth.c がこの構造体をインスタンス化し、管理します。
  • **呼び出し先**: ValidatorStartupCB, ValidatorShutdownCB, ValidatorValidateCB を実装する各バリデータモジュール（src/test/modules/oauth_validator/ 配下の各実装など）がこの構造体を参照します。

関連データ構造

  • OAuthValidatorCallbacks: この構造体を受け取るコールバック関数の定義群。

失敗モードと境界条件

  • private_data のメモリ管理はモジュール側の責務です。不適切なポインタ操作はセグメンテーション違反を引き起こす可能性があります。

保守担当者向けメモ

  • sversion は将来の拡張性のために予約されています。
  • private_data はモジュールが動的にロードされる際、状態を維持するための唯一の手段であるため、メモリリークが発生しないよう ValidatorShutdownCB で適切に解放する必要があります。
