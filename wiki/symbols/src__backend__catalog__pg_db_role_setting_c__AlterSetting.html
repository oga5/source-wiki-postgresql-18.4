AlterSetting

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_db_role_setting.c:23

目的

ALTER ROLE ... SET や ALTER DATABASE ... SET などの SQL コマンドに応じ、システムカタログ pg_db_role_setting に格納されたデータベースまたはロール単位の GUC パラメータ設定を永続化（追加・更新・削除）します。

シグネチャまたは形状

void AlterSetting(Oid databaseid, Oid roleid, VariableSetStmt *setstmt)

入力と出力

  • **入力**:
  • databaseid: 対象データベースの OID。
  • roleid: 対象ロールの OID。
  • setstmt: 実行する設定変更の内容を含む VariableSetStmt 構造体。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • pg_db_role_setting カタログに対する RowExclusiveLock を保持して実行されます。
  • 設定値は setconfig 列に text 型の配列として格納されます。
  • 変更の結果、設定配列が空になった場合は、カタログから該当タプルが削除されます。

アルゴリズムまたは振る舞い

1. ExtractSetVariableArgs で設定値を抽出します。 2. DbRoleSettingDatidRolidIndexId インデックスを使用して、指定された (databaseid, roleid) に対応する既存タプルを検索します。 3. setstmt->kind に基づき以下の処理を行います：

  • VAR_RESET_ALL: 既存の設定をすべて削除します。タプルが残る場合は更新し、空になる場合は削除します。
  • その他: GUCArrayAdd または GUCArrayDelete を用いて設定配列を操作します。タプルが存在すれば更新、存在しなければ新規挿入を行います。

    • 4. InvokeObjectPostAlterHookArg を呼び出し、変更を通知します。 5. systable_endscan でスキャンを終了し、table_close でリレーションを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/dbcommands.c: ALTER DATABASE ... SET 実行時。
  • src/backend/commands/user.c: ALTER ROLE ... SET 実行時。
  • **呼び出し先**:
  • GUCArrayAdd, GUCArrayDelete, GUCArrayReset (GUC 配列操作)
  • CatalogTupleUpdate, CatalogTupleInsert, CatalogTupleDelete (カタログ更新)
  • InvokeObjectPostAlterHookArg (フック通知)

関連データ構造

  • pg_db_role_setting: 操作対象のシステムカタログ。
  • VariableSetStmt: SQL 文の解析結果を保持する構造体。
  • HeapTuple: カタログの行データ。

失敗モードと境界条件

  • 指定されたキーに対するタプルが存在しない場合、RESET 以外のコマンドであれば新規挿入が行われます。
  • setconfig 列は TOAST テーブルが自動的に使用されるため、非常に大きな設定値も格納可能です。

保守担当者向けメモ

  • 本関数はカタログの「操作」のみを担当します。設定の適用ロジックは ApplySetting および utils/misc/guc.c を参照してください。
  • ロックは table_close 後もトランザクション終了まで保持される設計となっています。
