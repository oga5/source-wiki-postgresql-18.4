PGresAttValue

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/libpq-int.h:132

目的

クエリ結果セットにおける個々の属性値（カラム値）を保持するための内部データ構造。

シグネチャまたは形状

typedef struct pgresAttValue
{
    int         len;            /* 値のバイト長 */
    char       *value;          /* 実際の値（終端ヌル文字を含む） */
} PGresAttValue;

入力と出力

  • **入力**: データベースから受信したバイナリまたはテキスト形式のデータ。
  • **出力**: PGresult 構造体の一部として、クライアントアプリケーションが参照可能な形式で保持される。

前提条件と不変条件

  • value フィールドは、len で指定されたバイト数に加え、終端のヌル文字（\0）を保持する領域を確保している必要がある。
  • len が負の値の場合、その属性値は SQL の NULL を表すものとして扱われる（libpq の慣習に基づく）。

アルゴリズムまたは振る舞い

  • fe-exec.c 内の pqResultAlloc を介してメモリが割り当てられる。
  • クエリ結果の行（タプル）を構築する際、属性の数分だけ本構造体の配列が確保され、各カラムの値が格納される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-exec.c 内の pqAddTuple 関数などが、結果セットの構築過程で本構造体を操作する。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行わない。

関連データ構造

  • PGresult: 本構造体を配列として保持し、クエリ結果全体を管理する。

失敗モードと境界条件

  • メモリ不足時: pqResultAlloc による割り当て失敗が発生する可能性がある。
  • len の不整合: 実際の value の長さと len が一致しない場合、後続のデータ処理でバッファオーバーランや不正なメモリ参照を引き起こすリスクがある。

保守担当者向けメモ

  • 本構造体は libpq の内部実装であり、公開 API ではない。
  • value は malloc 等で確保されたメモリを指すため、PGresult の破棄時に適切に解放される必要がある。メモリ管理の詳細は fe-exec.c を参照すること。
