BitmapHeapScanNextBlock

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_handler.c:2459

目的

ビットマップスキャンにおいて、次に処理すべきヒープブロックを読み込み、そのブロック内の可視タプルを特定してスキャン状態を更新する。

シグネチャまたは形状

static bool BitmapHeapScanNextBlock(TableScanDesc scan,
                                    bool *recheck,
                                    uint64 *lossy_pages,
                                    uint64 *exact_pages)

入力と出力

  • **入力**:
  • scan: テーブルスキャン記述子。SO_TYPE_BITMAPSCAN フラグが必要。
  • recheck: ビットマップの再チェックが必要かを示すフラグの出力先。
  • lossy_pages, exact_pages: 処理したページ数のカウンタ（ポインタ）。
  • **出力**:
  • 戻り値: ブロックが正常に読み込まれた場合は true、ビットマップが枯渇した場合は false。

前提条件と不変条件

  • scan は BitmapHeapScanDesc としてキャスト可能であること。
  • hscan->rs_read_stream が初期化されていること。
  • 処理対象のブロック番号が有効であること。

アルゴリズムまたは振る舞い

1. 現在のバッファを解放し、read_stream_next_buffer を用いて次のブロックを読み込む。 2. ページに対して heap_page_prune_opt を実行し、断片化を解消する。 3. バッファを共有ロック（BUFFER_LOCK_SHARE）し、ビットマップの性質に応じて以下の処理を行う。

  • **非損失（Exact）**: tbm_extract_page_tuple で抽出されたオフセットに基づき、heap_hot_search_buffer を用いてHOTチェーンを辿り、可視タプルを特定する。
  • **損失（Lossy）**: ページ内の全タプルを走査し、HeapTupleSatisfiesVisibility で可視性を判定する。

    • 4. 特定された可視タプルのオフセットを hscan->rs_vistuples に格納し、カウンタを更新する。 5. 最後にロックを解除し、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_handler.c 内のビットマップスキャン処理（行2144付近）。
  • **呼び出し先**: read_stream_next_buffer, heap_page_prune_opt, LockBuffer, tbm_extract_page_tuple, heap_hot_search_buffer, HeapTupleSatisfiesVisibility, PredicateLockTID, HeapCheckForSerializableConflictOut 等。

関連データ構造

  • TableScanDesc / BitmapHeapScanDesc / HeapScanDesc: スキャン状態を保持する構造体。
  • TBMIterateResult: ビットマップの反復結果。
  • Buffer: ヒープページを保持するバッファ。

失敗モードと境界条件

  • ビットマップが枯渇した場合、read_stream_next_buffer が無効なバッファを返し、関数は false を返す。
  • ページ内に可視タプルが存在しない場合、rs_ntuples が 0 となり、呼び出し元で次のブロックへ進む制御が行われる。

保守担当者向けメモ

  • 本関数は heapam_handler.c 内のブリッジ関数であり、実際のタプル可視性判定やHOTチェーンの走査ロジックは heapam.c および heapam_visibility.c に依存している。
  • 損失ビットマップと非損失ビットマップで走査戦略が異なるため、修正時は両方のパスに注意すること。
