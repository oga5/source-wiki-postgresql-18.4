audit_failure

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_oat_hooks/test_oat_hooks.c:269

目的

PostgreSQLのオブジェクトアクセスフックにおいて、操作が拒否された際の監査メッセージを生成・出力します。

シグネチャまたは形状

static void audit_failure(const char *hook, char *action, char *objName);

入力と出力

  • **入力**:
  • hook: 発生元のフック名を示す文字列。
  • action: 拒否された操作内容を示す文字列。
  • objName: 操作対象のオブジェクト名を示す文字列。
  • **出力**: なし（void）。

前提条件と不変条件

  • 本関数は emit_audit_message を呼び出すため、同関数が並列ワーカープロセスにおけるログ重複を適切に制御しているという前提に依存します。

アルゴリズムまたは振る舞い

1. 引数として受け取った hook、action、objName を、ステータス "denied" と共に emit_audit_message へ渡します。 2. emit_audit_message を通じて、拒否された操作に関する監査ログを NOTICE レベルで出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_oat_hooks/test_oat_hooks.c 内の権限チェックロジック（例: REGRESS_exec_check_perms 等のフック関数内）。
  • **呼び出し先**:
  • emit_audit_message

関連データ構造

  • なし。

失敗モードと境界条件

  • 本関数自体はエラーを発生させませんが、呼び出し先である emit_audit_message の内部実装（NOTICE 出力や並列処理の判定）に依存します。

保守担当者向けメモ

  • 本関数はテストモジュール専用です。本番環境の監査ログ実装の参考にする場合は、ログ出力のパフォーマンスや、トランザクションのロールバックとの整合性に注意してください。
