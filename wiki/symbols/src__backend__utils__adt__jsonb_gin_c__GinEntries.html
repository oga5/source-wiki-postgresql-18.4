GinEntries

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonb_gin.c:80

目的

JSONBのGINインデックス構築およびクエリ解析プロセスにおいて、抽出されたキー（Datum）のリストを動的に管理するためのコンテナ構造体です。

シグネチャまたは形状

typedef struct GinEntries
{
    Datum      *buf;       /* キーを格納する配列 */
    int         count;     /* 現在格納されているキーの数 */
    int         allocated; /* bufに割り当てられている容量 */
} GinEntries;

入力と出力

  • **入力**: init_gin_entries を介して初期化され、add_gin_entry を通じて Datum 型のキーが追加されます。
  • **出力**: インデックス構築やクエリ処理の過程で、蓄積されたキーの配列が後続の処理（インデックスへの書き込みや論理ツリーの照合）に利用されます。

前提条件と不変条件

  • buf は Datum 型の配列を指し、count は現在格納されている要素数を、allocated は確保済みの領域サイズを保持します。
  • add_gin_entry を呼び出す前に init_gin_entries による初期化が必要です。

アルゴリズムまたは振る舞い

  • インデックス構築時、gin_extract_jsonb 等がJSONBドキュメントを走査し、抽出された各キーを add_gin_entry を用いて本構造体に順次追加します。
  • クエリ解析時、emit_jsp_gin_entries が JsonPathGinNode ツリーを走査し、検索条件となるキーを本構造体に収集します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • init_gin_entries (初期化)
  • add_gin_entry (要素追加)
  • emit_jsp_gin_entries (クエリ解析時の収集)
  • gin_extract_jsonb 等のインデックス抽出関数
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • Datum: GINインデックスに格納されるキーの基本単位。
  • JsonPathGinNode: クエリ解析時に本構造体へキーを供給する論理ツリー構造。

失敗モードと境界条件

  • メモリ割り当て失敗: add_gin_entry 内での動的なメモリ拡張時に発生する可能性があります。
  • 境界条件: count が allocated に達した際の再割り当て処理が適切に行われない場合、バッファオーバーフローのリスクがあります。

保守担当者向けメモ

  • 本構造体は jsonb_gin.c 内で局所的に使用されるユーティリティです。
  • count と allocated の整合性は add_gin_entry の実装に依存しているため、変更を加える際はメモリ管理ロジックを十分に確認してください。
