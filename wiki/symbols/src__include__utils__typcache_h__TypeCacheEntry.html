TypeCacheEntry

定義

  • 種別: struct
  • 定義箇所: src/include/utils/typcache.h:31

目的

データ型の属性、演算子、関数、および複合型やドメイン型の制約情報をキャッシュし、pg_type カタログへの頻繁なアクセスを回避してパフォーマンスを向上させるための構造体です。

入力と出力

  • **入力**: lookup_type_cache(type_id, flags) を介して要求された型情報。
  • **出力**: キャッシュされた型属性、演算子情報、関数呼び出し情報（FmgrInfo）、および複合型の場合はタプル記述子（TupleDesc）。

前提条件と不変条件

  • type_id はハッシュテーブルの検索キーであり、構造体の先頭に配置される必要がある。
  • キャッシュされた FmgrInfo はメモリリークを防ぐため、バックエンドの生存期間中適切に管理される。
  • tupDesc_identifier は、複合型の構造変更を検知するためにプロセス生存期間中一意である必要がある。

アルゴリズムまたは振る舞い

  • **遅延初期化**: 要求された情報（TYPECACHE_* フラグで指定）のみが計算され、キャッシュされる。
  • **無効化**: pg_type や pg_opclass の変更が検出された場合、キャッシュは無効化または再計算される。
  • **複合型管理**: 複合型の場合、tupDesc を保持し、構造変更を tupDesc_identifier で追跡する。

呼び出し元と呼び出し先

  • **呼び出し元**: postgres_fdw（deparse.c）、brin インデックス（brin_tuple.c）、gin インデックス（ginutil.c）、explain コマンド、executor など。
  • **呼び出し先**: lookup_type_cache 関数が本構造体の生成・検索を制御する。

関連データ構造

  • DomainConstraintCache: ドメイン制約を保持。
  • TupleDesc: 複合型の構造定義。
  • FmgrInfo: 関数呼び出し情報。
  • TypeCacheEnumData: 列挙型（Enum）の内部情報。

失敗モードと境界条件

  • 情報が未要求または存在しない場合、Oid フィールドは InvalidOid、ポインタフィールドは NULL、数値フィールドは 0 に初期化される。
  • 複合型でない場合、tupDesc は NULL となる。

保守担当者向けメモ

  • 本キャッシュはバックエンドプロセス単位で永続化される。
  • 演算子クラスや型定義の変更時には、typcache.c 内の無効化ロジックが正しく機能しているか確認すること。
