CopyToTextLikeStart

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyto.c:189

目的

COPY TO コマンドにおけるテキスト形式および CSV 形式の出力処理を開始する。主に、出力エンコーディングの準備と、要求された場合のヘッダー行（列名）の出力を行う。

シグネチャまたは形状

static void CopyToTextLikeStart(CopyToState cstate, TupleDesc tupDesc)

入力と出力

  • **入力**:
  • cstate: COPY TO 操作の現在の状態を保持する CopyToStateData 構造体。
  • tupDesc: 出力対象のタプル記述子。
  • **出力**: なし（副作用として出力バッファへデータを書き込む）。

前提条件と不変条件

  • cstate が有効であり、COPY 操作の初期化が完了していること。
  • cstate->need_transcoding が真の場合、null_print 文字列がサーバーエンコーディングからファイルエンコーディングへ変換される。

アルゴリズムまたは振る舞い

1. **エンコーディング変換**: need_transcoding が有効な場合、null_print を file_encoding に変換し、cstate->opts.null_print_client に格納する。 2. **ヘッダー出力**: cstate->opts.header_line が設定されている場合、attnumlist に基づいて列名を順次出力する。

  • 各列名の間には delim（区切り文字）を挿入する。
  • csv_mode が有効な場合は CopyAttributeOutCSV を、それ以外の場合は CopyAttributeOutText を使用して列名を出力する。

    • 3. **行終了**: ヘッダー出力の最後に CopySendTextLikeEndOfRow を呼び出し、行を終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyto.c 内の CopyToRoutine 構造体（CopyToStart フィールド）から参照される。
  • **呼び出し先**:
  • pg_server_to_any: エンコーディング変換に使用。
  • CopySendChar, CopyAttributeOutCSV, CopyAttributeOutText, CopySendTextLikeEndOfRow: 出力処理に使用。

関連データ構造

  • CopyToStateData: COPY 操作の状態管理。
  • TupleDesc: テーブルの属性定義情報。

失敗モードと境界条件

  • attnumlist が空の場合、ヘッダー行は出力されない。
  • エンコーディング変換に失敗した場合、基盤となる pg_server_to_any の挙動に依存する。

保守担当者向けメモ

  • 本関数はテキストおよび CSV 形式の共通初期化ロジックである。バイナリ形式には適用されない。
  • ヘッダー行の出力ロジックは、attnumlist に含まれる属性のみを対象とするため、COPY 文で指定された列順序が正しく反映される。
