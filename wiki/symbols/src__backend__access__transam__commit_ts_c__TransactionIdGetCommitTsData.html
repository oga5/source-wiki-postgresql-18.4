TransactionIdGetCommitTsData

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/commit_ts.c:273

目的

指定されたトランザクションID（XID）に対応するコミット時刻およびレプリケーション元ノードIDを取得する。

シグネチャまたは形状

bool TransactionIdGetCommitTsData(TransactionId xid, TimestampTz *ts, RepOriginId *nodeid);

入力と出力

  • **入力**:
  • xid: 検索対象のトランザクションID。
  • ts: コミット時刻を格納するポインタ（出力用）。
  • nodeid: レプリケーション元ノードIDを格納するポインタ（出力用、NULL可）。
  • **出力**:
  • 戻り値: 該当するコミット時刻が正常に取得でき、かつ ts が 0 でない場合に true を返す。それ以外は false。

前提条件と不変条件

  • track_commit_timestamp が有効であること。無効な場合はエラーが発生する。
  • xid は有効な正規のトランザクションIDである必要がある（無効なIDはエラー、凍結済み/ブートストラップIDは ts=0 を返し false となる）。
  • CommitTsLock および SLRU 関連のロックが内部で適切に管理される。

アルゴリズムまたは振る舞い

1. **バリデーション**: xid が有効か確認し、特殊なID（凍結済み等）であれば即座に false を返す。 2. **キャッシュ確認**: CommitTsLock を取得し、commitTsShared->xidLastCommit が要求された xid と一致すれば、キャッシュから値を返して終了する。 3. **範囲チェック**: TransamVariables から取得した有効範囲（oldestCommitTsXid ～ newestCommitTsXid）外の xid であれば false を返す。 4. **SLRU 読み込み**: SimpleLruReadPage_ReadOnly を呼び出し、該当するSLRUページから CommitTimestampEntry を読み込む。 5. **データ抽出**: 読み込んだエントリから時刻とノードIDを抽出し、ロックを解放して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/commit_ts.c (内部の検索処理)
  • src/backend/replication/logical/conflict.c
  • **呼び出し先**:
  • SimpleLruReadPage_ReadOnly
  • LWLockAcquire / LWLockRelease
  • SimpleLruGetBankLock

関連データ構造

  • CommitTimestampEntry: SLRUページ内に格納されるデータ構造。
  • CommitTsCtl: SLRU管理用の制御構造体。
  • CommitTimestampShared: 共有メモリ上の状態管理構造体。

失敗モードと境界条件

  • xid が無効な場合: ERROR をスローする。
  • commitTsActive が無効な場合: error_commit_ts_disabled() を呼び出しエラーとなる。
  • 検索対象の xid が保持している有効範囲外の場合: ts に 0 を設定し false を返す。

保守担当者向けメモ

  • 本関数は CommitTsLock を使用して共有メモリ上のキャッシュを保護し、SLRU 読み込み時には SimpleLruGetBankLock を使用してページ単位の排他制御を行う。
  • 頻繁に呼び出されるため、キャッシュヒット率が性能に大きく影響する。
