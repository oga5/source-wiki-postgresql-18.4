QTNode

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/ts_utils.h:232

目的

TSQuery を木構造（ツリー形式）に展開し、クエリの正規化、最適化、および比較操作を容易にするためのノード表現です。

シグネチャまたは形状

typedef struct QTNode
{
    QueryItem  *valnode;    /* クエリ項目へのポインタ */
    uint32      flags;      /* ノードの状態フラグ */
    int32       nchild;     /* 子ノードの数 */
    char       *word;       /* 関連する単語文字列 */
    uint32      sign;       /* クエリシグネチャ */
    struct QTNode **child;  /* 子ノードの配列 */
} QTNode;

入力と出力

  • **入力**: QT2QTN 関数等を通じて TSQuery 形式のデータから生成されます。
  • **出力**: QTN2QT 関数等を通じて、処理後のツリー構造を再び TSQuery 形式へ変換するために使用されます。

前提条件と不変条件

  • child 配列は nchild の数だけ要素を保持する必要があります。
  • valnode は有効な QueryItem を指している必要があります。

アルゴリズムまたは振る舞い

  • TSQuery を木構造に展開（QT2QTN）し、再帰的な走査を可能にします。
  • クエリの最適化（NOTの除去やストップワード処理）や正規化（QTNSort）を行う際のデータ構造として機能します。
  • 最終的に QTN2QT を用いて、操作済みのツリーを元の TSQuery 形式に再構築します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsquery_op.c および src/backend/utils/adt/tsquery_rewrite.c 内の関数群が、クエリ操作のために本構造体を生成・利用しています。
  • **呼び出し先**: 具体的な関数呼び出しの連鎖については、提供されたソース範囲内では明示されていません。

関連データ構造

  • QueryItem: valnode が参照するクエリの基本要素。
  • TSQuery: 本構造体が展開・再構築の対象とする元のデータ型。

失敗モードと境界条件

  • メモリ割り当て失敗: palloc0 等によるメモリ確保が失敗した場合、システムはエラーを発生させます。
  • nchild が 0 の場合、child ポインタは空である必要があります。

保守担当者向けメモ

  • QTNode は TSQuery の操作において中間表現として機能します。クエリの正規化ロジックを変更する際は、QT2QTN および QTN2QT の変換整合性に注意してください。
  • child 配列のメモリ管理は呼び出し側で行われるため、ノードの解放時には再帰的なメモリ解放が必要です。
