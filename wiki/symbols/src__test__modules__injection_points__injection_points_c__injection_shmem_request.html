injection_shmem_request

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_points.c:138

目的

PostgreSQLの共有メモリ初期化プロセスにおいて、injection_points モジュールが必要とする共有メモリ領域を確保するために使用されます。

シグネチャまたは形状

static void injection_shmem_request(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリの要求を登録）

前提条件と不変条件

  • 本関数は、PostgreSQLの共有メモリ要求フェーズで呼び出されることを前提としています。
  • prev_shmem_request_hook が設定されている場合、連鎖的に呼び出される必要があります。

アルゴリズムまたは振る舞い

1. prev_shmem_request_hook が存在する場合、それを実行して以前のフックを維持します。 2. InjectionPointSharedState 構造体のサイズを計算し、MAXALIGN でアライメントを調整します。 3. RequestAddinShmemSpace を呼び出し、計算したサイズの共有メモリ領域をシステムに要求します。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLの共有メモリ要求処理（shmem_request_hook を介して実行）。
  • 呼び出し先: prev_shmem_request_hook（存在する場合）、RequestAddinShmemSpace。

関連データ構造

  • InjectionPointSharedState: 本関数が確保を要求する共有メモリ上の状態管理構造体。

失敗モードと境界条件

  • RequestAddinShmemSpace が失敗した場合、PostgreSQLの起動プロセスが停止する可能性があります。
  • 共有メモリの要求は、shmem_request_hook が設定されるタイミング（モジュールロード時）に依存します。

保守担当者向けメモ

  • 本関数は shmem_request_hook に代入されることで機能します。フックの連鎖を維持するため、prev_shmem_request_hook の呼び出しを忘れないようにしてください。
  • 確保する構造体 InjectionPointSharedState のサイズが変更された場合、本関数内の計算ロジックも適切であるか確認が必要です。
