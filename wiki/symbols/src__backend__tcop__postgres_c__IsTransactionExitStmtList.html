IsTransactionExitStmtList

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/postgres.c:2874

目的

与えられた文リストが、トランザクションを終了させる単一のユーティリティコマンドであるかどうかを判定する。

シグネチャまたは形状

static bool IsTransactionExitStmtList(List *pstmts)

入力と出力

  • **入力**: pstmts (実行計画文のリスト List *)
  • **出力**: bool 型。リストが単一のトランザクション終了コマンドであれば true、それ以外は false。

前提条件と不変条件

  • pstmts は PlannedStmt ノードのリストであることを前提とする。
  • リストの長さが 1 でない場合、常に false を返す。

アルゴリズムまたは振る舞い

1. リスト pstmts の要素数が 1 であるかを確認する。 2. 要素数が 1 の場合、その要素を PlannedStmt として取得する。 3. commandType が CMD_UTILITY であり、かつ utilityStmt が IsTransactionExitStmt によってトランザクション終了コマンドであると判定された場合に true を返す。 4. 上記条件を満たさない場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/postgres.c:2249 (ポータルのステートメントリストの検証に使用)
  • **呼び出し先**:
  • list_length (リストの長さ判定)
  • linitial_node (リストの先頭要素の取得)
  • IsTransactionExitStmt (ユーティリティ文の種別判定)

関連データ構造

  • List: PostgreSQL の汎用リスト構造。
  • PlannedStmt: 実行計画を表すノード構造体。

失敗モードと境界条件

  • pstmts が空リストの場合、list_length は 0 となり、関数は false を返す。
  • pstmts 内の要素が PlannedStmt ではない場合、linitial_node マクロにより型チェックエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は postgres.c 内の静的関数であり、トランザクション制御のロジックにおいて特定のユーティリティコマンド（COMMIT, ROLLBACK 等）を識別するために使用される。
  • IsTransactionExitStmt の定義が変更された場合、本関数の判定ロジックも影響を受ける。
