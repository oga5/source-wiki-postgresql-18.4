findObjectByCatalogId

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/common.c:777

目的

CatalogId（OIDとテーブルOIDのペア）をキーとして、管理されている DumpableObject をハッシュテーブルから検索し取得する。

シグネチャまたは形状

DumpableObject *findObjectByCatalogId(CatalogId catalogId)

入力と出力

  • **入力**: CatalogId catalogId（検索対象のカタログ識別子）
  • **出力**: 該当する DumpableObject へのポインタ。見つからない場合やハッシュテーブルが未初期化の場合は NULL を返す。

前提条件と不変条件

  • catalogIdHash が初期化されている必要がある。
  • 検索対象のオブジェクトは、事前に AssignDumpId 等を通じて catalogIdHash に登録されている必要がある。

アルゴリズムまたは振る舞い

1. グローバル変数 catalogIdHash が NULL かどうかを確認し、NULL であれば即座に NULL を返す。 2. catalogid_lookup 関数を呼び出し、catalogIdHash から指定された catalogId に対応する CatalogIdMapEntry を検索する。 3. エントリが見つかった場合、そのエントリが保持する dobj（DumpableObject へのポインタ）を返す。 4. エントリが見つからない場合は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/common.c 内の複数の箇所から、オブジェクトの依存関係解決や参照のために呼び出されている。
  • **呼び出し先**: catalogid_lookup（ハッシュテーブル検索用関数）。

関連データ構造

  • CatalogId: 検索キーとなる構造体。
  • DumpableObject: 戻り値として返される、ダンプ対象オブジェクトの抽象構造体。
  • CatalogIdMapEntry: ハッシュテーブル内で CatalogId と DumpableObject を紐付ける要素。
  • catalogIdHash: オブジェクト検索用のハッシュテーブル（simplehash.h 実装）。

失敗モードと境界条件

  • catalogIdHash が未初期化（NULL）の場合、安全に NULL を返す。
  • 指定した CatalogId がハッシュテーブルに存在しない場合、NULL を返す。

保守担当者向けメモ

  • 本関数は pg_dump のオブジェクト管理インフラの核心部分である。
  • catalogIdHash のライフサイクルは getSchemaData の実行期間に依存するため、呼び出しタイミングに注意すること。
  • simplehash.h を利用しているため、ハッシュテーブルの再構築やメモリ管理の変更時には catalogid_lookup の挙動と整合性を取ること。
