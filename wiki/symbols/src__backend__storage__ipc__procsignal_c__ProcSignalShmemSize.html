ProcSignalShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/procsignal.c:117

目的

プロセス間シグナル通信（IPC）機能のために共有メモリ上で必要となる合計サイズを算出する。

シグネチャまたは形状

Size ProcSignalShmemSize(void)

入力と出力

  • **入力**: なし
  • **出力**: Size 型の共有メモリサイズ（バイト単位）

前提条件と不変条件

  • NumProcSignalSlots が正しく初期化されている必要がある。
  • 共有メモリの計算には mul_size および add_size を使用し、オーバーフローを防止している。

アルゴリズムまたは振る舞い

1. NumProcSignalSlots（全プロセススロット数）と ProcSignalSlot 構造体のサイズの積を計算する。 2. ProcSignalHeader 構造体のヘッダー部分（psh_slot メンバまでのオフセット）を加算する。 3. 計算結果を合計サイズとして返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/ipci.c:135（共有メモリ全体のサイズ計算時）
  • src/backend/storage/ipc/procsignal.c:134（ProcSignalShmemInit 内での検証用）
  • **呼び出し先**:
  • mul_size（ユーティリティ関数）
  • add_size（ユーティリティ関数）

関連データ構造

  • ProcSignalSlot: 各プロセスのシグナル状態を保持する構造体。
  • ProcSignalHeader: 共有メモリ上の管理ヘッダー。

失敗モードと境界条件

  • NumProcSignalSlots が 0 の場合、ヘッダーサイズのみが返される。
  • メモリサイズ計算時にオーバーフローが発生した場合、mul_size / add_size の内部実装によりエラーが報告される可能性がある。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウト構造に直接依存している。ProcSignalHeader や ProcSignalSlot の構造を変更した場合は、必ず本関数の計算ロジックを更新すること。
