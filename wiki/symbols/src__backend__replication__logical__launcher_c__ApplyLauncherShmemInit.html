ApplyLauncherShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/launcher.c:963

目的

論理レプリケーションランチャーが使用する共有メモリ領域（LogicalRepCtx）の確保および初期化を行う。

シグネチャまたは形状

void ApplyLauncherShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上の LogicalRepCtx を初期化する）

前提条件と不変条件

  • 本関数は、PostgreSQLの共有メモリ初期化フェーズで呼び出される必要がある。
  • max_logical_replication_workers が正しく設定されていること。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を呼び出し、"Logical Replication Launcher Data" という名称で共有メモリ領域を確保する。 2. 領域が新規作成された場合（found が false の場合）:

  • LogicalRepCtx 全体をゼロクリアする。
  • last_start_dsa および last_start_dsh を無効値（INVALID）に設定する。
  • max_logical_replication_workers の数だけループし、各ワーカーのスロットをゼロクリアした上で、個別のスピンロック（relmutex）を初期化する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:335（PostgreSQLの共有メモリ初期化プロセス）
  • 呼び出し先: ShmemInitStruct, memset, SpinLockInit, ApplyLauncherShmemSize

関連データ構造

  • LogicalRepCtx: 共有メモリ上のメインコンテキスト構造体。
  • LogicalRepWorker: 各ワーカーの状態を保持する構造体。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、ShmemInitStruct の内部実装によりプロセスが異常終了する可能性がある。
  • max_logical_replication_workers が 0 の場合、ループは実行されず、共有メモリ領域のみが確保される。

保守担当者向けメモ

  • 本関数は共有メモリの初期化時に一度だけ実行されることを想定している。
  • LogicalRepWorker のスピンロック初期化は、ワーカーのライフサイクル管理において重要であるため、構造体定義に変更を加える際は注意すること。
