gist_box_penalty

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistproc.c:198

目的

GiSTインデックスにおいて、新しいエントリを既存のノードに追加する際の「ペナルティ（コスト）」を計算します。この値は、挿入によって最小境界矩形（MBR）がどれだけ拡大するかを評価するために使用されます。

シグネチャまたは形状

Datum gist_box_penalty(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY *origentry（既存のインデックスエントリ）
  • PG_GETARG_POINTER(1): GISTENTRY *newentry（挿入対象の新しいエントリ）
  • PG_GETARG_POINTER(2): float *result（計算結果を格納するポインタ）
  • **出力**:
  • PG_RETURN_POINTER(result): 計算されたペナルティ値（float）へのポインタを返します。

前提条件と不変条件

  • origentry および newentry のキーは BOX 型として解釈可能である必要があります。
  • result は呼び出し元によってメモリが確保されている必要があります。

アルゴリズムまたは振る舞い

1. origentry および newentry からそれぞれの BOX 構造体を取得します。 2. 内部関数 box_penalty(origbox, newbox) を呼び出し、面積の増加量に基づいたペナルティ値を算出します。 3. 算出された値を result ポインタが指す領域に書き込み、そのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: GiSTインデックスの挿入処理（GiSTのインフラ層から呼び出されます）。
  • **呼び出し先**: box_penalty（同ファイル内のヘルパー関数）。

関連データ構造

  • GISTENTRY: GiSTインデックスのエントリを保持する構造体。
  • BOX: 2次元幾何データ型（矩形）。

失敗モードと境界条件

  • 入力された Datum が有効な BOX 型でない場合、DatumGetBoxP マクロの内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は box_penalty への単なるラッパーです。ロジックの変更が必要な場合は box_penalty を確認してください。
  • GiSTのペナルティ計算は、インデックスのバランスを維持するための重要な指標となるため、変更時は面積増加の計算ロジックに注意してください。
