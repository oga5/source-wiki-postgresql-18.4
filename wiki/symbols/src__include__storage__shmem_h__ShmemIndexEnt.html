ShmemIndexEnt

定義

  • 種別: struct
  • 定義箇所: src/include/storage/shmem.h:54

目的

共有メモリ上に割り当てられたデータ構造を管理・識別するためのエントリ構造体です。共有メモリ内の名前付きオブジェクトのレジストリ（ShmemIndex）を構成する要素として使用されます。

シグネチャまたは形状

typedef struct {
    char        key[SHMEM_INDEX_KEYSIZE]; /* 構造体の名前 */
    void       *location;                 /* 共有メモリ上の開始アドレス */
    Size        size;                     /* 要求されたサイズ（バイト） */
    Size        allocated_size;           /* 実際に割り当てられたサイズ（バイト） */
} ShmemIndexEnt;

入力と出力

本構造体はデータ保持用であり、関数ではありません。ShmemInitStruct 等の管理関数を通じて、共有メモリ上の構造体名（キー）と、その実体へのポインタおよびサイズ情報をマッピングするために利用されます。

前提条件と不変条件

  • key は SHMEM_INDEX_KEYSIZE（48バイト）以内の文字列である必要があります。
  • location は、共有メモリセグメントが全プロセスで同一の仮想アドレスにマッピングされていることを前提としています。
  • 共有メモリ上のデータ構造は一度割り当てられると解放できないため、本構造体で管理されるエントリも永続的に存在します。

アルゴリズムまたは振る舞い

共有メモリの初期化時や新しいデータ構造の確保時に、ShmemIndex ハッシュテーブルのエントリとして生成されます。ShmemInitStruct はこの構造体を参照して、指定された名前の構造体が既に存在するかを検索し、存在しない場合にのみ ShmemAlloc を呼び出して新しい領域を確保し、本構造体に情報を記録します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/shmem.c 内の ShmemInitStruct 等の管理関数が、ハッシュテーブルの要素として本構造体を操作します。また、src/backend/storage/ipc/ipci.c にて初期化時に参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • HTAB: ShmemIndex を構成するハッシュテーブル。
  • ShmemIndex: 共有メモリ上の全名前付き構造体を管理するグローバルなハッシュテーブル。

失敗モードと境界条件

  • key が SHMEM_INDEX_KEYSIZE を超える場合、切り詰めやオーバーフローのリスクがあります。
  • 共有メモリの枯渇時、ShmemAlloc が失敗することで本構造体の登録も失敗します。

保守担当者向けメモ

  • 本構造体は共有メモリのメタデータを保持するため、構造体のサイズ変更は共有メモリのレイアウトに影響を与える可能性があります。
  • EXEC_BACKEND 環境（Windows等）では、プロセス起動時にこの構造体を通じて共有メモリ上のポインタを再構築する処理が重要となります。
