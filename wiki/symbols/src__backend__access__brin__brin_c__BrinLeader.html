BrinLeader

定義

  • 種別: struct
  • 定義箇所: src/backend/access/brin/brin.c:122

目的

BRIN インデックスの並列構築時において、リーダープロセスが管理する状態を保持するための構造体です。

シグネチャまたは形状

typedef struct BrinLeader
{
    ParallelContext *pcxt;
    int             nparticipanttuplesorts;
    BrinShared      *brinshared;
    Sharedsort      *sharedsort;
    Snapshot        snapshot;
    WalUsage        *walusage;
    BufferUsage     *bufferusage;
} BrinLeader;

入力と出力

本構造体は関数引数や BrinBuildState のメンバとして保持され、並列処理の制御に使用されます。直接的な入出力関数は定義されていません。

前提条件と不変条件

  • nparticipanttuplesorts は、正常に起動したワーカープロセスの数と、リーダー自身がワーカーとして参加する場合はその分を加算した合計値である必要があります。
  • brinshared および sharedsort は、並列構築全体で共有される状態を指す必要があります。

アルゴリズムまたは振る舞い

  • 並列構築のライフサイクル管理において、リーダープロセスが共有メモリ上の状態（TOC）を繰り返し参照するオーバーヘッドを避けるためのキャッシュとして機能します。
  • _brin_end_parallel 関数において、並列処理の終了処理を行う際の主要な参照先となります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin.c 内の並列構築ロジック（_brin_end_parallel 等）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • BrinShared: インデックス構築全体の共有状態。
  • BrinBuildState: インデックス構築中の状態を保持する構造体。
  • ParallelContext: PostgreSQL の並列処理コンテキスト。
  • Sharedsort: tuplesort 管理下の共有状態。

失敗モードと境界条件

  • DISABLE_LEADER_PARTICIPATION が設定されている場合、リーダープロセスはワーカーとして参加せず、nparticipanttuplesorts の計算に影響を与えます。

保守担当者向けメモ

  • 本構造体は brin.c 内の並列構築処理に特化しています。並列構築のフローを追う際は、_brin_end_parallel および _brin_parallel_merge との連携を確認してください。
