ExtensionVersionInfo

定義

  • **種別**: struct
  • **定義箇所**: src/backend/commands/extension.c:109

目的

拡張機能のバージョン間における更新パスを計算するためのグラフ構造体です。Dijkstra法を用いた最短パス探索アルゴリズムにおいて、各バージョンをノードとして管理するために使用されます。

シグネチャまたは形状

typedef struct ExtensionVersionInfo
{
    char       *name;           /* 開始バージョンの名前 */
    List       *reachable;      /* 到達可能な ExtensionVersionInfo のリスト */
    bool        installable;    /* インストールスクリプトの有無 */
    bool        distance_known; /* Dijkstra法: 距離確定フラグ */
    int         distance;       /* Dijkstra法: 現在の推定距離 */
    struct ExtensionVersionInfo *previous; /* Dijkstra法: 最適な先行ノード */
} ExtensionVersionInfo;

入力と出力

  • **入力**: 拡張機能のバージョン名（文字列）。
  • **出力**: 指定されたバージョン名に対応する ExtensionVersionInfo 構造体（既存のものがない場合は新規作成）。

前提条件と不変条件

  • reachable リストは、当該バージョンから直接更新可能な他のバージョンへの参照を保持します。
  • distance および previous は、更新パス探索アルゴリズムの実行中にのみ有効な作業状態として使用されます。

アルゴリズムまたは振る舞い

  • 拡張機能の更新時、get_ext_ver_list を通じて全スクリプトを走査し、本構造体のグラフを構築します。
  • find_update_path 関数により、Dijkstra法を用いてソースバージョンからターゲットバージョンへの最短更新パスを決定します。
  • palloc を使用してメモリ上に動的に確保され、リスト構造で管理されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/extension.c 内のバージョン管理ロジック（get_ext_ver_list 等）。
  • **呼び出し先**: 構造体内部で List（PostgreSQLのリスト構造）を保持し、メモリ管理のために palloc を使用します。

関連データ構造

  • List: reachable フィールドで利用されるノードのリスト。
  • ExtensionControlFile: 拡張機能のメタデータを保持する構造体。

失敗モードと境界条件

  • 指定されたバージョン名が見つからない場合、または更新パスが存在しない場合、アルゴリズムは適切なエラーを報告します。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本構造体は更新パス計算のための「一時的な」グラフノードです。永続的なカタログ情報ではないため、セッション終了時にメモリが解放されることを前提としています。
  • reachable リストの構築ロジックを変更する際は、src/backend/commands/extension.c 内のグラフ探索アルゴリズムとの整合性に注意してください。
