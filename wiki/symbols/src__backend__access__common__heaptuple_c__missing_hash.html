missing_hash

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/heaptuple.c:103

目的

missing_cache_key 構造体に含まれる値をハッシュ化し、ハッシュテーブルのキーとして利用可能な uint32 型のハッシュ値を生成します。

シグネチャまたは形状

static uint32 missing_hash(const void *key, Size keysize)

入力と出力

  • **入力**:
  • key: missing_cache_key 構造体へのポインタ（void * として渡される）。
  • keysize: キーのサイズ（Size 型）。
  • **出力**:
  • uint32: entry->value を対象とした hash_bytes による計算結果。

前提条件と不変条件

  • key 引数は missing_cache_key 型のポインタとしてキャスト可能である必要があります。
  • entry->value が指すメモリ領域は、entry->len で指定された長さ分だけ有効である必要があります。

アルゴリズムまたは振る舞い

1. 引数 key を missing_cache_key 型のポインタにキャストします。 2. 構造体内の value フィールドと len フィールドを参照します。 3. hash_bytes 関数を呼び出し、value の内容に基づいたハッシュ値を計算して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/heaptuple.c:133 において、ハッシュテーブル制御構造体（hash_ctl）の hash コールバックとして設定されています。
  • **呼び出し先**: hash_bytes（common/hashfn.h で定義）。

関連データ構造

  • missing_cache_key: 本関数がハッシュ化対象とするデータ構造。

失敗モードと境界条件

  • key が NULL の場合や、entry->value が指すメモリが不正な場合は未定義の動作となります。
  • keysize 引数は本関数内では直接使用されていません。

保守担当者向けメモ

  • 本関数はハッシュテーブルのコールバックとして使用されるため、シグネチャは hash_ctl の要件に準拠しています。
  • hash_bytes の仕様変更や、missing_cache_key 構造体の定義変更時には本関数の実装を確認してください。
