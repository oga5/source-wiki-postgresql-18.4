PartClauseTarget

定義

  • 種別: enum
  • 定義箇所: src/backend/partitioning/partprune.c:92

目的

パーティションプルーニング（不要なパーティションの除外）を実行するタイミングを識別するための列挙型です。

シグネチャまたは形状

typedef enum PartClauseTarget
{
	PARTTARGET_PLANNER,
	PARTTARGET_INITIAL,
	PARTTARGET_EXEC,
} PartClauseTarget;

入力と出力

  • 本シンボル自体は列挙型であり、関数ではありません。
  • gen_partprune_steps などの関数において、プルーニングステップ生成の目的を指定する引数として使用されます。

前提条件と不変条件

  • プルーニングのフェーズ（プランニング時、実行開始時、スキャン実行時）のいずれか一つを明示的に指定する必要があります。

アルゴリズムまたは振る舞い

  • PARTTARGET_PLANNER: プランニング段階で静的にプルーニングを行う場合に指定します。
  • PARTTARGET_INITIAL: 実行開始時（executor startup）にプルーニングを行う場合に指定します。
  • PARTTARGET_EXEC: 各プランノードのスキャン実行時に動的にプルーニングを行う場合に指定します。

呼び出し元と呼び出し先

  • 呼び出し元: gen_partprune_steps 関数がこの型を引数として受け取ります。
  • 呼び出し先: 本シンボルは列挙型であり、直接の呼び出し先はありません。

関連データ構造

  • GeneratePruningStepsContext: ステップ生成中の状態を保持する構造体で、PartClauseTarget 型のメンバ target を保持します。

失敗モードと境界条件

  • 該当する情報はありません。

保守担当者向けメモ

  • 本列挙型は、partprune.c 内のプルーニングロジックが「いつ」評価されるかを制御する重要なフラグです。新しいプルーニングフェーズを追加する場合は、この列挙型を拡張する必要があります。
