find_or_create_child_node

定義

  • 種別: function
  • 定義箇所: contrib/fuzzystrmatch/daitch_mokotoff.c:215

目的

Daitch-Mokotoff Soundex アルゴリズムのコード生成過程において、親ノードから派生する子ノードを検索または新規作成し、音韻コードの蓄積状態を管理する。

シグネチャまたは形状

static dm_node *find_or_create_child_node(dm_node *parent, char code_digit, ArrayBuildState *soundex)

入力と出力

  • **入力**:
  • parent: 現在の音韻状態を保持する dm_node 構造体。
  • code_digit: 追加する音韻コードの数字（文字）。
  • soundex: 最終的な Soundex コードを格納するための ArrayBuildState。
  • **出力**:
  • 処理を継続すべき新しい dm_node へのポインタ。
  • コードが完了（6桁に到達）したか、既存ノードが完了済みである場合は NULL を返す。

前提条件と不変条件

  • parent->children 配列は、音韻コードの数字に対応するインデックスを持つ。
  • DM_CODE_DIGITS（6桁）に達したノードは完了とみなされる。
  • 新規作成されるノードは、start_node の初期値で初期化され、親ノードの soundex 状態を継承する。

アルゴリズムまたは振る舞い

1. code_digit に対応するインデックスから既存の dm_node を検索する。 2. 既存ノードが存在する場合:

  • 完了済み（soundex_length >= DM_CODE_DIGITS）であれば NULL を返す。

    • 3. 既存ノードが存在しない場合:

  • palloc_object で新しい dm_node を確保し、親の children 配列に登録する。
  • 親の soundex 情報をコピーし、code_digit を追加して長さを更新する。

    • 4. 新規ノードが 6 桁に達した場合:

  • cstring_to_text_with_len を用いてテキスト形式に変換し、accumArrayResult で soundex 配列に格納する。
  • NULL を返す。

    • 5. 6 桁未満であれば、作成したノードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/fuzzystrmatch/daitch_mokotoff.c 内のコード生成ロジック（daitch_mokotoff_coding 等から呼び出される）。
  • **呼び出し先**: palloc_object（メモリ確保）、cstring_to_text_with_len（テキスト変換）、accumArrayResult（配列蓄積）。

関連データ構造

  • dm_node: 音韻状態を保持するツリー構造体。
  • ArrayBuildState: PostgreSQL の配列構築用構造体。

失敗モードと境界条件

  • code_digit が範囲外の場合、配列アクセスで境界外参照が発生する可能性がある（呼び出し元での保証に依存）。
  • メモリ不足時は palloc_object がエラーを発生させる。
  • 6 桁に達したノードは再利用されず、出力配列へ送られた後に NULL が返される。

保守担当者向けメモ

  • 本関数はツリーの分岐を管理する重要な箇所である。dm_node の children 配列のインデックスが code_digit - '0' に依存しているため、ルールテーブルの変更時にはこの整合性に注意すること。
