pgaio_io_has_target

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_target.c:39

目的

指定されたI/Oハンドル（PgAioHandle）に有効なターゲットが割り当てられているかどうかを判定します。

シグネチャまたは形状

bool pgaio_io_has_target(PgAioHandle *ioh)

入力と出力

  • **入力**: PgAioHandle *ioh - 状態を確認するI/Oハンドルへのポインタ。
  • **出力**: bool - ターゲットが設定されている場合は true、それ以外（PGAIO_TID_INVALID）の場合は false を返します。

前提条件と不変条件

  • ioh は有効な PgAioHandle 構造体を指している必要があります。
  • ターゲットが PGAIO_TID_INVALID でない状態が、I/O操作実行の前提条件となります。

アルゴリズムまたは振る舞い

ioh->target の値が PGAIO_TID_INVALID と等しくないことを確認し、その結果を真偽値で返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio.c:430 (Assert内)
  • src/backend/storage/aio/aio_io.c:160 (Assert内)
  • **呼び出し先**:
  • 提供されたソース情報には、本関数から呼び出される関数は含まれていません。

関連データ構造

  • PgAioHandle: I/O操作の状態を保持する主要構造体。target フィールドを介して本関数と連携します。

失敗モードと境界条件

  • ioh が NULL の場合、未定義の動作（セグメンテーション違反）が発生する可能性があります。

保守担当者向けメモ

  • 本関数は主に Assert マクロ内で、I/O操作開始前の整合性チェックとして使用されています。
  • ターゲットの定義については src/backend/storage/aio/aio_target.c の pgaio_target_info レジストリを参照してください。
