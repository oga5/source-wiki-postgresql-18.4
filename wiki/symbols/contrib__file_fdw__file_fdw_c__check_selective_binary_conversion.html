check_selective_binary_conversion

定義

  • 種別: function
  • 定義箇所: contrib/file_fdw/file_fdw.c:937

目的

外部テーブルのクエリ実行において、COPY コマンドの列リストを最適化（選択的変換）できるかどうかを判定し、必要な列名のリストを生成します。

シグネチャまたは形状

static bool check_selective_binary_conversion(RelOptInfo *baserel, Oid foreigntableid, List **columns);

入力と出力

  • **入力**:
  • baserel: 外部テーブルのプランナリレーション情報。
  • foreigntableid: 外部テーブルの OID。
  • **出力**:
  • 戻り値: 最適化が可能な場合は true、それ以外は false。
  • *columns: 最適化が可能な場合、クエリで参照されている列名のリスト。最適化不可または全列必要な場合は NIL。

前提条件と不変条件

  • ファイルフォーマットが binary の場合は常に false を返します。
  • システム属性や削除済み列、生成列は対象外となります。
  • has_wholerow（行全体参照）が発生した場合は、最適化を断念し false を返します。

アルゴリズムまたは振る舞い

1. **フォーマット確認**: 外部テーブルのオプションから format を取得し、binary であれば即座に false を返します。 2. **列の特定**: pull_varattnos を使用して、ターゲットリストおよび制限句（baserestrictinfo）から参照されている属性を特定します。 3. **列名収集**: 特定された属性に対応する列名をシステムカタログから取得し、columns リストに追加します。 4. **最適化判定**:

  • 参照列が全属性数と一致する場合、または行全体参照が含まれる場合は最適化不要と判断し false を返します。
  • それ以外の場合、収集した列リストを返し true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/file_fdw/file_fdw.c 内のプランニング関連処理（行 565 付近）。
  • **呼び出し先**: GetForeignTable, pull_varattnos, table_open, RelationGetDescr, bms_next_member, TupleDescAttr, lappend, makeString, pstrdup, table_close 等の PostgreSQL 内部 API。

関連データ構造

  • RelOptInfo: プランナが保持するリレーション情報。
  • ForeignTable: 外部テーブルの定義情報。
  • Bitmapset: 参照されている属性番号の集合。
  • TupleDesc: テーブルの属性定義情報。

失敗モードと境界条件

  • **binary フォーマット**: 最適化対象外として処理されます。
  • **全列参照**: クエリがテーブルの全列を必要とする場合、最適化のメリットがないため false を返します。
  • **行全体参照**: SELECT * 等でテーブル全体が参照される場合、列の絞り込みができないため false を返します。

保守担当者向けメモ

  • 本関数は COPY コマンドの列リスト指定機能を利用して、不要な列の読み込みをスキップするためのものです。
  • COPY コマンドの仕様変更（特に列リストの制約）が直接影響するため、変更時は src/backend/commands/copyfrom.c の挙動と照らし合わせてください。
