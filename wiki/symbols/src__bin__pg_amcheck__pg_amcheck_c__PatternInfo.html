PatternInfo

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_amcheck/pg_amcheck.c:33

目的

コマンドラインから指定された検証対象パターンを保持し、データベース、スキーマ、リレーションの正規表現への分解結果およびフィルタリング条件を管理する。

シグネチャまたは形状

typedef struct PatternInfo
{
    const char *pattern;
    char       *db_regex;
    char       *nsp_regex;
    char       *rel_regex;
    bool        heap_only;
    bool        btree_only;
    bool        matched;
} PatternInfo;

入力と出力

  • **入力**: コマンドライン引数から取得した文字列（pattern）。
  • **出力**: 解析された正規表現文字列（db_regex, nsp_regex, rel_regex）および検証対象の制約フラグ（heap_only, btree_only）、マッチング状態（matched）。

前提条件と不変条件

  • pattern はコマンドラインから渡された変更されていない文字列を指す。
  • db_regex, nsp_regex, rel_regex は解析前は NULL であり、解析後に動的に割り当てられる。

アルゴリズムまたは振る舞い

1. コマンドライン引数からパターンを読み込み、構造体に格納する。 2. パターンをデータベース、スキーマ、リレーションの各正規表現に分解して保持する。 3. 検証対象がヒープのみか、B-tree 索引のみかをフラグで管理する。 4. データベース検索処理において、パターンが少なくとも1つのリレーションにマッチしたかどうかを matched フラグで追跡する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_amcheck/pg_amcheck.c 内のパターン解析およびリスト構築ロジック（extend_pattern_info_array 等）から参照される。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行わない。

関連データ構造

  • PatternInfoArray: PatternInfo 構造体の配列を管理するコンテナ。
  • AmcheckOptions: PatternInfo を含む検証オプション全体を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: pg_realloc を使用している箇所でメモリ確保に失敗した場合、プログラムは終了する可能性がある。
  • 正規表現の解析失敗: パターンが不正な形式である場合、解析ロジック側でエラーが発生する可能性がある。

保守担当者向けメモ

  • matched フラグは、検証実行後にパターンが有効であったかを判定するために使用される。
  • heap_only と btree_only は排他的な制約として扱われることが想定されるため、ロジック変更時は整合性に注意すること。
