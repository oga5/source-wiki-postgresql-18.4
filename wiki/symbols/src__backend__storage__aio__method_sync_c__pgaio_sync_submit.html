pgaio_sync_submit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_sync.c:41

目的

同期実行が強制されるI/Oメソッドにおいて、誤って非同期サブミット処理が呼び出された場合にエラーを発生させるガード関数です。

シグネチャまたは形状

static int pgaio_sync_submit(uint16 num_staged_ios, PgAioHandle **staged_ios)

入力と出力

  • 入力:
  • num_staged_ios: サブミット対象のI/O数。
  • staged_ios: PgAioHandle 構造体へのポインタ配列。
  • 出力:
  • 戻り値: 常に 0 を返しますが、到達した時点でエラーが発生します。

前提条件と不変条件

  • 本関数は、同期実行が必要なI/Oパスにおいて呼び出されるべきではありません。
  • 本関数が呼び出されることは、システム設計上の不変条件（同期I/Oは非同期サブミットを通らない）に対する違反とみなされます。

アルゴリズムまたは振る舞い

1. elog(ERROR, ...) を実行し、同期的に実行されるべきI/Oが非同期サブミットとして処理されたことを示すエラーを発生させます。 2. 処理を終了し、0 を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/aio/method_sync.c 内の pgaio_sync_ops 構造体（IoMethodOps）にて、.submit フィールドに関数ポインタとして登録されています。
  • 呼び出し先: elog マクロ（PostgreSQLのエラー報告機構）。

関連データ構造

  • PgAioHandle: I/O操作を管理するハンドル構造体。
  • IoMethodOps: I/Oメソッドの動作を定義する構造体。本関数はこの構造体を通じて参照されます。

失敗モードと境界条件

  • 本関数自体が呼び出された時点で、無条件に ERROR レベルのログが発行され、現在のトランザクションまたはプロセスが中断されます。

保守担当者向けメモ

  • 本関数は「同期I/Oのフォールバック」という設計意図を強制するためのガードレールです。実際の同期I/O処理は本関数ではなく、呼び出し側のロジックで行われる必要があります。
  • 呼び出し元に関する詳細な実行フローは、AIOサブシステムのメインエントリポイントである src/backend/storage/aio/aio.c を参照してください。
