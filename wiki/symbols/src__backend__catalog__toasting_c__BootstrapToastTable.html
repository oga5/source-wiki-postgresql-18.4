BootstrapToastTable

定義

  • **種別**: function
  • **定義箇所**: src/backend/catalog/toasting.c:97

目的

PostgreSQLのブートストラップ処理（initdb）において、指定されたテーブルに対してTOASTテーブルおよびそのインデックスを明示的に作成・関連付けます。

シグネチャまたは形状

void BootstrapToastTable(char *relName, Oid toastOid, Oid toastIndexOid)

入力と出力

  • **入力**:
  • relName: 対象テーブルの名前。
  • toastOid: TOASTテーブルに割り当てるOID。
  • toastIndexOid: TOASTテーブルのインデックスに割り当てるOID。
  • **出力**: なし（成功時はカタログ更新を行い、失敗時はエラーをスロー）。

前提条件と不変条件

  • 対象リレーションは RELKIND_RELATION（通常のテーブル）または RELKIND_MATVIEW（マテリアライズドビュー）である必要があります。
  • AccessExclusiveLock を取得して処理を行います。

アルゴリズムまたは振る舞い

1. table_openrv を使用して対象テーブルを排他ロックで開きます。 2. 対象がテーブルまたはマテリアライズドビューであることを検証し、該当しない場合はエラーをスローします。 3. create_toast_table を呼び出し、TOASTテーブルとインデックスの構築、カタログ更新、依存関係の登録を実行します。 4. create_toast_table が false を返した場合（TOASTテーブルが不要な場合）、エラーをスローします。 5. table_close を呼び出し、ロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: ブートストラップパーサー（src/include/catalog/genbki.h に言及あり）。
  • **呼び出し先**: table_openrv, create_toast_table, table_close。

関連データ構造

  • Relation: テーブル情報を保持する構造体。
  • Oid: オブジェクト識別子。

失敗モードと境界条件

  • 対象テーブルが存在しない場合、table_openrv によりエラーが発生します。
  • 対象がテーブルまたはマテリアライズドビューでない場合、elog(ERROR) により処理が中断されます。
  • create_toast_table が失敗した場合、elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数はブートストラップ専用のインターフェースです。通常のテーブル作成フローでは NewRelationCreateToastTable 等が使用されます。
  • create_toast_table に渡される引数のうち、Datum は (Datum) 0、is_internal は false、toastOid は InvalidOid 以外の値が指定されます。
