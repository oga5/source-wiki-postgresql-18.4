func_select_candidate

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_func.c:1007

目的

複数の関数候補リストから、引数の型情報に基づきヒューリスティックを用いて最適な関数を一つ選択する。

シグネチャまたは形状

FuncCandidateList func_select_candidate(int nargs, Oid *input_typeids, FuncCandidateList candidates)

入力と出力

  • **入力**:
  • nargs: 引数の数。
  • input_typeids: 入力引数の型OID配列。
  • candidates: 候補となる関数のリスト。
  • **出力**:
  • 解決された単一の FuncCandidateList、または解決不能な場合の NULL。

前提条件と不変条件

  • nargs が FUNC_MAX_ARGS を超える場合、エラーを報告する。
  • 入力型にドメインが含まれる場合、内部的にベース型に変換して評価する。

アルゴリズムまたは振る舞い

1. **正確な一致の検索**: 入力型と完全に一致する引数型を持つ候補を優先する。 2. **優先型の考慮**: 正確な一致が複数ある、または存在しない場合、型カテゴリに基づき「優先型（Preferred Type）」を持つ候補を優先する。 3. **不明な型（Unknown）の解決**: 引数に UNKNOWN 型が含まれる場合、型カテゴリの推論を試み、カテゴリ不一致の候補を除外する。 4. **絞り込み**: 各段階で候補が1つに絞り込まれた時点でその候補を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_func.c (複数箇所)
  • src/backend/parser/parse_oper.c:342
  • **呼び出し先**:
  • getBaseType (型解決用)
  • TypeCategory (型カテゴリ判定用)
  • IsPreferredType (優先型判定用)
  • ereport (エラー報告用)

関連データ構造

  • FuncCandidateList: 関数候補を保持する連結リスト構造。
  • Oid: PostgreSQLの型識別子。
  • TYPCATEGORY: 型のカテゴリ分類。

失敗モードと境界条件

  • nargs > FUNC_MAX_ARGS: ERRCODE_TOO_MANY_ARGUMENTS エラー。
  • 候補が絞り込めない場合: NULL を返す。
  • UNKNOWN 型の引数があり、かつカテゴリの推論が不可能な場合: NULL を返す。

保守担当者向けメモ

  • 本関数はPostgreSQLの型解決ロジックの核心部である。修正を行う際は、UNKNOWN 型の扱いおよび型カテゴリの優先順位付けが既存のクエリ挙動に与える影響を十分に検証すること。
