SearchNamedReplicationSlot

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slot.c:508

目的

指定された名前を持つレプリケーションスロットを共有メモリ上のスロット配列から検索し、該当するスロット構造体へのポインタを返します。

シグネチャまたは形状

ReplicationSlot *SearchNamedReplicationSlot(const char *name, bool need_lock)

入力と出力

  • **入力**:
  • name: 検索対象のスロット名（文字列）。
  • need_lock: 検索中に ReplicationSlotControlLock を取得・解放するかどうかを指定するフラグ。
  • **出力**:
  • 該当するスロットが見つかった場合はその ReplicationSlot 構造体へのポインタ。
  • 見つからない場合は NULL。

前提条件と不変条件

  • ReplicationSlotCtl が初期化済みであること。
  • need_lock が true の場合、呼び出し元は ReplicationSlotControlLock を保持していない状態で呼び出す必要があります（関数内で取得・解放を行うため）。

アルゴリズムまたは振る舞い

1. need_lock が true の場合、ReplicationSlotControlLock を共有モード（LW_SHARED）で取得します。 2. max_replication_slots まで定義された共有メモリ上のスロット配列を走査します。 3. 各スロットについて、in_use フラグが立っており、かつスロット名が name と一致するかを strcmp で確認します。 4. 一致するスロットが見つかった時点でループを終了し、そのポインタを保持します。 5. need_lock が true の場合、ReplicationSlotControlLock を解放します。 6. 検索結果（ポインタまたは NULL）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/slotsync.c
  • src/backend/replication/slot.c
  • src/backend/replication/walsender.c
  • src/backend/utils/activity/pgstat_replslot.c
  • src/backend/utils/adt/genfile.c
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease (条件付き)
  • strcmp

関連データ構造

  • ReplicationSlot: 検索対象となるスロットの構造体。
  • ReplicationSlotCtl: スロット配列を保持するグローバル制御構造体。
  • ReplicationSlotControlLock: スロット配列の走査を保護する軽量ロック。

失敗モードと境界条件

  • 指定された名前のスロットが存在しない場合、NULL を返します。
  • need_lock を false で呼び出す場合、呼び出し元で適切なロック（ReplicationSlotControlLock 等）が保持されている必要があります。さもないと、走査中にスロットの状態が変更される競合が発生する可能性があります。

保守担当者向けメモ

  • 本関数はスロットの検索のみを行い、スロットの所有権の取得や状態変更は行いません。
  • ReplicationSlotControlLock の取得階層（ReplicationSlotAllocationLock > ReplicationSlotControlLock）に注意してください。
