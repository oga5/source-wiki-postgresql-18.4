hstore_subscript_handler

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_subs.c:285

目的

PostgreSQL の実行エンジンに対し、hstore 型に対する添字演算（hstore_val['key']）を処理するためのルーチン群（SubscriptRoutines）を提供します。

シグネチャまたは形状

Datum hstore_subscript_handler(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（標準的な PostgreSQL 関数呼び出し規約）。
  • **出力**: SubscriptRoutines 構造体へのポインタを Datum として返します。

前提条件と不変条件

  • 本関数は PG_FUNCTION_INFO_V1 マクロによって登録されている必要があります。
  • 返される SubscriptRoutines は静的変数として定義されており、実行期間中不変です。

アルゴリズムまたは振る舞い

1. SubscriptRoutines 構造体を以下の設定で初期化します。

  • transform: hstore_subscript_transform を指定。
  • exec_setup: hstore_exec_setup を指定。
  • fetch_strict: true（入力が NULL の場合、結果も NULL を返す）。
  • fetch_leakproof: true（存在しないキーへのアクセスは NULL を返し、情報を漏洩させない）。
  • store_leakproof: false（代入操作はエラーを発生させる可能性があるため）。

    • 2. 初期化した構造体へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の実行エンジン（添字演算の初期化時）。
  • **呼び出し先**:
  • hstore_subscript_transform
  • hstore_exec_setup

関連データ構造

  • SubscriptRoutines: 添字演算の挙動を定義する構造体。

失敗モードと境界条件

  • 本関数自体は失敗しませんが、返されるハンドラ内の store_leakproof が false に設定されている通り、hstore への代入操作はエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本実装は hstore の添字演算が単一の text 型キーのみを扱うという単純な仕様に基づいています。
  • fetch_strict や leakproof フラグの変更が必要な場合は、この静的構造体の定義を更新してください。
