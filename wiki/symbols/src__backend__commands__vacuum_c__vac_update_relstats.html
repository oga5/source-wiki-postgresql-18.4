vac_update_relstats

定義

  • src/backend/commands/vacuum.c:1441

目的

指定されたリレーションの統計情報（ページ数、タプル数、可視性情報）およびトランザクション関連のメタデータ（relfrozenxid, relminmxid）を pg_class カタログに永続化する。

シグネチャまたは形状

void vac_update_relstats(Relation relation,
                         BlockNumber num_pages, double num_tuples,
                         BlockNumber num_all_visible_pages,
                         BlockNumber num_all_frozen_pages,
                         bool hasindex, TransactionId frozenxid,
                         MultiXactId minmulti,
                         bool *frozenxid_updated, bool *minmulti_updated,
                         bool in_outer_xact)

入力と出力

  • **入力**: 対象リレーション、更新対象の統計値、および更新フラグ。in_outer_xact が真の場合、DDL 関連のフラグ（relhasindex 等）の更新はスキップされる。
  • **出力**: なし（副作用として pg_class を更新）。frozenxid_updated および minmulti_updated を通じて更新の成否を呼び出し元に通知する。

前提条件と不変条件

  • pg_class に対する RowExclusiveLock が保持されていること。
  • relfrozenxid および relminmxid は、原則として過去の値より進んでいる必要がある。ただし、既存値が「未来」にあると判断された場合は、強制的に上書きされる。

アルゴリズムまたは振る舞い

1. pg_class を systable_inplace_update_begin で開き、対象リレーションのタプルをコピーする。 2. 統計値（relpages, reltuples, relallvisible, relallfrozen）を比較し、変更があれば更新フラグを立てる。 3. in_outer_xact が偽の場合、hasindex やルール/トリガーの存在有無に基づき、relhasindex 等のフラグを更新する。 4. frozenxid および minmulti を検証し、必要に応じて更新する。未来の XID が格納されていた場合は警告を発行する。 5. 変更があった場合のみ systable_inplace_update_finish を呼び出し、変更をコミットする。

呼び出し元と呼び出し先

  • **呼び出し元**: vacuumlazy.c（vacuum_rel 等）、analyze.c、relation_stats.c からの呼び出しが確認されている。
  • **呼び出し先**: table_open, systable_inplace_update_begin, systable_inplace_update_finish, table_close 等のカタログ操作関数。

関連データ構造

  • Relation: 対象テーブルのメタデータ。
  • Form_pg_class: pg_class カタログの行構造。

失敗モードと境界条件

  • pg_class 内に対象リレーションの行が見つからない場合、ERROR を発生させる。
  • relfrozenxid や relminmxid が不正な値（未来の値）であると判定された場合、警告（WARNING）を出力しつつ値を上書きする。

保守担当者向けメモ

  • 本関数は systable_inplace_update を使用しており、カタログ更新のオーバーヘッドを最小限に抑える設計となっている。
  • in_outer_xact フラグは、トランザクションのネスト状況に応じて DDL 属性の更新を制御するために使用される。変更を加える際は、このフラグがカタログの整合性に与える影響を考慮すること。
