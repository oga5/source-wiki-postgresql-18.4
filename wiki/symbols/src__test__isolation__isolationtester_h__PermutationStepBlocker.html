PermutationStepBlocker

定義

  • 種別: struct
  • 定義箇所: src/test/isolation/isolationtester.h:49

目的

分離レベルテストにおいて、特定のステップが実行される前に満たされるべきブロック条件（待機条件）を定義・保持するために使用されます。

シグネチャまたは形状

typedef struct {
    char       *stepname;
    PermutationStepBlockerType blocktype;
    int         num_notices;
    Step       *step;
    int         target_notices;
} PermutationStepBlocker;

入力と出力

  • **入力**: stepname（対象ステップ名）、blocktype（条件種別）、num_notices（通知数条件用）がパーサーにより設定されます。
  • **出力**: check_testspec() 実行後、step フィールドに参照先の Step 構造体へのポインタが格納されます。

前提条件と不変条件

  • check_testspec() が呼び出される前は、step フィールドは NULL である可能性があります。
  • num_notices は blocktype が PSB_NUM_NOTICES の場合にのみ使用されます。

アルゴリズムまたは振る舞い

  • テスト実行エンジンは、この構造体を参照して、対象のステップが実行可能か（ブロック条件が解除されているか）を判断します。
  • target_notices は実行時のワークスペースとして使用され、他セッションからの通知状況を追跡します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/isolation/isolationtester.c 内の各関数（338行目、680行目、843行目、1086行目など）で、PermutationStep から参照され、実行制御に使用されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • PermutationStep: 本構造体を保持する親構造体。blockers 配列を通じて管理されます。
  • Step: step フィールドが指し示す、実行対象のステップ定義。
  • PermutationStepBlockerType: blocktype に使用される列挙型。

失敗モードと境界条件

  • check_testspec() によるリンク解決が失敗した場合、step ポインタが正しく設定されず、実行時に予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 新しいブロック条件を追加する場合は、PermutationStepBlockerType 列挙型を拡張し、isolationtester.c 内の判定ロジックを更新してください。
  • target_notices は実行時の状態保持用であるため、テストの並列実行や再実行時には適切にリセットされる必要があります。
