calc_s2k_iter_salted

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-s2k.c:125

目的

OpenPGP仕様（RFC 4880）に基づき、ソルトと反復回数（Iterated）を指定してパスフレーズからバイナリ鍵を導出する。

シグネチャまたは形状

static int calc_s2k_iter_salted(PGP_S2K *s2k, PX_MD *md, const uint8 *key, unsigned key_len);

入力と出力

  • **入力**:
  • s2k: S2K設定（ソルト、反復回数、出力先バッファ、鍵長を含む）。
  • md: ハッシュ計算用インターフェース。
  • key: パスフレーズ文字列。
  • key_len: パスフレーズの長さ。
  • **出力**:
  • 成功時は 0 を返す。
  • s2k->key バッファに生成されたバイナリ鍵が格納される。

前提条件と不変条件

  • s2k->iter からデコードされた反復回数分、ハッシュ計算が繰り返される。
  • 必要な鍵長（s2k->key_len）を満たすまで、ハッシュ計算とバッファへの書き込みがループする。

アルゴリズムまたは振る舞い

1. s2k_decode_count を用いて反復回数を取得する。 2. 必要な鍵長に達するまで、以下の処理を繰り返す：

  • ハッシュコンテキストをリセットし、必要に応じて先行するゼロバイト（preload）をハッシュに追加する。
  • ソルトとパスフレーズをハッシュに供給する。
  • 反復回数に達するまで、ソルトとパスフレーズを交互にハッシュへ供給し続ける。
  • ハッシュ結果を s2k->key バッファにコピーする。

    • 3. 最後に px_memset を使用して、計算に使用した一時バッファをゼロクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-s2k.c 内の pgp_s2k_process から呼び出される（直接の参照箇所は pgp-s2k.c:301）。
  • **呼び出し先**: s2k_decode_count、px_md_result_size、px_md_reset、px_md_update、px_md_finish、px_memset。

関連データ構造

  • PGP_S2K: S2Kパラメータを保持する構造体。
  • PX_MD: ハッシュ関数抽象化インターフェース。

失敗モードと境界条件

  • remain（残りの鍵長）がハッシュの出力サイズより大きい場合と小さい場合で、コピー処理を切り替える。
  • 反復回数の計算において、curcnt が count を超えないよう制御されている。

保守担当者向けメモ

  • 本関数は機密情報（パスフレーズおよび生成された鍵）を扱うため、一時バッファのゼロクリアが必須である。
  • RFC 4880の仕様変更やハッシュアルゴリズムの拡張を行う際は、px_md インターフェースの挙動と整合性を確認すること。
