gbt_enumeq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_enum.c:46

目的

GiST インデックスにおける ENUM 型の等価性判定（=）を提供します。

シグネチャまたは形状

static bool gbt_enumeq(const void *a, const void *b, FmgrInfo *flinfo);

入力と出力

  • **入力**:
  • a, b: 比較対象の ENUM 値（内部的に Oid としてキャストして参照）。
  • flinfo: 関数マネージャー情報（本関数内では未使用）。
  • **出力**:
  • bool: a と b が等しい場合に true、それ以外は false を返します。

前提条件と不変条件

  • a および b は、有効な Oid を指すポインタである必要があります。
  • ENUM 型は内部的に Oid として表現されているという前提に基づいています。

アルゴリズムまたは振る舞い

1. 引数 a と b をそれぞれ const Oid * にキャストします。 2. 指し示す Oid 値同士を比較し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_enum.c:94 にて、tinfo 構造体の初期化時に参照されています。
  • **呼び出し先**: 直接的な関数呼び出しはありません。

関連データ構造

  • Oid: ENUM 型の内部表現。
  • tinfo: gbtree_ninfo 構造体。本関数を比較演算子として保持します。

失敗モードと境界条件

  • a または b が NULL ポインタである場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.c の汎用 GiST ユーティリティからコールバックされることを想定しています。
  • ENUM の比較ロジックは OID の比較と同一であるため、btree_oid.c の実装と整合しています。
