TupleHashEntryGetAdditional

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/executor/executor.h:187

目的

TupleHashTable 内の特定の TupleHashEntry に付随する追加データ領域へのポインタを取得します。

シグネチャまたは形状

void *TupleHashEntryGetAdditional(TupleHashTable hashtable, TupleHashEntry entry)

入力と出力

  • **入力**:
  • hashtable: 対象のハッシュテーブル構造体。
  • entry: データを取得したいハッシュエントリ。
  • **出力**:
  • 付随データ領域の先頭アドレス（void *）。データ領域が存在しない場合は NULL を返します。

前提条件と不変条件

  • hashtable->additionalsize が 0 より大きい場合、entry->firstTuple のメモリ上の直前位置に additionalsize 分の領域が確保されていることを前提とします。

アルゴリズムまたは振る舞い

1. hashtable->additionalsize が 0 より大きいかを確認します。 2. 0 より大きい場合、entry->firstTuple のポインタから hashtable->additionalsize を減算したアドレスを返します。 3. 0 以下の場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execGrouping.c
  • src/backend/executor/nodeAgg.c
  • src/backend/executor/nodeSetOp.c
  • **呼び出し先**:
  • 本関数はインライン関数であり、他の関数を呼び出しません。

関連データ構造

  • TupleHashTable: ハッシュテーブルの管理構造体。additionalsize を保持します。
  • TupleHashEntry: ハッシュエントリ構造体。firstTuple を保持します。

失敗モードと境界条件

  • hashtable->additionalsize が 0 の場合、データ領域は存在しないものとして NULL を返します。
  • entry が NULL の場合の挙動は定義されていません。

保守担当者向けメモ

  • 本関数は、ハッシュエントリのメモリレイアウト（firstTuple の直前に付随データを配置する）に強く依存しています。構造体のメモリ配置を変更する際は注意してください。
