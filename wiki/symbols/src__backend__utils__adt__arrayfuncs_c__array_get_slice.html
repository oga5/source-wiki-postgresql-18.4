array_get_slice

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arrayfuncs.c:2029

目的

多次元配列から指定された範囲（スライス）を抽出し、新しい配列として返します。

シグネチャまたは形状

Datum array_get_slice(Datum arraydatum, int nSubscripts, int *upperIndx, int *lowerIndx, bool *upperProvided, bool *lowerProvided, int arraytyplen, int elmlen, bool elmbyval, char elmalign)

入力と出力

  • **入力**:
  • arraydatum: 対象となる配列の Datum。
  • nSubscripts: 指定する次元数。
  • upperIndx, lowerIndx: スライスの開始および終了インデックスの配列。
  • upperProvided, lowerProvided: 各次元でインデックスが指定されているかを示すフラグ。
  • arraytyplen, elmlen, elmbyval, elmalign: 配列および要素の型情報。
  • **出力**: 抽出された部分配列を含む Datum。

前提条件と不変条件

  • 固定長配列（arraytyplen > 0）のスライスはサポートされておらず、エラーとなります。
  • 入力配列がTOASTされている場合、内部で DatumGetArrayTypeP により展開されます。
  • スライス範囲が配列の境界を超える場合、配列の境界に合わせて自動的に切り詰められます。
  • 結果の配列の各次元の下限（Lower Bound）は常に 1 に設定されます。

アルゴリズムまたは振る舞い

1. 入力配列の次元情報（ndim, dim, lb）を取得します。 2. 指定されたインデックスが配列の範囲外であれば、境界値に補正します。 3. 補正後の範囲が空であれば、construct_empty_array を呼び出し空の配列を返します。 4. mda_get_range を用いてスライス後の各次元のサイズ（span）を計算します。 5. array_slice_size を用いて、結果配列に必要なメモリサイズを算出します。 6. palloc0 でメモリを確保し、ヘッダー情報と次元情報をコピーします。 7. array_extract_slice を呼び出し、元の配列からデータを抽出して新しい配列へコピーします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/arrayfuncs.c (複数箇所)
  • src/backend/utils/adt/arraysubs.c (arraysubs.c 内のサブスクリプト処理)
  • **呼び出し先**:
  • DatumGetArrayTypeP, construct_empty_array, mda_get_range, array_slice_size, array_extract_slice, palloc0, SET_VARSIZE 等。

関連データ構造

  • ArrayType: PostgreSQLの配列内部表現。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • **エラー**: 固定長配列に対して呼び出された場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラーを報告します。
  • **境界条件**:
  • 指定されたスライスが配列の範囲外である場合、静かに切り詰められます。
  • スライス結果が空になる場合、空の配列が返されます。
  • ndim が MAXDIM を超える、または nSubscripts と整合しない場合、空の配列が返されます。

保守担当者向けメモ

  • 本関数は、結果の配列のNULLビットマップを、元の配列がNULLビットマップを持つ場合にのみ生成します。
  • 以前のバージョン（7.3以前）では下限値（Lower Bound）をコピーしていましたが、現在は一貫性を保つため常に 1 に設定されています。
