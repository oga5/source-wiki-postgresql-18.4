PLy_spi_execute_plan

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_spi.c:170

目的

Python から渡されたプリペアドステートメント（PLyPlanObject）を、指定された引数リストを用いて PostgreSQL の SPI (Server Programming Interface) を介して実行し、その結果を Python オブジェクトとして返します。

シグネチャまたは形状

PyObject *PLy_spi_execute_plan(PyObject *ob, PyObject *list, long limit)

入力と出力

  • **入力**:
  • ob: 実行対象の PLyPlanObject。
  • list: クエリ引数を含む Python シーケンス（NULL 可）。
  • limit: クエリの最大行数制限。
  • **出力**:
  • 成功時: 実行結果を格納した PLyResultObject。
  • 失敗時: NULL を返し、適切な Python 例外（TypeError や SPI エラー等）を設定。

前提条件と不変条件

  • list が指定されている場合、PySequence である必要があります。
  • 引数の数（nargs）は、プラン作成時に定義された数と一致しなければなりません。
  • 実行はサブトランザクション内で保護されます。

アルゴリズムまたは振る舞い

1. **引数検証**: list の型と要素数を検証し、プランの期待値と一致するか確認します。 2. **サブトランザクション開始**: PLy_spi_subtransaction_begin を呼び出し、実行環境を保護します。 3. **一時コンテキスト**: 引数変換用に AllocSetContextCreate で一時メモリコンテキストを作成します。 4. **型変換**: PLy_output_convert を使用して、Python オブジェクトを PostgreSQL の Datum 形式に変換します。 5. **クエリ実行**: SPI_execute_plan を呼び出し、結果を PLy_spi_execute_fetch_result で Python オブジェクトへ変換します。 6. **後処理**: サブトランザクションをコミットし、一時メモリコンテキストを解放します。 7. **例外処理**: PG_CATCH ブロックにて、エラー発生時にサブトランザクションをアボートし、エラー情報を Python 例外として再送出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_planobject.c:137
  • src/pl/plpython/plpy_spi.c:164
  • **呼び出し先**:
  • PLy_spi_subtransaction_begin / commit / abort
  • SPI_execute_plan
  • PLy_spi_execute_fetch_result
  • PLy_output_convert

関連データ構造

  • PLyPlanObject: 実行されるプリペアドステートメントの構造体。
  • PLyExecutionContext: 現在の実行コンテキスト。
  • SPITupleTable: SPI から返されるタプルテーブル。

失敗モードと境界条件

  • 引数の型不一致や個数不一致: TypeError を発生させます。
  • SPI_execute_plan の失敗: PLy_exc_spi_error を発生させます。
  • メモリ不足: palloc 等の PostgreSQL 内部エラーにより longjmp が発生し、PG_CATCH で捕捉されます。

保守担当者向けメモ

  • 本関数は PG_TRY ブロック内でメモリコンテキストを切り替えて操作するため、メモリリークを防ぐために PG_CATCH でのサブトランザクションアボートとコンテキスト解放が必須です。
  • SPI_execute_plan の戻り値が負の場合、SPI 固有のエラーとして処理されます。
