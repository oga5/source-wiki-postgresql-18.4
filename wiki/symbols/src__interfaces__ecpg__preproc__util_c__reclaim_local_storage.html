reclaim_local_storage

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/util.c:181

目的

ECPGプリプロセッサの構文解析サイクルで使用された一時メモリ領域（loc_chunk）を一括解放し、メモリリークを防止する。

シグネチャまたは形状

void reclaim_local_storage(void)

入力と出力

  • 入力: なし（グローバル変数 loc_chunks を参照）
  • 出力: なし（副作用としてメモリを解放し、loc_chunks を NULL にリセットする）

前提条件と不変条件

  • loc_chunks が指すメモリ領域は、loc_alloc によって確保されたものであること。
  • 本関数呼び出し後、以前に確保された一時メモリへのポインタはすべて無効となる。

アルゴリズムまたは振る舞い

1. グローバル変数 loc_chunks を起点に、連結リスト構造となっている各チャンクを順次走査する。 2. 各チャンクに対して free() を実行し、メモリを解放する。 3. 走査終了後、loc_chunks を NULL に初期化し、リストが空であることを示す。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/preproc/ecpg.c（解析サイクルの境界で呼び出されることが示唆されている）
  • 呼び出し先: free（標準ライブラリ）

関連データ構造

  • loc_chunk: 一時メモリ管理のためのチャンク構造体。

失敗モードと境界条件

  • loc_chunks が既に NULL の場合、ループは実行されず、安全に終了する。

保守担当者向けメモ

  • 本関数は個別の free を不要にするための基盤である。解析の節目で必ず呼び出されていることを確認すること。
  • loc_chunk は 8192 バイト単位で確保されるため、非常に大きな文字列を扱う場合はメモリ消費量に注意が必要である。
