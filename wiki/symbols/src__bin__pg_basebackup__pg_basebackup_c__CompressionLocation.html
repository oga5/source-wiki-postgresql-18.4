CompressionLocation

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_basebackup/pg_basebackup.c:122

目的

バックアップデータの圧縮処理をどこで実行するか（クライアント側かサーバー側か、あるいは未指定か）を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
	COMPRESS_LOCATION_UNSPECIFIED,
	COMPRESS_LOCATION_CLIENT,
	COMPRESS_LOCATION_SERVER,
} CompressionLocation;

入力と出力

  • 本シンボル自体は列挙型であり、関数への引数や変数として利用されます。
  • COMPRESS_LOCATION_UNSPECIFIED: 圧縮場所が未指定であることを示します。
  • COMPRESS_LOCATION_CLIENT: クライアント側での圧縮を示します。
  • COMPRESS_LOCATION_SERVER: サーバー側での圧縮を示します。

前提条件と不変条件

  • pg_basebackup における圧縮設定の選択肢として排他的に利用されます。

アルゴリズムまたは振る舞い

  • 圧縮の実行場所を決定するロジックにおいて、この列挙値に基づいて処理が分岐します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_basebackup/pg_basebackup.c 内の複数の関数（main 関数を含む）で、圧縮設定の保持や引数として参照されています。
  • 呼び出し先: 本シンボルは列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • 圧縮場所が COMPRESS_LOCATION_UNSPECIFIED のまま処理が進んだ場合、後続の処理で適切なデフォルト値への解決またはエラーハンドリングが必要です。

保守担当者向けメモ

  • 圧縮機能の拡張や変更を行う際は、この列挙型に新しい値を追加する必要があるか検討してください。現在の値は pg_basebackup の CLI 引数解析およびバックアップストリームの構築ロジックと密接に関連しています。
