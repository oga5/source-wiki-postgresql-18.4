SimpleLruInit

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/slru.c:251

目的

SLRU（Simple LRU）バッファ管理機構の初期化を行います。共有メモリ上のデータ構造の確保と初期化、および制御構造体（SlruCtl）の設定を担います。

シグネチャまたは形状

void SimpleLruInit(SlruCtl ctl, const char *name, int nslots, int nlsns,
                   const char *subdir, int buffer_tranche_id, int bank_tranche_id,
                   SyncRequestHandler sync_handler, bool long_segment_names)

入力と出力

  • **入力**:
  • ctl: 初期化対象の制御構造体。
  • name: 共有メモリ領域の識別名。
  • nslots: 管理するバッファスロット数。
  • nlsns: ページごとの LSN グループ数。
  • subdir: データ格納先のディレクトリパス。
  • buffer_tranche_id, bank_tranche_id: LWLock のトランシェ ID。
  • sync_handler: チェックポイント時の同期ハンドラ。
  • long_segment_names: セグメントファイル名の命名規則フラグ。
  • **出力**: なし（ctl および共有メモリ領域を副作用として初期化）。

前提条件と不変条件

  • nslots は SLRU_MAX_ALLOWED_BUFFERS 以下であること。
  • Postmaster 起動時（!IsUnderPostmaster）に共有メモリの初期化が行われること。
  • 呼び出し元で PagePrecedes コールバックが設定されていること。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を呼び出し、共有メモリ領域を取得または確保する。 2. プロセスが Postmaster の配下でない場合、共有メモリ内の SlruSharedData 構造体および各配列（ページバッファ、ステータス、ロック等）を初期化する。 3. ページバッファを BLCKSZ 単位で割り当て、各スロットを SLRU_PAGE_EMPTY 状態に設定する。 4. nbanks（nslots / SLRU_BANK_SIZE）個の銀行ロックを初期化する。 5. 最後に、渡された ctl 構造体に共有メモリへのポインタやディレクトリパス等の設定値を格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • clog.c, commit_ts.c, multixact.c, subtrans.c, async.c, predicate.c, test_slru.c 等の各サブシステム。
  • **呼び出し先**:
  • ShmemInitStruct, pg_atomic_init_u64, pgstat_get_slru_index, LWLockInitialize, SimpleLruShmemSize, strlcpy。

関連データ構造

  • SlruCtl: SLRU インスタンスの制御構造体。
  • SlruSharedData: 共有メモリ上の状態管理構造体。
  • LWLockPadded: ロック管理用構造体。

失敗モードと境界条件

  • nslots が許容最大値を超えた場合、Assert により停止する。
  • 共有メモリ確保時に found フラグの不整合（Postmaster 配下で未初期化、またはその逆）が発生した場合、Assert により停止する。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウトを決定するため、構造体のオフセット計算（MAXALIGN 等）を変更する際は、SimpleLruShmemSize との整合性に注意すること。
  • 銀行ロックの数は nslots に依存するため、SLRU_BANK_SIZE の変更はメモリレイアウト全体に影響する。
