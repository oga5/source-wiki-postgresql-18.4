PgStatShared_Common

定義

  • 種別: struct
  • 定義箇所: src/include/utils/pgstat_internal.h:121

目的

統計システムの共有メモリ上に配置されるデータ構造の共通ヘッダーとして機能し、統計データの整合性確認と排他制御を提供します。

シグネチャまたは形状

typedef struct PgStatShared_Common
{
    uint32      magic;      /* 妥当性確認用 */
    LWLock      lock;       /* 統計内容（ヘッダー直後のデータ）を保護するロック */
} PgStatShared_Common;

入力と出力

  • **入力**: 統計情報の更新やリセットを行う際、本構造体の lock を介して共有メモリ上のデータへアクセスします。
  • **出力**: 統計情報の読み取り時、magic フィールドを用いてデータ構造の妥当性を検証します。

前提条件と不変条件

  • 本構造体は共有メモリ上に配置されるため、ポインタではなくオフセットや dsa_pointer を用いてアクセスする必要があります。
  • 統計データ本体は本ヘッダーの直後に配置されることを前提としています。

アルゴリズムまたは振る舞い

  • **排他制御**: 統計データの更新時には lock を取得し、競合を防ぎます。
  • **妥当性検証**: magic フィールドにより、共有メモリ上の構造体が期待される形式であるかを実行時に確認します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c, pgstat_backend.c, pgstat_database.c, pgstat_replslot.c, pgstat_shmem.c 等の統計管理モジュールが、共有メモリ上の統計データへのアクセスヘッダーとして利用しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • PgStatShared_HashEntry: 共有統計ハッシュテーブルのエントリ。
  • LWLock: 共有メモリ上の統計データを保護するために使用される軽量ロック。

失敗モードと境界条件

  • magic フィールドの不一致は、共有メモリ上のデータ破損または初期化不良を示唆します。
  • lock の取得失敗やデッドロックは、統計システム全体のアクセス停止を招く可能性があります。

保守担当者向けメモ

  • 本構造体は共有メモリレイアウトの基盤であるため、フィールドの追加や変更は統計システムの全種別に影響を与えます。
  • 統計データへのアクセスは、必ず lock を適切に管理し、競合を避ける設計を維持してください。
