pgwin32_connect

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/socket.c:358

目的

Windows環境において、POSIX準拠の connect() システムコールをWinsock2 API上でエミュレートし、PostgreSQLのシグナル処理と統合された接続待機を実現する。

シグネチャまたは形状

int pgwin32_connect(SOCKET s, const struct sockaddr *addr, int addrlen);

入力と出力

  • **入力**:
  • s: 接続対象のソケットハンドル。
  • addr: 接続先アドレス情報へのポインタ。
  • addrlen: addr が指す構造体のサイズ。
  • **出力**:
  • 成功時は 0 を返す。
  • 失敗時は -1 を返し、内部で TranslateSocketError() を呼び出して errno を適切に設定する。

前提条件と不変条件

  • ソケット s はWinsock2で作成されていること。
  • 本関数は connect マクロを通じて呼び出されることを想定している（src/include/port/win32_port.h）。

アルゴリズムまたは振る舞い

1. WSAConnect を呼び出し、接続を試みる。 2. 接続が即座に成功した場合（r == 0）、そのまま 0 を返す。 3. WSAEWOULDBLOCK 以外のエラーが発生した場合は、TranslateSocketError() を経由してエラーを報告し -1 を返す。 4. WSAEWOULDBLOCK の場合、pgwin32_waitforsinglesocket を呼び出し、接続完了イベントまたはシグナル発生まで待機する。 5. 待機中にシグナルが配送された場合はループを継続し、接続完了まで待機を繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/win32_port.h にて connect マクロとして定義されており、PostgreSQLのネットワーク通信を行う各モジュールから呼び出される。
  • **呼び出し先**:
  • WSAConnect (Winsock2 API)
  • WSAGetLastError (Winsock2 API)
  • TranslateSocketError (内部ユーティリティ)
  • pgwin32_waitforsinglesocket (内部ユーティリティ)

関連データ構造

  • SOCKET (Winsock2型)
  • struct sockaddr (POSIXソケットアドレス構造体)

失敗モードと境界条件

  • WSAConnect が WSAEWOULDBLOCK 以外で失敗した場合、Winsockエラーコードが errno に変換される。
  • 接続待機中にシグナルが割り込んだ場合、pgwin32_waitforsinglesocket の戻り値に応じてループが制御される。

保守担当者向けメモ

  • 本関数はWindows特有の非同期イベントモデルを同期的な connect として見せるためのラッパーである。
  • pgwin32_waitforsinglesocket の戻り値が 0 の間はループを継続する設計となっており、シグナル処理との統合が維持されている。
