BumpBlockFree

定義

  • 種別: static inline function
  • 定義箇所: src/backend/utils/mmgr/bump.c:594

目的

バンプアロケータのメモリブロックをリストから削除し、システムメモリへ解放する。

シグネチャまたは形状

static inline void BumpBlockFree(BumpContext *set, BumpBlock *block)

入力と出力

  • **入力**:
  • set: 対象となる BumpContext へのポインタ。
  • block: 解放対象の BumpBlock へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • 指定された block が KeeperBlock（コンテキスト自体を保持するブロック）であってはならない（Assert により検証）。

アルゴリズムまたは振る舞い

1. IsKeeperBlock を使用して、対象ブロックが保護対象でないことを確認する。 2. dlist_delete を呼び出し、BumpContext が管理するブロックリストから当該ブロックを削除する。 3. set の mem_allocated カウンタから、解放するブロックのサイズ分を減算する。 4. CLOBBER_FREED_MEMORY が定義されている場合、wipe_mem を呼び出してブロック内のメモリを上書きする。 5. free(block) を呼び出し、メモリをシステムに返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/bump.c:262（BumpReset 内でのブロック解放処理）
  • **呼び出し先**:
  • IsKeeperBlock (マクロ)
  • dlist_delete
  • wipe_mem (条件付き)
  • free

関連データ構造

  • BumpContext: メモリコンテキストのメタデータ構造体。
  • BumpBlock: メモリ割り当ての単位となるブロック構造体。

失敗モードと境界条件

  • KeeperBlock を解放しようとした場合、Assert が有効なビルドではアボートする。
  • リストに含まれていないブロックを渡した場合の動作は未定義。

保守担当者向けメモ

  • 本関数は BumpReset 等から呼び出され、コンテキストのメモリ再利用プロセスの一部を担う。
  • pfree とは異なり、個別のチャンクではなくブロック単位での解放を行う点に注意すること。
