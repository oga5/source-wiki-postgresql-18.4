get_tablespace_io_concurrency

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/spccache.c:214

目的

指定されたテーブルスペースにおけるI/O並行性設定（effective_io_concurrency）を取得します。

シグネチャまたは形状

int get_tablespace_io_concurrency(Oid spcid)

入力と出力

  • **入力**: spcid (Oid) - 対象のテーブルスペースのOID。
  • **出力**: int - 適用されるI/O並行性の値。

前提条件と不変条件

  • get_tablespace() を介してキャッシュエントリが取得可能であること。
  • テーブルスペース固有の設定が存在しない、または値が負である場合は、システム全体のデフォルト値が使用される。

アルゴリズムまたは振る舞い

1. get_tablespace(spcid) を呼び出し、該当するテーブルスペースのキャッシュエントリを取得する。 2. キャッシュエントリ内のオプション（spc->opts）を確認する。 3. オプションが未設定（NULL）であるか、effective_io_concurrency が負の値であれば、グローバル変数 effective_io_concurrency を返す。 4. それ以外の場合は、キャッシュされたテーブルスペース固有の effective_io_concurrency を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/aio/read_stream.c:577 にて、ストリーム読み込み時の最大I/O数決定のために呼び出されている。
  • **呼び出し先**: get_tablespace() を内部で呼び出す。

関連データ構造

  • TableSpaceCacheEntry: キャッシュされたテーブルスペース情報とオプションを保持する構造体。

失敗モードと境界条件

  • spcid が無効な場合、get_tablespace() の振る舞いに依存する。
  • テーブルスペース固有の設定値が負の場合、システム全体のデフォルト値へフォールバックする仕様となっている。

保守担当者向けメモ

  • 本関数はプランナやストレージ層がコスト計算やI/O制御を行う際に参照する。
  • キャッシュの無効化は InvalidateTableSpaceCacheCallback によって行われるため、本関数が返す値は常に最新のカタログ状態を反映する。
