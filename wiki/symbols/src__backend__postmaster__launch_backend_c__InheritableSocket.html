InheritableSocket

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/launch_backend.c:76

目的

EXEC_BACKEND モードにおいて、親プロセスから子プロセスへソケットハンドルを安全に受け渡すための構造体です。Windows環境等でプロセス間でソケットを共有するために必要な情報を保持します。

シグネチャまたは形状

typedef struct
{
    SOCKET      origsocket;      /* 元のソケット値、または PGINVALID_SOCKET */
    WSAPROTOCOL_INFO wsainfo;    /* Windowsソケットのプロトコル情報 */
} InheritableSocket;

入力と出力

  • **入力**: write_inheritable_socket 関数において、対象の SOCKET と子プロセスの PID から本構造体へ情報が書き込まれます。
  • **出力**: read_inheritable_socket 関数において、本構造体から SOCKET 値が復元されます。

前提条件と不変条件

  • origsocket が有効なソケットでない場合、値は PGINVALID_SOCKET である必要があります。
  • wsainfo は、Windowsの WSADuplicateSocket APIによって生成された情報を含んでいる必要があります。

アルゴリズムまたは振る舞い

1. **書き込み**: write_inheritable_socket が WSADuplicateSocket を呼び出し、ソケット情報を wsainfo に格納します。 2. **読み込み**: read_inheritable_socket が WSASocket を呼び出し、wsainfo を使用して子プロセス内で利用可能なソケットハンドルを再構築します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c 内のプロセス起動処理フロー。
  • **呼び出し先**: write_inheritable_socket および read_inheritable_socket 関数が本構造体を引数として操作します。

関連データ構造

  • SOCKET: Windowsソケットハンドル。
  • WSAPROTOCOL_INFO: Windowsソケットのプロトコル情報を保持する構造体。

失敗モードと境界条件

  • WSADuplicateSocket が失敗した場合、ソケットの継承は行えません。
  • origsocket が PGINVALID_SOCKET の場合、読み込み側はソケットの復元を試みず、無効なソケットとして扱います。

保守担当者向けメモ

  • 本構造体は EXEC_BACKEND モード（主にWindows）でのみ機能します。
  • wsainfo の内容はOS依存のバイナリデータであるため、プロセス間でのシリアライズ/デシリアライズの整合性に注意してください。
