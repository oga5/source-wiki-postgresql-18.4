PGEvent

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/libpq-int.h:155

目的

libpq におけるイベント処理機構において、登録されたイベントハンドラとその状態を保持するために使用されます。

シグネチャまたは形状

typedef struct PGEvent
{
    PGEventProc proc;           /* イベント発生時に呼び出される関数 */
    char       *name;           /* エラーメッセージ用名称 */
    void       *passThrough;    /* 登録時に指定されたポインタ */
    void       *data;           /* インスタンス固有の状態データ */
    bool        resultInitialized; /* RESULTCREATE/COPY 成功時に true */
} PGEvent;

入力と出力

  • 入力: PGEventProc 型の関数ポインタ、名称、および任意のデータポインタを保持します。
  • 出力: イベント処理の結果として、data フィールドを通じて状態が更新されます。

前提条件と不変条件

  • name はエラーメッセージの識別に使用されます。
  • resultInitialized は、RESULTCREATE または COPY イベントが成功したことを示すフラグとして機能します。

アルゴリズムまたは振る舞い

  • libpq 内部でイベント配列として管理され、接続や結果オブジェクトに関連付けられたイベントハンドラを順次実行するために利用されます。
  • fe-exec.c および libpq-events.c において、メモリの動的確保（malloc/realloc）を伴う配列操作の対象となります。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-exec.c（dupEvents 関数内での複製処理）、src/interfaces/libpq/libpq-events.c（イベント配列の拡張処理）。
  • 呼び出し先: proc フィールドに格納された PGEventProc 型の関数が、イベント発生時に呼び出されます。

関連データ構造

  • PGconn: 接続状態を保持する構造体であり、events フィールドとして PGEvent の配列を保持します。
  • PGEventProc: イベント処理を行う関数型。

失敗モードと境界条件

  • メモリ確保失敗: malloc や realloc が失敗した場合、イベント配列の拡張や複製が正常に行われません。
  • resultInitialized の不整合: イベント処理のライフサイクルにおいて、初期化が正しく行われない場合、後続のイベント処理で誤った状態判定が行われる可能性があります。

保守担当者向けメモ

  • 本構造体は libpq-int.h で定義された内部構造体であり、公開 API ではありません。
  • イベント配列は PGconn 内で動的に拡張されるため、ポインタの生存期間とメモリ管理には注意が必要です。
