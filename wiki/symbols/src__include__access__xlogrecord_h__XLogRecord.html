XLogRecord

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlogrecord.h:41

目的

PostgreSQLのWAL（Write-Ahead Logging）レコードの固定長ヘッダーを定義し、レコードのメタデータ、トランザクション情報、および整合性検証のための情報を保持する。

シグネチャまたは形状

typedef struct XLogRecord
{
    uint32      xl_tot_len;     /* レコード全体の合計長 */
    TransactionId xl_xid;       /* トランザクションID */
    XLogRecPtr  xl_prev;        /* 直前のWALレコードへのポインタ */
    uint8       xl_info;        /* フラグビット */
    RmgrId      xl_rmid;        /* リソースマネージャID */
    /* 2バイトのパディング（ゼロ初期化） */
    pg_crc32c   xl_crc;         /* レコードのCRC */
} XLogRecord;

入力と出力

  • **入力**: WAL書き込み時に XLogInsert 等を通じて各フィールドが設定される。
  • **出力**: WALストリームの一部としてディスクまたはメモリ上に書き出される。

前提条件と不変条件

  • XLogRecord 構造体自体は MAXALIGN に整列される。
  • 構造体の直後には XLogRecordBlockHeader および XLogRecordDataHeader がパディングなしで続く。
  • メモリ上で操作する際は、整列されたバッファにコピーしてからアクセスする必要がある。

アルゴリズムまたは振る舞い

1. **ヘッダー構築**: レコードの先頭に配置され、後続のブロックヘッダーやデータペイロードのメタデータとして機能する。 2. **整合性検証**: xl_crc は、レコード全体の整合性を検証するために使用される。 3. **解析**: XLogReader は本構造体を読み取り、xl_tot_len を用いてレコードの境界を特定し、xl_rmid に基づいて適切なリソースマネージャへ処理を委譲する。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • contrib/pg_walinspect/pg_walinspect.c (ReadNextXLogRecord)
  • src/backend/access/transam/xlog.c (WAL書き込みおよびリカバリ処理)
  • src/backend/access/transam/twophase.c
  • **呼び出し先**: 構造体定義のため、直接的な関数呼び出しは存在しない。

関連データ構造

  • XLogRecordBlockHeader: レコードに関連付けられたブロックのメタデータ。
  • XLogRecordDataHeaderShort / Long: レコードデータのヘッダー。
  • RmgrId: リソースマネージャの識別子。
  • XLogRecPtr: WALストリーム内の位置を示すポインタ。

失敗モードと境界条件

  • **アライメント**: 構造体直後のデータは整列されていないため、直接アクセスするとプラットフォーム依存の不正アクセスが発生する可能性がある。
  • **CRCエラー**: 書き込まれた xl_crc と計算値が一致しない場合、WALレコードは破損しているとみなされる。
  • **ブロック数制限**: 1つのレコードが参照可能なブロック数は XLR_MAX_BLOCK_ID（32）に制限される。

保守担当者向けメモ

  • xl_info の上位4ビットはリソースマネージャが自由に使用可能である。
  • 本構造体はWALの物理フォーマットを規定するため、変更時はリカバリ互換性に注意すること。
