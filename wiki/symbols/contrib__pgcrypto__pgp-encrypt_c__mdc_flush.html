mdc_flush

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-encrypt.c:114

目的

OpenPGP ストリームの終了時に、計算済みの MDC（Modification Detection Code）パケットを生成し、出力フィルタへ書き出す。

シグネチャまたは形状

static int mdc_flush(PushFilter *dst, void *priv);

入力と出力

  • **入力**:
  • dst: データ書き込み先の PushFilter。
  • priv: MDC 計算に使用する PX_MD コンテキスト（void * として渡される）。
  • **出力**:
  • int: pushf_write の実行結果（成功時に 0、失敗時にエラーコード）。

前提条件と不変条件

  • priv は有効な PX_MD 構造体へのポインタであること。
  • mdc_flush が呼び出されるまでに、対象データに対する px_md_update が完了していること。

アルゴリズムまたは振る舞い

1. MDC パケットのヘッダ（タグ 0xD3、長さ 0x14）を構築する。 2. ヘッダ部分を px_md_update でハッシュ計算に含める。 3. px_md_finish を呼び出し、計算されたダイジェストをパケットの末尾に格納する。 4. pushf_write を使用して、構築したパケット（ヘッダ 2 バイト + ダイジェスト）を dst に書き出す。 5. セキュリティのため、ローカルの pkt バッファを px_memset でゼロクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていませんが、mdc_init, mdc_write, mdc_flush, mdc_free がセットで管理されていることが示唆されています。
  • **呼び出し先**:
  • px_md_update
  • px_md_finish
  • pushf_write
  • px_memset

関連データ構造

  • PushFilter: データ書き込みパイプラインの抽象化。
  • PX_MD: ハッシュ計算用コンテキスト。
  • MDC_DIGEST_LEN: MDC ダイジェストの長さ（定数）。

失敗モードと境界条件

  • pushf_write が失敗した場合、その戻り値がそのまま返される。
  • priv が不正なポインタである場合、px_md_update 等の内部関数でクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数は PGP の MDC パケット（RFC 4880）を生成する。パケット構造の変更が必要な場合は、ヘッダのバイト列（pkt[0], pkt[1]）を確認すること。
  • px_memset によるバッファクリアは、機密情報（ダイジェスト）のメモリ残留を防ぐための重要なセキュリティ対策である。
