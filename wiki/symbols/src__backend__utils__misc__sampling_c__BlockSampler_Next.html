BlockSampler_Next

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/sampling.c:63

目的

リレーションのブロックサンプリングにおいて、Knuthの「Algorithm S」に基づき、次にサンプリング対象となるブロック番号を決定して返します。

シグネチャまたは形状

BlockNumber BlockSampler_Next(BlockSampler bs)

入力と出力

  • **入力**: BlockSampler bs（サンプリング状態を保持する構造体へのポインタ）
  • **出力**: 次に選択された BlockNumber を返します。

前提条件と不変条件

  • BlockSampler_HasMore(bs) が真であること（bs->t < bs->N かつ bs->m < bs->n が保証されていること）。
  • 内部状態 bs->t（現在のブロック位置）および bs->m（選択済みブロック数）が適切に管理されていること。

アルゴリズムまたは振る舞い

1. 残りのブロック数 K と、まだ選択すべきブロック数 k を算出します。 2. k >= K の場合、残りの全ブロックを選択対象とし、bs->m と bs->t をインクリメントして現在の bs->t を返します。 3. それ以外の場合、sampler_random_fract を用いて乱数 V を生成し、Knuthの Algorithm S に基づく確率的スキップ処理を行います。

  • V < p（p はスキップ確率）を満たす間、ブロックをスキップし、bs->t を進め、K を減らして p を再計算します。

    • 4. 最終的に選択されたブロック位置 bs->t をインクリメントし、その値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/analyze.c:1162
  • **呼び出し先**:
  • BlockSampler_HasMore（前提条件の確認）
  • sampler_random_fract（乱数生成）

関連データ構造

  • BlockSamplerData（bs が指す構造体）

失敗モードと境界条件

  • K == k となった場合、ループ条件 V < p は p が 0 になるため必ず終了し、必要なブロック数が確実に選択されるよう設計されています。
  • 浮動小数点数の丸め誤差を考慮し、ループ条件には <= ではなく < を使用しています。

保守担当者向けメモ

  • 本関数は Knuth の Algorithm S を効率化しており、選択されるブロックごとに乱数生成を行うのではなく、スキップ判定を最適化しています。
  • sampler_random_fract は 0.0 を返さないため、対数計算や除算におけるゼロ除算のリスクを回避しています。
