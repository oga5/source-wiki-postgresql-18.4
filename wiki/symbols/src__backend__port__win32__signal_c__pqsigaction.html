pqsigaction

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/signal.c:209

目的

Windows環境において、POSIXの sigaction 関数と同等のシグナルハンドラ設定機能を提供します。

シグネチャまたは形状

int pqsigaction(int signum, const struct sigaction *act, struct sigaction *oldact);

入力と出力

  • **入力**:
  • signum: 設定対象のシグナル番号。
  • act: 新しく設定するシグナルハンドラ等の構造体（NULL可）。
  • oldact: 現在の設定を保存するための構造体（NULL可）。
  • **出力**: 成功時は 0、失敗時は -1 を返し、errno に EINVAL を設定します。

前提条件と不変条件

  • signum は 0 以上かつ PG_SIGNAL_COUNT 未満である必要があります。
  • pg_signal_array は、シグナル番号をインデックスとしてシグナルハンドラ情報を保持する内部配列です。

アルゴリズムまたは振る舞い

1. signum が範囲外の場合、errno を EINVAL に設定し -1 を返します。 2. oldact が指定されている場合、現在の pg_signal_array[signum] の内容を oldact にコピーします。 3. act が指定されている場合、pg_signal_array[signum] を act の内容で更新します。 4. 正常終了時に 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/libpq/pqsignal.h にて sigaction マクロとして定義されており、PostgreSQLの各所からPOSIX互換APIとして呼び出されます。
  • **呼び出し先**: 本関数は内部で他の関数を呼び出しません。

関連データ構造

  • struct sigaction: シグナルハンドラやマスク情報を保持するPOSIX標準の構造体。
  • pg_signal_array: src/backend/port/win32/signal.c 内で定義されている、シグナル設定を保持するグローバル配列。

失敗モードと境界条件

  • signum が負の値、または PG_SIGNAL_COUNT 以上である場合、無効な引数としてエラーを返します。

保守担当者向けメモ

  • 本関数はWindows環境におけるシグナルエミュレーションの基盤です。pg_signal_array へのアクセスは、シグナルスレッドとメインスレッド間での競合が発生しないよう、必要に応じて排他制御が考慮されていることを確認してください。
