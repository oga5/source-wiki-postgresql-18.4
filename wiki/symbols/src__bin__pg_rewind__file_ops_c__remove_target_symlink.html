remove_target_symlink

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/file_ops.c:290

目的

ターゲットデータディレクトリ内の指定されたシンボリックリンクを安全に削除します。

シグネチャまたは形状

static void remove_target_symlink(const char *path)

入力と出力

  • **入力**: path (ターゲットディレクトリからの相対パス)
  • **出力**: なし（成功時は削除を実行、失敗時は pg_fatal によりプロセスを終了）

前提条件と不変条件

  • path は path_is_safe_for_extraction によって検証され、ターゲットディレクトリ外へのアクセスを試みない安全なパスである必要があります。
  • グローバル変数 datadir_target が有効なターゲットディレクトリパスを保持している必要があります。

アルゴリズムまたは振る舞い

1. path_is_safe_for_extraction(path) を呼び出し、パスの安全性を確認します。不正な場合は pg_fatal で終了します。 2. グローバル変数 dry_run が真であれば、何もせず終了します。 3. datadir_target と path を結合し、完全なパスを dstpath に生成します。 4. unlink(dstpath) を実行してシンボリックリンクを削除します。 5. unlink が 0 以外を返した場合、エラーメッセージと共に pg_fatal を呼び出し、プロセスを終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/file_ops.c:149 （remove_target 関数内から呼び出されます）
  • **呼び出し先**: path_is_safe_for_extraction, pg_fatal, unlink

関連データ構造

  • なし

失敗モードと境界条件

  • **パスの安全性違反**: path_is_safe_for_extraction が偽を返すと pg_fatal が発生します。
  • **削除失敗**: unlink が失敗した場合（権限不足やファイル不在など）、pg_fatal によりエラー詳細が出力されプロセスが終了します。

保守担当者向けメモ

  • 本関数は dry_run モードをサポートしています。新しい削除ロジックを追加する際は、このパターンに従ってください。
  • datadir_target への依存があるため、本関数を呼び出す前にターゲットディレクトリの設定が完了していることを確認してください。
