JoinType

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/nodes.h:293

目的

クエリ処理における結合（JOIN）の種類を識別するための列挙型です。パーサー、プランナー、エグゼキューター間で結合の論理的な性質を共有するために使用されます。

シグネチャまたは形状

typedef enum JoinType {
    JOIN_INNER,
    JOIN_LEFT,
    JOIN_FULL,
    JOIN_RIGHT,
    JOIN_SEMI,
    JOIN_ANTI,
    JOIN_RIGHT_SEMI,
    JOIN_RIGHT_ANTI,
    JOIN_UNIQUE_OUTER,
    JOIN_UNIQUE_INNER
} JoinType;

入力と出力

  • 入力: 該当なし（列挙型定義）
  • 出力: 結合ノードの属性として、プランナーやエグゼキューターの処理フローを決定する識別子として機能します。

前提条件と不変条件

  • SQLの標準的なJOIN構文に対応する型（JOIN_INNER等）は、パーサー出力（JoinExprノード等）に現れる必要があります。
  • JOIN_UNIQUE_OUTER および JOIN_UNIQUE_INNER はプランナー内部でのみ使用され、エグゼキューターではサポートされません。

アルゴリズムまたは振る舞い

  • **SQL結合**: JOIN_INNER, JOIN_LEFT, JOIN_FULL, JOIN_RIGHT は標準的な結合動作を定義します。
  • **セミ結合/アンチ結合**: JOIN_SEMI, JOIN_ANTI 等は、EXISTS句などのイディオムをプランナーが変換した結果として生成されます。
  • JOIN_SEMI: 一致するRHS行が存在するLHS行を1つ出力します。
  • JOIN_ANTI: 一致するRHS行が存在しないLHS行を1つ出力します（結果はNULL拡張されます）。
  • **一意性制約**: JOIN_UNIQUE_OUTER/INNER は、結合前に特定のパスを一意にする必要があることを示します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • contrib/postgres_fdw/: get_jointype_name や外部テーブルの結合処理で参照。
  • src/backend/nodes/: outfuncs.c および readfuncs.c でシリアライズ/デシリアライズに使用。
  • src/backend/optimizer/: clausesel.c 等のプランナー処理で参照。
  • 呼び出し先: 情報なし。

関連データ構造

  • JoinExpr: パーサー出力で使用されるノード。
  • RelOptInfo: プランナー内で結合関係を保持する構造体。

失敗モードと境界条件

  • エグゼキューターがサポートしていない JOIN_UNIQUE_OUTER や JOIN_UNIQUE_INNER をエグゼキューター層へ渡すと、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 新しい結合タイプを追加する場合は、outfuncs.c および readfuncs.c の対応する関数を更新し、シリアライズ処理が欠落しないように注意してください。
  • JOIN_SEMI や JOIN_ANTI の出力仕様（NULL拡張の有無など）は、プランナーとエグゼキューター間の契約として厳密に維持する必要があります。
