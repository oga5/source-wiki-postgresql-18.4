CachedFunctionHashEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/funccache.c:41

目的

関数実行データのキャッシュを管理するハッシュテーブルにおいて、個々のエントリを保持するための構造体です。

シグネチャまたは形状

typedef struct CachedFunctionHashEntry
{
	CachedFunctionHashKey key;	/* hash key, must be first */
	CachedFunction *function;	/* points to data of language-specific size */
} CachedFunctionHashEntry;

入力と出力

  • 入力: ハッシュテーブルの検索キーとして CachedFunctionHashKey を保持します。
  • 出力: コンパイル済み関数データへのポインタ CachedFunction * を保持します。

前提条件と不変条件

  • key フィールドは構造体の先頭に配置される必要があります（ハッシュテーブルの検索ロジックがこのメモリレイアウトに依存しているため）。
  • function フィールドが指すデータは、言語固有のサイズを持つ動的な構造体です。

アルゴリズムまたは振る舞い

  • ハッシュテーブル（cfunc_hashtable）の要素として使用されます。
  • hash_search を通じて、キーに基づくエントリの検索、挿入、削除が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/funccache.c 内の関数（cached_function_compile 等）が、hash_search を呼び出す際にこの構造体型をキャストして使用します。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • CachedFunctionHashKey: キャッシュの検索キー。
  • CachedFunction: コンパイル済み関数の基底構造体。
  • cfunc_hashtable: 本エントリを格納するグローバルなハッシュテーブル。

失敗モードと境界条件

  • メモリ不足: hash_search による挿入時にメモリ不足が発生する可能性があります。
  • キャッシュ無効化: pg_proc の更新に伴い、本エントリが cfunc_hashtable_delete によって破棄される場合があります。

保守担当者向けメモ

  • 本構造体は cfunc_hashtable の entrysize として sizeof(CachedFunctionHashEntry) が指定されています。
  • function ポインタが指すメモリ領域は TopMemoryContext に確保されるため、バックエンドの生存期間中保持されます。
