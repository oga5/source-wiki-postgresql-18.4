ONEXIT

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/ipc/ipc.c:74

目的

プロセス終了時に実行されるコールバック関数と、その関数に渡される引数を保持するためのデータ構造です。

シグネチャまたは形状

struct ONEXIT
{
    pg_on_exit_callback function;
    Datum       arg;
};

入力と出力

  • function: 終了時に呼び出される pg_on_exit_callback 型の関数ポインタ。
  • arg: コールバック関数に渡される Datum 型の引数。

前提条件と不変条件

  • 本構造体は、プロセス終了処理の各フェーズ（on_proc_exit、before_shmem_exit、on_shmem_exit）で管理される配列の要素として使用されます。
  • 終了処理はLIFO（後入れ先出し）順で実行される設計となっています。

アルゴリズムまたは振る舞い

  • プロセス終了時に、本構造体に格納された function が arg を引数として呼び出されます。
  • 終了処理の再帰を防ぐため、実行対象のコールバックは実行前にリストから削除される仕組みが前提となっています。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipc.c 内の終了処理関数群（proc_exit や shmem_exit など）。
  • 呼び出し先: 本構造体に格納された pg_on_exit_callback 型の関数。

関連データ構造

  • on_proc_exit_list: プロセス終了時に実行されるコールバックの配列。
  • on_shmem_exit_list: 共有メモリ解放時に実行されるコールバックの配列。
  • before_shmem_exit_list: 共有メモリ解放の直前に実行されるコールバックの配列。
  • 上記すべてのリストは MAX_ON_EXITS サイズの ONEXIT 配列として定義されています。

失敗モードと境界条件

  • MAX_ON_EXITS を超えるコールバックの登録はできません。
  • コールバック実行中にエラーが発生した場合、そのコールバックは再実行されず、終了処理が継続される設計です。

保守担当者向けメモ

  • 本構造体は「IPC」モジュール内でのプロセス終了処理の基盤です。
  • on_exit_reset() を呼び出すことで、フォーク後のプロセスが親プロセスの終了処理を引き継がないように管理する必要があります。
