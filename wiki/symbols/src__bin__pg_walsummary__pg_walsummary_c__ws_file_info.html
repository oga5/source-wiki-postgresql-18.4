ws_file_info

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_walsummary/pg_walsummary.c:30

目的

WAL サマリファイルの解析処理において、現在処理中のファイルに関する状態（ファイル記述子とファイル名）を保持し、コールバック関数間で共有するために使用されます。

シグネチャまたは形状

typedef struct ws_file_info
{
	int			fd;
	char	   *filename;
} ws_file_info;

入力と出力

本構造体はデータ保持用であり、関数ではありません。

  • fd: オープンされたファイルのファイル記述子。
  • filename: ファイルパスを示す文字列へのポインタ。

前提条件と不変条件

  • fd は有効なファイル記述子である必要があります。
  • filename は解析対象の WAL サマリファイルを示す有効なパスである必要があります。

アルゴリズムまたは振る舞い

本構造体自体は状態保持のためのコンテナです。walsummary_read_callback などのコールバック関数内で callback_arg を通じて受け取られ、ファイル読み込み操作のコンテキストとして利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: main 関数（src/bin/pg_walsummary/pg_walsummary.c:100）でインスタンス化されます。
  • 呼び出し先: walsummary_read_callback（src/bin/pg_walsummary/pg_walsummary.c:247）において、引数からキャストされて参照されます。

関連データ構造

  • BlockRefTableReader: 本構造体が保持する fd を用いて読み込みを行うリーダーインターフェース。

失敗モードと境界条件

  • ファイルがオープンできない場合や、fd が無効な状態での読み込み試行時に、コールバック関数経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は walsummary_read_callback がファイルシステム操作を行うための唯一の参照先です。構造体のフィールドを変更する場合は、関連するコールバック関数のシグネチャおよびアクセス箇所をすべて更新してください。
