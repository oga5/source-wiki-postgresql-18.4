ExplainPrintSerialize

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain.c:998

目的

EXPLAIN コマンドの出力において、シリアライズ処理に関する統計情報（実行時間、データ量、フォーマット、バッファ使用量）を整形して出力する。

シグネチャまたは形状

static void ExplainPrintSerialize(ExplainState *es, SerializeMetrics *metrics)

入力と出力

  • **入力**:
  • es: ExplainState 構造体へのポインタ。出力バッファやフォーマット設定を保持。
  • metrics: SerializeMetrics 構造体へのポインタ。シリアライズ処理の計測値（時間、転送バイト数、バッファ使用量）を保持。
  • **出力**: なし（es->str に文字列を追記、または ExplainProperty* 関数を通じて出力）。

前提条件と不変条件

  • es->serialize が EXPLAIN_SERIALIZE_NONE である場合は呼び出されないこと。
  • es->serialize は EXPLAIN_SERIALIZE_TEXT または EXPLAIN_SERIALIZE_BINARY のいずれかであること。

アルゴリズムまたは振る舞い

1. es->serialize の値に基づき、フォーマット文字列（"text" または "binary"）を決定する。 2. ExplainOpenGroup を呼び出し、"Serialization" グループを開始する。 3. es->format が EXPLAIN_FORMAT_TEXT の場合:

  • インデントを適用し、計測された時間（es->timing が有効な場合）と転送データ量をテキスト形式で出力する。
  • es->buffers が有効かつバッファ使用量が取得可能な場合、インデントを深めて show_buffer_usage を実行する。

    • 4. es->format がそれ以外（JSON/XML等）の場合:

  • ExplainProperty* 関数群を使用し、時間、データ量、フォーマットを構造化データとして出力する。
  • es->buffers が有効な場合、show_buffer_usage を実行する。

    • 5. ExplainCloseGroup を呼び出し、グループを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/explain.c 内の ExplainPrintPlan 等から呼び出される（直接的な呼び出し元は src/backend/commands/explain.c:646）。
  • **呼び出し先**: ExplainOpenGroup, ExplainIndentText, appendStringInfo, peek_buffer_usage, show_buffer_usage, ExplainPropertyFloat, ExplainPropertyUInteger, ExplainPropertyText, ExplainCloseGroup。

関連データ構造

  • ExplainState: 出力状態を管理する構造体。
  • SerializeMetrics: シリアライズ処理の統計情報を保持する構造体。

失敗モードと境界条件

  • es->serialize が EXPLAIN_SERIALIZE_NONE の場合に呼び出された場合、アサーション失敗または予期せぬ動作を引き起こす可能性がある。
  • metrics が NULL の場合、ポインタ参照エラーが発生する。

保守担当者向けメモ

  • 本関数は EXPLAIN 出力におけるシリアライズ統計の表示を担う。表示形式の変更や新しい統計項目の追加は、TEXT 形式とそれ以外の形式（JSON/XML）の両方の分岐を更新する必要がある。
