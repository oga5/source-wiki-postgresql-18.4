secure_destroy

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure.c:87

目的

セキュア通信層に関連するリソースの解放および終了処理を行う。

シグネチャまたは形状

void secure_destroy(void)

入力と出力

  • 入力: なし
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • USE_SSL が定義されている環境下では、SSL関連のリソースが適切に初期化されている必要がある。

アルゴリズムまたは振る舞い

  • USE_SSL が定義されている場合、be_tls_destroy() を呼び出し、TLS/SSL関連の終了処理を実行する。
  • USE_SSL が定義されていない場合、本関数は何もしない。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/postmaster.c:2031
  • 呼び出し先: be_tls_destroy（USE_SSL 定義時のみ）

関連データ構造

  • なし

失敗モードと境界条件

  • USE_SSL 定義時、内部で呼び出される be_tls_destroy の実装に依存する。

保守担当者向けメモ

  • 本関数は libpq.h で外部公開されている。
  • SSL以外の暗号化方式（GSSAPI等）の終了処理が必要な場合は、本関数内で適切に条件分岐を追加する必要がある。
