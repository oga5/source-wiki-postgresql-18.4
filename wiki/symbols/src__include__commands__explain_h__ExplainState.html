ExplainState

定義

  • 種別: struct
  • 定義箇所: src/include/commands/explain.h:41

目的

EXPLAIN コマンドによる実行計画の出力処理において、出力フォーマット（JSON, YAML, Text 等）の設定、出力バッファ、および走査状態を保持するために使用されます。

シグネチャまたは形状

構造体の内部定義は src/include/commands/explain_state.h に存在します。本ヘッダーファイル内では、ポインタ型として各種関数やフックの引数に使用されます。

入力と出力

  • 入力: EXPLAIN 実行時のオプション設定および実行計画ツリー。
  • 出力: 指定されたフォーマットに従い、実行計画の各ノード情報や統計情報を出力先（標準出力やログ等）へ書き出します。

前提条件と不変条件

  • ExplainOnePlan 等の関数において、初期化済みの ExplainState インスタンスが渡される必要があります。
  • 実行計画の走査中、保持するフォーマット設定は一貫して維持されます。

アルゴリズムまたは振る舞い

1. EXPLAIN コマンド実行時に ExplainState が生成されます。 2. ExplainOnePlan が実行計画ツリーを再帰的に走査する際、この構造体を参照して出力形式を決定します。 3. 走査中に explain_per_node_hook 等のフックが呼び出される場合、この構造体が引数として渡され、拡張モジュールが情報を追加できるようになります。 4. 最終的に、保持しているバッファの内容が出力されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • contrib/auto_explain/auto_explain.c
  • contrib/file_fdw/file_fdw.c
  • contrib/pg_overexplain/pg_overexplain.c
  • 呼び出し先:
  • 構造体内部の詳細は src/include/commands/explain_state.h に依存しています。

関連データ構造

  • PlanState: 実行計画のノード状態。
  • PlannedStmt: 計画済みのクエリ。
  • explain_per_node_hook_type: ExplainState を引数に取るノード出力用フック。

失敗モードと境界条件

  • メモリ不足時、ExplainState の初期化や出力バッファの拡張に失敗する可能性があります。
  • 不正なフォーマット設定が与えられた場合、出力生成時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 拡張機能（contrib 配下のモジュールなど）で頻繁に利用されるため、構造体の変更は後方互換性に影響を与える可能性があります。
