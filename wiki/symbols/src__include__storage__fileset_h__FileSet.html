FileSet

定義

  • 種別: struct
  • 定義箇所: src/include/storage/fileset.h:22

目的

関連する一時ファイルを論理的な名前空間（ディレクトリ）としてグループ化し、管理するための構造体です。複数のバックエンド間での共有や、トランザクションを跨いだ永続化が必要な一時ファイルの管理を目的とします。

シグネチャまたは形状

typedef struct FileSet
{
	pid_t		creator_pid;	/* 作成プロセスID */
	uint32		number;			/* プロセスごとの識別番号 */
	int			ntablespaces;	/* 使用するテーブルスペース数 */
	Oid			tablespaces[8]; /* 使用するテーブルスペースのOID */
} FileSet;

入力と出力

  • **入力**: FileSetInit() を通じて現在のセッションの temp_tablespaces 設定がキャプチャされ、構造体に格納されます。
  • **出力**: FileSetCreate() や FileSetOpen() を呼び出す際、この構造体内の情報に基づいて物理的なファイルパスが生成されます。

前提条件と不変条件

  • tablespaces[8] という固定長配列を使用しているため、設定可能な一時テーブルスペース数には最大8つという上限があります。
  • FileSetDeleteAll() が明示的に呼び出されない限り、関連する一時ファイルは自動的に削除されません。

アルゴリズムまたは振る舞い

1. **初期化**: FileSetInit により、現在のセッション設定に基づいたテーブルスペース情報がキャッシュされます。 2. **ファイル操作**: FileSetCreate は内部で FilePath を生成し、PathNameCreateTemporaryFile を呼び出して物理ファイルを作成します。ディレクトリが存在しない場合は PathNameCreateTemporaryDir が呼び出されます。 3. **クリーンアップ**: FileSetDeleteAll は、セットに関連付けられた物理ディレクトリを走査し、含まれる全ファイルを削除します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/worker.c: FileSet のメモリ確保およびストリーミング処理での利用。
  • src/backend/storage/file/buffile.c: BufFile の基盤として FileSet を利用。
  • **呼び出し先**:
  • storage/fd.h で定義された低レベルファイル操作関数（PathNameCreateTemporaryFile 等）。

関連データ構造

  • src/include/storage/sharedfileset.h: FileSet を拡張し、共有メモリを介してプロセス間で共有可能にした構造体。

失敗モードと境界条件

  • 一時テーブルスペースの設定数が8を超えた場合、構造体の固定長配列により制限が発生します。
  • 明示的な削除処理を行わない場合、一時ファイルがディスク上に残り続ける可能性があります。

保守担当者向けメモ

  • 本構造体は src/backend/storage/file/fileset.c に実装されている関数群によって操作されます。
  • FileSet は、同じ名前で繰り返しオープン・クローズを行う必要がある場合や、トランザクションを跨ぐ生存期間が必要な場合に適しています。
