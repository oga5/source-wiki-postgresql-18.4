ExecAsyncRequestDone

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execAsync.c:136

目的

非同期実行リクエストが完了したことを記録し、取得した結果をリクエスト構造体に格納します。

シグネチャまたは形状

void ExecAsyncRequestDone(AsyncRequest *areq, TupleTableSlot *result)

入力と出力

  • areq: 完了状態を更新する対象の AsyncRequest 構造体へのポインタ。
  • result: 非同期処理によって取得された TupleTableSlot（完了を示す結果）。NULL を指定することも可能です。
  • 出力: なし（副作用として areq のメンバを更新）。

前提条件と不変条件

  • areq は有効な AsyncRequest 構造体を指している必要があります。
  • 本関数呼び出し後、areq->request_complete は必ず true となります。

アルゴリズムまたは振る舞い

1. areq->request_complete に true を代入し、リクエストの完了をマークします。 2. areq->result に引数 result を代入し、取得したタプルスロットを保持します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/postgres_fdw/postgres_fdw.c 内の複数の箇所から呼び出されています。
  • 呼び出し先: 本関数は直接的な関数呼び出しを行いません。

関連データ構造

  • AsyncRequest: 非同期リクエストの状態を管理する構造体。
  • TupleTableSlot: クエリ実行エンジンがタプルを保持するために使用する構造体。

失敗モードと境界条件

  • result に NULL が渡された場合、areq->result にも NULL が格納されます。これは非同期処理が完了したものの、返すべきタプルが存在しない（または終了した）ケースを表現します。

保守担当者向けメモ

  • 本関数は非同期ノードの実装において、処理完了を通知する標準的な手段です。新しい非同期対応ノードを実装する際は、本関数または ExecAsyncRequestPending を適切に使用してください。
