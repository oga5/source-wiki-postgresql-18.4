StartupProcExit

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/startup.c:202

目的

起動プロセス終了時にリカバリ環境を安全にシャットダウンする。

シグネチャまたは形状

static void StartupProcExit(int code, Datum arg)

入力と出力

  • code: 終了コード（proc_exit から渡される）。
  • arg: 補助引数（on_shmem_exit 経由で登録される際に使用）。
  • 出力: なし（戻り値なし）。

前提条件と不変条件

  • on_shmem_exit を介して登録されるコールバック関数として設計されている。
  • standbyState 変数の状態に基づき、リカバリ環境のクリーンアップを行う。

アルゴリズムまたは振る舞い

1. standbyState が STANDBY_DISABLED でない（スタンバイモードまたはリカバリ中である）ことを確認する。 2. 条件を満たす場合、ShutdownRecoveryTransactionEnvironment() を呼び出し、リカバリに関連するトランザクション環境を終了させる。

呼び出し元と呼び出し先

  • 呼び出し元: on_shmem_exit に登録されたコールバックとして、プロセス終了時に呼び出される（src/backend/postmaster/startup.c:224 で登録）。
  • 呼び出し先: ShutdownRecoveryTransactionEnvironment()。

関連データ構造

  • standbyState: リカバリ環境の状態を保持するグローバル変数。

失敗モードと境界条件

  • standbyState が STANDBY_DISABLED の場合、本関数は何も行わず正常に終了する。

保守担当者向けメモ

  • 本関数は on_shmem_exit のコールバックとして登録されるため、直接呼び出すのではなく、プロセス終了処理の一部として実行されることを前提としている。
