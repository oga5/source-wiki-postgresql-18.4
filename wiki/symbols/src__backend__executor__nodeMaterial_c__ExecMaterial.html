ExecMaterial

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMaterial.c:38

目的

子ノード（サブプラン）から出力されるタプルを Tuplestore に一時保存し、必要に応じて再スキャンや逆方向スキャンを可能にするための実行関数。

シグネチャまたは形状

TupleTableSlot *ExecMaterial(PlanState *pstate)

入力と出力

  • **入力**: pstate (PlanState 構造体へのポインタ。実際には MaterialState としてキャストされる)
  • **出力**: 取得したタプルを格納した TupleTableSlot、またはタプルが存在しない場合は NULL。

前提条件と不変条件

  • CHECK_FOR_INTERRUPTS() が呼び出され、割り込みがチェックされる。
  • MaterialState 内の tuplestorestate が未初期化かつ eflags が設定されている場合、初回実行時に tuplestore_begin_heap を用いて初期化される。
  • eof_underlying フラグは、子ノードが既に終了したことを示し、無駄な再呼び出しを防ぐために使用される。

アルゴリズムまたは振る舞い

1. **初期化**: 初回実行時に Tuplestore を作成し、必要に応じてマーク用読み取りポインタを割り当てる。 2. **Tuplestore からの取得**: 順方向または逆方向の走査指示に従い、Tuplestore からタプルの取得を試みる。 3. **逆方向走査の補正**: 逆方向走査かつ Tuplestore が EOF の場合、tuplestore_advance を用いて最後に追加されたタプルの手前まで位置を調整する。 4. **子ノードからの取得**: Tuplestore から取得できない場合、ExecProcNode を呼び出して子ノードからタプルを取得し、Tuplestore に保存した上で上位ノードへ返す。 5. **終了処理**: 子ノードが NULL を返した場合は eof_underlying を真に設定し、以降の呼び出しを短絡させる。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上では明示的な呼び出し元は特定されていないが、matstate->ss.ps.ExecProcNode = ExecMaterial; の代入により、実行エンジンのノード処理フローから呼び出される。
  • **呼び出し先**:
  • ExecProcNode (子ノードの実行)
  • tuplestore_begin_heap / tuplestore_gettupleslot / tuplestore_puttupleslot / tuplestore_advance (Tuplestore 操作)
  • ExecCopySlot / ExecClearTuple (スロット操作)

関連データ構造

  • MaterialState: ノードの状態を保持する構造体。
  • Tuplestorestate: タプルを一時保存するメモリ/ディスク領域の管理構造体。
  • TupleTableSlot: タプルを保持する実行時のデータコンテナ。

失敗モードと境界条件

  • Tuplestore が空の場合、子ノードの終了状態に応じて NULL を返す。
  • work_mem を超えるデータ量はディスクへの書き出しが発生する可能性がある。
  • 子ノードが NULL を返した後の再呼び出しは、eof_underlying により即座に NULL を返す。

保守担当者向けメモ

  • Tuplestore のメモリ消費は work_mem に依存するため、パフォーマンスチューニングの際は注意が必要。
  • eflags により Tuplestore を作成しない最適化パスが存在するため、条件分岐のロジック変更時は注意すること。
