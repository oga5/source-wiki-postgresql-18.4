get_partition_qual_relid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/partcache.c:298

目的

指定されたリレーション（OID）がパーティションである場合に、そのパーティションが満たすべき制約式（qual）を構築して返します。

シグネチャまたは形状

Expr *get_partition_qual_relid(Oid relid)

入力と出力

  • **入力**: relid (Oid) - 対象となるリレーションの識別子。
  • **出力**: Expr * - パーティション制約を表す式。対象がパーティションでない場合、または制約が存在しない場合は NULL を返します。

前提条件と不変条件

  • get_rel_relispartition(relid) が真である場合のみ、制約の構築処理が実行されます。
  • 内部で relation_open を使用するため、呼び出し元は適切なトランザクションコンテキストにいる必要があります。

アルゴリズムまたは振る舞い

1. get_rel_relispartition(relid) を呼び出し、対象がパーティションか確認します。 2. パーティションであれば、relation_open(relid, AccessShareLock) でリレーションを開きます。 3. generate_partition_qual(rel) を呼び出し、パーティション境界から制約式のリスト（and_args）を取得します。 4. 取得したリストを以下のルールで単一の Expr に変換します。

  • リストが空（NIL）の場合: NULL を返します。
  • リストの要素が1つより多い場合: makeBoolExpr(AND_EXPR, and_args, -1) を使用して論理積式を作成します。
  • リストの要素が1つの場合: その式をそのまま返します。

    • 5. relation_close(rel, NoLock) を呼び出し、ロックを保持したままリレーションを閉じます。

呼び出し元と呼び出し先

  • src/backend/utils/adt/ruleutils.c:2104

  • **呼び出し先**:

関連データ構造

  • Expr: PostgreSQLの式ツリーを表す構造体。
  • Relation: リレーションキャッシュのエントリ。

失敗モードと境界条件

  • 対象の relid がパーティションでない場合、処理はスキップされ NULL が返されます。
  • generate_partition_qual が空リストを返した場合、結果は NULL となります。

保守担当者向けメモ

  • 本関数は AccessShareLock を取得したままリレーションを閉じる（NoLock 指定）ことで、呼び出し元が安全にデパース（deparsing）を行えるように設計されています。
  • generate_partition_qual は再帰的な処理を含むため、非常に深いパーティション階層ではスタック消費に注意が必要です。
