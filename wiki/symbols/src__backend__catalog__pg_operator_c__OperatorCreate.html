OperatorCreate

定義

  • **種別**: function
  • **定義箇所**: src/backend/catalog/pg_operator.c:320

目的

PostgreSQLのシステムカタログ pg_operator に新しい演算子を登録、または既存のシェル演算子を実体化する。

シグネチャまたは形状

ObjectAddress OperatorCreate(const char *operatorName,
                             Oid operatorNamespace,
                             Oid leftTypeId,
                             Oid rightTypeId,
                             Oid procedureId,
                             List *commutatorName,
                             List *negatorName,
                             Oid restrictionId,
                             Oid joinId,
                             bool canMerge,
                             bool canHash)

入力と出力

  • **入力**: 演算子名、名前空間、左右の型OID、実装関数OID、交換子/否定子の名前リスト、制限/結合選択関数OID、および canMerge/canHash フラグ。
  • **出力**: 作成または更新された演算子の ObjectAddress を返す。

前提条件と不変条件

  • 演算子名が validOperatorName により構文的に妥当であること。
  • OperatorValidateParams により、指定された型と属性の整合性が検証されていること。
  • 既存の演算子と重複する場合、または所有権がない場合はエラーとなる。
  • 演算子が自身の否定子（negator）になることは禁止されている。

アルゴリズムまたは振る舞い

1. **検証**: 名前とパラメータの妥当性を確認し、既存の演算子が存在しないかチェックする。 2. **シェル解決**: 交換子や否定子が指定されている場合、get_other_operator を呼び出し、存在しない場合はシェル演算子を作成してOIDを取得する。 3. **属性設定**: pg_operator カタログの各カラムに対応する値を設定する。 4. **カタログ更新**: pg_operator テーブルを RowExclusiveLock で開き、既存のシェルがあれば heap_modify_tuple で更新し、なければ新規挿入を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/operatorcmds.c 内の CREATE OPERATOR 関連処理。
  • **呼び出し先**: validOperatorName, get_func_rettype, OperatorValidateParams, OperatorGet, get_other_operator, object_ownercheck, table_open, SearchSysCacheCopy1, heap_modify_tuple, CatalogTupleUpdate。

関連データ構造

  • pg_operator (システムカタログ)
  • ObjectAddress (戻り値)
  • HeapTuple (カタログ操作用)

失敗モードと境界条件

  • 演算子名が無効な場合、ERRCODE_INVALID_NAME でエラー。
  • 既に同名の演算子が存在する場合、ERRCODE_DUPLICATE_FUNCTION でエラー。
  • 演算子の所有者でない場合、aclcheck_error でエラー。
  • 自身を否定子として指定した場合、ERRCODE_INVALID_FUNCTION_DEFINITION でエラー。

保守担当者向けメモ

  • 本関数は pg_operator の更新と挿入の両方を担う。
  • 依存関係の構築（makeOperatorDependencies）や相互参照のリンク更新（OperatorUpd）が後続処理として必要となる。
  • シェル演算子の概念により、相互参照する演算子の作成順序問題を解決している。
