PolyNumAggState

定義

  • 種別: typedef
  • 定義箇所: src/backend/utils/adt/numeric.c:5659

目的

集約関数における中間計算状態を保持するための型定義です。

シグネチャまたは形状

typedef Int128AggState PolyNumAggState;

Int128AggState と同一の構造を持ちます。

入力と出力

  • 入力: 集約関数の引数として PG_GETARG_POINTER を介してポインタとして受け取られます。
  • 出力: 集約状態のシリアライズや、計算結果の保持に使用されます。

前提条件と不変条件

  • Int128AggState の構造に依存します。
  • 集約関数内では、PG_ARGISNULL による NULL チェックを経てポインタとして取得されることが一般的です。

アルゴリズムまたは振る舞い

  • Int128AggState のエイリアスとして、集約計算中の状態管理を行います。
  • 集約関数のシリアライズ（bytea への変換）や、複数の状態の結合処理において、PolyNumAggState 型としてキャストして操作されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/numeric.c 内の集約関数実装（numeric_avg_accum 等の関連関数）。
  • 呼び出し先: Int128AggState を操作する関数群。
  • 補足: 具体的な関数呼び出しの連鎖については、提供されたソース excerpt には詳細なコールグラフが含まれていません。

関連データ構造

  • Int128AggState: 本シンボルの実体。
  • NumericAggState: 関連する集約状態構造体。

失敗モードと境界条件

  • NULL 値の入力に対しては、PG_ARGISNULL によるガードが必要です。
  • メモリ上のポインタとして扱われるため、無効なポインタの参照や型不整合によるメモリ破壊のリスクがあります。

保守担当者向けメモ

  • 本シンボルは Int128AggState の別名です。変更を加える際は、Int128AggState 側の定義および依存関係を確認してください。
  • 集約関数のシリアライズ処理（bytea 変換）に関連するコードパスで頻繁に使用されています。
