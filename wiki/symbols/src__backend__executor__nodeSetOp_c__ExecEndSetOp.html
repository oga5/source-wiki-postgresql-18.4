ExecEndSetOp

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSetOp.c:680

目的

SetOp ノードの実行を終了し、関連するメモリリソースを解放して子ノードの終了処理を行う。

シグネチャまたは形状

void ExecEndSetOp(SetOpState *node)

入力と出力

  • **入力**: SetOpState *node - 終了対象となる SetOp ノードの実行状態構造体。
  • **出力**: なし。

前提条件と不変条件

  • node は有効な SetOpState 構造体を指している必要がある。
  • ハッシュテーブルが構築されている場合、node->tableContext にメモリコンテキストが保持されている。

アルゴリズムまたは振る舞い

1. node->tableContext が存在する場合、MemoryContextDelete を呼び出してハッシュテーブルを含む付随的なメモリ領域を解放する。 2. outerPlanState(node) を引数として ExecEndNode を呼び出し、外部子ノードの終了処理を行う。 3. innerPlanState(node) を引数として ExecEndNode を呼び出し、内部子ノードの終了処理を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:742
  • **呼び出し先**:
  • MemoryContextDelete (メモリ管理)
  • ExecEndNode (実行エンジン共通関数: 外部および内部子ノードに対して実行)

関連データ構造

  • SetOpState: SetOp ノードの実行状態を保持する構造体。tableContext をメンバとして含む。

失敗モードと境界条件

  • node が NULL の場合、あるいは node 内のポインタが不正な場合は未定義の動作となる。

保守担当者向けメモ

  • 本関数は SetOp ノードのクリーンアップを担当する。UNION 演算には使用されない点に注意すること。
  • ハッシュテーブルのメモリ管理は tableContext に集約されているため、ここでの削除がメモリリーク防止の要となる。
