variable_coerce_param_hook

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_param.c:186

目的

クエリ解析中に、型が未定（UNKNOWNOID）である外部パラメータの型を、文脈に応じて決定・検証するフック関数です。

シグネチャまたは形状

static Node *variable_coerce_param_hook(ParseState *pstate, Param *param,
                                       Oid targetTypeId, int32 targetTypeMod,
                                       int location)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態。
  • param: 型解決対象の Param ノード。
  • targetTypeId: 推論されたターゲットの型 OID。
  • targetTypeMod: ターゲットの型修飾子（本関数では使用されず無視される）。
  • location: 解析位置。
  • **出力**:
  • 型解決が成功した場合、更新された Param ノードを返す。
  • 外部パラメータでない、または型が既に確定している場合は NULL を返し、通常の型変換処理へ委譲する。

前提条件と不変条件

  • param->paramkind が PARAM_EXTERN であること。
  • param->paramtype が UNKNOWNOID であること。
  • pstate->p_ref_hook_state に VarParamState が格納されていること。

アルゴリズムまたは振る舞い

1. パラメータが外部かつ型未定であるか確認し、該当しない場合は NULL を返して終了する。 2. VarParamState からパラメータ型配列を取得し、paramid が有効範囲内か検証する。 3. 該当パラメータの型が UNKNOWNOID であれば targetTypeId を割り当てる。既に型が設定されている場合は、targetTypeId との一致を確認し、不一致であればエラーを投げる。 4. param->paramtype を targetTypeId に更新する。 5. param->paramtypmod を -1 に設定する（実行時の型チェックを強制するため）。 6. param->paramcollid を get_typcollation によりデフォルトの照合順序に設定する。 7. 必要に応じて param->location を更新し、ノードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ParseState の p_coerce_param_hook フィールドに登録され、パーサーの型変換処理から呼び出される。
  • **呼び出し先**: get_typcollation（型に対応する照合順序の取得）、ereport（エラー報告）。

関連データ構造

  • ParseState: 解析状態を保持する構造体。
  • Param: SQLパラメータを表すノード。
  • VarParamState: 可変パラメータの型情報を管理する内部構造体。

失敗モードと境界条件

  • **ERRCODE_UNDEFINED_PARAMETER**: paramid が範囲外の場合。
  • **ERRCODE_AMBIGUOUS_PARAMETER**: 既に確定した型と targetTypeId が矛盾する場合。

保守担当者向けメモ

  • paramtypmod を -1 に固定しているのは、実行時の型チェックを確実に行うための設計上の意図です。
  • 本フックは「明示的な $n 形式のパラメータ」のみを対象とします。
  • 戻り値が NULL の場合、呼び出し側は通常の型変換ロジックを継続します。
