gistfillbuffer

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistutil.c:33

目的

GiSTインデックスページに対して、指定された複数のインデックスタプルを連続して追加するユーティリティ関数です。

シグネチャまたは形状

void gistfillbuffer(Page page, IndexTuple *itup, int len, OffsetNumber off)

入力と出力

  • **入力**:
  • page: 操作対象のGiSTインデックスページ。
  • itup: 追加する IndexTuple の配列。
  • len: 追加するタプルの数。
  • off: 挿入を開始するオフセット番号。InvalidOffsetNumber を指定した場合、ページ末尾または空ページの場合は先頭から挿入されます。
  • **出力**: なし（成功時はページが更新されます）。

前提条件と不変条件

  • itup 配列には少なくとも len 個の有効な IndexTuple が含まれている必要があります。
  • ページ内にタプルを格納するための十分な空き容量が必要です。

アルゴリズムまたは振る舞い

1. off が InvalidOffsetNumber の場合、ページが空であれば FirstOffsetNumber を、そうでなければ現在の最大オフセットの次を挿入位置として決定します。 2. 指定された len 分だけループし、各タプルに対して PageAddItem を呼び出します。 3. 挿入位置 off は各タプル追加後にインクリメントされます。 4. PageAddItem が失敗（InvalidOffsetNumber を返却）した場合、elog(ERROR) を発生させ処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gist.c
  • src/backend/access/gist/gistbuild.c
  • src/backend/access/gist/gistxlog.c
  • **呼び出し先**:
  • PageAddItem (PostgreSQLコアのページ操作関数)
  • elog (エラー報告)

関連データ構造

  • Page: インデックスページ構造体。
  • IndexTuple: GiSTインデックスに格納されるタプル形式。
  • OffsetNumber: ページ内のアイテム位置を示す識別子。

失敗モードと境界条件

  • **失敗**: ページ容量不足により PageAddItem が失敗した場合、elog(ERROR) によりトランザクションがアボートされます。
  • **境界条件**: off に InvalidOffsetNumber を渡すことで、自動的に適切な挿入位置を計算する挙動が定義されています。

保守担当者向けメモ

  • 本関数はGiSTインデックスの構築やWALリカバリ時に頻繁に使用される低レベル関数です。
  • 挿入時のエラーは致命的（ERROR）として扱われるため、呼び出し側で事前に gistnospace 等を用いて十分な空き容量があるか確認することが推奨されます。
