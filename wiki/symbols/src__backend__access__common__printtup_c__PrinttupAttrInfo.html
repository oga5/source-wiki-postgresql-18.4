PrinttupAttrInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/access/common/printtup.c:45

目的

クエリ結果の各カラム（属性）に対して、出力関数や型情報をキャッシュし、タプル送信時の変換処理を効率化するために使用される構造体です。

シグネチャまたは形状

typedef struct {
    Oid         typoutput;      /* テキスト出力関数のOID */
    Oid         typsend;        /* バイナリ出力関数のOID */
    bool        typisvarlena;   /* 可変長型（TOAST可能）か否か */
    int16       format;         /* カラムのフォーマットコード */
    FmgrInfo    finfo;          /* 出力関数の事前計算済み呼び出し情報 */
} PrinttupAttrInfo;

入力と出力

本構造体自体はデータ保持用です。printtup_prepare_info 関数によって初期化され、printtup 関数内で各属性のシリアライズ処理の入力として参照されます。

前提条件と不変条件

  • タプル記述子（TupleDesc）が変更されない限り、キャッシュされた情報は再利用可能です。
  • finfo は、typoutput または typsend に基づいて事前に計算されている必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: printtup_create_DR 等の過程で、属性数分だけ PrinttupAttrInfo の配列が palloc0 で確保されます。 2. **キャッシュ**: printtup_prepare_info が各属性の型情報を検索し、対応する出力関数を finfo に格納します。 3. **利用**: printtup 実行時、各属性の変換において finfo を使用して関数呼び出しのオーバーヘッドを削減します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/printtup.c 内の printtup_prepare_info（初期化）および printtup（参照）。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接関数を呼び出すことはありませんが、内部の finfo が関数マネージャを介して出力関数を起動します。

関連データ構造

  • DR_printtup: 本構造体の配列を myinfo メンバとして保持する、DestReceiver の実装構造体。

失敗モードと境界条件

  • 属性数が 0 の場合、メモリ確保は行われません。
  • 型情報の検索に失敗した場合、finfo が正しく初期化されず、後続のタプル送信時にエラーが発生する可能性があります。

保守担当者向けメモ

  • PrinttupAttrInfo は行ごとのメモリコンテキスト（tmpcontext）とは独立して保持されます。タプル記述子が変更された場合は、キャッシュの再構築が必要です。
  • finfo の更新や変更を行う際は、fmgr インターフェースの仕様に従ってください。
