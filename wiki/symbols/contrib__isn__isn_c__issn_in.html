issn_in

定義

  • 種別: function
  • 定義箇所: contrib/isn/isn.c:1030

目的

入力された文字列形式のISSNを解析し、PostgreSQL内部の ean13 型へ変換する。

シグネチャまたは形状

Datum issn_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して受け取る文字列（ISSN）。
  • **出力**: ean13 型のデータ（Datum 形式）。解析失敗時は NULL を返す。

前提条件と不変条件

  • string2ean 関数がISSNの検証および変換のロジックを担う。
  • ean13 型の最下位ビットは、チェックデジットの有効性フラグとして予約されている。

アルゴリズムまたは振る舞い

1. PG_GETARG_CSTRING(0) で入力文字列を取得する。 2. string2ean 関数を呼び出し、入力文字列を ISSN 型として解析・検証する。 3. 解析に成功した場合、PG_RETURN_EAN13(result) を通じて変換後の ean13 値を返す。 4. 解析に失敗した場合、PG_RETURN_NULL() を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: string2ean（contrib/isn/isn.c 内）。

関連データ構造

  • ean13: 内部ストレージ形式（uint64）。
  • ISSN: string2ean に渡される識別子タイプ。

失敗モードと境界条件

  • 入力文字列がISSNの形式やチェックデジットの要件を満たさない場合、string2ean が false を返し、本関数は NULL を返す。

保守担当者向けメモ

  • 本関数は contrib/isn/ISSN.h で定義された範囲テーブルを利用して検証を行う。
  • チェックデジットの検証緩和（isn.weak）は string2ean 内部で処理される。
