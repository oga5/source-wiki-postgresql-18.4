md_readv_complete

定義

  • **種別**: function
  • **定義箇所**: src/backend/storage/smgr/md.c:1978

目的

非同期 I/O 操作（mdreadv）の完了時に呼び出され、OS レベルのバイト単位の読み込み結果をブロック単位に変換し、エラー判定および状態の正規化を行う。

シグネチャまたは形状

static PgAioResult md_readv_complete(PgAioHandle *ioh, PgAioResult prior_result, uint8 cb_data)

入力と出力

  • **入力**:
  • ioh: 非同期 I/O ハンドル。
  • prior_result: 直前の I/O 操作の結果（バイト数やエラーコードを含む）。
  • cb_data: コールバックデータ。
  • **出力**:
  • PgAioResult: 変換・判定後の結果。result フィールドには読み込まれたブロック数が格納される。

前提条件と不変条件

  • prior_result.result はバイト単位の読み込み量であること。
  • 読み込み後のブロック数（result.result / BLCKSZ）は、対象リレーションの総ブロック数（td->smgr.nblocks）を超えないこと（Assert による検証）。

アルゴリズムまたは振る舞い

1. **エラー判定**: prior_result.result が負の場合、エラーとして扱い、PGAIO_RS_ERROR を設定してサーバーログに記録し、結果を返す。 2. **単位変換**: バイト単位の読み込み結果を BLCKSZ で除算し、ブロック数に変換する。 3. **ゼロ読み込みの処理**: ブロック数が 0 の場合はエラーとみなし、ログ記録後に PGAIO_RS_ERROR を返す。 4. **部分読み込みの判定**: 読み込みブロック数が期待値（nblocks）より少ない場合、PGAIO_RS_PARTIAL ステータスを設定する。 5. **正常終了**: 変換されたブロック数を含む PgAioResult を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: 非同期 I/O サブシステム（PgAioHandle の完了コールバックとして登録）。
  • **呼び出し先**: pgaio_io_get_target_data（ターゲットデータ取得）、pgaio_result_report（ログ出力）。

関連データ構造

  • PgAioHandle: 非同期 I/O の状態管理構造体。
  • PgAioResult: I/O 操作の結果を保持する構造体。
  • PgAioTargetData: I/O 対象のメタデータ（ブロック数など）。

失敗モードと境界条件

  • **ハードエラー**: prior_result.result < 0 の場合、エラーとして処理される。
  • **ゼロ読み込み**: 読み込みブロック数が 0 の場合、エラーとして処理される。
  • **部分読み込み**: 要求されたブロック数に満たない場合、PGAIO_RS_PARTIAL としてマークされ、上位層での再試行を促す。

保守担当者向けメモ

  • 本関数は smgr API のブロック単位の抽象化を維持するための重要な変換層である。
  • ログ出力は LOG_SERVER_ONLY を使用しており、呼び出し元がエラーを処理する前に即座に情報を残す設計となっている。
