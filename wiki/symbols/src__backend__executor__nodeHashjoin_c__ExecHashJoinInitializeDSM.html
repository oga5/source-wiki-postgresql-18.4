ExecHashJoinInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHashjoin.c:1655

目的

並列ハッシュ結合（Parallel Hash Join）を実行するために必要な共有メモリ領域（DSM）の初期化およびデータ構造のセットアップを行う。

シグネチャまたは形状

void ExecHashJoinInitializeDSM(HashJoinState *state, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • state: 実行中のハッシュ結合ノードの状態（HashJoinState）。
  • pcxt: 並列実行コンテキスト（ParallelContext）。
  • **出力**: なし（副作用として state および共有メモリ上の ParallelHashJoinState を初期化する）。

前提条件と不変条件

  • pcxt->seg が NULL の場合、共有メモリセグメントが確保できていないため、本関数は何もせず即座にリターンする。
  • 成功時には、実行ノードの処理関数が ExecParallelHashJoin に設定される。

アルゴリズムまたは振る舞い

1. **実行プロシージャの切り替え**: ExecSetExecProcNode を呼び出し、実行ノードの処理を ExecParallelHashJoin に設定する。 2. **共有状態の確保**: shm_toc_allocate を使用して ParallelHashJoinState を共有メモリ上に確保し、shm_toc_insert でプランノードIDをキーとして登録する。 3. **初期値の設定**:

  • バッチ数、バケット数、タプル数などを初期化。
  • LWLockInitialize で並列結合用のロックを初期化。
  • BarrierInit を使用して、ビルド、バッチ拡張、バケット拡張用の各バリアを初期化。

    • 4. **ファイルセットの初期化**: SharedFileSetInit を呼び出し、共有一時ファイル用の領域を準備する。 5. **ハッシュノードへの紐付け**: innerPlanState(state) から取得した HashState の parallel_state に、作成した pstate を格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:512 からの呼び出しが確認されている。
  • **呼び出し先**: ExecSetExecProcNode, shm_toc_allocate, shm_toc_insert, pg_atomic_init_u32, LWLockInitialize, BarrierInit, SharedFileSetInit 等の共有メモリ管理および同期プリミティブ関数。

関連データ構造

  • HashJoinState: 結合ノードの実行状態。
  • ParallelHashJoinState: 並列結合の共有状態（バリア、ロック、バッチ管理情報を含む）。
  • ParallelContext: 並列実行のコンテキスト。
  • HashState: 内側リレーションのハッシュノード。

失敗モードと境界条件

  • pcxt->seg が NULL の場合、共有メモリが利用できないため、並列処理のセットアップをスキップして終了する。

保守担当者向けメモ

  • 本関数は並列ハッシュ結合の基盤となる共有メモリ構造を構築する。デバッグ時には ParallelHashJoinState の各フィールド（特にバリアの状態）が正しく初期化されているかを確認すること。
  • ExecParallelHashJoin への切り替えが必須であるため、実行フローの変更時には注意が必要。
