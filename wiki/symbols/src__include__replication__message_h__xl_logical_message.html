xl_logical_message

定義

  • 種別: struct
  • 定義箇所: src/include/replication/message.h:20

目的

論理デコーディングにおいて、ユーザー定義の任意のバイナリデータをWAL（Write-Ahead Log）に記録し、デコーディングプラグインへ伝達するためのヘッダー構造体です。

シグネチャまたは形状

typedef struct xl_logical_message
{
    Oid         dbId;           /* データベースOID */
    bool        transactional;  /* トランザクション属性の有無 */
    Size        prefix_size;    /* プレフィックスの長さ */
    Size        message_size;   /* メッセージのサイズ */
    char        message[FLEXIBLE_ARRAY_MEMBER]; /* ペイロード（プレフィックスを含む） */
} xl_logical_message;

入力と出力

  • **入力**: LogLogicalMessage 関数を通じて、データベースID、トランザクションフラグ、プレフィックス、およびメッセージ本体が本構造体に格納されます。
  • **出力**: WALレコードの一部として永続化され、論理デコーディング処理時にデコーディングプラグインへ渡されます。

前提条件と不変条件

  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体の固定サイズ計算には必ず SizeOfLogicalMessage マクロを使用する必要があります。
  • メッセージには、名前空間として機能するプレフィックスを付与することが推奨されます。

アルゴリズムまたは振る舞い

  • **トランザクション管理**: transactional が真の場合、トランザクションのコミット時に出力されます。偽の場合はWAL読み取り時に即座に出力されます。
  • **Redo処理**: リカバリ時には logicalmsg_redo が呼び出されますが、論理デコーディング専用データであるため、物理的な復旧は行わず「NOOP（何もしない）」として動作します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照）**:
  • src/backend/replication/logical/message.c: xlrec の構築に使用。
  • src/backend/replication/logical/decode.c: WALレコードの読み取り時にキャストして使用。
  • src/backend/access/rmgrdesc/logicalmsgdesc.c: レコードの記述に使用。
  • **呼び出し先**: 情報なし。

関連データ構造

  • SizeOfLogicalMessage: 構造体の固定サイズを計算するマクロ。
  • XLOG_LOGICAL_MESSAGE: 論理メッセージを示すWALレコード識別子。

失敗モードと境界条件

  • 可変長データを含むため、メモリレイアウトを直接操作する際は SizeOfLogicalMessage を使用しない場合、バッファオーバーランや不正なメモリ参照のリスクがあります。

保守担当者向けメモ

  • 本構造体は論理デコーディングのインターフェースの一部です。変更を加える際は src/backend/replication/logical/message.c および src/backend/replication/logical/decode.c への影響を確認してください。
