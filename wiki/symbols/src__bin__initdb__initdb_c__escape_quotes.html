escape_quotes

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/initdb.c:405

目的

入力文字列内のシングルクォートをエスケープし、SQL文や設定ファイル内で安全に使用可能な文字列を生成する。

シグネチャまたは形状

char *escape_quotes(const char *src);

入力と出力

  • **入力**: src (エスケープ対象の文字列へのポインタ)
  • **出力**: エスケープ処理済みの新しい文字列（呼び出し元で解放が必要）。メモリ不足時は pg_fatal によりプログラムを終了する。

前提条件と不変条件

  • 入力 src はヌル終端文字列であること。
  • 内部で escape_single_quotes_ascii を呼び出すため、ASCIIベースのエスケープルールに従う。

アルゴリズムまたは振る舞い

1. escape_single_quotes_ascii(src) を呼び出し、エスケープされた文字列を生成する。 2. 結果が NULL の場合（メモリ確保失敗）、pg_fatal を呼び出してエラーメッセージを出力し、プロセスを終了する。 3. 生成された文字列へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: initdb.c 内の各所（設定値の置換、ユーザー名やパスワードの処理など）。
  • **呼び出し先**: escape_single_quotes_ascii（fe_utils/string_utils.h で定義される関数）。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • **メモリ不足**: escape_single_quotes_ascii が NULL を返した場合、pg_fatal により即座に終了する。

保守担当者向けメモ

  • 本関数は src/bin/initdb/initdb.c 内で定義された静的関数ですが、同名の関数が src/bin/scripts/vacuumdb.c や src/fe_utils/recovery_gen.c にも存在します。修正時は影響範囲に注意してください。
  • 文字列のメモリ管理は呼び出し元に委ねられているため、メモリリークが発生しないよう注意が必要です。
