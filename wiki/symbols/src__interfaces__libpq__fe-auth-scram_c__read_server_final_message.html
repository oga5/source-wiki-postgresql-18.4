read_server_final_message

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth-scram.c:692

目的

SCRAM 認証プロトコルの最終ステップにおいて、サーバーから送信されたメッセージを解析し、サーバー署名を検証する。

シグネチャまたは形状

static bool read_server_final_message(fe_scram_state *state, char *input);

入力と出力

  • **入力**:
  • state: 現在の SCRAM 認証状態を保持する fe_scram_state 構造体へのポインタ。
  • input: サーバーから受信した SCRAM 最終メッセージ文字列（破壊的に解析される）。
  • **出力**:
  • bool: 成功した場合は true、エラー（メモリ不足、メッセージ形式不正、サーバーからのエラー通知）が発生した場合は false を返す。

前提条件と不変条件

  • state は初期化済みであること。
  • input は read_attr_value による解析が可能な形式であること。
  • 成功時、state->server_final_message に入力文字列のコピーが格納される。

アルゴリズムまたは振る舞い

1. input を state->server_final_message に複製する。 2. メッセージの先頭が 'e' で始まる場合、サーバーからのエラー通知とみなし、read_attr_value でエラー内容を取得して conn->errorMessage に記録し終了する。 3. 属性 'v'（サーバー署名）を解析し、Base64 デコードを行う。 4. デコード後の署名長が state->key_length と一致することを確認する。 5. 一致する場合、署名を state->ServerSignature にコピーする。 6. メッセージ末尾に余分なデータが存在しないことを確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-auth-scram.c 内の認証フロー（scram_exchange 等）から呼び出されることが示唆されている。
  • **呼び出し先**: read_attr_value（属性値抽出）、libpq_append_conn_error（エラー記録）、pg_b64_dec_len/pg_b64_decode（Base64 変換）。

関連データ構造

  • fe_scram_state: 認証セッションの状態と署名を保持する。
  • PGconn: エラーメッセージを格納する接続オブジェクト。

失敗モードと境界条件

  • **メモリ不足**: strdup や malloc の失敗時に false を返す。
  • **サーバーエラー**: メッセージが 'e' で始まる場合、認証失敗として処理する。
  • **形式不正**: 属性 'v' が見つからない、署名長が不一致、またはメッセージ末尾に不正なデータがある場合に false を返す。

保守担当者向けメモ

  • input 引数は read_attr_value によって内部的にポインタが移動・変更されるため、呼び出し後の再利用には注意が必要。
  • サーバー署名の検証ロジックは、この関数で署名を抽出した後に別途実行される。
