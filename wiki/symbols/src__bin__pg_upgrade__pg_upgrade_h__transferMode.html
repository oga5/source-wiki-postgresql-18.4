transferMode

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.h:259

目的

PostgreSQL のアップグレード処理において、旧クラスタから新クラスタへデータを移行する際の物理的なファイル転送方式を規定するために使用されます。

シグネチャまたは形状

typedef enum
{
	TRANSFER_MODE_CLONE,
	TRANSFER_MODE_COPY,
	TRANSFER_MODE_COPY_FILE_RANGE,
	TRANSFER_MODE_LINK,
	TRANSFER_MODE_SWAP,
} transferMode;

入力と出力

  • 入力: 該当なし（列挙型定義）
  • 出力: 以下のいずれかの定数値を保持します。
  • TRANSFER_MODE_CLONE: ファイルのクローン（Reflink）を使用。
  • TRANSFER_MODE_COPY: ファイルのコピーを使用。
  • TRANSFER_MODE_COPY_FILE_RANGE: copy_file_range システムコールを使用。
  • TRANSFER_MODE_LINK: ハードリンクを使用。
  • TRANSFER_MODE_SWAP: ファイルの入れ替え（スワップ）を使用。

前提条件と不変条件

  • pg_upgrade の実行時オプションまたは内部状態として、いずれか一つのモードが選択される必要があります。
  • ファイル操作を行う各関数は、この列挙値に基づいて適切なシステムコールを選択します。

アルゴリズムまたは振る舞い

本シンボルは、file.c 等のモジュールにおいて、実際のデータ移行ロジックを分岐させるための制御フラグとして機能します。例えば、ハードリンクが選択された場合、check_hard_link 関数を通じてリンクの可否が検証されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/bin/pg_upgrade/controldata.c: disable_old_cluster 関数
  • src/bin/pg_upgrade/file.c: check_hard_link 関数
  • src/bin/pg_upgrade/pg_upgrade.h: 構造体メンバ定義および関数プロトタイプ宣言
  • 呼び出し先:
  • 本シンボル自体は列挙型であり、直接的な呼び出し先はありません。

関連データ構造

  • ClusterInfo: アップグレード対象のクラスタ情報を保持する構造体であり、メンバとして transferMode transfer_mode を含みます。

失敗モードと境界条件

  • 不適切なモードが選択された場合、file.c 内のファイル操作関数がエラーを返す可能性があります。
  • 特定のファイルシステムがハードリンクやクローンをサポートしていない場合、選択したモードによっては移行処理が失敗します。

保守担当者向けメモ

  • 新しい転送方式を追加する場合は、本列挙型に値を追加し、file.c 内の各操作関数で switch 文等の分岐処理を更新してください。
  • pg_upgrade.h 内の transfer_mode メンバが正しく初期化されていることを確認してください。
