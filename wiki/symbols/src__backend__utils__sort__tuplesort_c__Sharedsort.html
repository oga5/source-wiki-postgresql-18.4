Sharedsort

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/sort/tuplesort.c:343

目的

並列ソート処理において、リーダープロセスとワーカープロセス間でソート状態や一時ファイル情報を共有・調整するための構造体です。

シグネチャまたは形状

struct Sharedsort {
    slock_t           mutex;
    int               currentWorker;
    int               workersFinished;
    SharedFileSet     fileset;
    int               nTapes;
    TapeShare         tapes[FLEXIBLE_ARRAY_MEMBER];
};

入力と出力

  • **入力**: 並列ソート実行時に共有メモリ領域（shm_toc_allocate）から割り当てられます。
  • **出力**: ワーカープロセスが tapes 配列を通じて自身のソート結果（テープ情報）をリーダーに報告し、リーダーがそれらを統合してマージ処理を行います。

前提条件と不変条件

  • mutex は tapes 配列より前の全フィールドを保護します。
  • currentWorker は 0 から始まる隙間のない連番として生成されます。
  • workersFinished がリーダー側の state.nParticipants と一致したとき、リーダーはマージを開始する準備が整ったと判断します。

アルゴリズムまたは振る舞い

1. **ワーカーの識別**: currentWorker を用いて各ワーカーに一意の ID を割り当てます。 2. **進捗報告**: 各ワーカーは処理完了時に workersFinished をインクリメントします。 3. **テープの統合**: リーダーは tapes 配列を参照し、各ワーカーが作成した一時ファイル（SharedFileSet）を連結して最終的なマージを実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin.c および src/backend/access/gin/gininsert.c において、並列ビルド時の共有メモリ管理として利用されています。
  • **呼び出し先**: 共有メモリ管理機能（shm_toc_allocate）および SharedFileSet を介した一時ファイル管理機能と連携します。

関連データ構造

  • SharedFileSet: 並列処理で共有される一時ファイル群を管理します。
  • TapeShare: 各ワーカーのテープ情報を格納する構造体です。

失敗モードと境界条件

  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズは nTapes に依存して動的に決定されます。
  • 共有メモリの割り当て失敗や、ワーカープロセスが正常に完了報告を行わない場合、リーダープロセスがマージフェーズへ移行できず待機状態となる可能性があります。

保守担当者向けメモ

  • 本構造体は tuplesort.c の並列ソートロジックの中核です。並列処理のデバッグ時には、mutex による排他制御と workersFinished のカウントアップが正しく行われているかを確認してください。
  • brin.c や gininsert.c が主な利用者ですが、他のインデックスアクセス手法で並列ソートを導入する際のテンプレートとしても機能します。
