cidr_out

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network.c:172

目的

cidr 型の内部表現を、人間が読み取り可能な文字列形式（CSTRING）に変換します。

シグネチャまたは形状

Datum cidr_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_INET_PP(0) を介して inet * 型のポインタを受け取ります。
  • **出力**: PG_RETURN_CSTRING を介して、ネットワークアドレスの文字列表現を返します。

前提条件と不変条件

  • 入力される inet 構造体は、cidr 型の不変条件（マスク長より右側のビットが 0 であること）を満たしている必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_INET_PP を使用して、入力された Datum から inet 構造体へのポインタを取得します。 2. 共通関数 network_out を呼び出します。その際、第2引数に true を渡すことで、cidr 型としての出力形式（マスク長を含む形式）を指定します。 3. network_out が生成した文字列を呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: network_out
  • **呼び出し元**: 提供されたソースコード内には、本関数を直接呼び出している箇所に関する情報はありません。

関連データ構造

  • inet: PostgreSQL のネットワーク型（inet および cidr）を保持する内部構造体。

失敗モードと境界条件

  • 入力ポインタが不正な場合、PG_GETARG_INET_PP マクロの挙動によりエラーが発生する可能性があります。
  • network_out の内部処理に依存し、メモリ不足等のシステムエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は cidr 型専用のラッパーであり、実質的な変換ロジックは network_out に集約されています。
  • cidr 型の出力形式を変更する必要がある場合は、network_out 側の実装を確認してください。
