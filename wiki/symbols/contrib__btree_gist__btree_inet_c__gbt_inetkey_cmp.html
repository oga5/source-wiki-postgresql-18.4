gbt_inetkey_cmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_inet.c:54

目的

GiST インデックス構築時における inetKEY 構造体同士の比較を行い、ソート順序を決定する。

シグネチャまたは形状

static int gbt_inetkey_cmp(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の Nsrt 構造体へのポインタ（内部で inetKEY にキャストされる）。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • 0: a と b が等しい場合。
  • 1: a が b より大きい場合。
  • -1: a が b より小さい場合。

前提条件と不変条件

  • 入力ポインタ a および b は、Nsrt 構造体を経由して inetKEY 構造体へ安全にキャスト可能であること。
  • inetKEY 構造体は lower および upper フィールドを持つこと。

アルゴリズムまたは振る舞い

1. a と b からそれぞれ inetKEY 構造体を取り出す。 2. まず lower 値を比較する。

  • ia->lower != ib->lower の場合、その大小関係を返す。

    • 3. lower 値が等しい場合、upper 値を比較する。

  • ia->upper == ib->upper ならば 0 を返す。
  • ia->upper > ib->upper ならば 1 を返し、そうでなければ -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は特定されていませんが、btree_gist のソートサポート（sortsupport）に関連する構造体初期化の一部として参照されています。
  • **呼び出し先**: 本関数は標準的な比較演算のみを行い、他の関数を呼び出しません。

関連データ構造

  • inetKEY: インデックス内で保持される lower と upper の境界値を持つ構造体。
  • Nsrt: ソート処理中に使用される中間データ構造。

失敗モードと境界条件

  • 入力ポインタが NULL の場合の挙動は定義されていません。
  • inetKEY 内の lower および upper は double 型として比較されるため、浮動小数点数の精度に依存します。

保守担当者向けメモ

  • 本関数は inet 型のソート順を定義する重要なロジックです。変更時は inetKEY の定義および btree_utils_num.c で提供される数値比較ロジックとの整合性に注意してください。
