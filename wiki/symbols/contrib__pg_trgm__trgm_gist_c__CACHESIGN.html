CACHESIGN

定義

  • 種別: struct
  • 定義箇所: contrib/pg_trgm/trgm_gist.c:745

目的

GiST インデックスの検索や距離計算処理において、インデックスキーのシグネチャ（ビットベクトル）をキャッシュし、計算コストを削減するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	bool		allistrue;
	BITVECP		sign;
} CACHESIGN;

入力と出力

  • allistrue (bool): すべてのビットがセットされている状態を示すフラグ。
  • sign (BITVECP): シグネチャを保持するビットベクトルへのポインタ。

前提条件と不変条件

  • sign は有効なビットベクトルを指している必要があります。
  • allistrue が true の場合、sign の内容は全ビットがセットされているものとして扱われます。

アルゴリズムまたは振る舞い

  • fillcache: インデックスキーから CACHESIGN 構造体へ情報をコピーし、キャッシュを構築します。
  • hemdistcache: 2つの CACHESIGN 構造体間のハミング距離を計算します。
  • GiST インデックスの検索時に、ノードごとのシグネチャをメモリ上に保持することで、繰り返し発生するビット演算のオーバーヘッドを抑制します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pg_trgm/trgm_gist.c 内の fillcache、hemdistcache、およびキャッシュ配列の割り当て処理。
  • 呼び出し先: BITVECP 型に関連するビット操作関数（port/pg_bitutils.h 等）。
  • 注記: src/backend/utils/adt/tsgistidx.c にも同名の構造体定義が存在しますが、本ページは contrib/pg_trgm/trgm_gist.c の定義を対象としています。

関連データ構造

  • BITVECP: シグネチャを表現するためのビットベクトル型。
  • gtrgm_consistent_cache: 本構造体を利用して検索時の状態を保持する上位のキャッシュ構造体。

失敗モードと境界条件

  • メモリ割り当て失敗: palloc を使用しているため、メモリ不足時には PostgreSQL のエラーハンドリングによりアボートします。
  • siglen の不一致: fillcache や hemdistcache に渡される siglen が、実際の sign のサイズと一致しない場合、メモリ破壊や不正な計算結果を招く可能性があります。

保守担当者向けメモ

  • 本構造体は tsgistidx.c と定義が重複しています。共通化の検討余地がありますが、現状は各モジュールで独立して管理されています。
  • sign フィールドのメモリ管理は呼び出し元に依存するため、キャッシュ配列の生存期間と palloc のライフサイクルに注意してください。
