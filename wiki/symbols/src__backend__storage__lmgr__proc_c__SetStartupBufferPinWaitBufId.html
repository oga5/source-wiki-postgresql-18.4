SetStartupBufferPinWaitBufId

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/proc.c:753

目的

共有メモリ上のグローバルなプロセス管理ヘッダー（ProcGlobal）に、スタートアッププロセスが現在待機しているバッファIDを記録します。

シグネチャまたは形状

void SetStartupBufferPinWaitBufId(int bufid)

入力と出力

  • **入力**: bufid (int型) - 待機対象のバッファID。待機を解除する場合は -1 が指定されます。
  • **出力**: なし。

前提条件と不変条件

  • ProcGlobal が初期化済みである必要があります。
  • コンパイラによるコードの並べ替えを防ぐため、volatile 修飾子を介して ProcGlobal にアクセスします。

アルゴリズムまたは振る舞い

1. グローバル変数 ProcGlobal を volatile ポインタとして取得します。 2. ProcGlobal が保持する startupBufferPinWaitBufId フィールドに、引数で渡された bufid を代入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c:5789 (バッファ待機開始時)
  • src/backend/storage/buffer/bufmgr.c:5793 (バッファ待機終了時)
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • PROC_HDR: ProcGlobal が指す構造体。startupBufferPinWaitBufId フィールドを保持します。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。
  • bufid に -1 を渡すことで、待機状態のクリアを表現する設計となっています。

保守担当者向けメモ

  • 本関数はスタートアッププロセスがバッファピンを待機する際の追跡に使用されます。
  • volatile 修飾子の使用は、最適化によるメモリ書き込み順序の変更を抑制するための重要な実装上の工夫です。
