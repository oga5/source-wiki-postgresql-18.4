tbm_get_pageentry

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/tidbitmap.c:1205

目的

指定されたブロック番号（pageno）に対応する PagetableEntry を取得、または存在しない場合は新規作成して返します。

シグネチャまたは形状

static PagetableEntry *tbm_get_pageentry(TIDBitmap *tbm, BlockNumber pageno)

入力と出力

  • **入力**:
  • tbm: 操作対象の TIDBitmap 構造体へのポインタ。
  • pageno: 取得対象のブロック番号。
  • **出力**:
  • 対象ブロックに対応する PagetableEntry 構造体へのポインタ。

前提条件と不変条件

  • TIDBitmap の状態（status）に応じて、単一ページ保持モードからハッシュテーブル管理モードへの移行が自動的に行われます。
  • 新規作成されたエントリはゼロクリアされ、blockno が設定されます。

アルゴリズムまたは振る舞い

1. **空の状態 (TBM_EMPTY)**: 固定スロット (entry1) を使用し、状態を TBM_ONE_PAGE に更新します。 2. **単一ページ保持状態 (TBM_ONE_PAGE)**:

  • 指定された pageno が既存の entry1 と一致すればそれを返します。
  • 一致しない場合、tbm_create_pagetable を呼び出してハッシュテーブル管理へ移行します。

    • 3. **ハッシュテーブル管理状態**: pagetable_insert を使用してエントリを検索または作成します。 4. **新規作成時**: エントリが新規作成された場合、TIDBitmap の nentries および npages カウンタをインクリメントし、エントリを初期化して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/tidbitmap.c 内の複数箇所（TID 追加処理やビットマップ結合処理など）。
  • **呼び出し先**:
  • tbm_create_pagetable
  • pagetable_insert
  • MemSet

関連データ構造

  • TIDBitmap: ビットマップ全体を管理する構造体。
  • PagetableEntry: ページ単位の情報を保持するハッシュテーブルのエントリ。

失敗モードと境界条件

  • メモリ不足時には pagetable_insert 経由でエラーが発生する可能性があります。
  • TBM_ONE_PAGE からハッシュテーブルへの移行時に、既存の entry1 の内容がハッシュテーブルへ統合される挙動を含みます。

保守担当者向けメモ

  • 本関数は TIDBitmap の内部状態遷移を制御する重要な役割を担っています。
  • TBM_ONE_PAGE からの移行ロジックを変更する場合は、PagetableEntry の初期化順序に注意してください。
