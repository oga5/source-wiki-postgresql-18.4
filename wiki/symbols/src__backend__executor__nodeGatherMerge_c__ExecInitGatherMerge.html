ExecInitGatherMerge

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGatherMerge.c:66

目的

GatherMerge プランノードを実行するための GatherMergeState 構造体を初期化し、並列クエリ実行環境におけるタプルマージ処理の準備を行う。

シグネチャまたは形状

GatherMergeState *ExecInitGatherMerge(GatherMerge *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の GatherMerge プランノード。
  • estate: 実行状態を保持する EState。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの GatherMergeState 構造体へのポインタ。

前提条件と不変条件

  • innerPlan(node) は NULL であること（GatherMerge は内側プランを持たない）。
  • node->plan.qual は存在しないこと（フィルタリングは子ノードで行うべきであるため）。
  • outerPlan(node) が存在し、ExecInitNode によって正常に初期化可能であること。

アルゴリズムまたは振る舞い

1. GatherMergeState を割り当て、ps.ExecProcNode に ExecGatherMerge を設定する。 2. ExecAssignExprContext で式コンテキストを作成する。 3. ExecInitNode を呼び出し、子ノード（outerPlan）を初期化する。 4. outeropsset を true に、outeropsfixed を false に設定し、スロット型の動的な変化に対応する。 5. 子ノードのタプル記述子（tupDesc）を取得し、結果型と射影情報を初期化する。 6. node->numCols が存在する場合、ソートキー情報を SortSupportData 配列として構築し、PrepareSortSupportFromOrderingOp で比較演算子を準備する。 7. gather_merge_setup(gm_state) を呼び出し、マージに必要なワークスペース（ヒープやバッファ）を確保する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c (ExecInitNode 経由)
  • **呼び出し先**:
  • ExecInitNode
  • ExecAssignExprContext
  • ExecGetResultType
  • ExecInitResultTypeTL
  • ExecConditionalAssignProjectionInfo
  • PrepareSortSupportFromOrderingOp
  • gather_merge_setup

関連データ構造

  • GatherMergeState: 本ノードの実行状態を保持する主要構造体。
  • GatherMerge: プランノード。
  • SortSupportData: ソートキーの比較ロジックを保持する構造体。
  • PlanState: ps メンバとして内包される基底構造体。

失敗モードと境界条件

  • 子ノードの初期化失敗時にエラーが発生する可能性がある。
  • ソートキーの準備において、指定された演算子が見つからない場合はエラーとなる。
  • メモリ不足時に palloc0 等でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は MergeAppend と類似したソートキー設定を行うが、並列実行環境特有の gather_merge_setup によるワークスペース確保が必要である。
  • ps.outeropsfixed = false としているのは、リーダープロセスとワーカープロセスでスロットの性質が異なる可能性があるためである。変更時はこの制約に注意すること。
