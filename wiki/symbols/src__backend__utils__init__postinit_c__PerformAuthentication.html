PerformAuthentication

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/postinit.c:193

目的

クライアント接続の認証プロセスを実行し、セッションの初期化を制御する。

シグネチャまたは形状

static void PerformAuthentication(Port *port)

入力と出力

  • **入力**: Port *port（クライアント接続情報を含む構造体）
  • **出力**: なし（成功時は認証を完了し、失敗時は FATAL エラー等でプロセスを終了させる）

前提条件と不変条件

  • ClientAuthInProgress フラグを true に設定し、認証中のログ出力を制限する。
  • 認証完了後に ClientAuthInProgress を false に戻す。
  • EXEC_BACKEND 環境下では、pg_hba.conf および pg_ident.conf の読み込みが必要。

アルゴリズムまたは振る舞い

1. **環境準備**: EXEC_BACKEND 定義時は PostmasterContext を作成し、load_hba() を実行（失敗時は FATAL）。load_ident() も試行する。 2. **タイムアウト設定**: AuthenticationTimeout に基づき STATEMENT_TIMEOUT を有効化する。 3. **認証実行**: set_ps_display("authentication") を呼び出し、ClientAuthentication(port) を実行する。 4. **後処理**: タイムアウトを無効化し、認証時間を記録する。 5. **ログ出力**: log_connections 設定に応じて、ユーザー名、データベース名、SSL/GSSAPI 情報をログに記録する。 6. **状態更新**: set_ps_display("startup") を呼び出し、認証フラグを解除する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/init/postinit.c 内の InitPostgres（MyProcPort を引数として呼び出し）。
  • **呼び出し先**: load_hba, load_ident, enable_timeout_after, ClientAuthentication, disable_timeout, ereport 等。

関連データ構造

  • Port: クライアント接続状態を保持する構造体。
  • ClientAuthInProgress: 認証中であることを示すグローバル変数。
  • conn_timing: 認証開始・終了時刻を保持する構造体。

失敗モードと境界条件

  • load_hba() の失敗: データベース接続が不可能なため FATAL エラーとなる。
  • ClientAuthentication() の失敗: 認証プロトコル内でエラーが発生した場合、関数は戻らない可能性がある。
  • タイムアウト: 認証中に AuthenticationTimeout を超過した場合、接続が切断される。

保守担当者向けメモ

  • EXEC_BACKEND 環境（Windows 等）では、認証設定ファイルの再読み込みが必須である点に注意すること。
  • 認証プロセスは STATEMENT_TIMEOUT インフラを利用しているため、認証中にデータベースアクセスが発生してもタイムアウトが適切に機能する。
