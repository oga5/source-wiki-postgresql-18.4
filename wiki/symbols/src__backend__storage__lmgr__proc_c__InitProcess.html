InitProcess

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/proc.c:389

目的

バックエンドプロセス（バックグラウンドワーカーや補助プロセスを含む）が共有メモリ上で自身の状態を管理するための PGPROC 構造体を確保し、初期化します。

シグネチャまたは形状

void InitProcess(void)

入力と出力

  • 入力: なし（グローバル変数 ProcGlobal およびプロセス種別判定関数に依存）
  • 出力: なし（副作用として MyProc グローバル変数へのポインタ設定、および共有メモリ上の PGPROC 構造体の初期化を行う）

前提条件と不変条件

  • ProcGlobal が初期化済みであること。
  • MyProc がまだ割り当てられていないこと（二重初期化の禁止）。
  • IsUnderPostmaster の状態に応じて、ポストマスターの子プロセスとして登録されること。

アルゴリズムまたは振る舞い

1. **プロセス登録**: ポストマスター配下であれば RegisterPostmasterChildActive() を呼び出します。 2. **リスト選択**: プロセス種別（AutoVacuum, BackgroundWorker, WalSender, その他）に基づき、ProcGlobal 内の適切なフリーリストを選択します。 3. **PGPROC確保**: ProcStructLock スピンロックを取得し、フリーリストから PGPROC をポップして MyProc に割り当てます。空きがない場合は FATAL エラーで終了します。 4. **構造体初期化**: MyProc の各フィールド（PID、トランザクションID、待機状態、同期レプリケーション用フラグ、グループロック用フィールド等）を初期状態に設定します。 5. **ラッチ設定**: OwnLatch() および SwitchToSharedLatch() を呼び出し、プロセスラッチを共有メモリ上のものへ切り替えます。 6. **クリーンアップ登録**: on_shmem_exit(ProcKill, 0) を登録し、終了時の解放処理を予約します。 7. **サブシステム初期化**: InitLWLockAccess() および InitDeadLockChecking() を呼び出し、ロックおよびデッドロック検出機能を有効化します。 8. **共有メモリ再アタッチ**: EXEC_BACKEND 定義環境下では、AttachSharedMemoryStructs() を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/bootstrap/bootstrap.c
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/bgworker.c
  • src/backend/replication/logical/slotsync.c
  • src/backend/replication/walsender.c
  • **呼び出し先**:
  • RegisterPostmasterChildActive, SpinLockAcquire/Release, set_spins_per_delay, dlist_pop_head_node, GetNumberFromPGProc, dlist_node_init, OwnLatch, SwitchToSharedLatch, pgstat_set_wait_event_storage, PGSemaphoreReset, on_shmem_exit, InitLWLockAccess, InitDeadLockChecking, AttachSharedMemoryStructs (EXEC_BACKEND時)

関連データ構造

  • PGPROC: プロセスごとの状態管理構造体。
  • ProcGlobal: 全プロセスの状態を管理する共有メモリ上のグローバルヘッダー。
  • ProcStructLock: フリーリスト操作を保護するスピンロック。

失敗モードと境界条件

  • ProcGlobal == NULL: PANIC で停止。
  • MyProc != NULL: ERROR で停止。
  • フリーリストが空（PGPROC 不足）: FATAL エラー（接続過多）。

保守担当者向けメモ

  • MyProc はプロセス固有の共有メモリ領域を指すため、初期化順序が重要です。
  • EXEC_BACKEND 環境では、共有メモリ構造体の再アタッチが必須となります。
  • autovacuum ランチャーは PROC_IS_AUTOVACUUM フラグを意図的に設定しない仕様です。
