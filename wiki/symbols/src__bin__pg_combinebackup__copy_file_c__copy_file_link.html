copy_file_link

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/copy_file.c:328

目的

ハードリンクを使用してソースファイルをデスティネーションへ複製し、必要に応じてチェックサムを計算する。

シグネチャまたは形状

static void copy_file_link(const char *src, const char *dest, pg_checksum_context *checksum_ctx);

入力と出力

  • src: コピー元ファイルのパス（const char *）。
  • dest: コピー先ファイルのパス（const char *）。
  • checksum_ctx: チェックサム計算用のコンテキスト（pg_checksum_context *）。NULL の場合は計算をスキップする。
  • 戻り値: なし（void）。失敗時は pg_fatal によりプロセスを終了する。

前提条件と不変条件

  • src および dest は有効なファイルパスであること。
  • link() システムコールが実行可能な権限とファイルシステム環境であること。

アルゴリズムまたは振る舞い

1. link(src, dest) を呼び出し、ハードリンクを作成する。 2. リンク作成に失敗した場合、pg_fatal を呼び出してエラーメッセージを出力し終了する。 3. 成功後、checksum_file(src, checksum_ctx) を呼び出し、ソースファイルの内容に基づいてチェックサムを計算・更新する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_combinebackup/copy_file.c 内の copy_file 関数において、strategy_implementation 関数ポインタに代入される形で参照されている。
  • 呼び出し先:
  • link() (システムコール)
  • pg_fatal()
  • checksum_file()

関連データ構造

  • pg_checksum_context: チェックサム計算の状態を保持する構造体。

失敗モードと境界条件

  • link() が失敗した場合（例: 同一ファイルシステム外へのリンク、権限不足、ファイル存在など）、pg_fatal によりプログラムが異常終了する。

保守担当者向けメモ

  • 本関数はハードリンクを利用するため、コピー元とコピー先が同一ファイルシステム上にあることが前提となる。
  • チェックサム計算はリンク作成後に別途行われるため、copy_file_blocks と比較して I/O パターンが異なる点に注意すること。
