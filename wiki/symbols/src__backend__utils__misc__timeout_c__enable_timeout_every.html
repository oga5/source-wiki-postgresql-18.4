enable_timeout_every

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/timeout.c:583

目的

指定されたタイムアウトIDに対して、初回期限および以降の繰り返し間隔を設定し、定期的なタイムアウトイベントをスケジュールする。

シグネチャまたは形状

void enable_timeout_every(TimeoutId id, TimestampTz fin_time, int delay_ms)

入力と出力

  • id: タイムアウトを識別する TimeoutId。
  • fin_time: 初回のタイムアウト発生時刻（TimestampTz）。
  • delay_ms: 繰り返し実行される際の待機間隔（ミリ秒単位）。
  • 戻り値: なし。

前提条件と不変条件

  • disable_alarm() を呼び出し、シグナル割り込みを一時的に無効化することで、データ構造の更新における安全性を確保している。

アルゴリズムまたは振る舞い

1. disable_alarm() を呼び出し、シグナルハンドラによる割り込みを抑制する。 2. GetCurrentTimestamp() で現在の時刻を取得する。 3. enable_timeout() を呼び出し、指定された id、現在時刻、初回期限、および繰り返し間隔を内部の管理配列に登録する。 4. schedule_alarm() を呼び出し、更新されたタイムアウト設定に基づいて setitimer を再設定する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/postmaster/startup.c:334
  • 呼び出し先:
  • disable_alarm()
  • GetCurrentTimestamp()
  • enable_timeout()
  • schedule_alarm()

関連データ構造

  • TimeoutId: タイムアウトの種類を定義する列挙型。
  • TimestampTz: PostgreSQLのタイムスタンプ型。

失敗モードと境界条件

  • シグナルハンドラ内での実行は想定されていない（disable_alarm による排他制御が必要なため）。
  • delay_ms が負の値や不適切な値である場合の挙動は、内部の enable_timeout の実装に依存する。

保守担当者向けメモ

  • 本関数は SIGALRM を使用するタイムアウト管理モジュールの一部である。
  • enable_timeout を経由して active_timeouts 配列を操作するため、並行処理の整合性に注意すること。
  • 繰り返し実行されるタスクは、handle_sig_alarm 内で再スケジュールされる仕組みとなっている。
