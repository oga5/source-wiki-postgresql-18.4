astreamer_gzip_writer_finalize

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_gzip.c:182

目的

gzip圧縮書き込み器に関連付けられたリソースを解放し、圧縮ファイルを適切にクローズする。

シグネチャまたは形状

static void astreamer_gzip_writer_finalize(astreamer *streamer)

入力と出力

  • **入力**: astreamer *streamer - astreamer_gzip_writer 構造体としてキャストされるストリーマーインスタンス。
  • **出力**: なし（void）。

前提条件と不変条件

  • streamer は astreamer_gzip_writer 型として安全にキャスト可能であること。
  • mystreamer->gzfile が有効な gzFile ハンドルを保持していること。

アルゴリズムまたは振る舞い

1. 引数の streamer を astreamer_gzip_writer 型にキャストする。 2. errno を 0 にリセットする（gzclose がエラー時に errno を設定しない場合に備える）。 3. gzclose(mystreamer->gzfile) を呼び出し、圧縮ファイルを閉じる。 4. gzclose が 0 以外を返した場合、pg_fatal を呼び出してエラーメッセージを出力し、プロセスを終了させる。 5. 成功後、mystreamer->gzfile を NULL に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の静的定義（src/fe_utils/astreamer_gzip.c:63）にて、astreamer インターフェースの finalize 関数ポインタとして登録されている。具体的な実行時の呼び出し元はソースから特定不可。
  • **呼び出し先**: gzclose（zlibライブラリ）、pg_fatal（PostgreSQLエラーハンドリング）。

関連データ構造

  • astreamer_gzip_writer: gzFile ハンドルおよびファイルパスを保持する構造体。

失敗モードと境界条件

  • gzclose が失敗した場合、pg_fatal によりプログラムが異常終了する。

保守担当者向けメモ

  • 本関数は gzclose を使用するため、ファイルハンドルを閉じる副作用を持つ。astreamer_gzip_writer_new で dup() を使用しているのは、この gzclose によるクローズ処理から元のファイルハンドルを保護するためである。
