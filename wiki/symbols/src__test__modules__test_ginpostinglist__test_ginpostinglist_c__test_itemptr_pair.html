test_itemptr_pair

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_ginpostinglist/test_ginpostinglist.c:40

目的

GINインデックスのポスティングリスト圧縮アルゴリズム（ginCompressPostingList）および展開アルゴリズム（ginPostingListDecode）の整合性を検証するヘルパー関数です。2つのItemPointerをペアとして処理し、圧縮後のサイズ制限と復元データの正確性を確認します。

シグネチャまたは形状

static void test_itemptr_pair(BlockNumber blk, OffsetNumber off, int maxsize)

入力と出力

  • **入力**:
  • blk (BlockNumber): 2番目のItemPointerのブロック番号。
  • off (OffsetNumber): 2番目のItemPointerのオフセット番号。
  • maxsize (int): 圧縮時に許容される最大バイト数。
  • **出力**: なし（成功時は通知ログを出力し、失敗時はelog(ERROR)によりテストを中断）。

前提条件と不変条件

  • 常に固定のTID (0, 1) と、引数で指定されたTID (blk, off) のペアがテスト対象となります。
  • 圧縮後のサイズは maxsize を超えてはなりません。
  • 展開後のTIDは、元の入力値と完全に一致する必要があります。

アルゴリズムまたは振る舞い

1. ItemPointerSet を使用して、(0, 1) と (blk, off) の2つのTIDを作成します。 2. ginCompressPostingList を呼び出し、指定された maxsize 以内で圧縮を実行します。 3. 圧縮結果のサイズが maxsize を超えていないか検証します。 4. ginPostingListDecode を呼び出し、圧縮データを元のTID配列に復元します。 5. 復元されたTIDと元のTIDを ItemPointerEquals で比較し、不一致がある場合はエラーを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**: test_ginpostinglist 関数（src/test/modules/test_ginpostinglist/test_ginpostinglist.c 内の各テストケース）。
  • **呼び出し先**:
  • ItemPointerSet
  • ginCompressPostingList
  • SizeOfGinPostingList
  • ginPostingListDecode
  • ItemPointerEquals
  • ItemPointerGetBlockNumber / ItemPointerGetOffsetNumber
  • elog

関連データ構造

  • ItemPointerData: テスト対象のTIDを保持する構造体。
  • GinPostingList: 圧縮されたポスティングリストのデータ構造。

失敗モードと境界条件

  • **サイズ超過**: 圧縮結果が maxsize を超えた場合、elog(ERROR) を発行します。
  • **データ不一致**: 復元されたTIDが元の値と異なる場合、elog(ERROR) を発行します。
  • **個数不一致**: 圧縮・展開されたTIDの数が期待値（2）と異なる場合、elog(NOTICE) で警告を出力します。

保守担当者向けメモ

  • 本関数はデルタエンコーディングの検証を目的としています。
  • maxsize 引数は、圧縮アルゴリズムのオーバーフローチェックを意図的に誘発させるために使用されます。
  • テスト実行には PostgreSQL 拡張モジュールとしてのビルドとインストールが必要です。
