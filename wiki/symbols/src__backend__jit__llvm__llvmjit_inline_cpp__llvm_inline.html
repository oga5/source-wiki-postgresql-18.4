llvm_inline

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_inline.cpp:166

目的

JIT コンパイル対象の LLVM モジュールに対し、外部モジュールから関数定義を検索・統合してインライン展開を行うためのメインエントリポイントです。

シグネチャまたは形状

void llvm_inline(LLVMModuleRef M)

入力と出力

  • **入力**: LLVMModuleRef M （インライン化対象となる LLVM モジュールへの参照）
  • **出力**: なし（副作用としてモジュール M が更新される）

前提条件と不変条件

  • 入力された LLVMModuleRef M は有効な LLVM モジュールを指している必要があります。
  • llvm::ManagedStatic で管理される module_cache および summary_cache が適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. LLVMGetModuleContext を使用してモジュールのコンテキストを取得し、LLVM C API のモジュール参照を llvm::Module へアンラップします。 2. llvm_build_inline_plan を呼び出し、現在のモジュール内の外部関数参照を解析して、インライン化すべき関数とモジュールの計画（ImportMapTy）を構築します。 3. 計画が生成されなかった場合（globalsToInline が空の場合）は処理を終了します。 4. llvm_execute_inline_plan を呼び出し、計画に基づいて llvm::IRMover を用いて外部モジュールから関数定義を現在のモジュールへ移動・統合します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit.c:726
  • **呼び出し先**:
  • llvm_build_inline_plan
  • llvm_execute_inline_plan
  • LLVM C API (LLVMGetModuleContext)
  • LLVM C++ API (llvm::unwrap)

関連データ構造

  • ImportMapTy: インライン化対象の関数とモジュールのマッピングを保持する構造。
  • LLVMModuleRef: LLVM モジュールを指す不透明なポインタ。

失敗モードと境界条件

  • llvm_build_inline_plan が nullptr を返した場合、インライン化の対象が見つからないか、計画構築が不要と判断され、関数は早期リターンします。

保守担当者向けメモ

  • 本関数はインライン化処理のオーケストレーションのみを担当します。詳細なコスト計算や統合ロジックについては、それぞれ llvm_build_inline_plan および llvm_execute_inline_plan を参照してください。
  • キャッシュの生存期間管理については、llvm_inline_reset_caches の呼び出しタイミングに注意してください。
