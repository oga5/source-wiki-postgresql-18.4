LexizeInit

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_parse.c:60

目的

全文検索における辞書処理の状態管理構造体 LexizeData を初期化し、辞書チェーン適用前のクリーンな状態を構築する。

シグネチャまたは形状

static void LexizeInit(LexizeData *ld, TSConfigCacheEntry *cfg)

入力と出力

  • **入力**:
  • ld: 初期化対象の LexizeData 構造体へのポインタ。
  • cfg: 辞書設定を含む TSConfigCacheEntry 構造体へのポインタ。
  • **出力**: なし（ld が指すメモリ領域を直接書き換える）。

前提条件と不変条件

  • ld は有効なメモリ領域を指している必要がある。
  • cfg は ts_cache.h で定義された有効な設定エントリであること。

アルゴリズムまたは振る舞い

1. ld->cfg に指定された設定をセットする。 2. ld->curDictId を InvalidOid に設定し、現在の辞書状態をリセットする。 3. ld->posDict を 0 に初期化する。 4. 辞書処理用のリスト構造（towork および waste）のヘッドとテールを NULL に設定する。 5. 作業用ポインタ lastRes および tmpRes を NULL に初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/ts_parse.c 内の parsetext 関数（2箇所で呼び出し）。
  • **呼び出し先**:
  • なし（本関数は構造体のメンバ初期化のみを行う）。

関連データ構造

  • LexizeData: 辞書処理の状態を保持するメイン構造体。
  • TSConfigCacheEntry: 全文検索設定のキャッシュ構造体。

失敗モードと境界条件

  • ld が NULL の場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は LexizeExec による辞書チェーン走査の開始前に必ず呼び出される必要がある。
  • LexizeData の構造に変更があった場合、本関数で全てのメンバが適切に初期化されているか確認すること。
