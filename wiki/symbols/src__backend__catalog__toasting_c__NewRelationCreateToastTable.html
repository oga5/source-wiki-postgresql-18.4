NewRelationCreateToastTable

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/toasting.c:70

目的

指定されたリレーションに対して、必要に応じてTOASTテーブルを生成し、メインテーブルと関連付けます。

シグネチャまたは形状

void NewRelationCreateToastTable(Oid relOid, Datum reloptions)

入力と出力

  • **入力**:
  • relOid: TOASTテーブルを作成する対象リレーションのOID。
  • reloptions: テーブル作成時に指定されたオプション（Datum形式）。
  • **出力**: なし（副作用としてシステムカタログの更新およびTOASTテーブルの作成を行う）。

前提条件と不変条件

  • 本関数は、テーブル作成処理の最終段階で呼び出されることを想定しています。
  • 内部で CommandCounterIncrement() を実行するため、呼び出し後に変更が可視化されます。

アルゴリズムまたは振る舞い

1. CheckAndCreateToastTable を呼び出し、以下の引数を渡して処理を委譲します。

  • relOid
  • reloptions
  • AccessExclusiveLock: 対象リレーションに対する排他ロック。
  • false: バイナリアップグレードモードではないことを指定。
  • InvalidOid: 既存のTOASTテーブルOIDを指定しない（新規作成）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/createas.c: CREATE TABLE AS 実行時に呼び出されます。
  • src/backend/tcop/utility.c: SQLコマンド実行時のユーティリティ処理から呼び出されます。
  • **呼び出し先**:
  • CheckAndCreateToastTable (直接の委譲先)

関連データ構造

  • Oid: リレーション識別子。
  • Datum: テーブルオプションを保持する汎用データ型。

失敗モードと境界条件

  • 対象リレーションがTOASTテーブルを必要としない場合（needs_toast_table による判定）、CheckAndCreateToastTable 内で処理がスキップされます。
  • 適切なロックが取得できない場合や、カタログ更新時にエラーが発生した場合はトランザクションがアボートされます。

保守担当者向けメモ

  • 本関数は公開インターフェースであり、主に新規テーブル作成時のエントリーポイントとして機能します。
  • 実際のTOASTテーブル生成ロジックは CheckAndCreateToastTable およびその内部の create_toast_table に集約されています。
