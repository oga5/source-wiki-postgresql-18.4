CopyQueryCompletion

定義

  • 種別: static inline function
  • 定義箇所: src/include/tcop/cmdtag.h:44

目的

QueryCompletion 構造体の内容を、あるインスタンスから別のインスタンスへコピーします。

シグネチャまたは形状

static inline void CopyQueryCompletion(QueryCompletion *dst, const QueryCompletion *src)

入力と出力

  • **入力**:
  • dst: コピー先の QueryCompletion 構造体へのポインタ。
  • src: コピー元の QueryCompletion 構造体へのポインタ（変更不可）。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • dst および src は有効な QueryCompletion 構造体を指している必要があります。
  • src の内容はコピー中も変更されないことが前提です。

アルゴリズムまたは振る舞い

1. src->commandTag を dst->commandTag に代入します。 2. src->nprocessed を dst->nprocessed に代入します。 構造体の全フィールドを個別にコピーする単純な代入操作です。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/pquery.c:774
  • src/backend/tcop/pquery.c:1038
  • src/backend/tcop/pquery.c:1361
  • **呼び出し先**:
  • 本関数はインライン関数であり、他の関数を呼び出しません。

関連データ構造

  • QueryCompletion: コマンドの完了状態（コマンドタグと処理行数）を保持する構造体。

失敗モードと境界条件

  • dst または src が NULL の場合、セグメンテーション違反が発生します。本関数自体には NULL チェックは含まれていません。

保守担当者向けメモ

  • 本関数は QueryCompletion 構造体のフィールド定義に依存しています。構造体に新しいフィールドが追加された場合は、本関数も更新する必要があります。
