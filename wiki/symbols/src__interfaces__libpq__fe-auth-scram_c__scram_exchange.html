scram_exchange

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth-scram.c:208

目的

SCRAM-SHA-256 認証プロトコルの各ステップを処理するステートマシン関数。クライアントとサーバー間のメッセージ交換を制御し、認証の成否を判定します。

シグネチャまたは形状

SASLStatus scram_exchange(void *opaq, bool final, char *input, int inputlen, char **output, int *outputlen)

入力と出力

  • **入力**:
  • opaq: fe_scram_state 構造体へのポインタ（認証状態を保持）。
  • final: 最終ステップかどうかを示すフラグ。
  • input: サーバーから受信したメッセージ文字列。
  • inputlen: 受信メッセージの長さ。
  • **出力**:
  • output: サーバーへ送信するメッセージ文字列（呼び出し元で解放が必要）。
  • outputlen: 送信メッセージの長さ。
  • **戻り値**: SASL_CONTINUE（継続）、SASL_COMPLETE（成功）、または SASL_FAILED（失敗）。

前提条件と不変条件

  • input 文字列は、解析過程で read_attr_value 等により破壊的に変更される可能性がある。
  • FE_SCRAM_INIT 以外の状態では、inputlen が 0 でないこと、および input の文字列長と一致することが必須。

アルゴリズムまたは振る舞い

fe_scram_state の状態に基づき以下の遷移を行う： 1. **FE_SCRAM_INIT**: クライアントの nonce を含む最初のメッセージを生成し、FE_SCRAM_NONCE_SENT へ遷移。 2. **FE_SCRAM_NONCE_SENT**: サーバーからの salt/nonce を解析し、クライアント証明を生成して送信。FE_SCRAM_PROOF_SENT へ遷移。 3. **FE_SCRAM_PROOF_SENT**: サーバーからの署名を検証。一致すれば SASL_COMPLETE を返し、client_finished_auth を true に設定。

呼び出し元と呼び出し先

  • **呼び出し元**: libpq の認証サブシステム（fe-auth.c 等から SASL フローの一部として呼び出される）。
  • **呼び出し先**: build_client_first_message, read_server_first_message, build_client_final_message, read_server_final_message, verify_server_signature。

関連データ構造

  • fe_scram_state: 認証セッション中の状態を保持する構造体。
  • fe_scram_state_enum: ステートマシンの遷移状態を定義する列挙型。

失敗モードと境界条件

  • メッセージの長さ不一致や空メッセージ受信時は SASL_FAILED を返し、libpq_append_conn_error でエラーを記録する。
  • サーバー署名の検証失敗や、不正なステート遷移が発生した場合は SASL_FAILED となる。

保守担当者向けメモ

  • 本関数は破壊的な入力処理を行うため、解析ロジックの変更時は注意が必要。
  • チャネルバインディングが有効な場合、SSL 証明書のハッシュ値が認証プロセスに組み込まれる点に留意すること。
