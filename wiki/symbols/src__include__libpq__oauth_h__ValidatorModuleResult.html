ValidatorModuleResult

定義

  • 種別: struct
  • 定義箇所: src/include/libpq/oauth.h:34

目的

OAuth認証プロセスにおいて、外部バリデータモジュールからコアシステムへ認証結果およびユーザー識別情報を伝達するために使用されます。

シグネチャまたは形状

typedef struct ValidatorModuleResult
{
    bool        authorized;
    char       *authn_id;
} ValidatorModuleResult;

入力と出力

  • **入力**: 外部バリデータモジュールが検証処理の結果を本構造体の各フィールドに書き込みます。
  • **出力**: コアシステム（auth-oauth.c 等）がこの構造体を参照し、接続の許可判定およびHBAマッピングを行います。

前提条件と不変条件

  • authn_id は palloc によってメモリ確保された文字列である必要があります。
  • authorized が false の場合でも、ログ記録のために authn_id を設定することが推奨されます。

アルゴリズムまたは振る舞い

1. バリデータモジュールの検証関数（validate_cb）が呼び出されます。 2. モジュールはトークンを検証し、結果を authorized に格納します。 3. モジュールは識別されたユーザー名を authn_id に格納します。 4. 呼び出し元は、これらの値に基づき接続の可否を決定し、HBAマッピングを実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-oauth.c 内の認証ロジック。
  • **呼び出し先**: 外部バリデータモジュール（src/test/modules/oauth_validator/ 配下の各実装など）。

関連データ構造

  • OAuthValidatorCallbacks: バリデータモジュールが実装する関数ポインタ群を保持する構造体。
  • ValidatorModuleState: モジュール固有のコンテキストを保持する構造体。

失敗モードと境界条件

  • authn_id が設定されていない場合、識別マッピングが必要な構成では認証が失敗する可能性があります。
  • palloc されたメモリの管理は呼び出し元に委ねられるため、メモリリークを避けるための適切な解放処理が必要です。

保守担当者向けメモ

  • 本構造体は libpq の認証フローにおいて、外部モジュールとのインターフェースの要となります。
  • authn_id のマッピングはHBA設定と密接に関係しているため、変更時は src/include/libpq/hba.h 関連のロジックへの影響を考慮してください。
