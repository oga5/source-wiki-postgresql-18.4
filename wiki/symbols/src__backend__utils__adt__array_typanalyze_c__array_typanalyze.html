array_typanalyze

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/array_typanalyze.c:97

目的

配列型カラムに対する ANALYZE 処理を拡張し、配列特有の統計情報（要素の頻度や一意な要素数の分布）を収集するためのフック関数です。

シグネチャまたは形状

Datum array_typanalyze(PG_FUNCTION_ARGS)
  • **引数**: PG_FUNCTION_ARGS (内部的に VacAttrStats * を取得)
  • **戻り値**: bool (成功した場合は true、失敗した場合は false または true を返す)

入力と出力

  • **入力**: stats (VacAttrStats *)。解析対象の属性統計構造体。
  • **出力**: 成功時、stats->compute_stats に配列専用の統計計算関数 compute_array_stats を設定し、stats->extra_data に ArrayAnalyzeExtraData 構造体を格納します。

前提条件と不変条件

  • stats->attrtypid が配列型（または配列型上のドメイン）である必要があります。
  • 配列要素型に対して、等価演算子 (eq_opr)、比較関数 (cmp_proc_finfo)、ハッシュ関数 (hash_proc_finfo) がシステムカタログから取得可能である必要があります。

アルゴリズムまたは振る舞い

1. 標準の std_typanalyze を呼び出し、基本的な統計設定を行います。 2. get_base_element_type を使用して配列の要素型 OID を特定します。 3. lookup_type_cache を介して要素型の比較・ハッシュ関数情報を取得します。必要な情報が欠落している場合は、標準の統計処理を維持して終了します。 4. ArrayAnalyzeExtraData を割り当て、要素型の型情報や関数ポインタを保存します。 5. stats->compute_stats を compute_array_stats に差し替え、元の compute_stats と extra_data を ArrayAnalyzeExtraData 内に退避させます。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の ANALYZE 処理（pg_type システムカタログの typanalyze 列で指定）。
  • **呼び出し先**:
  • std_typanalyze
  • get_base_element_type
  • lookup_type_cache
  • compute_array_stats (統計計算時に実行されるよう設定)

関連データ構造

  • VacAttrStats: ANALYZE 中の属性統計を保持する構造体。
  • ArrayAnalyzeExtraData: 配列解析に必要な型情報や関数ポインタを保持するカスタム構造体。
  • TypeCacheEntry: 要素型のキャッシュ情報。

失敗モードと境界条件

  • **エラー**: get_base_element_type が無効な OID を返した場合、配列型ではないと判断し elog(ERROR) を発生させます。
  • **早期終了**: std_typanalyze が失敗した場合、または要素型の必要な演算子・関数が取得できない場合、統計情報の拡張を行わずに終了します。

保守担当者向けメモ

  • 本関数は ANALYZE のフックとして機能します。統計情報の精度を調整する場合は、compute_array_stats 側のロジックを確認してください。
  • stats->minrows は std_typanalyze の設定値がそのまま維持されます。
