transformJoinOnClause

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_clause.c:364

目的

JOIN 句における ON 条件式の意味解析を行い、構文解析器が生成した生のノードをプランナが処理可能な形式へ変換します。

シグネチャまたは形状

Node *transformJoinOnClause(ParseState *pstate, JoinExpr *j, List *namespace)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態を保持する ParseState 構造体。
  • j: 解析対象の JoinExpr 構造体。
  • namespace: JOIN の左右のサブツリーから構成される名前空間リスト。
  • **出力**:
  • Node*: 意味解析済みの ON 条件式（transformWhereClause の結果）。

前提条件と不変条件

  • namespace に含まれる参照名の競合チェックは、呼び出し元の transformFromClauseItem ですでに完了しているものとします。
  • ON 句内のすべての名前空間アイテムは、LATERAL 状態に関わらず可視として扱われます。

アルゴリズムまたは振る舞い

1. setNamespaceLateralState を呼び出し、提供された namespace を ON 句のスコープとして設定します（LATERAL 状態を無効化し、可視性を強制）。 2. 現在の pstate->p_namespace を一時的に保存し、namespace に差し替えます。 3. transformWhereClause を呼び出し、j->quals を EXPR_KIND_JOIN_ON コンテキストで変換します。 4. 保存しておいた元の pstate->p_namespace を復元します。 5. 変換された式ノードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_clause.c:1405 において、JOIN 構文の解析中に呼び出されます。
  • **呼び出し先**:
  • setNamespaceLateralState
  • transformWhereClause

関連データ構造

  • ParseState: 解析中のクエリ状態を管理。
  • JoinExpr: JOIN 句の構文木ノード。

失敗モードと境界条件

  • transformWhereClause が内部で型不整合や未定義の列参照を検出した場合、エラーが発生し処理が中断されます。

保守担当者向けメモ

  • 本関数は ON 句のスコープを限定的に制御するためのラッパーです。LATERAL 参照の可視性制御については setNamespaceLateralState の仕様に依存しています。
