xl_heap_header

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/heapam_xlog.h:150

目的

ヒープテーブルのWAL（Write-Ahead Logging）レコードにおいて、タプルデータに先行して格納される共通ヘッダー情報を提供します。

シグネチャまたは形状

typedef struct xl_heap_header
{
	uint16		t_infomask2;
	uint16		t_infomask;
	uint8		t_hoff;
} xl_heap_header;

入力と出力

  • **入力**: ヒープタプルの属性情報（t_infomask2, t_infomask, t_hoff）。
  • **出力**: WALレコードの一部としてシリアライズされたバイナリデータ。

前提条件と不変条件

  • SizeOfHeapHeader マクロ（offsetof(xl_heap_header, t_hoff) + sizeof(uint8)）により、構造体のサイズが定義されています。
  • WALレコード内では、本構造体の直後に実際のタプルデータが続くレイアウトが想定されています。

アルゴリズムまたは振る舞い

  • ヒープ操作（挿入、更新等）の記録時に、タプルのメタデータ（t_infomask 等）をWALに永続化するために使用されます。
  • リカバリ時には、このヘッダーを読み取ることで、後続するタプルデータの構造や属性を再構築します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam.c（WAL生成時）、src/backend/access/heap/heapam_xlog.c（REDO処理時）、src/backend/replication/logical/decode.c（論理デコード時）。
  • **呼び出し先**: 直接的な関数呼び出し関係はありません。データ構造として参照されます。

関連データ構造

  • HeapTupleHeaderData（ヒープ上のタプルヘッダー）: 本構造体は、このタプルヘッダーの情報をWAL用に抽出したものです。

失敗モードと境界条件

  • WALレコードの読み込み時に、レコードの残りサイズが SizeOfHeapHeader 未満である場合、データ破損として扱われる可能性があります。

保守担当者向けメモ

  • XLOG_HEAP2_ 系操作を含むヒープ操作全般で共通して使用されます。
  • 構造体のフィールドを変更する場合は、SizeOfHeapHeader マクロおよび関連するREDOロジック（heap_redo 等）への影響を必ず確認してください。
