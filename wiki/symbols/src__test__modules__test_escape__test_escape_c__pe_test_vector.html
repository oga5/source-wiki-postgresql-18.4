pe_test_vector

定義

  • 種別: struct
  • 定義箇所: src/test/modules/test_escape/test_escape.c:76

目的

PostgreSQLの文字列エスケープ関数のテストにおいて、テストケースとなる入力文字列と、それに関連するエンコーディング情報を保持するために使用されるデータ構造です。

シグネチャまたは形状

typedef struct pe_test_vector
{
	const char *client_encoding;
	size_t		escape_len;
	const char *escape;
} pe_test_vector;

入力と出力

本構造体はデータ保持用であり、関数ではありません。

  • client_encoding: テスト対象のエンコーディング名（文字列）。
  • escape_len: escape フィールドが指す文字列の長さ。
  • escape: テスト対象となる入力文字列。

前提条件と不変条件

  • escape は escape_len で指定された長さを持つ有効な文字列である必要があります。
  • client_encoding は PostgreSQL が認識可能なエンコーディング名である必要があります。

アルゴリズムまたは振る舞い

本構造体は、pe_test_vectors 配列の要素として定義され、テスト実行時に各エスケープ関数へ渡される入力データを提供します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/test/modules/test_escape/test_escape.c:445: pe_test_vectors 配列の定義。
  • src/test/modules/test_escape/test_escape.c:634: test_one_vector_escape 関数が引数として受け取る。
  • src/test/modules/test_escape/test_escape.c:863: test_one_vector 関数が引数として受け取る。
  • 呼び出し先: 本構造体はデータ構造であるため、直接の呼び出し先はありません。

関連データ構造

  • pe_test_escape_func: テスト対象となるエスケープ関数の特性を定義する構造体。
  • pe_test_config: テスト全体の構成情報を保持する構造体。

失敗モードと境界条件

  • escape_len が実際の文字列長と一致しない場合、テスト実行時にバッファ読み取りエラーや予期せぬ挙動を引き起こす可能性があります。
  • 不正な client_encoding が指定された場合、pg_encoding_verifymbstr 等の検証関数でエラーが発生する可能性があります。

保守担当者向けメモ

  • 新しいテストケースを追加する場合は、pe_test_vectors 配列に本構造体のインスタンスを追加してください。
  • マルチバイト文字やヌルバイトを含む入力を追加する際は、escape_len の計算が正しいことを確認してください。
