cfunc_hashtable_init

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/funccache.c:58

目的

関数実行データのキャッシュを保持するグローバルなハッシュテーブル cfunc_hashtable を初期化します。

シグネチャまたは形状

static void cfunc_hashtable_init(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 cfunc_hashtable を初期化）

前提条件と不変条件

  • cfunc_hashtable が NULL であること（二重初期化の防止）。

アルゴリズムまたは振る舞い

1. HASHCTL 構造体にキーサイズ（CachedFunctionHashKey）およびエントリサイズ（CachedFunctionHashEntry）を設定します。 2. ハッシュ関数として cfunc_hash、マッチング関数として cfunc_match を指定します。 3. hash_create を呼び出し、"Cached function hash" という名称でハッシュテーブルを生成し、cfunc_hashtable に格納します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/funccache.c:174（cached_function_compile 内での遅延初期化として呼び出される可能性が高い）
  • 呼び出し先: hash_create（utils/hsearch.h）

関連データ構造

  • cfunc_hashtable: キャッシュを保持するグローバルなハッシュテーブル。
  • CachedFunctionHashKey: キャッシュ検索用のキー構造体。
  • CachedFunctionHashEntry: ハッシュテーブルに格納されるエントリ構造体。

失敗モードと境界条件

  • cfunc_hashtable が既に初期化されている場合、Assert により停止します。
  • メモリ不足時には hash_create がエラーを発生させます。

保守担当者向けメモ

  • 本関数は cached_function_compile から呼び出される初期化ルーチンです。
  • ハッシュテーブルのサイズは FUNCS_PER_USER 定数に基づいています。
