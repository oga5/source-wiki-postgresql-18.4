store_att_byval

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/tupmacs.h:210

目的

値渡し（byval）属性の Datum を、タプル内の指定されたメモリ位置へ書き込むための低レベルユーティリティです。fetch_att の逆操作として機能します。

シグネチャまたは形状

static inline void store_att_byval(void *T, Datum newdatum, int attlen)

入力と出力

  • T: 書き込み先のメモリ領域を指すポインタ。
  • newdatum: 書き込む値（Datum 型）。
  • attlen: 属性のバイト長（sizeof(char), sizeof(int16), sizeof(int32), または sizeof(Datum)）。
  • 戻り値: なし（void）。

前提条件と不変条件

  • attlen は、サポートされている型サイズ（1, 2, 4, または 64bit環境での8）のいずれかである必要があります。
  • T は、書き込むデータ型に応じた適切なアライメント境界に配置されている必要があります。

アルゴリズムまたは振る舞い

1. attlen に基づき switch 文で分岐します。 2. 各ケースにおいて、newdatum を適切な型（char, int16, int32, Datum）にキャストし、ポインタ T が指すメモリ領域へ値をコピーします。 3. サポートされていない attlen が指定された場合、elog(ERROR, ...) を呼び出し処理を中断します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • src/backend/access/brin/brin_minmax_multi.c
  • src/backend/access/common/heaptuple.c
  • src/backend/statistics/mcv.c
  • src/backend/utils/adt/arrayfuncs.c
  • src/backend/utils/adt/rangetypes.c
  • 呼び出し先:
  • elog（エラー発生時）

関連データ構造

  • Datum: PostgreSQL における汎用的なデータ表現型。
  • pg_type_d.h: アライメント定数等の定義元。

失敗モードと境界条件

  • サポート外の attlen が渡された場合、elog(ERROR) によりトランザクションがアボートされます。
  • T が不正なポインタである場合、メモリ破壊やセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は fetch_att と対になる操作を提供します。変更を加える際は、fetch_att 側の実装と整合性が取れているか確認してください。
  • SIZEOF_DATUM == 8 の条件付きコンパイルにより、プラットフォームごとの Datum サイズの違いを吸収しています。
