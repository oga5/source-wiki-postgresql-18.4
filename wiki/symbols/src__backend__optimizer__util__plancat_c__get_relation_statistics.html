get_relation_statistics

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/plancat.c:1503

目的

リレーションに関連付けられた拡張統計情報（Extended Statistics）を収集し、プランナがコスト見積もりに利用可能な形式（List）で返します。

シグネチャまたは形状

static List *get_relation_statistics(RelOptInfo *rel, Relation relation)

入力と出力

  • **入力**:
  • rel: 対象リレーションのプランナ用情報構造体（RelOptInfo）。
  • relation: 開かれたリレーションのキャッシュ構造体（Relation）。
  • **出力**:
  • 収集された統計情報構造体のリスト（List）。

前提条件と不変条件

  • relation は既にオープンされており、適切なロックが保持されている必要があります。
  • rel->relid が現在のクエリにおける正しいリレーション識別子（varno）として設定されている必要があります。

アルゴリズムまたは振る舞い

1. RelationGetStatExtList を呼び出し、対象リレーションの拡張統計オブジェクトの OID リストを取得します。 2. 各統計オブジェクトについて以下を実行します：

  • STATEXTOID システムキャッシュから pg_statistic_ext エントリを取得します。
  • 統計対象の列（stxkeys）をビットマップセットとして構築します。
  • 統計対象の式（stxexprs）が存在する場合、stringToNode で復元し、eval_const_expressions による定数畳み込みと fix_opfuncids を適用します。
  • 式に含まれる varno を、現在のクエリの varno に書き換えます（ChangeVarNodes）。
  • get_relation_statistics_worker を呼び出し、継承の有無（stxdinherit）の双方について統計情報を収集します。

    • 3. 最後に、一時的に確保したリソース（キャッシュ、ビットマップ）を解放し、収集したリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/plancat.c 内の get_relation_info から呼び出されます。
  • **呼び出し先**:
  • RelationGetStatExtList, SearchSysCache1, SysCacheGetAttr, ReleaseSysCache
  • stringToNode, eval_const_expressions, fix_opfuncids, ChangeVarNodes
  • get_relation_statistics_worker

関連データ構造

  • RelOptInfo: プランナがリレーションを表現するために使用する構造体。
  • pg_statistic_ext: 拡張統計情報を保持するシステムカタログ。

失敗モードと境界条件

  • STATEXTOID キャッシュのルックアップに失敗した場合、elog(ERROR) により処理を中断します。
  • 統計対象の式が空の場合、式に関する処理はスキップされます。

保守担当者向けメモ

  • 本関数は get_relation_info の一部として統計情報の収集を担います。
  • 式の varno 書き換えは、プランナがクエリ内の述語と統計情報を正しく照合するために不可欠です。
  • get_relation_statistics_worker を2回呼び出しているのは、継承されたテーブルとそうでないテーブルの両方の統計を考慮するためです。
