xl_btree_vacuum

定義

  • 種別: struct
  • 定義箇所: src/include/access/nbtxlog.h:223

目的

B-treeインデックスのバキューム処理（削除および更新）に関連するWAL（Write-Ahead Log）レコードのヘッダー構造を定義します。

シグネチャまたは形状

typedef struct xl_btree_vacuum
{
    uint16      ndeleted;
    uint16      nupdated;
    /* ペイロード（blk 0）: 削除対象オフセット、更新対象オフセット、更新タプルメタデータ */
} xl_btree_vacuum;

入力と出力

  • **入力**: バキューム処理時に nbtree の各操作から本構造体にデータが詰められ、WALへ書き込まれます。
  • **出力**: クラッシュリカバリやレプリケーション時に、btree_redo 等の関数が本構造体を読み取り、インデックスページの状態を復元します。

前提条件と不変条件

  • ndeleted および nupdated は、それぞれ削除および更新されたアイテムの数を保持します。
  • 実際の削除対象オフセット、更新対象オフセット、および更新タプルのメタデータは、WALレコードのブロック0のペイロード領域に格納されます。

アルゴリズムまたは振る舞い

  • 本構造体は、インデックスページ上の複数のアイテムに対する削除や更新操作を一度のWALレコードで記録するために使用されます。
  • REDO処理時には、この構造体の情報に基づき、対象ページ内の該当オフセットに対して削除や更新の再適用が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtpage.c で xlrec_vacuum としてインスタンス化されます。
  • **呼び出し先**: src/backend/access/nbtree/nbtxlog.c および src/backend/access/rmgrdesc/nbtdesc.c において、XLogRecGetData を介してレコードデータとして読み取られます。

関連データ構造

  • xl_btree_update: 更新タプルのメタデータ項目として本構造体のペイロード内で参照されます。
  • xl_btree_delete: 本構造体と同様にインデックス削除を記述するWALレコード構造体です。

失敗モードと境界条件

  • 構造体のサイズ計算には SizeOfBtreeVacuum マクロが使用されます。
  • ページ内のオフセット番号が不正な場合や、ペイロードのデータ長が不整合な場合、REDO処理中にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は xl_btree_delete と密接に関連しており、どちらもインデックスからのアイテム削除を表現します。
  • ログ解析時には nbtdesc.c の関数が本構造体の内容を人間が読める形式に変換します。
