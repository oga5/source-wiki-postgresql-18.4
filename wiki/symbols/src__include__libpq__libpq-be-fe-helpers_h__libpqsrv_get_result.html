libpqsrv_get_result

定義

  • 種別: function
  • 定義箇所: src/include/libpq/libpq-be-fe-helpers.h:333

目的

PostgreSQL バックエンド内で libpq を使用する際、クエリ結果が完全に受信されるまで待機し、安全に結果を取得するためのヘルパー関数です。

シグネチャまたは形状

PGresult *libpqsrv_get_result(PGconn *conn, uint32 wait_event_info);

入力と出力

  • **入力**:
  • conn: libpq の接続ハンドル (PGconn *)。
  • wait_event_info: 待機イベントの識別子 (uint32)。
  • **出力**:
  • 取得した PGresult * を返します。エラー発生時や結果がない場合は NULL を返します。

前提条件と不変条件

  • バックエンドのプロセスコンテキストで実行されることを前提としています。
  • MyLatch が有効であり、割り込み処理が可能な状態である必要があります。

アルゴリズムまたは振る舞い

1. PQisBusy(conn) が偽になるまでループし、データが到着するのを待ちます。 2. WaitLatchOrSocket を使用して、ソケットが読み取り可能になるか、ラッチがセットされるまで待機します。 3. ラッチがセットされた場合（割り込み要求など）、CHECK_FOR_INTERRUPTS() を呼び出して割り込みを処理します。 4. PQconsumeInput(conn) を呼び出し、ソケットからデータを消費します。消費に失敗した場合はループを抜けます。 5. ループ終了後、PQgetResult(conn) を呼び出して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • src/backend/replication/libpqwalreceiver/libpqwalreceiver.c
  • src/include/libpq/libpq-be-fe-helpers.h (内部的な再帰的利用)
  • **呼び出し先**:
  • PQisBusy, WaitLatchOrSocket, ResetLatch, CHECK_FOR_INTERRUPTS, PQconsumeInput, PQgetResult (いずれも libpq またはバックエンドのユーティリティ関数)。

関連データ構造

  • PGconn: libpq の接続状態を保持する構造体。
  • PGresult: クエリ結果を保持する構造体。
  • MyLatch: バックエンドのプロセス間通信用ラッチ。

失敗モードと境界条件

  • PQconsumeInput が 0 を返した場合（接続エラー等）、ループを中断し、その時点での PQgetResult の結果（通常は NULL）を返します。
  • 割り込みが発生した場合、CHECK_FOR_INTERRUPTS() により処理が中断される可能性があります。

保守担当者向けメモ

  • 本関数は libpq の非同期 I/O をバックエンドの割り込み処理と統合する重要なブリッジです。
  • wait_event_info は待機イベントの統計情報として使用されるため、適切な値を渡すことが重要です。
