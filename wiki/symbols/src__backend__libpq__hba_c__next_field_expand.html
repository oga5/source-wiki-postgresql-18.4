next_field_expand

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/hba.c:380

目的

認証設定ファイル（pg_hba.conf 等）の解析において、トークンを読み込み、必要に応じてファイルインクルード（@filename）を展開してトークンのリストを生成する。

シグネチャまたは形状

static List *next_field_expand(const char *filename, char **lineptr, int elevel, int depth, char **err_msg)

入力と出力

  • **入力**:
  • filename: 現在解析中のファイル名。
  • lineptr: 解析対象の文字列ポインタ。
  • elevel: エラー報告レベル。
  • depth: インクルードの再帰深さ。
  • err_msg: エラー発生時にメッセージを格納するポインタ。
  • **出力**:
  • List *: 解析されたトークンのリスト。

前提条件と不変条件

  • tokenize_context が適切に初期化されていること。
  • lineptr は有効な文字列を指していること。
  • インクルードの再帰深さが制限を超えていないこと（tokenize_expand_file 内部でチェックされる）。

アルゴリズムまたは振る舞い

1. initStringInfo で一時バッファを初期化する。 2. next_token を呼び出し、トークンを順次取得する。 3. トークンが @ で始まり、かつ引用符で囲まれていない場合、tokenize_expand_file を呼び出して外部ファイルの内容を再帰的に読み込む。 4. それ以外の場合、make_auth_token を使用してトークンを生成し、tokenize_context メモリコンテキスト上でリストに追加する。 5. トークンにカンマ（trailing_comma）が含まれている間、ループを継続する。 6. 最後に一時バッファを解放し、生成したトークンリストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/hba.c:782
  • **呼び出し先**:
  • next_token
  • tokenize_expand_file
  • make_auth_token
  • MemoryContextSwitchTo

  • pfree

関連データ構造

  • List: PostgreSQLのリスト構造。
  • StringInfoData: トークン文字列を保持する動的バッファ。
  • tokenize_context: トークン生成に使用されるメモリコンテキスト。

失敗モードと境界条件

  • next_token が失敗した場合、ループを終了する。
  • err_msg が設定された場合、ループを即座に中断する。
  • @ で始まるトークンがファイル参照として正しく解決できない場合、tokenize_expand_file がエラーを報告する。

保守担当者向けメモ

  • lappend はメモリ割り当てを行う可能性があるため、tokenize_context への切り替えが必須である。
  • カンマ区切りのトークンリストを扱うため、trailing_comma の状態管理が重要である。
