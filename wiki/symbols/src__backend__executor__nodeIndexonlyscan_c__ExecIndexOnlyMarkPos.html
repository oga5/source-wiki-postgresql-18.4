ExecIndexOnlyMarkPos

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexonlyscan.c:454

目的

インデックスのみスキャン（Index-Only Scan）において、現在のスキャン位置をマークし、後で ExecIndexOnlyRestrPos を用いて復帰できるようにする。

シグネチャまたは形状

void ExecIndexOnlyMarkPos(IndexOnlyScanState *node)

入力と出力

  • **入力**: IndexOnlyScanState *node（スキャンノードの状態構造体）
  • **出力**: なし（副作用としてスキャン位置を記録）

前提条件と不変条件

  • node は有効な IndexOnlyScanState を指している必要がある。
  • EvalPlanQual（EPQ）再チェック中である場合、対象リレーションに対して既にタプルが処理済み（relsubs_done が真）である必要がある。

アルゴリズムまたは振る舞い

1. **EPQチェック**: 現在の実行状態が EPQ 再チェック中であるかを確認する。 2. **EPQ検証**: EPQ 実行中かつ対象リレーションにテスト用タプルが存在する場合、relsubs_done フラグを確認する。フラグが偽であればエラーを投げる。 3. **位置の記録**: 上記の制約を満たす、または EPQ 実行中でない場合、node->ioss_ScanDesc を介して index_markpos を呼び出し、現在のインデックススキャン位置を保存する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:336（ExecMarkPos 経由の呼び出しが想定される）
  • **呼び出し先**: index_markpos（インデックスアクセスメソッドの API）

関連データ構造

  • IndexOnlyScanState: インデックスのみスキャンの実行状態を保持する構造体。
  • EState: 実行状態全般を管理する構造体。
  • EPQState: EvalPlanQual 再チェックの状態を管理する構造体。

失敗モードと境界条件

  • **エラー**: EPQ 再チェック中に、まだ処理されていないリレーションに対してマーク操作が行われた場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は ExecIndexOnlyRestrPos と対になる。
  • EPQ 実行中の制約は、インデックスのみスキャンが EPQ を完全にはサポートしていない（IndexOnlyRecheck がエラーを投げる）ことに関連している。
