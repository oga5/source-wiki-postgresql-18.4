pg_str_endswith

定義

  • 種別: function
  • 定義箇所: src/common/string.c:30

目的

文字列が特定の接尾辞（suffix）で終わるかどうかを判定します。

シグネチャまたは形状

bool pg_str_endswith(const char *str, const char *end);

入力と出力

  • **入力**:
  • str: 検査対象のヌル終端文字列。
  • end: 接尾辞として確認したいヌル終端文字列。
  • **出力**:
  • str が end で終わる場合は true、それ以外の場合は false を返します。

前提条件と不変条件

  • str および end は有効なヌル終端文字列である必要があります。
  • end の長さが str の長さより大きい場合、関数は直ちに false を返します。

アルゴリズムまたは振る舞い

1. strlen を用いて str と end の長さをそれぞれ取得します。 2. end の長さが str の長さより大きい場合、false を返します。 3. str のポインタを、末尾から end の長さ分だけ戻した位置まで進めます。 4. strcmp を用いて、進めた位置の文字列と end を比較し、一致すれば true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/slot.c:2223
  • src/bin/pg_rewind/filemap.c:786
  • **呼び出し先**:
  • strlen (標準Cライブラリ)
  • strcmp (標準Cライブラリ)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • str または end が NULL の場合、strlen 呼び出しにより未定義の動作（セグメンテーション違反等）が発生します。
  • 空文字列同士の比較（str が ""、end が ""）の場合、slen と elen は共に 0 となり、strcmp("", "") == 0 が評価され true を返します。

保守担当者向けメモ

  • 本関数は PostgreSQL のバックエンドおよびフロントエンドの両方から利用可能です。
  • パフォーマンス向上のため、文字列の長さを取得した後にポインタ演算を行っています。修正時は strlen の結果が正しいことを前提としています。
