PQssl_passwd_cb

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure-openssl.c:1938

目的

OpenSSL が暗号化された秘密鍵を読み込む際に使用するパスワード取得コールバック関数です。

シグネチャまたは形状

static int PQssl_passwd_cb(char *buf, int size, int rwflag, void *userdata);

入力と出力

  • buf: パスワードを格納するためのバッファ。
  • size: buf のサイズ。
  • rwflag: 読み込み（0）または書き込み（1）のフラグ。
  • userdata: PGconn 構造体へのポインタ。
  • 戻り値: パスワードの長さ（成功時）、またはエラー発生時の値。

前提条件と不変条件

  • userdata は PGconn 構造体としてキャスト可能である必要があります。
  • PQsslKeyPassHook が設定されている場合はその関数が優先され、設定されていない場合は PQdefaultSSLKeyPassHook_OpenSSL が使用されます。

アルゴリズムまたは振る舞い

1. userdata から PGconn オブジェクトを取得します。 2. グローバル変数 PQsslKeyPassHook が定義されているか確認します。 3. 定義されている場合は PQsslKeyPassHook(buf, size, conn) を呼び出し、その結果を返します。 4. 定義されていない場合は、デフォルトの PQdefaultSSLKeyPassHook_OpenSSL(buf, size, conn) を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • 呼び出し元: SSL_CTX_set_default_passwd_cb を介して OpenSSL ライブラリから呼び出されます（src/interfaces/libpq/fe-secure-openssl.c:796）。
  • 呼び出し先: PQsslKeyPassHook（関数ポインタ）または PQdefaultSSLKeyPassHook_OpenSSL。

関連データ構造

  • PGconn: libpq の接続状態を保持する構造体。

失敗モードと境界条件

  • PQsslKeyPassHook または PQdefaultSSLKeyPassHook_OpenSSL が失敗した場合、その戻り値がそのまま OpenSSL に伝播されます。
  • userdata が無効なポインタである場合の動作は未定義です。

保守担当者向けメモ

  • 本関数は OpenSSL の pem_password_cb 型に適合するように設計されています。
  • 認証情報のフック機構を抽象化しており、外部からパスワード取得ロジックを差し替え可能にしています。
