brin_inclusion_opcinfo

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_inclusion.c:93

目的

包含（Inclusion）演算子クラスを使用するBRINインデックスのメタデータ構造体（BrinOpcInfo および InclusionOpaque）を初期化し、インデックス構築やスキャンに必要な型キャッシュ情報を設定する。

シグネチャまたは形状

Datum brin_inclusion_opcinfo(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): インデックス対象列のデータ型OID。
  • **出力**:
  • BrinOpcInfo 構造体へのポインタを Datum として返す。

前提条件と不変条件

  • palloc0 によりメモリがゼロクリアされるため、InclusionOpaque 内の関数ポインタ配列やフラグは初期状態で未初期化（InvalidOid または false）として扱われる。
  • oi_nstored は 3 に固定される。
  • oi_regular_nulls は true に設定される。

アルゴリズムまたは振る舞い

1. BrinOpcInfo 構造体と InclusionOpaque 構造体を合わせた領域をメモリ確保する。 2. oi_opaque ポインタを確保したメモリ内の InclusionOpaque 領域へ向ける。 3. oi_typcache 配列の各要素を設定する:

  • INCLUSION_UNION: 指定されたデータ型の型キャッシュ。
  • INCLUSION_UNMERGEABLE: BOOLOID（結合不可フラグ用）。
  • INCLUSION_CONTAINS_EMPTY: BOOLOID（空要素フラグ用）。

    • 4. 確保した result を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に直接的な呼び出し元の記述なし。
  • **呼び出し先**:
  • palloc0
  • lookup_type_cache

関連データ構造

  • BrinOpcInfo: BRINインデックスの演算子クラス情報。
  • InclusionOpaque: 包含演算子クラス固有のメタデータ（キャッシュされた関数情報等）。
  • TypeCacheEntry: データ型のキャッシュ情報。

失敗モードと境界条件

  • lookup_type_cache が失敗した場合、PostgreSQLの標準的なエラーハンドリングにより例外が発生する可能性がある。

保守担当者向けメモ

  • InclusionOpaque の各フィールドは遅延初期化される設計である。
  • strategy_procinfos の要素は cached_subtype が変更された際に fn_oid をゼロクリアすることで無効化される。
  • extra_procinfo は一度ルックアップに失敗すると extra_proc_missing が true に設定され、再試行されない。
