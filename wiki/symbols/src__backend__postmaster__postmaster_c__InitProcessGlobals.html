InitProcessGlobals

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/postmaster.c:1932

目的

各プロセスにおける開始時刻の記録と、乱数生成器（PRNG）の初期化を行う。

シグネチャまたは形状

void InitProcessGlobals(void)

入力と出力

  • 入力: なし（グローバル変数 MyProcPid および pg_global_prng_state を使用）
  • 出力: なし（副作用としてグローバル変数の更新を行う）

前提条件と不変条件

  • プロセス起動時に一度だけ呼び出されることを想定している。
  • pg_global_prng_state が初期化可能な状態であること。

アルゴリズムまたは振る舞い

1. GetCurrentTimestamp() を用いて MyStartTimestamp を取得し、MyStartTime を設定する。 2. pg_prng_strong_seed を試行し、高品質な乱数シードで pg_global_prng_state を初期化する。 3. 高品質なシード取得が失敗した場合、プロセスID (MyProcPid) と開始時刻 (MyStartTimestamp) をビット演算で混合した値を用いて pg_prng_seed を実行する。 4. 非Windows環境では、生成された乱数シードを用いて srandom() を初期化し、レガシーな random(3) 関数との互換性を確保する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/postmaster/postmaster.c (Postmasterの初期化プロセス)
  • src/backend/utils/init/miscinit.c (バックエンドプロセスの初期化プロセス)
  • 呼び出し先:
  • GetCurrentTimestamp()
  • timestamptz_to_time_t()
  • pg_prng_strong_seed()
  • pg_prng_seed()
  • pg_prng_uint32()
  • srandom() (非Windows環境のみ)

関連データ構造

  • pg_global_prng_state: グローバルな乱数生成器の状態を保持する構造体。

失敗モードと境界条件

  • pg_prng_strong_seed が失敗した場合、プロセスIDとタイムスタンプに基づくフォールバックロジックが実行される。
  • Windows環境では srandom() が呼び出されない。

保守担当者向けメモ

  • 本関数はPostgreSQLの全プロセスで共通の初期化処理として機能する。
  • random(3) の使用はコア内では非推奨であるため、拡張モジュール向けに初期化を行っている点に注意すること。
