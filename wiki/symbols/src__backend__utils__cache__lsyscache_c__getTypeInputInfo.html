getTypeInputInfo

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/lsyscache.c:3051

目的

指定されたデータ型のOIDに基づき、その型の入力関数（typinput）および入力関数へ渡すパラメータ（typIOParam）を取得します。

シグネチャまたは形状

void getTypeInputInfo(Oid type, Oid *typInput, Oid *typIOParam)

入力と出力

  • **入力**:
  • type: 情報を取得するデータ型のOID。
  • typInput: 入力関数OIDを格納するためのポインタ（出力先）。
  • typIOParam: 入力関数に渡す補助パラメータOIDを格納するためのポインタ（出力先）。
  • **出力**: なし（引数ポインタ経由で値を返却）。

前提条件と不変条件

  • type に対応するシステムカタログ pg_type のエントリが存在すること。
  • 該当する型が「シェル型（未定義）」ではないこと。
  • 該当する型に入力関数が定義されていること。

アルゴリズムまたは振る舞い

1. TYPEOID システムキャッシュを検索し、対象型の HeapTuple を取得します。 2. キャッシュミスが発生した場合はエラーを報告します。 3. Form_pg_type 構造体へキャストし、typisdefined フラグを確認します。未定義の場合はエラーを発生させます。 4. typinput フィールドが有効なOIDか確認し、無効な場合はエラーを発生させます。 5. typinput を出力引数に代入します。 6. getTypeIOParam を呼び出し、その戻り値を typIOParam 出力引数に代入します。 7. ReleaseSysCache を呼び出し、取得したタプルの参照カウントを解放します。

関連データ構造

  • Form_pg_type: pg_type カタログの行構造体。
  • HeapTuple: システムキャッシュから取得されるタプル形式。

失敗モードと境界条件

  • **キャッシュ検索失敗**: elog(ERROR) によりトランザクションを中断します。
  • **シェル型**: typisdefined が偽の場合、ERRCODE_UNDEFINED_OBJECT でエラーを報告します。
  • **入力関数未定義**: typinput が無効なOIDの場合、ERRCODE_UNDEFINED_FUNCTION でエラーを報告します。

保守担当者向けメモ

  • 本関数は pg_type カタログの特定のフィールドを抽出するラッパーです。
  • 戻り値として typIOParam を取得するために getTypeIOParam を内部利用しています。
  • 呼び出し元でメモリ管理を行う必要はありません（値渡しのため）。
