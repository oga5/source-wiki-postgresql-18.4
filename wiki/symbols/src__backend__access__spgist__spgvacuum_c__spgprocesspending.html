spgprocesspending

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgvacuum.c:687

目的

SP-GiST インデックスの VACUUM 処理中に蓄積された保留中の TID（Tuple Identifier）リストを処理し、インデックスページ内の不要なタプルやリダイレクト情報をクリーンアップする。

シグネチャまたは形状

static void spgprocesspending(spgBulkDeleteState *bds)

入力と出力

  • **入力**: spgBulkDeleteState *bds（VACUUM 実行状態、保留リスト pendingList を含む）。
  • **出力**: なし（副作用としてインデックスページの内容を更新し、保留リストをクリアする）。

前提条件と不変条件

  • bds->pendingList に有効な保留アイテムが含まれていること。
  • 処理対象のインデックスページに対して排他ロックを取得・解放する。
  • ルートページがリダイレクト先として指定されることはない（発生した場合はエラー）。

アルゴリズムまたは振る舞い

1. bds->pendingList を走査し、未処理（done == false）のアイテムを順次処理する。 2. 各アイテムに対して vacuum_delay_point を呼び出し、バッファロックを保持しない状態で待機ポイントを設ける。 3. 参照先のページを読み込み、排他ロックを取得する。 4. ページ種別に応じた処理を実行：

  • **リーフページ**: vacuumLeafPage で不要タプルを削除し、vacuumRedirectAndPlaceholder でリダイレクト情報を整理する。同一ページを指す後続の保留アイテムも一括で done = true に設定する。
  • **内部ページ**: ページ内のインナータプルを走査し、SPGIST_LIVE ならノードのダウンリンクを、SPGIST_REDIRECT ならリダイレクト先を新たに spgAddPendingTID で保留リストに追加する。

    • 5. 最後に spgClearPendingList を呼び出し、リストを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgvacuum.c 内の spgbulkdelete（直接の呼び出しを確認）。
  • **呼び出し先**: vacuum_delay_point, ReadBufferExtended, LockBuffer, vacuumLeafPage, vacuumRedirectAndPlaceholder, SpGistSetLastUsedPage, spgAddPendingTID, spgClearPendingList 等。

関連データ構造

  • spgBulkDeleteState: VACUUM 処理の状態管理。
  • spgVacPendingItem: 保留中の TID を保持するリスト構造。
  • SpGistInnerTuple: 内部ページ上のタプル構造。

失敗モードと境界条件

  • **エラー**: ルートページがリダイレクト先として指定された場合、または予期しないタプル状態（tupstate）に遭遇した場合は elog(ERROR) を発生させる。
  • **境界条件**: 新規ページや削除済みページに遭遇した場合は無視する。

保守担当者向けメモ

  • 本関数は、インデックスの物理走査中に見つかったリダイレクト先を再訪する役割を持つ。
  • vacuumLeafPage 実行後、同一ページを指す保留アイテムをまとめて完了扱いにする最適化が含まれている。
  • 内部ページ処理時に spgAddPendingTID を再帰的に呼び出すため、保留リストのサイズが動的に変化することに留意すること。
