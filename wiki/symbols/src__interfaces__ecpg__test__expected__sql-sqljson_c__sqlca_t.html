sqlca_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-sqljson.c:32

目的

ECPG（Embedded SQL for C）において、SQL 文の実行結果やエラー状態を保持・伝達するための SQL 通信領域（SQL Communication Area）を定義する。

シグネチャまたは形状

struct sqlca_t {
    char        sqlcaid[8];
    long        sqlabc;
    long        sqlcode;
    struct {
        int     sqlerrml;
        char    sqlerrmc[SQLERRMC_LEN];
    }           sqlerrm;
    char        sqlerrp[8];
    long        sqlerrd[6];
    char        sqlwarn[8];
    char        sqlstate[5];
};

入力と出力

  • **入力**: ECPG ランタイムライブラリ（ECPGdo 等）が、SQL 文の実行結果に応じてこの構造体の各フィールドを更新する。
  • **出力**: アプリケーション側で sqlcode や sqlstate を参照し、エラーハンドリングや処理の分岐に使用する。

前提条件と不変条件

  • sqlwarn の要素 0 は、他の要素のいずれかが 'W' に設定された場合に 'W' となる。
  • sqlerrd[2] は、INSERT、UPDATE、DELETE 文の実行後に処理された行数を保持する。

アルゴリズムまたは振る舞い

SQL 文の実行後、ECPG ランタイムは実行ステータスをこの構造体に書き込む。アプリケーションは各クエリ実行後に sqlcode をチェックし、負の値であればエラーとして sqlprint() 等で診断を行うことが推奨される。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/ecpglib/connect.c
  • src/interfaces/ecpg/ecpglib/data.c
  • src/interfaces/ecpg/ecpglib/descriptor.c
  • **呼び出し先**: 構造体定義のため、特定の関数を呼び出すことはない。

関連データ構造

  • sqlda_t: 動的 SQL 用の記述子領域。
  • sqlvar_t: sqlda_t 内で使用される変数記述子。

失敗モードと境界条件

  • sqlcode が負の値の場合、SQL 文の実行失敗を示す。
  • sqlwarn の各要素は、切り捨てや非致命的な通知が発生した際に警告フラグとして機能する。

保守担当者向けメモ

  • 本構造体は sqlca.h に準拠した定義である。
  • 構造体の各フィールド（特に sqlerrd や sqlwarn）の解釈は、ECPG ランタイムの仕様に依存するため、変更時は ecpglib 側の実装と整合性を確認すること。
