smalldfa

定義

  • 種別: struct
  • 定義箇所: src/backend/regex/regexec.c:92

目的

小規模な正規表現のマッチングにおいて、ヒープメモリの割り当てを回避し、スタック上のメモリ領域を使用して DFA（決定性有限オートマトン）を効率的に管理するための構造体です。

シグネチャまたは形状

struct smalldfa {
    struct dfa dfa;                                /* 必須: 先頭に配置 */
    struct sset ssets[FEWSTATES * 2];              /* 状態セットのキャッシュ */
    unsigned statesarea[FEWSTATES * 2 + WORK];     /* 状態管理用ワークエリア */
    struct sset *outsarea[FEWSTATES * 2 * FEWCOLORS]; /* 出力遷移用エリア */
    struct arcp incarea[FEWSTATES * 2 * FEWCOLORS];   /* 入力遷移用エリア */
};

入力と出力

  • **入力**: 正規表現エンジンがマッチングを実行する際、メモリ最適化のために本構造体のインスタンスが提供されます。
  • **出力**: DFA の状態遷移およびマッチング結果を保持するコンテキストとして機能します。

前提条件と不変条件

  • struct dfa メンバが構造体の先頭に配置されている必要があります。
  • FEWSTATES および FEWCOLORS に基づく固定サイズの配列で構成されており、これを超える規模の DFA が必要な場合は、本構造体ではなく動的に割り当てられたメモリが使用されます。

アルゴリズムまたは振る舞い

正規表現エンジンは、マッチング開始時に smalldfa をスタック上に確保して利用を試みます。これにより、小規模な正規表現であれば malloc を呼び出すオーバーヘッドを排除します。大規模な正規表現やメモリ不足時には、DOMALLOC マクロ（NULL ポインタ）を介して動的メモリ割り当てへフォールバックする設計となっています。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regexec.c 内の pg_regexec 等でローカル変数として宣言されます。また、src/backend/regex/rege_dfa.c の関数群が引数として受け取ります。
  • **呼び出し先**: rege_dfa.c 内の DFA 構築・管理ロジックによって、この構造体内の領域が直接操作されます。

関連データ構造

  • struct dfa: DFA の中核となる状態管理構造体。
  • struct sset: DFA の状態セット。
  • struct arcp: DFA の遷移アーク。

失敗モードと境界条件

  • **境界条件**: FEWSTATES や FEWCOLORS で定義された容量を超える複雑な正規表現の場合、スタック上の smalldfa では不足するため、動的メモリ割り当てへ切り替える必要があります。
  • **失敗モード**: 構造体内の領域が不足し、かつ動的割り当てが失敗した場合には、正規表現エンジン全体がエラーを報告します。

保守担当者向けメモ

  • 本構造体はパフォーマンス最適化のための静的確保領域です。メンバのサイズを変更する場合は、スタックオーバーフローのリスクを考慮してください。
  • dfa メンバが先頭にあるという制約は、ポインタのキャストによる相互運用性に依存しているため、順序を変更しないでください。
