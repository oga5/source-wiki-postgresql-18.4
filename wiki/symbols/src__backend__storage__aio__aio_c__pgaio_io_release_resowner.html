pgaio_io_release_resowner

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio.c:265

目的

ResourceOwner に登録された非同期I/O（AIO）ハンドルを解放し、状態に応じて適切なクリーンアップ処理（ハンドルの再利用、未発行I/Oの強制発行、または警告の出力）を行う。

シグネチャまたは形状

void pgaio_io_release_resowner(dlist_node *ioh_node, bool on_error)

入力と出力

  • ioh_node: 解放対象の PgAioHandle に含まれる resowner_node へのポインタ。
  • on_error: エラー処理中（トランザクションアボート等）であれば true、正常終了時であれば false。
  • 戻り値: なし。

前提条件と不変条件

  • ioh_node から導出される PgAioHandle の resowner フィールドが非NULLであること。
  • 割り込みによる状態不整合を防ぐため、本関数は HOLD_INTERRUPTS() を呼び出し、終了時に RESUME_INTERRUPTS() を呼び出す。

アルゴリズムまたは振る舞い

1. dlist_container を用いて ioh_node から PgAioHandle を取得する。 2. ResourceOwnerForgetAioHandle を呼び出し、リソース所有者から当該ハンドルを登録解除する。 3. ハンドルの現在の状態（PgAioHandleState）に基づき分岐する：

  • PGAIO_HS_HANDED_OUT: ハンドルがバックエンドの handed_out_io と一致する場合、それをクリアし、pgaio_io_reclaim を実行してハンドルを回収する。
  • PGAIO_HS_DEFINED / PGAIO_HS_STAGED: 未発行のI/Oとみなされるため、pgaio_submit_staged を呼び出して発行を試みる。
  • PGAIO_HS_SUBMITTED 以降の完了状態: 特段の処理は行わない。

    • 4. on_error が false の場合、ハンドルがリークしているか、未発行のまま解放された際に警告（WARNING）を出力する。 5. ioh->report_return が設定されている場合、参照先メモリの無効化を想定して NULL にリセットする。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/resowner/resowner.c:743（ResourceOwner のクリーンアップ処理から）
  • 呼び出し先:
  • ResourceOwnerForgetAioHandle
  • pgaio_io_reclaim
  • pgaio_submit_staged
  • elog

関連データ構造

  • PgAioHandle: AIO操作を管理する主要構造体。
  • dlist_node: リソース所有者管理用のリストノード。

失敗モードと境界条件

  • PGAIO_HS_IDLE 状態で本関数が呼び出された場合、elog(ERROR) を発生させる。
  • on_error が false の場合、意図しないリークや未発行に対して警告ログが出力される。

保守担当者向けメモ

  • 本関数内での状態遷移は HOLD_INTERRUPTS() で保護されており、割り込みによる再帰的なI/O待ち合わせを防止している。
  • report_return のリセットは、I/O結果の報告先メモリが解放されることを前提としている。
