ForeignServer

定義

  • 種別: struct
  • 定義箇所: src/include/foreign/foreign.h:34

目的

PostgreSQLの外部サーバー（Foreign Server）に関するメタデータを保持し、FDW（外部データラッパー）が外部リソースへの接続や操作を行う際に必要な情報を管理する。

シグネチャまたは形状

typedef struct ForeignServer
{
	Oid			serverid;		/* server Oid */
	Oid			fdwid;			/* foreign-data wrapper */
	Oid			owner;			/* server owner user Oid */
	char	   *servername;		/* name of the server */
	char	   *servertype;		/* server type, optional */
	char	   *serverversion;	/* server version, optional */
	List	   *options;		/* srvoptions as DefElem list */
} ForeignServer;

入力と出力

  • 本構造体は、システムカタログから情報を取得する関数（GetForeignServerExtended 等）を通じてインスタンス化される。
  • options フィールドは DefElem のリストとして保持され、各FDWの実装側で解析される。

前提条件と不変条件

  • システムキャッシュから情報をコピーして作成されるため、使用後はメモリ管理（pfree 等）が必要である。
  • serverid は一意な識別子として機能する。

アルゴリズムまたは振る舞い

  • 外部サーバーの属性（所有者、型、バージョン、接続オプション）を保持するデータコンテナとして機能する。
  • 実際のデータ取得は src/backend/foreign/foreign.c が担い、システムカタログから取得した HeapTuple を本構造体にマッピングする。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の例）:
  • contrib/dblink/dblink.c
  • contrib/file_fdw/file_fdw.c
  • contrib/postgres_fdw/connection.c
  • 呼び出し先:
  • 本構造体自体はデータ構造であり、関数を呼び出すことはない。

関連データ構造

  • List: options フィールドの保持に使用。
  • Oid: サーバー、FDW、所有者の識別に使用。
  • DefElem: options リストの要素として使用。

失敗モードと境界条件

  • servertype や serverversion はオプションであり、NULLとなる可能性がある。
  • システムカタログに該当するサーバーが存在しない場合、取得関数側の設定（MISSING_OK フラグ）に応じてエラーまたはNULLが返される。

保守担当者向けメモ

  • 本構造体の options は DefElem のリストであるため、FDWの実装側で適切に解析・検証を行うこと。
  • システムキャッシュから生成されるため、永続的な保持が必要な場合は適切なメモリコンテキストでのコピーを検討すること。
