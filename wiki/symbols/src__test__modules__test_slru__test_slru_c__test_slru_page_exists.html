test_slru_page_exists

定義

  • **種別**: function
  • **定義箇所**: src/test/modules/test_slru/test_slru.c:129

目的

指定されたページ番号が物理的に存在するかどうかをSLRU（Simple Least Recently Used）キャッシュ上で確認し、その結果をSQLインターフェースへ返却する。

シグネチャまたは形状

Datum test_slru_page_exists(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_INT64(0) により、確認対象のページ番号（int64）を受け取る。
  • **出力**: PG_RETURN_BOOL(found) により、ページが存在すれば true、存在しなければ false を返す。

前提条件と不変条件

  • TestSlruCtl が初期化済みであること。
  • 呼び出し元はPostgreSQLの関数呼び出し規約に従うこと。

アルゴリズムまたは振る舞い

1. SimpleLruGetBankLock を使用して、対象ページ番号に対応するロックを取得する。 2. LWLockAcquire を呼び出し、対象のロックを排他モード（LW_EXCLUSIVE）で確保する。 3. SimpleLruDoesPhysicalPageExist を呼び出し、物理的なページ存在確認を行う。 4. LWLockRelease でロックを解放する。 5. 取得した真偽値を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1(test_slru_page_exists) により登録）。
  • **呼び出し先**:
  • SimpleLruGetBankLock
  • LWLockAcquire
  • SimpleLruDoesPhysicalPageExist
  • LWLockRelease

関連データ構造

  • TestSlruCtl: テスト用SLRUを制御する SlruCtlData 構造体。

失敗モードと境界条件

  • ロック取得時にデッドロックや競合が発生する可能性がある（SLRUのロック機構に依存）。
  • 無効なページ番号が渡された場合の挙動は SimpleLruDoesPhysicalPageExist の実装に依存する。

保守担当者向けメモ

  • 本関数は LW_EXCLUSIVE ロックを保持した状態でページ存在確認を行うため、高頻度な呼び出しはパフォーマンスに影響を与える可能性がある。
  • SimpleLruDoesPhysicalPageExist の内部実装に変更があった場合、本関数のロック戦略が適切か再評価すること。
