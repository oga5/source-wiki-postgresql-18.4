edge_failure

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_erx.c:371

目的

エッジ組み換え交叉（ERX）において、接続可能なエッジが枯渇した際に、残りの遺伝子からランダムに次の遺伝子を選択してツアーを継続させるフォールバック処理を行う。

シグネチャまたは形状

Gene edge_failure(PlannerInfo *root, Gene *gene, int index, Edge *edge_table, int num_gene)

入力と出力

  • **入力**:
  • root: クエリ最適化のコンテキスト情報。
  • gene: 現在構築中のツアー（遺伝子配列）。
  • index: 現在の遺伝子位置。
  • edge_table: 都市間の接続関係を保持するエッジテーブル。
  • num_gene: 遺伝子の総数。
  • **出力**: 次に選択された Gene（都市ID）。

前提条件と不変条件

  • edge_table 内の各都市において、unused_edges が -1 の場合はその都市が既にツアーに組み込まれていることを示す。
  • 正常系では、エッジテーブルから次の遺伝子が決定されるが、本関数はそれが不可能な異常系（または最終ステップ）で呼び出される。

アルゴリズムまたは振る舞い

1. **集計**: 未使用の遺伝子を走査し、初期エッジ数が4である遺伝子の数（four_count）と、残りの遺伝子数（remaining_edges）をカウントする。 2. **優先選択**:

  • four_count > 0 の場合: 初期エッジ数が4の遺伝子からランダムに1つを選択して返す。
  • remaining_edges > 0 の場合: 未使用の遺伝子からランダムに1つを選択して返す。

    • 3. **最終手段**: 上記のいずれにも該当しない場合（ツアーの最終点など）、unused_edges >= 0 である唯一の未使用遺伝子を探索して返す。 4. **エラー処理**: 遺伝子が見つからない場合は elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_erx.c 内の gimme_tour 関数から呼び出される。
  • **呼び出し先**: 乱数生成のために geqo_randint を呼び出す。

関連データ構造

  • Edge: 都市の接続関係と使用状況を保持する構造体。
  • Gene: 都市IDを表す型。

失敗モードと境界条件

  • **境界条件**: ツアーの最終ステップでは、エッジテーブルが空に見える状態になることがあるが、その場合は未使用の遺伝子を直接探索してリカバリする。
  • **失敗モード**: どの条件にも合致せず遺伝子を選択できない場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は、ERXアルゴリズムにおける接続の行き詰まりを解消するためのヒューリスティックなリカバリ手段である。
  • four_count を優先するロジックは、接続の制約が厳しい都市を優先的に処理するための設計である。
