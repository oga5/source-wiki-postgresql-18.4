ecpg_finish

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/connect.c:107

目的

データベース接続構造体（struct connection）を破棄し、関連するリソースの解放と接続リストからの削除を行う。

シグネチャまたは形状

static void ecpg_finish(struct connection *act)

入力と出力

  • **入力**: act (破棄対象の接続構造体へのポインタ)
  • **出力**: なし（副作用としてメモリ解放およびグローバル/スレッドローカルな接続状態の更新を行う）

前提条件と不変条件

  • 呼び出し元（ECPGdisconnect または ECPGconnect）が connections_mutex を保持している必要がある。
  • act が NULL の場合、ログ出力のみを行い何も処理しない。

アルゴリズムまたは振る舞い

1. ecpg_deallocate_all_conn を呼び出し、接続に関連するプリペアドステートメントを解放する。 2. PQfinish を呼び出し、背後の libpq 接続を閉じる。 3. all_connections リストから act を検索・削除する。 4. スレッドローカルな接続（actual_connection_key）およびグローバルな actual_connection が act を指している場合、それらをリストの先頭（または NULL）に更新する。 5. 接続名および構造体本体のメモリを解放する。 6. 全接続が閉じられた場合、ivlist（カーソル変数リスト）を走査して解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPGdisconnect および ECPGconnect（ソースコード内の直接的な呼び出し箇所として確認済み）。
  • **呼び出し先**: ecpg_deallocate_all_conn, PQfinish, pthread_getspecific, pthread_setspecific, ecpg_log, ecpg_free。

関連データ構造

  • struct connection: 接続情報を保持するメイン構造体。
  • all_connections: 全接続を管理する連結リストのヘッド。
  • actual_connection_key: スレッドごとのカレント接続を識別するキー。
  • ECPGtype_information_cache: 接続に関連付けられた型情報キャッシュ。
  • ivlist: カーソル変数を管理するリスト。

失敗モードと境界条件

  • act が NULL の場合: ログ出力のみを行い、安全に終了する。
  • 接続リスト内に act が存在しない場合: リストの走査は行われるが、削除処理はスキップされる。
  • 最後の接続が閉じられた場合: カーソル変数リスト（ivlist）のクリーンアップが実行される。

保守担当者向けメモ

  • 本関数は connections_mutex のロックを前提としているため、単独で呼び出す際は必ずロックを確保すること。
  • スレッドローカルな接続状態の更新において、pthread_setspecific を使用して actual_connection_key を適切に再設定している点に注意。
