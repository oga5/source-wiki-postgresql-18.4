SkipSupportData

定義

  • 種別: struct
  • 定義箇所: src/include/utils/skipsupport.h:61

目的

B-Tree インデックスのスキップスキャンにおいて、インデックス内の次の値を予測的に生成するための状態管理および演算子クラス固有のロジックを保持します。

シグネチャまたは形状

typedef struct SkipSupportData
{
    Datum low_elem;
    Datum high_elem;
    SkipSupportIncDec decrement;
    SkipSupportIncDec increment;
} SkipSupportData;

入力と出力

  • **入力**:
  • low_elem, high_elem: インデックス可能な値のドメインにおける最小値および最大値（昇順ソート時）。
  • decrement, increment: 値の増減を行う関数ポインタ。
  • **出力**:
  • 構造体インスタンスは、スキップスキャン実行時の状態として利用されます。

前提条件と不変条件

  • low_elem および high_elem は、対象データ型のソート順序に従って正しく設定されている必要があります。
  • decrement および increment 関数は、呼び出し元のメモリコンテキストでメモリを割り当てる必要があります。
  • これらの関数内でメモリリークを発生させてはなりません。
  • existing 引数には NULL が渡されることはありません。

アルゴリズムまたは振る舞い

  • スキャン中、現在の値から次の値を生成するために increment または decrement 関数が呼び出されます。
  • 増減関数が low_elem または high_elem と一致する値で呼び出された場合、overflow フラグをセットする必要があります。このとき、戻り値は未定義として扱われます。
  • 演算子クラスは、データ型の表現上の差異を吸収しつつ、ソート順序に影響しない情報（固定精度の表示スケールなど）を保持する必要はありません。
  • 予測が外れた場合、B-Tree スキャンは通常のインデックスプローブへフォールバックし、整合性を維持します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/skipsupport.c:41 にて palloc を介してインスタンス化されます。
  • **呼び出し先**: SkipSupportIncDec 型の関数ポインタが、インデックススキャン時の値の増減ロジックとして実行されます。

関連データ構造

  • SkipSupportIncDec: 値の増減を行う関数ポインタ型。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • 境界値（low_elem / high_elem）に到達した際、overflow フラグのセットを怠るとスキャンが正しく終了しません。
  • 連続的な型（浮動小数点数など）では次の値を予測できないため、本構造体の利用は不適切です。

保守担当者向けメモ

  • 本構造体は離散的なデータ型に特化しています。
  • increment/decrement 関数内でメモリを割り当てる際は、呼び出し元のコンテキストを考慮してください。
  • 逆順（DESC）スキャンの場合は、PrepareSkipSupportFromOpclass にて増減ロジックが入れ替えられる点に注意してください。
