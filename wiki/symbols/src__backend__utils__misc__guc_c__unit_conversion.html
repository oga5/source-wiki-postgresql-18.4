unit_conversion

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/misc/guc.c:103

目的

GUC（Grand Unified Configuration）システムにおいて、設定値の単位（例: "kB", "min"）を内部的な基準単位（base_unit）へ変換するための定義を保持する。

シグネチャまたは形状

typedef struct
{
	char		unit[MAX_UNIT_LEN + 1];
	int			base_unit;
	double		multiplier;
} unit_conversion;

入力と出力

  • 入力: 単位文字列（unit）、基準単位識別子（base_unit）、変換係数（multiplier）。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • unit は MAX_UNIT_LEN + 1 バイト以内の文字列であること。
  • multiplier は、指定された unit を base_unit に変換するための乗数として機能する。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持用であり、guc.c 内で定義される memory_unit_conversion_table や time_unit_conversion_table といった静的配列の要素として使用される。設定値の解析時に、これらのテーブルを参照して単位付きの値を正規化する処理の基盤となる。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/utils/misc/guc.c 内の memory_unit_conversion_table および time_unit_conversion_table の定義。
  • src/backend/utils/misc/guc.c 内の各関数（行 2676, 2732, 2774 付近）において、変換テーブルを走査するポインタとして使用されている。
  • 呼び出し先: なし（データ構造のため）。

関連データ構造

  • memory_unit_conversion_table: メモリ単位変換用の unit_conversion 配列。
  • time_unit_conversion_table: 時間単位変換用の unit_conversion 配列。

失敗モードと境界条件

  • unit 文字列が MAX_UNIT_LEN を超える場合、切り捨てまたはバッファオーバーフローのリスクがある（定義上の制約）。
  • 変換テーブル内に該当する単位が存在しない場合、GUC の値解析処理が失敗する可能性がある。

保守担当者向けメモ

  • 本構造体は guc.c 内で静的に定義された変換テーブルの型として機能している。
  • 新しい単位を追加する場合は、対応する base_unit（GUC_UNIT_XXX）を定義し、適切なテーブルへエントリを追加する必要がある。
