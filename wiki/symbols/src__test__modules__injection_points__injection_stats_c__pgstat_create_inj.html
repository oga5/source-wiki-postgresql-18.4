pgstat_create_inj

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_stats.c:123

目的

指定された名前のインジェクションポイントに対応する統計エントリを共有メモリ上に作成し、その統計データを初期化します。

シグネチャまたは形状

void pgstat_create_inj(const char *name)

入力と出力

  • **入力**: name (const char *) - 作成対象のインジェクションポイント名。
  • **出力**: なし。副作用として共有メモリ上の統計領域が初期化されます。

前提条件と不変条件

  • inj_stats_loaded および inj_stats_enabled が真である必要があります。これらが偽の場合、関数は何も行わずに即座に復帰します。

アルゴリズムまたは振る舞い

1. 統計機能が有効か確認し、無効であれば終了します。 2. PGSTAT_INJ_IDX(name) マクロを使用して名前からハッシュインデックスを生成し、pgstat_prep_pending_entry を呼び出して統計エントリの参照を取得します。 3. 取得した PgStatShared_InjectionPoint 構造体の stats メンバに対して memset を実行し、実行回数等の統計データを 0 に初期化します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/injection_points/injection_points.c:388
  • **呼び出し先**:
  • pgstat_prep_pending_entry (PostgreSQL統計サブシステム)
  • memset (標準ライブラリ)

関連データ構造

  • PgStatShared_InjectionPoint: 共有メモリ上に保持されるインジェクションポイントの統計データ構造体。
  • PgStat_EntryRef: 統計エントリへの参照を管理する構造体。

失敗モードと境界条件

  • 統計機能が無効な場合、処理はスキップされます。
  • name に対応するインジェクションポイントのハッシュ衝突や、統計サブシステム側のメモリ不足については、pgstat_prep_pending_entry の挙動に依存します。

保守担当者向けメモ

  • 本関数は PGSTAT_KIND_INJECTION 型の統計エントリを生成します。
  • 統計データの初期化は memset によるゼロクリアのみであるため、既存の統計値を保持したい場合には使用できません。
