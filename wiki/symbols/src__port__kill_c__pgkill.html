pgkill

定義

  • 種別: function
  • 定義箇所: src/port/kill.c:21

目的

Windows環境においてPOSIX準拠の kill() システムコールをエミュレートし、プロセスへのシグナル送信機能を提供します。

シグネチャまたは形状

int pgkill(int pid, int sig)

入力と出力

  • **入力**:
  • pid: 送信先プロセスのID（正の整数である必要があります）。
  • sig: 送信するシグナル番号。
  • **出力**:
  • 成功時: 0
  • 失敗時: -1 （errno が適切に設定されます）

前提条件と不変条件

  • Windows環境でのみ有効です。
  • pid <= 0 はサポートされておらず、EINVAL を返します。
  • sig は PG_SIGNAL_COUNT 未満かつ非負である必要があります。

アルゴリズムまたは振る舞い

1. **入力検証**: 引数の妥当性を確認し、不正な場合は EINVAL を返します。 2. **SIGKILL処理**: sig == SIGKILL の場合、OpenProcess でプロセスハンドルを取得し、TerminateProcess を用いて強制終了します。 3. **パイプ通信**: SIGKILL 以外の場合、ターゲットプロセスが作成した名前付きパイプ (\\.\pipe\pgsignal_<pid>) に対して CallNamedPipe を実行し、シグナル番号を送信します。 4. **エラー処理**: 通信失敗時、GetLastError() の結果に基づき、プロセス終了中（ERROR_BROKEN_PIPE 等）であれば成功とみなし、それ以外は ESRCH や EPERM 等のPOSIX準拠エラーへ変換します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/win32_port.h にて kill マクロとして定義されており、PostgreSQLのバックエンドコードから kill() として呼び出されます。
  • **呼び出し先**: Windows API (OpenProcess, TerminateProcess, CloseHandle, CallNamedPipe, GetLastError) を直接呼び出します。

関連データ構造

  • 特になし（標準的なWindows API型を使用）。

失敗モードと境界条件

  • pid が存在しない場合: ESRCH を返します。
  • アクセス権限がない場合: EPERM を返します。
  • パイプ通信が一時的なエラー（終了処理中など）で失敗した場合: 成功（0）を返します。
  • 不正なシグナル番号: EINVAL を返します。

保守担当者向けメモ

  • 本関数はプロセスグループへのシグナル送信をサポートしていません。
  • SIGKILL 以外のシグナルはターゲットプロセス側のパイプ受信準備に依存するため、プロセスがシグナル受信ループを維持していることが前提となります。
