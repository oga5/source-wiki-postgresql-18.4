CheckPWChallengeAuth

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth.c:822

目的

SCRAM または MD5 認証方式において、クライアントから提供されたパスワードチャレンジを検証し、ユーザーの認証を完了させる。

シグネチャまたは形状

static int CheckPWChallengeAuth(Port *port, const char **logdetail);

入力と出力

  • **入力**:
  • port: 接続情報を保持する Port 構造体へのポインタ。
  • logdetail: 認証失敗時の詳細情報を格納するためのポインタ。
  • **出力**:
  • int: 認証成功時に STATUS_OK、失敗時にそれ以外の値を返す。

前提条件と不変条件

  • port->hba->auth_method が uaSCRAM または uaMD5 であること（Assert により検証）。
  • 認証成功時には、set_authn_id を呼び出し、セッションの認証済みIDを確定させる。

アルゴリズムまたは振る舞い

1. get_role_password を呼び出し、データベースからユーザーのパスワードハッシュを取得する。 2. ユーザーが存在しない場合やパスワードが設定されていない場合でも、認証プロセスを継続する（ユーザーの存在を隠蔽するため）。 3. 認証方式の決定:

  • uaMD5 かつパスワードが MD5 形式の場合: CheckMD5Auth を実行。
  • それ以外の場合: CheckSASLAuth を使用し、SCRAM メカニズムで検証。

    • 4. 取得したパスワードハッシュ（shadow_pass）を pfree で解放する。 5. 認証成功時、set_authn_id を呼び出し、port->user_name を認証済みIDとして記録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth.c 内の認証フロー（ClientAuthentication からの呼び出しを確認）。
  • **呼び出し先**:
  • get_role_password
  • get_password_type
  • CheckMD5Auth
  • CheckSASLAuth
  • pfree
  • set_authn_id

関連データ構造

  • Port: 接続状態と認証設定（hba）を保持する構造体。
  • PasswordType: パスワードの暗号化形式を示す列挙型。

失敗モードと境界条件

  • ユーザーが存在しない場合、get_role_password が NULL を返す。この場合、Password_encryption 設定に基づいて擬似的なパスワードタイプを選択し、認証失敗を装うことで情報漏洩を防ぐ。
  • CheckSASLAuth または CheckMD5Auth が失敗した場合、認証結果は STATUS_OK 以外となり、呼び出し元へ伝播する。

保守担当者向けメモ

  • 本関数はユーザーの存在確認を隠蔽する設計となっているため、認証失敗時の挙動を変更する際は、セキュリティ上の影響（ユーザー列挙攻撃への耐性）を考慮すること。
  • 新しいパスワード暗号化方式を追加する場合は、pwtype の判定ロジックおよび CheckSASLAuth への引き渡しを確認すること。
