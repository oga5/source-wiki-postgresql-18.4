parse_oid

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/pg_combinebackup.c:798

目的

文字列形式の OID を解析し、数値型（Oid）に変換する。

シグネチャまたは形状

static bool parse_oid(char *s, Oid *result);

入力と出力

  • **入力**:
  • s: 解析対象の文字列。
  • result: 変換後の OID を格納するポインタ。
  • **出力**:
  • 成功時は true を返し、result に値を格納する。
  • 失敗時は false を返す。

前提条件と不変条件

  • s はヌル終端文字列であること。
  • 変換後の値は 1 以上 PG_UINT32_MAX 以下である必要がある。

アルゴリズムまたは振る舞い

1. strtoul を使用して文字列 s を 10 進数として変換する。 2. 以下のいずれかの条件に該当する場合、解析失敗として false を返す。

  • strtoul 実行中に errno が設定された場合。
  • 文字列の末尾まで完全に解析されなかった場合（*ep != '\0'）。
  • 変換結果が 1 未満または PG_UINT32_MAX を超える場合。

    • 3. すべての条件を満たす場合、result に値を代入し true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_combinebackup/pg_combinebackup.c:976
  • src/bin/pg_combinebackup/pg_combinebackup.c:1286
  • **呼び出し先**:
  • 標準ライブラリ関数 strtoul を呼び出す。

関連データ構造

  • Oid: PostgreSQL のオブジェクト識別子型。

失敗モードと境界条件

  • 範囲外の数値（0 以下や PG_UINT32_MAX 超）は false を返す。
  • errno が設定されるようなオーバーフロー等の異常系は false を返す。

保守担当者向けメモ

  • 本関数はディレクトリ名やファイル名から OID を抽出する用途で使用されている。
  • strtoul の結果を Oid 型にキャストして比較・代入しているため、プラットフォームごとの Oid 型の定義に依存する点に留意すること。
