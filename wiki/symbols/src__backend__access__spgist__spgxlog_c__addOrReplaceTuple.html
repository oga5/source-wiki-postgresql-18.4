addOrReplaceTuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgxlog.c:49

目的

SP-GiST インデックスページに対して、指定されたオフセット位置にタプルを追加、または既存のプレースホルダー（SPGIST_PLACEHOLDER）を新しいタプルで置換します。

シグネチャまたは形状

static void addOrReplaceTuple(Page page, Item tuple, int size, OffsetNumber offset)

入力と出力

  • **入力**:
  • page: 操作対象のインデックスページ。
  • tuple: 追加または置換するタプルデータ。
  • size: タプルのサイズ（バイト単位）。
  • offset: タプルを配置するオフセット番号。
  • **出力**: なし。成功時はページが更新され、失敗時はエラーをスローします。

前提条件と不変条件

  • offset が既存のページ内オフセットである場合、その位置にあるタプルは SPGIST_PLACEHOLDER 状態である必要があります。
  • プレースホルダーを置換する場合、ページ内の nPlaceholder カウンタが 1 以上であることを保証します。
  • offset は PageGetMaxOffsetNumber(page) + 1 以下である必要があります。

アルゴリズムまたは振る舞い

1. **置換判定**: offset が現在の最大オフセット以下の場合、既存のタプルを取得します。

  • 取得したタプルが SPGIST_PLACEHOLDER でない場合はエラーを発生させます。
  • プレースホルダーであれば、ページ内の nPlaceholder カウンタを減算し、PageIndexTupleDelete で既存の項目を削除します。

    • 2. **追加実行**: PageAddItem を呼び出し、指定された offset に新しいタプルを挿入します。 3. **検証**: PageAddItem の戻り値が期待される offset と一致しない場合、エラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • spgRedoAddLeaf
  • spgRedoMoveLeafs
  • spgRedoAddNode
  • spgRedoSplitTuple
  • spgRedoPickSplit
  • spgRedoVacuumLeaf
  • **呼び出し先**:
  • PageGetMaxOffsetNumber, PageGetItem, PageGetItemId, SpGistPageGetOpaque, PageIndexTupleDelete, PageAddItem (すべて PostgreSQL の標準ページ操作関数)

関連データ構造

  • SpGistDeadTuple: プレースホルダー状態を確認するために使用。
  • Page: インデックスページ構造。

失敗モードと境界条件

  • **エラー**:
  • 置換対象がプレースホルダーでない場合（elog(ERROR)）。
  • PageAddItem が指定されたオフセットに挿入できなかった場合（elog(ERROR)）。
  • **境界条件**: offset が PageGetMaxOffsetNumber(page) + 1 の場合、新規追加として処理されます。

保守担当者向けメモ

  • 本関数は WAL リプレイ時に使用されるため、ページの状態遷移（特にプレースホルダーの管理）が物理的なインデックス構造と厳密に一致している必要があります。
  • nPlaceholder カウンタの整合性は、インデックスの物理的な整合性に直結するため注意してください。
