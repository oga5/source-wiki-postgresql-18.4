parseAclItem

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/dumputils.c:422

目的

データベースオブジェクトの ACL（アクセス制御リスト）文字列を解析し、被付与者、付与者、および付与された権限を抽出して SQL 文構築用のバッファに格納します。

シグネチャまたは形状

static bool parseAclItem(const char *item, const char *type,
                         const char *name, const char *subname, int remoteVersion,
                         PQExpBuffer grantee, PQExpBuffer grantor,
                         PQExpBuffer privs, PQExpBuffer privswgo)

入力と出力

  • **入力**:
  • item: 解析対象の ACL 文字列。
  • type: オブジェクトの型（"TABLE", "FUNCTION" 等）。
  • name, subname: オブジェクト名およびサブオブジェクト名（列名など）。
  • remoteVersion: 接続先データベースのバージョン。
  • **出力**:
  • grantee, grantor: 被付与者名および付与者名を格納するバッファ。
  • privs, privswgo: 権限（GRANT OPTION なし/あり）を格納するバッファ。
  • 戻り値: 解析に成功した場合は true、失敗した場合は false。

前提条件と不変条件

  • item は grantee=privs/grantor の形式である必要があります。
  • privs および privswgo バッファは、関数内で resetPQExpBuffer により初期化されます。
  • subname が指定されている場合、権限文字列に付与されます。

アルゴリズムまたは振る舞い

1. item をコピーし、dequoteAclUserName を用いて被付与者と付与者を分離・抽出します。 2. type に応じて定義された権限コード（例: 'r'='SELECT', 'X'='EXECUTE'）を走査します。 3. 各権限コードが item に含まれるか確認し、*（GRANT OPTION）の有無に応じて privs または privswgo に AddAcl を呼び出します。 4. 全ての権限が特定のフラグ（all_with_go / all_without_go）を満たす場合、個別の権限列挙の代わりに "ALL" 文字列をバッファに書き込みます。 5. 最後にコピーした作業用バッファを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/dumputils.c 内の buildACLCommands 等から呼び出されます。
  • **呼び出し先**: dequoteAclUserName, AddAcl, resetPQExpBuffer, printfPQExpBuffer, appendPQExpBuffer, pg_strdup, pg_free 等。

関連データ構造

  • PQExpBuffer: SQL 文構築用の動的文字列バッファ。

失敗モードと境界条件

  • item の形式が不正（= や / が見つからない）な場合、false を返します。
  • 未知の type が渡された場合、abort() を呼び出しプロセスを終了させます。
  • subname が存在する場合、一部の権限（DELETE, TRIGGER, TRUNCATE, MAINTAIN）は列レベルの権限として適用されず、無視されます。

保守担当者向けメモ

  • 本関数は PostgreSQL の ACL 形式に強く依存しています。バックエンド側の src/backend/utils/adt/acl.c で ACL 形式が変更された場合、本関数の解析ロジックも同期させる必要があります。
  • CONVERT_PRIV マクロは、権限コードとキーワードの対応を簡潔に記述するための内部ヘルパーです。新しいオブジェクト型や権限を追加する際は、このマクロの利用を検討してください。
