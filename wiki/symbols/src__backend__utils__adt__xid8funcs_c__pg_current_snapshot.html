pg_current_snapshot

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xid8funcs.c:369

目的

現在のトランザクションスナップショットを取得し、64ビットトランザクションID（xid8）ベースの pg_snapshot 構造体として返します。

シグネチャまたは形状

Datum pg_current_snapshot(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出し規約に従う引数）。
  • **出力**: pg_snapshot 構造体へのポインタ（Datum 型）。

前提条件と不変条件

  • アクティブなスナップショットが存在する必要があります。存在しない場合、エラーが発生します。
  • 取得されたスナップショット内の各トランザクションIDは、next_fxid（現在の次のトランザクションID）に対して有効な範囲内にあることが保証されます。

アルゴリズムまたは振る舞い

1. ReadNextFullTransactionId() を呼び出し、現在のトランザクションIDの基準値を取得します。 2. GetActiveSnapshot() を通じて現在のスナップショットを取得します。 3. pg_snapshot 構造体に必要なメモリを確保します。 4. スナップショット内の xmin、xmax、および進行中のトランザクションID配列（xip）を、32ビットから64ビットの FullTransactionId へ変換して格納します。 5. sort_snapshot(snap) を呼び出し、xip 配列を昇順にソートし、重複を除去します。 6. 重複除去後のサイズに合わせて構造体のサイズを調整し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます（直接的なC言語レベルの呼び出し元はソースコード上では明示されていません）。
  • **呼び出し先**:
  • ReadNextFullTransactionId()
  • GetActiveSnapshot()
  • palloc()
  • FullTransactionIdFromAllowableAt()
  • sort_snapshot()
  • SET_VARSIZE()
  • PG_RETURN_POINTER()

関連データ構造

  • pg_snapshot: 64ビットトランザクションIDを保持するスナップショット構造体。
  • FullTransactionId: 64ビットのトランザクションID。
  • Snapshot: 内部的なスナップショット表現。

失敗モードと境界条件

  • アクティブなスナップショットが設定されていない場合、elog(ERROR) により処理が中断されます。
  • 2フェーズコミットの準備中など、一時的に重複したXIDが存在する場合、sort_snapshot による重複排除が行われます。

保守担当者向けメモ

  • 本関数はサブトランザクションIDを除外します。
  • sort_snapshot の実行後に SET_VARSIZE を呼び出すことで、重複排除によるサイズ変動を正しく反映させています。
