ZeroCLOGPage

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/clog.c:859

目的

指定されたページ番号（pageno）に対応する CLOG（Commit Log）ページをゼロクリアし、必要に応じてその操作を WAL（Write-Ahead Log）に記録する。

シグネチャまたは形状

int ZeroCLOGPage(int64 pageno, bool writeXlog)

入力と出力

  • **入力**:
  • pageno (int64): 初期化対象のページ番号。
  • writeXlog (bool): ページ初期化を WAL に記録するかどうか。
  • **出力**:
  • int: 初期化されたページが割り当てられた SLRU キャッシュ内のスロット番号。

前提条件と不変条件

  • XactCtl が初期化済みであること。
  • SimpleLruZeroPage が内部で呼び出されるため、SLRU の排他制御ルールに従うこと。

アルゴリズムまたは振る舞い

1. SimpleLruZeroPage(XactCtl, pageno) を呼び出し、指定ページをゼロクリアしてキャッシュスロットを確保する。 2. writeXlog が true の場合、WriteZeroPageXlogRec(pageno) を呼び出し、ページ初期化の WAL レコードを生成する。 3. 確保したスロット番号を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/clog.c:841 (BootStrapCLOG 内)
  • src/backend/access/transam/clog.c:978
  • src/backend/access/transam/clog.c:1125
  • **呼び出し先**:
  • SimpleLruZeroPage
  • WriteZeroPageXlogRec

関連データ構造

  • XactCtl: CLOG の共有メモリ状態を管理する SlruCtlData 構造体。

失敗モードと境界条件

  • SimpleLruZeroPage が失敗した場合（メモリ不足や I/O エラー等）、その関数内でエラーが発生し、本関数は復帰しない可能性がある。

保守担当者向けメモ

  • 本関数は CLOG の物理的な初期化を行う低レベル関数である。WAL 記録の有無を制御できるため、初期化時（BootStrapCLOG）と通常のページ拡張時で使い分けられている。
