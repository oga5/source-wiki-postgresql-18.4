interval

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/include/pgtypes_interval.h:19

目的

PostgreSQL の interval 型を C 言語環境（ECPG）で表現するためのデータ構造です。時間間隔を月・年単位と、それ以外の時間単位に分離して保持します。

シグネチャまたは形状

typedef struct
{
	int64		time;
	long		month;
} interval;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。ECPG ライブラリの変換関数（PGTYPESinterval_from_asc 等）を通じて、文字列形式との相互変換が行われます。

前提条件と不変条件

  • time: 月および年以外のすべての時間単位（マイクロ秒単位のオフセット）を保持します。
  • month: 月および年単位を保持します。
  • PostgreSQL 内部の interval 表現と整合させるため、これら二つのフィールドを分離して管理する必要があります。

アルゴリズムまたは振る舞い

本構造体は、時間間隔を「月・年」と「それ以外」に分けて格納することで、カレンダー計算と精密な時間計算の双方に対応可能な形式を提供します。

呼び出し元と呼び出し先

提供されたソース情報からは、本構造体を直接操作する特定の呼び出し元・呼び出し先関数は特定できません。ツールが検出した contrib 配下の各ファイル（btree_gin.c 等）での言及は、本構造体そのものの呼び出しではなく、PostgreSQL の interval 型に対する概念的な参照や、関連するインデックス・比較処理への言及である可能性が高いです。

関連データ構造

  • pgtypes_timestamp.h: タイムスタンプ型との演算で併用されます。
  • pgtypes_date.h: 日付型との演算で併用されます。

失敗モードと境界条件

  • メモリ管理: PGTYPESinterval_new を使用して動的に確保されたインスタンスは、必ず PGTYPESinterval_free で解放する必要があります。これを怠るとメモリリークが発生します。

保守担当者向けメモ

  • 本構造体は extern "C" ブロック内で定義されており、C++ からの利用が考慮されています。
  • 内部表現の変更は、ECPG ライブラリ全体の文字列変換ロジックおよびデータベースとのバイナリ互換性に直結するため注意してください。
