LockErrorCleanup

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/proc.c:812

目的

ロック取得待ちの最中にエラーが発生した際、プロセスを待機キューから安全に離脱させ、関連するタイムアウト設定を解除して整合性を回復する。

シグネチャまたは形状

void LockErrorCleanup(void)

入力と出力

  • 入力: なし（グローバルな MyProc および GetAwaitedLock() の状態に依存）
  • 出力: なし（副作用として共有メモリ上のロック待機状態を更新）

前提条件と不変条件

  • 割り込みが禁止された状態で呼び出される必要がある（HOLD_INTERRUPTS() を内部で実行）。
  • MyProc が有効なプロセス構造体を指していること。

アルゴリズムまたは振る舞い

1. AbortStrongLockAcquire() を呼び出し、強力なロック取得処理を中断する。 2. GetAwaitedLock() で現在待機中のロックを確認し、存在しない場合は即座に終了する。 3. DEADLOCK_TIMEOUT および LOCK_TIMEOUT タイマーを無効化する（LOCK_TIMEOUT のインジケータは保持）。 4. ロックのハッシュパーティションロックを取得する。 5. MyProc->links を確認し、待機キューに自身が残っている場合は RemoveFromWaitQueue() で離脱させる。 6. 既にキューから外れている（＝ロックが許可された、またはデッドロック検出された）場合、waitStatus が PROC_WAIT_STATUS_OK であれば GrantAwaitedLock() を呼び出し、ローカルロックテーブルに反映する。 7. ResetAwaitedLock() で待機状態をリセットし、パーティションロックを解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/xact.c (トランザクション終了処理)
  • src/backend/storage/lmgr/proc.c (内部的なエラーリカバリ)
  • 呼び出し先:
  • AbortStrongLockAcquire, GetAwaitedLock, disable_timeouts, LockHashPartitionLock, LWLockAcquire, dlist_node_is_detached, RemoveFromWaitQueue, GrantAwaitedLock, ResetAwaitedLock, LWLockRelease

関連データ構造

  • PGPROC: プロセスの状態および待機キューのリンクを保持。
  • LOCALLOCK: プロセスが現在待機しているロックの情報を保持。
  • LWLock: ロック待機キューを保護する軽量ロック。

失敗モードと境界条件

  • 待機中のロックが存在しない場合、何もせず正常終了する。
  • 既にロックが許可されている場合、GrantAwaitedLock() を通じてローカル状態を整合させる。

保守担当者向けメモ

  • 本関数は SIGINT 等の割り込み発生時に ProcessInterrupts より前に実行されることを想定しているため、LOCK_TIMEOUT のフラグ管理には注意が必要である。
  • 共有メモリ上のキュー操作を行うため、必ず partitionLock を介した排他制御を行うこと。
