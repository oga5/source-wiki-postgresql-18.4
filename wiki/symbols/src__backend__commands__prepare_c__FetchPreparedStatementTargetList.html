FetchPreparedStatementTargetList

定義

  • 種別: function
  • 定義箇所: src/backend/commands/prepare.c:488

目的

指定されたプリペアドステートメントのターゲットリスト（targetlist）を取得し、呼び出し元のメモリコンテキストへコピーして返します。

シグネチャまたは形状

List *FetchPreparedStatementTargetList(PreparedStatement *stmt)

入力と出力

  • **入力**: PreparedStatement *stmt - ターゲットリストを取得対象とするプリペアドステートメント構造体。
  • **出力**: List * - ステートメントのターゲットリストのコピー。

前提条件と不変条件

  • stmt は有効な PreparedStatement 構造体を指している必要があります。
  • 返されるリストは呼び出し元のメモリコンテキストに割り当てられるため、ステートメントの計画が後で無効化されても安全です。

アルゴリズムまたは振る舞い

1. stmt->plansource を使用して CachedPlanGetTargetList を呼び出し、計画のプライマリターゲットリストを取得します。 2. 取得したリストに対して copyObject を実行し、呼び出し元のメモリコンテキストへ複製します。 3. 複製したリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/pquery.c:405 からの呼び出しが確認されています。
  • **呼び出し先**: CachedPlanGetTargetList および copyObject を呼び出します。

関連データ構造

  • PreparedStatement: プリペアドステートメントの情報を保持する構造体。
  • CachedPlanSource: 計画のキャッシュソース。
  • List: PostgreSQLのリスト構造。

失敗モードと境界条件

  • stmt が NULL の場合、または stmt->plansource が無効な場合は、基盤となる CachedPlanGetTargetList の挙動に依存します。

保守担当者向けメモ

  • 本関数は、計画の無効化（invalidation）が発生しても呼び出し元が安全にターゲットリストを保持できるよう、明示的に copyObject を行っています。メモリリークを防ぐため、呼び出し元でリストの適切な解放が行われているか確認してください。
