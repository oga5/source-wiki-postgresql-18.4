shell_archive_file

定義

  • 種別: function
  • 定義箇所: src/backend/archive/shell_archive.c:56

目的

GUCパラメータ archive_command で指定されたシェルコマンドを実行し、WALファイルを外部ストレージへ転送する。

シグネチャまたは形状

static bool shell_archive_file(ArchiveModuleState *state, const char *file, const char *path);

入力と出力

  • **入力**:
  • state: アーカイブモジュールの状態（使用されず）。
  • file: アーカイブ対象のWALファイル名。
  • path: WALファイルのフルパス。
  • **出力**:
  • bool: コマンドが正常終了（終了コード0）した場合は true、それ以外は false を返す。

前提条件と不変条件

  • XLogArchiveCommand が設定されている必要がある。
  • path が提供される場合、make_native_path によりプラットフォーム固有の形式に変換される。
  • コマンド実行中は pgstat_report_wait_start により WAIT_EVENT_ARCHIVE_COMMAND 待機イベントが記録される。

アルゴリズムまたは振る舞い

1. replace_percent_placeholders を使用し、XLogArchiveCommand 内の %f（ファイル名）と %p（パス）を置換して実行コマンド文字列を生成する。 2. fflush(NULL) を呼び出し、バッファをフラッシュする。 3. system() を介してシェルコマンドを実行する。 4. 終了コードを解析する:

  • 正常終了（0）: true を返す。
  • 異常終了: wait_result_is_any_signal により、シグナルによる中断や致命的なエラーと判定された場合は FATAL レベルで報告し、それ以外は LOG レベルで報告して false を返す。

    • 5. 生成したコマンド文字列のメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: shell_archive_callbacks 構造体（ArchiveModuleCallbacks）の .archive_file_cb フィールドに登録され、アーカイブプロセスから間接的に呼び出される。
  • **呼び出し先**: pstrdup, make_native_path, replace_percent_placeholders, pgstat_report_wait_start, system, pgstat_report_wait_end, wait_result_is_any_signal, WIFEXITED, WEXITSTATUS, WIFSIGNALED, WTERMSIG, pg_strsignal, ereport, pfree, elog。

関連データ構造

  • ArchiveModuleState: アーカイブモジュールの状態管理用構造体。
  • ArchiveModuleCallbacks: アーカイブモジュールのコールバック関数を保持する構造体。

失敗モードと境界条件

  • system() がシグナルで中断された場合、アーカイブプロセスを終了させるために FATAL エラーを発生させる。
  • コマンドが見つからない等のシェルエラーが発生した場合、LOG または FATAL レベルでエラーを報告し、アーカイブは失敗する。
  • path が NULL の場合、プレースホルダー置換時にパス部分は処理されない。

保守担当者向けメモ

  • system() を使用しているため、archive_command に指定する文字列のセキュリティ（コマンドインジェクション等）に注意が必要。
  • シグナルによる中断を FATAL として扱うことで、ポストマスターによるアーカイブプロセスの再起動を促す設計となっている。
