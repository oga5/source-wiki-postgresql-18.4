WalSummarizerShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/walsummarizer.c:182

目的

WAL Summarizer プロセスが使用する共有メモリ領域の割り当ておよび初期化を行う。

シグネチャまたは形状

void WalSummarizerShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上の WalSummarizerCtl が初期化される）

前提条件と不変条件

  • ShmemInitStruct を介して共有メモリ領域を確保する。
  • 共有メモリが既に存在する場合（found が true）、初期化処理はスキップされる。
  • 初回起動時のみ、WalSummarizerData 構造体の各フィールドにデフォルト値（initialized = false, summarized_lsn = InvalidXLogRecPtr 等）が設定される。

アルゴリズムまたは振る舞い

1. WalSummarizerShmemSize() を呼び出し、必要な共有メモリサイズを算出する。 2. ShmemInitStruct を呼び出し、"Wal Summarizer Ctl" という名称で共有メモリを確保または取得する。 3. 共有メモリが新規作成された場合、WalSummarizerCtl の各メンバを初期化し、ConditionVariableInit を実行して条件変数をセットアップする。 4. 実際の要約開始に必要な詳細な初期化は、この関数内ではなく、後続の GetOldestUnsummarizedLSN() 呼び出し時に行われる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:333（PostgreSQL の共有メモリ初期化シーケンスの一部として呼び出される）
  • 呼び出し先: ShmemInitStruct, ConditionVariableInit, WalSummarizerShmemSize

関連データ構造

  • WalSummarizerData: 共有メモリ上に配置される制御構造体。
  • WalSummarizerCtl: WalSummarizerData を指すグローバルポインタ。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、ShmemInitStruct の内部実装に従いエラーが発生する。
  • 既に初期化済みの場合は、既存の共有メモリ状態を保持し、再初期化は行わない。

保守担当者向けメモ

  • 本関数は共有メモリの「枠」を確保する役割に特化している。要約処理の開始に必要な論理的な初期化は GetOldestUnsummarizedLSN に委譲されている点に注意すること。
