GinInitPage

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginutil.c:342

目的

GINインデックスのページヘッダーを初期化し、指定されたフラグとページサイズに基づいて不透明領域（Opaque）を設定します。

シグネチャまたは形状

void GinInitPage(Page page, uint32 f, Size pageSize)

入力と出力

  • page: 初期化対象のページバッファ。
  • f: ページに設定する GinPageOpaqueData のフラグ（GIN_DATA, GIN_LEAF 等）。
  • pageSize: ページのサイズ（バイト単位）。
  • 戻り値: なし。

前提条件と不変条件

  • page は PageInit を受け入れるために十分なメモリ領域を確保済みであること。
  • pageSize は PostgreSQL のブロックサイズ（BLCKSZ）と整合していること。

アルゴリズムまたは振る舞い

1. PageInit を呼び出し、ページ全体を初期化します。この際、sizeof(GinPageOpaqueData) を不透明領域のサイズとして指定します。 2. GinPageGetOpaque を使用して、ページ内の不透明領域へのポインタを取得します。 3. 渡されたフラグ f を opaque->flags に格納します。 4. opaque->rightlink を InvalidBlockNumber に設定し、右側の兄弟ページが存在しない状態として初期化します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginbtree.c
  • src/backend/access/gin/gindatapage.c
  • src/backend/access/gin/ginentrypage.c
  • src/backend/access/gin/ginutil.c（GinInitBuffer および GinInitMetabuffer 内）
  • **呼び出し先**:
  • PageInit (PostgreSQL 共通関数)
  • GinPageGetOpaque (マクロ)

関連データ構造

  • GinPageOpaqueData: GIN インデックスページの末尾に配置されるメタデータ構造体。フラグと右兄弟へのリンクを保持する。

失敗モードと境界条件

  • page が NULL の場合、または pageSize が不適切な場合、PageInit 内でセグメンテーション違反やメモリ破壊が発生する可能性があります。

保守担当者向けメモ

  • 本関数は GIN インデックスの物理的なページレイアウトを決定する基礎的な関数です。フラグの変更やページ構造の拡張を行う際は、GinPageOpaqueData の定義と整合性を確認してください。
