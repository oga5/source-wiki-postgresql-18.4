JsonUniqueParsingState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/json.c:59

目的

JSON解析時におけるキーの重複チェックおよび解析状態を管理するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct JsonUniqueParsingState
{
	JsonLexContext *lex;
	JsonUniqueCheckState check;
	JsonUniqueStackEntry *stack;
	int			id_counter;
	bool		unique;
} JsonUniqueParsingState;

入力と出力

  • **入力**: JSONの字句解析コンテキスト（lex）および重複チェック用の状態（check）。
  • **出力**: 解析の結果として、キーの重複有無を示すフラグ（unique）および解析スタックの状態を保持します。

前提条件と不変条件

  • lex は有効な JsonLexContext を指している必要があります。
  • unique フラグは、解析中に同一階層で重複キーが検出された場合に更新されます。

アルゴリズムまたは振る舞い

  • JSONオブジェクトの解析中に、現在の階層におけるキーの出現を追跡します。
  • id_counter を用いて解析対象の識別子を管理し、stack を介してネストされたJSON構造の重複チェックを再帰的にサポートします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/json.c 内の解析関数（1756, 1774, 1789, 1816 行付近）で参照・初期化されています。
  • **呼び出し先**: 構造体内部で JsonLexContext、JsonUniqueCheckState、JsonUniqueStackEntry を利用しており、これらに関連する関数群が呼び出されます。

関連データ構造

  • JsonLexContext: 字句解析の基盤。
  • JsonUniqueCheckState: 重複チェックのハッシュテーブル等の状態。
  • JsonUniqueStackEntry: ネストされた構造を追跡するためのスタック要素。

失敗モードと境界条件

  • メモリ不足によるスタック割り当て失敗の可能性があります。
  • 非常に深いネストを持つJSONデータに対しては、スタックの深さが制限を超える可能性があります。

保守担当者向けメモ

  • 本構造体は json.c 内のキー重複チェックロジックの中核です。修正時は JsonUniqueStackEntry の管理ロジックと整合性を保つ必要があります。
  • id_counter のオーバーフローについては、現実的なJSONの深さでは考慮不要ですが、極端なケースでは留意してください。
