pgstat_get_my_query_id

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/backend_status.c:1137

目的

現在のバックエンドプロセスに関連付けられたクエリID（st_query_id）を取得します。

シグネチャまたは形状

int64 pgstat_get_my_query_id(void);

入力と出力

  • 入力: なし
  • 出力: 現在のバックエンドのクエリID（int64）。バックエンドが初期化されていない場合は 0 を返します。

前提条件と不変条件

  • MyBEEntry が有効である必要があります。
  • 本関数は、呼び出し元が既に保護されている場合（例: pg_stat_get_activity）や、呼び出し元自身が現在のバックエンドである場合に呼び出されます。

アルゴリズムまたは振る舞い

1. グローバル変数 MyBEEntry が NULL かどうかを確認し、NULL であれば 0 を返します。 2. MyBEEntry が存在する場合、共有メモリ上の st_query_id フィールドの値を直接読み取って返します。 3. 読み取り時にロック（pgstat_begin_read_activity 等）を使用しません。これは、呼び出し元が保護されているか、あるいは自身のバックエンド状態を読み取るため競合が発生しないという前提に基づいています。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • src/backend/access/brin/brin.c
  • src/backend/access/nbtree/nbtsort.c
  • src/backend/commands/vacuumparallel.c
  • src/backend/executor/execParallel.c
  • src/backend/utils/error/csvlog.c
  • src/backend/utils/error/elog.c
  • src/backend/utils/error/jsonlog.c
  • 呼び出し先: なし

関連データ構造

  • PgBackendStatus: MyBEEntry が指す構造体。st_query_id を保持します。

失敗モードと境界条件

  • MyBEEntry が初期化されていない（NULL）場合、エラーにはならず 0 を返します。

保守担当者向けメモ

  • 本関数はロックフリーで実装されています。将来的に呼び出し元のコンテキストが変更され、自身のバックエンド以外の状態を読み取る可能性がある場合は、適切なロックプロトコル（st_changecount を用いた楽観的並行制御）の導入を検討してください。
