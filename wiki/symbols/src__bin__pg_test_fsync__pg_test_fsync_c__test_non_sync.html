test_non_sync

定義

  • 種別: function
  • 定義箇所: src/bin/pg_test_fsync/pg_test_fsync.c:571

目的

同期処理（fsync等）を伴わない単純な書き込み操作のベースラインパフォーマンスを測定する。

シグネチャまたは形状

static void test_non_sync(void)

入力と出力

  • 入力: なし（グローバル変数 filename、buf、XLOG_BLCKSZ を使用）
  • 出力: なし（標準出力へ測定結果を表示）

前提条件と不変条件

  • filename で指定されたファイルが書き込み可能であること。
  • buf に書き込み用のデータが準備されていること。
  • alarm_triggered フラグが時間計測の終了を制御する。

アルゴリズムまたは振る舞い

1. 「Non-sync'ed ... writes:」というヘッダーを標準出力に表示する。 2. open() を使用して一時ファイルを O_RDWR モードで開く。 3. START_TIMER マクロで計測を開始する。 4. alarm_triggered が true になるまで、pg_pwrite() を用いて XLOG_BLCKSZ サイズのデータをファイルへ繰り返し書き込む。 5. STOP_TIMER マクロで計測を終了する。 6. close() でファイル記述子を解放する。

呼び出し元と呼び出し先

  • 呼び出し元: main 関数（src/bin/pg_test_fsync/pg_test_fsync.c:138）
  • 呼び出し先: open, printf, fflush, pg_pwrite, die, close

関連データ構造

  • buf: 書き込みに使用されるデータバッファ。
  • XLOG_BLCKSZ: 書き込み単位となるブロックサイズ。

失敗モードと境界条件

  • open() が失敗した場合: die() を呼び出し、エラーメッセージを出力してプログラムを終了する。
  • pg_pwrite() が指定されたバイト数を書き込めなかった場合: die() を呼び出し、エラーメッセージを出力してプログラムを終了する。

保守担当者向けメモ

  • 本関数は同期なしの純粋な書き込み速度を測定するベースラインである。
  • alarm_triggered の状態に依存するため、シグナルハンドラが正しく設定されている環境で実行される必要がある。
