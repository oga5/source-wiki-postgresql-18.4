ZstdCompressorState

定義

  • **種別**: struct
  • **定義箇所**: src/bin/pg_dump/compress_zstd.c:39

目的

pg_dump における Zstandard (Zstd) 圧縮・解凍処理の実行コンテキストを保持する構造体です。ストリーム、バッファ、およびファイルポインタを管理し、アーカイブデータの透過的な読み書きを実現します。

シグネチャまたは形状

typedef struct ZstdCompressorState
{
    FILE         *fp;            /* 圧縮データの読み書き対象ファイル */
    ZSTD_CStream *cstream;       /* Zstd 圧縮ストリーム */
    ZSTD_DStream *dstream;       /* Zstd 解凍ストリーム */
    ZSTD_outBuffer output;       /* 出力用バッファ */
    ZSTD_inBuffer input;         /* 入力用バッファ */
    const char   *zstderror;     /* エラーメッセージ保持用ポインタ */
} ZstdCompressorState;

入力と出力

  • **入力**: pg_dump のアーカイブ処理から提供されるファイルポインタおよびデータストリーム。
  • **出力**: 圧縮または解凍されたデータが fp を通じてアーカイブへ読み書きされます。

前提条件と不変条件

  • ビルド時に USE_ZSTD が定義されている環境でのみ有効です。
  • pg_malloc0 等でメモリ確保された後、InitCompressorZstd または InitCompressFileHandleZstd によって適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

  • ZSTD_compressStream2 および ZSTD_decompressStream を用いたストリーム処理を抽象化します。
  • 圧縮・解凍の各フェーズにおいて、input および output バッファを介してデータを中継します。
  • zstderror は strerror() 等の静的文字列を保持し、エラー発生時の診断情報として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: InitCompressorZstd、InitCompressFileHandleZstd、および Zstd_read_internal 等の compress_zstd.c 内の各関数が、cs->private_data または CFH->private_data をキャストして本構造体を参照します。
  • **呼び出し先**: Zstandard ライブラリ（zstd.h）のストリーム操作関数群。

関連データ構造

  • CompressorState: pg_dump の汎用圧縮インターフェース。
  • CompressFileHandle: ファイルベースの圧縮ハンドル。

失敗モードと境界条件

  • メモリ不足時に pg_malloc0 が失敗する可能性があります。
  • ZSTD_compressStream2 や ZSTD_decompressStream の戻り値がエラーを示す場合、適切なエラーハンドリングが必要です。
  • 読み込み時、バッファの動的リサイズや EOF 判定において複雑な制御フローが発生します。

保守担当者向けメモ

  • 本構造体は compress_zstd.c 内の各関数で private_data としてキャストして利用されるため、構造体のメンバ変更時は関連する全ての関数を確認してください。
  • Zstd_read_internal は実装の複雑度が高いため、修正時はバッファ管理の整合性に注意してください。
