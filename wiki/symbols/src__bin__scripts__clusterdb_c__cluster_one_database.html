cluster_one_database

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/clusterdb.c:188

目的

指定されたデータベースに対して CLUSTER SQL コマンドを発行し、テーブルの物理的な再配置を実行する。

シグネチャまたは形状

static void cluster_one_database(const ConnParams *cparams, const char *table, const char *progname, bool verbose, bool echo);

入力と出力

  • **入力**:
  • cparams: 接続パラメータ（ConnParams 構造体）。
  • table: 対象テーブル名（NULL の場合はデータベース全体を対象とする）。
  • progname: ログ出力に使用するプログラム名。
  • verbose: VERBOSE オプションの有効/無効。
  • echo: SQL コマンドを標準出力に表示するかどうか。
  • **出力**: なし（成功時は終了、失敗時は exit(1) を呼び出す）。

前提条件と不変条件

  • cparams を用いてデータベースへの接続が確立可能であること。
  • executeMaintenanceCommand が正常に SQL を送信・実行できる状態であること。

アルゴリズムまたは振る舞い

1. connectDatabase を呼び出し、対象データベースへの接続を確立する。 2. PQExpBuffer を初期化し、CLUSTER コマンド文字列を構築する。

  • verbose が真の場合、VERBOSE を付与する。
  • table が指定されている場合、appendQualifiedRelation を用いて対象テーブルをクエリに追加する。

    • 3. executeMaintenanceCommand を介してサーバーへコマンドを送信する。 4. 実行失敗時は pg_log_error でエラー内容を出力し、接続を閉じてプロセスを終了する。 5. 成功時は接続を閉じ、バッファを解放して正常終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/scripts/clusterdb.c 内の main 関数および cluster_all_databases 関数から呼び出される。
  • **呼び出し先**:
  • connectDatabase
  • initPQExpBuffer, appendPQExpBufferStr, appendPQExpBufferChar, termPQExpBuffer
  • appendQualifiedRelation
  • executeMaintenanceCommand
  • PQdb, PQerrorMessage, PQfinish
  • pg_log_error, exit

関連データ構造

  • ConnParams: 接続情報を保持する構造体。
  • PQExpBufferData: SQL クエリ文字列を構築するためのバッファ。
  • PGconn: libpq のデータベース接続ハンドル。

失敗モードと境界条件

  • データベース接続失敗: connectDatabase が失敗した場合、処理は中断される。
  • SQL 実行失敗: executeMaintenanceCommand が偽を返した場合、エラーメッセージを出力して exit(1) で終了する。
  • テーブル指定なし: table が NULL の場合、データベース全体に対して CLUSTER が発行される。

保守担当者向けメモ

  • 本関数は libpq を直接叩くのではなく、fe_utils のユーティリティ関数を介してコマンドを実行する設計となっている。
  • CLUSTER コマンドは対象テーブルをロックするため、実行タイミングには注意が必要である。
