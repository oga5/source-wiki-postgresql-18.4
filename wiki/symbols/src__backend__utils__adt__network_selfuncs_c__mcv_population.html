mcv_population

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_selfuncs.c:553

目的

統計情報として保持されている最頻値（MCV）の出現頻度の合計を算出し、MCVリスト全体が占める選択率の総和を求める。

シグネチャまたは形状

Selectivity mcv_population(float4 *mcv_numbers, int mcv_nvalues)

入力と出力

  • **入力**:
  • mcv_numbers: 各MCVの出現頻度を格納した float4 配列。
  • mcv_nvalues: 配列内の要素数。
  • **出力**:
  • Selectivity (double): MCVリストに含まれる全要素の出現頻度の合計値。

前提条件と不変条件

  • mcv_numbers は mcv_nvalues 分の有効な要素を持つ配列であること。
  • 戻り値は、統計情報の定義に基づき、0.0 から 1.0 の範囲内となることが期待される。

アルゴリズムまたは振る舞い

1. sumcommon 変数を 0.0 で初期化する。 2. mcv_nvalues 回のループ処理を行い、mcv_numbers 配列の各要素を sumcommon に加算する。 3. 最終的な合計値 sumcommon を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network_selfuncs.c 内の以下の箇所から呼び出されている。
  • 317行目, 339行目, 444行目, 466行目
  • **呼び出し先**:
  • 特になし（標準的な加算演算のみ）。

関連データ構造

  • float4: PostgreSQLの統計情報で頻度を保持するために使用される単精度浮動小数点型。
  • Selectivity: 選択率を表す double 型のエイリアス。

失敗モードと境界条件

  • mcv_nvalues が 0 の場合、ループは実行されず 0.0 が返される。
  • 配列の内容が不正（負の値や合計が1を超える等）である場合、計算結果もその不整合を継承する。

保守担当者向けメモ

  • 本関数は単純な合計計算を行うユーティリティである。
  • 統計情報の取得元である pg_statistic のスロット構造が変更されない限り、ロジックの修正は不要である。
