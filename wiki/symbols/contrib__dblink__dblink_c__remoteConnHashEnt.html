remoteConnHashEnt

定義

  • 種別: struct
  • 定義箇所: contrib/dblink/dblink.c:163

目的

remoteConnHash ハッシュテーブルの各エントリとして、名前付きリモート接続情報を保持するために使用されます。

シグネチャまたは形状

typedef struct remoteConnHashEnt
{
	char		name[NAMEDATALEN];
	remoteConn	rconn;
} remoteConnHashEnt;

入力と出力

本構造体はデータ保持用であり、関数としての入力・出力はありません。ハッシュテーブル操作時にキーとして name が使用され、値として rconn が格納・取得されます。

前提条件と不変条件

  • name フィールドは NAMEDATALEN バイトの固定長配列であり、接続を識別する一意な名前を格納します。
  • rconn フィールドは remoteConn 型であり、有効なリモート接続状態を保持している必要があります。

アルゴリズムまたは振る舞い

ハッシュテーブルの要素として、接続名とそれに対応する接続状態をペアで管理します。hash_search を用いてこの構造体単位で検索・挿入・削除が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: dblink.c 内の接続管理関数（dblink_get_connections、dblink_disconnect、getConnectionByName 等）が、ハッシュテーブル操作の際に本構造体を参照・キャストしています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • remoteConn: 接続状態を保持する構造体。
  • remoteConnHash: 本構造体を要素として管理するハッシュテーブル。

失敗モードと境界条件

  • ハッシュテーブルの初期化（createConnHash）時に ctl.entrysize として sizeof(remoteConnHashEnt) が正しく設定されていない場合、メモリ破壊や検索エラーが発生する可能性があります。

保守担当者向けメモ

  • NAMEDATALEN のサイズ変更や remoteConn 構造体の定義変更に伴い、本構造体のメモリレイアウトが影響を受けるため注意してください。
