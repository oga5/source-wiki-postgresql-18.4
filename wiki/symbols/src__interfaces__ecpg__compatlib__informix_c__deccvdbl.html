deccvdbl

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/compatlib/informix.c:245

目的

double 型の値を Informix 互換の decimal 型構造体に変換します。

シグネチャまたは形状

int deccvdbl(double dbl, decimal *np)

入力と出力

  • **入力**:
  • dbl: 変換元の double 型の値。
  • np: 変換結果を格納する decimal 型構造体へのポインタ。
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時は ECPG_INFORMIX_OUT_OF_MEMORY などのエラーコードを返します。

前提条件と不変条件

  • np は有効な decimal 構造体を指している必要があります。
  • 内部的に PGTYPES ライブラリのメモリ管理機能を使用するため、PGTYPESnumeric_new() が成功する必要があります。

アルゴリズムまたは振る舞い

1. rsetnull を呼び出し、出力先 np を Informix 形式の NULL 値として初期化します。 2. risnull を使用して入力 dbl が NULL か判定し、NULL の場合は 0 を返して終了します。 3. PGTYPESnumeric_new() で一時的な numeric 型の領域を確保します。確保に失敗した場合は ECPG_INFORMIX_OUT_OF_MEMORY を返します。 4. PGTYPESnumeric_from_double() を使用して double を numeric に変換します。 5. 変換が成功した場合、PGTYPESnumeric_to_decimal() を使用して numeric を decimal 型構造体 np に格納します。 6. 最後に PGTYPESnumeric_free() で一時領域を解放し、処理結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/include/ecpg_informix.h に宣言が存在します。
  • 具体的な呼び出し元コードの詳細は提供されていません。
  • **呼び出し先**:
  • rsetnull, risnull
  • PGTYPESnumeric_new, PGTYPESnumeric_from_double, PGTYPESnumeric_to_decimal, PGTYPESnumeric_free

関連データ構造

  • decimal: Informix 互換の数値型構造体。
  • numeric: PostgreSQL 内部の数値型構造体。

失敗モードと境界条件

  • 入力が NULL の場合、0 を返して正常終了とみなされます。
  • メモリ不足により PGTYPESnumeric_new() が失敗した場合、ECPG_INFORMIX_OUT_OF_MEMORY が返されます。
  • PGTYPES ライブラリの変換関数がエラーを返した場合、そのエラーコードがそのまま呼び出し元へ伝播されます。

保守担当者向けメモ

  • 本関数は PGTYPES ライブラリを介して型変換を行うラッパーです。Informix 互換の decimal 型と PostgreSQL の numeric 型の橋渡しを行うため、変換ロジックの変更は PGTYPES 側の仕様に依存します。
