dsa_allocate_extended

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/dsa.c:670

目的

動的共有メモリ領域（DSA）から指定されたサイズのメモリを割り当て、その位置を示す dsa_pointer を返します。

シグネチャまたは形状

dsa_pointer dsa_allocate_extended(dsa_area *area, size_t size, int flags);

入力と出力

  • **入力**:
  • area: 割り当て対象の DSA 領域へのポインタ。
  • size: 割り当てるバイト数。
  • flags: 動作を指定するビットフラグ（DSA_ALLOC_HUGE, DSA_ALLOC_ZERO, DSA_ALLOC_NO_OOM）。
  • **出力**:
  • 成功時: 割り当てられたメモリを指す dsa_pointer。
  • 失敗時: InvalidDsaPointer（DSA_ALLOC_NO_OOM 指定時）。

前提条件と不変条件

  • size > 0 であること。
  • flags に応じて、AllocSizeIsValid または AllocHugeSizeIsValid によるサイズ検証を通過すること。
  • 内部のロック取得順序（DSA_AREA_LOCK → DSA_SCLASS_LOCK）を厳守すること。

アルゴリズムまたは振る舞い

1. **サイズ判定**:

  • 最大サイズクラスを超える場合: FreePageManager を使用し、ページ単位で直接割り当てを行う。get_best_segment または make_new_segment でセグメントを確保し、FreePageManagerGet でページを取得する。
  • それ以外の場合: サイズクラスに応じたプールから alloc_object を使用して割り当てる。

    • 2. **初期化**: DSA_ALLOC_ZERO フラグが指定されている場合、割り当てられたメモリ領域を memset で 0 クリアする。 3. **エラー処理**: 割り当て失敗時、DSA_ALLOC_NO_OOM フラグがない場合は ereport(ERROR) でプロセスを終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: dshash.c、tidbitmap.c、pgstat_shmem.c 等から呼び出される。
  • **呼び出し先**: alloc_object、get_best_segment、make_new_segment、FreePageManagerGet、init_span、dsa_get_address、dsa_free。

関連データ構造

  • dsa_area: 領域管理構造体。
  • dsa_pointer: 共有メモリ内の相対位置を示すポインタ。
  • dsa_area_pool: サイズクラスごとのメモリプール。
  • dsa_segment_map: セグメント管理用マップ。

失敗モードと境界条件

  • **メモリ不足**: 領域の拡張が不可能な場合、InvalidDsaPointer を返すかエラーを発生させる。
  • **不正なサイズ**: AllocSizeIsValid 等の制約に違反する場合、elog(ERROR) を発生させる。
  • **ページ割り当て失敗**: FreePageManagerGet が失敗した場合、FATAL エラーでプロセスを停止する。

保守担当者向けメモ

  • dsa_pointer は絶対アドレスではないため、アクセスには必ず dsa_get_address() を経由してください。
  • 大規模割り当て（DSA_SCLASS_SPAN_LARGE）は、通常のプール割り当てとは異なるパスを通るため、修正時は両方のロジックに注意してください。
