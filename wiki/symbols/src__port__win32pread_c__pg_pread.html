pg_pread

定義

  • 種別: function
  • 定義箇所: src/port/win32pread.c:19

目的

Windows環境において、POSIX標準の pread(2) 関数をエミュレートします。ファイルポインタを移動させずに、指定したオフセットからデータを読み取る機能を提供します。

シグネチャまたは形状

ssize_t pg_pread(int fd, void *buf, size_t size, off_t offset)

入力と出力

  • **入力**:
  • fd: ファイル記述子。
  • buf: 読み取ったデータを格納するバッファ。
  • size: 読み取るバイト数（最大1GBに制限）。
  • offset: 読み取りを開始するファイル内の位置。
  • **出力**:
  • 成功時: 実際に読み取ったバイト数を返します。
  • 失敗時: -1 を返し、errno を設定します。EOFに達した場合は 0 を返します。

前提条件と不変条件

  • size は 1GB を超えることはできません（DWORD 型のオーバーフロー防止のため、Min(size, 1024 * 1024 * 1024) が適用されます）。
  • fd は有効なWindowsファイルハンドルに変換可能である必要があります。

アルゴリズムまたは振る舞い

1. _get_osfhandle(fd) を使用して、Cランタイムのファイル記述子からWindowsネイティブのファイルハンドルを取得します。 2. OVERLAPPED 構造体の Offset フィールドに offset を設定します。 3. ReadFile APIを呼び出し、データを読み取ります。この際、OVERLAPPED 構造体を使用することで、ファイルポインタを更新せずに指定位置からの読み取りを行います。 4. ReadFile が失敗した場合、エラーコードが ERROR_HANDLE_EOF であれば 0 を返します。それ以外の場合は _dosmaperr を呼び出し、WindowsエラーをPOSIX準拠の errno に変換して -1 を返します。

関連データ構造

  • OVERLAPPED: Windowsの非同期I/O構造体。ここではオフセット指定のために使用されます。

失敗モードと境界条件

  • _get_osfhandle が無効なハンドルを返した場合、errno を EBADF に設定して -1 を返します。
  • ReadFile が失敗した場合、Windowsのエラーコードに基づき errno が設定されます。
  • size が 1GB を超える場合、自動的に 1GB に切り詰められます。

保守担当者向けメモ

  • ReadFile に OVERLAPPED 構造体を渡す際、ファイルポインタ自体は移動しませんが、Windowsのファイルシステムレベルでの挙動には注意が必要です。
  • 本関数は src/port/win32pwrite.c の書き込み操作と対になる実装です。
