defGetTypeLength

定義

  • 種別: function
  • 定義箇所: src/backend/commands/define.c:298

目的

DefElem ノードからデータ型の長さを表す整数値を取得します。特に「可変長」を意味する特殊な指定を解釈するために使用されます。

シグネチャまたは形状

int defGetTypeLength(DefElem *def)

入力と出力

  • **入力**: DefElem *def（解析対象の定義要素）
  • **出力**: int 型の長さ。可変長の場合は -1 を返します。

前提条件と不変条件

  • def および def->arg が NULL でないことが期待されます。def->arg が NULL の場合はエラーを報告します。

アルゴリズムまたは振る舞い

1. def->arg のノードタグを判定し、以下のいずれかの処理を行います。

  • T_Integer: 値をそのまま返します。
  • T_Float: 整数値が必要である旨のエラーを報告します。
  • T_String: 文字列が "variable" であれば -1 を返します。
  • T_TypeName: 型名が "variable" であれば -1 を返します。
  • T_List: 演算子名とみなして処理を継続します（現状ではエラーへ至ります）。

    • 2. 上記のいずれにも該当しない、または変換できない場合は ERRCODE_SYNTAX_ERROR を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/typecmds.c:360
  • **呼び出し先**:
  • intVal() (マクロ)
  • strVal() (マクロ)
  • TypeNameToString()
  • defGetString()
  • ereport() / elog()

関連データ構造

  • DefElem: SQL コマンドのオプション引数を保持する構造体。
  • Node: def->arg が指す汎用ノード。

失敗モードと境界条件

  • def->arg が NULL: ERRCODE_SYNTAX_ERROR を発生。
  • T_Float が指定された場合: ERRCODE_SYNTAX_ERROR を発生。
  • 認識できないノード型: elog(ERROR) により異常終了。
  • 上記以外の無効な引数: ERRCODE_SYNTAX_ERROR を発生。

保守担当者向けメモ

  • 本関数は typecmds.c 等でデータ型の定義を行う際のオプション解析に依存しています。
  • "variable" という文字列の判定には pg_strcasecmp を使用しており、大文字小文字を区別しません。
