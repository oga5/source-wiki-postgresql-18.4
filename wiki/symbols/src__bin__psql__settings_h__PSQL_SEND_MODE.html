PSQL_SEND_MODE

定義

  • 種別: enum
  • 定義箇所: src/bin/psql/settings.h:71

目的

psql クライアントがサーバーに対してクエリやコマンドを送信する際のプロトコル動作（通常クエリ、拡張プロトコル、パイプラインモード等）を制御・識別するために使用されます。

シグネチャまたは形状

typedef enum
{
	PSQL_SEND_QUERY,
	PSQL_SEND_EXTENDED_CLOSE,
	PSQL_SEND_EXTENDED_PARSE,
	PSQL_SEND_EXTENDED_QUERY_PARAMS,
	PSQL_SEND_EXTENDED_QUERY_PREPARED,
	PSQL_SEND_PIPELINE_SYNC,
	PSQL_SEND_START_PIPELINE_MODE,
	PSQL_SEND_END_PIPELINE_MODE,
	PSQL_SEND_FLUSH,
	PSQL_SEND_FLUSH_REQUEST,
	PSQL_SEND_GET_RESULTS,
} PSQL_SEND_MODE;

入力と出力

  • 入力: PsqlSettings 構造体内の send_mode フィールドとして保持されます。
  • 出力: クエリ実行ロジックにおいて、現在の送信モードに応じた libpq 関数呼び出しの分岐条件として機能します。

前提条件と不変条件

  • PsqlSettings 構造体の一部として管理されます。
  • 特定のモード（パイプライン関連など）への遷移は、サーバー側の対応状態と整合している必要があります。

アルゴリズムまたは振る舞い

  • PSQL_SEND_QUERY: 通常のクエリ送信。
  • PSQL_SEND_EXTENDED_*: 拡張クエリプロトコル（Parse, Bind, Execute 等）の各フェーズを実行。
  • PSQL_SEND_PIPELINE_*: パイプラインモードの開始、終了、同期を実行。
  • PSQL_SEND_FLUSH / PSQL_SEND_FLUSH_REQUEST: バッファのフラッシュ制御。
  • PSQL_SEND_GET_RESULTS: サーバーからの結果取得フェーズ。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/common.c において、クエリ処理後や送信リクエスト時に参照されます。
  • 呼び出し先: 本シンボル自体は列挙型であり、直接の関数呼び出しは行いません。

関連データ構造

  • PsqlSettings: send_mode フィールドを保持するグローバル設定構造体。

失敗モードと境界条件

  • 不適切なモード遷移（例：パイプラインモード外での PSQL_SEND_PIPELINE_SYNC 指定など）が発生した場合の挙動は、呼び出し側のロジックに依存します。

保守担当者向けメモ

  • pset.send_mode として使用されます。クエリ実行のフロー制御において重要な役割を果たすため、新しい送信プロトコルを追加する際は、この列挙型に適切な値を追加し、common.c 等の処理ロジックを更新してください。
