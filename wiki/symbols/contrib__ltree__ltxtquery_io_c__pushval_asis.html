pushval_asis

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltxtquery_io.c:181

目的

検索クエリの解析中に、オペランド（値）を QPRS_STATE が管理する内部バッファへ追加し、対応する ITEM 構造体をクエリツリーに登録する。

シグネチャまたは形状

static bool pushval_asis(QPRS_STATE *state, int type, char *strval, int lenval, uint16 flag)

入力と出力

  • **入力**:
  • state: 解析状態を保持する QPRS_STATE 構造体へのポインタ。
  • type: 追加する要素の型。
  • strval: 追加する文字列値。
  • lenval: 文字列の長さ。
  • flag: オペランドに関連付けられたフラグ。
  • **出力**:
  • 成功時は true、失敗時は false を返す。

前提条件と不変条件

  • lenval は 0xffff 以下である必要がある。
  • state->op バッファは、文字列の追加に伴い repalloc によって動的に拡張される可能性がある。
  • 成功時には state->curop が更新され、文字列がヌル終端された状態でバッファに格納される。

アルゴリズムまたは振る舞い

1. lenval が 0xffff を超える場合、エラーを報告して false を返す。 2. pushquery を呼び出し、クエリの ITEM 配列に新しい要素を登録する。 3. state->op バッファの残容量を確認し、不足している場合は repalloc でバッファサイズを倍増させる。 4. strval を state->curop が指す位置にコピーし、ヌル文字を付与する。 5. state->curop ポインタと state->sumlen を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/ltree/ltxtquery_io.c 内のクエリ解析ロジック（例: ltxtq_in 関連の処理）。
  • **呼び出し先**: pushquery（ITEM 登録用）、repalloc（メモリ拡張用）、ltree_crc32_sz（ハッシュ計算用）。
  • *注: src/backend/utils/adt/tsquery.c に同名のシンボルへの言及があるが、本ファイルにおける呼び出し関係は ltxtquery_io.c 内の処理に限定される。*

関連データ構造

  • QPRS_STATE: 解析中の状態（バッファ、演算子スタック、エラーコンテキスト）を保持する構造体。
  • ITEM: ltxtquery 内部で保持される、演算子とオペランドの固定長配列要素。

失敗モードと境界条件

  • lenval > 0xffff: ERRCODE_INVALID_PARAMETER_VALUE エラーが発生する。
  • メモリ不足: repalloc が失敗した場合、PostgreSQL のメモリ管理機構によりエラーがスローされる。

保守担当者向けメモ

  • 本関数は ltxtquery の構築においてオペランドをバッファへ書き込む重要な役割を担う。
  • state->op の再割り当て後に state->curop を再計算するロジックが含まれているため、バッファ操作の変更時には注意が必要。
