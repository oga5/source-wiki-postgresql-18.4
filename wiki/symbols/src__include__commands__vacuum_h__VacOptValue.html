VacOptValue

定義

  • 種別: enum
  • 定義箇所: src/include/commands/vacuum.h:200

目的

VACUUM コマンドの各オプション（index_cleanup や truncate など）における設定値の有効・無効・自動判定の状態を表現するために使用されます。

シグネチャまたは形状

typedef enum VacOptValue
{
    VACOPTVALUE_UNSPECIFIED = 0,
    VACOPTVALUE_AUTO,
    VACOPTVALUE_DISABLED,
    VACOPTVALUE_ENABLED,
} VacOptValue;

入力と出力

  • 入力: SQL コマンドのオプション引数から変換された値を受け取ります。
  • 出力: VacuumParams 構造体のメンバとして保持され、VACUUM 処理の実行ロジックにおける条件分岐に使用されます。

前提条件と不変条件

  • VACOPTVALUE_UNSPECIFIED（値 0）は、ユーザーが明示的にオプションを指定しなかった場合のデフォルト状態として扱われます。

アルゴリズムまたは振る舞い

  • VACUUM 実行時、各オプションに対してこの列挙型を用いることで、ユーザー指定の有無（UNSPECIFIED）と、指定された場合の動作（ENABLED/DISABLED）、あるいはシステムによる自動判断（AUTO）を区別します。

呼び出し元と呼び出し先

  • 呼び出し元（推測）: src/backend/commands/vacuum.c 内の get_vacoptval_from_boolean 関数が、SQL の DefElem から本型への変換を行っています。
  • 呼び出し先: VacuumParams 構造体のメンバ（index_cleanup, truncate）として参照されます。

関連データ構造

  • VacuumParams: VACUUM コマンドのパラメータを保持する構造体であり、本型をメンバとして含みます。

失敗モードと境界条件

  • 該当なし（列挙型のため、定義外の値が代入された場合の動作は未定義です）。

保守担当者向けメモ

  • 新しい VACUUM オプションを追加する際、三値論理（有効/無効/自動）が必要な場合は本型を再利用してください。
  • VACOPTVALUE_UNSPECIFIED が 0 であるため、構造体の初期化時にゼロクリアを行うと自動的に「未指定」状態となります。
