ExecUnique

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeUnique.c:45

目的

ソート済みの入力ストリームから隣接する重複行を排除し、一意な結果セットを生成する。

シグネチャまたは形状

static TupleTableSlot *ExecUnique(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（UniqueState 構造体としてキャストされる実行ノードの状態）。
  • **出力**: 重複排除された TupleTableSlot *。サブプランが終了した場合は NULL を返す。

前提条件と不変条件

  • 入力データはソート済みであること。ソートされていない場合、正しく重複排除されない。
  • UniqueState ノードの初期化が完了しており、比較関数（node->eqfunction）が準備されていること。

アルゴリズムまたは振る舞い

1. CHECK_FOR_INTERRUPTS() を実行し、割り込みを確認する。 2. ループ内でサブプランから ExecProcNode を呼び出し、タプルを取得する。 3. 取得したタプルが NULL の場合、resultTupleSlot をクリアして終了する。 4. resultTupleSlot が空（最初のタプル）であれば、そのタプルを保持してループを抜ける。 5. 既に保持しているタプルと新しいタプルを ExecQualAndReset で比較する。

  • 一致する場合: 重複とみなし、次のタプルを取得するためにループを継続する。
  • 不一致の場合: 新しいタプルを保持し、ループを抜ける。

    • 6. 最後に、保持したタプルを ExecCopySlot でコピーして返却する。これは、サブプランが次のフェッチでタプルを上書きする可能性があるためである。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecProcNode から呼び出される（uniquestate->ps.ExecProcNode = ExecUnique; による登録）。
  • **呼び出し先**:
  • ExecProcNode(outerPlan): サブプランからのタプル取得。
  • ExecQualAndReset: タプルの比較。
  • ExecCopySlot: タプルのコピー。
  • ExecClearTuple: スロットのクリア。

関連データ構造

  • UniqueState: ノードの実行時状態を保持する構造体。
  • TupleTableSlot: タプルを格納するスロット。
  • PlanState: 実行プランのノード状態。

失敗モードと境界条件

  • **サブプランの終了**: サブプランがタプルを返さなくなった場合、NULL を返して正常終了する。
  • **ソート順序の不備**: 入力がソートされていない場合、隣接する重複のみが排除され、離れた位置にある重複は排除されない。

保守担当者向けメモ

  • 本関数は射影（Projection）を行わないため、ps_ProjInfo は使用されない。
  • ExecCopySlot によるコピーは、サブプランのメモリ再利用によるデータ破壊を防ぐために必須である。
