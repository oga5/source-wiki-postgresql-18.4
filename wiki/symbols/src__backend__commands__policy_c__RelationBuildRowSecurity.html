RelationBuildRowSecurity

定義

  • 種別: function
  • 定義箇所: src/backend/commands/policy.c:192

目的

テーブルの relcache エントリに対して、行レベルセキュリティ（RLS）ポリシー情報を構築し、ロードする。

シグネチャまたは形状

void RelationBuildRowSecurity(Relation relation)

入力と出力

  • **入力**: relation (対象テーブルの Relation 構造体)
  • **出力**: なし（副作用として relation->rd_rsdesc に RowSecurityDesc を設定）

前提条件と不変条件

  • relation は有効なリレーションであること。
  • pg_policy カタログへのアクセス権限が必要。
  • メモリ管理のため、構築された RowSecurityDesc は CacheMemoryContext 配下に配置される。

アルゴリズムまたは振る舞い

1. ポリシー情報を保持するための専用メモリコンテキストを作成する。 2. pg_policy カタログを polrelid インデックスを用いてスキャンし、対象テーブルに関連する全ポリシーを取得する。 3. 各ポリシーについて以下を実行する:

  • RowSecurityPolicy 構造体を割り当て、名前、コマンド種別、許可/制限フラグをコピーする。
  • polroles（ロール配列）、polqual（USING句）、polwithcheck（WITH CHECK句）をカタログから抽出し、メモリコンテキストにコピーする。
  • 式ツリー内の SubLink の有無を判定する。
  • ポリシーリストを rsdesc->policies に追加する（逆順で構築される）。

    • 4. スキャン終了後、メモリコンテキストの親を CacheMemoryContext に変更し、永続化する。 5. 構築した rsdesc を relation->rd_rsdesc に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/relcache.c 内の relcache 構築プロセスから呼び出される。
  • **呼び出し先**: table_open, systable_beginscan, systable_getnext, systable_endscan, table_close, DatumGetArrayTypePCopy, stringToNode, checkExprHasSubLink 等。

関連データ構造

  • RowSecurityDesc: RLSポリシーの集合を保持する構造体。
  • RowSecurityPolicy: 個別のポリシー定義（コマンド種別、ロール、式ツリー等）。
  • pg_policy: ポリシー定義を格納するシステムカタログ。

失敗モードと境界条件

  • pg_policy.polroles が NULL の場合、エラーを発生させる。
  • メモリ不足時には MemoryContext 関連の関数がエラーを投げる可能性がある。
  • ポリシーが存在しない場合、rsdesc は空のリストを持つ。

保守担当者向けメモ

  • 本関数は relcache 構築時に実行されるため、パフォーマンスとメモリリークの回避が極めて重要である。
  • ポリシーの適用ロジック自体は src/backend/rewrite/rowsecurity.c に存在するため、本関数はあくまで「定義のロード」に専念すること。
  • リストが逆順に構築される歴史的経緯があるため、順序に依存する処理を追加する場合は注意が必要。
