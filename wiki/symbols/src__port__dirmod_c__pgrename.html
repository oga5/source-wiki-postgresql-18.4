pgrename

定義

  • 種別: function
  • 定義箇所: src/port/dirmod.c:51

目的

Windows 環境において、ファイル名の変更（リネーム）操作を堅牢に行うためのラッパー関数です。他プロセスによるファイルロックや共有違反が発生した場合に、短時間の再試行を行うことで POSIX 互換の rename 挙動をエミュレートします。

シグネチャまたは形状

int pgrename(const char *from, const char *to)

入力と出力

  • **入力**:
  • from: 移動元ファイルのパス（文字列）
  • to: 移動先ファイルのパス（文字列）
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時は -1 を返し、errno が適切に設定されます。

前提条件と不変条件

  • Windows 環境（WIN32 かつ !__CYGWIN__）では、MoveFileEx を使用して MOVEFILE_REPLACE_EXISTING フラグ付きで実行されます。
  • 非 Windows 環境では、標準の rename 関数を呼び出します。
  • 共有違反等のエラーが発生した場合、最大 10 秒間（100ms 間隔で最大 100 回）の再試行を行います。

アルゴリズムまたは振る舞い

1. MoveFileEx（Windows）または rename（その他）を実行します。 2. 失敗した場合、エラーコードを確認します。

  • Windows の場合: ERROR_ACCESS_DENIED、ERROR_SHARING_VIOLATION、ERROR_LOCK_VIOLATION のいずれかであれば再試行対象と判断します。
  • その他: errno が EACCES であれば再試行対象と判断します。

    • 3. 再試行対象外のエラー、または 10 秒経過しても成功しない場合は -1 を返します。 4. 再試行が必要な場合は pg_usleep(100000) で待機し、ループを継続します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/pg_upgrade.h にて pg_mv_file としてマクロ定義されています。
  • src/include/port.h にて rename マクロとして再定義されており、広範なコードから利用されています。
  • **呼び出し先**:
  • MoveFileEx (Windows API)
  • rename (標準ライブラリ)
  • _dosmaperr (Windows エラーコード変換)
  • pg_usleep (待機処理)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • **共有違反**: 他のプロセスがファイルをロックしている場合、最大 10 秒間リトライを試みます。
  • **タイムアウト**: 10 秒経過してもロックが解除されない場合は失敗とみなされます。
  • **権限エラー**: 再試行対象外のアクセス拒否エラーが発生した場合は即座に失敗します。

保守担当者向けメモ

  • 本関数は src/include/port.h で rename をオーバーライドしているため、Windows 上でのファイル操作の挙動を根本的に制御しています。
  • Windows SDK のバージョンやアンチウイルスソフトの挙動により、ERROR_LOCK_VIOLATION 等の追加が必要になる場合があります。
  • Cygwin 環境では標準の rename が使用されるため、本関数のリトライロジックは適用されません。
