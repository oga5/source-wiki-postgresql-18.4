InlineSearchPath

定義

  • 種別: typedef
  • 定義箇所: src/backend/jit/llvm/llvmjit_inline.cpp:67

目的

JIT コンパイルにおける関数インライン展開の際、外部モジュールのビットコードインデックスを検索するための順序付きリストを保持します。

シグネチャまたは形状

typedef llvm::SmallVector<llvm::ModuleSummaryIndex *, 2> InlineSearchPath;

入力と出力

  • **入力**: llvm::ModuleSummaryIndex へのポインタを要素として格納します。
  • **出力**: インライン化対象の関数定義を探索するための検索パスとして機能します。

前提条件と不変条件

  • llvm::SmallVector を使用しているため、初期容量として 2 つのポインタを保持可能です。
  • 検索パス内の各 ModuleSummaryIndex は、インライン化の計画策定時に有効である必要があります。

アルゴリズムまたは振る舞い

  • summaries_for_guid 関数において、特定の GUID に対応する関数定義を検索するための探索対象として利用されます。
  • add_module_to_inline_search_path 関数を通じて、検索対象のモジュールインデックスがこのパスに追加されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • llvm_build_inline_plan (インライン化計画の構築)
  • summaries_for_guid (インデックス検索)
  • add_module_to_inline_search_path (パスへのモジュール追加)
  • **呼び出し先**:
  • LLVM の SmallVector API を介して要素の追加や走査を行います。

関連データ構造

  • llvm::ModuleSummaryIndex: 検索パスの各要素が参照する LLVM のモジュール要約情報。
  • InlineWorkList: インライン化対象の関数を管理するリスト。

失敗モードと境界条件

  • 検索パスが空の場合、外部関数の定義を解決できず、インライン化がスキップされます。
  • llvm::SmallVector の容量を超えて要素を追加した場合、動的なメモリ再割り当てが発生します。

保守担当者向けメモ

  • 本構造体は llvmjit_inline.cpp 内で静的に定義された defaultSearchPath 等の管理に使用されます。
  • キャッシュされた ModuleSummaryIndex の生存期間と整合性に注意してください。llvm_inline_reset_caches が呼び出された際、このパスに含まれるポインタが無効化される可能性があります。
