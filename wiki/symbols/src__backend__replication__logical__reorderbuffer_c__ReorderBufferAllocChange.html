ReorderBufferAllocChange

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/reorderbuffer.c:505

目的

論理デコーディング処理で使用される ReorderBufferChange 構造体のメモリを確保し、初期化を行う。

シグネチャまたは形状

ReorderBufferChange *ReorderBufferAllocChange(ReorderBuffer *rb)

入力と出力

  • **入力**: rb (ReorderBuffer*) - 変更レコードを管理する ReorderBuffer 構造体へのポインタ。
  • **出力**: ReorderBufferChange* - rb->change_context メモリコンテキストから確保され、ゼロクリアされた ReorderBufferChange 構造体へのポインタ。

前提条件と不変条件

  • rb は有効な ReorderBuffer 構造体を指していること。
  • rb->change_context がメモリ確保のために初期化されていること。

アルゴリズムまたは振る舞い

1. rb->change_context をメモリコンテキストとして使用し、sizeof(ReorderBufferChange) 分のメモリを確保する。 2. 確保したメモリ領域全体を memset で 0 に初期化する。 3. 初期化済みのポインタを呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/decode.c (複数箇所)
  • src/backend/replication/logical/reorderbuffer.c (複数箇所)
  • **呼び出し先**:
  • MemoryContextAlloc (PostgreSQL 内部メモリ管理関数)
  • memset (標準ライブラリ関数)

関連データ構造

  • ReorderBuffer: デコーディングセッションの状態を保持する構造体。
  • ReorderBufferChange: 論理デコーディングにおける個別の変更内容（INSERT/UPDATE/DELETE 等）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAlloc がエラーを発生させ、処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数は ReorderBufferChange の確保に特化している。メモリ管理コンテキストの分離（change_context）により、トランザクション終了時やデコーディングセッション終了時の効率的な一括解放を可能にしている。
