upgrade_task_add_step

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/task.c:150

目的

UpgradeTask オブジェクトに対して、実行すべきクエリと、その結果を処理するためのコールバック関数をステップとして追加します。

シグネチャまたは形状

void upgrade_task_add_step(UpgradeTask *task, const char *query,
                           UpgradeTaskProcessCB process_cb, bool free_result,
                           void *arg);

入力と出力

  • task: ステップを追加する対象の UpgradeTask 構造体へのポインタ。
  • query: 実行する SQL クエリ文字列。
  • process_cb: クエリ結果を処理するコールバック関数。
  • free_result: クエリ実行後に結果セットを解放するかどうかを示すフラグ。
  • arg: コールバック関数に渡す任意のデータへのポインタ。
  • 出力: なし（void）。

前提条件と不変条件

  • task->steps 配列は pg_realloc によって動的に拡張されます。
  • 追加されたステップは、task->queries バッファにセミコロンで区切られて連結されます。
  • 複数のステップを追加した場合、それらは同一のデータベース接続上で順次実行されます。

アルゴリズムまたは振る舞い

1. task->steps 配列のサイズを num_steps の増加に合わせて pg_realloc で拡張します。 2. 新しいステップ（UpgradeTaskStep）を配列の末尾に確保し、引数で渡されたコールバック関数、フラグ、引数ポインタを格納します。 3. task->queries が保持する PQExpBuffer に、指定された query をセミコロン付きで追記します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_upgrade/check.c、src/bin/pg_upgrade/function.c、src/bin/pg_upgrade/info.c の各関数から呼び出されています。
  • 呼び出し先: pg_realloc（メモリ確保）、appendPQExpBuffer（クエリ文字列の連結）。

関連データ構造

  • UpgradeTask: タスク全体を管理する構造体。
  • UpgradeTaskStep: 個別のクエリステップと処理ロジックを保持する構造体。

失敗モードと境界条件

  • pg_realloc がメモリ不足により失敗した場合、プログラムは終了する可能性があります。

保守担当者向けメモ

  • 本関数は、タスクの実行前（upgrade_task_run 呼び出し前）にステップを定義するために使用されます。
  • 各タスクの最初のステップとして ALWAYS_SECURE_SEARCH_PATH_SQL が自動的に追加される仕様があるため、ステップの順序と内容には注意してください。
