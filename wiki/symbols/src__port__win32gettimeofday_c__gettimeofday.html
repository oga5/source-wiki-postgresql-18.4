gettimeofday

定義

  • 種別: function
  • 定義箇所: src/port/win32gettimeofday.c:52

目的

Windows環境においてPOSIX標準の gettimeofday() をエミュレートし、システム時刻を struct timeval 形式で取得する。

シグネチャまたは形状

int gettimeofday(struct timeval *tp, void *tzp);

入力と出力

  • **入力**:
  • tp: 現在時刻を格納する struct timeval へのポインタ。
  • tzp: タイムゾーン情報。POSIX仕様上は非推奨であり、PostgreSQL内では NULL であることが期待される。
  • **出力**:
  • 戻り値: 常に 0 を返す。
  • tp が指す構造体に、1970年1月1日からの経過秒数（tv_sec）およびマイクロ秒数（tv_usec）が書き込まれる。

前提条件と不変条件

  • tzp 引数は必ず NULL でなければならない（Assert(tzp == NULL) により検証）。
  • Windows API GetSystemTimePreciseAsFileTime が利用可能な環境であること。

アルゴリズムまたは振る舞い

1. GetSystemTimePreciseAsFileTime を呼び出し、現在のシステム時刻を FILETIME 形式で取得する。 2. FILETIME 値を ULARGE_INTEGER に変換し、定数 epoch（1601年と1970年の差分）を減算する。 3. 差分値を FILETIME_UNITS_PER_SEC で除算し、秒単位の値を算出する。 4. 差分値の剰余を FILETIME_UNITS_PER_USEC で除算し、マイクロ秒単位の値を算出する。 5. 計算結果を tp に格納し、0 を返す。

関連データ構造

  • struct timeval: POSIX標準の時刻構造体。
  • FILETIME: Windowsの時刻形式（1601年1月1日からの100ナノ秒単位）。
  • ULARGE_INTEGER: FILETIME を算術演算するために使用される64ビット整数型。

失敗モードと境界条件

  • tzp が NULL でない場合、アサーションによりプログラムが停止する可能性がある。
  • GetSystemTimePreciseAsFileTime は比較的新しいWindows APIであり、古いOS環境では動作しない可能性がある。
  • 本関数は高精度な「経過時間」計測用ではないため、そのような用途には適さない。

保守担当者向けメモ

  • epoch および FILETIME_UNITS_PER_SEC などの定数は、FILETIME からUNIXエポックへの変換に不可欠である。
  • 本関数はPOSIX互換レイヤーとして提供されており、Windows特有の時刻取得処理を隠蔽している。
