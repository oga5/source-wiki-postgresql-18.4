InitBufferManagerAccess

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/bufmgr.c:4007

目的

各バックエンドプロセスにおけるバッファ管理用のデータ構造を初期化し、プロセス終了時のクリーンアップ処理を登録する。

シグネチャまたは形状

void InitBufferManagerAccess(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数およびハッシュテーブルを初期化）

前提条件と不変条件

  • MyProc が初期化済みであること（Assert(MyProc != NULL) により検証）。
  • NBuffers および MaxBackends が設定済みであること。

アルゴリズムまたは振る舞い

1. **ピン制限の算出**: MaxProportionalPins を NBuffers / (MaxBackends + NUM_AUXILIARY_PROCS) として計算し、バックエンドごとのピン留め上限を設定する。 2. **参照カウント配列の初期化**: PrivateRefCountArray をゼロクリアする。 3. **ハッシュテーブルの作成**: PrivateRefCount を管理するためのハッシュテーブル PrivateRefCountHash を作成する。 4. **終了処理の登録**: on_shmem_exit を使用して、バックエンド終了時に AtProcExit_Buffers が呼び出されるよう登録する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/init/postinit.c:644
  • 呼び出し先: hash_create, on_shmem_exit, AtProcExit_Buffers（登録のみ）

関連データ構造

  • PrivateRefCountArray: バックエンドごとのピン留め回数を追跡する配列。
  • PrivateRefCountHash: PrivateRefCountEntry を格納するハッシュテーブル。
  • MaxProportionalPins: バックエンドが保持可能なピン留め数の上限値。

失敗モードと境界条件

  • MyProc が NULL の場合、アサーション失敗によりプロセスが停止する。
  • メモリ不足により hash_create が失敗した場合、PostgreSQL の標準的なメモリ割り当てエラーが発生する。

保守担当者向けメモ

  • 本関数はバックエンドの初期化フェーズで一度だけ実行される。
  • PrivateRefCountHash のサイズは現在 100 で固定されているが、必要に応じて調整可能である。
  • AtProcExit_Buffers は LWLock を使用するため、シャットダウン時の適切なタイミングで実行される必要がある。
