spg_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/spgdesc.c:19

目的

SP-GiSTインデックスのWALレコードの内容を人間が読める形式に変換し、StringInfo バッファへ出力する。主に pg_waldump 等のツールによるデバッグやリカバリ調査で使用される。

シグネチャまたは形状

void spg_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 出力文字列を格納する StringInfo へのポインタ。
  • record: 解析対象のWALレコード状態を保持する XLogReaderState へのポインタ。
  • **出力**: なし（buf を通じて文字列が追記される）。

前提条件と不変条件

  • record は有効なSP-GiSTのWALレコードを指していること。
  • XLogRecGetInfo で取得した情報から XLR_INFO_MASK を除外した値が、既知のSP-GiST操作タイプ（XLOG_SPGIST_ADD_LEAF 等）と一致すること。

アルゴリズムまたは振る舞い

1. XLogRecGetData を用いてWALレコードの生データポインタを取得する。 2. XLogRecGetInfo から補助フラグを除去し、操作タイプを特定する。 3. switch 文により操作タイプを判定し、対応する spgxlog 構造体（例: spgxlogAddLeaf）へキャストする。 4. 構造体内の各フィールド（オフセット、ブロック番号、フラグ等）を appendStringInfo を用いて buf に整形して書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h で定義されているリソースマネージャリストを通じて、WAL解析ツールから間接的に呼び出される。
  • **呼び出し先**: appendStringInfo などの文字列操作ユーティリティ。

関連データ構造

  • StringInfo: 文字列バッファ。
  • XLogReaderState: WALレコードの読み取り状態。
  • spgxlog... 構造体群（spgxlogAddLeaf, spgxlogMoveLeafs, spgxlogAddNode, spgxlogSplitTuple, spgxlogPickSplit, spgxlogVacuumLeaf, spgxlogVacuumRoot, spgxlogVacuumRedirect）: SP-GiSTのWALレコードデータ構造。

失敗モードと境界条件

  • 未知の操作タイプが渡された場合、switch 文で何も出力されず終了する。
  • 構造体のキャスト時にWALレコードのデータ長が不整合である場合、メモリ破壊や不正な値の読み取りが発生する可能性がある。

保守担当者向けメモ

  • 新しいWALレコードタイプをSP-GiSTに追加した際は、本関数の switch 文に必ず対応する case を追加すること。
  • XLR_INFO_MASK を用いて補助フラグを除外しているため、操作タイプ判定には純粋な識別子のみが使用される点に注意。
