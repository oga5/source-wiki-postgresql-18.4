目的

ECPG（Embedded SQL for C）プリプロセッサの機能テストを実行するエントリポイントです。特に EXEC SQL TYPE 指令による型定義と、ホスト変数としての配列の受け渡しが正しく動作することを検証します。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（コマンドライン引数は受け取らない）
  • 出力: 正常終了時に 0 を返却。標準出力にテスト結果を表示。

前提条件と不変条件

  • 実行前にデータベース ecpg1_regression が接続可能であること。
  • SQL 文の実行後、sqlca.sqlcode を確認し、負の値であれば sqlprint() を呼び出してエラーを表示する一貫したエラーハンドリングが行われること。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、データベースへ接続。 2. **DDL/DML**: test テーブルを作成し、データを挿入。各操作後に commit を実行。 3. **データ取得**: select * from test を実行し、結果をホスト変数（name, amount, letter）にバインド。 4. **反復処理**: sqlca.sqlerrd[2]（取得行数）に基づき、ループ内で各行のデータを整形して出力。 5. **終了処理**: テーブルの削除、トランザクションのコミット、およびデータベース接続の切断。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGdo, ECPGtrans, ECPGdisconnect, sqlprint, strncpy, printf

関連データ構造

  • sqlca: SQL 通信領域。実行結果のステータス確認に使用。
  • intarray: EXEC SQL TYPE で定義された int[6] 型のエイリアス。
  • string: char[8] 型のエイリアス。

失敗モードと境界条件

  • SQL 文の実行失敗時: sqlca.sqlcode < 0 となり、sqlprint() が診断情報を出力して処理を継続する。
  • データベース接続失敗時: ECPGconnect がエラーを返し、以降の SQL 操作が失敗する。

保守担当者向けメモ

  • 本関数はプリプロセッサによって自動生成されたテストコードです。直接編集せず、元の .pgc ファイルを修正してください。
  • sqlca.sqlerrd[2] を使用したループ処理は、ECPG における配列バインドの標準的な検証パターンです。
