RelationFindReplTupleSeq

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execReplication.c:354

目的

論理レプリケーションにおいて、インデックスを使用せずにシーケンシャルスキャンを用いて対象のタプルを検索し、指定されたモードでロックを取得します。

シグネチャまたは形状

bool RelationFindReplTupleSeq(Relation rel, LockTupleMode lockmode, TupleTableSlot *searchslot, TupleTableSlot *outslot)

入力と出力

  • **入力**:
  • rel: 対象のリレーション。
  • lockmode: 取得するロックモード（LockTupleMode）。
  • searchslot: 検索条件となるタプルを保持するスロット。
  • outslot: 検索結果のタプルを格納するスロット。
  • **出力**:
  • bool: タプルが見つかり、ロックに成功した場合は true、それ以外は false。

前提条件と不変条件

  • outslot のタプル記述子（TupleDesc）は、対象リレーション rel の記述子と一致している必要があります。
  • 実行には適切なスナップショットが必要です。

アルゴリズムまたは振る舞い

1. **スキャン開始**: table_beginscan を用いてテーブル全体のシーケンシャルスキャンを開始します。 2. **検索ループ**: table_scan_getnextslot で全行を走査し、tuples_equal を用いて searchslot と一致するタプルを探します。 3. **ロック待機**: 該当タプルが他トランザクションによってロックされている場合、XactLockTableWait で待機し、再試行（retry）します。 4. **ロック取得**: タプルが見つかった場合、table_tuple_lock を呼び出して指定された lockmode でロックを試みます。 5. **再試行判定**: should_refetch_tuple が真を返す場合（同時実行による更新等）、再度検索処理からやり直します。 6. **終了処理**: スキャンを終了し、使用したリソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/worker.c:2953 からの呼び出しが確認されています。
  • **呼び出し先**: table_beginscan, table_rescan, table_scan_getnextslot, table_tuple_lock, should_refetch_tuple, XactLockTableWait 等のテーブルアクセス層（AM）関数を呼び出します。

関連データ構造

  • Relation: 操作対象のテーブル情報。
  • TupleTableSlot: タプルデータの一時保持用構造体。
  • TM_FailureData: ロック失敗時の詳細情報。

失敗モードと境界条件

  • インデックスを使用しないため、テーブルサイズが大きい場合に非常に低速です。
  • 同時実行制御による競合が発生した場合、goto retry による再試行ループが発生します。

保守担当者向けメモ

  • 本関数はインデックスがない場合のフォールバックとして機能します。パフォーマンス上の理由から、実運用では適切なレプリカアイデンティティ（インデックス）の設定が強く推奨されます。
  • retry ラベルへのジャンプが多用されているため、状態遷移の整合性に注意してください。
