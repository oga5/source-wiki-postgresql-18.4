spgAddSearchItemToQueue

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgscan.c:107

目的

SP-GiST インデックスのスキャン処理において、次に探索すべきノード（SpGistSearchItem）を優先度付きキュー（scanQueue）に追加します。

シグネチャまたは形状

static void spgAddSearchItemToQueue(SpGistScanOpaque so, SpGistSearchItem *item)

入力と出力

  • **入力**:
  • so: スキャン状態を保持する SpGistScanOpaque 構造体。
  • item: キューに追加する SpGistSearchItem へのポインタ。
  • **出力**: なし（副作用としてキューの状態が更新されます）。

前提条件と不変条件

  • so->scanQueue が初期化済みである必要があります。
  • item は pairingheap で管理可能なノード（phNode）を含んでいる必要があります。

アルゴリズムまたは振る舞い

pairingheap_add を呼び出し、指定された item を so->scanQueue に挿入します。これにより、KNN 検索等の順序付き走査において、適切な優先順位でノードが管理されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgscan.c:146
  • src/backend/access/spgist/spgscan.c:587
  • src/backend/access/spgist/spgscan.c:737
  • **呼び出し先**:
  • pairingheap_add（外部ライブラリ関数）

関連データ構造

  • SpGistScanOpaque: スキャン状態を管理する構造体。
  • SpGistSearchItem: 走査対象のノード情報を保持する構造体。
  • pairingheap: 優先度付きキューの実装。

失敗モードと境界条件

  • メモリ不足時、pairingheap_add 内部でエラーが発生する可能性があります。
  • so または item が NULL の場合の挙動については、本ソースコード内では明示的なガードは記述されていません。

保守担当者向けメモ

  • 本関数は pairingheap への単なるラッパーです。キューの比較ロジックを変更する場合は、pairingheap_SpGistSearchItem_cmp を確認してください。
