tsm_system_time_handler

定義

  • 種別: function
  • 定義箇所: contrib/tsm_system_time/tsm_system_time.c:80

目的

PostgreSQL の TABLESAMPLE SYSTEM_TIME メソッドを登録するためのエントリポイントです。テーブルサンプリング API (TsmRoutine) を初期化し、各コールバック関数を割り当てます。

シグネチャまたは形状

Datum tsm_system_time_handler(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (PostgreSQL の関数呼び出し規約に基づく引数)
  • **出力**: TsmRoutine 構造体へのポインタを Datum 型として返す。

前提条件と不変条件

  • makeNode(TsmRoutine) によりメモリが確保されること。
  • parameterTypes に FLOAT8OID が設定されること。
  • repeatable_across_queries および repeatable_across_scans は、時間依存のサンプリングであるため false に設定される。

アルゴリズムまたは振る舞い

1. TsmRoutine 構造体を生成する。 2. サンプリングメソッドのメタデータ（パラメータ型、再実行可能性）を設定する。 3. 以下のコールバック関数を構造体に割り当てる:

  • SampleScanGetSampleSize: system_time_samplescangetsamplesize
  • InitSampleScan: system_time_initsamplescan
  • BeginSampleScan: system_time_beginsamplescan
  • NextSampleBlock: system_time_nextsampleblock
  • NextSampleTuple: system_time_nextsampletuple

    • 4. EndSampleScan には NULL を設定する。 5. 初期化済みの TsmRoutine を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の拡張機能ロードおよび TABLESAMPLE メソッド登録プロセス（PG_FUNCTION_INFO_V1 マクロ経由で参照される）。
  • **呼び出し先**: makeNode (PostgreSQL 内部関数)、および各コールバック関数（system_time_... 系関数）。

関連データ構造

  • TsmRoutine: テーブルサンプリングメソッドのインターフェースを定義する構造体。

失敗モードと境界条件

  • メモリ不足時には makeNode がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は TABLESAMPLE メソッドの登録用であり、実行時のロジックは割り当てられた各コールバック関数に委譲されます。
  • EndSampleScan が NULL であるため、スキャン終了時に特別なクリーンアップ処理は不要です。
