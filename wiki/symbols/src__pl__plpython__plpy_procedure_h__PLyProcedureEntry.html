PLyProcedureEntry

定義

  • 種別: struct
  • 定義箇所: src/pl/plpython/plpy_procedure.h:60

目的

PL/Pythonにおけるプロシージャキャッシュ用ハッシュテーブルの各エントリを保持するために使用されます。

シグネチャまたは形状

typedef struct PLyProcedureEntry
{
	PLyProcedureKey key;		/* hash key */
	PLyProcedure *proc;
} PLyProcedureEntry;

入力と出力

  • **入力**: ハッシュテーブル検索用の PLyProcedureKey（関数OIDおよびリレーションOID）。
  • **出力**: 対応する PLyProcedure 構造体へのポインタ。

前提条件と不変条件

  • 本構造体は、PostgreSQLのプロセス内メモリ空間で管理されるハッシュテーブルの一部として使用されます。
  • proc フィールドは、キャッシュされたプロシージャの実行に必要なメタデータおよびコンパイル済みコードを保持します。

アルゴリズムまたは振る舞い

  • キャッシュ検索時、key を用いてハッシュテーブルからエントリが特定されます。
  • キャッシュヒットした場合、proc を通じてプロシージャの実行コンテキストが取得されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_procedure.c 内のハッシュテーブル管理ロジックが、sizeof(PLyProcedureEntry) を用いてエントリサイズを定義し、エントリの参照および初期化を行います。
  • **呼び出し先**: 直接的な関数呼び出しは行いません。

関連データ構造

  • PLyProcedureKey: キャッシュ検索のキーとして使用されます。
  • PLyProcedure: 本エントリが指し示す、プロシージャの実行に必要な全情報を保持する構造体です。

失敗モードと境界条件

  • キャッシュミスが発生した場合、PLyProcedureEntry は新規作成され、システムカタログから情報が読み込まれます。
  • メモリ管理については、PLyProcedure が保持する MemoryContext の解放とエントリの削除が整合している必要があります。

保守担当者向けメモ

  • 本構造体は plpy_procedure.c のハッシュテーブル実装と密接に結合しています。エントリの追加や削除を行う際は、メモリコンテキストのライフサイクルに注意してください。
