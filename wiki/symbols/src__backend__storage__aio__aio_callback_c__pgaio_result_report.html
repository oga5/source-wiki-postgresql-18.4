pgaio_result_report

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_callback.c:172

目的

非同期I/O操作の失敗結果を報告し、登録されたコールバックを通じてエラーログ出力や診断処理を実行する。

シグネチャまたは形状

void pgaio_result_report(PgAioResult result, const PgAioTargetData *target_data, int elevel);

入力と出力

  • result: 報告対象の PgAioResult 構造体。
  • target_data: I/O操作のターゲット情報。
  • elevel: elog に渡すエラーレベル（DEBUG1, LOG_SERVER_ONLY 等）。
  • 戻り値: なし。

前提条件と不変条件

  • result.status は PGAIO_RS_UNKNOWN および PGAIO_RS_OK 以外であること（成功時は報告不要）。
  • 指定された result.id に対応するコールバックが aio_handle_cbs に存在し、かつ report 関数ポインタが定義されていること。

アルゴリズムまたは振る舞い

1. result.id を用いて aio_handle_cbs 配列から対応する PgAioHandleCallbacksEntry を取得する。 2. result.status が正常終了でないことをアサートで確認する。 3. 該当するコールバックエントリに report 関数が定義されていない場合、ERROR を発生させる。 4. 登録されている report コールバック関数を呼び出し、引数として result、target_data、elevel を渡す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/buffer/bufmgr.c
  • src/backend/storage/smgr/md.c
  • src/backend/storage/smgr/smgr.c
  • src/test/modules/test_aio/test_aio.c
  • 呼び出し先:
  • ce->cb->report(): 登録された各コールバック固有の報告関数。

関連データ構造

  • PgAioResult: I/O操作の結果状態を保持。
  • PgAioHandleCallbacksEntry: コールバック関数群を保持する構造体。
  • aio_handle_cbs: 利用可能なコールバック定義を保持する静的配列。

失敗モードと境界条件

  • result.id が不正な場合、配列外参照が発生する可能性がある。
  • ce->cb->report が NULL の場合、elog(ERROR) によりプロセスが中断される。

保守担当者向けメモ

  • 本関数はI/Oエラーの可視化を目的としており、成功した操作に対して呼び出すことは想定されていない。
  • report コールバックの実装側で、elevel に応じた適切なログ出力を行う必要がある。
