inet_opr_codenum

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_selfuncs.c:853

目的

ネットワーク関連の演算子（inet / cidr 型用）を、包含関係に基づく数値コード（-2〜2）に変換し、選択率推定ロジックで統一的に扱えるようにします。

シグネチャまたは形状

static int inet_opr_codenum(Oid operator)

入力と出力

  • **入力**: operator (Oid) - 評価対象の演算子識別子。
  • **出力**: int - 演算子の種類に対応する以下の数値。
  • -2: OID_INET_SUP_OP (上位包含)
  • -1: OID_INET_SUPEQ_OP (上位包含または等価)
  • 0: OID_INET_OVERLAP_OP (オーバーラップ)
  • 1: OID_INET_SUBEQ_OP (下位包含または等価)
  • 2: OID_INET_SUB_OP (下位包含)

前提条件と不変条件

  • 入力される operator は、pg_operator カタログで定義されたネットワーク関連の演算子である必要があります。
  • 未定義の演算子が渡された場合、elog(ERROR) により処理が中断されます。

アルゴリズムまたは振る舞い

1. switch 文を用いて、入力された operator Oid を判定します。 2. 演算子の意味論（包含関係の方向）に応じて、対応する整数値を返します。 3. どのケースにも該当しない場合はエラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network_selfuncs.c:105 (networksel 内)
  • src/backend/utils/adt/network_selfuncs.c:225 (networkjoinsel 内)
  • **呼び出し先**:
  • elog (エラー報告用)

関連データ構造

  • Oid: PostgreSQL のシステムカタログ識別子。

失敗モードと境界条件

  • ネットワーク演算子以外の Oid が渡された場合、elog(ERROR) によりクエリ実行が失敗します。

保守担当者向けメモ

  • 新しいネットワーク演算子を追加した場合は、本関数の switch 文に新しいケースを追加し、対応する数値コードを割り当てる必要があります。
  • 戻り値の数値（-2〜2）は、他の選択率推定関数（inet_hist_value_sel 等）における包含ロジックの分岐条件として利用されているため、変更時は関連関数への影響を考慮してください。
