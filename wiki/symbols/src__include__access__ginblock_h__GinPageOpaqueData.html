GinPageOpaqueData

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/ginblock.h:30

目的

GIN（Generalized Inverted Index）インデックスの各物理ページ末尾（pd_special 領域）に配置され、ページの種類や状態、リンク情報を保持するために使用されます。

シグネチャまたは形状

typedef struct GinPageOpaqueData
{
    BlockNumber rightlink;  /* 次のページへのリンク */
    OffsetNumber maxoff;    /* ページ内の項目数（GIN_DATA/GIN_LEAFページではPostingItem数、GIN_LISTページではヒープタプル数） */
    uint16      flags;      /* ページタイプ等のビットフラグ */
} GinPageOpaqueData;

入力と出力

  • **入力**: ページポインタから GinPageGetOpaque() マクロを介してアクセスされます。
  • **出力**: ページ内のデータ構造（PostingItem や GinPostingList）の解釈や、ページ間の走査順序を決定するためのメタデータを提供します。

前提条件と不変条件

  • PostgreSQLの標準的なページレイアウトに従い、pd_special 領域に配置されます。
  • PageInit 呼び出し時に、sizeof(GinPageOpaqueData) 分の領域が確保されている必要があります。
  • maxoff の意味は flags に設定されたページタイプ（GIN_DATA, GIN_LEAF, GIN_LIST 等）に依存します。

アルゴリズムまたは振る舞い

1. ページアクセス時に PageGetSpecialPointer を経由して本構造体を取得します。 2. flags を参照し、ページがリーフか、圧縮されているか等の状態を判定します。 3. rightlink を用いて、インデックス走査時の次ページへの移動を制御します。 4. maxoff を用いて、ページ内のデータ領域（PostingItem 等）のオフセット計算を行います。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照）**:
  • contrib/amcheck/verify_gin.c: ページ構造の整合性検証に使用。
  • contrib/pageinspect/ginfuncs.c: インデックス内容の検査に使用。
  • src/backend/access/gin/ginfast.c: ページヘッダー計算に使用。
  • src/backend/access/gin/ginutil.c: PageInit によるページ初期化時に使用。
  • **呼び出し先**: 構造体定義のため、特定の関数を呼び出すことはありません。

関連データ構造

  • GinMetaPageData: GINインデックスのメタページ構造。
  • PostingItem: GINデータページ内の項目。
  • GinPostingList: 圧縮されたポストリスト。

失敗モードと境界条件

  • PageGetSpecialSize が MAXALIGN(sizeof(GinPageOpaqueData)) と一致しない場合、ページ破損とみなされます。
  • flags に不正なビット組み合わせが設定されている場合、後続のデータアクセスでメモリ破壊や誤ったデータ解釈が発生する可能性があります。

保守担当者向けメモ

  • 本構造体はGINインデックスの物理レイアウトの根幹です。変更時はインデックスのバージョン管理および既存のWALログとの互換性に十分注意してください。
  • 直接ポインタ演算を行わず、必ず GinPageGetOpaque() マクロを介してアクセスしてください。
