member_verify_header

定義

  • 種別: function
  • 定義箇所: src/bin/pg_verifybackup/astreamer_verify.c:160

目的

tarアーカイブ内の各メンバー（ファイル）をバックアップマニフェストと照合し、検証の準備を行う。

シグネチャまたは形状

static void member_verify_header(astreamer *streamer, astreamer_member *member);

入力と出力

  • **入力**:
  • streamer: 検証状態を保持する astreamer_verify 構造体へのポインタ。
  • member: 現在処理中のアーカイブメンバー情報。
  • **出力**: なし（副作用として mystreamer の状態更新およびエラー報告を行う）。

前提条件と不変条件

  • member->is_regular が真であること（通常ファイルのみを対象とする）。
  • mystreamer->context が有効な検証コンテキストを保持していること。

アルゴリズムまたは振る舞い

1. **パス正規化**: member->pathname を基に、テーブルスペースの有無に応じてパスを構築し canonicalize_path を適用する。 2. **除外判定**: should_ignore_relpath により無視リストに含まれるか確認する。 3. **マニフェスト検索**: manifest_files_lookup でマニフェスト内のエントリを検索し、存在しない場合はエラーを報告する。 4. **メタデータ検証**: マニフェスト上のサイズとアーカイブ内のサイズを比較し、不一致があればエラーを報告し m->bad を設定する。 5. **検証フラグ設定**: チェックサム検証の要否および制御ファイル（pg_control）の追加検証フラグを決定する。 6. **初期化**: チェックサム検証が有効な場合、pg_checksum_init を呼び出しコンテキストを初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer_verify_content（src/bin/pg_verifybackup/astreamer_verify.c:103）。
  • **呼び出し先**:
  • canonicalize_path
  • should_ignore_relpath
  • manifest_files_lookup
  • report_backup_error
  • should_verify_checksum
  • pg_checksum_init

関連データ構造

  • astreamer_verify: 検証の全体状態を管理。
  • astreamer_member: アーカイブ内のファイル情報。
  • manifest_file: マニフェスト内のファイル定義。

失敗モードと境界条件

  • **ファイル不在**: マニフェストに存在しないファイルはエラーとして報告される。
  • **サイズ不一致**: マニフェストとアーカイブ間でサイズが異なる場合、エラー報告後に検証対象外（bad）としてマークされる。
  • **チェックサム初期化失敗**: pg_checksum_init が失敗した場合、エラー報告を行い、そのファイルに対するチェックサム検証を無効化する。

保守担当者向けメモ

  • テーブルスペースのパス処理において、pg_tblspc プレフィックスの付与と正規化が整合性検証の鍵となる。
  • 制御ファイルの検証はマニフェストバージョンが1より大きい場合にのみ行われるため、バージョン依存のロジックが含まれている。
