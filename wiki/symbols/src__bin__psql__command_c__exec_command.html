exec_command

定義

  • 種別: function
  • 定義箇所: src/bin/psql/command.c:314

目的

psql のバックスラッシュコマンド（\ で始まるコマンド）を解析し、対応する実行関数へディスパッチする。

シグネチャまたは形状

static backslashResult exec_command(
    const char *cmd,
    PsqlScanState scan_state,
    ConditionalStack cstack,
    PQExpBuffer query_buf,
    PQExpBuffer previous_buf
);

入力と出力

  • **入力**:
  • cmd: 実行対象のコマンド名。
  • scan_state: 字句解析器の状態。
  • cstack: 条件付き実行（\if）のスタック。
  • query_buf: 現在のクエリバッファ。
  • previous_buf: 前回のクエリバッファ。
  • **出力**:
  • backslashResult: コマンド実行後の状態（PSQL_CMD_SEND 等）。

前提条件と不変条件

  • conditional_active(cstack) を用いて、現在の \if ブロックがアクティブかどうかを判定する。
  • 非アクティブなブランチ内では、コマンドは実行されず、引数の読み飛ばしのみが行われる（各サブ関数がこの責務を負う）。

アルゴリズムまたは振る舞い

1. conditional_active で現在の実行可否を判定する。 2. 対話モードかつ非アクティブなブランチで、かつ分岐コマンド（\if 等）でない場合に警告を出力する。 3. cmd 文字列に基づき、対応する exec_command_* 関数を呼び出す。 4. 実行結果が PSQL_CMD_SEND であり、かつ query_buf が空の場合、copy_previous_query を呼び出して previous_buf を query_buf にコピーする。 5. 最終的な backslashResult を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c 内の HandleSlashCmds から呼び出される（直接の証拠あり）。
  • **呼び出し先**: exec_command_* で始まる多数のコマンド実行関数（例: exec_command_a, exec_command_connect 等）。

関連データ構造

  • backslashResult: 実行結果の列挙型。
  • PsqlScanState: 字句解析の状態管理。
  • ConditionalStack: \if ブロックのネスト管理。
  • PQExpBuffer: クエリ文字列を保持するバッファ。

失敗モードと境界条件

  • 未知のコマンドが渡された場合、PSQL_CMD_UNKNOWN を返す。
  • \if ブロックのネストや状態管理に失敗した場合、各サブ関数側のエラーハンドリングに依存する。

保守担当者向けメモ

  • 新しいコマンドを追加する場合は、本関数の if-else チェーンに登録すること。
  • PSQL_CMD_SEND を返すコマンドの共通処理（previous_buf のコピー）は本関数で行われるため、個別のサブ関数で重複実装しないこと。
