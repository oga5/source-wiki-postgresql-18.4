intset_create

定義

  • 種別: function
  • 定義箇所: src/backend/lib/integerset.c:282

目的

新しい空の IntegerSet 構造体を生成し、初期化する。

シグネチャまたは形状

IntegerSet *intset_create(void)

入力と出力

  • **入力**: なし。
  • **出力**: 初期化された IntegerSet 構造体へのポインタ。

前提条件と不変条件

  • 現在のメモリコンテキスト (CurrentMemoryContext) 上に IntegerSet が割り当てられる。
  • 戻り値の IntegerSet は、エントリ数 0、ルートノード NULL、バッファ空、イテレータ非アクティブの状態で初期化される。

アルゴリズムまたは振る舞い

1. palloc を使用して IntegerSet 構造体分のメモリを確保する。 2. CurrentMemoryContext を intset->context に保存し、初期メモリ使用量を記録する。 3. 集合の管理用メンバ（num_entries, highest_value, num_levels, root, leftmost_leaf）をゼロまたは NULL で初期化する。 4. rightmost_nodes 配列をゼロクリアする。 5. バッファおよびイテレータ関連のフラグとポインタを初期状態（無効または空）に設定する。 6. 初期化済みの intset ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistvacuum.c
  • src/test/modules/test_integerset/test_integerset.c
  • **呼び出し先**:
  • palloc (メモリ確保)
  • GetMemoryChunkSpace (メモリ使用量取得)
  • memset (メモリ初期化)

関連データ構造

  • IntegerSet: 本関数が生成するメインのデータ構造。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は IntegerSet のメモリコンテキストを保持するが、IntegerSet 自体を解放する関数は存在しない。メモリを解放する場合は、作成時に使用された MemoryContext を破棄すること。
