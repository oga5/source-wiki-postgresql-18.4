RecursiveUnion

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/plannodes.h:411

目的

再帰的クエリ（WITH RECURSIVE）を実行するための実行計画ノードを定義する。

シグネチャまたは形状

typedef struct RecursiveUnion
{
    Plan        plan;
    int         wtParam;
    int         numCols;
    AttrNumber *dupColIdx;
    Oid        *dupOperators;
    Oid        *dupCollations;
    long        numGroups;
} RecursiveUnion;

入力と出力

  • **入力**: plan メンバ（基底構造体）を通じて、再帰の非再帰項および再帰項となる子ノードを保持する。
  • **出力**: 再帰処理の結果として生成されたタプルセット。

前提条件と不変条件

  • Plan 構造体を最初のメンバとして保持し、プランノードとしての継承関係を維持する。
  • UNION ALL の場合、重複排除に関連するフィールド（numCols, dupColIdx 等）はゼロまたは NULL に設定される。

アルゴリズムまたは振る舞い

  • 再帰的クエリの実行において、wtParam を用いて作業テーブル（work table）を識別する。
  • 重複排除が必要な場合、numCols で指定された列数に基づき、dupColIdx、dupOperators、dupCollations を使用して比較を行う。
  • エグゼキュータは本ノードを再帰的に処理し、結果を蓄積する。

呼び出し元と呼び出し先

  • **呼び出し元**: プランナ（src/backend/optimizer/）が本ノードを生成する。
  • **呼び出し先**: エグゼキュータ（src/backend/executor/nodeRecursiveunion.c）が ExecInitRecursiveUnion を通じて初期化し、実行する。
  • **その他**: src/backend/commands/explain.c が実行計画の表示時に本ノードを参照する。

関連データ構造

  • Plan: 全てのプランノードの基底構造体。
  • PlanState: 実行時に本ノードの状態を保持する対応する実行時ノード。

失敗モードと境界条件

  • 重複排除を行わない UNION ALL 形式では、関連するメタデータフィールドが空であることを前提とする。
  • 再帰の停止条件が満たされない場合、無限ループに陥る可能性がある（これはエグゼキュータ側の制御に依存する）。

保守担当者向けメモ

  • 本ノードは式コンテキストを持たない。
  • dupColIdx 等の配列サイズは numCols に依存する。
  • 実行時の状態管理については src/include/nodes/execnodes.h および src/backend/executor/nodeRecursiveunion.c を参照すること。
