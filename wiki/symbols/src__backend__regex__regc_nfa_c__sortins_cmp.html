sortins_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_nfa.c:661

目的

NFA（非決定性有限オートマトン）の遷移（arc）を、特定の優先順位に基づいて比較し、ソート順を決定する。

シグネチャまたは形状

static int sortins_cmp(const void *a, const void *b)
  • a, b: 比較対象の struct arc * へのポインタ。

入力と出力

  • 入力: qsort 等から渡される、struct arc * 型のポインタへのポインタ。
  • 出力: 比較結果として -1（a < b）、1（a > b）、または 0（等価）を返す。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な struct arc * を指している必要がある。
  • 比較対象の arc 構造体は、from（状態）、co（色）、type フィールドが初期化されていること。

アルゴリズムまたは振る舞い

以下の優先順位でフィールドを比較し、差異が見つかった時点で結果を返す： 1. from->no（遷移元の状態番号） 2. co（遷移の色/条件） 3. type（遷移の型）

すべてのフィールドが一致する場合、0 を返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/regex/regc_nfa.c 内の複数の箇所で、qsort の比較関数として使用されている。
  • src/backend/regex/regc_nfa.c 内で、遷移の順序比較（switch (sortins_cmp(...))）として直接呼び出されている。
  • 呼び出し先:
  • 構造体のフィールドアクセスのみであり、他の関数呼び出しは行わない。

関連データ構造

  • struct arc: 遷移を定義する構造体。from（struct state へのポインタ）、co（color 型）、type を保持する。

失敗モードと境界条件

  • a または b が指すポインタが NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • 比較対象の from ポインタが NULL の場合、from->no へのアクセスで失敗する。

保守担当者向けメモ

  • 本関数は qsort の比較関数として設計されているため、比較ロジックを変更する際は、regc_nfa.c 内の他のソート処理や比較ロジックとの整合性に注意すること。
  • regcomp.c:164 にプロトタイプ宣言が存在する。
