setop_retrieve_sorted

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSetOp.c:196

目的

ソート済みの入力リレーション（外部および内部）に対して集合演算（INTERSECT または EXCEPT）を実行し、結果のタプルを順次取得する。

シグネチャまたは形状

TupleTableSlot *setop_retrieve_sorted(SetOpState *setopstate)

入力と出力

  • **入力**: SetOpState *setopstate（実行状態を保持するノード状態構造体）。
  • **出力**: 集合演算の結果として出力すべき TupleTableSlot。これ以上出力するタプルがない場合は NULL を返す。

前提条件と不変条件

  • 入力される外部および内部リレーションは、集合演算のキーに基づいてソート済みであること。
  • 初回呼び出し時に setopstate->need_init が真であれば、子プランからのタプル取得準備（初期化）が行われる。
  • setopstate->leftInput.firstTupleSlot が結果として返されるタプルのソースとなる。

アルゴリズムまたは振る舞い

1. **初期化**: 初回実行時、両入力の最初のタプルを読み込み、グループ化の準備を行う。外部リレーションが空であれば即座に終了する。 2. **グループ取得**: setop_load_group を呼び出し、外部および内部の現在のグループを読み込む。 3. **比較と判定**:

  • 外部グループと内部グループを setop_compare_slots で比較する。
  • 比較結果に基づき、現在のグループが演算結果として出力対象か否かを判断する。

    • 4. **出力生成**: set_output_count を用いて出力すべき行数を決定し、該当する場合は resultTupleSlot を返す。 5. **ループ**: 演算が完了するまで、または出力すべきタプルが見つかるまで、グループの読み込みと比較を繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecSetOp（src/backend/executor/nodeSetOp.c 内）。
  • **呼び出し先**: ExecProcNode（子ノードからのタプル取得）、setop_load_group、setop_compare_slots、set_output_count、ExecClearTuple。

関連データ構造

  • SetOpState: 実行ノードの状態を保持する主要構造体。
  • SetOpStatePerGroupData: 現在のグループにおける左側・右側のタプル出現数を保持する一時構造体。
  • TupleTableSlot: タプルを保持するスロット。

失敗モードと境界条件

  • **外部リレーションが空**: 演算結果は空となり、NULL を返す。
  • **内部リレーションが空**: 比較ロジックにより、外部グループが優先的に処理される。
  • **グループ不一致**: 比較結果が 0 でない場合、片方のグループをスキップして次のグループへ進む。

保守担当者向けメモ

  • 本関数はマージジョインのロジックに類似している。
  • UNION 演算には使用されないため、修正時は INTERSECT および EXCEPT の仕様に留意すること。
  • setopstate->need_init フラグによる初期化フローは、再スキャン時等の状態遷移において重要である。
