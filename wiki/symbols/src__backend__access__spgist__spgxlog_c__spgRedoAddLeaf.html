spgRedoAddLeaf

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgxlog.c:73

目的

SP-GiSTインデックスにおけるリーフタプルの追加または置換操作をWALリプレイ時に再現する。

シグネチャまたは形状

static void spgRedoAddLeaf(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）
  • **出力**: なし（副作用としてインデックスページを更新）

前提条件と不変条件

  • record から取得されるリーフタプルはアライメントされていないため、アクセス時には memcpy によるコピーが必要。
  • WALリプレイ中であるため、親ノードの更新よりも先にリーフページの更新を行うことが許容される。

アルゴリズムまたは振る舞い

1. WALレコードから spgxlogAddLeaf 構造体とリーフタプルデータを抽出する。 2. xldata->newPage が真の場合、SpGistInitBuffer で新規ページを初期化する。それ以外の場合は XLogReadBufferForRedo で既存ページを取得する。 3. ページに対して以下のいずれかの操作を実行する:

  • **新規追加**: addOrReplaceTuple を使用してタプルを挿入し、必要に応じてヘッドタプルのチェーンリンクを更新する。
  • **置換**: PageIndexTupleDelete で既存のDEADタプルを削除し、PageAddItem で新しいタプルを挿入する。

    • 4. ページにLSNを書き込み、バッファをダーティとしてマークする。 5. xldata->offnumParent が有効な場合、親ページを読み込み、spgUpdateNodeLink を用いてダウンリンクを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: spg_redo（src/backend/access/spgist/spgxlog.c:944）
  • **呼び出し先**: XLogRecGetData, XLogInitBufferForRedo, SpGistInitBuffer, XLogReadBufferForRedo, addOrReplaceTuple, PageGetItem, PageGetItemId, PageIndexTupleDelete, PageAddItem, PageSetLSN, MarkBufferDirty, UnlockReleaseBuffer, XLogRecGetBlockTag, spgUpdateNodeLink

関連データ構造

  • spgxlogAddLeaf: WALレコードのペイロード構造。
  • SpGistLeafTupleData: リーフタプルのヘッダー情報。
  • SpGistInnerTuple: 親ノードのタプル構造。

失敗モードと境界条件

  • PageAddItem が指定されたオフセットに挿入できなかった場合、elog(ERROR) を発生させる。
  • 親ノードの更新時に offnumParent が InvalidOffsetNumber の場合は更新をスキップする。

保守担当者向けメモ

  • リプレイ関数内では、スタンバイサーバーでの読み取り整合性を保つため、子ページ（リーフ）の更新を優先的に完了させる必要がある。
  • タプルデータはアライメントされていないため、直接キャストせず必ず memcpy を介してアクセスすること。
