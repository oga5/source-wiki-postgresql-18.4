consume_xids

定義

  • 種別: function
  • 定義箇所: src/test/modules/xid_wraparound/xid_wraparound.c:31

目的

指定された数だけトランザクションID（XID）を強制的に消費し、最終的なトランザクションIDを返すSQL呼び出し可能な関数です。主にXID周回テストの自動化に使用されます。

シグネチャまたは形状

Datum consume_xids(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_INT64(0) により int64 型の消費数（nxids）を受け取ります。
  • **出力**: PG_RETURN_FULLTRANSACTIONID により、消費後の最新の FullTransactionId を返します。

前提条件と不変条件

  • nxids は 0 以上である必要があります。負の値が指定された場合はエラーとなります。
  • 本関数はテスト専用であり、本番環境での実行はXID周回を早め、データベースの停止を招くリスクがあるため推奨されません。

アルゴリズムまたは振る舞い

1. 引数 nxids が 0 の場合、ReadNextFullTransactionId() を呼び出して現在の最新XIDを取得し、そのまま返します。 2. nxids が 0 より大きい場合、consume_xids_common を呼び出し、指定された数だけXIDを消費するロジックを実行します。 3. 処理完了後、到達した最終的な FullTransactionId を戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます。ソースコード上では PG_FUNCTION_INFO_V1(consume_xids) による登録が確認できます。
  • **呼び出し先**:
  • ReadNextFullTransactionId()
  • consume_xids_common()
  • elog() (エラー発生時)

関連データ構造

  • FullTransactionId: 64bitトランザクションIDを保持する型。
  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約に従う引数構造。

失敗モードと境界条件

  • nxids < 0: elog(ERROR, ...) により処理が中断されます。
  • nxids == 0: 消費を行わず、現在のIDを返す正常系として処理されます。

保守担当者向けメモ

  • 本関数は consume_xids_common に処理を委譲しているため、消費ロジックの変更は主にそちらを確認してください。
  • XidGenLock を取得する処理が含まれる場合があるため、実行中は他のトランザクションの進行をブロックする可能性があります。
