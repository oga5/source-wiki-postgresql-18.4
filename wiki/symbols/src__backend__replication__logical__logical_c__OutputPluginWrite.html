OutputPluginWrite

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/logical.c:706

目的

論理デコーディングにおいて、OutputPluginPrepareWrite で準備されたデータを、出力プラグインの書き込みコールバックを介して実際に転送する。

シグネチャまたは形状

void OutputPluginWrite(struct LogicalDecodingContext *ctx, bool last_write)

入力と出力

  • ctx: 論理デコーディングのコンテキスト構造体。
  • last_write: 今回の書き込みが、現在の論理メッセージにおける最後の断片であるかを示すフラグ。

前提条件と不変条件

  • ctx->prepared_write が true であること。これが false の場合、elog(ERROR) により処理が中断される。

アルゴリズムまたは振る舞い

1. ctx->prepared_write の状態を確認し、準備が完了していない場合はエラーを投げる。 2. ctx 内に保持されている write コールバック関数を呼び出し、ctx、write_location、write_xid、および last_write フラグを渡す。 3. 処理完了後、ctx->prepared_write を false にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/test_decoding/test_decoding.c 内の各コールバック関数（pg_decode_change, pg_decode_commit_txn 等）から頻繁に呼び出されている。
  • **呼び出し先**: ctx->write（LogicalDecodingContext 構造体で定義された関数ポインタ）。

関連データ構造

  • LogicalDecodingContext: 論理デコーディングの状態および出力プラグインのコールバック関数を保持する主要構造体。

失敗モードと境界条件

  • OutputPluginPrepareWrite を呼び出さずに本関数を呼び出した場合、elog(ERROR) が発生する。

保守担当者向けメモ

  • 本関数は OutputPluginPrepareWrite と対で使用される。状態管理（prepared_write）が必須であるため、プラグイン開発者がこの順序を遵守しているか確認すること。
  • ctx->write はプラグイン側で実装されるため、プラグインの挙動に依存する。
