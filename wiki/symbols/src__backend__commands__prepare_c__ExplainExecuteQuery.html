ExplainExecuteQuery

定義

  • 種別: function
  • 定義箇所: src/backend/commands/prepare.c:570

目的

プリペアドステートメントに対して EXPLAIN を実行し、その実行計画を解析・表示する。

シグネチャまたは形状

void ExplainExecuteQuery(ExecuteStmt *execstmt, IntoClause *into, ExplainState *es, ParseState *pstate, ParamListInfo params)

入力と出力

  • **入力**:
  • execstmt: 実行対象の EXECUTE 文ノード。
  • into: CREATE TABLE AS 等の出力先指定（NULL可）。
  • es: EXPLAIN の出力状態を保持する構造体。
  • pstate: 現在の解析状態。
  • params: 実行時に提供されるパラメータリスト。
  • **出力**: なし（es を通じて EXPLAIN 結果を出力）。

前提条件と不変条件

  • 対象のプリペアドステートメントは、結果セットの構造が固定されている必要がある（fixed_result が真であること）。
  • es->memory が有効な場合、一時的なメモリコンテキストを作成してプランナのメモリ消費量を計測する。

アルゴリズムまたは振る舞い

1. **準備**: es->memory が有効なら専用のメモリコンテキストを作成し、es->buffers が有効ならバッファ使用量の初期値を記録する。 2. **検索**: FetchPreparedStatement を使用して、指定された名前のプリペアドステートメントをハッシュテーブルから取得する。 3. **パラメータ評価**: ステートメントにパラメータが必要な場合、EvaluateParams を呼び出して ParamListInfo を構築する。 4. **計画取得**: GetCachedPlan を呼び出し、必要に応じて再計画を行い、実行可能な CachedPlan を取得する。 5. **実行計画の出力**: 取得した plan_list を反復処理し、各ステートメントに対して ExplainOnePlan（またはユーティリティ文の場合は ExplainOneUtility）を呼び出す。 6. **終了処理**: FreeExecutorState で一時的な実行状態を解放し、ReleaseCachedPlan でキャッシュ計画の参照カウントを減らす。

関連データ構造

  • PreparedStatement: プリペアドステートメントの情報を保持する構造体。
  • CachedPlan: キャッシュされた実行計画。
  • ExplainState: EXPLAIN の出力オプションや状態を管理。
  • ParamListInfo: クエリ実行時のパラメータ情報。

失敗モードと境界条件

  • **エラー**: プリペアドステートメントが可変結果（variable-result）を持つ場合、elog(ERROR) により処理を中断する。
  • **エラー**: FetchPreparedStatement が対象を見つけられない場合や、パラメータ評価で型変換等のエラーが発生した場合は、基盤側の ereport(ERROR) により処理が停止する。

保守担当者向けメモ

  • 本関数は EXPLAIN EXECUTE の実行パスを担う。EXPLAIN ANALYZE のような実行を伴う解析を行うため、GetCachedPlan による計画の取得と ExplainOnePlan による実行統計の収集が主眼となる。
  • es->memory や es->buffers の計測ロジックは ExplainOneQuery と同様のパターンに従っているため、修正時は整合性に注意すること。
