LWLockMode

定義

  • 種別: enum
  • 定義箇所: src/include/storage/lwlock.h:112

目的

軽量ロック（LWLock）を取得または解放する際のモードを指定するために使用されます。

シグネチャまたは形状

typedef enum LWLockMode
{
    LW_EXCLUSIVE,
    LW_SHARED,
    LW_WAIT_UNTIL_FREE,
} LWLockMode;

入力と出力

  • **入力**: LWLockAcquire や LWLockRelease などの関数において、ロックの取得・解放の挙動を制御する引数として渡されます。
  • **出力**: 該当なし（列挙型定数）。

前提条件と不変条件

  • LW_WAIT_UNTIL_FREE は PGPROC->lwWaitMode で使用される特殊なモードであり、LWLockAcquire の引数として直接使用してはなりません。

アルゴリズムまたは振る舞い

  • LW_EXCLUSIVE: 排他モード。他のプロセスがロックを保持していない場合にのみ取得可能です。
  • LW_SHARED: 共有モード。他のプロセスが共有モードでロックを保持していても取得可能です。
  • LW_WAIT_UNTIL_FREE: ロックが解放されるまで待機する状態を示します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c (AssertNotCatalogBufferLock)
  • src/backend/storage/lmgr/lwlock.c (LWLockAcquire, LWLockReleaseInternal, LWLockAttemptLock, LWLockQueueSelf, LWLockConditionalAcquire, LWLockAcquireOrWait, PRINT_LWDEBUG)
  • **呼び出し先**:
  • このシンボルは列挙型であり、関数を呼び出す側ではありません。

関連データ構造

  • LWLock: ロックの本体。
  • PGPROC: 待機プロセス情報（lwWaitMode フィールドで本列挙型を使用）。

失敗モードと境界条件

  • LW_WAIT_UNTIL_FREE を LWLockAcquire に渡した場合の動作は未定義または禁止されています。

保守担当者向けメモ

  • 本列挙型は src/backend/storage/lmgr/lwlock.c 内のロック取得・解放ロジックの根幹を成します。モードの追加や変更を行う際は、関連するアトミック操作および待機キューの処理に影響がないか確認してください。
