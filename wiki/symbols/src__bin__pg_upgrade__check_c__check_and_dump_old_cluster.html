check_and_dump_old_cluster

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/check.c:588

目的

pg_upgrade プロセスにおいて、旧クラスタの互換性を検証し、アップグレードに必要なメタデータの抽出およびダンプ準備を行う。

シグネチャまたは形状

void check_and_dump_old_cluster(void);

入力と出力

  • 入力: グローバル変数 old_cluster（旧クラスタ情報）および user_opts（ユーザーオプション）。
  • 出力: なし（副作用としてログ出力、検証失敗時の pg_fatal による終了、およびダンプファイルの生成を行う）。

前提条件と不変条件

  • old_cluster が初期化されていること。
  • 実行環境が pg_upgrade の事前検証フェーズにあること。

アルゴリズムまたは振る舞い

1. **環境準備**: live_check オプションが有効でない場合、旧クラスタのポストマスターを起動する。 2. **接続・情報収集**: データベース接続状態を確認し、DB、テーブル、論理レプリケーションスロット、テーブルスペース、ロード可能なライブラリの情報を取得する。 3. **互換性検証**: 以下の項目を順次チェックする。

  • インストールユーザー、準備されたトランザクション、ISN/INT8 型の受け渡し不整合。
  • PG17以降: 論理レプリケーションスロットの有効性とサブスクリプション状態。
  • データ型の使用状況、Unicode 更新の影響。
  • バージョン依存の制約チェック（エンコーディング変換関数、ポストフィックス演算子、ポリモーフィック関数、WITH OIDS、NOT NULL 制約の継承、ハッシュインデックス、pg_ ロールプレフィックス）。

    • 4. **ダンプ生成**: check オプションが有効でない場合、旧クラスタのダンプを実行する。 5. **終了処理**: live_check オプションが有効でない場合、ポストマスターを停止する。

関連データ構造

  • old_cluster (ClusterInfo型)
  • user_opts (UserOpts型)

失敗モードと境界条件

  • 接続不可や互換性チェックの不適合が検出された場合、pg_fatal によりプロセスが中断される。
  • バージョン固有のチェック（例: PG 13以下、PG 9.6以下）は、対象となる旧クラスタのメジャーバージョンに基づいて条件分岐する。

保守担当者向けメモ

  • 本関数は、PostgreSQLのバージョンアップに伴う破壊的変更を網羅する中心的な場所である。新しい互換性チェックを追加する場合は、適切なバージョン条件分岐を伴うチェック関数をここに追加すること。
