check_on_shmem_exit_lists_are_empty

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/ipc.c:438

目的

共有メモリ終了コールバックリスト（before_shmem_exit および on_shmem_exit）が空であることを検証し、意図しないタイミングでコールバックが登録・実行されていないかをチェックする。

シグネチャまたは形状

void check_on_shmem_exit_lists_are_empty(void)

入力と出力

  • 入力: なし
  • 出力: なし（正常時は何も返さない）

前提条件と不変条件

  • 共有メモリ関連の終了処理が開始される前の状態であること。
  • before_shmem_exit_index および on_shmem_exit_index が 0 であることが期待される。

アルゴリズムまたは振る舞い

1. before_shmem_exit_index が 0 でない場合、FATAL エラーを発生させプロセスを終了する。 2. on_shmem_exit_index が 0 でない場合、FATAL エラーを発生させプロセスを終了する。 3. 共有メモリのデタッチ状態については、上記リストが空であれば正常とみなす。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/tcop/backend_startup.c:365
  • 呼び出し先:
  • elog（エラー発生時）

関連データ構造

  • before_shmem_exit_index: before_shmem_exit リストの現在のインデックス。
  • on_shmem_exit_index: on_shmem_exit リストの現在のインデックス。

失敗モードと境界条件

  • リストにコールバックが残っている場合、FATAL エラーによりプロセスが強制終了される。これは、クリーンアップ処理が予期せぬタイミングで実行されたことを示す重大な不整合として扱われる。

保守担当者向けメモ

  • 本関数は主にデバッグや整合性チェックを目的としており、正常な実行パスではリストが空であることが前提となる。
