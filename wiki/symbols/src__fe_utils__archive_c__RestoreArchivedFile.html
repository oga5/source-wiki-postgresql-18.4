RestoreArchivedFile

定義

  • **種別**: 関数
  • **定義箇所**: src/fe_utils/archive.c:38

目的

外部アーカイブストレージからWALファイルを復元し、ローカル環境で読み取り可能な状態にする。

シグネチャまたは形状

int RestoreArchivedFile(const char *path, const char *xlogfname,
                        off_t expectedSize, const char *restoreCommand);

入力と出力

  • **入力**:
  • path: 復元先ディレクトリのベースパス。
  • xlogfname: 復元対象のWALファイル名。
  • expectedSize: 期待されるファイルサイズ（0の場合はチェックをスキップ）。
  • restoreCommand: 復元を実行するためのシェルコマンド文字列。
  • **出力**:
  • 成功時: 復元されたファイルのファイルディスクリプタ（int）。
  • 失敗時: -1 を返すか、致命的なエラーとしてプロセスを終了する。

前提条件と不変条件

  • restore_command は、指定されたパスへファイルをコピーする機能を持つ必要がある。
  • 復元に成功した場合、呼び出し元は返されたファイルディスクリプタを適切に close() する責任を負う。

アルゴリズムまたは振る舞い

1. **パス構築**: path と xlogfname を結合し、ローカルのフルパスを生成する。 2. **コマンド生成**: BuildRestoreCommand を使用して、restoreCommand 内のプレースホルダを実際のパスに置換する。 3. **実行**: system() を介してコマンドを実行する。 4. **検証**:

  • stat() を実行し、ファイルが存在することを確認する。
  • expectedSize > 0 の場合、stat_buf.st_size との不一致をチェックし、不一致であれば pg_fatal で終了する。

    • 5. **オープン**: open() でファイルを読み取り専用で開き、ディスクリプタを返す。 6. **エラー処理**:

  • コマンドがシグナルで中断された場合、pg_fatal で終了する。
  • ファイルが存在しない場合はエラーログを出力し、-1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/timeline.c
  • src/backend/access/transam/xlogarchive.c
  • src/backend/access/transam/xlogrecovery.c
  • src/bin/pg_rewind/parsexlog.c
  • **呼び出し先**:
  • BuildRestoreCommand (外部モジュール)
  • system(), stat(), open() (標準ライブラリ)
  • pg_fatal(), pg_log_error() (PostgreSQLロギングAPI)

関連データ構造

  • struct stat: ファイル属性の取得に使用。

失敗モードと境界条件

  • **シェルエラー**: restore_command が見つからない、または実行に失敗した場合は致命的エラーとして処理される。
  • **サイズ不一致**: 復元されたファイルのサイズが expectedSize と異なる場合、致命的エラーとなる。
  • **ファイル不在**: コマンド実行後にファイルが生成されない場合、エラーログを出力して -1 を返す。
  • **セキュリティ**: system() を使用しているため、restore_command に含まれるシェルインジェクションのリスクは呼び出し側に依存する。

保守担当者向けメモ

  • expectedSize に 0 を渡すとサイズ検証がスキップされる。
  • 復元失敗時の挙動は、シグナルによる中断か、単なるファイル不在かによって分岐する。
