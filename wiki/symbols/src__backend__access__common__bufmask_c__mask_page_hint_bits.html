mask_page_hint_bits

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/bufmask.c:45

目的

WAL（Write-Ahead Logging）の整合性チェックにおいて、ページ内のヒントビットや一時的なフラグを正規化（マスク）し、比較を容易にするために使用されます。

シグネチャまたは形状

void mask_page_hint_bits(Page page)

入力と出力

  • **入力**: Page page（操作対象のページポインタ）
  • **出力**: なし（インプレースでページ内容を書き換える）

前提条件と不変条件

  • 引数 page は有効な Page 型である必要があります。
  • 本関数はテスト目的の正規化ツールであり、実稼働時のページ更新には使用しないでください。

アルゴリズムまたは振る舞い

1. PageHeader にキャストし、pd_prune_xid を MASK_MARKER（通常 0）で上書きします。 2. PageClearFull を呼び出し、PD_PAGE_FULL フラグをクリアします。 3. PageClearHasFreeLinePointers を呼び出し、PD_HAS_FREE_LINES フラグをクリアします。 4. PageClearAllVisible を呼び出し、ページ全体の可視性フラグをクリアします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_xlog.c
  • src/backend/access/gin/ginxlog.c
  • src/backend/access/gist/gistxlog.c
  • src/backend/access/hash/hash_xlog.c
  • src/backend/access/heap/heapam_xlog.c
  • src/backend/access/nbtree/nbtxlog.c
  • src/backend/access/spgist/spgxlog.c
  • **呼び出し先**:
  • PageClearFull
  • PageClearHasFreeLinePointers
  • PageClearAllVisible

関連データ構造

  • PageHeader（src/include/storage/bufpage.h 等で定義されるページヘッダー構造体）

失敗モードと境界条件

  • 引数に NULL ポインタが渡された場合の挙動は定義されていません。
  • ページ構造が破損している場合、キャストやマクロの実行時に予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • MASK_MARKER は 0 として扱われます。比較対象間で値が異なっていても問題ない箇所を特定し、正規化する役割を担います。
  • PageClearAllVisible の呼び出しは、WAL リプレイ時にページ LSN がレコードの LSN を追い越している場合に、誤って全可視と判定されるのを防ぐための措置です。
