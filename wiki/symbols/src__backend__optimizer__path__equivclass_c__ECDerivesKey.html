ECDerivesKey

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/path/equivclass.c:108

目的

等価クラス（EquivalenceClass）から導出された等価制約をハッシュテーブルで管理するための検索キーとして使用されます。

シグネチャまたは形状

typedef struct
{
	EquivalenceMember *em1;
	EquivalenceMember *em2;
	EquivalenceClass *parent_ec;
} ECDerivesKey;

入力と出力

  • 入力: EquivalenceMember のペア（em1, em2）およびそれらが属する EquivalenceClass（parent_ec）。
  • 出力: ハッシュテーブル検索用のキー構造体。

前提条件と不変条件

  • parent_ec は、em1 および em2 が所属する有効な等価クラスである必要があります。
  • simplehash.h を用いたハッシュテーブルのキーとして利用されるため、hash_bytes 等による一貫したハッシュ値計算が前提となります。

アルゴリズムまたは振る舞い

  • 導出された等価制約の重複排除を行うための識別子として機能します。
  • fill_ec_derives_key 関数によって構造体の各フィールドが初期化されます。
  • hash_bytes を介して、構造体全体のメモリ内容に基づいたハッシュ値が計算されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/path/equivclass.c 内の導出制約管理処理（fill_ec_derives_key 等）。
  • 呼び出し先: lib/simplehash.h のハッシュ関数（hash_bytes）。

関連データ構造

  • EquivalenceClass: 導出元の等価クラス。
  • EquivalenceMember: 等価クラス内の個別の式。
  • ECDerivesEntry: このキーを保持し、ハッシュテーブルの要素として機能する構造体。

失敗モードと境界条件

  • 構造体内のポインタが NULL の場合、ハッシュ計算や比較時に未定義の動作を引き起こす可能性があるため、初期化時に適切な値が設定されている必要があります。

保守担当者向けメモ

  • 本構造体は simplehash のキーとして定義されています。ハッシュテーブルの検索効率を維持するため、フィールドの追加や変更を行う際は hash_bytes の対象範囲および比較ロジックの整合性に注意してください。
