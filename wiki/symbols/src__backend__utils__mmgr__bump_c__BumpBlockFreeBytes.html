BumpBlockFreeBytes

定義

  • 種別: static inline function
  • 定義箇所: src/backend/utils/mmgr/bump.c:584

目的

指定された BumpBlock 内で、現在利用可能な空きメモリサイズ（バイト単位）を算出します。

シグネチャまたは形状

static inline Size BumpBlockFreeBytes(BumpBlock *block)

入力と出力

  • **入力**: BumpBlock *block - 空き容量を確認する対象のバンプブロックへのポインタ。
  • **出力**: Size - ブロック内の endptr（終端）と freeptr（現在の割り当て開始位置）の差分。

前提条件と不変条件

  • 引数 block は有効な BumpBlock 構造体を指している必要があります。
  • block->endptr は常に block->freeptr 以上の値を持つことが前提となります。

アルゴリズムまたは振る舞い

1. ブロック構造体内の endptr（ブロックの末尾アドレス）から freeptr（次に割り当て可能なアドレス）を減算します。 2. その結果を Size 型として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/bump.c:522 において、メモリ割り当て要求サイズがブロックの空き容量を超えていないか判定するために使用されています。
  • **呼び出し先**:
  • 外部関数を呼び出さず、構造体のメンバアクセスのみを行います。

関連データ構造

  • BumpBlock: バンプアロケータが管理するメモリブロックのメタデータ構造体。freeptr および endptr を保持します。

失敗モードと境界条件

  • block が NULL の場合、ポインタ参照エラーが発生します。
  • freeptr が endptr を超えているようなメモリ破壊状態にある場合、計算結果は負の値（アンダーフロー）となります。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスを重視して頻繁に呼び出されることを想定しています。
  • 割り当てロジックの判定基準となる重要な関数であるため、変更時は BumpAlloc 等のブロック管理ロジックへの影響を考慮してください。
