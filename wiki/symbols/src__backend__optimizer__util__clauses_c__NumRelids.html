NumRelids

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/clauses.c:2130

目的

指定された式（clause）が参照するリレーション（テーブル）の数をカウントします。ただし、外部結合（outer join）に関連するリレーションは除外して計算します。

シグネチャまたは形状

int NumRelids(PlannerInfo *root, Node *clause)

入力と出力

  • **入力**:
  • root: プランナのコンテキスト情報（PlannerInfo）。
  • clause: 解析対象の式ツリー（Node）。
  • **出力**:
  • 外部結合のリレーションを除いた、式が参照するリレーションの数（int）。

前提条件と不変条件

  • root および clause は有効なポインタである必要があります。
  • root->outer_join_rels が適切に初期化されていることを前提としています。

アルゴリズムまたは振る舞い

1. pull_varnos(root, clause) を呼び出し、式ツリー内のすべての変数（varno）を抽出して Relids セットを取得します。 2. bms_del_members を使用し、取得したセットから root->outer_join_rels に含まれるリレーションを除去します。 3. bms_num_members を使用して、残ったリレーションの数をカウントします。 4. bms_free で一時的なセットを解放し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/clausesel.c
  • src/backend/utils/adt/selfuncs.c
  • **呼び出し先**:
  • pull_varnos
  • bms_del_members
  • bms_num_members
  • bms_free

関連データ構造

  • PlannerInfo: プランナのグローバルな状態を保持する構造体。
  • Relids: リレーションIDのビットマップセット。

失敗モードと境界条件

  • clause がリレーションを参照しない場合、結果は 0 となります。
  • clause が NULL の場合、pull_varnos の挙動に依存します。

保守担当者向けメモ

  • 本関数は、クエリの結合条件（Join Clause）か否かを判定する際の重要な指標として利用されます。
  • bms_del_members による除外処理は、外部結合の特殊な制約を考慮するためのものであり、このロジックを変更する際はプランナの結合順序決定への影響を考慮してください。
