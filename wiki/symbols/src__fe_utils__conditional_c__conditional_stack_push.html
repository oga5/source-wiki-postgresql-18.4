conditional_stack_push

定義

  • 種別: function
  • 定義箇所: src/fe_utils/conditional.c:52

目的

ネストされた \if ブロックの状態を管理するスタックに、新しい実行コンテキスト（ifState）をプッシュする。

シグネチャまたは形状

void conditional_stack_push(ConditionalStack cstack, ifState new_state)

入力と出力

  • cstack (ConditionalStack): 操作対象のスタック構造体。
  • new_state (ifState): 新たにスタックへ積むブロックの状態（IFSTATE_TRUE, IFSTATE_FALSE, IFSTATE_IGNORED 等）。
  • 戻り値: なし。

前提条件と不変条件

  • cstack は初期化済みであること。
  • pg_malloc によるメモリ確保が成功すること（失敗時は例外として処理される）。

アルゴリズムまたは振る舞い

1. IfStackElem 型の新しい要素をメモリ確保する。 2. 指定された new_state を要素の if_state に設定する。 3. query_len および paren_depth を初期値 -1 に設定する（これらは後続の set 関数で更新されるメタデータ用）。 4. 新しい要素をスタックの先頭（cstack->head）に挿入し、既存のスタックをその next に繋ぐ。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pgbench/pgbench.c
  • src/bin/psql/command.c
  • 呼び出し先:
  • pg_malloc (メモリ確保)

関連データ構造

  • ConditionalStack: スタック全体を管理する構造体。
  • IfStackElem: スタックの各要素を保持する構造体。
  • ifState: ブロックの状態を定義する列挙型。

失敗モードと境界条件

  • メモリ不足時: pg_malloc が失敗した場合、プログラムは終了する。

保守担当者向けメモ

  • 本関数はスタックの先頭を更新するのみであり、メタデータ（query_len 等）の具体的な値は、この後に呼び出される conditional_stack_set_query_len 等で設定される前提となっている。
