scram_get_mechanisms

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-scram.c:205

目的

クライアントに対して、サーバーがサポートするSASL SCRAM認証メカニズムのリストを通知する。

シグネチャまたは形状

static void scram_get_mechanisms(Port *port, StringInfo buf)

入力と出力

  • port: 現在の接続状態（SSL使用状況の確認に使用）。
  • buf: 認証メカニズム名がヌル文字区切りで追記されるバッファ。

前提条件と不変条件

  • buf は appendStringInfo 系関数で操作可能な状態であること。
  • SSLが有効かつ使用中の場合、チャネルバインディング対応のメカニズムが優先的に通知される。

アルゴリズムまたは振る舞い

1. SSL接続が確立されている場合、SCRAM_SHA_256_PLUS_NAME をバッファに追加し、ヌル文字で区切る。 2. 常に SCRAM_SHA_256_NAME をバッファに追加し、ヌル文字で区切る。 3. 優先度の高い（チャネルバインディング対応の）メカニズムがリストの先頭に来るように順序付けを行う。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/libpq/auth-scram.c 内の pg_be_scram_mech 構造体（認証メカニズム定義）から参照されている。
  • 呼び出し先: appendStringInfoString, appendStringInfoChar（PostgreSQLの文字列操作ユーティリティ）。

関連データ構造

  • Port: 接続情報を保持する構造体。
  • StringInfo: 動的文字列バッファ。

失敗モードと境界条件

  • SSLが無効な環境では、チャネルバインディング対応のメカニズムは通知されない。
  • メモリ不足により appendStringInfo が失敗した場合、PostgreSQLの標準的なエラーハンドリング（elog 等）によりプロセスが中断される可能性がある。

保守担当者向けメモ

  • 新たなSCRAMメカニズムを追加する場合は、この関数の順序と条件分岐を更新する必要がある。
  • 現在の設計では、チャネルバインディングのサポートはSSLの有無に完全に依存している。
