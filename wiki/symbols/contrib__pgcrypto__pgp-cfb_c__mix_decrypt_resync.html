mix_decrypt_resync

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-cfb.c:156

目的

古いPGP形式で採用されていた特殊な再同期（resync）処理を伴うCFB復号を行う。

シグネチャまたは形状

static int mix_decrypt_resync(PGP_CFB *ctx, const uint8 *data, int len, uint8 *dst)

入力と出力

  • ctx: 暗号化コンテキスト（PGP_CFB）。
  • data: 復号対象の入力データ。
  • len: 入力データの長さ。
  • dst: 復号結果を格納するバッファ。
  • 戻り値: 処理したバイト数。

前提条件と不変条件

  • ctx->resync が有効な場合に呼び出される。
  • ctx->block_no == 2 の場合、ブロック境界での特殊な再同期ロジックが適用される。

アルゴリズムまたは振る舞い

1. **ブロック#2の特殊処理**: ctx->block_no == 2 の場合、最初の2バイトを処理し、ctx->pos が2に達した時点で ctx->fr（フィードバックレジスタ）を再構成する。

  • ctx->encbuf の後半部分を ctx->fr の前半に、ctx->encbuf の前半2バイトを ctx->fr の後半にコピーする。
  • ctx->pos を0にリセットする。

    • 2. **データ処理**: 入力データと ctx->fre（暗号化されたフィードバックレジスタ）のXORを取り、dst に書き込む。 3. **状態更新**: ctx->pos を進め、処理したバイト数を返す。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/pgp-cfb.c 内の cfb_process（ctx->resync が真の場合に選択される）。
  • 呼び出し先: 直接的な関数呼び出しはなし（memcpy 等の標準ライブラリ関数を除く）。

関連データ構造

  • PGP_CFB: 暗号化コンテキスト。fr（フィードバックレジスタ）、fre（暗号化済みレジスタ）、encbuf、pos、block_no を保持。

失敗モードと境界条件

  • len が ctx->block_no == 2 の際の残りバイト数より小さい場合、部分的な処理が行われる。
  • ctx->pos がブロックサイズを跨ぐ際の境界処理は、ctx->pos の更新によって管理される。

保守担当者向けメモ

  • 本関数はPGPの古い仕様に依存した複雑なロジックを含んでいる。デバッグ時は ctx->resync フラグの状態と、ctx->block_no == 2 におけるレジスタの並び替えロジックに注意すること。
