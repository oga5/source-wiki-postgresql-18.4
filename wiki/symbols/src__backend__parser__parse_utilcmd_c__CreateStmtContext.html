CreateStmtContext

定義

  • 種別: struct
  • 定義箇所: src/backend/parser/parse_utilcmd.c:73

目的

CREATE TABLE や ALTER TABLE などの DDL コマンドの解析および変換処理において、変換中の状態を保持・管理するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    ParseState *pstate;
    const char *stmtType;
    RangeVar   *relation;
    Relation    rel;
    List       *inhRelations;
    bool        isforeign;
    bool        isalter;
    List       *columns;
    List       *ckconstraints;
    List       *nnconstraints;
    List       *fkconstraints;
    List       *ixconstraints;
    List       *likeclauses;
    List       *blist;
    List       *alist;
    IndexStmt  *pkey;
    bool        ispartitioned;
    PartitionBoundSpec *partbound;
    bool        ofType;
} CreateStmtContext;

入力と出力

  • **入力**: 変換対象の DDL 文の構文解析結果（ParseState 等）が初期化時に設定されます。
  • **出力**: 変換過程で生成された一連のユーティリティコマンドが blist（前処理用リスト）および alist（後処理用リスト）に蓄積されます。

前提条件と不変条件

  • blist にはテーブル作成前に実行すべきコマンドが格納されます。
  • alist にはテーブル作成後に実行すべきコマンドが格納されます。
  • isalter が真の場合、rel には既にオープンおよびロックされた Relation が保持されている必要があります。

アルゴリズムまたは振る舞い

1. **状態保持**: 変換処理の開始時に ParseState と共に初期化されます。 2. **要素の分類**: tableElts を走査し、列定義や各種制約を対応するリスト（ckconstraints, fkconstraints 等）に振り分けます。 3. **コマンド蓄積**: transformColumnDefinition や transformTableConstraint 等の関数を通じて、実行に必要なコマンドを blist または alist に追加します。 4. **ポストプロセス**: transformIndexConstraints 等により、制約に基づくインデックス生成コマンドを確定させます。

呼び出し元と呼び出し先

  • **呼び出し元**: transformCreateStmt および transformAlterTableStmt（直接の呼び出し関係はソースコードの構造から推測されます）。
  • **呼び出し先**: 以下の関数群が本構造体を引数として受け取ります。
  • transformColumnDefinition
  • transformTableConstraint
  • transformTableLikeClause
  • transformOfType
  • transformIndexConstraints
  • transformFKConstraints
  • transformCheckConstraints
  • transformConstraintAttrs
  • transformColumnType
  • transformPartitionCmd

関連データ構造

  • ParseState: 解析の全体的な状態。
  • List: コマンドや制約を保持するための汎用リスト。
  • IndexStmt: プライマリキー等のインデックス定義。
  • PartitionBoundSpec: パーティション境界の定義。

失敗モードと境界条件

  • isalter が真であるにもかかわらず rel が適切にロックされていない場合、後続の処理でエラーが発生する可能性があります。
  • LIKE 句の処理において、参照先テーブルの属性解決に失敗した場合、変換処理が中断されます。

保守担当者向けメモ

  • 本構造体は parse_utilcmd.c 内の変換ロジックのハブです。フィールドの追加や変更は、関連する transform* 系関数すべてに影響を与えるため注意してください。
  • blist と alist の使い分けは、テーブル作成の前後関係を制御する重要な設計です。特に LIKE 句や制約の適用順序を維持するために必須となります。
