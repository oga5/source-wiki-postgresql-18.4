CreateTupleDescCopyConstr

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tupdesc.c:339

目的

既存の TupleDesc を、関連する制約（TupleConstr）を含めて完全に複製する。

シグネチャまたは形状

TupleDesc CreateTupleDescCopyConstr(TupleDesc tupdesc)

入力と出力

  • **入力**: tupdesc (複製元の TupleDesc 構造体へのポインタ)
  • **出力**: 複製された新しい TupleDesc 構造体へのポインタ。

前提条件と不変条件

  • 入力 tupdesc は有効な TupleDesc であること。
  • 複製先の TupleDesc は CreateTemplateTupleDesc によって新しくメモリ確保される。
  • TupleConstr が存在する場合、その内部の文字列（adbin, ccname, ccbin）および missing 値は個別にメモリ確保（pstrdup / datumCopy）され、ディープコピーが行われる。

アルゴリズムまたは振る舞い

1. CreateTemplateTupleDesc で新しい記述子を初期化。 2. 属性配列を memcpy でフラットコピーし、各属性に対して populate_compact_attribute を実行して CompactAttribute を再構築する。 3. tupdesc->constr が存在する場合、TupleConstr 構造体全体を新規確保し、以下の要素をコピーする：

  • has_not_null, has_generated_stored, has_generated_virtual フラグ。
  • defval（デフォルト値）配列とその内部の adbin 文字列。
  • missing（欠損値）配列とその内部の datum 値（datumCopy を使用）。
  • check（チェック制約）配列とその内部の ccname, ccbin 文字列。

    • 4. 最後に tdtypeid および tdtypmod をコピーして返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c
  • src/backend/replication/pgoutput/pgoutput.c
  • src/backend/utils/cache/catcache.c
  • src/backend/utils/cache/typcache.c
  • **呼び出し先**:
  • CreateTemplateTupleDesc
  • populate_compact_attribute
  • palloc0, palloc, pstrdup, datumCopy（メモリ管理およびデータコピー関数）

関連データ構造

  • TupleDesc: タプル記述子のメイン構造体。
  • TupleConstr: 制約情報を保持する構造体。
  • FormData_pg_attribute: 属性定義データ。
  • CompactAttribute: 最適化用属性メタデータ。

失敗モードと境界条件

  • メモリ不足時、palloc 系関数によりエラーが発生する可能性がある。
  • constr が NULL の場合、制約のコピーはスキップされる。
  • num_defval や num_check が 0 の場合、対応する配列のコピー処理は実行されない。

保守担当者向けメモ

  • TupleDescCopy と異なり、本関数は制約（TupleConstr）のディープコピーを伴うため、メモリ消費量が増加する点に注意すること。
  • FormData_pg_attribute を直接操作した後は populate_compact_attribute が必須であるため、この関数内での順序を維持すること。
