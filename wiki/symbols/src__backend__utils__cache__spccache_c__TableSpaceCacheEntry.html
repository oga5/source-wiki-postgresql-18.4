TableSpaceCacheEntry

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/cache/spccache.c:38

目的

テーブルスペースのオプション設定をメモリ上にキャッシュし、システムカタログ（pg_tablespace）へのアクセス負荷を軽減するためのハッシュテーブル用エントリ構造体です。

シグネチャまたは形状

typedef struct
{
    Oid         oid;            /* 検索キー - 構造体の先頭である必要がある */
    TableSpaceOpts *opts;       /* オプション設定、設定がない場合は NULL */
} TableSpaceCacheEntry;

入力と出力

  • **入力**: oid（テーブルスペースのOID）をキーとしてハッシュテーブル検索に使用されます。
  • **出力**: opts（解析済みの TableSpaceOpts 構造体へのポインタ）を保持し、プランナ等の要求に応じて提供します。

前提条件と不変条件

  • oid フィールドは構造体の先頭に配置されなければなりません。
  • opts が NULL の場合、そのテーブルスペースには固有のオプション設定が存在しないことを示します。

アルゴリズムまたは振る舞い

  • TableSpaceCacheHash ハッシュテーブル内でエントリとして管理されます。
  • キャッシュミスが発生した場合、get_tablespace() 関数がシステムカタログから情報をロードし、本構造体を生成してハッシュテーブルに格納します。
  • InvalidateTableSpaceCacheCallback が呼び出されると、キャッシュ全体がフラッシュされるため、本エントリも破棄されます。

呼び出し元と呼び出し先

  • **呼び出し元**: get_tablespace() がキャッシュ検索およびエントリ生成の主要な呼び出し元です。また、get_tablespace_page_costs()、get_tablespace_io_concurrency()、get_tablespace_maintenance_io_concurrency() が本構造体を通じて設定値を取得します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • TableSpaceCacheHash: 本構造体を格納するハッシュテーブル。
  • TableSpaceOpts: opts フィールドが指し示す、解析済みのオプション設定構造体。

失敗モードと境界条件

  • **キャッシュミス**: 初回アクセス時や無効化直後はキャッシュミスが発生し、カタログ検索が行われます。
  • **メモリ不足**: ハッシュテーブルへの挿入時にメモリ不足が発生する可能性があります。
  • **無効化**: pg_tablespace が更新されると、本構造体を含むキャッシュ全体がクリアされます。

保守担当者向けメモ

  • 本構造体は各バックエンドプロセスが個別に保持するローカルキャッシュの一部です。
  • 現在の設計ではエントリ単位の無効化ではなく、キャッシュ全体のフラッシュを行う戦略をとっています。テーブルスペース数が増大した場合は、エントリ単位の管理への変更を検討してください。
