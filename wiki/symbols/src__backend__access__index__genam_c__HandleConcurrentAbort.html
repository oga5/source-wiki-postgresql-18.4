HandleConcurrentAbort

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/genam.c:490

目的

システムカタログスキャン中に、監視対象のトランザクション（CheckXidAlive）が並行してアボート（ロールバック）されたかどうかを検出し、エラーを発生させることでデータの一貫性を保護します。

シグネチャまたは形状

static inline void HandleConcurrentAbort(void)

入力と出力

  • 入力: グローバル変数またはスコープ内の CheckXidAlive（トランザクションID）。
  • 出力: なし（正常時は何も返さない）。
  • 副作用: CheckXidAlive がアボート済みであると判定された場合、ereport(ERROR, ...) によりトランザクションを強制終了させます。

前提条件と不変条件

  • CheckXidAlive が有効なトランザクションIDとして設定されていること。
  • 本関数は、システムカタログスキャンのループ内など、トランザクションの状態変化を監視すべき箇所で呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. CheckXidAlive が有効なIDであるかを確認します。 2. 以下の条件をすべて満たす場合、トランザクションがアボートしたと判断します。

  • CheckXidAlive が進行中（InProgress）ではない。
  • CheckXidAlive がコミット済み（DidCommit）ではない。

    • 3. 上記条件に合致する場合、ERRCODE_TRANSACTION_ROLLBACK を伴うエラーを発生させます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/index/genam.c 内のシステムカタログスキャン関連関数（systable_getnext 等の内部処理を含む複数箇所）。
  • 呼び出し先:
  • TransactionIdIsValid
  • TransactionIdIsInProgress
  • TransactionIdDidCommit
  • ereport

関連データ構造

  • TransactionId: トランザクション識別子。

失敗モードと境界条件

  • CheckXidAlive が無効な場合、本関数は何もしません。
  • トランザクションが正常に進行中またはコミット済みであれば、エラーは発生しません。
  • 競合が発生した場合、システムカタログスキャンは中断され、エラーが上位へ伝播します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスへの影響を最小限に抑える設計となっています。
  • CheckXidAlive の状態判定ロジックは、PostgreSQL のトランザクション管理の基本ルールに依存しているため、トランザクション状態遷移の仕様変更時には注意が必要です。
