SchemaQuery

定義

  • 種別: struct
  • 定義箇所: src/bin/psql/tab-complete.in.c:129

目的

psql の対話型シェルにおけるタブ補完機能において、データベースカタログ（テーブル、関数、制約など）から補完候補を動的に取得するためのクエリ定義を保持する。

シグネチャまたは形状

typedef struct SchemaQuery {
    int min_server_version;
    const char *catname;
    const char *selcondition;
    const char *viscondition;
    const char *namespace;
    const char *result;
    bool use_distinct;
    const char *const *keywords;
    const char *refname;
    const char *refviscondition;
    const char *refnamespace;
} SchemaQuery;

入力と出力

  • **入力**: 構造体の各フィールドに設定された SQL クエリの断片（カタログ名、条件式、結果フィールド名など）。
  • **出力**: 構造体定義自体はデータ保持用であり、この定義に基づき生成された SQL がバックエンドへ発行され、補完候補のリストが返却される。

前提条件と不変条件

  • catname にはクエリ対象のカタログ名（エイリアス含む）を指定する。
  • min_server_version が 0 でない場合、そのバージョン以降でのみ適用される。
  • completion_ref_object 等の参照機能を使用する場合、catname に定義カタログを含め、selcondition で結合条件を記述する必要がある。

アルゴリズムまたは振る舞い

1. 補完ロジックが SchemaQuery を参照し、catname や selcondition を組み合わせて SQL クエリを構築する。 2. 必要に応じて pg_namespace との結合や、スキーマ修飾の有無に応じた viscondition の適用を行う。 3. use_distinct が真の場合、DISTINCT 句を付与して重複を除去する。 4. keywords が指定されている場合、クエリ結果に加えて静的なキーワード候補を補完リストに追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/tab-complete.in.c 内の各静的インスタンス（Query_for_constraint_of_table, Query_for_list_of_datatypes 等）。
  • **呼び出し先**: 直接的な関数呼び出し関係は定義されていないが、補完実行時に _complete_from_query() 等の内部関数によって本構造体のデータが参照される。

関連データ構造

  • completion_squery: SchemaQuery へのポインタを保持する静的変数。
  • Query_for_... 系変数群: 本構造体を用いて定義された具体的な補完クエリ定義。

失敗モードと境界条件

  • catname が NULL の場合、そのエントリは無視される（リストの末尾等で使用）。
  • クエリ実行に失敗した場合、ユーザーにはエラーが表示されない設計となっている。
  • 補完候補が多すぎる場合やネットワーク遅延がある場合、補完動作が重くなる可能性がある。

保守担当者向けメモ

  • 本構造体は gen_tabcomplete.pl によるビルド処理の影響を受ける可能性がある。
  • catname に複数のテーブルを指定する場合は、selcondition で適切な結合条件を明示すること。
  • スキーマ修飾をサポートしない場合は、namespace フィールドを NULL に設定する。
