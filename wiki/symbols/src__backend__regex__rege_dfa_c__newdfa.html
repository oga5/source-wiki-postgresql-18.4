newdfa

定義

  • 種別: function
  • 定義箇所: src/backend/regex/rege_dfa.c:606

目的

正規表現エンジンにおけるDFA（決定性有限オートマトン）の実行状態を保持する dfa 構造体を初期化し、メモリを割り当てる。

シグネチャまたは形状

struct dfa *newdfa(struct vars *v, struct cnfa *cnfa, struct colormap *cm, struct smalldfa *sml)

入力と出力

  • **入力**:
  • v: 正規表現実行時のコンテキスト情報。
  • cnfa: コンパイル済みのNFA。
  • cm: 文字クラスの定義（カラーマップ）。
  • sml: 事前割り当て済みの smalldfa 構造体（NULL可）。
  • **出力**: 初期化された dfa 構造体へのポインタ。失敗時は NULL を返し、エラーコードをセットする。

前提条件と不変条件

  • cnfa は NULL であってはならず、状態数（nstates）は 0 より大きいこと。
  • wordsper（状態ビットマップのワード数）および ncolors が INT_MAX を超えるような巨大な値である場合、REG_ETOOBIG エラーとなる。

アルゴリズムまたは振る舞い

1. **メモリ割り当て戦略**:

  • 小規模なDFA（FEWSTATES かつ FEWCOLORS）の場合、提供された sml を使用するか、スタック/ヒープから smalldfa を確保する。
  • 大規模なDFAの場合、dfa 構造体および関連する配列（ssets, statesarea, outsarea, incarea）を個別に MALLOC する。

    • 2. **初期化**:

  • nssets は REG_SMALL フラグに応じて設定される。
  • backno を -1 に初期化し、search ポインタを ssets の先頭に向ける。
  • cnfa や cm への参照を保持し、各領域のポインタを適切に設定する。

    • 3. **エラー処理**: メモリ確保に失敗した場合は ERR(REG_ESPACE) を呼び出し、確保済みのリソースがあれば freedfa で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regexec.c 内の各所（newdfa を呼び出し、DFA 実行環境を構築する）。
  • **呼び出し先**: MALLOC, MALLOC_ARRAY, ERR, freedfa。

関連データ構造

  • struct dfa: DFA の実行状態を管理するメイン構造体。
  • struct cnfa: コンパイル済みのNFA。
  • struct smalldfa: 小規模DFA用の事前割り当て領域。
  • struct sset: DFA の状態セット。

失敗モードと境界条件

  • **メモリ不足**: MALLOC が失敗した場合、REG_ESPACE を報告し NULL を返す。
  • **サイズ制限**: cnfa の状態数や色数が大きすぎる場合、REG_ETOOBIG を報告し NULL を返す。

保守担当者向けメモ

  • sml が提供されない場合、小規模DFAは MALLOC される。このとき ismalloced フラグが true に設定され、後の freedfa で適切に解放される。
  • wordsper の計算は UBITS に依存しており、ビット演算による状態管理の効率化が図られている。
  • d->work は statesarea の末尾に配置されるため、メモリレイアウト変更時は注意が必要。
