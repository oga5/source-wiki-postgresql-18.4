pg_get_ruledef_worker

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ruleutils.c:596

目的

システムカタログ pg_rewrite から指定されたルール（ruleoid）の定義を読み取り、SQL 文として逆コンパイル（deparse）した文字列を生成する。

シグネチャまたは形状

static char *pg_get_ruledef_worker(Oid ruleoid, int prettyFlags)

入力と出力

  • **入力**:
  • ruleoid (Oid): 取得対象のルールの OID。
  • prettyFlags (int): 逆コンパイル時の整形オプション（インデントや改行の制御）。
  • **出力**:
  • char *: 逆コンパイルされた SQL 文の文字列。該当するルールが見つからない場合は NULL を返す。

前提条件と不変条件

  • SPI（Server Programming Interface）経由で pg_rewrite を検索するため、呼び出し元は SPI コンテキストを適切に管理する必要がある。
  • 内部的に plan_getrulebyoid を使用してクエリ計画をキャッシュする。

アルゴリズムまたは振る舞い

1. initStringInfo で出力バッファを初期化する。 2. SPI_connect を呼び出し、SPI マネージャに接続する。 3. plan_getrulebyoid が未初期化の場合、SPI_prepare を用いて pg_rewrite を検索するクエリ計画を作成し、SPI_keepplan で保持する。 4. SPI_execute_plan を実行し、指定された ruleoid に対応するタプルを取得する。 5. タプルが存在する場合、make_ruledef を呼び出してバッファに SQL 定義を書き込む。 6. SPI_finish を呼び出して SPI マネージャから切断する。 7. バッファが空であれば NULL を、そうでなければ構築された文字列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/ruleutils.c 内の pg_get_ruledef および pg_get_ruledef_ext から呼び出される。
  • **呼び出し先**:
  • SPI_connect, SPI_prepare, SPI_keepplan, SPI_execute_plan, SPI_finish (SPI API)
  • make_ruledef (逆コンパイルの実行)
  • initStringInfo, elog (PostgreSQL ユーティリティ)

関連データ構造

  • StringInfoData: 逆コンパイル結果を格納する可変長文字列バッファ。
  • HeapTuple, TupleDesc: pg_rewrite から取得したタプル情報。
  • SPIPlanPtr: pg_rewrite 検索用のキャッシュされたクエリ計画。

失敗モードと境界条件

  • SPI_prepare が失敗した場合、elog(ERROR) で終了する。
  • SPI_execute_plan が SPI_OK_SELECT 以外を返した場合、elog(ERROR) で終了する。
  • SPI_finish が失敗した場合、elog(ERROR) で終了する。
  • 対象の ruleoid が存在しない場合、バッファは空のままとなり、関数は NULL を返す。

保守担当者向けメモ

  • 本関数は pg_rewrite へのアクセスに SysCache ではなく SPI を使用している。これは、アクセス権限チェックを確実に行うための設計である。
  • plan_getrulebyoid は静的変数としてキャッシュされるため、モジュール終了時や計画の無効化に関する考慮が必要となる場合がある。
