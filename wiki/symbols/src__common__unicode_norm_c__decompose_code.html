decompose_code

定義

  • 種別: function
  • 定義箇所: src/common/unicode_norm.c:321

目的

Unicode 文字をその構成要素に分解し、指定されたバッファへ格納する。

シグネチャまたは形状

static void decompose_code(pg_wchar code, bool compat, pg_wchar **result, int *current)

入力と出力

  • code: 分解対象の Unicode コードポイント。
  • compat: 互換分解（Compatibility Decomposition）を行うかどうかのフラグ。
  • result: 分解結果を格納する配列へのポインタ（入出力）。
  • current: 現在の書き込み位置を示すインデックス（入出力）。

前提条件と不変条件

  • result が指すバッファは、分解後の全文字を格納するのに十分なサイズが確保されている必要がある。
  • ハングル文字（SBASE から SBASE + SCOUNT の範囲）は、テーブルを参照せずアルゴリズム的に分解される。

アルゴリズムまたは振る舞い

1. **ハングル処理**: 対象がハングル文字の場合、計算により L（初声）、V（中声）、T（終声）に分解し、result に順次格納する。 2. **テーブル検索**: ハングル以外の場合、get_code_entry を用いて分解情報を取得する。 3. **終了条件**: 分解情報が存在しない、または分解サイズが 0 の場合、対象文字をそのまま result に格納して終了する。 4. **再帰的分解**: 分解情報が存在する場合、get_code_decomposition で取得した各コードポイントに対して本関数を再帰的に呼び出し、完全な分解を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/unicode_norm.c 内の正規化処理フロー（unicode_normalize 等から間接的に呼び出される）。
  • **呼び出し先**:
  • get_code_entry
  • get_code_decomposition
  • decompose_code（再帰呼び出し）

関連データ構造

  • pg_unicode_decomposition: 文字の分解情報を保持する構造体。

失敗モードと境界条件

  • entry が NULL の場合、分解を行わず対象文字をそのまま出力する。
  • 互換分解が不要な設定（compat == false）で、対象が互換分解のみを持つ文字である場合、分解を行わずそのまま出力する。

保守担当者向けメモ

  • 本関数は再帰的に呼び出されるため、分解の深さがスタックオーバーフローを引き起こさないよう注意が必要である。
  • ハングル処理のロジックは Unicode TR15 に準拠している。変更時は当該仕様を確認すること。
