EnumUncommitted

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_enum.c:707

目的

指定された列挙型（OID）が、現在のトランザクション内で作成され、まだコミットされていない状態にあるかどうかを判定します。

シグネチャまたは形状

bool EnumUncommitted(Oid enum_id)

入力と出力

  • **入力**: enum_id (Oid) - 検査対象の列挙型の OID。
  • **出力**: bool - 未コミットであれば true、それ以外（コミット済み、または存在しない）であれば false を返します。

前提条件と不変条件

  • uncommitted_enum_values ハッシュテーブルが初期化されているか、あるいは NULL であることを前提とします。
  • トランザクション内で未コミットの列挙型値が追跡されている場合にのみ、正確な判定が可能です。

アルゴリズムまたは振る舞い

1. 未コミットの列挙型値を保持するハッシュテーブル uncommitted_enum_values が NULL の場合、未コミットの列挙型は存在しないと見なし false を返します。 2. ハッシュテーブルが存在する場合、hash_search を用いて enum_id をキーとして検索を行います。 3. 検索結果（found）に基づき、該当するエントリが存在すれば true を、存在しなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/enum.c:90
  • **呼び出し先**:
  • hash_search (PostgreSQL 内部ハッシュテーブル検索関数)

関連データ構造

  • uncommitted_enum_values: 未コミットの列挙型・値を追跡するために使用されるハッシュテーブル。

失敗モードと境界条件

  • uncommitted_enum_values が初期化されていない場合、安全に false を返す設計となっています。
  • 検索対象の enum_id がハッシュテーブル内に存在しない場合、正しく false を返します。

保守担当者向けメモ

  • 本関数は、トランザクションロールバック時にインデックスの不整合が発生するのを防ぐための安全策の一部です。
  • pg_enum.c 内の他の未コミット追跡関数（EnumTypeUncommitted 等）と併せて、列挙型の可視性制御に使用されます。
