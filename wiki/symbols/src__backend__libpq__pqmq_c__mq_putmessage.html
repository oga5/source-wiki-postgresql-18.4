mq_putmessage

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqmq.c:117

目的

共有メモリメッセージキュー（shm_mq）を介して、libpqプロトコル形式のメッセージを送信する。

シグネチャまたは形状

int mq_putmessage(char msgtype, const char *s, size_t len)

入力と出力

  • **入力**:
  • msgtype: メッセージタイプを示す文字。
  • s: 送信するデータ本体のバッファ。
  • len: データ本体の長さ。
  • **出力**:
  • 成功時は 0 を返す。
  • 失敗時（キューのデタッチやエラー）は EOF を返す。

前提条件と不変条件

  • pq_mq_handle が有効であること（NULL の場合は無視される）。
  • pq_mq_busy フラグにより、メッセージ送信中の再帰的な呼び出しを防止する。
  • 呼び出し時に shm_mq_sendv を使用して共有メモリへ書き込む。

アルゴリズムまたは振る舞い

1. **再入チェック**: pq_mq_busy が真の場合、デッドロック回避のためキューをデタッチして EOF を返す。 2. **初期化**: pq_mq_busy を真に設定し、メッセージタイプとデータを shm_mq_iovec に格納する。 3. **送信ループ**:

  • shm_mq_sendv を呼び出し、データをキューへ書き込む。
  • 並列リーダーのPIDが設定されている場合、SendProcSignal を発行してリーダーに通知する。
  • キューが満杯（SHM_MQ_WOULD_BLOCK）であれば、WaitLatch で空きを待ち、割り込みチェックを行う。

    • 4. **終了処理**: pq_mq_busy を偽に戻し、送信結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PqCommMqMethods 構造体を通じて、libpqの通信メソッドとして間接的に呼び出される。
  • **呼び出し先**:
  • shm_mq_sendv (共有メモリへの書き込み)
  • SendProcSignal (並列リーダーへの通知)
  • WaitLatch / ResetLatch (待機処理)
  • CHECK_FOR_INTERRUPTS (割り込み確認)
  • shm_mq_detach (再入時のクリーンアップ)

関連データ構造

  • shm_mq: 共有メモリメッセージキューのハンドル。
  • shm_mq_iovec: 送信データ構成用構造体。
  • pq_mq_handle: 現在使用中の共有メモリキューへのポインタ。
  • pq_mq_busy: 送信中状態を示すフラグ。

失敗モードと境界条件

  • **キューが満杯**: WaitLatch を使用して待機し、空きができるまでループする。
  • **再入**: 送信中に割り込み等で再度 mq_putmessage が呼ばれた場合、キューをデタッチして EOF を返す。
  • **キューの消失**: pq_mq_handle が NULL の場合、メッセージを無視して 0 を返す。
  • **送信失敗**: shm_mq_sendv が成功以外を返した場合、EOF を返す。

保守担当者向けメモ

  • 本関数は PqCommMqMethods の一部として、通常のソケット通信の代替として機能する。
  • SendProcSignal の送信先は、論理レプリケーションワーカーか並列ワーカーかによってシグナル種別が分岐する。
