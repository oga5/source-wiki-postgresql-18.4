parallel_vacuum_compute_workers

定義

  • 種別: function
  • 定義箇所: src/backend/commands/vacuumparallel.c:548

目的

並列VACUUM処理において、インデックスの特性とシステム設定に基づき、利用可能な並列ワーカー数を算出する。

シグネチャまたは形状

int parallel_vacuum_compute_workers(Relation *indrels, int nindexes, int nrequested, bool *will_parallel_vacuum)

入力と出力

  • **入力**:
  • indrels: インデックスリレーションの配列。
  • nindexes: インデックスの総数。
  • nrequested: 要求されたワーカー数。
  • will_parallel_vacuum: 各インデックスが並列処理対象となるかを格納する出力用配列。
  • **出力**:
  • 決定された並列ワーカー数（int）。

前提条件と不変条件

  • IsUnderPostmaster が真であること（スタンドアロンバックエンドでは並列不可）。
  • max_parallel_maintenance_workers が 0 より大きいこと。
  • min_parallel_index_scan_size 未満のサイズのインデックスは、並列処理対象から除外される。

アルゴリズムまたは振る舞い

1. **初期チェック**: 並列処理が許可されていない場合は 0 を返す。 2. **インデックス評価**: 各インデックスに対し、amparallelvacuumoptions を参照し、VACUUM_OPTION_NO_PARALLEL でないかつサイズ条件を満たすものを抽出する。 3. **カウント**: 並列バルク削除（VACUUM_OPTION_PARALLEL_BULKDEL）または並列クリーンアップ（VACUUM_OPTION_PARALLEL_CLEANUP / COND_CLEANUP）が可能なインデックス数を集計する。 4. **ワーカー数算出**:

  • 並列可能なインデックスの最大数から、リーダープロセス分として 1 を減算する。
  • 要求数（nrequested）と算出されたインデックス数、および max_parallel_maintenance_workers の最小値をとる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/vacuumparallel.c 内の並列VACUUM初期化プロセスから呼び出される（直接の関数名はソース抜粋からは特定不可）。
  • **呼び出し先**: RelationGetNumberOfBlocks を使用してインデックスサイズを判定する。

関連データ構造

  • Relation: インデックスのメタデータおよびアクセスメソッド情報（rd_indam）を参照するために使用。

失敗モードと境界条件

  • 並列可能なインデックスが 0 または 1 の場合、リーダープロセスのみで処理するため 0 を返す。
  • nrequested が 0 の場合、算出されたインデックス数を上限としてワーカー数を決定する。

保守担当者向けメモ

  • 本関数は、インデックスごとの並列実行能力を amparallelvacuumoptions に依存している。新しいアクセスメソッドを追加する際は、このオプションフラグの適切な設定が必要である。
