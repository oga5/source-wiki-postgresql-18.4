目的

clusterdb ユーティリティのエントリポイントとして、コマンドライン引数の解析、接続パラメータの初期化、および実行モード（全データベース対象または単一データベース対象）の決定と実行を行う。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。
  • 環境変数: PGDATABASE, PGUSER（データベース名が指定されない場合に使用）。
  • **出力**:
  • 正常終了時に 0 を返して終了。
  • 引数エラーや実行時エラー発生時に exit(1) を呼び出し終了。

前提条件と不変条件

  • getopt_long によるオプション解析が完了するまで、接続パラメータは初期状態である。
  • --all オプションと特定のデータベース名指定を同時に行うことはできない。

アルゴリズムまたは振る舞い

1. **初期化**: ロギング、ロケール、ヘルプ/バージョンオプションの処理を行う。 2. **引数解析**: getopt_long を使用し、接続オプション（ホスト、ポート、ユーザー等）および実行モード（--all、--table 等）を解析して変数に格納する。 3. **バリデーション**: 引数の過不足を確認し、エラー時は終了する。 4. **分岐**:

  • --all 指定時: cluster_all_databases を呼び出し、全データベースに対して処理を実行する。
  • 通常時: データベース名を特定し、指定されたテーブルリストがある場合は各テーブルに対して、ない場合はデータベース全体に対して cluster_one_database を順次呼び出す。

    • 5. **終了**: 処理完了後 exit(0) で終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: OS（コマンドラインからの実行）。
  • **呼び出し先**:
  • cluster_all_databases
  • cluster_one_database
  • get_progname, pg_logging_init, set_pglocale_pgservice, handle_help_version_opts, getopt_long, pg_strdup, simple_string_list_append, setup_cancel_handler, get_user_name_or_exit, pg_fatal, pg_log_error, pg_log_error_hint 等のユーティリティ関数。
  • ※ツールによる索引結果に含まれる contrib/amcheck 関連の参照は、本シンボルとは無関係なコードベース内の同名識別子への言及であり、本関数の呼び出し関係ではない。

関連データ構造

  • ConnParams: データベース接続情報を保持する構造体。
  • SimpleStringList: 対象テーブル名を保持するリスト。
  • struct option: getopt_long 用のオプション定義。

失敗モードと境界条件

  • 引数解析エラー: 不正なオプションや引数の過剰指定時にエラーメッセージを表示して終了する。
  • 競合する引数: --all とデータベース名指定が同時に行われた場合、pg_fatal により終了する。
  • データベース名未指定: 環境変数も存在しない場合、現在のユーザー名を取得して使用する。

保守担当者向けメモ

  • 本関数は libpq を直接叩かず、fe_utils 経由で抽象化されたインターフェースを利用している。
  • CLUSTER コマンド実行時のロック挙動に注意すること。
  • 類似ツール（vacuumdb, reindexdb）と共通の設計パターンを採用しているため、変更時はこれらとの一貫性を確認すること。
