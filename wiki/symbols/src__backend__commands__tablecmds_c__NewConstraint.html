NewConstraint

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/tablecmds.c:214

目的

テーブルに対する制約（CHECK制約や外部キー制約）の追加処理において、その定義情報を一時的に保持するための構造体です。

シグネチャまたは形状

typedef struct NewConstraint
{
    char       *name;           /* 制約名（なしの場合はNULL） */
    ConstrType  contype;        /* 制約種別（CHECKまたはFOREIGN） */
    Oid         refrelid;       /* 外部キーの場合の参照先リレーションOID */
    Oid         refindid;       /* 外部キーの場合の参照先インデックスOID */
    bool        conwithperiod;  /* 外部キーがPERIODを使用するかどうか */
    Oid         conid;          /* 外部キーの場合のpg_constraintエントリのOID */
    Node       *qual;           /* CHECK式または外部キー制約ノード */
    ExprState  *qualstate;      /* CHECK式の実行状態 */
} NewConstraint;

入力と出力

本構造体は、ALTER TABLE 等のコマンド実行中にメモリ上で生成・管理されます。palloc0 を用いてゼロ初期化された状態で割り当てられ、制約の検証やカタログ更新の過程で参照されます。

前提条件と不変条件

  • contype が FOREIGN の場合、refrelid や refindid などの外部キー関連フィールドが適切に設定されている必要があります。
  • qualstate は qual に基づく実行状態であり、実行フェーズ（Phase 3）で必要に応じて初期化されます。

アルゴリズムまたは振る舞い

ALTER TABLE コマンドの処理において、制約追加の要求をキューイングするために使用されます。特に複雑な制約（外部キーなど）は、テーブルのデータスキャンや検証が必要となるため、この構造体に情報を格納して処理の後半（Phase 3）まで持ち越されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/tablecmds.c 内の各コマンド処理関数（ALTER TABLE の準備フェーズなど）から palloc0 を介してインスタンス化されます。
  • **呼び出し先**: 構造体内部の qual や qualstate は、PostgreSQL の式評価エンジンによって参照されます。
  • **注記**: 具体的な関数呼び出し関係については、提供されたソース情報からは詳細なスタックトレースを特定できません。

関連データ構造

  • ConstrType: 制約の種類を定義する列挙型。
  • AlteredTableInfo: ALTER TABLE 実行中にテーブルごとの変更状態を保持する構造体であり、NewConstraint のリストを保持します。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • qual に不正な式が含まれている場合、後の検証フェーズでエラーとなる可能性があります。

保守担当者向けメモ

  • 本構造体は ALTER TABLE のフェーズ分けにおいて、制約の検証を遅延させるための重要な役割を担っています。
  • conwithperiod などのフラグは、特定の制約タイプにおける拡張機能のサポートに関連しています。変更を加える際は、src/backend/commands/tablecmds.c 内の Phase 3 処理への影響を確認してください。
