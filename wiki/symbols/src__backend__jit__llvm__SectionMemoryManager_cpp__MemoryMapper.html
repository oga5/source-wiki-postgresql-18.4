MemoryMapper

定義

  • 種別: class (デストラクタ定義)
  • 定義箇所: src/backend/jit/llvm/SectionMemoryManager.cpp:365

目的

OSからのメモリ確保を抽象化するためのインターフェースを提供します。SectionMemoryManager がJIT生成コードやデータを配置する際、具体的なメモリ割り当て処理を分離・モック化可能にするために使用されます。

シグネチャまたは形状

class MemoryMapper {
public:
    virtual ~MemoryMapper();
};

入力と出力

  • 入力: なし（デストラクタ）
  • 出力: なし

前提条件と不変条件

  • SectionMemoryManager のインスタンスによって所有または参照されます。
  • デストラクタは default で実装されており、派生クラスのクリーンアップを適切に行う必要があります。

アルゴリズムまたは振る舞い

  • 本シンボルは MemoryMapper クラスの仮想デストラクタの実装です。
  • 派生クラス（例: DefaultMMapper）が破棄される際に、基底クラスとしてのクリーンアップを実行します。

呼び出し元と呼び出し先

  • 呼び出し元: SectionMemoryManager の破棄時、または std::unique_ptr<MemoryMapper> OwnedMMapper がスコープを抜ける際に呼び出されます。
  • 呼び出し先: 派生クラスのデストラクタ（存在する場合）。

関連データ構造

  • SectionMemoryManager: 本クラスを利用するメインのメモリ管理クラス。
  • DefaultMMapper: MemoryMapper を継承した標準的な実装クラス。

失敗モードと境界条件

  • 派生クラスでリソース管理を行っている場合、デストラクタの呼び出し順序やメモリリークに注意が必要です。

保守担当者向けメモ

  • 本クラスは src/include/jit/SectionMemoryManager.h で宣言されています。
  • テスト時にメモリ確保の挙動を制御したい場合は、本クラスを継承したモッククラスを作成し、SectionMemoryManager に注入してください。
