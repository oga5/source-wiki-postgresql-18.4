datapagemap

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_rewind/datapagemap.h:14

目的

データページの変更状態を追跡するためのビットマップ構造を保持する。

シグネチャまたは形状

struct datapagemap
{
    char       *bitmap;
    int         bitmapsize;
};

入力と出力

  • bitmap: ページの状態を示すビットマップを格納する動的メモリ領域へのポインタ。
  • bitmapsize: bitmap が保持するメモリサイズ（バイト単位）。

前提条件と不変条件

  • bitmap は bitmapsize に応じた適切なサイズで割り当てられている必要がある。

アルゴリズムまたは振る舞い

本構造体は、特定のデータページが変更されたかどうかをビット単位で管理するための基盤を提供する。具体的なビット操作ロジックは、ペアとなる src/bin/pg_rewind/datapagemap.c 内の関数群によって実装される。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_rewind/filemap.c が本構造体を参照している。
  • 呼び出し先: 提供されたソース抜粋からは、本構造体が直接呼び出す関数や、本構造体を操作する具体的な関数呼び出しの詳細は特定できない。

関連データ構造

  • datapagemap_t: datapagemap 構造体の typedef エイリアス（src/bin/pg_rewind/datapagemap.h:20 で定義）。
  • datapagemap_iterator: 関連するイテレータ構造体。

失敗モードと境界条件

  • bitmap が NULL の場合や、bitmapsize が不正な値である場合の挙動については、ソース抜粋からは不明。

保守担当者向けメモ

  • 本構造体は pg_rewind におけるページレベルの変更追跡に使用される。
  • src/bin/pg_rewind/datapagemap.c に実装されている操作関数と併せて参照すること。
