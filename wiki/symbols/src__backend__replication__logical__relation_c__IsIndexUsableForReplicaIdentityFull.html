IsIndexUsableForReplicaIdentityFull

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/relation.c:820

目的

論理レプリケーションにおいて、特定のインデックスが REPLICA IDENTITY FULL モードでの更新・削除操作に使用可能かどうかを判定します。

シグネチャまたは形状

bool IsIndexUsableForReplicaIdentityFull(Relation idxrel, AttrMap *attrmap)

入力と出力

  • **入力**:
  • idxrel: 評価対象のインデックスリレーション。
  • attrmap: リモート列とローカル列の対応関係を保持するマップ。
  • **出力**:
  • bool: インデックスが使用可能な場合は true、それ以外は false。

前提条件と不変条件

  • インデックスは部分インデックス（Partial Index）であってはならない。
  • インデックスは少なくとも1つ以上のキー属性を持つ必要がある。

アルゴリズムまたは振る舞い

1. **部分インデックスチェック**: pg_index.indpred が存在する場合、使用不可と判定する。 2. **演算子クラスの検証**: インデックスの各キー列に対して、等価比較（COMPARE_EQ）が可能な演算子族が存在するかを確認する。 3. **型キャッシュの検証**: インデックスに含まれる全属性について、型キャッシュから等価演算子（eq_opr_finfo）が取得可能であることを確認する。 4. **左端フィールドの制約**: インデックスの左端フィールドが式（Expression）ではなく、かつ attrmap を通じてリモートリレーションの列を参照していることを確認する。 5. **AM機能の検証**: インデックスのアクセスメソッドが amgettuple を実装していることを確認する。

関連データ構造

  • Relation: インデックスおよびテーブルのメタデータ構造。
  • AttrMap: リモートとローカルの列マッピング情報。
  • TypeCacheEntry: 型ごとの演算子情報。

失敗モードと境界条件

  • 部分インデックスである場合、即座に false を返す。
  • 等価演算子を提供できないデータ型が含まれる場合、false を返す。
  • インデックスの左端フィールドがリモート列と対応していない場合、false を返す。
  • 使用するインデックスアクセスメソッドが amgettuple をサポートしていない場合、false を返す。

保守担当者向けメモ

  • 本関数は、論理レプリケーションのワーカーがリモートの変更をローカルに適用する際、行を一意に特定するためのインデックス選択ロジックの一部です。
  • attrmap の検証において、attnums が負の値（システム列や無効な列）である場合は使用不可とみなされます。
