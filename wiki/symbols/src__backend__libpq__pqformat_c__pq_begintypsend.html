pq_begintypsend

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqformat.c:325

目的

typsend（外部バイナリ形式）構築のための StringInfo バッファを初期化します。

シグネチャまたは形状

void pq_begintypsend(StringInfo buf)

入力と出力

  • **入力**: StringInfo buf（初期化対象のバッファ構造体へのポインタ）
  • **出力**: なし（引数の buf が更新されます）

前提条件と不変条件

  • 引数 buf は有効な StringInfo 構造体のアドレスである必要があります。
  • 呼び出し後、バッファの先頭4バイトは bytea 長さワード用に予約された状態となります。

アルゴリズムまたは振る舞い

1. initStringInfo(buf) を呼び出し、バッファを初期化します。 2. appendStringInfoCharMacro を4回呼び出し、バッファの先頭に4バイトのヌル文字（\0）を書き込みます。これは後続の処理で bytea の長さ情報を格納するための領域を確保するものです。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/cube/cube.c
  • contrib/hstore/hstore_io.c
  • contrib/ltree/ltree_io.c
  • contrib/ltree/ltxtquery_io.c
  • src/backend/utils/adt/array_userfuncs.c
  • src/backend/utils/adt/arrayfuncs.c
  • src/backend/utils/adt/bool.c
  • **呼び出し先**:
  • initStringInfo
  • appendStringInfoCharMacro

関連データ構造

  • StringInfo: メッセージ構築に使用される動的バッファ構造体。

失敗モードと境界条件

  • initStringInfo の内部実装に依存し、メモリ不足時にはエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は bytea 形式のデータ構築を前提としています。
  • 予約された4バイトは、この後に続くデータ構築処理において、最終的なデータ長を書き込むために使用されることを想定しています。
