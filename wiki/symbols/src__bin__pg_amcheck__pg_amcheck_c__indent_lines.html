indent_lines

定義

  • 種別: function
  • 定義箇所: src/bin/pg_amcheck/pg_amcheck.c:1007

目的

文字列の各行の先頭にインデント（スペース4つ）を挿入し、整形された文字列を生成する。主にエラーメッセージの表示を見やすくするために使用される。

シグネチャまたは形状

static char *indent_lines(const char *str)

入力と出力

  • **入力**: str (整形対象のヌル終端文字列)
  • **出力**: インデントが適用された新しい文字列（呼び出し元で解放が必要なメモリ領域）

前提条件と不変条件

  • str は有効なヌル終端文字列であること。
  • PQExpBuffer API を使用してバッファを管理するため、メモリ不足時には pstrdup やバッファ操作関数内でエラーが発生する可能性がある。

アルゴリズムまたは振る舞い

1. initPQExpBuffer で一時バッファを初期化する。 2. 文字列の先頭に " " を追加する。 3. 入力文字列 str を1文字ずつ走査する。 4. 各文字をバッファに追加する。 5. 改行文字 (\n) が出現し、かつそれが文字列の末尾でない場合、次の行の先頭に " " を追加する。 6. pstrdup を使用してバッファの内容を新しいメモリ領域にコピーし、termPQExpBuffer で一時バッファを解放して結果を返す。

呼び出し元と呼び出し先

関連データ構造

  • PQExpBufferData: 文字列構築のための一時バッファとして使用。

失敗モードと境界条件

  • 空文字列が入力された場合、インデントされたスペースのみが返される。
  • 文字列の末尾が改行で終わっている場合、その後の不要なインデントは挿入されない。
  • メモリ不足時は pstrdup 等の内部関数によりプログラムが終了する可能性がある。

保守担当者向けメモ

  • 本関数は PQExpBuffer を利用しているため、メモリリークを防ぐために termPQExpBuffer が確実に呼ばれる構造になっている。
  • 戻り値は pstrdup で確保されているため、呼び出し側で pfree を行う必要がある。
