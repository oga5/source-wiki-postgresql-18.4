findReplaceTo

定義

  • 種別: function
  • 定義箇所: contrib/unaccent/unaccent.c:310

目的

Trie 木構造を走査し、入力文字列に対して最長一致する置換ルールを検索します。

シグネチャまたは形状

static TrieChar *findReplaceTo(TrieChar *node, const unsigned char *src, int srclen, int *p_matchlen)

入力と出力

  • **入力**:
  • node: 検索を開始する Trie 木のノード。
  • src: 検索対象の文字列（バイト列）。
  • srclen: src の長さ。
  • p_matchlen: 最長一致した長さを格納するための出力用ポインタ。
  • **出力**:
  • 最長一致した置換ルールを持つ TrieChar ノードへのポインタ。一致がない場合は NULL を返します。

前提条件と不変条件

  • p_matchlen は有効な整数ポインタである必要があります。
  • Trie 木は initTrie によって適切に構築されている必要があります。

アルゴリズムまたは振る舞い

1. matchlen を 0 に初期化します。 2. node が存在し、かつ matchlen が srclen 未満である間、以下の処理を繰り返します。

  • 現在のノードから src[matchlen] に対応する子ノードへ遷移します。
  • matchlen をインクリメントします。
  • 現在のノードに replaceTo（置換文字列）が定義されている場合、そのノードを result として保持し、現在の matchlen を *p_matchlen に記録します。
  • 次の文字候補を node->nextChar から取得し、走査を継続します。

    • 3. 走査終了後、最後に記録された最長一致ノードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: unaccent_lexize (contrib/unaccent/unaccent.c:397) から呼び出されます。
  • **呼び出し先**: 提供されたソースコード内では、特定の関数呼び出しは確認されません。

関連データ構造

  • TrieChar: 置換ルールを保持する Trie 木のノード構造体。

失敗モードと境界条件

  • 一致するルールが存在しない場合、result は NULL となり、*p_matchlen は 0 となります。
  • srclen が 0 の場合、ループは実行されず NULL が返されます。

保守担当者向けメモ

  • 本関数はバイト単位で Trie 木を走査します。マルチバイト文字を扱う際は、入力データが正しいエンコーディングであることを前提としています。
