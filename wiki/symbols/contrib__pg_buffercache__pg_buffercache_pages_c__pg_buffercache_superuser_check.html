pg_buffercache_superuser_check

定義

  • 種別: function
  • 定義箇所: contrib/pg_buffercache/pg_buffercache_pages.c:665

目的

pg_buffercache 拡張機能内の特定の関数が、スーパーユーザー権限を持つユーザーによってのみ実行されることを保証するための権限チェックを行います。

シグネチャまたは形状

static void pg_buffercache_superuser_check(char *func_name);

入力と出力

  • **入力**:
  • func_name: 権限チェックに失敗した際にエラーメッセージへ埋め込まれる関数名（文字列）。
  • **出力**:
  • なし（戻り値なし）。

前提条件と不変条件

  • 呼び出し元は、PostgreSQL のスーパーユーザー権限を必要とする操作の直前に本関数を呼び出す必要があります。

アルゴリズムまたは振る舞い

1. superuser() を呼び出し、現在のセッションユーザーがスーパーユーザーであるかを確認します。 2. スーパーユーザーでない場合、ereport を使用して ERRCODE_INSUFFICIENT_PRIVILEGE エラーを発生させ、処理を中断します。 3. エラーメッセージには、引数として渡された func_name が含まれます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • pg_buffercache_evict
  • pg_buffercache_evict_relation
  • pg_buffercache_evict_all
  • **呼び出し先**:
  • superuser() (PostgreSQL 内部関数)
  • ereport() (PostgreSQL 内部関数)

関連データ構造

  • なし。

失敗モードと境界条件

  • 現在のユーザーがスーパーユーザー権限を持たない場合、常に ERROR レベルの例外が発生し、呼び出し元の関数の実行は直ちに終了します。

保守担当者向けメモ

  • 本関数は static であり、contrib/pg_buffercache/pg_buffercache_pages.c 内でのみ使用されます。
  • 権限チェックのロジックを変更する場合は、PostgreSQL の標準的な権限管理ポリシーに従っていることを確認してください。
