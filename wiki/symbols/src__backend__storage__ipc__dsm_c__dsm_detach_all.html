dsm_detach_all

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/dsm.c:774

目的

現在のプロセスが保持しているすべての動的共有メモリ（DSM）セグメントをデタッチし、制御セグメント自体もデタッチすることで、プロセス終了時のクリーンアップを行う。

シグネチャまたは形状

void dsm_detach_all(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてメモリのデタッチと制御セグメントの解放を行う）

前提条件と不変条件

  • dsm_segment_list が初期化されており、プロセスが現在接続しているセグメントを追跡していること。
  • dsm_control が有効な制御セグメントのアドレスを指しているか、あるいは NULL であること。

アルゴリズムまたは振る舞い

1. dsm_segment_list が空になるまで、リストの先頭要素を取得し dsm_detach() を呼び出して個別のセグメントをデタッチする。 2. dsm_control が NULL でない場合、dsm_impl_op を用いて制御セグメント自体をデタッチする。この際、DSM_OP_DETACH 操作を実行し、エラー発生時には ERROR を通知する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/postmaster/launch_backend.c:268
  • 呼び出し先:
  • dsm_detach
  • dsm_impl_op

関連データ構造

  • dsm_segment_list: プロセスが現在接続しているセグメントのリスト。
  • dsm_control: DSM 制御セグメントのアドレス。
  • dsm_control_handle: 制御セグメントのハンドル。
  • dsm_control_impl_private: 制御セグメントの OS 依存のプライベートデータ。
  • dsm_control_mapped_size: 制御セグメントのマップサイズ。

失敗モードと境界条件

  • dsm_control が NULL の場合、制御セグメントのデタッチ処理はスキップされる。
  • dsm_impl_op 実行時にエラーが発生した場合、PostgreSQL のエラー処理メカニズムにより ERROR が発行される。

保守担当者向けメモ

  • 本関数はプロセス終了時のクリーンアップを目的としているため、呼び出し元がトランザクションアボート中であっても安全に実行される必要がある。
  • dsm_segment_list の走査中に dsm_detach がリストを操作するため、リストの整合性が保たれていることを前提としている。
