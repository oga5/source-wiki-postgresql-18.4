gbt_date_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_date.c:150

目的

GiST インデックス検索において、インデックスノードが保持する日付範囲（dateKEY）が、指定されたクエリ条件を満たすかどうかを判定する。

シグネチャまたは形状

Datum gbt_date_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • entry (GISTENTRY*): インデックスエントリ。
  • query (DateADT): 検索対象の日付値。
  • strategy (StrategyNumber): 比較演算子の戦略番号。
  • recheck (bool*): 呼び出し元へ返す再チェックフラグ。
  • **出力**:
  • Datum (bool): 条件に一致する場合は true、一致しない場合は false を返す。

前提条件と不変条件

  • recheck は常に false に設定される（本関数による判定は完全であるため）。
  • entry->key は dateKEY 構造体として解釈可能である必要がある。

アルゴリズムまたは振る舞い

1. entry->key を dateKEY 構造体にキャストし、lower と upper の範囲を抽出する。 2. recheck フラグを false に設定する。 3. 抽出した範囲を GBT_NUMKEY_R 構造体に格納する。 4. gbt_num_consistent を呼び出し、汎用的な数値比較ロジックを用いて判定を行う。 5. 判定結果を PG_RETURN_BOOL で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス検索エンジン（直接の呼び出し元コードは提供されていない）。
  • **呼び出し先**: gbt_num_consistent（btree_utils_num.c で定義）。

関連データ構造

  • dateKEY: 日付の範囲（lower, upper）を保持する構造体。
  • GBT_NUMKEY_R: 比較対象となる範囲を保持する汎用構造体。
  • tinfo: date 型の処理に必要なメタデータを保持する gbtree_ninfo 構造体。

失敗モードと境界条件

  • entry が無効な場合や、dateKEY へのキャストが不適切な場合は予期せぬ動作を引き起こす可能性がある。
  • 内部的に呼び出される gbt_num_consistent が依存する比較関数（date_cmp 等）の挙動に依存する。

保守担当者向けメモ

  • 本関数は btree_utils_num.c が提供する汎用ロジックのラッパーである。ロジックの変更が必要な場合は、まず btree_utils_num.c の実装を確認すること。
  • recheck が常に false であることは、このインデックスが完全な一致判定を提供していることを示している。
