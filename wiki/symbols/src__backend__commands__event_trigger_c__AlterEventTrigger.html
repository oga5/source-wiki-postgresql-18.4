AlterEventTrigger

定義

  • 種別: function
  • 定義箇所: src/backend/commands/event_trigger.c:426

目的

既存のイベントトリガーの有効状態（tgenabled）を変更し、システムカタログ pg_event_trigger を更新する。

シグネチャまたは形状

Oid AlterEventTrigger(AlterEventTrigStmt *stmt)

入力と出力

  • **入力**: AlterEventTrigStmt *stmt（変更対象のトリガー名および新しい有効状態を含む構造体）
  • **出力**: Oid（変更されたイベントトリガーのOID）

前提条件と不変条件

  • pg_event_trigger システムカタログに対する RowExclusiveLock を取得する。
  • 呼び出し元は、対象のイベントトリガーに対する所有者権限（object_ownercheck）を有している必要がある。

アルゴリズムまたは振る舞い

1. EventTriggerRelationId を開いてロックを取得する。 2. EVENTTRIGGERNAME システムキャッシュを使用して、指定された名前のトリガーを検索する。存在しない場合はエラーを報告する。 3. 呼び出しユーザーがトリガーの所有者であることを確認し、権限がない場合は aclcheck_error を発行する。 4. 取得したタプルをコピーし、evtenabled フィールドを stmt->tgenabled で更新する。 5. CatalogTupleUpdate を呼び出し、カタログを更新する。 6. イベント名が login であり、かつ無効化されていない場合、SetDatabaseHasLoginEventTriggers() を呼び出してデータベースフラグを更新する。 7. InvokeObjectPostAlterHook を呼び出し、変更後のフック処理を実行する。 8. タプルを解放し、リレーションを閉じて終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c:899（DDL実行フロー）
  • **呼び出し先**:
  • table_open, table_close
  • SearchSysCacheCopy1
  • object_ownercheck
  • CatalogTupleUpdate
  • SetDatabaseHasLoginEventTriggers
  • InvokeObjectPostAlterHook
  • heap_freetuple

関連データ構造

  • AlterEventTrigStmt: 変更内容を保持するパーサー構造体。
  • Form_pg_event_trigger: pg_event_trigger カタログの行構造。
  • EventTriggerRelationId: pg_event_trigger カタログのOID。

失敗モードと境界条件

  • 指定された名前のイベントトリガーが存在しない場合、ERRCODE_UNDEFINED_OBJECT でエラー終了する。
  • 呼び出しユーザーが所有者でない場合、aclcheck_error によりエラー終了する。

保守担当者向けメモ

  • login イベントトリガーのフラグ更新は、ホットコードパスでの検索を高速化するための最適化である。
  • CatalogTupleUpdate を使用しているため、カタログのインデックス更新は自動的に処理される。
