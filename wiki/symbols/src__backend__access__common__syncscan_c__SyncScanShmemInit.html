SyncScanShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/syncscan.c:134

目的

同期スキャン機能で使用する共有メモリ領域（scan_locations）を確保および初期化する。

シグネチャまたは形状

void SyncScanShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（共有メモリ上の scan_locations グローバル変数を初期化）

前提条件と不変条件

  • IsUnderPostmaster が偽（Postmaster起動時）の場合、共有メモリ領域は未確保である必要がある。
  • IsUnderPostmaster が真（バックエンド起動時）の場合、共有メモリ領域は既に確保済みである必要がある。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を呼び出し、Sync Scan Locations List という名称で共有メモリを確保する。 2. IsUnderPostmaster が偽の場合（初期化プロセス）:

  • scan_locations の head をリストの先頭、tail を末尾に設定する。
  • SYNC_SCAN_NELEM 個の全スロットをループで走査し、各エントリの relfilelocator を Invalid 値、location を InvalidBlockNumber で初期化する。
  • 各エントリの prev および next ポインタを連結し、双方向リストを構築する。

    • 3. IsUnderPostmaster が真の場合:

  • 既に初期化済みであることを Assert(found) で確認する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:342（PostgreSQLの共有メモリ初期化シーケンスの一部）
  • 呼び出し先: ShmemInitStruct

関連データ構造

  • ss_scan_locations_t: 共有メモリ上のメイン構造体。
  • ss_lru_item_t: LRUリストを構成する各スロット。
  • SYNC_SCAN_NELEM: 保持するスキャン位置の最大数。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、ShmemInitStruct の内部でパニックが発生する可能性がある。
  • IsUnderPostmaster の状態と実際のメモリ確保状況が一致しない場合、Assert によりプロセスが停止する。

保守担当者向けメモ

  • 本関数はPostgreSQLの起動時に一度だけ実行される。
  • SYNC_SCAN_NELEM を変更する場合は、共有メモリサイズ計算ロジック（SyncScanShmemSize）との整合性に注意すること。
