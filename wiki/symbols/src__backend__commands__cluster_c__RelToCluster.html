RelToCluster

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/cluster.c:64

目的

クラスタリング対象となるテーブルとそのインデックスのペアを保持するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	Oid			tableOid;
	Oid			indexOid;
} RelToCluster;

入力と出力

  • 入力: なし（データ構造定義のため）
  • 出力: なし

前提条件と不変条件

  • tableOid は対象となるリレーションの OID を保持します。
  • indexOid はクラスタリングに使用するインデックスの OID を保持します。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のためのコンテナであり、特定のアルゴリズムは持ちません。主に CLUSTER コマンド実行時に、処理対象となるテーブルとインデックスの組み合わせをリストとして管理するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: get_tables_to_cluster および get_tables_to_cluster_partitioned 関数内で、処理対象リストを構築する際にインスタンス化されます。
  • 呼び出し先: 本構造体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • List: RelToCluster のインスタンスを複数保持するために使用されます。

失敗モードと境界条件

  • メモリ不足: palloc を使用してメモリ確保を行うため、システムメモリが枯渇した場合は失敗します。

保守担当者向けメモ

  • 本構造体は src/backend/commands/cluster.c 内でのみ使用される内部的なデータ構造です。
  • 複数テーブルのクラスタリング処理において、各テーブルのコンテキストを維持するために不可欠です。変更時は、リストを走査する箇所（lfirst を使用している箇所など）との整合性に注意してください。
