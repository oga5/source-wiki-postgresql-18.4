ReplicationSlotsShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slot.c:185

目的

レプリケーションスロット管理のために共有メモリ上に確保すべき合計サイズを計算します。

シグネチャまたは形状

Size ReplicationSlotsShmemSize(void)

入力と出力

  • **入力**: なし（グローバル変数 max_replication_slots を参照）。
  • **出力**: 必要な共有メモリサイズ（Size 型）。

前提条件と不変条件

  • max_replication_slots が 0 の場合、サイズとして 0 を返します。
  • 共有メモリの計算には ReplicationSlotCtlData 構造体のヘッダー部分と、max_replication_slots 分の ReplicationSlot 配列のサイズが含まれます。

アルゴリズムまたは振る舞い

1. max_replication_slots が 0 であれば 0 を返して終了します。 2. ReplicationSlotCtlData 構造体の replication_slots メンバまでのオフセットを取得します。 3. max_replication_slots に ReplicationSlot 構造体のサイズを乗算し、ヘッダーサイズに加算して合計サイズを算出します。 4. 算出した合計サイズを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/slot.c: ReplicationSlotsShmemInit 内で共有メモリ初期化時に呼び出し。
  • src/backend/storage/ipc/ipci.c: CreateSharedMemoryAndSemaphores 内でシステム全体の共有メモリサイズ算出時に呼び出し。
  • **呼び出し先**:
  • add_size (マクロ/関数): サイズの加算。
  • mul_size (マクロ/関数): サイズの乗算。

関連データ構造

  • ReplicationSlotCtlData: レプリケーションスロットの制御情報を保持する構造体。
  • ReplicationSlot: 個々のレプリケーションスロットの状態を保持する構造体。

失敗モードと境界条件

  • max_replication_slots が 0 の場合、共有メモリは割り当てられません。
  • add_size や mul_size 内部でオーバーフローが発生した場合の挙動は、それらのユーティリティ関数の実装に依存します。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウトを決定する重要な関数です。ReplicationSlotCtlData 構造体や ReplicationSlot 構造体のサイズが変更された場合、本関数の計算ロジックが整合しているか確認してください。
