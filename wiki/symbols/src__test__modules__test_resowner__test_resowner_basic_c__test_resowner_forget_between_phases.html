test_resowner_forget_between_phases

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_resowner/test_resowner_basic.c:185

目的

ResourceOwner の解放フェーズ開始後に ResourceOwnerForget を実行した際の挙動を検証するテスト関数です。

シグネチャまたは形状

Datum test_resowner_forget_between_phases(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS (PostgreSQLの関数呼び出し規約に従う引数)
  • 出力: Datum (通常は PG_RETURN_VOID() を返すが、本関数はエラーを期待するため到達しない)

前提条件と不変条件

  • ResourceOwnerRelease が既に呼び出されていること。
  • ResourceOwnerRelease 呼び出し後の ResourceOwnerForget はエラーを発生させる必要がある。

アルゴリズムまたは振る舞い

1. ResourceOwnerCreate を使用して新しいリソース所有者を作成する。 2. ResourceOwnerRemember を使用してリソースを登録する。 3. ResourceOwnerRelease を RESOURCE_RELEASE_BEFORE_LOCKS フェーズで呼び出し、リソース解放プロセスを開始する。 4. 解放フェーズ開始後に ResourceOwnerForget を呼び出す。 5. ResourceOwnerForget がエラーを投げることを期待し、後続の elog(ERROR, ...) および PG_RETURN_VOID() には到達しないことを確認する。

呼び出し元と呼び出し先

  • 呼び出し元: PG_FUNCTION_INFO_V1 マクロによる登録（直接的な関数呼び出しの文脈は提供されていません）。
  • 呼び出し先: ResourceOwnerCreate, ResourceOwnerEnlarge, ResourceOwnerRemember, ResourceOwnerRelease, ResourceOwnerForget, elog。

関連データ構造

  • ResourceOwner: リソース所有権を管理する構造体。
  • string_desc: リソースの解放処理を定義する記述子。

失敗モードと境界条件

  • ResourceOwnerForget がエラーを発生させなかった場合、elog(ERROR, ...) によりテスト失敗として扱われる。

保守担当者向けメモ

  • 本関数は意図的にエラーを発生させるテストケースです。ResourceOwner の状態遷移制約を検証する目的で使用してください。
