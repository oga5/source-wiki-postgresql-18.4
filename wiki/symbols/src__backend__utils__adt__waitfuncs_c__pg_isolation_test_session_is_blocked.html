pg_isolation_test_session_is_blocked

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/waitfuncs.c:38

目的

PostgreSQLのテストスイート（isolationtester）において、指定されたセッションが特定の競合状態によりブロックされているかを判定する。テスト対象のセッション間でのブロック関係のみを抽出するために使用される。

シグネチャまたは形状

Datum pg_isolation_test_session_is_blocked(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • blocked_pid (int32): 判定対象のセッションPID。
  • interesting_pids (int32[]): 監視対象のPIDリスト。
  • **出力**:
  • bool: 指定されたPIDがブロックされている場合は true、それ以外は false。

前提条件と不変条件

  • interesting_pids 配列は INT4OID 型であり、NULLを含んではならない。
  • blocked_pid が存在しない場合、セッションは終了しているとみなされ false を返す。

アルゴリズムまたは振る舞い

1. **インジェクションポイント判定**: BackendPidGetProc で対象プロセスの PGPROC を取得し、待機イベントが InjectionPoint であるかを確認する。該当すれば true を返す。 2. **重量級ロック判定**: pg_blocking_pids を呼び出し、ロックを保持しているPIDリストを取得する。このリストと interesting_pids を比較し、重複があれば true を返す。パフォーマンス上の理由から、単純な二重ループによる検索を行う。 3. **セーフスナップショット判定**: GetSafeSnapshotBlockingPids を呼び出し、スナップショット取得待ちが発生しているかを確認する。ブロック数が0より大きければ true を返す。 4. 上記いずれにも該当しない場合、false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/isolation/isolationtester.c:222 (SQL経由での呼び出し)
  • **呼び出し先**:
  • BackendPidGetProc
  • pgstat_get_wait_event_type
  • DirectFunctionCall1(pg_blocking_pids, ...)
  • GetSafeSnapshotBlockingPids

関連データ構造

  • PGPROC: プロセス管理構造体。
  • ArrayType: SQL配列型の内部表現。

失敗モードと境界条件

  • interesting_pids にNULLが含まれる場合、elog(ERROR) により処理が中断される。
  • blocked_pid に対応するプロセスが存在しない場合、安全に false を返す。

保守担当者向けメモ

  • 本関数は公式APIではなく、テスト専用ツールである。本番環境のアプリケーションコードからは呼び出さないこと。
  • debug_discard_caches が有効な環境でのオーバーヘッドを最小化するため、汎用的な arrayoverlaps ではなく単純なループ検索を採用している。
