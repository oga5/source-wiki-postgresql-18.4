get_file_fdw_attribute_options

定義

  • 種別: function
  • 定義箇所: contrib/file_fdw/file_fdw.c:449

目的

外部テーブルの各カラムに設定された force_not_null および force_null オプションを収集し、それらを COPY コマンドで利用可能な形式（DefElem のリスト）に変換して返します。

シグネチャまたは形状

static List *get_file_fdw_attribute_options(Oid relid)

入力と出力

  • **入力**: relid (Oid) - 対象となる外部テーブルの OID。
  • **出力**: List * - force_not_null および force_null オプションを含む DefElem のリスト。該当するオプションが存在しない場合は NIL を返します。

前提条件と不変条件

  • 指定された relid は有効なリレーションである必要があります。
  • table_open により AccessShareLock を取得して処理を行い、終了時に解放します。
  • 削除済み属性（attisdropped）はスキップされます。

アルゴリズムまたは振る舞い

1. 指定されたリレーションを開き、全属性を走査します。 2. 各属性に対して GetForeignColumnOptions を呼び出し、設定されているオプションを取得します。 3. force_not_null または force_null オプションが見つかり、かつその値が真である場合、属性名をリスト（fnncolumns または fncolumns）に追加します。 4. 収集したリストが空でない場合、それぞれ force_not_null および force_null という名前の DefElem を作成し、結果リストに追加して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/file_fdw/file_fdw.c 内の関数から呼び出されます（例: 404行目付近）。
  • **呼び出し先**: table_open, RelationGetDescr, TupleDescAttr, GetForeignColumnOptions, defGetBoolean, lappend, makeString, makeDefElem, table_close 等の PostgreSQL 内部 API。

関連データ構造

  • Relation: テーブル情報を保持する構造体。
  • TupleDesc: タプルの属性情報を保持する構造体。
  • DefElem: オプション定義を保持するノード。

失敗モードと境界条件

  • 該当するオプションが一つも設定されていない場合、空のリスト（NIL）を返します。
  • 属性が削除されている場合、その属性のオプションは無視されます。

保守担当者向けメモ

  • 本関数は COPY コマンドのオプション処理に依存しています。将来的に他のカラムオプションをサポートする場合、ループ内の条件分岐を拡張してください。
  • pstrdup で複製された属性名は、呼び出し元で適切にメモリ管理されることを前提としています。
