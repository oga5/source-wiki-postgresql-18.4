bpchar_input

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varchar.c:129

目的

入力文字列を BpChar (char(n)) 型の内部表現に変換し、指定された型修飾子 (atttypmod) に基づいて長さの検証、切り詰め、および空白埋めを行う。

シグネチャまたは形状

BpChar *bpchar_input(const char *s, size_t len, int32 atttypmod, Node *escontext)

入力と出力

  • **入力**:
  • s: 入力文字列のポインタ。
  • len: 入力文字列のバイト長。
  • atttypmod: 型修飾子（最大長情報を含む）。
  • escontext: エラー処理用コンテキスト。
  • **出力**:
  • 変換後の BpChar 構造体へのポインタ。

前提条件と不変条件

  • atttypmod が VARHDRSZ 未満の場合、入力文字列の長さがそのまま使用される。
  • atttypmod が指定されている場合、入力文字列は指定された文字数 (maxlen) に収まる必要がある。
  • 指定長を超える文字が含まれる場合、それらがすべて空白であれば切り詰めが許可される。空白以外が含まれる場合はエラーとなる。

アルゴリズムまたは振る舞い

1. **長さ検証**: atttypmod から最大文字数 (maxlen) を算出する。 2. **文字数計算**: pg_mbstrlen_with_len を使用し、マルチバイト文字を考慮した文字数を取得する。 3. **切り詰め処理**:

  • 文字数が maxlen を超える場合、pg_mbcharcliplen で許容範囲内のバイト数を特定する。
  • 超過部分に空白以外の文字が含まれていないか確認し、含まれていれば ERRCODE_STRING_DATA_RIGHT_TRUNCATION エラーを投げる。
  • 許容範囲内で切り詰める。

    • 4. **メモリ確保とコピー**: palloc で必要なメモリを確保し、データをコピーする。 5. **空白埋め**: 指定長に満たない場合は、末尾を空白 (' ') で埋める。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/varchar.c:114 (bpcharin/bpcharrecv の共通処理として)
  • src/backend/utils/adt/varchar.c:207
  • src/backend/utils/adt/varchar.c:242
  • **呼び出し先**:
  • pg_mbstrlen_with_len (マルチバイト文字数計算)
  • pg_mbcharcliplen (マルチバイト文字の切り詰めバイト数計算)
  • palloc (メモリ確保)
  • ereturn (エラー発生時の制御)

関連データ構造

  • BpChar: char(n) 型の内部表現（varlena 構造体）。
  • Node: エラーコンテキスト用構造体。

失敗モードと境界条件

  • **切り詰めエラー**: 指定長を超えた部分に空白以外の文字が存在する場合、ERRCODE_STRING_DATA_RIGHT_TRUNCATION を発生させる。
  • **メモリ不足**: palloc 失敗時はシステム標準の例外が発生する。

保守担当者向けメモ

  • 本関数は bpcharin および bpcharrecv の共通ロジックとして機能する。
  • マルチバイト文字の扱いは pg_mb* 系関数に依存しているため、エンコーディング変更時の挙動に注意すること。
  • maxlen の計算において、文字数とバイト数の混同を避けるため、pg_mbcharcliplen の戻り値を適切に扱う必要がある。
