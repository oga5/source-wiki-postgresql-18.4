TParserStateActionItem

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/wparser_def.c:209

目的

全文検索パーサーにおける有限オートマトン（FSM）の遷移ルールを定義する構造体です。特定の状態において、入力文字がどの条件を満たした場合に、どの状態へ遷移し、どのようなアクションを実行するかを規定します。

シグネチャまたは形状

typedef struct
{
	TParserCharTest isclass;  /* 文字クラス判定関数 */
	char		c;            /* 比較対象の文字 */
	uint16		flags;        /* アクションフラグ（PUSH/POP/BINGO等） */
	TParserState tostate;     /* 遷移先の状態 */
	int			type;         /* トークンタイプ */
	TParserSpecial special;   /* 特殊処理用関数ポインタ */
} TParserStateActionItem;

入力と出力

  • **入力**: パーサーの現在の状態と入力文字。
  • **出力**: 判定結果に基づく状態遷移、トークン確定（BINGO）、またはスタック操作（PUSH/POP）。

前提条件と不変条件

  • wparser_def.c 内で定義される各 actionTPS_* 配列の要素として使用されます。
  • isclass が指定されている場合、文字 c との比較よりも優先的に文字クラス判定が行われます。

アルゴリズムまたは振る舞い

1. パーサーは現在の TParserState に対応する TParserStateActionItem の配列を走査します。 2. 各アイテムの isclass または c を用いて入力文字を評価します。 3. 条件が一致した場合、flags に基づいて以下の処理を行います。

  • A_BINGO: 現在のトークンを確定させます。
  • A_PUSH: 現在の状態をスタックに保存し、tostate へ遷移します。
  • A_POP: スタックから状態を復元します。

    • 4. tostate に従って次の解析状態へ移行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/wparser_def.c 内の actionTPS_Base, actionTPS_InWord 等の静的配列から参照されます。
  • **呼び出し先**: isclass フィールドに格納された TParserCharTest 型の関数（p_is* 系関数など）が実行時に呼び出されます。

関連データ構造

  • TParserState: 状態遷移の基点となる列挙型。
  • TParser: 解析のメイン状態を保持する構造体。
  • TParserPosition: 状態スタックを管理する構造体。

失敗モードと境界条件

  • 配列の終端に到達しても一致する条件がない場合、パーサーはエラーまたはデフォルトの遷移処理を行う必要があります（具体的なエラーハンドリングの詳細はソース抜粋からは不明）。

保守担当者向けメモ

  • 新しいトークンタイプを追加する際は、対応する actionTPS_* 配列に本構造体のインスタンスを追加してください。
  • flags のビット定義については、src/backend/tsearch/wparser_def.c:219 以降のコメントを参照してください。
