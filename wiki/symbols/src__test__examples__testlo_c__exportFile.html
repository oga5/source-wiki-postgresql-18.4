exportFile

定義

  • 種別: function
  • 定義箇所: src/test/examples/testlo.c:149

目的

データベース内のラージオブジェクト（Large Object）の内容を、ローカルのファイルシステム上のファイルへエクスポートする補助関数です。

シグネチャまたは形状

static void exportFile(PGconn *conn, Oid lobjId, char *filename)

入力と出力

  • **入力**:
  • conn: PostgreSQL データベース接続へのポインタ。
  • lobjId: エクスポート対象のラージオブジェクトの OID。
  • filename: 書き出し先のローカルファイルパス。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • conn は有効なデータベース接続である必要があります。
  • lobjId で指定されたラージオブジェクトは、呼び出し前に存在している必要があります。
  • 呼び出し元は、本関数が失敗した場合（lo_open や open のエラー）に標準エラー出力へメッセージが出力されることを想定しています。

アルゴリズムまたは振る舞い

1. lo_open を使用して、指定された lobjId を読み取り専用 (INV_READ) で開きます。 2. open を使用して、指定された filename を書き込み専用・作成・切り詰め (O_CREAT | O_WRONLY | O_TRUNC) モードで開きます。 3. lo_read を用いてラージオブジェクトからバッファ (BUFSIZE) 単位でデータを読み込みます。 4. 読み込んだデータを write を用いてローカルファイルへ書き出します。 5. 読み込みが完了するまでループを継続します。 6. 最後に lo_close でラージオブジェクトを閉じ、close でローカルファイルを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/examples/testlo.c 内で定義されていますが、ソースコード内のコメントアウトされた箇所 (line 261) 以外での直接的な呼び出し関係は、提供されたソース excerpt からは特定できません。
  • **呼び出し先**:
  • lo_open, lo_read, lo_close (libpq ラージオブジェクト API)
  • open, write, close (標準 C ライブラリ)
  • fprintf (標準エラー出力用)

関連データ構造

  • PGconn: libpq のデータベース接続ハンドル。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • lo_open が失敗した場合（戻り値が負）、標準エラー出力にエラーメッセージが表示されますが、処理は継続されます。
  • open が失敗した場合、標準エラー出力にエラーメッセージが表示されます。
  • write が読み込んだバイト数より少ない場合、標準エラー出力にエラーメッセージが表示されます。
  • 本関数はエラー発生時に例外を投げたり終了したりせず、そのまま後続の処理（lo_close 等）へ進む実装となっています。

保守担当者向けメモ

  • 本関数は libpq のサンプルコードであり、エラーハンドリングは最小限です。実運用環境で使用する場合は、戻り値のチェックを強化し、エラー発生時に適切に処理を中断する設計への変更を推奨します。
  • src/test/examples/testlo64.c にも同名の関数が存在し、64ビットオフセットに対応した実装となっています。比較検討の際はそちらを参照してください。
