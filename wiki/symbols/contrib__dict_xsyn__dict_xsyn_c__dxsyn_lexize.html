dxsyn_lexize

定義

  • 種別: function
  • 定義箇所: contrib/dict_xsyn/dict_xsyn.c:201

目的

入力された単語を辞書データと照合し、設定されたオプション（keeporig, keepsynonyms）に基づいてシノニムを展開した TSLexeme 配列を生成する。

シグネチャまたは形状

Datum dxsyn_lexize(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): DictSyn *（辞書データ構造）
  • PG_GETARG_POINTER(1): char *（入力単語）
  • PG_GETARG_INT32(2): int（入力単語の長さ）
  • **出力**:
  • Datum: TSLexeme * 型の配列（終端は lexeme = NULL）。該当なしの場合は NULL を返却。

前提条件と不変条件

  • 入力単語の長さが 0、または辞書の要素数 d->len が 0 の場合は即座に NULL を返す。
  • 辞書データ d->syn は、compare_syn 関数を用いてソート済みである必要がある。
  • str_tolower を使用するため、辞書内のキーは小文字で保持されている必要がある。

アルゴリズムまたは振る舞い

1. 入力単語を小文字化し、一時的な Syn 構造体を作成する。 2. bsearch を使用し、d->syn 配列から該当するシノニムエントリを $O(\log n)$ で検索する。 3. ヒットした場合、found->value に格納されたシノニム文字列を find_word で順次パースする。 4. keeporig フラグが真の場合、または最初の単語以外の場合に res 配列へ追加する。 5. keepsynonyms フラグが偽の場合、最初の単語のみを処理して終了する。 6. 最終的に TSLexeme 配列を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの全文検索フレームワーク（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • pnstrdup, palloc, repalloc, pfree（メモリ管理）
  • str_tolower（文字列正規化）
  • bsearch（検索）
  • compare_syn（比較関数）
  • find_word（文字列パース）

関連データ構造

  • DictSyn: 辞書全体の設定とシノニム配列を保持。
  • Syn: 個別の単語とシノニム文字列を保持。
  • TSLexeme: 全文検索結果として返される単語構造体。

失敗モードと境界条件

  • 辞書に単語が見つからない場合、NULL を返す。
  • length が 0 の場合、または辞書が空の場合、NULL を返す。
  • メモリ不足時は palloc/repalloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は tsearch の辞書インターフェースに従っている。
  • res 配列のメモリ確保は palloc で行われ、nsyns に応じて repalloc で拡張される。
  • 辞書ファイル形式の変更や、新しいオプションの追加時には dxsyn_init との整合性に注意すること。
