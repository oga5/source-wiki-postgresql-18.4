ProcedureCreate

定義

  • **種別**: function
  • **定義箇所**: src/backend/catalog/pg_proc.c:97

目的

PostgreSQLのシステムカタログ pg_proc に対して、関数、プロシージャ、または集約関数の定義を新規作成または更新するために使用されます。

シグネチャまたは形状

ObjectAddress ProcedureCreate(const char *procedureName, Oid procNamespace, bool replace, bool returnsSet, Oid returnType, Oid proowner, Oid languageObjectId, Oid languageValidator, const char *prosrc, const char *probin, Node *prosqlbody, char prokind, bool security_definer, bool isLeakProof, bool isStrict, char volatility, char parallel, oidvector *parameterTypes, Datum allParameterTypes, Datum parameterModes, Datum parameterNames, List *parameterDefaults, Datum trftypes, List *trfoids, Datum proconfig, Oid prosupport, float4 procost, float4 prorows);

入力と出力

  • **入力**: 関数名、名前空間、言語、引数型、戻り値型、および関数の振る舞いを制御する各種フラグ（volatility, parallel 等）を受け取ります。
  • **出力**: 作成または更新されたオブジェクトの ObjectAddress を返します。

前提条件と不変条件

  • prosrc は有効なポインタである必要があります。
  • 引数の数が FUNC_MAX_ARGS を超えてはなりません。
  • 多相型（polymorphic）の戻り値型を使用する場合、型を推論可能な多相型の入力引数が存在する必要があります。
  • 戻り値型が INTERNAL の場合、少なくとも一つの入力引数が INTERNAL である必要があります。
  • allParameterTypes が指定された場合、それは1次元のOID配列である必要があります。

アルゴリズムまたは振る舞い

1. **検証**: 引数の数、多相型シグネチャ、および INTERNAL 型の使用に関する妥当性チェックを行います。 2. **配列解析**: allParameterTypes や parameterModes が提供されている場合、それらをデコードして内部形式に変換します。 3. **カタログ操作**: pg_proc システムカタログを検索し、replace フラグに応じて既存レコードの更新（CatalogTupleUpdate）または新規挿入（CatalogTupleInsert）を行います。 4. **依存関係記録**: 関数が依存する型や言語などのオブジェクトを pg_depend に登録します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/pg_aggregate.c, src/backend/commands/functioncmds.c, src/backend/commands/typecmds.c 等から呼び出されます。
  • **呼び出し先**: check_valid_polymorphic_signature, check_valid_internal_signature 等の検証関数を呼び出します。

関連データ構造

  • pg_proc: 関数定義を保持するシステムカタログ。
  • oidvector: 引数型を保持する構造体。
  • ObjectAddress: 作成されたオブジェクトを識別するための構造体。

失敗モードと境界条件

  • 引数が FUNC_MAX_ARGS を超える場合、ERRCODE_TOO_MANY_ARGUMENTS でエラーとなります。
  • 型シグネチャが不正な場合、ERRCODE_INVALID_FUNCTION_DEFINITION でエラーとなります。
  • 配列形式が期待される形式（1次元OID配列など）と異なる場合、elog(ERROR) が発生します。

保守担当者向けメモ

  • check_function_bodies が false の場合、一部のバリデーションがスキップされる仕様です。
  • allParameterTypes の解析において、deconstruct_array を使用せず直接ポインタ操作を行っているため、配列構造の変更時には注意が必要です。
