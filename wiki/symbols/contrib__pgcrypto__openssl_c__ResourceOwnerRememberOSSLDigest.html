ResourceOwnerRememberOSSLDigest

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/openssl.c:78

目的

OpenSSL のハッシュコンテキスト（OSSLDigest）を PostgreSQL の ResourceOwner に登録し、トランザクション終了時やエラー発生時に確実にリソースが解放されるよう管理する。

シグネチャまたは形状

static inline void ResourceOwnerRememberOSSLDigest(ResourceOwner owner, OSSLDigest *digest)

入力と出力

  • owner: リソースを所有する ResourceOwner オブジェクト。
  • digest: 登録対象の OSSLDigest 構造体へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • digest は有効な OSSLDigest 構造体を指している必要がある。
  • ResourceOwner 機構により、本関数で登録されたリソースは ResOwnerReleaseOSSLDigest を通じて自動的にクリーンアップされる。

アルゴリズムまたは振る舞い

1. ResourceOwnerRemember を呼び出し、digest ポインタを PointerGetDatum で変換して owner に登録する。 2. 登録時には、リソースの解放方法を定義した ossldigest_resowner_desc を指定する。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/openssl.c 内のハッシュコンテキスト初期化処理（例: digest->owner を引数として渡す箇所）。
  • 呼び出し先: ResourceOwnerRemember（PostgreSQL コアのリソース管理関数）。

関連データ構造

  • OSSLDigest: ハッシュ計算用のコンテキストを保持する構造体。
  • ResourceOwner: PostgreSQL のリソース所有権管理オブジェクト。
  • ossldigest_resowner_desc: リソース解放時のコールバックや説明を保持する記述子。

失敗モードと境界条件

  • owner が無効な場合、ResourceOwnerRemember 内部でエラーが発生する可能性がある。
  • メモリ不足等の理由で ResourceOwner への登録に失敗した場合、PostgreSQL のエラーハンドリング機構によりトランザクションがアボートされる。

保守担当者向けメモ

  • 本関数は static inline で定義されており、pgcrypto モジュール内でのリソースリーク防止の要となっている。
  • OSSLDigest 構造体のライフサイクルを変更する場合は、必ず ResourceOwnerForgetOSSLDigest による解除処理と対になるように設計すること。
