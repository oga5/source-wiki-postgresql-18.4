FakeRelCacheEntryData

定義

  • **種別**: struct
  • **定義箇所**: src/backend/access/transam/xlogutils.c:548

目的

リカバリ処理中に物理的なストレージ操作を行う際、本来のリレーションキャッシュ（RelCache）が存在しない環境下で、最小限のリレーション情報を保持するために使用される一時的なデータ構造です。

シグネチャまたは形状

typedef struct {
    RelationData reldata; /* 必須: 先頭に配置 */
    FormData_pg_class pgc;
} FakeRelCacheEntryData;

入力と出力

  • **入力**: CreateFakeRelcacheEntry 関数において、リレーションの物理的な属性情報（FormData_pg_class）を構築する際に使用されます。
  • **出力**: 物理的なブロック読み込みに必要な Relation 構造体として機能します。

前提条件と不変条件

  • reldata フィールドが構造体の先頭に配置されている必要があります。
  • 本構造体はリカバリ中（InRecovery）の限定的な用途のために設計されており、完全なリレーションキャッシュの代用にはなりません。

アルゴリズムまたは振る舞い

  • CreateFakeRelcacheEntry を通じてメモリ確保（palloc0）が行われ、物理的なストレージ操作に必要な最小限のメタデータを保持します。
  • 使用後は FreeFakeRelcacheEntry を用いて適切に解放する必要があります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlogutils.c 内の CreateFakeRelcacheEntry 関数が本構造体のメモリを確保します。
  • **呼び出し先**: 物理ストレージ層（storage/smgr.h）が、この構造体を介してリレーションの物理ブロックへアクセスします。

関連データ構造

  • RelationData: reldata フィールドとして内包。
  • FormData_pg_class: pgc フィールドとして内包。
  • FakeRelCacheEntry: 本構造体へのポインタ型（typedef FakeRelCacheEntryData *FakeRelCacheEntry;）。

失敗モードと境界条件

  • 本構造体はリカバリプロセス以外での使用を想定していません。通常のバックエンドプロセスから使用した場合の動作は保証されません。
  • メモリ確保失敗時は palloc0 がエラーを発生させます。

保守担当者向けメモ

  • 本構造体は、WALリプレイ中に物理ブロックを読み込むための「偽の」キャッシュエントリです。整合性チェックや完全なキャッシュ機能は期待しないでください。
  • reldata が構造体の先頭にあるというメモリレイアウトの制約は、キャストによるアクセスを前提としているため厳守してください。
