r_standard_suffix

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_german.c:336

目的

ドイツ語のステミング処理において、単語末尾の接尾辞を特定のルールに基づいて削除または置換し、単語を正規化する。

シグネチャまたは形状

static int r_standard_suffix(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z（ステミング環境の状態を保持する構造体）
  • 出力: 成功時は 1、失敗時は 0、エラー発生時は負の整数を返す。

前提条件と不変条件

  • z が指す環境において、r_mark_regions によって R1 および R2 領域が適切に設定されていること。
  • 処理対象の単語は UTF-8 エンコーディングであること。

アルゴリズムまたは振る舞い

単語の末尾から前方に向かって、以下の順序で接尾辞の照合と削除を行う： 1. **第一段階**: a_2 テーブルに基づき接尾辞を検索。R1 領域内にある場合、一致したルールに応じて削除または置換を実行。 2. **第二段階**: a_3 テーブルに基づき接尾辞を検索。R1 領域内にある場合、一致したルールに応じて削除を実行。 3. **第三段階**: a_5 テーブルに基づき接尾辞を検索。R2 領域内にある場合、一致したルールに応じて削除または置換を実行。 各段階で find_among_b を使用して後方からマッチングを行い、条件を満たす場合に slice_del または slice_from_s で単語を書き換える。

呼び出し元と呼び出し先

  • 呼び出し元: german_UTF_8_stem（直接の呼び出し元であることは文脈から推測されるが、本ソース excerpt には明示的な呼び出し関係の記述なし）。
  • 呼び出し先: find_among_b, r_R1, r_R2, slice_del, slice_from_s, eq_s_b, in_grouping_b_U, skip_b_utf8。

関連データ構造

  • struct SN_env: ステミングの実行状態（現在の単語、領域境界、カーソル位置等）を管理する構造体。
  • a_2, a_3, a_4, a_5: 接尾辞マッチング用のテーブル。

失敗モードと境界条件

  • R1 や R2 の領域条件を満たさない場合、接尾辞の削除は行われない。
  • slice_del 等の操作でメモリ不足やエラーが発生した場合、負の値を返して処理を中断する。
  • 単語が短すぎる場合や、マッチングテーブルに該当する接尾辞がない場合は何も変更せずに終了する。
