EOHPGetRWDatum

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/expandeddatum.h:138

目的

展開済みオブジェクトヘッダから、そのオブジェクトに対する読み取り書き込み可能（R/W）な Datum 表現を抽出する。

シグネチャまたは形状

static inline Datum EOHPGetRWDatum(const struct ExpandedObjectHeader *eohptr)

入力と出力

  • **入力**: eohptr - 展開済みオブジェクトの共通ヘッダ構造体へのポインタ。
  • **出力**: Datum - 展開済みオブジェクトの R/W ポインタを Datum 型に変換したもの。

前提条件と不変条件

  • eohptr は有効な ExpandedObjectHeader を指している必要がある。
  • 戻り値の Datum は、当該オブジェクトが R/W モードで操作可能であることを前提としている。

アルゴリズムまたは振る舞い

1. 引数として受け取った ExpandedObjectHeader 構造体の eoh_rw_ptr メンバにアクセスする。 2. PointerGetDatum マクロを使用して、そのポインタを Datum 型にキャストして返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/array_expanded.c
  • src/backend/utils/adt/array_userfuncs.c
  • src/backend/utils/adt/arrayfuncs.c
  • src/backend/utils/adt/expandeddatum.c

  • src/include/utils/array.h (マクロ PG_RETURN_EXPANDED_ARRAY 内)
  • src/include/utils/expandedrecord.h
  • **呼び出し先**:
  • PointerGetDatum (マクロ)

関連データ構造

  • ExpandedObjectHeader: 展開済みオブジェクトの共通ヘッダ。
  • Datum: PostgreSQL における汎用データ表現型。

失敗モードと境界条件

  • eohptr が NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • オブジェクトが R/W モードをサポートしていない、あるいは R/W ポインタが初期化されていない状態で呼び出した場合の挙動は未定義。

保守担当者向けメモ

  • 本関数はインライン関数であり、パフォーマンスを考慮してヘッダファイルで定義されている。
  • 読み取り専用（R/O）の Datum を取得する対となる関数 EOHPGetRODatum が存在する。
