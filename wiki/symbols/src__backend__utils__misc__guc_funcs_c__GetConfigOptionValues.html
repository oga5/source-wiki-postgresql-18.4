GetConfigOptionValues

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/guc_funcs.c:593

目的

GUC（Grand Unified Configuration）パラメータの属性情報を抽出し、SQLインターフェース（pg_settings ビュー等）で利用可能な文字列配列形式に変換する。

シグネチャまたは形状

static void GetConfigOptionValues(struct config_generic *conf, const char **values)

入力と出力

  • **入力**:
  • conf: 情報を取得する対象のGUCパラメータ構造体（config_generic）。
  • values: 属性値を格納するための const char * 型の配列（呼び出し側で確保）。
  • **出力**: なし（values 配列のインデックス0〜13に各属性値が書き込まれる）。

前提条件と不変条件

  • values 配列は、少なくとも14個の要素を保持できるサイズであること。
  • conf->vartype に基づき、values[9] から values[13] に型固有の属性（最小値、最大値、列挙値、初期値、リセット値）が設定される。
  • 文字列として返される値は pstrdup によってメモリ確保されるため、呼び出し側で適切なメモリ管理が必要。

アルゴリズムまたは振る舞い

1. **汎用属性の取得**: パラメータ名、現在値（ShowGUCOption を利用）、単位、グループ、説明文、コンテキスト、型、ソース情報を values[0]〜values[8] に格納する。 2. **型固有属性の取得**: conf->vartype に基づき switch 文で分岐し、以下の属性を処理する。

  • PGC_BOOL: "on"/"off" 文字列を生成。
  • PGC_INT / PGC_REAL: 数値を文字列に変換。
  • PGC_STRING: 文字列をそのままコピー。
  • PGC_ENUM: config_enum_get_options 等を用いて列挙値のリストおよび現在値を解決。

    • 3. **デフォルト値**: 未定義の属性や型に応じた不要なフィールドには NULL を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/guc_funcs.c 内の show_all_settings 関数（pg_settings ビューのバックエンド処理）。
  • **呼び出し先**: ShowGUCOption, get_config_unit_name, config_enum_get_options, config_enum_lookup_by_value, pstrdup, snprintf。

関連データ構造

  • struct config_generic: GUCパラメータの基底構造体。
  • struct config_bool, config_int, config_real, config_string, config_enum: 各型に対応する詳細構造体。

失敗モードと境界条件

  • conf->vartype が未知の値である場合、型固有の属性はすべて NULL に設定される。
  • PGC_ENUM 型において、列挙値にダブルクォーテーションが含まれる場合は正しく処理されない制約がある。
  • pstrdup によるメモリ確保が失敗した場合、PostgreSQLの標準的なエラーハンドリング（メモリ不足エラー）が発生する。

保守担当者向けメモ

  • NUM_PG_SETTINGS_ATTS 定数と values 配列のインデックスは密結合しているため、属性を追加・変更する際は整合性に注意すること。
  • ShowGUCOption を再利用しているため、表示ロジックの変更は一元化されている。
