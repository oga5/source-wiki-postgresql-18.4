proclist_push_tail_offset

定義

  • 種別: function
  • 定義箇所: src/include/storage/proclist.h:86

目的

共有メモリ上で管理される PGPROC の双方向連結リストに対し、指定されたノードを末尾に追加する。

シグネチャまたは形状

static inline void proclist_push_tail_offset(proclist_head *list, int procno, size_t node_offset)

入力と出力

  • list: 操作対象のリストヘッダー（proclist_head へのポインタ）。
  • procno: 追加するプロセスの識別番号。
  • node_offset: PGPROC 構造体内の proclist_node メンバへのオフセット。
  • 戻り値: なし。

前提条件と不変条件

  • 追加対象のノードは、現在どのリストにも属していないこと（node->next == 0 かつ node->prev == 0 であること）。
  • リストが空の場合、list->head と list->tail は INVALID_PROC_NUMBER であること。
  • リストが空でない場合、list->head および list->tail は有効な値であること。

アルゴリズムまたは振る舞い

1. proclist_node_get を使用して、procno と node_offset から対象ノードを取得する。 2. リストが空の場合:

  • ノードの next および prev を INVALID_PROC_NUMBER に設定する。
  • リストの head と tail を当該 procno に更新する。

    • 3. リストが空でない場合:

  • ノードの prev を現在の list->tail に設定する。
  • 現在の末尾ノードの next を当該 procno に更新する。
  • ノードの next を INVALID_PROC_NUMBER に設定する。
  • リストの tail を当該 procno に更新する。

呼び出し元と呼び出し先

  • 呼び出し先: proclist_node_get
  • 呼び出し元: src/include/storage/proclist.h 内の各所（offsetof マクロを介した呼び出しを含む）。

関連データ構造

  • proclist_head: リストの先頭と末尾を保持する構造体。
  • proclist_node: リストの前後関係を保持するノード構造体。
  • PGPROC: プロセス情報を保持する構造体。

失敗モードと境界条件

  • 追加対象ノードが既にリストに属している場合、Assert により停止する。
  • リストの整合性が崩れている場合（例: head があるのに tail がない等）、Assert により停止する。

保守担当者向けメモ

  • 本関数はインライン関数であり、共有メモリ上のアドレス空間の違いを吸収するため、ポインタではなく procno を使用している。
  • node_offset には offsetof(PGPROC, member_name) を指定すること。
