XLogEnsureRecordSpace

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xloginsert.c:174

目的

WALレコード構築時に使用する作業用メモリ領域（ブロック参照用配列およびデータチャンク用配列）を、指定されたサイズまで拡張します。

シグネチャまたは形状

void XLogEnsureRecordSpace(int max_block_id, int ndatas)

入力と出力

  • max_block_id: 必要な最大ブロックID（0始まり）。
  • ndatas: 必要なデータチャンク数。
  • 戻り値: なし。

前提条件と不変条件

  • CritSectionCount == 0 であること（クリティカルセクション内でのメモリ割り当て失敗を避けるため）。
  • max_block_id が XLR_MAX_BLOCK_ID を超える場合はエラーとなる。
  • 内部配列の最小サイズは XLR_NORMAL_MAX_BLOCK_ID および XLR_NORMAL_RDATAS によって保証される。

アルゴリズムまたは振る舞い

1. 引数 max_block_id と ndatas が最小値を下回る場合、それぞれ最小値に調整する。 2. max_block_id が許容上限を超えていないか検証する。 3. 現在の max_registered_buffers より max_block_id + 1 が大きい場合、registered_buffers 配列を repalloc で拡張し、新規領域を MemSet でゼロ初期化する。 4. 現在の max_rdatas より ndatas が大きい場合、rdatas 配列を repalloc で拡張する。

呼び出し元と呼び出し先

  • 呼び出し元: ginfast.c, gist.c, gistxlog.c, hashovfl.c, twophase.c, xloginsert.c 内の各関数。
  • 呼び出し先: repalloc, MemSet, elog。

関連データ構造

  • registered_buffers: 登録されたブロックのメタデータを保持する配列。
  • rdatas: WALレコードのデータチャンクを保持する XLogRecData 型の配列。

失敗モードと境界条件

  • max_block_id > XLR_MAX_BLOCK_ID: elog(ERROR) により処理が中断される。
  • メモリ不足: repalloc が失敗した場合、PostgreSQL の標準的なメモリ割り当てエラーが発生する。
  • クリティカルセクション内からの呼び出し: Assert によりデバッグビルドで停止する。

保守担当者向けメモ

  • 本関数は、標準的なレコードサイズを超える大規模なWALレコードを構築する前に呼び出す必要があります。
  • registered_buffers を拡張した際、パディングバイトを含む構造体全体をゼロ初期化している点に注意してください（WALデータに構造体の中身が直接含まれるため）。
