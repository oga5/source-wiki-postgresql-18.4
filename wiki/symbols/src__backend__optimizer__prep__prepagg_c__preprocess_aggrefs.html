preprocess_aggrefs

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepagg.c:109

目的

クエリ内の集約関数（Aggref）を最適化するためのメインエントリーポイントです。式ツリーを走査し、集約の統合や遷移状態の共有を決定してプランニング情報を構築します。

シグネチャまたは形状

void preprocess_aggrefs(PlannerInfo *root, Node *clause)

入力と出力

  • **入力**:
  • root: プランニングの全体情報を保持する PlannerInfo 構造体。
  • clause: 集約関数を含む可能性のある式ツリー（Node）。
  • **出力**: なし（副作用として root 内の集約関連情報を更新）。

前提条件と不変条件

  • 入力された式ツリー内の Aggref ノードを直接書き換えるインプレース更新を行います。
  • root->aggtransinfos 等のプランニング用データ構造が適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. preprocess_aggrefs_walker を呼び出し、指定された clause（式ツリー）を再帰的に走査します。 2. 走査中に Aggref ノードを発見するたびに、集約の統合（aggno の割り当て）や遷移状態の共有（transno の決定）を行い、PlannerInfo に情報を蓄積します。 3. 処理完了後、Aggref ノードのメタデータが更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planner.c: root->processed_tlist および parse->havingQual に対して呼び出されます。
  • **呼び出し先**:
  • preprocess_aggrefs_walker: 式ツリーの走査を実行します。

関連データ構造

  • PlannerInfo: プランニングのコンテキスト。
  • Aggref: クエリ内の集約関数ノード。
  • AggInfo / AggTransInfo: 集約および遷移状態に関する情報を保持する構造体（src/include/nodes/pathnodes.h で定義）。

失敗モードと境界条件

  • 供給されたソース情報には、特定の失敗モードや境界条件に関する明示的な記述はありません。

保守担当者向けメモ

  • 本関数はプランニングの初期段階で実行される必要があります。後続のプラン生成フェーズでは、本関数によって構築された AggInfo リストが参照されます。
  • 将来的には実行時のオーバーヘッド削減のため、これらの情報を Agg ノード自体に保持させる構想があります。
