ItemPointerCompare

定義

  • 種別: function
  • 定義箇所: src/backend/storage/page/itemptr.c:50

目的

2つの ItemPointer（TID）を比較し、その順序関係を判定します。主にB-treeインデックスやデータ構造のソート順序の決定に使用されます。

シグネチャまたは形状

int32 ItemPointerCompare(ItemPointer arg1, ItemPointer arg2)

入力と出力

  • **入力**: 比較対象となる2つの ItemPointer 型のポインタ（arg1, arg2）。
  • **出力**:
  • arg1 < arg2 の場合: -1
  • arg1 > arg2 の場合: 1
  • arg1 == arg2 の場合: 0

前提条件と不変条件

  • ItemPointerGetBlockNumberNoCheck および ItemPointerGetOffsetNumberNoCheck を使用するため、ip_posid == 0（無効なオフセット）を含むTIDであっても、メモリ上のビットパターンに基づいて一貫した比較結果を返します。

アルゴリズムまたは振る舞い

1. 各引数からブロック番号（BlockNumber）を取得し、比較します。 2. ブロック番号が異なる場合、その大小関係を即座に返します。 3. ブロック番号が等しい場合、オフセット番号（OffsetNumber）を取得し、比較結果を返します。 4. ブロック番号とオフセット番号の双方が等しい場合、0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c
  • contrib/amcheck/verify_nbtree.c
  • contrib/pg_surgery/heap_surgery.c
  • src/backend/access/brin/brin_minmax_multi.c
  • src/backend/access/gin/gindatapage.c
  • src/backend/access/gin/gininsert.c
  • **呼び出し先**:
  • ItemPointerGetBlockNumberNoCheck
  • ItemPointerGetOffsetNumberNoCheck

関連データ構造

  • ItemPointerData (TID): 6バイトの構造体。ブロック番号とオフセット番号を保持します。

失敗モードと境界条件

  • 特殊な値（SpecTokenOffsetNumber 等）や無効な TID が含まれる場合でも、エラーを発生させず、ビットパターンに基づいた順序を返します。

保守担当者向けメモ

  • 本関数は NoCheck 系アクセサを使用しており、通常のデータアクセスとは異なり、システムカタログやインデックス内の特殊な TID を許容する設計となっています。
  • 構造体のパディングによる影響を避けるため、ItemPointerData のサイズが 6 バイトであることを維持する必要があります。
