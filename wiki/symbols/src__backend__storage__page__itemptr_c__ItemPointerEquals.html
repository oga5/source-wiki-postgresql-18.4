ItemPointerEquals

定義

  • 種別: function
  • 定義箇所: src/backend/storage/page/itemptr.c:34

目的

2つの ItemPointer（TID: Tuple Identifier）が、同一のブロック番号およびオフセット番号を指しているかを判定する。

シグネチャまたは形状

bool ItemPointerEquals(ItemPointer pointer1, ItemPointer pointer2)

入力と出力

  • **入力**: 比較対象となる2つの ItemPointer 型のポインタ (pointer1, pointer2)。
  • **出力**: 両者が等しい場合は true、それ以外の場合は false を返す。

前提条件と不変条件

  • 入力されるポインタは有効な ItemPointerData を指している必要がある。
  • ItemPointerData 構造体は 6 バイトの固定サイズであり、パディングを含まないことが前提となっている。

アルゴリズムまたは振る舞い

1. ItemPointerGetBlockNumber を使用して、両ポインタのブロック番号を比較する。 2. ItemPointerGetOffsetNumber を使用して、両ポインタのオフセット番号を比較する。 3. 両方の条件が一致した場合にのみ true を返し、それ以外は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c
  • src/backend/access/brin/brin_revmap.c
  • src/backend/access/gin/ginfast.c
  • src/backend/access/gin/ginget.c
  • src/backend/access/gist/gist.c
  • src/backend/access/hash/hashutil.c
  • src/backend/access/heap/heapam.c
  • **呼び出し先**:
  • ItemPointerGetBlockNumber (インラインアクセサ)
  • ItemPointerGetOffsetNumber (インラインアクセサ)

関連データ構造

  • ItemPointerData: ディスク上のデータ位置を保持する 6 バイトの構造体。

失敗モードと境界条件

  • ポインタが NULL である場合の挙動については、本関数内では明示的なチェックを行っていない。
  • 無効な TID（ip_posid == 0 等）が含まれる場合でも、ビットパターンが一致すれば true を返す。

保守担当者向けメモ

  • 本関数は itemptr.h で定義されたインラインアクセサを組み合わせて比較を行う。ロジックの変更が必要な場合は、ItemPointerData の構造定義と整合性が取れているか確認すること。
