CreateExtensionInternal

定義

  • **種別**: function
  • **定義箇所**: src/backend/commands/extension.c:1783

目的

PostgreSQLの拡張機能（Extension）のインストールおよび更新処理のメインロジックを実行します。制御ファイルの読み込み、インストールパスの決定、ターゲットスキーマの特定、および必要なスキーマの作成を行います。

シグネチャまたは形状

static ObjectAddress CreateExtensionInternal(
    char *extensionName,
    char *schemaName,
    const char *versionName,
    bool cascade,
    List *parents,
    bool is_create
)

入力と出力

  • **入力**: 拡張機能名、ターゲットスキーマ名、バージョン名、CASCADEフラグ、親拡張機能リスト、作成フラグ。
  • **出力**: 作成された拡張機能の ObjectAddress を返します。

前提条件と不変条件

  • 制御ファイル（.control）が存在し、読み込み可能であること。
  • 指定されたバージョン名が有効であること。
  • 拡張機能が非リロケータブル（relocatable = false）な場合、制御ファイルで指定されたスキーマと異なるスキーマへのインストールは、cascade が真でない限りエラーとなります。
  • 一時名前空間がアクセスされた場合、MyXactFlags に XACT_FLAGS_ACCESSEDTEMPNAMESPACE が設定されます。

アルゴリズムまたは振る舞い

1. **制御ファイル読み込み**: read_extension_control_file で基本情報を取得します。 2. **バージョン決定**: バージョン未指定時はデフォルトバージョンを使用します。 3. **パス探索**: インストールスクリプトの存在を確認し、直接スクリプトがない場合は find_install_path を用いて更新パスを特定します。 4. **スキーマ特定**:

  • ユーザー指定のスキーマがある場合はそのOIDを取得します。
  • 制御ファイルでスキーマが指定されている場合は、そのスキーマの存在を確認し、なければ CreateSchemaCommand で作成します。
  • 指定がない場合は search_path の先頭をデフォルトとして使用します。

    • 5. **後続処理**: 依存する拡張機能の確認とインストールへ進みます（ソース抜粋以降の処理）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/extension.c 内の CreateExtension 等から呼び出されます。
  • **呼び出し先**: read_extension_control_file, get_extension_script_filename, get_ext_ver_list, find_install_path, CreateSchemaCommand 等。

関連データ構造

  • ExtensionControlFile: 制御ファイルの内容を保持。
  • ExtensionVersionInfo: バージョン更新パスのグラフ構造。
  • ObjectAddress: 作成されたオブジェクトの識別子。

失敗モードと境界条件

  • バージョン指定がないかつデフォルトバージョンが存在しない場合、エラーとなります。
  • インストールスクリプトや更新パスが見つからない場合、エラーとなります。
  • search_path が空でスキーマも指定されていない場合、エラーとなります。
  • スキーマ名が指定されているが、そのスキーマが存在しない場合、get_namespace_oid がエラーを発生させます。

保守担当者向けメモ

  • 本関数はトランザクション内で実行されます。CreateSchemaCommand 実行後に CommandCounterIncrement が行われるため、作成直後のスキーマが可視化される点に注意してください。
  • 拡張機能のインストール権限チェックは本関数内では行われず、スクリプト実行時のオブジェクト作成権限に委ねられています。
