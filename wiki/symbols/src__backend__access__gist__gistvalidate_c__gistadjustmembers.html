gistadjustmembers

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistvalidate.c:287

目的

GiSTインデックスの演算子族（opfamily）および演算子クラス（opclass）における、演算子とサポート関数の依存関係（ハード/ソフト）を適切に設定する。

シグネチャまたは形状

void gistadjustmembers(Oid opfamilyoid, Oid opclassoid, List *operators, List *functions)

入力と出力

  • **入力**:
  • opfamilyoid: 対象の演算子族のOID。
  • opclassoid: 対象の演算子クラスのOID。
  • operators: OpFamilyMember 構造体のリスト（演算子）。
  • functions: OpFamilyMember 構造体のリスト（サポート関数）。
  • **出力**: なし（引数として渡されたリスト内の OpFamilyMember 構造体のメンバを直接更新する）。

前提条件と不変条件

  • operators リスト内の各要素は、演算子族に対して常にソフト依存（ref_is_hard = false）として扱われる。
  • functions リスト内の各要素は、GiSTの必須サポート関数であるか否かに応じて依存関係が決定される。

アルゴリズムまたは振る舞い

1. **演算子の処理**: operators リストを走査し、すべての演算子に対して ref_is_hard = false、ref_is_family = true を設定し、refobjid を opfamilyoid に割り当てる。 2. **サポート関数の処理**: functions リストを走査し、各関数の番号（op->number）に基づいて以下を適用する。

  • **必須関数**: GIST_CONSISTENT_PROC, GIST_UNION_PROC, GIST_PENALTY_PROC, GIST_PICKSPLIT_PROC, GIST_EQUAL_PROC の場合、ref_is_hard = true を設定する。
  • **オプション関数**: GIST_COMPRESS_PROC, GIST_DECOMPRESS_PROC, GIST_DISTANCE_PROC, GIST_FETCH_PROC, GIST_OPTIONS_PROC, GIST_SORTSUPPORT_PROC, GIST_TRANSLATE_CMPTYPE_PROC の場合、ref_is_hard = false、ref_is_family = true を設定し、refobjid を opfamilyoid に割り当てる。
  • **無効な関数**: 上記以外の番号が指定された場合、ERROR を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gist.c: amroutine->amadjustmembers フィールドに登録されている。
  • **呼び出し先**:
  • ereport: エラー発生時に呼び出される。

関連データ構造

  • OpFamilyMember: 演算子族のメンバ（演算子または関数）を保持する構造体。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • 未定義のサポート関数番号が渡された場合、ERRCODE_INVALID_OBJECT_DEFINITION を伴うエラーが発生する。

保守担当者向けメモ

  • 本関数は DROP コマンド実行時の依存関係削除の挙動を制御する。必須関数はハード依存として扱われるため、演算子族から削除する際に厳格なチェックが行われる。
  • 新しいGiSTサポート関数が追加された場合は、必ず switch 文にケースを追加すること。
