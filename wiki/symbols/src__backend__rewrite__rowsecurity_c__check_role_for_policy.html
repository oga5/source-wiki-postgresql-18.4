check_role_for_policy

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rowsecurity.c:915

目的

指定された行レベルセキュリティ（RLS）ポリシーが、現在のユーザーに対して適用可能かどうかを判定する。

シグネチャまたは形状

static bool check_role_for_policy(ArrayType *policy_roles, Oid user_id)

入力と出力

  • **入力**:
  • policy_roles: ポリシーに関連付けられたロールのOID配列（ArrayType*）。
  • user_id: 判定対象となるユーザーのOID。
  • **出力**:
  • bool: ユーザーがポリシーの対象ロールに含まれる（または権限を継承している）場合は true、それ以外は false。

前提条件と不変条件

  • policy_roles は有効な ArrayType であること。
  • roles[0] が ACL_ID_PUBLIC である場合、すべてのユーザーに対して true を返す。

アルゴリズムまたは振る舞い

1. ポリシーのロール配列の先頭要素を確認し、ACL_ID_PUBLIC であれば直ちに true を返す。 2. それ以外の場合、配列内の各ロールについて has_privs_of_role(user_id, roles[i]) を呼び出し、ユーザーがそのロールの権限を持っているかを確認する。 3. いずれかのロールで権限が確認できれば true を返し、配列を走査し終えても一致しなければ false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/rewrite/rowsecurity.c 内の複数箇所（行 599, 636, 650）にて、ポリシー適用判定のために呼び出される。
  • **呼び出し先**:
  • has_privs_of_role: ユーザーが指定されたロールの権限を有しているか判定するために呼び出される。

関連データ構造

  • ArrayType: PostgreSQLの配列型構造体。
  • Oid: オブジェクト識別子。

失敗モードと境界条件

  • policy_roles が空の配列である場合、ループは実行されず false が返される。
  • has_privs_of_role の内部的な失敗については本関数では制御しない。

保守担当者向けメモ

  • 本関数は rowsecurity.c 内の静的関数であり、RLSポリシーのロールチェックにおける標準的なロジックをカプセル化している。
  • ACL_ID_PUBLIC のチェックは最適化のためのショートカットとして機能している。
