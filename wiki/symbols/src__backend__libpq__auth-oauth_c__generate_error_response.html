generate_error_response

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-oauth.c:484

目的

OAuth認証失敗時に、クライアントへ再試行に必要な情報（OpenID構成URIおよびスコープ）を含むJSON形式のエラー応答を生成する。

シグネチャまたは形状

static void generate_error_response(struct oauth_ctx *ctx, char **output, int *outputlen)

入力と出力

  • **入力**:
  • ctx: OAuth認証のコンテキスト構造体（issuer および scope を保持）。
  • output: 生成されたJSON文字列を格納するためのポインタ。
  • outputlen: 生成された文字列の長さを格納するためのポインタ。
  • **出力**: なし（引数経由で output および outputlen を更新）。

前提条件と不変条件

  • ctx->issuer および ctx->scope が設定されていること。設定されていない場合は FATAL エラーとして処理を中断する。
  • output および outputlen は呼び出し元によってメモリ管理が可能な領域を指している必要がある。

アルゴリズムまたは振る舞い

1. ctx->issuer および ctx->scope の存在を確認し、欠落している場合は FATAL を送出する。 2. issuer 文字列を基に、.well-known/openid-configuration パスを含むURIを構築する（既に含まれている場合はそのまま使用）。 3. JSON形式のレスポンス文字列を構築する。

  • status: "invalid_token" を固定値として設定。
  • openid-configuration: 構築したURIをJSONエスケープして設定。
  • scope: ctx->scope をJSONエスケープして設定。

    • 4. 構築した文字列を output に、その長さを outputlen に代入する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-oauth.c 内の認証フロー（oauth_exchange 等）から呼び出される。
  • **呼び出し先**: initStringInfo, appendStringInfoString, escape_json, pfree 等のユーティリティ関数。

関連データ構造

  • struct oauth_ctx: 認証セッションの状態と設定情報を保持する構造体。
  • StringInfoData: 文字列構築用バッファ。

失敗モードと境界条件

  • ctx->issuer または ctx->scope が NULL の場合、FATAL エラーによりプロセスが終了する。
  • JSONエスケープ処理は、URIやスコープに不正な文字が含まれる場合に対する防御的プログラミングとして機能する。

保守担当者向けメモ

  • 本関数はRFC 7628に基づくエラー応答を生成する。将来的にHBA設定でこれらのパラメータが必須化された場合、FATAL チェックの重要性が増す。
  • output に割り当てられたメモリは、呼び出し元で適切に解放される必要がある。
