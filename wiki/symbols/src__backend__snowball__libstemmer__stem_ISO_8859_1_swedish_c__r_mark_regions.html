r_mark_regions

目的

単語内の領域（R1など）を定義し、ステミング処理の対象範囲を制限するためのインデックスを計算・記録します。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • **入力**: SN_env 構造体（現在のカーソル位置 z->c、文字列長 z->l を含む）。
  • **出力**: 成功時に 1、失敗時に 0 を返します。副作用として z->I 配列（整数ワークスペース）に領域境界を書き込みます。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要があります。
  • z->I[0] および z->I[1] は、本関数によって領域境界として更新されます。

アルゴリズムまたは振る舞い

1. z->I[1] を文字列長 z->l で初期化します。 2. カーソル位置から3文字進めた位置を z->I[0] に記録します（範囲の最小境界）。 3. out_grouping を使用して、母音（g_v）以外の文字を検索し、その位置を特定します。 4. 続いて in_grouping を使用して、母音（g_v）の連続する範囲をスキップし、その後の位置を z->I[1] に更新します。 5. z->I[1] が z->I[0] より小さい場合、z->I[1] を z->I[0] に補正します。

呼び出し元と呼び出し先

  • **呼び出し元**: 外部からの直接的な呼び出し元はソースコードから特定できません。
  • **呼び出し先**: out_grouping および in_grouping（Snowballランタイム関数）を呼び出します。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。z->I（整数配列）、z->c（カーソル）、z->l（長さ）を使用します。

失敗モードと境界条件

  • z->c + 3 が文字列長 z->l を超える場合、0 を返して終了します。
  • out_grouping または in_grouping が失敗した場合、0 を返して終了します。
