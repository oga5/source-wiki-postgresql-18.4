getTypeIOParam

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/lsyscache.c:2480

目的

データ型の入出力関数に渡すべきパラメータ（OID）を決定します。

シグネチャまたは形状

Oid getTypeIOParam(HeapTuple typeTuple)

入力と出力

  • **入力**: typeTuple (HeapTuple) - pg_type カタログの行データ。
  • **出力**: Oid - 入出力関数に渡すパラメータとしての型 OID。

前提条件と不変条件

  • typeTuple は Form_pg_type 構造体として解釈可能な有効なタプルである必要があります。

アルゴリズムまたは振る舞い

1. GETSTRUCT マクロを使用して typeTuple を Form_pg_type 構造体にキャストします。 2. typeStruct->typelem が有効な OID であるかを確認します。 3. 配列型の場合（typelem が有効な場合）、その要素型である typelem を返します。 4. それ以外の場合、その型自身の OID (typeStruct->oid) を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_type.c
  • src/backend/utils/cache/lsyscache.c (内部の複数箇所)
  • src/pl/plperl/plperl.c
  • src/pl/tcl/pltcl.c
  • **呼び出し先**:
  • GETSTRUCT マクロ（access/htup_details.h）

関連データ構造

  • Form_pg_type: pg_type システムカタログの行構造体。

失敗モードと境界条件

  • typeTuple が無効な場合、GETSTRUCT の呼び出しにより未定義の動作が発生する可能性があります。

保守担当者向けメモ

  • src/backend/bootstrap/bootstrap.c 内のロジックと本関数が一致している必要がある旨のコメントがソースコード内に存在します。変更時は注意してください。
