heap2_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/heapdesc.c:433

目的

ヒープの補助操作（XLOG_HEAP2_*）に関連するWALレコードのフラグ値を、人間が判読可能な文字列形式に変換します。主に pg_waldump 等のツールでWALレコードの種類を識別するために使用されます。

シグネチャまたは形状

const char *heap2_identify(uint8 info);

入力と出力

  • **入力**: info (uint8型) - WALレコードの xl_info フィールドから取得されたフラグ値。
  • **出力**: const char * - 対応する操作名を示す文字列定数。該当するフラグがない場合は NULL を返します。

前提条件と不変条件

  • 入力 info は XLR_INFO_MASK でマスクされた後の値として扱われます。
  • 戻り値は静的な文字列リテラルへのポインタであり、呼び出し側で解放する必要はありません。

アルゴリズムまたは振る舞い

1. info から XLR_INFO_MASK を除去し、純粋な操作フラグを抽出します。 2. switch 文を用いて、抽出されたフラグ値と XLOG_HEAP2_* 定数を照合します。 3. 一致する定数に対応する文字列（例: "PRUNE_ON_ACCESS", "MULTI_INSERT" 等）を返します。 4. XLOG_HEAP2_MULTI_INSERT と XLOG_HEAP_INIT_PAGE の組み合わせなど、特定の複合フラグにも対応しています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/access/rmgrlist.h にて、リソースマネージャ（RM_HEAP2_ID）の記述子関数として登録されており、WAL解析エンジンから間接的に呼び出されます。
  • **呼び出し先**:
  • 本関数は他の関数を呼び出しません。

関連データ構造

  • access/heapam_xlog.h: XLOG_HEAP2_* 関連の定数定義が格納されています。

失敗モードと境界条件

  • 定義されていないフラグ値が渡された場合、関数は NULL を返します。呼び出し側はこの戻り値が NULL である可能性を考慮する必要があります。

保守担当者向けメモ

  • 新しい XLOG_HEAP2_* タイプのWALレコードを追加した際は、必ず本関数の switch 文に新しいケースを追加してください。
  • XLOG_HEAP_INIT_PAGE のような修飾フラグとの組み合わせが必要な場合は、既存の MULTI_INSERT の例を参考にしてください。
