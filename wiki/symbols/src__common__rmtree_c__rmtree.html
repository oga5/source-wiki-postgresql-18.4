rmtree

定義

  • 種別: function
  • 定義箇所: src/common/rmtree.c:49

目的

指定されたディレクトリとその配下の全ファイルおよびサブディレクトリを再帰的に削除します。

シグネチャまたは形状

bool rmtree(const char *path, bool rmtopdir)

入力と出力

  • **入力**:
  • path: 削除対象のディレクトリパス。
  • rmtopdir: true の場合、対象ディレクトリ自身も最後に削除します。
  • **出力**:
  • bool: 削除処理中にエラーが発生しなかった場合は true、一つでも削除失敗があった場合は false を返します。

前提条件と不変条件

  • OPENDIR マクロによりディレクトリを開く必要があります。
  • ファイル記述子の枯渇を防ぐため、サブディレクトリの走査は現在のディレクトリを閉じた後に再帰的に行われます。

アルゴリズムまたは振る舞い

1. OPENDIR で対象ディレクトリを開きます。 2. readdir でエントリを走査します。

  • . および .. はスキップします。
  • ファイルの場合は unlink で削除します。
  • ディレクトリの場合は、パスを動的配列 dirnames に蓄積します。

    • 3. CLOSEDIR でディレクトリを閉じます。 4. 蓄積したサブディレクトリのパスに対して、rmtree を再帰的に呼び出します。 5. rmtopdir が true の場合、最後に rmdir で対象ディレクトリ自身を削除します。 6. 削除失敗が発生しても処理を中断せず、可能な限り他のエントリの削除を試みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/dbcommands.c
  • src/backend/replication/slot.c
  • src/backend/storage/sync/sync.c
  • **呼び出し先**:
  • get_dirent_type (ヘルパー関数)
  • unlink, rmdir (システムコール)
  • palloc, repalloc, pstrdup, pfree (メモリ管理)
  • pg_log_warning (ログ出力)

関連データ構造

  • DIR: ディレクトリ操作用構造体。
  • dirent: ディレクトリ内のエントリ情報。
  • dirnames: サブディレクトリパスを一時保持するための動的配列。

失敗モードと境界条件

  • ディレクトリが開けない場合、警告を出力し false を返します。
  • ファイル削除 (unlink) やディレクトリ削除 (rmdir) に失敗した場合、警告を出力し、戻り値を false に設定した上で処理を継続します。
  • errno が設定されている場合、ディレクトリ読み込みエラーとして警告を出力します。

保守担当者向けメモ

  • 本関数はバックエンドおよびフロントエンドの両方から利用可能です。
  • ファイル記述子の節約のため、ディレクトリ走査と再帰処理を分離する設計になっています。
  • 削除失敗時も処理を継続する仕様のため、呼び出し元は戻り値を確認して適切なエラーハンドリングを行う必要があります。
