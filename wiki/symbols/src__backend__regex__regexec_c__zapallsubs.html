zapallsubs

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexec.c:664

目的

正規表現のマッチング結果を保持する配列において、すべてのサブ式（キャプチャグループ）のマッチ状態を「マッチなし」に初期化する。

シグネチャまたは形状

static void zapallsubs(regmatch_t *p, size_t n)

入力と出力

  • **入力**:
  • p: regmatch_t 型の配列へのポインタ。マッチ結果を格納する領域。
  • n: 配列 p の要素数。
  • **出力**: なし（戻り値なし）。配列 p の内容が直接更新される。

前提条件と不変条件

  • p は少なくとも n 個の要素を持つ有効なメモリ領域を指している必要がある。
  • インデックス 0（全体マッチ）は対象外とし、インデックス 1 から n-1 までのサブ式のみをリセットする。

アルゴリズムまたは振る舞い

1. 配列の末尾（n-1）から開始し、インデックス 1 に達するまで逆順にループする。 2. 各要素の rm_so（開始位置）および rm_eo（終了位置）に -1 を代入する。 3. これにより、該当するサブ式がマッチしなかったことを示す標準的な状態に設定される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regexec.c 内の複数箇所（行 237, 245, 325 など）で、マッチング処理の開始前やリセット時に呼び出されている。
  • **呼び出し先**: なし。

関連データ構造

  • regmatch_t: POSIX 正規表現ライブラリで使用される、マッチした部分文字列の開始・終了位置を保持する構造体。

失敗モードと境界条件

  • n が 0 または 1 の場合、ループ条件 i > 0 が即座に偽となるため、何も処理を行わずに終了する。

保守担当者向けメモ

  • 本関数は、マッチングの試行前に pmatch 配列をクリーンな状態にするために使用される。
  • インデックス 0（全体マッチ）は本関数ではクリアされないため、呼び出し側で別途管理する必要がある点に注意すること。
