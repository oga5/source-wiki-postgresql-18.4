AllocSetDelete

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/aset.c:606

目的

AllocSet メモリコンテキストを破棄し、関連付けられたメモリブロックを解放する。コンテキストが再利用可能なフリーリストの対象である場合は、破棄せずにフリーリストへ格納する。

シグネチャまたは形状

void AllocSetDelete(MemoryContext context)

入力と出力

  • **入力**: context (破棄対象の MemoryContext へのポインタ)
  • **出力**: なし

前提条件と不変条件

  • context は AllocSetIsValid を満たす有効な AllocSet であること。
  • MEMORY_CONTEXT_CHECKING が有効な場合、内部のメモリ整合性チェックが実行される。

アルゴリズムまたは振る舞い

1. **整合性チェック**: デバッグビルド時、AllocSetCheck を呼び出しメモリの破損やリークを検証する。 2. **フリーリスト管理**:

  • set->freeListIndex >= 0 の場合、コンテキストを再利用可能なフリーリストへ追加する。
  • 必要に応じて MemoryContextResetOnly を実行し、初期ブロック以外のメモリを解放する。
  • フリーリストが MAX_FREE_CONTEXTS に達している場合、既存のコンテキストを free() して空きを作る。
  • コンテキストをフリーリストの先頭に繋ぎ、処理を終了する。

    • 3. **メモリ解放**:

  • フリーリスト対象外の場合、保持している全ブロックを走査する。
  • 「キーパーブロック（コンテキストヘッダーと共有）」以外のブロックを free() し、mem_allocated を更新する。
  • CLOBBER_FREED_MEMORY が定義されている場合、解放前にメモリを上書きする。

    • 4. **ヘッダー解放**: 最後にコンテキストヘッダー（キーパーブロックを含む）を free() する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c: MCTX_ASET_ID の delete_context 関数ポインタとして登録。
  • **呼び出し先**:
  • AllocSetCheck (デバッグ時)
  • MemoryContextResetOnly
  • free() (標準ライブラリ)
  • wipe_mem (デバッグ時)

関連データ構造

  • AllocSetContext: 操作対象のメモリコンテキスト構造体。
  • AllocBlock: 管理対象のメモリブロック。
  • AllocSetFreeList: コンテキスト再利用のためのグローバルなフリーリスト。

失敗モードと境界条件

  • context が無効な場合、Assert により停止する。
  • フリーリストが満杯の場合、古いコンテキストを破棄してメモリを解放する。
  • キーパーブロックはコンテキストヘッダーの一部であるため、個別の free() 対象からは除外される。

保守担当者向けメモ

  • 本関数は MemoryContext インターフェースの一部であり、mcxt.c からの呼び出しが主である。
  • フリーリストの管理ロジックは、メモリ確保・解放のオーバーヘッドを削減するための重要な最適化である。
