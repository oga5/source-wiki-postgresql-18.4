Buffer

定義

  • 種別: typedef int
  • 定義箇所: src/include/storage/buf.h:23

目的

PostgreSQL のバッファ管理サブシステムにおいて、共有バッファまたはローカルバッファを識別するための共通の整数型。

シグネチャまたは形状

typedef int Buffer;

入力と出力

  • 本型は、バッファマネージャの関数（例: ReadBuffer）の戻り値として使用される。
  • 呼び出し元は、返された値に対して BufferIsInvalid マクロ等を用いて有効性を判定する。

前提条件と不変条件

  • 値が 0 の場合、InvalidBuffer（無効なバッファ）を示す。
  • 正の値は共有バッファを指す。
  • 負の値はローカルバッファ（現在のバックエンド専用）を指す。

アルゴリズムまたは振る舞い

本シンボル自体は型定義であり、直接的な制御フローは持たない。バッファの状態判定には、buf.h で定義される BufferIsInvalid(buffer) や BufferIsLocal(buffer) マクロが併用される。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の例）:
  • contrib/amcheck/verify_gin.c
  • contrib/amcheck/verify_heapam.c
  • contrib/amcheck/verify_nbtree.c
  • 呼び出し先:
  • 本型は純粋な型定義であり、関数ではないため呼び出し先は存在しない。

関連データ構造

  • BufferAccessStrategy: バッファ読み込み・書き出し時の戦略を管理する構造体へのポインタ。
  • buf_internals.h: バッファ記述子などの詳細な内部構造が定義されており、本型と組み合わせて使用される。

失敗モードと境界条件

  • 0（InvalidBuffer）は無効なバッファとして扱われるため、バッファ操作関数に渡す前に必ずチェックする必要がある。

保守担当者向けメモ

  • Buffer 型が負の値をとることでローカルバッファを表現している点は、PostgreSQL のメモリ管理における重要な設計上の特徴である。
  • バッファ管理のロジックを理解する際の「辞書」として参照すること。
