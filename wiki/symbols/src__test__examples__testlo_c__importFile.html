importFile

定義

  • 種別: function
  • 定義箇所: src/test/examples/testlo.c:33

目的

ローカルファイルシステム上のファイルを読み込み、PostgreSQL データベース内にラージオブジェクトとしてインポートする補助関数です。

シグネチャまたは形状

static Oid importFile(PGconn *conn, char *filename)

入力と出力

  • **入力**:
  • conn: データベース接続を管理する PGconn 構造体へのポインタ。
  • filename: インポート対象のローカルファイルパスを示す文字列。
  • **出力**:
  • 成功時: 作成されたラージオブジェクトの Oid を返します。
  • 失敗時: lo_creat が失敗した場合は 0 を返します。

前提条件と不変条件

  • conn は有効なデータベース接続状態である必要があります。
  • ファイルシステム上の filename は読み取り可能である必要があります。
  • ラージオブジェクトの操作は、トランザクションのコンテキスト内で行われることが想定されています。

アルゴリズムまたは振る舞い

1. 指定された filename を open 関数で読み取り専用モードで開きます。 2. lo_creat を呼び出し、読み書き可能なラージオブジェクトを作成して Oid を取得します。 3. lo_open を使用して、作成したラージオブジェクトを書き込みモードで開きます。 4. ローカルファイルから BUFSIZE 単位でデータを読み込み、lo_write を使用してラージオブジェクトへ順次書き込みます。 5. ファイルおよびラージオブジェクトのファイル記述子をそれぞれ close および lo_close で閉じます。 6. 最後に作成したラージオブジェクトの Oid を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/examples/testlo.c および src/test/examples/testlo64.c 内のコードから参照されています（コメントアウトされた例を含む）。
  • **呼び出し先**:
  • 標準ライブラリ: open, read, close, fprintf
  • libpq API: lo_creat, lo_open, lo_write, lo_close

関連データ構造

  • PGconn: libpq のデータベース接続ハンドル。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • ファイルのオープンに失敗した場合、標準エラー出力にエラーメッセージを表示しますが、関数は処理を継続します。
  • lo_creat が失敗した場合、0 を返し、標準エラー出力にエラーメッセージを表示します。
  • lo_write による書き込みバイト数が読み込みバイト数より少ない場合、エラーメッセージを表示します。
  • 本関数はエラー発生時に呼び出し元へ例外を投げたり終了したりせず、そのまま処理を続行して戻り値を返します。

保守担当者向けメモ

  • 本関数はサンプルコードとしての実装であり、エラーハンドリングは最小限です。本番環境で使用する場合は、PQerrorMessage を用いた詳細なエラーチェックを実装してください。
  • 実際のアプリケーション開発では、手動実装ではなくライブラリ提供の lo_import 関数の使用が推奨されます。
