ginxlogInsertEntry

定義

  • 種別: struct
  • 定義箇所: src/include/access/ginxlog.h:57

目的

GINインデックスの挿入操作をWAL（Write-Ahead Log）に記録するためのデータ構造です。リカバリ時に特定のインデックスページに対するタプルの挿入または削除を再現するために使用されます。

シグネチャまたは形状

typedef struct {
    OffsetNumber offset;
    bool         isDelete;
    IndexTupleData tuple; /* 可変長 */
} ginxlogInsertEntry;

入力と出力

  • **入力**: GINインデックス更新時に、挿入対象のオフセット、削除フラグ、およびインデックスタプルデータが本構造体に格納されます。
  • **出力**: WALレコードの一部として永続化され、リカバリ時に gin_redo を通じてページ状態の復元に使用されます。

前提条件と不変条件

  • tuple フィールドは可変長データとして構造体の末尾に配置されています。
  • 本構造体は memcpy を用いて直接読み書きされるため、フィールドの順序とサイズはバイナリ互換性を維持するために厳密に管理される必要があります。

アルゴリズムまたは振る舞い

1. **記録**: GINインデックスの更新時、ginxlogInsertEntry に情報を詰め、XLogInsert を介してWALへ書き出します。 2. **再生**: リカバリ時、gin_redo がレコードを読み取り、isDelete フラグを確認します。true であれば指定オフセットのタプルを削除し、false であれば挿入を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginentrypage.c において、インデックスエントリの挿入時に本構造体が初期化されます。
  • **呼び出し先**: src/backend/access/rmgrdesc/gindesc.c がWALダンプ出力時に本構造体を参照します。また、src/backend/access/gin/ginxlog.c がREDO処理時に本構造体をキャストして利用します。

関連データ構造

  • IndexTupleData: インデックス内のタプルデータを保持します。
  • ginxlogInsert: 挿入操作の共通ヘッダー構造体。

失敗モードと境界条件

  • **アライメント**: 構造体定義にフィールドを追加する場合、パディングがWALのバイナリ互換性を損なう可能性があるため注意が必要です。
  • **可変長データ**: tuple は可変長であるため、構造体サイズを計算する際は offsetof(ginxlogInsertEntry, tuple) を使用する必要があります。

保守担当者向けメモ

  • 本構造体はWALのバイナリ形式を直接規定しているため、フィールドの変更は後方互換性に影響を与えます。
  • デバッグ時には pg_waldump を使用して、gin_desc 関数経由で本構造体の内容を確認可能です。
