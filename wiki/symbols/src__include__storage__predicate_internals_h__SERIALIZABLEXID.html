SERIALIZABLEXID

定義

  • 種別: struct
  • 定義箇所: src/include/storage/predicate_internals.h:240

目的

トランザクションID（XID）と、それに対応する直列化可能トランザクション状態（SERIALIZABLEXACT）を共有メモリ上で関連付けるためのハッシュテーブルエントリ構造体です。

シグネチャまたは形状

typedef struct SERIALIZABLEXID
{
    SERIALIZABLEXIDTAG tag;      /* ハッシュキー */
    SERIALIZABLEXACT *myXact;    /* トップレベルトランザクションデータへのポインタ */
} SERIALIZABLEXID;

入力と出力

  • **入力**: SERIALIZABLEXIDTAG（ハッシュキーとして使用）
  • **出力**: SERIALIZABLEXACT 構造体への参照

前提条件と不変条件

  • 共有メモリ上のハッシュテーブル（SerializableXidHash）内で管理される。
  • myXact は、対応するトップレベルトランザクションの有効な状態構造体を指す必要がある。

アルゴリズムまたは振る舞い

  • predicate.c において、トランザクションIDからそのトランザクションのSSI（Serializable Snapshot Isolation）状態を高速に検索するために使用される。
  • hash_search を介して、ハッシュテーブルへの挿入、検索、削除が行われる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/lmgr/predicate.c 内の関数群（トランザクションの開始、状態追跡、クリーンアップ処理など）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行わない。

関連データ構造

  • SERIALIZABLEXIDTAG: ハッシュテーブルの検索キー。
  • SERIALIZABLEXACT: 本構造体が参照する、トランザクションのSSI状態を保持するメイン構造体。
  • SerializableXidHash: 本構造体を格納する共有メモリ上のハッシュテーブル。

失敗モードと境界条件

  • ハッシュテーブルの初期化失敗や、メモリ不足による hash_search の失敗が想定される。
  • トランザクション終了時のクリーンアップ漏れにより、ハッシュテーブル内にエントリが残存する可能性がある。

保守担当者向けメモ

  • 本構造体は SerializableXidHash のエントリサイズとして使用されるため、構造体のサイズ変更はハッシュテーブルの整合性に影響を与える。
  • SERIALIZABLEXIDTAG の定義変更時は、ハッシュ計算のパディングやアライメントに注意すること。
