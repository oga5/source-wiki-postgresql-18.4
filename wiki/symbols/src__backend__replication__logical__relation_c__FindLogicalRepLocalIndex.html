FindLogicalRepLocalIndex

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/relation.c:907

目的

論理レプリケーションにおいて、リモートの変更を適用するために使用可能なローカルテーブルのインデックス（Oid）を特定します。

シグネチャまたは形状

static Oid FindLogicalRepLocalIndex(Relation localrel, LogicalRepRelation *remoterel, AttrMap *attrMap)

入力と出力

  • **入力**:
  • localrel: 対象となるローカルリレーションの Relation 構造体。
  • remoterel: リモートリレーションの定義情報。
  • attrMap: ローカルとリモートの属性マッピング情報。
  • **出力**:
  • 使用可能なインデックスの Oid。該当するものがない場合は InvalidOid を返します。

前提条件と不変条件

  • パーティションテーブル（RELKIND_PARTITIONED_TABLE）に対しては、常に InvalidOid を返します（リーフパーティション側のインデックスを使用するため）。

アルゴリズムまたは振る舞い

1. **パーティション判定**: localrel がパーティションテーブルであれば即座に InvalidOid を返します。 2. **既存インデックスの確認**: GetRelationIdentityOrPK を呼び出し、プライマリキーまたはレプリカアイデンティティとして定義されたインデックスを優先的に取得します。有効な Oid が得られればそれを返します。 3. **REPLICA IDENTITY FULL の処理**: リモート側の設定が REPLICA_IDENTITY_FULL である場合、FindUsableIndexForReplicaIdentityFull を呼び出し、利用可能なインデックスを探索して返します。 4. **デフォルト**: 上記のいずれにも該当しない場合、InvalidOid を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/relation.c 内の logicalrep_rel_open（行486）および logicalrep_partition_open（行758）から呼び出されます。
  • **呼び出し先**:
  • GetRelationIdentityOrPK
  • FindUsableIndexForReplicaIdentityFull

関連データ構造

  • Relation: ローカルテーブルのメタデータ。
  • LogicalRepRelation: リモートリレーションの定義。
  • AttrMap: 属性マッピング情報。

失敗モードと境界条件

  • 対象テーブルに適切なインデックスが存在しない場合、InvalidOid を返します。
  • パーティションテーブルが渡された場合、インデックス探索を行わず InvalidOid を返します。

保守担当者向けメモ

  • 本関数はプランナを使用せず、インデックスの選択を行います。これはプランナのオーバーヘッドを回避し、メンテナンスコストを抑えるための設計です。
  • REPLICA_IDENTITY_FULL の場合、すべての列がインデックススキャンに使用可能であるという前提でインデックスを選択します。
