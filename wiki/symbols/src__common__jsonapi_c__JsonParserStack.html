JsonParserStack

定義

  • 種別: struct
  • 定義箇所: src/common/jsonapi.c:144

目的

インクリメンタル（非再帰的）なJSON構文解析において、解析状態と予測スタックを保持するために使用されます。

シグネチャまたは形状

struct JsonParserStack
{
    int         stack_size;
    char       *prediction;
    size_t      pred_index;
    char      **fnames;
    bool       *fnull;
    JsonTokenType scalar_tok;
    char       *scalar_val;
};

入力と出力

  • **入力**: pg_parse_json_incremental 実行時にコンテキストの一部として初期化され、解析の進行に伴い push_prediction や pop_prediction 等を通じて状態が更新されます。
  • **出力**: 解析結果として、スタック内の fnames（フィールド名）や scalar_val（スカラー値）がコールバック関数へ提供されます。

前提条件と不変条件

  • fnames および fnull は lex_level（解析の深さ）に基づいてインデックスされます。
  • インクリメンタル解析の進行中、スタックは現在の構文構造を正しく反映している必要があります。

アルゴリズムまたは振る舞い

  • 「Dragon Book」のアルゴリズム4.3に基づくテーブル駆動型パーサーの予測スタックとして機能します。
  • prediction 文字列と pred_index を用いて、次に期待される構文要素を追跡します。
  • 解析の深さに応じてフィールド名やNULLフラグを管理し、スカラー値の検出時には scalar_tok と scalar_val に値を保持します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/jsonapi.c: pg_parse_json_incremental 内でメモリ確保および参照。
  • **呼び出し先**:
  • src/common/jsonapi.c: push_prediction, pop_prediction, next_prediction, have_prediction などの操作関数によって状態が操作されます。

関連データ構造

  • JsonLexContext: 本構造体を保持する上位の解析コンテキスト。
  • td_entry: 予測テーブルの要素。

失敗モードと境界条件

  • メモリ不足時: ALLOC0 による確保失敗の可能性がある。
  • スタックオーバーフロー: 非常に深いネストを持つJSON入力に対して、stack_size の制限を超える可能性がある。

保守担当者向けメモ

  • 本構造体は src/include/common/jsonapi.h にも定義が存在します。
  • lex_level に依存する配列（fnames, fnull）のサイズ管理には注意が必要です。
