compare_fractional_path_costs

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/pathnode.c:126

目的

2つの Path ノードのコストを、指定された割合（fraction）に基づいて比較し、どちらがより低コストかを判定する。

シグネチャまたは形状

int compare_fractional_path_costs(Path *path1, Path *path2, double fraction)

入力と出力

  • **入力**:
  • path1, path2: 比較対象の Path 構造体へのポインタ。
  • fraction: コスト計算に用いる割合（0.0 から 1.0 の範囲）。
  • **出力**:
  • path1 が小さい場合は -1、path2 が小さい場合は +1、等しい場合は 0 を返す。

前提条件と不変条件

  • path1->disabled_nodes と path2->disabled_nodes が異なる場合、コスト計算よりも無効化されたノード数が優先される。
  • fraction が 0.0 以下または 1.0 以上の場合、compare_path_costs を用いて総コスト（TOTAL_COST）による比較にフォールバックする。

アルゴリズムまたは振る舞い

1. **無効ノード数の比較**: disabled_nodes を比較し、値が小さい方を優先する。 2. **範囲チェック**: fraction が [0, 1] の範囲外であれば、compare_path_costs を呼び出し、その結果を返す。 3. **コスト計算**: 各パスのコストを startup_cost + fraction * (total_cost - startup_cost) の式で算出する。 4. **比較**: 算出された cost1 と cost2 を比較し、大小関係を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/pathkeys.c
  • src/backend/optimizer/plan/planagg.c
  • src/backend/optimizer/plan/planner.c
  • **呼び出し先**:
  • compare_path_costs（fraction が範囲外の場合）

関連データ構造

  • Path: PostgreSQLのクエリ実行パスを表現する構造体。startup_cost, total_cost, disabled_nodes フィールドを使用。

失敗モードと境界条件

  • fraction が 0.0 または 1.0 の境界値では、compare_path_costs 経由で総コスト比較が行われる。
  • disabled_nodes の不一致がコスト比較よりも優先されるため、コストが低くても無効ノード数が多いパスは「大きい」と判定される。

保守担当者向けメモ

  • 本関数は Path のコスト比較における「部分的な実行」を評価するために使用される。
  • compare_path_costs_fuzzily と異なり、本関数は浮動小数点誤差を考慮した曖昧な比較を行わない点に注意すること。
