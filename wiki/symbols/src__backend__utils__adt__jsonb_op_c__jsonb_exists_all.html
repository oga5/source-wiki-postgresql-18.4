jsonb_exists_all

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_op.c:78

目的

jsonb 型のデータ内に、指定されたテキスト配列のすべての要素がキーまたは要素として存在するかを判定します（SQL演算子 ?& の実装）。

シグネチャまたは形状

Datum jsonb_exists_all(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_JSONB_P(0): 検索対象の jsonb オブジェクト。
  • PG_GETARG_ARRAYTYPE_P(1): 検索するキーの集合（text[] 型）。
  • **出力**:
  • bool: すべてのキーが存在すれば true、一つでも欠けていれば false を返します。

前提条件と不変条件

  • 入力配列の各要素は TEXTOID 型として扱われます。
  • 配列要素が NULL の場合はスキップされます。
  • 検索対象の jsonb は、トップレベルのキーまたは要素のみが探索対象となります（再帰的な探索は行われません）。

アルゴリズムまたは振る舞い

1. deconstruct_array_builtin を使用して、入力配列を Datum 配列に分解します。 2. 配列内の各要素をループで走査します。 3. 各要素を JsonbValue（jbvString 型）に変換します。 4. findJsonbValueFromContainer を呼び出し、jsonb コンテナ内に当該文字列が存在するかを確認します。 5. 一つでも存在しない要素が見つかった時点で false を返し終了します。 6. すべての要素の確認が完了した場合、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（fmgr）経由で呼び出されます。直接的なコード上の呼び出し元はソースコード内では特定されていません。
  • **呼び出し先**:
  • deconstruct_array_builtin
  • findJsonbValueFromContainer

関連データ構造

  • Jsonb: 検索対象となる jsonb 構造体。
  • ArrayType: 入力キー配列の内部表現。
  • JsonbValue: 検索キーを一時的に保持するための構造体。

失敗モードと境界条件

  • 入力配列が空の場合、ループは実行されず true を返します。
  • 配列内に NULL 要素が含まれる場合、その要素は無視されます。
  • jsonb がオブジェクトまたは配列構造でない場合、findJsonbValueFromContainer の仕様に従い検索が失敗します。

保守担当者向けメモ

  • 本関数は jsonb_exists 系演算子の一部であり、トップレベルの探索に限定されています。深い階層の探索が必要な場合は別の関数（jsonb_contains 等）を検討してください。
  • 配列要素の文字列変換時に VARDATA_ANY および VARSIZE_ANY_EXHDR を使用しており、TOAST されていないことを前提としています。
