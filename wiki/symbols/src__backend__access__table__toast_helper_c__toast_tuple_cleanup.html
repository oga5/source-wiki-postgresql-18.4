toast_tuple_cleanup

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/toast_helper.c:274

目的

TOAST処理が完了したタプルに関連する一時的なメモリリソースの解放、および更新（UPDATE）操作によって不要となった古い外部ストレージデータの物理削除を行う。

シグネチャまたは形状

void toast_tuple_cleanup(ToastTupleContext *ttc)

入力と出力

  • **入力**: ToastTupleContext *ttc （処理対象のTOASTコンテキストへのポインタ）
  • **出力**: なし（副作用としてメモリ解放および外部データの削除を実行）

前提条件と不変条件

  • ttc は toast_tuple_init 等によって適切に初期化されていること。
  • ttc->ttc_rel が有効なリレーション記述子を指していること。

アルゴリズムまたは振る舞い

1. **一時メモリの解放**: ttc->ttc_flags に TOAST_NEEDS_FREE が設定されている場合、各属性の tai_colflags を確認し、TOASTCOL_NEEDS_FREE が立っている属性に対して pfree を実行する。 2. **古い外部データの削除**: ttc->ttc_flags に TOAST_NEEDS_DELETE_OLD が設定されている場合、各属性の tai_colflags を確認し、TOASTCOL_NEEDS_DELETE_OLD が立っている属性に対して toast_delete_datum を呼び出し、物理ストレージからデータを削除する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heaptoast.c:333
  • **呼び出し先**: pfree（メモリ解放）、toast_delete_datum（外部データ削除）

関連データ構造

  • ToastTupleContext: TOAST処理の状態を保持する構造体。
  • ToastAttrInfo: 各属性のTOAST状態（フラグ等）を保持する構造体。

失敗モードと境界条件

  • ttc が NULL の場合、または ttc->ttc_rel が無効な場合はクラッシュする可能性がある。
  • toast_delete_datum 内部でエラーが発生した場合、トランザクションの異常終了に伴うロールバックが発生する。

保守担当者向けメモ

  • 本関数はTOAST処理の最終フェーズで呼び出される。メモリリークや外部データのゴミ残りを防ぐため、ToastTupleContext のフラグ管理が正確に行われていることが重要である。
  • toast_delete_datum の第3引数（is_speculative）は false で固定されている。
