AddSubscriptionRelState

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_subscription.c:266

目的

論理レプリケーションにおけるサブスクリプションとテーブルの紐付け状態（pg_subscription_rel）を新規作成し、カタログに挿入します。

シグネチャまたは形状

void AddSubscriptionRelState(Oid subid, Oid relid, char state, XLogRecPtr sublsn, bool retain_lock)

入力と出力

  • **入力**:
  • subid: サブスクリプションの OID。
  • relid: 対象テーブルの OID。
  • state: 設定する同期状態（srsubstate）。
  • sublsn: 同期状態に関連する LSN（InvalidXLogRecPtr の場合は NULL として扱われる）。
  • retain_lock: 処理終了後にロックを保持するかどうかを指定するフラグ。
  • **出力**: なし（カタログへの挿入のみ）。

前提条件と不変条件

  • pg_subscription_rel カタログに対する RowExclusiveLock を使用します。
  • 指定された subid と relid の組み合わせが既に存在する場合、エラーとなります。
  • 処理の開始時に SubscriptionRelationId に対して AccessShareLock を取得し、サブスクリプションの整合性を保護します。

アルゴリズムまたは振る舞い

1. LockSharedObject を呼び出し、サブスクリプションに対する共有ロックを取得します。 2. table_open で pg_subscription_rel を開きます。 3. SearchSysCacheCopy2 を使用して、指定された subid と relid のペアが既に存在しないか確認します。存在する場合は ERROR を発生させます。 4. heap_form_tuple を用いて、入力値から新しいタプルを構築します。 5. CatalogTupleInsert を呼び出し、カタログへタプルを挿入します。 6. retain_lock が false の場合、table_close でロックを解放し、UnlockSharedObject でサブスクリプションの共有ロックを解除します。true の場合はロックを保持したまま終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/subscriptioncmds.c
  • src/backend/utils/adt/pg_upgrade_support.c
  • **呼び出し先**:
  • LockSharedObject / UnlockSharedObject
  • table_open / table_close
  • SearchSysCacheCopy2
  • heap_form_tuple / heap_freetuple
  • CatalogTupleInsert

関連データ構造

  • pg_subscription_rel: サブスクリプションとテーブルの紐付けを保持するシステムカタログ。
  • Subscription: サブスクリプション情報を保持する構造体（直接の操作対象ではないが、subid で関連付けられる）。

失敗モードと境界条件

  • 指定した subid と relid のペアが既に存在する場合、elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数は pg_subscription_rel カタログを直接操作するため、カタログのスキーマ変更時には values 配列のインデックスや nulls の設定を適切に更新する必要があります。
  • retain_lock 引数は、呼び出し元が引き続き同じトランザクション内で関連するロックを必要とする場合に利用されます。
