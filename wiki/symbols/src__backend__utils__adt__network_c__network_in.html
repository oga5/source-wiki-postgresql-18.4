network_in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network.c:74

目的

文字列形式の IP アドレス（inet または cidr）を解析し、PostgreSQL 内部の inet 構造体に変換する。

シグネチャまたは形状

static inet *network_in(char *src, bool is_cidr, Node *escontext)

入力と出力

  • **入力**:
  • src: 解析対象の IP アドレス文字列。
  • is_cidr: cidr 型として扱う場合は true、inet 型の場合は false。
  • escontext: エラー報告用コンテキスト。
  • **出力**:
  • 成功時: 変換された inet 構造体へのポインタ。
  • 失敗時: NULL（escontext を介してエラーを報告）。

前提条件と不変条件

  • cidr 型として解析する場合、マスク長より右側のビットはすべて 0 である必要がある（addressOK による検証）。
  • 入力文字列に : が含まれる場合は IPv6、それ以外は IPv4 として扱われる。

アルゴリズムまたは振る舞い

1. palloc0 を用いて inet 構造体をメモリ確保する。 2. 文字列内の : の有無に基づき、アドレスファミリー（IPv4/IPv6）を決定する。 3. pg_inet_net_pton を呼び出し、文字列をバイナリ形式に変換する。 4. 変換結果が不正な場合、またはビット長が許容範囲外の場合はエラーを返す。 5. is_cidr が true の場合、addressOK を用いてマスク長以降のビットが 0 であることを検証する。 6. 構造体のビット長をセットし、SET_INET_VARSIZE マクロで可変長ヘッダを初期化して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • inet_in（inet 型の入力関数）
  • cidr_in（cidr 型の入力関数）
  • 内部的なホスト名解決処理（remote_host / local_host の取得時）
  • **呼び出し先**:
  • pg_inet_net_pton: 文字列からバイナリへの変換。
  • addressOK: cidr 型のビット整合性チェック。
  • ereturn: エラー発生時の制御。

関連データ構造

  • inet: ネットワーク型を保持する内部構造体。

失敗モードと境界条件

  • **無効な構文**: pg_inet_net_pton が失敗した場合、ERRCODE_INVALID_TEXT_REPRESENTATION を発生させる。
  • **ビット範囲外**: 指定されたビット長がアドレスファミリーの最大値を超えている場合。
  • **CIDR 不整合**: cidr 型において、マスク長より右側にビットが立っている場合、invalid cidr value エラーとなる。

保守担当者向けメモ

  • 本関数は inet と cidr の両方の入力を処理する共通ルーチンである。
  • cidr の不変条件（マスク長以降のビットが 0）は、この関数で厳密に検証される。
  • pg_inet_net_pton の戻り値が負の場合、解析失敗とみなされる。
