pg_visibility_map

定義

  • 種別: function
  • 定義箇所: contrib/pg_visibility/pg_visibility.c:83

目的

指定されたリレーションの特定のブロックにおける可視性マップ（VM）のビット状態（all-visible および all-frozen）を取得し、結果をレコードとして返します。

シグネチャまたは形状

Datum pg_visibility_map(PG_FUNCTION_ARGS);

入力と出力

  • **入力**:
  • relid (Oid): 対象リレーションのOID。
  • blkno (int64): 検査対象のブロック番号。
  • **出力**:
  • Datum: all_visible (boolean) と all_frozen (boolean) の2列からなるタプル。

前提条件と不変条件

  • 対象リレーションは可視性マップを持つ種類（check_relation_relkind による検証）であること。
  • blkno は 0 以上かつ MaxBlockNumber 以下であること。
  • 実行には対象リレーションに対する AccessShareLock が取得される。

アルゴリズムまたは振る舞い

1. relation_open を用いて対象リレーションを AccessShareLock モードで開く。 2. リレーションの種類がVMをサポートしているか確認する。 3. ブロック番号の妥当性を検証し、範囲外であればエラーを報告する。 4. visibilitymap_get_status を呼び出し、指定ブロックのVMビットを取得する。 5. 取得したビットから VISIBILITYMAP_ALL_VISIBLE および VISIBILITYMAP_ALL_FROZEN の状態を判定する。 6. relation_close でロックを解放する。 7. 判定結果を heap_form_tuple でタプル化し、PG_RETURN_DATUM で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（PG_FUNCTION_INFO_V1 による登録）。
  • **呼び出し先**:
  • relation_open / relation_close
  • check_relation_relkind
  • pg_visibility_tupdesc
  • visibilitymap_get_status
  • ReleaseBuffer
  • heap_form_tuple

関連データ構造

  • Relation: 対象リレーションのメタデータ。
  • Buffer: VMページを一時的に保持するために使用。
  • TupleDesc: 返却されるタプルの構造定義。

失敗モードと境界条件

  • 指定された blkno が負の値、または MaxBlockNumber を超える場合はエラーとなる。
  • VMをサポートしないリレーションに対して実行された場合、check_relation_relkind によりエラーが発生する。

保守担当者向けメモ

  • 本関数は AccessShareLock を使用するため、実行中にVMビットが変更される可能性がある。
  • visibilitymap_get_status 呼び出し後にバッファが返却された場合は、必ず ReleaseBuffer を実行してリークを防ぐ必要がある。
