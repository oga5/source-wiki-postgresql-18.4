rel_supports_distinctness

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/analyzejoins.c:951

目的

指定されたリレーション（RelOptInfo）が、一意性（Distinctness）を保証する性質を持っているかどうかを判定します。この情報は、不要な結合の削除やセミジョインの簡略化の判断基準として利用されます。

シグネチャまたは形状

static bool rel_supports_distinctness(PlannerInfo *root, RelOptInfo *rel)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel: 判定対象のリレーション情報。
  • **出力**:
  • bool: リレーションが一意性をサポートしている場合は true、そうでない場合は false を返します。

前提条件と不変条件

  • rel->reloptkind が RELOPT_BASEREL である必要があります。それ以外の場合は常に false を返します。
  • RTE_RELATION の場合、即時適用（immediate）かつ部分インデックスではない（indpred == NIL）一意インデックスが少なくとも1つ存在する必要があります。

アルゴリズムまたは振る舞い

1. **種別判定**: reloptkind が RELOPT_BASEREL でない場合は false を返します。 2. **リレーション種別ごとの処理**:

  • RTE_RELATION: rel->indexlist を走査し、unique かつ immediate かつ indpred == NIL であるインデックスを探索します。見つかれば true を返します。
  • RTE_SUBQUERY: root->simple_rte_array からサブクエリを取得し、query_supports_distinctness を呼び出して判定結果を返します。

    • 3. **デフォルト**: 上記以外のケースでは false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/analyzejoins.c 内の複数箇所（行206, 908, 1432など）で、結合削除やセミジョイン簡略化の判定に使用されています。
  • **呼び出し先**:
  • query_supports_distinctness（RTE_SUBQUERY の場合）

関連データ構造

  • PlannerInfo: プランナのコンテキスト情報。
  • RelOptInfo: リレーションの最適化情報。
  • IndexOptInfo: インデックスの最適化情報。

失敗モードと境界条件

  • RELOPT_BASEREL 以外のリレーション（結合結果など）に対しては、一意性の証明を試みず false を返します。
  • 部分インデックスや遅延評価される一意インデックスは、一意性の証明には使用されません。

保守担当者向けメモ

  • 本関数での一意性判定条件は、relation_has_unique_index_for と同期を保つ必要があります。
  • 現在のところ、RTE_RELATION と RTE_SUBQUERY 以外の rtekind に対する証明ルールは実装されていません。
