get_cached_uuid_t

定義

  • 種別: function
  • 定義箇所: contrib/uuid-ossp/uuid-ossp.c:156

目的

OSSP UUID ライブラリを使用する際、セッション内で再利用される uuid_t オブジェクトを管理し、生成時のオーバーヘッドを削減します。

シグネチャまたは形状

static uuid_t *get_cached_uuid_t(int which)

入力と出力

  • **入力**: which (int) - キャッシュするオブジェクトのインデックス（0 または 1）。
  • **出力**: uuid_t * - 要求されたインデックスに対応する UUID オブジェクトへのポインタ。

前提条件と不変条件

  • cached_uuid は静的配列として保持され、初回呼び出し時に NULL で初期化されます。
  • which には 0 または 1 が指定されることを想定しています。

アルゴリズムまたは振る舞い

1. 指定された which に対応するキャッシュが NULL かどうかを確認します。 2. NULL の場合、uuid_create() を呼び出して新しい uuid_t オブジェクトを生成します。 3. 生成に失敗した場合は pguuid_complain() を呼び出し、キャッシュを NULL に維持したままエラーを処理します。 4. 最後に、キャッシュされた（または新規生成された）uuid_t ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/uuid-ossp/uuid-ossp.c 内の UUID 生成関数群（インデックス 0 および 1 を指定して呼び出し）。
  • **呼び出し先**:
  • uuid_create() (OSSP ライブラリ関数)
  • pguuid_complain() (エラーハンドリング用)

関連データ構造

  • uuid_t: OSSP UUID ライブラリが定義する UUID オブジェクト型。

失敗モードと境界条件

  • uuid_create() が UUID_RC_OK 以外のリターンコードを返した場合、キャッシュは更新されず、pguuid_complain() を通じてエラーが報告されます。

保守担当者向けメモ

  • 本関数は MAC アドレス取得等のコストを抑えるために重要です。キャッシュのライフサイクルやインデックスの割り当てを変更する際は、UUID の一意性への影響を考慮してください。
