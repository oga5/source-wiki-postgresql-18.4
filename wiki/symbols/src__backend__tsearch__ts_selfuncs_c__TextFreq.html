TextFreq

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/ts_selfuncs.c:35

目的

全文検索における統計情報（MCELEM: Most Common Elements）を保持し、特定の語彙（Lexeme）とその出現頻度を関連付けるためのデータ構造です。

シグネチャまたは形状

typedef struct
{
    text       *element;
    float4      frequency;
} TextFreq;

入力と出力

  • element: 語彙を表す text 型のポインタ。
  • frequency: その語彙の出現頻度を示す float4 型の値。

前提条件と不変条件

  • pg_statistic から抽出された統計情報を格納するために使用されます。
  • bsearch を用いた検索を行うため、この構造体の配列は語彙順にソートされている必要があります。

アルゴリズムまたは振る舞い

  • mcelem_tsquery_selec 関数において、統計情報から動的にメモリ確保（palloc）され、配列として構築されます。
  • bsearch を使用して特定の語彙を検索し、対応する頻度を取得するために利用されます。
  • 比較関数 compare_lexeme_textfreq を介して、語彙の長さおよび内容に基づいた順序付けが行われます。

呼び出し元と呼び出し先

  • 呼び出し元: mcelem_tsquery_selec および tsquery_opr_selec がこの構造体の配列を生成・操作します。
  • 呼び出し先: bsearch（標準ライブラリ）がこの構造体の配列を検索するために呼び出されます。

関連データ構造

  • pg_statistic: 統計情報の元データ。
  • tsvector: 全文検索対象のデータ型。

失敗モードと境界条件

  • メモリ不足時: palloc によるメモリ確保が失敗する可能性があります。
  • 統計情報欠如時: 統計情報が存在しない場合、この構造体は生成されず、デフォルトの選択率が使用されます。

保守担当者向けメモ

  • compare_lexeme_textfreq での比較ロジックは、語彙の長さとバイト単位の比較に依存しているため、ソート順序の整合性に注意してください。
  • tsquery の再帰評価時に TextFreq 配列を走査するため、配列のサイズが大きい場合は検索コストに影響します。
