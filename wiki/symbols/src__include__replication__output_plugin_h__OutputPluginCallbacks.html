OutputPluginCallbacks

定義

  • 種別: struct
  • 定義箇所: src/include/replication/output_plugin.h:36

目的

論理デコーディングにおいて、出力プラグインが実装すべきコールバック関数群を保持し、コアエンジンとプラグイン間のインターフェースを定義する。

シグネチャまたは形状

関数ポインタをメンバとして保持する構造体。プラグインの初期化関数 _PG_output_plugin_init を通じて、この構造体の各メンバに実装関数を割り当てることで機能が登録される。

入力と出力

  • **入力**: _PG_output_plugin_init 関数が引数として受け取る struct OutputPluginCallbacks *cb に対して、プラグイン側で各コールバック関数を代入する。
  • **出力**: 登録されたコールバック関数群は、論理デコーディングの進行に応じてコアエンジンから呼び出される。

前提条件と不変条件

  • プラグインは共有ライブラリとしてロードされ、_PG_output_plugin_init が実行されることで構造体が初期化される必要がある。
  • 多くのコールバックはオプションだが、論理デコーディングの実行には startup_cb や change_cb などの必須コールバックの実装が求められる。

アルゴリズムまたは振る舞い

1. **初期化**: PostgreSQLがプラグインをロードし、_PG_output_plugin_init を呼び出して構造体を埋める。 2. **実行**: 論理デコーディングの各フェーズ（トランザクション開始、データ変更、コミット等）において、コアエンジンが構造体内の対応する関数ポインタを順次実行する。 3. **データ出力**: 各コールバック内で OutputPluginPrepareWrite や OutputPluginWrite を呼び出し、変換後のデータをクライアントへ送信する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/logical.c 内の LoadOutputPlugin 関数がプラグインをロードし、コールバックを登録する。
  • **呼び出し先**: contrib/test_decoding/test_decoding.c や src/backend/replication/pgoutput/pgoutput.c の _PG_output_plugin_init 実装が、この構造体に具体的な関数を割り当てる。

関連データ構造

  • LogicalDecodingContext: コールバック実行時のコンテキストとして渡される。
  • ReorderBufferTXN, ReorderBufferChange: 変更内容の保持に使用される。

失敗モードと境界条件

  • ストリーミング対応のコールバック（stream_ で始まるもの）を実装する場合、トランザクションの整合性を保つための状態管理が必要となる。

保守担当者向けメモ

  • 本構造体は新しいデコーディング機能（2フェーズコミット等）の追加に伴い拡張される。既存プラグインを移植する際は、最新の定義を確認すること。
  • src/backend/replication/pgoutput/pgoutput.c が標準的な実装例として参照できる。
