spgadjustmembers

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgvalidate.c:322

目的

SP-GiST 演算子ファミリー（opfamily）および演算子クラス（opclass）の登録時に、各メンバー（演算子およびサポート関数）の依存関係フラグを適切に設定する。

シグネチャまたは形状

void spgadjustmembers(Oid opfamilyoid, Oid opclassoid, List *operators, List *functions)

入力と出力

  • **入力**:
  • opfamilyoid: 対象の演算子ファミリーの OID。
  • opclassoid: 対象の演算子クラスの OID。
  • operators: OpFamilyMember 構造体のリスト（演算子）。
  • functions: OpFamilyMember 構造体のリスト（サポート関数）。
  • **出力**: なし（引数として渡されたリスト内の OpFamilyMember 構造体のフィールドを直接更新する）。

前提条件と不変条件

  • 演算子メンバーは常にソフト依存（ref_is_hard = false）として扱われる。
  • 必須のサポート関数はハード依存（ref_is_hard = true）として扱われる。
  • オプションのサポート関数はソフト依存として扱われる。

アルゴリズムまたは振る舞い

1. **演算子の処理**: operators リストを走査し、すべての演算子に対して以下を設定する。

  • ref_is_hard = false
  • ref_is_family = true
  • refobjid = opfamilyoid

    • 2. **サポート関数の処理**: functions リストを走査し、関数番号（op->number）に応じて以下を切り替える。

  • **必須関数**: SPGIST_CONFIG_PROC, SPGIST_CHOOSE_PROC, SPGIST_PICKSPLIT_PROC, SPGIST_INNER_CONSISTENT_PROC, SPGIST_LEAF_CONSISTENT_PROC の場合、ref_is_hard = true を設定する。
  • **オプション関数**: SPGIST_COMPRESS_PROC, SPGIST_OPTIONS_PROC の場合、ref_is_hard = false, ref_is_family = true, refobjid = opfamilyoid を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgutils.c 内で amroutine->amadjustmembers に代入されており、インデックス作成プロセスから呼び出される。
  • **呼び出し先**: 外部関数は呼び出さないが、ereport を使用してエラーを報告する。

関連データ構造

  • OpFamilyMember: 演算子や関数を保持する構造体。
  • SPGIST_..._PROC: サポート関数を識別するための定数。

失敗モードと境界条件

  • 定義されていない関数番号が functions リストに含まれている場合、ereport(ERROR, ...) により処理が中断される。

保守担当者向けメモ

  • SP-GiST の演算子クラスは通常演算子ファミリーと 1 対 1 で対応するため、依存関係の調整は主にファミリー単位で行われる。
  • 新しいサポート関数を SP-GiST に追加する場合は、本関数の switch 文にケースを追加する必要がある。
