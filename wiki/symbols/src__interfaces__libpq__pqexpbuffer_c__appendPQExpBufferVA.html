appendPQExpBufferVA

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/pqexpbuffer.c:293

目的

PQExpBuffer に対して vsnprintf 形式のフォーマット済み文字列を追加する。バッファ容量が不足している場合は自動的に拡張を試みる。

シグネチャまたは形状

bool appendPQExpBufferVA(PQExpBuffer str, const char *fmt, va_list args)

入力と出力

  • **入力**:
  • str: 操作対象の PQExpBuffer 構造体。
  • fmt: vsnprintf 互換のフォーマット文字列。
  • args: フォーマット引数のリスト (va_list)。
  • **出力**:
  • bool: 成功時に true、メモリ不足等の理由で失敗（またはバッファが broken 状態へ遷移）した際に false を返す。ただし、内部で markPQExpBufferBroken が呼ばれた場合は true を返す仕様となっている。

前提条件と不変条件

  • str は初期化済みであること。
  • str が「broken」状態（メモリ不足等）にある場合、呼び出しは即座にリターンされる（PQExpBufferBroken マクロによる保護）。
  • vsnprintf の結果が INT_MAX - 1 を超える場合、バッファは「broken」状態となる。

アルゴリズムまたは振る舞い

1. バッファに十分な空き容量（16バイト以上）があるか確認する。 2. 十分な場合、vsnprintf を実行してデータを書き込む。

  • 書き込み成功時、str->len を更新して true を返す。
  • vsnprintf がエラーを返した場合、markPQExpBufferBroken を呼び出し true を返す。

    • 3. 空き容量が不足している場合、または vsnprintf の出力がバッファサイズを超える場合：

  • 必要なサイズを計算し、enlargePQExpBuffer を呼び出してバッファを拡張する。
  • 拡張に失敗した場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq-oauth/oauth-utils.c
  • src/interfaces/libpq/fe-misc.c
  • src/interfaces/libpq/pqexpbuffer.c (内部の printfPQExpBuffer 等から参照)
  • **呼び出し先**:
  • vsnprintf (標準ライブラリ)
  • markPQExpBufferBroken
  • enlargePQExpBuffer

関連データ構造

  • PQExpBuffer: 文字列バッファを管理する構造体。

失敗モードと境界条件

  • **メモリ不足**: enlargePQExpBuffer が失敗した場合、関数は false を返す。
  • **フォーマットエラー**: vsnprintf が負の値を返した場合、markPQExpBufferBroken が呼ばれ、バッファは無効化される。
  • **サイズ制限**: 出力文字列が INT_MAX を超える場合、バッファは無効化される。

保守担当者向けメモ

  • 本関数は vsnprintf の戻り値を利用して必要なバッファサイズを推定する C99 準拠の挙動を前提としている。
  • enlargePQExpBuffer が失敗した際の戻り値が false である点に注意すること。
