test_resowner_many

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_resowner/test_resowner_many.c:202

目的

PostgreSQLの ResourceOwner 機構のストレステストを行うためのSQL呼び出し可能な関数です。大量のリソースの登録、部分的な破棄、およびフェーズごとの一括解放プロセスを通じて、リソース管理の順序整合性と優先順位が正しく維持されるかを検証します。

シグネチャまたは形状

Datum test_resowner_many(PG_FUNCTION_ARGS)

引数として5つの int32 値（リソース種類数、BeforeLocksの登録数/破棄数、AfterLocksの登録数/破棄数）を受け取ります。

入力と出力

  • **入力**:
  • nkinds: リソース種類の数。
  • nremember_bl, nforget_bl: BeforeLocksフェーズのリソース登録数および破棄数。
  • nremember_al, nforget_al: AfterLocksフェーズのリソース登録数および破棄数。
  • **出力**: なし（PG_RETURN_VOID()）。

前提条件と不変条件

  • 各引数は0以上である必要があります。
  • 破棄数（nforget_bl, nforget_al）は、対応する登録数以下である必要があります。
  • ResourceOwnerRelease 実行後、GetTotalResourceCount が0を返すこと（すべてのリソースが解放されたこと）をアサーションで検証します。

アルゴリズムまたは振る舞い

1. **初期化**: 指定された nkinds に基づき、BeforeLocksおよびAfterLocks用のリソース定義（ManyTestResourceKind）を生成します。 2. **リソース登録**: ResourceOwnerCreate で作成したオーナーに対し、RememberManyTestResources を用いて指定数のリソースを登録します。 3. **部分破棄**: ForgetManyTestResources を呼び出し、一部のリソースを明示的に解放します。 4. **一括解放**: 以下の順序で ResourceOwnerRelease を実行し、各フェーズでリソースが解放されることを確認します。

  • RESOURCE_RELEASE_BEFORE_LOCKS
  • RESOURCE_RELEASE_LOCKS
  • RESOURCE_RELEASE_AFTER_LOCKS

    • 5. **終了**: ResourceOwnerDelete を呼び出し、オーナーを破棄します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されることを想定しています。
  • **呼び出し先**: InitManyTestResourceKind, RememberManyTestResources, ForgetManyTestResources, ResourceOwnerCreate, ResourceOwnerRelease, ResourceOwnerDelete, GetTotalResourceCount 等の内部関数およびPostgreSQLの ResourceOwner API。

関連データ構造

  • ManyTestResourceKind: リソースの解放フェーズや優先順位を保持する構造体。
  • ResourceOwner: リソース管理の基盤となるPostgreSQLのデータ構造。

失敗モードと境界条件

  • 引数が負の値、または破棄数が登録数を超える場合は elog(ERROR) を発生させます。
  • 解放順序や優先順位が期待と異なる場合、Assert により検証エラーが発生します。

保守担当者向けメモ

  • 本関数はストレステスト用です。詳細な挙動を確認する場合は、PostgreSQLの設定で client_min_messages = debug1 を有効にしてください。
  • current_release_phase および last_release_priority といったグローバル変数の状態管理が、解放順序の検証において重要な役割を果たしています。
