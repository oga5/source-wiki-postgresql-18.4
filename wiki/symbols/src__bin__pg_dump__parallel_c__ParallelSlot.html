ParallelSlot

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/parallel.c:95

目的

pg_dump および pg_restore における並列処理の各ワーカープロセス（またはスレッド）の状態、通信パイプ、および識別情報を保持する内部構造体です。

シグネチャまたは形状

struct ParallelSlot {
    T_WorkerStatus workerStatus;
    ParallelCompletionPtr callback;
    void *callback_data;
    ArchiveHandle *AH;
    int pipeRead;
    int pipeWrite;
    int pipeRevRead;
    int pipeRevWrite;
#ifdef WIN32
    uintptr_t hThread;
    unsigned int threadId;
#else
    pid_t pid;
#endif
};

入力と出力

  • **入力**: ParallelState 内の配列要素として管理され、リーダープロセスからジョブ割り当て時に callback や AH が設定されます。
  • **出力**: ワーカーの実行状態（workerStatus）や、プロセス/スレッド識別子を保持します。

前提条件と不変条件

  • workerStatus が WRKR_WORKING の場合のみ、callback および callback_data が有効です。
  • プラットフォームに応じて、Unix系では pid が、Windows環境では hThread および threadId がワーカーの生存確認に使用されます。

アルゴリズムまたは振る舞い

  • リーダープロセスは、DispatchJobForTocEntry を通じてジョブを割り当て、パイプ経由でコマンドを送信します。
  • ワーカーは RunWorker ループ内でパイプを監視し、リーダーからの指示に従って ArchiveHandle を使用した処理を実行します。
  • 処理完了後、ワーカーはパイプ経由で結果を返送し、リーダーが ListenToWorkers でこれを回収します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/parallel.c 内の GetMyPSlot、RunWorker、set_cancel_slot_archive 等から参照されます。また、src/bin/pg_amcheck/pg_amcheck.c においても run_command 等で利用されています。
  • **呼び出し先**: 構造体内部に ArchiveHandle へのポインタを保持し、ワーカー処理の実行時に利用します。

関連データ構造

  • ParallelState: 本構造体を配列として保持する管理構造体。
  • T_WorkerStatus: ワーカーの状態（アイドル/実行中/終了）を定義する列挙型。
  • ArchiveHandle: ダンプ/リストア処理のコンテキストを保持する構造体。

失敗モードと境界条件

  • Windows環境では fork が利用できないため、スレッドとソケットを用いたパイプエミュレーション（pgpipe）に依存します。
  • シグナルハンドラ内でのアクセスは制限されており、安全な関数のみを使用する必要があります。

保守担当者向けメモ

  • プラットフォーム依存の差異（#ifdef WIN32）がプロセス/スレッド管理に含まれているため、OS固有の挙動変更時には注意が必要です。
  • デッドロック回避のため、ワーカーがテーブルロックを取得する際は NOWAIT が使用される設計となっています。
