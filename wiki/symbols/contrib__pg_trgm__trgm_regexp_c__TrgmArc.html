TrgmArc

定義

  • 種別: struct
  • 定義箇所: contrib/pg_trgm/trgm_regexp.c:349

目的

正規表現をトリグラムベースのグラフに変換する過程において、状態遷移を表現するためのエッジ（弧）を定義する。

シグネチャまたは形状

typedef struct
{
	ColorTrgm	ctrgm;			/* trigram needed to traverse arc */
	TrgmState  *target;			/* next state */
} TrgmArc;

入力と出力

  • **入力**: 遷移に必要なトリグラム情報 (ColorTrgm) および遷移先の状態 (TrgmState) を保持する。
  • **出力**: グラフ走査アルゴリズムにおいて、現在の状態から次の状態へ遷移可能かを判定するためのデータとして利用される。

前提条件と不変条件

  • TrgmState 構造体内のリスト（outgoing arcs）の一部として管理される。
  • 遷移には ctrgm で指定されたトリグラムの存在が必要である。

アルゴリズムまたは振る舞い

正規表現から生成された NFA をトリグラムの論理式へ変換する際、状態間の遷移条件を保持する。インデックススキャン時のグラフ走査において、特定のトリグラムが利用可能である場合に、この構造体を通じて次の状態へ遷移する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_trgm/trgm_regexp.c 内のグラフ構築および走査処理（palloc によるメモリ確保や lfirst によるリスト走査）。
  • **呼び出し先**:
  • 遷移先の TrgmState を参照する。

関連データ構造

  • TrgmState: 遷移元の状態および遷移先の状態として参照される。
  • ColorTrgm: 遷移の条件となるトリグラム情報。
  • TrgmNFA: 本構造体を含むグラフ構築の過程で利用される作業用構造体。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性がある。
  • グラフ構築時に MAX_TRGM_COUNT 等の制限を超えた場合、解析が中断される設計となっている。

保守担当者向けメモ

  • 本構造体は正規表現のインデックス最適化におけるグラフの「エッジ」を担う。
  • グラフ走査の効率は、この構造体が保持する ctrgm の選択と最適化に依存する。
  • 構造体の変更は、trigramsMatchGraph によるマッチング判定ロジックに直接影響を与えるため注意が必要。
