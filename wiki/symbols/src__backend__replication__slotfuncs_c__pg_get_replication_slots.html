pg_get_replication_slots

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slotfuncs.c:235

目的

システム内の全レプリケーションスロットの情報を集約し、SQLインターフェース（pg_replication_slots ビュー）を通じて提供するセット返却関数（SRF）です。

シグネチャまたは形状

Datum pg_get_replication_slots(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出しの引数）。
  • **出力**: Datum（pg_replication_slots ビューの各列に対応するタプルのセット）。

前提条件と不変条件

  • ReplicationSlotControlLock を共有ロックで保持し、各スロットの mutex を個別に取得してスナップショットをコピーすることで、システム全体を長時間ブロックせずに整合性を確保します。
  • スロット名などの情報は機密性が低いと見なされるため、特別な権限チェックは行われません。

アルゴリズムまたは振る舞い

1. InitMaterializedSRF を呼び出し、結果セットの初期化を行います。 2. GetXLogWriteRecPtr() で現在のWAL書き込み位置を取得します。 3. ReplicationSlotControlLock を共有モードで取得し、全スロットを反復走査します。 4. 使用中のスロット（in_use）について、mutex を用いてスロット内容を一時変数にコピーします。 5. コピーしたデータに基づき、スロットの種類（物理/論理）、永続性、アクティブなPID、LSN位置（restart_lsn, confirmed_flush）、WALの可用性状態などを算出します。 6. WALの可用性（walstate）に応じて、safe_wal_size（WALが削除されるまでの残りバイト数）を計算します。 7. 構築した値を tuplestore に格納し、最終的に呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（pg_replication_slots ビューのクエリ実行時）。
  • **呼び出し先**:
  • GetXLogWriteRecPtr()
  • LWLockAcquire / LWLockRelease
  • SpinLockAcquire / SpinLockRelease
  • GetWALAvailability()
  • XLByteToSeg 等のWAL操作関数

関連データ構造

  • ReplicationSlot: スロットの内部状態を保持する構造体。
  • ReplicationSlotCtl: 全スロットを管理する共有メモリ上の制御構造体。
  • ReturnSetInfo: SRFの結果セット情報を保持する構造体。

失敗モードと境界条件

  • スロットが削除済みまたは無効化されている場合、walstate は WALAVAIL_REMOVED または lost と判定されます。
  • max_slot_wal_keep_size_mb が負の値である場合、safe_wal_size は計算されず null となります。
  • restart_lsn が無効な場合、関連するLSNフィールドは null として出力されます。

保守担当者向けメモ

  • 本関数は ReplicationSlotControlLock を保持するため、スロット数が多い環境でもパフォーマンスへの影響を最小限に抑える設計になっています。
  • walstate の判定ロジックには、プロセスが生存している場合に「lost」ではなく「unreserved」と見なす等のヒューリスティックが含まれています。
