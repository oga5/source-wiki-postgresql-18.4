BufferManagerRelation

定義

  • 種別: struct
  • 定義箇所: src/include/storage/bufmgr.h:101

目的

リレーションの拡張操作（ExtendBufferedRel 等）において、対象となるリレーションの情報を抽象化して保持するための構造体です。

シグネチャまたは形状

typedef struct BufferManagerRelation
{
	Relation	rel;
	struct SMgrRelationData *smgr;
	char		relpersistence;
} BufferManagerRelation;

入力と出力

本構造体は、以下のマクロを通じて初期化されます。

  • BMR_REL(p_rel): Relation オブジェクトから生成。
  • BMR_SMGR(p_smgr, p_relpersistence): SMgrRelationData と永続性フラグから生成。

前提条件と不変条件

  • rel または smgr のいずれかが有効である必要があります。
  • relpersistence はリレーションの永続性（一時テーブルか否か等）を保持します。

アルゴリズムまたは振る舞い

バッファマネージャの拡張関数（ExtendBufferedRel 系）において、リレーションの物理的な拡張処理を行う際の引数として使用されます。これにより、上位層からのリレーション指定と、ストレージマネージャレベルでの直接指定の両方を単一のインターフェースで扱います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c: ExtendBufferedRelCommon, ExtendBufferedRelShared, ExtendBufferedRel, ExtendBufferedRelBy, ExtendBufferedRelTo
  • src/backend/storage/buffer/localbuf.c: ExtendBufferedRelLocal
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • Relation: リレーションキャッシュの定義。
  • SMgrRelationData: ストレージマネージャのリレーションデータ。

失敗モードと境界条件

  • rel と smgr が共に NULL の状態で拡張関数に渡された場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は bufmgr.h 内の BMR_REL および BMR_SMGR マクロによって生成されることが想定されています。直接インスタンス化する際は、これらのマクロの利用を検討してください。
