r_remove_question_prefixes

目的

タミル語のステミング処理において、単語の先頭に付与された特定の疑問形接頭辞を識別し、除去する。

シグネチャまたは形状

static int r_remove_question_prefixes(struct SN_env * z)

入力と出力

  • **入力**: z (Snowball ステミング環境を保持する SN_env 構造体へのポインタ)
  • **出力**: int 型。成功した場合は 1、マッチしなかった場合や処理が適用されなかった場合は 0、エラー発生時は負の値を返す。

前提条件と不変条件

  • z が有効な SN_env 構造体を指していること。
  • 内部で s_4 および s_5 という文字列定数、および a_1 という among 構造体の配列が定義されていること。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置 z->c を開始点 z->bra に設定する。 2. s_4 と一致するか確認し、一致しなければ終了する。 3. a_1 配列内のパターンを find_among で検索し、一致しなければ終了する。 4. 続いて s_5 と一致するか確認する。 5. 一致した場合、その範囲を終了点 z->ket とし、slice_del を呼び出して該当箇所を削除する。 6. 削除後、現在のカーソル位置を保持したまま r_fix_va_start を呼び出し、語幹の整合性を調整する。 7. 成功時に 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/libstemmer/stem_UTF_8_tamil.c 内のステミング処理フローから呼び出される。
  • **呼び出し先**: find_among (Snowball ランタイム), slice_del (Snowball ランタイム), r_fix_va_start (内部関数)。

関連データ構造

  • SN_env: ステミング環境を管理する構造体。
  • among: 接辞パターンを定義する構造体配列（a_1）。

失敗モードと境界条件

  • eq_s や find_among がマッチに失敗した場合は 0 を返し、文字列の変更は行われない。
  • slice_del が内部エラー（負の値）を返した場合、その値をそのまま呼び出し元に伝播させる。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによって自動生成されたコードである。ロジックの変更が必要な場合は、直接編集せず、対応する .sbl ファイルを修正すること。
