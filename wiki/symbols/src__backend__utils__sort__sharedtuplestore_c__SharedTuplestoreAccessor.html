SharedTuplestoreAccessor

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/sort/sharedtuplestore.c:71

目的

並列クエリ実行時、各バックエンドプロセスが共有タプルストアに対して読み書きを行うためのローカルな操作用構造体です。共有メモリ上の SharedTuplestore 状態と、各プロセスが担当する一時ファイル（SharedFileSet）を紐付け、バッファリングやスキャン状態を管理します。

シグネチャまたは形状

struct SharedTuplestoreAccessor {
    int participant;
    SharedTuplestore *sts;
    SharedFileSet *fileset;
    MemoryContext context;
    /* 読み取り用状態 */
    int read_participant;
    BufFile *read_file;
    int read_ntuples_available;
    int read_ntuples;
    size_t read_bytes;
    char *read_buffer;
    size_t read_buffer_size;
    BlockNumber read_next_page;
    /* 書き込み用状態 */
    SharedTuplestoreChunk *write_chunk;
    BufFile *write_file;
    BlockNumber write_page;
    char *write_pointer;
    char *write_end;
};

入力と出力

  • **入力**: 初期化時に SharedTuplestore へのポインタおよび SharedFileSet を保持します。
  • **出力**: BufFile を介して一時ファイルへの読み書きを行い、並列スキャン時にはタプルを呼び出し元へ提供します。

前提条件と不変条件

  • sts フィールドは、共有メモリ上に確保された有効な SharedTuplestore を指している必要があります。
  • 書き込み操作は、各プロセスに割り当てられた専用の write_file に対して行われるため、プロセス間での排他制御は不要です。
  • 読み取り操作（並列スキャン）時は、SharedTuplestore が管理するロックを使用して読み取り位置を同期する必要があります。

アルゴリズムまたは振る舞い

  • **書き込み**: sts_puttuple を通じて write_chunk にデータを蓄積し、チャンクが満杯になると sts_flush_chunk でファイルへ書き出します。
  • **読み取り**: sts_parallel_scan_next を通じて、全参加者のファイルを順次スキャンします。read_buffer を使用してチャンク単位でロードし、タプルを逐次的に抽出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHash.c および nodeHashjoin.c で使用されています。また、sts_filename、sts_flush_chunk、sts_end_write などの sharedtuplestore.c 内の関数で引数として参照されます。
  • **呼び出し先**: BufFile API（storage/buffile.h）を呼び出し、一時ファイルの物理的な読み書きを実行します。

関連データ構造

  • SharedTuplestore: 共有メモリ上の制御構造体。
  • SharedFileSet: 一時ファイルを管理する共有ファイルセット。
  • SharedTuplestoreChunk: 書き込み時のバッファ単位。
  • BufFile: 一時ファイルの抽象化レイヤー。

失敗モードと境界条件

  • **巨大タプル**: チャンクサイズを超えるタプルは、オーバーフローチャンクとして分割されます。読み取り側は read_buffer を用いてこれらを適切に再構成する必要があります。
  • **メモリ不足**: MemoryContext によるメモリ管理に依存しており、確保失敗時はエラーが発生します。

保守担当者向けメモ

  • 本構造体は各バックエンドのローカルメモリ上に配置されます。共有メモリ上の SharedTuplestore との同期状態（特に read_participant や read_next_page）の整合性に注意してください。
  • 書き込み時の write_pointer と write_end の更新は、sts_flush_chunk のロジックと密接に関連しています。
