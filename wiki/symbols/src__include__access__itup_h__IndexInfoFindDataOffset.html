IndexInfoFindDataOffset

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/itup.h:112

目的

インデックスタプル（IndexTuple）のヘッダー直後に続く、実際のデータ領域が開始するオフセットを算出する。

シグネチャまたは形状

static inline Size IndexInfoFindDataOffset(unsigned short t_info)

入力と出力

  • **入力**: t_info (インデックスタプルの制御情報ビットマスク)
  • **出力**: Size (データ領域開始位置までのバイト数)

前提条件と不変条件

  • t_info には INDEX_NULL_MASK が含まれているかどうかの情報が格納されている必要がある。
  • 戻り値は常に MAXALIGN によってアライメント調整された値となる。

アルゴリズムまたは振る舞い

1. t_info に対して INDEX_NULL_MASK ビットの論理積をとる。 2. ビットが立っていない（NULL値を含まない）場合、IndexTupleData のサイズをアライメント調整して返す。 3. ビットが立っている（NULL値を含む）場合、IndexTupleData と IndexAttributeBitMapData の合計サイズをアライメント調整して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pageinspect/btreefuncs.c
  • src/backend/access/common/indextuple.c
  • src/backend/access/hash/hashutil.c
  • src/include/access/ginblock.h
  • src/include/access/itup.h
  • **呼び出し先**:
  • MAXALIGN マクロ（直接的な依存関係）

関連データ構造

  • IndexTupleData: タプルの基本ヘッダー。
  • IndexAttributeBitMapData: NULL値を管理するためのビットマップ構造。

失敗モードと境界条件

  • 入力される t_info が不正な場合、誤ったオフセットを返す可能性がある。
  • MAXALIGN の境界条件により、アーキテクチャごとのアライメント要件に依存する。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、インライン関数として定義されている。
  • IndexTuple の物理レイアウト（ヘッダー → NULLビットマップ → データ）の変更時には、本関数の計算ロジックも必ず更新すること。
