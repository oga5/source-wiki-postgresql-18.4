heap_getsysattr

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/heaptuple.c:724

目的

ヒープタプルからシステム属性（ctid, xmin, xmax, cmin/cmax, tableoid）の値を抽出する。

シグネチャまたは形状

Datum heap_getsysattr(HeapTuple tup, int attnum, TupleDesc tupleDesc, bool *isnull)

入力と出力

  • **入力**:
  • tup: 対象の HeapTuple 構造体へのポインタ。
  • attnum: 取得対象のシステム属性番号。
  • tupleDesc: タプル記述子（本関数内では未使用）。
  • isnull: NULL判定結果を格納するブール値へのポインタ。
  • **出力**:
  • 戻り値: 属性値を表す Datum。
  • isnull: 常に false が設定される（システム属性はNULLにならないため）。

前提条件と不変条件

  • tup は非NULLである必要がある（Assert(tup) による検証）。
  • 指定された attnum が有効なシステム属性番号であること。無効な場合は elog(ERROR) が発生する。

アルゴリズムまたは振る舞い

1. isnull を false に初期化する。 2. attnum に基づき switch 文で分岐する。

  • SelfItemPointerAttributeNumber: tup->t_self を返す。
  • MinTransactionIdAttributeNumber: tup->t_data から xmin を抽出して返す。
  • MaxTransactionIdAttributeNumber: tup->t_data から xmax を抽出して返す。
  • MinCommandIdAttributeNumber / MaxCommandIdAttributeNumber: tup->t_data からコマンドIDを抽出して返す。
  • TableOidAttributeNumber: tup->t_tableOid を返す。

    • 3. 該当しない attnum の場合はエラーを投げる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execTuples.c
  • src/backend/utils/adt/expandedrecord.c
  • src/include/access/htup_details.h（インライン関数経由）
  • **呼び出し先**:
  • HeapTupleHeaderGetRawXmin, HeapTupleHeaderGetRawXmax, HeapTupleHeaderGetRawCommandId などのヘッダーアクセス用マクロ。
  • PointerGetDatum, TransactionIdGetDatum, CommandIdGetDatum, ObjectIdGetDatum などの型変換マクロ。

関連データ構造

  • HeapTuple: タプル本体。
  • TupleDesc: タプル記述子（本関数では引数として受け取るが、ロジック内では使用されない）。

失敗モードと境界条件

  • attnum が定義済みのシステム属性番号以外の場合、elog(ERROR) により処理が中断される。
  • tup がNULLの場合、アサーション違反によりプロセスが異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は tupleDesc を引数に取るが、現在の実装では使用していない。将来的な拡張やインターフェースの一貫性のために保持されている。
  • cmin と cmax は同一フィールドを共有しているため、どちらの属性番号を指定しても同じ値が返される。
