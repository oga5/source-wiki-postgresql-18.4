GetPageWithFreeSpace

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/freespace.c:136

目的

指定されたリレーションにおいて、要求された空き容量を満たすページを Free Space Map (FSM) から検索し、そのブロック番号を返します。

シグネチャまたは形状

BlockNumber GetPageWithFreeSpace(Relation rel, Size spaceNeeded)

入力と出力

  • **入力**:
  • rel: 検索対象のリレーション。
  • spaceNeeded: 必要な空き容量（バイト単位）。
  • **出力**:
  • 条件を満たすページの BlockNumber。該当するページが見つからない場合は InvalidBlockNumber が返されます。

前提条件と不変条件

  • FSM は「ヒント」として機能するため、返されたページが必ずしも要求された空き容量を保持しているとは限りません。呼び出し側は、実際にページを読み込んだ際に再度空き容量を確認する必要があります。
  • 空き容量は 256 段階のカテゴリに量子化されて管理されます。

アルゴリズムまたは振る舞い

1. fsm_space_needed_to_cat を呼び出し、spaceNeeded を FSM が管理する 256 段階のカテゴリ（min_cat）に変換します。 2. fsm_search を呼び出し、FSM の木構造をルートからリーフまで探索して、min_cat 以上の空き容量を持つページを特定します。 3. 特定されたページのブロック番号を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_pageops.c
  • src/backend/access/heap/hio.c
  • src/backend/storage/freespace/indexfsm.c
  • **呼び出し先**:
  • fsm_space_needed_to_cat (内部変換関数)
  • fsm_search (木構造探索関数)

関連データ構造

  • Relation: リレーション記述子。
  • BlockNumber: ページを特定する物理ブロック番号。
  • FSMAddress: FSM 内での論理的な位置を示す構造体（fsm_search 内部で使用）。

失敗モードと境界条件

  • リレーション内に十分な空き容量を持つページが存在しない場合、fsm_search は InvalidBlockNumber を返します。
  • spaceNeeded が極端に大きい場合、カテゴリ変換の結果として検索が失敗する可能性があります。

保守担当者向けメモ

  • 本関数は WAL ログを出力しません。これはパフォーマンス最適化のためであり、クラッシュリカバリ時には物理的なページ状態が正として扱われます。
  • 検索結果の不整合は、fsm_search がリーフノードまで降りて再確認する仕組みによって許容されています。
