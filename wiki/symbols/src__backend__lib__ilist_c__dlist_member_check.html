dlist_member_check

定義

  • 種別: function
  • 定義箇所: src/backend/lib/ilist.c:59

目的

指定された二重連結リスト（dlist_head）内に、特定のノード（dlist_node）が含まれているかを検証する。

シグネチャまたは形状

void dlist_member_check(const dlist_head *head, const dlist_node *node)

入力と出力

  • **入力**:
  • head: 走査対象となる二重連結リストの先頭ポインタ。
  • node: リスト内に存在するかを確認する対象のノードポインタ。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • head および node は有効なメモリ領域を指している必要がある。
  • リスト構造が破損していないことを前提とする。

アルゴリズムまたは振る舞い

1. リストの先頭（head->head.next）から順方向に末尾（&head->head）まで走査を行う。 2. 走査中の各ノードが引数の node と一致するかを比較する。 3. 一致するノードが見つかった場合は直ちに終了する。 4. リストの末尾に到達しても一致するノードが見つからなかった場合、elog(ERROR, ...) を呼び出し、エラーを発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/lib/ilist.h 内の複数のインライン関数（dlist_insert_after, dlist_insert_before, dlist_delete, dlist_move_node 等）から、デバッグ目的で呼び出されている。
  • **呼び出し先**: elog（PostgreSQLのエラー報告関数）。

関連データ構造

  • dlist_head: 二重連結リストの管理構造体。
  • dlist_node: 二重連結リストの要素構造体。

失敗モードと境界条件

  • **失敗モード**: 指定された node が head のリスト内に存在しない場合、ERROR レベルのログを出力して処理を中断する。
  • **境界条件**: 空のリストに対して呼び出した場合、ループは実行されず、直ちにエラーとなる。

保守担当者向けメモ

  • 本関数は const 修飾されたポインタを扱うため、イテレータマクロを使用せず、手動でループを記述している。
  • ILIST_DEBUG が有効なビルド環境において、リスト操作の整合性保証のために使用される。
