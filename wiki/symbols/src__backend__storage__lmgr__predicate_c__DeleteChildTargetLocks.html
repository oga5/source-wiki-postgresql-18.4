DeleteChildTargetLocks

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/predicate.c:2213

目的

指定されたターゲットタグ（newtargettag）によって包含される既存の述語ロックを特定し、それらを削除することで、より広範なロックへの昇格や冗長なロックの整理を行う。

シグネチャまたは形状

static void DeleteChildTargetLocks(const PREDICATELOCKTARGETTAG *newtargettag)

入力と出力

  • **入力**: newtargettag (削除対象を包含する親ターゲットのタグ)
  • **出力**: なし（副作用として共有メモリ上のハッシュテーブルおよびリストを更新）

前提条件と不変条件

  • 呼び出し元は、SerializablePredicateListLock を保持していない状態で呼び出す必要がある（関数内で取得するため）。
  • MySerializableXact が有効であること。
  • TargetTagIsCoveredBy が真となるロックのみが削除対象となる。

アルゴリズムまたは振る舞い

1. SerializablePredicateListLock を共有モードで取得する。 2. 並列モード（IsInParallelMode）であれば、現在のトランザクションの perXactPredicateListLock を排他モードで取得する。 3. 現在のトランザクションが保持する述語ロックリストを走査する。 4. 各ロックのターゲットが newtargettag に包含されているか判定する。 5. 包含されている場合、以下の処理を行う：

  • 該当するロックのパーティションロックを排他モードで取得する。
  • ロックをトランザクションおよびターゲットのリストから削除する。
  • PredicateLockHash からエントリを削除する。
  • RemoveTargetIfNoLongerUsed を呼び出し、ターゲットが不要であれば削除する。
  • パーティションロックを解放する。
  • DecrementParentLocks を呼び出し、親ロックの参照カウントを減らす。

    • 6. 取得したロックを順次解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/lmgr/predicate.c 内の複数箇所（ロック昇格時など）。
  • **呼び出し先**: TargetTagIsCoveredBy, PredicateLockTargetTagHashCode, PredicateLockHashPartitionLock, dlist_delete, hash_search_with_hash_value, RemoveTargetIfNoLongerUsed, DecrementParentLocks, IsInParallelMode。

関連データ構造

  • PREDICATELOCKTARGETTAG: 述語ロック対象の識別子。
  • SERIALIZABLEXACT: 直列化可能トランザクションの状態管理構造体。
  • PREDICATELOCK: 述語ロックの実体。
  • PredicateLockHash: 述語ロックを格納する共有メモリ上のハッシュテーブル。

失敗モードと境界条件

  • 並列モード時には perXactPredicateListLock の取得が必要であり、デッドロックを避けるためのロック順序が重要となる。
  • hash_search_with_hash_value での削除対象が見つからない場合、Assert により停止する可能性がある。

保守担当者向けメモ

  • 本関数はロックの粒度を粗くする（昇格させる）際に呼び出される。
  • dlist_foreach_modify を使用してリスト走査中の安全な削除を行っている。
  • 並列クエリ実行時のロック保護（perXactPredicateListLock）が適切に行われているか注意すること。
