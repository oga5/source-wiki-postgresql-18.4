setup_cancel_handler

定義

  • 種別: function
  • 定義箇所: src/fe_utils/cancel.c:182

目的

PostgreSQLフロントエンドアプリケーションにおいて、ユーザーからの割り込み（SIGINT）を捕捉し、クエリキャンセル処理をトリガーするためのシグナルハンドラを登録します。

シグネチャまたは形状

void setup_cancel_handler(void (*query_cancel_callback) (void))

入力と出力

  • **入力**: query_cancel_callback - 割り込み発生時に実行されるコールバック関数へのポインタ（NULLも指定可能）。
  • **出力**: なし。

前提条件と不変条件

  • シグナルハンドラ内で安全にメッセージを出力するため、本関数内でローカライズ済みメッセージ（cancel_sent_msg, cancel_not_sent_msg）のキャッシュが行われます。
  • pqsignal を使用して SIGINT に対して handle_sigint を登録します。

アルゴリズムまたは振る舞い

1. 引数で渡された query_cancel_callback をモジュール内の静的変数 cancel_callback に保持します。 2. _() マクロを使用して、キャンセル成功時および失敗時のメッセージを現在のロケールで取得し、静的変数にキャッシュします。 3. pqsignal を呼び出し、SIGINT シグナルが発生した際に handle_sigint が実行されるよう設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/pgbench/pgbench.c
  • src/bin/psql/common.c
  • src/bin/scripts/clusterdb.c
  • src/bin/scripts/reindexdb.c
  • src/bin/scripts/vacuumdb.c
  • **呼び出し先**:
  • pqsignal (シグナルハンドラ登録)
  • handle_sigint (シグナル発生時に実行される関数)

関連データ構造

  • cancel_callback: 割り込み時に実行されるコールバック関数を保持する静的変数。
  • cancel_sent_msg, cancel_not_sent_msg: シグナルハンドラ内で使用されるローカライズ済みメッセージ文字列。

失敗モードと境界条件

  • シグナルハンドラ内では gettext() などの複雑な関数が呼び出せないため、本関数で事前にメッセージをキャッシュする設計となっています。
  • Windows環境では SIGINT の代わりに SetConsoleCtrlHandler が使用されるため、本関数の動作はプラットフォームに依存します。

保守担当者向けメモ

  • CancelRequested フラグは自動的にリセットされないため、アプリケーション側でキャンセル処理後の状態に応じて適切にリセットする必要があります。
  • シグナル安全性を維持するため、ハンドラ内での処理は最小限に留めてください。
