g_int_union

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_gist.c:122

目的

GiST インデックス構築時、複数のインデックスエントリを統合して単一の親ノードキーを作成する。

シグネチャまたは形状

Datum g_int_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GistEntryVector *（統合対象のインデックスエントリ群）
  • PG_GETARG_POINTER(1): int *（出力される統合後のデータサイズを格納するポインタ）
  • **出力**:
  • 戻り値: Datum（統合・ソート・ユニーク化された ArrayType）
  • 副作用: *size に統合後の ArrayType のバイトサイズが書き込まれる。

前提条件と不変条件

  • 入力される各エントリは ArrayType として有効であること（CHECKARRVALID マクロによる検証）。
  • 統合後の配列はソートおよび重複排除が行われ、正規化された状態であること。

アルゴリズムまたは振る舞い

1. entryvec 内の全エントリを走査し、合計要素数を算出する。 2. 合計要素数分のメモリを確保し、各エントリの配列データを新しいメモリ領域へコピーする。 3. QSORT を用いて配列をソートする。 4. _int_unique を呼び出し、重複する整数を除去して配列を圧縮する。 5. 最終的な配列のサイズを *size に格納し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: GiST インデックスの構築プロセス（PostgreSQL の GiST インターフェース経由）。
  • **呼び出し先**: new_intArrayType, ARRNELEMS, ARRPTR, GETENTRY, CHECKARRVALID, memcpy, QSORT, _int_unique, VARSIZE。
  • 注: 具体的な呼び出し元関数名はソースコードの文脈からは特定されていません。

関連データ構造

  • GistEntryVector: GiST インデックスのページ分割や統合に使用されるエントリの集合。
  • ArrayType: PostgreSQL の配列型表現。

失敗モードと境界条件

  • 入力エントリが空の場合、またはメモリ不足が発生した場合の挙動は、基盤となる new_intArrayType やメモリ管理ルーチンに依存します。

保守担当者向けメモ

  • 本関数は gist__int_ops インデックス演算子クラスの一部です。
  • _int_unique による重複排除が必須であるため、この関数を変更する際はソート順序の整合性に注意してください。
