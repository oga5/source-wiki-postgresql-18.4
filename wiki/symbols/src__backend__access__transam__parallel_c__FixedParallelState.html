FixedParallelState

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/parallel.c:82

目的

並列クエリ実行において、リーダープロセスからワーカープロセスへ継承すべき固定サイズの共有状態を保持する。

シグネチャまたは形状

typedef struct FixedParallelState
{
    Oid            database_id;
    Oid            authenticated_user_id;
    Oid            session_user_id;
    Oid            outer_user_id;
    Oid            current_user_id;
    Oid            temp_namespace_id;
    Oid            temp_toast_namespace_id;
    int            sec_context;
    bool           session_user_is_superuser;
    bool           role_is_superuser;
    PGPROC        *parallel_leader_pgproc;
    pid_t          parallel_leader_pid;
    ProcNumber     parallel_leader_proc_number;
    TimestampTz    xact_ts;
    TimestampTz    stmt_ts;
    SerializableXactHandle serializable_xact_handle;
    slock_t        mutex;
    XLogRecPtr     last_xlog_end;
} FixedParallelState;

入力と出力

  • **入力**: リーダープロセスが InitializeParallelDSM 内で本構造体を共有メモリ上に確保し、値を設定する。
  • **出力**: ワーカープロセスが起動時に共有メモリから本構造体を読み取り、自身の実行環境をリーダーと同期させる。

前提条件と不変条件

  • mutex フィールドは、構造体内の他のフィールド（特に last_xlog_end）へのアクセスを保護するために使用される。
  • ワーカーは、リーダーから継承したこれらの状態を自身のメモリ空間に復元する必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: リーダーが shm_toc_allocate を用いて共有メモリ上に領域を確保し、現在のトランザクションやセッション情報を書き込む。 2. **同期**: ワーカーは起動後、共有メモリ上の本構造体を参照し、自身のセッション状態をリーダーと一致させる。 3. **更新**: ワーカーは実行中に last_xlog_end を更新する場合があり、その際は mutex による排他制御が行われる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/parallel.c 内の InitializeParallelDSM（初期化時）および各並列処理関数。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すものではない。
  • **注記**: MyFixedParallelState という静的変数を通じて、バックエンドローカルなコピーが保持される場合がある。

関連データ構造

  • ParallelContext: 並列実行のライフサイクルを管理する構造体。
  • PGPROC: リーダープロセスの識別に使用。
  • SerializableXactHandle: 直列化可能トランザクションのハンドル。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、並列クエリの初期化自体が失敗する。
  • mutex の競合により、ワーカーが last_xlog_end を更新する際に待機が発生する可能性がある。

保守担当者向けメモ

  • 本構造体は、ワーカーがリーダーの環境を再現するために不可欠な情報を網羅している。新しいセッション属性やトランザクション状態を並列化対象に追加する場合は、本構造体へのフィールド追加と、InitializeParallelDSM でのシリアライズ処理の更新が必要となる。
