oauth_state

定義

  • 種別: enum
  • 定義箇所: src/backend/libpq/auth-oauth.c:57

目的

SASL OAUTHBEARER 認証プロセスにおけるセッションの状態を管理するために使用されます。

シグネチャまたは形状

enum oauth_state
{
    OAUTH_STATE_INIT = 0,
    OAUTH_STATE_ERROR,
    OAUTH_STATE_FINISHED,
};

入力と出力

  • 入力: 状態遷移のトリガーとなる認証メッセージの処理結果。
  • 出力: 現在の認証フェーズを示す列挙値。

前提条件と不変条件

  • RFC 7628 に準拠したステートマシンの一部として機能します。
  • 認証失敗時の再試行フローをサポートするため、OAUTH_STATE_ERROR 状態が定義されています。

アルゴリズムまたは振る舞い

  • OAUTH_STATE_INIT: 認証プロセスの開始状態。
  • OAUTH_STATE_ERROR: 認証失敗時、エラー応答を生成した後の状態。
  • OAUTH_STATE_FINISHED: 認証プロセスが完了した状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/libpq/auth-oauth.c:67 にて、セッションごとの状態変数として宣言されています。
  • 呼び出し先: 認証ロジック（oauth_exchange 等）内で、現在の認証フェーズを判断するために参照されます。

関連データ構造

  • oauth_ctx: セッションごとの状態を保持する構造体であり、この oauth_state をメンバとして保持します。

失敗モードと境界条件

  • 認証失敗時に OAUTH_STATE_ERROR へ遷移し、RFC 7628 Sec 3.2.3 に基づく再試行フローを制御します。

保守担当者向けメモ

  • 状態遷移のロジックは oauth_exchange() 内で実装されています。新規の状態を追加する場合は、oauth_exchange におけるスイッチ文の網羅性を確認してください。
