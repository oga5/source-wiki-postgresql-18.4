shared_dependency_comparator

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_shdepend.c:609

目的

ShDependObjectInfo 構造体の配列をソートするための比較関数。共有依存関係のレコードを特定の順序で整列させるために使用される。

シグネチャまたは形状

static int shared_dependency_comparator(const void *a, const void *b)

入力と出力

  • 入力: const void *a, const void *b （実際には ShDependObjectInfo へのポインタとしてキャストされる）
  • 出力: int 型。a < b なら負、a > b なら正、等しければ 0 を返す。

前提条件と不変条件

  • 入力ポインタは ShDependObjectInfo 構造体を指している必要がある。
  • object.objectId、object.classId、object.objectSubId、deptype の順で比較が行われる。

アルゴリズムまたは振る舞い

以下の優先順位で昇順ソートを行う： 1. **OID (objectId)**: 最優先のソートキー。 2. **カタログ ID (classId)**: OID が同一の場合の識別用。 3. **サブ ID (objectSubId)**: unsigned int として比較。0（オブジェクト全体）が先頭に来る。 4. **依存タイプ (deptype)**: 最後に比較される。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/pg_shdepend.c:811 （qsort の引数として使用されていることが示唆される）。
  • 呼び出し先: 該当なし（比較ロジックのみを実行する純粋な関数）。

関連データ構造

  • ShDependObjectInfo: 比較対象となる共有依存関係情報を保持する構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合の挙動は定義されていない。
  • objectSubId は符号なし整数として扱われるため、負の値が渡された場合のソート順序に注意が必要。

保守担当者向けメモ

  • 本関数は qsort 等の標準ライブラリ関数から呼び出されることを想定している。
  • ソート順序を変更する場合は、pg_shdepend カタログの整合性や、この関数を利用している他の処理（重複排除や検索）への影響を考慮すること。
