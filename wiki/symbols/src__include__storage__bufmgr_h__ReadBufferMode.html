ReadBufferMode

定義

  • 種別: enum
  • 定義箇所: src/include/storage/bufmgr.h:44

目的

バッファマネージャがページを読み込む際、または新規作成する際の振る舞いを制御するためのモードを指定します。

シグネチャまたは形状

typedef enum {
    RBM_NORMAL,
    RBM_ZERO_AND_LOCK,
    RBM_ZERO_AND_CLEANUP_LOCK,
    RBM_ZERO_ON_ERROR,
    RBM_NORMAL_NO_LOG
} ReadBufferMode;

入力と出力

  • 入力: ReadBuffer 等のバッファ管理関数へ引数として渡されます。
  • 出力: 指定されたモードに基づき、バッファの初期化状態やロック状態が決定されます。

前提条件と不変条件

  • RBM_ZERO_AND_LOCK および RBM_ZERO_AND_CLEANUP_LOCK は、ディスクからの読み込みを行わず、呼び出し元がページを初期化することを前提とします。
  • RBM_NORMAL_NO_LOG は、WALリプレイ中のページ無効化ログ出力を抑制する点を除き、RBM_NORMAL と同等の振る舞いをします。

アルゴリズムまたは振る舞い

  • RBM_NORMAL: 通常の読み込み。
  • RBM_ZERO_AND_LOCK: ディスク読み込みをスキップし、ページをゼロクリアしてロックを取得します。
  • RBM_ZERO_AND_CLEANUP_LOCK: RBM_ZERO_AND_LOCK と同様ですが、ロックモードが「クリーンアップ」モードになります。
  • RBM_ZERO_ON_ERROR: 読み込みを試行し、エラー発生時は全ビットがゼロのページを返します。
  • RBM_NORMAL_NO_LOG: WALリプレイ時にページを無効としてログ記録しません。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）: src/backend/access/heap/hio.c、src/backend/access/transam/xlogutils.c、src/backend/storage/buffer/bufmgr.c 等の各関数。
  • 呼び出し先: 本シンボル自体は列挙型であり、関数ではないため呼び出し先はありません。

関連データ構造

  • Buffer: バッファを識別する整数型。
  • BufferAccessStrategy: バッファ置換戦略を定義する構造体。

失敗モードと境界条件

  • RBM_ZERO_ON_ERROR を指定した場合、I/Oエラー発生時に例外を発生させる代わりに、ゼロクリアされたページが返されるという境界条件があります。

保守担当者向けメモ

  • 本列挙型は ReadBuffer 系の関数の挙動を決定する重要なインターフェースです。新しい読み込みモードを追加する場合は、bufmgr.c 内の各処理ロジック（特にロック取得やWALログ出力箇所）への影響を精査してください。
