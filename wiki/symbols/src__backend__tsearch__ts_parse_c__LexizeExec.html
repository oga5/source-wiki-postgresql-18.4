LexizeExec

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_parse.c:172

目的

辞書チェーンを走査し、入力されたトークンを正規化して語彙（Lexeme）を生成する。

シグネチャまたは形状

static TSLexeme *LexizeExec(LexizeData *ld, ParsedLex **correspondLexem)

入力と出力

  • **入力**:
  • ld: 辞書処理の状態を保持する LexizeData 構造体へのポインタ。
  • correspondLexem: 対応する語彙情報を格納するためのポインタ。
  • **出力**:
  • 正規化された TSLexeme 構造体へのポインタ。正規化できない場合は NULL を返す。

前提条件と不変条件

  • ld->cfg に有効な設定が保持されていること。
  • 辞書関数（dict->lexize）は FunctionCall4 を介して呼び出されるインターフェースに準拠していること。
  • ld->curDictId が InvalidOid の場合は通常モード、それ以外の場合は複数単語を跨ぐモードとして動作する。

アルゴリズムまたは振る舞い

1. **通常モード**: ld->towork リスト内のトークンを順次取り出し、設定された辞書チェーンを適用する。 2. **辞書適用**: 辞書が getnext フラグを立てた場合、複数単語モードへ移行し、現在のトークンをスタックに積んで再帰的に呼び出しを行う。 3. **正規化**: 辞書が単語を認識した場合、その結果を返し、処理済みトークンを waste リストへ移動させる。 4. **複数単語モード**: ld->curDictId が有効な場合、辞書が要求する後続の単語を処理し、正規化が完了した時点で通常モードへ復帰する。 5. **再帰**: 辞書が単語を認識できない場合や、モード遷移が必要な場合に自身を再帰呼び出しする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/ts_parse.c 内の解析処理（parsetext 等）。
  • 自身（再帰呼び出し）。
  • **呼び出し先**:
  • lookup_ts_dictionary_cache: 辞書キャッシュの取得。
  • FunctionCall4: 辞書関数の実行。
  • RemoveHead, setNewTmpRes, setCorrLex, moveToWaste: LexizeData の状態操作関数。

関連データ構造

  • LexizeData: 辞書チェーン、作業中トークン、一時結果を保持。
  • TSLexeme: 正規化結果。
  • ParsedLex: 解析対象のトークンリスト。
  • TSDictionaryCacheEntry: 辞書の設定と関数ポインタ。

失敗モードと境界条件

  • **辞書未認識**: 辞書がトークンを認識できない場合、次の辞書へ進むか、通常モードへ復帰する。
  • **型不一致**: トークン型が辞書でサポートされていない場合、そのトークンをスキップする。
  • **メモリ管理**: palloc を多用するため、tmpRes 等のメモリリークに注意が必要。
  • **長大語**: MAXSTRLEN を超えるトークンは設定に応じて無視される。

保守担当者向けメモ

  • 辞書チェーンの複雑な状態遷移（towork と waste リスト）を追跡する際は、LexizeData の状態変化に注目すること。
  • 複数単語を跨ぐ正規化（シソーラス等）のデバッグでは、ld->curSub のポインタ遷移が重要となる。
