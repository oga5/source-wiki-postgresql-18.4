WHEN_TYPE

定義

  • 種別: enum
  • 定義箇所: src/interfaces/ecpg/preproc/type.h:77

目的

ECPGプリプロセッサにおいて、SQL文実行時のエラーや警告発生時に実行されるアクションの種類を定義します。

シグネチャまたは形状

enum WHEN_TYPE
{
	W_NOTHING,
	W_CONTINUE,
	W_BREAK,
	W_SQLPRINT,
	W_GOTO,
	W_DO,
	W_STOP
};

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし

前提条件と不変条件

  • ECPGの WHENEVER 文の解析結果として使用されます。
  • 各列挙子は、SQL実行後の制御フロー（継続、中断、ジャンプ等）を決定する一意の識別子として機能します。

アルゴリズムまたは振る舞い

  • W_NOTHING: アクションなし。
  • W_CONTINUE: 次の文へ処理を継続。
  • W_BREAK: ループからの脱出。
  • W_SQLPRINT: エラーメッセージの出力。
  • W_GOTO: 指定ラベルへのジャンプ。
  • W_DO: 指定関数または処理の実行。
  • W_STOP: プログラムの停止。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/preproc/type.h:90 にて when 構造体のメンバとして参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • struct when: WHEN_TYPE を保持し、WHENEVER 文の情報を管理する構造体。

失敗モードと境界条件

  • 列挙型であるため、定義外の値が代入された場合の動作は未定義です。

保守担当者向けメモ

  • 新しい WHENEVER アクションを追加する場合は、本列挙型に値を追加し、対応するコード生成ロジック（output.c 等）を更新する必要があります。
