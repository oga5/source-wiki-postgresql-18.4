xl_hash_squeeze_page

定義

  • 種別: struct
  • 定義箇所: src/include/access/hash_xlog.h:160

目的

ハッシュインデックスのページ圧縮（squeeze）操作をWAL（Write-Ahead Logging）に記録するためのデータ構造です。オーバーフローページからタプルを移動させ、ページを解放する際の物理的な変更情報を保持します。

シグネチャまたは形状

typedef struct xl_hash_squeeze_page
{
    BlockNumber prevblkno;
    BlockNumber nextblkno;
    uint16      ntups;
    bool        is_prim_bucket_same_wrt;
    bool        is_prev_bucket_same_wrt;
} xl_hash_squeeze_page;

入力と出力

  • **入力**: ハッシュインデックスのページ圧縮処理時に、移動元および移動先のブロック情報やタプル数が本構造体に格納され、WALレコードのペイロードとして書き込まれます。
  • **出力**: リカバリ時に XLogRecGetData を通じて読み出され、インデックスページの再構築に使用されます。

前提条件と不変条件

  • 本構造体は、ハッシュインデックスのオーバーフローページ解放に関連するWALレコードの一部として使用されます。
  • is_prim_bucket_same_wrt および is_prev_bucket_same_wrt は、移動先のページがプライマリバケットページまたは直前のオーバーフローページと同一であるかを示すフラグとして機能します。

アルゴリズムまたは振る舞い

1. **書き込み**: ハッシュインデックスの圧縮処理（hashovfl.c 等）において、移動対象のページ情報と移動先との関係を本構造体にセットし、WALへ書き出します。 2. **リカバリ**: hash_redo 処理において、WALレコードから本構造体をキャストして取得します。この情報に基づき、対象となるインデックスページを読み込み、タプルの移動とページ解放を再現します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/hash/hash_xlog.c:630: WALレコードからのデータ復元に使用。
  • src/backend/access/hash/hashovfl.c:647: 圧縮処理実行時に本構造体を初期化。
  • src/backend/access/rmgrdesc/hashdesc.c:86: WALログのデコード（説明出力）に使用。
  • **呼び出し先**: 情報なし。

関連データ構造

  • BlockNumber: PostgreSQLのブロック識別子。
  • xl_hash_squeeze_page を含むWALレコードは、ハッシュインデックスのリカバリ処理（hash_redo）において処理されます。

失敗モードと境界条件

  • offsetof を用いた構造体サイズの計算（src/include/access/hash_xlog.h:175）が行われており、構造体のパディングやサイズ変更が正しく行われない場合、WALレコードの読み込み時に不整合が発生する可能性があります。

保守担当者向けメモ

  • 本構造体はハッシュインデックスの物理的なページ構造に依存しています。ページレイアウトの変更時には、本構造体のフィールド構成と hash_redo での処理ロジックを同期させる必要があります。
