config_enum

定義

  • 種別: struct
  • 定義箇所: src/include/utils/guc_tables.h:286

目的

PostgreSQLのGUC（Grand Unified Configuration）システムにおいて、列挙型（enum）として定義された設定変数のメタデータと状態を保持する構造体です。

シグネチャまたは形状

struct config_enum
{
    struct config_generic gen;
    int            *variable;
    int             boot_val;
    const struct config_enum_entry *options;
    GucEnumCheckHook check_hook;
    GucEnumAssignHook assign_hook;
    GucShowHook show_hook;
    int             reset_val;
    void           *reset_extra;
};

入力と出力

  • **入力**: config_generic を基底として、設定値の検証（check_hook）、割り当て（assign_hook）、表示（show_hook）を行うための関数ポインタおよび列挙値の定義リストを受け取ります。
  • **出力**: variable ポインタを通じて、現在の設定値（int）をシステム全体で共有・参照可能にします。

前提条件と不変条件

  • gen フィールドは構造体の先頭に配置され、config_generic との互換性を持ちます。
  • variable、boot_val、options などの定数フィールドは、初期化時に正しく設定される必要があります。
  • reset_val および reset_extra は実行時に初期化されます。

アルゴリズムまたは振る舞い

  • 設定値の変更時、check_hook を介して値の妥当性を検証し、assign_hook を介して値を適用します。
  • config_enum_lookup_by_value や config_enum_lookup_by_name を用いて、内部的な整数値と人間が読み取り可能な文字列名との相互変換を行います。
  • config_enum_get_options を通じて、設定可能な選択肢のリストを取得できます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/guc.c 内の各関数（call_enum_check_hook、ShowGUCOption 等）が、この構造体を介して設定の検証や取得を行います。
  • **呼び出し先**: check_hook、assign_hook、show_hook に登録された関数が呼び出されます。

関連データ構造

  • config_generic: 全てのGUC変数の基底構造体。
  • config_enum_entry: 列挙型の名前と値のペアを定義する構造体。

失敗モードと境界条件

  • check_hook が無効な値を拒否した場合、設定変更は失敗します。
  • options リストに存在しない値を設定しようとした場合、検索関数は失敗またはエラーを返します。

保守担当者向けメモ

  • 本構造体は config_generic を継承しているため、gconf ポインタを struct config_enum * にキャストして操作する箇所が多く存在します。
  • extra ポインタ（reset_extra）は、フック関数間で複雑な状態を共有するために使用されます。
