sort_desc

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_op.c:239

目的

SQLから呼び出された整数配列（int[]）を降順にソートし、その結果を新しい配列として返します。

シグネチャまたは形状

Datum sort_desc(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される ArrayType 型の整数配列。
  • **出力**: PG_RETURN_POINTER を介して返される、降順にソートされた ArrayType 型の配列。

前提条件と不変条件

  • 入力配列は CHECKARRVALID マクロにより、PostgreSQLの配列として有効であることが検証されます。
  • 入力配列は PG_GETARG_ARRAYTYPE_P_COPY を使用してコピーされるため、元の配列データは変更されません。

アルゴリズムまたは振る舞い

1. PG_GETARG_ARRAYTYPE_P_COPY(0) により、入力配列のコピーを作成します。 2. CHECKARRVALID(a) で配列の整合性を確認します。 3. QSORT(a, 0) を実行します。第2引数の 0 はソート順序（降順）を指定するフラグとして機能します。 4. ソート済みの配列ポインタを返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1(sort_desc) により登録）。
  • **呼び出し先**: _int.h で定義されたマクロ CHECKARRVALID および QSORT。

関連データ構造

  • ArrayType: PostgreSQLの標準的な配列データ構造。

失敗モードと境界条件

  • CHECKARRVALID マクロにより、無効な配列構造が渡された場合はエラーが発生します。
  • メモリ不足時には、PostgreSQLのメモリ管理機構によりエラーがスローされます。

保守担当者向けメモ

  • 本関数はインプレースなソートを行いますが、入力時にコピーを作成しているため、呼び出し元のデータには影響しません。
  • ソートロジックの詳細は _int.h で定義された QSORT マクロに依存しています。
