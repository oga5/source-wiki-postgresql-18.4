ifState

定義

  • 種別: enum
  • 定義箇所: src/include/fe_utils/conditional.h:29

目的

クライアントアプリケーション（psql 等）におけるネストされた \if ブロックの実行状態を管理し、現在のコードブロックが実行対象か否かを判定するための状態値を定義する。

シグネチャまたは形状

typedef enum ifState
{
    IFSTATE_NONE,
    IFSTATE_TRUE,
    IFSTATE_FALSE,
    IFSTATE_IGNORED,
    IFSTATE_ELSE_TRUE,
    IFSTATE_ELSE_FALSE
} ifState;

入力と出力

  • 本シンボルは列挙型であり、入力や出力を持つ関数ではない。
  • conditional_stack_push や conditional_stack_poke などの関数において、状態遷移の引数として利用される。
  • conditional_stack_peek 等の関数から、現在のスタックトップの状態として返される。

前提条件と不変条件

  • 親ブロックの状態が子ブロックに継承される。
  • IFSTATE_IGNORED は、親が偽である場合、または既に真の分岐を通過した後の \elif であることを示す。

アルゴリズムまたは振る舞い

各値は以下の状態を表す：

  • IFSTATE_NONE: \if ブロック外。
  • IFSTATE_TRUE: 現在の \if/\elif が真であり、かつ全ての親ブロックも真。
  • IFSTATE_FALSE: 現在の \if/\elif が偽だが、まだ真の分岐に遭遇しておらず、かつ全ての親ブロックが真。
  • IFSTATE_IGNORED: 真の分岐を通過した後の \elif、または偽の親を持つブロック。
  • IFSTATE_ELSE_TRUE: \else が真であり、かつ全ての親ブロックが真。
  • IFSTATE_ELSE_FALSE: \else が偽または無視される状態。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/conditional.c 内のスタック操作関数（conditional_stack_push, conditional_stack_poke, conditional_stack_peek）がこの型を使用している。
  • **呼び出し先**: 本シンボル自体は列挙型であり、関数を呼び出さない。

関連データ構造

  • IfStackElem: スタックの各要素内で if_state フィールドとして保持される。
  • ConditionalStack: IfStackElem を管理するスタック構造体。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • psql のスクリプト実行制御において、非アクティブなブロックのテキストを破棄するかどうかの判断基準となる。
  • 新しい状態を追加する場合は、conditional_stack 関連のロジック（特に conditional_active()）が正しく判定を行えるか確認すること。
