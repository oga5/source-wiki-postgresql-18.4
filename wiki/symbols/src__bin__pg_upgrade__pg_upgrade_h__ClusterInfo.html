ClusterInfo

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.h:287

目的

PostgreSQL のアップグレード対象となる旧クラスタおよび新クラスタのメタデータ、構成情報、および実行状態を保持する主要なデータ構造です。

シグネチャまたは形状

typedef struct {
    ControlData controldata;
    DbLocaleInfo *template0;
    DbInfoArr    dbarr;
    char        *pgdata;
    char        *pgconfig;
    char        *bindir;
    char        *pgopts;
    char        *sockdir;
    unsigned short port;
    uint32       major_version;
    char         major_version_str[64];
    uint32       bin_version;
    const char  *tablespace_suffix;
    int          nsubs;
} ClusterInfo;

入力と出力

本構造体は、pg_upgrade の各フェーズ（チェック、ダンプ、転送、実行）において、クラスタの現在の状態を保持するコンテナとして機能します。関数への引数として渡されることで、対象クラスタのパスやポート、バージョン情報を提供します。

前提条件と不変条件

  • pg_upgrade のメイン処理において、旧クラスタと新クラスタのそれぞれに対してインスタンスが初期化される必要があります。
  • controldata フィールドは、pg_control ファイルの内容を保持し、バージョン間の差異を吸収する役割を担います。

アルゴリズムまたは振る舞い

各モジュール（info.c, check.c 等）が本構造体を参照し、クラスタの情報を収集・検証します。特に check.c 内の各チェック関数は、本構造体を通じてクラスタの構成がアップグレード可能であるかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/check.c 内の各種チェック関数（check_is_install_user, check_for_connection_status, check_for_prepared_transactions 等）が引数として受け取ります。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ControlData: クラスタの制御データ。
  • DbInfoArr: クラスタ内のデータベース情報の配列。
  • DbLocaleInfo: ロケール情報。

失敗モードと境界条件

  • pgdata や bindir などのパス情報が正しく設定されていない場合、後続のファイル操作やサーバー起動処理が失敗します。
  • major_version や bin_version の不整合は、アップグレードの可否判定においてエラーを引き起こす要因となります。

保守担当者向けメモ

  • 本構造体は PostgreSQL バックエンドのコードを直接再利用せず、独自の定義を使用しています。これはメジャーバージョン間での構造体定義の不一致を回避するためです。
  • 構造体内のフィールドを追加・変更する際は、info.c での初期化ロジックおよび各チェック関数への影響を確認してください。
