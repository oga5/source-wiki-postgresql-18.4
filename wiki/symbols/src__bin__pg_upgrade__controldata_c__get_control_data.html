get_control_data

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/controldata.c:37

目的

PostgreSQLのアップグレード対象となるクラスタの制御データ（pg_control）を外部コマンド経由で取得・解析し、クラスタの互換性および状態を検証します。

シグネチャまたは形状

void get_control_data(ClusterInfo *cluster)

入力と出力

  • **入力**: ClusterInfo *cluster（解析対象のクラスタ情報構造体へのポインタ）
  • **出力**: なし（副作用として cluster->controldata 構造体に解析結果を格納）

前提条件と不変条件

  • 実行環境のロケール設定（LC_ALL 等）は、コマンド出力をパースするために一時的に C ロケールへ変更されます。
  • 対象クラスタのバイナリパス（bindir）およびデータディレクトリ（pgdata）が ClusterInfo に設定されている必要があります。
  • 外部コマンド pg_controldata または pg_resetwal が実行可能である必要があります。

アルゴリズムまたは振る舞い

1. **環境変数の退避**: 現在のロケール環境変数を退避し、解析に必要な C ロケールを適用します。 2. **状態確認**: live_check が有効でない場合、pg_controldata を実行してクラスタが「shut down」状態であることを確認します。リカバリ中や異常終了している場合は pg_fatal で停止します。 3. **データ抽出**: pg_controldata または pg_resetwal -n を実行し、標準出力をパイプ経由で読み取ります。 4. **パース**: 出力行を一行ずつ読み込み、特定のキー文字列を検索して値を抽出します。バージョンに応じた条件分岐（GET_MAJOR_VERSION）を行い、適切なコマンド引数を選択します。 5. **後処理**: pclose() でコマンド終了を待ち、必須項目が取得できたかを確認します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/check.c 内の get_control_data(&old_cluster) および get_control_data(&new_cluster)。
  • **呼び出し先**: popen(), pclose(), fgets(), strstr(), pg_fatal(), pg_strip_crlf() 等の標準ライブラリおよびユーティリティ関数。

関連データ構造

  • ClusterInfo: クラスタの構成情報および制御データを保持する構造体。
  • ControlData: ClusterInfo 内で制御パラメータを格納する構造体。

失敗モードと境界条件

  • **コマンド実行失敗**: popen() が失敗した場合や、コマンドの終了ステータスが 0 でない場合に pg_fatal で停止します。
  • **解析失敗**: 必須の制御データ項目が取得できなかった場合、またはクラスタが正常にシャットダウンされていない場合にエラーを報告します。
  • **バージョン依存**: PostgreSQL 9.2 以前とそれ以降で、チェックサム等の取得ロジックが異なります。

保守担当者向けメモ

  • コマンド出力の文字列解析に依存しているため、pg_controldata の出力フォーマットが変更されるとパースロジックの修正が必要です。
  • ロケール環境変数の退避・復元処理は、他の処理に影響を与えないよう慎重に扱ってください。
  • live_check モード時は pg_resetwal ではなく pg_controldata が優先的に使用される点に注意してください。
