pg_strtoint16_safe

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/numutils.c:127

目的

文字列を int16 型の整数へ安全に変換します。エラー発生時には ereport を直接呼び出す代わりに、提供された escontext を介してエラー情報を伝播させます。

シグネチャまたは形状

int16 pg_strtoint16_safe(const char *s, Node *escontext);

入力と出力

  • **入力**:
  • s: 解析対象の文字列（NUL終端）。
  • escontext: エラー発生時に情報を格納するための Node 構造体（ErrorSaveContext 等）。
  • **出力**:
  • 変換後の int16 値。
  • 失敗時は escontext にエラーが記録され、呼び出し元へ制御が戻ります。

前提条件と不変条件

  • 入力文字列 s は有効なポインタである必要があります。
  • 変換結果が int16 の範囲（PG_INT16_MIN から PG_INT16_MAX）に収まる必要があります。
  • アンダースコア（_）は可読性のためのみに使用され、数値計算には影響しません。

アルゴリズムまたは振る舞い

1. **高速パス**: まず、単純な10進数文字列（符号付き）を想定して解析を試みます。 2. **低速パス**: 高速パスで解析できない場合（基数指定 0x, 0o, 0b やアンダースコアを含む場合、または先頭の空白など）、slow ラベルへジャンプし、より汎用的な解析を行います。 3. **エラー処理**: 範囲外の値や不正な構文を検出した場合、out_of_range または invalid_syntax ラベルへジャンプし、escontext を利用してエラーを通知します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/int.c:67
  • src/backend/utils/adt/numutils.c:124
  • **呼び出し先**:
  • isspace, isxdigit（標準ライブラリ）
  • pg_neg_u16_overflow（common/int.h）
  • ereport（エラー通知時）

関連データ構造

  • Node *escontext: エラーハンドリング用コンテキスト。
  • PG_INT16_MIN, PG_INT16_MAX: 境界値チェックに使用。

失敗モードと境界条件

  • **オーバーフロー**: int16 の範囲を超える値が入力された場合。
  • **不正な構文**: 数値として解釈できない文字が含まれる場合、またはアンダースコアの配置が不適切な場合。
  • **空の入力**: 数値を含まない文字列。

保守担当者向けメモ

  • 本関数は int16 専用です。他のサイズ（int32, int64）については、対応する pg_strtoint32_safe 等を参照してください。
  • 高速パスと低速パスの分岐ロジックは、パフォーマンスを最適化するために設計されています。修正時は高速パスの条件を維持するように注意してください。
