get_partition_col_typmod

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/partcache.h:91

目的

パーティションキーの特定の列に対応する型修飾子（typmod）を取得します。

シグネチャまたは形状

int32 get_partition_col_typmod(PartitionKey key, int col)

入力と出力

  • **入力**:
  • key: パーティションキー情報を保持する PartitionKey 構造体へのポインタ。
  • col: 取得対象の列インデックス（0から始まる整数）。
  • **出力**:
  • 指定された列の型修飾子（int32）。

前提条件と不変条件

  • key は有効な PartitionKey 構造体を指している必要があります。
  • パーティションキーは一度構築されると変更されないという不変条件があるため、リレーションを開いている間は安全に参照可能です。

アルゴリズムまたは振る舞い

key->parttypmod 配列の指定されたインデックス col に格納されている値を直接返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_utilcmd.c（パーティション定義の解析処理において、列の型情報を取得するために使用されています）。
  • **呼び出し先**:
  • なし（インライン関数として定義されており、直接的な関数呼び出しは行いません）。

関連データ構造

  • PartitionKey: パーティションキーのメタデータを保持する構造体。本関数はこの構造体内の parttypmod 配列を参照します。

失敗モードと境界条件

  • col が有効な範囲外である場合、メモリ破壊や不正な値の読み取りが発生する可能性があります（本関数内での境界チェックは行われていません）。

保守担当者向けメモ

  • 本関数は PartitionKey 構造体への直接アクセスをカプセル化するインライン関数群の一つです。将来的な構造体の変更に対する抽象化レイヤーとして機能します。
