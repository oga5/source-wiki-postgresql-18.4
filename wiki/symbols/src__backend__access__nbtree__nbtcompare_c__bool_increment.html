bool_increment

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtcompare.c:99

目的

B-tree インデックスのスキップスキャンにおいて、boolean 型の値を次の値へ増分する。

シグネチャまたは形状

static Datum bool_increment(Relation rel, Datum existing, bool *overflow)

入力と出力

  • **入力**:
  • rel: 対象となるリレーション（未使用）。
  • existing: 現在の boolean 値（Datum 型）。
  • overflow: 増分が不可能であることを示すフラグを格納するポインタ。
  • **出力**:
  • 戻り値: 増分後の Datum 値。
  • *overflow: existing が true の場合に true が設定される。

前提条件と不変条件

  • boolean 型の取りうる値は false (0) と true (1) である。
  • true を超える値への増分は定義されない。

アルゴリズムまたは振る舞い

1. existing を bool 型に変換する。 2. 値が true の場合:

  • *overflow を true に設定する。
  • 戻り値として (Datum) 0 を返す。

    • 3. 値が false の場合:

  • *overflow を false に設定する。
  • false (0) に 1 を加算した結果（true）を Datum に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtcompare.c 内で、SkipSupport 構造体の increment フィールドに本関数が登録されている（121行目）。
  • **呼び出し先**: 外部関数への直接的な呼び出しはソース内に存在しない。

関連データ構造

  • SkipSupport: インデックススキップスキャン用の増減ロジックを保持する構造体。

失敗モードと境界条件

  • existing が true の場合、これ以上増分できないため overflow が発生する。

保守担当者向けメモ

  • 本関数は SkipSupport インターフェースの一部として使用される。
  • rel 引数は現在の実装では使用されていないが、将来的な拡張や型固有の処理のためにシグネチャに含まれている。
