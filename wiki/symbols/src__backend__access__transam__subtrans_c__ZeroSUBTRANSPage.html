ZeroSUBTRANSPage

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/subtrans.c:295

目的

SUBTRANS（サブトランザクションログ）の指定されたページをゼロクリアし、初期化する。

シグネチャまたは形状

static int ZeroSUBTRANSPage(int64 pageno)

入力と出力

  • **入力**: pageno (初期化対象のページ番号)
  • **出力**: int (初期化されたSLRUスロット番号)

前提条件と不変条件

  • SubTransCtl が初期化済みであること。
  • 呼び出し元は、SLRUの排他制御（ロック）を適切に管理している必要がある。

アルゴリズムまたは振る舞い

1. SubTransCtl（SUBTRANS用のSLRU制御構造体）を引数として SimpleLruZeroPage を呼び出す。 2. SimpleLruZeroPage の戻り値（割り当てられたスロット番号）をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • BootStrapSUBTRANS (src/backend/access/transam/subtrans.c:278)
  • ExtendSUBTRANS (src/backend/access/transam/subtrans.c:338, 398)
  • **呼び出し先**:
  • SimpleLruZeroPage (src/backend/access/transam/slru.c)

関連データ構造

  • SubTransCtl: SUBTRANSのSLRU管理用共有メモリ構造体。

失敗モードと境界条件

  • SimpleLruZeroPage が内部でエラーを発生させた場合（メモリ不足やI/Oエラー等）、本関数も例外を投げる可能性がある。

保守担当者向けメモ

  • 本関数は SimpleLruZeroPage の単なるラッパーである。SUBTRANSのページ管理ロジックを変更する場合は、SubTransCtl の設定や slru.c 側の挙動に注意すること。
