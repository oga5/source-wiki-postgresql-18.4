print_double

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/pgtypeslib-num_test.c:32

目的

プラットフォーム（特にWindows）間での浮動小数点数出力の差異を吸収し、正規化された形式で出力するためのヘルパー関数。

シグネチャまたは形状

static void print_double(double x)

入力と出力

  • **入力**: double x（出力対象の浮動小数点数）
  • **出力**: なし（標準出力 stdout へ直接書き出す）

前提条件と不変条件

  • WIN32 環境下では、sprintf で生成された文字列の指数部が3桁（例: e+001）になる場合、これを2桁（例: e+01）に変換する処理が実行される。
  • 非 WIN32 環境では、標準の printf("%g", x) と同等の動作を行う。

アルゴリズムまたは振る舞い

1. WIN32 定義時:

  • sprintf を使用して double 値を文字列 convert に変換する。
  • 文字列の末尾付近を走査し、指数表記が e+00 のような3桁形式であるかを確認する。
  • 該当する場合、指数部の0を一つ削除して2桁に詰め、文字列を再構成する。
  • 最終的な文字列を printf で出力する。

    • 2. それ以外:

  • printf("%g", x) を直接実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/test/expected/compat_informix-dec_test.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test2.c
  • src/interfaces/ecpg/test/printf_hack.h
  • **呼び出し先**: sprintf, strlen, printf（標準Cライブラリ関数）

関連データ構造

  • なし

失敗モードと境界条件

  • convert バッファ（128バイト）を超える長さの文字列が生成される場合、sprintf によるバッファオーバーフローの可能性がある（ただし、通常の double 出力では十分に収まるサイズと想定される）。
  • 指数部が3桁でない場合や、%g の出力形式が想定と異なる場合は、文字列操作ロジックがスキップされ、そのまま出力される。

保守担当者向けメモ

  • 本関数はプリプロセッサ生成コードの一部であり、移植性維持のために重要である。
  • Windows環境でのテスト結果の期待値（expectedファイル）と実際の出力が一致しない場合、この正規化ロジックが現在のプラットフォームの出力形式と合致しているか確認すること。
