MultiXactMemberFreezeThreshold

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/multixact.c:3149

目的

MultiXact メンバーのストレージ使用状況に基づき、自動バキューム（autovacuum）が適用すべき動的な「MultiXact 凍結閾値（freeze age）」を算出する。

シグネチャまたは形状

int MultiXactMemberFreezeThreshold(void)

入力と出力

  • **入力**: なし（内部で ReadMultiXactCounts を呼び出し、現在の MultiXact 数およびメンバー数を取得）。
  • **出力**: 凍結処理を開始するまでの許容年齢（int 型）。

前提条件と不変条件

  • ReadMultiXactCounts が失敗した場合は、安全側に倒して 0 を返す。
  • 算出された凍結年齢は、常に autovacuum_multixact_freeze_max_age を超えないよう Min() 関数でクランプされる。

アルゴリズムまたは振る舞い

1. ReadMultiXactCounts を実行し、現在のシステム内の MultiXact 数とメンバー数を取得する。 2. メンバー数が MULTIXACT_MEMBER_SAFE_THRESHOLD 以下であれば、デフォルトの autovacuum_multixact_freeze_max_age を返す。 3. メンバー数が安全閾値を超えている場合、危険閾値（MULTIXACT_MEMBER_DANGER_THRESHOLD）までの超過割合を計算する。 4. その割合に応じて、システムから排除すべき MultiXact 数を算出し、現在の総数から差し引くことで、より積極的な（小さい）凍結年齢を算出する。 5. 算出結果が負にならないよう調整し、最終的に autovacuum_multixact_freeze_max_age との小さい方を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/vacuum.c: effective_multixact_freeze_max_age の算出に使用。
  • src/backend/postmaster/autovacuum.c: multiForceLimit の算出および effective_multixact_freeze_max_age の設定に使用。
  • **呼び出し先**:
  • ReadMultiXactCounts（MultiXact の統計情報取得）。
  • Min（マクロによる最小値選択）。

関連データ構造

  • autovacuum_multixact_freeze_max_age（GUC パラメータ）
  • MULTIXACT_MEMBER_SAFE_THRESHOLD / MULTIXACT_MEMBER_DANGER_THRESHOLD（定数）

失敗モードと境界条件

  • ReadMultiXactCounts が失敗した場合、0 を返す（即時の凍結処理を促す挙動）。
  • 算出された凍結年齢が 0 を下回ることはない。

保守担当者向けメモ

  • 本関数は、MultiXact の周回（Wraparound）を防止するための自動バキュームの挙動を制御する重要なロジックである。
  • 閾値定数（SAFE/DANGER）を変更する場合は、SLRU の容量設計と整合性が取れているか確認すること。
