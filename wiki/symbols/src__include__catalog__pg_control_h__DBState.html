DBState

定義

  • 種別: enum
  • 定義箇所: src/include/catalog/pg_control.h:89

目的

PostgreSQL クラスタのライフサイクルにおける現在の状態（起動、シャットダウン、リカバリ、稼働中など）を識別するために使用されます。

シグネチャまたは形状

typedef enum DBState
{
	DB_STARTUP = 0,
	DB_SHUTDOWNED,
	DB_SHUTDOWNED_IN_RECOVERY,
	DB_SHUTDOWNING,
	DB_IN_CRASH_RECOVERY,
	DB_IN_ARCHIVE_RECOVERY,
	DB_IN_PRODUCTION,
} DBState;

入力と出力

  • 入力: 該当なし（列挙型定義）。
  • 出力: データベースの状態を示す整数値（列挙子）。

前提条件と不変条件

  • pg_control ファイル内に格納されるメタデータの一部であり、クラスタの整合性チェックにおいて重要な役割を果たします。
  • 状態遷移は、データベースの起動、停止、およびリカバリプロセスによって制御されます。

アルゴリズムまたは振る舞い

データベースの起動時やリカバリ処理において、pg_control ファイルから読み込まれた DBState の値に基づき、システムが正常にシャットダウンされたか、あるいはクラッシュリカバリが必要かといった判断が行われます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/access/transam/xlogrecovery.c: dbstate_at_startup 変数として使用。
  • src/bin/pg_controldata/pg_controldata.c: dbState() 関数で状態表示に使用。
  • src/bin/pg_ctl/pg_ctl.c: get_control_dbstate() 関数および状態判定に使用。
  • src/include/catalog/pg_control.h: ControlFileData 構造体内のメンバとして使用。
  • 呼び出し先: 該当なし（列挙型のため）。

関連データ構造

  • ControlFileData: DBState を保持するメインの構造体。

失敗モードと境界条件

  • 不正な値: pg_control ファイルが破損している場合、定義外の値が読み込まれる可能性があります。
  • 整合性: DB_SHUTDOWNED 以外の状態で停止した場合、次回の起動時にリカバリプロセスがトリガーされます。

保守担当者向けメモ

  • 本列挙型に新しい状態を追加する場合は、pg_controldata ユーティリティ側の表示ロジックも併せて更新してください。
  • pg_control ファイルのバイナリ互換性に影響するため、順序の変更や削除は避けてください。
