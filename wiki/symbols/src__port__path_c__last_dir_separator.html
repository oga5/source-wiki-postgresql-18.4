last_dir_separator

定義

  • 種別: function
  • 定義箇所: src/port/path.c:144

目的

ファイルパス文字列内において、最後に現れるディレクトリ区切り文字へのポインタを返します。

シグネチャまたは形状

char *last_dir_separator(const char *filename);

入力と出力

  • **入力**: filename (ヌル終端文字列へのポインタ)
  • **出力**: パス内の最後のディレクトリ区切り文字を指す char *。区切り文字が見つからない場合は NULL を返します。

前提条件と不変条件

  • filename は有効なヌル終端文字列である必要があります。
  • 戻り値は const 修飾が外されたポインタとして返されます。

アルゴリズムまたは振る舞い

1. skip_drive(filename) を呼び出し、Windows のドライブレター部分をスキップして走査開始位置を決定します。 2. 文字列の末尾まで走査し、IS_DIR_SEP マクロで判定されるディレクトリ区切り文字（OS 依存）が現れるたびに、その位置を ret に記録します。 3. 走査終了後、最後に記録された区切り文字の位置を返します。

  • unconstify (マクロ)

関連データ構造

  • なし

失敗モードと境界条件

  • パス内にディレクトリ区切り文字が含まれない場合、NULL を返します。
  • ドライブレターのみのパスや、区切り文字を含まないファイル名のみの入力に対しては安全に動作します。

保守担当者向けメモ

  • 本関数は const char * を受け取りつつ char * を返すため、呼び出し側で返されたポインタを介して元の文字列を書き換えることが可能です（例: *last_dir_separator(...) = '\0' によるパスの切り詰め）。この挙動に依存しているコードが多いため、変更時は注意してください。
