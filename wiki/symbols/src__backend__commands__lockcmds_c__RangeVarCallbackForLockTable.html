RangeVarCallbackForLockTable

定義

  • 種別: function
  • 定義箇所: src/backend/commands/lockcmds.c:70

目的

LOCK TABLE コマンド実行時、対象リレーションの存在確認、型検証、および権限チェックを行うためのコールバック関数。

シグネチャまたは形状

static void RangeVarCallbackForLockTable(const RangeVar *rv, Oid relid, Oid oldrelid, void *arg)
  • rv: 対象リレーションの識別子。
  • relid: 対象リレーションの OID。
  • oldrelid: 以前の OID（未使用）。
  • arg: LOCKMODE へのポインタ。

入力と出力

  • 入力: RangeVar 構造体、OID、および LOCKMODE を保持する arg。
  • 出力: なし（成功時は正常終了、失敗時は ereport(ERROR) を発行）。

前提条件と不変条件

  • relid が有効でない場合、またはリレーションが並行して削除された場合は、権限チェックを行わずに即座に復帰する。
  • 一時リレーションがアクセスされた場合、MyXactFlags に XACT_FLAGS_ACCESSEDTEMPNAMESPACE が設定される。

アルゴリズムまたは振る舞い

1. get_rel_relkind(relid) を呼び出し、リレーションの種類を確認する。 2. リレーションが「通常のテーブル」「パーティションテーブル」「ビュー」のいずれかであることを検証する。それ以外の場合は ERRCODE_WRONG_OBJECT_TYPE エラーを発生させる。 3. 一時リレーションであれば、トランザクションフラグを更新する。 4. LockTableAclCheck を呼び出し、現在のユーザーが指定された lockmode で対象リレーションをロックする権限があるか確認する。 5. 権限がない場合、aclcheck_error を通じてエラーを報告する。

呼び出し元と呼び出し先

  • 呼び出し元: ソースコード上の直接的な呼び出し元は明示されていませんが、RangeVar を解決するプロセスの一部として利用されます。
  • 呼び出し先: get_rel_relkind, get_rel_persistence, LockTableAclCheck, GetUserId, get_relkind_objtype, ereport。

関連データ構造

  • RangeVar: リレーションを識別するための構造体。
  • LOCKMODE: ロックの強度を定義する列挙型。

失敗モードと境界条件

  • リレーションが存在しない場合: 処理をスキップして正常終了。
  • リレーションがテーブル・ビュー以外（インデックスやシーケンス等）の場合: ERRCODE_WRONG_OBJECT_TYPE でエラー。
  • 権限不足の場合: aclcheck_error によりエラー。

保守担当者向けメモ

  • 本関数は LOCK TABLE の検証フェーズに特化しています。新しいリレーションタイプをサポートする場合は、relkind のチェック条件を更新する必要があります。
