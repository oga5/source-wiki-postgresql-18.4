bloom_create

定義

  • 種別: function
  • 定義箇所: src/backend/lib/bloomfilter.c:86

目的

指定された要素数とメモリ制限に基づき、ブルームフィルタ構造体を初期化し、必要なメモリを割り当てる。

シグネチャまたは形状

bloom_filter *bloom_create(int64 total_elems, int bloom_work_mem, uint64 seed)

入力と出力

  • **入力**:
  • total_elems: フィルタに格納する想定要素数。
  • bloom_work_mem: 使用可能なメモリ量（KB単位）。
  • seed: ハッシュ計算に使用するシード値。
  • **出力**:
  • 初期化済みの bloom_filter 構造体へのポインタ。

前提条件と不変条件

  • ビットセットのサイズは 2 のべき乗に調整される。
  • メモリ割り当ては palloc0 を使用し、ビットセット領域はゼロ初期化される。
  • 最小メモリサイズは 1MB に制限される。
  • 偽陽性率を 1% 未満に抑えるため、要素あたり 2 バイトのメモリ確保を目標とする。

アルゴリズムまたは振る舞い

1. bloom_work_mem と total_elems から目標とするビットセットサイズを算出する。 2. my_bloom_power を呼び出し、サイズを 2 のべき乗に切り下げる。 3. palloc0 を用いて bloom_filter 構造体とビットセット領域を確保する。 4. optimal_k を用いて最適なハッシュ関数数 k を決定し、構造体に格納する。 5. シード値とビットセットサイズ m を構造体に設定して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_nbtree.c
  • src/backend/utils/adt/acl.c
  • src/test/modules/test_bloomfilter/test_bloomfilter.c
  • **呼び出し先**:
  • my_bloom_power
  • optimal_k
  • palloc0

関連データ構造

  • bloom_filter: フィルタの状態（ビットセット、ハッシュ関数数、シード値、サイズ）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる可能性がある。
  • bloom_work_mem が極端に小さい場合でも、最小サイズ（1MB）が適用される。

保守担当者向けメモ

  • ビットセットサイズは 32 ビットハッシュ値の範囲内に収めるため、最大 512MB に制限されている。
  • 拡張ダブルハッシュ法を採用しているため、ビットセットサイズが 2 のべき乗であっても衝突を適切に分散できる。
