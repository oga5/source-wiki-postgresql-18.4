BTIndexStat

定義

  • 種別: struct
  • 定義箇所: contrib/pgstattuple/pgstatindex.c:77

目的

B-treeインデックスの物理的な状態および統計情報を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct BTIndexStat
{
	uint32		version;
	uint32		level;
	BlockNumber root_blkno;
	uint64		internal_pages;
	uint64		leaf_pages;
	uint64		empty_pages;
	uint64		deleted_pages;
	uint64		max_avail;
	uint64		free_space;
	uint64		fragments;
} BTIndexStat;

入力と出力

本構造体は関数への引数や戻り値として直接渡されるものではなく、インデックス走査中に統計情報を蓄積するためのコンテナとして使用されます。

前提条件と不変条件

  • インデックスのメタページおよび各データページを走査する過程で、各フィールドがインクリメントまたは更新されます。
  • pgstatindex_impl 関数内での統計収集処理において、対象インデックスのロックが保持されている必要があります。

アルゴリズムまたは振る舞い

B-treeインデックスの全ページを走査し、ページタイプ（リーフ、内部、未使用等）を判別して、対応するカウンタ（internal_pages, leaf_pages, empty_pages, deleted_pages 等）を更新します。また、空き容量や断片化状況を計算するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgstattuple/pgstatindex.c 内の pgstatindex_impl 関数（indexStat 変数としてインスタンス化）。
  • 呼び出し先: 本構造体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • GinIndexStat: GINインデックス用の統計構造体。
  • HashIndexStat: Hashインデックス用の統計構造体。

失敗モードと境界条件

  • インデックスがB-tree形式でない場合、本構造体を用いた統計収集は行われません。
  • 巨大なインデックスを走査する場合、I/O負荷が高くなるため、BAS_BULKREAD 戦略を用いたバッファ管理が行われます。

保守担当者向けメモ

  • 本構造体のフィールド定義を変更する場合、統計情報を出力するSQL関数側の戻り値型定義（get_call_result_type 等で構築されるタプル構造）との整合性に注意してください。
