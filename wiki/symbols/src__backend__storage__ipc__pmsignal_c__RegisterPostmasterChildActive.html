RegisterPostmasterChildActive

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/pmsignal.c:289

目的

Postmasterの子プロセスがアクティブ状態へ移行したことを共有メモリ上で記録し、プロセス終了時に自動的に非アクティブ化するためのクリーンアップ処理を登録します。

シグネチャまたは形状

void RegisterPostmasterChildActive(void)

入力と出力

  • 入力: なし（グローバル変数 MyPMChildSlot を使用）
  • 出力: なし（共有メモリ上の PMSignalState を更新）

前提条件と不変条件

  • MyPMChildSlot が有効な範囲（1以上かつ PMSignalState->num_child_flags 以下）であること。
  • 対象のスロットが事前に PM_CHILD_ASSIGNED 状態であること。

アルゴリズムまたは振る舞い

1. 現在のプロセスに割り当てられたスロット番号 MyPMChildSlot を取得します。 2. 共有メモリ上の PMSignalState->PMChildFlags 配列の該当インデックスを PM_CHILD_ACTIVE に更新します。 3. on_shmem_exit を呼び出し、プロセス終了時に MarkPostmasterChildInactive が実行されるよう登録します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/lmgr/proc.c:410
  • src/backend/storage/lmgr/proc.c:631
  • 呼び出し先:
  • on_shmem_exit (クリーンアップフックの登録)
  • MarkPostmasterChildInactive (終了時に実行されるコールバック)

関連データ構造

  • PMSignalState: 共有メモリ上のシグナル管理構造体。
  • PMChildFlags: 子プロセスの状態を保持する配列。

失敗モードと境界条件

  • Assert により、スロット番号の範囲外アクセスや、期待される状態（PM_CHILD_ASSIGNED）でない場合にプログラムが停止します。

保守担当者向けメモ

  • 本関数は、子プロセスが初期化を終え、アクティブに動作を開始する直前に呼び出されることを想定しています。
  • on_shmem_exit を使用しているため、プロセスが異常終了または正常終了する際、自動的に MarkPostmasterChildInactive が呼び出され、スロットが解放されます。
