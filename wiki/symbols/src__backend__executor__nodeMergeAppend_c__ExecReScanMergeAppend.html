ExecReScanMergeAppend

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMergeAppend.c:354

目的

MergeAppend ノードの実行状態をリセットし、再スキャン可能な状態にする。

シグネチャまたは形状

void ExecReScanMergeAppend(MergeAppendState *node)

入力と出力

  • **入力**: MergeAppendState *node（再スキャン対象の実行状態構造体）
  • **出力**: なし（副作用として node の内部状態を更新）

前提条件と不変条件

  • node は有効な MergeAppendState を指している必要がある。
  • パーティション・プルーニングが有効な場合、ms_prune_state が設定されている。

アルゴリズムまたは振る舞い

1. **プルーニング状態の更新**: ms_prune_state が存在し、かつ chgParam（変更されたパラメータ）がプルーニング式で使用されている場合、ms_valid_subplans を解放して NULL にリセットする。 2. **サブプランの再スキャン**: 全てのサブプラン（node->mergeplans）をループ処理する。

  • 各サブプランに対して UpdateChangedParamSet を呼び出し、パラメータ変更を通知する。
  • サブプランの chgParam が NULL の場合のみ、ExecReScan を呼び出してサブプランを再スキャンする。

    • 3. **ヒープのリセット**: binaryheap_reset を呼び出し、マージ用の優先度付きキューを初期化する。 4. **初期化フラグの解除**: node->ms_initialized を false に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:150 からの呼び出しが確認されている。
  • **呼び出し先**:
  • UpdateChangedParamSet
  • ExecReScan
  • binaryheap_reset
  • bms_free (条件付き)

関連データ構造

  • MergeAppendState: ノードの実行状態を保持。
  • PlanState: 各サブプランの状態。
  • binaryheap: マージ処理に使用される優先度付きキュー。

失敗モードと境界条件

  • ms_prune_state が NULL の場合、プルーニング関連の処理はスキップされる。
  • サブプランの chgParam が非 NULL の場合、そのサブプランに対する ExecReScan はスキップされる（最初の ExecProcNode 呼び出し時に再スキャンされるため）。

保守担当者向けメモ

  • 本関数は ExecReScan のインターフェースの一部として実装されている。
  • binaryheap_reset によりヒープが空になるため、次回の実行時には ExecMergeAppend 内でヒープの再構築が必要となる。
