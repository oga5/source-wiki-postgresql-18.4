additional_numeric_locale_len

定義

  • 種別: function
  • 定義箇所: src/fe_utils/print.c:288

目的

数値文字列をロケール固有の形式（桁区切り文字や小数点記号の挿入）に変換する際に、元の文字列長に対して追加で必要となるバイト数を算出します。

シグネチャまたは形状

static int additional_numeric_locale_len(const char *my_str)

入力と出力

  • **入力**: my_str (数値を含む文字列へのポインタ)
  • **出力**: ロケール適用後に増加する文字列の長さ（バイト数）を int 型で返します。

前提条件と不変条件

  • グローバル変数 groupdigits（桁区切りの間隔）、thousands_sep（桁区切り文字）、decimal_point（小数点記号）が適切に設定されている必要があります。
  • my_str は有効な数値文字列であると想定されています。

アルゴリズムまたは振る舞い

1. integer_digits() を呼び出し、数値の整数部分の桁数を取得します。 2. 整数部分の桁数が groupdigits を超える場合、挿入される桁区切り文字の総バイト数を計算して加算します。 3. 文字列内に . が含まれる場合、現在の小数点記号とロケール固有の小数点記号の長さの差分（strlen(decimal_point) - 1）を加算します。 4. 最終的な追加バイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/print.c 内の数値フォーマット処理（new_len = strlen(my_str) + additional_numeric_locale_len(my_str); など）。
  • **呼び出し先**:
  • integer_digits()
  • strlen()
  • strchr()

関連データ構造

  • 本関数は特定のデータ構造を操作しませんが、printTable 関連の出力処理において、バッファサイズを動的に決定するために利用されます。

失敗モードと境界条件

  • my_str に小数点が含まれない場合、小数点に関する追加長は計算されません。
  • 整数部分の桁数が groupdigits 以下の場合、桁区切り文字による追加長は発生しません。

保守担当者向けメモ

  • 本関数は static であり、src/fe_utils/print.c 内でのみ使用されます。
  • ロケール設定が変更された場合、グローバル変数の整合性が保たれていることを確認してください。
