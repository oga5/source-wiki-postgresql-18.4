TSDictionaryCacheEntry

定義

  • **種別**: struct
  • **定義箇所**: src/include/tsearch/ts_cache.h:51

目的

全文検索（Tsearch）における辞書オブジェクトのメタデータをメモリ上に保持し、カタログ（pg_ts_dict）へのアクセスを回避して辞書操作を高速化するためのキャッシュエントリ構造体です。

シグネチャまたは形状

typedef struct TSDictionaryCacheEntry
{
    Oid         dictId;         /* ハッシュキー（先頭必須） */
    bool        isvalid;        /* キャッシュの有効性フラグ */
    Oid         lexizeOid;      /* lexize関数のOID */
    FmgrInfo    lexize;         /* lexize関数の実行情報 */
    MemoryContext dictCtx;      /* 辞書固有データ用メモリコンテキスト */
    void       *dictData;       /* 辞書固有データへのポインタ */
} TSDictionaryCacheEntry;

入力と出力

  • **入力**: 辞書ID（Oid）をキーとしてハッシュテーブルから検索されます。
  • **出力**: 辞書の lexize メソッドを実行するための FmgrInfo 構造体および、辞書固有のデータポインタを提供します。

前提条件と不変条件

  • dictId は構造体の最初のメンバである必要があります（ハッシュテーブルのキーとして使用されるため）。
  • isvalid が false の場合、キャッシュは無効とみなされ、再構築が必要です。
  • 辞書固有データは dictCtx に割り当てられるため、キャッシュの再構築時にはメモリリークに注意が必要です。

アルゴリズムまたは振る舞い

1. **ルックアップ**: TSDictionaryCacheHash を使用して dictId に基づくエントリを検索します。 2. **キャッシュミス/無効時**: カタログから情報を読み込み、lexize 関数を初期化し、dictData を設定します。 3. **無効化**: システムキャッシュの更新が検知されると、isvalid が false に設定され、次回アクセス時に再構築がトリガーされます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/ts_cache.c 内のキャッシュ管理関数、および src/backend/tsearch/dict.c、src/backend/tsearch/dict_thesaurus.c、src/backend/tsearch/ts_parse.c、contrib/unaccent/unaccent.c 等の辞書利用箇所。
  • **呼び出し先**: FmgrInfo を介して辞書の lexize 関数を呼び出します。

関連データ構造

  • TSAnyCacheEntry: キャッシュエントリの共通基盤。
  • TSDictionaryCacheHash: 本構造体を格納するハッシュテーブル。
  • FmgrInfo: 関数呼び出しを高速化するためのインターフェース。

失敗モードと境界条件

  • **メモリ不足**: dictCtx の割り当て失敗時にエラーが発生する可能性があります。
  • **無効化**: カタログ変更時に isvalid が false になるため、クエリ実行中にキャッシュが再構築される可能性があります。

保守担当者向けメモ

  • 本構造体はバックエンドの生存期間中保持されます。
  • dictData に格納される辞書固有データは、キャッシュの再構築時に適切に解放・再確保される必要があります。
  • dictId の配置順序は、汎用的なキャッシュアクセスを維持するために厳守してください。
