geqo

定義

  • **種別**: function
  • **定義箇所**: src/backend/optimizer/geqo/geqo_main.c:71

目的

結合順序の決定というNP困難な最適化問題を、遺伝的アルゴリズム（GA）を用いて近似的に解き、最適なクエリ実行計画（RelOptInfo）を生成する。

シグネチャまたは形状

RelOptInfo *geqo(PlannerInfo *root, int number_of_rels, List *initial_rels)

入力と出力

  • **入力**:
  • root: プランナのコンテキスト情報。
  • number_of_rels: 結合対象となるリレーションの数。
  • initial_rels: 初期リレーションのリスト。
  • **出力**:
  • 最適化された結合順序に基づく RelOptInfo 構造体へのポインタ。

前提条件と不変条件

  • root を介してメモリ管理が行われるため、呼び出し元で適切なメモリコンテキストが維持されている必要がある。
  • コンパイル時に ERX, PMX, CX, PX, OX1, OX2 のいずれかの交叉アルゴリズムが定義されている必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: geqo_set_seed で乱数シードを設定し、gimme_pool_size 等でGAパラメータを決定する。 2. **個体群生成**: alloc_pool でメモリを確保し、random_init_pool で初期個体を生成後、sort_pool で適合度順にソートする。 3. **進化ループ**: 指定された世代数分、以下を繰り返す。

  • geqo_selection: 親個体を選択。
  • 交叉: コンパイル時に選択されたアルゴリズム（ERX, PMX等）で子個体を生成。
  • geqo_eval: 子個体の適合度（コスト）を評価。
  • spread_chromo: 子個体を個体群に挿入し、世代を更新。

    • 4. **終了**: 最良の個体からクエリ木を構築し、リソースを解放して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は本 excerpt 内では特定されていない。
  • **呼び出し先**: geqo_set_seed, gimme_pool_size, gimme_number_generations, alloc_pool, random_init_pool, sort_pool, alloc_chromo, geqo_selection, geqo_eval, spread_chromo 等のGEQOサブモジュール関数。

関連データ構造

  • PlannerInfo, RelOptInfo, Chromosome, Pool, Gene, Edge (ERX時), City (CX/PX/OX時)。

失敗モードと境界条件

  • コンパイル時に交叉アルゴリズムが一つも定義されていない場合、ビルドエラーが発生する。
  • GEQO_DEBUG 定義時には、世代ごとのログ出力やエッジ失敗時のログ出力が行われる。

保守担当者向けメモ

  • 本関数は root（PlannerInfo）のメモリコンテキストに依存しているため、メモリリークを防ぐには alloc_* 系関数と対になる解放処理の整合性が重要である。
  • 交叉アルゴリズムはマクロ定義により切り替わるため、特定のアルゴリズムの挙動を追う際はビルド構成を確認すること。
