am_tablesync_worker

定義

  • 種別: function
  • 定義箇所: src/include/replication/worker_internal.h:334

目的

現在のプロセスがテーブル同期（Tablesync）ワーカーとして動作しているかどうかを判定します。

シグネチャまたは形状

static inline bool am_tablesync_worker(void)

入力と出力

  • **入力**: なし（グローバル変数 MyLogicalRepWorker を参照）。
  • **出力**: 現在のプロセスがテーブル同期ワーカーであれば true、そうでなければ false を返します。

前提条件と不変条件

  • MyLogicalRepWorker が適切に初期化されている環境で呼び出される必要があります。

アルゴリズムまたは振る舞い

グローバル変数 MyLogicalRepWorker を引数として isTablesyncWorker マクロ（または関数）を評価し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/tablesync.c: tablesync.c 内の処理で利用。
  • src/backend/replication/logical/worker.c: ワーカーの役割に応じた条件分岐やエラー報告（pgstat_report_subscription_error）の判定に使用。
  • **呼び出し先**: isTablesyncWorker（マクロまたは関数）。

関連データ構造

  • MyLogicalRepWorker: 現在のワーカーの状態を保持するグローバル変数。

失敗モードと境界条件

  • MyLogicalRepWorker が NULL の場合、isTablesyncWorker の実装に依存しますが、セグメンテーションフォールトや予期しない動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数はインライン関数であり、頻繁に呼び出されるため、判定ロジックの変更はパフォーマンスに影響を与える可能性があります。
  • am_leader_apply_worker() 等の他の役割判定関数と併用されることが多いため、役割の排他性や重複については worker.c のロジックと整合性を保つ必要があります。
