ExecParallelCreateReaders

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execParallel.c:889

目的

並列クエリ実行において、各ワーカープロセスからタプルを受信するための TupleQueueReader を初期化し、リーダー側の ParallelExecutorInfo に格納する。

シグネチャまたは形状

void ExecParallelCreateReaders(ParallelExecutorInfo *pei)

入力と出力

  • **入力**: ParallelExecutorInfo *pei (並列実行のコンテキスト情報)
  • **出力**: なし（副作用として pei->reader に TupleQueueReader の配列が割り当てられ、各キューが初期化される）

前提条件と不変条件

  • pei->reader が NULL であること（二重初期化の防止）。
  • pei->pcxt->nworkers_launched が 0 より大きい場合、pei->tqueue および pei->pcxt->worker が適切に設定されていること。

アルゴリズムまたは振る舞い

1. pei->pcxt->nworkers_launched から起動されたワーカー数を確認する。 2. ワーカー数が 0 より大きい場合：

  • TupleQueueReader ポインタの配列を palloc で確保し、pei->reader に代入する。
  • 各ワーカーについてループし、以下の処理を行う：
  • shm_mq_set_handle を呼び出し、共有メモリキュー (pei->tqueue[i]) とワーカーのバックグラウンドワーカーハンドル (pei->pcxt->worker[i].bgwhandle) を紐付ける。
  • CreateTupleQueueReader を呼び出し、当該キューからタプルを読み取るためのリーダーオブジェクトを作成して pei->reader[i] に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeGather.c
  • src/backend/executor/nodeGatherMerge.c
  • **呼び出し先**:
  • shm_mq_set_handle
  • CreateTupleQueueReader
  • palloc

関連データ構造

  • ParallelExecutorInfo: 並列実行の全体状態を保持する構造体。
  • TupleQueueReader: ワーカーからのタプルストリームを読み取るための構造体。
  • shm_mq: プロセス間通信用の共有メモリキュー。

失敗モードと境界条件

  • nworkers_launched が 0 の場合、リーダーの初期化はスキップされ、pei->reader は NULL のままとなる。
  • メモリ不足時には palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は Gather や GatherMerge ノードの初期化フェーズで呼び出される。
  • shm_mq_set_handle によるハンドル設定は、ワーカーがタプルを送信する前に完了している必要がある。
