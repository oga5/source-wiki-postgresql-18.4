readTimeLineHistory

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/timeline.c:75

目的

指定されたタイムラインID（TLI）に対応する履歴ファイルを読み込み、TimeLineHistoryEntry 構造体のリストとして再構築する。

シグネチャまたは形状

List *readTimeLineHistory(TimeLineID targetTLI)

入力と出力

  • **入力**: targetTLI (読み込む対象のタイムラインID)
  • **出力**: List * (最新のタイムラインを先頭とする TimeLineHistoryEntry のリスト)

前提条件と不変条件

  • タイムライン1は履歴ファイルを持たない特別なIDとして扱われる。
  • 履歴ファイル内のタイムラインIDは昇順である必要がある。
  • 履歴ファイル内の各タイムラインIDは、対象の targetTLI 未満である必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: targetTLI が1の場合、単一の空エントリを返して終了する。 2. **ファイル取得**: アーカイブリカバリ時は RestoreArchivedFile を使用し、それ以外は TLHistoryFilePath でパスを特定する。 3. **パース**:

  • ファイルを1行ずつ読み込み、<tli> <switchpoint_hi>/<switchpoint_lo> 形式を解析する。
  • # で始まる行や空行は無視する。
  • 読み込んだ各行から TimeLineHistoryEntry を作成し、リストの先頭に挿入する（lcons）。

    • 4. **終端処理**: 履歴ファイルの末尾（最新のタイムライン）に対応するエントリをリストの先頭に追加する。 5. **アーカイブ対応**: アーカイブから取得したファイルであれば、将来の参照のために KeepFileRestoredFromArchive を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/access/transam/xlogutils.c
  • src/backend/backup/backup_manifest.c
  • src/backend/backup/basebackup_incremental.c
  • src/backend/postmaster/walsummarizer.c
  • **呼び出し先**:
  • TLHistoryFileName, TLHistoryFilePath
  • RestoreArchivedFile, KeepFileRestoredFromArchive
  • AllocateFile, FreeFile
  • pgstat_report_wait_start/end

  • ereport (エラーハンドリング)

関連データ構造

  • TimeLineHistoryEntry: タイムラインID、開始LSN、終了LSNを保持する構造体。
  • List: PostgreSQLの標準的なリスト構造。

失敗モードと境界条件

  • **ファイル不在**: errno が ENOENT であれば、親を持たないものとして空エントリを返す。それ以外は FATAL エラー。
  • **構文エラー**: 履歴ファイルの形式が不正（数値でない、フィールド不足）な場合、FATAL エラー。
  • **順序不正**: タイムラインIDが昇順でない、または targetTLI 以上のIDが含まれる場合、FATAL エラー。

保守担当者向けメモ

  • 履歴ファイルはタブ区切りのテキスト形式である。
  • リストは最新のタイムラインが先頭に来るように構築されるため、検索時は先頭から順に走査することが想定されている。
  • アーカイブから取得したファイルは、リカバリ後に pg_wal へ保存されることで整合性が維持される。
