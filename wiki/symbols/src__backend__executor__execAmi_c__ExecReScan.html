ExecReScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execAmi.c:76

目的

プランノードの状態を初期化し、パラメータ変更に伴う再実行を可能にする。

シグネチャまたは形状

void ExecReScan(PlanState *node)

入力と出力

  • **入力**: PlanState *node（再スキャン対象のプランノード状態）
  • **出力**: なし（副作用としてノードの状態をリセットする）

前提条件と不変条件

  • node->instrument が存在する場合、InstrEndLoop を呼び出して実行統計を終了させる。
  • node->chgParam が設定されている場合、変更されたパラメータセットを子ノード（initPlan, subPlan, outerPlan, innerPlan）へ伝播させる必要がある。

アルゴリズムまたは振る舞い

1. 実行統計（instrument）の終了処理を行う。 2. node->chgParam が非空の場合、UpdateChangedParamSet を用いて、関連する initPlan、subPlan、および outerPlan/innerPlan に対して変更を伝播させる。 3. node->ps_ExprContext が存在する場合、ReScanExprContext を呼び出して式評価コンテキストをリセットする。 4. nodeTag(node) に基づき、各ノードタイプ専用の ExecReScan<Node> 関数（例: ExecReScanSeqScan）を呼び出し、ノード固有の再スキャン処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execMain.c
  • src/backend/executor/execAsync.c
  • src/backend/executor/execProcnode.c
  • src/backend/executor/nodeAgg.c
  • src/backend/executor/nodeAppend.c
  • src/backend/executor/nodeBitmapAnd.c
  • （その他、プランノードの再実行を必要とする各ノード実装）
  • **呼び出し先**:
  • UpdateChangedParamSet
  • ExecReScanSetParamPlan
  • ReScanExprContext
  • 各種 ExecReScan<Node> 関数（ExecReScanSeqScan, ExecReScanHashJoin 等）

関連データ構造

  • PlanState: 実行中のプランノードの状態を保持する構造体。
  • SubPlanState: サブプランの状態を保持する構造体。
  • ExprContext: 式評価のためのコンテキスト。

失敗モードと境界条件

  • node が NULL の場合の挙動は定義されていない。
  • chgParam の伝播において、InitPlan 間の依存関係が複雑な場合、現在の実装では順序依存性がある（リストの先頭から順に処理される）。

保守担当者向けメモ

  • 新しいプランノードを追加した際は、本関数の switch 文に必ず新しいノードタイプ用のケースを追加すること。
  • ExecReScan は再帰的に子ノードの状態を更新するため、プランツリーの構造変更時には注意が必要である。
