fix_opfuncids_walker

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/nodeFuncs.c:1843

目的

ノードツリーを走査し、演算子式（OpExpr 等）に含まれる演算子関数 ID（opfuncid）が未設定の場合に、システムカタログを参照して適切な ID を設定する。

シグネチャまたは形状

static bool fix_opfuncids_walker(Node *node, void *context)

入力と出力

  • **入力**:
  • node: 走査対象のノード。
  • context: 走査の文脈（本関数では使用されない）。
  • **出力**:
  • bool: 走査を継続する場合は false、中断する場合は true を返す。

前提条件と不変条件

  • node が NULL の場合は直ちに false を返す。
  • 演算子に関連するノード型（OpExpr, DistinctExpr, NullIfExpr, ScalarArrayOpExpr）に対しては、適切な設定関数を呼び出す。

アルゴリズムまたは振る舞い

1. node が NULL ならば false を返して終了する。 2. ノード型を判定し、以下の処理を行う：

  • OpExpr, DistinctExpr, NullIfExpr: set_opfuncid を呼び出し、演算子関数 ID を設定する。
  • ScalarArrayOpExpr: set_sa_opfuncid を呼び出し、演算子関数 ID を設定する。

    • 3. expression_tree_walker を再帰的に呼び出し、ツリー全体を走査する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • fix_opfuncids（src/backend/nodes/nodeFuncs.c 内）から呼び出される。
  • **呼び出し先**:
  • set_opfuncid
  • set_sa_opfuncid
  • expression_tree_walker

関連データ構造

  • Node: 走査対象の汎用ノード。
  • OpExpr, DistinctExpr, NullIfExpr, ScalarArrayOpExpr: 演算子関数 ID を保持する具体的なノード型。

失敗モードと境界条件

  • node が NULL の場合、安全に処理を終了する。
  • 演算子関数 ID が解決できない場合、set_opfuncid 等の内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は expression_tree_walker を利用した再帰的な走査を行うため、スタックオーバーフローの可能性がある深いツリー構造には注意が必要である。
  • 新しい演算子関連のノード型が追加された場合は、本関数の switch（または if-else）ブロックにケースを追加すること。
