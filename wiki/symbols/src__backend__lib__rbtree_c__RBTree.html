RBTree

定義

  • 種別: struct
  • 定義箇所: src/backend/lib/rbtree.c:41

目的

PostgreSQLにおける汎用的な赤黒木（平衡二分探索木）の制御構造体です。メモリ管理や比較ロジックを呼び出し側に委譲する侵入型設計を採用し、$O(\lg n)$ の検索・挿入・削除性能を提供します。

シグネチャまたは形状

struct RBTree
{
    RBTNode    *root;           /* ルートノード、空の場合はRBTNIL */
    Size        node_size;      /* ノードのサイズ */
    rbt_comparator comparator;  /* 比較関数 */
    rbt_combiner combiner;      /* 重複キー挿入時の結合関数 */
    rbt_allocfunc allocfunc;    /* メモリ確保関数 */
    rbt_freefunc freefunc;      /* メモリ解放関数 */
    void       *arg;            /* 各関数に渡される引数 */
};

入力と出力

  • **入力**: rbt_create を通じて初期化され、ノードサイズおよび各種コールバック関数が登録されます。
  • **出力**: 木のルートポインタを保持し、各操作関数（rbt_insert, rbt_delete, rbt_find 等）の第一引数として木の状態を伝播します。

前提条件と不変条件

  • node_size、comparator、combiner、allocfunc、freefunc、arg は rbt_create による初期化後、変更不可（定数）として扱われます。
  • RBTNIL は静的な番兵として使用され、NULLの代わりに境界条件の判定に用いられます。
  • 侵入型設計のため、RBTNode は呼び出し側が定義する構造体の先頭メンバである必要があります。

アルゴリズムまたは振る舞い

  • **平衡維持**: 挿入および削除操作において、回転（Rotation）と色変更を行い、赤黒木の不変条件を維持します。
  • **抽象化**: データの比較やメモリ操作は、構造体に登録されたコールバック関数を介して実行されます。
  • **走査**: rbt_begin_iterate 等のイテレータ関数により、スタックを使用せずに親ポインタを辿る手法で昇順・降順の走査が可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: rbt_create（初期化）、rbt_insert（挿入）、rbt_delete（削除）、rbt_find（検索）などの関数が本構造体を操作します。
  • **呼び出し先**: 構造体内に保持された comparator、combiner、allocfunc、freefunc などの関数ポインタを介して、呼び出し側が提供したロジックを実行します。

関連データ構造

  • RBTNode: 木の各ノードの先頭に配置されるヘッダー構造体。
  • RBTNIL: 葉ノードを表す静的な番兵。

失敗モードと境界条件

  • **メモリ管理**: freefunc はノード自体のメモリ解放のみを想定しており、ノード内のサブデータまで再帰的に解放する設計にはなっていません。
  • **重複キー**: rbt_insert 時にキーが重複する場合、combiner コールバックが呼び出されます。

保守担当者向けメモ

  • 本構造体は src/include/lib/rbtree.h と対になっており、利用者はヘッダーファイルを参照してコールバック関数を定義する必要があります。
  • 侵入型設計であるため、メモリ確保・解放の責任はすべて呼び出し側にあります。
