setup_background_workers

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_shm_mq/setup.c:173

目的

テスト用バックグラウンドワーカーの登録および管理用オブジェクトの初期化を行います。

シグネチャまたは形状

static worker_state *setup_background_workers(int nworkers, dsm_segment *seg);

入力と出力

  • **入力**:
  • nworkers: 起動するワーカーの数。
  • seg: ワーカーがアタッチする動的共有メモリ（DSM）セグメント。
  • **出力**:
  • worker_state*: 登録されたワーカーのハンドルを保持する構造体へのポインタ。

前提条件と不変条件

  • メモリコンテキストの切り替えにより、worker_state およびその内部の BackgroundWorkerHandle は TopTransactionContext に割り当てられます。これにより、トランザクション終了後も生存期間が維持されます。
  • on_dsm_detach フックとして cleanup_background_workers が登録され、セグメント解放時にワーカーが停止されることが保証されます。

アルゴリズムまたは振る舞い

1. TopTransactionContext 上に worker_state オブジェクトを確保します。 2. on_dsm_detach を設定し、異常終了時やセグメント解放時にワーカーがクリーンアップされるよう予約します。 3. BackgroundWorker 構造体を初期化します。ライブラリ名に test_shm_mq、関数名に test_shm_mq_main を指定し、DSMセグメントハンドルを引数として渡します。 4. nworkers 分だけ RegisterDynamicBackgroundWorker を呼び出し、ワーカーを登録します。 5. 登録したワーカーのハンドルを worker_state に格納して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_shm_mq/setup.c 内の関数（test_shm_mq_setup からの呼び出しが確認されています）。
  • **呼び出し先**: RegisterDynamicBackgroundWorker, on_dsm_detach, MemoryContextAlloc, MemoryContextSwitchTo 等のPostgreSQL内部API。

関連データ構造

  • worker_state: ワーカーのハンドル配列を保持する構造体。
  • BackgroundWorker: ワーカーの起動設定を保持する構造体。
  • dsm_segment: 共有メモリセグメント。

失敗モードと境界条件

  • RegisterDynamicBackgroundWorker が失敗した場合、ERRCODE_INSUFFICIENT_RESOURCES エラーを発生させます。これは max_worker_processes の上限に達している場合に発生します。

保守担当者向けメモ

  • ワーカーの生存期間を管理するため、メモリコンテキストの操作が必須です。コンテキストの変更を忘れると、on_dsm_detach フック実行前にメモリが解放されるリスクがあります。
  • bgw_restart_time は BGW_NEVER_RESTART に設定されており、ワーカーは一度終了すると再起動しません。
