ReadBufferBI

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/hio.c:87

目的

BulkInsertState を活用して、ヒープリレーションのバッファ読み込みを最適化します。大量挿入時に同一ページへの連続アクセスが発生する場合、バッファの再利用を行うことで効率を高めます。

シグネチャまたは形状

static Buffer ReadBufferBI(Relation relation, BlockNumber targetBlock,
                           ReadBufferMode mode, BulkInsertState bistate)

入力と出力

  • **入力**:
  • relation: 対象のリレーション。
  • targetBlock: 読み込み対象のブロック番号。
  • mode: バッファ読み込みモード（ReadBufferMode）。
  • bistate: 大量挿入状態を保持する構造体。
  • **出力**:
  • Buffer: 確保されたバッファのハンドル。

前提条件と不変条件

  • bistate が NULL の場合、通常の ReadBufferExtended と同等の挙動をとります。
  • bistate が提供されている場合、bistate->current_buf に保持されたバッファが targetBlock と一致すれば、それを再利用します。
  • RBM_ZERO_AND_LOCK および RBM_ZERO_AND_CLEANUP_LOCK モードは、リレーション拡張時に使用されるため、既存バッファの再利用パスには到達しません。

アルゴリズムまたは振る舞い

1. **通常モード**: bistate が NULL ならば ReadBufferExtended を呼び出し終了します。 2. **キャッシュ確認**: bistate->current_buf が有効かつ targetBlock と一致する場合、そのバッファの参照カウントを増やして返却します。 3. **キャッシュ更新**: 一致しない場合、現在の bistate->current_buf を解放し、ReadBufferExtended を用いて新しいバッファを読み込みます。 4. **状態保存**: 新しく読み込んだバッファの参照カウントを増やし、bistate->current_buf に格納して次回の挿入に備えます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/hio.c 内の関数（RelationGetBufferForTuple 等から呼び出されることが確認されています）。
  • **呼び出し先**: ReadBufferExtended, BufferGetBlockNumber, IncrBufferRefCount, ReleaseBuffer。

関連データ構造

  • Buffer: バッファマネージャが管理するバッファハンドル。
  • BulkInsertState: 大量挿入時のバッファ戦略と現在のバッファを保持する構造体。
  • Relation: リレーション記述子。

失敗モードと境界条件

  • bistate が提供されている場合、targetBlock がキャッシュと一致しないと、既存のバッファが解放され、新しいバッファの読み込みが発生します。
  • 読み込み対象のブロックが物理的に存在しない場合、ReadBufferExtended の挙動に従いエラーまたは新規ブロック作成が行われます。

保守担当者向けメモ

  • 本関数は BulkInsertState の状態を更新するため、呼び出し側で bistate のライフサイクルを適切に管理する必要があります。
  • mode にロックを伴うフラグを指定する場合、キャッシュ再利用ロジックとの整合性に注意してください。
