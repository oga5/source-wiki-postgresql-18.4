ReleaseSysCache

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/syscache.c:268

目的

システムカタログキャッシュから取得したタプルの参照カウントを減らし、必要に応じてキャッシュエントリを解放可能にするために使用されます。

シグネチャまたは形状

void ReleaseSysCache(HeapTuple tuple)

入力と出力

  • **入力**: tuple (HeapTuple) - 解放対象となる、システムキャッシュから取得されたタプル。
  • **出力**: なし。

前提条件と不変条件

  • 引数 tuple は、SearchSysCache 等のシステムキャッシュ検索関数によって返された有効なタプルである必要があります。
  • 本関数を呼び出すことで、呼び出し側は当該タプルへの参照を放棄したものとみなされます。

アルゴリズムまたは振る舞い

1. 内部的に ReleaseCatCache(tuple) を呼び出します。 2. catcache.c が管理する参照カウントのデクリメント処理に委譲し、参照カウントがゼロになった場合にキャッシュエントリの解放を試みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blvalidate.c
  • contrib/postgres_fdw/connection.c
  • contrib/postgres_fdw/deparse.c
  • contrib/sepgsql/dml.c
  • contrib/sepgsql/proc.c
  • contrib/sepgsql/relation.c
  • contrib/tcn/tcn.c
  • **呼び出し先**:
  • ReleaseCatCache (定義箇所不明、catcache.c 内の関数と推測される)

関連データ構造

  • HeapTuple: PostgreSQLのヒープタプル構造体。

失敗モードと境界条件

  • 無効なポインタや、キャッシュ由来ではないタプルを渡した場合の動作は未定義です。
  • 既に解放済みのタプルに対して本関数を呼び出した場合の挙動については、本ソース excerpt からは判断できません。

保守担当者向けメモ

  • 本関数は catcache.c のラッパーです。キャッシュ管理の根本的なロジック変更は catcache.c を参照してください。
  • SearchSysCacheCopy で取得したタプルはキャッシュの直接参照ではないため、本関数ではなく heap_freetuple 等で解放する必要があります。混同しないよう注意してください。
