pgstat_get_entry_ref_cached

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_shmem.c:400

目的

統計エントリへの参照（PgStat_EntryRef）をローカルキャッシュから取得、または初期化します。共有メモリ上のハッシュテーブル検索を最適化し、キャッシュミス時のメモリ割り当てを管理します。

シグネチャまたは形状

static bool pgstat_get_entry_ref_cached(PgStat_HashKey key, PgStat_EntryRef **entry_ref_p)

入力と出力

  • **入力**:
  • key: 統計エントリを特定するための PgStat_HashKey。
  • entry_ref_p: 取得した PgStat_EntryRef 構造体へのポインタを格納する出力先。
  • **出力**:
  • bool: キャッシュが有効（既存のエントリが正しく参照されている）な場合は true、それ以外（新規作成または再初期化が必要）の場合は false を返します。

前提条件と不変条件

  • pgStatEntryRefHash が初期化されている必要があります。
  • キャッシュヒット時、entry_ref->shared_stats は 0xdeadbeef のマジックナンバーを持ち、かつ参照カウントが 0 より大きいことが保証されます。

アルゴリズムまたは振る舞い

1. pgstat_entry_ref_hash_insert を呼び出し、ローカルキャッシュから指定キーのエントリを取得または作成します。 2. エントリが新規作成された場合、またはキャッシュ内の entry_ref が空の場合は、pgStatSharedRefContext からメモリを割り当て、各フィールドを NULL で初期化して false を返します。 3. キャッシュ内に entry_ref が存在するが shared_stats が NULL の場合、再初期化が必要と判断し false を返します。 4. 上記以外（キャッシュヒット）の場合、アサーションで整合性を確認し、true を返します。 5. 最後に entry_ref_p にキャッシュ内の参照を格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: pgstat_get_entry_ref（src/backend/utils/activity/pgstat_shmem.c:501）から呼び出されます。
  • **呼び出し先**: pgstat_entry_ref_hash_insert および MemoryContextAlloc を呼び出します。

関連データ構造

  • PgStat_EntryRef: 統計エントリへの参照を保持する構造体。
  • PgStat_EntryRefHashEntry: ローカルキャッシュのハッシュテーブルエントリ。
  • pgStatEntryRefHash: バックエンドローカルの統計参照キャッシュ。
  • pgStatSharedRefContext: 参照構造体を割り当てるためのメモリコンテキスト。

失敗モードと境界条件

  • メモリ不足時: MemoryContextAlloc がエラーを発生させる可能性があります。
  • キャッシュミス: found が false となり、呼び出し元で共有メモリ側の検索や初期化処理が実行されます。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、キャッシュ挿入を先行させることでハッシュテーブルの重複検索を回避しています。
  • shared_stats が NULL の状態は、エントリが削除済みまたは未接続であることを示唆しており、呼び出し元での適切なハンドリングが必要です。
