test_slru_page_sync

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_slru/test_slru.c:143

目的

指定されたページ番号が属するSLRUセグメントファイルをディスクへ同期（フラッシュ）し、その結果を通知するテスト用関数です。

シグネチャまたは形状

Datum test_slru_page_sync(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_INT64(0) により、対象となるページ番号（int64）を受け取ります。
  • **出力**: PG_RETURN_VOID() により、戻り値なし（void）を返します。

前提条件と不変条件

  • 本関数はPostgreSQLのSQLインターフェースを通じて呼び出されることを前提としています。
  • TestSlruCtl が初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. 引数からページ番号を取得します。 2. ページ番号を SLRU_PAGES_PER_SEGMENT で除算し、対象となるセグメント番号（ftag.segno）を算出します。 3. SlruSyncFileTag を呼び出し、該当するセグメントファイル全体をディスクへフラッシュします。 4. elog(NOTICE, ...) を使用して、同期対象のセグメント番号とファイルパスをログに出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1(test_slru_page_sync) により登録）。
  • **呼び出し先**: SlruSyncFileTag（PostgreSQLのSLRUコアAPI）。

関連データ構造

  • TestSlruCtl: テスト用SLRUを制御する SlruCtlData 構造体。
  • FileTag: 同期対象のファイルを識別するための構造体。

失敗モードと境界条件

  • 指定されたページ番号が負の値である場合や、SLRUの範囲外である場合の挙動は SlruSyncFileTag の実装に依存します。
  • ファイルシステム上の同期処理が失敗した場合、SlruSyncFileTag 内部でエラーが発生し、トランザクションが中断される可能性があります。

保守担当者向けメモ

  • この関数はページ単位ではなく、セグメント単位（ファイル全体）でフラッシュを行う点に注意してください。
  • long_segment_names = true 設定下での動作を前提としています。
