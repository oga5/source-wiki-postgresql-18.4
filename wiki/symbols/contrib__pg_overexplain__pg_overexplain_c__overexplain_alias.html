overexplain_alias

定義

  • 種別: function
  • 定義箇所: contrib/pg_overexplain/pg_overexplain.c:685

目的

Alias 構造体の内容を整形し、EXPLAIN 出力の一部として ExplainState にプロパティとして追加する。

シグネチャまたは形状

static void overexplain_alias(const char *qlabel, Alias *alias, ExplainState *es)

入力と出力

  • **入力**:
  • qlabel: 出力時のプロパティ名。
  • alias: 表示対象の Alias 構造体へのポインタ。
  • es: EXPLAIN の状態を保持する ExplainState 構造体。
  • **出力**: なし（ExplainPropertyText を介して es に出力内容を書き込む）。

前提条件と不変条件

  • alias 引数は NULL であってはならない（Assert による検証）。

アルゴリズムまたは振る舞い

1. initStringInfo を用いて一時的な文字列バッファを初期化する。 2. alias->aliasname を quote_identifier で修飾し、バッファの先頭に書き込む。 3. alias->colnames リストを走査し、各カラム名を quote_identifier で修飾してカンマ区切りでバッファに追加する。 4. ExplainPropertyText を呼び出し、構築した文字列を qlabel をキーとして ExplainState に出力する。 5. pfree を呼び出し、バッファに使用したメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_overexplain/pg_overexplain.c 内の overexplain_range_table 関数（rte->alias および rte->eref の出力用として呼び出し）。
  • **呼び出し先**:
  • initStringInfo, appendStringInfo, appendStringInfoChar, quote_identifier, ExplainPropertyText, pfree（いずれも PostgreSQL 内部 API）。

関連データ構造

  • Alias: SQL のエイリアス情報を保持する構造体。
  • ExplainState: EXPLAIN コマンドの実行状態と出力先を管理する構造体。

失敗モードと境界条件

  • alias が NULL の場合、アサーション違反により異常終了する。
  • alias->colnames が空の場合、括弧内には何も出力されない（aliasname () の形式となる）。

保守担当者向けメモ

  • 本関数は Alias 構造体の表示に特化している。colnames の走査には foreach_node マクロを使用しており、PostgreSQL のノードリスト走査規約に従っている。
