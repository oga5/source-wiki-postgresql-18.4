pgoutput_prepare_txn

定義

  • 種別: function
  • 定義箇所: src/backend/replication/pgoutput/pgoutput.c:678

目的

論理レプリケーションにおける2フェーズコミットの「準備（PREPARE）」フェーズを処理し、トランザクションの準備情報をサブスクライバーへ送信する。

シグネチャまたは形状

static void pgoutput_prepare_txn(LogicalDecodingContext *ctx, ReorderBufferTXN *txn, XLogRecPtr prepare_lsn)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • txn: 準備対象のトランザクション情報。
  • prepare_lsn: 準備操作のWALログシーケンス番号。
  • **出力**: なし（副作用として出力ストリームへデータを書き出す）。

前提条件と不変条件

  • ctx および txn は有効なポインタであること。
  • 論理レプリケーションプロトコルに従い、適切な出力バッファが確保されていること。

アルゴリズムまたは振る舞い

1. OutputPluginUpdateProgress を呼び出し、進捗状況を更新する。 2. OutputPluginPrepareWrite を呼び出し、出力バッファを準備する。 3. logicalrep_write_prepare を呼び出し、トランザクションの準備情報をバイナリ形式で書き出す。 4. OutputPluginWrite を呼び出し、バッファの内容をサブスクライバーへ送信する。

呼び出し元と呼び出し先

  • **呼び出し元**: _PG_output_plugin_init 内で prepare_cb として登録されており、論理デコーディングのフレームワークから呼び出される。
  • **呼び出し先**:
  • OutputPluginUpdateProgress
  • OutputPluginPrepareWrite
  • logicalrep_write_prepare
  • OutputPluginWrite

関連データ構造

  • LogicalDecodingContext: 論理デコーディングのセッション状態を保持。
  • ReorderBufferTXN: トランザクションの変更内容や状態を保持。

失敗モードと境界条件

  • 出力バッファの書き込み失敗や、ネットワークエラーが発生した場合、基盤となる出力プラグイン関数がエラーをスローする可能性がある。

保守担当者向けメモ

  • 本関数は2フェーズコミットの準備段階のみを担う。実際のコミット処理は pgoutput_commit_txn 等の別のコールバックが担当する。
  • logicalrep_write_prepare は論理レプリケーションプロトコルの仕様に直接依存しているため、プロトコル変更時には注意が必要。
