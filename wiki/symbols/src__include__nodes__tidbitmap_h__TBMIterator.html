TBMIterator

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/tidbitmap.h:51

目的

TIDBitmap を走査するための統合イテレータ構造体です。プライベートイテレータと共有イテレータを単一のインターフェースで抽象化し、ビットマップ内の TID（Tuple Identifier）を順次取得するために使用されます。

シグネチャまたは形状

typedef struct TBMIterator
{
    bool        shared;
    union
    {
        TBMPrivateIterator *private_iterator;
        TBMSharedIterator *shared_iterator;
    }           i;
} TBMIterator;

入力と出力

  • **入力**: tbm_begin_iterate 関数により初期化されます。
  • **出力**: tbm_iterate 関数を通じて TBMIterateResult 構造体に走査結果が格納されます。

前提条件と不変条件

  • shared フラグが true の場合は i.shared_iterator が、false の場合は i.private_iterator が有効である必要があります。
  • tbm_end_iterate を呼び出すまで、イテレータに関連付けられたリソースは保持されます。

アルゴリズムまたは振る舞い

1. **初期化**: tbm_begin_iterate により、対象の TIDBitmap に応じたイテレータが生成され、本構造体に格納されます。 2. **走査**: tbm_iterate をループ内で呼び出すことで、ビットマップ内のタプル情報を順次取得します。 3. **終了**: tbm_end_iterate を呼び出し、イテレータが保持するメモリや共有リソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeBitmapHeapscan.c
  • src/include/access/relscan.h
  • **呼び出し先**:
  • tbm_begin_iterate (初期化)
  • tbm_iterate (走査)
  • tbm_end_iterate (終了)
  • tbm_exhausted (状態確認)

関連データ構造

  • TIDBitmap: 走査対象となるビットマップ構造体。
  • TBMPrivateIterator: プライベート走査用の内部構造体。
  • TBMSharedIterator: パラレルクエリ用の共有走査用内部構造体。
  • TBMIterateResult: 走査結果を保持する構造体。

失敗モードと境界条件

  • tbm_iterate は、ビットマップの走査が完了した際に false を返します。
  • 共有イテレータを使用する場合、dsa_area を介したメモリ管理に依存するため、共有メモリ領域の整合性が前提となります。

保守担当者向けメモ

  • 本構造体は union を使用して実装されているため、メンバへの直接アクセスは避け、必ず提供されている API（tbm_iterate 等）を介して操作してください。
  • 内部構造は tidbitmap.c で定義されています。
