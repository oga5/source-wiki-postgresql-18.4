entryGetItem

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginget.c:811

目的

GINスキャンにおいて、特定の GinScanEntry から advancePast で指定された位置よりも後の、次に合致するアイテム（TID）を取得し、entry->curItem に格納する。

シグネチャまたは形状

static void entryGetItem(GinState *ginstate, GinScanEntry entry, ItemPointerData advancePast)

入力と出力

  • **入力**:
  • ginstate: GINインデックスの状態管理構造体。
  • entry: 走査対象の GinScanEntry 構造体。
  • advancePast: この値より大きいTIDを検索するための基準点。
  • **出力**: なし（entry->curItem を更新し、必要に応じて entry->isFinished を設定する）。

前提条件と不変条件

  • entry->isFinished が false であること。
  • entry->curItem が無効であるか、または entry->curItem <= advancePast であること。

アルゴリズムまたは振る舞い

entry の保持するデータソースに応じて以下のいずれかの処理を行う： 1. **ビットマップ結果 (entry->matchBitmap)**: tbm_private_iterate を用いてビットマップを走査し、advancePast を超えるTIDを抽出する。Lossyページの場合はページ単位で返し、Exactページの場合はオフセットを走査する。 2. **ポストリスト (!BufferIsValid(entry->buffer))**: メモリ上のリストを直接走査し、advancePast を超える最初のアイテムを特定する。 3. **ポストツリー (BufferIsValid(entry->buffer))**: entryLoadMoreItems を呼び出してページ単位でデータをロードし、リストを走査する。

いずれのケースも、entry->reduceResult が有効な場合は dropItem を呼び出し、条件を満たさないアイテムをスキップする。

呼び出し元と呼び出し先

  • **呼び出し元**: keyGetItem（src/backend/access/gin/ginget.c 内の複数箇所）から呼び出される。
  • **呼び出し先**:
  • tbm_private_iterate, tbm_end_private_iterate, tbm_extract_page_tuple（ビットマップ処理時）
  • entryLoadMoreItems（ポストツリー処理時）
  • ginCompareItemPointers, dropItem

関連データ構造

  • GinScanEntry: スキャン中のエントリ状態を保持する構造体。
  • ItemPointerData: TID（タプル識別子）。
  • GinState: GINインデックスのメタデータ。

失敗モードと境界条件

  • **終了条件**: 走査対象のデータが尽きた場合、entry->isFinished が true に設定され、entry->curItem は無効化される。
  • **Lossyページ**: ページ全体が合致とみなされる場合、ItemPointerSetLossyPage が使用される。
  • **スキップ処理**: advancePast 以下のアイテムは無視され、次の有効なアイテムまで走査が継続される。

保守担当者向けメモ

  • 本関数は keyGetItem から呼び出され、インデックスの物理構造（ビットマップ、ポストリスト、ポストツリー）を抽象化してTIDを供給する。
  • reduceResult が設定されている場合、dropItem による追加のフィルタリングが行われるため、パフォーマンスに影響を与える可能性がある。
