r_case

定義

目的

ハンガリー語のステミング処理において、単語末尾の格変化接尾辞を特定し、条件を満たす場合に削除および正規化を行う。

シグネチャまたは形状

static int r_case(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z （ステミング環境の状態を保持する構造体）
  • 出力: int （成功時に 1、失敗または処理中断時に 0 または負の値を返す）

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • 処理対象の単語は z 内のバッファに格納されている。

アルゴリズムまたは振る舞い

1. z->ket を現在のカーソル位置 z->c に設定する。 2. a_4 配列（44個の接尾辞パターン）を用いて、末尾からマッチングを行う（find_among_b）。マッチしない場合は 0 を返して終了する。 3. z->bra を現在のカーソル位置に設定する。 4. r_R1 を呼び出し、マッチした接尾辞が有効な領域（R1）内にあるかを確認する。結果が 0 以下ならその値を返して終了する。 5. slice_del を呼び出し、マッチした接尾辞を削除する。結果が負ならその値を返して終了する。 6. r_v_ending を呼び出し、削除後の単語に対して母音の正規化処理を行う。結果を返して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_ISO_8859_2_hungarian.c:809 および src/backend/snowball/libstemmer/stem_UTF_8_hungarian.c:815 からの呼び出しが確認されている。
  • 呼び出し先: find_among_b, r_R1, slice_del, r_v_ending

関連データ構造

  • struct SN_env: ステミング環境を管理する構造体。
  • a_4: 格変化接尾辞を定義した among 構造体の配列。

失敗モードと境界条件

  • find_among_b がマッチに失敗した場合、0 を返す。
  • r_R1 または slice_del がエラー（負の値）を返した場合、そのエラーコードを伝播させる。
  • r_v_ending が 0 または負の値を返した場合、その値を返す。
