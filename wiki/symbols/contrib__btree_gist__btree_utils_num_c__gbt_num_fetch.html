gbt_num_fetch

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_utils_num.c:106

目的

GiST インデックスのリーフノードに格納された圧縮済みデータから、元のデータ型に対応する Datum を復元し、新しい GISTENTRY として返します。

シグネチャまたは形状

GISTENTRY *gbt_num_fetch(GISTENTRY *entry, const gbtree_ninfo *tinfo)

入力と出力

  • **入力**:
  • entry: 処理対象の GISTENTRY 構造体。
  • tinfo: データ型の情報（型識別子やサイズ）を保持する gbtree_ninfo 構造体。
  • **出力**:
  • 復元された Datum を保持する、動的に割り当てられた GISTENTRY へのポインタ。

前提条件と不変条件

  • tinfo->indexsize >= 2 * tinfo->size であること（アサーションにより検証）。
  • リーフエントリにおいて、格納されている下限値と上限値は同一であるとみなされます。

アルゴリズムまたは振る舞い

1. tinfo->t（型識別子）に基づき、switch 文で適切なデータ型を選択します。 2. entry->key が指すメモリ領域から、該当する型（bool, int16, int32, int64, Oid, float4, float8, DateADT, TimeADT, Timestamp, Cash）として値を読み取ります。 3. 読み取った値を Datum 型に変換します。 4. palloc を使用して新しい GISTENTRY 構造体をメモリ上に確保します。 5. gistentryinit マクロを使用して、元のエントリの関連情報（リレーション、ページ、オフセット）を継承しつつ、復元した Datum を設定して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下の各ファイル内の fetch 関数から呼び出されています。
  • btree_bool.c, btree_cash.c, btree_date.c, btree_enum.c, btree_float4.c, btree_float8.c, btree_int2.c, btree_int4.c, btree_int8.c, btree_interval.c, btree_macaddr.c, btree_macaddr8.c
  • **呼び出し先**:
  • palloc (PostgreSQL メモリ管理)
  • gistentryinit (GiST インデックス用マクロ)

関連データ構造

  • GISTENTRY: GiST インデックスのエントリを表現する構造体。
  • gbtree_ninfo: 数値型 GiST インデックスのメタデータ（型情報やサイズ）を保持する構造体。

失敗モードと境界条件

  • tinfo->t がサポートされていない型の場合、default 節により entry->key がそのまま Datum として扱われます。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 新しい数値型をサポートする場合、switch 文に新しい case を追加する必要があります。
  • 本関数はリーフノードでの使用を前提としており、下限値のみを抽出して返します。
