setRuleCheckAsUser

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteDefine.c:630

目的

ルール内のクエリツリーを走査し、権限チェック用のユーザーID（userid）を伝播させるために使用されます。

シグネチャまたは形状

void setRuleCheckAsUser(Node *node, Oid userid)

入力と出力

  • **入力**:
  • node: 走査対象となるクエリツリーまたはノードのポインタ。
  • userid: 権限チェックに使用するユーザーの OID。
  • **出力**: なし（副作用としてノード内のユーザーIDを更新）。

前提条件と不変条件

  • node は有効なクエリツリーまたはノード構造である必要があります。
  • 内部で呼び出される setRuleCheckAsUser_walker がノードの再帰的な走査をサポートしている必要があります。

アルゴリズムまたは振る舞い

1. 受け取った userid を setRuleCheckAsUser_walker に渡します。 2. setRuleCheckAsUser_walker を介して、指定されたノードツリー全体を再帰的に走査します。 3. 走査中に遭遇した権限チェックに関連するノードに対して、userid を適用します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/rewrite/rowsecurity.c: securityQuals および withCheckOptions の処理。
  • src/backend/utils/cache/relcache.c: ルールアクションおよびクエリの修飾子（qual）の処理。
  • **呼び出し先**:
  • setRuleCheckAsUser_walker（直接の内部呼び出し）。

関連データ構造

  • Node: 走査対象となる汎用ノード構造。
  • Oid: ユーザー識別子。

失敗モードと境界条件

  • node が NULL の場合、setRuleCheckAsUser_walker の実装に依存しますが、一般的には何も行われません。
  • 複雑なクエリツリーにおいて、走査対象外のノード型が存在する場合、それらは無視されます。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の走査ロジックは setRuleCheckAsUser_walker に集約されています。変更を加える際は、再帰的な走査ロジックの整合性に注意してください。
