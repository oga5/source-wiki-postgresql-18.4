pg_realloc_mul

定義

  • 種別: function
  • 定義箇所: src/common/fe_memutils.c:287

目的

メモリ確保サイズの乗算（s1 * s2）を伴う再割り当てを、整数オーバーフローおよびサイズ上限チェックを行いながら安全に実行する。

シグネチャまたは形状

void *pg_realloc_mul(void *p, Size s1, Size s2)

入力と出力

  • **入力**:
  • p: 再割り当て対象の既存メモリブロックへのポインタ（NULL の場合は新規確保として振る舞う）。
  • s1, s2: 確保するメモリサイズの計算に使用する乗数。
  • **出力**:
  • 再割り当て後のメモリブロックへのポインタ。
  • 確保失敗時はプロセスを終了する（pg_realloc の挙動に準拠）。

前提条件と不変条件

  • s1 * s2 の結果が SIZE_MAX / 2 を超えてはならない。これを超える要求は、物理メモリの有無に関わらずエラーとみなされる。
  • 内部で pg_mul_size_overflow を使用し、乗算時のオーバーフローを検出する。

アルゴリズムまたは振る舞い

1. pg_mul_size_overflow(s1, s2, &req) を呼び出し、乗算結果がオーバーフローするかを確認する。 2. 計算結果 req が SIZE_MAX / 2 を超えているかを確認する。 3. 上記のいずれかに該当する場合、mul_size_error(s1, s2) を呼び出し、エラー処理（プロセス終了）を行う。 4. チェックを通過した場合、pg_realloc(p, req) を呼び出してメモリを再割り当てし、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/common/fe_memutils.h にて pg_realloc_array マクロの内部実装として利用されている。
  • **呼び出し先**:
  • pg_mul_size_overflow (インライン展開)
  • mul_size_error
  • pg_realloc

関連データ構造

  • Size: メモリサイズを表す型（通常 size_t）。

失敗モードと境界条件

  • **整数オーバーフロー**: s1 * s2 が Size 型の最大値を超える場合、mul_size_error により終了する。
  • **サイズ制限超過**: 計算結果が SIZE_MAX / 2 を超える場合、mul_size_error により終了する。
  • **メモリ不足**: pg_realloc が失敗した場合、pg_realloc 内部のロジックによりプロセスが終了する。

保守担当者向けメモ

  • 本関数は fe_memutils.c 内で inline 的な役割を果たすため、パフォーマンスを考慮して mul_size のロジックを直接組み込んでいる。
  • pg_realloc_array マクロ経由で配列の再割り当てを行う際の標準的なインターフェースであるため、変更時は型安全性を維持すること。
