UpdateXmaxHintBits

定義

  • 種別: static void
  • 定義箇所: src/backend/access/heap/heapam.c:1990

目的

タプルの xmax トランザクションが終了した際、その状態をタプルのヒントビット（t_infomask）に反映させる。

シグネチャまたは形状

static void UpdateXmaxHintBits(HeapTupleHeader tuple, Buffer buffer, TransactionId xid)

入力と出力

  • tuple: 更新対象のタプルヘッダー。
  • buffer: タプルが格納されているバッファ。
  • xid: タプルの xmax に記録されているトランザクションID。
  • 出力: なし（副作用として buffer 内のタプルヘッダーのヒントビットを更新）。

前提条件と不変条件

  • HeapTupleHeaderGetRawXmax(tuple) が xid と一致すること。
  • tuple->t_infomask に HEAP_XMAX_IS_MULTI フラグが含まれていないこと。

アルゴリズムまたは振る舞い

1. タプルの t_infomask に HEAP_XMAX_COMMITTED または HEAP_XMAX_INVALID が既に設定されている場合は何もしない。 2. 未設定の場合、以下の条件でヒントビットを更新する：

  • HEAP_XMAX_IS_LOCKED_ONLY が偽であり、かつ TransactionIdDidCommit(xid) が真であれば、HEAP_XMAX_COMMITTED を設定する。
  • それ以外（ロックのみの場合、またはトランザクションがアボート済みの場合）は、HEAP_XMAX_INVALID を設定する。

    • 3. HeapTupleSetHintBits を呼び出し、バッファのダーティフラグ設定とヒントビットの書き込みを行う。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/heapam.c 内の複数箇所（行 2952, 3646, 5078 など）。
  • 呼び出し先: TransactionIdDidCommit（トランザクション状態確認）、HeapTupleSetHintBits（ヒントビット書き込み）。

関連データ構造

  • HeapTupleHeader: タプルヘッダー構造体。
  • Buffer: バッファ管理用ハンドル。
  • TransactionId: トランザクション識別子。

失敗モードと境界条件

  • Assert 文により、前提条件（xmax の一致、MultiXact ではないこと）が満たされない場合はプロセスが異常終了する。
  • TransactionIdDidCommit の結果に基づき、コミット済みか無効かのいずれかの状態へ遷移する。

保守担当者向けメモ

  • 本関数は MultiXact（複数トランザクションによるロック）を扱わないため、呼び出し側で HEAP_XMAX_IS_MULTI が立っていないことを保証する必要がある。
  • HeapTupleSetHintBits を介してバッファの更新を行うため、呼び出し側で適切なバッファロックが保持されていることが前提となる。
