reduce_unique_semijoins

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/analyzejoins.c:875

目的

クエリ内のセミジョイン（JOIN_SEMI）のうち、右側リレーションの一意性が結合条件によって保証されるものを特定し、不要なセミジョイン情報をプランナの管理リストから削除することで最適化を行います。

シグネチャまたは形状

void reduce_unique_semijoins(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root（クエリ全体のプランナ情報へのポインタ）
  • **出力**: なし（副作用として root->join_info_list を直接更新）

前提条件と不変条件

  • root->join_info_list に格納された SpecialJoinInfo を走査対象とする。
  • 対象となるセミジョインは、右側が単一のベースリレーション（min_righthand がシングルトン）である必要がある。
  • JOIN_SEMI 以外の結合タイプは無視される。

アルゴリズムまたは振る舞い

1. root->join_info_list を反復処理し、各 SpecialJoinInfo を検査する。 2. JOIN_SEMI かつ右側が単一リレーションであるかを確認する。 3. rel_supports_distinctness を呼び出し、対象リレーションがそもそも一意性をサポートしているか高速に判定する。 4. generate_join_implied_equalities を用いて、結合条件および等価クラス（EC）から導出される制約リストを生成する。 5. innerrel_is_unique を呼び出し、生成された制約下で右側リレーションが一意であるかを検証する。 6. 一意性が証明された場合、foreach_delete_current を用いて該当する SpecialJoinInfo を join_info_list から削除する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planmain.c:234
  • **呼び出し先**:
  • find_base_rel
  • rel_supports_distinctness

関連データ構造

  • PlannerInfo: プランニング中のクエリ全体の状態を保持。
  • SpecialJoinInfo: セミジョインや外部結合の制約情報を保持。
  • RelOptInfo: ベースリレーションの最適化情報を保持。

失敗モードと境界条件

  • 右側リレーションが複数リレーションから構成される場合、本関数では処理されずスキップされる。
  • rel_supports_distinctness で一意性が判定できない場合、または innerrel_is_unique で一意性が証明できない場合は、結合の削除は行われない。

保守担当者向けメモ

  • 本関数はセミジョインを内部結合へ格下げする前段階の最適化として機能する。
  • join_info_list の走査中に要素を削除するため、foreach_delete_current を使用した安全なリスト操作を行っている。
  • 結合条件の導出には generate_join_implied_equalities を使用しており、EC（等価クラス）の変更が本関数の判定結果に影響を与える可能性がある。
