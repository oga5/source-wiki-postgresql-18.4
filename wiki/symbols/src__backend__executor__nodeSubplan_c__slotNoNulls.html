slotNoNulls

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSubplan.c:781

目的

TupleTableSlot 内のすべての属性が NULL でないかどうかを判定する。

シグネチャまたは形状

static bool slotNoNulls(TupleTableSlot *slot)

入力と出力

  • **入力**: TupleTableSlot *slot（検査対象のタプルスロット）
  • **出力**: bool（スロット内の全属性が非 NULL であれば true、一つでも NULL が含まれていれば false を返す）

前提条件と不変条件

  • slot は有効な TupleTableSlot 構造体を指している必要がある。
  • slot->tts_tupleDescriptor が初期化されており、属性数（natts）が正しく設定されていること。

アルゴリズムまたは振る舞い

1. スロットのタプル記述子から属性数（ncols）を取得する。 2. 1番目の属性から ncols 番目の属性まで順にループする。 3. slot_attisnull(slot, i) を呼び出し、各属性が NULL かどうかを検査する。 4. NULL を検出した時点で直ちに false を返し終了する。 5. 全ての属性を検査し、NULL が見つからなかった場合に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeSubplan.c 内の複数箇所（行150、行610付近）。
  • **呼び出し先**: slot_attisnull（マクロまたはインライン関数）。

関連データ構造

  • TupleTableSlot: タプルデータを保持する実行エンジン内の標準的な構造体。

失敗モードと境界条件

  • slot が NULL の場合、あるいは tts_tupleDescriptor が適切に設定されていない場合はセグメンテーション違反を引き起こす可能性がある。
  • 属性数が 0 のスロットに対しては、ループが実行されず true を返す。

保守担当者向けメモ

  • 本関数は nodeSubplan.c 内のサブクエリ評価ロジックにおいて、NULL 値の有無による条件分岐の判定に使用されている。
  • 類似の関数として slotAllNulls が存在するため、用途（全 NULL か、非 NULL か）を混同しないよう注意すること。
