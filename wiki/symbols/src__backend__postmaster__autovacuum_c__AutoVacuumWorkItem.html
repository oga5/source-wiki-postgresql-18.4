AutoVacuumWorkItem

定義

  • **種別**: struct
  • **定義箇所**: src/backend/postmaster/autovacuum.c:263

目的

自動バキュームシステムにおいて、特定のデータベースやリレーションに対する非同期的な作業要求を保持・管理するために使用される構造体です。

シグネチャまたは形状

typedef struct AutoVacuumWorkItem
{
    AutoVacuumWorkItemType avw_type;
    bool        avw_used;       /* データが有効か */
    bool        avw_active;     /* 処理中か */
    Oid         avw_database;   /* 対象データベースOID */
    Oid         avw_relation;   /* 対象リレーションOID */
    BlockNumber avw_blockNumber;/* 対象ブロック番号 */
} AutoVacuumWorkItem;

入力と出力

本構造体は共有メモリ上の配列 av_workItems の要素として配置されます。

  • **入力**: perform_work_item 関数等を通じて、作業内容（タイプ、DB、リレーション、ブロック）が設定されます。
  • **出力**: avw_active フラグや各OIDフィールドを通じて、ワーカープロセスが実行すべき作業を特定します。

前提条件と不変条件

  • avw_used が true の場合のみ、他のフィールドの値が有効であるとみなされます。
  • avw_active は、現在その作業項目がワーカーによって処理中であることを示します。

アルゴリズムまたは振る舞い

  • 自動バキュームのランチャーおよびワーカーは、共有メモリ上の AutoVacuumShmem->av_workItems 配列を走査し、avw_used が有効な項目を特定します。
  • perform_work_item 関数は、この構造体を参照して具体的なバキュームやアナライズの作業を実行します。
  • autovac_report_workitem 関数は、作業項目の状態を更新するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/autovacuum.c 内の複数の箇所で、共有メモリ上の配列要素として参照されています（例: 2532行目, 3313行目）。
  • **呼び出し先**: perform_work_item および autovac_report_workitem が本構造体を引数として受け取ります。

関連データ構造

  • AutoVacuumShmemStruct: 本構造体の配列を保持する共有メモリ上の親構造体。
  • AutoVacuumWorkItemType: 作業の種類を定義する列挙型（本構造体の avw_type フィールドで使用）。

失敗モードと境界条件

  • NUM_WORKITEMS で定義された上限数を超える作業要求は保持できません。
  • 共有メモリ上のデータであるため、プロセス間での競合が発生する可能性があり、適切なロック（AutovacuumLock 等）の取得が前提となります。

保守担当者向けメモ

  • 本構造体は src/include/postmaster/autovacuum.h にも関連定義が存在します。
  • 作業項目の追加や削除を行う際は、必ず AutovacuumLock を介した排他制御を行ってください。
