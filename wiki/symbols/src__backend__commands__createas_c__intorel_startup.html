intorel_startup

定義

  • 種別: function
  • 定義箇所: src/backend/commands/createas.c:458

目的

CREATE TABLE AS (CTAS) および CREATE MATERIALIZED VIEW コマンド実行時のエグゼキュータ初期化処理。ターゲットとなるリレーションの物理作成、およびタプル挿入のための DestReceiver 状態のセットアップを行う。

シグネチャまたは形状

static void intorel_startup(DestReceiver *self, int operation, TupleDesc typeinfo)

入力と出力

  • **入力**:
  • self: DR_intorel 構造体へのポインタ（DestReceiver としてキャスト）。
  • operation: 実行される操作の種類。
  • typeinfo: 挿入されるタプルの記述子。
  • **出力**: なし（副作用として self 内のフィールドを初期化し、システムカタログへリレーションを登録する）。

前提条件と不変条件

  • self->into が設定されていること。
  • ターゲットリレーションに対して行レベルセキュリティ (RLS) が有効になっていないこと（有効な場合はエラー）。
  • RelationGetTargetBlock(intoRelationDesc) が InvalidBlockNumber であること。

アルゴリズムまたは振る舞い

1. **列定義の構築**: typeinfo から列名と型情報を取得し、CREATE TABLE 用の ColumnDef リストを作成する。列名リストが指定されている場合はそれを優先する。 2. **照合順序の検証**: 列の型が照合順序を保持する場合、照合順序が解決可能であることを確認する。 3. **リレーション作成**: create_ctas_internal を呼び出し、物理テーブルを作成する。 4. **RLS チェック**: check_enable_rls を実行し、RLS が有効な場合はエラーを投げる。 5. **マテリアライズドビューの更新**: マテリアライズドビューかつ WITH NO DATA でない場合、SetMatViewPopulatedState で状態を更新する。 6. **状態の保存**: myState にリレーション記述子、オブジェクトアドレス、コマンド ID、バルク挿入状態 (GetBulkInsertState) を格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/createas.c 内で DestReceiver の rStartup フィールドとして登録され、エグゼキュータから呼び出される。
  • **呼び出し先**: create_ctas_internal, table_open, check_enable_rls, SetMatViewPopulatedState, GetCurrentCommandId, GetBulkInsertState。

関連データ構造

  • DR_intorel: DestReceiver を継承した CTAS 専用の制御構造体。
  • IntoClause: CREATE TABLE AS の構文解析結果を保持する構造体。
  • TupleDesc: 挿入されるタプルのメタデータ。

失敗モードと境界条件

  • **列名不一致**: 指定された列名数が実際の列数と一致しない場合にエラー。
  • **照合順序未解決**: 照合順序が決定できない場合にエラー。
  • **RLS 有効**: RLS が有効なテーブルを作成しようとした場合にエラー。

保守担当者向けメモ

  • 本関数は CREATE TABLE AS と CREATE MATERIALIZED VIEW の両方をサポートする。
  • WITH NO DATA が指定されている場合、バルク挿入用の状態 (bistate) は初期化されない。
  • 挿入されるタプルは intorel_receive で処理され、最終的に intorel_shutdown でクリーンアップされる。
