pg_hmac_final

定義

  • 種別: function
  • 定義箇所: src/common/hmac.c:243

目的

HMAC 計算の最終段階を実行し、RFC 2104 に基づくメッセージ認証コードを生成します。

シグネチャまたは形状

int pg_hmac_final(pg_hmac_ctx *ctx, uint8 *dest, size_t len);

入力と出力

  • **入力**:
  • ctx: HMAC 計算状態を保持する pg_hmac_ctx 構造体へのポインタ。
  • dest: 計算結果を格納するバッファへのポインタ。
  • len: dest バッファのサイズ。
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時は -1 を返し、ctx->error にエラーコードを設定します。

前提条件と不変条件

  • ctx は pg_hmac_init によって初期化済みである必要があります。
  • ctx が NULL の場合、関数は即座に -1 を返します。

アルゴリズムまたは振る舞い

1. 内部ハッシュコンテキストを pg_cryptohash_final で確定させ、一時バッファ h に格納します。 2. 内部ハッシュ処理に失敗した場合、PG_HMAC_ERROR_INTERNAL を設定して終了します。 3. k_opad（外部パディング）と一時バッファ h を用いて、再度ハッシュ計算（H(K XOR opad, tmp)）を行います。 4. 最終的なハッシュ値を dest に書き込みます。 5. 使用した一時メモリ h を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth-scram.c
  • src/common/scram-common.c
  • src/interfaces/libpq/fe-auth-scram.c
  • **呼び出し先**:
  • pg_cryptohash_final
  • pg_cryptohash_init
  • pg_cryptohash_update
  • pg_cryptohash_error
  • ALLOC / FREE (マクロ)

関連データ構造

  • pg_hmac_ctx: HMAC 計算の状態を保持する不透明な構造体。

失敗モードと境界条件

  • ctx が NULL: -1 を返します。
  • メモリ割り当て失敗: PG_HMAC_ERROR_OOM を設定し -1 を返します。
  • 内部ハッシュ関数エラー: PG_HMAC_ERROR_INTERNAL を設定し、pg_cryptohash_error から取得した詳細情報を ctx->errreason に格納して -1 を返します。

保守担当者向けメモ

  • 本関数はフォールバック実装です。OpenSSL 等のライブラリが利用可能な環境では、src/common/hmac_openssl.c の実装が優先される場合があります。
  • ALLOC/FREE マクロは環境（バックエンド/フロントエンド）に応じて palloc/pfree または malloc/free に切り替わります。フロントエンドで使用する際はメモリリークに注意してください。
