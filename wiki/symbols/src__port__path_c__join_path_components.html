join_path_components

定義

  • 種別: function
  • 定義箇所: src/port/path.c:285

目的

2つのパスコンポーネント（head と tail）を結合し、必要に応じてディレクトリ区切り文字を挿入して単一のパス文字列を生成します。

シグネチャまたは形状

void join_path_components(char *ret_path, const char *head, const char *tail);

入力と出力

  • ret_path: 結合結果を格納するバッファ（MAXPGPATH 以上のサイズが必要）。
  • head: パスの先頭部分。
  • tail: パスの末尾に追加する部分。
  • 出力: ret_path に結合されたパスが書き込まれます。

前提条件と不変条件

  • ret_path は MAXPGPATH バイトの書き込みに耐えうる領域である必要があります。
  • head が ret_path と異なる場合、head の内容が ret_path にコピーされます。

アルゴリズムまたは振る舞い

1. ret_path が head と異なる場合、strlcpy を使用して head を ret_path にコピーします。 2. tail が空でない場合、以下の処理を行います。

  • head がドライブレターのみ、あるいは空でない場合、パスの連結部分に / を挿入します（skip_drive を使用して判定）。
  • snprintf を用いて、ret_path の末尾に /（必要な場合）と tail を連結します。
  • . や .. の解決は行わず、後続の canonicalize_path() に委ねます。

関連データ構造

  • MAXPGPATH: パス文字列の最大長を規定する定数。

失敗モードと境界条件

  • ret_path が十分なバッファサイズを持たない場合、snprintf により切り詰めが発生する可能性があります。
  • head が空の場合、tail がそのまま ret_path に書き込まれます。

保守担当者向けメモ

  • 本関数はパスの連結のみを行い、正規化は行いません。パスの安全性を確保するためには、呼び出し後に canonicalize_path() を適用することが推奨されます。
