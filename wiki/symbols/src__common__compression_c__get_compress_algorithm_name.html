get_compress_algorithm_name

定義

  • 種別: function
  • 定義箇所: src/common/compression.c:68

目的

圧縮アルゴリズムを示す列挙型 pg_compress_algorithm を、対応する人間が読み取り可能な文字列（"none", "gzip", "lz4", "zstd"）に変換する。

シグネチャまたは形状

const char *get_compress_algorithm_name(pg_compress_algorithm algorithm);

入力と出力

  • **入力**: pg_compress_algorithm 型の列挙値。
  • **出力**: アルゴリズム名を表す静的な文字列リテラル（const char *）。

前提条件と不変条件

  • 入力される algorithm は、PG_COMPRESSION_NONE, PG_COMPRESSION_GZIP, PG_COMPRESSION_LZ4, PG_COMPRESSION_ZSTD のいずれかである必要がある。

アルゴリズムまたは振る舞い

1. switch 文を用いて入力された列挙値を評価する。 2. 各ケースに対応する文字列を返却する。 3. 定義されていない値が渡された場合、Assert(false) を実行し、コンパイラ警告を回避するための "???" を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/compress_io.c
  • src/bin/pg_dump/pg_backup_archiver.c
  • src/common/compression.c（複数箇所）
  • **呼び出し先**:
  • 本関数は標準ライブラリや他の関数を呼び出さない。

関連データ構造

  • pg_compress_algorithm（列挙型）

失敗モードと境界条件

  • 定義外の列挙値が渡された場合、Assert が有効なビルドではアボートし、無効なビルドでは "???" を返す。

保守担当者向けメモ

  • 新しい圧縮アルゴリズムを pg_compress_algorithm に追加した際は、必ず本関数の switch 文にもケースを追加すること。default ケースを意図的に設けていないため、未対応の列挙値がある場合はコンパイル時に警告が発生する設計となっている。
