shm_mq_set_sender

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/storage/ipc/shm_mq.c:223

目的

共有メモリメッセージキュー（shm_mq）に対して、送信側プロセス（PGPROC）を登録する。

シグネチャまたは形状

void shm_mq_set_sender(shm_mq *mq, PGPROC *proc)

入力と出力

  • **入力**:
  • mq: 操作対象の共有メモリメッセージキューへのポインタ。
  • proc: 送信側として設定するプロセス構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • mq->mq_sender は、設定時に NULL である必要がある（Assert により検証）。
  • 内部状態の変更は mq->mq_mutex スピンロックによって保護される。

アルゴリズムまたは振る舞い

1. mq->mq_mutex を取得する。 2. mq->mq_sender が NULL であることを確認する。 3. mq->mq_sender に proc を代入する。 4. 現在の mq->mq_receiver をローカル変数に退避し、スピンロックを解放する。 5. 受信側プロセスが既に設定されている場合（receiver != NULL）、そのプロセスのラッチ（procLatch）をセットし、送信側が設定されたことを通知する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c
  • src/backend/executor/execParallel.c
  • src/backend/replication/logical/applyparallelworker.c
  • src/test/modules/test_shm_mq/setup.c
  • src/test/modules/test_shm_mq/worker.c
  • **呼び出し先**:
  • SpinLockAcquire / SpinLockRelease
  • SetLatch

関連データ構造

  • shm_mq: メッセージキューの制御ヘッダー。
  • PGPROC: PostgreSQL のプロセス状態を保持する構造体。

失敗モードと境界条件

  • mq->mq_sender が既に設定されている場合、アサーション違反によりプロセスが停止する可能性がある。
  • receiver がまだ設定されていない場合、通知は行われない。

保守担当者向けメモ

  • 本関数は送信側の登録のみを行い、受信側の登録は shm_mq_set_receiver が担当する。
  • 送信側が設定されたことを受信側に通知する仕組みは、非同期的なプロセス間通信の開始点となる。
