BuildQueryCompletionString

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/cmdtag.c:120

目的

SQLコマンドの実行完了時に、クライアントへ通知するための完了文字列（Command Tag）を構築する。

シグネチャまたは形状

Size BuildQueryCompletionString(char *buff, const QueryCompletion *qc, bool nameonly)

入力と出力

  • **入力**:
  • buff: 完了文字列を格納するバッファ（COMPLETION_TAG_BUFSIZE 以上のサイズが必要）。
  • qc: コマンドタグと処理行数を含む QueryCompletion 構造体。
  • nameonly: true の場合、コマンド名のみを生成する。
  • **出力**:
  • 戻り値: 生成された文字列の長さ（Size 型）。

前提条件と不変条件

  • buff は COMPLETION_TAG_BUFSIZE（64バイト）以上の領域を確保していること。
  • tagname は ASCII 文字列であることを前提としている。
  • taglen はバッファをオーバーランしないサイズ（COMPLETION_TAG_BUFSIZE - MAXINT8LEN - 4 以下）であること。

アルゴリズムまたは振る舞い

1. GetCommandTagNameAndLen を呼び出し、コマンド名とその長さを取得する。 2. コマンド名を buff にコピーする。 3. command_tag_display_rowcount が真かつ nameonly が偽の場合、処理行数を追記する。

  • CMDTAG_INSERT の場合は、プロトコル互換性のため、行数の前に 0 を挿入する。

    • 4. 文字列の末尾に \0 を付与する。 5. 生成した文字列の長さを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/dest.c:180
  • **呼び出し先**:
  • GetCommandTagNameAndLen
  • command_tag_display_rowcount
  • pg_ulltoa_n

関連データ構造

  • QueryCompletion: コマンドタグと処理行数を保持する構造体。
  • CommandTag: コマンドの種類を示す列挙型。

失敗モードと境界条件

  • バッファサイズ不足: taglen が規定の安全な長さを超える場合、Assert により停止する。
  • INSERT コマンド: 過去の OID 埋め込み仕様との互換性のため、常に 0 が付与される。

保守担当者向けメモ

  • 新しいコマンドタグを追加する場合は src/include/tcop/cmdtaglist.h を編集すること。
  • INSERT の完了タグに 0 が含まれるのは仕様であり、削除しないこと。
