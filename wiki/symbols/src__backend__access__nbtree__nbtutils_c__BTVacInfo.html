BTVacInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/access/nbtree/nbtutils.c:3492

目的

B-tree インデックスにおける VACUUM 操作の状態を共有メモリ上で管理するための構造体です。

シグネチャまたは形状

typedef struct BTVacInfo
{
    BTCycleId    cycle_ctr;        /* 最近割り当てられたサイクルID */
    int          num_vacuums;      /* 現在アクティブなVACUUMの数 */
    int          max_vacuums;      /* vacuums[] 配列の確保済みサイズ */
    BTOneVacInfo vacuums[FLEXIBLE_ARRAY_MEMBER];
} BTVacInfo;

入力と出力

  • 本構造体は共有メモリ領域として確保され、BTreeShmemInit 等のプロセスを通じてシステム全体で共有されます。

前提条件と不変条件

  • vacuums メンバは FLEXIBLE_ARRAY_MEMBER を使用しており、構造体自体のサイズは動的に決定されます。
  • 共有メモリ上の状態として管理されるため、アクセスには適切なロック制御が必要です。

アルゴリズムまたは振る舞い

  • インデックスの VACUUM 処理において、現在実行中の VACUUM プロセスの追跡と、サイクル管理のための識別子（cycle_ctr）を保持します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/nbtree/nbtutils.c 内の共有メモリ初期化処理（ShmemInitStruct）にて、"BTree Vacuum State" という名称で初期化されます。
  • 呼び出し先: 構造体内部の vacuums 配列は BTOneVacInfo 型の要素を保持します。

関連データ構造

  • BTOneVacInfo: 各 VACUUM プロセスの詳細情報を保持する構造体。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、システムは初期化できません。
  • max_vacuums を超える VACUUM プロセスが同時に発生した場合の挙動については、本構造体定義のみからは判断できません。

保守担当者向けメモ

  • 本構造体は offsetof(BTVacInfo, vacuums) を用いてサイズ計算が行われています。共有メモリのレイアウトを変更する際は、この計算式が整合しているか確認してください。
