word

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regcomp.c:1476

目的

正規表現における「単語境界（word boundary）」の制約を NFA（非決定性有限オートマトン）上に構築する。

シグネチャまたは形状

static void word(struct vars *v, int dir, struct state *lp, struct state *rp)

入力と出力

  • v: コンパイル状態を保持する vars 構造体へのポインタ。
  • dir: 境界の方向。AHEAD または BEHIND を指定する。
  • lp, rp: NFA の開始状態および終了状態を表す state 構造体へのポインタ。
  • 出力: なし（副作用として NFA に遷移を追加）。

前提条件と不変条件

  • dir は必ず AHEAD または BEHIND である必要がある（assert による検証あり）。

アルゴリズムまたは振る舞い

1. v->nfa 内の指定された状態 lp から rp へ、v->wordchrs（単語構成文字セット）に基づく遷移を複製する。 2. cloneouts 関数を呼び出し、dir に応じて単語境界の遷移を NFA に追加する。

呼び出し元と呼び出し先

  • 呼び出し元: ソースコード内の直接的な呼び出し元は、提供された抜粋からは特定できません。
  • 呼び出し先: cloneouts（src/backend/regex/regc_nfa.c 内の関数と推測される）。
  • 注意: ツールが提示した contrib/ 配下の word 関連の参照は、本関数とは異なるシンボルを指している可能性が高く、本関数の呼び出し元とは見なせません。

関連データ構造

  • struct vars: コンパイルコンテキスト。
  • struct state: NFA の状態ノード。
  • v->nfa: 構築対象の NFA。
  • v->wordchrs: 単語構成文字を定義するカラーマップまたは文字セット。

失敗モードと境界条件

  • dir が AHEAD または BEHIND 以外の場合、アサーション失敗によりプログラムが終了する。

保守担当者向けメモ

  • 本関数は正規表現の \m や \M といった単語境界メタ文字の処理に使用されます。
  • cloneouts を介して NFA を操作するため、NFA の状態遷移の整合性に依存しています。
