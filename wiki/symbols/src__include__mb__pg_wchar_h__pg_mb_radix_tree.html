pg_mb_radix_tree

定義

  • 種別: struct
  • 定義箇所: src/include/mb/pg_wchar.h:447

目的

UTF-8と他エンコーディング間の文字コード変換を高速化するための基数木（Radix Tree）データ構造です。

シグネチャまたは形状

typedef struct {
    const uint16 *chars16;
    const uint32 *chars32;
    uint32 b1root; uint8 b1_lower; uint8 b1_upper;
    uint32 b2root; uint8 b2_1_lower; uint8 b2_1_upper; uint8 b2_2_lower; uint8 b2_2_upper;
    uint32 b3root; uint8 b3_1_lower; uint8 b3_1_upper; uint8 b3_2_lower; uint8 b3_2_upper; uint8 b3_3_lower; uint8 b3_3_upper;
    uint32 b4root; uint8 b4_1_lower; uint8 b4_1_upper; uint8 b4_2_lower; uint8_b4_2_upper; uint8 b4_3_lower; uint8 b4_3_upper; uint8 b4_4_lower; uint8 b4_4_upper;
} pg_mb_radix_tree;

入力と出力

  • **入力**: 1〜4バイトの入力バイト列。
  • **出力**: chars16 または chars32 配列から参照される変換後の値。

前提条件と不変条件

  • chars16 または chars32 のいずれか一方が使用されます（値の幅に応じて選択）。
  • 各バイト数（1〜4バイト）ごとに、有効な入力範囲を示す lower および upper 境界値が定義されています。

アルゴリズムまたは振る舞い

  • 入力されたバイト列を基数木の階層（1〜4バイト）に沿って探索し、対応する変換先コードポイントを chars 配列から取得します。
  • bXroot フィールドは、各階層におけるテーブルのオフセットを保持します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の痕跡）**:
  • src/backend/utils/mb/conv.c
  • src/backend/utils/mb/conversion_procs/utf8_and_iso8859/utf8_and_iso8859.c
  • src/backend/utils/mb/conversion_procs/utf8_and_win/utf8_and_win.c
  • src/include/mb/pg_wchar.h 内の関数プロトタイプ
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • pg_wchar: PostgreSQL内部の文字型。
  • pg_wchar_tbl: エンコーディング変換関数を保持するテーブル。

失敗モードと境界条件

  • 入力バイトが各階層の lower から upper の範囲外である場合、変換は失敗または未定義となります。
  • chars16 と chars32 の選択が不適切な場合、メモリ参照エラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は変換テーブルの静的な定義に使用されます。新しいエンコーディング変換を追加する際は、この基数木構造に従ったテーブル作成が必要です。
  • 呼び出し元情報の詳細は、src/backend/utils/mb/conv.c の実装を参照してください。
