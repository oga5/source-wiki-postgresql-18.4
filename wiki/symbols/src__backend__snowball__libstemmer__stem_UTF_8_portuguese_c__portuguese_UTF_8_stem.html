portuguese_UTF_8_stem

定義

  • **種別**: function
  • **定義箇所**: src/backend/snowball/libstemmer/stem_UTF_8_portuguese.c:886

目的

UTF-8 エンコーディングされたポルトガル語テキストに対し、接尾辞の削除を行うことで単語の語幹を抽出する。PostgreSQL の全文検索エンジンにおける正規化処理のメインエントリーポイントとして機能する。

シグネチャまたは形状

int portuguese_UTF_8_stem(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z（Snowball ランタイムの状態を保持する構造体。処理対象の文字列データを含む）
  • **出力**: int（成功時に 1、エラー発生時に負の値を返す）

前提条件と不変条件

  • z は有効な SN_env 構造体へのポインタであること。
  • 内部処理において z->c（現在のカーソル位置）や z->l（文字列長）などの状態が適切に更新される。

アルゴリズムまたは振る舞い

1. **前処理**: r_prelude を呼び出し、アクセント記号等の正規化を行う。 2. **領域分割**: r_mark_regions を呼び出し、RV, R1, R2 領域を特定する。 3. **接尾辞削除**:

  • r_standard_suffix（名詞・形容詞）または r_verb_suffix（動詞）を順次試行する。
  • 特定の条件（末尾が 'i' であり、その前が 'c' ではない等）を満たす場合、r_RV 領域内での削除を実行する。
  • 上記が適用されない場合、r_residual_suffix を試行する。

    • 4. **残余処理**: r_residual_form を呼び出し、語幹の微調整を行う。 5. **後処理**: r_postlude を呼び出し、一時変換した文字を元に戻す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は提供されていない。
  • **呼び出し先**: r_prelude, r_mark_regions, r_standard_suffix, r_verb_suffix, r_RV, slice_del, r_residual_suffix, r_residual_form, r_postlude。

関連データ構造

  • struct SN_env: Snowball ランタイムの状態管理構造体。

失敗モードと境界条件

  • 各内部関数が負の値を返した場合、その値をそのまま呼び出し元へ伝播させる。
  • 文字列が空の場合や、領域（RV, R1, R2）が定義できない場合、各内部関数が適切に処理をスキップまたは終了する。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードである。ロジックの修正が必要な場合は、対応する .sbl ファイルを編集し、再生成すること。
  • UTF-8 マルチバイト文字の扱いは header.h で定義されたユーティリティ関数に依存している。
