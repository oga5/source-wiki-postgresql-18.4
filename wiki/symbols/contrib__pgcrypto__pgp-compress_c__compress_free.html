compress_free

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-compress.c:169

目的

圧縮処理で使用された ZipStat 構造体および zlib ストリームのリソースを解放し、メモリをクリーンアップする。

シグネチャまたは形状

static void compress_free(void *priv)

入力と出力

  • priv: 圧縮フィルタの状態を保持する ZipStat 構造体へのポインタ（void * として受け取る）。
  • 戻り値: なし。

前提条件と不変条件

  • priv は compress_init によって初期化された有効な ZipStat 構造体を指している必要がある。
  • 本関数呼び出し後、priv が指すメモリ領域は解放されるため、以降のアクセスは禁止される。

アルゴリズムまたは振る舞い

1. deflateEnd(&st->stream) を呼び出し、zlib の圧縮ストリーム状態を終了させる。 2. px_memset を使用して ZipStat 構造体の中身をゼロクリアする。 3. pfree(st) を呼び出し、PostgreSQL のメモリ管理機構を通じて構造体自体を解放する。

呼び出し元と呼び出し先

  • 呼び出し元: ソースコード内に明示的な呼び出し元情報は記載されていない。
  • 呼び出し先: deflateEnd (zlib), px_memset, pfree (PostgreSQL)。

関連データ構造

  • ZipStat: 圧縮処理の状態を保持する構造体。

失敗モードと境界条件

  • priv が NULL の場合、pfree 等でクラッシュする可能性がある。
  • deflateEnd がエラーを返した場合の処理は本関数内には含まれていない。

保守担当者向けメモ

  • 本関数は PushFilter の終了処理として設計されている。
  • zlib のメモリ管理は z_alloc / z_free を介して PostgreSQL の palloc / pfree に統合されているため、本関数での解放は一貫性を保っている。
