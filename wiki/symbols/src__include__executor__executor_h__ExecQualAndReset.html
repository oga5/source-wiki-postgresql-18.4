ExecQualAndReset

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/executor/executor.h:541

目的

式評価（Qual）の実行と、それに続くタプル単位のメモリコンテキストのリセットを単一の操作として提供し、実行エンジンにおけるメモリ管理を簡素化する。

シグネチャまたは形状

bool ExecQualAndReset(ExprState *state, ExprContext *econtext)

入力と出力

  • **入力**:
  • state: 評価対象の式状態（ExprState）。
  • econtext: 式評価に使用するコンテキスト（ExprContext）。
  • **出力**:
  • bool: 式の評価結果。

前提条件と不変条件

  • econtext が有効であり、そのメンバ ecxt_per_tuple_memory が初期化されていること。
  • 本関数を呼び出すことで、econtext->ecxt_per_tuple_memory に割り当てられたメモリは解放される。

アルゴリズムまたは振る舞い

1. ExecQual(state, econtext) を呼び出し、式の評価結果を取得する。 2. econtext->ecxt_per_tuple_memory に対して MemoryContextReset を実行し、タプル処理用のメモリ領域をクリアする。 3. 手順1で得た評価結果を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • ExecQual: 式の評価を実行。
  • MemoryContextReset: メモリコンテキストの解放。
  • **呼び出し元**:
  • src/backend/executor/execGrouping.c
  • src/backend/executor/nodeAgg.c
  • src/backend/executor/nodeBitmapHeapscan.c
  • src/backend/executor/nodeGroup.c
  • src/backend/executor/nodeHash.c
  • src/backend/executor/nodeIndexonlyscan.c
  • src/backend/executor/nodeIndexscan.c

関連データ構造

  • ExprState: 式の実行状態を保持する構造体。
  • ExprContext: 式評価時の実行環境およびメモリコンテキストを保持する構造体。

失敗モードと境界条件

  • econtext が NULL の場合、MemoryContextReset 呼び出し時にセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内での順序依存を避けるために ResetExprContext マクロの代わりに MemoryContextReset を直接呼び出している。
  • 頻繁に呼び出されるため、パフォーマンスに直結する。変更時はインライン展開の影響を考慮すること。
