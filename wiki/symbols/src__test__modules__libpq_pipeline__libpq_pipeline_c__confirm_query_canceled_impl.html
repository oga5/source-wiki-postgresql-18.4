confirm_query_canceled_impl

定義

  • 種別: function
  • 定義箇所: src/test/modules/libpq_pipeline/libpq_pipeline.c:94

目的

クエリが期待通りにキャンセルされたことを検証するテストヘルパー関数です。クエリ結果がエラー（PGRES_FATAL_ERROR）であり、かつエラーコードがキャンセルを示す 57014 であることを確認します。

シグネチャまたは形状

static void confirm_query_canceled_impl(int line, PGconn *conn);

入力と出力

  • line: 呼び出し元の行番号（エラー発生時のログ出力に使用）。
  • conn: 検証対象のデータベース接続（PGconn *）。
  • 出力: なし。検証失敗時は pg_fatal_impl を呼び出し、プログラムを終了させます。

前提条件と不変条件

  • conn は有効なデータベース接続である必要があります。
  • 本関数は、クエリがキャンセルされた直後の結果取得フェーズで呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. PQgetResult(conn) を呼び出し、クエリの結果を取得します。 2. 結果が NULL の場合、エラーメッセージを出力して終了します。 3. 結果ステータスが PGRES_FATAL_ERROR でない場合、エラーとして終了します。 4. PQresultErrorField を使用し、SQLSTATE が 57014（query_canceled）であることを確認します。一致しない場合はエラーとして終了します。 5. PQclear(res) で結果オブジェクトを解放します。 6. PQisBusy が偽になるまで PQconsumeInput をループ実行し、接続の入力を完全に消費します。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/libpq_pipeline/libpq_pipeline.c 内で定義されたマクロ #define confirm_query_canceled(conn) confirm_query_canceled_impl(__LINE__, conn) を介して呼び出されます。
  • 呼び出し先: PQgetResult, pg_fatal_impl, PQerrorMessage, PQresultStatus, PQresultErrorField, PQclear, PQisBusy, PQconsumeInput。

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • PGresult: libpq のクエリ結果ハンドル。

失敗モードと境界条件

  • PQgetResult が NULL を返した場合（接続断や予期せぬ状態）。
  • クエリがエラーにならなかった場合。
  • エラーコードが 57014 以外の場合。
  • 上記のいずれかの場合、pg_fatal_impl を通じてテストが異常終了します。

保守担当者向けメモ

  • 本関数はテスト専用のユーティリティです。
  • SQLSTATE 57014 は PostgreSQL におけるクエリキャンセルの標準コードです。将来的にキャンセル以外のエラーを検証する際は、別の関数を用意してください。
