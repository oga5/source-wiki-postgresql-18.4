PortalSetResultFormat

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/pquery.c:623

目的

ポータル（Portal）が結果を返す際のデータ形式（テキストまたはバイナリ）を各列ごとに設定する。

シグネチャまたは形状

void PortalSetResultFormat(Portal portal, int nFormats, int16 *formats)

入力と出力

  • **入力**:
  • portal: 設定対象のポータル構造体。
  • nFormats: 指定されたフォーマットの数。
  • formats: 各列のフォーマットコード（int16）を格納した配列。
  • **出力**: なし（副作用として portal->formats を更新）。

前提条件と不変条件

  • portal->tupDesc が NULL の場合、本関数は何も行わず終了する。
  • portal->formats は portal->portalContext メモリコンテキスト内に割り当てられる。

アルゴリズムまたは振る舞い

1. portal->tupDesc が存在しない場合は即座にリターンする。 2. ポータルの列数（natts）に基づき、portal->portalContext 上に formats 配列を確保する。 3. nFormats の値に応じて以下のいずれかの処理を行う：

  • nFormats > 1: 各列に対して個別にフォーマットを指定する。指定数と列数が一致しない場合は ERRCODE_PROTOCOL_VIOLATION でエラーを投げる。
  • nFormats > 0: 配列の最初の要素を全列のフォーマットとして適用する。
  • nFormats == 0: 全列に対してデフォルト（0: テキスト形式）を適用する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/postgres.c:1257
  • src/backend/tcop/postgres.c:2057
  • **呼び出し先**:
  • MemoryContextAlloc (メモリ確保)
  • ereport (エラー報告)

関連データ構造

  • Portal: ポータルの実行状態を保持する構造体。tupDesc（タプル記述子）および formats（結果形式配列）をメンバに持つ。

失敗モードと境界条件

  • nFormats > 1 かつ nFormats != natts の場合、プロトコル違反としてエラーを発生させる。

保守担当者向けメモ

  • 本関数はフロントエンドからのバインドメッセージ処理に関連して呼び出される。
  • portal->formats のメモリ管理は portal->portalContext に依存しているため、ポータルのライフサイクルと同期している必要がある。
