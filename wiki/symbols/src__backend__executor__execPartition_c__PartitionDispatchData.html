PartitionDispatchData

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/execPartition.c:143

目的

パーティションテーブルの階層構造において、特定のレベルにおけるルーティング情報を保持します。タプルルーティング時に、どのパーティションへタプルを振り分けるかを決定するためのメタデータを提供します。

シグネチャまたは形状

typedef struct PartitionDispatchData
{
	Relation	reldesc;
	PartitionKey key;
	List	   *keystate;		/* list of ExprState */
	PartitionDesc partdesc;
	TupleTableSlot *tupslot;
	AttrMap    *tupmap;
	int			indexes[FLEXIBLE_ARRAY_MEMBER];
} PartitionDispatchData;

入力と出力

  • **入力**: パーティション化されたリレーションの記述子、パーティションキー、および階層情報。
  • **出力**: 構造体内の各フィールドを通じて、タプルルーティングに必要なリレーション情報、式評価状態、およびパーティション記述子を提供します。

前提条件と不変条件

  • indexes フィールドは FLEXIBLE_ARRAY_MEMBER として定義されており、構造体の末尾に配置されます。
  • パーティションがサブパーティションを持つ場合、この構造体は再帰的にリンクされ、階層を降りて探索が行われます。

アルゴリズムまたは振る舞い

  • タプルルーティング時、ExecFindPartition がこの構造体を参照してパーティションキーを抽出し、対象パーティションを特定します。
  • 必要に応じて tupmap を使用して親テーブルとパーティション間の属性マッピング（型変換等）を行います。
  • indexes 配列は、パーティションの探索や管理のためのインデックス情報を保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execPartition.c 内のルーティング初期化処理（ExecInitPartitionDispatchInfo 等）により割り当てられます。
  • **呼び出し先**: 直接的な関数呼び出し関係はソースから特定できませんが、PartitionDispatch 型（本構造体へのポインタ）としてルーティングロジック全体で参照されます。

関連データ構造

  • PartitionTupleRouting: ルーティング全体の管理構造体。
  • PartitionDispatch: PartitionDispatchData へのポインタ型。
  • ResultRelInfo: ルーティング先として特定されるパーティションの実行時情報。

失敗モードと境界条件

  • メモリ割り当て時、offsetof(PartitionDispatchData, indexes) を用いて可変長配列分を含めたサイズ計算が行われます。この計算が不正確な場合、メモリ破壊の可能性があります。
