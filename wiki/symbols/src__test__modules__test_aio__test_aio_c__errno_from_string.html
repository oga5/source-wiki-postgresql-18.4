errno_from_string

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_aio/test_aio.c:135

目的

文字列として与えられたエラー名（例: "EIO"）を、対応するシステムエラーコード（errno）の整数値に変換して返すテスト用ヘルパー関数です。

シグネチャまたは形状

Datum errno_from_string(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_TEXT_PP(0) を介して受け取る text 型の文字列。
  • **出力**: 対応する int32 型の errno 値を PG_RETURN_INT32 で返します。

前提条件と不変条件

  • 入力文字列は、サポートされている特定の文字列（"EIO", "EAGAIN", "EINTR", "ENOSPC", "EROFS"）のいずれかである必要があります。

アルゴリズムまたは振る舞い

1. 入力された text 型引数を text_to_cstring で C 文字列に変換します。 2. strcmp を用いて、入力文字列を定義済みのエラー名と比較します。 3. 一致する文字列が見つかった場合、対応するマクロ値（EIO, EAGAIN 等）を返します。 4. 一致する文字列がない場合、ERRCODE_INVALID_PARAMETER_VALUE を伴う ereport(ERROR, ...) を発行し、処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(errno_from_string) マクロにより、PostgreSQL の関数マネージャから呼び出されることが示唆されています。具体的な SQL クエリからの呼び出し元はソースコード上では明示されていません。
  • **呼び出し先**: text_to_cstring、ereport、および strcmp などの標準的な PostgreSQL/C ライブラリ関数を呼び出します。

関連データ構造

  • なし。

失敗モードと境界条件

  • サポートされていない文字列が渡された場合、PostgreSQL のエラーとして処理され、トランザクションがアボートされます。

保守担当者向けメモ

  • 本関数はテストモジュール専用です。新しいエラーコードのサポートが必要な場合は、if-else ブロックに条件を追加してください。
