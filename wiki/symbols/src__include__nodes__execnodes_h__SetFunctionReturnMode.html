SetFunctionReturnMode

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/execnodes.h:333

目的

集合を返す関数（Set-returning functions）の実行結果を、呼び出し元へどのように引き渡すかというモードを定義します。

シグネチャまたは形状

typedef enum {
    SFRM_ValuePerCall = 0x01,
    SFRM_Materialize = 0x02,
    SFRM_Materialize_Random = 0x04,
    SFRM_Materialize_Preferred = 0x08,
} SetFunctionReturnMode;

入力と出力

本シンボル自体は列挙型であり、関数呼び出しの戻り値の形式を指定するフラグとして機能します。

前提条件と不変条件

  • 各値はビットフラグとして定義されており、特定のモードを識別するために使用されます。

アルゴリズムまたは振る舞い

  • SFRM_ValuePerCall: 関数呼び出しごとに1つの値を返します。
  • SFRM_Materialize: 結果セットを Tuplestore に格納して返します。
  • SFRM_Materialize_Random: Tuplestore がランダムアクセスを必要とする場合に指定されます。
  • SFRM_Materialize_Preferred: 呼び出し元が Tuplestore 形式での受け取りを優先する場合に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/nodes/execnodes.h 内の returnMode フィールド（ReturnSetInfo 構造体等で使用されると推測されます）。
  • 呼び出し先: 本シンボルは列挙型定義であり、直接的な関数呼び出し先はありません。

関連データ構造

  • ReturnSetInfo: 実行エンジン内でこの列挙型を保持する構造体として参照されています。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 本シンボルは実行時の戻り値制御に使用されます。新しい戻り値モードを追加する場合は、ビットフラグの重複がないか確認してください。
