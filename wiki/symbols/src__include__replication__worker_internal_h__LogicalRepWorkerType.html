LogicalRepWorkerType

定義

  • 種別: enum
  • 定義箇所: src/include/replication/worker_internal.h:29

目的

論理レプリケーションにおけるワーカープロセスの役割（種類）を識別するために使用されます。

シグネチャまたは形状

typedef enum LogicalRepWorkerType
{
	WORKERTYPE_UNKNOWN = 0,
	WORKERTYPE_TABLESYNC,
	WORKERTYPE_APPLY,
	WORKERTYPE_PARALLEL_APPLY,
} LogicalRepWorkerType;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし

前提条件と不変条件

  • WORKERTYPE_UNKNOWN は初期値または未定義状態として扱われます。
  • ワーカーのライフサイクル管理において、この値はワーカーの役割を固定するために使用されます。

アルゴリズムまたは振る舞い

  • ワーカーの起動時に役割を決定し、LogicalRepWorker 構造体の type フィールドに格納されます。
  • am_leader_apply_worker() や am_parallel_apply_worker() などの判定関数において、現在のプロセスがどの役割であるかを識別する根拠となります。

呼び出し元と呼び出し先

  • 呼び出し元: logicalrep_worker_launch 関数（src/backend/replication/logical/launcher.c）がワーカー起動時に引数として受け取ります。
  • 呼び出し先: なし（データ型定義のため）。

関連データ構造

  • LogicalRepWorker: ワーカーの共通状態を保持する構造体で、この列挙型をメンバとして持ちます。

失敗モードと境界条件

  • 該当なし。

保守担当者向けメモ

  • 新しいワーカータイプを追加する場合は、本列挙型への追加に加え、関連する判定関数（am_... 系）の更新が必要になる可能性があります。
