visibilitymap_prepare_truncate

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/visibilitymap.c:444

目的

リレーションの切り詰め（Truncation）に伴い、可視性マップ（Visibility Map）のサイズを調整し、切り詰められたヒープブロックに対応する不要なビットをクリアする。

シグネチャまたは形状

BlockNumber visibilitymap_prepare_truncate(Relation rel, BlockNumber nheapblocks)

入力と出力

  • **入力**:
  • rel: 対象のリレーション。
  • nheapblocks: 切り詰め後のヒープブロック数。
  • **出力**:
  • 切り詰め後の可視性マップのブロック数（BlockNumber）。切り詰めが不要な場合やマップが存在しない場合は InvalidBlockNumber を返す。

前提条件と不変条件

  • 可視性マップが存在しない場合、何も行わず InvalidBlockNumber を返す。
  • 切り詰め後のヒープサイズがマップのページ境界と一致しない場合、境界をまたぐビットをクリアする必要がある。
  • ビット操作はクリティカルセクション内で行われ、必要に応じて WAL ログが記録される。

アルゴリズムまたは振る舞い

1. 指定された nheapblocks に対応するマップ上の位置（ブロック、バイト、オフセット）を算出する。 2. マップファイルが存在しない場合は終了する。 3. 切り詰め境界がページ境界と一致しない場合：

  • 対象のマップページを vm_readbuf で読み込む。
  • 排他ロックを取得し、クリティカルセクションを開始する。
  • MemSet を使用して、切り詰め範囲外のバイトを 0 クリアする。
  • 境界となるバイト内の不要なビットをビットマスク演算でクリアする。
  • ページをダーティにし、必要に応じて log_newpage_buffer で WAL 記録を行う。
  • クリティカルセクションを終了し、ロックを解放する。

    • 4. 最終的なマップのブロック数が切り詰め後のサイズより大きい場合、その値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/storage.c（smgrtruncate 関連処理）
  • contrib/pg_visibility/pg_visibility.c
  • **呼び出し先**:
  • smgrexists, smgrnblocks
  • vm_readbuf
  • LockBuffer, START_CRIT_SECTION, END_CRIT_SECTION
  • MarkBufferDirty, log_newpage_buffer
  • UnlockReleaseBuffer

関連データ構造

  • Relation: 操作対象のリレーション情報。
  • Buffer: 可視性マップのページを保持するバッファ。
  • VISIBILITYMAP_FORKNUM: 可視性マップが格納されるフォーク番号。

失敗モードと境界条件

  • マップファイルが存在しない場合、処理はスキップされる。
  • 既にマップファイルが要求サイズより小さい場合、何もせず InvalidBlockNumber を返す。
  • vm_readbuf が無効なバッファを返した場合、処理を中断する。

保守担当者向けメモ

  • 本関数は、ヒープの切り詰め時にマップの整合性を保つために不可欠である。
  • チェックサムが有効な場合、切り詰め後のページ保護のために WAL 記録が強制される点に注意すること。
  • visibilitymap_truncation_length は本関数と同様の計算を行うが、変更は行わない。
