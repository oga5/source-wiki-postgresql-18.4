enlarge_trgm_array

定義

  • 種別: function
  • 定義箇所: contrib/pg_trgm/trgm_op.c:123

目的

動的に拡張可能なトリグラム配列（growable_trgm_array）のバッファ容量を、指定された追加要素数分だけ拡張します。

シグネチャまたは形状

static void enlarge_trgm_array(growable_trgm_array *arr, int needed)

入力と出力

  • arr: 拡張対象の growable_trgm_array 構造体へのポインタ。
  • needed: 追加で確保が必要な要素数。
  • 出力: なし（副作用として arr の datum ポインタと allocated フィールドが更新されます）。

前提条件と不変条件

  • arr は初期化済みである必要があります。
  • MaxAllocSize を超えるメモリ確保は許可されません。

アルゴリズムまたは振る舞い

1. 現在の配列長（arr->length）に needed を加算し、必要な総容量を算出します。 2. 必要な容量が現在の確保済みサイズ（arr->allocated）を超える場合のみ処理を行います。 3. MaxAllocSize を超える場合は ERROR を発生させます。 4. repalloc を使用してメモリを再確保し、arr->allocated を更新します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pg_trgm/trgm_op.c 内の関数（例: 365 行目付近）。
  • 呼び出し先: repalloc（PostgreSQL メモリ管理関数）、ereport。

関連データ構造

  • growable_trgm_array: トリグラム配列を管理するための構造体。
  • trgm: トリグラムを保持する型。

失敗モードと境界条件

  • MaxAllocSize / sizeof(trgm) を超える要求があった場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラー終了します。

保守担当者向けメモ

  • repalloc を使用しているため、メモリ断片化に注意してください。
  • CALCGTSIZE マクロを使用して必要なメモリサイズを計算しているため、構造体の定義変更時は trgm.h の定義と整合しているか確認してください。
