jsonb_get_element

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonfuncs.c:1530

目的

JSONBデータから、指定されたパス（キーまたは配列インデックスの配列）に従って特定の要素を抽出する。

シグネチャまたは形状

Datum jsonb_get_element(Jsonb *jb, Datum *path, int npath, bool *isnull, bool as_text)

入力と出力

  • jb: 対象となる Jsonb 構造体。
  • path: 抽出パスを構成する Datum 型の配列。
  • npath: パスの深さ（要素数）。
  • isnull: 抽出結果がNULLまたはパスが見つからない場合に true が設定される出力引数。
  • as_text: true の場合、結果をテキスト形式で返す。false の場合は Jsonb 形式で返す。
  • 戻り値: 抽出された要素の Datum。

前提条件と不変条件

  • npath <= 0 の場合、入力されたJSONB全体（またはスカラー値）をそのまま返す。
  • パス走査中に要素が見つからない場合、isnull を true に設定し、NULL を返す。
  • 配列に対する負のインデックス指定は、末尾からのオフセットとして解釈される。

アルゴリズムまたは振る舞い

1. ルート要素がオブジェクト、配列、スカラーのいずれであるかを判定する。 2. npath が0以下であれば、現在のコンテナをそのまま返して終了する。 3. npath 回のループでパスを順次走査する：

  • 現在のコンテナがオブジェクトであれば、path[i] をキーとして getKeyJsonValueFromContainer を呼び出す。
  • 現在のコンテナが配列であれば、path[i] を整数に変換し、getIthJsonbValueFromContainer を呼び出す。
  • スカラー値に対してパス指定がある場合は抽出不可とみなし、isnull を true にする。

    • 4. 抽出された jbvp がバイナリ形式であれば、次の反復のためにコンテナを更新する。 5. 最終的な抽出結果を as_text フラグに応じてテキストまたは Jsonb 形式に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:

  • **呼び出し先**:
  • getKeyJsonValueFromContainer
  • getIthJsonbValueFromContainer
  • JsonbValueAsText
  • JsonbValueToJsonb
  • cstring_to_text / JsonbToCString

関連データ構造

  • Jsonb: JSONBのルート構造体。
  • JsonbContainer: JSONBの内部コンテナ表現。
  • JsonbValue: JSONBの各ノード（スカラー、オブジェクト、配列）を表現する構造体。

失敗モードと境界条件

  • パス指定が不正（数値変換不可など）な場合、isnull を true にして終了する。
  • 負のインデックスが配列サイズを超過する場合、isnull を true にして終了する。
  • スカラー値に対してパス指定（npath > 0）を行った場合、抽出失敗として扱う。

保守担当者向けメモ

  • 本関数は jsonb 型のサブスクリプト操作のバックエンド実装として機能する。
  • as_text が true の場合、jbvp->type == jbvNull であれば明示的に isnull を true に設定する点に注意。
