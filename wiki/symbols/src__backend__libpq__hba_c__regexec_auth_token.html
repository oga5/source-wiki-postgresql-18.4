regexec_auth_token

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/hba.c:347

目的

認証設定ファイル（pg_hba.conf 等）内のトークンが正規表現として指定されている場合、対象文字列に対してその正規表現を適用し、マッチングを行う。

シグネチャまたは形状

static int regexec_auth_token(const char *match, AuthToken *token, size_t nmatch, regmatch_t pmatch[])

入力と出力

  • **入力**:
  • match: 照合対象となる文字列（ユーザー名やデータベース名など）。
  • token: 正規表現情報を含む AuthToken 構造体。
  • nmatch: pmatch 配列のサイズ。
  • pmatch: マッチした部分文字列の位置を格納する配列。
  • **出力**:
  • int: pg_regexec の実行結果（成功時は REG_OKAY 等）。

前提条件と不変条件

  • token->string の先頭文字が '/' であること。
  • token->regex が既にコンパイル済みであること。
  • 上記は Assert により検証される。

アルゴリズムまたは振る舞い

1. 入力文字列 match を pg_wchar 形式に変換する。 2. pg_regexec を呼び出し、token->regex を用いて正規表現マッチングを実行する。 3. 変換に使用した一時的なメモリ領域を解放する。 4. マッチング結果を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/hba.c 内の以下の箇所から呼び出されることが確認されている。
  • ロール名の照合 (line 971)
  • データベース名の照合 (line 1027)
  • 識別マッピングの照合 (line 2849)
  • **呼び出し先**:
  • palloc / pfree (メモリ管理)
  • pg_mb2wchar_with_len (文字コード変換)
  • pg_regexec (正規表現エンジン)

関連データ構造

  • AuthToken: トークンの文字列とコンパイル済み正規表現 (regex) を保持する構造体。
  • regmatch_t: 正規表現のマッチ位置を保持する標準的な構造体。

失敗モードと境界条件

  • match 文字列が空の場合や、正規表現のコンパイル状態が不正な場合は pg_regexec の挙動に依存する。
  • メモリ不足時には palloc がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は pg_hba.conf 等で / で始まるトークンを処理するために使用される。
  • pg_wchar への変換は、正規表現エンジンがワイド文字を扱うために必要である。
