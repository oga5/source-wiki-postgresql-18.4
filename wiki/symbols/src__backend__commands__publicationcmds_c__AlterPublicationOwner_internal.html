AlterPublicationOwner_internal

定義

  • **種別**: function
  • **定義箇所**: src/backend/commands/publicationcmds.c:1999

目的

パブリケーションの所有者を変更し、システムカタログの更新および所有権に関連する依存関係の再構築を行う。

シグネチャまたは形状

static void AlterPublicationOwner_internal(Relation rel, HeapTuple tup, Oid newOwnerId)

入力と出力

  • **入力**:
  • rel: pg_publication システムカタログの開かれたリレーション。
  • tup: 変更対象のパブリケーションのタプル。
  • newOwnerId: 新しい所有者の OID。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • rel は pg_publication カタログである必要がある。
  • 呼び出し元は、対象パブリケーションに対する適切な権限チェックを完了しているか、スーパーユーザーである必要がある。

アルゴリズムまたは振る舞い

1. **重複チェック**: 現在の所有者が newOwnerId と同一であれば、何もせず終了する。 2. **権限検証**: スーパーユーザーでない場合、以下のチェックを行う:

  • 実行者がパブリケーションの所有者であること。
  • 実行者が newOwnerId にロールを変更できること。
  • newOwnerId が現在のデータベースに対する CREATE 権限を持つこと。
  • FOR ALL TABLES パブリケーションの場合、新しい所有者がスーパーユーザーであること。
  • スキーマパブリケーションの場合、新しい所有者がスーパーユーザーであること。

    • 3. **カタログ更新**: pg_publication タプルの pubowner を更新し、CatalogTupleUpdate を実行する。 4. **依存関係更新**: changeDependencyOnOwner を呼び出し、所有者に関する依存関係を更新する。 5. **フック実行**: InvokeObjectPostAlterHook を呼び出し、変更を通知する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の参照に基づき、src/backend/commands/publicationcmds.c 内のパブリケーション所有権変更処理から呼び出される。
  • **呼び出し先**: GETSTRUCT, superuser, object_ownercheck, aclcheck_error, check_can_set_role, object_aclcheck, get_database_name, superuser_arg, is_schema_publication, ereport, CatalogTupleUpdate, changeDependencyOnOwner, InvokeObjectPostAlterHook。

関連データ構造

  • Relation: システムカタログへのアクセスに使用。
  • HeapTuple: pg_publication のタプルデータ。
  • Form_pg_publication: pg_publication の構造体定義。

失敗モードと境界条件

  • **権限エラー**: 所有者でない、またはスーパーユーザー権限が必要な条件で非スーパーユーザーが所有者変更を試みた場合、ereport(ERROR, ...) により中断される。
  • **データベース権限**: 新しい所有者が対象データベースへの CREATE 権限を持たない場合、エラーとなる。

保守担当者向けメモ

  • 本関数は pg_publication のカタログ更新を直接行うため、呼び出し前に適切なロックが保持されていることを確認すること。
  • FOR ALL TABLES やスキーマパブリケーションに対する所有権制限は、セキュリティ上の要件であるため、変更時は慎重に検討すること。
