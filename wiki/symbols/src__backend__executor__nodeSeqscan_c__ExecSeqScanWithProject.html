ExecSeqScanWithProject

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSeqscan.c:149

目的

フィルタ条件（Qual）を含まず、プロジェクション（射影）のみが必要なシーケンシャルスキャンを実行する。

シグネチャまたは形状

TupleTableSlot *ExecSeqScanWithProject(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（実行中のスキャンノードの状態）。
  • **出力**: TupleTableSlot *（プロジェクション適用後のタプル、またはスキャン終了時は空）。

前提条件と不変条件

  • pstate->state->es_epq_active が NULL であること（EvalPlanQual実行中ではない）。
  • pstate->qual が NULL であること（フィルタ条件が存在しない）。
  • pstate->ps_ProjInfo が NULL ではないこと（プロジェクション情報が必須）。

アルゴリズムまたは振る舞い

1. pstate を SeqScanState にキャストする。 2. ExecScanExtended を呼び出し、以下のメソッドを渡してスキャンを実行する。

  • アクセスメソッド: SeqNext
  • 再チェックメソッド: SeqRecheck
  • フィルタ条件: NULL
  • プロジェクション情報: pstate->ps_ProjInfo

    • 3. ExecScanExtended の戻り値をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeSeqscan.c:269（ExecInitSeqScan 内で ExecProcNode に割り当てられる）。
  • **呼び出し先**:
  • SeqNext（タプル取得用）
  • SeqRecheck（再チェック用）
  • ExecScanExtended（スキャン実行の共通基盤）

関連データ構造

  • SeqScanState: スキャンノードの実行状態を保持する構造体。
  • PlanState: 実行計画ノードの共通状態。
  • TupleTableSlot: タプルを保持するスロット。

失敗モードと境界条件

  • Assert 文により、前提条件（Qualの不在やProjInfoの存在）が満たされない場合は異常終了する。
  • スキャン対象のテーブルが空、または全件走査が完了した場合は NULL を返す。

保守担当者向けメモ

  • 本関数は、クエリ実行時の最適化パスの一つです。Qualが存在する場合は ExecSeqScanWithQual や ExecSeqScanWithQualProject が使用されるため、条件分岐を最小化する設計となっています。
