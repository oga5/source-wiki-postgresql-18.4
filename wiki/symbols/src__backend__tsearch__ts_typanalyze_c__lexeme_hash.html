lexeme_hash

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_typanalyze.c:477

目的

LexemeHashKey 構造体に含まれる語彙文字列をハッシュ化し、ハッシュテーブルでの検索や格納に使用するハッシュ値を生成する。

シグネチャまたは形状

static uint32 lexeme_hash(const void *key, Size keysize)

入力と出力

  • **入力**:
  • key: LexemeHashKey 構造体へのポインタ（void * として渡される）。
  • keysize: キーのサイズ（本関数内では未使用）。
  • **出力**:
  • uint32: hash_any 関数によって計算されたハッシュ値。

前提条件と不変条件

  • key は LexemeHashKey 型の有効なポインタである必要がある。
  • LexemeHashKey 内の lexeme フィールドは有効な文字列を指し、length フィールドは正しい長さを保持している必要がある。

アルゴリズムまたは振る舞い

1. 引数 key を LexemeHashKey 型のポインタ l にキャストする。 2. l->lexeme と l->length を hash_any 関数に渡し、ハッシュ値を計算する。 3. 計算結果を DatumGetUInt32 を介して uint32 型に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/ts_typanalyze.c 内のハッシュテーブル初期化処理（hash_ctl.hash = lexeme_hash;）にて、ハッシュテーブルのコールバック関数として登録される。
  • **呼び出し先**: hash_any（PostgreSQL 内部のハッシュ計算ユーティリティ）。

関連データ構造

  • LexemeHashKey: 語彙を識別するためのキー構造体。

失敗モードと境界条件

  • key が NULL の場合、あるいは LexemeHashKey の内容が不正な場合は未定義の動作となる。

保守担当者向けメモ

  • 本関数は dynahash 形式のハッシュテーブルで使用されることを前提としている。
  • keysize 引数はインターフェース上存在するが、本実装では使用されていない。
