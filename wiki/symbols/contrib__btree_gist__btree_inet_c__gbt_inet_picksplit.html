gbt_inet_picksplit

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_inet.c:165

目的

GiST インデックス構築時、インデックスページが満杯になった際にノードを分割するためのサポート関数です。inet 型のデータを適切に分割し、インデックスのバランスを維持します。

シグネチャまたは形状

Datum gbt_inet_picksplit(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して、GistEntryVector（分割対象のデータ）および GIST_SPLITVEC（分割結果を格納する構造体）を受け取ります。
  • **出力**: Datum 型として、分割処理の結果を返します。

前提条件と不変条件

  • tinfo（gbtree_ninfo 構造体）が定義されており、inet 型の比較ロジックが提供されている必要があります。
  • 内部的に btree_utils_num.c の汎用関数を利用するため、inet 型が double スカラ値へ変換可能であることが前提となります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) から GistEntryVector を取得します。 2. PG_GETARG_POINTER(1) から GIST_SPLITVEC を取得します。 3. tinfo および関数呼び出し情報（fcinfo->flinfo）を引数として、btree_utils_num.c で定義された汎用関数 gbt_num_picksplit を呼び出します。 4. gbt_num_picksplit の戻り値を PG_RETURN_POINTER で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス管理エンジンから呼び出されます。
  • **呼び出し先**: contrib/btree_gist/btree_utils_num.c 内の gbt_num_picksplit を呼び出します。

関連データ構造

  • GistEntryVector: 分割対象のインデックスエントリを保持する構造体。
  • GIST_SPLITVEC: 分割後のエントリ配置情報を格納する構造体。
  • tinfo: inet 型の比較・サイズ情報を定義した gbtree_ninfo 構造体。

失敗モードと境界条件

  • gbt_num_picksplit に依存しているため、メモリ不足や無効なインデックスエントリが渡された場合にエラーが発生する可能性があります。
  • inet 型を double に変換する際の精度限界により、非常に近い値を持つネットワークアドレスの分割において、GiST インデックスの効率が低下する可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.c への単なるラッパーです。ロジックの変更が必要な場合は、まず tinfo の定義内容を確認してください。
  • inet 型特有の複雑な比較ロジックを直接実装するのではなく、数値変換を介した汎用的な分割ロジックを利用する設計となっています。
