heapam_tuple_satisfies_snapshot

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_handler.c:215

目的

指定されたスナップショットに基づき、ヒープタプルが可視であるかどうかを判定する。

シグネチャまたは形状

bool heapam_tuple_satisfies_snapshot(Relation rel, TupleTableSlot *slot, Snapshot snapshot)

入力と出力

  • **入力**:
  • rel: 対象のリレーション。
  • slot: 判定対象のタプルを保持する TupleTableSlot。
  • snapshot: 可視性判定に使用するスナップショット。
  • **出力**:
  • bool: タプルがスナップショットに対して可視であれば true、そうでなければ false。

前提条件と不変条件

  • slot は TTS_IS_BUFFERTUPLE 型である必要がある。
  • slot が保持するバッファは有効（BufferIsValid）であること。
  • 呼び出し元は当該バッファのピンを保持している必要があるが、ロックは保持していないこと。

アルゴリズムまたは振る舞い

1. slot を BufferHeapTupleTableSlot にキャストし、内部のバッファを取得する。 2. LockBuffer を使用してバッファを共有モード（BUFFER_LOCK_SHARE）でロックする。 3. HeapTupleSatisfiesVisibility を呼び出し、タプルの可視性を判定する。 4. LockBuffer を使用してバッファのロックを解除（BUFFER_LOCK_UNLOCK）する。 5. 判定結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_handler.c 内の heapam_methods 構造体（TableAmRoutine）のメンバとして登録されている。
  • **呼び出し先**:
  • LockBuffer
  • HeapTupleSatisfiesVisibility

関連データ構造

  • Relation: リレーション情報。
  • TupleTableSlot: タプルを保持するスロット。
  • BufferHeapTupleTableSlot: バッファ付きタプルスロット。
  • Snapshot: 可視性判定用スナップショット。

失敗モードと境界条件

  • slot がバッファ付きタプルスロットでない場合、Assert により停止する。
  • バッファが無効な場合、Assert により停止する。

保守担当者向けメモ

  • 本関数は TableAmRoutine のコールバックとして機能する。
  • 実際の可視性判定ロジックは HeapTupleSatisfiesVisibility に委譲されているため、判定アルゴリズムの変更が必要な場合はそちらを参照すること。
