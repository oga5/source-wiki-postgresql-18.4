pg_regoff_t

定義

  • 種別: typedef
  • 定義箇所: src/include/regex/regex.h:126

目的

正規表現マッチングにおける文字列内のオフセット（位置）を表現するための型です。

シグネチャまたは形状

typedef long pg_regoff_t;

入力と出力

  • 入力: なし（型定義）
  • 出力: long 型の整数値として、文字列内のバイト位置または文字位置を保持します。

前提条件と不変条件

  • long 型として定義されており、システム上のオフセット値を格納するのに十分なサイズを持つことが想定されています。

アルゴリズムまたは振る舞い

  • 正規表現エンジンがマッチした部分文字列の開始位置および終了位置を記録するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/include/regex/regex.h 内の pg_regmatch_t 構造体（rm_so および rm_eo メンバ）。
  • src/include/regex/regex.h 内の #define regoff_t pg_regoff_t によるエイリアス定義。
  • 呼び出し先: なし（プリミティブ型へのエイリアスであるため）。

関連データ構造

  • pg_regmatch_t: マッチ結果を格納する構造体であり、この型を使用してマッチ範囲を保持します。

失敗モードと境界条件

  • 型のサイズ（long）を超えるオフセット値を持つ文字列を扱う場合、オーバーフローのリスクがあります。

保守担当者向けメモ

  • 本シンボルは POSIX の regoff_t に相当する PostgreSQL 内部の型です。
  • src/include/regex/regex.h 内で regoff_t として再定義されているため、外部インターフェースとの整合性に注意してください。
