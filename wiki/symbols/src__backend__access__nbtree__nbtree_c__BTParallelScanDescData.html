BTParallelScanDescData

定義

  • 種別: struct
  • 定義箇所: src/backend/access/nbtree/nbtree.c:67

目的

B-tree インデックスの並列スキャン実行時に、複数のワーカー間で共有される状態を保持する。

シグネチャまたは形状

typedef struct BTParallelScanDescData
{
    BlockNumber btps_nextScanPage;
    BlockNumber btps_lastCurrPage;
    BTPS_State  btps_pageStatus;
    LWLock      btps_lock;
    ConditionVariable btps_cv;
    int         btps_arrElems[FLEXIBLE_ARRAY_MEMBER];
} BTParallelScanDescData;

入力と出力

  • 入力: 並列スキャン開始時に btestimateparallelscan を通じて必要なメモリサイズが算出され、共有メモリ上に割り当てられる。
  • 出力: 構造体内の各フィールドは、並列スキャン中のページ位置や同期状態を保持し、各ワーカーが参照・更新する。

前提条件と不変条件

  • btps_lock は、構造体内の共有状態（btps_nextScanPage 等）へのアクセスを保護するために使用される。
  • btps_arrElems は可変長配列であり、ScalarArrayOp を含むスキャンキーのオフセットを保持する。

アルゴリズムまたは振る舞い

  • 並列スキャン中、ワーカーは _bt_parallel_seize を呼び出して btps_lock を取得し、次にスキャンすべきページ（btps_nextScanPage）を特定する。
  • ページ処理完了後、_bt_parallel_release を呼び出して状態を更新し、他ワーカーに制御を戻す。
  • btps_cv は、並列スキャン実行時のワーカー間同期に使用される。

呼び出し元と呼び出し先

  • 呼び出し元: btestimateparallelscan（サイズ見積もり）、および並列スキャンを管理する nbtree.c 内の関数群。
  • 呼び出し先: 具体的な関数呼び出しの詳細はソース抜粋からは不明。

関連データ構造

  • BTParallelScanDesc: BTParallelScanDescData へのポインタ型。
  • BTPS_State: 並列スキャンのページ状態を示す列挙型。

失敗モードと境界条件

  • 共有メモリの割り当て不足: btestimateparallelscan による見積もりが不正確な場合、並列スキャンの初期化に失敗する可能性がある。
  • ロック競合: 多数のワーカーが同時に btps_lock を取得しようとすると、パフォーマンスが低下する可能性がある。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を含んでおり、構造体末尾に ScalarArrayOp やスキップ配列用の追加データが格納される設計となっている。
  • _bt_parallel_seize と _bt_parallel_release のペアが正しく管理されていることが、並列実行時の整合性維持に不可欠である。
