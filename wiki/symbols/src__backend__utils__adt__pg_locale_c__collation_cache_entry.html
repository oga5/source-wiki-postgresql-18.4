collation_cache_entry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/pg_locale.c:146

目的

照合順序（Collation）の検索を高速化するためのキャッシュエントリを保持する構造体です。simplehash を用いたキャッシュ機構において、OID からロケールオブジェクトへのマッピングを管理します。

シグネチャまたは形状

typedef struct {
    Oid         collid;    /* ハッシュキー: pg_collation OID */
    pg_locale_t locale;    /* ロケール構造体、無効な場合は 0 */
    uint32      hash;      /* simplehash 用のハッシュ値 */
    char        status;    /* エントリの状態 */
} collation_cache_entry;

入力と出力

  • 入力: collid（pg_collation システムカタログの OID）をキーとして使用。
  • 出力: locale（pg_locale_t 型のロケール情報）を保持。

前提条件と不変条件

  • collid は pg_collation テーブルの有効な OID である必要があります。
  • locale が 0 の場合、その OID に対応するロケールは無効または未初期化であることを示します。
  • simplehash の要素として使用されるため、hash フィールドはハッシュ計算結果と一致している必要があります。

アルゴリズムまたは振る舞い

  • simplehash 実装の一部として、OID をキーとしたハッシュテーブルの要素として機能します。
  • 照合順序の検索時に、まずこの構造体を通じてキャッシュを照会し、キャッシュミスが発生した場合にのみロケールの再生成が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/pg_locale.c 内のキャッシュ管理関数（pg_newlocale_from_collation 等）から参照されます。
  • 呼び出し先: 構造体定義であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • pg_locale_t: 実際のロケールプロパティを保持する構造体。
  • simplehash.h: 本構造体を要素として管理するためのハッシュテーブル実装。

失敗モードと境界条件

  • collid が無効な場合、locale に 0 が設定され、キャッシュエントリとして保持されます。
  • メモリ不足時には、キャッシュの拡張や新規エントリの作成が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は simplehash の SH_ELEMENT_TYPE として定義されています。キャッシュのライフサイクルはバックエンドの生存期間に依存するため、メモリリークや古いロケール情報の保持に注意してください。
