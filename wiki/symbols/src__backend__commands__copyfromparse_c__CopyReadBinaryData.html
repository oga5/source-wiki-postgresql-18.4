CopyReadBinaryData

定義

  • **種別**: function
  • **定義箇所**: src/backend/commands/copyfromparse.c:700

目的

COPY FROM コマンドのバイナリモードにおいて、入力データソースから指定されたバイト数（nbytes）を読み取り、指定されたバッファ（dest）へコピーする。

シグネチャまたは形状

int CopyReadBinaryData(CopyFromState cstate, char *dest, int nbytes)

入力と出力

  • **入力**:
  • cstate: CopyFromState 構造体（現在の解析状態を保持）。
  • dest: データを格納する宛先バッファ。
  • nbytes: 読み込むバイト数。
  • **出力**:
  • 実際に読み込まれたバイト数を int 型で返す。

前提条件と不変条件

  • cstate->raw_buf には、データソースから読み込まれた未処理のバイト列が格納されている必要がある。
  • cstate->raw_buf_index は、次に読み込むべき raw_buf 内のオフセットを指している。

アルゴリズムまたは振る舞い

1. **バッファ確認**: raw_buf 内に要求された nbytes 分のデータが存在するか確認する。 2. **直接コピー**: 十分なデータがある場合、memcpy で dest にコピーし、raw_buf_index を進める。 3. **不足時の補充**: データが不足している場合、do-while ループで以下の処理を繰り返す。

  • raw_buf が空であれば CopyLoadRawBuf を呼び出し、ソースからデータを補充する。
  • raw_reached_eof が真であればループを終了する。
  • raw_buf にある利用可能なデータと、残りの必要バイト数の小さい方を memcpy でコピーする。

    • 4. **戻り値**: 最終的にコピーできた合計バイト数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfromparse.c 内の複数箇所（バイナリヘッダー、属性値、およびパディングの読み込み時）。
  • **呼び出し先**: CopyLoadRawBuf（バッファ補充用）。

関連データ構造

  • CopyFromState: commands/copyfrom_internal.h で定義される、COPY FROM の実行状態を管理する構造体。

失敗モードと境界条件

  • **EOF**: CopyLoadRawBuf が raw_reached_eof を設定した場合、要求された nbytes に達していなくても、その時点までに読み込めたバイト数を返して終了する。
  • **バッファサイズ**: nbytes が内部バッファサイズを超える場合でも、ループ処理により正しく分割して読み込みを行う。

保守担当者向けメモ

  • 本関数は memcpy を使用してメモリ転送を行うため、呼び出し側で dest バッファのオーバーフローが発生しないよう注意すること。
  • CopyLoadRawBuf の呼び出しにより、ファイルやクライアントからの I/O が発生する可能性がある。
