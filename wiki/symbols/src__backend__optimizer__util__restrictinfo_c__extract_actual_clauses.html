extract_actual_clauses

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/restrictinfo.c:484

目的

RestrictInfo ノードのリストから、プラン作成に必要な生の式（Expr）のリストを抽出します。

シグネチャまたは形状

List *extract_actual_clauses(List *restrictinfo_list, bool pseudoconstant)

入力と出力

  • **入力**:
  • restrictinfo_list: 抽出対象となる RestrictInfo ノードのリスト。
  • pseudoconstant: 抽出する式の pseudoconstant 属性の指定値（true または false）。
  • **出力**:
  • 指定された条件に合致する RestrictInfo 内の clause フィールドを格納した List。

前提条件と不変条件

  • restrictinfo_list 内の各要素は RestrictInfo 型である必要があります。
  • rinfo_is_constant_true が真を返す（常に真と評価される）条件式は、結果リストから除外されます。

アルゴリズムまたは振る舞い

1. restrictinfo_list を走査します。 2. 各 RestrictInfo ノードについて、以下の条件を両方満たすか確認します。

  • rinfo->pseudoconstant が引数 pseudoconstant と一致する。
  • rinfo_is_constant_true(rinfo) が偽である。

    • 3. 条件を満たす場合、その rinfo->clause を結果リストに追加します。 4. 走査終了後、構築したリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/file_fdw/file_fdw.c
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/optimizer/plan/createplan.c
  • **呼び出し先**:
  • rinfo_is_constant_true (インライン関数またはマクロと推測される)
  • lfirst_node (マクロ)
  • lappend (マクロ)

関連データ構造

  • RestrictInfo: プランナが使用する制約条件のラッパー構造体。
  • List: PostgreSQL の汎用リスト構造。

失敗モードと境界条件

  • restrictinfo_list が NIL の場合、空のリストを返します。
  • すべての要素が条件に合致しない場合、空のリストを返します。

保守担当者向けメモ

  • 本関数は、プランナが最適化の最終段階で RestrictInfo から実行可能な式を取り出す際に多用されます。
  • pseudoconstant フラグのフィルタリングは、定数として扱える式とそうでない式を分離するために重要です。
