pgaio_uring_context_shmem_size

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_io_uring.c:231

目的

io_uring を使用する全バックエンドプロセス分の PgAioUringContext 構造体を格納するために必要な共有メモリサイズを算出します。

シグネチャまたは形状

size_t pgaio_uring_context_shmem_size(void)

入力と出力

  • 入力: なし
  • 出力: size_t 型のバイト数。pgaio_uring_procs()（バックエンド数）と sizeof(PgAioUringContext) の積を返します。

前提条件と不変条件

  • pgaio_uring_procs() がシステム上の有効なバックエンド数を正しく返す状態であること。
  • mul_size 関数により、計算結果が size_t の範囲を超えないことが保証されていること。

アルゴリズムまたは振る舞い

1. pgaio_uring_procs() を呼び出し、対象となるプロセス数を取得します。 2. PgAioUringContext 構造体のサイズと乗算し、必要なメモリ量を計算して返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/aio/method_io_uring.c:272
  • src/backend/storage/aio/method_io_uring.c:297
  • 呼び出し先:
  • pgaio_uring_procs()

関連データ構造

  • PgAioUringContext: バックエンドごとの io_uring インスタンスとロックを保持する構造体。

失敗モードと境界条件

  • pgaio_uring_procs() が 0 を返す場合、結果として 0 バイトが返されます。
  • メモリ計算においてオーバーフローが発生する可能性がある場合、mul_size が適切に処理します。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウト計算の一部です。PgAioUringContext 構造体の定義に変更があった場合は、本関数が返すサイズが適切か確認してください。
