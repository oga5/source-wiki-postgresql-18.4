multirange_get_range

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/multirangetypes.c:696

目的

MultirangeType 構造体から、指定されたインデックス i に位置する個別の RangeType を抽出し、メモリ上に再構築して返します。

シグネチャまたは形状

RangeType *multirange_get_range(TypeCacheEntry *rangetyp, const MultirangeType *multirange, int i)

入力と出力

  • **入力**:
  • rangetyp: レンジ型のキャッシュエントリ。
  • multirange: 対象となるマルチレンジ構造体。
  • i: 抽出するレンジのインデックス。
  • **出力**:
  • 抽出・構築された RangeType へのポインタ。

前提条件と不変条件

  • i < multirange->rangeCount であること（Assert で検証）。
  • multirange は正規化（ソート済みかつ非重複）されている必要があります。
  • rangetyp は有効な型キャッシュエントリである必要があります。

アルゴリズムまたは振る舞い

1. multirange_get_bounds_offset を使用して、対象レンジのデータ開始位置を特定します。 2. MultirangeGetFlagsPtr から該当レンジのフラグを取得します。 3. レンジ境界値のサイズを計算します。境界値はアライメントが必要なため、att_addlength_pointer および att_align_pointer を使用して、フラグに基づき境界値の長さを走査します。 4. 必要なメモリを palloc0 で確保し、RangeType ヘッダーを初期化します。 5. 抽出した境界データとフラグを新しい RangeType 領域にコピーし、返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/multirangetypes.c 内の複数箇所（インデックス指定によるレンジ取得時）。
  • **呼び出し先**:
  • multirange_get_bounds_offset
  • MultirangeGetFlagsPtr
  • MultirangeGetBoundariesPtr
  • att_addlength_pointer
  • att_align_pointer
  • palloc0

関連データ構造

  • MultirangeType: マルチレンジのメモリ表現。
  • RangeType: 個別のレンジのメモリ表現。
  • TypeCacheEntry: 型情報のキャッシュ。

失敗モードと境界条件

  • i が rangeCount 以上の場合、アサーション違反によりプロセスが停止します（デバッグビルド時）。
  • メモリ不足時には palloc0 がエラーを発生させます。

保守担当者向けメモ

  • 本関数は、マルチレンジ内の可変長境界値を正確に特定するために、フラグを確認しながらポインタを走査する設計になっています。境界値のレイアウト変更時には、この走査ロジックの整合性に注意してください。
