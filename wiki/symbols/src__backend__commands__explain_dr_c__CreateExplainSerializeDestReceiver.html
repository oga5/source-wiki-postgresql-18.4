CreateExplainSerializeDestReceiver

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_dr.c:273

目的

EXPLAIN (SERIALIZE) オプション実行時に、クエリ結果のシリアライズコストを計測するための専用 DestReceiver オブジェクトを生成・初期化する。

シグネチャまたは形状

DestReceiver *CreateExplainSerializeDestReceiver(ExplainState *es)

入力と出力

  • **入力**: ExplainState *es （計測状態を保持する構造体へのポインタ）
  • **出力**: DestReceiver * （初期化されたシリアライズ計測用レシーバー）

前提条件と不変条件

  • palloc0 を使用して SerializeDestReceiver 構造体分のメモリを確保する。
  • 返されるレシーバーの mydest フィールドは DestExplainSerialize に設定される。

アルゴリズムまたは振る舞い

1. SerializeDestReceiver 構造体用にメモリをゼロクリアで割り当てる。 2. DestReceiver の関数ポインタ（receiveSlot, rStartup, rShutdown, rDestroy）を、本モジュール内の対応する計測用関数（serializeAnalyzeReceive 等）に割り当てる。 3. 引数として受け取った ExplainState を構造体内部の es フィールドに保持する。 4. 構築したレシーバーを DestReceiver 型にキャストして返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/explain.c:545
  • src/backend/tcop/dest.c:157
  • **呼び出し先**:
  • palloc0 (メモリ確保)
  • serializeAnalyzeReceive, serializeAnalyzeStartup, serializeAnalyzeShutdown, serializeAnalyzeDestroy (関数ポインタの代入先)

関連データ構造

  • SerializeDestReceiver: 本関数が生成するメインのデータ構造。DestReceiver を継承し、ExplainState への参照と計測用関数群を保持する。
  • ExplainState: 計測結果を格納するための状態管理構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる。
  • 引数 es が NULL の場合でも関数は実行される（src/backend/tcop/dest.c での呼び出し例あり）。

保守担当者向けメモ

  • 本関数は「実際にネットワークへデータを送信しない」レシーバーを生成するファクトリです。ロジックを変更する際は、DestReceiver の各フック関数が正しく計測用ロジックを指しているか確認してください。
