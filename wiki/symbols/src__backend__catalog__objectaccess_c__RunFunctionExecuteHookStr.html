RunFunctionExecuteHookStr

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaccess.c:264

目的

関数実行イベントを外部のフック関数に通知します。特に、オブジェクトIDが確定する前や、名前ベースでの識別が必要な状況において、プラグインが関数実行処理に介入するためのインターフェースを提供します。

シグネチャまたは形状

void RunFunctionExecuteHookStr(const char *objectName)

入力と出力

  • **入力**: objectName (関数名を表す文字列)
  • **出力**: なし（void）

前提条件と不変条件

  • グローバル変数 object_access_hook_str が NULL であってはなりません。実行時に Assert により検証されます。

アルゴリズムまたは振る舞い

1. object_access_hook_str が設定されていることを確認します。 2. 登録されたフック関数を以下の引数で呼び出します。

  • イベントタイプ: OAT_FUNCTION_EXECUTE
  • クラスID: ProcedureRelationId
  • オブジェクト名: objectName
  • その他の引数: 0 および NULL

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • **呼び出し先**: グローバル関数ポインタ object_access_hook_str が指す外部プラグイン関数。

関連データ構造

  • OAT_FUNCTION_EXECUTE: フックに渡されるイベントタイプ定数。
  • ProcedureRelationId: pg_proc カタログを識別するOID。

失敗モードと境界条件

  • object_access_hook_str が NULL の状態で呼び出された場合、アサーション失敗によりプロセスが異常終了します。

保守担当者向けメモ

  • 本関数は名前ベースのフック呼び出し口です。OIDが利用可能な場合は RunFunctionExecuteHook の使用を検討してください。
  • 実際のロジックはロードされた共有ライブラリ（プラグイン）側に存在するため、デバッグ時はフックが正しく登録されているかを確認してください。
