translate_col_privs_multilevel

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/inherit.c:759

目的

継承階層において、最上位の親テーブルで定義された列権限（Bitmapset）を、多段階の継承関係を辿って対象の子テーブルの列インデックスへ変換する。

シグネチャまたは形状

static Bitmapset *translate_col_privs_multilevel(PlannerInfo *root, RelOptInfo *rel, RelOptInfo *parent_rel, Bitmapset *parent_cols)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel: 変換対象の子テーブル（RelOptInfo）。
  • parent_rel: 権限の起点となる親テーブル。
  • parent_cols: 変換対象の列権限ビットマップ。
  • **出力**:
  • 変換後の列権限ビットマップ。parent_cols が NULL の場合は NULL を返す。

前提条件と不変条件

  • root->append_rel_array が初期化されていること。
  • rel が parent_rel の子孫関係にあること。
  • rel が parent_rel と一致しない場合、rel->parent が存在すること。

アルゴリズムまたは振る舞い

1. parent_cols が NULL ならば即座に NULL を返す。 2. rel が直近の親（rel->parent）と一致しない場合、再帰的に translate_col_privs_multilevel を呼び出し、中間階層の親テーブルを経由して権限を変換する。 3. rel->parent が存在しない場合はエラー（elog）を発生させる。 4. 最終的に、root->append_rel_array から対象テーブルの AppendRelInfo を取得し、translate_col_privs を用いて親の列ビットマップを子の列ビットマップへ変換する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/inherit.c 内の関数（updatedCols の算出時など）。
  • **呼び出し先**:
  • translate_col_privs（直接の変換処理）。
  • translate_col_privs_multilevel（再帰呼び出し）。

関連データ構造

  • PlannerInfo: プランナの全体状態を保持。
  • RelOptInfo: リレーションの最適化情報を保持。
  • AppendRelInfo: 継承関係における親から子への属性変換情報（translated_vars）を保持。

失敗モードと境界条件

  • rel->parent が NULL かつ rel != parent_rel の場合、継承関係の不整合としてエラーを投げる。
  • root->append_rel_array が NULL の場合、アサーション違反となる。

保守担当者向けメモ

  • 本関数は多段階継承（孫テーブルなど）をサポートするために再帰構造をとっている。
  • AppendRelInfo の translated_vars を利用しているため、root->append_rel_array が正しく構築されているタイミングで呼び出す必要がある。
