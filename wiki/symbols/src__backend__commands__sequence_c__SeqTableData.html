SeqTableData

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/sequence.c:76

目的

セッション内において、特定のシーケンスに対する現在のキャッシュ状態（最後に払い出した値およびプリフェッチ済みの値）を保持し、nextval() 呼び出し時のパフォーマンスを最適化するために使用されます。

シグネチャまたは形状

typedef struct SeqTableData
{
    Oid            relid;           /* pg_class OID (ハッシュキー) */
    RelFileNumber  filenumber;      /* 最後に確認したリレーションファイル番号 */
    LocalTransactionId lxid;        /* 最後にシーケンス操作を行ったトランザクション */
    bool           last_valid;      /* "last" 値が有効か */
    int64          last;            /* nextval で最後に返した値 */
    int64          cached;          /* nextval 用にキャッシュ済みの最終値 */
    int64          increment;       /* シーケンスの増分値 */
} SeqTableData;

入力と出力

  • **入力**: シーケンスのOIDをキーとしてハッシュテーブル（seqhashtab）に格納されます。
  • **出力**: nextval_internal 等の関数が、この構造体を参照してシーケンスの現在値を更新し、呼び出し元へ値を返します。

前提条件と不変条件

  • increment フィールドは、nextval_internal() が初めて呼び出されるまで 0 に初期化されます。
  • last != cached の場合、キャッシュ内にまだ使用可能な値が残っていることを示します。
  • シーケンスはヒープとして扱われますが、VACUUM の対象外です。

アルゴリズムまたは振る舞い

1. **初期化**: 初回アクセス時に init_sequence がハッシュテーブルを検索し、存在しない場合はエントリを作成します。 2. **値の払い出し**: nextval() 呼び出し時、キャッシュに空きがあれば即座に値を返します。 3. **プリフェッチ**: キャッシュが枯渇した場合、read_seq_tuple を介してディスクから最新状態を読み込み、SEQ_LOG_VALS（32）個分の値を確保してWALを記録します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/sequence.c 内のシーケンス管理関数群。
  • **呼び出し先**: 具体的な関数呼び出し関係はソースコードの断片からは特定できません。

関連データ構造

  • SeqTable: SeqTableData * 型のポインタ。
  • seqhashtab: セッション内でアクセスされたシーケンスを追跡するハッシュテーブル。

失敗モードと境界条件

  • nextval() はトランザクションのロールバックに関わらず値を消費します。
  • ALTER SEQUENCE による変更は、物理ファイルを再作成することでトランザクションの原子性を保証します。

保守担当者向けメモ

  • 本構造体はセッションローカルなキャッシュであるため、プロセス間での状態共有は行われません。
  • last_used_seq ポインタを介して、直近に使用されたエントリへの高速アクセスが行われる場合があります。
