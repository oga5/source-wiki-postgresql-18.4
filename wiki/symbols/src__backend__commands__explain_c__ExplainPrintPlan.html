ExplainPrintPlan

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain.c:758

目的

クエリの実行計画（PlannedStmt）を走査し、ExplainState を介して指定されたフォーマット（TEXT, JSON, XML 等）で出力するための初期化および実行を行う。

シグネチャまたは形状

void ExplainPrintPlan(ExplainState *es, QueryDesc *queryDesc)

入力と出力

  • **入力**:
  • es: 出力状態とフォーマット設定を保持する ExplainState 構造体。
  • queryDesc: 実行計画（plannedstmt）およびプランツリー（planstate）を含むクエリ記述子。
  • **出力**: なし（ExplainState のバッファに結果が蓄積される）。

前提条件と不変条件

  • queryDesc->plannedstmt が NULL であってはならない。
  • ExplainState は呼び出し前に適切に初期化されている必要がある。

アルゴリズムまたは振る舞い

1. **状態の初期化**: queryDesc から plannedstmt や rtable を抽出し、ExplainState の各フィールド（rtable_names, deparse_cxt 等）をセットアップする。 2. **範囲テーブルの調整**: RTE_GROUP 型の範囲テーブルエントリを検出し、rtable_size を調整する。 3. **プランツリーの走査**:

  • GatherState ノードが「不可視（invisible）」としてマークされている場合、そのノードをスキップし、ワーカー詳細の表示を抑制する。
  • ExplainNode を呼び出し、プランツリーを再帰的に走査して出力を行う。

    • 4. **付加情報の出力**:

  • ExplainPrintSettings を呼び出し、デフォルトと異なる GUC パラメータを表示する。
  • verbose モードかつ queryId が存在する場合、ExplainPropertyInteger を使用してクエリ識別子を出力する（ただし COMPUTE_QUERY_ID_REGRESS 設定時は除く）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/explain.c（内部）
  • contrib/auto_explain/auto_explain.c
  • **呼び出し先**:
  • ExplainPreScanNode, select_rtable_names_for_explain, deparse_context_for_plan_tree
  • ExplainNode
  • ExplainPrintSettings
  • ExplainPropertyInteger

関連データ構造

  • ExplainState: 出力制御および状態管理用構造体。
  • QueryDesc: 実行計画およびプランツリーの保持用。
  • PlannedStmt, PlanState: 実行計画のツリー構造。

失敗モードと境界条件

  • Gather ノードの不可視設定により、プランツリーの先頭がスキップされる場合がある。
  • queryId の出力は、グローバル設定 compute_query_id の値に依存する。

保守担当者向けメモ

  • 本関数はプラン出力のメインエントリポイントである。プランノードの表示形式を変更する場合は、呼び出し先の ExplainNode を参照すること。
  • rtable の走査ロジックは、RTE_GROUP の存在によって rtable_size が変動する点に注意が必要。
