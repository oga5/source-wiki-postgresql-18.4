spg_mask

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgxlog.c:993

目的

SP-GiSTインデックスページの非決定的な情報（LSN、チェックサム、ヒントビット、未使用領域）をマスクし、リカバリ後の整合性検証を可能にする。

シグネチャまたは形状

void spg_mask(char *pagedata, BlockNumber blkno)

入力と出力

  • pagedata: マスク対象となるページデータへのポインタ。
  • blkno: ページ番号（本関数内では未使用）。
  • 出力: なし（pagedata が直接更新される）。

前提条件と不変条件

  • pagedata は有効な Page 構造体としてキャスト可能であること。
  • ページヘッダーの pd_lower が SizeOfPageHeaderData 以上である場合、未使用領域のマスクが実行される。

アルゴリズムまたは振る舞い

1. mask_page_lsn_and_checksum(page) を呼び出し、ページLSNとチェックサムをマスクする。 2. mask_page_hint_bits(page) を呼び出し、ヒントビットをマスクする。 3. ページヘッダーの pd_lower が有効な範囲（SizeOfPageHeaderData 以上）にある場合、mask_unused_space(page) を呼び出し、ページ内の未使用領域をマスクする。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/access/rmgrlist.h において、SP-GiSTのリソースマネージャ定義（PG_RMGR）の一部として登録されている。
  • 呼び出し先: mask_page_lsn_and_checksum、mask_page_hint_bits、mask_unused_space（いずれも access/bufmask.h 由来）。

関連データ構造

  • Page: pagedata をキャストして使用。
  • PageHeader: page のヘッダー情報にアクセスするために使用。

失敗モードと境界条件

  • pd_lower が SizeOfPageHeaderData 未満の場合、未使用領域のマスクはスキップされる。

保守担当者向けメモ

  • 本関数は、スタンバイサーバーでのページ検証プロセスにおいて、物理的な変更がなくても発生しうる非決定的な差異を排除するために不可欠である。
  • blkno 引数は現在の実装では使用されていないが、将来的な拡張やデバッグ目的で保持されている。
