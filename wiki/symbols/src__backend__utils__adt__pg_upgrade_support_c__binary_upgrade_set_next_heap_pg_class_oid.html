binary_upgrade_set_next_heap_pg_class_oid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_upgrade_support.c:96

目的

PostgreSQL のアップグレードプロセスにおいて、次に作成されるヒープリレーション（テーブル）の pg_class OID を強制的に指定するために使用されます。

シグネチャまたは形状

Datum binary_upgrade_set_next_heap_pg_class_oid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介して、次に割り当てるべき OID を受け取ります。
  • **出力**: なし（PG_RETURN_VOID() を返します）。

前提条件と不変条件

  • サーバーが binary_upgrade モードで起動されている必要があります。CHECK_IS_BINARY_UPGRADE マクロにより、この条件が満たされない場合はエラーが発生します。

アルゴリズムまたは振る舞い

1. 引数として渡された OID を取得します。 2. CHECK_IS_BINARY_UPGRADE を実行し、アップグレードモードであることを検証します。 3. グローバル変数 binary_upgrade_next_heap_pg_class_oid に、取得した OID を代入します。 4. 処理を終了し、void を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c 内の SQL クエリから呼び出されることが確認されています。
  • **呼び出し先**: 内部的なグローバル変数 binary_upgrade_next_heap_pg_class_oid への書き込みを行います。

関連データ構造

  • binary_upgrade_next_heap_pg_class_oid: アップグレード時に次に使用される OID を保持するグローバル変数。

失敗モードと境界条件

  • binary_upgrade モード以外で呼び出された場合、CHECK_IS_BINARY_UPGRADE により処理が中断されます。

保守担当者向けメモ

  • 本関数はシステムカタログの整合性に直接影響を与えるため、アップグレードツール以外からの利用は想定されていません。
  • 他の binary_upgrade_set_next_* 系関数と同様の定型的な実装となっています。
