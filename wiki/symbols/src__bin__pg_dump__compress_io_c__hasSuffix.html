hasSuffix

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_io.c:162

目的

指定されたファイル名が特定のサフィックス（拡張子）で終わっているかを判定する。

シグネチャまたは形状

static int hasSuffix(const char *filename, const char *suffix)

入力と出力

  • **入力**:
  • filename: 判定対象の文字列（ファイルパス）。
  • suffix: 比較対象のサフィックス文字列。
  • **出力**:
  • 1 (真): filename が suffix で終わる場合。
  • 0 (偽): それ以外の場合。

前提条件と不変条件

  • filename および suffix は有効なヌル終端文字列である必要がある。

アルゴリズムまたは振る舞い

1. strlen を使用して filename と suffix の長さを取得する。 2. filename の長さが suffix の長さより短い場合、直ちに 0 を返す。 3. filename の末尾から suffix の長さ分だけオフセットした位置と、suffix を memcmp で比較する。 4. 一致すれば 1 を、一致しなければ 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/compress_io.c 内のファイルパス解析処理において、.gz、.lz4、.zst の各拡張子判定に使用されている。
  • **呼び出し先**:
  • 標準 C ライブラリの strlen および memcmp を呼び出す。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • filename が suffix より短い場合、正しく 0 を返す。
  • filename または suffix が空文字列の場合の挙動は、strlen の結果に基づき適切に処理される（空のサフィックスは常に一致とみなされる）。

保守担当者向けメモ

  • 本関数は static であり、compress_io.c 内でのみ使用されるユーティリティである。
  • 拡張子判定のロジックを変更する場合は、比較対象のサフィックスが正しく指定されているか確認すること。
