pgstat_count_io_op_time

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_io.c:121

目的

I/O操作の実行時間を計測し、統計情報として蓄積する。

シグネチャまたは形状

void pgstat_count_io_op_time(IOObject io_object, IOContext io_context, IOOp io_op, instr_time start_time, uint32 cnt, uint64 bytes)

入力と出力

  • **入力**:
  • io_object: 対象のI/Oオブジェクト（例: IOOBJECT_RELATION）
  • io_context: I/Oが発生したコンテキスト
  • io_op: 実行された操作（例: IOOP_READ, IOOP_WRITE）
  • start_time: 計測開始時刻（instr_time型）
  • cnt: 操作回数
  • bytes: 操作したバイト数
  • **出力**: なし（副作用として統計変数を更新）

前提条件と不変条件

  • start_time がゼロでない場合のみ、時間計測処理が実行される。
  • IOOBJECT_WAL 以外のオブジェクトに対しては、バッファ使用統計（pgBufferUsage）の更新が行われる。

アルゴリズムまたは振る舞い

1. start_time がゼロでない場合、現在時刻を取得し、start_time との差分（io_time）を算出する。 2. IOOBJECT_WAL 以外の場合:

  • IOOP_WRITE または IOOP_EXTEND の場合、pgstat_count_buffer_write_time を呼び出し、対象がリレーションであれば pgBufferUsage の書き込み時間を加算する。
  • IOOP_READ の場合、pgstat_count_buffer_read_time を呼び出し、対象がリレーションであれば pgBufferUsage の読み取り時間を加算する。

    • 3. PendingIOStats の該当する統計領域に io_time を加算する。 4. pgstat_count_backend_io_op_time を呼び出し、バックエンドごとの統計を更新する。 5. 最後に pgstat_count_io_op を呼び出し、回数とバイト数を記録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogreader.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/replication/walreceiver.c
  • src/backend/storage/buffer/bufmgr.c
  • **呼び出し先**:
  • pgstat_count_buffer_write_time
  • pgstat_count_buffer_read_time
  • pgstat_count_backend_io_op_time
  • pgstat_count_io_op

関連データ構造

  • PendingIOStats: バックエンドごとの未フラッシュ統計を保持する静的変数。
  • pgBufferUsage: バッファ使用状況を追跡するグローバル構造体。
  • instr_time: 計測用時刻データ型。

失敗モードと境界条件

  • start_time がゼロ（INSTR_TIME_IS_ZERO）の場合、時間計測および pgBufferUsage への加算はスキップされ、pgstat_count_io_op のみが実行される。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、オーバーヘッドを最小限に抑える設計となっている。
  • IOOBJECT_WAL は pgBufferUsage の更新対象外である点に注意すること。
