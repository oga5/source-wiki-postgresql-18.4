r_instrum

定義

  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_hungarian.c:569

目的

ハンガリー語のステミング処理において、単語末尾の特定の接尾辞（楽器格などに関連するもの）を特定し、条件を満たす場合に削除および正規化を行う。

シグネチャまたは形状

static int r_instrum(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z （ステミング実行環境の状態を保持する構造体）
  • 出力: int （成功時に 1、失敗または条件不一致時に 0、エラー時に負の値を返す）

前提条件と不変条件

  • z->c が単語内の処理対象位置を指していること。
  • z->lb が単語の境界（下限）を定義していること。
  • a_3 配列に定義された接尾辞パターンとのマッチングが必要。

アルゴリズムまたは振る舞い

1. 現在位置 z->c を z->ket に設定する。 2. 単語の末尾から1文字前が l (ASCII: 108) であるかを確認し、境界条件をチェックする。 3. find_among_b を使用し、a_3 配列に基づいて末尾の接尾辞を検索する。 4. マッチした場合、z->bra を現在位置に設定する。 5. r_R1 を呼び出し、対象が R1 領域内にあるか検証する。 6. r_double を呼び出し、重複文字の処理を行う。 7. slice_del を呼び出し、マッチした接尾辞を削除する。 8. r_undouble を呼び出し、削除後の単語に対して重複文字の復元処理を行う。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_hungarian.c:809 および src/backend/snowball/libstemmer/stem_ISO_8859_2_hungarian.c:803 からの呼び出しが確認されている。
  • 呼び出し先: find_among_b, r_R1, r_double, slice_del, r_undouble。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • a_3: 接尾辞マッチングに使用される among 構造体の配列。

失敗モードと境界条件

  • z->c - 1 <= z->lb: 単語の境界を超えた場合は 0 を返す。
  • p[z->c - 1] != 108: 末尾の文字が l でない場合は 0 を返す。
  • find_among_b がマッチしない場合は 0 を返す。
  • r_R1, r_double, slice_del, r_undouble が負の値を返した場合、その値をそのまま伝播させる。
