InitializeShmemGUCs

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/ipci.c:356

目的

共有メモリのサイズ、必要な巨大ページ（Huge Pages）数、および OS セマフォ数を計算し、それらの値を内部的な GUC（Grand Unified Configuration）パラメータとして設定する。

シグネチャまたは形状

void InitializeShmemGUCs(void)

入力と出力

  • **入力**: なし（内部的に CalculateShmemSize および GetHugePageSize を呼び出す）。
  • **出力**: なし（副作用として GUC パラメータ shared_memory_size、shared_memory_size_in_huge_pages、num_os_semaphores を更新する）。

前提条件と不変条件

  • CalculateShmemSize が正しく共有メモリの総バイト数とセマフォ数を算出できる状態であること。
  • GUC システムが SetConfigOption を受け入れ可能な状態であること。

アルゴリズムまたは振る舞い

1. CalculateShmemSize を呼び出し、必要な共有メモリサイズ（バイト単位）とセマフォ数を取得する。 2. 共有メモリサイズをメガバイト単位に切り上げ、shared_memory_size に設定する。 3. GetHugePageSize を呼び出し、巨大ページサイズが有効な場合、必要な巨大ページ数を計算して shared_memory_size_in_huge_pages に設定する。 4. 算出したセマフォ数を num_os_semaphores に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/postmaster.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • CalculateShmemSize
  • GetHugePageSize
  • add_size
  • SetConfigOption

関連データ構造

  • なし（GUC パラメータの更新のみを行う）。

失敗モードと境界条件

  • GetHugePageSize が 0 を返す場合、巨大ページ関連の GUC 設定はスキップされる。
  • add_size による計算でオーバーフローが発生する可能性があるが、本関数内では直接的なエラーハンドリングは行われていない。

保守担当者向けメモ

  • 本関数は共有メモリの初期化プロセスにおいて、設定値を外部から参照可能にするための報告用関数である。
  • 新しい共有メモリ領域を定義した際は、CalculateShmemSize 側でサイズ計算が正しく行われていることを確認すること。
