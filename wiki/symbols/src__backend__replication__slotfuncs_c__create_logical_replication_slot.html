create_logical_replication_slot

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slotfuncs.c:116

目的

論理レプリケーションスロットを作成し、初期化および出力プラグインの検証を行います。

シグネチャまたは形状

static void create_logical_replication_slot(
    char *name, 
    char *plugin,
    bool temporary, 
    bool two_phase,
    bool failover,
    XLogRecPtr restart_lsn,
    bool find_startpoint
)

入力と出力

  • **入力**:
  • name: スロット名。
  • plugin: 使用する論理デコーディングプラグイン名。
  • temporary: 一時スロットとして作成するかどうか。
  • two_phase: 2フェーズコミットをサポートするかどうか。
  • failover: フェイルオーバーを許可するかどうか。
  • restart_lsn: デコーディング開始位置の起点となるLSN。
  • find_startpoint: デコーディング開始位置を検索するかどうか。
  • **出力**: なし（成功時はスロットが作成され、失敗時は例外が発生）。

前提条件と不変条件

  • MyReplicationSlot が NULL であること（Assertにより検証）。
  • 呼び出し元で適切な権限チェックが完了していること。

アルゴリズムまたは振る舞い

1. **スロット作成**: ReplicationSlotCreate を呼び出し、共有メモリ上にスロットを確保します。一時スロット以外は、エラー発生時の自動クリーンアップを容易にするため、初期状態を RS_EPHEMERAL（一時的）として作成します。 2. **デコーディングコンテキスト作成**: CreateInitDecodingContext を呼び出し、出力プラグインの検証および restart_lsn と xmin の更新を行います。 3. **開始位置検索**: find_startpoint が真の場合、DecodingContextFindStartpoint を呼び出してデコーディング開始位置を特定します。 4. **クリーンアップ**: FreeDecodingContext を呼び出し、コンテキストを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/slotfuncs.c 内の関数（pg_create_logical_replication_slot 等）。
  • src/bin/pg_basebackup/pg_createsubscriber.c 内の関数。
  • **呼び出し先**:
  • ReplicationSlotCreate
  • CreateInitDecodingContext
  • DecodingContextFindStartpoint
  • FreeDecodingContext

関連データ構造

  • LogicalDecodingContext: 論理デコーディングの実行状態を保持する構造体。
  • XLogRecPtr: WAL上の位置を示すポインタ。

失敗モードと境界条件

  • **名前の競合**: ReplicationSlotCreate 内で既存スロット名との競合が検出された場合、エラーが発生します。
  • **プラグイン検証失敗**: CreateInitDecodingContext で指定されたプラグインが読み込めない、または初期化できない場合にエラーが発生します。
  • **トランザクション失敗**: 本関数実行中にトランザクションがロールバックされた場合、作成中のスロットは自動的に破棄されます。

保守担当者向けメモ

  • 本関数はSQLインターフェースと内部ロジックの仲介役です。スロットの永続化（RS_PERSISTENT への昇格）は、この関数を呼び出した上位の関数で行われる設計となっています。
