plpgsql_ns_pop

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_funcs.c:66

目的

PL/pgSQL の名前空間管理スタックにおいて、現在のスコープから直近のラベル（PLPGSQL_NSTYPE_LABEL）を含むエントリまでを破棄し、スコープを一段階戻します。

シグネチャまたは形状

void plpgsql_ns_pop(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 ns_top を更新）

前提条件と不変条件

  • ns_top が NULL であってはならない（Assert により検証）。
  • 名前空間スタックは ns_top を起点とするリンクリストで構成されている。

アルゴリズムまたは振る舞い

1. ns_top が NULL でないことを確認する。 2. ns_top->itemtype が PLPGSQL_NSTYPE_LABEL になるまで、ns_top をリストの前の要素（prev）へ順次移動させる。 3. ラベルに到達した後、さらに ns_top = ns_top->prev を実行し、ラベル自体もスタックから取り除く。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは具体的な呼び出し箇所は特定できません。
  • 呼び出し先: なし（言語組み込みのポインタ操作のみ）。

関連データ構造

  • PLpgSQL_nsitem: 名前空間スタックを構成する要素。itemtype および prev ポインタを持つ。
  • ns_top: 名前空間スタックの先頭を指すグローバル変数。

失敗モードと境界条件

  • ns_top が NULL の状態で呼び出された場合、Assert によりプログラムが異常終了する。
  • スタック内に PLPGSQL_NSTYPE_LABEL 型のアイテムが存在しない場合、リストの終端（NULL）に到達してセグメンテーションフォールトが発生する可能性がある。

保守担当者向けメモ

  • 本関数は plpgsql_ns_push と対になる操作です。
  • ラベル単位でのスコープ管理を行うため、ブロック構造の終了時に呼び出されることを想定しています。
