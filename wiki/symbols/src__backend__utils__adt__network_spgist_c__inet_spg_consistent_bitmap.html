inet_spg_consistent_bitmap

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_spgist.c:373

目的

SP-GiSTインデックスの検索時、クエリ条件（ScanKey）に基づいて探索すべきノードを特定するためのビットマップを生成します。不要なノードを早期に排除（枝刈り）することで検索効率を向上させます。

シグネチャまたは形状

static int inet_spg_consistent_bitmap(const inet *prefix, int nkeys, ScanKey scankeys, bool leaf);

入力と出力

  • **入力**:
  • prefix: 現在のノードが表すネットワークプレフィックス。
  • nkeys: scankeys 配列の要素数。
  • scankeys: 検索条件の配列。
  • leaf: 現在のノードがリーフノードかどうかを示すフラグ。
  • **出力**:
  • 探索対象となるノードを示すビットマップ（整数値）。

前提条件と不変条件

  • leaf が true の場合、初期ビットマップは 1 です。
  • leaf が false の場合、初期ビットマップは 1 | (1 << 1) | (1 << 2) | (1 << 3) （全ノード探索可能）です。
  • 検索条件の評価において、ビットマップが 0 になった時点で探索は打ち切られます。

アルゴリズムまたは振る舞い

1. **初期化**: leaf フラグに基づき、探索可能なノードのビットマップを初期化します。 2. **ファミリ判定**: 検索対象の argument と prefix のアドレスファミリ（IPv4/IPv6）を比較し、不一致の場合は戦略に応じてビットマップを 0 に設定します。 3. **ネットワークビット数判定**: ip_bits を比較し、サブネット/スーパーネット演算子（<<, >> 等）に基づいて探索範囲を絞り込みます。 4. **共通プレフィックス判定**: bitncmp を用いて、共通ビット部分がクエリと一致するかを確認し、不一致の場合はビットマップを 0 に設定します。 5. **次ビット判定**: 共通ビットの次のビットを確認し、探索不要な枝（ノード2または3）をビットマップから除外します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network_spgist.c 内の inet_spg_inner_consistent および関連するインデックス操作関数から呼び出されます。
  • **呼び出し先**:
  • ip_bits(), ip_family(), ip_addr(), DatumGetInetPP(), bitncmp() などのネットワーク型操作関数。

関連データ構造

  • inet: ネットワーク型（inet, cidr）の内部表現。
  • ScanKey: インデックススキャンで使用される検索条件構造体。

失敗モードと境界条件

  • 検索条件とプレフィックスのファミリが異なる場合、一部の演算子を除き探索は即座に終了（ビットマップ 0）します。
  • commonbits が ip_bits(argument) を超える場合や、ビット比較で不一致が生じた場合、探索範囲が適切に絞り込まれます。

保守担当者向けメモ

  • 本関数は、ネットワーク演算子の特性をインデックス構造に最適化するため、複雑な条件分岐を含みます。
  • 4ノード構成において、ノード0/1はプレフィックス長が一致する値、ノード2/3はプレフィックス長がより長い値に対応しています。
  • 「Check 3: next network bit」の処理は検索パフォーマンス向上のための最適化であり、削除しても正当性は維持されます。
