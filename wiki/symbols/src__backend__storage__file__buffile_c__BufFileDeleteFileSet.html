BufFileDeleteFileSet

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/buffile.c:363

目的

FileSet 内に存在する、分割された物理ファイル群から構成される論理的なバッファ付きファイルを削除する。

シグネチャまたは形状

void BufFileDeleteFileSet(FileSet *fileset, const char *name, bool missing_ok)

入力と出力

  • fileset: 対象の FileSet 構造体へのポインタ。
  • name: 削除対象の論理ファイル名。
  • missing_ok: true の場合、ファイルが存在しなくてもエラーを発生させない。false の場合、初期セグメントが見つからないとエラーとなる。
  • 戻り値: なし（成功時）。

前提条件と不変条件

  • FileSet 内の物理ファイルは、FileSetSegmentName によって生成される命名規則に従って分割保存されている必要がある。
  • 削除処理中に割り込みが発生した場合、CHECK_FOR_INTERRUPTS() により処理が中断される可能性がある。

アルゴリズムまたは振る舞い

1. セグメント番号 0 から開始し、FileSetSegmentName を用いて物理ファイル名を生成する。 2. FileSetDelete を呼び出し、該当する物理セグメントを削除する。 3. 削除に成功した場合は found フラグを立て、次のセグメント番号へ進む。 4. FileSetDelete が false を返した時点で、これ以上のセグメントは存在しないと判断しループを終了する。 5. ループ終了後、一度もセグメントが見つからず、かつ missing_ok が false であれば elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/replication/logical/worker.c (複数箇所)
  • 呼び出し先:
  • FileSetSegmentName
  • FileSetDelete
  • CHECK_FOR_INTERRUPTS
  • elog

関連データ構造

  • FileSet: 共有一時ファイルを管理するための構造体。

失敗モードと境界条件

  • 指定された名前の初期セグメント（セグメント0）が存在せず、かつ missing_ok が false の場合、エラーが発生する。

保守担当者向けメモ

  • 本関数は、論理ファイルがいくつの物理セグメントに分割されているかを事前に把握していないため、削除失敗が確認されるまでセグメントを順次削除する設計となっている。
  • fd.c のコメントによれば、本関数はファイルが存在しないケースを許容する仕組みをサポートするために設計されている。
