AbsoluteConfigLocation

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/conffiles.c:35

目的

指定された設定ファイルまたはディレクトリのパスを、絶対パスに変換して返します。

シグネチャまたは形状

char *AbsoluteConfigLocation(const char *location, const char *calling_file)

入力と出力

  • **入力**:
  • location: 解決対象のパス（相対パスまたは絶対パス）。
  • calling_file: 呼び出し元の設定ファイルパス（NULL可）。
  • **出力**:
  • 変換された絶対パスを保持する、palloc された文字列へのポインタ。

前提条件と不変条件

  • location が相対パスであり、かつ calling_file が NULL の場合、グローバル変数 DataDir が初期化されている必要があります（Assert(DataDir) による検証）。

アルゴリズムまたは振る舞い

1. is_absolute_path(location) を判定し、既に絶対パスであればそのコピーを返します。 2. 相対パスの場合:

  • calling_file が提供されている場合、その親ディレクトリを基準パスとして location を結合します。
  • calling_file が NULL の場合、DataDir を基準パスとして location を結合します。

    • 3. 結合後のパスに対して canonicalize_path を実行し、冗長な要素を除去して正規化します。 4. 最終的な絶対パスを pstrdup で複製して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/hba.c
  • src/backend/utils/misc/conffiles.c
  • **呼び出し先**:
  • is_absolute_path
  • pstrdup
  • get_parent_directory
  • join_path_components
  • canonicalize_path

関連データ構造

  • MAXPGPATH: パス文字列を格納するバッファサイズの上限。

失敗モードと境界条件

  • DataDir が設定されていない状態で calling_file が NULL の場合、アサーション違反によりプロセスが停止する可能性があります。
  • メモリ不足時には pstrdup 経由でエラーが発生します。

保守担当者向けメモ

  • 本関数が返す文字列は palloc されているため、呼び出し側で適切にメモリを解放する必要があります。
  • パス解決の基準となる DataDir は PostgreSQL の実行環境において必須のグローバル変数です。
