ossl_EVP_cipher_func

定義

  • 種別: typedef
  • 定義箇所: contrib/pgcrypto/openssl.c:234

目的

OpenSSL の EVP_CIPHER 構造体を取得するための関数ポインタ型を定義し、暗号アルゴリズムの動的な解決を可能にする。

シグネチャまたは形状

typedef const EVP_CIPHER *(*ossl_EVP_cipher_func) (void);

入力と出力

  • 入力: なし (void)
  • 出力: const EVP_CIPHER * 型の OpenSSL 暗号アルゴリズム定義へのポインタ。

前提条件と不変条件

  • 返される EVP_CIPHER ポインタは、OpenSSL ライブラリによって管理される定数構造体である。
  • 本型を使用する関数は、OpenSSL の初期化が完了している環境で呼び出される必要がある。

アルゴリズムまたは振る舞い

  • 特定の暗号アルゴリズム（例: AES, Blowfish）に対応する EVP_CIPHER を返す関数を指すための抽象化レイヤーとして機能する。
  • pgcrypto モジュール内で、アルゴリズム名から具体的な実装を取得する際のルックアップテーブルの一部として利用される。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/openssl.c 内の ossl_cipher_lookup 関数において、この型を持つ変数が定義されている。
  • 呼び出し先: 具体的な実装関数（例: EVP_aes_256_cbc 等をラップする関数）がこの型に適合する。

関連データ構造

  • EVP_CIPHER: OpenSSL が提供する暗号アルゴリズムの定義構造体。
  • OSSLCipher: 本型を使用して取得したアルゴリズム情報を保持する pgcrypto 側のコンテキスト構造体。

失敗モードと境界条件

  • 該当するアルゴリズムが存在しない場合、あるいは OpenSSL のバージョン差異により関数が利用できない場合、呼び出し先の実装は NULL を返す可能性がある。

保守担当者向けメモ

  • 本型は pgcrypto が OpenSSL の EVP API を抽象化する際の基盤である。新しい暗号アルゴリズムを追加する際は、この型に適合する関数を定義し、ルックアップテーブルに登録する必要がある。
