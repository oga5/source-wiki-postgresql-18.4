SetXidCommitTsInPage

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/commit_ts.c:221

目的

指定されたトランザクションID（XID）およびサブトランザクションID群のコミット時刻とレプリケーション元ノードIDを、SLRU（Simple Least Recently Used）キャッシュ上の特定のページに書き込みます。

シグネチャまたは形状

static void SetXidCommitTsInPage(TransactionId xid, int nsubxids, TransactionId *subxids, TimestampTz ts, RepOriginId nodeid, int64 pageno)

入力と出力

  • **入力**:
  • xid: メインのトランザクションID。
  • nsubxids: サブトランザクションIDの数。
  • subxids: サブトランザクションIDの配列。
  • ts: 記録するコミット時刻（TimestampTz）。
  • nodeid: レプリケーション元ノードID。
  • pageno: 書き込み対象のSLRUページ番号。
  • **出力**: なし（副作用としてSLRUバッファを更新）。

前提条件と不変条件

  • CommitTsCtl が初期化済みであること。
  • 指定された pageno に対応するSLRUバンクの排他ロックが取得可能であること。

アルゴリズムまたは振る舞い

1. SimpleLruGetBankLock を使用して、対象ページに対応するLWLockを取得します。 2. LWLockAcquire で排他ロックを確保します。 3. SimpleLruReadPage を呼び出し、対象ページをSLRUバッファに読み込みます。 4. TransactionIdSetCommitTs を呼び出し、メインの xid の情報を書き込みます。 5. ループ処理により、subxids 配列内のすべてのサブトランザクションIDに対して TransactionIdSetCommitTs を実行します。 6. CommitTsCtl->shared->page_dirty[slotno] を true に設定し、ページが更新されたことをマークします。 7. LWLockRelease でロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: TransactionTreeSetCommitTsData（src/backend/access/transam/commit_ts.c 内）。
  • **呼び出し先**: SimpleLruGetBankLock, LWLockAcquire, SimpleLruReadPage, TransactionIdSetCommitTs, LWLockRelease。

関連データ構造

  • CommitTsCtl: コミットタイムスタンプ管理用のSLRU制御構造体。
  • TransactionId: トランザクション識別子。
  • TimestampTz: 時刻データ型。
  • RepOriginId: レプリケーション元ノード識別子。

失敗モードと境界条件

  • SLRUバッファの読み込み失敗やロック取得の競合が発生した場合、SimpleLruReadPage または LWLockAcquire の内部挙動に従いエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はSLRUのページ単位でロックを管理するため、高負荷時には CommitTsCtl のバンク数やバッファ設定が競合に影響します。
  • TransactionIdSetCommitTs はインライン化または単純なメモリ書き込みを行う関数であると想定されます。
