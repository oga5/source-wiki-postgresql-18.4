RI_FKey_noaction_del

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ri_triggers.c:638

目的

外部キー制約における NO ACTION 削除操作を処理するトリガープロシージャです。削除対象の行が他のテーブルから参照されている場合に、制約違反をチェックします。

シグネチャまたは形状

Datum RI_FKey_noaction_del(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (トリガー実行コンテキストを含む)
  • **出力**: Datum (トリガー実行結果。通常は PointerGetDatum(NULL) またはトリガー成功を示す値)

前提条件と不変条件

  • 本関数は、DELETE 操作に関連付けられたトリガーとして呼び出される必要があります。
  • 内部で ri_CheckTrigger を呼び出し、トリガーの呼び出しタイミングとイベントが RI_TRIGTYPE_DELETE と一致していることを検証します。

アルゴリズムまたは振る舞い

1. ri_CheckTrigger を呼び出し、トリガーの実行環境が適切かを確認します。 2. ri_restrict 関数を呼び出し、削除操作が参照整合性に違反していないかをチェックします。この際、ri_restrict に対して true を渡し、NO ACTION 相当の制限付き削除ロジックを実行させます。 3. ri_restrict の戻り値をそのまま返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c: 外部キー制約の定義時や更新時に、トリガー関数として本シンボルが登録されます。
  • **呼び出し先**:
  • ri_CheckTrigger: トリガーの妥当性検証。
  • ri_restrict: 実際の制約チェックおよび参照アクションの実行。

関連データ構造

  • TriggerData: トリガー実行時に渡されるコンテキスト情報。
  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約に基づく引数構造体。

失敗モードと境界条件

  • ri_CheckTrigger による検証で、トリガーの呼び出しタイミングやイベントが不適切な場合、エラーが発生します。
  • ri_restrict の実行中に外部キー制約違反が検出された場合、エラーがスローされトランザクションが中断されます。

保守担当者向けメモ

  • 本関数は ri_restrict のラッパーとして機能しており、RESTRICT 削除と共通のロジックを共有しています。
  • 動作変更を行う際は、ri_restrict 側の実装が他のトリガー関数（RI_FKey_restrict_del 等）に与える影響を考慮してください。
