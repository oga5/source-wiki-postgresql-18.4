ExecParallelEstimate

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execParallel.c:232

目的

並列クエリ実行において、各プランノードが動的共有メモリ（DSM）上で必要とするメモリ領域のサイズを再帰的に推定し、集計する。

シグネチャまたは形状

static bool ExecParallelEstimate(PlanState *planstate, ExecParallelEstimateContext *e);

入力と出力

  • **入力**:
  • planstate: 推定対象のプランツリーのルートノード。
  • e: 推定コンテキスト（ノード数カウンタおよびDSM管理用ポインタ pcxt を保持）。
  • **出力**:
  • 戻り値: planstate_tree_walker の走査結果（通常は false）。
  • 副作用: e->nnodes がインクリメントされ、各ノード固有の推定関数（ExecSeqScanEstimate 等）を通じて e->pcxt に必要なメモリサイズが加算される。

前提条件と不変条件

  • planstate が NULL の場合は即座に false を返す。
  • プランツリー全体を走査するため、新しい並列対応ノードを追加する際は、本関数の switch 文に適切なケースを追加する必要がある。

アルゴリズムまたは振る舞い

1. 現在のノードをカウントする（e->nnodes++）。 2. nodeTag(planstate) に基づき、対応するノード型の推定関数を呼び出す。

  • parallel_aware フラグが立っているノード（SeqScan, HashJoin 等）は、そのノードが並列実行時に必要とするメモリを推定する。
  • EXPLAIN ANALYZE 用に、並列非対応であっても統計情報保持のために推定を行うノード（Sort, Agg, IndexScan 等）が存在する。

    • 3. planstate_tree_walker を使用して、プランツリーの全ノードに対して再帰的に本関数を適用する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execParallel.c:704（ExecInitParallelPlan 内から呼び出される）。
  • planstate_tree_walker による再帰呼び出し。
  • **呼び出し先**:
  • 各ノード型の推定関数（ExecSeqScanEstimate, ExecHashJoinEstimate, ExecSortEstimate 等）。
  • planstate_tree_walker。

関連データ構造

  • PlanState: 実行プランのノード状態。
  • ExecParallelEstimateContext: 推定処理中に状態を保持するコンテキスト構造体。

失敗モードと境界条件

  • planstate が NULL の場合、安全に終了する。
  • 未知のノード型が渡された場合、default ケースにより何も行わず走査を継続する。

保守担当者向けメモ

  • 本関数はプランツリーの走査と各ノードのメモリ要求の集約を担う中心的なルーチンである。
  • PARALLEL_KEY_* を用いたDSMのオフセット管理と密接に関連しているため、メモリ割り当てロジックを変更する際は、対応する ExecParallelInitializeDSM 側の実装と整合性を保つこと。
