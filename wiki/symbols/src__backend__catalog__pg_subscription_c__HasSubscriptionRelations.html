HasSubscriptionRelations

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_subscription.c:511

目的

指定されたサブスクリプション（subid）に関連付けられたテーブル（pg_subscription_rel）が存在するかどうかを判定します。

シグネチャまたは形状

bool HasSubscriptionRelations(Oid subid)

入力と出力

  • **入力**: subid (Oid) - 検査対象のサブスクリプション識別子。
  • **出力**: bool - 関連するテーブルが1つでも存在すれば true、存在しなければ false を返します。

前提条件と不変条件

  • pg_subscription_rel システムカタログに対する読み取り権限が必要です。
  • 呼び出し元は、対象のサブスクリプションに対して適切なロックを保持していることが推奨されます。

アルゴリズムまたは振る舞い

1. SubscriptionRelRelationId（pg_subscription_rel）を AccessShareLock モードで開きます。 2. pg_subscription_rel_srsubid 属性をキーとして、指定された subid を検索するスキャンを開始します。 3. systable_getnext を呼び出し、タプルが1つでも取得できるかを確認します。 4. 取得できた場合は true、取得できなかった場合は false を結果として保持します。 5. スキャンを終了し、リレーションを閉じて結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/tablesync.c:1654
  • **呼び出し先**:
  • table_open
  • ScanKeyInit
  • systable_beginscan
  • systable_getnext
  • systable_endscan
  • table_close

関連データ構造

  • pg_subscription_rel: サブスクリプションとテーブルの紐付けを管理するシステムカタログ。
  • Relation: システムカタログへのアクセスに使用する構造体。
  • ScanKeyData: カタログ検索条件を保持する構造体。

失敗モードと境界条件

  • subid が無効な場合、検索結果は空となり false を返します。
  • システムカタログのロック取得に失敗した場合、データベースの例外処理メカニズムにより処理が中断される可能性があります。

保守担当者向けメモ

  • 本関数は pg_subscription_rel を直接スキャンするため、キャッシュを利用する他の関数と比較してコストがかかる可能性があります。頻繁に呼び出される場合は注意してください。
  • AccessShareLock を使用しているため、サブスクリプションの削除や変更と競合する可能性があります。必要に応じて呼び出し元で適切なロック管理を行ってください。
