PMSignalReason

定義

  • 種別: enum
  • 定義箇所: src/include/storage/pmsignal.h:33

目的

ポストマスター（親プロセス）に対して、子プロセスから非同期に通知すべきイベントの種類を定義します。

シグネチャまたは形状

typedef enum {
    PMSIGNAL_RECOVERY_STARTED,
    PMSIGNAL_RECOVERY_CONSISTENT,
    PMSIGNAL_BEGIN_HOT_STANDBY,
    PMSIGNAL_ROTATE_LOGFILE,
    PMSIGNAL_START_AUTOVAC_LAUNCHER,
    PMSIGNAL_START_AUTOVAC_WORKER,
    PMSIGNAL_BACKGROUND_WORKER_CHANGE,
    PMSIGNAL_START_WALRECEIVER,
    PMSIGNAL_ADVANCE_STATE_MACHINE,
    PMSIGNAL_XLOG_IS_SHUTDOWN
} PMSignalReason;

入力と出力

  • 本シンボル自体は列挙値であり、SendPostmasterSignal(PMSignalReason) の引数として入力され、CheckPostmasterSignal(PMSignalReason) の引数として状態確認に使用されます。

前提条件と不変条件

  • 通知は共有メモリ上の PMSignalFlags を介して行われます。
  • PMSignalFlags は sig_atomic_t 型であり、アトミックな読み書きが保証されています。

アルゴリズムまたは振る舞い

1. 子プロセスが SendPostmasterSignal を呼び出し、指定された PMSignalReason に対応するフラグを共有メモリ上でセットし、ポストマスターへ SIGUSR1 を送信します。 2. ポストマスターは CheckPostmasterSignal を呼び出し、共有メモリ上のフラグを読み取って対応するイベント処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/pmsignal.c 内の SendPostmasterSignal および CheckPostmasterSignal 関数がこの型を引数として利用します。
  • **呼び出し先**: 情報なし。

関連データ構造

  • PMSignalData: 共有メモリ上に配置される通信用データ構造。

失敗モードと境界条件

  • 共有メモリの初期化前に呼び出された場合の挙動については、本ソース excerpt からは判断できません。

保守担当者向けメモ

  • 新しいイベントを追加する場合は、本列挙型に値を追加し、pmsignal.c 側の CheckPostmasterSignal での処理を実装してください。
  • 共有メモリレイアウトの変更を避けるため、構造体の内部詳細は pmsignal.c に隠蔽されています。
