newRegisNode

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/regis.c:73

目的

正規表現パターンを保持するための新しい RegisNode 構造体をメモリ上に確保し、連結リストを構築する。

シグネチャまたは形状

static RegisNode *newRegisNode(RegisNode *prev, int len)

入力と出力

  • **入力**:
  • prev: 直前のノードへのポインタ（リストの先頭の場合は NULL）。
  • len: 確保するノードのデータ領域の長さ。
  • **出力**:
  • 新規に確保・初期化された RegisNode へのポインタ。

前提条件と不変条件

  • palloc0 を使用するため、PostgreSQL のメモリコンテキストが有効である必要がある。
  • len はノードのデータ領域サイズとして正の整数が期待される。

アルゴリズムまたは振る舞い

1. RNHDRSZ + len + 1 バイトのメモリを palloc0 でゼロ初期化して確保する。 2. prev が指定されている場合、prev->next に新規ノードを代入し、連結リストを接続する。 3. 新規ノードへのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: RS_compile（src/backend/tsearch/regis.c 内の複数箇所から呼び出し）。
  • **呼び出し先**: palloc0（PostgreSQL メモリ管理関数）。

関連データ構造

  • RegisNode: パターンを保持する連結リストのノード。next ポインタ、型情報、およびデータ領域を持つ。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させ、処理が中断される。
  • prev が NULL の場合、リストの接続は行われず、単一のノードとして作成される。

保守担当者向けメモ

  • 本関数は RS_compile による正規表現コンパイルプロセスの一部として使用される。
  • 確保されたメモリは RS_free を通じて解放される必要がある。
