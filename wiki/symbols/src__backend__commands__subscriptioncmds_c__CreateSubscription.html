CreateSubscription

定義

  • 種別: function
  • 定義箇所: src/backend/commands/subscriptioncmds.c:538

目的

論理レプリケーションにおける新しいサブスクリプションを作成し、システムカタログ pg_subscription への登録およびリモートノード（パブリッシャー）との初期接続・同期準備を行う。

シグネチャまたは形状

ObjectAddress CreateSubscription(ParseState *pstate, CreateSubscriptionStmt *stmt, bool isTopLevel)

入力と出力

  • **入力**:
  • pstate: パーサー状態。
  • stmt: CREATE SUBSCRIPTION 文の構文解析結果。
  • isTopLevel: トランザクションブロック内かどうかのフラグ。
  • **出力**:
  • 作成されたサブスクリプションの ObjectAddress を返す。

前提条件と不変条件

  • pg_create_subscription ロールの権限が必要。
  • データベースに対する CREATE 権限が必要。
  • create_slot = true を指定する場合、トランザクションブロック内での実行は禁止。
  • 非スーパーユーザーが password_required = false を指定することは不可。
  • 同一データベース内でサブスクリプション名は一意である必要がある。

アルゴリズムまたは振る舞い

1. **オプション解析**: parse_subscription_options を呼び出し、SubOpts 構造体に設定を格納。 2. **権限検証**: ユーザー権限およびデータベース権限を確認。 3. **カタログ操作**: pg_subscription に新しいタプルを挿入し、所有権の依存関係を記録。 4. **レプリケーション起点作成**: replorigin_create を呼び出し、レプリケーションの起点を作成。 5. **リモート接続**: opts.connect が真の場合、libpqwalreceiver をロードし、walrcv_connect を介してパブリッシャーへ接続を試みる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c (SQLコマンド実行時)
  • **呼び出し先**:
  • parse_subscription_options, PreventInTransactionBlock, has_privs_of_role, object_aclcheck, table_open, GetSysCacheOid2, load_file, walrcv_check_conninfo, GetNewOidWithIndex, CatalogTupleInsert, recordDependencyOnOwner, replorigin_create, walrcv_connect 等。

関連データ構造

  • CreateSubscriptionStmt: SQL 文の解析結果を保持。
  • SubOpts: サブスクリプション作成オプションを保持する内部構造体。
  • pg_subscription: サブスクリプション情報を格納するシステムカタログ。

失敗モードと境界条件

  • **権限エラー**: 必要なロールや権限がない場合に ERROR を発行。
  • **重複エラー**: 同名のサブスクリプションが既に存在する場合に ERRCODE_DUPLICATE_OBJECT を発行。
  • **接続エラー**: パブリッシャーへの接続失敗時に ERRCODE_CONNECTION_FAILURE を発行。
  • **トランザクション制約**: create_slot 指定時にトランザクションブロック内で実行された場合、PreventInTransactionBlock によりエラーとなる。

保守担当者向けメモ

  • リモート接続処理は PG_TRY/PG_FINALLY ブロックによる保護が推奨される（ソース内のコメント参照）。
  • libpqwalreceiver のロードは動的に行われるため、環境依存のライブラリパスに注意すること。
  • AlterSubscription と共通のオプション解析ロジックを使用しているため、オプション追加時は parse_subscription_options の修正が必要。
