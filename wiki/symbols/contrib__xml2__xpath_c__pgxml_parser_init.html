pgxml_parser_init

定義

  • 種別: function
  • 定義箇所: contrib/xml2/xpath.c:66

目的

libxml2 パーサーの初期化と、PostgreSQL のエラーハンドリング機構との統合を行う。

シグネチャまたは形状

PgXmlErrorContext *pgxml_parser_init(PgXmlStrictness strictness)

入力と出力

  • **入力**: strictness (PgXmlStrictness 型) - XML パース時の厳密さを指定するフラグ。
  • **出力**: PgXmlErrorContext * - 初期化されたエラーコンテキストへのポインタ。

前提条件と不変条件

  • pg_xml_init が正常にエラーコンテキストを生成できること。
  • xmlInitParser() の呼び出しが例外（elog）を発生させないこと。

アルゴリズムまたは振る舞い

1. pg_xml_init(strictness) を呼び出し、PostgreSQL のコアと共有するエラーハンドラーをセットアップする。 2. xmlInitParser() を呼び出し、libxml2 ライブラリを初期化する。 3. 生成された PgXmlErrorContext を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/xml2/xpath.c: xpath_table 関数内（2箇所）
  • contrib/xml2/xslt_proc.c: XSLT 処理関数内
  • **呼び出し先**:
  • pg_xml_init (PostgreSQL コアの XML サポート機能)
  • xmlInitParser (libxml2 ライブラリ)

関連データ構造

  • PgXmlErrorContext: XML パース中のエラー情報を保持する構造体。

失敗モードと境界条件

  • pg_xml_init がメモリ不足等で失敗した場合、PostgreSQL のエラーハンドリングにより elog(ERROR) が発生する可能性がある。

保守担当者向けメモ

  • 本関数は libxml2 のグローバルな初期化を行うため、モジュール内の XML 処理の起点となる。
  • libxml2 のリソース解放については、本関数ではなく cleanup_workspace 等の終了処理関数が担当する点に注意すること。
