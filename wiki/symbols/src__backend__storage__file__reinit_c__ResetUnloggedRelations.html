ResetUnloggedRelations

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/reinit.c:46

目的

PostgreSQLの起動リカバリ時において、アンログド（unlogged）リレーションの整合性を回復するために、全テーブルスペースを走査してクリーンアップまたは初期化処理を実行する。

シグネチャまたは形状

void ResetUnloggedRelations(int op)

入力と出力

  • **入力**:
  • op: 処理内容を指定するビットマスクフラグ。UNLOGGED_RELATION_CLEANUP（不要なフォークの削除）および UNLOGGED_RELATION_INIT（_init フォークからの再構築）を指定可能。
  • **出力**: なし（副作用としてファイルシステム上のリレーションファイルを操作する）。

前提条件と不変条件

  • 呼び出し時に CurrentMemoryContext が有効であること。
  • PG_TBLSPC_DIR および各テーブルスペースディレクトリへのアクセス権限が必要。
  • _init フォークは決して削除されないという不変条件がある。

アルゴリズムまたは振る舞い

1. **メモリ管理**: 操作専用のテンポラリメモリコンテキストを作成し、処理終了後に解放する。 2. **進捗報告**: begin_startup_progress_phase() を呼び出し、リカバリの進捗報告を開始する。 3. **デフォルト領域の処理**: pg_default（base ディレクトリ）に対して ResetUnloggedRelationsInTablespaceDir を実行する。 4. **全テーブルスペースの走査**: PG_TBLSPC_DIR を開き、各テーブルスペースディレクトリを反復処理する。

  • . および .. ディレクトリはスキップする。
  • 各テーブルスペース内の TABLESPACE_VERSION_DIRECTORY に対して ResetUnloggedRelationsInTablespaceDir を再帰的に実行する。

    • 5. **後処理**: メモリコンテキストを復元し、テンポラリコンテキストを削除する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c（リカバリプロセスから UNLOGGED_RELATION_CLEANUP および UNLOGGED_RELATION_INIT フラグを伴って呼び出される）。
  • **呼び出し先**:
  • ResetUnloggedRelationsInTablespaceDir
  • AllocateDir, ReadDir, FreeDir（storage/fd.h）
  • MemoryContextSwitchTo, AllocSetContextCreate, MemoryContextDelete
  • begin_startup_progress_phase

関連データ構造

  • unlogged_relation_entry: クリーンアップ時に処理対象を追跡するために使用されるハッシュテーブル用構造体。

失敗モードと境界条件

  • ディレクトリの読み取り失敗やファイル操作権限がない場合、PostgreSQLの標準的なエラーハンドリング（elog 等）によりプロセスが中断される可能性がある。
  • op に無効なフラグが渡された場合、意図しない動作を引き起こす可能性がある（ログ出力で確認可能）。

保守担当者向けメモ

  • 本関数はリカバリプロセスの重要な一部であり、ファイルシステム構造の変更（特にテーブルスペースのレイアウト）に影響を受ける。
  • ResetUnloggedRelationsInTablespaceDir が内部でハッシュテーブルを使用しているため、メモリ消費量はリレーション数に比例する。
