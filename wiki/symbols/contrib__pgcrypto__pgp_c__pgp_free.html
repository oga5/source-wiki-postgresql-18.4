pgp_free

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp.c:212

目的

PGP_Context 構造体が保持するリソースを解放し、メモリを安全に破棄します。

シグネチャまたは形状

int pgp_free(PGP_Context *ctx);

入力と出力

  • **入力**: ctx (PGP_Context *) - 解放対象のPGPコンテキストへのポインタ。
  • **出力**: int - 常に 0 を返します。

前提条件と不変条件

  • ctx は pgp_init() によって初期化された有効なポインタである必要があります。
  • 解放後は ctx を参照してはなりません。

アルゴリズムまたは振る舞い

1. ctx->pub_key が存在する場合、pgp_key_free() を呼び出して公開鍵リソースを解放します。 2. px_memset() を使用して、ctx 構造体全体をゼロクリアし、機密情報がメモリ上に残らないようにします。 3. pfree() を呼び出して、コンテキスト構造体自体が占有していたメモリを解放します。 4. 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-pgsql.c (複数箇所)
  • **呼び出し先**:
  • pgp_key_free (条件付き)
  • px_memset
  • pfree

関連データ構造

  • PGP_Context: PGP処理の状態を保持する主要な構造体。

失敗モードと境界条件

  • ctx が NULL の場合、pfree 等の内部関数でセグメンテーション違反が発生する可能性があります（ソースコード上に明示的なNULLチェックはありません）。

保守担当者向けメモ

  • 本関数は機密情報のクリアを目的として px_memset を使用しています。構造体の定義が変更された場合、sizeof *ctx によるクリア範囲が適切か確認してください。
