MatchesArray

定義

  • 種別: function
  • 定義箇所: src/bin/psql/tab-complete.in.c:1728

目的

入力された単語列が、指定された補完パターン（args）と一致するかを判定します。ワイルドカード（MatchAnyN）を含むパターンマッチングをサポートします。

シグネチャまたは形状

static bool MatchesArray(bool case_sensitive,
                         int previous_words_count, char **previous_words,
                         int narg, const char *const *args)

入力と出力

  • **入力**:
  • case_sensitive: 大文字小文字を区別するかどうか。
  • previous_words: ユーザーが入力した単語の配列。
  • previous_words_count: 入力単語数。
  • narg: パターン配列 args の要素数。
  • args: 比較対象となるパターン文字列の配列。空文字列 "" は MatchAnyN（任意の単語列にマッチ）として扱われます。
  • **出力**:
  • bool: パターンと一致した場合は true、それ以外は false。

前提条件と不変条件

  • previous_words_count が narg - 1 未満の場合、常に false を返します。

アルゴリズムまたは振る舞い

1. args 内に空文字列（MatchAnyN）が存在するかを走査します。 2. **通常ケース（MatchAnyN なし）**:

  • previous_words_count と narg が一致することを確認します。
  • HeadMatchesArray を呼び出し、全単語が一致するか判定します。

    • 3. **ワイルドカードケース（MatchAnyN あり）**:

  • MatchAnyN の位置を境界として、前半を HeadMatchesArray で、後半を TailMatchesArray でそれぞれ検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/tab-complete.in.c 内の複数箇所から呼び出されています。
  • **呼び出し先**: HeadMatchesArray, TailMatchesArray。

関連データ構造

  • previous_words: 入力行を分解した単語リスト。
  • args: 補完ルールを定義する文字列配列。

失敗モードと境界条件

  • 入力単語数がパターンに必要な最小数に満たない場合は即座に false を返します。
  • MatchAnyN が含まれる場合、その位置に応じて分割された各セグメントが一致しない限り false を返します。

保守担当者向けメモ

  • 本関数は gen_tabcomplete.pl によって生成されるコードの一部として利用されることを想定しています。
  • MatchAnyN の判定ロジックは args 内の空文字列に依存しているため、パターン定義時に注意が必要です。
