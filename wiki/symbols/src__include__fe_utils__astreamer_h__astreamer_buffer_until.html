astreamer_buffer_until

定義

  • 種別: function
  • 定義箇所: src/include/fe_utils/astreamer.h:185

目的

ストリーム内のバッファが指定されたバイト数（target_bytes）に達するまで、入力データからデータを蓄積する。

シグネチャまたは形状

static inline bool astreamer_buffer_until(astreamer *streamer, const char **data, int *len, int target_bytes)

入力と出力

  • **入力**:
  • streamer: 操作対象の astreamer 構造体。
  • data: 入力データポインタへのポインタ。処理に応じて更新される。
  • len: 入力データの残り長へのポインタ。処理に応じて更新される。
  • target_bytes: 蓄積目標とするバイト数。
  • **出力**:
  • 戻り値: 目標バイト数に達した場合は true、不足している場合は false を返す。

前提条件と不変条件

  • streamer->bbs_buffer が初期化されていること。
  • data および len は、呼び出し元で追跡されている現在のストリーム位置を指していること。

アルゴリズムまたは振る舞い

1. 現在のバッファ長（streamer->bbs_buffer.len）が target_bytes 以上であれば、何もせず true を返す。 2. 現在のバッファ長と入力データの合計が target_bytes 未満であれば、入力データすべてを astreamer_buffer_bytes でバッファに蓄積し、false を返す。 3. 上記以外の場合、目標長に達するまでの不足分のみを astreamer_buffer_bytes で蓄積し、true を返す。

関連データ構造

  • astreamer: ストリーム処理の基底構造体。bbs_buffer を保持する。

失敗モードと境界条件

  • target_bytes が現在のバッファ長以下の場合は即座に成功（true）とみなされる。
  • 入力データが不足している場合は、利用可能なすべてのデータがバッファに移動される。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で直接展開される。
  • astreamer_buffer_bytes を内部的に呼び出しているため、バッファ管理の挙動はこの関数に依存する。
