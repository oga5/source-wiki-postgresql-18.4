add_descriptor

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/descriptor.c:74

目的

ECPGプリプロセッサにおいて、SQL記述子（Descriptor）を管理するためのグローバルリストに新しい記述子エントリを追加します。

シグネチャまたは形状

void add_descriptor(const char *name, const char *connection)

入力と出力

  • name: 記述子の名前（文字列）。
  • connection: 記述子が関連付けられる接続名（文字列、またはNULL）。
  • 出力: なし（副作用としてグローバルな記述子リストを更新）。

前提条件と不変条件

  • name の先頭文字がダブルクォーテーション（"）でない場合、関数は何も行わずに終了します。
  • 記述子の名前と接続情報は、mm_strdup を介してメモリ上に複製されます。

アルゴリズムまたは振る舞い

1. name の先頭が " でない場合、処理を中断します。 2. struct descriptor 型のメモリを確保します。 3. 引数として渡された name および connection を複製して構造体に格納します。 4. 新しい記述子をグローバルな descriptors リストの先頭に挿入します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • 呼び出し先: mm_alloc（メモリ確保）、mm_strdup（文字列複製）。

関連データ構造

  • struct descriptor: 記述子の名前、接続情報、およびリストの次要素へのポインタを保持する構造体。

失敗モードと境界条件

  • name が " で始まらない場合、登録は無視されます。
  • メモリ不足時には mm_alloc または mm_strdup 内でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は descriptors というグローバル変数に依存しています。
  • 記述子の名前は引用符付きで管理される仕様であるため、呼び出し側で名前の形式を保証する必要があります。
