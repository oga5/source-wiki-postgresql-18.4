AclItem

定義

  • 種別: struct
  • 定義箇所: src/include/utils/acl.h:54

目的

データベースオブジェクトに対する個別の権限付与（GRANT）情報を保持する最小単位のデータ構造です。

シグネチャまたは形状

typedef struct AclItem
{
	Oid			ai_grantee;		/* 権限が付与されるロールのID */
	Oid			ai_grantor;		/* 権限を付与したロールのID */
	AclMode		ai_privs;		/* 権限ビット */
} AclItem;

入力と出力

  • 本構造体は、システムカタログから読み込まれたACL（アクセス制御リスト）の各要素として利用されます。
  • ai_privs は AclMode 型であり、下位32ビットが実際の権限、上位32ビットがGRANT OPTIONを表します。

前提条件と不変条件

  • Acl（AclItemの配列）内で使用される際、GRANT OPTIONを持つ権限は、それに依存する権限よりも前に配置される必要があります。
  • Acl はTOASTされる可能性があるため、処理時にはデトースト処理（PG_DETOAST_DATUM）が必須です。

アルゴリズムまたは振る舞い

  • aclmask() 関数により、特定のロールに対する権限ビットを抽出する際に参照されます。
  • aclupdate を通じて、既存のACL配列に対する追加・削除操作が行われます。
  • qsort を用いたソート処理において、aclitemComparator と組み合わせて配列の順序を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/aclchk.c および src/backend/utils/adt/acl.c 内のACL操作関数群。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。
  • 注: 呼び出し元・呼び出し先の詳細なコールグラフは、提供されたソース範囲内では限定的です。

関連データ構造

  • Acl: AclItem の配列。
  • AclMode: 権限ビットを保持する型。
  • AclResult: 権限チェックの結果を保持する列挙型。

失敗モードと境界条件

  • AclItem の配列順序が不適切な場合、権限チェックの整合性が損なわれる可能性があります。
  • Acl 配列の操作時にTOAST処理を怠ると、メモリ破損や不正なアクセスが発生します。

保守担当者向けメモ

  • ai_privs のビット操作には、必ず ACLITEM_GET_* マクロを使用してください。
  • aclparse や aclitem_match などの acl.c 内の関数が、本構造体の比較や解析の基準となります。
