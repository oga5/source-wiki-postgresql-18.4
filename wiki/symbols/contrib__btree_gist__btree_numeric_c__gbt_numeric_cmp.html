gbt_numeric_cmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_numeric.c:68

目的

PostgreSQL の numeric 型に対する比較操作を GiST インデックスの構築および操作プロセスへ提供します。

シグネチャまたは形状

int32 gbt_numeric_cmp(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**: 比較対象となる2つの numeric 型データへのポインタ (a, b)、照合順序 (collation)、関数情報 (flinfo)。
  • **出力**: a が b より小さい場合は負の整数、等しい場合は 0、大きい場合は正の整数を返します。

前提条件と不変条件

  • 入力ポインタ a および b は有効な numeric 型のデータ構造を指している必要があります。
  • 内部的に PostgreSQL コアの numeric_cmp 関数を利用するため、その実行環境（メモリコンテキスト等）が整っている必要があります。

アルゴリズムまたは振る舞い

1. 入力されたポインタ a および b を PointerGetDatum を用いて Datum 型に変換します。 2. PostgreSQL コアの numeric_cmp 関数を DirectFunctionCall2 を介して呼び出します。 3. 得られた結果を DatumGetInt32 で int32 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_numeric.c 内の静的初期化構造体（tinfo 等）から参照されています。
  • **呼び出し先**: PostgreSQL コアの numeric_cmp 関数。

関連データ構造

  • numeric: PostgreSQL の可変長数値型。
  • Datum: PostgreSQL の汎用データ表現型。

失敗モードと境界条件

  • 入力が NULL の場合、DirectFunctionCall2 の仕様に従い、呼び出し元で適切な処理が行われない限りエラーが発生する可能性があります。
  • numeric_cmp が内部的に発生させるエラー（数値の不正など）は、そのまま本関数から伝播します。

保守担当者向けメモ

  • 本関数は numeric 型の比較ロジックをコア関数に委譲するアダプターです。
  • btree_gist モジュールにおける numeric 型のソートやインデックス構築の基盤となるため、比較ロジックの変更はインデックスの整合性に直結します。
