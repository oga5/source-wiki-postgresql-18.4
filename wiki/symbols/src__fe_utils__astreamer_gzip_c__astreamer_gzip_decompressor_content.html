astreamer_gzip_decompressor_content

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_gzip.c:284

目的

入力されたgzip圧縮データチャンクを解凍し、解凍後のデータを後続の astreamer パイプラインへ転送する。

シグネチャまたは形状

static void astreamer_gzip_decompressor_content(
    astreamer *streamer,
    astreamer_member *member,
    const char *data,
    int len,
    astreamer_archive_context context
)

入力と出力

  • **入力**:
  • streamer: 解凍器のインスタンス（astreamer_gzip_decompressor）。
  • member: 現在処理中のアーカイブメンバー情報。
  • data: 解凍対象の圧縮データバッファ。
  • len: data のバイト長。
  • context: アーカイブ処理のコンテキスト。
  • **出力**: なし（副作用として後続のストリーマーへデータを渡す）。

前提条件と不変条件

  • streamer は astreamer_gzip_decompressor 型としてキャスト可能であること。
  • mystreamer->bytes_written は常に bbs_buffer.maxlen 未満であること。

アルゴリズムまたは振る舞い

1. 入力された data を z_stream の入力バッファに設定する。 2. inflate() を呼び出し、データを解凍して mystreamer->base.bbs_buffer に格納する。 3. 解凍処理中にエラーが発生した場合（Z_OK, Z_STREAM_END, Z_BUF_ERROR 以外）、pg_fatal を呼び出し処理を中断する。 4. バッファが満杯になった場合、astreamer_content を呼び出して後続のストリーマーへデータを転送し、bytes_written をリセットする。 5. 入力データが残っている限り、上記処理を繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer インターフェースの content メソッド経由で呼び出される（具体的な呼び出し元関数はソースコードの構造上、動的なディスパッチが行われるため、本抜粋からは特定不可）。
  • **呼び出し先**:
  • inflate() (zlib)
  • astreamer_content() (後続のストリーマー)
  • pg_fatal()

関連データ構造

  • astreamer_gzip_decompressor: z_stream を保持する解凍器の構造体。
  • z_stream: zlibの解凍状態を管理する構造体。
  • astreamer: ストリーミング処理の基底構造体。

失敗モードと境界条件

  • **解凍エラー**: inflate() が Z_OK, Z_STREAM_END, Z_BUF_ERROR 以外の値を返した場合、致命的エラーとして終了する。
  • **バッファ境界**: bytes_written が bbs_buffer.maxlen に達した際に正しくフラッシュが行われる。

保守担当者向けメモ

  • 本関数は zlib に強く依存している。
  • astreamer_gzip_decompressor 構造体の bbs_buffer を介してデータをバッファリングしているため、メモリ管理は astreamer の基底実装に従う。
