LabelProvider

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/seclabel.c:28

目的

外部セキュリティプロバイダー（SELinux 等）が提供するラベル検証ロジックをシステムに登録・管理するための構造体です。

シグネチャまたは形状

typedef struct
{
	const char *provider_name;
	check_object_relabel_type hook;
} LabelProvider;

入力と出力

  • provider_name: プロバイダーを一意に識別する名称（文字列）。
  • hook: ラベルの妥当性を検証するための関数ポインタ（check_object_relabel_type 型）。

前提条件と不変条件

  • provider_name は、label_provider_list 内で一意である必要があります。
  • hook は、セキュリティラベルの付与時に呼び出される検証ロジックを指す必要があります。

アルゴリズムまたは振る舞い

  • 拡張機能が register_label_provider を通じて本構造体を登録します。
  • SECURITY LABEL コマンド実行時、指定されたプロバイダー名と一致する LabelProvider が label_provider_list から検索されます。
  • 検索されたプロバイダーの hook が呼び出され、対象オブジェクトに対するラベルの妥当性が検証されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/seclabel.c 内の register_label_provider および ExecSecLabelStmt 等。
  • 呼び出し先: hook フィールドに格納された関数（check_object_relabel_type）。

関連データ構造

  • label_provider_list: 登録された LabelProvider のリストを保持する静的変数。

失敗モードと境界条件

  • プロバイダーが複数ロードされている状態で、SQL 文でプロバイダー名が明示されない場合、検証は失敗します。
  • 登録されていないプロバイダー名を指定した場合、検索は失敗します。

保守担当者向けメモ

  • 本構造体はラベルの「保存場所」を管理するものであり、ラベル自体の意味や妥当性検証は hook に委譲されています。
  • provider_name の比較は、プロバイダーの識別に不可欠です。
