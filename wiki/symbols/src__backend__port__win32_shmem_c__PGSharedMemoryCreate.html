PGSharedMemoryCreate

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/port/win32_shmem.c:206

目的

Windows 環境において、PostgreSQL の共有メモリセグメントを新規作成し、プロセスのアドレス空間にマッピングする。

シグネチャまたは形状

PGShmemHeader *PGSharedMemoryCreate(Size size, PGShmemHeader **shim)

入力と出力

  • **入力**:
  • size: 要求する共有メモリのサイズ（バイト単位）。
  • shim: PGShmemHeader へのポインタを格納する出力用引数。
  • **出力**:
  • 成功時: マッピングされた共有メモリの先頭アドレス（PGShmemHeader 型）。
  • 失敗時: elog(FATAL) によりプロセスを終了。

前提条件と不変条件

  • size は PGShmemHeader を格納するのに十分な大きさであること（Assert で検証）。
  • huge_pages 設定が有効な場合、OS の「メモリ内のページをロック」権限が必要。
  • 共有メモリ領域の断片化を防ぐため、VirtualAlloc による保護領域（ShmemProtectiveRegion）の確保が先行して行われる。

アルゴリズムまたは振る舞い

1. **保護領域の確保**: VirtualAlloc で PROTECTIVE_REGION_SIZE 分のメモリを予約。 2. **Huge Pages 設定**: huge_pages が有効な場合、SEC_LARGE_PAGES を使用し、必要に応じてサイズを調整。 3. **セグメント作成**: CreateFileMapping を呼び出し、共有メモリセグメントを作成。既存の場合は最大10回までリトライ（1秒間隔）。 4. **ハンドル継承**: DuplicateHandle を使用して、子プロセスに継承可能なハンドルを作成。 5. **マッピング**: MapViewOfFileEx を使用して、共有メモリをプロセスのアドレス空間にマッピング。 6. **初期化**: マッピングされた領域の先頭に PGShmemHeader を配置し、creatorPID を設定。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/ipci.c の CreateSharedMemoryAndSemaphores から呼び出されることが確認されている。
  • **呼び出し先**: GetSharedMemName, GetLargePageMinimum, EnableLockPagesPrivilege, CreateFileMapping, DuplicateHandle, MapViewOfFileEx, getpid 等の Windows API および内部関数。

関連データ構造

  • PGShmemHeader: 共有メモリの先頭に配置される管理構造体。
  • UsedShmemSegAddr: 現在使用中の共有メモリのベースアドレスを保持するグローバル変数。

失敗モードと境界条件

  • **リソース不足**: CreateFileMapping が ERROR_NO_SYSTEM_RESOURCES を返した場合、Huge Pages を無効化して再試行するロジックがある。
  • **競合**: 既に同名の共有メモリが存在し、解放されない場合は FATAL エラーとなる。
  • **権限不足**: Huge Pages 使用時に権限がない場合、設定に応じてエラーまたは警告が発生する。

保守担当者向けメモ

  • Windows のアドレス空間の断片化を避けるため、ShmemProtectiveRegion の予約処理は必須である。
  • CreateFileMapping のリトライ処理は、前回のプロセスが完全に終了していないケースを考慮した暫定的な実装である。
  • hdr->creatorPID の設定順序は、他プロセスがヘッダーを誤認しないよう慎重に制御されている。
