bt_multi_page_stats

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/btreefuncs.c:342

目的

指定されたB-treeインデックスのページ範囲について、各ページの統計情報をSRF（Set Returning Function）として取得する。

シグネチャまたは形状

Datum bt_multi_page_stats(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • relname (text): インデックス名。
  • blkno (int64): 開始ブロック番号。
  • blk_count (int64): 取得するページ数（負の値を指定すると全ページを対象とする）。
  • **出力**:
  • ページごとの統計情報を含む行セット（BTPageStat 構造体の各フィールドに対応する11列）。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要。
  • 対象リレーションは有効なB-treeインデックスであること。
  • 処理中は対象インデックスに対して AccessShareLock を保持する。

アルゴリズムまたは振る舞い

1. **初期化**: SRF_IS_FIRSTCALL にてリレーションを開き、bt_index_block_validate で対象がB-treeであることを検証する。blk_count が正の場合は範囲をチェックする。 2. **状態保持**: FuncCallContext の user_fctx に ua_page_stats を格納し、リレーションのOIDと現在のブロック位置を維持する。 3. **反復処理**:

  • 各呼び出しで ReadBuffer を使用して対象ページを読み込み、共有ロックを取得する。
  • GetBTPageStatistics を呼び出してページ統計を収集する。
  • 収集した統計を文字列配列に変換し、BuildTupleFromCStrings で行を構築して返す。
  • blkno をインクリメントし、blk_count をデクリメントする。

    • 4. **終了**: blk_count が0に達した時点でロックを解放し、SRF_RETURN_DONE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（PG_FUNCTION_INFO_V1 マクロによる登録）。
  • **呼び出し先**:
  • bt_index_block_validate, check_relation_block_range (検証)
  • GetBTPageStatistics (統計収集)
  • relation_open, relation_close (リレーション管理)
  • ReadBuffer, LockBuffer, UnlockReleaseBuffer (バッファ管理)

関連データ構造

  • BTPageStat: ページ統計情報を保持する構造体。
  • ua_page_stats: SRFの反復間で状態を保持するための内部構造体。
  • FuncCallContext: SRFの実行状態管理用。

失敗モードと境界条件

  • スーパーユーザー権限がない場合、エラーを報告する。
  • 指定されたリレーションがB-treeでない場合、またはブロック範囲が不正な場合、エラーを報告する。
  • 戻り値の型が期待される複合型と一致しない場合、エラーを報告する。
  • blk_count が負の場合、インデックスの全ページを走査するモード（allpages）として動作する。

保守担当者向けメモ

  • 統計情報の各フィールドは文字列として構築されるため、将来的な型変更時には BuildTupleFromCStrings の引数構成に注意すること。
