AppendIncrementalManifestData

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_incremental.c:193

目的

増分バックアップ用マニフェストデータの断片を逐次的に受け取り、内部バッファへの蓄積および必要に応じたインクリメンタルなJSON解析を実行します。

シグネチャまたは形状

void AppendIncrementalManifestData(IncrementalBackupInfo *ib, const char *data, int len)

入力と出力

  • ib: 増分バックアップのメタデータと状態を保持する IncrementalBackupInfo 構造体へのポインタ。
  • data: 追加するマニフェストデータの文字列バッファ。
  • len: 追加するデータの長さ。
  • 出力: なし（副作用として ib 内部のバッファおよび解析状態が更新されます）。

前提条件と不変条件

  • ib は CreateIncrementalBackupInfo によって適切に初期化されている必要があります。
  • 内部メモリコンテキスト (ib->mcxt) が有効である必要があります。

アルゴリズムまたは振る舞い

1. ib->mcxt にメモリコンテキストを切り替えます。 2. 蓄積されたバッファサイズが MAX_CHUNK を超える場合、MIN_CHUNK を残して json_parse_manifest_incremental_chunk を呼び出し、部分的なJSON解析を実行します。 3. 解析済みのデータをバッファから削除し、残りのデータを先頭に寄せます。 4. 新しいデータ data をバッファの末尾に追加します。 5. 元のメモリコンテキストに復帰します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/walsender.c (確認済み)
  • 呼び出し先: json_parse_manifest_incremental_chunk (JSON解析用外部関数)

関連データ構造

  • IncrementalBackupInfo: 増分バックアップの解析状態を保持する主要構造体。

失敗モードと境界条件

  • バッファサイズが MIN_CHUNK を下回る間は解析が保留され、蓄積のみが行われます。
  • メモリコンテキストの切り替え失敗や、JSON形式の不正による解析エラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はマニフェストのストリーミング解析を前提としています。MIN_CHUNK および MAX_CHUNK の定数は、解析効率とメモリ消費のトレードオフを決定します。
  • memmove を使用してバッファの先頭を詰める処理が含まれるため、バッファサイズが極端に大きい場合はパフォーマンスに影響する可能性があります。
