WalkInnerWith

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_cte.c:811

目的

ネストされた WITH 句（CTE）の依存関係を解析し、名前の可視性スコープを管理する。

シグネチャまたは形状

static void WalkInnerWith(Node *stmt, WithClause *withClause, CteState *cstate)

入力と出力

  • stmt: 解析対象のノード。
  • withClause: 処理対象の WITH 句構造体。
  • cstate: CTEの解析状態を保持する CteState 構造体。
  • 出力: なし（cstate->innerwiths を更新し、依存関係をグラフに反映させる）。

前提条件と不変条件

  • withClause が再帰的か否かに応じて、cstate->innerwiths のスタック操作が正しく行われる必要がある。
  • cstate->innerwiths は、現在のスコープで参照可能なCTEのリストを保持するスタックとして機能する。

アルゴリズムまたは振る舞い

1. **再帰的CTEの場合 (recursive == true)**:

  • すべてのCTE名を cstate->innerwiths にプッシュする。
  • 各CTEのクエリおよびメインの stmt に対して makeDependencyGraphWalker を実行し、依存関係を特定する。
  • 処理終了後、スタックからポップする。

    • 2. **非再帰的CTEの場合 (recursive == false)**:

  • 空のリストを cstate->innerwiths にプッシュする。
  • 各CTEを順次処理し、処理済みのCTEを現在のスコープ（innerwiths の先頭）に追加する。これにより、後続のCTEから先行するCTEが参照可能になる。
  • 最後に stmt に対して makeDependencyGraphWalker を実行し、処理終了後にポップする。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/parser/parse_cte.c 内の各箇所（makeDependencyGraphWalker 等から再帰的に呼び出される）。
  • 呼び出し先: makeDependencyGraphWalker、raw_expression_tree_walker。

関連データ構造

  • WithClause: WITH 句の構文情報。
  • CteState: 解析状態を保持する構造体。
  • CommonTableExpr: 個別のCTE定義。

失敗モードと境界条件

  • innerwiths リストの操作中に cstate の状態が不整合になると、名前解決エラーやクラッシュを引き起こす可能性がある。
  • 再帰的CTEの定義がSQL標準に違反している場合、後続の検証フェーズでエラーとなる。

保守担当者向けメモ

  • innerwiths はスタック構造であり、lcons と list_delete_first によるプッシュ・ポップが必須である。
  • 非再帰的CTEにおける「後続のCTEから先行するCTEへの参照」を許可するロジックは、innerwiths の先頭リストへの追加によって実現されている。
