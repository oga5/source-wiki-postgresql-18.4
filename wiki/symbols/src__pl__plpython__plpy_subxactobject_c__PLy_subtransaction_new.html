PLy_subtransaction_new

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_subxactobject.c:66

目的

Python側から plpy.subtransaction() が呼び出された際に、新しいサブトランザクション管理オブジェクト（PLySubtransactionObject）を生成・初期化します。

シグネチャまたは形状

PyObject *PLy_subtransaction_new(PyObject *self, PyObject *unused)

入力と出力

  • **入力**:
  • self: Pythonのメソッド呼び出しにおけるインスタンスまたはモジュールオブジェクト。
  • unused: 使用されない引数。
  • **出力**:
  • 成功時: 新規生成された PLySubtransactionObject へのポインタ（PyObject *）。
  • 失敗時: NULL。

前提条件と不変条件

  • PLy_SubtransactionType が事前に初期化されている必要があります。
  • 生成されたオブジェクトの started および exited フラグは、初期状態として false に設定されます。

アルゴリズムまたは振る舞い

1. PyObject_New を使用して PLySubtransactionObject 型のメモリを確保し、初期化します。 2. Python 3.8 未満の環境では、PLy_SubtransactionType の参照カウントをインクリメントし、Pythonのバージョン依存の問題（Issue 35810）を回避します。 3. オブジェクトの内部状態である started と exited を false に初期化します。 4. 初期化されたオブジェクトを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_plpymodule.c:87 にて、plpy モジュールのメソッド定義の一部として登録されています。
  • **呼び出し先**:
  • PyObject_New (Python C API)
  • Py_INCREF (Python C API)

関連データ構造

  • PLySubtransactionObject: サブトランザクションの状態を保持する構造体。
  • PLy_SubtransactionType: Python型システムにおけるサブトランザクションの型定義。

失敗モードと境界条件

  • PyObject_New がメモリ不足等で失敗した場合、NULL を返します。

保守担当者向けメモ

  • 本関数は単なるオブジェクトのインスタンス化を行うものであり、実際のサブトランザクションの開始（BeginInternalSubTransaction の呼び出し）は、後続の __enter__ メソッド（PLy_subtransaction_enter）で行われます。
