simple_oid_list_destroy

定義

  • 種別: function
  • 定義箇所: src/fe_utils/simple_list.c:105

目的

SimpleOidList 構造体によって管理されている単方向連結リストの全セルをメモリから解放する。

シグネチャまたは形状

void simple_oid_list_destroy(SimpleOidList *list)

入力と出力

  • **入力**: list - 解放対象の SimpleOidList 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • list は有効な SimpleOidList 構造体を指している必要がある。
  • リスト内の各セルは pg_malloc 等で確保されたメモリ領域である必要がある。

アルゴリズムまたは振る舞い

1. リストの先頭 (list->head) から走査を開始する。 2. 各セルについて、次のセルへのポインタを一時的に保持する。 3. pg_free を呼び出し、現在のセルを解放する。 4. 次のセルへ移動し、リストが空になるまで繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/scripts/reindexdb.c:483 での呼び出しが確認されている。
  • **呼び出し先**: pg_free（メモリ解放用ユーティリティ）。

関連データ構造

  • SimpleOidList: リストのヘッドを保持する構造体。
  • SimpleOidListCell: リストの各要素を構成するセル構造体。

失敗モードと境界条件

  • list が NULL の場合、あるいはリストが既に空（head が NULL）の場合、関数は何も行わず正常に終了する。

保守担当者向けメモ

  • 本関数はリストの各セルを解放するが、SimpleOidList 構造体自体のメモリ解放は呼び出し元で行う必要がある。
  • simple_ptr_list_destroy とは異なり、OID は値として保持されているため、本関数による解放でリストの内容は完全に破棄される。
