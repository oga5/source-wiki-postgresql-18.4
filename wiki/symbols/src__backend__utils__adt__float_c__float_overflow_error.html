float_overflow_error

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/float.c:85

目的

浮動小数点演算においてオーバーフローが発生した際、PostgreSQLの標準的なエラー報告メカニズムを通じて例外を発生させる。

シグネチャまたは形状

void float_overflow_error(void)

入力と出力

  • 入力: なし
  • 出力: なし（関数は正常に復帰せず、ereport によりエラーをスローする）

前提条件と不変条件

  • 本関数は pg_noinline として定義されており、呼び出し元でのインライン展開は抑制される。
  • 呼び出しにより、トランザクションの異常終了を伴う ERROR レベルの例外が発行される。

アルゴリズムまたは振る舞い

1. ereport マクロを呼び出す。 2. エラーコード ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE を設定する。 3. "value out of range: overflow" というメッセージをエラーログに出力する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/adt/float.c 内の各種算術演算関数（浮動小数点演算の結果が範囲外となった箇所）。
  • contrib/btree_gist/btree_float4.c および contrib/btree_gist/btree_float8.c 内のインデックス処理関数。
  • 呼び出し先:
  • ereport（PostgreSQLの内部エラー報告関数）。

関連データ構造

  • なし

失敗モードと境界条件

  • 本関数自体がエラー処理を目的としているため、本関数内での失敗モードは定義されていない。
  • 呼び出し元が浮動小数点演算のオーバーフローを検知した直後に呼び出されることが前提である。

保守担当者向けメモ

  • 本関数は、浮動小数点演算エラーの報告を一元化するために使用されている。エラーメッセージやエラーコードを変更する場合は、他の類似関数（float_underflow_error 等）との整合性に注意すること。
