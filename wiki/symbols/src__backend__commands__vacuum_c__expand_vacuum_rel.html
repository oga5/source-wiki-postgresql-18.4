expand_vacuum_rel

定義

  • 種別: function
  • 定義箇所: src/backend/commands/vacuum.c:898

目的

VACUUM または ANALYZE コマンドの対象リレーションを拡張し、継承関係にある子テーブルやパーティションを含む処理対象リストを生成する。

シグネチャまたは形状

List *expand_vacuum_rel(VacuumRelation *vrel, MemoryContext vac_context, int options)

入力と出力

  • **入力**:
  • vrel: 処理対象の VacuumRelation 構造体。
  • vac_context: リストを割り当てるためのメモリコンテキスト。
  • options: VACOPT_* フラグを含む整数値。
  • **出力**:
  • 処理対象となる VacuumRelation のリスト (List *)。

前提条件と不変条件

  • vrel->oid が有効な場合、そのリレーションのみを含むリストを即座に返す。
  • AmAutoVacuumWorkerProcess() が真となるプロセスからは呼び出されないこと（アサーションにより強制）。
  • パーティションテーブルに対して ONLY オプションが指定された場合、処理は行われず警告が発行される。

アルゴリズムまたは振る舞い

1. **OIDの確認**: vrel->oid が有効なら、そのリレーションをリストに追加して終了する。 2. **リレーションの特定**: RangeVarGetRelidExtended を使用して対象リレーションの OID を取得する。ロックが取得できない場合は警告を発して空リストを返す。 3. **権限チェック**: vacuum_is_permitted_for_relation を呼び出し、現在のユーザーが対象リレーションを処理する権限を持つか確認する。 4. **継承関係の探索**: vrel->relation->inh が真（ONLY 指定なし）の場合、find_all_inheritors を用いて子テーブルおよびパーティションの OID を取得し、リストに追加する。 5. **クリーンアップ**: 取得したシステムキャッシュを解放し、一時的に取得した AccessShareLock を UnlockRelationOid で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/vacuum.c 内の vacuum() 関数から呼び出される。
  • **呼び出し先**: RangeVarGetRelidExtended, SearchSysCache1, vacuum_is_permitted_for_relation, makeVacuumRelation, find_all_inheritors, UnlockRelationOid 等。

関連データ構造

  • VacuumRelation: VACUUM 対象を定義する構造体。
  • List: PostgreSQL のリスト構造。
  • MemoryContext: メモリ管理コンテキスト。

失敗モードと境界条件

  • **ロック競合**: VACOPT_SKIP_LOCKED が指定されている場合、ロックが取得できなければ警告を出してスキップする。
  • **キャッシュルックアップ失敗**: SearchSysCache1 でリレーションが見つからない場合、elog(ERROR) により処理を中断する。
  • **デッドロック回避**: 継承テーブルの探索時にはロックを保持せず、処理対象リストへの追加のみを行う。

保守担当者向けメモ

  • 本関数はトランザクションをまたいで実行されることを想定しており、ロックを一時的に取得・解放する設計となっている。
  • 処理対象の OID を取得した後にテーブルが削除・改名される可能性があるため、後続の処理で RangeVar が無効化されるリスクを考慮する必要がある。
