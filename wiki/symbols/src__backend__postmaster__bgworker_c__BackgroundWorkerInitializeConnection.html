BackgroundWorkerInitializeConnection

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/bgworker.c:852

目的

バックグラウンドワーカープロセスがデータベースに接続し、初期化処理を行うためのインターフェースを提供します。

シグネチャまたは形状

void BackgroundWorkerInitializeConnection(const char *dbname, const char *username, uint32 flags)

入力と出力

  • dbname: 接続先データベース名（NULL可）。
  • username: 接続に使用するロール名（NULL可）。
  • flags: 接続時の挙動を制御するビットフラグ。
  • BGWORKER_BYPASS_ALLOWCONN: datallowconn チェックをスキップ。
  • BGWORKER_BYPASS_ROLELOGINCHECK: rolcanlogin チェックをスキップ。
  • 戻り値: なし。

前提条件と不変条件

  • ワーカー登録時に BGWORKER_BACKEND_DATABASE_CONNECTION フラグが設定されている必要があります。設定されていない場合、FATAL エラーとなります。
  • 呼び出し時点でプロセスが Init モードである必要があります。

アルゴリズムまたは振る舞い

1. flags に基づき、InitPostgres に渡す内部フラグ（INIT_PG_OVERRIDE_ALLOW_CONNS, INIT_PG_OVERRIDE_ROLE_LOGIN）を構築します。 2. ワーカーの登録情報（MyBgworkerEntry）を確認し、データベース接続が許可されているか検証します。 3. InitPostgres を呼び出し、指定されたデータベースとユーザーでセッションを初期化します。 4. 現在の処理モードが InitProcessingMode であることを確認し、その後 SetProcessingMode(NormalProcessing) を呼び出して通常処理モードへ移行します。

呼び出し元と呼び出し先

  • 呼び出し元（直接の参照）:
  • src/backend/replication/logical/launcher.c
  • src/test/modules/worker_spi/worker_spi.c
  • 呼び出し先:
  • InitPostgres
  • SetProcessingMode
  • ereport

関連データ構造

  • BackgroundWorker (MyBgworkerEntry を介して参照)

失敗モードと境界条件

  • 登録時にデータベース接続フラグがない場合: ERRCODE_PROGRAM_LIMIT_EXCEEDED で FATAL 終了。
  • 呼び出し時の処理モードが InitProcessingMode でない場合: ERROR で終了。

保守担当者向けメモ

  • InitPostgres の呼び出しにおいて、session_preload_libraries は意図的に無視される仕様です。
  • SetProcessingMode による状態遷移は、ワーカーが初期化を完了し、通常のクエリ実行が可能な状態へ移行するために必須です。
