pivot_field

定義

  • 種別: struct
  • 定義箇所: src/bin/psql/crosstabview.c:22

目的

\crosstabview コマンドにおいて、クロス集計表の行または列のヘッダー情報を保持するために使用されるデータ構造です。

シグネチャまたは形状

typedef struct _pivot_field
{
    char   *name;        /* PQgetvalue() から取得したヘッダー値 */
    char   *sort_value;  /* ソート用カラムの値 */
    int     rank;        /* 0から始まる順位 */
} pivot_field;

入力と出力

  • **入力**: PQgetvalue() を介して取得された文字列ポインタ（name, sort_value）。
  • **出力**: rank フィールドは、初期状態では出現順、ソート指定時は rankSort 関数による計算結果が格納されます。

前提条件と不変条件

  • name および sort_value が NULL の場合、SQL の NULL 値として扱われます。
  • rank は 0 以上の整数です。
  • 同一の name が複数回出現する場合、最初に出現した際の sort_value が保持されます。

アルゴリズムまたは振る舞い

1. **初期化**: クエリ結果の走査時に、一意な値が pivot_field として抽出されます。 2. **ランク付け**: 初期状態では、結果セット内での出現順序が rank に割り当てられます。 3. **ソート**: ソート列が指定されている場合、最終フェーズで rankSort が呼び出され、指定された順序に基づいて rank が更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/crosstabview.c 内の PrintResultInCrosstab および関連する集計処理関数。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • avl_tree: ヘッダー値の重複排除とソート済み配列生成に使用されます。
  • printTable: 最終的なクロス集計表の出力に使用されます。

失敗モードと境界条件

  • **メモリ不足**: pg_malloc によるメモリ確保失敗時にプロセスが終了する可能性があります。
  • **列数制限**: CROSSTABVIEW_MAX_COLUMNS (1600) を超えるヘッダーが生成される場合、エラーとなります。
  • **ソート列の不整合**: ソート列が数値として解釈できない場合、ランク 0 として扱われます。

保守担当者向けメモ

  • 本構造体は crosstabview.c 内で動的に配列として確保されます（pg_malloc(sizeof(pivot_field) * num_columns) 等）。
  • name と sort_value は libpq の結果セットへのポインタであるため、元の結果セットが解放される前に本構造体を使用する必要があります。
