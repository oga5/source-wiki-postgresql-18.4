lexstart

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_lex.c:69

目的

正規表現コンパイルの字句解析フェーズを開始し、初期オプションの処理と解析コンテキストの決定、および最初のトークンの読み込みを行う。

シグネチャまたは形状

void lexstart(struct vars *v)

入力と出力

  • **入力**: struct vars *v （正規表現コンパイルの状態を保持する構造体）
  • **出力**: なし（副作用として v 内のコンテキスト、nexttype、および最初のトークン情報が更新される）

前提条件と不変条件

  • v は有効な vars 構造体へのポインタであること。
  • prefixes(v) の実行後にエラーが発生していないこと（NOERR() マクロによる検証）。
  • REG_QUOTE、REG_EXTENDED、REG_ADVANCED などのフラグ設定に基づき、排他的なコンテキスト（L_Q, L_ERE, L_BRE）が正しく選択されること。

アルゴリズムまたは振る舞い

1. prefixes(v) を呼び出し、正規表現の先頭に付与されたフラグ（*** や (?...)）を解析・適用する。 2. v->cflags を参照し、以下のいずれかのコンテキストを INTOCON マクロで設定する。

  • REG_QUOTE が有効な場合: L_Q
  • REG_EXTENDED が有効な場合: L_ERE
  • それ以外の場合: L_BRE

    • 3. v->nexttype を EMPTY に初期化する。 4. next(v) を呼び出し、最初のトークンを特定して v に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regcomp.c:454
  • **呼び出し先**:
  • prefixes(v)
  • next(v)
  • NOERR() （マクロ）
  • INTOCON() （マクロ）

関連データ構造

  • struct vars: 正規表現コンパイルの全状態を保持する主要構造体。
  • lexcon: 現在の字句解析コンテキスト（L_Q, L_ERE, L_BRE）。

失敗モードと境界条件

  • prefixes(v) 内でエラーが発生した場合、NOERR() により処理が中断される。
  • REG_QUOTE と他のフラグ（REG_ADVANCED 等）が同時に指定されている場合、assert によりデバッグビルド時に検証される。

保守担当者向けメモ

  • 本関数は regcomp.c から呼び出される初期化ルーチンである。
  • lexcon の状態遷移は字句解析の根幹であるため、新しい正規表現フレーバーを追加する際は、ここでのコンテキスト選択ロジックに注意が必要である。
