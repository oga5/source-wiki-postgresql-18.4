PLy_info

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_plpymodule.c:289

目的

Python スクリプトから PostgreSQL の INFO レベルのメッセージをログ出力するためのインターフェースを提供します。

シグネチャまたは形状

static PyObject *PLy_info(PyObject *self, PyObject *args, PyObject *kw);

入力と出力

  • **入力**:
  • self: Python モジュールオブジェクト。
  • args: Python から渡された可変長引数（タプル）。
  • kw: Python から渡されたキーワード引数（辞書）。
  • **出力**:
  • PLy_output 関数の戻り値（通常は Py_None を含む PyObject*）。

前提条件と不変条件

  • 本関数は Python の C API 規約に従い、PyObject ポインタを引数として受け取ります。
  • 内部で呼び出される PLy_output が、Python の引数を PostgreSQL の ereport 形式へ変換できる必要があります。

アルゴリズムまたは振る舞い

1. INFO 定数を引数として PLy_output 関数を呼び出します。 2. PLy_output が Python の引数を解析し、PostgreSQL のログ出力機能（ereport）を介してメッセージを送信します。 3. PLy_output の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Python インタプリタ（plpy.info() メソッド経由）。
  • **呼び出し先**: PLy_output（src/pl/plpython/plpy_plpymodule.c 内で定義）。
  • **登録**: PLy_methods テーブルにて info メソッドとして登録されています。

関連データ構造

  • PyObject: Python のオブジェクト表現。
  • PLy_methods: plpy モジュールのメソッド定義テーブル。

失敗モードと境界条件

  • PLy_output 内部での型変換エラーや、PostgreSQL 側のメモリ不足等により ereport がエラーを発生させた場合、Python の例外として再送出される可能性があります。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際のログ出力ロジックは PLy_output に集約されています。
  • ログレベルを変更する場合は、PLy_output に渡す定数を変更してください。
  • Python の参照カウント管理は PLy_output 側で行われるため、本関数内での追加の参照操作は不要です。
