basebackup_progress_wait_wal_archive

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_progress.c:205

目的

ベースバックアップの進捗状況を pg_stat_progress_basebackup ビューに反映させ、現在のフェーズが「WALアーカイブ待機中」であることをシステムに通知する。

シグネチャまたは形状

void basebackup_progress_wait_wal_archive(bbsink_state *state)

入力と出力

  • **入力**: bbsink_state *state - バックアップの現在の状態（テーブルスペースのリスト等）を保持する構造体。
  • **出力**: なし（副作用として pgstat 統計情報を更新）。

前提条件と不変条件

  • state は有効な bbsink_state 構造体を指している必要がある。
  • pgstat サブシステムが初期化されていること。

アルゴリズムまたは振る舞い

1. PROGRESS_BASEBACKUP_PHASE（フェーズ）および PROGRESS_BASEBACKUP_TBLSPC_STREAMED（ストリーム済みテーブルスペース数）を更新対象として指定する。 2. フェーズを PROGRESS_BASEBACKUP_PHASE_WAIT_WAL_ARCHIVE に設定する。 3. state->tablespaces のリスト長を、ストリーム済みのテーブルスペース数として設定する。 4. pgstat_progress_update_multi_param を呼び出し、統計情報を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup.c:395
  • **呼び出し先**: pgstat_progress_update_multi_param（pgstat.h で定義）

関連データ構造

  • bbsink_state: バックアップの進行状況やテーブルスペース情報を管理する構造体。

失敗モードと境界条件

  • state が NULL の場合、プログラムは異常終了する可能性がある。
  • pgstat 統計更新の失敗については、本関数内では明示的なエラーハンドリングを行っていない。

保守担当者向けメモ

  • 本関数は、データファイルの転送が終了し、WALの転送待ち状態へ移行するタイミングで呼び出される。
  • val[1] に list_length(state->tablespaces) を設定することで、テーブルスペースの転送完了を報告する仕様となっている。
