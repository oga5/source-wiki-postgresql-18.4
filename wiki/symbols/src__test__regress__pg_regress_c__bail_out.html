bail_out

定義

  • 種別: function
  • 定義箇所: src/test/regress/pg_regress.c:254

目的

テスト実行中に致命的なエラーが発生した際、TAP (Test Anything Protocol) 形式でエラーメッセージを出力し、プロセスを終了させるために使用されます。

シグネチャまたは形状

static void bail_out(bool noatexit, const char *fmt, ...);

入力と出力

  • noatexit (bool): true の場合、atexit ハンドラをスキップして即座に終了します。
  • fmt (const char *): 可変長引数を受け取るフォーマット文字列。
  • ...: フォーマット文字列に対応する引数。
  • 出力: なし（プロセスが終了するため）。

前提条件と不変条件

  • emit_tap_output_v を介して BAIL 型の TAP 出力が行われます。
  • 終了ステータスは常に 2 です。

アルゴリズムまたは振る舞い

1. 可変長引数リスト ap を初期化します。 2. emit_tap_output_v を呼び出し、BAIL タイプとしてエラーメッセージを標準出力へ書き出します。 3. noatexit が true の場合、_exit(2) を呼び出し、プロセスを即座に終了します。 4. それ以外の場合、exit(2) を呼び出し、標準的な終了処理を経てプロセスを終了します。

呼び出し元と呼び出し先

  • 呼び出し先: emit_tap_output_v, _exit, exit
  • 呼び出し元:
  • pg_regress.c 内の bail_noatexit マクロおよび bail マクロ経由で呼び出されます。
  • ※注: src/backend/utils/misc/guc.c に同名のラベルが存在しますが、本関数との直接的な呼び出し関係はソースコード上からは確認できません。

関連データ構造

  • なし

失敗モードと境界条件

  • 本関数自体がエラーハンドラであるため、呼び出し時点でプログラムの継続は不可能とみなされます。
  • noatexit の指定により、クリーンアップ処理（atexit）の実行可否が制御されます。

保守担当者向けメモ

  • pg_regress.c 内で bail_noatexit(...) および bail(...) マクロとしてラップされており、通常はこれらを通じて利用されます。
  • TAP 出力フォーマットを変更する場合は、emit_tap_output_v の仕様と整合性を取る必要があります。
