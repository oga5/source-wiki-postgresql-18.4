DatumTupleFields

定義

  • 種別: struct
  • 定義箇所: src/include/access/htup_details.h:134

目的

複合型（レコード型）の値を保持するデータ構造において、その型情報および物理的な長さを管理するために使用されます。

シグネチャまたは形状

typedef struct DatumTupleFields
{
    int32       datum_len_;     /* varlenaヘッダー */
    int32       datum_typmod;   /* 型修飾子 (-1 またはレコード型ID) */
    Oid         datum_typeid;   /* 複合型のOIDまたはRECORDOID */
} DatumTupleFields;

入力と出力

  • 入力: 複合型のデータ表現。
  • 出力: 構造体内の各フィールド（長さ、型修飾子、型OID）へのアクセス。

前提条件と不変条件

  • datum_len_ は varlena ヘッダーとして扱われるため、直接操作してはなりません。
  • datum_typeid はドメイン上の複合型ではなく、純粋な複合型である必要があります。
  • CoerceToDomain は物理表現を変更しないという原則に従います。

アルゴリズムまたは振る舞い

複合型データのメタデータを保持し、実行エンジンや型変換処理において、対象データがどの型に属するか、またその物理的なサイズがどれだけかを識別するために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/postgres_fdw/postgres_fdw.c、src/backend/commands/typecmds.c、src/backend/utils/adt/rowtypes.c 等で参照されています。
  • 呼び出し先: 提供されたソース情報からは、特定の関数呼び出し関係は特定できません。

関連データ構造

  • HeapTupleFields: 同様のコンテキストで定義される共用体。
  • varlena: 可変長データ型の基底構造。

失敗モードと境界条件

  • datum_typeid にドメイン上の複合型を指定した場合の挙動は定義されていません。
  • 将来的に Oid が64ビットに拡張される可能性を考慮したフィールド順序になっています。

保守担当者向けメモ

  • datum_len_ への直接アクセスは避けてください。
  • フィールドの配置は将来の Oid 拡張を考慮して設計されています。変更時はアライメントに注意してください。
