ArrayGetIntegerTypmods

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arrayutils.c:232

目的

cstring[] 型の配列を、整数値（int32）の配列に変換して取得します。主に型修飾子（typmod）のリストを解析するために使用されます。

シグネチャまたは形状

int32 *ArrayGetIntegerTypmods(ArrayType *arr, int *n)

入力と出力

  • **入力**:
  • arr: 変換対象の ArrayType ポインタ。
  • n: 配列の要素数を格納するための出力用整数ポインタ。
  • **出力**:
  • 変換された int32 値の配列（palloc 済み）。呼び出し元で pfree する必要があります。

前提条件と不変条件

  • 入力配列 arr は cstring 型である必要があります。
  • 入力配列 arr は1次元である必要があります。
  • 入力配列 arr に NULL 要素が含まれていてはなりません。

アルゴリズムまたは振る舞い

1. 配列の型が CSTRINGOID であるか検証し、異なればエラーを発生させます。 2. 配列が1次元であることを検証し、異なればエラーを発生させます。 3. 配列内に NULL が含まれていないか検証し、含まれていればエラーを発生させます。 4. deconstruct_array_builtin を使用して、配列を Datum の配列に分解します。 5. 要素数 *n に基づいて int32 配列をメモリ確保します。 6. 各要素を pg_strtoint32 で文字列から整数へ変換し、結果配列に格納します。 7. 一時的な elem_values 配列を解放し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/date.c
  • src/backend/utils/adt/numeric.c
  • src/backend/utils/adt/timestamp.c
  • src/backend/utils/adt/varbit.c
  • src/backend/utils/adt/varchar.c
  • **呼び出し先**:
  • ereport (エラー報告)
  • array_contains_nulls
  • deconstruct_array_builtin
  • palloc / pfree
  • pg_strtoint32

関連データ構造

  • ArrayType: PostgreSQLの配列内部表現。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • **エラー発生**:
  • cstring[] 以外の型が渡された場合。
  • 多次元配列が渡された場合。
  • 配列内に NULL が含まれる場合。
  • **メモリ**: palloc が失敗した場合はシステム標準のメモリ不足エラーが発生します。

保守担当者向けメモ

  • 本関数は cstring から int32 への変換を前提としています。将来的に他の型への変換が必要な場合は、別の関数を検討するか、本関数を汎用化する必要があります。
  • deconstruct_array_builtin の結果である elem_values は pfree で解放されますが、result 配列は呼び出し元が管理する責任がある点に注意してください。
