DropPreparedStatement

定義

  • 種別: function
  • 定義箇所: src/backend/commands/prepare.c:518

目的

セッション内で管理されているプリペアドステートメントを破棄し、関連するキャッシュ済み計画のリソースを解放する。

シグネチャまたは形状

void DropPreparedStatement(const char *stmt_name, bool showError)

入力と出力

  • stmt_name (const char *): 削除対象のプリペアドステートメント名。
  • showError (bool): 指定した名前のステートメントが見つからない場合にエラーを発生させるかどうか。
  • 戻り値: なし。

前提条件と不変条件

  • prepared_queries ハッシュテーブルが初期化されていること。
  • 本関数はセッションローカルなスコープで動作し、他セッションのステートメントには影響を与えない。

アルゴリズムまたは振る舞い

1. FetchPreparedStatement を呼び出し、指定された名前のステートメントをハッシュテーブルから検索する。showError が真でステートメントが存在しない場合はエラーを発生させる。 2. ステートメントが見つかった場合、DropCachedPlan を呼び出して plansource（キャッシュ済み計画）の参照カウントを減らし、リソースを解放する。 3. hash_search を HASH_REMOVE モードで呼び出し、prepared_queries ハッシュテーブルから該当エントリを削除する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/prepare.c (DEALLOCATE文処理など)
  • src/backend/tcop/postgres.c (拡張クエリプロトコルでのクローズ処理)
  • **呼び出し先**:
  • FetchPreparedStatement
  • DropCachedPlan
  • hash_search

関連データ構造

  • PreparedStatement: プリペアドステートメントの情報を保持する構造体。
  • prepared_queries: セッション内の全プリペアドステートメントを格納するハッシュテーブル。
  • CachedPlanSource: DropCachedPlan を通じて管理される計画キャッシュのソース。

失敗モードと境界条件

  • 指定された stmt_name が存在しない場合、showError が真であれば FetchPreparedStatement 内で ereport(ERROR) が発生する。
  • 存在しない場合に showError が偽であれば、何も行わず正常終了する。

保守担当者向けメモ

  • 本関数は DEALLOCATE コマンドのバックエンド処理の核となる。
  • DropCachedPlan による計画の解放は、計画キャッシュの参照カウント管理と密接に関連しているため、変更時は plancache.c の仕様を確認すること。
