LockRelationOid

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lmgr.c:106

目的

指定されたリレーション（OID）に対して指定されたモードのロックを取得し、必要に応じてリレーションキャッシュのインバリデーションメッセージを処理することで、キャッシュの整合性を保証する。

シグネチャまたは形状

void LockRelationOid(Oid relid, LOCKMODE lockmode)

入力と出力

  • **入力**:
  • relid (Oid): ロック対象のリレーション識別子。
  • lockmode (LOCKMODE): 取得するロックのモード。
  • **出力**: なし（成功時はロックを取得して終了、失敗時は例外をスロー）。

前提条件と不変条件

  • SetLocktagRelationOid を用いて、入力された relid から LOCKTAG が構築される。
  • ロック取得後、既に同一モードのロックを保持している場合（LOCKACQUIRE_ALREADY_CLEAR）を除き、AcceptInvalidationMessages() を呼び出してキャッシュの整合性を維持する必要がある。

アルゴリズムまたは振る舞い

1. SetLocktagRelationOid を呼び出し、対象リレーションのタグを生成する。 2. LockAcquireExtended を呼び出し、共有メモリ上でロックを取得する。 3. 取得結果が LOCKACQUIRE_ALREADY_CLEAR でない場合、AcceptInvalidationMessages() を実行して stale なリレーションキャッシュエントリを更新またはフラッシュする。 4. MarkLockClear(locallock) を呼び出し、当該ロックに対するインバリデーション処理が完了したことをマークする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_prewarm/pg_prewarm.c
  • src/backend/access/common/relation.c
  • src/backend/catalog/dependency.c
  • src/backend/catalog/heap.c
  • src/backend/catalog/index.c
  • src/backend/catalog/namespace.c
  • **呼び出し先**:
  • SetLocktagRelationOid

  • AcceptInvalidationMessages
  • MarkLockClear

関連データ構造

  • LOCKTAG: ロック対象を識別するための構造体。
  • LOCALLOCK: プロセスローカルなロック管理情報。

失敗モードと境界条件

  • LockAcquireExtended がロック取得に失敗した場合、例外が発生し処理が中断される可能性がある。
  • 再帰的な呼び出しにおいて、既にインバリデーションメッセージを処理中の場合、LOCKACQUIRE_ALREADY_CLEAR の判定により二重処理が回避される。

保守担当者向けメモ

  • 本関数は RangeVarGetRelid() 等のキャッシュ整合性に依存する処理の基盤となっている。
  • インバリデーション処理の省略は、古い relcache エントリの使用による予期せぬエラーを招くため、慎重に行うこと。
  • 自身のトランザクションによる変更は CommandCounterIncrement で処理されるため、本関数内でのインバリデーション対象外である。
