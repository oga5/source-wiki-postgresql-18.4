AlterObjectNamespace_oid

定義

  • 種別: function
  • 定義箇所: src/backend/commands/alter.c:624

目的

指定されたオブジェクトを新しいスキーマ（名前空間）へ移動するための汎用的なディスパッチャ関数です。

シグネチャまたは形状

Oid AlterObjectNamespace_oid(Oid classId, Oid objid, Oid nspOid, ObjectAddresses *objsMoved)

入力と出力

  • **入力**:
  • classId: 対象オブジェクトのシステムカタログOID。
  • objid: 対象オブジェクトのOID。
  • nspOid: 移動先のスキーマOID。
  • objsMoved: 移動対象オブジェクトを追跡するための ObjectAddresses 構造体へのポインタ。
  • **出力**:
  • Oid: 移動前の古いスキーマOIDを返します。

前提条件と不変条件

  • classId に対応するオブジェクト型がスキーマ修飾をサポートしている必要があります。
  • サポート外のオブジェクト型が渡された場合、Assert により検証されます。

アルゴリズムまたは振る舞い

1. classId に基づき、オブジェクト型に応じた処理へ分岐します。 2. **テーブル (RelationRelationId)**: relation_open で排他ロックを取得し、AlterTableNamespaceInternal を呼び出して移動を実行後、ロックを解放します。 3. **型 (TypeRelationId)**: AlterTypeNamespace_oid を呼び出します。 4. **その他カタログオブジェクト**: ProcedureRelationId や OperatorRelationId 等のリストに含まれる場合、table_open で行排他ロックを取得し、AlterObjectNamespace_internal を呼び出してカタログ更新を行い、table_close でロックを解放します。 5. スキーマ修飾を持たないオブジェクト型が渡された場合、get_object_attnum_namespace が InvalidAttrNumber を返すことを確認します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/alter.c (自身)
  • src/backend/commands/extension.c
  • src/backend/commands/typecmds.c
  • **呼び出し先**:
  • AlterTableNamespaceInternal
  • AlterTypeNamespace_oid
  • AlterObjectNamespace_internal
  • relation_open / relation_close
  • table_open / table_close

関連データ構造

  • ObjectAddresses: 移動対象のオブジェクト集合を管理する構造体。
  • Relation: テーブル操作時に使用されるリレーション記述子。

失敗モードと境界条件

  • 未対応の classId が渡された場合、Assert が失敗します。
  • ロック取得時にデッドロックや権限不足が発生した場合、PostgreSQLの標準的なエラー処理メカニズムにより中断されます。

保守担当者向けメモ

  • 新しいオブジェクト型をスキーマ移動可能にする場合は、switch 文のケース節に追加し、適切な内部関数（AlterObjectNamespace_internal 等）を呼び出すようにしてください。
  • get_object_attnum_namespace が正しく実装されていることが前提となります。
