TransactionIdAsyncCommitTree

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/transam.c:251

目的

トランザクションとそのサブトランザクション群のステータスを「コミット済み」として一括設定します。非同期コミットに関連するログシーケンス番号（LSN）を記録するために使用されます。

シグネチャまたは形状

void TransactionIdAsyncCommitTree(TransactionId xid, int nxids, TransactionId *xids, XLogRecPtr lsn)

入力と出力

  • xid: 親トランザクションのID。
  • nxids: サブトランザクションの数。
  • xids: サブトランザクションIDの配列。
  • lsn: このコミットに関連付けられるログシーケンス番号。
  • 戻り値: なし。

前提条件と不変条件

  • TransactionIdSetTreeStatus を呼び出すためのラッパーとして機能します。
  • 渡される xids 配列は、指定された nxids の数だけ有効である必要があります。

アルゴリズムまたは振る舞い

1. TransactionIdSetTreeStatus を呼び出し、親トランザクション xid およびサブトランザクション配列 xids のステータスを TRANSACTION_STATUS_COMMITTED に設定します。 2. その際、指定された lsn をコミット情報として記録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c:1531
  • src/backend/access/transam/xact.c:6186
  • **呼び出し先**:
  • TransactionIdSetTreeStatus (内部関数)

関連データ構造

  • TransactionId: トランザクションを一意に識別する32ビット値。
  • XLogRecPtr: WAL内の位置を示すログシーケンス番号。

失敗モードと境界条件

  • TransactionIdSetTreeStatus の内部実装に依存します。CLOG（Commit Log）の共有メモリ領域への書き込みが発生するため、メモリ不足やロック競合が発生する可能性があります。

保守担当者向けメモ

  • 本関数は TransactionIdCommitTree と同様の処理を行いますが、非同期コミットの文脈で明示的にLSNを扱うために提供されています。
  • TransactionIdSetTreeStatus の挙動変更が本関数に直接影響するため、修正時は注意してください。
