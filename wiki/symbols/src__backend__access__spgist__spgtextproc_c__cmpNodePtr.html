cmpNodePtr

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgtextproc.c:323

目的

spgNodePtr 構造体の配列を、保持する文字コード（c）に基づいて昇順にソートするための比較関数を提供します。

シグネチャまたは形状

static int cmpNodePtr(const void *a, const void *b)

入力と出力

  • **入力**: 比較対象となる2つの spgNodePtr 構造体へのポインタ（void * 型として受け取る）。
  • **出力**: 比較結果を返す int 型。a->c が b->c より小さければ負、等しければ0、大きければ正の値を返す。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な spgNodePtr 構造体を指している必要があります。
  • spgNodePtr 構造体には、比較対象となる int16 型のメンバ c が存在することが前提です。

アルゴリズムまたは振る舞い

1. 引数 a および b を spgNodePtr 型のポインタにキャストします。 2. pg_cmp_s16 関数を呼び出し、それぞれの構造体が持つ c メンバの値を比較します。 3. 比較結果をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgtextproc.c 内の qsort 呼び出し（393行目）にて、ノード配列のソートに使用されています。
  • **呼び出し先**: pg_cmp_s16（common/int.h で定義される比較ユーティリティ）。

関連データ構造

  • spgNodePtr: 本関数が比較対象とする構造体。

失敗モードと境界条件

  • a または b が NULL の場合、あるいは指し示す先が spgNodePtr ではない場合、未定義の動作を引き起こします。
  • c メンバの値は int16 の範囲内に収まる必要があります。

保守担当者向けメモ

  • 本関数は qsort のコールバックとして設計されています。ソート順序を変更する必要がある場合は、この関数を修正してください。
  • pg_cmp_s16 を利用しているため、比較ロジックは PostgreSQL の標準的な整数比較ルールに従います。
