ObjectAddress

定義

  • 種別: struct
  • 定義箇所: src/include/tcop/deparse_utility.h:54
  • 補足: 本定義は catalog/objectaddress.h:24 で定義されている構造体を参照しています。

目的

データベース内のオブジェクトを一意に特定するための識別子を保持します。主にユーティリティコマンドの実行過程において、対象となるオブジェクトを追跡・記録するために使用されます。

シグネチャまたは形状

ソースコード上の定義は catalog/objectaddress.h に依存しており、本ヘッダー内では CollectedCommand 構造体の一部として、以下のように simple 構造体内で利用されています。

struct {
    ObjectAddress address;
    ObjectAddress secondaryObject;
} simple;

入力と出力

  • 入力: データベースオブジェクトの識別情報（クラスID、オブジェクトID、サブID等）。
  • 出力: 特定されたオブジェクトのメタデータ。

前提条件と不変条件

  • catalog/objectaddress.h で定義された構造体定義に準拠する必要があります。
  • ユーティリティコマンドの追跡において、対象オブジェクトが有効な OID を持っていることが前提となります。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持用であり、ロジックは持ちません。コマンド実行時に ObjectAddress が生成され、CollectedCommand 構造体に格納されることで、イベントトリガーや監査ログ等の後続処理で再利用可能な形式を提供します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/sepgsql/ 配下の各モジュール（database.c, dml.c, label.c, proc.c）にて、オブジェクトのラベル付けや権限チェックの文脈で参照されています。
  • 呼び出し先: 本シンボル自体はデータ構造であるため、直接的な呼び出し先はありません。

関連データ構造

  • CollectedCommand: 本構造体を含む、コマンド実行状態を保持するメインの構造体。
  • ObjectType: オブジェクトの種類を識別するための列挙型。
  • Oid: オブジェクトを一意に識別するための型。

失敗モードと境界条件

  • 該当するオブジェクトが存在しない場合や、無効な OID が指定された場合の挙動は、本構造体ではなく、これを利用する側の関数（catalog/objectaddress.h 関連の関数群）に依存します。

保守担当者向けメモ

  • 本構造体は catalog/objectaddress.h の定義に強く依存しています。変更を加える際は、カタログ定義との整合性に注意してください。
  • CollectedCommand 内で共用体の一部として使用されているため、アクセス時には type フィールドによる型チェックが必須です。
