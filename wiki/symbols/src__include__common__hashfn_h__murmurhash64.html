murmurhash64

定義

  • 種別: function
  • 定義箇所: src/include/common/hashfn.h:105

目的

64ビット整数値に対して、ハッシュテーブルのインデックス計算等に適した高速な非暗号学的ハッシュ値を生成する。

シグネチャまたは形状

static inline uint64 murmurhash64(uint64 data)

入力と出力

  • **入力**: uint64 data（ハッシュ化対象の64ビット整数）
  • **出力**: uint64（計算されたハッシュ値）

前提条件と不変条件

  • 本関数はインライン関数として定義されており、呼び出し側のオーバーヘッドを最小化する設計となっている。
  • 出力される64ビット値は、ハッシュテーブルの性能を維持するため、ビット全体が均等にランダムに分散されることが期待される。

アルゴリズムまたは振る舞い

入力値 data に対して以下のステップでビット混合を行う： 1. h を data で初期化。 2. h を右に33ビットシフトし、XORをとる。 3. h に定数 0xff51afd7ed558ccd を乗算する。 4. 再度、右に33ビットシフトし、XORをとる。 5. h に定数 0xc4ceb9fe1a85ec53 を乗算する。 6. 最後に、右に33ビットシフトし、XORをとって結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/resowner/resowner.c:235 にて、hash_combine64 の引数として呼び出されている。
  • **呼び出し先**: 外部関数への依存はなく、算術演算とビット演算のみで構成される。

関連データ構造

  • 特になし（プリミティブ型 uint64 のみを扱う）。

失敗モードと境界条件

  • 入力値がどのような値であっても計算は常に成功する。
  • 算術演算およびビットシフト演算のみで構成されるため、境界条件によるエラーは発生しない。

保守担当者向けメモ

  • 本関数は src/include/common/hashfn.h 内でインライン定義されているため、変更時はヘッダーの再コンパイルが必要となる。
  • 非暗号学的ハッシュであるため、セキュリティ上の要件（ハッシュ衝突攻撃耐性）が求められる箇所では、シード値を受け取る _extended 系関数や他のアルゴリズムの検討が必要である。
