ExecNamedTuplestoreScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeNamedtuplestorescan.c:66

目的

名前付き一時タプルストア（ENR）からタプルを順次スキャンして取得する実行ノードのメイン処理を実行します。

シグネチャまたは形状

static TupleTableSlot *ExecNamedTuplestoreScan(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（NamedTuplestoreScanState 構造体としてキャストして使用）
  • **出力**: TupleTableSlot *（取得したタプル、またはスキャン終了時は空のスロット）

前提条件と不変条件

  • pstate は NamedTuplestoreScanState 型のノードである必要があります。
  • 本関数は前方スキャンのみをサポートしており、EXEC_FLAG_BACKWARD は禁止されています。

アルゴリズムまたは振る舞い

1. pstate を NamedTuplestoreScanState にキャストします。 2. ExecScan 関数を呼び出し、以下のコールバックを使用してタプルストアからタプルを取得します。

  • NamedTuplestoreScanNext: 次のタプルをフェッチする実処理。
  • NamedTuplestoreScanRecheck: タプルの再チェック処理。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン内のノード実行処理（scanstate->ss.ps.ExecProcNode に本関数が割り当てられる）。
  • **呼び出し先**:
  • ExecScan（共通スキャンインフラ）
  • NamedTuplestoreScanNext（コールバック）
  • NamedTuplestoreScanRecheck（コールバック）

関連データ構造

  • NamedTuplestoreScanState: 本ノードの実行状態を保持する構造体。
  • PlanState: 実行計画のノード状態。
  • TupleTableSlot: タプルを保持するスロット。

失敗モードと境界条件

  • タプルストアが空の場合、ExecScan を通じて空のスロットが返されます。
  • 読み取りポインタが初期化されていない場合や、無効な状態での呼び出しは ExecScan 内部の依存関係によりエラーとなる可能性があります。

保守担当者向けメモ

  • 本関数は ExecScan を利用するラッパーです。詳細なスキャンロジックは NamedTuplestoreScanNext に委譲されています。
  • 読み取りポインタの管理は tuplestore 側の機能に依存しており、本関数自体はポインタのライフサイクル管理を行いません。
