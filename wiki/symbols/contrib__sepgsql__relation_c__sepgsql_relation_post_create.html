sepgsql_relation_post_create

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/relation.c:239

目的

新規作成されたリレーション（テーブル、ビュー、シーケンス等）に対し、SELinuxセキュリティラベルの計算、権限検証、およびラベルの永続化を行う。

シグネチャまたは形状

void sepgsql_relation_post_create(Oid relOid)

入力と出力

  • **入力**: relOid (作成されたリレーションのOID)
  • **出力**: なし（副作用として pg_seclabel カタログへのラベル書き込みを行う）

前提条件と不変条件

  • SnapshotSelf を使用してカタログをスキャンするため、トランザクション内で未コミットのレコードも参照可能である。
  • インデックスは親テーブルの属性として扱われるため、本関数はインデックスに対しては sepgsql_index_modify を呼び出し、直接のラベル割り当ては行わない。
  • TOASTテーブルに関連付けられたインデックスは処理対象外とする。

アルゴリズムまたは振る舞い

1. pg_class から対象リレーションの情報を取得する。 2. 親スキーマに対する db_schema:{add_name} 権限を検証する。 3. リレーションの種類（テーブル、ビュー、シーケンス）に応じたセキュリティクラスを決定する。 4. sepgsql_compute_create を用いてデフォルトのセキュリティラベルを計算する。 5. 対象オブジェクトに対する db_xxx:{create} 権限を検証する。 6. SetSecurityLabel を呼び出し、計算したラベルを pg_seclabel に保存する。 7. リレーションがテーブルまたはパーティションテーブルの場合、各カラムに対しても同様にデフォルトラベルの計算、権限検証、およびラベル割り当てを行う。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/hooks.c:135
  • **呼び出し先**: table_open, systable_beginscan, sepgsql_avc_check_perms, sepgsql_compute_create, sepgsql_avc_check_perms_label, SetSecurityLabel, sepgsql_index_modify 等

関連データ構造

  • Form_pg_class: リレーションのメタデータ構造体。
  • Form_pg_attribute: カラムのメタデータ構造体。
  • ObjectAddress: セキュリティラベルを適用する対象を識別する構造体。

失敗モードと境界条件

  • pg_class に該当するOIDが存在しない場合、elog(ERROR) により処理を中断する。
  • インデックス操作時は sepgsql_index_modify へ転送され、本関数内でのラベル割り当て処理はスキップされる。
  • 権限チェックで拒否された場合、sepgsql_avc_check_perms 等がエラーを発生させる。

保守担当者向けメモ

  • 本関数はDDLフックとして動作するため、カタログ操作には SnapshotSelf が必須である。
  • カラムのラベル割り当ては、テーブル作成時のみ再帰的に実行される。
  • audit_name の生成には quote_identifier を使用し、ログ出力時の識別子を適切にエスケープしている。
