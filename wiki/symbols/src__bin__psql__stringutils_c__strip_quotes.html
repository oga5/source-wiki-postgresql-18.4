strip_quotes

定義

  • 種別: function
  • 定義箇所: src/bin/psql/stringutils.c:239

目的

文字列内の引用符（クォート）を除去し、エスケープされた文字を正規化して、インプレースで文字列を書き換えます。

シグネチャまたは形状

void strip_quotes(char *source, char quote, char escape, int encoding)

入力と出力

  • source: 処理対象の文字列（入力かつ出力）。インプレースで書き換えられます。
  • quote: 除去対象の引用符文字。
  • escape: エスケープ文字。
  • encoding: マルチバイト文字を考慮するためのエンコーディング識別子。
  • 戻り値: なし（void）。

前提条件と不変条件

  • source は NULL であってはなりません。
  • quote はヌル文字（'\0'）であってはなりません。
  • source 文字列は、PQmblenBounded を用いてマルチバイト文字として正しく走査可能である必要があります。

アルゴリズムまたは振る舞い

1. source の先頭が quote と一致する場合、その文字をスキップします。 2. 文字列を走査し、以下のルールで処理を行います：

  • 文字列末尾の quote は除去します。
  • 連続する quote（quote + quote）は、1つの quote として扱います。
  • escape 文字に続く文字は、エスケープされたものとしてそのまま保持します。

    • 3. PQmblenBounded を使用してマルチバイト文字の境界を維持しながら、処理済みの文字を source の先頭方向へ詰め（インプレース書き換え）、最後にヌル終端を付与します。

関連データ構造

失敗モードと境界条件

  • source が空文字列の場合、何も行われません。
  • quote で囲まれていない文字列に対しては、先頭の quote チェックがスキップされ、そのままの文字列が維持されます。
  • 不完全なエスケープシーケンス（文字列末尾に escape 文字のみがある場合）は、そのまま保持されます。

保守担当者向けメモ

  • 本関数はインプレースで文字列を操作するため、呼び出し元で source が指すメモリ領域が書き換え可能であることを確認してください。
  • quote と escape が同じ文字である場合の挙動については、呼び出し側のロジックに依存します。
