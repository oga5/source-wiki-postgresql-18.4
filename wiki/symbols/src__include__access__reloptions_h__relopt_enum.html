relopt_enum

定義

  • 種別: struct
  • 定義箇所: src/include/access/reloptions.h:123

目的

リレーションオプション（reloptions）において、列挙型（enum）として定義されるパラメータの構造を保持します。

シグネチャまたは形状

typedef struct relopt_enum
{
    relopt_gen    gen;
    relopt_enum_elt_def *members;
    int           default_val;
    const char   *detailmsg;
} relopt_enum;

入力と出力

  • **入力**: relopt_enum_elt_def 型の配列（members）により、許容される列挙値のリストとデフォルト値（default_val）を定義します。
  • **出力**: reloptions.c 内の解析処理において、ユーザー指定値と照合され、内部的な整数値として変換・保持されます。

前提条件と不変条件

  • gen フィールドは relopt_gen 型であり、すべてのリレーションオプション構造体の先頭に配置される共通ヘッダーです。
  • members は NULL 終端された配列である必要があります。

アルゴリズムまたは振る舞い

  • reloptions.c 内の build_reloptions 等の関数により、ユーザーが指定した文字列オプションを、この構造体で定義された列挙値にマッピングします。
  • オプションの初期化時には init_enum_reloption が使用され、メモリ確保と構造体の設定が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/reloptions.c 内の enumRelOpts 定義や、init_enum_reloption を呼び出す各アクセスメソッドの初期化処理。
  • **呼び出し先**: 構造体内部の gen フィールドを通じて relopt_gen の機能を利用します。
  • **注記**: 具体的な関数呼び出しの連鎖については、提供されたソース範囲内では詳細なコールグラフは特定できません。

関連データ構造

  • relopt_gen: 共通ヘッダー。
  • relopt_enum_elt_def: 列挙値の定義に使用される構造体。

失敗モードと境界条件

  • members 配列が適切に NULL 終端されていない場合、解析処理中にメモリ境界外アクセスが発生する可能性があります。
  • default_val が members 配列の範囲外を指している場合、予期しない動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は reloptions.c 内で RELOPT_TYPE_ENUM として扱われます。新しい列挙型オプションを追加する際は、relopt_enum_elt_def を正しく定義し、init_enum_reloption を介して登録してください。
