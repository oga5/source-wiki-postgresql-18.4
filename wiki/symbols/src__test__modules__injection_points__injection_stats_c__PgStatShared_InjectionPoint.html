PgStatShared_InjectionPoint

定義

  • 種別: struct
  • 定義箇所: src/test/modules/injection_points/injection_stats.c:31

目的

PostgreSQLの統計サブシステム（pgstat）において、共有メモリ上でインジェクションポイントの統計情報を保持するための共有データ構造です。

シグネチャまたは形状

typedef struct PgStatShared_InjectionPoint
{
    PgStatShared_Common header;
    PgStat_StatInjEntry stats;
} PgStatShared_InjectionPoint;

入力と出力

  • 本構造体はデータ保持用であり、直接的な関数入力や出力は持ちません。
  • pgstat サブシステムが共有メモリ上の統計エントリを管理する際に、この構造体を通じて stats フィールド（PgStat_StatInjEntry）にアクセスします。

前提条件と不変条件

  • PgStatShared_Common 型の header を先頭に含み、pgstat の共有メモリ管理規約に従う必要があります。
  • injection_stats 定義において、shared_data_off は stats フィールドのオフセットを指し、shared_data_len は stats フィールドのサイズと一致している必要があります。

アルゴリズムまたは振る舞い

  • 共有メモリ上の統計エントリとして、pgstat のフラッシュ処理や取得処理の際にキャストして使用されます。
  • injection_stats_flush_cb 等のコールバック関数内で、共有メモリ上の統計データへのポインタとしてキャストされ、ローカルの更新分を反映するために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/injection_points/injection_stats.c 内の統計管理関数（injection_stats_flush_cb や統計取得処理）が、entry_ref->shared_stats を本構造体にキャストして参照します。
  • 呼び出し先: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • PgStatShared_Common: 共有統計エントリの共通ヘッダー。
  • PgStat_StatInjEntry: インジェクションポイントの具体的な統計データ（numcalls 等）を保持する構造体。

失敗モードと境界条件

  • 共有メモリ上の entry_ref->shared_stats が正しく初期化されていない状態で本構造体にキャストしてアクセスした場合、メモリ破壊や不正な読み取りが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は pgstat のカスタム統計種別実装の一部です。構造を変更する場合は、injection_stats 定義内の shared_size、shared_data_off、shared_data_len の設定を必ず更新してください。
