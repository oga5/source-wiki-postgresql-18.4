get_tablespace_paths

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/tablespace.c:39

目的

旧クラスタからユーザー定義のテーブルスペースパスを収集し、それらが物理的に存在し、かつディレクトリであることを検証して os_info 構造体に格納する。

シグネチャまたは形状

static void get_tablespace_paths(void)

入力と出力

  • **入力**: なし（グローバルな old_cluster および os_info を参照）。
  • **出力**: なし（副作用として os_info.old_tablespaces および os_info.num_old_tablespaces を更新）。

前提条件と不変条件

  • old_cluster が接続可能な状態であること。
  • pg_tablespace_location 関数が利用可能な PostgreSQL バージョンであること。

アルゴリズムまたは振る舞い

1. connectToServer を使用して旧クラスタに接続する。 2. pg_catalog.pg_tablespace をクエリし、pg_default および pg_global を除外した全テーブルスペースのパスを取得する。 3. 取得したパスの数を os_info.num_old_tablespaces に設定し、パス文字列を格納するメモリを確保する。 4. 各パスに対して stat() を実行し、以下の検証を行う：

  • パスが存在しない場合は PG_FATAL で終了。
  • パスがディレクトリでない場合は PG_FATAL で終了。

    • 5. クエリ結果と接続を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: init_tablespaces（src/bin/pg_upgrade/tablespace.c 内で確認）。
  • **呼び出し先**: connectToServer, executeQueryOrDie, pg_malloc, pg_strdup, stat, report_status, PQntuples, PQfnumber, PQgetvalue, PQclear, PQfinish。

関連データ構造

  • os_info: テーブルスペースのパス情報および個数を保持するグローバル構造体。
  • PGconn, PGresult: libpq の接続およびクエリ結果構造体。

失敗モードと境界条件

  • **ディレクトリ不在**: stat() が ENOENT を返した場合、report_status(PG_FATAL, ...) により処理を中断する。
  • **パスがディレクトリでない**: S_ISDIR チェックに失敗した場合、PG_FATAL で終了する。
  • **接続失敗**: connectToServer や executeQueryOrDie 内部でエラーが発生した場合、呼び出し元と同様にプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数はテーブルスペースの物理パスの妥当性を検証する重要な事前チェック機能である。
  • os_info への書き込みを行うため、呼び出し順序（init_tablespaces 内での位置）を変更する際は注意が必要である。
  • 取得したパスは pg_strdup で複製されるため、呼び出し側でメモリ管理を意識する必要はない。
