astreamer_verify_content_new

定義

  • 種別: function
  • 定義箇所: src/bin/pg_verifybackup/astreamer_verify.c:66

目的

バックアップ検証用ストリーマー（astreamer_verify）のインスタンスを生成および初期化し、検証パイプラインに組み込むためのコンストラクタ。

シグネチャまたは形状

astreamer *astreamer_verify_content_new(astreamer *next, verifier_context *context, char *archive_name, Oid tblspc_oid);

入力と出力

  • **入力**:
  • next: パイプライン内の後続ストリーマー。
  • context: 検証全体の状態を保持する verifier_context へのポインタ。
  • archive_name: 対象アーカイブ名。
  • tblspc_oid: テーブルスペースの OID。
  • **出力**:
  • 初期化された astreamer 構造体へのポインタを返す。

前提条件と不変条件

  • palloc0 により astreamer_verify 構造体がゼロクリアされた状態で割り当てられる。
  • astreamer_verify_ops が bbs_ops に設定され、ストリーマーとしてのインターフェースが確立される。
  • context->skip_checksums が偽の場合、チェックサム計算用のコンテキストが動的に割り当てられる。

アルゴリズムまたは振る舞い

1. astreamer_verify 構造体メモリを確保する。 2. astreamer_verify_ops を bbs_ops に代入し、ストリーマーの操作セットを定義する。 3. 引数で受け取った next（後続ストリーマー）、context、archive_name、tblspc_oid を構造体に格納する。 4. context 内のフラグを確認し、チェックサム検証が有効な場合は pg_checksum_context を割り当てる。 5. 初期化済みの base メンバ（astreamer 型）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_verifybackup/pg_verifybackup.c:1300
  • **呼び出し先**: palloc0（メモリ確保）、pg_malloc（チェックサムコンテキスト確保）

関連データ構造

  • astreamer_verify: 本関数で初期化されるメインの検証用構造体。
  • astreamer: ストリーマーの基底構造体。
  • verifier_context: 検証設定および状態を保持する構造体。
  • pg_checksum_context: チェックサム計算用コンテキスト。

失敗モードと境界条件

  • メモリ不足時には palloc0 または pg_malloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は astreamer インターフェースの一部として実装されており、astreamer_ops を介して呼び出されることを前提としている。
  • context->skip_checksums の状態に応じてチェックサムコンテキストの有無が決定されるため、後続の処理ではこのポインタが NULL である可能性を考慮する必要がある。
