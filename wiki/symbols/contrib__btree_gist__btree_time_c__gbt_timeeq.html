gbt_timeeq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_time.c:63

目的

PostgreSQLのGiSTインデックスにおいて、2つの TimeADT 値が等しいかどうかを判定します。

シグネチャまたは形状

static bool gbt_timeeq(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の TimeADT 型データへのポインタ。
  • flinfo: 関数マネージャ情報（本関数内では直接使用されません）。
  • **出力**:
  • 2つの値が等しい場合は true、そうでない場合は false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な TimeADT 値を指している必要があります。

アルゴリズムまたは振る舞い

1. 入力された void * 型のポインタを TimeADT * 型にキャストします。 2. TimeADTGetDatumFast マクロを使用して、各値を Datum 形式に変換します。 3. PostgreSQLのコア関数 time_eq を DirectFunctionCall2 経由で呼び出し、比較結果を取得します。 4. 取得した結果を DatumGetBool で bool 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_time.c 内の静的テーブル定義（gbtree_ninfo 構造体の初期化時）で参照されています。
  • **呼び出し先**: time_eq（PostgreSQLコアの時刻比較関数）。

関連データ構造

  • TimeADT: PostgreSQL内部で時刻を表現するデータ型。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、未定義の動作を引き起こす可能性があります。
  • time_eq 関数がエラーを発生させた場合、本関数もその例外を伝播します。

保守担当者向けメモ

  • 本関数は btree_utils_num.c が提供する汎用的な比較ロジックの一部として利用されます。
  • timetz 型の比較を行う場合は、別途正規化処理が必要となるため、本関数は純粋な time 型の比較に特化しています。
