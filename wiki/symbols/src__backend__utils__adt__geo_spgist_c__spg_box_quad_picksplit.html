spg_box_quad_picksplit

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_spgist.c:440

目的

SP-GiSTインデックス構築時、入力された矩形（BOX）の集合を4次元空間上の四分木（Quad Tree）の16象限に分割するためのメディアン計算およびノード割り当てを行う。

シグネチャまたは形状

Datum spg_box_quad_picksplit(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgPickSplitIn（入力タプル群）および spgPickSplitOut（出力構造体）を受け取る。
  • **出力**: void。out 構造体の prefixDatum（重心）、nNodes（16）、mapTuplesToNodes（各タプルの象限割り当て）、leafTupleDatums を設定する。

前提条件と不変条件

  • 入力データは BOX 型として解釈可能であること。
  • 4次元座標（low.x, high.x, low.y, high.y）の各軸においてメディアンを算出し、それらを組み合わせた centroid を基準点とする。
  • 象限の割り当てには getQuadrant 関数を使用する。

アルゴリズムまたは振る舞い

1. 全入力タプルの4次元座標を抽出し、各次元ごとにソートしてメディアンを特定する。 2. 特定したメディアン値を組み合わせて centroid（BOX）を生成し、これをインデックスのプレフィックスとして設定する。 3. nNodes を 16 に設定し、各タプルに対して getQuadrant を呼び出して、どの象限（0-15）に属するかを判定する。 4. 判定結果を mapTuplesToNodes に格納し、タプルを対応するノードへ割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiSTインデックス構築プロセス（詳細はソースコード内に明示的な呼び出し元記述なし）。
  • **呼び出し先**: getQuadrant（象限判定）、qsort（メディアン計算用）、palloc（メモリ確保）、DatumGetBoxP / BoxPGetDatum（型変換）。

関連データ構造

  • BOX: 幾何データ型。
  • spgPickSplitIn / spgPickSplitOut: SP-GiSTの picksplit インターフェース用構造体。

失敗モードと境界条件

  • 入力タプル数が 0 の場合、median 計算や配列アクセスで境界条件に抵触する可能性がある（実装上は in->nTuples に依存）。
  • メモリ不足時には palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は4次元空間の分割を行うため、getQuadrant とのロジック整合性が重要である。
  • mapTuplesToNodes および leafTupleDatums は palloc で確保されており、SP-GiSTのメモリ管理規約に従う。
  • 呼び出し元に関する情報は、現在のソースコード抜粋からはインデックス構築プロセスという推測以上の詳細なコールスタックは特定できない。
