missing_match

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/heaptuple.c:111

目的

missing_cache_key 構造体同士の比較を行い、ハッシュテーブル内での一致判定を行うための比較関数です。

シグネチャまたは形状

static int missing_match(const void *key1, const void *key2, Size keysize)

入力と出力

  • **入力**:
  • key1, key2: 比較対象となる missing_cache_key 構造体へのポインタ。
  • keysize: キーのサイズ（本関数内では直接使用されません）。
  • **出力**:
  • 0: 両者が一致する場合。
  • 1 または -1: 両者が不一致の場合（長さの比較結果に基づく）。

前提条件と不変条件

  • key1 および key2 は missing_cache_key 型としてキャスト可能である必要があります。
  • entry1->value および entry2->value は Datum 型として保持されており、DatumGetPointer でアクセス可能です。

アルゴリズムまたは振る舞い

1. key1 と key2 を missing_cache_key 型にキャストします。 2. まず len フィールドを比較します。長さが異なる場合は、その大小関係に応じて 1 または -1 を返します。 3. 長さが等しい場合、DatumGetPointer を介して取得したデータの実体を memcmp で比較し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/heaptuple.c:134 において、ハッシュテーブルの制御構造体 hash_ctl.match に関数ポインタとして代入されています。
  • **呼び出し先**: memcmp（標準ライブラリ関数）。

関連データ構造

  • missing_cache_key: 比較対象となるキー構造体。

失敗モードと境界条件

  • key1 または key2 が NULL の場合、あるいは Datum が指すメモリ領域が不正な場合は未定義の動作となります。

保守担当者向けメモ

  • 本関数はハッシュテーブルの比較用コールバックとして設計されています。keysize 引数はインターフェース上存在しますが、実装では無視されています。
