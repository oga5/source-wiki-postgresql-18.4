usage

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/option.c:303

目的

pg_upgrade コマンドの利用方法、コマンドラインオプションの一覧、および実行前の必須手順を標準出力に表示する。

シグネチャまたは形状

static void usage(void);

入力と出力

  • 入力: なし（グローバル変数 old_cluster、new_cluster、os_info の値を参照してデフォルト値を表示する）。
  • 出力: 標準出力（stdout）へヘルプメッセージを出力する。

前提条件と不変条件

  • printf を使用してロケール対応のメッセージを出力する。
  • Windows 環境（WIN32）か否かで、環境変数の設定例の表示形式を切り替える。

アルゴリズムまたは振る舞い

1. pg_upgrade の概要を表示する。 2. 利用可能なコマンドラインオプションの一覧を、デフォルト値を含めて表示する。 3. 実行前に必要な準備作業（クラスタの作成、停止など）を列挙する。 4. 必須となる引数（データディレクトリ、バイナリディレクトリ）を説明する。 5. コマンドライン引数および環境変数を用いた実行例を表示する。 6. バグ報告先およびプロジェクトのホームページ情報を表示する。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース excerpt 内には、本関数を呼び出している箇所は記載されていない。
  • 呼び出し先: printf（標準ライブラリ）。

関連データ構造

  • old_cluster: 旧クラスタ情報を保持する構造体（ポート番号の参照に使用）。
  • new_cluster: 新クラスタ情報を保持する構造体（ポート番号の参照に使用）。
  • os_info: OS 関連情報を保持する構造体（ユーザー名の参照に使用）。

失敗モードと境界条件

  • 特になし。

保守担当者向けメモ

  • オプションの追加や変更を行った際は、本関数内のヘルプメッセージも必ず更新すること。
  • PACKAGE_BUGREPORT、PACKAGE_NAME、PACKAGE_URL マクロを使用して情報を出力しているため、これらはビルドシステム側で適切に定義されている必要がある。
