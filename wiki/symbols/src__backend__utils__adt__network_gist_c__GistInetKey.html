GistInetKey

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/network_gist.c:79

目的

PostgreSQLのネットワーク型（INET / CIDR）に対するGiSTインデックスにおいて、インデックスキーとして使用される構造体です。アドレスの包含関係や階層構造を効率的に検索するために、アドレス値だけでなくプレフィックス長やマスク長を保持します。

シグネチャまたは形状

typedef struct GistInetKey
{
	uint8		va_header;		/* varlena header */
	unsigned char family;		/* PGSQL_AF_INET, PGSQL_AF_INET6, or zero */
	unsigned char minbits;		/* 最小ネットマスク長 */
	unsigned char commonbits;	/* 共通プレフィックス長 */
	unsigned char ipaddr[16];	/* 共通アドレス部分（最大128ビット） */
} GistInetKey;

入力と出力

  • **入力**: inet_gist_compress 関数により、INET 型の値から本構造体へ変換されます。
  • **出力**: inet_gist_fetch 関数により、インデックスキーから元の INET 型を復元するために使用されます。

前提条件と不変条件

  • varlena 形式に従うため、メモリレイアウトは可変長データとして扱われます。
  • ipaddr フィールドは、アドレスファミリーに応じた有効なビット長を保持します。
  • 内部構造は INET 型とは異なるため、直接的なキャストは行わず、DatumGetInetKeyP マクロ等を経由してアクセスする必要があります。

アルゴリズムまたは振る舞い

  • **検索**: inet_gist_consistent 関数において、minbits（最小ネットマスク長）と commonbits（共通プレフィックス長）を比較することで、インデックスの枝刈りを行います。
  • **統合**: calc_inet_union_params を用いて、複数のキーから共通のプレフィックス長と最小ネットマスク長を算出し、親ノードの代表値を生成します。
  • **分割**: inet_gist_picksplit は、アドレスファミリーやビット単位の共通性に基づき、インデックスページの分割戦略を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: inet_gist_compress, inet_gist_consistent, inet_gist_union, inet_gist_penalty, inet_gist_picksplit 等のGiSTサポート関数から参照されます。
  • **呼び出し先**: 直接的な関数呼び出しは行いませんが、palloc0 等のメモリ管理関数によってインスタンス化されます。

関連データ構造

  • INET: ネットワーク型の内部表現。
  • varlena: PostgreSQLの可変長データ型ヘッダー。

失敗モードと境界条件

  • アドレスファミリー（family）が一致しない場合、インデックスの検索や統合において高いコストが割り当てられるか、不整合として扱われます。
  • ipaddr のサイズ（16バイト）を超えるアドレスは扱えません。

保守担当者向けメモ

  • 本構造体は varlena 形式ですが、SET_VARSIZE_SHORT を使用してサイズを管理する際は offsetof(GistInetKey, ipaddr) を基準とした計算が必要です。
  • 検索効率を最適化するため、commonbits と minbits の計算ロジックを変更する場合は、関連する consistent 関数および union 関数の整合性を再確認してください。
