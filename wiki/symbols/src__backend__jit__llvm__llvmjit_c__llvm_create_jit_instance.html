llvm_create_jit_instance

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit.c:1215

目的

LLVM の ORC JIT インスタンス（LLVMOrcLLJITRef）を構築し、PostgreSQL プロセス内のシンボル解決およびカスタムシンボル解決機能を備えた実行環境を初期化する。

シグネチャまたは形状

static LLVMOrcLLJITRef llvm_create_jit_instance(LLVMTargetMachineRef tm)

入力と出力

  • **入力**: tm (LLVM のターゲットマシン参照)
  • **出力**: 初期化済みの LLVMOrcLLJITRef への参照。失敗した場合は elog(ERROR) により処理が中断される。

前提条件と不変条件

  • 引数 tm は有効な LLVMTargetMachineRef であること。
  • LLVM の C API が初期化済みであること。

アルゴリズムまたは振る舞い

1. LLVMOrcCreateLLJITBuilder を用いてビルダーを生成し、ターゲットマシンを設定する。 2. llvm_create_object_layer をオブジェクトリンクレイヤーの作成関数として登録する。 3. LLVMOrcCreateLLJIT で JIT インスタンスを生成する。 4. llvm_log_jit_error をエラーレポーターとして設定する。 5. LLVMOrcCreateDynamicLibrarySearchGeneratorForProcess を使用し、PostgreSQL バイナリおよびロード済みライブラリ内のシンボルを解決するためのジェネレータを MainJITDylib に追加する。 6. LLVMOrcCreateCustomCAPIDefinitionGenerator を使用し、SQL 呼び出し可能な関数等の特殊なシンボルを解決するためのジェネレータ（llvm_resolve_symbols）を MainJITDylib に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit.c 内の llvm_session_initialize（最適化レベル 0 および 3 用のインスタンス生成時）。
  • **呼び出し先**:
  • LLVMOrcCreateLLJITBuilder, LLVMOrcJITTargetMachineBuilderCreateFromTargetMachine, LLVMOrcLLJITBuilderSetJITTargetMachineBuilder, LLVMOrcLLJITBuilderSetObjectLinkingLayerCreator, LLVMOrcCreateLLJIT, LLVMOrcExecutionSessionSetErrorReporter, LLVMOrcCreateDynamicLibrarySearchGeneratorForProcess, LLVMOrcJITDylibAddGenerator, LLVMOrcCreateCustomCAPIDefinitionGenerator 等の LLVM C API。
  • llvm_create_object_layer (コールバックとして登録)
  • llvm_resolve_symbols (コールバックとして登録)

関連データ構造

  • LLVMOrcLLJITRef: 生成される JIT インスタンス。
  • LLVMTargetMachineRef: JIT のターゲットアーキテクチャ情報。

失敗モードと境界条件

  • LLVM インスタンスの生成失敗、またはシンボルジェネレータの作成失敗時には elog(ERROR) が発生し、PostgreSQL のエラー処理系へ制御が移る。

保守担当者向けメモ

  • LLVM 14 以前とそれ以降で LLVMOrcCreateCustomCAPIDefinitionGenerator の引数構成が異なるため、プリプロセッサディレクティブによる条件分岐が含まれている。
  • 本関数で登録されるジェネレータは、JIT コンパイルされたコードが PostgreSQL の既存関数を呼び出すために不可欠である。
