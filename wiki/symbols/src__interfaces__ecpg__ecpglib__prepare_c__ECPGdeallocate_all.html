ECPGdeallocate_all

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/prepare.c:349

目的

指定されたデータベース接続に関連付けられたすべてのプリペアドステートメントを解放（DEALLOCATE）します。

シグネチャまたは形状

bool ECPGdeallocate_all(int lineno, int compat, const char *connection_name)

入力と出力

  • **入力**:
  • lineno: 呼び出し元のソースコード行番号（エラー報告用）。
  • compat: 互換性モードを指定するフラグ。
  • connection_name: 対象となる接続名（NULLの場合はデフォルト接続）。
  • **出力**:
  • 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • ecpg_get_connection を介して有効な接続オブジェクトが取得可能である必要があります。
  • ecpg_init による初期化が成功している必要があります。

アルゴリズムまたは振る舞い

1. ecpg_get_connection を呼び出し、connection_name に対応する接続オブジェクトを取得します。 2. ecpg_init を呼び出し、接続の初期化と検証を行います。失敗した場合は false を返します。 3. ecpg_deallocate_all_conn を呼び出し、当該接続に関連付けられたすべてのプリペアドステートメントを破棄する処理を実行し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/output.c（プリプロセッサによるコード生成）
  • src/interfaces/ecpg/test/expected/ 配下の各テストコード
  • **呼び出し先**:
  • ecpg_get_connection
  • ecpg_init
  • ecpg_deallocate_all_conn

関連データ構造

  • connection: データベース接続状態を保持する構造体。

失敗モードと境界条件

  • 指定された connection_name が存在しない場合、ecpg_init が失敗し false を返します。
  • 接続が初期化できない場合、処理は中断され false を返します。

保守担当者向けメモ

  • 本関数はラッパーであり、実質的な解放処理は ecpg_deallocate_all_conn に委譲されています。
  • 接続終了時のクリーンアップ処理の一部として利用されるため、メモリ管理の整合性に注意してください。
