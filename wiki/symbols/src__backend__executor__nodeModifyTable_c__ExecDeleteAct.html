ExecDeleteAct

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeModifyTable.c:1491

目的

テーブルの削除操作における実処理を実行し、その結果を呼び出し元へ報告するサブルーチンです。

シグネチャまたは形状

static TM_Result ExecDeleteAct(ModifyTableContext *context, ResultRelInfo *resultRelInfo, ItemPointer tupleid, bool changingPart);

入力と出力

  • **入力**:
  • context: 操作の状態（ModifyTableContext）を保持するポインタ。
  • resultRelInfo: ターゲットリレーション情報。
  • tupleid: 削除対象タプルの物理位置（ItemPointer）。
  • changingPart: パーティション変更を伴う削除かどうかを示すフラグ。
  • **出力**:
  • TM_Result: 削除操作の結果（成功、失敗、競合など）。

前提条件と不変条件

  • context->estate が有効な実行状態（EState）を指していること。
  • resultRelInfo がターゲットリレーションの記述子（ri_RelationDesc）を保持していること。

アルゴリズムまたは振る舞い

1. context->estate から実行状態（スナップショットやコマンドID）を取得します。 2. table_tuple_delete を呼び出し、ストレージ層に対して指定された tupleid の削除を依頼します。

  • 削除時には estate->es_output_cid（コマンドID）および es_snapshot を使用します。
  • es_crosscheck_snapshot による整合性チェックを行います。
  • 削除の確定を待機する設定（wait for commit = true）で実行します。

    • 3. 処理結果を context->tmfd に格納し、呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecDelete（src/backend/executor/nodeModifyTable.c 内の複数箇所）。
  • **呼び出し先**: table_tuple_delete（テーブルアクセスメソッド層）。

関連データ構造

  • ModifyTableContext: 操作全体のコンテキスト。
  • ResultRelInfo: ターゲットテーブルのメタデータ。
  • TM_Result: ストレージ層からの操作結果ステータス。

失敗モードと境界条件

  • table_tuple_delete が返す TM_Result に依存します。同時実行制御による競合や、スナップショットの不整合が発生した場合に適切なステータスが返されます。

保守担当者向けメモ

  • 本関数は ExecDelete の内部ロジックを分離したものです。削除処理の共通化や、将来的な削除前後のフック追加を行う場合は本関数を修正してください。
  • context->tmfd を介して結果を伝播させる設計となっているため、コンテキスト構造の変更時には注意が必要です。
