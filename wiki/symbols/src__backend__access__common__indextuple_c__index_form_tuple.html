index_form_tuple

定義

  • src/backend/access/common/indextuple.c:43

目的

指定された属性値の配列から、新しい IndexTuple を構築します。

シグネチャまたは形状

IndexTuple index_form_tuple(TupleDesc tupleDescriptor, const Datum *values, const bool *isnull);

入力と出力

  • **入力**:
  • tupleDescriptor: タプルの構造を定義する TupleDesc。
  • values: 属性値の Datum 配列。
  • isnull: 各属性が NULL かどうかを示すブール値の配列。
  • **出力**:
  • 構築された IndexTuple へのポインタ。

前提条件と不変条件

  • CurrentMemoryContext をメモリ割り当てに使用します。
  • インデックス行は外部ストレージ（TOAST）を保持できないため、必要に応じて内部でデータのインライン化や圧縮が行われます。

アルゴリズムまたは振る舞い

1. CurrentMemoryContext を指定して index_form_tuple_context を呼び出します。 2. 内部的には、heap_compute_data_size で必要なサイズを計算し、MemoryContextAllocZero でメモリを確保した後、heap_fill_tuple を用いてデータを書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_nbtree.c
  • src/backend/access/common/indextuple.c (内部の切り詰め処理等)
  • src/backend/access/gin/ginentrypage.c
  • **呼び出し先**:
  • index_form_tuple_context (直接の委譲先)

関連データ構造

  • IndexTuple: インデックス行のメモリ表現。
  • TupleDesc: タプルの属性定義。
  • Datum: 属性値の汎用型。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAllocZero によりエラーが発生する可能性があります。
  • index_form_tuple は結果のサイズを MAXALIGN するため、実際のデータサイズよりわずかに大きくなる場合があります。

保守担当者向けメモ

  • 本関数は index_form_tuple_context のラッパーです。特定のメモリコンテキストでタプルを作成したい場合は、直接 index_form_tuple_context を使用してください。
  • TOASTデータの扱いは TOAST_INDEX_HACK に依存しており、インデックス作成時に強制的なデトーストや圧縮が適用されます。
