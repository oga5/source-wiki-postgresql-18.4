XLogArchiveNotify

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogarchive.c:443

目的

指定されたWALファイルに対してアーカイブが必要であることを示す .ready ファイルを作成し、アーカイブプロセス（pgarch）に通知を行う。

シグネチャまたは形状

void XLogArchiveNotify(const char *xlog)

入力と出力

  • **入力**: xlog (アーカイブ対象のWALファイル名)
  • **出力**: なし（成功時）。失敗時はログ出力を行う。

前提条件と不変条件

  • xlog 引数は有効なWALファイル名であること。
  • pg_wal/archive_status ディレクトリへの書き込み権限が必要。

アルゴリズムまたは振る舞い

1. StatusFilePath を使用し、対象WALファイル名に .ready サフィックスを付与したパスを生成する。 2. AllocateFile を用いて当該パスに空のファイルを書き込みモードで作成する。 3. ファイル作成に失敗した場合は LOG レベルでエラーを報告し、処理を中断する。 4. タイムライン履歴ファイル（IsTLHistoryFileName が真）の場合は、PgArchForceDirScan を呼び出し、アーカイブプロセスに即時スキャンを強制する。 5. IsUnderPostmaster が真の場合、PgArchWakeup を呼び出し、アーカイブプロセスを起動（ウェイクアップ）させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/timeline.c
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogarchive.c
  • src/backend/replication/walreceiver.c
  • **呼び出し先**:
  • StatusFilePath
  • AllocateFile / FreeFile
  • IsTLHistoryFileName
  • PgArchForceDirScan
  • PgArchWakeup

関連データ構造

  • .ready ファイル: アーカイブ待ち状態を示すフラグファイル。
  • MAXPGPATH: ファイルパスの最大長を規定する定数。

失敗モードと境界条件

  • **ファイル作成失敗**: AllocateFile が NULL を返す場合、アーカイブ状態の通知に失敗し、ログに記録される。
  • **ファイルクローズ失敗**: FreeFile が非ゼロを返す場合、書き込みエラーとしてログに記録される。
  • **競合**: 既に同名の .ready ファイルが存在する場合の挙動は、AllocateFile の実装に依存する。

保守担当者向けメモ

  • 本関数はアーカイブプロセスとの連携の起点となる。.ready ファイルの作成は、アーカイブプロセスが当該ファイルを認識するための不可欠なトリガーである。
  • タイムライン履歴ファイルに対する PgArchForceDirScan の呼び出しは、スタンバイ昇格時のタイムライン競合を避けるための優先度制御である。
