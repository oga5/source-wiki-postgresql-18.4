buf_init

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xid8funcs.c:221

目的

pg_snapshot 構造体を初期化し、指定されたトランザクションID範囲（xmin, xmax）を持つバイナリバッファ（StringInfo）を生成します。

シグネチャまたは形状

static StringInfo buf_init(FullTransactionId xmin, FullTransactionId xmax)

入力と出力

  • **入力**:
  • xmin: スナップショットの最小トランザクションID。
  • xmax: スナップショットの最大トランザクションID。
  • **出力**:
  • StringInfo: 初期化された pg_snapshot 構造体（nxip = 0）を格納したバッファ。

前提条件と不変条件

  • pg_snapshot 構造体の nxip（進行中のトランザクションID数）は常に 0 に初期化されます。
  • makeStringInfo() によってメモリが割り当てられるため、呼び出し元で適切な解放が必要です。

アルゴリズムまたは振る舞い

1. pg_snapshot 型のローカル変数 snap を宣言します。 2. snap の xmin および xmax に引数の値を代入し、nxip を 0 に設定します。 3. makeStringInfo() を呼び出し、空のバッファを作成します。 4. PG_SNAPSHOT_SIZE(0) を用いて、nxip が 0 の状態の pg_snapshot 構造体をバッファにバイナリコピーします。 5. 作成したバッファを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/xid8funcs.c:292 にて呼び出されています。
  • **呼び出し先**:
  • makeStringInfo()
  • appendBinaryStringInfo()
  • 注記: ツールが提示した src/backend/storage/buffer/buf_init.c 等の参照は、ファイル名が同一であることによるヒューリスティックな一致であり、本関数との直接的な呼び出し関係を示すものではありません。

関連データ構造

  • FullTransactionId: 64ビットのトランザクションID。
  • pg_snapshot: トランザクションスナップショットを保持する構造体。
  • StringInfo: PostgreSQLの動的文字列バッファ。

失敗モードと境界条件

  • メモリ不足時には makeStringInfo() 内部でエラーが発生し、処理が中断される可能性があります。

保守担当者向けメモ

  • 本関数は pg_snapshot の初期化専用です。nxip > 0 のスナップショットを作成する場合は、別途 buf_add_txid 等を用いてバッファを拡張する必要があります。
  • src/backend/storage/buffer/buf_init.c と名前が重複していますが、本関数は xid8funcs.c 内の静的関数であり、ストレージ管理の buf_init とは別物です。
