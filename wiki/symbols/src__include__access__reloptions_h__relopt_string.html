relopt_string

定義

  • 種別: struct
  • 定義箇所: src/include/access/reloptions.h:139

目的

リレーションやテーブルスペースのオプション設定において、文字列型のパラメータを定義・管理するための構造体です。

シグネチャまたは形状

typedef struct relopt_string
{
    relopt_gen    gen;
    int           default_len;
    bool          default_isnull;
    validate_string_relopt validate_cb;
    fill_string_relopt fill_cb;
    char         *default_val;
} relopt_string;

入力と出力

  • **入力**: オプション定義時に default_val（デフォルト値）、validate_cb（検証用コールバック）、fill_cb（値設定用コールバック）を指定します。
  • **出力**: build_reloptions 等の処理を通じて、解析された文字列オプション値が構造体として保持されます。

前提条件と不変条件

  • gen フィールドは、すべてのオプション型に共通するヘッダーとして機能します。
  • 文字列オプションを定義する際は、validate_string_relopt による検証ロジックの実装が推奨されます。

アルゴリズムまたは振る舞い

1. **初期化**: init_string_reloption 等を介して、指定された種類（relopt_kind）とデフォルト値で初期化されます。 2. **検証**: ユーザーが設定した値は validate_cb を通じて妥当性がチェックされます。 3. **変換**: build_reloptions 実行時、登録された定義に基づき、入力された Datum が適切なメモリレイアウトへマッピングされます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/reloptions.c 内の各関数（stringRelOpts 配列の定義、allocate_reloption、init_string_reloption 等）から参照されています。
  • **呼び出し先**: validate_cb および fill_cb を通じて、各モジュール固有の検証・設定関数が呼び出されます。

関連データ構造

  • relopt_gen: 本構造体の先頭メンバであり、共通ヘッダーとして機能します。
  • relopt_kind: オプションの適用対象（Heap, B-tree等）を識別します。

失敗モードと境界条件

  • 文字列の長さが default_len を超える場合や、validate_cb による検証に失敗した場合、オプション設定処理はエラーを返します。

保守担当者向けメモ

  • 文字列オプションを扱う際は、メモリレイアウトの整合性を保つため、relopt_parse_elt を用いて構造体内のオフセットを正確に管理してください。
  • 実行計画やデータ読み取りに影響を与えるオプションを追加する場合は、適切な LOCKMODE を選択する必要があります。
