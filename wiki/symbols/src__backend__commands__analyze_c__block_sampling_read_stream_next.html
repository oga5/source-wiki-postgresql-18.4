block_sampling_read_stream_next

定義

  • 種別: function
  • 定義箇所: src/backend/commands/analyze.c:1155

目的

ReadStream インターフェースを介して、ブロックサンプリングにおける次の対象ブロック番号を取得する。

シグネチャまたは形状

static BlockNumber block_sampling_read_stream_next(ReadStream *stream, void *callback_private_data, void *per_buffer_data)

入力と出力

  • **入力**:
  • stream: 読み込みストリームのポインタ。
  • callback_private_data: BlockSamplerData 構造体へのポインタ。
  • per_buffer_data: 使用されていない（void *）。
  • **出力**:
  • 次の BlockNumber。サンプリング対象が残っていない場合は InvalidBlockNumber を返す。

前提条件と不変条件

  • callback_private_data は BlockSamplerData 型としてキャスト可能であること。
  • BlockSampler_HasMore および BlockSampler_Next が提供するサンプリングロジックに依存する。

アルゴリズムまたは振る舞い

1. callback_private_data を BlockSamplerData 型の bs にキャストする。 2. BlockSampler_HasMore(bs) を評価する。 3. 真であれば BlockSampler_Next(bs) を呼び出し、その結果（次のブロック番号）を返す。 4. 偽であれば InvalidBlockNumber を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は確認されていないが、ReadStream のコールバックとして登録される想定である。
  • **呼び出し先**: BlockSampler_HasMore および BlockSampler_Next（utils/sampling.h 関連）。

関連データ構造

  • BlockSamplerData: サンプリングの状態を保持する構造体。
  • ReadStream: 読み込みストリームを制御する構造体。
  • BlockNumber: PostgreSQL のブロック識別子。

失敗モードと境界条件

  • サンプリング対象が枯渇した際、正しく InvalidBlockNumber を返すことで終了を通知する。
  • callback_private_data が NULL の場合の挙動は定義されていない。

保守担当者向けメモ

  • 本関数は ReadStream のコールバックとして設計されており、バッチ処理において安全に使用できることが示唆されている。
  • BlockSampler 関連の API が変更された場合、本関数のロジックも追従する必要がある。
