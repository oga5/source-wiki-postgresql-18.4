TU_UpdateIndexes

定義

  • 種別: enum
  • 定義箇所: src/include/access/tableam.h:109

目的

タプル更新操作において、どのインデックスの更新が必要かを分類・管理するための列挙型です。

シグネチャまたは形状

typedef enum TU_UpdateIndexes
{
    TU_None,        /* インデックス列の更新なし（TID変更もなし） */
    TU_All,         /* 非要約インデックス列の更新、またはTIDの変更あり */
    TU_Summarizing  /* 要約列のみの更新、TID変更なし */
} TU_UpdateIndexes;

入力と出力

  • 本シンボルは、タプル更新処理（heapam.c や tableam.c 等）において、更新後のインデックス処理の要否を決定するフラグとして使用されます。

前提条件と不変条件

  • TU_None: インデックス列およびTIDに変更がない場合に指定されます。
  • TU_All: インデックス列の変更、または物理的なTIDの変更が発生した場合に指定されます。
  • TU_Summarizing: インデックスの要約列のみが変更され、TIDに変更がない場合に指定されます。

アルゴリズムまたは振る舞い

  • タプル更新処理の過程で、更新内容に応じてこの列挙値が設定されます。
  • 呼び出し側は、この値に基づいてインデックスの再構築や更新処理をスキップするか、あるいは実行するかを判断します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/heapam_handler.c
  • src/backend/access/table/tableam.c
  • src/backend/catalog/indexing.c
  • src/backend/executor/execReplication.c
  • src/backend/executor/nodeModifyTable.c
  • src/include/access/heapam.h
  • **呼び出し先**:
  • 本シンボルは列挙型であり、関数ではないため呼び出し先はありません。

関連データ構造

  • HeapTuple: 更新対象となるタプル構造体。
  • TM_IndexDelete: インデックス削除状態を管理する関連構造体。

失敗モードと境界条件

  • 適切な値が設定されない場合、インデックスの不整合が発生する可能性があります。特にTIDが変更されたにもかかわらず TU_None が選択された場合、インデックスが古いTIDを指し続けるリスクがあります。

保守担当者向けメモ

  • 本シンボルは tableam インターフェースの一部であり、新しいストレージエンジンを実装する際は、タプル更新時にこの列挙型を適切に選択して返す必要があります。
