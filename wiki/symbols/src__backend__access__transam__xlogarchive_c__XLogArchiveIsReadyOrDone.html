XLogArchiveIsReadyOrDone

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogarchive.c:663

目的

指定されたWALセグメントが、アーカイブ処理の完了（.done）またはアーカイブ待ち（.ready）のいずれかの状態にあるかを判定します。

シグネチャまたは形状

bool XLogArchiveIsReadyOrDone(const char *xlog)

入力と出力

  • **入力**: xlog (WALセグメント名を示す文字列)
  • **出力**: true（アーカイブ済み、またはアーカイブ待ち状態である場合）、false（それ以外の場合）

前提条件と不変条件

  • pg_wal/archive_status ディレクトリ内に、対応する .ready または .done ファイルが存在するかをファイルシステムレベルで確認します。
  • 呼び出し元は、xlog 引数として有効なWALファイル名を渡す必要があります。

アルゴリズムまたは振る舞い

1. 指定されたWALセグメントの .done ファイルの存在を確認し、存在すれば true を返します。 2. 次に .ready ファイルの存在を確認し、存在すれば true を返します。 3. アーカイブプロセスとの競合（直前にアーカイブが完了した可能性）を考慮し、再度 .done ファイルの存在を確認します。 4. いずれも存在しない場合、false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c:5385
  • **呼び出し先**:
  • stat() (システムコール)
  • StatusFilePath() (マクロ/関数: アーカイブ状態ファイルのパス構築)

関連データ構造

  • struct stat: ファイルの存在確認に使用。
  • MAXPGPATH: パス文字列の最大長定義。

失敗モードと境界条件

  • ファイルシステムエラーや権限不足により stat() が失敗した場合、その状態は無視され、最終的に false を返す可能性があります。
  • 競合状態（Race condition）を考慮した再チェックを行っていますが、ファイルシステム上の操作と完全に同期しているわけではありません。

保守担当者向けメモ

  • 本関数は、WALセグメントがアーカイブプロセスによって処理中であるか、既に完了しているかを判断するための安全なチェック手段を提供します。
  • StatusFilePath を使用して状態ファイルパスを生成しているため、ファイル配置場所の変更時は当該マクロの定義を確認してください。
