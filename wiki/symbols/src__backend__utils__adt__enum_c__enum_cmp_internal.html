enum_cmp_internal

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/enum.c:251

目的

列挙型（ENUM）の値を比較するための共通エンジン。OIDの直接比較による高速パスと、システムキャッシュを利用した順序比較を切り替えて実行する。

シグネチャまたは形状

static int enum_cmp_internal(Oid arg1, Oid arg2, FunctionCallInfo fcinfo)

入力と出力

  • **入力**:
  • arg1, arg2: 比較対象の列挙値（OID）。
  • fcinfo: 関数呼び出し情報。flinfo->fn_extra をキャッシュ領域として使用する。
  • **出力**:
  • 0: arg1 == arg2 の場合。
  • -1: arg1 < arg2 の場合。
  • 1: arg1 > arg2 の場合。

前提条件と不変条件

  • fcinfo->flinfo が NULL でないこと（アサーションで検証）。
  • arg1 または arg2 が奇数の場合、比較には TypeCacheEntry が必要となる。
  • fcinfo->flinfo->fn_extra は、初回実行時に TypeCacheEntry を保持するために使用される。

アルゴリズムまたは振る舞い

1. **等価判定**: arg1 == arg2 であれば即座に 0 を返す。 2. **高速パス**: 両方のOIDが偶数であれば、OIDの大小関係をそのまま順序として返し、終了する。 3. **キャッシュ検索**: 偶数でない場合、fcinfo->flinfo->fn_extra を確認する。

  • キャッシュが未設定の場合、arg1 をキーに pg_enum システムカタログを検索し、対応する型OIDを取得。lookup_type_cache を通じて TypeCacheEntry を生成し、fn_extra に保存する。

    • 4. **比較実行**: compare_values_of_enum を呼び出し、キャッシュされた順序情報に基づいて比較結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • enum_lt, enum_le, enum_eq, enum_ne, enum_ge, enum_gt（比較演算子）
  • 列挙型の最小値・最大値取得関数
  • **呼び出し先**:
  • SearchSysCache1
  • lookup_type_cache
  • compare_values_of_enum

関連データ構造

  • Oid: 列挙値の内部表現。
  • FunctionCallInfo: 関数呼び出しのコンテキスト。
  • TypeCacheEntry: 列挙型の順序情報を保持するキャッシュ構造体。
  • Form_pg_enum: pg_enum システムカタログの行構造。

失敗モードと境界条件

  • arg1 に対応する列挙型がシステムカタログに見つからない場合、ERROR を発生させる。
  • fcinfo->flinfo が NULL の場合、アサーション違反となる。

保守担当者向けメモ

  • 本関数は列挙型の比較におけるパフォーマンスの要である。高速パス（偶数OID）の条件は、列挙値の順序が変更されていない場合に有効である。
  • compare_values_of_enum へのフォールバックは、列挙値の順序が変更された場合にのみ発生する。
  • fn_extra を介したキャッシュ機構は、同一セッション内での繰り返し呼び出しを最適化する。
