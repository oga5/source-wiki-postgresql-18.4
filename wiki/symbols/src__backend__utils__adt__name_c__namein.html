namein

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/name.c:47

目的

C 文字列（cstring）を PostgreSQL の name 型（固定長識別子）に変換します。

シグネチャまたは形状

Datum namein(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される cstring 型の Datum。
  • **出力**: name 型の Datum。

前提条件と不変条件

  • name 型は NAMEDATALEN バイトの固定長です。
  • 入力文字列が NAMEDATALEN を超える場合、マルチバイト文字の整合性を保ちつつ切り詰められます。
  • 戻り値のメモリ領域は palloc0 により確保され、ゼロクリアされるため、末尾のパディングが保証されます。

アルゴリズムまたは振る舞い

1. 入力文字列の長さを取得します。 2. 長さが NAMEDATALEN 以上の場合、pg_mbcliplen を使用して NAMEDATALEN - 1 バイト以内に収まるよう切り詰めます。 3. palloc0(NAMEDATALEN) を呼び出し、ゼロ初期化されたメモリ領域を確保します。 4. memcpy を使用して、切り詰められた文字列を確保した領域にコピーします。 5. PG_RETURN_NAME マクロを介して変換後の Datum を返します。

関連データ構造

  • Name: name 型の内部表現。
  • NAMEDATALEN: name 型の固定長サイズを定義する定数。

失敗モードと境界条件

  • 入力文字列が NAMEDATALEN を超える場合、切り詰め処理が発生します。
  • メモリ確保失敗時は palloc0 がエラーをスローします。

保守担当者向けメモ

  • name 型のサイズ制限は歴史的な理由によるものです。変更が必要な場合は NAMEDATALEN 定数を確認してください。
  • palloc0 を使用しているため、未初期化メモリによる比較の不整合は発生しません。
