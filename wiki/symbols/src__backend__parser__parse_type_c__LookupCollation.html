LookupCollation

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_type.c:514

目的

SQL文で指定された照合順序（Collation）の名前リストを解決し、対応するシステムカタログ上の OID を取得します。

シグネチャまたは形状

Oid LookupCollation(ParseState *pstate, List *collnames, int location)

入力と出力

  • **入力**:
  • pstate: 現在の構文解析状態（エラー位置報告用）。NULL の場合、エラー位置の追跡は行われません。
  • collnames: 照合順序名を指定する List。
  • location: エラー発生時の SQL 文中の位置情報。
  • **出力**:
  • 解決された照合順序の Oid を返します。

前提条件と不変条件

  • pstate が提供されている場合、エラー発生時に parser_errposition を用いて正確な位置を報告するためのコールバックが設定されます。
  • 内部で get_collation_oid を呼び出し、照合順序の検索を行います。

アルゴリズムまたは振る舞い

1. pstate が有効な場合、setup_parser_errposition_callback を呼び出し、エラー発生時の位置情報を設定します。 2. get_collation_oid(collnames, false) を呼び出し、システムカタログから照合順序の OID を検索します。 3. pstate が有効な場合、cancel_parser_errposition_callback を呼び出し、設定したコールバックを解除します。 4. 取得した colloid を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_clause.c
  • src/backend/parser/parse_expr.c
  • src/backend/parser/parse_type.c
  • src/backend/parser/parse_utilcmd.c
  • **呼び出し先**:
  • get_collation_oid
  • setup_parser_errposition_callback (条件付き)
  • cancel_parser_errposition_callback (条件付き)

関連データ構造

  • ParseState: 構文解析のコンテキストを保持する構造体。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • get_collation_oid が指定された名前の照合順序を見つけられない場合、内部でエラーが発生します。
  • pstate が NULL の場合、エラー位置の報告は行われません。

保守担当者向けメモ

  • 本関数は src/include/parser/parse_type.h で公開されています。
  • 照合順序の検索ロジックを変更する場合は、get_collation_oid の仕様と整合性を取る必要があります。
