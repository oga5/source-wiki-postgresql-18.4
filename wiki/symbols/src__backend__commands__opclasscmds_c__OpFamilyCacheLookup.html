OpFamilyCacheLookup

定義

  • 種別: function
  • 定義箇所: src/backend/commands/opclasscmds.c:80

目的

指定されたアクセスメソッド（AM）と演算子ファミリー名に基づき、システムカタログから演算子ファミリーのタプルを検索・取得します。

シグネチャまたは形状

HeapTuple OpFamilyCacheLookup(Oid amID, List *opfamilyname, bool missing_ok)

入力と出力

  • **入力**:
  • amID: 検索対象のアクセスメソッドのOID。
  • opfamilyname: 演算子ファミリー名を指定するリスト（修飾名）。
  • missing_ok: true の場合、見つからない場合に NULL を返し、false の場合はエラーを発生させる。
  • **出力**:
  • 成功時: 該当する演算子ファミリーの HeapTuple。
  • 失敗時: missing_ok が true なら NULL。

前提条件と不変条件

  • opfamilyname は DeconstructQualifiedName で分解可能な形式である必要があります。
  • missing_ok が false の場合、検索失敗時には必ずエラー（elog または ereport）が発生し、関数は戻りません。

アルゴリズムまたは振る舞い

1. DeconstructQualifiedName を使用して、opfamilyname をスキーマ名とファミリー名に分解します。 2. スキーマ名が指定されている場合、LookupExplicitNamespace で名前空間を特定し、SearchSysCache3（OPFAMILYAMNAMENSP）を用いて検索します。 3. スキーマ名が指定されていない場合、OpfamilynameGetOpfid で検索パスからOIDを特定し、SearchSysCache1（OPFAMILYOID）を用いて検索します。 4. タプルが見つからず missing_ok が false の場合、アクセスメソッド自体の存在を確認した上で、ERRCODE_UNDEFINED_OBJECT エラーを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の参照情報に基づくと、src/backend/commands/opclasscmds.c 内の他の関数から呼び出されています。
  • **呼び出し先**:
  • DeconstructQualifiedName
  • LookupExplicitNamespace
  • SearchSysCache3 / SearchSysCache1
  • OpfamilynameGetOpfid
  • NameListToString
  • elog / ereport

関連データ構造

  • HeapTuple: 検索結果として返されるシステムカタログのタプル。
  • Oid: アクセスメソッドおよび演算子ファミリーの識別子。
  • Form_pg_am: エラー報告時にアクセスメソッド名を取得するために使用。

失敗モードと境界条件

  • 指定されたアクセスメソッドが存在しない場合、エラー報告の過程で AMOID キャッシュ検索が失敗し、elog(ERROR, ...) が発生します。
  • 演算子ファミリーが見つからない場合、missing_ok が false であれば ereport(ERROR, ...) によりトランザクションが中断されます。

保守担当者向けメモ

  • 本関数は、演算子ファミリーの検索における標準的なインターフェースです。新しいアクセスメソッドやカタログ検索ロジックを追加する際は、本関数のキャッシュ検索キー（OPFAMILYAMNAMENSP 等）が適切か確認してください。
