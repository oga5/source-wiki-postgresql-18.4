ResourceReleaseCallbackItem

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/resowner/resowner.c:188

目的

リソース解放時に実行されるコールバック関数とその引数を保持し、連結リストとして管理するためのデータ構造です。

シグネチャまたは形状

typedef struct ResourceReleaseCallbackItem
{
	struct ResourceReleaseCallbackItem *next;
	ResourceReleaseCallback callback;
	void	   *arg;
} ResourceReleaseCallbackItem;

入力と出力

  • **入力**: 構造体生成時に callback 関数ポインタおよび arg（任意のデータへのポインタ）を保持します。
  • **出力**: リストの next ポインタにより、後続のコールバック項目を指し示します。

前提条件と不変条件

  • next ポインタは、リストの終端では NULL である必要があります。
  • callback は、リソース解放フェーズにおいて実行可能な関数である必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は単方向連結リストを構成します。
  • ResourceRelease_callbacks 静的変数（resowner.c:195）を起点として、登録されたコールバックが順次管理されます。
  • リソース解放処理において、このリストを走査することで登録された各コールバックが呼び出されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/resowner/resowner.c 内の関数（963行目付近および977行目付近）で、リストへの追加や走査のためにインスタンス化および参照が行われています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありませんが、保持している callback フィールドを通じて外部関数が実行されます。

関連データ構造

  • ResourceReleaseCallback: コールバック関数の型定義。
  • ResourceRelease_callbacks: 本構造体のリストを保持する静的変数。

失敗モードと境界条件

  • メモリ不足時: malloc（または相当するメモリ確保関数）が失敗した場合、リストへの追加ができません。
  • リストの空状態: ResourceRelease_callbacks が NULL の場合、解放処理は何も行われません。

保守担当者向けメモ

  • 本構造体は resowner.c 内の静的リスト管理に使用されます。リストの操作（追加・削除）を行う際は、メモリリークを防ぐため、適切に pfree 等で解放されているか確認してください。
  • 呼び出し元・呼び出し先の詳細な関数名は、ソースコード内の具体的な登録・実行関数（RegisterResourceReleaseCallback 等の推測されるインターフェース）と照らし合わせて確認してください。
