ssl_protocol_versions

定義

  • 種別: enum
  • 定義箇所: src/include/libpq/libpq.h:128

目的

PostgreSQL バックエンドにおける SSL/TLS 通信で使用可能なプロトコルバージョンを識別・指定するための列挙型です。

シグネチャまたは形状

enum ssl_protocol_versions
{
	PG_TLS_ANY = 0,
	PG_TLS1_VERSION,
	PG_TLS1_1_VERSION,
	PG_TLS1_2_VERSION,
	PG_TLS1_3_VERSION,
};

入力と出力

  • **入力**: 該当なし（定数定義）。
  • **出力**: PG_TLS_ANY から PG_TLS1_3_VERSION までの整数値。

前提条件と不変条件

  • PG_TLS_ANY はデフォルト値として 0 が割り当てられています。
  • 各列挙子は、PostgreSQL がサポートする特定の TLS バージョンに対応します。

アルゴリズムまたは振る舞い

  • SSL/TLS 接続設定において、許可するプロトコルバージョンの範囲や特定のバージョンを指定するために使用されます。
  • PG_TLS_ANY は、特定のバージョンを強制せず、利用可能なプロトコルを許容する設定を意図しています。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数情報は、提供されたソース抜粋には含まれていません。

関連データ構造

  • Port 構造体（libpq-be.h で定義）などの通信状態を管理する構造体において、SSL 設定の一部として参照される可能性があります。

失敗モードと境界条件

  • 定義された範囲外の整数値をこの列挙型としてキャストして使用した場合の挙動は未定義です。

保守担当者向けメモ

  • 本シンボルはバックエンド側の SSL 設定用です。クライアント側の libpq-fe.h で定義される同様の定数と混同しないよう注意してください。
  • 新しい TLS バージョンがサポートされる際は、この列挙型に新しいメンバを追加する必要があります。
