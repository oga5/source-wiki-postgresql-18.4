multixact_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/mxactdesc.c:49

目的

マルチトランザクション（MultiXact）に関連するWALレコードの内容を解析し、人間が読める形式の文字列としてバッファに書き出す。主に pg_waldump 等のログ解析ツールで使用される。

シグネチャまたは形状

void multixact_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 解析結果を格納する StringInfo バッファ。
  • record: 解析対象のWALレコードを保持する XLogReaderState 構造体。
  • **出力**: なし（buf を通じて文字列が書き出される）。

前提条件と不変条件

  • record は有効な XLogReaderState を指していること。
  • XLogRecGetInfo(record) で取得される情報ビットは、XLR_INFO_MASK を除外した状態で、既知のマルチトランザクションWALレコードタイプ（ZERO_OFF_PAGE, ZERO_MEM_PAGE, CREATE_ID, TRUNCATE_ID）のいずれかに一致すること。

アルゴリズムまたは振る舞い

1. XLogRecGetData を用いてレコードの生データポインタを取得する。 2. XLogRecGetInfo からフラグを除去し、レコードタイプを特定する。 3. レコードタイプに応じた処理を実行する:

  • XLOG_MULTIXACT_ZERO_OFF_PAGE または XLOG_MULTIXACT_ZERO_MEM_PAGE: ページ番号（int64）を読み取り、バッファに書き出す。
  • XLOG_MULTIXACT_CREATE_ID: xl_multixact_create 構造体としてデータをキャストし、マルチトランザクションID、オフセット、メンバ数を書き出した後、各メンバ情報を out_member 関数で順次書き出す。
  • XLOG_MULTIXACT_TRUNCATE_ID: xl_multixact_truncate 構造体としてデータをキャストし、オフセットとメンバの切り詰め範囲を書き出す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h で定義されている RM_MULTIXACT_ID リソースマネージャの記述子ルーチンとして登録されており、WAL再生・解析システムから呼び出される。
  • **呼び出し先**:
  • XLogRecGetData, XLogRecGetInfo (WAL解析用API)
  • appendStringInfo (文字列整形用API)
  • out_member (メンバ情報出力用ヘルパー関数)

関連データ構造

  • StringInfo: 出力バッファ。
  • XLogReaderState: WALレコードの読み取り状態。
  • xl_multixact_create, xl_multixact_truncate: マルチトランザクションWALレコードのデータ構造。

失敗モードと境界条件

  • レコードタイプが上記以外の場合、バッファには何も書き出されない。
  • xl_multixact_create のメンバ数が不正な場合、ループ処理でメモリ境界外を参照する可能性がある（WALレコードの整合性に依存）。

保守担当者向けメモ

  • out_member 関数と密結合しているため、MultiXactMember 構造体やロック状態の定義が変更された場合は本関数および out_member の双方を確認すること。
  • XLR_INFO_MASK によるビットマスク処理は、WALレコードのフラグ分離に必須である。
