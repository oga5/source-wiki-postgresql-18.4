autovac_init

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/autovacuum.c:3341

目的

自動バキューム（Autovacuum）システムの起動可否を判定し、必要な設定チェックを行う。

シグネチャまたは形状

void autovac_init(void)

入力と出力

  • 入力: なし（GUCパラメータ autovacuum_start_daemon および pgstat_track_counts を参照）
  • 出力: なし（副作用として警告の出力や設定チェックを実行）

前提条件と不変条件

  • autovacuum_start_daemon が有効であること。
  • pgstat_track_counts が有効であること（無効な場合は警告を発行して終了）。

アルゴリズムまたは振る舞い

1. autovacuum_start_daemon が偽であれば、何もせず即座に復帰する。 2. pgstat_track_counts が偽であれば、警告（WARNING）を発行し、自動バキュームを起動しない。 3. 上記条件を満たす場合、check_av_worker_gucs() を呼び出し、ワーカーに関連するGUCパラメータの整合性を検証する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/postmaster.c:1324
  • 呼び出し先: check_av_worker_gucs

関連データ構造

  • autovacuum_start_daemon (GUC)
  • pgstat_track_counts (GUC)

失敗モードと境界条件

  • pgstat_track_counts が無効な場合、自動バキューム機能は抑制される。
  • 設定の不整合がある場合、check_av_worker_gucs 内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は Postmaster の初期化プロセスの一部として呼び出される。
  • 自動バキュームの起動要件に変更がある場合は、本関数の条件分岐を更新すること。
