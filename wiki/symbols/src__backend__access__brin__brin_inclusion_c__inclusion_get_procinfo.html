inclusion_get_procinfo

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_inclusion.c:543

目的

BRINインデックスの「包含（Inclusion）」演算子クラスにおいて、指定されたサポート関数（procnum）の FmgrInfo 構造体を取得します。システムカタログへの反復的なアクセスを避けるため、InclusionOpaque 構造体へのキャッシュ機能を提供します。

シグネチャまたは形状

static FmgrInfo *inclusion_get_procinfo(BrinDesc *bdesc, uint16 attno, uint16 procnum, bool missing_ok)

入力と出力

  • **入力**:
  • bdesc: インデックスの記述子。
  • attno: 対象の属性番号。
  • procnum: 取得対象のサポート関数番号。
  • missing_ok: 関数が見つからない場合にエラーを発生させるか（false）、NULL を返すか（true）のフラグ。
  • **出力**:
  • 成功時: 対象関数の FmgrInfo へのポインタ。
  • 失敗時: missing_ok が true の場合に NULL。それ以外はエラーで終了。

前提条件と不変条件

  • bdesc->bd_info[attno - 1]->oi_opaque が InclusionOpaque 型としてキャスト可能であること。
  • procnum は PROCNUM_BASE を基準としたオフセット計算に使用される。

アルゴリズムまたは振る舞い

1. InclusionOpaque 内のキャッシュ（extra_proc_missing）を確認し、過去に検索して存在しなかった関数であれば即座に NULL を返す。 2. キャッシュ（extra_procinfos）に fn_oid が存在すればそれを返す。 3. キャッシュにない場合、index_getprocid を用いてシステムカタログから関数OIDを取得する。 4. OIDが有効であれば fmgr_info_copy を用いて情報をキャッシュにコピーし、そのポインタを返す。 5. OIDが無効な場合、missing_ok が false ならばエラーを報告し、true ならばキャッシュに「欠落」フラグを立てて NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • brin_inclusion_add_value
  • brin_inclusion_consistent
  • brin_inclusion_union
  • 同ファイル内の各サポート関数呼び出し箇所（PROCNUM_EMPTY, PROCNUM_CONTAINS, PROCNUM_MERGEABLE, PROCNUM_MERGE 等）。
  • **呼び出し先**:
  • index_getprocid
  • index_getprocinfo
  • fmgr_info_copy
  • ereport

関連データ構造

  • InclusionOpaque: インデックス列ごとのメタデータと関数キャッシュを保持する構造体。
  • BrinDesc: インデックス記述子。
  • FmgrInfo: 関数マネージャの実行情報。

失敗モードと境界条件

  • missing_ok が false の状態で、インデックス定義に存在しないサポート関数を要求した場合、ERRCODE_INVALID_OBJECT_DEFINITION でエラー終了する。
  • 属性番号（attno）が範囲外の場合の挙動は、bdesc->bd_info へのアクセス時に未定義の動作となる可能性がある。

保守担当者向けメモ

  • 本関数は minmax_get_strategy_procinfo と設計が共通しています。修正時は整合性を確認してください。
  • キャッシュの更新は InclusionOpaque のライフサイクルに依存します。インデックスのクローズ時にキャッシュが適切に解放されるか注意してください。
