recurse_set_operations

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepunion.c:208

目的

集合演算（UNION, INTERSECT, EXCEPT）のクエリツリーを再帰的に走査し、各ノードに対応する物理的な実行パス（RelOptInfo）を構築する。

シグネチャまたは形状

static RelOptInfo *recurse_set_operations(Node *setOp, PlannerInfo *root,
                                          SetOperationStmt *parentOp,
                                          List *colTypes, List *colCollations,
                                          List *refnames_tlist,
                                          List **pTargetList,
                                          bool *istrivial_tlist)

入力と出力

  • **入力**:
  • setOp: 処理対象のノード（RangeTblRef または SetOperationStmt）。
  • root: 現在のクエリレベルの PlannerInfo。
  • parentOp: 親の集合演算ノード（サブクエリのプランニング時に伝播）。
  • colTypes, colCollations: 集合演算結果に要求される型と照合順序。
  • refnames_tlist: 参照名解決用のターゲットリスト。
  • **出力**:
  • 戻り値: 構築された RelOptInfo。
  • pTargetList: 決定されたターゲットリスト。
  • istrivial_tlist: ターゲットリストが自明（変換不要）かどうかを示すフラグ。

前提条件と不変条件

  • check_stack_depth() を呼び出し、再帰的なネストによるスタックオーバーフローを防止する。
  • RangeTblRef の場合、サブクエリ内に他の集合演算ノードへの外部参照が含まれていないことを保証する。
  • SetOperationStmt の場合、生成されたパスのターゲットリストが要求された型・照合順序と一致しない場合は、Result ノードによる射影（Projection）を適用する。

アルゴリズムまたは振る舞い

1. **末端ノード (RangeTblRef)**:

  • build_simple_rel で RelOptInfo を作成。
  • subquery_planner を呼び出し、サブクエリのパスを生成。
  • generate_setop_tlist を用いて、集合演算に必要なターゲットリストを構築。

    • 2. **中間ノード (SetOperationStmt)**:

  • op->op が SETOP_UNION なら generate_union_paths を、それ以外なら generate_nonunion_paths を呼び出し、子ノードのパスを結合。
  • 戻り値のターゲットリストが要求と一致しない場合、apply_projection_to_path または create_projection_path を用いて射影を適用。
  • postprocess_setop_rel を呼び出し、コスト計算と最適パスの選択を行う。

    • 3. **エラー処理**: 未知のノード型が渡された場合は elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • plan_set_operations（src/backend/optimizer/prep/prepunion.c）
  • 再帰的に自身を呼び出す（SetOperationStmt の左右の子ノード処理）。
  • **呼び出し先**:
  • subquery_planner, build_simple_rel, generate_setop_tlist, generate_union_paths, generate_nonunion_paths, apply_projection_to_path, create_projection_path, postprocess_setop_rel。

関連データ構造

  • RelOptInfo: 集合演算の結果として生成されるパス情報を保持。
  • SetOperationStmt: 集合演算の論理構造。
  • PlannerInfo: プランニング中のクエリ状態。

失敗モードと境界条件

  • **スタックオーバーフロー**: 集合演算が過度にネストされている場合、check_stack_depth() によりエラーとなる。
  • **外部参照**: サブクエリ内に予期せぬ外部参照がある場合、elog(ERROR) で停止する。
  • **未知のノード**: setOp が RangeTblRef でも SetOperationStmt でもない場合、elog(ERROR) を発生させる。

保守担当者向けメモ

  • istrivial_tlist は初期値として true に設定されるが、処理過程で更新される。
  • varno 0 を使用した射影は、setrefs.c での fix_upper_expr との整合性を保つための特殊な実装である。
  • generate_setop_tlist の呼び出し時に varno を 0 に設定する箇所は、射影ノードの生成ロジックと密接に関連している。
