pg_atomic_fetch_add_u64_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/arch-ppc.h:217

目的

PowerPC アーキテクチャにおいて、64bit 整数に対するアトミックな加算操作を行い、加算前の値を返します。

シグネチャまたは形状

uint64 pg_atomic_fetch_add_u64_impl(volatile pg_atomic_uint64 *ptr, int64 add_)

入力と出力

  • **入力**:
  • ptr: 操作対象となる pg_atomic_uint64 型へのポインタ。
  • add_: 加算する値（int64）。
  • **出力**:
  • 加算操作が行われる直前のメモリ上の値（uint64）。

前提条件と不変条件

  • 本関数は 64bit モードでの実行を前提としています。
  • ptr はアトミック操作のために適切にアライメントされている必要があります。
  • PowerPC の ldarx/stdcx. 命令のペアによる排他制御を利用するため、メモリモデルの整合性が保証されます。

アルゴリズムまたは振る舞い

1. sync 命令により、先行するメモリ操作を完了させます。 2. ldarx 命令でメモリから値をロードし、予約をセットします。 3. 加算処理を実行します（HAVE_I_CONSTRAINT__BUILTIN_CONSTANT_P が有効かつ add_ が 16bit 範囲内の定数の場合は addi 命令、それ以外は add 命令を使用）。 4. stdcx. 命令で計算結果をメモリに書き込みます。 5. 書き込みが失敗した場合（予約が解除された場合）、bne 命令により ldarx の直前へ戻り再試行します。 6. 成功後、lwsync 命令によりメモリ順序を確定させ、加算前の値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/atomics/generic.h 等の汎用アトミックインターフェースから呼び出されます。
  • **呼び出し先**: 直接的な関数呼び出しはありません。インラインアセンブラによる CPU 命令（ldarx, stdcx. 等）を実行します。

関連データ構造

  • pg_atomic_uint64: 操作対象となるアトミック変数型。

失敗モードと境界条件

  • stdcx. 命令によるストア条件付き操作が失敗した場合、ループによる再試行が行われます。
  • 64bit モード以外で呼び出された場合の動作は定義されていません。

保守担当者向けメモ

  • 本関数は GCC インラインアセンブラを使用しています。修正時は memory および cc 制約が適切に設定されていることを確認してください。
  • ldarx/stdcx. のループ構造は PowerPC における標準的なアトミック操作パターンです。最適化を行う際は、この予約メカニズムを破壊しないよう注意してください。
