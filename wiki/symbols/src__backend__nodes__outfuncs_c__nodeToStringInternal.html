nodeToStringInternal

定義

  • 種別: static function
  • 定義箇所: src/backend/nodes/outfuncs.c:783

目的

PostgreSQLの内部ノード構造を、指定された設定に基づいてASCII文字列形式へ変換する内部的なシリアライズ関数です。

シグネチャまたは形状

static char *nodeToStringInternal(const void *obj, bool write_loc_fields)

入力と出力

  • **入力**:
  • obj: 文字列化対象のノード構造体へのポインタ。
  • write_loc_fields: パース時の位置情報（location）を出力に含めるかどうかを制御するフラグ。
  • **出力**:
  • ノードをシリアライズした結果の文字列（char *）。

前提条件と不変条件

  • StringInfoData を使用して文字列を構築するため、メモリ割り当てが成功する必要があります。
  • グローバルな状態フラグ write_location_fields を一時的に変更し、処理終了後に復元する不変条件を持ちます。

アルゴリズムまたは振る舞い

1. 現在のグローバルな write_location_fields の値を退避します。 2. 引数 write_loc_fields をグローバルな write_location_fields に設定します。 3. initStringInfo で文字列バッファを初期化します。 4. outNode を呼び出し、ノードツリーを再帰的に走査してバッファに書き込みます。 5. 退避していた write_location_fields を復元します。 6. 構築された文字列データ（str.data）を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • nodeToString (位置情報なしで呼び出し)
  • nodeToStringWithLocations (位置情報ありで呼び出し)
  • **呼び出し先**:
  • initStringInfo
  • outNode

関連データ構造

  • StringInfoData: 文字列構築用バッファ。
  • Node: シリアライズ対象のノード構造体。

失敗モードと境界条件

  • outNode が再帰的に呼び出されるため、ノードツリーが極端に深い場合はスタックオーバーフローのリスクがあります（outNode 内部で check_stack_depth() により保護されています）。
  • メモリ不足時には initStringInfo 内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は write_location_fields というグローバル変数を一時的に書き換える「マニューバ」を行っています。この実装は stringinfo.h の設計方針に従っており、マルチスレッド環境や再入可能性については、PostgreSQLのバックエンドの実行モデル（プロセスベース）に依存しています。
