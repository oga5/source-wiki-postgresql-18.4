HbaLine

定義

  • 種別: struct
  • 定義箇所: src/include/libpq/hba.h:95

目的

PostgreSQL のホストベース認証設定ファイル（pg_hba.conf）における単一行の設定内容を保持するデータ構造です。接続条件（IPアドレス、データベース、ユーザー等）と、それに対応する認証方式およびパラメータを格納します。

シグネチャまたは形状

typedef struct HbaLine {
    char *sourcefile;
    int linenumber;
    char *rawline;
    ConnType conntype;
    List *databases;
    List *roles;
    struct sockaddr_storage addr;
    int addrlen;
    struct sockaddr_storage mask;
    int masklen;
    IPCompareMethod ip_cmp_method;
    char *hostname;
    UserAuth auth_method;
    char *usermap;
    /* ... LDAP, RADIUS, OAuth, Kerberos 等の認証オプションフィールド ... */
} HbaLine;

入力と出力

  • **入力**: load_hba() 関数が設定ファイルを読み込み、parse_hba_line() を通じて本構造体の各フィールドに値が設定されます。
  • **出力**: 認証プロセスにおいて、hba_getauthmethod() が特定の接続情報と照合するために本構造体のリストを参照します。

前提条件と不変条件

  • addrlen および masklen は、有効なアドレス/マスクが存在しない場合に 0 となります。
  • UserAuth 列挙型で定義された認証方式に応じて、関連するオプション（LDAP設定やOAuth設定など）が適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. 設定ファイルのパース時に palloc0 でメモリが確保され、各トークンが対応するフィールドに割り当てられます。 2. クライアント接続時、メモリ上に保持された HbaLine のリストを走査し、接続情報と一致するルールを特定します。 3. 特定されたルールに基づき、auth.c が実際の認証処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/hba.c（パースおよびリスト管理）
  • src/backend/libpq/auth-oauth.c（check_oauth_validator 内での参照）
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ConnType: 接続タイプ（host, local 等）
  • UserAuth: 認証方式（password, scram, gss 等）
  • List: PostgreSQL 標準のリスト構造（データベース名やロール名の保持に使用）
  • struct sockaddr_storage: IPアドレスおよびマスクの保持に使用

失敗モードと境界条件

  • 設定ファイル内の構文エラーは parse_hba_auth_opt 等のパース関数で検出されます。
  • addrlen や masklen が 0 の場合、IPアドレスによる制限が適用されない（あるいは全許可/全拒否の）境界条件として扱われます。

保守担当者向けメモ

  • HbaLine は非常に多くのフィールドを持つ巨大な構造体です。新しい認証方式（LDAP, RADIUS, OAuth 等）を追加する際は、本構造体にフィールドを追加し、hba.c のパースロジックを更新する必要があります。
  • UserAuth 列挙型を変更した場合は、hba.c 内の関連する配列との同期を忘れないでください。
