bt_page_print_tuples

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/btreefuncs.c:479

目的

B-treeインデックスページ内の個々のタプルを解析し、SQLで表示可能な形式（HeapTuple）に変換して返します。

シグネチャまたは形状

static Datum bt_page_print_tuples(ua_page_items *uargs)

入力と出力

  • **入力**: ua_page_items *uargs（解析対象のページ、オフセット、ページ属性を含む構造体）
  • **出力**: Datum（heap_form_tuple で生成された、タプルの詳細情報を含む行データ）

前提条件と不変条件

  • uargs->page は有効なB-treeページである必要があります。
  • ItemIdIsValid(id) が真である必要があります（無効なアイテムIDの場合はエラーをスロー）。
  • dlen（データ長）が負または INDEX_SIZE_MASK を超える場合はエラーをスローします。
  • ピボットタプル（ispivottuple）の場合、ItemIdIsDead(id) は偽である必要があります。

アルゴリズムまたは振る舞い

1. 指定されたオフセットから ItemId を取得し、IndexTuple を抽出します。 2. タプルの基本情報（オフセット、TID、サイズ、Nullフラグ、可変長フラグ）を values 配列に格納します。 3. タプルのデータ部分を16進数文字列に変換します。この際、ポストティングリストやピボットタプルのTID領域をデータ長から除外します。 4. ページの種類（リーフか否か）とオフセットに基づき、ピボットタプルかどうかを判定します。 5. ピボットタプルでない場合は LP_DEAD ビットの状態を、ピボットタプルの場合は NULL を設定します。 6. ヒープTIDを取得し、ピボットタプルかつ非ヒープキー空間の場合は NULL を設定します。 7. ポストティングリストを持つ場合は、TIDの配列を構築して格納します。 8. heap_form_tuple を使用して結果を構築し、Datum として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pageinspect/btreefuncs.c 内の bt_page_items_internal（702行目および818行目付近）。
  • **呼び出し先**: PageGetItemId, PageGetItem, IndexTupleSize, IndexInfoFindDataOffset, BTreeTupleIsPosting, BTreeTupleGetPostingOffset, BTreeTupleIsPivot, BTreeTupleGetHeapTID, palloc0, CStringGetTextDatum, ItemIdIsDead, BTreeTupleGetPosting, BTreeTupleGetNPosting, construct_array_builtin, heap_form_tuple 等。

関連データ構造

  • ua_page_items: ページ解析に必要なコンテキスト情報を保持。
  • IndexTuple: B-treeインデックス内のタプル構造。
  • BTPageOpaqueData（access/nbtree.h 経由で間接的に参照）。

失敗モードと境界条件

  • ItemId が無効な場合、elog(ERROR) を発生させます。
  • 計算されたデータ長 dlen が不正な場合、elog(ERROR) を発生させます。
  • !heapkeyspace インデックスにおけるピボットタプルの判定には制限があり、ページの種類やオフセット位置による推論に依存します。

保守担当者向けメモ

  • 本関数は pageinspect モジュールの一部であり、スーパーユーザー権限でのみ実行可能です。
  • BTreeTupleIsPivot() の挙動はインデックスのバージョン（heapkeyspace の有無）に依存するため、将来的なPostgreSQLの内部構造変更時にはこの判定ロジックの再検証が必要です。
