WalSummaryFile

定義

  • 種別: struct
  • 定義箇所: src/include/backup/walsummary.h:27

目的

インクリメンタルバックアップにおいて、特定の WAL セグメント範囲を要約したファイルのメタデータを保持する。物理的な WAL ファイルを走査することなく、バックアップに必要な LSN 範囲の網羅性を検証するために使用される。

シグネチャまたは形状

typedef struct WalSummaryFile
{
	XLogRecPtr	start_lsn;
	XLogRecPtr	end_lsn;
	TimeLineID	tli;
} WalSummaryFile;

入力と出力

本構造体はデータ保持用であり、関数ではない。

  • **入力**: GetWalSummaries() 等により、ファイルシステム上のサマリーファイル名からパースされた値が格納される。
  • **出力**: WalSummariesAreComplete() 等の検証関数や、ソート用比較関数において、各フィールドの値が参照される。

前提条件と不変条件

  • start_lsn は、そのサマリーがカバーする開始位置を示す。
  • end_lsn は、そのサマリーがカバーする終了位置を示す。
  • tli は、当該サマリーが生成されたタイムラインを識別する。
  • リストとして管理される場合、start_lsn に基づいてソートされることが一般的である（walsummary.c 内の比較関数による）。

アルゴリズムまたは振る舞い

本構造体自体はデータコンテナであり、ロジックは含まない。関連する振る舞いとして、walsummary.c 内で以下の処理が行われる：

  • ディレクトリ走査によるリスト構築。
  • start_lsn を基準としたリストのソート。
  • 指定された LSN 範囲に対する網羅性チェック。
  • 特定の期限（cutoff_time）に基づく古いサマリーの削除判定。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_incremental.c
  • src/backend/backup/walsummary.c
  • src/backend/backup/walsummaryfuncs.c
  • **呼び出し先**:
  • 本構造体はデータ構造であるため、関数呼び出しは発生しない。

関連データ構造

  • WalSummaryIO: ファイル操作（I/O）の状態を管理する構造体。
  • List (PostgreSQL の nodes/pg_list.h): WalSummaryFile オブジェクトの集合を管理するために使用される。

失敗モードと境界条件

  • 構造体自体に失敗モードはないが、本構造体を利用する関数において、start_lsn と end_lsn の不整合や、タイムラインの不一致が検出される場合がある。
  • 網羅性検証時、リスト内に隙間がある場合は WalSummariesAreComplete が失敗（または欠落箇所の報告）を行う。

保守担当者向けメモ

  • 本構造体は src/backend/backup/walsummary.c 内のファイル名パース処理と密接に連携している。構造体のフィールドを変更する場合は、ファイル名のフォーマット検証ロジックとの整合性に注意すること。
  • タイムライン（tli）の考慮は本構造体単体では行われないため、利用側で適切なフィルタリングを行う必要がある。
