目的

ECPG（Embedded SQL for C）の Informix 互換ライブラリ関数 intoasc の動作を検証する回帰テスト用エントリポイント。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム実行時に内部で interval 構造体を生成）。
  • 出力: 変換前後の dirty_str の内容を標準出力へ出力。終了コード 0 を返す。

前提条件と不変条件

  • interval 型のメモリ割り当てが成功していること。
  • dirty_str が intoasc による書き込みを受け入れ可能な状態であること。

アルゴリズムまたは振る舞い

1. interval 構造体へのメモリを動的に確保する。 2. interval_ptr の time フィールドに 100000000、month フィールドに 240 を設定する。 3. 変換前の dirty_str の内容を標準出力に表示する。 4. intoasc(interval_ptr, dirty_str) を呼び出し、interval データを ASCII 文字列へ変換して dirty_str に格納する。 5. 変換後の dirty_str の内容を標準出力に表示し、変換結果を検証する。 6. 0 を返して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: テスト実行環境（OS またはテストスイート）。
  • 呼び出し先: malloc, printf, intoasc。
  • 注記: 呼び出し元に関する詳細なソースコード上の参照情報は提供されていません。

関連データ構造

  • interval: pgtypes_interval.h で定義される時間間隔構造体。
  • dirty_str: 変換結果を保持する固定長文字配列。

失敗モードと境界条件

  • malloc が失敗した場合、以降のポインタ参照でセグメンテーション違反が発生する可能性がある。
  • intoasc が期待する形式で変換できない場合、dirty_str の内容が不正になる可能性がある。

保守担当者向けメモ

  • 本関数は ECPG プリプロセッサが生成する「期待値」コードの一部であり、手動編集は推奨されません。
  • Informix 互換モード特有の挙動を確認するためのテストであり、PostgreSQL ネイティブな実装とは異なるインターフェースである点に留意してください。
