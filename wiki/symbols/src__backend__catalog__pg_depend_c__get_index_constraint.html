get_index_constraint

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_depend.c:1060

目的

指定されたインデックス（indexId）に紐付いている制約（Constraint）の OID を取得します。

シグネチャまたは形状

Oid get_index_constraint(Oid indexId)

入力と出力

  • **入力**: indexId (Oid) - 検索対象のインデックスの OID。
  • **出力**: Oid - 見つかった制約の OID。該当する制約が存在しない場合は InvalidOid を返します。

前提条件と不変条件

  • pg_depend システムカタログに対する AccessShareLock を取得してスキャンを行います。
  • 検索対象は pg_depend テーブルの classid が RelationRelationId（インデックス自身）であるレコードに限定されます。

アルゴリズムまたは振る舞い

1. pg_depend テーブルを DependDependerIndexId インデックスを用いてスキャンします。 2. 検索条件として、classid が RelationRelationId、objid が指定された indexId、objsubid が 0 であるレコードを抽出します。 3. スキャン結果をループし、以下の条件をすべて満たすレコードを探します。

  • refclassid が ConstraintRelationId であること。
  • refobjsubid が 0 であること。
  • deptype が DEPENDENCY_INTERNAL であること。

    • 4. 条件に合致したレコードの refobjid（制約の OID）を返し、スキャンを終了します。

関連データ構造

  • pg_depend: 依存関係を保持するシステムカタログ。
  • Form_pg_depend: pg_depend テーブルのタプル構造体。

失敗モードと境界条件

  • 指定された indexId に対応する制約が存在しない場合、InvalidOid を返します。
  • 内部依存（DEPENDENCY_INTERNAL）としてマークされていない制約は無視されます。

保守担当者向けメモ

  • 本関数は、インデックスが特定の制約（主キーやユニーク制約など）によって作成されたものであるかを判定するために使用されます。
  • refobjsubid が 0 であることを確認しているため、テーブル全体やインデックス全体に対する依存関係のみを対象としています。
