jsonb_contains

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_op.c:111

目的

2つの jsonb 値を比較し、第1引数が第2引数を包含しているかどうか（@> 演算子）を判定します。

シグネチャまたは形状

Datum jsonb_contains(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_JSONB_P(0): 包含される側（スーパーセット）の jsonb ポインタ。
  • PG_GETARG_JSONB_P(1): 包含する側（サブセット）の jsonb ポインタ。
  • **出力**:
  • bool: 包含関係が成立する場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力された jsonb は有効な構造体である必要があります。
  • 包含関係の判定において、両者のルート構造（オブジェクトか否か）が一致しない場合、包含関係は成立しません。

アルゴリズムまたは振る舞い

1. PG_GETARG_JSONB_P を使用して2つの jsonb 引数を取得します。 2. JB_ROOT_IS_OBJECT マクロを用いて、両者のルートが共にオブジェクトであるかを確認し、不一致であれば即座に false を返します。 3. JsonbIteratorInit を用いて、両方の jsonb に対してイテレータを初期化します。 4. JsonbDeepContains を呼び出し、再帰的に包含関係を検証した結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（fmgr）を介してSQL演算子 @> から呼び出されます。
  • **呼び出し先**:
  • JsonbIteratorInit
  • JsonbDeepContains

関連データ構造

  • Jsonb: jsonb 型の内部表現。
  • JsonbIterator: jsonb 構造を走査するためのイテレータ。

失敗モードと境界条件

  • 両引数のルート型が異なる場合（例：一方がオブジェクトで他方が配列）、包含関係は成立せず false となります。
  • 内部的な走査において、JsonbDeepContains が再帰的に構造を比較します。

保守担当者向けメモ

  • 本関数は jsonb_op.c 内で定義されており、SQLレベルの @> 演算子に対応しています。
  • 包含判定のロジックは JsonbDeepContains に依存しているため、包含の定義を変更する場合はそちらの確認が必要です。
