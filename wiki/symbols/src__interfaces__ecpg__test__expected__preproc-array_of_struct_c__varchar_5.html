目的

ECPG（Embedded SQL for C）において、可変長文字列（VARCHAR）を表現するためのデータ構造です。データベースとの間で文字列データをやり取りする際のホスト変数として使用されます。

シグネチャまたは形状

struct varchar_5 {
    int len;
    char arr[50];
} onlyname[2];
  • len: 文字列の長さを保持する整数型。
  • arr: 最大50バイトの文字列を格納する固定長配列。
  • onlyname: この構造体の要素を2つ持つ配列として定義されています。

入力と出力

  • 入力: SQL の SELECT 文等の結果セットから取得される文字列データ。
  • 出力: C 言語側で処理された文字列データ。

前提条件と不変条件

  • arr 配列のサイズは50バイトに固定されています。
  • ECPG プリプロセッサが生成するコードにおいて、データベースの列定義とこの構造体のメンバ順序が一致している必要があります。

アルゴリズムまたは振る舞い

ECPG プリプロセッサによってホスト変数として認識され、データベースとのデータ転送時に len メンバが文字列の長さを、arr メンバが実際のデータ内容を保持するように自動的に管理されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/test/expected/preproc-array_of_struct.c:258 において、ECPGdo 関数へ引数として渡されています。
  • 呼び出し先: データベースとの通信を行う ECPGdo ライブラリ関数が、この構造体を介してデータ操作を行います。

関連データ構造

  • cust_ind: 構造体メンバに対応するインジケータ変数。

失敗モードと境界条件

  • arr のサイズ（50バイト）を超える文字列が格納された場合、バッファオーバーフローやデータの切り捨てが発生する可能性があります。
  • len の値が arr の物理サイズを超えないことが期待されます。
