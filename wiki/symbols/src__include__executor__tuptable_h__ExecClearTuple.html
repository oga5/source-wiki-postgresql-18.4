ExecClearTuple

定義

  • 種別: function
  • 定義箇所: src/include/executor/tuptable.h:457

目的

タプルテーブルスロット（TupleTableSlot）に保持されている現在のタプルデータを解放し、スロットを空の状態にする。

シグネチャまたは形状

static inline TupleTableSlot *ExecClearTuple(TupleTableSlot *slot)

入力と出力

  • **入力**: slot (TupleTableSlot *) - クリア対象のスロット。
  • **出力**: TupleTableSlot * - クリア処理後の slot ポインタをそのまま返す。

前提条件と不変条件

  • slot は有効な TupleTableSlot 構造体を指している必要がある。
  • slot->tts_ops が適切に初期化されており、clear 関数ポインタが実装されていること。

アルゴリズムまたは振る舞い

1. slot が保持する TupleTableSlotOps（仮想関数テーブル）の clear メソッドを呼び出す。 2. clear メソッドは、スロットの型（仮想、ヒープ、バッファ等）に応じた適切なリソース解放（参照カウントの減少やメモリのクリア等）を行う。 3. 処理完了後、入力された slot ポインタを戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/file_fdw/file_fdw.c
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/access/common/tupconvert.c
  • src/backend/access/heap/heapam.c
  • **呼び出し先**:
  • slot->tts_ops->clear(slot)（TupleTableSlotOps で定義された実装依存のクリア関数）。

関連データ構造

  • TupleTableSlot: タプルを保持する基本構造体。
  • TupleTableSlotOps: スロットの挙動を定義する関数ポインタテーブル。

失敗モードと境界条件

  • slot が NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • 既に空のスロットに対して呼び出した場合の挙動は、各 TupleTableSlotOps 実装の clear メソッドに依存する。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、実行エンジン全体で頻繁に呼び出される。
  • 新しいタプル形式を追加する場合は、TupleTableSlotOps 構造体で clear メソッドを適切に実装する必要がある。
