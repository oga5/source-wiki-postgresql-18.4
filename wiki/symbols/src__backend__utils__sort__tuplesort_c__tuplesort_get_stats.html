tuplesort_get_stats

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplesort.c:2498

目的

ソート操作の実行統計（使用リソース量および適用されたソート手法）を抽出し、TuplesortInstrumentation 構造体に格納する。

シグネチャまたは形状

void tuplesort_get_stats(Tuplesortstate *state, TuplesortInstrumentation *stats)

入力と出力

  • **入力**:
  • state: ソート状態を保持する Tuplesortstate へのポインタ。
  • stats: 統計情報を書き込む TuplesortInstrumentation へのポインタ。
  • **出力**: なし（stats 引数を通じて結果を返却）。

前提条件と不変条件

  • state は初期化済みであること。
  • 外部ソート（ディスク使用）の場合、呼び出し元がタプルを取得した後の正確なメモリ使用量は追跡されないため、stats->spaceUsed はディスクソート開始時点までの最大値に基づく。

アルゴリズムまたは振る舞い

1. tuplesort_updatemax(state) を呼び出し、現在の最大リソース使用量を更新する。 2. state->isMaxSpaceDisk に基づき、stats->spaceType を SORT_SPACE_TYPE_DISK または SORT_SPACE_TYPE_MEMORY に設定する。 3. state->maxSpace をキロバイト単位に換算し、stats->spaceUsed に格納する。 4. state->maxSpaceStatus に基づき、以下のいずれかのソート手法を stats->sortMethod に設定する：

  • TSS_SORTEDINMEM: boundUsed が真なら SORT_TYPE_TOP_N_HEAPSORT、それ以外は SORT_TYPE_QUICKSORT。
  • TSS_SORTEDONTAPE: SORT_TYPE_EXTERNAL_SORT。
  • TSS_FINALMERGE: SORT_TYPE_EXTERNAL_MERGE。
  • デフォルト: SORT_TYPE_STILL_IN_PROGRESS。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/explain.c
  • src/backend/executor/nodeIncrementalSort.c
  • src/backend/executor/nodeSort.c
  • **呼び出し先**:
  • tuplesort_updatemax（内部関数）

関連データ構造

  • Tuplesortstate: ソートの実行状態を保持するメイン構造体。
  • TuplesortInstrumentation: 統計情報を格納する構造体。

失敗モードと境界条件

  • ディスクベースのソートにおいて、タプル返却後のメモリ使用量は正確に反映されない。
  • 並列ソート環境下では、本関数は単一ワーカーの役割に関する統計のみを報告する。

保守担当者向けメモ

  • メモリコンテキストの実際の使用量を正確に取得するAPIが存在しないため、現在の実装ではメモリ使用量の追跡に制限がある。このオーバーヘッドを解消する修正は現時点では優先されていない。
