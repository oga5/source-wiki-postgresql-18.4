CleanupProcSignalState

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/procsignal.c:229

目的

プロセス終了時に、共有メモリ上の ProcSignalSlot を解放し、後続のシグナル処理やバリア待機による誤動作を防ぐ。

シグネチャまたは形状

static void CleanupProcSignalState(int status, Datum arg);

入力と出力

  • **入力**:
  • status: 終了ステータス（on_shmem_exit 経由で渡される）。
  • arg: 補助データ（未使用）。
  • **出力**: なし（副作用として共有メモリ状態を更新）。

前提条件と不変条件

  • MyProcSignalSlot が有効なスロットを指していること。
  • プロセス終了処理の一環として呼び出されること。

アルゴリズムまたは振る舞い

1. MyProcSignalSlot を NULL に設定し、以降の SIGUSR1 受信時に当該スロットへアクセスしないようにする。 2. pss_mutex を取得し、スロット内の PID が自プロセスと一致するか検証する。一致しない場合はログを出力して終了する。 3. pss_pid を 0 に設定し、スロットを未使用状態にする。 4. pss_cancel_key_len を 0 にリセットする。 5. pss_barrierGeneration を PG_UINT64_MAX に設定し、バリア待機中のプロセスが当該スロットでブロックされないようにする。 6. pss_mutex を解放後、ConditionVariableBroadcast を呼び出し、バリア待機中の全プロセスへ通知を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: on_shmem_exit に登録されたコールバックとして、共有メモリ終了時に呼び出される。
  • **呼び出し先**: SpinLockAcquire/Release、pg_atomic_read_u32/write_u32/write_u64、ConditionVariableBroadcast。

関連データ構造

  • ProcSignalSlot: プロセスごとのシグナル状態を保持する構造体。
  • ProcSignal: 共有メモリ上のヘッダー構造体。

失敗モードと境界条件

  • スロット内の PID が自プロセスと一致しない場合、エラーを発生させずにログ出力のみを行い処理を中断する（無限ループ回避のため）。

保守担当者向けメモ

  • 本関数は on_shmem_exit を通じて実行されるため、呼び出し元コードの明示的な呼び出しは不要です。
  • pss_barrierGeneration を最大値に設定する処理は、終了中のプロセスがバリアの進行を阻害しないための重要な不変条件維持策です。
