testprs_lextype

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_parser/test_parser.c:107

目的

PostgreSQLのカスタムパーサーがサポートするトークン型のリストをシステムに提供します。

シグネチャまたは形状

Datum testprs_lextype(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（PostgreSQLの関数呼び出し規約に基づく引数）。
  • **出力**: LexDescr 構造体の配列へのポインタを Datum 型として返します。

前提条件と不変条件

  • palloc を使用してメモリを確保するため、PostgreSQLのメモリコンテキスト内で実行される必要があります。
  • 配列の末尾には lexid = 0 を持つ要素を配置し、リストの終端を示す必要があります。

アルゴリズムまたは振る舞い

1. LexDescr 構造体の配列を3要素分（トークン型2種 + 終端1種）メモリ確保します。 2. インデックス0に lexid = 3（"word"）を設定します。 3. インデックス1に lexid = 12（"blank"）を設定します。 4. インデックス2に lexid = 0 を設定して終端を明示します。 5. PG_RETURN_POINTER マクロを使用して、構築した配列を返します。

  • 注: ここで指定する lexid は、既存の標準パーサーのハイライト機能と互換性を持たせるために意図的に選択されています。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（PG_FUNCTION_INFO_V1(testprs_lextype) により登録）。
  • **呼び出し先**: palloc および pstrdup（PostgreSQLのメモリ管理関数）。

関連データ構造

  • LexDescr: トークンのID、エイリアス、説明を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • 戻り値の配列サイズが不足すると、パーサーのメタデータ読み込み時に不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本関数で定義されるトークンID（3と12）は、PostgreSQLの標準パーサーと共通化されています。変更を加える際は、既存の全文検索機能（ハイライト機能など）への影響を考慮してください。
