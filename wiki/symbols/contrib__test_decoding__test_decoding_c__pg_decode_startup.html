pg_decode_startup

定義

  • 種別: function
  • 定義箇所: contrib/test_decoding/test_decoding.c:158

目的

論理デコーディングプラグインの初期化を行い、ユーザー指定のオプションを解析してプラグイン固有の状態（TestDecodingData）を構築・設定する。

シグネチャまたは形状

static void pg_decode_startup(LogicalDecodingContext *ctx, OutputPluginOptions *opt, bool is_init)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • opt: 出力プラグインのオプション設定構造体。
  • is_init: 初期化フラグ。
  • **出力**: なし（副作用として ctx->output_plugin_private に設定値を格納し、opt を更新する）。

前提条件と不変条件

  • ctx->output_plugin_options に格納された各オプション値は、DefElem 構造体として提供される必要がある。
  • parse_bool による解析が失敗した場合、ereport によりエラーがスローされる。

アルゴリズムまたは振る舞い

1. TestDecodingData 構造体をメモリ確保し、デフォルト値を設定する。 2. ctx->output_plugin_private に確保したデータを割り当てる。 3. デフォルトの出力形式を OUTPUT_PLUGIN_TEXTUAL_OUTPUT に設定する。 4. ctx->output_plugin_options を反復処理し、以下のオプションを解析・適用する：

  • include-xids, include-timestamp, force-binary, skip-empty-xacts, only-local, include-rewrites, stream-changes

    • 5. 不明なオプションが指定された場合はエラーを報告する。 6. ctx->streaming を enable_streaming フラグに基づいて更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: _PG_output_plugin_init 内でコールバックとして登録され、論理デコーディングの起動時にコアエンジンから呼び出される。
  • **呼び出し先**: palloc0, AllocSetContextCreate, lfirst, strcmp, strVal, parse_bool, ereport 等のPostgreSQL内部API。

関連データ構造

  • TestDecodingData: プラグインの設定状態を保持する構造体。
  • LogicalDecodingContext: 論理デコーディングの実行コンテキスト。
  • OutputPluginOptions: 出力プラグインの動作オプション。

失敗モードと境界条件

  • 未知のオプションが渡された場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー終了する。
  • オプション値のパース（parse_bool）に失敗した場合、エラー終了する。

保守担当者向けメモ

  • 本関数はプラグインの初期化フェーズで一度だけ実行される。
  • 新しいオプションを追加する場合は、foreach ループ内に条件分岐を追加し、TestDecodingData 構造体へフィールドを追加する必要がある。
