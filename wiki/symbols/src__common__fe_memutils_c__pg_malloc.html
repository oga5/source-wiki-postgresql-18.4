pg_malloc

定義

  • 種別: function
  • 定義箇所: src/common/fe_memutils.c:52

目的

PostgreSQLのフロントエンドプログラムにおいて、安全なメモリ確保を行うためのラッパー関数です。メモリ確保失敗時のエラーハンドリングを統一し、呼び出し側の負担を軽減します。

シグネチャまたは形状

void *pg_malloc(size_t size)

入力と出力

  • **入力**: size_t size（確保したいメモリのバイト数）
  • **出力**: 確保されたメモリ領域へのポインタ。確保に成功した場合のみ戻り、失敗した場合はプロセスが終了するため、呼び出し側は戻り値が NULL かどうかをチェックする必要はありません。

前提条件と不変条件

  • FRONTEND マクロが定義された環境でのみ使用可能です。
  • size が SIZE_MAX / 2 を超える場合、安全上の制約によりエラーとして扱われます。
  • malloc(0) が要求された場合、移植性を考慮して内部的に malloc(1) として処理されます。

アルゴリズムまたは振る舞い

1. 内部関数 pg_malloc_internal(size, 0) を呼び出します。 2. pg_malloc_internal 内で標準の malloc を実行します。 3. malloc が失敗した場合、stderr にエラーメッセージを出力し、exit(EXIT_FAILURE) を呼び出してプロセスを終了します。

呼び出し元と呼び出し先

  • **呼び出し先**: pg_malloc_internal
  • **呼び出し元**:
  • contrib/oid2name/oid2name.c (複数箇所)
  • src/bin/initdb/initdb.c (複数箇所)

関連データ構造

  • なし（標準の size_t および void * を使用）

失敗モードと境界条件

  • **メモリ不足**: malloc が NULL を返した場合、プロセスは即座に終了します。
  • **サイズ制限**: size > SIZE_MAX / 2 の場合、エラーとしてプロセスが終了します。

保守担当者向けメモ

  • 本関数は pg_malloc_internal の単純なラッパーです。フラグ制御が必要な場合は pg_malloc_extended の使用を検討してください。
  • バックエンドの palloc との互換性を維持するため、メモリ確保の挙動を変更する際は src/common/fe_memutils.c 全体の設計方針に従ってください。
