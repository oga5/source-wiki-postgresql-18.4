range_get_typcache

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes.c:1766

目的

指定された範囲型のメタデータ（TypeCacheEntry）をキャッシュから取得または生成し、関数呼び出しの実行コンテキスト（fn_extra）に保持します。

シグネチャまたは形状

TypeCacheEntry *range_get_typcache(FunctionCallInfo fcinfo, Oid rngtypid)

入力と出力

  • **入力**:
  • fcinfo: 現在の関数呼び出し情報。fn_extra を介してキャッシュを永続化するために使用。
  • rngtypid: 取得対象の範囲型の OID。
  • **出力**:
  • TypeCacheEntry*: 範囲型に関する情報（要素型、比較関数等）を含むキャッシュエントリ。

前提条件と不変条件

  • fcinfo->flinfo->fn_extra は、呼び出し間でキャッシュを保持するために使用されます。
  • 指定された rngtypid がシステムカタログ上で有効な範囲型である必要があります。

アルゴリズムまたは振る舞い

1. fcinfo->flinfo->fn_extra から既存のキャッシュを取得します。 2. キャッシュが空であるか、キャッシュされている型 ID が要求された rngtypid と一致しない場合、lookup_type_cache を呼び出して情報を再取得します。 3. 取得したエントリの rngelemtype が NULL の場合、指定された型が範囲型ではないと判断し、エラーを発生させます。 4. 新しく取得したキャッシュを fcinfo->flinfo->fn_extra に格納し、呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/rangetypes.c 内の範囲型演算関数群（range_lower, range_upper, range_constructor2, range_constructor3 等）。
  • **呼び出し先**: lookup_type_cache（utils/lsyscache.h）。

関連データ構造

  • TypeCacheEntry: 範囲型の要素型情報や比較関数を保持する構造体。
  • FunctionCallInfo: PostgreSQL の関数呼び出し規約におけるコンテキスト構造体。

失敗モードと境界条件

  • rngtypid が範囲型ではない場合、elog(ERROR) によりトランザクションが中断されます。

保守担当者向けメモ

  • 本関数は fn_extra を利用してキャッシュを保持するため、同一の FunctionCallInfo を再利用する関数内でのみ有効です。
  • lookup_type_cache に渡される TYPECACHE_RANGE_INFO フラグにより、範囲型特有のメタデータが確実にロードされることが保証されています。
