local2local

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conv.c:32

目的

単一バイト文字セット間でのエンコーディング変換を、提供された変換テーブルを用いて実行します。

シグネチャまたは形状

int local2local(const unsigned char *l,
                unsigned char *p,
                int len,
                int src_encoding,
                int dest_encoding,
                const unsigned char *tab,
                bool noError)

入力と出力

  • **入力**:
  • l: 変換元文字列へのポインタ。
  • p: 変換先文字列を格納するバッファへのポインタ。
  • len: 変換対象のバイト数。
  • src_encoding, dest_encoding: 変換元および変換先のエンコーディング識別子。
  • tab: 変換テーブル（HIGHBIT をオフセットとして使用）。
  • noError: 変換失敗時にエラーを報告せず中断するかどうかを指定するフラグ。
  • **出力**:
  • 戻り値: 実際に変換・処理されたバイト数。

前提条件と不変条件

  • tab は、上位ビットがセットされた文字（HIGHBIT 以上）を変換するためのルックアップテーブルである必要があります。
  • 変換後のバッファ p は、入力文字列の長さを収容するのに十分な領域が必要です。
  • 処理終了後、p が指す位置にヌル終端文字 (\0) が書き込まれます。

アルゴリズムまたは振る舞い

1. 入力文字列をバイト単位で走査します。 2. IS_HIGHBIT_SET(c1) が偽（ASCII範囲）の場合、そのまま出力バッファへコピーします。 3. IS_HIGHBIT_SET(c1) が真の場合、tab[c1 - HIGHBIT] を参照して変換後の文字を取得します。 4. 変換後の文字が存在しない場合、noError が偽であれば report_untranslatable_char を呼び出し、真であれば処理を中断します。 5. 入力文字が 0（ヌル）の場合、noError が偽であれば report_invalid_encoding を呼び出し、真であれば処理を中断します。 6. 処理したバイト数をカウントし、最後に p にヌル終端を付与して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mb/conversion_procs/cyrillic_and_mic/cyrillic_and_mic.c 内の各変換関数（koi2win1251, win12512koi, koi2win866, win8662koi, win8662win1251, win12512win866, iso2koi, koi2iso, iso2win1251, win12512iso）。
  • **呼び出し先**:
  • report_invalid_encoding（エラー発生時）
  • report_untranslatable_char（変換不可文字発生時）

関連データ構造

  • tab: 変換テーブル（unsigned char 配列）。

失敗モードと境界条件

  • **不正なエンコーディング**: 入力に 0 が含まれる場合、noError が偽であればエラーを報告します。
  • **変換不可文字**: 変換テーブルで対応する値が見つからない場合、noError が偽であればエラーを報告します。
  • **境界条件**: len が 0 に達した時点でループを終了します。

保守担当者向けメモ

  • 本関数は単一バイト文字セット専用です。マルチバイト文字を含む変換には使用できません。
  • noError フラグの挙動は、呼び出し元が変換失敗を許容するかどうかに直結するため、エラーハンドリングの設計変更時には注意が必要です。
