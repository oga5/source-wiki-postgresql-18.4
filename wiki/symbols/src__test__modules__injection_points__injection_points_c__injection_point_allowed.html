injection_point_allowed

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_points.c:200

目的

指定されたインジェクションポイントの条件（InjectionPointCondition）が、現在の実行コンテキストにおいて適用可能かどうかを判定します。

シグネチャまたは形状

static bool injection_point_allowed(InjectionPointCondition *condition)

入力と出力

  • **入力**: InjectionPointCondition *condition - 評価対象の条件構造体。
  • **出力**: bool - 条件が満たされている場合は true、それ以外は false を返します。

前提条件と不変条件

  • condition 引数は有効なポインタである必要があります。
  • condition->type に基づいて条件分岐が行われます。

アルゴリズムまたは振る舞い

1. 初期値として result を true に設定します。 2. condition->type に応じて以下の判定を行います。

  • INJ_CONDITION_PID: 現在のプロセスID (MyProcPid) が condition->pid と一致するかを確認します。一致しない場合、result を false に設定します。
  • INJ_CONDITION_ALWAYS: 条件に関わらず result は true のまま維持されます。

    • 3. 最終的な result を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:

  • **呼び出し先**:
  • 本関数は直接的な外部関数の呼び出しを行わず、グローバル変数 MyProcPid を参照します。

関連データ構造

  • InjectionPointCondition: 条件の種類と詳細（PID等）を保持する構造体。

失敗モードと境界条件

  • condition が NULL の場合、未定義の動作（セグメンテーション違反）が発生する可能性があります。
  • 未知の condition->type が渡された場合、デフォルトで true を返します。

保守担当者向けメモ

  • 本関数はテストモジュール専用のロジックです。
  • 新しい条件タイプを追加する場合は、switch 文にケースを追加してください。
