CopyMethod

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_combinebackup/copy_file.h:19

目的

バックアップの結合処理において、ファイルデータをコピーまたは転送する際に使用する手法を識別・指定するために定義されています。

シグネチャまたは形状

typedef enum CopyMethod
{
	COPY_METHOD_CLONE,
	COPY_METHOD_COPY,
	COPY_METHOD_COPY_FILE_RANGE,
#ifdef WIN32
	COPY_METHOD_COPYFILE,
#endif
	COPY_METHOD_LINK,
} CopyMethod;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし（列挙型定義）

前提条件と不変条件

  • WIN32 環境下では COPY_METHOD_COPYFILE が定義に含まれます。
  • それ以外の環境では、プラットフォーム固有の定数を除いた上記列挙子が使用されます。

アルゴリズムまたは振る舞い

本シンボルは、ファイル操作の戦略を選択するための識別子として機能します。具体的なコピー処理のロジックは、この列挙値を受け取る関数（copy_file.c 等）によって実装されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/bin/pg_combinebackup/copy_file.c
  • src/bin/pg_combinebackup/pg_combinebackup.c
  • src/bin/pg_combinebackup/reconstruct.c
  • src/bin/pg_combinebackup/reconstruct.h
  • 呼び出し先: なし（本シンボルはデータ型定義であり、関数ではないため）

関連データ構造

  • common/checksum_helper.h が直接インクルードされています。

失敗モードと境界条件

  • 列挙型であるため、定義されていない値をキャスト等で代入した場合の動作は、これを利用する側の実装に依存します。

保守担当者向けメモ

  • 新しいコピー手法を追加する場合は、プラットフォーム固有の条件分岐（#ifdef）を適切に考慮してください。
  • 本列挙型を使用する関数群（reconstruct.c 等）の引数として渡されるため、変更時は関連する関数シグネチャとの整合性に注意してください。
