scan_file

定義

  • 種別: function
  • 定義箇所: src/bin/pg_checksums/pg_checksums.c:174

目的

データファイルを開き、ブロック単位で読み込み、チェックサムの検証（CHECKモード）または計算と書き込み（ENABLEモード）を行う。

シグネチャまたは形状

static void scan_file(const char *fn, int segmentno)

入力と出力

  • **入力**:
  • fn: 対象ファイルパス（文字列）
  • segmentno: セグメント番号（整数）
  • **出力**: なし（副作用としてファイルへの書き込みやログ出力を行う）

前提条件と不変条件

  • mode が PG_MODE_ENABLE または PG_MODE_CHECK であること。
  • クラスタが停止状態であること（稼働中の実行はデータ破損の誤検知を招く）。

アルゴリズムまたは振る舞い

1. 指定されたファイルを mode に応じた権限（O_RDWR または O_RDONLY）で開く。 2. ファイルを BLCKSZ 単位でループ読み込みする。 3. PageIsNew で新規ページと判定された場合はスキップする。 4. pg_checksum_page を用いてチェックサムを計算する。 5. CHECK モードの場合、計算値とページヘッダーの pd_checksum を比較し、不一致があればエラーを記録する。 6. ENABLE モードの場合、計算値と既存の pd_checksum が異なる場合にのみ、ページヘッダーを更新してファイルに書き戻す。 7. 進捗表示が有効な場合、progress_report を呼び出す。 8. 処理終了後、ファイルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_checksums/pg_checksums.c 内の scan_directory（推測されるが、提供されたソース内では直接の呼び出し箇所は明示されていない）。
  • **呼び出し先**: pg_checksum_page, read, write, lseek, pg_fatal, pg_log_error, pg_log_info, progress_report, close。

関連データ構造

  • PGIOAlignedBlock: ページ読み込み用バッファ。
  • PageHeader: ページヘッダー構造体。
  • ControlFileData: クラスタのメタデータ（チェックサムバージョン確認に使用）。

失敗モードと境界条件

  • ファイルのオープン失敗、読み込み失敗、書き込み失敗、または lseek 失敗時に pg_fatal を呼び出しプロセスを終了する。
  • 読み込みサイズが BLCKSZ と一致しない場合、エラーとして処理される。

保守担当者向けメモ

  • 進捗表示（showprogress）が有効な場合、ファイルサイズ計算のために走査が二重に行われる仕様である点に注意すること。
  • 新しいシステムファイルを追加する際は、skip[] 配列の除外リストを確認すること。
