PLpgSQL_type

定義

  • 種別: struct
  • 定義箇所: src/pl/plpgsql/src/plpgsql.h:210

目的

PL/pgSQL におけるデータ型のメタデータおよび実行時情報を保持する構造体です。コンパイル時に型情報をキャッシュし、実行エンジンが変数の型を識別するために使用されます。

シグネチャまたは形状

typedef struct PLpgSQL_type
{
	char	   *typname;
	Oid			typoid;
	PLpgSQL_type_type ttype;
	int16		typlen;
	bool		typbyval;
	char		typtype;
	Oid			collation;
	bool		typisarray;
	int32		atttypmod;
	TypeName   *origtypname;
	TypeCacheEntry *tcache;
	uint64		tupdesc_id;
} PLpgSQL_type;

入力と出力

本構造体は、plpgsql_build_variable 等の関数を通じて生成・参照されます。入力として PostgreSQL のシステムカタログ（pg_type）から取得した情報を受け取り、PL/pgSQL 内部で利用可能な形式に変換して保持します。

前提条件と不変条件

  • typoid は有効な PostgreSQL の型 OID である必要があります。
  • origtypname、tcache、tupdesc_id の各フィールドは、名前付き複合型（Named Composite Types）でのみ使用され、RECORD 型では使用されません。

アルゴリズムまたは振る舞い

  • コンパイルフェーズにおいて、型情報を pg_type から抽出し、PLpgSQL_type 構造体にコピーします。
  • 複合型の場合、tcache を介して型キャッシュエントリを保持し、tupdesc_id によってタプル記述子の変更を追跡します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**: src/pl/plpgsql/src/pl_comp.c 内の build_datatype、plpgsql_build_variable 等の関数が本構造体を生成・操作します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • PLpgSQL_type_type: 型の分類を示す列挙型。
  • TypeCacheEntry: PostgreSQL の型キャッシュ構造体。
  • TypeName: ユーザーが指定した型名を保持する構造体。

失敗モードと境界条件

  • RECORD 型として扱われる場合、複合型特有のフィールド（origtypname 等）は初期化されないか、無効な値となります。
  • tupdesc_id が不一致の場合、複合型の定義が変更されたことを示唆し、再評価が必要となる場合があります。

保守担当者向けメモ

  • 本構造体は plpgsql.h で定義されており、PL/pgSQL のコンパイルおよび実行の基盤となります。
  • フィールドの追加や変更を行う際は、pl_comp.c における型構築ロジックとの整合性に注意してください。
