scram_build_secret

定義

  • 種別: function
  • 定義箇所: src/common/scram-common.c:208

目的

SCRAM認証に必要な各キー（StoredKey, ServerKey）を導出し、データベース保存用の標準フォーマット文字列（SCRAM-SHA-256$iterations:salt$stored_key:server_key）を生成する。

シグネチャまたは形状

char *scram_build_secret(pg_cryptohash_type hash_type, int key_length,
                         const uint8 *salt, int saltlen, int iterations,
                         const char *password, const char **errstr)

入力と出力

  • **入力**:
  • hash_type: 使用するハッシュアルゴリズム（現在は PG_SHA256 のみサポート）。
  • key_length: キーの長さ。
  • salt, saltlen: ソルト値とその長さ。
  • iterations: PBKDF2の反復回数。
  • password: 正規化済みのパスワード文字列。
  • errstr: エラー発生時に詳細メッセージを格納するポインタ。
  • **出力**:
  • 成功時: 構築された認証文字列（呼び出し元で解放が必要）。
  • 失敗時: NULL を返し、errstr にエラー理由を設定。

前提条件と不変条件

  • hash_type は PG_SHA256 であること。
  • iterations は 0 より大きいこと。
  • 入力パスワードは事前にSASLprepで正規化されていること。
  • 戻り値のメモリは、フロントエンドでは malloc、バックエンドでは palloc で確保される。

アルゴリズムまたは振る舞い

1. scram_SaltedPassword を呼び出し、パスワードからソルト付きハッシュを生成する。 2. scram_ClientKey および scram_ServerKey を用いて各キーを導出する。 3. scram_H を用いて StoredKey をハッシュ化する。 4. 各バイナリデータを pg_b64_encode でBase64形式に変換する。 5. 指定されたフォーマットに従い、各要素を $ および : で連結した文字列を構築する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth-scram.c:506
  • src/interfaces/libpq/fe-auth-scram.c:945
  • **呼び出し先**:
  • scram_SaltedPassword
  • scram_ClientKey
  • scram_H
  • scram_ServerKey
  • pg_b64_encode

関連データ構造

  • pg_cryptohash_type: ハッシュアルゴリズムの識別子。
  • SCRAM_MAX_KEY_LEN: キー計算に使用されるバッファの最大長。

失敗モードと境界条件

  • メモリ確保失敗時: errstr に "out of memory" を設定し NULL を返す。
  • 暗号計算失敗時: 各内部関数がエラーを返し、errstr が設定される。バックエンド環境では elog(ERROR, ...) が発生する。
  • Base64エンコード失敗時: errstr にエラー理由を設定し、確保したメモリを解放して NULL を返す。

保守担当者向けメモ

  • 本関数はフロントエンドとバックエンドの両方から利用されるため、メモリ管理（malloc vs palloc）がマクロで切り替えられている点に注意すること。
  • 戻り値の文字列は呼び出し元で解放する責任がある。
