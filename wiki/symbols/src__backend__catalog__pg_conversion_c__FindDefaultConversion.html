FindDefaultConversion

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_conversion.c:151

目的

指定された名前空間およびエンコーディングペア（変換元・変換先）に対して、デフォルトとして設定されている変換関数のOIDを検索・取得します。

シグネチャまたは形状

Oid FindDefaultConversion(Oid name_space, int32 for_encoding, int32 to_encoding)

入力と出力

  • **入力**:
  • name_space (Oid): 検索対象の名前空間。
  • for_encoding (int32): 変換元のエンコーディングID。
  • to_encoding (int32): 変換先のエンコーディングID。
  • **出力**:
  • Oid: 見つかったデフォルト変換関数のプロシージャOID。該当するものがない場合は InvalidOid を返します。

前提条件と不変条件

  • CONDEFAULT システムキャッシュが利用可能である必要があります。
  • 戻り値は変換規則（pg_conversion）のOIDではなく、変換を実行するプロシージャ（pg_proc）のOIDです。

アルゴリズムまたは振る舞い

1. SearchSysCacheList3 を使用し、CONDEFAULT キャッシュから name_space, for_encoding, to_encoding をキーとして変換規則のリストを取得します。 2. 取得したリストを走査し、condefault フラグが真であるタプルを特定します。 3. 該当タプルが見つかった場合、そのタプル内の conproc（プロシージャOID）を保持します。 4. ReleaseSysCacheList でキャッシュリストを解放し、特定したプロシージャOIDを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/namespace.c:4097
  • src/backend/catalog/pg_conversion.c:72
  • **呼び出し先**:
  • SearchSysCacheList3 (システムキャッシュ検索)
  • ReleaseSysCacheList (キャッシュ解放)
  • GETSTRUCT (タプル構造体へのアクセス)

関連データ構造

  • Form_pg_conversion: pg_conversion システムカタログのタプル形式。
  • CatCList: SearchSysCacheList3 が返すキャッシュリスト構造体。

失敗モードと境界条件

  • 指定された条件に一致するデフォルト変換が存在しない場合、InvalidOid を返します。
  • CONDEFAULT キャッシュの検索キー（名前空間、エンコーディングID）が不正確な場合、意図した変換関数が取得できません。

保守担当者向けメモ

  • 本関数は変換規則のOIDではなく、実行用のプロシージャOIDを返す点に注意してください。
  • CONDEFAULT キャッシュは connamespace, conforencoding, contoencoding の3つのキーで構成されています。キャッシュの定義変更時には本関数の検索ロジックも確認が必要です。
