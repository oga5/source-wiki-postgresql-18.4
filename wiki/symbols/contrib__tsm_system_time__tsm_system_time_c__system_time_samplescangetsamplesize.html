system_time_samplescangetsamplesize

定義

  • 種別: function
  • 定義箇所: contrib/tsm_system_time/tsm_system_time.c:104

目的

TABLESAMPLE SYSTEM_TIME を使用したクエリのプランニング段階において、指定された時間制限内にスキャン可能なページ数とタプル数を概算します。

シグネチャまたは形状

static void system_time_samplescangetsamplesize(
    PlannerInfo *root,
    RelOptInfo *baserel,
    List *paramexprs,
    BlockNumber *pages,
    double *tuples
)

入力と出力

  • **入力**:
  • root: プランナのコンテキスト情報。
  • baserel: 対象リレーションの統計情報。
  • paramexprs: TABLESAMPLE 句で指定された時間制限（ミリ秒）の式リスト。
  • **出力**:
  • pages: 推定されるスキャン対象ページ数（*pages に格納）。
  • tuples: 推定されるスキャン対象タプル数（*tuples に格納）。

前提条件と不変条件

  • paramexprs の先頭要素は、時間制限を表す式である必要があります。
  • baserel には有効なリレーション統計（ページ数、タプル数）が含まれていることが期待されます。

アルゴリズムまたは振る舞い

1. **時間制限の抽出**: paramexprs から時間制限（ミリ秒）を評価します。評価不能または無効な値の場合はデフォルトの 1000ms を使用します。 2. **コスト見積もり**: get_tablespace_page_costs を用いて取得した spc_random_page_cost を基準に、指定時間をページ数に換算します。 3. **ページ数の算出**: 換算したページ数をリレーションの総ページ数でクランプ（制限）します。 4. **タプル数の算出**: リレーションのタプル密度（タプル数/ページ数）に基づき、推定ページ数からタプル数を算出します。データが不足している場合は 1 ページあたり 1 タプルと仮定し、最終的にリレーションの総タプル数でクランプします。

呼び出し元と呼び出し先

  • **呼び出し元**: tsm_system_time_handler 内で TsmRoutine 構造体の SampleScanGetSampleSize フィールドに割り当てられ、プランナから呼び出されます。
  • **呼び出し先**: estimate_expression_value（式評価用）、get_tablespace_page_costs（コスト取得用）、clamp_row_est（値の範囲制限用）。

関連データ構造

  • PlannerInfo: プランナのグローバル情報。
  • RelOptInfo: リレーションの統計情報。
  • TsmRoutine: テーブルサンプリング API のルーチン定義。

失敗モードと境界条件

  • 時間制限が負数または NaN の場合、デフォルト値として 1000ms が使用されます。
  • spc_random_page_cost が 0 以下の場合は、コスト計算をスキップし、時間制限値をそのままページ数として扱います。
  • リレーションの統計情報（baserel->tuples や pages）が 0 の場合、タプル数は推定ページ数と同値として扱われます。

保守担当者向けメモ

  • 本関数によるコスト計算は、ページコストをミリ秒に換算するというヒューリスティックな手法に基づいています。これは厳密な予測ではなく、あくまでプランナに対する概算の提供を目的としています。
