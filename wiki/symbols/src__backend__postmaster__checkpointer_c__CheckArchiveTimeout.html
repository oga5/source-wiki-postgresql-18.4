CheckArchiveTimeout

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/checkpointer.c:683

目的

archive_timeout 設定に基づき、WALセグメントの強制切り替えが必要かどうかを判定し、必要に応じて実行する。これにより、アイドル状態のデータベースでも定期的にWALアーカイブが進行するようにする。

シグネチャまたは形状

static void CheckArchiveTimeout(void)

入力と出力

  • 入力: なし（グローバル変数 XLogArchiveTimeout、last_xlog_switch_time、およびシステム時刻を参照）
  • 出力: なし（副作用としてWALセグメントの切り替え要求を行う）

前提条件と不変条件

  • XLogArchiveTimeout > 0 であること。
  • リカバリ中（RecoveryInProgress()）ではないこと。
  • システムがアイドル状態であっても、前回の切り替えから一定時間が経過していれば切り替えを試みる。

アルゴリズムまたは振る舞い

1. XLogArchiveTimeout が無効またはリカバリ中の場合は即座に終了する。 2. 現在時刻と last_xlog_switch_time を比較し、タイムアウトに達していない場合は終了する。 3. GetLastSegSwitchData を呼び出し、最新のセグメント切り替え情報を取得して last_xlog_switch_time を更新する。 4. 再度タイムアウト判定を行い、条件を満たす場合：

  • GetLastImportantRecPtr() が前回の切り替え位置より進んでいる（重要なWALが記録されている）か確認する。
  • 条件を満たす場合、RequestXLogSwitch(true) を呼び出し、チェックポイントを誘発しない形式でWALセグメントの切り替えを要求する。

    • 5. 処理の最後に last_xlog_switch_time を現在時刻で更新し、アイドル時の過剰な再試行を防止する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/checkpointer.c 内のメインループ（行550および801付近）。
  • 呼び出し先: RecoveryInProgress, GetLastSegSwitchData, GetLastImportantRecPtr, RequestXLogSwitch, XLogSegmentOffset, elog。

関連データ構造

  • XLogRecPtr: WAL上の位置を指すポインタ。
  • pg_time_t: 時刻管理に使用。

失敗モードと境界条件

  • XLogArchiveTimeout が 0 以下の場合、機能は無効化される。
  • リカバリ中は実行されない。
  • 重要なWALが記録されていない場合、タイムアウトしてもセグメント切り替えは行われない。
  • 切り替え要求の結果がセグメント境界と一致する場合、切り替えは行われなかったものとみなされる。

保守担当者向けメモ

  • 本関数はチェックポインターのメインループから定期的に呼び出される。
  • last_xlog_switch_time は、切り替えが実行された時刻だけでなく、切り替えが「要求された」時刻も保持する。
  • RequestXLogSwitch(true) の引数 true は、チェックポイントを誘発させないためのフラグである。
