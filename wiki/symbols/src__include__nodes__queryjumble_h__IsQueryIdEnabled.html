IsQueryIdEnabled

定義

  • 種別: static inline function
  • 定義箇所: src/include/nodes/queryjumble.h:103

目的

現在のシステム設定に基づき、クエリ識別子（Query ID）の計算処理を有効化すべきかどうかを判定する。

シグネチャまたは形状

static inline bool IsQueryIdEnabled(void)

入力と出力

  • 入力: なし（グローバル変数 compute_query_id および query_id_enabled を参照）
  • 出力: bool 型。Query ID の計算が有効な場合は true、無効な場合は false を返す。

前提条件と不変条件

  • compute_query_id GUC パラメータが COMPUTE_QUERY_ID_OFF の場合は常に false を返す。
  • compute_query_id GUC パラメータが COMPUTE_QUERY_ID_ON の場合は常に true を返す。

アルゴリズムまたは振る舞い

1. compute_query_id が COMPUTE_QUERY_ID_OFF であれば false を返す。 2. compute_query_id が COMPUTE_QUERY_ID_ON であれば true を返す。 3. 上記以外（AUTO 等）の場合、query_id_enabled 変数の値を返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/commands/createas.c
  • src/backend/commands/explain.c
  • src/backend/commands/portalcmds.c
  • src/backend/nodes/queryjumblefuncs.c
  • src/backend/parser/analyze.c
  • 呼び出し先: なし（関数呼び出しを含まないインライン関数）

関連データ構造

  • ComputeQueryIdType（列挙型）: compute_query_id の設定値を定義。

失敗モードと境界条件

  • 本関数自体は失敗しない。
  • compute_query_id が AUTO に設定されている場合、query_id_enabled の状態に依存するため、実行時の環境設定に結果が左右される。

保守担当者向けメモ

  • 頻繁に呼び出されるため、インライン関数として定義されている。
  • pg_stat_statements 等の拡張機能がクエリのフィンガープリントを必要とする際、直接的なフラグチェックの代わりに使用すること。
