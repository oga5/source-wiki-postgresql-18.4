pthread_key_t

定義

  • typedef ULONG pthread_key_t;
  • 定義箇所: src/port/pthread-win32.h:7

目的

Windows 環境において、POSIX スレッド固有データ（TSD: Thread-Specific Data）のキーを識別するための型定義です。

シグネチャまたは形状

ULONG 型のエイリアスです。

入力と出力

  • 入力: pthread_setspecific および pthread_getspecific 関数の引数として使用されます。
  • 出力: 該当なし（型定義のため）。

前提条件と不変条件

  • Windows 上での移植性維持を目的としています。
  • 現在の実装では、TSD 機能はスタブ化されており、完全な POSIX 準拠の動作は保証されません。

アルゴリズムまたは振る舞い

  • 本シンボル自体はデータ型の定義であり、ロジックを持ちません。
  • 関連する操作（pthread_setspecific, pthread_getspecific）は src/interfaces/libpq/pthread-win32.c に実装されていますが、現時点では空の実装（スタブ）です。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/interfaces/ecpg/ecpglib/connect.c: actual_connection_key
  • src/interfaces/ecpg/ecpglib/descriptor.c: descriptor_key
  • src/interfaces/ecpg/ecpglib/memory.c: auto_mem_key
  • src/interfaces/ecpg/ecpglib/misc.c: sqlca_key
  • 呼び出し先:
  • pthread_setspecific および pthread_getspecific（src/port/pthread-win32.h で宣言）が本型を引数として受け取ります。

関連データ構造

  • pthread_mutex_t（同ヘッダー内で定義される同期プリミティブ）

失敗モードと境界条件

  • 本型を利用する TSD 操作はスタブであるため、Windows 上でスレッド固有データを正しく保持・取得できない可能性があります。

保守担当者向けメモ

  • 本シンボルは Windows 環境専用の互換レイヤーです。
  • pthread_setspecific や pthread_getspecific を利用する際は、Windows 上での動作制限について留意してください。
