bpchar_sortsupport

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varchar.c:929

目的

bpchar（char(n)）型のソート処理を最適化するためのサポート関数を提供します。

シグネチャまたは形状

Datum bpchar_sortsupport(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して SortSupport 構造体へのポインタを受け取ります。
  • **出力**: void（PG_RETURN_VOID() を使用）。SortSupport 構造体の内部状態を更新することでソートの最適化設定を行います。

前提条件と不変条件

  • ssup->ssup_cxt（メモリコンテキスト）が有効である必要があります。
  • ssup->ssup_collation を介して、比較に使用するロケール情報が提供されている必要があります。

アルゴリズムまたは振る舞い

1. ssup->ssup_cxt にメモリコンテキストを切り替えます。 2. 汎用的な文字列ソートサポート関数である varstr_sortsupport を呼び出します。 3. その際、型識別子として BPCHAROID を渡し、bpchar 型特有の比較ルール（末尾空白の無視など）を適用させます。 4. 元のメモリコンテキストに戻し、処理を終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソート処理を行う PostgreSQL の実行エンジン（直接の呼び出し元情報はソースコード断片からは特定できません）。
  • **呼び出し先**: varstr_sortsupport（src/backend/utils/adt/varlena.c に定義されていると推測される汎用関数）。

関連データ構造

  • SortSupport: ソートの最適化情報を保持する構造体。
  • BPCHAROID: bpchar 型を識別するための OID。

失敗モードと境界条件

  • メモリコンテキストの切り替えに失敗した場合、システムエラーが発生する可能性があります。
  • 提供された SortSupport ポインタが NULL の場合、未定義の動作となります。

保守担当者向けメモ

  • 本関数は varstr_sortsupport への単なるラッパーです。bpchar 型の比較ロジックに変更を加える場合は、varstr_sortsupport 側の実装を確認してください。
