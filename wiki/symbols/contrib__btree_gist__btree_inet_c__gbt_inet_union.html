gbt_inet_union

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_inet.c:143

目的

GiST インデックス構築時、複数の inet 型キーを包含する最小の境界（Union）を計算する。

シグネチャまたは形状

Datum gbt_inet_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GistEntryVector *（結合対象のキー集合）。
  • PG_GETARG_POINTER(1): int *（結果のサイズを格納するポインタ）。
  • **出力**:
  • 戻り値: Datum（計算された inetKEY 構造体へのポインタ）。
  • 副作用: 第2引数に sizeof(inetKEY) を書き込む。

前提条件と不変条件

  • tinfo（gbtree_ninfo 構造体）がモジュール内で定義されており、inet 型の比較ロジックを提供していること。
  • inetKEY 構造体がメモリ上に確保可能であること。

アルゴリズムまたは振る舞い

1. palloc を使用して inetKEY サイズのメモリ領域を確保する。 2. 第2引数に sizeof(inetKEY) を設定する。 3. btree_utils_num.c で定義された汎用関数 gbt_num_union を呼び出し、entryvec 内の全キーを包含する最小の inetKEY を計算して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス構築プロセス（直接的な呼び出し元コードはソース抜粋に含まれていない）。
  • **呼び出し先**: gbt_num_union（btree_utils_num.c 内の汎用関数）。

関連データ構造

  • inetKEY: インデックス内で保持される lower と upper の境界値を持つ構造体。
  • GistEntryVector: 結合対象のキーを保持する GiST の標準構造体。
  • tinfo: inet 型の比較ロジックを定義した gbtree_ninfo 構造体。

失敗モードと境界条件

  • メモリ不足時に palloc がエラーを発生させる可能性がある。
  • entryvec が空の場合の挙動は gbt_num_union の実装に依存する。

保守担当者向けメモ

  • 本関数は btree_utils_num.c の汎用ロジックをラップする薄いインターフェースである。
  • inet 型特有の複雑な処理は tinfo を通じて委譲されているため、ロジックの変更が必要な場合は tinfo の定義を確認すること。
