ExplainState

定義

  • 種別: struct
  • 定義箇所: src/include/commands/explain_state.h:44

目的

EXPLAIN コマンドの実行中、クエリプランの走査から結果の整形・出力に至るまでの状態を保持・管理するためのメイン構造体です。

シグネチャまたは形状

typedef struct ExplainState {
    StringInfo  str;
    bool        verbose, analyze, costs, buffers, wal, timing, summary, memory, settings, generic;
    ExplainSerializeOption serialize;
    ExplainFormat format;
    int         indent;
    List       *grouping_stack;
    PlannedStmt *pstmt;
    List       *rtable;
    List       *rtable_names;
    List       *deparse_cxt;
    Bitmapset  *printed_subplans;
    bool        hide_workers;
    int         rtable_size;
    ExplainWorkersState *workers_state;
    void      **extension_state;
    int         extension_state_allocated;
} ExplainState;

入力と出力

  • **入力**: EXPLAIN コマンドのオプション（ANALYZE, VERBOSE 等）がフラグとして設定されます。
  • **出力**: StringInfo 型の str フィールドに、整形されたクエリプランの出力結果が蓄積されます。

前提条件と不変条件

  • str フィールドは StringInfo で管理されており、メモリリークを防ぐため適切なライフサイクル管理が必要です。
  • indent や grouping_stack などのフォーマット用状態は、プランツリーの走査中に関数間で共有・更新されます。

アルゴリズムまたは振る舞い

1. **初期化**: NewExplainState() によりデフォルト値で生成されます。 2. **オプション適用**: ParseExplainOptionList() 等によりフラグが更新されます。 3. **プラン走査**: ExplainPrintPlan 等の実行中、プランツリーの走査に伴いインデントレベルや出力バッファが更新されます。 4. **拡張**: extension_state 配列を通じて、拡張機能が独自のオプションや状態を動的に保持・参照可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/auto_explain/auto_explain.c や contrib/pg_overexplain/pg_overexplain.c など、拡張機能やコマンド実行エンジンから参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すものではありませんが、ExplainPrintPlan 等の関数に引数として渡され、状態が参照・更新されます。

関連データ構造

  • StringInfo: 出力バッファ。
  • ExplainFormat: 出力形式（TEXT, JSON等）。
  • ExplainWorkersState: 並列クエリ実行時のワーカーごとの状態。
  • PlannedStmt: 実行対象のプランツリー。

失敗モードと境界条件

  • extension_state の管理において、割り当てられたサイズを超えたアクセスを行わないよう注意が必要です。
  • 並列クエリ実行時には workers_state が適切に設定されている必要があります。

保守担当者向けメモ

  • ExplainState はプランツリーの走査中に再帰的に渡されるため、関数間で共有される重要なコンテキストです。
  • 独自の EXPLAIN オプションを追加する場合は、extension_state 配列と関連アクセサ（GetExplainExtensionState 等）を利用してください。
