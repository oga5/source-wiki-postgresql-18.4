DumpSignalInformation

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/parallel.c:165

目的

並列処理実行中におけるシグナルハンドリングの状態を保持し、割り込み発生時に適切なキャンセル処理を行うための情報を管理する。

シグネチャまたは形状

typedef struct DumpSignalInformation
{
	ArchiveHandle *myAH;		/* キャンセルを発行するデータベース接続 */
	ParallelState *pstate;		/* 並列実行状態（存在する場合） */
	bool		handler_set;	/* プロセス内でシグナルハンドラが設定済みか */
#ifndef WIN32
	bool		am_worker;		/* ワーカープロセスであるか */
#endif
} DumpSignalInformation;

入力と出力

本構造体は関数引数として渡されるものではなく、src/bin/pg_dump/parallel.c 内で static volatile DumpSignalInformation signal_info; として静的に定義され、シグナルハンドラとメイン処理間で共有されるグローバルな状態として機能する。

前提条件と不変条件

  • handler_set は、シグナルハンドラが正常に登録された場合に true となる。
  • WIN32 環境以外では、am_worker フラグにより自身がリーダーかワーカーかを識別する。

アルゴリズムまたは振る舞い

  • ユーザーによる割り込み（Ctrl+C等）が発生した際、シグナルハンドラはこの構造体を参照する。
  • myAH を通じてデータベース接続のキャンセルを試み、pstate が存在する場合は並列実行中の全ワーカーに対して終了要求を伝播させる。

呼び出し元と呼び出し先

  • 呼び出し元: シグナルハンドラ（OSから呼び出される）および並列処理の初期化・終了処理を行う関数群。
  • 呼び出し先: ArchiveHandle が保持するキャンセル用関数や、ParallelState を操作する並列処理関数。
  • 注記: 具体的な関数呼び出しの連鎖については、提供されたソース excerpt には含まれていない。

関連データ構造

  • ArchiveHandle: データベース接続およびアーカイブのメタデータを保持。
  • ParallelState: 並列実行全体の管理構造体。

失敗モードと境界条件

  • シグナルハンドラ内での実行であるため、非同期シグナル安全（async-signal-safe）な操作のみが許容される。
  • myAH や pstate が初期化される前にシグナルが発生した場合の挙動については、本構造体の定義のみからは判断できない。

保守担当者向けメモ

  • 本構造体は volatile 修飾された静的変数として使用されるため、シグナルハンドラ内でのアクセスが競合しないよう設計されている。
  • Windows環境とそれ以外で構造体のメンバが異なるため、プラットフォーム固有の条件付きコンパイルに注意すること。
