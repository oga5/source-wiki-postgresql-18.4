NIImportAffixes

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/spell.c:1436

目的

ISpell または Hunspell 形式の接辞（affix）ファイルを読み込み、辞書構造体 IspellDict にルールを登録する。

シグネチャまたは形状

void NIImportAffixes(IspellDict *Conf, const char *filename)

入力と出力

  • **入力**:
  • Conf: 接辞ルールを格納する IspellDict 構造体へのポインタ。
  • filename: 読み込む接辞ファイル（.affix）のパス。
  • **出力**: なし（Conf が更新される）。エラー時は ereport(ERROR, ...) により中断される。

前提条件と不変条件

  • Conf は初期化済みであること。
  • ファイル内の接辞定義形式（旧来の ISpell 形式と新しい Hunspell 形式）が混在している場合、エラーとなる。

アルゴリズムまたは振る舞い

1. tsearch_readline_begin でファイルを開く。 2. Conf のフラグ設定を初期化する。 3. ファイルを一行ずつ読み込み、小文字化して解析する。

  • compoundwords, suffixes, prefixes, flag などのキーワードを検出し、旧形式のルールをパースする。
  • parse_affentry を使用して接辞エントリを抽出し、NIAddAffix で Conf に追加する。

    • 4. 新形式（COMPOUNDFLAG, PFX, SFX 等）のキーワードが検出された場合、isnewformat ラベルへ遷移し、NIImportOOAffixes を呼び出して処理を委譲する。 5. 処理終了後、tsearch_readline_end でファイルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/dict_ispell.c:65
  • **呼び出し先**:
  • tsearch_readline_begin, tsearch_readline, tsearch_readline_end
  • str_tolower, findchar2, addCompoundAffixFlagValue, parse_affentry, NIAddAffix
  • NIImportOOAffixes

関連データ構造

  • IspellDict: 辞書の状態と接辞ルールを保持するメイン構造体。

失敗モードと境界条件

  • ファイルが開けない場合、ERROR を発生させる。
  • 旧形式と新形式のコマンドが同一ファイル内に混在している場合、ERROR を発生させる。
  • メモリ不足時は palloc 等の内部関数によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は ISpell 形式と Hunspell 形式の自動判別を行う。
  • NIImportOOAffixes は Hunspell 形式の処理を担当する。
  • pfree によるメモリ解放がループ内で行われているため、読み込み処理の変更時はメモリリークに注意すること。
