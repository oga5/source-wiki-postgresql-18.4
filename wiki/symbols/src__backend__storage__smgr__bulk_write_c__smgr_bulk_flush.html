smgr_bulk_flush

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/bulk_write.c:241

目的

BulkWriteState 内に保持されている保留中の書き込みキューを物理ストレージへフラッシュし、必要に応じて WAL への記録およびチェックサムの計算を行う。

シグネチャまたは形状

static void smgr_bulk_flush(BulkWriteState *bulkstate)

入力と出力

  • **入力**: BulkWriteState *bulkstate（書き込み状態を保持する構造体）
  • **出力**: なし（副作用としてストレージへの書き込みおよび bulkstate->npending のリセットを行う）

前提条件と不変条件

  • bulkstate は有効なポインタであること。
  • 書き込み対象のページデータは bulkstate->pending_writes に格納されていること。
  • 呼び出し後、bulkstate->npending は 0 にリセットされる。

アルゴリズムまたは振る舞い

1. **ソート**: 保留中の書き込みが複数ある場合、buffer_cmp を用いてブロック番号順にソートする。 2. **WAL 記録**: bulkstate->use_wal が真の場合、キュー内の全ページを log_newpages を用いて WAL に記録する。 3. **チェックサム計算**: 各ページに対して PageSetChecksumInplace を実行する。 4. **物理書き込み**:

  • ブロック番号が現在のリレーションサイズ (bulkstate->relsize) を超える場合、その間の空き領域をゼロ埋め（smgrextend）してから対象ページを書き込む。
  • 既存領域内の場合は smgrwrite を使用する。

    • 5. **メモリ解放**: 書き込み完了後、各ページバッファを pfree で解放する。 6. **状態更新**: bulkstate->npending を 0 に戻す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/smgr/bulk_write.c 内の smgr_bulk_write（キュー溢れ時）および smgr_bulk_finish（終了時）。
  • **呼び出し先**:
  • qsort, buffer_cmp
  • log_newpages
  • PageSetChecksumInplace
  • smgrextend, smgrwrite
  • pfree

関連データ構造

  • BulkWriteState: 書き込み操作の全体状態を管理。
  • PendingWrite: 個別の書き込み要求（ブロック番号とバッファデータ）。

失敗モードと境界条件

  • npending == 0 の場合は何もせず即座に復帰する。
  • 非連続的なブロック書き込みが発生した場合、ファイルシステムの断片化を防ぐためにゼロ埋め処理が実行される。
  • log_newpages 呼び出し時、標準外のページレイアウトが混在している場合は、すべて非標準としてログ記録される。

保守担当者向けメモ

  • 本関数は BulkWriteState が保持するバッファの所有権を消費（pfree）するため、呼び出し後にバッファへアクセスしてはならない。
  • smgrextend によるゼロ埋め処理は、論理的には必須ではないが、ファイルシステムの断片化抑制を目的としている。
