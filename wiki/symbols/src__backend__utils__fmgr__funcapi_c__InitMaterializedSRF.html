InitMaterializedSRF

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/funcapi.c:75

目的

マテリアライズドモード（Tuplestore）で結果を返す集合を返す関数（SRF）の初期化を行います。呼び出し側の ReturnSetInfo 構造体を設定し、結果格納用のタプルストアを準備します。

シグネチャまたは形状

void InitMaterializedSRF(FunctionCallInfo fcinfo, bits32 flags)

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報。resultinfo フィールドに ReturnSetInfo が格納されている必要があります。
  • flags: 動作フラグ。MAT_SRF_USE_EXPECTED_DESC（期待される記述子の使用）や MAT_SRF_BLESS（タプル記述子の祝福）を指定可能。
  • **出力**: なし（副作用として fcinfo->resultinfo を更新）。

前提条件と不変条件

  • fcinfo->resultinfo が ReturnSetInfo 型であること。
  • 呼び出し側のコンテキストが SFRM_Materialize モードを許可していること。
  • MAT_SRF_USE_EXPECTED_DESC フラグ使用時は、rsinfo->expectedDesc が非 NULL であること。

アルゴリズムまたは振る舞い

1. **コンテキスト検証**: rsinfo が有効か、および SFRM_Materialize が許可されているかを確認し、不正な場合はエラーを報告します。 2. **メモリ管理**: rsinfo->econtext->ecxt_per_query_memory に切り替え、永続的なメモリ領域でタプルストアを構築します。 3. **タプル記述子の生成**: flags に応じて、expectedDesc をコピーするか、get_call_result_type を用いて戻り値の型から動的に構築します。 4. **祝福（Blessing）**: MAT_SRF_BLESS 指定時は BlessTupleDesc を実行します。 5. **タプルストアの初期化**: tuplestore_begin_heap を呼び出し、rsinfo の returnMode、setResult、setDesc を設定します。 6. **復帰**: 元のメモリコンテキストに戻ります。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_heapam.c、contrib/dblink/dblink.c、contrib/pageinspect/ 各種、contrib/pg_buffercache/、contrib/pg_stat_statements/、contrib/pg_walinspect/、contrib/pgrowlocks/、contrib/postgres_fdw/ など、多くの拡張モジュールから呼び出されています。
  • **呼び出し先**: CreateTupleDescCopy、get_call_result_type、BlessTupleDesc、tuplestore_begin_heap、MemoryContextSwitchTo。

関連データ構造

  • FunctionCallInfo: 関数呼び出しのメタデータ。
  • ReturnSetInfo: SRFの実行状態を保持する構造体。
  • Tuplestorestate: マテリアライズされた結果を保持するストア。
  • TupleDesc: タプルの構造定義。

失敗モードと境界条件

  • rsinfo が NULL または ReturnSetInfo ではない場合: ERRCODE_FEATURE_NOT_SUPPORTED でエラー。
  • SFRM_Materialize が許可されていない場合: ERRCODE_FEATURE_NOT_SUPPORTED でエラー。
  • 戻り値が複合型ではない場合: get_call_result_type が TYPEFUNC_COMPOSITE 以外を返すとエラー。

保守担当者向けメモ

  • 本関数は per-query メモリコンテキストで動作するため、メモリリークを防ぐために MemoryContextSwitchTo の戻り値によるコンテキスト復帰が必須です。
  • get_call_result_type の呼び出しはコストが高いため、必要に応じてキャッシュ戦略を検討してください。
