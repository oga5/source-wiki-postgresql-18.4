variable

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/ecpglib/ecpglib_extern.h:135

目的

ECPG（Embedded SQL for C）において、ホスト変数とSQL変数間のマッピングおよびデータ転送情報を保持するために使用されます。

シグネチャまたは形状

struct variable
{
	enum ECPGttype type;
	void	   *value;
	void	   *pointer;
	long		varcharsize;
	long		arrsize;
	long		offset;
	enum ECPGttype ind_type;
	void	   *ind_value;
	void	   *ind_pointer;
	long		ind_varcharsize;
	long		ind_arrsize;
	long		ind_offset;
	struct variable *next;
};

入力と出力

  • **入力**: ECPGプリプロセッサによって生成されたホスト変数の型情報、値へのポインタ、およびインジケータ変数の情報。
  • **出力**: 連結リスト構造（next）を介して、SQL実行に必要な変数の集合を後続の処理（ecpg_execute 等）へ提供します。

前提条件と不変条件

  • next ポインタにより、複数の変数をリストとして管理します。
  • 各変数は、対応するインジケータ変数（ind_ で始まるメンバ）の情報を保持可能です。

アルゴリズムまたは振る舞い

  • ECPGランタイムライブラリにおいて、SQL文の実行時にホスト変数とデータベース間の型変換およびデータ転送を行う際のメタデータとして機能します。
  • type および ind_type には ECPGttype 列挙型が使用され、データの型を識別します。
  • value および pointer は、ホスト変数のメモリ上の位置や値を指し示します。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数情報は、提供されたソース excerpt には含まれていません。

関連データ構造

  • ECPGttype: 変数の型を定義する列挙型。
  • var_list: variable 構造体のリストを管理するための構造体（推測される関連構造）。

失敗モードと境界条件

  • value や pointer が NULL の場合、データ転送時に不正なメモリアクセスが発生する可能性があります。
  • arrsize や varcharsize が不適切な値である場合、バッファオーバーフローやデータ切り捨てのリスクがあります。

保守担当者向けメモ

  • 本構造体は ecpglib の内部インターフェースの一部です。
  • 構造体の変更は、プリプロセッサが生成するコードとの互換性に直結するため注意が必要です。
  • ツールが提示した contrib/ 配下の変数に関する言及は、本構造体とは無関係な一般的な用語としての「variable」であり、本シンボルとは別のものです。
