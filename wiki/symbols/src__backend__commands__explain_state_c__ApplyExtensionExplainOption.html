ApplyExtensionExplainOption

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_state.c:366

目的

EXPLAIN コマンドのオプション解析において、コア機能で認識されない未知のオプションを、登録済みの拡張機能（Extension）のハンドラへ委譲して処理させる。

シグネチャまたは形状

bool ApplyExtensionExplainOption(ExplainState *es, DefElem *opt, ParseState *pstate)

入力と出力

  • **入力**:
  • es: ExplainState 構造体へのポインタ。
  • opt: 解析対象の DefElem（オプション名と値のペア）。
  • pstate: 解析状態を保持する ParseState 構造体。
  • **出力**:
  • bool: 該当するオプション名を持つ拡張機能ハンドラが見つかり、実行された場合は true を返す。見つからなかった場合は false を返す。

前提条件と不変条件

  • ExplainExtensionOptionsAssigned が現在登録されている拡張オプションの数を保持していること。
  • ExplainExtensionOptionArray に有効なハンドラが格納されていること。

アルゴリズムまたは振る舞い

1. ExplainExtensionOptionsAssigned を上限として、ExplainExtensionOptionArray を順次走査する。 2. 各要素の option_name と、入力された opt->defname を strcmp で比較する。 3. 一致する名前が見つかった場合、その要素の option_handler を呼び出し、es, opt, pstate を渡す。 4. 実行後、直ちに true を返して終了する。 5. ループを完了しても一致するものがない場合、false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/explain_state.c 内のオプション解析処理（ParseExplainOptionList 等）から呼び出される。
  • **呼び出し先**: ExplainExtensionOptionArray に登録された各拡張機能の option_handler 関数。

関連データ構造

  • ExplainState: EXPLAIN の実行状態を保持する構造体。
  • DefElem: SQL コマンドのオプション定義を保持する構造体。
  • ExplainExtensionOption: 拡張機能のオプション名とハンドラを保持する構造体（ExplainExtensionOptionArray の要素型）。

失敗モードと境界条件

  • 登録されたオプションが存在しない場合、false を返す。
  • opt->defname が NULL の場合や、ExplainExtensionOptionArray が初期化されていない場合の挙動については、本ソース抜粋からは判断できない。

保守担当者向けメモ

  • 本関数は EXPLAIN の拡張性を提供するためのディスパッチャである。新しい拡張オプションを追加する場合は、RegisterExtensionExplainOption を通じて登録を行う必要がある。
  • 拡張機能側のハンドラ内で発生したエラーは、本関数を介して上位へ伝播する。
