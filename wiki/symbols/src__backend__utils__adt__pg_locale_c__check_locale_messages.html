check_locale_messages

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale.c:411

目的

GUC（Grand Unified Configuration）パラメータとして設定された lc_messages の値が、システム上で有効なロケール設定であるかを検証します。

シグネチャまたは形状

bool check_locale_messages(char **newval, void **extra, GucSource source)

入力と出力

  • **入力**:
  • newval: 設定しようとしているロケール名の文字列へのポインタ。
  • extra: 未使用。
  • source: 設定のソース（GucSource）。
  • **出力**:
  • bool: 検証に成功した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • newval が空文字列（""）の場合、source が PGC_S_DEFAULT であれば常に true を返します。
  • Windows 環境では、ロケール値の妥当性チェックを行わず常に true を返します。
  • LC_MESSAGES カテゴリが定義されているプラットフォーム（かつ Windows 以外）では、check_locale 関数を介して検証が行われます。

アルゴリズムまたは振る舞い

1. newval が空文字列かを確認します。空であれば、デフォルト設定時のみ true を返し、それ以外は false を返します。 2. プラットフォームが LC_MESSAGES をサポートし、かつ Windows でない場合、check_locale(LC_MESSAGES, *newval, NULL) を呼び出し、その結果を返します。 3. 上記以外の環境（Windows または LC_MESSAGES 未定義）では、検証をスキップして true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/guc_tables.c: GUC パラメータのフックとして登録されています。
  • **呼び出し先**:
  • check_locale (条件付き): ロケールの妥当性確認に使用されます。

関連データ構造

  • GucSource: 設定の由来を示す列挙型。

失敗モードと境界条件

  • newval が空文字列かつデフォルト設定以外の場合、検証失敗として false を返します。
  • LC_MESSAGES が定義されていない環境や Windows 環境では、値の妥当性に関わらず常に true を返すため、無効なロケール名が設定される可能性があります。

保守担当者向けメモ

  • 本関数は lc_messages GUC パラメータのバリデーションフックです。
  • LC_MESSAGES のサポート状況はプラットフォームに依存するため、移植性を考慮した実装となっています。
  • check_locale への依存があるため、ロケール検証ロジックの変更時には本関数の挙動も再確認してください。
