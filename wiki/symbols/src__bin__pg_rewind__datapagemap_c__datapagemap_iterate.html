datapagemap_iterate

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/datapagemap.c:74

目的

datapagemap_t 構造体で管理されている変更済みデータページのビットマップを走査するためのイテレータを生成します。

シグネチャまたは形状

datapagemap_iterator_t *datapagemap_iterate(datapagemap_t *map);

入力と出力

  • **入力**: map (変更済みブロックを保持する datapagemap_t へのポインタ)
  • **出力**: 初期化された datapagemap_iterator_t 構造体へのポインタ

前提条件と不変条件

  • map は有効な datapagemap_t 構造体を指している必要があります。
  • 戻り値として得られたイテレータは、呼び出し側で pg_free() を使用して明示的に解放する必要があります。

アルゴリズムまたは振る舞い

1. pg_malloc を使用して datapagemap_iterator_t 構造体分のメモリを確保します。 2. 確保したイテレータの map フィールドに引数の map を代入します。 3. 走査開始位置を示す nextblkno を 0 に初期化します。 4. 初期化済みのイテレータを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_rewind/filemap.c:531
  • src/bin/pg_rewind/pg_rewind.c:585
  • src/bin/pg_rewind/datapagemap.c:122
  • **呼び出し先**:
  • pg_malloc (メモリ確保)

関連データ構造

  • datapagemap_t: 変更済みブロックを保持するビットマップ構造体。
  • datapagemap_iterator_t: イテレーションの状態（対象マップと現在のブロック番号）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には pg_malloc がエラーを発生させ、処理が中断される可能性があります。

保守担当者向けメモ

  • 本関数で生成されたイテレータは、後続の datapagemap_next() 関数で走査に使用されます。
  • メモリリークを防ぐため、イテレータの解放忘れがないよう注意してください。
