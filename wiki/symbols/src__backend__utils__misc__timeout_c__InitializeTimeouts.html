InitializeTimeouts

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/timeout.c:469

目的

PostgreSQL バックエンドプロセスにおけるタイムアウト管理機能の初期化および再初期化を行う。

シグネチャまたは形状

void InitializeTimeouts(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として内部状態の初期化とシグナルハンドラの登録を行う）

前提条件と不変条件

  • 本関数は、タイムアウト管理機能を利用する前に呼び出される必要がある。
  • 呼び出し後、all_timeouts_initialized フラグが true に設定される。

アルゴリズムまたは振る舞い

1. disable_alarm() を呼び出し、既存のタイマー設定を無効化する。 2. アクティブなタイムアウト数を保持する num_active_timeouts を 0 にリセットする。 3. all_timeouts 配列を走査し、全要素のインデックス、アクティブ状態、ハンドラ、時間設定を初期値（無効状態）にリセットする。 4. all_timeouts_initialized を true に設定する。 5. pqsignal を使用して、SIGALRM シグナルに対するハンドラとして handle_sig_alarm を登録する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/bgworker.c
  • src/backend/postmaster/startup.c
  • src/backend/replication/logical/slotsync.c
  • src/backend/replication/walsender.c
  • src/backend/tcop/backend_startup.c
  • src/backend/tcop/postgres.c
  • 呼び出し先:
  • disable_alarm()
  • pqsignal()
  • handle_sig_alarm (関数ポインタとして登録)

関連データ構造

  • all_timeouts: タイムアウト設定を保持する静的配列。
  • num_active_timeouts: 現在有効なタイムアウトの数。
  • all_timeouts_initialized: 初期化完了を示すフラグ。

失敗モードと境界条件

  • 本関数自体はエラーを返さない。
  • pqsignal によるシグナルハンドラの登録が失敗した場合の挙動は、OS の sigaction 等の挙動に依存する。

保守担当者向けメモ

  • 本関数は再初期化にも対応しているため、プロセス内で複数回呼び出される可能性がある（例: backend_startup.c 内での再呼び出し）。
  • SIGALRM ハンドラとして登録される handle_sig_alarm は、シグナル安全性に配慮した実装が必要である。
