ObjectAccessDrop

定義

  • 種別: struct
  • 定義箇所: src/include/catalog/objectaccess.h:74

目的

SQLオブジェクト削除時、拡張機能に対して削除のコンテキスト情報を伝達するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
    int dropflags;
} ObjectAccessDrop;

入力と出力

  • **入力**: 削除操作時に dropflags が設定されます。
  • **出力**: 登録されたフック関数がこの構造体を参照し、削除の性質を判定します。

前提条件と不変条件

  • dropflags は src/include/catalog/dependency.h で定義される PERFORM_DELETION_* フラグと整合している必要があります。

アルゴリズムまたは振る舞い

1. コアコード（deleteOneObject 等）が削除処理を開始する際、ObjectAccessDrop 構造体を初期化します。 2. memset でゼロクリアされた後、適切な dropflags が格納されます。 3. InvokeObjectAccessHook マクロを通じて、登録済みの外部フック関数へこの構造体へのポインタが渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/objectaccess.c 内の関数がこの構造体をインスタンス化し、フックへ渡します。
  • **呼び出し先**: contrib/sepgsql/hooks.c および src/test/modules/test_oat_hooks/test_oat_hooks.c がこの構造体をキャストして受け取り、内容を参照します。

関連データ構造

  • dependency.h: dropflags の定義元である PERFORM_DELETION_* フラグ群。

失敗モードと境界条件

  • フックが登録されていない場合、本構造体は生成されず、呼び出しも行われません。
  • 構造体のポインタが NULL でないことを前提としてフック側でキャストが行われるため、呼び出し側の初期化漏れは未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は deleteOneObject() の引数をパッケージ化する役割を担っています。
  • 拡張機能側で利用する際は、必ず ObjectAccessDrop * へのキャストを行ってください。
