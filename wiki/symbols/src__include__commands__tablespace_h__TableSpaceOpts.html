TableSpaceOpts

定義

  • 種別: struct
  • 定義箇所: src/include/commands/tablespace.h:41

目的

テーブルスペースごとのI/Oコスト設定（random_page_cost、seq_page_cost、I/O並行性設定）を保持し、データベースのクエリプランナがストレージ性能を考慮した実行計画を立てるためのパラメータを提供します。

シグネチャまたは形状

typedef struct TableSpaceOpts
{
    int32       vl_len_;        /* varlena header */
    float8      random_page_cost;
    float8      seq_page_cost;
    int         effective_io_concurrency;
    int         maintenance_io_concurrency;
} TableSpaceOpts;

入力と出力

  • 本構造体は varlena ヘッダー（vl_len_）を含む可変長データ構造として扱われます。
  • 入力: SQLの CREATE TABLESPACE または ALTER TABLESPACE コマンドで指定されたオプション値。
  • 出力: クエリプランナが参照するテーブルスペース固有のコストパラメータ。

前提条件と不変条件

  • vl_len_ フィールドは varlena ヘッダーであるため、直接操作してはなりません。
  • メモリレイアウトは reloptions システムと整合している必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、テーブルスペースの物理的なI/O特性を定義します。
  • random_page_cost および seq_page_cost は、プランナがインデックススキャンとシーケンシャルスキャンのコストを比較する際に使用されます。
  • effective_io_concurrency および maintenance_io_concurrency は、非同期I/Oの並行度を制御するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/access/common/reloptions.c: reloptions システムにより、各フィールドがSQLオプションとマッピングされています。
  • src/backend/utils/cache/spccache.c: テーブルスペースキャッシュの管理において、本構造体へのポインタが保持されます。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • pg_tablespace: 本構造体の設定値が格納されるシステムカタログ。
  • reloptions: 本構造体のフィールドを定義・検証するためのフレームワーク。

失敗モードと境界条件

  • varlena ヘッダーの不適切な操作はメモリ破壊を引き起こす可能性があります。
  • 負の値や無効なコスト設定が指定された場合、reloptions の検証ロジックによりエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は varlena ヘッダーを含むため、メモリ割り当てやコピーを行う際は PostgreSQL の varlena 関連マクロを使用してください。
  • フィールドを追加・変更する場合は、src/backend/access/common/reloptions.c の定義も併せて更新する必要があります。
