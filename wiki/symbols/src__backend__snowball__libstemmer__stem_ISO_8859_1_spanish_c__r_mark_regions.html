r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_spanish.c:506

目的

単語内のステミング処理に必要な領域（RV, R1, R2）の境界位置を計算し、SN_env 構造体に記録する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • 入力: SN_env * z （ステミング環境。現在の単語データとカーソル位置 c を含む）
  • 出力: int （常に 1 を返す）
  • 副作用: z->I[0], z->I[1], z->I[2] に計算された境界オフセットが書き込まれる。

前提条件と不変条件

  • z は有効な SN_env 構造体を指していること。
  • g_v（母音グループ）が定義されていること。

アルゴリズムまたは振る舞い

1. 全ての領域境界（I[0], I[1], I[2]）を単語の末尾 z->l で初期化する。 2. **RV領域の計算**:

  • 母音の後に子音が続く場合、その子音の直後をRVの開始位置とする。
  • 母音の後に母音が続く場合、その母音の直後をRVの開始位置とする。
  • 上記に該当しない場合、単語の3文字目（インデックス2）をRVの開始位置とする。

    • 3. **R1, R2領域の計算**:

  • 母音と子音の遷移をスキャンし、最初の遷移の直後をR1、さらにその後の遷移の直後をR2として記録する。

    • 4. 処理中、カーソル位置 z->c は一時的に移動するが、終了時には呼び出し時の位置に復元される。

呼び出し元と呼び出し先

  • 呼び出し元: spanish_ISO_8859_1_stem（直接の言及はないが、モジュール内の制御フローより推測される）。
  • 呼び出し先: in_grouping, out_grouping（Snowballランタイムのユーティリティ）。
  • 他の言語用ステマー（basque, catalan, danish等）からも同名の関数が定義・利用されているが、本シンボルはスペイン語版の定義である。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。I 配列（整数型ワークスペース）を領域保持に使用。

失敗モードと境界条件

  • 単語が短く、母音・子音のパターンが見つからない場合、境界は初期値（単語末尾）のままとなる。
  • z->c が単語長を超えるような不正な状態は、in_grouping 等のガードにより回避される。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによる自動生成コードである。ロジックの変更が必要な場合は、対応する .sbl ソースファイルを修正すること。
  • I[0]〜I[2] のインデックスがどの領域に対応するかは、Snowballの内部仕様に依存する。
