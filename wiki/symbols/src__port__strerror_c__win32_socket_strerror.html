win32_socket_strerror

定義

  • 種別: function
  • 定義箇所: src/port/strerror.c:274

目的

Windows 環境において、Winsock エラーコードを人間が読める形式の文字列に変換する。

シグネチャまたは形状

static char *win32_socket_strerror(int errnum, char *buf, size_t buflen);

入力と出力

  • **入力**:
  • errnum: 変換対象の Winsock エラーコード。
  • buf: メッセージを格納するバッファへのポインタ。
  • buflen: buf のサイズ。
  • **出力**:
  • 変換されたエラーメッセージが格納された buf へのポインタを返す。

前提条件と不変条件

  • netmsg.dll を動的にロードしてエラーメッセージを取得する。
  • 内部で静的変数 handleDLL を保持し、初回呼び出し時にロードを行う。

アルゴリズムまたは振る舞い

1. handleDLL が未ロードの場合、LoadLibraryEx を用いて netmsg.dll をロードする。ロード失敗時はエラーメッセージを buf に書き込み即時復帰する。 2. ZeroMemory でバッファを初期化する。 3. FormatMessage を使用し、netmsg.dll から errnum に対応するエラー文字列を取得する。 4. FormatMessage が失敗した場合は、「unrecognized winsock error %d」という文字列を buf に書き込む。 5. 最終的に buf を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/strerror.c 内の関数（pg_strerror_r からの呼び出しが示唆されている）、および src/include/port/win32_port.h からの参照。
  • **呼び出し先**: Windows API (LoadLibraryEx, GetLastError, ZeroMemory, FormatMessage, MAKELANGID)。

関連データ構造

  • HANDLE: Windows の DLL ハンドル。

失敗モードと境界条件

  • netmsg.dll のロード失敗: GetLastError() を含むエラーメッセージを生成する。
  • FormatMessage による変換失敗: 「unrecognized winsock error %d」というフォールバックメッセージを生成する。

保守担当者向けメモ

  • 本関数は Windows 専用のロジックである。
  • netmsg.dll のロードは初回のみ行われる。
  • FormatMessage の引数 buflen - 1 により、バッファオーバーランを防止している。
