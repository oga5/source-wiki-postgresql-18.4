RT_NODE_48

定義

  • 種別: struct
  • 定義箇所: src/include/lib/radixtree.h:542

目的

適応型基数木（ART）において、子ノード数が中規模（最大48個）の際に使用されるノード構造体です。メモリ効率を最適化しつつ、高速な子ノード検索を実現します。

シグネチャまたは形状

typedef struct RT_NODE_48 {
    RT_NODE      base;
    bitmapword   isset[RT_BM_IDX(RT_FANOUT_48_MAX)];
    uint8        slot_idxs[RT_NODE_MAX_SLOTS];
    RT_PTR_ALLOC children[FLEXIBLE_ARRAY_MEMBER];
} RT_NODE_48;

入力と出力

  • **入力**: 検索対象のキー（チャンク）を引数として、RT_NODE_48_GET_CHILD 等のマクロを介して子ノードポインタを取得します。
  • **出力**: children 配列内の RT_PTR_ALLOC 型ポインタ（または共有メモリ上のハンドル）。

前提条件と不変条件

  • slot_idxs は RT_NODE_MAX_SLOTS のサイズを持ち、子ノードへのインデックスを保持します。
  • children はフレキシブル配列メンバーであり、ノードのサイズクラスに応じて動的に確保されます。
  • RT_INVALID_SLOT_IDX (0xFF) は、スロットが割り当てられていないことを示す不変の値として使用されます。

アルゴリズムまたは振る舞い

  • **検索**: isset ビットマップでチャンクの存在を確認し、slot_idxs をルックアップテーブルとして使用して children 配列内のインデックスを特定します。
  • **初期化**: memset を使用して slot_idxs 以前の固定サイズメンバーをゼロ初期化する操作が確認されています。
  • **動的管理**: ノードが満杯になった際や空きができた際に、他のノード種別（RT_NODE_16 や RT_NODE_256）との間で変換が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: 基数木の挿入・検索・削除処理を行う関数群（src/include/lib/radixtree.h 内の各操作関数）。
  • **呼び出し先**: 直接的な関数呼び出しではなく、マクロ（RT_NODE_48_IS_CHUNK_USED, RT_NODE_48_GET_CHILD）を介したデータアクセスが主です。

関連データ構造

  • RT_NODE: 基底構造体。
  • RT_PTR_ALLOC: 子ノードを指すポインタまたはDSAハンドル。
  • RT_NODE_4, RT_NODE_16, RT_NODE_256: 他のサイズクラスのノード構造体。

失敗モードと境界条件

  • children 配列の範囲外アクセス: RT_FANOUT_48 の計算式に基づき、メモリ確保サイズが適切でない場合に発生する可能性があります。
  • RT_INVALID_SLOT_IDX の誤判定: チャンクが範囲外である場合や、未初期化のスロットを参照した場合に発生します。

保守担当者向けメモ

  • slot_idxs は memset で個別に初期化されるため、構造体のメモリレイアウト変更時には offsetof(RT_NODE_48, slot_idxs) の依存関係に注意してください。
  • 共有メモリ環境下では、ポインタ操作が dsa_pointer を介して行われる点に留意してください。
