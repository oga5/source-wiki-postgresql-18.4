CompareType

定義

  • 種別: enum
  • 定義箇所: src/include/access/cmptype.h:31

目的

PostgreSQL における演算子の比較セマンティクス（等価、不等価、大小比較、包含関係など）を抽象化し、インデックスアクセスメソッド（AM）や式評価器が演算子の意味を共通言語で解釈できるようにするためのブリッジとして機能します。

シグネチャまたは形状

typedef enum CompareType
{
	COMPARE_INVALID = 0,
	COMPARE_LT = 1,
	COMPARE_LE = 2,
	COMPARE_EQ = 3,
	COMPARE_GE = 4,
	COMPARE_GT = 5,
	COMPARE_NE = 6,
	COMPARE_OVERLAP,
	COMPARE_CONTAINED_BY,
} CompareType;

入力と出力

  • 入力: 演算子の意味論を表現する列挙値。
  • 出力: インデックス AM の戦略番号や、式ノード（RowCompareExpr 等）における比較ロジックの解釈結果として利用されます。

前提条件と不変条件

  • COMPARE_LT から COMPARE_GT までの値は、B-tree インデックスの戦略番号（BTLessStrategyNumber 等）と直接対応しています。
  • COMPARE_NE 以降のシンボルは、B-tree 以外のインデックス AM や複雑な比較セマンティクスを表現するために使用されます。

アルゴリズムまたは振る舞い

システムが特定の演算子を CompareType に変換することで、インデックス AM がその演算子の論理的な意味を理解し、適切な検索や評価を実行可能にします。

呼び出し元と呼び出し先

  • contrib/btree_gist/btree_gist.c

  • 呼び出し先: 本シンボルは列挙型の定義であり、直接的な関数呼び出し先はありません。

関連データ構造

  • src/include/access/stratnum.h: インデックス AM の戦略番号が定義されており、概念的に密接に関連しています。
  • src/include/nodes/primnodes.h: RowCompareExpr など、この列挙型を利用して比較ロジックを構築するノードが定義されています。

失敗モードと境界条件

  • COMPARE_INVALID は無効な比較状態を示します。
  • 現在のマッピングは主に B-tree の戦略番号に依存しているため、GiST や GIN などの他のインデックス AM で利用する際、マッピングが完全ではない可能性があります。

保守担当者向けメモ

  • 新しい演算子やインデックス AM を追加する際、システムがその演算子の意味を理解する必要がある場合に本列挙型の拡張を検討してください。
  • 現在の定義は B-tree への依存が強いため、他 AM への適用時にはマッピングの網羅性を確認してください。
