uint32

定義

  • 種別: 符号なし32ビット整数型（uint32）
  • 定義箇所: src/include/replication/walreceiver.h:180

目的

論理レプリケーションにおけるプロトコルバージョンを保持するために使用されます。

シグネチャまたは形状

uint32 は、logical 構造体内の最初のメンバとして定義されています。

struct
{
    uint32 proto_version;
    /* ... 他のメンバ ... */
} logical;

入力と出力

  • 入力: 論理レプリケーションのストリーミング開始時に、プロトコルバージョンとして設定されます。
  • 出力: プライマリサーバーとのネゴシエーションにおいて、サポートする論理プロトコルのバージョンを伝達するために参照されます。

前提条件と不変条件

  • 32ビットの符号なし整数として表現可能である必要があります。
  • 共有メモリ構造体の一部として使用される場合、適切なアトミック操作やロックによる保護が前提となります。

アルゴリズムまたは振る舞い

本シンボル自体はデータ型であり、特定のアルゴリズムを実装するものではありません。論理レプリケーションの通信において、送信側と受信側が互換性のあるプロトコルバージョンを使用しているかを検証するための比較対象として機能します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは、本シンボルを直接操作する具体的な関数呼び出し元は特定できません。
  • 呼び出し先: 本シンボルはプリミティブ型であるため、関数を呼び出すことはありません。

関連データ構造

  • WalRcvStreamOptions: 論理レプリケーションのストリーミングオプションを保持する構造体の一部として含まれます。

失敗モードと境界条件

  • 境界条件: 32ビットの最大値（UINT32_MAX）を超える値は保持できません。
  • 失敗モード: 不正なプロトコルバージョンが設定された場合、レプリケーションの接続確立時にプロトコル不一致エラーが発生する可能性があります。

保守担当者向けメモ

  • 本シンボルは標準的な整数型であり、walreceiver.h 内では論理レプリケーションのプロトコルバージョン管理という特定の文脈で使用されています。
  • 他のモジュール（contrib/amcheck 等）でも同名の型が広く利用されていますが、本ページは walreceiver.h における定義に焦点を当てています。
