SystemRowsSamplerData

定義

  • 種別: struct
  • 定義箇所: contrib/tsm_system_rows/tsm_system_rows.c:46

目的

SYSTEM_ROWS テーブルサンプルメソッドにおけるスキャン実行時の状態管理を行うための構造体です。スキャン中の進捗状況や、再スキャン時の一貫性を維持するためのパラメータを保持します。

シグネチャまたは形状

typedef struct {
    uint32      seed;           /* 乱数シード */
    int64       ntuples;        /* 返すべきタプル数 */
    OffsetNumber lt;            /* 現在のブロックで最後に返したタプルのオフセット */
    BlockNumber doneblocks;     /* 走査済みのブロック数 */
    BlockNumber lb;             /* 最後に訪問したブロック番号 */
    BlockNumber nblocks;        /* リレーション内の総ブロック数 */
    BlockNumber firstblock;     /* サンプリング開始ブロック */
    BlockNumber step;           /* ブロック走査のステップ幅（未設定時は0） */
} SystemRowsSamplerData;

入力と出力

本構造体は関数引数として直接受け渡されるものではなく、TsmRoutine を通じた実行コンテキストの一部として palloc0 によりメモリ確保され、各サンプリング関数内でキャストして参照されます。

前提条件と不変条件

  • nblocks、firstblock、step の3つの値は、一度設定されると再スキャン時にも変更されません。
  • step が 0 の場合、まだ初期化が行われていない状態を示します。

アルゴリズムまたは振る舞い

  • system_rows_nextsampleblock 関数において、step と firstblock を用いたスキップ走査アルゴリズムの状態保持に使用されます。
  • doneblocks をインクリメントすることで、全ブロックを網羅したかどうかの判定に寄与します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • system_rows_initsamplescan (メモリ確保)
  • system_rows_beginsamplescan (状態参照)
  • system_rows_nextsampleblock (状態参照および更新)
  • system_rows_nextsampletuple (状態参照および更新)
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • TsmRoutine: SYSTEM_ROWS メソッドのルーチン定義。
  • BlockNumber, OffsetNumber: PostgreSQL の標準的な型。

失敗モードと境界条件

  • step が 0 の状態でサンプリングが開始された場合、初回実行時に random_relative_prime を用いて初期化が行われる必要があります。
  • リレーションのブロック数が変動した場合、再スキャン時に nblocks の不一致によりサンプリング結果が変化する可能性があります。

保守担当者向けメモ

  • 本構造体は node->tsm_state を通じてアクセスされます。キャストの際は SystemRowsSamplerData * への変換が必須です。
  • 構造体内の nblocks 等のフィールドは、クエリ実行中の再スキャンにおいて一貫性を保つための重要な役割を担っています。
