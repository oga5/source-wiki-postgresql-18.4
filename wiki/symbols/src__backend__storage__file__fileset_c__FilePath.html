FilePath

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/fileset.c:196

目的

FileSet 内の特定のファイル名に対応する物理的なファイルパスを構築する。

シグネチャまたは形状

static void FilePath(char *path, FileSet *fileset, const char *name)

入力と出力

  • **入力**:
  • path: パス文字列を格納するためのバッファ（MAXPGPATH 以上のサイズが必要）。
  • fileset: 対象となる FileSet 構造体へのポインタ。
  • name: ファイル名。
  • **出力**:
  • path 引数に、構築されたフルパスが書き込まれる。

前提条件と不変条件

  • path 引数は MAXPGPATH バイト以上のメモリ領域を指していること。
  • fileset は FileSetInit() によって初期化済みであること。

アルゴリズムまたは振る舞い

1. ChooseTablespace(fileset, name) を呼び出し、ファイル名に基づいて配置先のテーブルスペースを決定する。 2. FileSetPath() を呼び出し、決定されたテーブルスペースに対応するディレクトリパスを dirpath に取得する。 3. snprintf を使用し、dirpath と name を結合して path に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/file/fileset.c 内の FileSetCreate、FileSetOpen、FileSetDelete 等から呼び出される。
  • **呼び出し先**:
  • ChooseTablespace
  • FileSetPath

関連データ構造

  • FileSet: 一時ファイルの集合を管理する構造体。

失敗モードと境界条件

  • name が長すぎる場合、snprintf の制限によりパスが切り詰められる可能性がある。
  • path バッファが MAXPGPATH 未満の場合、バッファオーバーフローが発生する可能性がある。

保守担当者向けメモ

  • 本関数は static であり、fileset.c 内でのみ使用される内部ヘルパー関数である。
  • パス生成ロジックを変更する場合は、FileSetPath および ChooseTablespace との整合性に注意すること。
