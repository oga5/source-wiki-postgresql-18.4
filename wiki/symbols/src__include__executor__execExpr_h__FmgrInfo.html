FmgrInfo

定義

  • 種別: struct
  • 定義箇所: src/include/executor/execExpr.h:386

目的

PostgreSQLの関数呼び出しにおけるルックアップデータおよび実行に必要な情報を保持する構造体です。式評価エンジン内では、関数実行ステップのコンテキストの一部として利用されます。

シグネチャまたは形状

src/include/executor/execExpr.h 内の func 構造体メンバとして以下の構成で定義されています。

struct {
    FmgrInfo   *finfo;          /* 関数ルックアップデータ */
    FunctionCallInfo fcinfo_data; /* 引数等の情報 */
    PGFunction  fn_addr;        /* 実際の呼び出しアドレス */
    int         nargs;          /* 引数の数 */
    bool        make_ro;        /* arg0を読み取り専用にするか（NULLIF用） */
} func;

入力と出力

  • **入力**: 関数実行に必要なメタデータ（finfo）、引数情報（fcinfo_data）、および実行アドレス（fn_addr）。
  • **出力**: 実行エンジンが関数を呼び出すための直接的な参照情報を提供します。

前提条件と不変条件

  • fn_addr は、間接参照を避けるためにキャッシュされた関数ポインタです。
  • make_ro フラグは NULLIF 命令の実行時のみ使用されます。

アルゴリズムまたは振る舞い

式評価ステップにおいて、関数呼び出しのオーバーヘッドを最小化するために使用されます。finfo を介して関数情報を取得し、fn_addr を直接参照することで、実行時のディスパッチを効率化します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/executor/execExpr.h 内の式評価ステップ（ExprEvalStep）から参照されます。
  • **呼び出し先**: 拡張モジュール（contrib/bloom, contrib/btree_gist 等）の各関数で、引数として渡されることが確認されています。

関連データ構造

  • ExprEvalStep: 式評価の最小単位であり、本構造体を含む func メンバを保持します。
  • FunctionCallInfo: 関数呼び出し時の引数や状態を管理する構造体。

失敗モードと境界条件

  • finfo が適切に初期化されていない場合、関数呼び出し時に不正なメモリ参照や実行時エラーが発生する可能性があります。
  • nargs が実際の引数数と一致しない場合、関数実行時にスタックや引数処理の不整合を招く恐れがあります。

保守担当者向けメモ

  • 本構造体は ExprEvalStep の一部としてキャッシュラインを意識した設計になっています。メンバの追加や変更を行う際は、構造体全体のサイズがキャッシュ効率に与える影響を考慮してください。
  • fn_addr への直接アクセスはパフォーマンス向上のための最適化です。この設計を維持するため、関数呼び出しのパスを変更する際は注意が必要です。
