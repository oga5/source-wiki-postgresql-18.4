PLySavedArgs

定義

  • 種別: struct
  • 定義箇所: src/pl/plpython/plpy_procedure.h:15

目的

PL/Pythonプロシージャ実行時における引数スタックの管理を目的とした構造体です。関数呼び出しの再帰やネストが発生する際に、現在の引数状態を保存・復元するために使用されます。

シグネチャまたは形状

typedef struct PLySavedArgs
{
    struct PLySavedArgs *next;
    PyObject   *args;
    PyObject   *td;
    int         nargs;
    PyObject   *namedargs[FLEXIBLE_ARRAY_MEMBER];
} PLySavedArgs;

入力と出力

  • **入力**: PLy_function_save_args 関数により、現在の実行コンテキストから引数情報が抽出され、本構造体に格納されます。
  • **出力**: PLy_function_restore_args 関数により、保存された引数情報がプロシージャの実行環境へ書き戻されます。

前提条件と不変条件

  • namedargs は可変長配列（FLEXIBLE_ARRAY_MEMBER）として定義されており、構造体割り当て時に nargs に基づくメモリ確保が必要です。
  • next ポインタにより、複数の引数セットをリンクリストとして保持可能です。

アルゴリズムまたは振る舞い

1. **保存**: PLy_function_save_args が現在の引数（args）およびトリガーデータ（td）を本構造体にコピーします。 2. **復元**: PLy_function_restore_args が、保存された値をプロシージャのグローバル辞書等に再設定します。 3. **解放**: PLy_function_drop_args により、保持していた PyObject への参照カウントを適切に処理し、構造体メモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_exec.c 内の関数群が主に利用しています。
  • PLy_function_save_args: 引数状態の保存。
  • PLy_function_restore_args: 引数状態の復元。
  • PLy_function_drop_args: 構造体の破棄。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • PLyProcedure: プロシージャのメタデータを保持する構造体。本構造体と組み合わせて実行コンテキストを管理します。

失敗モードと境界条件

  • メモリ確保失敗: PLy_function_save_args 内での malloc 等の失敗時にエラーが発生する可能性があります。
  • 参照カウントの不整合: PyObject を保持しているため、解放処理（PLy_function_drop_args）が正しく行われない場合、Python側のメモリリークを招く可能性があります。

保守担当者向けメモ

  • namedargs が可変長配列であるため、インスタンス生成時には offsetof(PLySavedArgs, namedargs) を用いたサイズ計算が必須です。
  • トリガー関数実行時には td フィールドが有効になりますが、通常の関数では NULL になる点に注意してください。
