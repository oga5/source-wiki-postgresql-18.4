btvalidate

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtvalidate.c:39

目的

B-tree インデックスの演算子クラス（opclass）および演算子族（opfamily）の定義が、PostgreSQL の B-tree 実装要件を満たしているかを検証する。

シグネチャまたは形状

bool btvalidate(Oid opclassoid)

入力と出力

  • **入力**: opclassoid (検証対象の演算子クラスの OID)
  • **出力**: bool (検証に成功した場合は true、失敗した場合は false)

前提条件と不変条件

  • 指定された opclassoid がシステムカタログ pg_opclass に存在すること。
  • 演算子族内の各演算子およびサポート関数が、B-tree が規定する戦略番号およびシグネチャの要件を満たしていること。

アルゴリズムまたは振る舞い

1. **メタデータ取得**: opclassoid から演算子クラスおよび関連する演算子族の情報をシステムカタログから取得する。 2. **サポート関数検証**: 演算子族に登録された各サポート関数について、amprocnum に基づきシグネチャを検証する。

  • BTORDER_PROC, BTSORTSUPPORT_PROC, BTINRANGE_PROC, BTEQUALIMAGE_PROC, BTOPTIONS_PROC, BTSKIPSUPPORT_PROC の各ケースに対し、期待される引数型と戻り値型をチェックする。

    • 3. **演算子検証**: 演算子族内の各演算子について、戦略番号が有効範囲内か、ORDER BY 演算子が含まれていないか、および演算子のシグネチャが正しいかを検証する。 4. **グループ整合性チェック**: identify_opfamily_groups を使用して演算子と関数のグループを分類し、データ型ごとの完全性を確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtree.c 内で amroutine->amvalidate に割り当てられ、インデックス作成時等に呼び出される。
  • **呼び出し先**: SearchSysCache1, get_opfamily_name, check_amproc_signature, check_amop_signature, identify_opfamily_groups 等のユーティリティ関数。

関連データ構造

  • Form_pg_opclass: 演算子クラスの定義情報。
  • Form_pg_amproc: 演算子族のサポート関数情報。
  • Form_pg_amop: 演算子族の演算子情報。
  • OpFamilyOpFuncGroup: 演算子族内の演算子と関数のグループ化構造。

失敗モードと境界条件

  • **キャッシュ検索失敗**: opclassoid が無効な場合、elog(ERROR) で停止する。
  • **検証エラー**: サポート関数の番号が不正、シグネチャが不一致、または演算子の戦略番号が範囲外の場合、ereport(INFO, ...) で詳細を報告し、戻り値として false を返す。

保守担当者向けメモ

  • 本関数は switch 文で B-tree がサポートする全サポート関数を網羅している。新しいサポート関数を追加する際は、この switch 文への追記が必要。
  • identify_opfamily_groups 以降の処理は、データ型間の整合性チェックを行うための重要なロジックである。
