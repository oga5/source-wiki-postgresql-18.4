VersionedQuery

定義

  • 種別: struct
  • 定義箇所: src/bin/psql/tab-complete.in.c:107

目的

PostgreSQLのサーバーバージョンに応じて、タブ補完用のSQLクエリを切り替えるためのデータ構造を提供します。

シグネチャまたは形状

typedef struct VersionedQuery
{
	int			min_server_version;
	const char *query;
} VersionedQuery;

入力と出力

  • min_server_version: 適用対象となる最小のサーバーバージョン（整数）。
  • query: 実行されるSQLクエリ文字列。

前提条件と不変条件

  • クエリの詳細はサーバーバージョンに依存するため、通常は VersionedQuery の配列として定義されます。
  • サーバーバージョンが min_server_version を満たす場合に、対応する query が補完候補取得のために使用されます。

アルゴリズムまたは振る舞い

タブ補完の実行時、接続先サーバーのバージョンに基づき、適切な VersionedQuery が選択されます。これにより、古いサーバーでは存在しない機能やカタログに対するクエリを回避し、バージョン固有の補完を可能にします。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/tab-complete.in.c 内の補完ロジック（completion_vquery や Query_for_list_of_publications 等の配列経由）。
  • 呼び出し先: 直接的な関数呼び出しではなく、補完エンジンがこの構造体を参照してSQLを発行します。

関連データ構造

  • SchemaQuery: データベースオブジェクト補完用のクエリ定義構造体。
  • completion_vquery: 現在の補完処理で参照されている VersionedQuery へのポインタ。

失敗モードと境界条件

  • サーバーバージョンがどの min_server_version にも適合しない場合、補完候補が正しく生成されない可能性があります。
  • 補完クエリ自体が失敗した場合、ユーザーにはエラーが表示されない設計となっています。

保守担当者向けメモ

  • 本構造体は gen_tabcomplete.pl によるビルド時変換の影響を受ける可能性があります。
  • 新しいサーバー機能に対応する補完を追加する際は、適切な min_server_version を設定した VersionedQuery の配列を定義してください。
