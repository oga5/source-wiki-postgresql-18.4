create_tablespace_directories

定義

  • **種別**: function
  • **定義箇所**: src/backend/commands/tablespace.c:571

目的

テーブルスペース作成時に必要な物理ディレクトリ構造（バージョンディレクトリ）の構築および、$PGDATA/pg_tblspc 配下へのシンボリックリンク作成を行う。

シグネチャまたは形状

static void create_tablespace_directories(const char *location, const Oid tablespaceoid)

入力と出力

  • **入力**:
  • location: テーブルスペースの物理的な格納先パス。空文字列の場合は「インプレース（in-place）」モードとして動作する。
  • tablespaceoid: 作成対象のテーブルスペースの OID。
  • **出力**: なし（成功時はディレクトリ作成およびシンボリックリンク作成の副作用が発生）。

前提条件と不変条件

  • location が空でない場合、対象ディレクトリは事前に存在し、適切な権限設定が可能である必要がある。
  • in_place モード（location が空）は開発・回帰テスト専用である。
  • バージョンディレクトリ（TABLESPACE_VERSION_DIRECTORY）の存在により、同一パスへの複数テーブルスペース作成を防止する。

アルゴリズムまたは振る舞い

1. **パス生成**: PG_TBLSPC_DIR 配下に OID を用いたシンボリックリンクパスを生成。 2. **インプレース処理**: location が空の場合、リンク先パス自体をディレクトリとして作成。 3. **権限確認**: インプレースモード以外では、location に対して chmod を実行し、適切なディレクトリ作成権限があるか検証。 4. **バージョンディレクトリ作成**: location 配下にバージョンディレクトリを作成。既に存在する場合はディレクトリか否かを確認し、リカバリ中でなければエラーを投げる。 5. **シンボリックリンク作成**: リカバリ中の場合は既存のリンクを削除し、location を指すシンボリックリンクを PGDATA 配下に作成。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • CreateTableSpace（DDL 実行時）
  • tblspc_redo（WAL リプレイ時）
  • **呼び出し先**:
  • MakePGDirectory
  • chmod
  • stat
  • remove_tablespace_symlink
  • symlink
  • psprintf, pfree

関連データ構造

  • Oid: テーブルスペース識別子。
  • struct stat: ディレクトリ状態確認用。

失敗モードと境界条件

  • **ディレクトリ作成失敗**: MakePGDirectory が失敗した場合、エラーを報告。
  • **権限エラー**: chmod が失敗した場合、またはディレクトリが存在しない場合にエラーを報告。
  • **型不一致**: バージョンディレクトリ位置にディレクトリ以外のファイルが存在する場合、エラーを報告。
  • **重複利用**: 既にディレクトリが存在し、かつリカバリ中でない場合、その場所が既にテーブルスペースとして使用中であると判断しエラーを報告。
  • **リンク作成失敗**: symlink が失敗した場合、エラーを報告。

保守担当者向けメモ

  • 本関数は pg_tablespace_spcname_index によるカタログレベルの競合防止を前提としている。
  • リカバリ中のシンボリックリンク削除は、誤った場所を指している可能性を考慮した安全策である。
