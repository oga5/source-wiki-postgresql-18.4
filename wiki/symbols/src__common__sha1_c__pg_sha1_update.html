pg_sha1_update

定義

  • src/common/sha1.c:331

目的

SHA-1 ハッシュ計算において、入力データストリームをコンテキストに追加し、内部バッファが64バイトに達するごとにハッシュ変換処理を実行する。

シグネチャまたは形状

void pg_sha1_update(pg_sha1_ctx *ctx, const uint8 *data, size_t len)

入力と出力

  • ctx: SHA-1 コンテキスト構造体（入力および更新対象）。
  • data: ハッシュ計算対象のバイト列。
  • len: data のバイト長。
  • 出力: なし（ctx の内部状態が更新される）。

前提条件と不変条件

  • ctx は事前に pg_sha1_init によって初期化されていること。
  • COUNT マクロはコンテキスト内の現在のバッファオフセットを追跡しており、64バイト単位の境界を維持する。

アルゴリズムまたは振る舞い

1. 入力データ data をループで処理する。 2. 現在のバッファの空き容量（gaplen）を計算し、入力データからコピー可能な分だけ ctx->m.b8 に memmove する。 3. コピーしたバイト数分だけ COUNT を進め、メッセージ長（ビット単位）を ctx->c.b64[0] に加算する。 4. COUNT が64の倍数（ブロック境界）に達した場合、sha1_step(ctx) を呼び出してブロックのハッシュ変換を行う。 5. 全データが処理されるまで繰り返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/cryptohash.c:147 からの呼び出しが確認されている。
  • 呼び出し先: sha1_step を内部で呼び出す。

関連データ構造

  • pg_sha1_ctx: sha1_int.h で定義される SHA-1 状態保持用構造体。

失敗モードと境界条件

  • len が0の場合、ループは実行されず何も行われない。
  • バッファの境界（64バイト）を跨ぐデータ入力時、適切に分割して sha1_step が呼び出される。

保守担当者向けメモ

  • RFC 3174 に準拠した実装である。
  • WORDS_BIGENDIAN マクロによるエンディアン依存の処理が含まれるため、移植時には注意が必要。
  • COUNT マクロおよび ctx->c.b64 の更新ロジックは、ハッシュの整合性に直結するため変更時は慎重を期すこと。
