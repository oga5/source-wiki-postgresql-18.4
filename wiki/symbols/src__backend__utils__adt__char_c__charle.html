charle

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/char.c:153

目的

PostgreSQLの内部型 "char"（1バイト文字型）の2つの値を受け取り、第1引数が第2引数以下であるかを判定します。

シグネチャまたは形状

Datum charle(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの "char" 型の値を受け取ります。
  • **出力**: 第1引数が第2引数以下であれば true、そうでなければ false を返します（PG_RETURN_BOOL）。

前提条件と不変条件

  • 入力値は char 型として扱われますが、比較時には uint8（符号なし8ビット整数）にキャストされます。
  • この比較ロジックは、int4 との変換時に使用される符号付きの解釈とは異なる場合があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_CHAR マクロを使用して、2つの入力引数を取得します。 2. それぞれの値を uint8 にキャストします。 3. キャスト後の値を用いて、arg1 <= arg2 の比較演算を行います。 4. 結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（演算子呼び出し経由）。
  • **呼び出し先**: PG_GETARG_CHAR および PG_RETURN_BOOL マクロ。
  • 注: ソースコード内に直接的な関数呼び出しの記述はありません。

関連データ構造

  • Datum: PostgreSQLの汎用データ型表現。
  • "char": 1バイト固定長の内部文字型。

失敗モードと境界条件

  • 比較は符号なし8ビット整数として行われるため、文字コードの大小関係は 0 から 255 の範囲で評価されます。

保守担当者向けメモ

  • 本関数は uint8 として比較を行いますが、int4 への変換関数（chartoi4 等）は符号付きとして扱うため、型変換と比較演算の間で一貫性が欠ける可能性があります。変更を加える際は、この歴史的な不整合に注意してください。
