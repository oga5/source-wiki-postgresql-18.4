sepgsql_proc_execute

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/proc.c:314

目的

PostgreSQLの関数実行時において、SELinuxポリシーに基づく db_procedure:{execute} 権限の検証を行う。

シグネチャまたは形状

void sepgsql_proc_execute(Oid functionId)

入力と出力

  • **入力**: functionId (Oid) - 実行対象となる関数のオブジェクト識別子。
  • **出力**: なし。権限チェックに失敗した場合はエラーを発生させて処理を中断する。

前提条件と不変条件

  • functionId は pg_proc カタログ内の有効な関数を参照している必要がある。
  • 呼び出し側は、関数が実行される直前のコンテキストで本関数を呼び出す必要がある。

アルゴリズムまたは振る舞い

1. ObjectAddress 構造体を初期化し、classId を ProcedureRelationId、objectId を引数の functionId に設定する。 2. getObjectIdentity を呼び出し、監査ログ用のオブジェクト識別文字列を取得する。 3. sepgsql_avc_check_perms を呼び出し、以下の条件で権限を検証する:

  • クラス: SEPG_CLASS_DB_PROCEDURE
  • 権限: SEPG_DB_PROCEDURE__EXECUTE
  • 監査フラグ: true (拒否時にログを出力)

    • 4. 取得した監査文字列を pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/hooks.c:277 (直接の証拠あり)
  • **呼び出し先**:
  • getObjectIdentity (PostgreSQLコアAPI)
  • sepgsql_avc_check_perms (SE-PostgreSQL AVCインターフェース)
  • pfree (PostgreSQLメモリ管理)

関連データ構造

  • ObjectAddress: PostgreSQLのオブジェクトを特定するための構造体。
  • ProcedureRelationId: pg_proc カタログを指す定数。

失敗モードと境界条件

  • 権限チェックが失敗した場合、sepgsql_avc_check_perms 内で例外が発生し、トランザクションがアボートされる。
  • getObjectIdentity が失敗した場合、またはメモリ不足が発生した場合は、PostgreSQLの標準的なエラー処理に従う。

保守担当者向けメモ

  • 本関数は pg_proc に対するフックとして機能する。
  • 監査ログの生成に getObjectIdentity を使用しているため、関数名やスキーマ名が変更された場合でも適切な識別子がログに残る。
