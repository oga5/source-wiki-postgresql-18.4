ReportSlotConnectionError

定義

  • 種別: function
  • 定義箇所: src/backend/commands/subscriptioncmds.c:2322

目的

論理レプリケーションにおいて、パブリッシャーノードへの接続失敗時にエラーを報告し、関連するテーブル同期用レプリケーションスロットの削除失敗について警告を出力する。

シグネチャまたは形状

static void ReportSlotConnectionError(List *rstates, Oid subid, char *slotname, char *err)

入力と出力

  • rstates: SubscriptionRelState 構造体のリスト。各テーブルの同期状態を保持する。
  • subid: 対象サブスクリプションの OID。
  • slotname: 接続失敗したレプリケーションスロット名。
  • err: 接続失敗の詳細を示すエラーメッセージ。
  • 戻り値: なし（内部で ereport(ERROR, ...) を呼び出し、トランザクションを中断する）。

前提条件と不変条件

  • rstates 内の各 SubscriptionRelState は、呼び出し元によって状態が変更されないことが保証されている必要がある。
  • subid は有効なサブスクリプションを指している必要がある。

アルゴリズムまたは振る舞い

1. rstates リストを走査し、各要素（rstate）を確認する。 2. relid が有効なテーブル同期ワーカーについて、状態が SUBREL_STATE_SYNCDONE でない場合、ReplicationSlotNameForTablesync を使用してスロット名を特定し、削除失敗の警告（WARNING）を出力する。 3. 最後に、パブリッシャーへの接続失敗を理由として ereport(ERROR) を発行し、ユーザーに対して ALTER SUBSCRIPTION コマンドを用いた対処方法を提示する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/subscriptioncmds.c 内の関数（詳細はソースコードの文脈に依存するが、主にサブスクリプション削除や接続処理に関連する箇所から参照される）。
  • 呼び出し先: ReplicationSlotNameForTablesync（警告出力用）、elog、ereport。

関連データ構造

  • SubscriptionRelState: テーブルごとの同期状態を管理する構造体。
  • List: PostgreSQL の汎用リスト構造。

失敗モードと境界条件

  • rstates が空の場合、警告出力はスキップされ、接続エラーのみが報告される。
  • relid が無効（!OidIsValid）な要素は無視される。
  • ereport(ERROR) により、この関数を呼び出したトランザクションはアボートされる。

保守担当者向けメモ

  • 本関数は、レプリケーションスロットのクリーンアップが不完全な状態で接続エラーが発生した際のユーザーへのガイダンスを提供する。
  • 警告メッセージの出力は、あくまで接続失敗という致命的なエラーの補足情報として扱われる。
