ER_flatten_into

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/expandedrecord.c:763

目的

展開されたレコード（ExpandedRecordHeader）の内部状態を、物理的な HeapTupleHeader 形式へ再構築（フラット化）します。

シグネチャまたは形状

static void ER_flatten_into(ExpandedObjectHeader *eohptr, void *result, Size allocated_size)

入力と出力

  • **入力**:
  • eohptr: 展開されたレコードのヘッダー。
  • result: フラット化されたデータを書き込むためのメモリ領域。
  • allocated_size: result に割り当てられたサイズ。
  • **出力**: なし（result が指すメモリ領域にタプルデータが書き込まれます）。

前提条件と不変条件

  • erh->er_magic が ER_MAGIC であること。
  • allocated_size は、事前に計算された erh->flat_size と一致していること。

アルゴリズムまたは振る舞い

1. **高速パス**: 既にフラット化された値が存在し、かつ外部ストレージ（TOAST）フィールドを含まない場合、erh->fvalue を result にコピーし、ヘッダー情報（型ID、型修飾子、データ長）を更新して終了します。 2. **通常パス**:

  • result 領域をゼロクリアします。
  • HeapTupleHeader のヘッダーフィールド（データ長、型ID、型修飾子、t_ctid、属性数、t_hoff）を設定します。
  • heap_fill_tuple を呼び出し、erh->dvalues（フィールド値）と erh->dnulls（NULLフラグ）からデータ領域を構築します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の言及として src/backend/utils/adt/expandedrecord.c 内の複数箇所で参照されていますが、具体的な呼び出しフローの詳細は提供されていません。
  • **呼び出し先**: expanded_record_get_tupdesc、heap_fill_tuple、および HeapTupleHeader 操作用マクロ（HeapTupleHeaderSetDatumLength 等）。

関連データ構造

  • ExpandedRecordHeader: 展開されたレコードのメイン構造体。
  • HeapTupleHeader: 物理的なタプルのヘッダー構造。
  • TupleDesc: タプルの記述子。

失敗モードと境界条件

  • allocated_size が期待値と一致しない場合、Assert により停止します。
  • dvalues や dnulls が適切に初期化されていない場合、heap_fill_tuple の実行時に未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は ER_methods プロトコルの一部として実装されています。
  • ER_FLAG_HAVE_EXTERNAL が設定されている場合、単純なコピーではなく heap_fill_tuple による再構築が必須となります。
  • t_hoff（データオフセット）の管理は erh->hoff に依存しているため、レコード構築時のオフセット計算との整合性に注意してください。
