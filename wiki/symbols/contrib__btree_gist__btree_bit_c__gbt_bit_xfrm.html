gbt_bit_xfrm

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bit.c:74

目的

BIT 型および VARBIT 型のデータを GiST インデックスに格納する際、メモリ配置をアライメント調整済み（INTALIGN）の形式に変換するために使用されます。

シグネチャまたは形状

static bytea *gbt_bit_xfrm(bytea *leaf)

入力と出力

  • **入力**: leaf (変換対象の bytea 型ビット列データ)
  • **出力**: アライメント調整およびパディングが施された新しい bytea 型のポインタ。

前提条件と不変条件

  • 入力データは有効な VARBIT 構造である必要があります。
  • 出力バッファは palloc によって確保され、パディング領域はゼロで初期化されます。
  • 返されるデータサイズは INTALIGN によって調整されたサイズとなります。

アルゴリズムまたは振る舞い

1. 入力データのサイズ（ヘッダ込み）を計算し、INTALIGN を用いてアライメント調整後のサイズ padded_sz を算出します。 2. palloc で padded_sz 分のメモリを確保します。 3. パディング領域（sz から padded_sz まで）をゼロで埋めます。 4. SET_VARSIZE を使用して、新しいバッファのサイズを padded_sz に設定します。 5. memcpy を使用して、元のビットデータを新しいバッファのデータ領域へコピーします。 6. 変換後のバッファを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_bit.c:100 (内部ノード処理時)
  • contrib/btree_gist/btree_bit.c:156 (クエリ処理時)
  • **呼び出し先**:
  • palloc (メモリ確保)
  • SET_VARSIZE (PostgreSQL マクロ)
  • memcpy (メモリコピー)
  • VARBITBYTES, VARHDRSZ, VARDATA, VARBITS (PostgreSQL マクロ)

関連データ構造

  • bytea: PostgreSQL の可変長データ型。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • 入力ポインタが NULL の場合の挙動は定義されていません。

保守担当者向けメモ

  • 本関数は GiST インデックスの内部構造への適合を目的としており、インデックスの検索時および構築時の両方でデータの一貫性を保つために不可欠です。
  • パディング領域をゼロクリアすることで、比較時の不定なメモリ内容による誤判定を防いでいます。
