array_prepend

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_userfuncs.c:222

目的

配列の先頭に新しい要素を追加し、新しい配列を生成します。

シグネチャまたは形状

Datum array_prepend(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して、追加する要素（anyelement）と対象の配列（anyarray）を受け取ります。
  • **出力**: Datum 型として、要素が先頭に追加された新しい配列を返します。

前提条件と不変条件

  • 入力配列は空（次元数0）または1次元である必要があります。それ以外の次元を持つ配列が渡された場合はエラーとなります。
  • fetch_array_arg_replace_nulls を使用して、NULL入力の処理と配列の展開を行います。

アルゴリズムまたは振る舞い

1. PG_ARGISNULL を用いて追加要素がNULLかどうかを判定します。 2. fetch_array_arg_replace_nulls で対象配列を ExpandedArrayHeader 形式で取得します。 3. 次元数を確認し、1次元の場合は現在のインデックスの下限（lbound）から新しいインデックスを計算します（オーバーフローチェックを含む）。次元数が0の場合はインデックスを1とします。 4. array_set_element を呼び出し、指定した位置に要素を挿入します。 5. 1次元配列の場合、結果の配列の下限（lbound）を元の配列の下限値に再設定します。 6. 最終的な配列を Datum として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースから直接呼び出されます。また、src/bin/pg_upgrade/check.c 等で参照されています。
  • **呼び出し先**: fetch_array_arg_replace_nulls（引数取得）、array_set_element（要素挿入）、pg_sub_s32_overflow（算術チェック）、ereport（エラー報告）。

関連データ構造

  • ExpandedArrayHeader: 展開された配列のヘッダー構造。
  • ArrayMetaState: 配列の要素型情報（typlen, typbyval, typalign）を保持するキャッシュ構造体。

失敗モードと境界条件

  • **データ例外**: 1次元または空以外の配列が渡された場合、ERRCODE_DATA_EXCEPTION でエラーとなります。
  • **数値範囲外**: インデックス計算時に整数オーバーフローが発生した場合、ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE でエラーとなります。

保守担当者向けメモ

  • 本関数は fn_extra を利用して型情報をキャッシュしており、同一クエリ内での繰り返し実行時のパフォーマンスを最適化しています。
  • 1次元配列の先頭追加という性質上、下限値（lbound）の維持が重要であり、挿入後に明示的な再設定を行っています。
