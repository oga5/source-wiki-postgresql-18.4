gistUnloadNodeBuffers

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistbuildbuffers.c:271

目的

メモリ上にロードされているすべての GiST ノードバッファを一時ファイルへ書き出し（フラッシュ）、メモリ上の状態をクリアする。

シグネチャまたは形状

void gistUnloadNodeBuffers(GISTBuildBuffers *gfbb)

入力と出力

  • **入力**: gfbb - GiST バッファリングビルドの状態を保持する GISTBuildBuffers 構造体へのポインタ。
  • **出力**: なし（副作用としてメモリ上のバッファがディスクへ退避される）。

前提条件と不変条件

  • gfbb が有効な GISTBuildBuffers 構造体を指していること。
  • 実行後、gfbb->loadedBuffersCount は必ず 0 になる。

アルゴリズムまたは振る舞い

1. gfbb->loadedBuffersCount に格納されている、現在メモリ上にロードされているバッファのリストを走査する。 2. 各バッファに対して gistUnloadNodeBuffer を呼び出し、ディスクへ書き出す。 3. 処理完了後、gfbb->loadedBuffersCount を 0 にリセットし、メモリ上にロードされたバッファが存在しない状態にする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistbuild.c:1317
  • **呼び出し先**: gistUnloadNodeBuffer

関連データ構造

  • GISTBuildBuffers: バッファリングビルドの全体状態を管理する構造体。

失敗モードと境界条件

  • gistUnloadNodeBuffer 内部で一時ファイル書き込みエラーが発生した場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は、メモリ上のバッファを強制的にフラッシュする際に使用される。
  • loadedBuffersCount をリセットする前に全てのバッファを処理する必要があるため、ループの順序やカウンタの更新タイミングには注意すること。
