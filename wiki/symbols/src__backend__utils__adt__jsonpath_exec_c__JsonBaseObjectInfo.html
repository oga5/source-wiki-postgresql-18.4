JsonBaseObjectInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonpath_exec.c:82

目的

JSONPath 実行エンジンにおいて、.keyvalue() メソッド等の評価対象となる「ベースオブジェクト」を保持するために使用されます。

シグネチャまたは形状

typedef struct JsonBaseObjectInfo
{
	JsonbContainer *jbc;
	int			id;
} JsonBaseObjectInfo;

入力と出力

  • jbc: 評価対象の JsonbContainer へのポインタ。
  • id: ベースオブジェクトを識別するための整数値。

前提条件と不変条件

  • jbc は有効な JsonbContainer を指している必要があります。
  • 実行コンテキスト内で、特定のパス評価の起点となるオブジェクトを特定するために利用されます。

アルゴリズムまたは振る舞い

  • setBaseObject() 関数によって初期化され、JsonPathExecContext 内のベースオブジェクトとして設定されます。
  • .keyvalue() メソッドの実行時など、パスの走査中に現在のコンテキストからベースオブジェクトを参照するために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: setBaseObject 関数（定義: src/backend/utils/adt/jsonpath_exec.c:332）がこの構造体を生成します。
  • 呼び出し先: 実行エンジン内の各評価関数（executeItem 等）が、コンテキストからこの構造体を参照します。
  • 参照箇所: src/backend/utils/adt/jsonpath_exec.c 内の複数箇所（103, 750, 2873, 3495行目付近）で、ローカル変数として宣言または代入されています。

関連データ構造

  • JsonbContainer: JSONB データの内部構造体。
  • JsonPathExecContext: 実行時の状態を保持する構造体。本構造体はこのコンテキストの一部として管理されます。

失敗モードと境界条件

  • jbc が NULL の場合、または無効なコンテナを指している場合の挙動については、本ソース excerpt からは明示されていません。

保守担当者向けメモ

  • 本構造体は .keyvalue() メソッドの評価に特化した情報保持用です。
  • JsonPathExecContext 内の baseObject フィールドと密接に関連しているため、変更時は実行コンテキストのライフサイクルに注意してください。
