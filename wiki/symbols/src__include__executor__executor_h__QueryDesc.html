QueryDesc

目的

クエリ実行エンジンにおいて、実行対象となるクエリプランと、その実行に必要な状態情報を保持するための記述子です。

シグネチャまたは形状

typedef struct QueryDesc QueryDesc; （詳細は src/include/executor/execdesc.h に定義）

入力と出力

本シンボル自体はデータ構造ですが、実行ライフサイクル関数（ExecutorStart 等）の引数として渡され、実行状態の受け渡しに使用されます。

前提条件と不変条件

  • 実行エンジンによる処理の開始前に初期化されている必要があります。
  • 実行ライフサイクル（Start -> Run -> Finish -> End）を通じて、一貫した実行状態を維持します。

アルゴリズムまたは振る舞い

クエリ実行の各フェーズにおいて、プランツリー、実行時パラメータ、および実行結果を格納するコンテナとして機能します。実行エンジンは、この構造体を参照してプランノードのトラバースや式評価を行います。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • contrib/auto_explain/auto_explain.c: explain_ExecutorStart, explain_ExecutorRun, explain_ExecutorFinish, explain_ExecutorEnd
  • contrib/pg_stat_statements/pg_stat_statements.c: pgss_ExecutorStart, pgss_ExecutorRun, pgss_ExecutorFinish, pgss_ExecutorEnd
  • **呼び出し先**:
  • 本シンボルはデータ構造であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • EState (実行状態)
  • Plan / PlannedStmt (クエリプラン)
  • TupleTableSlot (タプル操作用)

失敗モードと境界条件

  • 構造体が適切に初期化されていない状態で実行関数に渡された場合、実行エンジンは未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • QueryDesc の詳細なメンバ構成については src/include/executor/execdesc.h を参照してください。
  • 実行エンジンのフック（ExecutorStart_hook 等）を実装する際は、この構造体を介して実行コンテキストにアクセスします。
