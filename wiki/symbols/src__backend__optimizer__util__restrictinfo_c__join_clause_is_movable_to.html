join_clause_is_movable_to

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/optimizer/util/restrictinfo.c:574

目的

結合条件（RestrictInfo）が、特定のベースリレーション（RelOptInfo）へ移動（プッシュダウン）可能かどうかを判定します。

シグネチャまたは形状

bool join_clause_is_movable_to(RestrictInfo *rinfo, RelOptInfo *baserel)

入力と出力

  • **入力**:
  • rinfo: 判定対象の RestrictInfo 構造体。
  • baserel: 移動先候補となる RelOptInfo 構造体。
  • **出力**:
  • bool: 移動可能な場合は true、そうでない場合は false を返します。

前提条件と不変条件

  • rinfo および baserel は有効なポインタである必要があります。
  • rinfo->clause_relids には、対象の baserel が含まれている必要があります。

アルゴリズムまたは振る舞い

以下の条件をすべて満たす場合に true を返します。 1. rinfo が baserel を物理的に参照していること。 2. rinfo が外部結合（Outer Join）の条件である場合、その結合の外部側（outer side）へ移動しようとしていないこと。 3. rinfo 内の変数が、外部結合によって NULL 化される対象ではないこと（rinfo->clause_relids と baserel->nulling_relids が重複していないこと）。 4. baserel が LATERAL 参照を持つ場合、その参照先と rinfo が競合していないこと。 5. rinfo がクローン（is_clone）ではないこと。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/optimizer/path/indxpath.c
  • src/backend/optimizer/path/tidpath.c
  • src/backend/optimizer/util/orclauses.c
  • **呼び出し先**:
  • bms_is_member (Bitmapset 操作)
  • bms_overlap (Bitmapset 操作)

関連データ構造

  • RestrictInfo: 結合条件のメタデータを含む構造体。
  • RelOptInfo: リレーションの最適化情報を保持する構造体。

失敗モードと境界条件

  • rinfo が baserel を参照していない場合、直ちに false を返します。
  • rinfo がクローンである場合、移動不可として false を返します。
  • 外部結合の制約に抵触する場合、移動不可として false を返します。

保守担当者向けメモ

  • 本関数は、プランナが結合条件を適切なリレーションまでプッシュダウンできるかを判断する重要なガードレールです。
  • nulling_relids のチェックは、外部結合による NULL 伝播を考慮した安全な移動を保証するために不可欠です。
