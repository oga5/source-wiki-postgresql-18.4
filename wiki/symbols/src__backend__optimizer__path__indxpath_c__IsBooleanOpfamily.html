IsBooleanOpfamily

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/indxpath.c:2791

目的

指定された演算子族（opfamily）が、ブール型を扱うための演算子族であるかどうかを判定する。

シグネチャまたは形状

static bool IsBooleanOpfamily(Oid opfamily)

入力と出力

  • **入力**: opfamily (Oid) - 判定対象の演算子族のオブジェクト識別子。
  • **出力**: bool - 指定された演算子族がブール型をサポートしている場合は true、そうでない場合は false。

前提条件と不変条件

  • opfamily がシステムカタログ上の有効な演算子族 ID であることを前提とする。

アルゴリズムまたは振る舞い

1. opfamily が FirstNormalObjectId 未満（組み込みオブジェクト）である場合、IsBuiltinBooleanOpfamily(opfamily) を呼び出して判定する。 2. それ以外の場合、op_in_opfamily を使用し、BooleanEqualOperator が当該演算子族に含まれているかどうかを確認することで判定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/indxpath.c 内の複数箇所（行 156, 2731, 4374）から呼び出されている。
  • **呼び出し先**:
  • IsBuiltinBooleanOpfamily（組み込み判定時）
  • op_in_opfamily（ユーザー定義演算子族判定時）

関連データ構造

  • Oid: PostgreSQL のオブジェクト識別子。
  • BooleanEqualOperator: ブール型の等価演算子。

失敗モードと境界条件

  • opfamily が無効な OID である場合、op_in_opfamily の内部実装に依存する挙動となる。

保守担当者向けメモ

  • 本関数は、インデックスパス生成時に特定の演算子族がブール型を扱えるかを確認するために使用される。
  • IsBooleanOpfamily() が true を返す場合のみ、後続のブール型特有の処理（indxpath.c 内のコメントで言及されている処理）が実行される設計となっている。
