hemdistsign

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_intbig_gist.c:216

目的

2つのビットベクトル（シグネチャ）間のハミング距離を計算し、GiST インデックスにおけるノード間の類似度やコストを評価します。

シグネチャまたは形状

int hemdistsign(BITVECP a, BITVECP b, int siglen)

入力と出力

  • **入力**:
  • a, b: 比較対象のビットベクトル（BITVECP 型）。
  • siglen: ビットベクトルの長さ（バイト単位）。
  • **出力**:
  • int: 2つのビットベクトル間でビットが異なる箇所の総数（ハミング距離）。

前提条件と不変条件

  • a および b は、少なくとも siglen バイト以上の有効なメモリ領域を指している必要があります。
  • pg_number_of_ones 配列が、0〜255 の各値に対するビットカウントを保持している必要があります。

アルゴリズムまたは振る舞い

1. LOOPBYTE(siglen) マクロを使用して、ビットベクトルをバイト単位で走査します。 2. 各バイトに対して XOR (^) 演算を行い、ビットが異なる箇所を特定します。 3. XOR の結果（diff）をインデックスとして pg_number_of_ones テーブルを参照し、そのバイト内のセットビット数を加算します。 4. 全バイトの合計値を最終的なハミング距離として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_intbig_gist.c 内の g_intbig_picksplit 等から呼び出されます。
  • その他、contrib/hstore/hstore_gist.c、contrib/ltree/_ltree_gist.c、contrib/pg_trgm/trgm_gist.c からの呼び出しが確認されています。
  • **呼び出し先**:
  • pg_number_of_ones（グローバルなビットカウントテーブル）を参照します。

関連データ構造

  • BITVECP: ビットベクトルを表現するためのポインタ型。

失敗モードと境界条件

  • siglen に負の値や不正なサイズが渡された場合の動作は定義されていません。
  • メモリ境界外へのアクセスを防ぐため、呼び出し側で siglen の整合性を保証する必要があります。

保守担当者向けメモ

  • 本関数は popcount 命令を使用せず、ルックアップテーブル (pg_number_of_ones) を使用して実装されています。これは、特定の環境下での最適化判断によるものです。
  • 複数の contrib モジュールで共通利用されているため、変更時は他モジュールへの影響を考慮してください。
