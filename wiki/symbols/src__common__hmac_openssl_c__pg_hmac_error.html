pg_hmac_error

定義

  • 種別: function
  • 定義箇所: src/common/hmac_openssl.c:335

目的

HMAC 計算処理に関連するエラー状態を、人間が読み取れる形式の文字列（const char *）に変換して返します。

シグネチャまたは形状

const char *pg_hmac_error(pg_hmac_ctx *ctx);

入力と出力

  • **入力**: pg_hmac_ctx *ctx（HMAC コンテキストへのポインタ）。NULL を許容します。
  • **出力**: エラー内容を示す文字列。

前提条件と不変条件

  • ctx が NULL の場合、メモリ不足（"out of memory"）を示す文字列を返します。
  • ctx->errreason が設定されている場合、その内容が優先的に返されます。

アルゴリズムまたは振る舞い

1. ctx が NULL かを確認し、真であれば "out of memory" を返します。 2. ctx->errreason が存在すれば、その文字列を返します。 3. 上記以外の場合、ctx->error の値に基づいて以下のいずれかを返します。

  • PG_HMAC_ERROR_NONE: "success"
  • PG_HMAC_ERROR_DEST_LEN: "destination buffer too small"
  • PG_HMAC_ERROR_OPENSSL: "OpenSSL failure"

    • 4. 予期しないエラーコードが渡された場合は Assert(false) を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth-scram.c
  • src/common/hmac.c
  • src/common/scram-common.c
  • **呼び出し先**:
  • 提供されたソースコード内では、特定の関数呼び出しは行われていません。

関連データ構造

  • pg_hmac_ctx: HMAC 計算の状態を保持する構造体。エラーコード（error）およびエラー理由（errreason）を保持します。

失敗モードと境界条件

  • ctx が NULL の場合、安全に "out of memory" を返す設計となっています。
  • 定義されていないエラーコードが ctx->error に格納されている場合、Assert(false) により異常終了します。

保守担当者向けメモ

  • 本関数は OpenSSL のエラーキューを直接操作するのではなく、pg_hmac_ctx に記録された状態を翻訳する役割を担います。
  • 新しいエラータイプを追加する場合は、switch 文のケースを拡張してください。
