vacuumLeafRoot

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/spgist/spgvacuum.c:408

目的

SP-GiST インデックスのルートページ（リーフページでもある場合）から、不要になったライブタプルを削除し、インデックスの整合性を維持する。

シグネチャまたは形状

static void vacuumLeafRoot(spgBulkDeleteState *bds, Relation index, Buffer buffer)

入力と出力

  • **入力**:
  • bds: VACUUM 実行状態を保持する spgBulkDeleteState 構造体。
  • index: 対象のインデックスリレーション。
  • buffer: 処理対象のルートページを含むバッファ。
  • **出力**: なし（副作用としてページ内容の更新と統計情報の加算を行う）。

前提条件と不変条件

  • 対象ページは SP-GiST のルートページであること。
  • ルートページ内のすべてのタプルは SPGIST_LIVE 状態である必要がある（それ以外の状態はエラーとして扱われる）。

アルゴリズムまたは振る舞い

1. ページ内の全タプルを走査し、bds->callback を用いて削除対象か判定する。 2. 削除対象のタプルオフセットを toDelete 配列に蓄積し、統計情報（tuples_removed または num_index_tuples）を更新する。 3. 削除対象が存在する場合:

  • クリティカルセクションを開始する。
  • PageIndexMultiDelete を使用してページからタプルを物理的に削除する。
  • バッファをダーティとしてマークする。
  • WAL が必要な場合、XLOG_SPGIST_VACUUM_ROOT レコードを生成し、削除情報を記録する。
  • クリティカルセクションを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: spgvacuumpage（src/backend/access/spgist/spgvacuum.c 内の 646 行目付近で確認）。
  • **呼び出し先**:
  • BufferGetPage, PageGetMaxOffsetNumber, PageGetItem, PageGetItemId (ページ操作)
  • bds->callback (削除判定)
  • PageIndexMultiDelete (タプル削除)
  • MarkBufferDirty, RelationNeedsWAL, XLogBeginInsert, XLogRegisterData, XLogRegisterBuffer, XLogInsert, PageSetLSN (WAL 記録)

関連データ構造

  • spgBulkDeleteState: VACUUM の進捗とコールバック関数を保持。
  • spgxlogVacuumRoot: ルートページ削除用の WAL ログ構造体。
  • SpGistLeafTuple: ページ内に格納されているリーフタプル。

失敗モードと境界条件

  • **エラー**: ルートページ内に SPGIST_LIVE 以外の状態のタプルが存在する場合、elog(ERROR) を発生させる。
  • **境界条件**: 削除対象のタプルが存在しない場合、処理を即座に終了する。

保守担当者向けメモ

  • 本関数はルートページ専用の削除ロジックである。通常のリーフページ処理には vacuumLeafPage が使用される。
  • WAL 記録時に toDelete 配列の内容を登録しているため、xlrec.nDelete の整合性に注意すること。
