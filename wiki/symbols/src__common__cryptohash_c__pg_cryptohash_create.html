pg_cryptohash_create

定義

  • 種別: function
  • 定義箇所: src/common/cryptohash.c:73

目的

ハッシュ計算を行うためのコンテキスト構造体（pg_cryptohash_ctx）を生成および初期化する。

シグネチャまたは形状

pg_cryptohash_ctx *pg_cryptohash_create(pg_cryptohash_type type)

入力と出力

  • **入力**: type（使用するハッシュアルゴリズムを指定する pg_cryptohash_type 型の値）
  • **出力**: 初期化された pg_cryptohash_ctx へのポインタ。メモリ割り当てに失敗した場合は NULL を返す。

前提条件と不変条件

  • 割り当てられるメモリサイズは、サポートされている中で最大のハッシュアルゴリズム（SHA512）に合わせて固定されている。
  • 返されるコンテキストの type フィールドには指定されたアルゴリズムが記録され、error フィールドは PG_CRYPTOHASH_ERROR_NONE に設定される。

アルゴリズムまたは振る舞い

1. ALLOC マクロを使用して pg_cryptohash_ctx 構造体分のメモリを確保する。 2. 確保したメモリ領域を memset でゼロクリアする。 3. 構造体の type メンバに引数の type を代入する。 4. 構造体の error メンバを PG_CRYPTOHASH_ERROR_NONE に初期化する。 5. 初期化済みのコンテキストポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/uuid-ossp/uuid-ossp.c
  • src/backend/backup/backup_manifest.c
  • src/backend/libpq/auth-scram.c
  • src/backend/utils/adt/cryptohashfuncs.c
  • src/common/checksum_helper.c
  • **呼び出し先**:
  • ALLOC（環境に応じて palloc または malloc に展開される）
  • memset

関連データ構造

  • pg_cryptohash_ctx: ハッシュ計算の状態を保持する不透明な構造体。
  • pg_cryptohash_type: アルゴリズムの種類を定義する列挙型。

失敗モードと境界条件

  • メモリ不足により ALLOC が NULL を返した場合、関数は NULL を返す。

保守担当者向けメモ

  • 本関数は、アルゴリズムごとのメモリサイズの差異を考慮せず、常に最大サイズを確保する設計となっている。これはコードの複雑性を低減するための意図的な判断である。
