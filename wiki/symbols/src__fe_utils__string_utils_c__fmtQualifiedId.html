fmtQualifiedId

定義

  • 種別: function
  • 定義箇所: src/fe_utils/string_utils.c:295

目的

スキーマ名と識別子を組み合わせ、SQLクエリ内で安全に使用できる「schema.identifier」形式の文字列を生成します。

シグネチャまたは形状

const char *fmtQualifiedId(const char *schema, const char *id);

入力と出力

  • **入力**:
  • schema: スキーマ名（NULL可）。
  • id: 識別子名。
  • **出力**:
  • クォートおよびエスケープ処理済みの schema.id 形式の文字列へのポインタ。

前提条件と不変条件

  • 戻り値は内部の静的バッファを指すため、呼び出し直後に使用するか、内容をコピーする必要があります。
  • 内部バッファはスレッドセーフではありません。
  • 現在設定されているエンコーディング（getFmtEncoding()）に基づいて処理が行われます。

アルゴリズムまたは振る舞い

1. 現在のエンコーディング設定を getFmtEncoding() で取得します。 2. fmtQualifiedIdEnc を呼び出し、スキーマ名と識別子に対して適切なクォート処理（必要に応じたダブルクォートの付与）を実行します。 3. 生成された文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: fmtQualifiedIdEnc
  • **呼び出し元**:
  • src/bin/pg_dump/parallel.c
  • src/bin/pg_dump/pg_backup_archiver.c
  • src/bin/pg_dump/pg_dump.c

関連データ構造

  • PQExpBuffer（内部的に使用されるバッファ管理構造体）

失敗モードと境界条件

  • メモリ不足時には内部でエラーが発生し、プログラムが終了する可能性があります。
  • 戻り値のバッファは後続の fmtId 系関数の呼び出しによって上書きされるため、保持には注意が必要です。

保守担当者向けメモ

  • 本関数は fmtQualifiedIdEnc のラッパーです。エンコーディングを明示的に指定する必要がある場合は fmtQualifiedIdEnc を直接使用してください。
  • マルチスレッド環境で利用する場合は、スレッドローカルストレージへの移行や排他制御の検討が必要です。
