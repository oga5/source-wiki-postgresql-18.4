pg_queue_signal

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/signal.c:258

目的

Windows環境において、受信したシグナルをスレッドセーフにキューへ格納し、メインスレッドへ通知を行う。

シグネチャまたは形状

void pg_queue_signal(int signum)

入力と出力

  • **入力**: signum (int) - キューイング対象のシグナル番号。
  • **出力**: なし。

前提条件と不変条件

  • pgwin32_signal_event が初期化済みであること（Assertにより検証）。
  • pg_signal_crit_sec クリティカルセクションが初期化済みであること。

アルゴリズムまたは振る舞い

1. シグナル番号が有効範囲（0 < signum < PG_SIGNAL_COUNT）外であれば、何もせず終了する。 2. pg_signal_crit_sec を取得し、排他制御下で pg_signal_queue ビットマスクに該当シグナルをセットする。 3. クリティカルセクションを解放する。 4. SetEvent(pgwin32_signal_event) を呼び出し、待機中のメインスレッドに対してシグナル到着を通知する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/port/win32/signal.c 内のシグナルリスナースレッドおよびコンソールハンドラ。
  • src/backend/port/win32/timer.c (SIGALRM用)
  • src/backend/postmaster/postmaster.c (SIGCHLD用)
  • **呼び出し先**:
  • EnterCriticalSection / LeaveCriticalSection (Windows API)
  • SetEvent (Windows API)

関連データ構造

  • pg_signal_queue: シグナル状態を保持するビットマスク。
  • pg_signal_crit_sec: 同期用クリティカルセクション。
  • pgwin32_signal_event: 通知用Windowsイベントオブジェクト。

失敗モードと境界条件

  • signum が 0 または PG_SIGNAL_COUNT 以上の場合、関数は即座にリターンし、シグナルは無視される。
  • pgwin32_signal_event が NULL の場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数はシグナルスレッドとメインスレッド間で共有されるリソースを操作するため、必ず pg_signal_crit_sec による保護を維持すること。
  • SetEvent はイベントをシグナル状態にするため、メインスレッド側で適切にイベントをリセットする処理（pgwin32_dispatch_queued_signals 等）と対になっていることを確認すること。
