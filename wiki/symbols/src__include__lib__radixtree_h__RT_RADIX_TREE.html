RT_RADIX_TREE

定義

  • 種別: struct
  • 定義箇所: src/include/lib/radixtree.h:270

目的

適応型基数木（Adaptive Radix Tree: ART）のメイン構造体であり、キーと値のペアを階層的に保持・管理する。

シグネチャまたは形状

typedef struct RT_RADIX_TREE RT_RADIX_TREE;

入力と出力

  • 本構造体は、RT_CREATE 関数によって生成され、RT_RADIX_TREE 型のポインタとして返される。
  • 共有メモリモードでは dsa_area を、ローカルメモリモードでは MemoryContext を入力として初期化される。

前提条件と不変条件

  • キーは符号なし64ビット整数（uint64）に限定される。
  • マクロによるテンプレート展開を通じて生成されるため、特定のバリュー型に対して型安全性が保証される。

アルゴリズムまたは振る舞い

  • キーを8ビット単位（RT_SPAN）で分割し、階層的にノードを辿ることで検索や挿入を行う。
  • 子ノード数に応じて、4, 16, 48, 256スロットのノード構造を動的に切り替え、メモリ効率を最適化する。
  • 共有メモリ使用時は dsa_pointer を介してポインタ操作を行い、LWLock による排他制御が適用される。

呼び出し元と呼び出し先

  • **呼び出し元**: RT_CREATE, RT_ATTACH, RT_DETACH, RT_GET_HANDLE, RT_LOCK_EXCLUSIVE, RT_LOCK_SHARE, RT_UNLOCK, RT_FREE, RT_FIND, RT_SET などの各操作関数が本構造体を引数として受け取る。
  • **呼び出し先**: 構造体内部で RT_NODE や RT_CHILD_PTR を保持し、ノードの階層構造を管理する。

関連データ構造

  • RT_NODE: 木を構成する各ノード。
  • RT_HANDLE: 木へのハンドル。
  • RT_ITER: イテレーション用構造体。
  • dsa_area / MemoryContext: メモリ管理基盤。

失敗モードと境界条件

  • 共有メモリ環境下でのメモリ不足や、ロック競合による待機が発生する可能性がある。
  • キーの範囲は uint64 の全域をサポートするが、ノードの適応型サイズクラスの制限により、挿入時のノード成長処理が伴う。

保守担当者向けメモ

  • 本構造体はテンプレートの一部であり、RT_MAKE_NAME マクロによって生成される。
  • 将来的に「Optimistic Lock Coupling」への移行が検討されているため、並行制御の実装変更には注意が必要である。
  • RT_DEBUG を定義することで、ノード統計やダンプ機能を利用可能。
