AtPrepare_MultiXact

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/multixact.c:1985

目的

2相コミット（Two-Phase Commit）の準備フェーズにおいて、現在のトランザクションが保持している最も古い MultiXactId を永続化するための準備を行います。

シグネチャまたは形状

void AtPrepare_MultiXact(void)

入力と出力

  • **入力**: なし（内部的に MyOldestMemberMXactIdSlot() を介して現在のトランザクション状態を取得）。
  • **出力**: なし（成功時、RegisterTwoPhaseRecord を通じて 2相コミット用レコードを登録）。

前提条件と不変条件

  • 本関数は、トランザクションの準備（PREPARE TRANSACTION）処理中に呼び出される必要があります。
  • MultiXactIdIsValid() が真となる有効な MultiXactId が存在する場合のみ、レコードが登録されます。

アルゴリズムまたは振る舞い

1. MyOldestMemberMXactIdSlot() を呼び出し、現在のトランザクションに関連付けられた最も古い MultiXactId を取得します。 2. 取得した ID が有効（MultiXactIdIsValid）であるかを確認します。 3. 有効な場合、RegisterTwoPhaseRecord を使用して、リソースマネージャ ID TWOPHASE_RM_MULTIXACT_ID を指定し、当該 MultiXactId を 2相コミットのステートファイルに書き込むためのレコードとして登録します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c:2673（PrepareTransaction 内から呼び出される）。
  • **呼び出し先**:
  • MyOldestMemberMXactIdSlot()
  • MultiXactIdIsValid()
  • RegisterTwoPhaseRecord()

関連データ構造

  • MultiXactId: MultiXact を識別する 32 ビット ID。
  • TWOPHASE_RM_MULTIXACT_ID: 2相コミットにおける MultiXact 用のリソースマネージャ識別子。

失敗モードと境界条件

  • MyOldestMemberMXactIdSlot() が無効な ID を返す場合、本関数は何の処理も行わず正常終了します。

保守担当者向けメモ

  • 本関数は、準備されたトランザクションが再開またはコミットされた際に、MultiXact の整合性を維持するために不可欠です。
  • RegisterTwoPhaseRecord に渡されるデータサイズは sizeof(MultiXactId) で固定されています。
