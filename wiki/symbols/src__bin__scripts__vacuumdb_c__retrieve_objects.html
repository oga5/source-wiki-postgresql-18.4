retrieve_objects

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/vacuumdb.c:793

目的

PostgreSQLのシステムカタログをクエリし、vacuumdbの実行対象となるテーブルやスキーマのリストを生成する。

シグネチャまたは形状

static SimpleStringList *retrieve_objects(PGconn *conn, vacuumingOptions *vacopts, SimpleStringList *objects, bool echo)

入力と出力

  • **入力**:
  • conn: データベース接続。
  • vacopts: 実行オプション（min_xid_age、min_mxid_age、missing_stats_only等）。
  • objects: ユーザーが指定した対象オブジェクトのリスト。
  • echo: クエリ実行時にSQLを表示するかどうかのフラグ。
  • **出力**:
  • 処理対象として特定されたオブジェクトを格納した SimpleStringList 構造体へのポインタ。

前提条件と不変条件

  • objfilter グローバル変数（VacObjFilter）に基づき、クエリのフィルタリング条件が動的に決定される。
  • 一時テーブル（relpersistence == 't'）は常に除外される。
  • 統計情報の不足チェック（missing_stats_only）を行う場合、メインテーブルとTOASTテーブルの両方の統計情報を考慮する。

アルゴリズムまたは振る舞い

1. objects が指定されている場合、WITH 句（CTE）を使用して対象オブジェクトを一時テーブルとしてクエリに組み込む。 2. pg_class および pg_namespace を結合し、対象オブジェクトのOID、名前、スキーマ名を抽出する。 3. objfilter の設定に応じて、テーブル単位またはスキーマ単位でのフィルタリングを行う。 4. vacopts の条件（XID年齢、MXID年齢、統計情報の有無）に基づき、WHERE 句でクエリを絞り込む。 5. 構築したクエリを conn を通じて実行し、結果を SimpleStringList に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/scripts/vacuumdb.c 内のメイン処理フロー（main関数付近のロジックから呼び出される）。
  • **呼び出し先**: initPQExpBuffer、appendPQExpBufferStr、appendStringLiteralConn、splitTableColumnsSpec、PQclientEncoding 等のユーティリティ関数。

関連データ構造

  • vacuumingOptions: 実行オプションを保持。
  • SimpleStringList: 処理対象オブジェクトのリスト管理に使用。
  • PQExpBufferData: 動的なSQLクエリ構築に使用。

失敗モードと境界条件

  • データベース接続が切断されている場合、クエリ実行時に失敗する。
  • ユーザー指定のオブジェクト名がカタログ内に存在しない場合、クエリ結果が空になる可能性がある。
  • objfilter の設定と objects リストの内容に矛盾がある場合、意図しないクエリが生成される可能性がある。

保守担当者向けメモ

  • 本関数はSQLインジェクションを防ぐため、appendStringLiteralConn を使用して入力値をエスケープしている。変更時はこのセキュリティ設計を維持すること。
  • missing_stats_only のクエリは複雑なサブクエリを含むため、パフォーマンス上の懸念がある場合はカタログ統計の更新状況を確認すること。
