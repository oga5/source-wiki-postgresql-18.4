shm_toc_estimator

定義

  • 種別: struct
  • 定義箇所: src/include/storage/shm_toc.h:43

目的

動的共有メモリ（DSM）セグメント上に shm_toc を構築する際、必要な共有メモリの総サイズを事前に算出するためのヘルパー構造体です。

シグネチャまたは形状

typedef struct
{
	Size		space_for_chunks;
	Size		number_of_keys;
} shm_toc_estimator;

入力と出力

  • **入力**: 確保したいチャンクの合計サイズ（space_for_chunks）および、登録するキーの総数（number_of_keys）をフィールドに設定します。
  • **出力**: shm_toc_estimate(shm_toc_estimator *e) 関数に渡すことで、TOC 構造体自体と各エントリを格納するために必要な合計サイズ（Size）が算出されます。

前提条件と不変条件

  • space_for_chunks には、TOC が管理する各データ領域のサイズの合計を指定する必要があります。
  • number_of_keys には、TOC に登録するキーの総数を指定する必要があります。

アルゴリズムまたは振る舞い

1. 利用者は shm_toc_estimator 構造体をインスタンス化し、必要な領域サイズとキー数を設定します。 2. shm_toc_estimate 関数を呼び出すことで、共有メモリセグメントの確保に必要な総バイト数が計算されます。 3. この計算結果に基づき、shm_toc_create を用いて共有メモリ領域の初期化が行われます。

呼び出し元と呼び出し先

関連データ構造

  • shm_toc: 本構造体で見積もられたサイズに基づいて作成される、実際の目次管理構造体。

失敗モードと境界条件

  • キーの数が極端に多い場合、TOC 自体のオーバーヘッドが増大します。多数のオブジェクトを管理する場合は、TOC にはルート構造体のみを登録し、その配下で管理する設計が推奨されます。

保守担当者向けメモ

  • 本構造体は「一度確保したら解放しない」メモリ管理モデルを前提としています。
  • shm_toc_estimate の計算ロジックを変更する場合は、shm_toc_create 内のメモリ配置ロジックと整合性が取れていることを確認してください。
