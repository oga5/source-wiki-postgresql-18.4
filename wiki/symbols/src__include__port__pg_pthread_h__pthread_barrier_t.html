pthread_barrier_t

定義

  • 種別: struct
  • 定義箇所: src/include/port/pg_pthread.h:24

目的

POSIXスレッドのバリア同期機能が標準提供されていないプラットフォーム（主にmacOS）において、スレッド間の同期を実現するための代替実装を提供します。

シグネチャまたは形状

typedef struct pg_pthread_barrier
{
    bool            sense;      /* フェーズ切り替え用フラグ */
    int             count;      /* 待機対象のスレッド数 */
    int             arrived;    /* 到達済みスレッド数 */
    pthread_mutex_t mutex;      /* 排他制御用ミューテックス */
    pthread_cond_t  cond;       /* 到達待ち合わせ用条件変数 */
} pthread_barrier_t;

入力と出力

本構造体は、pthread_barrier_init、pthread_barrier_wait、pthread_barrier_destroy の各関数に対して、状態保持のための引数として渡されます。

前提条件と不変条件

  • HAVE_PTHREAD_BARRIER_WAIT が定義されていない環境でのみ使用されます。
  • 内部の mutex および cond は、POSIXスレッドの標準的なミューテックスおよび条件変数として初期化されている必要があります。

アルゴリズムまたは振る舞い

  • pthread_barrier_wait 呼び出し時、内部の mutex で排他制御を行い、arrived カウンタを更新します。
  • 指定された count に達するまで、スレッドは cond 上で待機します。
  • sense フラグにより、バリアの再利用（フェーズの切り替え）を管理します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照）**:
  • src/backend/storage/ipc/barrier.c
  • src/bin/pgbench/pgbench.c
  • src/port/pthread_barrier_wait.c
  • **呼び出し先**:
  • 内部的に <pthread.h> で定義された pthread_mutex_t および pthread_cond_t を使用します。

関連データ構造

  • pthread_mutex_t
  • pthread_cond_t

失敗モードと境界条件

  • count に達する前に pthread_barrier_destroy が呼び出された場合の動作は未定義です。
  • 内部のミューテックスや条件変数の初期化に失敗した場合、後続の wait 操作は正常に機能しません。

保守担当者向けメモ

  • 本構造体は軽量な同期を目的としています。複雑なスレッド管理が必要な場合は、他の同期プリミティブの検討が必要です。
  • src/port/pthread_barrier_wait.c に実装の詳細が含まれています。
