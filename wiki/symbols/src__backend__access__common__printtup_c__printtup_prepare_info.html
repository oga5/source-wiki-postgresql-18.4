printtup_prepare_info

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/printtup.c:249

目的

タプル出力処理において、各カラムのデータ変換に必要な関数情報（出力関数または送信関数）をキャッシュし、DR_printtup 状態構造体を初期化する。

シグネチャまたは形状

static void printtup_prepare_info(DR_printtup *myState, TupleDesc typeinfo, int numAttrs)

入力と出力

  • **入力**:
  • myState: DR_printtup 構造体へのポインタ。現在のポータル形式情報およびキャッシュ領域を保持。
  • typeinfo: タプルの属性定義を含む TupleDesc。
  • numAttrs: 処理対象のカラム数。
  • **出力**: なし（myState->myinfo にキャッシュ情報を書き込み、myState->attrinfo を更新する）。

前提条件と不変条件

  • myState->portal->formats が各カラムのフォーマット（テキスト: 0, バイナリ: 1）を保持していること。
  • numAttrs が 0 以下の場合は早期リターンし、キャッシュの初期化は行われない。

アルゴリズムまたは振る舞い

1. 既存の myState->myinfo が存在する場合は pfree で解放する。 2. numAttrs 分の PrinttupAttrInfo 配列を palloc0 で確保する。 3. 各カラムについて以下を反復する:

  • フォーマットコードが 0（テキスト）の場合: getTypeOutputInfo で出力関数を検索し、fmgr_info で関数呼び出し情報をキャッシュする。
  • フォーマットコードが 1（バイナリ）の場合: getTypeBinaryOutputInfo で送信関数を検索し、fmgr_info で関数呼び出し情報をキャッシュする。
  • それ以外の場合: 不正なフォーマットコードとしてエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/printtup.c 内のタプル送信処理フロー（初回実行時など）。
  • **呼び出し先**:
  • pfree, palloc0 (メモリ管理)
  • getTypeOutputInfo, getTypeBinaryOutputInfo (型情報検索)
  • fmgr_info (関数呼び出し情報の取得)
  • ereport (エラー処理)

関連データ構造

  • DR_printtup: printtup の状態を保持する構造体。
  • PrinttupAttrInfo: 各カラムの型変換関数やフォーマット情報を格納する構造体。
  • TupleDesc: タプルのメタデータ構造。

失敗モードと境界条件

  • numAttrs <= 0: 処理をスキップして正常終了する。
  • format が 0 または 1 以外: ERRCODE_INVALID_PARAMETER_VALUE でエラー終了する。

保守担当者向けメモ

  • 本関数は printtup の実行効率を左右するキャッシュ構築を担う。キャッシュの再構築が必要な場合は、必ず古い myinfo を解放してから再確保すること。
  • explain_dr.c において、本関数の処理の一部が類似の目的で使用されている点に注意。
