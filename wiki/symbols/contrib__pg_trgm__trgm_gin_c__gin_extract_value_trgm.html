gin_extract_value_trgm

定義

  • 種別: function
  • 定義箇所: contrib/pg_trgm/trgm_gin.c:34

目的

GINインデックス構築時、入力されたテキストデータからトリグラム（3文字の連続）を抽出し、インデックスキーとして利用可能な Datum 配列に変換する。

シグネチャまたは形状

Datum gin_extract_value_trgm(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_PP(0): インデックス対象のテキストデータ。
  • PG_GETARG_POINTER(1): 出力されるキーの個数を格納するための int32 型ポインタ。
  • **出力**:
  • 戻り値: 抽出されたトリグラムの Datum 配列（Datum *）。
  • 副作用: 第2引数 nentries に抽出されたトリグラムの総数が書き込まれる。

前提条件と不変条件

  • 入力テキストが有効な text 型であること。
  • nentries ポインタが有効なメモリ領域を指していること。
  • 抽出されたトリグラムが存在しない場合、nentries は 0 となり、戻り値は NULL となる。

アルゴリズムまたは振る舞い

1. generate_trgm を呼び出し、入力テキストからトリグラムのリスト（TRGM 構造体）を生成する。 2. 生成されたトリグラムの個数（trglen）を確認する。 3. trglen > 0 の場合、palloc を使用して Datum 配列を確保する。 4. trgm2int を使用して各トリグラムを int32 型の整数値に変換し、Datum 配列に格納する。 5. 最終的に Datum 配列のポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GIN インデックス構築プロセス（直接の呼び出し元関数名はソースコードの文脈からは特定不可）。
  • **呼び出し先**:
  • generate_trgm（trgm_op.c で定義）
  • trgm2int
  • palloc（PostgreSQL メモリ管理）

関連データ構造

  • TRGM: トリグラムのリストを保持する構造体。
  • Datum: PostgreSQL の汎用データ型。

失敗モードと境界条件

  • 入力テキストが空、またはトリグラムを生成できない短い文字列の場合、nentries は 0 に設定され、空のインデックスキーとして処理される。
  • メモリ不足時には palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は GIN 演算クラスの extractValue メソッドとして登録される。
  • trgm2int による変換は、トリグラムをインデックスのキーとして効率的に比較・格納するために不可欠である。
  • 類似度検索等の精度を維持するため、トリグラム生成ロジック（generate_trgm）の変更は本関数にも影響を与える点に留意すること。
