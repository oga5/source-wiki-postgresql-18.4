add_join_clause_to_rels

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/joininfo.c:97

目的

指定された RestrictInfo（結合条件）を、それが参照する各ベースリレーションの joininfo リストに追加し、プランナが結合順序を決定する際に利用可能な状態にする。

シグネチャまたは形状

void add_join_clause_to_rels(PlannerInfo *root, RestrictInfo *restrictinfo, Relids join_relids)

入力と出力

  • **入力**:
  • root: 現在のクエリのプランナ情報。
  • restrictinfo: 追加対象の結合条件。
  • join_relids: この条件が参照するリレーションの集合（ビットマップセット）。
  • **出力**: なし（副作用として各 RelOptInfo の joininfo リストを更新）。

前提条件と不変条件

  • restrictinfo が常に真（TRUE）と評価される場合、リストには追加されない。
  • restrictinfo が常に偽（FALSE）と評価される場合、定数 FALSE を持つ新しい RestrictInfo に置換される。この際、rinfo_serial および root->last_rinfo_serial は元の値を維持する必要がある。

アルゴリズムまたは振る舞い

1. restriction_is_always_true を確認し、真であれば即座に終了する。 2. restriction_is_always_false を確認し、真であれば make_restrictinfo を用いて定数 FALSE に置換する。このとき、シリアル番号の整合性を保つために rinfo_serial を退避・復元する。 3. join_relids に含まれる各リレーション ID を bms_next_member で順次取得する。 4. find_base_rel_ignore_join を用いて対応する RelOptInfo を取得し、存在する場合のみその joininfo リストに restrictinfo を追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/initsplan.c:3318
  • **呼び出し先**:
  • restriction_is_always_true / restriction_is_always_false
  • make_restrictinfo
  • bms_next_member
  • find_base_rel_ignore_join
  • lappend

関連データ構造

  • PlannerInfo: プランナのグローバル状態。
  • RestrictInfo: 結合条件を保持する構造体。
  • RelOptInfo: リレーションの最適化情報を保持する構造体。

失敗モードと境界条件

  • find_base_rel_ignore_join が NULL を返す場合（ベースリレーションではない場合）、そのリレーションへの追加はスキップされる。

保守担当者向けメモ

  • 本関数はベースリレーション（baserel）のみを対象としています。
  • RestrictInfo は複数のリレーションで共有されるため、メモリ管理に注意してください。
  • remove_join_clause_from_rels は本関数の逆操作を行います。
