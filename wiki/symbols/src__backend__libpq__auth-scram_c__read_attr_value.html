read_attr_value

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-scram.c:742

目的

SCRAMメッセージ文字列から、指定された属性キーに対応する値（key=value形式のvalue部分）を抽出します。

シグネチャまたは形状

static char *read_attr_value(char **input, char attr)

入力と出力

  • **入力**:
  • input: 解析対象の文字列へのポインタ。解析が進むと、次の属性の開始位置までポインタが更新されます。
  • attr: 検索対象の属性キー（1文字）。
  • **出力**:
  • 抽出された値の先頭アドレスを返します。

前提条件と不変条件

  • input が指す文字列は、SCRAMプロトコルに従った key=value 形式の連続である必要があります。
  • 属性値はカンマ（,）または文字列終端（\0）で終了している必要があります。

アルゴリズムまたは振る舞い

1. 現在のポインタ位置が指定された attr と一致するか確認し、一致しない場合はエラーを報告します。 2. 次の文字が = であることを確認し、一致しない場合はエラーを報告します。 3. = の直後から、, または終端文字が現れるまでを値として特定します。 4. 値の末尾に終端文字 \0 を挿入して文字列を分離します。 5. input ポインタを、次の属性の開始位置（, の次）へ進めます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-scram.c 内の以下の箇所から呼び出されます。
  • channel_binding_type の取得 (p 属性)
  • state->client_username の取得 (n 属性)
  • state->client_nonce の取得 (r 属性)
  • channel_binding の取得 (c 属性)
  • state->client_final_nonce の取得 (r 属性)
  • **呼び出し先**: sanitize_char（エラーメッセージ生成時）。

関連データ構造

  • scram_state: 認証セッションの状態を保持する構造体（呼び出し元でこの構造体のフィールドを更新するために使用）。

失敗モードと境界条件

  • **プロトコル違反**: 指定された属性キーが見つからない、または = が存在しない場合、ereport(ERROR, ...) により処理が中断されます。
  • **終端処理**: 文字列の末尾が属性値である場合、正しく終端文字を処理して input ポインタを更新します。

保守担当者向けメモ

  • 本関数はサーバー側の実装です。src/interfaces/libpq/fe-auth-scram.c にも同名の関数が存在しますが、シグネチャ（エラーバッファの有無）が異なるため混同しないよう注意してください。
  • 破壊的な変更（*end = '\0'）を行うため、渡される文字列は書き込み可能である必要があります。
