interval_cmp_lower

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistproc.c:314

目的

SplitInterval 構造体の lower 値に基づき、2つの間隔を比較するための比較関数です。主にソート処理の基準として使用されます。

シグネチャまたは形状

static int interval_cmp_lower(const void *i1, const void *i2)

入力と出力

  • **入力**:
  • i1, i2: SplitInterval 構造体へのポインタ（const void * として渡される）。
  • **出力**:
  • float8_cmp_internal の結果を返します。lower1 が lower2 より小さい場合は負、等しい場合は0、大きい場合は正の整数を返します。

前提条件と不変条件

  • 入力ポインタ i1 および i2 は、有効な SplitInterval 構造体を指している必要があります。
  • SplitInterval 構造体には lower フィールドが存在することが前提です。

アルゴリズムまたは振る舞い

1. 引数 i1 および i2 を SplitInterval 型のポインタにキャストします。 2. それぞれの lower フィールド（float8 型）を取得します。 3. float8_cmp_internal を呼び出し、取得した2つの値を比較してその結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistproc.c 内の分割ロジック（qsort 等の比較関数として使用）。
  • **呼び出し先**:
  • float8_cmp_internal
  • **注記**: src/backend/utils/adt/rangetypes_gist.c に同名の関数が存在しますが、シグネチャが異なるため、本シンボルとは別の実装である可能性があります。

関連データ構造

  • SplitInterval: gistproc.c 内で定義されている、分割処理に使用される間隔情報を保持する構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、未定義の動作を引き起こします。
  • float8 の比較において NaN が含まれる場合の挙動は float8_cmp_internal の仕様に依存します。

保守担当者向けメモ

  • 本関数は qsort 等の標準的な比較関数インターフェースに適合するように設計されています。変更を加える際は、ソート順序の整合性に注意してください。
