namene

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/name.c:156

目的

2つの name 型引数が等しくないかどうかを判定し、その結果を真偽値として返します。

シグネチャまたは形状

Datum namene(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを介して2つの Name 型引数を受け取ります。
  • **出力**: 2つの引数が等しくない場合は true、等しい場合は false を Datum 型として返します。

前提条件と不変条件

  • 引数は PG_GETARG_NAME マクロで取得可能である必要があります。
  • 比較には現在の照合順序（Collation）が使用されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_NAME(0) および PG_GETARG_NAME(1) を使用して、2つの Name 型引数を取得します。 2. namecmp 関数を呼び出し、現在の照合順序（PG_GET_COLLATION()）に基づいて2つの引数を比較します。 3. namecmp の戻り値が 0 でない場合（等しくない場合）、true を返し、0 の場合（等しい場合）は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL レベルの演算子（<>）から呼び出されます。
  • **呼び出し先**: namecmp 関数を呼び出します。

関連データ構造

  • Name: PostgreSQL のシステムカタログ等で使用される固定長（NAMEDATALEN）の文字列型。

失敗モードと境界条件

  • namecmp の仕様に従い、NAMEDATALEN を超える部分は比較対象外となります。
  • 照合順序の設定によっては、比較結果がロケールに依存します。

保守担当者向けメモ

  • 本関数は name 型の不等価比較を実装する標準的なラッパーです。ロジックの変更が必要な場合は、比較の根幹である namecmp を確認してください。
