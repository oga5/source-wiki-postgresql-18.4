ExecInitTidRangeScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTidrangescan.c:358

目的

TID範囲スキャン（TID Range Scan）を実行するための実行ノード TidRangeScanState を初期化し、スキャン対象のリレーションのオープンおよび式評価の準備を行う。

シグネチャまたは形状

TidRangeScanState *ExecInitTidRangeScan(TidRangeScan *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 実行対象の TidRangeScan プランノード。
  • estate: クエリ実行状態を保持する EState。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの TidRangeScanState 構造体へのポインタ。

前提条件と不変条件

  • node は TidRangeScan 型の有効なプランノードであること。
  • estate は初期化済みであること。
  • 内部状態 trss_inScan は false に設定され、TID範囲は未計算の状態から開始される。

アルゴリズムまたは振る舞い

1. TidRangeScanState 構造体を割り当て、プランノードと実行状態をリンクする。 2. ExecProcNode 関数ポインタに ExecTidRangeScan を割り当てる。 3. ExecAssignExprContext を呼び出し、式評価用のコンテキストを作成する。 4. ExecOpenScanRelation を使用してスキャン対象のリレーションをオープンし、状態構造体に格納する。 5. ExecInitScanTupleSlot を呼び出し、リレーションの記述子に基づいてタプルスロットを初期化する。 6. ExecInitResultTypeTL および ExecAssignScanProjectionInfo を呼び出し、結果型と射影情報を設定する。 7. ExecInitQual を呼び出し、プランノードの qual（フィルタ条件）を初期化する。 8. TidExprListCreate を呼び出し、TID範囲計算に必要な式リストを構築する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内の ExecInitNode 等から呼び出される。
  • **呼び出し先**:
  • makeNode, ExecAssignExprContext, ExecOpenScanRelation, ExecInitScanTupleSlot, ExecInitResultTypeTL, ExecAssignScanProjectionInfo, ExecInitQual, TidExprListCreate

関連データ構造

  • TidRangeScanState: 本関数が初期化するメインの実行状態構造体。
  • TidRangeScan: プランノード。
  • EState: 実行状態。
  • Relation: スキャン対象のリレーション記述子。

失敗モードと境界条件

  • リレーションのオープン失敗やメモリ割り当て失敗時には、PostgreSQL の標準的なエラーハンドリングにより例外が発生する。
  • ss.ss_currentScanDesc は本関数内では NULL に初期化される（テーブルスキャン自体はここでは開始されない）。

保守担当者向けメモ

  • 本関数はノードの初期化のみを担当する。実際のTID範囲の評価は実行時の TidRangeEval で行われるため、初期化段階では範囲の計算は行われない。
  • table_slot_callbacks を使用して Table AM に依存したスロット初期化を行っている点に注意すること。
