LtreeSignature

定義

  • 種別: struct
  • 定義箇所: contrib/ltree/_ltree_gist.c:411

目的

GiST インデックスにおいて、ltree 配列の包含関係を高速に判定するためのビットベクトルシグネチャとその長さを保持する。

シグネチャまたは形状

typedef struct LtreeSignature
{
	BITVECP		sign;
	int			siglen;
} LtreeSignature;

入力と出力

  • sign: BITVECP 型（ビットベクトルへのポインタ）。各パスのハッシュ値がマッピングされる。
  • siglen: int 型。ビットベクトルの長さ（バイト単位）。

前提条件と不変条件

  • siglen はインデックスオプションにより決定される。
  • ビットベクトルはハッシュ衝突を許容する近似的な表現であるため、この構造体を用いた判定結果は常に「再検証（recheck）」を必要とする。

アルゴリズムまたは振る舞い

  • _ltree_compress 等の関数内で、配列内の各 ltree パスから計算された CRC32 ハッシュをビットベクトルにセットするために使用される。
  • hemdist 関数において、2つの LtreeSignature 間のハミング距離を計算する際のデータコンテナとして機能する。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照）:
  • contrib/ltree/_ltree_gist.c
  • contrib/ltree/ltree_gist.c
  • 呼び出し先:
  • 提供されたソース情報からは、特定の関数呼び出し関係は特定できません。

関連データ構造

  • BITVECP: ビットベクトルを指す型。
  • ltree_gist: ltree 型の GiST インデックス用格納形式。

失敗モードと境界条件

  • 構造体内の sign が指すメモリ領域が siglen に基づいて適切に確保されていない場合、メモリ破壊や不正なビットアクセスが発生する可能性がある。

保守担当者向けメモ

  • 本構造体は _ltree_gist.c（配列用）と ltree_gist.c（単一用）の両方で定義・利用されている。修正時は両方のファイルでの整合性に注意すること。
  • siglen の動的変更がインデックスの再構築なしにどのように扱われるかについては、PostgreSQL の GiST インデックスの仕様と ltree モジュールの設定管理を確認すること。
