like_escape

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like.c:429

目的

SQLの LIKE 演算子で使用されるパターン文字列に対し、指定されたエスケープ文字を適用してエスケープ処理を行う。

シグネチャまたは形状

Datum like_escape(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_PP(0): パターン文字列 (text *)
  • PG_GETARG_TEXT_PP(1): エスケープ文字を含む文字列 (text *)
  • **出力**:
  • PG_RETURN_TEXT_P(result): エスケープ処理後の新しい text オブジェクト

前提条件と不変条件

  • 入力されるパターンおよびエスケープ文字列は、PostgreSQLの text 型として有効であること。
  • データベースのエンコーディング設定に基づき、適切な処理関数が選択される。

アルゴリズムまたは振る舞い

1. データベースの最大文字長 (pg_database_encoding_max_length) を確認する。 2. シングルバイトエンコーディングの場合は SB_do_like_escape を呼び出す。 3. マルチバイトエンコーディングの場合は MB_do_like_escape を呼び出す。 4. 処理結果を text 型として返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（直接的な呼び出し元コードの詳細は提供されていない）。
  • **呼び出し先**:
  • SB_do_like_escape (シングルバイト用)
  • MB_do_like_escape (マルチバイト用)

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • PG_FUNCTION_ARGS: SQL関数呼び出し規約のための引数構造体。

失敗モードと境界条件

  • 入力文字列が不正なエンコーディングである場合、内部の SB_do_like_escape または MB_do_like_escape がエラーを発生させる可能性がある。
  • メモリ不足時には、text 型の確保時にエラーが発生する。

保守担当者向けメモ

  • 本関数は単なるディスパッチャであり、実際の変換ロジックは SB_do_like_escape および MB_do_like_escape に依存している。
  • like_match.c 内のコメントによれば、本関数はパターン内のエスケープ文字を適切に変換する役割を担う。
