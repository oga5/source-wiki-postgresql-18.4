InternalIpcMemoryCreate

定義

  • 種別: function
  • 定義箇所: src/backend/port/sysv_shmem.c:120

目的

System V 共有メモリセグメントを新規作成し、プロセスのアドレス空間にアタッチする。

シグネチャまたは形状

static void *InternalIpcMemoryCreate(IpcMemoryKey memKey, Size size)

入力と出力

  • **入力**:
  • memKey: 共有メモリセグメントを識別するためのキー。
  • size: 確保する共有メモリのサイズ（バイト単位）。
  • **出力**:
  • 成功時: アタッチされた共有メモリ領域の先頭アドレス（void *）。
  • 失敗時: 既存セグメントとの衝突時は NULL を返し、それ以外の致命的なエラー時は ereport(FATAL, ...) によりプロセスを終了する。

前提条件と不変条件

  • IPC_CREAT | IPC_EXCL フラグを使用してセグメントを作成するため、指定したキーで既にセグメントが存在する場合は衝突とみなされる。
  • EXEC_BACKEND ビルド環境では、環境変数 PG_SHMEM_ADDR を介してアタッチ先アドレスの指定が可能。

アルゴリズムまたは振る舞い

1. **アドレス決定**: EXEC_BACKEND 定義時は環境変数またはプラットフォーム固有のデフォルト値からアタッチ先アドレスを決定する。 2. **セグメント作成**: shmget を呼び出し、セグメントを生成する。

  • EINVAL が発生した場合、一部の BSD 系カーネルの挙動を考慮し、サイズ 0 で再試行して衝突の有無を確認する。
  • 衝突（EEXIST, EACCES, EIDRM）が確認された場合は NULL を返す。
  • その他のエラーはカーネルパラメータ（SHMMAX, SHMALL 等）の制限とみなし、FATAL エラーとして報告する。

    • 3. **終了時処理の登録**: on_shmem_exit を使用し、プロセス終了時に IpcMemoryDelete（削除）および IpcMemoryDetach（デタッチ）が実行されるよう登録する。 4. **アタッチ**: shmat を呼び出し、メモリをプロセス空間にマッピングする。失敗時は FATAL エラーとする。 5. **メタデータ記録**: 作成したキーと ID をデータディレクトリのロックファイルに書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/port/sysv_shmem.c 内の PGSharedMemoryCreate から呼び出される。
  • **呼び出し先**: shmget, shmctl, shmat, on_shmem_exit, AddToDataDirLockFile 等のシステムコールおよび内部関数。

関連データ構造

  • IpcMemoryKey: 共有メモリの識別キー。
  • IpcMemoryId: shmget が返す共有メモリ識別子。

失敗モードと境界条件

  • **衝突**: 既に同名のセグメントが存在する場合、NULL を返して呼び出し元に制御を戻す。
  • **リソース制限**: カーネルの SHMMAX, SHMALL, SHMMNI 制限を超過した場合、FATAL エラーで停止する。
  • **アタッチ失敗**: shmat が失敗した場合、FATAL エラーで停止する。

保守担当者向けメモ

  • 本関数は shmget のエラーコードを詳細に解析し、プラットフォームごとの差異（特に BSD 系）を吸収するロジックを含んでいる。
  • EXEC_BACKEND 環境におけるアドレス指定はデバッグ目的の回避策であるため、将来的な GUC 等への移行が望ましい。
