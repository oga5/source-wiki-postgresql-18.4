missing_cache_key

定義

  • 種別: struct
  • 定義箇所: src/backend/access/common/heaptuple.c:95

目的

属性のデフォルト値（missing value）をキャッシュするためのハッシュテーブルにおいて、検索キーとして使用される構造体です。

シグネチャまたは形状

typedef struct
{
	int			len;
	Datum		value;
} missing_cache_key;

入力と出力

  • **入力**: len（属性の長さ）、value（属性値そのもの）。
  • **出力**: ハッシュテーブルのキーとして機能し、特定の属性値に対応するキャッシュエントリを特定するために使用されます。

前提条件と不変条件

  • hash_ctl.keysize および hash_ctl.entrysize の設定において、本構造体のサイズ（sizeof(missing_cache_key)）が使用されます。
  • ハッシュテーブルの検索および比較処理において、この構造体へのキャストを介してアクセスされます。

アルゴリズムまたは振る舞い

  • ハッシュテーブルのキーとして、属性の長さと値を保持します。
  • 属性値のキャッシュ検索時に、このキーを用いて既存のエントリと照合を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/heaptuple.c 内のハッシュテーブル操作関数群（missing_hash、missing_match、init_missing_cache 等）から参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • hash_ctl: ハッシュテーブルの初期化に使用される制御構造体。

失敗モードと境界条件

  • 構造体のサイズがハッシュテーブルの定義と一致しない場合、メモリ破壊や検索エラーが発生する可能性があります。
  • Datum 型の特性上、値がポインタである場合はその有効期間に依存します。

保守担当者向けメモ

  • 本構造体は heaptuple.c 内のキャッシュ機構専用です。変更を加える際は、対応するハッシュ関数および比較関数（missing_hash, missing_match）の整合性を確認してください。
