PathHashStack

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonb_gin.c:73

目的

JSONBのパスハッシュ値を保持するためのスタック構造体です。主にjsonb_path_opsインデックスにおいて、ネストされたJSONB構造を走査する際のパスのハッシュ計算過程を追跡するために使用されます。

シグネチャまたは形状

typedef struct PathHashStack
{
    uint32                hash;
    struct PathHashStack *parent;
} PathHashStack;

入力と出力

  • **入力**: palloc を用いてメモリ確保され、hash メンバに計算されたハッシュ値が格納されます。
  • **出力**: parent ポインタを介して、スタックの親要素（一つ外側の階層のハッシュ）を参照します。

前提条件と不変条件

  • parent は、より外側の階層の PathHashStack を指すか、スタックの底を示す NULL である必要があります。
  • hash は、現在のパス階層までのハッシュ計算結果を保持します。

アルゴリズムまたは振る舞い

JSONBドキュメントの走査中に、現在の階層のハッシュ値を保持したまま、新しい階層へ進む際にスタックとして連結されます。これにより、再帰的な構造を持つJSONBデータに対して、パスのハッシュ値を階層的に管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonb_gin.c 内のインデックス抽出処理（gin_extract_jsonb_path 等）において、スタックの初期化および要素の追加が行われます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • JsonbIterator: JSONBドキュメントの走査に使用されます。
  • gin_extract_jsonb_path: 本構造体を利用してインデックスキーを生成する関数です。

失敗モードと境界条件

  • メモリ不足: palloc によるメモリ確保時にエラーが発生する可能性があります。
  • スタックオーバーフロー: JSONBのネストが極端に深い場合、スタックの深さが制限を超える可能性があります（check_stack_depth() による保護が想定されます）。

保守担当者向けメモ

  • 本構造体は jsonb_path_ops インデックスの構築に特化しています。
  • parent ポインタによる連結リスト構造であるため、走査終了後のメモリ解放が適切に行われているか確認してください。
