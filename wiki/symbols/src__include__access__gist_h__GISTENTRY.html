GISTENTRY

定義

  • 種別: struct
  • 定義箇所: src/include/access/gist.h:161

目的

GiST（Generalized Search Tree）インデックス内の各エントリを表現し、キー値とその配置場所（リレーション、ページ、オフセット）を保持するための構造体です。

シグネチャまたは形状

typedef struct GISTENTRY
{
    Datum       key;
    Relation    rel;
    Page        page;
    OffsetNumber offset;
    bool        leafkey;
} GISTENTRY;

入力と出力

本構造体は、GiST インデックスのサポート関数（Consistent, Union, Compress, Decompress, Penalty, PickSplit 等）において、インデックスエントリの情報を渡すための引数として使用されます。

前提条件と不変条件

  • key はインデックス対象のデータ（Datum）を保持します。
  • page はエントリが格納されているページへのポインタです。
  • leafkey は、そのエントリがリーフノードにあるかどうかを示すフラグです。

アルゴリズムまたは振る舞い

GiST インデックスの操作において、インデックスエントリの属性をカプセル化します。各サポート関数は、この構造体を通じてインデックス内の特定のタプル情報にアクセスし、検索や挿入、分割処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist モジュール内の各ファイル（btree_bit.c, btree_bool.c, btree_bytea.c 等）において、PG_GETARG_POINTER を介して引数として取得されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • Relation: インデックスが属するリレーション情報。
  • Page: インデックスページへのポインタ。
  • OffsetNumber: ページ内のタプル位置。
  • GIST_SPLITVEC: ページ分割時に使用される関連構造体。

失敗モードと境界条件

  • page が無効なポインタである場合や、offset がページ内の有効な範囲外である場合、インデックス操作中に不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は GiST インデックスの公開 API の一部です。変更を加える際は、既存のすべてのサポート関数および contrib 以下の拡張モジュールへの影響を考慮してください。
