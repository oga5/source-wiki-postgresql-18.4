PgFdwAnalyzeState

定義

  • 種別: struct
  • 定義箇所: contrib/postgres_fdw/postgres_fdw.c:257

目的

外部テーブルに対する ANALYZE 操作中に、統計情報収集のための状態を保持するために使用されます。

シグネチャまたは形状

typedef struct PgFdwAnalyzeState
{
    Relation    rel;                /* 外部テーブルの relcache エントリ */
    AttInMetadata *attinmeta;       /* 属性データ型変換メタデータ */
    List       *retrieved_attrs;    /* クエリで取得する属性番号のリスト */

    /* サンプリングされた行 */
    HeapTuple  *rows;               /* targrows サイズの配列 */
    int         targrows;           /* サンプル行の目標数 */
    int         numrows;            /* 収集済みのサンプル行数 */

    /* ランダムサンプリング用 */
    double      samplerows;         /* フェッチした行数 */
    double      rowstoskip;         /* 次のサンプルまでにスキップする行数 */
    ReservoirStateData rstate;      /* リザーバサンプリングの状態 */

    /* 作業用メモリコンテキスト */
    MemoryContext anl_cxt;          /* ANALYZE 期間中のデータ用コンテキスト */
    MemoryContext temp_cxt;         /* タプルごとの一時データ用コンテキスト */
} PgFdwAnalyzeState;

入力と出力

本構造体は関数間で状態を共有するためのコンテナであり、直接的な入力引数や戻り値はありません。analyze_row_processor などの関数において、サンプリング処理の進捗管理やデータ変換のメタデータ保持に使用されます。

前提条件と不変条件

  • rows 配列は targrows に基づいて確保される必要があります。
  • anl_cxt および temp_cxt は、それぞれ ANALYZE のライフサイクルおよびタプル処理のライフサイクルに合わせて適切に管理される必要があります。

アルゴリズムまたは振る舞い

外部テーブルから行をサンプリングする際、リザーバサンプリング（rstate）を用いて統計情報を収集します。samplerows や rowstoskip を更新しながら、リモートから取得したデータを attinmeta を介してローカルのタプル形式へ変換し、rows 配列に蓄積します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/postgres_fdw.c 内の ANALYZE 関連処理においてインスタンス化されます。
  • **呼び出し先**: analyze_row_processor 関数が本構造体を引数として受け取り、行の処理を行います。

関連データ構造

  • Relation: テーブルのメタデータ保持に使用。
  • AttInMetadata: リモートから取得したデータの型変換に使用。
  • ReservoirStateData: 統計情報収集のためのサンプリングアルゴリズムの状態保持に使用。
  • MemoryContext: メモリ管理に使用。

失敗モードと境界条件

  • targrows を超える行の収集や、メモリコンテキストの枯渇が発生する可能性があります。
  • リモートサーバーからのデータ取得失敗時、attinmeta による変換処理が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は postgres_fdw の ANALYZE 処理に特化しています。統計情報の精度やサンプリングロジックを変更する場合は、rstate の更新ロジックと temp_cxt の解放タイミングに注意してください。
