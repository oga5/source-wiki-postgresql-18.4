text_to_stavalues

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/attribute_stats.c:716

目的

外部から提供されたテキスト形式の統計データを、PostgreSQLの属性型に適合する配列データへ変換および検証します。

シグネチャまたは形状

static Datum text_to_stavalues(const char *staname, FmgrInfo *array_in, Datum d, Oid typid, int32 typmod, bool *ok)

入力と出力

  • **入力**:
  • staname: 統計項目名（エラーメッセージ用）。
  • array_in: 型変換用の関数情報。
  • d: 変換対象のテキストデータ。
  • typid, typmod: 変換先のデータ型と型修飾子。
  • **出力**:
  • 戻り値: 変換後の配列データ（Datum）。
  • ok: 変換および検証が成功したかを示すフラグ。

前提条件と不変条件

  • d はテキスト形式のデータである必要があります。
  • 変換後のデータは1次元配列である必要があります。
  • 変換後の配列に NULL 値が含まれていてはなりません。

アルゴリズムまたは振る舞い

1. TextDatumGetCString を用いて入力データを文字列に変換します。 2. FunctionCallInvoke を呼び出し、指定された array_in 関数を使用して型変換を実行します。 3. 変換中にエラーが発生した場合は、ErrorSaveContext を通じて WARNING を発行し、ok を false に設定して終了します。 4. 変換結果が1次元配列でない場合、または配列内に NULL が含まれる場合は、WARNING を発行し、ok を false に設定して終了します。 5. 検証を通過した場合、ok を true に設定し、変換後の配列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/attribute_stats.c 内の各統計更新処理（most_common_vals, histogram_bounds, most_common_elems, range_bounds_histogram, range_length_histogram の各設定箇所）。
  • **呼び出し先**: FunctionCallInvoke（型変換関数）、TextDatumGetCString、array_contains_nulls、ereport 等。

関連データ構造

  • FmgrInfo: 関数呼び出し情報を保持する構造体。
  • ErrorSaveContext: エラー発生時の詳細情報を保持するコンテキスト。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • **変換失敗**: 型変換関数がエラーを返した場合、WARNING として報告され、処理は中断されます。
  • **次元数不一致**: 1次元配列以外が生成された場合、WARNING が発行されます。
  • **NULL値**: 配列内に NULL が含まれる場合、WARNING が発行されます。

保守担当者向けメモ

  • 本関数は ANALYZE 以外の経路から統計情報をインポートする際のバリデーション層として機能します。
  • エラー発生時に ereport ではなく WARNING を使用しているため、呼び出し側は ok フラグを確認して後続の処理を制御する必要があります。
