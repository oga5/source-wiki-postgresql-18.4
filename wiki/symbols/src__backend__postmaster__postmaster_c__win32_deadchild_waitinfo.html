win32_deadchild_waitinfo

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/postmaster.c:466

目的

Windows環境において、終了した子プロセスの情報を保持し、waitpid() 相当の処理を行うために使用されるデータ構造です。

シグネチャまたは形状

typedef struct
{
	HANDLE		waitHandle;
	HANDLE		procHandle;
	DWORD		procId;
} win32_deadchild_waitinfo;

入力と出力

  • 入力: 本構造体は palloc を用いてメモリ確保され、終了した子プロセスのハンドルおよびIDを格納します。
  • 出力: キューに格納された後、waitpid() 処理を行う関数によって消費されます。

前提条件と不変条件

  • Windows環境でのみ使用される構造体です。
  • waitHandle および procHandle は有効なWindowsハンドルである必要があります。

アルゴリズムまたは振る舞い

1. 子プロセスの終了を検知した際、本構造体のインスタンスが生成されます。 2. 生成されたオブジェクトはキューにポストされ、waitpid() 処理を行うルーチンによって参照されます。 3. 処理ルーチンは、キューから本構造体を取り出し、保持されているハンドル情報を用いて終了処理を完了させます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/postmaster.c 内の終了処理ロジック（palloc による生成およびキューへのポスト）。
  • 呼び出し先: waitpid() 相当の処理を行う関数（キューからの取り出しおよび参照）。

関連データ構造

  • HANDLE (Windows API)
  • DWORD (Windows API)

失敗モードと境界条件

  • メモリ不足: palloc が失敗した場合、プロセス終了処理が正常に行われない可能性があります。
  • ハンドル無効: waitHandle や procHandle が無効な場合、終了処理が正しく完了しない可能性があります。

保守担当者向けメモ

  • 本構造体はWindows固有のプロセス管理ロジックに深く依存しています。プラットフォーム固有の挙動を変更する際は、src/backend/postmaster/postmaster.c 内のキュー操作ロジックと併せて確認してください。
