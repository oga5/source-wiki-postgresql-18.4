list_second_cell

定義

  • 種別: static inline function
  • 定義箇所: src/include/nodes/pg_list.h:141

目的

List 構造体から、2番目の要素を保持する ListCell へのポインタを取得します。

シグネチャまたは形状

ListCell *list_second_cell(const List *l)

入力と出力

  • **入力**: const List *l （対象のリストへのポインタ）
  • **出力**: リストの要素数が2以上であれば2番目の要素の ListCell* を返し、それ以外（リストが NULL または要素数が2未満）の場合は NULL を返します。

前提条件と不変条件

  • リストが空（NIL）の場合、l は NULL として扱われます。
  • リストの要素数は l->length に格納されています。

アルゴリズムまたは振る舞い

1. 引数 l が有効であり、かつ l->length が2以上であることを確認します。 2. 条件を満たす場合、内部配列 elements のインデックス1（2番目の要素）のアドレスを返します。 3. 条件を満たさない場合、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_coerce.c:1353
  • src/backend/utils/adt/jsonpath_exec.c:3565
  • **呼び出し先**: 本関数は直接的な関数呼び出しを行わず、List 構造体のメンバアクセスのみを行います。

関連データ構造

  • List: リストのヘッダ構造体。length と elements 配列を保持します。
  • ListCell: リスト内の各要素を保持する共用体。

失敗モードと境界条件

  • l が NULL の場合、安全に NULL を返します。
  • 要素数が0または1のリストに対して呼び出した場合、NULL を返します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結しています。
  • リストの要素数判定には l->length を使用しているため、リストの整合性が保たれていることが前提となります。
