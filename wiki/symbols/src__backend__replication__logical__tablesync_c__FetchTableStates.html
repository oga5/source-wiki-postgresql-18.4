FetchTableStates

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/tablesync.c:1607

目的

サブスクリプションに関連付けられたテーブルのうち、同期が完了していない（READY 状態ではない）テーブルのリストをキャッシュへ再構築し、サブスクリプションがテーブルを保持しているかを確認する。

シグネチャまたは形状

static bool FetchTableStates(bool *started_tx)

入力と出力

  • **入力**:
  • started_tx: トランザクションを本関数内で開始したかどうかを格納するフラグへのポインタ。
  • **出力**:
  • 戻り値: サブスクリプションがテーブルを保持している場合は true、そうでない場合は false。
  • *started_tx: 本関数内で StartTransactionCommand() が実行された場合に true が設定される。

前提条件と不変条件

  • table_states_validity が SYNC_TABLE_STATE_VALID 以外の場合にのみ、キャッシュの再構築が行われる。
  • キャッシュされたテーブル状態は CacheMemoryContext に永続化される。

アルゴリズムまたは振る舞い

1. table_states_validity を確認し、無効であれば再構築を開始する。 2. トランザクション状態でない場合は StartTransactionCommand() を呼び出し、started_tx を true に設定する。 4. 取得したリストを CacheMemoryContext にコピーし、table_states_not_ready を更新する。 5. table_states_not_ready が空であっても、HasSubscriptionRelations を用いてサブスクリプション自体にテーブルが存在するかを再確認する。 6. 再構築完了後、table_states_validity を SYNC_TABLE_STATE_VALID に戻す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/tablesync.c 内の複数箇所（行135, 434, 1773）。
  • **呼び出し先**:
  • list_free_deep, IsTransactionState, StartTransactionCommand, GetSubscriptionRelations, MemoryContextSwitchTo, palloc, lfirst, lappend, HasSubscriptionRelations。

関連データ構造

  • SubscriptionRelState: 同期状態を保持する構造体。
  • table_states_validity: キャッシュの有効性を管理する列挙型変数。

失敗モードと境界条件

  • トランザクション外で呼び出された場合、内部でトランザクションを開始する。
  • キャッシュ再構築中に無効化が発生した場合、構築中のリストを返すが、状態は無効のまま維持される（次回の再構築を促す）。

保守担当者向けメモ

  • 本関数は CacheMemoryContext を操作するため、メモリリークやコンテキストの切り替えミスに注意すること。
  • table_states_validity の状態遷移は、並行処理における整合性を保つための重要なガードである。
