check_copy_file_range

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/file.c:399

目的

実行環境が copy_file_range システムコールによるファイル転送をサポートしているかを検証する診断関数です。

シグネチャまたは形状

void check_copy_file_range(void)

入力と出力

  • **入力**: なし（グローバル変数 old_cluster および new_cluster を参照）
  • **出力**: なし（成功時は正常終了、失敗時は pg_fatal によりプロセスを終了）

前提条件と不変条件

  • old_cluster.pgdata および new_cluster.pgdata が有効なパスとして設定されていること。
  • 実行環境で HAVE_COPY_FILE_RANGE が定義されている場合、copy_file_range システムコールが正常に動作すること。

アルゴリズムまたは振る舞い

1. 旧クラスタの PG_VERSION ファイルを読み取り専用で開きます。 2. 新クラスタのデータディレクトリ内に一時ファイル PG_VERSION.copy_file_range_test を作成します。 3. HAVE_COPY_FILE_RANGE が定義されている場合、copy_file_range を呼び出して一時ファイルへのデータ転送を試みます。 4. 転送が失敗した場合は pg_fatal でエラーを報告します。 5. 最後に一時ファイルを削除します。 6. HAVE_COPY_FILE_RANGE が定義されていない環境では、即座に pg_fatal を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/check.c:726
  • **呼び出し先**: open, unlink, close, copy_file_range (システムコール)、pg_fatal (エラーハンドリング)

関連データ構造

  • old_cluster, new_cluster: pg_upgrade.h で定義されるクラスタ情報を保持する構造体。

失敗モードと境界条件

  • ファイルのオープン失敗、作成失敗、または copy_file_range の実行失敗時に pg_fatal が発生します。
  • HAVE_COPY_FILE_RANGE が未定義のプラットフォームでは、常に失敗します。

保守担当者向けメモ

  • 本関数は pg_upgrade の事前チェックフェーズで使用されます。
  • copy_file_range のサポート状況はプラットフォームのビルド設定に依存するため、移植性を考慮する場合は HAVE_COPY_FILE_RANGE マクロの定義を確認してください。
