make_incremental_rfile

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/reconstruct.c:455

目的

増分バックアップファイルからヘッダー情報を読み込み、再構築用の rfile 構造体を初期化および検証する。

シグネチャまたは形状

static rfile *make_incremental_rfile(char *filename)

入力と出力

  • **入力**: filename (読み込む増分バックアップファイルのパス)
  • **出力**: 初期化済みの rfile 構造体へのポインタ。

前提条件と不変条件

  • ファイルは make_rfile によって既にオープンされている必要がある。
  • ファイルの先頭には INCREMENTAL_MAGIC が格納されている必要がある。
  • num_blocks および truncation_block_length は RELSEG_SIZE 以下でなければならない。

アルゴリズムまたは振る舞い

1. make_rfile を呼び出し、基本構造体を生成する。 2. マジックナンバーを読み込み、INCREMENTAL_MAGIC と一致するか検証する。 3. ブロック数 (num_blocks) と切り詰めブロック長 (truncation_block_length) を読み込み、それぞれが RELSEG_SIZE を超えていないか確認する。 4. ブロックが存在する場合、relative_block_numbers 配列をメモリ確保し、ファイルから読み込む。 5. ヘッダー全体の長さを計算し、ブロック内容が BLCKSZ に整列するようにパディングを加算する。

関連データ構造

  • rfile: 再構築対象ファイルのメタデータとファイル記述子を保持する構造体。

失敗モードと境界条件

  • **マジックナンバー不一致**: pg_fatal によりプロセスが終了する。
  • **サイズ超過**: num_blocks または truncation_block_length が RELSEG_SIZE を超える場合、pg_fatal により終了する。
  • **メモリ不足**: pg_malloc0 失敗時に終了する可能性がある。

保守担当者向けメモ

  • ヘッダーの整列処理は、ブロックデータが BLCKSZ 境界から開始されることを保証するために重要である。
  • num_blocks が 0 の場合、relative_block_numbers のメモリ確保はスキップされる。
