xidrecv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xid.c:55

目的

外部バイナリ形式のデータを読み込み、PostgreSQLの内部的なトランザクション識別子（TransactionId）型へ変換します。

シグネチャまたは形状

Datum xidrecv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される StringInfo 型のバッファ（PG_GETARG_POINTER(0)）。
  • **出力**: TransactionId 型の値を Datum 形式に変換して返却します。

前提条件と不変条件

  • 入力バッファは libpq/pqformat.h で定義されたバイナリ形式に従っている必要があります。
  • TransactionId のサイズは sizeof(TransactionId) に依存します。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から StringInfo バッファを取得します。 2. pq_getmsgint を呼び出し、バッファから sizeof(TransactionId) バイト分のデータを読み取ります。 3. 読み取った値を TransactionId 型にキャストします。 4. PG_RETURN_TRANSACTIONID マクロを使用して、結果を Datum として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（fmgr）経由で、SQLレベルのバイナリ入力操作から呼び出されます。
  • **呼び出し先**:
  • pq_getmsgint（libpq/pqformat.h）: バイナリデータのデコードに使用。

関連データ構造

  • StringInfo: ネットワーク経由のバイナリデータ保持用構造体。
  • TransactionId: PostgreSQLのトランザクション識別子。

失敗モードと境界条件

  • 入力バッファが TransactionId のサイズに満たない場合、pq_getmsgint 内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は xid 型のバイナリ入力関数です。同様の処理を行う xidin（文字列入力）とは異なるパスを辿ります。
  • TransactionId の定義変更やバイナリプロトコルの変更時には、本関数の読み取りサイズを確認してください。
