initialize_reloptions

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/reloptions.c:590

目的

システム全体で利用可能なリレーションオプション（ストレージパラメータ）の定義テーブルを統合・初期化し、実行時に効率的にアクセス可能な単一の配列 relOpts を構築します。

シグネチャまたは形状

static void initialize_reloptions(void)

入力と出力

  • **入力**: なし（静的なオプション定義配列 boolRelOpts, intRelOpts, realRelOpts, enumRelOpts, stringRelOpts および custom_options を参照）。
  • **出力**: なし（グローバル変数 relOpts を更新し、need_initialization フラグを false に設定）。

前提条件と不変条件

  • 各オプション定義の lockmode が DoLockModesConflict を満たすこと（自己矛盾がないこと）。
  • relOpts が既に割り当てられている場合は pfree で解放されるため、メモリ管理の整合性が保たれていること。

アルゴリズムまたは振る舞い

1. 各種オプション配列（bool, int, real, enum, string）を走査し、総数をカウントします。 2. num_custom_options を加算し、必要なメモリ量を算出します。 3. TopMemoryContext 上に relopt_gen * 型の配列を確保します。 4. 各オプションのメタデータ（型、名前の長さ）を補完しながら、確保した配列にポインタを格納します。 5. 最後に NULL ポインタを末尾に配置してリストを終了させます。 6. need_initialization を false に更新し、初期化完了を通知します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/reloptions.c 内の 1528 行目および 2153 行目から呼び出されています。
  • **呼び出し先**: DoLockModesConflict（マクロまたは関数）、pfree、MemoryContextAlloc、strlen を使用しています。

関連データ構造

  • relopt_gen: オプションの基底構造体。
  • relOpts: 初期化後に格納されるグローバルなオプション定義リスト。
  • boolRelOpts 等の各型別静的配列。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAlloc がエラーを発生させます。
  • オプション定義の各配列が NULL で終端されていない場合、ループ処理が範囲外アクセスを引き起こす可能性があります。

保守担当者向けメモ

  • 新しいオプション型を追加する場合は、本関数のループ処理に該当する型を追加し、relOpts[j]->type の設定を更新する必要があります。
  • TopMemoryContext を使用しているため、プロセスの生存期間中保持されるメモリとして扱われます。
