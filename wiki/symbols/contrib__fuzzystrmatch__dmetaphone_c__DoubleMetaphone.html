DoubleMetaphone

定義

  • 種別: function
  • 定義箇所: contrib/fuzzystrmatch/dmetaphone.c:394

目的

入力文字列から発音に基づいた「プライマリ」および「オルタネート（代替）」の 2 種類のコードを生成し、名前の曖昧検索や機能インデックスの構築を支援します。

シグネチャまたは形状

static void DoubleMetaphone(char *str, char **codes)

入力と出力

  • **入力**: str (変換対象の文字列)、codes (結果を格納する 2 要素の文字列配列へのポインタ)。
  • **出力**: なし（codes を通じてプライマリおよびオルタネートのコードを返却）。

前提条件と不変条件

  • 入力文字列は NewMetaString を通じて metastring 構造体に変換され、末尾にパディングが付与されます。
  • primary および secondary の各コードは、最大 4 文字まで生成されます。
  • primary->free_string_on_destroy および secondary->free_string_on_destroy は 0 に設定され、メモリ管理が制御されます。

アルゴリズムまたは振る舞い

1. 入力文字列を大文字に変換し、初期化を行います。 2. 特定の接頭辞（GN, KN, PN, WR, PS）や 'X' で始まる場合の特殊ルールを適用します。 3. 文字列を走査し、switch 文で現在の文字に応じた発音ルールを適用して primary および secondary バッファにコードを追加します。 4. primary または secondary のいずれかの長さが 4 に達するまでループを継続します。 5. 最終的に生成されたコードを codes 配列に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/fuzzystrmatch/dmetaphone.c 内の SQL エントリポイント関数（dmetaphone および dmetaphone_alt 等）から呼び出されます。
  • **呼び出し先**: NewMetaString, MetaphAdd, MakeUpper, StringAt, GetAt, IsVowel 等のヘルパー関数を呼び出します。

関連データ構造

  • metastring: 文字列バッファと現在の長さを保持する構造体。

失敗モードと境界条件

  • 入力文字列が空の場合や、特定の文字パターンに一致しない場合は、デフォルトのルールに従って処理されます。
  • primary および secondary の長さが 4 に達した時点でループが終了します。
  • current インデックスが文字列長を超えないよう境界チェックが行われます。

保守担当者向けメモ

  • 本関数は Perl の Text::DoubleMetaphone モジュールの移植版です。
  • dmetaphone と dmetaphone_alt の両方が常に両方のコードを計算する仕様であり、パフォーマンス上のボトルネックとなる可能性があります。
  • メモリ管理は PostgreSQL のメモリコンテキストに依存しており、palloc/repalloc が内部的に使用されています。
