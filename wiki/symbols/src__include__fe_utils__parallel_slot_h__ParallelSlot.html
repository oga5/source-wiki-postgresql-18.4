ParallelSlot

定義

  • 種別: struct
  • 定義箇所: src/include/fe_utils/parallel_slot.h:21

目的

PostgreSQLのフロントエンドツールにおいて、並列処理用のデータベース接続（PGconn）を管理し、各接続に対するクエリ実行と結果処理のコールバックを紐付けるためのコンテナとして機能します。

シグネチャまたは形状

typedef struct ParallelSlot
{
    PGconn     *connection;
    bool        inUse;
    ParallelSlotResultHandler handler;
    void       *handler_context;
} ParallelSlot;

入力と出力

  • **入力**:
  • connection: PGconn オブジェクトへのポインタ。
  • handler: クエリ結果を処理するための ParallelSlotResultHandler 型の関数ポインタ。
  • handler_context: ハンドラ関数に渡される任意のコンテキストデータ。
  • **出力**:
  • inUse: スロットが現在使用中であるかを示すブール値。

前提条件と不変条件

  • connection に対してコマンドやクエリを発行する前に、必要に応じて handler および handler_context を登録する必要があります。
  • 未設定の場合、handler および handler_context は NULL である必要があります。

アルゴリズムまたは振る舞い

1. **接続保持**: 単一の PGconn を保持します。 2. **状態管理**: inUse フラグにより、スロットが現在利用可能か否かを管理します。 3. **コールバック実行**: クエリ発行時に登録された handler は、結果が返された際に呼び出され、handler_context を介して状態を共有します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_amcheck/pg_amcheck.c: run_command 関数内で使用。
  • src/bin/pg_dump/parallel.c: GetMyPSlot や RunWorker 等で並列ワーカーの接続管理に使用。
  • **呼び出し先**:
  • libpq-fe.h: PGconn 型の定義および関連する接続処理。

関連データ構造

  • ParallelSlotArray: 複数の ParallelSlot を保持するコンテナ。
  • ParallelSlotResultHandler: ParallelSlot に登録される結果処理用コールバックの型定義。

失敗モードと境界条件

  • handler が設定されていない場合、結果処理のロジックが欠落する可能性があります。
  • handler 内での PQclear の実行有無については、呼び出し元とハンドラ間での責務分担が求められます。

保守担当者向けメモ

  • 本構造体は ParallelSlotArray の一部として管理されることが前提です。
  • Windows環境では FD_SETSIZE の制限が並列接続数に影響を与えるため、スロット数設計時には注意してください。
