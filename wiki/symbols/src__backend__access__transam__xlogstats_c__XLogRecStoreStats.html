XLogRecStoreStats

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogstats.c:53

目的

解析済みのWALレコードから統計情報を抽出し、XLogStats 構造体へ集計します。リソースマネージャ（rmgr）単位およびレコードタイプ単位でのレコード数、データ長、FPI（Full Page Image）長の累積を行います。

シグネチャまたは形状

void XLogRecStoreStats(XLogStats *stats, XLogReaderState *record)

入力と出力

  • **入力**:
  • stats: 統計情報を格納する XLogStats 構造体へのポインタ。
  • record: 解析済みのWALレコード状態を保持する XLogReaderState 構造体へのポインタ。
  • **出力**: なし（stats が指す構造体の内容が更新されます）。

前提条件と不変条件

  • stats および record は NULL であってはなりません（Assert により検証されます）。
  • record は XLogReaderState として有効な状態である必要があります。

アルゴリズムまたは振る舞い

1. **全体集計**: stats->count をインクリメントします。 2. **レコード長算出**: XLogRecGetLen を呼び出し、レコードのデータ長（rec_len）とFPI長（fpi_len）を取得します。 3. **Rmgr単位の集計**: XLogRecGetRmid で取得した RmgrId をキーとして、stats->rmgr_stats 配列の該当エントリにレコード数、データ長、FPI長を加算します。 4. **レコードタイプ単位の集計**:

  • XLogRecGetInfo で取得した xl_info の上位4ビットを recid として抽出します。
  • RM_XACT_ID（トランザクション関連）の場合は、recid に対して 0x07 のビットマスクを適用し、opcodeを正規化します。
  • 算出した rmid と recid をインデックスとして、stats->record_stats 二次元配列の該当エントリに各値を加算します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_walinspect/pg_walinspect.c:785
  • src/bin/pg_waldump/pg_waldump.c:1287
  • **呼び出し先**:
  • XLogRecGetRmid
  • XLogRecGetLen
  • XLogRecGetInfo

関連データ構造

  • XLogStats: 統計情報を保持する構造体（src/include/access/xlogstats.h で定義）。
  • XLogReaderState: WALレコードの解析状態を保持する構造体。
  • RmgrId: リソースマネージャを識別するID。

失敗モードと境界条件

  • stats または record に NULL が渡された場合、アサーション違反によりプロセスが停止します。
  • RM_XACT_ID 以外のレコードにおいて、xl_info の上位4ビットが想定外の範囲にある場合、record_stats 配列の境界外アクセスが発生する可能性があります（各リソースマネージャが適切に xl_info を管理していることが前提となります）。

保守担当者向けメモ

  • 本関数は XLogRecGetLen の実装に依存しており、リーダー側のAPI変更時には追従が必要です。
  • record_stats のインデックス計算は xl_info の上位4ビットに依存しているため、新しいリソースマネージャを追加する際は、このビット領域の利用ルールを遵守する必要があります。
