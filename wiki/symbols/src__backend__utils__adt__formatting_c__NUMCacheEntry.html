NUMCacheEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/formatting.c:403

目的

数値フォーマット文字列の解析結果をキャッシュし、再利用することで変換処理のオーバーヘッドを削減するために使用されます。

シグネチャまたは形状

typedef struct
{
	FormatNode	format[NUM_CACHE_SIZE + 1];
	char		str[NUM_CACHE_SIZE + 1];
	bool		valid;
	int			age;
	NUMDesc		Num;
} NUMCacheEntry;

入力と出力

  • 本構造体自体はデータ保持用であり、関数呼び出しの引数や戻り値として直接渡されることはありません。
  • NUM_cache_fetch(const char *str) 等のキャッシュ検索関数が、この構造体へのポインタを返却します。

前提条件と不変条件

  • format 配列には、解析済みの FormatNode ツリーが格納されます。
  • str には、対応するフォーマット文字列が保持されます。
  • valid が true の場合、キャッシュエントリは有効な解析結果を保持しています。
  • age はキャッシュの置換アルゴリズム（LRU等）で使用される経過時間または優先度を示します。

アルゴリズムまたは振る舞い

  • 数値変換処理（TO_CHAR や TO_NUMBER）において、フォーマット文字列が指定された際、まず NUM_cache_search を通じて既存の NUMCacheEntry を探索します。
  • キャッシュに存在しない場合、NUM_cache_getnew を呼び出して新しいエントリを確保し、parse_format で解析した結果を格納します。
  • NUMCache 配列内のエントリは、必要に応じて MemoryContextAllocZero を用いて TopMemoryContext 上に動的に割り当てられます。

呼び出し元と呼び出し先

  • **呼び出し元:**
  • src/backend/utils/adt/formatting.c 内のキャッシュ管理関数群（NUM_cache_getnew, NUM_cache_search, NUM_cache_fetch）。
  • **呼び出し先:**
  • 本構造体はデータ保持用であり、特定の関数を呼び出すことはありません。

関連データ構造

  • FormatNode: フォーマット文字列の解析結果を保持するノード。
  • NUMDesc: 数値フォーマットの属性（精度、符号、ロケール設定など）を保持する構造体。
  • NUMCache: NUMCacheEntry を格納する静的な配列。

失敗モードと境界条件

  • NUM_CACHE_SIZE を超えるフォーマット文字列が指定された場合、キャッシュの範囲外となるため、キャッシュ機構は利用されません。
  • メモリ不足時には MemoryContextAllocZero がエラーを発生させる可能性があります。

保守担当者向けメモ

  • NUMCacheEntry のサイズは、キャッシュ効率を考慮して調整されるべきです。
  • キャッシュの置換ロジックを変更する場合は、age フィールドの更新メカニズムを NUM_cache_getnew 内で確認してください。
