eqsel_internal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/selfuncs.c:243

目的

等価演算子（=）または不等価演算子（<>）を用いた制限句の選択率を推定する。

シグネチャまたは形状

static double eqsel_internal(PG_FUNCTION_ARGS, bool negate)

入力と出力

  • **入力**:
  • PG_FUNCTION_ARGS: プランナ情報、演算子OID、引数リスト、リレーションIDを含む関数引数。
  • negate: true の場合、不等価演算子（<>）として処理する。
  • **出力**:
  • double: 0.0 から 1.0 の範囲で算出された選択率。

前提条件と不変条件

  • negate が true の場合、元の演算子に対応する否定演算子（get_negator）が存在する必要がある。存在しない場合はデフォルト値が返される。
  • 入力式が 変数 = 定数 または 定数 = 変数 の形式（あるいはそれに準ずるもの）でない場合、デフォルトの選択率が返される。

アルゴリズムまたは振る舞い

1. **否定演算子の処理**: negate が true の場合、get_negator を用いて演算子を反転させる。 2. **変数情報の抽出**: get_restriction_variable を呼び出し、統計情報（VariableStatData）と対象の式を特定する。失敗した場合はデフォルト値を返す。 3. **選択率の計算**:

  • other（変数以外の側）が Const 型の場合、var_eq_const を呼び出して詳細な統計情報に基づく推定を行う。
  • それ以外の場合、var_eq_non_const を呼び出して推定を行う。

    • 4. **リソース解放**: ReleaseVariableStats を呼び出し、取得した統計情報を解放する。 5. **結果返却**: 計算された選択率を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/selfuncs.c 内の eqsel（negate=false）および neqsel（negate=true）から呼び出される。
  • **呼び出し先**:
  • get_negator, get_restriction_variable, var_eq_const, var_eq_non_const, ReleaseVariableStats。

関連データ構造

  • VariableStatData: 統計情報取得時に使用されるメタデータ構造体。
  • PlannerInfo: クエリプランナのコンテキスト情報。
  • Const: 定数ノード。

失敗モードと境界条件

  • 演算子の否定形が見つからない場合、1.0 - DEFAULT_EQ_SEL を返す。
  • 変数情報の抽出に失敗した場合、negate の状態に応じて DEFAULT_EQ_SEL または 1.0 - DEFAULT_EQ_SEL を返す。

保守担当者向けメモ

  • 本関数は eqsel と neqsel の共通ロジックを保持している。
  • Const ノードはクエリ内の単純な定数だけでなく、推定過程で生成された定数も含まれる可能性がある点に注意すること。
