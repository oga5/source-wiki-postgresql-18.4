GinLeader

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gin/gininsert.c:110

目的

GINインデックスの並列構築において、リーダープロセスが管理する状態を保持するための構造体です。並列構築の調整、共有メモリへのアクセス、およびワーカープロセスの管理を担います。

シグネチャまたは形状

typedef struct GinLeader
{
    ParallelContext *pcxt;
    int             nparticipanttuplesorts;
    GinBuildShared *ginshared;
    Sharedsort     *sharedsort;
    Snapshot        snapshot;
    WalUsage       *walusage;
    BufferUsage    *bufferusage;
} GinLeader;

入力と出力

  • **入力**: 並列構築開始時に palloc0 を介してメモリ確保され、各フィールドが初期化されます。
  • **出力**: _gin_end_parallel 関数等に渡され、並列処理の終了処理やリソース解放に使用されます。

前提条件と不変条件

  • nparticipanttuplesorts は、正常に起動したワーカープロセスの数に、リーダー自身がワーカーとして参加する場合は1を加えた値となります。
  • ginshared および sharedsort は、リーダーがTOC（Table of Contents）ルックアップを回避するための共有状態への直接ポインタとして機能します。

アルゴリズムまたは振る舞い

  • 並列構築のライフサイクル全体を通じて、リーダープロセスが共有メモリ状態（GinBuildShared）やソート状態（Sharedsort）を追跡するために使用されます。
  • _gin_end_parallel 関数において、並列処理のシャットダウンおよびリソースのクリーンアップの制御に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: ginbuild（並列構築パス）においてインスタンス化されます。
  • **呼び出し先**: _gin_end_parallel 関数が本構造体を引数として受け取り、並列処理の終了処理を実行します。

関連データ構造

  • GinBuildShared: インデックス構築全体の共有状態。
  • Sharedsort: tuplesort によって管理される共有ソート状態。
  • GinBuildState: 本構造体を保持し、構築プロセス全体の管理を行う構造体。

失敗モードと境界条件

  • DISABLE_LEADER_PARTICIPATION が設定されている場合、リーダーはワーカーとして参加せず、nparticipanttuplesorts の計算に影響を与えます。
  • 並列コンテキスト（ParallelContext）の初期化やワーカーの起動に失敗した場合、本構造体を用いた終了処理が適切に行われる必要があります。

保守担当者向けメモ

  • 本構造体は並列構築の複雑な同期ロジックの中核です。ginshared へのアクセスは、リーダーがTOCルックアップを避けるための最適化であることを理解してください。
  • walusage および bufferusage は、並列構築中の統計情報を集約するために使用されます。
