PGTYPESnumeric_cmp

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/numeric.c:1280

目的

2つの numeric 型変数を比較し、その大小関係を整数値で返します。

シグネチャまたは形状

int PGTYPESnumeric_cmp(numeric *var1, numeric *var2)

入力と出力

  • **入力**: 比較対象となる numeric 構造体へのポインタ var1 および var2。
  • **出力**:
  • var1 > var2 の場合: 正の整数
  • var1 < var2 の場合: 負の整数
  • var1 == var2 の場合: 0
  • エラー発生時: INT_MAX

前提条件と不変条件

  • var1 および var2 は有効な numeric 構造体を指している必要があります。
  • 内部的に cmp_abs 関数を使用して絶対値の比較を行います。

アルゴリズムまたは振る舞い

1. **両者が正**: cmp_abs(var1, var2) の結果を返します。 2. **両者が負**: cmp_abs(var2, var1) の結果を返します（負数同士の比較における逆転を考慮）。 3. **異符号**: var1 が正で var2 が負なら 1 を、逆なら -1 を返します。 4. **エラー**: 上記のいずれにも該当しない（符号が不正な）場合、errno に PGTYPES_NUM_BAD_NUMERIC を設定し、INT_MAX を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test2.c
  • **呼び出し先**:
  • cmp_abs（同ファイル内の静的関数）

関連データ構造

  • numeric: 数値を保持する構造体（sign フィールドで正負を判定）。

失敗モードと境界条件

  • var1 または var2 の符号が NUMERIC_POS または NUMERIC_NEG 以外の場合、errno が PGTYPES_NUM_BAD_NUMERIC に設定され、INT_MAX が返されます。

保守担当者向けメモ

  • 本関数は numeric 構造体の符号フラグに依存しています。符号の定義に変更があった場合は注意してください。
  • cmp_abs は絶対値の比較を行うため、符号の判定ロジックと分離されています。
