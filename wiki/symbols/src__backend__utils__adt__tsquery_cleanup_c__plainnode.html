plainnode

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_cleanup.c:61

目的

NODE 構造体で表現されたクエリツリーを、フラットな QueryItem 配列形式へ再帰的に変換（シリアライズ）します。

シグネチャまたは形状

static void plainnode(PLAINTREE *state, NODE *node);

入力と出力

  • **入力**:
  • state: 変換状態を保持する PLAINTREE 構造体（出力先配列、現在のインデックス、配列サイズを管理）。
  • node: 変換対象の NODE 構造体。
  • **出力**: なし（state 内の ptr 配列を更新）。

前提条件と不変条件

  • state->ptr は QueryItem の配列として確保されている必要があります。
  • 再帰呼び出しを行うため、スタックオーバーフローを防止する check_stack_depth() が呼び出されます。
  • 処理終了後、引数 node は pfree() により解放されます。

アルゴリズムまたは振る舞い

1. スタック深さをチェックします。 2. state->ptr の容量が不足している場合、repalloc でサイズを倍増させます。 3. node->valnode を state->ptr の現在位置にコピーします。 4. ノードの型に応じて分岐します：

  • QI_VAL (値ノード): インデックスをインクリメントして終了。
  • OP_NOT (NOT演算子): left フラグを立て、右ノードに対して再帰的に plainnode を呼び出します。
  • その他 (二項演算子): 現在のインデックスを保持し、右ノード、左ノードの順に再帰呼び出しを行います。その後、left フィールドに左右のノード間の距離を格納します。

    • 5. 最後に node をメモリ解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery_cleanup.c:106 (plaintree 関数内から呼び出し)
  • **呼び出し先**:
  • check_stack_depth()
  • repalloc()
  • pfree()
  • plainnode() (自身を再帰呼び出し)

関連データ構造

  • NODE: クエリツリーのノード構造体。
  • PLAINTREE: 変換中の状態を管理する構造体。
  • QueryItem: tsquery のフラットな要素構造体。

失敗モードと境界条件

  • スタック深さが制限を超えた場合、check_stack_depth() によりエラーが発生します。
  • メモリ不足時には repalloc() がエラーをスローします。

保守担当者向けメモ

  • 本関数は再帰的にツリーを走査するため、非常に深いクエリツリーが入力された場合にスタックオーバーフローのリスクがあります。
  • OP_NOT 以外の演算子における left フィールドの計算（state->cur - cur）は、フラットな配列上でのツリー構造の復元に不可欠です。変更時はこの距離計算の整合性に注意してください。
