proclist_pop_head_node_offset

定義

  • 種別: function
  • 定義箇所: src/include/storage/proclist.h:172

目的

共有メモリ上の双方向連結リストから先頭ノードを取り出し、対応する PGPROC 構造体へのポインタを返します。

シグネチャまたは形状

static inline PGPROC *proclist_pop_head_node_offset(proclist_head *list, size_t node_offset)

入力と出力

  • **入力**:
  • list: 操作対象の proclist_head 構造体へのポインタ。
  • node_offset: PGPROC 構造体内の proclist_node メンバへのオフセット（offsetof で取得）。
  • **出力**:
  • 戻り値: リストから削除された先頭ノードに対応する PGPROC 構造体へのポインタ。

前提条件と不変条件

  • リストが空であってはなりません（proclist_is_empty(list) が真である場合、Assert により停止します）。
  • node_offset は、リスト内の各 PGPROC が保持する proclist_node の正しいオフセットである必要があります。

アルゴリズムまたは振る舞い

1. リストが空でないことを確認します。 2. list->head（先頭プロセスの番号）を使用して、GetPGProcByNumber により対象の PGPROC 構造体を取得します。 3. proclist_delete_offset を呼び出し、リストから当該ノードを削除します。 4. 取得した PGPROC 構造体へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/storage/proclist.h 内で直接参照されています。
  • **呼び出し先**:
  • proclist_is_empty
  • GetPGProcByNumber
  • proclist_delete_offset

関連データ構造

  • proclist_head: リストの先頭と末尾を管理する構造体。
  • PGPROC: プロセス情報を保持する構造体。
  • proclist_node: リストの前後関係を保持するノード構造体。

失敗モードと境界条件

  • リストが空の状態で呼び出した場合、アサーション違反が発生します。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、共有メモリ上のデータ構造を安全に操作するために procno を介した間接参照を行っています。
  • 呼び出し時には必ず offsetof(PGPROC, member_name) を使用して、正しいノードメンバを指定してください。
