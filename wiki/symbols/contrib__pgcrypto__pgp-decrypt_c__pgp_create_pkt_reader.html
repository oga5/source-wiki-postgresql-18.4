pgp_create_pkt_reader

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-decrypt.c:222

目的

PGP パケットの読み取りを行うための PullFilter を生成し、パケット固有のメタデータ（タイプと長さ）を管理下に置く。

シグネチャまたは形状

int pgp_create_pkt_reader(PullFilter **pf_p, PullFilter *src, int len, int pkttype, PGP_Context *ctx);

入力と出力

  • **入力**:
  • pf_p: 生成されたフィルタを格納するポインタ。
  • src: 読み取り元のソースフィルタ。
  • len: パケットの長さ。
  • pkttype: パケットのタイプ。
  • ctx: PGP コンテキスト（本関数内では直接使用されないがシグネチャに含まれる）。
  • **出力**:
  • 成功時は 0 を返し、pf_p にフィルタが設定される。
  • 失敗時は負の値を返す。

前提条件と不変条件

  • palloc によるメモリ確保が成功する必要がある。
  • pullf_create が正常にフィルタを構築できる必要がある。

アルゴリズムまたは振る舞い

1. PktData 構造体用のメモリを palloc で確保する。 2. 指定された pkttype と len を PktData に格納する。 3. pktreader_filter を用いて pullf_create を呼び出し、フィルタチェーンを構築する。 4. pullf_create が失敗した場合は、確保した PktData を pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-decrypt.c (複数箇所)
  • contrib/pgcrypto/pgp-info.c:136
  • contrib/pgcrypto/pgp-pubkey.c:479
  • **呼び出し先**:
  • palloc / pfree (PostgreSQL メモリ管理)
  • pullf_create (フィルタ生成)

関連データ構造

  • PullFilter: データの読み取りフィルタ。
  • PktData: パケットのタイプと長さを保持する構造体。
  • PGP_Context: 復号コンテキスト。

失敗モードと境界条件

  • メモリ不足により palloc が失敗した場合、PostgreSQL のエラーハンドリングにより制御が移る。
  • pullf_create が失敗した場合、負の戻り値が返され、内部で確保したメモリは適切に解放される。

保守担当者向けメモ

  • 本関数は pktreader_filter を使用してパケット読み取りを抽象化している。
  • ctx 引数は現在の実装では使用されていないが、将来的な拡張やインターフェースの一貫性のために保持されている。
