AssignTypeArrayOid

定義

  • **種別**: function
  • **定義箇所**: src/backend/commands/typecmds.c:2454

目的

新しいデータ型に対応する配列型のOIDを割り当てます。

シグネチャまたは形状

Oid AssignTypeArrayOid(void)

入力と出力

  • **入力**: なし
  • **出力**: 新しく割り当てられた配列型のOID（Oid型）

前提条件と不変条件

  • 通常運用時は、pg_type カタログへのアクセス権限が必要です。
  • バイナリアップグレードモード（IsBinaryUpgrade が真）の場合、事前に binary_upgrade_next_array_pg_type_oid が有効なOIDに設定されている必要があります。

アルゴリズムまたは振る舞い

1. **バイナリアップグレードの判定**: IsBinaryUpgrade が真の場合、グローバル変数 binary_upgrade_next_array_pg_type_oid からOIDを取得し、同変数を InvalidOid にリセットします。 2. **通常モードの処理**: pg_type リレーションを AccessShareLock で開き、GetNewOidWithIndex を使用して pg_type カタログの TypeOidIndexId インデックスに基づいた新しいOIDを生成します。 3. **リソース解放**: 通常モード時は table_close を呼び出し、リレーションを閉じます。 4. **戻り値**: 決定されたOIDを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c
  • src/backend/commands/typecmds.c（複数の箇所で型定義時に呼び出し）
  • **呼び出し先**:
  • table_open / table_close
  • GetNewOidWithIndex
  • ereport（バイナリアップグレード時のエラー報告）

関連データ構造

  • pg_type（カタログテーブル）
  • TypeRelationId
  • TypeOidIndexId
  • binary_upgrade_next_array_pg_type_oid（バイナリアップグレード用グローバル変数）

失敗モードと境界条件

  • **バイナリアップグレード時のエラー**: IsBinaryUpgrade が真であるにもかかわらず binary_upgrade_next_array_pg_type_oid が無効な場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告します。

保守担当者向けメモ

  • 本関数は、型定義プロセスにおいて配列型を自動生成する際に不可欠です。
  • バイナリアップグレード時のOID強制割り当てロジックは、pg_upgrade の整合性を保つために重要です。通常運用時には影響しません。
