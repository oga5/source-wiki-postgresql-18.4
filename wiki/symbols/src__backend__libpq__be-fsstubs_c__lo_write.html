lo_write

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-fsstubs.c:181

目的

開いているラージオブジェクト（Large Object）に対して、指定されたバッファからデータを書き込みます。

シグネチャまたは形状

int lo_write(int fd, const char *buf, int len)

入力と出力

  • **入力**:
  • fd: 書き込み対象のラージオブジェクトディスクリプタ。
  • buf: 書き込むデータが格納されたバッファ。
  • len: 書き込むデータのバイト数。
  • **出力**:
  • 成功時: 書き込まれたバイト数を返します（inv_write の戻り値に依存）。
  • 失敗時: ereport によりエラーが発生し、制御が戻りません。

前提条件と不変条件

  • fd は有効な範囲（0 以上 cookies_size 未満）であり、かつ cookies[fd] が NULL であってはなりません。
  • 対象のラージオブジェクトディスクリプタは、オープン時に書き込み権限（IFS_WRLOCK フラグ）が付与されている必要があります。

アルゴリズムまたは振る舞い

1. fd が有効なディスクリプタであるか検証し、無効な場合は ERRCODE_UNDEFINED_OBJECT でエラーを報告します。 2. cookies 配列から該当する LargeObjectDesc を取得します。 3. lobj->flags を確認し、IFS_WRLOCK が設定されていない場合は ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラーを報告します。 4. inv_write を呼び出し、実際の書き込み処理を実行します。 5. inv_write の結果を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-fsstubs.c 内の関数（直接的な呼び出しが確認されています）。
  • その他、外部モジュールからの呼び出しについては、提供されたソースコードの範囲内では具体的な呼び出し元関数名は特定されていません。
  • **呼び出し先**:
  • inv_write（src/backend/storage/large_object/inv_api.c で定義されるラージオブジェクトの物理書き込み関数）。

関連データ構造

  • LargeObjectDesc: ラージオブジェクトの状態を保持する構造体。
  • cookies: 開いているラージオブジェクトディスクリプタを管理するグローバルな動的配列。

失敗モードと境界条件

  • **無効なディスクリプタ**: fd が範囲外、またはクローズ済みの場合、ERROR をスローします。
  • **書き込み権限不足**: 読み取り専用でオープンされたディスクリプタに対して書き込もうとした場合、ERROR をスローします。
  • **物理書き込み失敗**: inv_write 内部で発生したエラー（トランザクション異常やストレージエラー等）は、そのまま呼び出し元へ伝播します。

保守担当者向けメモ

  • 本関数は be-fsstubs.c 内のラージオブジェクト操作のラッパーです。物理的な書き込みロジックを変更する場合は inv_write を参照してください。
  • cookies 配列の整合性は、be_lo_open および AtEOXact_LargeObject によるライフサイクル管理に依存しています。
