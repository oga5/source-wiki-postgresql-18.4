clauselist_selectivity

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/clausesel.c:99

目的

クエリプランナがコスト見積もりのために使用する、AND 条件リスト全体の選択率（Selectivity）を計算する主要なエントリポイントです。

シグネチャまたは形状

Selectivity clauselist_selectivity(PlannerInfo *root,
                                   List *clauses,
                                   int varRelid,
                                   JoinType jointype,
                                   SpecialJoinInfo *sjinfo)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • clauses: 評価対象となる RestrictInfo ノードのリスト。
  • varRelid: 評価対象の単一リレーションID（または 0）。
  • jointype: 結合の種類。
  • sjinfo: 特殊な結合に関する情報。
  • **出力**:
  • Selectivity (double型): 0.0 から 1.0 の範囲で、条件に適合する行の割合を返します。

前提条件と不変条件

  • clauses は RestrictInfo ノードのリストである必要があります。
  • 拡張統計（Extended Statistics）が利用可能な場合、列間の相関を考慮した計算が行われます。

アルゴリズムまたは振る舞い

1. 内部的に clauselist_selectivity_ext を呼び出します。 2. その際、拡張統計の使用を許可するフラグを true に設定して渡します。 3. 範囲クエリが含まれる場合は、単純な積ではなく、範囲の重なりを考慮した補正計算を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/file_fdw/file_fdw.c
  • contrib/postgres_fdw/postgres_fdw.c
  • **呼び出し先**:
  • clauselist_selectivity_ext (直接呼び出し)

関連データ構造

  • RestrictInfo: 条件式と計算済みの選択率を保持する構造体。
  • PlannerInfo: プランナのコンテキスト情報。
  • SpecialJoinInfo: 結合の制約情報。

失敗モードと境界条件

  • clauses が空リストの場合、選択率は 1.0 を返します。
  • 統計情報が不足している場合、デフォルトの選択率推定値が使用されます。

保守担当者向けメモ

  • 本関数は clauselist_selectivity_ext のラッパーです。ロジックの変更は通常 clauselist_selectivity_ext に対して行います。
  • 拡張統計の適用有無により推定値が大きく変動するため、デバッグ時には use_extended_stats フラグの挙動を確認してください。
