INFIX

定義

  • 種別: struct
  • 定義箇所: contrib/ltree/ltxtquery_io.c:490

目的

検索クエリ（ltxtquery）の内部バイナリ形式から、人間が読みやすい中置記法（infix notation）の文字列へ復元する処理において、変換状態を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	ITEM	   *curpol;    /* 現在処理中のITEM配列ポインタ */
	char	   *buf;       /* 出力用バッファ */
	char	   *cur;       /* バッファ内の現在位置 */
	char	   *op;        /* 演算子スタックまたは関連する操作用ポインタ */
	int32		buflen;    /* バッファの残り容量 */
} INFIX;

入力と出力

  • 本構造体は、infix 関数（contrib/ltree/ltxtquery_io.c:513 等）の引数として渡されます。
  • curpol を通じて入力である ITEM 配列を走査し、buf を通じて変換後の文字列を出力します。

前提条件と不変条件

  • buf は十分な長さのメモリ領域を指している必要があります。
  • curpol は有効な ITEM 配列の終端または開始位置を指している必要があります。

アルゴリズムまたは振る舞い

  • infix 関数内で再帰的に走査され、ITEM 配列の演算子とオペランドを読み取り、中置記法の文字列を構築します。
  • cur ポインタを更新しながら、buflen の範囲内で文字列を書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: ltxtq_out 関数から infix 関数を介して利用されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出しません。
  • **他モジュールでの利用**: contrib/intarray/_int_bool.c および src/backend/utils/adt/tsquery.c でも同名の構造体が定義・利用されていますが、本ページは contrib/ltree/ltxtquery_io.c の定義に焦点を当てています。

関連データ構造

  • ITEM: ltxtquery の構成要素である演算子またはオペランドの配列要素。

失敗モードと境界条件

  • buflen を超える書き込みが発生した場合の挙動は、infix 関数の実装に依存します。
  • curpol が無効なメモリ領域を指している場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は ltree モジュール以外（intarray や tsquery）でも類似の目的で定義されています。変更を加える際は、他モジュールとの互換性や定義の重複に注意してください。
