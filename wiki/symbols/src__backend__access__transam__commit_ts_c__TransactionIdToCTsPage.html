TransactionIdToCTsPage

定義

  • 種別: static inline 関数
  • 定義箇所: src/backend/access/transam/commit_ts.c:71

目的

指定されたトランザクションID（XID）が格納されているコミットタイムスタンプ（Commit Timestamp）のSLRUページ番号を算出する。

シグネチャまたは形状

int64 TransactionIdToCTsPage(TransactionId xid)

入力と出力

  • **入力**: xid (TransactionId) - ページ番号を特定したい対象のトランザクションID。
  • **出力**: int64 - 対応するSLRUページ番号。

前提条件と不変条件

  • COMMIT_TS_XACTS_PER_PAGE 定数が定義されていること。
  • 入力される xid は有効なトランザクションIDであること。

アルゴリズムまたは振る舞い

入力された xid を COMMIT_TS_XACTS_PER_PAGE で除算し、その商をページ番号として返却する。これにより、トランザクションIDの範囲を固定サイズのページにマッピングする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/commit_ts.c 内の以下の箇所から呼び出されている。
  • TransactionTreeSetCommitTsData (180行目、185行目)
  • TransactionIdSetCommitTs (277行目)
  • TransactionIdGetCommitTsData (727行目)
  • GetLatestCommitTsData (871行目)
  • CommitTsPagePrecedes (898行目)
  • **呼び出し先**:
  • 外部関数呼び出しなし（算術演算のみ）。

関連データ構造

  • COMMIT_TS_XACTS_PER_PAGE: 1ページあたりのトランザクション数を定義する定数。

失敗モードと境界条件

  • xid が非常に大きい場合や、0（InvalidTransactionId）などの特殊な値が渡された場合の挙動は、呼び出し側のロジックに依存する。
  • 算術演算であるため、除算による例外が発生する可能性は極めて低い。

保守担当者向けメモ

  • 本関数はSLRUのページングロジックの根幹である。COMMIT_TS_XACTS_PER_PAGE の値を変更する場合は、既存のデータとの互換性に注意すること。
