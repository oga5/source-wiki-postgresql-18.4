SpGistPageOpaqueData

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/spgist_private.h:60

目的

SP-GiSTインデックスの各物理ページ末尾に配置されるメタデータ構造体であり、ページ内のタプル状態や識別情報を保持します。

シグネチャまたは形状

typedef struct SpGistPageOpaqueData
{
    uint16      flags;          /* ビット定義に基づくフラグ */
    uint16      nRedirection;   /* ページ内のリダイレクションタプル数 */
    uint16      nPlaceholder;   /* ページ内のプレースホルダタプル数 */
    uint16      spgist_page_id; /* SP-GiSTインデックス識別用ID */
} SpGistPageOpaqueData;

入力と出力

本構造体はページ初期化時に書き込まれ、インデックス操作時にページ内のタプル管理状況を追跡するために読み取られます。

前提条件と不変条件

  • ページ内の LIVE タプルおよび DEAD タプルの個数は本構造体では管理されません。
  • 構造体のサイズは MAXALIGN に従う必要があります（src/include/access/spgist_private.h:451 参照）。

アルゴリズムまたは振る舞い

  • ページ初期化時（PageInit）に、ページ末尾の特殊領域として本構造体が配置されます。
  • nRedirection および nPlaceholder は、ページ内のタプル状態遷移（REDIRECT や PLACEHOLDER）に伴い更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgutils.c 内の PageInit 呼び出しにおいて、ページ初期化時に使用されます。
  • **呼び出し先**: 直接的な関数呼び出しはありませんが、インデックスの物理レイアウトを定義するデータ構造として、SP-GiSTの各操作関数から参照されます。

関連データ構造

  • SpGistPageOpaque: SpGistPageOpaqueData へのポインタ型。
  • SpGistMetaPageData: インデックス全体のメタデータ構造。

失敗モードと境界条件

  • ページサイズ（BLCKSZ）に対して本構造体のサイズが適切にアライメントされていない場合、メモリレイアウトの不整合が発生する可能性があります。

保守担当者向けメモ

  • 本構造体はSP-GiSTの物理ページレイアウトの根幹です。フィールドの追加や変更を行う際は、既存の MAXALIGN 制約およびインデックスのバイナリ互換性に注意してください。
