bytealike

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like.c:325

目的

bytea 型のデータに対して、SQL の LIKE 演算子によるパターンマッチングを実行します。

シグネチャまたは形状

Datum bytealike(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_BYTEA_PP(0): 検索対象の bytea データ。
  • PG_GETARG_BYTEA_PP(1): パターンとして使用する bytea データ。
  • **出力**:
  • PG_RETURN_BOOL(result): マッチした場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力データは bytea 型のポインタとして取得されます。
  • VARDATA_ANY および VARSIZE_ANY_EXHDR を使用して、TOAST 展開済みのデータ本体とヘッダを除いた長さを取得します。

アルゴリズムまたは振る舞い

1. 入力された bytea データから、データ本体へのポインタと長さを抽出します。 2. SB_MatchText 関数を呼び出し、シングルバイトベースのパターンマッチングを実行します。 3. SB_MatchText の戻り値が LIKE_TRUE であるかどうかを判定し、その結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェースから呼び出されます（直接的な呼び出し元情報の詳細はソースコード内に記載なし）。
  • **呼び出し先**: SB_MatchText（like_match.c で定義されたマクロにより生成される関数）。

関連データ構造

  • bytea: PostgreSQL の可変長バイナリデータ型。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • 入力データが NULL の場合、PG_GETARG_BYTEA_PP マクロの挙動に従い、呼び出し元で適切に処理される必要があります。
  • SB_MatchText はシングルバイトの比較を行うため、マルチバイト文字を含む bytea データに対しては、バイト単位の厳密な一致判定が行われます。

保守担当者向けメモ

  • 本関数は SB_MatchText を直接呼び出しており、エンコーディングの差異を考慮しないシングルバイト比較に特化しています。
  • like_match.c をインクルードして生成される他の関数群と同様、SB_MatchText の挙動変更が本関数に直接影響します。
