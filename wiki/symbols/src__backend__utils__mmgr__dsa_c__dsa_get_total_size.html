dsa_get_total_size

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/dsa.c:1026

目的

指定された動的共有メモリ領域（DSA）において、現在割り当てられている全セグメントの合計サイズを取得する。

シグネチャまたは形状

size_t dsa_get_total_size(dsa_area *area)

入力と出力

  • **入力**: area (DSA領域を管理する dsa_area 構造体へのポインタ)
  • **出力**: size_t 型の合計サイズ（バイト単位）

前提条件と不変条件

  • area は有効な dsa_area 構造体を指している必要がある。
  • 本関数は内部で DSA_AREA_LOCK を取得するため、呼び出し元で当該ロックを保持している必要はない。

アルゴリズムまたは振る舞い

1. area に関連付けられた DSA_AREA_LOCK を排他モード（LW_EXCLUSIVE）で取得する。 2. 制御構造体 area->control 内の total_segment_size フィールドから現在の合計サイズを読み取る。 3. DSA_AREA_LOCK を解放する。 4. 読み取ったサイズを戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/lib/radixtree.h 内の処理から呼び出される（tree->dsa を引数として使用）。
  • **呼び出し先**:
  • LWLockAcquire
  • LWLockRelease

関連データ構造

  • dsa_area: DSA領域の状態を保持する構造体。
  • dsa_area_control: 共有メモリ領域の制御情報を保持する構造体（total_segment_size を含む）。

失敗モードと境界条件

  • area が NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • ロック取得時に他のプロセスとの競合が発生した場合、ロックが解放されるまで待機する。

保守担当者向けメモ

  • 本関数はロックを取得するため、呼び出し元で既に DSA_AREA_LOCK を保持しているとデッドロックが発生する可能性がある。
  • total_segment_size は動的に変化するため、本関数が返す値は呼び出し時点でのスナップショットである。
