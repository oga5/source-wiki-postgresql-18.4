tlist_member

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/tlist.c:87

目的

指定された式（Expr）と一致する式を持つ TargetEntry をターゲットリスト（List）から検索します。

シグネチャまたは形状

TargetEntry *tlist_member(Expr *node, List *targetlist)

入力と出力

  • **入力**:
  • node: 検索対象の式（Expr*）。
  • targetlist: 検索対象となる TargetEntry のリスト（List*）。
  • **出力**:
  • 一致する TargetEntry へのポインタ。見つからない場合は NULL を返します。

前提条件と不変条件

  • targetlist は TargetEntry ノードを含むリストである必要があります。
  • 式の一致判定には equal() 関数が使用されます。

アルゴリズムまたは振る舞い

1. targetlist 内の各要素を foreach マクロで順次走査します。 2. 各 TargetEntry（tlentry）の expr フィールドと、引数 node を equal() 関数で比較します。 3. equal() が真を返した場合、その tlentry を即座に返します。 4. リストの末尾まで一致するものが見つからなかった場合、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/optimizer/plan/createplan.c
  • src/backend/optimizer/plan/setrefs.c
  • src/backend/optimizer/prep/preptlist.c
  • **呼び出し先**:
  • equal()（式の一致判定に使用）

関連データ構造

  • TargetEntry: ターゲットリストの各要素を構成するノード。
  • Expr: 評価対象の式。

失敗モードと境界条件

  • targetlist が空（NIL）の場合、ループは実行されず NULL が返されます。
  • node が NULL の場合、equal() の挙動に依存します。

保守担当者向けメモ

  • equal() を使用した厳密な比較を行うため、特定の条件下で比較を緩和したい場合は tlist_member_match_var 等の代替関数の検討が必要です。
  • 本関数はリストの先頭から線形探索を行うため、非常に長いターゲットリストに対して頻繁に呼び出す場合はパフォーマンスへの影響を考慮してください。
