BlockSampler_Init

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/sampling.c:38

目的

リレーションのブロックサンプリングを行うための状態構造体（BlockSamplerData）を初期化し、サンプリング処理の準備を行う。

シグネチャまたは形状

BlockNumber BlockSampler_Init(BlockSampler bs, BlockNumber nblocks, int samplesize, uint32 randseed)

入力と出力

  • **入力**:
  • bs: 初期化対象の BlockSampler 構造体へのポインタ。
  • nblocks: 対象リレーションの総ブロック数。
  • samplesize: 抽出したいサンプル数。
  • randseed: 乱数生成器の初期化に使用するシード値。
  • **出力**:
  • 実際にサンプリング対象となるブロック数（Min(samplesize, nblocks) を返却）。

前提条件と不変条件

  • bs は有効なメモリ領域を指している必要がある。
  • 内部状態として bs->t（走査済みブロック数）と bs->m（選択済みブロック数）が 0 にリセットされる。

アルゴリズムまたは振る舞い

1. 引数 nblocks を bs->N に格納する。 2. samplesize を bs->n に格納する。 3. 走査済みブロック数 bs->t および選択済みブロック数 bs->m を 0 に初期化する。 4. sampler_random_init_state を呼び出し、指定された randseed で乱数生成器の状態を初期化する。 5. samplesize と nblocks の小さい方の値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/analyze.c:1228 にて、統計情報収集の準備として呼び出される。
  • **呼び出し先**: sampler_random_init_state（乱数生成器の初期化）。

関連データ構造

  • BlockSamplerData: サンプリングの状態を保持する構造体。N（総数）、n（目標数）、t（現在位置）、m（選択数）、randstate（乱数状態）を含む。

失敗モードと境界条件

  • samplesize が nblocks を超える場合、戻り値は nblocks となる。
  • randseed が同一であれば、生成されるサンプリング順序は再現可能である。

保守担当者向けメモ

  • 本関数は Knuth の「Algorithm S」を用いたサンプリングの初期化フェーズを担う。
  • samplesize を動的に削減するロジックを追加する場合は、本関数内のコメント箇所で実装すること。
