SyncRepInitConfig

定義

  • 種別: function
  • 定義箇所: src/backend/replication/syncrep.c:444

目的

現在のWAL送信プロセス（WALSender）が同期スタンバイとして機能する優先度を判定し、共有メモリ上の状態を更新します。

シグネチャまたは形状

void SyncRepInitConfig(void)

入力と出力

  • **入力**: なし（内部的に MyWalSnd および synchronous_standby_names 設定を参照）。
  • **出力**: なし（副作用として MyWalSnd->sync_standby_priority を更新）。

前提条件と不変条件

  • MyWalSnd が有効なWALSender状態を指している必要があります。
  • 呼び出し元はWALSenderプロセス内である必要があります。

アルゴリズムまたは振る舞い

1. SyncRepGetStandbyPriority() を呼び出し、現在の設定に基づく自身の同期優先度を取得します。 2. 取得した優先度が MyWalSnd->sync_standby_priority と異なる場合、スピンロック（MyWalSnd->mutex）を取得して値を更新します。 3. 更新が行われた場合、DEBUG1 レベルでログを出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walsender.c 内の複数箇所（レプリケーション設定変更時や接続状態遷移時など）。
  • **呼び出し先**:
  • SyncRepGetStandbyPriority()
  • SpinLockAcquire / SpinLockRelease
  • ereport

関連データ構造

  • WalSnd (共有メモリ構造体): MyWalSnd を通じてアクセスされる。
  • sync_standby_priority: WalSnd 構造体内のメンバ。

失敗モードと境界条件

  • 設定が変更されていない場合、更新処理はスキップされます。
  • MyWalSnd が初期化されていない環境で呼び出された場合の動作は未定義です（ソースコード上はWALSenderプロセス内での呼び出しを前提としています）。

保守担当者向けメモ

  • 本関数は synchronous_standby_names の変更を反映するために頻繁に呼び出されます。
  • 優先度の更新はスピンロックで保護されていますが、頻繁な呼び出しによる競合が発生しないよう注意してください。
