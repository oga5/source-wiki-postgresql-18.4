ObjectAddressStack

定義

  • 種別: struct
  • 定義箇所: src/backend/catalog/dependency.c:124

目的

依存関係の再帰的な探索処理において、現在訪問中のオブジェクトの経路を追跡し、循環参照を検出・防止するためのスタック構造を提供します。

シグネチャまたは形状

typedef struct ObjectAddressStack
{
	const ObjectAddress *object;	/* 訪問中のオブジェクト */
	int			flags;			/* 現在のフラグビット */
	struct ObjectAddressStack *next;	/* 次のスタックレベルへのポインタ */
} ObjectAddressStack;

入力と出力

  • 入力: 探索対象の ObjectAddress および現在のフラグ状態。
  • 出力: 探索の深さに応じて動的に構築される連結リスト形式のスタック。

前提条件と不変条件

  • next ポインタは、より外側のスタックレベル（呼び出し元）を指します。
  • 循環参照を避けるため、探索の各ステップでこのスタックが参照されます。

アルゴリズムまたは振る舞い

  • findDependentObjects 関数等の再帰的な依存関係探索において、現在処理中のオブジェクトをスタックに積むことで、探索経路を保持します。
  • 探索中に同じオブジェクトがスタック内に存在するかを確認することで、無限ループを回避します。

呼び出し元と呼び出し先

  • 呼び出し元: findDependentObjects 等の依存関係探索関数が、スタックの構築および検索に使用します。
  • 呼び出し先: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ObjectAddress: 探索対象となるデータベースオブジェクトの識別子。
  • ObjectAddresses: 削除対象オブジェクトのリストを保持する構造体。

失敗モードと境界条件

  • 依存関係が極めて深い場合、スタックの深さがシステム制限を超える可能性があります（check_stack_depth() による保護が前提）。

保守担当者向けメモ

  • 本構造体は dependency.c 内の再帰探索ロジックに特化しています。
  • 循環参照の検出ロジックを変更する際は、このスタックの走査方法を併せて確認してください。
