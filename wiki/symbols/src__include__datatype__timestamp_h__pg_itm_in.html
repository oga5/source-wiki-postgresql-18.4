pg_itm_in

定義

  • 種別: struct
  • 定義箇所: src/include/datatype/timestamp.h:82

目的

日時計算や分解処理において、日付（年・月・日）およびマイクロ秒単位の時刻情報を保持するための構造体です。

シグネチャまたは形状

struct pg_itm_in
{
    int64       tm_usec;    /* マイクロ秒単位の時刻 */
    int         tm_mday;    /* 日 */
    int         tm_mon;     /* 月 */
    int         tm_year;    /* 年 */
};

入力と出力

本シンボルはデータ構造の定義であり、関数ではないため直接的な入出力はありません。ただし、src/backend/utils/adt/datetime.c 内の各関数において、引数としてポインタ経由で受け渡され、日時情報の読み書きに使用されます。

前提条件と不変条件

  • tm_usec は int64 型であり、広範な時刻範囲を保持できる必要があります。
  • 日時計算の過程で、各フィールドは正規化されることが想定されます。

アルゴリズムまたは振る舞い

本構造体自体はデータの保持のみを行います。具体的な振る舞いは、この構造体を操作する src/backend/utils/adt/datetime.c 内の関数（例: AdjustMonths 等）に依存します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/datetime.c 内の複数の関数が、引数として struct pg_itm_in * を受け取ります。
  • **呼び出し先**: 本構造体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • pg_itm: 日時分解に関連する対となる構造体。
  • Timestamp / TimestampTz: 本構造体と相互変換される可能性のある絶対時間表現。

失敗モードと境界条件

  • tm_usec が int64 の範囲を超えた場合のオーバーフローについては、本構造体定義単体では規定されていません。
  • 各フィールド（年・月・日）の妥当性チェックは、本構造体を利用する側のロジックに依存します。

保守担当者向けメモ

  • src/backend/utils/adt/datetime.c で頻繁に参照されているため、フィールドの追加や変更を行う際は、同ファイル内のすべての参照箇所で整合性が取れているか確認してください。
