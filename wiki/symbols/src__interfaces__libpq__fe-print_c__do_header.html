do_header

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-print.c:455

目的

クエリ結果の表形式出力において、ヘッダー行および境界線（枠線）を生成・出力します。

シグネチャまたは形状

static char *do_header(FILE *fout, const PQprintOpt *po, const int nFields, int *fieldMax, const char **fieldNames, unsigned char *fieldNotNum, const int fs_len, const PGresult *res);

入力と出力

  • **入力**:
  • fout: 出力先ファイルストリーム。
  • po: 出力オプション（PQprintOpt）。HTML形式や標準形式の指定に使用。
  • nFields: フィールド数。
  • fieldMax: 各列の最大幅を格納した配列。
  • fieldNames: フィールド名の配列。
  • fieldNotNum: 数値以外のフィールドを示すフラグ配列。
  • fs_len: フィールド区切り文字の長さ。
  • res: PGresult オブジェクト。
  • **出力**:
  • 成功時: 生成された境界線文字列（char *）。呼び出し元で解放が必要。
  • 失敗時: NULL。

前提条件と不変条件

  • po->standard が有効な場合、境界線文字列の計算において pg_add_size_overflow を用いたオーバーフローチェックが行われます。
  • fieldMax 配列は、ヘッダーの長さを考慮して必要に応じて更新されます。

アルゴリズムまたは振る舞い

1. **HTMLモード**: po->html3 が真の場合、<tr> タグと <th> タグを用いてヘッダーを出力します。 2. **標準モード**:

  • 各列の幅と区切り文字に基づき、境界線（例: +---+---+）の長さを計算します。
  • 計算したサイズでメモリを確保し、境界線文字列を構築します。
  • 境界線、フィールド名、区切り文字を順次 fout に書き出します。

    • 3. **エラー処理**: メモリ確保失敗時やサイズ計算でオーバーフローが発生した場合は、エラーメッセージを出力して NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-print.c 内の PQprint 関数から呼び出されます。
  • **呼び出し先**: PQfname（libpq API）、pg_add_size_overflow（共通ユーティリティ）、および標準のファイル出力関数（fputs, fprintf）。

関連データ構造

  • PQprintOpt: 出力形式を制御する構造体。
  • PGresult: クエリ結果を保持する構造体。

失敗モードと境界条件

  • **メモリ不足**: malloc が失敗した場合、stderr にエラーを出力して NULL を返します。
  • **オーバーフロー**: 境界線のサイズ計算が size_t の範囲を超える場合、overflow ラベルへ遷移し NULL を返します。

保守担当者向けメモ

  • 本関数はレガシーな出力機能の一部です。
  • 境界線文字列の動的確保を行っているため、呼び出し元でのメモリリークに注意してください。
  • fieldMax を直接書き換える副作用があるため、呼び出し時の状態管理に留意してください。
