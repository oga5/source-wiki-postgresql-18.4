LargeObjectDesc

定義

  • 種別: struct
  • 定義箇所: src/include/storage/large_object.h:39

目的

PostgreSQLにおいて、オープンされたラージオブジェクト（LO）の操作セッション状態を管理するための構造体です。

シグネチャまたは形状

typedef struct LargeObjectDesc
{
    Oid         id;             /* LOの識別子 */
    Snapshot    snapshot;       /* 使用するスナップショット */
    SubTransactionId subid;     /* 所有するサブトランザクションID */
    uint64      offset;         /* 現在のシークポインタ */
    int         flags;          /* 状態フラグ (IFS_RDLOCK, IFS_WRLOCK) */
} LargeObjectDesc;

入力と出力

  • 本構造体は inv_open 関数によって初期化され、ラージオブジェクト操作のハンドルとして返されます。
  • inv_read や inv_write などのAPIにおいて、操作対象のコンテキストとして入力されます。

前提条件と不変条件

  • flags フィールドには、オープン時の権限状態を示す IFS_RDLOCK（読み取り）または IFS_WRLOCK（書き込み）が設定されます。
  • offset は現在の読み書き位置を保持し、操作に応じて更新されます。

アルゴリズムまたは振る舞い

  • ラージオブジェクトのオープン時に生成され、セッション中のシーク位置やスナップショット情報を保持します。
  • inv_close が呼び出されるまで、ラージオブジェクトに対する一連の操作（読み取り、書き込み、シーク等）のステートフルな管理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/be-fsstubs.c 内の関数群（inv_open 等のラッパー）が、本構造体をメモリコンテキスト上で管理・参照しています。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を直接呼び出すことはありません。

関連データ構造

  • Snapshot: ラージオブジェクト読み取り時に使用されるスナップショット。
  • SubTransactionId: ライフサイクル管理のためのサブトランザクション識別子。

失敗モードと境界条件

  • 構造体内の offset は uint64 で管理されており、ラージオブジェクトの最大サイズ（INT_MAX * LOBLKSIZE）を超えるシークや書き込みは制限されます。

保守担当者向けメモ

  • 本構造体は src/backend/libpq/be-fsstubs.c にてメモリコンテキスト上に割り当てられるため、メモリリークを防ぐための適切なクローズ処理（inv_close）が必須です。
  • flags のビット定義（IFS_RDLOCK, IFS_WRLOCK）を変更する場合は、関連するAPIの権限チェックロジックとの整合性に注意してください。
