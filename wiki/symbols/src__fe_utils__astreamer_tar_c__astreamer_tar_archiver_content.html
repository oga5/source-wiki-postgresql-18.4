astreamer_tar_archiver_content

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_tar.c:425

目的

tarアーカイブのストリーム処理において、型付きチャンクを受け取り、必要に応じてヘッダーの再構築やパディングの補完を行い、後続のストリーマーへデータを転送する。

シグネチャまたは形状

static void astreamer_tar_archiver_content(astreamer *streamer,
                                           astreamer_member *member,
                                           const char *data, int len,
                                           astreamer_archive_context context)

入力と出力

  • streamer: アーカイバーのインスタンス。
  • member: 現在処理中のtarメンバー情報。
  • data: 入力データバッファ。
  • len: 入力データの長さ。
  • context: 現在のアーカイブコンテキスト（ヘッダー、コンテンツ、トレーラー等）。
  • 出力: なし（astreamer_content を介して後続ストリーマーへデータを渡す）。

前提条件と不変条件

  • context は ASTREAMER_UNKNOWN であってはならない。
  • ASTREAMER_MEMBER_HEADER コンテキストにおいて len が0の場合、tarCreateHeader を用いてヘッダーが再構築される。
  • ASTREAMER_MEMBER_TRAILER コンテキストにおいて、ヘッダーが再構築された場合はパディングも再計算・補完される。

アルゴリズムまたは振る舞い

1. **ヘッダー再構築**: context が ASTREAMER_MEMBER_HEADER で len が0の場合、tarCreateHeader でヘッダーを生成し、rearchive_member フラグを立てる。 2. **パディング補完**: context が ASTREAMER_MEMBER_TRAILER で rearchive_member が真の場合、tarPaddingBytesRequired に基づいてパディングを生成し、フラグを解除する。 3. **アーカイブ終端**: context が ASTREAMER_ARCHIVE_TRAILER の場合、2ブロック分のNULバイトを生成する。 4. **転送**: 処理済みのデータ（または元のデータ）を astreamer_content を通じて後続のストリーマーへ渡す。

呼び出し元と呼び出し先

  • 呼び出し元: src/fe_utils/astreamer_tar.c 内の astreamer_tar_archiver 構造体の .content フィールドとして参照されている。
  • 呼び出し先: tarCreateHeader、tarPaddingBytesRequired、astreamer_content。

関連データ構造

  • astreamer_tar_archiver: 本関数が操作する状態保持構造体。rearchive_member フラグを保持する。
  • astreamer_member: メンバーのメタデータ（パス名、サイズ、モード等）を保持。

失敗モードと境界条件

  • context が ASTREAMER_UNKNOWN の場合、Assert により停止する。
  • ASTREAMER_MEMBER_HEADER で len が0以外の場合、ヘッダー再構築は行われない。

保守担当者向けメモ

  • 本関数は astreamer パイプラインの一部として動作する。
  • ヘッダーの再構築が行われた場合、後続のパディング処理も連動して更新される必要があるため、rearchive_member フラグの状態管理に注意すること。
