IncreaseBuffer

定義

  • 種別: function
  • 定義箇所: contrib/fuzzystrmatch/dmetaphone.c:272

目的

metastring 構造体が保持する文字列バッファを動的に拡張し、必要な文字数を格納できるようにします。

シグネチャまたは形状

static void IncreaseBuffer(metastring *s, int chars_needed)

入力と出力

  • **入力**:
  • s: 拡張対象の metastring 構造体へのポインタ。
  • chars_needed: 追加で必要となる最小文字数。
  • **出力**: なし（副作用として s が指すバッファを更新）。

前提条件と不変条件

  • s は有効な metastring 構造体を指している必要があります。
  • META_REALLOC マクロが PostgreSQL のメモリ管理機構（repalloc 等）をラップしていることを前提としています。
  • 拡張後のバッファサイズは、現在のサイズに chars_needed と 10 文字の余裕を加えた値になります。

アルゴリズムまたは振る舞い

1. META_REALLOC を使用して、現在のバッファサイズ (s->bufsize) に chars_needed と 10 を加えたサイズでメモリを再確保します。 2. assert を用いて、再確保後のポインタが NULL でないことを検証します。 3. s->bufsize を新しいサイズに更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: MetaphAdd 関数から呼び出されます。
  • **呼び出し先**: META_REALLOC マクロ（PostgreSQL のメモリ再確保機能）。

関連データ構造

  • metastring: 文字列バッファとそのサイズを管理する構造体。

失敗モードと境界条件

  • メモリ不足が発生した場合、META_REALLOC の実装に依存しますが、通常は PostgreSQL のエラーハンドリングによりプロセスが中断されます。
  • assert により、メモリ確保失敗時はデバッグビルド環境でアボートします。

保守担当者向けメモ

  • 本関数は MetaphAdd 内でバッファ不足を検知した際に呼び出されます。
  • 10 文字という固定の余裕値は、頻繁な再確保を避けるためのヒューリスティックなバッファリングです。
