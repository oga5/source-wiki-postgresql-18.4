ItemPointerDec

定義

  • 種別: function
  • 定義箇所: src/backend/storage/page/itemptr.c:113

目的

ItemPointer（TID）を論理的に1つ前の位置へデクリメントします。ブロック境界を跨いだ移動をサポートし、オフセットが0になった場合にブロック番号を繰り下げます。

シグネチャまたは形状

void ItemPointerDec(ItemPointer pointer)

入力と出力

  • **入力**: pointer (ItemPointer型) - 操作対象のタプル識別子へのポインタ。
  • **出力**: なし（引数 pointer が指すメモリ領域が直接更新されます）。

前提条件と不変条件

  • pointer は有効な ItemPointerData を指している必要があります。
  • ItemPointerData 構造体は 6 バイトのサイズを維持する必要があります。

アルゴリズムまたは振る舞い

1. ItemPointerGetBlockNumberNoCheck および ItemPointerGetOffsetNumberNoCheck を使用して、現在のブロック番号とオフセットを取得します。 2. オフセットが 0 の場合:

  • ブロック番号が 0 でなければ、オフセットを PG_UINT16_MAX に設定し、ブロック番号を 1 減らします。
  • ブロック番号が 0 の場合は、オフセットは 0 のまま維持されます。

    • 3. オフセットが 0 でない場合:

  • オフセットを単純に 1 減らします。

    • 4. ItemPointerSet を使用して、計算された新しいブロック番号とオフセットを pointer に書き戻します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeTidrangescan.c:199
  • **呼び出し先**:
  • ItemPointerGetBlockNumberNoCheck
  • ItemPointerGetOffsetNumberNoCheck
  • ItemPointerSet

関連データ構造

  • ItemPointerData: ディスク上のタプル位置を保持する 6 バイトの構造体。

失敗モードと境界条件

  • ブロック番号が 0 かつオフセットが 0 の場合、それ以上デクリメントされず値は維持されます。
  • オフセットが 0 の状態からデクリメントを行うと、ブロック番号が繰り下がります。

保守担当者向けメモ

  • 本関数は NoCheck 系のアクセサを使用しており、無効な TID 値に対してもビットパターンに基づいた一貫した演算を行います。
  • ページ内の有効なオフセット範囲（MaxOffsetNumber）とは独立して動作するため、呼び出し側で必要に応じて範囲チェックを行う必要があります。
