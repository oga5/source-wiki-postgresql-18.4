getConnectionByName

定義

  • 種別: function
  • 定義箇所: contrib/dblink/dblink.c:2533

目的

名前付き接続の識別子を使用して、ハッシュテーブルから対応する remoteConn 構造体を取得します。

シグネチャまたは形状

static remoteConn *getConnectionByName(const char *name)

入力と出力

  • **入力**: name (接続名を表す文字列)
  • **出力**: 接続が見つかり、かつ有効な PGconn を保持している場合は remoteConn* を返します。それ以外（接続が存在しない、または接続が確立されていない）の場合は NULL を返します。

前提条件と不変条件

  • remoteConnHash が初期化されていない場合、内部で createConnHash() を呼び出して初期化を行います。
  • 入力された name は、内部で truncate_identifier を用いて PostgreSQL の識別子長制限に合わせて切り詰められた上で検索に使用されます。

アルゴリズムまたは振る舞い

1. remoteConnHash が存在しない場合、createConnHash() を呼び出してハッシュテーブルを生成します。 2. 入力された name を pstrdup で複製し、truncate_identifier で正規化します。 3. hash_search を使用し、HASH_FIND モードでハッシュテーブルから該当する remoteConnHashEnt を検索します。 4. エントリが存在し、かつそのエントリ内の rconn.conn が NULL でない（接続が有効である）ことを確認します。 5. 条件を満たす場合、その remoteConn 構造体へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: dblink.c 内の複数の関数（dblink_connect, dblink_disconnect, dblink_exec, dblink_record 等）から、接続名を指定した操作を行う際に呼び出されます。
  • **呼び出し先**: createConnHash（初期化時）、pstrdup、truncate_identifier、hash_search。

関連データ構造

  • remoteConn: リモート接続状態を保持する構造体。
  • remoteConnHashEnt: remoteConnHash ハッシュテーブルの各エントリを構成する構造体。
  • remoteConnHash: 接続名をキーとして remoteConn を管理するハッシュテーブル。

失敗モードと境界条件

  • 指定された名前の接続が存在しない場合、NULL を返します。
  • 接続が存在しても、PGconn が NULL（接続が切断されている等）である場合は NULL を返します。
  • メモリ不足時には pstrdup 等でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は接続名の正規化（truncate_identifier）を行うため、呼び出し元で指定する名前とハッシュテーブル内のキーが一致するよう設計されています。
  • 接続の有効性チェックとして hentry->rconn.conn != NULL を行っている点に注意してください。
