AffixNode

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/dicts/spell.h:138

目的

接辞（affix）を格納するためのトライ（Trie）構造のノードを表現します。

シグネチャまたは形状

typedef struct AffixNode
{
    uint32      isvoid:1,
                length:31;
    AffixNodeData data[FLEXIBLE_ARRAY_MEMBER];
} AffixNode;

入力と出力

  • **入力**: 辞書構築時に palloc0 等を用いてメモリを確保し、ANHRDSZ（ヘッダーサイズ）と AffixNodeData の配列サイズを組み合わせてインスタンス化されます。
  • **出力**: トライ構造の一部として、接辞検索時のノードとして利用されます。

前提条件と不変条件

  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズ計算には ANHRDSZ マクロを使用する必要があります。
  • length フィールドは、後続する data 配列の要素数を保持します。

アルゴリズムまたは振る舞い

  • 辞書内の接辞を木構造で保持し、FindAffixes 関数等を通じて単語の接辞マッチングを行う際にトラバースされます。
  • 構築フェーズでは、NISortAffixes 等の処理を経て、効率的な検索が可能なトライ構造へと変換されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/spell.c 内の辞書構築および検索ロジック（FindAffixes 等）から参照されます。
  • **呼び出し先**: 構造体内部で AffixNodeData を保持しています。

関連データ構造

  • AffixNodeData: トライの各ノードが保持する具体的なデータ要素。
  • IspellDict: 辞書全体の状態を管理する構造体であり、AffixNode を含むトライ構造を保持します。

失敗モードと境界条件

  • メモリ確保時、ANHRDSZ を正しく計算せずにメモリを割り当てた場合、バッファオーバーフローや不正なメモリ参照が発生する可能性があります。
  • 辞書構築フェーズ（NIStartBuild から NIFinishBuild まで）以外での動的な操作は、メモリ管理の不整合を招く恐れがあります。

保守担当者向けメモ

  • 本構造体は可変長配列（FLEXIBLE_ARRAY_MEMBER）を含みます。メモリレイアウトを直接操作する際は、ANHRDSZ マクロの定義を必ず参照してください。
  • 辞書構築時のメモリコンテキスト（buildCxt）とライフサイクルを意識し、NIFinishBuild による適切なクリーンアップが行われていることを確認してください。
