choose_popcount_functions

定義

  • 種別: function
  • 定義箇所: src/port/pg_popcount_aarch64.c:74

目的

AArch64環境において、実行時のCPU機能判定に基づき、最適なビットカウント（popcount）実装関数を選択してグローバルな関数ポインタを更新する。

シグネチャまたは形状

static inline void choose_popcount_functions(void)

入力と出力

  • 入力: なし（内部で pg_popcount_sve_available() を呼び出し、CPUのSVEサポート状況を判定する）。
  • 出力: なし（副作用として pg_popcount_optimized および pg_popcount_masked_optimized を適切な実装関数へ書き換える）。

前提条件と不変条件

  • POPCNT_AARCH64 が定義されている環境でのみ有効。
  • 本関数は、最適化されたpopcount関数が初めて呼び出される際に実行されることを想定している。

アルゴリズムまたは振る舞い

1. pg_popcount_sve_available() を実行し、SVE命令セットが利用可能かを確認する。 2. 利用可能な場合:

  • pg_popcount_optimized を pg_popcount_sve に設定。
  • pg_popcount_masked_optimized を pg_popcount_masked_sve に設定。

    • 3. 利用不可能な場合:

  • pg_popcount_optimized を pg_popcount_neon に設定。
  • pg_popcount_masked_optimized を pg_popcount_masked_neon に設定。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/port/pg_bitutils.c (複数箇所)
  • src/port/pg_popcount_aarch64.c (複数箇所)
  • 呼び出し先:
  • pg_popcount_sve_available()

関連データ構造

  • pg_popcount_optimized: 最適化されたpopcount関数を保持する関数ポインタ。
  • pg_popcount_masked_optimized: マスク付きpopcount関数を保持する関数ポインタ。

失敗モードと境界条件

  • USE_SVE_POPCNT_WITH_RUNTIME_CHECK が定義されていない場合、SVEの判定は行われず、常にNeon実装が選択される。

保守担当者向けメモ

  • 本関数はインライン関数として定義されている。
  • 関数ポインタの書き換えはスレッドセーフではない可能性があるため、呼び出し元のコンテキスト（初期化処理）を確認すること。
