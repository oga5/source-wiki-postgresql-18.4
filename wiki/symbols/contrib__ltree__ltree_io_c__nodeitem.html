nodeitem

定義

  • 種別: struct
  • 定義箇所: contrib/ltree/ltree_io.c:17

目的

ltree および lquery の文字列解析処理において、一時的なラベル情報（ノード）を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct
{
	const char *start; /* ラベルの開始位置 */
	int			len;   /* バイト単位の長さ */
	int			flag;  /* フラグ情報 */
	int			wlen;  /* 文字単位の長さ */
} nodeitem;

入力と出力

本構造体はデータ保持用であり、関数ではないため直接の入出力はありません。解析プロセスにおいて、文字列から抽出された情報が各フィールドに格納されます。

前提条件と不変条件

  • len はバイト単位の長さを、wlen は文字単位の長さを保持します。
  • 解析完了後、finish_nodeitem 関数によってラベルの妥当性が検証されます。

アルゴリズムまたは振る舞い

文字列解析中に一時的なリストとしてメモリ上に確保され、各ラベルの境界や属性を記録します。解析終了後、これらの情報は最終的な ltree または lquery 構造体へコピーされます。

呼び出し元と呼び出し先

  • **呼び出し元**: parse_ltree や parse_lquery などの解析関数が、文字列スキャン中に本構造体の配列を palloc で確保し、利用します。
  • **呼び出し先**: finish_nodeitem 関数が、本構造体へのポインタを引数として受け取り、ラベルの検証を行います。

関連データ構造

  • ltree: 階層パスを格納する最終的な構造体。
  • lquery: 検索パターンを格納する最終的な構造体。

失敗モードと境界条件

  • メモリ確保時（palloc）にメモリ不足が発生した場合、PostgreSQL のエラーハンドリング機構により中断されます。
  • finish_nodeitem による検証でラベル長や構文の制約に違反した場合、エラーが報告されます。

保守担当者向けメモ

  • 本構造体は解析時の一時的なバッファとして機能します。ltree_io.c 内の GETVAR マクロ等でポインタ操作が行われているため、構造体のメモリレイアウトを変更する際は注意してください。
