r_postlude

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_french.c:601

目的

フランス語のステミング処理における最終段階として、単語内の特定の文字パターンを置換または削除し、語幹を確定させる。

シグネチャまたは形状

static int r_postlude(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（Snowballランタイム環境構造体）
  • 出力: 成功時に 1、エラー発生時に負の値を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • a_1 テーブルに基づくパターンマッチングが実行される。

アルゴリズムまたは振る舞い

1. 無限ループ内で find_among を使用し、現在のカーソル位置から a_1 テーブルに定義されたパターンを単語末尾方向へ検索する。 2. マッチしたパターン（among_var）に応じて以下の処理を行う：

  • 1～5: slice_from_s を呼び出し、対応する文字列（s_8～s_12）に置換する。
  • 6: slice_del を呼び出し、該当箇所を削除する。
  • 7: マッチしない場合はカーソルを1文字進める。

    • 3. マッチするパターンがなくなった時点でループを終了し、1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_ISO_8859_1_french.c:1275（french_ISO_8859_1_stem 内から呼び出される）。
  • 呼び出し先: find_among（パターン検索）、slice_from_s（文字列置換）、slice_del（文字列削除）。

関連データ構造

  • SN_env: ステミング処理の状態を保持する構造体。
  • a_1: among 構造体の配列。パターンマッチングに使用される。

失敗モードと境界条件

  • slice_from_s または slice_del が負の値を返した場合、その値をそのまま呼び出し元へ伝播させる。
  • カーソルが文字列の末尾に達した場合、ループを終了する。
