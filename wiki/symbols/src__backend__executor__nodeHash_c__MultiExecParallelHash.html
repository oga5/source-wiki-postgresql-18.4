MultiExecParallelHash

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHash.c:218

目的

並列ハッシュ結合におけるハッシュテーブル構築フェーズを、複数のバックエンド間で同期・協調して実行する。

シグネチャまたは形状

static void MultiExecParallelHash(HashState *node)

入力と出力

  • **入力**: HashState *node（ハッシュ結合の実行状態を保持するノード構造体）。
  • **出力**: なし（副作用として共有メモリ上のハッシュテーブル構築およびディスクへのバッチ書き出しを行う）。

前提条件と不変条件

  • hashtable が初期化済みであること。
  • Barrier（build_barrier）の状態に基づき、各ワーカーが現在の構築フェーズを正しく認識していること。
  • PHJ_BUILD_ALLOCATE フェーズ以降で呼び出されること。

アルゴリズムまたは振る舞い

1. **同期**: build_barrier を使用して、全ワーカーがハッシュテーブル構築の各段階（メモリ確保、ハッシュ計算、バッチ処理）で同期する。 2. **動的拡張**: 必要に応じて ExecParallelHashIncreaseNumBatches および ExecParallelHashIncreaseNumBuckets を呼び出し、バッチ数やバケット数を動的に拡張する。 3. **タプル挿入**: 子ノードからタプルを取得し、ハッシュ値を計算して共有ハッシュテーブルへ挿入する。メモリ制限を超過したタプルはバッチファイルへ書き出す。 4. **終了処理**: 全ワーカーがタプル処理を完了した後、共有カウンタをマージし、バッチファイルを閉じる。 5. **状態固定**: 最終的に pstate->growth を PHJ_GROWTH_DISABLED に設定し、バッチ構成を固定する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の明示的な呼び出し元は提供されていません。
  • **呼び出し先**:
  • BarrierArriveAndWait, BarrierAttach, BarrierDetach
  • ExecParallelHashIncreaseNumBatches, ExecParallelHashIncreaseNumBuckets
  • ExecParallelHashEnsureBatchAccessors, ExecParallelHashTableInsert, ExecParallelHashMergeCounters
  • ExecProcNode, ExecEvalExprSwitchContext

関連データ構造

  • HashState: 実行ノードの状態。
  • HashJoinTable: ハッシュ結合の全状態を保持する主要構造体。
  • ParallelHashJoinState: 並列処理用の共有状態。
  • Barrier: 並列実行の同期プリミティブ。

失敗モードと境界条件

  • **メモリ不足**: work_mem を超える場合、バッチ数を増やしてディスクへスピルする。
  • **スキュー**: 頻出値によるスキューが発生した場合、バッチ処理の効率が低下する可能性がある。
  • **同期待ち**: 他のワーカーが処理を完了するまで BarrierArriveAndWait で待機するため、ワーカー間の処理負荷の偏りが全体の待機時間となる。

保守担当者向けメモ

  • 本関数は並列処理の核心部であり、Barrier のフェーズ遷移が非常に重要です。
  • nbatch > 1 の場合、BufFile を介したディスクI/Oが発生するため、パフォーマンス監視の対象としてください。
  • ExecParallelHashIncreaseNumBatches が呼び出されると、既存タプルの再パーティショニングが発生し、実行コストが急増します。
