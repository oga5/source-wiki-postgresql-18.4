pgstat_btree_page

定義

  • 種別: function
  • 定義箇所: contrib/pgstattuple/pgstattuple.c:410

目的

B-tree インデックスの指定されたページを走査し、統計情報（空き容量やタプル数）を収集・集計する。

シグネチャまたは形状

static void pgstat_btree_page(pgstattuple_type *stat, Relation rel, BlockNumber blkno, BufferAccessStrategy bstrategy)

入力と出力

  • **入力**:
  • stat: 統計情報を格納する pgstattuple_type 構造体へのポインタ。
  • rel: 対象のインデックスリレーション。
  • blkno: 走査対象のブロック番号。
  • bstrategy: バッファアクセスのための戦略オブジェクト。
  • **出力**: なし（stat 構造体のフィールドを直接更新する）。

前提条件と不変条件

  • rel は B-tree インデックスである必要がある。
  • 呼び出し元で適切なロック（AccessShareLock 等）が保持されていること。
  • ページ読み込み時に BT_READ モードでロックを取得し、処理終了後に解放する。

アルゴリズムまたは振る舞い

1. ReadBufferExtended を使用して指定ブロックを読み込み、BT_READ モードでロックする。 2. ページが新規（PageIsNew）の場合、ページ全体を空き容量として stat->free_space に加算する。 3. ページが B-tree 構造を持つ場合、BTPageOpaqueData を取得して以下の判定を行う：

  • P_IGNORE（削除済みまたは半死状態）の場合、ページ全体を空き容量として加算する。
  • リーフページ（P_ISLEAF）の場合、pgstat_index_page を呼び出し、ページ内のタプルを走査して統計を更新する。
  • 内部ページの場合、統計更新は行わずスキップする。

    • 4. _bt_relbuf を呼び出し、バッファのロックを解放して読み込みを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgstattuple/pgstattuple.c 内のインデックス走査ロジック（pgstat_index 等から間接的に呼び出される）。
  • **呼び出し先**:
  • ReadBufferExtended, LockBuffer, BufferGetPage, _bt_relbuf（PostgreSQL ストレージマネージャ）
  • PageIsNew, PageGetSpecialSize, BTPageGetOpaque, P_IGNORE, P_ISLEAF, P_FIRSTDATAKEY, PageGetMaxOffsetNumber（B-tree アクセスメソッド）
  • pgstat_index_page（統計集計用ヘルパー）

関連データ構造

  • pgstattuple_type: 統計結果を保持する構造体。
  • BTPageOpaqueData: B-tree ページのメタデータ構造。

失敗モードと境界条件

  • ページが破損している場合、PostgreSQL のバッファマネージャまたは B-tree アクセスメソッドの内部チェックによりエラーが発生する可能性がある。
  • CHECK_FOR_INTERRUPTS() は直接含まれていないが、呼び出し元の走査ループに含まれていることが想定される。

保守担当者向けメモ

  • 本関数は B-tree 特有のページ構造に依存している。他のインデックス型（Hash, GiST 等）については、それぞれ pgstat_hash_page や pgstat_gist_page が対応する。
  • 内部ページ（非リーフ）の統計は現在無視されているため、将来的にインデックスの深さや内部ノードの統計が必要になった場合は拡張が必要である。
