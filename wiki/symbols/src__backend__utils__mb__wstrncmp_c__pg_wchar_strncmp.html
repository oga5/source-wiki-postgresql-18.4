pg_wchar_strncmp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/wstrncmp.c:39

目的

2つの pg_wchar 配列を、最大 n 文字まで辞書順で比較します。

シグネチャまたは形状

int pg_wchar_strncmp(const pg_wchar *s1, const pg_wchar *s2, size_t n)

入力と出力

  • s1: 比較対象のワイド文字配列（入力）
  • s2: 比較対象のワイド文字配列（入力）
  • n: 比較する最大文字数（入力）
  • 戻り値:
  • 0: 両者が一致、または n 文字以内にヌル文字に到達した場合。
  • 0以外: 不一致が発生した位置での *s1 と *s2 の差分。

前提条件と不変条件

  • n が 0 の場合、比較を行わず即座に 0 を返します。
  • 比較中にヌル文字（0）に到達した場合は、その時点で比較を終了し、一致とみなして 0 を返します。

アルゴリズムまたは振る舞い

1. n が 0 ならば 0 を返す。 2. s1 と s2 の各要素を順次比較する。 3. 不一致が見つかった場合、*s1 - *s2 の値を返す。 4. s1 がヌル文字に到達した場合はループを終了し、0 を返す。 5. n 文字比較するまで、またはヌル文字に到達するまでループを継続する。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • 呼び出し先: 標準的な比較演算およびポインタ演算のみを使用しており、他の関数呼び出しは行いません。

関連データ構造

  • pg_wchar: PostgreSQL内部で使用されるワイド文字型。mb/pg_wchar.h で定義されています。

失敗モードと境界条件

  • n が 0 の場合: 常に 0 を返します。
  • 配列の長さが n 未満でヌル文字に到達した場合: 比較は正常に終了し 0 を返します。
  • メモリ保護違反: s1 または s2 が指すメモリ領域が n 文字分確保されていない場合、未定義の動作となります。

保守担当者向けメモ

  • 標準 C ライブラリの strncmp と同様のセマンティクスを持ちますが、対象が pg_wchar 型である点に留意してください。
  • 型変換を伴う比較が必要な場合は、本関数ではなく pg_char_and_wchar_strncmp を使用してください。
