text_to_cstring_buffer

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varlena.c:256

目的

PostgreSQLのtext型データを、指定されたバッファサイズのC文字列（ヌル終端文字列）へ安全にコピーします。

シグネチャまたは形状

void text_to_cstring_buffer(const text *src, char *dst, size_t dst_len)

入力と出力

  • **入力**:
  • src: 変換元のtext型データへのポインタ。
  • dst: コピー先のバッファへのポインタ。
  • dst_len: dstバッファの全サイズ（バイト単位）。
  • **出力**: なし（dstバッファに結果が書き込まれる）。

前提条件と不変条件

  • srcはTOAST（圧縮・外部化）されている可能性があるため、内部でpg_detoast_datum_packedにより展開されます。
  • dst_lenが0の場合、コピーは行われません。
  • dst_lenが1以上の場合、必ずヌル終端文字（\0）が書き込まれます。

アルゴリズムまたは振る舞い

1. pg_detoast_datum_packedを用いてsrcをメモリ上に展開します。 2. dst_lenが0より大きい場合、以下の処理を行います。

  • バッファサイズをヌル終端分確保するためdst_lenを1減らします。
  • srcのデータ長がdst_lenを超える場合、マルチバイト文字の整合性を維持するためpg_mbcliplenを用いて切り詰めを行います。
  • memcpyでデータをコピーし、最後にヌル終端文字を付与します。

    • 3. 展開のために一時メモリを割り当てた場合（srcunpacked != src）、pfreeで解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgcrypto.c
  • src/backend/libpq/be-fsstubs.c
  • src/backend/utils/adt/date.c
  • src/backend/utils/adt/timestamp.c
  • **呼び出し先**:
  • pg_detoast_datum_packed
  • pg_mbcliplen
  • memcpy
  • pfree

関連データ構造

  • text: PostgreSQLの可変長文字列型。

失敗モードと境界条件

  • dst_lenが0の場合、何も書き込まれません。

保守担当者向けメモ

  • src引数はconstですが、内部でunconstifyして展開処理を行うため、呼び出し側は元のデータが変更されないことを期待できます。
  • pg_detoast_datum_packedを使用しているため、TOASTされたデータに対しても透過的に動作します。
