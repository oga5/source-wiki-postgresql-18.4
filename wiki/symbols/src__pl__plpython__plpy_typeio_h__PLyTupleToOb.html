PLyTupleToOb

定義

  • 種別: struct
  • 定義箇所: src/pl/plpython/plpy_typeio.h:40

目的

PostgreSQLの複合型（タプル）をPythonオブジェクトへ変換する際のメタデータおよび変換状態を保持する。

シグネチャまたは形状

typedef struct PLyTupleToOb
{
    TupleDesc    recdesc;
    TypeCacheEntry *typentry;
    uint64       tupdescid;
    PLyDatumToOb *atts;
    int          natts;
} PLyTupleToOb;

入力と出力

本構造体は変換処理のコンテキストとして機能します。

  • **入力**: 変換対象のタプル情報（recdesc または typentry）。
  • **出力**: atts フィールドに格納された各カラムごとの変換情報（PLyDatumToOb 配列）。

前提条件と不変条件

  • atts および natts は、初期化前はそれぞれ NULL および 0 である必要があります。
  • typentry は名前付き複合型の場合に設定され、recdesc は RECORD 型の場合に使用されます。

アルゴリズムまたは振る舞い

  • 複合型の各属性（カラム）に対して再帰的に変換を行うための情報を管理します。
  • tupdescid を用いて、typcache 内のタプル記述子の変更を追跡し、必要に応じて変換情報の再構築を判断します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元情報は提供されていません。
  • **呼び出し先**: PLyDatumToOb 構造体を参照し、各カラムの変換処理を委譲します。

関連データ構造

  • TupleDesc: タプルの構造定義。
  • TypeCacheEntry: 型情報のキャッシュ。
  • PLyDatumToOb: 各属性の変換状態を保持する構造体。

失敗モードと境界条件

  • natts が 0 の場合、変換対象の属性が存在しないか、初期化が完了していないことを示します。
  • typentry や recdesc が無効な場合、変換処理は失敗します。

保守担当者向けメモ

  • 本構造体は plpy_typeio.c での利用を前提としています。
  • 変換処理は再帰的であるため、メモリコンテキストの管理には十分注意してください。
  • tupdescid の不一致を検知した際は、変換情報の再セットアップが必要です。
