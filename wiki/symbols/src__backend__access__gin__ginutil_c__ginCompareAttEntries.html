ginCompareAttEntries

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginutil.c:414

目的

GINインデックスにおける2つのエントリを、属性番号を優先キーとして比較します。

シグネチャまたは形状

int ginCompareAttEntries(GinState *ginstate,
                         OffsetNumber attnuma, Datum a, GinNullCategory categorya,
                         OffsetNumber attnumb, Datum b, GinNullCategory categoryb)

入力と出力

  • **入力**:
  • ginstate: GINインデックスの実行時状態。
  • attnuma, attnumb: 比較対象の属性番号。
  • a, b: 比較対象のデータ値（Datum）。
  • categorya, categoryb: 値がNULLであるか、あるいはどのようなカテゴリに属するかを示すGinNullCategory。
  • **出力**:
  • 比較結果を整数で返します。attnuma < attnumb なら -1、attnuma > attnumb なら 1。属性番号が等しい場合は ginCompareEntries の結果を返します。

前提条件と不変条件

  • ginstate は initGinState 等で適切に初期化されている必要があります。
  • 属性番号が異なる場合、値の内容に関わらず属性番号の大小関係が優先されます。

アルゴリズムまたは振る舞い

1. 属性番号 attnuma と attnumb を比較します。 2. 属性番号が異なれば、その大小関係に基づき -1 または 1 を即座に返します。 3. 属性番号が等しい場合、ginCompareEntries を呼び出し、データ値とカテゴリに基づいて詳細な比較を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c
  • src/backend/access/gin/ginbulk.c
  • src/backend/access/gin/ginentrypage.c
  • **呼び出し先**:
  • ginCompareEntries (同ファイル内)

関連データ構造

  • GinState: GINインデックスの演算クラス情報や比較関数を保持する構造体。
  • GinNullCategory: NULL値の扱いを定義する列挙型。

失敗モードと境界条件

  • ginstate が無効な場合、ginCompareEntries 内部でセグメンテーション違反や予期せぬ動作が発生する可能性があります。
  • 属性番号が一致し、かつ ginCompareEntries が提供する比較関数が適切に定義されていない場合、比較結果は不定となります。

保守担当者向けメモ

  • 本関数は属性番号によるソート順を強制するラッパーです。比較ロジックの変更は ginCompareEntries 側で行う必要があります。
  • contrib/amcheck から頻繁に呼び出されているため、変更時はインデックス整合性チェックへの影響を考慮してください。
