ts_typanalyze

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_typanalyze.c:57

目的

tsvector 型カラムに対する ANALYZE 処理をカスタマイズし、統計情報収集のための設定を行う。

シグネチャまたは形状

Datum ts_typanalyze(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (内部的に VacAttrStats 構造体へのポインタを抽出)
  • **出力**: Datum (常に true を返す)

前提条件と不変条件

  • 引数として渡される stats ポインタが有効であること。
  • stats->attstattarget が負の値の場合、デフォルトの統計ターゲット値で上書きされる。

アルゴリズムまたは振る舞い

1. stats->attstattarget が負であれば、システム全体の default_statistics_target を適用する。 2. 統計計算関数として compute_tsvector_stats を stats->compute_stats に登録する。 3. 統計収集に必要な最小行数 minrows を 300 * stats->attstattarget として設定する。 4. 処理成功として true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の ANALYZE コマンド実行プロセス（直接的な呼び出し元コードは本ソース抜粋には含まれていない）。
  • **呼び出し先**: compute_tsvector_stats（関数ポインタとして代入）。

関連データ構造

  • VacAttrStats: ANALYZE 処理中に統計情報を保持する構造体。

失敗モードと境界条件

  • attstattarget が負の場合の補正処理が行われる。
  • 最小行数 minrows の計算において、attstattarget の値に依存したスケーリングが行われる。

保守担当者向けメモ

  • 本関数は tsvector 型専用の typanalyze エントリポイントである。
  • minrows の計算式（300 * attstattarget）については、src/backend/commands/analyze.c 内のコメントを参照すること。
