jsonb_exec_setup

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonbsubs.c:352

目的

jsonb 型の添字操作（subscripting）を実行するためのワークスペースを初期化し、実行時に必要なメソッドを登録する。

シグネチャまたは形状

static void jsonb_exec_setup(const SubscriptingRef *sbsref,
                             SubscriptingRefState *sbsrefstate,
                             SubscriptExecSteps *methods)

入力と出力

  • **入力**:
  • sbsref: 添字操作の構文解析結果を含む SubscriptingRef 構造体。
  • sbsrefstate: 実行状態を保持する SubscriptingRefState 構造体。
  • methods: 実行ステップの関数ポインタを格納する SubscriptExecSteps 構造体。
  • **出力**: なし（sbsrefstate->workspace の設定および methods 構造体への関数ポインタ代入による副作用）。

前提条件と不変条件

  • sbsref->refupperindexpr が存在し、添字式のリストとして有効であること。
  • sizeof(Datum) >= sizeof(Oid) であることを前提に、メモリレイアウトが計算されている。

アルゴリズムまたは振る舞い

1. JsonbSubWorkspace 構造体と、添字数に応じた Datum および Oid 配列を保持するためのメモリを palloc0 で確保する。 2. workspace->expectArray を false に初期化する。 3. 確保したメモリ領域を workspace->index および workspace->indexOid に割り当てる。 4. sbsrefstate->workspace に作成したワークスペースを格納する。 5. sbsref->refupperindexpr を走査し、各添字式の型（Oid）を workspace->indexOid に記録する。 6. methods 構造体に以下の関数ポインタを登録する:

  • sbs_check_subscripts: jsonb_subscript_check_subscripts
  • sbs_fetch: jsonb_subscript_fetch
  • sbs_assign: jsonb_subscript_assign
  • sbs_fetch_old: jsonb_subscript_fetch_old

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonbsubs.c 内の jsonb_subscript_handler 構造体（.exec_setup フィールド）を通じて呼び出される。
  • **呼び出し先**:
  • palloc0 (メモリ確保)
  • exprType (式から型を取得)
  • jsonb_subscript_check_subscripts, jsonb_subscript_fetch, jsonb_subscript_assign, jsonb_subscript_fetch_old (関数ポインタの代入先)

関連データ構造

  • JsonbSubWorkspace: 実行時の添字情報や型情報を保持する構造体。
  • SubscriptingRef: 構文解析された添字情報。
  • SubscriptingRefState: 実行時の状態管理。
  • SubscriptExecSteps: 実行ステップの関数テーブル。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる。
  • sbsref->refupperindexpr が空の場合、nupper が 0 となり、配列領域の確保は行われない。

保守担当者向けメモ

  • 本関数は jsonb の添字操作のセットアップを行うブリッジ関数である。
  • methods 構造体への関数ポインタ代入は、jsonb の添字操作のライフサイクルを決定するため、新しい操作関数を追加する際はここを更新する必要がある。
