validator_shutdown

定義

  • 種別: function
  • 定義箇所: src/test/modules/oauth_validator/validator.c:108

目的

OAuth検証モジュールの終了処理において、保持している内部状態（private_data）の整合性を検証する。

シグネチャまたは形状

static void validator_shutdown(ValidatorModuleState *state)

入力と出力

  • **入力**: ValidatorModuleState *state（検証モジュールの状態を保持する構造体へのポインタ）
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • state->private_data には、validator_startup で設定された PRIVATE_COOKIE が格納されている必要がある。

アルゴリズムまたは振る舞い

1. 引数 state が保持する private_data の値を確認する。 2. private_data が PRIVATE_COOKIE と一致しない場合、elog(PANIC, ...) を呼び出し、サーバープロセスを異常終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: validator_callbacks 構造体を通じて、PostgreSQLサーバーのOAuth検証フックから呼び出される（src/test/modules/oauth_validator/validator.c:36）。
  • **呼び出し先**: elog（PostgreSQLのログ出力およびエラー報告関数）。

関連データ構造

  • ValidatorModuleState: 検証モジュールの状態を管理する構造体。
  • PRIVATE_COOKIE: 状態の整合性を確認するために使用される定数。

失敗モードと境界条件

  • private_data が期待値（PRIVATE_COOKIE）と異なる場合、PANIC レベルのエラーが発生し、サーバープロセスが停止する。

保守担当者向けメモ

  • 本関数はテストモジュール用であり、実運用環境では推奨されないデバッグ用の整合性チェックが含まれている。
  • 呼び出し元に関する情報は、validator_callbacks 構造体への登録経由であること以外、ソースコード上では明示的な直接呼び出し箇所は確認できない。
