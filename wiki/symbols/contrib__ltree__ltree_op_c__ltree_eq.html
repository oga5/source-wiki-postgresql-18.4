ltree_eq

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_op.c:108

目的

2つの ltree 型の値が等しいかどうかを判定し、SQLの等価演算子（=）のバックエンドとして機能します。

シグネチャまたは形状

Datum ltree_eq(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される2つの ltree 型のポインタ。
  • **出力**: bool 型（Datum 形式）。両者が等しい場合は true、それ以外は false を返します。

前提条件と不変条件

  • RUNCMP マクロが定義されている環境で実行される必要があります。
  • 入力される ltree は有効な構造体である必要があります。

アルゴリズムまたは振る舞い

1. RUNCMP マクロを実行し、内部的に ltree_compare を呼び出して2つの ltree を比較します。 2. 比較結果が格納された変数 res を評価します。 3. res == 0（比較結果が等価）であれば true を、そうでなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジンから演算子として呼び出されます。ソースコード内では PG_FUNCTION_INFO_V1(ltree_eq) による登録が確認されています。
  • **呼び出し先**: RUNCMP マクロを通じて ltree_compare を呼び出します。

関連データ構造

  • ltree: 階層構造を持つラベルパスを格納するデータ型。
  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約に従う引数コンテナ。

失敗モードと境界条件

  • 入力が NULL の場合、PostgreSQL の関数呼び出し規約により NULL が返されます。
  • ltree の構造が破損している場合、ltree_compare の内部で予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は ltree_compare の結果に完全に依存しています。比較ロジックの変更が必要な場合は ltree_compare を参照してください。
  • RUNCMP マクロは引数の取得と比較を簡略化する共通マクロであるため、変更時は他の比較関数への影響を考慮してください。
