remove_new_subdir

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.c:707

目的

新クラスタのデータディレクトリ配下にある指定されたサブディレクトリ内のファイルを削除し、クリーンアップを行う。

シグネチャまたは形状

static void remove_new_subdir(const char *subdir, bool rmtopdir)

入力と出力

  • **入力**:
  • subdir: 削除対象のサブディレクトリ名（new_cluster.pgdata からの相対パス）。
  • rmtopdir: true の場合、指定されたディレクトリ自体も削除する。false の場合、ディレクトリ内のファイルのみを削除する。
  • **出力**: なし（成功時はログ出力を行い、失敗時は pg_fatal でプロセスを終了する）。

前提条件と不変条件

  • new_cluster グローバル構造体が初期化されていること。
  • 指定された subdir が new_cluster.pgdata 配下に存在すること。

アルゴリズムまたは振る舞い

1. prep_status を呼び出し、削除対象のサブディレクトリ名をログに記録する。 2. new_cluster.pgdata と subdir を結合し、完全なパスを new_path に生成する。 3. rmtree(new_path, rmtopdir) を呼び出し、ファイルまたはディレクトリの削除を実行する。 4. 削除に失敗した場合は pg_fatal を呼び出し、エラーメッセージを出力して終了する。 5. 成功した場合は check_ok() を呼び出し、完了ステータスを表示する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/pg_upgrade.c:730

  • **呼び出し先**:
  • prep_status (外部関数)

関連データ構造

  • new_cluster: pg_upgrade.c で定義されるグローバル構造体。pgdata メンバが新クラスタのデータディレクトリパスを保持する。

失敗モードと境界条件

  • rmtree が失敗した場合（権限不足やファイルロック等）、pg_fatal によりプログラムが異常終了する。

保守担当者向けメモ

  • 本関数は pg_upgrade における物理的なファイル削除操作を抽象化している。
  • rmtree の挙動に依存するため、Windows 環境等でのファイルロックの影響を受ける可能性がある。
