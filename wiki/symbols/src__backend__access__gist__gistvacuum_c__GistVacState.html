GistVacState

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gist/gistvacuum.c:29

目的

GiST インデックスの VACUUM 処理中に、スキャン状態、統計情報、および削除対象のページ情報を保持するための一時的なコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    IndexVacuumInfo *info;
    IndexBulkDeleteResult *stats;
    IndexBulkDeleteCallback callback;
    void *callback_state;
    GistNSN startNSN;
    IntegerSet *internal_page_set;
    IntegerSet *empty_leaf_set;
    MemoryContext page_set_context;
} GistVacState;

入力と出力

  • **入力**: VACUUM 実行時に gistvacuumscan 内で初期化され、各ページ処理関数（gistvacuumpage 等）へポインタとして渡されます。
  • **出力**: 処理結果として stats（IndexBulkDeleteResult）が更新され、empty_leaf_set 等に蓄積された情報が gistvacuum_delete_empty_pages で利用されます。

前提条件と不変条件

  • page_set_context は、internal_page_set および empty_leaf_set のメモリ管理に使用されます。
  • startNSN は、スキャン開始時のインデックスの状態を追跡するために使用されます。

アルゴリズムまたは振る舞い

1. **状態保持**: VACUUM のスキャン全体を通じて、削除対象のタプル判定用コールバックと、空になったリーフページおよび内部ページの集合を保持します。 2. **ページ管理**: ページスキャン中に空になったリーフページを empty_leaf_set に記録し、後段の gistvacuum_delete_empty_pages でまとめてツリーから切り離すための基盤となります。 3. **メモリ管理**: page_set_context を通じて、ページセット管理に必要なメモリを効率的に確保・解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: gistvacuumscan（src/backend/access/gist/gistvacuum.c:129）でインスタンス化されます。
  • **呼び出し先**:
  • gistvacuumpage（src/backend/access/gist/gistvacuum.c:308）に引数として渡されます。
  • gistvacuum_delete_empty_pages（src/backend/access/gist/gistvacuum.c:504）に引数として渡されます。

関連データ構造

  • IntegerSet: 空ページ番号の管理に使用。
  • IndexBulkDeleteResult: VACUUM の統計情報格納用。
  • MemoryContext: 構造体内の動的メモリ管理用。

失敗モードと境界条件

  • メモリ不足: page_set_context でのメモリ確保失敗時にエラーが発生する可能性があります。
  • ページセットの肥大化: インデックス内の空ページが極端に多い場合、IntegerSet がメモリを消費します。

保守担当者向けメモ

  • 本構造体は gistvacuumscan の実行期間中のみ有効な一時的な状態保持用です。
  • internal_page_set と empty_leaf_set は、空ページの再帰的な削除処理において重要な役割を果たすため、これらのセットの整合性は gistvacuumpage 内のロジックに依存しています。
