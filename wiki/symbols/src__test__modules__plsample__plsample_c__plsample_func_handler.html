plsample_func_handler

定義

  • 種別: function
  • 定義箇所: src/test/modules/plsample/plsample.c:92

目的

PostgreSQL の手続き型言語（PL）ハンドラにおける、通常の関数呼び出しを処理するためのテンプレート実装。関数定義のメタデータ取得、引数の解析とログ出力、および戻り値の変換を行う。

シグネチャまたは形状

static Datum plsample_func_handler(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（fcinfo を通じて関数呼び出し情報を受け取る）。
  • **出力**: Datum 型の戻り値。戻り値の型が TEXTOID の場合はソースコード文字列を返し、それ以外の場合は NULL を返す。

前提条件と不変条件

  • fcinfo->flinfo->fn_oid が有効な関数 OID であること。
  • pg_proc システムカタログから対象関数のメタデータが取得可能であること。
  • 戻り値の型が TEXT 以外の場合、PG_RETURN_NULL() が実行される。

アルゴリズムまたは振る舞い

1. **メタデータ取得**: SearchSysCache1 を使用して pg_proc から関数情報を取得し、ソースコードを NOTICE レベルでログ出力する。 2. **コンテキスト作成**: AllocSetContextCreate により、関数実行用のメモリコンテキストを生成する。 3. **引数処理**: get_func_arg_info で引数情報を取得し、各引数の値を OutputFunctionCall で変換して NOTICE ログに出力する。 4. **戻り値処理**:

  • 戻り値の型が TEXT の場合、InputFunctionCall を用いてソースコードを Datum に変換して返す。
  • それ以外の場合は PG_RETURN_NULL() を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: plsample_call_handler（src/test/modules/plsample/plsample.c:76）。
  • **呼び出し先**: PostgreSQL の内部 API（SearchSysCache1, SysCacheGetAttr, AllocSetContextCreate, get_func_arg_info, fmgr_info_cxt, OutputFunctionCall, InputFunctionCall, ReleaseSysCache 等）。

関連データ構造

  • HeapTuple: システムカタログ（pg_proc, pg_type）の行データ。
  • Form_pg_proc: 関数定義の構造体。
  • Form_pg_type: 型定義の構造体。
  • FmgrInfo: 関数呼び出し管理情報。
  • MemoryContext: 関数実行時のメモリ管理コンテキスト。

失敗モードと境界条件

  • SearchSysCache1 が失敗した場合、elog(ERROR) により処理を中断する。
  • 関数ソースコードが取得できない場合、elog(ERROR) を発生させる。
  • 引数や戻り値の型情報がシステムキャッシュから取得できない場合、elog(ERROR) を発生させる。

保守担当者向けメモ

  • 本関数はテンプレートであり、実際の言語処理系は含まれていない。
  • AllocSetContextCreate は関数実行ごとのメモリ管理の雛形として機能する。
  • 戻り値の処理は TEXT 型に限定されており、他の型をサポートする場合は InputFunctionCall の呼び出しロジックを拡張する必要がある。
