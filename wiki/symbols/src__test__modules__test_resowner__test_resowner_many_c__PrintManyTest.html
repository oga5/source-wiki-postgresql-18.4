PrintManyTest

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_resowner/test_resowner_many.c:81

目的

ResourceOwner によって管理されるリソースがリークした際に、デバッグ出力用の文字列を生成し、該当するリソース種別のリークカウンタをインクリメントします。

シグネチャまたは形状

static char *PrintManyTest(Datum res);

入力と出力

  • **入力**: Datum res（ManyTestResource 構造体へのポインタを保持する Datum 型）
  • **出力**: char *（リソースの種類を示す説明文字列）

前提条件と不変条件

  • 入力 res は ManyTestResource 構造体としてキャスト可能である必要があります。
  • mres->kind が有効な ManyTestResourceKind を指している必要があります。

アルゴリズムまたは振る舞い

1. 引数 res を ManyTestResource 型のポインタに変換します。 2. 該当リソースの種別（mres->kind）が保持するリークカウンタ nleaked を 1 増加させます。 3. psprintf を使用して、リソース種別の名前を含む説明文字列を動的に生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ResourceOwner のデバッグ出力機構から呼び出されることが想定されています（ソースコード内のコメントより）。
  • **呼び出し先**: psprintf（文字列生成用）。
  • **補足**: 直接的な呼び出し元はソースコードの文脈からは特定されていませんが、ManyTestResourceKind の DebugPrint フィールドに本関数が代入されています。

関連データ構造

  • ManyTestResource: リソースのインスタンスを保持する構造体。
  • ManyTestResourceKind: リソースの種別とリークカウンタを保持する構造体。

失敗モードと境界条件

  • mres が NULL の場合、あるいは mres->kind が初期化されていない場合はセグメンテーション違反が発生する可能性があります。
  • 本関数は「各リークしたリソースに対して一度だけ呼び出される」という前提で実装されています。

保守担当者向けメモ

  • 本関数は ResourceOwner のデバッグ用コールバックとして機能します。
  • nleaked のインクリメントは副作用であるため、テスト実行時のリーク検証ロジックと密接に関連しています。変更時は ResourceOwner のデバッグ出力の呼び出し頻度を確認してください。
