DictInt

定義

  • 種別: struct
  • 定義箇所: contrib/dict_int/dict_int.c:23

目的

PostgreSQL の全文検索（TSearch）において、整数値を正規化またはフィルタリングするための辞書設定値を保持する構造体です。

シグネチャまたは形状

typedef struct
{
	int			maxlen;
	bool		rejectlong;
	bool		absval;
} DictInt;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。ただし、dintdict_init 関数によって設定値が読み込まれ、dintdict_lexize 関数によってトークン処理の制御パラメータとして参照されます。

前提条件と不変条件

  • dintdict_init を通じてメモリ上に構築される必要があります。
  • palloc0 によりゼロ初期化された状態で確保されることが一般的です。

アルゴリズムまたは振る舞い

本構造体は以下の設定値を保持し、全文検索時のトークン処理を制御します。

  • maxlen: 整数トークンの最大長。
  • rejectlong: true の場合、maxlen を超えるトークンを破棄します。false の場合、maxlen で切り詰めます。
  • absval: true の場合、トークンの先頭にある符号（+ / -）を無視して処理します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • dintdict_init（contrib/dict_int/dict_int.c:41）: 構造体のメモリ確保に使用。
  • dintdict_lexize（contrib/dict_int/dict_int.c:82）: 辞書設定の取得に使用。
  • 呼び出し先: 情報なし。

関連データ構造

  • TSLexeme: dintdict_lexize がトークン処理結果を返す際に使用するデータ構造。

失敗モードと境界条件

  • 構造体自体の失敗モードは定義されていませんが、maxlen の設定値が不適切な場合、dintdict_lexize におけるトークン処理の挙動（切り詰めまたは破棄）に直接影響します。

保守担当者向けメモ

  • 本構造体は palloc0 で確保されるため、メモリ管理は PostgreSQL のメモリコンテキストに依存します。
  • 構造体のフィールド追加や変更を行う際は、dintdict_init でのオプション解析ロジックと、dintdict_lexize での参照ロジックの両方を更新してください。
