getRootTableInfo

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dump.c:2762

目的

パーティションテーブルの階層を遡り、その最上位にあるルートテーブル（パーティション化されていない親テーブル）の TableInfo を取得する。

シグネチャまたは形状

static TableInfo *getRootTableInfo(const TableInfo *tbinfo)

入力と出力

  • **入力**: tbinfo (対象となるパーティションテーブルの TableInfo 構造体へのポインタ)
  • **出力**: パーティション階層の最上位に位置する TableInfo 構造体へのポインタ

前提条件と不変条件

  • 入力された tbinfo はパーティションであること (tbinfo->ispartition が真)。
  • 入力された tbinfo は単一の親を持つこと (tbinfo->numParents == 1)。
  • 階層を遡る過程で遭遇する各親テーブルも、同様に単一の親を持つこと。

アルゴリズムまたは振る舞い

1. 引数 tbinfo がパーティションであることを確認する。 2. tbinfo->parents[0] を現在の親として取得する。 3. 現在の親がパーティションである限り、その親の親を辿るループを継続する。 4. パーティションではない（ispartition が偽の）親に到達した時点で、そのテーブル情報を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c:2579
  • src/bin/pg_dump/pg_dump.c:2843
  • **呼び出し先**:
  • 外部関数呼び出しなし。構造体メンバの参照のみを行う。

関連データ構造

  • TableInfo: pg_dump におけるテーブル定義を保持する構造体。ispartition フラグおよび parents 配列を介して継承・パーティション階層を管理する。

失敗モードと境界条件

  • tbinfo->ispartition が偽の場合、Assert によりプログラムが停止する。
  • tbinfo->numParents が 1 ではない場合、Assert によりプログラムが停止する。

保守担当者向けメモ

  • 本関数は TableInfo の階層構造が単一の継承パス（numParents == 1）であることを前提としている。将来的に多重継承や複雑なパーティション構成が導入された場合、このロジックは再検討が必要となる可能性がある。
