GetNamedDSMSegment

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/dsm_registry.c:131

目的

名前付き動的共有メモリ（DSM）セグメントを検索または作成し、呼び出し元のアドレス空間にマッピングしてその先頭アドレスを返します。

シグネチャまたは形状

void *GetNamedDSMSegment(const char *name, size_t size, void (*init_callback) (void *ptr), bool *found);

入力と出力

  • **入力**:
  • name: セグメントを一意に識別する文字列（64バイト未満）。
  • size: セグメントのサイズ（0は不可）。
  • init_callback: セグメント新規作成時に実行される初期化関数（任意）。
  • found: セグメントが既に存在したかを示すフラグを格納するポインタ。
  • **出力**:
  • 戻り値: マッピングされた共有メモリセグメントの先頭アドレス。

前提条件と不変条件

  • found は非NULLであること。
  • name は空文字列であってはならない。
  • 既存のセグメントと要求サイズが一致しない場合はエラーとなる。
  • 処理中のメモリ割り当ては TopMemoryContext で行われる。

アルゴリズムまたは振る舞い

1. TopMemoryContext へ切り替え、init_dsm_registry() を呼び出してレジストリを準備する。 2. dshash_find_or_insert を使用してハッシュテーブルから name に対応するエントリを検索する。 3. **新規作成時**:

  • dsm_create でセグメントを作成し、init_callback が指定されていれば実行する。
  • dsm_pin_segment および dsm_pin_mapping を実行してセグメントを永続化する。

    • 4. **既存時**:

  • dsm_find_mapping で既にアタッチされているか確認し、未アタッチであれば dsm_attach を行う。

    • 5. dshash_release_lock でハッシュテーブルのロックを解放し、メモリコンテキストを復元してアドレスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_prewarm/autoprewarm.c
  • src/test/modules/injection_points/injection_points.c
  • src/test/modules/test_dsm_registry/test_dsm_registry.c
  • **呼び出し先**:
  • init_dsm_registry
  • dshash_find_or_insert / dshash_release_lock
  • dsm_create / dsm_attach / dsm_find_mapping / dsm_segment_address / dsm_segment_handle / dsm_pin_segment / dsm_pin_mapping
  • MemoryContextSwitchTo

関連データ構造

  • DSMRegistryEntry: ハッシュテーブルに格納される管理構造体。
  • dsm_segment: DSMセグメントの制御構造体。

失敗モードと境界条件

  • name が空または長すぎる（64バイト以上）場合、エラー。
  • size が0の場合、エラー。
  • 既存セグメントと要求サイズが不一致の場合、エラー。
  • dsm_attach に失敗した場合、エラー。

保守担当者向けメモ

  • init_callback はセグメントが最初に作成された時のみ実行される点に注意すること。
  • DSM関連の割り当てはバックエンドの寿命を超えて永続的である必要があるため、TopMemoryContext を使用している。
