avc_cache

定義

  • 種別: struct
  • 定義箇所: contrib/sepgsql/uavc.c:30

目的

SELinux のアクセス制御決定（権限判定結果）をユーザー空間で保持し、カーネルへのシステムコール呼び出し頻度を削減することで、SELinux 統合によるパフォーマンス低下を最小限に抑えるためのキャッシュエントリ構造体です。

シグネチャまたは形状

typedef struct {
    uint32      hash;               /* キャッシュエントリのハッシュ値 */
    char       *scontext;           /* 主体（Subject）のセキュリティコンテキスト */
    char       *tcontext;           /* 対象（Target）のセキュリティコンテキスト */
    uint16      tclass;             /* 対象のオブジェクトクラス */
    uint32      allowed;            /* 許可される権限ビットマスク */
    uint32      auditallow;         /* 許可時に監査する権限ビットマスク */
    uint32      auditdeny;          /* 拒否時に監査する権限ビットマスク */
    bool        permissive;         /* permissive ルール適用時は true */
    bool        hot_cache;          /* 最近参照された場合は true */
    bool        tcontext_is_valid;  /* tcontext が有効な場合は true */
    char       *ncontext;           /* 信頼されたプロシージャ実行時のテンポラリ scontext */
} avc_cache;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。キャッシュ生成時には palloc0 を介してメモリが割り当てられ、検索や計算の結果が各フィールドに格納されます。

前提条件と不変条件

  • キャッシュは avc_mem_cxt という専用のメモリコンテキストで管理されます。
  • ncontext フィールドは、DB_PROCEDURE クラスの権限判定において、実行時に切り替えるべきセキュリティラベルを保持するために使用されます。それ以外の場合は NULL です。

アルゴリズムまたは振る舞い

  • **LRU 管理**: hot_cache フラグにより、参照頻度に基づいたメモリ管理（再利用）をサポートします。
  • **キャッシュ検索**: sepgsql_avc_lookup を通じてハッシュテーブルから検索されます。
  • **ポリシー更新**: SELinux ポリシーのリロードが検知されると、キャッシュの妥当性が検証され、必要に応じてフラッシュされます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/uavc.c 内の検索関数（sepgsql_avc_lookup 等）や、キャッシュオブジェクトを生成する関数から参照されます。
  • **呼び出し先**: 本構造体はデータ構造であるため、直接的な呼び出し先はありません。

関連データ構造

  • sepgsql.h: SEPG_CLASS_* などのオブジェクトクラス定義。
  • avc_mem_cxt: キャッシュエントリを保持するメモリコンテキスト。

失敗モードと境界条件

  • キャッシュエントリが閾値を超えた場合、sepgsql_avc_reclaim により参照頻度の低いエントリが解放されます。
  • 権限判定中にポリシーが更新された場合、sepgsql_avc_check_valid を用いた再試行ループが必要です。

保守担当者向けメモ

  • 権限チェックはシステムコールを跨ぐ可能性があるため、本構造体を利用する際は sepgsql_avc_check_valid をループ内で使用するイディオムを厳守してください。
  • DB_PROCEDURE クラスのキャッシュ時には、ncontext の取り扱いに注意してください。
