init_trgm_array

定義

  • 種別: function
  • 定義箇所: contrib/pg_trgm/trgm_op.c:91

目的

動的に拡張可能なトリグラム配列管理構造体（growable_trgm_array）を初期化し、トリグラム格納用のメモリ領域を確保する。

シグネチャまたは形状

void init_trgm_array(growable_trgm_array *arr, int slen)

入力と出力

  • **入力**:
  • arr: 初期化対象の growable_trgm_array 構造体へのポインタ。
  • slen: 入力文字列の長さ。
  • **出力**: なし（arr の各フィールドが更新される）。

前提条件と不変条件

  • arr は有効なメモリ領域を指している必要がある。
  • slen はトリグラム生成の基となる文字列の長さである。
  • palloc によるメモリ確保が成功すること。

アルゴリズムまたは振る舞い

1. 入力文字列長 slen に基づき、必要な初期バッファサイズを計算する（slen + 1）。 2. MaxAllocSize を超えるメモリ要求が発生しないかチェックし、超過する場合は ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラーを報告する。 3. CALCGTSIZE マクロを使用して必要なメモリ量を算出し、palloc で確保する。 4. 構造体 arr の datum に確保したメモリを割り当て、allocated にサイズを、length に 0 を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_trgm/trgm_op.c 内の generate_trgm 等（ソースコード内の直接的な呼び出し箇所として line 460 および line 1063 が確認されている）。
  • **呼び出し先**:
  • palloc (PostgreSQL メモリ管理関数)
  • ereport (エラー報告用マクロ)

関連データ構造

  • growable_trgm_array: トリグラム配列を動的に管理するための構造体。
  • trgm: トリグラムを表現する型。

失敗モードと境界条件

  • init_size が MaxAllocSize / sizeof(trgm) を超える場合、メモリ確保前にエラーを発生させる。
  • palloc 自体がメモリ不足で失敗した場合は、PostgreSQL の標準的なメモリ管理機構によりエラーがスローされる。

保守担当者向けメモ

  • 本関数はトリグラム生成の初期化フェーズで必須となる。
  • CALCGTSIZE マクロの定義および growable_trgm_array の構造については trgm.h を参照すること。
  • メモリ確保後の拡張が必要な場合は、対となる enlarge_trgm_array を使用すること。
