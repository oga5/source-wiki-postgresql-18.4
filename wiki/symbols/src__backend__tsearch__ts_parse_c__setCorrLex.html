setCorrLex

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_parse.c:119

目的

LexizeData 構造体が保持する waste リスト（処理済みトークン群）の所有権を呼び出し元に委譲するか、あるいはメモリを解放してリストをクリアする。

シグネチャまたは形状

static void setCorrLex(LexizeData *ld, ParsedLex **correspondLexem)

入力と出力

  • **入力**:
  • ld: 処理状態を保持する LexizeData 構造体へのポインタ。
  • correspondLexem: ParsedLex リストの先頭アドレスを受け取るためのポインタ（NULL許容）。
  • **出力**: なし（副作用として ld->waste を更新）。

前提条件と不変条件

  • ld は有効な LexizeData 構造体を指していること。
  • ld->waste リスト内の各要素は palloc で確保されていること。

アルゴリズムまたは振る舞い

1. correspondLexem が NULL でない場合、ld->waste.head を *correspondLexem に代入し、呼び出し元へリストの所有権を渡す。 2. correspondLexem が NULL の場合、ld->waste.head から始まるリストを走査し、各要素を pfree で解放する。 3. 最後に ld->waste.head および ld->waste.tail を NULL にリセットし、リストを空にする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/ts_parse.c 内の LexizeExec 関数（行240, 332, 345）。
  • **呼び出し先**:
  • pfree（メモリ解放用）。

関連データ構造

  • LexizeData: 辞書処理の状態を管理する構造体。
  • ParsedLex: トークン情報を保持する構造体。

失敗モードと境界条件

  • ld->waste.head が NULL の場合、リストの解放処理はスキップされ、単に head と tail が NULL に再設定される。

保守担当者向けメモ

  • 本関数は LexizeExec における辞書適用後の後処理として機能する。
  • correspondLexem を介してリストを外部に渡す場合、呼び出し元で適切にメモリ管理（解放）が行われる必要がある点に注意すること。
