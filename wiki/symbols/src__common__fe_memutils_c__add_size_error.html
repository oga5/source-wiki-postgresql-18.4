add_size_error

定義

  • 種別: function
  • 定義箇所: src/common/fe_memutils.c:206

目的

メモリ確保サイズの計算時に発生した加算オーバーフローを報告し、プロセスを異常終了させるためのエラーハンドラです。

シグネチャまたは形状

pg_noreturn void add_size_error(Size s1, Size s2);

入力と出力

  • **入力**:
  • s1 (Size): 加算対象のサイズ値。
  • s2 (Size): 加算対象のサイズ値。
  • **出力**: なし（関数は復帰しません）。

前提条件と不変条件

  • 本関数は、メモリ確保サイズの加算においてオーバーフローが検出された直後に呼び出されることを想定しています。
  • pg_noreturn 属性により、呼び出し元へ制御が戻ることはありません。

アルゴリズムまたは振る舞い

1. 標準エラー出力 (stderr) に対して、invalid memory allocation request size s1 + s2 という形式のエラーメッセージを出力します。 2. exit(EXIT_FAILURE) を実行し、プロセスを終了させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/fe_memutils.c 内のサイズ計算ロジックから呼び出されます。
  • src/backend/utils/mmgr/mcxt.c からの参照が確認されています。
  • **呼び出し先**:
  • fprintf (標準ライブラリ)
  • exit (標準ライブラリ)

関連データ構造

  • Size: メモリサイズを表す型（通常 size_t）。

失敗モードと境界条件

  • 本関数自体が失敗することはありませんが、呼び出し元でサイズ計算のオーバーフローを正しく検知できていない場合、本関数は呼び出されません。

保守担当者向けメモ

  • 本関数はフロントエンドおよびバックエンドの両方から参照される可能性があるため、変更時は両方のコンテキストへの影響を考慮してください。
  • エラーメッセージは国際化（_() マクロ）に対応しています。
