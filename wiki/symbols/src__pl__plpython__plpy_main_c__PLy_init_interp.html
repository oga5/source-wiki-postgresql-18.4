PLy_init_interp

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_main.c:140

目的

Python インタプリタのグローバル環境（__main__ モジュール）を初期化し、PL/Python 用の共有データ辞書（GD）を設定します。

シグネチャまたは形状

static void PLy_init_interp(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 PLy_interp_globals および GD 辞書を更新）

前提条件と不変条件

  • Python インタプリタが既に起動している必要があります。
  • PLy_interp_globals が外部から参照可能であること。

アルゴリズムまたは振る舞い

1. PyImport_AddModule("__main__") を呼び出し、Python の __main__ モジュールを取得します。 2. 取得したモジュールの辞書を PLy_interp_globals に格納します。 3. 新しい Python 辞書オブジェクトを生成し、PLy_interp_safe_globals（静的変数）に保持します。 4. PLy_interp_globals 内に "GD" というキーで、作成した辞書オブジェクトを登録します。 5. 処理中にエラーが発生した場合は PLy_elog(ERROR, ...) を呼び出し、PostgreSQL のエラーメカニズムへ制御を移します。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpython/plpy_main.c 内の初期化プロセス（PLy_initialize からの呼び出しが示唆されています）。
  • 呼び出し先: Python C API (PyImport_AddModule, Py_INCREF, PyModule_GetDict, PyDict_New, PyDict_SetItemString, Py_DECREF, PyErr_Occurred) および PLy_elog。

関連データ構造

  • PLy_interp_globals: Python の __main__ モジュールの辞書を保持するグローバル変数。
  • PLy_interp_safe_globals: GD 辞書として使用される静的変数。

失敗モードと境界条件

  • PyImport_AddModule が NULL を返す、または Python エラーが発生した場合、PLy_elog によりエラー終了します。
  • PyDict_New がメモリ不足等で NULL を返した場合、PLy_elog によりエラー終了します。

保守担当者向けメモ

  • 本関数は static であり、plpy_main.c 内でのみ使用されます。
  • GD 辞書は Python コードからグローバルデータとしてアクセス可能な領域となるため、この初期化は PL/Python のセッション間データ共有の基盤となります。
