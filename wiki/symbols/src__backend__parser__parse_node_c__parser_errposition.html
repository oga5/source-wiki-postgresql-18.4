parser_errposition

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_node.c:105

目的

SQL解析中に発生したエラーに対し、ソーステキスト内のバイトオフセットを文字インデックスへ変換し、ereport 機構へエラー位置を報告する。

シグネチャまたは形状

int parser_errposition(ParseState *pstate, int location);

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する ParseState 構造体へのポインタ。
  • location: エラーが発生したソーステキスト内のバイトオフセット。
  • **出力**:
  • errposition() の戻り値（通常はエラー報告の一部として使用される整数）。

前提条件と不変条件

  • location が負の値である場合、本関数は何も行わず 0 を返す。
  • pstate が NULL であるか、pstate->p_sourcetext が NULL である場合、エラー位置の特定は行われず 0 を返す。

アルゴリズムまたは振る舞い

1. location が負でないことを確認する。 2. pstate および pstate->p_sourcetext が有効であることを確認する。 3. pg_mbstrlen_with_len を使用し、指定されたバイトオフセットをマルチバイト文字を考慮した文字インデックスに変換する。 4. 変換後の値に 1 を加算し、errposition() を呼び出してエラー位置を登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_subs.c
  • src/backend/catalog/heap.c
  • src/backend/commands/cluster.c
  • src/backend/commands/collationcmds.c
  • src/backend/commands/copy.c
  • **呼び出し先**:
  • pg_mbstrlen_with_len (文字数計算)
  • errposition (エラー報告)

関連データ構造

  • ParseState: 解析中のクエリ状態を保持する構造体。p_sourcetext を介して元のSQL文字列を参照する。

失敗モードと境界条件

  • location が 0 未満の場合、エラー位置の報告はスキップされる。
  • pstate が初期化されていない、またはソーステキストが欠落している場合、エラー位置の報告はスキップされる。

保守担当者向けメモ

  • 本関数はバイトオフセットを文字インデックスへ変換するため、マルチバイト文字を含むSQL文でも正確な位置を報告可能である。
  • errposition への引数は1から始まる文字インデックスである必要があるため、計算結果に 1 を加算している。
