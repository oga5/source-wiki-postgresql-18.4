ExecAsyncRequest

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execAsync.c:25

目的

非同期実行に対応したノードに対し、タプルの取得を要求するエントリポイントとして機能します。

シグネチャまたは形状

void ExecAsyncRequest(AsyncRequest *areq)

入力と出力

  • **入力**: AsyncRequest *areq （非同期リクエストの状態を保持する構造体）
  • **出力**: なし（副作用として areq->result にタプルが格納されるか、保留状態が更新されます）

前提条件と不変条件

  • areq->requestee が非同期実行をサポートしている必要があります。
  • areq->requestee に instrument が設定されている場合、実行統計の計測が自動的に行われます。

アルゴリズムまたは振る舞い

1. **再スキャン判定**: areq->requestee->chgParam が存在する場合、ExecReScan を呼び出してノードの状態をリセットします。 2. **計測開始**: instrument が有効な場合、InstrStartNode を呼び出します。 3. **ディスパッチ**: nodeTag(areq->requestee) に基づき、適切なノード固有の非同期処理関数を呼び出します（現在は T_ForeignScanState のみ対応）。 4. **応答処理**: ExecAsyncResponse(areq) を呼び出し、結果を親ノードへ伝播させます。 5. **計測終了**: instrument が有効な場合、InstrStopNode を呼び出し、タプルの有無に応じて計測値を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeAppend.c （ExecAsyncRequest(areq) を呼び出す箇所が確認されています）
  • **呼び出し先**:
  • ExecReScan
  • InstrStartNode / InstrStopNode
  • ExecAsyncForeignScanRequest （T_ForeignScanState の場合）
  • ExecAsyncResponse
  • elog （未対応のノード型が渡された場合）

関連データ構造

  • AsyncRequest: 非同期リクエストを管理する構造体。
  • ForeignScanState: 非同期実行をサポートする具体的なノード型。

失敗モードと境界条件

  • 非同期実行をサポートしていないノード型が渡された場合、elog(ERROR, ...) によりエラーが発生します。

保守担当者向けメモ

  • 新しい非同期対応ノードを追加する場合は、本関数の switch 文に新しい nodeTag を追加し、対応するノード側で専用の ExecAsync... 関数を実装してください。
  • 実行統計（Instrumentation）の開始・終了処理は、非同期実行のコストを正確に計測するために必須です。
