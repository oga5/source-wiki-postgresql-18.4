intorel_shutdown

定義

  • 種別: function
  • 定義箇所: src/backend/commands/createas.c:613

目的

CREATE TABLE AS (CTAS) 実行における DestReceiver の終了処理を行い、リソースの解放とテーブルのクローズを行う。

シグネチャまたは形状

static void intorel_shutdown(DestReceiver *self)

入力と出力

  • **入力**: DestReceiver *self（DR_intorel 構造体としてキャストして使用）
  • **出力**: なし（副作用としてリソース解放とリレーションのクローズを実行）

前提条件と不変条件

  • self は DR_intorel 型の構造体である必要がある。
  • myState->rel が有効なリレーションであることを前提とする。

アルゴリズムまたは振る舞い

1. self を DR_intorel 型にキャストし、IntoClause 情報を取得する。 2. into->skipData が偽（データ挿入が行われた場合）であれば、以下の処理を実行する:

  • FreeBulkInsertState を呼び出し、バルク挿入状態を解放する。
  • table_finish_bulk_insert を呼び出し、テーブルへのバルク挿入を完了させる。

    • 3. table_close を呼び出し、対象リレーションをクローズする（ロックはコミットまで保持するため NoLock を指定）。 4. myState->rel を NULL に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: DestReceiver インターフェースの rShutdown フィールドを通じて呼び出される（実行終了時）。
  • **呼び出し先**: FreeBulkInsertState, table_finish_bulk_insert, table_close。

関連データ構造

  • DR_intorel: 本関数が操作する DestReceiver の拡張構造体。
  • IntoClause: skipData フラグを保持する構造体。

失敗モードと境界条件

  • into->skipData が真の場合、バルク挿入関連のクリーンアップはスキップされる。
  • table_close に NoLock を渡すことで、トランザクション終了までロックを維持する設計となっている。

保守担当者向けメモ

  • 本関数は DR_intorel のライフサイクルにおける最終段階を担当する。
  • ロックの保持期間については、table_close の引数 NoLock が意図的な設計であることを留意すること。
