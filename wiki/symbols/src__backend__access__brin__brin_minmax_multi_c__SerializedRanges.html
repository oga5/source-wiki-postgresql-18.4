SerializedRanges

定義

  • 種別: struct
  • 定義箇所: src/backend/access/brin/brin_minmax_multi.c:205

目的

BRINインデックスの「Multi Min/Max」演算クラスにおいて、メモリ上の統計情報（Ranges）をディスク格納に適したコンパクトな bytea 形式へシリアライズするための物理格納構造体です。

シグネチャまたは形状

typedef struct SerializedRanges
{
    int32       vl_len_;        /* varlenaヘッダー */
    Oid         typid;          /* 格納データの型 */
    int         nranges;        /* 格納されている範囲数 */
    int         nvalues;        /* データ配列内の全値の数 */
    int         maxvalues;      /* 最大許容値数（reloption） */
    char        data[FLEXIBLE_ARRAY_MEMBER]; /* 実際のデータ */
} SerializedRanges;

入力と出力

  • **入力**: brin_range_serialize 関数により Ranges 構造体から変換されます。
  • **出力**: brin_range_deserialize 関数により、インデックスページから読み出され、メモリ上の Ranges 構造体へ復元されます。

前提条件と不変条件

  • (2 * nranges + nvalues) <= maxvalues という制約を満たす必要があります。
  • data メンバは可変長配列として扱われ、vl_len_ は PostgreSQL の varlena 形式に従う必要があります。

アルゴリズムまたは振る舞い

  • インデックスページへの書き込み時に、メモリ上の Ranges をこの構造体に詰め込み、bytea として永続化します。
  • 読み出し時には、PG_DETOAST_DATUM 等を用いてデトーストされたデータから本構造体へキャストし、各フィールドを解析します。

呼び出し元と呼び出し先

  • **呼び出し元**: brin_minmax_multi_serialize（シリアライズ時）、brin_minmax_multi_consistent（スキャン時）、および PG_DETOAST_DATUM を介したデータ取得処理。
  • **呼び出し先**: brin_range_serialize および brin_range_deserialize がこの構造体の構築と分解を直接担当します。

関連データ構造

  • Ranges: メモリ上での管理用構造体。
  • ExpandedRange: マージ処理のための中間表現。

失敗モードと境界条件

  • maxvalues を超えるデータが格納されようとした場合、インデックスの整合性が損なわれる可能性があります（AssertCheckRanges 等で検証されます）。
  • vl_len_ が正しく設定されていない場合、メモリ破壊や不正なデータ読み込みを引き起こす可能性があります。

保守担当者向けメモ

  • data フィールドは FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体自体のサイズ計算には offsetof(SerializedRanges, data) を使用してください。
  • 本構造体は bytea として格納されるため、直接的なポインタ操作を行う際は、PostgreSQL のトースト処理（PG_DETOAST_DATUM）を必ず経由してください。
