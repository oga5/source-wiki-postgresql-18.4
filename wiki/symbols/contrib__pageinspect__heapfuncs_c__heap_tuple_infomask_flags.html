heap_tuple_infomask_flags

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/heapfuncs.c:519

目的

PostgreSQL のヒープタプルが持つ t_infomask および t_infomask2 ビットフィールドをデコードし、人間が判読可能なフラグ文字列の配列として SQL レベルで提供する。

シグネチャまたは形状

Datum heap_tuple_infomask_flags(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_INT16(0): t_infomask (uint16)
  • PG_GETARG_INT16(1): t_infomask2 (uint16)
  • **出力**:
  • 2つの text[] 型配列を含む複合型（レコード）を Datum として返す。
  • 第1要素: 個別のフラグ（例: HEAP_HASNULL, HEAP_HOT_UPDATED 等）
  • 第2要素: 複合フラグ（例: HEAP_XMAX_SHR_LOCK 等）

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要。
  • 戻り値の型は複合型（row type）である必要がある。
  • 破損したページデータに対しても動作するよう設計されている。

アルゴリズムまたは振る舞い

1. **権限チェック**: superuser() を確認し、権限不足時はエラーを投げる。 2. **型検証**: get_call_result_type で戻り値の型を検証する。 3. **ビットカウント**: pg_popcount を使用して、設定されているフラグの総数を計算し、メモリを確保する。 4. **デコード処理**:

  • t_infomask および t_infomask2 の各ビットを順次チェックし、該当するフラグ文字列を flags 配列に格納する。
  • 複合フラグ（HEAP_XMAX_SHR_LOCK 等）は、ビットマスクの組み合わせとして別途判定する。

    • 5. **構築と返却**: construct_array_builtin で text[] を作成し、heap_form_tuple でレコードを構築して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出される（直接的な C 言語レベルの呼び出し元はソース内に明示されていない）。
  • **呼び出し先**:
  • superuser(), get_call_result_type(), pg_popcount(), palloc0(), CStringGetTextDatum(), construct_array_builtin(), construct_empty_array(), heap_form_tuple(), PG_RETURN_DATUM() 等の PostgreSQL 内部 API。

関連データ構造

  • TupleDesc: 結果のレコード構造を定義。
  • HeapTuple: 構築された結果タプル。
  • ArrayType: フラグ文字列を格納する配列。

失敗モードと境界条件

  • **権限エラー**: スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE で終了する。
  • **型エラー**: 戻り値の型が複合型でない場合、elog(ERROR) で終了する。
  • **フラグなし**: bitcnt が 0 の場合、空の配列を格納したレコードを返す。

保守担当者向けメモ

  • 本関数は contrib/pageinspect の一部であり、デバッグ用途である。
  • t_infomask の定義は access/htup_details.h に依存しているため、将来的なフラグ追加時は同ヘッダと本関数の判定ロジックを同期させる必要がある。
