length_hist_bsearch

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_selfuncs.c:656

目的

範囲型の長さヒストグラム（length_hist_values）を二分探索し、指定された値（value）に対応するインデックスを特定します。

シグネチャまたは形状

static int length_hist_bsearch(Datum *length_hist_values, int length_hist_nvalues, double value, bool equal)

入力と出力

  • **入力**:
  • length_hist_values: 昇順にソートされた長さヒストグラムの Datum 配列。
  • length_hist_nvalues: 配列の要素数。
  • value: 検索対象の長さ（double）。
  • equal: 比較時に等価条件を含めるかどうかを示すフラグ。
  • **出力**:
  • 検索条件を満たす最大のインデックス（int）。見つからない場合は -1 を返します。

前提条件と不変条件

  • length_hist_values は昇順にソートされている必要があります。
  • Datum は float8 として解釈可能である必要があります。

アルゴリズムまたは振る舞い

1. 探索範囲を lower = -1 から upper = length_hist_nvalues - 1 に初期化します。 2. while (lower < upper) ループ内で、中央値 middle を計算します。 3. middle 位置の値を DatumGetFloat8 で取得し、value と比較します。 4. equal が true の場合、middleval <= value を条件として含め、条件を満たす限り lower を middle に更新します。 5. equal が false の場合、middleval < value のみを条件として lower を更新します。 6. 条件を満たさない場合は upper を middle - 1 に更新します。 7. 最終的な lower の値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/rangetypes_selfuncs.c 内の複数箇所（統計計算ロジック）。
  • src/backend/utils/adt/multirangetypes_selfuncs.c 内の複数箇所。
  • **呼び出し先**:
  • DatumGetFloat8 マクロ（Datum から double への変換）。

関連データ構造

  • Datum: PostgreSQL の汎用データ表現。
  • 範囲型の統計情報（pg_statistic に格納されるヒストグラムデータ）。

失敗モードと境界条件

  • length_hist_nvalues が 0 の場合、lower は -1 のまま返されます。
  • 検索値がヒストグラムの全要素より小さい場合、戻り値は -1 となります。

保守担当者向けメモ

  • 本関数は rangetypes および multirangetypes の両方から利用される共通ユーティリティです。
  • 比較ロジックは equal フラグによって厳密な不等号と等号付き不等号を切り替える設計となっており、ヒストグラムのビン境界を扱う際のオフセット計算に直結しています。
