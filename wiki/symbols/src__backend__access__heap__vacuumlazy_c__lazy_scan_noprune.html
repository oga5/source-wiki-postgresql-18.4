lazy_scan_noprune

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/vacuumlazy.c:2238

目的

クリーンアップロックが取得できない場合に、ヒープページを物理的に変更（Pruning/Freezing）することなく、統計情報の収集とデッドタプルの追跡のみを行う。

シグネチャまたは形状

static bool lazy_scan_noprune(LVRelState *vacrel, Buffer buf, BlockNumber blkno, Page page, bool *has_lpdead_items);

入力と出力

  • **入力**:
  • vacrel: VACUUM実行状態を保持する LVRelState 構造体。
  • buf: 対象ページのバッファ。
  • blkno: 対象ページのブロック番号。
  • page: 対象ページのポインタ。
  • has_lpdead_items: LP_DEADアイテムの有無を返す出力用ポインタ。
  • **出力**:
  • bool: 処理が成功した場合は true。Aggressive VACUUMにおいて凍結が必要なタプルが存在し、かつロック取得が必要な場合は false を返す。

前提条件と不変条件

  • BufferGetBlockNumber(buf) == blkno であること。
  • 呼び出し側は対象ページに対する共有ロックを保持している必要がある。

アルゴリズムまたは振る舞い

1. ページ内の全オフセットを走査し、各タプルの状態を確認する。 2. ItemIdIsDead なアイテムを deadoffsets に記録する。 3. heap_tuple_should_freeze を用いて凍結が必要か判定する。

  • vacrel->aggressive が有効で凍結が必要な場合、即座に false を返し、呼び出し側に lazy_scan_prune への切り替えを促す。

    • 4. HeapTupleSatisfiesVacuum を用いてタプルの可視性を判定し、ライブタプル数やデッドタプル数をカウントする。 5. 処理終了後、vacrel->NewRelfrozenXid および vacrel->NewRelminMxid を更新する。 6. インデックスが存在する場合、見つかった LP_DEAD アイテムを dead_items_add で記録する。

呼び出し元と呼び出し先

  • **呼び出し元**: lazy_scan_heap（src/backend/access/heap/vacuumlazy.c 内のヒープスキャンループ）。
  • **呼び出し先**: PageGetMaxOffsetNumber, PageGetItemId, ItemIdIsUsed, ItemIdIsRedirected, ItemIdIsDead, heap_tuple_should_freeze, ItemPointerSet, PageGetItem, RelationGetRelid, HeapTupleSatisfiesVacuum, dead_items_add。

関連データ構造

  • LVRelState: VACUUM全体の制御状態。
  • HeapTupleData: ヒープタプルのメタデータ。
  • ItemIdData: ページ内のアイテム識別子。

失敗モードと境界条件

  • **Aggressive VACUUM時の制約**: 凍結が必要なタプルが存在し、かつ vacrel->aggressive が真の場合、本関数は処理を完遂できず false を返す。
  • **インデックスなしのケース**: インデックスが存在しない場合、LP_DEAD アイテムは missed_dead_tuples として計上される。

保守担当者向けメモ

  • 本関数は「Pruningを行わない」という性質上、ページ内の LP_DEAD アイテムを即座に LP_UNUSED にはしない。これらは後続のインデックス掃除フェーズで処理されるか、次回のVACUUMまで持ち越される。
  • lazy_scan_prune とのロジックの乖離を避けるため、タプルのカウント処理は両者で一貫させる必要がある。
