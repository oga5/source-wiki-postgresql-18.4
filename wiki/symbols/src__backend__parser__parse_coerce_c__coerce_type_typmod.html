coerce_type_typmod

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_coerce.c:750

目的

式（Node）に対して、ターゲットとなる型修飾子（typmod）を適用または強制します。必要に応じて型変換関数を呼び出し、あるいは RelabelType ノードを挿入することで、式が指定された型修飾子を持つことを保証します。

シグネチャまたは形状

Node *coerce_type_typmod(Node *node, Oid targetTypeId, int32 targetTypMod,
                         CoercionContext ccontext, CoercionForm cformat,
                         int location, bool hideInputCoercion)

入力と出力

  • **入力**:
  • node: 変換対象の式ノード。
  • targetTypeId: ターゲットとなる型のOID。
  • targetTypMod: 適用する型修飾子。
  • ccontext, cformat, location: 変換の文脈、形式、および構文上の位置情報。
  • hideInputCoercion: 真の場合、内部的な変換ステップの表示を抑制します。
  • **出力**:
  • 変換済み、または型修飾子を付与された Node。

前提条件と不変条件

  • targetTypMod が現在の式の typmod と一致する場合、変換は行われず入力ノードがそのまま返されます。
  • targetTypMod が負の値である場合、実際の型変換は行われず、RelabelType を用いて型修飾子のみが更新されます。

アルゴリズムまたは振る舞い

1. **早期リターン**: targetTypMod が現在の式の typmod と等しければ、何もせず node を返す。 2. **表示抑制**: hideInputCoercion が真であれば、hide_coercion_node を呼び出し、ネストされた変換ステップの表示を抑制する。 3. **変換パスの特定**: targetTypMod が負でなければ、find_typmod_coercion_function を使用して、型修飾子を適用するための変換関数（funcId）を検索する。 4. **ノード生成**:

  • 変換関数が見つかった場合（pathtype != COERCION_PATH_NONE）、build_coercion_expression を呼び出し、変換ノードを構築する。
  • 変換関数が見つからない場合、applyRelabelType を呼び出し、型修飾子を反映させるための RelabelType ノードを適用する。

    • 5. **結果返却**: 最終的なノードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_coerce.c 内の各所（coerce_type 等の変換ロジックから呼び出される）。
  • **呼び出し先**:
  • exprTypmod, exprCollation
  • hide_coercion_node
  • find_typmod_coercion_function
  • build_coercion_expression
  • applyRelabelType

関連データ構造

  • Node: PostgreSQLの構文解析ツリーを構成する基本構造。
  • Oid: 型識別子。
  • CoercionContext, CoercionForm: 変換の性質を定義する列挙型。

失敗モードと境界条件

  • targetTypMod が負の場合、変換関数は検索されず、常に RelabelType による処理が行われます。
  • 変換関数が見つからない場合でも、エラーにはならず RelabelType による型修飾子の強制が行われます。

保守担当者向けメモ

  • 本関数は、型変換の最終段階で型修飾子（長さ制限など）を適用するために使用されます。
  • RelabelType は物理的なデータ変換を行わず、パーサーに対して型の解釈を指示するものである点に注意してください。
