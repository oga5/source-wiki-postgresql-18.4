downcase_truncate_identifier

定義

  • 種別: function
  • 定義箇所: src/backend/parser/scansup.c:36

目的

SQL 識別子を小文字に変換し、データベースの制限（NAMEDATALEN）に合わせて切り詰めるための主要なエントリーポイントです。

シグネチャまたは形状

char *downcase_truncate_identifier(const char *ident, int len, bool warn)

入力と出力

  • **入力**:
  • ident: 処理対象の識別子文字列（const char *）。
  • len: 入力文字列の長さ。
  • warn: 切り詰めが発生した場合に NOTICE を発行するかどうかを指定するフラグ。
  • **出力**:
  • 処理済み文字列を格納したメモリ領域へのポインタ（palloc により割り当て済み）。

前提条件と不変条件

  • 返却される文字列は palloc されているため、呼び出し側でメモリ管理（解放）を行う必要があります。
  • マルチバイト文字を含む場合、pg_mbcliplen を介して文字境界を維持した切り詰めが行われます。

アルゴリズムまたは振る舞い

内部的に downcase_identifier を呼び出し、切り詰めフラグを true に設定して処理を委譲します。これにより、小文字化処理と NAMEDATALEN に基づく切り詰め処理が実行されます。

呼び出し元と呼び出し先

  • **呼び出し先**: downcase_identifier
  • **呼び出し元（直接的な証拠）**:
  • contrib/pgcrypto/pgcrypto.c
  • src/backend/utils/adt/date.c
  • src/backend/utils/adt/datetime.c
  • src/backend/utils/adt/timestamp.c

関連データ構造

  • NAMEDATALEN: 識別子の最大長を規定する定数（src/include/pg_config_manual.h で定義）。

失敗モードと境界条件

  • 入力文字列が NAMEDATALEN を超える場合、warn が true であればユーザーに通知が行われます。

保守担当者向けメモ

  • 本関数は downcase_identifier のラッパーです。ロケール依存の tolower() を使用しているため、Unicode 準拠の正規化が必要な場合には注意が必要です。
