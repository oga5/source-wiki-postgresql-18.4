test_slru_page_write

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_slru/test_slru.c:56

目的

SQLインターフェースを通じて、指定されたページ番号のSLRUバッファを初期化し、任意のデータを書き込んでディスクへフラッシュするテスト用関数です。

シグネチャまたは形状

Datum test_slru_page_write(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • pageno (int64): 書き込み対象のページ番号（第1引数）。
  • data (text): ページに書き込む文字列データ（第2引数）。
  • **出力**: void (PG_RETURN_VOID)

前提条件と不変条件

  • TestSlruCtl が初期化されていること。
  • SimpleLruZeroPage の実行後、TestSlruCtl->shared->page_number[slotno] が指定した pageno と一致すること。

アルゴリズムまたは振る舞い

1. SimpleLruGetBankLock を使用して、対象ページに対応する排他ロックを取得する。 2. SimpleLruZeroPage を呼び出し、対象ページをバッファに割り当てて初期化する。 3. 該当スロットのステータスを SLRU_PAGE_VALID に設定し、ダーティフラグを立てる。 4. strncpy を使用して、バッファサイズ（BLCKSZ - 1）の制限内でデータをコピーする。 5. SimpleLruWritePage を呼び出し、バッファの内容をディスクへ書き出す。 6. 取得したロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（PG_FUNCTION_INFO_V1 マクロによる登録あり）。
  • **呼び出し先**:
  • SimpleLruGetBankLock
  • LWLockAcquire / LWLockRelease
  • SimpleLruZeroPage
  • SimpleLruWritePage
  • text_to_cstring

関連データ構造

  • TestSlruCtl: テスト用SLRUを制御する SlruCtlData 構造体。
  • SlruSharedData: TestSlruCtl->shared を介してアクセスされる共有メモリ上の状態管理構造。

失敗モードと境界条件

  • data の長さが BLCKSZ - 1 を超える場合、切り詰められる。
  • ロック取得時に他のプロセスと競合する場合、待機が発生する。

保守担当者向けメモ

  • 本関数は SimpleLruZeroPage を使用しているため、既存のページ内容を上書きする点に注意してください。
  • long_segment_names = true 設定下での動作を前提としています。
