pgaio_worker_queue_shmem_size

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_worker.c:103

目的

非同期I/Oワーカープロセス方式で使用される共有メモリ上のサブミッションキュー（PgAioWorkerSubmissionQueue）に必要なメモリサイズを算出します。

シグネチャまたは形状

size_t pgaio_worker_queue_shmem_size(int *queue_size)

入力と出力

  • **入力**: queue_size（ポインタ）: 算出の基準となるキューサイズ。
  • **出力**: 戻り値として必要なバイト数を size_t で返します。また、引数 queue_size には、マスク計算のために2のべき乗に切り上げられた値が格納されます。

前提条件と不変条件

  • io_worker_queue_size がグローバル変数として定義されている必要があります。
  • キューサイズは、ビットマスク操作を可能にするため、必ず2のべき乗に切り上げられます。

アルゴリズムまたは振る舞い

1. グローバル変数 io_worker_queue_size を pg_nextpower2_32 関数で2のべき乗に切り上げ、引数 queue_size に書き戻します。 2. PgAioWorkerSubmissionQueue 構造体の固定部分（sqes メンバのオフセット）と、切り上げたキューサイズ分（int 型の配列）の合計サイズを計算して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/method_worker.c:126
  • src/backend/storage/aio/method_worker.c:140
  • **呼び出し先**:
  • pg_nextpower2_32（port/pg_bitutils.h で定義）

関連データ構造

  • PgAioWorkerSubmissionQueue: この関数がサイズを算出する対象の共有メモリ構造体。

失敗モードと境界条件

  • io_worker_queue_size が極端に大きい場合、pg_nextpower2_32 の挙動および size_t の範囲内であることに依存します。

保守担当者向けメモ

  • 本関数は、共有メモリの割り当て時にキューのサイズを決定する重要な役割を担います。キューの構造を変更（sqes メンバの型変更など）した場合は、必ずこの計算式を更新してください。
