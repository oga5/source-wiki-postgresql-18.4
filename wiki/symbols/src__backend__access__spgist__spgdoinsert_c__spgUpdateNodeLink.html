spgUpdateNodeLink

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgdoinsert.c:51

目的

SP-GiSTインナータプル内の指定されたノードが指し示すリンク先（ブロック番号およびオフセット番号）を更新します。

シグネチャまたは形状

void spgUpdateNodeLink(SpGistInnerTuple tup, int nodeN, BlockNumber blkno, OffsetNumber offset)

入力と出力

  • **入力**:
  • tup: 更新対象の SpGistInnerTuple。
  • nodeN: 更新対象となるノードのインデックス番号。
  • blkno: 新しく設定するブロック番号。
  • offset: 新しく設定するオフセット番号。
  • **出力**: なし（副作用として tup 内のノード情報が更新されます）。

前提条件と不変条件

  • tup は有効な SpGistInnerTuple である必要があります。
  • 指定された nodeN が tup 内に存在しなければなりません。

アルゴリズムまたは振る舞い

1. SGITITERATE マクロを使用して、インナータプル内のノードを順次走査します。 2. 現在のノードインデックスが nodeN と一致した場合、そのノードの t_tid（ItemPointer）を blkno と offset で更新します。 3. 更新完了後、直ちにリターンします。 4. 走査終了までに nodeN が見つからなかった場合、elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgdoinsert.c 内の挿入ロジック。
  • src/backend/access/spgist/spgxlog.c 内のWALリプレイ処理。
  • **呼び出し先**:
  • ItemPointerSet（マクロ）
  • elog（エラー報告用）

関連データ構造

  • SpGistInnerTuple: SP-GiSTの内部タプル構造。
  • SpGistNodeTuple: インナータプル内に含まれる個々のノード構造。

失敗モードと境界条件

  • 指定された nodeN がタプル内に存在しない場合、ERROR レベルで処理が中断されます。

保守担当者向けメモ

  • 本関数はインナータプルの物理的な構造を走査するため、ノード数が多い場合には計算コストに注意してください。
  • spgUpdateNodeLink は src/include/access/spgist_private.h で外部公開されています。
