pgstat_index_page

定義

  • 種別: function
  • 定義箇所: contrib/pgstattuple/pgstattuple.c:580

目的

インデックスページを走査し、ページ内の空き容量およびライブ/デッドタプルの統計情報を集計して pgstattuple_type 構造体に反映させる。

シグネチャまたは形状

static void pgstat_index_page(pgstattuple_type *stat, Page page, OffsetNumber minoff, OffsetNumber maxoff)

入力と出力

  • **入力**:
  • stat: 集計結果を格納する pgstattuple_type 構造体へのポインタ。
  • page: 走査対象のインデックスページ。
  • minoff, maxoff: 走査を開始および終了するオフセット番号。
  • **出力**: なし（stat が指す構造体の各フィールドが更新される）。

前提条件と不変条件

  • page は有効なインデックスページであること。
  • minoff から maxoff までの範囲が、対象ページ内の有効なオフセット範囲内であること。

アルゴリズムまたは振る舞い

1. PageGetExactFreeSpace(page) を呼び出し、ページ内の空き容量を stat->free_space に加算する。 2. minoff から maxoff まで OffsetNumberNext を用いて順次走査する。 3. 各オフセットの ItemId を取得し、ItemIdIsDead で生存状態を判定する。

  • **デッドタプルの場合**: stat->dead_tuple_count をインクリメントし、ItemIdGetLength で取得した長さを stat->dead_tuple_len に加算する。
  • **ライブタプルの場合**: stat->tuple_count をインクリメントし、長さを stat->tuple_len に加算する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgstattuple/pgstattuple.c 内のインデックス走査関数（pgstat_btree_page 等の各アクセスメソッド用関数から呼び出される）。
  • **呼び出し先**:
  • PageGetExactFreeSpace
  • PageGetItemId
  • ItemIdIsDead
  • ItemIdGetLength
  • OffsetNumberNext

関連データ構造

  • pgstattuple_type: 統計情報を保持する構造体。
  • Page: PostgreSQL のページデータ構造。
  • ItemId: ページ内のアイテム識別子。

失敗モードと境界条件

  • ページが破損している場合、PageGetItemId 等のアクセスマクロが予期しない動作をする可能性がある。
  • minoff が maxoff より大きい場合、ループは実行されない。

保守担当者向けメモ

  • 本関数は汎用的なインデックスページ走査ロジックであるため、特定のインデックス型（B-tree, Hash, GiST 等）に依存しない。
  • 走査対象のページは呼び出し元で適切にロックされている必要がある。
