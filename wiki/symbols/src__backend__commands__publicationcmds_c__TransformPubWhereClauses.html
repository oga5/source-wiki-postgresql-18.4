TransformPubWhereClauses

定義

  • 種別: function
  • 定義箇所: src/backend/commands/publicationcmds.c:706

目的

パブリケーション定義に含まれる行フィルタ（WHERE句）を解析し、構文チェック、型変換、および論理レプリケーションの制約に適合しているかの検証を行う。

シグネチャまたは形状

static void TransformPubWhereClauses(List *tables, const char *queryString, bool pubviaroot)

入力と出力

  • **入力**:
  • tables: PublicationRelInfo 構造体のリスト。各要素は解析対象のリレーションとWHERE句を含む。
  • queryString: 解析に使用する元のSQLクエリ文字列。
  • pubviaroot: publish_via_partition_root オプションの有効状態。
  • **出力**: なし（副作用として tables 内の PublicationRelInfo 構造体の whereClause フィールドが解析済みの式に更新される）。

前提条件と不変条件

  • tables リスト内の各 PublicationRelInfo は初期化済みであること。
  • 行フィルタ式は、論理レプリケーションの安全性を確保するため、check_simple_rowfilter_expr によって制限された「単純な式」である必要がある。

アルゴリズムまたは振る舞い

1. tables リストを走査し、WHERE句が設定されているリレーションのみを処理する。 2. pubviaroot が false で、かつ対象がパーティションテーブルである場合、エラーを発生させる（パーティションテーブルへの直接的なWHERE句適用は許可されない）。 3. make_parsestate で一時的な ParseState を作成し、対象リレーションを範囲テーブル（RangeTable）に追加する。 4. transformWhereClause を呼び出し、WHERE句の構文解析と型変換を行う。 5. assign_expr_collations で照合順序を確定させ、expand_generated_columns_in_expr で生成列を展開する。 6. check_simple_rowfilter_expr を実行し、式がレプリケーションに適した単純な形式であることを検証する。 7. 解析済みの式を pri->whereClause に格納し、ParseState を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/publicationcmds.c 内のパブリケーション作成・変更処理（CreatePublication や AlterPublication 関連のフローから呼び出される）。
  • **呼び出し先**:
  • make_parsestate, addRangeTableEntryForRelation, addNSItemToQuery, transformWhereClause, assign_expr_collations, expand_generated_columns_in_expr, check_simple_rowfilter_expr, free_parsestate 等のパーサー関連関数。

関連データ構造

  • PublicationRelInfo: パブリケーションに関連付けられたリレーション情報とWHERE句を保持する構造体。
  • ParseState: SQL解析中の状態を保持する構造体。

失敗モードと境界条件

  • pubviaroot が false の状態でパーティションテーブルにWHERE句を指定した場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーとなる。
  • check_simple_rowfilter_expr の制約に違反する複雑な式（サブクエリや揮発性関数など）が含まれる場合、エラーとなる。

保守担当者向けメモ

  • 本関数は、パブリケーションの定義変更時にキャッシュ無効化とセットで実行される必要がある。
  • 行フィルタの制約は、過去のデータスナップショットに対する安全な評価を保証するためのものであるため、変更時は check_simple_rowfilter_expr_walker のロジックと整合性を保つこと。
