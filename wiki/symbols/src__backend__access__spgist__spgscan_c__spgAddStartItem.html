spgAddStartItem

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgscan.c:129

目的

SP-GiST インデックススキャンを開始するための初期探索項目（SpGistSearchItem）を生成し、スキャンキューに追加します。

シグネチャまたは形状

static void spgAddStartItem(SpGistScanOpaque so, bool isnull)

入力と出力

  • **入力**:
  • so: スキャン状態を保持する SpGistScanOpaque 構造体。
  • isnull: NULL 値の検索か否かを示すフラグ。
  • **出力**: なし（副作用として so 内のキューに項目が追加されます）。

前提条件と不変条件

  • so は有効な SpGistScanOpaque 構造体である必要があります。
  • so->zeroDistances が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. spgAllocSearchItem を呼び出し、so のコンテキスト内で新しい探索項目を割り当てます。 2. isnull フラグに基づき、探索開始位置を決定します。

  • isnull が真の場合: SPGIST_NULL_BLKNO を使用。
  • isnull が偽の場合: SPGIST_ROOT_BLKNO を使用。

    • 3. 探索項目の各フィールドを初期化します（isLeaf = false, level = 0 など）。 4. spgAddSearchItemToQueue を呼び出し、設定済みの項目をスキャンキューに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgscan.c 内の関数（spgrescan 等から呼び出されることが示唆されています）。
  • **呼び出し先**:
  • spgAllocSearchItem
  • ItemPointerSet
  • spgAddSearchItemToQueue

関連データ構造

  • SpGistScanOpaque: スキャン状態を管理する主要構造体。
  • SpGistSearchItem: 走査対象のノードやリーフを保持するキュー項目。

失敗モードと境界条件

  • メモリ不足時には spgAllocSearchItem 経由でエラーが発生する可能性があります。
  • isnull の値によって、インデックスのルートページまたは NULL 用の特殊ブロックのいずれかが適切に選択される必要があります。

保守担当者向けメモ

  • 本関数はスキャン開始時の初期状態を決定する重要な役割を担います。インデックスの木構造の走査開始点（ルートまたは NULL 領域）を制御しているため、SP-GiST のインデックス構造変更時には本関数のロジックを確認してください。
