astreamer_gzip_decompressor

定義

  • **種別**: struct
  • **定義箇所**: src/fe_utils/astreamer_gzip.c:46

目的

gzip形式の圧縮データを解凍し、後続の astreamer パイプラインへデータを転送するための状態管理構造体です。

シグネチャまたは形状

typedef struct astreamer_gzip_decompressor
{
    astreamer    base;           /* 基底ストリーマーインターフェース */
    z_stream     zstream;        /* zlibの解凍用ストリーム状態 */
    size_t       bytes_written;  /* 解凍済みデータの累積サイズ */
} astreamer_gzip_decompressor;

入力と出力

  • **入力**: astreamer_gzip_decompressor_content を通じて供給される圧縮データチャンク。
  • **出力**: 解凍されたデータは astreamer インターフェースの規約に従い、後続のストリーマーへ転送されます。

前提条件と不変条件

  • zlib ライブラリが利用可能であること。
  • astreamer_gzip_decompressor_new によって適切に初期化されていること。
  • zstream メンバは inflateInit2 等の zlib 関数によって管理される必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: astreamer_gzip_decompressor_new で z_stream を初期化。 2. **解凍処理**: content メソッド呼び出し時に inflate を実行し、入力データを解凍。 3. **転送**: 解凍後のデータはバッファ（bbs_buffer）に蓄積され、満杯時に後続のストリーマーへ渡される。 4. **終了処理**: finalize メソッドで残存バッファをフラッシュし、後続ストリーマーの終了処理を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer_gzip_decompressor_new（インスタンス生成）、astreamer_gzip_decompressor_content、astreamer_gzip_decompressor_finalize、astreamer_gzip_decompressor_free 等の各関数内でキャストして使用。
  • **呼び出し先**: zlib ライブラリ関数（inflate 等）、および astreamer インターフェースで定義された後続のストリーマー関数。

関連データ構造

  • astreamer: 本構造体が継承する基底インターフェース。
  • z_stream: zlib が提供する圧縮・解凍状態管理構造体。

失敗モードと境界条件

  • zlib の解凍処理中にエラーが発生した場合、get_gz_error 等を通じてエラーが報告される。
  • メモリ不足時には palloc0 がエラーを発生させる可能性がある。
  • 入力データが不正なgzip形式である場合、inflate がエラーを返す。

保守担当者向けメモ

  • 本構造体は astreamer インターフェースの一部として実装されています。
  • zlib のリソース（zstream）は astreamer_gzip_decompressor_free で適切に解放される必要があります。
  • 他の圧縮モジュール（LZ4, Zstd）と異なり、本モジュールはgzip特有の zlib 依存関係を直接保持しています。
