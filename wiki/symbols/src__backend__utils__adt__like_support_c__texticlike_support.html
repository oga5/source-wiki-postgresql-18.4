texticlike_support

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like_support.c:122

目的

大文字小文字を区別しない ILIKE 演算子に対するプランナのサポート関数として機能します。インデックススキャンが利用可能な条件式への変換や、選択率の推定をプランナに提供します。

シグネチャまたは形状

Datum texticlike_support(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して Node *rawreq を受け取ります。これはプランナからのサポート要求（インデックス条件生成または選択率推定）を表すノードです。
  • **出力**: Datum 型として、like_regex_support の実行結果を返します。

前提条件と不変条件

  • rawreq は有効な SupportRequest ノードである必要があります。
  • 内部的に Pattern_Type_Like_IC（大文字小文字を区別しない LIKE パターン）を固定引数として使用します。

アルゴリズムまたは振る舞い

1. 引数から rawreq を取得します。 2. like_regex_support 関数を呼び出し、その際に Pattern_Type_Like_IC を指定して処理を委譲します。 3. like_regex_support の戻り値をそのまま返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: プランナ（インデックスの利用可能性や選択率を評価する際に呼び出されます）。
  • **呼び出し先**: like_regex_support（実際の条件生成および選択率推定ロジックのメインエントリポイント）。

関連データ構造

  • Pattern_Type: パターンマッチングの種類を定義する列挙型。本関数は Pattern_Type_Like_IC を使用します。
  • Node: プランナからの要求を保持する汎用ノード構造体。

失敗モードと境界条件

  • like_regex_support がサポートしていないパターンや無効な要求を受け取った場合、その関数に準じたエラーまたはデフォルトの挙動が発生します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的なロジックは like_regex_support に集約されています。ロジックの変更が必要な場合は like_regex_support を参照してください。
