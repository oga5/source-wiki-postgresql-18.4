parse_fcall_arguments

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/fastpath.c:328

目的

クライアントから送信された Fastpath 関数呼び出しメッセージの引数データを解析し、FunctionCallInfo 構造体に格納して、戻り値のフォーマットコードを返します。

シグネチャまたは形状

int16 parse_fcall_arguments(StringInfo msgBuf, struct fp_info *fip, FunctionCallInfo fcinfo)

入力と出力

  • **入力**:
  • msgBuf: クライアントからのメッセージデータを含む StringInfo。
  • fip: 対象関数のメタデータ（fp_info）。
  • fcinfo: 引数値を格納する FunctionCallInfo。
  • **出力**:
  • 戻り値: クライアントが要求する結果のフォーマットコード（int16）。

前提条件と不変条件

  • msgBuf は PQfn プロトコルに従った形式であること。
  • fip には対象関数の正しい引数情報が設定されていること。
  • nargs が FUNC_MAX_ARGS を超えないこと。

アルゴリズムまたは振る舞い

1. メッセージから引数のフォーマットコード（aformats）を読み取る。 2. 引数の数（nargs）を読み取り、fip 内の定義と一致するか検証する。 3. 各引数について以下の処理を繰り返す:

  • argsize を読み取り、-1 の場合は isnull = true とする。
  • argsize が有効な場合、abuf にデータを読み込む。
  • aformat に応じて変換を行う:
  • 0 (text): OidInputFunctionCall を使用して変換。必要に応じてクライアントからサーバーへのエンコーディング変換を行う。
  • 1 (binary): OidReceiveFunctionCall を使用して変換。

    • 4. 最後にメッセージから結果のフォーマットコードを読み取り、呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/fastpath.c 内の関数（HandleFunctionRequest からの呼び出しが確認されている）。
  • **呼び出し先**: pq_getmsgint, pq_getmsgbytes, getTypeInputInfo, OidInputFunctionCall, getTypeBinaryInputInfo, OidReceiveFunctionCall 等。

関連データ構造

  • StringInfo: メッセージバッファ。
  • fp_info: 関数メタデータ。
  • FunctionCallInfo: 関数実行に必要な引数と状態を保持する構造体。

失敗モードと境界条件

  • ERRCODE_PROTOCOL_VIOLATION: 引数の数やフォーマットコードの不整合、無効な引数サイズが検出された場合に発生。
  • ERRCODE_INVALID_BINARY_REPRESENTATION: バイナリデータが期待通りに消費されなかった場合に発生。
  • ERRCODE_INVALID_PARAMETER_VALUE: サポートされていないフォーマットコードが指定された場合に発生。

保守担当者向けメモ

  • 本関数内での palloc は MessageContext で行われるため、明示的な解放は不要です。
  • aformat が 0 の場合、エンコーディング変換後の文字列メモリは pfree で解放されます。
  • キャッシュは存在せず、呼び出しごとにシステムカタログを検索する設計となっています。
