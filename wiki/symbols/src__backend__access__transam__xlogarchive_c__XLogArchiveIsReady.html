XLogArchiveIsReady

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogarchive.c:693

目的

指定されたWALセグメントファイルが、アーカイブプロセスによる処理待ち（.ready状態）であるかどうかを判定します。

シグネチャまたは形状

bool XLogArchiveIsReady(const char *xlog);

入力と出力

  • **入力**: xlog (WALセグメントファイル名)
  • **出力**: true（.readyファイルが存在する場合）、false（存在しない場合）

前提条件と不変条件

  • xlog 引数は有効なWALセグメントファイル名である必要があります。
  • アーカイブ状態ファイルは pg_wal/archive_status ディレクトリ内に配置されていることを前提としています。

アルゴリズムまたは振る舞い

1. StatusFilePath マクロを使用して、指定されたWALファイル名に対応する .ready ファイルのパスを構築します。 2. stat() を呼び出し、当該パスのファイルが存在するかを確認します。 3. ファイルが存在すれば true を返し、存在しなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c:3983
  • **呼び出し先**:
  • stat() (システムコール)
  • StatusFilePath (マクロ)

関連データ構造

  • struct stat: ファイルの存在確認に使用される標準構造体。

失敗モードと境界条件

  • stat() がファイル非存在以外のエラー（権限不足等）を返した場合、本関数は false を返します。
  • xlog 引数がNULLまたは不正なパスである場合の挙動は、StatusFilePath の実装に依存します。

保守担当者向けメモ

  • 本関数は単に .ready ファイルの存在を確認するのみであり、アーカイブが実際に進行中であるか（.doneの有無など）は考慮しません。
  • アーカイブ状態の完全な判定には、XLogArchiveIsReadyOrDone 等の関連関数との組み合わせが必要になる場合があります。
