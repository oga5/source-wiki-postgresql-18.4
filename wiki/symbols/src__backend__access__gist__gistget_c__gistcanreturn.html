gistcanreturn

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistget.c:796

目的

指定されたインデックス属性が、インデックスのみのスキャン（Index-Only Scan）において元の値を復元可能かどうかを判定する。

シグネチャまたは形状

bool gistcanreturn(Relation index, int attno)

入力と出力

  • **入力**:
  • index: 対象のGiSTインデックスリレーション。
  • attno: 判定対象の属性番号。
  • **出力**:
  • true: 属性値が復元可能（Index-Only Scanをサポート）。
  • false: 復元不可能。

前提条件と不変条件

  • attno がインデックスのキー属性数を超えている場合、または GIST_FETCH_PROC が定義されている場合、あるいは GIST_COMPRESS_PROC が定義されていない場合は true を返す。

アルゴリズムまたは振る舞い

1. attno がインデックスのキー属性数より大きい場合、true を返す。 2. index に対して GIST_FETCH_PROC（フェッチ関数）が定義されている場合、true を返す。 3. index に対して GIST_COMPRESS_PROC（圧縮関数）が定義されていない場合、true を返す。 4. 上記のいずれにも該当しない場合、false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gist.c: amroutine->amcanreturn に設定される。
  • src/backend/access/gist/gistutil.c: インデックスのプロパティ確認時に参照される可能性がある。
  • **呼び出し先**:
  • IndexRelationGetNumberOfKeyAttributes: インデックスのキー属性数を取得。
  • index_getprocid: インデックスのサポートプロシージャ（GIST_FETCH_PROC, GIST_COMPRESS_PROC）のIDを取得。
  • OidIsValid: プロシージャIDの有効性を確認。

関連データ構造

  • Relation: インデックスリレーション構造体。

失敗モードと境界条件

  • attno が無効な場合や、インデックスリレーションが正しく初期化されていない場合の挙動については、本関数内では明示的なエラーハンドリングを行わず、index_getprocid 等の依存関数の仕様に依存する。

保守担当者向けメモ

  • 本関数はGiSTの IndexAmRoutine の amcanreturn フィールドとして登録される。
  • GIST_FETCH_PROC が存在すれば、圧縮されたデータから元の値を復元できると見なされるため、Index-Only Scanが可能となる。
