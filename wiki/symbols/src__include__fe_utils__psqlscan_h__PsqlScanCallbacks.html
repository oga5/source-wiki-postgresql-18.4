PsqlScanCallbacks

定義

  • 種別: struct
  • 定義箇所: src/include/fe_utils/psqlscan.h:61

目的

SQL解析器（レキサー）において、変数置換などの外部拡張機能を提供するためのコールバック関数を保持する構造体です。

シグネチャまたは形状

typedef struct PsqlScanCallbacks
{
    char *(*get_variable) (const char *varname, PsqlScanQuoteType quote, void *passthrough);
} PsqlScanCallbacks;

入力と出力

  • **入力**: get_variable 関数ポインタは、変数名 (varname)、引用符の状態 (quote)、およびユーザー定義のデータポインタ (passthrough) を受け取ります。
  • **出力**: 変数の値を保持する文字列を返します。変数が不明な場合は NULL を返します。返される文字列は呼び出し元で解放（free）可能である必要があります。

前提条件と不変条件

  • 変数置換が不要な場合、get_variable ポインタは NULL に設定可能です。
  • get_variable が返す文字列は、呼び出し側がメモリを管理（解放）できる形式である必要があります。

アルゴリズムまたは振る舞い

解析器が SQL 文のトークン化を行う際、変数置換が必要な箇所に到達すると、登録された get_variable コールバックを呼び出します。これにより、解析器自体は変数の具体的な保持方法を知ることなく、外部から値を取得できます。

呼び出し元と呼び出し先

  • **呼び出し元**: psql_scan_create() 関数に引数として渡され、解析器の初期化に使用されます。
  • **呼び出し先**: 解析器の内部ロジックから、変数置換が必要なタイミングで get_variable が呼び出されます。
  • **参照元**:
  • src/bin/pgbench/pgbench.c
  • src/bin/psql/mainloop.c
  • src/test/modules/test_escape/test_escape.c

関連データ構造

  • PsqlScanState: 解析器の内部状態を保持する構造体。
  • PsqlScanQuoteType: 変数置換時の引用符の状態を示す列挙型。

失敗モードと境界条件

  • get_variable が NULL を返した場合、解析器は変数が未定義であると判断します。
  • get_variable ポインタ自体が NULL の場合、変数置換機能は無効化されます。

保守担当者向けメモ

  • 本構造体は psql や pgbench 等のフロントエンドツールで共通して利用されます。拡張を行う際は、既存のコールバック実装（psqlscan_callbacks 等）との互換性に注意してください。
