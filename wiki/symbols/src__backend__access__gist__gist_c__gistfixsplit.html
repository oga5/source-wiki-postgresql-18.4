gistfixsplit

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gist.c:1199

目的

GiSTインデックスにおいて、不完全なページ分割（incomplete split）を修復し、親ノードへのダウンリンク挿入を完了させるために使用されます。

シグネチャまたは形状

static void gistfixsplit(GISTInsertState *state, GISTSTATE *giststate)

入力と出力

  • **入力**:
  • state: 現在の挿入状態を保持する GISTInsertState 構造体。スタック情報を含む。
  • giststate: GiSTインデックスの操作状態を保持する GISTSTATE 構造体。
  • **出力**: なし（副作用としてインデックスページおよび親ノードを更新）。

前提条件と不変条件

  • state->stack が有効であること。
  • stack->downlinkoffnum が有効なオフセット番号であること。

アルゴリズムまたは振る舞い

1. ログに分割修復の開始を出力します。 2. GistFollowRight フラグが立っているページチェーンを右方向（rightlink）に辿ります。 3. 各ページに対して gistformdownlink を呼び出し、親ノードへ挿入するためのダウンリンクタプルを生成します。 4. 生成したダウンリンク情報を splitinfo リストに蓄積します。 5. 最後に gistfinishsplit を呼び出し、収集した情報を元に親ノードへのダウンリンク挿入を完了させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gist.c:720（gistdoinsert 内の挿入処理フローから呼び出される）。
  • **呼び出し先**:
  • gistformdownlink
  • gistfinishsplit
  • ReadBuffer / LockBuffer（ページチェーン走査時）

関連データ構造

  • GISTInsertState: 挿入操作のコンテキストとスタックを保持。
  • GISTSTATE: GiSTインデックスのメタデータと状態。
  • GISTPageSplitInfo: ページ分割情報とダウンリンクを保持する構造体。

失敗モードと境界条件

  • GistFollowRight フラグが立っていない場合、ループは即座に終了します。
  • ページチェーンの走査中に排他ロック（GIST_EXCLUSIVE）を取得するため、デッドロックの回避が重要です。

保守担当者向けメモ

  • 本関数は、並行処理中にページ分割が中断された際、ツリーの整合性を保つためのリカバリ処理として機能します。
  • gistfinishsplit への依存が強いため、親ノードの更新ロジックに変更を加える際は両方の整合性を確認してください。
