histogram_selectivity

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/selfuncs.c:830

目的

カラムのヒストグラム統計情報を利用して、特定の演算子に対する選択率（0.0〜1.0）を推定します。

シグネチャまたは形状

double histogram_selectivity(VariableStatData *vardata,
                             FmgrInfo *opproc, Oid collation,
                             Datum constval, bool varonleft,
                             int min_hist_size, int n_skip,
                             int *hist_size)

入力と出力

  • **入力**:
  • vardata: 対象カラムの統計情報を含む構造体。
  • opproc: 評価対象の演算子関数情報。
  • collation: 照合順序。
  • constval: 比較対象の定数値。
  • varonleft: 変数が演算子の左側にあるかを示すフラグ。
  • min_hist_size: 推定を行うために必要なヒストグラムの最小要素数。
  • n_skip: ヒストグラムの両端から除外する要素数。
  • **出力**:
  • 戻り値: 推定された選択率（成功時）。統計情報が不十分な場合は -1 を返す。
  • hist_size: 取得されたヒストグラムのサイズ（出力パラメータ）。

前提条件と不変条件

  • n_skip >= 0 であること。
  • min_hist_size > 2 * n_skip であること。
  • vardata に有効な統計情報が含まれていること。

アルゴリズムまたは振る舞い

1. get_attstatsslot を使用して STATISTIC_KIND_HISTOGRAM を取得します。 2. ヒストグラムのサイズが min_hist_size 以上であるか確認します。 3. 条件を満たす場合、ヒストグラムの各要素に対して opproc を適用し、条件を満たす要素数（nmatch）をカウントします。 4. nmatch を、有効なヒストグラム要素数（全体から 2 * n_skip を引いた数）で除算し、選択率を算出します。 5. 統計情報が欠落している、またはサイズが不足している場合は -1 を返します。 6. 最後に free_attstatsslot でメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/ts_selfuncs.c
  • src/backend/utils/adt/like_support.c
  • src/backend/utils/adt/selfuncs.c (内部の複数の箇所)
  • **呼び出し先**:
  • get_attstatsslot
  • FunctionCallInvoke
  • free_attstatsslot
  • statistic_proc_security_check

関連データ構造

  • VariableStatData: カラムの統計情報を保持する構造体。
  • AttStatsSlot: pg_statistic から抽出された統計データスロット。
  • FmgrInfo: 関数呼び出し情報を管理する構造体。

失敗モードと境界条件

  • 統計情報が取得できない場合、またはヒストグラムサイズが min_hist_size 未満の場合は -1 を返します。
  • n_skip がヒストグラムサイズに対して大きすぎる場合、アサーションエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は generic_restriction_selectivity 等から利用される汎用的なロジックです。
  • 演算子の適用結果が NULL になる場合を考慮し、FunctionCallInvoke を直接呼び出す実装になっています。
  • 統計情報の取得には get_attstatsslot を使用しているため、pg_statistic のスキーマ変更時には注意が必要です。
