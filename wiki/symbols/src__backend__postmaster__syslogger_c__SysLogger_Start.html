SysLogger_Start

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/syslogger.c:592

目的

PostgreSQL のログ収集プロセス（Syslogger）を起動し、Postmaster からの stderr 出力をパイプ経由で集約するための環境を構築する。

シグネチャまたは形状

int SysLogger_Start(int child_slot)

入力と出力

  • **入力**: child_slot（Postmaster が管理する子プロセススロット番号）。
  • **出力**: 成功時は起動した Syslogger プロセスの PID（pid_t）、失敗時は 0 を返す。

前提条件と不変条件

  • Logging_collector が有効であること（Assert により検証）。
  • ログ出力先ディレクトリ（Log_directory）が書き込み可能であること。
  • syslogPipe が未初期化または既存のパイプを再利用する状態であること。

アルゴリズムまたは振る舞い

1. **パイプの準備**: syslogPipe が未作成の場合、プラットフォームに応じて pipe()（Unix）または CreatePipe()（Windows）でパイプを作成する。 2. **ディレクトリ作成**: MakePGDirectory を呼び出し、ログディレクトリを確保する。 3. **初期ログファイルの作成**: logfile_open を使用して、標準ログ、および設定に応じて CSV/JSON ログファイルを開く。 4. **プロセス起動**: postmaster_child_launch を呼び出し、Syslogger プロセスを生成する。 5. **stderr のリダイレクト**: 成功時、dup2 を使用して標準エラー出力（および標準出力）をパイプの書き込み端へリダイレクトし、不要になった書き込み端のファイル記述子を閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/postmaster.c
  • **呼び出し先**: logfile_getname, logfile_open, postmaster_child_launch, syslogger_fdget（EXEC_BACKEND 定義時）, dup2, pipe/CreatePipe

関連データ構造

  • syslogPipe: プロセス間通信用パイプのファイル記述子。
  • syslogFile, csvlogFile, jsonlogFile: ログ書き込み用のファイルポインタ。
  • SysloggerStartupData: EXEC_BACKEND 環境下でプロセス起動時に渡されるデータ構造。

失敗モードと境界条件

  • パイプ作成失敗時: FATAL エラーで停止。
  • ログファイルオープン失敗時: logfile_open 内のロジックによりエラー処理される。
  • プロセス起動失敗時: LOG レベルでエラーを記録し、0 を返して呼び出し元へ制御を戻す。
  • リダイレクト失敗時: FATAL エラーで停止。

保守担当者向けメモ

  • Windows 環境では pipeThread を介した特殊な処理が行われるため、プラットフォーム固有のコードパスに注意すること。
  • syslogPipe は Syslogger がクラッシュして再起動する際にも再利用されるため、Postmaster 側で読み込み端を保持し続ける必要がある。
