format_type_with_typemod

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/format_type.c:361

目的

データ型OIDと型修飾子（typmod）を、SQL形式の文字列（例: varchar(255)）に変換します。

シグネチャまたは形状

char *format_type_with_typemod(Oid type_oid, int32 typemod)

入力と出力

  • **入力**:
  • type_oid: 変換対象のデータ型のOID。
  • typemod: 型修飾子（例: varchar の長さ制限）。
  • **出力**:
  • 変換された型名と修飾子を含む文字列（palloc されたメモリ領域）。

前提条件と不変条件

  • typemod が -1 の場合、型修飾子は出力されません。
  • 内部で format_type_extended を呼び出すため、システムカタログ（pg_type）へのアクセスが必要です。

アルゴリズムまたは振る舞い

1. FORMAT_TYPE_TYPEMOD_GIVEN フラグを指定して format_type_extended を呼び出します。 2. format_type_extended 内で、型名解決、スキーマ修飾、および typmodout 関数による修飾子の文字列化が行われます。 3. 生成された文字列を呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/deparse.c
  • contrib/tablefunc/tablefunc.c
  • src/backend/access/common/attmap.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • format_type_extended（直接の委譲先）

関連データ構造

  • Oid: PostgreSQLのシステムオブジェクト識別子。
  • int32: 型修飾子の内部表現。

失敗モードと境界条件

  • 無効な type_oid が渡された場合、format_type_extended の仕様に従い、エラーが発生するか、あるいは ??? のような代替文字列が返されます。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数は format_type_extended の単なるラッパーです。型フォーマットのロジックを変更する場合は format_type_extended を参照してください。
  • typmod の値が -1（未指定）の場合の挙動は、SQLパーサーの解釈と整合している必要があります。
