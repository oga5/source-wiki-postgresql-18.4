current_database

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/misc.c:194

目的

現在接続中のデータベース名を取得し、SQLの name 型として返却します。

シグネチャまたは形状

Datum current_database(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出し規約に基づく引数）。
  • **出力**: 現在のデータベース名を表す Datum（Name 型）。

前提条件と不変条件

  • MyDatabaseId が有効なデータベースを指している必要があります。
  • メモリ割り当て（palloc）が成功することが前提です。

アルゴリズムまたは振る舞い

1. NAMEDATALEN バイトのメモリを palloc で確保し、Name 型のポインタ db を作成します。 2. get_database_name(MyDatabaseId) を呼び出し、現在のバックエンドのデータベース名を取得します。 3. namestrcpy を使用して、取得した名前を db にコピーします。 4. PG_RETURN_NAME(db) を介して結果を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c:3249（実行エンジン）
  • 各種クライアントツール（pg_dump, pg_upgrade, oid2name 等）のSQLクエリ内。
  • **呼び出し先**:
  • palloc: メモリ確保。
  • get_database_name: システムカタログからデータベース名を取得。
  • namestrcpy: Name 型への文字列コピー。

関連データ構造

  • Name: PostgreSQLのシステム識別子（固定長文字列）を保持する型。
  • MyDatabaseId: 現在のセッションが接続しているデータベースのOID。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • MyDatabaseId が無効な場合、get_database_name の挙動に依存します。

保守担当者向けメモ

  • 本関数は misc.c 内のシステム状態照会機能の一部です。
  • Name 型の固定長制限（NAMEDATALEN）に注意してください。
