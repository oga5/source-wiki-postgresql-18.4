bbsink_copystream

定義

  • **種別**: struct
  • **定義箇所**: src/backend/backup/basebackup_copy.c:39

目的

PostgreSQLのベースバックアップデータをクライアントへ転送するための「COPY OUT」プロトコル実装を管理する構造体です。複数のアーカイブファイルやマニフェストを単一のCOPYストリーム内で多重化して送信する役割を担います。

シグネチャまたは形状

typedef struct bbsink_copystream
{
    bbsink      base;
    bool        send_to_client;
    char       *msgbuffer;
    TimestampTz last_progress_report_time;
    uint64      bytes_done_at_last_time_check;
} bbsink_copystream;

入力と出力

  • **入力**: bbsink インターフェースを介したバックアップイベント（アーカイブ開始、データ転送、マニフェスト生成など）。
  • **出力**: CopyData プロトコルメッセージ（タイプバイト付与済み）および進捗報告メッセージ。

前提条件と不変条件

  • base フィールドにより bbsink インターフェースを継承しています。
  • msgbuffer は、プロトコルメッセージの先頭にタイプバイト（'d'等）を配置し、その直後からデータが格納されるように設計されています。

アルゴリズムまたは振る舞い

  • **プロトコル多重化**: msgbuffer の先頭にタイプバイトを埋め込み、pq_putmessage を用いてクライアントへ送信します。
  • **進捗管理**: archive_contents 実行時に、前回の報告から一定量（PROGRESS_REPORT_BYTE_INTERVAL）以上のデータが転送された場合のみ時刻を確認し、閾値を超えていれば進捗メッセージ（'p'）を送信します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup_copy.c 内の各関数（bbsink_copystream_new 等）でインスタンス化およびキャストして利用されます。
  • **呼び出し先**: bbsink インターフェースを介して basebackup サブシステムから呼び出されます。

関連データ構造

  • bbsink: 基底となるシンクインターフェース。
  • bbsink_copystream_ops: bbsink インターフェースの関数ポインタテーブル。

失敗モードと境界条件

  • 進捗報告の頻度制御において、システムコール（時刻取得）のコストを避けるため、バイト数チェックを先行させるヒューリスティックな設計を採用しています。

保守担当者向けメモ

  • msgbuffer の先頭にタイプバイトを埋め込む際、アライメントを考慮して MAXIMUM_ALIGNOF 分のパディングを確保する工夫がなされています。
  • 本構造体は、古い「アーカイブごとにCOPY OUTを繰り返す」方式を置き換えるために実装されました。
