TypeInfo

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_dump.h:203

目的

PostgreSQLのデータベースダンプ処理において、システムカタログ上のデータ型情報をメモリ上で表現し、ダンプ対象としての属性や依存関係を管理する。

シグネチャまたは形状

typedef struct _typeInfo {
    DumpableObject dobj;
    DumpableAcl dacl;
    char       *ftypname;
    const char *rolname;
    Oid         typelem;
    Oid         typrelid;
    Oid         typarray;
    char        typrelkind;
    char        typtype;
    bool        isArray;
    bool        isMultirange;
    bool        isDefined;
    struct _shellTypeInfo *shellType;
    struct _constraintInfo *notnull;
    int         nDomChecks;
    struct _constraintInfo *domChecks;
} TypeInfo;

入力と出力

  • 入力: システムカタログ（pg_type等）から取得した情報。
  • 出力: pg_dump の各ダンプ関数（dumpType, dumpDomain 等）へ渡され、SQL生成の基盤となる。

前提条件と不変条件

  • dobj.name は pg_type.typname の生値である。
  • ftypname は format_type() の結果であり、必要に応じてクォートされ、スキーマ修飾される。
  • shellType は、必要に応じて生成されるシェル型エントリへのリンクを保持する。

アルゴリズムまたは振る舞い

  • pg_dump の収集フェーズでインスタンス化され、ダンプ実行時に dumpType 系の関数によって型定義のSQL生成に使用される。
  • ドメイン型の場合、notnull 制約や domChecks（CHECK制約）を保持し、型定義の一部として出力される。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/common.c（型情報の取得・変換）、src/bin/pg_dump/pg_dump.c（ダンプ処理のメインロジック）。
  • 呼び出し先: dumpType, dumpBaseType, dumpEnumType, dumpRangeType, dumpUndefinedType, dumpDomain, dumpCompositeType 等のダンプ関数。

関連データ構造

  • DumpableObject: 基底構造体。
  • DumpableAcl: アクセス権限情報。
  • _shellTypeInfo: シェル型情報。
  • _constraintInfo: ドメイン制約情報。

失敗モードと境界条件

  • shellType が存在しない場合、ポインタは NULL となる。
  • typrelkind や typtype の値により、ダンプ対象の型（基本型、複合型、ドメイン等）が分岐する。

保守担当者向けメモ

  • 新しい型カテゴリを追加する場合は、pg_dump.c 内のダンプ関数群および pg_dump_sort.c の依存関係処理を更新すること。
  • メモリ節約のため、詳細情報の取得は必要になるまで遅延される設計となっている。
