array_agg_serialize

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_userfuncs.c:698

目的

並列クエリ実行時において、集約関数 array_agg の中間状態（ArrayBuildState）をシリアライズし、ワーカープロセス間で転送可能なバイナリ形式（bytea）に変換する。

シグネチャまたは形状

Datum array_agg_serialize(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して ArrayBuildState へのポインタを受け取る。
  • **出力**: シリアライズされたデータを含む bytea 型の Datum を返す。

前提条件と不変条件

  • AggCheckCallContext により、集約コンテキスト内から呼び出されている必要がある。
  • state->element_type、nelems、typlen、typbyval、typalign、および dnulls 配列がシリアライズの先頭部分に格納される。
  • typbyval が真の場合、dvalues 配列はそのままバイナリとして送出される。
  • typbyval が偽の場合、各要素に対して型固有の送信関数（typsend）を呼び出し、個別にシリアライズする。

アルゴリズムまたは振る舞い

1. pq_begintypsend でバッファを初期化する。 2. 配列のメタデータ（型情報、要素数、アライメント等）をバッファに書き込む。 3. dnulls 配列を書き込む。 4. typbyval の値に応じて分岐する:

  • **真**: dvalues 配列をメモリコピーで直接書き込む。
  • **偽**: fn_extra にキャッシュされた SerialIOData を利用し、各要素を SendFunctionCall で変換して書き込む。この際、NULL要素はスキップされる。

    • 5. pq_endtypsend で bytea を生成し返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていないが、src/backend/optimizer/prep/prepagg.c において並列集約のシリアライズ処理として言及されている。
  • **呼び出し先**: pq_begintypsend, pq_sendint32, pq_sendint64, pq_sendint16, pq_sendbyte, pq_sendbytes, getTypeBinaryOutputInfo, fmgr_info_cxt, SendFunctionCall, pq_endtypsend 等。

関連データ構造

  • ArrayBuildState: 集約状態を保持する構造体。
  • SerialIOData: fn_extra にキャッシュされる、型送信関数情報を保持する構造体。
  • StringInfoData: シリアライズ用バッファ。

失敗モードと境界条件

  • fn_extra が未初期化の場合、カタログ検索を行いキャッシュを構築する。
  • typbyval が偽の場合、NULL要素は dvalues に含まれていても送信処理ではスキップされる。

保守担当者向けメモ

  • array_agg_deserialize とのプロトコル整合性を維持すること。
  • fn_extra を使用したキャッシュ戦略により、同一クエリ内でのカタログ検索コストを削減している。
  • 並列クエリの実行計画において、集約状態の転送が必要な場合にのみ呼び出される。
