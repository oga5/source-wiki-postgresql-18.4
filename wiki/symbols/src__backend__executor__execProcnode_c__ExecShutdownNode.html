ExecShutdownNode

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execProcnode.c:771

目的

プランツリーの実行状態（PlanState）に対して、実行終了に伴うシャットダウン処理を再帰的に実行する。

シグネチャまたは形状

void ExecShutdownNode(PlanState *node)

入力と出力

  • **入力**: PlanState *node - シャットダウン対象のプランノードの実行状態ツリーのルート。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 node は有効な PlanState 構造体を指している必要がある。
  • 内部で呼び出される ExecShutdownNode_walker が、ツリー全体を走査して各ノードのシャットダウン処理を完了させる。

アルゴリズムまたは振る舞い

1. ExecShutdownNode_walker(node, NULL) を呼び出す。 2. ExecShutdownNode_walker はツリーを再帰的に走査し、各ノードに対して必要なシャットダウン処理（リソースの解放やクリーンアップ）を適用する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execMain.c:1769 から呼び出されていることが確認できる。
  • **呼び出し先**: ExecShutdownNode_walker を直接呼び出す。

関連データ構造

  • PlanState: プランノードの実行状態を保持する構造体。

失敗モードと境界条件

  • node が NULL の場合の挙動については、ExecShutdownNode_walker の実装に依存する。

保守担当者向けメモ

  • 本関数は ExecShutdownNode_walker への単なるラッパーである。シャットダウン処理のロジックを変更する場合は ExecShutdownNode_walker を参照すること。
