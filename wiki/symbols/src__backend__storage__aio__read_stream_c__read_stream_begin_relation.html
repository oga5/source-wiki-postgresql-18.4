read_stream_begin_relation

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/read_stream.c:735

目的

リレーションのデータ読み込みを効率化するための ReadStream オブジェクトを初期化し、ストリームを開始する。

シグネチャまたは形状

ReadStream *read_stream_begin_relation(int flags,
                                       BufferAccessStrategy strategy,
                                       Relation rel,
                                       ForkNumber forknum,
                                       ReadStreamBlockNumberCB callback,
                                       void *callback_private_data,
                                       size_t per_buffer_data_size);

入力と出力

  • **入力**:
  • flags: ストリームの動作を制御するビットフラグ。
  • strategy: バッファアクセスの戦略（BufferAccessStrategy）。
  • rel: 対象のリレーション。
  • forknum: 読み込み対象のフォーク番号。
  • callback: 次のブロック番号を取得するためのコールバック関数。
  • callback_private_data: コールバックに渡す任意のデータ。
  • per_buffer_data_size: 各バッファに関連付ける追加データのサイズ。
  • **出力**:
  • 初期化された ReadStream 構造体へのポインタ。

前提条件と不変条件

  • rel は有効なリレーションであること。
  • callback は、ストリームが要求するブロック番号を正しく提供できること。

アルゴリズムまたは振る舞い

1. rel から SmgrRelation を取得する。 2. rel->rd_rel->relpersistence からリレーションの永続性情報を取得する。 3. 取得した情報と引数を read_stream_begin_impl に渡し、ストリームの実装を初期化してその結果を返す。

関連データ構造

  • ReadStream: ストリームの状態を保持するメイン構造体。
  • Relation: リレーション情報を保持する構造体。
  • BufferAccessStrategy: バッファ管理の戦略定義。

失敗モードと境界条件

  • メモリ不足時や無効な引数による read_stream_begin_impl 内部でのエラー発生時に失敗する可能性がある。

保守担当者向けメモ

  • 本関数は read_stream_begin_impl のラッパーであり、リレーションから必要な情報を抽出して実装層へ委譲する役割を持つ。
  • 変更を加える際は、read_stream_begin_impl の引数構成との整合性に注意すること。
