paraminfo_get_equal_hashops

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/joinpath.c:438

目的

Memoize パス生成のために、結合条件およびラテラル変数からハッシュキーとして使用可能な式と演算子のリストを抽出する。

シグネチャまたは形状

bool paraminfo_get_equal_hashops(PlannerInfo *root, ParamPathInfo *param_info,
                                 RelOptInfo *outerrel, RelOptInfo *innerrel,
                                 List *ph_lateral_vars, List **param_exprs,
                                 List **operators, bool *binary_mode)

入力と出力

  • **入力**:
  • param_info: 結合条件を含む ParamPathInfo。
  • outerrel, innerrel: 結合対象のリレーション。
  • ph_lateral_vars: プレースホルダーのラテラル変数リスト。
  • **出力**:
  • param_exprs: ハッシュキーとして使用する式のリスト。
  • operators: 各式に対応するハッシュ等価演算子の OID リスト。
  • binary_mode: ビット単位の比較が必要な場合に true となるフラグ。
  • 戻り値: 抽出に成功した場合は true、失敗した場合は false。

前提条件と不変条件

  • param_info が提供される場合、その ppi_clauses は 2 引数の OpExpr である必要がある。
  • ラテラル変数に揮発性（volatile）関数が含まれている場合は失敗する。
  • ハッシュ関数や等価演算子が取得できない型が含まれる場合は失敗する。

アルゴリズムまたは振る舞い

1. **結合条件の処理**: param_info 内の各 RestrictInfo を走査し、ハッシュ可能な等価演算子を特定して param_exprs と operators に追加する。 2. **バイナリモードの判定**: 結合演算子がハッシュ可能でない場合、またはラテラル変数を扱う場合は、論理的な等価性ではなくビット単位の比較を行うため binary_mode を true に設定する。 3. **ラテラル変数の処理**: ph_lateral_vars および innerrel->lateral_vars を走査し、型キャッシュからハッシュプロシージャと等価演算子を取得してキーに追加する。 4. 重複する式はリストに追加しない。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/joinpath.c 内の get_memoize_path（推測される文脈上の参照）。
  • **呼び出し先**: clause_sides_match_join, list_member, lappend_oid, contain_volatile_functions, lookup_type_cache 等。

関連データ構造

  • ParamPathInfo: パラメータ化されたパスの制約情報を保持。
  • RestrictInfo: 結合条件式を保持。
  • TypeCacheEntry: 型のハッシュ関数や等価演算子情報。

失敗モードと境界条件

  • 結合条件が OpExpr ではない、または引数が 2 つでない場合。
  • 結合条件が指定されたリレーションの辺と一致しない場合。
  • ハッシュ等価演算子が見つからない場合。
  • ラテラル変数に揮発性関数が含まれる場合。
  • 型キャッシュから必要なハッシュプロシージャや等価演算子が取得できない場合。

保守担当者向けメモ

  • 本関数は Memoize パスの生成可否を決定する重要なロジックである。
  • binary_mode の設定は、浮動小数点数などの「論理的等価」と「ビット単位の等価」が異なる型を扱う際に重要となる。
