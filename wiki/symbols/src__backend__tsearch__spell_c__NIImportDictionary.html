NIImportDictionary

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/spell.c:519

目的

ISpell 形式の辞書ファイル（.dict）を読み込み、各単語とその接辞フラグを解析して、辞書構造体へ登録する。

シグネチャまたは形状

void NIImportDictionary(IspellDict *Conf, const char *filename)

入力と出力

  • **入力**:
  • Conf: 辞書の状態を保持する IspellDict 構造体へのポインタ。
  • filename: 読み込む辞書ファイルのパス。
  • **出力**: なし（副作用として Conf に単語データが追加される）。

前提条件と不変条件

  • ファイルは tsearch_readline で読み取り可能な形式であること。
  • 辞書内のフラグは、1バイト文字かつ表示可能かつ空白を含まない文字である必要がある（それ以外は切り捨てられる）。

アルゴリズムまたは振る舞い

1. tsearch_readline_begin を用いてファイルを開く。 2. ファイルの各行を tsearch_readline で順次読み込む。 3. 行内の / を区切り文字として、単語本体と接辞フラグを分離する。 4. 単語末尾の空白を除去し、lowerstr_ctx を用いて小文字化する。 5. NIAddSpell を呼び出し、処理済みの単語とフラグを Conf に登録する。 6. 読み込み終了後、tsearch_readline_end でリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/dict_ispell.c (辞書初期化プロセスの一部として呼び出される)。
  • **呼び出し先**: tsearch_readline_begin, tsearch_readline, findchar, pg_mblen_cstr, isprint, isspace, lowerstr_ctx, NIAddSpell, pfree, tsearch_readline_end。

関連データ構造

  • IspellDict: 辞書の状態を保持するメイン構造体。

失敗モードと境界条件

  • ファイルを開けない場合、ereport(ERROR, ...) により処理が中断される。
  • フラグ部分にマルチバイト文字や制御文字が含まれる場合、その位置でフラグの解析が打ち切られる。

保守担当者向けメモ

  • 本関数は辞書構築の初期段階で実行される。メモリ消費が大きいため、構築用コンテキストの管理に注意すること。
  • 辞書ファイルのフォーマット変更や、新しい接辞フラグの仕様追加時には、フラグ解析ロジックの修正が必要となる可能性がある。
