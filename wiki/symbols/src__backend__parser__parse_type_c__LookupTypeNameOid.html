LookupTypeNameOid

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_type.c:231

目的

TypeName 構造体から対応するシステムカタログ上の型 OID を検索・取得します。

シグネチャまたは形状

Oid LookupTypeNameOid(ParseState *pstate, const TypeName *typeName, bool missing_ok)

入力と出力

  • **入力**:
  • pstate: パーサーの状態（エラー報告用）。NULL 指定可能。
  • typeName: 解決対象の型名構造体。
  • missing_ok: 型が見つからない場合にエラーを発生させるか（false）、InvalidOid を返すか（true）を指定するフラグ。
  • **出力**:
  • 成功時: 対応する型の Oid。
  • 失敗時: missing_ok が true の場合は InvalidOid。

前提条件と不変条件

  • 取得した内部キャッシュエントリは、関数終了時に ReleaseSysCache によって解放されます。
  • missing_ok が false の場合、型が見つからなければ ereport(ERROR, ...) により処理が中断されます。

アルゴリズムまたは振る舞い

1. LookupTypeName を呼び出し、TypeName に対応する Type（HeapTuple）を取得します。 2. tup が NULL の場合:

  • missing_ok が false ならば、ERRCODE_UNDEFINED_OBJECT でエラーを報告します。
  • missing_ok が true ならば、InvalidOid を返して終了します。

    • 3. tup が有効な場合:

  • GETSTRUCT マクロを用いて Form_pg_type 構造体へキャストし、oid フィールドを抽出します。
  • ReleaseSysCache(tup) を呼び出し、キャッシュ参照を解放します。

    • 4. 抽出した typoid を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c
  • src/backend/commands/dropcmds.c
  • src/backend/parser/parse_func.c
  • src/backend/parser/parse_oper.c
  • **呼び出し先**:
  • LookupTypeName (同ファイル内)
  • TypeNameToString (同ファイル内)
  • ReleaseSysCache (utils/syscache.h)
  • ereport / parser_errposition (postgres.h)

関連データ構造

  • TypeName: 構文解析された型名の表現。
  • Form_pg_type: pg_type システムカタログの行構造。
  • Type: HeapTuple への typedef。

失敗モードと境界条件

  • 指定された型名がシステムカタログに存在しない場合、missing_ok が false であればエラーとなります。
  • pstate が NULL の場合、エラー発生時の位置情報報告は行われません。

保守担当者向けメモ

  • 本関数は LookupTypeName のラッパーであり、呼び出し側で ReleaseSysCache を意識する必要がないよう設計されています。
  • 型の存在確認のみが必要な場合に適したインターフェースです。
