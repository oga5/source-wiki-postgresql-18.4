PGSharedMemoryNoReAttach

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32_shmem.c:471

目的

Windows 環境において、子プロセスが共有メモリセグメントを再アタッチしないことを明示し、関連するリソースを適切に解放するために使用されます。

シグネチャまたは形状

void PGSharedMemoryNoReAttach(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数の更新とリソース解放を行う）

前提条件と不変条件

  • ShmemProtectiveRegion が NULL でないこと。
  • UsedShmemSegAddr が NULL でないこと。
  • IsUnderPostmaster が真であること（Postmaster 配下の子プロセスであること）。

アルゴリズムまたは振る舞い

1. UsedShmemSegAddr を NULL にリセットし、現在のプロセスが共有メモリにアタッチされていない状態であることを示します。 2. PGSharedMemoryDetach() を呼び出し、継承された共有メモリセグメントのハンドルをクローズします。これにより、Windows が当該プロセスを共有メモリの保持者として認識し続けることを防ぎます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/postmaster/launch_backend.c:669
  • src/backend/port/win32_shmem.c:458（直接の呼び出し元として示唆）
  • src/backend/port/sysv_shmem.c（参照として示唆）
  • 呼び出し先:
  • PGSharedMemoryDetach()

関連データ構造

  • UsedShmemSegAddr: 共有メモリのベースアドレスを保持するグローバル変数。
  • ShmemProtectiveRegion: アドレス空間保護のための領域。

失敗モードと境界条件

  • 前提条件の Assert に失敗した場合、プログラムは異常終了します。
  • PGSharedMemoryDetach の内部処理が失敗した場合、共有メモリハンドルがリークする可能性があります。

保守担当者向けメモ

  • 本関数は Windows 特有のメモリ管理ロジックです。子プロセスが共有メモリを必要としない場合に、OS 側の参照カウントを正しく減らすために不可欠です。
  • sysv_shmem.c からの参照については、プラットフォーム間のインターフェース整合性によるものですが、本関数の実装自体は win32_shmem.c に限定されています。
