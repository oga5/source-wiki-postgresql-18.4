ExecEndRecursiveUnion

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeRecursiveunion.c:284

目的

WITH RECURSIVE クエリ実行ノードの終了処理を行い、再帰処理で使用したリソースを解放する。

シグネチャまたは形状

void ExecEndRecursiveUnion(RecursiveUnionState *node)

入力と出力

  • **入力**: RecursiveUnionState *node （再帰実行ノードの状態構造体）
  • **出力**: なし（副作用としてメモリ解放とサブプランの終了を実行）

前提条件と不変条件

  • node は ExecInitRecursiveUnion によって初期化済みであること。
  • node->working_table および node->intermediate_table が有効なタプルストアを指していること。

アルゴリズムまたは振る舞い

1. tuplestore_end を呼び出し、再帰処理で使用した working_table と intermediate_table を解放する。 2. node->tempContext および node->tableContext が存在する場合、MemoryContextDelete を呼び出し、関連するメモリコンテキスト（ハッシュテーブル等を含む）を破棄する。 3. ExecEndNode を呼び出し、outerPlanState(node)（非再帰項）および innerPlanState(node)（再帰項）の終了処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:609 （実行エンジンからの終了要求）
  • **呼び出し先**:
  • tuplestore_end
  • MemoryContextDelete
  • ExecEndNode

関連データ構造

  • RecursiveUnionState: 再帰実行ノードの状態を保持する構造体。
  • tuplestore: 再帰クエリの結果を一時的に保持するストレージ。
  • MemoryContext: メモリ管理用コンテキスト。

失敗モードと境界条件

  • node が NULL の場合、あるいは node 内の各ポインタが初期化されていない状態での呼び出しは未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は RecursiveUnionState に紐付くすべてのリソースをクリーンアップする責務を持つ。新しいリソース（メモリやファイルハンドル等）を RecursiveUnionState に追加した場合は、必ず本関数で解放処理を追加すること。
