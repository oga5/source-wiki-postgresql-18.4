_SPI_connection

定義

  • 種別: struct
  • 定義箇所: src/include/executor/spi_priv.h:22

目的

SPI（Server Programming Interface）呼び出しにおける実行状態を保持し、ネストされた呼び出しのスタック管理を行うための内部構造体です。

シグネチャまたは形状

typedef struct {
    uint64 processed;
    SPITupleTable *tuptable;
    SubTransactionId execSubid;
    slist_head tuptables;
    MemoryContext procCxt;
    MemoryContext execCxt;
    MemoryContext savedcxt;
    SubTransactionId connectSubid;
    QueryEnvironment *queryEnv;
    bool atomic;
    bool internal_xact;
    uint64 outer_processed;
    SPITupleTable *outer_tuptable;
    int outer_result;
} _SPI_connection;

入力と出力

本構造体は関数引数ではなく、SPI 実行時の内部状態としてメモリ上に保持されます。SPI 呼び出しの開始時に初期化され、終了時に破棄またはスタックからポップされます。

前提条件と不変条件

  • _SPI_connection は SPI の内部的な状態管理用であり、外部モジュールからの直接利用は想定されていません。
  • ネストされた呼び出しが発生した場合、上位レベルの実行状態（outer_ で始まるメンバ）が本構造体に退避されます。

アルゴリズムまたは振る舞い

  • SPI 呼び出しが開始されると、現在のメモリコンテキストやサブトランザクション ID が記録されます。
  • 実行中のクエリ結果（processed 行数や tuptable）を保持し、実行コンテキスト（execCxt）やプロシージャコンテキスト（procCxt）を管理します。
  • atomic フラグにより、トランザクション制御が許可されるコンテキストか否かを判定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/spi.c 内の SPI 関連関数（SPI_connect 等）が本構造体をスタック（_SPI_stack）として管理しています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出し先はありません。

関連データ構造

  • SPITupleTable: 実行結果のタプルを保持するリスト。
  • MemoryContext: 実行状態の生存期間を制御するメモリ領域。
  • QueryEnvironment: クエリ実行環境の設定。
  • SubTransactionId: 実行に関連するサブトランザクションの識別子。

失敗モードと境界条件

  • SPI のネストが深すぎる場合や、メモリコンテキストの管理不整合が発生した場合に、実行状態の復元が正しく行われない可能性があります。
  • internal_xact が設定されている場合、SPI 管理下のトランザクション境界としてクリーンアップ処理がスキップされる挙動をとります。

保守担当者向けメモ

  • _SPI_connection 内の outer_ メンバ変数がスタックとしてどのように機能しているかを追うことで、SPI のネストレベルを理解できます。
  • 本構造体は src/backend/executor/spi.c の _SPI_stack および _SPI_current によって参照・管理されています。
