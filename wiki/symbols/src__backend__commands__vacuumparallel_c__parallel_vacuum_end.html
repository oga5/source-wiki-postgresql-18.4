parallel_vacuum_end

定義

  • 種別: function
  • 定義箇所: src/backend/commands/vacuumparallel.c:435

目的

並列VACUUM処理の終了処理を行い、共有メモリリソースの解放と統計情報の回収を行う。

シグネチャまたは形状

void parallel_vacuum_end(ParallelVacuumState *pvs, IndexBulkDeleteResult **istats)

入力と出力

  • **入力**:
  • pvs: 並列VACUUMの状態を保持する ParallelVacuumState 構造体へのポインタ。
  • istats: 各インデックスの統計情報を格納するための IndexBulkDeleteResult ポインタの配列。
  • **出力**: なし。istats 配列を通じて更新された統計情報が呼び出し元に返される。

前提条件と不変条件

  • 本関数はリーダープロセスによって呼び出される必要がある（!IsParallelWorker() のアサーションにより強制）。

アルゴリズムまたは振る舞い

1. pvs->indstats を走査し、istat_updated が真であるインデックスの統計情報を istats 配列へコピーする。 2. TidStoreDestroy を呼び出し、共有メモリ上の削除対象タプル情報を破棄する。 3. DestroyParallelContext を呼び出し、並列実行コンテキストを解放する。 4. ExitParallelMode を呼び出し、並列モードを終了する。 5. pvs->will_parallel_vacuum および pvs 自体のメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/vacuumlazy.c
  • **呼び出し先**: TidStoreDestroy, DestroyParallelContext, ExitParallelMode, palloc0, memcpy, pfree

関連データ構造

  • ParallelVacuumState: 並列VACUUMの制御状態を保持。
  • IndexBulkDeleteResult: インデックスのバルク削除後の統計情報。
  • PVIndStats: インデックスごとの統計情報と更新フラグ。

失敗モードと境界条件

  • pvs が NULL の場合や、既に解放済みのメモリを参照した場合はクラッシュする可能性がある。
  • 並列ワーカーが予期せず終了している場合、統計情報の回収が不完全になる可能性がある。

保守担当者向けメモ

  • 本関数は並列VACUUMの最終フェーズを担当する。メモリリークを防ぐため、ParallelVacuumState に追加された新しい動的メモリ領域は必ずここで解放すること。
