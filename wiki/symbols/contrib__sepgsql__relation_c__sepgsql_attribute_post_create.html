sepgsql_attribute_post_create

定義

  • **種別**: function
  • **定義箇所**: contrib/sepgsql/relation.c:42

目的

新規作成されたテーブルカラムに対し、SELinuxポリシーに基づいたデフォルトのセキュリティラベルを計算・適用し、その作成権限を検証する。

シグネチャまたは形状

void sepgsql_attribute_post_create(Oid relOid, AttrNumber attnum)

入力と出力

  • **入力**:
  • relOid (Oid): カラムが属するリレーションのOID。
  • attnum (AttrNumber): 作成されたカラムの属性番号。
  • **出力**: なし（副作用としてシステムカタログ pg_seclabel を更新）。

前提条件と不変条件

  • 対象のリレーションが RELKIND_RELATION または RELKIND_PARTITIONED_TABLE であること（それ以外の場合は何もせず終了する）。
  • SnapshotSelf を使用して、現在のトランザクション内で作成されたカラムのカタログレコードが参照可能であること。

アルゴリズムまたは振る舞い

1. **対象確認**: リレーションの種類を判定し、対象外であれば早期リターンする。 2. **カタログ取得**: pg_attribute を SnapshotSelf でスキャンし、指定された relOid と attnum に該当するタプルを取得する。 3. **ラベル計算**:

  • クライアントのラベルと親リレーションのラベルを取得。
  • sepgsql_compute_create を呼び出し、カラム名に基づいた新しいセキュリティラベルを生成する。

    • 4. **権限検証**: sepgsql_avc_check_perms_label を呼び出し、db_column:{create} 権限を検証する。 5. **ラベル適用**: SetSecurityLabel を使用して、生成したラベルを対象カラムに永続化する。 6. **クリーンアップ**: スキャン終了、テーブルクローズ、およびメモリ確保したラベル文字列の解放を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/hooks.c:138（DDLフック経由）。
  • **呼び出し先**:
  • table_open, table_close
  • systable_beginscan, systable_getnext, systable_endscan
  • sepgsql_get_client_label, sepgsql_get_label, sepgsql_compute_create
  • sepgsql_avc_check_perms_label
  • SetSecurityLabel

関連データ構造

  • Form_pg_attribute: カラムのメタデータ構造体。
  • ObjectAddress: ラベル適用対象を特定するための構造体。
  • StringInfoData: 監査ログ用の識別名生成に使用。

失敗モードと境界条件

  • **エラー**: 指定された relOid と attnum に対応するタプルがカタログに見つからない場合、elog(ERROR) を発生させる。
  • **境界条件**: インデックスなど、個別のセキュリティラベルを持たないオブジェクトに対しては処理を行わない。

保守担当者向けメモ

  • 本関数は SnapshotSelf を使用しているため、DDL実行直後の未コミット状態のカタログレコードを正しく参照できる。
  • 監査ログには getObjectIdentity で取得したリレーション名とカラム名を連結した文字列が使用される。
