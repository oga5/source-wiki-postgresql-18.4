dataGetLeftMostPage

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/gindatapage.c:363

目的

GINポスティングツリーの内部ノードにおいて、最も左側にある子ノードのブロック番号を取得する。

シグネチャまたは形状

static BlockNumber dataGetLeftMostPage(GinBtree btree, Page page)

入力と出力

  • **入力**:
  • btree: GIN B-treeの制御構造体。
  • page: 対象となるGINデータページのポインタ。
  • **出力**:
  • BlockNumber: ページ内の最初のオフセットに格納されている子ノードのブロック番号。

前提条件と不変条件

  • 対象ページはリーフページであってはならない (!GinPageIsLeaf(page))。
  • 対象ページはGINデータページである必要がある (GinPageIsData(page))。
  • ページ内の最大オフセット番号は FirstOffsetNumber 以上である必要がある。

アルゴリズムまたは振る舞い

1. ページ内の最初の有効なエントリ（FirstOffsetNumber）を GinDataPageGetPostingItem を用いて取得する。 2. 取得した PostingItem から PostingItemGetBlockNumber を介してブロック番号を抽出し、それを戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/gindatapage.c:1890 にて、btree->getLeftMostChild 関数ポインタに本関数が代入されている。
  • **呼び出し先**:
  • GinDataPageGetPostingItem (マクロ/インライン関数)
  • PostingItemGetBlockNumber (マクロ/インライン関数)

関連データ構造

  • GinBtree: GIN B-treeの操作状態を保持する構造体。
  • Page: PostgreSQLの標準的なページ構造体。
  • PostingItem: ポスティングツリー内のノード間リンクを保持する構造体。

失敗モードと境界条件

  • 前提条件の Assert に違反する場合（リーフページを渡した場合や、空のページを渡した場合）、デバッグビルドではアサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は GinBtree インターフェースの一部として使用されるため、直接呼び出すよりも btree->getLeftMostChild を介した利用が想定されている。
