BTScanInsertData

定義

  • 種別: struct
  • 定義箇所: src/include/access/nbtree.h:795

目的

B-tree インデックスの挿入操作および検索操作において、ツリーを降下するために必要なプライベートな状態を保持する構造体です。

シグネチャまたは形状

typedef struct BTScanInsertData
{
	bool		heapkeyspace;
	bool		allequalimage;
	bool		anynullkeys;
	bool		nextkey;
	bool		backward;
	ItemPointer scantid;
	int			keysz;
	ScanKeyData scankeys[INDEX_MAX_KEYS];
} BTScanInsertData;

入力と出力

  • **入力**: 挿入または検索の開始時に、インデックスのキー情報やスキャン方向、タイブレーカーとしての scantid を設定して初期化されます。
  • **出力**: 構造体自体は状態保持用であり、ツリー降下プロセスを通じてインデックス内の適切な位置を特定するための情報源として機能します。

前提条件と不変条件

  • scankeys 配列は構造体の最後に配置される必要があります。
  • keysz は INDEX_MAX_KEYS を超えない範囲で、有効なスキャンキーの数を保持します。

アルゴリズムまたは振る舞い

  • B-tree のルートからリーフへの降下処理において、現在のスキャン状態を管理します。
  • backward フラグにより、インデックススキャンの方向を制御します。
  • heapkeyspace や allequalimage などのフラグは、インデックスのキー比較や重複排除のロジックにおいて、現在のキーがどのように扱われるべきかを決定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtsearch.c (インデックス検索の初期化時)
  • src/backend/access/nbtree/nbtutils.c (スキャンキーのメモリ確保時)
  • **呼び出し先**:
  • 提供されたソース情報には、本構造体を利用する具体的な関数呼び出しの詳細は含まれていません。

関連データ構造

  • ScanKeyData: インデックスの検索条件を格納する配列要素。
  • BTScanInsert: BTScanInsertData へのポインタ型。

失敗モードと境界条件

  • keysz が INDEX_MAX_KEYS を超えて設定された場合、メモリ破壊や未定義動作を引き起こす可能性があります。
  • scantid が適切に設定されていない場合、重複キーが存在するインデックスでの検索順序が保証されません。

保守担当者向けメモ

  • 本構造体は nbtree の挿入・検索ロジックの根幹に関わるため、変更時は nbtsearch.c および nbtinsert.c への影響を十分に確認してください。
  • BTScanInsert 型（ポインタ）を介して操作されることが一般的です。
