dblink_get_pkey

定義

  • 種別: function
  • 定義箇所: contrib/dblink/dblink.c:1512

目的

指定されたリレーション（テーブル）の主キー情報を取得し、その属性名と位置をセットとして返す SRF（Set-Returning Function）です。

シグネチャまたは形状

Datum dblink_get_pkey(PG_FUNCTION_ARGS)
  • 入力: SQL 関数引数（リレーション名を表す text 型）
  • 出力: (position int4, colname text) 型のタプルセット

入力と出力

  • **入力**: PG_GETARG_TEXT_PP(0) を介してリレーション名を受け取ります。
  • **出力**: SRF_RETURN_NEXT を通じて、主キーの各列について position（1始まりのインデックス）と colname（列名）を含むタプルを返します。

前提条件と不変条件

  • 呼び出し元は、対象リレーションに対する AccessShareLock を取得できる権限（ACL_SELECT）を有している必要があります。
  • SRF_IS_FIRSTCALL() ブロック内でメモリコンテキストを funcctx->multi_call_memory_ctx に切り替え、複数回の呼び出しにわたる永続性を確保します。

アルゴリズムまたは振る舞い

1. **初回呼び出し時**:

  • get_rel_from_relname で対象リレーションを開き、get_pkey_attnames で主キーの属性名配列を取得します。
  • CreateTemplateTupleDesc で出力用のタプル記述子（position と colname）を作成し、TupleDescGetAttInMetadata でメタデータを初期化します。
  • 主キーが存在しない場合は直ちに終了します。

    • 2. **毎回の呼び出し時**:

  • funcctx から保持された主キー情報を取り出します。
  • call_cntr が max_calls に達するまで、現在の属性情報を BuildTupleFromCStrings でタプル化して返却します。
  • 全ての属性を返し終えたら SRF_RETURN_DONE を発行します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は特定されていませんが、SQL インターフェースから呼び出されることを想定しています。
  • **呼び出し先**: get_rel_from_relname, get_pkey_attnames, relation_close, CreateTemplateTupleDesc, TupleDescInitEntry, TupleDescGetAttInMetadata, BuildTupleFromCStrings 等の PostgreSQL 内部関数を呼び出します。

関連データ構造

  • FuncCallContext: SRF の状態保持に使用。
  • AttInMetadata: C 文字列からタプルを構築するためのメタデータ。
  • Relation: 対象テーブルのメタデータ保持に使用。

失敗モードと境界条件

  • 対象リレーションが存在しない場合や、アクセス権限がない場合はエラーが発生します。
  • 主キーが定義されていないリレーションに対しては、結果を返さずに終了します。

保守担当者向けメモ

  • 本関数は SRF_FIRSTCALL_INIT を使用しているため、メモリリークを防ぐために MemoryContext の切り替えが適切に行われていることを確認してください。
  • get_pkey_attnames が返す配列のメモリ管理は funcctx->user_fctx に依存しているため、コンテキストの生存期間に注意してください。
