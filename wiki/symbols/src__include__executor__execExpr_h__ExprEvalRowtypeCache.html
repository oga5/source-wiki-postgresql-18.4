ExprEvalRowtypeCache

定義

  • 種別: struct
  • 定義箇所: src/include/executor/execExpr.h:45

目的

複合型（Composite Type）または匿名 RECORD 型のタプル記述子（TupleDesc）をキャッシュし、実行時の型解決を効率化するために使用されます。

シグネチャまたは形状

typedef struct ExprEvalRowtypeCache
{
    void       *cacheptr;
    uint64      tupdesc_id;
} ExprEvalRowtypeCache;

入力と出力

  • **入力**: 実行エンジンが評価する式に関連付けられたタプル記述子情報。
  • **出力**: cacheptr を介して提供される TypeCacheEntry へのポインタ、または匿名 RECORD 型の場合はキャッシュされた TupleDesc へのポインタ。

前提条件と不変条件

  • 初期状態では cacheptr は NULL です。
  • tupdesc_id が 0 でない場合、cacheptr は複合型の TypeCacheEntry を指します。
  • tupdesc_id が 0 の場合、cacheptr は匿名 RECORD 型の TupleDesc を直接指します。

アルゴリズムまたは振る舞い

実行エンジンは、この構造体を使用してタプル記述子の変更を追跡します。tupdesc_id を比較することで、前回の評価時と型が変更されていないかを確認し、変更がない場合はキャッシュされた cacheptr を再利用します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execExpr.c および src/backend/executor/execExprInterp.c 内の式評価ロジック。
  • **呼び出し先**: 提供されたソース情報からは、具体的な関数呼び出し関係は特定できません。

関連データ構造

  • TypeCacheEntry: cacheptr が指す可能性のある構造体。
  • TupleDesc: 匿名 RECORD 型の場合にキャッシュされる構造体。

失敗モードと境界条件

  • cacheptr が NULL の初期状態から、どのように最初のキャッシュが生成されるかについては、本ソース抜粋には記載がありません。
  • メモリ制約のため、cacheptr と tupdesc_id は単一の構造体にまとめられており、これ以上のフィールド追加は推奨されません。

保守担当者向けメモ

  • ExprEvalStep の一部として使用されることが多く、メモリレイアウトの効率性が重視されています。
  • tupdesc_id の値が 0 か否かで cacheptr の指す先が異なるため、アクセス時には必ずこのフラグを確認する必要があります。
