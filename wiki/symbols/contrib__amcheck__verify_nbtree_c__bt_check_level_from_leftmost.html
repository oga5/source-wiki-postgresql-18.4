bt_check_level_from_leftmost

定義

  • 種別: function
  • 定義箇所: contrib/amcheck/verify_nbtree.c:623

目的

B-Tree インデックスの特定のレベルにおいて、左端のページから右リンク（btpo_next）を順次辿り、各ページが B-Tree の構造的不変条件を満たしているかを検証する。

シグネチャまたは形状

static BtreeLevel bt_check_level_from_leftmost(BtreeCheckState *state, BtreeLevel level)

入力と出力

  • **入力**:
  • state: 検証状態を保持する BtreeCheckState 構造体。
  • level: 検証対象のレベル情報（開始ブロック番号など）。
  • **出力**:
  • BtreeLevel: 次のレベル（下位レベル）の開始情報を格納した構造体。リーフレベルの場合は leftmost が P_NONE となる。

前提条件と不変条件

  • 呼び出し元は、指定された level.leftmost がそのレベルの有効な開始点であることを期待する。
  • readonly モードの場合、ページが完全に削除（P_ISDELETED）されていることは許容されない。
  • 兄弟リンク（btpo_prev）と現在の走査順序は整合している必要がある。
  • 循環リンク（current == leftcurrent 等）は存在してはならない。

アルゴリズムまたは振る舞い

1. 指定された level.leftmost から開始し、btpo_next を辿ってレベル内の全ページを走査する。 2. 各ページに対して以下を実行する:

  • CHECK_FOR_INTERRUPTS() を呼び出し、割り込みをチェックする。
  • ページが削除済み（P_IGNORE）の場合、readonly モードであればエラーを報告し、それ以外はスキップする。
  • レベルの最初のページであれば、readonly モード時に左端の正当性とルートの整合性を検証し、下位レベルの開始情報を取得する。
  • 兄弟リンクの整合性を確認し、必要に応じて bt_recheck_sibling_links を呼び出す。
  • bt_target_page_check を呼び出し、ページ内のキー順序や不変条件を検証する。

    • 3. 循環リンクを検出し、エラーを報告する。 4. 走査終了後、下位レベルの情報を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_nbtree.c 内の bt_check_every_level から呼び出される。
  • **呼び出し先**:
  • bt_target_page_check
  • bt_recheck_sibling_links
  • bt_leftmost_ignoring_half_dead
  • PageGetItemIdCareful
  • BTreeTupleGetDownLink

関連データ構造

  • BtreeCheckState: 検証の全体状態を保持。
  • BtreeLevel: レベルの走査状態を保持。
  • BTPageOpaque: B-Tree ページのメタデータ（リンク情報やレベルなど）。

失敗モードと境界条件

  • **インデックス破損**: 循環リンク、兄弟リンクの不整合、レベル不一致、または削除済みページへの到達が検出された場合に ERROR を発生させる。
  • **境界条件**:
  • ページが右端（P_RIGHTMOST）であるにもかかわらず、期待される走査が完了していない場合。
  • リーフレベルに到達した際の nextleveldown の初期化。

保守担当者向けメモ

  • 本関数は readonly モードにおいて、より厳密な整合性チェック（削除済みページの検出など）を行う。
  • ページ走査中に MemoryContextSwitchTo を使用してメモリ管理を行っているため、コンテキストの切り替えに注意が必要。
