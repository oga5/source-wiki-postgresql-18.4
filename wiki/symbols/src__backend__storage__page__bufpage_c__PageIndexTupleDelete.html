PageIndexTupleDelete

定義

  • 種別: function
  • 定義箇所: src/backend/storage/page/bufpage.c:1050

目的

ページ内の指定されたオフセット番号に対応するタプルを削除し、ページ内の断片化を解消して空き領域を再構成します。

シグネチャまたは形状

void PageIndexTupleDelete(Page page, OffsetNumber offnum)

入力と出力

  • **入力**:
  • page: 操作対象のページへのポインタ。
  • offnum: 削除対象のタプルを示す OffsetNumber。
  • **出力**: なし（ページの内容が直接更新されます）。

前提条件と不変条件

  • ページヘッダーの pd_lower、pd_upper、pd_special が整合していること。
  • offnum がページ内の有効な範囲内であること。
  • 削除対象のアイテムIDがストレージを保持していること。

アルゴリズムまたは振る舞い

1. **検証**: ページヘッダーのポインタ値が破損していないかチェックし、offnum が有効か確認します。 2. **アイテムIDの削除**: 削除対象の offnum 以降のアイテムID配列を前方へ詰め、配列を再構成します。 3. **タプルの移動**: 削除されたタプルよりページ先頭側にあるタプルデータを前方へ移動させ、空き領域を統合します。 4. **ポインタ更新**: ページヘッダーの pd_upper（タプル領域の開始位置）を増加させ、pd_lower（アイテムID配列の終了位置）を減少させます。 5. **オフセット調整**: ページ内の残存する全アイテムIDを走査し、移動したタプルを指すオフセット値を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: GIN、GiST、B-tree、SP-GiST の各インデックス管理コード（ginentrypage.c, ginvacuum.c, ginxlog.c, gist.c, gistvacuum.c, gistxlog.c, nbtpage.c, nbtxlog.c, spgdoinsert.c など）。
  • **呼び出し先**: PageGetMaxOffsetNumber, PageGetItemId, ItemIdHasStorage, ItemIdGetLength, ItemIdGetOffset, PageIsEmpty などのマクロおよび memmove。

関連データ構造

  • PageHeaderData: ページヘッダー構造体。
  • ItemIdData: アイテムID配列の各要素。

失敗モードと境界条件

  • ページヘッダーのポインタ値が不正な場合、ERRCODE_DATA_CORRUPTED でエラーを報告します。
  • offnum が範囲外の場合、elog(ERROR) を発生させます。
  • 削除対象のタプルがページ先頭にある場合、データ移動はスキップされます。

保守担当者向けメモ

  • 本関数はページ内の物理的な再配置を伴うため、インデックスの整合性を維持するクリティカルなパスで使用されます。
  • memmove を使用してメモリを操作するため、ページ内のデータ構造（スロット付きページ）のレイアウトに強く依存しています。
