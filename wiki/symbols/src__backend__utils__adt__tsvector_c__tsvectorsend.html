tsvectorsend

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector.c:407

目的

tsvector 型のデータを、ネットワーク転送やデータベース間通信に適したバイナリ形式にシリアライズします。

シグネチャまたは形状

Datum tsvectorsend(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_TSVECTOR(0) を介して取得される TSVector 構造体。
  • **出力**: bytea 型のバイナリデータ（PG_RETURN_BYTEA_P を使用）。

前提条件と不変条件

  • tsvector 内の語彙（lexeme）はソート済みである必要があります。
  • sizeof(WordEntry) が 4 バイトであることを前提としています。
  • TSVector 内の文字列はヌル終端ではないため、送信時に個別にヌル終端を付与する必要があります。

アルゴリズムまたは振る舞い

1. pq_begintypsend を呼び出し、バイナリバッファを初期化します。 2. tsvector の要素数（vec->size）を書き込みます。 3. 各 WordEntry を順次走査します。

  • 語彙文字列を pq_sendtext で書き込み、続いてヌル終端文字を pq_sendbyte で追加します。
  • POSDATALEN マクロを用いて位置情報の長さを取得し、書き込みます。
  • 位置情報が存在する場合、WordEntryPos 配列を走査して各位置情報を pq_sendint16 で書き込みます。

    • 4. pq_endtypsend を呼び出し、構築されたバッファを bytea として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されます（直接的な C 言語レベルの呼び出し元は提供されていません）。
  • **呼び出し先**: libpq/pqformat.h で定義される pq_begintypsend, pq_sendint32, pq_sendtext, pq_sendbyte, pq_sendint16, pq_endtypsend を利用しています。

関連データ構造

  • TSVector: 全文検索用データ型の内部表現。
  • WordEntry: 語彙のメタデータ（位置、長さ、オフセット）を保持する構造体。
  • WordEntryPos: 位置情報と重みを保持する 16 ビットのデータ型。
  • StringInfoData: バイナリ転送用のバッファ構造体。

失敗モードと境界条件

  • npos（位置情報の数）が 0 の場合、位置情報の書き込みはスキップされます。
  • WordEntry のサイズに関する前提（4 バイト）が崩れた場合、データ構造の読み取りに失敗する可能性があります。

保守担当者向けメモ

  • 本関数は tsvectorrecv と対になる処理です。バイナリ形式を変更する場合は、必ず両方の関数を更新してください。
  • WordEntry の構造や WordEntryPos のビット操作に変更を加える際は、src/include/tsearch/ts_type.h の定義と整合性を確認してください。
