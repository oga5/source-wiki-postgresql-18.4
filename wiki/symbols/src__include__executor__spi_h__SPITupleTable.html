SPITupleTable

定義

  • 種別: struct
  • 定義箇所: src/include/executor/spi.h:22

目的

Server Programming Interface (SPI) を介して実行された SQL コマンドの結果（タプル群）と、それに関連するメタデータを保持するために使用されます。

シグネチャまたは形状

typedef struct SPITupleTable
{
    TupleDesc    tupdesc;       /* タプル記述子 */
    HeapTuple   *vals;          /* タプル配列 */
    uint64       numvals;       /* 有効なタプル数 */
    uint64       alloced;       /* vals 配列の割り当て済みサイズ */
    MemoryContext tuptabcxt;    /* 結果テーブルのメモリコンテキスト */
    slist_node   next;          /* 内部管理用リンク */
    SubTransactionId subid;     /* 作成されたサブトランザクション ID */
} SPITupleTable;

入力と出力

  • 入力: SPI 実行エンジンによって生成されるタプルデータおよびメタデータ。
  • 出力: 呼び出し元（PL 言語ハンドラや拡張機能など）に対し、vals 配列を介してタプルへのアクセスを提供します。

前提条件と不変条件

  • vals 配列は numvals 個の有効なタプルを保持します。
  • tuptabcxt は、この構造体および関連するタプルが割り当てられているメモリコンテキストを指します。
  • 構造体内の next フィールドは、SPI 内部のリスト管理に使用されます。

アルゴリズムまたは振る舞い

SPI 実行結果はグローバル変数 SPI_tuptable に格納されます。呼び出し元は、この構造体を通じてクエリ結果を走査します。SPI_freetuptable 関数を介して、関連するメモリコンテキストと共に構造体全体が解放される設計となっています。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • contrib/dblink/dblink.c
  • contrib/tablefunc/tablefunc.c
  • contrib/xml2/xpath.c
  • src/backend/executor/spi.c
  • 呼び出し先: 構造体内部のフィールド操作および slist 関連のユーティリティ関数（slist_container 等）が使用されます。

関連データ構造

  • TupleDesc: タプルの構造定義。
  • HeapTuple: 個別のタプルデータ。
  • MemoryContext: メモリ管理コンテキスト。
  • slist_node: 内部リスト管理用ノード。

失敗モードと境界条件

  • SPI 呼び出しのたびにグローバル変数 SPI_tuptable が上書きされるため、結果を保持する必要がある場合は、呼び出し元で速やかにコピーを作成する必要があります。
  • 適切な SPI_finish や SPI_freetuptable の呼び出しを怠ると、メモリリークが発生する可能性があります。

保守担当者向けメモ

  • 本構造体には公開メンバーと非公開メンバー（alloced, tuptabcxt, next, subid）が混在しています。外部コードは公開メンバーのみを操作すべきです。
  • subid は、タプルテーブルがどのサブトランザクションで作成されたかを追跡しており、トランザクション境界でのリソース管理に重要です。
