tuplesort_free

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplesort.c:896

目的

ソート操作に関連するリソースを解放し、一時ファイルの後始末を行う。

シグネチャまたは形状

static void tuplesort_free(Tuplesortstate *state)

入力と出力

  • **入力**: Tuplesortstate *state（解放対象のソート状態構造体）
  • **出力**: なし（副作用としてメモリおよび一時ファイルが解放される）

前提条件と不変条件

  • state は有効な Tuplesortstate 構造体を指している必要がある。
  • state->base.sortcontext が有効なメモリコンテキストとして存在している必要がある。

アルゴリズムまたは振る舞い

1. state->base.sortcontext にメモリコンテキストを切り替える。 2. ソートの種類（外部ソートかメモリ内ソートか）に応じて、使用されたディスクブロック数またはメモリ使用量を算出する。 3. state->tapeset が存在する場合、LogicalTapeSetClose を呼び出して一時ファイルを閉じる。 4. trace_sort が有効な場合、ソートの統計情報（使用リソース量、経過時間等）をログに出力する。 5. TRACE_POSTGRESQL_SORT_DONE プローブを起動する。 6. FREESTATE(state) マクロを実行し、状態構造体に関連する内部リソースを解放する。 7. MemoryContextReset を呼び出し、ソート専用のメモリコンテキストをリセットして全作業メモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/tuplesort.c 内の複数箇所（tuplesort_reset 等の終了処理フロー）。
  • **呼び出し先**: LogicalTapeSetClose, LogicalTapeSetBlocks, MemoryContextSwitchTo, MemoryContextReset, pg_rusage_show, elog。

関連データ構造

  • Tuplesortstate: ソートのメイン状態を保持する構造体。
  • LogicalTapeSet: 外部ソート時に使用される一時ファイルの集合。

失敗モードと境界条件

  • state が NULL の場合、ポインタ参照エラーが発生する可能性がある（ソースコード上でのガードは明示されていない）。
  • メモリコンテキストが既に破棄されている場合、MemoryContextSwitchTo 等で不正なアクセスが発生する可能性がある。

保守担当者向けメモ

  • 本関数は MemoryContextReset を用いてメモリを解放するため、呼び出し後に state ポインタ自体を再利用することはできない。
  • LogicalTapeSet の個別のテープファイルは、sortcontext の破棄に伴い自動的にクリーンアップされる設計となっている。
