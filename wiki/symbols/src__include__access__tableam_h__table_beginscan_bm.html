table_beginscan_bm

定義

  • 種別: function
  • 定義箇所: src/include/access/tableam.h:919

目的

ビットマップスキャンを実行するためのテーブルスキャン記述子（TableScanDesc）を初期化・作成します。

シグネチャまたは形状

static inline TableScanDesc
table_beginscan_bm(Relation rel, Snapshot snapshot,
                   int nkeys, struct ScanKeyData *key)

入力と出力

  • **入力**:
  • rel: スキャン対象のリレーション。
  • snapshot: スキャンに使用するMVCCスナップショット。
  • nkeys: スキャンキーの数。
  • key: スキャンキーの配列。
  • **出力**:
  • 初期化された TableScanDesc 構造体へのポインタ。

前提条件と不変条件

  • rel は有効なリレーション構造体である必要があります。
  • rel->rd_tableam が適切に初期化されており、scan_begin 関数ポインタが実装されている必要があります。

アルゴリズムまたは振る舞い

1. ビットマップスキャンを示すフラグ SO_TYPE_BITMAPSCAN およびページモードを許可する SO_ALLOW_PAGEMODE を設定します。 2. rel->rd_tableam->scan_begin を呼び出し、具体的なテーブルアクセス手法（AM）にスキャン開始処理を委譲します。 3. その際、第5引数（parallel_scan）には NULL を渡し、作成されたフラグを渡します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeBitmapHeapscan.c:109
  • **呼び出し先**:
  • rel->rd_tableam->scan_begin（具体的なAM実装に依存）

関連データ構造

  • TableScanDesc: スキャン状態を保持する構造体。
  • Relation: テーブルのメタデータとAMルーチンへの参照を保持。
  • ScanKeyData: スキャン条件を定義する構造体。
  • Snapshot: MVCCの可視性制御に使用。

失敗モードと境界条件

  • rel->rd_tableam が未定義または scan_begin が実装されていない場合、セグメンテーションフォールトが発生する可能性があります。
  • メモリ不足により TableScanDesc の割り当てに失敗する可能性があります。

保守担当者向けメモ

  • 本関数はビットマップスキャン専用のラッパーです。汎用的なスキャンには table_beginscan を使用してください。
  • table_beginscan_bm で開始されたスキャンは、終了時に table_endscan を呼び出す必要があります。
