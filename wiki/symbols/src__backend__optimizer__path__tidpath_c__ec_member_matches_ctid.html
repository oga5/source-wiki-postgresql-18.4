ec_member_matches_ctid

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/tidpath.c:480

目的

EquivalenceClass（等価クラス）のメンバーが、指定されたリレーションの ctid 列を参照しているかどうかを判定します。

シグネチャまたは形状

static bool ec_member_matches_ctid(PlannerInfo *root, RelOptInfo *rel,
                                   EquivalenceClass *ec, EquivalenceMember *em,
                                   void *arg);

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel: 対象となるリレーション情報。
  • ec: 評価対象の等価クラス。
  • em: 評価対象の等価メンバー。
  • arg: 未使用の汎用引数。
  • **出力**:
  • bool: メンバーが ctid を参照している場合は true、それ以外は false を返します。

前提条件と不変条件

  • em->em_expr が存在し、それが Var ノードであることを前提としています。
  • IsCTIDVar 関数を使用して、対象の Var が指定された rel の ctid 列であるかを検証します。

アルゴリズムまたは振る舞い

1. em->em_expr が Var ノードであるかを確認します。 2. IsCTIDVar を呼び出し、その Var が対象リレーションの ctid 属性を指しているか判定します。 3. 両条件を満たす場合に true を返し、それ以外の場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/tidpath.c 内の関数から参照されています（詳細はソースコードの文脈に依存）。
  • **呼び出し先**: IsCTIDVar を呼び出します。

関連データ構造

  • EquivalenceMember: 等価クラス内の式を保持する構造体。
  • RelOptInfo: リレーションの最適化情報を保持する構造体。
  • Var: 式ツリー内の変数ノード。

失敗モードと境界条件

  • em->em_expr が NULL の場合や、Var ノードではない場合は false を返します。

保守担当者向けメモ

  • 本関数は EquivalenceClass のメンバーを走査する際のコールバックとして使用されることを想定しています。
  • IsCTIDVar の判定ロジックに変更があった場合、本関数の挙動も影響を受けます。
