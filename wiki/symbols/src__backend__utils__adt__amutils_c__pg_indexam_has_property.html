pg_indexam_has_property

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/amutils.c:408

目的

SQL インターフェースを通じて、指定されたインデックス・アクセスメソッド（AM）が特定のプロパティをサポートしているかどうかを判定します。

シグネチャまたは形状

Datum pg_indexam_has_property(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): 対象となるアクセスメソッドの OID (amoid)。
  • PG_GETARG_TEXT_PP(1): 照会するプロパティ名（文字列）。
  • **出力**:
  • Datum: プロパティの有無を示す真偽値（bool）。

前提条件と不変条件

  • amoid は pg_am カタログに存在する有効なアクセスメソッドを指している必要があります。
  • プロパティ名は IndexAMProperty 列挙型および am_propnames 配列で定義された既知の名称である必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP で受け取ったプロパティ名を text_to_cstring で C 文字列に変換します。 2. 内部関数 indexam_property を呼び出し、以下の引数を渡して処理を委譲します。

  • fcinfo: 関数呼び出し情報。
  • propname: 変換済みのプロパティ名。
  • amoid: 入力されたアクセスメソッド OID。
  • InvalidOid: インデックス OID は指定されないため無効値として渡されます。
  • 0: 列番号（attno）は指定されないため 0 が渡されます。

    • 3. indexam_property の戻り値をそのまま返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェースから直接呼び出されます。
  • **呼び出し先**: indexam_property（同ファイル内の内部関数）。

関連データ構造

  • IndexAmRoutine: アクセスメソッドの機能定義を保持する構造体。
  • IndexAMProperty: 照会可能なプロパティを定義する列挙型。

失敗モードと境界条件

  • 不正なプロパティ名が指定された場合、indexam_property 内でエラーが発生する可能性があります。
  • 存在しない amoid が指定された場合、GetIndexAmRoutineByAmId（indexam_property 内部で呼び出される）がエラーを発生させます。

保守担当者向けメモ

  • 本関数は SQL レベルのラッパーであり、実ロジックは indexam_property に集約されています。
  • 新しいプロパティを追加する場合は、IndexAMProperty 列挙型と am_propnames 配列の更新が必要です。
