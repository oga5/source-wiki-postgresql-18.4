plpgsql_stmt_typename

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_funcs.c:231

目的

PL/pgSQL ステートメントの内部型（cmd_type）に対応する、人間が読み取り可能な名称（文字列）を返します。主にエラーメッセージやデバッグ出力に使用されます。

シグネチャまたは形状

const char *plpgsql_stmt_typename(PLpgSQL_stmt *stmt)

入力と出力

  • **入力**: PLpgSQL_stmt *stmt - 型名を判定したいステートメント構造体へのポインタ。
  • **出力**: const char * - ステートメントの種類を表す文字列。

前提条件と不変条件

  • 引数 stmt は有効な PLpgSQL_stmt 構造体を指している必要があります。
  • 返される文字列は静的メモリ領域または翻訳済み文字列リテラルであり、呼び出し側で解放する必要はありません。

アルゴリズムまたは振る舞い

1. stmt->cmd_type をスイッチ文で評価します。 2. 各 PLPGSQL_STMT_* 定数に対し、対応する名称を返します。 3. 一部のステートメント（EXIT, GETDIAG, FETCH, CALL など）は、構造体内のフラグ（is_exit, is_stacked, is_move, is_call）を参照して、より詳細な型名を動的に決定します。 4. 既知の型に一致しない場合は "unknown" を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/pl_exec.c:1297（エラー発生時のステートメント特定に使用）。
  • **呼び出し先**: 本関数は直接的な関数呼び出しを行わず、stmt 構造体のメンバを参照します。

関連データ構造

  • PLpgSQL_stmt: ステートメントの基底構造体。
  • PLpgSQL_stmt_exit, PLpgSQL_stmt_getdiag, PLpgSQL_stmt_fetch, PLpgSQL_stmt_call: 各種ステートメントの派生構造体。

失敗モードと境界条件

  • stmt が NULL の場合、構造体メンバへのアクセスによりセグメンテーション違反が発生する可能性があります。
  • 未定義の cmd_type が渡された場合、デフォルト値 "unknown" を返します。

保守担当者向けメモ

  • 新しいステートメント型を plpgsql.h に追加した際は、本関数のスイッチ文にも必ずケースを追加してください。
  • 多言語対応のため、一部の戻り値には _() マクロが適用されています。翻訳対象の文字列を変更する際は注意してください。
