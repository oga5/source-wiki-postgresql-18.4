PQregisterEventProc

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/libpq-events.c:39

目的

PGconn オブジェクトに関連付けられたイベントプロシージャ（コールバック）を登録し、ライフサイクルイベントの通知を開始します。

シグネチャまたは形状

int PQregisterEventProc(PGconn *conn, PGEventProc proc, const char *name, void *passThrough);

入力と出力

  • **入力**:
  • conn: 対象の PGconn オブジェクト。
  • proc: 登録するイベント処理関数。
  • name: イベントプロシージャを識別するための名称（内部でコピーされます）。
  • passThrough: イベント発生時にプロシージャへ渡される任意のデータポインタ。
  • **出力**:
  • 成功時は true (1)、失敗時は false (0) を返します。

前提条件と不変条件

  • proc、conn、name がすべて有効（NULL でない）である必要があります。
  • 同一の proc アドレスを持つ関数を同一の conn に対して二重登録することはできません。
  • name は空文字列であってはなりません。

アルゴリズムまたは振る舞い

1. 引数の妥当性を検証し、既に登録済みでないか確認します。 2. conn 内のイベント配列に空きがない場合、配列サイズを拡張（倍増）します。 3. 新しいイベントエントリを配列に追加し、name を strdup で複製して保持します。 4. PGEVT_REGISTER イベントを引数 passThrough と共に即座に呼び出します。 5. コールバックが失敗（0 を返却）した場合は、登録をロールバックし、確保したメモリを解放して false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の明示的な呼び出し元情報は提供されていません。
  • **呼び出し先**: 引数として渡された proc（PGEventProc 型の関数ポインタ）を直接呼び出します。

関連データ構造

  • PGconn: 接続状態を保持する構造体。events 配列を管理します。
  • PGEvent: イベントプロシージャとそのメタデータを保持する内部構造体。
  • PGEventRegister: PGEVT_REGISTER イベント時に渡される構造体。

失敗モードと境界条件

  • 引数が無効な場合、またはメモリ確保（malloc/realloc/strdup）に失敗した場合は false を返します。
  • 既に登録済みのプロシージャを登録しようとした場合は false を返します。
  • proc が PGEVT_REGISTER イベント内でエラーを返した場合、登録は取り消され false を返します。

保守担当者向けメモ

  • イベントの識別には関数ポインタのアドレスが使用されるため、同一プロシージャの複数回登録は不可です。
  • name 引数は内部で strdup されるため、呼び出し側で元の文字列を保持し続ける必要はありません。
  • 拡張ライブラリがリソース管理を行うための基盤であり、接続や結果セットの生存期間に応じたクリーンアップ処理に利用されます。
