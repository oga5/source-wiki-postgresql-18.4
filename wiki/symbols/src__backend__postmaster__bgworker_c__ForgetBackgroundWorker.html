ForgetBackgroundWorker

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/bgworker.c:428

目的

登録済みのバックグラウンドワーカーをシステムから削除し、関連する共有メモリ上のスロットを解放してメモリをクリーンアップします。

シグネチャまたは形状

void ForgetBackgroundWorker(RegisteredBgWorker *rw)

入力と出力

  • **入力**: rw (RegisteredBgWorker *) - 削除対象のバックグラウンドワーカー構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • rw->rw_shmem_slot が max_worker_processes 未満であること。
  • 対応する共有メモリ上の BackgroundWorkerSlot が in_use 状態であること。

アルゴリズムまたは振る舞い

1. rw が指す共有メモリ上のスロットを取得します。 2. ワーカーが BGWORKER_CLASS_PARALLEL フラグを持つ場合、共有メモリ上の parallel_terminate_count をインクリメントします。 3. メモリバリア (pg_memory_barrier) を発行し、終了処理の完了を保証した上で、スロットの in_use フラグを false に設定します。 4. デバッグログを出力します。 5. BackgroundWorkerList から当該ワーカーのノードを削除し、rw 構造体自体を pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/bgworker.c 内の複数箇所
  • src/backend/postmaster/postmaster.c 内の複数箇所
  • **呼び出し先**:
  • dlist_delete
  • pfree
  • ereport
  • pg_memory_barrier

関連データ構造

  • RegisteredBgWorker: ワーカーのローカル管理用構造体。
  • BackgroundWorkerSlot: 共有メモリ上のワーカー状態管理スロット。
  • BackgroundWorkerData: 共有メモリ上のワーカー管理データ全体。

失敗モードと境界条件

  • rw が無効なポインタである場合、または rw_shmem_slot が範囲外である場合はアサーション違反となります。
  • slot->in_use が false の状態で呼び出された場合、アサーション違反となります。

保守担当者向けメモ

  • 本関数は共有メモリの更新を伴うため、メモリバリアの順序が重要です。parallel_terminate_count の更新と in_use のクリア順序を変更しないでください。
  • pfree を実行するため、本関数を呼び出した後は rw ポインタへのアクセスは禁止されます。
