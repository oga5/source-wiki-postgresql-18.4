pgTypeNameCompare

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dump_sort.c:483

目的

2つのデータ型（OID）を比較し、ダンプ順序を決定するためのソートキーを生成する。

シグネチャまたは形状

int pgTypeNameCompare(Oid typid1, Oid typid2)

入力と出力

  • **入力**: 比較対象となる2つの Oid (typid1, typid2)。
  • **出力**: 比較結果を示す整数。0は等価、負の値は typid1 が先、正の値は typid2 が先であることを示す。

前提条件と不変条件

  • findTypeByOid により、入力されたOIDに対応する TypeInfo が取得可能であること。
  • 比較対象の型が属する名前空間（スキーマ）名は有効であること。

アルゴリズムまたは振る舞い

1. OIDが同一であれば0を返す。 2. findTypeByOid を用いて各OIDに対応する TypeInfo を検索する。 3. 検索に失敗した場合（カタログ破損とみなす）、Assert(false) を実行し、0を返して比較を継続させる。 4. 両方の型が名前空間を持つ場合、まず名前空間名（namespace->dobj.name）で strcmp を行い、結果が0でなければそれを返す。 5. 名前空間が同一（または存在しない）場合、型名（dobj.name）で strcmp を行い、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump_sort.c 内の関数（関数引数の型比較や演算子の左辺・右辺の型比較に使用）。
  • **呼び出し先**:
  • findTypeByOid（型情報取得のため）。
  • strcmp（文字列比較のため）。

関連データ構造

  • TypeInfo: 型情報を保持する構造体。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • **カタログ破損**: findTypeByOid が型情報を返さない場合、Assert により異常終了（デバッグビルド時）または0を返して比較を続行する。
  • **名前空間の欠如**: 型が名前空間を持たない場合、カタログ破損とみなして Assert を実行する。

保守担当者向けメモ

  • 本関数はソートの安定性を担保するために重要である。修正時は DOTypeNameCompare との整合性に注意すること。
  • 比較ロジックは「スキーマ名」→「型名」の順序で行われるため、スキーマ間での型名の重複はスキーマ名によって分離される。
