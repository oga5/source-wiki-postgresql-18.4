SendRowDescriptionMessage

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/common/printtup.c:165

目的

クエリ結果のメタデータ（カラム定義）をフロントエンドへ通知するための RowDescription メッセージを構築・送信する。

シグネチャまたは形状

void SendRowDescriptionMessage(StringInfo buf, TupleDesc typeinfo, List *targetlist, int16 *formats)

入力と出力

  • **入力**:
  • buf: メッセージ構築用の StringInfo バッファ。
  • typeinfo: 送信するタプルの記述子 (TupleDesc)。
  • targetlist: カラムのソース情報を含む TargetEntry のリスト。
  • formats: 各カラムのフォーマットコード（テキスト/バイナリ）の配列（NULL可）。
  • **出力**: なし（副作用として buf にメッセージが書き込まれ、送信される）。

前提条件と不変条件

  • typeinfo は有効なタプル記述子であること。
  • buf は pq_beginmessage_reuse で初期化可能な状態であること。
  • カラムがドメイン型の場合、getBaseTypeAndTypmod により基底型へ解決される。

アルゴリズムまたは振る舞い

1. PqMsg_RowDescription メッセージを開始し、カラム数 (natts) を送信する。 2. メッセージ全体のサイズを事前に計算し、enlargeStringInfo でバッファを拡張して再割り当てを抑制する。 3. 各カラムについて以下の処理を繰り返す:

  • getBaseTypeAndTypmod を呼び出し、ドメイン型を基底型に変換する。
  • targetlist から対応する TargetEntry を検索し、テーブル名やカラム番号の情報を取得する（見つからない場合は 0 を使用）。
  • formats 配列からフォーマットコードを取得する。
  • カラム名、テーブルOID、カラム番号、型OID、型長、型修飾子、フォーマットコードを順次バッファへ書き込む。

    • 4. pq_endmessage_reuse を呼び出し、メッセージ送信を完了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/printtup.c (printtup_startup)
  • src/backend/tcop/postgres.c (exec_describe_statement_message 等の文脈)
  • **呼び出し先**:
  • pq_beginmessage_reuse, pq_sendint16, pq_writestring, pq_writeint32, pq_writeint16, pq_endmessage_reuse (libpq プロトコル関数)
  • getBaseTypeAndTypmod (型解決)
  • enlargeStringInfo (バッファ管理)

関連データ構造

  • StringInfo: メッセージ構築用バッファ。
  • TupleDesc: タプルの構造定義。
  • TargetEntry: クエリのターゲットリスト項目。

失敗モードと境界条件

  • targetlist が空または不一致の場合、resorigtbl および resorigcol には 0 が設定される。
  • formats が NULL の場合、すべてのカラムでフォーマット 0（テキスト）が使用される。
  • メモリ不足時は enlargeStringInfo 経由でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はカタログアクセスを伴うため、呼び出しタイミングには注意が必要である。
  • NAMEDATALEN を用いたサイズ見積もりは、文字セットのオーバーヘッドを考慮して過大に見積もられている。
