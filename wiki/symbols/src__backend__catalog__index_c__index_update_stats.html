index_update_stats

定義

  • **種別**: function
  • **定義箇所**: src/backend/catalog/index.c:2808

目的

CREATE INDEX または REINDEX 実行後、システムカタログ pg_class 内の対象リレーション（インデックスまたはテーブル）の統計情報およびフラグを更新します。

シグネチャまたは形状

static void index_update_stats(Relation rel, bool hasindex, double reltuples)

入力と出力

  • **入力**:
  • rel: 更新対象のリレーション（Relation）。
  • hasindex: pg_class.relhasindex に設定する真偽値。
  • reltuples: 更新後のタプル数（負の値は更新対象外）。
  • **出力**: なし（副作用として pg_class を直接更新）。

前提条件と不変条件

  • pg_class に対する RowExclusiveLock を保持していること。
  • reltuples が 0 でかつ既存の reltuples が負の場合、reltuples は -1 として扱われます。
  • バイナリアップグレード中は統計情報の更新をスキップします。
  • オートバキュームが無効な場合、統計情報の更新をスキップする場合があります。

アルゴリズムまたは振る舞い

1. **更新判定**: reltuples の値やオートバキューム設定に基づき、統計情報を更新するかを決定します。 2. **物理情報取得**: 更新が必要な場合、RelationGetNumberOfBlocks でページ数を取得し、インデックス以外であれば visibilitymap_count で可視・凍結ページ数を計算します。 3. **インプレース更新**: systable_inplace_update_begin を使用し、トランザクションを介さない非トランザクション的な更新を行います。 4. **値の適用**: relhasindex、relpages、reltuples、relallvisible、relallfrozen を比較し、変更がある場合のみ systable_inplace_update_finish で書き出します。 5. **キャッシュ無効化**: 変更の有無に関わらず、必要に応じて CacheInvalidateRelcacheByTuple を呼び出し、リレーションキャッシュの整合性を維持します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/index.c 内の index_create や index_build 等のインデックス構築・再構築処理から呼び出されます。
  • **呼び出し先**:
  • table_open, table_close
  • systable_inplace_update_begin, systable_inplace_update_finish, systable_inplace_update_cancel
  • visibilitymap_count
  • CacheInvalidateRelcacheByTuple
  • heap_freetuple

関連データ構造

  • Relation: 操作対象のリレーション情報。
  • Form_pg_class: pg_class システムカタログのタプル構造。
  • ScanKeyData: pg_class 検索用のキー。

失敗モードと境界条件

  • **タプル未検出**: 指定された relid に対応する pg_class タプルが見つからない場合、elog(ERROR) を発生させます。
  • **パーティションインデックス**: RELKIND_PARTITIONED_INDEX に対してはアサーション失敗により停止します。
  • **並行更新**: 他の CREATE INDEX と競合する場合でも、非トランザクション更新によりデッドロックや更新失敗を回避する設計となっています。

保守担当者向けメモ

  • 本関数は pg_class を直接書き換えるため、非常に慎重な設計が求められます。
  • systable_inplace_update を使用しているため、トランザクションのロールバック後も更新が反映されたままになる点に注意してください。
  • relpages や relallvisible の更新は、reltuples が提供されている場合に限定されます。
