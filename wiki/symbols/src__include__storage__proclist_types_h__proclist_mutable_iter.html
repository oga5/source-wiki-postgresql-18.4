proclist_mutable_iter

定義

  • 種別: struct
  • 定義箇所: src/include/storage/proclist_types.h:47

目的

プロセス番号（ProcNumber）で構成される双方向連結リストを走査する際、走査中に現在の要素が削除されても、次の要素を安全に参照し続けるためのイテレータ構造体です。

シグネチャまたは形状

typedef struct proclist_mutable_iter
{
	ProcNumber	cur;			/* pgprocno of the current PGPROC */
	ProcNumber	next;			/* pgprocno of the next PGPROC */
} proclist_mutable_iter;

入力と出力

  • **入力**: リスト走査の初期化時に、リストの先頭情報などが渡されます（操作関数は proclist.h で定義）。
  • **出力**: 走査の各ステップで cur に現在の要素、next に次の要素が保持されます。

前提条件と不変条件

  • ProcNumber 型（storage/procnumber.h 由来）のインデックスを使用します。
  • リストの走査中に要素が削除された場合でも、next を保持することで走査を継続できる設計となっています。

アルゴリズムまたは振る舞い

  • リストの走査中に、現在のノードがリストから削除されることを許容します。
  • cur が現在の処理対象を指し、next が削除発生時に備えてあらかじめ次のノードを保持する役割を果たします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/lmgr/lwlock.c 内の複数の箇所で、リスト走査用変数として宣言されています。
  • **呼び出し先**: src/include/storage/proclist.h:207 のマクロ（AssertVariableIsOfTypeMacro）により、型チェックの対象として参照されています。

関連データ構造

  • proclist_node: リストの各要素。
  • proclist_head: リストの先頭と末尾を管理するヘッダー。
  • ProcNumber: リストのノードを識別するためのプロセス番号。

失敗モードと境界条件

  • リストが空の場合、INVALID_PROC_NUMBER が適切に扱われる必要があります。
  • 走査中にリストの整合性が崩れた場合（循環参照など）、無限ループや不正なメモリ参照のリスクがあります。

保守担当者向けメモ

  • 本構造体は proclist.h で提供される API を介して操作することが必須です。構造体メンバを直接操作するとリストの整合性が損なわれる可能性があります。
  • 共有メモリ内での配置を考慮し、ポインタではなく整数インデックス（ProcNumber）を使用している点に注意してください。
