do_object_end

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_json_parser/test_json_parser_incremental.c:260

目的

JSON オブジェクトの終了イベントを処理し、パース結果を標準出力へ再構成（エコーバック）するコールバック関数です。

シグネチャまたは形状

static JsonParseErrorType do_object_end(void *state)

入力と出力

  • **入力**: void *state - DoState 構造体へのポインタ。
  • **出力**: JsonParseErrorType 型の戻り値（ソースコード上では明示的な return 文が省略されていますが、関数定義のシグネチャに従います）。

前提条件と不変条件

  • 引数 state は DoState 構造体としてキャスト可能である必要があります。

アルゴリズムまたは振る舞い

1. 引数 state を DoState 型のポインタ _state にキャストします。 2. 標準出力に対して、JSON オブジェクトの終了を示す \n}\n を出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: JsonSemAction 構造体のメンバ .object_end に関数ポインタとして登録されており、pg_parse_json_incremental を通じてパース処理中に呼び出されます。
  • **呼び出し先**: 標準ライブラリの printf を呼び出します。

関連データ構造

  • DoState: パース中の状態を保持する構造体。
  • JsonSemAction: パースイベントを処理するコールバック関数のテーブル。

失敗モードと境界条件

  • ソースコードの抜粋範囲内では、エラーハンドリングや戻り値の生成に関するロジックは記述されていません。

保守担当者向けメモ

  • 本関数は JsonSemAction のコールバックとして機能します。戻り値の型が JsonParseErrorType であるため、将来的にエラーチェックを追加する場合は適切な列挙値を返す必要があります。
