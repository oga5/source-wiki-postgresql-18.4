ECPGdo

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/execute.c:2274

目的

ECPG（Embedded SQL for C）アプリケーションから発行される SQL 文を実行するための主要なエントリーポイントです。

シグネチャまたは形状

bool ECPGdo(const int lineno, const int compat, const int force_indicator, const char *connection_name, const bool questionmarks, const int st, const char *query, ...)

入力と出力

  • **入力**:
  • lineno: 呼び出し元のソースコード行番号。
  • compat: 互換性モード。
  • force_indicator: インジケータ変数の強制使用フラグ。
  • connection_name: 接続名（NULL の場合はデフォルト接続）。
  • questionmarks: ? をプレースホルダーとして許可するかどうか。
  • st: ステートメントの種類。
  • query: 実行する SQL 文の文字列。
  • ...: 可変長引数として、SQL 文に関連付けられたホスト変数のリスト。
  • **出力**:
  • bool: 実行が成功した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • 可変長引数は va_start で初期化され、va_end で終了される必要があります。
  • 内部で ecpg_do を呼び出すための引数スタックを構築します。

アルゴリズムまたは振る舞い

1. va_start を使用して可変長引数リストを取得します。 2. 取得した引数を ecpg_do 関数へ転送し、実際の SQL 実行処理（パラメータ構築、クエリ送信、結果格納）を委譲します。 3. va_end を呼び出して引数リストを解放します。 4. ecpg_do の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPG プリプロセッサが生成した C コード、および ECPG のテストスイート（例: compat_informix-describe.c など）。
  • **呼び出し先**: ecpg_do（同一ファイル内の内部関数）。

関連データ構造

  • va_list: 可変長引数を処理するための標準 C ライブラリ型。

失敗モードと境界条件

  • 内部で呼び出される ecpg_do が失敗した場合、本関数も false を返します。
  • 接続が確立されていない場合や、SQL 文の解析・実行中にエラーが発生した場合に失敗します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際のロジックは ecpg_do に存在します。
  • プリプロセッサが生成するコードのインターフェースであるため、シグネチャの変更には注意が必要です。
