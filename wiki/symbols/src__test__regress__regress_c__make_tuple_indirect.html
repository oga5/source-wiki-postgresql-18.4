make_tuple_indirect

定義

  • 種別: function
  • 定義箇所: src/test/regress/regress.c:334

目的

入力されたタプル内の可変長属性（varlena）を強制的に間接参照（TOAST indirect）形式に変換し、PostgreSQL の内部データ構造の検証を行うためのテスト用関数です。

シグネチャまたは形状

Datum make_tuple_indirect(PG_FUNCTION_ARGS);

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して HeapTupleHeader 型のタプルを受け取ります。
  • **出力**: 変換後のタプルデータ（HeapTupleHeader）を Datum として返します。

前提条件と不変条件

  • 入力タプルは有効な HeapTupleHeader である必要があります。
  • 変換対象は、attisdropped ではなく、かつ NULL でない可変長属性（attlen == -1）であり、かつ TYPSTORAGE_PLAIN ではない属性に限られます。
  • 既に間接参照されている属性はスキップされます。

アルゴリズムまたは振る舞い

1. 入力タプルの型情報から TupleDesc を取得し、heap_deform_tuple で各属性を分解します。 2. TopTransactionContext にメモリコンテキストを切り替えます。 3. 各属性を走査し、条件を満たす可変長属性をメモリ上にコピー（または detoast_external_attr で取得）します。 4. struct varatt_indirect を構築し、SET_VARTAG_EXTERNAL マクロを用いて VARTAG_INDIRECT 形式のデータに書き換えます。 5. heap_form_tuple を用いて新しいタプルを再構築します。 6. 構築したタプルの t_data を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 により登録）。
  • **呼び出し先**: lookup_rowtype_tupdesc, heap_deform_tuple, MemoryContextSwitchTo, detoast_external_attr, heap_form_tuple, ReleaseTupleDesc 等の PostgreSQL 内部 API。

関連データ構造

  • HeapTupleHeader: 入力および出力のタプル構造。
  • varatt_indirect: 間接参照ポインタを保持する構造体。
  • TupleDesc: タプルの属性定義情報。

失敗モードと境界条件

  • 変換結果を PG_RETURN_HEAPTUPLEHEADER で返すと、TOAST ポインタが即座にフラット化されるため、意図的に PG_RETURN_POINTER を使用して生の t_data を返却しています。
  • この関数が返すタプルをコンポジット型（レコードや配列など）に挿入すると、PostgreSQL の制約に違反する可能性があるため、テストスクリプト内での利用範囲には注意が必要です。

保守担当者向けメモ

  • 本関数は回帰テスト専用です。本番環境のコードでは使用しないでください。
  • 内部データ構造を意図的に操作しているため、PostgreSQL の TOAST 処理の学習用サンプルとして適しています。
