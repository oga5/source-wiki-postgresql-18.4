replace_guc_value

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/initdb.c:527

目的

設定ファイルの内容（文字列配列）に含まれる特定のGUC（Grand Unified Configuration）パラメータの値を置換、または新規追加する。

シグネチャまたは形状

static char **replace_guc_value(char **lines, const char *guc_name, const char *guc_value, bool mark_as_comment)

入力と出力

  • **入力**:
  • lines: 設定ファイルの各行を保持するNULL終端の文字列配列。
  • guc_name: 置換対象のパラメータ名。
  • guc_value: 設定する新しい値。
  • mark_as_comment: true の場合、パラメータをコメントアウト（#を付与）して設定する。
  • **出力**: 更新された文字列配列へのポインタ。

前提条件と不変条件

  • lines はNULL終端の文字列配列であること。
  • guc_value が引用符を必要とする場合、escape_quotes を用いて適切にエスケープされる。
  • 既存の行を置換する場合、元の行のコメント（#以降）を可能な限り維持する。

アルゴリズムまたは振る舞い

1. 新しい設定行を PQExpBuffer を用いて構築する。 2. lines 配列を走査し、guc_name に一致する行を探す。 3. 一致した場合、その行を新しい値で置き換え、元の行に存在したコメントをインデントを維持した状態で新しい行に付与する。 4. 一致する行が見つからない場合、配列の末尾に新しい設定行を追加し、配列を再確保（pg_realloc_array）する。 5. 処理後の文字列配列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/initdb/initdb.c 内の main 関数周辺（max_connections, shared_buffers, timezone 等の設定時）。
  • **呼び出し先**: createPQExpBuffer, guc_value_requires_quotes, escape_quotes, pg_strncasecmp, pg_realloc_array 等。

関連データ構造

  • PQExpBuffer: 文字列構築用バッファ。
  • char **: 設定ファイルの各行を保持する配列。

失敗モードと境界条件

  • guc_name が見つからない場合、配列の末尾に新規追加される。
  • guc_value にタブが含まれる場合、インデント調整処理では考慮されない。
  • lines 配列のメモリ再確保に失敗した場合、プログラムは終了する可能性がある（pg_realloc_array の挙動に依存）。

保守担当者向けメモ

  • 本関数は postgresql.conf 等のテンプレートファイルを動的に書き換えるために使用される。
  • 既存の行を置換する際、# で始まる行や空白文字を許容する柔軟な検索ロジックとなっている。
  • newline->data は free されずに呼び出し元へ引き渡されるため、メモリリークを防ぐ設計となっている。
