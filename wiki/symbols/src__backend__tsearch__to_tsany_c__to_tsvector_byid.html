to_tsvector_byid

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/to_tsany.c:242

目的

指定された全文検索設定（Config OID）を使用して、入力テキストを tsvector 型へ変換する。

シグネチャまたは形状

Datum to_tsvector_byid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): 全文検索設定の OID。
  • PG_GETARG_TEXT_PP(1): 変換対象のテキストデータ。
  • **出力**:
  • PG_RETURN_TSVECTOR(out): 変換された tsvector 構造体。

前提条件と不変条件

  • 入力テキストのサイズに基づき、ParsedText 用のメモリを動的に割り当てる。
  • MaxAllocSize を超えないよう、単語数の見積もりに上限を設けている。

アルゴリズムまたは振る舞い

1. 入力テキストのサイズから単語数を推定し、ParsedText 構造体を初期化する。 2. parsetext を呼び出し、指定された設定に基づいてテキストを解析・正規化し、ParsedText に格納する。 3. 入力テキストのメモリを解放する。 4. make_tsvector を呼び出し、解析結果から最終的な tsvector バイナリを構築する。 5. 構築した tsvector を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • SQL インターフェース経由での呼び出し。
  • src/backend/tsearch/to_tsany.c 内で DirectFunctionCall2 を用いた呼び出し（証拠あり）。
  • **呼び出し先**:
  • parsetext (解析処理)
  • make_tsvector (ベクトル構築)
  • palloc (メモリ確保)

関連データ構造

  • ParsedText: 解析中の一時的な単語列を保持する構造体。
  • TSVector: 最終的な検索用データ構造。
  • ParsedWord: ParsedText 内の個々の単語情報を保持する構造体。

失敗モードと境界条件

  • 入力テキストが空、または単語数が極端に少ない場合、ParsedText の初期化サイズを最小 2 に調整する。
  • メモリ確保時、MaxAllocSize を超える場合は上限値で切り詰める。

保守担当者向けメモ

  • 本関数は tsvector の内部バイナリレイアウトを直接扱うため、メモリ管理とアライメントには注意が必要である。
  • parsetext による解析結果は make_tsvector に依存しており、両者のインターフェース変更時は整合性に留意すること。
