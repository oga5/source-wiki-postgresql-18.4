ExecutorEnd

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execMain.c:465

目的

クエリ実行の最終段階として、実行環境のリソース解放およびクリーンアップ処理を実行する。

シグネチャまたは形状

void ExecutorEnd(QueryDesc *queryDesc)

入力と出力

  • **入力**: QueryDesc *queryDesc（実行対象のクエリ記述子）
  • **出力**: なし（void）

前提条件と不変条件

  • ExecutorStart によって初期化された有効な QueryDesc が渡される必要がある。
  • クエリ実行のライフサイクルにおける最終フェーズとして呼び出される。

アルゴリズムまたは振る舞い

1. ExecutorEnd_hook が設定されている場合、それを呼び出してクリーンアップを委譲する。 2. フックが設定されていない場合、標準の standard_ExecutorEnd を呼び出し、リレーションのクローズやメモリコンテキストの破棄などの標準的な終了処理を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyto.c
  • src/backend/commands/createas.c
  • src/backend/commands/explain.c
  • src/backend/commands/extension.c
  • src/backend/commands/matview.c
  • contrib/auto_explain/auto_explain.c
  • contrib/pg_stat_statements/pg_stat_statements.c
  • **呼び出し先**:
  • ExecutorEnd_hook（設定されている場合）
  • standard_ExecutorEnd（デフォルト）

関連データ構造

  • QueryDesc: クエリ実行に必要なプランツリーや実行状態（EState）を保持する構造体。

失敗モードと境界条件

  • queryDesc が NULL の場合の挙動については、本ソース抜粋からは判断不可。
  • フック関数内でエラーが発生した場合、そのエラーが呼び出し元に伝播する。

保守担当者向けメモ

  • 本関数は実行エンジンの終了処理のゲートウェイである。拡張機能が実行終了時に独自の統計収集やログ出力を行う場合は、ExecutorEnd_hook を利用する。
  • standard_ExecutorEnd の詳細な内部処理については、execMain.c 内の当該関数の定義を参照すること。
