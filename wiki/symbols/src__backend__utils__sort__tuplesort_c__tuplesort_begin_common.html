tuplesort_begin_common

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/sort/tuplesort.c:641

目的

ソート操作に必要なメモリコンテキストの作成、Tuplesortstate 構造体の初期化、および並列ソート時の調整用パラメータの設定を行う。

シグネチャまたは形状

Tuplesortstate *tuplesort_begin_common(int workMem, SortCoordinate coordinate, int sortopt)

入力と出力

  • **入力**:
  • workMem: ソートに使用可能なメモリ量（KB単位）。最小値は64KBに強制される。
  • coordinate: 並列ソート時の調整情報。NULL の場合はシリアルソートとして動作。
  • sortopt: ソートの動作オプション（TUPLESORT_RANDOMACCESS 等）。
  • **出力**:
  • 初期化済みの Tuplesortstate 構造体へのポインタ。

前提条件と不変条件

  • coordinate が指定されている場合、TUPLESORT_RANDOMACCESS オプションは許可されない。
  • workMem は内部で Max(workMem, 64) * 1024 として allowedMem に格納される。
  • Tuplesortstate は maincontext 内に割り当てられ、明示的な pfree を不要としている。

アルゴリズムまたは振る舞い

1. **メモリコンテキスト作成**: maincontext（永続用）と sortcontext（操作用）を作成。 2. **状態初期化**: Tuplesortstate をゼロクリアで割り当て、allowedMem や初期配列サイズ（INITIAL_MEMTUPSIZE）を設定。 3. **バッチ初期化**: tuplesort_begin_batch(state) を呼び出し、バッチごとの状態を構築。 4. **並列処理設定**: coordinate の有無に基づき、シリアル、ワーカー、リーダーのいずれかの役割を state に反映。 5. **コンテキスト復帰**: 初期化前のメモリコンテキストへ戻し、作成した状態を返却。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/tuplesortvariants.c 内の各ソート開始関数（tuplesort_begin_heap 等）。
  • **呼び出し先**: AllocSetContextCreate, palloc0, pg_rusage_init, tuplesort_begin_batch, worker_get_identifier。

関連データ構造

  • Tuplesortstate: ソート操作のメイン状態を保持。
  • SortCoordinate: 並列ソート時のリーダー/ワーカー情報を保持。
  • MemoryContext: メモリ管理に使用。

失敗モードと境界条件

  • coordinate が存在し、かつ TUPLESORT_RANDOMACCESS が指定されている場合、elog(ERROR) により強制終了する。
  • workMem が64未満の場合、64KBとして扱われる。

保守担当者向けメモ

  • 本関数は tuplesort_reset を跨いで生存する maincontext を作成する。メモリリークを防ぐため、コンテキストの階層構造を維持すること。
  • 並列ソートのロジックは coordinate のフラグに依存しているため、並列処理の仕様変更時は coordinate のハンドリング箇所を重点的に確認すること。
