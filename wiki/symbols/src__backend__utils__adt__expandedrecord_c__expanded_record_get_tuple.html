expanded_record_get_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/expandedrecord.c:883

目的

展開されたレコード（ExpandedRecordHeader）から、物理的な HeapTuple 表現を取得します。

シグネチャまたは形状

HeapTuple expanded_record_get_tuple(ExpandedRecordHeader *erh)

入力と出力

  • **入力**: erh - 展開されたレコードのヘッダー構造体へのポインタ。
  • **出力**:
  • 成功時: レコードの内容を表す HeapTuple。
  • レコードが空の場合: NULL。

前提条件と不変条件

  • erh は有効な ExpandedRecordHeader を指している必要があります。
  • 内部状態に応じて、元のタプルを再利用するか、現在のフィールド値からタプルを再構築します。

アルゴリズムまたは振る舞い

1. **元のタプル保持時**: ER_FLAG_FVALUE_VALID フラグが立っている場合、保持されている元のタプル（erh->fvalue）をそのまま返します。 2. **フィールド分解時**: 上記に該当せず、ER_FLAG_DVALUES_VALID フラグが立っている場合、現在のフィールド値（erh->dvalues）とヌルビットマップ（erh->dnulls）から heap_form_tuple を用いて新しいタプルを生成して返します。 3. **空の場合**: どちらのフラグも立っていない場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpgsql/src/pl_exec.c (複数箇所でレコードのタプル化に使用)
  • **呼び出し先**:
  • heap_form_tuple (タプル再構築時)

関連データ構造

  • ExpandedRecordHeader: レコードのメタデータ、フラグ、フィールド値、タプル記述子を保持するメイン構造体。
  • HeapTuple: PostgreSQL の物理的なタプル表現。

失敗モードと境界条件

  • erh が NULL の場合の挙動は定義されていません（セグメンテーション違反の可能性があります）。
  • レコードが空（フィールド値も元のタプルも存在しない）の場合は NULL を返します。

保守担当者向けメモ

  • 本関数は遅延評価の仕組みの一部であり、必要に応じてタプルの再構築を行います。
  • ER_FLAG_FVALUE_VALID および ER_FLAG_DVALUES_VALID フラグの状態が、レコードの現在の有効な表現を決定します。
