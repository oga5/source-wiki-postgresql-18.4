astreamer_zstd_frame

定義

  • **種別**: struct
  • **定義箇所**: src/fe_utils/astreamer_zstd.c:29

目的

Zstandard (zstd) を用いたデータ圧縮および伸長処理において、ストリーミングパイプラインのコンテキストとバッファを保持するために使用される構造体です。

シグネチャまたは形状

typedef struct astreamer_zstd_frame
{
    astreamer       base;         /* 基底ストリーマー構造体 */
    ZSTD_CCtx      *cctx;         /* 圧縮コンテキスト */
    ZSTD_DCtx      *dctx;         /* 伸長コンテキスト */
    ZSTD_outBuffer  zstd_outBuf;  /* zstd 出力バッファ */
} astreamer_zstd_frame;

入力と出力

  • **入力**: 本構造体自体はデータ入力ではなく、astreamer インフラストラクチャを通じて渡されるデータチャンクを処理するための状態保持に使用されます。
  • **出力**: 圧縮または伸長されたデータが zstd_outBuf を経由して後続のストリーマーへ転送されます。

前提条件と不変条件

  • USE_ZSTD が定義された環境でのみ有効です。
  • base フィールドは astreamer 型の先頭要素として配置され、ポリモーフィックなストリーマー操作を可能にします。

アルゴリズムまたは振る舞い

  • astreamer パイプラインの一部として、astreamer_zstd_compressor または astreamer_zstd_decompressor の各関数内でキャストされ、zstd のコンテキスト管理およびバッファリング状態を維持します。
  • 圧縮時は ZSTD_CCtx を、伸長時は ZSTD_DCtx を使用してストリーム処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer_zstd_compressor_new および astreamer_zstd_decompressor_new にてインスタンスが生成されます。また、各ストリーマーの content, finalize, free 関数内でキャストされて参照されます。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数呼び出しは行いません。

関連データ構造

  • astreamer: 本構造体が継承する基底ストリーマー構造体。
  • ZSTD_CCtx, ZSTD_DCtx: libzstd が提供する圧縮・伸長コンテキスト。
  • ZSTD_outBuffer: libzstd の出力バッファ管理構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時（palloc0 使用）は PostgreSQL のエラーハンドリングにより終了します。
  • zstd ライブラリの処理エラーは pg_fatal を通じて報告されます。
  • 構造体内のポインタが初期化されていない状態でアクセスされた場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • astreamer_zstd_frame は astreamer の派生型として扱われるため、base メンバのレイアウト変更には注意が必要です。
  • 圧縮レベルやスレッド設定は、本構造体ではなく生成時に渡される仕様に基づきます。
  • 伸長と圧縮の両方のコンテキストを保持していますが、通常は用途に応じていずれか一方が使用されます。
