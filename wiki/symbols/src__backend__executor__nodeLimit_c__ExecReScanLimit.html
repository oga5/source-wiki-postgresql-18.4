ExecReScanLimit

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeLimit.c:540

目的

Limit ノードの再スキャン処理を実行し、LIMIT および OFFSET 式の再評価と内部状態のリセットを行う。

シグネチャまたは形状

void ExecReScanLimit(LimitState *node)

入力と出力

  • **入力**: LimitState *node（再スキャン対象の Limit ノードの状態構造体）
  • **出力**: なし（副作用としてノードの状態を更新）

前提条件と不変条件

  • recompute_limits(node) を呼び出し、パラメータ変更を反映させる必要がある。
  • 下位ノード（outerPlan）が Sort 等のパラメータ依存を持つ場合、下位ノードの再スキャン前に本関数で LIMIT/OFFSET の再計算を完了させる必要がある。

アルゴリズムまたは振る舞い

1. recompute_limits(node) を呼び出し、LIMIT/OFFSET 式を再評価して内部状態をリセットする。 2. 下位ノード（outerPlan）の chgParam を確認する。 3. chgParam が NULL の場合、下位ノードに対して ExecReScan(outerPlan) を呼び出し、明示的に再スキャンを指示する。 4. chgParam が NULL でない場合、下位ノードは最初の ExecProcNode 呼び出し時に自動的に再スキャンされるため、本関数では何もしない。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:298
  • **呼び出し先**:
  • recompute_limits (内部関数)
  • ExecReScan (下位ノード用)

関連データ構造

  • LimitState: Limit ノードの実行状態を保持する構造体。
  • PlanState: outerPlan を参照するために使用。

失敗モードと境界条件

  • 下位ノードが NULL の場合、outerPlanState(node) の呼び出しで未定義の動作が発生する可能性がある（ソースコード上は outerPlan が存在することを前提としている）。

保守担当者向けメモ

  • WITH TIES 指定時の境界値比較ロジックは recompute_limits および ExecLimit に依存しているため、再スキャン時の状態整合性に注意すること。
  • chgParam による条件分岐は、実行エンジン全体の再スキャン最適化ルールに従っている。
