pglz_decompress_datum

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/common/toast_compression.c:81

目的

PGLZアルゴリズムで圧縮されたTOASTデータ（varlena）を伸長し、元のデータ形式に戻す。

シグネチャまたは形状

struct varlena *pglz_decompress_datum(const struct varlena *value)

入力と出力

  • **入力**: value - 圧縮されたデータを含む varlena 構造体へのポインタ。
  • **出力**: 伸長後のデータを含む、新しく割り当てられた varlena 構造体へのポインタ。

前提条件と不変条件

  • 入力 value はPGLZ形式で圧縮されている必要がある。
  • 伸長後のサイズは VARDATA_COMPRESSED_GET_EXTSIZE(value) により取得可能であること。

アルゴリズムまたは振る舞い

1. VARDATA_COMPRESSED_GET_EXTSIZE を用いて伸長後のサイズを特定し、palloc でメモリを確保する。 2. pglz_decompress を呼び出し、圧縮データ（ヘッダーを除く）を伸長先バッファへ展開する。 3. pglz_decompress が負の値を返した場合は、データ破損とみなして ereport(ERROR, ...) を発行する。 4. 伸長後のサイズを SET_VARSIZE でヘッダーに設定し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/detoast.c:485
  • **呼び出し先**: pglz_decompress（common/pg_lzcompress.h で定義）

関連データ構造

  • varlena: PostgreSQLの可変長データ型。
  • VARHDRSZ: 可変長データのヘッダーサイズ。
  • VARHDRSZ_COMPRESSED: 圧縮された可変長データのヘッダーサイズ。

失敗モードと境界条件

  • **データ破損**: 伸長処理中に pglz_decompress が負の値を返した場合、ERRCODE_DATA_CORRUPTED エラーが発生する。
  • **メモリ不足**: palloc が失敗した場合、PostgreSQLのメモリ管理機構によりエラーがスローされる。

保守担当者向けメモ

  • 本関数はPGLZ専用である。LZ4など他のアルゴリズムについては別の関数が用意されている。
  • 伸長後のサイズ取得には varlena ヘッダーの特定のビットフィールドを利用しているため、ヘッダー構造の変更時には注意が必要である。
