zmm_regs_available

定義

  • 種別: function
  • 定義箇所: src/port/pg_crc32c_sse42_choose.c:46

目的

AVX-512命令セットの実行に必要なZMMレジスタが、OSによって適切に保存・復元可能（有効化）であるかを判定します。

シグネチャまたは形状

static bool zmm_regs_available(void)

入力と出力

  • 入力: なし
  • 出力: bool 型。ZMMレジスタが利用可能な場合は true、それ以外は false を返します。

前提条件と不変条件

  • 本関数を呼び出す前に、必ず CPUID 命令を用いて OSXSAVE フラグがセットされていることを確認する必要があります。

アルゴリズムまたは振る舞い

1. HAVE_XSAVE_INTRINSICS が定義されている場合、_xgetbv(0) を呼び出し、返された値のビットマスクが 0xe6（XCR0のビット0, 1, 2, 5, 6, 7の組み合わせ）と一致するかを確認します。 2. 一致する場合、OSがZMMレジスタの状態管理をサポートしていると判断し true を返します。 3. HAVE_XSAVE_INTRINSICS が定義されていない環境では、常に false を返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/port/pg_crc32c_sse42_choose.c (判定ロジック内)
  • src/port/pg_popcount_avx512.c (機能判定時)
  • 呼び出し先:
  • _xgetbv (組み込み関数。HAVE_XSAVE_INTRINSICS 定義時)

関連データ構造

  • なし

失敗モードと境界条件

  • OSXSAVE フラグの確認を怠った状態で本関数を呼び出した場合、CPUのサポート状況に関わらず誤った判定結果を返す可能性があります。

保守担当者向けメモ

  • 本関数は、AVX-512関連機能を利用するモジュール間で共通して使用されます。判定ロジックを変更する場合は、関連するすべてのモジュール（pg_crc32c_sse42_choose.c および pg_popcount_avx512.c）への影響を考慮してください。
