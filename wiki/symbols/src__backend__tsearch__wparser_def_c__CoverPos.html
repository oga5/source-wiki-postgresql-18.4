CoverPos

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/wparser_def.c:1955

目的

全文検索結果のハイライト生成（mark_hl_fragments）において、検索語を含むテキストの断片（フラグメント）を管理するために使用される構造体です。

シグネチャまたは形状

typedef struct {
    int32 startpos;   /* フラグメントの開始単語インデックス */
    int32 endpos;     /* 終了単語インデックス（含む） */
    int32 poslen;     /* 興味深い単語の数 */
    int32 curlen;     /* 単語の総数 */
    bool  chosen;     /* 選択済みか */
    bool  excluded;   /* 除外済みか */
} CoverPos;

入力と出力

本構造体は関数引数や戻り値として直接渡されるほか、palloc / repalloc を用いて動的に確保される配列の要素として利用されます。

前提条件と不変条件

  • startpos および endpos は単語インデックスとして整合性が取れている必要があります。
  • curlen はフラグメント内の単語総数を表します。

アルゴリズムまたは振る舞い

ハイライト生成処理において、検索語の出現位置に基づいてフラグメントを特定し、chosen や excluded フラグを更新することで、最終的に表示するハイライト範囲を決定するロジックの一部として機能します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tsearch/wparser_def.c 内のハイライト生成関連関数。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • CoverPos の配列が palloc を通じて動的に確保されます。

失敗モードと境界条件

  • メモリ不足時: palloc / repalloc が失敗した場合、PostgreSQL のエラーハンドリングにより処理が中断されます。
  • インデックスの範囲外アクセス: startpos や endpos が不正な値である場合、後続のハイライト処理で予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は mark_hl_fragments 関数に関連するフラグメント管理に特化しています。
  • 配列のサイズ変更（repalloc）が発生するため、メモリ管理には注意が必要です。
