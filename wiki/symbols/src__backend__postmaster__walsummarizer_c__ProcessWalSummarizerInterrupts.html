ProcessWalSummarizerInterrupts

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/walsummarizer.c:861

目的

WAL Summarizer プロセスにおいて、保留中のシグナルや設定変更、終了要求を処理し、プロセスの状態を適切に同期させる。

シグネチャまたは形状

static void ProcessWalSummarizerInterrupts(void)

入力と出力

  • 入力: なし（グローバルなフラグ変数を参照）
  • 出力: なし（副作用としてプロセスの終了や設定の再読み込みを行う）

前提条件と不変条件

  • 本関数は walsummarizer.c 内のメインループおよび待機処理の随所で呼び出されることを想定している。
  • summarize_wal GUC パラメータが false に変更された場合、本関数はプロセスを終了させる。

アルゴリズムまたは振る舞い

1. **シグナルバリアの処理**: ProcSignalBarrierPending が真であれば ProcessProcSignalBarrier() を実行。 2. **設定再読み込み**: ConfigReloadPending が真であれば、フラグをリセットし ProcessConfigFile(PGC_SIGHUP) を実行。 3. **終了要求の確認**: ShutdownRequestPending が真、または summarize_wal が無効であれば、ログを出力して proc_exit(0) を呼び出しプロセスを終了する。 4. **メモリコンテキストのログ出力**: LogMemoryContextPending が真であれば ProcessLogMemoryContextInterrupt() を実行。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/walsummarizer.c 内の複数箇所（WalSummarizerMain ループ内および待機処理中）。
  • **呼び出し先**:
  • ProcessProcSignalBarrier
  • ProcessConfigFile
  • proc_exit
  • ProcessLogMemoryContextInterrupt

関連データ構造

  • ProcSignalBarrierPending (グローバルフラグ)
  • ConfigReloadPending (グローバルフラグ)
  • ShutdownRequestPending (グローバルフラグ)
  • summarize_wal (GUC パラメータ)
  • LogMemoryContextPending (グローバルフラグ)

失敗モードと境界条件

  • proc_exit(0) が呼び出されると、プロセスは即座に終了し、呼び出し元へは戻らない。

保守担当者向けメモ

  • 本関数はプロセスが割り込みを安全に処理するためのゲートウェイである。新しい割り込み要因を追加する場合は、本関数内に処理を追加すること。
  • summarize_wal の状態変化を監視しているため、このプロセスが不要になった際の終了処理が自動的に行われる設計となっている。
