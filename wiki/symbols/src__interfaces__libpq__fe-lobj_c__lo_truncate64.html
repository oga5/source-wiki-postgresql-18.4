lo_truncate64

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-lobj.c:194

目的

データベース内のラージオブジェクトを指定されたサイズに切り詰める（truncate）。

シグネチャまたは形状

int lo_truncate64(PGconn *conn, int fd, int64_t len);

入力と出力

  • **入力**:
  • conn: データベース接続を管理する PGconn 構造体へのポインタ。
  • fd: 切り詰め対象のラージオブジェクト記述子。
  • len: 切り詰め後の新しいサイズ（64ビット整数）。
  • **出力**:
  • 成功時は 0 を返す。
  • 失敗時は -1 を返し、libpq のエラー状態が更新される。

前提条件と不変条件

  • 内部的に lo_initialize(conn) を呼び出し、サーバー側関数のOID解決とキャッシュを行う。
  • サーバー側で lo_truncate64 関数が利用可能である必要がある（conn->lobjfuncs->fn_lo_truncate64 が 0 でないこと）。

アルゴリズムまたは振る舞い

1. lo_initialize を実行し、接続状態を初期化する。 2. サーバー側の lo_truncate64 関数のOIDがキャッシュされているか確認し、未解決の場合はエラーを返す。 3. 引数 fd を整数として、len をネットワークバイトオーダー（lo_hton64）に変換して PQArgBlock に格納する。 4. PQfn を介してサーバー側の lo_truncate64 関数を呼び出す。 5. サーバーからの応答（PGresult）を確認し、PGRES_COMMAND_OK であれば成功として retval を返し、それ以外はエラーとして -1 を返す。 6. 最後に PQclear(res) を呼び出し、結果セットを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/examples/testlo64.c 等のクライアントアプリケーション。
  • **呼び出し先**: lo_initialize（内部初期化）、PQfn（サーバー関数実行）、lo_hton64（バイトオーダー変換）、libpq_append_conn_error（エラー報告）、PQresultStatus / PQclear（結果処理）。

関連データ構造

  • PGconn: 接続状態とラージオブジェクト関数のOIDキャッシュを保持。
  • PQArgBlock: PQfn に渡す引数構造体。

失敗モードと境界条件

  • lo_initialize が失敗した場合、またはサーバー側で関数がサポートされていない場合は -1 を返す。
  • サーバー側での実行中にエラーが発生した場合、PQresultStatus が PGRES_COMMAND_OK 以外となり、-1 を返す。

保守担当者向けメモ

  • 本関数は64ビット対応版であり、古いサーバーバージョンでは利用できない可能性があるため、実行時にOIDの有無を確認するガード処理が含まれている。
  • トランザクション内で実行されるため、失敗時はトランザクションがアボート状態になる点に注意すること。
