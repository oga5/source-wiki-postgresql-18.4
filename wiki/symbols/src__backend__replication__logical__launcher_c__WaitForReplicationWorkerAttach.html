WaitForReplicationWorkerAttach

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/launcher.c:173

目的

論理レプリケーションのバックグラウンドワーカーが共有メモリにアタッチし、稼働を開始するまで待機する。

シグネチャまたは形状

static bool WaitForReplicationWorkerAttach(LogicalRepWorker *worker, uint16 generation, BackgroundWorkerHandle *handle);

入力と出力

  • **入力**:
  • worker: 待機対象の LogicalRepWorker 構造体へのポインタ。
  • generation: 待機対象のワーカー世代番号。
  • handle: バックグラウンドワーカーのハンドル。
  • **出力**:
  • bool: ワーカーが正常にアタッチした場合は true、ワーカーが起動前に終了した場合は false を返す。

前提条件と不変条件

  • LogicalRepWorkerLock を介して共有メモリ上のワーカー状態を保護する。
  • 待機中に割り込み（CHECK_FOR_INTERRUPTS）が発生した場合は即座に中断する。

アルゴリズムまたは振る舞い

1. ループ内で LogicalRepWorkerLock を共有モードで取得し、worker->proc が設定されている（アタッチ完了）か、worker->in_use が偽（ワーカー終了）かを確認する。 2. ワーカーがまだ起動中の場合、GetBackgroundWorkerPid でプロセスの生存を確認する。 3. BGWH_STOPPED の場合は、排他ロックを取得して logicalrep_worker_cleanup を実行し、false を返して終了する。 4. 上記以外の場合、WaitLatch を用いて 10ms のタイムアウト付きで待機する。 5. 待機中にラッチがセットされた場合はリセットし、ループを継続する。 6. 終了時に、待機中にラッチを消費していた場合は SetLatch を再実行してイベントを復元する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/launcher.c 内のワーカー起動処理（logicalrep_worker_launch 等）。
  • **呼び出し先**: CHECK_FOR_INTERRUPTS, LWLockAcquire, LWLockRelease, GetBackgroundWorkerPid, logicalrep_worker_cleanup, WaitLatch, ResetLatch, SetLatch。

関連データ構造

  • LogicalRepWorker: 論理レプリケーションワーカーの共有メモリ状態を保持する構造体。
  • BackgroundWorkerHandle: バックグラウンドワーカーの制御ハンドル。
  • LogicalRepWorkerLock: ワーカー状態を保護する LWLock。

失敗モードと境界条件

  • ワーカーがアタッチする前にプロセスが終了した場合、BGWH_STOPPED を検知してクリーンアップを行い false を返す。
  • 待機中にポストマスターが終了した場合、WaitLatch の WL_EXIT_ON_PM_DEATH により待機が終了する。

保守担当者向けメモ

  • 本関数はワーカーのアタッチを待機するためのものであり、タイムアウト（10ms）は通知メカニズムがないための暫定的なポーリング間隔である。
  • 待機中にラッチを消費した場合、呼び出し元がイベントを見失わないよう dropped_latch フラグを用いて終了時にラッチを再設定する設計となっている。
