bbsink_zstd_begin_backup

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_zstd.c:87

目的

Zstandard圧縮シンクの初期化を行い、圧縮コンテキストの生成とパラメータ設定、および後続のシンクへのバックアップ開始通知を行う。

シグネチャまたは形状

static void bbsink_zstd_begin_backup(bbsink *sink)

入力と出力

  • **入力**: bbsink *sink （bbsink_zstd 構造体としてキャストして使用）
  • **出力**: なし（副作用として mysink->cctx の生成、mysink->base.bbs_buffer の割り当て、および後続シンクへの bbsink_begin_backup 呼び出しを行う）

前提条件と不変条件

  • sink は bbsink_zstd 型のインスタンスであること。
  • mysink->compress に有効な圧縮仕様が格納されていること。
  • mysink->base.bbs_buffer_length が適切に設定されていること。

アルゴリズムまたは振る舞い

1. ZSTD_createCCtx() を呼び出し、圧縮コンテキストを生成する。 2. 圧縮レベル、スレッド数（ZSTD_c_nbWorkers）、および長距離マッチング設定（ZSTD_c_enableLongDistanceMatching）を ZSTD_CCtx_setParameter で適用する。 3. palloc を用いて mysink->base.bbs_buffer を確保する。 4. ZSTD_compressBound を使用して、圧縮後のデータが収まるバッファサイズを計算し、BLCKSZ の倍数に切り上げる。 5. 後続のシンクに対して bbsink_begin_backup を呼び出し、初期化を伝播させる。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_zstd_ops 構造体を通じて、バックアップパイプラインの開始時に呼び出される。
  • **呼び出し先**:
  • ZSTD_createCCtx
  • ZSTD_CCtx_setParameter
  • palloc
  • ZSTD_compressBound
  • bbsink_begin_backup （後続のシンク）

関連データ構造

  • bbsink_zstd: 圧縮状態を保持する構造体。
  • pg_compress_specification: 圧縮設定（レベル、スレッド数等）を保持。
  • ZSTD_CCtx: libzstd の圧縮コンテキスト。

失敗モードと境界条件

  • ZSTD_createCCtx が失敗した場合、elog(ERROR) で終了する。
  • 圧縮パラメータ（レベル、スレッド数、長距離マッチング）の設定が ZSTD_isError を返す場合、elog または ereport でエラーを報告する。

保守担当者向けメモ

  • 本関数は USE_ZSTD が定義されている環境でのみ有効です。
  • ZSTD_c_nbWorkers の設定は、libzstd のバージョンやビルド時のスレッドサポート状況に依存するため、設定失敗時のエラーハンドリングが重要です。
  • bbs_buffer のサイズ計算において BLCKSZ によるアライメント調整を行っている点に注意してください。
