shmem_exit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/ipc.c:228

目的

プロセス終了シーケンスの一環として、共有メモリ関連のリソース解放および登録されたクリーンアップコールバックの実行を管理する。

シグネチャまたは形状

void shmem_exit(int code)

入力と出力

  • **入力**: code (int) - 終了ステータスコード。各コールバック関数に引数として渡される。
  • **出力**: なし（void）。

前提条件と不変条件

  • shmem_exit_inprogress フラグが実行開始時に true に設定され、終了時に false に戻される。
  • コールバック実行前に LWLockReleaseAll() が呼び出され、保持しているすべての LWLocks が解放される。
  • コールバックリストは LIFO（後入れ先出し）順で実行される。

アルゴリズムまたは振る舞い

1. **ロック解放**: LWLockReleaseAll() を呼び出し、デタッチされた DSM セグメントへのアクセスを防ぐ。 2. **高レベルコールバック実行**: before_shmem_exit_list に登録された関数を逆順（LIFO）で実行する。 3. **DSM シャットダウン**: dsm_backend_shutdown() を呼び出し、動的共有メモリセグメントを解放する。 4. **低レベルコールバック実行**: on_shmem_exit_list に登録された関数を逆順（LIFO）で実行する。 5. **インデックスリセット**: 各リストのインデックスを 0 にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/ipc.c 内の proc_exit（終了シーケンスの主要な一部として）。
  • src/backend/postmaster/postmaster.c:3196（直接呼び出し）。
  • **呼び出し先**:
  • LWLockReleaseAll()
  • dsm_backend_shutdown()
  • before_shmem_exit_list に登録された各関数
  • on_shmem_exit_list に登録された各関数

関連データ構造

  • before_shmem_exit_list / before_shmem_exit_index
  • on_shmem_exit_list / on_shmem_exit_index
  • shmem_exit_inprogress（状態フラグ）

失敗モードと境界条件

  • コールバック実行中にエラーが発生した場合、再帰的な呼び出しを防ぐため、各コールバックは実行前にリストから削除される設計となっている。
  • dsm_backend_shutdown() は、個別のコールバックがエラーを投げても残りのコールバックが実行されるよう設計されている。

保守担当者向けメモ

  • 本関数は「共有メモリのクリーンアップ」に特化しており、プロセス終了処理のフェーズ管理において重要な役割を果たす。
  • before_shmem_exit はシステムがまだ稼働している必要がある処理（一時リレーションの削除など）に、on_shmem_exit は低レベルなリソース解放に使用する。
