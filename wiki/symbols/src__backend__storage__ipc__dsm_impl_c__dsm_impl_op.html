dsm_impl_op

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/dsm_impl.c:158

目的

動的共有メモリ（DSM）に対する低レベルな操作（作成、アタッチ、デタッチ、破棄）を、実行環境のOSに応じた実装へディスパッチする抽象化レイヤーです。

シグネチャまたは形状

bool dsm_impl_op(dsm_op op, dsm_handle handle, Size request_size,
                 void **impl_private, void **mapped_address, Size *mapped_size,
                 int elevel);

入力と出力

  • **入力**:
  • op: 操作種別（dsm_op 列挙型）。
  • handle: 操作対象のDSMハンドル。
  • request_size: 作成時の要求サイズ（それ以外は0）。
  • impl_private: 実装固有のデータ（ファイル記述子やハンドル等）を保持するポインタ。
  • mapped_address: マッピング済みアドレスを格納するポインタ。
  • mapped_size: マッピング済みサイズを格納するポインタ。
  • elevel: エラー発生時のログレベル。
  • **出力**: 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • op が DSM_OP_CREATE 以外の場合、request_size は 0 でなければならない。
  • op が DSM_OP_CREATE または DSM_OP_ATTACH の場合、*mapped_address は NULL かつ *mapped_size は 0 でなければならない。

アルゴリズムまたは振る舞い

1. 引数の前提条件を Assert で検証します。 2. グローバル変数 dynamic_shared_memory_type を参照し、現在のプラットフォームに応じた実装関数（dsm_impl_posix, dsm_impl_sysv, dsm_impl_windows, dsm_impl_mmap のいずれか）へ処理を委譲します。 3. 該当する実装が存在しない場合、elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/dsm.c 内の各関数（dsm_create, dsm_attach, dsm_detach, dsm_destroy 等）。
  • **呼び出し先**: dynamic_shared_memory_type に応じて、dsm_impl_posix, dsm_impl_sysv, dsm_impl_windows, dsm_impl_mmap のいずれかが呼び出されます。

関連データ構造

  • dsm_op: 操作種別を定義する列挙型。
  • dynamic_shared_memory_type: 使用中のDSM実装方式を保持するGUC変数。

失敗モードと境界条件

  • dynamic_shared_memory_type が不正な値の場合、elog(ERROR) により処理が中断されます。
  • 各プラットフォーム固有の実装関数が失敗した場合、その戻り値（false）がそのまま返されます。

保守担当者向けメモ

  • 本関数は単なるディスパッチャです。個別のOS固有のロジックは各 dsm_impl_*.c 関数に記述されています。
  • 新しいOSや共有メモリ方式を追加する場合は、dynamic_shared_memory_type の列挙値追加と、本関数の switch 文へのケース追加が必要です。
