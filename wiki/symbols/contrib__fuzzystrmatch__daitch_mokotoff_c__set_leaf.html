set_leaf

定義

  • 種別: function
  • 定義箇所: contrib/fuzzystrmatch/daitch_mokotoff.c:195

目的

Daitch-Mokotoff Soundex アルゴリズムの処理過程において、特定のノードを「リーフ（末端）」としてマークし、リーフのリンクリストに登録する。

シグネチャまたは形状

static void set_leaf(dm_node *first_node[2], dm_node *last_node[2], dm_node *node, int ix_node)

入力と出力

  • first_node[2]: リーフリストの先頭ノードを保持する配列（入出力）。
  • last_node[2]: リーフリストの末尾ノードを保持する配列（入出力）。
  • node: リーフとして設定する対象の dm_node。
  • ix_node: 処理対象のインデックス（0 または 1）。

前提条件と不変条件

  • node は有効な dm_node 構造体へのポインタであること。
  • ix_node は 0 または 1 であること。

アルゴリズムまたは振る舞い

1. node->is_leaf が既に設定されている場合は何もしない。 2. node->is_leaf を 1 に設定する。 3. first_node[ix_node] が NULL の場合、node をリストの先頭として設定する。 4. NULL でない場合、現在の last_node[ix_node]->next[ix_node] に node を接続する。 5. last_node[ix_node] を node に更新し、node->next[ix_node] を NULL に設定してリストの終端を確定させる。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/fuzzystrmatch/daitch_mokotoff.c 内の update_leaves 等の処理から呼び出される（行 324 の呼び出しを確認）。
  • 呼び出し先: 直接的な関数呼び出しは行わない。

関連データ構造

  • dm_node: 音韻コード生成のツリー構造を構成するノード。

失敗モードと境界条件

  • node が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • ix_node が配列境界（0, 1）外の場合、メモリ破壊を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は dm_node の状態遷移を管理する重要なユーティリティである。
  • first_node および last_node 配列は、アルゴリズムの並列的な音韻パス（最大2系統）を追跡するために使用されている。
