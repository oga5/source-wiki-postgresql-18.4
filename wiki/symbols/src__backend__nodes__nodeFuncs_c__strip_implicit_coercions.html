strip_implicit_coercions

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/nodeFuncs.c:704

目的

ノードツリーの最上位にある「暗黙的な型変換（implicit coercion）」ノードを再帰的に除去し、変換対象の元のノードを返します。

シグネチャまたは形状

Node *strip_implicit_coercions(Node *node)

入力と出力

  • **入力**: Node *node（解析対象のノードツリーのルート）
  • **出力**: Node *（暗黙的変換が除去された後のノード、または入力が変換ノードでない場合は入力そのもの）

前提条件と不変条件

  • 入力が NULL の場合は NULL を返します。
  • 変換ノードが連鎖している場合、再帰的にすべて除去されます。
  • COERCE_IMPLICIT_CAST 以外の形式を持つ変換ノードは除去されず、そのまま返されます。

アルゴリズムまたは振る舞い

以下のノード型について、その format フィールドが COERCE_IMPLICIT_CAST であるかを確認し、該当する場合はその内部ノードに対して再帰的に本関数を呼び出します。

  • FuncExpr
  • RelabelType
  • CoerceViaIO
  • ArrayCoerceExpr
  • ConvertRowtypeExpr
  • CoerceToDomain

いずれの条件にも一致しないノードに到達した時点で、そのノードを返して再帰を終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/deparse.c
  • src/backend/commands/tablecmds.c
  • src/backend/parser/parse_clause.c
  • src/backend/rewrite/rewriteHandler.c
  • **呼び出し先**:
  • 自身（再帰呼び出し）

関連データ構造

  • Node (基底構造体)
  • FuncExpr, RelabelType, CoerceViaIO, ArrayCoerceExpr, ConvertRowtypeExpr, CoerceToDomain (各変換ノード構造体)

失敗モードと境界条件

  • 入力が NULL の場合、安全に NULL を返します。
  • 変換ノードがネストしている場合、スタックの深さに応じて再帰が発生します。

保守担当者向けメモ

  • 新しい種類の暗黙的変換ノードが PostgreSQL に追加された場合、本関数の if-else ブロックに新しい IsA チェックを追加する必要があります。
  • 変換形式の判定には COERCE_IMPLICIT_CAST 定数を使用してください。
