ObjectAccessPostAlter

定義

  • 種別: struct
  • 定義箇所: src/include/catalog/objectaccess.h:86

目的

SQLオブジェクトの変更（ALTER）操作後に発生するイベントにおいて、フック関数へコンテキスト情報を伝達するために使用されます。

シグネチャまたは形状

typedef struct {
    Oid     auxiliary_id;
    bool    is_internal;
} ObjectAccessPostAlter;

入力と出力

本構造体は、InvokeObjectPostAlterHook マクロを通じてフック関数に渡される引数の一部として機能します。

前提条件と不変条件

  • auxiliary_id: pg_inherits、pg_db_role_setting、pg_user_mapping のエントリを識別する場合にのみ使用されます。それ以外の場合は InvalidOid を設定する必要があります。
  • is_internal: ユーザーが明示的に変更を要求したものではなく、システム内部の理由（例: CLUSTER 中のテンポラリヒープの制約変更）で実行された場合に true となります。

アルゴリズムまたは振る舞い

フック関数（セキュリティプラグインや監査ログ機能）は、この構造体を参照して以下の判断を行います。

  • is_internal が true の場合、権限チェックをスキップするなどの制御を行うことが推奨されます。
  • auxiliary_id を用いて、対象となるシステムカタログの特定のタプルを特定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/objectaccess.c 内の関数が、この構造体を初期化してフックを呼び出します。
  • **呼び出し先**: contrib/sepgsql/hooks.c や src/test/modules/test_oat_hooks/test_oat_hooks.c など、object_access_hook を実装する外部拡張機能がこの構造体を受け取ります。

関連データ構造

  • ObjectAccessType: 監視対象イベントの種類を定義します。
  • Oid: PostgreSQLのオブジェクト識別子型。

失敗モードと境界条件

  • auxiliary_id に不適切な値を設定した場合、フック関数側で誤ったオブジェクトを対象として処理が行われる可能性があります。
  • is_internal の判定が不正確な場合、本来必要な権限チェックがスキップされる、あるいは不要なチェックが実行される可能性があります。

保守担当者向けメモ

  • 本構造体は InvokeObjectPostAlterHook マクロ経由でのみ使用してください。
  • is_internal フラグは、プラグイン側でセキュリティポリシーを柔軟に適用するための重要な情報源です。変更を加える際は、このフラグのセマンティクスを維持してください。
