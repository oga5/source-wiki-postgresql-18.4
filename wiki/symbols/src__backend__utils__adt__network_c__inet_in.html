inet_in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network.c:120

目的

文字列形式の IP アドレス（inet 型）を PostgreSQL 内部の inet 構造体へ変換します。

シグネチャまたは形状

Datum inet_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して取得される、IP アドレスを表す文字列（char *）。
  • **出力**: PG_RETURN_INET_P マクロを介して返される、inet 型の Datum 表現。

前提条件と不変条件

  • 入力文字列は有効な IP アドレス形式である必要があります。
  • network_in 関数へ is_cidr 引数として false を渡すことで、inet 型として処理されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_CSTRING(0) で入力文字列を取得します。 2. network_in 関数を呼び出し、文字列を内部表現に変換します。この際、is_cidr に false を指定し、呼び出し元のコンテキスト情報を渡します。 3. 変換結果を PG_RETURN_INET_P を通じて返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gin/btree_gin.c
  • src/backend/utils/adt/pgstatfuncs.c
  • （その他、SQL インターフェース経由で呼び出される可能性があります）
  • **呼び出し先**:
  • network_in（src/backend/utils/adt/network.c 内の共通ルーチン）

関連データ構造

  • inet: ネットワークアドレスを保持する内部構造体。
  • Datum: PostgreSQL の汎用データ表現。

失敗モードと境界条件

  • 不正な形式の文字列が入力された場合、network_in 内部でエラーが発生し、トランザクションがアボートされる可能性があります。

保守担当者向けメモ

  • 本関数は inet 型専用の入力関数です。cidr 型の入力には cidr_in が使用されます。
  • 変換ロジックの詳細は network_in に依存しているため、共通のパース処理を変更する場合は network_in を参照してください。
