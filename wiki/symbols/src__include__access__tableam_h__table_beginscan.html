table_beginscan

定義

  • 種別: function
  • 定義箇所: src/include/access/tableam.h:874

目的

テーブルスキャンを開始するための汎用インターフェースを提供し、ストレージエンジン（Table AM）に依存しないスキャン処理の初期化を行う。

シグネチャまたは形状

TableScanDesc table_beginscan(Relation rel, Snapshot snapshot, int nkeys, struct ScanKeyData *key)

入力と出力

  • **入力**:
  • rel: スキャン対象のリレーション（Relation）。
  • snapshot: 可視性判定に使用するMVCCスナップショット。
  • nkeys: スキャンキーの数。
  • key: スキャン条件を定義する ScanKeyData 構造体の配列。
  • **出力**:
  • TableScanDesc: 初期化されたスキャン記述子。

前提条件と不変条件

  • 呼び出し側は、対象のリレーションに対して適切なロックを保持している必要がある。
  • rel->rd_tableam が有効な TableAmRoutine を指していること。

アルゴリズムまたは振る舞い

1. シーケンシャルスキャン、同期スキャン、ページモードを許可するデフォルトフラグ（SO_TYPE_SEQSCAN 等）を設定する。 2. rel->rd_tableam->scan_begin を呼び出し、具体的なストレージエンジンの実装へ処理を委譲する。 3. 委譲先の実装から返された TableScanDesc を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgrowlocks/pgrowlocks.c
  • src/backend/access/heap/heapam_handler.c
  • src/backend/commands/copyto.c
  • src/backend/commands/tablecmds.c
  • src/backend/commands/typecmds.c
  • src/backend/executor/execReplication.c
  • src/backend/executor/nodeSeqscan.c
  • src/backend/partitioning/partbounds.c
  • **呼び出し先**:
  • rel->rd_tableam->scan_begin（具体的なAM実装の関数）。

関連データ構造

  • Relation: テーブルのメタデータとアクセス手法へのポインタを保持。
  • TableScanDesc: スキャン状態を保持する構造体。
  • ScanKeyData: スキャン時のフィルタ条件。
  • TableAmRoutine: テーブルアクセス手法の関数ポインタ群。

失敗モードと境界条件

  • rel->rd_tableam が未定義または不正な場合、セグメンテーションフォールトが発生する可能性がある。
  • スナップショットが適切に提供されない場合、MVCCの可視性判定が正しく行われない。

保守担当者向けメモ

  • 本関数はインライン関数であり、実際のロジックは各AMの scan_begin 実装に依存する。
  • 新しいAMを追加する際は、scan_begin が本関数で指定されたフラグを適切に処理できるか確認すること。
