dlist_delete_from

定義

  • 種別: static inline function
  • 定義箇所: src/include/lib/ilist.h:428

目的

二重連結リストから指定されたノードを削除し、その際に対象ノードがリストに含まれているかどうかの整合性チェックを行う。

シグネチャまたは形状

static inline void dlist_delete_from(dlist_head *head, dlist_node *node)

入力と出力

  • head: 削除対象のノードが属するリストのヘッドへのポインタ。
  • node: リストから削除するノードへのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • node は head が指すリストの有効なメンバーである必要がある。
  • dlist_member_check が内部で呼び出されるため、ILIST_DEBUG が有効な場合、リストの整合性が検証される。

アルゴリズムまたは振る舞い

1. dlist_member_check(head, node) を実行し、ノードがリスト内に存在するかを検証する。 2. dlist_delete(node) を呼び出し、ノードの前後ポインタを繋ぎ変えてリストから切り離す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/mmgr/slab.c (複数箇所)
  • src/backend/utils/resowner/resowner.c:1112
  • src/include/lib/ilist.h:767
  • 呼び出し先:
  • dlist_member_check
  • dlist_delete

関連データ構造

  • dlist_head: リストの起点構造体。
  • dlist_node: リストの要素となるリンク構造体。

失敗モードと境界条件

  • ILIST_DEBUG が定義されている環境において、node が head に属していない場合、アサーション失敗やエラーが発生する可能性がある。
  • リストが空の場合や、ノードがリストに含まれていない状態で呼び出した場合の挙動は、dlist_member_check の実装に依存する。

保守担当者向けメモ

  • 本関数は dlist_delete のラッパーであり、整合性チェックのステップが追加されている。
  • ノードの next/prev ポインタを NULL にリセットしたい場合は、dlist_delete_thoroughly の使用を検討すること。
