pgstat_reset_slru_counter_internal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_slru.c:246

目的

共有メモリ上の特定のSLRU統計情報をリセットし、リセット時刻を更新する。

シグネチャまたは形状

static void pgstat_reset_slru_counter_internal(int index, TimestampTz ts)

入力と出力

  • index (int): リセット対象のSLRUインデックス。
  • ts (TimestampTz): リセット時刻として記録するタイムスタンプ。
  • 出力: なし（副作用として共有メモリを直接更新）。

前提条件と不変条件

  • pgStatLocal.shmem->slru が初期化済みであること。
  • 呼び出し側は、index が有効な範囲内であることを保証する必要がある。

アルゴリズムまたは振る舞い

1. 共有メモリ上のSLRU統計領域 (pgStatLocal.shmem->slru) への排他ロック (LW_EXCLUSIVE) を取得する。 2. 指定された index に対応する PgStat_SLRUStats 構造体の全メンバを memset で 0 クリアする。 3. 同構造体の stat_reset_timestamp フィールドに引数 ts を代入する。 4. 排他ロックを解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • pgstat_reset_slru (特定のSLRU名指定時)
  • pgstat_reset_slru (全SLRUリセットループ内)
  • 呼び出し先:
  • LWLockAcquire
  • memset
  • LWLockRelease

関連データ構造

  • PgStatShared_SLRU: 共有メモリ上のSLRU統計管理構造体。
  • PgStat_SLRUStats: 個別のSLRU統計カウンタおよびリセット時刻を保持する構造体。

失敗モードと境界条件

  • index が範囲外の場合、メモリ破壊が発生する可能性がある（境界チェックは本関数内では行われない）。
  • ロック取得時に他のプロセスとの競合が発生した場合、ロックが解放されるまで待機する。

保守担当者向けメモ

  • 本関数は共有メモリを直接操作するため、必ず適切なロック下で実行される必要がある。
  • memset によるクリアは、PgStat_SLRUStats 構造体の全フィールドがゼロ初期化可能であることを前提としている。
