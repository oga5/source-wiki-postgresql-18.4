ArrayGetNItems

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arrayutils.c:56

目的

多次元配列の次元情報から、配列に含まれる全要素数を計算します。

シグネチャまたは形状

int ArrayGetNItems(int ndim, const int *dims);

入力と出力

  • **入力**:
  • ndim: 配列の次元数。
  • dims: 各次元のサイズを格納した整数配列。
  • **出力**:
  • 配列の全要素数を表す int 値を返します。

前提条件と不変条件

  • dims は少なくとも ndim 個の要素を持つ配列を指している必要があります。
  • 内部的に ArrayGetNItemsSafe を呼び出しており、計算結果がシステム制限を超える場合はエラーが発生する可能性があります。

アルゴリズムまたは振る舞い

1. ArrayGetNItemsSafe を NULL コンテキストで呼び出します。 2. ArrayGetNItemsSafe の計算結果（全要素数）をそのまま返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: ArrayGetNItemsSafe
  • **呼び出し元**:
  • contrib/cube/cube.c
  • contrib/dblink/dblink.c
  • contrib/intarray/_int.h
  • contrib/ltree/_ltree_gist.c, _ltree_op.c, lquery_op.c, ltree_gist.c
  • contrib/pg_surgery/heap_surgery.c

関連データ構造

  • ArrayType: PostgreSQLの配列データ型の内部表現。本関数は ArrayType の次元情報（ARR_NDIM, ARR_DIMS）を引数として受け取る用途で多用されます。

失敗モードと境界条件

  • ndim が 0 の場合、要素数は 0 とみなされます。
  • 計算過程で要素数が MaxArraySize を超える場合、または整数オーバーフローが発生する場合、ArrayGetNItemsSafe の仕様に基づきエラーがスローされます。

保守担当者向けメモ

  • 本関数は ArrayGetNItemsSafe のラッパーです。エラーハンドリングの柔軟性が必要な場合は、直接 ArrayGetNItemsSafe を使用してください。
  • contrib モジュール内の多くの拡張機能で、配列の要素数取得の標準的な手段として利用されています。
