tbm_free_shared_area

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/tidbitmap.c:329

目的

並列クエリ実行時に共有メモリ（DSA）上に確保された TBMSharedIteratorState および関連するデータ構造の参照カウントを減らし、必要に応じてメモリを解放する。

シグネチャまたは形状

void tbm_free_shared_area(dsa_area *dsa, dsa_pointer dp)

入力と出力

  • **入力**:
  • dsa: 共有メモリ領域を管理する dsa_area へのポインタ。
  • dp: 解放対象の TBMSharedIteratorState を指す dsa_pointer。
  • **出力**: なし。

前提条件と不変条件

  • dp は有効な TBMSharedIteratorState を指している必要がある。
  • 内部の各配列（pagetable, spages, schunks）は、dsa_pointer が有効な場合にのみ参照カウントの減算と解放が行われる。
  • 参照カウントが 0 に達した構造体のみが dsa_free によって解放される。

アルゴリズムまたは振る舞い

1. dsa_get_address を使用して dp から TBMSharedIteratorState の実体を取得する。 2. pagetable、spages、schunks の各ポインタが有効か確認する。 3. 有効な各構造体について、pg_atomic_sub_fetch_u32 を用いて refcount をアトミックに 1 減算する。 4. 減算後の値が 0 になった場合、dsa_free を呼び出して当該メモリ領域を解放する。 5. 最後に、TBMSharedIteratorState 自体（dp）を dsa_free で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeBitmapHeapscan.c:546
  • **呼び出し先**:
  • dsa_get_address (DSA API)
  • pg_atomic_sub_fetch_u32 (アトミック操作)
  • dsa_free (DSA API)

関連データ構造

  • TBMSharedIteratorState: 共有イテレータの状態を保持する構造体。
  • PTEntryArray, PTIterationArray: ページテーブルやページ/チャンク情報を保持する共有配列構造体。
  • dsa_area, dsa_pointer: PostgreSQL の動的共有メモリ管理用構造体。

失敗モードと境界条件

  • dsa_pointer が無効な場合、対応する構造体の解放処理はスキップされる。
  • 参照カウントが 0 に達しない限り、共有メモリ上のデータは保持され続ける。

保守担当者向けメモ

  • 本関数は並列クエリの終了処理において重要である。参照カウントの管理はアトミック操作に依存しているため、並列実行環境下での整合性が保証されている。
  • 構造体内の各ポインタが DsaPointerIsValid でチェックされているため、部分的に初期化された状態でも安全に呼び出し可能である。
