GetRelationIdentityOrPK

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/relation.c:890

目的

論理レプリケーションにおいて、更新や削除操作の識別に使用すべきインデックス（REPLICA IDENTITY または主キー）の OID を取得します。

シグネチャまたは形状

Oid GetRelationIdentityOrPK(Relation rel);

入力と出力

  • **入力**: rel (対象テーブルの Relation 構造体ポインタ)
  • **出力**: 識別に使用可能なインデックスの Oid。該当するインデックスが存在しない場合は InvalidOid を返します。

前提条件と不変条件

  • rel は有効なリレーション構造体である必要があります。
  • 戻り値が OidIsValid である場合、その OID はテーブルの REPLICA IDENTITY または主キーとして利用可能なインデックスを指します。

アルゴリズムまたは振る舞い

1. RelationGetReplicaIndex(rel) を呼び出し、REPLICA IDENTITY として設定されたインデックスの OID を取得します。 2. 手順1で有効な OID が得られなかった場合、RelationGetPrimaryKeyIndex(rel, false) を呼び出し、主キーインデックスの OID を取得します。 3. 最終的に特定されたインデックスの OID を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execReplication.c
  • src/backend/replication/logical/conflict.c
  • src/backend/replication/logical/relation.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • RelationGetReplicaIndex
  • RelationGetPrimaryKeyIndex

関連データ構造

  • Relation: テーブルのメタデータおよびインデックス情報を保持する構造体。

失敗モードと境界条件

  • テーブルに REPLICA IDENTITY が設定されておらず、かつ主キーも存在しない場合、InvalidOid が返されます。

保守担当者向けメモ

  • 本関数は、論理レプリケーションのワーカーがターゲットテーブルの行を特定する際に不可欠なインデックスを決定する標準的な手段です。
  • RelationGetPrimaryKeyIndex の第2引数に false を渡しているため、主キーが存在しない場合にエラーを発生させず、単に InvalidOid を返す挙動となります。
