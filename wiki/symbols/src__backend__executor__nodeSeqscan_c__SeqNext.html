SeqNext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSeqscan.c:50

目的

テーブルアクセスメソッド（AM）を介して、シーケンシャルスキャンにおける次のタプルを順次取得します。

シグネチャまたは形状

TupleTableSlot *SeqNext(SeqScanState *node)

入力と出力

  • **入力**: SeqScanState *node（スキャンの実行状態を保持するノード構造体）
  • **出力**: TupleTableSlot *（取得したタプルを格納するスロット、またはスキャン終了時は NULL）

前提条件と不変条件

  • node は初期化済みであること。
  • node->ss.ss_currentScanDesc が NULL の場合、非並列スキャンとして table_beginscan を実行し、スキャン記述子を初期化する。

アルゴリズムまたは振る舞い

1. node から現在のスキャン記述子 (scandesc)、実行状態 (estate)、スキャン方向、およびタプル格納用スロットを取得します。 2. scandesc が未初期化（NULL）であれば、table_beginscan を呼び出してリレーションをスキャン用に開き、node->ss.ss_currentScanDesc に保存します。 3. table_scan_getnextslot を呼び出し、現在のスキャン方向に従って次のタプルをスロットに読み込みます。 4. タプルが取得できた場合はそのスロットを返し、取得できなかった（スキャン終了）場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeSeqscan.c 内の ExecScanAccessMtd を介して実行エンジンから呼び出されます。
  • **呼び出し先**:
  • table_beginscan（スキャン記述子の初期化）
  • table_scan_getnextslot（タプルの取得）

関連データ構造

  • SeqScanState: スキャン実行時の状態を保持する構造体。
  • TableScanDesc: テーブルアクセスメソッドによるスキャン状態の記述子。
  • TupleTableSlot: タプルデータを保持するスロット。

失敗モードと境界条件

  • スキャン開始時にリレーションが正しく開けない場合、table_beginscan 内部でエラーが発生する可能性があります。
  • スキャンが終了した場合は NULL を返します。

保守担当者向けメモ

  • 本関数は ExecScanAccessMtd として登録されており、スキャンノードの実行ループから直接呼び出されます。
  • 並列スキャンと非並列スキャンの両方に対応しており、scandesc の遅延初期化ロジックが含まれています。
