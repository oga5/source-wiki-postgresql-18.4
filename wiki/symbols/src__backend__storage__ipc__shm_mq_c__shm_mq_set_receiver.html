shm_mq_set_receiver

定義

  • src/backend/storage/ipc/shm_mq.c:205

目的

共有メモリメッセージキュー（shm_mq）に対して、メッセージを受信するプロセス（PGPROC）を登録します。

シグネチャまたは形状

void shm_mq_set_receiver(shm_mq *mq, PGPROC *proc)

入力と出力

  • **入力**:
  • mq: 操作対象の共有メモリメッセージキューへのポインタ。
  • proc: 受信側として設定するプロセス構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • mq->mq_receiver が NULL であること（既に受信者が設定されていないこと）。この条件は Assert で検証されます。
  • mq->mq_mutex スピンロックにより、キューのヘッダー情報が保護されます。

アルゴリズムまたは振る舞い

1. mq->mq_mutex スピンロックを取得します。 2. mq->mq_receiver に指定された proc を代入します。 3. 現在の送信側プロセス（mq->mq_sender）をローカル変数に保持します。 4. スピンロックを解放します。 5. 送信側プロセスが既に存在する場合（sender != NULL）、そのプロセスのラッチ（procLatch）をセットし、受信者が設定されたことを通知します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c
  • src/backend/executor/execParallel.c
  • src/backend/replication/logical/applyparallelworker.c
  • src/test/modules/test_shm_mq/setup.c
  • src/test/modules/test_shm_mq/worker.c
  • **呼び出し先**:
  • SpinLockAcquire / SpinLockRelease
  • SetLatch

関連データ構造

  • shm_mq: メッセージキューの制御ヘッダー。
  • PGPROC: PostgreSQLのプロセス情報を保持する構造体。

失敗モードと境界条件

  • mq->mq_receiver が既に設定されている場合、Assert によりアボートします。
  • mq が無効なメモリ領域を指している場合の動作は未定義です。

保守担当者向けメモ

  • 本関数は、キューの初期化プロセスの一部として呼び出されることを想定しています。
  • 送信側への通知は、スピンロックを解放した後に実行されるため、デッドロックのリスクを低減しています。
