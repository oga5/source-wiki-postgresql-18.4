FreeErrorDataContents

定義

  • 種別: function
  • 定義箇所: src/backend/utils/error/elog.c:1834

目的

ErrorData 構造体が保持する動的に割り当てられた補助データ（文字列フィールド等）を解放する。

シグネチャまたは形状

static void FreeErrorDataContents(ErrorData *edata)

入力と出力

  • **入力**: edata - 解放対象の ErrorData 構造体へのポインタ。
  • **出力**: なし（副作用としてメモリを解放）。

前提条件と不変条件

  • edata は有効な ErrorData 構造体を指している必要がある。
  • 解放される各フィールドは、palloc 等で割り当てられたメモリであるか、あるいは NULL である必要がある。

アルゴリズムまたは振る舞い

ErrorData 構造体内の以下の各フィールドが NULL でない場合、pfree を呼び出してメモリを解放する。

  • message, detail, detail_log, hint, context, backtrace
  • schema_name, table_name, column_name, datatype_name, constraint_name
  • internalquery

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/error/elog.c 内の FreeErrorData 関数およびエラー処理フローの各所から呼び出される。
  • **呼び出し先**:
  • pfree（メモリ解放用関数）。

関連データ構造

  • ErrorData: エラーの詳細情報を保持する構造体。

失敗モードと境界条件

  • edata が NULL の場合、デリファレンスによりクラッシュする可能性がある（ソースコード上でのガードは存在しない）。
  • 各フィールドが NULL の場合は単にスキップされるため、安全に処理される。

保守担当者向けメモ

  • 本関数は ErrorData 構造体自体のメモリは解放せず、構造体が保持する「中身（Contents）」のみを解放する。構造体自体の解放が必要な場合は FreeErrorData を使用すること。
  • 新たなフィールドが ErrorData に追加された場合は、本関数にも解放処理を追加する必要がある。
