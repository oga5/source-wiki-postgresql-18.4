sqlda_common_total_size

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/sqlda.c:64

目的

PGresult の指定された行に含まれる各フィールドのデータサイズを計算し、アライメントを考慮した合計オフセット値を算出します。

シグネチャまたは形状

static long sqlda_common_total_size(const PGresult *res, int row, enum COMPAT_MODE compat, long offset)

入力と出力

  • **入力**:
  • res: 処理対象の PGresult オブジェクト。
  • row: 処理対象の行インデックス。
  • compat: 互換モード（Informix またはネイティブ）。
  • offset: 計算を開始する初期オフセット値。
  • **出力**:
  • 戻り値: 全フィールドのデータ配置後の最終的な合計オフセット値。

前提条件と不変条件

  • res は有効な PGresult であること。
  • ecpg_sqlda_align_add_size を用いて、各データ型に応じたアライメント規則が適用されること。
  • numeric 型の場合、内部的に PGTYPESnumeric_from_asc を使用して動的にサイズを算出するため、メモリ確保の前に値の解析が必要となる。

アルゴリズムまたは振る舞い

1. PQnfields(res) でフィールド数を取得し、各フィールドをループ処理する。 2. sqlda_dynamic_type を用いて、PostgreSQL の型を ECPG の型へ変換する。 3. 型ごとのサイズとアライメント要件に基づき、ecpg_sqlda_align_add_size を呼び出して offset を更新する。 4. numeric 型の場合、PGTYPESnumeric_from_asc で値を一時的にデコードし、実際のデータ長（digits 配列のサイズ）を計算に含める。 5. 文字列型（char, string 等）の場合は、strlen に 1 を加えた値をデータ長として使用する。 6. 全フィールドの処理完了後、最終的な offset を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/sqlda.c 内の sqlda_compat_total_size および sqlda_native_total_size から呼び出される。
  • **呼び出し先**:
  • PQnfields, PQftype, PQgetisnull, PQgetvalue (libpq)
  • sqlda_dynamic_type
  • ecpg_sqlda_align_add_size
  • PGTYPESnumeric_from_asc, PGTYPESnumeric_free

関連データ構造

  • PGresult: libpq の結果セット構造体。
  • COMPAT_MODE: 互換モードを定義する列挙型。
  • ECPGttype: ECPG で定義される型列挙。

失敗モードと境界条件

  • PGTYPESnumeric_from_asc が失敗した場合（num が NULL）、そのフィールドのサイズ加算はスキップされる。
  • PQgetvalue が返す文字列が NULL の場合、strlen でクラッシュする可能性がある（ソースコード上はチェックなし）。

保守担当者向けメモ

  • numeric 型のサイズ計算は、値のデコードを伴うため他の型よりコストが高い。
  • ecpg_sqlda_align_add_size のアライメント規則は、プラットフォーム固有の構造体パディングを模倣しているため、変更時は注意が必要。
