OidOptions

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_dump/pg_dump.c:141

目的

OID（Object Identifier）がゼロである場合に、getFormattedTypeName 関数がどのように振る舞うかを制御するためのオプションフラグを定義します。

シグネチャまたは形状

typedef enum OidOptions
{
    zeroIsError = 1,
    zeroAsStar = 2,
    zeroAsNone = 4,
} OidOptions;

入力と出力

  • 入力: getFormattedTypeName 関数の引数として渡されます。
  • 出力: なし（列挙型定数として機能します）。

前提条件と不変条件

  • 各値はビットフラグとして機能するように定義されています。
  • 呼び出し側は、OID がゼロの場合の期待される挙動に応じて、これらの中から適切な値を選択する必要があります。

アルゴリズムまたは振る舞い

  • zeroIsError: OID がゼロの場合、エラーとして扱います。
  • zeroAsStar: OID がゼロの場合、*（アスタリスク）として表現します。
  • zeroAsNone: OID がゼロの場合、none として表現します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/pg_dump.c 内の getFormattedTypeName を呼び出す箇所（例: 20210行目付近）。
  • 呼び出し先: 情報なし。

関連データ構造

  • Oid: PostgreSQL のオブジェクト識別子型。

失敗モードと境界条件

  • 複数のフラグを同時に指定した場合の挙動については、本ソース excerpt からは判断できません。
  • OID がゼロ以外の場合の挙動については、本ソース excerpt からは判断できません。

保守担当者向けメモ

  • getFormattedTypeName 関数の実装において、これらのフラグがどのように条件分岐に使用されているかを確認してください。
