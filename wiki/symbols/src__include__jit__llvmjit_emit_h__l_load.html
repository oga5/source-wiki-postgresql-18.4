l_load

定義

  • 種別: function
  • 定義箇所: src/include/jit/llvmjit_emit.h:119

目的

LLVM IR におけるメモリからの値のロード処理を抽象化し、冗長な記述を簡略化するラッパー関数です。

シグネチャまたは形状

static inline LLVMValueRef l_load(LLVMBuilderRef b, LLVMTypeRef t, LLVMValueRef v, const char *name)

入力と出力

  • **入力**:
  • b: 命令を挿入する LLVMBuilderRef。
  • t: ロードする値の型を表す LLVMTypeRef。
  • v: ロード元となるポインタを示す LLVMValueRef。
  • name: 生成される命令に付与する名前（文字列）。
  • **出力**:
  • ロードされた値を表す LLVMValueRef を返します。

前提条件と不変条件

  • 本関数は USE_LLVM が定義されている環境でのみ有効です。
  • b は有効な LLVMBuilderRef である必要があります。

アルゴリズムまたは振る舞い

内部で LLVMBuildLoad2 を呼び出し、指定されたビルダー b に対して、型 t を持つポインタ v から値をロードする命令を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit.c
  • src/backend/jit/llvm/llvmjit_deform.c
  • src/backend/jit/llvm/llvmjit_expr.c
  • **呼び出し先**:
  • LLVMBuildLoad2 (LLVM C API)

関連データ構造

  • LLVMBuilderRef: 命令生成器。
  • LLVMValueRef: LLVM IR 上の値。
  • LLVMTypeRef: LLVM IR 上の型。

失敗モードと境界条件

  • v が有効なポインタ型でない場合や、t がロード対象として不適切な場合、LLVM C API の仕様に従いエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はインライン関数であり、呼び出し側のコンテキストで直接展開されます。
  • LLVM C API のバージョンアップに伴う変更を吸収する層であるため、直接 LLVMBuildLoad2 を呼ぶ代わりに本関数を使用してください。
