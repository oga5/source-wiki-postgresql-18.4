ExecEndCustomScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeCustom.c:124

目的

カスタムスキャン実行ノードの終了処理を実行します。拡張機能側で定義されたクリーンアップロジックを呼び出すためのインターフェースです。

シグネチャまたは形状

void ExecEndCustomScan(CustomScanState *node)

入力と出力

  • **入力**: CustomScanState *node - 終了対象となるカスタムスキャンノードの状態構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • node->methods が有効であり、かつ node->methods->EndCustomScan 関数ポインタが NULL でないことが必須です（Assert により検証）。

アルゴリズムまたは振る舞い

1. node->methods->EndCustomScan が定義されていることを確認します。 2. 拡張機能が提供する EndCustomScan コールバック関数を呼び出し、拡張機能固有のリソース解放や終了処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:684
  • **呼び出し先**: node->methods->EndCustomScan（拡張機能側で実装された関数）

関連データ構造

  • CustomScanState: カスタムスキャンの実行状態を保持する構造体。
  • CustomExecMethods: 拡張機能が実装すべきコールバック関数のテーブル。

失敗モードと境界条件

  • node->methods->EndCustomScan が NULL の場合、アサーション違反によりプロセスが停止します。

保守担当者向けメモ

  • 本関数は単なるディスパッチャーです。実際の終了処理ロジックは、各拡張機能が提供する CustomExecMethods 内に実装されています。
  • 拡張機能側でメモリリークやリソースの解放漏れが発生している場合は、拡張機能側の EndCustomScan 実装を確認してください。
