gbt_byteale

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bytea.c:47

目的

bytea型に対する「小なりまたは等しい（less than or equal to）」比較をGiSTインデックスの内部ロジックへ提供します。

シグネチャまたは形状

bool gbt_byteale(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象のbyteaデータへのポインタ。
  • collation: 照合順序（本関数では使用されません）。
  • flinfo: 関数マネージャ情報（本関数では使用されません）。
  • **出力**:
  • a <= b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は有効なbyteaデータへのポインタである必要があります。
  • PostgreSQLの標準関数 byteale が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. a と b を PointerGetDatum を用いてPostgreSQLのDatum形式に変換します。 2. DirectFunctionCall2 を介して標準の byteale 関数を呼び出します。 3. 戻り値を DatumGetBool でbool型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は、contrib/btree_gist/btree_bytea.c:79 において関数ポインタとして参照されています。
  • **呼び出し先**: PostgreSQLの標準関数 byteale を呼び出します。

関連データ構造

  • bytea: PostgreSQLの可変長バイナリデータ型。

失敗モードと境界条件

  • a または b がNULLの場合、標準の byteale 関数の挙動に従います。
  • 比較対象のデータが不正なbytea形式である場合、標準関数側でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的な比較ロジックはPostgreSQLコアの byteale に依存しています。
  • btree_gist モジュール内の他の比較関数（gbt_bytealt 等）と同様のパターンで実装されています。
