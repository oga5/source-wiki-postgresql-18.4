PendingWrite

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/smgr/bulk_write.c:51

目的

バルク書き込み処理において、物理ストレージへフラッシュされるのを待機している個々のページデータとそのメタデータを保持するために使用されます。

シグネチャまたは形状

typedef struct PendingWrite
{
	BulkWriteBuffer buf;
	BlockNumber blkno;
	bool		page_std;
} PendingWrite;

入力と出力

本構造体はデータ保持用であり、直接的な関数入力・出力インターフェースを持ちません。BulkWriteState 内の配列要素として管理されます。

前提条件と不変条件

  • BulkWriteState の管理下で、MAX_PENDING_WRITES を上限とする配列の一部として利用されます。
  • 書き込み順序を最適化するため、フラッシュ直前に buffer_cmp を用いて blkno に基づくソートが行われます。

アルゴリズムまたは振る舞い

  • smgr_bulk_write によりページがキューイングされる際、本構造体にバッファとブロック番号が格納されます。
  • smgr_bulk_flush 実行時に、保持している blkno を基準にソートされ、物理的な書き込み順序が決定されます。

呼び出し元と呼び出し先

  • 呼び出し元: smgr_bulk_write（書き込みキューへの追加）、smgr_bulk_flush（ソートおよび書き込み処理）、buffer_cmp（ソート時の比較関数）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • BulkWriteState: 本構造体の配列を保持し、バルク書き込み全体の状態を管理します。
  • BulkWriteBuffer: ページデータを指す型。

失敗モードと境界条件

  • MAX_PENDING_WRITES を超える書き込み要求が発生した場合、smgr_bulk_write 内で自動的にフラッシュ処理がトリガーされます。

保守担当者向けメモ

  • 本構造体は bulk_write.c 内のローカルな管理用データ構造です。
  • ソートアルゴリズム（buffer_cmp）を変更する場合は、blkno の比較ロジックが正しく維持されていることを確認してください。
