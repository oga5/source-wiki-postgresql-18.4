GetCursorNumber

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/connection.c:912

目的

リモートサーバー上で使用するカーソル名の一意な識別番号を生成・取得します。

シグネチャまたは形状

unsigned int GetCursorNumber(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn（リモートサーバーへの接続ハンドル。ただし、関数内部では使用されていません）
  • **出力**: unsigned int（インクリメントされた新しいカーソル番号）

前提条件と不変条件

  • 内部的な静的カウンタ cursor_number をインクリメントして返します。
  • 呼び出しごとに必ず一意な値が返されることが期待されます。

アルゴリズムまたは振る舞い

1. 静的変数 cursor_number を前置インクリメント（++cursor_number）します。 2. インクリメント後の値を戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c:1546
  • contrib/postgres_fdw/postgres_fdw.c:5275
  • **呼び出し先**:
  • 特になし（標準的なインクリメント操作のみを実行）。

関連データ構造

  • PGconn: libpq の接続構造体（本関数では引数として受け取るのみ）。

失敗モードと境界条件

  • 戻り値の型が unsigned int であるため、カウンタが最大値に達した場合のオーバーフローについては考慮されていません。

保守担当者向けメモ

  • 本関数は conn 引数を受け取りますが、実装内では使用していません。将来的な拡張やインターフェースの統一のために保持されている可能性があります。
  • 類似の関数として GetPrepStmtNumber が存在し、そちらはカウンタをリセットしないという特性を持ちます。
