macaddr_ge

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mac.c:236

目的

2つの macaddr 型の値を受け取り、左辺が右辺以上（>=）であるかを判定するSQL比較関数です。

シグネチャまたは形状

Datum macaddr_ge(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの macaddr 型ポインタ（a1, a2）を受け取ります。
  • **出力**: bool 型の Datum を返します。a1 >= a2 であれば true、そうでなければ false です。

前提条件と不変条件

  • 入力される macaddr ポインタは有効である必要があります。
  • 比較ロジックは macaddr_cmp_internal に依存しており、6バイトのMACアドレスを効率的に比較します。

アルゴリズムまたは振る舞い

1. PG_GETARG_MACADDR_P マクロを使用して、引数から2つの macaddr 構造体へのポインタを取得します。 2. macaddr_cmp_internal(a1, a2) を呼び出し、比較結果を取得します。 3. 比較結果が 0 以上（>= 0）であるかを判定し、その結果を PG_RETURN_BOOL で返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_macaddr.c:38（直接的な呼び出しの証拠あり）
  • **呼び出し先**:
  • macaddr_cmp_internal（比較ロジックの実行）

関連データ構造

  • macaddr: 6バイト（EUI-48）のMACアドレスを保持する構造体。

失敗モードと境界条件

  • 引数が NULL の場合、PostgreSQLの関数呼び出し規約に従い、呼び出し元で適切に処理される必要があります。
  • 比較ロジック自体は macaddr_cmp_internal の境界条件（6バイトの全ビット比較）に準拠します。

保守担当者向けメモ

  • 本関数は標準的な比較演算子の一部です。比較ロジックを変更する場合は macaddr_cmp_internal を修正してください。
  • contrib/btree_gist からの参照があるため、インデックス関連の動作変更時には影響範囲を確認してください。
