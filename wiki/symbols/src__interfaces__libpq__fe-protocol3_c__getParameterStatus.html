getParameterStatus

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-protocol3.c:1539

目的

サーバーから送信されたパラメータ名と値のペアを読み取り、PGconn 構造体に保存する。

シグネチャまたは形状

static int getParameterStatus(PGconn *conn);

入力と出力

  • **入力**: PGconn *conn（現在の接続状態を保持する構造体）
  • **出力**: 成功時は 0、失敗時（EOF またはメモリ不足）は EOF を返す。

前提条件と不変条件

  • conn は有効な PGconn オブジェクトであること。
  • 呼び出し時点で、通信バッファ上にパラメータ名および値のメッセージデータが存在していること。

アルゴリズムまたは振る舞い

1. pqGets を使用して、通信バッファからパラメータ名を読み取り conn->workBuffer に格納する。 2. initPQExpBuffer で一時バッファ valueBuf を初期化し、pqGets を使用してパラメータ値を読み取る。 3. 読み取りに成功した場合、pqSaveParameterStatus を呼び出してパラメータ名と値を接続状態に保存する。 4. 保存処理がメモリ不足により失敗した場合は、libpq_append_conn_error でエラーを記録し、handleFatalError を呼び出して接続を終了させる。 5. 最後に termPQExpBuffer で一時バッファを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-protocol3.c 内のメッセージ解析ロジック（pqParseInput3 等のディスパッチ先）。
  • **呼び出し先**:
  • pqGets: バッファからの文字列読み取り。
  • initPQExpBuffer / termPQExpBuffer: 一時バッファの管理。
  • pqSaveParameterStatus: パラメータの保存。
  • libpq_append_conn_error / handleFatalError: エラー処理。

関連データ構造

  • PGconn: 接続状態を管理する主要構造体。
  • PQExpBufferData: パラメータ値の一時保持に使用される動的バッファ。

失敗モードと境界条件

  • **EOF**: pqGets がデータの読み取りに失敗した場合、関数は即座に EOF を返す。
  • **メモリ不足**: pqSaveParameterStatus が失敗した場合、致命的エラーとして接続が切断される。

保守担当者向けメモ

  • 本関数はプロトコル 3.0 専用の処理である。
  • パラメータ値は非常に大きくなる可能性があるため、PQExpBuffer を使用して動的にメモリを確保・解放している。
