deparseDeleteSql

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/deparse.c:2359

目的

リモートの PostgreSQL サーバーで実行するための DELETE 文の SQL 文字列を構築します。

シグネチャまたは形状

void deparseDeleteSql(StringInfo buf, RangeTblEntry *rte, Index rtindex, Relation rel, List *returningList, List **retrieved_attrs)

入力と出力

  • **入力**:
  • buf: SQL 文字列を格納する StringInfo バッファ。
  • rte: 対象テーブルの RangeTblEntry。
  • rtindex: テーブルの範囲テーブルインデックス。
  • rel: 対象テーブルの Relation 記述子。
  • returningList: RETURNING 句の式リスト。
  • **出力**:
  • buf: DELETE FROM ... WHERE ctid = $1 および RETURNING 句が追記されます。
  • retrieved_attrs: RETURNING 句で取得される属性のリストが格納されます。

前提条件と不変条件

  • ctid を使用して削除対象を特定するため、リモートテーブルの ctid が利用可能である必要があります。
  • rel が有効な Relation 構造体であることを前提とします。

アルゴリズムまたは振る舞い

1. DELETE FROM に続けて deparseRelation を呼び出し、対象テーブル名をバッファに追加します。 2. WHERE ctid = $1 を追加し、特定の行を識別します。 3. deparseReturningList を呼び出し、RETURNING 句の構築と取得属性の解決を行います。その際、テーブルに AFTER ROW トリガーが存在するかどうかの判定（rel->trigdesc の確認）を引数として渡します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c:1893
  • **呼び出し先**:
  • deparseRelation
  • deparseReturningList

関連データ構造

  • StringInfo: SQL 文字列構築用バッファ。
  • RangeTblEntry: クエリの範囲テーブルエントリ。
  • Relation: テーブルのメタデータ構造体。

失敗モードと境界条件

  • deparseRelation や deparseReturningList 内部でサポートされていないノード型に遭遇した場合、エラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は ctid を用いた単一行削除を前提としています。
  • RETURNING 句の処理において、リモート側のトリガー状態に依存した構築が行われるため、トリガー定義の変更時には注意が必要です。
