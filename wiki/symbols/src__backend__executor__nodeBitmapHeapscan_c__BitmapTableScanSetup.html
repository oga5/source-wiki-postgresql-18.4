BitmapTableScanSetup

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapHeapscan.c:62

目的

ビットマップヒープスキャンの実行準備を行います。インデックススキャンから得られた TIDBitmap を取得し、ヒープテーブルに対するスキャンを開始するためのイテレータを初期化します。

シグネチャまたは形状

static void BitmapTableScanSetup(BitmapHeapScanState *node)

入力と出力

  • **入力**: BitmapHeapScanState *node（スキャンノードの状態構造体）
  • **出力**: なし（副作用として node 内の tbm、ss.ss_currentScanDesc、initialized フラグを更新）

前提条件と不変条件

  • node->ss.ps.state->es_query_dsa が有効であること。
  • 並列実行時、pstate が存在する場合は共有状態の同期が適切に行われていること。
  • MultiExecProcNode を介して取得される結果が TIDBitmap 型であること。

アルゴリズムまたは振る舞い

1. **ビットマップの取得**:

  • 並列実行でない場合、MultiExecProcNode を呼び出し TIDBitmap を取得。
  • 並列実行の場合、BitmapShouldInitializeSharedState を用いてリーダープロセスがビットマップを生成し、tbm_prepare_shared_iterate で共有イテレータを準備。その後 BitmapDoneInitializingSharedState で他プロセスに通知。

    • 2. **イテレータの開始**: tbm_begin_iterate を呼び出し、ビットマップ走査用のイテレータを作成。 3. **テーブルスキャンの開始**: node->ss.ss_currentScanDesc が未初期化の場合、table_beginscan_bm を呼び出してスキャン記述子を作成。 4. **状態の確定**: 作成したイテレータをスキャン記述子にセットし、node->initialized を true に設定。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の参照に基づくと、nodeBitmapHeapscan.c 内の関数から呼び出されています（具体的な関数名はソース抜粋からは特定不可）。
  • **呼び出し先**: MultiExecProcNode、BitmapShouldInitializeSharedState、tbm_prepare_shared_iterate、BitmapDoneInitializingSharedState、tbm_begin_iterate、table_beginscan_bm。

関連データ構造

  • BitmapHeapScanState: スキャンノードの実行状態を保持。
  • TIDBitmap: 走査対象のタプルID集合。
  • ParallelBitmapHeapState: 並列実行時の共有状態。
  • TBMIterator: ビットマップ走査用イテレータ。

失敗モードと境界条件

  • MultiExecProcNode の戻り値が TIDBitmap ではない場合、elog(ERROR) により異常終了します。
  • table_beginscan_bm の呼び出しは、初回スキャン時のみ実行されます。

保守担当者向けメモ

  • 本関数は initialized フラグを管理しており、二重初期化を防ぐ設計となっています。
  • 並列実行時のリーダーとワーカーの同期ロジック（BitmapShouldInitializeSharedState 等）は、共有メモリ上のイテレータ状態に依存しているため、変更時は並列実行の整合性に注意してください。
