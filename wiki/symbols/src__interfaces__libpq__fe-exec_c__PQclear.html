PQclear

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-exec.c:726

目的

PGresult オブジェクトに関連付けられたメモリおよびリソースを解放し、オブジェクトを破棄します。

シグネチャまたは形状

void PQclear(PGresult *res);

入力と出力

  • **入力**: res (PGresult *) - 解放対象の PGresult オブジェクトへのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • res が NULL の場合、何も行わずに正常終了します。
  • res が静的な OOM_result を指している場合、何も行わずに正常終了します。
  • 呼び出し後、res が指していたメモリ領域は無効となります。

アルゴリズムまたは振る舞い

1. **ガード節**: res が NULL または OOM_result であれば即座に復帰します。 2. **イベント破棄**: res に登録されたイベントがある場合、各イベントに対して PGEVT_RESULTDESTROY を通知し、イベント名を解放します。 3. **イベント配列解放**: イベント管理用配列 res->events を解放します。 4. **データブロック解放**: res->curBlock から始まる連結リストを辿り、すべての補助データブロックを順次解放します。 5. **タプル配列解放**: res->tuples を解放します。 6. **ポインタの無効化**: プログラミングエラーを検知しやすくするため、構造体内の各ポインタフィールドを NULL に設定し、nEvents を 0 にリセットします。 7. **構造体解放**: PGresult 構造体本体を free します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/dblink/dblink.c 等の外部モジュールから頻繁に呼び出されます。
  • **呼び出し先**: 登録されたイベントプロシージャ（res->events[i].proc）を呼び出します。

関連データ構造

  • PGresult: 解放対象となる主要な構造体。
  • PGresult_data: PGresult 内部でメモリを管理するためのブロック構造体。

失敗モードと境界条件

  • NULL ポインタや OOM_result を渡した場合は安全に無視されます。
  • 二重解放（double free）を防ぐための明示的なポインタクリア処理が含まれていますが、呼び出し元で解放済みのポインタを再度渡さないよう注意が必要です。

保守担当者向けメモ

  • PGresult は独自のメモリ管理（PQresultAlloc 等によるブロック割り当て）を行っているため、個別のフィールドを個別に free するのではなく、この関数を通じて一括解放する必要があります。
  • イベント処理の順序は、PGresult の破棄プロセスにおいて重要です。
