pg_database_size_oid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/dbsize.c:167

目的

指定された OID に対応するデータベースの物理的なディスクサイズ（バイト単位）を計算し、SQL インターフェースへ返却します。

シグネチャまたは形状

Datum pg_database_size_oid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介してデータベースの OID を受け取ります。
  • **出力**: データベースの合計サイズを int64 型で返します。サイズが 0 の場合は NULL を返します。

前提条件と不変条件

  • 対象のデータベースがシステムカタログ（pg_database）に存在する必要があります。存在しない場合はエラーを発生させます。
  • 実行ユーザーは、対象データベースに対する適切なアクセス権限（または pg_read_all_stats ロール）を有している必要があります（権限チェックは内部で呼び出される関数に依存します）。

アルゴリズムまたは振る舞い

1. SearchSysCacheExists1 を使用して、指定された OID のデータベースが存在するかを確認します。存在しない場合は ERRCODE_UNDEFINED_OBJECT でエラーを報告します。 2. calculate_database_size(dbOid) を呼び出し、対象データベースの全テーブルスペース上の物理サイズを集計します。 3. 集計結果が 0 の場合は PG_RETURN_NULL() を返します。 4. それ以外の場合は、計算されたサイズを PG_RETURN_INT64(size) で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェースから直接呼び出されることを想定しています。
  • **呼び出し先**:
  • SearchSysCacheExists1 (システムカタログ検索)
  • calculate_database_size (サイズ計算の実行)
  • ereport (エラー報告)

関連データ構造

  • Oid: データベース識別子。
  • int64: サイズの保持に使用。

失敗モードと境界条件

  • **データベース不在**: 指定された OID が存在しない場合、エラーが発生します。
  • **サイズ 0**: データベースが空、またはアクセス権限等の理由でサイズが 0 と判定された場合、NULL が返されます。

保守担当者向けメモ

  • 本関数は calculate_database_size のラッパーとして機能します。サイズ計算ロジックの変更は calculate_database_size 側で行ってください。
  • 権限チェックの詳細は calculate_database_size の実装に依存します。
