extract_nonindex_conditions

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/costsize.c:849

目的

インデックスによって既に処理される条件を除外し、インデックススキャン後にフィルタリング（再評価）が必要な条件（qual_clauses）を抽出する。

シグネチャまたは形状

static List *extract_nonindex_conditions(List *qual_clauses, List *indexclauses)

入力と出力

  • **入力**:
  • qual_clauses: 評価対象となる RestrictInfo リスト。
  • indexclauses: インデックスによって処理されることが確定している RestrictInfo リスト。
  • **出力**: インデックス条件と重複せず、かつ擬似定数（pseudoconstant）ではない条件を含む List。

前提条件と不変条件

  • qual_clauses 内の各要素は RestrictInfo 型である必要がある。
  • 擬似定数は結果から除外される。
  • is_redundant_with_indexclauses により、インデックス条件と等価または派生関係にある条件は除外される。

アルゴリズムまたは振る舞い

1. qual_clauses を反復処理する。 2. 各 RestrictInfo について以下を判定する：

  • pseudoconstant フラグが立っている場合はスキップする。
  • is_redundant_with_indexclauses を呼び出し、インデックス条件と重複する場合はスキップする。

    • 3. 上記のいずれにも該当しない条件を result リストに追加し、最終的に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/costsize.c 内の複数箇所（インデックスパスのコスト計算時）。
  • src/backend/optimizer/plan/createplan.c:3066（実行計画生成時）。
  • **呼び出し先**:
  • is_redundant_with_indexclauses（重複判定）。
  • lfirst_node, lappend（リスト操作）。

関連データ構造

  • RestrictInfo: クエリ内の制約条件を保持する構造体。
  • List: PostgreSQLの汎用リスト構造。

失敗モードと境界条件

  • 入力リストが空の場合、空のリストを返す。
  • すべての条件がインデックスでカバーされている場合、空のリストを返す。

保守担当者向けメモ

  • 本関数は、インデックススキャン後に適用すべき「残りのフィルタ条件」を特定する重要な役割を担う。
  • is_redundant_with_indexclauses の判定ロジックに依存しているため、等価クラス（EquivalenceClass）の扱いに変更がある場合は注意が必要。
