pgstat_drop_relation

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_relation.c:178

目的

リレーションの破棄に伴い、統計システムから当該リレーションの追跡情報を削除し、現在のトランザクション内での統計カウンタをリセットする。

シグネチャまたは形状

void pgstat_drop_relation(Relation rel)

入力と出力

  • **入力**: rel (破棄対象のリレーションへのポインタ)
  • **出力**: なし（副作用として統計状態を更新）

前提条件と不変条件

  • rel は有効なリレーション構造体であること。
  • pgstat_should_count_relation(rel) が真の場合、リレーションの統計情報はトランザクションレベルで管理されている必要がある。

アルゴリズムまたは振る舞い

1. pgstat_drop_transactional を呼び出し、共有統計システムから当該リレーションの情報を削除する（共有か否かに応じてデータベースIDを判定）。 2. pgstat_should_count_relation を確認し、追跡対象外であれば即座に終了する。 3. 現在のトランザクションネストレベルを取得する。 4. リレーションの pgstat_info を参照し、現在のネストレベルと一致するトランザクション統計情報が存在する場合：

  • save_truncdrop_counters を呼び出し、現在のカウンタ値を保存する。
  • 挿入、更新、削除の各カウンタを 0 にリセットする。これにより、トランザクション内の pg_stat_xact_all_tables 参照時に 0 が返されることを保証する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c:1899
  • src/backend/catalog/index.c:2329
  • **呼び出し先**:
  • pgstat_drop_transactional
  • pgstat_should_count_relation
  • save_truncdrop_counters
  • GetCurrentTransactionNestLevel
  • RelationGetRelid

関連データ構造

  • Relation: リレーションのメタデータと統計情報 (pgstat_info) を保持。
  • PgStat_TableStatus: リレーションの統計状態を管理する構造体。

失敗モードと境界条件

  • pgstat_should_count_relation が偽を返す場合、カウンタのリセット処理はスキップされる。
  • 現在のネストレベルと pgstat_info->trans->nest_level が一致しない場合、カウンタのリセットは行われない。

保守担当者向けメモ

  • 本関数はリレーションの破棄時に呼ばれるため、トランザクションのロールバックやコミットの整合性を維持する重要な役割を担う。
  • save_truncdrop_counters によるカウンタ保存は、TRUNCATEやDROP後の統計の一貫性を保つための特殊な処理である点に留意すること。
