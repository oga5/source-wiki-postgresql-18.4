llvm_mutable_module

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit.c:315

目的

JIT コンパイル対象となる LLVM モジュールを取得、または必要に応じて新規作成する。

シグネチャまたは形状

LLVMModuleRef llvm_mutable_module(LLVMJitContext *context)

入力と出力

  • **入力**: LLVMJitContext *context - JIT コンパイルのコンテキスト状態を保持する構造体。
  • **出力**: LLVMModuleRef - 現在のコンテキストに関連付けられた、変更可能な LLVM モジュールへの参照。

前提条件と不変条件

  • llvm_assert_in_fatal_section() により、致命的なエラーセクション内であることが検証される。
  • context->module が存在しない場合、新しいモジュールが生成され、context->compiled は false に設定される。

アルゴリズムまたは振る舞い

1. context->module が既に存在するかを確認する。 2. 存在しない場合:

  • context->compiled を false に設定する。
  • グローバルな llvm_generation をインクリメントし、context->module_generation に割り当てる。
  • LLVMModuleCreateWithNameInContext を用いて "pg" という名前の新規モジュールを作成する。
  • LLVMSetTarget および LLVMSetDataLayout を呼び出し、グローバルなターゲットとデータレイアウトをモジュールに適用する。

    • 3. 存在するモジュール、または新規作成したモジュールを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit_deform.c:102
  • src/backend/jit/llvm/llvmjit_expr.c:156
  • **呼び出し先**:
  • llvm_assert_in_fatal_section
  • LLVMModuleCreateWithNameInContext
  • LLVMSetTarget
  • LLVMSetDataLayout

関連データ構造

  • LLVMJitContext: JIT コンパイルのライフサイクルとモジュール状態を管理する構造体。

失敗モードと境界条件

  • LLVM API 内部でメモリ不足や不正な状態が発生した場合、LLVM のエラーハンドリング機構を通じて PostgreSQL のエラー処理系へ伝播する可能性がある。

保守担当者向けメモ

  • 本関数は、JIT コンパイルのパイプラインにおいてモジュールが「変更可能（mutable）」な状態であることを保証するゲートウェイである。
  • context->module_generation は、モジュールの世代管理に使用されるため、変更時には整合性に注意すること。
