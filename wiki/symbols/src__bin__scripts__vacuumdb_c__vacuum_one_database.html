vacuum_one_database

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/vacuumdb.c:523

目的

指定されたデータベースに対して VACUUM または ANALYZE 操作を実行する。サーバーバージョンに応じたオプションのバリデーション、対象オブジェクトの取得、および必要に応じた並列処理の準備を行う。

シグネチャまたは形状

static void vacuum_one_database(
    ConnParams *cparams,
    vacuumingOptions *vacopts,
    int stage,
    SimpleStringList *objects,
    SimpleStringList **found_objs,
    int concurrentCons,
    const char *progname,
    bool echo,
    bool quiet
)

入力と出力

  • **入力**:
  • cparams: データベース接続パラメータ。
  • vacopts: 実行オプション（vacuumingOptions 構造体）。
  • stage: 分析ステージ（ANALYZE_NO_STAGE 等）。
  • objects: 処理対象オブジェクトのリスト。
  • found_objs: 取得済みオブジェクトリストの参照（出力兼用）。
  • concurrentCons: 並列接続数。
  • progname, echo, quiet: 実行時設定。
  • **出力**: なし（副作用としてデータベースに対する SQL 実行および found_objs への結果格納を行う）。

前提条件と不変条件

  • stage は ANALYZE_NO_STAGE であるか、有効なステージ範囲内であること（Assert による検証）。
  • 接続先のサーバーバージョンが、指定された vacopts の各オプションをサポートしていること。

アルゴリズムまたは振る舞い

1. connectDatabase を呼び出し、対象データベースへ接続する。 2. サーバーバージョンを確認し、vacopts で指定された各オプション（--skip-locked, --parallel 等）がサポートされていない場合はエラー終了する。 3. サーバーバージョンが 16.0 以上であれば vacopts->skip_database_stats を有効化する。 4. found_objs が提供されていない場合、retrieve_objects を呼び出して対象オブジェクトリストを取得する。 5. 対象オブジェクトが存在しない場合は早期リターンする。 6. concurrentCons をオブジェクト数に合わせて調整する。 7. stage が指定されている場合、対応する stage_commands を実行してセッションパラメータを初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/scripts/vacuumdb.c 内の main 関数およびデータベースループ処理箇所から呼び出される。
  • **呼び出し先**: connectDatabase, PQserverVersion, PQfinish, pg_fatal, retrieve_objects, executeCommand 等。

関連データ構造

  • vacuumingOptions: 実行オプションを保持。
  • SimpleStringList: 処理対象オブジェクトのリスト。
  • ConnParams: 接続情報。

失敗モードと境界条件

  • **バージョン不整合**: 指定されたオプションが接続先サーバーのバージョンでサポートされていない場合、pg_fatal により終了する。
  • **接続失敗**: connectDatabase が失敗した場合、処理は継続されない。
  • **対象なし**: 処理対象のテーブルやオブジェクトが 0 件の場合、何も実行せずに終了する。

保守担当者向けメモ

  • 本関数は vacuumdb の中心的な実行ロジックであり、サーバーバージョンごとの機能制限がハードコードされている。新しい VACUUM オプションを追加する際は、ここでのバージョンチェックの追加が必要。
  • stage 処理は ANALYZE の段階的実行を制御しており、stage_commands 配列と密結合している。
