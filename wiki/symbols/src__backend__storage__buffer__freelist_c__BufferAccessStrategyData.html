BufferAccessStrategyData

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/buffer/freelist.c:72

目的

特定の操作（スキャンやバルク処理など）において、共有バッファプール全体を汚染することなく再利用するための「リングバッファ」構造を保持する。

シグネチャまたは形状

typedef struct BufferAccessStrategyData {
    BufferAccessStrategyType btype;
    int         nbuffers;
    int         current;
    Buffer      buffers[FLEXIBLE_ARRAY_MEMBER];
} BufferAccessStrategyData;

入力と出力

  • **入力**: 構造体生成時に nbuffers（リング内のバッファ数）が決定される。
  • **出力**: buffers 配列を通じて、現在割り当てられているバッファの識別子（Buffer）を保持する。

前提条件と不変条件

  • buffers 配列は FLEXIBLE_ARRAY_MEMBER を使用しており、構造体本体と連続したメモリ領域として palloc される。
  • buffers 内の InvalidBuffer（値: 0）は、そのスロットがまだ選択されていないことを示す。

アルゴリズムまたは振る舞い

  • リングバッファとして機能し、current インデックスを用いて次に再利用すべきバッファを追跡する。
  • 大規模なスキャン時に、この構造体で管理されるバッファのみを循環させることで、キャッシュ効率を維持する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/freelist.c 内のメモリ割り当て関数（palloc0 を使用して生成）。
  • **呼び出し先**: 情報なし。
  • **参照**: src/include/storage/buf.h にて BufferAccessStrategy 型として定義され、freelist.c 外部からも参照される。

関連データ構造

  • BufferAccessStrategyType: 戦略の種類（BAS_BULKREAD, BAS_BULKWRITE 等）を定義。
  • Buffer: バッファ識別子。

失敗モードと境界条件

  • nbuffers が 0 の場合、リングバッファとしての機能は無効となる。
  • メモリ割り当て時に palloc が失敗した場合はプロセスが終了する。

保守担当者向けメモ

  • 本構造体は freelist.c に対してプライベートな実装詳細であるため、直接的な操作は freelist.c 内の関数を通じて行うこと。
  • buffers 配列のサイズは動的であるため、構造体へのアクセス時には offsetof を用いた計算が推奨される。
