hashint2

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hashfunc.c:59

目的

int16 型の値をハッシュ値に変換し、PostgreSQL のハッシュアクセスメソッドやハッシュ結合で使用可能な形式で提供します。

シグネチャまたは形状

Datum hashint2(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを介して渡される int16 型の SQL 引数。
  • **出力**: Datum 型として返される 32 ビットのハッシュ値。

前提条件と不変条件

  • 入力値は int16 として取得されます。
  • 等価な入力値に対しては、常に同一のハッシュ値を返す必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT16(0) を使用して、第 1 引数を int16 型として取得します。 2. 取得した値を int32 型にキャストします。 3. hash_uint32 関数を呼び出し、その結果を戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し先**: hash_uint32
  • **呼び出し元**: ソースコード内の直接的な呼び出し元情報は提供されていませんが、本関数は pg_amproc システムカタログを通じてハッシュ演算子クラスから呼び出される設計です。

関連データ構造

  • Datum: PostgreSQL の汎用データ型表現。
  • PG_FUNCTION_ARGS: 関数呼び出しの引数コンテキスト。

失敗モードと境界条件

  • 特記事項なし。int16 の全範囲を int32 にキャストして処理するため、型変換による情報欠落は発生しません。

保守担当者向けメモ

  • 本関数は int16 用の標準ハッシュ関数です。ハッシュ結合のパフォーマンス最適化が必要な場合は、シード値を受け取る hashint2extended の実装と併せて管理してください。
  • hashint4 とは論理的に等価な入力に対して同一のハッシュ値を生成するよう設計されています。
