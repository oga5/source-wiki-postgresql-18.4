CountDBSubscriptions

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_subscription.c:153

目的

指定されたデータベース OID (dbid) に紐付いているサブスクリプションの総数をカウントします。

シグネチャまたは形状

int CountDBSubscriptions(Oid dbid)

入力と出力

  • **入力**: dbid (Oid) - カウント対象のデータベース識別子。
  • **出力**: int - 指定されたデータベースに関連付けられたサブスクリプションの数。

前提条件と不変条件

  • pg_subscription システムカタログに対する読み取り権限が必要です。
  • table_open(SubscriptionRelationId, RowExclusiveLock) を呼び出すため、呼び出し元は適切なトランザクションコンテキストに存在する必要があります。

アルゴリズムまたは振る舞い

1. pg_subscription リレーションを RowExclusiveLock モードで開きます。 2. subdbid カラムが引数の dbid と一致するタプルを検索するためのスキャンキーを初期化します。 3. systable_beginscan を使用してシステムテーブルスキャンを開始します。 4. systable_getnext をループで呼び出し、一致するタプルが見つかるたびにカウンタ nsubs をインクリメントします。 5. スキャンを終了し、リレーションをクローズしてカウント値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/dbcommands.c:1767 (データベース削除時のチェック等で使用)
  • **呼び出し先**:
  • table_open / table_close
  • ScanKeyInit
  • systable_beginscan / systable_getnext / systable_endscan

関連データ構造

  • pg_subscription: サブスクリプション情報を保持するシステムカタログ。
  • Relation: システムカタログへのアクセスに使用するリレーション構造体。
  • ScanKeyData: スキャン条件を保持する構造体。

失敗モードと境界条件

  • 指定された dbid に該当するサブスクリプションが存在しない場合、0 を返します。
  • table_open 失敗時やカタログアクセス時のエラーは、PostgreSQL の標準的なエラーハンドリング機構により例外として処理されます。

保守担当者向けメモ

  • 本関数は RowExclusiveLock を取得しますが、読み取りのみを行うため、必要に応じてロックレベルの最適化（AccessShareLock への変更など）が可能か検討してください。
  • systable_beginscan の第2引数に InvalidOid を渡しているため、インデックスを使用しないシーケンシャルスキャンが行われます。サブスクリプション数が多い環境で頻繁に呼び出される場合は、インデックススキャンの利用を検討してください。
