BloomBuildState

定義

  • 種別: struct
  • 定義箇所: contrib/bloom/blinsert.c:34

目的

Bloom 索引の構築（Build）処理中、一時的な状態を保持するために使用される構造体です。ページキャッシュの管理やメモリコンテキストの制御を行い、インデックス構築の効率化を担います。

シグネチャまたは形状

typedef struct {
    BloomState      blstate;      /* bloom index state */
    int64           indtuples;    /* total number of tuples indexed */
    MemoryContext   tmpCtx;       /* temporary memory context */
    PGAlignedBlock  data;         /* cached page */
    int             count;        /* number of tuples in cached page */
} BloomBuildState;

入力と出力

本構造体は関数引数として渡され、構築プロセス全体を通じて状態が更新されます。

  • **入力**: blbuild 関数内で初期化され、bloomBuildCallback を通じて各タプルの処理に利用されます。
  • **出力**: 構築完了後、保持していたページキャッシュがディスクへフラッシュされ、メモリコンテキストが破棄されます。

前提条件と不変条件

  • tmpCtx は、タプル処理ごとにリセットされるメモリコンテキストとして管理される必要があります。
  • data（キャッシュされたページ）は、count が上限に達した際に flushCachedPage を通じてディスクへ書き出される必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: blbuild にて BloomBuildState が生成され、インデックス構築の準備が行われます。 2. **タプル処理**: bloomBuildCallback が呼び出されるたびに、タプルがキャッシュページに追加されます。 3. **フラッシュ**: キャッシュがいっぱいになると flushCachedPage が呼び出され、ページが物理ストレージへ書き出されます。 4. **終了**: 全タプルの処理後、残りのキャッシュがフラッシュされ、構築が完了します。

呼び出し元と呼び出し先

  • **呼び出し元**: blbuild（contrib/bloom/blinsert.c:126）にてインスタンス化されます。
  • **呼び出し先**:
  • flushCachedPage（contrib/bloom/blinsert.c:48）
  • initCachedPage（contrib/bloom/blinsert.c:65）
  • bloomBuildCallback（contrib/bloom/blinsert.c:78）

関連データ構造

  • BloomState: インデックスの基本状態を保持します。
  • PGAlignedBlock: ページデータを保持するためのアライメントされたメモリブロックです。
  • MemoryContext: 構築中のメモリリークを防ぐための管理単位です。

失敗モードと境界条件

  • メモリ不足: tmpCtx の管理が不適切な場合、大量のタプル処理時にメモリリークが発生する可能性があります。
  • ページ溢れ: count がページ容量を超えないよう、flushCachedPage による適切なフラッシュタイミングの制御が不可欠です。

保守担当者向けメモ

  • 本構造体は GenericXLog を使用したページ書き込みと密接に関連しています。変更を加える際は、WAL ログの整合性に注意してください。
  • tmpCtx のリセットタイミングは、メモリ効率を維持するために重要です。タプル処理ごとのリセットが正しく行われているか確認してください。
