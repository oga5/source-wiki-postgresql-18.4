TidStoreIter

定義

  • 種別: struct
  • 定義箇所: src/backend/access/common/tidstore.c:135 および src/include/access/tidstore.h:21

目的

TidStore に格納された全 TID（ItemPointerData）を順次走査するためのイテレータ状態を保持する構造体です。

シグネチャまたは形状

struct TidStoreIter
{
    TidStore   *ts;
    union
    {
        shared_ts_iter *shared;
        local_ts_iter *local;
    } tree_iter;
    TidStoreIterResult output;
};

入力と出力

  • **入力**: TidStoreBeginIterate を通じて初期化され、対象となる TidStore へのポインタを保持します。
  • **出力**: TidStoreIterateNext を呼び出すことで、走査結果が output フィールド（TidStoreIterResult 型）に格納されます。

前提条件と不変条件

  • TidStore が共有メモリ上にあるかローカルメモリ上にあるかに応じて、tree_iter 内の適切なイテレータ（shared または local）が使用される必要があります。
  • TidStoreIterateNext の呼び出しにより、内部のラディックツリー走査状態が更新されます。

アルゴリズムまたは振る舞い

1. TidStoreBeginIterate により、呼び出し側のメモリコンテキストで TidStoreIter が palloc0 されます。 2. TidStoreIterateNext が呼び出されるたびに、内部のラディックツリーイテレータが次のブロックおよびオフセットを検索します。 3. 走査が完了すると、TidStoreEndIterate を呼び出してリソースを解放する必要があります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/tidstore.c（TidStoreBeginIterate, TidStoreIterateNext, TidStoreEndIterate）
  • src/backend/access/heap/vacuumlazy.c（VACUUM 処理中のコールバック内）
  • src/test/modules/test_tidstore/test_tidstore.c
  • **呼び出し先**:
  • 内部的に lib/radixtree.h のイテレータ機能を利用します。

関連データ構造

  • TidStore: 走査対象となるデータストア。
  • TidStoreIterResult: 走査結果を格納する構造体。
  • shared_ts_iter / local_ts_iter: 内部で使用されるラディックツリーのイテレータ型。

失敗モードと境界条件

  • TidStore が空の場合、イテレータは即座に終了状態を返します。
  • 共有メモリ版の TidStore を使用する場合、適切なロックが保持されていない状態で走査を行うと未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • TidStoreIter は palloc0 で確保されるため、使用後は必ず TidStoreEndIterate を呼び出してメモリリークを防いでください。
  • tree_iter は共用体であるため、TidStoreIsShared() の判定結果に基づいて正しくアクセスする必要があります。
