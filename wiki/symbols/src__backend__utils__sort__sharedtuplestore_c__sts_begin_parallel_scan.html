sts_begin_parallel_scan

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/sharedtuplestore.c:252

目的

共有タプルストアに対する並列スキャンを開始するための初期化処理を行います。

シグネチャまたは形状

void sts_begin_parallel_scan(SharedTuplestoreAccessor *accessor)

入力と出力

  • **入力**: SharedTuplestoreAccessor *accessor - 操作対象の共有タプルストアへのアクセス用構造体。
  • **出力**: なし（副作用として accessor の内部状態が更新されます）。

前提条件と不変条件

  • すべての参加バックエンドが sts_end_write() を呼び出し、バッファのフラッシュとファイル書き込みの停止が完了している必要があります。
  • 実行時に accessor->sts->participants[i].writing がすべて false であることがアサーションにより検証されます。

アルゴリズムまたは振る舞い

1. sts_end_parallel_scan(accessor) を呼び出し、既存のスキャン状態があれば終了させます。 2. 全参加者の書き込み状態が終了していることを確認します。 3. スキャン開始時の読み取り位置を、当該バックエンド自身が書き込んだファイル（accessor->participant）に設定します。これはキャッシュの局所性を考慮した最適化です。 4. read_file を NULL に、read_next_page を 0 に初期化し、スキャン開始準備を整えます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeHash.c
  • src/backend/executor/nodeHashjoin.c
  • **呼び出し先**:
  • sts_end_parallel_scan

関連データ構造

  • SharedTuplestoreAccessor: スキャン状態（読み取り対象の参加者、ファイル、ページ位置）を保持します。
  • SharedTuplestore: 参加者リストと書き込み状態を管理する共有構造体。

失敗モードと境界条件

  • 参加者がまだ書き込み中の場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数はスキャンを開始するだけであり、実際のタプル取得は後続の sts_parallel_scan_next で行われます。
  • 読み取り順序は保証されません。並列スキャンは全参加者の全タプルを読み出すための機構です。
