DetachSession

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/common/session.c:200

目的

現在のバックエンドプロセスが保持しているセッション共有メモリ（DSMセグメントおよびDSA領域）から明示的にデタッチし、リソースを解放します。

シグネチャまたは形状

void DetachSession(void)

入力と出力

  • **入力**: なし（グローバル変数 CurrentSession を参照）
  • **出力**: なし（副作用として共有メモリのデタッチを実行）

前提条件と不変条件

  • CurrentSession が初期化済みであること。
  • 本関数実行後、CurrentSession->segment および CurrentSession->area は NULL に設定される。

アルゴリズムまたは振る舞い

1. 登録されているデタッチフックを実行する。 2. dsm_detach を呼び出し、CurrentSession->segment が指す動的共有メモリセグメントをデタッチする。 3. CurrentSession->segment を NULL に更新する。 4. dsa_detach を呼び出し、CurrentSession->area が指す動的共有メモリ領域をデタッチする。 5. CurrentSession->area を NULL に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c:1582
  • **呼び出し先**:
  • dsm_detach (外部関数)
  • dsa_detach (外部関数)

関連データ構造

  • CurrentSession: セッション情報を保持するグローバル構造体。
  • dsm_segment: 動的共有メモリセグメント。
  • dsa_area: 動的共有メモリ領域。

失敗モードと境界条件

  • CurrentSession が NULL の状態で呼び出された場合の動作については、ソースコード上では明示的なガード条件は確認できません。

保守担当者向けメモ

  • 本関数はバックエンド終了時、または並列クエリ実行後のクリーンアップ時に呼び出されます。
  • dsm_pin_mapping を使用しているため、明示的にこの関数を呼び出さない限り、バックエンドの生存期間中はメモリが保持されます。
