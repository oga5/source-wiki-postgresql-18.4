GucAction

定義

  • 種別: enum
  • 定義箇所: src/include/utils/guc.h:200

目的

PostgreSQLのGUC（Grand Unified Configuration）システムにおいて、set_config_option 関数が実行する設定変更操作の種類を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
    GUC_ACTION_SET,    /* 通常の SET コマンド */
    GUC_ACTION_LOCAL,  /* SET LOCAL コマンド */
    GUC_ACTION_SAVE,   /* 関数内での SET または一時的な割り当て */
} GucAction;

入力と出力

  • 本シンボルは、設定変更関数（set_config_option 等）の引数として入力されます。
  • 出力はありません。

前提条件と不変条件

  • 設定変更のコンテキスト（GucContext）やソース（GucSource）と組み合わせて、設定値の適用優先順位や永続性を決定するロジックの一部として機能します。

アルゴリズムまたは振る舞い

  • GUC_ACTION_SET: 通常のセッション設定変更に使用されます。
  • GUC_ACTION_LOCAL: トランザクション終了時にロールバックされる SET LOCAL 操作に使用されます。
  • GUC_ACTION_SAVE: 関数実行中の一時的な設定変更や、内部的な値の退避処理に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • src/backend/utils/fmgr/fmgr.c: GUC_ACTION_SAVE を指定して呼び出し。
  • src/backend/utils/misc/guc.c: push_old_value 等の内部関数で引数として使用。
  • src/backend/utils/misc/guc_funcs.c: SET コマンドの種類に応じて GUC_ACTION_SET または GUC_ACTION_LOCAL を選択。
  • **呼び出し先**:
  • 本シンボル自体は列挙型であり、関数ではないため呼び出し先はありません。

関連データ構造

  • GucContext: 設定変更が許可されるタイミングを定義。
  • GucSource: 設定値の優先順位を定義。
  • config_generic: GUC変数の基底構造体。

失敗モードと境界条件

  • 該当なし（列挙型のため）。

保守担当者向けメモ

  • GucAction の値は、guc.c 内の push_old_value や set_config_option のロジックにおいて、古い値を保存すべきか、あるいはトランザクション境界で復元すべきかを判断する重要なフラグとして機能します。新しいアクションタイプを追加する場合は、これらの関数内のスイッチ文を更新する必要があります。
