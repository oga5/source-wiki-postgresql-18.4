check_locale

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale.c:300

目的

指定されたロケール名がシステムで有効かどうかを検証し、必要に応じて正規化されたロケール名を取得します。

シグネチャまたは形状

bool check_locale(int category, const char *locale, char **canonname);

入力と出力

  • **入力**:
  • category: setlocale に渡すロケールカテゴリ（例: LC_COLLATE, LC_CTYPE）。
  • locale: 検証対象のロケール名文字列。
  • canonname: 正規化されたロケール名を格納するためのポインタ（不要な場合は NULL）。
  • **出力**:
  • 戻り値: ロケールが有効であれば true、無効または非ASCII文字が含まれる場合は false。
  • canonname: 成功時に正規化されたロケール名が割り当てられます（呼び出し元で pfree が必要）。

前提条件と不変条件

  • 入力された locale 文字列および取得された正規化ロケール名に非ASCII文字が含まれる場合、検証は失敗します（Windows環境への配慮）。
  • 処理の前後で、元のロケール設定が復元されることが保証されます。

アルゴリズムまたは振る舞い

1. 入力 locale が非ASCII文字を含むかチェックし、含む場合は警告を出して false を返します。 2. 現在のロケール設定を setlocale(category, NULL) で保存します。 3. setlocale(category, locale) を呼び出し、システムがそのロケールを受け入れるか確認します。 4. canonname が指定されている場合、成功したロケール名を pstrdup で保存します。 5. 手順2で保存した元のロケールを setlocale で復元します。 6. 手順4で取得した正規化名が非ASCII文字を含む場合、警告を出してメモリを解放し、false を返します。 7. 手順3の結果が有効であれば true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/dbcommands.c: データベース作成時のロケール検証。
  • src/backend/utils/adt/pg_locale.c: GUC 変数（lc_monetary, lc_numeric, lc_time, lc_messages）の割り当て検証。
  • **呼び出し先**:
  • setlocale (libc)
  • pg_is_ascii
  • pstrdup, pfree
  • ereport, elog

関連データ構造

  • なし（標準的な C 文字列と setlocale カテゴリ定数を使用）。

失敗モードと境界条件

  • setlocale が失敗した場合、関数は false を返します。
  • 非ASCII文字が含まれるロケール名は、OSの制限により拒否されます。
  • 元のロケールの復元に失敗した場合、elog(WARNING) が発生します。

保守担当者向けメモ

  • 本関数は Windows 環境での非 ASCII ロケール名に起因する問題を回避するためのガードレールを含んでいます。
  • canonname を使用する場合、呼び出し元で必ず pfree を行う必要があります。
