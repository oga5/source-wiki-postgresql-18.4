getJsonbOffset

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_util.c:133

目的

JsonbContainer 内の指定されたインデックスにある要素の物理的なオフセットを算出します。

シグネチャまたは形状

uint32 getJsonbOffset(const JsonbContainer *jc, int index)

入力と出力

  • **入力**:
  • jc: 対象となる JsonbContainer 構造体へのポインタ。
  • index: オフセットを求めたい要素のインデックス（0 始まり）。
  • **出力**:
  • 指定されたインデックスの要素が開始する位置を示す uint32 型のオフセット値。

前提条件と不変条件

  • jc->children 配列が有効であること。
  • index は jc 内の有効な要素範囲内であること。

アルゴリズムまたは振る舞い

1. 指定された index より前の要素から逆方向に走査を開始します。 2. 各要素の長さ（JBE_OFFLENFLD）を累積加算します。 3. JBE_HAS_OFF マクロで判定される「オフセット情報を持つエントリ」に到達した時点で走査を終了し、それまでの累積値を返します。 4. このロジックは、特定の要素の開始オフセットが、直前のエントリの終了オフセットに依存するという jsonb のバイナリ構造に基づいています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonb_util.c 内の複数の箇所（インデックス指定による要素アクセスやイテレータの現在位置計算など）。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありませんが、JBE_OFFLENFLD および JBE_HAS_OFF マクロを使用して jc->children の各要素を評価します。

関連データ構造

  • JsonbContainer: jsonb のバイナリ表現を保持する構造体。
  • JEntry: JsonbContainer 内の各要素の長さやオフセット情報を格納する型。

失敗モードと境界条件

  • index が 0 の場合、ループは実行されず 0 が返されます。
  • index が範囲外の場合の挙動は定義されておらず、メモリ破壊や不正なオフセット値の算出を招く可能性があります。

保守担当者向けメモ

  • 本関数は jsonb のバイナリ構造に強く依存しています。jsonb の物理レイアウト（JEntry 配列の配置）に変更を加える場合は、本関数のオフセット計算ロジックが整合性を保っているか確認してください。
