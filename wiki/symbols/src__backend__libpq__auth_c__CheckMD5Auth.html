CheckMD5Auth

定義

  • **種別**: function
  • **定義箇所**: src/backend/libpq/auth.c:882

目的

MD5ハッシュベースの認証プロトコルを実行し、クライアントから提供されたパスワードを検証する。

シグネチャまたは形状

static int CheckMD5Auth(Port *port, char *shadow_pass, const char **logdetail)

入力と出力

  • **入力**:
  • port: 接続状態を保持する Port 構造体へのポインタ。
  • shadow_pass: データベースに保存されているユーザーのパスワードハッシュ。
  • logdetail: 認証失敗時の詳細情報を格納するためのポインタ。
  • **出力**:
  • int: 認証結果（STATUS_OK, STATUS_ERROR, または STATUS_EOF）。

前提条件と不変条件

  • pg_strong_random による4バイトのソルト生成が成功する必要がある。
  • shadow_pass が NULL の場合、認証は即座に STATUS_ERROR となる。

アルゴリズムまたは振る舞い

1. pg_strong_random を使用して4バイトのランダムなソルト（md5Salt）を生成する。生成失敗時は LOG レベルでエラーを記録し STATUS_ERROR を返す。 2. sendAuthRequest を呼び出し、AUTH_REQ_MD5 型の認証要求と生成したソルトをクライアントへ送信する。 3. recv_password_packet を呼び出し、クライアントからのパスワード応答を待機する。応答がない場合は STATUS_EOF を返す。 4. md5_crypt_verify を呼び出し、提供されたパスワード、ソルト、および shadow_pass を照合する。 5. 取得したパスワードバッファを pfree で解放し、検証結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth.c 内の認証フロー（auth_result = CheckMD5Auth(...)）。
  • **呼び出し先**:
  • pg_strong_random
  • ereport
  • sendAuthRequest
  • recv_password_packet
  • md5_crypt_verify
  • pfree

関連データ構造

  • Port: 接続情報を保持する構造体。
  • STATUS_OK, STATUS_ERROR, STATUS_EOF: 認証結果を示す定数。

失敗モードと境界条件

  • **ソルト生成失敗**: pg_strong_random が失敗した場合、認証は中断される。
  • **クライアント応答なし**: recv_password_packet が NULL を返した場合、STATUS_EOF を返す。
  • **パスワード不一致**: md5_crypt_verify が検証に失敗した場合、STATUS_ERROR を返す。
  • **無効な shadow_pass**: shadow_pass が NULL の場合、検証を行わず STATUS_ERROR を返す。

保守担当者向けメモ

  • 本関数は libpq の認証ディスパッチャーの一部として機能する。
  • 認証失敗時の詳細は logdetail を通じて呼び出し元に伝達されるため、セキュリティ上の理由からクライアントには詳細なエラー内容を直接返さないよう注意すること。
