TSVector

定義

  • 種別: typedef
  • 定義箇所: src/include/tsearch/ts_type.h:98

目的

PostgreSQL の全文検索機能において、検索対象となるドキュメントのバイナリ表現を保持する TSVectorData 構造体へのポインタ型です。

シグネチャまたは形状

typedef TSVectorData *TSVector;

入力と出力

  • 入力: TSVectorData 構造体へのポインタ。
  • 出力: TSVector 型（TSVectorData *）。

前提条件と不変条件

  • TSVectorData は可変長配列（FLEXIBLE_ARRAY_MEMBER）を含むため、メモリレイアウトはヘッダー、WordEntry 配列、文字列データ、位置情報ベクトルの順に連続して配置される必要があります。
  • 構造体のサイズ計算には CALCDATASIZE や COMPUTESIZE マクロを使用する必要があります。

アルゴリズムまたは振る舞い

  • TSVector は、PostgreSQL の Datum 型と内部構造体との仲介を行う fmgr マクロ（PG_GETARG_TSVECTOR 等）を通じて、TOAST 解除や型キャストを伴うデータアクセスに使用されます。
  • ARRPTR() や STRPTR() マクロを用いることで、メモリ上の特定のセクション（語彙や位置情報）へ直接アクセス可能です。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/tsearch/to_tsany.c: 複数の関数内で TSVector 型の変数宣言および palloc0 によるメモリ確保に使用されています。
  • 呼び出し先:
  • 提供されたソース excerpt には、この型が呼び出す具体的な関数や、この型を呼び出す関数の詳細な制御フローは含まれていません。

関連データ構造

  • TSVectorData: 実体となる構造体。
  • WordEntry: 語彙を管理するメタデータ。
  • WordEntryPos: 位置情報を保持する構造体。

失敗モードと境界条件

  • 可変長データ構造であるため、直接的なポインタ演算を行う際はアライメントや varlena ヘッダーの扱いに注意が必要です。
  • 構造体サイズを誤って計算した場合、メモリ破壊や不正なアクセスが発生する可能性があります。

保守担当者向けメモ

  • TSVector は TSVectorData へのポインタであるため、メモリ管理は palloc 等の PostgreSQL メモリコンテキストに依存します。
  • 構造体定義の変更時は、tsvector のバイナリ互換性に影響を与えるため、ディスク保存形式との整合性に注意してください。
