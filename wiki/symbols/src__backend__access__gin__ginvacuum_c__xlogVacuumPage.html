xlogVacuumPage

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginvacuum.c:88

目的

GINインデックスのバキューム処理において、エントリツリーのリーフページに対する変更をWAL（Write-Ahead Logging）に記録します。

シグネチャまたは形状

static void xlogVacuumPage(Relation index, Buffer buffer)

入力と出力

  • **入力**:
  • index: 対象となるインデックスのリレーション。
  • buffer: バキューム対象のページを含むバッファ。
  • **出力**: なし（副作用としてWALログを生成し、ページにLSNをセットする）。

前提条件と不変条件

  • 対象ページはエントリツリーのリーフページであること（!GinPageIsData(page) かつ GinPageIsLeaf(page) のアサーションにより保証）。
  • データベースがWALを必要とする設定であること。

アルゴリズムまたは振る舞い

1. RelationNeedsWAL を確認し、WALが不要な場合は即座に終了する。 2. XLogBeginInsert を呼び出し、WAL挿入シーケンスを開始する。 3. XLogRegisterBuffer を使用し、REGBUF_FORCE_IMAGE フラグを付与してページ全体のフルイメージを登録する（変更内容が細かいため、差分ではなくページ全体を記録する）。 4. XLogInsert を呼び出し、RM_GIN_ID および XLOG_GIN_VACUUM_PAGE 型のログを書き込む。 5. 書き込まれたログのLSN（Log Sequence Number）を PageSetLSN で対象ページに適用する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginvacuum.c 内のバキューム処理フローから呼び出されます（ソース内の直接的な参照として確認）。
  • **呼び出し先**:
  • BufferGetPage
  • GinPageIsData / GinPageIsLeaf
  • RelationNeedsWAL
  • XLogBeginInsert
  • XLogRegisterBuffer
  • XLogInsert
  • PageSetLSN

関連データ構造

  • Relation: インデックスのメタデータ。
  • Buffer: ページデータへのアクセスに使用。
  • Page: GINインデックスのページ構造。

失敗モードと境界条件

  • WALが不要な設定（RelationNeedsWAL が偽）の場合、ログ出力を行わずに正常終了する。
  • ページがリーフページでない場合、アサーション違反により異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は、ページ内の変更を細かく追跡するコストを避けるため、常にフルページイメージを記録しています。将来的な最適化の余地があります。
  • ログタイプとして XLOG_GIN_VACUUM_PAGE を使用しています。WALのリカバリロジックを変更する際は、このログ型の処理も併せて確認してください。
