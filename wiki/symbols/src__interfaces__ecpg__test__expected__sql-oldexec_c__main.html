目的

ECPG（Embedded SQL for C）における動的SQL実行、プリペアドステートメント、およびカーソル操作の動作を検証するための回帰テスト用エントリーポイントです。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は使用しません）。
  • 出力: 標準出力へテスト結果（挿入されたタプル数、フェッチしたデータの内容）を出力します。

前提条件と不変条件

  • データベース ecpg1_regression への接続が成功している必要があります。
  • 各SQL操作の直後に sqlca.sqlcode < 0 をチェックし、エラー発生時には sqlprint() を呼び出すことで不変条件（SQLの正常実行）を維持しています。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGconnect でデータベースに接続し、test テーブルを作成してトランザクションをコミットします。 2. **動的SQL実行**: sprintf で構築した INSERT 文を ECPGst_exec_immediate を介して複数回実行します。 3. **プリペアド実行**: ECPGprepare でパラメータ付きSQLを準備し、ECPGst_execute でホスト変数 increment をバインドして実行します。 4. **カーソル操作**:

  • DECLARE CURSOR でクエリを定義し、FETCH を使用して結果をホスト変数配列（name, amount, letter）に格納します。
  • 取得したデータをループ処理し、標準出力へ整形して表示します。
  • CLOSE でカーソルを閉じます。

    • 5. **終了処理**: 最後に別のカーソル（CUR3）を用いたフェッチ操作を行い、プログラムを終了します。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリーポイント）。
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGdo, ECPGtrans, ECPGprepare, ECPGprepared_statement, sqlprint, sprintf, printf, strncpy。

関連データ構造

  • sqlca: SQL通信領域。エラーコードや処理行数（sqlca.sqlerrd[2]）の取得に使用。
  • ホスト変数配列: amount (int[8]), name (char[8][8]), letter (char[8][1])。

失敗モードと境界条件

  • SQL実行時にエラーが発生した場合、sqlprint() が呼び出され、標準エラー出力にエラー情報が表示されます。
  • sqlca.sqlcode が負の値になる場合、各ステップでエラーハンドリングが行われます。
