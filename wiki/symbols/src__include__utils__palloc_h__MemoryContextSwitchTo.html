MemoryContextSwitchTo

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/palloc.h:137

目的

現在のメモリ割り当て先コンテキストを一時的に別のコンテキストへ切り替え、後で元のコンテキストへ復帰できるように、切り替え前のコンテキストを保持・返却する。

シグネチャまたは形状

MemoryContext MemoryContextSwitchTo(MemoryContext context)

入力と出力

  • **入力**: context (切り替え先の MemoryContext)
  • **出力**: 切り替え前の CurrentMemoryContext (呼び出し元が後で復帰するために使用する)

前提条件と不変条件

  • CurrentMemoryContext グローバル変数を直接書き換えることは禁止されており、必ず本関数を経由すること。
  • 戻り値として受け取ったコンテキストは、作業終了後に再度 MemoryContextSwitchTo に渡すことで元の状態に戻す必要がある。

アルゴリズムまたは振る舞い

1. 現在の CurrentMemoryContext をローカル変数 old に退避する。 2. CurrentMemoryContext を引数 context に更新する。 3. 退避しておいた old を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c
  • contrib/amcheck/verify_nbtree.c
  • contrib/auto_explain/auto_explain.c
  • contrib/bloom/blinsert.c
  • **呼び出し先**:
  • 本関数はインライン関数であり、直接的な関数呼び出しは行わない。

関連データ構造

  • MemoryContext: メモリ割り当ての生存期間を管理する抽象型。
  • CurrentMemoryContext: 現在の割り当て先を保持するグローバル変数。

失敗モードと境界条件

  • 引数 context が無効なポインタである場合、その後の palloc 等のメモリ割り当て操作でクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数は非常に頻繁に呼び出されるため、インライン化によるオーバーヘッドの最小化が維持されている。
  • コンテキストの切り替えは必ずペア（退避と復帰）で行うこと。リークや不正なコンテキストへの割り当てを防ぐため、呼び出しパターンの遵守が重要である。
