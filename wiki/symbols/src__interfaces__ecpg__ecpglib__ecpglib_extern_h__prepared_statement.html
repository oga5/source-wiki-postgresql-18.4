prepared_statement

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/ecpglib/ecpglib_extern.h:92

目的

ECPGライブラリにおいて、プリペアドステートメント（準備済みSQL文）の状態を管理し、接続ごとに保持されるリスト構造を構成するために使用されます。

シグネチャまたは形状

struct prepared_statement
{
	char	   *name;
	bool		prepared;
	struct statement *stmt;
	struct prepared_statement *next;
};

入力と出力

  • **入力**: 構造体の各メンバ（SQL文の名前、準備状態、ステートメント本体へのポインタ、次要素へのポインタ）。
  • **出力**: 接続に関連付けられたプリペアドステートメントのリスト管理。

前提条件と不変条件

  • next メンバにより、単方向連結リストを形成します。
  • stmt メンバは、実行されるSQL文のライフサイクルを管理する statement 構造体を指します。

アルゴリズムまたは振る舞い

  • ecpg_alloc を介してメモリが割り当てられ、動的にリストに追加・管理されます。
  • ecpg_find_prepared_statement 関数により、名前をキーとしてリストから特定のステートメントを検索可能です。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/interfaces/ecpg/ecpglib/descriptor.c: 記述子操作における参照。
  • src/interfaces/ecpg/ecpglib/prepare.c: プリペアドステートメントの生成、検索、管理ロジック。
  • src/interfaces/ecpg/ecpglib/ecpglib_extern.h: 構造体定義および検索関数プロトタイプ。
  • **呼び出し先**:
  • struct statement: 実行コンテキストの保持。

関連データ構造

  • struct statement: SQL実行のライフサイクルを管理する構造体。
  • struct connection: プリペアドステートメントのリスト（prep_stmts）を保持する親構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時（ecpg_alloc 経由）の挙動は、ECPGライブラリのメモリ管理ポリシーに依存します。
  • リストの末尾では next が NULL となります。

保守担当者向けメモ

  • 本構造体はライブラリ内部での管理用です。
  • prepare.c 内のロジックでリストの走査や挿入が行われるため、変更時はリストの整合性に注意してください。
