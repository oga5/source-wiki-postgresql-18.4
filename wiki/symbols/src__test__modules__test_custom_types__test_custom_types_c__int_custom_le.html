int_custom_le

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_custom_types/test_custom_types.c:143

目的

カスタム型に対する「以下（less than or equal to）」比較演算を実装し、PostgreSQLの演算子フレームワークに提供する。

シグネチャまたは形状

Datum int_custom_le(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される2つの int32 型の値。
  • **出力**: 第1引数が第2引数以下であれば true、そうでなければ false を示す bool 型の Datum。

前提条件と不変条件

  • 入力値は int32 として解釈可能であること。
  • PostgreSQLの関数マネージャインターフェース（fmgr.h）の規約に従うこと。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT32 を使用して、第1引数と第2引数を取得する。 2. C言語の比較演算子 <= を用いて両者を比較する。 3. PG_RETURN_BOOL を使用して、比較結果を Datum 形式で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの演算子実行エンジン（SQLレベルで本関数がバインドされた演算子が呼び出された際）。
  • **呼び出し先**: fmgr.h で定義された PG_GETARG_INT32 および PG_RETURN_BOOL マクロ。
  • **注記**: ソースコード内の PG_FUNCTION_INFO_V1(int_custom_le) は、本関数がPostgreSQLの関数マネージャに登録されていることを示している。

関連データ構造

  • int32: 比較対象となる内部データ型。
  • Datum: PostgreSQLの汎用データ表現。

失敗モードと境界条件

  • int32 の範囲外の値が渡された場合の挙動は、呼び出し元の型変換処理に依存する。
  • 本関数自体は単純な比較を行うのみであり、特有の失敗モードは定義されていない。

保守担当者向けメモ

  • 本関数はテストモジュールの一部であり、int32 の標準的な比較ロジックを模倣している。
  • 比較ロジックに変更を加える場合は、他の比較関数（int_custom_lt, int_custom_gt 等）との整合性に注意すること。
