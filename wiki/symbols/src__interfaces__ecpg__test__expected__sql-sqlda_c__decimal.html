decimal

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-sqlda.c:78

目的

ECPG（Embedded SQL for C）において、固定小数点数（Numeric）データを表現するための構造体です。

シグネチャまたは形状

typedef struct
{
    int             ndigits;        /* digits[] 内の桁数 */
    int             weight;         /* 先頭桁の重み */
    int             rscale;         /* 結果のスケール */
    int             dscale;         /* 表示スケール */
    int             sign;           /* 符号 (NUMERIC_POS, NUMERIC_NEG, NUMERIC_NAN) */
    NumericDigit    digits[DECSIZE]; /* 10進数桁データ */
} decimal;

入力と出力

  • 入力: 構造体の各メンバ（ndigits, weight, rscale, dscale, sign, digits）に数値を設定することで、任意の固定小数点数を表現します。
  • 出力: 構造体として保持された数値データは、ECPGの動的SQL実行時やデータダンプ処理において参照されます。

前提条件と不変条件

  • digits 配列のサイズは DECSIZE に依存します。
  • sign メンバは、定義済みの定数（NUMERIC_POS, NUMERIC_NEG, NUMERIC_NAN）のいずれかである必要があります。

アルゴリズムまたは振る舞い

本構造体は、PostgreSQLの numeric 型の内部表現をECPG環境で扱うためのデータコンテナとして機能します。sql-sqlda.c 内の dump_sqlda 関数等において、SQLDA経由で取得された数値データの解析や表示に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内では、特定の関数による直接的な呼び出し関係は明示されていません。
  • 呼び出し先: NumericDigit 型（digits 配列の要素）に依存しています。

関連データ構造

  • NumericDigit: digits 配列の型定義。
  • sqlda_t: SQLDAインターフェースの主要構造体。
  • sqlvar_t: SQLDA内の列情報を保持する構造体。

失敗モードと境界条件

  • ndigits が 0 になる可能性があります。
  • digits 配列の容量（DECSIZE）を超える桁数の数値は保持できません。

保守担当者向けメモ

  • 本構造体は _ECPG_INFORMIX_H マクロ等の環境設定により定義が切り替わる可能性があるため、sqlda-compat.h および sqlda-native.h の定義と併せて確認してください。
  • 他のモジュール（contrib/ 配列など）で言及される「decimal」という用語は、本構造体とは異なる文脈（Pythonモジュールや文字列処理）である可能性が高いため、混同に注意してください。
