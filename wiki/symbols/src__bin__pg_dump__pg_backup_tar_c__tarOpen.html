tarOpen

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_tar.c:301

目的

TAR形式のアーカイブ内における特定のファイルメンバーを読み込みまたは書き込み用にオープンし、対応する TAR_MEMBER 構造体を初期化する。

シグネチャまたは形状

static TAR_MEMBER *tarOpen(ArchiveHandle *AH, const char *filename, char mode)

入力と出力

  • **入力**:
  • AH: アーカイブのハンドル。
  • filename: 操作対象のファイル名。読み込みモードで NULL の場合は次のファイルを選択する。
  • mode: 操作モード（'r': 読み込み, 'w': 書き込み）。
  • **出力**:
  • 成功時: 初期化された TAR_MEMBER 構造体へのポインタ。
  • 失敗時: 読み込みモードでファイルが見つからない場合は NULL、または pg_fatal による異常終了。

前提条件と不変条件

  • アーカイブの圧縮アルゴリズムが PG_COMPRESSION_NONE であること（圧縮は未サポート）。
  • 書き込みモード時は、一時ファイル作成のために umask が一時的に変更される。

アルゴリズムまたは振る舞い

1. **読み込みモード ('r')**:

  • _tarPositionTo を呼び出し、指定された filename までアーカイブ内をシークする。
  • ファイルが見つからない場合、filename が指定されていればエラー終了し、NULL ならそのまま戻る。
  • tm->nFH にアーカイブのファイルハンドルを割り当てる。

    • 2. **書き込みモード ('w')**:

  • TAR_MEMBER 構造体をメモリ確保する。
  • tmpfile()（Windows環境では _tempnam と open のループ）を使用して一時ファイルを作成し、tm->tmpFH に割り当てる。
  • ファイル作成時のパーミッションを制御するため umask を操作する。
  • tm->targetFile にファイル名を保存する。

    • 3. **共通**:

  • tm->mode にモードを格納し、tm->tarFH にコンテキストのファイルハンドルを設定して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_backup_tar.c 内の各所（toc.dat の読み書き、restore.sql の生成、ラージオブジェクトの処理など）。
  • **呼び出し先**: _tarPositionTo（読み込み時）、pg_malloc0_object、tmpfile、_tempnam、open、fdopen、pg_fatal、pg_strdup。

関連データ構造

  • TAR_MEMBER: アーカイブ内のメンバー情報を保持する構造体。
  • ArchiveHandle: アーカイブ全体の管理構造体。
  • lclContext: TAR形式特有のコンテキスト（AH->formatData から取得）。

失敗モードと境界条件

  • **圧縮の不一致**: PG_COMPRESSION_NONE 以外が指定されている場合、pg_fatal で停止する。
  • **ファイル検索失敗**: 読み込み時に指定ファイルが存在しない場合、pg_fatal で停止する。
  • **一時ファイル作成失敗**: 書き込み時に一時ファイルが生成できない場合、pg_fatal で停止する。

保守担当者向けメモ

  • 本関数は圧縮をサポートしていない。将来的な拡張が必要な場合は、tm->nFH の割り当てロジックを圧縮ストリームに対応させる必要がある。
  • Windows環境での一時ファイル作成ロジック（_tempnam 使用）は、競合を避けるためのループ構造を含んでいる。
