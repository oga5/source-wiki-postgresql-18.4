process_matched_tle

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteHandler.c:1080

目的

INSERTまたはUPDATE文において、同一カラムに対する複数の代入操作を統合し、単一のターゲットリストエントリ（TargetEntry）へ正規化する。

シグネチャまたは形状

TargetEntry *process_matched_tle(TargetEntry *src_tle, TargetEntry *prior_tle, const char *attrName)

入力と出力

  • **入力**:
  • src_tle: 新しく適用される代入操作を含むターゲットエントリ。
  • prior_tle: 既に存在する代入操作を含むターゲットエントリ（初回はNULL）。
  • attrName: エラーメッセージ用に使用されるカラム名。
  • **出力**: 統合された式を持つ新しい TargetEntry を返す。

前提条件と不変条件

  • prior_tle がNULLの場合、src_tle をそのまま返す。
  • 複数の代入は FieldStore または SubscriptingRef 操作である必要がある。
  • 統合される各代入の基底となる式（get_assignment_input で取得）が一致していなければならない。
  • CoerceToDomain が存在する場合、それらは同一の型である必要がある。

アルゴリズムまたは振る舞い

1. **初期チェック**: prior_tle がNULLなら src_tle を返す。 2. **ドメイン剥離**: 両方の式が CoerceToDomain であれば、一時的に剥離して内部の代入式を比較する。 3. **入力検証**: get_assignment_input を使用して、両方の代入が同一の基底（Var参照等）に対して行われているかを確認する。一致しない場合はエラーを報告する。 4. **統合処理**:

  • FieldStore の場合: 複数のフィールド更新を単一の FieldStore にマージするか、ネスト構造を作成する。
  • SubscriptingRef の場合: prior_expr を refexpr に設定してネストする。

    • 5. **再構築**: 剥離した CoerceToDomain があれば、統合後の式をラップして戻す。 6. **結果生成**: flatCopyTargetEntry を用いて新しいエントリを作成し、統合した式を格納して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/rewrite/rewriteHandler.c 内の関数（rewriteTargetListIU 等から呼び出されることが示唆されている）。
  • **呼び出し先**: get_assignment_input, makeNode, list_concat_copy, equal, flatCopyTargetEntry, ereport, elog。

関連データ構造

  • TargetEntry: クエリのターゲットリストの各要素。
  • FieldStore: フィールド更新操作を保持するノード。
  • SubscriptingRef: 配列等の添字参照操作を保持するノード。
  • CoerceToDomain: ドメイン型へのキャストを保持するノード。

失敗モードと境界条件

  • **エラー**:
  • 同一カラムへの複数代入で、基底となる式が一致しない場合（ERRCODE_SYNTAX_ERROR）。
  • サポートされていない式タイプが渡された場合（elog(ERROR, "cannot happen")）。
  • **境界条件**:
  • prior_tle がNULL（初回代入）の場合は正常系として即時リターンする。

保守担当者向けメモ

  • 本関数は、UPDATE tab SET col.fld1 = x, col.fld2 = y のような複雑な代入を処理する。
  • FieldStore の統合は、複数のフィールド更新を単一ノードにまとめることで最適化を図っている。
  • CoerceToDomain の処理は、ドメイン制約がすべてのフィールド更新完了後に適用されるように設計されている。
