LockClauseStrength

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/lockoptions.h:21

目的

SQLの SELECT FOR 句で指定される行レベルのロック強度を表現し、プランナーが複数のロック指定を統合する際の優先順位を決定するために使用されます。

シグネチャまたは形状

typedef enum LockClauseStrength
{
    LCS_NONE,           /* ロック指定なし（PlanRowMark用） */
    LCS_FORKEYSHARE,    /* FOR KEY SHARE */
    LCS_FORSHARE,       /* FOR SHARE */
    LCS_FORNOKEYUPDATE, /* FOR NO KEY UPDATE */
    LCS_FORUPDATE,      /* FOR UPDATE */
} LockClauseStrength;

入力と出力

  • 入力: SQLパーサーが解析したロック指定に基づき生成されます。
  • 出力: プランナーやリライターが、最終的なロック要件を決定するための比較値として利用します。

前提条件と不変条件

  • 列挙値の順序は優先順位と一致しており、値が大きいほどロック強度が強くなります。
  • LCS_NONE はロック指定がない状態を示し、主に実行計画ノード（PlanRowMark）で使用されます。

アルゴリズムまたは振る舞い

  • パーサーが SQL 文からロック指定を解析し、この列挙型を ParseState や Query 構造体に格納します。
  • プランナーの applyLockingClause 関数などが、複数のロック指定が競合した際に列挙値の大小を比較し、より強いロック要件を選択します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/optimizer/plan/planner.c (select_rowmark_type)
  • src/backend/parser/analyze.c (CheckSelectLocking, LCS_asString)
  • src/backend/rewrite/rewriteHandler.c
  • src/include/foreign/fdwapi.h
  • src/include/nodes/execnodes.h (PlanRowMark 構造体)
  • 呼び出し先: 本シンボルは列挙型定義であり、直接的な関数呼び出しは行いません。

関連データ構造

  • RowMarkClause (src/include/nodes/parsenodes.h): SQLのロック句を保持する構造体。
  • PlanRowMark (src/include/nodes/plannodes.h): 実行計画におけるロック要件を保持する構造体。
  • LockTupleMode: より低レイヤーのタプルロック処理を表現する列挙型。

失敗モードと境界条件

  • 未定義の整数値がキャストされた場合、elog(ERROR, "unrecognized LockClauseStrength %d", ...) によりエラーが発生します（planner.c および ruleutils.c で確認）。

保守担当者向けメモ

  • 新しいロックモードを追加する際は、既存の列挙順序が applyLockingClause 等での優先順位判定ロジックと整合しているか確認してください。
  • LockClauseStrength は SQL レベルの指定であり、内部的な LockTupleMode とは役割が異なる点に注意してください。
