xl_invalid_page

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/xlogutils.c:72

目的

WALリプレイ中に発生した「存在しないページ」や「ゼロ埋めされたページ」への参照を追跡し、リカバリ処理の整合性を検証するためのハッシュテーブル用エントリ構造体です。

シグネチャまたは形状

typedef struct xl_invalid_page
{
    xl_invalid_page_key key;    /* ハッシュキー（構造体の先頭である必要がある） */
    bool        present;        /* ページは存在するがゼロ埋めされていたか */
} xl_invalid_page;

入力と出力

  • **入力**: xl_invalid_page_key をキーとしてハッシュテーブルに格納されます。
  • **出力**: present フラグにより、該当ページが物理的に存在したか（ゼロ埋め状態）を識別します。

前提条件と不変条件

  • key フィールドは構造体の先頭に配置されなければなりません。
  • 本構造体は invalid_page_tab ハッシュテーブルの要素として使用されます。
  • リカバリ終了時に本構造体を含むエントリがハッシュテーブルに残存している場合、データベースの整合性に問題があるとみなされます。

アルゴリズムまたは振る舞い

1. **登録**: WALリプレイ中にページ読み込みが失敗またはゼロ埋めページを検出した際、この構造体がハッシュテーブルに挿入されます。 2. **追跡**: リカバリの進行に伴い、該当ページが正当化されるか、あるいは削除されるまで状態を保持します。 3. **検証**: リカバリの最終段階で XLogCheckInvalidPages がハッシュテーブルを走査し、残存するエントリがないかを確認します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlogutils.c 内のハッシュテーブル操作関数（hash_seq_search 等）から参照されます。
  • **呼び出し先**: 直接的な関数呼び出しは行いません。データ保持のための構造体です。

関連データ構造

  • xl_invalid_page_key: ハッシュテーブルの検索キー。
  • invalid_page_tab: 本構造体を要素として格納するハッシュテーブル。

失敗モードと境界条件

  • **メモリ不足**: ハッシュテーブルへの挿入時にメモリ不足が発生する可能性があります。
  • **整合性エラー**: リカバリ終了時にエントリが残存している場合、PANIC レベルのエラーが発生し、リカバリが中断されます。

保守担当者向けメモ

  • 本構造体はリカバリプロセス専用です。通常のバックエンドプロセスでは使用されません。
  • present フラグは、フルページ書き込みが無効な環境下でのリカバリにおいて、ページが物理的に存在するか否かを区別するために重要です。
