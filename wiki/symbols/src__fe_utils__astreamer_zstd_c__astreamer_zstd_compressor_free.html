astreamer_zstd_compressor_free

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_zstd.c:242

目的

Zstandard圧縮ストリーマーに関連付けられたリソースを解放し、メモリリークを防ぐ。

シグネチャまたは形状

static void astreamer_zstd_compressor_free(astreamer *streamer)

入力と出力

  • **入力**: astreamer *streamer - 解放対象のストリーマー構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • streamer は astreamer_zstd_frame 型としてキャスト可能であること。
  • streamer->bbs_next が有効なストリーマーであるか、または NULL であること（astreamer_free の仕様に依存）。

アルゴリズムまたは振る舞い

1. astreamer_free(streamer->bbs_next) を呼び出し、パイプラインの後続ストリーマーを再帰的に解放する。 2. ZSTD_freeCCtx(mystreamer->cctx) を呼び出し、zstd圧縮コンテキストを破棄する。 3. pfree(streamer->bbs_buffer.data) を呼び出し、内部バッファを解放する。 4. pfree(streamer) を呼び出し、ストリーマー構造体自体を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/astreamer_zstd.c 内のストリーマー定義構造体（.free メンバ）から参照されている。
  • **呼び出し先**:
  • astreamer_free (後続ストリーマーの解放)
  • ZSTD_freeCCtx (zstdライブラリ関数)
  • pfree (メモリ解放)

関連データ構造

  • astreamer: ストリーマーの基底構造体。
  • astreamer_zstd_frame: 圧縮コンテキストとバッファを保持する派生構造体。

失敗モードと境界条件

  • streamer が NULL の場合、pfree 等でセグメンテーション違反が発生する可能性がある（呼び出し元でガードされていることが前提）。
  • bbs_next が既に解放済みである場合、二重解放によるクラッシュのリスクがある。

保守担当者向けメモ

  • 本関数は astreamer インターフェースの free コールバックとして実装されています。パイプラインの構造変更時には、bbs_next の解放順序が適切であることを確認してください。
  • ZSTD_CCtx の解放は libzstd の仕様に従っています。zstdのバージョンアップ時には、この解放関数が適切か再確認してください。
