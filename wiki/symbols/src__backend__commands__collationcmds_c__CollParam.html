CollParam

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/collationcmds.c:758

目的

照合順序（Collation）のインポート処理において、対象の名前空間（Namespace）と、処理結果を追跡するためのカウンタ変数を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	Oid			nspid;      /* 対象の名前空間ID */
	int		   *ncreatedp;  /* 作成された照合順序数のカウンタ */
	int		   *nvalidp;    /* 有効な照合順序数のカウンタ */
} CollParam;

入力と出力

本構造体は、照合順序のインポート処理中にパラメータを渡すためのコンテナとして機能します。

  • **入力**: 呼び出し元で初期化された nspid およびカウンタへのポインタ。
  • **出力**: 処理の過程で ncreatedp および nvalidp が指す先の整数値がインクリメントされます。

前提条件と不変条件

  • ncreatedp および nvalidp は、呼び出し元によって有効なメモリ領域を指すポインタとして提供される必要があります。
  • nspid は有効な名前空間のOIDである必要があります。

アルゴリズムまたは振る舞い

照合順序のインポート処理（pg_import_system_collations 等）において、この構造体はコールバック関数や補助関数へ渡されます。関数内では CollParam *param = (CollParam *) lparam; のようにキャストして参照され、インポートの進捗状況を呼び出し元のスコープへ反映させるために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/collationcmds.c 内の照合順序インポート関連関数。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • ポインタ ncreatedp や nvalidp が NULL の場合、参照時にセグメンテーション違反が発生する可能性があります。
  • 呼び出し元でカウンタの初期化を忘れた場合、不正なメモリ領域への書き込みが発生するリスクがあります。

保守担当者向けメモ

  • 本構造体は pg_import_system_collations のような一括処理のコンテキストでのみ使用されています。
  • 構造体のフィールドを追加・変更する場合は、この構造体を利用しているすべてのインポート関連関数でのキャストおよび参照箇所を確認してください。
