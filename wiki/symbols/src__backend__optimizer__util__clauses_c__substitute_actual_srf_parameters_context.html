substitute_actual_srf_parameters_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/util/clauses.c:77

目的

セットを返す関数（SRF: Set-Returning Function）のインライン展開処理において、関数呼び出し時の実引数を保持し、式ツリー内のパラメータを置換するためのコンテキストとして使用されます。

シグネチャまたは形状

typedef struct
{
	int			nargs;
	List	   *args;
	int			sublevels_up;
} substitute_actual_srf_parameters_context;

入力と出力

  • nargs: 処理対象の引数の数。
  • args: 置換対象となる実引数のリスト。
  • sublevels_up: 現在の式ツリーのネストレベル（サブクエリの深さ）。

前提条件と不変条件

  • 本構造体は、SRFのインライン化処理中に一時的な状態保持のためにスタック上で生成されます。
  • args リストは、関数定義と整合する実引数を含んでいる必要があります。

アルゴリズムまたは振る舞い

  • src/backend/optimizer/util/clauses.c 内の関数において、式ツリーを走査する際のパラメータ置換ロジックの制御構造として利用されます。
  • ツリー走査中にパラメータノード（Param）に遭遇した際、このコンテキスト内の情報を用いて、対応する実引数への置換が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/util/clauses.c 内の SRF インライン化関連関数（inline_set_returning_function 等）から参照されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • List: PostgreSQLの汎用リスト構造。引数の保持に使用されます。
  • Param: 式ツリー内で置換対象となるパラメータノード。

失敗モードと境界条件

  • args リストの要素数と nargs が不一致の場合、置換処理中に不正なメモリ参照や論理エラーが発生する可能性があります。
  • sublevels_up が正しく設定されていない場合、ネストされたクエリ内でのパラメータ置換が誤ったスコープで行われる可能性があります。

保守担当者向けメモ

  • 本構造体は substitute_actual_parameters_context と類似していますが、SRF 特有のインライン化ロジックに特化しています。変更を加える際は、clauses.c 内の SRF 関連の走査関数への影響を確認してください。
