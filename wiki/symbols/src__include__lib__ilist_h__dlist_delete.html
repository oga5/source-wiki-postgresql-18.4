dlist_delete

定義

  • 種別: function
  • 定義箇所: src/include/lib/ilist.h:404

目的

二重連結リストから指定されたノードを削除し、リストの整合性を維持するために前後のノードのリンクを繋ぎ変える。

シグネチャまたは形状

static inline void dlist_delete(dlist_node *node)

入力と出力

  • **入力**: node (削除対象の dlist_node へのポインタ)
  • **出力**: なし（副作用としてリスト構造を更新）

前提条件と不変条件

  • node は有効な二重連結リストに含まれている必要がある。
  • node がリストの先頭や末尾である場合、リストの head 構造体が適切に循環参照を維持している必要がある。
  • 本関数はメモリの解放を行わない。ノードが保持するデータ構造のメモリ管理は呼び出し側に委ねられる。

アルゴリズムまたは振る舞い

1. node->prev の next ポインタを node->next に更新する。 2. node->next の prev ポインタを node->prev に更新する。 これにより、リストから node が論理的に切り離される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gindatapage.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/transam/parallel.c
  • src/backend/access/transam/xlogprefetcher.c
  • src/backend/executor/nodeMemoize.c
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/bgworker.c
  • src/backend/postmaster/pmchild.c
  • src/backend/replication/logical/reorderbuffer.c
  • **呼び出し先**: なし（インライン関数として直接ポインタ操作を実行）

関連データ構造

  • dlist_node: リストのリンクを保持する構造体。
  • dlist_head: リストの起点となる構造体。

失敗モードと境界条件

  • node がリストに含まれていない場合、または node が NULL の場合、不正なメモリアクセスが発生する。
  • リストが空の状態や、単一要素のみのリストに対する操作は、dlist_head の構造が循環していることを前提としている。

保守担当者向けメモ

  • 本関数はインライン化されており、パフォーマンス重視の設計となっている。
  • ILIST_DEBUG が定義されている場合、ilist.c 内のデバッグ用関数による整合性チェックと併用されることが想定される。
  • 削除後のノードのポインタは更新されないため、削除後にノードを再利用する場合は注意が必要。
