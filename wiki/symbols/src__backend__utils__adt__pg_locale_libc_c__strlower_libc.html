strlower_libc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_libc.c:122

目的

指定されたロケール設定に基づき、文字列を小文字に変換する。データベースのエンコーディングに応じて、適切な変換処理へルーティングするブリッジ関数。

シグネチャまたは形状

size_t strlower_libc(char *dst, size_t dstsize, const char *src, ssize_t srclen, pg_locale_t locale)

入力と出力

  • dst: 変換後の文字列を格納するバッファ。
  • dstsize: dst バッファのサイズ。
  • src: 変換元の文字列。
  • srclen: src の長さ（バイト単位）。
  • locale: 使用するロケール情報（pg_locale_t）。
  • 戻り値: 変換後の文字列の長さ（バイト単位）。

前提条件と不変条件

  • データベースのエンコーディング情報がシステムから取得可能であること。
  • pg_database_encoding_max_length() の結果に基づき、シングルバイトまたはマルチバイトの処理が選択される。

アルゴリズムまたは振る舞い

1. pg_database_encoding_max_length() を呼び出し、データベースの最大文字長を確認する。 2. 最大長が 1 より大きい場合、strlower_libc_mb を呼び出してマルチバイト対応の変換を行う。 3. 最大長が 1 以下の場合、strlower_libc_sb を呼び出してシングルバイト用の変換を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/pg_locale.c 内のロケール抽象化層から呼び出される。
  • **呼び出し先**:
  • strlower_libc_mb (マルチバイト環境時)
  • strlower_libc_sb (シングルバイト環境時)

関連データ構造

  • pg_locale_t: ロケール情報を保持する構造体。

失敗モードと境界条件

  • dstsize が不足している場合の挙動は、内部で呼び出される _mb または _sb 系関数の実装に依存する。
  • データベースエンコーディングの判定結果により、処理パスが分岐する。

保守担当者向けメモ

  • 本関数は単なるディスパッチャーであるため、変換ロジックの修正が必要な場合は strlower_libc_mb または strlower_libc_sb を参照すること。
