test_open

定義

  • 種別: function
  • 定義箇所: src/bin/pg_test_fsync/pg_test_fsync.c:240

目的

テスト用の一時ファイルを作成し、基本的な書き込みと同期が可能であることを検証する。

シグネチャまたは形状

static void test_open(void)

入力と出力

  • 入力: なし（グローバル変数 filename および full_buf を使用）
  • 出力: なし（成功時は一時ファイルを作成し、失敗時は die() を呼び出しプロセスを終了する）

前提条件と不変条件

  • filename で指定されたパスへの書き込み権限が必要。
  • full_buf には DEFAULT_XLOG_SEG_SIZE 分のデータが格納されていること。

アルゴリズムまたは振る舞い

1. open() を使用して一時ファイルを作成（O_RDWR | O_CREAT | PG_BINARY）。 2. needs_unlink フラグを 1 に設定し、終了時の削除対象とする。 3. full_buf から DEFAULT_XLOG_SEG_SIZE 分のデータをファイルに書き込む。 4. fsync() を実行し、書き込みバッファをフラッシュして後続のテストへの影響を排除する。 5. close() でファイル記述子を閉じる。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_test_fsync/pg_test_fsync.c:126
  • 呼び出し先: open, write, fsync, close, die（いずれも標準ライブラリまたは共通ユーティリティ）

関連データ構造

  • filename: テスト対象のファイルパスを保持するグローバル変数。
  • full_buf: 書き込み用データバッファ。
  • needs_unlink: 一時ファイルの削除フラグ。

失敗モードと境界条件

  • ファイルのオープン失敗、書き込み失敗、または fsync 失敗時に die() を呼び出し、エラーメッセージを出力してプログラムを終了する。

保守担当者向けメモ

  • 本関数はテストの初期化フェーズで実行される。ここで作成されたファイルは、テスト終了時に signal_cleanup 等によって削除される必要がある。
