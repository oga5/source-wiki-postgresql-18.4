libpqrcv_get_backend_pid

定義

  • 種別: function
  • 定義箇所: src/backend/replication/libpqwalreceiver/libpqwalreceiver.c:1064

目的

レプリケーション接続において、接続先のプライマリサーバー側で動作しているバックエンドプロセスのプロセスID（PID）を取得します。

シグネチャまたは形状

static pid_t libpqrcv_get_backend_pid(WalReceiverConn *conn)

入力と出力

  • **入力**: WalReceiverConn *conn（アクティブなレプリケーション接続を保持する構造体へのポインタ）
  • **出力**: pid_t（サーバー側のバックエンドPID。接続が無効な場合は libpq の仕様に従う）

前提条件と不変条件

  • 引数 conn は有効な WalReceiverConn 構造体を指している必要があります。
  • 内部の conn->streamConn が初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った WalReceiverConn 構造体から、内部の streamConn（PGconn 型）を参照します。 2. libpq の関数である PQbackendPID を呼び出し、サーバー側のバックエンドPIDを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: WalReceiverFunctions 構造体を通じて、レプリケーション管理プロセスから間接的に呼び出されます（src/backend/replication/libpqwalreceiver/libpqwalreceiver.c:113 にて関数ポインタとして登録）。
  • **呼び出し先**: libpq ライブラリの PQbackendPID 関数。

関連データ構造

  • WalReceiverConn: レプリケーション接続の状態を保持する構造体。本関数はこの構造体内の streamConn メンバを利用します。

失敗モードと境界条件

  • conn が NULL の場合、あるいは conn->streamConn が初期化されていない場合、セグメンテーション違反が発生する可能性があります。
  • PQbackendPID の仕様に基づき、接続が確立されていない場合は 0 が返される可能性があります。

保守担当者向けメモ

  • 本関数は WalReceiverFunctions インターフェースの一部として実装されています。
  • libpq の PQbackendPID をラップしているだけの単純な関数ですが、レプリケーション接続の監視やデバッグにおいて重要な役割を果たします。
