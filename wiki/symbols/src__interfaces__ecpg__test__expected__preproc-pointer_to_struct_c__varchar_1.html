varchar_1

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c:36

目的

ECPG（Embedded SQL for C）において、可変長文字列（VARCHAR）を表現するためのデータ構造です。

シグネチャまたは形状

struct varchar_1 {
    int len;
    char arr[50];
};

入力と出力

  • 入力: 構造体メンバ len（文字列長）および arr（文字列バッファ）。
  • 出力: ECPGプリプロセッサが生成するSQL実行コードにおいて、ホスト変数としてデータベースとのデータ転送に使用されます。

前提条件と不変条件

  • arr は最大50バイトの文字データを保持します。
  • ECPGの仕様に基づき、len には arr に格納された有効な文字列の長さが設定される必要があります。

アルゴリズムまたは振る舞い

ECPGプリプロセッサによって生成されるコード内で、SQLの VARCHAR 型とC言語の構造体との間のマッピングを仲介します。SQLクエリ実行時に、この構造体のメモリレイアウトがライブラリ関数によって参照されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/test/expected/preproc-cursor.c 内の複数の箇所で、ECPGt_varchar 型として本構造体のサイズ（sizeof(struct varchar_1)）が参照されています。
  • 呼び出し先: ECPGライブラリ関数（ECPGdo 等）が、この構造体のアドレスを介してデータの読み書きを行います。

関連データ構造

  • varchar_2, varchar_3, varchar_4: 同様に定義された可変長文字列用構造体。
  • customer: 本構造体を利用するテスト用構造体。

失敗モードと境界条件

  • arr のサイズ（50バイト）を超えるデータを格納しようとした場合、バッファオーバーフローやデータ切り捨てが発生する可能性があります。
  • len の値が arr の確保サイズを超過した場合、ECPGライブラリの処理において未定義の挙動を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体はECPGプリプロセッサの出力テスト用です。手動での変更は行わず、プリプロセッサの変換ロジックの変更に合わせてテスト期待値を更新してください。
  • sizeof を使用した型チェックが行われているため、メンバ構成を変更する場合は関連するテストコードの整合性に注意してください。
