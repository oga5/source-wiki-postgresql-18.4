Type

定義

  • 種別: typedef
  • 定義箇所: src/include/parser/parse_type.h:20

目的

SQL文中の型名を解決した結果として得られる、システムカタログ pg_type の行情報を保持するための型定義です。

シグネチャまたは形状

typedef HeapTuple Type;

HeapTuple へのエイリアスであり、システムカタログの行データを指すポインタ構造体です。

入力と出力

  • **入力**: LookupTypeName 等の検索関数が pg_type を検索し、その結果として Type を返します。
  • **出力**: 呼び出し元は、この Type を通じて OID、型長、値渡しフラグ、照合順序などの属性情報を取得します。

前提条件と不変条件

  • Type はシステムキャッシュから取得されるため、使用後は必ず ReleaseSysCache を呼び出してメモリを解放する必要があります。
  • pg_type カタログの有効な行を指していることが前提です。

アルゴリズムまたは振る舞い

パーサーが構文解析中に TypeName ノードを解決する際、システムカタログを検索して得られた HeapTuple を Type として保持します。これにより、型名文字列から具体的な型属性へのアクセスが可能になります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/objectaddress.c および src/backend/commands/functioncmds.c で、Type 型の変数（tup や typtup）が宣言されていることが確認されています。
  • **呼び出し先**: HeapTuple を操作するシステムキャッシュ関連関数（ReleaseSysCache 等）が利用されます。
  • 注: ツールによる呼び出し元情報は、変数名の一致に基づくヒューリスティックなものであり、すべての参照を網羅しているわけではありません。

関連データ構造

  • HeapTuple: Type の実体。
  • TypeName: パーサーが扱う型名のノード構造体。
  • pg_type: 型情報の定義元であるシステムカタログ。

失敗モードと境界条件

  • 型が見つからない場合、LookupTypeName の設定に応じてエラーが発生するか、NULL が返されます。
  • ReleaseSysCache を忘れるとメモリリークが発生します。

保守担当者向けメモ

  • Type は HeapTuple の別名であるため、メモリ管理の責任が呼び出し元にある点に注意してください。
  • ISCOMPLEX マクロなどの型判定ユーティリティと組み合わせて使用されます。
