lookup_ts_dictionary_cache

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/ts_cache.c:207

目的

全文検索用辞書の定義および初期化データをキャッシュから取得します。キャッシュミスが発生した場合は、システムカタログから情報を読み込み、辞書固有のメモリコンテキストを作成してキャッシュエントリを構築します。

シグネチャまたは形状

TSDictionaryCacheEntry *lookup_ts_dictionary_cache(Oid dictId)

入力と出力

  • **入力**: dictId (Oid) - 取得対象の辞書のオブジェクトID。
  • **出力**: TSDictionaryCacheEntry * - 辞書の定義、初期化データ、および lexize メソッド情報を含むキャッシュエントリへのポインタ。

前提条件と不変条件

  • 初回呼び出し時に TSDictionaryCacheHash ハッシュテーブルおよび CacheMemoryContext が初期化されます。
  • キャッシュエントリはバックエンドの生存期間中保持されます。
  • isvalid フラグが false の場合、キャッシュエントリは再構築されます。

アルゴリズムまたは振る舞い

1. **初期化**: ハッシュテーブル未作成時は作成し、pg_ts_dict および pg_ts_template に対する無効化コールバックを登録します。 2. **高速ルックアップ**: lastUsedDictionary（単一エントリキャッシュ）を確認し、一致すれば即座に返します。 3. **ハッシュ検索**: TSDictionaryCacheHash を検索し、存在しないか無効な場合は構築処理へ進みます。 4. **構築**:

  • pg_ts_dict および関連する pg_ts_template をシステムカタログから取得します。
  • 辞書固有のメモリコンテキスト (dictCtx) を作成またはリセットします。
  • テンプレートの tmplinit メソッドが存在する場合、辞書オプションを読み込み、dictCtx 上で初期化メソッドを実行して dictData を格納します。
  • fmgr_info_cxt を使用して lexize メソッドの情報をキャッシュします。

    • 5. **更新**: lastUsedDictionary を更新し、エントリを返します。

関連データ構造

  • TSDictionaryCacheEntry: 辞書キャッシュの構造体。
  • TSDictionaryCacheHash: 辞書キャッシュを保持するハッシュテーブル。
  • lastUsedDictionary: 直近に使用された辞書エントリを保持するポインタ。

失敗モードと境界条件

  • **エラー**: 指定された dictId がシステムカタログに存在しない場合、またはテンプレートに lexize メソッドが定義されていない場合は elog(ERROR) により中断されます。
  • **メモリ**: キャッシュ構築には CacheMemoryContext 配下の専用コンテキストを使用します。

保守担当者向けメモ

  • 辞書キャッシュは dictCtx という個別のメモリコンテキストを持ちます。辞書固有のデータはここに格納されるため、キャッシュ無効化時にはこのコンテキストがリセットされます。
  • lastUsedDictionary は単一エントリの高速アクセスのために使用されており、頻繁にアクセスされる辞書に対して有効です。
