find_base_rel_ignore_join

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/relnode.c:453

目的

指定された relid に対応する RelOptInfo を PlannerInfo から取得する。通常のベースリレーション検索と異なり、特定の結合リレーション（内部結合以外の結合）を無視して NULL を返すことを許容する。

シグネチャまたは形状

RelOptInfo *find_base_rel_ignore_join(PlannerInfo *root, int relid)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報構造体。
  • relid: 検索対象のリレーションID。
  • **出力**:
  • 成功時: 対応する RelOptInfo 構造体へのポインタ。
  • 結合リレーション（非内部結合）の場合: NULL。
  • その他: エラー発生時は elog(ERROR) により処理が中断される。

前提条件と不変条件

  • relid は root->simple_rel_array_size 未満である必要がある。
  • root->simple_rel_array および root->simple_rte_array が初期化されていること。

アルゴリズムまたは振る舞い

1. relid が有効な範囲内かを確認する（負のインデックスアクセス防止のため uint32 にキャストして比較）。 2. root->simple_rel_array[relid] にエントリが存在すれば、それを即座に返す。 3. エントリが存在しない場合、root->simple_rte_array[relid] を確認する。 4. 対象が RTE_JOIN かつ JOIN_INNER 以外の結合タイプであれば NULL を返す。 5. 上記以外の場合は、対応するエントリが見つからないものとして elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/initsplan.c:1012
  • src/backend/optimizer/util/appendinfo.c:775
  • src/backend/optimizer/util/joininfo.c:140, 170
  • **呼び出し先**:
  • 直接的な関数呼び出しはなし（マクロや標準ライブラリを除く）。

関連データ構造

  • RelOptInfo: プランナが管理するリレーション情報。
  • PlannerInfo: プランナの実行状態を保持する構造体。
  • RangeTblEntry: レンジテーブルエントリ。

失敗モードと境界条件

  • relid が simple_rel_array_size を超える場合、または有効なベースリレーションでも非内部結合でもない場合は elog(ERROR) が発生する。
  • 内部結合以外の結合リレーションに対しては、エラーではなく NULL を返すことで安全にスキップする。

保守担当者向けメモ

  • 本関数は、プランナがベースリレーションを走査する際に、結合ノードを意図的に無視したいケースで使用される。
  • デバッグ目的で、relid が予期せぬ結合タイプでないかを検証するロジックが含まれている。
