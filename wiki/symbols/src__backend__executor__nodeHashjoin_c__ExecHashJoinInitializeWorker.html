ExecHashJoinInitializeWorker

定義

  • **種別**: function
  • **定義箇所**: src/backend/executor/nodeHashjoin.c:1751

目的

並列ハッシュ結合を実行するワーカープロセスにおいて、共有メモリ上のハッシュ結合状態をアタッチし、実行ノードの処理関数を並列実行用に切り替える。

シグネチャまたは形状

void ExecHashJoinInitializeWorker(HashJoinState *state, ParallelWorkerContext *pwcxt)

入力と出力

  • **入力**:
  • state: 実行中のハッシュ結合ノードの状態（HashJoinState）。
  • pwcxt: 並列ワーカーのコンテキスト情報（ParallelWorkerContext）。
  • **出力**: なし（副作用として state および hashNode の内部状態を更新）。

前提条件と不変条件

  • pwcxt->toc から plan_node_id に対応する ParallelHashJoinState が取得可能であること。
  • state の内側プランノードが HashState であること。

アルゴリズムまたは振る舞い

1. state からプランノードIDを取得し、pwcxt->toc を介して共有メモリ上の ParallelHashJoinState をルックアップする。 2. SharedFileSetAttach を呼び出し、共有一時ファイルセットを現在のワーカープロセスにアタッチする。 3. innerPlanState(state) を通じて取得した HashState に、ルックアップした parallel_state を設定する。 4. ExecSetExecProcNode を呼び出し、当該ノードの実行関数を ExecParallelHashJoin に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:1380（直接的な証拠）。
  • **呼び出し先**:
  • shm_toc_lookup
  • SharedFileSetAttach
  • innerPlanState
  • ExecSetExecProcNode

関連データ構造

  • HashJoinState: ハッシュ結合の実行状態を保持する構造体。
  • ParallelHashJoinState: 並列ハッシュ結合のための共有メモリ上の状態。
  • ParallelWorkerContext: 並列ワーカーの共有メモリコンテキスト。
  • HashState: 内側プランのハッシュ状態。

失敗モードと境界条件

  • shm_toc_lookup が指定された plan_node_id を見つけられない場合、false 引数によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は並列クエリ実行時の初期化フェーズで呼び出される。
  • ExecSetExecProcNode による関数ポインタの書き換えにより、以降の実行はシリアル版ではなく並列版のロジックへ誘導される。
