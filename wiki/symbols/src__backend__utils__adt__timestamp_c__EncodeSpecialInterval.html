EncodeSpecialInterval

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/timestamp.c:1597

目的

Interval 型の値が「無限大」などの特殊な状態（NOBEGIN または NOEND）である場合に、対応する文字列（EARLY または LATE）をバッファに書き込みます。

シグネチャまたは形状

static void EncodeSpecialInterval(const Interval *interval, char *str)

入力と出力

  • **入力**:
  • interval: 判定対象の Interval 構造体へのポインタ。
  • str: 結果の文字列を格納するバッファへのポインタ。
  • **出力**: なし（str が指すメモリ領域に文字列が書き込まれます）。

前提条件と不変条件

  • interval は有効なポインタである必要があります。
  • str は、EARLY または LATE の文字列を格納するのに十分なサイズを持つ必要があります。

アルゴリズムまたは振る舞い

1. INTERVAL_IS_NOBEGIN(interval) が真の場合、str に EARLY をコピーします。 2. INTERVAL_IS_NOEND(interval) が真の場合、str に LATE をコピーします。 3. 上記のいずれでもない場合、elog(ERROR, ...) を呼び出し、エラーとして処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/timestamp.c 内の interval_out 等の関数から呼び出されます（ソース内の参照箇所: 982行目）。
  • **呼び出し先**: strcpy（標準ライブラリ）、elog（PostgreSQL エラー報告関数）。

関連データ構造

  • Interval: src/include/datatype/timestamp.h で定義される、時間間隔を保持する構造体。

失敗モードと境界条件

  • **失敗モード**: interval が NOBEGIN でも NOEND でもない場合、elog(ERROR) によりトランザクションがアボートされます。
  • **境界条件**: interval が通常の時間間隔値である場合は、この関数はエラーを発生させます。

保守担当者向けメモ

  • 本関数は特殊な Interval 値の出力専用です。通常の Interval 値の出力には使用しないでください。
  • EARLY および LATE は、datetime.c 等で定義されている定数文字列を指します。
