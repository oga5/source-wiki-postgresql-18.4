freeGISTstate

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gist.c:1663

目的

GiST インデックス操作に関連するメモリリソースを解放し、GISTSTATE 構造体に関連付けられたメモリコンテキストを破棄します。

シグネチャまたは形状

void freeGISTstate(GISTSTATE *giststate)

入力と出力

  • **入力**: GISTSTATE *giststate - 解放対象の GiST 状態構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • 引数 giststate は有効な GISTSTATE 構造体を指している必要があります。
  • giststate->scanCxt が初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. giststate->scanCxt（スキャン用メモリコンテキスト）を MemoryContextDelete を用いて削除します。 2. この操作により、当該コンテキスト内に割り当てられたすべてのメモリが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistbuild.c: インデックス構築処理の終了時。
  • src/backend/access/gist/gistscan.c: インデックススキャンの終了時（gistendscan 等から）。
  • **呼び出し先**:
  • MemoryContextDelete（PostgreSQL メモリ管理 API）。

関連データ構造

  • GISTSTATE: GiST インデックス操作の実行状態を保持する構造体。gist_private.h で定義。

失敗モードと境界条件

  • giststate が NULL の場合、メモリ保護違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は GISTSTATE が保持する scanCxt を削除するだけであり、構造体自体のメモリ解放は呼び出し元に依存する場合があります。
  • gistbeginscan で作成されたリソースをクリーンアップするための標準的な手段です。
