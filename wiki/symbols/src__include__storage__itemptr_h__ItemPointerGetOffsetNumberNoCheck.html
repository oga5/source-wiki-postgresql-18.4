ItemPointerGetOffsetNumberNoCheck

定義

  • 種別: function
  • 定義箇所: src/include/storage/itemptr.h:113

目的

ItemPointerData 構造体から、検証処理（Assert）を介さずに直接オフセット番号（OffsetNumber）を抽出する。

シグネチャまたは形状

static inline OffsetNumber ItemPointerGetOffsetNumberNoCheck(const ItemPointerData *pointer)

入力と出力

  • **入力**: pointer (const ItemPointerData *) - 読み取り対象のアイテムポインタへのポインタ。
  • **出力**: OffsetNumber - 構造体内の ip_posid フィールドの値。

前提条件と不変条件

  • pointer は有効な ItemPointerData 構造体を指している必要がある。
  • 本関数は Assert による検証を行わないため、呼び出し側でポインタの妥当性を保証する必要がある。

アルゴリズムまたは振る舞い

1. 引数として受け取った pointer が指す構造体の ip_posid メンバに直接アクセスする。 2. 取得した値をそのまま戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c (複数箇所)
  • contrib/amcheck/verify_nbtree.c (複数箇所)
  • **呼び出し先**: なし（インライン関数として定義されており、直接的な関数呼び出しは発生しない）。

関連データ構造

  • ItemPointerData: タプル位置を保持する 6 バイトの構造体。
  • OffsetNumber: src/include/storage/off.h で定義される、ページ内のタプル位置を示す型。

失敗モードと境界条件

  • pointer が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • ip_posid に MaxOffsetNumber を超える特殊値（SpecTokenOffsetNumber や MovedPartitionsOffsetNumber 等）が含まれている場合、それらの値がそのまま返される。

保守担当者向けメモ

  • NoCheck の接尾辞が示す通り、パフォーマンスが極めて重要なパスや、既に妥当性が検証済みであると確信できる状況でのみ使用すること。
  • ItemPointerData のメモリレイアウト（パディングの有無）に依存するため、構造体定義を変更する際は本関数の動作に影響がないか確認が必要である。
