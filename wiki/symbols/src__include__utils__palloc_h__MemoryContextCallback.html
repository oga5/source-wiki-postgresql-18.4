MemoryContextCallback

定義

  • 種別: struct
  • 定義箇所: src/include/utils/palloc.h:47

目的

メモリコンテキスト（MemoryContext）がリセットまたは削除される直前に実行されるクリーンアップ処理を登録するための構造体です。

シグネチャまたは形状

typedef struct MemoryContextCallback
{
    MemoryContextCallbackFunction func; /* 実行する関数 */
    void       *arg;                    /* 関数に渡す引数 */
    struct MemoryContextCallback *next; /* リスト内の次の要素 */
} MemoryContextCallback;

入力と出力

  • **入力**: 登録時に func（関数ポインタ）と arg（任意のデータへのポインタ）を指定します。
  • **出力**: コンテキストの破棄時に、登録された func が arg を引数として呼び出されます。

前提条件と不変条件

  • 呼び出し元は、この構造体のメモリ領域を確保する責任を負います。
  • next ポインタにより、単方向リストとして管理されます。

アルゴリズムまたは振る舞い

1. MemoryContextRegisterResetCallback を使用してコンテキストに登録されます。 2. コンテキストがリセットまたは削除される際、登録されたコールバックリストが順次走査されます。 3. 各コールバックの func が arg を伴って実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/postgres_fdw.c、src/backend/executor/functions.c、src/backend/libpq/auth-oauth.c、src/backend/replication/pgoutput/pgoutput.c、src/include/utils/expandedrecord.h などで利用されています。
  • **呼び出し先**: 登録された MemoryContextCallbackFunction 型の関数が実行されます。

関連データ構造

  • MemoryContext: このコールバックを保持するメモリコンテキスト。
  • MemoryContextCallbackFunction: コールバック関数の型定義。
  • MemoryContextData (定義は nodes/memnodes.h): reset_cbs フィールドを通じてコールバックリストを保持します。

失敗モードと境界条件

  • 構造体自体のメモリ確保に失敗した場合、登録処理自体が失敗します。
  • コールバック関数内で例外が発生した場合の挙動は、コンテキスト管理側の実装に依存します。

保守担当者向けメモ

  • src/backend/utils/mmgr/mcxt.c にあるコールバック処理の実装を参照してください。
  • コンテキストの破棄時に実行されるため、リソース解放の順序には注意が必要です。
