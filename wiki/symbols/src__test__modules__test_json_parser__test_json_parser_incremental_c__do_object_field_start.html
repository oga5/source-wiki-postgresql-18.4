do_object_field_start

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_json_parser/test_json_parser_incremental.c:271

目的

JSONオブジェクトのフィールド開始イベントを処理し、フィールド名をエスケープして標準出力へ整形表示するコールバック関数。

シグネチャまたは形状

static JsonParseErrorType do_object_field_start(void *state, char *fname, bool isnull)

入力と出力

  • **入力**:
  • state: DoState 構造体へのポインタ（void * として渡される）。
  • fname: フィールド名文字列。
  • isnull: フィールド値がNULLかどうかを示すフラグ（本関数内では未使用）。
  • **出力**:
  • JSON_SUCCESS: 処理成功時に常に返される。

前提条件と不変条件

  • state 引数は DoState 型にキャスト可能であること。
  • _state->buf が初期化済みであること。

アルゴリズムまたは振る舞い

1. state を DoState 型にキャストする。 2. _state->elem_is_first が false の場合、先行するフィールドとの区切りとして ",\n" を出力する。 3. _state->buf をリセットし、escape_json を用いて fname をエスケープしてバッファに格納する。 4. エスケープされたフィールド名を標準出力に書き出し、続けて : を出力する。 5. _state->elem_is_first を false に更新する。 6. JSON_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_json_parser/test_json_parser_incremental.c 内の JsonSemAction 構造体（object_field_start フィールド）を通じて、パーサーから間接的に呼び出される。
  • **呼び出し先**:
  • escape_json: フィールド名のエスケープ処理。
  • resetStringInfo: バッファのクリア。
  • printf: 標準出力への書き出し。

関連データ構造

  • DoState: パース状態を保持する構造体。elem_is_first フラグと buf（StringInfo）を含む。

失敗モードと境界条件

  • 本関数自体はエラーを返さない設計となっている。
  • fname が NULL の場合の挙動は定義されていない。

保守担当者向けメモ

  • 本関数はテストモジュールの一部であり、JSONの構造を視覚的に再構成するために使用される。
  • isnull 引数は現在の実装では無視されているが、将来的にフィールド値が NULL の場合の表示制御が必要になった場合は活用可能である。
