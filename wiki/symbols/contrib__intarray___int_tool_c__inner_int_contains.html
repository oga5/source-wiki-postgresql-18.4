inner_int_contains

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_tool.c:14

目的

2つの整数配列（ArrayType）を比較し、配列 b のすべての要素が配列 a に含まれているか（包含関係にあるか）を判定します。

シグネチャまたは形状

bool inner_int_contains(ArrayType *a, ArrayType *b)

入力と出力

  • **入力**:
  • a: 比較対象となる整数配列（ArrayType*）。
  • b: 含まれているかを確認する整数配列（ArrayType*）。
  • **出力**:
  • bool: b の全要素が a に存在する場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力される ArrayType *a および ArrayType *b は、**ソート済み**である必要があります。ソートされていない配列を渡した場合、正しい結果は保証されません。
  • ARRNELEMS および ARRPTR マクロにより、配列の要素数とデータポインタが正しく取得できる有効な ArrayType であることが前提です。

アルゴリズムまたは振る舞い

1. 両方の配列の要素数（na, nb）とデータポインタ（da, db）を取得します。 2. 2つのポインタを先頭から同時に走査し、以下のロジックで比較を行います。

  • da[i] < db[j]: a の現在の要素が小さいため、a のポインタをインクリメントします。
  • da[i] == db[j]: 一致したため、一致数 n を増やし、両方のポインタをインクリメントします。
  • da[i] > db[j]: b の現在の要素が a に存在しないことが確定するため、ループを終了します。

    • 3. 最終的に一致した要素数 n が b の全要素数 nb と等しいかどうかを判定して返します。 4. 計算量は $O(N+M)$ （$N, M$ は各配列の要素数）です。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_gist.c: GiST インデックスのサポート関数内で使用。
  • contrib/intarray/_int_op.c: SQL 演算子（@> 等）の実装から使用。
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行わず、ARRNELEMS や ARRPTR といった PostgreSQL の配列操作マクロのみを使用します。

関連データ構造

  • ArrayType: PostgreSQL の標準的な配列データ構造。

失敗モードと境界条件

  • 空の配列 b が渡された場合、n == nb（0 == 0）となり true を返します（空集合はすべての集合の部分集合であるため）。
  • a が空で b が空でない場合、false を返します。

保守担当者向けメモ

  • 本関数は線形時間で動作するよう設計されています。パフォーマンス維持のため、ロジックを変更する際は走査の効率を損なわないように注意してください。
  • 入力配列がソート済みであるという前提は、呼び出し側（_int_op.c や _int_gist.c）で保証されている必要があります。
