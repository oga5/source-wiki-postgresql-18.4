add_variable_to_head

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/variable.c:400

目的

SQL文の入出力引数リスト（arguments）の先頭に、新しい変数およびインジケータ変数のペアを追加します。

シグネチャまたは形状

void add_variable_to_head(struct arguments **list, struct variable *var, struct variable *ind)

入力と出力

  • **入力**:
  • list: 更新対象の arguments リストへのポインタ。
  • var: リストに追加する変数（struct variable）。
  • ind: 変数に関連付けられたインジケータ変数（struct variable、NULL可）。
  • **出力**: なし（list が指すリストが更新されます）。

前提条件と不変条件

  • list は有効な arguments リストのポインタである必要があります。
  • mm_alloc を使用してメモリを確保するため、メモリ不足時はプロセスが終了する可能性があります。

アルゴリズムまたは振る舞い

1. mm_alloc を使用して、新しい arguments 構造体インスタンスをメモリ上に確保します。 2. 確保した構造体の variable フィールドに var を、indicator フィールドに ind を格納します。 3. 新しい構造体の next ポインタを現在のリストの先頭（*list）に向けます。 4. リストの先頭ポインタを、新しく作成した構造体を指すように更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での具体的な呼び出し箇所は、提供された情報からは特定できません。
  • **呼び出し先**: mm_alloc（メモリ確保用）。

関連データ構造

  • struct arguments: SQL文の引数リストを構成するリンクリスト構造体。
  • struct variable: ECPGプリプロセッサで管理されるホスト変数の定義。

失敗モードと境界条件

  • mm_alloc が失敗した場合、内部的に mmfatal が呼び出され、プログラムが異常終了します。

保守担当者向けメモ

  • 本関数は add_variable_to_tail と対になる関数です。リストの順序がコード生成時の引数順序に直結するため、呼び出し時には順序の意図を確認してください。
