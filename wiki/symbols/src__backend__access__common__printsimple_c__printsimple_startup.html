printsimple_startup

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/printsimple.c:30

目的

DestReceiver インターフェースの一部として、クエリ結果の列メタデータをフロントエンドへ送信する。カタログアクセスが制限された環境において、RowDescription メッセージを構築し、列定義を伝達する役割を担う。

シグネチャまたは形状

void printsimple_startup(DestReceiver *self, int operation, TupleDesc tupdesc)

入力と出力

  • **入力**:
  • self: DestReceiver 構造体へのポインタ。
  • operation: 実行される操作の種類（本関数内では未使用）。
  • tupdesc: 送信対象となるタプルの記述子（TupleDesc）。
  • **出力**: なし（戻り値は void）。副作用としてフロントエンドへ RowDescription メッセージを送信する。

前提条件と不変条件

  • プロトコルバージョン 3.0 で動作することを前提とする。
  • tupdesc は有効な列情報を含んでいる必要がある。
  • カタログアクセスが不可能な環境（walsender 等）での使用を想定している。

アルゴリズムまたは振る舞い

1. pq_beginmessage を呼び出し、PqMsg_RowDescription メッセージの構築を開始する。 2. tupdesc->natts を用いて列数を送信する。 3. 各列についてループし、以下の情報を順次送信する:

  • 列名 (attname)
  • テーブル OID（固定値 0）
  • 属性番号（固定値 0）
  • 属性型 OID (atttypid)
  • 属性長 (attlen)
  • 属性修飾子 (atttypmod)
  • フォーマットコード（固定値 0）

    • 4. pq_endmessage を呼び出し、メッセージの送信を完了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/dest.c 内の DestReceiver 関連処理から呼び出されることが示唆されている。
  • **呼び出し先**: pq_beginmessage, pq_sendint16, pq_sendstring, pq_sendint32, pq_endmessage などの libpq メッセージ構築関数群。

関連データ構造

  • DestReceiver: 出力先を抽象化する構造体。
  • TupleDesc: タプルの構造を定義する記述子。
  • Form_pg_attribute: 列の属性情報。

失敗モードと境界条件

  • tupdesc が NULL の場合、あるいは不正なメモリ領域を指している場合はクラッシュする可能性がある。
  • メッセージ構築中に通信エラーが発生した場合、libpq 側のエラーハンドリングに依存する。

保守担当者向けメモ

  • 本関数はカタログを参照しないため、列のメタデータは TupleDesc から直接取得される。
  • table oid や attnum が 0 に固定されている点は、本モジュールが軽量な転送に特化していることによる仕様である。
