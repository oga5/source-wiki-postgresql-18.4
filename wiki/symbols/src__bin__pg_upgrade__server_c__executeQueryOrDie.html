executeQueryOrDie

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/server.c:123

目的

データベースに対してクエリを実行し、成功した場合は結果を返し、失敗した場合はエラーログを出力してプログラムを即座に終了させるためのラッパー関数です。

シグネチャまたは形状

PGresult *executeQueryOrDie(PGconn *conn, const char *fmt, ...);

入力と出力

  • **入力**:
  • conn: PGconn へのポインタ。
  • fmt: printf 形式のクエリ文字列。
  • ...: fmt に対応する可変長引数。
  • **出力**:
  • 成功時: クエリ結果を保持する PGresult へのポインタ。
  • 失敗時: 戻り値なし（プログラムが exit(1) で終了するため）。

前提条件と不変条件

  • conn は有効なデータベース接続である必要があります。
  • 戻り値として受け取った PGresult は、呼び出し元で PQclear を実行して解放する必要があります。

アルゴリズムまたは振る舞い

1. 可変長引数を用いてクエリ文字列を構築します（QUERY_ALLOC サイズの静的バッファを使用）。 2. PQexec を呼び出し、クエリを実行します。 3. PQresultStatus を確認し、結果が PGRES_TUPLES_OK または PGRES_COMMAND_OK であることを検証します。 4. 成功した場合は PGresult を返します。 5. 失敗した場合は、エラーメッセージをログに出力し、PQclear で結果を解放、PQfinish で接続を閉じた後、exit(1) でプロセスを終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/check.c (複数箇所)
  • src/bin/pg_upgrade/info.c (複数箇所)
  • **呼び出し先**:
  • libpq 関数群 (PQexec, PQresultStatus, PQerrorMessage, PQclear, PQfinish)
  • pg_log (ログ出力用)

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • PGresult: libpq のクエリ結果ハンドル。
  • ExecStatusType: クエリ実行ステータスの列挙型。

失敗モードと境界条件

  • クエリ実行が PGRES_TUPLES_OK または PGRES_COMMAND_OK 以外を返した場合、プログラムは即座に終了します。
  • 静的バッファ query のサイズ（QUERY_ALLOC）を超えるクエリ文字列は切り詰められる可能性があります。

保守担当者向けメモ

  • 本関数はエラー時に PQfinish(conn) を呼び出して接続を破棄するため、呼び出し側で接続の再利用を想定している場合は注意が必要です。
  • 戻り値の PQclear 忘れはメモリリークの原因となるため、必ず呼び出し側で管理してください。
