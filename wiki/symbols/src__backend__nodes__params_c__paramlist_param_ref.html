paramlist_param_ref

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/params.c:130

目的

パーサーがクエリ内のパラメータ参照（ParamRef）を解決する際に、ParamListInfo から対応するパラメータ情報を取得し、Param ノードを生成する。

シグネチャまたは形状

static Node *paramlist_param_ref(ParseState *pstate, ParamRef *pref)

入力と出力

  • **入力**:
  • pstate: パーサーの状態（p_ref_hook_state に ParamListInfo が格納されている）。
  • pref: 解決対象のパラメータ参照ノード。
  • **出力**:
  • 成功時: 解決された Param ノードへのポインタ。
  • 失敗時: NULL（パラメータ番号が範囲外、または型情報が未定義の場合）。

前提条件と不変条件

  • pstate->p_ref_hook_state は ParamListInfo 構造体としてキャスト可能であること。
  • ParamListInfo 内の numParams が正しく設定されていること。

アルゴリズムまたは振る舞い

1. pstate->p_ref_hook_state から ParamListInfo を取得する。 2. パラメータ番号が 1 から numParams の範囲内か検証する。範囲外であれば NULL を返す。 3. paramFetch フックが設定されていればそれを呼び出し、そうでなければ内部の params 配列から ParamExternData を取得する。 4. 取得したパラメータの型（ptype）が有効か確認する。無効であれば NULL を返す。 5. Param ノードを新規作成し、以下の値を設定する：

  • paramkind: PARAM_EXTERN
  • paramid: パラメータ番号
  • paramtype: 取得した型
  • paramcollid: get_typcollation で取得した照合順序
  • location: pref->location

    • 6. 生成した Param ノードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/params.c 内の paramlist_parser_setup 関数において、pstate->p_paramref_hook に関数ポインタとして代入される。
  • **呼び出し先**:
  • paramLI->paramFetch（動的パラメータ取得時）
  • makeNode(Param)（ノード生成）
  • get_typcollation（型情報の取得）

関連データ構造

  • ParamListInfo: パラメータリストの保持構造体。
  • ParamRef: パーサーが生成するパラメータ参照ノード。
  • ParamExternData: パラメータのメタデータ（型情報など）を保持する構造体。
  • Param: 実行エンジンが利用するパラメータノード。

失敗モードと境界条件

  • パラメータ番号が 0 以下、または numParams を超える場合は NULL を返す。
  • パラメータの型（ptype）が OidIsValid でない（未定義）場合は NULL を返す。

保守担当者向けメモ

  • 本関数は ParseState のフックとして動作するため、直接呼び出されることはなく、パーサーの解析プロセスを通じて間接的に実行される。
  • paramtypmod は常に -1 に固定されている。
