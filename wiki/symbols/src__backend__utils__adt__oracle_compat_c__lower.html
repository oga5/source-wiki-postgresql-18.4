lower

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oracle_compat.c:48

目的

入力された text 型文字列を小文字に変換し、新しい text 型として返します。Oracle データベースとの互換性を目的とした関数です。

シグネチャまたは形状

Datum lower(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される text 型のデータ。
  • **出力**: 小文字に変換された新しい text 型の Datum。

前提条件と不変条件

  • 入力文字列は PostgreSQL の text 型である必要があります。
  • データベースの照合順序（Collation）設定に依存して変換が行われます。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP(0) を使用して入力文字列を取得します。 2. str_tolower を呼び出し、入力文字列のデータ、サイズ、および現在の照合順序を渡して小文字変換を実行します。 3. cstring_to_text を使用して、変換後の C 文字列を PostgreSQL の text 型に変換します。 4. 変換に使用した一時的なメモリ領域を pfree で解放します。 5. PG_RETURN_TEXT_P を使用して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報には、本関数を直接呼び出している具体的な関数名は記載されていません（contrib/amcheck 等のソースコードに含まれる "lower" という文字列は、本関数への参照ではなく、B-tree インデックスの構造や境界条件に関する記述です）。
  • **呼び出し先**:
  • str_tolower (src/backend/utils/formatting.c)
  • cstring_to_text
  • pfree

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • メモリ不足時には palloc（cstring_to_text 内部で使用）によりエラーが発生する可能性があります。
  • 照合順序の設定によっては、予期しない文字変換結果となる場合があります。

保守担当者向けメモ

  • 本関数は utils/formatting.h で定義されている str_tolower をラップする薄いラッパーです。
  • マルチバイト文字の処理は str_tolower に依存しているため、文字エンコーディングに関連する修正が必要な場合は、まず formatting.c 側の実装を確認してください。
