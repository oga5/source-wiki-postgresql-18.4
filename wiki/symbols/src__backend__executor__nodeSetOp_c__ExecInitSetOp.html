ExecInitSetOp

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSetOp.c:563

目的

SQLの集合演算（INTERSECT / EXCEPT）を実行するための SetOpState ノードを初期化し、実行に必要なメモリコンテキスト、ハッシュテーブル、またはソートサポート情報を準備する。

シグネチャまたは形状

SetOpState *ExecInitSetOp(SetOp *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 実行対象の SetOp プランノード。
  • estate: クエリ実行状態（EState）。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの SetOpState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK が含まれていないこと（Assert で検証）。
  • SetOp ノードは射影（Projection）を行わないため、ps.ps_ProjInfo は NULL に設定される。

アルゴリズムまたは振る舞い

1. **状態構造体の作成**: SetOpState を割り当て、実行関数 ExecSetOp を登録する。 2. **メモリ管理**: SETOP_HASHED 戦略の場合、ハッシュテーブル保持用に tableContext を作成する。 3. **子ノードの初期化**: outerPlan および innerPlan を初期化する。ハッシュ戦略時は EXEC_FLAG_REWIND を無効化する。 4. **スロットの初期化**: ExecInitResultTupleSlotTL を呼び出し、ソート戦略時は右入力用に firstTupleSlot を追加で確保する。 5. **比較・ハッシュ準備**:

  • SETOP_HASHED の場合: execTuplesHashPrepare で等価関数とハッシュ関数を準備し、build_hash_table でハッシュテーブルを構築する。
  • SETOP_SORTED の場合: SortSupportData を割り当て、各列のソート順序や照合順序を準備する。

    • 6. **戻り値**: 初期化された setopstate を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内の ExecInitNode から呼び出される。
  • **呼び出し先**: makeNode, ExecAssignExprContext, AllocSetContextCreate, ExecInitNode, ExecInitResultTupleSlotTL, ExecInitExtraTupleSlot, execTuplesHashPrepare, PrepareSortSupportFromOrderingOp, build_hash_table。

関連データ構造

  • SetOpState: 本関数が初期化する主要な状態構造体。
  • SetOp: プランノード。
  • SortSupportData: ソート戦略時に使用される比較サポート情報。
  • EState: 実行状態。

失敗モードと境界条件

  • メモリ不足時に AllocSetContextCreate や palloc0 がエラーを発生させる可能性がある。
  • 不正な eflags が渡された場合、Assert によりデバッグビルドで停止する。

保守担当者向けメモ

  • SETOP_HASHED モードでは tableContext を使用してメモリを管理しており、ExecReScanSetOp でこのコンテキストをリセットすることで効率的な再利用を実現している。
  • UNION 演算はこのノードでは処理されないため、本関数は INTERSECT または EXCEPT のみに特化している。
