pg_sha384_init

定義

  • 種別: function
  • 定義箇所: src/common/sha2.c:933

目的

SHA-384 ハッシュ計算のためのコンテキスト構造体を初期化し、計算開始可能な状態にリセットします。

シグネチャまたは形状

void pg_sha384_init(pg_sha384_ctx *context)

入力と出力

  • **入力**: pg_sha384_ctx *context（初期化対象のコンテキスト構造体へのポインタ）
  • **出力**: なし（副作用として context が書き換えられます）

前提条件と不変条件

  • context が NULL の場合、関数は何も行わず直ちに復帰します。
  • 正常な呼び出し後、context->state は sha384_initial_hash_value で初期化され、context->buffer はゼロクリアされ、context->bitcount は 0 にリセットされます。

アルゴリズムまたは振る舞い

1. context が NULL かどうかを確認し、該当する場合は終了します。 2. sha384_initial_hash_value を用いて context->state を初期ハッシュ値で上書きします。 3. context->buffer を PG_SHA384_BLOCK_LENGTH 分だけゼロで埋めます。 4. context->bitcount の 2 つの要素を 0 に設定し、処理済みビット数をリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/cryptohash.c:120 からの呼び出しが確認されています。
  • **呼び出し先**: memcpy および memset（標準ライブラリ関数）を呼び出します。

関連データ構造

  • pg_sha384_ctx: SHA-384 の状態を保持する構造体（src/common/sha2_int.h で定義）。

失敗モードと境界条件

  • context に NULL ポインタが渡された場合、安全に無視されます。
  • それ以外の境界条件やエラー報告の仕組みは本関数には存在しません。

保守担当者向けメモ

  • 本関数は SHA-512 の実装をベースとした SHA-384 用の初期化ルーチンです。
  • sha384_initial_hash_value 定数および PG_SHA384_BLOCK_LENGTH の定義については sha2_int.h を参照してください。
