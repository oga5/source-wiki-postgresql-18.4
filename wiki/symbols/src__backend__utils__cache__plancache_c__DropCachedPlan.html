DropCachedPlan

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/plancache.c:573

目的

キャッシュされたクエリ計画（CachedPlanSource）を破棄し、関連するメモリリソースを解放する。

シグネチャまたは形状

void DropCachedPlan(CachedPlanSource *plansource)

入力と出力

  • **入力**: plansource (CachedPlanSource *) - 破棄対象のキャッシュ済み計画ソースへのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • plansource->magic が CACHEDPLANSOURCE_MAGIC と一致している必要がある（アサーションによる検証）。

アルゴリズムまたは振る舞い

1. **リストからの除外**: plansource->is_saved が真であれば、保持されているリストから削除し、フラグを偽に更新する。 2. **参照カウントの解放**: ReleaseGenericPlan(plansource) を呼び出し、汎用計画の参照カウントを減算する。 3. **無効化**: plansource->magic を 0 に設定し、以降の利用を無効化する。 4. **メモリ解放**: plansource->is_oneshot が偽である場合、MemoryContextDelete を用いて plansource->context を破棄する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/prepare.c
  • src/backend/executor/functions.c
  • src/backend/executor/spi.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • dlist_delete
  • ReleaseGenericPlan
  • MemoryContextDelete

関連データ構造

  • CachedPlanSource: クエリのソース情報とキャッシュ状態を保持する構造体。

失敗モードと境界条件

  • is_oneshot が真の場合、メモリコンテキストの削除はスキップされる。
  • 呼び出し時に magic 値が不正な場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は CachedPlanSource のライフサイクル終了時に呼び出される。
  • is_oneshot の場合はメモリ管理の挙動が異なるため、呼び出し元で CachedPlanSource の生成方法を把握しておく必要がある。
