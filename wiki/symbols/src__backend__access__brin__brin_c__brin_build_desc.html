brin_build_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin.c:1580

目的

BRINインデックス操作に必要な記述子（BrinDesc）を初期化し、各インデックス列に対応する演算クラス情報（BrinOpcInfo）を収集・構築する。

シグネチャまたは形状

BrinDesc *brin_build_desc(Relation rel)

入力と出力

  • **入力**: Relation rel（対象のBRINインデックスリレーション）
  • **出力**: 初期化された BrinDesc 構造体へのポインタ。

前提条件と不変条件

  • rel は有効なBRINインデックスリレーションであること。
  • 各インデックス列に対して BRIN_PROCNUM_OPCINFO サポート関数が定義されていること。
  • 戻り値の BrinDesc は、関数内で作成された専用のメモリコンテキスト（bd_context）に割り当てられる。

アルゴリズムまたは振る舞い

1. brin desc cxt という名称の新しいメモリコンテキストを作成し、以降の割り当てをそこに切り替える。 2. インデックスの各列に対し、index_getprocinfo を用いて BRIN_PROCNUM_OPCINFO を取得し、演算クラス固有の情報をロードする。 3. 全列の oi_nstored を合計し、インデックス全体で保持する格納データの総数を算出する。 4. BrinDesc 構造体を割り当て、リレーション情報、タプル記述子、および各列の BrinOpcInfo 配列を格納する。 5. bd_disktdesc は遅延生成のため初期値 NULL に設定する。 6. 元のメモリコンテキストに復帰し、構築した BrinDesc を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c 内の brininsert、bringetbitmap、brinbuild 等の各処理。
  • contrib/pageinspect/brinfuncs.c
  • **呼び出し先**:
  • index_getprocinfo (PostgreSQL インデックスAMインターフェース)
  • AllocSetContextCreate, MemoryContextSwitchTo (メモリ管理)
  • palloc, pfree (メモリ割り当て)

関連データ構造

  • BrinDesc: インデックスのメタデータと演算クラス情報を保持する主要構造体。
  • BrinOpcInfo: 各演算クラスが提供する、インデックス列ごとの要約情報に関するメタデータ。
  • Relation: PostgreSQL のリレーション記述子。

失敗モードと境界条件

  • メモリ不足時には palloc 等がエラーを発生させる。
  • インデックス列に BRIN_PROCNUM_OPCINFO が定義されていない場合、index_getprocinfo がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数で作成された BrinDesc は、呼び出し側で不要になった際に brin_free_desc を使用して適切にメモリ解放を行う必要がある。
  • bd_disktdesc の遅延生成ロジックが将来的に変更される場合、本関数の初期化処理との整合性に注意すること。
