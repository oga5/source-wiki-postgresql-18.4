heapcheck_read_stream_next_unskippable

定義

  • 種別: function
  • 定義箇所: contrib/amcheck/verify_heapam.c:880

目的

ヒープリレーションの走査において、指定されたスキップオプション（SKIP_PAGES_ALL_FROZEN または SKIP_PAGES_ALL_VISIBLE）に基づき、検証対象外のページをスキップして次に検証すべきブロック番号を返します。

シグネチャまたは形状

static BlockNumber heapcheck_read_stream_next_unskippable(
    ReadStream *stream,
    void *callback_private_data,
    void *per_buffer_data
)

入力と出力

  • **入力**:
  • stream: 使用される ReadStream オブジェクト。
  • callback_private_data: HeapCheckReadStreamData 構造体へのポインタ。走査範囲およびスキップ設定を保持。
  • per_buffer_data: 現在未使用。
  • **出力**:
  • 次に検証すべき BlockNumber。走査範囲が終了した場合は InvalidBlockNumber を返します。

前提条件と不変条件

  • callback_private_data は HeapCheckReadStreamData 型としてキャスト可能であること。
  • p->range に有効な走査範囲（current_blocknum から last_exclusive まで）が設定されていること。
  • p->rel がオープン状態であること。

アルゴリズムまたは振る舞い

1. p->range.current_blocknum から p->range.last_exclusive までブロック番号を順次インクリメントしながらループします。 2. 各ブロックに対して visibilitymap_get_status を呼び出し、可視性マップのビット情報を取得します。 3. p->skip_option の設定を確認します：

  • SKIP_PAGES_ALL_FROZEN が指定されている場合、VISIBILITYMAP_ALL_FROZEN ビットが立っていればそのページをスキップします。
  • SKIP_PAGES_ALL_VISIBLE が指定されている場合、VISIBILITYMAP_ALL_VISIBLE ビットが立っていればそのページをスキップします。

    • 4. スキップ条件に該当しないページが見つかった場合、そのブロック番号を返します。 5. ループ終了まで該当ページが見つからない場合、InvalidBlockNumber を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_heapam.c 内で ReadStream のコールバックとして設定されます（ソースコード行 467）。
  • **呼び出し先**: visibilitymap_get_status を呼び出します。

関連データ構造

  • HeapCheckReadStreamData: 走査範囲、リレーション情報、スキップオプションを保持する構造体。
  • ReadStream: PostgreSQL のページ走査用 API。

失敗モードと境界条件

  • 走査範囲が空の場合、直ちに InvalidBlockNumber を返します。
  • 破損した可視性マップ情報が読み込まれた場合、誤ったスキップ判断が行われる可能性があります。

保守担当者向けメモ

  • 本関数は ReadStream のコールバックとして動作するため、ロックの取得タイミング等、呼び出し元のコンテキストに依存する制約に注意してください。
  • per_buffer_data は現在の実装では使用されていませんが、将来的な拡張のためにシグネチャに含まれています。
