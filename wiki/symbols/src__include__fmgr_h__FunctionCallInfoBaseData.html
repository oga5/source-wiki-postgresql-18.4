FunctionCallInfoBaseData

定義

  • 種別: struct
  • 定義箇所: src/include/fmgr.h:85

目的

PostgreSQL の関数呼び出しにおける引数、実行コンテキスト、および戻り値の状態を保持するための基底データ構造です。

シグネチャまたは形状

typedef struct FunctionCallInfoBaseData
{
    FmgrInfo   *flinfo;
    fmNodePtr   context;
    fmNodePtr   resultinfo;
    Oid         fncollation;
    bool        isnull;
    short       nargs;
    NullableDatum args[FLEXIBLE_ARRAY_MEMBER];
} FunctionCallInfoBaseData;

入力と出力

  • **入力**: 関数呼び出しに必要なメタデータ（flinfo）、コンテキスト情報、照合順序（fncollation）、および可変長の引数配列（args）。
  • **出力**: 関数実行結果の NULL 状態（isnull）および args 配列内の NullableDatum を通じた引数情報の受け渡し。

前提条件と不変条件

  • args は可変長配列（FLEXIBLE_ARRAY_MEMBER）として定義されており、構造体インスタンスの確保時には nargs に応じた十分なメモリ領域が必要です。
  • isnull は、関数が NULL を返す場合に呼び出された関数側で true に設定される必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: 呼び出し元は flinfo や nargs を設定し、必要に応じて args 配列を埋めます。 2. **実行**: 関数マネージャを通じて関数が呼び出される際、この構造体が引数コンテナとして機能します。 3. **結果処理**: 呼び出された関数は args から引数を取得し、処理結果を Datum として返し、必要に応じて isnull を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execExprInterp.c、src/backend/utils/sort/sortsupport.c 等で、関数実行の準備段階としてインスタンス化または参照されています。
  • **呼び出し先**: 具体的な関数呼び出し先は、flinfo メンバが指す FmgrInfo に依存します。

関連データ構造

  • FunctionCallInfo: 本構造体へのポインタ型（typedef struct FunctionCallInfoBaseData *FunctionCallInfo）。
  • FmgrInfo: 関数呼び出しのメタデータを保持する構造体。
  • NullableDatum: args 配列の各要素を構成する型。

失敗モードと境界条件

  • nargs を超える args へのアクセスは、メモリ破壊を引き起こす可能性があります。
  • 構造体確保時に nargs 分のメモリを確保していない場合、args 配列へのアクセスで境界外参照が発生します。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を含むため、スタック上に直接宣言する際は注意が必要です。
  • アライメントを保証するため、FunctionCallInfoBaseData との共用体（union）を使用するマクロが fmgr.h 内で提供されています。
