build_pgstattuple_type

定義

  • 種別: function
  • 定義箇所: contrib/pgstattuple/pgstattuple.c:91

目的

集計済みの統計情報構造体（pgstattuple_type）を、SQL 関数から返却可能な Datum（複合型）形式に変換する。

シグネチャまたは形状

Datum build_pgstattuple_type(pgstattuple_type *stat, FunctionCallInfo fcinfo)

入力と出力

  • **入力**:
  • stat: 統計データが格納された pgstattuple_type 構造体へのポインタ。
  • fcinfo: 呼び出し元の関数情報。
  • **出力**:
  • Datum: 構築された HeapTuple を変換した結果。

前提条件と不変条件

  • fcinfo から取得される戻り値の型は、複合型（TYPEFUNC_COMPOSITE）である必要がある。
  • stat->table_len が 0 の場合、算出されるパーセンテージ（tuple_percent, dead_tuple_percent, free_percent）はすべて 0.0 となる。

アルゴリズムまたは振る舞い

1. get_call_result_type を用いて、戻り値のタプル記述子（TupleDesc）を取得する。 2. TupleDescGetAttInMetadata を使用し、C 文字列からタプルを構築するためのメタデータを準備する。 3. stat の各フィールド値に基づき、ライブタプル、デッドタプル、空き容量の割合を計算する。 4. 9 つの統計項目（テーブル長、タプル数、タプル長、タプル割合、デッドタプル数、デッドタプル長、デッドタプル割合、空き容量、空き容量割合）を文字列バッファに格納する。 5. BuildTupleFromCStrings を呼び出し、文字列配列から HeapTuple を生成する。 6. HeapTupleGetDatum を介して Datum に変換し、呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgstattuple/pgstattuple.c:404
  • contrib/pgstattuple/pgstattuple.c:574
  • **呼び出し先**:
  • get_call_result_type
  • TupleDescGetAttInMetadata
  • BuildTupleFromCStrings
  • HeapTupleGetDatum

関連データ構造

  • pgstattuple_type: 統計情報の保持に使用される構造体。
  • TupleDesc: タプルの構造定義。
  • AttInMetadata: 文字列からタプルを構築するためのメタデータ。

失敗モードと境界条件

  • get_call_result_type が複合型を返さない場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は 9 つの固定カラム（NCOLUMNS）を前提としており、出力形式を変更する場合は NCOLUMNS および snprintf の順序を同期させる必要がある。
  • NCHARS は各フィールドの文字列長制限として定義されている。
