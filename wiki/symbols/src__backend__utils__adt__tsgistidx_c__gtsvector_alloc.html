gtsvector_alloc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsgistidx.c:146

目的

GiST インデックス用の SignTSVector 構造体をメモリ上に確保し、初期化します。

シグネチャまたは形状

static SignTSVector *gtsvector_alloc(int flag, int len, BITVECP sign)

入力と出力

  • **入力**:
  • flag: SignTSVector のフラグ（ARRKEY, SIGNKEY, ALLISTRUE 等）。
  • len: 確保するデータの長さ。
  • sign: ビットベクトルデータへのポインタ（SIGNKEY が指定されている場合に使用）。
  • **出力**:
  • 初期化済みの SignTSVector 構造体へのポインタ。

前提条件と不変条件

  • CALCGTSIZE(flag, len) マクロにより、ヘッダーサイズとデータ長を合わせた適切なメモリサイズが計算されます。
  • palloc を使用するため、PostgreSQL のメモリコンテキスト内で呼び出される必要があります。

アルゴリズムまたは振る舞い

1. CALCGTSIZE を用いて必要なメモリサイズを算出します。 2. palloc でメモリを確保します。 3. SET_VARSIZE で構造体の可変長サイズを設定し、flag を格納します。 4. flag に SIGNKEY が含まれ、かつ ALLISTRUE フラグが立っていない場合、引数 sign から GETSIGN(res) へデータをコピーします。 5. 確保したメモリ領域へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsgistidx.c 内の各所（gtsvector_compress, gtsvector_picksplit 等）から、インデックスキーの生成時に呼び出されます。
  • **呼び出し先**:
  • palloc (PostgreSQL メモリ管理)
  • SET_VARSIZE, CALCGTSIZE, GETSIGN (マクロ)
  • memcpy (標準ライブラリ)

関連データ構造

  • SignTSVector: GiST インデックス内に格納されるキー構造体。
  • BITVECP: ビットベクトルを指すポインタ型。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • sign が NULL の状態で SIGNKEY を指定した場合、メモリのコピーは行われません。

保守担当者向けメモ

  • 本関数は tsvector の GiST インデックス構築におけるメモリ確保の共通インターフェースです。構造体のレイアウト変更やヘッダーサイズの計算ロジックを変更する際は、CALCGTSIZE マクロと併せて確認してください。
