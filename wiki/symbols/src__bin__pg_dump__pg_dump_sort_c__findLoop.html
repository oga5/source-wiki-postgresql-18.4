findLoop

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dump_sort.c:854

目的

依存関係グラフにおいて、指定された startPoint オブジェクトへ戻る循環参照（ループ）を深さ優先探索で特定する。

シグネチャまたは形状

static int findLoop(DumpableObject *obj, DumpId startPoint, bool *processed, DumpId *searchFailed, DumpableObject **workspace, int depth);

入力と出力

  • **入力**:
  • obj: 現在探索中の DumpableObject。
  • startPoint: 探索の起点となる DumpId。
  • processed: 既に処理済みのオブジェクトを追跡するフラグ配列。
  • searchFailed: 特定の startPoint から到達不能と判明したオブジェクトを記録する配列。
  • workspace: 現在の探索パスを保持するスタック領域。
  • depth: 現在の再帰深さ。
  • **出力**:
  • 戻り値: ループが見つかった場合は depth（パスの長さ）、見つからなかった場合は 0。

前提条件と不変条件

  • workspace 配列は、循環参照による無限再帰を防ぐために現在の探索パスを保持する。
  • searchFailed 配列により、一度探索してループが見つからなかったパスの再探索をスキップする。
  • processed 配列により、既に処理済みのループとの重複検出を回避する。

アルゴリズムまたは振る舞い

1. processed または searchFailed に該当する場合、探索を打ち切る。 2. workspace 内を走査し、現在の obj が既にパスに含まれている場合は無限再帰を防ぐため 0 を返す。 3. obj を workspace に追加する。 4. obj の直接の依存関係の中に startPoint が存在するか確認し、あればループとして depth を返す。 5. 存在しない場合、依存先オブジェクトに対して再帰的に findLoop を呼び出す。 6. どの枝からも startPoint に到達できない場合、searchFailed に startPoint を記録し 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump_sort.c 内の循環依存解決ロジックから呼び出される。
  • **呼び出し先**: findObjectByDumpId（依存関係の解決）、および自身（再帰呼び出し）。

関連データ構造

  • DumpableObject: ダンプ対象のデータベースオブジェクト構造体。
  • DumpId: オブジェクトを一意に識別するID。

失敗モードと境界条件

  • 依存先が NULL（ダンプ対象外）の場合は探索をスキップする。
  • 循環参照が存在しない場合、探索は全ノードを走査した後に 0 を返す。
  • workspace のサイズ制限により、過度に深い依存関係がある場合はスタックオーバーフローのリスクがある（ただし、workspace への追加チェックにより無限ループは防止されている）。

保守担当者向けメモ

  • 本関数は短いサイクルを優先的に検出する特性がある。
  • searchFailed 配列の活用により、探索効率が最適化されている。修正時はこのキャッシュロジックを破壊しないよう注意すること。
