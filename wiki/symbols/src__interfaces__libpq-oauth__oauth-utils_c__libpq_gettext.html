libpq_gettext

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq-oauth/oauth-utils.c:124

目的

NLS（国際化）対応のメッセージ翻訳機能を提供します。libpq の翻訳インフラを libpq-oauth モジュールから利用可能にするためのラッパーです。

シグネチャまたは形状

char *libpq_gettext(const char *msgid);

入力と出力

  • **入力**: msgid (翻訳対象の文字列定数)
  • **出力**: 翻訳された文字列へのポインタ。NLS が無効な場合は、入力された msgid がそのまま返されます。

前提条件と不変条件

  • 呼び出し元は、戻り値を const char * として扱う必要があります（NLS 無効時に引数がそのまま返されるため）。
  • libpq_gettext_impl 関数ポインタが初期化されている場合、その実装が優先されます。

アルゴリズムまたは振る舞い

1. libpq_gettext_impl が設定されているか確認します。 2. 設定されている場合、その関数ポインタを呼び出して翻訳結果を返します。 3. 設定されていない場合（libpq が NLS 無効でビルドされている場合など）、unconstify マクロを介して msgid をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq-oauth/oauth-curl.c 内の各所（エラーメッセージの生成やユーザーへの通知出力時）。
  • **呼び出し先**: libpq_gettext_impl（初期化時に注入された関数ポインタ）。

関連データ構造

  • libpq_gettext_func: libpq_gettext_impl を保持するために使用される関数ポインタ型。

失敗モードと境界条件

  • libpq_gettext_impl が未設定の場合でも、エラーにはならず引数をそのまま返すことで安全に動作します。

保守担当者向けメモ

  • 本関数は libpq-int.h への依存を避けるための抽象化レイヤーの一部です。
  • libpq 側の NLS 設定と libpq-oauth 側の NLS 設定が不一致の場合でも、翻訳なしの文字列を返すことで整合性を保つ設計になっています。
