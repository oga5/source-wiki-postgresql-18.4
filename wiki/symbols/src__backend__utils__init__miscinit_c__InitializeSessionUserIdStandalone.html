InitializeSessionUserIdStandalone

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/miscinit.c:890

目的

シングルユーザーモード、オートバキュームワーカー、スロット同期ワーカー、およびバックグラウンドワーカーにおいて、セッションユーザーIDをブートストラップスーパーユーザーとして初期化します。

シグネチャまたは形状

void InitializeSessionUserIdStandalone(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 AuthenticatedUserId およびセッション権限設定を更新）

前提条件と不変条件

  • 呼び出し元は、シングルユーザーモードであるか、オートバキューム、スロット同期、またはバックグラウンドワーカーのいずれかのプロセスである必要があります。
  • AuthenticatedUserId が未設定（OidIsValid が偽）である状態で、一度だけ呼び出される必要があります。

アルゴリズムまたは振る舞い

1. プロセス種別の妥当性を Assert で検証します。 2. AuthenticatedUserId が初期化済みでないことを Assert で確認します。 3. AuthenticatedUserId に BOOTSTRAP_SUPERUSERID を代入します。 4. SetSessionAuthorization を呼び出し、セッション権限をブートストラップスーパーユーザーに設定します。 5. SetCurrentRoleId を呼び出し、現在のロールIDを InvalidOid に設定して初期化を完了します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/init/postinit.c 内の初期化処理から複数箇所で呼び出されています。
  • 呼び出し先:
  • SetSessionAuthorization
  • SetCurrentRoleId

関連データ構造

  • AuthenticatedUserId (グローバル変数)
  • BOOTSTRAP_SUPERUSERID (定数)

失敗モードと境界条件

  • 本関数は pg_authid テーブルの行が破損している場合でも起動を継続できるように設計されています。
  • GUC（session_authorization）を直接設定しないため、このセッション内では current_setting() は空文字列を返します。

保守担当者向けメモ

  • 本関数は SetSessionAuthorization を使用しますが、GUCの更新を伴わない特殊なケースです。将来的にGUCとの整合性を取る際は、この設計意図（pg_authid への依存回避）を考慮してください。
