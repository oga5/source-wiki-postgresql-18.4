dump_sqlda

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/compat_informix-sqlda.c:122

目的

Informix 互換モードの ECPG テストにおいて、SQLDA（SQL Descriptor Area）構造体の中身を標準出力へダンプし、デバッグやテスト結果の検証を支援する。

シグネチャまたは形状

static void dump_sqlda(sqlda_t *sqlda)

入力と出力

  • **入力**: sqlda_t 構造体へのポインタ。
  • **出力**: なし（標準出力への printf による情報表示のみ）。

前提条件と不変条件

  • sqlda が NULL の場合、エラーメッセージを出力して即座に終了する。
  • sqlda->sqld には記述子内の有効な列数が格納されている必要がある。
  • 各 sqlvar[i] の sqltype に応じて、sqldata ポインタが適切な型（int*, double*, decimal* など）としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. sqlda が NULL かどうかを判定し、該当する場合は警告を出力してリターンする。 2. sqlda->sqld の回数分ループし、各列の情報を順次処理する。 3. sqlind（インジケータ）が -1 の場合、値が NULL である旨を表示する。 4. それ以外の場合、sqltype に基づき以下の型として値を表示する:

  • SQLCHAR: 文字列として表示。
  • SQLINT: 整数として表示。
  • SQLFLOAT: 浮動小数点数として表示。
  • SQLDECIMAL: dectoasc 関数を用いて文字列に変換し、DECIMAL 値として表示。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内のテスト実行フロー（main 関数内）から複数回呼び出される。また、src/interfaces/ecpg/test/expected/sql-sqlda.c からも呼び出されている。
  • **呼び出し先**: 標準ライブラリの printf、および Informix 互換ライブラリの dectoasc。

関連データ構造

  • sqlda_t: 記述子全体を保持する構造体。
  • sqlvar_t: 各列の属性（型、名前、データポインタ、インジケータ）を保持する構造体。
  • decimal: SQLDECIMAL 型の処理に使用される構造体。

失敗モードと境界条件

  • sqlda が NULL の場合、安全に処理を中断する。
  • sqltype が上記で定義されたケース（SQLCHAR, SQLINT, SQLFLOAT, SQLDECIMAL）以外の場合、その列の出力はスキップされる。

保守担当者向けメモ

  • 本関数はテスト用コードであり、実運用環境のライブラリには含まれない。
  • sqltype の追加や変更があった場合は、switch 文に新しいケースを追加する必要がある。
  • dectoasc の呼び出し時に decimal 型へのキャストを行っているため、sqldata のメモリレイアウトが decimal 構造体と一致していることが前提となる。
