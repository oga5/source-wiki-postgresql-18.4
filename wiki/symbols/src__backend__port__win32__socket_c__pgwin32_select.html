pgwin32_select

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/socket.c:516

目的

Windows環境において、POSIX準拠の select() 関数をWinsock APIを用いてエミュレートし、ソケットのI/O待機とPostgreSQLのシグナル処理を統合する。

シグネチャまたは形状

int pgwin32_select(int nfds, fd_set *readfds, fd_set *writefds, fd_set *exceptfds, const struct timeval *timeout);

入力と出力

  • **入力**:
  • nfds: POSIX互換のための引数（無視される）。
  • readfds, writefds: 監視対象のソケット集合。
  • exceptfds: サポート対象外（NULLである必要がある）。
  • timeout: 待機時間。
  • **出力**:
  • 成功時: 準備完了したソケットの総数。
  • タイムアウト時: 0。
  • 失敗時: -1（errnoが設定される）。

前提条件と不変条件

  • exceptfds は NULL でなければならない（Assertにより検証）。
  • Windowsのソケットは内部的に非ブロッキングモードとして扱われる。
  • 呼び出しの冒頭で pgwin32_poll_signals() が実行され、保留中のシグナルが処理される。

アルゴリズムまたは振る舞い

1. **シグナル確認**: pgwin32_poll_signals() を呼び出し、シグナルがあれば即座に -1 を返す。 2. **書き込み準備の強制確認**: writefds が指定されている場合、WSASend を用いたダミー送信を行い、即座に書き込み可能なソケットがあれば結果を返して終了する。 3. **イベント設定**: WSACreateEvent で各ソケットにイベントを割り当て、WSAEventSelect で監視対象のイベント（FD_READ, FD_WRITE 等）を登録する。 4. **待機**: WaitForMultipleObjectsEx を使用し、ソケットイベントまたはシグナルイベント（pgwin32_signal_event）のいずれかが発生するまで待機する。 5. **結果収集**: WSAEnumNetworkEvents を用いて、準備完了したソケットを特定し、readfds/writefds を更新する。 6. **クリーンアップ**: WSAEventSelect(..., NULL, 0) でイベント関連付けを解除し、WSACloseEvent でリソースを解放する。 7. **シグナル処理**: シグナルイベントで復帰した場合は pgwin32_dispatch_queued_signals() を呼び出し、errno を EINTR に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/win32_port.h にて select マクロとして定義されており、PostgreSQLの各所で select() として呼び出される。
  • **呼び出し先**: pgwin32_poll_signals, WSASend, WSACreateEvent, WSAEventSelect, WaitForMultipleObjectsEx, WSAEnumNetworkEvents, WSACloseEvent, pgwin32_dispatch_queued_signals 等。

関連データ構造

  • fd_set: ソケットの集合を保持する構造体。
  • WSAEVENT: Windowsのイベントオブジェクト。

失敗モードと境界条件

  • exceptfds が非NULLの場合、アサーション違反となる。
  • WSAEventSelect が失敗した場合、エラーを変換し、割り当て済みのイベントを解放して -1 を返す。
  • タイムアウト発生時は readfds および writefds をクリアし 0 を返す。

保守担当者向けメモ

  • exceptfds は現在サポートされていない。
  • Windowsの FD_WRITE イベントの仕様上、書き込み可能判定にはダミー送信による確認が必要である。
  • WaitForMultipleObjectsEx の待機対象には、ソケットイベントに加えてシグナルイベントが含まれている。
