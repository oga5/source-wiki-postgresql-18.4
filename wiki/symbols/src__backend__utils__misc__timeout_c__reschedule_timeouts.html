reschedule_timeouts

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/timeout.c:539

目的

現在アクティブなタイムアウトイベントの状態に基づき、システムタイマー（setitimer）を再スケジュールする。

シグネチャまたは形状

void reschedule_timeouts(void)

入力と出力

  • 入力: なし（内部状態 all_timeouts_initialized および num_active_timeouts を参照）
  • 出力: なし（副作用としてタイマー設定を更新）

前提条件と不変条件

  • all_timeouts_initialized が偽の場合、何も行わずに即座に復帰する。
  • 実行中、disable_alarm() を呼び出してシグナル割り込みを一時的に抑制する。

アルゴリズムまたは振る舞い

1. 初期化状態を確認し、未初期化であれば終了する。 2. disable_alarm() を呼び出し、シグナル割り込みを無効化する。 3. num_active_timeouts が 0 より大きい場合、GetCurrentTimestamp() を現在時刻として取得し、schedule_alarm() を呼び出してタイマーを再設定する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/xact.c (トランザクション処理内)
  • src/backend/tcop/postgres.c (トップレベルのコマンド処理)
  • 呼び出し先:
  • disable_alarm()
  • schedule_alarm()
  • GetCurrentTimestamp()

関連データ構造

  • num_active_timeouts: 現在有効なタイムアウトの数。
  • all_timeouts_initialized: タイムアウト管理システムが初期化済みかを示すフラグ。

失敗モードと境界条件

  • タイムアウトが一つもアクティブでない場合（num_active_timeouts == 0）、タイマーの再スケジュールはスキップされる。
  • 初期化前に呼び出された場合、安全に無視される。

保守担当者向けメモ

  • 本関数はシグナルハンドラとメイン処理間の競合を避けるため、disable_alarm() による保護下で実行される。
  • schedule_alarm() は setitimer を呼び出すため、呼び出し頻度が高い場合はパフォーマンスへの影響を考慮すること。
