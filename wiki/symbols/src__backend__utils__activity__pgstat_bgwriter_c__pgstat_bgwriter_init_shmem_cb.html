pgstat_bgwriter_init_shmem_cb

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/activity/pgstat_bgwriter.c:79

目的

バックグラウンドライター（bgwriter）の統計情報を保持する共有メモリ領域の初期化を行います。

シグネチャまたは形状

void pgstat_bgwriter_init_shmem_cb(void *stats)

入力と出力

  • **入力**: void *stats - 初期化対象となる共有メモリ領域へのポインタ。内部で PgStatShared_BgWriter 型にキャストされます。
  • **出力**: なし（副作用として共有メモリ内のロックが初期化されます）。

前提条件と不変条件

  • 引数 stats は、PgStatShared_BgWriter 構造体として適切に割り当てられた共有メモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. 受け取った stats ポインタを PgStatShared_BgWriter * 型にキャストします。 2. 構造体内の lock メンバに対して LWLockInitialize を実行し、LWTRANCHE_PGSTATS_DATA トランシェを使用してロックを初期化します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c:411 において、統計システムの初期化プロセスからコールバックとして参照されています。
  • **呼び出し先**: LWLockInitialize（PostgreSQLのロック管理関数）。

関連データ構造

  • PgStatShared_BgWriter: バックグラウンドライターの統計情報を共有メモリ上で管理するための構造体。本関数はこの構造体内のロックを初期化します。

失敗モードと境界条件

  • stats が NULL の場合、あるいは有効な共有メモリ領域を指していない場合、メモリ保護違反や不正なメモリ操作が発生する可能性があります。

保守担当者向けメモ

  • 本関数は統計サブシステムの初期化フェーズで呼び出されるコールバックです。ロックのトランシェ（LWTRANCHE_PGSTATS_DATA）を変更する際は、統計システム全体のロック階層設計に影響がないか確認してください。
