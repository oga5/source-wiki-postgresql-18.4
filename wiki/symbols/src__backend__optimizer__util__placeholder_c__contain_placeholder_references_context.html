contain_placeholder_references_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/util/placeholder.c:27

目的

式ツリー内を走査する際、特定の PlaceHolderVar への参照が含まれているかを判定するための状態保持用コンテキスト構造体です。

シグネチャまたは形状

typedef struct contain_placeholder_references_context
{
	int			relid;          /* 検索対象のリレーションID */
	int			sublevels_up;   /* 現在のクエリレベルからの相対深さ */
} contain_placeholder_references_context;

入力と出力

  • 入力: 走査対象の式ツリーおよび本構造体のインスタンス。
  • 出力: 走査の結果として、指定されたリレーションIDおよびレベルに合致する PlaceHolderVar が存在するかどうかの真偽値（主に contain_placeholder_references_to 関数経由で返される）。

前提条件と不変条件

  • relid は有効なリレーション識別子である必要があります。
  • sublevels_up は、現在のクエリレベルを基準とした非負の整数である必要があります。

アルゴリズムまたは振る舞い

式ツリーを再帰的に探索する contain_placeholder_references_walker 関数において、現在のノードが PlaceHolderVar である場合に、その phrels（依存リレーション集合）が本構造体の relid を含んでいるか、かつ sublevels_up が一致するかを照合するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: contain_placeholder_references_to 関数が本構造体を初期化し、expression_tree_walker を介して走査を開始します。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出しません。

関連データ構造

  • PlaceHolderVar: 走査対象となるノード。
  • contain_placeholder_references_walker: 本構造体を利用してツリーを走査する関数。

失敗モードと境界条件

  • sublevels_up が不適切な値（クエリの深さを超える値など）である場合、誤った判定結果を返す可能性があります。
  • 走査対象の式が NULL の場合、あるいは PlaceHolderVar を含まない式ツリーである場合は、常に「参照なし」と判定されます。

保守担当者向けメモ

  • 本構造体は contain_placeholder_references_to 関数専用のコンテキストとして設計されています。
  • 拡張を行う際は、contain_placeholder_references_walker 内での比較ロジックと整合性を保つ必要があります。
