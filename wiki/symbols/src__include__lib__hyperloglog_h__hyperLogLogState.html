hyperLogLogState

定義

  • 種別: struct
  • 定義箇所: src/include/lib/hyperloglog.h:53

目的

HyperLogLog (HLL) アルゴリズムを用いて、大規模なデータセットのカーディナリティ（ユニークな要素数）を固定メモリ量で近似計算するための状態管理構造体です。

シグネチャまたは形状

typedef struct hyperLogLogState
{
    uint8       registerWidth;  /* レジスタのビット幅 */
    Size        nRegisters;     /* レジスタ数 */
    double      alphaMM;        /* バイアス補正用の計算済み係数 */
    uint8      *hashesArr;      /* レジスタ配列へのポインタ */
    Size        arrSize;        /* 配列のサイズ */
} hyperLogLogState;

入力と出力

本構造体は、addHyperLogLog 関数への入力として状態を保持し、estimateHyperLogLog 関数によってカーディナリティの推定値（double）を出力するための基盤となります。

前提条件と不変条件

  • registerWidth（ビット幅）は 4 から 16 の範囲で指定する必要があります。
  • メモリ使用量は 2^registerWidth に比例します。
  • hashesArr は palloc0 等で適切に確保されている必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: initHyperLogLog により、指定されたビット幅に基づいてレジスタ配列が割り当てられ、alphaMM が算出されます。 2. **更新**: addHyperLogLog がハッシュ値を受け取り、ハッシュ値の先頭ビットでレジスタを選択し、残りのビットから先行ゼロの数をカウントしてレジスタを更新します。 3. **推定**: estimateHyperLogLog が全レジスタの調和平均を計算し、alphaMM を用いてバイアスを補正した結果を返します。

関連データ構造

  • port/pg_bitutils.h: 先行ゼロカウント（pg_count_leading_zeros）等のビット操作機能を提供し、アルゴリズムの計算を支えています。

失敗モードと境界条件

  • メモリ不足: palloc0 等によるメモリ確保失敗時にエラーが発生する可能性があります。
  • 精度限界: 指定した registerWidth が小さい場合、推定誤差が大きくなります。
