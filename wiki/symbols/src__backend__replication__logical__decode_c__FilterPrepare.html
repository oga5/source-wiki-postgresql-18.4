FilterPrepare

定義

  • 種別: static inline function
  • 定義箇所: src/backend/replication/logical/decode.c:550

目的

論理デコーディングにおいて、特定の2相コミット（2PC）トランザクションをフィルタリング（除外）すべきかどうかを判定する。

シグネチャまたは形状

static inline bool FilterPrepare(LogicalDecodingContext *ctx, TransactionId xid, const char *gid)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • xid: 対象トランザクションのID。
  • gid: 2相コミットのグローバル識別子。
  • **出力**:
  • true: トランザクションをフィルタリング（スキップ）する場合。
  • false: トランザクションを処理対象とする場合。

前提条件と不変条件

  • ctx->twophase が無効な場合、すべての2相トランザクションはフィルタリング対象（true）とみなされる。
  • ctx->callbacks.filter_prepare_cb が設定されていない場合、フィルタリングは行われず、すべての準備済みトランザクションが処理対象となる。

アルゴリズムまたは振る舞い

1. ctx->twophase を確認し、無効であれば true を返す。 2. ctx->callbacks.filter_prepare_cb が NULL であれば、フィルタリングを行わず false を返す。 3. 上記以外の場合、filter_prepare_cb_wrapper を呼び出し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/decode.c 内の複数箇所（行239, 267, 326付近）。
  • **呼び出し先**:
  • filter_prepare_cb_wrapper（定義箇所はソース抜粋外のため詳細不明）。

関連データ構造

  • LogicalDecodingContext: 論理デコーディングの実行状態およびコールバック関数を保持する構造体。

失敗モードと境界条件

  • ctx が NULL の場合の挙動については、本ソース抜粋からは判断できない。
  • filter_prepare_cb_wrapper が内部でエラーを発生させた場合の動作は、本ソース抜粋からは判断できない。

保守担当者向けメモ

  • 本関数は static inline で定義されており、デコーディングの各フェーズで頻繁に呼び出されるため、パフォーマンスへの影響を考慮すること。
  • 2相コミットのサポートが有効でない場合、強制的にスキップされる仕様である点に注意。
