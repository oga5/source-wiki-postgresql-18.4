RemoveTypeById

定義

  • 種別: function
  • 定義箇所: src/backend/commands/typecmds.c:656

目的

指定されたOIDに対応するデータ型をシステムカタログから削除します。

シグネチャまたは形状

void RemoveTypeById(Oid typeOid)

入力と出力

  • **入力**: typeOid (削除対象のデータ型のOID)
  • **出力**: なし（成功時はカタログから削除、失敗時はエラーをスロー）

前提条件と不変条件

  • pg_type システムカタログに対する RowExclusiveLock を取得して操作を行います。
  • 指定された typeOid が pg_type に存在する必要があります。

アルゴリズムまたは振る舞い

1. pg_type リレーションをオープンし、行排他ロックを取得します。 2. TYPEOID システムキャッシュを使用して、対象の型タプルを検索します。見つからない場合はエラーを発生させます。 3. CatalogTupleDelete を呼び出し、pg_type から該当タプルを削除します。 4. 型の種類（typtype）を確認し、以下の特殊な型の場合は関連するカタログエントリも削除します。

  • TYPTYPE_ENUM: EnumValuesDelete を呼び出し、pg_enum のエントリを削除します。
  • TYPTYPE_RANGE: RangeDelete を呼び出し、pg_range のエントリを削除します。

    • 5. システムキャッシュを解放し、リレーションをクローズします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/dependency.c:1392 からの呼び出しが確認されています。
  • **呼び出し先**: table_open, SearchSysCache1, CatalogTupleDelete, EnumValuesDelete, RangeDelete, ReleaseSysCache, table_close を呼び出します。

関連データ構造

  • pg_type: 型定義を保持するシステムカタログ。
  • pg_enum: 列挙型の値を保持するシステムカタログ。
  • pg_range: 範囲型の定義を保持するシステムカタログ。

失敗モードと境界条件

  • 指定した typeOid が存在しない場合、elog(ERROR) によりトランザクションが中断されます。
  • 削除対象の型が他のオブジェクトから依存されている場合、カタログレベルの制約により削除が失敗する可能性があります。

保守担当者向けメモ

  • 本関数は依存関係の自動管理を行わないため、EnumやRangeのような特殊な型については、明示的に関連カタログを削除するロジックが含まれています。将来的に新しい型カテゴリが追加された場合は、ここでの削除処理の拡張が必要になる可能性があります。
