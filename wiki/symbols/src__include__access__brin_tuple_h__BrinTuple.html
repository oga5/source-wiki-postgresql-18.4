BrinTuple

定義

  • 種別: struct
  • 定義箇所: src/include/access/brin_tuple.h:63

目的

BRIN (Block Range Index) において、ページ範囲の要約情報をディスク上に格納するための物理的なタプル構造を定義します。

シグネチャまたは形状

typedef struct BrinTuple
{
    BlockNumber bt_blkno;
    uint8       bt_info;
} BrinTuple;

入力と出力

  • **入力**: 構造体内の bt_blkno（ヒープブロック番号）および bt_info（メタデータフラグ）。
  • **出力**: 可変長データ領域の先頭位置を示すオフセット情報（bt_info の下位 5 ビット）。

前提条件と不変条件

  • bt_info のビット構成は以下の通り固定されています。
  • 第 7 ビット (高位): has nulls (範囲内に NULL が存在)
  • 第 6 ビット: is placeholder tuple (プレースホルダー)
  • 第 5 ビット: range is empty (範囲が空)
  • 第 4-0 ビット: データ領域のオフセット

アルゴリズムまたは振る舞い

BRIN インデックスのディスク表現として機能します。インデックス更新時には brin_form_tuple によって BrinMemTuple から生成され、検索時には brin_deform_tuple によってメモリ上の構造へ展開されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin.c 内の各関数（brtup、lastPageTuple、newtup 等として参照）、および contrib/pageinspect/brinfuncs.c。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • BrinMemTuple: インデックス処理中に使用されるメモリ上の対応構造体。
  • BrinValues: BrinMemTuple 内で各カラムの要約値を保持する構造体。

失敗モードと境界条件

  • NULL ビットマップ（hasnulls および allnulls）の管理において、bt_info のビットフラグとデータ領域の整合性が損なわれると、インデックスの読み取りエラーが発生します。
  • TOAST されたデータを含む場合、デトースト処理が適切に行われないとデータ破損の可能性があります。

保守担当者向けメモ

  • 本構造体はディスク上の物理レイアウトを規定するため、フィールドの順序やビット定義の変更はディスクフォーマットの互換性に影響します。
  • メモリ上の操作には必ず BrinMemTuple を使用し、ディスクとの変換は src/backend/access/brin/brin_tuple.c のインターフェースを介してください。
