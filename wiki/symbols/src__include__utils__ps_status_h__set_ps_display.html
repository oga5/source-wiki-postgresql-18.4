set_ps_display

定義

  • 種別: function
  • 定義箇所: src/include/utils/ps_status.h:39

目的

バックエンドプロセスの現在の活動内容を OS 上のプロセス名（ps コマンド等で確認可能）に反映させるためのインターフェースを提供します。

シグネチャまたは形状

static inline void set_ps_display(const char *activity)

入力と出力

  • **入力**: activity (現在の活動内容を示す文字列へのポインタ)
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • プロセス起動時に save_ps_display_args() が呼び出され、OS の引数領域が保護・管理されている必要があります。
  • update_process_title GUC 変数が有効である場合にのみ、OS 側のプロセス名が更新されます。

アルゴリズムまたは振る舞い

1. 引数として渡された activity 文字列の長さを strlen() で算出します。 2. 内部関数 set_ps_display_with_len(activity, strlen(activity)) を呼び出します。 3. set_ps_display_with_len は、プラットフォーム固有の実装（setproctitle の利用や argv 領域の直接書き換え）に従い、OS にプロセス名を通知します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/backup/basebackup.c
  • src/backend/commands/async.c
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/pgarch.c
  • **呼び出し先**:
  • set_ps_display_with_len (直接の呼び出し先)

関連データ構造

  • update_process_title (プロセス名更新の有効/無効を制御する GUC 変数)

失敗モードと境界条件

  • **プラットフォーム依存**: OS によってプロセス名更新のメカニズムが異なるため、実装は ps_status.c 内の #ifdef 分岐に依存します。
  • **パフォーマンス**: Windows 環境では更新処理のオーバーヘッドが大きいため、デフォルトで無効化されています。
  • **副作用**: Linux 等の環境では、本関数の呼び出しが /proc/$pid/environ の内容を破壊する可能性があることが指摘されています。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されることを想定し、インライン関数として定義されています。
  • 文字列長をコンパイル時に評価できるよう工夫されていますが、動的な文字列を渡す場合は strlen のコストが発生します。
  • 移植性を考慮する場合、src/backend/utils/misc/ps_status.c のプラットフォーム固有の実装詳細を確認してください。
