heap_free_minimal_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/heaptuple.c:1529

目的

MinimalTuple 形式のタプルが占有しているメモリ領域を解放します。

シグネチャまたは形状

void heap_free_minimal_tuple(MinimalTuple mtup)

入力と出力

  • **入力**: mtup (MinimalTuple) - 解放対象となるミニマルタプルへのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • mtup は palloc 等によって割り当てられた有効なメモリ領域を指している必要があります。
  • 本関数呼び出し後、mtup が指すメモリ領域へのアクセスは無効となります。

アルゴリズムまたは振る舞い

内部的に pfree(mtup) を呼び出し、該当するメモリ領域をメモリコンテキストから解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execTuples.c
  • src/backend/executor/nodeHash.c
  • src/backend/executor/nodeHashjoin.c
  • src/backend/utils/sort/tuplestore.c
  • **呼び出し先**:
  • pfree (PostgreSQLのメモリ管理関数)

関連データ構造

  • MinimalTuple: タプルの物理的レイアウトを簡略化した構造体。src/include/access/htup_details.h に定義されています。

失敗モードと境界条件

  • mtup が NULL の場合、pfree の仕様に依存します（通常、PostgreSQLの pfree は NULL ポインタを渡すと何もしません）。

保守担当者向けメモ

  • 本関数は単なる pfree のラッパーです。将来的に MinimalTuple の構造が複雑化し、内部で追加のメモリ解放が必要になった場合、この関数内で処理を拡張してください。
