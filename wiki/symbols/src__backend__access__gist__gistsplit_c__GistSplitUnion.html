GistSplitUnion

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gist/gistsplit.c:31

目的

GiSTインデックスのページ分割処理において、分割後の各ページ（左・右）のユニオンキーを再計算するための作業用構造体です。

シグネチャまたは形状

typedef struct {
    OffsetNumber *entries;
    int          len;
    Datum        *attr;
    bool         *isnull;
    bool         *dontcare;
} GistSplitUnion;

入力と出力

本構造体はデータ保持用であり、関数ではありません。gistSplitByKey 等の処理において、分割対象のタプル情報（オフセット、属性値、NULLフラグ、移動コストゼロの判定フラグ）を保持するために使用されます。

前提条件と不変条件

  • len は entries 配列の要素数と一致している必要があります。
  • attr、isnull、dontcare は、それぞれ len 分の要素を持つ配列を指している必要があります。

アルゴリズムまたは振る舞い

分割処理中に以下の目的で使用されます。 1. **タプル管理**: 分割対象となるタプルのオフセットと属性値を保持します。 2. **Don't-care 判定**: dontcare フラグにより、特定の列においてペナルティなしで移動可能なタプルを識別します。 3. **ユニオンキー再計算**: 保持されたデータに基づき、分割後のページにおけるインデックスの整合性を維持するためのキー計算を支援します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistsplit.c 内の gistSplitByKey 等の関数が、この構造体をインスタンス化して使用しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • OffsetNumber: タプルの位置を指す識別子。
  • Datum: インデックス化される属性値。

失敗モードと境界条件

  • メモリ割り当てが不十分な場合、構造体内の各配列へのアクセスで境界外参照が発生する可能性があります。
  • len が 0 の場合、ユニオンキーの計算は無効な状態となります。

保守担当者向けメモ

  • 本構造体は gistSplitByKey におけるマルチカラム分割の最適化プロセスにおいて重要な役割を果たします。
  • 構造体内の各ポインタが指すメモリ領域のライフサイクルは、呼び出し元の関数（主に gistSplitByKey）によって管理されています。
