WFW_WaitOption

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_dump/parallel.h:30

目的

pg_dump および pg_restore の並列処理において、リーダープロセスがワーカープロセスの状態を監視する際の待機条件を指定するために使用されます。

シグネチャまたは形状

typedef enum
{
	WFW_NO_WAIT,
	WFW_GOT_STATUS,
	WFW_ONE_IDLE,
	WFW_ALL_IDLE,
} WFW_WaitOption;

入力と出力

本シンボルは WaitForWorkers 関数の引数 mode として入力されます。出力値はありません。

前提条件と不変条件

  • WaitForWorkers 関数に渡される列挙値として、並列処理の同期制御を決定します。
  • 各列挙値は、ワーカーの終了待ちやアイドル状態の確認といった特定の待機動作に対応しています。

アルゴリズムまたは振る舞い

  • WFW_NO_WAIT: 待機せずに即座に制御を戻します。
  • WFW_GOT_STATUS: ステータスが取得できるまで待機します。
  • WFW_ONE_IDLE: いずれか1つのワーカーがアイドル状態になるまで待機します。
  • WFW_ALL_IDLE: 全てのワーカーがアイドル状態になるまで待機します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/parallel.c 内の WaitForWorkers 関数がこの型を引数として受け取ります。
  • 呼び出し先: 情報なし。

関連データ構造

  • ParallelState: 並列処理の管理構造体であり、WaitForWorkers を通じて本オプションによる制御を受けます。

失敗モードと境界条件

  • 該当なし。

保守担当者向けメモ

  • 本列挙型は parallel.c の WaitForWorkers 関数におけるロジック分岐の根幹を成します。動作変更時は WaitForWorkers 内の待機ロジックとの整合性に注意してください。
