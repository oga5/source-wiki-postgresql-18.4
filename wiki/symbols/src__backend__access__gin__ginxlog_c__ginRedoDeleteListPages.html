ginRedoDeleteListPages

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginxlog.c:674

目的

GINインデックスのリストページ削除に関するWALレコードをリプレイし、メタページの更新および削除対象ページの初期化を行う。

シグネチャまたは形状

static void ginRedoDeleteListPages(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）。
  • **出力**: なし（副作用としてバッファの状態を更新）。

前提条件と不変条件

  • record から取得したメタページブロック番号が GIN_METAPAGE_BLKNO であること。
  • 削除対象のページは、メタページロックによって保護されるため、リプレイ中に他のリーダーが削除中のページにアクセスすることはない。

アルゴリズムまたは振る舞い

1. **メタページの更新**:

  • XLogInitBufferForRedo でメタページを取得し、GinInitMetabuffer で初期化する。
  • WALレコード内の metadata をメタページにコピーし、LSNを更新してダーティにする。

    • 2. **削除対象ページの初期化**:

  • WALレコードに含まれる削除対象ページ数（data->ndeleted）分だけループする。
  • 各ページを XLogInitBufferForRedo で取得し、GinInitBuffer を用いて GIN_DELETED 状態として初期化する。
  • 各ページのLSNを更新し、ダーティにしてからバッファを解放する。

    • 3. **終了処理**:

  • 最後にメタページのバッファを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginxlog.c 内の gin_redo 関数から呼び出される（WALレコードの種別に基づくディスパッチ）。
  • **呼び出し先**: XLogInitBufferForRedo, BufferGetBlockNumber, BufferGetPage, GinInitMetabuffer, GinPageGetMeta, PageSetLSN, MarkBufferDirty, GinInitBuffer, UnlockReleaseBuffer。

関連データ構造

  • ginxlogDeleteListPages: WALレコードのデータ構造。
  • GinMetaPageData: GINインデックスのメタページ情報。

失敗モードと境界条件

  • 削除対象ページが0の場合、ループは実行されずメタページの更新のみが行われる。
  • XLogInitBufferForRedo が失敗した場合はリプレイ処理が中断される（PostgreSQLの標準的なWALリプレイ動作に準拠）。

保守担当者向けメモ

  • 本関数は、通常の運用時における shiftList() とは異なり、削除対象ページを一つずつロックして処理する。これはWALリプレイの特性上、安全であると判断されている。
  • 削除されたページは空の GIN_DELETED 状態として再初期化され、ページ内の右リンク（right-link）は保持されない。
