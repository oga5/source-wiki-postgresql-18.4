px_crypt_algo

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/px-crypt.c:80

目的

pgcrypto モジュールにおいて、サポートされる各パスワードハッシュアルゴリズムを統一的に扱うためのメタデータ構造です。

シグネチャまたは形状

struct px_crypt_algo
{
    char       *id;         /* アルゴリズム識別子（ソルトのプレフィックス） */
    unsigned    id_len;     /* id の長さ */
    char       *(*crypt) (const char *psw, const char *salt,
                          char *buf, unsigned len); /* ハッシュ計算関数へのポインタ */
};

入力と出力

  • 本構造体自体は入力や出力を行いません。
  • メンバの crypt 関数ポインタは、パスワード (psw) とソルト (salt) を入力として受け取り、ハッシュ化された文字列をバッファ (buf) に書き込みます。

前提条件と不変条件

  • id はソルト文字列の先頭部分と一致する必要があります。
  • crypt 関数ポインタは、pgcrypto がサポートする各ハッシュアルゴリズムの実装（crypt-*.c）を指す必要があります。

アルゴリズムまたは振る舞い

  • px_crypt_list テーブル内で、特定のハッシュアルゴリズムを識別し、対応する計算関数をディスパッチするためのインターフェースとして機能します。
  • px_crypt() 関数は、この構造体のリストを走査して id を照合し、一致したアルゴリズムの crypt 関数を呼び出します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/px-crypt.c 内の px_crypt() 関数が、この構造体の配列（px_crypt_list）を走査して利用します。
  • 呼び出し先: crypt メンバは、run_crypt_des, run_crypt_md5, run_crypt_bf, run_crypt_sha などの関数を指します。

関連データ構造

  • px_crypt_list: px_crypt_algo 型の静的配列。アルゴリズムの定義リストを保持します。

失敗モードと境界条件

  • id が一致しない場合、px_crypt は適切なアルゴリズムを見つけられず処理を継続できません。
  • crypt 関数ポインタが NULL の場合、実行時にクラッシュする可能性があります。

保守担当者向けメモ

  • px_crypt_list 内の順序は重要です。プレフィックスが短いもの（空文字など）が後に配置されるよう注意してください。
  • 新しいハッシュアルゴリズムを追加する場合は、この構造体を用いて px_crypt_list にエントリを追加し、対応する run_crypt_* 関数を実装してください。
