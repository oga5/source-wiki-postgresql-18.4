window_percent_rank_support

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/windowfuncs.c:287

目的

PERCENT_RANK ウィンドウ関数のクエリ最適化をサポートします。オプティマイザに対し、関数の単調性情報や、実行時の計算コストを削減するためのフレーム設定を伝達します。

シグネチャまたは形状

Datum window_percent_rank_support(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して Node *rawreq を受け取ります。
  • **出力**: SupportRequestWFuncMonotonic または SupportRequestOptimizeWindowClause の要求に応じて、更新された要求構造体へのポインタを返します。要求に該当しない場合は NULL を返します。

前提条件と不変条件

  • rawreq は SupportRequestWFuncMonotonic または SupportRequestOptimizeWindowClause のいずれかである必要があります。
  • PERCENT_RANK は単調増加関数であるという不変条件に基づき動作します。

アルゴリズムまたは振る舞い

1. **単調性の通知**: SupportRequestWFuncMonotonic が要求された場合、req->monotonic を MONOTONICFUNC_INCREASING に設定します。 2. **フレーム最適化**: SupportRequestOptimizeWindowClause が要求された場合、フレームオプションを ROWS モード（FRAMEOPTION_ROWS）に設定し、RANGE モードのデフォルト動作を上書きします。これにより、実行エンジンが「ピア行（同順位の行）」をチェックするコストを回避します。

  • 設定値: FRAMEOPTION_NONDEFAULT | FRAMEOPTION_ROWS | FRAMEOPTION_START_UNBOUNDED_PRECEDING | FRAMEOPTION_END_CURRENT_ROW

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のクエリプランナ（オプティマイザ）から、ウィンドウ関数の最適化フェーズで呼び出されます。
  • **呼び出し先**: 提供されたソースコード内での直接的な関数呼び出しはありません。

関連データ構造

  • SupportRequestWFuncMonotonic: ウィンドウ関数の単調性を問い合わせるための構造体。
  • SupportRequestOptimizeWindowClause: ウィンドウ句のフレーム最適化を要求するための構造体。

失敗モードと境界条件

  • 未知の rawreq 型が渡された場合、NULL を返して処理を終了します。

保守担当者向けメモ

  • 本関数は row_number のサポート関数と同様のフレーム最適化戦略を採用しています。将来的に PERCENT_RANK の計算ロジックに変更が生じた場合、このフレーム設定が依然として妥当であるか確認してください。
