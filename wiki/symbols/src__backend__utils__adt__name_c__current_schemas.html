current_schemas

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/name.c:293

目的

現在のセッションにおける検索パス（search path）に含まれるスキーマ名のリストを、name 型の配列として取得します。

シグネチャまたは形状

Datum current_schemas(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_BOOL(0) を介して、システムスキーマを含めるかどうかを示すブール値を受け取ります。
  • **出力**: 検索パス上のスキーマ名を格納した ArrayType（name 型の配列）を Datum として返します。

前提条件と不変条件

  • fetch_search_path を呼び出して検索パスを取得します。
  • 取得した検索パスのリストは、処理終了後に list_free で解放されます。
  • 削除された名前空間（スキーマ）は結果から除外されます。

アルゴリズムまたは振る舞い

1. fetch_search_path を呼び出し、検索パスの OID リストを取得します。 2. リスト内の各 OID に対して get_namespace_name を実行し、スキーマ名を取得します。 3. スキーマ名が存在する場合、DirectFunctionCall1 を介して namein 関数を呼び出し、cstring を name 型に変換して配列に格納します。 4. 収集した name 型の配列を construct_array_builtin で構築し、返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c（SQL クエリ経由での参照）
  • src/bin/pgbench/pgbench.c（SQL クエリ経由での参照）
  • **呼び出し先**:
  • fetch_search_path
  • get_namespace_name
  • namein
  • construct_array_builtin

関連データ構造

  • List: 検索パスの管理に使用。
  • ArrayType: 返却値の型。
  • name: 配列の要素型。

失敗モードと境界条件

  • 検索パスが空の場合、空の配列が返されます。
  • 検索パスに含まれる OID が既に削除されている場合、そのスキーマは結果からスキップされます。

保守担当者向けメモ

  • 本関数は SQL インターフェースとして公開されています。
  • namein を直接呼び出しているため、name 型の内部表現（NAMEDATALEN 固定長）の制約を受けます。
