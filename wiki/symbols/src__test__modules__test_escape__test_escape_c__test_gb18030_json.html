test_gb18030_json

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_escape/test_escape.c:212

目的

GB18030エンコーディングにおける不正なJSONユニコードエスケープシーケンスに対する、パーサーの境界処理とメモリ安全性を検証する。

シグネチャまたは形状

static void test_gb18030_json(pe_test_config *tc)

入力と出力

  • **入力**: pe_test_config 構造体へのポインタ（テスト設定）。
  • **出力**: なし（report_result を通じてテスト結果を標準出力へ記録）。

前提条件と不変条件

  • 入力データとして {"\u\xFE を使用する。
  • NEVER_ACCESS_STR をバッファ末尾に配置し、VALGRIND_MAKE_MEM_NOACCESS を用いて、パーサーが入力長を超えてメモリを読み取らないことを強制する。

アルゴリズムまたは振る舞い

1. createPQExpBuffer で入力バッファを確保し、テスト文字列と NEVER_ACCESS_STR を格納する。 2. VALGRIND_MAKE_MEM_NOACCESS を呼び出し、入力長以降のメモリ領域を保護する。 3. makeJsonLexContextCstringLen を使用し、PG_GB18030 エンコーディングを指定してJSONレキシカルコンテキストを作成する。 4. pg_parse_json を実行し、結果が JSON_UNICODE_ESCAPE_FORMAT エラーであることを確認する。 5. report_result で検証結果を報告し、確保したリソース（コンテキスト、バッファ）を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_escape/test_escape.c:958（main 関数内と推測される箇所より呼び出し）。
  • **呼び出し先**: createPQExpBuffer, appendBinaryPQExpBuffer, appendPQExpBufferStr, VALGRIND_MAKE_MEM_NOACCESS, escapify, makeJsonLexContextCstringLen, pg_parse_json, report_result, json_errdetail, freeJsonLexContext, destroyPQExpBuffer。

関連データ構造

  • pe_test_config: テスト設定を保持する構造体。
  • PQExpBuffer: 文字列バッファ管理用構造体。
  • JsonLexContext: JSON解析用コンテキスト。
  • JsonSemAction: JSON解析時のセマンティックアクション（本関数では空）。

失敗モードと境界条件

  • pg_parse_json が期待される JSON_UNICODE_ESCAPE_FORMAT 以外の値を返した場合、テストは失敗として報告される。
  • 入力バッファの境界を超えて読み取りが発生した場合、Valgrind等のメモリデバッガによって検出される。

保守担当者向けメモ

  • 本関数はGB18030特有のマルチバイト境界における脆弱性をテストする目的がある。
  • NEVER_ACCESS_STR の配置と VALGRIND_MAKE_MEM_NOACCESS の使用は、メモリ安全性の担保に不可欠であるため、変更時は注意すること。
