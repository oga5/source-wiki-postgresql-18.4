LLVMJitContext

定義

  • **種別**: struct
  • **定義箇所**: src/include/jit/llvmjit.h:43

目的

PostgreSQL の JIT コンパイルセッションにおけるライフサイクル管理およびリソース追跡を行うためのコンテキスト構造体です。LLVM のコンパイル状態、モジュール、および関連するメモリリソースを保持します。

シグネチャまたは形状

typedef struct LLVMJitContext
{
    JitContext    base;
    ResourceOwner resowner;
    size_t        module_generation;
    LLVMContextRef llvm_context;
    LLVMModuleRef module;
    bool          compiled;
    int           counter;
    List         *handles;
} LLVMJitContext;

入力と出力

  • **入力**: コンテキスト生成時に JitContext 基底構造体および ResourceOwner を受け取ります。
  • **出力**: コンパイルされた関数ハンドル（handles）および LLVM モジュール状態を保持します。

前提条件と不変条件

  • resowner は、コンテキストのクリーンアップを保証するために使用されます。
  • llvm_context は複数のコンパイル間で再利用されますが、メモリ消費を抑えるために適宜リセットされる可能性があります。
  • module は「書き込み可能」な状態として管理されます。

アルゴリズムまたは振る舞い

1. **初期化**: llvm_create_context により生成され、ResourceOwner に登録されます。 2. **コード生成**: llvm_mutable_module を通じて現在のモジュールを取得し、LLVM IR を生成します。 3. **管理**: counter を使用して、生成されるオブジェクト名の一意性を保証します。 4. **終了処理**: ResourceOwner を通じて、関連する LLVM リソースおよびメモリが安全に解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/jit/llvm/llvmjit.c 内の各関数（llvm_compile_module, llvm_optimize_module, llvm_mutable_module, llvm_get_function 等）から参照されます。
  • **呼び出し先**: LLVM C API（LLVMContextRef, LLVMModuleRef 等）を直接操作します。

関連データ構造

  • JitContext: 基底構造体。
  • ResourceOwner: リソースの所有権管理に使用。
  • List: Orc JIT 経由で生成されたコードハンドルを保持。

失敗モードと境界条件

  • メモリ不足（OOM）発生時、ResourceOwner を介した安全な終了処理が求められます。
  • llvm_context 内に型定義が蓄積しすぎた場合、メモリ節約のためにコンテキストのリセットが発生します。

保守担当者向けメモ

  • 本構造体は USE_LLVM が定義されている環境でのみ有効です。
  • Struct... 型定義と PostgreSQL の C 構造体のメモリレイアウトが一致していることを維持してください。
  • handles リストは Orc JIT 経由で生成されたコードの追跡に使用されます。
