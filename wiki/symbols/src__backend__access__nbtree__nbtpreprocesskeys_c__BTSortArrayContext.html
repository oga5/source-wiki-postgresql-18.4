BTSortArrayContext

定義

  • 種別: struct
  • 定義箇所: src/backend/access/nbtree/nbtpreprocesskeys.c:32

目的

B-tree インデックススキャンにおける配列型検索条件（IN 句や ANY 句など）のソート処理において、比較関数や照合順序（Collation）の状態を保持するために使用されます。

シグネチャまたは形状

typedef struct BTSortArrayContext
{
	FmgrInfo   *sortproc;   /* 比較用関数情報 */
	Oid			collation;  /* 照合順序 */
	bool		reverse;    /* ソート順序の反転フラグ */
} BTSortArrayContext;

入力と出力

本構造体は関数間で受け渡されるコンテキストオブジェクトとして機能します。

  • **入力**: _bt_preprocess_array_keys 内で初期化され、比較処理に必要な情報が格納されます。
  • **出力**: 比較関数（sortproc）の実行時や、ソート順序の判定に使用されます。

前提条件と不変条件

  • sortproc は有効な FmgrInfo 構造体を指している必要があります。
  • collation は、対象となるインデックス属性の照合順序と一致している必要があります。

アルゴリズムまたは振る舞い

配列条件の処理中、この構造体は比較ロジックのパラメータとして参照されます。特に reverse フラグは、インデックスのソート順（ASC/DESC）に応じて比較結果を調整するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtpreprocesskeys.c 内の _bt_preprocess_array_keys 関数にてインスタンス化されます。
  • **呼び出し先**: 比較処理の過程で、sortproc を介して内部的な比較関数が呼び出されます。

関連データ構造

  • FmgrInfo: PostgreSQL の関数マネージャ情報。
  • BTScanKeyPreproc: 本ファイル内で定義される、スキャンキーの事前処理状態を保持する構造体。

失敗モードと境界条件

  • sortproc が初期化されていない場合、比較実行時にセグメンテーションフォールトや予期せぬ動作を引き起こす可能性があります。
  • collation が不適切な場合、文字列比較において誤ったソート順序が適用される可能性があります。

保守担当者向けメモ

  • 本構造体は _bt_preprocess_array_keys の実行範囲内でのみ使用される一時的なコンテキストです。
  • 配列キーの処理ロジックを変更する際は、reverse フラグがインデックスの物理的な格納順序と論理的なクエリ順序の整合性を正しく保っているか確認してください。
