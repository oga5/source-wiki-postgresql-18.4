entryBeginPlaceToPage

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginentrypage.c:526

目的

GINエントリーツリーへのタプル挿入に先立ち、対象ページに十分な空き容量があるかを確認し、必要に応じてページ分割を行う。

シグネチャまたは形状

static GinPlaceToPageRC entryBeginPlaceToPage(
    GinBtree btree, 
    Buffer buf, 
    GinBtreeStack *stack,
    void *insertPayload, 
    BlockNumber updateblkno,
    void **ptp_workspace,
    Page *newlpage, 
    Page *newrpage
)

入力と出力

  • **入力**:
  • btree: GIN B-tree操作用構造体。
  • buf: 挿入対象のバッファ。
  • stack: 挿入位置を示すスタック情報。
  • insertPayload: 挿入するデータ（GinBtreeEntryInsertData型）。
  • updateblkno: 更新対象のブロック番号。
  • **出力**:
  • 戻り値: GPTP_SPLIT（分割が必要な場合）または GPTP_INSERT（そのまま挿入可能な場合）。
  • newlpage, newrpage: 分割が発生した場合、新しい左右のページデータが格納される。

前提条件と不変条件

  • insertPayload は GinBtreeEntryInsertData 型としてキャスト可能であること。
  • stack は有効な挿入位置（off）を保持していること。

アルゴリズムまたは振る舞い

1. entryIsEnoughSpace を呼び出し、現在のページに挿入データが収まるか判定する。 2. 容量不足の場合:

  • entrySplitPage を呼び出し、ページを分割して左右のページを生成する。
  • GPTP_SPLIT を返す。

    • 3. 容量が十分な場合:

  • 何もせず GPTP_INSERT を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginbtree.c の汎用B-treeエンジンから、btree->beginPlaceToPage 経由で呼び出される。
  • **呼び出し先**:
  • entryIsEnoughSpace
  • entrySplitPage

関連データ構造

  • GinBtree: GINインデックスのB-tree操作を抽象化する構造体。
  • GinBtreeEntryInsertData: エントリーツリーへの挿入データ。
  • GinBtreeStack: ページ探索時のスタック情報。

失敗モードと境界条件

  • ページ容量が不足している場合、entrySplitPage が実行される。
  • 呼び出し元や呼び出し先の詳細なエラーハンドリングについては、本ソース抜粋からは判断できない。

保守担当者向けメモ

  • 本関数は GinBtree のコールバックとして登録されるため、インターフェースの変更には注意が必要である。
  • ページ分割のロジックは entrySplitPage に委譲されているため、分割アルゴリズムの詳細はそちらを参照すること。
