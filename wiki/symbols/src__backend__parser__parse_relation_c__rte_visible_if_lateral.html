rte_visible_if_lateral

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_relation.c:3891

目的

指定された RangeTblEntry (RTE) が、LATERAL 結合のコンテキストにおいてのみ可視であるかどうかを判定します。

シグネチャまたは形状

static bool rte_visible_if_lateral(ParseState *pstate, RangeTblEntry *rte)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態を保持する ParseState 構造体へのポインタ。
  • rte: 可視性を確認対象とする RangeTblEntry 構造体へのポインタ。
  • **出力**:
  • bool: 指定された RTE が LATERAL 専用として可視であれば true、そうでなければ false を返します。

前提条件と不変条件

  • pstate->p_lateral_active が true の場合、本関数は常に false を返します。これは、LATERAL 処理が既にアクティブな状況下では、この関数による判定が不適切であるためです。

アルゴリズムまたは振る舞い

1. pstate->p_lateral_active を確認し、アクティブであれば直ちに false を返します。 2. findNSItemForRTE を呼び出し、対象の rte に対応する ParseNamespaceItem を検索します。 3. 対応する nsitem が見つかった場合、その p_lateral_only（LATERAL 専用フラグ）と p_lateral_ok（LATERAL 参照が許可されているか）の両方が true である場合にのみ true を返します。 4. 上記以外の場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_relation.c 内の複数箇所（3752行目、3811行目）で、RTE の可視性判定のために呼び出されています。
  • **呼び出し先**:
  • findNSItemForRTE（ParseState から RangeTblEntry に対応する名前空間アイテムを取得するために使用）。

関連データ構造

  • ParseState: クエリ解析の全体状態を保持。
  • RangeTblEntry: クエリ内のリレーション（テーブル、サブクエリ等）を表現。
  • ParseNamespaceItem: 名前空間内のアイテムを表現し、LATERAL 関連のフラグを保持。

失敗モードと境界条件

  • findNSItemForRTE が NULL を返した場合（対象の RTE が名前空間に見つからない場合）、本関数は false を返します。
  • p_lateral_active が設定されている場合、LATERAL 参照の可視性判定は抑制されます。

保守担当者向けメモ

  • 本関数は、LATERAL 結合における名前解決の制約を検証するための補助的なロジックです。
  • p_lateral_only と p_lateral_ok のフラグがどのように設定されるかは、ParseNamespaceItem の構築プロセスに依存します。変更を加える際は、名前空間の管理ロジックと整合性を取ってください。
