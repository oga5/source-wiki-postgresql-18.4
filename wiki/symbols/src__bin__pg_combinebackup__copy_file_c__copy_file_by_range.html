copy_file_by_range

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/copy_file.c:272

目的

copy_file_range システムコールを利用して、ソースファイルからデスティネーションファイルへ効率的にデータをコピーし、必要に応じてチェックサムを計算する。

シグネチャまたは形状

static void copy_file_by_range(const char *src, const char *dest, pg_checksum_context *checksum_ctx);

入力と出力

  • src (const char *): コピー元ファイルのパス。
  • dest (const char *): コピー先ファイルのパス。
  • checksum_ctx (pg_checksum_context *): チェックサム計算用のコンテキスト。
  • 戻り値: なし（成功時は終了、失敗時は pg_fatal によりプロセスを終了）。

前提条件と不変条件

  • HAVE_COPY_FILE_RANGE マクロが定義されている環境（Linux 等の対応 OS）であること。
  • src ファイルが読み取り可能であり、dest ファイルが作成可能であること。

アルゴリズムまたは振る舞い

1. src を読み取り専用、dest を書き込み・作成専用（排他モード）でオープンする。 2. copy_file_range をループ内で呼び出し、全データを転送する。 3. 転送完了後、ファイルディスクリプタをクローズする。 4. checksum_file を呼び出し、ファイル内容の整合性を検証する。 5. プラットフォームが copy_file_range をサポートしていない場合、pg_fatal を発生させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_combinebackup/copy_file.c 内のコピー戦略決定ロジック（strategy_implementation への代入）。
  • 呼び出し先: open, copy_file_range, close (システムコール)、checksum_file (補助関数)、pg_fatal (エラーハンドリング)。

関連データ構造

  • pg_checksum_context: チェックサム計算の状態を保持する構造体。

失敗モードと境界条件

  • ファイルのオープン失敗: pg_fatal によりエラーログを出力して終了。
  • copy_file_range の実行エラー: pg_fatal によりエラーログを出力して終了。
  • プラットフォーム非対応: HAVE_COPY_FILE_RANGE が未定義の場合、実行時に pg_fatal を発生させる。

保守担当者向けメモ

  • 本関数はカーネルレベルのコピー最適化を利用するため、OS のサポート状況に強く依存する。
  • copy_file_range は転送のみを行い、チェックサム計算は別途 checksum_file を呼び出す必要がある点に注意すること。
