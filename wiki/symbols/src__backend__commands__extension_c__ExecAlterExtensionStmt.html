ExecAlterExtensionStmt

定義

  • 種別: function
  • 定義箇所: src/backend/commands/extension.c:3407

目的

SQLコマンド ALTER EXTENSION を処理し、既存の拡張機能のバージョン更新を実行する。

シグネチャまたは形状

ObjectAddress ExecAlterExtensionStmt(ParseState *pstate, AlterExtensionStmt *stmt)

入力と出力

  • **入力**:
  • pstate: パーサー状態。
  • stmt: ALTER EXTENSION 文の構文解析木。extname（拡張機能名）および options（new_version 等の指定）を含む。
  • **出力**:
  • ObjectAddress: 更新された拡張機能のオブジェクトアドレス。既に指定バージョンである場合は InvalidObjectAddress を返す。

前提条件と不変条件

  • グローバル変数 creating_extension が真の場合、入れ子状の ALTER EXTENSION は許可されずエラーとなる。
  • 対象の拡張機能は pg_extension カタログに存在する必要がある。
  • 実行ユーザーは対象の拡張機能の所有者でなければならない。
  • new_version が指定されていない場合、制御ファイルの default_version が使用される。

アルゴリズムまたは振る舞い

1. **検証**: creating_extension フラグを確認し、入れ子呼び出しを拒否する。 2. **検索**: pg_extension をスキャンし、対象拡張機能の OID と現在のバージョン名を取得する。 3. **権限確認**: object_ownercheck により所有者権限を検証する。 4. **設定読み込み**: read_extension_control_file で制御ファイルを読み込む。 5. **バージョン特定**: stmt->options を走査し、更新先バージョンを決定する。 6. **更新パス特定**: identify_update_path を呼び出し、現在のバージョンから目標バージョンまでの更新スクリプトのリストを取得する。 7. **実行**: ApplyExtensionUpdates を呼び出し、更新スクリプトを順次実行してカタログを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c: ProcessUtility 経由で呼び出される。
  • **呼び出し先**:
  • table_open, systable_beginscan, systable_getnext, systable_endscan, table_close
  • read_extension_control_file
  • identify_update_path
  • ApplyExtensionUpdates
  • object_ownercheck, aclcheck_error

関連データ構造

  • AlterExtensionStmt: SQL文の解析結果を保持するノード。
  • ExtensionControlFile: 拡張機能の制御ファイル情報を保持する構造体。
  • pg_extension: 拡張機能のメタデータを格納するシステムカタログ。

失敗モードと境界条件

  • **エラー**:
  • 入れ子状の ALTER EXTENSION 呼び出し。
  • 拡張機能が存在しない場合。
  • 実行ユーザーが所有者でない場合。
  • バージョン名が指定されておらず、かつ制御ファイルに default_version がない場合。
  • 不明なオプションが指定された場合。
  • **境界条件**:
  • 現在のバージョンと更新先バージョンが同一の場合、通知（NOTICE）を出して処理を終了する。

保守担当者向けメモ

  • 本関数は ApplyExtensionUpdates を介してスクリプト実行を行うため、スクリプト実行中のエラーハンドリングは ApplyExtensionUpdates 側の実装に依存する。
  • creating_extension フラグの管理は、拡張機能のインストール/更新の一貫性を保つための重要な不変条件である。
