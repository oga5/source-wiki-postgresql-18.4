areasel

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_selfuncs.c:47

目的

幾何演算子（重なり等）に対する選択率（Selectivity）の推定値をプランナに提供する。

シグネチャまたは形状

Datum areasel(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（クエリの統計情報やデータ分布は参照されない）
  • 出力: float8 型の固定値 0.005 を PG_RETURN_FLOAT8 マクロ経由で返却する。

前提条件と不変条件

  • 常に固定値 0.005 を返す。
  • 入力引数の内容は評価に影響を与えない。

アルゴリズムまたは振る舞い

1. 関数が呼び出されると、統計情報に基づく計算を行わず、直ちに 0.005 を返却する。 2. この値は、プランナが幾何インデックス（GiST等）の利用を検討する際に、条件句の選択率として使用される。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLのクエリプランナ（幾何演算子のコスト算出時）。
  • 呼び出し先: なし。
  • 補足: ソースコード内のコメントにて、他の推定関数（areajoinsel等）との比較対象として言及されている。

関連データ構造

  • Datum: PostgreSQLの汎用データ型表現。
  • PG_FUNCTION_ARGS: 関数マネージャ（fmgr）の引数構造。

失敗モードと境界条件

  • 失敗モード: なし（常に固定値を返すため）。
  • 境界条件: 統計情報が欠如している場合や、データ分布に関わらず常に一定の選択率を提示する。

保守担当者向けメモ

  • 本関数は「完全にデタラメ（totally bogus）」な実装であり、統計情報に基づいた動的な推定を行っていない。
  • 幾何インデックスの利用を促すための暫定的なヒューリスティック値として機能している。
  • 将来的に幾何学的なデータ分布を考慮したアルゴリズムへ置き換える必要がある。
