tidgt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tid.c:211

目的

2つの TID (Tuple Identifier) を比較し、第一引数が第二引数よりも大きいかどうかを判定する。

シグネチャまたは形状

Datum tidgt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの ItemPointer を受け取る。
  • **出力**: bool 型の Datum。第一引数が第二引数より大きい場合は true、それ以外は false を返す。

前提条件と不変条件

  • 入力される ItemPointer は有効なポインタである必要がある。
  • ItemPointerCompare の比較ロジックに基づき、ブロック番号、次にオフセット番号の順で比較が行われる。

アルゴリズムまたは振る舞い

1. PG_GETARG_ITEMPOINTER を使用して、引数から2つの ItemPointer (arg1, arg2) を取得する。 2. ItemPointerCompare(arg1, arg2) を呼び出し、比較結果を取得する。 3. 比較結果が 0 より大きい（arg1 > arg2）場合、true を返す。それ以外は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL エンジン（演算子 > の実装として呼び出される）。
  • **呼び出し先**: ItemPointerCompare

関連データ構造

  • ItemPointer: タプルの物理位置（ブロック番号とオフセット番号）を保持するデータ構造。

失敗モードと境界条件

  • 引数が NULL の場合、PostgreSQL の関数呼び出し規約に従い、呼び出し元で適切に処理される必要がある（通常は strict 属性により自動的に NULL が返される）。
  • ItemPointerCompare の仕様に依存し、ブロック番号とオフセット番号の大小関係がそのまま比較結果に反映される。

保守担当者向けメモ

  • 本関数は TID 型の比較演算子の一部である。比較ロジックを変更する場合は ItemPointerCompare を参照すること。
  • 物理的なタプル位置を比較するため、VACUUM FULL 等でタプルが移動した後の値に対して使用する際は注意が必要である。
