mulShift

定義

  • 種別: function
  • 定義箇所: src/common/d2s.c:161

目的

Ryu アルゴリズムの一部として、128ビット精度の乗算とビットシフトを組み合わせた計算を行い、浮動小数点数の変換に必要なスケーリングを実行します。

シグネチャまたは形状

static inline uint64 mulShift(const uint64 m, const uint64 *const mul, const int32 j)

入力と出力

  • **入力**:
  • m: 乗数となる64ビット整数。
  • mul: 2つの64ビット整数からなる配列（乗算用定数）。
  • j: シフト量を決定する32ビット整数。
  • **出力**:
  • 計算結果の64ビット整数。

前提条件と不変条件

  • mul 配列は少なくとも2つの要素を持つ必要があります。
  • 内部で uint128 型を使用するため、コンパイル環境が128ビット整数演算をサポートしている必要があります。

アルゴリズムまたは振る舞い

1. 入力 m と mul[0]、mul[1] をそれぞれ128ビットにキャストして乗算します。 2. b0（m * mul[0]）の上位64ビットと b2（m * mul[1]）を加算します。 3. 加算結果を j - 64 ビット分だけ右シフトし、結果を64ビットに切り詰めて返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/d2s.c 内の複数の箇所（d2d ロジックに関連する計算）および src/common/f2s.c から呼び出されています。
  • **呼び出し先**: 外部関数への呼び出しはありません。

関連データ構造

  • uint64, uint128, int32: 演算に使用される固定幅整数型。

失敗モードと境界条件

  • j の値が64未満の場合、右シフト演算が未定義の挙動を引き起こす可能性があります。
  • 128ビット演算が利用できない環境では、RYU_ONLY_64_BIT_OPS マクロによる代替実装が使用されます。

保守担当者向けメモ

  • 本関数は非常に頻繁に呼び出されるため、インライン化が維持されています。
  • 変更を加える際は、d2d 内の丸め処理や f2s.c での利用と整合しているか、ビット演算のロジックを慎重に検証してください。
