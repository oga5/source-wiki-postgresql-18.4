pg_lsn_ne

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_lsn.c:126

目的

2つの pg_lsn 値が等しくないかどうかを判定し、SQLの不一致演算子（<>）のバックエンド実装を提供します。

シグネチャまたは形状

Datum pg_lsn_ne(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの pg_lsn 型の引数を受け取ります。
  • **出力**: bool 型（Datum 形式）。2つの値が異なる場合は true、等しい場合は false を返します。

前提条件と不変条件

  • 入力引数は PG_GETARG_LSN マクロを使用して取得されます。
  • pg_lsn は内部的に64ビット整数（XLogRecPtr）として扱われます。

アルゴリズムまたは振る舞い

1. PG_GETARG_LSN(0) および PG_GETARG_LSN(1) を呼び出し、2つの XLogRecPtr 値を取得します。 2. 取得した2つの値を比較演算子 != で比較します。 3. 比較結果を PG_RETURN_BOOL マクロで返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジンから演算子 <> を通じて呼び出されます。
  • **呼び出し先**: PG_GETARG_LSN および PG_RETURN_BOOL マクロを使用します。これら以外の具体的な関数呼び出しはソースコード内に存在しません。

関連データ構造

  • XLogRecPtr: LSNの内部表現である64ビット整数型。

失敗モードと境界条件

  • 引数が正しく渡されない場合、PG_GETARG_LSN マクロの仕様に従いエラーが発生する可能性があります。
  • 64ビット整数の比較であるため、値の範囲による境界条件は存在しません。

保守担当者向けメモ

  • 本関数は単純な比較ロジックであり、pg_lsn_eq の否定結果を返すのと同等です。
  • 比較対象が XLogRecPtr であるため、算術演算のような numeric 型への変換コストは発生しません。
