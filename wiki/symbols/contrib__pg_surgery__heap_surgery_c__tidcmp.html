tidcmp

定義

  • 種別: function
  • 定義箇所: contrib/pg_surgery/heap_surgery.c:356

目的

ItemPointer（TID）のペアを比較し、qsort 等のソートアルゴリズムで使用可能な比較結果を返すための比較関数です。

シグネチャまたは形状

static int32 tidcmp(const void *a, const void *b)

入力と出力

  • **入力**:
  • a: 比較対象の ItemPointer へのポインタ（const void *）。
  • b: 比較対象の ItemPointer へのポインタ（const void *）。
  • **出力**:
  • int32: a が b より小さい場合は負の値、等しい場合は 0、大きい場合は正の値を返します。

前提条件と不変条件

  • 入力 a および b は、有効な ItemPointer 構造体へのポインタである必要があります。

アルゴリズムまたは振る舞い

1. 引数 a と b を ItemPointer 型にキャストします。 2. ItemPointerCompare 関数を呼び出し、その結果をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_surgery/heap_surgery.c 内の heap_force_common 関数（137行目）において、TID 配列をソートするために qsort の引数として呼び出されます。
  • **呼び出し先**: ItemPointerCompare（PostgreSQL のコア関数）。

関連データ構造

  • ItemPointer: タプル識別子（TID）を保持するデータ構造。

失敗モードと境界条件

  • 入力ポインタが NULL の場合の動作は定義されていません。

保守担当者向けメモ

  • 本関数は qsort のコールバックとしてのみ使用される静的関数です。比較ロジックを変更する必要がある場合は、ItemPointerCompare の仕様を確認してください。
