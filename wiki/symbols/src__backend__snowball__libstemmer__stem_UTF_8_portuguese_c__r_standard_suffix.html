r_standard_suffix

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_portuguese.c:644

目的

ポルトガル語のステミング処理において、単語の末尾から標準的な接尾辞を特定し、削除または置換を行う。

シグネチャまたは形状

static int r_standard_suffix(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（ステミング対象の単語データと現在の処理状態を保持する構造体）
  • **出力**: int（成功時に 1、失敗または条件不一致時に 0、エラー発生時に負の値を返す）

前提条件と不変条件

  • z->c（現在のカーソル位置）が単語の末尾付近にあること。
  • find_among_b 関数により、定義済みの接尾辞テーブル a_5 との照合が行われる。
  • 多くのケースで、接尾辞の削除前に r_R1 または r_R2 関数による領域チェックが必須となる。

アルゴリズムまたは振る舞い

1. z->ket を現在のカーソル位置に設定し、末尾の文字が特定の条件を満たすか確認する。 2. find_among_b を使用して、テーブル a_5（45個の接尾辞パターン）から一致するものを検索する。 3. 一致した among_var に応じて以下の処理を行う：

  • **ケース 1-4, 6-8**: r_R2 領域内であることを確認後、接尾辞を削除または指定文字列に置換する。一部のケースでは、削除後にさらに別の接尾辞（a_2, a_3, a_4 等）の削除を試みる。
  • **ケース 5**: r_R1 領域内であることを確認後、接尾辞を削除し、さらに a_2 テーブルに基づく追加の削除を試みる。
  • **ケース 9**: r_RV 領域内であることを確認後、末尾の 'e' を削除し、指定文字列に置換する。

呼び出し元と呼び出し先

  • **呼び出し元**: portuguese_UTF_8_stem（メインのステミング関数）から呼び出される。
  • **呼び出し先**: find_among_b, r_R1, r_R2, r_RV, slice_del, slice_from_s, eq_s_b。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_5: 削除対象となる標準接尾辞の定義テーブル。
  • a_2, a_3, a_4: 追加削除対象となる接尾辞テーブル。

失敗モードと境界条件

  • r_R1 や r_R2 の領域チェックに失敗した場合、処理は中断され 0 または負の値が返される。
  • 文字列の先頭や境界（z->lb）に達した場合、安全に終了する。
  • slice_del 等のメモリ操作関数が負の値を返した場合、エラーとして伝播する。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードであるため、直接編集せず .sbl ソースファイルを修正すること。
  • among_var の各ケースはポルトガル語の文法規則に基づいている。ロジックの変更は、対応する接尾辞テーブルの定義と整合性を取る必要がある。
