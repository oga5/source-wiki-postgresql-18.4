TransactionLogFetch

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/transam.c:51

目的

指定されたトランザクションID（XID）のコミット状態を CLOG（Commit Log）から取得し、結果を単一アイテムキャッシュに保持することで、頻繁な参照を最適化します。

シグネチャまたは形状

static XidStatus TransactionLogFetch(TransactionId transactionId)

入力と出力

  • **入力**: transactionId (判定対象のトランザクションID)
  • **出力**: XidStatus 型のステータス値（TRANSACTION_STATUS_COMMITTED, TRANSACTION_STATUS_ABORTED, TRANSACTION_STATUS_IN_PROGRESS, TRANSACTION_STATUS_SUB_COMMITTED のいずれか）

前提条件と不変条件

  • cachedFetchXid に格納されたキャッシュは、状態が確定している（IN_PROGRESS や SUB_COMMITTED ではない）トランザクションのみを対象とします。
  • BootstrapTransactionId および FrozenTransactionId は常にコミット済みとして扱われます。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: transactionId が cachedFetchXid と一致する場合、キャッシュされたステータスを即座に返します。 2. **特殊ID判定**: TransactionIdIsNormal でない場合、BootstrapTransactionId または FrozenTransactionId ならばコミット済み、それ以外はアボート済みと判定します。 3. **CLOG参照**: TransactionIdGetStatus を呼び出し、実際のステータスと対応するLSNを取得します。 4. **キャッシュ更新**: 取得したステータスが確定状態（IN_PROGRESS または SUB_COMMITTED 以外）である場合のみ、cachedFetchXid、cachedFetchXidStatus、cachedCommitLSN を更新します。 5. **結果返却**: 取得したステータスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/transam.c 内の TransactionIdDidCommit および TransactionIdDidAbort から呼び出されます。
  • **呼び出し先**:
  • TransactionIdEquals (マクロ)
  • TransactionIdIsNormal (マクロ)
  • TransactionIdGetStatus (CLOG層の関数)

関連データ構造

  • XidStatus: トランザクションの状態を定義する列挙型。
  • cachedFetchXid, cachedFetchXidStatus, cachedCommitLSN: transam.c 内で定義された単一アイテムキャッシュ用の静的変数。

失敗モードと境界条件

  • **ラップアラウンド**: XIDは循環するため、直接的な比較ではなく TransactionIdEquals 等の専用マクロを使用する必要があります。
  • **未確定状態**: IN_PROGRESS や SUB_COMMITTED の状態はキャッシュされないため、呼び出し元は必要に応じて再帰的な解決（親トランザクションの追跡など）を行う必要があります。

保守担当者向けメモ

  • 本関数は static であり、モジュール内部でのみ使用されます。
  • キャッシュは単一アイテムのみであるため、複数のトランザクションを頻繁に切り替えて参照する処理ではキャッシュミスが発生します。
  • TransactionIdGetStatus が返すLSN情報もキャッシュ対象に含まれるため、変更時には整合性に注意してください。
