test_huge_distances

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_integerset/test_integerset.c:514

目的

IntegerSet における Simple-8b エンコーディングの制約（2^60）を跨ぐ距離を持つ値の追加・検索・走査が正しく機能することを検証する。

シグネチャまたは形状

static void test_huge_distances(void)

入力と出力

  • 入力: なし
  • 出力: なし（検証失敗時は elog(ERROR) によりテストを中断）

前提条件と不変条件

  • IntegerSet が空の状態から開始される。
  • 追加される値は昇順であり、かつ uint64 の範囲内に収まる必要がある。

アルゴリズムまたは振る舞い

1. **データ生成**: 2^60 を境界とする距離を持つ一連の値を生成し、最後にランダムな値を追加して合計1000個の値を準備する。 2. **構築**: intset_create() で生成した集合に対し、intset_add_member() を用いて全値を登録する。 3. **メンバ判定テスト**: 各値の前後（±1）および値自身に対して intset_is_member() を実行し、期待値と一致するか確認する。 4. **イテレーションテスト**: intset_begin_iterate() および intset_iterate_next() を使用し、登録した順序と値が完全に一致するかを検証する。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/test_integerset/test_integerset.c 内のメインテストルーチンから呼び出される。
  • 呼び出し先: intset_create, intset_add_member, intset_is_member, intset_begin_iterate, intset_iterate_next（すべて lib/integerset.h 定義）。

関連データ構造

  • IntegerSet: テスト対象の整数集合構造体。

失敗モードと境界条件

  • 2^60 を超える距離を持つ値の処理において、内部的なバッファリングやツリー構造へのフラッシュが正しく行われない場合に失敗する。
  • intset_is_member が境界値（±1）を誤判定した場合にエラーを報告する。
  • イテレーション中に値の欠落や順序の不一致が発生した場合にエラーを報告する。

保守担当者向けメモ

  • 本関数は Simple-8b エンコーディングの限界をテストする重要なケースである。実装の変更時には、特に 2^60 前後の値のエンコード・デコードロジックが正しく動作しているか確認すること。
