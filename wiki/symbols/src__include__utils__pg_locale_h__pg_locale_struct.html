pg_locale_struct

定義

  • 種別: struct
  • 定義箇所: src/include/utils/pg_locale.h:115

目的

PostgreSQLにおけるロケールおよび照合順序（Collation）処理の抽象化レイヤーを担う構造体です。libc、ICU、および組み込みロケールプロバイダーを統一的に扱い、実行時の照合順序や文字変換の挙動をカプセル化します。

シグネチャまたは形状

struct pg_locale_struct
{
    char        provider;
    bool        deterministic;
    bool        collate_is_c;
    bool        ctype_is_c;
    bool        is_default;
    const struct collate_methods *collate;
    union {
        struct { const char *locale; bool casemap_full; } builtin;
        locale_t    lt;
#ifdef USE_ICU
        struct { const char *locale; UCollator *ucol; } icu;
#endif
    } info;
};

入力と出力

  • **入力**: データベースのロケール設定（GUC）や照合順序OIDに基づき、pg_newlocale_from_collation 等を通じて初期化されます。
  • **出力**: 構造体内の collate メソッドテーブルおよび info 共用体を通じて、後続の文字列操作関数（pg_strcoll 等）へロケール固有の処理ロジックを提供します。

前提条件と不変条件

  • collate_is_c が真の場合、collate メンバは NULL となります。
  • provider の値に応じて、info 共用体の適切なメンバ（builtin, lt, icu）が使用されます。
  • ICUプロバイダーを使用する場合、USE_ICU マクロが定義されている必要があります。

アルゴリズムまたは振る舞い

本構造体はポリモーフィズムを実現しており、文字列操作関数は collate メソッドテーブルを参照して適切な実装（libcまたはICU固有の関数）へディスパッチします。collate_is_c や ctype_is_c が真の場合は、重いロケール処理をスキップする高速パスが選択されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale.c
  • src/backend/utils/adt/pg_locale_builtin.c
  • src/backend/utils/adt/pg_locale_icu.c
  • src/backend/utils/adt/pg_locale_libc.c

    • （上記各ファイルでメモリ確保および初期化が行われます）

  • **呼び出し先**:
  • collate_methods 構造体（メソッドテーブルとして参照）
  • UCollator（ICU使用時）

関連データ構造

  • pg_locale_t: struct pg_locale_struct * への型定義。
  • collate_methods: 照合順序の挙動を定義する関数ポインタテーブル。
  • UCollator: ICUライブラリの照合順序オブジェクト。

失敗モードと境界条件

  • メモリ不足時に MemoryContextAllocZero が失敗する可能性があります。
  • ICUが有効でない環境でICUプロバイダーのロケールを扱おうとした場合の挙動は、ビルド構成に依存します。

保守担当者向けメモ

  • 本構造体は実行時のプロバイダー切り替えを支える重要な抽象化レイヤーです。変更時は collate_methods との整合性に注意してください。
  • collate_is_c 等のフラグはパフォーマンス最適化に直結するため、ロジック追加時はこれらのフラグによる分岐が適切か確認してください。
