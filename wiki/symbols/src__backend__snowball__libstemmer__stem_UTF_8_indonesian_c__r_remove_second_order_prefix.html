r_remove_second_order_prefix

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_indonesian.c:273

目的

インドネシア語のステミング処理において、単語の先頭から「第二順位の接頭辞（second-order prefix）」を特定し、除去または置換を行う。

シグネチャまたは形状

static int r_remove_second_order_prefix(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（ステミング環境の状態を保持する構造体）
  • **出力**: int（成功時に 1、失敗または対象外の場合に 0、エラー発生時に負の値を返す）

前提条件と不変条件

  • z->c（現在のカーソル位置）が単語の先頭付近にあること。
  • z->p（文字バッファ）の z->c + 1 位置に文字コード 101（'e'）が存在すること。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置 z->c を z->bra に設定する。 2. z->c + 1 がバッファ範囲内であり、かつ文字が 'e' であるかを確認する。 3. find_among を使用し、配列 a_4（サイズ 6）に基づいて接頭辞パターンを検索する。 4. マッチした場合、z->ket を現在のカーソル位置に設定し、マッチしたパターン（among_var）に応じて以下の処理を行う：

  • **ケース 1/3**: slice_del で接頭辞を削除し、状態フラグ z->I[0] を更新する。
  • **ケース 2/4**: slice_from_s で接頭辞を別の文字列に置換する。

    • 5. 全ケース共通で、ステミングの閾値 z->I[1] を 1 減算する。

呼び出し元と呼び出し先

  • **呼び出し元**: indonesian_UTF_8_stem（stem_UTF_8_indonesian.c 内のステミングメイン処理）から呼び出される。
  • **呼び出し先**: find_among（パターン検索）、slice_del（削除）、slice_from_s（置換）。

関連データ構造

  • SN_env: ステミング環境を管理する構造体。I[0]（状態フラグ）、I[1]（閾値）、c（カーソル）、bra/ket（範囲指定）を含む。
  • a_4: 第二順位の接頭辞パターンを定義する among 構造体の配列。

失敗モードと境界条件

  • z->c + 1 がバッファの末尾を超える場合、または 'e' 以外の場合は 0 を返す。
  • find_among がマッチしない場合は 0 を返す。
  • slice_del または slice_from_s が内部エラー（負の値）を返した場合、その値をそのまま呼び出し元に伝播させる。
