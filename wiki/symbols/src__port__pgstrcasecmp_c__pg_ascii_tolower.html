pg_ascii_tolower

定義

  • 種別: function
  • 定義箇所: src/port/pgstrcasecmp.c:145

目的

ロケール設定に依存せず、ASCII文字（'A'-'Z'）を小文字（'a'-'z'）に変換する。

シグネチャまたは形状

unsigned char pg_ascii_tolower(unsigned char ch)

入力と出力

  • **入力**: unsigned char ch（変換対象の文字）
  • **出力**: unsigned char（変換後の文字。ASCIIの大文字であれば小文字に変換され、それ以外は入力値がそのまま返される）

前提条件と不変条件

  • 入力値が 'A' から 'Z' の範囲内である場合、必ず対応する小文字に変換される。
  • 入力値が ASCII の範囲外、または小文字である場合、値は変更されない。
  • ロケール設定（環境変数等）の影響を受けない。

アルゴリズムまたは振る舞い

1. 入力文字 ch が 'A' 以上かつ 'Z' 以下であるか判定する。 2. 条件を満たす場合、ch に 'a' - 'A'（差分）を加算して小文字に変換する。 3. 変換後の値、または条件を満たさなかった元の値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/crc32.c
  • contrib/ltree/lquery_op.c
  • src/backend/regex/regc_pg_locale.c
  • src/backend/utils/adt/formatting.c
  • **呼び出し先**:
  • 本関数は標準ライブラリ関数を呼び出さず、算術演算のみで完結する。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • **境界条件**: 入力値が 'A' または 'Z' の場合、正しく変換が行われる。
  • **失敗モード**: 8ビット文字（ASCII範囲外）に対しては変換を行わないため、マルチバイト文字セットの処理には適さない。

保守担当者向けメモ

  • 本関数はロケールに依存しない厳密なASCII変換を保証する。ロケール依存の変換が必要な場合は pg_tolower を使用すること。
  • src/backend/parser/scansup.c 等の他の識別子正規化ロジックと整合性を保つ必要がある。
