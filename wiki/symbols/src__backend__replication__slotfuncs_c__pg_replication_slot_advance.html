pg_replication_slot_advance

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slotfuncs.c:509

目的

指定されたレプリケーションスロットのWAL読み取り位置（LSN）を、指定されたターゲット位置まで強制的に進めます。

シグネチャまたは形状

Datum pg_replication_slot_advance(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • slotname (Name): 対象とするレプリケーションスロット名。
  • moveto (XLogRecPtr): 進捗先のターゲットLSN。
  • **出力**:
  • HeapTuple: スロット名と、実際に到達したLSNを含む行データ。

前提条件と不変条件

  • 呼び出し元プロセスは、現在他のレプリケーションスロットを保持していないこと（Assert(!MyReplicationSlot)）。
  • ターゲットLSNが有効であること。
  • スロットが過去に一度もWALを予約していない、または無効化されている場合は進捗不可。
  • ターゲットLSNは、現在のスロットの最小LSN（物理スロットなら restart_lsn、論理スロットなら confirmed_flush）より前であってはならない。

アルゴリズムまたは振る舞い

1. 権限チェックを行い、ターゲットLSNが有効か確認する。 2. データベースの状態（リカバリ中か否か）に応じて、ターゲットLSNをフラッシュ済みまたはリプレイ済みのLSNでクランプ（上限設定）する。 3. ReplicationSlotAcquire を呼び出し、対象スロットを排他的に確保する。 4. スロットの種類（論理か物理か）を判定し、それぞれ pg_logical_replication_slot_advance または pg_physical_replication_slot_advance を呼び出して更新を実行する。 5. 更新後、システム全体の xmin および LSN の必要量を再計算する。 6. スロットを解放し、到達したLSNを結果として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（詳細はソースコード外のSQLレイヤーに依存）。
  • **呼び出し先**:
  • ReplicationSlotAcquire / ReplicationSlotRelease
  • pg_logical_replication_slot_advance / pg_physical_replication_slot_advance
  • ReplicationSlotsComputeRequiredXmin / ReplicationSlotsComputeRequiredLSN

関連データ構造

  • MyReplicationSlot: 現在のバックエンドが保持するスロット情報。
  • XLogRecPtr: WAL上の位置を示すデータ型。

失敗モードと境界条件

  • **無効なLSN**: moveto が無効な場合、エラーを報告。
  • **権限不足**: CheckSlotPermissions により権限がない場合はエラー。
  • **進捗不可**: スロットが初期化されていない、または無効化されている場合にエラー。
  • **逆行**: ターゲットLSNが現在の最小LSNより小さい場合、エラーを報告。
  • **型不一致**: 戻り値の型が複合型でない場合、エラーを報告。

保守担当者向けメモ

  • 本関数はSQLインターフェースであり、内部的なスロット状態の更新は slot.c のロジックに依存しています。
  • ターゲットLSNのクランプ処理は、リカバリ状態によって参照先（GetFlushRecPtr vs GetXLogReplayRecPtr）が切り替わるため注意してください。
