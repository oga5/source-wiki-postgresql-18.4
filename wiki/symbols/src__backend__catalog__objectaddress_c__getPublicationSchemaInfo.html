getPublicationSchemaInfo

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaddress.c:2863

目的

ObjectAddress で指定されたパブリケーションとスキーマの関連付け情報をシステムカタログから取得し、パブリケーション名とスキーマ名を解決します。

シグネチャまたは形状

static bool getPublicationSchemaInfo(const ObjectAddress *object, bool missing_ok, char **pubname, char **nspname);

入力と出力

  • **入力**:
  • object: 情報を取得する対象の ObjectAddress。
  • missing_ok: true の場合、対象が見つからなくてもエラーにせず false を返す。
  • pubname: パブリケーション名を格納するポインタのポインタ。
  • nspname: スキーマ名を格納するポインタのポインタ。
  • **出力**:
  • 成功時に true を返し、pubname と nspname に値を設定する。失敗時は false を返す。

前提条件と不変条件

  • object->objectId は pg_publication_namespace カタログの有効な OID である必要があります。
  • 呼び出し側は、出力引数として渡した char * ポインタのメモリ管理（解放）に責任を持つ必要があります。

アルゴリズムまたは振る舞い

1. PUBLICATIONNAMESPACE システムキャッシュを object->objectId で検索します。 2. タプルが見つからない場合、missing_ok に応じてエラーを投げるか false を返します。 3. タプルから pg_publication_namespace の情報を抽出します。 4. get_publication_name を呼び出し、パブリケーション名を取得します。取得失敗時はキャッシュを解放して false を返します。 5. get_namespace_name を呼び出し、スキーマ名を取得します。取得失敗時はパブリケーション名のメモリを解放し、キャッシュを解放して、missing_ok に応じてエラーを投げるか false を返します。 6. 全ての取得が成功した場合、キャッシュを解放して true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/objectaddress.c 内の関数（行 2857, 3994, 5918 付近で参照）。
  • **呼び出し先**: SearchSysCache1, GETSTRUCT, get_publication_name, get_namespace_name, ReleaseSysCache, pfree, elog。

関連データ構造

  • ObjectAddress: オブジェクトを特定するための構造体。
  • Form_pg_publication_namespace: pg_publication_namespace カタログの行構造。

失敗モードと境界条件

  • 対象の OID がカタログに存在しない場合、missing_ok が false なら ERROR を発生させます。
  • パブリケーション名またはスキーマ名が解決できない場合、関数は false を返し、確保したメモリを適切に解放します。

保守担当者向けメモ

  • 本関数は pg_publication_namespace カタログに依存しています。スキーマやパブリケーションの定義変更に伴うキャッシュの整合性に注意してください。
