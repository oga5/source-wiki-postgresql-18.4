lquery_level

定義

  • 種別: struct
  • 定義箇所: contrib/ltree/ltree.h:88

目的

lquery 構造体の一部として、検索パターンにおける単一のレベル（階層）を定義します。パスの照合において、特定のラベルのバリエーションや繰り返し回数を管理するために使用されます。

シグネチャまたは形状

typedef struct {
    uint16      totallen;       /* このレベルの合計バイト数 */
    uint16      flag;           /* LQL_xxx および LVAR_xxx フラグ */
    uint16      numvar;         /* バリアントの数（0は '*' を意味する） */
    uint16      low;            /* 最小繰り返し回数 */
    uint16      high;           /* 最大繰り返し回数 */
    char        variants[FLEXIBLE_ARRAY_MEMBER]; /* lquery_variant 構造体の配列 */
} lquery_level;

入力と出力

  • **入力**: lquery 構造体の一部としてメモリ上に配置されます。
  • **出力**: checkLevel 関数等において、ltree_level との照合結果を判定するためのデータソースとして利用されます。

前提条件と不変条件

  • variants フィールドは可変長配列であるため、構造体のサイズ計算には sizeof を直接使用せず、LQL_HDRSIZE マクロおよび totallen を用いて計算する必要があります。
  • variants 配列内の各要素は MAXALIGN に準拠している必要があります。
  • numvar が 0 の場合、ワイルドカード * として扱われます。

アルゴリズムまたは振る舞い

  • lquery 内の各レベルを走査し、flag に格納されたバリアント情報に基づいて、対象となる ltree のレベルとマッチングを行います。
  • LQL_NEXT マクロを使用して、メモリ上の次の lquery_level へオフセット計算により移動します。
  • LQL_CANLOOKSIGN マクロにより、インデックス検索時にシグネチャを確認可能かどうかの判定に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照）**:
  • contrib/ltree/_ltree_gist.c: LQUERY_FIRST を介してクエリの先頭レベルを取得。
  • contrib/ltree/lquery_op.c: checkLevel 関数および checkCond 関数内で、パスとの照合ロジックに使用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。ロジックは lquery_op.c 等の関数によって処理されます。

関連データ構造

  • lquery: lquery_level を複数保持する親構造体。
  • lquery_variant: variants 配列内に格納される、個別のラベル条件を定義する構造体。
  • ltree_level: 比較対象となるパスの階層構造。

失敗モードと境界条件

  • **アライメント違反**: FLEXIBLE_ARRAY_MEMBER を含むため、MAXALIGN を考慮しないポインタ演算を行うとメモリ破壊や不正なアクセスが発生します。
  • **範囲外アクセス**: totallen を超えて variants を走査した場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • lquery_level の構造を変更する場合、LQL_NEXT や LQL_HDRSIZE マクロの定義も併せて更新する必要があります。
  • 過去のディスクフォーマットとの互換性維持のため、low および high フィールドの解釈には注意してください。
