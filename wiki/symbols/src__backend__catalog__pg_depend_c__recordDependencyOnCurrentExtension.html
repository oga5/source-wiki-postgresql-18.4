recordDependencyOnCurrentExtension

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_depend.c:194

目的

現在実行中の拡張機能（CurrentExtensionObject）と、指定されたオブジェクトとの間に依存関係（DEPENDENCY_EXTENSION）を記録します。

シグネチャまたは形状

void recordDependencyOnCurrentExtension(const ObjectAddress *object, bool isReplace)

入力と出力

  • object: 依存関係の対象となるオブジェクトのアドレス（ObjectAddress 構造体）。
  • isReplace: 既存の依存関係を置換するかどうかを示すフラグ。
  • 出力: なし（成功時）。失敗時はエラーをスローします。

前提条件と不変条件

  • object->objectSubId は 0 である必要があります（拡張機能のメンバーはオブジェクト全体である必要があるため）。
  • creating_extension が真である場合にのみ処理が実行されます。

アルゴリズムまたは振る舞い

1. creating_extension が真であることを確認します。 2. isReplace が真の場合、対象オブジェクトが既に他の拡張機能に属していないかを確認します。

  • 既に現在の拡張機能のメンバーであれば、何もせず終了します。
  • 他の拡張機能のメンバーである場合、またはどの拡張機能にも属していない（かつ置換を試みている）場合は、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE エラーを発生させます。

    • 3. 上記のチェックを通過した場合、CurrentExtensionObject を参照先として、recordDependencyOn を呼び出し、DEPENDENCY_EXTENSION 型の依存関係を永続化します。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下のソースファイルから呼び出されていることが確認されています。
  • src/backend/catalog/heap.c
  • src/backend/catalog/pg_cast.c
  • src/backend/catalog/pg_collation.c
  • src/backend/catalog/pg_conversion.c
  • src/backend/catalog/pg_namespace.c
  • src/backend/catalog/pg_operator.c
  • src/backend/catalog/pg_proc.c
  • src/backend/catalog/pg_type.c
  • src/backend/commands/amcmds.c
  • **呼び出し先**: getExtensionOfObject, recordDependencyOn, getObjectDescription, get_extension_name 等。

関連データ構造

  • ObjectAddress: オブジェクトを識別するための構造体。
  • ExtensionRelationId: 拡張機能カタログのOID。
  • CurrentExtensionObject: 現在処理中の拡張機能OIDを保持するグローバル変数。

失敗モードと境界条件

  • isReplace が真の際、対象オブジェクトが既に別の拡張機能に属している場合、またはどの拡張機能にも属していない場合にエラーとなります。
  • objectSubId が 0 でない場合、Assert により異常終了します。

保守担当者向けメモ

  • 本関数は CREATE EXTENSION 実行中の整合性維持に不可欠です。
  • isReplace が偽の場合、呼び出し元は CommandCounterIncrement を実行していない可能性があるため、カタログルックアップの安全性には注意が必要です。
