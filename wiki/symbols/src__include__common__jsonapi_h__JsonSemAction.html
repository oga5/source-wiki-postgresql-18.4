JsonSemAction

定義

  • 種別: struct
  • 定義箇所: src/include/common/jsonapi.h:151

目的

JSON 解析プロセスにおいて、各トークンや構造の開始・終了時に呼び出されるコールバック関数のテーブルを保持し、解析結果をユーザー定義の状態（semstate）へ抽出するためのインターフェースを提供します。

シグネチャまたは形状

typedef struct JsonSemAction
{
    void           *semstate;
    json_struct_action object_start;
    json_struct_action object_end;
    json_struct_action array_start;
    json_struct_action array_end;
    json_ofield_action object_field_start;
    json_ofield_action object_field_end;
    json_aelem_action array_element_start;
    json_aelem_action array_element_end;
    json_scalar_action scalar;
} JsonSemAction;

入力と出力

  • **入力**: ユーザーが定義した semstate（任意のデータ構造へのポインタ）および各 JSON 要素に対応する関数ポインタ。
  • **出力**: パーサーが解析中に各コールバックを呼び出すことで、semstate を介してデータが構築・蓄積されます。

前提条件と不変条件

  • 各コールバック関数は、解析エラー発生時に JSON_SEM_ACTION_FAILED を返すことで異常終了を通知できます。
  • semstate は、コールバック関数内で適切にキャストして利用される必要があります。

アルゴリズムまたは振る舞い

1. パーサー（pg_parse_json 等）が JSON 入力を走査します。 2. トークンや構造（オブジェクト、配列、フィールド、スカラー値）を認識するたびに、対応する関数ポインタが呼び出されます。 3. ユーザー定義のコールバック関数は、semstate を更新して JSON の内容を抽出・変換します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/json.c および jsonb.c、jsonfuncs.c 内の関数群が本構造体を初期化し、パーサーへ渡しています。
  • **呼び出し先**: 構造体内の各関数ポインタは、解析の進捗に応じてパーサーから呼び出されます。

関連データ構造

  • JsonLexContext: 解析状態を保持するメイン構造体。
  • json_struct_action, json_ofield_action, json_aelem_action, json_scalar_action: 各コールバック関数の型定義。

失敗モードと境界条件

  • コールバック関数が JSON_SEM_ACTION_FAILED を返した場合、パーサーは解析を停止し、エラーとして扱います。この際、エラー詳細はコールバック内で報告済みであると見なされます。

保守担当者向けメモ

  • 本構造体はイベント駆動型解析の要です。新しい解析アクションを追加する場合は、jsonapi.c 側のパーサーロジックと整合性を取る必要があります。
  • メモリ管理については、JsonLexContext の setJsonLexContextOwnsTokens 設定と組み合わせて、トークンの所有権を適切に管理してください。
