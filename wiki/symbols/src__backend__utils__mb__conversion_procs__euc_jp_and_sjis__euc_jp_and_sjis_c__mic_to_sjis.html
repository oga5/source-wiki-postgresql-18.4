mic_to_sjis

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_jp_and_sjis/euc_jp_and_sjis.c:143

目的

MULE_INTERNAL (MIC) エンコーディングから SJIS エンコーディングへの文字列変換を行うための PostgreSQL 関数マネージャインターフェースです。

シグネチャまたは形状

Datum mic_to_sjis(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を取得します。
  • src: 変換元文字列（CSTRING）
  • dest: 変換先バッファ（CSTRING）
  • len: 変換対象の長さ（INT32）
  • noError: エラーハンドリングフラグ（BOOL）
  • **出力**: 変換されたバイト数（INT32）を PG_RETURN_INT32 で返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_MULE_INTERNAL, PG_SJIS) により、ソースエンコーディングが PG_MULE_INTERNAL、ターゲットエンコーディングが PG_SJIS であることが検証されます。

アルゴリズムまたは振る舞い

1. PG_FUNCTION_ARGS から引数を抽出します。 2. エンコーディングの整合性を検証します。 3. 内部関数 mic2sjis を呼び出し、実際の変換処理を実行します。 4. mic2sjis が返した変換済みバイト数を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 マクロによる登録経由）。
  • **呼び出し先**: mic2sjis（内部変換ロジック）。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQL の関数引数構造体。
  • PG_MULE_INTERNAL, PG_SJIS: エンコーディング識別子。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、PostgreSQL のエラーハンドリング機構により処理が中断されます。
  • 変換不能文字や不正なバイト列に対する挙動は、引数 noError の値に基づき mic2sjis 内で制御されます。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の変換ロジックは mic2sjis に依存しています。変換ロジックの修正が必要な場合は mic2sjis を参照してください。
