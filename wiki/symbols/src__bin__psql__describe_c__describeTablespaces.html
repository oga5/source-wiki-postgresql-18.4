describeTablespaces

定義

  • 種別: function
  • 定義箇所: src/bin/psql/describe.c:221

目的

データベース内のテーブルスペース一覧をシステムカタログから取得し、整形して表示する。

シグネチャまたは形状

bool describeTablespaces(const char *pattern, bool verbose)

入力と出力

  • **入力**:
  • pattern: フィルタリング対象のテーブルスペース名パターン（NULL の場合は全件）。
  • verbose: 詳細表示（ACL、オプション、サイズ、説明）を有効にするフラグ。
  • **出力**:
  • bool: 処理が成功した場合は true、失敗した場合は false を返す。

前提条件と不変条件

  • pset グローバル変数が初期化されており、データベース接続が確立されていること。
  • pattern が指定された場合、validateSQLNamePattern により SQL インジェクションを防ぐための検証が行われる。

アルゴリズムまたは振る舞い

1. PQExpBuffer を初期化し、pg_catalog.pg_tablespace を対象とした SELECT 文を構築する。 2. verbose が true の場合、ACL、オプション、サイズ、説明を取得するカラムを追加する。 3. validateSQLNamePattern を呼び出し、pattern に基づく WHERE 句を構築する。 4. PSQLexec を実行してクエリ結果を取得する。 5. printQuery を使用して、結果を pset.queryFout に整形出力する。 6. 使用したバッファと結果セットを解放し、終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c:1099（\db コマンド実行時）。
  • **呼び出し先**:
  • initPQExpBuffer, printfPQExpBuffer, appendPQExpBuffer, appendPQExpBufferStr, termPQExpBuffer
  • printACLColumn
  • validateSQLNamePattern
  • PSQLexec
  • printQuery
  • PQclear

関連データ構造

  • PQExpBufferData: SQL クエリ構築用バッファ。
  • PGresult: クエリ実行結果の保持。
  • printQueryOpt: 出力オプション（pset.popt を継承）。

失敗モードと境界条件

  • validateSQLNamePattern が失敗した場合、バッファを解放して false を返す。
  • PSQLexec が NULL を返した場合（クエリ実行失敗）、false を返す。
  • データベース接続が切断されている場合、PSQLexec 経由でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は \db メタコマンドのバックエンド実装である。
  • 新しいテーブルスペース属性を追加する場合は、verbose 分岐内のクエリと printQuery の設定を更新すること。
