generate_old_dump

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/dump.c:15

目的

旧データベースクラスタからスキーマ定義およびグローバルオブジェクトを抽出し、アップグレード用のダンプファイルを作成する。

シグネチャまたは形状

void generate_old_dump(void)

入力と出力

  • **入力**: 引数なし。グローバル変数 old_cluster（クラスタ情報）、new_cluster（バイナリパス）、log_opts（ログ設定）、user_opts（ユーザー設定）を参照する。
  • **出力**: なし（副作用としてファイルシステム上にダンプファイルを出力する）。

前提条件と不変条件

  • old_cluster が初期化され、接続可能な状態であること。
  • new_cluster.bindir に適切なバージョンの pg_dumpall および pg_dump が存在すること。
  • 実行ユーザーがダンプ出力先ディレクトリへの書き込み権限を持つこと。

アルゴリズムまたは振る舞い

1. **グローバルオブジェクトのダンプ**: pg_dumpall --globals-only を同期的に実行し、ロールやテーブルスペース情報を保存する。 2. **スキーマダンプのループ**: old_cluster.dbarr 内の全データベースに対し、以下の処理を繰り返す。

  • データベースごとの接続文字列を構築（必要に応じてプロトコルバージョンを調整）。
  • pg_dump を --binary-upgrade モードで呼び出し、並列実行キューに投入する。

    • 3. **並列実行の待機**: reap_child() を呼び出し、すべての pg_dump プロセスが完了するまで待機する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/check.c:702
  • **呼び出し先**:
  • exec_prog (同期実行)
  • parallel_exec_prog (非同期実行)
  • reap_child (プロセス終了待機)
  • initPQExpBuffer, appendPQExpBufferStr, appendConnStrVal, appendShellString, termPQExpBuffer (接続文字列構築)
  • prep_status, check_ok, pg_log (ログ出力)

関連データ構造

  • ClusterInfo (グローバル変数 old_cluster, new_cluster の型)
  • DbInfo (データベース情報)
  • PQExpBufferData (接続文字列バッファ)

失敗モードと境界条件

  • 外部コマンド（pg_dump 等）の実行失敗時に exec_prog または parallel_exec_prog がエラーを報告する。
  • プロトコルバージョン 3.0 未満の古いクラスタに対しては、接続オプションに max_protocol_version=3.0 を付与して互換性を維持する。
  • user_opts.transfer_mode が TRANSFER_MODE_SWAP でない場合、--sequence-data オプションが追加される。

保守担当者向けメモ

  • 本関数は --binary-upgrade フラグを使用してシステムカタログを直接出力する特殊なダンプを行う。通常のバックアップ用途には使用できない。
  • 接続文字列の構築には appendShellString を使用しており、シェルインジェクション対策が施されている。変更時はこのエスケープ処理を維持すること。
