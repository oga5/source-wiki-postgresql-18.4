IndexAMProperty

定義

  • 種別: enum
  • 定義箇所: src/include/access/amapi.h:36

目的

インデックス・アクセスメソッド（AM）が提供する能力や特性を識別するための列挙型です。システムカタログやプランナが、特定のインデックスが特定の操作（スキャンタイプ、順序付け、列の特性など）をサポートしているかを問い合わせる際に使用されます。

シグネチャまたは形状

typedef enum IndexAMProperty
{
    AMPROP_UNKNOWN = 0,
    AMPROP_ASC,
    AMPROP_DESC,
    AMPROP_NULLS_FIRST,
    AMPROP_NULLS_LAST,
    AMPROP_ORDERABLE,
    AMPROP_DISTANCE_ORDERABLE,
    AMPROP_RETURNABLE,
    AMPROP_SEARCH_ARRAY,
    AMPROP_SEARCH_NULLS,
    AMPROP_CLUSTERABLE,
    AMPROP_INDEX_SCAN,
    AMPROP_BITMAP_SCAN,
    AMPROP_BACKWARD_SCAN,
    AMPROP_CAN_ORDER,
    AMPROP_CAN_UNIQUE,
    AMPROP_CAN_MULTI_COL,
    AMPROP_CAN_EXCLUDE,
    AMPROP_CAN_INCLUDE,
} IndexAMProperty;

入力と出力

本シンボルは列挙値であり、関数への引数として渡されることで、対象のインデックスが持つ特定のプロパティの有無を判定するフラグとして機能します。

前提条件と不変条件

  • AMPROP_UNKNOWN はコアコードで認識されないプロパティを示します。
  • 各列挙値は、インデックスの列特性、インデックス全体の特性、またはAM自体の特性のいずれかに分類されます。

アルゴリズムまたは振る舞い

pg_index_column_has_property 等の関数において、この列挙値を引数として渡すことで、インデックスのメタデータに基づいた真偽値の判定が行われます。

関連データ構造

  • IndexAmRoutine: インデックスAMの全機能へのエントリポイントを保持する構造体。本プロパティと組み合わせて使用されます。

失敗モードと境界条件

  • 未定義のプロパティを渡した場合、または特定のAMがサポートしていないプロパティを問い合わせた場合、AMPROP_UNKNOWN または適切なエラーハンドリングが実行されます。

保守担当者向けメモ

  • 新しいインデックスAM機能を追加する際は、本列挙型に新しいプロパティを追加するだけでなく、src/backend/utils/adt/amutils.c 等の関連する判定ロジックを更新する必要があります。
