gbt_bool_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bool.c:104

目的

GiSTインデックスの検索時、指定されたクエリ条件とインデックスキーが一致するかを判定します。

シグネチャまたは形状

Datum gbt_bool_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY 構造体（インデックスエントリ）。
  • PG_GETARG_INT16(1): 比較対象のブール値クエリ。
  • PG_GETARG_UINT16(2): StrategyNumber（比較演算の種類）。
  • PG_GETARG_POINTER(4): recheck フラグへのポインタ。
  • **出力**:
  • Datum 型のブール値（一致判定結果）。
  • *recheck: 常に false が設定されます（本関数による判定は完全であるため）。

前提条件と不変条件

  • entry->key は boolKEY 構造体として解釈可能であること。
  • 内部で利用される tinfo（gbtree_ninfo 構造体）が適切に初期化されていること。

アルゴリズムまたは振る舞い

1. entry->key から boolKEY を取得し、GBT_NUMKEY_R 構造体に変換します。 2. *recheck を false に設定します。 3. gbt_num_consistent を呼び出し、tinfo に定義された比較ロジックを用いてクエリとキーを比較します。 4. 判定結果を PG_RETURN_BOOL で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのGiSTインデックスエンジン（直接的な呼び出し元コードの詳細はソース抜粋に含まれていません）。
  • **呼び出し先**: gbt_num_consistent（contrib/btree_gist/btree_utils_num.c で定義）。

関連データ構造

  • boolKEY: インデックス内でブール値の範囲を保持する構造体。
  • GISTENTRY: GiSTインデックスのエントリ情報。
  • GBT_NUMKEY_R: 数値範囲を表現するための内部用構造体。

失敗モードと境界条件

  • 本関数は常に *recheck = false を返すため、インデックス検索後の再チェックは不要です。
  • 比較ロジックは tinfo に依存しており、tinfo が正しく設定されていない場合は予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.h の共通ユーティリティを流用する薄いラッパーです。
  • bool 型を数値（0/1）として扱う前提で実装されているため、比較ロジックの変更が必要な場合は tinfo の定義を確認してください。
