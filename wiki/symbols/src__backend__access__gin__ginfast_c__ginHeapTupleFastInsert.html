ginHeapTupleFastInsert

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/gin/ginfast.c:218

目的

GINインデックスの「ペンディングリスト（Pending List）」に対し、タプルを効率的に挿入する。インデックスへの直接挿入負荷を避けるため、データを一時的なページに蓄積する。

シグネチャまたは形状

void ginHeapTupleFastInsert(GinState *ginstate, GinTupleCollector *collector)

入力と出力

  • **入力**:
  • ginstate: GINインデックスの状態管理構造体。
  • collector: 挿入対象のタプル群とサイズ情報を保持する構造体。
  • **出力**: なし（副作用としてインデックスのペンディングリストを更新）。

前提条件と不変条件

  • collector->ntuples が 0 の場合、処理は即座に終了する。
  • ペンディングリストの操作は、メタページ（GIN_METAPAGE_BLKNO）の排他ロック下で行われる。
  • シリアライザブルなスキャンとの競合を避けるため、メタページへのアクセス時に CheckForSerializableConflictIn が呼び出される。

アルゴリズムまたは振る舞い

1. **挿入戦略の決定**:

  • 挿入データが単一ページに収まらない、または現在のテールページに空きがない場合、makeSublist を用いて新しいサブリストを作成する。
  • 収まる場合は、既存のテールページへ直接追記する。

    • 2. **サブリスト作成時**:

  • 新規サブリストを作成し、既存リストの末尾（metadata->tail）にリンクする。
  • メタページの head, tail, nPendingPages, nPendingHeapTuples を更新する。

    • 3. **テールページ直接挿入時**:

  • 現在のテールページをロックし、PageAddItem を用いてタプルを順次追加する。
  • メタページの nPendingHeapTuples およびページ内の maxoff をインクリメントする。

    • 4. **WAL記録**:

  • RelationNeedsWAL が真の場合、XLogBeginInsert を呼び出し、メタページやデータページの変更をWALに記録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gininsert.c:890
  • **呼び出し先**:
  • makeSublist (サブリスト作成時)
  • ReadBuffer, LockBuffer, PageAddItem (ストレージ操作)
  • XLogBeginInsert, XLogRegisterBuffer (WAL記録)
  • CheckForSerializableConflictIn (シリアライザブル整合性チェック)

関連データ構造

  • GinState: GINインデックスの操作状態。
  • GinTupleCollector: 挿入待ちタプルのバッファ。
  • GinMetaPageData: ペンディングリストのヘッド/テールを管理するメタページ構造体。

失敗モードと境界条件

  • PageAddItem が InvalidOffsetNumber を返した場合、elog(ERROR) により処理を中断する。
  • ペンディングリストのサイズが GinListPageSize を超える場合、強制的にサブリスト化が行われる。

保守担当者向けメモ

  • 本関数はペンディングリストの整合性を維持するため、クリティカルセクション（START_CRIT_SECTION）内でメタページおよびデータページの更新を行う。
  • 高い並行性を維持するため、サブリスト作成の準備段階ではメタページのロックを一時的に解放する設計となっている。
