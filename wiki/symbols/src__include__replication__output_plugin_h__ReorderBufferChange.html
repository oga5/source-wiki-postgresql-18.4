ReorderBufferChange

定義

  • 種別: struct
  • 定義箇所: src/include/replication/output_plugin.h:61 (および replication/reorderbuffer.h:76)

目的

論理デコーディングにおいて、WAL（Write-Ahead Log）から抽出された個別のデータ変更内容（挿入、更新、削除など）を保持し、出力プラグインへ伝達するために使用されます。

シグネチャまたは形状

構造体の詳細なメンバ構成は本ソース抜粋には含まれていませんが、LogicalDecodeChangeCB コールバック関数の引数として渡されるポインタ型として定義されています。

入力と出力

  • **入力**: 論理デコーディングのコアエンジンがWALを解析し、トランザクション内の変更を特定した際に生成されます。
  • **出力**: LogicalDecodeChangeCB を通じて出力プラグインに渡され、プラグイン側で変換処理（JSON化やプロトコル変換など）が行われます。

前提条件と不変条件

  • LogicalDecodeChangeCB の引数として渡される際、有効な ReorderBufferTXN および Relation オブジェクトと関連付けられている必要があります。

アルゴリズムまたは振る舞い

論理デコーディングの実行フェーズにおいて、コアエンジンはトランザクション内の各変更を ReorderBufferChange 構造体に格納し、登録された出力プラグインのコールバック関数を呼び出します。プラグインはこの構造体を参照して、変更内容を外部へ出力します。

呼び出し元と呼び出し先

  • **呼び出し元（生成・利用）**: src/backend/replication/logical/decode.c 内のデコーディングロジックが本構造体を生成し、コールバックへ渡します。
  • **呼び出し先（利用）**: contrib/test_decoding/test_decoding.c などの出力プラグイン実装が、この構造体を受け取り、内容を解析・処理します。

関連データ構造

  • ReorderBufferTXN: 変更が属するトランザクション情報を保持します。
  • Relation: 変更対象のテーブル定義情報を提供します。
  • LogicalDecodingContext: デコーディングの実行コンテキストを保持します。

失敗モードと境界条件

  • 構造体メンバの具体的な境界条件については、本ソース抜粋からは判断できません。

保守担当者向けメモ

  • 本構造体の定義は replication/reorderbuffer.h と output_plugin.h の両方に存在します。変更を加える際は、両方の定義の整合性に注意してください。
  • 構造体の詳細なメンバについては replication/reorderbuffer.h を参照してください。
