gbt_uuid_union

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_uuid.c:155

目的

GiST インデックス構築時、複数の UUID キーを包含する最小の範囲（Union）を計算する。

シグネチャまたは形状

Datum gbt_uuid_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GistEntryVector *（結合対象のキー集合）。
  • PG_GETARG_POINTER(1): int *（出力サイズを格納するポインタ）。
  • **出力**:
  • Datum: 結合された uuidKEY 構造体へのポインタ。
  • 副作用: 第2引数に sizeof(uuidKEY) が書き込まれる。

前提条件と不変条件

  • tinfo（gbtree_ninfo 型の静的変数）が UUID 型の比較関数や型情報を保持していること。
  • palloc により uuidKEY サイズのメモリが確保可能であること。

アルゴリズムまたは振る舞い

1. palloc を使用して uuidKEY サイズのメモリ領域を確保する。 2. 第2引数に sizeof(uuidKEY) を設定する。 3. btree_utils_num.c で定義された汎用関数 gbt_num_union を呼び出し、entryvec 内の全キーを包含する最小範囲を計算して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: GiST インデックスの構築プロセス（PostgreSQL の GiST インターフェース経由）。
  • **呼び出し先**: gbt_num_union（contrib/btree_gist/btree_utils_num.c 内の汎用関数）。

関連データ構造

  • uuidKEY: インデックスノードに格納される UUID の範囲（lower/upper）を保持する構造体。
  • GistEntryVector: 結合対象の GiST エントリを保持する構造体。
  • tinfo: UUID 型のメタデータを含む gbtree_ninfo 構造体。

失敗モードと境界条件

  • メモリ不足時に palloc がエラーを発生させる可能性がある。
  • entryvec が空の場合の挙動は gbt_num_union の実装に依存する。

保守担当者向けメモ

  • 本関数は btree_utils_num.c の汎用ロジックをラップしているため、詳細な結合ロジック（範囲の拡大方法など）については gbt_num_union を参照すること。
  • PG_FUNCTION_INFO_V1(gbt_uuid_union) により、PostgreSQL の関数マネージャに登録されている。
