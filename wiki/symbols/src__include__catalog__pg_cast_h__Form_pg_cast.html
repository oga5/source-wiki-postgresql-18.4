Form_pg_cast

定義

  • 種別: typedef
  • 定義箇所: src/include/catalog/pg_cast.h:57

目的

システムカタログ pg_cast のタプル（行データ）を指すポインタ型を提供します。

シグネチャまたは形状

typedef FormData_pg_cast *Form_pg_cast;

FormData_pg_cast 構造体へのポインタとして定義されています。

入力と出力

  • **入力**: GETSTRUCT(tuple) マクロ等で取得したタプルデータ。
  • **出力**: FormData_pg_cast 構造体のメンバ（castsource, casttarget, castfunc, castcontext, castmethod 等）へのアクセス。

前提条件と不変条件

  • GETSTRUCT マクロを介して、有効な pg_cast システムカタログのタプルからキャストされる必要があります。
  • 構造体の各フィールドは、データベース内の pg_cast テーブルのスキーマ定義と一致している必要があります。

アルゴリズムまたは振る舞い

本シンボル自体は型定義であり、実行時の振る舞いは持ちません。主に pg_cast カタログの行データを C 言語の構造体として扱い、型変換ルール（変換元・変換先・変換関数・コンテキスト・手法）をプログラムから参照するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/catalog/objectaddress.c: GETSTRUCT の結果をキャストして使用。
  • src/backend/parser/parse_coerce.c: GETSTRUCT の結果をキャストして使用。
  • **呼び出し先**:
  • 本シンボルは型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • FormData_pg_cast: pg_cast テーブルの実際の行構造体。
  • CoercionCodes: キャストコンテキスト（暗黙的、代入、明示的）を定義する列挙型。
  • CoercionMethod: キャスト手法（関数、バイナリ互換、入出力）を定義する列挙型。

失敗モードと境界条件

  • GETSTRUCT に渡されるタプルが pg_cast 以外のカタログのものである場合、フィールド参照時にメモリレイアウトの不整合が発生します。
  • NULL ポインタをキャストして参照した場合、セグメンテーション違反が発生します。

保守担当者向けメモ

  • FormData_pg_cast の定義が変更された場合、このポインタ型を通じてアクセスしているすべての箇所で整合性を確認する必要があります。
  • Catalog.pm によるカタログ生成プロセスに依存しているため、構造体の変更時は関連する BKI 定義との同期に注意してください。
