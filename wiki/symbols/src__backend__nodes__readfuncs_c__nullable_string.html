nullable_string

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/readfuncs.c:187

目的

シリアライズされた文字列から、PostgreSQL のノードフィールドとして文字列を復元する。特に outfuncs.c が出力した特殊な表現（NULL、空文字列、エスケープされた文字列）を元の形式に戻す役割を持つ。

シグネチャまたは形状

static char *nullable_string(const char *token, int length)

入力と出力

  • **入力**:
  • token: pg_strtok によって抽出されたトークン文字列。
  • length: トークンの長さ。
  • **出力**:
  • 復元された文字列（char *）。NULL の場合は NULL を返す。

前提条件と不変条件

  • token は pg_strtok が生成した有効な文字列であること。
  • outToken が出力した形式（<>、""、またはエスケープされた文字列）に従っていることを前提とする。

アルゴリズムまたは振る舞い

1. **NULL 判定**: length が 0 の場合、NULL を返す（outToken が NULL を <> として出力し、pg_strtok がそれを空文字列として扱うため）。 2. **空文字列判定**: length が 2 で、かつ token が "" である場合、空文字列 "" を pstrdup して返す。 3. **通常文字列の復元**: 上記以外の場合、debackslash 関数を呼び出し、outToken によって付与された保護用のバックスラッシュを除去して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/readfuncs.c:124
  • src/backend/nodes/readfuncs.c:548
  • **呼び出し先**:
  • debackslash（文字列のエスケープ解除用）
  • pstrdup（メモリ確保用）

関連データ構造

  • Node 構造体の文字列フィールドのデシリアライズに使用される。

失敗モードと境界条件

  • length が 0 の場合は NULL を返すため、呼び出し側は戻り値が NULL になる可能性を考慮する必要がある。
  • debackslash の内部実装に依存しており、不正なエスケープシーケンスが含まれる場合の挙動は debackslash の仕様に準ずる。

保守担当者向けメモ

  • 本関数は outfuncs.c の出力形式と密接に結合している。シリアライズ形式を変更する場合は、必ず本関数のロジックと整合性を確認すること。
  • location フィールドの復元とは異なり、純粋な文字列データの復元に特化している。
