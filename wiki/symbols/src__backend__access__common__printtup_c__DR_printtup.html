DR_printtup

定義

  • 種別: struct
  • 定義箇所: src/backend/access/common/printtup.c:54

目的

DestReceiver インターフェースを継承し、クエリ結果のタプルをフロントエンドへ送信するためのプライベート状態を保持する構造体です。

シグネチャまたは形状

typedef struct {
    DestReceiver pub;           /* 公開されている関数ポインタ */
    Portal       portal;        /* 出力元のポータル */
    bool         sendDescrip;   /* 起動時に RowDescription を送信するか */
    TupleDesc    attrinfo;      /* 設定済みの属性情報 */
    int          nattrs;        /* 属性数 */
    PrinttupAttrInfo *myinfo;   /* 各属性のキャッシュ情報 */
    StringInfoData buf;         /* 出力バッファ（tmpcontext外） */
    MemoryContext tmpcontext;   /* 行ごとのワークスペース用メモリコンテキスト */
} DR_printtup;

入力と出力

  • **入力**: DestReceiver インターフェースを介したタプルデータ。
  • **出力**: StringInfoData buf を経由したフロントエンドへの DataRow メッセージ送信。

前提条件と不変条件

  • buf は tmpcontext とは独立したメモリ領域で管理される必要があります。
  • myinfo は printtup_prepare_info によって初期化され、タプル記述子が変更されない限り再利用されます。

アルゴリズムまたは振る舞い

1. printtup_create_DR によりメモリ確保と初期化が行われます。 2. printtup 呼び出し時、tmpcontext をリセットしてメモリを解放します。 3. slot_getallattrs でタプルをデコードし、キャッシュされた myinfo を用いて各属性を変換します。 4. 変換後のデータを buf に書き込み、pq_endmessage_reuse を用いて送信します。

呼び出し元と呼び出し先

  • **呼び出し元**: printtup_create_DR, printtup_startup, printtup, printtup_prepare_info, printtup_shutdown 等の printtup.c 内の関数。
  • **呼び出し先**: 構造体内の関数ポインタ（DestReceiver 経由）および printtup_prepare_info 等の補助関数。

関連データ構造

  • DestReceiver: 基底となるインターフェース構造体。
  • PrinttupAttrInfo: 各カラムの出力関数や型情報を保持するキャッシュ構造体。
  • StringInfoData: 送信メッセージ構築用のバッファ。

失敗モードと境界条件

  • メモリ不足時: palloc0 等のメモリ確保失敗によりエラーが発生する可能性があります。
  • 属性数不一致: nattrs と attrinfo の不整合は、タプル処理時のクラッシュやデータ破損を招く可能性があります。

保守担当者向けメモ

  • 行ごとのメモリ管理に tmpcontext を使用しているため、このコンテキスト内でのメモリリークには特に注意してください。
  • buf は永続的なバッファであるため、tmpcontext のリセットの影響を受けません。
