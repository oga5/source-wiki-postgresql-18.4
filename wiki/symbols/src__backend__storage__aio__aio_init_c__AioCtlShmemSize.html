AioCtlShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_init.c:30

目的

PostgreSQL の非同期 I/O (AIO) サブシステムにおいて、共有メモリ上に配置される制御構造体 PgAioCtl のサイズを算出します。

シグネチャまたは形状

static Size AioCtlShmemSize(void)

入力と出力

  • 入力: なし
  • 出力: PgAioCtl 構造体のサイズ（Size 型）を返します。

前提条件と不変条件

  • 本関数は、共有メモリのレイアウトを決定する初期化フェーズで呼び出されることを前提としています。
  • 返されるサイズは PgAioCtl 型のメモリ占有量と一致します。

アルゴリズムまたは振る舞い

sizeof(PgAioCtl) を評価し、その値をそのまま返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • AioShmemSize (src/backend/storage/aio/aio_init.c:135): 共有メモリ総サイズの計算時に呼び出されます。
  • AioShmemInit (src/backend/storage/aio/aio_init.c:157, 162): 共有メモリの確保および初期化（memset）時に呼び出されます。
  • 呼び出し先: なし（C 言語の sizeof 演算子のみを使用）。

関連データ構造

  • PgAioCtl: AIO サブシステムのメイン制御構造体。

失敗モードと境界条件

  • 本関数自体は計算のみを行うため、失敗モードはありません。

保守担当者向けメモ

  • PgAioCtl 構造体の定義が変更された場合、本関数は自動的に正しいサイズを返すため、個別の修正は不要です。
  • 共有メモリのレイアウト変更時には、AioShmemSize 内での加算順序や AioShmemInit での初期化ロジックと整合性が取れているか確認してください。
