pgstattuple_type

定義

  • 種別: struct
  • 定義箇所: contrib/pgstattuple/pgstattuple.c:57

目的

PostgreSQL のテーブルおよびインデックスの物理的な統計情報（タプル数、デッドタプル数、空き容量など）を保持し、SQL 関数を通じてユーザーへ返却するためのデータ構造です。

シグネチャまたは形状

typedef struct pgstattuple_type
{
	uint64		table_len;
	uint64		tuple_count;
	uint64		tuple_len;
	uint64		dead_tuple_count;
	uint64		dead_tuple_len;
	uint64		free_space;
} pgstattuple_type;

入力と出力

本構造体は直接的な入力・出力関数ではありませんが、build_pgstattuple_type() 関数を介して SQL の複合型（Datum）へ変換され、結果として出力されます。

前提条件と不変条件

  • 各フィールドは uint64 型であり、物理的なバイト数やカウント数を保持します。
  • free_space は、ページ内の再利用可能な空き容量（バイト単位）を指します。
  • 統計情報の集計処理中、対象リレーションに対して AccessShareLock が保持されている必要があります。

アルゴリズムまたは振る舞い

pgstat_heap() や pgstat_index() などの走査関数によって、リレーションの各ページが読み込まれるたびに、本構造体の各フィールドが加算・更新されます。最終的に build_pgstattuple_type() がこの構造体の値を基に HeapTuple を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: pgstattuple() 等の SQL 関数実装、および pgstat_heap()、pgstat_index()、pgstat_btree_page() 等の統計集計関数。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • pgstat_page (関数ポインタ型): 統計集計時に本構造体を引数として受け取るコールバック関数。

失敗モードと境界条件

  • 大規模なテーブルに対して実行した場合、全ページ走査に伴う高い I/O 負荷が発生します。
  • 走査処理中に CHECK_FOR_INTERRUPTS() が呼び出されるため、ユーザーによるクエリキャンセルが発生した場合は処理が中断されます。

保守担当者向けメモ

  • 本構造体のフィールド定義を変更する場合、対応する SQL 側の複合型定義および build_pgstattuple_type() 内の変換ロジックを同期させる必要があります。
