am_propname

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/amutils.c:25

目的

インデックス・アクセスメソッド（AM）のプロパティ名（文字列）と、それに対応する内部列挙型 IndexAMProperty を関連付けるためのマッピング構造体です。

シグネチャまたは形状

struct am_propname
{
	const char *name;
	IndexAMProperty prop;
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。

  • name: プロパティ名を表す文字列へのポインタ。
  • prop: 対応する IndexAMProperty 列挙値。

前提条件と不変条件

  • am_propnames 配列の要素として使用されることを前提としています。
  • 新しいプロパティを追加する場合は、IndexAMProperty 列挙型と am_propnames 配列の両方を更新する必要があります。

アルゴリズムまたは振る舞い

本構造体自体はロジックを持ちません。この構造体の配列（am_propnames）は、SQL レベルで指定されたプロパティ名を内部的な列挙型に変換する際の検索テーブルとして利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/amutils.c:31 にて am_propnames 配列の定義の一部として参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • IndexAMProperty: プロパティの種類を定義する列挙型。
  • am_propnames: am_propname 型の静的配列。

失敗モードと境界条件

  • 構造体定義自体に失敗モードはありません。
  • 検索時に一致する name が存在しない場合、呼び出し側のロジックでエラーハンドリングを行う必要があります。

保守担当者向けメモ

  • 本構造体は、インデックスの特性を動的に照会する indexam_property 関連の機能を支える基盤です。
  • 構造体の変更時は、関連する am_propnames 配列の整合性を確認してください。
