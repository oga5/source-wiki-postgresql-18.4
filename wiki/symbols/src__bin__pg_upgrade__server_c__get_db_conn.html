get_db_conn

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/server.c:56

目的

pg_upgrade において、指定されたデータベースクラスターへの接続文字列を構築し、libpq を介してデータベース接続を確立します。

シグネチャまたは形状

static PGconn *get_db_conn(ClusterInfo *cluster, const char *db_name)

入力と出力

  • **入力**:
  • cluster: 接続先クラスターの情報を含む ClusterInfo 構造体へのポインタ。
  • db_name: 接続対象のデータベース名。
  • **出力**:
  • 確立された接続を表す PGconn *。接続失敗時は libpq の仕様に従い、接続状態が無効な PGconn オブジェクトが返される場合があります。

前提条件と不変条件

  • cluster 引数は有効なメモリ領域を指している必要があります。
  • os_info.user が適切に初期化されている必要があります。
  • 接続文字列の構築には PQExpBuffer を使用しており、関数終了時にバッファは解放されます。

アルゴリズムまたは振る舞い

1. initPQExpBuffer で接続文字列用バッファを初期化します。 2. dbname、user、port をバッファに追加します。 3. cluster->sockdir が存在する場合、host として追加します。 4. protocol_negotiation_supported(cluster) が偽の場合、互換性維持のため max_protocol_version=3.0 を付与します。 5. PQconnectdb を呼び出し、構築した文字列で接続を試みます。 6. termPQExpBuffer でバッファを破棄し、接続オブジェクトを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/server.c 内の複数箇所で、データベース接続の初期化に使用されています。
  • **呼び出し先**:
  • libpq 関数: PQconnectdb
  • fe_utils/string_utils.h 等のユーティリティ: initPQExpBuffer, appendPQExpBufferStr, appendConnStrVal, appendPQExpBuffer, termPQExpBuffer
  • protocol_negotiation_supported (定義箇所は本ソース外)

関連データ構造

  • ClusterInfo: 接続先クラスターのポート番号やソケットディレクトリ情報を提供します。
  • PGconn: libpq の接続ハンドル。
  • PQExpBufferData: 接続文字列構築用の一時バッファ。

失敗モードと境界条件

  • PQconnectdb が失敗した場合、関数は NULL ではなく、接続エラー状態を含む PGconn オブジェクトを返す可能性があります。呼び出し側で PQstatus(conn) != CONNECTION_OK の確認が必要です。
  • メモリ不足により PQExpBuffer の拡張が失敗した場合、プログラムが異常終了する可能性があります。

保守担当者向けメモ

  • 本関数は静的関数であり、server.c 内でのみ使用されます。
  • 接続オプションの追加が必要な場合は、appendPQExpBuffer を使用して適切にエスケープ処理を行ってください。
  • 呼び出し側で PQfinish(conn) を呼び出し、リソースを解放する責任があります。
