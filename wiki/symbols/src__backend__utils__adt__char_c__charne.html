charne

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/char.c:135

目的

PostgreSQLの内部型 "char"（1バイト文字型）の2つの値が等しくないかどうかを判定し、ブール値を返します。

シグネチャまたは形状

Datum charne(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの "char" 型の値を受け取ります。
  • **出力**: 2つの値が等しくない場合は true、等しい場合は false を Datum 型として返します。

前提条件と不変条件

  • 入力値は "char" 型（1バイト）として扱われます。
  • 比較演算は、符号なし8ビット整数 (uint8) として実行されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_CHAR マクロを使用して、引数から2つの文字値を取得します。 2. 取得した2つの値を比較し、それらが一致しないことを確認します。 3. PG_RETURN_BOOL マクロを使用して、比較結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL演算子インターフェースから呼び出されます（直接的な呼び出し元の詳細は提供されていません）。
  • **呼び出し先**: PG_GETARG_CHAR および PG_RETURN_BOOL マクロを使用します。

関連データ構造

  • "char": PostgreSQLの1バイト固定長データ型。

失敗モードと境界条件

  • 本関数自体は単純な比較であるため、特有の失敗モードはありません。
  • 比較は uint8 として行われますが、int4 との変換ロジックとは符号の扱いが異なる場合があるため、型変換を伴う操作との整合性に注意が必要です。

保守担当者向けメモ

  • 本関数は SQL の != 演算子に対応しています。
  • "char" 型は SQL 標準の CHAR(n) とは異なる型であるため、混同を避けてください。
