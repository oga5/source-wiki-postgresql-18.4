typinfo

定義

  • **種別**: struct
  • **定義箇所**: src/backend/bootstrap/bootstrap.c:73

目的

PostgreSQLのブートストラッププロセスにおいて、システムカタログ pg_type が構築される以前の段階で、データ型の属性（サイズ、アライメント、入出力関数等）を保持・管理するために使用される構造体です。

シグネチャまたは形状

struct typinfo
{
	char		name[NAMEDATALEN];
	Oid			oid;
	Oid			elem;
	int16		len;
	bool		byval;
	char		align;
	char		storage;
	Oid			collation;
	Oid			inproc;
	Oid			outproc;
};

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。入力データは TypInfo 配列の初期化時に与えられ、出力はブートストラップ中の型解決処理において参照されます。

前提条件と不変条件

  • name フィールドは NAMEDATALEN バイトの固定長配列です。
  • ブートストラップの初期段階において、システムカタログが未構築の状態でも型情報を解決できるよう、ハードコードされた定義として存在します。

アルゴリズムまたは振る舞い

  • TypInfo 配列の要素として定義され、ブートストラップ中に型名から OID や属性を検索する際のキャッシュまたは参照元として機能します。
  • gettype 関数等の内部処理において、型情報の解決に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/bootstrap/bootstrap.c:87: TypInfo 配列の定義にて使用。
  • src/backend/bootstrap/bootstrap.c:140: n_types の計算（配列サイズ算出）にて使用。
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • TypInfo: typinfo 構造体の配列。ブートストラップに必要な基本型が格納されています。

失敗モードと境界条件

  • TypInfo 配列に含まれない型を解決しようとした場合、ブートストラップ処理が失敗する可能性があります。
  • NAMEDATALEN を超える型名は切り詰められるか、定義時に制約を受けます。

保守担当者向けメモ

  • pg_type が構築された後は、この構造体によるハードコードされた情報よりもカタログ上の情報が優先される場合があります。gettype 関数の挙動変更時には、この構造体とカタログの整合性に注意してください。
