ProcSignalShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/procsignal.c:131

目的

プロセス間シグナル通信（ProcSignal）に必要な共有メモリ領域を割り当て、初期化する。

シグネチャまたは形状

void ProcSignalShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上の ProcSignal グローバル変数が初期化される）

前提条件と不変条件

  • 共有メモリの初期化プロセス中に呼び出される必要がある。
  • ProcSignalShmemSize() が返すサイズが、共有メモリ確保に十分であること。

アルゴリズムまたは振る舞い

1. ProcSignalShmemSize() を呼び出し、必要な共有メモリサイズを取得する。 2. ShmemInitStruct を使用して "ProcSignal" という名前の共有メモリ領域を確保または取得する。 3. 既に領域が初期化されている場合（found が真）は何も行わない。 4. 初回呼び出しの場合（found が偽）:

  • グローバルなバリア生成番号 psh_barrierGeneration を 0 に初期化する。
  • NumProcSignalSlots 分の各 ProcSignalSlot に対して以下を実行する:
  • スピンロック (pss_mutex) の初期化。
  • PID を 0 に設定。
  • シグナルフラグ配列のクリア。
  • バリア生成番号を PG_UINT64_MAX に初期化。
  • バリアチェックマスクを 0 に初期化。
  • 条件変数 (pss_barrierCV) の初期化。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:326 (PostgreSQLの共有メモリ初期化シーケンスの一部)
  • 呼び出し先: ShmemInitStruct, pg_atomic_init_u64, SpinLockInit, pg_atomic_init_u32, MemSet, ConditionVariableInit

関連データ構造

  • ProcSignalHeader: 共有メモリ上の管理ヘッダー。
  • ProcSignalSlot: 各プロセスのシグナル状態を保持する構造体。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、ShmemInitStruct 内部でエラーが発生し、プロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数はシステム起動時に一度だけ実行されることを想定している。
  • ProcSignalSlot の初期化順序や各フィールドの初期値は、シグナル伝達の正当性に直結するため変更時は注意が必要である。
