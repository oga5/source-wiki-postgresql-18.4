anl_random_fract

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/sampling.c:265

目的

レガシーなサンプリング処理において、0から1の範囲（0を含まない）の一様な乱数を生成する。

シグネチャまたは形状

double anl_random_fract(void)

入力と出力

  • 入力: なし
  • 出力: double 型の乱数値。範囲は (0.0, 1.0) である。

前提条件と不変条件

  • 内部状態 oldrs が初回呼び出し時に初期化される。
  • sampler_random_fract の仕様により、戻り値は常に 0.0 を除外する。

アルゴリズムまたは振る舞い

1. 静的な初期化フラグ oldrs_initialized を確認する。 2. 初回呼び出しの場合、pg_global_prng_state から取得した値を用いて oldrs.randstate を初期化し、フラグを立てる。 3. oldrs.randstate を使用して sampler_random_fract を呼び出し、その結果を返す。

呼び出し元と呼び出し先

  • 呼び出し元: 外部モジュール（src/include/commands/vacuum.h および src/include/utils/sampling.h で宣言されており、統計情報収集等のレガシーなサンプリング処理から参照される）。
  • 呼び出し先: sampler_random_init_state（初期化時）、sampler_random_fract（乱数生成時）、pg_prng_uint32（初期化時の乱数シード取得）。

関連データ構造

  • oldrs: anl_random_fract が内部的に保持する静的な乱数生成器の状態構造体。

失敗モードと境界条件

  • 乱数生成器の初期化は初回のみ行われるため、マルチスレッド環境や状態のリセットが必要な状況下での挙動には注意が必要である。
  • 戻り値が 0.0 になることはないため、対数計算の引数として安全に使用できる。

保守担当者向けメモ

  • 本関数はレガシーなAPIである。新規実装では BlockSampler や ReservoirState を使用する新しいAPIを優先すること。
