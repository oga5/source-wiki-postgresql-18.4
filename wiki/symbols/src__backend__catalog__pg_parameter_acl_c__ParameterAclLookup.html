ParameterAclLookup

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_parameter_acl.c:34

目的

指定された設定パラメータ（GUC）名に対応する pg_parameter_acl カタログ内のOIDを検索します。

シグネチャまたは形状

Oid ParameterAclLookup(const char *parameter, bool missing_ok)

入力と出力

  • **入力**:
  • parameter (const char *): 検索対象のパラメータ名。
  • missing_ok (bool): true の場合、パラメータが見つからなくてもエラーを発生させず InvalidOid を返します。
  • **出力**:
  • Oid: 見つかったパラメータのOID。存在しない場合は InvalidOid。

前提条件と不変条件

  • parameter は convert_GUC_name_for_parameter_acl によってカタログ格納用の形式に正規化されます。
  • pg_parameter_acl は共有カタログ（BKI_SHARED_RELATION）であるため、マルチテナント環境や共有メモリの整合性に留意する必要があります。

アルゴリズムまたは振る舞い

1. convert_GUC_name_for_parameter_acl を呼び出し、入力名をカタログ格納用の形式に変換します。 2. GetSysCacheOid1 を使用し、PARAMETERACLNAME システムキャッシュから Anum_pg_parameter_acl_oid をキーとしてOIDを検索します。 3. OIDが見つからず、かつ missing_ok が false の場合、ERRCODE_UNDEFINED_OBJECT エラーを発生させます。 4. 変換に使用したメモリ領域を pfree で解放し、OIDを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/aclchk.c:760
  • src/backend/catalog/objectaddress.c:1310
  • **呼び出し先**:
  • convert_GUC_name_for_parameter_acl (内部関数)
  • GetSysCacheOid1 (システムキャッシュAPI)
  • cstring_to_text (データ型変換)
  • ereport (エラー報告)
  • pfree (メモリ解放)

関連データ構造

  • pg_parameter_acl: 本関数が検索対象とするシステムカタログ。
  • FormData_pg_parameter_acl: カタログのスキーマ定義（src/include/catalog/pg_parameter_acl.h に定義）。

失敗モードと境界条件

  • パラメータが存在しない場合: missing_ok が false ならばエラー終了、true ならば InvalidOid を返します。

保守担当者向けメモ

  • 本関数はシステムキャッシュを利用して高速な検索を提供します。
  • pg_parameter_acl のスキーマ変更時には、GetSysCacheOid1 の引数やキャッシュIDの整合性を確認してください。
