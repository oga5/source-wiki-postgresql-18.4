changeDependencyOnOwner

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_shdepend.c:315

目的

オブジェクトの所有者が変更された際に、pg_shdepend カタログ内の共有依存関係を更新し、整合性を維持します。

シグネチャまたは形状

void changeDependencyOnOwner(Oid classId, Oid objectId, Oid newOwnerId)

入力と出力

  • classId: 変更対象オブジェクトのシステムカタログ OID。
  • objectId: 変更対象オブジェクトの OID。
  • newOwnerId: 新しい所有者のロール OID。
  • 出力: なし（副作用として pg_shdepend を更新）。

前提条件と不変条件

  • pg_shdepend カタログに対する RowExclusiveLock が必要です。
  • 所有者（Owner）は SHARED_DEPENDENCY_ACL エントリを持ってはならないという不変条件があります。

アルゴリズムまたは振る舞い

1. pg_shdepend を RowExclusiveLock で開きます。 2. shdepChangeDep を呼び出し、既存の SHARED_DEPENDENCY_OWNER エントリを新しい所有者情報で更新します。 3. shdepDropDependency を呼び出し、新しい所有者に対する SHARED_DEPENDENCY_ACL エントリが存在すれば削除します（所有者が権限を二重に保持することを防ぐため）。 4. SHARED_DEPENDENCY_INITACL エントリについては、所有者の変更に関わらず保持されるため、変更を加えません。 5. 最後に pg_shdepend を閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/alter.c
  • src/backend/commands/dbcommands.c
  • src/backend/commands/event_trigger.c
  • src/backend/commands/foreigncmds.c
  • src/backend/commands/publicationcmds.c
  • src/backend/commands/schemacmds.c
  • src/backend/commands/subscriptioncmds.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • table_open / table_close
  • shdepChangeDep
  • shdepDropDependency

関連データ構造

  • SharedDependRelationId: 操作対象のシステムカタログ。
  • SHARED_DEPENDENCY_OWNER, SHARED_DEPENDENCY_ACL, SHARED_DEPENDENCY_INITACL: 依存関係の種類を示す列挙値。

失敗モードと境界条件

  • 共有カタログのロック取得失敗や、カタログ更新時の制約違反が発生する可能性があります。
  • 呼び出し元で適切なトランザクションコンテキストが確保されていない場合、整合性が損なわれる可能性があります。

保守担当者向けメモ

  • 本関数は ALTER OWNER コマンドの実行に伴う依存関係の修正を一元管理します。
  • SHARED_DEPENDENCY_ACL の削除は、所有者変更時に発生しうる権限の重複を排除するための重要な処理です。
