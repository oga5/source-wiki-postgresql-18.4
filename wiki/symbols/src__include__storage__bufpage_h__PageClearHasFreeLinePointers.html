PageClearHasFreeLinePointers

定義

  • 種別: static inline function
  • 定義箇所: src/include/storage/bufpage.h:407

目的

ページヘッダーのフラグから PD_HAS_FREE_LINES ビットをクリアし、ページ内に再利用可能なラインポインタが存在しない（またはその状態を追跡しない）ことを示すために使用されます。

シグネチャまたは形状

static inline void PageClearHasFreeLinePointers(Page page)

入力と出力

  • **入力**: Page page（操作対象のページへのポインタ）
  • **出力**: なし（副作用として page が指すヘッダーの pd_flags を直接変更する）

前提条件と不変条件

  • page は有効な PageHeaderData を持つページを指している必要があります。
  • PD_HAS_FREE_LINES はWALログに記録されない「ヒントビット」として扱われるため、この操作は永続的な状態変更ではなく、メモリ上の最適化状態を更新するものです。

アルゴリズムまたは振る舞い

1. 引数 page を PageHeader 型にキャストします。 2. pd_flags フィールドに対してビット演算 &= ~PD_HAS_FREE_LINES を実行し、該当ビットを強制的にオフにします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/bufmask.c:55
  • src/backend/storage/page/bufpage.c:279
  • src/backend/storage/page/bufpage.c:811
  • src/backend/storage/page/bufpage.c:894
  • **呼び出し先**:
  • 本関数はインライン関数であり、他の関数を呼び出しません。

関連データ構造

  • PageHeaderData: ページ先頭のメタデータ構造体。pd_flags を保持します。
  • Page: char * として定義されるページへのポインタ型。

失敗モードと境界条件

  • page が NULL または不正なメモリ領域を指している場合、セグメンテーション違反が発生します。
  • ページヘッダーが初期化されていない状態で呼び出した場合、意図しないメモリ領域のフラグが書き換わる可能性があります。

保守担当者向けメモ

  • 本関数はヒントビットを操作する低レベルなユーティリティです。このフラグの正確性は保証されない場合があるため、フラグの状態のみに依存したクリティカルなロジックを組む際は注意してください。
