pg_lfind32_one_by_one_helper

定義

  • 種別: function
  • 定義箇所: src/include/port/pg_lfind.h:89

目的

SIMD命令が利用できない環境、または要素数が少ない場合に、uint32型の配列から指定された値を線形探索するためのフォールバック関数です。

シグネチャまたは形状

static inline bool pg_lfind32_one_by_one_helper(uint32 key, const uint32 *base, uint32 nelem)

入力と出力

  • **入力**:
  • key: 検索対象の uint32 値。
  • base: 検索対象となる uint32 配列の先頭ポインタ。
  • nelem: 配列内の要素数。
  • **出力**:
  • bool: 配列内に key が存在する場合は true、存在しない場合は false を返します。

前提条件と不変条件

  • base は少なくとも nelem 個の要素を持つ有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. 0 から nelem - 1 までのインデックスでループを実行します。 2. 各反復において、base[i] が key と一致するかを比較します。 3. 一致した時点で直ちに true を返します。 4. ループが終了しても一致が見つからない場合、false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/port/pg_lfind.h 内の pg_lfind32 実装において、SIMD処理のフォールバック先として、または USE_ASSERT_CHECKING 有効時の検証用として呼び出されます。
  • **呼び出し先**:
  • 本関数は標準的な比較演算のみを行い、他の関数を呼び出しません。

関連データ構造

  • uint32: 検索対象および配列要素の型。

失敗モードと境界条件

  • nelem が 0 の場合、ループは実行されず、常に false を返します。
  • メモリ境界外へのアクセスは発生しません（nelem の範囲内で安全に動作します）。

保守担当者向けメモ

  • 本関数は static inline で定義されており、コンパイル時に呼び出し元へ展開されることが想定されています。
  • SIMD最適化版（pg_lfind32_simd_helper）との結果整合性を保つため、ロジックの変更時は注意が必要です。
