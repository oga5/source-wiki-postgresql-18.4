pg_atomic_uint64

定義

  • 種別: struct
  • 定義箇所: src/include/port/atomics/generic-sunpro.h:54

目的

Oracle Solaris Studio (SunPro) コンパイラ環境において、アトミック操作の対象となる64ビット符号なし整数値を保持し、適切なメモリ配置を保証するための構造体です。

シグネチャまたは形状

typedef struct pg_atomic_uint64
{
    volatile uint64 value pg_attribute_aligned(8);
} pg_atomic_uint64;

入力と出力

  • **入力**: 構造体内の value フィールドに対し、PostgreSQL のアトミック操作関数（pg_atomic_compare_exchange_u64_impl 等）がポインタ経由でアクセスします。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • value フィールドは volatile 修飾されており、コンパイラによる最適化で値がキャッシュされることを防ぎます。
  • pg_attribute_aligned(8) により、8バイト境界へのアライメントが強制されます。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持に特化しており、具体的なアトミック操作（比較交換や交換など）は、generic-sunpro.h 内で定義された関数が本構造体の value フィールドを操作することで実行されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/transam/xlog.c: insertingAt, unloggedLSN, logInsertResult, logWriteResult, logFlushResult, xlblocks 等の変数定義およびメモリ割り当てで使用。
  • src/backend/access/transam/xlogprefetcher.c: reset_time, prefetch, hit 等の変数定義で使用。
  • **呼び出し先**: 構造体自体はデータ型であるため、直接的な呼び出し先はありません。

関連データ構造

  • pg_atomic_uint32: 32ビット版のアトミック操作用構造体。
  • src/include/port/atomics/generic.h: 本構造体が準拠する共通インターフェース定義。

失敗モードと境界条件

  • **アライメント**: pg_attribute_aligned(8) の挙動はコンパイラのバージョンに依存します。古い環境では厳密な検証が必要です。
  • **64ビットサポート**: コンパイラが64ビットアトミック操作をサポートしていない環境では、本構造体を利用した操作が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は Solaris 環境専用です。他プラットフォーム向けの実装については generic-gcc.h 等を参照してください。
  • Solaris の atomic_cas_64 等のシステム関数と組み合わせて使用されることを前提としています。
