XLTW_Oper

定義

  • 種別: enum
  • 定義箇所: src/include/storage/lmgr.h:24

目的

トランザクション待機時（XactLockTableWait 等）の操作種別を識別するために使用されます。デッドロック検出や待機時のエラーメッセージ生成において、どの処理が待機を引き起こしたかを特定する役割を持ちます。

シグネチャまたは形状

typedef enum XLTW_Oper
{
    XLTW_None,
    XLTW_Update,
    XLTW_Delete,
    XLTW_Lock,
    XLTW_LockUpdated,
    XLTW_InsertIndex,
    XLTW_InsertIndexUnique,
    XLTW_FetchUpdated,
    XLTW_RecheckExclusionConstr,
} XLTW_Oper;

入力と出力

  • 入力: 該当する操作の文脈に応じて、列挙子の一つが関数引数として渡されます。
  • 出力: 該当なし（列挙型定義）。

前提条件と不変条件

  • 待機処理を行う関数（XactLockTableWait など）において、現在の操作内容を正確に反映する値を選択する必要があります。

アルゴリズムまたは振る舞い

ロックマネージャが競合を検出し、プロセスが待機状態に入る際、この列挙値が待機理由として記録されます。これにより、システムログやデバッグ出力において、単なる「待機」ではなく「どの操作（例: XLTW_Update や XLTW_InsertIndex）による待機か」を区別可能にします。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/access/heap/heapam.c: 更新、削除、インデックス挿入処理などで使用。
  • src/backend/executor/execIndexing.c: インデックス処理の待機理由として使用。
  • src/backend/storage/lmgr/lmgr.c: ロックマネージャの内部処理で引数として使用。
  • 呼び出し先: 該当なし（列挙型のため）。

関連データ構造

  • LOCKTAG: ロック対象を識別する構造体。
  • LOCKMODE: ロックの強度を定義する列挙型。
  • XactLockTableWait: 本列挙型を引数として受け取り、待機処理を実行する関数。

失敗モードと境界条件

  • 適切な XLTW_Oper を指定しなかった場合、デッドロック発生時のエラーメッセージが不正確になり、デバッグが困難になる可能性があります。

保守担当者向けメモ

  • 新しいロック操作や待機シナリオを追加する際は、本列挙型に適切な値が定義されているか確認してください。
  • src/backend/storage/lmgr/lmgr.c の実装と照らし合わせ、エラーメッセージ生成ロジックが新しい列挙子を適切に処理しているか確認が必要です。
