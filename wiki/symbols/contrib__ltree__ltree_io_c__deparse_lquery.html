deparse_lquery

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_io.c:641

目的

内部バイナリ形式の lquery 構造体を、人間が読み取り可能な文字列形式（クエリパターン）に変換する。

シグネチャまたは形状

static char *deparse_lquery(const lquery *in)

入力と出力

  • **入力**: const lquery *in - 変換対象の lquery 構造体へのポインタ。
  • **出力**: char * - 変換後の文字列を格納したメモリ領域（palloc により確保）。

前提条件と不変条件

  • 入力 in は有効な lquery 構造体であること。
  • 戻り値の文字列は呼び出し元で適切に解放される必要がある。

アルゴリズムまたは振る舞い

1. **サイズ計算**: lquery 内の各レベル（lquery_level）とバリアント（lquery_variant）を走査し、出力文字列に必要な合計バイト数を算出する。 2. **メモリ確保**: palloc を使用して計算したサイズのバッファを確保する。 3. **文字列構築**:

  • 各レベルを . で連結する。
  • LQL_NOT フラグがある場合は ! を付与する。
  • 各バリアントを | で連結し、ラベル名および修飾子（%：部分一致、@：大文字小文字無視、*：前方一致）を書き込む。
  • LQL_COUNT フラグがある、またはレベルが空（numvar == 0）の場合、{low,high} 形式の範囲指定を付与する。

    • 4. **終端**: 文字列の末尾に \0 を付与して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/ltree_io.c 内の lquery_out 関数（PG_RETURN_POINTER 経由）。
  • contrib/ltree/ltree_io.c 内の他の内部処理。
  • **呼び出し先**:
  • palloc (PostgreSQL メモリ管理)
  • memcpy, sprintf, strchr (標準 C ライブラリ)

関連データ構造

  • lquery: 検索パターンを保持するメイン構造体。
  • lquery_level: 各階層の情報を保持。
  • lquery_variant: 各ラベルのバリアント情報を保持。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • LTREE_MAX_LEVELS に達する範囲指定は {,} などの特殊形式で出力される。
  • numvar == 0 の場合は * として出力される。

保守担当者向けメモ

  • lquery の複雑なメタ文字（|, !, *, {}）のシリアライズロジックを含んでいるため、構文の変更時には parse_lquery との整合性に注意すること。
  • sprintf を使用しているため、バッファサイズ計算（totallen）のロジックに漏れがないか厳密に確認が必要。
