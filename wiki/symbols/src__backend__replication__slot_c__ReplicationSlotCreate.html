ReplicationSlotCreate

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slot.c:352

目的

新しいレプリケーションスロットを割り当て、ディスク上に永続化し、共有メモリ上でアクティブ化する。

シグネチャまたは形状

void ReplicationSlotCreate(const char *name, bool db_specific,
                           ReplicationSlotPersistency persistency,
                           bool two_phase, bool failover, bool synced)

入力と出力

  • **入力**:
  • name: スロット名。
  • db_specific: データベース固有のスロットか否か。
  • persistency: 永続性（RS_TEMPORARY 等）。
  • two_phase, failover, synced: 論理レプリケーションおよびフェイルオーバー用のフラグ。
  • **出力**: なし（成功時は共有メモリおよびディスクに反映）。

前提条件と不変条件

  • MyReplicationSlot が NULL であること（既にスロットを所有していないこと）。
  • ReplicationSlotAllocationLock を排他モードで取得して実行する。
  • スロット名が有効であること（ReplicationSlotValidateName による検証）。

アルゴリズムまたは振る舞い

1. **検証**: スタンバイ側でのフェイルオーバー有効化や、一時スロットでのフェイルオーバー有効化など、許可されない組み合わせをエラーにする。 2. **排他制御**: ReplicationSlotAllocationLock を取得し、スロット作成中の競合を防ぐ。 3. **重複確認**: ReplicationSlotControlLock (共有) を取得し、既存スロットとの名前衝突を確認。空きスロットを検索する。 4. **初期化**: 共有メモリ上の ReplicationSlot 構造体をゼロクリアし、引数およびデフォルト値で初期化する。 5. **永続化**: CreateSlotOnDisk(slot) を呼び出し、ディスク上にディレクトリを作成する。 6. **アクティブ化**: ReplicationSlotControlLock (排他) を取得し、in_use を true に設定。active_pid を現在のプロセスIDに設定し、MyReplicationSlot に割り当てる。 7. **統計情報**: 論理スロットの場合は pgstat_create_replslot を呼び出す。 8. **解放**: ロックを解放し、ConditionVariableBroadcast で待機中のプロセスに通知する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/slotsync.c
  • src/backend/replication/slotfuncs.c
  • src/backend/replication/walsender.c
  • **呼び出し先**:
  • ReplicationSlotValidateName
  • CreateSlotOnDisk
  • pgstat_create_replslot
  • LWLockAcquire / LWLockRelease
  • SpinLockAcquire / SpinLockRelease
  • ConditionVariableBroadcast

関連データ構造

  • ReplicationSlot: スロットの共有メモリ状態を保持する構造体。
  • ReplicationSlotCtl: スロット配列を管理するグローバル制御構造体。
  • MyReplicationSlot: 現在のバックエンドが所有するスロットへのポインタ。

失敗モードと境界条件

  • **名前衝突**: 同名のスロットが既に存在する場合、ERRCODE_DUPLICATE_OBJECT でエラー。
  • **リソース枯渇**: max_replication_slots に達している場合、ERRCODE_CONFIGURATION_LIMIT_EXCEEDED でエラー。
  • **不正な構成**: スタンバイでのフェイルオーバー有効化や、一時スロットでのフェイルオーバー有効化は ERRCODE_FEATURE_NOT_SUPPORTED でエラー。

保守担当者向けメモ

  • 本関数は ReplicationSlotAllocationLock を保持したまま終了するため、呼び出し元でのロック管理に注意すること。
  • CreateSlotOnDisk が失敗した場合、スロットはまだ in_use になっていないため、特別なクリーンアップは不要である。
