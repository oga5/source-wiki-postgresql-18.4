extractModify

定義

  • 種別: function
  • 定義箇所: src/backend/commands/aggregatecmds.c:477

目的

CREATE AGGREGATE コマンドのパラメータとして指定された finalfunc_modify 属性の文字列値を、内部的な定数（AGGMODIFY_*）へ変換します。

シグネチャまたは形状

static char extractModify(DefElem *defel)

入力と出力

  • **入力**: DefElem *defel（SQL パーサーから渡される定義要素）。
  • **出力**: char 型の内部定数（AGGMODIFY_READ_ONLY, AGGMODIFY_SHAREABLE, または AGGMODIFY_READ_WRITE）。

前提条件と不変条件

  • 入力された defel は defGetString() によって文字列として取得可能である必要があります。

アルゴリズムまたは振る舞い

1. defGetString(defel) を呼び出し、パラメータの値を文字列として取得します。 2. 文字列値を比較し、以下のいずれかに該当すれば対応する定数を返します。

  • "read_only" → AGGMODIFY_READ_ONLY
  • "shareable" → AGGMODIFY_SHAREABLE
  • "read_write" → AGGMODIFY_READ_WRITE

    • 3. 上記のいずれにも一致しない場合、ereport(ERROR, ...) を呼び出し、構文エラーとして処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/aggregatecmds.c 内の DefineAggregate 関数（153行目および155行目）。
  • **呼び出し先**: defGetString（commands/defrem.h で定義）、ereport（PostgreSQL エラー報告メカニズム）。

関連データ構造

  • DefElem: SQL 定義要素を保持する構造体。
  • AGGMODIFY_*: 集約関数の変更属性を示す内部定数。

失敗モードと境界条件

  • 指定された文字列が "read_only", "shareable", "read_write" のいずれでもない場合、ERRCODE_SYNTAX_ERROR を伴うエラーが発生します。

保守担当者向けメモ

  • 新しい変更属性が追加された場合は、この関数の比較ロジックを更新する必要があります。
  • 戻り値の char 型は、カタログ上の属性値として直接利用されることを想定しています。
