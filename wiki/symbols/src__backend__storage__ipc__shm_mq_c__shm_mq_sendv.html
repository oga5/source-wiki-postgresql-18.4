shm_mq_sendv

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shm_mq.c:360

目的

共有メモリメッセージキューに対し、複数のデータ断片（shm_mq_iovec）を連結して送信する。

シグネチャまたは形状

shm_mq_result shm_mq_sendv(shm_mq_handle *mqh, shm_mq_iovec *iov, int iovcnt, bool nowait, bool force_flush)

入力と出力

  • **入力**:
  • mqh: キューへのアクセスハンドル。
  • iov: 送信するデータ断片の配列。
  • iovcnt: iov 配列の要素数。
  • nowait: true の場合、ブロッキングせずに SHM_MQ_WOULD_BLOCK を返す。
  • force_flush: true の場合、書き込み後に即座に共有メモリ上のポインタを更新し、受信側に通知する。
  • **出力**:
  • SHM_MQ_SUCCESS: 送信成功。
  • SHM_MQ_WOULD_BLOCK: バッファ不足により待機が必要。
  • SHM_MQ_DETACHED: 受信側がデタッチ済み。

前提条件と不変条件

  • 呼び出し元プロセスがキューの送信側（mq->mq_sender == MyProc）であること。
  • 送信メッセージの総サイズが MaxAllocSize を超えないこと（超える場合はエラー）。
  • mqh 内の mqh_partial_bytes および mqh_length_word_complete は、メッセージ送信の進捗を追跡するために使用される。

アルゴリズムまたは振る舞い

1. **メッセージ長書き込み**: メッセージ全体のサイズをキューの先頭ワードとして書き込む。完了するまで shm_mq_send_bytes を繰り返す。 2. **データ書き込み**:

  • 各 iov を順次処理する。
  • 中間チャンクが MAXIMUM_ALIGNOF に整列されていない場合、一時バッファを使用して後続のデータと結合し、整列を維持する。
  • 最終チャンク以外は MAXALIGN_DOWN でサイズを調整して書き込む。

    • 3. **状態リセット**: メッセージ送信完了後、mqh の進捗状態をリセットする。 4. **通知**: force_flush が指定されているか、未送信データ量がリングサイズの1/4を超えた場合、共有メモリを更新し受信側に通知する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/pqmq.c:164
  • src/backend/storage/ipc/shm_mq.c:337 (shm_mq_send からの呼び出し)
  • **呼び出し先**:
  • shm_mq_send_bytes: 実際の共有メモリへのコピーと同期を行う。
  • SpinLockAcquire/Release: 受信側プロセスの確認に使用。

関連データ構造

  • shm_mq_handle: キューのプライベート状態を保持。
  • shm_mq: 共有メモリ上のキュー本体。
  • shm_mq_iovec: 送信データの断片を定義する構造体。

失敗モードと境界条件

  • MaxAllocSize を超えるメッセージサイズは ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラー終了する。
  • 受信側がデタッチされた場合、SHM_MQ_DETACHED を返し、内部状態をリセットする。
  • nowait が true の場合、バッファが満杯であれば即座に SHM_MQ_WOULD_BLOCK を返す。

保守担当者向けメモ

  • 本関数は shm_mq_send の内部実装として利用されるほか、直接呼び出されることもある。
  • mqh_counterparty_attached フラグにより、受信側が一度でもアタッチしたことが確認できれば、スピンロックを省略して mq_receiver を参照する最適化が行われている。
