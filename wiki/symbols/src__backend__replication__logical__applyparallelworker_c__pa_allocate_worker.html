pa_allocate_worker

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/applyparallelworker.c:469

目的

論理レプリケーションのストリーミングトランザクションを並列適用するためのワーカーを確保し、トランザクションIDとワーカー情報のマッピングを管理する。

シグネチャまたは形状

void pa_allocate_worker(TransactionId xid)

入力と出力

  • **入力**: xid (適用対象のトランザクションID)
  • **出力**: なし（副作用として共有メモリ上のハッシュテーブルおよびワーカー状態を更新）

前提条件と不変条件

  • pa_can_start() が真を返すこと（ワーカー起動が許可されている状態であること）。
  • ParallelApplyTxnHash が未初期化の場合は、本関数内で初期化される。
  • 指定された xid が既にハッシュテーブルに存在する場合、ハッシュテーブルの破損とみなされエラーが発生する。

アルゴリズムまたは振る舞い

1. pa_can_start() を確認し、不可であれば即時終了する。 2. pa_launch_parallel_worker() を呼び出し、利用可能なワーカー情報 (ParallelApplyWorkerInfo) を取得する。取得失敗時は終了する。 3. ParallelApplyTxnHash が未作成であれば、ApplyContext をメモリコンテキストとしてハッシュテーブルを生成する。 4. xid をキーとしてハッシュテーブルにエントリを作成する。 5. 取得したワーカーの共有メモリ状態を更新し、トランザクション状態を PARALLEL_TRANS_UNKNOWN に設定する。 6. ワーカーの in_use フラグを立て、serialize_changes を無効化し、エントリにワーカー情報を格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/worker.c:1516
  • **呼び出し先**:
  • pa_can_start()
  • pa_launch_parallel_worker()
  • hash_create()
  • hash_search()
  • SpinLockAcquire() / SpinLockRelease()

関連データ構造

  • ParallelApplyWorkerInfo: ワーカーの実行状態を保持する構造体。
  • ParallelApplyWorkerEntry: トランザクションIDとワーカー情報のマッピングを保持するハッシュエントリ。
  • ParallelApplyTxnHash: トランザクションIDをキーとするハッシュテーブル。

失敗モードと境界条件

  • pa_can_start() が偽を返す場合、ワーカーは割り当てられない。
  • pa_launch_parallel_worker() が NULL を返す場合、ワーカーは割り当てられない。
  • 既に存在する xid に対して呼び出された場合、elog(ERROR) によりトランザクションが中断される。

保守担当者向けメモ

  • 本関数は、論理レプリケーションの並列適用におけるワーカーのライフサイクル管理の起点となる。
  • ハッシュテーブルの初期化は初回呼び出し時に行われるため、スレッドセーフな初期化が必要な環境下での動作に注意すること。
  • 共有メモリ上の xact_state 更新時にはスピンロックが必須である。
