CleanupPriorWALFiles

定義

  • 種別: function
  • 定義箇所: src/bin/pg_archivecleanup/pg_archivecleanup.c:91

目的

アーカイブディレクトリを走査し、指定された基準ファイル（exclusiveCleanupFileName）よりも古い WAL セグメントおよびバックアップ履歴ファイルを削除する。

シグネチャまたは形状

static void CleanupPriorWALFiles(void)

入力と出力

  • **入力**: グローバル変数 archiveLocation（ディレクトリパス）、exclusiveCleanupFileName（削除境界となるファイル名）、dryrun（削除のシミュレーションフラグ）、cleanBackupHistory（バックアップ履歴削除フラグ）。
  • **出力**: なし（成功時はディレクトリ内のファイルを削除、失敗時は pg_fatal によりプロセス終了）。

前提条件と不変条件

  • archiveLocation で指定されたディレクトリが読み取り可能であること。
  • exclusiveCleanupFileName が SetWALFileNameForCleanup によって適切に初期化されていること。
  • ファイル名の比較は、タイムライン部分（先頭8文字）を無視したアルファベット順の比較に基づいている。

アルゴリズムまたは振る舞い

1. opendir でアーカイブディレクトリを開く。 2. ディレクトリ内の各ファイルを順次読み込み、以下の処理を行う：

  • TrimExtension で拡張子を除去し、IsXLogFileName 等を用いて WAL ファイルまたはバックアップ履歴ファイルか判定する。
  • ファイル名（タイムライン以降）を exclusiveCleanupFileName と比較し、辞書順で小さい（古い）ものを特定する。
  • dryrun が真の場合はファイル名を表示してスキップする。
  • dryrun が偽の場合は unlink を実行してファイルを削除する。

    • 3. 走査完了後、closedir でディレクトリを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/bin/pg_archivecleanup/pg_archivecleanup.c:394）。
  • **呼び出し先**: opendir, readdir, closedir, TrimExtension, IsXLogFileName, IsPartialXLogFileName, IsBackupHistoryFileName, snprintf, printf, unlink, pg_log_debug, pg_fatal。

関連データ構造

  • DIR（ディレクトリストリーム）
  • struct dirent（ディレクトリ内のエントリ情報）

失敗モードと境界条件

  • ディレクトリが開けない、または読み取り中にエラーが発生した場合は pg_fatal で終了する。
  • unlink に失敗した場合（権限不足等）は pg_fatal で終了する。
  • タイムラインが異なる場合でも、ファイル名のアルファベット順序のみで削除判定を行うため、古いタイムラインのファイルが削除される可能性がある。

保守担当者向けメモ

  • 本関数はローカルファイルシステムを前提としている。ネットワークストレージ等への対応が必要な場合は、unlink やディレクトリ走査ロジックの修正を検討すること。
  • additional_ext が極端に長い場合、ファイル名の切り詰め処理に影響が出る可能性がある点に注意。
