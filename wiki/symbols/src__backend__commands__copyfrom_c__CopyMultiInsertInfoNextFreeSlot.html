CopyMultiInsertInfoNextFreeSlot

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfrom.c:734

目的

バルク挿入バッファ内の次に利用可能な TupleTableSlot を取得、または必要に応じて新規作成します。

シグネチャまたは形状

TupleTableSlot *CopyMultiInsertInfoNextFreeSlot(CopyMultiInsertInfo *miinfo, ResultRelInfo *rri)

入力と出力

  • **入力**:
  • miinfo: バルク挿入の状態を管理する構造体（本関数内では直接使用されず、将来的な拡張用）。
  • rri: ターゲットテーブルの情報を保持する ResultRelInfo。ri_CopyMultiInsertBuffer を介してバッファ状態を参照します。
  • **出力**:
  • 挿入に使用可能な TupleTableSlot へのポインタ。

前提条件と不変条件

  • rri->ri_CopyMultiInsertBuffer が NULL であってはならない。
  • バッファ内の使用済みスロット数 (buffer->nused) が MAX_BUFFERED_TUPLES 未満であること。

アルゴリズムまたは振る舞い

1. rri から CopyMultiInsertBuffer を取得します。 2. 現在のバッファ使用数 nused を取得します。 3. buffer->slots[nused] が NULL の場合、table_slot_create を呼び出して新しいスロットを生成し、バッファに格納します。 4. buffer->slots[nused] を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfrom.c 内の COPY 実行ループ（行データのバッファリング時）。
  • **呼び出し先**:
  • table_slot_create（テーブルアクセス層の関数）。

関連データ構造

  • CopyMultiInsertBuffer: バルク挿入用のスロット配列と使用状況を保持する構造体。
  • ResultRelInfo: ターゲットリレーションの実行時情報を保持する構造体。
  • TupleTableSlot: タプルを保持するためのメモリ領域。

失敗モードと境界条件

  • buffer が NULL の場合、アサーション違反により停止します。
  • nused が MAX_BUFFERED_TUPLES に達している場合、アサーション違反により停止します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスを重視して設計されています。
  • miinfo 引数は現在の実装では使用されていませんが、将来的に CopyMultiInsertInfo 全体の状態管理が必要になった場合に備えてシグネチャに含まれています。
