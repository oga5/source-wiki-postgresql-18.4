get_partition_ancestors

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/partition.c:133

目的

指定されたリレーション（パーティション）から階層を遡り、最上位の親テーブルに至るまでのすべての祖先リレーションのOIDリストを取得します。

シグネチャまたは形状

List *get_partition_ancestors(Oid relid)

入力と出力

  • **入力**: relid (Oid) - 祖先を検索する対象リレーションのOID。
  • **出力**: List * - 祖先リレーションのOIDを格納したリスト。リストは最上位の親から順に並びます。

前提条件と不変条件

  • InheritsRelationId（pg_inherits システムカタログ）へのアクセス権限が必要です。
  • 呼び出し側は、対象リレーションがパーティション階層の一部であることを前提とすることが推奨されます。

アルゴリズムまたは振る舞い

1. InheritsRelationId を AccessShareLock でオープンします。 2. 内部関数 get_partition_ancestors_worker を呼び出し、再帰的に親を辿って result リストを構築します。 3. table_close を呼び出し、ロックを解放します。 4. 構築された祖先リストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/index.c
  • src/backend/catalog/pg_depend.c
  • src/backend/catalog/pg_publication.c
  • src/backend/executor/execMain.c
  • src/backend/executor/execPartition.c
  • src/backend/replication/pgoutput/pgoutput.c
  • src/backend/utils/adt/partitionfuncs.c
  • **呼び出し先**:
  • table_open / table_close
  • get_partition_ancestors_worker (再帰的探索用)

関連データ構造

  • List: PostgreSQLの標準的なリスト構造。
  • InheritsRelationId: pg_inherits システムカタログのOID。

失敗モードと境界条件

  • 対象の relid がパーティション階層に属さない場合、空のリストが返されます。
  • システムカタログへのアクセス失敗やロック取得の競合が発生した場合、PostgreSQLの標準的なエラー処理メカニズムにより例外がスローされます。

保守担当者向けメモ

  • 本関数はカタログ操作に特化しています。実行時のパーティションルーティング処理については src/backend/partitioning/ を参照してください。
  • 祖先リストの構築は get_partition_ancestors_worker に依存しているため、階層探索ロジックの変更時は同関数も併せて確認してください。
