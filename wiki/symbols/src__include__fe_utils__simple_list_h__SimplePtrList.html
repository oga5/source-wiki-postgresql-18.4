SimplePtrList

定義

  • 種別: struct
  • 定義箇所: src/include/fe_utils/simple_list.h:52

目的

PostgreSQL のフロントエンドツールにおいて、汎用ポインタを格納・管理するための軽量な単方向連結リストの管理構造体です。

シグネチャまたは形状

typedef struct SimplePtrList
{
    SimplePtrListCell *head;
    SimplePtrListCell *tail;
} SimplePtrList;

入力と出力

  • **入力**: リスト操作関数（simple_ptr_list_append 等）に対して、リストのポインタおよび格納する汎用ポインタ（void *）を受け取ります。
  • **出力**: リストの先頭（head）および末尾（tail）のセルへのポインタを保持します。

前提条件と不変条件

  • リストの初期化時には、head および tail を NULL に設定する必要があります。
  • 本構造体はリストの管理のみを行い、保持するポインタが指す先のメモリ管理（解放等）は行いません。
  • メモリリークを防ぐため、リストの使用終了時には対応する破棄関数を呼び出す必要があります。

アルゴリズムまたは振る舞い

  • **追加**: simple_ptr_list_append を通じて、リストの末尾に新しいセルを動的に割り当てて連結します。
  • **破棄**: リストを走査し、各セルを pg_free で解放します。

呼び出し元と呼び出し先

  • **呼び出し元（確認されたもの）**:
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/pg_dump/pg_dump.c
  • src/bin/pg_dump/pg_dump.h
  • src/bin/pg_verifybackup/pg_verifybackup.c
  • **呼び出し先**:
  • src/fe_utils/simple_list.c（simple_ptr_list_append 等の操作関数が実装されています）

関連データ構造

  • SimplePtrListCell: リストの各要素を構成するセル構造体。

失敗モードと境界条件

  • メモリ不足時には pg_malloc 等の内部関数によりエラーが発生する可能性があります。
  • 空のリストに対して操作を行う場合、head および tail が NULL であることを前提とした安全な処理が必要です。

保守担当者向けメモ

  • バックエンドの List 構造体とは異なり、機能が最小限に限定されています。複雑なリスト操作が必要な場合は、他のデータ構造の利用を検討してください。
  • 保持しているポインタ先のメモリは管理対象外である点に注意してください。
