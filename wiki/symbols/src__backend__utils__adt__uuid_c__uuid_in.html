uuid_in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/uuid.c:77

目的

SQLの文字列型（cstring）からPostgreSQLの内部的な uuid 型（pg_uuid_t）への変換を行います。

シグネチャまたは形状

Datum uuid_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して取得される、UUIDの文字列表現を含むC文字列。
  • **出力**: pg_uuid_t 構造体へのポインタを Datum 型として返却します。

前提条件と不変条件

  • 入力文字列は、string_to_uuid 関数が許容する形式（標準的なUUIDの文字列表現）である必要があります。
  • palloc によるメモリ割り当てが成功することが前提です。

アルゴリズムまたは振る舞い

1. PG_GETARG_CSTRING(0) で入力文字列を取得します。 2. palloc を使用して、16バイトの pg_uuid_t 構造体を保持するためのメモリを確保します。 3. string_to_uuid を呼び出し、入力文字列を解析して確保したメモリ領域にバイナリ形式のUUIDを書き込みます。 4. PG_RETURN_UUID_P マクロを使用して、変換後の pg_uuid_t ポインタを Datum として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/uuid-ossp/uuid-ossp.c 内の各関数（DirectFunctionCall1 を介した呼び出し）。
  • **呼び出し先**:
  • string_to_uuid（文字列解析の実行）
  • palloc（メモリ確保）

関連データ構造

  • pg_uuid_t: UUIDを保持する16バイトの構造体。

失敗モードと境界条件

  • 入力文字列が不正な形式である場合、string_to_uuid 内でエラーが発生し、トランザクションがアボートされる可能性があります。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数は uuid 型の入力ハンドラとして機能します。文字列解析のロジックを変更する場合は string_to_uuid を参照してください。
