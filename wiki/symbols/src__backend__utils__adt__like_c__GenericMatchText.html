GenericMatchText

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like.c:149

目的

データベースのエンコーディングおよび照合順序（Collation）に基づき、適切な文字列パターンマッチング関数へ処理を振り分ける汎用ディスパッチャです。

シグネチャまたは形状

static inline int GenericMatchText(const char *s, int slen, const char *p, int plen, Oid collation)

入力と出力

  • **入力**:
  • s: 検索対象の文字列。
  • slen: s の長さ。
  • p: パターン文字列。
  • plen: p の長さ。
  • collation: 照合順序を識別する Oid。
  • **出力**:
  • マッチング結果を示す整数値（LIKE_TRUE 等）。

前提条件と不変条件

  • collation が有効な Oid である必要があります。無効な場合は ERRCODE_INDETERMINATE_COLLATION エラーを発生させます。
  • データベースのエンコーディング設定に基づき、内部的に SB_MatchText、UTF8_MatchText、または MB_MatchText のいずれかが呼び出されます。

アルゴリズムまたは振る舞い

1. OidIsValid(collation) を確認し、無効な場合はエラーを報告します。 2. pg_newlocale_from_collation(collation) を使用してロケール情報を取得します。 3. データベースの最大文字長およびエンコーディングを確認します。

  • 最大長が 1 の場合: SB_MatchText を実行。
  • UTF-8 の場合: UTF8_MatchText を実行。
  • それ以外の場合: MB_MatchText を実行。

    • 4. 選択された関数がマッチング結果を返し、それを呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/like.c 内の textlike、textnlike、namelike、namenlike 等の SQL 関数インターフェースから呼び出されます。
  • **呼び出し先**:
  • pg_newlocale_from_collation
  • pg_database_encoding_max_length
  • GetDatabaseEncoding
  • SB_MatchText / UTF8_MatchText / MB_MatchText

関連データ構造

  • Oid: 照合順序の識別子。
  • pg_locale_t: ロケール依存の比較処理に使用される構造体。

失敗モードと境界条件

  • collation が解決できない場合、ereport(ERROR, ...) により処理が中断されます。
  • エンコーディング判定ロジックにより、データベース設定に応じた適切な関数が選択されない場合、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は like_match.c で定義されるマクロベースの関数群をラップしています。エンコーディング追加やマッチングロジックの変更時は、like_match.c のインクルード構成と各 MatchText 関数の実装を確認してください。
