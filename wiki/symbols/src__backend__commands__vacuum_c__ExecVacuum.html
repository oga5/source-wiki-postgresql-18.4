ExecVacuum

定義

  • 種別: function
  • 定義箇所: src/backend/commands/vacuum.c:161

目的

SQL パーサーから受け取った VacuumStmt を解析し、VACUUM または ANALYZE 操作を実行するためのパラメータ（VacuumParams）を構築・検証するコマンド実行準備関数です。

シグネチャまたは形状

void ExecVacuum(ParseState *pstate, VacuumStmt *vacstmt, bool isTopLevel)

入力と出力

  • **入力**:
  • pstate: パーサーの状態情報。
  • vacstmt: 実行対象の VACUUM または ANALYZE 文の構文木。
  • isTopLevel: トップレベルのコマンドかどうかを示すフラグ。
  • **出力**: なし（副作用として VacuumParams を内部で構築し、後続の処理へ引き継ぐ）。

前提条件と不変条件

  • params.options には VACOPT_VACUUM または VACOPT_ANALYZE のいずれか（あるいは両方）が含まれている必要がある。
  • VACUUM FULL を実行する場合、VACOPT_VACUUM が指定されていなければならない。
  • VACUUM FULL と並列実行（nworkers > 0）を同時に指定することはできない。

アルゴリズムまたは振る舞い

1. **初期化**: VacuumParams の各フィールドをデフォルト値（index_cleanup や truncate は未指定、並列ワーカー数は 0 など）で初期化する。 2. **オプション解析**: vacstmt->options リストを走査し、verbose, buffer_usage_limit, analyze, freeze, full, parallel などの各オプションを解析・変換する。

  • buffer_usage_limit は指定された値が許容範囲内（MIN_BAS_VAC_RING_SIZE_KB ～ MAX_BAS_VAC_RING_SIZE_KB）にあるか検証する。
  • parallel オプションは 0 ～ MAX_PARALLEL_WORKER_LIMIT の範囲で検証し、0 の場合は並列処理を無効化する。

    • 3. **ビットマスク構築**: 解析結果を params.options ビットマスクに集約する。 4. **妥当性検証**: 組み合わせの矛盾（VACUUM FULL と並列実行の併用など）をチェックし、不正な場合はエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c: SQL 文の実行プロセスから呼び出される。
  • **呼び出し先**:
  • defGetBoolean, defGetString, defGetInt32: オプション値の取得に使用。
  • ereport: エラー報告に使用。
  • get_vacoptval_from_boolean: オプション値の変換に使用。

関連データ構造

  • VacuumStmt: SQL 文の構文木。
  • VacuumParams: 実行パラメータを保持する構造体。
  • ParseState: パーサーの状態を保持する構造体。

失敗モードと境界条件

  • **構文エラー**: 未知のオプションが指定された場合、または ANALYZE 専用のオプションが VACUUM 以外で指定された場合にエラーとなる。
  • **値の範囲外**: buffer_usage_limit や parallel のワーカー数が許容範囲外の場合にエラーとなる。
  • **機能制限**: VACUUM FULL と並列実行を同時に指定した場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラーとなる。

保守担当者向けメモ

  • 本関数はコマンドの準備を行うのみであり、実際のデータ処理は vacuum() 等の関数へ委譲されます。
  • BUFFER_USAGE_LIMIT は VACUUM FULL では使用できないため、将来的な拡張や仕様変更の際は、この制約の妥当性を再確認してください。
