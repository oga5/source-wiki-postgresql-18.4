expand_appendrel_subquery

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/inherit.c:798

目的

サブクエリを含む継承階層において、親リレーションに関連付けられた子リレーションの RelOptInfo を構築し、必要に応じて再帰的に継承展開を行う。

シグネチャまたは形状

static void expand_appendrel_subquery(PlannerInfo *root, RelOptInfo *rel, RangeTblEntry *rte, Index rti)

入力と出力

  • root: プランナのグローバル情報構造体。
  • rel: 展開対象である親リレーションの RelOptInfo。
  • rte: 親リレーションに対応する RangeTblEntry。
  • rti: 親リレーションの範囲テーブルインデックス。
  • 出力: なし（root->simple_rel_array および root->append_rel_list を介して副作用を及ぼす）。

前提条件と不変条件

  • root->append_rel_list には、対象の親リレーションに関連する AppendRelInfo が格納されている必要がある。
  • 子リレーションの RangeTblEntry は、既に root->simple_rte_array に存在している必要がある。

アルゴリズムまたは振る舞い

1. root->append_rel_list を走査し、引数 rti と一致する parent_relid を持つ AppendRelInfo を抽出する。 2. 各子リレーションに対して build_simple_rel を呼び出し、RelOptInfo を生成する。 3. 子リレーションの RangeTblEntry で inh フラグが有効な場合、expand_inherited_rtentry を再帰的に呼び出し、さらに深い継承階層を展開する。

呼び出し元と呼び出し先

  • 呼び出し元: expand_inherited_rtentry（src/backend/optimizer/util/inherit.c:100 付近）。
  • 呼び出し先: build_simple_rel, expand_inherited_rtentry。

関連データ構造

  • PlannerInfo: プランニング中のクエリ全体の状態を保持。
  • RelOptInfo: リレーションの最適化情報を保持。
  • AppendRelInfo: 継承関係（親と子）のメタデータを保持。
  • RangeTblEntry: 範囲テーブルの各エントリ。

失敗モードと境界条件

  • childRTindex が root->simple_rel_array_size を超える場合、または対応する childrte が存在しない場合は Assert により停止する。

保守担当者向けメモ

  • 本関数はサブクエリの継承展開に特化している。通常のテーブル継承と異なり、既に定義済みの AppendRelInfo を基点として処理を進める点に注意すること。
  • 再帰呼び出しを行うため、継承階層が深い場合にスタックオーバーフローが発生しないよう設計されている。
