VirtualTransactionId

定義

  • 種別: struct
  • 定義箇所: src/include/storage/lock.h:60

目的

バックエンドプロセスが保持するトランザクションを識別するための仮想的なトランザクションIDです。主にトランザクション終了待ちのロック管理に使用されます。

シグネチャまたは形状

typedef struct
{
	ProcNumber	procNumber;		/* proc number of the PGPROC */
	LocalTransactionId localTransactionId;	/* lxid from PGPROC */
} VirtualTransactionId;

入力と出力

  • 入力: ProcNumber（プロセス番号）および LocalTransactionId（ローカルトランザクションID）の組み合わせ。
  • 出力: 構造体として、特定のバックエンドにおける一意なトランザクション識別子を保持します。

前提条件と不変条件

  • procNumber は有効な PGPROC を指す必要があります。
  • 構造体は src/include/storage/lock.h で定義されており、ロックマネージャのコンテキスト内で使用されます。

アルゴリズムまたは振る舞い

  • 物理的なトランザクションID（XID）が割り当てられる前のトランザクションや、読み取り専用トランザクションを識別するために使用されます。
  • ロックマネージャにおいて、特定のプロセスとトランザクションの紐付けを維持します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/access/transam/twophase.c
  • src/backend/access/transam/xact.c
  • src/backend/access/transam/xlog.c
  • src/backend/commands/indexcmds.c
  • src/backend/storage/ipc/procarray.c
  • 呼び出し先: 提供されたソース情報には、本構造体が直接呼び出す関数やメソッドの情報は含まれていません。

関連データ構造

  • PGPROC: プロセス情報を保持する構造体。
  • LocalTransactionId: ローカルトランザクションID。

失敗モードと境界条件

  • 構造体内のフィールドが適切に初期化されていない場合、無効なトランザクションとして扱われる可能性があります。
  • 共有メモリ上のプロセス配列との整合性が失われた場合、誤ったトランザクションを識別するリスクがあります。

保守担当者向けメモ

  • 本構造体は LOCKTAG 等と異なり、ロック対象そのものではなく、ロックを保持する「主体」を識別するものです。
  • procarray.c 等で配列として確保されることが多いため、メモリレイアウトの変更には注意が必要です。
