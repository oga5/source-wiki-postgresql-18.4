pgstat_init_function_usage

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_function.c:71

目的

SQL関数の実行開始時に、統計収集のためのコンテキストを初期化し、実行時間計測の基準値を記録する。

シグネチャまたは形状

void pgstat_init_function_usage(FunctionCallInfo fcinfo, PgStat_FunctionCallUsage *fcu)

入力と出力

  • **入力**:
  • fcinfo: 実行対象の関数情報（OIDや統計設定を含む）。
  • fcu: 計測状態を保持するための出力用構造体。
  • **出力**: なし（fcu 構造体の各フィールドを更新）。

前提条件と不変条件

  • pgstat_track_functions GUCパラメータが関数の統計収集を許可していること。
  • 関数が実行中に削除された場合、AcceptInvalidationMessages() を経由してシステムカタログを再確認し、存在しない場合はエラーを投げる。

アルゴリズムまたは振る舞い

1. **統計収集の判定**: pgstat_track_functions の設定に基づき、対象外であれば fcu->fs を NULL にして終了する。 2. **エントリ準備**: pgstat_prep_pending_entry を呼び出し、共有メモリ上の統計エントリを準備する。 3. **整合性チェック**: 新規エントリ作成時、関数が既に削除されていないか SearchSysCacheExists1 で確認する。削除済みであれば pgstat_drop_entry を呼び出し、ERRCODE_UNDEFINED_FUNCTION でエラーを発生させる。 4. **計測値の保存**:

  • fcu->fs: 統計エントリへの参照を保持。
  • fcu->save_f_total_time: 現在の関数の累積実行時間を保存（再帰呼び出しの補正用）。
  • fcu->save_total: バックエンド全体の累積実行時間を保存。
  • fcu->start: 現在のクロック時刻を記録。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（execExprInterp.c, execSRF.c）、コマンド処理（event_trigger.c, functioncmds.c, trigger.c）、関数管理（fmgr.c）。
  • **呼び出し先**: pgstat_prep_pending_entry, AcceptInvalidationMessages, SearchSysCacheExists1, pgstat_drop_entry, INSTR_TIME_SET_CURRENT。

関連データ構造

  • PgStat_FunctionCallUsage: 計測用コンテキスト。
  • PgStat_FunctionCounts: 関数統計の累積データ。
  • FunctionCallInfo: 関数呼び出し情報。

失敗モードと境界条件

  • **関数削除**: 実行開始直前にシステムカタログから関数が削除されていた場合、ERROR を発生させる。
  • **統計無効**: pgstat_track_functions 設定により統計が不要な場合、計測を行わず正常終了する。

保守担当者向けメモ

  • 本関数は再帰呼び出しを考慮した統計算出の起点となる。pgstat_end_function_usage と対で使用されることが前提である。
  • キャッシュ無効化メッセージの処理は、関数実行時の整合性を保つために重要である。
