test_single_value

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_integerset/test_integerset.c:316

目的

IntegerSet に単一の値を格納した際の、基本操作（要素数取得、メンバ判定、イテレーション）の正確性を検証する。

シグネチャまたは形状

static void test_single_value(uint64 value)

入力と出力

  • 入力: value (テスト対象とする uint64 型の整数値)
  • 出力: なし（検証失敗時に elog(ERROR) を発行してテストを中断する）

前提条件と不変条件

  • intset_create() によって生成された空の IntegerSet に対して、単一の value を追加した状態を前提とする。
  • 集合内の要素数は常に 1 である必要がある。

アルゴリズムまたは振る舞い

1. intset_create() で集合を作成し、intset_add_member() で value を追加する。 2. intset_num_entries() が 1 を返すことを確認する。 3. intset_is_member() を使用し、0, 1, PG_UINT64_MAX, および value 自体に対する判定結果が期待値と一致するか確認する。 4. intset_begin_iterate() および intset_iterate_next() を使用し、集合から value が正しく取得できること、およびその後のイテレーションで要素が残っていないことを確認する。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/test_integerset/test_integerset.c 内のテスト実行ロジック（test_integerset 関数から呼び出される）。
  • 呼び出し先: intset_create, intset_add_member, intset_num_entries, intset_is_member, intset_begin_iterate, intset_iterate_next（すべて lib/integerset.h で定義された API）。

関連データ構造

  • IntegerSet: テスト対象の整数集合データ構造。

失敗モードと境界条件

  • intset_num_entries が 1 以外を返した場合にエラーとなる。
  • intset_is_member の判定が期待値（value と比較対象が一致するか）と異なる場合にエラーとなる。
  • イテレーションで value が取得できない、または期待しない値が取得された場合にエラーとなる。
  • 境界値として 0 や PG_UINT64_MAX を渡した場合の挙動を検証する。

保守担当者向けメモ

  • 本関数は test_integerset の初期化フェーズで、特定の境界値を検証するために繰り返し呼び出される。
  • intset_iterate_next の戻り値が true であることと、取得した値が value と一致することを厳密にチェックしているため、イテレータの実装変更時には注意が必要である。
