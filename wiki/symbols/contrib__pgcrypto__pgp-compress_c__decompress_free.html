decompress_free

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-compress.c:312

目的

PGP伸長処理で使用されたリソース（zlib ストリームおよび関連するメモリ領域）を解放し、安全に破棄します。

シグネチャまたは形状

static void decompress_free(void *priv)

入力と出力

  • **入力**: priv - 伸長処理のコンテキストを保持する DecomprData 構造体へのポインタ（void * として受け取る）。
  • **出力**: なし。

前提条件と不変条件

  • priv は DecomprData 構造体として初期化済みであること。
  • 本関数呼び出し後、priv が指すメモリ領域は無効化されるため、呼び出し元は当該ポインタを参照してはならない。

アルゴリズムまたは振る舞い

1. inflateEnd(&dec->stream) を呼び出し、zlib が保持している伸長用ストリームの状態を解放する。 2. px_memset(dec, 0, sizeof(*dec)) を実行し、DecomprData 構造体の中身をゼロクリアして機密情報の残留を防ぐ。 3. pfree(dec) を呼び出し、構造体自体が占有していたメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し先**: inflateEnd (zlib), px_memset, pfree (PostgreSQL メモリ管理)。
  • **呼び出し元**: ソースコード内の記述から、伸長フィルタのライフサイクル管理を行う関数群（decompress_init 等のコンテキスト）から参照されていることが示唆されるが、具体的な呼び出し箇所は本抜粋からは特定できない。

関連データ構造

  • DecomprData: 伸長処理の状態を保持する内部構造体。z_stream をメンバとして含む。

失敗モードと境界条件

  • priv が NULL の場合、pfree 等の内部関数で不正なメモリアクセスが発生する可能性がある。

保守担当者向けメモ

  • 本関数は pgcrypto のフィルタパイプラインにおけるクリーンアップ処理の一部である。
  • px_memset によるゼロクリアは、暗号化関連モジュールとしてのセキュリティ要件（メモリ上の機密データ消去）を満たすために重要である。
