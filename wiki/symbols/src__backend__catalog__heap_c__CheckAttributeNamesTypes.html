CheckAttributeNamesTypes

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/heap.c:451

目的

リレーション（テーブル等）の列定義が妥当であるかを検証する。具体的には、列数の上限チェック、システム列名との重複確認、列名の重複確認、および各列のデータ型の妥当性検証を行う。

シグネチャまたは形状

void CheckAttributeNamesTypes(TupleDesc tupdesc, char relkind, int flags)

入力と出力

  • **入力**:
  • tupdesc: 検証対象のタプル記述子。
  • relkind: リレーションの種類（RELKIND_VIEW や RELKIND_COMPOSITE_TYPE 等）。
  • flags: CheckAttributeType に渡されるフラグビット。
  • **出力**: なし（検証失敗時は ereport(ERROR, ...) により例外をスローする）。

前提条件と不変条件

  • tupdesc は有効なポインタであること。
  • カタログ操作を伴うため、トランザクション内での実行が前提となる。

アルゴリズムまたは振る舞い

1. **列数チェック**: tupdesc->natts が MaxHeapAttributeNumber を超えていないか確認する。 2. **システム列名との衝突確認**: relkind がビューや複合型でない場合、各列名がシステム列名と重複していないか SystemAttributeByName を用いて検証する。 3. **列名の重複確認**: 全ての列名について、同一リレーション内で名前が重複していないか確認する。 4. **データ型検証**: 各列に対して CheckAttributeType を呼び出し、型定義の妥当性を検証する。attisdropped（削除済み）の列はスキップされる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/heap.c 内のテーブル作成処理（heap_create_with_catalog 等）や、src/backend/parser/parse_relation.c から呼び出される。
  • **呼び出し先**:
  • SystemAttributeByName
  • CheckAttributeType
  • TupleDescAttr（マクロ）

関連データ構造

  • TupleDesc: リレーションの列定義を保持する構造体。
  • Form_pg_attribute: pg_attribute カタログの行構造。

失敗モードと境界条件

  • **ERRCODE_TOO_MANY_COLUMNS**: 列数が MaxHeapAttributeNumber を超えた場合に発生。
  • **ERRCODE_DUPLICATE_COLUMN**: 列名がシステム列名と重複、または列名が重複している場合に発生。
  • **その他**: CheckAttributeType 内部での型検証失敗時にエラーが発生する。

保守担当者向けメモ

  • relkind による分岐は、システム列を持たないリレーション（ビュー等）を考慮している。
  • CheckAttributeType は再帰的に呼び出される可能性があるため、スタック深さの制限に注意が必要。
  • システム属性の定義は heap.c 内で静的に管理されており、変更時には関連するシステムカタログ定義との整合性に留意すること。
