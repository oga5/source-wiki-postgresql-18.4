rebuild_relation

定義

  • 種別: function
  • 定義箇所: src/backend/commands/cluster.c:628

目的

既存のリレーション（テーブル）を、指定されたインデックス順序、あるいは物理順序で再構築し、領域の最適化や物理的な並び替えを行う。

シグネチャまたは形状

static void rebuild_relation(Relation OldHeap, Relation index, bool verbose)

入力と出力

  • **入力**:
  • OldHeap: 再構築対象のリレーション（AccessExclusiveLock 保持必須）。
  • index: クラスタリングに使用するインデックス（指定なしの場合は NULL）。
  • verbose: 処理状況をログ出力するかどうかのフラグ。
  • **出力**: なし（副作用としてテーブルの物理ファイルが置換される）。

前提条件と不変条件

  • OldHeap および index（指定時）に対して、呼び出し元で AccessExclusiveLock を取得していること。
  • make_new_heap により、新しいヒープに対して AccessExclusiveLock が取得される。

アルゴリズムまたは振る舞い

1. **インデックス設定**: index が指定されている場合、mark_index_clustered を呼び出し、対象インデックスをクラスタリング用としてマークする。 2. **一時テーブル作成**: make_new_heap を呼び出し、再構築用の空のテンポラリテーブルを作成する。 3. **データコピー**: copy_table_data を呼び出し、OldHeap から NewHeap へデータを指定順序でコピーする。 4. **リソース解放**: OldHeap、index、NewHeap のリレーションキャッシュエントリを閉じる。 5. **ファイルスワップ**: finish_heap_swap を呼び出し、物理ファイルを入れ替え、インデックスを再構築し、一時テーブルを削除する。

呼び出し元と呼び出し先

  • **呼び出し元**: cluster_rel（src/backend/commands/cluster.c 内）。
  • **呼び出し先**: mark_index_clustered, make_new_heap, table_open, copy_table_data, table_close, index_close, finish_heap_swap。

関連データ構造

  • Relation: リレーション記述子。
  • Oid: オブジェクト識別子。
  • TransactionId, MultiXactId: トランザクション管理用識別子。

失敗モードと境界条件

  • ロック未取得状態で呼び出された場合、Assert により異常終了する。
  • make_new_heap や finish_heap_swap 内部でのカタログ操作やファイルシステム操作が失敗した場合、トランザクション全体がアボートされる。

保守担当者向けメモ

  • 本関数は「1テーブル1トランザクション」の設計に従い、長時間ロックを保持する。
  • finish_heap_swap はシステムカタログの整合性に直結するクリティカルな処理であるため、変更時は注意が必要。
  • index が NULL の場合、物理順序での再構築（VACUUM FULL 相当の動作）が行われる。
