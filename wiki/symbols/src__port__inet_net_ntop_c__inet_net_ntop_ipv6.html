inet_net_ntop_ipv6

定義

  • 種別: function
  • 定義箇所: src/port/inet_net_ntop.c:177

目的

IPv6 アドレス（バイナリ形式）を、CIDR 表記を含む人間が読み取り可能な文字列形式へ変換します。

シグネチャまたは形状

static char *inet_net_ntop_ipv6(const u_char *src, int bits, char *dst, size_t size);

入力と出力

  • src: 変換対象の IPv6 アドレス（ネットワークバイトオーダーのバイト配列）。
  • bits: CIDR 接頭辞のビット数（-1 は指定なし、0〜128 が有効）。
  • dst: 結果を格納するバッファ。
  • size: dst バッファのサイズ。
  • 戻り値: 成功時は dst へのポインタ、失敗時は NULL。

前提条件と不変条件

  • bits が -1 未満または 128 を超える場合、errno に EINVAL を設定して NULL を返します。
  • 変換後の文字列が size を超える場合、errno に EMSGSIZE を設定して NULL を返します。

アルゴリズムまたは振る舞い

1. **ワード変換**: 入力バイト配列を 16 ビット単位のワード配列に変換します。 2. **ゼロ圧縮の特定**: 連続する 0 の最長範囲を特定し、:: で省略可能な箇所を決定します。 3. **文字列構築**:

  • 16 ビットワードを 16 進数文字列として一時バッファ tmp に書き込みます。
  • 特定された最長ゼロ範囲は :: に置き換えます。
  • IPv4 互換アドレスの場合は decoct を呼び出して末尾をドット区切り形式に変換します。

    • 4. **CIDR 付与**: bits が指定されている場合、文字列の末尾に /<bits> を追加します。 5. **コピー**: 最終的な文字列を dst にコピーします。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/inet_net_ntop.c 内の pg_inet_net_ntop から呼び出されます。
  • 呼び出し先: decoct（IPv4 互換アドレス変換時）。

関連データ構造

  • words: 16 ビット単位のアドレス表現を保持する配列。
  • best, cur: ゼロ圧縮の範囲（開始位置と長さ）を追跡するための構造体。

失敗モードと境界条件

  • bits が範囲外の場合: EINVAL を返却。
  • バッファサイズ不足: EMSGSIZE を返却。
  • IPv4 互換アドレス変換失敗時: EMSGSIZE を返却。

保守担当者向けメモ

  • IPv6 のゼロ圧縮ロジック（best の計算）は複雑であるため、変更時は最長ゼロランの判定ロジックを慎重に確認してください。
  • SPRINTF マクロを使用しており、標準の sprintf とは戻り値の扱いが異なる点に注意してください。
