pg_checksum_raw_context

定義

  • 種別: typedef (共用体)
  • 定義箇所: src/include/common/checksum_helper.h:42

目的

チェックサム計算アルゴリズムの種類に応じて、計算に必要な内部状態を保持するための抽象化コンテナです。

シグネチャまたは形状

typedef union pg_checksum_raw_context
{
    pg_crc32c           c_crc32c;
    pg_cryptohash_ctx  *c_sha2;
} pg_checksum_raw_context;

入力と出力

  • 入力: 本共用体は pg_checksum_context 構造体の一部として保持されます。
  • 出力: 選択されたアルゴリズム（CRC-32C または SHA-2）に応じた計算状態を格納します。

前提条件と不変条件

  • pg_checksum_context 内のアルゴリズム識別子と、本共用体で実際に使用されているメンバが一致している必要があります。
  • SHA-2 を使用する場合、c_sha2 は pg_cryptohash_ctx への有効なポインタである必要があります。

アルゴリズムまたは振る舞い

  • CRC-32C 使用時: c_crc32c メンバを使用して計算状態を直接保持します。
  • SHA-2 ファミリー使用時: c_sha2 メンバを使用して、暗号学的ハッシュのコンテキストへのポインタを保持します。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/common/checksum_helper.h 内の pg_checksum_context 構造体定義で使用されています。
  • 呼び出し先: 具体的な計算処理を行う src/common/checksum_helper.c 内の関数群が、この共用体のメンバを操作します。

関連データ構造

  • pg_checksum_context: 本共用体を内部に持つ、チェックサム計算用のメイン構造体。
  • pg_cryptohash_ctx: c_sha2 が指し示す暗号学的ハッシュのコンテキスト構造体。

失敗モードと境界条件

  • SHA-2 使用時に c_sha2 が初期化されていない、または解放済みである状態でアクセスすると未定義の動作を引き起こします。

保守担当者向けメモ

  • 本共用体は pg_checksum_context の一部として管理されます。新しいチェックサムアルゴリズムを追加する場合は、この共用体に新しいメンバを追加し、pg_checksum_init / update / final の各関数で適切にハンドリングしてください。
