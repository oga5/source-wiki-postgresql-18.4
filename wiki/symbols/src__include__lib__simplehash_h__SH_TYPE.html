SH_TYPE

定義

  • 種別: struct
  • 定義箇所: src/include/lib/simplehash.h:145

目的

マクロ定義に基づき生成される、型安全かつ高性能なオープンアドレス法ハッシュテーブルのメイン構造体です。

シグネチャまたは形状

typedef struct SH_TYPE
{
    uint64      size;           /* バケット配列のサイズ */
    uint32      members;        /* 格納されている要素数 */
    uint32      sizemask;       /* サイズ計算用マスク */
    uint32      grow_threshold; /* 拡張閾値 */
    SH_ELEMENT_TYPE *data;      /* ハッシュバケット配列 */
#ifndef SH_RAW_ALLOCATOR
    MemoryContext ctx;          /* メモリコンテキスト */
#endif
    void       *private_data;   /* コールバック用ユーザーデータ */
} SH_TYPE;

入力と出力

  • **入力**: SH_CREATE マクロにより初期化され、SH_INSERT 等の操作関数を通じて要素が格納されます。
  • **出力**: SH_LOOKUP 等の操作関数により、SH_ELEMENT_TYPE 型の要素へのポインタが返されます。

前提条件と不変条件

  • size は 64 ビットで管理され、最大 UINT32_MAX までのハッシュテーブルを扱えます。
  • 実際の要素数は SH_MAX_FILLFACTOR によって制限されます。
  • SH_RAW_ALLOCATOR が定義されていない場合、MemoryContext を介したメモリ管理が必須となります。

アルゴリズムまたは振る舞い

  • **Robin Hoodハッシュ**: 衝突時に最適位置から遠い要素を移動させることで、検索性能を安定させます。
  • **動的拡張**: members が grow_threshold を超えると、SH_GROW が呼び出され、テーブルサイズが 2 倍に拡張されます。
  • **メモリ管理**: MemoryContext またはカスタムアロケータを使用してバケット配列を保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: SH_CREATE, SH_DESTROY, SH_RESET, SH_GROW, SH_INSERT, SH_INSERT_HASH, SH_LOOKUP, SH_LOOKUP_HASH, SH_DELETE_ITEM などの操作関数が本構造体を引数として受け取ります。
  • **呼び出し先**: SH_RAW_ALLOCATOR が未定義の場合、utils/memutils.h のメモリ管理関数が利用されます。

関連データ構造

  • SH_ELEMENT_TYPE: テーブル内に格納される要素の型。
  • SH_ITERATOR: ハッシュテーブル走査用構造体。
  • SH_STATUS: バケット状態管理用列挙型。

失敗モードと境界条件

  • **メモリ不足**: SH_CREATE や SH_GROW 実行時にメモリ割り当てが失敗する可能性があります。
  • **サイズ制限**: size が UINT32_MAX を超えるような大規模なテーブル生成はサポートされていません。

保守担当者向けメモ

  • 本構造体はマクロによるコード生成の一部であるため、直接編集するのではなく、生成元のマクロ定義を確認してください。
  • SH_STORE_HASH が定義されている場合、要素内にハッシュ値が保持され、検索効率が向上します。
