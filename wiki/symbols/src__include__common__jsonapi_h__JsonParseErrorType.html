JsonParseErrorType

定義

  • 種別: enum
  • 定義箇所: src/include/common/jsonapi.h:34

目的

JSON 解析処理における成功状態、および発生したエラーの種類を分類・識別するために使用されます。

シグネチャまたは形状

typedef enum JsonParseErrorType {
    JSON_SUCCESS,
    JSON_INCOMPLETE,
    JSON_INVALID_LEXER_TYPE,
    JSON_NESTING_TOO_DEEP,
    JSON_ESCAPING_INVALID,
    JSON_ESCAPING_REQUIRED,
    JSON_EXPECTED_ARRAY_FIRST,
    JSON_EXPECTED_ARRAY_NEXT,
    JSON_EXPECTED_COLON,
    JSON_EXPECTED_END,
    JSON_EXPECTED_JSON,
    JSON_EXPECTED_MORE,
    JSON_EXPECTED_OBJECT_FIRST,
    JSON_EXPECTED_OBJECT_NEXT,
    JSON_EXPECTED_STRING,
    JSON_INVALID_TOKEN,
    JSON_OUT_OF_MEMORY,
    JSON_UNICODE_CODE_POINT_ZERO,
    JSON_UNICODE_ESCAPE_FORMAT,
    JSON_UNICODE_HIGH_ESCAPE,
    JSON_UNICODE_UNTRANSLATABLE,
    JSON_UNICODE_HIGH_SURROGATE,
    JSON_UNICODE_LOW_SURROGATE,
    JSON_SEM_ACTION_FAILED
} JsonParseErrorType;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: JSON 解析関数（pg_parse_json 等）の戻り値として、解析結果の状態を返します。

前提条件と不変条件

  • JSON_SUCCESS は解析が正常に完了したことを示します。
  • JSON_SEM_ACTION_FAILED が返される場合、エラーの詳細は既にコールバック関数内で報告済みであると見なされます。

アルゴリズムまたは振る舞い

パーサーは入力ストリームを走査し、構文エラーやメモリ不足、Unicode 変換エラーなどを検知した際に、対応する列挙値を返して処理を中断します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/utils/adt/json.c
  • src/backend/utils/adt/jsonb.c
  • 呼び出し先: なし（本シンボルは列挙型であり、関数ではないため）

関連データ構造

  • JsonLexContext: 解析状態を保持する構造体。
  • JsonSemAction: 解析中に実行されるコールバック関数のテーブル。

失敗モードと境界条件

  • JSON_OUT_OF_MEMORY: メモリ割り当て失敗時に発生。
  • JSON_NESTING_TOO_DEEP: JSON のネストが深すぎる場合に発生。
  • JSON_SEM_ACTION_FAILED: ユーザー定義のセマンティックアクション（コールバック）内でエラーが発生した場合。
  • 各種 JSON_UNICODE_* エラー: 不正な Unicode エスケープシーケンスやサロゲートペアの不整合時に発生。

保守担当者向けメモ

  • 新しいエラータイプを追加する場合は、jsonapi.c 側のエラーハンドリングロジックも併せて更新してください。
  • JSON_SEM_ACTION_FAILED を返す際は、必ず呼び出し元でエラーメッセージが報告されていることを確認してください。
