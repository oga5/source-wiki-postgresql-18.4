port_bio_write

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-openssl.c:935

目的

OpenSSL の BIO インターフェースを介して、PostgreSQL のソケットへデータを書き込むためのカスタムメソッドを提供します。

シグネチャまたは形状

static int port_bio_write(BIO *h, const char *buf, int size);

入力と出力

  • **入力**:
  • h: 書き込み対象の BIO オブジェクト。
  • buf: 書き込むデータが格納されたバッファ。
  • size: 書き込むデータのバイト数。
  • **出力**:
  • 成功時は書き込んだバイト数（正の整数）、失敗時は 0 または負の値を返します。

前提条件と不変条件

  • BIO_get_data(h) を通じて、有効な Port 構造体ポインタが取得できる必要があります。
  • secure_raw_write が内部で呼び出されるため、ソケットの状態が書き込み可能である必要があります。

アルゴリズムまたは振る舞い

1. BIO オブジェクトから Port 構造体を取得し、secure_raw_write を呼び出してソケットへデータを書き込みます。 2. BIO_clear_retry_flags(h) を呼び出し、既存の再試行フラグをクリアします。 3. 書き込み結果が 0 以下の場合、errno を確認します。 4. errno が EINTR、EWOULDBLOCK、または EAGAIN のいずれかである場合、BIO_set_retry_write(h) を呼び出し、呼び出し元に対して再試行を要求します。 5. 最終的な書き込み結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-secure-openssl.c:998 にて、BIO_meth_set_write を介して OpenSSL のメソッドテーブルに登録されます。
  • **呼び出し先**:
  • secure_raw_write (直接の書き込み処理)
  • BIO_get_data (OpenSSL API)
  • BIO_clear_retry_flags (OpenSSL API)
  • BIO_set_retry_write (OpenSSL API)

関連データ構造

  • BIO: OpenSSL の I/O 抽象化構造体。
  • Port: PostgreSQL の接続状態を保持する構造体。

失敗モードと境界条件

  • secure_raw_write がエラーを返した場合、errno の値に応じて再試行フラグが設定されます。
  • size が 0 の場合や、ソケットがクローズされている場合の挙動は secure_raw_write の実装に依存します。

保守担当者向けメモ

  • 本関数は OpenSSL の BIO_METHOD として登録されることを前提としています。
  • errno の判定は非ブロッキング I/O 処理と密接に関連しているため、変更時は pqcomm.c 側のソケット処理との整合性に注意してください。
