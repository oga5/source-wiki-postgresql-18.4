shell_check_detail

定義

  • 種別: function
  • 定義箇所: contrib/basebackup_to_shell/basebackup_to_shell.c:101

目的

バックアップターゲットとして shell が指定された際に、実行ユーザーが外部コマンド実行に必要な権限を有しているかを検証する。

シグネチャまたは形状

static void *shell_check_detail(char *target, char *target_detail);

入力と出力

  • **入力**:
  • target: ターゲット名（"shell"）。
  • target_detail: バックアップターゲットの詳細情報。
  • **出力**:
  • 成功時: 入力された target_detail をそのまま返す。
  • 失敗時: 権限不足の場合、ereport(ERROR, ...) により処理を中断する。

前提条件と不変条件

  • shell_required_role が空文字でない場合、現在のユーザーが指定されたロールの権限を持っている必要がある。
  • トランザクション管理関数（StartTransactionCommand / CommitTransactionCommand）が呼び出し可能な状態であること。

アルゴリズムまたは振る舞い

1. shell_required_role が設定されているか確認する。 2. 設定されている場合、トランザクションを開始する。 3. get_role_oid を使用して、必要なロールの OID を取得する。 4. has_privs_of_role を使用して、現在のユーザー（GetUserId()）がそのロールの権限を持っているか検証する。 5. 権限がない場合は ERRCODE_INSUFFICIENT_PRIVILEGE でエラーを発生させる。 6. トランザクションをコミットし、target_detail を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: BaseBackupAddTarget への引数として登録され、バックアップターゲットの初期化プロセスから呼び出される。
  • **呼び出し先**:
  • StartTransactionCommand / CommitTransactionCommand
  • get_role_oid
  • has_privs_of_role
  • GetUserId
  • ereport

関連データ構造

  • shell_required_role: 権限チェックに使用されるグローバルなロール名。

失敗モードと境界条件

  • **権限不足**: ユーザーが shell_required_role の権限を持たない場合、バックアップ処理自体がエラーで停止する。
  • **ロール未定義**: get_role_oid が指定されたロールを見つけられない場合の挙動は、PostgreSQL の標準的なロール検索仕様に依存する。

保守担当者向けメモ

  • 本関数は BaseBackupAddTarget に渡されるコールバック関数である。
  • 外部コマンド実行のセキュリティを担保する重要なゲートキーパーであるため、権限チェックロジックの変更には注意が必要である。
