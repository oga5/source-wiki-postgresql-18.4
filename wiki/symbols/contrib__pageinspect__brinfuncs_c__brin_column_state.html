brin_column_state

定義

  • 種別: struct
  • 定義箇所: contrib/pageinspect/brinfuncs.c:33

目的

BRIN インデックスの各カラムに対して、インデックス項目をテキスト形式へ変換するための出力関数情報（FmgrInfo）を保持する構造体です。

シグネチャまたは形状

typedef struct brin_column_state
{
	int			nstored;
	FmgrInfo	outputFn[FLEXIBLE_ARRAY_MEMBER];
} brin_column_state;

入力と出力

  • 入力: nstored（格納された属性数）、outputFn（可変長配列としての出力関数情報）。
  • 出力: なし（データ構造体）。

前提条件と不変条件

  • outputFn は FLEXIBLE_ARRAY_MEMBER を使用しており、構造体の末尾に配置されています。
  • インスタンス化の際は、offsetof(brin_column_state, outputFn) を用いて必要なメモリサイズを計算する必要があります。

アルゴリズムまたは振る舞い

  • brin_page_items 関数において、インデックス内の各カラムを SQL 結果セットとして出力する際、型に応じた変換処理を効率的に実行するために使用されます。
  • 実行時に palloc を介して動的にメモリ確保され、インデックスの属性数に応じて outputFn 配列が拡張されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pageinspect/brinfuncs.c 内の関数（brin_page_items 等）。
  • 呼び出し先: 直接的な関数呼び出しは行いませんが、保持する FmgrInfo を通じて各型の出力関数が間接的に呼び出されます。

関連データ構造

  • FmgrInfo: PostgreSQL の関数マネージャ情報構造体。
  • BrinMemTuple: brin_page_items で使用されるメモリ上のタプル表現。

失敗モードと境界条件

  • メモリ確保時: RelationGetDescr(indexRel)->natts に基づいてメモリを確保するため、インデックスの属性数が極端に多い場合に palloc が失敗する可能性があります。
  • バージョン不一致: 拡張のバージョンが古い場合、関数定義と実装の不一致により、本構造体を用いたデータ変換時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は brinfuncs.c 内でのみ使用される内部的な状態保持用です。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズ計算には必ず offsetof を使用してください。
