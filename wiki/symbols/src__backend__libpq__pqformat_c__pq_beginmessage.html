pq_beginmessage

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqformat.c:87

目的

フロントエンド・バックエンド間通信プロトコルにおけるメッセージ構築を開始し、StringInfo バッファを初期化する。

シグネチャまたは形状

void pq_beginmessage(StringInfo buf, char msgtype)

入力と出力

  • buf: 初期化対象となる StringInfo 構造体へのポインタ。
  • msgtype: メッセージの種類を示す識別子（文字）。
  • 戻り値: なし。

前提条件と不変条件

  • buf は呼び出し元によってメモリが確保されている必要がある。
  • pq_sendXXX 系関数が buf->cursor を変更しないことを前提としている。

アルゴリズムまたは振る舞い

1. initStringInfo(buf) を呼び出し、バッファを初期化する。 2. 引数 msgtype を buf->cursor フィールドに格納する。これは、後続の pq_endmessage 等でメッセージタイプとして利用されるための待避場所として機能する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/printsimple.c
  • src/backend/backup/basebackup_copy.c
  • src/backend/commands/async.c
  • src/backend/commands/copyfromparse.c
  • src/backend/commands/copyto.c
  • src/backend/libpq/auth.c
  • **呼び出し先**:
  • initStringInfo (内部で呼び出し)

関連データ構造

  • StringInfo: メッセージ構築に使用される動的バッファ構造体。

失敗モードと境界条件

  • initStringInfo がメモリ不足によりエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数で設定した buf->cursor は、メッセージ構築の終了時に pq_endmessage が参照する。このフィールドを他の目的で上書きしないよう注意すること。
