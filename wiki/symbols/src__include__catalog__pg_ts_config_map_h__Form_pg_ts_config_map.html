Form_pg_ts_config_map

定義

  • 種別: typedef
  • 定義箇所: src/include/catalog/pg_ts_config_map.h:45

目的

システムカタログ pg_ts_config_map の各行データ（FormData_pg_ts_config_map）を指すためのポインタ型を提供します。

シグネチャまたは形状

typedef FormData_pg_ts_config_map *Form_pg_ts_config_map;

入力と出力

  • 入力: GETSTRUCT() マクロ等を通じて取得された、pg_ts_config_map カタログのタプル（HeapTuple）データ。
  • 出力: FormData_pg_ts_config_map 構造体へのポインタ。

前提条件と不変条件

  • GETSTRUCT() を介してキャストされる際、対象のタプルが pg_ts_config_map カタログの物理構造と一致している必要があります。

アルゴリズムまたは振る舞い

本シンボル自体は型定義であり、実行時の振る舞いは持ちません。実体は、全文検索設定（mapcfg）と辞書（mapdict）の紐付け情報を保持する構造体へのアクセス手段として機能します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/commands/tsearchcmds.c (複数箇所)
  • src/backend/utils/cache/ts_cache.c:487
  • 呼び出し先:
  • 本シンボルは型定義であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • FormData_pg_ts_config_map: カタログの各行を表現する基底構造体。
  • pg_ts_config_map: 本構造体が定義するシステムカタログ。

失敗モードと境界条件

  • 不適切なタプルに対してキャストを行った場合、メモリレイアウトの不整合により不正な値へのアクセスが発生する可能性があります。

保守担当者向けメモ

  • pg_ts_config_map の列定義（mapcfg, maptokentype, mapseqno, mapdict）に変更を加えた場合は、FormData_pg_ts_config_map 構造体と併せて整合性を確認してください。
