tstoreReceiveSlot_tupmap

定義

  • 種別: function
  • 定義箇所: src/backend/executor/tstoreReceiver.c:191

目的

実行エンジンから受け取ったタプルを、ターゲットの TupleDesc に適合するように変換し、Tuplestore に格納する。

シグネチャまたは形状

static bool tstoreReceiveSlot_tupmap(TupleTableSlot *slot, DestReceiver *self)

入力と出力

  • **入力**:
  • slot: 変換対象のタプルを保持する TupleTableSlot。
  • self: TStoreState としてキャストされる DestReceiver オブジェクト。
  • **出力**:
  • 常に true を返す。

前提条件と不変条件

  • self は TStoreState 構造体として初期化されていること。
  • myState->tupmap->attrMap および myState->mapslot が有効であること。
  • detoast 処理と本関数（タプル変換）を同時に使用しないこと（SetTuplestoreDestReceiverParams 内の制約）。

アルゴリズムまたは振る舞い

1. myState から変換マップ（attrMap）と作業用スロット（mapslot）を取得する。 2. execute_attr_map_slot を呼び出し、入力 slot を変換ルールに従って mapslot へ変換・コピーする。 3. 変換後の mapslot を tuplestore_puttupleslot を用いて Tuplestore に書き込む。 4. 成功を示す true を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジンのタプル処理ループ（直接の呼び出し元はソースコード上では明示されていないが、TStoreState の receiveSlot 関数ポインタとして設定される）。
  • **呼び出し先**:
  • execute_attr_map_slot (変換処理)
  • tuplestore_puttupleslot (ストレージへの書き込み)

関連データ構造

  • TStoreState: 本関数が操作する状態管理構造体。
  • TupleTableSlot: タプルの保持および変換用ワークスペース。
  • Tuplestore: 格納先の永続化ストレージ。

失敗モードと境界条件

  • execute_attr_map_slot または tuplestore_puttupleslot 内部でエラーが発生した場合、例外がスローされる可能性がある。
  • メモリ不足時には palloc 関連の例外が発生する可能性がある。

保守担当者向けメモ

  • 本関数は tstoreStartupReceiver において、変換が必要な場合に receiveSlot として割り当てられる。
  • 変換ロジックの変更が必要な場合は execute_attr_map_slot の引数や mapslot の構成を確認すること。
