GetPermutation

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_rbtree/test_rbtree.c:93

目的

指定されたサイズ size の整数配列に対し、Fisher-Yates シャッフルアルゴリズム（Inside-out 変種）を用いて、0 から size - 1 までの値をランダムに並べ替えた順列を生成する。

シグネチャまたは形状

int *GetPermutation(int size)

入力と出力

  • **入力**: size (生成する順列の要素数)
  • **出力**: palloc によってメモリ確保された、ランダムに並べ替えられた int 型の配列へのポインタ。

前提条件と不変条件

  • size は正の整数であること。
  • pg_global_prng_state が初期化されており、乱数生成に使用可能であること。
  • 戻り値の配列は呼び出し元で pfree 等により適切に解放する必要がある。

アルゴリズムまたは振る舞い

1. size 分のメモリを palloc で確保する。 2. 配列の先頭要素 permutation[0] を 0 で初期化する。 3. i = 1 から size - 1 までループし、以下の処理を行う：

  • pg_prng_uint64_range を使用して、0 から i の範囲で乱数 j を生成する。
  • j < i の場合、既存の permutation[j] の値を permutation[i] にコピーする。
  • permutation[j] に現在のインデックス i を代入する。

    • 4. 完成した配列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_rbtree/test_rbtree.c 内の rbt_populate 関数から呼び出される。
  • **呼び出し先**: palloc (メモリ確保)、pg_prng_uint64_range (乱数生成)。

関連データ構造

  • pg_global_prng_state: 乱数生成器の状態を保持するグローバル変数。

失敗モードと境界条件

  • size が 0 または負の場合の挙動はソースコード上では明示されていないが、palloc の引数やループ条件に依存する。
  • メモリ不足時には palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は Fisher-Yates シャッフルの「Inside-out」アルゴリズムを実装しており、挿入とスワップを同時に行うことで効率化を図っている。
  • 乱数生成に pg_global_prng_state を使用しているため、テスト実行ごとに異なる順列が生成される。
