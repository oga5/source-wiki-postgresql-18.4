hstore_delete_array

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_op.c:291

目的

hstore 値から、指定されたキー配列に含まれるキーを持つエントリを削除し、新しい hstore を生成する。

シグネチャまたは形状

Datum hstore_delete_array(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_HSTORE_P(0): 操作対象の HStore 構造体。
  • PG_GETARG_ARRAYTYPE_P(1): 削除対象のキーを格納した text[] 配列。
  • **出力**:
  • Datum: 削除後の新しい HStore 構造体へのポインタ。

前提条件と不変条件

  • 入力 hstore は内部的にキーの長さと内容でソートされている必要がある。
  • hstoreArrayToPairs により変換されたキー配列も、同様の順序でソートされている必要がある。
  • メモリ管理は palloc を使用し、結果の hstore は HS_FINALIZE マクロによって最終的なサイズとエントリ数が確定される。

アルゴリズムまたは振る舞い

1. 入力 hstore を解析し、削除対象のキー配列を Pairs 構造体の配列に変換する。 2. 削除対象のキーが空の場合、入力 hstore のコピーをそのまま返す。 3. 入力 hstore とソート済みの削除対象キー配列をマージソートのように走査する。 4. 両者のキーを比較し、一致しない（削除対象ではない）エントリのみを新しいメモリ領域にコピーする。 5. HS_FINALIZE を呼び出し、最終的な hstore のサイズとエントリ数を設定して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • hstoreArrayToPairs: キー配列の変換とソート。
  • HS_COPYITEM, HS_FINALIZE: hstore 構造の構築と確定。
  • memcmp: キーの比較。

関連データ構造

  • HStore: 操作対象のメイン構造体。
  • HEntry: hstore 内の各エントリのメタデータ。
  • Pairs: hstoreArrayToPairs が生成する、キーと長さのペアを保持する構造体。

失敗モードと境界条件

  • 削除対象のキー配列が空の場合、入力のコピーを返す。
  • 削除対象のキーが hstore 内に存在しない場合、そのキーは無視され、元の hstore が維持される。
  • メモリ不足時は palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は hstore のソート済み特性を前提としたマージアルゴリズムを使用している。ロジックを変更する際は、キーの比較順序（長さ優先、次に内容）が維持されていることを確認すること。
  • HS_SETCOUNT は一時的な値であり、最終的な確定には HS_FINALIZE が必須である点に注意。
