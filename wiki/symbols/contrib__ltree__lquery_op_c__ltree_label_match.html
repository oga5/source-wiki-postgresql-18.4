ltree_label_match

定義

  • 種別: function
  • 定義箇所: contrib/ltree/lquery_op.c:80

目的

ltree のラベル（ノード名）と lquery のパターンが一致するかを判定します。大文字小文字を区別しない比較や、プレフィックスマッチングをサポートします。

シグネチャまたは形状

bool ltree_label_match(const char *pred, size_t pred_len, const char *label, size_t label_len, bool prefix, bool ci)

入力と出力

  • **入力**:
  • pred: 検索パターン文字列。
  • pred_len: パターン文字列の長さ。
  • label: 対象となるラベル文字列。
  • label_len: ラベル文字列の長さ。
  • prefix: true の場合、プレフィックスマッチング（前方一致）を行います。
  • ci: true の場合、大文字小文字を区別しない比較を行います。
  • **出力**:
  • 一致する場合は true、それ以外は false を返します。

前提条件と不変条件

  • ci が false の場合、バイナリ一致（またはバイナリプレフィックス一致）のみが行われます。
  • ci が true の場合、ロケールに基づいたケースフォールディングが行われます。
  • pg_strfold を使用してケースフォールディングを行う際、必要に応じてバッファの再割り当て（repalloc）が行われます。

アルゴリズムまたは振る舞い

1. **高速パス**: ci が false かつバイナリ一致が成立する場合、即座に true を返します。 2. **Cロケール処理**: ロケールが C の場合、pg_ascii_tolower を用いて各文字を比較します。 3. **低速パス（ケースインセンシティブ）**:

  • pg_strfold を使用して pred と label をケースフォールディング後の文字列に変換します。
  • 変換後の文字列長が初期バッファサイズを超える場合は、メモリを再確保して再試行します。
  • 変換後の文字列に対して strncmp を実行し、一致を判定します。
  • 最後に pfree で一時バッファを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/lquery_op.c 内のロジック（checkLevel 等の文脈）
  • contrib/ltree/ltxtquery_op.c:71
  • **呼び出し先**:
  • pg_newlocale_from_collation
  • pg_ascii_tolower
  • pg_strfold
  • palloc, repalloc, pfree

関連データ構造

  • ltree: 階層パスを保持するデータ構造。
  • lquery: 検索パターンを保持するデータ構造。

失敗モードと境界条件

  • ci が false で一致しない場合、即座に false を返します。
  • ケースフォールディング後の文字列長が元のバッファサイズを超えた場合、repalloc による動的なメモリ拡張が行われます。
  • メモリ不足時には palloc/repalloc がエラーを発生させます。

保守担当者向けメモ

  • pg_strfold はロケール設定により計算コストが変動するため、バイナリ一致の高速パスが優先される設計になっています。
  • ci が true の場合、ケースフォールディング後のバイト長が元の長さと異なる可能性があるため、単純な長さ比較のみで判定を打ち切ることはできません。
