pg_tzenumerate_next

定義

  • 種別: function
  • 定義箇所: src/timezone/pgtz.c:425

目的

タイムゾーンデータベースを再帰的に走査し、利用可能なタイムゾーンを一つずつ列挙するイテレータ関数です。

シグネチャまたは形状

pg_tz *pg_tzenumerate_next(pg_tzenum *dir)

入力と出力

  • **入力**: pg_tzenum *dir（走査状態を保持する構造体へのポインタ）
  • **出力**: 成功時は次に発見されたタイムゾーンの pg_tz 構造体へのポインタを返し、走査が完了した場合は NULL を返します。

前提条件と不変条件

  • dir は pg_tzenumerate_start によって適切に初期化されている必要があります。
  • dir 構造体内のメモリコンテキストは、palloc を使用するため適切に管理されている必要があります。
  • 走査中に dir->depth が MAX_TZDIR_DEPTH を超えることは許容されません。

アルゴリズムまたは振る舞い

1. dir->depth が 0 以上である限り、現在のディレクトリを ReadDir で走査します。 2. ディレクトリの終端に達した場合、FreeDir でディレクトリを閉じ、スタックを一つ戻して次の走査を継続します。 3. . で始まるファイルやディレクトリはスキップします。 4. サブディレクトリを発見した場合、スタックを深め（dir->depth++）、AllocateDir で新しいディレクトリを開いて走査を再開します。 5. ファイルを発見した場合、tzload を呼び出してタイムゾーンデータを読み込みます（キャッシュには格納しません）。 6. pg_tz_acceptable で閏秒を含むゾーンなどを除外し、有効な場合はその名前を dir->tz.TZname にコピーして返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/datetime.c:5354
  • **呼び出し先**: ReadDir, FreeDir, pfree, pstrdup, AllocateDir, tzload, pg_tz_acceptable

関連データ構造

  • pg_tzenum: 走査状態（ディレクトリスタック、現在の深さ、ベースパス等）を保持する構造体。
  • pg_tz: タイムゾーン定義を保持する構造体。

失敗モードと境界条件

  • **ディレクトリスタックオーバーフロー**: MAX_TZDIR_DEPTH を超える深い階層構造がある場合、ERROR を発生させます。
  • **ディレクトリオープン失敗**: AllocateDir が失敗した場合、ERROR を発生させます。
  • **読み込み失敗**: tzload が失敗したファイルは無視され、次のエントリへ進みます。
  • **無効なゾーン**: pg_tz_acceptable を満たさないゾーン（閏秒ゾーン等）は無視されます。

保守担当者向けメモ

  • 本関数はキャッシュを汚染しないよう tzload を直接使用しています。
  • palloc を使用しているため、呼び出し側で適切なメモリコンテキストがアクティブであることを確認してください。
  • dir->tz へのポインタを返しますが、これは pg_tzenum 構造体内部の領域を指しているため、次の呼び出しまでのみ有効です。
