cleanup_workspace

定義

  • 種別: function
  • 定義箇所: contrib/xml2/xpath.c:421

目的

xpath_workspace 構造体が保持する libxml2 関連のリソースを解放し、メモリリークを防ぐためのクリーンアップ関数です。

シグネチャまたは形状

static void cleanup_workspace(xpath_workspace *workspace)

入力と出力

  • **入力**: xpath_workspace *workspace （解放対象のワークスペース構造体へのポインタ）
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • 引数 workspace は有効なポインタである必要があります。
  • 関数実行後、workspace 内の各ポインタ（res, ctxt, doctree）は NULL にリセットされます。

アルゴリズムまたは振る舞い

1. workspace->res が存在する場合、xmlXPathFreeObject を呼び出して XPath 評価結果を解放し、res を NULL に設定します。 2. workspace->ctxt が存在する場合、xmlXPathFreeContext を呼び出して XPath コンテキストを解放し、ctxt を NULL に設定します。 3. workspace->doctree が存在する場合、xmlFreeDoc を呼び出して XML ドキュメントツリーを解放し、doctree を NULL に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/xml2/xpath.c 内の各 XPath 評価関数（xpath_nodeset, xpath_list, xpath_string, xpath_number, xpath_bool 等）および内部処理から呼び出されます。
  • **呼び出し先**: libxml2 ライブラリの xmlXPathFreeObject, xmlXPathFreeContext, xmlFreeDoc を呼び出します。

関連データ構造

  • xpath_workspace: XML ドキュメント、XPath コンテキスト、評価結果を保持する管理構造体。

失敗モードと境界条件

  • workspace が指す各メンバが既に NULL である場合、対応する解放関数はスキップされ、安全に処理が継続されます。

保守担当者向けメモ

  • libxml2 のメモリ管理は PostgreSQL のメモリコンテキストとは独立しているため、本関数による明示的な解放が必須です。
  • 新たなリソースを xpath_workspace に追加した場合は、必ず本関数で解放処理を追記してください。
