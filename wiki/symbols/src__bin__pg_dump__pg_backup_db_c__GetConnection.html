GetConnection

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_db.c:192

目的

アーカイブハンドルから、現在保持しているデータベース接続（PGconn）を取得する。

シグネチャまたは形状

PGconn *GetConnection(Archive *AHX)

入力と出力

  • **入力**: Archive *AHX（ArchiveHandle 構造体へのポインタとして扱われる）
  • **出力**: PGconn *（libpq の接続オブジェクト、または接続が確立されていない場合は NULL）

前提条件と不変条件

  • 入力された AHX は、内部で ArchiveHandle 型にキャストされるため、有効な ArchiveHandle 構造体へのポインタである必要がある。

アルゴリズムまたは振る舞い

1. 引数 AHX を ArchiveHandle * 型にキャストする。 2. キャストされた構造体内の connection メンバを直接参照し、その値を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し先**: なし（構造体のメンバアクセスのみ）。
  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c 内の各関数（fsstate->conn 等の取得に使用）。
  • contrib/postgres_fdw/connection.c（接続状態の管理や参照カウントに関連する処理）。
  • ※注: pg_dump モジュール内での直接的な呼び出し元については、提供されたソース抜粋からは特定できない。

関連データ構造

  • Archive: アーカイブ処理の抽象インターフェース。
  • ArchiveHandle: pg_backup_archiver.h で定義される、接続情報を含む内部構造体。
  • PGconn: libpq のデータベース接続オブジェクト。

失敗モードと境界条件

  • AHX が NULL の場合、構造体のメンバアクセスによりセグメンテーション違反が発生する可能性がある。
  • 接続が確立されていない場合、AH->connection は NULL を保持しているため、そのまま NULL が返される。

保守担当者向けメモ

  • 本関数は単なるゲッターであり、接続の確立や再接続は行わない。接続のライフサイクル管理が必要な場合は ConnectDatabaseAhx や ReconnectToServer を参照すること。
