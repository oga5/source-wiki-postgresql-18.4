gbt_float4lt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_float4.c:48

目的

float4 型の2つの値が「より小さい（less than）」という関係にあるかを判定し、GiSTインデックスの比較処理をサポートします。

シグネチャまたは形状

bool gbt_float4lt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の float4 値へのポインタ。
  • b: 比較対象の float4 値へのポインタ。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • bool: *a < *b が真であれば true、それ以外は false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な float4 型の値を指している必要があります。

アルゴリズムまたは振る舞い

1. ポインタ a および b を const float4 * 型にキャストします。 2. それぞれの指す値をデリファレンスし、標準的な比較演算子 < を用いて大小比較を行います。 3. 比較結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_float4.c 内の静的変数 tinfo の初期化において参照されています。
  • **呼び出し先**: 本関数は直接的な関数呼び出しを行わず、C言語の組み込み比較演算子のみを使用します。

関連データ構造

  • float4: PostgreSQLの単精度浮動小数点数型。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、未定義の動作（セグメンテーション違反など）を引き起こす可能性があります。
  • float4 値が NaN（Not a Number）である場合の比較結果は、標準的な浮動小数点数の比較規則に従います。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で定義された汎用的な数値処理ロジックから呼び出されることを想定したラッパー関数です。
  • 比較ロジックを変更する場合は、他の比較関数（gbt_float4gt, gbt_float4le 等）との整合性に注意してください。
