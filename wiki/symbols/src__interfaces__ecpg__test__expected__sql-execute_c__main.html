main

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-execute.c:30

目的

ECPG（Embedded SQL for C）における動的SQL実行機能（EXECUTE IMMEDIATE、PREPARE、EXECUTE、カーソル操作）の正当性を検証する回帰テストのエントリポイント。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（引数なし）
  • 出力: 終了ステータス（int）

前提条件と不変条件

  • データベース ecpg1_regression への接続が成功していること。
  • sqlca 構造体が各SQL操作後に更新され、エラーチェック（sqlca.sqlcode < 0）が行われること。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、データベースへ接続する。 2. **DDL実行**: test テーブルを作成し、コミットする。 3. **即時実行テスト**: EXECUTE IMMEDIATE を使用して、リテラルおよびホスト変数 command に格納された INSERT 文を実行する。 4. **準備済み実行テスト**: PREPARE でクエリを準備し、EXECUTE でパラメータ（increment）をバインドして実行する。 5. **カーソル操作**: DECLARE CURSOR でクエリを定義し、FETCH を使用して結果をホスト変数（name, amount, letter）に格納する。 6. **結果出力**: フェッチした行をループで走査し、標準出力へ表示する。 7. **終了処理**: カーソルのクローズ、プリペアドステートメントの解放を行う。

呼び出し元と呼び出し先

  • 呼び出し元: 不明（テスト実行環境から呼び出されるエントリポイント）
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGdo, ECPGtrans, ECPGprepare, ECPGdeallocate, ECPGprepared_statement, sqlprint, sprintf, printf, strncpy

関連データ構造

  • sqlca: SQL実行結果のステータスやエラー情報を保持する構造体。
  • amount, increment, name, letter, command: SQL操作に使用されるホスト変数。

失敗モードと境界条件

  • 各SQL操作後に sqlca.sqlcode < 0 を判定し、エラーが発生した場合は sqlprint() を呼び出してエラー情報を出力する。
  • カーソル操作において、フェッチする行数（8行）がバッファサイズと一致している必要がある。

保守担当者向けメモ

  • 本関数はプリプロセッサの出力結果であるため、直接編集せず、元の .pgc ファイルを修正すること。
  • sqlca.sqlerrd[2] を参照して影響を受けた行数を確認するパターンは、ECPGテストの標準的な検証手法である。
