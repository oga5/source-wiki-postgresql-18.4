check_exclusion_or_unique_constraint

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execIndexing.c:704

目的

テーブルへのタプル挿入時、指定されたインデックスに基づいて一意性制約または排他制約の違反を検出する。

シグネチャまたは形状

static bool check_exclusion_or_unique_constraint(
    Relation heap, Relation index, IndexInfo *indexInfo,
    ItemPointer tupleid, const Datum *values, const bool *isnull,
    EState *estate, bool newIndex, CEOUC_WAIT_MODE waitMode,
    bool violationOK, ItemPointer conflictTid
)

入力と出力

  • **入力**: 対象リレーション（heap, index）、インデックス情報、挿入対象のキー値（values, isnull）、実行状態（estate）、待機モード（waitMode）等。
  • **出力**: 制約違反がなければ true を返し、違反がある場合は呼び出し元の設定に応じてエラーを投げるか、競合情報を conflictTid に格納して false を返す。

前提条件と不変条件

  • ii_WithoutOverlaps が設定されている場合、空の範囲（empty range/multirange）の挿入を禁止するチェックが先行する。
  • ii_NullsNotDistinct が偽の場合、入力値に NULL が含まれると制約チェックは即座にパスする。

アルゴリズムまたは振る舞い

1. **制約情報の特定**: indexInfo から排他制約または一意性制約の演算子・戦略を取得する。 2. **空範囲チェック**: WITHOUT OVERLAPS 制約がある場合、ExecWithoutOverlapsNotEmpty を呼び出し、空の範囲が含まれていないか検証する。 3. **スキャン準備**: DirtySnapshot を使用して、未コミットのタプルを含むインデックススキャンを初期化する。 4. **スキャン実行**: インデックスをスキャンし、挿入対象のキー値と一致する既存タプルを検索する。 5. **自己除外**: 挿入対象のタプル自身（tupleid）は競合判定から除外する。 6. **再チェック**: 損失のあるインデックススキャン（lossy scan）の場合は index_recheck_constraint を用いて再検証を行う。 7. **競合処理**: 競合が見つかった場合、トランザクションの可視性に応じて待機処理（waitMode に従う）またはエラー処理を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execIndexing.c 内の ExecCheckIndexConstraints 等から呼び出される。
  • **呼び出し先**: index_beginscan, index_rescan, index_getnext_slot, ExecWithoutOverlapsNotEmpty, index_recheck_constraint 等。

関連データ構造

  • IndexInfo: インデックスのメタデータと制約情報。
  • CEOUC_WAIT_MODE: 排他制約チェック時の待機動作を制御する列挙型。
  • SnapshotData: 可視性判定に使用するスナップショット。

失敗モードと境界条件

  • **デッドロック**: 競合する他トランザクションの終了を待機する際、デッドロックが発生する可能性がある。
  • **損失のあるスキャン**: index_scan->xs_recheck が真の場合、インデックスの不完全な一致を補完するための再チェックが必須となる。
  • **NULL値**: ii_NullsNotDistinct の設定により、NULL の扱いが異なる。

保守担当者向けメモ

  • 本関数は INSERT ... ON CONFLICT の競合検出ロジックの中核を担う。
  • 競合検出時に retry ラベルへ戻ることで、待機後の再スキャンを実現している。
  • 呼び出し元との詳細な依存関係は、src/backend/executor/execIndexing.c 内の他の関数との連携を考慮すること。
