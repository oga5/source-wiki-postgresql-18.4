pgaio_uring_shmem_init

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_io_uring.c:278

目的

PostgreSQL のバックエンドプロセスが使用する io_uring インスタンスの共有メモリ領域を確保し、各コンテキストの初期化を行う。

シグネチャまたは形状

static void pgaio_uring_shmem_init(bool first_time)

入力と出力

  • **入力**: first_time (bool) - 共有メモリの初期割り当て時か否かを示すフラグ。
  • **出力**: なし（副作用として共有メモリの確保と io_uring キューの初期化を行う）。

前提条件と不変条件

  • ShmemInitStruct を介して "AioUringContext" という名前の共有メモリ領域が確保される。
  • pgaio_uring_caps.mem_init_size が正の場合、io_uring のリングデータはページ境界にアライメントされる。
  • 実行環境の RLIMIT_NOFILE が、全バックエンド分の io_uring インスタンスを保持するのに十分なファイルディスクリプタ数を提供している必要がある。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を呼び出し、PgAioUringContext 配列用の共有メモリを確保する。既に確保済みであれば即座に終了する。 2. pgaio_uring_caps.mem_init_size が有効な場合、共有メモリ領域からリングデータ用のメモリを切り出し、ページ境界にアライメントする。 3. 全バックエンド数 (TotalProcs) 分のループを実行し、各 PgAioUringContext に対して以下の処理を行う：

  • io_uring_queue_init_mem (サポート時) または io_uring_queue_init を呼び出し、io_uring インスタンスを初期化する。
  • 初期化失敗時には、errno に基づいて適切なエラーコード（EPERM, EMFILE, ENOSYS 等）とヒントを付与して ereport(ERROR, ...) を発行する。
  • LWLockInitialize を呼び出し、各コンテキストの completion_lock を初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の参照情報に基づくと、pgaio_uring_ops 構造体の .shmem_init フィールドに登録されている。
  • **呼び出し先**: ShmemInitStruct, pgaio_uring_procs, pgaio_uring_shmem_size, pgaio_uring_context_shmem_size, pgaio_uring_ring_shmem_size, sysconf, TYPEALIGN, io_uring_queue_init_mem, io_uring_queue_init, LWLockInitialize。

関連データ構造

  • PgAioUringContext: 各バックエンドの io_uring 状態とロックを保持する構造体。
  • pgaio_uring_contexts: 共有メモリ上に確保された PgAioUringContext の配列。
  • pgaio_uring_caps: io_uring の機能サポート状況を保持する構造体。

失敗モードと境界条件

  • io_uring_queue_init が失敗した場合、EPERM (権限不足)、EMFILE (ファイルディスクリプタ制限超過)、ENOSYS (カーネル未サポート) に応じたエラーメッセージとヒントを表示してプロセスを停止する。
  • TotalProcs が過大である場合、ファイルディスクリプタ制限により初期化が失敗する可能性がある。

保守担当者向けメモ

  • io_uring のワーカー共有機能の活用や、RLIMIT_NOFILE の自動調整については将来的な検討課題である。
  • io_uring_queue_init_mem のサポート状況は実行時に pgaio_uring_check_capabilities で判定される。
