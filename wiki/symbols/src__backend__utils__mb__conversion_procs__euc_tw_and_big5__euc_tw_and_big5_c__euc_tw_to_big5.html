euc_tw_to_big5

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_tw_and_big5/euc_tw_and_big5.c:51

目的

EUC_TW エンコーディングから BIG5 エンコーディングへの文字列変換を行うための PostgreSQL 変換プロシージャです。

シグネチャまたは形状

Datum euc_tw_to_big5(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の引数を受け取ります。
  • src (CSTRING): 変換元の文字列バッファ。
  • dest (CSTRING): 変換後の文字列を格納するバッファ。
  • len (INT32): 入力文字列のバイト長。
  • noError (BOOL): 変換エラー発生時の挙動制御フラグ。
  • **出力**: 変換されたバイト数を Datum 型（INT32）として返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_EUC_TW, PG_BIG5) により、変換元が EUC_TW、変換先が BIG5 であることが保証されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得します。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングの妥当性を検証します。 3. 内部関数 euc_tw2big5 を呼び出し、実際のバイト列変換を実行します。 4. 変換されたバイト数を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 により登録）。
  • **呼び出し先**: euc_tw2big5（実際の変換ロジック）。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約で使用される引数構造体。

失敗モードと境界条件

  • noError が true の場合、変換エラーが発生しても例外を発生させず、処理を中断してその時点までの変換バイト数を返します。
  • noError が false の場合、不正な文字シーケンスに対してはエラーが発生する可能性があります（詳細は euc_tw2big5 の実装に依存）。

保守担当者向けメモ

  • 本関数はインターフェース層であり、具体的な変換ロジックは euc_tw2big5 に委譲されています。
  • 変換テーブルや計算式を変更する場合は、euc_tw2big5 または関連するヘルパー関数を確認してください。
