GenericXLogStart

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/generic_xlog.c:268

目的

汎用WALレコード構築のための状態管理構造体（GenericXLogState）を初期化し、メモリを割り当てる。

シグネチャまたは形状

GenericXLogState *GenericXLogStart(Relation relation)

入力と出力

  • **入力**: relation (対象となるリレーション)
  • **出力**: 初期化された GenericXLogState 構造体へのポインタ。

前提条件と不変条件

  • palloc_aligned により、PG_IO_ALIGN_SIZE にアライメントされたメモリが確保される。
  • state->isLogged は、対象リレーションがWAL記録を必要とするかどうかに基づいて設定される。
  • state->pages 配列内の各要素について、buffer は InvalidBuffer で初期化され、image は内部の images 配列を指すように設定される。

アルゴリズムまたは振る舞い

1. GenericXLogState サイズ分のメモリをアライメントを考慮して確保する。 2. RelationNeedsWAL(relation) を呼び出し、WAL記録の要否を判定して isLogged に格納する。 3. MAX_GENERIC_XLOG_PAGES 分のループを回し、各ページスロットのバッファを InvalidBuffer にリセットし、イメージ用データ領域を初期化する。 4. 初期化済みの状態構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blinsert.c
  • contrib/bloom/blutils.c
  • contrib/bloom/blvacuum.c
  • **呼び出し先**:
  • palloc_aligned
  • RelationNeedsWAL

関連データ構造

  • GenericXLogState: 汎用WALレコード構築中の状態を保持する構造体。
  • Relation: PostgreSQLのリレーション定義。

失敗モードと境界条件

  • メモリ不足時には palloc_aligned がエラーを発生させる。
  • MAX_GENERIC_XLOG_PAGES を超えるページ登録は、後続の GenericXLogRegisterBuffer 呼び出し時に制限を受ける。

保守担当者向けメモ

  • 本関数はWAL構築の開始点であり、ここで確保されたメモリは GenericXLogFinish または GenericXLogAbort で解放される必要がある。
  • GenericXLogState はアライメントされているため、構造体のメモリレイアウト変更時には注意が必要。
