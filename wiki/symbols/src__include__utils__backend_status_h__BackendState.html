BackendState

定義

  • 種別: enum
  • 定義箇所: src/include/utils/backend_status.h:24

目的

PostgreSQLの各バックエンドプロセスにおける現在のライフサイクル状態を定義し、共有メモリ上でプロセスごとの活動状況を監視するための識別子として機能します。

シグネチャまたは形状

typedef enum BackendState
{
    STATE_UNDEFINED,
    STATE_STARTING,
    STATE_IDLE,
    STATE_RUNNING,
    STATE_IDLEINTRANSACTION,
    STATE_FASTPATH,
    STATE_IDLEINTRANSACTION_ABORTED,
    STATE_DISABLED,
} BackendState;

入力と出力

  • 入力: pgstat_report_activity 関数などの状態更新APIにおいて、現在のプロセス状態を示す引数として使用されます。
  • 出力: PgBackendStatus 構造体の st_state フィールドとして保持され、統計情報システムや監視ツールから参照されます。

前提条件と不変条件

  • 状態の更新は、PGSTAT_BEGIN_WRITE_ACTIVITY および PGSTAT_END_WRITE_ACTIVITY マクロで保護されたクリティカルセクション内で行う必要があります。
  • 読み取り側は、チェンジカウンタ（st_changecount）の整合性を確認することで、更新中の不整合な状態を読み取らないことが保証されます。

アルゴリズムまたは振る舞い

バックエンドプロセスの活動変化に応じて、pgstat_report_activity を通じて適切な BackendState 値が共有メモリ上の PgBackendStatus に書き込まれます。これにより、外部プロセスは各バックエンドが現在「実行中」か「待機中」か等の動的な状態をリアルタイムに把握可能です。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/activity/backend_status.c 内の pgstat_report_activity 関数が、状態更新の主要なインターフェースとして本シンボルを利用しています。
  • 呼び出し先: 本シンボル自体は列挙型であり、直接的な呼び出し先はありません。

関連データ構造

  • PgBackendStatus: 各バックエンドの状態を保持する共有メモリ上の構造体。
  • st_state: PgBackendStatus 内で BackendState 型として定義されているフィールド。

失敗モードと境界条件

  • 状態更新マクロ内でエラーが発生した場合、PANIC を引き起こす可能性があるため、状態遷移を伴うコードは極力短く記述する必要があります。
  • 状態遷移の順序や整合性は、呼び出し側のロジックに依存します。

保守担当者向けメモ

  • 本シンボルは pgstat が管理する累積統計とは異なり、プロセスの「現在の状態」を追跡するものです。
  • 状態の追加や変更を行う際は、backend_status.c 側の関連処理が正しく追従しているか確認してください。
