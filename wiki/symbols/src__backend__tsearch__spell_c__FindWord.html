FindWord

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/spell.c:604

目的

ISpell/Hunspell 辞書構造体（Trie）を探索し、指定された単語が辞書に存在するか、および特定の接辞フラグ条件を満たすかを判定します。

シグネチャまたは形状

int FindWord(IspellDict *Conf, const char *word, const char *affixflag, int flag)

入力と出力

  • **入力**:
  • Conf: 辞書データ構造体へのポインタ。
  • word: 検索対象の単語文字列。
  • affixflag: 適用すべき接辞フラグ。
  • flag: 複合語処理のためのビットマスク（FF_COMPOUNDFLAGMASK でフィルタリング）。
  • **出力**:
  • 1: 単語が見つかり、かつ接辞フラグ条件に合致した場合。
  • 0: 単語が見つからない、または条件に合致しない場合。

前提条件と不変条件

  • Conf->Dictionary は有効な Trie 構造であること。
  • flag は FF_COMPOUNDFLAGMASK に基づいて評価される。
  • 探索中、StopMiddle->compoundflag と flag の論理積により、複合語としての有効性が検証される。

アルゴリズムまたは振る舞い

1. Conf->Dictionary をルートとして、単語の各文字を Trie 上で順次探索します。 2. 各ノードにおいて、SPNodeData の配列を二分探索し、現在の文字と一致するノードを特定します。 3. 単語の末尾に到達した際、そのノードが単語の終端（isword）であることを確認します。 4. 複合語フラグのチェックを行います：

  • flag が 0 の場合、FF_COMPOUNDONLY フラグを持つ単語は除外されます。
  • flag が指定されている場合、単語の compoundflag との論理積が 0 であれば除外されます。

    • 5. 最後に IsAffixFlagInUse を呼び出し、指定された affixflag が当該単語に関連付けられているかを確認します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/spell.c 内の複数箇所（単語の正規化および複合語分割処理中）。
  • **呼び出し先**:
  • IsAffixFlagInUse: 接辞フラグの照合に使用。

関連データ構造

  • IspellDict: 辞書全体の状態を保持。
  • SPNode / SPNodeData: Trie 構造を構成するノードおよびデータ。

失敗モードと境界条件

  • 単語が辞書に存在しない場合、0 を返します。
  • flag 条件に適合しない場合、単語が存在しても 0 を返します。
  • word が空文字列の場合、探索は行われません。

保守担当者向けメモ

  • 本関数は辞書探索の核心部であり、頻繁に呼び出されるため、二分探索の効率が検索性能に直結します。
  • IsAffixFlagInUse の呼び出しコストがボトルネックとなる可能性があるため、パフォーマンスチューニング時はこの経路を考慮してください。
