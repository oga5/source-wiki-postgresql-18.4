TuplesortIndexBTreeArg

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/sort/tuplesortvariants.c:132

目的

B-Treeインデックスのソート処理において、インデックス固有のパラメータと一意性制約の情報を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	TuplesortIndexArg index;

	bool		enforceUnique;	/* 重複タプル検出時にエラーを発生させるか */
	bool		uniqueNullsNotDistinct; /* 一意性制約におけるNULLの扱い */
} TuplesortIndexBTreeArg;

入力と出力

本構造体は、ソート状態（Tuplesortstate）の一部としてメモリ上に割り当てられ、ソート処理の初期化時に設定されます。直接的な関数入出力ではなく、ソートエンジンが参照する設定データとして機能します。

前提条件と不変条件

  • TuplesortIndexArg を継承（包含）しており、インデックスソートに必要な共通情報を前提とします。
  • enforceUnique が真の場合、ソート過程で重複キーが検出されるとエラーが発生します。

アルゴリズムまたは振る舞い

本構造体は、tuplesort_begin_index_btree 等の初期化関数によってメモリ確保および初期化が行われます。ソートエンジンは、この構造体を参照することで、一意性チェックの要否やNULLの比較ルールを動的に切り替えます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/tuplesortvariants.c 内の tuplesort_begin_index_btree 関連関数が、本構造体を palloc で確保し初期化します。
  • **呼び出し先**: ソート処理の実行中、base->arg を通じて本構造体にアクセスし、一意性制約の判定ロジックに利用されます。

関連データ構造

  • TuplesortIndexArg: インデックスソートの基底となる共通引数構造体。
  • Tuplesortstate: 本構造体を内部に保持し、ソート処理の全状態を管理するメイン構造体。

失敗モードと境界条件

  • メモリ不足時: palloc による確保失敗が発生する可能性があります。
  • 一意性制約違反: enforceUnique が真の状態で重複タプルが投入された場合、ソート処理中にエラーが送出されます。

保守担当者向けメモ

  • 本構造体は tuplesortvariants.c 内のインデックスソート専用の拡張データです。一意性制約のロジックを変更する場合は、この構造体のフラグと、それを利用する比較関数の両方を更新する必要があります。
