ginInsertValue

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginbtree.c:815

目的

GINインデックスのB-tree構造に対して、指定されたデータを挿入します。必要に応じてページ分割を伴う再帰的なツリー更新を管理します。

シグネチャまたは形状

void ginInsertValue(GinBtree btree, GinBtreeStack *stack, void *insertdata, GinStatsData *buildStats)

入力と出力

  • **入力**:
  • btree: GIN B-treeの操作コンテキスト。
  • stack: 挿入対象のリーフページを含むツリーパス情報。
  • insertdata: 挿入するデータ。
  • buildStats: インデックス構築統計情報（オプション）。
  • **出力**: なし（副作用としてツリー構造を更新）。

前提条件と不変条件

  • stack は挿入対象のリーフページを指している必要があります。
  • 挿入処理中に遭遇した不完全な分割（GIN_INCOMPLETE_SPLIT）は、本関数内で自動的に修復されます。

アルゴリズムまたは振る舞い

1. **不完全分割の修復**: stack が指すリーフページが不完全な分割状態にある場合、ginFinishOldSplit を呼び出して整合性を確保します。 2. **ページへの挿入**: ginPlaceToPage を呼び出し、ページ内にデータを挿入します。 3. **完了判定**:

  • 挿入が成功（ページ溢れなし）した場合、バッファをアンロックし、スタックを解放します。
  • ページ溢れが発生した場合、ginFinishSplit を呼び出し、親ページへのダウンリンク挿入を含む再帰的な分割処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gindatapage.c
  • src/backend/access/gin/gininsert.c
  • **呼び出し先**:
  • ginFinishOldSplit
  • ginPlaceToPage
  • ginFinishSplit
  • freeGinBtreeStack
  • LockBuffer

関連データ構造

  • GinBtree: GIN B-treeの操作を定義する構造体。
  • GinBtreeStack: ツリー降下時のパスを保持するスタック構造体。
  • GinStatsData: インデックス構築時の統計情報。

失敗モードと境界条件

  • ページ容量不足: ginPlaceToPage が false を返すことで検知され、ginFinishSplit による分割処理へ移行します。
  • 不完全な分割: ページフラグ GIN_INCOMPLETE_SPLIT が設定されている場合、処理の冒頭で修復が試みられます。

保守担当者向けメモ

  • 本関数はエントリツリーとデータツリーの両方で共通して使用されます。
  • ginPlaceToPage の戻り値に基づいて後続の分割処理を制御する設計となっており、再帰的なツリー更新の起点となります。
