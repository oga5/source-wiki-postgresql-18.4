tsquery_not

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_op.c:158

目的

入力された tsquery 型の値に対して論理否定（NOT）演算を適用し、新しい tsquery を生成する。

シグネチャまたは形状

Datum tsquery_not(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して tsquery 型の引数（a）を受け取る。
  • **出力**: 否定演算が適用された新しい tsquery を Datum 型として返す。

前提条件と不変条件

  • 入力 a の size が 0 の場合、演算を行わず入力値をそのまま返す。
  • 内部的に QTNode ツリー構造へ変換して処理を行うため、メモリ確保が必要となる。
  • QTN_NEEDFREE フラグが設定されたノードは、処理終了後に QTNFree で解放される必要がある。

アルゴリズムまたは振る舞い

1. PG_GETARG_TSQUERY_COPY(0) で入力 tsquery を取得する。 2. size が 0 であれば即座に終了する。 3. QTNode を割り当て、OP_NOT 演算子ノードとして初期化する。 4. QT2QTN を用いて入力 tsquery をツリー構造に変換し、NOT ノードの子として接続する。 5. QTN2QT を用いてツリー構造を再び tsquery 型へシリアライズする。 6. 一時的な QTNode を QTNFree で解放し、入力引数のメモリを PG_FREE_IF_COPY で適切に管理する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の明示的な呼び出し元は確認されていない（SQL 演算子 ! の実装として呼び出される想定）。
  • **呼び出し先**:
  • QT2QTN: tsquery をツリー構造へ変換。
  • QTN2QT: ツリー構造を tsquery へ変換。
  • QTNFree: QTNode ツリーのメモリ解放。
  • palloc0: メモリ確保。

関連データ構造

  • TSQuery: 全文検索クエリの内部表現。
  • QTNode: tsquery を操作するためのツリー構造体。
  • QueryItem: tsquery 内の個々のノード（演算子や単語）。

失敗モードと境界条件

  • 入力が空（size == 0）の場合は、演算をスキップしてそのまま返す。
  • メモリ不足時には palloc0 がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は tsquery を一度ツリー構造（QTNode）に展開してから再シリアライズするため、複雑なクエリではオーバーヘッドが発生する。
  • QTN_NEEDFREE フラグの管理がメモリリーク防止の鍵となるため、ツリー構造を変更する際は注意が必要。
