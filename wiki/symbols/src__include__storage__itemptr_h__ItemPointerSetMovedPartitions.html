ItemPointerSetMovedPartitions

定義

  • 種別: function
  • 定義箇所: src/include/storage/itemptr.h:209

目的

指定された ItemPointerData を、タプルがパーティション間を移動したことを示す特殊な状態に設定します。

シグネチャまたは形状

static inline void ItemPointerSetMovedPartitions(ItemPointerData *pointer)

入力と出力

  • **入力**: pointer (ItemPointerData *) - 状態を更新する対象のアイテムポインタへのポインタ。
  • **出力**: なし（副作用として pointer が指すメモリの内容が書き換えられます）。

前提条件と不変条件

  • pointer は有効な ItemPointerData 構造体を指している必要があります。
  • 本関数は、通常のタプル位置ではなく、パーティション間移動という特殊な状態を表現するために使用されます。

アルゴリズムまたは振る舞い

内部的に ItemPointerSet を呼び出し、ブロック番号に MovedPartitionsBlockNumber を、オフセット番号に MovedPartitionsOffsetNumber を設定します。これにより、当該ポインタが通常のデータページを指していないことが示されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/access/htup_details.h:493 にて、tup->t_ctid に対して呼び出されています。
  • **呼び出し先**:
  • ItemPointerSet（インライン関数）

関連データ構造

  • ItemPointerData: 物理的なタプル位置を保持する 6 バイトの構造体。
  • MovedPartitionsBlockNumber: 移動状態を示すために使用される特殊なブロック番号。
  • MovedPartitionsOffsetNumber: 移動状態を示すために使用される特殊なオフセット番号。

失敗モードと境界条件

  • pointer が NULL の場合、未定義の動作を引き起こします。
  • 本関数は Assert 等による検証を行わないため、呼び出し側でポインタの有効性を保証する必要があります。

保守担当者向けメモ

  • 本関数は ItemPointerIndicatesMovedPartitions と対になる操作です。状態判定ロジックを変更する場合は、必ず両方の関数を整合させる必要があります。
  • ItemPointerData のメモリレイアウトはパディングを抑制しているため、この構造体への直接的な操作はバイナリ互換性に影響を与える可能性があります。
