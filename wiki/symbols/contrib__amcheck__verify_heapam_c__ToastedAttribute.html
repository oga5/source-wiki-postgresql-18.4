ToastedAttribute

定義

  • **種別**: struct
  • **定義箇所**: contrib/amcheck/verify_heapam.c:74

目的

ヒープテーブル内のTOAST（外部格納）属性を検証する際、その属性のメタデータと格納場所を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct ToastedAttribute
{
    struct varatt_external toast_pointer;
    BlockNumber blkno;      /* メインテーブル内のブロック番号 */
    OffsetNumber offnum;    /* メインテーブル内のオフセット番号 */
    AttrNumber   attnum;    /* メインテーブル内の属性番号 */
} ToastedAttribute;

入力と出力

本構造体はデータ保持用であり、関数ではありません。検証プロセスにおいて、ヒープ走査中に収集されたTOAST属性情報が本構造体に格納され、後続の check_toasted_attribute 等の検証関数へ渡されます。

前提条件と不変条件

  • toast_pointer は、外部格納されたデータへのポインタ情報を含みます。
  • blkno, offnum, attnum は、当該TOAST属性が本来属しているメインテーブル内の位置を正確に指し示している必要があります。

アルゴリズムまたは振る舞い

1. ヒープ走査中にTOAST属性が検出されると、palloc0 を用いて本構造体がメモリ確保されます。 2. 属性のメタデータが構造体にコピーされます。 3. ページ内の全タプル処理後、蓄積された ToastedAttribute リストが check_toasted_attribute に渡され、TOASTテーブルとの整合性が検証されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_heapam.c 内のヒープ走査ロジック（check_tuple 等）が本構造体を生成します。
  • **呼び出し先**: check_toasted_attribute、report_toast_corruption 等の検証関数が本構造体を引数として受け取ります。

関連データ構造

  • HeapCheckContext: 検証全体の状態を保持する構造体であり、本構造体のリストを管理します。
  • varatt_external: TOASTデータの外部参照情報を保持する標準的なPostgreSQL構造体。

失敗モードと境界条件

  • メモリ不足時: palloc0 によるメモリ確保が失敗する可能性があります。
  • 破損データ: toast_pointer の内容が不正な場合、後続の check_toasted_attribute で検証エラーが報告されます。

保守担当者向けメモ

  • 本構造体は verify_heapam.c 内でのみ使用される一時的なデータ保持用です。
  • TOAST検証のロジックを変更する場合は、blkno や offnum が指すメインテーブルのコンテキストが正しく維持されているか確認してください。
