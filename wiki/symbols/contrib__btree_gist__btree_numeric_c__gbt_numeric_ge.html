gbt_numeric_ge

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_numeric.c:36

目的

PostgreSQL の numeric 型に対する「以上 (>=)」の比較演算を GiST インデックスのサポート関数として提供します。

シグネチャまたは形状

bool gbt_numeric_ge(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象となる numeric 型のデータへのポインタ。
  • collation: 照合順序（本関数では使用されません）。
  • flinfo: 関数呼び出し情報（本関数では使用されません）。
  • **出力**:
  • a >= b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な numeric 型のデータ構造を指している必要があります。
  • PostgreSQL のコア関数 numeric_ge が正常に動作する環境である必要があります。

アルゴリズムまたは振る舞い

1. a と b を PointerGetDatum を用いて Datum 型に変換します。 2. DirectFunctionCall2 を介して PostgreSQL コアの numeric_ge 関数を呼び出します。 3. 戻り値を DatumGetBool で bool 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_numeric.c 内の tinfo 構造体（gbtree_vinfo）の初期化時に参照されています。
  • **呼び出し先**: PostgreSQL コアの numeric_ge 関数。

関連データ構造

  • gbtree_vinfo: numeric 型の比較関数群を保持し、GiST インデックス操作へ提供するための構造体。

失敗モードと境界条件

  • numeric 型の比較ルールに従い、入力が NaN を含む場合の挙動はコアの numeric_ge に依存します。
  • メモリ上の不正なポインタが渡された場合、DirectFunctionCall2 呼び出し時にセグメンテーション違反等のエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は numeric 型専用の比較アダプターです。ロジックの変更が必要な場合は、コアの numeric_ge の仕様を確認してください。
  • collation および flinfo 引数は現在の実装では使用されていませんが、インターフェースの互換性維持のためシグネチャに含まれています。
