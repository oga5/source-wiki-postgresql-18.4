pgstat_report_vacuum

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_relation.c:209

目的

VACUUM実行後の統計情報を共有メモリ上のリレーション統計エントリに反映し、関連するカウンタを更新します。

シグネチャまたは形状

void pgstat_report_vacuum(Oid tableoid, bool shared, PgStat_Counter livetuples, PgStat_Counter deadtuples, TimestampTz starttime)

入力と出力

  • **入力**:
  • tableoid: 対象リレーションのOID。
  • shared: 共有カタログか否かを示すフラグ。
  • livetuples: VACUUM後に生存しているタプル数。
  • deadtuples: VACUUM後に削除されたタプル数。
  • starttime: VACUUM開始時刻。
  • **出力**: なし（副作用として共有メモリ上の統計情報を更新）。

前提条件と不変条件

  • pgstat_track_counts が有効であること。無効な場合は即座にリターンします。
  • 呼び出し元は、VACUUM処理の開始時刻を starttime として保持している必要があります。

アルゴリズムまたは振る舞い

1. pgstat_track_counts を確認し、無効なら終了。 2. 現在時刻と starttime から経過時間を算出。 3. pgstat_get_entry_ref_locked を呼び出し、対象リレーションの統計エントリをロックして取得。 4. livetuples および deadtuples を統計エントリに書き込み、ins_since_vacuum を 0 にリセット。 5. 実行プロセスが AutoVacuum ワーカーか否かを判定し、対応する統計カウンタ（last_autovacuum_time / last_vacuum_time 等）と累積時間を更新。 6. pgstat_unlock_entry でロックを解放。 7. pgstat_flush_io および pgstat_flush_backend を呼び出し、IO統計をフラッシュ。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/vacuumlazy.c:938 (直接的な証拠)
  • **呼び出し先**:
  • GetCurrentTimestamp
  • TimestampDifferenceMilliseconds
  • pgstat_get_entry_ref_locked
  • AmAutoVacuumWorkerProcess
  • pgstat_unlock_entry
  • pgstat_flush_io
  • pgstat_flush_backend

関連データ構造

  • PgStat_EntryRef: 統計エントリへの参照とロック状態を管理。
  • PgStatShared_Relation: 共有メモリ上に保持されるリレーション統計構造体。
  • PgStat_StatTabEntry: リレーションごとの統計値（タプル数、VACUUM回数など）。

失敗モードと境界条件

  • pgstat_track_counts が無効な場合、統計更新は行われません。
  • ロック取得失敗時の挙動については、pgstat_get_entry_ref_locked の仕様に依存します。

保守担当者向けメモ

  • 本関数はVACUUM終了時に呼ばれますが、IO統計のフラッシュを即座に行うため、頻繁な呼び出しはパフォーマンスに影響を与える可能性があります。
  • ins_since_vacuum のリセットは、次回のAutoVacuumトリガー判定に影響します。
