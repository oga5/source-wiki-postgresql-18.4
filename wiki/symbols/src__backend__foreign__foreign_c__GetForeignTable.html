GetForeignTable

定義

  • 種別: function
  • 定義箇所: src/backend/foreign/foreign.c:254

目的

指定された OID に対応する外部テーブルの定義情報をシステムカタログから取得し、メモリ上の ForeignTable 構造体として構築する。

シグネチャまたは形状

ForeignTable *GetForeignTable(Oid relid)

入力と出力

  • **入力**: relid (外部テーブルの OID)
  • **出力**: 外部テーブルの定義を保持する ForeignTable 構造体へのポインタ。

前提条件と不変条件

  • relid は pg_foreign_table システムカタログに存在する有効な外部テーブルの OID である必要がある。
  • 呼び出し側は、返された ForeignTable 構造体が不要になった時点で適切にメモリを解放する責任を負う。

アルゴリズムまたは振る舞い

1. FOREIGNTABLEREL システムキャッシュを使用して、指定された relid に対応する pg_foreign_table タプルを検索する。 2. タプルが見つからない場合は elog(ERROR) を発生させる。 3. palloc を使用して ForeignTable 構造体をメモリ上に確保する。 4. タプルから ftserver (サーバーID) を取得し、構造体に格納する。 5. ftoptions 属性を抽出し、untransformRelOptions を介して List 形式に変換して構造体に格納する（属性が NULL の場合は NIL を設定）。 6. ReleaseSysCache を呼び出してタプルを解放し、構築した構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/file_fdw/file_fdw.c
  • contrib/postgres_fdw/deparse.c
  • contrib/postgres_fdw/postgres_fdw.c
  • **呼び出し先**:
  • SearchSysCache1
  • GETSTRUCT
  • palloc
  • SysCacheGetAttr
  • untransformRelOptions
  • ReleaseSysCache

関連データ構造

  • ForeignTable: 外部テーブルのメタデータを保持する構造体。
  • Form_pg_foreign_table: pg_foreign_table システムカタログのタプル形式。

失敗モードと境界条件

  • 指定された relid が pg_foreign_table に存在しない場合、elog(ERROR) によりトランザクションが中断される。
  • ftoptions が設定されていない場合、options フィールドには NIL が格納される。

保守担当者向けメモ

  • 本関数はシステムキャッシュを利用しているため、カタログ更新が頻繁な環境ではキャッシュの整合性に注意が必要である。
  • untransformRelOptions の結果として返されるリストは、メモリコンテキストの生存期間に依存するため、呼び出し側のメモリ管理方針と整合しているか確認すること。
