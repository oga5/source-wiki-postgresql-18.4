ArrayGetNItemsSafe

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/arrayutils.c:66

目的

多次元配列の全要素数を計算し、その値がシステム制限（MaxArraySize）を超えていないか、または計算過程でオーバーフローが発生していないかを検証します。

シグネチャまたは形状

int ArrayGetNItemsSafe(int ndim, const int *dims, struct Node *escontext)

入力と出力

  • **入力**:
  • ndim: 配列の次元数。
  • dims: 各次元のサイズを格納した整数配列。
  • escontext: エラー処理用コンテキスト（Node 構造体）。
  • **出力**:
  • 成功時: 計算された全要素数（int）。
  • 失敗時: escontext を介してエラーを報告し、-1 を返す。

前提条件と不変条件

  • ndim <= 0 の場合、要素数は 0 とみなされます。
  • 計算結果の ret は常に 0 以上である必要があります（Assert による検証）。
  • 各次元サイズ dims[i] は負であってはなりません（負の値は境界計算のオーバーフローとみなされます）。

アルゴリズムまたは振る舞い

1. ndim <= 0 ならば 0 を即座に返します。 2. ret を 1 で初期化し、各次元のサイズを順次乗算します。 3. 乗算の過程で int64 を使用してオーバーフローを検出し、int32 の範囲を超える場合はエラーを報告します。 4. 各次元のサイズが負である場合、または最終的な全要素数が MaxArraySize を超える場合、escontext を通じて ERRCODE_PROGRAM_LIMIT_EXCEEDED エラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/arrayutils.c:59（ArrayGetNItems 関数からの呼び出し）
  • **呼び出し先**:
  • ereturn マクロ（エラー発生時）

関連データ構造

  • MaxArraySize: 配列の最大要素数を規定するシステム定数。
  • Node: エラー処理コンテキストとして使用される構造体。

失敗モードと境界条件

  • **負の次元**: dims[i] < 0 の場合、エラーを返します。
  • **算術オーバーフロー**: 次元数の積が int32 の範囲を超える場合、エラーを返します。
  • **サイズ制限超過**: 計算結果が MaxArraySize を超える場合、エラーを返します。

保守担当者向けメモ

  • 本関数は escontext を受け取ることで、呼び出し元がエラーハンドリングを制御できる設計になっています。
  • int64 を用いた中間計算は、32ビット環境および64ビット環境双方での安全なオーバーフローチェックを目的としています。
