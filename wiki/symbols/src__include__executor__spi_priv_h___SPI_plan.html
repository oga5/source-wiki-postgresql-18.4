_SPI_plan

定義

  • 種別: struct
  • 定義箇所: src/include/executor/spi_priv.h:90

目的

SPI（Server Programming Interface）において、SQL コマンドの実行計画を管理するための内部構造体です。計画のライフサイクルやメモリ管理、実行時のパラメータを保持します。

シグネチャまたは形状

typedef struct _SPI_plan
{
    int            magic;
    bool           saved;
    bool           oneshot;
    List          *plancache_list;
    MemoryContext  plancxt;
    RawParseMode   parse_mode;
    int            cursor_options;
    int            nargs;
    Oid           *argtypes;
    ParserSetupHook parserSetup;
    void          *parserSetupArg;
} _SPI_plan;

入力と出力

  • **入力**: 計画作成時に nargs（引数数）、argtypes（引数型）、parserSetup（パーサー設定フック）などが設定されます。
  • **出力**: 実行時に plancache_list を介してキャッシュされた実行計画（CachedPlanSource）が提供されます。

前提条件と不変条件

  • magic フィールドは常に _SPI_PLAN_MAGIC と一致する必要があります。
  • plancxt が NULL の場合、その計画は「一時的な計画」として扱われ、関数終了時に自動的に解放されます。

アルゴリズムまたは振る舞い

  • SPI 呼び出しの実行計画を保持し、plancache.c と連携して計画の無効化や再計画を管理します。
  • saved フラグにより、計画が永続的か一時的かを区別します。
  • oneshot フラグは、計画が一度限りの実行を想定しているかを示します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/spi.c 内の複数の関数（SPI_prepare 等）で、ローカル変数としてインスタンス化および初期化が行われています。
  • **呼び出し先**: plancache.c の関数群が plancache_list を操作するために参照します。

関連データ構造

  • CachedPlanSource: plancache_list に格納される実際の計画ソース。
  • MemoryContext: plancxt として保持され、計画に関連するデータの生存期間を制御します。

失敗モードと境界条件

  • plancxt が適切に管理されていない場合、メモリリークや不正なメモリ参照が発生する可能性があります。
  • magic 値の不一致は、構造体の破損を示唆します。

保守担当者向けメモ

  • 本構造体は _ で始まる内部用定義であり、外部モジュールからの直接利用は避けてください。
  • plancxt のメモリ管理は複雑であるため、計画の生存期間を設計する際は注意が必要です。
  • ネストされた SPI 呼び出しにおけるスタック管理については、_SPI_connection 構造体との連携を確認してください。
