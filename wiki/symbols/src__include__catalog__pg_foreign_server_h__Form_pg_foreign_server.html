Form_pg_foreign_server

定義

  • 種別: typedef
  • 定義箇所: src/include/catalog/pg_foreign_server.h:48

目的

システムカタログ pg_foreign_server の各行データ（FormData_pg_foreign_server）を指すポインタ型を提供し、外部サーバーのメタデータ操作を簡略化する。

シグネチャまたは形状

typedef FormData_pg_foreign_server *Form_pg_foreign_server;

入力と出力

  • **入力**: HeapTuple 構造体から抽出されたデータポインタ。
  • **出力**: FormData_pg_foreign_server 構造体への型キャストされたポインタ。

前提条件と不変条件

  • GETSTRUCT(tup) マクロによって取得された HeapTuple のデータ領域が、FormData_pg_foreign_server 構造体のレイアウトと一致している必要がある。
  • 本型は、カタログ行の読み取り専用または更新用ポインタとして使用される。

アルゴリズムまたは振る舞い

1. HeapTuple を取得する。 2. GETSTRUCT マクロを使用して、タプル内のデータ領域へのポインタを取得する。 3. 取得したポインタを Form_pg_foreign_server 型にキャストすることで、サーバー名、所有者、FDW OID、オプション等の各フィールドに構造体メンバとしてアクセス可能にする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/foreigncmds.c
  • src/backend/foreign/foreign.c
  • **呼び出し先**:
  • GETSTRUCT マクロ（タプルからデータ構造への変換に使用）。

関連データ構造

  • FormData_pg_foreign_server: カタログの物理的な行構造。
  • HeapTuple: カタログ行を保持する汎用的なタプル構造体。

失敗モードと境界条件

  • GETSTRUCT に渡されるタプルが pg_foreign_server カタログのものではない場合、フィールドアクセス時にメモリ破壊や不正な値の読み取りが発生する可能性がある。

保守担当者向けメモ

  • 本型は pg_foreign_server のスキーマ変更（フィールドの追加・削除）に直接依存する。構造体定義を変更した場合は、必ず pg_foreign_server_d.h の定数および関連するカタログ操作ロジックとの整合性を確認すること。
