create_physical_replication_slot

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slotfuncs.c:35

目的

物理レプリケーションスロットを作成し、必要に応じてWALの予約およびディスクへの永続化を行う。

シグネチャまたは形状

static void create_physical_replication_slot(
    char *name, 
    bool immediately_reserve, 
    bool temporary, 
    XLogRecPtr restart_lsn
)

入力と出力

  • **入力**:
  • name: 作成するスロット名。
  • immediately_reserve: true の場合、直ちにWALを予約する。
  • temporary: true の場合、一時スロットとして作成する。
  • restart_lsn: immediately_reserve が true の際に使用する開始LSN。無効な場合は現在のLSNを予約する。
  • **出力**: なし（副作用としてスロットを作成）。

前提条件と不変条件

  • MyReplicationSlot が NULL であること（アサーションにより強制）。
  • ReplicationSlotCreate により、スロット名の競合チェックが行われる。

アルゴリズムまたは振る舞い

1. ReplicationSlotCreate を呼び出し、共有メモリ上に指定された名前でスロットを確保する。 2. immediately_reserve が true の場合:

  • restart_lsn が無効であれば ReplicationSlotReserveWal() を呼び出し、現在のWAL位置を予約する。
  • restart_lsn が有効であれば、その値を MyReplicationSlot->data.restart_lsn に設定する。
  • ReplicationSlotMarkDirty() および ReplicationSlotSave() を呼び出し、スロットの状態をディスクへ書き出す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/slotfuncs.c 内の関数（pg_create_physical_replication_slot 等から参照されていることが示唆される）。
  • **呼び出し先**:
  • ReplicationSlotCreate
  • ReplicationSlotReserveWal
  • ReplicationSlotMarkDirty
  • ReplicationSlotSave

関連データ構造

  • ReplicationSlot (グローバル変数 MyReplicationSlot を介してアクセス)
  • XLogRecPtr

失敗モードと境界条件

  • スロット名が既存のものと重複する場合、ReplicationSlotCreate 内でエラーが発生する。
  • ディスク書き込み失敗時、ReplicationSlotSave がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は static であり、主にSQLインターフェース層からの呼び出しを想定している。
  • エラー発生時のクリーンアップは、トランザクションのロールバックメカニズムに依存する。
