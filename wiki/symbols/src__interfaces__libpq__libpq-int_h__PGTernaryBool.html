PGTernaryBool

定義

  • 種別: enum
  • 定義箇所: src/interfaces/libpq/libpq-int.h:254

目的

真偽値（True/False）に加えて「不明（Unknown）」の状態を表現するための列挙型です。

シグネチャまたは形状

typedef enum
{
    PG_BOOL_UNKNOWN = 0,
    PG_BOOL_YES,
    PG_BOOL_NO
} PGTernaryBool;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: PG_BOOL_UNKNOWN (0), PG_BOOL_YES, PG_BOOL_NO のいずれかの値。

前提条件と不変条件

  • 値は必ず定義された3つの列挙子のいずれかをとる必要があります。
  • PG_BOOL_UNKNOWN は初期状態や未確定状態を示すために使用されます。

アルゴリズムまたは振る舞い

  • 状態の保持: 接続パラメータやトランザクション状態など、値が確定していない可能性がある属性の管理に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/interfaces/libpq/libpq-int.h: default_transaction_read_only および in_hot_standby メンバの型として使用。
  • src/interfaces/libpq-oauth/oauth-curl.c: init_successful 変数の型として使用。
  • 呼び出し先: なし（本シンボルはデータ型定義であり、関数呼び出しは含みません）。

関連データ構造

  • PGconn: libpq-int.h 内で定義される構造体。本型をメンバとして保持します。

失敗モードと境界条件

  • 境界条件: 整数値としてキャストされた場合、0, 1, 2 以外の値が代入されると予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本型は libpq 内部でのみ使用されるプライベートな定義です。公開 API である libpq-fe.h には含まれていません。
  • 新しい状態を追加する場合は、既存のロジック（特に条件分岐）がすべて網羅されているか確認してください。
