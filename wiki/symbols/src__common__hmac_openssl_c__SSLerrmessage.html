SSLerrmessage

定義

  • 種別: function
  • 定義箇所: src/common/hmac_openssl.c:97

目的

OpenSSL のエラーコードを人間が読み取れる形式の文字列に変換します。

シグネチャまたは形状

static const char *SSLerrmessage(unsigned long ecode)

入力と出力

  • **入力**: ecode (unsigned long) - OpenSSL のエラーコード（ERR_get_error() 等で取得される値）。
  • **出力**: const char * - エラー内容を示す文字列。ecode が 0 の場合は NULL を返します。

前提条件と不変条件

  • ecode が 0 の場合、関数は即座に NULL を返します。
  • OpenSSL の ERR_reason_error_string を利用するため、OpenSSL ライブラリが初期化されている必要があります。

アルゴリズムまたは振る舞い

1. 引数 ecode が 0 かどうかを判定し、0 であれば NULL を返します。 2. 0 でない場合、OpenSSL の ERR_reason_error_string(ecode) を呼び出し、その結果を返します。 3. ERR_reason_error_string が NULL を返す可能性があることを前提としており、呼び出し元で適切なフォールバック処理を行う設計となっています。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/be-secure-openssl.c 内の複数の箇所から、SSL 関連のエラー発生時に ERR_get_error() の戻り値を引数として呼び出されています。
  • **呼び出し先**: OpenSSL ライブラリの ERR_reason_error_string を呼び出します。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • ecode に無効なエラーコードが渡された場合、ERR_reason_error_string の仕様に従い NULL が返される可能性があります。

保守担当者向けメモ

  • 本関数は src/common/hmac_openssl.c 内で定義されていますが、主に src/backend/libpq/be-secure-openssl.c から利用されています。
  • エラーメッセージの取得に失敗した場合の挙動は、呼び出し側のエラーハンドリングロジックに依存します。
