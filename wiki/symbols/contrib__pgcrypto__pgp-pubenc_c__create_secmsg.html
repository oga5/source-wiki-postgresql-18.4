create_secmsg

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pubenc.c:82

目的

セッション鍵、アルゴリズムID、およびチェックサムを結合して「秘密メッセージ」を構築し、PKCS#1 v1.5 パディングを施した上で、暗号化可能な MPI（Multi-Precision Integer）形式に変換する。

シグネチャまたは形状

static int create_secmsg(PGP_Context *ctx, PGP_MPI **msg_p, int full_bytes)

入力と出力

  • **入力**:
  • ctx: PGPコンテキスト（セッション鍵およびアルゴリズム情報を含む）。
  • full_bytes: パディング後のターゲットバイト数。
  • **出力**:
  • msg_p: 生成された MPI へのポインタを格納する領域。
  • 戻り値: 成功時は 0 以上、失敗時は負のエラーコード。

前提条件と不変条件

  • ctx->sess_key が有効なセッション鍵を保持していること。
  • 処理終了時には、一時的に確保したメモリ（secmsg および padded）は必ず px_memset でゼロクリアされ、pfree で解放されること。

アルゴリズムまたは振る舞い

1. **チェックサム計算**: セッション鍵の各バイトの合計を算出する。 2. **メッセージ構築**: [アルゴリズムID][セッション鍵][チェックサム(2バイト)] の順でバッファを構成する。 3. **パディング**: pad_eme_pkcs1_v15 を呼び出し、指定された full_bytes に合わせて PKCS#1 v1.5 形式のパディングを適用する。 4. **MPI変換**: パディング済みデータを pgp_mpi_create により MPI 形式へ変換する。 5. **クリーンアップ**: 使用した一時バッファをゼロクリアしてメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-pubenc.c 内の encrypt_and_write_elgamal
  • contrib/pgcrypto/pgp-pubenc.c 内の encrypt_and_write_rsa
  • **呼び出し先**:
  • pad_eme_pkcs1_v15
  • pgp_mpi_create
  • px_memset, palloc, pfree

関連データ構造

  • PGP_Context: セッション鍵やアルゴリズムIDを保持する構造体。
  • PGP_MPI: 多倍長整数を表現する構造体。

失敗モードと境界条件

  • pad_eme_pkcs1_v15 が失敗した場合、MPI は生成されずエラーコードが返される。
  • メモリ確保に失敗した場合は palloc が例外を投げる可能性がある。

保守担当者向けメモ

  • 本関数は機密情報（セッション鍵）を扱うため、メモリ解放時の px_memset によるゼロクリアが必須である。
  • パディング形式や MPI 変換ロジックを変更する場合は、PGP 仕様および pgp-mpi.c の実装との整合性に注意すること。
