arraycontsel

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_selfuncs.c:240

目的

PostgreSQLのクエリプランナにおいて、配列型に対する演算子（@>、<@、&&）の制限選択率（Selectivity）を推定します。

シグネチャまたは形状

Datum arraycontsel(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • root: PlannerInfo 構造体（プランナ情報）
  • operator: 評価対象の演算子OID
  • args: 制限句の引数リスト
  • varRelid: 変数のリレーションID
  • **出力**:
  • float8: 0.0から1.0の範囲で算出された選択率。

前提条件と不変条件

  • 評価対象の式が (variable op constant) または (constant op variable) の形式であること。
  • 定数側が配列型であり、かつカラムの要素型と一致していること。
  • 演算子が &&、@>、<@ のいずれかであること。
  • 統計情報が不足している場合は、デフォルトの選択率が返されます。

アルゴリズムまたは振る舞い

1. **検証**: 入力が変数と定数の比較であることを確認し、定数がNULLであれば選択率0.0を返します。 2. **正規化**: 変数が右辺にある場合は演算子を交換し、常に変数が左辺にある状態へ正規化します。 3. **型チェック**: 定数と変数の要素型が一致するか確認します。 4. **推定**: calc_arraycontsel を呼び出し、統計情報（MCELEM、DECHIST）に基づいた選択率を算出します。 5. **後処理**: 算出された値を CLAMP_PROBABILITY マクロで0.0〜1.0の範囲に収め、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_selfuncs.c（intarray 拡張モジュールからの直接呼び出し）
  • **呼び出し先**:
  • calc_arraycontsel: 統計情報を用いた実際の計算ロジック。
  • get_restriction_variable: 式の解析。
  • ReleaseVariableStats: 統計情報の解放。
  • get_base_element_type: 型情報の取得。

関連データ構造

  • PlannerInfo: プランナのコンテキスト。
  • VariableStatData: 変数の統計情報保持用構造体。
  • Const: 定数ノード。

失敗モードと境界条件

  • **型不一致**: 要素型が一致しない場合、デフォルトの選択率を返します。
  • **非定数**: 定数以外の値が比較対象の場合、デフォルトの選択率を返します。
  • **NULL定数**: 演算子が厳密（strict）であるため、NULL定数に対しては選択率0.0を返します。
  • **未認識の演算子**: 統計情報や計算ロジックが対応していない演算子の場合、エラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は array_selfuncs.c 内のメインエントリポイントです。ロジックの詳細は calc_arraycontsel に委譲されています。
  • 統計情報が不足している場合のフォールバック値（DEFAULT_SEL）は、演算子ごとに定義されています。
  • intarray 拡張モジュールから直接利用されているため、変更時は互換性に注意してください。
