fsync_parent_path

定義

  • 種別: function
  • 定義箇所: src/common/file_utils.c:455

目的

指定されたファイルまたはディレクトリの親ディレクトリに対して fsync を実行し、メタデータの永続化を保証します。

シグネチャまたは形状

int fsync_parent_path(const char *fname);

入力と出力

  • **入力**: fname (対象となるファイルまたはディレクトリのパス文字列)
  • **出力**: 成功時に 0、失敗時に -1 を返します。

前提条件と不変条件

  • fname は MAXPGPATH 以下の長さである必要があります。
  • get_parent_directory の仕様により、入力がファイル名のみ（親ディレクトリを含まない）の場合は、カレントディレクトリ (.) を対象として処理します。

アルゴリズムまたは振る舞い

1. 入力パス fname を parentpath バッファにコピーします。 2. get_parent_directory を呼び出し、パスから親ディレクトリ部分を抽出します。 3. 抽出結果が空文字列の場合、カレントディレクトリを示す . に置き換えます。 4. fsync_fname(parentpath, true) を呼び出し、親ディレクトリを同期します。

呼び出し元と呼び出し先

  • **呼び出し先**: get_parent_directory, fsync_fname
  • **呼び出し元**:
  • src/backend/storage/file/fd.c (複数の箇所で呼び出しあり)
  • src/bin/pg_basebackup/walmethods.c
  • src/bin/pg_upgrade/relfilenumber.c
  • src/common/file_utils.c (durable_rename 内など)

関連データ構造

  • MAXPGPATH: パス名の最大長を定義する定数。

失敗モードと境界条件

  • fsync_fname が失敗した場合、本関数も -1 を返します。
  • 親ディレクトリの取得に失敗した場合や、ファイルシステムが fsync をサポートしていない環境での挙動は fsync_fname の実装に依存します。

保守担当者向けメモ

  • 本関数は FRONTEND 定義時にコンパイルされるフロントエンド用ユーティリティですが、バックエンドコード（src/backend/storage/file/fd.c）からも同名の関数が参照されています。バックエンド側の実装との整合性に注意してください。
