system_rows_nextsampleblock

定義

  • 種別: function
  • 定義箇所: contrib/tsm_system_rows/tsm_system_rows.c:206

目的

TABLESAMPLE SYSTEM_ROWS メソッドにおいて、リレーション内の全ブロックをランダムな順序で網羅的に走査するための次のブロック番号を決定する。

シグネチャまたは形状

BlockNumber system_rows_nextsampleblock(SampleScanState *node, BlockNumber nblocks)

入力と出力

  • **入力**:
  • node: サンプリングスキャンの状態を保持する SampleScanState 構造体。
  • nblocks: 現在のリレーションが持つ総ブロック数。
  • **出力**:
  • 次に読み込むべき BlockNumber。走査が完了した場合は InvalidBlockNumber を返す。

前提条件と不変条件

  • node->tsm_state は SystemRowsSamplerData 型としてキャスト可能であること。
  • 初回呼び出し時に sampler->step が 0 の場合、乱数生成器を用いて開始位置とスキップ幅が初期化される。
  • スキップ幅（step）は、リレーション内の全ブロックを重複なく巡回するために random_relative_prime で算出される。

アルゴリズムまたは振る舞い

1. **初期化**: 初回呼び出し時、sampler->step が未設定であれば、sampler_random_init_state で乱数状態を初期化し、開始ブロックとスキップ幅を決定する。 2. **終了判定**: 全ブロックを走査済み、または必要なタプル数に到達した場合は InvalidBlockNumber を返す。 3. **ブロック選択**: 線形合同法的なスキップ走査（lb = (lb + step) % nblocks）を行い、次のブロック番号を算出する。 4. **検証**: 算出されたブロック番号が現在の nblocks を超える場合は、有効な値になるまで再計算を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/tsm_system_rows/tsm_system_rows.c 内の tsm_system_rows_handler（TsmRoutine 構造体への関数ポインタ登録経由）。
  • **呼び出し先**:
  • sampler_random_init_state
  • sampler_random_fract
  • random_relative_prime

関連データ構造

  • SystemRowsSamplerData: スキャン状態（step, firstblock, lb, doneblocks 等）を保持する構造体。
  • SampleScanState: PostgreSQL のスキャン実行状態を保持する構造体。

失敗モードと境界条件

  • nblocks が 0 の場合、走査対象がないため InvalidBlockNumber を返す。
  • スキャン中にリレーションのブロック数が増減した場合、do-while ループにより現在の nblocks に適合するブロック番号が選択される。

保守担当者向けメモ

  • 本関数は SystemRowsSamplerData の状態を更新するため、再スキャン時の挙動に影響を与える。
  • sampler->lb の更新には uint64 へのキャストを使用し、オーバーフローを回避している点に注意すること。
