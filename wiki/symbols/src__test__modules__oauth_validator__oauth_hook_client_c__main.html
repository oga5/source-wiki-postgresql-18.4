目的

OAuth 認証フック機能の動作を検証するためのテストドライバとして、コマンドライン引数の解析、認証フックの登録、およびデータベース接続の実行を行う。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。接続文字列（conninfo）およびテストモードを指定するオプション（--expected-scope, --token, --misbehave, --stress-async 等）を受け取る。
  • **出力**:
  • 正常終了時は 0 を返す。
  • 引数エラーや接続失敗時は 1 を返す。

前提条件と不変条件

  • libpq が利用可能であること。
  • 接続文字列（conninfo）が引数の最後に指定されていること。

アルゴリズムまたは振る舞い

1. **引数解析**: getopt_long を使用し、テスト用の設定フラグ（expected_scope, token, misbehave_mode 等）をグローバル変数に格納する。 2. **フック登録**: PQsetAuthDataHook(handle_auth_data) を呼び出し、認証プロセスにフック関数を注入する。 3. **接続実行**:

  • --stress-async 指定時: PQconnectStart を呼び出し、PQconnectPoll をループさせて非同期接続を強制的に完了させる。
  • 通常時: PQconnectdb を使用して同期接続を行う。

    • 4. **検証**: 接続状態を PQstatus で確認し、成功すればメッセージを出力して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: Perl テストスクリプト（t/002_client.pl）から実行される。
  • **呼び出し先**:
  • usage(): 引数エラー時やヘルプ表示時に呼び出される。
  • PQsetAuthDataHook(): libpq API。
  • PQconnectStart() / PQconnectPoll() / PQconnectdb(): libpq API。
  • handle_auth_data(): 認証時に libpq からコールバックされる。

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • struct option: getopt_long 用のコマンドラインオプション定義。

失敗モードと境界条件

  • **引数不正**: 必須の接続文字列が欠けている場合、usage() を表示して終了する。
  • **接続失敗**: PQstatus が CONNECTION_OK でない場合、エラーメッセージを出力して終了する。
  • **非同期テスト**: stress-async モードでは、PQconnectPoll のループが PGRES_POLLING_FAILED に達した場合に終了する。

保守担当者向けメモ

  • 本関数はテストツールの一部であり、実運用コードではない。
  • stress-async モードは、非同期接続の競合やタイミング問題を誘発させるためのビジーウェイト実装である。
  • ツールによる「呼び出し元」の索引結果（amcheck 等）は、本シンボルとは無関係な誤検知である。
