pgaio_uring_wait_one

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_io_uring.c:583

目的

特定の非同期 I/O (PgAioHandle) が完了するまで待機し、完了イベントを処理する。

シグネチャまたは形状

static void pgaio_uring_wait_one(PgAioHandle *ioh, uint64 ref_generation)

入力と出力

  • **入力**:
  • ioh: 待機対象の PgAioHandle 構造体へのポインタ。
  • ref_generation: 待機対象の I/O 世代番号。
  • **出力**: なし（完了時に状態更新を行う）。

前提条件と不変条件

  • ioh->owner_procno に基づく PgAioUringContext が初期化済みであること。
  • 呼び出し元は completion_lock を保持していないこと（関数内で排他的に取得するため）。

アルゴリズムまたは振る舞い

1. 対象の PgAioUringContext の completion_lock を排他取得する。 2. ループ内で以下を繰り返す：

  • pgaio_io_was_recycled を確認し、対象 I/O が既に完了・再利用されている場合はループを抜ける。
  • io_uring_cq_ready で完了キューを確認し、イベントがあれば pgaio_uring_drain_locked を呼び出して処理する。
  • 完了していない場合、io_uring_wait_cqes を呼び出し、カーネルからの完了通知を待機する（WAIT_EVENT_AIO_IO_URING_EXECUTION を報告）。
  • io_uring_wait_cqes が EINTR を返した場合は再試行し、その他のエラー時は PANIC を発生させる。

    • 3. completion_lock を解放して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/aio/method_io_uring.c 内の IoMethodOps 構造体（pgaio_uring_ops）を通じて間接的に呼び出される。
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease
  • pgaio_io_was_recycled
  • io_uring_cq_ready
  • pgstat_report_wait_start / pgstat_report_wait_end
  • io_uring_wait_cqes
  • pgaio_uring_drain_locked

関連データ構造

  • PgAioHandle: I/O 操作の状態を保持するハンドル。
  • PgAioUringContext: io_uring インスタンスと completion_lock を含むコンテキスト。

失敗モードと境界条件

  • io_uring_wait_cqes が EINTR 以外のエラーを返した場合、errno を設定して PANIC する。
  • 他のバックエンドによって I/O が既に処理済みの場合、待機せずに即座に復帰する。

保守担当者向けメモ

  • 現在のロック実装は、所有プロセス以外のバックエンドが完了処理を行うことを想定しているが、所有プロセス自身が待機する場合でもロックを取得するため、オーバーヘッドが発生している。将来的な最適化の余地がある。
