SpGistGetBuffer

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/spgist/spgutils.c:568

目的

SP-GiSTインデックスにおいて、指定された条件（リーフ/内部ノード、NULL格納の有無）を満たす空き領域を持つバッファを取得または新規割り当てする。

シグネチャまたは形状

Buffer SpGistGetBuffer(Relation index, int flags, int needSpace, bool *isNew)

入力と出力

  • **入力**:
  • index: 対象のインデックスリレーション。
  • flags: ページの種類を指定するフラグ（GBUF_LEAF, GBUF_NULLS 等）。
  • needSpace: 要求する最小空きバイト数。
  • **出力**:
  • isNew: 取得したバッファが新規初期化されたものである場合に true が設定される。
  • **戻り値**: ロックされた Buffer。

前提条件と不変条件

  • needSpace は SPGIST_PAGE_CAPACITY を超えてはならない。
  • キャッシュされたページ（lup->blkno）が固定ページ（SpGistBlockIsFixed）であってはならない。
  • 取得したバッファは呼び出し元で適切に解放する必要がある。

アルゴリズムまたは振る舞い

1. **要求サイズの調整**: SpGistGetTargetPageFreeSpace を加算し、SPGIST_PAGE_CAPACITY を上限として調整する。 2. **キャッシュ確認**: GET_LUP でキャッシュされたページ情報を取得。キャッシュがない場合は allocNewBuffer を呼び出す。 3. **ページ検証**:

  • キャッシュされたページを ReadBuffer で読み込み、ConditionalLockBuffer を試行。ロック失敗時は allocNewBuffer へフォールバック。
  • ページが空または削除済みの場合、SpGistInitBuffer で初期化して返す。
  • ページの種類（リーフ/内部、NULL対応）と空き容量が要求を満たすか確認し、適合すればそのバッファを返す。

    • 4. **フォールバック**: キャッシュされたページが不適合な場合、allocNewBuffer を呼び出して新規ページを確保する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgdoinsert.c 内の各所（インデックス挿入処理）。
  • **呼び出し先**: spgGetCache, allocNewBuffer (内部関数), ReadBuffer, ConditionalLockBuffer, BufferGetPage, PageIsNew, SpGistPageIsDeleted, PageIsEmpty, SpGistInitBuffer, PageGetExactFreeSpace, UnlockReleaseBuffer。

関連データ構造

  • SpGistCache: インデックスの実行時状態を保持。
  • SpGistLastUsedPage: 直近で使用されたページのキャッシュ情報。
  • Buffer: PostgreSQLのバッファ管理構造体。

失敗モードと境界条件

  • needSpace が SPGIST_PAGE_CAPACITY を超える場合、elog(ERROR) で停止する。
  • ページロックの競合が発生した場合、キャッシュを無視して allocNewBuffer を呼び出し、新規ページ確保を試みる。

保守担当者向けメモ

  • lastUsedPages キャッシュは必ずしも最新ではないため、ページ取得後に型や空き容量の再検証を行っている。
  • fillfactor を考慮した空き容量計算を行っているため、タプル挿入時のページ選択効率に影響を与える。
