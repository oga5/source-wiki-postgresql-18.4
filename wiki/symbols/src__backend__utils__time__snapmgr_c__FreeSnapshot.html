FreeSnapshot

定義

  • 種別: function
  • 定義箇所: src/backend/utils/time/snapmgr.c:661

目的

不要になったスナップショット構造体（SnapshotData）のメモリを解放する。

シグネチャまたは形状

static void FreeSnapshot(Snapshot snapshot)

入力と出力

  • **入力**: snapshot (解放対象の Snapshot 構造体へのポインタ)
  • **出力**: なし

前提条件と不変条件

  • snapshot->regd_count == 0 であること（登録済み参照がないこと）。
  • snapshot->active_count == 0 であること（アクティブな参照がないこと）。
  • snapshot->copied が真であること（コピーされたスナップショットであること）。

アルゴリズムまたは振る舞い

1. 渡された snapshot が解放可能であることを前提条件の通りアサーションで検証する。 2. pfree(snapshot) を呼び出し、スナップショット構造体が占有していたメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/time/snapmgr.c 内の複数の箇所（行785, 898, 998など）から呼び出されている。
  • **呼び出し先**:
  • pfree (PostgreSQLのメモリ解放関数)

関連データ構造

  • SnapshotData (または Snapshot): スナップショット情報を保持する構造体。

失敗モードと境界条件

  • 前提条件（参照カウントが0であること等）が満たされていない場合、アサーション違反によりプロセスが異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は static であり、snapmgr.c 内部でのみ使用される。
  • メモリリークを防ぐため、スナップショットの参照カウントが適切に管理されていることを確認してから呼び出す必要がある。
