T_Action

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_dump/pg_backup_archiver.h:116

目的

アーカイブ処理における操作の種類（ダンプまたは復元）を識別するために使用される列挙型です。

シグネチャまたは形状

typedef enum T_Action
{
	ACT_DUMP,
	ACT_RESTORE,
} T_Action;

入力と出力

  • 入力: なし（列挙定数）
  • 出力: ACT_DUMP（ダンプ操作を示す）、ACT_RESTORE（復元操作を示す）

前提条件と不変条件

  • pg_dump または pg_restore の実行コンテキストにおいて、アーカイブ操作の方向性を決定するために使用されます。

アルゴリズムまたは振る舞い

  • 並列処理において、ワーカープロセスが実行すべきタスクがダンプであるか復元であるかを区別するために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/bin/pg_dump/parallel.c: buildWorkerCommand, parseWorkerCommand, buildWorkerResponse 等の関数で引数として使用。
  • src/bin/pg_dump/parallel.h: 関数シグネチャ内で使用。
  • 呼び出し先: なし（列挙型のため）。

関連データ構造

  • ArchiveHandle: アーカイブ操作の状態管理に使用される構造体。
  • TocEntry: ダンプ対象オブジェクトのメタデータ。

失敗モードと境界条件

  • 定義された ACT_DUMP または ACT_RESTORE 以外の値が割り当てられた場合、後続の処理で予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本シンボルは並列復元処理（parallel.c）において頻繁に参照されています。操作の追加が必要な場合は、本列挙型とそれを利用する parallel.c 内のロジックを併せて更新してください。
