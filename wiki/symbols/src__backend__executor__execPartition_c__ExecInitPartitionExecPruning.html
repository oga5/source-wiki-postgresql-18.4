ExecInitPartitionExecPruning

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execPartition.c:1879

目的

クエリ実行時におけるパーティションプルーニングの初期化およびコンテキスト設定を行う。

シグネチャまたは形状

PartitionPruneState *ExecInitPartitionExecPruning(
    PlanState *planstate,
    int n_total_subplans,
    int part_prune_index,
    Bitmapset *relids,
    Bitmapset **initially_valid_subplans
)

入力と出力

  • **入力**:
  • planstate: 実行中のプランノードの状態。
  • n_total_subplans: 全サブプラン数。
  • part_prune_index: EState 内のプルーニング情報インデックス。
  • relids: プランノードに関連付けられたリレーションIDセット。
  • **出力**:
  • 戻り値: 初期化された PartitionPruneState。
  • initially_valid_subplans: 初期プルーニングの結果として有効なサブプランのビットマップ。

前提条件と不変条件

  • EState 内の es_part_prune_infos および es_part_prune_states に、指定された part_prune_index が存在すること。
  • 引数 relids は、pruneinfo->relids と一致しなければならない（不一致時はエラー）。
  • PartitionPruneState は事前に ExecDoInitialPruning() によって生成されている必要がある。

アルゴリズムまたは振る舞い

1. EState から対応する PartitionPruneInfo を取得し、relids の整合性を検証する。 2. EState から PartitionPruneState を取得する。 3. do_initial_prune が真であれば、es_part_prune_results から初期有効サブプランを取得する。偽であれば、全サブプランを有効とするビットマップを生成する。 4. do_exec_prune が真であれば、InitExecPartitionPruneContexts() を呼び出し、実行時プルーニング用のコンテキストを初期化する。この際、初期プルーニングで除外されたサブプランを考慮してインデックスの再シーケンスを行う。 5. 初期化された PartitionPruneState を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAppend.c
  • src/backend/executor/nodeMergeAppend.c
  • **呼び出し先**:
  • list_nth_node (PostgreSQL内部リスト操作)
  • bms_equal, bms_add_range (ビットマップ操作)
  • InitExecPartitionPruneContexts (実行時コンテキスト初期化)

関連データ構造

  • PartitionPruneState: プルーニングの実行状態を保持する構造体。
  • PartitionPruneInfo: プランナーが生成したプルーニング情報。
  • PlanState: 実行ノードの共通状態。
  • EState: クエリ実行全体の実行状態。

失敗モードと境界条件

  • relids が pruneinfo->relids と一致しない場合、elog(ERROR) により処理を中断する。
  • n_total_subplans が 0 以下の場合、Assert により検証される。

保守担当者向けメモ

  • 本関数は ExecDoInitialPruning の結果を利用するため、呼び出し順序に依存関係がある。
  • InitExecPartitionPruneContexts を呼び出す際、初期プルーニングで削除されたサブプランを考慮したインデックス再計算が行われる点に注意すること。
