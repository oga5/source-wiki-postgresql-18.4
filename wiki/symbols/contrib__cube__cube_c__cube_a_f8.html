cube_a_f8

定義

  • 種別: function
  • 定義箇所: contrib/cube/cube.c:209

目的

PostgreSQL の配列型から cube 型（NDBOX）のポイントを作成します。

シグネチャまたは形状

Datum cube_a_f8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_ARRAYTYPE_P(0) を介して ArrayType（浮動小数点数の配列）を受け取ります。
  • **出力**: PG_RETURN_NDBOX_P を介して、入力配列の要素を座標とする NDBOX 構造体を返します。

前提条件と不変条件

  • 入力配列に NULL が含まれていないこと。
  • 配列の要素数（次元数）が CUBE_MAX_DIM 以下であること。

アルゴリズムまたは振る舞い

1. 入力配列が NULL を含む場合、ERRCODE_ARRAY_ELEMENT_ERROR でエラーを報告します。 2. 配列の要素数（dim）が CUBE_MAX_DIM を超える場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラーを報告します。 3. POINT_SIZE(dim) に基づいて NDBOX 構造体に必要なメモリを割り当てます。 4. SET_VARSIZE、SET_DIM、SET_POINT_BIT マクロを使用して、NDBOX のヘッダー情報を初期化します。 5. 入力配列の各要素を NDBOX の座標配列 x にコピーします。 6. 構築した NDBOX を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数インターフェース（PG_FUNCTION_INFO_V1(cube_a_f8) による登録）。
  • **呼び出し先**: array_contains_nulls, ARRNELEMS, ARRPTR, POINT_SIZE, palloc0, SET_VARSIZE, SET_DIM, SET_POINT_BIT, PG_RETURN_NDBOX_P（いずれも PostgreSQL の内部マクロまたは関数）。

関連データ構造

  • NDBOX: 多次元ボックスを表現するデータ構造。
  • ArrayType: PostgreSQL の配列型。

失敗モードと境界条件

  • **NULL 値**: 配列内に NULL が存在する場合、エラーとなります。
  • **次元数超過**: 配列の長さが CUBE_MAX_DIM を超える場合、エラーとなります。

保守担当者向けメモ

  • 本関数はポイント（点）の作成に特化しています。NDBOX の内部構造（x 配列の配置）に依存しているため、cubedata.h の定義変更時には注意が必要です。
