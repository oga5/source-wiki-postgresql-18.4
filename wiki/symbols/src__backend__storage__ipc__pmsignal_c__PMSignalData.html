PMSignalData

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/ipc/pmsignal.c:72

目的

Postmaster（親プロセス）とバックエンドプロセス（子プロセス）間での非同期通信および状態管理を行うための共有メモリ構造体です。

シグネチャまたは形状

struct PMSignalData
{
    sig_atomic_t PMSignalFlags[NUM_PMSIGNALS];
    QuitSignalReason sigquit_reason;
    int            num_child_flags;
    sig_atomic_t   PMChildFlags[FLEXIBLE_ARRAY_MEMBER];
};

入力と出力

本構造体は関数ではなくデータ構造ですが、共有メモリを介して以下の入出力を行います。

  • **入力**: 子プロセスによる PMSignalFlags の更新（通知）、Postmasterによる PMChildFlags の状態管理。
  • **出力**: Postmasterが読み取るシグナルフラグ、子プロセスが読み取る sigquit_reason。

前提条件と不変条件

  • 共有メモリ上に配置される必要があります。
  • PMSignalFlags および PMChildFlags は sig_atomic_t 型であり、シグナルハンドラ内での更新がアトミックに行われることが保証されています。
  • PMChildFlags は可変長配列として定義されており、メモリ確保時に num_child_flags に基づいてサイズが決定されます。

アルゴリズムまたは振る舞い

  • **シグナル通知**: 子プロセスが PMSignalFlags を更新し、Postmasterへ SIGUSR1 を送信することで非同期イベントを伝達します。
  • **シャットダウン理由の伝達**: Postmasterが SIGQUIT を送出する際、その理由を sigquit_reason に格納し、子プロセスがこれを参照します。
  • **状態追跡**: PMChildFlags を用いて、各バックエンドプロセスの状態（UNUSED/ASSIGNED/ACTIVE/WALSENDER等）を管理します。

呼び出し元と呼び出し先

  • **参照元**: src/backend/postmaster/launch_backend.c および src/backend/storage/ipc/pmsignal.c 内のグローバル変数 PMSignalState を通じてアクセスされます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。操作は SendPostmasterSignal や CheckPostmasterSignal 等のモジュール内関数を介して行われます。

関連データ構造

  • PMSignalState: PMSignalData を指すグローバルなポインタ。
  • QuitSignalReason: シャットダウン理由を保持する列挙型。

失敗モードと境界条件

  • PMChildFlags のサイズは num_child_flags に依存するため、共有メモリ確保時のサイズ計算（offsetof を使用）が不適切である場合、メモリ破壊やアクセス違反が発生する可能性があります。
  • 共有メモリが初期化される前にアクセスした場合、PMSignalState が NULL であるためクラッシュします。

保守担当者向けメモ

  • 本構造体は不透明な構造体として扱われるべきであり、詳細は pmsignal.c 内に限定されています。
  • PMChildFlags の可変長配列部分は、PMSignalShmemSize() で計算されるサイズと整合している必要があります。
  • 変更を加える際は、シグナルハンドラ内での安全性を維持するため、ロックを使用しない設計を維持してください。
