目的

ECPG（Embedded SQL for C）における日付、タイムスタンプ、および間隔（interval）型の変換ロジックとフォーマット機能の正確性を検証する回帰テストのエントリポイント。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は受け取らない）
  • 出力: 成功時に 0 を返し、標準出力にテスト結果のログを出力する。エラー発生時は sqlprint() により標準エラー出力へSQLエラー情報が出力される。

前提条件と不変条件

  • ECPGプリプロセッサによって生成されたコードであり、実行にはデータベース接続環境（ecpg1_regression）が必要。
  • 実行中に date_test テーブルの作成および操作を行うため、データベースに対する書き込み権限が必要。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、データベースへ接続。 2. **環境設定**: datestyle を iso に、intervalstyle を postgres_verbose に設定。 3. **データ型変換テスト**:

  • PGTYPES*_from_asc を使用して文字列から内部表現へ変換。
  • PGTYPES*_to_asc を使用して内部表現から文字列へ変換し、結果を標準出力へ表示。
  • PGTYPESdate_mdyjul / PGTYPESdate_julmdy によるユリウス日変換の検証。
  • PGTYPESdate_fmt_asc および PGTYPEStimestamp_fmt_asc を用いたカスタムフォーマットの入出力検証。
  • PGTYPESdate_defmt_asc を用いた多様な日付文字列フォーマットの解析検証。

    • 4. **DB操作**: ECPGdo を介してSQLの挿入および選択を行い、ECPGの型マッピングの整合性を確認。 5. **終了**: 各種メモリ（text, out）を解放し、テストを終了する。

呼び出し元と呼び出し先

  • 呼び出し元: OSのプロセス実行環境（エントリポイント）。
  • 呼び出し先: ECPGconnect, ECPGdo, PGTYPES* 系API（PGTYPESdate_from_asc, PGTYPEStimestamp_to_asc 等）、sqlprint, printf, malloc, free。

関連データ構造

  • date, timestamp, interval: ECPGで定義される日付・時刻型。
  • sqlca: SQL通信領域（エラーコード確認に使用）。

失敗モードと境界条件

  • SQLエラー発生時: sqlca.sqlcode < 0 の場合に sqlprint() が呼び出される。
  • メモリ不足: malloc 失敗時のハンドリングは明示されていない。
  • フォーマット解析失敗: PGTYPESdate_defmt_asc 等で不正な入力文字列が与えられた場合の挙動は、ライブラリ側の仕様に依存する。

保守担当者向けメモ

  • 本ファイルはプリプロセッサ生成物であるため、直接編集せず dt_test.pgc を修正すること。
  • テスト結果は回帰テストの期待値として固定されているため、出力フォーマットを変更する場合は期待値ファイルとの整合性に注意が必要。
