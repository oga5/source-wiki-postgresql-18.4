parse_kvpairs_for_auth

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-oauth.c:385

目的

SASL OAUTHBEARER メッセージから、RFC 7628 に基づくキー・値ペア（kvpair）を解析し、認証トークン（auth キーの値）を抽出する。

シグネチャまたは形状

static char *parse_kvpairs_for_auth(char **input)

入力と出力

  • **入力**: char **input - 解析対象の文字列へのポインタ。呼び出し元でヌル終端されていることが前提。
  • **出力**: char * - 抽出された auth キーに対応する値のポインタ。見つからない場合は NULL。

前提条件と不変条件

  • gs2-header および最初の kvsep は解析済みであること。
  • 入力文字列は KVSEP（%x01）で区切られた key=value ペアの形式であること。
  • 認証トークン（auth キー）はメッセージ内に最大で1つのみ存在すること。

アルゴリズムまたは振る舞い

1. KVSEP を区切り文字として、入力文字列を順次走査する。 2. 空のペア（KVSEP が連続する箇所）に到達した場合、解析を終了し、現在の auth ポインタを返す。 3. 各ペアについて = を検索し、キーと値を分離する。 4. validate_kvpair を呼び出し、キーと値の妥当性を検証する。 5. キーが AUTH_KEY と一致する場合、その値を auth 変数に保持する。重複がある場合はエラーとする。 6. 未知のキーは無視する。 7. 最終的な終端子が見つからない場合、プロトコル違反としてエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-oauth.c:268 (oauth_exchange 内)
  • **呼び出し先**: validate_kvpair

関連データ構造

  • AUTH_KEY: 認証トークンを識別するための定数キー。
  • KVSEP: ペアを区切るための制御文字（%x01）。

失敗モードと境界条件

  • **プロトコル違反 (ERRCODE_PROTOCOL_VIOLATION)**:
  • KVSEP で終端されていないペアが存在する場合。
  • = が含まれないキーが存在する場合。
  • auth キーが複数回出現する場合。
  • 最終的な終端子（空のペア）が存在しない場合。

保守担当者向けメモ

  • 本関数は RFC 7628 の仕様に基づき、host や port といった OAUTHBEARER で使用されないキーを明示的に無視する設計となっている。
  • 入力文字列は破壊的に変更（\0 による終端）されるため、呼び出し元でのメモリ管理に注意すること。
