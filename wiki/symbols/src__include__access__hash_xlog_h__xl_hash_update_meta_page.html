xl_hash_update_meta_page

定義

  • 種別: struct
  • 定義箇所: src/include/access/hash_xlog.h:202

目的

ハッシュインデックスのメタページ更新操作をWAL（Write-Ahead Logging）に記録するためのデータ構造です。

シグネチャまたは形状

typedef struct xl_hash_update_meta_page
{
    double      ntuples;
} xl_hash_update_meta_page;

入力と出力

  • 入力: ntuples（ハッシュインデックス内のタプル総数を示す double 型の値）
  • 出力: なし（本構造体はWALレコードのペイロードとして書き込まれます）

前提条件と不変条件

  • 本構造体は、ハッシュインデックスのメタページを更新する際のWALレコードの一部として使用されます。
  • ntuples は、更新後のインデックスにおける正確なタプル数である必要があります。

アルゴリズムまたは振る舞い

1. ハッシュインデックスの更新処理において、メタページの変更が必要な場合に本構造体がインスタンス化されます。 2. XLogRecGetData を通じてWALレコードから本構造体が読み取られます。 3. リカバリ処理（hash_redo）において、この構造体に含まれる ntuples の値がメタページに反映されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/hash/hash.c（メタページ更新時に構造体を初期化）
  • 呼び出し先: src/backend/access/hash/hash_xlog.c（リカバリ時にレコードからデータを抽出）、src/backend/access/rmgrdesc/hashdesc.c（ログ出力時にレコードを解析）

関連データ構造

  • xl_hash_init_meta_page: メタページの初期化に使用される構造体。
  • hash_redo: 本構造体を利用してリカバリを実行する関数。

失敗モードと境界条件

  • WALレコードの読み込み時にデータが破損している場合、XLogRecGetData によるキャスト後の値が不正になる可能性があります。
  • ntuples の値が負数や異常値である場合のバリデーションについては、本構造体定義自体には含まれていません。

保守担当者向けメモ

  • 本構造体は offsetof(xl_hash_update_meta_page, ntuples) + sizeof(double) としてサイズ計算が行われています。構造体メンバを追加・変更する場合は、この計算式に影響がないか確認してください。
