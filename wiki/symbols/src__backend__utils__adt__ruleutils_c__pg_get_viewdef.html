pg_get_viewdef

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ruleutils.c:677

目的

指定された OID を持つビューの定義を、人間が読み取り可能な SQL 文字列として逆コンパイル（deparse）して取得します。

シグネチャまたは形状

Datum pg_get_viewdef(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介してビューの OID を受け取ります。
  • **出力**: ビューの定義を表す text 型の値を返します。定義が見つからない場合は NULL を返します。

前提条件と不変条件

  • PRETTYFLAG_INDENT がデフォルトの整形フラグとして適用されます。
  • 内部的に WRAP_COLUMN_DEFAULT を使用して出力の折り返し制御が行われます。

アルゴリズムまたは振る舞い

1. 引数からビューの OID を取得します。 2. pg_get_viewdef_worker を呼び出し、インデント付きでビュー定義の文字列を生成します。 3. 生成された文字列が NULL の場合、SQL の NULL を返します。 4. 文字列が生成された場合、string_to_text を介して text 型に変換し、PG_RETURN_TEXT_P で返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c: ビュー定義のダンプ用クエリとして使用。
  • src/bin/psql/command.c および describe.c: psql の \d コマンド等でビュー定義を表示するために使用。
  • **呼び出し先**:
  • pg_get_viewdef_worker: 実際の逆コンパイル処理を実行する内部関数。

関連データ構造

  • Datum: PostgreSQL の汎用データ型。
  • PG_FUNCTION_ARGS: 関数呼び出しの引数コンテキスト。

失敗モードと境界条件

  • 指定された OID が存在しない、またはビューとして解決できない場合、pg_get_viewdef_worker が NULL を返し、本関数も NULL を返します。

保守担当者向けメモ

  • 本関数は OID による単一のビュー定義取得用です。名前による指定や、より詳細な整形オプションが必要な場合は、pg_get_viewdef_ext や pg_get_viewdef_name などの関連関数を参照してください。
  • pg_get_viewdef_worker の実装変更は、本関数および psql 等の出力形式に直接影響を与えます。
