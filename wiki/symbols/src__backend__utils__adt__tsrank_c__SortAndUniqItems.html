SortAndUniqItems

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsrank.c:154

目的

TSQuery 内の全オペランド（QI_VAL 型のアイテム）を抽出し、重複を除去したリストを作成する。

シグネチャまたは形状

QueryOperand **SortAndUniqItems(TSQuery q, int *size)

入力と出力

  • **入力**:
  • q: 対象の TSQuery 構造体。
  • size: クエリ内のアイテム総数へのポインタ（入力時は総数、出力時は重複除去後の要素数）。
  • **出力**:
  • QueryOperand 構造体へのポインタ配列。呼び出し元で palloc されたメモリを解放する必要がある。

前提条件と不変条件

  • size は TSQuery 内のアイテム数と一致している必要がある。
  • compareQueryOperand 関数がオペランドの比較に利用可能であること。

アルゴリズムまたは振る舞い

1. TSQuery から GETQUERY マクロでアイテム列を取得し、QI_VAL 型のアイテムのみをポインタ配列 res に収集する。 2. qsort_arg を使用し、compareQueryOperand を用いてオペランドをソートする。 3. ソート済み配列を走査し、隣接する重複要素を排除してリストを圧縮する。 4. 最終的な要素数を size に書き戻し、配列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsrank.c:222
  • src/backend/utils/adt/tsrank.c:302
  • **呼び出し先**:
  • compareQueryOperand（比較関数）
  • palloc（メモリ確保）

関連データ構造

  • TSQuery: 全文検索クエリの内部表現。
  • QueryItem: クエリ内の個々の要素。
  • QueryOperand: クエリ内のオペランド（検索語）。

失敗モードと境界条件

  • size が 2 未満の場合、ソートや重複除去を行わずそのまま配列を返す。
  • メモリ不足時は palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は TSQuery の構造を破壊せず、ポインタの配列を生成する。
  • compareQueryOperand の実装に依存しているため、比較ロジックを変更する場合は注意が必要。
