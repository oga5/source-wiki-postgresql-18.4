TransactionIdPrecedesOrEquals

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/transam.c:298

目的

2つのトランザクションID（XID）を比較し、id1 が論理的に id2 と等しいか、それよりも先行しているかを判定します。

シグネチャまたは形状

bool TransactionIdPrecedesOrEquals(TransactionId id1, TransactionId id2)

入力と出力

  • **入力**:
  • id1: 比較対象のトランザクションID。
  • id2: 比較対象のトランザクションID。
  • **出力**:
  • bool: id1 が id2 に対して論理的に先行または等しい場合に true を返します。

前提条件と不変条件

  • XIDは32ビットの循環値であるため、通常の比較演算子（<=）を直接使用することはできません。
  • TransactionIdIsNormal を満たさない特殊なXID（BootstrapTransactionId 等）が含まれる場合、通常の比較演算子が適用されます。

アルゴリズムまたは振る舞い

1. id1 または id2 が正規のXIDでない場合、単純な比較 id1 <= id2 の結果を返します。 2. 両者が正規のXIDである場合、差分 (int32)(id1 - id2) を計算します。 3. 計算された差分が 0 以下であれば true を返し、それ以外の場合は false を返します。これにより、XIDのラップアラウンドを考慮した前後関係の判定を行います。

関連データ構造

  • TransactionId: 32ビットのトランザクション識別子。

失敗モードと境界条件

  • XIDのラップアラウンドが発生する境界において、通常の比較演算子を使用すると誤った結果を招くため、本関数を使用する必要があります。
  • 特殊なXID（FrozenTransactionId 等）が入力された際の挙動は、TransactionIdIsNormal の定義に依存します。

保守担当者向けメモ

  • 本関数は、XIDの循環的な性質を扱うための標準的な比較ツールです。ロジックを変更する際は、TransactionIdPrecedes や TransactionIdFollows との整合性を維持してください。
