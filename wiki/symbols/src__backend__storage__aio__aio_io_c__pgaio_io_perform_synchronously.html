pgaio_io_perform_synchronously

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_io.c:115

目的

非同期I/Oサブシステムにおいて、同期的なファイルI/O（preadv/pwritev）を実行し、その結果を PgAioHandle に反映させるためのフォールバックまたは直接実行用インターフェースを提供します。

シグネチャまたは形状

void pgaio_io_perform_synchronously(PgAioHandle *ioh)

入力と出力

  • **入力**: PgAioHandle *ioh - 実行対象のI/Oハンドル。操作内容（read/write）、ファイル記述子、オフセット、バッファ情報を含みます。
  • **出力**: なし（副作用として ioh->result に実行結果を格納し、完了処理をトリガーします）。

前提条件と不変条件

  • ioh は有効な PgAioHandle を指している必要があります。
  • START_CRIT_SECTION() により、実行中はクリティカルセクション内に置かれます。

アルゴリズムまたは振る舞い

1. START_CRIT_SECTION() を呼び出し、処理の不可分性を保証します。 2. ioh->op に基づき、PGAIO_OP_READV または PGAIO_OP_WRITEV を選択します。 3. 適切な待機イベント（WAIT_EVENT_DATA_FILE_READ または WAIT_EVENT_DATA_FILE_WRITE）を報告しつつ、pg_preadv または pg_pwritev を実行します。 4. 実行結果を ioh->result に格納します（エラー時は -errno を設定）。 5. pgaio_io_process_completion(ioh, ioh->result) を呼び出し、完了処理を実行します。 6. END_CRIT_SECTION() でクリティカルセクションを終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio.c:476
  • src/backend/storage/aio/method_worker.c:289, 565
  • **呼び出し先**:
  • pg_preadv / pg_pwritev (低レベルファイルI/O)
  • pgstat_report_wait_start / pgstat_report_wait_end (統計情報)
  • pgaio_io_process_completion (完了処理)
  • elog (エラーハンドリング)

関連データ構造

  • PgAioHandle: I/O操作のパラメータと結果を保持する構造体。
  • PgAioOp: 操作種別（READV, WRITEV, INVALID）。

失敗モードと境界条件

  • PGAIO_OP_INVALID が指定された場合、elog(ERROR) により処理が中断されます。
  • I/O操作が負の値を返した場合、errno が ioh->result に格納されます。

保守担当者向けメモ

  • 本関数はクリティカルセクション内で実行されるため、呼び出し元は割り込みが許可されていない状態であることを確認する必要があります。
  • pgaio_ctl->iovecs を介して iovec 配列にアクセスするため、ioh->iovec_off の整合性が保たれていることが前提となります。
