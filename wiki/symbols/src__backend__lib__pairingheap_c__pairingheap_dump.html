pairingheap_dump

定義

  • 種別: function
  • 定義箇所: src/backend/lib/pairingheap.c:317

目的

ペアリングヒープの構造を文字列としてダンプし、デバッグや可視化を支援する。

シグネチャまたは形状

char *pairingheap_dump(pairingheap *heap,
                       void (*dumpfunc) (pairingheap_node *node, StringInfo buf, void *opaque),
                       void *opaque)

入力と出力

  • **入力**:
  • heap: ダンプ対象の pairingheap 構造体へのポインタ。
  • dumpfunc: 各ノードの内容を StringInfo バッファに書き込むためのコールバック関数。
  • opaque: dumpfunc に渡される任意のユーザーデータ。
  • **出力**:
  • ヒープ構造を表現する文字列（palloc されたメモリ領域）。呼び出し側で解放が必要。

前提条件と不変条件

  • heap は有効な pairingheap 構造体を指している必要がある。
  • PAIRINGHEAP_DEBUG が定義されている環境での利用を想定している。

アルゴリズムまたは振る舞い

1. ヒープが空（ph_root が NULL）の場合、"(empty)" という文字列を複製して返す。 2. ヒープが空でない場合、initStringInfo でバッファを初期化する。 3. pairingheap_dump_recurse を呼び出し、ルートノードから再帰的にヒープ構造を走査してバッファに書き込む。 4. 構築された StringInfo のデータ部分を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は提供されていない。
  • **呼び出し先**:
  • initStringInfo
  • pairingheap_dump_recurse
  • pstrdup

関連データ構造

  • pairingheap: ヒープ全体を管理する構造体。
  • pairingheap_node: ヒープ内の各要素。
  • StringInfo: PostgreSQL の動的文字列バッファ。

失敗モードと境界条件

  • heap が NULL の場合、プログラムはクラッシュする可能性がある。
  • メモリ不足時には palloc 関連の関数によりエラーが発生する。

保守担当者向けメモ

  • 本関数はデバッグ用であるため、本番環境のパフォーマンスクリティカルなパスでは使用しないこと。
  • dumpfunc を通じてノードごとの詳細情報を柔軟に出力できる設計になっている。
