PGP_COMPR_TYPE

定義

  • 種別: enum
  • 定義箇所: contrib/pgcrypto/pgp.h:90

目的

OpenPGP (RFC 4880) 規格で定義されるデータ圧縮アルゴリズムを識別するための列挙型です。

シグネチャまたは形状

enum PGP_COMPR_TYPE
{
    PGP_COMPR_NONE = 0,
    PGP_COMPR_ZIP = 1,
    PGP_COMPR_ZLIB = 2,
    PGP_COMPR_BZIP2 = 3,
};

入力と出力

  • 入力: なし（定数定義）
  • 出力: PGP_COMPR_TYPE 型の列挙値

前提条件と不変条件

  • PGP_COMPR_NONE は必須（must）として扱われます。
  • PGP_COMPR_ZIP は推奨（should）として扱われます。

アルゴリズムまたは振る舞い

PGP データの圧縮方式を指定するために使用されます。各値は RFC 4880 に準拠した圧縮アルゴリズムに対応しており、暗号化処理の過程で適切な圧縮フィルタを選択する際の識別子として機能します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース抜粋には、この列挙型を参照している具体的な関数呼び出しは含まれていません。
  • 呼び出し先: なし（定数定義のため）

関連データ構造

  • PGP_Context: 暗号化/復号のセッション状態を保持する構造体であり、本列挙型を用いて圧縮設定を管理する可能性があります。

失敗モードと境界条件

  • 定義外の整数値をキャストして代入した場合の挙動は未定義です。

保守担当者向けメモ

  • RFC 4880 に強く依存しています。新しい圧縮アルゴリズムを追加する場合は、RFC の仕様を確認してください。
  • pgp-compress.c が本列挙型を利用した圧縮処理の実装を保持している可能性が高いため、詳細なロジックを確認する際は同ファイルを調査してください。
