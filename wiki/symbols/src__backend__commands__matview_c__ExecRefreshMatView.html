ExecRefreshMatView

定義

  • 種別: function
  • 定義箇所: src/backend/commands/matview.c:120

目的

REFRESH MATERIALIZED VIEW SQLコマンドの実行を開始し、対象のマテリアライズド・ビュー（MV）に対する適切なロックの取得と、更新処理の委譲を行う。

シグネチャまたは形状

ObjectAddress ExecRefreshMatView(RefreshMatViewStmt *stmt, const char *queryString, QueryCompletion *qc)

入力と出力

  • **入力**:
  • stmt: REFRESH MATERIALIZED VIEW 文の解析結果構造体。
  • queryString: 実行元のクエリ文字列。
  • qc: クエリ完了状態を保持する構造体。
  • **出力**:
  • 更新されたMVの ObjectAddress を返す。

前提条件と不変条件

  • stmt->concurrent が真の場合、ExclusiveLock を取得する。
  • stmt->concurrent が偽の場合、AccessExclusiveLock を取得する。
  • ロックはトランザクション終了まで保持される。

アルゴリズムまたは振る舞い

1. stmt->concurrent の値に基づき、必要なロックモード（ExclusiveLock または AccessExclusiveLock）を決定する。 2. RangeVarGetRelidExtended を呼び出し、対象MVのOIDを取得すると同時にロックを獲得する。この際、RangeVarCallbackMaintainsTable コールバックを使用してテーブルの整合性を検証する。 3. 取得したOIDと引数を RefreshMatViewByOid に渡し、実際の更新処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c:1693 (SQLコマンド実行のディスパッチャ)
  • **呼び出し先**:
  • RangeVarGetRelidExtended
  • RefreshMatViewByOid

関連データ構造

  • RefreshMatViewStmt: REFRESH 文の構文解析結果を保持する構造体。
  • ObjectAddress: オブジェクトの識別子。
  • QueryCompletion: クエリ実行結果のメタデータ。

失敗モードと境界条件

  • 対象の RangeVar が存在しない、あるいはMVではない場合、RangeVarGetRelidExtended によりエラーが発生する。
  • ロック取得時にデッドロックが発生した場合、PostgreSQLの標準的なロック管理機構により処理が中断される。

保守担当者向けメモ

  • 本関数は単なるエントリーポイントであり、実際の更新ロジックは RefreshMatViewByOid に集約されている。
  • CONCURRENTLY 指定時のロック強度の違いが、同時実行制御の成否に直結するため、ロックモードの変更には注意が必要である。
