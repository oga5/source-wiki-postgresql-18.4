ExecSort

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSort.c:49

目的

クエリ実行エンジンにおいて、下位ノードから取得したタプルをソートし、ソート済みシーケンスとして提供する。

シグネチャまたは形状

TupleTableSlot *ExecSort(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（SortState 構造体としてキャストして使用）
  • **出力**: ソート済みの TupleTableSlot *。ソート完了後は順次取得したタプルを返し、終了時は空のスロットを返す。

前提条件と不変条件

  • 初回呼び出し時に node->sort_Done が false であること。
  • CHECK_FOR_INTERRUPTS() が呼び出され、クエリキャンセル要求を監視していること。
  • datumSort フラグにより、単一 Datum ソートかタプルソートかが決定される。

アルゴリズムまたは振る舞い

1. **初回実行時**:

  • tuplesort_begin_datum または tuplesort_begin_heap を用いてソート状態を初期化。
  • 下位ノード（outerNode）から全タプルを読み込み、tuplesort_put... でソート用バッファへ投入。
  • tuplesort_performsort を呼び出し、ソートを実行。
  • 並列ワーカーの場合は、統計情報を shared_info に格納。
  • node->sort_Done を true に設定。

    • 2. **2回目以降の呼び出し**:

  • tuplesort_getdatum または tuplesort_gettupleslot を使用し、ソート済みデータを順次取得して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジンのノード処理（ExecProcNode 経由で呼び出される）。
  • **呼び出し先**: tuplesort.c の各関数（tuplesort_begin_*, tuplesort_put*, tuplesort_performsort, tuplesort_get*）、ExecProcNode（下位ノードの取得）。

関連データ構造

  • SortState: ノードの状態を保持する構造体。
  • Tuplesortstate: tuplesort.c が管理するソート状態。
  • TupleTableSlot: タプルデータを保持するスロット。

失敗モードと境界条件

  • **ブロッキング**: 全入力を読み込むまで最初のタプルを返せないため、パイプライン処理のボトルネックとなる。
  • **メモリ制限**: work_mem を超える場合は一時ファイルが使用される。
  • **割り込み**: CHECK_FOR_INTERRUPTS() により、ソート処理中にユーザーによるキャンセルが可能。

保守担当者向けメモ

  • datumSort は単一カラムの場合のみ適用される最適化である。
  • ExecSort 内の CHECK_FOR_INTERRUPTS() は、大規模ソート時の応答性を維持するために不可欠である。
  • 並列クエリ実行時の統計情報収集ロジックが含まれている点に注意すること。
