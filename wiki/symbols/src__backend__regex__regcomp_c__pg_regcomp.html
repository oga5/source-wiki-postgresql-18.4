pg_regcomp

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regcomp.c:371

目的

正規表現パターンをコンパイルし、実行エンジンで使用可能な regex_t 構造体および内部の NFA（非決定性有限オートマトン）とサブ式ツリーを構築する。

シグネチャまたは形状

int pg_regcomp(regex_t *re, const chr *string, size_t len, int flags, Oid collation)

入力と出力

  • **入力**:
  • re: 初期化対象の regex_t 構造体へのポインタ。
  • string: コンパイル対象の正規表現パターン文字列。
  • len: パターンの長さ。
  • flags: コンパイルオプション（REG_EXTENDED, REG_ICASE 等）。
  • collation: ロケール照合順序。
  • **出力**:
  • 成功時は 0 を返す。
  • 失敗時はエラーコード（REG_INVARG, REG_ESPACE 等）を返す。

前提条件と不変条件

  • re および string は NULL であってはならない。
  • REG_QUOTE フラグと REG_ADVANCED 等の競合するフラグを同時に指定することはできない。
  • コンパイル処理中、内部の vars 構造体が状態を保持し、エラー発生時には freev() を介してリソースが解放される。

アルゴリズムまたは振る舞い

1. **初期化**: ロケール設定を行い、vars 構造体および guts 構造体をセットアップする。 2. **解析**: lexstart() で字句解析を開始し、parse() を再帰的に呼び出して NFA とサブ式ツリーを構築する。 3. **最適化**: optimize() および makesearch() を実行し、NFA を検索効率の良い形式に変換する。 4. **コンパクト化**: compact() を用いて実行エンジン用の圧縮 NFA を生成する。 5. **終了処理**: 構築した情報を regex_t に格納し、一時的な作業領域を freev() で解放して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_trgm/trgm_regexp.c
  • src/backend/libpq/hba.c
  • src/backend/tsearch/spell.c
  • src/backend/utils/adt/regexp.c
  • src/test/modules/test_regex/test_regex.c
  • **呼び出し先**:
  • pg_set_regex_collation, lexstart, parse, optimize, makesearch, compact, freev 等の内部関数。

関連データ構造

  • regex_t: 公開される正規表現オブジェクト。
  • struct vars: コンパイル中の状態を保持するコンテキスト。
  • struct guts: コンパイル結果（NFA、カラーマップ、ツリー等）を保持する内部構造体。
  • struct subre: 正規表現の構造を表現するツリーノード。

失敗モードと境界条件

  • 引数が NULL の場合、REG_INVARG を返す。
  • メモリ割り当て失敗時、REG_ESPACE を返す。
  • 不正なフラグの組み合わせが指定された場合、REG_INVARG を返す。
  • 解析中にエラーが発生した場合、freev() を呼び出してリソースを解放し、エラーコードを返す。

保守担当者向けメモ

  • 本関数は regcomp.c のメインエントリポイントであり、コンパイルの全工程を統括する。
  • vars 構造体は多くの内部関数で共有されるため、フィールドの変更には注意が必要。
  • REG_DEBUG マクロを有効にすると、コンパイル過程のダンプ出力が可能。
