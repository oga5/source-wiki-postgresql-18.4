init_MultiFuncCall

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/funcapi.c:132

目的

集合を返す関数（SRF）の初回呼び出し時に、複数回の呼び出しにまたがって状態を保持するための FuncCallContext を初期化し、メモリコンテキストを構築する。

シグネチャまたは形状

FuncCallContext *init_MultiFuncCall(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（fcinfo を含む関数呼び出し情報）。
  • **出力**: 初期化された FuncCallContext 構造体へのポインタ。

前提条件と不変条件

  • fcinfo->resultinfo が ReturnSetInfo 型として有効であること。
  • 本関数はSRFの初回呼び出し時にのみ実行されること（2回目以降の呼び出しはエラーとなる）。

アルゴリズムまたは振る舞い

1. **コンテキスト検証**: fcinfo->resultinfo が ReturnSetInfo であるかを確認し、不適切な場合はエラーを報告する。 2. **初回呼び出し判定**: fcinfo->flinfo->fn_extra が NULL であることを確認する。 3. **メモリ確保**: fn_mcxt を親として、SRFのライフサイクルを管理するための長寿命なメモリコンテキスト（SRF multi-call context）を作成する。 4. **構造体初期化**: FuncCallContext をゼロ初期化し、各フィールド（call_cntr, max_calls, user_fctx 等）をセットアップする。 5. **状態保存**: fcinfo->flinfo->fn_extra に作成したコンテキストを保存し、後続の呼び出しで参照可能にする。

呼び出し元と呼び出し先

  • **呼び出し元**: SRFの実装内（通常は SRF_FIRSTCALL_INIT() マクロ経由）。
  • **呼び出し先**: AllocSetContextCreate, MemoryContextAllocZero, RegisterExprContextCallback, elog（エラー時）。

関連データ構造

  • FuncCallContext: SRFの状態保持用構造体。
  • ReturnSetInfo: SRFの実行コンテキスト情報。
  • FmgrInfo: 関数呼び出し情報（fn_extra を保持）。

失敗モードと境界条件

  • **コンテキストエラー**: SRFとして呼び出されていない場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラー終了する。
  • **二重初期化**: 既に初期化済みの状態で呼び出された場合、elog(ERROR) により強制終了する。

保守担当者向けメモ

  • 本関数は SRF_FIRSTCALL_INIT() マクロを通じて利用されることが前提である。
  • fn_extra を介した状態管理を行っているため、SRFの実装側で fn_extra を直接書き換えないよう注意が必要である。
