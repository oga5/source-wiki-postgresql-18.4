run_all_permutations_recurse

定義

  • 種別: function
  • 定義箇所: src/test/isolation/isolationtester.c:445

目的

テスト仕様（TestSpec）に基づき、各セッションのステップを可能な限りすべての順列で生成し、各順列が完成するたびに実行する。

シグネチャまたは形状

static void run_all_permutations_recurse(TestSpec *testspec, int *piles, int nsteps, PermutationStep **steps)

入力と出力

  • testspec: テスト対象の仕様構造体。
  • piles: 各セッションで現在何番目のステップまで処理したかを保持する配列。
  • nsteps: 現在の再帰深さ（これまでに蓄積されたステップ数）。
  • steps: 生成中の順列を格納する PermutationStep ポインタの配列。
  • 戻り値: なし。

前提条件と不変条件

  • steps 配列は、全ステップ数を格納するのに十分なサイズが確保されている必要がある。
  • piles 配列の各要素は、対応するセッションのステップ数を超えない範囲で更新される。
  • run_all_permutations によって、steps 配列内の未使用フィールドは事前にゼロクリアされていることが期待される。

アルゴリズムまたは振る舞い

1. 各セッションに対してループを行い、まだ実行していないステップが存在するかを確認する。 2. ステップが存在する場合、そのステップを steps[nsteps] に割り当て、piles をインクリメントして再帰的に自身を呼び出す。 3. 再帰呼び出しから戻った後、piles をデクリメントしてバックトラックを行う。 4. すべてのセッションのステップを使い果たし、found フラグが false のままループを抜けた場合、順列が完成したとみなして run_permutation を呼び出し、テストを実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/isolation/isolationtester.c:438 (run_all_permutations 内)
  • src/test/isolation/isolationtester.c:469 (自身による再帰呼び出し)
  • **呼び出し先**:
  • run_permutation (順列完成時)

関連データ構造

  • TestSpec: テスト全体の仕様。
  • PermutationStep: 順列を構成する個別のステップ情報。
  • Step: セッション内の個別の操作単位。

失敗モードと境界条件

  • ステップ数が非常に多い場合、再帰によるスタックオーバーフローのリスクがある。
  • testspec が適切に初期化されていない場合、セッションやステップへのアクセスで不正なメモリ参照が発生する可能性がある。

保守担当者向けメモ

  • 本関数は自動生成される順列の探索を行うため、PermutationStep にブロック条件（blocker）を付与しない前提で設計されている。
  • piles 配列の管理はバックトラッキングの要であるため、変更時はステップのインクリメント・デクリメントの整合性に注意すること。
