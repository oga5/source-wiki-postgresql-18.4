gbt_uuid_fetch

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_uuid.c:124

目的

GiST インデックスのリーフノードから、UUID 型の値をフェッチ（取得）します。

シグネチャまたは形状

Datum gbt_uuid_fetch(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体へのポインタを受け取ります。
  • **出力**: Datum 型として、インデックスから取得された UUID 値を返します。

前提条件と不変条件

  • PG_GETARG_POINTER(0) により、有効な GISTENTRY へのポインタが取得できる必要があります。
  • 静的変数 tinfo（gbtree_ninfo 型）が、UUID 型のインデックス操作に必要なメタデータとして正しく初期化されている必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を呼び出し、引数から GISTENTRY 構造体を取得します。 2. 取得した entry と、モジュール内で定義された tinfo を引数として gbt_num_fetch を呼び出します。 3. gbt_num_fetch の戻り値を PG_RETURN_POINTER マクロで返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス管理機構（直接的な呼び出し元コードはソース内に明示されていません）。
  • **呼び出し先**: gbt_num_fetch（btree_utils_num.c で定義された汎用関数）。

関連データ構造

  • GISTENTRY: GiST インデックスのノード情報を保持する構造体。
  • tinfo: UUID 型の比較関数や型情報を保持する gbtree_ninfo 型の静的変数。

失敗モードと境界条件

  • entry が NULL である場合や、tinfo が適切に設定されていない場合、gbt_num_fetch の内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で提供される汎用的なフェッチロジックのラッパーです。UUID 特有の複雑な処理は含まれておらず、インデックスの型情報（tinfo）を介して汎用関数へ処理を委譲しています。
