deccall3

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/ecpg/compatlib/informix.c:85

目的

Informix の decimal 型に対する算術演算を、PostgreSQL の numeric 型を用いた演算へ委譲するための内部ラッパー関数です。

シグネチャまたは形状

static int deccall3(decimal *arg1, decimal *arg2, decimal *result, int (*ptr) (numeric *, numeric *, numeric *))

入力と出力

  • **入力**:
  • arg1, arg2: 演算対象となる decimal 型のポインタ。
  • result: 演算結果を格納する decimal 型のポインタ。
  • ptr: 実行する numeric 型演算関数への関数ポインタ。
  • **出力**:
  • 成功時は 0 を返し、result に計算結果を格納します。
  • 失敗時は ECPG_INFORMIX_OUT_OF_MEMORY などのエラーコードを返します。

前提条件と不変条件

  • arg1 または arg2 が NULL 値（risnull で判定）である場合、演算は行われず即座に 0 を返します。

アルゴリズムまたは振る舞い

1. arg1 と arg2 が NULL か確認し、該当すれば終了します。 2. PGTYPESnumeric_new() を用いて、計算用の一時的な numeric 型変数（a1, a2, nres）を確保します。 3. PGTYPESnumeric_from_decimal() を使用し、入力された decimal 値を numeric 型へ変換します。 4. 引数 ptr で渡された関数を実行し、結果を nres に格納します。 5. 演算が成功（i == 0）した場合、rsetnull() で result を初期化した後、PGTYPESnumeric_to_decimal() で nres の値を result に書き戻します。 6. 確保した一時変数を PGTYPESnumeric_free() で解放し、演算結果のステータスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • decadd (加算)
  • decdiv (除算)
  • decmul (乗算)
  • decsub (減算)
  • **呼び出し先**:
  • risnull, rsetnull
  • PGTYPESnumeric_new, PGTYPESnumeric_free
  • PGTYPESnumeric_from_decimal, PGTYPESnumeric_to_decimal
  • ptr 引数で渡される PGTYPESnumeric_* 演算関数

関連データ構造

  • decimal: Informix 互換の数値型。
  • numeric: PostgreSQL 内部の数値型。

失敗モードと境界条件

  • メモリ不足時: PGTYPESnumeric_new() が失敗した場合、確保済みのメモリを解放し ECPG_INFORMIX_OUT_OF_MEMORY を返します。
  • 変換失敗時: PGTYPESnumeric_from_decimal() が失敗した場合、同様にメモリを解放しエラーを返します。

保守担当者向けメモ

  • 本関数は decimal 型の演算を numeric 型に変換して処理する共通パターンを実装しています。新しい decimal 演算関数を追加する際は、このラッパーを利用することが推奨されます。
