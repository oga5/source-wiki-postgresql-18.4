pg_tm

定義

  • 種別: struct
  • 定義箇所: src/include/pgtime.h:34

目的

PostgreSQL 内部での時刻表現を保持するための構造体です。標準 C ライブラリの struct tm に似ていますが、PostgreSQL の日付時刻関数との整合性を考慮して定義されています。

シグネチャまたは形状

struct pg_tm
{
    int         tm_sec;
    int         tm_min;
    int         tm_hour;
    int         tm_mday;
    int         tm_mon;
    int         tm_year;
    int         tm_wday;
    int         tm_yday;
    int         tm_isdst;
    long int    tm_gmtoff;
    const char *tm_zone;
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。pg_localtime() や pg_gmtime() などの変換関数を通じて、pg_time_t 型から本構造体へ値が書き込まれます。

前提条件と不変条件

  • tm_mon（月）は 0 起点、tm_year（年）は 1900 年基準という IANA タイムゾーンライブラリの慣習に従います。
  • PostgreSQL の他の日付時刻関数（1 起点や紀元前基準）とは基準が異なるため、相互運用時には調整が必要です。

アルゴリズムまたは振る舞い

時刻の各要素（秒、分、時、日、月、年、曜日、通算日、夏時間フラグ、UTC オフセット、タイムゾーン名）を個別のフィールドとして保持します。

呼び出し元と呼び出し先

  • **呼び出し元（参照例）**:
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/postmaster/syslogger.c
  • src/backend/utils/adt/date.c
  • **呼び出し先**: 本構造体はデータ型であるため、直接的な呼び出し先はありません。

関連データ構造

  • pg_time_t: 64 ビット幅の時刻表現型。
  • pg_tz: タイムゾーン情報を保持する不透明な構造体。

失敗モードと境界条件

  • 構造体内部の各フィールド値が、IANA タイムゾーンライブラリの仕様範囲外となった場合の挙動は、変換を行う関数（pg_localtime 等）の仕様に依存します。

保守担当者向けメモ

  • tm_mon や tm_year の基準値が PostgreSQL の他の日付時刻関数と異なる点に注意してください。
  • タイムゾーン関連の操作を行う際は、内部構造を直接操作せず、提供されている API 関数を経由してください。
