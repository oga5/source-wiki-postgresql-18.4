buf_add_txid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xid8funcs.c:236

目的

StringInfo バッファに FullTransactionId を追加し、スナップショット内のトランザクションIDカウントを更新します。

シグネチャまたは形状

static void buf_add_txid(StringInfo buf, FullTransactionId fxid)

入力と出力

  • **入力**:
  • buf: StringInfo 型。pg_snapshot 構造体として解釈されるデータ領域を保持。
  • fxid: FullTransactionId 型。バッファに追加する64ビットトランザクションID。
  • **出力**: なし（副作用として buf を更新）。

前提条件と不変条件

  • buf->data は pg_snapshot 構造体としてキャスト可能である必要があります。
  • appendBinaryStringInfo によるバッファの再割り当て（realloc）が発生する可能性があるため、カウントのインクリメントは再割り当て前に行われます。

アルゴリズムまたは振る舞い

1. buf->data を pg_snapshot * にキャストします。 2. snap->nxip（スナップショット内のトランザクションID数）をインクリメントします。 3. appendBinaryStringInfo を呼び出し、fxid の値をバッファの末尾にバイナリ形式で追記します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/xid8funcs.c:309（文脈から、スナップショット構築プロセスの一部として呼び出されていることが示唆されます）。
  • **呼び出し先**: appendBinaryStringInfo（PostgreSQL 内部の文字列バッファ操作関数）。

関連データ構造

  • StringInfo: 可変長文字列バッファ。
  • FullTransactionId: 64ビットのトランザクションID。
  • pg_snapshot: トランザクションIDの配列を保持するスナップショット構造体。

失敗モードと境界条件

  • buf が指すメモリ領域が不正な場合、セグメンテーション違反が発生する可能性があります。
  • appendBinaryStringInfo 内部でメモリ不足が発生した場合、PostgreSQL のエラーハンドリング機構によりアボートされます。

保守担当者向けメモ

  • 本関数は pg_snapshot の構築過程で使用されます。nxip の更新とバッファへの書き込み順序が重要であり、変更を加える際は pg_snapshot 構造体のメモリレイアウトとの整合性に注意してください。
