pg_base64_decode

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-armor.c:94

目的

Base64 エンコードされた ASCII 文字列をバイナリデータに復元します。

シグネチャまたは形状

int pg_base64_decode(const uint8 *src, unsigned len, uint8 *dst)

入力と出力

  • **入力**:
  • src: デコード対象の Base64 文字列（uint8 ポインタ）。
  • len: 入力文字列の長さ。
  • dst: デコード結果を格納するバッファ（十分なサイズが必要）。
  • **出力**:
  • 成功時: デコードされたバイナリデータのバイト数（int）。
  • 失敗時: PXE_PGP_CORRUPT_ARMOR（エラーコード）。

前提条件と不変条件

  • 入力文字列には A-Z, a-z, 0-9, +, / のみが含まれる必要があります。
  • 空白文字（スペース、タブ、改行、復帰）は無視されます。
  • パディング文字 = は、Base64 ブロックの終端として正しく配置されている必要があります。

アルゴリズムまたは振る舞い

1. 入力文字列を 1 文字ずつ走査し、Base64 値（0-63）に変換します。 2. 4 文字ごとに 24 ビットのバッファに蓄積し、3 バイトのバイナリデータとして dst に書き出します。 3. = が出現した場合、パディング位置を記録し、最終ブロックのデコード時に余分なバイトを切り捨てます。 4. 処理終了時にバッファ内に未処理のデータが残っている場合、または不正な文字やパディング配置が検出された場合はエラーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-armor.c 内の pgp_armor_decode 等から呼び出されます。
  • src/backend/utils/adt/encode.c に同名の関数が存在しますが、本シンボルとは実装が異なります。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • uint8: PostgreSQL の基本型（unsigned char）。

失敗モードと境界条件

  • **PXE_PGP_CORRUPT_ARMOR**:
  • 不正な文字が含まれている場合。
  • パディング = が不適切な位置にある場合。
  • 入力文字列の長さが 4 の倍数（有効な文字のみを考慮）にならない場合。

保守担当者向けメモ

  • 本実装は contrib/pgcrypto 固有のものであり、PostgreSQL の他のモジュール（src/backend/utils/adt/encode.c など）にある Base64 実装と重複しています。修正時はこの重複に注意してください。
