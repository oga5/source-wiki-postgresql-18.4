run_simple_command

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/libpq_source.c:190

目的

リモートの PostgreSQL サーバーに対して SQL コマンドを実行し、その結果が成功したことを保証する。

シグネチャまたは形状

static void run_simple_command(PGconn *conn, const char *sql)

入力と出力

  • **入力**:
  • conn: 接続済みの PGconn オブジェクト。
  • sql: 実行する SQL コマンド文字列。
  • **出力**: なし（成功時は正常終了、失敗時は pg_fatal によりプロセスを終了）。

前提条件と不変条件

  • conn は有効な接続状態であること。
  • 実行される SQL は、結果セットを返さない（コマンド完了のみを期待する）ものであること。

アルゴリズムまたは振る舞い

1. PQexec を使用して、指定された sql をリモートサーバーで実行する。 2. PQresultStatus を確認し、結果が PGRES_COMMAND_OK でない場合は pg_fatal を呼び出してエラーメッセージと共にプログラムを終了させる。 3. 成功した場合は PQclear を呼び出し、結果オブジェクトを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_rewind/libpq_source.c 内の初期化処理において、タイムアウト設定（statement_timeout 等）や読み取り専用モードへの移行を行う際に呼び出される。
  • **呼び出し先**:
  • libpq ライブラリの PQexec、PQresultStatus、PQresultErrorMessage、PQclear。

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • PGresult: libpq のクエリ結果ハンドル。

失敗モードと境界条件

  • SQL 実行がサーバー側で失敗した場合、pg_fatal により即座にプロセスが終了する。
  • 接続が切断されている場合、PQexec の挙動に依存してエラーが発生する。

保守担当者向けメモ

  • 本関数はコマンド実行の成否のみを判定するため、結果セットを取得する必要があるクエリには使用しないこと。
  • エラー発生時に pg_fatal を使用するため、呼び出し側で個別のエラーハンドリングを行う必要はない。
