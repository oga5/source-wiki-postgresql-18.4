ginCompressPostingList

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginpostinglist.c:196

目的

ItemPointer（TID）の配列を差分エンコーディング（Varbyte）を用いて圧縮し、指定されたサイズ制限内に収まる GinPostingList 構造体を生成します。

シグネチャまたは形状

GinPostingList *ginCompressPostingList(const ItemPointer ipd, int nipd, int maxsize, int *nwritten)

入力と出力

  • **入力**:
  • ipd: ソート済みの ItemPointer 配列。
  • nipd: 配列内の要素数。
  • maxsize: 圧縮後のデータが占有可能な最大バイト数。
  • nwritten: （オプション）実際に圧縮された要素数を格納するポインタ。
  • **出力**: 圧縮されたデータを保持する GinPostingList 構造体へのポインタ。

前提条件と不変条件

  • 入力 ipd は昇順にソートされている必要があります。
  • maxsize は SHORTALIGN 境界に切り捨てられます。
  • 圧縮後の GinPostingList のサイズは maxsize を超えません。
  • 内部の差分計算において、後続のTIDは常に先行するTIDより大きい必要があります。

アルゴリズムまたは振る舞い

1. 最初のTIDを result->first に格納し、これを基準点（prev）とします。 2. 後続の各TIDについて、prev との差分（Delta）を計算します。 3. encode_varbyte を使用して差分を可変長バイト列に変換し、バッファに書き込みます。 4. バッファの残容量が不足した場合は、その時点で処理を中断します。 5. 最終的なバイト数が奇数の場合、パディングバイトをゼロクリアします。 6. CHECK_ENCODING_ROUNDTRIP が定義されている場合、デコード結果が元データと一致するか検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: gindatapage.c、gininsert.c、ginvacuum.c、ginxlog.c の各関数から呼び出されます。
  • **呼び出し先**: itemptr_to_uint64、encode_varbyte、ginPostingListDecode（検証時のみ）。

関連データ構造

  • GinPostingList: 圧縮されたTIDリストを保持する構造体。
  • ItemPointer: GINインデックスで管理されるタプル識別子。

失敗モードと境界条件

  • maxsize が小さすぎる場合、最初のTIDのみが格納されるか、あるいは処理が即座に終了します。
  • バッファの末尾付近で MaxBytesPerInteger 未満の空きしかない場合、一時バッファを使用して書き込み可否を判定します。

保守担当者向けメモ

  • 本関数は差分エンコーディングの特性上、入力のソート順序に依存します。
  • MaxBytesPerInteger はエンコード時の最大バイト数であり、バッファオーバーフローを防ぐための重要な定数です。
  • 圧縮アルゴリズムの変更は、ディスク上のデータフォーマット互換性に直接影響するため注意してください。
