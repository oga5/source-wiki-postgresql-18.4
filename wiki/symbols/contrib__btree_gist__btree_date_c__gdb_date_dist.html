gdb_date_dist

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_date.c:90

目的

2つの DateADT 型の値の間の距離を計算し、float8 型として返します。

シグネチャまたは形状

static float8 gdb_date_dist(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の DateADT 値を指すポインタ。
  • flinfo: 関数マネージャ情報（本関数内では直接使用されず、DirectFunctionCall2 に渡される）。
  • **出力**:
  • 2つの日付間の差分（絶対値）を float8 型で返します。

前提条件と不変条件

  • a および b は有効な DateADT 値を指している必要があります。
  • 日付の差分計算においてオーバーフローが発生しないことを前提としています。

アルゴリズムまたは振る舞い

1. DirectFunctionCall2 を介して PostgreSQL コアの date_mi 関数を呼び出し、2つの日付の差分を算出します。 2. 算出された差分（Datum）を int32 に変換します。 3. abs() 関数を用いて差分の絶対値を取得し、float8 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の記述に基づくと、同ファイル内の他の関数から参照されている可能性がありますが、具体的な呼び出し元関数名は提供されていません。
  • **呼び出し先**:
  • date_mi (PostgreSQL コア関数)
  • DateADTGetDatum (マクロ)
  • DatumGetInt32 (マクロ)
  • abs (標準ライブラリ)

関連データ構造

  • DateADT: PostgreSQL の日付型表現。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、未定義の動作を引き起こします。
  • 日付の差分が int32 の範囲を超える場合、計算結果が不正確になる可能性があります（前提条件としてオーバーフローしないと仮定されています）。

保守担当者向けメモ

  • 本関数は btree_gist における date 型の距離計算ロジックを提供します。
  • date_mi を利用しているため、コア側の date_mi 実装に変更があった場合、影響を受ける可能性があります。
