xl_hash_vacuum_one_page

定義

  • 種別: struct
  • 定義箇所: src/include/access/hash_xlog.h:252

目的

ハッシュインデックスのバキューム処理において、単一ページからタプルを削除した際の変更内容をWAL（Write-Ahead Logging）に記録するためのデータ構造です。

シグネチャまたは形状

typedef struct xl_hash_vacuum_one_page
{
    TransactionId snapshotConflictHorizon;
    uint16      ntuples;
    bool        isCatalogRel;
    OffsetNumber offsets[FLEXIBLE_ARRAY_MEMBER];
} xl_hash_vacuum_one_page;

入力と出力

  • **入力**: バキューム対象のページ情報、削除対象のタプルオフセット、およびリカバリ時の競合解決に必要なメタデータ。
  • **出力**: WALレコードのペイロードとして書き込まれるバイナリデータ。

前提条件と不変条件

  • offsets は FLEXIBLE_ARRAY_MEMBER として定義されており、ntuples で指定された数だけ後続のメモリ領域に格納されます。
  • isCatalogRel は、スタンバイ側での論理デコーディング中にリカバリ競合を適切に処理するために使用されます。

アルゴリズムまたは振る舞い

1. **書き込み**: バキューム処理実行時に、削除したタプルのオフセット番号を offsets 配列に格納し、WALレコードとして書き出します。 2. **リカバリ**: hash_redo 関数がWALレコードを読み込み、この構造体にキャストして内容を取得します。その後、指定されたページから offsets に含まれるタプルを物理的に削除し、ページを更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/hash/hashinsert.c（WALレコードの生成時）
  • **呼び出し先**:
  • src/backend/access/hash/hash_xlog.c（リカバリ処理時）
  • src/backend/access/rmgrdesc/hashdesc.c（ログのデコード・表示時）

関連データ構造

  • SizeOfHashVacuumOnePage: offsetof(xl_hash_vacuum_one_page, offsets) として定義される、ヘッダー部分のサイズ定数。

失敗モードと境界条件

  • ntuples が0の場合、削除対象がないことを示しますが、構造体としては有効です。
  • offsets 配列のサイズがページ内の有効なオフセット数を超えないことが前提となります。

保守担当者向けメモ

  • 本構造体は可変長配列（FLEXIBLE_ARRAY_MEMBER）を含むため、サイズ計算には必ず SizeOfHashVacuumOnePage マクロを使用してください。
  • リカバリ時の競合処理（isCatalogRel）は、論理レプリケーションの整合性に直結するため、変更時には注意が必要です。
