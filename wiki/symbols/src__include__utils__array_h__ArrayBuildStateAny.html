ArrayBuildStateAny

定義

  • 種別: struct
  • 定義箇所: src/include/utils/array.h:226

目的

スカラー値の配列または配列の配列を構築する際、その構築状態を統一的に保持するためのコンテナ構造体です。

シグネチャまたは形状

typedef struct ArrayBuildStateAny
{
    ArrayBuildState *scalarstate;
    ArrayBuildStateArr *arraystate;
} ArrayBuildStateAny;

入力と出力

  • 入力: 構築対象となる要素（スカラーまたは配列）。
  • 出力: 最終的に makeArrayResult 等の関数を通じて生成される ArrayType。

前提条件と不変条件

  • scalarstate と arraystate のうち、必ず一方が NULL であり、もう一方が有効な状態を保持している必要があります。

アルゴリズムまたは振る舞い

  • 配列構築の初期化時に initArrayResultAny を使用して本構造体を生成します。
  • 構築対象がスカラーか配列かに応じて、内部の scalarstate または arraystate のいずれか一方のみが使用されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/executor/nodeSubplan.c
  • src/backend/utils/adt/array_userfuncs.c
  • 呼び出し先:
  • src/backend/utils/adt/arrayfuncs.c 内の initArrayResultAny 関数により初期化されます。

関連データ構造

  • ArrayBuildState: スカラー要素の配列構築用。
  • ArrayBuildStateArr: 配列の配列（多次元配列等）構築用。

失敗モードと境界条件

  • メモリ不足時には palloc 経由でエラーが発生する可能性があります。
  • 構築中に型不整合が発生した場合の挙動は、呼び出し元のロジックに依存します。

保守担当者向けメモ

  • 本構造体は、異なる構築状態を単一のインターフェースで扱うためのラッパーです。新しい構築ロジックを追加する際は、既存の scalarstate / arraystate の排他制御が維持されていることを確認してください。
