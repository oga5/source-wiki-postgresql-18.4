AdjustMonths

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/datetime.c:658

目的

pg_itm_in 構造体内の月（tm_mon）に対して、指定された値を加算し、オーバーフローチェックを行う。

シグネチャまたは形状

static bool AdjustMonths(int64 val, struct pg_itm_in *itm_in)

入力と出力

  • **入力**:
  • val: 加算する月数（int64）。
  • itm_in: 月の値を保持する pg_itm_in 構造体へのポインタ。
  • **出力**:
  • bool: 成功した場合は true、加算値が int32 の範囲外であるか、加算処理でオーバーフローが発生した場合は false を返す。

前提条件と不変条件

  • val は INT_MIN から INT_MAX の範囲内である必要がある。
  • itm_in は有効な pg_itm_in 構造体を指している必要がある。

アルゴリズムまたは振る舞い

1. val が int32 の範囲（INT_MIN ～ INT_MAX）に収まっているか確認する。範囲外であれば false を返す。 2. pg_add_s32_overflow を使用し、itm_in->tm_mon に val を加算する。 3. 加算結果を itm_in->tm_mon に格納し、オーバーフローが発生しなかった場合に true を返す。

関連データ構造

  • struct pg_itm_in: 日付・時刻の内部表現を保持する構造体。

失敗モードと境界条件

  • val が int32 の範囲を超えている場合、処理は失敗し false を返す。
  • itm_in->tm_mon への加算時に int32 のオーバーフローが発生した場合、処理は失敗し false を返す。

保守担当者向けメモ

  • 本関数は static であり、datetime.c 内でのみ使用される。
  • pg_add_s32_overflow の挙動に依存しているため、プラットフォーム固有の整数オーバーフロー処理の変更には注意が必要である。
