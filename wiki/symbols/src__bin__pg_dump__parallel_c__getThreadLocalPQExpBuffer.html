getThreadLocalPQExpBuffer

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/parallel.c:290

目的

スレッドローカルストレージ（TLS）または静的変数を使用して、再利用可能な PQExpBuffer を提供する。これにより、並列処理環境下で各スレッドが独立したバッファを保持しつつ、メモリの再利用を可能にする。

シグネチャまたは形状

PQExpBuffer getThreadLocalPQExpBuffer(void)

入力と出力

  • 入力: なし
  • 出力: PQExpBuffer 型のポインタ（初期化済み、または内容がリセットされたバッファ）

前提条件と不変条件

  • parallel_init_done フラグの状態に応じて、TLS（TlsGetValue/TlsSetValue）または静的変数 s_id_return のいずれかを使用してバッファを管理する。
  • TlsGetValue は、値が設定されていない場合に 0 を返すことを前提としている。

アルゴリズムまたは振る舞い

1. parallel_init_done が真の場合、TlsGetValue(tls_index) を使用してスレッド固有のバッファを取得する。偽の場合は静的変数 s_id_return を使用する。 2. 取得したバッファが存在する場合、resetPQExpBuffer を呼び出して内容をクリアする。 3. バッファが存在しない場合、createPQExpBuffer で新規作成する。 4. 新規作成したバッファを、parallel_init_done の状態に応じて TLS または静的変数に保存する。 5. 最終的にバッファのポインタを返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_dump/parallel.c:923 (getLocalPQExpBuffer への代入)
  • 呼び出し先:
  • resetPQExpBuffer
  • createPQExpBuffer
  • TlsGetValue / TlsSetValue (プラットフォーム依存)

関連データ構造

  • PQExpBuffer: 文字列操作のためのバッファ構造体。

失敗モードと境界条件

  • createPQExpBuffer がメモリ不足等で失敗した場合の挙動については、本ソース内では明示されていない。
  • parallel_init_done が切り替わるタイミングでの整合性は、呼び出し側の制御に依存する。

保守担当者向けメモ

  • 本関数は pg_dump の並列処理において、スレッドセーフなバッファ管理を行うための重要なユーティリティである。
  • pg_dump.c:2334 のコメントにある通り、特定のコンテキスト（fmtId 呼び出し時など）では使用できない制約があるため注意が必要である。
