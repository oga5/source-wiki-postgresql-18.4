interval_lerp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/orderedsetaggs.c:511

目的

2つの interval 型の値の間を、指定された割合（pct）で線形補間（Linear Interpolation）します。

シグネチャまたは形状

Datum interval_lerp(Datum lo, Datum hi, double pct)

入力と出力

  • **入力**:
  • lo: 補間の開始点となる interval 型の Datum。
  • hi: 補間の終了点となる interval 型の Datum。
  • pct: 補間比率を示す double 型の値（通常 0.0 から 1.0 の範囲）。
  • **出力**:
  • lo と hi の間を pct で線形補間した結果を Datum として返します。

前提条件と不変条件

  • lo および hi は有効な interval 型の Datum である必要があります。
  • 内部で interval_mi（減算）、interval_mul（乗算）、interval_pl（加算）を呼び出すため、これらの関数が正常に動作する環境が必要です。

アルゴリズムまたは振る舞い

以下の数式に基づき計算を行います： result = lo + (hi - lo) * pct

1. interval_mi を呼び出し、hi - lo を計算します。 2. interval_mul を呼び出し、差分に pct を乗算します。 3. interval_pl を呼び出し、開始点 lo に乗算結果を加算して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/orderedsetaggs.c 内の percentile_cont_final_common 関数から、連続パーセンタイル計算の補間ロジックとして呼び出されます。
  • **呼び出し先**:
  • interval_mi
  • interval_mul
  • interval_pl

関連データ構造

  • Datum: PostgreSQL の汎用データ表現型。
  • interval: PostgreSQL の時間間隔データ型。

失敗モードと境界条件

  • interval 型の演算においてオーバーフローが発生する場合、基底関数（interval_mi 等）の挙動に従いエラーが発生します。
  • pct が 0.0 の場合は lo を、1.0 の場合は hi を返します。

保守担当者向けメモ

  • 本関数は percentile_cont のような順序集合集約関数における連続値の補間に特化しています。
  • 類似の補間関数として float8_lerp が存在します。ロジックを変更する際は、他の型に対する補間関数との整合性に注意してください。
