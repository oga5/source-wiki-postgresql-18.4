RecoveryLockXidEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/ipc/standby.c:59

目的

ホットスタンバイ環境において、特定のトランザクションID（XID）に関連付けられたロックのリストを管理するためのハッシュテーブルエントリです。

シグネチャまたは形状

typedef struct RecoveryLockXidEntry
{
    TransactionId xid;              /* ハッシュキー（先頭である必要がある） */
    struct RecoveryLockEntry *head; /* ロックリストの先頭ポインタ */
} RecoveryLockXidEntry;

入力と出力

  • **入力**: ハッシュテーブル RecoveryLockXidHash のキーとして TransactionId を受け取ります。
  • **出力**: 該当するXIDが保持する RecoveryLockEntry のチェーン（head）を返します。

前提条件と不変条件

  • xid フィールドは構造体の先頭に配置され、ハッシュキーとして機能する必要があります。
  • head は、そのトランザクションが保持するロックのリストを指すか、ロックがない場合は NULL である必要があります。

アルゴリズムまたは振る舞い

  • RecoveryLockXidHash ハッシュテーブル内で、特定のXIDに関連するロック情報を検索・保持するために使用されます。
  • StandbyReleaseXidEntryLocks 関数において、このエントリに関連付けられたすべてのロックを解放する際の起点として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/standby.c 内のロック管理関数群（StandbyReleaseXidEntryLocks 等）から参照されます。
  • **呼び出し先**: RecoveryLockEntry 構造体（head ポインタ経由）。
  • 注: 呼び出し元・呼び出し先の詳細なコールグラフは、提供されたソース抜粋からは限定的です。

関連データ構造

  • RecoveryLockEntry: この構造体が保持するロックリストの各要素。
  • RecoveryLockXidHash: 本構造体を要素として格納するハッシュテーブル。

失敗モードと境界条件

  • ハッシュテーブルのサイズ設定（hash_ctl.entrysize）が本構造体のサイズと一致しない場合、メモリ破壊や検索エラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はスタンバイ側でのロック追跡の基盤です。xid がハッシュキーとして機能するため、構造体のメンバ順序を変更する際は注意してください。
