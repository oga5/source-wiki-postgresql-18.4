sts_initialize

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/sharedtuplestore.c:125

目的

共有メモリ上の SharedTuplestore 構造体を初期化し、呼び出し元のバックエンドがストアを操作するための SharedTuplestoreAccessor を生成・返却する。

シグネチャまたは形状

SharedTuplestoreAccessor *sts_initialize(
    SharedTuplestore *sts, 
    int participants,
    int my_participant_number,
    size_t meta_data_size,
    int flags,
    SharedFileSet *fileset,
    const char *name
);

入力と出力

  • **入力**:
  • sts: 初期化対象の共有メモリ上の制御構造体。
  • participants: 参加するバックエンドの総数。
  • my_participant_number: 自身の識別番号（0 から participants - 1）。
  • meta_data_size: 各タプルに付与するメタデータのサイズ。
  • flags: 動作フラグ。
  • fileset: 一時ファイルを管理する SharedFileSet。
  • name: ストアの識別名。
  • **出力**:
  • 成功時: 操作用構造体 SharedTuplestoreAccessor へのポインタ。
  • 失敗時: エラー発生時は elog(ERROR) により中断。

前提条件と不変条件

  • my_participant_number < participants であること。
  • meta_data_size + sizeof(uint32) が STS_CHUNK_DATA_SIZE を超えないこと。
  • name の長さが sts->name のバッファサイズを超えないこと。

アルゴリズムまたは振る舞い

1. sts 構造体に引数のメタデータ、フラグ、名前をコピーする。 2. 全参加者分の SharedTuplestoreParticipant 構造体に対して、LWLock を初期化し、読み取り位置や書き込み状態をリセットする。 3. palloc0 を使用して SharedTuplestoreAccessor をメモリ確保し、自身の参加者番号、共有ストアへの参照、ファイルセット、現在のメモリコンテキストを格納して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeHash.c: ParallelHashJoinBatchInner および ParallelHashJoinBatchOuter から呼び出される。
  • **呼び出し先**:
  • LWLockInitialize: 各参加者のロック初期化に使用。
  • palloc0: アクセサ構造体のメモリ確保に使用。
  • elog: 引数検証失敗時のエラー報告に使用。

関連データ構造

  • SharedTuplestore: 共有メモリ上のメイン制御構造体。
  • SharedTuplestoreAccessor: 各バックエンドが保持するローカル操作用構造体。
  • SharedTuplestoreParticipant: 各参加者の状態（ロック、読み取り位置等）を保持。
  • SharedFileSet: 一時ファイル管理用構造体。

失敗モードと境界条件

  • name が長すぎる場合: ERROR を発生させる。
  • meta_data_size が大きすぎる場合: ERROR を発生させる。
  • my_participant_number が不正な場合: Assert によりデバッグ時に停止する。

保守担当者向けメモ

  • 本関数は共有メモリ上のストアを初期化するのみであり、実際のファイル作成や書き込みは後続の sts_puttuple 等で行われる。
  • メタデータサイズが固定である前提で設計されているため、将来的に可変長にする場合は sts_puttuple および sts_read_tuple の修正が必要となる。
