pg_log_level

定義

  • 種別: enum
  • 定義箇所: src/include/common/logging.h:16

目的

PostgreSQLのフロントエンドプログラムにおけるログ出力の重要度を定義し、メッセージのフィルタリングや出力制御を行うための列挙型です。

シグネチャまたは形状

enum pg_log_level {
    PG_LOG_NOTSET = 0,
    PG_LOG_DEBUG,
    PG_LOG_INFO,
    PG_LOG_WARNING,
    PG_LOG_ERROR,
    PG_LOG_OFF
};

入力と出力

  • 本シンボル自体は列挙値の集合です。
  • ログ出力関数（pg_log_generic等）の引数として渡され、現在のログしきい値（__pg_log_level）と比較されます。

前提条件と不変条件

  • フロントエンドプログラム専用です。バックエンド（サーバー）コードでは使用できません。
  • PG_LOG_NOTSET および PG_LOG_OFF は、実際のメッセージレベルとして使用してはなりません。

アルゴリズムまたは振る舞い

  • PG_LOG_DEBUG: デフォルトでは出力されません。
  • PG_LOG_INFO: デフォルトで出力されます（プログラムの標準出力とは別に stderr へ出力）。
  • PG_LOG_WARNING: 警告およびエラーに近いメッセージに使用されます。
  • PG_LOG_ERROR: エラーメッセージに使用されます。
  • ログ出力時は、現在のしきい値と比較され、対象レベルがしきい値以上であれば stderr へ書き出されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照先）**:
  • __pg_log_level（グローバル変数）
  • pg_logging_set_level()
  • pg_log_generic()
  • pg_log_generic_v()
  • **呼び出し先**: 本シンボルは列挙型定義であり、関数呼び出しは行いません。

関連データ構造

  • __pg_log_level: 現在のログ出力しきい値を保持するグローバル変数。

失敗モードと境界条件

  • PG_LOG_NOTSET をログレベルとして指定した場合の動作は未定義です。
  • PG_LOG_OFF を指定すると、すべてのログ出力が抑制されます。

保守担当者向けメモ

  • 本列挙型は src/common/logging.c の実装と密接に関連しています。
  • ログレベルの追加や変更を行う際は、pg_log_generic 側のフィルタリングロジックへの影響を確認してください。
