sanitize_char

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-scram.c:806

目的

ログ出力やエラーメッセージにおいて、制御文字や非表示文字を含む可能性のある単一の文字を、安全に表示可能な文字列形式に変換する。

シグネチャまたは形状

static char *sanitize_char(char c)

入力と出力

  • **入力**: char c （変換対象の文字）
  • **出力**: char * （変換後の文字列を保持する静的バッファへのポインタ）

前提条件と不変条件

  • 戻り値は静的バッファ (static char buf[5]) を指すため、呼び出しごとに内容が上書きされる。
  • 戻り値の文字列は、最大で 'x' (3文字) または 0xHH (4文字) の形式となる。

アルゴリズムまたは振る舞い

1. 入力文字 c が表示可能な範囲（ASCII 0x21 から 0x7E まで）にあるか判定する。 2. 範囲内の場合、'c' の形式でバッファに書き込む。 3. 範囲外の場合、0xHH （16進数表記）の形式でバッファに書き込む。 4. バッファのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth-scram.c 内の複数箇所（属性解析や検証時のログ出力）。
  • src/backend/libpq/auth-oauth.c 内の複数箇所。
  • **呼び出し先**: 標準ライブラリ関数 snprintf を使用。

関連データ構造

  • 特になし（静的バッファ buf を内部的に使用）。

失敗モードと境界条件

  • 入力文字が 0x21 未満または 0x7E を超える場合、常に16進数表記にフォールバックされる。
  • 静的バッファを使用しているため、マルチスレッド環境や再帰的な呼び出しにおいて、戻り値のポインタが指す内容が競合する可能性がある（PostgreSQLのバックエンドプロセスモデルにおいては通常単一スレッドで動作するため、この制約下で設計されている）。

保守担当者向けメモ

  • 本関数は auth-scram.c と auth-oauth.c の両方で定義されている可能性がある（ツールによる参照情報に基づく）。重複定義や共通化の必要性について確認すること。
  • 戻り値が静的バッファであるため、呼び出し直後に結果をコピーまたは使用する必要がある。
