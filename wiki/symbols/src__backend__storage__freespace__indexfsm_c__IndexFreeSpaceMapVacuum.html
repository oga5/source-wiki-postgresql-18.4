IndexFreeSpaceMapVacuum

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/indexfsm.c:70

目的

インデックスリレーションに関連付けられたFSM（Free Space Map）をスキャンし、物理ページの状態との不整合を修正する。

シグネチャまたは形状

void IndexFreeSpaceMapVacuum(Relation rel)

入力と出力

  • **入力**: rel (対象となるインデックスリレーションの Relation 構造体ポインタ)
  • **出力**: なし（void）

前提条件と不変条件

  • 引数 rel は有効なインデックスリレーションを指している必要がある。
  • インデックスのFSM構造が初期化されていることが前提となる。

アルゴリズムまたは振る舞い

1. 汎用的なFSM保守関数である FreeSpaceMapVacuum(rel) を呼び出す。 2. FreeSpaceMapVacuum がインデックスリレーション全体のFSMツリーを走査し、実際のページ状態とFSM上の記録との不整合を解消する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blvacuum.c
  • src/backend/access/gin/ginfast.c
  • src/backend/access/gin/ginvacuum.c
  • src/backend/access/gist/gistvacuum.c
  • src/backend/access/nbtree/nbtree.c
  • src/backend/access/spgist/spgvacuum.c
  • **呼び出し先**:
  • FreeSpaceMapVacuum (汎用FSMモジュール)

関連データ構造

  • Relation: インデックスリレーションのメタデータを保持する構造体。

失敗モードと境界条件

  • 物理的なストレージアクセスエラーが発生した場合、基盤となる FreeSpaceMapVacuum のエラー処理に従い、例外が発生する可能性がある。

保守担当者向けメモ

  • 本関数は freespace.c で定義されている汎用的な FreeSpaceMapVacuum への単なるラッパーである。
  • インデックス特有のFSMロジック（ページが「空き」か「使用中」かの二値管理）の整合性維持は、すべて FreeSpaceMapVacuum の実装に依存している。
