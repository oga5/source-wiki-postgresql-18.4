index_expression_changed_walker

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execIndexing.c:1125

目的

インデックス式が更新対象の列を参照しているかどうかを判定し、インデックス更新の最適化（ヒントの付与）が可能かを判断する。

シグネチャまたは形状

static bool index_expression_changed_walker(Node *node, Bitmapset *allUpdatedCols)

入力と出力

  • **入力**:
  • node: 走査対象の式ツリー（Node）。
  • allUpdatedCols: 更新された列番号のビットマップセット（Bitmapset）。
  • **出力**:
  • bool: 式が更新された列を参照している場合は true、それ以外は false。

前提条件と不変条件

  • node が NULL の場合、常に false を返す。
  • Var ノードに到達した際、varattno が allUpdatedCols に含まれていれば、その式は変更されたとみなされる。

アルゴリズムまたは振る舞い

1. ノードが NULL なら false を返す。 2. ノードが Var 型の場合:

  • varattno を allUpdatedCols と照合する。
  • 含まれていれば true を返し、含まれていなければ false を返す。

    • 3. ノードが Var 以外の場合:

  • expression_tree_walker を再帰的に呼び出し、子ノードを探索する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execIndexing.c:1098 (内部的な判定処理)
  • **呼び出し先**:
  • expression_tree_walker (再帰探索のため)

関連データ構造

  • Node: PostgreSQLの式ツリー構造。
  • Bitmapset: 更新された列を保持するビットマップ。
  • Var: 式内の変数参照。

失敗モードと境界条件

  • node が NULL の場合は安全に処理される。
  • Var 以外のノードで再帰が深すぎる場合、スタックオーバーフローの可能性がある（ただし、通常の式ツリーの深さであれば問題ない）。

保守担当者向けメモ

  • 本関数は index_unchanged_by_update から利用され、インデックス更新をスキップする最適化の可否を決定する。
  • allUpdatedCols のビット演算において FirstLowInvalidHeapAttributeNumber を用いたオフセット調整が行われている点に注意すること。
