initHyperLogLogError

定義

  • 種別: function
  • 定義箇所: src/backend/lib/hyperloglog.c:127

目的

指定された許容誤差（error）を満たす最小のビット幅を算出し、hyperLogLogState を初期化します。

シグネチャまたは形状

void initHyperLogLogError(hyperLogLogState *cState, double error)

入力と出力

  • cState: 初期化対象の hyperLogLogState 構造体へのポインタ。
  • error: 許容する相対誤差（double型）。
  • 出力: なし（cState が内部で初期化される）。

前提条件と不変条件

  • bwidth（ビット幅）は 4 から 16 の範囲で決定されます。
  • initHyperLogLog を呼び出すため、同関数が要求するメモリ確保の前提条件に従います。

アルゴリズムまたは振る舞い

1. ビット幅 bwidth を初期値 4 から開始します。 2. 各 bwidth に対して、レジスタ数 m = 2^bwidth を計算します。 3. 1.04 / sqrt(m) < error という条件式を満たすまで bwidth をインクリメントします。 4. 決定した bwidth を用いて initHyperLogLog(cState, bwidth) を呼び出し、状態を初期化します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは特定できません。
  • 呼び出し先: initHyperLogLog（同ファイル内）。

関連データ構造

  • hyperLogLogState: HLL の状態を保持する構造体。

失敗モードと境界条件

  • bwidth が 16 に達した時点でループは終了します。
  • error の値が極端に小さい場合、bwidth は最大値の 16 で固定されます。

保守担当者向けメモ

  • 本関数は 1.04 / sqrt(m) という近似式に基づきビット幅を決定しています。
  • initHyperLogLog との依存関係があるため、初期化ロジックを変更する際は両方の整合性を確認してください。
