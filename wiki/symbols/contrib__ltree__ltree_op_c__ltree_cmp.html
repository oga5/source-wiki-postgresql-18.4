ltree_cmp

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_op.c:87

目的

PostgreSQL の ltree 型に対する比較演算（大小比較）を SQL インターフェース経由で実行する。

シグネチャまたは形状

Datum ltree_cmp(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの ltree 型引数を受け取る。
  • **出力**: int32 型。第1引数が第2引数より小さい場合は負、等しい場合は0、大きい場合は正の値を返す。

前提条件と不変条件

  • RUNCMP マクロが定義されている環境で実行されること。
  • 入力引数は有効な ltree 構造体であること。

アルゴリズムまたは振る舞い

1. RUNCMP マクロを呼び出し、引数の取得および内部的な比較関数（ltree_compare）の実行を行う。 2. 比較結果を res に格納する。 3. PG_RETURN_INT32(res) を通じて、比較結果を PostgreSQL の Datum 形式で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL レベルの比較演算子（<, <=, =, >=, > 等）から呼び出される。
  • **呼び出し先**: RUNCMP マクロ内で ltree_compare が呼び出される。

関連データ構造

  • ltree: 比較対象となる階層ラベルパス構造体。
  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約に基づく引数コンテナ。

失敗モードと境界条件

  • 入力が不正なメモリ領域を指している場合、セグメンテーション違反が発生する可能性がある。
  • RUNCMP マクロの内部実装に依存するため、比較対象のメモリレイアウトが ltree の仕様に適合しない場合は未定義の動作となる。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_INFO_V1(ltree_cmp) により PostgreSQL の関数として登録されている。
  • 比較ロジックの詳細は ltree_compare に依存しているため、比較アルゴリズムの変更が必要な場合はそちらを参照すること。
