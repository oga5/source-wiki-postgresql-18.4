目的

バスク語のステミング処理において、名詞に関連する接尾辞の識別および削除・置換を行う。

シグネチャまたは形状

static int r_izenak(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z （ステミング環境の状態を保持する構造体）
  • **出力**: int （処理が成功した場合は 1、失敗した場合は 0 または負の値を返す）

前提条件と不変条件

  • z->c は現在の単語内のカーソル位置を示す。
  • 処理開始前に z->ket に現在のカーソル位置が保存される。
  • 文字列の末尾から前方に向かって検索を行うため、z->lb（下限）および z->c の整合性が保たれている必要がある。

アルゴリズムまたは振る舞い

1. **事前チェック**: 現在の文字が特定のビットマスク条件を満たさない場合、即座に 0 を返す。 2. **接尾辞検索**: find_among_b を使用し、探索テーブル a_1（サイズ 295）を用いて単語末尾から名詞接尾辞を検索する。 3. **条件分岐**: 検索結果（among_var）に基づき、以下のいずれかの処理を実行する。

  • case 1, 2, 4: r_RV, r_R2, r_R1 のいずれかの領域条件を満たすか確認し、条件を満たせば slice_del で接尾辞を削除する。
  • case 3, 5-10: slice_from_s を使用し、該当する接尾辞を別の文字列に置換する。

    • 4. **終了**: 処理が成功すれば 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/libstemmer/stem_ISO_8859_1_basque.c および src/backend/snowball/libstemmer/stem_UTF_8_basque.c 内のメインステミング関数から呼び出される。
  • **呼び出し先**: find_among_b, r_RV, r_R2, r_R1, slice_del, slice_from_s （いずれも Snowball ランタイム関数）。

関連データ構造

  • struct SN_env: ステミング環境を管理する構造体。
  • a_1: 名詞接尾辞の探索に使用される among 構造体の配列。

失敗モードと境界条件

  • find_among_b が一致する接尾辞を見つけられない場合、0 を返す。
  • r_RV, r_R1, r_R2 などの領域チェック関数が失敗した場合、その戻り値（負の値を含む）をそのまま返す。
  • slice_del や slice_from_s がメモリ操作等でエラーを返した場合、その値を伝播させる。
