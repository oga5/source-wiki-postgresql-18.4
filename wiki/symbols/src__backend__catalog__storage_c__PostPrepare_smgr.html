PostPrepare_smgr

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/storage.c:933

目的

トランザクションの PREPARE 実行成功後に、メモリ上に保持されている削除待ちリレーションのリスト（pendingDeletes）を解放し、クリーンアップを行う。

シグネチャまたは形状

void PostPrepare_smgr(void)

入力と出力

  • 入力: なし（グローバル変数 pendingDeletes を参照）
  • 出力: なし（副作用としてメモリ解放を実行）

前提条件と不変条件

  • pendingDeletes リスト内の各エントリは、pfree で解放可能なメモリ領域を指していること。
  • 本関数はトランザクションの PREPARE 処理の文脈で呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. グローバル変数 pendingDeletes が指すリストを先頭から順に走査する。 2. 各エントリに対して pfree を実行し、メモリを解放する。 3. リストの走査が完了するまでこれを繰り返し、最終的に pendingDeletes を空（NULL）にする。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c:2739
  • 呼び出し先: pfree（メモリ解放関数）

関連データ構造

  • PendingRelDelete: 削除待ちリレーションを保持するリスト構造体。

失敗モードと境界条件

  • pendingDeletes が NULL の場合、ループは実行されず即座に終了する（安全に動作する）。

保守担当者向けメモ

  • 本関数は PREPARE 後のクリーンアップ専用である。通常のコミットやアボート時の処理（smgrDoPendingDeletes 等）とは異なるため、混同しないこと。
  • リストの走査中に next ポインタを一時保持することで、安全にメモリ解放を行っている。
