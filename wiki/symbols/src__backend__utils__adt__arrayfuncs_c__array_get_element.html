array_get_element

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arrayfuncs.c:1819

目的

PostgreSQLの配列データ型から、指定された添字（サブスクリプト）に対応する要素を取得します。

シグネチャまたは形状

Datum array_get_element(Datum arraydatum, int nSubscripts, int *indx, int arraytyplen, int elmlen, bool elmbyval, char elmalign, bool *isNull)

入力と出力

  • **入力**:
  • arraydatum: 操作対象の配列（Datum）。
  • nSubscripts: 添字の数。
  • indx: 添字の配列。
  • arraytyplen, elmlen, elmbyval, elmalign: 要素の型に関するメタ情報。
  • **出力**:
  • 戻り値: 指定された位置の要素（Datum）。
  • isNull: 要素がNULLであるか、または添字が無効な場合に true が設定される。

前提条件と不変条件

  • 配列が外部展開（Expanded Array）形式の場合、array_get_element_expanded に処理を委譲します。
  • 固定長配列の場合、1次元かつ0ベースのインデックスとみなされます。
  • 添字が配列の次元数や境界外である場合、エラーではなく isNull を true に設定して終了します。

アルゴリズムまたは振る舞い

1. **型判定**: 配列の型情報に基づき、固定長配列、展開済み配列、または通常の可変長配列（ArrayType）として処理を分岐します。 2. **バリデーション**: 配列の次元数と添字の数が一致するか、および各添字が境界内にあるかを確認します。 3. **オフセット計算**: ArrayGetOffset を使用して、多次元配列における線形オフセットを算出します。 4. **NULLチェック**: array_get_isnull を使用し、対象位置がNULLビットマップ上でNULLとしてマークされているかを確認します。 5. **データ取得**: array_seek を用いてデータポインタを特定し、ArrayCast を介して適切な型で値を返却します。

関連データ構造

  • ArrayType: 配列の内部表現構造体。
  • Datum: PostgreSQLの汎用データ表現。

失敗モードと境界条件

  • **無効な添字**: 配列の次元数と nSubscripts が不一致、または添字が範囲外の場合、isNull を true にして (Datum) 0 を返します。
  • **NULL要素**: 対象要素がNULLの場合、isNull を true にして (Datum) 0 を返します。

保守担当者向けメモ

  • 本関数は array_seek を使用してデータポインタを計算するため、配列のメモリレイアウト（ヘッダー、NULLビットマップ、データ本体）の変更には注意が必要です。
  • 展開済み配列（Expanded Array）の処理は array_get_element_expanded に分離されているため、ロジックの修正時は両方の関数を確認してください。
