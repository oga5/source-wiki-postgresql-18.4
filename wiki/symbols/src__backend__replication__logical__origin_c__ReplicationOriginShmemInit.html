ReplicationOriginShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/origin.c:548

目的

論理レプリケーションの起点（Replication Origin）管理に必要な共有メモリ領域を確保および初期化する。

シグネチャまたは形状

void ReplicationOriginShmemInit(void)

入力と出力

  • 入力: なし（GUC変数 max_active_replication_origins を参照）
  • 出力: なし（共有メモリ上のグローバル変数 replication_states_ctl および replication_states を初期化）

前提条件と不変条件

  • max_active_replication_origins が 0 の場合、本関数は何も行わずに終了する。
  • 共有メモリの確保は ShmemInitStruct を介して行われる。
  • 初回起動時（found == false）のみ、メモリのゼロクリアおよび各状態構造体のロックと条件変数の初期化が行われる。

アルゴリズムまたは振る舞い

1. max_active_replication_origins が 0 ならば即座にリターンする。 2. ShmemInitStruct を呼び出し、ReplicationOriginState という名前で共有メモリを確保する。 3. 確保された領域の先頭アドレスを replication_states_ctl に格納し、その中の states 配列を replication_states に割り当てる。 4. 初回確保時（!found）の場合:

  • ReplicationOriginShmemSize() で計算されたサイズ分、メモリをゼロクリアする。
  • tranche_id に LWTRANCHE_REPLICATION_ORIGIN_STATE を設定する。
  • max_active_replication_origins の数だけループし、各 ReplicationState の lock（LWLock）と origin_cv（ConditionVariable）を初期化する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:330（PostgreSQLの共有メモリ初期化プロセスの一部として呼び出される）
  • 呼び出し先: ShmemInitStruct, ReplicationOriginShmemSize, MemSet, LWLockInitialize, ConditionVariableInit

関連データ構造

  • ReplicationStateCtl: 共有メモリ上の制御構造体。
  • ReplicationState: 各起点の進捗を保持する構造体。
  • LWLock: 各状態を保護するための軽量ロック。
  • ConditionVariable: 起点削除時の待機に使用される同期プリミティブ。

失敗モードと境界条件

  • max_active_replication_origins が 0 に設定されている場合、共有メモリは確保されない。
  • 共有メモリの確保に失敗した場合、ShmemInitStruct がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数はPostgreSQLの起動シーケンス（CreateSharedMemoryAndSemaphores）から呼び出される必要がある。
  • max_active_replication_origins の値は、共有メモリサイズを決定する重要なパラメータであるため、変更時は ReplicationOriginShmemSize との整合性に注意すること。
