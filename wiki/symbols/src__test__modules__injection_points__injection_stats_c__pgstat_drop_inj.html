pgstat_drop_inj

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_stats.c:145

目的

指定された名前のインジェクションポイントに関連付けられた統計エントリを、PostgreSQLの統計サブシステムから削除します。

シグネチャまたは形状

void pgstat_drop_inj(const char *name)

入力と出力

  • **入力**: name (const char *) - 削除対象のインジェクションポイント名。
  • **出力**: なし（void）。

前提条件と不変条件

  • inj_stats_loaded および inj_stats_enabled が真である場合にのみ処理が実行されます。
  • 統計サブシステムの状態に依存しており、無効な状態では即座に復帰します。

アルゴリズムまたは振る舞い

1. 統計機能が無効な場合、何もせず終了します。 2. PGSTAT_INJ_IDX(name) マクロを使用して名前からハッシュインデックスを生成し、pgstat_drop_entry を呼び出して統計エントリの削除を試みます。 3. pgstat_drop_entry が失敗した場合（戻り値が偽）、pgstat_request_entry_refs_gc() を呼び出し、参照カウントのガベージコレクションを要求します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/injection_points/injection_points.c (2箇所)
  • **呼び出し先**:
  • pgstat_drop_entry (PostgreSQL統計サブシステム)
  • pgstat_request_entry_refs_gc (PostgreSQL統計サブシステム)

関連データ構造

  • PGSTAT_KIND_INJECTION: 統計種別識別子。
  • PGSTAT_INJ_IDX(): インジェクションポイント名から統計インデックスを生成するマクロ。

失敗モードと境界条件

  • 統計機能が無効な場合、削除処理はスキップされます。
  • 削除対象のエントリが存在しない、または参照が保持されている場合、pgstat_drop_entry は失敗し、ガベージコレクションの要求がトリガーされます。

保守担当者向けメモ

  • 本関数は injection_points モジュールの統計管理の一部です。統計種別 PGSTAT_KIND_INJECTION のライフサイクル管理において重要な役割を果たします。
  • pgstat_drop_entry の戻り値に基づく条件分岐は、統計サブシステムのメモリ管理ルールに従っています。
