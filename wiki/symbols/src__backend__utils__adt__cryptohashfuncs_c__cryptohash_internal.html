cryptohash_internal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/cryptohashfuncs.c:79

目的

SHA-2ファミリーのハッシュ計算を共通化し、PostgreSQLの関数呼び出し規約に適応させる内部ヘルパー関数。

シグネチャまたは形状

static inline bytea *cryptohash_internal(pg_cryptohash_type type, bytea *input)

入力と出力

  • **入力**:
  • type: 計算するハッシュアルゴリズムの種類（pg_cryptohash_type）。
  • input: ハッシュ対象のデータを含む bytea 型ポインタ。
  • **出力**:
  • 計算されたハッシュ値を含む bytea 型ポインタ。

前提条件と不変条件

  • type は PG_SHA224, PG_SHA256, PG_SHA384, PG_SHA512 のいずれかである必要がある。
  • palloc0 を使用してメモリを確保するため、PostgreSQLのメモリコンテキスト内で実行される必要がある。

アルゴリズムまたは振る舞い

1. 指定された type に基づき、ハッシュの長さと識別文字列を決定する。PG_MD5 または PG_SHA1 が指定された場合はエラーを投げる。 2. palloc0 で結果格納用の bytea 領域を確保する。 3. pg_cryptohash_create でコンテキストを生成し、pg_cryptohash_init、pg_cryptohash_update、pg_cryptohash_final を順次呼び出してハッシュを計算する。 4. 各ステップでエラーが発生した場合は elog(ERROR, ...) を呼び出し、処理を中断する。 5. 最後に pg_cryptohash_free でコンテキストを解放し、SET_VARSIZE で結果のサイズを設定して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • sha224_bytea
  • sha256_bytea
  • sha384_bytea
  • sha512_bytea
  • **呼び出し先**:
  • pg_cryptohash_create
  • pg_cryptohash_init
  • pg_cryptohash_update
  • pg_cryptohash_final
  • pg_cryptohash_free
  • pg_cryptohash_error

関連データ構造

  • pg_cryptohash_type: ハッシュアルゴリズムの列挙型。
  • pg_cryptohash_ctx: ハッシュ計算の内部状態を保持する構造体。
  • bytea: PostgreSQLの可変長バイナリデータ型。

失敗モードと境界条件

  • サポートされていないハッシュタイプが渡された場合、elog(ERROR) により処理が停止する。
  • ハッシュ計算の各フェーズ（init/update/final）でエラーが発生した場合、pg_cryptohash_error を用いてエラーメッセージと共に elog(ERROR) が発生する。

保守担当者向けメモ

  • 本関数は palloc0 を使用しているため、メモリリークの心配はないが、PostgreSQLのメモリコンテキスト管理に依存している点に留意すること。
  • MD5やSHA-1のサポートは意図的に除外されているため、これらを追加する場合はロジックの拡張が必要。
