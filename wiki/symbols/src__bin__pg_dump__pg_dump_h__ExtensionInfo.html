ExtensionInfo

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_dump.h:193

目的

PostgreSQLの拡張機能（Extension）に関するメタデータを保持し、pg_dump におけるダンプ処理の対象として管理するための構造体です。

シグネチャまたは形状

typedef struct _extensionInfo
{
    DumpableObject dobj;
    char      *namespace;       /* 拡張機能のオブジェクトを含むスキーマ */
    bool       relocatable;
    char      *extversion;
    char      *extconfig;       /* 設定テーブルに関する情報 */
    char      *extcondition;
} ExtensionInfo;

入力と出力

  • 本構造体は、システムカタログから読み取られた拡張機能情報を格納するコンテナとして機能します。
  • pg_dump の各処理フェーズにおいて、拡張機能の定義や構成情報を出力するためのソースデータとして利用されます。

前提条件と不変条件

  • dobj フィールドを通じて DumpableObject としての共通属性（ID、依存関係など）を保持します。
  • 拡張機能が所有するオブジェクトを特定するために、カタログIDとの関連付けが必要です。

アルゴリズムまたは振る舞い

  • pg_dump の収集フェーズでインスタンス化され、ダンプ順序の決定や依存関係の構築に使用されます。
  • dumpExtension 関数などを通じて、アーカイブへの書き出し処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c 内の dumpExtension や selectDumpableExtension などで参照されます。また、src/bin/pg_dump/common.c 内の recordExtensionMembership や findOwningExtension でオブジェクトと拡張機能の紐付けに使用されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • DumpableObject: 基底構造体。
  • CatalogId: 拡張機能が所有するオブジェクトを識別するために使用されます。

失敗モードと境界条件

  • 拡張機能の構成情報（extconfig 等）が取得できない場合、ダンプ出力に不整合が生じる可能性があります。
  • 依存関係の解決において、拡張機能が他のオブジェクトと循環参照を持つ場合、pg_dump_sort.c によるソート処理で境界条件が発生します。

保守担当者向けメモ

  • ExtensionInfo を拡張する場合は、pg_dump.c 内の関連するダンプロジックおよび pg_dump_sort.c の依存関係管理ロジックへの影響を確認してください。
  • メモリ管理については、pg_dump の実行期間中生存するため、動的に割り当てられた文字列フィールド（namespace, extversion 等）の解放タイミングに注意してください。
