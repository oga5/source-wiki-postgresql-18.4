ExplainSaveGroup

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_format.c:427

目的

EXPLAIN 出力処理において、現在のグループ化状態（ネストレベルやスタック）を一時的に退避させ、後で復元可能にするための状態保存を行う。

シグネチャまたは形状

void ExplainSaveGroup(ExplainState *es, int depth, int *state_save)

入力と出力

  • es: 出力状態を保持する ExplainState 構造体へのポインタ。
  • depth: インデントを調整するための深さの値。
  • state_save: 退避したグループ化状態を格納するための出力用ポインタ。

前提条件と不変条件

  • es->format に基づいて処理が分岐する。
  • EXPLAIN_FORMAT_JSON および EXPLAIN_FORMAT_YAML の場合、es->grouping_stack が有効なリストであることが前提となる。

アルゴリズムまたは振る舞い

1. es->format を判定する。 2. EXPLAIN_FORMAT_TEXT: 何も行わない。 3. EXPLAIN_FORMAT_XML: es->indent から depth を減算する。 4. EXPLAIN_FORMAT_JSON / EXPLAIN_FORMAT_YAML:

  • es->indent から depth を減算する。
  • es->grouping_stack の先頭要素を state_save に格納する。
  • es->grouping_stack から先頭要素を削除する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/explain.c:4930 にて確認。
  • 呼び出し先: list_delete_first および linitial_int（PostgreSQL のリスト操作関数）を呼び出す。

関連データ構造

  • ExplainState: 出力状態を管理する構造体。
  • grouping_stack: ExplainState 内のリスト構造で、ネスト状態を追跡する。

失敗モードと境界条件

  • grouping_stack が空の状態で JSON または YAML フォーマットの処理が行われた場合、リスト操作関数によるエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数で退避した状態は、対となる関数（ExplainRestoreGroup 等）で復元されることを想定している。
  • EXPLAIN_FORMAT_TEXT はインデント管理を行わないため、本関数では何もしない仕様となっている。
