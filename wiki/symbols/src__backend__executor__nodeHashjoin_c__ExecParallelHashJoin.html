ExecParallelHashJoin

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHashjoin.c:699

目的

並列クエリ実行環境におけるハッシュ結合の実行エントリーポイント。

シグネチャまたは形状

TupleTableSlot *ExecParallelHashJoin(PlanState *pstate)

入力と出力

  • **入力**: pstate (結合ノードの実行状態を保持する PlanState 構造体)
  • **出力**: 結合条件を満たす TupleTableSlot、または結合終了時に NULL を返す。

前提条件と不変条件

  • 本関数は並列実行コンテキストで呼び出されることを前提とする。
  • 内部で呼び出される ExecHashJoinImpl は、並列実行フラグ（true）を渡すことで並列処理専用の分岐を有効化する。

アルゴリズムまたは振る舞い

1. ExecHashJoinImpl を呼び出し、並列実行モードでハッシュ結合のロジックを実行する。 2. コンパイラの最適化により、並列処理に不要なシリアル実行用の分岐はインライン展開時に除去されることが期待される。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecSetExecProcNode を介して実行ノードのプロシージャとして登録される（src/backend/executor/nodeHashjoin.c:1669, 1767）。
  • **呼び出し先**: ExecHashJoinImpl（直接の呼び出し先）。

関連データ構造

  • PlanState: 実行ノードの状態管理。
  • ParallelHashJoinState: 並列ハッシュ結合の同期状態を管理する構造体（ExecHashJoinImpl 経由で利用）。

失敗モードと境界条件

  • 呼び出し元および呼び出し先の詳細なエラーハンドリングについては、本ソース抜粋からは特定不可。

保守担当者向けメモ

  • 本関数は ExecHashJoinImpl への単なるラッパーである。ロジックの修正は ExecHashJoinImpl に対して行うこと。
  • 並列実行時は Barrier を用いた同期が行われるため、デッドロックを避けるための制約（バリア待機中にタプルを返さない等）を維持する必要がある。
