shm_toc_create

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shm_toc.c:39

目的

共有メモリセグメントの先頭に「目次（Table of Contents）」構造体（shm_toc）を初期化し、動的共有メモリ（DSM）領域を論理的な管理対象としてセットアップする。

シグネチャまたは形状

shm_toc *shm_toc_create(uint64 magic, void *address, Size nbytes)

入力と出力

  • **入力**:
  • magic: 領域の識別に使用する64ビットのマジックナンバー。
  • address: 共有メモリセグメントの開始アドレス。
  • nbytes: 確保された共有メモリの総サイズ。
  • **出力**:
  • 初期化された shm_toc 構造体へのポインタ。

前提条件と不変条件

  • nbytes は offsetof(shm_toc, toc_entry) より大きい必要がある（アサーションで検証）。
  • shm_toc_allocate() で使用されるアライメント計算のため、開始アドレスはバッファアライメントされていることが前提となる。

アルゴリズムまたは振る舞い

1. 指定された address を shm_toc 構造体としてキャストする。 2. toc_magic にマジックナンバーを格納する。 3. toc_mutex を SpinLockInit で初期化する。 4. toc_total_bytes に nbytes をバッファアライメント（BUFFERALIGN_DOWN）して格納する。 5. toc_allocated_bytes（割り当て済みバイト数）と toc_nentry（エントリ数）を 0 にリセットする。 6. 初期化済みの toc ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/session.c
  • src/backend/access/transam/parallel.c
  • src/backend/replication/logical/applyparallelworker.c
  • src/test/modules/test_shm_mq/setup.c
  • **呼び出し先**:
  • SpinLockInit（マクロ）

関連データ構造

  • shm_toc: 目次を保持する不透明な構造体。
  • shm_toc_entry: 目次に格納される各エントリの構造体。

失敗モードと境界条件

  • nbytes が shm_toc 構造体のヘッダーサイズを下回る場合、アサーション違反によりプロセスが停止する。

保守担当者向けメモ

  • 本関数はセグメントの初期化のみを行う。既存のセグメントに接続する場合は shm_toc_attach を使用すること。
  • toc_total_bytes の計算において BUFFERALIGN_DOWN を使用しているため、入力サイズがアライメント境界に満たない端数は切り捨てられる点に注意。
