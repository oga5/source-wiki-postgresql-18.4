GetNewTransactionId

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/varsup.c:76

目的

新規トランザクションまたはサブトランザクションに対して、一意なトランザクションID（XID）を払い出します。

シグネチャまたは形状

FullTransactionId GetNewTransactionId(bool isSubXact)

入力と出力

  • **入力**: isSubXact (bool) - サブトランザクションであるかを示すフラグ。
  • **出力**: FullTransactionId - 新規に割り当てられた64bitトランザクションID。

前提条件と不変条件

  • 並列モード（Parallel Mode）実行中は呼び出し不可。
  • リカバリ中（Recovery In Progress）は呼び出し不可。
  • ブートストラップ処理モード時は、特別なブートストラップ用XIDを返し、isSubXact は false である必要がある。
  • XidGenLock を排他制御（LW_EXCLUSIVE）で取得して処理を行う。

アルゴリズムまたは振る舞い

1. **並列/リカバリチェック**: 並列モードまたはリカバリ中であればエラーを発生させる。 2. **ブートストラップ処理**: ブートストラップモードであれば、専用のIDを割り当てて即時終了する。 3. **XID生成とWraparound保護**:

  • XidGenLock を取得し、現在の nextXid を参照する。
  • xidVacLimit を超えている場合、ロックを一時解放して自動バキュームの起動信号送信や警告・エラー（xidStopLimit 超過時）を行う。
  • 警告・エラー処理後、再度ロックを取得し直す。

    • 4. **ストレージ拡張**: ExtendCLOG、ExtendCommitTs、ExtendSUBTRANS を呼び出し、必要な領域を確保する。 5. **状態更新**: nextXid を進め、ProcArray に新しいXIDを登録する。 6. **終了処理**: ロックを解放し、割り当てた FullTransactionId を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c（トランザクション開始時）など。
  • **呼び出し先**: ExtendCLOG, ExtendCommitTs, ExtendSUBTRANS, FullTransactionIdAdvance, SendPostmasterSignal, get_database_name, ereport 等。

関連データ構造

  • TransamVariables: 共有メモリ上のID生成状態管理構造体。
  • MyProc: 現在のプロセスの管理構造体。
  • ProcGlobal: プロセス全体のXID状態を保持する共有メモリ構造体。

失敗モードと境界条件

  • **Wraparound保護**: xidStopLimit を超えた場合、データベースのデータ損失を防ぐため ERROR を発生させトランザクションを拒否する。
  • **CLOG拡張失敗**: ストレージ拡張が失敗した場合、XIDの更新は行われない。
  • **並列モード**: 並列クエリ実行中のXID割り当ては禁止されており、ERROR となる。

保守担当者向けメモ

  • XidGenLock は頻繁に取得されるため、関数内の処理は最小限に抑えられている。
  • 警告や信号送信時はデッドロック回避のためロックを一時解放する設計となっている。
  • 64bitの FullTransactionId を扱うが、内部で32bit XIDとの変換が行われる点に注意すること。
  • ProcArray への登録は、XidGenLock 解放前に完了させる必要がある。
