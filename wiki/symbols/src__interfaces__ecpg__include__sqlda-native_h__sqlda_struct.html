sqlda_struct

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/include/sqlda-native.h:33

目的

ECPG（Embedded SQL for C）において、動的SQL実行時の結果セットやパラメータのメタデータを保持するネイティブ形式のSQLDA（SQL Descriptor Area）を提供します。

シグネチャまたは形状

struct sqlda_struct
{
    char        sqldaid[8];
    long        sqldabc;
    short       sqln;
    short       sqld;
    struct sqlda_struct *desc_next;
    struct sqlvar_struct sqlvar[1];
};

入力と出力

  • **入力**: ECPGランタイムライブラリがデータベースから取得した結果セットやバインドパラメータ。
  • **出力**: アプリケーションがアクセス可能なカラム属性（型、長さ、データポインタ、NULLインジケータ）。

前提条件と不変条件

  • sqlvar[1] は構造体の末尾に続く可変長領域の先頭として機能します。
  • メモリ確保時には、sqld（カラム数）に基づいたサイズ計算（sizeof(struct sqlda_struct) + (sqld - 1) * sizeof(struct sqlvar_struct)）が必須です。

アルゴリズムまたは振る舞い

  • データベースのクエリ結果を構造体にマッピングし、desc_next を用いて複数の記述子を連結するリスト構造を形成します。
  • sqld は保持するカラム数を示し、実際のデータアクセスは sqlvar 配列を介して行われます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/interfaces/ecpg/ecpglib/descriptor.c
  • src/interfaces/ecpg/ecpglib/execute.c
  • src/interfaces/ecpg/ecpglib/sqlda.c
  • **呼び出し先**:
  • struct sqlvar_struct（sqlvar メンバとして使用）

関連データ構造

  • struct sqlvar_struct: 個々のカラム属性を定義。
  • sqlda.h / sqlda-compat.h: 他のSQLDA定義形式。

失敗モードと境界条件

  • sqld の値と実際のメモリ確保サイズが不一致の場合、バッファオーバーフローのリスクが生じます。
  • NAMEDATALEN（64バイト）を超える識別子を使用した場合、メタデータの切り詰めが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はECPGネイティブ形式であり、Informix互換の sqlda-compat.h とは別物です。
  • sqlvar[1] はC言語の慣用句による可変長配列の実装であるため、ポインタ演算やメモリ確保の際には注意してください。
