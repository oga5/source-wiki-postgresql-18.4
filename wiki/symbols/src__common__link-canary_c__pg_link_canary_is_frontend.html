pg_link_canary_is_frontend

定義

  • 種別: function
  • 定義箇所: src/common/link-canary.c:28

目的

現在のバイナリがフロントエンド（クライアントライブラリ等）としてビルドされているかを実行時に判定する診断用関数です。共有ライブラリが意図した環境で動作しているかを確認する「カナリア」として機能します。

シグネチャまたは形状

bool pg_link_canary_is_frontend(void);

入力と出力

  • 入力: なし
  • 出力: bool 型。コンパイル時に FRONTEND マクロが定義されている場合は true を、そうでない場合は false を返します。

前提条件と不変条件

  • コンパイル時の FRONTEND マクロの定義状態に依存します。
  • 実行時の動的な環境変化には対応せず、ビルド時に確定した値を返します。

アルゴリズムまたは振る舞い

プリプロセッサの条件分岐（#ifdef FRONTEND）を利用し、コンパイル時に決定された環境情報を即座に返します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • src/backend/bootstrap/bootstrap.c:370
  • src/interfaces/libpq/fe-connect.c:2718
  • 呼び出し先: なし（本関数は定数を返すのみです）

関連データ構造

  • なし

失敗モードと境界条件

  • ELF 形式のシステムにおいて、共有ライブラリがバックエンドにロードされた際、同名のシンボルがバックエンド側で定義されているとシンボル衝突が発生する可能性があります。本関数は、このようなリンクの不整合を検証するための診断ツールであり、ランタイムの環境判定を目的としたものではありません。

保守担当者向けメモ

  • 本関数は src/common に配置されており、フロントエンドとバックエンドの両方からリンクされます。
  • 開発時やデバッグ時に、意図しないシンボルが解決されていないかを検証する目的で使用してください。
