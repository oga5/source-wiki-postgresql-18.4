sift_up

定義

  • 種別: function
  • 定義箇所: src/common/binaryheap.c:269

目的

バイナリヒープ内の指定されたノードを、ヒープの順序条件を満たすまで親方向に浮上（sift up）させ、ヒープの整合性を維持する。

シグネチャまたは形状

static void sift_up(binaryheap *heap, int node_off)

入力と出力

  • **入力**:
  • heap: 操作対象の binaryheap 構造体へのポインタ。
  • node_off: 浮上を開始するノードの配列インデックス。
  • **出力**: なし（副作用として heap->bh_nodes 配列の内容が書き換わる）。

前提条件と不変条件

  • node_off は heap->bh_nodes の有効なインデックスであること。
  • ヒープ内の他のノードは、対象ノードを除いてヒープ条件を満たしていること。

アルゴリズムまたは振る舞い

1. 指定された node_off の値を「穴（hole）」として保持し、一時的に配列から取り出した状態にする。 2. node_off がルート（0）に達するまでループを繰り返す。 3. 現在のノードの親ノードを取得し、比較関数 heap->bh_compare を呼び出す。 4. 対象ノードの値が親ノードの値以下であれば、ヒープ条件が満たされたと判断しループを終了する。 5. そうでない場合、親ノードの値を現在の node_off 位置に移動させ、node_off を親のインデックスに更新して継続する。 6. ループ終了後、最終的な node_off の位置に保持していた値を格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • binaryheap_add (ヒープ末尾への追加後)
  • binaryheap_remove_node (ノード削除後の再配置)
  • **呼び出し先**:
  • parent_offset (親インデックスの計算)
  • heap->bh_compare (比較関数の実行)

関連データ構造

  • binaryheap: ヒープの管理構造体。bh_nodes 配列、比較関数 bh_compare、および比較関数に渡す引数 bh_arg を保持する。

失敗モードと境界条件

  • node_off が 0 の場合、ループは実行されず、値の再格納のみが行われる。
  • 比較関数が予期しない値を返す場合、ヒープの整合性が損なわれる可能性がある。

保守担当者向けメモ

  • 本関数は「穴」を移動させる手法を採用しており、要素の交換（swap）を繰り返すよりもデータコピーの回数が削減されている。
  • bh_node_type は環境により Datum または void * に解決されるため、比較関数の実装には注意が必要である。
