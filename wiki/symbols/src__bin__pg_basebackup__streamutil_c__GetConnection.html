GetConnection

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/streamutil.c:59

目的

PostgreSQLサーバーへのレプリケーション専用接続を確立し、初期設定（search_pathのクリア等）を行う。

シグネチャまたは形状

PGconn *GetConnection(void)

入力と出力

  • **入力**: グローバル変数（connection_string, dbname, dbhost, dbuser, dbport, dbgetpassword, password, progname）を参照。
  • **出力**: 接続成功時は PGconn* を返す。接続失敗時やエラー発生時は NULL を返すか、pg_fatal によりプロセスを終了する。

前提条件と不変条件

  • dbname と connection_string が同時に非NULLであってはならない（Assertによる検証）。
  • 接続文字列が指定されている場合、PQconninfoParse が成功する必要がある。

アルゴリズムまたは振る舞い

1. **接続情報の構築**: コマンドライン引数や接続文字列から keywords と values の配列を生成する。replication モードを有効化し、アプリケーション名を progname に設定する。 2. **接続試行**: PQconnectdbParams を使用してサーバーに接続する。 3. **パスワード要求**: 接続時にパスワードが必要な場合、simple_prompt を用いて対話的に取得し、再試行するループ構造を持つ。 4. **接続検証**: 接続状態が CONNECTION_OK でない場合はエラーログを出力し、リソースを解放して NULL を返す。 5. **環境初期化**: PostgreSQL 10以降のサーバーに対しては、セキュリティ向上のため ALWAYS_SECURE_SEARCH_PATH_SQL を実行し、search_path をクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**: ツール内の具体的な呼び出し元はソース抜粋からは特定不可。
  • *注記*: ツールが提示した contrib/postgres_fdw の呼び出し元情報は、本関数のシグネチャ（引数なし）と一致しないため、本ソースファイルにおける直接の呼び出し元とは見なせない。
  • **呼び出し先**: PQconnectdbParams, PQstatus, PQconnectionNeedsPassword, PQfinish, PQerrorMessage, PQconninfoParse, PQconninfoFree, PQexec, PQclear, PQserverVersion, simple_prompt, pg_fatal, pg_log_error, pg_malloc0。

関連データ構造

  • PGconn: libpqの接続ハンドル。
  • PQconninfoOption: 接続オプションの解析結果を保持。

失敗モードと境界条件

  • 接続文字列の解析失敗: pg_fatal で終了。
  • サーバーへの接続失敗: pg_log_error でエラーを出力し NULL を返す。
  • search_path のクリア失敗: pg_log_error を出力し exit(1) で終了。
  • メモリ不足: PQconnectdbParams が NULL を返した場合、pg_fatal で終了。

保守担当者向けメモ

  • 本関数はフロントエンド用であり、バックエンドの関数は呼び出せない。
  • PostgreSQL 10未満のサーバーとの互換性を維持するため、バージョンチェック（PQserverVersion）による条件分岐が含まれている。
  • 接続文字列の解析と引数のマージ処理は、将来的な接続オプションの追加時に整合性を確認すること。
