oid_array_to_list

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_proc.c:1204

目的

PostgreSQLの Datum 型として保持されているOID配列を、メモリ上の List 構造体へ変換する。

シグネチャまたは形状

List *oid_array_to_list(Datum datum)

入力と出力

  • **入力**: Datum datum（OIDの配列を含む ArrayType へのポインタを保持する Datum）
  • **出力**: List *（OIDを要素として含むリスト。リストが空の場合は NIL）

前提条件と不変条件

  • 入力 datum は有効な ArrayType を指している必要がある。
  • 内部的に deconstruct_array_builtin を使用するため、配列の要素型は OIDOID であることが前提となる。

アルゴリズムまたは振る舞い

1. DatumGetArrayTypeP を使用して、入力 datum から ArrayType 構造体を取得する。 2. deconstruct_array_builtin を呼び出し、配列をOIDの配列（values）と要素数（nelems）に分解する。 3. nelems 回ループし、lappend_oid を用いて各OIDを result リストに追加する。 4. 構築したリストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plperl/plperl.c:2821
  • src/pl/plpython/plpy_procedure.c:192
  • **呼び出し先**:
  • DatumGetArrayTypeP
  • deconstruct_array_builtin
  • lappend_oid

関連データ構造

  • ArrayType: PostgreSQLの配列データ構造。
  • List: PostgreSQLの標準的な連結リスト構造。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • 入力 datum が有効な配列でない場合、DatumGetArrayTypeP の内部でエラーが発生する可能性がある。
  • 配列が空の場合、nelems が 0 となり、NIL が返される。

保守担当者向けメモ

  • 本関数は OIDOID 型の配列専用である。他の型を扱う場合は別の変換関数が必要となる。
  • deconstruct_array_builtin の使用により、メモリ割り当ては現在のメモリコンテキストで行われる。
