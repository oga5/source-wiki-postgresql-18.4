TidStoreLockExclusive

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tidstore.c:286

目的

共有メモリ上に配置された TidStore に対して排他ロックを取得し、並行アクセスを制御する。

シグネチャまたは形状

void TidStoreLockExclusive(TidStore *ts)

入力と出力

  • **入力**: TidStore *ts - 操作対象の TidStore 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 ts は有効な TidStore を指している必要がある。
  • 共有メモリを使用する TidStore に対してのみ有効であり、ローカルメモリ版に対して呼び出した場合の動作は本関数内では明示されていない。

アルゴリズムまたは振る舞い

1. TidStoreIsShared(ts) を用いて、対象の TidStore が共有メモリを使用しているか判定する。 2. 共有メモリを使用している場合、内部の共有ラディックツリーに対して shared_ts_lock_exclusive を呼び出し、排他ロックを取得する。 3. 共有メモリを使用していない場合、何も行わずに終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_tidstore/test_tidstore.c:199
  • **呼び出し先**:
  • TidStoreIsShared (マクロまたはインライン関数と推測される)
  • shared_ts_lock_exclusive (内部関数)

関連データ構造

  • TidStore: TidStore の状態を保持するメイン構造体。

失敗モードと境界条件

  • ts が NULL の場合の動作は定義されていない。
  • 共有メモリ版ではない TidStore に対して呼び出した場合、ロックは取得されない。

保守担当者向けメモ

  • 本関数は共有メモリ版 TidStore の排他制御用である。共有メモリを使用する際は、呼び出し側で本関数によるロック管理が必須となる。
  • shared_ts_lock_exclusive の具体的な実装は src/backend/access/common/tidstore.c 内の他箇所または依存する lib/radixtree.h に依存する。
