gistScanPage

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistget.c:327

目的

GiSTインデックスの指定されたページを読み込み、検索条件に合致するタプルを抽出して、スキャン状態のキューへの追加や結果セットへの格納を行う。

シグネチャまたは形状

static void gistScanPage(IndexScanDesc scan, GISTSearchItem *pageItem, IndexOrderByDistance *myDistances, TIDBitmap *tbm, int64 *ntids)

入力と出力

  • **入力**:
  • scan: インデックススキャンの記述子。
  • pageItem: 走査対象のページ情報（ブロック番号、親LSN等）。
  • myDistances: 順序付きスキャン用の距離情報。
  • tbm: ビットマップスキャン時に使用するTIDビットマップ。
  • ntids: ビットマップに追加されたタプル数のカウンタ。
  • **出力**: なし（副作用としてスキャン状態の更新やビットマップへの追加を行う）。

前提条件と不変条件

  • pageItem はヒープタプルを指していないこと（Assert(!GISTSearchItemIsHeap(*pageItem))）。
  • ページ読み込み時に GIST_SHARE ロックを取得する。
  • ページが削除されている場合、処理を中断してバッファを解放する。

アルゴリズムまたは振る舞い

1. **ページ読み込み**: 指定されたブロックを読み込み、ロックを取得して gistcheckpage で検証する。 2. **右兄弟リンクの追跡**: 並行分割が発生している場合、rightlink を辿り、新しい GISTSearchItem を作成して優先度付きキューに追加する。 3. **削除済みページのスキップ**: ページが削除されている場合は即座に終了する。 4. **タプル走査**: ページ内の全タプルを順次評価する。

  • gistindex_keytest を呼び出し、検索条件との一致および距離を判定する。
  • ignore_killed_tuples が有効な場合、死んだタプルはスキップする。

    • 5. **結果の格納**:

  • ビットマップスキャン時: tbm_add_tuples でTIDを格納。
  • 非順序スキャン時: so->pageData 配列に格納。
  • 順序付きスキャンまたは非リーフページ時: GISTSearchItem を作成し、pairingheap に追加。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistget.c 内の各所（gistgettuple や gistgetbitmap 等から、再帰的または反復的に呼び出される）。
  • **呼び出し先**: ReadBuffer, LockBuffer, gistcheckpage, GistFollowRight, pairingheap_add, gistindex_keytest, tbm_add_tuples, gistFetchTuple 等。

関連データ構造

  • GISTScanOpaque: スキャン状態（キュー、メモリコンテキスト等）を保持。
  • GISTSearchItem: 検索対象のインデックスページまたはヒープタプルを表現する構造体。
  • TIDBitmap: ビットマップスキャン用の結果格納領域。

失敗モードと境界条件

  • **並行分割**: 親LSNと現在のページNSNを比較し、分割が発生していれば右兄弟をキューに追加する。
  • **ページ削除**: ページが削除済みの場合、処理をスキップする。
  • **メモリ管理**: gistindex_keytest 実行時は tempCxt を使用し、タプル取得時は pageDataCxt を使用してメモリリークを防ぐ。

保守担当者向けメモ

  • LP_DEAD ヒントの適用は、読み込み時のLSN（so->curPageLSN）を保存することで、MVCCスキャンにおいて安全に行われる。
  • 順序付きスキャン（ORDER BY）の場合、myDistances を用いてキューへの挿入順序が制御される。
