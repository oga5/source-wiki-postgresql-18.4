copy_byval_expanded_array

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_expanded.c:184

目的

既存の展開済み配列（ExpandedArrayHeader）の内容を、別のメモリコンテキスト上にコピーして複製を作成する。

シグネチャまたは形状

static void copy_byval_expanded_array(ExpandedArrayHeader *eah, ExpandedArrayHeader *oldeah)

入力と出力

  • **入力**:
  • eah: コピー先の展開済み配列ヘッダー（初期化済み）。
  • oldeah: コピー元の展開済み配列ヘッダー。
  • **出力**: なし（eah の各フィールドにコピー元から値が設定される）。

前提条件と不変条件

  • eah->hdr.eoh_context は、コピー先として有効なメモリコンテキストであること。
  • oldeah は有効な展開済み配列データを持っていること。
  • コピー先 eah のフラット表現（fvalue, fstartptr, fendptr）は NULL に初期化される。

アルゴリズムまたは振る舞い

1. oldeah から次元数（ndims）とデータ長（dvalueslen）を取得する。 2. eah->hdr.eoh_context を使用して、次元配列（dims, lbound）および要素データ配列（dvalues）をメモリ確保する。 3. oldeah から eah へ、次元情報、型情報（element_type, typlen 等）、および要素データ（dvalues）を memcpy でコピーする。 4. oldeah に NULL ビットマップ（dnulls）が存在する場合、同様にメモリ確保とコピーを行う。 5. 最後に、フラット表現に関連するポインタ（fvalue, fstartptr, fendptr）を NULL に設定し、シリアライズされていない状態であることを明示する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/array_expanded.c 内の関数から参照されている（詳細はソースコードの静的解析による）。
  • **呼び出し先**: MemoryContextAlloc（メモリ確保）、memcpy（メモリコピー）。

関連データ構造

  • ExpandedArrayHeader: 展開済み配列のヘッダー構造体。
  • MemoryContext: メモリ割り当ての管理単位。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAlloc がエラーを発生させる。
  • oldeah->dnulls が NULL の場合、コピー先でも dnulls は NULL として扱われる。

保守担当者向けメモ

  • 本関数は「byval（値渡し）」型の配列要素を扱うためのコピー処理を想定しているが、実装上は memcpy を使用しているため、要素型が typbyval であるかどうかにかかわらず、メモリ上のバイナリコピーを行う。
  • フラットな表現（fvalue 等）はコピーされず、常に NULL にリセットされる点に注意すること。
