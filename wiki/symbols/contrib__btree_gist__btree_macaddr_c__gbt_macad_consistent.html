gbt_macad_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_macaddr.c:122

目的

GiSTインデックス検索において、インデックス内の macaddr 範囲キーがクエリ条件と一致するかを判定する。

シグネチャまたは形状

Datum gbt_macad_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • entry (GISTENTRY *): 検査対象のインデックスエントリ。
  • query (macaddr *): 検索条件となる値。
  • strategy (StrategyNumber): 比較演算子（<, <=, =, >=, > 等）。
  • recheck (bool *): 検索結果の再評価が必要かを示すフラグ。
  • **出力**:
  • Datum (bool): 検索条件に合致すれば true を返す。
  • *recheck: 本関数は常に false を設定する（完全一致判定のため）。

前提条件と不変条件

  • entry->key は macKEY 構造体として解釈可能であること。
  • recheck は常に false に設定される。これは、本関数が提供する比較が正確（exact）であることを示す。

アルゴリズムまたは振る舞い

1. entry->key から macKEY 構造体を取得し、その lower および upper 境界を GBT_NUMKEY_R 構造体にマッピングする。 2. recheck を false に設定する。 3. 汎用ユーティリティ関数 gbt_num_consistent を呼び出し、tinfo（macaddr 用の比較関数テーブル）を使用してクエリとインデックスキーの比較を行う。 4. gbt_num_consistent の戻り値をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**: GiSTインデックスの検索プロセス（PostgreSQLのインデックススキャン機構）。
  • **呼び出し先**:
  • gbt_num_consistent (汎用ユーティリティ)
  • tinfo に登録された比較関数（gbt_macadlt 等）

関連データ構造

  • macKEY: インデックスノードに格納される lower と upper の境界値を持つ構造体。
  • GBT_NUMKEY_R: 汎用的な数値範囲を表現するための構造体。
  • tinfo: macaddr 型の比較ロジックを保持する静的構造体。

失敗モードと境界条件

  • entry が不正なポインタである場合、または entry->key が macKEY 構造体と一致しないメモリレイアウトである場合は未定義動作となる。
  • strategy がサポートされていない演算子である場合、gbt_num_consistent 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は btree_utils_num.h のパターンに従ったラッパーである。比較ロジックを変更する場合は、tinfo が参照している各比較関数（gbt_macadlt 等）を確認すること。
  • recheck が常に false であるため、GiSTインデックスの検索結果に対して追加のフィルタリングは不要である。
