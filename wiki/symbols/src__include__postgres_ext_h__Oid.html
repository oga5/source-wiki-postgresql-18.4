Oid

定義

  • 種別: typedef
  • 定義箇所: src/include/postgres_ext.h:32

目的

PostgreSQL のシステムオブジェクトを識別するための、バックエンドおよびフロントエンド間で共有されるオブジェクト識別子（Object Identifier）型。

シグネチャまたは形状

typedef unsigned int Oid;

入力と出力

  • 入力: 符号なし整数値。
  • 出力: システムカタログ内の特定の行やオブジェクトを指し示す識別子として機能する。

前提条件と不変条件

  • unsigned int 型として定義されており、非負の整数値をとる。
  • PostgreSQL のデータモデルの根幹として、システムオブジェクトの識別に使用される。

アルゴリズムまたは振る舞い

  • 本シンボル自体は型定義であり、実行時のロジックは持たない。
  • システムカタログの検索や、クライアントライブラリ（libpq）がオブジェクトを特定する際の共通言語として利用される。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • contrib/amcheck/verify_common.c
  • contrib/amcheck/verify_common.h
  • contrib/amcheck/verify_gin.c
  • contrib/amcheck/verify_heapam.c
  • contrib/amcheck/verify_nbtree.c
  • 呼び出し先:
  • 本シンボルは型定義であるため、直接的な呼び出し先は存在しない。

関連データ構造

  • InvalidOid: 無効な OID を表す定数（postgres_ext.h 内で定義）。

失敗モードと境界条件

  • InvalidOid を使用することで、オブジェクトが存在しない、あるいは未指定であることを表現する。
  • 整数型の範囲を超える値の扱いは、プラットフォームの unsigned int の仕様に依存する。

保守担当者向けメモ

  • 本型はフロントエンド（クライアント）からも参照される公開 API であるため、変更には高い互換性維持の責任が伴う。
  • 内部専用の識別子が必要な場合は、本ヘッダーではなく postgres.h 等の内部ヘッダーの使用を検討すること。
