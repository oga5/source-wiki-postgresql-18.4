RI_FKey_restrict_del

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/ri_triggers.c:658

目的

外部キー制約における RESTRICT アクションを実装するトリガープロシージャです。参照先の行が削除されようとした際、その行を参照している外部キーが存在する場合にエラーを発生させ、削除を阻止します。

シグネチャまたは形状

Datum RI_FKey_restrict_del(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（トリガー実行コンテキストを含む）。
  • **出力**: Datum 型（通常は PointerGetDatum(NULL) またはトリガーの戻り値）。

前提条件と不変条件

  • 本関数は、DELETE 操作に関連付けられたトリガーとして呼び出される必要があります。
  • 内部で ri_CheckTrigger を呼び出し、トリガーの呼び出しタイミングとイベントが RI_TRIGTYPE_DELETE に適合しているかを検証します。

アルゴリズムまたは振る舞い

1. ri_CheckTrigger を呼び出し、トリガー呼び出しの妥当性を検証します。 2. ri_restrict 関数を呼び出し、実際の制約チェックを実行します。この際、第2引数に false を渡すことで、NO ACTION および RESTRICT 共通のロジックを共有します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/tablecmds.c において、外部キー制約のトリガー定義時に本関数名が指定されます。
  • **呼び出し先**:
  • ri_CheckTrigger（トリガーの妥当性検証）
  • ri_restrict（制約チェックのメインロジック）

関連データ構造

  • TriggerData: トリガー実行時のコンテキスト情報。
  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約に基づく引数構造体。

失敗モードと境界条件

  • ri_CheckTrigger の検証に失敗した場合、エラーが発生します。
  • 参照整合性違反（参照している行が存在する状態で削除しようとした場合）が発生した場合、ri_restrict 経由でエラーが報告されます。

保守担当者向けメモ

  • 本関数は ri_restrict へのラッパーとして機能しています。ロジックの変更が必要な場合は、まず ri_restrict の実装を確認してください。
  • src/backend/commands/tablecmds.c からの参照が確認されており、外部キー制約の定義時に動的に割り当てられるため、関数名の変更には注意が必要です。
