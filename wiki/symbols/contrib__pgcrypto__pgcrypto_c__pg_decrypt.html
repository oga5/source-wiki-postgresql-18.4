pg_decrypt

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgcrypto.c:300

目的

SQL インターフェースから対称鍵暗号化されたデータを復号するためのメインエントリポイントです。

シグネチャまたは形状

Datum pg_decrypt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_BYTEA_PP(0): 復号対象のデータ (bytea)
  • PG_GETARG_BYTEA_PP(1): 復号に使用する鍵 (bytea)
  • PG_GETARG_TEXT_PP(2): アルゴリズム名 (text)
  • **出力**:
  • 復号されたデータ (bytea)

前提条件と不変条件

  • find_provider により、指定されたアルゴリズム名に対応する PX_Combo コンテキストが正常に取得できること。
  • PostgreSQL のメモリコンテキスト内で palloc によるメモリ確保が成功すること。
  • 復号処理終了後、px_combo_free を呼び出してリソースを解放すること。

アルゴリズムまたは振る舞い

1. find_provider を呼び出し、指定されたアルゴリズム名から PX_Combo プロバイダーを検索・初期化します。 2. 入力データと鍵のサイズを取得します。 3. px_combo_decrypt_len を使用して、復号後のデータに必要なバッファサイズを計算し、palloc でメモリを確保します。 4. px_combo_init で鍵を設定し、px_combo_decrypt を実行してデータを復号します。 5. 処理完了後、px_combo_free でコンテキストを解放します。 6. 復号結果のサイズを SET_VARSIZE で設定し、PG_RETURN_BYTEA_P で返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1(pg_decrypt) により登録）。
  • **呼び出し先**: find_provider, px_combo_decrypt_len, px_combo_init, px_combo_decrypt, px_combo_free, px_strerror。

関連データ構造

  • PX_Combo: 暗号化/復号処理を抽象化するコンテキスト構造体。
  • bytea, text: PostgreSQL のデータ型。

失敗モードと境界条件

  • アルゴリズムが見つからない場合や、復号処理中にエラーが発生した場合は ereport(ERROR, ...) を発行し、処理を中断します。
  • 復号エラー時には px_strerror(err) を通じて詳細なエラーメッセージが報告されます。

保守担当者向けメモ

  • 本関数は SQL と内部の px レイヤーを繋ぐブリッジです。暗号化アルゴリズム自体の実装については px.c や openssl.c を参照してください。
  • PG_FREE_IF_COPY を使用して、PostgreSQL のメモリ管理ルールに従ったメモリ解放を行っています。
