oidvectortypes

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/format_type.c:446

目的

oidvector（型OIDの配列）を、カンマ区切りの型名リスト（例: "integer, text"）を表す文字列に変換します。

シグネチャまたは形状

Datum oidvectortypes(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_POINTER(0) を介して oidvector * を受け取ります。
  • **出力**: text 型の Datum を返却します。

前提条件と不変条件

  • 入力された oidvector は check_valid_oidvector によって妥当性が検証される必要があります。
  • 内部で palloc を使用してメモリを確保し、必要に応じて repalloc で拡張します。

アルゴリズムまたは振る舞い

1. 入力された oidvector の要素数（dim1）を取得します。 2. 各要素（型OID）に対し、format_type_extended(oid, -1, FORMAT_TYPE_ALLOW_INVALID) を呼び出し、型名を文字列として取得します。 3. 取得した型名をカンマとスペース（, ）で連結し、一つの文字列を構築します。 4. 文字列の構築中、バッファが不足した場合は repalloc で動的に拡張します。 5. 最終的な文字列を cstring_to_text で text 型に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/tab-complete.in.c において、psql のタブ補完機能から参照されている記述があります。
  • **呼び出し先**:
  • check_valid_oidvector
  • format_type_extended
  • palloc / repalloc
  • cstring_to_text

関連データ構造

  • oidvector: 型OIDを格納する配列構造体。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • oidvector が無効な場合、check_valid_oidvector によりエラーが発生する可能性があります。
  • メモリ不足時には palloc / repalloc がエラーをスローします。
  • format_type_extended が無効な型OIDを処理する場合、フラグ FORMAT_TYPE_ALLOW_INVALID によりエラーを回避しつつ処理を継続します。

保守担当者向けメモ

  • 本関数は format_type_extended に依存しており、型名の解決ロジックに変更があった場合は影響を受けます。
  • 文字列連結のバッファ管理は手動で行われているため、要素数が多い場合のメモリ再割り当て効率に注意してください。
