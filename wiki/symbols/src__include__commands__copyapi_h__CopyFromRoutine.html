CopyFromRoutine

定義

  • 種別: struct
  • 定義箇所: src/include/commands/copyapi.h:61

目的

PostgreSQL の COPY FROM コマンドにおいて、データ形式（CSV, Text, Binary 等）ごとのインポート処理を抽象化し、プラグイン可能なインターフェースを提供します。

シグネチャまたは形状

typedef struct CopyFromRoutine
{
    void (*CopyFromInFunc) (CopyFromState cstate, Oid atttypid, FmgrInfo *finfo, Oid *typioparam);
    void (*CopyFromStart) (CopyFromState cstate, TupleDesc tupDesc);
    bool (*CopyFromOneRow) (CopyFromState cstate, ExprContext *econtext, Datum *values, bool *nulls);
    void (*CopyFromEnd) (CopyFromState cstate);
} CopyFromRoutine;

入力と出力

  • **CopyFromInFunc**: 入力関数情報を設定します。finfo（カタログ情報）および typioparam（型OID）を更新します。
  • **CopyFromStart**: 初期化処理を行います。戻り値はありません。
  • **CopyFromOneRow**: 1行分のデータを読み込み、values 配列と nulls 配列を埋めます。読み込むタプルが残っている場合は true、終了時は false を返します。
  • **CopyFromEnd**: 終了処理を行います。戻り値はありません。

前提条件と不変条件

  • 各コールバックに渡される cstate は、フォーマット固有のプライベートデータへのアクセスポイントとして機能します。
  • この構造体はサーバーの生存期間中保持されるため、通常は static const として定義する必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: CopyFromInFunc で入力関数情報を設定し、CopyFromStart でフォーマット固有の初期化を実行します。 2. **ループ**: CopyFromOneRow が繰り返し呼び出され、データソースから行を読み取ります。 3. **終了**: CopyFromEnd によりリソースの解放や後処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfrom.c 内の CopyFromRoutineText、CopyFromRoutineCSV、CopyFromRoutineBinary として定義されています。
  • **呼び出し先**: src/include/commands/copyfrom_internal.h の routine フィールドを通じて参照されます。

関連データ構造

  • CopyFromState: COPY FROM の状態管理用構造体。
  • TupleDesc: リレーションのタプル記述子。
  • ExprContext: デフォルト値の評価に使用されるコンテキスト。

失敗モードと境界条件

  • CopyFromOneRow が false を返すと、読み込みループが終了します。
  • econtext はデフォルト値が使用されない場合 NULL になる可能性があります。

保守担当者向けメモ

  • 新しいデータフォーマットを追加する際は、本構造体を実装し、コアの COPY ロジックから呼び出されるように登録してください。
  • 各コールバック関数は、フォーマット固有のロジックをコアエンジンから分離する役割を担います。
