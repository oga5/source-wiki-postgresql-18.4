ST_SORT

定義

  • 種別: function
  • 定義箇所: src/include/lib/sort_template.h:291

目的

コンパイル時に型特化されたクイックソート関数を生成し、汎用的な qsort に伴う関数呼び出しオーバーヘッドを排除してソート処理を高速化する。

シグネチャまたは形状

ST_SCOPE void ST_SORT(ST_ELEMENT_TYPE * data, size_t n
                      ST_SORT_PROTO_ELEMENT_SIZE
                      ST_SORT_PROTO_COMPARE
                      ST_SORT_PROTO_ARG)

※ ST_SORT_PROTO_* マクロにより、要素サイズ、比較関数、およびコンテキスト引数が追加される。

入力と出力

  • **入力**:
  • data: ソート対象配列の先頭ポインタ。
  • n: 配列の要素数。
  • ST_SORT_PROTO_*: 比較ロジックやコンテキストを含む追加パラメータ。
  • **出力**: なし（data が指す配列がインプレースでソートされる）。

前提条件と不変条件

  • DO_CHECK_FOR_INTERRUPTS() マクロが定義されている環境であること。
  • ST_ELEMENT_TYPE、ST_COMPARE 等の必要なマクロが定義された状態で本ヘッダーがインクルードされていること。
  • 再帰呼び出しにおいて、常に小さいパーティションを先に処理することでスタック深さを $O(\log N)$ に制限する。

アルゴリズムまたは振る舞い

1. **挿入ソートへのフォールバック**: 要素数 n < 7 の場合、挿入ソートを実行して終了する。 2. **事前ソートチェック**: 配列が既にソート済みであれば早期終了する。 3. **ピボット選択**: ST_MED3 を使用してピボットを選択する（要素数が多い場合は中央値の計算を強化）。 4. **パーティショニング**: Bentley-McIlroy クイックソートに基づき、3方向分割（等価な要素を中央にまとめる）を行う。 5. **再帰と反復**: 小さいパーティションを再帰的にソートし、大きいパーティションは goto loop を用いた反復処理（末尾再帰最適化）で処理し、スタック消費を抑える。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c (sort_pending_writebacks)
  • src/backend/utils/sort/tuplesort.c (qsort_tuple_signed, qsort_tuple_int32, qsort_tuple, qsort_ssup)
  • **呼び出し先**:
  • DO_COMPARE, DO_SWAP, DO_SWAPN, DO_MED3, DO_SORT（いずれもマクロとして定義された関数または処理）。

関連データ構造

  • ST_ELEMENT_TYPE: ソート対象のデータ型。
  • ST_POINTER_TYPE: 配列の走査に使用されるポインタ型。

失敗モードと境界条件

  • **スタックオーバーフロー**: 再帰の深さを $O(\log N)$ に抑える設計だが、極端なデータ分布やスタック制限が厳しい環境では注意が必要。
  • **割り込み**: DO_CHECK_FOR_INTERRUPTS() が呼び出されるため、ソート中に処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数はマクロ展開によって生成されるため、デバッグ時は生成後の関数名で検索すること。
  • 比較ロジックのインライン化は、複雑な関数よりも単純な if-else 構造の方がコンパイラの最適化効率が高い。
