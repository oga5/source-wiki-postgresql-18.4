CatalogIndexInsert

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/indexing.c:74

目的

システムカタログのタプル挿入に伴い、関連するすべてのインデックスに対してエントリを挿入し、整合性を維持する。

シグネチャまたは形状

static void CatalogIndexInsert(CatalogIndexState indstate, HeapTuple heapTuple, TU_UpdateIndexes updateIndexes)

入力と出力

  • **入力**:
  • indstate: インデックス情報を含む CatalogIndexState 構造体。
  • heapTuple: 挿入対象のヒープタプル。
  • updateIndexes: インデックス更新の範囲を指定する列挙型（TU_All または TU_Summarizing）。
  • **出力**: なし（副作用としてインデックスへの挿入を実行）。

前提条件と不変条件

  • システムカタログは式インデックス、部分インデックス、除外制約、遅延ユニーク制約をサポートしない（Assert により検証）。
  • indstate に含まれるインデックスは即時更新（indimmediate）である必要がある。
  • onlySummarized モードの場合、タプルは HOT (Heap Only Tuple) 更新である必要がある。

アルゴリズムまたは振る舞い

1. **HOT更新の判定**: HeapTupleIsHeapOnly を確認し、HOT更新であればインデックス更新をスキップする（USE_ASSERT_CHECKING が有効な場合は整合性チェックのみ行う）。 2. **初期化**: indstate からインデックス情報を取得し、タプルを保持するための TupleTableSlot を作成する。 3. **インデックス走査**: 各インデックスに対して以下を繰り返す。

  • ii_ReadyForInserts が偽であればスキップ。
  • onlySummarized モードで ii_Summarizing が偽であればスキップ。
  • FormIndexDatum を使用してインデックスキーを生成。
  • index_insert を呼び出し、インデックスにエントリを挿入。

    • 4. **終了**: 使用した TupleTableSlot を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/indexing.c 内の CatalogTupleInsert、CatalogTupleInsertWithInfo、CatalogTupleUpdate、CatalogTupleUpdateWithInfo、CatalogTuplesMultiInsertWithInfo から呼び出される。
  • **呼び出し先**: FormIndexDatum（キー生成）、index_insert（インデックス挿入）、ExecStoreHeapTuple / MakeSingleTupleTableSlot / ExecDropSingleTupleTableSlot（スロット管理）。

関連データ構造

  • CatalogIndexState: インデックス情報（IndexInfo 配列やリレーション記述子）を保持する構造体。
  • HeapTuple: 操作対象のタプル。
  • TupleTableSlot: タプルを一時的に保持する実行時構造体。

失敗モードと境界条件

  • インデックスが ii_ReadyForInserts でない場合は無視される。
  • numIndexes == 0 の場合は何もせず即座に復帰する。
  • システムカタログの制約（式インデックス等がないこと）に違反した場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数は HOT 更新を検知してインデックス更新を回避する最適化を含んでいる。
  • システムカタログ特有の制約が Assert で厳格に保護されているため、カタログ定義を変更する際は注意が必要。
  • CatalogIndexState のキャッシュ機構は存在しないため、複数操作時は *WithInfo 系関数経由での利用を推奨。
