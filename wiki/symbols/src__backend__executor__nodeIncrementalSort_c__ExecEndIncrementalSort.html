ExecEndIncrementalSort

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIncrementalSort.c:1076

目的

IncrementalSortState ノードの実行を終了し、関連するリソースを解放する。

シグネチャまたは形状

void ExecEndIncrementalSort(IncrementalSortState *node)

入力と出力

  • **入力**: IncrementalSortState *node（終了対象のインクリメンタルソートノードの状態構造体）
  • **出力**: なし

前提条件と不変条件

  • node は有効な IncrementalSortState を指している必要がある。
  • 呼び出し後、node->fullsort_state および node->prefixsort_state は NULL に設定される。

アルゴリズムまたは振る舞い

1. SO_printf を用いて終了処理の開始をログ出力する。 2. ExecDropSingleTupleTableSlot を呼び出し、group_pivot および transfer_tuple スロットを解放する。 3. fullsort_state が存在する場合、tuplesort_end を呼び出してリソースを解放し、ポインタを NULL にする。 4. prefixsort_state が存在する場合、同様に tuplesort_end を呼び出してリソースを解放し、ポインタを NULL にする。 5. outerPlanState(node) を引数として ExecEndNode を呼び出し、サブプランの終了処理を行う。 6. 終了処理の完了をログ出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:714（エグゼキュータのノード終了処理経由）
  • **呼び出し先**:
  • ExecDropSingleTupleTableSlot
  • tuplesort_end
  • ExecEndNode

関連データ構造

  • IncrementalSortState: 本ノードの実行状態を保持する構造体。
  • TupleTableSlot: タプル保持用のスロット。

失敗モードと境界条件

  • node が NULL の場合、あるいは node 内の各ポインタが不正な場合は未定義の動作となる。
  • tuplesort_end は NULL ポインタに対しては安全に動作するよう設計されている（関数内でチェック済み）。

保守担当者向けメモ

  • 本関数は EXEC_FLAG_BACKWARD や EXEC_FLAG_MARK をサポートしていないため、それらに関連するリソース解放処理は含まれていない。
  • サブプランの終了には ExecEndNode を使用しており、ノード階層のクリーンアップ順序を維持している。
