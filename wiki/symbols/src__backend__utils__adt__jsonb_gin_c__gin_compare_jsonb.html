gin_compare_jsonb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_gin.c:202

目的

GINインデックスのキー比較に使用される比較関数です。2つのjsonbキー（text型として表現）を比較し、その順序関係を返します。

シグネチャまたは形状

Datum gin_compare_jsonb(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される2つの text 型のポインタ（arg1, arg2）。
  • **出力**: 比較結果を示す int32 値（arg1 < arg2 なら負、arg1 == arg2 なら0、arg1 > arg2 なら正）。

前提条件と不変条件

  • 入力される text 型データは、VARDATA_ANY および VARSIZE_ANY_EXHDR マクロで適切に展開・サイズ取得可能である必要があります。
  • 比較は常に C_COLLATION_OID（Cロケール）を使用して行われます。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を使用して、入力された2つの引数を取得します。 2. VARDATA_ANY および VARSIZE_ANY_EXHDR を用いて、各テキストデータのデータ本体へのポインタと長さを抽出します。 3. varstr_cmp 関数を呼び出し、Cロケールに基づいて2つの文字列を比較します。 4. PG_FREE_IF_COPY を使用して、必要に応じて一時的なコピーを解放します。 5. PG_RETURN_INT32 を使用して、比較結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は確認されていません。GINインデックスの演算子クラス定義において、比較関数として登録されることを想定しています。
  • **呼び出し先**:
  • varstr_cmp (PostgreSQL内部の文字列比較関数)
  • PG_GETARG_TEXT_PP, VARDATA_ANY, VARSIZE_ANY_EXHDR, PG_FREE_IF_COPY, PG_RETURN_INT32 (PostgreSQLの関数マネージャマクロ)

関連データ構造

  • text: PostgreSQLの可変長文字列型。

失敗モードと境界条件

  • 入力がNULLの場合、PG_GETARG_TEXT_PP の仕様に従いエラーが発生する可能性があります。
  • 文字列の長さが異なる場合や、内容が一致する場合の比較は varstr_cmp の挙動に依存します。

保守担当者向けメモ

  • 本関数は jsonb_ops 演算子クラスにおいて、GINインデックスのキー順序を決定するために使用されます。
  • 比較ロジックを変更する場合は、インデックスの整合性が損なわれないよう注意してください。
