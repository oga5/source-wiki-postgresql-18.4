check_errno

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/pgtypeslib-num_test2.c:264

目的

グローバル変数 errno の値を判定し、対応するエラーメッセージを標準出力へ書き出すデバッグ用関数。

シグネチャまたは形状

static void check_errno(void)

入力と出力

  • 入力: グローバル変数 errno（errno.h で定義されるシステムエラーコードまたは pgtypes_error.h で定義される数値型エラーコード）。
  • 出力: なし（標準出力 stdout への文字列出力のみ）。

前提条件と不変条件

  • errno が適切な値に設定されている状態で呼び出されることを想定している。
  • printf を使用するため、標準出力が利用可能である必要がある。

アルゴリズムまたは振る舞い

1. errno の値を switch 文で評価する。 2. 0 の場合は (no errno set) を出力する。 3. PGTYPES_NUM_OVERFLOW, PGTYPES_NUM_UNDERFLOW, PGTYPES_NUM_BAD_NUMERIC, PGTYPES_NUM_DIVIDE_ZERO のいずれかに該当する場合、対応するエラー名を括弧付きで出力する。 4. 上記以外（default）の場合、数値としての errno と、strerror(errno) によって取得したシステムエラーメッセージを出力する。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/test/expected/compat_informix-dec_test.c 内の各テストケースから、エラー発生時（戻り値が負または非ゼロの場合）に呼び出される。
  • 呼び出し先: printf, strerror（標準ライブラリ）。

関連データ構造

  • pgtypes_error.h: エラーコード定数（PGTYPES_NUM_*）の定義元。

失敗モードと境界条件

  • errno が定義された定数以外の値である場合、default ケースにより数値とシステムエラー文字列が出力される。
  • strerror がスレッドセーフでない環境や、特定のロケールに依存する可能性がある。

保守担当者向けメモ

  • 本関数は主に回帰テストのデバッグ出力を目的としている。
  • pgtypes ライブラリで新しいエラーコードが追加された場合は、switch 文に case を追加すること。
