FmgrInfo

定義

  • 種別: struct
  • 定義箇所: src/include/fmgr.h:56

目的

PostgreSQL の関数マネージャ（fmgr）において、SQL 関数を C 言語で実行するために必要なメタデータおよび実行時情報を保持するキャッシュ構造体です。

シグネチャまたは形状

typedef struct FmgrInfo
{
	PGFunction	fn_addr;
	Oid			fn_oid;
	short		fn_nargs;
	bool		fn_strict;
	bool		fn_retset;
	unsigned char fn_stats;
	void	   *fn_extra;
	MemoryContext fn_mcxt;
	fmNodePtr	fn_expr;
} FmgrInfo;

入力と出力

  • **入力**: fmgr_info() を介して OID から初期化されます。
  • **出力**: 関数呼び出し時に FunctionCallInvoke 等へ渡され、実行制御に使用されます。

前提条件と不変条件

  • fn_nargs は 0 から FUNC_MAX_ARGS の範囲内である必要があります。
  • fn_extra に格納されるデータは、fn_mcxt で指定されたメモリコンテキストの生存期間に依存します。

アルゴリズムまたは振る舞い

1. **初期化**: fmgr_info() により、システムカタログから関数情報を取得し、この構造体にキャッシュします。 2. **実行制御**: fn_strict が true の場合、引数に NULL が含まれると関数本体を呼び出さずに NULL を返します。 3. **拡張性**: fn_extra フィールドは、関数ハンドラが実行時に必要な状態を保持するための領域として利用可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist や contrib/bloom などの拡張モジュールにおいて、インデックス操作や比較関数を呼び出す際に利用されています。
  • **呼び出し先**: fn_addr に格納された関数ポインタが、実際の C 言語実装関数を指します。

関連データ構造

  • FunctionCallInfo: 関数呼び出し時の引数やコンテキストを保持する構造体。
  • MemoryContext: fn_extra のメモリ管理に使用されるコンテキスト。

失敗モードと境界条件

  • fn_mcxt が不適切な場合、fn_extra へのアクセス時にメモリ破壊や無効なポインタ参照が発生する可能性があります。
  • fn_addr が正しくロードされていない状態で呼び出すと、セグメンテーションフォールトが発生します。

保守担当者向けメモ

  • FmgrInfo の内容はキャッシュされるため、関数定義の変更（CREATE OR REPLACE FUNCTION 等）が発生した場合は、キャッシュの無効化または再初期化が必要です。
  • 長期的なキャッシュを行う場合は fmgr_info_cxt を使用してメモリコンテキストを明示的に管理してください。
