pg_strdup

定義

  • 種別: function
  • 定義箇所: src/common/fe_memutils.c:90

目的

入力された文字列を複製し、新しいメモリ領域を確保して返却します。メモリ確保に失敗した場合や、不正な入力が与えられた場合にはプロセスを終了させることで、呼び出し側のエラーハンドリングを簡略化します。

シグネチャまたは形状

char *pg_strdup(const char *in);

入力と出力

  • **入力**: in (複製対象のヌル終端文字列へのポインタ)
  • **出力**: 複製された文字列を保持する、新しく確保されたメモリ領域へのポインタ。

前提条件と不変条件

  • in は NULL であってはなりません。
  • 戻り値として返されるポインタは、呼び出し側で free()（または pg_free()）を使用して解放する必要があります。

アルゴリズムまたは振る舞い

1. 入力 in が NULL かどうかを判定します。NULL の場合はエラーメッセージを stderr に出力し、exit(EXIT_FAILURE) でプロセスを終了します。 2. 標準ライブラリの strdup(in) を呼び出し、メモリを確保して文字列をコピーします。 3. strdup の戻り値が NULL（メモリ不足）の場合、stderr に "out of memory" と出力し、exit(EXIT_FAILURE) でプロセスを終了します。 4. 確保したメモリ領域へのポインタを返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/oid2name/oid2name.c
  • contrib/vacuumlo/vacuumlo.c
  • src/bin/initdb/initdb.c

    • （上記はツールによる参照候補であり、網羅的ではありません）

  • **呼び出し先**:
  • strdup (標準ライブラリ)
  • fprintf (標準ライブラリ)
  • exit (標準ライブラリ)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • **入力が NULL**: 内部エラーとして扱い、プロセスを終了します。
  • **メモリ不足**: strdup が失敗した場合、エラーメッセージを出力してプロセスを終了します。

保守担当者向けメモ

  • 本関数はフロントエンド専用のユーティリティです。バックエンドコードとの互換性を維持するため、メモリ確保失敗時には必ずプロセスを終了させる設計となっています。
