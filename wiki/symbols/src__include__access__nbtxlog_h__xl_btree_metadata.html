xl_btree_metadata

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/nbtxlog.h:49

目的

B-treeインデックスのメタページを再構築するために必要な最小限の情報を保持し、WAL（Write-Ahead Log）への記録およびクラッシュリカバリ時の復元に使用されます。

シグネチャまたは形状

typedef struct xl_btree_metadata
{
    uint32      version;
    BlockNumber root;
    uint32      level;
    BlockNumber fastroot;
    uint32      fastlevel;
    uint32      last_cleanup_num_delpages;
    bool        allequalimage;
} xl_btree_metadata;

入力と出力

  • **入力**: B-tree操作時に現在のメタページ状態から各フィールドが設定されます。
  • **出力**: WALレコードの一部として書き出され、REDO処理時にメタページの物理的な状態を復元するために使用されます。

前提条件と不変条件

  • sizeof(xl_btree_metadata) は、WALレコードの読み込み時に整合性チェック（Assert）の基準として使用されます。
  • REDO処理においては、LSN（Log Sequence Number）チェックを介して冪等性が維持される必要があります。

アルゴリズムまたは振る舞い

1. **記録**: nbtree のインデックス操作（メタページ更新等）が発生した際、本構造体に現在のメタページ情報を格納します。 2. **WAL書き出し**: XLogRegisterBufData を通じてWALレコードに登録されます。 3. **復元**: クラッシュリカバリ時、btree_redo がWALレコードを読み込み、この構造体の値を用いてメタページの内容を再構築します。

呼び出し元と呼び出し先

  • **呼び出し元（記録側）**:
  • src/backend/access/nbtree/nbtinsert.c
  • src/backend/access/nbtree/nbtpage.c
  • **呼び出し先（REDO側）**:
  • src/backend/access/nbtree/nbtxlog.c （btree_redo 関数内で利用）

関連データ構造

  • XLogReaderState: WALレコードの読み取りに使用。
  • nbtree 関連の他のWAL構造体（xl_btree_insert, xl_btree_split 等）。

失敗モードと境界条件

  • WALレコードのサイズが sizeof(xl_btree_metadata) と一致しない場合、nbtxlog.c 内の Assert により不整合が検出されます。

保守担当者向けメモ

  • 本構造体はメタページの物理構造を反映しているため、メタページの定義（nbtree.h）に変更を加える際は、本構造体のフィールド構成も同期させる必要があります。
