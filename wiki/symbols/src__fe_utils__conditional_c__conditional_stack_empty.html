conditional_stack_empty

定義

  • 種別: function
  • 定義箇所: src/fe_utils/conditional.c:129

目的

ConditionalStack が現在空である（スタック内に要素が存在しない）かどうかを判定します。

シグネチャまたは形状

bool conditional_stack_empty(ConditionalStack cstack)

入力と出力

  • **入力**: cstack (型: ConditionalStack) - 検査対象のスタック構造体。
  • **出力**: bool - スタックが空であれば true、要素が存在すれば false を返します。

前提条件と不変条件

  • cstack は有効な ConditionalStack 構造体を指している必要があります。

アルゴリズムまたは振る舞い

スタック構造体の head メンバが NULL であるかどうかを直接評価し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pgbench/pgbench.c: 複数の箇所でスタックの状態確認およびアサーションに使用。
  • src/bin/psql/mainloop.c: \if ブロックのネスト状態確認に使用。
  • src/fe_utils/conditional.c: 同一ファイル内の他の関数（conditional_stack_pop 等）から内部的に使用。
  • **呼び出し先**:
  • 本関数は他の関数を呼び出しません。

関連データ構造

  • ConditionalStack: fe_utils/conditional.h で定義される、\if ブロックのネストを管理するスタック構造体。

失敗モードと境界条件

  • cstack が NULL の場合、デリファレンスによりクラッシュする可能性があります。

保守担当者向けメモ

  • 本関数はスタックの空状態を判定する軽量な述語です。スタックのメモリ管理（create/destroy）とは独立して動作します。
