InitializeFastPathLocks

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/postinit.c:579

目的

バックエンドプロセスごとの高速パスロック（FastPath Lock）グループ数を決定し、グローバル変数 FastPathLockGroupsPerBackend を初期化します。

シグネチャまたは形状

void InitializeFastPathLocks(void)

入力と出力

  • **入力**: GUC 変数 max_locks_per_xact を参照します。
  • **出力**: グローバル変数 FastPathLockGroupsPerBackend を設定します。

前提条件と不変条件

  • 本関数はプロセス起動時に一度だけ呼び出される必要があります（FastPathLockGroupsPerBackend == 0 であることをアサート）。
  • 設定される値は 2 のべき乗である必要があります。
  • 設定値は最低 1 以上、かつ FP_LOCK_GROUPS_PER_BACKEND_MAX を上限とします。

アルゴリズムまたは振る舞い

1. max_locks_per_xact を基に、必要なロックグループ数を計算します。 2. pg_nextpower2_32 を使用して、計算結果を 2 のべき乗に調整します。 3. FP_LOCK_SLOTS_PER_GROUP で除算し、上限値（FP_LOCK_GROUPS_PER_BACKEND_MAX）と下限値（1）の範囲内に収めます。 4. 最終的な値が 2 のべき乗であることをアサートで検証します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/bootstrap/bootstrap.c
  • src/backend/postmaster/postmaster.c
  • src/backend/storage/ipc/ipci.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • pg_nextpower2_32（マクロまたは関数）
  • Max, Min（マクロ）
  • Assert（マクロ）

関連データ構造

  • FastPathLockGroupsPerBackend（グローバル変数）
  • max_locks_per_xact（GUC 変数）

失敗モードと境界条件

  • max_locks_per_xact が極端に小さい場合でも、最低 1 グループが割り当てられるよう制御されています。
  • 初期化が二重に行われた場合、アサートによりプロセスが停止します。

保守担当者向けメモ

  • 本関数はロックマネージャの初期化において重要な役割を果たします。変更を加える際は、src/include/storage/proc.h 等で定義されているロックグループの計算ロジックとの整合性に注意してください。
