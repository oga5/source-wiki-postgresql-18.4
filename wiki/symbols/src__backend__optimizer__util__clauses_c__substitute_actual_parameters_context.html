substitute_actual_parameters_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/util/clauses.c:70

目的

SQL関数やセットを返す関数（SRF）のインライン化処理において、関数呼び出し時の実引数を式ツリー内のパラメータと置換するためのコンテキスト情報を保持します。

シグネチャまたは形状

typedef struct {
    int         nargs;      /* 引数の数 */
    List       *args;       /* 実引数のリスト */
    int        *usecounts;  /* 各引数の使用回数追跡用配列 */
} substitute_actual_parameters_context;

入力と出力

本構造体は、関数呼び出しの引数リスト（args）と、その引数が式内で何回参照されたかを追跡するためのカウンタ（usecounts）を管理します。

前提条件と不変条件

  • nargs は args リストの長さと一致している必要があります。
  • usecounts 配列は、少なくとも nargs 分のメモリ領域が確保されている必要があります。

アルゴリズムまたは振る舞い

式ツリーの走査中に、特定のパラメータノードを実引数に置換する際の補助データとして使用されます。usecounts を通じて、引数が複数回評価されるべきか、あるいは一度の評価で済むかを制御する情報源となります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/clauses.c 内のインライン化関連関数（inline_function 等）から参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • List: PostgreSQLの汎用リスト構造。
  • substitute_actual_srf_parameters_context: SRF（セットを返す関数）の置換に特化した類似のコンテキスト構造体。

失敗モードと境界条件

  • args リストが期待される引数の数と一致しない場合、置換処理中に不正なメモリ参照が発生する可能性があります。
  • usecounts が初期化されていない、またはサイズが不足している場合、インデックス範囲外アクセスが発生するリスクがあります。

保守担当者向けメモ

  • 本構造体は clauses.c 内のインライン化ロジックと密接に結合しています。引数の置換ロジックを変更する際は、usecounts の更新タイミングが正しいか確認してください。
  • 呼び出し元の詳細なコールグラフは、現在のソース解析範囲では限定的です。必要に応じて inline_function 周辺のフローを追跡してください。
