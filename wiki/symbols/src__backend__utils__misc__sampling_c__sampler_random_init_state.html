sampler_random_init_state

定義

  • src/backend/utils/misc/sampling.c:233

目的

サンプリング処理で使用する乱数生成器の状態（pg_prng_state）を、指定されたシード値を用いて初期化する。

シグネチャまたは形状

void sampler_random_init_state(uint32 seed, pg_prng_state *randstate)

入力と出力

  • **入力**:
  • seed (uint32): 乱数生成器の初期化に使用するシード値。
  • randstate (pg_prng_state *): 初期化対象となる乱数生成器の状態構造体へのポインタ。
  • **出力**: なし（randstate が更新される）。

前提条件と不変条件

  • randstate は有効なメモリ領域を指している必要がある。
  • 内部で pg_prng_seed を呼び出すため、pg_prng モジュールの仕様に従い、シード値に基づいた決定論的な乱数系列が生成される状態となる。

アルゴリズムまたは振る舞い

1. 受け取った seed を uint64 にキャストする。 2. pg_prng_seed 関数を呼び出し、randstate を指定されたシード値で初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/tsm_system_rows/tsm_system_rows.c
  • contrib/tsm_system_time/tsm_system_time.c
  • src/backend/utils/misc/sampling.c（複数の箇所で初期化に使用）
  • **呼び出し先**:
  • pg_prng_seed

関連データ構造

  • pg_prng_state: 乱数生成器の状態を保持する構造体。

失敗モードと境界条件

  • randstate が NULL の場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は pg_prng のラッパーとして機能している。乱数生成のアルゴリズムを変更する場合は、pg_prng 側の実装を確認すること。
  • 新規のサンプリングロジックを実装する際は、本関数を用いて乱数状態を初期化することが推奨される。
