secure_raw_read

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure.c:267

目的

暗号化層を介さず、Port 構造体が保持するソケットから直接データを読み取る低レベルなI/O関数です。

シグネチャまたは形状

ssize_t secure_raw_read(Port *port, void *ptr, size_t len);

入力と出力

  • **入力**:
  • port: 通信状態を保持する Port 構造体へのポインタ。
  • ptr: 読み取ったデータを格納するバッファ。
  • len: 読み取りを要求するバイト数。
  • **出力**:
  • 読み取ったバイト数（ssize_t）。エラー発生時は -1 を返し、errno が設定されます。

前提条件と不変条件

  • port は有効なポインタである必要があります。
  • port->raw_buf が存在する場合、raw_buf_remaining が正の数であれば、ソケット読み取りの前にバッファ内のデータが優先的に消費されます。

アルゴリズムまたは振る舞い

1. **バッファ確認**: port->raw_buf_remaining が 0 より大きい場合、バッファ内の未処理データを ptr にコピーし、バッファのオフセットと残量を更新して即座に返します。 2. **ソケット読み取り**: バッファが空の場合、recv() を呼び出してソケットから直接データを読み取ります。

  • Windows環境では、recv() 呼び出しの前後で pgwin32_noblock フラグを一時的に切り替え、非ブロッキング動作を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-secure-gssapi.c (GSSAPI受信処理)
  • src/backend/libpq/be-secure-openssl.c (OpenSSL受信処理)
  • src/backend/libpq/be-secure.c (内部的な読み取り処理)
  • **呼び出し先**:
  • recv() (OS標準のソケット読み取り関数)

関連データ構造

  • Port: 通信状態、ソケット記述子、および未処理の生データバッファ（raw_buf）を保持する構造体。

失敗モードと境界条件

  • recv() がエラーを返した場合、その戻り値（通常 -1）をそのまま返します。
  • バッファ消費時、要求された len が raw_buf_remaining より大きい場合は、バッファ内の残量分のみを読み取ります。

保守担当者向けメモ

  • 本関数は「生の」読み取りを行うため、SSL/TLSやGSSAPIの暗号化処理は含まれません。暗号化された通信が必要な場合は、上位の secure_read を使用してください。
  • raw_buf は、ハンドシェイク開始前に読み込まれてしまった平文データを保持するための領域であり、接続確立時に適切に消費される設計となっています。
