toast_save_datum

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/toast_internals.c:118

目的

巨大な属性値（Datum）をTOASTテーブルに分割保存し、外部ストレージを参照するためのポインタ（varatt_external）を生成します。

シグネチャまたは形状

Datum toast_save_datum(Relation rel, Datum value, struct varlena *oldexternal, int options)

入力と出力

  • **入力**:
  • rel: 対象テーブルのリレーション。
  • value: 保存対象のデータ（varlena形式）。
  • oldexternal: テーブル再構築時などに使用される既存の外部ポインタ（任意）。
  • options: 操作オプション。
  • **出力**:
  • 外部ストレージを参照する varatt_external 構造体を格納した Datum。

前提条件と不変条件

  • 入力 value は外部データ（VARATT_IS_EXTERNAL）であってはならない。
  • TOASTテーブルおよびそのインデックスは RowExclusiveLock で開かれる。
  • トランザクション内で呼び出される必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: 対象テーブルのTOASTリレーションとインデックスを開く。 2. **メタデータ計算**: 入力データの状態（ショート、圧縮済み、通常）に応じて、va_rawsize（元のサイズ）および va_extinfo（外部格納時の情報）を算出する。 3. **OID決定**:

  • 通常時: GetNewOidWithIndex を使用して一意な va_valueid を生成。
  • 再構築時（rel->rd_toastoid が有効な場合）: oldexternal から既存のOIDを再利用し、重複がある場合はデータ書き込みをスキップする最適化を行う。

    • 4. **データ格納**: チャンクデータ（TOAST_MAX_CHUNK_SIZE単位）をTOASTテーブルへ挿入する準備を行う（ソース抜粋はここまで）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/table/toast_helper.c
  • src/backend/commands/cluster.c
  • **呼び出し先**:
  • table_open
  • toast_open_indexes
  • GetNewOidWithIndex
  • toastrel_valueid_exists
  • toastid_valueid_exists

関連データ構造

  • varatt_external: 外部ストレージ上のデータを指すポインタ構造体。
  • Relation: テーブルおよびTOASTテーブルのメタデータ。
  • varlena: 可変長データのヘッダー構造。

失敗モードと境界条件

  • oldexternal が指定されている場合、そのデータが VARATT_IS_EXTERNAL_ONDISK であることを検証する。
  • テーブル再構築時に既存のOIDと競合する場合、toastid_valueid_exists を用いて一意性を保証する。

保守担当者向けメモ

  • rd_toastoid を用いた再利用ロジックは、CLUSTER等のテーブル再構築時にTOAST領域の重複を防ぐための重要な最適化である。
  • チャンクの物理的な挿入ループは、本関数の後半（ソース抜粋範囲外）で実行される。
