perform_spin_delay

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/s_lock.c:125

目的

スピンロック取得の試行において、競合が発生した際の待機戦略（指数バックオフおよびスタック検出）を実行します。

シグネチャまたは形状

void perform_spin_delay(SpinDelayStatus *status)

入力と出力

  • **入力**: SpinDelayStatus *status（現在のスピン試行回数、遅延時間、および呼び出し元の位置情報を含む構造体へのポインタ）
  • **出力**: なし（副作用として status 内のカウンタおよび遅延時間を更新）

前提条件と不変条件

  • status は呼び出し元で適切に初期化されている必要があります。
  • spins_per_delay（グローバル変数）が正しく設定されていること。
  • pg_global_prng_state が初期化されていること。

アルゴリズムまたは振る舞い

1. **CPU遅延**: SPIN_DELAY() マクロを実行し、ハードウェアレベルの短い待機を行います。 2. **スピンカウント**: status->spins をインクリメントし、spins_per_delay に達したか確認します。 3. **スタック検出**: 待機回数が NUM_DELAYS を超えた場合、s_lock_stuck を呼び出し致命的エラーとして処理します。 4. **指数バックオフ**:

  • 初回待機時は MIN_DELAY_USEC を設定します。
  • pgstat_report_wait_start/end を用いて待機イベントを記録し、pg_usleep で待機します。
  • 次回の待機時間をランダムに 1 倍から 2 倍の範囲で増加させます。
  • MAX_DELAY_USEC を超えた場合は MIN_DELAY_USEC にリセットします。

    • 5. **リセット**: status->spins を 0 に戻し、次回の試行に備えます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c
  • src/backend/storage/lmgr/lwlock.c
  • src/backend/storage/lmgr/s_lock.c
  • **呼び出し先**:
  • s_lock_stuck
  • pgstat_report_wait_start / pgstat_report_wait_end
  • pg_usleep
  • pg_prng_double

関連データ構造

  • SpinDelayStatus: スピンロックの試行状態を保持する構造体。

失敗モードと境界条件

  • **スタック検出**: status->delays が NUM_DELAYS を超えると、ロック取得不能と判断し s_lock_stuck によりプロセスを終了（PANIC）させます。
  • **遅延時間**: MAX_DELAY_USEC を超えると最小値にラップアラウンドします。

保守担当者向けメモ

  • 本関数は S_LOCK_TEST マクロが定義されている場合、標準出力に * を出力するデバッグ挙動を含みます。
  • 待機イベントの報告は pgstat を介して行われますが、より詳細な粒度での報告が必要な場合は将来的な拡張が検討可能です。
