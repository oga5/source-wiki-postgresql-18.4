ginRedoInsert

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginxlog.c:346

目的

GINインデックスの挿入操作に関するWALレコードをリプレイし、インデックスページの状態を復元します。

シグネチャまたは形状

static void ginRedoInsert(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）
  • **出力**: なし（副作用としてバッファの状態を更新）

前提条件と不変条件

  • record は ginxlogInsert 構造体を含む有効なWALレコードであること。
  • 非リーフページへの挿入の場合、ginRedoClearIncompleteSplit を呼び出して分割完了処理を行うこと。
  • ページタイプ（データページかエントリページか）に応じて、適切な挿入関数（ginRedoInsertData または ginRedoInsertEntry）が呼び出されること。

アルゴリズムまたは振る舞い

1. WALレコードから ginxlogInsert データを取得します。 2. 非リーフページへの挿入であれば、ginRedoClearIncompleteSplit を呼び出し、子ページの不完全な分割フラグをクリアします。 3. XLogReadBufferForRedo を使用して対象ページを読み込みます。 4. BLK_NEEDS_REDO が返された場合：

  • ページがデータページであれば ginRedoInsertData を、エントリページであれば ginRedoInsertEntry を呼び出し、ペイロードを挿入します。
  • ページのLSNを更新し、MarkBufferDirty でダーティ状態にします。

    • 5. 最後にバッファをアンロックおよび解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: gin_redo（src/backend/access/gin/ginxlog.c:744）
  • **呼び出し先**:
  • ginRedoClearIncompleteSplit
  • XLogReadBufferForRedo
  • ginRedoInsertData
  • ginRedoInsertEntry
  • PageSetLSN
  • MarkBufferDirty
  • UnlockReleaseBuffer

関連データ構造

  • XLogReaderState: WALレコードの読み取り状態を保持。
  • ginxlogInsert: 挿入操作のWALデータ構造。
  • Buffer: インデックスページを保持するバッファ。

失敗モードと境界条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返さない場合、ページ更新はスキップされます。
  • ページタイプ（GIN_INSERT_ISDATA フラグ）と実際のページ内容が一致しない場合、Assert により停止する可能性があります。

保守担当者向けメモ

  • 本関数はGINインデックスのWALリプレイにおける主要な挿入ロジックです。
  • leftChildBlkno に関するコードは現在 NOT_USED マクロで無効化されています。
  • ページタイプに応じた分岐（ginRedoInsertData vs ginRedoInsertEntry）が正しく行われているか確認してください。
