SendBackupManifest

定義

  • 種別: function
  • 定義箇所: src/backend/backup/backup_manifest.c:316

目的

バックアップ・マニフェストの構築を完了させ、マニフェスト全体の整合性を検証するためのSHA-256チェックサムを付与した上で、bbsink インターフェースを通じてクライアントへ送信する。

シグネチャまたは形状

void SendBackupManifest(backup_manifest_info *manifest, bbsink *sink)

入力と出力

  • **入力**:
  • manifest: マニフェスト生成の状態を保持する backup_manifest_info 構造体。
  • sink: マニフェストデータの送信先となる bbsink オブジェクト。
  • **出力**: なし（副作用として sink を介したストリーム送信が行われる）。

前提条件と不変条件

  • マニフェスト生成が有効であること（IsManifestEnabled が真を返すこと）。
  • マニフェスト全体のチェックサム計算には、ファイルごとのアルゴリズム設定に関わらず、常にSHA-256が使用される。

アルゴリズムまたは振る舞い

1. IsManifestEnabled を確認し、無効であれば即座に終了する。 2. manifest->still_checksumming を false に設定し、ハッシュ計算を終了する。 3. pg_cryptohash_final を呼び出し、蓄積されたデータからSHA-256ダイジェストを算出する。 4. AppendStringToManifest を使用して、JSON形式の末尾に "Manifest-Checksum": "<SHA-256値>" を追記する。 5. JSON構造を閉じるための } と改行を追記する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c:650
  • **呼び出し先**:
  • IsManifestEnabled
  • pg_cryptohash_final
  • AppendStringToManifest
  • hex_encode
  • elog (エラー発生時)

関連データ構造

  • backup_manifest_info: マニフェスト生成のコンテキスト（ハッシュ計算状態を含む）。
  • bbsink: バックアップデータの送信先抽象化レイヤー。

失敗モードと境界条件

  • pg_cryptohash_final が失敗した場合、elog(ERROR) により処理が中断される。
  • マニフェストが無効な場合、何も送信せずに正常終了する。

保守担当者向けメモ

  • 本関数はマニフェストの最終的なJSON構造を確定させる役割を持つ。JSONの構文（カンマや閉じ括弧）は AppendStringToManifest を介して手動で制御されているため、修正時はJSON構造の整合性に注意すること。
  • マニフェスト全体のチェックサムアルゴリズムを将来的に変更する場合、クライアント側での読み取り順序やアルゴリズム判別ロジックの調整が必要となる。
