SPELL

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/dicts/spell.h:61

目的

辞書ファイルから読み込まれた単語の生データおよびその属性情報を保持するための構造体です。辞書構築プロセスにおいて、単語とそれに関連する接辞（affix）情報を管理します。

シグネチャまたは形状

typedef struct spell_struct
{
    union {
        const char *flag;
        struct {
            int affix;
            int len;
        } d;
    } p;
    char word[FLEXIBLE_ARRAY_MEMBER];
} SPELL;

入力と出力

  • **入力**: 辞書ファイル読み込み時に word 文字列と flag 文字列が格納されます。
  • **出力**: 構築フェーズの最適化（NISortDictionary）を経て、p.d 構造体（接辞インデックスと単語長）が利用可能になります。

前提条件と不変条件

  • FLEXIBLE_ARRAY_MEMBER を使用しているため、メモリ確保時には SPELLHDRSZ マクロを用いたサイズ計算が必須です。
  • 辞書構築のフェーズにより union p の解釈が異なります。NIImportDictionary() 実行直後は flag が有効ですが、NISortDictionary() 実行後は d が使用されます。

アルゴリズムまたは振る舞い

1. **構築初期**: NIImportDictionary により、単語とフラグ文字列が読み込まれます。 2. **ソート・最適化**: NISortDictionary を経て、単語リストがソートされます。この際、cmpspell 関数等を用いて word や flag を比較します。 3. **変換**: mkSPNode 等の処理において、flag 文字列から affix インデックスへの変換が行われ、p.d が設定されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/spell.c 内の辞書構築ロジック（NIImportDictionary, NISortDictionary, NIFinishBuild 等）から参照・操作されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • IspellDict: 辞書全体の状態を保持し、SPELL の配列を管理します。
  • SPNode: SPELL から変換された Trie 構造のノード。

失敗モードと境界条件

  • メモリ確保失敗: tmpalloc 等でのメモリ不足時にエラーが発生します。
  • 構築フェーズの不整合: NISortDictionary 実行前に p.d を参照すると、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • union p の切り替えタイミングは辞書構築のライフサイクルに厳密に依存しています。コード修正時は、現在が「インポートフェーズ」か「ソート済みフェーズ」かを必ず確認してください。
  • SPELLHDRSZ マクロは offsetof(SPELL, word) として定義されており、メモリレイアウト操作時に活用してください。
