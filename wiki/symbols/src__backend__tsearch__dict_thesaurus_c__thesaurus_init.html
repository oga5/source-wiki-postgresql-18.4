thesaurus_init

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/dict_thesaurus.c:595

目的

シソーラス辞書の初期化およびコンパイルを行う。設定ファイルからルールを読み込み、サブ辞書と連携して検索可能な内部構造を構築する。

シグネチャまたは形状

Datum thesaurus_init(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して List *dictoptions を受け取る。これには dictfile（設定ファイルパス）と dictionary（サブ辞書名）が含まれる必要がある。
  • **出力**: 初期化された DictThesaurus 構造体へのポインタを Datum 型として返す。

前提条件と不変条件

  • dictoptions に dictfile と dictionary の両方が指定されていること。
  • dictfile および dictionary パラメータはそれぞれ1回のみ指定可能。
  • subdictname から取得されるサブ辞書がシステム内に存在すること。

アルゴリズムまたは振る舞い

1. DictThesaurus 構造体をメモリ確保する。 2. dictoptions を走査し、thesaurusRead で設定ファイルを読み込む。 3. 指定されたサブ辞書名を stringToQualifiedNameList で解析し、get_ts_dict_oid および lookup_ts_dictionary_cache を用いてサブ辞書のキャッシュを取得する。 4. compileTheLexeme および compileTheSubstitute を呼び出し、読み込んだルールを最適化・コンパイルする。 5. 構築した構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は明示されていない（PostgreSQLの辞書初期化インターフェースから呼び出される）。
  • **呼び出し先**: thesaurusRead, compileTheLexeme, compileTheSubstitute, stringToQualifiedNameList, get_ts_dict_oid, lookup_ts_dictionary_cache。

関連データ構造

  • DictThesaurus: シソーラス辞書のメイン構造体。
  • DefElem: 設定オプションを保持する構造体。

失敗モードと境界条件

  • dictfile または dictionary パラメータが欠落している場合、エラーを報告する。
  • 同一パラメータが複数回指定された場合、エラーを報告する。
  • 不明なパラメータが指定された場合、エラーを報告する。

保守担当者向けメモ

  • 本関数は辞書の初期化時に一度だけ実行される。
  • compileTheLexeme 等のコンパイル処理はサブ辞書に依存するため、サブ辞書のロード順序やキャッシュ状態に注意すること。
