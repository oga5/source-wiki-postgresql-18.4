ECPGset_desc_header

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/descriptor.c:580

目的

指定された名前の SQL 記述子（Descriptor）に対して、ヘッダー情報である項目数（count）を設定します。

シグネチャまたは形状

bool ECPGset_desc_header(int lineno, const char *desc_name, int count)

入力と出力

  • **入力**:
  • lineno: エラー報告に使用される行番号。
  • desc_name: 操作対象の記述子名。
  • count: 設定する項目数。
  • **出力**:
  • bool: 成功した場合は true、指定された名前の記述子が見つからない場合は false を返します。

前提条件と不変条件

  • 記述子は事前に ECPGallocate_desc 等で割り当てられている必要があります。
  • スレッドローカルストレージにより、呼び出し元のスレッドに紐付いた記述子リストから desc_name を検索します。

アルゴリズムまたは振る舞い

1. ecpg_find_desc を呼び出し、現在のスレッドの記述子リストから desc_name に一致する記述子構造体を取得します。 2. 記述子が見つからない場合、false を返して終了します。 3. 記述子が見つかった場合、その構造体の count メンバに引数 count の値を代入します。 4. true を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPG プリプロセッサが生成する C コード（src/interfaces/ecpg/preproc/descriptor.c 等）から呼び出されます。
  • **呼び出し先**: ecpg_find_desc（記述子の検索に使用）。

関連データ構造

  • struct descriptor: 記述子の状態を保持する構造体。count メンバを含みます。

失敗モードと境界条件

  • 指定した desc_name が存在しない場合、関数は false を返し、記述子の更新は行われません。

保守担当者向けメモ

  • 本関数は SET DESCRIPTOR ... HEADER 文に対応しています。
  • 記述子の検索はスレッドセーフに行われますが、desc_name の一意性はスレッド単位で管理されている点に注意してください。
