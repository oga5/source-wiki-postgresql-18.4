AdjustTimestampForTypmod

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/timestamp.c:367

目的

指定された型修飾子（typmod）に基づき、Timestamp 値の精度を丸める。

シグネチャまたは形状

bool AdjustTimestampForTypmod(Timestamp *time, int32 typmod, Node *escontext)

入力と出力

  • **入力**:
  • time: 変更対象の Timestamp へのポインタ。
  • typmod: ターゲットとなる精度（0〜6の範囲）。
  • escontext: エラー報告用コンテキスト（Node *）。
  • **出力**:
  • 成功時は true を返し、time が指す値が丸められる。
  • typmod が範囲外の場合は false を返す。

前提条件と不変条件

  • Timestamp はマイクロ秒単位の int64 として扱われる。
  • TIMESTAMP_NOT_FINITE(*time) が真である場合（無限大など）、値は変更されない。
  • typmod が -1（指定なし）または MAX_TIMESTAMP_PRECISION（最大精度）の場合、値は変更されない。

アルゴリズムまたは振る舞い

1. typmod が有効範囲（0〜6）内にあるか検証し、範囲外であれば escontext を通じてエラーを報告する。 2. TimestampScales および TimestampOffsets 配列を使用して、指定された精度まで値を丸める。 3. 値が正またはゼロの場合、*time = ((*time + offset) / scale) * scale の計算を行う。 4. 値が負の場合、符号を反転させて同様の丸め処理を行い、最後に符号を戻す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/timestamp.c 内の各関数（timestamp_in 等）。
  • src/backend/utils/adt/formatting.c（to_timestamp 等）。
  • src/backend/utils/adt/jsonpath_exec.c。
  • src/backend/utils/adt/date.c。
  • **呼び出し先**:
  • ereturn マクロ（エラー発生時）。

関連データ構造

  • Timestamp: int64 型のエイリアス。
  • MAX_TIMESTAMP_PRECISION: 6（定数）。
  • TimestampScales / TimestampOffsets: 精度変換用の静的配列。

失敗モードと境界条件

  • typmod が 0 未満または MAX_TIMESTAMP_PRECISION を超える場合、エラーが発生する。
  • *time が infinity または -infinity の場合、丸め処理はスキップされる。

保守担当者向けメモ

  • 本関数は date.c 等の他ファイルでも同様のロジックが使用されている。修正時は重複箇所への影響を考慮すること。
  • Timestamp の内部表現はマイクロ秒単位であるため、typmod による精度調整は 10^n 単位の除算・乗算で行われる。
