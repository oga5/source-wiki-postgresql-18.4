astreamer_gzip_decompressor_finalize

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_gzip.c:340

目的

gzip解凍処理の終了時に、バッファ内に残存するデータを後続のストリーマーへ転送し、パイプラインの終了処理を完了させる。

シグネチャまたは形状

static void astreamer_gzip_decompressor_finalize(astreamer *streamer)

入力と出力

  • **入力**: astreamer *streamer（astreamer_gzip_decompressor 構造体としてキャストして使用）
  • **出力**: なし（副作用として後続ストリーマーへのデータ転送および終了処理を実行）

前提条件と不変条件

  • 引数 streamer は astreamer_gzip_decompressor 型のインスタンスであること。
  • mystreamer->base.bbs_next が有効な後続ストリーマーを指していること。

アルゴリズムまたは振る舞い

1. 引数の streamer を astreamer_gzip_decompressor 型にキャストする。 2. bytes_written が 0 より大きい場合、バッファ（bbs_buffer.data）に残っているデータを astreamer_content を介して後続のストリーマーへ転送する。 3. 最後に astreamer_finalize を呼び出し、後続のストリーマーの終了処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer インターフェースの finalize メソッドとして、ストリーミング処理の終了時に呼び出される（ソースコード内の構造体初期化箇所で参照）。
  • **呼び出し先**:
  • astreamer_content（後続ストリーマーへのデータ転送）
  • astreamer_finalize（後続ストリーマーの終了処理）

関連データ構造

  • astreamer_gzip_decompressor: 解凍器の状態を保持する構造体。bbs_buffer（バッファ）および bbs_next（後続ストリーマー）を含む。

失敗モードと境界条件

  • bytes_written が 0 の場合、バッファ転送処理はスキップされる。
  • 後続ストリーマーが存在しない場合、bbs_next へのアクセスにより未定義の動作が発生する可能性がある（呼び出し側の構成に依存）。

保守担当者向けメモ

  • 本関数は astreamer インターフェースの規約に従い、パイプラインの終端処理を担う。
  • zlib の解凍状態（z_stream）の解放は、本関数ではなく astreamer_gzip_decompressor_free で行われる点に注意すること。
