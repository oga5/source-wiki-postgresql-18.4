r_standard_suffix

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_french.c:676

目的

フランス語のステミング処理において、単語末尾の標準的な接尾辞を識別し、ルールに基づいて削除または置換を行う。

シグネチャまたは形状

static int r_standard_suffix(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z（ステミング環境の状態を保持する構造体）
  • 出力: int（成功時は 1、失敗または処理対象外の場合は 0、エラー時は負の値を返す）

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • 内部で r_R1、r_R2、r_RV などの領域判定関数を呼び出すため、事前に r_mark_regions が実行され、領域が正しく設定されている必要がある。

アルゴリズムまたは振る舞い

1. find_among_b を使用し、単語末尾が配列 a_4 に含まれる 43 種類の接尾辞のいずれかに一致するかを後方検索する。 2. 一致した場合、among_var の値に応じて以下の処理を行う：

  • **ケース 1-5**: r_R2 領域内であることを確認し、接尾辞を削除するか、特定の文字列に置換する。
  • **ケース 6**: r_RV 領域内であることを確認して削除した後、さらに a_2 配列に基づく追加の接尾辞マッチングを行い、条件に応じて削除や置換を再帰的に試みる。
  • **ケース 7**: r_R2 領域内であることを確認して削除した後、a_3 配列に基づく追加マッチングを行う。

    • 3. 各ステップで slice_del（削除）または slice_from_s（置換）を実行し、バッファを更新する。

呼び出し元と呼び出し先

  • 呼び出し元: french_UTF_8_stem（直接の言及はないが、ファイル概要の制御フローより推論）。
  • 呼び出し先: find_among_b, r_R1, r_R2, r_RV, slice_del, slice_from_s, eq_s_b。
  • 注: 他の言語ファイル（stem_ISO_8859_1_catalan.c 等）にも同名のシンボルが存在するが、本ページは stem_UTF_8_french.c の実装を対象としている。

関連データ構造

  • struct SN_env: ステミング対象の文字列バッファ、カーソル位置、領域境界を保持する。
  • a_2, a_3, a_4: 接尾辞マッチングに使用される among 構造体の配列。

失敗モードと境界条件

  • find_among_b が一致を見つけられない場合、0 を返して終了する。
  • r_R1, r_R2, r_RV のチェックで領域外と判定された場合、処理は中断される。
  • メモリ操作関数（slice_del 等）が負の値を返した場合、そのエラーコードをそのまま伝播させる。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによって自動生成されたコードである。ロジックの変更が必要な場合は、直接編集せず、対応する .sbl ファイルを修正すること。
  • デバッグ時は z->c（カーソル位置）と z->l（文字列長）の推移を追跡し、among_var の分岐が意図通りか確認すること。
