PROCLOCK_PRINT

定義

  • 種別: inline static function
  • 定義箇所: src/backend/storage/lmgr/lock.c:393

目的

PROCLOCK 構造体の状態をデバッグログとして出力する。

シグネチャまたは形状

void PROCLOCK_PRINT(const char *where, const PROCLOCK *proclockP)

入力と出力

  • **入力**:
  • where: ログ出力時に付与する識別用文字列。
  • proclockP: 出力対象の PROCLOCK 構造体へのポインタ。
  • **出力**: なし（elog を介してログを出力する）。

前提条件と不変条件

  • proclockP は有効な PROCLOCK 構造体を指している必要がある。
  • LOCK_DEBUG_ENABLED マクロによる判定が真の場合のみ出力が行われる。

アルゴリズムまたは振る舞い

1. proclockP->tag.myLock のタグ情報に基づき、LOCK_DEBUG_ENABLED でデバッグ出力が有効か確認する。 2. 有効な場合、elog(LOG, ...) を呼び出し、以下の情報を出力する:

  • 呼び出し元で指定された where 文字列
  • proclockP のアドレス
  • 関連する LOCK 構造体のアドレス
  • ロックメソッド識別子
  • 関連する PGPROC 構造体のアドレス
  • holdMask（保持しているロックモードのビットマスク）

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/lmgr/lock.c 内の各関数（LockHasWaiters, LockAcquire, LockCheckConflicts など）。
  • **呼び出し先**: elog（PostgreSQL のログ出力関数）。

関連データ構造

  • PROCLOCK: プロセスとロックオブジェクトのペアを管理する構造体。
  • LOCK: ロック対象オブジェクトの状態を管理する構造体。
  • PGPROC: ロックを保持または待機しているプロセス情報。

失敗モードと境界条件

  • LOCK_DEBUG_ENABLED が偽を返す場合、何も出力せず終了する。
  • proclockP が NULL の場合、proclockP->tag.myLock へのアクセスによりセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は LOCK_DEBUG が有効な環境でのデバッグ用である。
  • src/backend/storage/lmgr/lock.c:406 には、デバッグが無効な場合に何もしない（((void) 0)）マクロ定義が存在する。
