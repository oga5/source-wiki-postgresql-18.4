SyncRepGetNthLatestSyncRecPtr

定義

  • 種別: function
  • 定義箇所: src/backend/replication/syncrep.c:692

目的

複数のスタンバイサーバーから報告されたWALの書き込み・フラッシュ・適用位置（LSN）のリストから、指定された順位（N番目）の最新位置を算出する。

シグネチャまたは形状

static void SyncRepGetNthLatestSyncRecPtr(
    XLogRecPtr *writePtr,
    XLogRecPtr *flushPtr,
    XLogRecPtr *applyPtr,
    SyncRepStandbyData *sync_standbys,
    int num_standbys,
    uint8 nth
)

入力と出力

  • **入力**:
  • sync_standbys: スタンバイサーバーの進捗データ配列。
  • num_standbys: 配列の要素数。
  • nth: 取得対象の順位（1以上、num_standbys以下）。
  • **出力**:
  • writePtr, flushPtr, applyPtr: それぞれの項目におけるN番目に新しいLSNが格納される。

前提条件と不変条件

  • nth > 0 かつ nth <= num_standbys であること（アサーションにより検証）。
  • sync_standbys 配列には有効な進捗データが格納されていること。

アルゴリズムまたは振る舞い

1. 各スタンバイのWrite/Flush/Apply位置を個別の配列にコピーする。 2. qsort と比較関数 cmp_lsn を使用し、各配列を降順（新しい順）にソートする。 3. ソート済み配列の nth - 1 番目の要素を抽出し、出力ポインタに書き込む。 4. 確保した一時メモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の SyncRepGetSyncRecPtr から呼び出される（ソースコード内の直接的な記述に基づく）。
  • **呼び出し先**: palloc, qsort, pfree を使用。

関連データ構造

  • SyncRepStandbyData: 各スタンバイの進捗（write, flush, apply）を保持する構造体。
  • XLogRecPtr: WALのログ位置を示すデータ型。

失敗モードと境界条件

  • nth が範囲外の場合、アサーション違反によりプロセスが異常終了する可能性がある。
  • メモリ不足時、palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数はクォーラムベースの同期レプリケーションにおいて、特定の順位のスタンバイの進捗を特定するために使用される。
  • ソート処理に qsort を使用しているため、スタンバイ数が多い場合は計算量に注意が必要。
