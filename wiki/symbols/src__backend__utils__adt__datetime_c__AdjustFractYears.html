AdjustFractYears

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/datetime.c:610

目的

浮動小数点数で表現された年単位の端数（frac）を、指定されたスケール（scale）に基づいて月数に変換し、pg_itm_in 構造体の月フィールドに加算します。

シグネチャまたは形状

static bool AdjustFractYears(double frac, int scale, struct pg_itm_in *itm_in)

入力と出力

  • **入力**:
  • frac: 加算対象の年単位の端数（double）。
  • scale: frac をスケーリングするための倍率（int）。
  • itm_in: 更新対象の pg_itm_in 構造体へのポインタ。
  • **出力**:
  • 戻り値: 加算処理がオーバーフローせずに成功した場合は true、オーバーフローした場合は false を返します。

前提条件と不変条件

  • abs(frac) < 1 であることを前提としています。このため、scale が一般的な値であればオーバーフローは発生しないと想定されています。

アルゴリズムまたは振る舞い

1. frac * scale * MONTHS_PER_YEAR を計算し、rint() 関数で最も近い整数に丸めて extra_months を算出します。 2. pg_add_s32_overflow() を使用して、itm_in->tm_mon に extra_months を加算します。 3. 加算結果が 32 ビット符号付き整数の範囲内に収まるかを確認し、その成否を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/datetime.c 内の複数箇所（行 3749, 3756, 3763, 3770, 3997, 4038 など）から呼び出されています。
  • **呼び出し先**: rint()（標準数学ライブラリ）および pg_add_s32_overflow()（PostgreSQL 内部のオーバーフローチェックマクロ）を呼び出します。

関連データ構造

  • struct pg_itm_in: 日付・時刻の構成要素を保持する内部構造体。本関数はこの構造体内の tm_mon フィールドを直接操作します。

失敗モードと境界条件

  • itm_in->tm_mon への加算結果が 32 ビット符号付き整数の最大値または最小値を超える場合、pg_add_s32_overflow により false が返されます。

保守担当者向けメモ

  • 本関数は static であり、主に datetime.c 内の複雑な日付計算処理の一部として利用されます。
  • MONTHS_PER_YEAR 定数に依存しているため、年と月の換算ロジックに変更がある場合は注意してください。
