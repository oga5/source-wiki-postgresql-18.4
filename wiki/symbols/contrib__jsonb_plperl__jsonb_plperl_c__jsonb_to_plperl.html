jsonb_to_plperl

定義

  • 種別: function
  • 定義箇所: contrib/jsonb_plperl/jsonb_plperl.c:275

目的

PostgreSQL の jsonb 型データを Perl のデータ構造（スカラー、配列、ハッシュ）へ変換する SQL 呼び出し可能なエントリポイントです。

シグネチャまたは形状

Datum jsonb_to_plperl(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される jsonb 型の PostgreSQL データ。
  • **出力**: PointerGetDatum により変換された Perl オブジェクト（SV*）を Datum 型として返却します。

前提条件と不変条件

  • dTHX マクロにより、Perl インタプリタのコンテキストが初期化されている必要があります。
  • 入力データは有効な Jsonb 構造体である必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_JSONB_P(0) を使用して、引数から Jsonb 構造体を取得します。 2. Jsonb_to_SV 関数を呼び出し、Jsonb のルートノードから Perl の SV (Scalar Value) への再帰的な変換を実行します。 3. 変換された SV を PointerGetDatum でラップし、PostgreSQL が認識可能な形式で返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェースから呼び出されます（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: Jsonb_to_SV を呼び出し、再帰的に変換処理を委譲します。

関連データ構造

  • Jsonb: PostgreSQL 側の JSON バイナリ表現。
  • SV (Scalar Value): Perl の汎用データ構造。

失敗モードと境界条件

  • メモリ管理: PostgreSQL の palloc と Perl のメモリ管理が混在するため、変換プロセスにおけるメモリリークに注意が必要です。
  • 再帰の深さ: Jsonb が深くネストしている場合、再帰的な変換処理によりスタックオーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 本関数は jsonb から Perl への変換の最上位エントリポイントです。詳細な変換ロジックは Jsonb_to_SV に依存しています。
  • Perl の UV (Unsigned Integer) 型が関与する場合、文字列経由で numeric 型として扱われる仕様に留意してください。
