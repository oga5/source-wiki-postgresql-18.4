jsonb_in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb.c:72

目的

入力された文字列（C-string）を PostgreSQL の内部バイナリ形式である Jsonb 型の Datum に変換する。

シグネチャまたは形状

Datum jsonb_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る cstring 型の JSON 文字列。
  • **出力**: Datum 型として返される Jsonb 構造体へのポインタ。

前提条件と不変条件

  • 入力文字列は有効な JSON 形式である必要がある。
  • fcinfo->context を介してメモリコンテキストが提供されていること。

アルゴリズムまたは振る舞い

1. PG_GETARG_CSTRING(0) を使用して、入力引数から JSON 文字列を取得する。 2. jsonb_from_cstring を呼び出し、文字列の長さ、パースの可否（false）、および現在のメモリコンテキストを渡して変換を実行する。 3. jsonb_from_cstring の戻り値をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c
  • src/backend/parser/parse_expr.c
  • src/backend/utils/adt/jsonfuncs.c
  • src/backend/utils/adt/jsonpath_exec.c
  • **呼び出し先**:
  • jsonb_from_cstring (同ファイル内)

関連データ構造

  • Jsonb: jsonb 型の内部バイナリ表現。
  • Datum: PostgreSQL の汎用データ型表現。
  • PG_FUNCTION_ARGS: SQL 関数呼び出し規約のための引数構造体。

失敗モードと境界条件

  • 入力文字列が不正な JSON 形式である場合、jsonb_from_cstring 内でエラーが発生し、トランザクションがアボートされる可能性がある。

保守担当者向けメモ

  • 本関数は SQL インターフェースの入口であり、実際のパースロジックは jsonb_from_cstring に委譲されている。
  • パース処理は JsonLexContext を利用して行われるため、詳細な構文解析ロジックについては jsonb_from_cstring および関連する JsonSemAction コールバックを参照すること。
