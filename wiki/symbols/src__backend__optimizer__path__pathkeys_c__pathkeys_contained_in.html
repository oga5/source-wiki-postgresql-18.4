pathkeys_contained_in

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/pathkeys.c:342

目的

あるパスキーリスト（keys1）が、別のパスキーリスト（keys2）によって提供される順序に包含されているか（あるいは等価であるか）を判定します。

シグネチャまたは形状

bool pathkeys_contained_in(List *keys1, List *keys2)

入力と出力

  • **入力**:
  • keys1: 比較対象となるパスキーのリスト。
  • keys2: 基準となるパスキーのリスト。
  • **出力**:
  • bool: keys1 が keys2 と等価、または keys2 が提供する順序が keys1 を包含している場合に true を返します。

前提条件と不変条件

  • 入力される keys1 および keys2 は、正規化された PathKey のリストである必要があります。
  • compare_pathkeys の結果に基づき、包含関係を判定します。

アルゴリズムまたは振る舞い

1. compare_pathkeys(keys1, keys2) を呼び出し、2 つのリストの順序関係を比較します。 2. 比較結果が PATHKEYS_EQUAL（両者が等価）または PATHKEYS_BETTER2（keys2 が keys1 よりも詳細な順序を提供している、あるいは keys1 を包含している）である場合、true を返します。 3. それ以外の場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/foreign/foreign.c
  • src/backend/optimizer/path/allpaths.c
  • src/backend/optimizer/path/costsize.c
  • src/backend/optimizer/path/joinpath.c
  • **呼び出し先**:
  • compare_pathkeys

関連データ構造

  • List: PostgreSQL の汎用リスト構造。
  • PathKey: 順序を表現するデータ構造。

失敗モードと境界条件

  • 空のリストが渡された場合の挙動は compare_pathkeys の仕様に依存します。
  • keys1 が keys2 よりも順序が厳しい（PATHKEYS_BETTER1）場合や、順序が全く異なる（PATHKEYS_DIFFERENT）場合は false となります。

保守担当者向けメモ

  • 本関数は、プランナが特定のパスが要求される順序を満たしているかを検証する際の標準的な手段です。
  • 包含関係の判定ロジックを変更する場合は、compare_pathkeys の戻り値の定義と整合性を取る必要があります。
