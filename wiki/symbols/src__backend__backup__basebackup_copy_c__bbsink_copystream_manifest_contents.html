bbsink_copystream_manifest_contents

定義

  • **種別**: function
  • **定義箇所**: src/backend/backup/basebackup_copy.c:272

目的

バックアップマニフェストのコンテンツを、COPYプロトコルを通じてクライアントへ送信する。

シグネチャまたは形状

void bbsink_copystream_manifest_contents(bbsink *sink, size_t len)

入力と出力

  • **入力**:
  • sink: bbsink 構造体へのポインタ。実際には bbsink_copystream としてキャストされる。
  • len: 送信するマニフェストデータのバイト長。
  • **出力**: なし（副作用としてネットワークへメッセージを送信）。

前提条件と不変条件

  • sink は bbsink_copystream 型のインスタンスであること。
  • mysink->msgbuffer には、送信対象のデータが格納されていること。
  • len は msgbuffer に格納された有効なデータ長を示すこと。

アルゴリズムまたは振る舞い

1. sink を bbsink_copystream 型にキャストする。 2. mysink->send_to_client が真であるかを確認する。 3. 真の場合、pq_putmessage を使用して、タイプバイト 'd' を先頭に付与したメッセージを送信する。 4. 送信サイズは、タイプバイト分を含めた len + 1 となる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup_copy.c 内の bbsink_copystream_ops 構造体を通じて、バックアップ処理のメインループから間接的に呼び出される。
  • **呼び出し先**: pq_putmessage（libpqの関数）。

関連データ構造

  • bbsink_copystream: 本関数が操作するメインの構造体。msgbuffer や send_to_client フラグを保持する。
  • bbsink: 基底となるシンク構造体。

失敗モードと境界条件

  • send_to_client が偽の場合、データ送信はスキップされる。
  • pq_putmessage が失敗した場合の挙動は、libpqの標準的なエラー処理に依存する。

保守担当者向けメモ

  • 本関数は、マニフェストデータを CopyData メッセージとして送信する役割を担う。
  • タイプバイトとして 'd' がハードコードされている点に注意。これはマニフェストデータも通常のデータストリームの一部として扱われることを示唆している。
  • len + 1 の計算は、msgbuffer の先頭にタイプバイトを配置する設計に基づいている。
