OffsetVarNodes_context

定義

  • 種別: struct
  • 定義箇所: src/backend/rewrite/rewriteManip.c:381

目的

OffsetVarNodes 関数がクエリツリーを走査する際、現在の走査状態（オフセット量とネストレベル）を保持するために使用されるコンテキスト構造体です。

シグネチャまたは形状

typedef struct
{
	int			offset;
	int			sublevels_up;
} OffsetVarNodes_context;

入力と出力

本構造体はデータ保持用であり、関数ではありません。

  • offset: Var ノードの varno に加算する値。
  • sublevels_up: 現在の走査対象が何階層上のクエリレベルにあるかを示す値。

前提条件と不変条件

  • sublevels_up は、Query ノードを通過するたびに適切にインクリメントおよびデクリメントされる必要があります。
  • 本構造体は OffsetVarNodes_walker によって参照されます。

アルゴリズムまたは振る舞い

OffsetVarNodes_walker がツリーを再帰的に走査する際、この構造体を通じて現在の sublevels_up を追跡し、対象となる Var ノードの varno を offset 分だけシフトさせるための情報を共有します。

呼び出し元と呼び出し先

  • **呼び出し元**: OffsetVarNodes 関数（src/backend/rewrite/rewriteManip.c:478）でインスタンス化されます。
  • **呼び出し先**: OffsetVarNodes_walker（src/backend/rewrite/rewriteManip.c:388）に引数として渡されます。

関連データ構造

  • OffsetVarNodes_walker: 本構造体を利用してツリーを走査する関数。
  • Query: 走査対象となるノードツリーのルート。

失敗モードと境界条件

  • 呼び出し側が sublevels_up の管理（サブクエリ突入時のインクリメント等）を誤った場合、意図しない階層の Var ノードが書き換えられる可能性があります。

保守担当者向けメモ

  • 本構造体はインプレース修正を行うための補助データです。ツリーの整合性を保つため、走査ロジック内での sublevels_up の更新ルールを厳守してください。
