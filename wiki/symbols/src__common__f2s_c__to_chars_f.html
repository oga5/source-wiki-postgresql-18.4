to_chars_f

定義

  • 種別: function
  • 定義箇所: src/common/f2s.c:439

目的

Ryuアルゴリズムによって計算された浮動小数点数の仮数と指数を、指定されたバッファへ最短の10進数文字列としてフォーマットする。

シグネチャまたは形状

static inline int to_chars_f(const floating_decimal_32 v, const uint32 olength, char *const result)

入力と出力

  • **入力**:
  • v: 変換対象の仮数と指数を保持する floating_decimal_32 構造体。
  • olength: 出力する数値の桁数。
  • result: 文字列を書き込むためのバッファ。
  • **出力**:
  • 戻り値: 書き込まれた文字列の総長（int）。

前提条件と不変条件

  • 呼び出し元は、負の符号（-）が必要な場合に既にバッファへ書き込み済みであること。
  • mantissa * 10^exponent が出力すべき値であること。
  • 指数が6以上の場合は科学的表記法へ切り替わるため、本関数内では考慮不要。

アルゴリズムまたは振る舞い

1. **初期化**: 指数と出力長に基づき、小数点位置を決定する。 2. **バッファ準備**: 指数に応じて、先行ゼロの挿入や小数点位置の調整を行う。 3. **数値展開**: DIGIT_TABLE を使用し、output（仮数）を4桁ずつ文字列バッファへ高速に書き出す。 4. **フォーマット調整**:

  • 指数が負の場合、0.000... 形式を構築。
  • 指数が正の場合、dddd.dddd 形式を構築し、必要に応じて memmove で小数点を挿入。

    • 5. **完了**: 最終的な文字列長を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/f2s.c 内の関数（float_to_shortest_decimal_bufn 等から間接的に参照される）。
  • **呼び出し先**: memcpy, memset, memmove（標準Cライブラリ）。

関連データ構造

  • floating_decimal_32: 変換対象の仮数と指数を保持する構造体。
  • DIGIT_TABLE: 数値から文字列への高速変換用テーブル。

失敗モードと境界条件

  • nexp（指数と長さの合計）が負の場合、Assert(nexp >= -3) により範囲が制限される。
  • exp が6以上の場合、本関数は科学的表記法を想定しないため、呼び出し元で制御する必要がある。
  • index == 1 の場合、nexp < 7 であることが保証される。

保守担当者向けメモ

  • DIGIT_TABLE を使用した変換は printf より高速だが、インデックス計算が複雑であるため、変更時は memcpy のオフセットに注意すること。
  • コンパイラの最適化を促進するため、memset や memcpy で固定長（8バイト等）を扱う箇所がある。
