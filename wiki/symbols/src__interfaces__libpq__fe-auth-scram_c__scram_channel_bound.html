scram_channel_bound

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth-scram.c:157

目的

SCRAM 認証セッションにおいて、チャネルバインディング（tls-server-end-point）が使用され、かつ認証が正常に完了しているかを判定する。

シグネチャまたは形状

bool scram_channel_bound(void *opaq)

入力と出力

  • **入力**: opaq (SCRAM 認証状態を保持する fe_scram_state 構造体へのポインタ)
  • **出力**: bool 型。条件を満たす場合は true、それ以外は false を返す。

前提条件と不変条件

  • opaq が NULL の場合、認証は行われていないとみなされ false を返す。
  • 認証状態が FE_SCRAM_FINISHED である必要がある。

アルゴリズムまたは振る舞い

1. 引数 opaq を fe_scram_state 型にキャストする。 2. state が NULL であれば false を返す。 3. 認証状態 (state->state) が FE_SCRAM_FINISHED でない場合、false を返す。 4. 使用されている SASL メカニズム名 (state->sasl_mechanism) が SCRAM_SHA_256_PLUS_NAME と一致するか比較する。 5. 上記すべての条件を満たす場合にのみ true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の参照情報に基づくと、src/interfaces/libpq/fe-auth-scram.c 内で定義および使用されている。具体的な外部からの呼び出し関係は提供されたソースからは特定できない。
  • **呼び出し先**: strcmp (標準ライブラリ関数) を使用してメカニズム名の比較を行う。

関連データ構造

  • fe_scram_state: 認証セッション中の状態を保持する構造体。
  • fe_scram_state_enum: 認証の進捗状態を定義する列挙型。

失敗モードと境界条件

  • opaq が NULL の場合、安全に false を返す。
  • 認証が完了していない、またはチャネルバインディングを伴わないメカニズム（SCRAM-SHA-256 等）が使用されている場合、false を返す。

保守担当者向けメモ

  • 本関数は、チャネルバインディングの有効性を確認するための述語として機能する。
  • SCRAM_SHA_256_PLUS_NAME は common/scram-common.h で定義されている定数である。
