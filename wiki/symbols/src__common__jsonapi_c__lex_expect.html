lex_expect

定義

  • 種別: function
  • 定義箇所: src/common/jsonapi.c:316

目的

JSON解析において、現在のトークンが期待される型と一致するかを検証し、一致する場合は次のトークンへ進め、不一致の場合はエラーを報告する。

シグネチャまたは形状

static inline JsonParseErrorType lex_expect(JsonParseContext ctx, JsonLexContext *lex, JsonTokenType token)

入力と出力

  • **入力**:
  • ctx: 解析コンテキスト。
  • lex: 現在の字句解析状態を保持する JsonLexContext へのポインタ。
  • token: 期待される JsonTokenType。
  • **出力**:
  • 成功時: json_lex の戻り値（次のトークン読み込み結果）。
  • 失敗時: report_parse_error が返す JsonParseErrorType。

前提条件と不変条件

  • lex は有効な JsonLexContext を指している必要がある。
  • lex_peek が現在のトークンを正しく取得できる状態であること。

アルゴリズムまたは振る舞い

1. lex_peek(lex) を呼び出し、現在のトークンが引数 token と一致するか確認する。 2. 一致する場合、json_lex(lex) を呼び出してトークンを消費し、その結果を返す。 3. 一致しない場合、report_parse_error(ctx, lex) を呼び出し、解析エラーとして処理を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/jsonapi.c 内の各解析関数（pg_parse_json 等の内部処理）から、JSON構造の整合性チェックのために呼び出される。
  • **呼び出し先**:
  • lex_peek: 現在のトークン確認に使用。
  • json_lex: トークンの消費に使用。
  • report_parse_error: 不一致時のエラー報告に使用。

関連データ構造

  • JsonLexContext: 字句解析の状態を保持する構造体。
  • JsonTokenType: トークンの種類を定義する列挙型。
  • JsonParseErrorType: 解析エラーの種類を示す戻り値型。

失敗モードと境界条件

  • 現在のトークンが期待値と異なる場合、report_parse_error を経由してエラーが返される。
  • lex が指すバッファの終端や不正なJSON構造に遭遇した場合、lex_peek または json_lex の内部仕様に従いエラーが伝播する。

保守担当者向けメモ

  • 本関数は static inline で定義されており、頻繁に呼び出される解析ループ内でのオーバーヘッドを最小化している。
  • エラー発生時の詳細な診断情報は report_parse_error に依存しているため、エラーメッセージの変更が必要な場合はそちらを確認すること。
