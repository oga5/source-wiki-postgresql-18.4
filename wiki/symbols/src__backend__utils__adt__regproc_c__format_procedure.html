format_procedure

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/regproc.c:298

目的

指定された関数OIDに対応する、人間が読み取り可能な関数名（スキーマ修飾名を含む）の文字列を生成します。

シグネチャまたは形状

char *format_procedure(Oid procedure_oid)

入力と出力

  • **入力**: procedure_oid (Oid) - 情報を取得する対象関数のOID。
  • **出力**: char * - 関数名を表す文字列（例: schema.funcname）。呼び出し元でメモリを解放する必要があります。

前提条件と不変条件

  • procedure_oid はシステムカタログ pg_proc に存在する有効なOIDである必要があります。
  • 内部で format_procedure_extended を呼び出すため、その前提条件（システムキャッシュへのアクセス権等）に準拠します。

アルゴリズムまたは振る舞い

1. 引数として受け取った procedure_oid をそのまま format_procedure_extended 関数へ渡します。 2. format_procedure_extended に対してフラグ 0 を指定して呼び出し、標準的なフォーマットで文字列を取得して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blvalidate.c
  • src/backend/access/brin/brin_validate.c
  • src/backend/access/gin/ginvalidate.c
  • src/backend/access/gist/gistvalidate.c
  • src/backend/access/hash/hashvalidate.c

    • (主にインデックスアクセスメソッドのバリデーション処理で使用)

  • **呼び出し先**: format_procedure_extended

関連データ構造

  • Oid: PostgreSQLのシステムオブジェクト識別子。
  • pg_proc: 関数情報を保持するシステムカタログ。

失敗モードと境界条件

  • 指定された procedure_oid がシステムカタログに存在しない場合、format_procedure_extended の挙動に従い、通常はエラーが発生するか、OIDそのものを文字列として返す可能性があります。

保守担当者向けメモ

  • 本関数は format_procedure_extended(oid, 0) の単なるラッパーです。将来的にデフォルトのフォーマットを変更する必要がある場合は、format_procedure_extended の実装を確認してください。
