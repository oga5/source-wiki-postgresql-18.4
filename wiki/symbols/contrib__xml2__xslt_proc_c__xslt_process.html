xslt_process

定義

  • 種別: function
  • 定義箇所: contrib/xml2/xslt_proc.c:44

目的

SQL から XML ドキュメントと XSLT スタイルシートを受け取り、libxslt を用いて変換を実行し、その結果をテキストとして返す。

シグネチャまたは形状

Datum xslt_process(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • 第1引数: XML ドキュメント (text)
  • 第2引数: XSLT スタイルシート (text)
  • 第3引数 (オプション): パラメータ文字列 (text, key=value 形式)
  • **出力**: 変換結果のテキスト (text) または NULL

前提条件と不変条件

  • USE_LIBXSLT マクロが定義されている環境でビルドされていること。
  • libxslt のセキュリティ設定により、ファイルシステムへの読み書き、ディレクトリ作成、およびネットワークアクセスは厳格に禁止される。
  • PG_TRY/PG_CATCH ブロックにより、エラー発生時にも libxml/libxslt が確保したメモリが確実に解放されること。

アルゴリズムまたは振る舞い

1. **パラメータ解析**: 第3引数がある場合、parse_params を呼び出して libxslt 用の配列に変換する。 2. **初期化**: pgxml_parser_init でエラーハンドリングをセットアップする。 3. **解析**: xmlReadMemory を使用して XML ドキュメントとスタイルシートをメモリ上に展開する。 4. **変換準備**: xsltParseStylesheetDoc でスタイルシートをコンパイルし、xsltNewTransformContext で変換コンテキストを作成する。 5. **セキュリティ適用**: xsltSetSecurityPrefs を用いて外部リソースへのアクセスを禁止する。 6. **変換実行**: xsltApplyStylesheetUser で変換を行い、結果を文字列に変換する。 7. **クリーンアップ**: 正常終了・異常終了の双方において、確保したリソース（ドキュメント、コンテキスト、スタイルシート等）を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • parse_params (内部ヘルパー)
  • pgxml_parser_init / pg_xml_done (xpath.c 由来)
  • libxml2 / libxslt API (xmlReadMemory, xsltApplyStylesheetUser 等)

関連データ構造

  • PgXmlErrorContext: エラーハンドリング用コンテキスト。
  • xsltStylesheetPtr, xmlDocPtr, xsltSecurityPrefsPtr, xsltTransformContextPtr: libxslt/libxml が管理する内部構造体。

失敗モードと境界条件

  • USE_LIBXSLT が未定義の場合: ERRCODE_FEATURE_NOT_SUPPORTED エラーを発生させる。
  • XML 解析失敗時: ERRCODE_INVALID_XML_DOCUMENT エラーを発生させる。
  • スタイルシート解析失敗時: ERRCODE_INVALID_ARGUMENT_FOR_XQUERY エラーを発生させる。
  • セキュリティ設定失敗時: ereport(ERROR) を発生させる。
  • 変換結果が空の場合: 空文字列を返す。

保守担当者向けメモ

  • libxml/libxslt のメモリ管理は PostgreSQL の palloc と独立しているため、PG_CATCH ブロック内での解放処理を漏らさないこと。
  • resstat < 0 の場合に PG_RETURN_NULL() を返す挙動は、将来的にエラーを投げる形へ修正することが推奨される。
