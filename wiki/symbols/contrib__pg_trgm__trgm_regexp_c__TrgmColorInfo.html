TrgmColorInfo

定義

  • 種別: struct
  • 定義箇所: contrib/pg_trgm/trgm_regexp.c:261

目的

正規表現のNFA（非決定性有限オートマトン）における各「色（color）」の特性を保持するための構造体です。正規表現からトリグラムを抽出する際、各状態遷移がどのような文字構成を持つかを管理します。

シグネチャまたは形状

typedef struct
{
	bool		expandable;
	bool		containsNonWord;
	int			wordCharsCount;
	trgm_mb_char *wordChars;
} TrgmColorInfo;

入力と出力

  • **入力**: 正規表現の解析過程で、各色に対して wordChars（文字配列）やフラグが設定されます。
  • **出力**: グラフ構築やマッチング判定の際、特定の遷移がトリグラムとして展開可能か、非単語文字を含むか等の情報を後続のアルゴリズムへ提供します。

前提条件と不変条件

  • wordChars は trgm_mb_char 型の配列を指し、wordCharsCount によってその要素数が管理されます。
  • expandable が false の場合、その色はトリグラム展開の対象外として扱われる可能性があります。

アルゴリズムまたは振る舞い

  • 正規表現のコンパイル後、TrgmNFA 構造体の一部としてメモリが確保されます。
  • 各色に対して、正規表現の文字クラス情報から wordChars が抽出・格納されます。
  • グラフ走査時、この構造体を参照して、特定の遷移がトリグラムを生成できるかどうかが判定されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_trgm/trgm_regexp.c 内の正規表現解析ロジック（TrgmNFA の初期化および構築フェーズ）。
  • **呼び出し先**: 直接的な関数呼び出しではなく、TrgmNFA を介して printSourceNFA やグラフ構築・走査関数から参照されます。

関連データ構造

  • TrgmNFA: 本構造体を保持する親構造体。
  • trgm_mb_char: wordChars 配列の要素として使用されるマルチバイト文字表現。

失敗モードと境界条件

  • メモリ不足時: palloc0 によるメモリ確保に失敗する可能性があります。
  • 複雑な正規表現: wordChars が非常に多くなる場合、解析コストが増大するため、定数による制限が適用される場合があります。

保守担当者向けメモ

  • 本構造体は正規表現からトリグラムを抽出する際のメタデータとして機能します。
  • wordChars のメモリ管理は TrgmNFA のライフサイクルに依存しているため、メモリリークや不正なポインタ参照が発生しないよう注意してください。
