pg_log_generic

定義

  • 種別: function
  • 定義箇所: src/common/logging.c:207

目的

PostgreSQLフロントエンドプログラムにおいて、可変長引数を受け取り、ログレベルと出力部位に応じたメッセージを標準エラー出力（stderr）へ書き出すための汎用ログ出力関数。

シグネチャまたは形状

void pg_log_generic(enum pg_log_level level, enum pg_log_part part, const char *pg_restrict fmt, ...);

入力と出力

  • **入力**:
  • level: ログの重要度（enum pg_log_level）。
  • part: メッセージの構成要素（enum pg_log_part）。
  • fmt: printf 形式のフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**: なし（void）。副作用として stderr へのログ出力を行う。

前提条件と不変条件

  • pg_logging_init() が呼び出され、ログシステムが初期化されていること。
  • フォーマット文字列 fmt の末尾に改行コードを含めないこと（フレームワーク側で自動付与されるため）。
  • バックエンドコードからは呼び出さないこと。

アルゴリズムまたは振る舞い

1. va_start を用いて可変長引数リストを初期化する。 2. pg_log_generic_v を呼び出し、実際のログ処理（フィルタリング、整形、出力）を委譲する。 3. va_end を用いて引数リストを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_utils.h
  • src/common/file_utils.c
  • src/include/common/logging.h（各種ログ出力マクロ経由）
  • **呼び出し先**:
  • pg_log_generic_v（同一ファイル内）

関連データ構造

  • enum pg_log_level: ログの重要度を定義。
  • enum pg_log_part: ログの出力部位（primary, detail, hint等）を定義。

失敗モードと境界条件

  • __pg_log_level によるフィルタリングにより、指定レベルが現在の閾値未満の場合は出力されない。
  • メモリ不足時には pg_malloc_extended のフォールバック機構により vfprintf が直接呼び出される。

保守担当者向けメモ

  • 本関数を直接呼び出すことは避け、src/include/common/logging.h で定義されている pg_log_error() や pg_log_debug() などのマクロを使用すること。
