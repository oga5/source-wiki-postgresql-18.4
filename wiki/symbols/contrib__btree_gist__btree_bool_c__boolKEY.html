boolKEY

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_bool.c:10

目的

PostgreSQLのGiSTインデックスにおいて、boolean型の値を範囲として表現し、インデックス構造内に格納するためのデータ構造です。

シグネチャまたは形状

typedef struct boolkey
{
	bool		lower;
	bool		upper;
} boolKEY;

入力と出力

  • 入力: bool 型の値。
  • 出力: boolKEY 構造体（lower および upper フィールドにブール値が格納される）。

前提条件と不変条件

  • リーフノードにおいて、lower == upper である必要があります。

アルゴリズムまたは振る舞い

  • GiSTインデックスのキーとして機能し、bool 型の値を範囲（lower から upper）として保持します。
  • インデックス構築時や検索時に、gbt_bool_compress 等の関数を通じて bool 型の値がこの構造体に変換されます。
  • 内部的には btree_utils_num.h の数値系ユーティリティを流用しており、bool 値を 0/1 の数値として扱います。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/btree_gist/btree_bool.c 内の各サポート関数（gbt_bool_compress, gbt_bool_consistent, gbt_bool_union, gbt_bool_penalty 等）が、この構造体を型キャストして利用しています。
  • 呼び出し先: 直接的な関数呼び出し関係はありません（データ構造定義のため）。

関連データ構造

  • gbtree_ninfo: bool 型の比較ロジックやサイズ情報を保持する構造体。
  • GISTENTRY: GiSTインデックスの各エントリを保持する構造体。

失敗モードと境界条件

  • 構造体のサイズは sizeof(boolKEY) に依存します。
  • 範囲の整合性が保たれない場合（リーフノードでの lower != upper など）、インデックス検索結果が不正確になる可能性があります。

保守担当者向けメモ

  • 本構造体は btree_utils_num.c の共通ロジックと密接に連携しています。修正を行う際は、数値系ユーティリティへの影響を考慮してください。
  • bool 型は実質的に 0/1 の数値として扱われるため、比較ロジックの変更は gbt_boolgt 等の関連関数と併せて確認してください。
