get_partition_parent_worker

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/partition.c:84

目的

指定されたリレーション（パーティション）の直接の親リレーションのOIDを取得し、そのパーティションがデタッチ処理中であるかを確認します。

シグネチャまたは形状

static Oid get_partition_parent_worker(Relation inhRel, Oid relid, bool *detach_pending)

入力と出力

  • **入力**:
  • inhRel: pg_inherits カタログを開いたリレーション記述子。
  • relid: 親を検索する対象リレーションのOID。
  • detach_pending: デタッチ状態を格納するためのブール値へのポインタ。
  • **出力**:
  • 戻り値: 親リレーションのOID。親が存在しない場合は InvalidOid。
  • *detach_pending: 対象パーティションがデタッチ保留中の場合に true が設定されます。

前提条件と不変条件

  • inhRel は pg_inherits カタログを指している必要があります。
  • relid は有効なリレーションOIDである必要があります。

アルゴリズムまたは振る舞い

1. detach_pending を false に初期化します。 2. pg_inherits カタログに対して、inhrelid が指定された relid と一致し、かつ inhseqno が 1 であるタプルを検索するためのスキャンキーを構築します。 3. systable_beginscan を使用してスキャンを開始し、systable_getnext で該当タプルを取得します。 4. タプルが見つかった場合、その inhdetachpending フィールドを確認し、true であれば detach_pending を更新します。 5. inhparent フィールドから親のOIDを取得し、スキャンを終了して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/partition.c 内の関数（get_partition_parent 等）から呼び出されます。
  • **呼び出し先**: systable_beginscan, systable_getnext, systable_endscan などのシステムカタログスキャン用API。

関連データ構造

  • pg_inherits: パーティション階層を保持するシステムカタログ。
  • Relation: リレーション記述子。
  • HeapTuple: カタログから取得されるタプル。

失敗モードと境界条件

  • 指定された relid に対応する親が存在しない場合、InvalidOid を返します。
  • pg_inherits に該当するエントリがない場合、スキャンは何も返さず、初期値の InvalidOid が返されます。

保守担当者向けメモ

  • 本関数は pg_inherits のスキャンに特化した内部ワーカー関数です。
  • inhseqno が 1 であるタプルのみを対象としているため、多重継承や特殊な継承構造における挙動については、呼び出し側の意図と整合しているか確認してください。
