pg_atomic_fetch_add_u32_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/arch-ppc.h:130

目的

PowerPCアーキテクチャにおいて、指定された32ビットアトミック変数に対して加算を行い、加算前の値を返す。

シグネチャまたは形状

static inline uint32 pg_atomic_fetch_add_u32_impl(volatile pg_atomic_uint32 *ptr, int32 add_)

入力と出力

  • **入力**:
  • ptr: 操作対象のアトミック変数へのポインタ。
  • add_: 加算する値（int32）。
  • **出力**: 加算操作が行われる前の ptr の値。

前提条件と不変条件

  • ptr は volatile 修飾された pg_atomic_uint32 型であること。
  • GCCインラインアセンブラが利用可能であること。
  • PowerPCのメモリモデルに従い、lwarx/stwcx. 命令による予約ループがアトミック性を保証する。

アルゴリズムまたは振る舞い

1. sync 命令によりメモリバリアを張る。 2. lwarx 命令でメモリをロードし、予約をセットする。 3. 加算処理を行う（定数 add_ が16ビット範囲内の場合は addi 命令、それ以外は add 命令を使用）。 4. stwcx. 命令で結果をストアする。予約が解除されていた場合は bne 命令でステップ2へ戻り再試行する。 5. lwsync 命令でメモリ順序を確定させ、加算前の値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/atomics/generic.h 等の汎用アトミックインターフェースから呼び出される。
  • **呼び出し先**: 直接的な関数呼び出しはない。インラインアセンブラによるPowerPC命令（sync, lwarx, stwcx., lwsync 等）を実行する。

関連データ構造

  • pg_atomic_uint32: 操作対象となるアトミック変数型。

失敗モードと境界条件

  • stwcx. 命令が失敗した場合、予約が失われたと判断してループ内で再試行を行うため、競合が激しい環境ではループ回数が増加する。
  • add_ が PG_INT16_MAX を超える場合、最適化パスを通らず汎用的な add 命令が使用される。

保守担当者向けメモ

  • 本関数はPowerPC特有のアセンブラ命令に依存している。修正時は memory および cc 制約が正しく設定されていることを確認すること。
  • HAVE_I_CONSTRAINT__BUILTIN_CONSTANT_P が定義されている環境では、定数最適化が有効になる。
