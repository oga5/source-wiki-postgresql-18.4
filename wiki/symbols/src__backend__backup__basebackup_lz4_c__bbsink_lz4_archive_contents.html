bbsink_lz4_archive_contents

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_lz4.c:179

目的

LZ4圧縮シンクにおいて、入力バッファ内のデータを圧縮し、後続のシンクへ転送する。

シグネチャまたは形状

static void bbsink_lz4_archive_contents(bbsink *sink, size_t avail_in)

入力と出力

  • sink: 処理対象の bbsink_lz4 構造体へのポインタ。
  • avail_in: 圧縮対象となる入力データのバイト数。
  • 出力: なし（副作用として後続シンクへのデータ転送および内部バッファの更新を行う）。

前提条件と不変条件

  • sink は bbsink_lz4 型としてキャスト可能であること。
  • LZ4F_compressUpdate がエラーを返さないこと。

アルゴリズムまたは振る舞い

1. LZ4F_compressBound を使用して、入力データに必要な最大圧縮後サイズを計算する。 2. 後続シンクのバッファ残量が計算したサイズを下回る場合、bbsink_archive_contents を呼び出して後続シンクへ既存の圧縮データをフラッシュし、書き込み済みバイト数をリセットする。 3. LZ4F_compressUpdate を呼び出し、入力バッファのデータを圧縮して後続シンクのバッファへ書き込む。 4. 圧縮処理が失敗した場合は elog(ERROR) で中断する。 5. 圧縮されたバイト数を mysink->bytes_written に加算し、状態を更新する。

呼び出し元と呼び出し先

  • 呼び出し元: bbsink_lz4_ops 構造体を通じて、バックアップ処理のパイプラインから間接的に呼び出される。
  • 呼び出し先:
  • bbsink_archive_contents (後続シンクへのフラッシュ用)
  • LZ4F_compressUpdate (LZ4ライブラリ)
  • LZ4F_compressBound (LZ4ライブラリ)
  • LZ4F_isError / LZ4F_getErrorName (LZ4ライブラリ)

関連データ構造

  • bbsink_lz4: 圧縮コンテキスト (ctx)、設定 (prefs)、書き込み済みバイト数 (bytes_written) を保持する構造体。
  • bbsink: パイプラインの基底構造体。

失敗モードと境界条件

  • 圧縮処理が失敗した場合、LZ4F_isError によりエラーが検出され、elog(ERROR) が発生する。
  • 後続シンクのバッファが不足していると判断された場合、自動的にフラッシュ処理が実行される。

保守担当者向けメモ

  • 本関数は USE_LZ4 が定義されている環境でのみ有効である。
  • マニフェストデータは本関数の対象外であり、圧縮されずに転送される。
