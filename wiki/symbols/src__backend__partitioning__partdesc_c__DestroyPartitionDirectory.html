DestroyPartitionDirectory

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partdesc.c:483

目的

PartitionDirectory が保持するすべてのパーティションリレーションの参照カウントを減らし、ディレクトリ構造を破棄するためのクリーンアップ処理を行います。

シグネチャまたは形状

void DestroyPartitionDirectory(PartitionDirectory pdir)

入力と出力

  • **入力**: pdir (PartitionDirectory) - 破棄対象のパーティションディレクトリ構造体。
  • **出力**: なし。

前提条件と不変条件

  • pdir は有効な PartitionDirectory 構造体へのポインタである必要があります。
  • pdir 内のハッシュテーブル pdir_hash に格納されている各 PartitionDirectoryEntry は、有効なリレーション (rel) を保持している必要があります。

アルゴリズムまたは振る舞い

1. hash_seq_init を使用して、pdir->pdir_hash に格納された全エントリを走査するためのシーケンシャルスキャンを開始します。 2. hash_seq_search を用いて、ハッシュテーブル内の各 PartitionDirectoryEntry (pde) を順次取得します。 3. 各エントリに対して RelationDecrementReferenceCount(pde->rel) を呼び出し、リレーションの参照カウントを減少させます。 4. すべてのエントリの処理が完了するまで繰り返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execUtils.c:221
  • src/backend/optimizer/plan/planner.c:630
  • **呼び出し先**:
  • hash_seq_init
  • hash_seq_search
  • RelationDecrementReferenceCount

関連データ構造

  • PartitionDirectory: 複数のパーティションリレーションを管理するコンテナ。
  • PartitionDirectoryEntry: ハッシュテーブル内の各要素であり、リレーション (rel) を保持する構造体。

失敗モードと境界条件

  • pdir が NULL の場合、あるいは pdir->pdir_hash が初期化されていない場合は未定義の動作となります。
  • ハッシュテーブルが空の場合、ループは実行されず即座に終了します。

保守担当者向けメモ

  • 本関数はディレクトリ内のリレーションの参照カウントを減らすのみであり、PartitionDirectory 構造体自体のメモリ解放（pfree 等）は本関数の責務に含まれていません。呼び出し元で適切にメモリ管理が行われていることを確認してください。
