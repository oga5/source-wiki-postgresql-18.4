ExplainPropertyInteger

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_format.c:222

目的

EXPLAIN コマンドの出力において、整数型のプロパティを現在のフォーマット（TEXT, XML, JSON, YAML）に合わせて整形し、出力バッファへ書き込む。

シグネチャまたは形状

void ExplainPropertyInteger(const char *qlabel, const char *unit, int64 value, ExplainState *es)

入力と出力

  • qlabel: プロパティのラベル名（例: "Batch Size"）。
  • unit: プロパティの単位（例: "b"）。不要な場合は NULL を指定。
  • value: 出力対象の int64 型の整数値。
  • es: 出力状態を管理する ExplainState 構造体へのポインタ。
  • 出力: 戻り値なし。副作用として es 内の出力バッファにフォーマット済みの文字列が追加される。

前提条件と不変条件

  • es は ExplainBeginOutput によって初期化済みであること。
  • value は int64 型の範囲内であること。

アルゴリズムまたは振る舞い

1. snprintf を使用し、INT64_FORMAT マクロに従って value を一時バッファ buf（サイズ32バイト）に文字列変換する。 2. 変換された文字列を ExplainProperty 関数へ渡し、ラベル、単位、値、および数値であることを示すフラグ（true）と共にフォーマット処理を委譲する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/commands/explain.c
  • contrib/file_fdw/file_fdw.c
  • contrib/pg_overexplain/pg_overexplain.c
  • contrib/postgres_fdw/postgres_fdw.c
  • 呼び出し先:
  • ExplainProperty (同ファイル内)

関連データ構造

  • ExplainState: 出力形式やバッファ状態を保持する構造体。

失敗モードと境界条件

  • buf のサイズ（32バイト）は int64 の最大値を表現するのに十分な長さである。
  • qlabel や unit が NULL の場合の挙動は、内部で呼び出される ExplainProperty の実装に依存する。

保守担当者向けメモ

  • 本関数は ExplainProperty のラッパーである。数値のフォーマットロジックを変更する場合は、ExplainProperty 側の処理を確認すること。
  • 新しい出力フォーマットを追加する場合は、ExplainProperty 内の switch 文の拡張が必要となる。
