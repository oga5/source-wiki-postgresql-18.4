AlterEnum

定義

  • 種別: function
  • 定義箇所: src/backend/commands/typecmds.c:1304

目的

SQLの ALTER TYPE ... ADD VALUE または RENAME VALUE コマンドを処理し、列挙型（Enum）のラベル追加または名称変更を実行する。

シグネチャまたは形状

ObjectAddress AlterEnum(AlterEnumStmt *stmt)

入力と出力

  • **入力**: AlterEnumStmt *stmt（パーサーが生成した列挙型変更用の構文木構造体）。
  • **出力**: ObjectAddress（変更された列挙型のOIDとカタログIDを含むオブジェクトアドレス）。

前提条件と不変条件

  • 指定された型名がシステムカタログ pg_type 上で列挙型として存在すること。
  • 呼び出し元が対象の列挙型に対する適切な所有権限を持っていること（checkEnumOwner により検証）。

アルゴリズムまたは振る舞い

1. **型解決**: makeTypeNameFromNameList と typenameTypeId を使用して、入力された型名からOIDを特定する。 2. **検証**: SearchSysCache1 で pg_type を検索し、対象が有効な型であることを確認後、checkEnumOwner で所有権をチェックする。 3. **操作の分岐**:

  • stmt->oldVal が存在する場合: RenameEnumLabel を呼び出し、既存のラベル名を変更する。
  • stmt->oldVal が存在しない場合: AddEnumLabel を呼び出し、新しいラベルを挿入する。

    • 4. **フックと戻り値**: InvokeObjectPostAlterHook を呼び出して変更を通知し、対象型の ObjectAddress を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（SQLコマンド実行のディスパッチャ）。
  • **呼び出し先**:
  • makeTypeNameFromNameList, typenameTypeId
  • SearchSysCache1, ReleaseSysCache
  • checkEnumOwner
  • RenameEnumLabel（oldVal 指定時）
  • AddEnumLabel（oldVal 未指定時）
  • InvokeObjectPostAlterHook

関連データ構造

  • AlterEnumStmt: SQLコマンドの解析結果を保持するノード。
  • ObjectAddress: オブジェクトの識別情報。
  • pg_type: 列挙型の定義を保持するシステムカタログ。

失敗モードと境界条件

  • 指定された型名が見つからない場合、または列挙型ではない場合、エラーが発生する可能性がある。
  • SearchSysCache1 で対象型が見つからない場合、elog(ERROR) により処理が中断される。
  • 権限不足の場合、checkEnumOwner 内でエラーが発生する。

保守担当者向けメモ

  • 本関数は pg_enum カタログの直接的な操作を RenameEnumLabel や AddEnumLabel に委譲している。列挙型の内部表現や順序付けに関するロジックを変更する場合は、それらの関数を参照すること。
  • InvokeObjectPostAlterHook は、イベントトリガーや他のサブシステムへの通知のために必須である。
