MultirangeType

定義

  • 種別: struct
  • 定義箇所: src/include/utils/multirangetypes.h:26

目的

PostgreSQLにおいて、複数のレンジ型（range）を一つの値として保持・管理するためのメモリ上のデータ構造を提供します。

シグネチャまたは形状

typedef struct {
    int32    vl_len_;         /* varlenaヘッダー */
    Oid      multirangetypid; /* マルチレンジ型自身のOID */
    uint32   rangeCount;      /* 格納されているレンジの数 */
    /* 続いて ShortRangeType 構造体の配列が配置される */
} MultirangeType;

入力と出力

  • 本構造体は varlena 型として扱われます。
  • 入力: PG_GETARG_MULTIRANGE_P マクロ等を用いて Datum から取得されます。
  • 出力: make_multirange 関数等で構築され、MultirangeTypePGetDatum を通じて Datum に変換されます。

前提条件と不変条件

  • vl_len_ メンバを直接操作してはなりません。
  • 構造体内部のレンジ配列は可変長であり、各レンジ自体も varlena である可能性があるため、インデックスによる直接アクセスは禁止されています。
  • アクセスには必ず提供されているマクロや関数（multirange_get_range 等）を使用する必要があります。

アルゴリズムまたは振る舞い

  • 内部的に複数の ShortRangeType を保持します。
  • レンジの数は rangeCount で管理されます。
  • 演算時には TypeCacheEntry を介して型情報を参照し、効率的な処理を行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execIndexing.c および src/backend/utils/adt/multirangetypes.c 内の各関数から参照されています。
  • 呼び出し先: 本構造体自体はデータ構造であり、操作には multirange_deserialize や make_multirange などのAPIが利用されます。

関連データ構造

  • ShortRangeType: 内部に保持される個別のレンジ型。
  • TypeCacheEntry: 型情報のキャッシュ管理に使用。
  • varlena: PostgreSQLの可変長データ型基底。

失敗モードと境界条件

  • レンジのサイズが可変であるため、メモリレイアウト上のオフセット計算には att_align_nominal 等の適切なアライメント処理が必須です。
  • 構造体メンバの直接参照は、メモリ破壊や不正なデータ読み取りを引き起こす可能性があります。

保守担当者向けメモ

  • 構造体定義の変更時は、multirangetypes.c 内のサイズ計算マクロ（sizeof(MultirangeType) を含む箇所）に影響がないか確認してください。
  • 内部のレンジ配列へアクセスする際は、必ず multirangetypes.h で定義されたアクセサ関数経由で行う設計を維持してください。
