gbt_int2key_cmp

定義

  • contrib/btree_gist/btree_int2.c:55

目的

GiST インデックス内で int2 型の範囲（int16KEY）を比較し、ソートや順序付けのための順序関係を判定する。

シグネチャまたは形状

int gbt_int2key_cmp(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の Nsrt 構造体へのポインタ。内部で int16KEY 型にキャストされる。
  • flinfo: 関数呼び出し情報（本関数内では未使用）。
  • **出力**:
  • 0: a と b の範囲が等しい場合。
  • 1: a が b より大きい場合。
  • -1: a が b より小さい場合。

前提条件と不変条件

  • 入力ポインタ a および b は、Nsrt 構造体として適切に初期化されており、そのメンバ t が int16KEY 型のデータ構造を指している必要がある。

アルゴリズムまたは振る舞い

1. a と b からそれぞれ int16KEY 構造体を取り出す。 2. まず lower 値を比較する。

  • ia->lower > ib->lower であれば 1 を返す。
  • ia->lower < ib->lower であれば -1 を返す。

    • 3. lower 値が等しい場合、upper 値を比較する。

  • ia->upper > ib->upper であれば 1 を返す。
  • ia->upper < ib->upper であれば -1 を返す。

    • 4. 両方の値が等しければ 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_int2.c 内の静的初期化箇所（tinfo 構造体の定義など）から参照されている。
  • **呼び出し先**: 直接的な関数呼び出しは行わず、ポインタ演算と構造体メンバの比較のみを行う。

関連データ構造

  • int16KEY: int2 型の範囲を保持する構造体（lower, upper を持つ）。
  • Nsrt: ソート処理で使用されるラッパー構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合の動作は定義されていない。
  • int16KEY の lower および upper 値が int16 の範囲外である場合の挙動は、呼び出し元のデータ整合性に依存する。

保守担当者向けメモ

  • 本関数は btree_utils_num が提供する汎用的な比較ロジックを int2 型に特化させたものである。
  • 比較ロジックを変更する場合は、int16KEY の定義および他の整数型（int4, int8）の実装との整合性に注意すること。
