bbsink_server_end_archive

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_server.c:193

目的

ベースバックアップのアーカイブ処理を終了し、書き込み中のファイルをディスクへ同期してクローズする。

シグネチャまたは形状

static void bbsink_server_end_archive(bbsink *sink)

入力と出力

  • **入力**: sink (bbsink *) - サーバー保存用シンクのインスタンス。
  • **出力**: なし（副作用としてファイルシステムへの書き込みとクローズを行う）。

前提条件と不変条件

  • sink は bbsink_server 型にキャスト可能であること。
  • mysink->file が有効なファイル記述子を保持していること。

アルゴリズムまたは振る舞い

1. sink を bbsink_server 型にキャストし、内部のファイル記述子を取得する。 2. FileSync を呼び出し、書き込み内容をディスクへフラッシュする。 3. FileClose を呼び出し、ファイルをクローズする。 4. mysink->file および mysink->filepos をリセットする。 5. bbsink_forward_end_archive(sink) を呼び出し、後続のシンクへ終了通知を転送する。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_server_ops 構造体を通じて、バックアップ処理のメインフローから間接的に呼び出される。
  • **呼び出し先**:
  • FileSync
  • FileClose
  • bbsink_forward_end_archive

関連データ構造

  • bbsink_server: サーバー保存用シンクの状態を保持する構造体。
  • bbsink: 基底となるシンクインターフェース。

失敗モードと境界条件

  • FileSync が失敗した場合、ereport(ERROR, ...) を発行し、処理を中断する。この際、リカバリ不能な状態を避けるため PANIC ではなく ERROR が使用される。

保守担当者向けメモ

  • 本関数はバックアップの整合性を保証するために fsync を実行する重要な箇所である。
  • 呼び出し元に関する直接的なコード上の参照は bbsink_server_ops 構造体への代入（src/backend/backup/basebackup_server.c:48）が確認されている。
