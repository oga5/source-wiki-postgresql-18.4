FileFdwPlanState

定義

  • 種別: struct
  • 定義箇所: contrib/file_fdw/file_fdw.c:100

目的

外部データラッパー（FDW）のプランニング段階において、読み込み対象のファイルやプログラムに関する情報、およびコスト見積もりに必要な統計情報を保持するために使用されます。

シグネチャまたは形状

typedef struct FileFdwPlanState
{
	char	   *filename;		/* 読み込み対象のファイルパスまたはプログラム名 */
	bool		is_program;		/* filename が OS コマンドである場合は true */
	List	   *options;		/* filename/is_program を除く、マージ済みの COPY オプション */
	BlockNumber pages;			/* ファイルの物理サイズの推定値 */
	double		ntuples;		/* データ行数の推定値 */
} FileFdwPlanState;

入力と出力

  • **入力**: プランニング時に palloc を介してメモリが割り当てられ、各フィールドが初期化されます。
  • **出力**: baserel->fdw_private に格納され、後続のプランニング処理や実行フェーズへ引き継がれます。

前提条件と不変条件

  • filename は有効なファイルパスまたは実行可能なプログラム名である必要があります。
  • options リストには、COPY コマンドで利用可能なオプションが含まれます。
  • pages および ntuples は、プランナがコスト計算を行うための推定値として保持されます。

アルゴリズムまたは振る舞い

1. **初期化**: プランニング中に palloc でメモリが確保されます。 2. **保持**: 外部テーブルのオプション解析結果や、estimate_size 等で算出された統計情報が各フィールドに格納されます。 3. **伝播**: この構造体は fdw_private としてプランナの RelOptInfo に保持され、実行フェーズへ渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/file_fdw/file_fdw.c 内のプランニング関連関数（fileGetForeignRelSize 等）から生成・参照されます。
  • **呼び出し先**: 実行フェーズの関数（fileBeginForeignScan 等）が、baserel->fdw_private をキャストして本構造体の情報にアクセスします。

関連データ構造

  • FileFdwExecutionState: 実行フェーズで使用されるコンテキスト構造体。
  • RelOptInfo: プランナがリレーション情報を保持するために使用する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • filename や options が不正な場合、プランニング段階で検証エラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はプランニングフェーズから実行フェーズへの情報受け渡しを担うため、フィールドを追加する際はプランニングと実行の両方のロジックに影響を与える点に注意してください。
  • COPY コマンドの仕様変更に伴い、options の扱いが変更される可能性があります。
