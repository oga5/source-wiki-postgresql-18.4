CreatePolicy

定義

  • 種別: function
  • 定義箇所: src/backend/commands/policy.c:568

目的

SQLの CREATE POLICY 文を処理し、指定されたテーブルに対する行レベルセキュリティ（RLS）ポリシーをシステムカタログ pg_policy に登録する。

シグネチャまたは形状

ObjectAddress CreatePolicy(CreatePolicyStmt *stmt)

入力と出力

  • **入力**: CreatePolicyStmt *stmt（解析済みの CREATE POLICY 文の構文木）
  • **出力**: ObjectAddress（作成されたポリシーオブジェクトの識別子）

前提条件と不変条件

  • 対象テーブルに対する適切な権限が必要（RangeVarCallbackForPolicy による検証）。
  • 同一テーブル内に同一名のポリシーが存在してはならない。
  • SELECT または DELETE コマンドに対して WITH CHECK 句を指定することはできない。
  • INSERT コマンドに対して USING 句（qual）を指定することはできない。

アルゴリズムまたは振る舞い

1. parse_policy_command でコマンド種別を特定し、構文の妥当性を検証する。 2. policy_role_list_to_array を用いてロールリストをOID配列に変換する。 3. RangeVarGetRelidExtended で対象テーブルのOIDを取得し、権限チェックを行う。 4. transformWhereClause を使用して、ポリシーの USING および WITH CHECK 句を式ツリーに変換し、照合順序（collation）を割り当てる。 5. pg_policy カタログをスキャンし、同名のポリシーが存在しないことを確認する。 6. GetNewOidWithIndex で新しいポリシーOIDを生成し、CatalogTupleInsert でカタログに挿入する。 7. recordDependencyOn および recordSharedDependencyOn を呼び出し、テーブル、式、ロールに対する依存関係を記録する。 8. InvokeObjectPostCreateHook を呼び出し、作成後のフック処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（SQL解析後のユーティリティ実行フェーズ）
  • **呼び出し先**: parse_policy_command, policy_role_list_to_array, RangeVarGetRelidExtended, transformWhereClause, recordDependencyOn, recordSharedDependencyOn 等。

関連データ構造

  • CreatePolicyStmt: SQL文の構文木。
  • pg_policy: ポリシー定義を保持するシステムカタログ。
  • ObjectAddress: 作成されたオブジェクトの識別子。

失敗モードと境界条件

  • 指定されたテーブルが存在しない場合、エラーとなる。
  • ポリシー名が重複している場合、ERRCODE_DUPLICATE_OBJECT エラーとなる。
  • 不正な句（SELECT に対する WITH CHECK 等）が指定された場合、ERRCODE_SYNTAX_ERROR エラーとなる。

保守担当者向けメモ

  • 本関数はカタログへの書き込みのみを行う。キャッシュの無効化（CacheInvalidateRelcache）は、この関数の呼び出し元または後続の処理で行われる必要がある（ソース抜粋の末尾参照）。
  • pg_policy の polroles は配列として格納されるため、ロールの変更には注意が必要。
