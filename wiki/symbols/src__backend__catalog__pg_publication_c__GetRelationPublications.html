GetRelationPublications

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_publication.c:750

目的

指定されたリレーション（テーブル）が登録されているすべてのパブリケーションのOIDリストを取得する。

シグネチャまたは形状

List *GetRelationPublications(Oid relid)

入力と出力

  • **入力**: relid (Oid) - 対象となるリレーションのOID。
  • **出力**: List * - パブリケーションのOID（pubid）を格納したリスト。

前提条件と不変条件

  • relid は有効なリレーションを指している必要がある。
  • 呼び出し元は、返されたリストに対して list_free() 等を用いてメモリを解放する責任を負う。

アルゴリズムまたは振る舞い

1. PUBLICATIONRELMAP システムキャッシュを使用して、指定された relid に関連付けられたすべての pg_publication_rel エントリを検索する。 2. 取得したキャッシュリストを走査し、各エントリからパブリケーションID (prpubid) を抽出する。 3. 抽出したIDを lappend_oid を用いて結果リストに追加する。 4. ReleaseSysCacheList を呼び出し、キャッシュリストを解放する。 5. 構築したリストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_publication.c
  • src/backend/commands/tablecmds.c
  • src/backend/replication/pgoutput/pgoutput.c
  • src/backend/utils/cache/relcache.c
  • **呼び出し先**:
  • SearchSysCacheList1 (システムキャッシュ検索)
  • GETSTRUCT (タプル構造体へのアクセス)
  • lappend_oid (リスト操作)
  • ReleaseSysCacheList (キャッシュ解放)

関連データ構造

  • pg_publication_rel: パブリケーションとリレーションの紐付けを管理するシステムカタログ。
  • CatCList: システムキャッシュから取得されるリスト構造。

失敗モードと境界条件

  • 指定されたリレーションがどのパブリケーションにも含まれていない場合、空のリスト（NIL）が返される。
  • システムキャッシュの検索失敗やメモリ不足時には、PostgreSQLの標準的なエラーハンドリング（elog 等）により処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数は PUBLICATIONRELMAP キャッシュに依存している。カタログ更新時にはキャッシュの無効化が正しく行われていることを確認すること。
  • パーティションテーブルの扱いについては、呼び出し元で GetPubPartitionOptionRelations 等と組み合わせて考慮する必要がある場合がある。
