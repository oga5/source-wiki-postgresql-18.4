index_close

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/indexam.c:176

目的

インデックスリレーションのクローズ処理および、必要に応じたロックの解放を行う。

シグネチャまたは形状

void index_close(Relation relation, LOCKMODE lockmode)

入力と出力

  • **入力**:
  • relation: クローズ対象のインデックスリレーション記述子。
  • lockmode: 解放するロックのモード（NoLock を指定した場合はロック解放を行わない）。
  • **出力**: なし（void）。

前提条件と不変条件

  • lockmode は NoLock 以上かつ MAX_LOCKMODES 未満である必要がある（Assert による検証）。

アルゴリズムまたは振る舞い

1. 指定された relation からロック識別子（LockRelId）を取得する。 2. RelationClose(relation) を呼び出し、リレーションキャッシュの参照カウントを減らす等のクローズ処理を実行する。 3. lockmode が NoLock でない場合、UnlockRelationId を呼び出し、対応するロックを解放する。

関連データ構造

  • Relation: インデックスリレーションの構造体。
  • LOCKMODE: ロックの強度を指定する列挙型。

失敗モードと境界条件

  • lockmode に不正な値が渡された場合、アサーションエラーが発生する可能性がある。
  • RelationClose 内部でのリレーション状態に関するエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はリレーションのクローズとロック解放を統合的に行うためのラッパーである。
  • ロックの解放はトランザクション終了まで保持される設計が一般的であるため、呼び出しタイミングには注意が必要である。
