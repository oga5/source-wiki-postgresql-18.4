Oid

定義

  • 種別: typedef
  • 定義箇所: src/include/commands/explain.h:49

目的

PostgreSQL システム内のデータベースオブジェクト（テーブル、インデックス、関数等）を一意に識別するための識別子型です。

シグネチャまたは形状

typedef として定義された数値型（通常は unsigned int）です。

入力と出力

  • 入力: オブジェクトを特定するための識別子として使用されます。
  • 出力: explain_get_index_name_hook_type 等の関数において、特定のオブジェクトを指し示す引数として渡されます。

前提条件と不変条件

  • システムカタログ内の各オブジェクトに対して一意であることが期待されます。
  • データベースクラスタ全体で有効な識別子として機能します。

アルゴリズムまたは振る舞い

特定のオブジェクトを検索・参照するためのキーとして機能します。explain.h の文脈では、インデックスの OID を渡すことで、対応するインデックス名を取得するフック関数の引数として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: explain.h 内のフック定義（explain_get_index_name_hook_type）で引数として使用されます。
  • 呼び出し先: 本シンボル自体は型定義であり、直接的な呼び出し先はありません。
  • 補足: contrib/amcheck 等の外部モジュールで、オブジェクト識別用として広範に使用されています。

関連データ構造

  • explain_get_index_name_hook_type: Oid を引数に取り、インデックス名を返す関数ポインタ型。

失敗モードと境界条件

  • 無効な OID（InvalidOid）が渡された場合、対応するオブジェクトが見つからず、フック関数がエラーまたは NULL を返す可能性があります。

保守担当者向けメモ

  • 本シンボルは PostgreSQL 全体で共通の型定義です。explain.h は単にこの型をフック関数のインターフェースとして利用しているに過ぎません。
  • 拡張機能開発において、オブジェクトを特定する際には必ずこの型を使用してください。
