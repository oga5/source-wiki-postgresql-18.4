set_plain_rel_pathlist

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/allpaths.c:767

目的

プレーンなリレーション（テーブル等）に対して、利用可能なスキャンパス（SeqScan, TIDScan, IndexScan）を生成し、RelOptInfo に登録する。

シグネチャまたは形状

static void set_plain_rel_pathlist(PlannerInfo *root, RelOptInfo *rel, RangeTblEntry *rte)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel: パスを生成対象とするリレーション情報。
  • rte: リレーションに対応するレンジテーブルエントリ。
  • **出力**: なし（副作用として rel->pathlist 等にパスを追加する）。

前提条件と不変条件

  • rel->lateral_relids に基づき、LATERAL 参照によるパラメータ化が必要な場合は required_outer に保持される。
  • CurrentOfExpr を含むクエリの場合、TID スキャンが強制される。

アルゴリズムまたは振る舞い

1. **TID スキャンの検討**: create_tidscan_paths を呼び出す。これが真を返した場合、TID スキャンが唯一の有効なパスであるため、他のパスを生成せずに即座に終了する。 2. **シーケンシャルスキャンの追加**: create_seqscan_path を呼び出し、結果を add_path で登録する。 3. **並列スキャンの検討**: rel->consider_parallel が真かつ required_outer が空（パラメータ化されていない）の場合、create_plain_partial_paths を呼び出し、並列実行可能なパスを生成する。 4. **インデックススキャンの追加**: create_index_paths を呼び出し、利用可能なインデックススキャンパスを生成する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/allpaths.c 内の set_rel_pathlist から呼び出される。
  • **呼び出し先**:
  • create_tidscan_paths
  • create_seqscan_path
  • add_path
  • create_plain_partial_paths
  • create_index_paths

関連データ構造

  • PlannerInfo: プランナの全体状態。
  • RelOptInfo: パス生成対象のリレーション情報。
  • RangeTblEntry: クエリ内のテーブル定義情報。

失敗モードと境界条件

  • CurrentOfExpr が存在する場合、TID スキャン以外のパス生成はスキップされる。
  • required_outer が空でない場合、並列スキャンパスの生成は行われない。

保守担当者向けメモ

  • 本関数は set_rel_pathlist からの分岐先の一つであり、主に通常のテーブルスキャンを担当する。
  • 結合条件のプッシュダウンはサポートされていないが、LATERAL 参照によるパラメータ化には対応している。
