BTInsertStateData

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/nbtree.h:820

目的

B-tree へのタプル挿入処理中に使用される作業用データ構造です。挿入対象のタプル情報、スキャンキー、および挿入先バッファの状態を保持します。

シグネチャまたは形状

typedef struct BTInsertStateData
{
    IndexTuple    itup;           /* 挿入対象のタプル */
    Size          itemsz;         /* itup のサイズ（MAXALIGN 済みであること） */
    BTScanInsert  itup_key;       /* 挿入用スキャンキー */
    Buffer        buf;            /* 挿入先となるリーフページのバッファ */
    bool          bounds_valid;   /* バッファ内の境界キャッシュの有効性 */
    OffsetNumber  low;            /* 検索範囲の開始オフセット */
    OffsetNumber  stricthigh;     /* 検索範囲の終了オフセット */
    int           postingoff;     /* ポスティングリスト内の位置（-1 は無効/未設定） */
} BTInsertStateData;

入力と出力

本構造体は関数間で状態を共有するために使用されます。

  • **入力**: 挿入対象の IndexTuple および BTScanInsert 構造体。
  • **出力**: 挿入処理の結果として更新されたバッファ状態およびオフセット情報。

前提条件と不変条件

  • itemsz は MAXALIGN() されている必要があります。
  • bounds_valid が true の場合、low および stricthigh は現在の buf 内で有効な検索範囲を示します。
  • postingoff が -1 の場合、ポスティングリスト内での位置は特定されていないか、あるいは LP_DEAD ビットが設定されたタプルとの重複を示します。

アルゴリズムまたは振る舞い

  • _bt_binsrch_insert および _bt_findinsertloc において、挿入位置の特定と重複チェック（_bt_check_unique）の効率化のために使用されます。
  • ページ内の検索範囲をキャッシュすることで、再検索のコストを削減します。
  • ポスティングリスト（重複排除されたタプル）への挿入時、リスト内の正確な位置を保持します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtinsert.c
  • contrib/amcheck/verify_nbtree.c
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。関連するロジック（_bt_binsrch_insert 等）によって操作されます。

関連データ構造

  • BTScanInsertData: 挿入時の検索キーを定義する構造体。
  • IndexTuple: インデックスに格納される実際のデータ構造。

失敗モードと境界条件

  • postingoff が -1 の場合、既存のポスティングリストタプルが LP_DEAD であることを示唆しており、挿入ロジック側で適切なハンドリングが必要です。
  • buf が無効な場合、挿入操作は継続できません。

保守担当者向けメモ

  • 本構造体は nbtree の挿入ロジックの内部状態を保持するものであり、外部モジュールからの直接的な操作は避けてください。
  • BTREE_VERSION の変更に伴い、タプルの構造やポスティングリストの扱いが変更される可能性があるため、関連するインライン関数（BTreeTupleIsPivot 等）と併せて確認してください。
