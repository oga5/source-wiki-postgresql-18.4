guc_free

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/guc.c:688

目的

GUC（Grand Unified Configuration）システムに関連するメモリ領域を安全に解放します。pfree() とは異なり、NULL ポインタの入力を許容するラッパーとして機能します。

シグネチャまたは形状

void guc_free(void *ptr)

入力と出力

  • **入力**: ptr (解放対象のメモリ領域へのポインタ)
  • **出力**: なし

前提条件と不変条件

  • ptr が NULL でない場合、そのメモリ領域は GUCMemoryContext に割り当てられている必要があります。この条件は Assert によって検証されます。

アルゴリズムまたは振る舞い

1. ptr が NULL かどうかを確認します。 2. ptr が NULL の場合は何もしません。 3. ptr が NULL でない場合、GetMemoryChunkContext(ptr) を呼び出し、対象メモリが GUCMemoryContext に属していることをアサートします。 4. pfree(ptr) を呼び出してメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/variable.c
  • src/backend/utils/cache/ts_cache.c
  • src/backend/utils/misc/guc.c 内の各所
  • **呼び出し先**:
  • GetMemoryChunkContext
  • pfree

関連データ構造

  • GUCMemoryContext: GUC 関連のデータが保持される専用メモリコンテキスト。

失敗モードと境界条件

  • ptr が NULL の場合、安全に無視されます。
  • ptr が GUCMemoryContext 以外のメモリコンテキストに属している場合、デバッグビルドでは Assert 失敗によりプロセスが停止します。

保守担当者向けメモ

  • 本関数は、誤って malloc で確保されたメモリが渡されることを防ぐためのガードレールとして機能します。
  • pfree() との互換性ではなく、GUC システムの歴史的な慣習（free(NULL) の許容）を維持するために存在しています。
