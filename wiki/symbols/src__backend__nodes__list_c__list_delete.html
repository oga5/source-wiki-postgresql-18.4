list_delete

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/list.c:852

目的

ポインタリストから指定された値と一致する最初の要素を検索し、削除する。

シグネチャまたは形状

List *list_delete(List *list, void *datum)

入力と出力

  • **入力**:
  • list: 操作対象のポインタリスト（List *）。
  • datum: 削除対象の要素値（void *）。
  • **出力**:
  • 削除後のリストを指すポインタ（List *）。要素が見つからなかった場合は元のリストがそのまま返される。

前提条件と不変条件

  • list はポインタリストである必要がある（IsPointerList(list) が真であること）。
  • 操作の開始時に check_list_invariants(list) が呼び出され、リストの整合性が検証される。

アルゴリズムまたは振る舞い

1. リスト内の各要素を foreach マクロで順次走査する。 2. equal(lfirst(cell), datum) を用いて、現在の要素が datum と等しいか判定する。 3. 一致する要素が見つかった場合、list_delete_cell(list, cell) を呼び出して当該セルを削除し、その結果を返す。 4. リストの末尾まで一致するものが見つからなかった場合、変更を加えずに元のリストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/commands/subscriptioncmds.c
  • src/backend/executor/execUtils.c
  • src/backend/optimizer/path/indxpath.c
  • src/backend/utils/misc/queryenvironment.c
  • src/test/modules/injection_points/injection_points.c
  • **呼び出し先**:
  • list_delete_cell (一致が見つかった場合)
  • equal (要素比較用)

関連データ構造

  • List: リストのヘッダ構造体。
  • ListCell: リスト内の個別の要素を保持する構造体。

失敗モードと境界条件

  • list がポインタリストでない場合、Assert により異常終了する可能性がある。
  • リスト内に datum と一致する要素が存在しない場合、リストは変更されず、元のポインタがそのまま返される。
  • リストの削除操作により内部配列が再配置される可能性があるため、呼び出し元はリスト操作中に保持していた ListCell へのポインタが無効化されるリスクを考慮する必要がある。

保守担当者向けメモ

  • 本関数はポインタリスト専用である。整数やOIDなどの値を扱う場合は、対応する list_delete_int や list_delete_oid を使用すること。
  • 配列ベースの実装であるため、削除操作は $O(N)$ の計算量となる。頻繁な削除が必要な場合はデータ構造の選択を再検討すること。
