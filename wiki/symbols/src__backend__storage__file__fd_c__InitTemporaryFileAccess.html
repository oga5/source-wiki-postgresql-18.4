InitTemporaryFileAccess

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/fd.c:932

目的

一時ファイル操作を許可するための初期化処理を行います。プロセス終了時に一時ファイルを適切にクリーンアップするためのフックを登録します。

シグネチャまたは形状

void InitTemporaryFileAccess(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として一時ファイル操作の許可状態を更新し、終了フックを登録）

前提条件と不変条件

  • InitFileAccess() が既に実行され、SizeVfdCache が初期化されていること。
  • 本関数はプロセス内で一度だけ呼び出されること。

アルゴリズムまたは振る舞い

1. SizeVfdCache が0でないことを確認し、InitFileAccess が実行済みであることを検証します。 2. temporary_files_allowed が偽であることを確認し、二重初期化を防止します。 3. before_shmem_exit を呼び出し、共有メモリ終了時に BeforeShmemExit_Files が実行されるよう登録します。これにより、統計情報を報告可能な段階で一時ファイルが確実に削除されます。 4. アサーション有効時（USE_ASSERT_CHECKING）のみ、temporary_files_allowed を真に設定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/init/postinit.c:650
  • 呼び出し先: before_shmem_exit（PostgreSQL内部の終了フック登録関数）

関連データ構造

  • temporary_files_allowed: 一時ファイル操作の許可状態を保持するグローバル変数。

失敗モードと境界条件

  • InitFileAccess() より前に呼び出された場合、アサーションエラーが発生します。
  • 二度呼び出された場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は、PostgreSQLの起動プロセスにおいて一時ファイルの使用を開始する直前に呼び出される必要があります。
  • BeforeShmemExit_Files の登録は、プロセス終了時のリソースリークを防ぐために不可欠です。
