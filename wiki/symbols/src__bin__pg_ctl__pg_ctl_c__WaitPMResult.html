WaitPMResult

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_ctl/pg_ctl.c:44

目的

PostgreSQL サーバー（postmaster）の起動待機処理における、現在の状態および結果を表現するために使用されます。

シグネチャまたは形状

typedef enum
{
	POSTMASTER_READY,
	POSTMASTER_STILL_STARTING,
	POSTMASTER_SHUTDOWN_IN_RECOVERY,
	POSTMASTER_FAILED,
} WaitPMResult;

入力と出力

本シンボルは列挙型であり、関数 wait_for_postmaster_start の戻り値として、サーバーの起動監視結果を呼び出し元へ伝達するために出力されます。

前提条件と不変条件

  • サーバーの起動監視（-w オプション等）を行うプロセスにおいて、状態遷移の判定基準として利用されます。
  • 各列挙値は、サーバーが接続を受け入れ可能か、あるいは起動に失敗したか等の排他的な状態を表します。

アルゴリズムまたは振る舞い

  • POSTMASTER_READY: サーバーが正常に起動し、接続を受け入れ可能な状態。
  • POSTMASTER_SHUTDOWN_IN_RECOVERY: サーバーがリカバリ中にシャットダウンした状態。
  • POSTMASTER_FAILED: サーバーの起動が失敗した状態。

呼び出し元と呼び出し先

  • 呼び出し元: wait_for_postmaster_start 関数がこの型を返します。
  • 呼び出し先: 該当なし（列挙型のため）。

関連データ構造

  • CtlCommand: pg_ctl の操作コマンドを定義する列挙型。
  • wait_for_postmaster_start: この型を戻り値として使用する主要な関数。

失敗モードと境界条件

  • サーバーの起動がタイムアウトした場合や、PID ファイルの読み取りに失敗した場合など、POSTMASTER_FAILED が返されることで異常系への分岐が行われます。

保守担当者向けメモ

  • サーバーの起動状態判定ロジックに変更を加える際は、この列挙型に新しい状態が必要か検討してください。
  • wait_for_postmaster_start 関数内での状態遷移ロジックと密結合しているため、修正時は当該関数の実装を併せて確認してください。
