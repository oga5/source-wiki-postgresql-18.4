JsonLikeRegexContext

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonpath_exec.c:120

目的

JSONパス実行エンジンにおいて、正規表現マッチング操作に必要なパラメータを保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct JsonLikeRegexContext
{
	text	   *regex;
	int			cflags;
} JsonLikeRegexContext;

入力と出力

  • **入力**:
  • regex: 正規表現パターンを格納する text 型へのポインタ。
  • cflags: 正規表現のコンパイルオプション（フラグ）。
  • **出力**: 本構造体自体はデータ保持用であり、関数呼び出しの引数として正規表現実行ロジックへ渡されます。

前提条件と不変条件

  • regex フィールドは有効な text 型の正規表現パターンを指している必要があります。
  • cflags は regex/regex.h で定義された正規表現エンジンが解釈可能なフラグである必要があります。

アルゴリズムまたは振る舞い

本構造体は、JSONパス内の正規表現マッチング処理において、実行に必要な設定をカプセル化します。実行時にこの構造体が初期化され、正規表現エンジンへ渡されることで、特定の jsonb 値に対するパターンマッチングが実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonpath_exec.c:1861 にてローカル変数として初期化される箇所が確認されています。
  • **呼び出し先**:
  • src/backend/utils/adt/jsonpath_exec.c:2269 にて、関数引数（param）として受け取られ、正規表現実行ロジック内で参照されています。

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • regex/regex.h: 本構造体が利用する正規表現エンジンの定義ヘッダー。

失敗モードと境界条件

  • regex が NULL の場合、または不正な正規表現パターンが含まれている場合、正規表現エンジン側でエラーが発生する可能性があります。
  • cflags に無効なフラグが設定された場合、マッチング処理が意図しない挙動を示す可能性があります。

保守担当者向けメモ

  • 本構造体は jsonpath_exec.c 内の正規表現評価ロジックに特化しています。
  • 正規表現のコンパイルや実行に関する詳細な挙動は、regex/regex.h の仕様に依存します。
