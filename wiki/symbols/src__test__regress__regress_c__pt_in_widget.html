pt_in_widget

定義

  • 種別: function
  • 定義箇所: src/test/regress/regress.c:220

目的

指定された Point が、テスト用データ型 WIDGET の領域（中心点と半径で定義される円）に含まれているかどうかを判定する。

シグネチャまたは形状

Datum pt_in_widget(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINT_P(0): 判定対象となる Point 構造体へのポインタ。
  • PG_GETARG_POINTER(1): 判定基準となる WIDGET 構造体へのポインタ。
  • **出力**:
  • PG_RETURN_BOOL: 点が円の内部にある場合は true、そうでない場合は false を返す。

前提条件と不変条件

  • WIDGET 構造体は、center（Point 型）と radius（float8 型）を保持している必要がある。
  • DirectFunctionCall2 を介して point_distance 関数を呼び出すため、PostgreSQL のジオメトリ演算環境が初期化されている必要がある。

アルゴリズムまたは振る舞い

1. 第1引数から Point 構造体を取得する。 2. 第2引数から WIDGET 構造体を取得する。 3. DirectFunctionCall2 を使用し、標準の point_distance 関数を呼び出して、入力点と WIDGET の中心点との距離を算出する。 4. 算出した距離が WIDGET の半径 (radius) より小さいかどうかを比較する。 5. 比較結果をブール値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由（PG_FUNCTION_INFO_V1(pt_in_widget) によりエクスポート）。具体的な SQL クエリからの呼び出し履歴は提供されていない。
  • **呼び出し先**:
  • point_distance (PostgreSQL 内部関数)

関連データ構造

  • Point: 座標を保持する構造体。
  • WIDGET: src/test/regress/regress.c で定義されるテスト用カスタムデータ型。

失敗モードと境界条件

  • 引数が正しく渡されない場合、PG_GETARG_* マクロの内部でエラーが発生する可能性がある。
  • point_distance が予期しない値を返した場合、距離計算に影響する可能性がある。

保守担当者向けメモ

  • 本関数は回帰テスト専用であり、本番環境での利用は想定されていない。
  • WIDGET 構造体の定義変更時には、本関数のアクセス方法も追従する必要がある。
