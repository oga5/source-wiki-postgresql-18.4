bbsink_forward_begin_manifest

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_sink.c:75

目的

bbsink（バックアップシンク）チェーンにおいて、マニフェスト生成の開始処理を後続のシンクへ転送する。

シグネチャまたは形状

void bbsink_forward_begin_manifest(bbsink *sink)

入力と出力

  • **入力**: sink (bbsink *) - 現在のシンクオブジェクト。
  • **出力**: なし。

前提条件と不変条件

  • sink->bbs_next が NULL であってはならない（アサーションにより検証済み）。

アルゴリズムまたは振る舞い

1. sink->bbs_next が存在することを確認する。 2. sink->bbs_next（次のシンク）の begin_manifest メソッドを呼び出し、処理を委譲する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/basebackup_to_shell/basebackup_to_shell.c
  • src/backend/backup/basebackup_server.c
  • **呼び出し先**:
  • bbsink_begin_manifest（sink->bbs_next に対して実行）
  • **参照元（メソッド割り当て）**:
  • src/backend/backup/basebackup_gzip.c
  • src/backend/backup/basebackup_lz4.c
  • src/backend/backup/basebackup_progress.c
  • src/backend/backup/basebackup_throttle.c
  • src/backend/backup/basebackup_zstd.c

関連データ構造

  • bbsink: バックアップデータの転送先を抽象化した構造体。bbs_next メンバを通じて次のシンクを保持する。

失敗モードと境界条件

  • sink->bbs_next が NULL の場合、アサーション違反によりプログラムが停止する。

保守担当者向けメモ

  • 本関数は転送専用のヘルパーである。独自のシンクを実装する際、マニフェスト生成の開始処理を単に次へ流すだけであれば、bbsink_ops 構造体の begin_manifest フィールドに本関数を直接割り当てることが可能である。
