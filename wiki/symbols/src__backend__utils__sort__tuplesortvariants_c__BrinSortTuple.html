BrinSortTuple

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/sort/tuplesortvariants.c:169

目的

BRIN（Block Range Index）インデックスのソート処理において、メモリ上またはディスク上の論理テープに格納されるタプルを保持するための構造体です。

シグネチャまたは形状

typedef struct BrinSortTuple
{
	Size		tuplen;
	BrinTuple	tuple;
} BrinSortTuple;

入力と出力

本構造体はデータ保持用であり、直接的な関数入力・出力インターフェースを持ちません。関連するマクロ BRINSORTTUPLE_SIZE(len) を通じて、BrinTuple の長さに応じたメモリ確保サイズを算出するために使用されます。

前提条件と不変条件

  • tuplen は、後続する tuple フィールドの実際のサイズと一致している必要があります。
  • BRINSORTTUPLE_SIZE マクロにより、offsetof(BrinSortTuple, tuple) と len を加算したサイズが、この構造体を格納するために必要なメモリ量として定義されています。

アルゴリズムまたは振る舞い

BRINインデックスのソート時に、tuplesort エンジンが扱う汎用的なタプル形式を、BRIN固有のデータ構造に適合させるためのラッパーとして機能します。メモリ不足時にディスクへ書き出す際や、ソート済みデータを読み出す際に、この構造体を通じて BrinTuple データへのアクセスが行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/tuplesortvariants.c 内の BRIN ソート関連関数（tuplesort_begin_index_brin 等の内部処理）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、特定の関数を呼び出すことはありません。

関連データ構造

  • BrinTuple: 本構造体が保持する実データ。
  • SortTuple: tuplesort.c がソート時に使用する汎用的なタプル構造体。BrinSortTuple はこの SortTuple の一部としてキャストされて扱われることがあります。

失敗モードと境界条件

  • tuplen の値が不正確な場合、メモリの読み書き時にバッファオーバーランや不正なメモリ参照が発生する可能性があります。
  • BRINSORTTUPLE_SIZE を使用せずに手動でサイズ計算を行うと、アライメントやオフセットの不整合によりデータ破損を招く恐れがあります。

保守担当者向けメモ

  • 本構造体は tuplesortvariants.c 内でのみ使用される内部的な型です。
  • BrinTuple の構造に変更があった場合、本構造体の定義および BRINSORTTUPLE_SIZE マクロの整合性を確認してください。
  • tuplesort_readtup_alloc 等の関数でキャストして使用されるため、型変換時の安全性に注意してください。
