copy_options

定義

  • 種別: struct
  • 定義箇所: src/bin/psql/copy.c:53

目的

psql の \copy メタコマンドにおける解析結果を保持し、サーバーとのデータ転送処理（COPY プロトコル）に必要な設定情報を管理するために使用されます。

シグネチャまたは形状

struct copy_options
{
    char       *before_tofrom;  /* COPY string before TO/FROM */
    char       *after_tofrom;   /* COPY string after TO/FROM filename */
    char       *file;           /* NULL = stdin/stdout */
    bool        program;        /* is 'file' a program to popen? */
    bool        psql_inout;     /* true = use psql stdin/stdout */
    bool        from;           /* true = FROM, false = TO */
};

入力と出力

  • 本構造体は、parse_slash_copy 関数によって引数から解析・生成されます。
  • do_copy 関数などの転送処理ロジックにおいて、読み取り専用の構成情報として参照されます。

前提条件と不変条件

  • file が NULL の場合、標準入出力が対象であることを示します。
  • program が true の場合、file は popen で実行されるコマンド文字列として扱われます。
  • psql_inout が true の場合、psql の標準入出力ストリームが優先的に使用されます。

アルゴリズムまたは振る舞い

  • \copy コマンドの構文解析時にインスタンス化され、転送方向（FROM/TO）や対象ファイルパス、実行プログラムの有無といった実行時パラメータを保持します。
  • 転送処理の終了後、関連するメモリは free_copy_options 関数によって解放される必要があります。

呼び出し元と呼び出し先

  • **呼び出し元**: parse_slash_copy（生成）、do_copy（参照）。
  • **呼び出し先**: free_copy_options（メモリ解放）。
  • ※上記以外の具体的な呼び出し関係は、提供されたソース範囲内では確認できません。

関連データ構造

  • src/bin/psql/copy.h: 公開インターフェース定義。

失敗モードと境界条件

  • file 文字列のメモリ管理が不適切な場合、メモリリークが発生する可能性があります。
  • program フラグと file のパス指定が矛盾する場合、popen 実行時にエラーとなる可能性があります。

保守担当者向けメモ

  • 本構造体は psql の \copy 実装に特化しています。新しい COPY オプションを追加する場合は、この構造体にフィールドを追加し、parse_slash_copy での解析ロジックを更新してください。
