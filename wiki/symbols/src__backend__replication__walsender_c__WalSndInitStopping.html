WalSndInitStopping

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walsender.c:3804

目的

稼働中のすべての WAL 送信プロセス（WalSender）に対し、シャットダウン準備を開始するようシグナルを送信する。

シグネチャまたは形状

void WalSndInitStopping(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として各 WAL 送信プロセスへシグナルを送信）

前提条件と不変条件

  • WalSndCtl が初期化済みであること。
  • max_wal_senders が有効な値であること。

アルゴリズムまたは振る舞い

1. WalSndCtl が管理するすべての WAL 送信スロット（0 から max_wal_senders - 1）を反復処理する。 2. 各スロットの mutex を取得し、現在のプロセス ID (pid) を確認する。 3. pid が 0 でない（プロセスが稼働中である）場合、SendProcSignal を用いて当該プロセスに PROCSIG_WALSND_INIT_STOPPING シグナルを送信する。 4. mutex を解放し、次のスロットへ進む。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlog.c:6658
  • 呼び出し先: SendProcSignal

関連データ構造

  • WalSndCtl: 全 WAL 送信プロセスの状態を保持する共有メモリ上の制御構造体。
  • WalSnd: 個別の WAL 送信プロセスの状態を保持する構造体。

失敗モードと境界条件

  • max_wal_senders が 0 の場合、ループは実行されず何も行われない。
  • 特定の WalSnd スロットが空（pid == 0）の場合、そのスロットはスキップされる。

保守担当者向けメモ

  • 本関数はシャットダウンシーケンスの一部として呼び出される。シグナルを受け取った各 WAL 送信プロセスがどのように停止処理を行うかは、各プロセスのシグナルハンドラまたはメインループ内のチェックロジックに依存する。
