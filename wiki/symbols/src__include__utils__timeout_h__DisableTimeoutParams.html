DisableTimeoutParams

定義

  • 種別: struct
  • 定義箇所: src/include/utils/timeout.h:69

目的

複数のタイムアウト設定を一度に解除する際、解除対象の識別子と、タイムアウト発生時にセットされるインジケータフラグを保持し続けるか否かを指定するための構造体です。

シグネチャまたは形状

typedef struct
{
	TimeoutId	id;				/* 解除対象のタイムアウトID */
	bool		keep_indicator; /* インジケータフラグを保持するか */
} DisableTimeoutParams;

入力と出力

  • 本構造体は disable_timeouts 関数への入力として使用されます。
  • 出力はありません（本構造体自体は設定保持用です）。

前提条件と不変条件

  • id には有効な TimeoutId を指定する必要があります。
  • disable_timeouts に渡す際は、配列の末尾を特定するために count 引数と組み合わせて使用されます。

アルゴリズムまたは振る舞い

  • disable_timeouts 関数において、指定された id に対応するタイムアウトを無効化します。
  • keep_indicator が true の場合、タイムアウト発生時にセットされたインジケータフラグをクリアせずに保持します。false の場合はフラグもクリアされます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/lmgr/proc.c 内の複数箇所で、DisableTimeoutParams 型の配列として定義・使用されています。
  • **呼び出し先**: src/backend/utils/misc/timeout.c の disable_timeouts 関数が本構造体を受け取り、処理を実行します。

関連データ構造

  • TimeoutId: タイムアウトの種類を識別する列挙型。
  • EnableTimeoutParams: タイムアウトを有効化するための対となる構造体。

失敗モードと境界条件

  • 無効な TimeoutId を指定した場合の挙動については、disable_timeouts 関数の実装に依存します。
  • 配列の count が実際の要素数と一致しない場合、メモリ破壊や意図しないタイムアウトの解除が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は disable_timeouts のインターフェースの一部です。変更を加える際は、src/backend/utils/misc/timeout.c の実装と整合性を取ってください。
