find_expr_references_context

定義

  • 種別: struct
  • 定義箇所: src/backend/catalog/dependency.c:139

目的

式（expression）の走査中に、参照されているデータベースオブジェクトを収集し、関連する範囲テーブル（rangetable）を解決するためのコンテキスト情報を保持する。

シグネチャまたは形状

typedef struct
{
	ObjectAddresses *addrs;		/* 蓄積されるオブジェクトアドレス */
	List	   *rtables;		/* Varを解決するための範囲テーブルのリスト */
} find_expr_references_context;

入力と出力

本構造体は関数への引数として渡され、走査処理を通じて addrs フィールドにオブジェクトアドレスが蓄積される。

前提条件と不変条件

  • addrs は、オブジェクトアドレスを格納するために初期化された ObjectAddresses 構造体を指している必要がある。
  • rtables は、式内の Var ノードを解決するために必要な範囲テーブルのリストを保持する。

アルゴリズムまたは振る舞い

式ツリーを走査する際に、このコンテキストを使用して発見されたオブジェクトを addrs リストに追加する。また、式に含まれる Var ノードの解決が必要な場合、rtables を参照して適切なコンテキスト情報を取得する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/dependency.c 内の複数の箇所（例: 1557, 1602 行目付近）でローカル変数としてインスタンス化される。
  • 呼び出し先: 本構造体を受け取る関数（find_expr_references_walker 等）が、このコンテキストを介してオブジェクトの収集を行う。

関連データ構造

  • ObjectAddresses: 収集されたオブジェクトのリストを管理する。
  • List: PostgreSQLの標準的なリスト構造。rtables の保持に使用される。

失敗モードと境界条件

  • addrs が NULL の状態で走査関数に渡された場合、メモリ保護違反が発生する可能性がある。
  • 循環参照や複雑な式ツリーの走査において、スタック深さの制限（check_stack_depth()）に抵触する可能性がある。

保守担当者向けメモ

  • 本構造体は find_expr_references_walker と密接に関連している。式走査ロジックを変更する際は、このコンテキストが保持する情報が十分か確認すること。
