TCPattern

定義

  • 種別: struct
  • 定義箇所: src/bin/psql/tab-complete.in.c:1364

目的

psql のタブ補完機能において、入力された単語列と照合するためのパターン定義を保持する。

シグネチャまたは形状

typedef struct {
    int id;                     /* match_previous_words で使用される case ラベル */
    enum TCPatternKind kind;    /* マッチングの種類 */
    int nwords;                 /* words 配列の長さ */
    const char *const *words;   /* マッチ対象の単語配列 */
} TCPattern;

入力と出力

  • 入力: match_previous_words 関数内で、この構造体の配列 (tcpatterns) が参照される。
  • 出力: 構造体の各フィールドは、補完ロジックにおける条件分岐およびマッチングの判定基準として利用される。

前提条件と不変条件

  • nwords は words 配列の実際の要素数と一致している必要がある。
  • id は match_previous_words 内の制御フロー（switch 文のラベル等）と整合している必要がある。

アルゴリズムまたは振る舞い

  • tcpatterns 配列の要素として静的に定義され、補完候補を決定するためのパターンマッチングの基盤となる。
  • ユーザー入力の単語列が words 配列の内容と一致するかを判定し、一致した場合に id に基づく処理へ分岐する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/tab-complete.in.c 内の match_previous_words 関数が、tcpatterns 配列を通じて本構造体を参照する。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはない。

関連データ構造

  • TCPatternKind: マッチングの種類を定義する列挙型。
  • tcpatterns: TCPattern 型の静的配列。

失敗モードと境界条件

  • nwords と words の不整合が発生した場合、配列外参照や誤ったマッチング判定を招く可能性がある。

保守担当者向けメモ

  • 本構造体は gen_tabcomplete.pl による自動生成プロセスと密接に関連している。構造を変更する場合は、生成スクリプトおよび match_previous_words のロジックへの影響を考慮すること。
