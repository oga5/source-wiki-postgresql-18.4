pg_get_line_buf

定義

  • 種別: function
  • 定義箇所: src/common/pg_get_line.c:94

目的

既存の StringInfo バッファを再利用して、指定されたストリームから1行を読み込む。

シグネチャまたは形状

bool pg_get_line_buf(FILE *stream, StringInfo buf)

入力と出力

  • **入力**:
  • stream: 読み込み対象の FILE ポインタ。
  • buf: 読み込み先となる StringInfo 構造体へのポインタ。
  • **出力**:
  • bool: 読み込みが成功した場合は true、EOF またはエラーが発生した場合は false を返す。

前提条件と不変条件

  • buf は初期化済みの StringInfo であること。
  • 呼び出しのたびに、buf 内の既存データは破棄される。

アルゴリズムまたは振る舞い

1. resetStringInfo(buf) を呼び出し、バッファの内容をリセットする。 2. pg_get_line_append(stream, buf, NULL) を呼び出し、ストリームから行を読み込む。 3. 読み込み結果（成功/失敗）をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/ts_locale.c
  • src/bin/initdb/initdb.c
  • src/bin/pg_dump/filter.c
  • src/bin/pg_dump/pg_backup_archiver.c
  • src/interfaces/ecpg/test/pg_regress_ecpg.c
  • **呼び出し先**:
  • resetStringInfo (lib/stringinfo.h)
  • pg_get_line_append (src/common/pg_get_line.c)

関連データ構造

  • StringInfo: 動的に拡張可能な文字列バッファを保持する構造体。

失敗モードと境界条件

  • ストリームが EOF に達した場合、または I/O エラーが発生した場合は false を返す。
  • メモリ不足（OOM）が発生した場合は、pg_get_line_append の内部実装に従いエラーが発生する。

保守担当者向けメモ

  • fgets と同様、末尾の改行文字はバッファ内に保持される。
  • 既存のバッファを再利用するため、ループ内で繰り返し読み込みを行う場合にメモリ効率が良い。
