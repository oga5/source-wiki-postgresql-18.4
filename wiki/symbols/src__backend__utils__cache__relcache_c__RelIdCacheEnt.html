RelIdCacheEnt

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/relcache.c:128

目的

リレーションキャッシュ（RelationIdCache）において、リレーションの OID をキーとして、対応するリレーション記述子（Relation）を保持するためのハッシュテーブルエントリ構造体です。

シグネチャまたは形状

typedef struct relidcacheent
{
	Oid			reloid;
	Relation	reldesc;
} RelIdCacheEnt;

入力と出力

  • reloid: ハッシュテーブルの検索キーとなるリレーションの OID。
  • reldesc: キャッシュされたリレーション記述子へのポインタ。

前提条件と不変条件

  • RelationIdCache ハッシュテーブルの各エントリとして使用されます。
  • reldesc が指す Relation 構造体は、キャッシュのライフサイクル管理（参照カウント等）に従います。

アルゴリズムまたは振る舞い

  • hash_search を用いたハッシュテーブル操作において、reloid をキーとして reldesc を取得・格納するために使用されます。
  • キャッシュミス時には、この構造体を介して新しいリレーション記述子がハッシュテーブルに登録されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/relcache.c 内の RelationIdGetRelation、RelationBuildDesc、RelationCacheInvalidateEntry 等の関数が、hash_search を通じて本構造体を操作します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • RelationIdCache: 本構造体を要素として格納するメインのハッシュテーブル。
  • RelationData (Relation): reldesc が指す実体。

失敗モードと境界条件

  • メモリ不足時: hash_search が失敗する可能性があります。
  • キャッシュ無効化時: reldesc が指すリレーションが古くなった場合、RelationCacheInvalidateEntry によってエントリが処理されます。

保守担当者向けメモ

  • 本構造体は relcache.c 内のハッシュテーブル専用のラッパーです。構造体のフィールドを変更する場合は、RelationIdCache を操作するすべての hash_search 呼び出し箇所に影響が及ぶため注意してください。
