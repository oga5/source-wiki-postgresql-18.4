check_amproc_signature

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/amvalidate.c:151

目的

インデックスのサポート関数が、アクセスメソッドの要件とするシグネチャ（戻り値型、引数の数、引数の型）に適合しているかを検証する。

シグネチャまたは形状

bool check_amproc_signature(Oid funcid, Oid restype, bool exact, int minargs, int maxargs, ...)

入力と出力

  • **入力**:
  • funcid: 検証対象の関数OID。
  • restype: 期待される戻り値の型OID。
  • exact: true の場合、引数型の一致に厳密な型比較を行う。false の場合、バイナリ互換性（IsBinaryCoercible）を許容する。
  • minargs, maxargs: 期待される引数の数の範囲。
  • ...: 可変長引数として、期待される各引数の型OIDを渡す。
  • **出力**:
  • bool: すべての条件を満たす場合に true、それ以外は false を返す。

前提条件と不変条件

  • funcid はシステムカタログ pg_proc に存在しなければならない。存在しない場合、elog(ERROR) により処理が中断される。
  • 可変長引数で指定された型リストは、maxargs の数と一致している必要がある。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用して pg_proc から関数定義を取得する。 2. 戻り値の型が restype と一致し、かつ proretset（集合を返す関数）ではないことを確認する。 3. 引数の数が minargs 以上 maxargs 以下であることを確認する。 4. 可変長引数で指定された各型と、実際の関数の引数型を比較する。exact フラグに応じて、厳密な一致またはバイナリ互換性を判定する。 5. ReleaseSysCache を呼び出してキャッシュエントリを解放し、判定結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blvalidate.c
  • src/backend/access/brin/brin_validate.c
  • src/backend/access/gin/ginvalidate.c
  • src/backend/access/gist/gistvalidate.c
  • **呼び出し先**:
  • SearchSysCache1 / ReleaseSysCache (システムカタログ検索)
  • IsBinaryCoercible (型互換性チェック)
  • elog (エラー報告)

関連データ構造

  • Form_pg_proc: pg_proc カタログの行構造体。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • funcid が無効な場合、キャッシュルックアップ失敗としてエラーが発生する。
  • 引数の数が minargs 未満または maxargs を超える場合、false を返す。
  • 戻り値が集合（proretset が true）である場合、false を返す。

保守担当者向けメモ

  • 本関数は va_list を使用して可変長引数を処理するため、呼び出し側で渡す引数の数と maxargs の整合性に注意すること。
  • exact フラグの切り替えにより、型チェックの厳密さが変わるため、アクセスメソッドごとの要件に合わせて適切に指定すること。
