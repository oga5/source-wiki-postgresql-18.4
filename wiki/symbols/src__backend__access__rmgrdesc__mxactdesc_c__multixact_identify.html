multixact_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/mxactdesc.c:83

目的

マルチトランザクション（MultiXact）に関連するWALレコードの info ビットを解析し、人間が判読可能なレコードタイプ名（文字列）を返します。主に pg_waldump 等のログ解析ツールで使用されます。

シグネチャまたは形状

const char *multixact_identify(uint8 info);

入力と出力

  • **入力**: info (uint8) - WALレコードの info フィールド。
  • **出力**: const char * - レコードタイプに対応する識別文字列。該当しない場合は NULL を返します。

前提条件と不変条件

  • 入力された info ビットから XLR_INFO_MASK を除外した値が、定義済みのマルチトランザクションWALレコードタイプと一致する必要があります。

アルゴリズムまたは振る舞い

1. info 引数に対してビット単位の AND 演算を行い、XLR_INFO_MASK をマスクして不要なフラグを除去します。 2. マスク後の値に基づき switch 文で以下のいずれかに分岐します。

  • XLOG_MULTIXACT_ZERO_OFF_PAGE: "ZERO_OFF_PAGE" を返す。
  • XLOG_MULTIXACT_ZERO_MEM_PAGE: "ZERO_MEM_PAGE" を返す。
  • XLOG_MULTIXACT_CREATE_ID: "CREATE_ID" を返す。
  • XLOG_MULTIXACT_TRUNCATE_ID: "TRUNCATE_ID" を返す。

    • 3. 一致するケースがない場合、初期値である NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h において、RM_MULTIXACT_ID リソースマネージャの識別子関数として登録されています。
  • **呼び出し先**: 本関数は他の関数を呼び出しません。

関連データ構造

  • XLR_INFO_MASK: WALレコードのフラグ部分を分離するために使用される定数。
  • XLOG_MULTIXACT_*: マルチトランザクションの各WALレコードタイプを定義する列挙値。

失敗モードと境界条件

  • info ビットが既知のマルチトランザクションレコードタイプと一致しない場合、関数は NULL を返します。呼び出し側はこの戻り値が NULL である可能性を考慮する必要があります。

保守担当者向けメモ

  • 新しいマルチトランザクションWALレコードタイプが追加された場合は、switch 文に新しい case を追加してください。
  • XLR_INFO_MASK の定義が変更された場合、ビットマスク処理が正しく機能しているか確認してください。
