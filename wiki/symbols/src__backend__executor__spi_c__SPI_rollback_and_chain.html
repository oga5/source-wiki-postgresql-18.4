SPI_rollback_and_chain

定義

  • 種別: function
  • 定義箇所: src/backend/executor/spi.c:418

目的

現在のトランザクションをロールバックし、即座に新しいトランザクションを開始（チェーン）します。

シグネチャまたは形状

void SPI_rollback_and_chain(void)

入力と出力

  • 入力: なし
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • SPI コンテキスト内での呼び出しを前提とします。
  • トランザクション制御が許可されているコンテキスト（例: 非アトミックなプロシージャ内）である必要があります。

アルゴリズムまたは振る舞い

1. 内部関数 _SPI_rollback(true) を呼び出します。 2. 引数 true により、ロールバック後に新しいトランザクションを開始するチェーン処理が実行されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpgsql/src/pl_exec.c:4983
  • 呼び出し先: _SPI_rollback（内部関数）

関連データ構造

  • _SPI_stack: SPI 呼び出しのネストを管理する内部スタック（間接的に参照）。

失敗モードと境界条件

  • トランザクション制御が許可されていないコンテキストで呼び出された場合、エラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は _SPI_rollback のラッパーです。トランザクションの連鎖（chaining）に関する詳細なロジックは _SPI_rollback 側に依存しています。
