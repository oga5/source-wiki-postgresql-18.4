xl_xact_dbinfo

定義

  • 種別: struct
  • 定義箇所: src/include/access/xact.h:255

目的

トランザクションのWAL（Write-Ahead Logging）レコードにおいて、データベース識別子（dbId）とテーブルスペース識別子（tsId）を保持するために使用されます。

シグネチャまたは形状

typedef struct xl_xact_dbinfo
{
    Oid         dbId;           /* MyDatabaseId */
    Oid         tsId;           /* MyDatabaseTableSpace */
} xl_xact_dbinfo;

入力と出力

  • **入力**: トランザクションのコミットやアボート処理時に、現在のデータベースおよびテーブルスペースのOIDが本構造体に格納されます。
  • **出力**: WALレコードの一部としてディスクに書き込まれ、リカバリ時に xactdesc.c 等で読み取られます。

前提条件と不変条件

  • WALレコードの XINFO_HAS_DBINFO フラグが設定されている場合にのみ、本構造体がWALデータ内に存在することが期待されます。

アルゴリズムまたは振る舞い

  • トランザクション終了処理において、xl_xact_dbinfo 型の変数が初期化され、WALレコードのペイロードとしてシリアライズされます。
  • WAL解析時（xactdesc.c）、データポインタから本構造体へキャストして各OIDを取得します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c 内のトランザクションログ記録処理（XactLogCommitRecord 等）から参照されます。
  • **呼び出し先**: src/backend/access/rmgrdesc/xactdesc.c 内のWALレコード解析・記述処理において、データポインタのオフセット計算および値の抽出に使用されます。

関連データ構造

  • xl_xact_xinfo: WALレコードのメタデータフラグを保持し、xl_xact_dbinfo の有無を制御します。

失敗モードと境界条件

  • WALレコードの解析時に、データポインタの整合性が取れていない場合、キャスト後の dbId や tsId が不正な値となる可能性があります。

保守担当者向けメモ

  • 本構造体は可変長データを含むWALレコードの一部として扱われるため、解析時は必ず XINFO_HAS_DBINFO フラグを確認し、sizeof(xl_xact_dbinfo) を用いてポインタを適切に進める必要があります。
