tsm_system_handler

定義

  • 種別: function
  • 定義箇所: src/backend/access/tablesample/system.c:66

目的

TABLESAMPLE SYSTEM メソッドを実装するための TsmRoutine 構造体を初期化し、PostgreSQL のサンプリングシステムへ登録するエントリポイントとして機能します。

シグネチャまたは形状

Datum tsm_system_handler(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（標準的な PostgreSQL 関数呼び出し引数）。
  • **出力**: TsmRoutine 構造体へのポインタを Datum 型として返します。

前提条件と不変条件

  • makeNode(TsmRoutine) によりメモリが確保される必要があります。
  • 返される TsmRoutine は、TABLESAMPLE SYSTEM に必要なメソッド（NextSampleBlock や NextSampleTuple 等）が正しく設定されている必要があります。

アルゴリズムまたは振る舞い

1. TsmRoutine 構造体を動的に割り当てます。 2. パラメータ型として FLOAT4OID を設定します。 3. サンプリングの再現性（repeatable_across_queries および repeatable_across_scans）を true に設定します。 4. system.c 内の各関数（system_samplescangetsamplesize, system_initsamplescan, system_beginsamplescan, system_nextsampleblock, system_nextsampletuple）を構造体の関数ポインタに割り当てます。 5. EndSampleScan は NULL に設定されます。 6. 構築した構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は特定されていません。
  • **呼び出し先**:
  • makeNode
  • list_make1_oid
  • system_samplescangetsamplesize
  • system_initsamplescan
  • system_beginsamplescan
  • system_nextsampleblock
  • system_nextsampletuple

関連データ構造

  • TsmRoutine: サンプリングメソッドのインターフェースを定義する構造体。

失敗モードと境界条件

  • メモリ不足時には makeNode がエラーを発生させる可能性があります。
  • EndSampleScan が NULL であるため、スキャン終了時に特別なクリーンアップ処理は実行されません。

保守担当者向けメモ

  • 本関数は TABLESAMPLE SYSTEM のインターフェース定義を担います。メソッドの追加や変更を行う場合は、本関数内の関数ポインタ割り当てを更新してください。
  • PG_FUNCTION_ARGS を使用しているため、PostgreSQL の関数マネージャ経由で呼び出されることを前提としています。
