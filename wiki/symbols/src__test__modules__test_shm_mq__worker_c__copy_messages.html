copy_messages

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_shm_mq/worker.c:175

目的

共有メモリメッセージキュー（shm_mq）からメッセージを受信し、別のキューへ転送するループ処理を実行します。

シグネチャまたは形状

static void copy_messages(shm_mq_handle *inqh, shm_mq_handle *outqh)

入力と出力

  • **入力**:
  • inqh: 受信元メッセージキューのハンドル。
  • outqh: 送信先メッセージキューのハンドル。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • inqh および outqh は、呼び出し前に shm_mq_attach 等で適切に初期化されている必要があります。
  • 本関数はループ内で CHECK_FOR_INTERRUPTS() を呼び出すため、PostgreSQLのバックグラウンドワーカー環境下で実行されることを前提としています。

アルゴリズムまたは振る舞い

1. 無限ループを開始し、CHECK_FOR_INTERRUPTS() を実行してシグナル（SIGTERM等）を監視します。 2. shm_mq_receive を呼び出し、入力キューからメッセージを取得します。 3. 受信結果が SHM_MQ_SUCCESS でない場合、ループを終了します。 4. shm_mq_send を呼び出し、取得したデータを送信先キューへ書き出します。 5. 送信結果が SHM_MQ_SUCCESS でない場合、ループを終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_shm_mq/worker.c 内のメインロジックから呼び出されます。
  • **呼び出し先**:
  • CHECK_FOR_INTERRUPTS()
  • shm_mq_receive
  • shm_mq_send

関連データ構造

  • shm_mq_handle: メッセージキューの接続状態を保持する構造体。
  • shm_mq_result: メッセージキュー操作の結果ステータス。

失敗モードと境界条件

  • shm_mq_receive または shm_mq_send が SHM_MQ_SUCCESS 以外の値を返した場合、ループは即座に終了します。これはキューの切断やエラー発生時を想定した挙動です。

保守担当者向けメモ

  • 本関数はワーカープロセスのデータ転送ロジックの中核です。計算処理を追加する場合は、shm_mq_receive と shm_mq_send の間にロジックを挿入してください。
  • ループ内の CHECK_FOR_INTERRUPTS() は、プロセスが適切に終了シグナルに応答するために不可欠です。削除しないでください。
