ntile_context

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/windowfuncs.c:32

目的

ウィンドウ関数 NTILE の計算において、パーティションごとの現在のバケット状態および進行状況を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    int32   ntile;           /* 現在の計算結果（バケット番号） */
    int64   rows_per_bucket; /* 現在のバケットに含まれる行数 */
    int64   boundary;        /* バケットに割り当てるべき目標行数 */
    int64   remainder;       /* 全行数 % バケット数（余り） */
} ntile_context;

入力と出力

  • 入力: WinGetPartitionLocalMemory を介してウィンドウ関数実行エンジンから確保・初期化されます。
  • 出力: NTILE 関数の計算過程で各フィールドが更新され、最終的に現在の行に対するバケット番号が算出されます。

前提条件と不変条件

  • ウィンドウ関数実行時に、パーティションごとにメモリが確保される必要があります。
  • remainder は、パーティション内の全行を均等にバケット分割した際の余りとして計算され、最初の remainder 個のバケットには 1 行多く割り当てられるロジックの基準となります。

アルゴリズムまたは振る舞い

1. 初期化時に WinGetPartitionLocalMemory を通じてコンテキストが割り当てられます。 2. 各行の処理において、現在の行位置と boundary、remainder を比較し、必要に応じて ntile（バケット番号）をインクリメントします。 3. rows_per_bucket を更新し、現在のバケットの境界を追跡します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/windowfuncs.c 内の NTILE 関数実装（window_ntile 等）。
  • 呼び出し先: WinGetPartitionLocalMemory（windowapi.h）を呼び出し、パーティション固有のメモリ領域を確保します。

関連データ構造

  • windowapi.h: ウィンドウ関数実行用APIを提供し、本構造体のメモリ管理の基盤となります。

失敗モードと境界条件

  • パーティションが空の場合や、NTILE の引数（バケット数）が 0 または負の場合の挙動については、本構造体定義からは直接読み取れません。
  • メモリ確保失敗時は WinGetPartitionLocalMemory の仕様に従い、エラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は windowfuncs.c 内の NTILE 関数専用のステート管理用です。
  • rows_per_bucket や remainder の計算ロジックを変更する場合は、NTILE の仕様（余りの分配ルール）と整合しているか確認してください。
