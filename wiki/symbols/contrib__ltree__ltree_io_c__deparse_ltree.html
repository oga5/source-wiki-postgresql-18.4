deparse_ltree

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_io.c:144

目的

内部バイナリ形式の ltree 構造体を、人間が読み取り可能なドット区切りの文字列形式に変換（デシリアライズ）します。

シグネチャまたは形状

static char *deparse_ltree(const ltree *in)

入力と出力

  • **入力**: const ltree *in - 変換対象の ltree 構造体へのポインタ。
  • **出力**: char * - 変換後の文字列を格納したメモリ領域へのポインタ。呼び出し元で解放が必要な palloc 済みメモリを返します。

前提条件と不変条件

  • 入力 in は有効な ltree 構造体である必要があります。
  • VARSIZE(in) を使用してメモリを確保するため、入力データが varlena 形式の正しいサイズ情報を持っていることが前提です。

アルゴリズムまたは振る舞い

1. VARSIZE(in) に基づいて、結果文字列を格納するためのメモリを palloc で確保します。 2. LTREE_FIRST(in) を使用して、ltree 内の最初のレベル構造体を取得します。 3. in->numlevel 回のループ処理を行い、各レベルのラベル名（name）をバッファにコピーします。 4. 2番目以降のレベルについては、ラベルの前にドット（.）を挿入して階層を表現します。 5. LEVEL_NEXT マクロを使用して次のレベルへ順次ポインタを進めます。 6. 最後に終端文字 \0 を付与して文字列を完成させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • ltree_out 関数（contrib/ltree/ltree_io.c 内）から呼び出されます。
  • **呼び出し先**:
  • palloc（PostgreSQL メモリ管理）
  • memcpy（標準 C ライブラリ）
  • VARSIZE, LTREE_FIRST, LEVEL_NEXT（ltree.h で定義されたマクロ）

関連データ構造

  • ltree: 階層パスを保持する可変長構造体。
  • ltree_level: ltree 内の各ノード（ラベル）を表現する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • numlevel が 0 の場合、空の文字列（\0 のみ）が返されます。

保守担当者向けメモ

  • 本関数は ltree のシリアライズ専用です。lquery の文字列変換には deparse_lquery が別途存在するため、混同しないよう注意してください。
  • palloc されたメモリは呼び出し元で適切に管理されることを前提としています。
