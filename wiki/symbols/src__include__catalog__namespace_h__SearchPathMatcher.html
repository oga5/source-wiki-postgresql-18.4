SearchPathMatcher

定義

  • **種別**: struct
  • **定義箇所**: src/include/catalog/namespace.h:59

目的

現在のセッションにおけるスキーマ検索パス環境をキャッシュし、環境設定が変更されたかどうかを高速に判定するための構造体です。

シグネチャまたは形状

typedef struct SearchPathMatcher
{
	List	   *schemas;		/* 明示的に指定されたスキーマのOIDリスト */
	bool		addCatalog;		/* pg_catalogを暗黙的に先頭に追加するか */
	bool		addTemp;		/* 一時スキーマを暗黙的に先頭に追加するか */
	uint64		generation;		/* アクティブな環境との等価性を高速判定するための世代番号 */
} SearchPathMatcher;

入力と出力

  • **入力**: GetSearchPathMatcher 関数により、現在のメモリコンテキストに基づいて生成されます。
  • **出力**: SearchPathMatchesCurrentEnvironment 関数において、現在の検索パス環境と一致するかどうかの判定結果（bool）を提供します。

前提条件と不変条件

  • generation フィールドは、検索パス環境の変更を追跡するための識別子として機能します。
  • schemas リストには、検索対象となるスキーマのOIDが格納されます。

アルゴリズムまたは振る舞い

1. **キャッシュ生成**: GetSearchPathMatcher を通じて、現在の検索パス状態を構造体に保持します。 2. **環境比較**: SearchPathMatchesCurrentEnvironment を呼び出すことで、現在のセッションの検索パス設定がキャッシュ時点から変更されていないかを generation 等を用いて比較します。 3. **複製**: CopySearchPathMatcher を使用して、既存の構造体を別のメモリ領域へ複製可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/namespace.c 内の検索パス管理ロジックから呼び出されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出し先はありません。
  • **関連関数**: GetSearchPathMatcher, CopySearchPathMatcher, SearchPathMatchesCurrentEnvironment が本構造体を操作します。

関連データ構造

  • List: schemas フィールドで使用されるPostgreSQLのリスト構造。
  • MemoryContext: GetSearchPathMatcher の引数として使用されます。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性があります。
  • schemas リストが空の場合、暗黙的なスキーマ（addCatalog, addTemp）のみが検索対象となります。

保守担当者向けメモ

  • 本構造体は頻繁に参照されるため、generation による比較ロジックの変更はパフォーマンスに直結します。
  • src/backend/catalog/namespace.c に実装されている関連関数とセットで管理してください。
