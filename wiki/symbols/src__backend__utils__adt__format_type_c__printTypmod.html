printTypmod

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/format_type.c:370

目的

データ型名に対して、型修飾子（typmod）を付与した文字列を生成します。

シグネチャまたは形状

char *printTypmod(const char *typname, int32 typmod, Oid typmodout)

入力と出力

  • **入力**:
  • typname: 型名（文字列）。
  • typmod: 型修飾子（int32）。
  • typmodout: 型固有の修飾子出力関数のOID。
  • **出力**:
  • 型名と修飾子を結合した文字列（palloc されたメモリ領域）。

前提条件と不変条件

  • typmod は -1 であってはなりません（Assert(typmod >= 0) により強制）。

アルゴリズムまたは振る舞い

1. typmodout が InvalidOid の場合:

  • デフォルトの挙動として、typname(typmod) の形式で文字列を生成します。

    • 2. typmodout が有効なOIDの場合:

  • 指定された typmodout 関数を呼び出し、typmod を文字列に変換します。
  • typname と変換された文字列を連結して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/format_type.c 内の各箇所（bit, character, numeric, interval, time, timestamp 等の型名フォーマット処理）。
  • **呼び出し先**:
  • OidFunctionCall1: typmodout 関数を実行するために使用。
  • psprintf: 文字列の組み立てに使用。

関連データ構造

  • Oid: 型や関数の識別子。
  • int32: typmod の型。

失敗モードと境界条件

  • typmod が -1 の場合、アサーション違反により停止します。
  • typmodout 関数が失敗した場合、その関数のエラーハンドリングに従います。

保守担当者向けメモ

  • 本関数は format_type_extended 等から呼び出され、SQL標準に準拠した型定義の出力に使用されます。
  • 新しいデータ型を追加し、独自の typmod 表現が必要な場合は、pg_type カタログの typmodout 列に適切な関数を設定してください。
