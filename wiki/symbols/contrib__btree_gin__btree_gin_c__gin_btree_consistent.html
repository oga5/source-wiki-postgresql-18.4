gin_btree_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gin/btree_gin.c:174

目的

GINインデックスにおいて、検索条件に対するインデックスエントリの一貫性を判定するサポート関数です。

シグネチャまたは形状

Datum gin_btree_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介してGINエンジンから渡される検索コンテキスト。
  • **出力**: Datum 型として true を返します。また、引数 recheck（第6引数）を false に設定します。

前提条件と不変条件

  • 本関数は常に true を返し、再チェック（recheck）を不要とみなします。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(5) を使用して、呼び出し元から渡された recheck フラグのポインタを取得します。 2. *recheck に false を代入します。これは、インデックスエントリが検索条件を完全に満たしていることを示します。 3. PG_RETURN_BOOL(true) を実行し、検索条件に適合したことを通知して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: GINインデックスの検索エンジン（直接の呼び出し元情報はソースコード上では明示されていません）。
  • **呼び出し先**: なし。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約に基づく引数コンテナ。

失敗モードと境界条件

  • 供給されたソースコード上では、特定の失敗モードや境界条件は定義されていません。

保守担当者向けメモ

  • 本関数は、GINインデックス上でB-tree的な比較を行う際、インデックスエントリが条件を満たすかどうかの判定を簡略化しています。再チェックが不要であるという前提が、将来的なインデックス構造の変更によって覆されないか注意してください。
