PLy_spi_execute_fetch_result

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_spi.c:335

目的

PostgreSQL の SPI (Server Programming Interface) で実行されたクエリの結果セット（SPITupleTable）を、Python の PLyResultObject に変換して返却する。

シグネチャまたは形状

PyObject *PLy_spi_execute_fetch_result(SPITupleTable *tuptable, uint64 rows, int status)

入力と出力

  • **入力**:
  • tuptable: SPI 実行結果のタプルテーブル。
  • rows: 処理された行数。
  • status: SPI 実行のステータスコード。
  • **出力**:
  • 変換後の PyObject*（PLyResultObject 型）。失敗時は NULL を返す。

前提条件と不変条件

  • rows が PY_SSIZE_T_MAX を超える場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラーが発生する。
  • tuptable が存在する場合、処理終了後に SPI_freetuptable(tuptable) が呼び出され、メモリが解放される。
  • result->tupdesc は TopMemoryContext にコピーされ、SPI コンテキスト終了後も生存する。

アルゴリズムまたは振る舞い

1. PLy_result_new() で新しい結果オブジェクトを生成する。 2. status を Python オブジェクトに変換し、結果オブジェクトに格納する。 3. tuptable が存在する場合、一時的なメモリコンテキストを作成し、各タプルを Python オブジェクトへ変換してリストに格納する。 4. PG_TRY ブロック内でタプルの変換を行い、エラー発生時は PG_CATCH で一時コンテキストの削除と結果オブジェクトの解放を行う。 5. 成功時は SPI_freetuptable を呼び出し、作成した一時コンテキストを削除して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_spi.c 内のクエリ実行関数（PLy_spi_execute 等）。
  • **呼び出し先**: PLy_result_new, SPI_freetuptable, PyList_New, PLy_input_from_tuple, CreateTupleDescCopy 等。

関連データ構造

  • SPITupleTable: PostgreSQL SPI の結果セット構造体。
  • PLyResultObject: Python 側でクエリ結果を保持するオブジェクト。
  • PLyExecutionContext: 現在の実行コンテキスト。

失敗モードと境界条件

  • PLy_result_new() が失敗した場合、tuptable を解放して NULL を返す。
  • rows が Python のリストサイズ制限（PY_SSIZE_T_MAX）を超える場合、PostgreSQL エラーとして報告される。
  • PyList_New() 等の Python API が失敗した場合、結果オブジェクトを解放し NULL を返す。

保守担当者向けメモ

  • 本関数は PG_TRY / PG_CATCH を使用しており、PostgreSQL のエラー発生時のメモリリークを防ぐため、PG_CATCH ブロック内での MemoryContextDelete と Py_DECREF が必須である。
  • tupdesc のコピーは TopMemoryContext に対して行われるため、PLy_result_dealloc による適切なクリーンアップが期待されている。
