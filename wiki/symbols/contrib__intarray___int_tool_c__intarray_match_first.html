intarray_match_first

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_tool.c:337

目的

PostgreSQL の整数配列（ArrayType）内を先頭から走査し、指定された値と一致する最初の要素の「1から始まるインデックス」を返します。

シグネチャまたは形状

int32 intarray_match_first(ArrayType *a, int32 elem)

入力と出力

  • **入力**:
  • a: 検索対象の ArrayType ポインタ。
  • elem: 検索する整数値。
  • **出力**:
  • 一致する要素が見つかった場合: その要素のインデックス（1-based index）。
  • 一致する要素が見つからない場合: 0。

前提条件と不変条件

  • CHECKARRVALID(a) マクロにより、入力された a が有効な整数配列構造であることが保証されている必要があります。

アルゴリズムまたは振る舞い

1. ARRNELEMS(a) を用いて配列内の要素数を取得します。 2. ARRPTR(a) を用いて配列のデータ領域へのポインタを取得します。 3. 配列の先頭から順に要素を比較します。 4. elem と一致する値が見つかった時点で、その位置のインデックスに 1 を加算して即座に返します。 5. 配列の末尾まで一致するものがない場合は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_op.c:270: SQL 演算子の実装内で呼び出されています。
  • **呼び出し先**:
  • PostgreSQL のマクロ CHECKARRVALID, ARRNELEMS, ARRPTR を使用しています。

関連データ構造

  • ArrayType: PostgreSQL の標準的な配列型構造体。

失敗モードと境界条件

  • 入力配列が空の場合、ループは実行されず 0 を返します。
  • 配列内に重複する値が存在する場合、最も先頭に近い（インデックスが小さい）要素の位置を返します。

保守担当者向けメモ

  • 本関数は線形探索（$O(N)$）を行います。ソート済み配列を前提とした他の inner_int_ 系関数とは異なり、順序を問わず動作しますが、大規模な配列に対して頻繁に呼び出す場合はパフォーマンスに注意してください。
