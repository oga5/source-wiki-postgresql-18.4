CommitTimestampEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/commit_ts.c:54

目的

トランザクションのコミット時刻およびレプリケーション元ノードIDをSLRU（Simple Least Recently Used）ページ内に格納するためのデータ構造です。

シグネチャまたは形状

typedef struct CommitTimestampEntry
{
	TimestampTz time;
	RepOriginId nodeid;
} CommitTimestampEntry;

入力と出力

  • **入力**: 構造体メンバとして TimestampTz 型のコミット時刻と RepOriginId 型のノードIDを保持します。
  • **出力**: 本構造体はSLRUページ内のデータとして書き込まれ、TransactionIdGetCommitTsData 等のAPIを通じて検索結果として返されます。

前提条件と不変条件

  • 本構造体は track_commit_timestamp GUCパラメータが有効な場合にのみ使用されます。
  • SLRUページ内に配置されるため、メモリ配置およびアライメントはSLRUの管理規則に従います。

アルゴリズムまたは振る舞い

  • TransactionTreeSetCommitTsData 等の関数により、トランザクションのコミット時に本構造体のインスタンスが生成・更新され、SLRUバッファへ書き込まれます。
  • SizeOfCommitTimestampEntry マクロ（offsetof を使用）により、構造体サイズが計算され、SLRU上のオフセット管理に利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/commit_ts.c 内の関数（TransactionTreeSetCommitTsData, TransactionIdSetCommitTs 等）が本構造体をインスタンス化し、SLRUへの書き込みを行います。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • CommitTsCtl: 本構造体を格納するSLRUの制御構造体。
  • TimestampTz: PostgreSQLのタイムスタンプ型。
  • RepOriginId: レプリケーション元の識別子。

失敗モードと境界条件

  • **境界条件**: SizeOfCommitTimestampEntry マクロによるサイズ計算において、構造体のパディングが考慮されている必要があります。
  • **失敗モード**: SLRUバッファの不足やディスクI/Oエラーが発生した場合、本構造体の読み書きが失敗し、トランザクションのコミット情報取得が不可能になる可能性があります。

保守担当者向けメモ

  • 本構造体の定義を変更する場合、SizeOfCommitTimestampEntry マクロの定義も併せて確認してください。
  • SLRUページ内のデータレイアウトに直接影響するため、バイナリ互換性に注意が必要です。
