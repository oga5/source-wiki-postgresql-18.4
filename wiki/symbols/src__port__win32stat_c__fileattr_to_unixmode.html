fileattr_to_unixmode

定義

  • 種別: function
  • 定義箇所: src/port/win32stat.c:47

目的

Windows のファイル属性（dwFileAttributes）を、PostgreSQL が内部的に利用する POSIX 準拠の st_mode（unsigned short）へ変換します。

シグネチャまたは形状

static unsigned short fileattr_to_unixmode(int attr);

入力と出力

  • **入力**: int attr - Windows API から取得したファイル属性ビットマスク。
  • **出力**: unsigned short - POSIX 互換のファイルモードビット。

前提条件と不変条件

  • 入力 attr は Windows の GetFileInformationByHandle 等で得られるファイル属性ビットであることを想定しています。
  • 戻り値の st_mode は、所有者の権限のみを考慮して生成されます。グループや他人の権限ビットは設定されません。

アルゴリズムまたは振る舞い

1. **ファイルタイプ判定**: FILE_ATTRIBUTE_DIRECTORY が含まれていれば _S_IFDIR を、そうでなければ _S_IFREG を設定します。 2. **読み書き権限の判定**: FILE_ATTRIBUTE_READONLY が含まれていれば _S_IREAD を設定し、含まれていなければ _S_IREAD | _S_IWRITE を設定します。 3. **実行権限の付与**: 無条件に _S_IEXEC を付与します（Windows の PATHEXT 等による複雑な実行権限シミュレーションは行いません）。 4. 構築したビットマスクを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/win32stat.c 内の st_mode 設定箇所（例: buf->st_mode = fileattr_to_unixmode(fiData.dwFileAttributes);）。
  • **呼び出し先**: 直接的な関数呼び出しはありません。標準の C マクロ（_S_IFDIR, _S_IFREG, _S_IREAD, _S_IWRITE, _S_IEXEC）を利用します。

関連データ構造

  • struct stat（st_mode フィールド）

失敗モードと境界条件

  • 本関数自体はエラーを返しません。
  • 入力 attr に予期しないビットが含まれていても、定義済みのビットマスク操作のみを行うため、安全に処理されます。

保守担当者向けメモ

  • 本関数は所有者の権限のみを考慮する設計です。より詳細なアクセス制御が必要な場合は、win32security.c 等のセキュリティ記述子を扱う関数群との連携を検討してください。
