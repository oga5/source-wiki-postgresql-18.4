DbInfoArr

定義

  • **種別**: struct
  • **定義箇所**: src/bin/pg_upgrade/pg_upgrade.h:218

目的

アップグレード対象となるデータベース情報の配列を保持し、クラスタ内の全データベースに関するメタデータを一括管理するために使用されます。

シグネチャまたは形状

typedef struct
{
	DbInfo	   *dbs;			/* array of db infos */
	int			ndbs;			/* number of db infos */
} DbInfoArr;

入力と出力

  • **入力**: DbInfo 構造体の配列（dbs）およびその要素数（ndbs）。
  • **出力**: 本構造体自体が、データベース情報のコンテナとして機能します。

前提条件と不変条件

  • dbs は DbInfo 構造体の配列を指すポインタである必要があります。
  • ndbs は配列内の有効な要素数と一致している必要があります。

アルゴリズムまたは振る舞い

本構造体は、pg_upgrade における旧クラスタおよび新クラスタのデータベース情報を保持するデータストアとして機能します。info.c 等のモジュールによって構築・参照され、アップグレード処理の各フェーズでデータベース単位の操作を行う際の基盤となります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/parallel.c: parallel_transfer_all_new_dbs 等で旧・新クラスタのデータベース情報として使用。
  • src/bin/pg_upgrade/pg_upgrade.h: ClusterInfo 構造体の一部として定義。
  • **呼び出し先**:
  • src/bin/pg_upgrade/info.c: free_db_and_rel_infos（メモリ解放）、print_db_infos（情報出力）にて引数として受け取り処理されます。

関連データ構造

  • DbInfo: 個々のデータベース情報を保持する構造体。
  • ClusterInfo: DbInfoArr を内包し、クラスタ全体の情報を管理する構造体。

失敗モードと境界条件

  • dbs が NULL の場合や、ndbs が 0 の場合のメモリ管理（解放処理など）において、呼び出し側の実装が適切に処理を行う必要があります。

保守担当者向けメモ

  • 本構造体は pg_upgrade の中核的なデータ保持構造の一つです。info.c での構築ロジックに変更を加える際は、parallel.c 等での参照整合性に注意してください。
