oidKEY

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_oid.c:10

目的

PostgreSQLのGiSTインデックスにおいて、Oid（Object Identifier）型の値の範囲（最小値と最大値）を保持し、インデックスノード内で表現するために使用されます。

シグネチャまたは形状

typedef struct
{
	Oid			lower;
	Oid			upper;
} oidKEY;

入力と出力

  • **入力**: Oid 型の lower および upper 値。
  • **出力**: GiSTインデックスのノードに格納される構造体としてのデータ。

前提条件と不変条件

  • リーフノードにおいては、lower == upper である必要があります。

アルゴリズムまたは振る舞い

  • Oid 型の値を範囲として管理し、GiSTインデックスの検索や圧縮処理において、クエリ条件との適合性を判定するための基盤となります。
  • 内部的には btree_utils_num.h で定義された数値型用の汎用GiSTユーティリティ関数によって操作されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_enum.c 内の複数の箇所で、oidKEY 型へのキャストやメモリ確保、ポインタ経由でのアクセスが行われています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。操作ロジックは btree_utils_num.c 等のユーティリティ関数に委譲されています。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子型。
  • GISTENTRY: GiSTインデックスの各エントリを保持する構造体。

失敗モードと境界条件

  • lower > upper となるような不正な範囲設定は、インデックスの整合性を損なう可能性があります。
  • メモリ割り当て時（palloc）の失敗は、PostgreSQLの標準的なエラー処理に従います。

保守担当者向けメモ

  • 本構造体は Oid を数値として扱うためのアダプターです。複雑な比較ロジックやGiST特有の動作については、contrib/btree_gist/btree_utils_num.c を参照してください。
