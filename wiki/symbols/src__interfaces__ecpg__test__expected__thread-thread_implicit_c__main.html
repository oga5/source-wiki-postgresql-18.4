main

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/thread-thread_implicit.c:41

目的

ECPG（Embedded SQL for C）のマルチスレッド環境における並行接続およびトランザクション処理の整合性を検証する回帰テストプログラムのエントリポイント。

シグネチャまたは形状

int main()

入力と出力

  • 入力: なし（プログラム実行時に環境変数や引数に依存せず、定数 nthreads および iterations を使用）。
  • 出力: 標準出力にテスト結果（"Success." またはエラーメッセージ）を出力し、終了ステータスとして 0（成功）または 1（メモリ確保失敗時）を返す。

前提条件と不変条件

  • データベース ecpg1_regression が接続可能であること。
  • nthreads 分のスレッドが並行して test_thread テーブルへ挿入処理を行う。
  • 最終的なテーブルの行数が nthreads * iterations と一致することが成功の不変条件である。

アルゴリズムまたは振る舞い

1. **環境準備**: ecpg1_regression に接続し、既存の test_thread テーブルを削除・再作成してコミットする。 2. **スレッド生成**: nthreads 個のスレッドを生成し、各スレッドで test_thread 関数を並行実行する。

  • POSIX環境では pthread_create、Windows環境では CreateThread を使用。

    • 3. **同期**: 全スレッドの終了を待機する（pthread_join または WaitForMultipleObjects）。 4. **検証**: 再接続し、SELECT count(*) FROM test_thread を実行して結果を取得。期待値と比較して結果を判定する。 5. **後処理**: メモリ解放および接続切断を行う。

呼び出し元と呼び出し先

  • 呼び出し元: OSのプロセスローダー。
  • 呼び出し先:
  • ECPGライブラリ関数: ECPGconnect, ECPGdo, ECPGtrans, ECPGdisconnect。
  • スレッド関数: test_thread（各スレッドの実行本体）。
  • 標準ライブラリ: calloc, free, fprintf, printf。
  • OS依存API: pthread_create/pthread_join (POSIX) または CreateThread/WaitForMultipleObjects (Windows)。

関連データ構造

  • pthread_t / HANDLE: スレッドハンドルを格納する配列。
  • l_rows: 挿入された行数を格納するホスト変数（int）。

失敗モードと境界条件

  • calloc によるメモリ確保失敗時にエラーメッセージを出力して終了する。
  • DROP TABLE が失敗する可能性がある（コメントに記載あり）。
  • スレッドの実行順序は不定であり、競合が発生しないことを前提としている。

保守担当者向けメモ

  • 本関数は回帰テスト用であり、デバッグ出力（ECPGdebug）は無効化されている。
  • WindowsとPOSIXでスレッド制御のAPIが分岐しているため、修正時は両方のブロックを確認すること。
  • 期待値の判定は「最終的な行数」のみで行われるため、スレッド間の実行順序に依存しない設計となっている。
