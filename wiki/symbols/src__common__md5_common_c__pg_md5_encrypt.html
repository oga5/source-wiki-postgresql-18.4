pg_md5_encrypt

定義

  • 種別: function
  • 定義箇所: src/common/md5_common.c:144

目的

パスワードとソルトを連結し、md5 プレフィックスを付与したハッシュ文字列を生成します。

シグネチャまたは形状

bool pg_md5_encrypt(const char *passwd, const uint8 *salt, size_t salt_len, char *buf, const char **errstr);

入力と出力

  • **入力**:
  • passwd: パスワード文字列。
  • salt: ソルトデータ。
  • salt_len: ソルトのバイト長。
  • buf: 結果を格納するバッファ（十分なサイズが必要）。
  • errstr: エラー発生時にエラーメッセージを格納するポインタ。
  • **出力**:
  • 成功時に true、失敗時に false を返します。
  • buf には "md5" で始まるハッシュ文字列が書き込まれます。

前提条件と不変条件

  • buf は、md5 プレフィックスとハッシュ値（32文字）を格納するために十分なメモリ領域を確保している必要があります。
  • passwd はヌル終端文字列である必要があります。

アルゴリズムまたは振る舞い

1. passwd の長さを取得し、passwd と salt を連結するためのメモリを malloc で確保します。 2. crypt_buf に passwd をコピーし、その直後に salt を連結します。 3. buf の先頭に "md5" を書き込みます。 4. pg_md5_hash を呼び出し、crypt_buf の内容からハッシュを計算して buf の 4 文字目以降に書き込みます。 5. 確保した crypt_buf を解放し、処理結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/crypt.c
  • src/interfaces/libpq/fe-auth.c
  • **呼び出し先**:
  • pg_md5_hash (計算実行)
  • malloc, free, strlen, memcpy, strcpy (標準ライブラリ)

関連データ構造

  • uint8: ソルトデータに使用される符号なし8ビット整数型。

失敗モードと境界条件

  • malloc が失敗した場合、errstr に "out of memory" を設定し false を返します。
  • pg_md5_hash が失敗した場合、そのエラー状態を伝播し false を返します。

保守担当者向けメモ

  • 内部で malloc を使用するため、呼び出し元はメモリ不足の可能性を考慮する必要があります。
  • セキュリティ上の理由から、MD5は認証目的以外での新規採用を避けるべきです。
  • ソルトはハッシュ計算の末尾に配置される仕様です。
