lo_creat

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-lobj.c:437

目的

データベース内に新しいラージオブジェクトを作成し、そのオブジェクト識別子（OID）を返します。

シグネチャまたは形状

Oid lo_creat(PGconn *conn, int mode);

入力と出力

  • **入力**:
  • conn: データベース接続を管理する PGconn 構造体へのポインタ。
  • mode: ラージオブジェクトのアクセスモード（ビットマスク）。
  • **出力**:
  • 成功時: 作成されたラージオブジェクトの Oid。
  • 失敗時: InvalidOid。

前提条件と不変条件

  • lo_initialize(conn) が成功し、サーバー側のラージオブジェクト関数（fn_lo_creat）のOIDがキャッシュされている必要があります。
  • 呼び出しは現在のトランザクションコンテキスト内で実行されます。

アルゴリズムまたは振る舞い

1. lo_initialize を呼び出し、必要なサーバー関数OIDが利用可能か確認します。失敗した場合は InvalidOid を返します。 2. PQArgBlock を使用して mode を引数として設定します。 3. PQfn（Fastpath関数呼び出し）を介してサーバー側の lo_creat 関数を実行します。 4. サーバーからの応答が PGRES_COMMAND_OK であれば、結果からOIDを取得して返します。 5. 処理終了後、PQclear を呼び出して結果セットを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/examples/testlo.c および testlo64.c
  • src/interfaces/libpq/fe-lobj.c 内の内部処理
  • **呼び出し先**:
  • lo_initialize
  • PQfn
  • PQresultStatus
  • PQclear

関連データ構造

  • PGconn: 接続状態およびキャッシュされた関数OIDを保持。
  • PQArgBlock: サーバーへ渡す引数構造体。
  • PGresult: サーバーからの実行結果を保持。

失敗モードと境界条件

  • lo_initialize が失敗した場合、InvalidOid を返します。
  • サーバー側でエラーが発生した場合、PQresultStatus が PGRES_COMMAND_OK 以外となり、InvalidOid を返します。
  • トランザクションがアボート状態にある場合、操作は失敗します。

保守担当者向けメモ

  • 本関数は libpq のラージオブジェクトAPIの基本機能です。
  • サーバー側の lo_creat 関数は、pg_proc システムカタログから動的に解決されます。
  • 戻り値の Oid は、後続の lo_open 等の操作で使用されます。
