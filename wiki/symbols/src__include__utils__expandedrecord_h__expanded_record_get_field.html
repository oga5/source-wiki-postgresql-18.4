expanded_record_get_field

定義

  • 種別: function
  • 定義箇所: src/include/utils/expandedrecord.h:227

目的

展開されたレコード（ExpandedRecordHeader）から、指定されたフィールド番号の値を Datum として取得する。

シグネチャまたは形状

static inline Datum
expanded_record_get_field(ExpandedRecordHeader *erh, int fnumber, bool *isnull)

入力と出力

  • **入力**:
  • erh: ターゲットとなる展開されたレコードへのポインタ。
  • fnumber: 取得対象のフィールド番号（1始まり）。
  • isnull: フィールドが NULL かどうかを格納するための出力用ポインタ。
  • **出力**:
  • 戻り値: フィールドの値（Datum）。

前提条件と不変条件

  • erh は有効な ExpandedRecordHeader を指している必要がある。
  • fnumber は 1 以上 erh->nfields 以下である必要がある。

アルゴリズムまたは振る舞い

1. erh->flags に ER_FLAG_DVALUES_VALID がセットされており、かつ fnumber が有効範囲内である場合、キャッシュされている erh->dvalues および erh->dnulls 配列から直接値を読み取り、isnull を設定して即座に返す。 2. 上記条件を満たさない場合、expanded_record_fetch_field を呼び出し、必要に応じて物理タプルからのデコード処理を経て値を取得する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c
  • src/pl/plpgsql/src/pl_exec.c
  • **呼び出し先**:
  • expanded_record_fetch_field（キャッシュが無効な場合）

関連データ構造

  • ExpandedRecordHeader: 展開されたレコードのメモリ上での表現。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • fnumber が範囲外の場合、expanded_record_fetch_field 側の実装に依存するが、通常はエラーが発生する可能性がある。
  • ER_FLAG_DVALUES_VALID が未設定の場合、タプルのデコード処理（expanded_record_fetch_field）が実行されるため、処理コストが増大する。

保守担当者向けメモ

  • 本関数は static inline で定義されており、頻繁に呼び出されるフィールドアクセスを最適化する役割を持つ。
  • dvalues 配列の有効性は ER_FLAG_DVALUES_VALID フラグによって管理されているため、このフラグの整合性を維持することが重要である。
