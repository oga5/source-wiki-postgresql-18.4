astreamer_gzip_writer_free

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_gzip.c:200

目的

astreamer_gzip_writer 構造体に関連付けられたメモリリソースを解放し、破棄します。

シグネチャまたは形状

static void astreamer_gzip_writer_free(astreamer *streamer);

入力と出力

  • **入力**: astreamer *streamer - 解放対象のストリーマーオブジェクトへのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • streamer は astreamer_gzip_writer 型としてキャスト可能であること。
  • 呼び出し時点で、後続のストリーマー（bbs_next）および gzip ファイルハンドル（gzfile）が既にクローズまたは解放済みであり、NULL であること（Assert による検証）。

アルゴリズムまたは振る舞い

1. 引数の streamer を astreamer_gzip_writer 型にキャストします。 2. bbs_next および gzfile が NULL であることを確認します。 3. 構造体内部で保持している pathname 文字列を pfree で解放します。 4. 最後に astreamer_gzip_writer 構造体自体を pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/astreamer_gzip.c 内の astreamer インターフェースのメソッドテーブル（.free メンバ）として登録されています。具体的な実行時の呼び出し元については、提供されたソース範囲内では特定できません。
  • **呼び出し先**: pfree（メモリ解放関数）。

関連データ構造

  • astreamer_gzip_writer: 本関数が管理・解放する主要なデータ構造。

失敗モードと境界条件

  • bbs_next または gzfile が NULL でない場合、Assert によりプログラムが異常終了します。これは、リソースの解放順序が正しくない（finalize が適切に呼ばれていない等）ことを示唆します。

保守担当者向けメモ

  • 本関数は astreamer インターフェースの破棄処理の一部です。gzFile のクローズ処理は本関数ではなく、astreamer_gzip_writer_finalize 等の先行する処理で行われる設計となっています。
