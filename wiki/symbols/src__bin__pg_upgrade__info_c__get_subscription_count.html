get_subscription_count

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/info.c:759

目的

指定された PostgreSQL クラスタ内のサブスクリプション数を取得し、ClusterInfo 構造体に格納する。

シグネチャまたは形状

void get_subscription_count(ClusterInfo *cluster)

入力と出力

  • **入力**: ClusterInfo *cluster（対象クラスタ情報へのポインタ）
  • **出力**: なし（副作用として cluster->nsubs を更新する）

前提条件と不変条件

  • cluster 引数は有効な ClusterInfo 構造体を指している必要がある。
  • connectToServer 関数が template1 データベースへの接続を確立できること。

アルゴリズムまたは振る舞い

1. connectToServer を使用して、対象クラスタの template1 データベースに接続する。 2. pg_catalog.pg_subscription テーブルに対して SELECT count(*) クエリを実行する。 3. クエリ結果からサブスクリプション数を取得し、整数値に変換して cluster->nsubs に代入する。 4. PQclear で結果セットを解放し、PQfinish でデータベース接続を閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/check.c:633
  • **呼び出し先**:
  • connectToServer（接続確立）
  • executeQueryOrDie（クエリ実行）
  • PQgetvalue, PQclear, PQfinish（libpq 関数）

関連データ構造

  • ClusterInfo: クラスタ情報を保持する構造体。nsubs メンバにサブスクリプション数が格納される。

失敗モードと境界条件

  • executeQueryOrDie は、クエリ実行に失敗した場合にプログラムを終了させる可能性がある。
  • 接続先のクラスタが pg_subscription テーブルを持たない古いバージョンである場合、クエリ実行時にエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は pg_upgrade におけるサブスクリプション移行の事前チェックや情報収集プロセスの一部として利用される。
  • cluster->nsubs の値は、後のアップグレード処理における論理レプリケーション関連の判定に使用される。
