SyncRepQueueInsert

定義

  • 種別: function
  • 定義箇所: src/backend/replication/syncrep.c:371

目的

同期レプリケーションの待機キュー（SyncRepQueue）に対し、現在のバックエンドプロセス（MyProc）を待機対象として挿入する。キューをLSN（Log Sequence Number）順に維持することで、効率的な待機解除を可能にする。

シグネチャまたは形状

static void SyncRepQueueInsert(int mode)

入力と出力

  • **入力**:
  • mode: 待機モード（NUM_SYNC_REP_WAIT_MODE 未満の整数）。
  • **出力**: なし。副作用として共有メモリ上の待機キューが更新される。

前提条件と不変条件

  • SyncRepLock が保持されている状態で呼び出される必要がある（共有メモリ操作のため）。
  • MyProc->waitLSN が適切に設定されていること。
  • 待機キューは常に waitLSN の昇順でソートされている必要がある。

アルゴリズムまたは振る舞い

1. 指定された mode に対応する WalSndCtl->SyncRepQueue[mode] を取得する。 2. キューを末尾から逆順に走査し、proc->waitLSN < MyProc->waitLSN となる要素を探す。 3. 条件を満たす要素が見つかった場合、その直後に MyProc を挿入する。 4. キューが空、または MyProc が最も小さいLSNを持つ場合は、キューの先頭（dlist_push_head）に挿入する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/syncrep.c 内の SyncRepWaitForLSN から呼び出される。
  • **呼び出し先**: dlist 操作関数（dlist_reverse_foreach, dlist_insert_after, dlist_push_head）。

関連データ構造

  • WalSndCtl: 同期レプリケーションの制御情報を保持する共有メモリ構造体。
  • PGPROC: バックエンドプロセスの状態を保持する構造体。syncRepLinks を介してキューに連結される。
  • dlist: PostgreSQLの二重連結リスト。

失敗モードと境界条件

  • **空のキュー**: キューが空の場合、MyProc が先頭に挿入される。
  • **LSNの最小値**: MyProc のLSNがキュー内のどの要素よりも小さい場合、先頭に挿入される。
  • **アサーション**: mode が有効な範囲外である場合、Assert により停止する。

保守担当者向けメモ

  • 本関数は SyncRepLock の保護下で実行されることを前提としている。ロックの取得漏れがないか注意すること。
  • キューの順序維持は、SyncRepReleaseWaiters がLSNの小さい順に効率よく待機解除を行うために不可欠である。
