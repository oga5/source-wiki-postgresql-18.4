systable_getnext

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/genam.c:513

目的

システムカタログスキャンにおいて、次のタプルを透過的に取得する。インデックスが使用可能な場合はインデックススキャンを、そうでない場合はヒープスキャンを自動的に選択する。

シグネチャまたは形状

HeapTuple systable_getnext(SysScanDesc sysscan)

入力と出力

  • **入力**: SysScanDesc sysscan (スキャン状態を保持する記述子)
  • **出力**: HeapTuple (次のタプル、またはスキャン終了時は NULL)

前提条件と不変条件

  • sysscan は systable_beginscan によって初期化済みであること。
  • 取得したタプルのメモリ管理において、ExecFetchSlotHeapTuple の第2引数は false であり、呼び出し元はタプルの解放（pfree）を行ってはならない。

アルゴリズムまたは振る舞い

1. sysscan->irel が存在する場合（インデックススキャン）:

  • index_getnext_slot を呼び出し、結果を sysscan->slot に格納する。
  • インデックス条件が損失的（lossy）である場合、エラーを発生させる。

    • 2. sysscan->irel が存在しない場合（ヒープスキャン）:

  • table_scan_getnextslot を呼び出し、結果を sysscan->slot に格納する。

    • 3. いずれの場合も、ExecFetchSlotHeapTuple を介してスロットから HeapTuple を抽出する。 4. 最後に HandleConcurrentAbort() を呼び出し、論理ストリーミング中の同時実行アボートを処理する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • contrib/sepgsql/ 配下の各ファイル (database.c, label.c, proc.c, relation.c, schema.c)
  • src/backend/access/common/toast_internals.c
  • **呼び出し先**:
  • index_getnext_slot
  • table_scan_getnextslot
  • ExecFetchSlotHeapTuple
  • HandleConcurrentAbort

関連データ構造

  • SysScanDesc: スキャン状態を管理する構造体。
  • HeapTuple: 返却されるタプル型。
  • TupleTableSlot: スキャン結果の一時保持に使用されるスロット。

失敗モードと境界条件

  • **損失的インデックス条件**: インデックススキャンにおいて xs_recheck が真の場合、システムカタログスキャンではサポートされていないため elog(ERROR) を発生させる。
  • **スキャン終了**: 次のタプルが存在しない場合、NULL を返す。

保守担当者向けメモ

  • 本関数はシステムカタログスキャン専用であり、汎用的なインデックススキャンとは異なる制約（損失的条件の非サポートなど）を持つ。
  • HandleConcurrentAbort() の呼び出しは、論理レプリケーション等の文脈で重要であるため、削除や変更には注意が必要。
