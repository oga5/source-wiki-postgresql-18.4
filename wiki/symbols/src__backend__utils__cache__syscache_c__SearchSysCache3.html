SearchSysCache3

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/syscache.c:242

目的

3つのキーを使用してシステムカタログキャッシュからタプルを検索し、取得します。

シグネチャまたは形状

HeapTuple SearchSysCache3(int cacheId, Datum key1, Datum key2, Datum key3)

入力と出力

  • **入力**:
  • cacheId: 検索対象のキャッシュID（SysCache 配列のインデックス）。
  • key1, key2, key3: 検索に使用する3つのキー値（Datum 型）。
  • **出力**:
  • 成功時: 見つかったタプルへのポインタ（HeapTuple）。
  • 失敗時: NULL（該当するタプルが存在しない場合）。

前提条件と不変条件

  • cacheId が有効な範囲内であり、対応するキャッシュが初期化されていること。
  • 指定されたキャッシュが3つのキーを使用する定義（cc_nkeys == 3）であること。これに違反する場合、アサーションエラーが発生します。

アルゴリズムまたは振る舞い

1. cacheId が有効であること、および対象キャッシュのキー数が3であることを Assert で検証します。 2. SysCache[cacheId] を参照し、SearchCatCache3 を呼び出して実際の検索処理を委譲します。 3. 検索結果として得られた HeapTuple を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/aclchk.c
  • src/backend/catalog/namespace.c
  • src/backend/catalog/objectaddress.c
  • src/backend/catalog/pg_proc.c
  • src/backend/commands/analyze.c
  • src/backend/commands/indexcmds.c
  • src/backend/commands/opclasscmds.c
  • src/backend/commands/user.c
  • src/backend/executor/nodeHash.c
  • **呼び出し先**:
  • SearchCatCache3（src/backend/utils/cache/catcache.c 内の関数）

関連データ構造

  • SysCache[]: システムキャッシュを保持する CatCache 構造体の配列。
  • HeapTuple: 検索結果として返されるタプル構造体。

失敗モードと境界条件

  • cacheId が範囲外の場合、またはキャッシュが初期化されていない場合にアサーション失敗。
  • キャッシュ定義のキー数と関数呼び出しのキー数が一致しない場合にアサーション失敗。
  • 該当するタプルが見つからない場合は NULL を返します。

保守担当者向けメモ

  • 本関数が返すタプルはキャッシュ内部のポインタです。呼び出し側で直接変更してはいけません。変更が必要な場合は SearchSysCacheCopy を使用してください。
  • 新しいキャッシュを追加する場合は syscache_info.h を編集し、キー数が3であることを確認してください。
