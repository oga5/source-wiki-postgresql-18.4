dm_node

定義

  • 種別: struct
  • 定義箇所: contrib/fuzzystrmatch/daitch_mokotoff.c:64

目的

Daitch-Mokotoff Soundex アルゴリズムにおける音韻コード生成過程の状態を保持するツリー構造体です。入力文字列の走査に伴う音韻的な分岐と状態遷移を管理します。

シグネチャまたは形状

typedef struct dm_node {
    int         soundex_length;
    char        soundex[DM_CODE_DIGITS];
    int         is_leaf;
    int         last_update;
    char        code_digit;
    char        prev_code_digits[2];
    char        next_code_digits[2];
    int         prev_code_index;
    int         next_code_index;
    struct dm_node *children[10];
    struct dm_node *next[2];
} dm_node;

入力と出力

  • **入力**: アルゴリズムの各ステップで initialize_node や find_or_create_child_node を介して初期化・更新されます。
  • **出力**: 最終的に is_leaf が真となったノードが、生成された 6 桁の Soundex コードとして蓄積されます。

前提条件と不変条件

  • children[10] は 0-9 のコードに対応する分岐を保持します。
  • prev_code_digits および next_code_digits に 2 つの数字が格納される場合、一方は必ず 'X' です。
  • 重複するコードや 'X' は同一ノードへ戻る構造をとります。

アルゴリズムまたは振る舞い

  • 入力文字列の走査に応じて children を介してツリーを拡張します。
  • next ポインタにより、各イテレーションにおけるノードの連結リストを維持します。
  • 音韻ルールに基づき、update_node や update_leaves を通じて状態を遷移させ、6 桁のコードが完成した時点で is_leaf フラグが立てられます。

呼び出し元と呼び出し先

  • **呼び出し元**: initialize_node, add_next_code_digit, set_leaf, find_or_create_child_node 等の関数がこの構造体を操作します。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • daitch_mokotoff.h: 音韻ルールテーブルを定義しており、ノード生成の基準となります。
  • ArrayBuildState: 最終的な Soundex コードの配列構築に使用されます。

失敗モードと境界条件

  • 入力文字列の長さに対して指数的に分岐が増加する可能性があるため、メモリ消費に注意が必要です。
  • 6 桁の制限に達した後の挙動は is_leaf フラグによって制御されます。

保守担当者向けメモ

  • 本構造体は動的プログラミング的なアプローチで利用されます。children と next ポインタの役割を理解することが、コード生成ロジックを追跡する鍵となります。
  • ルール定義（daitch_mokotoff.h）の変更は、この構造体を通じたツリー構築に直接影響を与えます。
