定義

  • 定義箇所: src/bin/pg_dump/pg_restore.c:58

目的

PostgreSQL のバックアップアーカイブからデータベース定義やデータを復元するコマンドラインユーティリティ pg_restore のメインエントリポイントです。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**: コマンドライン引数 (argc, argv)。アーカイブファイルパスや復元オプション（スキーマ指定、並列ジョブ数など）を含む。
  • **出力**: 復元処理の成否を示す終了コード（成功時は 0、失敗時は非ゼロ）。

前提条件と不変条件

  • pg_logging_init および init_parallel_dump_utils が呼び出され、実行環境が初期化されていること。
  • NewRestoreOptions により復元オプション構造体が適切に割り当てられていること。

アルゴリズムまたは振る舞い

1. **初期化**: ログ設定、ロケール設定、並列処理ユーティリティの初期化を行う。 2. **引数解析**: getopt_long を使用し、コマンドライン引数を解析して RestoreOptions 構造体に反映させる。 3. **検証**: 競合するオプションのチェックを行い、整合性を確保する。 4. **アーカイブ処理**:

  • OpenArchive でアーカイブを読み込む。
  • -l オプション指定時は PrintTOCSummary で目次を表示する。
  • 通常時は ProcessArchiveRestoreOptions で復元順序を確定し、RestoreArchive を呼び出して復元を実行する。

    • 5. **終了**: CloseArchive でリソースを解放し、終了コードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（プログラムのエントリポイント）。
  • **呼び出し先**:
  • NewRestoreOptions, getopt_long, pg_logging_init 等の初期化関数。
  • OpenArchive, SetArchiveOptions, RestoreArchive, CloseArchive 等のアーカイブ操作関数。
  • usage, exit_nicely 等のユーティリティ関数。
  • ※提供されたソース情報には、本関数を呼び出す他の内部関数は記載されていません。

関連データ構造

  • RestoreOptions: 復元時の動作を制御する設定を保持する構造体。
  • Archive: アーカイブのメタデータと状態を保持するハンドル。
  • option: getopt_long で使用されるコマンドラインオプション定義構造体。

失敗モードと境界条件

  • 引数解析時に不正なオプションが指定された場合、usage を表示して終了する。
  • 競合するオプション（例: --data-only と --schema-only の組み合わせ等）が指定された場合、整合性チェックによりエラーとなる可能性がある。
  • アーカイブファイルのオープン失敗や復元処理中のエラー発生時は、exit_nicely を通じて適切な終了コードで終了する。

保守担当者向けメモ

  • 本関数は主に引数解析と設定の橋渡しを担当しています。実際の復元ロジックの詳細は pg_backup_archiver.c に委譲されています。
  • 復元処理の挙動を追跡する場合は、RestoreArchive(AH) の呼び出し箇所から pg_backup_archiver.c を参照してください。
  • --filter オプションによる複雑な制御は read_restore_filters 関数で処理されます。
