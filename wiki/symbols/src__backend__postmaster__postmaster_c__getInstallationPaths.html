getInstallationPaths

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/postmaster.c:1461

目的

PostgreSQL 実行ファイルおよびライブラリ（pkglib）のインストールパスを特定し、インストール環境が正常であることを検証する。

シグネチャまたは形状

static void getInstallationPaths(const char *argv0)

入力と出力

  • 入力: argv0（起動時のプログラム名）
  • 出力: なし（成功時）。内部的なグローバル変数 my_exec_path および pkglib_path を設定する。失敗時は ereport によりプロセスを終了させる。

前提条件と不変条件

  • argv0 が有効な文字列であること。
  • find_my_exec および get_pkglib_path が実行環境のパスを正しく解決できること。
  • pkglib_path で指定されるディレクトリが読み取り可能であること。

アルゴリズムまたは振る舞い

1. find_my_exec を呼び出し、自身の実行ファイルパスを my_exec_path に格納する。失敗時は FATAL エラーとする。 2. EXEC_BACKEND が定義されている場合、find_other_exec を使用して postgres 実行ファイルのパスを postgres_exec_path に格納する。 3. get_pkglib_path を呼び出し、ライブラリディレクトリのパスを pkglib_path に格納する。 4. AllocateDir を使用して pkglib_path がディレクトリとして存在し、かつ読み取り可能であることを確認する。 5. 検証に失敗した場合は ERROR を送出し、インストールが不完全または破損している旨を報告する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/postmaster.c 内の PostmasterMain（行537）。
  • 呼び出し先: find_my_exec, find_other_exec (条件付き), get_pkglib_path, AllocateDir, FreeDir, ereport。

関連データ構造

  • my_exec_path (グローバル変数)
  • pkglib_path (グローバル変数)
  • postgres_exec_path (グローバル変数、EXEC_BACKEND 定義時のみ)

失敗モードと境界条件

  • find_my_exec 失敗: 自身の実行ファイルが特定できない場合、FATAL で終了。
  • find_other_exec 失敗: EXEC_BACKEND 環境下で対応する postgres 実行ファイルが見つからない場合、FATAL で終了。
  • ディレクトリアクセス失敗: pkglib_path が存在しない、または権限不足で開けない場合、ERROR で終了。

保守担当者向けメモ

  • 本関数は PostmasterMain の初期化フェーズで早期に呼び出される。
  • pkglib_path の検証は、設定ファイル読み込み後にモジュールをロードする際の依存関係を保証するために重要である。
