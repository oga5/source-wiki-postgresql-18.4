wait_result_is_signal

定義

  • 種別: function
  • 定義箇所: src/common/wait_error.c:101

目的

プロセス終了ステータスが、指定されたシグナル番号によって発生したものであるかを判定します。シェル経由で実行されたプロセスの終了コード（128 + シグナル番号）も考慮します。

シグネチャまたは形状

bool wait_result_is_signal(int exit_status, int signum)

入力と出力

  • **入力**:
  • exit_status: wait(2) や waitpid(2) から返されるプロセス終了ステータス。
  • signum: 判定対象のシグナル番号。
  • **出力**:
  • bool: 指定されたシグナルで終了した場合は true、それ以外は false。

前提条件と不変条件

  • exit_status は wait 関連のシステムコールから取得された有効なステータス値である必要があります。
  • POSIX シェルの慣習に基づき、シグナルによる終了を WIFSIGNALED による判定と、終了コード 128 + signum の両面から検証します。

アルゴリズムまたは振る舞い

1. WIFSIGNALED(exit_status) が真かつ WTERMSIG(exit_status) が signum と一致する場合、true を返します。 2. 上記に該当しない場合、WIFEXITED(exit_status) が真かつ WEXITSTATUS(exit_status) が 128 + signum と一致する場合、true を返します。 3. いずれにも該当しない場合、false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogarchive.c:264
  • src/backend/commands/copyfrom.c:1963
  • **呼び出し先**:
  • WIFSIGNALED, WTERMSIG, WIFEXITED, WEXITSTATUS マクロ（<sys/wait.h> 由来）。

関連データ構造

  • 特になし（標準的な int 型の終了ステータスを使用）。

失敗モードと境界条件

  • Windows 環境では WTERMSIG 等の挙動がプラットフォーム依存となるため、src/include/port.h 等で定義された移植性レイヤーに依存します。
  • exit_status が不正な値である場合、マクロの挙動は未定義です。

保守担当者向けメモ

  • シェル経由の実行（system(3) 等）では、シグナル終了が終了コード 128 + signum に変換されるため、この関数はその両方をカバーする設計になっています。
  • 類似の判定を行う wait_result_is_any_signal との使い分けに注意してください。
