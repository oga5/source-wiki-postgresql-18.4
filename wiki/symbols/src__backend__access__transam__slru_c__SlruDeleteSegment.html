SlruDeleteSegment

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/slru.c:1525

目的

指定されたセグメント番号（segno）に対応する SLRU セグメントファイルを物理的に削除する。削除に先立ち、メモリ上のバッファキャッシュから当該セグメントに属するページを排除またはフラッシュする。

シグネチャまたは形状

void SlruDeleteSegment(SlruCtl ctl, int64 segno)

入力と出力

  • ctl: 対象となる SLRU インスタンスの制御構造体。
  • segno: 削除対象のセグメント番号。
  • 戻り値: なし。

前提条件と不変条件

  • 呼び出し元は、削除対象のセグメントが既に不要であることを保証しなければならない。
  • 処理中、対象セグメントに関連するバッファスロットのロック（銀行ロック）が排他的に取得される。
  • 処理の過程で I/O が発生する場合、銀行ロックは一時的に解放される。

アルゴリズムまたは振る舞い

1. shared->bank_locks を用いて、全スロットを走査する。 2. 各スロットについて、対象セグメントに属するページを特定する。 3. ページが SLRU_PAGE_VALID かつダーティである場合は SlruInternalWritePage でディスクへ書き出し、クリーンな場合は単に SLRU_PAGE_EMPTY 状態へ遷移させる。 4. ページが I/O 中の場合は SimpleLruWaitIO で完了を待機する。 5. I/O 発生によりロックを解放した場合は、再走査（restart）を行い、競合により読み込まれた可能性のある対象セグメントのページを再度クリーンアップする。 6. 全スロットの確認完了後、SlruInternalDeleteSegment を呼び出して物理ファイルを削除する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/multixact.c
  • src/test/modules/test_slru/test_slru.c
  • **呼び出し先**:
  • SlotGetBankNumber (マクロ)
  • LWLockAcquire / LWLockRelease
  • SlruInternalWritePage
  • SimpleLruWaitIO
  • SlruInternalDeleteSegment

関連データ構造

  • SlruCtl: SLRU インスタンスの制御構造体。
  • SlruSharedData: 共有メモリ上の状態管理構造体（page_status, page_number, page_dirty, bank_locks を含む）。

失敗モードと境界条件

  • 削除対象のセグメントが既に存在しない場合や、I/O エラーが発生した場合は SlruInternalDeleteSegment 内の挙動に依存する。
  • 処理中に新しいページが読み込まれた場合、did_write フラグによる再走査ループにより整合性が維持される。

保守担当者向けメモ

  • 本関数は SimpleLruTruncate と同様の排他制御ロジックを採用している。ロックの取得順序や解放タイミングを変更する際は、デッドロックの可能性に注意すること。
  • SlruInternalDeleteSegment は物理的なファイル削除を行うため、呼び出し前に必ず当該セグメントへの参照がキャッシュから完全に排除されている必要がある。
