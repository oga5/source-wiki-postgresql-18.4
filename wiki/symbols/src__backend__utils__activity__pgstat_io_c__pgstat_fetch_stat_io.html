pgstat_fetch_stat_io

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_io.c:163

目的

I/O統計情報のスナップショットを取得し、ローカルの統計構造体へのポインタを返します。

シグネチャまたは形状

PgStat_IO *pgstat_fetch_stat_io(void)

入力と出力

  • 入力: なし
  • 出力: PgStat_IO 構造体へのポインタ。現在のバックエンドにおけるI/O統計スナップショットを指します。

前提条件と不変条件

  • 呼び出し前に、統計サブシステムが初期化されている必要があります。
  • 返されるポインタは、次に pgstat_snapshot_fixed が呼び出されるか、スナップショットが更新されるまでの間有効です。

アルゴリズムまたは振る舞い

1. pgstat_snapshot_fixed(PGSTAT_KIND_IO) を呼び出し、I/O統計の最新スナップショットを生成・更新します。 2. pgStatLocal.snapshot.io のアドレスを返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/adt/pgstatfuncs.c:1556 (SQL関数等から統計を参照する際に使用)
  • 呼び出し先:
  • pgstat_snapshot_fixed (統計スナップショットの更新)

関連データ構造

  • PgStat_IO: I/O統計情報を保持する構造体。
  • pgStatLocal: バックエンドローカルの統計情報を管理するグローバル変数。

失敗モードと境界条件

  • 統計サブシステムが未初期化の状態で呼び出された場合の挙動については、本ソースコード内では定義されていません。

保守担当者向けメモ

  • 本関数は pg_stat_io ビューのデータソースとして機能します。統計の取得ロジックを変更する場合は、pgstat_snapshot_fixed の挙動と整合性が取れているか確認してください。
