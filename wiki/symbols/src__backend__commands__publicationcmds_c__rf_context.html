rf_context

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/publicationcmds.c:56

目的

論理レプリケーションにおける行フィルタ（row filter）の妥当性を検証する際、関連するリレーション情報やREPLICA IDENTITYの制約を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct rf_context
{
    Bitmapset  *bms_replident; /* レプリカアイデンティティ列のビットセット */
    bool        pubviaroot;    /* 親リレーションの行フィルタを検証中か否か */
    Oid         relid;         /* 対象リレーションのOID */
    Oid         parentid;      /* 親リレーションのOID */
} rf_context;

入力と出力

本構造体は関数間で状態を共有するために使用されます。

  • 入力: contain_invalid_rfcolumn_walker 等の関数へ引数として渡されます。
  • 出力: 構造体内の各フィールドが検証処理の判断基準として参照されます。

前提条件と不変条件

  • relid および parentid は有効なリレーションOIDである必要があります。
  • bms_replident は、対象リレーションのREPLICA IDENTITYを構成する列を正確に反映している必要があります。

アルゴリズムまたは振る舞い

行フィルタ式を走査する際、式中に含まれる列がREPLICA IDENTITYに含まれているかを判定するために使用されます。pubviaroot が真の場合、パーティションテーブルの親階層に対する検証が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/publicationcmds.c:323 にて初期化されます。
  • 呼び出し先: contain_invalid_rfcolumn_walker（src/backend/commands/publicationcmds.c:237）が本構造体を引数として受け取り、式内の列を検証します。

関連データ構造

  • Bitmapset: 列のインデックスを保持するために使用されます。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • bms_replident が適切に初期化されていない場合、行フィルタの検証が誤った結果を返す可能性があります。
  • pubviaroot の設定が実際のテーブル階層と不一致の場合、パーティションテーブルの行フィルタ検証が正しく機能しません。

保守担当者向けメモ

  • 本構造体は、行フィルタの構文解析および検証フェーズにおいて、式ツリーを走査する際の「状態」を保持する役割に特化しています。
  • 変更を加える際は、contain_invalid_rfcolumn_walker での参照方法と整合性を取る必要があります。
