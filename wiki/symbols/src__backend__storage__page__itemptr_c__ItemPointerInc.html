ItemPointerInc

定義

  • 種別: function
  • 定義箇所: src/backend/storage/page/itemptr.c:83

目的

ItemPointer（TID）を論理的にインクリメントし、オフセット番号が上限に達した際にブロック番号を繰り上げることで、TID の連続的な移動をサポートします。

シグネチャまたは形状

void ItemPointerInc(ItemPointer pointer)

入力と出力

  • **入力**: ItemPointer pointer（操作対象のポインタへのポインタ）
  • **出力**: なし（引数 pointer が指すメモリの内容を直接更新）

前提条件と不変条件

  • pointer は有効な ItemPointerData を指している必要があります。
  • ページ内の有効なオフセット範囲（MaxOffsetNumber）とは独立して動作し、PG_UINT16_MAX を境界として計算を行います。

アルゴリズムまたは振る舞い

1. ItemPointerGetBlockNumberNoCheck および ItemPointerGetOffsetNumberNoCheck を使用して、現在のブロック番号とオフセット番号を取得します。 2. オフセット番号が PG_UINT16_MAX に達している場合:

  • ブロック番号が InvalidBlockNumber でなければ、オフセットを 0 にリセットし、ブロック番号を 1 インクリメントします。

    • 3. オフセット番号が PG_UINT16_MAX 未満の場合:

  • オフセット番号を 1 インクリメントします。

    • 4. ItemPointerSet を使用して、更新された値を pointer に書き戻します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeTidrangescan.c:181
  • **呼び出し先**:
  • ItemPointerGetBlockNumberNoCheck
  • ItemPointerGetOffsetNumberNoCheck
  • ItemPointerSet

関連データ構造

  • ItemPointerData（TID）: 6 バイトの構造体で、ブロック番号とオフセット番号を保持します。

失敗モードと境界条件

  • blk が InvalidBlockNumber の場合、オフセットが PG_UINT16_MAX に達してもブロック番号の繰り上げは行われません。
  • オフセットが PG_UINT16_MAX に達していない場合は、単純にオフセットが加算されます。

保守担当者向けメモ

  • 本関数はインライン化に適さない複雑な演算を実装しています。
  • NoCheck 系のアクセサを使用しているため、無効な TID 値に対してもビットパターンに基づいた一貫した演算結果を返します。
