indexed_tlist

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/setrefs.c:49

目的

サブプランのターゲットリスト（tlist）を高速に検索・参照するためのインデックス付き構造体です。プランの最終的な後処理において、変数参照の解決を効率化するために使用されます。

シグネチャまたは形状

typedef struct {
    List       *tlist;
    int         num_vars;
    bool        has_ph_vars;
    bool        has_non_vars;
    tlist_vinfo vars[FLEXIBLE_ARRAY_MEMBER];
} indexed_tlist;

入力と出力

  • 入力: build_tlist_index 関数により、既存の List *tlist から生成されます。
  • 出力: 構造体内の vars 配列を通じて、ターゲットリスト内の各要素への高速なアクセスを提供します。

前提条件と不変条件

  • vars 配列は num_vars 個の要素を持ちます。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、メモリ割り当て時に num_vars 分の領域が確保されている必要があります。

アルゴリズムまたは振る舞い

プランツリーの走査中、式ツリー内の変数をサブプランの出力位置へマッピングする際に参照されます。has_ph_vars（PlaceHolderVarの有無）や has_non_vars（その他の要素の有無）フラグにより、検索対象の絞り込みを最適化します。

呼び出し元と呼び出し先

  • 呼び出し元: set_plan_references および関連する式変換関数群（fix_scan_expr 等）が、プランノードの参照解決時にこの構造体を利用します。
  • 呼び出し先: build_tlist_index 関数が本構造体の構築を担います。

関連データ構造

  • tlist_vinfo: indexed_tlist 内の各要素を保持する構造体。
  • List: ターゲットリストの基底構造。

失敗モードと境界条件

  • num_vars が 0 の場合、vars 配列へのアクセスは無効です。
  • ターゲットリストが空の場合、本構造体は最小限の初期化状態となります。

保守担当者向けメモ

  • 本構造体は setrefs.c 内でのみ使用される内部的な最適化構造体です。
  • vars 配列は可変長であるため、構造体のコピーやメモリ管理には注意が必要です。
