ecpg_pthreads_init

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/connect.c:29

目的

スレッド固有の接続データを管理するためのキー（actual_connection_key）が、プロセス内で一度だけ確実に初期化されることを保証します。

シグネチャまたは形状

void ecpg_pthreads_init(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてスレッド関連の初期化を実行）

前提条件と不変条件

  • pthread_once_t 型の静的変数 actual_connection_key_once が初期化済みであること。
  • ecpg_actual_connection_init 関数が pthread_once の引数として適切に定義されていること。

アルゴリズムまたは振る舞い

1. pthread_once を呼び出し、actual_connection_key_once コントロールブロックを使用して ecpg_actual_connection_init を実行します。 2. これにより、マルチスレッド環境下でも actual_connection_key の初期化処理が一度だけ実行されることが保証されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/interfaces/ecpg/ecpglib/connect.c 内の接続管理関数群
  • src/interfaces/ecpg/ecpglib/execute.c:1964
  • 呼び出し先:
  • pthread_once（システムライブラリ）
  • ecpg_actual_connection_init（初期化ルーチン）

関連データ構造

  • actual_connection_key_once: pthread_once_t 型の制御変数。
  • actual_connection_key: スレッドごとのカレント接続を保持するために使用される pthread_key_t。

失敗モードと境界条件

  • pthread_once の内部実装に依存します。スレッドライブラリが利用できない環境や、初期化関数が失敗した場合の挙動はプラットフォームの pthread 実装に準拠します。

保守担当者向けメモ

  • 本関数は、ECPGライブラリ内のスレッドセーフな接続管理機能を利用する前に必ず呼び出される必要があります。
  • ecpg-pthread-win32.h が含まれているため、Windows環境でのスレッド互換性も考慮されています。
