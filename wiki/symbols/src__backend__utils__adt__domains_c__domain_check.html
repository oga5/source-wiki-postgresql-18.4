domain_check

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/domains.c:345

目的

指定されたデータ（Datum）が、特定のドメイン型に対して定義された制約（NOT NULL や CHECK 制約）を満たしているかを検証します。

シグネチャまたは形状

void domain_check(Datum value, bool isnull, Oid domainType, void **extra, MemoryContext mcxt);

入力と出力

  • **入力**:
  • value: 検証対象のデータ。
  • isnull: 値が NULL かどうかを示すフラグ。
  • domainType: 検証対象のドメイン型の OID。
  • extra: キャッシュされたドメイン情報を保持するためのポインタ（DomainIOData への参照）。
  • mcxt: 制約評価に使用するメモリコンテキスト。
  • **出力**: なし（戻り値は void）。制約違反がある場合はエラーをスローします。

前提条件と不変条件

  • extra は、初回呼び出し時に NULL であれば初期化され、以降はキャッシュとして再利用されます。
  • 制約評価には ExprContext を使用するため、呼び出し元は適切なメモリ管理（ReScanExprContext 等）を考慮する必要があります。

アルゴリズムまたは振る舞い

1. 内部関数 domain_check_internal を呼び出します。 2. domain_check_internal は、domainType に基づいてキャッシュされた制約情報を取得または生成します。 3. NOT NULL 制約および CHECK 制約を順次評価します。 4. 制約違反が検出された場合、エラーを報告して処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_io.c
  • src/backend/utils/adt/expandedrecord.c
  • src/backend/utils/adt/domains.c（内部的な再帰的参照やテスト用途）
  • **呼び出し先**:
  • domain_check_internal（直接の委譲先）

関連データ構造

  • DomainIOData: ドメインの制約情報や実行コンテキストを保持する構造体。
  • Datum: PostgreSQL の汎用データ表現。

失敗モードと境界条件

  • 制約違反: CHECK 制約の評価結果が偽である場合、または NOT NULL 制約に違反する場合、エラーが発生します。
  • メモリ不足: 制約評価中にメモリ不足が発生した場合、標準的なエラー処理に従います。

保守担当者向けメモ

  • 本関数はエラー発生時に例外をスローします。例外を発生させずに検証結果のみを得たい場合は、代わりに domain_check_safe を使用してください。
  • extra 引数は、同一のドメイン型に対する繰り返し呼び出し時のパフォーマンスを最適化するために重要です。
