attype

定義

  • 種別: struct
  • 定義箇所: src/timezone/zic.c:390

目的

タイムゾーンの遷移時刻（Transition Time）とその属性を保持するための構造体です。zic コンパイラが TZif バイナリを生成する過程で、各ゾーンの遷移情報を管理するために使用されます。

シグネチャまたは形状

static struct attype
{
    zic_t       at;         /* 遷移時刻 */
    bool        dontmerge;  /* マージを禁止するフラグ */
    unsigned char type;     /* 遷移先のタイプインデックス */
} *attypes;

入力と出力

  • 入力: タイムゾーン定義ファイル（.zi）から解析されたルールに基づき、outzone() 関数等で本構造体の配列（attypes）が構築されます。
  • 出力: writezone() 関数において、この構造体のデータが TZif 形式のバイナリファイルへ書き出されます。

前提条件と不変条件

  • attypes は、遷移時刻に基づいてソートされた状態で管理されることが想定されています。
  • at フィールドは zic_t 型であり、システムで定義された時刻表現に依存します。

アルゴリズムまたは振る舞い

  • タイムゾーンの遷移時刻を計算する際、複数の遷移が連続する場合の最適化や、TZif 仕様への適合処理が行われます。
  • qsort 等の比較関数内で、at フィールドを参照して遷移時刻の順序を決定する処理が含まれます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/timezone/zic.c:2039-2040: 遷移時刻の比較処理（qsort 用の比較関数内）で参照されています。
  • src/timezone/zic.c:3325: lastat ポインタとして一時的に参照されています。
  • 呼び出し先:
  • 情報なし。

関連データ構造

  • zic_t: 時刻表現に使用される型。
  • struct zone: タイムゾーン定義を保持する構造体。
  • struct rule: タイムゾーン切り替えルールを保持する構造体。

失敗モードと境界条件

  • メモリ不足: attypes 配列の動的確保時にメモリが枯渇した場合、処理が中断される可能性があります。
  • 遷移時刻の重複: 適切に処理されない場合、dontmerge フラグの有無に関わらず、バイナリ生成時に冗長なデータが含まれる可能性があります。

保守担当者向けメモ

  • 本構造体は zic.c 内で静的に定義されており、コンパイル時のタイムゾーン生成プロセスに特化しています。
  • at フィールドの比較を行う際は、zic_t の型定義（int64 等）がプラットフォーム間で一貫していることを確認してください。
