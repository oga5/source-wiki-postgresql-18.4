pg_disable_aslr

定義

  • 種別: function
  • 定義箇所: src/common/exec.c:478

目的

実行プロセスにおけるアドレス空間配置のランダム化（ASLR）を無効化し、メモリレイアウトの予測可能性を確保する。

シグネチャまたは形状

int pg_disable_aslr(void);

入力と出力

  • 入力: なし
  • 出力: 成功時は 0、失敗時は -1 を返す。失敗時は errno が設定される。

前提条件と不変条件

  • プラットフォームが HAVE_SYS_PERSONALITY_H または HAVE_SYS_PROCCTL_H をサポートしている必要がある。
  • サポートされていない環境では、常に errno を ENOSYS に設定し -1 を返す。

アルゴリズムまたは振る舞い

1. HAVE_SYS_PERSONALITY_H が定義されている場合、personality(ADDR_NO_RANDOMIZE) を呼び出し、ASLR を無効化する。 2. 上記が定義されておらず、HAVE_SYS_PROCCTL_H および PROC_ASLR_FORCE_DISABLE が定義されている場合、procctl(P_PID, 0, PROC_ASLR_CTL, &data) を呼び出し、ASLR を強制的に無効化する。 3. いずれの条件も満たさない場合、errno を ENOSYS に設定し、-1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_ctl/pg_ctl.c:451
  • src/test/regress/pg_regress.c:1215
  • 呼び出し先:
  • システムコール personality または procctl（プラットフォーム依存）。

関連データ構造

  • なし

失敗モードと境界条件

  • サポートされていないOS環境では、機能が提供されず ENOSYS エラーとなる。
  • システムコールが失敗した場合、そのエラーコードがそのまま返される。

保守担当者向けメモ

  • 本関数はプラットフォーム固有のASLR制御を抽象化するものである。新しいOSやアーキテクチャでASLR無効化が必要な場合は、プリプロセッサ条件分岐に新たな定義を追加すること。
