dsm_attach

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/dsm.c:664

目的

既存の動的共有メモリ（DSM）セグメントを現在のプロセスにアタッチし、アクセス可能な状態にする。

シグネチャまたは形状

dsm_segment *dsm_attach(dsm_handle h)

入力と出力

  • **入力**: dsm_handle h（アタッチ対象のセグメントハンドル）
  • **出力**: 成功時は dsm_segment 構造体へのポインタ、セグメントが見つからない場合は NULL を返す。

前提条件と不変条件

  • IsUnderPostmaster が真であること（ポストマスタープロセス内での呼び出しは不可）。
  • 同一プロセス内で同じセグメントハンドルに対して二重にアタッチしようとするとエラーが発生する。

アルゴリズムまたは振る舞い

1. 必要に応じて dsm_backend_startup() を呼び出し、DSM システムを初期化する。 2. 既にアタッチ済みのセグメントリストを走査し、指定されたハンドルが重複していないか確認する。 3. dsm_create_descriptor() で新しいセグメント記述子を作成する。 4. DynamicSharedMemoryControlLock を排他取得し、共有メモリ上の制御領域から該当ハンドルを検索する。 5. ハンドルが見つかった場合、参照カウント（refcnt）をインクリメントし、セグメントのメモリマッピング情報を設定する。 6. メイン共有メモリ領域内のセグメントであれば、オフセットからアドレスを計算する。それ以外の場合は dsm_impl_op を呼び出し、OS レベルでのアタッチ（マッピング）を行う。 7. 制御領域にハンドルが存在しない場合、アタッチを中止し dsm_detach を実行して NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_prewarm/autoprewarm.c, src/backend/access/common/session.c, src/backend/access/transam/parallel.c, src/backend/replication/logical/applyparallelworker.c, src/backend/storage/ipc/dsm_registry.c, src/backend/utils/mmgr/dsa.c, src/test/modules/test_shm_mq/worker.c
  • **呼び出し先**: dsm_backend_startup, dsm_create_descriptor, LWLockAcquire, LWLockRelease, is_main_region_dsm_handle, dsm_detach, dsm_impl_op

関連データ構造

  • dsm_segment: プロセスローカルなセグメント管理構造体。
  • dsm_control_header / dsm_control_item: 共有メモリ上の制御情報。
  • dsm_segment_list: プロセスが現在保持しているセグメントのリスト。

失敗モードと境界条件

  • 指定したハンドルが制御領域に見つからない場合、NULL を返す。
  • 既にアタッチ済みのハンドルを再度指定した場合、ERROR が発生する。
  • dsm_impl_op によるマッピングに失敗した場合、エラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は dsm_find_mapping() を呼び出してから使用することが推奨される。
  • 制御領域の検索ループにおいて、refcnt <= 1 のスロットは無視される（破棄プロセス中のため）。
  • メイン共有メモリ領域（dsm_main_space_begin）を利用するセグメントと、OS 依存のセグメントでマッピング処理が分岐する点に注意すること。
