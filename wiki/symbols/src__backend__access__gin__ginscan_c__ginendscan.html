ginendscan

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginscan.c:505

目的

GINインデックススキャンの終了処理を行い、スキャンに関連するリソースを解放します。

シグネチャまたは形状

void ginendscan(IndexScanDesc scan)

入力と出力

  • **入力**: IndexScanDesc scan - 終了対象となるインデックススキャン記述子。
  • **出力**: なし。

前提条件と不変条件

  • scan->opaque には、ginbeginscan によって初期化された GinScanOpaque 構造体が格納されている必要があります。

アルゴリズムまたは振る舞い

1. scan->opaque を GinScanOpaque 型として取得します。 2. ginFreeScanKeys を呼び出し、スキャンキーに関連付けられたメモリおよびリソースを解放します。 3. GinScanOpaque 内の tempCtx（一時メモリコンテキスト）を削除します。 4. GinScanOpaque 内の keyCtx（スキャンキー用メモリコンテキスト）を削除します。 5. GinScanOpaque 構造体自体を pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginutil.c において、amroutine->amendscan に本関数が割り当てられています。
  • **呼び出し先**:
  • ginFreeScanKeys
  • MemoryContextDelete
  • pfree

関連データ構造

  • IndexScanDesc: PostgreSQLの標準的なインデックススキャン記述子。
  • GinScanOpaque: GINスキャン中の状態を保持する構造体。

失敗モードと境界条件

  • scan が NULL の場合、または scan->opaque が正しく初期化されていない場合は、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は ginbeginscan で確保されたメモリコンテキストのライフサイクルを終了させる役割を持ちます。メモリリークを防ぐため、新しいリソースを GinScanOpaque に追加した場合は、必ず ginFreeScanKeys または本関数内で解放処理を行ってください。
