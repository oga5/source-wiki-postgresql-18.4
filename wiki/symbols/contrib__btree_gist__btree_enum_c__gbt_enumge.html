gbt_enumge

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_enum.c:39

目的

PostgreSQL の ENUM 型に対する「以上（greater than or equal to）」比較を GiST インデックスの検索ロジック内で実行する。

シグネチャまたは形状

static bool gbt_enumge(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の ENUM 値を指すポインタ（内部的に Oid として扱われる）。
  • flinfo: PostgreSQL の関数マネージャー情報。
  • **出力**:
  • bool: a >= b が真であれば true、そうでなければ false を返す。

前提条件と不変条件

  • a および b は、有効な Oid を指すポインタである必要がある。
  • ENUM 型は内部的に Oid として表現されており、この関数は enum_ge 関数を介してその順序関係を評価する。

アルゴリズムまたは振る舞い

1. a と b が指す Oid 値を ObjectIdGetDatum で Datum 型に変換する。 2. CallerFInfoFunctionCall2 を使用して、PostgreSQL 標準の enum_ge 関数を呼び出す。 3. 比較結果を DatumGetBool で論理値に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_enum.c 内の tinfo 構造体（gbtree_ninfo）の初期化時に参照される。
  • **呼び出し先**:
  • enum_ge（PostgreSQL 標準の ENUM 比較関数）
  • CallerFInfoFunctionCall2（PostgreSQL 関数呼び出しインターフェース）

関連データ構造

  • Oid: ENUM 型の内部表現。
  • FmgrInfo: 関数呼び出しに必要なメタデータ。
  • tinfo: gbtree_ninfo 構造体（本関数を保持し、GiST ユーティリティ関数へ提供する）。

失敗モードと境界条件

  • a または b が無効なメモリ領域を指している場合、セグメンテーション違反が発生する可能性がある。
  • flinfo が適切に初期化されていない場合、CallerFInfoFunctionCall2 の実行時にエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で定義された汎用 GiST ユーティリティからコールバックされることを前提としている。
  • ENUM 型の順序は Oid の順序と一致するため、OID ベースの比較ロジックと互換性がある。
