pg_current_xact_id

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xid8funcs.c:333

目的

現在のトランザクションID（FullTransactionId / xid8）を取得し、SQLインターフェースを通じてユーザーに公開します。

シグネチャまたは形状

Datum pg_current_xact_id(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出し規約に基づく引数）。
  • **出力**: Datum 型としてラップされた FullTransactionId（64ビットトランザクションID）。

前提条件と不変条件

  • 呼び出し元は、有効なトランザクションコンテキスト内である必要があります。
  • リカバリ実行中（InRecovery）の呼び出しは禁止されています。

アルゴリズムまたは振る舞い

1. PreventCommandDuringRecovery("pg_current_xact_id()") を呼び出し、リカバリ中の実行を阻止します。リカバリ中に実行すると、トランザクションIDが割り当てられていない場合に強制的な割り当てが発生し、失敗するためです。 2. GetTopFullTransactionId() を呼び出し、現在のトップレベルトランザクションIDを取得します。 3. 取得したIDを PG_RETURN_FULLTRANSACTIONID マクロを介して返却します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • PreventCommandDuringRecovery: リカバリ状態のチェック。
  • GetTopFullTransactionId: 現在のトランザクションIDの取得。
  • **呼び出し元**: SQLインターフェースから直接呼び出されることを想定しています。

関連データ構造

  • FullTransactionId: 64ビットのトランザクションID（エポックを含む）。

失敗モードと境界条件

  • **リカバリ中の実行**: PreventCommandDuringRecovery によりエラーが発生します。
  • **トランザクション未割り当て**: トランザクションIDがまだ割り当てられていない場合、GetTopFullTransactionId の内部ロジックにより新しいIDが割り当てられます。

保守担当者向けメモ

  • 本関数は常に有効なトランザクションIDを返すことが期待されています。NULLや無効なIDを返すような変更は避けてください。
  • サブトランザクションIDは除外され、トップレベルのトランザクションIDのみが返されます。
  • pg_current_xact_id_if_assigned とは異なり、本関数は必要に応じてトランザクションIDを強制的に割り当てます。
