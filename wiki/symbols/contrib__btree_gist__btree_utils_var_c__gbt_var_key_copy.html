gbt_var_key_copy

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_utils_var.c:88

目的

GBT_VARKEY_R 構造体で保持されている下限（lower）および上限（upper）の可変長データを、単一の GBT_VARKEY 構造体へコピーして統合します。

シグネチャまたは形状

GBT_VARKEY *gbt_var_key_copy(const GBT_VARKEY_R *u)

入力と出力

  • **入力**: u (const GBT_VARKEY_R *) - コピー元となる下限・上限ポインタを保持する構造体。
  • **出力**: GBT_VARKEY * - 統合されたデータを含む、メモリ上に割り当てられた新しい GBT_VARKEY 構造体。

前提条件と不変条件

  • u->lower および u->upper は有効な varlena 形式のデータである必要があります。
  • palloc0 を使用してメモリを確保するため、PostgreSQL のメモリコンテキスト内で呼び出される必要があります。

アルゴリズムまたは振る舞い

1. VARSIZE マクロを用いて、下限および上限データのサイズを取得します。 2. INTALIGN を用いて下限データのサイズをアライメントし、合計サイズ（下限＋上限＋ヘッダーサイズ）を算出します。 3. palloc0 で必要なメモリ領域を確保します。 4. memcpy を使用して、確保した領域の先頭に下限データを、アライメント後のオフセット位置に上限データをコピーします。 5. SET_VARSIZE を使用して、新しい GBT_VARKEY のヘッダーに合計サイズを書き込みます。 6. 構築した GBT_VARKEY ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_bit.c:102
  • contrib/btree_gist/btree_numeric.c:165
  • contrib/btree_gist/btree_utils_var.c:279, 285, 345
  • **呼び出し先**:
  • palloc0, memcpy (PostgreSQL 内部メモリ管理およびメモリ操作関数)

関連データ構造

  • GBT_VARKEY: 可変長キーを格納する構造体。
  • GBT_VARKEY_R: 可変長キーの下限と上限を個別のポインタとして保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • 入力ポインタ u が NULL の場合、未定義の動作（セグメンテーション違反）を引き起こします。

保守担当者向けメモ

  • 本関数は varlena 型のメモリレイアウトに強く依存しています。INTALIGN によるパディングが正しく適用されているため、構造の変更時にはアライメント要件を維持してください。
