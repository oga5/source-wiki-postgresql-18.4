ActionList

定義

  • **種別**: struct
  • **定義箇所**: src/backend/commands/async.c:345

目的

トランザクションのネストレベルごとに、保留中の LISTEN / UNLISTEN アクションを管理するためのスタック構造を提供します。

シグネチャまたは形状

typedef struct ActionList
{
    int             nestingLevel;   /* 現在のトランザクションのネスト深さ */
    List           *actions;        /* ListenAction 構造体のリスト */
    struct ActionList *upper;       /* 上位トランザクションレベルのリストへのポインタ */
} ActionList;

入力と出力

  • **入力**: トランザクション開始時やネストされたトランザクションの開始時に、TopTransactionContext 上でメモリ確保および初期化が行われます。
  • **出力**: トランザクションのコミットまたはロールバックに伴い、リスト内のアクションが処理され、メモリが解放されます。

前提条件と不変条件

  • upper フィールドは、現在のトランザクションより上位のレベルが存在しない場合は NULL となります。
  • actions リストには ListenAction 構造体が格納されます。

アルゴリズムまたは振る舞い

  • トランザクションのネストレベルに応じて、ActionList がスタック状に連結されます。
  • 新しいネストレベルに入る際、既存の pendingActions を upper に保持した新しい ActionList が作成されます。
  • トランザクション終了時に、この構造体を通じて保留中のアクションが順次処理されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/async.c 内のトランザクション管理処理（pendingActions の操作に関連する箇所）。
  • **呼び出し先**: ListenAction 構造体（actions リストの要素として）。
  • *注: 具体的な関数呼び出しのフローについては、提供されたソース情報からは詳細な依存関係を特定できません。*

関連データ構造

  • ListenAction: actions リスト内に格納される個々のアクション定義。

失敗モードと境界条件

  • メモリ不足時: MemoryContextAlloc が失敗した場合、PostgreSQLの標準的なエラーハンドリングによりトランザクションがアボートされます。

保守担当者向けメモ

  • ActionList および NotificationList オブジェクト自体は、トランザクション終了時に適切にクリーンアップされる必要があります。
  • ネストされたトランザクションにおける nestingLevel の整合性に注意してください。
