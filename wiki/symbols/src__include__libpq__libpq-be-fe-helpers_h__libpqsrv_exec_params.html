libpqsrv_exec_params

定義

  • 種別: function
  • 定義箇所: src/include/libpq/libpq-be-fe-helpers.h:268

目的

PostgreSQL バックエンドから libpq を介して、パラメータ付きクエリを安全に実行し、その結果を取得します。

シグネチャまたは形状

PGresult *libpqsrv_exec_params(PGconn *conn,
                               const char *command,
                               int nParams,
                               const Oid *paramTypes,
                               const char *const *paramValues,
                               const int *paramLengths,
                               const int *paramFormats,
                               int resultFormat,
                               uint32 wait_event_info)

入力と出力

  • **入力**:
  • conn: 接続済みの PGconn オブジェクト。
  • command: 実行する SQL クエリ文字列。
  • nParams, paramTypes, paramValues, paramLengths, paramFormats: クエリパラメータに関する情報。
  • resultFormat: 結果の形式（0: テキスト, 1: バイナリ）。
  • wait_event_info: 待機イベントの識別子。
  • **出力**:
  • 成功時: クエリ結果を含む PGresult へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • conn は有効な接続状態であること。
  • バックエンドの FD 管理ルールに従い、呼び出し元は適切なリソース保護下にあること。

アルゴリズムまたは振る舞い

1. PQsendQueryParams を呼び出し、クエリをサーバーへ送信する。 2. 送信に失敗した場合は直ちに NULL を返す。 3. 送信成功後、libpqsrv_get_result_last を呼び出し、クエリの最終結果を取得する。この過程で WaitLatchOrSocket を用いた待機と CHECK_FOR_INTERRUPTS() による割り込み処理が行われる。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • **呼び出し先**:
  • PQsendQueryParams (libpq)
  • libpqsrv_get_result_last (本ライブラリ内)

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • PGresult: クエリ実行結果を保持する構造体。

失敗モードと境界条件

  • PQsendQueryParams が失敗した場合、NULL を返して終了する。
  • クエリ実行中にバックエンドで割り込み（停止要求など）が発生した場合、libpqsrv_get_result_last の挙動により処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル経由でインライン展開されます。
  • 拡張機能開発時のみ利用可能です。バックエンド本体のビルド時には使用が制限されています。
