StateFileChunk

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/twophase.c:996

目的

準備済みトランザクション（2PC）の情報をディスクへ永続化する際、メモリ上のデータを分割して保持するための連結リストのノードとして機能します。

シグネチャまたは形状

typedef struct StateFileChunk
{
	char	   *data;
	uint32		len;
	struct StateFileChunk *next;
} StateFileChunk;

入力と出力

  • data: 保持するデータブロックへのポインタ。
  • len: data が指すブロックのバイト長。
  • next: リスト内の次のチャンクへのポインタ。

前提条件と不変条件

  • twophase.c 内で、準備済みトランザクションの情報を WAL および状態ファイルへ書き出す際に使用されるチェーン構造の一部です。
  • リストの末尾は next が NULL となります。

アルゴリズムまたは振る舞い

  • 準備済みトランザクションの永続化プロセスにおいて、動的にメモリを確保（palloc0）し、チェーンを構築してデータを順次格納します。
  • 複数の StateFileChunk を連結することで、可変長のデータを効率的に管理します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/twophase.c 内の準備済みトランザクション永続化処理（palloc0 を用いたインスタンス生成が確認されています）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • GlobalTransactionData: 準備済みトランザクションの状態を管理する構造体。
  • TwoPhaseFileHeader: 状態ファイルのヘッダー情報。

失敗モードと境界条件

  • メモリ不足時: palloc0 が失敗した場合、PostgreSQL の標準的なエラー処理によりトランザクションがアボートされます。
  • 連結リストの操作: tail->next へのアクセス時に tail が正しく初期化されていない場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は twophase.c 内のローカルなデータ管理用です。
  • 連結リストの操作を行う際は、メモリリークを防ぐため、使用後の適切なメモリ解放（pfree）が行われているか確認してください。
