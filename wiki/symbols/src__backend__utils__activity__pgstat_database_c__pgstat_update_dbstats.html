pgstat_update_dbstats

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_database.c:322

目的

バックエンドがローカルに蓄積したトランザクション統計およびI/Oタイミング情報を、データベース単位の統計エントリへ転送・集計します。

シグネチャまたは形状

void pgstat_update_dbstats(TimestampTz ts)

入力と出力

  • **入力**: ts (現在時刻を示す TimestampTz)
  • **出力**: なし（副作用として共有メモリ上の統計エントリを更新）

前提条件と不変条件

  • MyDatabaseId が有効（OidIsValid）であること。無効な場合は処理を中断します。
  • 統計情報の更新は、pgstat_track_counts フラグが有効な環境下で呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. pgstat_prep_database_pending を呼び出し、現在のデータベースに対応するローカル統計エントリを取得します。 2. グローバル変数（pgStatXactCommit, pgStatXactRollback, pgStatBlockReadTime, pgStatBlockWriteTime）の値を、取得したエントリに加算します。 3. pgstat_should_report_connstat() が真の場合、セッション時間（session_time）、アクティブ時間（active_time）、トランザクション待機時間（idle_in_transaction_time）を計算し、更新します。 4. 転送完了後、加算元のグローバル変数をすべて 0 にリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/pgstat.c:758
  • **呼び出し先**:
  • pgstat_prep_database_pending (統計エントリの取得)
  • pgstat_should_report_connstat (接続統計報告の要否判定)
  • TimestampDifference (セッション時間の計算)

関連データ構造

  • PgStat_StatDBEntry: データベース単位の統計情報を保持する構造体。
  • グローバル変数群: pgStatXactCommit, pgStatXactRollback, pgStatBlockReadTime, pgStatBlockWriteTime, pgStatActiveTime, pgStatTransactionIdleTime。

失敗モードと境界条件

  • MyDatabaseId が InvalidOid の場合、統計の更新を行わずに即時リターンします。
  • 接続統計の報告対象外である場合、セッション関連のメトリクス更新はスキップされます。

保守担当者向けメモ

  • 本関数は、統計情報の「蓄積」から「集計」への橋渡しを行う重要なポイントです。
  • グローバル変数のリセット処理が含まれているため、呼び出しタイミングが統計の精度に直結します。
  • 共有メモリへのフラッシュ処理（pgstat_database_flush_cb）とは別のレイヤーであり、本関数はあくまでローカルな集計処理を担当します。
