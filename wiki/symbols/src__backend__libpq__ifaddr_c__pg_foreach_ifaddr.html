pg_foreach_ifaddr

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/ifaddr.c:229

目的

システムの全ネットワークインターフェースを列挙し、各インターフェースのアドレスとサブネットマスクに対して指定されたコールバック関数を適用します。

シグネチャまたは形状

int pg_foreach_ifaddr(PgIfAddrCallback callback, void *cb_data);

入力と出力

  • **入力**:
  • callback: 各インターフェースに対して呼び出される PgIfAddrCallback 型の関数ポインタ。
  • cb_data: コールバック関数に渡される任意のデータポインタ。
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時は -1 を返し、必要に応じて errno を設定します。

前提条件と不変条件

  • 本実装は Windows 環境（WIN32）における WSAIoctl を使用したインターフェース取得処理です。
  • 呼び出し側は、コールバック関数が適切にアドレスとマスクを処理できることを前提とします。

アルゴリズムまたは振る舞い

1. WSASocket を使用して UDP ソケットを作成します。 2. SIO_GET_INTERFACE_LIST を用いてシステム上のインターフェース情報を取得します。

  • バッファサイズが不足している場合（WSAEFAULT または WSAENOBUFS）、バッファを拡張しながら最大 1024 個のインターフェースまで再試行します。

    • 3. 取得したインターフェースリストを走査し、各要素に対して run_ifaddr_callback を呼び出します。 4. 最後にソケットを閉じ、割り当てたメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/hba.c: クライアントの接続元IPが許可範囲内かを確認する際に使用。
  • src/tools/ifaddrs/test_ifaddrs.c: テスト用ユーティリティ。
  • **呼び出し先**:
  • run_ifaddr_callback: 各インターフェース情報に対して実行される内部関数。

関連データ構造

  • PgIfAddrCallback: コールバック関数の型定義。
  • INTERFACE_INFO: Windows API で定義されるネットワークインターフェース情報構造体。

失敗モードと境界条件

  • ソケット作成失敗時、またはメモリ確保失敗時に -1 を返します。
  • インターフェース数が 1024 を超える場合や、WSAIoctl が予期せぬエラーを返した場合は処理を中断し -1 を返します。

保守担当者向けメモ

  • 本関数は OS 依存の実装であり、Windows 環境専用です。他の OS では ifaddrs.h 等を用いた別の実装がプリプロセッサ条件分岐によって選択されます。
  • 修正を行う際は、src/backend/libpq/ifaddr.c 内の該当する #ifdef WIN32 ブロックのみを確認してください。
