print_edge_table

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_misc.c:111

目的

遺伝的アルゴリズムの交叉処理（ERXなど）で使用されるエッジテーブル（Edge Table）の内容をデバッグ目的で可視化し、指定されたファイルストリームに出力します。

シグネチャまたは形状

void print_edge_table(FILE *fp, Edge *edge_table, int num_gene)

入力と出力

  • fp: 出力先のファイルストリーム。
  • edge_table: 表示対象の Edge 構造体の配列。
  • num_gene: 遺伝子の数（エッジテーブルの走査範囲）。
  • 戻り値: なし。

前提条件と不変条件

  • GEQO_DEBUG マクロが定義されている環境でのみ有効です。
  • edge_table は少なくとも num_gene 以上の要素を持つ有効な配列である必要があります。
  • edge_table[i] は unused_edges および edge_list メンバを持つ構造体である必要があります。

アルゴリズムまたは振る舞い

1. fp に対して "EDGE TABLE" というヘッダーを出力します。 2. 1から num_gene までのインデックスでループし、各遺伝子番号と、その遺伝子に関連付けられたエッジリスト（edge_list）の内容を順次出力します。 3. 出力終了後に改行を挿入し、fflush(fp) を呼び出してバッファをフラッシュし、即座に書き込みを確定させます。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内には、本関数を呼び出している具体的な関数名は記載されていません。
  • 呼び出し先: 標準ライブラリの fprintf および fflush を使用します。

関連データ構造

  • Edge: エッジテーブルを構成する構造体。unused_edges（未使用エッジ数）と edge_list（エッジのリスト）を保持します。

失敗モードと境界条件

  • fp が無効なストリームである場合、書き込み時にエラーが発生する可能性があります。
  • num_gene が edge_table の実際のサイズを超えている場合、メモリ境界外アクセスが発生するリスクがあります。

保守担当者向けメモ

  • 本関数はデバッグ用であり、本番環境のパフォーマンスには影響しません。
  • 遺伝子番号が1から始まる前提でループ処理が記述されているため、データ構造側のインデックス設計と整合しているか確認してください。
