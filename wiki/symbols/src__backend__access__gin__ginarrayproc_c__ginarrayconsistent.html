ginarrayconsistent

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginarrayproc.c:141

目的

GINインデックスにおいて、配列型に対する検索条件（演算子）とインデックス内のキーの整合性を判定し、検索結果が条件を満たすかを確認します。

シグネチャまたは形状

Datum ginarrayconsistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • check[] (bool*): 各インデックスキーがクエリ条件を満たすかのフラグ配列。
  • strategy (StrategyNumber): 実行する演算子戦略（Overlap, Contains, Contained, Equal）。
  • nkeys (int32): キーの数。
  • nullFlags[] (bool*): 各キーのNULLフラグ。
  • **出力**:
  • res (bool): 整合性判定結果。
  • recheck (bool*): インデックスによる判定が不完全な場合に true を設定し、再確認を要求するフラグ。

前提条件と不変条件

  • strategy が未知の値である場合、elog(ERROR) を発生させます。
  • GinContainedStrategy および GinEqualStrategy では、インデックスのみでは完全な判定ができないため、必ず recheck が true に設定されます。

アルゴリズムまたは振る舞い

strategy に基づき以下の判定を行います。

  • **GinOverlapStrategy**: いずれかの非NULLキーが一致すれば true。recheck は false。
  • **GinContainsStrategy**: 全てのキーが一致し、かつNULLが含まれていなければ true。recheck は false。
  • **GinContainedStrategy**: インデックスによる判定は不完全なため、常に true を返し、recheck を true に設定。
  • **GinEqualStrategy**: 全てのキーが一致すれば true。recheck は true。

呼び出し元と呼び出し先

  • 呼び出し元: ソースコード内に直接の呼び出し元情報は記載されていません。
  • 呼び出し先: elog（エラー発生時）。

関連データ構造

  • StrategyNumber: 演算子クラスの戦略番号。
  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約に基づく引数コンテナ。

失敗モードと境界条件

  • 未知の strategy が渡された場合、elog(ERROR) により処理が中断されます。
  • GinContainedStrategy および GinEqualStrategy は、インデックスの性質上、常に再確認（recheck）を必要とする境界条件として扱われます。

保守担当者向けメモ

  • 本関数は ginarraytriconsistent と同様の論理を扱いますが、こちらは2値論理（TRUE/FALSE）で判定を行います。
  • recheck フラグの制御は検索パフォーマンスに直結するため、ロジック変更時は注意が必要です。
