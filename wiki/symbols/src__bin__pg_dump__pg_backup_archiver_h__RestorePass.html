RestorePass

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_dump/pg_backup_archiver.h:201

目的

pg_restore におけるアーカイブ復元時の順序制御（マルチパス処理）を定義します。

シグネチャまたは形状

typedef enum {
    RESTORE_PASS_MAIN = 0,
    RESTORE_PASS_ACL,
    RESTORE_PASS_POST_ACL,
    #define RESTORE_PASS_LAST RESTORE_PASS_POST_ACL
} RestorePass;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: 復元処理の各フェーズを識別する定数

前提条件と不変条件

  • RESTORE_PASS_MAIN が 0 から開始されます。
  • RESTORE_PASS_LAST マクロにより、最終フェーズが RESTORE_PASS_POST_ACL であることが保証されます。

アルゴリズムまたは振る舞い

復元処理を以下の3段階に分割して制御します。 1. RESTORE_PASS_MAIN: 大半のTOCアイテムを処理するメインパス。 2. RESTORE_PASS_ACL: ACL（アクセス制御リスト）関連アイテムを処理するパス。 3. RESTORE_PASS_POST_ACL: イベントトリガーやマテリアライズドビューの更新など、ACL適用後に実行すべきアイテムを処理するパス。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/pg_backup_archiver.c 内の復元ロジック（_tocEntryRestorePass 等）から参照されています。
  • 呼び出し先: なし（列挙型のため）。

関連データ構造

  • ArchiveHandle: 並列復元時に restorePass メンバとして本型を保持します。
  • TocEntry: 各エントリがどのパスで復元されるかを判定するために使用されます。

失敗モードと境界条件

  • 境界条件: RESTORE_PASS_LAST を超える値が指定された場合の動作は定義されていません。

保守担当者向けメモ

  • 復元順序の変更を検討する際は、src/bin/pg_dump/pg_dump_sort.c のソートロジックおよび pg_backup_archiver.c のマルチパス構造との整合性に注意してください。
  • 並列復元時には ArchiveHandle 内の restorePass メンバが状態管理に使用されます。
