SaveCachedPlan

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/plancache.c:529

目的

CachedPlanSource を永続的なキャッシュとして登録し、バックエンドの生存期間中保持されるようにする。

シグネチャまたは形状

void SaveCachedPlan(CachedPlanSource *plansource)

入力と出力

  • **入力**: plansource (CachedPlanSource *) - 保存対象のキャッシュソース。
  • **出力**: なし。

前提条件と不変条件

  • plansource の magic フィールドが CACHEDPLANSOURCE_MAGIC であること。
  • plansource が完了状態 (is_complete) であること。
  • plansource が未保存 (!is_saved) であること。
  • is_oneshot フラグが立っていないこと（ワンショット計画は保存不可）。

アルゴリズムまたは振る舞い

1. 既存の汎用計画 (generic plan) が存在する場合、ReleaseGenericPlan を呼び出して破棄する。 2. plansource->context の親メモリコンテキストを CacheMemoryContext に変更し、永続化する。 3. saved_plan_list（グローバルリスト）の末尾に plansource を追加する。 4. plansource->is_saved を true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/prepare.c
  • src/backend/executor/functions.c
  • src/backend/executor/spi.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • ReleaseGenericPlan
  • MemoryContextSetParent
  • dlist_push_tail

関連データ構造

  • CachedPlanSource: クエリのソース情報を保持する構造体。
  • saved_plan_list: 永続化された CachedPlanSource を管理するグローバルな双方向リスト。
  • CacheMemoryContext: キャッシュを永続保持するためのメモリコンテキスト。

失敗モードと境界条件

  • is_oneshot が true の場合、elog(ERROR) により処理が中断される。
  • 呼び出し順序が不適切な場合（is_complete が偽など）、Assert によりデバッグビルドで停止する。

保守担当者向けメモ

  • 本関数は、計画生成前に呼び出されることが想定されている。
  • 汎用計画の破棄は、メモリ管理の安全性を確保するための措置である。
