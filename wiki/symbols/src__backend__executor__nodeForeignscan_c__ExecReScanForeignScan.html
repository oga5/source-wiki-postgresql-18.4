ExecReScanForeignScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeForeignscan.c:322

目的

外部テーブルスキャンノードの状態をリセットし、スキャンを最初から再実行可能にする。

シグネチャまたは形状

void ExecReScanForeignScan(ForeignScanState *node)

入力と出力

  • **入力**: ForeignScanState *node（スキャン対象の外部テーブル実行状態）
  • **出力**: なし（副作用としてノードの状態を更新）

前提条件と不変条件

  • node は初期化済みであること。
  • EvalPlanQual（EPQ）実行中かつ CMD_SELECT 以外の操作（直接更新等）である場合、本関数は何も行わず即座に復帰する。

アルゴリズムまたは振る舞い

1. **EPQチェック**: estate->es_epq_active が有効かつ plan->operation が CMD_SELECT でない場合、処理をスキップする。 2. **FDWリセット**: node->fdwroutine->ReScanForeignScan(node) を呼び出し、FDW固有のリセット処理を実行する。 3. **サブプランの再スキャン**: outerPlan が存在し、かつ chgParam が NULL の場合、ExecReScan(outerPlan) を呼び出してサブプランを再スキャンする。 4. **スキャン状態の更新**: ExecScanReScan(&node->ss) を呼び出し、共通のスキャン状態をリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c（ExecReScan 経由での呼び出しが確認されている）。
  • **呼び出し先**:
  • node->fdwroutine->ReScanForeignScan
  • ExecReScan
  • ExecScanReScan

関連データ構造

  • ForeignScanState: 外部テーブルスキャンの実行状態を保持する構造体。
  • FdwRoutine: 外部データソースごとの操作関数ポインタ群。
  • ForeignScan: 実行計画ノード。

失敗モードと境界条件

  • outerPlan が NULL の場合、サブプランの再スキャンはスキップされる。
  • outerPlan->chgParam が NULL でない場合、サブプランは後続の ExecProcNode 呼び出し時に再スキャンされるため、本関数内では再スキャンされない。

保守担当者向けメモ

  • 本関数は「薄いラッパー」であり、実際の再スキャンロジックは fdwroutine に委譲されている。
  • EvalPlanQual 実行中の直接更新（DirectModify）は無視される仕様である点に注意すること。
