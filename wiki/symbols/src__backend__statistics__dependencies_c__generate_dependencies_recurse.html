generate_dependencies_recurse

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/dependencies.c:90

目的

関数従属性の計算に必要な、属性の組み合わせ（順列）を再帰的に生成します。

シグネチャまたは形状

static void generate_dependencies_recurse(DependencyGenerator state, int index, AttrNumber start, AttrNumber *current);

入力と出力

  • **入力**:
  • state: 属性生成の状態管理構造体。
  • index: 現在生成中の組み合わせのインデックス。
  • start: 組み合わせ生成を開始する属性番号。
  • current: 現在生成中の属性の組み合わせを保持する配列。
  • **出力**: なし（副作用として state->dependencies に結果を格納）。

前提条件と不変条件

  • state->k は生成する組み合わせのサイズ（要素数）を保持している必要があります。
  • state->n は対象となる属性の総数を保持している必要があります。
  • index < (state->k - 1) の間、current 配列の要素は昇順で生成されます。

アルゴリズムまたは振る舞い

1. **再帰ステップ (index < k - 1)**:

  • start から state->n までの範囲でループし、current[index] に値を設定して再帰的に呼び出します。これにより、最初の k-1 個の要素が昇順で生成されます。

    • 2. **終端ステップ (index == k - 1)**:

  • 最後の要素を決定します。0 から state->n までの値のうち、既に current に含まれていない値を探します。
  • 重複がない場合、その組み合わせを state->dependencies 配列に追加し、state->ndependencies をインクリメントします。
  • repalloc を使用して、必要に応じて state->dependencies のメモリ領域を拡張します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/dependencies.c 内の generate_dependencies 関数から呼び出されます。
  • 本関数自身が再帰的に呼び出されます。
  • **呼び出し先**:
  • repalloc (メモリ管理)
  • memcpy (メモリコピー)

関連データ構造

  • DependencyGenerator: 属性の組み合わせ生成プロセスを管理する構造体。dependencies（結果格納用配列）、k（組み合わせサイズ）、n（属性数）、ndependencies（生成済み数）を保持します。

失敗モードと境界条件

  • repalloc がメモリ不足により失敗した場合、PostgreSQL の標準的なエラー処理によりアボートします。
  • state->n が state->k 未満の場合、組み合わせは生成されません。

保守担当者向けメモ

  • 本関数は k 個の属性の組み合わせを生成しますが、最後の要素のみが他の要素と重複しないことを条件としています。これは関数従属性の定義（a -> b のようなペア）を効率的に列挙するための実装です。
  • repalloc をループ内で呼び出しているため、生成される組み合わせ数が多い場合、メモリ再割り当てのコストが増大する可能性があります。
