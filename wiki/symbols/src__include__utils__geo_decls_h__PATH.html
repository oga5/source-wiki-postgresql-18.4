PATH

定義

  • 種別: struct
  • 定義箇所: src/include/utils/geo_decls.h:115

目的

PostgreSQLにおける幾何データ型「PATH（パス）」を表現するためのC言語構造体。SQLレベルのパス型とメモリ上のデータ構造を橋渡しする。

シグネチャまたは形状

typedef struct {
    int32       vl_len_;
    int32       npts;
    int32       closed;
    int32       dummy;
    Point       p[FLEXIBLE_ARRAY_MEMBER];
} PATH;

入力と出力

  • **入力**: PG_GETARG_PATH_P(0) 等のマクロを通じて、PostgreSQLの Datum から本構造体へのポインタとして取得される。
  • **出力**: 幾何演算の結果として、メモリ上に確保された PATH 構造体へのポインタが返される。

前提条件と不変条件

  • varlena 型（可変長データ型）であるため、アクセス前に必ず PG_DETOAST_DATUM を経由してTOAST解除を行う必要がある。
  • vl_len_ フィールドは直接操作してはならない。
  • p フィールドは FLEXIBLE_ARRAY_MEMBER を使用しており、構造体の末尾に可変長の Point 配列が続く。

アルゴリズムまたは振る舞い

  • npts に格納された点数に基づき、動的にメモリを確保してパスを構築する。
  • closed フラグにより、パスが閉じているか（多角形を形成するか）を判定する。
  • dummy フィールドは、double 型のアライメントを維持するためのパディングとして機能する。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • src/backend/utils/adt/geo_ops.c: PG_GETARG_PATH_P を用いた引数取得、および palloc によるメモリ確保と初期化。
  • **呼び出し先**:
  • 構造体定義のため、特定の関数を呼び出すことはない。ただし、Point 型の配列を保持する。

関連データ構造

  • Point: パスを構成する各頂点の座標を保持する構造体。

失敗モードと境界条件

  • **メモリ不足**: palloc 失敗時にエラーが発生する。
  • **不正な点数**: npts が負の値や、メモリ確保時に INT_MAX を超えるような不正な値である場合、geo_ops.c 内のチェックロジックにより拒否される。
  • **NaNの扱い**: 幾何演算全般の制約として、NaNが含まれる場合は比較関数が FALSE を返す。

保守担当者向けメモ

  • 本構造体は varlena 型であるため、メモリレイアウトの変更には細心の注意を払うこと。
  • dummy フィールドはアライメント調整用であり、削除や変更を行うとプラットフォーム依存のクラッシュを招く可能性がある。
  • 数値解析の専門的な最適化は施されていないため、厳密な計算が必要な場合は注意が必要。
