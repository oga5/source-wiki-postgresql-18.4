INT128

定義

  • 種別: struct
  • 定義箇所: src/include/common/int128.h:107

目的

コンパイラがネイティブな128ビット整数型をサポートしていない環境において、64ビット整数のペアを用いて128ビット整数演算を実現するためのポータブルなデータ構造を提供します。

シグネチャまたは形状

typedef struct {
#ifdef WORDS_BIGENDIAN
    int64  hi; /* 上位64ビット（符号を含む） */
    uint64 lo; /* 下位64ビット（符号なし） */
#else
    uint64 lo; /* 下位64ビット（符号なし） */
    int64  hi; /* 上位64ビット（符号を含む） */
#endif
} INT128;

入力と出力

本構造体はデータ保持用であり、直接的な入出力関数ではありません。関連する演算関数（int128_add_int64_mul_int64 等）の引数または戻り値として使用されます。

前提条件と不変条件

  • WORDS_BIGENDIAN の定義に基づき、メモリレイアウトが決定されます。
  • ソフトウェア実装時、ネイティブな int128 型のメモリレイアウトと整合するように hi と lo が配置されます。
  • 算術右シフト操作が正しく機能する環境である必要があります。

アルゴリズムまたは振る舞い

  • ネイティブ型が利用できない環境では、この構造体を用いて加算や乗算が行われます。
  • 加算時は lo 側の加算を行い、キャリーが発生した際に hi 側を調整します。
  • 乗算時は64ビット入力を32ビット単位に分割し、部分積を計算して128ビットの範囲内で加算します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/timestamp.c において、interval の比較や線形表現の保持に使用されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な呼び出し先はありません。演算ロジックは src/include/common/int128.h 内のインライン関数群によって提供されます。

関連データ構造

  • int64, uint64: 構成要素となる64ビット整数型。
  • int128 (ネイティブ型): コンパイラがサポートする場合の代替型。

失敗モードと境界条件

  • ソフトウェア実装において、算術右シフトが期待通りに動作しない環境では計算結果が不正になる可能性があります（コンパイル時に StaticAssertDecl で検証されます）。
  • ネイティブ型が利用可能な環境（USE_NATIVE_INT128）での利用が推奨されており、ソフトウェア実装はパフォーマンス上のオーバーヘッドを伴います。

保守担当者向けメモ

  • 本構造体はプラットフォーム依存性を吸収するための抽象化レイヤーです。変更を加える際は、エンディアン依存のレイアウトがネイティブな int128 と一致していることを確認してください。
  • ソフトウェア実装のロジックは複雑なビット操作を含むため、src/tools/testint128.c を用いた検証が必須です。
