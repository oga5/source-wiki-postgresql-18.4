PLpgSQL_nsitem_type

定義

  • 種別: enum
  • 定義箇所: src/pl/plpgsql/src/plpgsql.h:42

目的

PL/pgSQL の名前空間（namespace）に登録されるアイテムの種類を識別するために使用されます。

シグネチャまたは形状

typedef enum PLpgSQL_nsitem_type
{
    PLPGSQL_NSTYPE_LABEL,    /* ブロックラベル */
    PLPGSQL_NSTYPE_VAR,      /* スカラー変数 */
    PLPGSQL_NSTYPE_REC,      /* 複合変数 */
} PLpgSQL_nsitem_type;

入力と出力

  • 本シンボルは列挙型であり、関数への引数や構造体のメンバとして利用されます。
  • 主な出力先は、名前空間管理関数（plpgsql_ns_additem 等）の識別子としての利用です。

前提条件と不変条件

  • 名前空間に追加される各アイテムは、必ずこのいずれかの型に分類される必要があります。

アルゴリズムまたは振る舞い

  • コンパイル時、名前空間に識別子を登録する際に、その識別子が「ラベル」「スカラー変数」「複合変数」のいずれであるかを区別するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（利用箇所）**:
  • src/pl/plpgsql/src/pl_comp.c: add_parameter_name 関数にてパラメータの型指定に使用。
  • src/pl/plpgsql/src/pl_funcs.c: plpgsql_ns_additem 関数にて名前空間へのアイテム追加時に使用。
  • **呼び出し先**:
  • 本シンボル自体は列挙型であり、直接的な関数呼び出しは行いません。

関連データ構造

  • PLpgSQL_nsitem（名前空間アイテム構造体、本型をメンバとして保持）
  • PLpgSQL_execstate（実行時状態管理）

失敗モードと境界条件

  • 定義外の値をキャストして代入した場合の挙動は未定義です。

保守担当者向けメモ

  • 新しい名前空間アイテムの種類（例: カーソルやエイリアスなど）を追加する必要がある場合は、この列挙型を拡張してください。
