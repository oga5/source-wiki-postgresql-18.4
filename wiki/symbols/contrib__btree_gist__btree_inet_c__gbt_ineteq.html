gbt_ineteq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_inet.c:38

目的

inet 型の GiST インデックスにおいて、2つの値が等しいかどうかを判定する比較関数です。

シグネチャまたは形状

static bool gbt_ineteq(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象となる double 型へ変換済みの値へのポインタ。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • bool: a と b が等しい場合に true、それ以外の場合に false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、double 型の値を指している必要があります。
  • 本関数は inet 型の値を double に変換した後のスカラ比較を行うため、変換後の値が等価であることに依存します。

アルゴリズムまたは振る舞い

1. 引数 a と b をそれぞれ const double * にキャストします。 2. キャストされたポインタが指す double 値同士を比較演算子 == で評価します。 3. 比較結果をそのまま戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_inet.c 内の tinfo 構造体（gbtree_ninfo）の初期化において、比較関数として参照されています。
  • **呼び出し先**: 直接的な関数呼び出しはありません。

関連データ構造

  • inetKEY: inet 型のインデックス格納用構造体。本関数は、この構造体に含まれるスカラ変換後の値に対して操作を行います。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、未定義の動作を引き起こします。
  • double 型への変換精度に依存するため、非常に大きなネットワーク範囲や複雑なサブネットマスクの比較において、期待される inet 型の等価性と一致しない可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.c が提供する汎用的な数値比較ロジックの一部として機能します。
  • flinfo 引数は現在の実装では使用されていませんが、将来的な拡張やインターフェースの互換性のために保持されています。
