PQencryptPasswordConn

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/libpq/fe-auth.c:1405

目的

クライアント側でパスワードを暗号化し、サーバーへ送信するためのハッシュ値を生成する。SQLコマンド（ALTER USER 等）でパスワードを平文で送ることを避けるための公開APIである。

シグネチャまたは形状

char *PQencryptPasswordConn(PGconn *conn, const char *passwd, const char *user, const char *algorithm);

入力と出力

  • **入力**:
  • conn: 接続オブジェクト。
  • passwd: 暗号化対象の平文パスワード。
  • user: ユーザー名（MD5ハッシュ計算に使用）。
  • algorithm: 使用するアルゴリズム名（scram-sha-256, md5 等）。NULL の場合、サーバーの password_encryption 設定を照会する。
  • **出力**:
  • 暗号化されたパスワード文字列（成功時）。呼び出し元で free() する必要がある。
  • NULL（失敗時。conn->errorMessage に詳細が格納される）。

前提条件と不変条件

  • conn が有効であること。
  • algorithm が NULL の場合、サーバーへの SHOW password_encryption クエリが成功し、有効なアルゴリズム名が返される必要がある。
  • algorithm に "on" または "off" が指定された場合は "md5" として扱われる。

アルゴリズムまたは振る舞い

1. conn が無効な場合は NULL を返す。 2. algorithm が指定されていない場合、PQexec でサーバーの password_encryption 設定を取得する。 3. アルゴリズムが scram-sha-256 の場合、pg_fe_scram_build_secret を呼び出す。 4. アルゴリズムが md5 の場合、pg_md5_encrypt を使用してハッシュを生成する。 5. 未知のアルゴリズムが指定された場合はエラーを記録し NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/scripts/createuser.c 等のクライアントツール。
  • **呼び出し先**: PQexec（アルゴリズム取得時）、pg_fe_scram_build_secret、pg_md5_encrypt。

関連データ構造

  • PGconn: 接続状態およびエラーメッセージ保持に使用。
  • PGresult: サーバー設定取得時に使用。

失敗モードと境界条件

  • conn が NULL の場合、即座に NULL を返す。
  • サーバー設定取得失敗時や、未知のアルゴリズム指定時には libpq_append_conn_error を通じてエラーを記録し NULL を返す。
  • メモリ不足時には NULL を返す。

保守担当者向けメモ

  • 本関数は password_encryption の設定値として "on"/"off" を受け入れるが、これは歴史的経緯によるものであり、内部的には "md5" に変換される。
  • 新しい認証アルゴリズムを追加する場合は、strcmp による分岐条件を拡張する必要がある。
