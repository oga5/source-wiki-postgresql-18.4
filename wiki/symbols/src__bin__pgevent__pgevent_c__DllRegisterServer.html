DllRegisterServer

定義

  • 種別: function
  • 定義箇所: src/bin/pgevent/pgevent.c:64

目的

Windowsイベントログシステムに対し、PostgreSQLのイベントソース情報をレジストリへ登録する。

シグネチャまたは形状

STDAPI DllRegisterServer(void)

入力と出力

  • **入力**: グローバル変数 g_module（DLLハンドル）および event_source（イベントソース名）。
  • **出力**: HRESULT 型のステータスコード。成功時は S_OK、失敗時は SELFREG_E_TYPELIB を返す。

前提条件と不変条件

  • 実行には HKEY_LOCAL_MACHINE を操作するための管理者権限が必要。
  • g_module が DllMain によって正しく初期化されていること。

アルゴリズムまたは振る舞い

1. GetModuleFileName を使用して、現在のDLLのフルパスを取得する。 2. SYSTEM\CurrentControlSet\Services\EventLog\Application\<event_source> キーをレジストリに作成する。 3. 作成したキー配下に以下の値を設定する:

  • EventMessageFile: DLLのフルパス（REG_EXPAND_SZ）。
  • TypesSupported: サポートするイベントタイプ（エラー、警告、情報）のビットマスク（REG_DWORD）。

    • 4. レジストリキーを閉じ、成功を返す。 5. 各ステップで失敗した場合は、エラーメッセージボックスを表示し、SELFREG_E_TYPELIB を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: regsvr32 コマンド、および src/bin/pgevent/pgevent.c 内の DllInstall 関数から呼び出される。
  • **呼び出し先**: Windows API（GetModuleFileName, RegCreateKey, RegSetValueEx, RegCloseKey, MessageBox）。

関連データ構造

  • HKEY: レジストリキーハンドル。
  • event_source: レジストリパスの一部として使用される文字列変数。

失敗モードと境界条件

  • GetModuleFileName の失敗: DLLパス取得不可時に SELFREG_E_TYPELIB を返す。
  • RegCreateKey の失敗: レジストリキー作成権限がない場合などに SELFREG_E_TYPELIB を返す。
  • RegSetValueEx の失敗: レジストリ値の書き込み失敗時に SELFREG_E_TYPELIB を返す。

保守担当者向けメモ

  • regsvr32 の仕様により、インストール時に DllInstall の前に呼び出される場合がある。意図しない登録を防ぐため、regsvr32 /n オプションの使用が推奨される。
  • レジストリ操作は HKEY_LOCAL_MACHINE に対して行われるため、実行環境の権限に注意すること。
