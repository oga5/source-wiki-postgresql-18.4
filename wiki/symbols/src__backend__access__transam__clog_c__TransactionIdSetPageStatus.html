TransactionIdSetPageStatus

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/clog.c:292

目的

指定されたトランザクションID（XID）およびサブトランザクションの状態を、CLOG（Commit Log）の特定のページに書き込む。

シグネチャまたは形状

static void TransactionIdSetPageStatus(
    TransactionId xid, 
    int nsubxids, 
    TransactionId *subxids, 
    XidStatus status, 
    XLogRecPtr lsn, 
    int64 pageno, 
    bool all_xact_same_page
)

入力と出力

  • **入力**:
  • xid: 対象のトランザクションID。
  • nsubxids: サブトランザクションの数。
  • subxids: サブトランザクションIDの配列。
  • status: 設定する状態（XidStatus）。
  • lsn: 関連する WAL の LSN。
  • pageno: 対象の CLOG ページ番号。
  • all_xact_same_page: すべての XID が同一ページ内にあるかを示すフラグ。
  • **出力**: なし（副作用として CLOG ページを更新）。

前提条件と不変条件

  • THRESHOLD_SUBTRANS_CLOG_OPT が PGPROC_MAX_CACHED_SUBXIDS 以下であること（静的アサーション）。
  • XactCtl が初期化されていること。

アルゴリズムまたは振る舞い

1. **ロック取得**: SimpleLruGetBankLock を使用して、対象ページに対応する SLRU バンクロックを取得する。 2. **グループ更新の試行**: 以下の条件を満たす場合、ロック競合を回避するためにグループ更新を試みる。

  • all_xact_same_page が真。
  • xid が現在のプロセス（MyProc）の XID と一致。
  • サブトランザクション数が最適化閾値以下であり、かつ MyProc の情報と一致する。
  • LWLockConditionalAcquire で即座にロックが取れれば直接更新し、取れなければ TransactionGroupUpdateXidStatus を呼び出す。

    • 3. **通常更新**: グループ更新が適用できない、または成功しなかった場合、LWLockAcquire で排他ロックを確実に取得し、TransactionIdSetPageStatusInternal を呼び出して状態を更新する。 4. **終了**: ロックを解放して処理を完了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/clog.c 内の TransactionIdSetTreeStatus 等から呼び出される。
  • **呼び出し先**:
  • SimpleLruGetBankLock
  • TransactionIdSetPageStatusInternal
  • TransactionGroupUpdateXidStatus
  • LWLockAcquire / LWLockRelease / LWLockConditionalAcquire

関連データ構造

  • XactCtl: CLOG の共有メモリ状態を管理する構造体。
  • MyProc: 現在のバックエンドプロセスの情報を保持する構造体。
  • SLRU: CLOG が利用するページキャッシュ機構。

失敗モードと境界条件

  • **ロック競合**: ページへのアクセスが集中している場合、グループ更新メカニズムへフォールバックする。
  • **ページ境界**: all_xact_same_page が偽の場合、グループ更新はスキップされ、通常のロック取得処理へ移行する。

保守担当者向けメモ

  • 本関数は TransactionIdSetTreeStatus の下位ルーチンであり、CLOG への書き込みにおける排他制御と最適化の分岐点となる。
  • グループ更新の条件判定（MyProc との照合）は、並列処理における整合性維持のために重要である。
