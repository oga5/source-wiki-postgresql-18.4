GetForeignServerIdByRelId

定義

  • 種別: function
  • 定義箇所: src/backend/foreign/foreign.c:355

目的

指定された外部テーブルのリレーションID（relid）から、対応する外部サーバーのOIDを取得します。

シグネチャまたは形状

Oid GetForeignServerIdByRelId(Oid relid)

入力と出力

  • **入力**: relid (Oid) - 検索対象の外部テーブルのリレーションID。
  • **出力**: serverid (Oid) - 外部テーブルに関連付けられた外部サーバーのOID。

前提条件と不変条件

  • 入力された relid がシステムカタログ pg_foreign_table に存在する必要があります。存在しない場合、関数は elog(ERROR) を発生させます。

アルゴリズムまたは振る舞い

1. FOREIGNTABLEREL システムキャッシュを使用して、指定された relid に対応する pg_foreign_table のタプルを検索します。 2. タプルが見つからない場合はエラーを報告します。 3. タプルから ftserver フィールド（外部サーバーID）を抽出します。 4. ReleaseSysCache を呼び出してタプルの参照を解放します。 5. 抽出した serverid を戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c
  • src/backend/foreign/foreign.c
  • src/backend/optimizer/util/plancat.c
  • **呼び出し先**:
  • SearchSysCache1 (システムキャッシュ検索)
  • ReleaseSysCache (キャッシュ解放)
  • elog (エラー報告)

関連データ構造

  • Form_pg_foreign_table: pg_foreign_table カタログのタプル構造体。ftserver フィールドを保持します。

失敗モードと境界条件

  • 指定された relid が外部テーブルとして存在しない場合、elog(ERROR) によりトランザクションが中断されます。

保守担当者向けメモ

  • 本関数はシステムキャッシュを利用するため、カタログの更新頻度が高い環境でも効率的に動作します。
  • 戻り値の Oid はキャッシュ解放後も有効な識別子として利用可能です。
