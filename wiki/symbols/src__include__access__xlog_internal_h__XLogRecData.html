XLogRecData

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlog_internal.h:312

目的

WAL（先行書き込みログ）レコードを構成するデータ断片を連結リストとして保持し、複数のメモリ領域から一つのWALレコードを組み立てるための構造体です。

シグネチャまたは形状

typedef struct XLogRecData
{
    struct XLogRecData *next;    /* 次の要素へのポインタ、またはNULL */
    const void *data;            /* リソースマネージャデータの開始位置 */
    uint32      len;             /* データの長さ */
} XLogRecData;

入力と出力

  • **入力**: 連結リストの各ノードは、WALレコードの一部となるデータ領域（data）とそのサイズ（len）を保持します。
  • **出力**: リストの末尾は next が NULL であることで示されます。

前提条件と不変条件

  • WALレコードの先頭となる XLogRecData は、レコードヘッダーを指す必要があります。
  • リストのチェーンは、XLogInsertRecord や XLogRecordAssemble などの関数によって構築・管理されます。

アルゴリズムまたは振る舞い

  • WAL書き込み時、複数のデータ断片を XLogRecData でチェーン状に繋ぐことで、メモリ上の散在したデータをコピーすることなく、一連のWALレコードとして構成します。
  • WAL読み込み時、XLogReaderState はこの構造体定義を参照し、レコードの継続やヘッダー情報を解析します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/access/transam/xlog.c: XLogInsertRecord および CopyXLogRecordToWAL でレコードデータの組み立てに使用。
  • src/backend/access/transam/xloginsert.c: XLogRecordAssemble によるチェーン構築および rdata_head / rdata_tail 等での管理。
  • src/backend/access/transam/multixact.c: ステータスフラグやXIDの保持に使用。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • XLogReaderState: WAL読み込み時に本構造体を利用してレコードを復元します。
  • RmgrData: リソースマネージャのメソッドテーブル。WALレコードの処理時に連携します。

失敗モードと境界条件

  • next ポインタが正しく終端（NULL）されていない場合、WALレコードの読み込みや書き込み処理でメモリ破壊やセグメンテーション違反が発生する可能性があります。
  • len が実際のデータサイズと一致しない場合、WALログの破損を招きます。

保守担当者向けメモ

  • 本構造体はフロントエンド（pg_receivewal 等）からもインクルードされるため、バックエンド専用のロジックを含めないようにしてください。
  • XLogRecData のチェーン構築ロジックを変更する場合は、xloginsert.c のアセンブル処理との整合性に注意してください。
