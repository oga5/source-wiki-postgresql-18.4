ForeignDataWrapper

定義

  • 種別: struct
  • 定義箇所: src/include/foreign/foreign.h:24

目的

PostgreSQLにおける外部データラッパー（FDW）の定義情報を保持し、システムカタログから取得したメタデータを管理するための構造体です。

シグネチャまたは形状

typedef struct ForeignDataWrapper
{
	Oid			fdwid;			/* FDW Oid */
	Oid			owner;			/* FDW owner user Oid */
	char	   *fdwname;		/* Name of the FDW */
	Oid			fdwhandler;		/* Oid of handler function, or 0 */
	Oid			fdwvalidator;	/* Oid of validator function, or 0 */
	List	   *options;		/* fdwoptions as DefElem list */
} ForeignDataWrapper;

入力と出力

本構造体は、GetForeignDataWrapper などの関数を通じてシステムキャッシュから情報を読み込み、生成されます。出力として、FDWの識別子、所有者、ハンドラ関数、バリデータ関数、およびオプション設定のリストを提供します。

前提条件と不変条件

  • options フィールドは DefElem のリストとして保持されます。
  • システムキャッシュから情報をコピーして作成されるため、使用後は呼び出し元で適切なメモリ管理（pfree 等）を行う必要があります。

アルゴリズムまたは振る舞い

システムカタログ pg_foreign_data_wrapper から取得した HeapTuple の値を、GETSTRUCT および SysCacheGetAttr を用いて本構造体の各フィールドにマッピングします。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • src/backend/commands/foreigncmds.c（GetForeignDataWrapper を介した取得）
  • contrib/dblink/dblink.c
  • contrib/file_fdw/file_fdw.c
  • src/backend/catalog/objectaddress.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**: 情報なし。

関連データ構造

  • List: options フィールドで使用。
  • Oid: 各種識別子に使用。
  • DefElem: options リストの要素として使用。

失敗モードと境界条件

  • fdwhandler および fdwvalidator は、設定されていない場合に 0（InvalidOid）となります。
  • オブジェクト取得時に MISSING_OK フラグを使用しない場合、対象のFDWが存在しないとエラーが発生します。

保守担当者向けメモ

  • 本構造体は src/backend/foreign/foreign.c で実装される関数群によって値が設定されます。
  • options は各FDWの実装側で解析される設計となっているため、変更を加える際は各FDW側の解析ロジックとの整合性に注意してください。
