sql_fn_parser_setup

定義

  • 種別: function
  • 定義箇所: src/backend/executor/functions.c:339

目的

SQL関数の解析時に使用されるパーサーフックをセットアップし、列参照やパラメータ参照の解決ロジックを構成する。

シグネチャまたは形状

void sql_fn_parser_setup(struct ParseState *pstate, SQLFunctionParseInfoPtr pinfo)

入力と出力

  • pstate: 解析状態を保持する ParseState 構造体（入力・出力）。
  • pinfo: SQL関数の解析に必要な情報を含む SQLFunctionParseInfoPtr（入力）。

前提条件と不変条件

  • pstate は有効なポインタである必要がある。
  • pinfo は、関数内の列参照やパラメータ参照を解決するためのコンテキスト情報として pstate->p_ref_hook_state に格納される。

アルゴリズムまたは振る舞い

1. pstate->p_pre_columnref_hook を NULL に設定する。 2. pstate->p_post_columnref_hook を sql_fn_post_column_ref に設定する。 3. pstate->p_paramref_hook を sql_fn_param_ref に設定する。 4. pstate->p_ref_hook_state に pinfo を割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_proc.c
  • src/backend/commands/functioncmds.c
  • src/backend/optimizer/util/clauses.c
  • **呼び出し先**:
  • 本関数はフック関数（sql_fn_post_column_ref, sql_fn_param_ref）を ParseState に登録するのみであり、直接的な関数呼び出しは行わない。

関連データ構造

  • ParseState: パーサーの実行状態を管理する構造体。
  • SQLFunctionParseInfo: SQL関数の解析に必要なメタデータ。

失敗モードと境界条件

  • pstate が NULL の場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は ParserSetupHook として利用される。
  • p_coerce_param_hook は使用されないため、明示的に設定をスキップしている。
