test_lfind32

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_lfind/test_lfind.c:119

目的

PostgreSQLの最適化された32ビット値線形探索関数 pg_lfind32 の正確性を検証するためのテスト関数です。

シグネチャまたは形状

Datum test_lfind32(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS (SQLインターフェース経由)
  • 出力: PG_RETURN_VOID() (成功時)
  • 失敗時: elog(ERROR) によりトランザクションをアボートし、エラーメッセージを出力します。

前提条件と不変条件

  • TEST_ARRAY_SIZE は 135 で固定されています。
  • テスト対象の配列 test_array は 0 で初期化され、インデックス 8, 64, 134 にそれぞれ値 1, 2, 3 が配置されます。

アルゴリズムまたは振る舞い

1. 135要素の uint32 配列を確保し、特定のインデックスに値を配置します。 2. pg_lfind32 を呼び出し、以下の条件を検証します。

  • 探索範囲がキーの配置位置より小さい場合に、値が見つからないこと。
  • 探索範囲がキーの配置位置を含む場合に、値が正しく見つかること。

    • 3. 存在しない値（4）を検索し、見つからないことを確認します。 4. 全ての検証が成功した場合に PG_RETURN_VOID() を返します。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLの関数管理インターフェース（PG_FUNCTION_INFO_V1(test_lfind32) を通じてSQLから呼び出し可能）。
  • 呼び出し先: pg_lfind32（検証対象）、elog（エラー報告用）。

関連データ構造

  • uint32 test_array[135]: テスト用のデータバッファ。

失敗モードと境界条件

  • 期待値と異なる結果（見つかるべき値が見つからない、またはその逆）が得られた場合、elog(ERROR) が発生します。
  • ベクトル演算の境界（SIMD処理の範囲）と、末尾の端数処理（テール処理）が発生する範囲の両方でテストが行われます。

保守担当者向けメモ

  • 本関数はSQL経由で実行されるユニットテストです。
  • pg_lfind32 の実装に変更を加えた際は、本関数がカバーするインデックス（8, 64, 134）および探索範囲が、SIMDレジスタの境界条件を適切に網羅しているか確認してください。
