build_join_rel_hash

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/relnode.c:485

目的

結合リレーション（joinrel）を高速に検索するためのハッシュテーブルを構築し、既存の結合リレーションをそのテーブルに登録する。

シグネチャまたは形状

static void build_join_rel_hash(PlannerInfo *root)

入力と出力

  • **入力**: root (PlannerInfo *) - 現在のクエリプランナの情報を保持する構造体。
  • **出力**: なし（副作用として root->join_rel_hash にハッシュテーブルが設定される）。

前提条件と不変条件

  • root->join_rel_list に既に存在する結合リレーションが格納されていること。
  • root->join_rel_hash がまだ初期化されていない状態で呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. bitmap_hash および bitmap_match を使用して、Relids をキーとするハッシュテーブル（JoinRelHashTable）を作成する。 2. root->join_rel_list を走査し、リスト内の各 RelOptInfo を取得する。 3. 各リレーションの relids をキーとしてハッシュテーブルに挿入し、JoinHashEntry にリレーションへのポインタを格納する。 4. 構築したハッシュテーブルを root->join_rel_hash に代入する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/relnode.c 内の結合リレーション管理関数（find_join_rel 等からの間接的なトリガー）。
  • **呼び出し先**: hash_create（ハッシュテーブル生成）、hash_search（エントリ挿入）。

関連データ構造

  • PlannerInfo: 結合リレーションのリストとハッシュテーブルを保持する。
  • RelOptInfo: 結合リレーションの情報を保持する構造体。
  • JoinHashEntry: ハッシュテーブル内のエントリ構造体。
  • Relids: ハッシュキーとして使用されるビットマップ。

失敗モードと境界条件

  • メモリ不足時には hash_create がエラーを発生させる可能性がある。
  • Assert(!found) により、既にハッシュテーブルに存在するキーを挿入しようとした場合はアサーション違反となる。

保守担当者向けメモ

  • 本関数は結合リレーションの数が増加した際に動的に呼び出される。
  • Relids をキーとしているため、結合対象のリレーション集合が同一であれば同一のハッシュエントリにマッピングされる。
