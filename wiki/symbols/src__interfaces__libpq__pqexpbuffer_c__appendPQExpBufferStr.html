appendPQExpBufferStr

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/pqexpbuffer.c:366

目的

PQExpBuffer 構造体が管理するバッファの末尾に、ヌル終端文字列を安全に追加します。

シグネチャまたは形状

void appendPQExpBufferStr(PQExpBuffer str, const char *data)

入力と出力

  • str (入力/出力): 操作対象の PQExpBuffer 構造体へのポインタ。
  • data (入力): 追加するヌル終端文字列。
  • 戻り値: なし。

前提条件と不変条件

  • str は initPQExpBuffer 等で初期化済みであること。
  • data は有効なヌル終端文字列であること。
  • バッファが「broken」状態（メモリ不足等）にある場合、本関数は内部的に処理をスキップし、何も行いません。

アルゴリズムまたは振る舞い

1. strlen(data) を使用して追加する文字列の長さを算出します。 2. appendBinaryPQExpBuffer を呼び出し、算出した長さ分のデータをバッファの末尾に追加します。 3. 必要に応じて enlargePQExpBuffer が内部で呼び出され、バッファの拡張が行われます。

呼び出し元と呼び出し先

  • **呼び出し先**: appendBinaryPQExpBuffer
  • **呼び出し元**:
  • src/bin/initdb/initdb.c (複数箇所)
  • src/bin/pg_amcheck/pg_amcheck.c (複数箇所)

関連データ構造

  • PQExpBuffer: 動的に拡張可能な文字列バッファを保持する構造体。

失敗モードと境界条件

  • メモリ不足が発生した場合、バッファは「broken」状態に遷移し、以降の追加操作は無視されます。
  • data が NULL の場合の動作は定義されていません（セグメンテーション違反の可能性があります）。

保守担当者向けメモ

  • 本関数は appendBinaryPQExpBuffer のラッパーであり、文字列の長さを自動計算する利便性を提供します。
  • 性能が極めて重要な箇所で、既に文字列長が判明している場合は、直接 appendBinaryPQExpBuffer を使用することを検討してください。
