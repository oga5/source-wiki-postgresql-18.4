tzEntry

定義

  • 種別: struct
  • 定義箇所: src/include/utils/tzparser.h:23

目的

タイムゾーン設定ファイル（省略形定義）の各行を解析・保持し、PostgreSQL の日付・時刻処理サブシステムへ構造化されたデータを提供するために使用されます。

シグネチャまたは形状

typedef struct tzEntry
{
    char       *abbrev;    /* TZ abbreviation (downcased) */
    char       *zone;      /* zone name if dynamic abbrev, else NULL */
    int         offset;    /* offset in seconds from UTC */
    bool        is_dst;    /* true if a DST abbreviation */
    int         lineno;    /* source line number */
    const char *filename;  /* source filename */
} tzEntry;

入力と出力

  • **入力**: 設定ファイルから読み込まれた各フィールド（省略形、オフセット、DSTフラグ等）。
  • **出力**: load_tzoffsets を通じて構築される TimeZoneAbbrevTable の構成要素として、日付計算処理へ渡されます。

前提条件と不変条件

  • abbrev フィールドは、比較の整合性を保つために内部で小文字に変換されている必要があります。
  • 動的省略形（zone が NULL でない場合）では、offset および is_dst フィールドは使用されません。

アルゴリズムまたは振る舞い

1. tzparser.c 内のパーサーが設定ファイルを走査し、各行を tzEntry 構造体にパースします。 2. validateTzEntry 関数により、オフセット範囲や省略形の長さが妥当か検証されます。 3. 解析された tzEntry の配列は、最終的に TimeZoneAbbrevTable にまとめられ、時刻変換ロジックで利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/tzparser.c 内の解析関数（validateTzEntry, splitTzLine, addToArray 等）が本構造体を操作します。
  • **呼び出し先**: src/backend/utils/adt/datetime.c の ConvertTimeZoneAbbrevs 関数が、本構造体の配列を参照してタイムゾーン変換を行います。

関連データ構造

  • TimeZoneAbbrevTable: tzEntry の配列を保持する上位のデータ構造。

失敗モードと境界条件

  • 設定ファイル内の定義が不正な場合、validateTzEntry による検証でエラーが発生します。
  • エラー報告時には、構造体内の filename と lineno を使用してユーザーに通知されます。

保守担当者向けメモ

  • 本構造体は GUC（timezone_abbreviations）のチェックフックから利用されることを想定しています。
  • エラー発生時は elog(ERROR) でプロセスを終了させるのではなく、GUC_check_errmsg() を使用してユーザーに情報を伝達する設計に従ってください。
