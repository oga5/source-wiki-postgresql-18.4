hashendscan

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash.c:433

目的

ハッシュインデックススキャンの終了処理を行い、スキャンに関連するリソースを解放します。

シグネチャまたは形状

void hashendscan(IndexScanDesc scan)

入力と出力

  • **入力**: IndexScanDesc scan（終了対象のスキャン記述子）
  • **出力**: なし（副作用としてメモリ解放とバッファの解放を行う）

前提条件と不変条件

  • scan は有効な IndexScanDesc である必要があります。
  • scan->opaque は HashScanOpaque 構造体を指している必要があります。

アルゴリズムまたは振る舞い

1. scan->opaque から HashScanOpaque 構造体を取得します。 2. 現在のスキャン位置（currPos）が有効な場合、_hash_kill_items を呼び出し、削除対象としてマークされたアイテム（killed items）の処理を完了させます。 3. _hash_dropscanbuf を呼び出し、スキャン中に保持していたバッファを解放します。 4. so->killedItems が存在する場合は pfree で解放します。 5. so（HashScanOpaque）自体を pfree で解放します。 6. scan->opaque を NULL に設定し、スキャン記述子をクリーンな状態にします。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のインデックスアクセスメソッドインターフェース（amroutine->amendscan に登録）。
  • **呼び出し先**:
  • _hash_kill_items
  • _hash_dropscanbuf
  • pfree

関連データ構造

  • IndexScanDesc: スキャン全体の状態を保持する構造体。
  • HashScanOpaque: ハッシュインデックス固有のスキャン状態（バッファ、削除対象アイテムリスト等）を保持する構造体。

失敗モードと境界条件

  • scan->opaque が既に NULL の場合、あるいは不正なポインタである場合の動作は未定義です。
  • _hash_kill_items や _hash_dropscanbuf 内部でのエラー発生時は、呼び出し元に例外が伝播します。

保守担当者向けメモ

  • 本関数は、インデックススキャン終了時のリソースリークを防ぐために重要です。特に _hash_kill_items による遅延削除処理が正しく完了することを確認してください。
  • HashScanOpaque のメモリ管理は本関数に依存しているため、構造体のメンバ追加時には pfree の順序や対象に注意してください。
