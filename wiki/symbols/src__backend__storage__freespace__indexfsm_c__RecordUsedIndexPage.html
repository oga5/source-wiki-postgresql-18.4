RecordUsedIndexPage

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/indexfsm.c:61

目的

インデックスリレーション内の指定されたページを、FSM（Free Space Map）上で「使用中」として記録する。

シグネチャまたは形状

void RecordUsedIndexPage(Relation rel, BlockNumber usedBlock)

入力と出力

  • **入力**:
  • rel: 対象となるインデックスリレーションの Relation 構造体。
  • usedBlock: 使用中としてマークする対象の BlockNumber。
  • **出力**: なし（void）。

前提条件と不変条件

  • 内部的に 0 を「使用中」の状態として扱う。
  • 汎用的な RecordPageWithFreeSpace 関数を介してFSMを更新する。

アルゴリズムまたは振る舞い

1. 指定された rel と usedBlock を引数として受け取る。 2. RecordPageWithFreeSpace(rel, usedBlock, 0) を呼び出す。 3. FSM上で該当ページの使用可能容量を 0 に設定することで、当該ページが使用中であることを記録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/freespace/indexfsm.c 内の GetFreeIndexPage 関数（空きページ検索後に使用中としてマークするために呼び出される）。
  • **呼び出し先**:
  • RecordPageWithFreeSpace（storage/freespace.h で定義された汎用FSM操作関数）。

関連データ構造

  • Relation: インデックスリレーションを表現する構造体。
  • BlockNumber: ページ番号を識別する型。

失敗モードと境界条件

  • 供給されたソース情報には、特定の失敗モードや境界条件に関する明示的な記述はありません。

保守担当者向けメモ

  • 本関数は、ヒープ用のFSM機能をインデックス管理に適合させるためのラッパーです。
  • 内部的な値の解釈（0 = 使用中）は、src/backend/storage/freespace/indexfsm.c の設計慣習に基づいています。
