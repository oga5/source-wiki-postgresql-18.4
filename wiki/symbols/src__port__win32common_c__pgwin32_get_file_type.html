pgwin32_get_file_type

定義

  • 種別: function
  • 定義箇所: src/port/win32common.c:30

目的

Windows APIの GetFileType をラップし、PostgreSQLの移植レイヤーで共通して利用可能なファイルタイプ取得機能を提供します。特に、無効なハンドルに対するガード処理と、WindowsエラーコードからPOSIX準拠の errno への変換を統合的に行います。

シグネチャまたは形状

DWORD pgwin32_get_file_type(HANDLE hFile)

入力と出力

  • **入力**: hFile (Windowsの HANDLE)
  • **出力**: 成功時はファイルタイプ（FILE_TYPE_DISK, FILE_TYPE_CHAR 等）、失敗時は FILE_TYPE_UNKNOWN を返します。

前提条件と不変条件

  • errno は関数呼び出しの冒頭で 0 にリセットされます。
  • INVALID_HANDLE_VALUE またはストリーム未割り当てを示す -2 が渡された場合、errno を EINVAL に設定し、即座に FILE_TYPE_UNKNOWN を返します。

アルゴリズムまたは振る舞い

1. 入力された hFile が有効なハンドルか検証します。 2. GetFileType(hFile) を呼び出し、結果を保持します。 3. GetLastError() を呼び出し、直前のAPI実行状態を確認します。 4. GetFileType が FILE_TYPE_UNKNOWN を返し、かつ GetLastError() が NO_ERROR 以外である場合、_dosmaperr() を呼び出してWindowsエラーコードをPOSIXの errno に変換します。 5. 最終的なファイルタイプを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/port/win32fseek.c
  • src/port/win32stat.c
  • **呼び出し先**:
  • GetFileType (Windows API)
  • GetLastError (Windows API)
  • _dosmaperr (MSVCランタイム関数)

関連データ構造

  • HANDLE (Windows API型)
  • DWORD (Windows API型)

失敗モードと境界条件

  • **無効なハンドル**: INVALID_HANDLE_VALUE または -2 が渡された場合、errno が EINVAL に設定され、FILE_TYPE_UNKNOWN が返されます。
  • **APIエラー**: GetFileType が失敗し、かつシステムエラーが発生している場合、_dosmaperr により errno が更新され、FILE_TYPE_UNKNOWN が返されます。

保守担当者向けメモ

  • 本関数は win32fseek や win32stat など、Windows環境でのファイル操作を行う移植コードの基盤となります。
  • _dosmaperr によるエラー変換は、Windows特有のシステムエラーをPostgreSQLの標準的なエラーハンドリングに統合するために不可欠です。
