report_manifest_error

定義

  • 種別: function
  • 定義箇所: src/bin/pg_verifybackup/pg_verifybackup.c:508

目的

backup_manifest ファイルの解析中に発生したエラーをログに記録し、プログラムを異常終了させるためのコールバック関数です。

シグネチャまたは形状

static void report_manifest_error(JsonManifestParseContext *context, const char *fmt, ...);

入力と出力

  • **入力**:
  • context: 解析コンテキスト（JsonManifestParseContext 構造体へのポインタ）。
  • fmt: エラーメッセージの書式文字列。
  • ...: 書式文字列に対応する可変長引数。
  • **出力**: なし（関数内で exit(1) を呼び出すため、呼び出し元へは復帰しません）。

前提条件と不変条件

  • マニフェスト解析処理において、致命的なエラーが発生した際に呼び出されることを前提としています。
  • 本関数が呼び出された場合、プログラムは即座に終了します。

アルゴリズムまたは振る舞い

1. va_start を使用して可変長引数リストを初期化します。 2. pg_log_generic_v を呼び出し、PG_LOG_ERROR レベルでエラーメッセージを標準エラー出力等へ出力します。 3. va_end で引数リストを終了します。 4. exit(1) を実行し、プロセスを終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_verifybackup/pg_verifybackup.c: マニフェスト解析コンテキストの error_cb として設定されています。
  • src/bin/pg_combinebackup/load_manifest.c: 同様に解析コンテキストの error_cb として設定されています。
  • **呼び出し先**:
  • pg_log_generic_v: ログ出力のために呼び出されます。
  • exit: プロセス終了のために呼び出されます。

関連データ構造

  • JsonManifestParseContext: マニフェスト解析の状態を保持する構造体。本関数はこの構造体のエラーハンドラとして登録されます。

失敗モードと境界条件

  • 本関数自体がエラーを発生させることはありませんが、呼び出し元でマニフェストの構文解析が継続不可能な状態になった場合に実行されます。

保守担当者向けメモ

  • 本関数は pg_noreturn 属性を持つことが期待される設計ですが、ソースコード上では static void として定義されています。
  • マニフェスト解析エラーは常に致命的（fatal）として扱われるため、この関数が呼ばれると検証プロセスは即座に停止します。
