pg_lfind8_le

定義

  • 種別: function
  • 定義箇所: src/include/port/pg_lfind.h:57

目的

uint8 型の配列から、指定された値 key 以下（<=）の要素を高速に検索する。

シグネチャまたは形状

static inline bool pg_lfind8_le(uint8 key, uint8 *base, uint32 nelem)

入力と出力

  • **入力**:
  • key: 検索対象の基準値（uint8）。
  • base: 検索対象配列の先頭ポインタ（uint8 *）。
  • nelem: 配列の要素数（uint32）。
  • **出力**:
  • bool: 配列内に key 以下の要素が存在する場合は true、存在しない場合は false を返す。

前提条件と不変条件

  • base は少なくとも nelem 個の要素を持つ有効なメモリ領域を指している必要がある。
  • Vector8 型のサイズが 2 のべき乗であることを前提としている。

アルゴリズムまたは振る舞い

1. **SIMD 処理**: nelem を Vector8 のサイズ単位で切り捨てた範囲まで、SIMD 命令を用いて並列比較を行う。

  • vector8_load でデータをロードし、vector8_has_le で key 以下の要素が含まれるかを判定する。
  • 該当要素が見つかった時点で true を返して終了する。

    • 2. **フォールバック処理**: SIMD 処理で対象が見つからなかった場合、残りの要素（nelem の端数分）をループで一つずつ比較する。 3. **終了**: 全ての要素を走査しても見つからない場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/jsonapi.c:2221
  • src/test/modules/test_lfind/test_lfind.c:84, 86, 88, 94, 96
  • **呼び出し先**:
  • vector8_load（port/simd.h 経由）
  • vector8_has_le（port/simd.h 経由）

関連データ構造

  • Vector8: SIMD 演算用のベクトル型。

失敗モードと境界条件

  • nelem が 0 の場合、ループは実行されず即座に false を返す。
  • nelem が Vector8 のサイズ未満の場合、SIMD ループはスキップされ、フォールバックのループのみが実行される。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結している。
  • SIMD 命令の利用可否は port/simd.h の抽象化レイヤーに依存する。
  • vector8_has_le の実装がプラットフォームごとの SIMD 命令セットに最適化されているため、移植時は simd.h の定義を確認すること。
