InvalidateTSCacheCallBack

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/ts_cache.c:93

目的

システムカタログの更新に伴い、全文検索（Tsearch）関連のキャッシュエントリを無効化し、整合性を維持するために使用されます。

シグネチャまたは形状

static void InvalidateTSCacheCallBack(Datum arg, int cacheid, uint32 hashvalue)

入力と出力

  • **入力**:
  • arg: キャッシュ対象のハッシュテーブル（HTAB *）へのポインタ。
  • cacheid: 無効化のトリガーとなったシステムカタログのID。
  • hashvalue: 無効化対象のハッシュ値（本関数内では未使用）。
  • **出力**: なし（副作用としてキャッシュ状態を更新）。

前提条件と不変条件

  • arg は有効な HTAB 構造体へのポインタである必要があります。
  • キャッシュエントリは TSAnyCacheEntry 構造体としてキャスト可能である必要があります。

アルゴリズムまたは振る舞い

1. hash_seq_init および hash_seq_search を使用して、指定されたハッシュテーブル内の全エントリを走査します。 2. 各エントリの isvalid フラグを false に設定します。これにより、次回のルックアップ時に再構築が強制されます。 3. arg が TSConfigCacheHash と一致する場合、現在のデフォルト全文検索設定を保持する TSCurrentConfigCache を InvalidOid にリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**: CacheRegisterSyscacheCallback を介して、以下のシステムカタログの更新時に呼び出されます。
  • TSPARSEROID
  • TSDICTOID
  • TSTEMPLATEOID
  • TSCONFIGOID
  • TSCONFIGMAP
  • **呼び出し先**:
  • hash_seq_init
  • hash_seq_search

関連データ構造

  • HTAB: キャッシュを管理するハッシュテーブル。
  • TSAnyCacheEntry: キャッシュエントリの共通構造体。
  • TSConfigCacheHash: 設定キャッシュ用のハッシュテーブル（比較対象）。

失敗モードと境界条件

  • arg が無効なポインタである場合、セグメンテーション違反が発生する可能性があります。
  • キャッシュが空の場合、ループは即座に終了し、副作用は発生しません。

保守担当者向けメモ

  • 本関数は CacheRegisterSyscacheCallback のコールバックとして登録されるため、シグネチャを変更する際は utils/inval.h の定義と整合させる必要があります。
  • TSCurrentConfigCache のリセットは、設定キャッシュが更新された場合のみ実行されるよう条件分岐されています。
