array_get_isnull

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arrayfuncs.c:4786

目的

PostgreSQLの配列内部表現における、特定の要素がNULLであるかどうかを判定します。

シグネチャまたは形状

static bool array_get_isnull(const bits8 *nullbitmap, int offset)

入力と出力

  • **入力**:
  • nullbitmap: 配列のNULLビットマップへのポインタ（NULLの場合あり）。
  • offset: 判定対象となる要素のインデックス（0始まり）。
  • **出力**:
  • bool: 要素がNULLであれば true、そうでなければ false を返します。

前提条件と不変条件

  • nullbitmap が NULL の場合、配列内にNULL要素は存在しないと見なし、常に false を返します。
  • offset は、配列内の有効な要素インデックス範囲内である必要があります。

アルゴリズムまたは振る舞い

1. nullbitmap が NULL かどうかを確認し、NULL ならば false を返します。 2. offset を用いてビットマップ内の該当バイト（offset / 8）とビット位置（offset % 8）を特定します。 3. ビットマップの該当ビットが 1 であれば、その要素は「非NULL」であるため false を返します。 4. ビットマップの該当ビットが 0 であれば、その要素は「NULL」であるため true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/arrayfuncs.c 内の複数の箇所から呼び出されています（例: 行1902, 2413, 4693, 4725, 5082）。
  • **呼び出し先**: 本関数は他の関数を呼び出しません。

関連データ構造

  • ArrayType: 配列の内部表現構造体。NULLビットマップはこの構造体の一部として保持されます。

失敗モードと境界条件

  • nullbitmap が NULL の場合、関数は安全に false を返します。
  • offset が負の値や配列の要素数を超えた値である場合の挙動は、呼び出し側の責任に依存します（本関数内での境界チェックはありません）。

保守担当者向けメモ

  • 本関数は ArrayType のメモリレイアウト（ヘッダー、NULLビットマップ、データ本体）に強く依存しています。ビットマップのビット操作ロジックを変更する際は、ArrayType の定義と整合性を確認してください。
