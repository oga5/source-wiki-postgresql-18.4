show_log_timezone

定義

  • 種別: function
  • 定義箇所: src/backend/commands/variable.c:464

目的

GUC（Grand Unified Configuration）パラメータ log_timezone の現在の設定値を、表示用に正規化されたタイムゾーン名として取得します。

シグネチャまたは形状

const char *show_log_timezone(void)

入力と出力

  • **入力**: なし（グローバル変数 log_timezone を参照）。
  • **出力**: タイムゾーンの正規名を表す文字列（const char *）。解決できない場合は "unknown" を返します。

前提条件と不変条件

  • log_timezone が有効なタイムゾーン設定を保持している必要があります。
  • pg_get_timezone_name がタイムゾーンの解決に使用されます。

アルゴリズムまたは振る舞い

1. pg_get_timezone_name を呼び出し、現在の log_timezone に対応する正規のタイムゾーン名を取得します。 2. 取得結果が NULL でない場合はその文字列を返します。 3. 取得結果が NULL の場合は、フォールバックとして "unknown" という文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: GUCサブシステム（src/backend/utils/misc/guc_tables.c）により、log_timezone パラメータの show_hook として登録・利用されています。
  • **呼び出し先**: pg_get_timezone_name（タイムゾーン名の解決）。

関連データ構造

  • log_timezone: 内部的に保持されているタイムゾーン設定変数。

失敗モードと境界条件

  • タイムゾーン名が解決できない場合、関数はエラーを発生させず "unknown" を返します。

保守担当者向けメモ

  • 本関数は log_timezone の表示専用フックです。設定値の変更や検証を行う場合は、対応する check_log_timezone や assign_log_timezone を参照してください。
