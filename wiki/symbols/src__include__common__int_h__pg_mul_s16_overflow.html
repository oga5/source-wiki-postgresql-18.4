pg_mul_s16_overflow

定義

  • 種別: function
  • 定義箇所: src/include/common/int.h:102

目的

2つの int16 型整数の乗算を行い、結果が int16 の範囲を超える（オーバーフローする）かどうかを判定する。

シグネチャまたは形状

static inline bool pg_mul_s16_overflow(int16 a, int16 b, int16 *result)

入力と出力

  • **入力**:
  • a: 乗数 (int16)
  • b: 被乗数 (int16)
  • result: 計算結果を格納するポインタ (int16 *)
  • **出力**:
  • 戻り値: オーバーフローが発生した場合は true、それ以外は false。
  • *result: 演算成功時は乗算結果。オーバーフロー時はダミー値 0x5EED が書き込まれる。

前提条件と不変条件

  • コンパイラが HAVE__BUILTIN_OP_OVERFLOW をサポートしている場合、組み込み関数が優先的に使用される。
  • サポートされていない環境では、int32 型へのキャストによる拡張を用いて計算が行われる。

アルゴリズムまたは振る舞い

1. HAVE__BUILTIN_OP_OVERFLOW が定義されている場合、__builtin_mul_overflow を呼び出して結果を返す。 2. 定義されていない場合:

  • a と b を int32 にキャストして乗算し、一時変数 res に格納する。
  • res が PG_INT16_MAX を超えるか、PG_INT16_MIN 未満であるかを判定する。
  • オーバーフロー時は *result に 0x5EED を代入し true を返す。
  • 正常時は *result に res をキャストして代入し false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/int.c:969 にて、arg1 と arg2 の乗算結果の検証に使用されている。
  • **呼び出し先**: コンパイラ組み込み関数 __builtin_mul_overflow（環境依存）。

関連データ構造

  • int16, int32: PostgreSQL で定義された固定幅整数型。

失敗モードと境界条件

  • int16 の範囲（-32,768 から 32,767）を超える乗算結果はすべてオーバーフローとして扱われる。
  • オーバーフロー発生時の *result の値は、警告回避のための固定値 0x5EED であり、計算結果としては無効である。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、ヘッダーファイル内で完結している。
  • 移植性を担保するため、コンパイラ組み込み関数が利用できない環境でのフォールバック処理が実装されている。
