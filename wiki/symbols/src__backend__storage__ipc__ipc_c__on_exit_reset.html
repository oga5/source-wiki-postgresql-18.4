on_exit_reset

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/ipc.c:422

目的

プロセス終了時に実行されるコールバックの登録状態をリセットします。主にフォーク直後のバックエンドプロセスが、親プロセスから引き継いだ終了処理用インデックスをクリアするために使用されます。

シグネチャまたは形状

void on_exit_reset(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルな終了処理用インデックス変数を初期化）

前提条件と不変条件

  • 本関数は、プロセスが新しい実行コンテキストを開始する際（例: フォーク直後）に呼び出されることを想定しています。
  • 呼び出し後、before_shmem_exit_index、on_shmem_exit_index、on_proc_exit_index はすべて 0 に設定されます。

アルゴリズムまたは振る舞い

1. before_shmem_exit_index を 0 にリセットします。 2. on_shmem_exit_index を 0 にリセットします。 3. on_proc_exit_index を 0 にリセットします。 4. reset_on_dsm_detach() を呼び出し、動的共有メモリのデタッチ用コールバック状態をリセットします。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/init/miscinit.c:122
  • 呼び出し先:
  • reset_on_dsm_detach()（内部関数）

関連データ構造

  • before_shmem_exit_index
  • on_shmem_exit_index
  • on_proc_exit_index

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。本関数は状態を強制的に初期化する操作であり、エラーを返しません。

保守担当者向けメモ

  • 本関数は「IPC」モジュールの一部ですが、プロセス間通信そのものではなく、プロセス終了時のクリーンアップ管理における「状態の初期化」を担っています。
  • 親プロセスから引き継いだ終了コールバックリストを無効化する重要な役割を持つため、新しいプロセス生成フローを変更する際は本関数の呼び出しが適切に行われているか確認してください。
