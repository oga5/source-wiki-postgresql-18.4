sqlca_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/compat_informix-test_informix2.c:36

目的

ECPG（Embedded SQL for C）において、SQL 文の実行結果、エラーコード、警告状態、および診断情報を保持するための SQL 通信領域（SQLCA: SQL Communication Area）を定義する構造体です。

シグネチャまたは形状

struct sqlca_t
{
    char        sqlcaid[8];
    long        sqlabc;
    long        sqlcode;
    struct {
        int     sqlerrml;
        char    sqlerrmc[SQLERRMC_LEN];
    }           sqlerrm;
    char        sqlerrp[8];
    long        sqlerrd[6];
    char        sqlwarn[8];
    char        sqlstate[5];
};

入力と出力

  • **入力**: SQL 文の実行結果に基づき、ECPG ライブラリが本構造体の各メンバを更新します。
  • **出力**: アプリケーションは sqlcode や sqlstate を参照してエラーハンドリングを行い、sqlerrd[2] を通じて処理された行数を確認します。

前提条件と不変条件

  • sqlerrd[2] は、INSERT、UPDATE、DELETE 文の実行後に処理された行数を格納します。
  • sqlwarn[0] は、他の警告フラグ（sqlwarn[1] 等）が設定された場合に 'W' に設定されます。
  • sqlwarn[1] は、ホスト変数への格納時に文字列の切り捨てが発生した場合に 'W' に設定されます。

アルゴリズムまたは振る舞い

ECPG ライブラリは SQL 文の実行ごとにこの構造体を更新します。アプリケーションは各 SQL 操作の直後にこの構造体の内容を検査し、異常系であればロールバック等の処理へ分岐します。

呼び出し元と呼び出し先

  • **呼び出し元（参照・更新）**: ECPGget_sqlca() を介してライブラリ内部（src/interfaces/ecpg/ecpglib/connect.c, descriptor.c, data.c 等）から頻繁に参照・更新されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • sqlda_t: 動的 SQL 文の記述子構造体。
  • sqlvar_t: sqlda_t 内で使用される変数定義構造体。

失敗モードと境界条件

  • sqlcode が 0 以外の場合、エラーまたは警告が発生しています。
  • sqlerrd の各要素は、特定の SQL 文が適用されない場合には空（または未定義）となります。
  • sqlerrmc は SQLERRMC_LEN で定義された長さを超えるエラーメッセージを保持できません。

保守担当者向けメモ

  • 本構造体は Informix 互換モードを含む ECPG の標準的なインターフェースです。
  • src/interfaces/ecpg/compatlib/informix.c にて初期化およびコピー処理が行われているため、変更時は互換性への影響を確認してください。
