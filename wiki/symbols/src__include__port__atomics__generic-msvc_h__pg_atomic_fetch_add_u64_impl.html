pg_atomic_fetch_add_u64_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/generic-msvc.h:109

目的

64ビット符号なし整数に対して、アトミックな加算操作を行い、加算前の値を返す。

シグネチャまたは形状

static inline uint64 pg_atomic_fetch_add_u64_impl(volatile pg_atomic_uint64 *ptr, int64 add_)

入力と出力

  • **入力**:
  • ptr: 操作対象となる pg_atomic_uint64 構造体への揮発性ポインタ。
  • add_: 加算する値（int64）。
  • **出力**:
  • 加算操作が実行される直前の、ptr が指す値（uint64）。

前提条件と不変条件

  • 本関数は MSVC 環境下でのみ使用される。
  • ptr は適切にアライメントされた pg_atomic_uint64 を指している必要がある。
  • 内部で _InterlockedExchangeAdd64 を使用するため、64ビットビルド環境での実行が前提となる。

アルゴリズムまたは振る舞い

1. 引数として受け取った ptr->value のアドレスを、MSVC の組み込み関数 _InterlockedExchangeAdd64 に渡す。 2. _InterlockedExchangeAdd64 により、メモリ上でアトミックに加算を行い、加算前の値を即座に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/atomics.h 等の抽象化レイヤーから呼び出される。
  • **呼び出し先**: MSVC 組み込み関数 _InterlockedExchangeAdd64（<intrin.h> で定義）。

関連データ構造

  • pg_atomic_uint64: アトミック操作対象の値を保持する構造体。

失敗モードと境界条件

  • 64ビット未満の環境（32ビットビルド等）では、本関数は利用できない（コンパイルエラーまたは未定義の動作となる可能性がある）。
  • 整数オーバーフローが発生した場合の挙動は、標準的な符号なし整数のラップアラウンド規則に従う。

保守担当者向けメモ

  • 本関数は atomics.h を通じて利用されることを想定している。直接呼び出さず、PostgreSQL 標準の pg_atomic_fetch_add_u64 インターフェースを経由すること。
  • _InterlockedExchangeAdd64 はメモリバリアを内包しているため、追加のバリア操作は不要である。
