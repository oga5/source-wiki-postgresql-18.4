RelationInitIndexAccessInfo

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/relcache.c:1444

目的

リレーション（インデックス）のアクセスに必要なメタデータおよびサポート情報をシステムカタログから読み込み、Relation 構造体のインデックス関連フィールドを初期化する。

シグネチャまたは形状

void RelationInitIndexAccessInfo(Relation relation)

入力と出力

  • **入力**: relation (初期化対象の Relation 構造体ポインタ)
  • **出力**: なし（副作用として relation 構造体の各フィールドを更新）

前提条件と不変条件

  • relation->rd_rel->relam が InvalidOid でないこと。
  • relation が有効なインデックスリレーションであること。
  • CacheMemoryContext がメモリ管理の基盤として利用可能であること。

アルゴリズムまたは振る舞い

1. **カタログ取得**: pg_index から対象インデックスのタプルを SearchSysCache1 で取得し、rd_indextuple にコピーする。 2. **AMハンドラ設定**: pg_am からアクセスメソッドのハンドラ OID を取得し、rd_amhandler に格納する。 3. **メモリ確保**: インデックスアクセス情報専用のメモリコンテキスト (rd_indexcxt) を作成する。 4. **API初期化**: InitIndexAmRoutine(relation) を呼び出し、AM固有のルーチンをロードする。 5. **配列割り当て**: rd_opfamily, rd_opcintype, rd_indcollation, rd_indoption 等の配列を rd_indexcxt 上に確保する。 6. **データ抽出**: fastgetattr を使用して pg_index から照合順序（collation）、演算子クラス（opclass）、オプション情報を抽出し、各配列にコピーする。 7. **サポート情報初期化**: IndexSupportInitialize を呼び出し、サポートプロシージャの OID 配列を構築する。 8. **初期化完了**: インデックスの式、述語、排他制約キャッシュを NIL または NULL で初期化する。

失敗モードと境界条件

  • pg_index または pg_am のキャッシュルックアップに失敗した場合、elog(ERROR) を発生させる。
  • relnatts と indnatts の不整合がある場合、elog(ERROR) を発生させる。

保守担当者向けメモ

  • 本関数は rd_indexcxt を作成するため、リレーションの再構築時やクローズ時にメモリリークが発生しないよう注意が必要。
  • rd_indexprs や rd_indpred 等の複雑なキャッシュフィールドは本関数内では初期化されず、後続の処理で遅延評価される設計となっている。
