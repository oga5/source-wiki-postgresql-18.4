locate_windowfunc_context

定義

  • 種別: struct
  • 定義箇所: src/backend/rewrite/rewriteManip.c:39

目的

クエリツリーの走査中にウィンドウ関数（WindowFunc）ノードを探索し、その位置情報を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct
{
	int			win_location;
} locate_windowfunc_context;

入力と出力

  • win_location: 走査中に発見されたウィンドウ関数の位置（location）を格納する出力フィールドです。

前提条件と不変条件

  • 本構造体は、locate_windowfunc_walker 関数によるツリー走査の過程で、ウィンドウ関数の存在確認や位置特定のために使用されます。

アルゴリズムまたは振る舞い

  • locate_windowfunc_walker 関数がツリーを再帰的に探索する際、この構造体をコンテキストとして受け渡し、ウィンドウ関数ノードに遭遇した際にその位置情報を win_location に記録します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/rewrite/rewriteManip.c 内の関数（locate_windowfunc_walker を呼び出す箇所）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • WindowFunc: 探索対象となるノード型。
  • locate_windowfunc_walker: 本構造体を利用してツリーを走査する関数。

失敗モードと境界条件

  • ウィンドウ関数が見つからなかった場合、win_location の値は初期化時の状態（通常は -1 等）のままとなります。

保守担当者向けメモ

  • 本構造体は rewriteManip.c 内の特定の探索ロジックに特化しています。他の探索コンテキスト構造体と同様に、walker 関数とのペアで管理されています。
