LockManagerShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lock.c:442

目的

PostgreSQL の共有メモリ上に、重量級ロック管理に必要なハッシュテーブルおよび高速パス（Fast-Path）用データ構造を初期化します。

シグネチャまたは形状

void LockManagerShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（共有メモリ上のグローバル変数 LockMethodLockHash、LockMethodProcLockHash、FastPathStrongRelationLocks を初期化）

前提条件と不変条件

  • LockManagerShmemSize() で計算されたサイズと整合性が取れている必要があります。
  • 共有メモリの初期化フェーズで呼び出される必要があります。

アルゴリズムまたは振る舞い

1. **LOCK ハッシュテーブルの作成**: NLOCKENTS() に基づき、オブジェクト単位のロック情報を保持する LockMethodLockHash を ShmemInitHash で作成します。 2. **PROCLOCK ハッシュテーブルの作成**: ロックとプロセスの対応関係を保持する LockMethodProcLockHash を作成します。サイズは LOCK テーブルの 2 倍に設定されます。 3. **高速パス構造体の初期化**: FastPathStrongRelationLocks を共有メモリ上に確保し、初回作成時のみその内部のミューテックスを SpinLockInit で初期化します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:300（PostgreSQL の共有メモリ初期化プロセスの一部として呼び出されます）
  • 呼び出し先: ShmemInitHash, ShmemInitStruct, SpinLockInit

関連データ構造

  • LOCK: ロック対象オブジェクトごとの情報を保持。
  • PROCLOCK: 特定のプロセスとロック対象の関連を保持。
  • FastPathStrongRelationLockData: 高速パス最適化を無効化すべき強力なロックの管理用構造体。

失敗モードと境界条件

  • 共有メモリが不足している場合、ShmemInitHash または ShmemInitStruct がエラーを発生させ、PostgreSQL の起動が停止します。

保守担当者向けメモ

  • 本関数でのサイズ計算ロジックを変更した場合は、必ず LockManagerShmemSize() も併せて修正してください。
  • LOCK_DEBUG を有効にすると、ロック管理の挙動を詳細に追跡可能です。
