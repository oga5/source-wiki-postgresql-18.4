encoding_match

定義

  • 種別: struct
  • 定義箇所: src/port/chklocale.c:39

目的

OS 固有のエンコーディング名と、PostgreSQL 内部で使用されるエンコーディング識別子（pg_enc）を対応付けるためのデータ構造です。

シグネチャまたは形状

struct encoding_match
{
	enum pg_enc pg_enc_code;
	const char *system_enc_name;
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。encoding_match_list 配列の要素として使用され、システムから取得したエンコーディング文字列との照合に利用されます。

前提条件と不変条件

  • pg_enc_code は src/include/mb/pg_wchar.h で定義された pg_enc 列挙型に準拠する必要があります。
  • system_enc_name は、照合時に pg_strcasecmp を用いて大文字小文字を区別せずに比較されます。

アルゴリズムまたは振る舞い

本構造体は、encoding_match_list という静的テーブルを構成します。ロケールから取得した文字列をこのテーブル内の system_enc_name と照合し、一致した場合に対応する pg_enc_code を特定するルックアップ処理の基盤となります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/chklocale.c 内の encoding_match_list 定義（45行目）で参照されています。
  • **呼び出し先**: 情報なし。

関連データ構造

  • enum pg_enc: PostgreSQL 内部のエンコーディング識別子。
  • encoding_match_list: encoding_match 構造体の配列。

失敗モードと境界条件

  • 照合対象の文字列が encoding_match_list に存在しない場合、エンコーディングの特定は失敗し、呼び出し元のロジックで -1 が返されます。

保守担当者向けメモ

  • 新しいエンコーディングや別名を追加する場合は、encoding_match_list に本構造体のインスタンスを追加してください。
  • C または POSIX ロケールが指定された場合の挙動は、本構造体による照合とは別に処理される点に注意してください。
