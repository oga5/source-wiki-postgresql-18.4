GetForeignDataWrapperExtended

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/foreign/foreign.c:49

目的

システムカタログ pg_foreign_data_wrapper から指定された OID に対応する外部データラッパー（FDW）の定義情報を取得し、メモリ上の構造体 ForeignDataWrapper に格納して返します。

シグネチャまたは形状

ForeignDataWrapper *GetForeignDataWrapperExtended(Oid fdwid, bits16 flags)

入力と出力

  • **入力**:
  • fdwid (Oid): 取得対象の FDW の OID。
  • flags (bits16): 動作フラグ（FDW_MISSING_OK を指定可能）。
  • **出力**:
  • 成功時: ForeignDataWrapper 構造体へのポインタ。
  • 失敗時: flags に FDW_MISSING_OK が含まれていない場合、エラーにより中断。含まれている場合は NULL を返す。

前提条件と不変条件

  • SearchSysCache1 を使用するため、システムキャッシュが初期化されている必要があります。
  • 返される ForeignDataWrapper 構造体は palloc によってメモリ確保されるため、呼び出し側で適切に管理する必要があります。

アルゴリズムまたは振る舞い

1. FOREIGNDATAWRAPPEROID システムキャッシュを使用して、指定された fdwid のタプルを検索します。 2. タプルが見つからない場合、flags を確認し、FDW_MISSING_OK が設定されていなければ elog(ERROR) を発生させます。 3. タプルが存在する場合、GETSTRUCT で Form_pg_foreign_data_wrapper を取得します。 4. palloc で ForeignDataWrapper 構造体を確保し、OID、所有者、名前、ハンドラ、バリデーターの各フィールドをコピーします。 5. SysCacheGetAttr を用いて fdwoptions 属性を取得し、untransformRelOptions を介して List 形式に変換して構造体に格納します。 6. ReleaseSysCache でタプルを解放し、構造体ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c (複数箇所)
  • src/backend/foreign/foreign.c (ラッパー関数 GetForeignDataWrapper からの呼び出し)
  • **呼び出し先**:
  • SearchSysCache1
  • HeapTupleIsValid
  • GETSTRUCT
  • palloc
  • pstrdup
  • NameStr
  • SysCacheGetAttr
  • untransformRelOptions
  • ReleaseSysCache

関連データ構造

  • ForeignDataWrapper: FDW の定義情報を保持する構造体。
  • Form_pg_foreign_data_wrapper: システムカタログ pg_foreign_data_wrapper のタプル形式。

失敗モードと境界条件

  • 指定された OID が存在しない場合、FDW_MISSING_OK フラグがない限りエラーとなります。
  • fdwoptions が空（isnull）の場合、options フィールドには NIL が設定されます。

保守担当者向けメモ

  • 本関数は GetForeignDataWrapper の内部実装として利用されています。
  • fdwoptions の変換に untransformRelOptions を使用しているため、カタログ側のデータ形式変更に依存します。
