gets_fromFile

定義

  • 種別: function
  • 定義箇所: src/bin/psql/input.c:185

目的

指定されたファイルストリームから1行（改行文字まで）を読み込み、動的に確保された文字列として返却する。

シグネチャまたは形状

char *gets_fromFile(FILE *source)

入力と出力

  • **入力**: source (読み込み対象の FILE ポインタ)
  • **出力**: 読み込んだ行を含む char*（呼び出し元で解放が必要）。EOFまたはエラー時は NULL。

前提条件と不変条件

  • 静的バッファ PQExpBuffer を内部的に保持し、初回呼び出し時に初期化される。
  • sigint_interrupt_jmp が設定されている環境で呼び出される必要がある（sigint_interrupt_enabled フラグを一時的に有効化するため）。

アルゴリズムまたは振る舞い

1. 静的バッファが未初期化なら createPQExpBuffer で作成し、既存なら resetPQExpBuffer でクリアする。 2. ループ内で fgets を使用して入力を取得する。

  • fgets 実行中は sigint_interrupt_enabled を true に設定し、割り込みを許可する。

    • 3. 読み込んだ断片を appendPQExpBufferStr でバッファに追加する。 4. バッファ末尾に改行文字 \n が含まれた場合、それを終端文字 \0 に置換し、pg_strdup で複製した文字列を返却する。 5. EOFに達した場合、バッファ内に残存データがあればそれを返し、なければ NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/command.c:2667
  • src/bin/psql/input.c:105
  • src/bin/psql/mainloop.c:171
  • **呼び出し先**:
  • createPQExpBuffer, resetPQExpBuffer, appendPQExpBufferStr, PQExpBufferBroken (PQExpBuffer API)
  • fgets (標準ライブラリ)
  • pg_log_error, pg_strdup (PostgreSQL ユーティリティ)

関連データ構造

  • PQExpBuffer: 読み込み用の一時バッファとして使用される動的文字列構造体。

失敗モードと境界条件

  • **メモリ不足**: PQExpBufferBroken が真となった場合、エラーログを出力し NULL を返す。
  • **読み込みエラー**: ferror が真となった場合、エラーログを出力し NULL を返す。
  • **EOF**: 入力ストリームの終端で、バッファが空であれば NULL を返す。

保守担当者向けメモ

  • 本関数は readline を使用しない環境でのフォールバック、またはファイル入力用として使用される。
  • sigint_interrupt_enabled の操作は、SIGINT 発生時に longjmp を行うための重要な制御であるため、変更時は注意が必要。
