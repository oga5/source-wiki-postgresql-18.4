SUBTRANSShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/subtrans.c:213

目的

サブトランザクションログ（SUBTRANS）を管理するために共有メモリ上で必要な合計サイズを算出する。

シグネチャまたは形状

Size SUBTRANSShmemSize(void)

入力と出力

  • 入力: なし
  • 出力: Size 型の共有メモリサイズ（バイト単位）

前提条件と不変条件

  • SUBTRANSShmemBuffers() が返すバッファ数が有効であること。
  • 共有メモリの初期化前に呼び出される必要がある。

アルゴリズムまたは振る舞い

SimpleLruShmemSize を呼び出し、SUBTRANSShmemBuffers() で定義されたバッファ数に基づいて、SLRU 制御構造体とバッファ領域に必要な合計サイズを計算して返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:126（共有メモリの全体サイズ計算時）
  • 呼び出し先: SUBTRANSShmemBuffers()

関連データ構造

  • SimpleLruShmemSize: SLRU 共有メモリサイズ計算用の汎用関数。
  • SUBTRANSShmemBuffers(): SUBTRANS 用のバッファ数を定義する関数。

失敗モードと境界条件

  • 共有メモリサイズが計算不能なほど巨大な値になる場合、SimpleLruShmemSize の内部実装に依存する。

保守担当者向けメモ

  • 本関数は PostgreSQL の共有メモリ初期化プロセスの一部であり、SUBTRANS 関連のメモリ割り当てを変更する場合は SUBTRANSShmemBuffers の定義と併せて確認すること。
