PUTENVPROC

定義

  • 種別: typedef
  • 定義箇所: src/port/win32env.c:31

目的

Windows 環境において、異なる C ランタイムライブラリ（CRT）間で環境変数を同期させるために、各 CRT が提供する _putenv 関数を動的に呼び出すための関数ポインタ型を定義します。

シグネチャまたは形状

typedef int (_cdecl * PUTENVPROC) (const char *);

入力と出力

  • 入力: const char * 型の環境変数定義文字列（例: "VAR=VALUE"）。
  • 出力: int 型。成功した場合は 0、失敗した場合は非 0 を返します。

前提条件と不変条件

  • _cdecl 呼び出し規約に従う関数である必要があります。
  • 呼び出し元は、この型を使用して取得した関数ポインタが有効なメモリ領域を指していることを保証する必要があります。

アルゴリズムまたは振る舞い

本シンボル自体は型定義ですが、実行時には GetProcAddress を介して取得された各 CRT の _putenv 関数のアドレスがこの型にキャストされ、環境変数の更新処理に利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/win32env.c 内の環境変数更新ロジック（pgwin32_putenv 等）において、動的にロードされたモジュールから _putenv を取得する際に使用されます。
  • 呼び出し先: 実行時には、各 CRT モジュール内の _putenv 関数が呼び出されます。

関連データ構造

  • pgwin32_putenv: 本型を利用して環境変数の同期を行う関数。

失敗モードと境界条件

  • GetProcAddress が _putenv を見つけられなかった場合、この型へのキャストや呼び出しは未定義の動作を引き起こす可能性があるため、呼び出し側での NULL チェックが必須です。

保守担当者向けメモ

  • 本型は Windows 特有の CRT 混在問題を解決するためのものです。将来的に新しい CRT が導入された場合、modulenames 配列の更新とともに、この型が適合するか確認してください。
