qsort_tuple_signed_compare

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplesort.c:517

目的

符号付きデータ型をキーとするタプルの比較を行い、ソート順序を決定する。

シグネチャまたは形状

static pg_attribute_always_inline int
qsort_tuple_signed_compare(SortTuple *a, SortTuple *b, Tuplesortstate *state)

入力と出力

  • **入力**: 比較対象の SortTuple 型ポインタ a および b、およびソート状態を保持する Tuplesortstate 構造体。
  • **出力**: a が b より小さい場合は負の整数、等しい場合は 0、大きい場合は正の整数を返す。

前提条件と不変条件

  • state->base.sortKeys[0] に有効なソートキーが設定されていること。
  • pg_attribute_always_inline が指定されており、コンパイル時に呼び出し元へインライン展開されることが期待される。

アルゴリズムまたは振る舞い

1. ApplySignedSortComparator を呼び出し、第1ソートキーに基づいて a と b を比較する。 2. 比較結果が 0 でない（順序が確定した）場合、その値を即座に返す。 3. 比較結果が 0 の場合、state->base.onlyKey が存在すれば（他のキーによるソートが不要であれば）0 を返す。 4. それ以外の場合、state->base.comparetup_tiebreak を呼び出し、タイブレーク（同点決勝）処理を実行して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/tuplesort.c 内のソート処理（ST_COMPARE マクロ経由での利用が確認されている）。
  • **呼び出し先**: ApplySignedSortComparator および state->base.comparetup_tiebreak。

関連データ構造

  • SortTuple: ソート対象のタプルを保持する構造体。
  • Tuplesortstate: ソート操作の全体状態を管理する構造体。

失敗モードと境界条件

  • comparetup_tiebreak が設定されていない状態でタイブレークが必要になった場合、関数ポインタの呼び出しによりクラッシュする可能性がある（呼び出し元の初期化状態に依存）。

保守担当者向けメモ

  • 本関数はインライン展開を前提としているため、変更時は tuplesort.c 内のパフォーマンスへの影響を考慮すること。
  • ApplySignedSortComparator は符号付き比較専用であるため、符号なしデータ型を扱う場合は qsort_tuple_unsigned_compare を使用すること。
