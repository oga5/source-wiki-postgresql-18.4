pgstat_subscription_flush_cb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_subscription.c:102

目的

バックエンドローカルに蓄積されたサブスクリプション統計情報を、共有メモリ上の統計構造体へ同期（フラッシュ）するために使用されるコールバック関数です。

シグネチャまたは形状

bool pgstat_subscription_flush_cb(PgStat_EntryRef *entry_ref, bool nowait)

入力と出力

  • **入力**:
  • entry_ref: 統計エントリへの参照（ローカルおよび共有メモリ上の統計データを含む）。
  • nowait: ロック取得時に待機するかどうかを指定するフラグ。
  • **出力**:
  • bool: 成功した場合は true、ロック取得に失敗した場合は false を返します。

前提条件と不変条件

  • entry_ref->pending は PgStat_BackendSubEntry 型としてキャスト可能である必要があります。
  • entry_ref->shared_stats は PgStatShared_Subscription 型としてキャスト可能である必要があります。
  • localent（ローカル統計）には、共有メモリへ加算すべき非ゼロの統計内容が含まれていることが前提です。

アルゴリズムまたは振る舞い

1. entry_ref からローカル統計 (localent) と共有メモリ上の統計 (shsubent) を取得します。 2. pgstat_lock_entry を呼び出し、共有メモリ上の統計エントリをロックします。nowait が true でロック取得に失敗した場合は、即座に false を返します。 3. SUB_ACC マクロを使用して、以下の値を共有メモリへ加算します。

  • apply_error_count
  • sync_error_count
  • conflict_count 配列（全競合タイプ）

    • 4. pgstat_unlock_entry を呼び出してロックを解放します。 5. true を返して処理を完了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/pgstat.c:362: 統計システム内のフラッシュ処理用コールバックとして登録されています。
  • **呼び出し先**:
  • pgstat_lock_entry
  • pgstat_unlock_entry

関連データ構造

  • PgStat_EntryRef: 統計エントリの参照管理構造体。
  • PgStat_BackendSubEntry: バックエンドローカルなサブスクリプション統計。
  • PgStatShared_Subscription: 共有メモリ上のサブスクリプション統計。

失敗モードと境界条件

  • nowait が true の場合、共有メモリのロックが競合すると false を返し、統計のフラッシュは行われません。

保守担当者向けメモ

  • SUB_ACC マクロは本関数内でのみ定義・使用されています。PgStatShared_Subscription 構造体に新しい統計フィールドを追加した場合は、このマクロの適用範囲を更新する必要があります。
