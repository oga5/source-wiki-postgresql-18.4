uint16

定義

  • 種別: typedef（PostgreSQL 基本型）
  • 定義箇所: contrib/sepgsql/selinux.c:30（ソースコード上の使用箇所）

目的

16ビット符号なし整数を保持するためのデータ型です。contrib/sepgsql/selinux.c 内では、SELinux ポリシーのオブジェクトクラスコードを格納するために使用されます。

シグネチャまたは形状

typedef uint16; // 16-bit unsigned integer

入力と出力

  • 入力: 0 から 65535 までの整数値。
  • 出力: 16ビット幅のバイナリ表現。

前提条件と不変条件

  • PostgreSQL の基本型定義に依存しており、プラットフォーム間で 16 ビット幅であることが保証されています。
  • selinux_catalog 構造体内では、class_code として SELinux のオブジェクトクラス識別子を保持します。

アルゴリズムまたは振る舞い

本シンボル自体はデータ型であり、特定のアルゴリズムを持ちません。selinux_catalog 構造体の一部として、PostgreSQL 内部のオブジェクトクラスと SELinux 外部コードの対応付けを保持する役割を担います。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/sepgsql/selinux.c 内の selinux_catalog 定義。
  • 呼び出し先: なし（基本型のため）。
  • 補足: ツールによる検索結果から、contrib/amcheck や contrib/bloom など他のモジュールでも同名の型が広く利用されていますが、これらは本シンボルの直接的な呼び出し元ではありません。

関連データ構造

  • selinux_catalog: class_code フィールドとして本型を保持する構造体。
  • uint32: selinux_catalog 内で権限コード（av_code）を保持するために併用される型。

失敗モードと境界条件

  • 値が 65535 を超える場合、オーバーフローが発生します。
  • SELinux のクラスコードが 16 ビットの範囲を超える場合、本型では表現できません。

保守担当者向けメモ

  • 本シンボルは PostgreSQL の標準的な型定義です。contrib/sepgsql 固有の定義ではないため、変更の際は PostgreSQL 全体の型定義への影響を考慮してください。
  • selinux_catalog 内での使用において、将来的にクラスコードの範囲が拡大される可能性がある場合は、型定義の再検討が必要になる可能性があります。
