printfPQExpBuffer

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/pqexpbuffer.c:234

目的

PQExpBuffer バッファの内容を、指定された printf 形式のフォーマット文字列で上書きします。

シグネチャまたは形状

void printfPQExpBuffer(PQExpBuffer str, const char *fmt, ...)

入力と出力

  • **入力**:
  • str: 操作対象の PQExpBuffer 構造体へのポインタ。
  • fmt: printf 形式のフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**: なし（副作用として str の内容が更新されます）。

前提条件と不変条件

  • str は初期化済みである必要があります。
  • バッファが既に「broken」（メモリ不足等によるエラー状態）である場合、本関数は何も行わずに即座にリターンします。

アルゴリズムまたは振る舞い

1. resetPQExpBuffer(str) を呼び出し、バッファを空の状態にリセットします。 2. バッファが「broken」状態であれば処理を終了します。 3. do-while ループ内で appendPQExpBufferVA を呼び出し、フォーマットされた文字列をバッファに追加します。 4. appendPQExpBufferVA がバッファ不足を検知して false を返した場合、内部でバッファが拡張され、ループが継続されます。 5. 成功するまで（done が true になるまで）書き込みを試行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/initdb/initdb.c、src/bin/pg_dump/dumputils.c、src/bin/pg_dump/pg_dump.c 等の各所から呼び出されています。
  • **呼び出し先**: resetPQExpBuffer、PQExpBufferBroken（マクロ）、va_start/va_end、appendPQExpBufferVA。

関連データ構造

  • PQExpBuffer: 文字列バッファを管理する構造体。

失敗モードと境界条件

  • **メモリ不足**: appendPQExpBufferVA 内部でメモリ確保に失敗した場合、バッファは「broken」状態に遷移し、以降の操作は無効化されます。
  • **フォーマットエラー**: vsnprintf に依存しているため、不正なフォーマット文字列が渡された場合の挙動は標準ライブラリの仕様に従います。

保守担当者向けメモ

  • 本関数は errno を保存・復元することで、フォーマット処理中の副作用から errno を保護しています。
  • バッファの拡張が必要な場合、appendPQExpBufferVA が再試行を制御する設計となっています。
