vac_estimate_reltuples

定義

  • 種別: function
  • 定義箇所: src/backend/commands/vacuum.c:1345

目的

テーブルの全ページをスキャンしなかった場合に、pg_class.reltuples に格納する新しいタプル数の推定値を算出する。

シグネチャまたは形状

double vac_estimate_reltuples(Relation relation,
                              BlockNumber total_pages,
                              BlockNumber scanned_pages,
                              double scanned_tuples)

入力と出力

  • **入力**:
  • relation: 対象リレーションの記述子。
  • total_pages: テーブルの総ページ数。
  • scanned_pages: 今回のスキャンで走査したページ数。
  • scanned_tuples: 今回のスキャンで確認されたタプル数。
  • **出力**:
  • 推定されたタプル総数（double 型）。

前提条件と不変条件

  • relation->rd_rel が有効であり、既存の統計情報（relpages, reltuples）が参照可能であること。
  • 戻り値は floor(x + 0.5) により整数値に丸められる。

アルゴリズムまたは振る舞い

1. **全スキャン時**: scanned_pages >= total_pages の場合、scanned_tuples をそのまま返す。 2. **過小スキャン時の保護**:

  • テーブルサイズが不変で、かつスキャン範囲が全ページの2%未満である場合、既存の reltuples を維持する。
  • スキャン範囲が1ページ以下の場合も、既存の reltuples を維持する。

    • 3. **密度計算**:

  • 既存の統計情報が不明（reltuples < 0 または relpages == 0）な場合、スキャンしたタプル密度を全ページに外挿する。
  • 通常時は、既存のタプル密度（old_reltuples / old_relpages）を算出し、未スキャンページ分を補完して合計する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/vacuumlazy.c: vacrel->new_live_tuples の算出に使用。
  • contrib/pgstattuple/pgstatapprox.c: stat->tuple_count の算出に使用。
  • **呼び出し先**:
  • 算術演算および floor() 関数（math.h）を使用。

関連データ構造

  • Relation: relation->rd_rel を通じて Form_pg_class の統計情報にアクセスする。

失敗モードと境界条件

  • old_rel_pages が 0 の場合、ゼロ除算を回避するロジックが組み込まれている。
  • 既存の統計情報が不明な場合（負の値など）、スキャン結果のみに基づく単純な外挿を行う。

保守担当者向けメモ

  • 本関数は、VACUUM がテーブルの一部のみをスキャンする際に、統計情報の急激な変動を防ぐためのヒューリスティックを含んでいる。
  • vacuumlazy.c の実装詳細（最終ページを常にスキャンする傾向など）に起因する統計の歪みを補正する意図があるため、スキャンロジックを変更する際は本関数の保護条件も再検討すること。
