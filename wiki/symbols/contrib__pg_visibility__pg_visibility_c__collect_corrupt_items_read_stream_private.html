collect_corrupt_items_read_stream_private

定義

  • 種別: struct
  • 定義箇所: contrib/pg_visibility/pg_visibility.c:48

目的

ReadStream を使用してリレーションをスキャンする際、破損したアイテムを収集するためのプライベートなコンテキスト情報を保持する構造体です。

シグネチャまたは形状

struct collect_corrupt_items_read_stream_private
{
	bool		all_frozen;
	bool		all_visible;
	BlockNumber current_blocknum;
	BlockNumber last_exclusive;
	Relation	rel;
	Buffer		vmbuffer;
};

入力と出力

本構造体は関数への引数ではなく、スキャン処理中の状態保持に使用されます。

  • 入力: rel（対象リレーション）、vmbuffer（可視性マップバッファ）など。
  • 出力: all_frozen, all_visible（現在のブロックの可視性状態）、current_blocknum（現在処理中のブロック番号）。

前提条件と不変条件

  • rel は有効なリレーション記述子である必要があります。
  • vmbuffer は、スキャン対象の可視性マップページを指す有効なバッファである必要があります。

アルゴリズムまたは振る舞い

本構造体は、ReadStream を用いた全ページ走査において、各ブロックの可視性ビット状態（all_frozen, all_visible）を一時的に保持し、整合性チェックのロジックに提供します。スキャンが進むにつれ、current_blocknum が更新されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pg_visibility/pg_visibility.c 内の関数（例: collect_corrupt_items 相当の処理）において、ローカル変数として宣言および初期化されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • Relation: 対象となるリレーション情報。
  • Buffer: 可視性マップのバッファ管理に使用。
  • BlockNumber: ブロック位置の追跡に使用。

失敗モードと境界条件

  • vmbuffer が無効な場合、可視性ビットの読み取りに失敗する可能性があります。
  • rel がクローズされている状態でアクセスした場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本構造体は ReadStream のコールバック処理内で callback_private_data としてキャストして利用されます。
  • 構造体のフィールド追加や変更を行う際は、ReadStream を利用するスキャンロジック全体での整合性に注意してください。
