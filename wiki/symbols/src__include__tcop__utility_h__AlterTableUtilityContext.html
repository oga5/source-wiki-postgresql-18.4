AlterTableUtilityContext

定義

  • 種別: struct
  • 定義箇所: src/include/tcop/utility.h:30

目的

ALTER TABLE コマンドの実行中に、再帰的なサブコマンド処理や関連する実行コンテキスト情報を保持・伝播するために使用されます。

シグネチャまたは形状

typedef struct AlterTableUtilityContext
{
	PlannedStmt *pstmt;			/* 外側の ALTER TABLE コマンドの計画 */
	const char *queryString;	/* クエリ文字列 */
	Oid			relid;			/* 対象テーブルの OID */
	ParamListInfo params;		/* パラメータ情報 */
	QueryEnvironment *queryEnv; /* 実行環境 */
} AlterTableUtilityContext;

入力と出力

本構造体は、ALTER TABLE 処理の各段階で必要なメタデータおよび実行状態を入力として受け取り、サブコマンド実行関数へ引き継ぎます。出力としての戻り値はありません。

前提条件と不変条件

  • pstmt、queryString、relid、params、queryEnv は、ALTER TABLE の実行フローにおいて整合性が保たれている必要があります。
  • 構造体内の各フィールドは、実行中のコマンドのスコープ内でのみ有効です。

アルゴリズムまたは振る舞い

ALTER TABLE の実行プロセスにおいて、このコンテキスト構造体がサブコマンド処理関数に渡されることで、トップレベルのコマンド情報（クエリ文字列やパラメータ等）をサブコマンド側から参照可能にします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/tablecmds.c 内の各関数が、このコンテキストを引数として受け取ります。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • PlannedStmt: 実行計画ノード。
  • ParamListInfo: パラメータリスト。
  • QueryEnvironment: クエリ実行環境。

失敗モードと境界条件

  • 構造体内のポインタが NULL である場合、参照先の関数でセグメンテーション違反が発生する可能性があります。
  • relid が無効な OID である場合、後続のテーブル操作が失敗します。

保守担当者向けメモ

  • 本構造体は ALTER TABLE の再帰的な処理に深く関与しているため、フィールドの追加や変更を行う際は src/backend/commands/tablecmds.c 側のすべての参照箇所で整合性が取れていることを確認してください。
