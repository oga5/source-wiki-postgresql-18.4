UserAuth

定義

  • 種別: enum
  • 定義箇所: src/include/libpq/hba.h:25

目的

PostgreSQLのクライアント接続認証において、サポートされる認証方式を定義する列挙型です。

シグネチャまたは形状

typedef enum UserAuth
{
    uaReject,
    uaImplicitReject,
    uaTrust,
    uaIdent,
    uaPassword,
    uaMD5,
    uaSCRAM,
    uaGSS,
    uaSSPI,
    uaPAM,
    uaBSD,
    uaLDAP,
    uaCert,
    uaRADIUS,
    uaPeer,
    uaOAuth,
    /* USER_AUTH_LAST は uaOAuth */
} UserAuth;

入力と出力

  • 本型は、認証設定（HbaLine）や接続情報（Port）のフィールドとして保持されます。
  • hba_authname(UserAuth auth_method) 関数を通じて、列挙値に対応する文字列名を取得可能です。

前提条件と不変条件

  • USER_AUTH_LAST マクロにより、列挙型の最後の値が uaOAuth であることが保証されています。
  • 新しい認証方式を追加する場合、src/backend/libpq/hba.c 内の UserAuthName 配列を同期して更新し、両者の整合性を維持する必要があります。

アルゴリズムまたは振る舞い

  • クライアント接続時に hba_getauthmethod() が適用すべき認証方式を決定するために使用されます。
  • 決定された値に基づき、src/backend/libpq/auth.c が実際の認証処理を実行します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/libpq/hba.c (認証方式の名称変換および検証)
  • src/backend/utils/init/miscinit.c
  • src/include/libpq/hba.h (構造体定義内)
  • src/include/libpq/libpq-be.h (接続情報構造体内)
  • 呼び出し先:
  • 本シンボルは列挙型であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • HbaLine: pg_hba.conf の各行の設定を保持する構造体で、認証方式を UserAuth 型で保持します。
  • Port: クライアント接続情報を保持する構造体で、適用される認証方式を保持します。

失敗モードと境界条件

  • 列挙型定義と UserAuthName 配列の不整合は、認証方式の名称解決時にエラーや予期せぬ挙動を引き起こす可能性があります。
  • uaImplicitReject はユーザーから直接指定できない内部的な値として扱われます。

保守担当者向けメモ

  • 新しい認証方式を追加する際は、hba.c 内の UserAuthName[] 配列が UserAuth 列挙型と一致していることを確認してください。
  • hba_authname() 関数は、この列挙型を文字列に変換する際に使用されます。
