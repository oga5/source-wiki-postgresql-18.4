pg_GSS_error_int

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-gssapi-common.c:24

目的

GSSAPIのステータスコード（メジャーまたはマイナー）を人間が読める文字列に変換し、指定されたバッファに格納する内部ヘルパー関数です。

シグネチャまたは形状

static void pg_GSS_error_int(char *s, size_t len, OM_uint32 stat, int type)

入力と出力

  • s: エラーメッセージを書き込むための出力先バッファ。
  • len: バッファ s のサイズ。
  • stat: 変換対象のGSSAPIステータスコード。
  • type: ステータスの種類（GSS_C_GSS_CODE または GSS_C_MECH_CODE）。
  • 出力: 戻り値なし。バッファ s に文字列が書き込まれます。

前提条件と不変条件

  • バッファ s は、少なくとも len バイトの書き込み可能な領域を持つ必要があります。
  • gss_display_status が正常終了する限り、メッセージは連結されます。

アルゴリズムまたは振る舞い

1. gss_display_status を呼び出し、ステータスコードに対応するメッセージを取得します。 2. 複数のメッセージが存在する場合、スペースで区切りながらバッファ s に連結します。 3. 各メッセージ取得後に gss_release_buffer を呼び出し、GSSAPIが確保したメモリを解放します。 4. 最後にヌル終端文字を付与します。

呼び出し元と呼び出し先

  • 呼び出し元: 同一ファイル内の pg_GSS_error から、メジャーコードとマイナーコードの変換のために呼び出されます。
  • 呼び出し先: gss_display_status および gss_release_buffer（GSSAPIライブラリ関数）、elog（PostgreSQLログ出力関数）。
  • 注: src/interfaces/libpq/fe-gssapi-common.c に同名の関数が存在しますが、引数構成が異なるため、本関数とは別個の実装です。

関連データ構造

  • gss_buffer_desc: GSSAPIから返されるメッセージ文字列を保持する構造体。
  • OM_uint32: GSSAPIのステータスコード型。

失敗モードと境界条件

  • バッファサイズ不足: メッセージが len を超える場合、elog(COMMERROR, ...) を発行してエラー報告が不完全であることを通知し、バッファの末尾をヌル終端して切り詰めます。
  • GSSAPIエラー: gss_display_status が GSS_S_COMPLETE を返さない場合、ループを中断します。

保守担当者向けメモ

  • 本関数は COMMERROR レベルでログを出力します。これは、エラー報告処理自体が再帰的にGSSAPIエラーを引き起こすのを防ぐための設計です。
  • バッファサイズは呼び出し元で固定長（例: 128バイト）として確保されることが想定されています。
