Delete

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/fd.c:1269

目的

LRU（Least Recently Used）キャッシュ管理用の二重連結リストから、指定された仮想ファイル記述子（VFD）エントリを削除します。

シグネチャまたは形状

static void Delete(File file)

入力と出力

  • **入力**: file (File 型) - 削除対象のVFDインデックス。
  • **出力**: なし。

前提条件と不変条件

  • file != 0 であること（Assert により検証）。
  • VfdCache 配列が初期化されており、対象の file が有効なLRUリストの一部であること。

アルゴリズムまたは振る舞い

1. 指定された file に対応する Vfd 構造体を取得します。 2. 連結リストのポインタを操作し、対象ノードをリストから切り離します。

  • vfdP->lruLessRecently（直前のノード）の lruMoreRecently ポインタを、vfdP->lruMoreRecently（直後のノード）へ繋ぎ変えます。
  • vfdP->lruMoreRecently（直後のノード）の lruLessRecently ポインタを、vfdP->lruLessRecently（直前のノード）へ繋ぎ変えます。

    • 3. FDDEBUG マクロが有効な場合、操作前後にLRUリストの状態をログ出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/file/fd.c 内のファイルアクセス関数（FileAccess 等）から、LRUリングの順序を更新する際に呼び出されます。
  • **呼び出し先**: 直接的な関数呼び出しはありませんが、VfdCache グローバル配列および _dump_lru（デバッグ時）を参照します。
  • **注意**: ツールが提示した contrib/postgres_fdw 等の Delete は、SQLレベルの行削除操作を指すものであり、本関数とは無関係です。

関連データ構造

  • Vfd: src/backend/storage/file/fd.c で定義される仮想ファイル記述子構造体。LRUリストの前後ノードを指すインデックスを保持します。
  • VfdCache: VFDの配列。LRUリストのアンカーおよび各ノードの実体。

失敗モードと境界条件

  • file が 0 の場合、Assert により異常終了します。
  • リストの整合性が壊れている場合、ポインタの繋ぎ変えによりメモリ破壊や無限ループが発生する可能性があります。

保守担当者向けメモ

  • 本関数はリストのリンクを外すのみであり、物理的なファイルクローズや Vfd 構造体の解放は行いません。
  • 物理的なクローズを伴う削除が必要な場合は、LruDelete を使用してください。
