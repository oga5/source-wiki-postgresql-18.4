injection_wait

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_points.c:281

目的

PostgreSQLのコードパス上に挿入されたインジェクションポイントにおいて、外部からのシグナル（injection_points_wakeup）が送られるまでプロセスを待機させる。

シグネチャまたは形状

void injection_wait(const char *name, const void *private_data, void *arg)

入力と出力

  • **入力**:
  • name: インジェクションポイントの名称。
  • private_data: InjectionPointCondition 構造体へのポインタ。
  • arg: 未使用。
  • **出力**: なし（void）。

前提条件と不変条件

  • 共有メモリ状態 inj_state が初期化されていること（未初期化時は injection_init_shmem() が呼び出される）。
  • injection_point_allowed により実行が許可されていること。
  • 共有メモリ内に待機用の空きスロット（INJ_MAX_WAIT）が存在すること。

アルゴリズムまたは振る舞い

1. injection_point_allowed で条件判定を行い、不許可なら即時終了する。 2. WaitEventInjectionPointNew で待機イベントを生成する。 3. 共有メモリの inj_state->name 配列から空きスロットを確保し、現在の wait_counts を記録する。 4. 空きスロットがない場合は ERROR を発生させる。 5. ConditionVariablePrepareToSleep を呼び出し、ループ内で wait_counts が変化するまで ConditionVariableSleep を繰り返す。 6. 待機解除後、ConditionVariableCancelSleep を実行し、確保していたスロットを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/injection_points/injection_points.c 内でインジェクションポイントのコールバックとして登録される。
  • **呼び出し先**:
  • injection_init_shmem
  • injection_point_allowed
  • pgstat_report_inj
  • WaitEventInjectionPointNew
  • SpinLockAcquire / SpinLockRelease
  • ConditionVariablePrepareToSleep / ConditionVariableSleep / ConditionVariableCancelSleep
  • elog

関連データ構造

  • InjectionPointSharedState: 共有メモリ上の状態管理構造体。
  • InjectionPointCondition: インジェクション条件を保持する構造体。

失敗モードと境界条件

  • **スロット不足**: 共有メモリ上の待機スロットが満杯の場合、elog(ERROR, ...) によりエラーが発生する。

保守担当者向けメモ

  • 本関数はテスト専用であり、本番環境での使用は想定されていない。
  • 待機イベント名は WaitEventInjectionPointNew で生成されるが、テスト用途のため解放処理は行われない。
  • 共有メモリのロックにはスピンロックを使用しているため、待機中の処理は短時間であることが期待される。
