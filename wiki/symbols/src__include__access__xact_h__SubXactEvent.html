SubXactEvent

定義

  • 種別: enum
  • 定義箇所: src/include/access/xact.h:140

目的

サブトランザクションのライフサイクルにおける各フェーズ（開始、コミット、アボート、コミット直前）を識別し、登録されたコールバック関数をトリガーするために使用されます。

シグネチャまたは形状

typedef enum
{
	SUBXACT_EVENT_START_SUB,
	SUBXACT_EVENT_COMMIT_SUB,
	SUBXACT_EVENT_ABORT_SUB,
	SUBXACT_EVENT_PRE_COMMIT_SUB,
} SubXactEvent;

入力と出力

  • 入力: トランザクション制御ロジックにおいて、サブトランザクションの状態遷移時にイベント種別として渡されます。
  • 出力: なし（列挙型定数）。

前提条件と不変条件

  • SubXactCallback 型の関数ポインタを介して使用されます。
  • 各イベントは、サブトランザクションの特定の状態遷移（開始、コミット、アボート、コミット直前）と一対一で対応します。

アルゴリズムまたは振る舞い

サブトランザクションの状態が変化する際、CallSubXactCallbacks 関数を通じて登録済みのコールバック関数にこのイベントが通知されます。これにより、外部モジュール（postgres_fdw、sepgsql、plpgsql 等）はサブトランザクションの終了やアボートに合わせたリソースのクリーンアップや状態更新を実行できます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c 内の CallSubXactCallbacks 関数。
  • 呼び出し先（コールバック登録者）:
  • contrib/postgres_fdw/connection.c: pgfdw_subxact_callback
  • contrib/sepgsql/label.c: sepgsql_subxact_callback
  • src/pl/plpgsql/src/pl_exec.c: plpgsql_subxact_cb

関連データ構造

  • SubXactCallback: SubXactEvent を引数に取るコールバック関数の型定義。
  • SubTransactionId: コールバック時にイベント対象を特定するために併用される識別子。

失敗モードと境界条件

  • 該当するイベントが発生しない場合、コールバックは呼び出されません。
  • コールバック関数内でエラーが発生した場合、トランザクション全体の異常終了処理に影響を与える可能性があります。

保守担当者向けメモ

  • 新しいサブトランザクションイベントを追加する場合は、本列挙型を拡張し、CallSubXactCallbacks を呼び出す箇所で適切にハンドリングしてください。
  • コールバック関数はトランザクションの整合性に直結するため、実装時は副作用に十分注意してください。
