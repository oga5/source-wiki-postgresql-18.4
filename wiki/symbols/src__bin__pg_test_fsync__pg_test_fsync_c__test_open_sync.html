test_open_sync

定義

  • 種別: function
  • 定義箇所: src/bin/pg_test_fsync/pg_test_fsync.c:466

目的

O_SYNC フラグを使用したファイル書き込みのパフォーマンスを測定し、ベンチマーク結果を出力する。

シグネチャまたは形状

void test_open_sync(const char *msg, int writes_size)

入力と出力

  • msg: 出力ラベルとして使用される文字列。
  • writes_size: 1回あたりの書き込みサイズ（KB単位）。
  • 出力: 標準出力に対し、計測結果または n/a（未サポート時）を表示する。

前提条件と不変条件

  • O_SYNC マクロが定義されている環境でのみ、実際の書き込みテストが実行される。
  • filename がグローバルスコープ等で定義され、アクセス可能であること。
  • buf が書き込み用のデータバッファとして準備されていること。

アルゴリズムまたは振る舞い

1. 指定された msg をラベルとして出力する。 2. O_SYNC が定義されている場合、open_direct を用いてファイルを開く。 3. 開けない場合は n/a* を出力する。 4. 開けた場合、alarm_triggered が true になるまで以下のループを繰り返す：

  • 16 / writes_size 回の書き込みを pg_pwrite で実行する。
  • 各書き込みは writes_size * 1024 バイトを対象とする。

    • 5. 計測終了後、ファイルを閉じる。 6. O_SYNC が未定義の場合は n/a を出力する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_test_fsync/pg_test_fsync.c 内のメイン処理フロー（行456-460付近）。
  • 呼び出し先: open_direct, printf, fflush, START_TIMER, pg_pwrite, STOP_TIMER, close, die。

関連データ構造

  • buf: 書き込みに使用されるデータバッファ。
  • filename: テスト対象のファイルパス。
  • alarm_triggered: 計測期間の終了を判定するフラグ。

失敗モードと境界条件

  • open_direct が失敗した場合、n/a* を表示して終了する。
  • pg_pwrite が期待されるバイト数を書き込めなかった場合、die 関数によりエラー終了する。
  • O_SYNC がプラットフォームでサポートされていない場合、テストはスキップされ n/a と表示される。

保守担当者向けメモ

  • writes_size は 16 の約数であることを前提としたループ構造になっているため、呼び出し側の引数設定に注意すること。
  • O_SYNC の挙動はプラットフォーム（OS/ファイルシステム）に強く依存するため、結果の解釈には注意が必要。
