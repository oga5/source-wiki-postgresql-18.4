EnablePortalManager

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/portalmem.c:103

目的

ポータル管理システムを初期化し、ポータル構造体を保持するためのメモリコンテキストと、ポータルを名前で管理するためのハッシュテーブルを作成する。

シグネチャまたは形状

void EnablePortalManager(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 TopPortalContext および PortalHashTable を初期化する）

前提条件と不変条件

  • TopPortalContext が NULL であること（二重初期化の防止）。

アルゴリズムまたは振る舞い

1. TopPortalContext が NULL であることをアサートで確認する。 2. TopMemoryContext を親として、ポータル専用のメモリコンテキスト TopPortalContext を作成する。 3. ポータル名（MAX_PORTALNAME_LEN）をキーとし、PortalHashEnt をエントリとするハッシュテーブル PortalHashTable を作成する。 4. ハッシュテーブルの初期サイズには PORTALS_PER_USER を使用する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/init/postinit.c:812
  • 呼び出し先: AllocSetContextCreate（メモリコンテキスト作成）、hash_create（ハッシュテーブル作成）

関連データ構造

  • TopPortalContext: ポータル構造体自体を割り当てるためのメモリコンテキスト。
  • PortalHashTable: 全ポータルを管理するハッシュテーブル。
  • PortalHashEnt: ハッシュテーブルのエントリ構造体。

失敗モードと境界条件

  • メモリ不足時: AllocSetContextCreate または hash_create 内部でメモリ確保に失敗した場合、PostgreSQLの標準的なエラーハンドリングによりアボートする。

保守担当者向けメモ

  • 本関数はバックエンドの初期化プロセスにおいて一度だけ呼び出されることを想定している。
  • PortalHashTable の初期サイズ PORTALS_PER_USER は、システム全体のポータル利用状況に応じて調整が必要となる場合がある。
