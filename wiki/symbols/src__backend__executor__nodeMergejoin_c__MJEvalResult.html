MJEvalResult

定義

  • 種別: enum
  • 定義箇所: src/backend/executor/nodeMergejoin.c:143

目的

マージ結合処理において、入力タプルが結合に適しているか、あるいは結合を終了すべきかを判定するための評価結果を表現する。

シグネチャまたは形状

typedef enum
{
	MJEVAL_MATCHABLE,
	MJEVAL_NONMATCHABLE,
	MJEVAL_ENDOFJOIN,
} MJEvalResult;

入力と出力

  • 本シンボルは列挙型であり、関数等の戻り値として使用される。
  • MJEVAL_MATCHABLE: 通常の結合可能なタプル。
  • MJEVAL_NONMATCHABLE: NULL値を含む等の理由で結合不可能なタプル。
  • MJEVAL_ENDOFJOIN: 入力の終端（物理的または論理的）。

前提条件と不変条件

  • マージ結合の実行中、タプルの評価結果として必ずいずれかの値が返される。
  • NULL値を含むタプルは、結合条件に関わらず MJEVAL_NONMATCHABLE として扱われる。

アルゴリズムまたは振る舞い

  • マージ結合のメインループにおいて、入力タプルの状態を識別するために使用される。
  • 結合条件の評価結果に基づき、ステートマシンが次に取るべきアクション（スキップ、結合、終了）を決定する際の判断基準となる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/nodeMergejoin.c 内のタプル評価関数（MJEvalOuterValues, MJEvalInnerValues 等）から返される。
  • 呼び出し先: 呼び出し元に関する具体的な情報は提供されていない。

関連データ構造

  • MergeJoinClauseData: 結合キーの比較に使用される構造体。
  • ExecMergeJoin: 本列挙型を利用してステートマシンを駆動するメイン関数。

失敗モードと境界条件

  • 入力ストリームが空の場合や、結合条件が満たされない場合に MJEVAL_ENDOFJOIN や MJEVAL_NONMATCHABLE が適切に返されることが、結合の正確性を保証する。

保守担当者向けメモ

  • 本列挙型はマージ結合のステートマシン制御の根幹に関わるため、新しい結合状態を追加する際は、既存の ExecMergeJoin 内の分岐ロジックへの影響を十分に検討すること。
