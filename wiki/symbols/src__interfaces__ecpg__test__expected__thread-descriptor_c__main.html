定義

目的

ECPG（Embedded SQL for C）におけるマルチスレッド環境下での SQL 記述子（Descriptor）の割り当ておよび解放処理の安定性を検証する回帰テストのエントリポイントです。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は使用しません）。
  • 出力: 正常終了時に 0 を返します。

前提条件と不変条件

  • THREADS 定数で定義された数のスレッドを並行実行できる環境であること。
  • プラットフォームに応じて pthread または Windows スレッド API が利用可能であること。

アルゴリズムまたは振る舞い

1. **スレッド生成**: THREADS で指定された回数分、各プラットフォームの API（_beginthreadex または pthread_create）を用いてスレッドを起動し、関数 fn を実行させます。 2. **並行実行**: 各スレッドは fn 内で SQL 記述子の割り当てと解放を繰り返します。 3. **同期**: WaitForMultipleObjects（Windows）または pthread_join（POSIX）を使用して、すべてのスレッドが終了するまで待機します。 4. **リソース解放**: Windows 環境の場合、生成したスレッドハンドルを CloseHandle で解放します。

呼び出し元と呼び出し先

  • 呼び出し元: OS のプロセスローダー（エントリポイント）。
  • 呼び出し先:
  • fn（スレッド実行関数）
  • _beginthreadex / pthread_create（スレッド生成）
  • WaitForMultipleObjects / pthread_join（同期）
  • CloseHandle（リソース解放）

関連データ構造

  • threads: スレッド識別子を保持する配列（HANDLE または pthread_t）。

失敗モードと境界条件

  • スレッド生成に失敗した場合の挙動は、各プラットフォームの API 仕様に依存します。
  • fn 内で SQL エラーが発生した場合、sqlca.sqlcode を通じてエラーが検出され、標準出力へ出力されます。

保守担当者向けメモ

  • 本関数はプリプロセッサが生成したテストコードの一部です。直接編集せず、生成元のテンプレートやプリプロセッサのロジックを確認してください。
  • Windows と POSIX で条件付きコンパイルが分かれているため、スレッド制御の変更時は両方のパスを考慮する必要があります。
