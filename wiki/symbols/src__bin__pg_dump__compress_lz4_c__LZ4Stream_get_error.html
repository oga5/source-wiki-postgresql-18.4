LZ4Stream_get_error

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_lz4.c:329

目的

LZ4 圧縮ストリームに関連するエラーメッセージを取得する。

シグネチャまたは形状

const char *LZ4Stream_get_error(CompressFileHandle *CFH)

入力と出力

  • **入力**: CompressFileHandle *CFH（LZ4 圧縮ストリームのハンドル）。
  • **出力**: エラー内容を示す文字列（const char *）。

前提条件と不変条件

  • CFH->private_data は LZ4State 構造体としてキャスト可能であること。
  • state->errcode には、直近の LZ4 操作の結果が保持されていること。

アルゴリズムまたは振る舞い

1. CFH->private_data から LZ4State を取得する。 2. LZ4F_isError(state->errcode) を評価する。

  • 真の場合: LZ4F_getErrorName を呼び出し、LZ4 ライブラリ固有のエラー文字列を返す。
  • 偽の場合: strerror(errno) を呼び出し、システムエラー文字列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/compress_lz4.c 内の読み込みエラー発生箇所（pg_fatal 呼び出し時）。
  • **呼び出し先**:
  • LZ4F_isError (liblz4)
  • LZ4F_getErrorName (liblz4)
  • strerror (libc)

関連データ構造

  • CompressFileHandle: 圧縮ストリームの抽象化ハンドル。
  • LZ4State: LZ4 圧縮・解凍のコンテキストおよびエラーコードを保持する構造体。

失敗モードと境界条件

  • CFH が NULL の場合、あるいは private_data が正しく初期化されていない場合は未定義の動作となる。
  • state->errcode が LZ4 のエラーコードでない場合、strerror(errno) がフォールバックとして使用される。

保守担当者向けメモ

  • 本関数は CFH->get_error_func に割り当てられ、エラーハンドリングのインターフェースとして機能する。
  • LZ4F ライブラリのバージョンアップに伴い、エラーコードの判定ロジックに変更が必要になる可能性がある。
