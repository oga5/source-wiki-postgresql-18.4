freeJsonLexContext

定義

  • 種別: function
  • 定義箇所: src/common/jsonapi.c:687

目的

JSON字句解析コンテキスト（JsonLexContext）に関連付けられたリソースを解放し、メモリリークを防ぐ。

シグネチャまたは形状

void freeJsonLexContext(JsonLexContext *lex)

入力と出力

  • **入力**: JsonLexContext *lex - 解放対象のコンテキストへのポインタ。
  • **出力**: なし（副作用としてメモリ解放および構造体のリセットを行う）。

前提条件と不変条件

  • lex が NULL または failed_oom（メモリ不足時の特殊な静的ポインタ）である場合、関数は何もせず即座に復帰する。
  • JSONLEX_FREE_STRUCT フラグが設定されている場合、コンテキスト構造体自体も解放される。設定されていない場合は、構造体の中身がゼロクリアされる。

アルゴリズムまたは振る舞い

1. **ガードチェック**: lex が無効な場合は終了。 2. **文字列バッファの解放**: JSONLEX_FREE_STRVAL フラグがあれば lex->strval を、lex->errormsg が存在すればそれを解放。 3. **インクリメンタル状態の解放**: lex->incremental が真の場合、以下のリソースを順次解放：

  • inc_state->partial_token
  • inc_state 自体
  • pstack->prediction
  • JSONLEX_CTX_OWNS_TOKENS フラグがある場合、pstack->fnames 内の各トークン文字列
  • pstack の各メンバ（fnames, fnull, scalar_val）および pstack 自体

    • 4. **構造体の処理**: JSONLEX_FREE_STRUCT フラグの有無に応じて、構造体自体の解放またはゼロクリアを行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/json.c、jsonb.c、および jsonfuncs.c の各所。主にJSON解析処理の終了時に呼び出される。
  • **呼び出し先**: jsonapi_destroyStringInfo、jsonapi_termStringInfo、および環境に応じた FREE マクロ。

関連データ構造

  • JsonLexContext: 解析状態を保持するメイン構造体。
  • JsonIncrementalState: インクリメンタル解析時に使用される状態保持構造体。

失敗モードと境界条件

  • lex が NULL の場合、安全に無視される。
  • failed_oom ポインタが渡された場合、安全に無視される。
  • インクリメンタル解析時、pstack や inc_state が適切に初期化されていない状態で呼び出された場合の挙動は、構造体の初期化状態に依存する。

保守担当者向けメモ

  • 本関数は、JSONAPI_USE_PQEXPBUFFER マクロの定義状況に応じて、バックエンド（palloc）とフロントエンド（malloc）のメモリ管理を抽象化する FREE マクロを利用している。
  • インクリメンタル解析用のスタック構造（pstack）の解放順序は、依存関係に従っているため変更時は注意が必要。
