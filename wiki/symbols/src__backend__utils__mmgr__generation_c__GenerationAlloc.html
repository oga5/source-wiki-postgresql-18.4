GenerationAlloc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/generation.c:526

目的

GenerationContext 内でメモリチャンクを割り当てる。FIFO（先入れ先出し）に近い割り当てパターンに最適化されており、ブロックの再利用を通じてメモリ断片化を抑制する。

シグネチャまたは形状

void *GenerationAlloc(MemoryContext context, Size size, int flags)

入力と出力

  • **入力**:
  • context: 対象の GenerationContext。
  • size: 要求するメモリサイズ（バイト単位）。
  • flags: 割り当てオプション（MCXT_ALLOC_HUGE 等）。
  • **出力**: 確保されたメモリ領域へのポインタ。

前提条件と不変条件

  • context は GenerationIsValid を満たす有効な GenerationContext であること。
  • MEMORY_CONTEXT_CHECKING が有効な場合、センチネルバイト用に size + 1 のアライメント調整が行われる。

アルゴリズムまたは振る舞い

1. **サイズ判定**: 要求サイズが allocChunkLimit を超える場合、GenerationAllocLarge に委譲する。 2. **現在のブロック確認**: 現在のブロックに要求サイズ分の空きがあるか確認する。 3. **ブロック再利用**: 現在のブロックに空きがない場合、freeblock（空のブロック）が存在すればそれを現在のブロックに昇格させ、そこから割り当てる。 4. **新規確保**: freeblock が存在しない、またはサイズ不足の場合は、GenerationAllocFromNewBlock を呼び出し、新しいブロックを確保して割り当てる。 5. **チャンク割り当て**: 決定したブロックから GenerationAllocChunkFromBlock を用いてチャンクを切り出す。

関連データ構造

  • GenerationContext: メモリコンテキストのメイン構造体。
  • GenerationBlock: メモリ管理単位。
  • MemoryContext: 抽象インターフェース。

失敗モードと境界条件

  • 要求サイズが allocChunkLimit を超える場合、GenerationAllocLarge が呼び出される。
  • freeblock が存在する場合でも、サイズが不足していれば新規ブロックが作成される。
  • KeeperBlock は再利用対象外であり、コンテキストのリセット時まで生存する。

保守担当者向けメモ

  • FIFO ワークロードでの断片化を防ぐため、現在のブロックと freeblock 間での頻繁な切り替え（ピンポン現象）を避ける設計となっている。
  • MEMORY_CONTEXT_CHECKING 定義時のセンチネルバイトの扱いに注意すること。
