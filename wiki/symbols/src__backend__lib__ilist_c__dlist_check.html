dlist_check

定義

  • 種別: function
  • 定義箇所: src/backend/lib/ilist.c:76

目的

二重連結リストのポインタ整合性を検証し、リスト構造の破損を検知する。主にデバッグ用途で使用される。

シグネチャまたは形状

void dlist_check(const dlist_head *head)

入力と出力

  • **入力**: head (検証対象の二重連結リストのヘッドポインタ)
  • **出力**: なし（正常時は復帰）
  • **副作用**: リストの破損が検知された場合、elog(ERROR, ...) を呼び出しプロセスを中断する。

前提条件と不変条件

  • head が NULL であってはならない。
  • リストが空（head->head.next および prev が NULL）の場合、初期化済みとみなして正常終了する。
  • 正常なリストでは、任意のノード cur に対して cur->prev->next == cur かつ cur->next->prev == cur が成立しなければならない。

アルゴリズムまたは振る舞い

1. head が NULL の場合、エラーを報告する。 2. リストがゼロ初期化されている場合は直ちに復帰する。 3. **順方向走査**: head->head.next から開始し、head->head に戻るまで各ノードを辿る。各ノードおよびその隣接ポインタが NULL でないこと、および相互参照関係（prev->next と next->prev）が正しいことを確認する。 4. **逆方向走査**: head->head.prev から開始し、head->head に戻るまで同様の検証を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/lib/ilist.h 内の複数のマクロ（dlist_insert_after 等のデバッグ用ラッパー）から呼び出される。
  • **呼び出し先**: elog（エラー報告用）。

関連データ構造

  • dlist_head: 二重連結リストのヘッド構造体。
  • dlist_node: リストの各要素を構成するノード構造体。

失敗モードと境界条件

  • head が NULL の場合、エラーが発生する。
  • リストのリンクが途切れている（NULL を含む）、または相互参照が不整合な場合、エラーが発生する。
  • 循環リストの終端判定（&head->head への到達）が正しく行われない場合、無限ループに陥る可能性がある。

保守担当者向けメモ

  • 本関数は ILIST_DEBUG が有効なビルド環境でのみ機能するよう設計されている。
  • O(n) の計算量を持つため、本番環境での呼び出しはパフォーマンスに影響を与える可能性がある。
