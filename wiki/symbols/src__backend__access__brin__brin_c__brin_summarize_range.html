brin_summarize_range

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin.c:1380

目的

指定されたBRINインデックスの特定のヒープページ範囲を要約し、インデックス情報を更新する。主にオートバキュームやメンテナンス操作から呼び出される。

シグネチャまたは形状

Datum brin_summarize_range(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): 対象となるBRINインデックスのOID。
  • PG_GETARG_INT64(1): 要約対象のヒープブロック範囲（heapBlk64）。
  • **出力**:
  • PG_RETURN_INT32: 要約された範囲の数（int32）。

前提条件と不変条件

  • データベースのリカバリ中ではないこと。
  • 指定されたブロック番号が有効範囲内であること。
  • 対象のOIDが有効なBRINインデックスであること。
  • 呼び出し元がインデックスの所有者権限を有していること。
  • インデックスが有効（indisvalid）であること。

アルゴリズムまたは振る舞い

1. **検証**: リカバリ状態、ブロック番号の範囲、インデックスの型、および所有者権限をチェックする。 2. **セットアップ**: ヒープリレーションとインデックスリレーションを ShareUpdateExclusiveLock でロックする。オートバキューム等のために、実行コンテキストをテーブル所有者のユーザーIDに切り替え、検索パスを制限する。 3. **要約実行**: インデックスが有効な場合、brinsummarize を呼び出して指定範囲の要約処理を実行する。 4. **クリーンアップ**: GUC設定のロールバック、ユーザーIDとセキュリティコンテキストの復元を行い、リレーションをクローズする。 5. **戻り値**: 要約された範囲数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/autovacuum.c (直接的な呼び出し)
  • src/backend/access/brin/brin.c 内の内部処理
  • **呼び出し先**:
  • brinsummarize (要約処理の実行)
  • IndexGetRelation, table_open, index_open, relation_close (リレーション管理)
  • GetUserIdAndSecContext, SetUserIdAndSecContext (権限管理)

関連データ構造

  • Relation: インデックスおよびヒープリレーションの管理に使用。
  • BRIN_AM_OID: BRINインデックスであることを確認するための定数。

失敗モードと境界条件

  • **エラー**:
  • リカバリ中に実行された場合。
  • ブロック番号が範囲外の場合。
  • 対象がBRINインデックスではない場合。
  • 呼び出し元がインデックスの所有者ではない場合。
  • インデックスの親テーブルが開けない場合。
  • **境界条件**:
  • インデックスが無効（indisvalid が偽）な場合は要約を行わず、DEBUG1 レベルのログを出力して終了する。

保守担当者向けメモ

  • 本関数は ShareUpdateExclusiveLock を使用するため、通常のインデックス操作と並行して実行可能である。
  • brinsummarize への引数として true を渡しており、これは要約処理が強制的に実行されることを意味する。
  • 実行中にGUCの変更が発生した場合、AtEOXact_GUC によりトランザクション終了時と同様のロールバックが行われる。
