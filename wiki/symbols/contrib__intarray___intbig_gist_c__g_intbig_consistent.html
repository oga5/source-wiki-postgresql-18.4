g_intbig_consistent

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_intbig_gist.c:464

目的

GiST インデックスにおいて、検索条件（演算子）とインデックスエントリを照合し、インデックスツリーの探索を継続すべきかを判定する。

シグネチャまたは形状

Datum g_intbig_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • entry: GISTENTRY* (インデックスエントリ)
  • query: ArrayType* (検索対象の配列)
  • strategy: StrategyNumber (演算子戦略)
  • recheck: bool* (出力用: 常に true を設定)
  • **出力**:
  • bool: 検索条件に一致する可能性がある場合は true、そうでない場合は false。

前提条件と不変条件

  • 本関数は「損失あり（lossy）」の検索を行うため、呼び出し元は常に *recheck = true を受け取り、検索結果に対して再チェックを行う必要がある。
  • ISALLTRUE マクロで判定される全ビットがセットされたエントリは、常に一致とみなされる。

アルゴリズムまたは振る舞い

1. *recheck を true に設定する。 2. インデックスエントリが全ビットセットされている場合、直ちに true を返す。 3. BooleanSearchStrategy の場合、signconsistent を呼び出してビットベクトルとの照合を行う。 4. その他の戦略（RTOverlapStrategyNumber, RTSameStrategyNumber, RTContainsStrategyNumber 等）については、以下の通り処理する:

  • RTOverlapStrategyNumber / RTContainsStrategyNumber: _intbig_overlap または _intbig_contains を呼び出す。
  • RTSameStrategyNumber: リーフノードであればビット単位の比較を行い、内部ノードであれば _intbig_contains を使用する。
  • RTContainedByStrategyNumber: リーフノードではビット単位の包含チェックを行い、内部ノードでは常に true を返す（空配列の存在を考慮した全探索）。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックスエンジン（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: signconsistent, _intbig_overlap, _intbig_contains, HASH, LOOPBYTE 等の内部ヘルパー関数およびマクロ。

関連データ構造

  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。
  • GISTTYPE: intbig 型のインデックス用データ構造。
  • ArrayType: 検索クエリとして渡される配列。

失敗モードと境界条件

  • 未サポートの戦略が渡された場合、false を返す。
  • RTContainedByStrategyNumber は現在非推奨だが、古い SQL 定義との互換性のために保持されている。
  • 損失あり検索のため、偽陽性が発生する。

保守担当者向けメモ

  • 本関数は intarray 1.4 以降、<@ 演算子が削除されたため、RTContainedByStrategyNumber の一部ロジックは到達不能となっている。
  • siglen は GET_SIGLEN() マクロにより動的に取得されるため、インデックス作成時の設定値に依存する。
