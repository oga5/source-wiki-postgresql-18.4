plpgsql_compile

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_comp.c:105

目的

PL/pgSQL 関数のソーステキストを解析し、実行可能な PLpgSQL_function 構造体（実行ツリー）を構築する主要なエントリーポイントです。

シグネチャまたは形状

PLpgSQL_function *plpgsql_compile(FunctionCallInfo fcinfo, bool forValidator)

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報。キャッシュの管理や関数定義の取得に使用。
  • forValidator: バリデーター（CREATE FUNCTION 時の検証など）として実行されているかを示すフラグ。
  • **出力**:
  • 構築された PLpgSQL_function 構造体へのポインタ。

前提条件と不変条件

  • fcinfo->flinfo->fn_extra を使用して、コンパイル済み関数のキャッシュを永続的に保持します。
  • コンパイル処理は再入不可であり、コンパイル中に別の PL/pgSQL 関数が呼び出されることは想定されていません。

アルゴリズムまたは振る舞い

1. cached_function_compile を呼び出し、既存のキャッシュがあれば再利用を試みます。 2. キャッシュがない場合、plpgsql_compile_callback を介してソースコードの解析とツリー構築を行います。 3. 構築された PLpgSQL_function を fcinfo->flinfo->fn_extra に格納し、次回以降の呼び出しで再検索を回避します。 4. 最終的にコンパイル済みの関数構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpgsql/src/pl_handler.c: plpgsql_call_handler 等から呼び出されます。
  • **呼び出し先**:
  • cached_function_compile (内部関数)
  • plpgsql_compile_callback (コンパイル実体)
  • plpgsql_delete_callback (キャッシュ削除用)

関連データ構造

  • PLpgSQL_function: コンパイル結果として生成される実行ツリーのルート構造体。
  • FunctionCallInfo: PostgreSQL の関数呼び出し状態を保持する構造体。

失敗モードと境界条件

  • コンパイル中のメモリ管理は plpgsql_compile_tmp_cxt で行われ、成功時にのみ CacheMemoryContext へ昇格されるため、失敗時のメモリリークが抑制される設計となっています。

保守担当者向けメモ

  • 本関数は pl_handler.c からの呼び出しが主であり、PL/pgSQL のフロントエンドにおけるコンパイル処理の起点です。
  • 複雑な名前解決やパーサーフックの設定は、内部で呼び出される plpgsql_compile_callback および関連するパーサー設定関数群で行われます。
