range_send

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes.c:262

目的

PostgreSQL の範囲型（RangeType）をクライアントへ送信するためのバイナリ形式（bytea）に変換します。

シグネチャまたは形状

Datum range_send(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される RangeType オブジェクト。
  • **出力**: bytea 型のバイナリデータ。

前提条件と不変条件

  • 入力された RangeType は PG_GETARG_RANGE_P を通じて取得可能であること。
  • check_stack_depth() により、サブタイプが再帰的な範囲型である場合のスタックオーバーフローを防止します。
  • get_range_io_data を呼び出し、対象の範囲型に対応する I/O キャッシュを取得する必要があります。

アルゴリズムまたは振る舞い

1. get_range_io_data を用いて、型キャッシュおよび送信関数情報を取得します。 2. range_deserialize を実行し、RangeType から境界値（lower, upper）と空フラグを抽出します。 3. range_get_flags で範囲のフラグを取得します。 4. pq_begintypsend で送信バッファを初期化し、フラグを書き込みます。 5. 境界が存在する場合（RANGE_HAS_LBOUND / RANGE_HAS_UBOUND）、各境界値を SendFunctionCall でシリアライズし、その長さとデータをバッファに追加します。 6. pq_endtypsend を呼び出し、構築されたバッファを bytea として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（SQL インターフェース経由）。
  • **呼び出し先**: check_stack_depth, get_range_io_data, range_deserialize, range_get_flags, pq_begintypsend, pq_sendbyte, SendFunctionCall, pq_sendint32, pq_sendbytes, pq_endtypsend。

関連データ構造

  • RangeType: 範囲型の内部表現。
  • RangeBound: 範囲の境界値を保持する構造体。
  • RangeIOData: 範囲型の I/O 処理に必要なキャッシュ情報。
  • StringInfo: バイナリ送信用のバッファ構造体。

失敗モードと境界条件

  • 入力範囲が TOAST 圧縮されている場合、内部的にデトースト処理が行われます。
  • 境界値のシリアライズ時に、要素型固有の送信関数が失敗した場合はエラーが発生します。

保守担当者向けメモ

  • 本関数はバイナリプロトコルにおける範囲型のシリアライズを担います。要素型の送信関数（cache->typioproc）に依存しているため、要素型の変更が範囲型のバイナリ表現に直接影響します。
