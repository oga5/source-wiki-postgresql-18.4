pull_vars_of_level

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/var.c:338

目的

クエリツリーまたは式ツリーから、指定されたネストレベル（levelsup）に一致する Var ノードおよび PlaceHolderVar ノードを抽出し、リストとして返します。

シグネチャまたは形状

List *pull_vars_of_level(Node *node, int levelsup)

入力と出力

  • **入力**:
  • node: 走査対象の Node（Query または式ツリー）。
  • levelsup: 抽出対象とする変数のネストレベル（0は現在のクエリレベル）。
  • **出力**:
  • 抽出されたノードの List。該当するものがない場合は NIL。

前提条件と不変条件

  • node が Query である場合、その内部の sublevels_up はインクリメントされません。
  • 内部的に pull_vars_context を使用して走査状態を管理します。

アルゴリズムまたは振る舞い

1. pull_vars_context を初期化し、vars を NIL に、sublevels_up を引数の levelsup に設定します。 2. query_or_expression_tree_walker を呼び出し、pull_vars_walker を用いてツリーを再帰的に走査します。 3. 走査中に指定されたレベルと一致する変数ノードが見つかると、コンテキスト内のリストに追加されます。 4. 走査完了後、収集されたリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/joinpath.c
  • src/backend/optimizer/plan/initsplan.c
  • **呼び出し先**:
  • query_or_expression_tree_walker
  • pull_vars_walker（内部的な走査関数）

関連データ構造

  • pull_vars_context: 走査中の状態（収集したリストとターゲットレベル）を保持する構造体。
  • Var, PlaceHolderVar: 抽出対象となるノード型。

失敗モードと境界条件

  • 入力 node が NULL の場合、空のリストを返します。
  • 該当するレベルの変数が存在しない場合、NIL を返します。

保守担当者向けメモ

  • 本関数は expression_tree_walker の仕組みを利用しています。サブクエリや SubLink を含む複雑なツリーを扱う際は、sublevels_up の値が正しく管理されていることを確認してください。
  • PlaceHolderVar は Var と同様に扱われるため、オプティマイザの変換処理においてこれらが混在する点に留意してください。
