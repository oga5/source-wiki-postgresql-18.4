render_newlen

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-encrypt.c:44

目的

OpenPGP 仕様（RFC 4880）に基づき、パケットのデータ長を可変長形式（New Length Encoding）でエンコードし、指定されたバッファへ書き込む。

シグネチャまたは形状

uint8 *render_newlen(uint8 *h, int len)

入力と出力

  • **入力**:
  • h: 書き込み先バッファのポインタ。
  • len: エンコード対象のデータ長（整数）。
  • **出力**:
  • 書き込み後のバッファポインタの次位置を返す。

前提条件と不変条件

  • h が指すバッファは、エンコード結果を格納するために十分な領域（最大5バイト）を確保している必要がある。

アルゴリズムまたは振る舞い

len の値に応じて以下のいずれかの形式でエンコードを行う： 1. **len <= 191**: 1バイトで表現。len をそのまま格納。 2. **191 < len <= 8383**: 2バイトで表現。上位バイトに ((len - 192) >> 8) + 192、下位バイトに (len - 192) & 255 を格納。 3. **len > 8383**: 5バイトで表現。先頭に 255 を書き込み、続く4バイトに len をビッグエンディアン形式で格納。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-encrypt.c 内の複数箇所（82行目、266行目、288行目など）から呼び出されている。
  • **呼び出し先**:
  • 本関数は外部関数を呼び出さない。

関連データ構造

  • uint8: PGP パケットのバイト列操作に使用される型。

失敗モードと境界条件

  • len が負の値の場合の動作は定義されていない。
  • バッファ h のサイズが不足している場合、メモリ破壊が発生する可能性がある。

保守担当者向けメモ

  • 本関数は RFC 4880 の「New Packet Lengths」セクションの実装である。仕様変更やパケット形式の拡張が必要な場合は、このエンコードロジックを再確認すること。
