ExecHashJoinOuterGetTuple

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHashjoin.c:978

目的

ハッシュ結合における外部（Outer）リレーションから、結合処理対象となるタプルをフェッチする。メモリ内に収まらない場合は、ディスク上のバッチファイルからタプルを復元する。

シグネチャまたは形状

TupleTableSlot *ExecHashJoinOuterGetTuple(PlanState *outerNode, HashJoinState *hjstate, uint32 *hashvalue)

入力と出力

  • **入力**:
  • outerNode: 外部リレーションの実行状態（PlanState）。
  • hjstate: ハッシュ結合の実行状態（HashJoinState）。
  • hashvalue: 計算されたハッシュ値を格納するポインタ。
  • **出力**:
  • 結合対象となる TupleTableSlot。タプルがない場合は NULL を返す。

前提条件と不変条件

  • hjstate->hj_HashTable が初期化されていること。
  • curbatch == 0 の場合、hjstate->hj_FirstOuterTupleSlot に未処理のタプルがあればそれが優先される。
  • curbatch > 0 の場合、hashtable->outerBatchFile[curbatch] が有効であること。

アルゴリズムまたは振る舞い

1. **バッチ0（初回パス）の処理**:

  • hj_FirstOuterTupleSlot にタプルがあればそれを使用し、なければ ExecProcNode(outerNode) で外部ノードからタプルを取得する。
  • 取得したタプルに対し、hj_OuterHash を用いてハッシュ値を計算する。
  • ハッシュ値が計算できない（NULL）タプルはスキップし、次のタプルを取得する。
  • 有効なタプルが得られた場合、hj_OuterNotEmpty を true に設定し、そのタプルを返す。

    • 2. **バッチ1以降の処理**:

  • hashtable->outerBatchFile[curbatch] から ExecHashJoinGetSavedTuple を呼び出し、退避されていたタプルとハッシュ値を復元する。
  • 復元されたタプルを返す。

    • 3. **終了条件**:

  • 該当するバッチでタプルが取得できない場合、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHashjoin.c 内の結合メインループ（ExecHashJoinImpl 等から参照される）。
  • **呼び出し先**:
  • ExecProcNode（外部ノードからのタプル取得）
  • ExecEvalExprSwitchContext（ハッシュ値計算）
  • ExecHashJoinGetSavedTuple（ディスクからのタプル復元）

関連データ構造

  • HashJoinState: 結合の実行状態を保持。
  • HashJoinTable: ハッシュテーブルおよびバッチ管理情報。
  • TupleTableSlot: タプルデータを保持するコンテナ。

失敗モードと境界条件

  • curbatch が 0 の場合、外部リレーションが空であれば NULL を返す。
  • curbatch > 0 の場合、バッチファイルが空であれば NULL を返す。
  • ハッシュ値計算時に NULL が発生した場合、そのタプルは結合対象から除外される。

保守担当者向けメモ

  • 本関数はシリアル実行用です。並列実行時には ExecParallelHashJoinOuterGetTuple が使用されます。
  • curbatch の遷移は hjstate->hj_JoinState のステートマシンと連動しているため、デバッグ時はステートの整合性に注意してください。
