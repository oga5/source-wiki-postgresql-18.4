NODE

定義

  • 種別: struct
  • 定義箇所: contrib/ltree/ltxtquery_io.c:27

目的

ltxtquery（検索クエリ）の構文解析中に、トークンを一時的に保持し、連結リスト形式でツリー構造を構築するために使用されるデータ構造です。

シグネチャまたは形状

typedef struct NODE
{
	int32		type;
	int32		val;
	int16		distance;
	int16		length;
	uint16		flag;
	struct NODE *next;
} NODE;

入力と出力

  • 本構造体は、makepol 関数等において palloc を介して動的にメモリ確保されます。
  • 最終的に ltxtquery 内部の ITEM 配列へ変換されるため、本構造体自体は解析用の一時的な中間表現として機能します。

前提条件と不変条件

  • next ポインタにより、単方向連結リストを形成します。
  • type および val は、クエリ内の演算子やオペランドの種類を識別するために使用されます。

アルゴリズムまたは振る舞い

  • ltxtq_in によるクエリ解析プロセスにおいて、トークン化された要素を順次 NODE として生成し、リストに追加します。
  • 構築されたリストは、演算子の優先順位に従って後置記法（逆ポーランド記法）のツリー構造へ変換されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/ltree/ltxtquery_io.c 内の makepol 関数などが、クエリ解析時に本構造体をインスタンス化します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。
  • 注: contrib/intarray/_int_bool.c にも同名の構造体定義が存在しますが、本ページでは ltxtquery_io.c の定義に焦点を当てています。

関連データ構造

  • QPRS_STATE: 解析中の状態を保持する構造体であり、NODE リストの管理に使用されます。
  • ITEM: ltxtquery の最終的なバイナリ形式を構成する要素です。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • 連結リストの構築過程で、クエリの構文エラーやスタックオーバーフロー（再帰深さ制限）が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は ltree モジュールと intarray モジュールで重複定義されています。修正を行う際は、両方のモジュールへの影響を考慮してください。
  • 解析用の一時構造体であるため、解析終了後は適切にメモリが解放されるか、あるいは ITEM 配列へ変換された後に破棄されるフローであることを確認してください。
