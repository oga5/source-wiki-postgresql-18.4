pgsocket

定義

  • 種別: typedef
  • 定義箇所: src/include/port.h:29

目的

PostgreSQL がサポートする多様なプラットフォーム（Unix系およびWindows）間でのソケット記述子の型を抽象化し、移植性を確保するために使用されます。

シグネチャまたは形状

typedef int pgsocket;

入力と出力

  • **入力**: int 型のソケット記述子。
  • **出力**: ソケット記述子を保持する変数として機能します。

前提条件と不変条件

  • プラットフォーム固有のソケット型を int に統一して扱います。
  • 無効なソケット状態を表すために PGINVALID_SOCKET が併用されることが一般的です。

アルゴリズムまたは振る舞い

本シンボルは型定義であり、実行時のアルゴリズムは持ちません。ネットワーク通信を行う際、OS 依存のソケット型を pgsocket にキャストまたは代入することで、コアコードが OS の差異を意識せずにソケットを操作できるようにします。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/libpq/auth.c: Ident サーバーとの通信用ソケットとして使用。
  • src/backend/libpq/ifaddr.c: ネットワークインターフェースのアドレス取得用。
  • src/backend/libpq/pqcomm.c: 接続待受ソケットの管理および接続受け入れ（AcceptConnection）に使用。
  • src/backend/postmaster/postmaster.c: 待受ソケットの配列管理に使用。
  • src/backend/replication/libpqwalreceiver/libpqwalreceiver.c: WAL 受信用の通信に使用。
  • src/backend/replication/logical/tablesync.c / worker.c: 論理レプリケーションの通信用。
  • **呼び出し先**:
  • 本シンボルは型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • PGINVALID_SOCKET: 無効なソケットを示す定数（本シンボルと組み合わせて使用）。
  • ClientSocket: 接続されたクライアントソケットを保持する構造体。

失敗モードと境界条件

  • int 型の範囲を超える記述子を扱う環境では、本定義の適合性に注意が必要です。
  • Windows 環境では、標準的な socket 関数が返す値と pgsocket の互換性が port/win32_port.h 等のレイヤーで保証されている必要があります。

保守担当者向けメモ

  • 本シンボルは src/include/port.h におけるプラットフォーム抽象化の基盤です。
  • ソケット関連の関数シグネチャを変更する際は、必ず pgsocket 型を使用し、int を直接使用しないようにしてください。
  • Windows 環境でのデバッグ時には、ソケット記述子が pgwin32_ 系関数によってどのようにラップされているかを確認してください。
