PLy_result_coltypes

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_resultobject.c:165

目的

PLyResult オブジェクトが保持するクエリ結果の各列のデータ型ID（OID）を、Pythonのリストとして取得します。

シグネチャまたは形状

static PyObject *PLy_result_coltypes(PyObject *self, PyObject *unused)

入力と出力

  • **入力**:
  • self: PLyResultObject へのポインタ。
  • unused: 使用されない引数（METH_NOARGS 用）。
  • **出力**:
  • 成功時: 各列の型IDを格納した PyListObject。
  • 失敗時: NULL（例外がセットされます）。

前提条件と不変条件

  • self は PLyResultObject 型である必要があります。
  • ob->tupdesc が有効（NULLでない）である必要があります。

アルゴリズムまたは振る舞い

1. ob->tupdesc が存在するか確認し、存在しない場合は PLy_exc_error を発生させます。 2. tupdesc->natts（列数）のサイズを持つ新しい Python リストを作成します。 3. 各列について TupleDescAttr を用いて属性情報を取得し、その atttypid を PyLong_FromLong で Python の整数オブジェクトに変換してリストに格納します。 4. 完成したリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Python インタープリタから PLyResult オブジェクトのメソッドとして呼び出されます（src/pl/plpython/plpy_resultobject.c:29 のメソッド定義テーブルを参照）。
  • **呼び出し先**:
  • PLy_exception_set（エラー時）
  • PyList_New
  • TupleDescAttr
  • PyList_SetItem
  • PyLong_FromLong

関連データ構造

  • PLyResultObject: クエリ結果を保持するC構造体。
  • TupleDesc: PostgreSQLの行記述子。

失敗モードと境界条件

  • tupdesc が NULL の場合: 「command did not produce a result set」というエラーメッセージと共に NULL を返します。
  • メモリ不足時: PyList_New が失敗し、NULL を返します。

保守担当者向けメモ

  • 本関数は METH_NOARGS として登録されています。
  • tupdesc の各属性へのアクセスには TupleDescAttr マクロを使用しており、PostgreSQLの内部構造変更に対する抽象化が行われています。
