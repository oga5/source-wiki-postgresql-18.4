generate_series_int4

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/int.c:1531

目的

整数範囲を生成するセットリターニング関数（SRF）として機能し、SQLクエリ内で指定された開始値から終了値までの整数シーケンスを生成します。

シグネチャまたは形状

  • Datum generate_series_int4(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される、SQLレベルの引数（開始値、終了値、およびオプションのステップ値）。
  • **出力**: Datum 型として、生成された整数のシーケンスを返します。

前提条件と不変条件

  • 本関数はセットリターニング関数（SRF）として実装されており、呼び出し間で状態を保持するために FuncCallContext を利用します。
  • 内部的には generate_series_step_int4 に処理を委譲しています。

アルゴリズムまたは振る舞い

1. SQLから呼び出されると、PG_FUNCTION_ARGS を介して引数を受け取ります。 2. generate_series_step_int4(fcinfo) を呼び出し、実際のシーケンス生成ロジックを実行します。 3. 生成された結果を Datum 形式で返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（SQLクエリからの呼び出し）。また、src/test/regress/regress.c:779 に本関数への言及が存在します。
  • **呼び出し先**: generate_series_step_int4（直接の委譲先）。

関連データ構造

  • FuncCallContext: 呼び出し間での状態（generate_series_fctx）を保持するために使用されます。
  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約に従う引数コンテナ。

失敗モードと境界条件

  • 算術演算においてオーバーフローが発生した場合、common/int.h のオーバーフローチェック関数により ereport(ERROR, ...) が発行され、トランザクションが中断されます。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的なロジックは generate_series_step_int4 に存在します。修正やデバッグを行う際は、委譲先の関数を確認してください。
