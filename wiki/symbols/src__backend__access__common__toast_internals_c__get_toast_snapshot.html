get_toast_snapshot

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/toast_internals.c:637

目的

TOASTデータの読み取り（デトースト）操作に使用する、静的なTOAST用スナップショット（SnapshotToastData）を取得します。

シグネチャまたは形状

Snapshot get_toast_snapshot(void);

入力と出力

  • 入力: なし
  • 出力: Snapshot 型のポインタ（&SnapshotToastData）

前提条件と不変条件

  • 呼び出し時に、セッション内でアクティブなスナップショットが登録されている必要があります。
  • HaveRegisteredOrActiveSnapshot() が真であることを要求します。

アルゴリズムまたは振る舞い

1. HaveRegisteredOrActiveSnapshot() を呼び出し、アクティブなスナップショットが存在するかを確認します。 2. スナップショットが存在しない場合、elog(ERROR) を発生させ処理を中断します。 3. 存在する場合、静的変数 SnapshotToastData のアドレスを返します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • contrib/amcheck/verify_heapam.c:1887
  • src/backend/access/common/toast_internals.c:428
  • src/backend/access/heap/heaptoast.c:688
  • 呼び出し先:
  • HaveRegisteredOrActiveSnapshot()（マクロまたは関数）

関連データ構造

  • Snapshot: スナップショット情報を保持する構造体。
  • SnapshotToastData: TOAST操作専用の静的スナップショットインスタンス。

失敗モードと境界条件

  • トランザクションがコミットされた後にTOASTデータを読み取ろうとするなど、アクティブなスナップショットが存在しない状況下で呼び出された場合、エラーが発生します。これは、コミット後にTOASTデータが削除される可能性があるため、安全性を確保するための制約です。

保守担当者向けメモ

  • 本関数は、TOASTデータの操作がトランザクションの有効期間内で行われていることを保証するためのガードレールとして機能します。
  • 呼び出し元がトランザクションの境界を跨いでデトーストを行おうとしていないか注意してください。
