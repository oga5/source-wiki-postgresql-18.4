xl_xact_relfilelocators

定義

  • 種別: struct
  • 定義箇所: src/include/access/xact.h:268

目的

トランザクションのコミットやアボート時に、WAL（Write-Ahead Logging）へ記録されるリレーションの識別子（RelFileLocator）のリストを保持するために使用されます。

シグネチャまたは形状

typedef struct xl_xact_relfilelocators
{
    int         nrels;          /* リレーション数 */
    RelFileLocator xlocators[FLEXIBLE_ARRAY_MEMBER];
} xl_xact_relfilelocators;

入力と出力

  • **入力**: トランザクション処理中に削除対象となったリレーションの RelFileLocator 配列。
  • **出力**: WALレコードの一部としてシリアライズされたリレーション識別子の集合。

前提条件と不変条件

  • xlocators は FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体自体は可変長です。
  • WALレコードの XINFO_HAS_RELFILELOCATORS フラグが設定されている場合にのみ、この構造体がWALレコード内に存在します。

アルゴリズムまたは振る舞い

1. トランザクション終了時、削除対象のリレーションがある場合に本構造体が作成されます。 2. nrels にリレーションの総数が格納され、続くメモリ領域に RelFileLocator の配列が配置されます。 3. リカバリ時やデバッグ出力時には、この構造体へキャストしてリレーション情報を読み取ります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c 内のトランザクション終了処理（XactLogCommitRecord 等）にて、リレーション情報をWALに書き込む際に使用されます。
  • **呼び出し先**: src/backend/access/rmgrdesc/xactdesc.c にて、WALレコードの解析およびデバッグ表示のために参照されます。

関連データ構造

  • RelFileLocator: リレーションを物理的に識別するための構造体。
  • xl_xact_commit / xl_xact_abort: 本構造体が含まれる可能性のある上位のWALレコード構造体。

失敗モードと境界条件

  • nrels が 0 の場合、本構造体は実質的に空のデータとして扱われます。
  • 可変長構造体であるため、直接的なメモリコピーやサイズ計算には MinSizeOfXactRelfileLocators マクロ（offsetof(xl_xact_relfilelocators, xlocators)）を使用する必要があります。

保守担当者向けメモ

  • 本構造体は可変長データを含むため、解析時には必ず Parse...Record 系関数や適切なオフセット計算を介してアクセスしてください。
  • XINFO_HAS_RELFILELOCATORS フラグとの整合性を常に確認してください。
