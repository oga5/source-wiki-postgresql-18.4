get_matching_location

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_clause.c:3175

目的

指定された sortgroupref に対応する式（expr）のソースコード上の位置情報を取得します。

シグネチャまたは形状

static int get_matching_location(int sortgroupref, List *sortgrouprefs, List *exprs)

入力と出力

  • **入力**:
  • sortgroupref: 検索対象のソート/グループ参照ID。
  • sortgrouprefs: sortgroupref を格納した整数のリスト。
  • exprs: 対応する式（Node）のリスト。
  • **出力**:
  • 成功時: 対応する式のソース位置（int）。
  • 失敗時: elog(ERROR) により処理を中断。

前提条件と不変条件

  • sortgrouprefs リストと exprs リストは同じ長さである必要があります。
  • sortgroupref が sortgrouprefs リスト内に存在することが期待されます。

アルゴリズムまたは振る舞い

1. forboth マクロを使用して sortgrouprefs と exprs を同時に走査します。 2. 現在の sortgrouprefs の要素が sortgroupref と一致するか確認します。 3. 一致した場合、対応する exprs の要素に対して exprLocation() を呼び出し、その結果を返します。 4. リストの最後まで一致が見つからなかった場合、elog(ERROR) を発行してエラー終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_clause.c 内の複数箇所から呼び出されています（例: 3118行目付近）。
  • **呼び出し先**: exprLocation() を呼び出します。

関連データ構造

  • List: PostgreSQLの汎用リスト構造。
  • Node: 式を表現するノード構造体。

失敗モードと境界条件

  • sortgroupref がリスト内に存在しない場合、elog(ERROR) が発生し、プログラムの実行が停止します。

保守担当者向けメモ

  • 本関数は sortgroupref と式の対応付けが必須であるコンテキストで使用されます。呼び出し側で必ず対応が存在することを保証してください。
