mm_strdup

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/util.c:96

目的

文字列を複製し、メモリ確保に失敗した場合には致命的エラーとしてプロセスを終了させる、エラーチェック付きの strdup ラッパーを提供します。

シグネチャまたは形状

char *mm_strdup(const char *string);

入力と出力

  • **入力**: string (複製対象のヌル終端文字列)
  • **出力**: 複製された文字列へのポインタ。成功時は呼び出し元で free が必要です。

前提条件と不変条件

  • 入力 string は有効なヌル終端文字列である必要があります。
  • メモリ確保が成功した場合、戻り値は malloc された領域を指します。

アルゴリズムまたは振る舞い

1. 標準ライブラリの strdup(string) を呼び出し、メモリを確保して内容をコピーします。 2. 戻り値が NULL の場合、mmfatal を呼び出して OUT_OF_MEMORY エラーを報告し、プロセスを終了します。 3. 確保に成功した場合は、そのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し先**: strdup (標準ライブラリ), mmfatal (同ファイル内)
  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/descriptor.c (複数箇所)
  • src/interfaces/ecpg/preproc/ecpg.c (複数箇所)
  • src/interfaces/ecpg/preproc/type.c

関連データ構造

  • 特になし（標準の文字列型 char * を扱う）。

失敗モードと境界条件

  • メモリ不足時: mmfatal を呼び出し、プログラムを終了させます。
  • 入力 string が NULL の場合: strdup の仕様に依存し、未定義動作となる可能性があります。

保守担当者向けメモ

  • 本関数は mmfatal を使用するため、呼び出し元でメモリ確保失敗のハンドリングを行う必要はありません。
  • loc_alloc や loc_strdup とは異なり、本関数で確保されたメモリは reclaim_local_storage では解放されません。呼び出し元で適切に free する必要があります。
