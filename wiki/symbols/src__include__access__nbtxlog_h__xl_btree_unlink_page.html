xl_btree_unlink_page

定義

  • 種別: struct
  • 定義箇所: src/include/access/nbtxlog.h:310

目的

B-treeインデックスにおいて、ページ削除（unlink）操作をWAL（Write-Ahead Log）に記録するためのデータ構造です。この構造体は、クラッシュリカバリ時に削除対象ページとその周辺のリンク状態を再構築するために使用されます。

シグネチャまたは形状

typedef struct xl_btree_unlink_page
{
    BlockNumber leftsib;
    BlockNumber rightsib;
    uint32      level;
    FullTransactionId safexid;
    BlockNumber leafleftsib;
    BlockNumber leafrightsib;
    BlockNumber leaftopparent;
} xl_btree_unlink_page;

入力と出力

  • **入力**: 削除対象ページのブロック番号、左右の兄弟ページ、レベル、および削除操作に関連するトランザクションID（safexid）。内部ページ削除時には、ハーフデッド（half-dead）なリーフページの再構築に必要な兄弟関係および親リンク情報が含まれます。
  • **出力**: WALレコードの一部として書き出され、REDO処理時にインデックスページの物理的なリンク状態を復元するために利用されます。

前提条件と不変条件

  • 内部ページが削除対象である場合、leafleftsib、leafrightsib、leaftopparent フィールドが有効な情報として使用されます。
  • XLOG_BTREE_UNLINK_PAGE_META フラグが設定されている場合、構造体の直後に xl_btree_metadata が続くことが期待されます。

アルゴリズムまたは振る舞い

  • 本構造体は、nbtree のページ削除処理において、削除対象ページをインデックス構造から切り離す際の整合性を保証します。
  • REDOルーチンは、この構造体の情報を用いて対象ページを再構築します。内部ページ削除時には、ハーフデッドなリーフページをゼロから再構築するアプローチをとることで、処理の単純化を図っています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtpage.c (インデックスページ削除処理時)
  • **呼び出し先**:
  • src/backend/access/nbtree/nbtxlog.c (REDO処理)
  • src/backend/access/rmgrdesc/nbtdesc.c (ログ解析・デバッグ出力)

関連データ構造

  • xl_btree_metadata: 構造体の直後に付随する可能性があるメタデータ構造体。
  • BlockNumber: PostgreSQLのブロック識別子。
  • FullTransactionId: トランザクションID。

失敗モードと境界条件

  • 内部ページ削除時とリーフページ削除時で、使用されるフィールドの範囲が異なります。
  • ログの読み込み時に XLOG_BTREE_UNLINK_PAGE_META フラグの有無を正しく判定できない場合、後続のメタデータ読み取りでオフセットの不整合が発生する可能性があります。

保守担当者向けメモ

  • SizeOfBtreeUnlinkPage マクロが leaftopparent までのオフセットを計算するために定義されています。構造体に変更を加える際は、このマクロの定義も併せて確認してください。
  • 内部ページ削除時のハーフデッドページ再構築ロジックは、REDOの冪等性を維持するように設計されています。
