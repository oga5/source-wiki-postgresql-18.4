pg_get_shmem_allocations_numa

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shmem.c:547

目的

共有メモリ上の各割り当て領域が、どのNUMAノードに配置されているかを可視化するためのSQL関数を提供します。

シグネチャまたは形状

Datum pg_get_shmem_allocations_numa(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (PostgreSQLの関数呼び出し規約に従う引数)
  • **出力**: Datum (内部的には tuplestore を介して、共有メモリのキー名、NUMAノードID、およびそのノードに割り当てられたバイト数を表す行セットを返します)

前提条件と不変条件

  • pg_numa_init() が成功していること（NUMAがサポートされている環境であること）。
  • ShmemIndexLock を共有モードで取得して実行されます。
  • 共有メモリの各セグメントは、OSのページ境界に合わせて開始・終了アドレスが調整されます。

アルゴリズムまたは振る舞い

1. pg_numa_init を呼び出し、NUMA初期化を確認します。 2. ShmemIndex ハッシュテーブルを走査し、各共有メモリ割り当てエントリを取得します。 3. 各エントリの開始・終了アドレスをOSのページサイズ単位にアライメントします。 4. pg_numa_touch_mem_if_required を使用して、必要に応じてメモリページにアクセス（タッチ）し、NUMAノード情報を取得可能にします。 5. pg_numa_query_pages を呼び出し、各ページがどのNUMAノードに属するかを特定します。 6. 各NUMAノードごとの合計サイズを計算し、結果を tuplestore に格納します。 7. 最後に、NUMAノードに割り当てられなかった（またはスワップアウトされた）ページ分を別エントリとして出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます（直接的なC言語レベルの呼び出し元は提供されたソース内にはありません）。
  • **呼び出し先**: pg_numa_init, pg_numa_get_max_node, pg_get_shmem_pagesize, hash_seq_init, hash_seq_search, pg_numa_touch_mem_if_required, pg_numa_query_pages, tuplestore_putvalues 等。

関連データ構造

  • ShmemIndex: 共有メモリ上の全データ構造を管理するハッシュテーブル。
  • ShmemIndexEnt: 共有メモリ割り当ての各エントリ。
  • ReturnSetInfo: SRF（Set Returning Function）の結果を保持する構造体。

失敗モードと境界条件

  • pg_numa_init が失敗した場合、ERROR をスローします。
  • pg_numa_query_pages が失敗した場合、ERROR をスローします。
  • NUMAノードIDが想定範囲外（[0, max_nodes]）である場合、ERROR をスローします。
  • ページがスワップアウトされている場合（s == -2）、その分は「NUMAノードなし」として集計されます。

保守担当者向けメモ

  • 本関数は firstNumaTouch フラグを使用して、初回実行時にメモリページをタッチする処理を行います。
  • 共有メモリのサイズがOSページサイズと一致しない場合でも、ページ境界へのアライメント処理により正確な統計を取得するよう設計されています。
