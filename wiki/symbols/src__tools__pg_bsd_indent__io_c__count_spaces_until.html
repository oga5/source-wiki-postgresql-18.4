count_spaces_until

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/io.c:516

目的

指定されたバッファ内のテキストを、開始カラム位置から順次出力した際に到達する最終的なカラム位置を算出します。

シグネチャまたは形状

int count_spaces_until(int cur, char *buffer, char *end);

入力と出力

  • **入力**:
  • cur: 開始カラム位置（整数）。
  • buffer: 走査対象の文字列バッファ。
  • end: 走査を終了するポインタ（この位置に達するか、ヌル文字 \0 に達すると終了）。
  • **出力**:
  • 処理後の最終的なカラム位置を整数で返します。

前提条件と不変条件

  • buffer はヌル終端されているか、end ポインタによって範囲が制限されている必要があります。
  • tabsize グローバル変数がタブ幅の計算に使用されます。

アルゴリズムまたは振る舞い

バッファ内の各文字を順次走査し、文字種に応じてカラム位置 cur を更新します。

  • \n (改行) または 014 (フォームフィード): cur を 1 にリセットします。
  • \t (タブ): tabsize に基づき、次のタブストップ位置まで cur を進めます。
  • 010 (バックスペース): cur を 1 減らします。
  • その他: cur を 1 増やします。

    • 走査は *buf == '\0' または buf == end になるまで継続されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/tools/pg_bsd_indent/indent.c
  • src/tools/pg_bsd_indent/io.c
  • src/tools/pg_bsd_indent/pr_comment.c
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行わず、グローバル変数 tabsize を参照します。

関連データ構造

  • tabsize: インデント計算に使用されるグローバルなタブ幅設定。

失敗モードと境界条件

  • buffer が空の場合、入力された cur がそのまま返されます。
  • end が buffer より前を指している場合、ループは即座に終了し cur を返します。
  • tabsize が 0 以下の値に設定されている場合、タブ計算時に除算エラーが発生する可能性があります（ソースコード上の制約）。

保守担当者向けメモ

  • 本関数は indent.c や pr_comment.c において、コードやコメントの配置位置を決定するための重要なユーティリティです。
  • tabsize の値が正しく設定されていることを前提としているため、変更時はグローバル設定との整合性に注意してください。
