setup_parser_errposition_callback

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_node.c:139

目的

SQL解析中に発生したエラーに対し、ソーステキスト上の位置情報を付与するためのエラーコールバックを error_context_stack に登録します。

シグネチャまたは形状

void setup_parser_errposition_callback(ParseCallbackState *pcbstate, ParseState *pstate, int location)

入力と出力

  • **入力**:
  • pcbstate: 状態を保持するための ParseCallbackState 構造体へのポインタ。
  • pstate: 現在の解析状態を示す ParseState 構造体へのポインタ。
  • location: エラーが発生したソーステキスト内の位置（バイトオフセット）。
  • **出力**: なし（副作用として pcbstate の初期化とグローバルなエラーコンテキストスタックの更新を行う）。

前提条件と不変条件

  • pcbstate は呼び出し元によってメモリが確保されている必要があります。
  • error_context_stack はグローバルなエラー処理スタックであり、本関数は現在のコールバックをその先頭に挿入します。

アルゴリズムまたは振る舞い

1. 引数 pstate と location を pcbstate に格納します。 2. pcbstate->errcallback の callback フィールドに pcb_error_callback を設定します。 3. pcbstate->errcallback の arg に pcbstate 自身を設定します。 4. 現在の error_context_stack を pcbstate->errcallback.previous に退避させます。 5. error_context_stack を更新し、新しいコールバックをスタックの先頭に配置します。

呼び出し元と呼び出し先

  • **呼び出し元**: analyze.c, parse_clause.c, parse_coerce.c, parse_func.c, parse_node.c, parse_oper.c, parse_relation.c 等の各解析ルーチンから、エラー発生の可能性がある処理の直前に呼び出されます。
  • **呼び出し先**: pcb_error_callback（関数ポインタとして登録）。

関連データ構造

  • ParseCallbackState: エラー報告に必要なコンテキストを保持する構造体。
  • ParseState: 解析セッションの状態を保持する構造体。
  • error_context_stack: PostgreSQL のエラー報告メカニズムで使用されるグローバルスタック。

失敗モードと境界条件

  • location が負の値（エラー位置が不明な場合）であっても、コールバック自体は登録されます。
  • 呼び出し元で cancel_parser_errposition_callback を呼び出してスタックを復元しない場合、エラー発生時に不正なメモリ参照や誤ったエラー位置報告が発生する可能性があります。

保守担当者向けメモ

  • 本関数は ereport() が呼び出された際に、pcb_error_callback を介して parser_errposition を実行するための準備を行います。
  • 登録したコールバックは、解析処理の終了時に必ず cancel_parser_errposition_callback で解除する必要があります。
