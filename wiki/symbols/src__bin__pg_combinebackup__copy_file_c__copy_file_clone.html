copy_file_clone

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/copy_file.c:226

目的

OSのファイルクローン機能（Copy-on-Write）を利用して、ソースファイルをデスティネーションへ効率的に複製し、必要に応じてチェックサムを計算する。

シグネチャまたは形状

static void copy_file_clone(const char *src, const char *dest, pg_checksum_context *checksum_ctx);

入力と出力

  • src: コピー元ファイルのパス（入力）。
  • dest: コピー先ファイルのパス（入力）。
  • checksum_ctx: チェックサム計算用のコンテキスト（入力）。
  • 戻り値: なし（成功時）。失敗時は pg_fatal によりプロセスを終了する。

前提条件と不変条件

  • プラットフォームが HAVE_COPYFILE (macOS等) または Linux の FICLONE ioctl をサポートしている必要がある。
  • dest ファイルは存在しない状態であること（O_EXCL フラグによる排他作成）。

アルゴリズムまたは振る舞い

1. **プラットフォーム固有のクローン実行**:

  • HAVE_COPYFILE が定義されている場合: copyfile(src, dest, NULL, COPYFILE_CLONE_FORCE) を実行。
  • Linux の場合: src を読み取り専用、dest を書き込み・作成・排他モードで開き、ioctl(dest_fd, FICLONE, src_fd) を実行。

    • 3. **チェックサム計算**: クローン完了後、checksum_file(src, checksum_ctx) を呼び出し、ソースファイルの内容に基づいたチェックサムを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_combinebackup/copy_file.c 内の copy_file 関数（戦略選択時に strategy_implementation へ代入される）。
  • **呼び出し先**: checksum_file（チェックサム計算用）、およびOSシステムコール（open, ioctl, close, unlink）。

関連データ構造

  • pg_checksum_context: チェックサム計算の状態を保持する構造体。

失敗モードと境界条件

  • **プラットフォーム非対応**: サポートされていない環境では pg_fatal を発生させる。
  • **I/O エラー**: ファイルのオープン、クローン操作、またはクローズに失敗した場合、エラーメッセージと共に pg_fatal で終了する。
  • **排他制御**: dest ファイルが既に存在する場合、O_EXCL によりオープンが失敗し、エラーとなる。

保守担当者向けメモ

  • 本関数はコピー後に別途 checksum_file を呼び出すため、ブロック単位のコピー（copy_file_blocks）とは異なり、コピー完了後に再度ソースファイルを読み込む必要がある。
  • Linux環境での ioctl 失敗時には、不完全なファイルが残らないよう unlink を確実に行うこと。
