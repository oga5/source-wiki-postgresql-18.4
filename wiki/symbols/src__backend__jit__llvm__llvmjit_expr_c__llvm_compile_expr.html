llvm_compile_expr

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_expr.c:79

目的

PostgreSQL の式評価（ExprState）を LLVM IR に変換し、実行時にネイティブコードとして実行可能な関数を生成する。

シグネチャまたは形状

bool llvm_compile_expr(ExprState *state)

入力と出力

  • **入力**: ExprState *state（コンパイル対象の式状態）
  • **出力**: bool（コンパイルの成否。成功時に true）

前提条件と不変条件

  • state->parent が存在すること（EState へのアクセスが必要なため）。
  • llvm_enter_fatal_on_oom() が呼び出され、メモリ不足時のエラーハンドリングが設定されていること。

アルゴリズムまたは振る舞い

1. **コンテキスト準備**: parent->state->es_jit を確認し、存在しない場合は llvm_create_context で新規作成する。 2. **LLVM 環境構築**: llvm_mutable_module からモジュールを取得し、LLVMBuilderRef を初期化する。 3. **関数生成**: evalexpr という名前の関数をモジュールに追加し、ExecInterpExprStillValid と同等の型シグネチャを割り当てる。 4. **IR 構築**:

  • entry 基本ブロックを作成し、ビルダーを配置する。
  • ExprState から resvalue および resnull へのポインタを取得する。
  • ExprContext からスキャン、内部、外部、旧、新タプルスロットおよびそれらの値・NULL 配列へのポインタをロードする。
  • ExprState から結果スロットを取得する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/jit/llvm/llvmjit.c（cb->compile_expr への代入を通じて間接的に呼び出される）。
  • **呼び出し先**: llvm_enter_fatal_on_oom, llvm_create_context, llvm_mutable_module, LLVMCreateBuilderInContext, llvm_expand_funcname, LLVMAddFunction, llvm_pg_var_func_type, LLVMSetLinkage, LLVMSetVisibility, llvm_copy_attributes, LLVMAppendBasicBlockInContext, LLVMGetParam, LLVMPositionBuilderAtEnd, l_struct_gep, l_load_struct_gep。

関連データ構造

  • ExprState: コンパイル対象の式状態。
  • LLVMJitContext: JIT コンパイルの管理コンテキスト。
  • LLVMBuilderRef: LLVM IR 生成用ビルダー。
  • LLVMModuleRef: 生成されるコードを保持するモジュール。

失敗モードと境界条件

  • parent が NULL の場合、Assert により停止する。
  • メモリ不足時は llvm_enter_fatal_on_oom により致命的エラーとして処理される。

保守担当者向けメモ

  • 本関数は巨大な switch 文の入り口であり、以降の処理で各 ExprEvalStep を順次 LLVM IR に変換する。
  • l_ プレフィックスを持つ関数は llvmjit_emit.h で定義されたラッパーであり、LLVM API の冗長な呼び出しを隠蔽している。
  • 生成された IR は llvmjit.c 側の設定によりダンプ可能である。
