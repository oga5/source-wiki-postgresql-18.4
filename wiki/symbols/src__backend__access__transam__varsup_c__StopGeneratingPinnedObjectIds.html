StopGeneratingPinnedObjectIds

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/varsup.c:651

目的

システムが生成するオブジェクトID（OID）の範囲を、固定された（Pinned）OIDの生成を終了し、通常のOID生成シーケンスへ切り替えるために使用されます。

シグネチャまたは形状

void StopGeneratingPinnedObjectIds(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上の次OID状態を更新）

前提条件と不変条件

  • SetNextObjectId を介して、OID生成の起点が FirstUnpinnedObjectId に設定されます。

アルゴリズムまたは振る舞い

1. SetNextObjectId 関数を呼び出します。 2. 引数として FirstUnpinnedObjectId を渡し、次に払い出されるOIDを固定値以降の範囲にリセットします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/catalog.c 内の関数から呼び出されることが確認されています。
  • 呼び出し先: SetNextObjectId を呼び出します。

関連データ構造

  • FirstUnpinnedObjectId: 固定OIDと通常OIDの境界を示す定数。
  • TransamVariables: 共有メモリ上でOIDの生成状態を保持する構造体（SetNextObjectId 経由で更新）。

失敗モードと境界条件

  • 共有メモリへのアクセスを伴うため、適切なロック（OidGenLock）が SetNextObjectId 内部で管理されている必要があります。

保守担当者向けメモ

  • 本関数は、システムカタログの初期化や特定のメタデータ操作の完了時に、OIDの払い出し範囲を制御するために使用されます。
  • src/include/access/transam.h に外部公開用の宣言が存在します。
