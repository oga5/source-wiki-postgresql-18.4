MJFillInner

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMergejoin.c:477

目的

マージ結合において、外部リレーション（Outer）のタプルが結合条件に一致しなかった場合に、NULL補完された内部リレーション（Inner）のタプルを生成・返却する。

シグネチャまたは形状

static TupleTableSlot *MJFillInner(MergeJoinState *node)

入力と出力

  • **入力**: MergeJoinState *node （マージ結合の実行状態を保持する構造体）
  • **出力**: 結合条件を満たした場合は投影（Projection）後の TupleTableSlot * を返し、満たさない場合は NULL を返す。

前提条件と不変条件

  • node->js.ps.ps_ExprContext が初期化されていること。
  • node->mj_NullOuterTupleSlot にNULL補完用のタプルが設定されていること。
  • node->mj_InnerTupleSlot に現在の内部タプルが保持されていること。

アルゴリズムまたは振る舞い

1. ResetExprContext を呼び出し、式評価用のコンテキストをリセットする。 2. 式コンテキストの ecxt_outertuple にNULL補完用スロットを、ecxt_innertuple に現在の内部タプルスロットを割り当てる。 3. ExecQual を用いて結合条件（otherqual）を評価する。 4. 条件が真であれば、ExecProject を実行して投影後のタプルを返却する。 5. 条件が偽であれば、InstrCountFiltered2 を呼び出してフィルタリングされた行数をカウントし、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeMergejoin.c 内の複数箇所（行 726, 861, 1286, 1356）。
  • **呼び出し先**:
  • ResetExprContext
  • ExecQual
  • ExecProject
  • InstrCountFiltered2
  • MJ_printf （デバッグ用マクロ）

関連データ構造

  • MergeJoinState: マージ結合の実行状態を管理する主要構造体。
  • TupleTableSlot: タプルを保持するスロット。
  • ExprContext: 式評価のための実行コンテキスト。

失敗モードと境界条件

  • otherqual がNULLを評価した場合、結合条件は不一致とみなされ、NULL が返される。
  • 投影（Projection）が失敗した場合の挙動は、ExecProject の実装に依存する。

保守担当者向けメモ

  • 本関数は右外部結合や完全外部結合のNULL補完ロジックの一部として機能する。
  • MJ_dump マクロが有効な場合、MJ_printf を通じて実行トレースが出力される。
