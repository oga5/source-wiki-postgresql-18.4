cleanup_output_dirs

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/util.c:62

目的

プログラム終了時に、pg_upgrade が生成した一時ディレクトリやログファイルを削除し、リソースを解放する。

シグネチャまたは形状

void cleanup_output_dirs(void)

入力と出力

  • 入力: なし（グローバル変数 log_opts を参照）
  • 出力: なし（副作用としてファイルシステム上のディレクトリ削除およびログファイルのクローズを行う）

前提条件と不変条件

  • log_opts.internal が有効なファイルポインタとして開かれていること。
  • log_opts.basedir および log_opts.rootdir が削除対象のディレクトリパスとして適切に設定されていること。

アルゴリズムまたは振る舞い

1. log_opts.internal をクローズする。 2. log_opts.retain が真であれば、削除処理をスキップして終了する。 3. rmtree を使用して log_opts.basedir を再帰的に削除する。Windows環境での削除失敗を考慮し、失敗した場合は再試行する。 4. pg_check_dir を用いて log_opts.rootdir の状態を確認する。

  • ディレクトリが空、またはドットファイルのみの場合は rmtree で削除する（同様に再試行ロジックあり）。
  • ディレクトリが存在し、かつ空でない場合は、過去のログ活動が含まれていると判断して削除を保留する。
  • アクセス不可等のエラー時は pg_log で警告を出力する。

関連データ構造

  • LogOpts (グローバル変数 log_opts): ログ出力設定およびディレクトリパスを保持する構造体。

失敗モードと境界条件

  • rmtree が失敗した場合、Windows環境でのファイルロック等を想定して一度だけ再試行を行う。
  • pg_check_dir が予期しない値を返した場合、pg_log を通じて警告メッセージを出力する。
  • log_opts.rootdir が空でない場合、意図的に削除を行わない設計となっている。

保守担当者向けメモ

  • 本関数は pg_upgrade の終了処理の要であり、ファイルシステム操作を伴うため、Windows環境での挙動（ファイルロックによる削除失敗）に対する堅牢性が考慮されている。
  • rmtree の再試行ロジックは、プラットフォーム固有の制約を回避するための重要な実装である。
