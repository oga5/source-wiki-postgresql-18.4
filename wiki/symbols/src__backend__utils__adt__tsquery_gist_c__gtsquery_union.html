gtsquery_union

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_gist.c:88

目的

GiST インデックスの構築時、複数の tsquery シグネチャを統合し、親ノードの代表的なシグネチャ（ビットマップ）を生成する。

シグネチャまたは形状

Datum gtsquery_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GistEntryVector *entryvec（統合対象のインデックスエントリ群）。
  • PG_GETARG_POINTER(1): int *size（結果のサイズを格納する出力用ポインタ）。
  • **出力**:
  • Datum: 統合された TSQuerySign（ビット単位の論理和）。
  • *size: sizeof(TSQuerySign) が設定される。

前提条件と不変条件

  • entryvec は有効な GistEntryVector 構造体を指している必要がある。
  • TSQuerySign は固定長のビットマップ型である。

アルゴリズムまたは振る舞い

1. 初期値 sign を 0 に設定する。 2. entryvec->n 個の全エントリをループで走査する。 3. 各エントリのシグネチャを GETENTRY マクロで取得し、現在の sign に対してビット単位の論理和（|=）を適用する。 4. 最終的な sign を返し、size に TSQuerySign のサイズを格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: GiST インデックスの構築または更新プロセス（PostgreSQL の GiST インデックス管理機構）。
  • **呼び出し先**: GETENTRY マクロ（entryvec からエントリを取得）。
  • 注: ソースコードの文脈上、具体的な呼び出し元の関数名は明示されていない。

関連データ構造

  • GistEntryVector: GiST インデックスのノードに含まれるエントリの集合。
  • TSQuerySign: tsquery を圧縮したビットマップ表現。

失敗モードと境界条件

  • entryvec->n が 0 の場合、sign は 0 となる。
  • entryvec が NULL の場合、プログラムは異常終了する可能性がある（防御的チェックは含まれていない）。

保守担当者向けメモ

  • 本関数は GiST インデックスの union メソッドとして機能する。
  • 常に TSQuerySign 型の固定サイズを返すため、インデックスの構造は静的である。
