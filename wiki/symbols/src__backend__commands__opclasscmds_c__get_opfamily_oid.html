get_opfamily_oid

定義

  • 種別: function
  • 定義箇所: src/backend/commands/opclasscmds.c:138

目的

指定されたアクセスメソッド（AM）と演算子ファミリー名に基づき、システムカタログから対応する演算子ファミリーのOIDを取得します。

シグネチャまたは形状

Oid get_opfamily_oid(Oid amID, List *opfamilyname, bool missing_ok)

入力と出力

  • **入力**:
  • amID: 演算子ファミリーが属するアクセスメソッドのOID。
  • opfamilyname: 演算子ファミリー名を指定するリスト（スキーマ修飾名を含む）。
  • missing_ok: true の場合、対象が見つからない場合にエラーを発生させず InvalidOid を返します。
  • **出力**:
  • 成功時は演算子ファミリーのOID、見つからない場合は InvalidOid を返します。

前提条件と不変条件

  • OpFamilyCacheLookup を介してシステムキャッシュにアクセスするため、適切なトランザクションコンテキストが必要です。
  • 取得したタプルは関数内で ReleaseSysCache により解放されます。

アルゴリズムまたは振る舞い

1. OpFamilyCacheLookup を呼び出し、指定されたAMと名前で演算子ファミリーのシステムカタログタプルを検索します。 2. タプルが無効（!HeapTupleIsValid）であれば、InvalidOid を返して終了します。 3. 有効なタプルが取得できた場合、GETSTRUCT マクロを使用して Form_pg_opfamily 構造体へキャストします。 4. 構造体から演算子ファミリーのOID（oid）を抽出します。 5. ReleaseSysCache を呼び出してタプルの参照を解放します。 6. 抽出したOIDを呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • OpFamilyCacheLookup
  • ReleaseSysCache
  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c
  • src/backend/commands/opclasscmds.c（複数箇所）

関連データ構造

  • Form_pg_opfamily: pg_opfamily システムカタログのタプル構造体。
  • HeapTuple: システムキャッシュから返されるタプル表現。

失敗モードと境界条件

  • missing_ok が false の場合、OpFamilyCacheLookup 内部で対象が見つからないとエラーが発生する可能性があります。
  • amID が無効な場合、検索は失敗します。

保守担当者向けメモ

  • 本関数は名前解決のラッパーであり、実際の検索ロジックは OpFamilyCacheLookup に依存しています。
  • 演算子ファミリーの定義変更や検索ロジックの修正を行う際は、pg_opfamily カタログの定義と整合性を確認してください。
