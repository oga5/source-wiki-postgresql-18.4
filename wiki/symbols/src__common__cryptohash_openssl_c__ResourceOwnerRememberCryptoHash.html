ResourceOwnerRememberCryptoHash

定義

  • 種別: static inline void
  • 定義箇所: src/common/cryptohash_openssl.c:90

目的

PostgreSQLのバックエンド環境において、暗号学的ハッシュコンテキスト（pg_cryptohash_ctx）を ResourceOwner に登録し、トランザクション終了時やエラー発生時に自動的に解放されるよう管理する。

シグネチャまたは形状

void ResourceOwnerRememberCryptoHash(ResourceOwner owner, pg_cryptohash_ctx *ctx)

入力と出力

  • owner: 登録先の ResourceOwner オブジェクト。
  • ctx: 管理対象となる pg_cryptohash_ctx 構造体へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • 本関数はバックエンド環境でのみ使用される。
  • ctx は有効な pg_cryptohash_ctx を指している必要がある。
  • cryptohash_resowner_desc がリソース管理用の記述子として定義されている必要がある。

アルゴリズムまたは振る舞い

ResourceOwnerRemember 関数を呼び出し、ctx を PointerGetDatum で変換して ResourceOwner に登録する。これにより、リソース所有者が解放される際に、関連付けられたハッシュコンテキストが追跡対象となる。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/cryptohash_openssl.c 内のコンテキスト作成処理（pg_cryptohash_create 等）から呼び出される。
  • 呼び出し先: ResourceOwnerRemember（PostgreSQLコアのメモリ管理API）。

関連データ構造

  • pg_cryptohash_ctx: ハッシュ計算の状態を保持する構造体。
  • ResourceOwner: リソースのライフサイクルを管理するバックエンドのオブジェクト。
  • cryptohash_resowner_desc: リソース所有者管理用のリソース記述子。

失敗モードと境界条件

  • owner が無効な場合、ResourceOwnerRemember の仕様に従いエラーが発生する可能性がある。
  • メモリ不足等の理由で ResourceOwner への登録に失敗した場合、PostgreSQLの標準的なエラーハンドリング機構により例外がスローされる。

保守担当者向けメモ

  • 本関数は static inline で定義されており、主に pg_cryptohash_create からの呼び出しを想定している。
  • コンテキストの解放が必要な場合は、対となる ResourceOwnerForgetCryptoHash を使用すること。
