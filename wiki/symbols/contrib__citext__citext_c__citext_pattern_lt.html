citext_pattern_lt

定義

  • 種別: function
  • 定義箇所: contrib/citext/citext.c:316

目的

citext 型の2つの値に対し、照合順序（Collation）を無視したバイナリ比較を行い、左側の値が右側の値より小さいかどうかを判定する。

シグネチャまたは形状

Datum citext_pattern_lt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの text 型データ（citext としてキャストされたもの）を受け取る。
  • **出力**: PG_RETURN_BOOL を介して、左側が右側より小さい場合に true、それ以外の場合に false を返す。

前提条件と不変条件

  • 入力された text 型データは、PG_GETARG_TEXT_PP を使用して適切に展開される必要がある。
  • 処理終了時には PG_FREE_IF_COPY を使用して、コピーされた入力データのメモリを解放し、メモリリークを防ぐ必要がある。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を使用して、第1引数と第2引数を取得する。 2. internal_citext_pattern_cmp を呼び出し、照合順序を無視した比較を実行する。 3. 比較結果が 0 未満（cmp < 0）であるかを判定し、結果を保持する。 4. PG_FREE_IF_COPY を使用して、入力引数のメモリを解放する。 5. 判定結果を PG_RETURN_BOOL で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(citext_pattern_lt) マクロにより登録されている。直接的な関数呼び出し元はソースコード上では明示されていない。
  • **呼び出し先**: internal_citext_pattern_cmp を呼び出す。

関連データ構造

  • text: PostgreSQL の可変長文字列型。citext 型の基盤として使用される。

失敗モードと境界条件

  • 入力が NULL の場合、PostgreSQL の関数呼び出し規約に従い、呼び出し元で適切に処理される必要がある（本関数内ではチェックされていない）。
  • internal_citext_pattern_cmp の戻り値に依存する。

保守担当者向けメモ

  • 本関数はパターンマッチング用の比較演算子として実装されており、通常の citext_lt と異なり、照合順序の影響を受けないバイナリ比較を行う点に注意すること。
