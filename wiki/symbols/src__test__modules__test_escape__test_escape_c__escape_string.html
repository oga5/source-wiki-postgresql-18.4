escape_string

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_escape/test_escape.c:326

目的

指定された文字列をSQL文字列リテラルとしてエスケープし、PQExpBuffer バッファへ追加する。

シグネチャまたは形状

static bool escape_string(PGconn *conn, PQExpBuffer target, const char *unescaped, size_t unescaped_len, PQExpBuffer escape_err)

入力と出力

  • **入力**:
  • conn: PostgreSQL接続ハンドル（本関数内では未使用）。
  • target: エスケープ結果を格納する PQExpBuffer。
  • unescaped: エスケープ対象の文字列。
  • unescaped_len: unescaped の長さ。
  • escape_err: エラー情報を格納する PQExpBuffer（本関数内では未使用）。
  • **出力**:
  • 常に true を返す。target にエスケープされた文字列が追記される。

前提条件と不変条件

  • target は初期化済みの PQExpBuffer であること。
  • unescaped は少なくとも unescaped_len バイトの有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. target の末尾に開始クォート ' を追加する。 2. unescaped_len * 2 + 1 分の領域を target に確保する。 3. PQescapeString を呼び出し、unescaped を target の現在の末尾位置にエスケープして書き込む。 4. 書き込まれたバイト数分だけ target->len を進める。 5. target の末尾に終了クォート ' を追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_escape/test_escape.c 内の pe_test_escape_func 構造体で参照されている。
  • **呼び出し先**: appendPQExpBufferChar, enlargePQExpBuffer, PQescapeString。
  • **注意**: src/bin/psql/tab-complete.in.c に同名の関数が存在するが、シグネチャが異なるため、本関数とは別の実装である。

関連データ構造

  • PQExpBuffer: 文字列バッファ管理用構造体。
  • PGconn: PostgreSQL接続用構造体。

失敗モードと境界条件

  • target のメモリ確保に失敗した場合、enlargePQExpBuffer 内部でプロセスが終了する可能性がある。
  • unescaped_len が極端に大きい場合、バッファ拡張に伴うメモリ不足が発生し得る。

保守担当者向けメモ

  • 本関数はテストモジュール専用の実装であり、conn や escape_err 引数を利用していない。実運用コードの escape_string と混同しないよう注意すること。
