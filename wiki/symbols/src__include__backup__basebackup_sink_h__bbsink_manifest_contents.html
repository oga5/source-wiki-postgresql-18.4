bbsink_manifest_contents

定義

  • 種別: function
  • 定義箇所: src/include/backup/basebackup_sink.h:233

目的

バックアップマニフェストのデータを、シンクチェーン内の現在のシンクから次段のシンクへ転送するために使用されます。

シグネチャまたは形状

static inline void bbsink_manifest_contents(bbsink *sink, size_t len)

入力と出力

  • sink: 操作対象となる bbsink 構造体へのポインタ。
  • len: 転送するデータのバイト数。
  • 戻り値: なし。

前提条件と不変条件

  • sink は NULL であってはならない。
  • len は 0 より大きく、かつ sink->bbs_buffer_length 以下である必要がある。

アルゴリズムまたは振る舞い

1. sink が NULL でないことを確認する。 2. len が有効な範囲内（0 < len <= bbs_buffer_length）であることを検証する。 3. sink->bbs_ops->manifest_contents コールバックを呼び出し、実際のデータ転送処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/backup_manifest.c
  • src/backend/backup/basebackup_gzip.c
  • src/backend/backup/basebackup_lz4.c
  • src/backend/backup/basebackup_sink.c
  • src/backend/backup/basebackup_zstd.c
  • **呼び出し先**:
  • sink->bbs_ops->manifest_contents（bbsink_ops 構造体で定義された関数ポインタ）

関連データ構造

  • bbsink: シンクの基本構造体。
  • bbsink_ops: シンクの挙動を定義する関数テーブル。

失敗モードと境界条件

  • sink が NULL の場合、または len が 0 以下、あるいはバッファ長を超える場合は Assert により停止する。

保守担当者向けメモ

  • 本関数はインライン関数であり、各シンクの実装における manifest_contents コールバックへのディスパッチを担います。
  • 処理の委譲を行う際は、bbsink_forward_manifest_contents 等のユーティリティ関数と混同しないよう注意してください。
