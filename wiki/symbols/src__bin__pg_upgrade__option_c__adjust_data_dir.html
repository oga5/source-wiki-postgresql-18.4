adjust_data_dir

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/option.c:429

目的

設定ファイルのみが配置されたディレクトリ（config-only）から、実際のデータディレクトリパスを特定し、ClusterInfo 構造体を更新する。

シグネチャまたは形状

void adjust_data_dir(ClusterInfo *cluster)

入力と出力

  • **入力**: ClusterInfo *cluster（対象クラスタの情報を保持する構造体へのポインタ）
  • **出力**: なし（副作用として cluster->pgdata を更新する）

前提条件と不変条件

  • cluster->pgdata には初期値として設定ディレクトリのパスが格納されていること。
  • cluster->bindir に有効なバイナリパスが設定されていること。
  • 対象ディレクトリが config-only である場合、postgres コマンドが実行可能であること。

アルゴリズムまたは振る舞い

1. cluster->pgconfig に pgdata の値をコピーする。 2. postgresql.conf が存在しない場合、または PG_VERSION が存在する場合は、config-only ディレクトリではないと判断し終了する。 3. config-only ディレクトリと判定された場合、postgres -D [pgconfig] -C data_directory コマンドを実行する。 4. コマンドの標準出力から実際のデータディレクトリパスを取得し、末尾の改行コードを除去する。 5. 取得したパスで cluster->pgdata を上書き更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/pg_upgrade.c: old_cluster および new_cluster に対して呼び出される。
  • **呼び出し先**:
  • popen, fgets, pclose: 外部コマンド実行用。
  • pg_fatal: エラー発生時の強制終了用。
  • pg_strip_crlf: 文字列の改行除去用。
  • check_ok: 成功時のステータス表示用。

関連データ構造

  • ClusterInfo: pg_upgrade.h で定義。pgdata（データディレクトリ）、pgconfig（設定ディレクトリ）、bindir（バイナリパス）を保持する。

失敗モードと境界条件

  • postgresql.conf が読み取れない、または PG_VERSION が存在する場合は処理をスキップする。
  • postgres コマンドの実行に失敗した場合、またはコマンドがデータディレクトリを返さない場合は pg_fatal によりプログラムが終了する。
  • 本関数は PostgreSQL 9.2 以降のバイナリでのみ動作する。

保守担当者向けメモ

  • src/bin/pg_ctl/pg_ctl.c にも同名の関数が存在するが、本ページは pg_upgrade 用の定義を対象としている。
  • pg_upgrade 実行時にデータディレクトリが特定できない場合、この関数が失敗することで処理が中断される。
