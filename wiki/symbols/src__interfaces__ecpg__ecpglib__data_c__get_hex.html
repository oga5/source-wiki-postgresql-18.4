get_hex

定義

  • 種別: static inline 関数
  • 定義箇所: src/interfaces/ecpg/ecpglib/data.c:139

目的

16進数文字（'0'-'9', 'a'-'f', 'A'-'F'）を対応する数値（0-15）に変換します。

シグネチャまたは形状

static inline char get_hex(char c)

入力と出力

  • **入力**: char c（変換対象の文字）
  • **出力**: char 型として返される数値（0〜15）。変換不能な文字の場合は -1 を返します。

前提条件と不変条件

  • 入力文字 c が 0 より大きく 127 未満（ASCII範囲内）であることを前提としています。
  • 内部の hexlookup テーブルは、128要素の int8 配列であり、有効な16進数文字以外には -1 が格納されています。

アルゴリズムまたは振る舞い

1. 128要素の静的ルックアップテーブル hexlookup を参照します。 2. 入力文字 c が有効な範囲内であれば、テーブルから対応する値をインデックスとして取得します。 3. 範囲外または無効な文字の場合は -1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル src/interfaces/ecpg/ecpglib/data.c 内の hex_decode（推定）等の関数から呼び出されています。
  • **呼び出し先**: なし（純粋な変換関数）。
  • 注: ツールにより src/interfaces/libpq/fe-exec.c 等の他ファイルでも同名の関数が検出されていますが、これらは別定義の可能性が高いです。

関連データ構造

  • static const int8 hexlookup[128]: 変換用ルックアップテーブル。

失敗モードと境界条件

  • 入力文字が16進数として無効な場合、-1 を返します。
  • ASCII範囲外の文字が入力された場合、テーブル参照を行わず -1 を返します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、コンパイル時に展開されることを意図しています。
  • hexlookup テーブルのインデックスは unsigned char にキャストして安全にアクセスしています。変更時はテーブルサイズとインデックス範囲の整合性に注意してください。
