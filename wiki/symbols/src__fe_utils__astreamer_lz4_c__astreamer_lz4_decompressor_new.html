astreamer_lz4_decompressor_new

定義

  • **種別**: 関数
  • **定義箇所**: src/fe_utils/astreamer_lz4.c:276

目的

LZ4形式で圧縮されたデータストリームを伸長するための astreamer インスタンスを生成・初期化します。

シグネチャまたは形状

astreamer *astreamer_lz4_decompressor_new(astreamer *next);

入力と出力

  • **入力**: next (後続の処理を行う astreamer オブジェクトへのポインタ)
  • **出力**: 伸長処理用に設定された astreamer オブジェクトへのポインタ。失敗時は pg_fatal によりプロセスを終了します。

前提条件と不変条件

  • next は NULL であってはなりません。
  • ビルド時に USE_LZ4 が定義されている必要があります（未定義の場合は実行時にエラーとなります）。

アルゴリズムまたは振る舞い

1. astreamer_lz4_frame 構造体用のメモリを確保します。 2. bbs_ops に伸長用の操作関数群 (astreamer_lz4_decompressor_ops) を割り当てます。 3. bbs_next に引数の next を設定し、内部バッファ (bbs_buffer) を初期化します。 4. LZ4F_createDecompressionContext を呼び出し、LZ4伸長コンテキストを作成します。 5. コンテキスト作成に失敗した場合は pg_fatal でエラーを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c:1270
  • src/bin/pg_verifybackup/pg_verifybackup.c:1310
  • **呼び出し先**:
  • palloc0 (メモリ確保)
  • initStringInfo (バッファ初期化)
  • LZ4F_createDecompressionContext (LZ4ライブラリ関数)
  • LZ4F_isError / LZ4F_getErrorName (エラー判定・報告)
  • pg_fatal (エラーハンドリング)

関連データ構造

  • astreamer_lz4_frame: 伸長状態を保持する内部構造体。
  • astreamer: ストリーマーの基底インターフェース。

失敗モードと境界条件

  • USE_LZ4 が定義されていない環境で呼び出された場合、pg_fatal により即座に終了します。
  • LZ4F_createDecompressionContext がエラーを返した場合、pg_fatal によりエラーメッセージを出力して終了します。

保守担当者向けメモ

  • 本関数は astreamer パイプラインの伸長ステップを構築します。
  • 伸長コンテキストの解放は、後続の astreamer_lz4_decompressor_free で行われる設計となっています。
