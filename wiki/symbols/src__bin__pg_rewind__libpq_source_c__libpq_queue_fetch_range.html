libpq_queue_fetch_range

定義

  • **種別**: 関数
  • **場所**: src/bin/pg_rewind/libpq_source.c:354

目的

リモートサーバーから取得するファイル範囲のリクエストをキューに蓄積し、必要に応じてバッチ処理（process_queued_fetch_requests）をトリガーすることで、通信オーバーヘッドを削減します。

シグネチャまたは形状

static void libpq_queue_fetch_range(rewind_source *source, const char *path, off_t off, size_t len)

入力と出力

  • **入力**:
  • source: libpq_source 構造体へのポインタ（rewind_source としてキャスト）。
  • path: 取得対象のファイルパス。
  • off: 取得を開始するオフセット。
  • len: 取得するデータの長さ。
  • **出力**: なし（副作用として src->request_queue にリクエストが追加される）。

前提条件と不変条件

  • source は libpq_source 型として正しく初期化されている必要があります。
  • path 引数は、同一ファイルに対しては常に同じポインタ値が渡されることが期待されています（ポインタ比較による最適化のため）。
  • MAX_CHUNK_SIZE を超えるリクエストは、自動的に分割されます。

アルゴリズムまたは振る舞い

1. **マージ処理**: 直前のリクエストとパスが一致し、かつオフセットが連続している場合、MAX_CHUNK_SIZE に達するまで既存のリクエストを拡張します。 2. **キューイング**: 残りのリクエスト範囲を MAX_CHUNK_SIZE 単位に分割し、src->request_queue に順次追加します。 3. **バッチ実行**: キュー内のリクエスト数が MAX_CHUNKS_PER_QUERY に達した場合、process_queued_fetch_requests を呼び出して蓄積されたリクエストを一括実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_rewind/libpq_source.c 内の libpq_queue_fetch_file などから呼び出されます。
  • libpq_source 構造体の関数ポインタ queue_fetch_range に割り当てられています。
  • **呼び出し先**:
  • process_queued_fetch_requests: キューが満杯になった際に実行されます。

関連データ構造

  • libpq_source: リクエストキュー (request_queue) と現在のリクエスト数 (num_requests) を保持する構造体。
  • fetch_range_request: 個別のファイル取得リクエストを定義する構造体。

失敗モードと境界条件

  • **キューの制限**: MAX_CHUNKS_PER_QUERY に達すると自動的にフラッシュされるため、メモリ不足やバッファオーバーフローを防ぎます。
  • **パスの不一致**: path ポインタが一致しない場合、マージ処理はスキップされますが、正当性は維持されます。

保守担当者向けメモ

  • path の比較にポインタ等価性を使用しているため、呼び出し側でファイルパスの文字列ポインタが安定していることを前提としています。
  • MAX_CHUNK_SIZE および MAX_CHUNKS_PER_QUERY の定数は、通信パフォーマンスに直接影響するため、調整時はスループットへの影響を考慮してください。
