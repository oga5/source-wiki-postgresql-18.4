appendStringLiteral

定義

  • 種別: function
  • 定義箇所: src/fe_utils/string_utils.c:350

目的

入力文字列をSQLリテラル形式（シングルクォートで囲まれた形式）に変換し、指定された PQExpBuffer に追加します。

シグネチャまたは形状

void appendStringLiteral(PQExpBuffer buf, const char *str, int encoding, bool std_strings)

入力と出力

  • **入力**:
  • buf: 書き込み先の PQExpBuffer。
  • str: リテラル化する対象の文字列。
  • encoding: 文字列のエンコーディング（pg_wchar 関連）。
  • std_strings: 標準文字列構文を使用するかどうか（true の場合、バックスラッシュのエスケープを抑制）。
  • **出力**: なし（buf の内容が更新されます）。

前提条件と不変条件

  • buf は初期化済みである必要があります。
  • enlargePQExpBuffer が失敗した場合、関数は何もせず終了します。
  • 内部バッファは 2 * length + 2 のサイズを確保し、オーバーフローを防ぎます。

アルゴリズムまたは振る舞い

1. buf の容量を 2 * length + 2 に拡張します。 2. バッファの先頭に ' を書き込みます。 3. 文字列を1文字ずつ走査します。

  • **ASCII文字**: IS_HIGHBIT_SET が偽の場合、SQL_STR_DOUBLE に基づいて必要に応じてクォートを重ね、バッファに追加します。
  • **マルチバイト文字**: PQmblen で長さを判定します。無効なマルチバイトシーケンスが検出された場合、pg_encoding_set_invalid を使用してサーバー側でエラーとなる無効なバイト列に置換します。

    • 4. 最後に ' を書き込み、\0 で終端し、buf->len を更新します。

関連データ構造

  • PQExpBuffer: 文字列バッファ管理構造体。

失敗モードと境界条件

  • enlargePQExpBuffer がメモリ不足で失敗した場合、処理は中断されます。
  • 不正なマルチバイト文字が含まれる場合、サーバー側でエラーを誘発する形式に置換されます。

保守担当者向けメモ

  • 本関数は std_strings 引数により、バックスラッシュの扱いが変化します。変更を加える際は、SQL標準との互換性に注意してください。
  • マルチバイト文字の検証は、セキュリティ上の脆弱性（クォート文字のスキップ等）を防ぐために重要です。
