contsel

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_selfuncs.c:85

目的

幾何演算子（包含関係）に対するクエリの選択率（Selectivity）を推定し、プランナに提供する。

シグネチャまたは形状

Datum contsel(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（クエリの条件句に関する情報を含むが、本関数内では参照されない）。
  • **出力**: float8 型の固定値 0.001 を PG_RETURN_FLOAT8 マクロ経由で返却する。

前提条件と不変条件

  • 入力されたクエリの統計情報やデータ分布は一切参照されない。
  • 常に固定値 0.001 を返す。

アルゴリズムまたは振る舞い

クエリプランナが幾何インデックス（GiST等）の利用を検討する際、包含演算子に対するコスト見積もりの根拠として、ハードコードされた固定値 0.001 を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのクエリプランナ（直接的な呼び出し元情報はソースコード上では明示されていない）。
  • **呼び出し先**: なし。

関連データ構造

  • Datum: PostgreSQLの汎用データ型表現。
  • PG_FUNCTION_ARGS: 関数呼び出し規約のためのマクロ。

失敗モードと境界条件

  • 統計情報に基づいた動的な推定を行わないため、実際のデータ分布と乖離していてもエラーにはならず、常に一定の値を返す。

保守担当者向けメモ

  • 現在の実装は統計情報を使用しない「完全にデタラメ（totally bogus）」な値である。
  • 幾何学的なデータ分布を考慮した真の選択率推定アルゴリズムが実装されるまでの暫定的な処置である。
  • 幾何インデックスが利用可能な場合にプランナがそれを選択するように、十分に小さい値を提示することが目的である。
