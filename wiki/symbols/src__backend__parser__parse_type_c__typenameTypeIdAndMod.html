typenameTypeIdAndMod

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_type.c:309

目的

TypeName 構造体から、対応するシステムカタログ上の型 OID と、検証済みの型修飾子（typmod）を同時に取得する。

シグネチャまたは形状

void typenameTypeIdAndMod(ParseState *pstate, const TypeName *typeName, Oid *typeid_p, int32 *typmod_p)

入力と出力

  • **入力**:
  • pstate: パーサーの状態（エラー位置報告に使用）。
  • typeName: 解決対象の型名構造体。
  • **出力**:
  • typeid_p: 解決された型の OID が格納されるポインタ。
  • typmod_p: 解決された型修飾子が格納されるポインタ。

前提条件と不変条件

  • typeName は有効な構造体であること。
  • 呼び出し元は、typeid_p および typmod_p が指すメモリ領域を確保しておく必要がある。

アルゴリズムまたは振る舞い

1. typenameType を呼び出し、型名から Type（HeapTuple）を取得し、同時に typmod_p に型修飾子を書き込む。 2. 取得した HeapTuple から GETSTRUCT マクロを用いて Form_pg_type 構造体へアクセスし、oid を抽出して typeid_p に代入する。 3. ReleaseSysCache を呼び出し、取得した Type キャッシュエントリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c (テーブル定義や列定義の解析)
  • src/backend/parser/parse_clause.c
  • src/backend/parser/parse_expr.c (式解析)
  • src/backend/parser/parse_jsontable.c
  • **呼び出し先**:
  • typenameType (内部関数)
  • ReleaseSysCache (システムキャッシュ解放)

関連データ構造

  • TypeName: SQL 文中の型指定を保持するノード。
  • ParseState: 構文解析中の状態管理構造体。
  • Form_pg_type: pg_type システムカタログの行構造。

失敗モードと境界条件

  • 型名が見つからない場合や、型修飾子の検証（typmodin）でエラーが発生した場合は、内部でエラーがスローされ、処理が中断される。

保守担当者向けメモ

  • 本関数は Type キャッシュを内部で取得・解放するため、呼び出し側で ReleaseSysCache を行う必要はない。
  • pstate はエラー発生時のコンテキスト提供に使用されるため、可能な限り渡すことが推奨される。
