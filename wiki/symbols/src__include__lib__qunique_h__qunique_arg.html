qunique_arg

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/lib/qunique.h:45

目的

ソート済みの配列に対し、ユーザー定義の比較関数と任意の引数（arg）を用いてインプレースで重複要素を削除し、ユニークな要素のみを残す。

シグネチャまたは形状

size_t qunique_arg(void *array, size_t elements, size_t width,
                   int (*compare) (const void *, const void *, void *),
                   void *arg);

入力と出力

  • **入力**:
  • array: 操作対象の配列へのポインタ。
  • elements: 配列の要素数。
  • width: 各要素のバイトサイズ。
  • compare: 2つの要素を比較し、等しければ0を返す関数ポインタ。
  • arg: 比較関数に渡されるユーザー定義データ。
  • **出力**:
  • 重複排除後のユニークな要素数（size_t）。

前提条件と不変条件

  • 入力配列は、指定された compare 関数に基づいて事前にソートされている必要がある。
  • 配列はインプレースで書き換えられるため、元のデータが不要であることを呼び出し側で保証すること。

アルゴリズムまたは振る舞い

1. elements が 1 以下の場合は、そのまま elements を返す。 2. 配列を先頭から走査し、直前のユニークな要素（インデックス j）と現在の要素（インデックス i）を compare 関数で比較する。 3. 比較結果が 0 でない（異なる）場合、j をインクリメントし、i と j が異なる位置であれば memcpy を用いて要素を移動させる。 4. 最終的にユニークな要素数 j + 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_tool.c:318
  • src/backend/access/nbtree/nbtpreprocesskeys.c:2746
  • **呼び出し先**:
  • compare 関数（引数として渡されたもの）
  • memcpy（標準ライブラリ）

関連データ構造

  • 特になし（汎用的な void * 配列を扱う）。

失敗モードと境界条件

  • elements が 0 または 1 の場合、処理を行わず即座に値を返す。
  • 配列がソートされていない場合、重複が完全には排除されない。

保守担当者向けメモ

  • 本関数はインライン関数として定義されているため、呼び出しコストは低いが、要素サイズ（width）が大きい場合は memcpy のオーバーヘッドに留意すること。
  • string.h の memcpy を使用しているため、利用側で適切にインクルードされている必要がある。
