CreateRestrictedProcess

定義

  • **種別**: function
  • **定義箇所**: src/bin/pg_ctl/pg_ctl.c:1767

目的

Windows環境において、管理者権限や特定の特権を制限したセキュリティトークンを用いて新しいプロセスを生成し、ジョブオブジェクトによるリソース制限を適用する。

シグネチャまたは形状

int CreateRestrictedProcess(char *cmd, PROCESS_INFORMATION *processInfo, bool as_service)

入力と出力

  • **入力**:
  • cmd: 実行するコマンドライン文字列。
  • processInfo: 生成されたプロセスの情報を受け取る PROCESS_INFORMATION 構造体へのポインタ。
  • as_service: サービスとして実行するかを示すフラグ。
  • **出力**:
  • 成功時は TRUE (非ゼロ)、失敗時は 0 を返す。

前提条件と不変条件

  • 呼び出し元はWindows環境であること。
  • InheritStdHandles により、標準入出力ハンドルが子プロセスに継承されるよう設定される。
  • 管理者グループおよびパワーユーザーグループのSID、ならびに不要な特権がトークンから削除される。

アルゴリズムまたは振る舞い

1. InheritStdHandles を呼び出し、標準ハンドルを継承可能にする。 2. 現在のプロセスからトークンを取得し、管理者/パワーユーザーSIDおよび不要な特権を除去した制限付きトークンを作成する。 3. CreateProcessAsUser を CREATE_SUSPENDED フラグ付きで呼び出し、プロセスを一時停止状態で生成する。 4. プロセスがジョブオブジェクトに属していない場合、新しいジョブオブジェクトを作成し、以下の制限を適用する:

  • 未処理例外時の終了、優先度クラスの制限。
  • デスクトップ、クリップボード、システムパラメータへのアクセス制限。
  • セキュリティ制限（管理者権限の無効化）。

    • 5. プロセスをジョブオブジェクトに割り当てる。 6. ResumeThread を呼び出し、子プロセスの実行を開始する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_ctl/pg_ctl.c 内の do_start 等から呼び出される（直接的な証拠あり）。
  • **呼び出し先**: InheritStdHandles, GetPrivilegesToDelete, CreateRestrictedToken, AddUserToTokenDacl, CreateProcessAsUser, CreateJobObject, SetInformationJobObject, AssignProcessToJobObject, ResumeThread 等のWindows API。

関連データ構造

  • PROCESS_INFORMATION: プロセスおよびスレッドのハンドルとIDを保持。
  • STARTUPINFO: プロセスの起動設定。
  • JOBOBJECT_*: ジョブオブジェクトの制限設定用構造体群。

失敗モードと境界条件

  • プロセストークンの取得失敗、SIDの割り当て失敗、制限付きトークンの作成失敗時にエラーメッセージを stderr に出力し 0 を返す。
  • CreateProcessAsUser が失敗した場合、その結果を返す。

保守担当者向けメモ

  • ジョブオブジェクトのハンドルは、pg_ctl の終了まで生存させる意図で意図的にクローズしていない。
  • src/common/restricted_token.c にも同名の関数が存在するが、本関数は pg_ctl.c 内で定義されたローカルな実装である。
