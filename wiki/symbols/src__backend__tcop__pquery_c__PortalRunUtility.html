PortalRunUtility

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/pquery.c:1121

目的

ポータル内で実行されるユーティリティコマンド（VACUUM, CREATE TABLE 等）の実行を管理し、必要なトランザクションスナップショットの制御と実行後のクリーンアップを行う。

シグネチャまたは形状

static void PortalRunUtility(Portal portal, PlannedStmt *pstmt,
                             bool isTopLevel, bool setHoldSnapshot,
                             DestReceiver *dest, QueryCompletion *qc)

入力と出力

  • portal: 実行対象のポータル構造体。
  • pstmt: 実行するユーティリティコマンドの計画済みステートメント。
  • isTopLevel: 最上位レベルのクエリかどうかを示すフラグ。
  • setHoldSnapshot: スナップショットをポータルに保持するかどうか。
  • dest: 結果の出力先。
  • qc: コマンド完了情報を格納する構造体。
  • 戻り値: なし（void）。

前提条件と不変条件

  • PlannedStmtRequiresSnapshot(pstmt) が真の場合、スナップショットがアクティブなスナップショットスタックにプッシュされる。
  • 実行終了時、portal->portalSnapshot は NULL にリセットされる。
  • ユーティリティコマンド実行後、メモリコンテキストは portal->portalContext に切り替えられる。

アルゴリズムまたは振る舞い

1. **スナップショット管理**: コマンドがスナップショットを必要とする場合、GetTransactionSnapshot() を取得し、必要に応じて RegisterSnapshot で保持する。その後 PushActiveSnapshotWithLevel でアクティブ化し、ポータルに記録する。 2. **実行**: ProcessUtility を呼び出し、実際のユーティリティコマンドを実行する。 3. **コンテキスト復帰**: MemoryContextSwitchTo を使用して、ポータルのメモリコンテキストを確実に復元する。 4. **スナップショット解放**: 実行中にスナップショットがスタックから削除されていないことを確認し、PopActiveSnapshot でスタックを整理する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/pquery.c 内の PortalRun 等から呼び出される。
  • **呼び出し先**: ProcessUtility, GetTransactionSnapshot, RegisterSnapshot, PushActiveSnapshotWithLevel, GetActiveSnapshot, MemoryContextSwitchTo, ActiveSnapshotSet, PopActiveSnapshot。

関連データ構造

  • Portal: ポータルの状態を保持する構造体。
  • PlannedStmt: 実行対象の計画済みステートメント。
  • DestReceiver: クエリ結果の配送先インターフェース。
  • QueryCompletion: コマンド実行結果のメタデータ。

失敗モードと境界条件

  • VACUUM 等のコマンドは実行中にスナップショットスタックを操作（ポップ）する場合があるため、ActiveSnapshotSet() を確認して安全に処理を行う。
  • portalSnapshot が NULL の場合、スナップショットのポップ処理はスキップされる。

保守担当者向けメモ

  • 本関数はユーティリティコマンド実行時のスナップショット管理に特化している。EnsurePortalSnapshotExists との関連性に注意すること。
  • isTopLevel フラグは ProcessUtility に渡され、トップレベルのトランザクション制御に影響を与える。
