pg_b64_enc_len

定義

  • 種別: function
  • 定義箇所: src/common/base64.c:223

目的

バイナリデータをBase64エンコードする際に必要な出力バッファの最小サイズを算出します。

シグネチャまたは形状

int pg_b64_enc_len(int srclen)

入力と出力

  • **入力**: srclen (エンコード対象のバイナリデータのバイト数)
  • **出力**: エンコード後のBase64文字列を格納するために必要なバッファサイズ（バイト単位）。

前提条件と不変条件

  • 入力 srclen は非負の整数であることを想定しています。
  • 戻り値は、Base64の仕様に基づき、3バイトの入力が4文字の出力に変換される計算式によって算出されます。

アルゴリズムまたは振る舞い

入力されたバイト数 srclen に対して以下の計算を行います。 1. (srclen + 2) / 3 により、3バイト単位のブロック数を算出します（端数切り上げ）。 2. そのブロック数に4を乗算し、必要な文字数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • contrib/postgres_fdw/connection.c
  • src/backend/libpq/auth-scram.c
  • src/common/scram-common.c
  • **呼び出し先**:
  • 本関数は算術演算のみを行うため、他の関数を呼び出しません。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • srclen が 0 の場合、戻り値は 0 となります。
  • 負の値を入力した場合の挙動は定義されていません。

保守担当者向けメモ

  • 本関数はバッファオーバーフローを防ぐためのメモリ確保計算に使用されます。エンコード処理（pg_b64_encode）を呼び出す前に、必ずこの関数で算出したサイズ以上のバッファを確保してください。
