validate_log_connections_options

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/backend_startup.c:994

目的

log_connections GUC（設定パラメータ）に指定された文字列リストを検証し、対応するビットフラグへ変換する。PostgreSQL 18以降のリスト形式への対応と、旧バージョンとの互換性を維持する役割を持つ。

シグネチャまたは形状

static bool validate_log_connections_options(List *elemlist, uint32 *flags)

入力と出力

  • **入力**:
  • elemlist: 検証対象の文字列リスト。
  • flags: 変換後のビットフラグを格納する出力用ポインタ。
  • **出力**:
  • bool: 検証が成功した場合は true、無効なオプションが含まれる場合は false を返す。

前提条件と不変条件

  • elemlist が空の場合、flags は 0 に初期化され、検証は成功（true）とみなされる。
  • 互換性オプション（off, true 等）が指定された場合、リストにはその単一要素のみが含まれている必要がある。複数の要素と同時に指定された場合はエラーとなる。

アルゴリズムまたは振る舞い

1. **互換性チェック**: compat_options 配列を参照し、入力が旧来のブール値表現（on, off, true, false 等）と一致するか確認する。一致した場合は、リスト長が1であることを確認した上で、対応するフラグをセットして終了する。 2. **詳細オプションの解析**: リスト内の各要素を options 配列（receipt, authentication, authorization, setup_durations, all）と照合する。 3. **フラグの集約**: 一致したオプションに対応するビットフラグを *flags に対してビット単位の論理和（OR）で累積する。 4. **エラーハンドリング**: どのオプションにも一致しない文字列が含まれていた場合、GUC_check_errdetail を通じてエラー詳細を記録し、false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/backend_startup.c 内の check_log_connections（1106行目付近）から呼び出される。
  • **呼び出し先**: 内部で list_length, linitial, pg_strcasecmp, GUC_check_errdetail などのユーティリティ関数を使用する。

関連データ構造

  • config_enum_entry: GUC設定の名称と値のペアを定義する構造体。
  • List: PostgreSQLのリスト構造。

失敗モードと境界条件

  • **無効なオプション**: 定義されていない文字列がリストに含まれる場合、false を返し、エラー詳細を設定する。
  • **互換性オプションの混在**: 互換性オプション（例: on）と他のオプションが同時にリストに含まれる場合、エラーとして処理される。

保守担当者向けメモ

  • 本関数は log_connections の設定変更時に呼び出されるため、新しい接続ログのカテゴリを追加する場合は、内部の options 配列を更新する必要がある。
  • 互換性オプションのリストは、過去の挙動を維持するために固定されている。
