XLogRecPtr

定義

  • 種別: typedef (または struct としての参照)
  • 定義箇所: access/xlogreader.h:62 (本体)、src/backend/access/transam/xlogprefetcher.c:113 (構造体メンバとしての利用)

目的

WAL（Write-Ahead Log）内の特定のレコード位置を指し示すためのポインタ型です。

シグネチャまたは形状

64ビットの整数値として定義されており、WALストリーム上の論理的なオフセットを表します。

入力と出力

  • 入力: WALの読み込み位置や、ページヘッダ内のLSN（Log Sequence Number）として使用されます。
  • 出力: WALレコードの走査、リカバリの進捗管理、およびページが最後に更新された時点の識別子として返されます。

前提条件と不変条件

  • 値は単調増加します。
  • 常に有効なWALレコードの開始位置、またはページ内のLSNを指している必要があります。

アルゴリズムまたは振る舞い

WALのデコードやリプレイ処理において、現在の処理位置を追跡するために使用されます。xlogprefetcher.c 内では、先読みキュー（LsnReadQueue）において、特定のI/Oリクエストに関連付けられたWAL位置を保持するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/amcheck/verify_nbtree.c、contrib/pg_walinspect/pg_walinspect.c 等のWAL解析ツールや、リカバリ処理プロセスから参照されます。
  • 呼び出し先: 本シンボル自体はデータ型であるため、直接的な呼び出し先はありません。

関連データ構造

  • XLogReaderState: WAL読み込み状態を保持する構造体。
  • LsnReadQueue: 先読みI/Oリクエストを管理するキュー構造体（XLogRecPtr をメンバとして保持）。

失敗モードと境界条件

  • 無効なLSN（InvalidXLogRecPtr）を指定した場合、WALの読み込みや比較処理においてエラーまたは予期せぬ動作を引き起こす可能性があります。
  • 64ビットの範囲を超えることは想定されていません。

保守担当者向けメモ

  • 本シンボルは access/xlogreader.h で定義されているコア型です。xlogprefetcher.c 内の構造体メンバとして現れる際は、先読み対象のWAL位置を特定するために利用されています。
  • 修正や変更を行う際は、WALの互換性に直結するため、xlogreader.h の定義を優先して参照してください。
