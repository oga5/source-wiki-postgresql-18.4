initTrie

定義

  • 種別: function
  • 定義箇所: contrib/unaccent/unaccent.c:96

目的

指定された設定ファイルから置換ルールを読み込み、文字列置換のための Trie 木構造を構築する。

シグネチャまたは形状

static TrieChar *initTrie(const char *filename)

入力と出力

  • **入力**: filename (設定ファイル名)
  • **出力**: 構築された Trie 木のルートノードへのポインタ (TrieChar *)。

前提条件と不変条件

  • filename は get_tsearch_config_filename を通じて解決される。
  • 設定ファイルは "src" または "src trg" の形式で記述されている必要がある。
  • ファイル読み込み時にエンコーディング変換エラーが発生した場合、その行はスキップされる。

アルゴリズムまたは振る舞い

1. get_tsearch_config_filename でファイルパスを特定し、tsearch_readline_begin で読み込みを開始する。 2. ファイルの各行をループで処理する。 3. 状態機械を用いて行内の src（置換元）と trg（置換先）を解析する。

  • trg が省略された場合は空文字列として扱う。
  • trg が引用符で囲まれている場合は、引用符を除去しエスケープを処理する。

    • 4. 解析した各ルールを placeChar 関数に渡し、Trie 木に登録する。 5. 構文エラー（文字列が3つ以上ある、引用符が閉じられていない等）が発生した場合は WARNING を発行する。 6. 処理終了後、メモリを解放し、構築した Trie 木のルートを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: unaccent_init (ソースコード内の直接的な呼び出し箇所として確認)
  • **呼び出し先**: get_tsearch_config_filename, tsearch_readline_begin, tsearch_readline, pg_mblen_cstr, placeChar, palloc, pfree, ereport

関連データ構造

  • TrieChar: 置換ルールを保持する Trie 木のノード構造体。
  • tsearch_readline_state: ファイル読み込み状態を管理する構造体。

失敗モードと境界条件

  • **ファイルオープン失敗**: tsearch_readline_begin が失敗した場合、ERROR を発生させ処理を中断する。
  • **構文エラー**: 行の形式が不正な場合、WARNING を発行し、その行をスキップして処理を継続する。
  • **エンコーディングエラー**: tsearch_readline 内部で発生する変換エラーは PG_TRY ブロックで捕捉され、該当行をスキップする。

保守担当者向けメモ

  • Trie 木の検索はバイト単位で行われるため、マルチバイト文字を含むルールを扱う際は、入力データとルールファイルのエンコーディングの一致が重要である。
  • 引用符で囲まれた trg 内の "" は、単一の " としてエスケープされる仕様である。
