int128_add_int64_mul_int64

定義

  • 種別: function
  • 定義箇所: src/include/common/int128.h:61

目的

2つの64ビット整数の積を計算し、その結果を既存の128ビット整数値に加算します。

シグネチャまたは形状

static inline void int128_add_int64_mul_int64(INT128 *i128, int64 x, int64 y)

入力と出力

  • **入力**:
  • i128: 加算対象となる128ビット整数へのポインタ（入出力兼用）。
  • x: 乗数となる64ビット整数。
  • y: 乗数となる64ビット整数。
  • **出力**: なし（i128 が指すメモリ領域が直接更新されます）。

前提条件と不変条件

  • i128 は有効な INT128 型のメモリ領域を指している必要があります。
  • 演算結果が128ビットの範囲を超える場合のオーバーフロー挙動は、コンパイラの int128 型の仕様に依存します。

アルゴリズムまたは振る舞い

  • x と y をそれぞれ int128 型にキャストし、乗算を行います。
  • その積を i128 が指す値に加算します。
  • ネイティブな int128 がサポートされている環境では、コンパイラの組み込み演算を利用します。サポートされていない環境では、ソフトウェア実装（32ビット単位の分割計算）へフォールバックされます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/timestamp.c:2562
  • src/tools/testint128.c:127
  • **呼び出し先**:
  • コンパイラ組み込みの算術演算子（ネイティブ環境時）または int128.h 内のソフトウェア実装用ロジック。

関連データ構造

  • INT128: 128ビット整数を保持する型。環境に応じてネイティブ型または構造体として定義されます。

失敗モードと境界条件

  • ソフトウェア実装環境では、算術右シフトの挙動に依存するため、コンパイル時に StaticAssertDecl による検証が行われます。この検証に失敗する環境では正しく動作しません。

保守担当者向けメモ

  • ソフトウェア実装は複雑な分岐とシフト演算を含むため、パフォーマンス上の理由からネイティブな int128 型が利用可能な環境（USE_NATIVE_INT128）での利用が強く推奨されます。
  • エンディアン（WORDS_BIGENDIAN）を考慮した設計となっているため、移植性を変更する際は注意が必要です。
