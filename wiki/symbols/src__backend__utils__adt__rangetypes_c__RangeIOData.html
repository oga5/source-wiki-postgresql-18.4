RangeIOData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/rangetypes.c:51

目的

範囲型（Range Type）の入出力処理に必要なメタデータを保持するための構造体です。範囲型の要素型に関するキャッシュ情報や、I/O 関数呼び出しに必要な情報を管理します。

シグネチャまたは形状

typedef struct RangeIOData
{
    TypeCacheEntry *typcache;   /* 範囲型の型キャッシュエントリ */
    FmgrInfo       typioproc;   /* 要素型のI/O関数情報 */
    Oid            typioparam;  /* 要素型のI/Oパラメータ */
} RangeIOData;

入力と出力

本構造体自体はデータ保持用ですが、get_range_io_data 関数等を通じて初期化され、範囲型の入出力処理において参照されます。

前提条件と不変条件

  • typcache は、range_get_typcache を通じて取得された有効な TypeCacheEntry を指している必要があります。
  • typioproc は、要素型の入出力を行うための関数マネージャ情報として適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

範囲型の入出力関数（range_in, range_out 等）において、要素型の型情報を動的に取得・保持するために使用されます。関数呼び出しの fn_extra にキャッシュとして格納することで、繰り返し呼び出される処理のオーバーヘッドを削減します。

呼び出し元と呼び出し先

  • **呼び出し元**: get_range_io_data 関数により生成されます。また、range_in や range_out などの範囲型関連関数が、fn_extra を介して本構造体を取得・利用します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありませんが、保持する typioproc を通じて要素型の入出力関数が実行されます。

関連データ構造

  • TypeCacheEntry: 範囲型の型情報を保持するキャッシュ構造体。
  • FmgrInfo: PostgreSQL の関数マネージャ情報。
  • RangeType: 範囲型の内部表現（varlena）。

失敗モードと境界条件

  • fn_extra が初期化されていない場合、またはメモリ確保に失敗した場合は、呼び出し側の関数でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は src/backend/utils/adt/rangetypes.c 内で完結して利用されています。
  • 範囲型の入出力ロジックを変更する際は、typioproc や typioparam の設定が正しく維持されているか確認してください。
  • fn_extra を介したメモリ管理（MemoryContextAlloc）が行われているため、メモリリークや不正なポインタ参照に注意してください。
