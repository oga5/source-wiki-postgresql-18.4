statext_mcv_deserialize

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/mcv.c:995

目的

pg_statistic_ext_data カタログに格納されたバイナリ形式（bytea）の多変量MCVリストを、メモリ上の MCVList 構造体へ復元する。

シグネチャまたは形状

MCVList *statext_mcv_deserialize(bytea *data)

入力と出力

  • **入力**: bytea *data - シリアライズされたMCVリストデータ。
  • **出力**: MCVList * - 復元されたMCVリスト構造体。data が NULL の場合は NULL を返す。

前提条件と不変条件

  • 入力データは STATS_MCV_MAGIC を保持し、STATS_MCV_TYPE_BASIC 型である必要がある。
  • ndimensions は 1 以上 STATS_MAX_DIMENSIONS 以下であること。
  • nitems は 1 以上 STATS_MCVLIST_MAX_ITEMS 以下であること。
  • 入力データのサイズは、ヘッダーおよび次元情報、アイテム数と整合している必要がある。

アルゴリズムまたは振る舞い

1. **ヘッダー検証**: VARSIZE_ANY を用いて最小サイズをチェックし、マジックナンバー、型、次元数、アイテム数の妥当性を検証する。 2. **構造体初期化**: MCVList 構造体を割り当て、ヘッダー情報をコピーする。 3. **メタデータ抽出**: 次元情報（DimensionInfo）を読み取り、各次元のデータサイズを検証する。 4. **メモリ確保**: 復元後のデータ保持に必要なメモリ量を計算し、repalloc で MCVList を再割り当てする。 5. **データ展開**: 参照型データ（by-ref）のコピーを確保し、値と isnull フラグの配列を配置する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/mcv.c:577
  • src/backend/statistics/mcv.c:1355
  • **呼び出し先**:
  • palloc0, palloc, repalloc (メモリ管理)
  • elog (エラー報告)
  • memcpy (データコピー)

関連データ構造

  • MCVList: 多変量統計情報を保持するメイン構造体。
  • DimensionInfo: 各次元の統計メタデータ。
  • MCVItem: 個別のMCVアイテム情報。

失敗モードと境界条件

  • data が NULL の場合は NULL を返す。
  • ヘッダーの整合性チェック（マジックナンバー、型、次元数、アイテム数）に失敗した場合、elog(ERROR) を発生させる。
  • データの総サイズが期待値と一致しない場合、elog(ERROR) を発生させる。

保守担当者向けメモ

  • 本関数は、シリアライズされたデータが不適切なアライメントを持つ可能性があるため、メモリ再配置時に MAXALIGN を使用して安全なメモリレイアウトを構築している。
  • DimensionInfo のアクセスは、サイズ検証が完了した後にのみ行われるよう順序が厳密に制御されている。
