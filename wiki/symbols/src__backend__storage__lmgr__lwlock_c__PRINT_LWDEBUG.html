PRINT_LWDEBUG

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lwlock.c:273

目的

軽量ロック（LWLock）の状態をデバッグログとして出力する。

シグネチャまたは形状

inline static void PRINT_LWDEBUG(const char *where, LWLock *lock, LWLockMode mode)

入力と出力

  • **入力**:
  • where: ログ出力元を示す文字列（関数名など）。
  • lock: 状態を出力する対象の LWLock 構造体へのポインタ。
  • mode: ロックモード（ログ出力には直接使用されない）。
  • **出力**: なし（ereport によるログ出力のみ）。

前提条件と不変条件

  • グローバル変数 Trace_lwlocks が真である場合にのみログが出力される。
  • lock ポインタは有効な LWLock 構造体を指している必要がある。

アルゴリズムまたは振る舞い

1. Trace_lwlocks が有効か確認する。 2. pg_atomic_read_u32 を使用して、対象 lock の現在の state をアトミックに読み取る。 3. ereport(LOG, ...) を呼び出し、以下の情報をログに出力する：

  • プロセスID (MyProcPid)
  • 呼び出し元 (where)
  • ロック名 (T_NAME(lock))
  • 排他ロックの有無 (LW_VAL_EXCLUSIVE)
  • 共有ロック数 (LW_SHARED_MASK)
  • 待機者の有無 (LW_FLAG_HAS_WAITERS)
  • 待機者数 (nwaiters)
  • 解放許可フラグ (LW_FLAG_RELEASE_OK)

    • 4. ログ出力時は、ステートメントおよびコンテキスト情報を隠蔽する設定 (errhidestmt(true), errhidecontext(true)) が適用される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • LWLockAcquire
  • LWLockConditionalAcquire
  • LWLockAcquireOrWait
  • LWLockWaitForVar
  • LWLockUpdateVar
  • LWLockRelease
  • **呼び出し先**:
  • pg_atomic_read_u32
  • ereport
  • T_NAME (マクロ)

関連データ構造

  • LWLock: ロックの状態と待機情報を保持する構造体。

失敗モードと境界条件

  • Trace_lwlocks が偽の場合、本関数は何も行わない。
  • コンパイル設定により、PRINT_LWDEBUG は空の定義（((void)0)）に置換される場合がある。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、デバッグ目的のトレース用である。
  • ログ出力が非常に冗長になることを避けるため、errhidestmt および errhidecontext を使用して出力内容を制限している。
