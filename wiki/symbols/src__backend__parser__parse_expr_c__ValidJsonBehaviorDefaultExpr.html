ValidJsonBehaviorDefaultExpr

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_expr.c:4708

目的

SQL/JSON関数における DEFAULT 句の式が、実行時に安全かつ適切に評価可能な形式であるかを検証します。

シグネチャまたは形状

static bool ValidJsonBehaviorDefaultExpr(Node *expr, void *context)

入力と出力

  • **入力**:
  • expr: 検証対象の式ツリーノード。
  • context: expression_tree_walker に渡される補助コンテキスト。
  • **出力**:
  • bool: 式が許可されたノード構成であれば true、そうでなければ false を返します。

前提条件と不変条件

  • expr が NULL の場合は false を返します。
  • 再帰的な検証において、ノードの型が許可されたリストに含まれている必要があります。

アルゴリズムまたは振る舞い

1. nodeTag(expr) を用いてノードの種類を判定します。 2. 以下のノードは直接許可（true）します。

  • T_Const
  • T_FuncExpr
  • T_OpExpr

    • 3. 以下のノードは、その子ノードが再帰的に検証を通過する場合に許可（true）します。

  • T_CoerceViaIO, T_CoerceToDomain, T_ArrayCoerceExpr, T_ConvertRowtypeExpr, T_RelabelType, T_CollateExpr
  • これらは expression_tree_walker を介して再帰的に処理されます。

    • 4. 上記以外のノードはすべて false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_expr.c:4763 (検証の実行)
  • src/backend/parser/parse_expr.c:4709 (自身を再帰的に呼び出し)
  • **呼び出し先**:
  • expression_tree_walker (ツリー走査用ユーティリティ)

関連データ構造

  • Node: PostgreSQLの汎用ノード構造体。
  • NodeTag: ノードの種類を識別する列挙型。

失敗モードと境界条件

  • expr が NULL の場合、即座に false を返します。
  • 許可されていないノード型（例：サブクエリや変数参照など）が含まれている場合、検証は失敗し false となります。

保守担当者向けメモ

  • 本関数は SQL/JSON の DEFAULT 値として許容される式の範囲を制限するホワイトリストとして機能します。新しいノード型を許可する必要がある場合は、switch 文の適切なセクションに追加してください。
