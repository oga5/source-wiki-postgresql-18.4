GISTSearchItem

定義

  • 種別: struct
  • 定義箇所: src/include/access/gist_private.h:130

目的

GiSTインデックスの検索処理において、探索対象となるインデックスページまたはヒープタプルを管理するためのキュー要素です。

シグネチャまたは形状

typedef struct GISTSearchItem
{
    pairingheap_node phNode;
    BlockNumber blkno;
    union {
        GistNSN parentlsn;
        GISTSearchHeapItem heap;
    } data;
    IndexOrderByDistance distances[FLEXIBLE_ARRAY_MEMBER];
} GISTSearchItem;

入力と出力

  • **入力**: 検索対象のブロック番号、親ページのLSN（インデックスページの場合）、またはヒープ情報（ヒープタプルの場合）、および距離情報。
  • **出力**: pairingheap を介して優先順位付けされた探索候補として利用されます。

前提条件と不変条件

  • distances フィールドは可変長配列として定義されており、numberOfOrderBys に応じたエントリを保持します。
  • インデックスページを指す場合、parentlsn を保持することで、検索中にページ分割が発生したかどうかを検出します。

アルゴリズムまたは振る舞い

  • pairingheap のノードとして機能し、GiSTスキャン時の優先度付きキュー管理に使用されます。
  • 検索アルゴリズムは、この構造体をキューから取り出し、インデックスページであれば子ノードを探索し、ヒープタプルであれば結果として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistget.c 内の gistScanPage および関連するスキャン関数群が、新しい検索アイテムの生成およびキューからの取り出しを行っています。
  • **呼び出し先**: pairingheap API（pairingheap_remove_first 等）が本構造体を操作します。

関連データ構造

  • pairingheap_node: キュー管理用。
  • GistNSN: インデックスページの分割検出用。
  • GISTSearchHeapItem: ヒープタプル情報保持用。
  • IndexOrderByDistance: 距離順検索のための距離情報。

失敗モードと境界条件

  • 呼び出し元は、取得した GISTSearchItem を使用後に pfree する責任があります。
  • blkno が InvalidBlockNumber の場合、インデックスページ以外の状態を示します。

保守担当者向けメモ

  • 本構造体はGiSTの検索エンジンの中核であり、メモリ管理は呼び出し側のスキャンコンテキストに依存します。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、インスタンス生成時には適切なメモリサイズを確保する必要があります。
