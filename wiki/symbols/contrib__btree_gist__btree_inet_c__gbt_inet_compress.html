gbt_inet_compress

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_inet.c:91

目的

GiST インデックス構築時、inet 型の値をインデックス格納用の inetKEY 構造体に変換（圧縮）します。

シグネチャまたは形状

Datum gbt_inet_compress(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体を受け取ります。
  • **出力**: GISTENTRY へのポインタを Datum として返します。

前提条件と不変条件

  • entry->leafkey が真の場合、convert_network_to_scalar が成功し、failure フラグが偽であることを前提とします（Assert で検証）。
  • リーフノードでは、生成される inetKEY の lower と upper は同一の値に設定されます。

アルゴリズムまたは振る舞い

1. entry->leafkey が真（リーフノード）の場合:

  • inetKEY 構造体をメモリ確保します。
  • convert_network_to_scalar を呼び出し、inet 型の値を double 型のスカラ値に変換します。
  • lower および upper フィールドに変換後の値を代入します。
  • gistentryinit を使用して、新しい GISTENTRY を初期化します。

    • 2. entry->leafkey が偽（内部ノード）の場合:

  • 入力された entry をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス構築プロセス（直接的な呼び出し元コードはソース抜粋に含まれていません）。
  • **呼び出し先**: convert_network_to_scalar（inet 型の変換）、gistentryinit（GiST エントリの初期化）、palloc（メモリ確保）。

関連データ構造

  • inetKEY: インデックス内で保持される境界値（lower, upper）を持つ構造体。
  • GISTENTRY: GiST インデックスの各ノードエントリを表現する構造体。

失敗モードと境界条件

  • convert_network_to_scalar が失敗した場合、Assert によりプロセスが中断されます。
  • リーフノード以外では変換処理を行わず、入力をそのまま透過的に返します。

保守担当者向けメモ

  • 本関数は inet 型を double に変換して処理するため、非常に大きなネットワーク範囲や複雑なサブネットマスクの精度には限界がある可能性があります。
  • inetKEY 構造体はリーフノードにおいて lower == upper となるよう設計されています。
