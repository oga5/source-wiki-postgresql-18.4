leftmostvalue_float8

定義

  • 種別: function
  • 定義箇所: contrib/btree_gin/btree_gin.c:241

目的

float8 型のデータに対して、GINインデックスを用いた範囲検索（特に BTLess 系戦略）を行う際、インデックスの走査開始点として使用される最小値（負の無限大）を提供します。

シグネチャまたは形状

static Datum leftmostvalue_float8(void)

入力と出力

  • 入力: なし
  • 出力: float8 型の負の無限大を表す Datum 値。

前提条件と不変条件

  • float8 型の演算および get_float8_infinity() が利用可能であること。
  • GINインデックスのサポート関数として呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. get_float8_infinity() を呼び出し、浮動小数点の無限大値を取得します。 2. その値に負の符号を付与します。 3. Float8GetDatum マクロを使用して、計算結果を PostgreSQL の Datum 形式に変換して返します。

呼び出し元と呼び出し先

  • 呼び出し元: GIN_SUPPORT(float8, false, leftmostvalue_float8, btfloat8cmp) マクロ定義内から参照されています。
  • 呼び出し先: get_float8_infinity()（utils/float.h 由来）および Float8GetDatum。

関連データ構造

  • Datum: PostgreSQL の汎用データ表現型。

失敗モードと境界条件

  • 浮動小数点の演算環境に依存します。システムが IEEE 754 準拠の無限大をサポートしていない環境では、期待通りの最小値として機能しない可能性があります。

保守担当者向けメモ

  • 本関数は GIN_SUPPORT マクロを通じて登録されます。新しいデータ型を追加する際、同様の「最小値」を返す関数を定義する際のテンプレートとして利用してください。
  • numeric 型など、絶対的な最小値が存在しない型とは異なり、float8 は明確な負の無限大を持つため、ダミー値の考慮は不要です。
