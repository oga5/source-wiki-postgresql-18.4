dosqlprint

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/compat_informix-test_informix.c:28

目的

SQL 操作でエラーが発生した際、グローバルな SQL 通信領域（sqlca）に格納されたエラーメッセージを標準エラー出力へ表示するヘルパー関数です。

シグネチャまたは形状

static void dosqlprint(void)

入力と出力

  • 入力: なし（グローバル変数 sqlca を参照）
  • 出力: なし（標準エラー出力への書き込みのみ）

前提条件と不変条件

  • sqlca 構造体が適切に初期化され、有効なエラー情報が保持されている必要があります。

アルゴリズムまたは振る舞い

1. printf を使用し、sqlca.sqlerrm.sqlerrmc に格納されているエラーメッセージ文字列を "doSQLprint: Error: %s\n" という形式で標準出力（stdout）へ出力します。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/test/expected/compat_informix-test_informix.c 内の各 SQL 文実行後のエラーチェック処理（if (sqlca.sqlcode < 0) dosqlprint ( );）から頻繁に呼び出されます。
  • 呼び出し先: 標準ライブラリの printf を呼び出します。

関連データ構造

  • sqlca: SQL 通信領域を保持するグローバル構造体。本関数は特に sqlca.sqlerrm.sqlerrmc を参照します。

失敗モードと境界条件

  • sqlca.sqlerrm.sqlerrmc が空または不正な文字列である場合、意図しない出力が行われる可能性があります。

保守担当者向けメモ

  • 本関数は ECPG の Informix 互換テストにおけるエラーハンドリング用です。
  • printf を使用しているため、テスト実行時のログ出力先が標準出力に固定されている点に注意してください。
