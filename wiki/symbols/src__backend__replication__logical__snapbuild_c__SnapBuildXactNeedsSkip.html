SnapBuildXactNeedsSkip

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/snapbuild.c:303

目的

論理デコーディングにおいて、指定されたWALレコードがデコーディング開始時点よりも前のものであるかを判定し、処理をスキップすべきかどうかを判断する。

シグネチャまたは形状

bool SnapBuildXactNeedsSkip(SnapBuild *builder, XLogRecPtr ptr)

入力と出力

  • **入力**:
  • builder: スナップショット構築の状態を保持する SnapBuild 構造体へのポインタ。
  • ptr: 判定対象となるWALレコードのLSN（XLogRecPtr）。
  • **出力**:
  • bool: ptr が builder->start_decoding_at より小さい場合に true を返し、それ以外の場合に false を返す。

前提条件と不変条件

  • builder は初期化済みであること。
  • builder->start_decoding_at には、論理デコーディングを開始すべき有効なLSNが格納されていること。

アルゴリズムまたは振る舞い

1. 引数として受け取ったWALレコードのLSN ptr と、builder 内に保持されているデコーディング開始位置 start_decoding_at を比較する。 2. ptr < builder->start_decoding_at が真であれば、そのレコードはデコーディング対象外とみなして true を返す。 3. それ以外の場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/decode.c: WALレコードのデコード処理内で、レコードのスキップ判定に使用。
  • src/backend/replication/logical/reorderbuffer.c: トランザクションの変更適用や読み取り処理において、対象レコードが範囲内かを確認するために使用。
  • **呼び出し先**:
  • 本関数は比較演算のみを行うため、他の関数を呼び出さない。

関連データ構造

  • SnapBuild: スナップショット構築のメイン状態構造体。start_decoding_at メンバを保持する。
  • XLogRecPtr: WAL上の位置を示すデータ型。

失敗モードと境界条件

  • builder が NULL の場合、ポインタ参照によりクラッシュする可能性がある。
  • ptr が無効なLSNである場合、比較結果は不定となる。

保守担当者向けメモ

  • 本関数は非常に軽量な述語関数であり、論理デコーディングのループ内で頻繁に呼び出される。ロジックを変更する場合は、パフォーマンスへの影響を考慮すること。
