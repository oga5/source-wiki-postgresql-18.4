gb18030_to_utf8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_gb18030/utf8_and_gb18030.c:196

目的

GB18030 エンコーディングから UTF-8 エンコーディングへの文字コード変換を実行する、PostgreSQL の SQL 呼び出し可能な関数です。

シグネチャまたは形状

Datum gb18030_to_utf8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて以下の引数を受け取ります。
  • src (CSTRING): 変換元の GB18030 文字列。
  • dest (CSTRING): 変換後の UTF-8 文字列を格納するバッファ。
  • len (INT32): 変換対象のバイト長。
  • noError (BOOL): 変換エラー発生時の挙動を制御するフラグ。
  • **出力**: Datum 型として、実際に変換されたバイト数を返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_GB18030, PG_UTF8) マクロにより、変換元が GB18030、変換先が UTF-8 であることが検証されます。

アルゴリズムまたは振る舞い

1. LocalToUtf 関数を呼び出し、変換処理を開始します。 2. 変換には gb18030_to_unicode_tree マッピングテーブルを使用します。 3. マッピングテーブルで処理できない特殊な範囲（主に 4 バイト文字）については、conv_18030_to_utf8 関数を介して処理します。 4. 最終的に変換されたバイト数を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の変換サブシステム（src/backend/utils/mb/mbutils.c 等から間接的に参照される可能性あり）。
  • **呼び出し先**:
  • LocalToUtf (PostgreSQL 内部関数)
  • conv_18030_to_utf8 (補助変換関数)

関連データ構造

  • gb18030_to_unicode_tree: 変換用マッピングテーブル。

失敗モードと境界条件

  • noError 引数が false の場合、不正なバイト列が入力されるとエラーが発生します。
  • 4 バイト文字などの特殊な範囲は、conv_18030_to_utf8 を通じた計算ロジックに依存します。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_INFO_V1 マクロにより、PostgreSQL の関数管理インターフェースに登録されています。
  • 基本的な変換は .map ファイルのツリー構造で行われますが、4 バイト文字の計算ロジックについては conv_18030_to_utf8 の実装を確認してください。
