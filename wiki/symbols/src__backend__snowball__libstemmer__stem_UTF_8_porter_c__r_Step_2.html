r_Step_2

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_porter.c:369

目的

Porter Stemming アルゴリズムの第 2 段階として、特定の接尾辞を語幹の領域 R1 内で置換する。

シグネチャまたは形状

static int r_Step_2(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境の状態を保持する構造体）
  • 出力: 成功時に 1、失敗または条件不一致時に 0、エラー発生時に負の値を返す。

前提条件と不変条件

  • z->c が現在のカーソル位置を指していること。
  • r_R1 関数により、対象の接尾辞が語幹の領域 R1 内に存在することが検証される。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置から後方に向かって、a_3 で定義された 20 種類の接尾辞を find_among_b で検索する。 2. マッチした接尾辞のインデックス（among_var）を取得する。 3. r_R1 を呼び出し、マッチした接尾辞が R1 領域内にあるか確認する。 4. among_var の値に基づき、switch 文で対応する接尾辞を別の文字列に置換（slice_from_s）する。 5. 処理が完了すれば 1 を返し、マッチしない場合や R1 領域外の場合は 0 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: porter_UTF_8_stem（直接の呼び出しは stem_UTF_8_porter.c 等の各ステマー実装内で行われる）。
  • 呼び出し先: find_among_b, r_R1, slice_from_s。

関連データ構造

  • SN_env: ステミング処理の状態を管理する構造体。
  • a_3: r_Step_2 で使用される接尾辞の検索用 among 構造体配列。

失敗モードと境界条件

  • z->c - 2 <= z->lb: 文字列の長さが不足している場合、処理を中断し 0 を返す。
  • find_among_b がマッチしない場合、0 を返す。
  • r_R1 が失敗した場合、その戻り値をそのまま返す。
  • slice_from_s がメモリ不足等で失敗した場合、負の値を返す。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによって自動生成されている。ロジックの変更が必要な場合は、直接編集せず、対応する .sbl ファイルを修正すること。
  • among_var に対応する各 s_n 文字列定数は、Snowball の生成ルールに従って定義されている。
