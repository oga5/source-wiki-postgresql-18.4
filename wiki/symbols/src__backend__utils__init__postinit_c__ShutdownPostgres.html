ShutdownPostgres

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/postinit.c:1342

目的

バックエンドプロセスの終了時に、アクティブなトランザクションの強制終了およびユーザーロックの解放を行うクリーンアップ処理を実行します。

シグネチャまたは形状

static void ShutdownPostgres(int code, Datum arg)

入力と出力

  • code (int): 終了コード（on_proc_exit 等のコールバック規約に従う）。
  • arg (Datum): 補助引数。
  • 出力: なし（戻り値なし）。

前提条件と不変条件

  • 本関数は、プロセス終了処理の過程で呼び出されることを想定しています。
  • トランザクション終了後も保持される「ユーザーロック」を明示的に解放する必要があります。

アルゴリズムまたは振る舞い

1. AbortOutOfAnyTransaction() を呼び出し、実行中のアクティブなトランザクションを強制的に終了させます。 2. LockReleaseAll(USER_LOCKMETHOD, true) を呼び出し、すべてのユーザーロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/init/postinit.c:829 (before_shmem_exit に登録されるコールバックとして使用)
  • **呼び出し先**:
  • AbortOutOfAnyTransaction()
  • LockReleaseAll()
  • **注記**: src/backend/postmaster/auxprocess.c および src/backend/replication/logical/worker.c 内のコメントで本関数への言及がありますが、直接的な呼び出し関係については提供されたソースから断定できません。

関連データ構造

  • USER_LOCKMETHOD: ロック解放対象を指定するための定数。

失敗モードと境界条件

  • トランザクションが既に終了している場合でも AbortOutOfAnyTransaction は安全に処理を完了します。
  • 共有メモリが既にデタッチされている環境下での呼び出しには注意が必要です。

保守担当者向けメモ

  • 本関数は before_shmem_exit を介して登録されるため、プロセス終了時のクリーンアップ順序において重要な役割を果たします。
  • ユーザーロック以外のリソース解放が必要になった場合は、本関数内に適切な解放処理を追加してください。
