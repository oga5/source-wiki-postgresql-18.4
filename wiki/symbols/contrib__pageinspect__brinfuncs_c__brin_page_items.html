brin_page_items

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/brinfuncs.c:128

目的

BRIN（Block Range Index）インデックスの物理ページを解析し、ページ内に格納されているインデックス項目（タプル）をデコードして SQL 結果セットとして返します。主にデバッグやインデックスの整合性確認に使用されます。

シグネチャまたは形状

Datum brin_page_items(PG_FUNCTION_ARGS)

  • 引数: bytea（raw_page）, Oid（indexRelid）
  • 戻り値: Datum（セットリターニング関数として tuplestore を介して行を返却）

入力と出力

  • 入力:
  • raw_page: get_raw_page 等で取得した BRIN インデックスのページデータ。
  • indexRelid: 対象となる BRIN インデックスの OID。
  • 出力:
  • ページ内の各タプルについて、オフセット番号、ブロック番号、属性番号、NULL フラグ、プレースホルダーフラグ、空範囲フラグ、および属性値の文字列表現を含む行を返します。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要です。
  • pageinspect 拡張のバージョンが BRIN_PAGE_ITEMS_V1_12 以上である必要があります。
  • 指定されたリレーションは BRIN インデックスである必要があります。
  • ページは有効な BRIN レギュラーページである必要があります。

アルゴリズムまたは振る舞い

1. 権限チェックを行い、拡張のバージョンが古い場合はエラーを発生させます。 2. インデックスリレーションを開き、BRIN インデックスであることを検証します。 3. brin_build_desc を使用してインデックスの記述子を構築し、各カラムの出力関数を初期化します。 4. ページ内の各オフセットを走査し、brin_deform_tuple を用いて物理タプルを BrinMemTuple 構造体に変換します。 5. 変換されたデータを SQL の出力形式に整形し、tuplestore に格納して返却します。

呼び出し元と呼び出し先

  • 呼び出し元: SQL インターフェース（SELECT * FROM brin_page_items(...)）から呼び出されます。
  • 呼び出し先: verify_brin_page, brin_build_desc, brin_deform_tuple, index_open, getTypeOutputInfo, fmgr_info, OutputFunctionCall 等。

関連データ構造

  • brin_column_state: 各カラムの出力関数情報を保持する内部構造体。
  • BrinMemTuple: デコードされたインデックス項目のメモリ内表現。
  • BrinDesc: BRIN インデックスの記述子。

失敗モードと境界条件

  • スーパーユーザー権限がない場合、エラーとなります。
  • 拡張のバージョンが古い場合、カラム数不一致によりエラーとなります。
  • 指定したリレーションが BRIN インデックスでない場合、エラーとなります。
  • ページが空（PageIsNew）の場合、NULL を返します。
  • ページ内の項目が使用されていない場合、そのオフセットはスキップまたは NULL 値として扱われます。

保守担当者向けメモ

  • 本関数は出力カラム数に厳格です。将来的にカラムを追加する場合は、BRIN_PAGE_ITEMS_V1_12 のようなバージョンチェック定数を更新し、既存の SQL 定義との整合性に注意してください。
  • brin_deform_tuple の結果を tuplestore に書き出す際の型変換ロジックは、インデックスの属性型に依存するため、型定義の変更時には注意が必要です。
