xl_hash_move_page_contents

定義

  • 種別: struct
  • 定義箇所: src/include/access/hash_xlog.h:136

目的

ハッシュインデックスのページ間でタプルを移動させる操作をWAL（Write-Ahead Logging）に記録するためのデータ構造です。

シグネチャまたは形状

typedef struct xl_hash_move_page_contents
{
    uint16      ntups;
    bool        is_prim_bucket_same_wrt;
} xl_hash_move_page_contents;

入力と出力

  • **入力**:
  • ntups: 移動対象となるタプルの数。
  • is_prim_bucket_same_wrt: 移動先のページがプライマリバケットページと同一である場合に true となるフラグ。
  • **出力**: なし（WALレコードのペイロードとして永続化される）。

前提条件と不変条件

  • 本構造体は、ハッシュインデックスのページ再構成操作（タプルの移動）が発生する際に、WALレコードのデータ部として書き込まれます。
  • is_prim_bucket_same_wrt は、リカバリ時に移動先ページのコンテキストを正しく判定するために使用されます。

アルゴリズムまたは振る舞い

1. **書き込み**: ハッシュインデックスの操作（hashovfl.c 等）において、タプル移動が発生した際に本構造体に値を設定し、WALレコードとして書き出します。 2. **リカバリ**: hash_redo 関数がWALレコードを読み込む際、XLogRecGetData を通じて本構造体をキャストして取得します。この情報に基づき、移動元および移動先ページに対して適切な再適用処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元（書き込み側）**: src/backend/access/hash/hashovfl.c（タプル移動処理時）
  • **呼び出し先（リカバリ側）**: src/backend/access/hash/hash_xlog.c（hash_redo 内でのレコード解析）
  • **その他**: src/backend/access/rmgrdesc/hashdesc.c にて、ログのデバッグ出力（hash_desc）のために参照されています。

関連データ構造

  • src/include/access/hash_xlog.h: 本構造体が定義されているヘッダーファイル。

失敗モードと境界条件

  • ntups が 0 の場合、移動対象がないことを示しますが、WALレコードの生成ロジックがこのケースを許容するかは実装依存です。
  • 構造体のパディングやアライメントについては、offsetof を用いた計算（src/include/access/hash_xlog.h:145）により、バイナリ互換性が維持されています。

保守担当者向けメモ

  • 本構造体は、ハッシュインデックスの物理的なページ再構成を再現するための重要なメタデータです。変更を加える際は、hash_redo におけるリカバリロジックと、hash_desc におけるログ出力形式の両方に整合性を持たせてください。
