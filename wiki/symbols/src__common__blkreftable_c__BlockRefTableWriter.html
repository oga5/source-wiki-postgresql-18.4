BlockRefTableWriter

定義

  • 種別: struct
  • 定義箇所: src/common/blkreftable.c:217

目的

ブロック参照テーブルのデータを外部ストレージへシリアライズ（書き出し）するための状態管理構造体です。

シグネチャまたは形状

struct BlockRefTableWriter
{
    BlockRefTableBuffer buffer;
};

入力と出力

  • **入力**: CreateBlockRefTableWriter を介して初期化され、BlockRefTableWriteEntry に渡される BlockRefTableEntry を順次処理します。
  • **出力**: 内部の BlockRefTableBuffer を通じて、登録された io_callback_fn によりバイナリデータとして出力されます。

前提条件と不変条件

  • BlockRefTableWriter は CreateBlockRefTableWriter によってメモリが割り当てられ、初期化されている必要があります。
  • シリアライズされるエントリは、tablespace -> database -> relfilenumber -> forknumber の順序でソートされている必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: CreateBlockRefTableWriter により、I/O コールバック関数を保持した状態で生成されます。 2. **書き込み**: BlockRefTableWriteEntry が呼び出されると、指定された BlockRefTableEntry をバッファ経由でシリアライズします。 3. **終了**: DestroyBlockRefTableWriter により、関連するバッファと構造体が解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/blkreftable.c 内のシリアライズ処理フロー。
  • **呼び出し先**:
  • BlockRefTableWriteEntry (書き込み実行)
  • DestroyBlockRefTableWriter (破棄)
  • CreateBlockRefTableWriter (生成)

関連データ構造

  • BlockRefTableBuffer: 書き込み用バッファとして内部保持。
  • BlockRefTableEntry: 書き込み対象となるデータ単位。

失敗モードと境界条件

  • I/O コールバック関数がエラーを返した場合、書き込み処理は中断されます。
  • メモリ不足時には palloc0 による割り当て失敗が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は io_callback_fn を介して I/O を抽象化しているため、ファイルだけでなくメモリバッファやネットワークストリームへの書き出しにも対応可能です。
  • シリアライズ時のソート順序（BlockRefTableComparator に準拠）が守られていない場合、読み込み時に整合性が保てなくなるため注意してください。
