ExecBitmapIndexScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapIndexscan.c:37

目的

BitmapIndexScan ノードに対して、標準的な ExecProcNode インターフェース経由での呼び出しを禁止し、誤用を防ぐためのガード関数として機能します。

シグネチャまたは形状

static TupleTableSlot *ExecBitmapIndexScan(PlanState *pstate)

入力と出力

  • 入力: PlanState *pstate （実行中のプランノードの状態）
  • 出力: なし（常にエラーを発生させて終了するため、戻り値は到達しません）

前提条件と不変条件

  • 本関数は ExecProcNode として呼び出されることを想定していません。
  • BitmapIndexScan ノードはタプルを直接返さないため、本関数を呼び出すことは実行エンジンにおける設計上の誤りです。

アルゴリズムまたは振る舞い

1. elog(ERROR, ...) を呼び出し、BitmapIndexScan ノードが ExecProcNode 呼び出し規約をサポートしていない旨のメッセージを出力して処理を中断します。 2. コンパイラの警告を避けるためのダミーとして NULL を返しますが、実際にはエラーにより制御が戻ることはありません。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/nodeBitmapIndexscan.c:237 において、indexstate->ss.ps.ExecProcNode に関数ポインタとして代入されています。
  • 呼び出し先: elog（エラー報告用関数）。

関連データ構造

  • PlanState: プランノードの実行状態を保持する構造体。
  • TupleTableSlot: 本来であればタプルを格納する構造体ですが、本関数では使用されません。

失敗モードと境界条件

  • 本関数が呼び出された時点で、常にエラー（ERROR レベル）が発生します。

保守担当者向けメモ

  • 本関数は ExecProcNode インターフェースを意図的に無効化するためのものです。BitmapIndexScan ノードの実行には必ず MultiExecBitmapIndexScan を使用してください。
