LockTableRecurse

定義

  • 種別: function
  • 定義箇所: src/backend/commands/lockcmds.c:116

目的

指定されたリレーション（テーブル）の継承ツリーを辿り、すべての子テーブルに対して指定されたロックモードで再帰的にロックを適用する。

シグネチャまたは形状

void LockTableRecurse(Oid reloid, LOCKMODE lockmode, bool nowait)

入力と出力

  • **入力**:
  • reloid: ロック対象の親リレーションの OID。
  • lockmode: 適用するロックモード（LOCKMODE）。
  • nowait: true の場合、ロックが即座に取得できなければエラーを発生させる。
  • **出力**: なし（void）。

前提条件と不変条件

  • 親リレーションは本関数呼び出し前に既にロックされていることが前提である。
  • 継承ツリーの探索には find_all_inheritors を使用する。

アルゴリズムまたは振る舞い

1. find_all_inheritors を呼び出し、対象リレーションのすべての子テーブルの OID リストを取得する。 2. リスト内の各子テーブルに対して以下を繰り返す。

  • 親自身（childreloid == reloid）であればスキップする。
  • nowait が false の場合、LockRelationOid でロックを獲得する。
  • nowait が true の場合、ConditionalLockRelationOid を試行し、失敗した場合はエラー（ERRCODE_LOCK_NOT_AVAILABLE）を報告する。
  • ロック獲得後、SearchSysCacheExists1 を使用して子テーブルが並行して削除されていないか確認する。削除されている場合はロックを解放し、処理をスキップする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/lockcmds.c 内の LockTableCommand および LockViewRecurse から呼び出される。
  • **呼び出し先**:
  • find_all_inheritors
  • LockRelationOid
  • ConditionalLockRelationOid
  • get_rel_name
  • ereport
  • SearchSysCacheExists1
  • UnlockRelationOid

関連データ構造

  • Oid: リレーション識別子。
  • LOCKMODE: ロックの強度を指定する列挙型。
  • List: 子テーブルの OID を保持するリスト。

失敗モードと境界条件

  • **ロック競合**: nowait が有効な場合、ロックが取得できないと ERROR を発生させる。
  • **並行削除**: ロック取得後にテーブルが削除された場合、システムキャッシュの存在確認により検出し、ロックを解放して安全に処理を継続する。
  • **名前解決失敗**: get_rel_name が失敗した場合（テーブルが削除された直後など）、エラー報告をスキップする。

保守担当者向けメモ

  • 本関数は継承関係のみを対象とする。ビューの再帰的ロックについては LockViewRecurse が担当する。
  • ロック取得後のシステムキャッシュ確認は、競合状態（Race Condition）を避けるための重要な不変条件である。
