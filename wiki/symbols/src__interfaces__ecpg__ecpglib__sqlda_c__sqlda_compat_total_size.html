sqlda_compat_total_size

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/sqlda.c:156

目的

Informix 互換モードにおける SQLDA 構造体およびそのデータ領域を格納するために必要な、合計メモリサイズを算出します。

シグネチャまたは形状

static long sqlda_compat_total_size(const PGresult *res, int row, enum COMPAT_MODE compat)

入力と出力

  • **入力**:
  • res: 取得した結果セット (PGresult)。
  • row: 対象となる行番号。負の場合はメタデータ領域のみを計算対象とします。
  • compat: 互換モード指定 (enum COMPAT_MODE)。
  • **出力**:
  • 必要なメモリサイズを long 型で返します。

前提条件と不変条件

  • res は有効な PGresult ポインタである必要があります。
  • row が 0 以上の場合、sqlda_common_total_size を介してデータ領域のサイズが加算されます。

アルゴリズムまたは振る舞い

1. sqlda_compat_empty_size(res) を呼び出し、構造体本体およびメタデータに必要な初期サイズを取得します。 2. row が負の値であれば、その初期サイズをそのまま返します。 3. row が 0 以上であれば、sqlda_common_total_size を呼び出し、指定された行のデータ格納に必要なサイズを初期サイズに加算して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/sqlda.c 内の関数（例: ecpg_build_compat_sqlda 等）から呼び出されます。
  • **呼び出し先**:
  • sqlda_compat_empty_size
  • sqlda_common_total_size

関連データ構造

  • PGresult: libpq の結果セット構造体。
  • SQLDA: Informix 互換の記述子領域。
  • COMPAT_MODE: 互換モードを定義する列挙型。

失敗モードと境界条件

  • row < 0 の場合、データ領域の計算をスキップし、メタデータのみのサイズを返します。
  • PGresult が空の場合の挙動は、内部で呼び出される sqlda_compat_empty_size および sqlda_common_total_size の実装に依存します。

保守担当者向けメモ

  • 本関数は Informix 互換モード専用です。ネイティブモードの場合は sqlda_native_total_size を使用してください。
  • メモリレイアウトの変更が必要な場合は、sqlda_common_total_size との整合性に注意してください。
