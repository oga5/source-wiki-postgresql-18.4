A_Expr_Kind

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/parsenodes.h:327

目的

SQLのパースツリーにおいて、A_Expr（式）ノードがどのような種類の演算や構文構造を表すかを識別するために使用されます。

シグネチャまたは形状

typedef enum A_Expr_Kind
{
    AEXPR_OP, AEXPR_OP_ANY, AEXPR_OP_ALL,
    AEXPR_DISTINCT, AEXPR_NOT_DISTINCT,
    AEXPR_NULLIF, AEXPR_IN,
    AEXPR_LIKE, AEXPR_ILIKE, AEXPR_SIMILAR,
    AEXPR_BETWEEN, AEXPR_NOT_BETWEEN,
    AEXPR_BETWEEN_SYM, AEXPR_NOT_BETWEEN_SYM
} A_Expr_Kind;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: A_Expr 構造体内の kind フィールドとして保持されます。

前提条件と不変条件

特定の列挙値には、対応する演算子名（name フィールド）に関する制約が存在します。

  • AEXPR_DISTINCT / AEXPR_NOT_DISTINCT / AEXPR_NULLIF: name は "=" である必要があります。
  • AEXPR_IN: name は "=" または "<>" である必要があります。
  • AEXPR_LIKE: name は "~~" または "!~~" である必要があります。
  • AEXPR_ILIKE: name は "~~*" または "!~~*" である必要があります。
  • AEXPR_SIMILAR: name は "~" または "!~" である必要があります。
  • AEXPR_BETWEEN 系: name は対応するキーワード文字列である必要があります。

アルゴリズムまたは振る舞い

パーサーがSQL文を解析する際、演算子や特定の構文（BETWEEN等）を検出すると、この列挙値を用いて A_Expr ノードを初期化します。後続の解析フェーズでは、この値に基づいて適切な意味解析処理へ分岐します。

呼び出し元と呼び出し先

  • 呼び出し元（生成側）: makeA_Expr および makeSimpleA_Expr（src/backend/nodes/makefuncs.c）がこの型を引数として受け取り、ノードを生成します。
  • 呼び出し先（参照側）: src/backend/nodes/outfuncs.c 等で、ノードのシリアライズやデバッグ出力時に値の判定に使用されます。

関連データ構造

  • A_Expr: この列挙型を kind フィールドとして保持する構造体。

失敗モードと境界条件

  • 未定義の列挙値が渡された場合、outfuncs.c 等の処理において elog(ERROR, ...) が発生し、処理が中断されます。

保守担当者向けメモ

  • 新しい演算子構文を追加する場合は、本列挙型に値を追加し、関連する makefuncs.c や出力関数を更新してください。
  • 各列挙値が要求する name フィールドの制約は、コード上のコメントとして記述されていますが、実行時のバリデーションロジックと整合しているか確認が必要です。
