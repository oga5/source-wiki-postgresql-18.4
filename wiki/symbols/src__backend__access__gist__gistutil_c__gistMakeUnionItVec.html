gistMakeUnionItVec

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistutil.c:154

目的

インデックスタプルのベクトルから、GiSTインデックスの親ノードに格納するための「和（Union）」キーを計算し、属性ごとに配列へ格納する。

シグネチャまたは形状

void gistMakeUnionItVec(GISTSTATE *giststate, IndexTuple *itvec, int len, Datum *attr, bool *isnull)

入力と出力

  • **入力**:
  • giststate: GiSTインデックスの状態管理構造体。
  • itvec: 処理対象となる IndexTuple の配列。
  • len: itvec 内のタプル数。
  • **出力**:
  • attr: 計算されたUnionキー（Datum）を格納する配列。
  • isnull: 各属性がNULLであるかを示すフラグ配列。

前提条件と不変条件

  • giststate は初期化済みであること。
  • attr および isnull 配列は、インデックスの属性数分以上の領域が確保されていること。
  • itvec に含まれる各タプルは、giststate->leafTupdesc に適合する形式であること。

アルゴリズムまたは振る舞い

1. GistEntryVector をメモリ上に確保する。 2. インデックスの各属性（列）について以下を繰り返す：

  • itvec 内の全タプルから、当該列の値を抽出する。
  • NULL値を除外して evec に収集する。
  • 全てがNULLの場合、結果の attr をNULLとしてマークする。
  • 値が1つだけの場合、unionFn の要件を満たすためにダミーとして自身を複製し、要素数を2にする。
  • giststate->unionFn を呼び出し、収集した全値のUnionを計算して attr に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistsplit.c:64
  • **呼び出し先**:
  • index_getattr (マクロ)
  • gistdentryinit
  • FunctionCall2Coll (GiSTの union サポート関数を実行)

関連データ構造

  • GISTSTATE: インデックスの操作に必要なサポート関数や型情報を含む。
  • GistEntryVector: 複数の GISTENTRY を保持し、union 関数へ渡すための構造体。
  • IndexTuple: インデックス内の各レコード。

失敗モードと境界条件

  • **全要素がNULL**: 該当属性の isnull が true に設定される。
  • **要素数が1**: unionFn が2つ以上の入力を期待する場合に備え、内部的に要素を複製して処理を継続する。
  • **メモリ不足**: palloc によるメモリ確保失敗時にエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はGiSTのインデックス構築およびページ分割処理における中核的なデータ変換ロジックである。
  • unionFn はインデックスの各属性に対して定義されたサポート関数であり、この関数の挙動がインデックスの圧縮率や検索性能に直結する。
