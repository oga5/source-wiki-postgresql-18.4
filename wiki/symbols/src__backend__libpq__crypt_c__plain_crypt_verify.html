plain_crypt_verify

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/crypt.c:256

目的

クライアントから平文で提供されたパスワードを、システムカタログに保存されているハッシュ形式（SCRAM-SHA-256 または MD5）に基づいて検証する。

シグネチャまたは形状

int plain_crypt_verify(const char *role, const char *shadow_pass,
                       const char *client_pass,
                       const char **logdetail);

入力と出力

  • **入力**:
  • role: ユーザー名。
  • shadow_pass: システムカタログに保存されているパスワードハッシュ文字列。
  • client_pass: クライアントから送信された平文パスワード。
  • logdetail: 認証失敗時の詳細メッセージを格納するためのポインタ。
  • **出力**:
  • STATUS_OK (0): 検証成功。
  • STATUS_ERROR (-1): 検証失敗、または形式不明。

前提条件と不変条件

  • shadow_pass は get_password_type で識別可能な形式であること。
  • パスワードの保存形式として平文（PASSWORD_TYPE_PLAINTEXT）は想定されていない。
  • 認証失敗時の詳細情報は、セキュリティ上の理由からクライアントには返さず、サーバーログ出力用として logdetail に格納される。

アルゴリズムまたは振る舞い

1. get_password_type(shadow_pass) を呼び出し、保存されているハッシュ形式を判定する。 2. **SCRAM-SHA-256 の場合**: scram_verify_plain_password を呼び出し、結果を返す。 3. **MD5 の場合**:

  • pg_md5_encrypt を使用して、client_pass と role から MD5 ハッシュを生成する。
  • timingsafe_bcmp を使用して、生成されたハッシュと shadow_pass を比較する（タイミング攻撃対策）。

    • 4. 形式が不明な場合、または一致しない場合は STATUS_ERROR を返し、logdetail にエラーメッセージを設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/passwordcheck/passwordcheck.c
  • src/backend/commands/user.c
  • src/backend/libpq/auth.c
  • **呼び出し先**:
  • get_password_type
  • scram_verify_plain_password
  • pg_md5_encrypt
  • timingsafe_bcmp
  • psprintf

関連データ構造

  • PasswordType (列挙型: PASSWORD_TYPE_SCRAM_SHA_256, PASSWORD_TYPE_MD5, PASSWORD_TYPE_PLAINTEXT)

失敗モードと境界条件

  • get_password_type が不明な形式を返した場合、認証は失敗する。
  • MD5 ハッシュ生成時にエラーが発生した場合、そのエラー内容が logdetail に反映される。
  • パスワード不一致時は、ユーザー名を含むエラーメッセージが logdetail に設定される。

保守担当者向けメモ

  • 本関数は平文パスワードを受け取るため、SCRAM や MD5 への変換ロジックを内包している。
  • timingsafe_bcmp を使用している箇所は、サイドチャネル攻撃を防ぐための重要な実装であるため、変更時は注意すること。
