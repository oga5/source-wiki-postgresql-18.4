ArrayGetOffset

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arrayutils.c:31

目的

多次元配列における指定されたインデックス位置を、メモリ上の線形オフセット（0から始まる要素番号）に変換します。

シグネチャまたは形状

int ArrayGetOffset(int n, const int *dim, const int *lb, const int *indx)

入力と出力

  • **入力**:
  • n: 配列の次元数。
  • dim: 各次元のサイズを格納した配列。
  • lb: 各次元の下限値（lower bound）を格納した配列。
  • indx: 目的の要素の各次元におけるインデックス。
  • **出力**:
  • 計算された線形オフセット（int 型）。

前提条件と不変条件

  • dim、lb、indx は少なくとも n 個の要素を持つ必要があります。
  • 本関数はインデックスの境界チェックを行いません。呼び出し元が事前に ArrayCheckBoundsSafe 等を用いて妥当性を確認していることが前提です。

アルゴリズムまたは振る舞い

1. 最下位の次元（n-1）から順に、各次元のインデックスと下限値の差分を計算します。 2. その差分に、その次元より下位の次元サイズの積（scale）を乗算します。 3. 全次元について加算を行い、最終的な線形オフセットを算出します。 4. scale はループごとに更新され、多次元構造を線形化するための重みとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/arrayfuncs.c 内の複数の箇所（配列の要素アクセスやコピー処理など）。
  • **呼び出し先**:
  • 特筆すべき関数呼び出しはありません（純粋な算術計算関数です）。

関連データ構造

  • ArrayType: PostgreSQLの配列内部表現。本関数は、この構造体に含まれる次元情報や下限値情報を利用して計算を行います。

失敗モードと境界条件

  • **境界条件**: n=0 の場合、ループは実行されず offset=0 を返します。
  • **失敗モード**: インデックスが配列の範囲外である場合、計算結果は負の値や配列サイズを超える値になりますが、本関数自体はエラーを検知しません。

保守担当者向けメモ

  • 本関数は mda_ 系関数と同様に、呼び出し元が次元や境界の妥当性を保証していることを前提とした低レベルな計算ルーチンです。
  • 非常に頻繁に呼び出されるため、計算ロジックを変更する際はパフォーマンスへの影響を考慮してください。
