hmac_finish

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/px-hmac.c:99

目的

HMAC 計算の最終段階を実行し、内部ダイジェストの結果に対して opad（Outer Pad）を用いた二段目のハッシュ計算を行い、最終的な HMAC 値を生成します。

シグネチャまたは形状

static void hmac_finish(PX_HMAC *h, uint8 *dst)

入力と出力

  • **入力**:
  • h: 初期化および更新済みの PX_HMAC 構造体へのポインタ。
  • dst: 最終的な HMAC 値を格納するためのバッファ（呼び出し元が十分な領域を確保している必要がある）。
  • **出力**: なし（dst に結果が書き込まれる）。

前提条件と不変条件

  • h は有効な PX_HMAC 構造体であり、既に hmac_init および hmac_update が呼び出されていること。
  • h->md に関連付けられたダイジェストコンテキストが有効であること。

アルゴリズムまたは振る舞い

1. 現在のダイジェストコンテキストのブロックサイズおよび結果サイズを取得します。 2. 一時バッファを割り当て、現在のダイジェスト計算を完了（px_md_finish）させ、その結果を一時バッファに保存します。 3. ダイジェストコンテキストをリセットし、opad を適用します。 4. 一時バッファの内容をダイジェストコンテキストに更新（px_md_update）します。 5. 最終的なハッシュ値を dst に書き込みます。 6. セキュリティのため、一時バッファを px_memset でゼロクリアし、メモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/px-hmac.c 内で PX_HMAC 構造体の finish 関数ポインタとして登録されています。具体的な呼び出し元コードの詳細は提供されていません。
  • **呼び出し先**: px_md_block_size, px_md_result_size, px_md_finish, px_md_reset, px_md_update, px_memset, palloc, pfree。

関連データ構造

  • PX_HMAC: HMAC 操作の状態を保持する構造体。
  • PX_MD: 基盤となるメッセージダイジェストコンテキスト。

失敗モードと境界条件

  • palloc がメモリ不足により失敗した場合、PostgreSQL のエラーハンドリングにより処理が中断されます。
  • dst バッファが結果サイズに対して小さい場合、バッファオーバーフローが発生する可能性があります（呼び出し元で適切なサイズ確保が必須）。

保守担当者向けメモ

  • 本関数は RFC 2104 に準拠しています。
  • px_memset を使用したゼロクリアは、機密情報（中間ハッシュ）のメモリ残留を防ぐための重要なセキュリティ対策です。変更時はこのクリア処理を維持してください。
