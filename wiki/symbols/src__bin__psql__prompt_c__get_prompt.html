get_prompt

定義

  • 種別: function
  • 定義箇所: src/bin/psql/prompt.c:67

目的

psql の対話型シェルにおいて、現在のセッション状態に基づいたプロンプト文字列を動的に生成して返します。

シグネチャまたは形状

char *get_prompt(promptStatus_t status, ConditionalStack cstack);

入力と出力

  • **入力**:
  • status: プロンプトの種類（PROMPT_READY, PROMPT_CONTINUE 等）。
  • cstack: 条件付き実行スタック（PROMPT_READY 時の条件判定に使用）。
  • **出力**:
  • 生成されたプロンプト文字列を保持する静的バッファ（destination）へのポインタ。

前提条件と不変条件

  • pset 構造体が初期化されている必要があります。
  • 返されるポインタは静的バッファを指すため、呼び出し元は結果を即座に使用するか、別のメモリ領域へコピーする必要があります。
  • プロンプトの最大長は MAX_PROMPT_SIZE（256バイト）に制限されます。

アルゴリズムまたは振る舞い

1. status に応じて pset から対応するテンプレート文字列（prompt1, prompt2, prompt3）を選択します。 2. テンプレート文字列を1文字ずつ走査し、% で始まるエスケープシーケンスを解釈します。 3. % に続く文字に応じて、データベース名、ホスト名、ポート、ユーザー名、PID、パイプライン状態などの情報を libpq API や環境変数から取得し、バッファに連結します。 4. %R シーケンスでは、現在の status や接続状態、条件スタックの状態に基づいた文字（@, !, =, ^ 等）を生成します。 5. 生成された文字列を destination に格納し、そのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/copy.c: get_prompt(PROMPT_COPY, NULL) を呼び出し。
  • src/bin/psql/mainloop.c: get_prompt(PROMPT_READY, cond_stack) および get_prompt(prompt_status, cond_stack) を呼び出し。
  • **呼び出し先**:
  • libpq API（PQdb, PQhost, PQport, PQbackendPID, PQpipelineStatus 等）
  • session_username()
  • GetVariable()
  • conditional_active()

関連データ構造

  • promptStatus_t: プロンプトの状態を定義する列挙型。
  • pset: psql の設定および接続情報を保持するグローバル構造体。
  • ConditionalStack: 条件付き実行の状態を管理するスタック。

失敗モードと境界条件

  • pset.db が NULL の場合、接続情報に依存するエスケープシーケンス（%p, %n 等）は空またはデフォルト値として処理されます。
  • 文字列が MAX_PROMPT_SIZE を超える場合、切り詰められます。
  • 存在しないエスケープシーケンスや不正な入力に対する明示的なエラーハンドリングはソース内に記載されていません。

保守担当者向けメモ

  • 新しいエスケープシーケンスを追加する場合は、switch (*p) ブロック内にケースを追加してください。
  • destination が静的バッファであるため、再入可能性（reentrancy）はありません。
  • マルチバイト文字の表示幅計算については、PQdsplen の挙動に依存します。
