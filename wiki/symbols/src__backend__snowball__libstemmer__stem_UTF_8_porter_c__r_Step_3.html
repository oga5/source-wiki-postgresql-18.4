r_Step_3

定義

  • **種別**: function
  • **定義箇所**: src/backend/snowball/libstemmer/stem_UTF_8_porter.c:449

目的

Porter Stemming アルゴリズムにおける第 3 段階の処理を実行し、特定の接尾辞を置換または削除して語幹を還元する。

シグネチャまたは形状

static int r_Step_3(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（ステミング環境を保持する構造体）
  • **出力**: 処理が成功した場合は 1、失敗または条件不一致の場合は 0、エラー発生時は負の値を返す。

前提条件と不変条件

  • z が有効な SN_env 構造体を指していること。
  • 内部で r_R1 関数を呼び出し、対象の接尾辞が領域 R1 内にあることを検証する。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置 z->c を z->ket に設定する。 2. 文字列の末尾付近の文字コードに基づき、高速な事前チェックを行う。 3. find_among_b を使用して、定義された接尾辞リスト a_4（7要素）との後方マッチングを行う。 4. マッチした場合、z->bra を現在のカーソル位置に設定し、r_R1 を実行して領域 R1 内であることを確認する。 5. マッチした接尾辞の種類（among_var）に応じて以下のいずれかを行う：

  • 1: 接尾辞を s_19 に置換。
  • 2: 接尾辞を s_20 に置換。
  • 3: 接尾辞を削除。

呼び出し元と呼び出し先

  • **呼び出し元**: porter_UTF_8_stem（および各言語実装のステミング関数から間接的に参照される可能性がある）。
  • **呼び出し先**: r_R1, find_among_b, slice_from_s, slice_del。

関連データ構造

  • SN_env: ステミング処理の状態（カーソル、領域境界など）を保持。
  • a_4: find_among_b で使用される接尾辞の探索用構造体。

失敗モードと境界条件

  • 文字列長が不足している場合や、R1 領域外である場合は 0 を返して終了する。
  • slice_from_s や slice_del が内部でエラー（負の値）を返した場合、その値をそのまま伝播させる。
