replace_variables

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/prepare.c:103

目的

SQL文内のホスト変数（: または ?）を、PostgreSQLが認識可能な $n 形式のプレースホルダに変換します。

シグネチャまたは形状

static bool replace_variables(char **text, int lineno);

入力と出力

  • **入力**:
  • text: 変換対象のSQL文字列へのポインタ。変換後の新しい文字列で上書きされます。
  • lineno: エラー発生時に使用される行番号。
  • **出力**:
  • 成功時は true、メモリ確保失敗時は false を返します。

前提条件と不変条件

  • text が指す文字列は ecpg_alloc で確保されたメモリ領域である必要があります（関数内で ecpg_free が呼び出されるため）。
  • 文字列内のシングルクォートで囲まれた範囲は、変数置換の対象外となります。
  • ::（型キャスト演算子）は変数と見なされず、スキップされます。

アルゴリズムまたは振る舞い

1. 文字列を走査し、シングルクォート内か、: または ? 以外の文字であればスキップします。 2. :: を検出した場合は、変数と誤認しないようスキップします。 3. 変数（: または ?）を検出した場合：

  • $n 形式の文字列（n はカウンタ）を生成します。
  • isvarchar を用いて変数名の長さを特定し、元の文字列から該当部分を置換します。
  • ecpg_alloc で新しいメモリ領域を確保し、旧領域を ecpg_free で解放します。

    • 4. 変換後の文字列を *text に格納し、カウンタをインクリメントして走査を継続します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/prepare.c 内の prepare_common 等から呼び出されます。
  • **呼び出し先**:
  • ecpg_alloc
  • ecpg_free
  • isvarchar（マクロまたは関数）

関連データ構造

  • char **text: 変換対象のSQLコマンド文字列。

失敗モードと境界条件

  • **メモリ不足**: ecpg_alloc が失敗した場合、false を返して終了します。
  • **文字列末尾**: 文字列の終端に達した際のポインタ操作が適切に行われるよう制御されています。

保守担当者向けメモ

  • ecpg_alloc および ecpg_free を使用しているため、メモリ管理の整合性に注意してください。
  • isvarchar の定義に依存して変数名の認識範囲が決まるため、SQL構文の拡張時にはこの判定ロジックを確認してください。
