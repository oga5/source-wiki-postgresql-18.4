bbsink_forward_end_manifest

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_sink.c:100

目的

バックアップマニフェストの終了処理を、bbsink チェーン内の次のシンクへ転送する。

シグネチャまたは形状

void bbsink_forward_end_manifest(bbsink *sink)

入力と出力

  • **入力**: sink (現在の bbsink オブジェクトへのポインタ)
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • sink->bbs_next が NULL であってはならない（後続のシンクが存在することが必須）。

アルゴリズムまたは振る舞い

1. sink->bbs_next が存在することをアサーションで検証する。 2. sink->bbs_next を引数として bbsink_end_manifest を呼び出し、処理を後続のシンクへ委譲する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/basebackup_to_shell/basebackup_to_shell.c
  • src/backend/backup/basebackup_server.c
  • **呼び出し先**:
  • bbsink_end_manifest (後続のシンクのメソッド)
  • **参照元（メソッド割り当て）**:
  • src/backend/backup/basebackup_gzip.c
  • src/backend/backup/basebackup_lz4.c
  • src/backend/backup/basebackup_progress.c
  • src/backend/backup/basebackup_throttle.c
  • src/backend/backup/basebackup_zstd.c

関連データ構造

  • bbsink: バックアップデータの転送先を抽象化した構造体。bbs_next メンバを通じてチェーンを形成する。

失敗モードと境界条件

  • sink->bbs_next が NULL の場合、アサーション違反によりプログラムが異常終了する。

保守担当者向けメモ

  • 本関数は、独自の bbsink 実装において、マニフェスト終了時に特別な処理が不要な場合にコールバックとして割り当てるためのヘルパーである。
  • bbsink_end_manifest を直接呼び出すのではなく、必ずこの転送関数を経由させることで、シンクチェーンの整合性を維持すること。
