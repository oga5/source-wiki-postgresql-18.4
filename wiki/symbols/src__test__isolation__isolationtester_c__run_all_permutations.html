run_all_permutations

定義

  • 種別: function
  • 定義箇所: src/test/isolation/isolationtester.c:405

目的

テスト仕様（TestSpec）に含まれる全セッションのステップを網羅する、すべての可能な実行順列を生成して実行する。

シグネチャまたは形状

static void run_all_permutations(TestSpec *testspec)

入力と出力

  • **入力**: TestSpec *testspec（実行対象のテスト仕様構造体）
  • **出力**: なし（副作用として各順列の実行を行う）

前提条件と不変条件

  • testspec が有効なポインタであること。
  • testspec 内の各セッションにステップが定義されていること。

アルゴリズムまたは振る舞い

1. 全セッションのステップ数を合計し、作業用メモリ（steps, stepptrs）を確保する。 2. 各セッションの進行状況を管理する piles 配列を初期化する。 3. run_all_permutations_recurse を呼び出し、再帰的にステップの順列を生成・実行する。 4. 処理完了後、確保したメモリを解放する。

  • 順列生成のロジックは、各セッションを「山（pile）」と見なし、すべての山が空になるまでステップを順次選択する手法をとる。

呼び出し元と呼び出し先

  • **呼び出し元**: run_testspec（src/test/isolation/isolationtester.c:399）
  • **呼び出し先**: run_all_permutations_recurse

関連データ構造

  • TestSpec: テスト全体の仕様を保持する構造体。
  • PermutationStep: 各ステップの情報を保持する構造体。

失敗モードと境界条件

  • メモリ確保（pg_malloc0, pg_malloc）に失敗した場合、プログラムは終了する。
  • ステップ数が 0 の場合、順列生成は行われない。

保守担当者向けメモ

  • 本関数は順列生成の初期化と後始末を担当する。再帰的な探索ロジックは run_all_permutations_recurse に依存している。
  • メモリ管理は pg_malloc 系関数を使用しているため、呼び出し元でのエラーハンドリングは不要。
