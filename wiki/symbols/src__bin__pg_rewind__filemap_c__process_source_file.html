process_source_file

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/filemap.c:279

目的

同期元（ソース）のファイル情報を収集し、ハッシュテーブル（filehash）に登録して同期計画の基礎データを作成する。

シグネチャまたは形状

void process_source_file(const char *path, file_type_t type, size_t size, const char *link_target)

入力と出力

  • **入力**:
  • path: ファイルパス。
  • type: ファイル種別（file_type_t）。
  • size: ファイルサイズ。
  • link_target: シンボリックリンクのターゲットパス（存在する場合）。
  • **出力**: なし（副作用としてハッシュテーブルを更新）。

前提条件と不変条件

  • pg_wal はシンボリックリンクであってもディレクトリとして扱う。
  • データファイル（isRelDataFile が真）は必ず FILE_TYPE_REGULAR である必要がある。

アルゴリズムまたは振る舞い

1. **正規化**: path が pg_wal であり、かつ type が FILE_TYPE_SYMLINK の場合、type を FILE_TYPE_DIRECTORY に上書きする。 2. **検証**: データファイルが通常ファイルでない場合、pg_fatal を呼び出し異常終了する。 3. **登録**: insert_filehash_entry(path) を呼び出し、ハッシュテーブルから該当エントリを取得する。 4. **重複チェック**: 既に source_exists が真であれば、重複ソースファイルとして pg_fatal を呼び出す。 5. **更新**: エントリに source_exists（真）、source_type、source_size を設定し、link_target があれば複製して格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/pg_rewind.c:475 において、source->traverse_files のコールバックとして渡されている。
  • **呼び出し先**: insert_filehash_entry（ハッシュテーブル操作）、isRelDataFile（パス判定）、pg_fatal（エラー終了）、pg_strdup（メモリ確保）。

関連データ構造

  • file_entry_t: ファイルのメタデータと同期状態を保持する構造体。
  • filehash: ファイルパスをキーとするハッシュテーブル。

失敗モードと境界条件

  • **重複エラー**: 同一パスが二度登録された場合、pg_fatal によりプロセスが終了する。
  • **型不整合**: データファイルが通常ファイルでない場合、pg_fatal によりプロセスが終了する。

保守担当者向けメモ

  • 本関数は process_target_file と対になる処理を行う。修正時は filemap.h で定義される file_entry_t の構造変更に注意すること。
  • pg_wal の扱いに関するロジックは、ターゲット側の処理と整合性を保つ必要がある。
