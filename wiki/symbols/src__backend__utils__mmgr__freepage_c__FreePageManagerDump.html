FreePageManagerDump

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/freepage.c:423

目的

FreePageManager の現在の状態（メタデータ、B-tree 構造、リサイクルリスト、フリーリスト）を人間が読み取り可能な文字列形式でダンプし、デバッグや状態確認を支援する。

シグネチャまたは形状

char *FreePageManagerDump(FreePageManager *fpm)

入力と出力

  • **入力**: fpm - ダンプ対象の FreePageManager 構造体へのポインタ。
  • **出力**: char * - 状態情報が格納された動的に割り当てられた文字列（StringInfo のデータ）。呼び出し元はメモリの解放に責任を持つ必要がある。

前提条件と不変条件

  • fpm は有効な FreePageManager 構造体を指している必要がある。
  • fpm_segment_base(fpm) が、管理対象の共有メモリセグメントのベースアドレスを正しく解決できること。

アルゴリズムまたは振る舞い

1. StringInfoData を初期化し、管理対象のメタデータ（自己参照オフセット、最大連続ページ数）を記録する。 2. B-tree の深さが 0 より大きい場合は FreePageManagerDumpBtree を呼び出し、構造をダンプする。深さが 0 の場合は singleton_npages の情報を記録する。 3. btree_recycle リストが存在する場合、FreePageManagerDumpSpans を用いてリサイクル対象のスパンをダンプする。 4. FPM_NUM_FREELISTS に定義されたすべてのフリーリストを走査し、空でないリストについて FreePageManagerDumpSpans を用いてスパン情報をダンプする。 5. 構築した文字列バッファのデータを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は提供されていない。
  • **呼び出し先**:
  • initStringInfo, appendStringInfo, appendStringInfoString (lib/stringinfo.h)
  • fpm_segment_base, relptr_offset, relptr_access, relptr_is_null (utils/relptr.h)
  • FreePageManagerDumpBtree
  • FreePageManagerDumpSpans

関連データ構造

  • FreePageManager: 管理対象のメイン構造体。
  • FreePageSpanLeader: フリーリストやリサイクルリストのノード。
  • StringInfoData: 出力文字列を構築するためのバッファ。

失敗モードと境界条件

  • fpm が NULL の場合、fpm_segment_base 等の内部関数でクラッシュする可能性がある。
  • メモリ不足時には StringInfo の内部的なメモリ割り当てが失敗する可能性がある。

保守担当者向けメモ

  • 本関数は StringInfo を使用して動的にメモリを確保して文字列を返すため、呼び出し側で pfree() 等によるメモリ解放が必須である。
  • FPM_EXTRA_ASSERTS が有効な環境では、内部的な整合性チェックと併用されることが想定される。
