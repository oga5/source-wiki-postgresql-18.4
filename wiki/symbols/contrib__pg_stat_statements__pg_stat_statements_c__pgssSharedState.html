pgssSharedState

定義

  • 種別: struct
  • 定義箇所: contrib/pg_stat_statements/pg_stat_statements.c:246

目的

pg_stat_statements 拡張機能において、共有メモリ上でクエリ統計のハッシュテーブル、クエリテキストの外部ファイル管理、およびグローバルな統計情報を一元管理するための状態構造体です。

シグネチャまたは形状

typedef struct pgssSharedState
{
	LWLock	   *lock;			/* ハッシュテーブルの検索/変更を保護 */
	double		cur_median_usage;	/* ハッシュテーブル内の現在のメディアン使用量 */
	Size		mean_query_len; /* 現在の平均クエリテキスト長 */
	slock_t		mutex;			/* 以下のフィールドのみを保護 */
	Size		extent;			/* クエリファイルの現在のエクステント */
	int			n_writers;		/* クエリファイルへのアクティブな書き込み数 */
	int			gc_count;		/* クエリファイルのガベージコレクション回数 */
	pgssGlobalStats stats;		/* pgss のグローバル統計 */
} pgssSharedState;

入力と出力

本構造体は関数ではなくデータ構造であるため、入力と出力の概念は適用されません。共有メモリ領域として確保され、モジュール内の各フック関数から参照・更新されます。

前提条件と不変条件

  • lock (LWLock) はハッシュテーブルの構造変更（エントリの追加・削除）時に取得する必要があります。
  • mutex (SpinLock) は extent、n_writers、gc_count、stats の更新時に取得する必要があります。
  • デッドロックを回避するため、lock を保持した状態で mutex を取得することは可能ですが、逆の順序での取得は禁止されています。

アルゴリズムまたは振る舞い

  • **ハッシュテーブル管理**: lock を用いて、クエリ統計を保持するハッシュテーブルへの安全なアクセスを提供します。
  • **クエリテキスト管理**: クエリ文字列は共有メモリ節約のため外部ファイルに保存されます。extent や n_writers を通じてファイル領域の管理およびガベージコレクションの制御を行います。
  • **統計集計**: stats フィールドを通じて、システム全体での統計情報の集計値を保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: pgss_shmem_startup にて初期化され、pgss_store 等の統計更新処理から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • pgssEntry: ハッシュテーブルの各要素。
  • pgssGlobalStats: pgssSharedState 内に含まれるグローバル統計構造体。
  • LWLock, slock_t: 排他制御に使用されるプリミティブ。

失敗モードと境界条件

  • 共有メモリ不足時、pgss_shmem_request による領域確保が失敗する可能性があります。
  • クエリテキストの外部ファイル書き込み時にディスク容量や権限によるエラーが発生する可能性があります。

保守担当者向けメモ

  • クエリテキストは外部ファイルに保存されるため、ファイル操作とメモリ上のオフセット管理の同期が非常に複雑です。修正を加える際は、gc_qtexts 等のガベージコレクションロジックへの影響を十分に考慮してください。
  • nesting_level 等の変数と併用し、再帰的なクエリ呼び出しが統計を二重計上しないよう注意が必要です。
