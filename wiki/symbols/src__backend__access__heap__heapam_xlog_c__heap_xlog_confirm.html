heap_xlog_confirm

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_xlog.c:957

目的

WALリプレイ時、指定されたタプルが実際に挿入されたことを確定させるために、タプルの t_ctid を自身の位置（ブロック番号とオフセット番号）で更新します。

シグネチャまたは形状

static void heap_xlog_confirm(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（WALレコード情報を含む）
  • **出力**: なし（副作用としてバッファの内容を更新）

前提条件と不変条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返す場合のみ、ページ更新処理が実行されます。
  • 対象のオフセット番号 offnum がページ内の有効なアイテムポインタを指している必要があります。無効な場合は PANIC で停止します。

アルゴリズムまたは振る舞い

1. XLogReadBufferForRedo を呼び出し、対象のヒープページをバッファに読み込みます。 2. レコードから取得した offnum を使用して、ページ内の ItemId を特定します。 3. ItemId が正常であることを確認し、該当するタプルヘッダーを取得します。 4. タプルの t_ctid を、現在のブロック番号と offnum で上書きします。 5. ページのLSNをレコードの終了LSNで更新し、バッファをダーティとしてマークします。 6. 最後にバッファのロックを解除し、解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_xlog.c 内のWALリプレイディスパッチャから呼び出されます。
  • **呼び出し先**: XLogReadBufferForRedo, BufferGetPage, PageGetMaxOffsetNumber, PageGetItemId, ItemIdIsNormal, PageGetItem, ItemPointerSet, BufferGetBlockNumber, PageSetLSN, MarkBufferDirty, BufferIsValid, UnlockReleaseBuffer。

関連データ構造

  • XLogReaderState: WALレコードの読み取り状態。
  • xl_heap_confirm: 挿入確定情報を含むWALレコード構造体。
  • Buffer: ヒープページを保持するバッファ。
  • HeapTupleHeader: タプルのヘッダー情報。

失敗モードと境界条件

  • PageGetMaxOffsetNumber(page) < offnum または !ItemIdIsNormal(lp) の場合、整合性エラーとして PANIC を発生させます。

保守担当者向けメモ

  • 本関数は、挿入操作の確定という特定のWALイベントを処理します。新しいWALレコードタイプを追加する際は、heap_redo 等のディスパッチャ経由で適切に呼び出されることを確認してください。
