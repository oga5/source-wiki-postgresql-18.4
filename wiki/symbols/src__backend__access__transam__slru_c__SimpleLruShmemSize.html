SimpleLruShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/slru.c:197

目的

SLRU（Simple LRU）バッファ管理機構を共有メモリ上に初期化するために必要なバイト数を計算する。

シグネチャまたは形状

Size SimpleLruShmemSize(int nslots, int nlsns)

入力と出力

  • **入力**:
  • nslots: 管理するバッファスロットの総数。
  • nlsns: ページごとに保持する LSN（Log Sequence Number）の数（0 の場合は LSN を保持しない）。
  • **出力**:
  • 共有メモリ領域として確保すべき合計サイズ（Size 型）。

前提条件と不変条件

  • nslots は SLRU_MAX_ALLOWED_BUFFERS 以下であること。
  • nslots は SLRU_BANK_SIZE の倍数であること。
  • 内部計算においてオーバーフローが発生しない前提で設計されている。

アルゴリズムまたは振る舞い

1. SlruSharedData 構造体自体のサイズをベースに、スロット数に応じた各管理配列（ページバッファ、ステータス、ダーティフラグ、ページ番号、LRU カウンタ、バッファロック）のサイズを加算する。 2. 銀行（Bank）単位のロックおよびカウンタ用の領域を nbanks（nslots / SLRU_BANK_SIZE）に基づいて加算する。 3. nlsns > 0 の場合、LSN 格納用の領域を追加する。 4. 最終的に BUFFERALIGN でアライメント調整を行い、実際のページデータ領域（BLCKSZ * nslots）を加算して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • clog.c: CLOGShmemBuffers() を使用。
  • commit_ts.c: CommitTsShmemBuffers() を使用。
  • multixact.c: multixact_offset_buffers および multixact_member_buffers を使用。
  • subtrans.c: SUBTRANSShmemBuffers() を使用。
  • async.c: notify_buffers を使用。
  • predicate.c: serializable_buffers を使用。
  • test_slru.c: NUM_TEST_BUFFERS を使用。
  • **呼び出し先**:
  • MAXALIGN（マクロ）
  • BUFFERALIGN（マクロ）

関連データ構造

  • SlruSharedData: 共有メモリ上の SLRU 状態管理構造体。
  • LWLockPadded: ロック管理用構造体。
  • XLogRecPtr: LSN 格納用型。

失敗モードと境界条件

  • nslots が SLRU_BANK_SIZE の倍数でない場合、Assert により停止する。
  • nslots が SLRU_MAX_ALLOWED_BUFFERS を超える場合、Assert により停止する。

保守担当者向けメモ

  • 本関数は共有メモリ割り当ての計算のみを行うため、実際のメモリ確保は ShmemInitStruct 等を呼び出す側で行う必要がある。
  • nlsns を変更する場合は、group_lsn 配列の計算ロジックに影響するため注意すること。
