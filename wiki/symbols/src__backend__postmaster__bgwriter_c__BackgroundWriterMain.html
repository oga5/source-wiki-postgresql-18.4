BackgroundWriterMain

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/bgwriter.c:87

目的

バックグラウンドライター（bgwriter）プロセスのメインループを実行し、共有バッファ内のダーティページを定期的にディスクへ書き出すことで、バックエンドプロセスの負荷を軽減する。

シグネチャまたは形状

void BackgroundWriterMain(const void *startup_data, size_t startup_data_len)

入力と出力

  • **入力**: startup_data（未使用、startup_data_len は 0 である必要がある）
  • **出力**: なし（プロセス終了までループ内で動作）

前提条件と不変条件

  • AuxiliaryProcessMainCommon() を呼び出し、補助プロセスとしての初期化が完了していること。
  • sigsetjmp によるエラーリカバリ用のスタックが常に有効であること。
  • PG_exception_stack が local_sigjmp_buf を指していること。

アルゴリズムまたは振る舞い

1. **初期化**: シグナルハンドラの設定、メモリコンテキストの作成、WritebackContext の初期化を行う。 2. **エラーリカバリ**: sigsetjmp を使用し、エラー発生時にはリソース（LWLocks、バッファ、一時ファイル等）を解放し、1秒間待機した後に処理を再開する。 3. **メインループ**:

  • ProcessMainLoopInterrupts() で割り込みを処理。
  • BgBufferSync() を呼び出し、ダーティバッファの書き出しを行う。
  • 統計情報を報告し、チェックポイント直後であれば smgrdestroyall() で不要なファイルハンドルを解放。
  • スタンバイ状態であれば、必要に応じて xl_running_xacts を WAL に記録。
  • WaitLatch() を用いて次のサイクルまで待機（ハイバネーション制御を含む）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c（プロセスの起動時に参照）
  • **呼び出し先**: AuxiliaryProcessMainCommon, BgBufferSync, pgstat_report_bgwriter, smgrdestroyall, sigsetjmp, siglongjmp（エラー時）など。

関連データ構造

  • sigjmp_buf: エラーリカバリ用スタック。
  • MemoryContext: プロセス専用のメモリ管理コンテキスト。
  • WritebackContext: 書き出し制御用コンテキスト。

失敗モードと境界条件

  • **エラー発生時**: sigsetjmp により捕捉され、リソース解放後に pg_usleep で1秒待機してからループを継続する。
  • **ハイバネーション**: システムがアイドル状態の場合、BgBufferSync の戻り値に基づいてスリープ時間を延長する。

保守担当者向けメモ

  • PG_TRY ではなく sigsetjmp を直接使用しているのは、エラーリカバリ中も例外ハンドラを有効に保つためである。
  • チェックポイント処理は checkpointer.c に分離されているため、本関数には含まれない。
  • 終了処理やシグナル処理の変更時は、pqsignal の設定と sigprocmask の整合性に注意すること。
