find_plan

定義

  • 種別: function
  • 定義箇所: contrib/spi/refint.c:616

目的

キャッシュされた実行計画（EPlan）を識別子に基づいて検索し、存在しない場合は新規に確保・初期化して返す。

シグネチャまたは形状

static EPlan *find_plan(char *ident, EPlan **eplan, int *nplans)

入力と出力

  • **入力**:
  • ident: 検索対象の識別子文字列（トリガー名とリレーションIDの組み合わせ）。
  • eplan: EPlan 構造体の配列へのポインタ。
  • nplans: 現在保持されている計画数へのポインタ。
  • **出力**:
  • 検索または新規作成された EPlan 構造体へのポインタ。

前提条件と不変条件

  • メモリ割り当ては TopMemoryContext で行われるため、セッションをまたいで計画が生存する。
  • eplan 配列は repalloc により動的に拡張される。

アルゴリズムまたは振る舞い

1. TopMemoryContext へメモリコンテキストを切り替える。 2. nplans > 0 の場合、既存の eplan 配列を走査し、ident が一致する EPlan を探す。一致すればそのポインタを返す。 3. 一致しない場合、または nplans == 0 の場合、palloc または repalloc を使用して配列を拡張し、新しい EPlan 領域を確保する。 4. 新規確保した EPlan に対して、ident のコピー、nplans の初期化（0）、splan の初期化（NULL）を行い、nplans をインクリメントする。 5. 元のメモリコンテキストに戻し、確保した EPlan を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/spi/refint.c 内の check_primary_key (131行目)
  • contrib/spi/refint.c 内の check_foreign_key (358行目)
  • **呼び出し先**:
  • MemoryContextSwitchTo
  • palloc / repalloc
  • pstrdup
  • strcmp

関連データ構造

  • EPlan: 実行計画（SPIPlanPtr）と識別子を保持するキャッシュ用構造体。

失敗モードと境界条件

  • メモリ不足時には palloc / repalloc がエラーを発生させる。
  • nplans が 0 の場合、新規配列として初期化される。

保守担当者向けメモ

  • 本関数は SPI_keepplan と組み合わせて使用されることを前提としている。
  • ident 文字列は pstrdup で複製されるため、呼び出し元で渡した文字列の寿命に依存しない。
