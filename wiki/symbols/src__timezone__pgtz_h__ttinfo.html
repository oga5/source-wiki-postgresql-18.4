ttinfo

定義

  • 種別: struct
  • 定義箇所: src/timezone/pgtz.h:26

目的

タイムゾーンの遷移における時刻タイプ情報（time type information）を保持するための構造体です。IANA タイムゾーンデータベースのファイル形式における各時刻タイプの属性をメモリ上で表現します。

シグネチャまたは形状

struct ttinfo {
    int32    tt_utoff;      /* UTからのオフセット（秒） */
    bool     tt_isdst;      /* 夏時間（DST）フラグ */
    int      tt_desigidx;   /* 略称リストのインデックス */
    bool     tt_ttisstd;    /* 遷移が標準時刻かどうか */
    bool     tt_ttisut;     /* 遷移がUTかどうか */
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。tzload() 等の処理を通じて、タイムゾーンファイルから読み込まれたバイナリデータが本構造体の各フィールドにマッピングされます。

前提条件と不変条件

  • tt_utoff は秒単位のオフセットを保持します。
  • tt_desigidx は、関連する state 構造体が保持する略称文字列配列へのインデックスとして機能します。

アルゴリズムまたは振る舞い

本構造体は、特定の時刻における UTC との時差や、その時刻が夏時間であるか、および表示上の略称を決定するための参照データとして使用されます。時刻変換処理において、state 構造体内の ttinfo 配列から適切なエントリが選択されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/timezone/localtime.c 内の各関数（init_ttinfo による初期化や、時刻変換ロジック内での参照）。
  • **呼び出し先**: 本構造体はデータ構造であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • struct state: タイムゾーンの遷移情報全体を保持する構造体であり、ttinfo の配列をメンバとして含みます。
  • tzfile.h: 本構造体のフィールドの元となるバイナリフォーマットを定義しています。

失敗モードと境界条件

  • tt_desigidx が不正なインデックスを指している場合、略称の取得時にメモリ破壊や誤った文字列参照が発生する可能性があります。
  • ttinfo 配列のサイズは TZ_MAX_TIMES（tzfile.h で定義）に依存します。

保守担当者向けメモ

  • 本構造体は IANA タイムゾーンデータベースの仕様と密接に関連しています。構造体の変更は localtime.c での解析ロジックおよび tzfile.h の定義との整合性に注意してください。
