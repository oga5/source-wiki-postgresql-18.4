SetClientEncoding

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/mbutils.c:216

目的

クライアントエンコーディングを切り替え、サーバーとの間で文字変換を行うための変換関数（FmgrInfo）をキャッシュからセットアップする。

シグネチャまたは形状

int SetClientEncoding(int encoding)

入力と出力

  • **入力**: encoding (設定するエンコーディングID)
  • **出力**: 成功時は 0、失敗時（無効なエンコーディングまたはキャッシュ未検出）は -1 を返す。

前提条件と不変条件

  • PG_VALID_FE_ENCODING(encoding) が真である必要がある。
  • バックエンド起動完了前（backend_startup_complete が偽）の場合、設定は保留され 0 を返す。
  • 変換関数を使用する場合、事前に PrepareClientEncoding によって ConvProcList にキャッシュが準備されている必要がある。

アルゴリズムまたは振る舞い

1. 引数のエンコーディングが有効か検証する。 2. 起動処理中の場合は pending_client_encoding に値を保存して終了する。 3. サーバーとクライアントのエンコーディングが同一、または一方が PG_SQL_ASCII の場合は変換関数を NULL に設定し、変換なしとして終了する。 4. ConvProcList を走査し、現在のサーバーエンコーディングと指定されたクライアントエンコーディングに一致する ConvProcInfo を検索する。 5. 一致するものが見つかれば、ClientEncoding および変換関数ポインタ（ToServerConvProc, ToClientConvProc）を更新する。重複するキャッシュエントリがあれば解放する。 6. キャッシュが見つかれば 0 を、見つからなければ -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c
  • src/backend/commands/variable.c
  • src/backend/utils/mb/mbutils.c
  • **呼び出し先**:
  • GetDatabaseEncoding()
  • pfree()
  • foreach_delete_current() (マクロ)
  • lfirst() (マクロ)

関連データ構造

  • ConvProcList: 変換関数情報を保持するリスト。
  • ConvProcInfo: サーバー・クライアント間の変換関数情報（FmgrInfo）を格納する構造体。
  • ClientEncoding: 現在のクライアントエンコーディング情報を保持するグローバル変数。
  • pg_enc2name_tbl: エンコーディング名とIDの対応テーブル。

失敗モードと境界条件

  • PG_VALID_FE_ENCODING が偽の場合、-1 を返す。
  • PrepareClientEncoding による事前のキャッシュ準備がない状態で、変換が必要なエンコーディングを指定した場合、-1 を返す。

保守担当者向けメモ

  • 本関数は ConvProcList を操作するため、メモリ管理（pfree）とリストの整合性に注意すること。
  • 変換関数は TopMemoryContext にキャッシュされるため、セッション終了まで保持される。
  • PrepareClientEncoding との対で使用される設計である。
