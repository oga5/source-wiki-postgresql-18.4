PLyResultObject

定義

  • 種別: struct
  • 定義箇所: src/pl/plpython/plpy_resultobject.h:13

目的

PostgreSQL のクエリ実行結果（SPI 経由）を Python オブジェクトとして保持し、Python 側からリストや辞書のように操作可能にするためのデータ構造です。

シグネチャまたは形状

typedef struct PLyResultObject
{
    PyObject_HEAD
    PyObject   *nrows;
    PyObject   *rows;
    PyObject   *status;
    TupleDesc   tupdesc;
} PLyResultObject;

入力と出力

  • **入力**: PLy_result_new() を通じて初期化されます。
  • **出力**: Python 側へクエリ結果（行データ、行数、ステータス）を提供します。

前提条件と不変条件

  • PyObject_HEAD を含み、Python の拡張型として動作します。
  • tupdesc は PostgreSQL 側のリソースであり、PLy_result_dealloc 等で適切に解放される必要があります。
  • rows はデータがない場合、空のリストとして保持されます。

アルゴリズムまたは振る舞い

1. PLy_result_new() によりメモリ確保と初期化が行われます。 2. SPI 実行結果に基づき、tupdesc（タプル記述子）、rows（行データ）、nrows（行数）、status（SPI ステータス）が設定されます。 3. Python 側からのメソッド呼び出し（colnames, nrows 等）に応じて、構造体内の各フィールドが参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_cursorobject.c および src/pl/plpython/plpy_resultobject.c 内の関数から参照・操作されています。
  • **呼び出し先**: Python C API（PyObject_New 等）および PostgreSQL のタプル記述子管理関数を利用します。

関連データ構造

  • TupleDesc: PostgreSQL のタプル記述子。
  • PyObject: Python のオブジェクト基底型。

失敗モードと境界条件

  • PyObject_New によるメモリ確保が失敗した場合、NULL が返されます。
  • tupdesc の解放漏れはメモリリークを引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は Python の参照カウンタと PostgreSQL のメモリコンテキストの両方を管理対象とするため、デストラクタ（PLy_result_dealloc）でのリソース解放には細心の注意を払ってください。
