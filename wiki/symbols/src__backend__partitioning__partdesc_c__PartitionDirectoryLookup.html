PartitionDirectoryLookup

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partdesc.c:455

目的

指定されたリレーションのパーティション記述子（PartitionDesc）を PartitionDirectory から取得、または必要に応じて新規作成して返します。

シグネチャまたは形状

PartitionDesc PartitionDirectoryLookup(PartitionDirectory pdir, Relation rel)

入力と出力

  • **入力**:
  • pdir: パーティション記述子を管理するディレクトリ構造体。
  • rel: 検索対象のリレーション。
  • **出力**:
  • 対象リレーションに対応する PartitionDesc 構造体。

前提条件と不変条件

  • pdir は有効な PartitionDirectory であること。
  • rel は有効なリレーションであること。
  • 新規作成時には RelationIncrementReferenceCount(rel) が呼び出され、リレーションの参照カウントが保持される。

アルゴリズムまたは振る舞い

1. pdir->pdir_hash に対して、リレーションの OID をキーとしてハッシュ検索を行う。 2. キャッシュが存在しない場合（!found）:

  • RelationIncrementReferenceCount(rel) を実行し、リレーションの参照カウントを増加させる。
  • pdir->omit_detached フラグを使用して RelationGetPartitionDesc(rel, ...) を呼び出し、記述子を生成する。
  • 生成した記述子とリレーションを PartitionDirectoryEntry に格納する。

    • 3. キャッシュ済み、または新規作成した記述子を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execPartition.c
  • src/backend/optimizer/util/inherit.c
  • src/backend/optimizer/util/plancat.c
  • **呼び出し先**:
  • hash_search
  • RelationGetRelid
  • RelationIncrementReferenceCount
  • RelationGetPartitionDesc

関連データ構造

  • PartitionDirectory: 複数のリレーションに対する記述子をハッシュテーブルで管理するコンテナ。
  • PartitionDirectoryEntry: ハッシュテーブルの各エントリ。リレーションと対応する PartitionDesc を保持する。
  • PartitionDesc: パーティションの構造情報。

失敗モードと境界条件

  • RelationGetPartitionDesc が NULL を返すことは想定されておらず、Assert(pde->pd != NULL) により検証される。

保守担当者向けメモ

  • 本関数は PartitionDirectory を介してパーティション記述子のライフサイクルを管理します。リレーションの参照カウントを操作するため、ディレクトリの破棄時には適切なクリーンアップが必要です。
