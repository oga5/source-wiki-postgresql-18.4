ExecEndWindowAgg

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeWindowAgg.c:2786

目的

WindowAgg 実行ノードの終了処理を行い、割り当てられたメモリリソースおよびタプルバッファを解放する。

シグネチャまたは形状

void ExecEndWindowAgg(WindowAggState *node)

入力と出力

  • **入力**: WindowAggState *node （ウィンドウ集約ノードの実行時状態構造体）
  • **出力**: なし（副作用としてリソース解放を実行）

前提条件と不変条件

  • node は ExecInitWindowAgg によって初期化済みであること。
  • node->buffer が存在する場合、tuplestore_end を呼び出してクリーンアップを行う。

アルゴリズムまたは振る舞い

1. **タプルストアの解放**: node->buffer が存在する場合、tuplestore_end を呼び出して解放し、node->buffer を NULL に設定する。 2. **パーティションの解放**: release_partition(node) を呼び出し、現在のパーティションに関連するリソースを解放する。 3. **メモリコンテキストの削除**:

  • 各集約関数ごとの aggcontext を削除（node->aggcontext と異なる場合）。
  • node->partcontext および node->aggcontext を削除。

    • 4. **メモリの解放**: node->perfunc および node->peragg 配列を pfree で解放する。 5. **プランノードの終了**: outerPlanState(node) を取得し、ExecEndNode を呼び出して下位プランノードの終了処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:730 （ExecEndNode 経由での呼び出しが確認されている）
  • **呼び出し先**: tuplestore_end, release_partition, MemoryContextDelete, pfree, outerPlanState, ExecEndNode

関連データ構造

  • WindowAggState: 実行ノードの状態を保持する主要構造体。
  • PlanState: 下位プランノードの実行状態。

失敗モードと境界条件

  • node が NULL の場合、あるいは node 内のポインタが不正な場合はセグメンテーション違反を引き起こす可能性がある。
  • tuplestore_end 呼び出し前に node->buffer が既に解放されている場合、二重解放の危険性がある（本関数では NULL チェックと代入によりこれを回避している）。

保守担当者向けメモ

  • peragg 配列内の各 aggcontext が node->aggcontext と同一である場合、個別の削除は行われないため、メモリ管理の所有権に注意すること。
  • release_partition は node->buffer を使用する可能性があるため、順序として tuplestore_end の後に実行されることが重要である。
