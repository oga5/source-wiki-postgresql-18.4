gbt_inet_penalty

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_inet.c:153

目的

GiST インデックスの挿入時において、新しい値を既存のインデックスツリーのノードに追加する際の「ペナルティ（コスト）」を計算します。

シグネチャまたは形状

Datum gbt_inet_penalty(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): 既存のインデックスエントリ (inetKEY へのポインタ)。
  • PG_GETARG_POINTER(1): 追加対象の新しいエントリ (inetKEY へのポインタ)。
  • PG_GETARG_POINTER(2): 計算結果を格納する float 型のポインタ。
  • **出力**: PG_RETURN_POINTER(result) を通じて、計算されたペナルティ値（float）を返します。

前提条件と不変条件

  • 入力される GISTENTRY のキーは inetKEY 構造体としてキャスト可能である必要があります。
  • inetKEY 構造体は lower および upper 境界値を含んでいる必要があります。

アルゴリズムまたは振る舞い

1. 引数から既存エントリ (origentry) と新規エントリ (newentry) を取得します。 2. penalty_num 関数を呼び出し、origentry の範囲と newentry の範囲を比較して、挿入に伴う範囲の拡大量を計算します。 3. 計算されたペナルティ値を result ポインタに書き込み、返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス構築・挿入プロセス（直接的な呼び出し元関数名はソース内に明示されていません）。
  • **呼び出し先**: penalty_num（btree_utils_num.c で定義された汎用関数）。

関連データ構造

  • inetKEY: inet 型のインデックス範囲を保持する構造体。
  • GISTENTRY: GiST インデックスの各エントリを表現する PostgreSQL 内部構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合や、不正なメモリ領域を指している場合はクラッシュする可能性があります。
  • inetKEY の lower または upper が適切に初期化されていない場合、計算結果が不定となります。

保守担当者向けメモ

  • 本関数は btree_utils_num.c の penalty_num に処理を委譲しているため、ロジックの変更が必要な場合は btree_utils_num 側の実装を確認してください。
  • inet 型特有の比較ロジックは inetKEY の構造に依存しています。
