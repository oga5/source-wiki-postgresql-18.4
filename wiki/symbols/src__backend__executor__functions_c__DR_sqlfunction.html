DR_sqlfunction

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/functions.c:45

目的

SQL関数実行時の結果収集および変換を行うための DestReceiver 派生構造体です。関数実行エンジンが生成したタプルを一時的に保持し、必要に応じて型変換を行います。

シグネチャまたは形状

typedef struct {
    DestReceiver pub;           /* 公開されている関数ポインタ */
    Tuplestorestate *tstore;    /* 結果タプルの格納先、またはNULL */
    JunkFilter *filter;         /* タプル型変換用のジャンクフィルタ */
} DR_sqlfunction;

入力と出力

  • **入力**: DestReceiver インターフェースを介して受け取るタプル。
  • **出力**: tstore に蓄積された結果タプル、または filter を通じて変換されたタプル。

前提条件と不変条件

  • pub フィールドは DestReceiver として初期化されている必要があります。
  • tstore が NULL の場合、結果の蓄積は行われません。

アルゴリズムまたは振る舞い

1. **初期化**: CreateSQLFunctionDestReceiver を通じてメモリ確保と初期化が行われます。 2. **結果収集**: SQL関数の実行中に生成されたタプルを tstore に格納します。 3. **変換**: 必要に応じて filter を使用し、実行結果のタプル構造を関数の戻り値型に適合させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/functions.c 内の関数実行ロジック。
  • **呼び出し先**: Tuplestore API（タプル蓄積用）および JunkFilter API（変換用）。
  • 注: 具体的な関数呼び出しの連鎖については、提供されたソース excerpt に詳細なコールグラフが含まれていないため、これ以上の情報は不明です。

関連データ構造

  • DestReceiver: 本構造体の基底となるインターフェース。
  • Tuplestorestate: 結果セットを保持するためのメモリ/ディスク上のストレージ。
  • JunkFilter: 不要な属性の除去や型変換を行うための構造体。

失敗モードと境界条件

  • メモリ不足: palloc0 による確保失敗時にエラーが発生する可能性があります。
  • tstore の未初期化: tstore が適切に設定されていない状態でタプルを挿入しようとした場合の動作は未定義です。

保守担当者向けメモ

  • 本構造体は src/backend/executor/functions.c 内の SQL 関数実行エンジン専用です。
  • DestReceiver の契約に従い、pub フィールドの関数ポインタが正しく設定されていることを確認してください。
