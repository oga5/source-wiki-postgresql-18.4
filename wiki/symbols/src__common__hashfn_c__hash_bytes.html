hash_bytes

定義

  • 種別: function
  • 定義箇所: src/common/hashfn.c:145

目的

可変長バイト列から、ハッシュテーブル等の用途に適した 32 ビットのハッシュ値を生成します。

シグネチャまたは形状

uint32 hash_bytes(const unsigned char *k, int keylen)

入力と出力

  • **入力**:
  • k: ハッシュ化対象のバイト列へのポインタ。
  • keylen: バイト列の長さ。
  • **出力**:
  • 算出された uint32 型のハッシュ値。

前提条件と不変条件

  • hash_bytes は内部で elog(ERROR) を発生させない設計となっており、クリティカルなパスでも安全に使用可能です。
  • 入力データがワード境界（UINT32_ALIGN_MASK）に整列している場合、最適化されたコードパスが選択されます。

アルゴリズムまたは振る舞い

1. **初期化**: 内部状態 a, b, c を定数およびキー長に基づいて初期化します。 2. **データ処理**:

  • 12 バイト単位で mix マクロを適用し、状態を攪拌します。
  • アライメントに応じて、ワード単位の読み込みまたはバイト単位の結合を行います。
  • エンディアン（WORDS_BIGENDIAN）を考慮し、バイト列を uint32 に変換します。

    • 3. **残余処理**: 12 バイト未満の残りのデータに対し、switch 文を用いて残りのバイトを状態 a, b, c に加算します。 4. **最終化**: 最終的なハッシュ値を確定させ、呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/heaptuple.c
  • src/backend/backup/basebackup_incremental.c
  • src/backend/optimizer/path/equivclass.c
  • src/backend/utils/adt/datum.c
  • src/backend/utils/adt/json.c
  • src/bin/pg_dump/common.c
  • src/common/blkreftable.c
  • **呼び出し先**:
  • mix マクロ（ハッシュ計算の核となるビット混合処理）。

関連データ構造

  • uint32: ハッシュ値および内部状態の保持に使用。
  • UINT32_ALIGN_MASK: メモリのアライメントチェックに使用。

失敗モードと境界条件

  • keylen が 0 の場合、初期化された定数に基づくハッシュ値が返されます。
  • データの整列状態により実行されるコードパスが異なりますが、結果の一貫性は維持されます。

保守担当者向けメモ

  • 本関数は Bob Jenkins 氏のアルゴリズムをベースにしています。mix や final マクロを修正する際は、ハッシュの分布特性（アバランチ効果）に十分注意してください。
  • 64 ビットのハッシュ値が必要な場合は、本関数ではなく hash_bytes_extended を使用してください。
