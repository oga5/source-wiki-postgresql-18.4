SerialInit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/predicate.c:805

目的

直列化可能スナップショット分離（SSI）で使用される pg_serial データの SLRU 管理を初期化し、共有メモリ上の SerialControlData 構造体を確保またはアタッチする。

シグネチャまたは形状

static void SerialInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリおよび SLRU 制御構造を初期化）

前提条件と不変条件

  • SerialSlruCtl が初期化済みであること。
  • 共有メモリの初期化プロセスにおいて、SerialControlData の存在確認と IsUnderPostmaster の状態が一致していること（Assert による検証）。

アルゴリズムまたは振る舞い

1. **SLRU 初期化**: SerialSlruCtl に対して SerialPagePrecedesLogically をページ順序判定関数として設定し、SimpleLruInit を呼び出して pg_serial 用のバッファ管理を初期化する。 2. **テスト実行**: USE_ASSERT_CHECKING が定義されている場合、SerialPagePrecedesLogicallyUnitTests および SlruPagePrecedesUnitTests を実行して整合性を検証する。 3. **共有メモリ確保**: ShmemInitStruct を使用して SerialControlData を共有メモリ上に確保、または既存の領域にアタッチする。 4. **初期化判定**: 新規作成時（found == false）のみ、SerialControlLock を排他ロックし、headPage を -1、headXid および tailXid を InvalidTransactionId に設定して初期状態を構築する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/lmgr/predicate.c 内の初期化プロセス（直接的な呼び出し箇所がソース内に存在）。
  • 注: src/backend/backup/basebackup.c および src/bin/pg_rewind/filemap.c に言及があるが、これらが直接本関数を呼び出しているか、あるいは単なる参照であるかはソースコードの文脈からは確定できない。
  • 呼び出し先: SimpleLruInit, ShmemInitStruct, LWLockAcquire, LWLockRelease 等の共有メモリおよびロック管理関数。

関連データ構造

  • SerialControlData: 直列化可能トランザクションの SLRU 状態を保持する共有メモリ構造体。
  • SerialSlruCtl: SLRU 管理のための制御構造。

失敗モードと境界条件

  • 共有メモリの確保失敗や、Assert による found フラグと IsUnderPostmaster の不一致が発生した場合、プロセスは異常終了する。

保守担当者向けメモ

  • 本関数は SSI の基盤となる共有メモリ領域を準備する重要な初期化ルーチンである。
  • SerialControlData の初期化ロジックは、PostgreSQL の起動シーケンスにおける共有メモリ割り当てのタイミングに依存している。
