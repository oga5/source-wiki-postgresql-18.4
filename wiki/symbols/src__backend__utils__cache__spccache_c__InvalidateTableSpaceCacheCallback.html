InvalidateTableSpaceCacheCallback

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/spccache.c:54

目的

システムカタログ pg_tablespace の更新イベントを検知し、メモリ上のテーブルスペースキャッシュ（TableSpaceCacheHash）を全クリアすることで、キャッシュの整合性を維持します。

シグネチャまたは形状

static void InvalidateTableSpaceCacheCallback(Datum arg, int cacheid, uint32 hashvalue)

入力と出力

  • **入力**:
  • arg: コールバック登録時に渡されたデータ（未使用）。
  • cacheid: 無効化対象のキャッシュID。
  • hashvalue: 無効化対象のハッシュ値。
  • **出力**: なし（副作用としてキャッシュの破棄を実行）。

前提条件と不変条件

  • TableSpaceCacheHash が初期化済みであること。
  • 本関数はシステムカタログの無効化通知メカニズムから呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. hash_seq_init を使用して TableSpaceCacheHash の全エントリを走査します。 2. 各エントリ（TableSpaceCacheEntry）に対して以下の処理を行います。

  • spc->opts が存在する場合、pfree でメモリを解放します。
  • hash_search を HASH_REMOVE モードで呼び出し、ハッシュテーブルからエントリを削除します。

    • 3. 削除処理中に hash_search が失敗した場合、elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/spccache.c 内の初期化処理（CacheRegisterSyscacheCallback 経由での登録が示唆されています）。
  • **呼び出し先**:
  • hash_seq_init, hash_seq_search, hash_search（utils/hsearch.h）
  • pfree（メモリ管理）
  • elog（エラー報告）

関連データ構造

  • TableSpaceCacheHash: キャッシュの実体であるハッシュテーブル。
  • TableSpaceCacheEntry: キャッシュ内の各エントリ。

失敗モードと境界条件

  • キャッシュが空の場合、走査は即座に終了します。
  • hash_search が予期せず失敗した場合、elog(ERROR) によりトランザクションまたはプロセスが異常終了します。

保守担当者向けメモ

  • 現在の設計では、テーブルスペースの変更時にキャッシュを全フラッシュする単純な戦略をとっています。テーブルスペース数が増大した場合は、エントリ単位の無効化への変更を検討してください。
