jsonb_subscript_handler

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonbsubs.c:401

目的

jsonb 型に対する添字（subscripting）操作を管理するための SubscriptRoutines 構造体をシステムに提供するエントリポイントです。

シグネチャまたは形状

Datum jsonb_subscript_handler(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（標準的な PostgreSQL 関数呼び出し規約）。
  • **出力**: SubscriptRoutines 構造体へのポインタを Datum 型として返却します。

前提条件と不変条件

  • 本関数は static const な SubscriptRoutines 構造体を定義し、そのアドレスを返却します。
  • 構造体内の各関数ポインタ（transform, exec_setup）は、jsonb 型の添字操作に必要なロジックを指し示している必要があります。

アルゴリズムまたは振る舞い

1. SubscriptRoutines 構造体を静的に初期化します。

  • transform: jsonb_subscript_transform を設定。
  • exec_setup: jsonb_exec_setup を設定。
  • fetch_strict: true（入力が NULL の場合、取得結果も NULL となる）。
  • fetch_leakproof: true（不正な添字の場合、取得結果は NULL となる）。
  • store_leakproof: false（代入操作において不正な添字はエラーをスローする）。

    • 2. 初期化した構造体のアドレスを PG_RETURN_POINTER マクロを介して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は明示されていませんが、PostgreSQL の添字フレームワークによってシステム起動時または型定義の参照時に呼び出されることが想定されます。
  • **呼び出し先**:
  • jsonb_subscript_transform
  • jsonb_exec_setup

関連データ構造

  • SubscriptRoutines: 添字操作の挙動を定義する構造体。
  • PG_FUNCTION_ARGS: 関数呼び出しの引数コンテキスト。

失敗モードと境界条件

  • 本関数自体は静的な構造体のアドレスを返すのみであるため、関数内部での失敗モードは定義されていません。
  • 実行時の挙動（fetch_strict 等）については、返却された構造体を利用する側のフレームワークに依存します。

保守担当者向けメモ

  • 本関数は jsonb の添字操作の構成を定義する中心的な場所です。フラグ（fetch_leakproof 等）を変更する場合は、実行時のエラーハンドリングやセキュリティ上の影響を考慮してください。
  • スライス操作はサポート対象外であるため、将来的な機能追加の際には SubscriptRoutines の定義拡張が必要となる可能性があります。
