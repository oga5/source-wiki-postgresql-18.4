PrepareSortSupportFromOrderingOp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/sortsupport.c:133

目的

指定された順序付け演算子（Ordering Operator）に基づき、SortSupportData 構造体を初期化し、ソート実行に必要な比較ロジックを準備します。

シグネチャまたは形状

void PrepareSortSupportFromOrderingOp(Oid orderingOp, SortSupport ssup)

入力と出力

  • **入力**:
  • orderingOp (Oid): 順序付けに使用する演算子のOID。
  • ssup (SortSupport): 初期化対象となる SortSupportData 構造体へのポインタ。
  • **出力**: なし（ssup の内部フィールドが更新されます）。

前提条件と不変条件

  • ssup->comparator が NULL であること（Assert により検証）。
  • orderingOp は有効な順序付け演算子である必要があります。

アルゴリズムまたは振る舞い

1. get_ordering_op_properties を呼び出し、演算子に関連する演算子ファミリー（opfamily）、入力型、比較タイプ（COMPARE_LT または COMPARE_GT）を取得します。 2. 取得できない場合はエラーをスローします。 3. 比較タイプが COMPARE_GT の場合、ssup->ssup_reverse を true に設定します。 4. FinishSortSupportFunction を呼び出し、演算子ファミリーから最適な比較関数を検索・設定します。

  • src/backend/commands/analyze.c

  • **呼び出し先**:
  • get_ordering_op_properties

  • elog

関連データ構造

  • SortSupportData (およびその別名 SortSupport): ソートの比較ロジックや状態を保持する中心的な構造体。

失敗モードと境界条件

  • orderingOp が有効な順序付け演算子でない場合、elog(ERROR, ...) により処理が中断されます。

保守担当者向けメモ

  • 本関数は、演算子からソートサポートを構築する際のエントリーポイントです。
  • ssup->comparator の設定は FinishSortSupportFunction に委譲されており、必要に応じて comparison_shim が適用されます。
