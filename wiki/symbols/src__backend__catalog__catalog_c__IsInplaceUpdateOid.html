IsInplaceUpdateOid

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/catalog.c:192

目的

指定されたOIDが、インプレース更新（In-place update）の対象となるシステムカタログであるかを判定します。

シグネチャまたは形状

bool IsInplaceUpdateOid(Oid relid)

入力と出力

  • **入力**: relid (Oid) - 判定対象のリレーションOID。
  • **出力**: bool - 対象がインプレース更新対象であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 本関数はカタログアクセスを行いません。ロック取得中や再帰的な呼び出しにおいても安全に使用可能です。
  • 判定対象は RelationRelationId（pg_class）または DatabaseRelationId（pg_database）に限定されます。

アルゴリズムまたは振る舞い

引数として渡された relid が、以下のいずれかの定数と一致するかを比較します。

  • RelationRelationId
  • DatabaseRelationId

一致する場合に true を返し、それ以外の場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/catalog.c:185 (内部的な判定処理)
  • **呼び出し先**:
  • 提供されたソース情報には、本関数が呼び出している関数に関する情報はありません。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。
  • RelationRelationId, DatabaseRelationId: システムカタログのOIDを保持する定数。

失敗モードと境界条件

  • 境界条件: 入力されたOIDがシステムカタログのOIDであっても、上記2つのいずれでもない場合は false を返します。
  • 失敗モード: 特筆すべき失敗モードはありません。

保守担当者向けメモ

  • 本関数は、システムカタログの更新方式を識別するための低レイヤーのユーティリティです。
  • 新たなインプレース更新対象のカタログを追加する必要がある場合は、本関数の条件式を更新してください。
