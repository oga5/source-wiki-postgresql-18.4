nodeTokenType

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/read.c:245

目的

シリアライズされた文字列形式のトークンを解析し、PostgreSQLの内部ノード構造における型（NodeTag）を識別する。

シグネチャまたは形状

NodeTag nodeTokenType(const char *token, int length)

入力と出力

  • **入力**:
  • token: 解析対象の文字列トークン。
  • length: トークンの長さ。
  • **出力**:
  • NodeTag: トークンの種類を示す列挙値（T_Integer, T_Float, T_Boolean, T_String, T_BitString, LEFT_PAREN, RIGHT_PAREN, LEFT_BRACE, OTHER_TOKEN など）。

前提条件と不変条件

  • token は pg_strtok によって切り出された有効な文字列であることを想定している。
  • 数値判定において、strtoint がトークンの末尾まで正しく解析できることを前提としている。

アルゴリズムまたは振る舞い

1. **数値判定**: トークンが数値（整数または浮動小数点数）かを確認する。strtoint を使用して範囲と構文を検証し、T_Integer または T_Float を返す。 2. **記号判定**: (, ), { の単一文字トークンをそれぞれ対応する括弧型に分類する。 3. **キーワード/リテラル判定**:

  • "true" または "false" の場合は T_Boolean。
  • 両端が " で囲まれている場合は T_String。
  • 'b' または 'x' で始まる場合は T_BitString。

    • 4. **デフォルト**: 上記のいずれにも該当しない場合は OTHER_TOKEN を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/read.c 内の nodeRead 関数から呼び出される。
  • **呼び出し先**: strtoint（標準的な数値変換関数）。

関連データ構造

  • NodeTag: ノードの種類を識別するための列挙型。
  • List, Bitmapset: nodeRead が本関数の結果に基づいて構築するデータ構造。

失敗モードと境界条件

  • 数値判定時に errno が ERANGE になる場合や、解析結果がトークン長と一致しない場合は T_Float として扱われる。
  • 引用符で囲まれた文字列において、長さが1以下の場合は T_String と判定されない。

保守担当者向けメモ

  • src/backend/nodes/outfuncs.c において、ビット文字列（'b' や 'x'）のエスケープを行わない仕様が本関数の判定ロジックに依存しているため、変更時は注意が必要。
