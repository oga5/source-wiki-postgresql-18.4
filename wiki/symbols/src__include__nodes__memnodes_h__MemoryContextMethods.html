MemoryContextMethods

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/memnodes.h:58

目的

PostgreSQLのメモリ管理サブシステムにおいて、各メモリコンテキスト実装が提供すべき仮想関数テーブル（vtable）を定義する。これにより、メモリ確保や解放といった操作を、具体的な実装（AllocSetやSlabなど）に依存しない共通インターフェースとして抽象化する。

シグネチャまたは形状

関数ポインタをメンバとして保持する構造体。主な操作として alloc、free_p、realloc、reset、delete_context、get_chunk_context、get_chunk_space、is_empty、stats を定義する。MEMORY_CONTEXT_CHECKING が定義されている場合は check 関数が追加される。

入力と出力

  • **alloc**: (MemoryContext, Size, int) を受け取り、確保されたメモリ領域のポインタを返す。
  • **free_p**: (void *) を受け取り、領域を解放する。
  • **realloc**: (void *, Size, int) を受け取り、再確保された領域のポインタを返す。
  • **reset/delete_context**: (MemoryContext) を受け取り、コンテキストの状態を更新または破棄する。
  • **get_chunk_context**: (void *) から所属する MemoryContext を特定して返す。
  • **get_chunk_space**: (void *) から確保サイズ（オーバーヘッド含む）を返す。
  • **is_empty**: (MemoryContext) が空かどうかを bool で返す。
  • **stats**: (MemoryContext, ...) を通じて統計情報を出力または集約する。

前提条件と不変条件

  • alloc および realloc は、MCXT_ALLOC_HUGE および MCXT_ALLOC_NO_OOM フラグを適切に処理しなければならない。
  • MCXT_ALLOC_ZERO フラグの処理は呼び出し元で行われる。
  • MemoryContext は構造体へのポインタとして扱われる。

アルゴリズムまたは振る舞い

メモリ管理の操作（palloc等）が呼び出されると、対象の MemoryContext が保持する MemoryContextMethods の関数ポインタが間接的に呼び出される。これにより、コンテキストの型に応じた動的なメモリ管理が実行される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mmgr/mcxt.c 内のメモリ管理API（palloc 等）が、この構造体の関数ポインタを介して実装を呼び出す。
  • **呼び出し先**: src/backend/utils/mmgr/ 配下の具体的なメモリ管理実装（aset.c 等）が、この構造体の各メンバに具体的な関数を割り当てる。

関連データ構造

  • MemoryContextData: メモリコンテキストの基底構造体。この構造体の methods フィールドとして保持される。
  • MemoryContextCounters: stats メソッドで使用される統計情報構造体。

失敗モードと境界条件

  • alloc や realloc においてメモリ確保に失敗した場合、MCXT_ALLOC_NO_OOM フラグの有無に応じてエラーハンドリングを行う必要がある。
  • check メソッド（定義時）は、コンテキストの整合性を検証し、異常があれば WARNING を発行する。

保守担当者向けメモ

  • 新しいメモリコンテキスト型を追加する際は、この構造体の全関数ポインタを実装し、MemoryContextData の methods に設定する必要がある。
  • MEMORY_CONTEXT_CHECKING の有無で構造体のサイズが変わる点に注意すること。
