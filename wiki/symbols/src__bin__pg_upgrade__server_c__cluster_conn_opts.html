cluster_conn_opts

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/server.c:93

目的

PostgreSQL クラスターへの接続に必要なコマンドラインオプション文字列を生成します。主に pg_dump や pg_restore などの外部コマンド実行時に渡す引数を作成するために使用されます。

シグネチャまたは形状

char *cluster_conn_opts(ClusterInfo *cluster);

入力と出力

  • **入力**: ClusterInfo *cluster（接続先クラスターの情報を含む構造体）
  • **出力**: char *（--host、--port、--username を含むシェルエスケープ済みのオプション文字列）

前提条件と不変条件

  • os_info.user が適切に設定されている必要があります。
  • 内部で静的バッファ (static PQExpBuffer) を使用しているため、本関数を連続して呼び出すと以前の戻り値の内容は上書きされます。

アルゴリズムまたは振る舞い

1. 静的バッファ buf が未初期化であれば createPQExpBuffer() で作成し、初期化済みであれば resetPQExpBuffer() で内容をクリアします。 2. cluster->sockdir が存在する場合、--host オプションとソケットディレクトリのパスをシェルエスケープしてバッファに追加します。 3. cluster->port を用いて --port オプションを追加します。 4. os_info.user を用いて --username オプションをシェルエスケープして追加します。 5. バッファ内の文字列データ (buf->data) を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/dump.c
  • src/bin/pg_upgrade/pg_upgrade.c
  • **呼び出し先**:
  • createPQExpBuffer / resetPQExpBuffer (libpq)
  • appendPQExpBufferStr / appendPQExpBuffer / appendPQExpBufferChar (libpq)
  • appendShellString (fe_utils/string_utils.h)

関連データ構造

  • ClusterInfo: 接続先クラスターのポート番号やソケットディレクトリ情報を保持します。
  • PQExpBuffer: 文字列構築用のバッファ構造体です。

失敗モードと境界条件

  • メモリ不足により createPQExpBuffer が失敗した場合、プログラムは終了する可能性があります（libpq の実装に依存）。
  • cluster ポインタが NULL の場合、セグメンテーション違反が発生します。

保守担当者向けメモ

  • 本関数は静的バッファを再利用するため、戻り値を長期間保持したり、複数の呼び出し結果を同時に参照したりすることはできません。必要に応じて呼び出し側で文字列をコピーしてください。
