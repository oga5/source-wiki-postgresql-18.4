get_source_line

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_elog.c:447

目的

指定されたソースコード文字列から、特定の行番号に対応するソース行を抽出して返します。

シグネチャまたは形状

static char *get_source_line(const char *src, int lineno)

入力と出力

  • **入力**:
  • src: ソースコード全体を含むヌル終端文字列。
  • lineno: 取得対象の行番号（1始まり）。
  • **出力**:
  • 抽出された行の文字列（pstrdup または pnstrdup でメモリ確保されたもの）。
  • 指定行が見つからない場合や lineno <= 0 の場合は NULL。

前提条件と不変条件

  • lineno は 1 以上である必要があります。
  • 戻り値は PostgreSQL のメモリコンテキストで確保されるため、呼び出し元で適切に管理する必要があります。

アルゴリズムまたは振る舞い

1. lineno <= 0 の場合は直ちに NULL を返します。 2. 改行文字（\n）を区切りとして、指定された lineno に到達するまでソース文字列を走査します。 3. 指定行が見つかった場合、行頭の空白文字をスキップします。 4. 行の末尾（次の改行または文字列終端）までを切り出し、メモリを確保して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_elog.c:316 にて、エラー発生時のコンテキスト情報取得のために呼び出されています。
  • **呼び出し先**: strchr（標準ライブラリ）、isspace（標準ライブラリ）、pstrdup（PostgreSQL）、pnstrdup（PostgreSQL）。

関連データ構造

  • exec_ctx->curr_proc->src: 呼び出し元で使用されるソースコード文字列へのポインタ。

失敗モードと境界条件

  • lineno がソースの総行数を超えている場合: NULL を返します。
  • lineno <= 0 の場合: NULL を返します。
  • 行がすべて空白文字で構成されている場合: 空の文字列または NULL を返す可能性があります（next < s のチェックにより安全性を確保）。

保守担当者向けメモ

  • 本関数は static であり、plpy_elog.c 内でのみ使用されます。
  • Python のスタックトレースから得られた行番号と、PL/Python が保持するソースコードの整合性が取れていることが前提となります。
