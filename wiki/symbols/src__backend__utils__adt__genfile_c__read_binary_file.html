read_binary_file

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/genfile.c:102

目的

サーバー上の指定されたファイルからバイナリデータを読み込み、PostgreSQLの bytea 型として返却する。

シグネチャまたは形状

bytea *read_binary_file(const char *filename, int64 seek_offset, int64 bytes_to_read, bool missing_ok)

入力と出力

  • **入力**:
  • filename: 読み込むファイルのパス。
  • seek_offset: 読み込み開始位置（0以上で先頭からのオフセット、負の値で末尾からのオフセット）。
  • bytes_to_read: 読み込むバイト数（負の値の場合はファイル末尾まで読み込む）。
  • missing_ok: true の場合、ファイルが存在しない場合にエラーとせず NULL を返す。
  • **出力**:
  • 読み込んだデータを含む bytea 型のポインタ。

前提条件と不変条件

  • bytes_to_read が MaxAllocSize - VARHDRSZ を超える場合はエラーとなる。
  • ファイル操作には AllocateFile および FreeFile を使用し、PostgreSQLのファイルディスクリプタ管理下で実行される。

アルゴリズムまたは振る舞い

1. **検証**: 要求サイズがメモリ割り当て制限を超えていないか確認する。 2. **オープン**: AllocateFile でファイルを開く。存在しない場合、missing_ok が true なら NULL を返し、そうでなければエラーを報告する。 3. **シーク**: fseeko を使用して指定されたオフセットへ移動する。 4. **読み込み**:

  • bytes_to_read >= 0 の場合: 指定されたサイズ分を palloc で確保したバッファに読み込む。
  • bytes_to_read < 0 の場合: StringInfo を用いてファイル末尾まで動的にバッファを拡張しながら読み込む。

    • 5. **終了処理**: ferror で読み込みエラーを確認し、FreeFile でファイルを閉じる。 6. **返却**: 読み込んだデータサイズを SET_VARSIZE で設定し、bytea を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/genfile.c 内の read_text_file（推測される参照）
  • src/backend/utils/adt/genfile.c 内の pg_read_binary_file_common
  • src/backend/utils/adt/genfile.c 内の pg_read_file_common（推測される参照）
  • **呼び出し先**:
  • AllocateFile, FreeFile (storage/fd.h)
  • palloc, ereport, fseeko, fread, initStringInfo, enlargeStringInfo

関連データ構造

  • bytea: PostgreSQLの可変長バイナリデータ型。
  • StringInfoData: 可変長文字列/バイナリ構築用構造体。

失敗モードと境界条件

  • **ファイルオープン失敗**: missing_ok が false の場合、またはファイルアクセス権限がない場合にエラー。
  • **シーク失敗**: fseeko が失敗した場合にエラー。
  • **読み込みエラー**: fread 中にエラーが発生した場合にエラー。
  • **サイズ超過**: 読み込み要求サイズが MaxAllocSize を超える場合、またはファイルが大きすぎてメモリ制限に達する場合にエラー。

保守担当者向けメモ

  • 本関数は低レベルなファイル読み込みを担当するため、呼び出し元で必ず convert_and_check_filename 等によるパスの検証と権限チェックが行われていることを前提としている。
  • bytes_to_read < 0 の場合の読み込みロジックは、メモリ消費を抑えつつファイル全体を読み込むために StringInfo を利用している。修正時はメモリ割り当ての挙動に注意すること。
