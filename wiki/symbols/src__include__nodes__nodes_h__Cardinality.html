Cardinality

定義

  • 種別: typedef
  • 定義箇所: src/include/nodes/nodes.h:258

目的

クエリ処理における行数やその他の整数カウントの推定値を保持するために使用されます。

シグネチャまたは形状

typedef double Cardinality;

入力と出力

  • 入力: double 型の数値。
  • 出力: double 型の数値。

前提条件と不変条件

  • 統計的な推定値（行数など）を表現することを意図しています。
  • 整数値として扱う必要がある場合は、明示的な変換（clamp_cardinality_to_long 等）が必要です。

アルゴリズムまたは振る舞い

  • 本質的には double のエイリアスであり、計算コストや行数の見積もりにおいて、コードの意図を明確にするための型として機能します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/include/nodes/parsenodes.h: enrtuples フィールド
  • src/include/nodes/pathnodes.h: total_table_pages, limit_tuples, rows, tuples, ppi_rows フィールド
  • src/backend/optimizer/path/costsize.c: clamp_cardinality_to_long 関数
  • 呼び出し先:
  • 特定の関数呼び出しは定義されていません。

関連データ構造

  • Selectivity や Cost と同様の目的で定義された、ノード関連の数値型です。

失敗モードと境界条件

  • double 型であるため、非常に大きな値や精度上の制約を受けます。
  • 整数値として利用する際、範囲外の値や負の値が渡された場合の挙動は、利用側の関数（例: clamp_cardinality_to_long）の実装に依存します。

保守担当者向けメモ

  • Node 構造体の一部として利用されることが多いため、この型の定義を変更する場合は、関連する pathnodes.h や parsenodes.h への影響を考慮してください。
