LogicalRepTupleData

定義

  • 種別: struct
  • 定義箇所: src/include/replication/logicalproto.h:84

目的

論理レプリケーションにおいて、リモート側から受信したタプルの値を保持するためのメモリ表現を提供します。

シグネチャまたは形状

typedef struct LogicalRepTupleData
{
    StringInfoData *colvalues; /* 各列ごとの値（StringInfo） */
    char           *colstatus; /* 各列ごとの状態（null/unchanged/text/binary） */
    int             ncols;     /* 配列の長さ（列数） */
} LogicalRepTupleData;

入力と出力

  • **入力**: 論理レプリケーションのワイヤプロトコル経由で受信したバイトストリームから、logicalrep_read_tuple 等の関数を通じてデータが読み込まれます。
  • **出力**: 読み込まれたデータは colvalues（列データ）および colstatus（列の状態を示すマーカー）として格納され、レプリケーションワーカーがローカルテーブルへ適用するために利用します。

前提条件と不変条件

  • colvalues および colstatus は ncols のサイズを持つ配列である必要があります。
  • リモート側のテーブル構造に基づいて構築されるため、ローカル側のテーブル定義と一致しているとは限りません。

アルゴリズムまたは振る舞い

1. **デシリアライズ**: logicalrep_read_tuple 関数がストリームから列数と各列のステータス、値を読み取り、構造体の各フィールドに割り当てます。 2. **データ適用**: レプリケーションワーカーは、この構造体を参照して、更新対象の列データをローカルのタプルスロットへ反映します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/proto.c: logicalrep_read_insert, logicalrep_read_delete 等のデコード関数。
  • src/backend/replication/logical/worker.c: レプリケーションワーカーの適用処理。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • StringInfoData: colvalues の各要素として使用される、可変長文字列バッファ。
  • LogicalRepRelation: リモート側のテーブル定義を保持する構造体。

失敗モードと境界条件

  • ncols が 0 の場合、配列へのアクセスは発生しません。
  • リモートとローカルで列数が一致しない場合、適用時に不整合が発生する可能性があります。

保守担当者向けメモ

  • colstatus の各文字は、列の値が NULL か、変更なし（unchanged）か、テキスト形式か、バイナリ形式かを示すマーカーとして機能します。
  • 本構造体はリモートのスキーマに依存するため、スキーマ変更時の互換性には注意が必要です。
