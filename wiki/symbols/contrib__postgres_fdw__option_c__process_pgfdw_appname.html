process_pgfdw_appname

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/option.c:499

目的

外部サーバー接続時の application_name に含まれる % エスケープシーケンスを、現在のプロセス情報（プロセスID、データベース名、ユーザー名など）で置換し、動的なアプリケーション名を生成する。

シグネチャまたは形状

char *process_pgfdw_appname(const char *appname)

入力と出力

  • **入力**: appname (置換対象の文字列)
  • **出力**: 置換後の文字列を保持する palloc されたメモリ領域へのポインタ。

前提条件と不変条件

  • StringInfoData を使用して動的に文字列を構築する。
  • 呼び出し元は、返された文字列のメモリ管理（解放）に責任を持つ必要がある。

アルゴリズムまたは振る舞い

1. initStringInfo でバッファを初期化する。 2. 入力文字列を走査し、% 以外の文字はそのままバッファにコピーする。 3. % を検出した場合、次の文字を読み取り、以下のエスケープシーケンスを処理する：

  • %%: % リテラル
  • %a: application_name (GUC)
  • %c: MyStartTime と MyProcPid の組み合わせ
  • %C: cluster_name
  • %d: 現在のデータベース名 (MyProcPort->database_name)
  • %p: 現在のプロセスID (MyProcPid)
  • %u: 現在のユーザー名 (MyProcPort->user_name)

    • 4. 未定義のシーケンスや、% で終わる不正な形式は無視される。 5. 最終的なバッファの内容（buf.data）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/connection.c:542
  • **呼び出し先**: initStringInfo, appendStringInfoChar, appendStringInfoString, appendStringInfo (すべて PostgreSQL 内部ユーティリティ)

関連データ構造

  • StringInfoData: 文字列構築用バッファ。
  • MyProcPort: 現在の接続ポート情報（データベース名、ユーザー名の取得に使用）。

失敗モードと境界条件

  • MyProcPort が NULL の場合、%d や %u の置換は "[unknown]" となる。
  • 文字列の末尾が単独の % で終わる場合、その % は無視される。
  • 不明なエスケープシーケンスは無視され、何も出力されない。

保守担当者向けメモ

  • 新しいエスケープシーケンスを追加する場合は、switch 文にケースを追加すること。
  • MyProcPort へのアクセスは、バックエンドプロセス内での実行を前提としている。
