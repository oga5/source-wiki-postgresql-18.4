compute_remaining_iovec

定義

  • 種別: function
  • 定義箇所: src/common/file_utils.c:613

目的

ベクトルI/O操作において、部分的に転送が完了した iovec 配列から、未転送のデータ範囲を特定し、配列の先頭へ再配置する。

シグネチャまたは形状

int compute_remaining_iovec(struct iovec *destination,
                            const struct iovec *source,
                            int iovcnt,
                            size_t transferred)

入力と出力

  • **入力**:
  • destination: 未転送データを格納する出力先配列。
  • source: 現在の iovec 配列のポインタ。
  • iovcnt: source 配列内の要素数。
  • transferred: カーネルによって既に転送された合計バイト数。
  • **出力**:
  • 戻り値: 未転送データが残っている iovec の個数。すべて転送済みの場合は 0。

前提条件と不変条件

  • iovcnt > 0 であること。
  • transferred は、要求された合計転送量を超えないこと（カーネルの挙動が期待通りであること）。
  • destination は、source と同じか、少なくとも iovcnt 分の struct iovec を保持できる領域を指していること。

アルゴリズムまたは振る舞い

1. transferred を用いて、完全に転送済みの iovec 要素をスキップし、source ポインタを進め iovcnt を減らす。 2. 全ての要素が転送済みとなった場合、transferred が 0 であることを確認し 0 を返す。 3. 未転送の要素が残っている場合、source から destination へ残りの要素を memmove でコピーする。 4. 先頭の iovec 要素について、部分的に転送された分を iov_base のオフセット加算と iov_len の減算によって調整する。 5. 調整後の残りの要素数 iovcnt を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/smgr/md.c (複数箇所)
  • src/common/file_utils.c:692 (pg_pwritev_with_retry 内)
  • **呼び出し先**:
  • 標準ライブラリ関数: memmove

関連データ構造

  • struct iovec (POSIX定義のベクトルI/O構造体)

失敗モードと境界条件

  • iovcnt が 0 になる場合、転送が完全に完了したと見なし 0 を返す。
  • transferred が 0 の場合、配列のコピーのみが行われ、元の個数がそのまま返される。
  • Assert により、カーネルが要求以上のデータを転送した場合や、iovcnt が不正な場合にプログラムを停止させる。

保守担当者向けメモ

  • 本関数は pg_pwritev_with_retry 等の堅牢なI/O操作において、部分書き込み発生時の再試行ループを制御するために不可欠である。
  • destination と source が同一のメモリ領域を指す場合でも memmove を使用しているため安全である。
