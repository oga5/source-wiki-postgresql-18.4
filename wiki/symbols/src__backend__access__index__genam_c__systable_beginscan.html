systable_beginscan

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/genam.c:387

目的

システムカタログに対するスキャンを開始します。インデックスの使用可否を自動的に判定し、インデックスが利用可能な場合はそれを使用し、不可能な場合はヒープスキャンへ透過的にフォールバックします。

シグネチャまたは形状

SysScanDesc systable_beginscan(Relation heapRelation, Oid indexId, bool indexOK, Snapshot snapshot, int nkeys, ScanKey key);

入力と出力

  • **入力**:
  • heapRelation: スキャン対象のシステムテーブル（リレーション）。
  • indexId: 使用するインデックスのOID。
  • indexOK: インデックスの使用を許可するかどうかのフラグ。
  • snapshot: スキャンに使用するスナップショット。NULLの場合はカタログスナップショットが自動登録される。
  • nkeys: スキャンキーの数。
  • key: スキャン条件を定義する ScanKey 配列。
  • **出力**:
  • SysScanDesc: スキャン状態を保持する記述子。

前提条件と不変条件

  • indexOK が真であっても、IgnoreSystemIndexes が設定されている場合や、対象インデックスが Reindex 処理中の場合は、強制的にヒープスキャンが選択されます。
  • ヒープスキャンへフォールバックする場合、同期スキャン（synchronized scan）は無効化されます。
  • snapshot が提供されない場合、GetCatalogSnapshot を用いて取得・登録されます。

アルゴリズムまたは振る舞い

1. **インデックスの判定**: indexOK、IgnoreSystemIndexes、ReindexIsProcessingIndex の状態に基づき、インデックスを開くか（index_open）を決定します。 2. **スキャン記述子の初期化**: SysScanDesc を割り当て、ヒープリレーションとスロットを初期化します。 3. **スナップショット管理**: スナップショットが未指定の場合、カタログスナップショットを登録し、スキャン終了時に解放可能な状態にします。 4. **スキャン実行**:

  • **インデックス使用時**: ScanKey の属性番号をインデックス列番号に変換し、index_beginscan および index_rescan を呼び出します。
  • **ヒープスキャン時**: table_beginscan_strat を呼び出し、ヒープ全体をスキャンします。

    • 5. **トランザクション追跡**: CheckXidAlive が有効な場合、システムテーブルスキャン中であることを示すフラグ bsysscan を立てます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/dblink/dblink.c、contrib/sepgsql/ 配下の各ファイル、src/backend/access/common/toast_internals.c など。
  • **呼び出し先**: index_open, table_slot_create, RegisterSnapshot, GetCatalogSnapshot, index_beginscan, index_rescan, table_beginscan_strat 等。

関連データ構造

  • SysScanDesc: スキャン状態を保持する構造体。
  • Relation: システムテーブルのリレーション。
  • ScanKey: スキャン条件。
  • Snapshot: MVCC スナップショット。

失敗モードと境界条件

  • key で指定された属性番号がインデックスに含まれていない場合、elog(ERROR, "column is not in index") が発生します。
  • システムカタログのインデックスが破損している場合、呼び出し元で indexOK=false を指定することでヒープスキャンへ回避可能です。

保守担当者向けメモ

  • 本関数は MVCC をバイパスする操作や、システムカタログの整合性維持に直結するため、変更には細心の注意が必要です。
  • bsysscan フラグの管理は、トランザクションの生存確認と連動しているため、xact.c のコメントも併せて参照してください。
