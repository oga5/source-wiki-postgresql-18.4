tfuncFetchRows

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTableFuncscan.c:267

目的

テーブル関数（XMLTABLE や JSON_TABLE 等）の実行において、入力ドキュメントを解析し、全行を tuplestore にロードしてマテリアライズする。

シグネチャまたは形状

static void tfuncFetchRows(TableFuncScanState *tstate, ExprContext *econtext)

入力と出力

  • **入力**:
  • tstate: テーブル関数スキャンノードの状態構造体（TableFuncScanState）。
  • econtext: 式評価コンテキスト。
  • **出力**: なし（副作用として tstate->tupstore に結果が格納される）。

前提条件と不変条件

  • tstate->opaque が NULL であること。
  • tstate->perTableCxt が、1つのテーブル関数呼び出しのライフサイクルに紐づくメモリコンテキストとして適切に管理されていること。

アルゴリズムまたは振る舞い

1. tuplestore_begin_heap を使用して、結果格納用の tuplestore を初期化する。 2. tstate->perTableCxt にメモリコンテキストを切り替え、routine->InitOpaque を呼び出して解析エンジンを初期化する。 3. ExecEvalExpr でドキュメント式を評価する。 4. 結果が非 NULL の場合:

  • tfuncInitialize で名前空間やフィルタを設定する。
  • 序数カウンタ（ordinal）を 1 に初期化する。
  • tfuncLoadRows を呼び出し、全行を解析して tuplestore にロードする。

    • 5. PG_CATCH ブロックにより、エラー発生時に routine->DestroyOpaque を確実に呼び出す。 6. 最後に routine->DestroyOpaque を呼び出し、メモリコンテキストを復元・リセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeTableFuncscan.c 内の TableFuncNext（初回実行時）。
  • **呼び出し先**:
  • tuplestore_begin_heap
  • routine->InitOpaque / routine->DestroyOpaque
  • ExecEvalExpr
  • tfuncInitialize
  • tfuncLoadRows

関連データ構造

  • TableFuncScanState: スキャンノードの実行状態を保持。
  • TableFuncRoutine: XML/JSON 解析のための関数ルーチン定義。
  • tuplestore: 抽出された行を保持するメモリ/ディスク上のストレージ。

失敗モードと境界条件

  • ドキュメント式が NULL を返した場合、行はロードされず即座に終了する。
  • 巨大な入力ドキュメントを処理する場合、tuplestore が work_mem を超えるとディスクへ溢れる可能性がある。
  • エラー発生時は PG_RE_THROW により例外が再スローされる。

保守担当者向けメモ

  • 本関数は「マテリアライズ型」スキャンの中核であり、一度の呼び出しで全行を処理する。
  • opaque ポインタは解析エンジン固有の状態保持に使用されるため、ライフサイクル管理（Init / Destroy）の整合性に注意すること。
  • perTableCxt はクエリ全体よりも短く、タプル単位よりも長い生存期間を持つため、メモリリークの温床にならないよう注意が必要。
