plpgsql_parser_setup

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_comp.c:984

目的

SQL 式の解析中に、PL/pgSQL 特有の変数参照やパラメータを解決するためのパーサーフックを設定します。

シグネチャまたは形状

void plpgsql_parser_setup(struct ParseState *pstate, PLpgSQL_expr *expr)

入力と出力

  • **入力**:
  • pstate: PostgreSQL のパーサー状態を保持する構造体。
  • expr: 解析対象となる PL/pgSQL 式。
  • **出力**: なし（pstate のフック関数ポインタを更新する副作用を持つ）。

前提条件と不変条件

  • pstate は有効な ParseState 構造体を指している必要があります。
  • expr は解析対象の式として有効な PLpgSQL_expr である必要があります。

アルゴリズムまたは振る舞い

1. pstate の p_pre_columnref_hook に plpgsql_pre_column_ref を割り当てます。 2. pstate の p_post_columnref_hook に plpgsql_post_column_ref を割り当てます。 3. pstate の p_paramref_hook に plpgsql_param_ref を割り当てます。 4. pstate の p_ref_hook_state に expr を格納し、フック関数が解析対象の式情報にアクセスできるようにします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpgsql/src/pl_exec.c: estate->paramLI->parserSetup および options.parserSetup として設定されます。
  • **呼び出し先**:
  • 直接的な関数呼び出しは行いませんが、設定されたフックを通じて plpgsql_pre_column_ref、plpgsql_post_column_ref、plpgsql_param_ref が後続の解析プロセスで呼び出されます。

関連データ構造

  • ParseState: PostgreSQL のパーサー状態。
  • PLpgSQL_expr: PL/pgSQL の式を表現する構造体。

失敗モードと境界条件

  • pstate が NULL の場合、セグメンテーションフォールトが発生する可能性があります。

保守担当者向けメモ

  • 本関数は ParserSetupHook 型として使用されます。
  • p_coerce_param_hook は使用されていません。
  • フック関数（plpgsql_pre_column_ref 等）は、p_ref_hook_state を介して渡された PLpgSQL_expr を参照して名前解決を行います。
