ECPGset_noind_null

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/misc.c:291

目的

インジケータ変数を使用しない環境において、ECPGの型システムで定義された「NULL」を表す特殊値を、指定されたメモリ領域に書き込む。

シグネチャまたは形状

void ECPGset_noind_null(enum ECPGttype type, void *ptr)

入力と出力

  • type: 設定するデータの型（enum ECPGttype）。
  • ptr: NULL値を書き込む対象のメモリ領域へのポインタ。
  • 出力: なし（ptr が指すメモリの内容を直接変更する）。

前提条件と不変条件

  • ptr は type に対応する有効なメモリ領域を指している必要がある。
  • ECPGis_noind_null と対になっており、ここでセットされた値は後続の判定処理でNULLとして認識される。

アルゴリズムまたは振る舞い

type に応じて以下の値を ptr に書き込む：

  • 文字列系 (char, string 等): '\0' を設定。
  • 整数系 (short, int, long 等): 各型の最小値 (SHRT_MIN, INT_MIN, LONG_MIN 等) を設定。
  • 浮動小数点系 (float, double): memset を用いて全ビットを 0xff で埋める。
  • varchar / bytea: 長さ (len) を 0 に設定。
  • decimal / numeric: sign フィールドに NUMERIC_NULL を設定。
  • interval / timestamp: memset を用いて全ビットを 0xff で埋める。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/compatlib/informix.c、src/interfaces/ecpg/ecpglib/data.c、src/interfaces/ecpg/ecpglib/sqlda.c からの呼び出しが確認されている。
  • 呼び出し先: memset（標準ライブラリ）。

関連データ構造

  • enum ECPGttype: 型の識別子。
  • struct ECPGgeneric_varchar: varchar 型の内部表現。
  • struct ECPGgeneric_bytea: bytea 型の内部表現。
  • decimal, numeric: 数値型構造体。

失敗モードと境界条件

  • ptr が NULL の場合、セグメンテーションフォールトが発生する可能性がある。
  • 未定義の type が渡された場合、何も行わずに終了する。

保守担当者向けメモ

  • 新しい型をECPGに追加した際は、本関数の switch 文にも対応するNULL表現を追加する必要がある。
  • ECPGis_noind_null との整合性を常に維持すること。
