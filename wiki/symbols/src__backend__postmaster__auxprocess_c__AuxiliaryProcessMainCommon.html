AuxiliaryProcessMainCommon

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/auxprocess.c:38

目的

PostgreSQLの補助プロセス（Auxiliary Process）が起動直後に行う共通の初期化処理を実行し、共有メモリへの接続およびプロセス状態のセットアップを行う。

シグネチャまたは形状

void AuxiliaryProcessMainCommon(void)

入力と出力

  • 入力: なし（Postmasterからフォークされた環境に依存）
  • 出力: なし（副作用としてプロセス状態を更新）

前提条件と不変条件

  • IsUnderPostmaster が真であること。
  • 呼び出し時の処理モードが InitProcessing であること。

アルゴリズムまたは振る舞い

1. **メモリ管理**: PostmasterContext を破棄し、不要なメモリを解放する。 2. **プロセス表示**: init_ps_display(NULL) によりプロセス状態表示を初期化する。 3. **インデックス制御**: IgnoreSystemIndexes を true に設定する。 4. **共有メモリ接続**: InitAuxiliaryProcess() を呼び出し、PGPROC を作成してLWLocksおよび共有メモリへのアクセス権を確立する。 5. **基盤初期化**: BaseInit() および ProcSignalInit() を実行する。 6. **リソース管理**: CreateAuxProcessResourceOwner() を呼び出し、トランザクション外でのバッファピン管理用リソースオーナーを作成する。 7. **統計情報**: pgstat 関連の初期化（beinit, bestart_initial, bestart_final）を行う。 8. **終了処理登録**: before_shmem_exit に ShutdownAuxiliaryProcess を登録し、プロセス終了時のLWLock解放を保証する。 9. **モード遷移**: 処理モードを NormalProcessing に変更する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • bgwriter.c, checkpointer.c, pgarch.c, startup.c, walsummarizer.c, walwriter.c
  • src/backend/replication/walreceiver.c
  • src/backend/storage/aio/method_worker.c
  • **呼び出し先**:
  • MemoryContextDelete, init_ps_display, InitAuxiliaryProcess, BaseInit, ProcSignalInit, CreateAuxProcessResourceOwner, pgstat_beinit, pgstat_bestart_initial, pgstat_bestart_final, before_shmem_exit, SetProcessingMode

関連データ構造

  • PGPROC: 共有メモリ上のプロセス管理構造体。
  • PostmasterContext: 破棄対象となるメモリコンテキスト。

失敗モードと境界条件

  • PostmasterContext が存在しない場合、削除処理はスキップされる。
  • InitAuxiliaryProcess 等の内部関数が失敗した場合、プロセスは異常終了する可能性がある。

保守担当者向けメモ

  • 本関数はトランザクションを実行しないプロセス向けに設計されている。
  • ShutdownAuxiliaryProcess の登録は、システム全体のデッドロックを防ぐために必須である。
  • InitPostgres とは異なり、軽量な初期化パスである点に注意すること。
