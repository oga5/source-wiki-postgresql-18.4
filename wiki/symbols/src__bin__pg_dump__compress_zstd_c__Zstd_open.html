Zstd_open

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_zstd.c:504

目的

Zstandard 圧縮を使用するファイルハンドルを初期化し、CompressFileHandle 構造体に Zstd 固有のコンテキストを割り当てる。

シグネチャまたは形状

static bool Zstd_open(const char *path, int fd, const char *mode, CompressFileHandle *CFH)

入力と出力

  • **入力**:
  • path: ファイルパス（fd < 0 の場合に使用）。
  • fd: ファイル記述子（fd >= 0 の場合に使用）。
  • mode: ファイルオープンモード（fopen / fdopen に渡される）。
  • CFH: 初期化対象の CompressFileHandle 構造体へのポインタ。
  • **出力**:
  • 成功時は true を返し、CFH->private_data に初期化された ZstdCompressorState を格納する。
  • 失敗時は false を返し、CFH->private_data を NULL に設定する。

前提条件と不変条件

  • CFH は有効なポインタであること。
  • ZstdCompressorState のメモリ確保に失敗した場合、errno が ENOMEM に設定される。

アルゴリズムまたは振る舞い

1. CFH->private_data を NULL に初期化する。 2. pg_malloc_extended を使用して ZstdCompressorState 用のメモリを確保する（ゼロ初期化）。 3. fd が 0 以上であれば dup(fd) を経由して fdopen を呼び出し、それ以外であれば path に対して fopen を呼び出す。 4. ファイルオープンに成功した場合、取得した FILE * を zstdcs->fp に格納し、CFH->private_data に zstdcs を割り当てて true を返す。 5. ファイルオープンに失敗した場合は、確保した zstdcs を解放して false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_zstd.c 内で CFH->open_func に代入されている。直接的な呼び出し元はソースコード上では明示されていない。
  • **呼び出し先**: pg_malloc_extended, dup, fdopen, fopen, pg_free。

関連データ構造

  • CompressFileHandle: 圧縮処理の抽象化ハンドル。
  • ZstdCompressorState: Zstd 圧縮・解凍のコンテキストを保持する構造体。

失敗モードと境界条件

  • メモリ不足時: pg_malloc_extended が失敗し、errno を ENOMEM に設定して false を返す。
  • ファイルオープン失敗時: fopen または fdopen が NULL を返し、確保済みの zstdcs を解放して false を返す。

保守担当者向けメモ

  • 本関数は USE_ZSTD マクロが定義されている環境でのみ有効です。
  • fd を渡す際に dup(fd) を行っているため、呼び出し側で元の fd を閉じる必要があるかどうかのライフサイクル管理に注意してください。
