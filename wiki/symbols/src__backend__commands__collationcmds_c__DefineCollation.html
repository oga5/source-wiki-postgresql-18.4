DefineCollation

定義

  • 種別: function
  • 定義箇所: src/backend/commands/collationcmds.c:52

目的

SQLの CREATE COLLATION コマンドを実行し、指定されたパラメータに基づいて新しい照合順序（Collation）オブジェクトをシステムカタログ pg_collation に定義する。

シグネチャまたは形状

ObjectAddress DefineCollation(ParseState *pstate, List *names, List *parameters, bool if_not_exists)

入力と出力

  • **入力**:
  • pstate: パーサの状態。
  • names: 照合順序の名称（修飾名）。
  • parameters: CREATE COLLATION 文で指定されたオプションのリスト（FROM, LOCALE, LC_COLLATE, LC_CTYPE, PROVIDER, DETERMINISTIC, RULES, VERSION）。
  • if_not_exists: IF NOT EXISTS 句の有無。
  • **出力**:
  • 作成された照合順序の ObjectAddress を返す。

前提条件と不変条件

  • 呼び出し元は、対象の名前空間に対する CREATE 権限を持っている必要がある。
  • LOCALE オプションと LC_COLLATE または LC_CTYPE を同時に指定することはできない。
  • FROM オプションが指定された場合、他のオプションを同時に指定することはできない。
  • COLLPROVIDER_DEFAULT（デフォルト照合順序）のコピーは許可されない。

アルゴリズムまたは振る舞い

1. 名前空間の解決と権限チェックを行う。 2. parameters リストを走査し、各オプション（DefElem）を解析・検証する。 3. FROM オプションがある場合、既存の照合順序をシステムキャッシュ（COLLOID）から検索し、その属性（プロバイダ、決定論的フラグ、エンコーディング、ロケール設定等）を継承する。 4. FROM オプションがない場合、指定されたプロバイダ（builtin, icu, libc）に基づき設定を初期化する。デフォルトは libc プロバイダ。 5. 認識されないオプションや矛盾する設定がある場合はエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c (SQLコマンド実行時)
  • **呼び出し先**:
  • QualifiedNameGetCreationNamespace
  • object_aclcheck
  • get_collation_oid
  • SearchSysCache1 / SysCacheGetAttr
  • defGetQualifiedName / defGetString / defGetBoolean
  • ereport (エラー報告)

関連データ構造

  • ObjectAddress: 作成されたオブジェクトの識別子。
  • DefElem: SQLコマンドの各パラメータを保持する構造体。
  • pg_collation: 照合順序情報を格納するシステムカタログ。

失敗モードと境界条件

  • 権限不足: aclcheck_error を発生させる。
  • 構文エラー: 未知の属性や矛盾するオプション指定時に ERRCODE_SYNTAX_ERROR を発生させる。
  • キャッシュ検索失敗: elog によりエラー終了する。
  • 不正なプロバイダ指定: ERRCODE_INVALID_OBJECT_DEFINITION を発生させる。

保守担当者向けメモ

  • 本関数は CREATE COLLATION の解析フェーズを担当する。カタログへの実際の挿入処理は、この後の工程（ソースコードの続き部分）で行われる。
  • COLLPROVIDER_DEFAULT の取り扱いには注意が必要であり、既存のコードベースとの整合性を維持すること。
  • プラットフォーム依存のロケール処理が含まれるため、移植時には pg_locale.h 関連の依存関係に留意すること。
