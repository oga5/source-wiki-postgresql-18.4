create_target_symlink

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/file_ops.c:273

目的

ターゲットデータディレクトリ内にシンボリックリンクを作成します。

シグネチャまたは形状

static void create_target_symlink(const char *path, const char *link)

入力と出力

  • path: 作成するシンボリックリンクの相対パス。
  • link: シンボリックリンクが指し示すターゲットパス。
  • 出力: なし（成功時）。失敗時は pg_fatal によりプロセスを終了します。

前提条件と不変条件

  • path は path_is_safe_for_extraction によって安全性が検証されている必要があります。
  • グローバル変数 datadir_target がターゲットディレクトリのベースパスとして設定されている必要があります。

アルゴリズムまたは振る舞い

1. path_is_safe_for_extraction(path) を呼び出し、パスが安全でない場合はエラーで終了します。 2. グローバル変数 dry_run が真であれば、何もせず処理を終了します。 3. datadir_target と path を結合し、絶対パス dstpath を生成します。 4. symlink(link, dstpath) を実行し、リンクを作成します。 5. 作成に失敗した場合は pg_fatal を呼び出し、エラーメッセージを出力して終了します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_rewind/file_ops.c 内の create_target 関数から呼び出されます。
  • 呼び出し先: path_is_safe_for_extraction（セキュリティチェック）、symlink（OSシステムコール）、pg_fatal（エラーハンドリング）。

関連データ構造

  • datadir_target（グローバル変数）: ターゲットディレクトリのパス。
  • dry_run（グローバル変数）: ドライランモードのフラグ。

失敗モードと境界条件

  • path_is_safe_for_extraction が偽を返した場合、pg_fatal により異常終了します。
  • symlink システムコールが失敗した場合（権限不足、パスの競合など）、pg_fatal により異常終了します。

保守担当者向けメモ

  • 本関数は pg_rewind のファイル同期プロセスの一部であり、ターゲットディレクトリ外への不正なシンボリックリンク作成を防止するセキュリティチェックが必須です。
  • dry_run モードのサポートを維持するため、新しいファイル操作を追加する際は必ずフラグチェックを組み込んでください。
