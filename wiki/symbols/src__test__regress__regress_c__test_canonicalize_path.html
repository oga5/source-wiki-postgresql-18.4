test_canonicalize_path

定義

  • 種別: function
  • 定義箇所: src/test/regress/regress.c:324

目的

PostgreSQL のパス正規化ロジックを SQL レベルからテストするための補助関数です。

シグネチャまたは形状

Datum test_canonicalize_path(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_TEXT_PP(0) を介して受け取る text 型のパス文字列。
  • **出力**: PG_RETURN_TEXT_P を介して返される、正規化後の text 型文字列。

前提条件と不変条件

  • 入力引数として有効なパス文字列が提供されていること。
  • canonicalize_path 関数が呼び出し可能な環境であること。

アルゴリズムまたは振る舞い

1. text_to_cstring を使用して、入力された text 型のパスを C 文字列に変換します。 2. canonicalize_path(path) を呼び出し、パスの正規化（冗長なセパレータの削除や . の解決など）をインプレースで実行します。 3. 正規化された C 文字列を cstring_to_text で text 型に再変換し、呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 マクロによりエクスポートされており、SQL クエリから呼び出し可能）。
  • **呼び出し先**: text_to_cstring, canonicalize_path, cstring_to_text。

関連データ構造

  • Datum: PostgreSQL の汎用データ型。
  • text: PostgreSQL の可変長文字列型。

失敗モードと境界条件

  • 入力が NULL の場合、PG_GETARG_TEXT_PP の挙動に従いエラーとなる可能性があります。
  • canonicalize_path の内部仕様に依存し、不正なパス形式に対する挙動は当該関数に準じます。

保守担当者向けメモ

  • 本関数は回帰テスト専用です。本番環境のコードパスには影響を与えません。
  • canonicalize_path の実装変更に伴う回帰テストとして利用してください。
