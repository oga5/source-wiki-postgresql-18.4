shm_toc_estimate

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shm_toc.c:262

目的

動的共有メモリ（DSM）セグメント内に shm_toc 構造体および関連するエントリ、データチャンクを格納するために必要な合計バイト数を見積もる。

シグネチャまたは形状

Size shm_toc_estimate(shm_toc_estimator *e)

入力と出力

  • **入力**: shm_toc_estimator *e（見積もりに必要なキー数 number_of_keys およびデータ領域サイズ space_for_chunks を保持する構造体へのポインタ）
  • **出力**: Size（必要な共有メモリの合計サイズ。BUFFERALIGN によるアライメント調整済み）

前提条件と不変条件

  • 入力される shm_toc_estimator 構造体の各フィールドは、呼び出し前に適切に設定されている必要がある。
  • add_size および mul_size 関数により、計算過程でのオーバーフローが防止される。

アルゴリズムまたは振る舞い

1. shm_toc 構造体のヘッダー部分（toc_entry 配列の直前まで）のサイズを初期値とする。 2. キーの数（e->number_of_keys）に基づき、shm_toc_entry 配列に必要なサイズを加算する。 3. ユーザーが指定したデータ格納領域のサイズ（e->space_for_chunks）を加算する。 4. 最終的な合計サイズを BUFFERALIGN マクロでアライメント調整して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/session.c
  • src/backend/access/transam/parallel.c
  • src/backend/replication/logical/applyparallelworker.c
  • src/test/modules/test_shm_mq/setup.c
  • **呼び出し先**:
  • add_size（内部ユーティリティ）
  • mul_size（内部ユーティリティ）
  • BUFFERALIGN（マクロ）

関連データ構造

  • shm_toc: 目次を保持するメイン構造体。
  • shm_toc_estimator: 見積もりに必要なパラメータを保持する構造体。
  • shm_toc_entry: TOC 内の各エントリを定義する構造体。

失敗モードと境界条件

  • e->number_of_keys や e->space_for_chunks が極端に大きい場合、add_size / mul_size の内部ロジックにより安全に処理される（オーバーフロー時はエラーを発生させる設計）。
  • アライメント調整により、計算値よりもわずかに大きな値が返される場合がある。

保守担当者向けメモ

  • 本関数は shm_toc の構造変更（メンバーの追加や削除）が発生した際に、必ず offsetof(shm_toc, toc_entry) の計算が正しいか確認すること。
  • 共有メモリの割り当てサイズを決定する重要な起点となるため、見積もりロジックの変更はメモリ不足やセグメント破壊のリスクを伴う。
