diinsert

定義

  • 種別: function
  • 定義箇所: src/test/modules/dummy_index_am/dummy_index_am.c:163

目的

PostgreSQLのインデックス・アクセスメソッド（AM）インターフェースにおける、インデックスへのデータ挿入処理のスタブ実装です。

シグネチャまたは形状

static bool
diinsert(Relation index, Datum *values, bool *isnull,
         ItemPointer ht_ctid, Relation heapRel,
         IndexUniqueCheck checkUnique,
         bool indexUnchanged,
         IndexInfo *indexInfo)

入力と出力

  • **入力**:
  • index: 挿入対象のインデックスリレーション。
  • values, isnull: 挿入する属性値とそのNULLフラグ配列。
  • ht_ctid: 挿入されるタプルのヒープ内TID。
  • heapRel: ヒープリレーション。
  • checkUnique: 一意性制約のチェックモード。
  • indexUnchanged: インデックス値が変更されていないかを示すフラグ。
  • indexInfo: インデックス情報構造体。
  • **出力**:
  • bool: 常に false を返します。

前提条件と不変条件

  • 本関数は dummy_index_am モジュールの一部であり、実際のデータ挿入やインデックス構造の更新は行いません。

アルゴリズムまたは振る舞い

  • 呼び出された際、内部で何も処理を行わず、直ちに false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのインデックス操作エンジン（IndexAmRoutine を介して呼び出されます）。
  • **呼び出し先**: 本関数内から他の関数を呼び出すことはありません。
  • **直接的な証拠**: src/test/modules/dummy_index_am/dummy_index_am.c:307 にて、amroutine->aminsert に本関数が代入されています。

関連データ構造

  • Relation: インデックスおよびヒープリレーションの定義。
  • IndexAmRoutine: 本関数が登録されるAMインターフェース構造体。
  • IndexInfo: インデックスに関するメタ情報。

失敗モードと境界条件

  • 処理を行わないため、本関数自体に起因する失敗モードはありません。

保守担当者向けメモ

  • 本関数はテンプレートとしてのスタブです。実際のインデックスAMを実装する際は、この関数を適切な挿入ロジックで置き換えてください。
