MultirangeIOData

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/adt/multirangetypes.c:49

目的

マルチレンジ型の入出力処理（I/O）において、型情報やプロシージャ情報を保持するためのキャッシュ構造体です。

シグネチャまたは形状

typedef struct MultirangeIOData
{
    TypeCacheEntry *typcache;    /* マルチレンジ型の型キャッシュエントリ */
    FmgrInfo       typioproc;    /* レンジ型のI/Oプロシージャ */
    Oid            typioparam;   /* レンジ型のI/Oパラメータ */
} MultirangeIOData;

入力と出力

本構造体は関数呼び出しのコンテキスト（FunctionCallInfo）内でキャッシュとして生成・保持されます。直接的な入出力関数ではありません。

前提条件と不変条件

  • typcache は、対象となるマルチレンジ型の有効なキャッシュエントリを指している必要があります。
  • typioproc は、レンジ型の入出力に必要な関数情報を含んでいる必要があります。

アルゴリズムまたは振る舞い

  • マルチレンジ型のI/O処理中に、型情報のルックアップコストを削減するために使用されます。
  • get_multirange_io_data 関数によって初期化され、fcinfo->flinfo->fn_extra に格納されます。
  • 初回アクセス時にメモリコンテキスト（fn_mcxt）へ割り当てられ、以降の呼び出しで再利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/multirangetypes.c 内の各種I/O関連関数（multirange_in, multirange_out, multirange_recv 等）。
  • **呼び出し先**: get_multirange_io_data 関数が本構造体の生成と管理を行います。

関連データ構造

  • TypeCacheEntry: PostgreSQLの型キャッシュ情報。
  • FmgrInfo: 関数マネージャが使用する関数呼び出し情報。
  • MultirangeType: マルチレンジ型のメモリ上の表現。

失敗モードと境界条件

  • メモリ割り当て失敗時（MemoryContextAlloc）にエラーが発生する可能性があります。
  • fcinfo->flinfo->fn_extra が初期化されていない場合、または不正な型情報が渡された場合の挙動は、呼び出し側の実装に依存します。

保守担当者向けメモ

  • 本構造体は fn_extra を介して永続化されるため、メモリリークを防ぐために適切なメモリコンテキスト（fn_mcxt）で割り当てられていることを確認してください。
  • レンジ型の定義変更に伴うキャッシュの無効化ロジックについては、typcache の仕様に従う必要があります。
