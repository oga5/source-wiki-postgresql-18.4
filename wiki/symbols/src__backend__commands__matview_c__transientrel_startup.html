transientrel_startup

定義

  • 種別: function
  • 定義箇所: src/backend/commands/matview.c:482

目的

マテリアライズド・ビューの更新処理において、一時テーブルへのデータ挿入を開始するための実行エンジン初期化を行う。

シグネチャまたは形状

static void transientrel_startup(DestReceiver *self, int operation, TupleDesc typeinfo)

入力と出力

  • self: DR_transientrel 構造体へのポインタ。
  • operation: 実行される操作の種類。
  • typeinfo: 挿入されるタプルの記述子。
  • 出力: なし（self が指す構造体のフィールドを更新する）。

前提条件と不変条件

  • self は DR_transientrel 構造体としてキャスト可能であること。
  • 対象となる一時テーブルの transientoid が self に設定されていること。
  • RelationGetTargetBlock(transientrel) が InvalidBlockNumber であること（未書き込み状態であること）。

アルゴリズムまたは振る舞い

1. myState->transientoid を用いて対象リレーションを table_open（ロックなし）で開く。 2. 開いたリレーションを myState->transientrel に格納する。 3. 現在のコマンドIDを myState->output_cid に設定する。 4. 挿入オプション ti_options を TABLE_INSERT_SKIP_FSM | TABLE_INSERT_FROZEN に設定する。 5. GetBulkInsertState() を呼び出し、一括挿入用の状態を myState->bistate に格納する。 6. アサーションにより、対象リレーションが既に書き込み済みでないことを検証する。

呼び出し元と呼び出し先

  • 呼び出し元: DestReceiver 構造体の rStartup フィールドを通じて呼び出される（直接の呼び出し元は実行エンジン）。
  • 呼び出し先: table_open, GetCurrentCommandId, GetBulkInsertState, RelationGetTargetBlock。

関連データ構造

  • DR_transientrel: 一時テーブルへのデータ挿入を管理する DestReceiver の拡張構造体。

失敗モードと境界条件

  • table_open が失敗した場合、例外が発生する。
  • RelationGetTargetBlock が InvalidBlockNumber でない場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は REFRESH MATERIALIZED VIEW のデータ生成フェーズの開始時に実行される。
  • TABLE_INSERT_SKIP_FSM および TABLE_INSERT_FROZEN を使用しているため、この処理は一時テーブルの初期構築に特化しており、通常のテーブル更新とは異なる最適化が行われている。
