pq_cleanup_redirect_to_shm_mq

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqmq.c:66

目的

共有メモリメッセージキュー（shm_mq）へのリダイレクト設定を解除し、関連するリソースの状態を初期化します。

シグネチャまたは形状

static void pq_cleanup_redirect_to_shm_mq(dsm_segment *seg, Datum arg)

入力と出力

  • **入力**:
  • seg: クリーンアップ対象の動的共有メモリセグメントへのポインタ。
  • arg: on_dsm_detach コールバック用の汎用データ（本関数では使用されません）。
  • **出力**: なし（副作用としてグローバル変数を更新）。

前提条件と不変条件

  • 本関数は on_dsm_detach コールバックとして登録されることを前提としています。
  • 実行後、pq_mq_handle は NULL にリセットされ、出力先指定 whereToSendOutput は DestNone に設定されます。

アルゴリズムまたは振る舞い

1. pq_mq_handle を NULL に設定し、共有メモリキューへの参照を破棄します。 2. whereToSendOutput を DestNone に設定し、出力先を無効化します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/pqmq.c 内の on_dsm_detach 登録処理から間接的に呼び出されます。
  • **呼び出し先**:
  • 提供されたソースコード内では、本関数が直接呼び出している関数はありません。

関連データ構造

  • dsm_segment: 動的共有メモリセグメントを管理する構造体。
  • pq_mq_handle: 共有メモリメッセージキューのハンドルを保持する内部変数。
  • whereToSendOutput: メッセージの出力先を制御するグローバル変数。

失敗モードと境界条件

  • 本関数自体はエラーを返さない設計ですが、on_dsm_detach の仕組みに依存しているため、共有メモリセグメントのデタッチタイミングが適切でない場合、意図しないタイミングで出力先が DestNone に切り替わる可能性があります。

保守担当者向けメモ

  • 本関数は、共有メモリセグメントがデタッチされる際に自動的に呼び出されるクリーンアップルーチンです。通信経路の切り替え状態を安全に終了させるために使用されます。
