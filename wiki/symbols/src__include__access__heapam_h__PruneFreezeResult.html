PruneFreezeResult

定義

  • 種別: struct
  • 定義箇所: src/include/access/heapam.h:225

目的

ヒープページに対する剪定（Prune）および凍結（Freeze）処理の結果を保持し、VACUUM 処理におけるページの状態更新や可視性マップ（VM）の更新判断を支援する。

シグネチャまたは形状

typedef struct PruneFreezeResult
{
    int         ndeleted;
    int         nnewlpdead;
    int         nfrozen;
    int         live_tuples;
    int         recently_dead_tuples;
    bool        all_visible;
    bool        all_frozen;
    TransactionId vm_conflict_horizon;
    bool        hastup;
    int         lpdead_items;
    OffsetNumber deadoffsets[MaxHeapTuplesPerPage];
} PruneFreezeResult;

入力と出力

  • **入力**: 剪定および凍結処理の実行結果（タプル数、オフセット、フラグ等）が格納される。
  • **出力**: VACUUM 処理において、ページが全可視・全凍結状態であるかの判定や、リレーション切り詰め（Truncation）の安全性の判断に使用される。

前提条件と不変条件

  • vm_conflict_horizon は、nfrozen > 0 かつ all_frozen が真である場合にのみ有効。
  • all_visible および all_frozen は、HEAP_PRUNE_FREEZE オプションが指定された場合にのみ設定される。
  • hastup は、ページ内に LP_DEAD 項目が存在する場合でも、ページがリレーション切り詰めに対して安全でないことを示すために true となる。

アルゴリズムまたは振る舞い

1. ページ内のタプルを走査し、不要なタプルを削除（ndeleted）および凍結（nfrozen）する。 2. 処理後の生存タプル数（live_tuples）および最近死亡したタプル数（recently_dead_tuples）を更新する。 3. ページ内の LP_DEAD 項目を deadoffsets 配列に記録し、その総数を lpdead_items に保持する。 4. ページの状態に基づき、可視性マップのビット設定可否（all_visible, all_frozen）を決定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/pruneheap.c および src/backend/access/heap/vacuumlazy.c において、剪定処理の結果を受け取るために使用されている。
  • **呼び出し先**: 構造体定義のため、特定の関数を呼び出すことはない。

関連データ構造

  • OffsetNumber: ページ内のタプル位置を示す。
  • TransactionId: トランザクション ID。
  • MaxHeapTuplesPerPage: ページ内に格納可能な最大タプル数。

失敗モードと境界条件

  • deadoffsets 配列は MaxHeapTuplesPerPage で固定されているため、ページ内の全タプルが LP_DEAD となった場合でもオーバーフローしない。
  • hastup が true の場合、VACUUM はリレーション切り詰め前に LP_DEAD 項目の削除を完了させる必要がある。

保守担当者向けメモ

  • 本構造体は HEAP_PRUNE_FREEZE オプションの有無によって設定されるフィールドが異なるため、利用時にはオプションの指定状況を確認すること。
  • vm_conflict_horizon の有効条件（nfrozen > 0 かつ all_frozen）は、可視性マップの更新ロジックにおいて厳密に守る必要がある。
