inet_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network.c:249

目的

バイナリ形式の入力ストリームから inet 型のデータを読み込み、PostgreSQL の内部表現（Datum）に変換します。

シグネチャまたは形状

Datum inet_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される StringInfo 型のバッファ（PG_GETARG_POINTER(0)）。
  • **出力**: inet 型の内部表現である Datum を返します。

前提条件と不変条件

  • 入力バッファ buf は、PostgreSQL のバイナリプロトコルに従ったネットワークデータを含んでいる必要があります。
  • 内部的に network_recv を呼び出すため、その関数が要求するメモリ管理およびデータ構造の制約に従います。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から StringInfo 形式のバッファを取得します。 2. network_recv(buf, false) を呼び出し、バイナリデータを解析して inet 構造体に変換します。ここで第2引数に false を渡すことで、cidr 型ではなく inet 型としての処理を指定しています。 3. PG_RETURN_INET_P マクロを使用して、変換された inet 構造体を Datum として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（SQL インターフェース経由）。
  • **呼び出し先**: network_recv（src/backend/utils/adt/network.c 内の共通ルーチン）。

関連データ構造

  • StringInfo: バイナリデータの読み込みに使用される可変長バッファ構造体。
  • inet: ネットワークアドレスを保持する内部データ構造。

失敗モードと境界条件

  • 入力バッファが不正な形式である場合、network_recv 内でエラーが発生し、トランザクションがアボートされる可能性があります。
  • メモリ不足が発生した場合、PostgreSQL のメモリ管理機構によりエラーがスローされます。

保守担当者向けメモ

  • 本関数は inet 型専用のラッパーです。cidr 型のバイナリ入力には cidr_recv が使用されます。
  • 共通ロジックの変更が必要な場合は、network_recv を確認してください。
