mcv_selectivity

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/selfuncs.c:739

目的

統計情報に含まれる最頻値（MCV: Most Common Values）リストを走査し、指定された演算子に基づいて条件が真となる確率（選択率）を推定する。

シグネチャまたは形状

double mcv_selectivity(VariableStatData *vardata, FmgrInfo *opproc, Oid collation,
                       Datum constval, bool varonleft,
                       double *sumcommonp)

入力と出力

  • **入力**:
  • vardata: 変数の統計情報（statsTuple を含む）。
  • opproc: 評価対象の演算子関数。
  • collation: 照合順序。
  • constval: 比較対象の定数値。
  • varonleft: 変数が演算子の左側にある場合は true。
  • sumcommonp: MCV リスト全体の頻度の合計を格納する出力用ポインタ。
  • **出力**:
  • 戻り値: MCV リスト内で条件を満たす値の頻度の合計（選択率）。

前提条件と不変条件

  • vardata->statsTuple が有効であること。
  • statistic_proc_security_check が成功すること。
  • get_attstatsslot により STATISTIC_KIND_MCV 統計情報が取得可能であること。

アルゴリズムまたは振る舞い

1. mcv_selec（条件合致頻度）と sumcommon（MCV 全体頻度）を 0.0 に初期化する。 2. get_attstatsslot で MCV リストを取得する。 3. InitFunctionCallInfoData を用いて、演算子関数を直接呼び出すための環境を構築する。 4. MCV リストの各要素に対してループ処理を行う：

  • varonleft の値に応じて、変数値と定数値を演算子の引数に割り当てる。
  • FunctionCallInvoke を用いて演算子を実行する。
  • 結果が真であれば、その要素の頻度（sslot.numbers[i]）を mcv_selec に加算する。
  • 常にその要素の頻度を sumcommon に加算する。

    • 5. free_attstatsslot でメモリを解放する。 6. sumcommon を sumcommonp に書き込み、mcv_selec を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/like_support.c
  • src/backend/utils/adt/network_selfuncs.c
  • src/backend/utils/adt/selfuncs.c（内部の複数箇所）
  • **呼び出し先**:
  • get_attstatsslot
  • FunctionCallInvoke
  • free_attstatsslot

関連データ構造

  • VariableStatData: 変数の統計メタデータ。
  • AttStatsSlot: pg_statistic から抽出された統計情報のスロット。
  • FmgrInfo: 関数マネージャ情報。

失敗モードと境界条件

  • 統計情報（MCV）が存在しない場合、mcv_selec は 0.0、sumcommon は 0.0 のまま返される。
  • 演算子関数が NULL を返す可能性がある場合、FunctionCallInvoke を直接呼び出すことで安全に処理する設計となっている。

保守担当者向けメモ

  • 本関数は pg_statistic の MCV 情報を直接操作する。演算子の適用順序（varonleft）を正しく扱う必要がある。
  • sumcommon は、MCV 以外のデータに対する選択率推定（ヒストグラム等）を行う際の補正値として利用されるため、正確な算出が重要である。
