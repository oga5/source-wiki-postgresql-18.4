cclasscvec

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/regex/regc_locale.c:568

目的

正規表現コンパイル時に、指定された文字クラス（char_classes）に対応する文字集合（cvec）を生成する。

シグネチャまたは形状

struct cvec *cclasscvec(struct vars *v, enum char_classes cclasscode, int cases)

入力と出力

  • **入力**:
  • v: コンパイルコンテキスト（struct vars）。
  • cclasscode: 生成対象の文字クラスを示す列挙型。
  • cases: 大文字小文字を区別しないマッチングを行うかを示すフラグ（非ゼロで有効）。
  • **出力**:
  • 指定された文字クラスを含む struct cvec へのポインタ。メモリ不足時は NULL を返し、内部で ERR(REG_ESPACE) を呼び出す。

前提条件と不変条件

  • cases が真で、かつ cclasscode が CC_LOWER または CC_UPPER の場合、内部的に CC_ALPHA として処理される。
  • switch 文の各ケースは、cclass_column_index() と同期している必要がある。

アルゴリズムまたは振る舞い

1. **正規化**: 大文字小文字を区別しない場合、LOWER/UPPER クラスを ALPHA に変換する。 2. **クラス判定**:

  • ロケール依存のクラス（ALPHA, DIGIT, SPACE 等）: pg_ctype_get_cache() を呼び出し、PostgreSQLのロケールキャッシュから結果を取得する。
  • ハードコードされたクラス（ASCII, BLANK, CNTRL, XDIGIT）: getcvec() で領域を確保し、addrange() や addchr() を用いて直接文字集合を構築する。

    • 3. **エラーハンドリング**: cv が NULL の場合、REG_ESPACE エラーを記録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regcomp.c: 正規表現のコンパイルプロセス内で、文字クラスの解決時に複数回呼び出される。
  • **呼び出し先**:
  • pg_ctype_get_cache(): ロケール依存の文字クラス取得用。
  • getcvec(): cvec 構造体の初期化用。
  • addrange() / addchr(): cvec への要素追加用。

関連データ構造

  • struct cvec: 文字集合を保持する構造体。
  • enum char_classes: regguts.h で定義される文字クラスの列挙型。
  • struct vars: コンパイル状態を保持する構造体。

失敗モードと境界条件

  • **メモリ不足**: getcvec() または pg_ctype_get_cache() が失敗して cv が NULL になった場合、ERR(REG_ESPACE) が発生する。
  • **ロケール依存性**: XDIGIT はロケールに関わらずハードコードされた定義を使用する。

保守担当者向けメモ

  • 本関数内の switch 文に新しい文字クラスを追加する際は、必ず cclass_column_index() にも同様の変更を加え、ロジックの整合性を維持すること。
  • pg_wc_* 系関数はデータベースのエンコーディングに依存するため、直接 libc の関数を呼び出さないこと。
