validate_string_relopt

定義

  • 種別: typedef
  • 定義箇所: src/include/access/reloptions.h:133

目的

リレーションオプションとして定義された文字列型の値に対して、妥当性検証を行うためのコールバック関数の型を定義します。

シグネチャまたは形状

typedef void (*validate_string_relopt) (const char *value);

入力と出力

  • **入力**: value (検証対象の文字列へのポインタ)
  • **出力**: なし（void）。検証に失敗した場合は、エラーをスローすることで呼び出し元に通知します。

前提条件と不変条件

  • 文字列オプションを定義する際、このコールバックを実装することが推奨されます。
  • 検証関数は、渡された文字列がそのオプションにとって許容可能な形式であるかをチェックする必要があります。

アルゴリズムまたは振る舞い

  • ユーザーが指定したオプション文字列が、特定のルール（形式や内容の制約）に従っているかを検証します。
  • 検証ロジックは、この型を実装する各関数に委ねられます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/reloptions.c 内のオプション解析処理から呼び出されます。
  • **呼び出し先**: 不明（具体的な検証ロジックは、この型を実装する個別の関数に依存します）。

関連データ構造

  • relopt_string: 文字列オプションを定義する構造体であり、このコールバックを保持するフィールドを含みます。

失敗モードと境界条件

  • 検証に失敗した場合、関数内でエラーを発生させる必要があります。
  • value が NULL の場合の挙動については、実装側の定義に依存します。

保守担当者向けメモ

  • 文字列オプションを新規追加する際は、必ずこのコールバックを実装し、適切な検証を行ってください。
  • src/backend/access/common/reloptions.c の各箇所でこの型が使用されているため、変更時は影響範囲を確認してください。
