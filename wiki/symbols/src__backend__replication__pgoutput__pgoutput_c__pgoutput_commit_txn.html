pgoutput_commit_txn

定義

  • 種別: function
  • 定義箇所: src/backend/replication/pgoutput/pgoutput.c:629

目的

論理レプリケーションにおけるトランザクションのコミット処理を実行し、必要に応じてサブスクライバーへコミットメッセージを送信する。

シグネチャまたは形状

static void pgoutput_commit_txn(LogicalDecodingContext *ctx, ReorderBufferTXN *txn, XLogRecPtr commit_lsn)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • txn: コミット対象のトランザクション情報（ReorderBufferTXN）。
  • commit_lsn: コミットのログシーケンス番号。
  • **出力**: なし（副作用としてストリームへの書き込みおよびメモリ解放を行う）。

前提条件と不変条件

  • txn->output_plugin_private に PGOutputTxnData が格納されている必要がある。
  • 処理終了後、txn->output_plugin_private は NULL に設定される。

アルゴリズムまたは振る舞い

1. txn->output_plugin_private から PGOutputTxnData を取得し、トランザクション内で変更が送信されたか（sent_begin_txn）を確認する。 2. OutputPluginUpdateProgress を呼び出し、進捗を更新する。 3. PGOutputTxnData をメモリから解放し、txn->output_plugin_private を NULL にクリアする。 4. sent_begin_txn が false の場合、空のトランザクションとして処理を終了する（デバッグログを出力）。 5. sent_begin_txn が true の場合、OutputPluginPrepareWrite を呼び出し、logicalrep_write_commit を使用してコミットメッセージを書き込み、OutputPluginWrite で送信を確定させる。

呼び出し元と呼び出し先

  • **呼び出し元**: _PG_output_plugin_init 内で commit_cb として登録されており、論理デコーディングのフレームワークから呼び出される。
  • **呼び出し先**:
  • OutputPluginUpdateProgress
  • pfree
  • elog
  • OutputPluginPrepareWrite
  • logicalrep_write_commit
  • OutputPluginWrite

関連データ構造

  • LogicalDecodingContext: 論理デコーディングのセッション状態。
  • ReorderBufferTXN: トランザクションのメタデータ。
  • PGOutputTxnData: プラグイン固有のトランザクション状態（sent_begin_txn フラグを保持）。

失敗モードと境界条件

  • txndata が NULL の場合、Assert により停止する。
  • トランザクション内に送信対象の変更が含まれていない場合、コミットメッセージは送信されず、空のトランザクションとしてスキップされる。

保守担当者向けメモ

  • 本関数はトランザクションの終了処理を担当するため、メモリリークを防ぐために PGOutputTxnData の解放が確実に行われる必要がある。
  • sent_begin_txn フラグは、サブスクライバーへ不要な空のトランザクション情報を送らないための重要な制御フラグである。
