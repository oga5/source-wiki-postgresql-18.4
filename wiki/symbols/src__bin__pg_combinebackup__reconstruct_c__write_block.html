write_block

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/reconstruct.c:752

目的

再構築中のデータファイルに対して、単一ブロック（BLCKSZ）の書き込みを行い、同時にチェックサム計算を更新する。

シグネチャまたは形状

static void write_block(int fd, char *output_filename, uint8 *buffer, pg_checksum_context *checksum_ctx)

入力と出力

  • fd: 書き込み対象のファイル記述子。
  • output_filename: エラーログ出力用のファイル名。
  • buffer: 書き込むデータを含むバッファ（BLCKSZ サイズ）。
  • checksum_ctx: チェックサム計算用のコンテキスト。

前提条件と不変条件

  • buffer は少なくとも BLCKSZ バイトの有効なデータを含んでいること。
  • fd は書き込み可能な状態で開かれていること。

アルゴリズムまたは振る舞い

1. write() システムコールを使用して、buffer から BLCKSZ バイトを fd に書き込む。 2. 書き込みバイト数が BLCKSZ と一致しない場合、エラー内容に応じて pg_fatal を呼び出しプロセスを終了する。 3. pg_checksum_update() を呼び出し、書き込んだブロックデータを用いて checksum_ctx を更新する。 4. チェックサム更新が失敗した場合は pg_fatal を呼び出しプロセスを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_combinebackup/reconstruct.c 内の write_reconstructed_file 関数から呼び出される。
  • **呼び出し先**: write()（システムコール）、pg_checksum_update()。

関連データ構造

  • BLCKSZ: storage/block.h で定義されるブロックサイズ。
  • pg_checksum_context: チェックサム計算の状態を保持する構造体。

失敗モードと境界条件

  • 書き込み失敗（wb < 0）: システムエラーメッセージと共に致命的エラーとして終了。
  • 部分書き込み（wb != BLCKSZ）: 書き込みバイト数を報告し、致命的エラーとして終了。
  • チェックサム更新失敗: 致命的エラーとして終了。

保守担当者向けメモ

  • 本関数は BLCKSZ 単位の書き込みに特化している。
  • pg_fatal を使用しているため、呼び出し側でエラーハンドリングを個別に実装する必要はない。
