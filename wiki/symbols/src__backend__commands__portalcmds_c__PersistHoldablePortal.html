PersistHoldablePortal

定義

  • 種別: function
  • 定義箇所: src/backend/commands/portalcmds.c:325

目的

トランザクション終了後も保持される「ホールド可能カーソル（Holdable Portal）」に対し、クエリの実行結果を Tuplestore へ退避させ、永続化を行う。

シグネチャまたは形状

void PersistHoldablePortal(Portal portal)

入力と出力

  • **入力**: Portal portal（永続化対象のポータル構造体）
  • **出力**: なし（副作用としてポータル内の holdStore に結果が格納され、queryDesc が解放される）

前提条件と不変条件

  • portal->createSubid が有効であること。
  • portal->holdContext および portal->holdStore が初期化済みであること。
  • portal->holdSnapshot が NULL であること。
  • 呼び出し時に、ポータルを作成したトランザクション内にあること。

アルゴリズムまたは振る舞い

1. **メタデータ保護**: tupDesc をポータルの長期メモリコンテキストへコピーする。 2. **実行環境の準備**: ActivePortal 等のグローバル変数を一時保存し、ポータル固有のコンテキストへ切り替える。 3. **データ転送**:

  • DestReceiver を DestTuplestore に設定し、クエリ実行結果を holdStore に書き込む。
  • スクロール可能なカーソルの場合は ExecutorRewind を実行する。

    • 4. **クリーンアップ**: ExecutorFinish および ExecutorEnd を呼び出し、クエリ実行エンジンを終了させる。 5. **位置調整**: atEnd フラグの状態に応じて、tuplestore の読み取り位置を再設定する。 6. **エラーハンドリング**: PG_TRY/PG_CATCH ブロックにより、実行中のエラー発生時に MarkPortalFailed を呼び出し、ポータルを無効化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mmgr/portalmem.c（トランザクション終了処理に関連する箇所）
  • **呼び出し先**: ExecutorRun, ExecutorFinish, ExecutorEnd, CreateTupleDescCopy, tuplestore_skiptuples, tuplestore_rescan 等。

関連データ構造

  • Portal: ポータルの状態を保持する主要構造体。
  • QueryDesc: クエリ実行プランと状態を保持。
  • Tuplestore: 結果セットを永続化するためのストレージ。

失敗モードと境界条件

  • **エラー発生時**: PG_CATCH により MarkPortalFailed が実行され、ポータルは破棄対象となる。
  • **スクロール不可カーソル**: CURSOR_OPT_SCROLL が指定されていない場合、tuplestore のリポジショニングは行われない。
  • **二重シャットダウン防止**: portal->queryDesc を NULL に設定することで、クリーンアップ時の二重解放を回避している。

保守担当者向けメモ

  • 本関数はトランザクション終了時の重要な処理であり、メモリ消費量やディスクI/Oに直接影響する。
  • tuplestore への書き込み時に detoast を行うため、スナップショットなしでもデータが安全に保持される設計となっている。
