PgFdwOption

定義

  • 種別: struct
  • 定義箇所: contrib/postgres_fdw/option.c:29

目的

postgres_fdw における外部サーバー、外部テーブル、ユーザーマッピング等のオプション定義を保持し、各オプションが libpq 接続パラメータとして使用可能か、またどのカタログコンテキストで有効かを管理する。

シグネチャまたは形状

typedef struct PgFdwOption
{
	const char *keyword;
	Oid			optcontext;		/* OID of catalog in which option may appear */
	bool		is_libpq_opt;	/* true if it's used in libpq */
} PgFdwOption;

入力と出力

本構造体自体はデータ保持用であり、関数ではない。初期化時には InitPgFdwOptions によって libpq のデフォルトオプションと non_libpq_options 配列が統合され、動的に割り当てられた postgres_fdw_options 配列の要素として格納される。

前提条件と不変条件

  • keyword はオプション名を指す文字列。
  • optcontext は、そのオプションが適用可能なシステムカタログの OID を示す。
  • is_libpq_opt が true の場合、そのオプションは libpq の接続パラメータとして直接渡される。

アルゴリズムまたは振る舞い

postgres_fdw のオプション検証および抽出プロセスにおいて、以下の役割を果たす： 1. **検証**: postgres_fdw_validator がユーザー指定のオプションを走査する際、定義済みの PgFdwOption リストと照合し、存在とコンテキストの妥当性を確認する。 2. **抽出**: ExtractConnectionOptions が接続確立時に、is_libpq_opt が true であるエントリのみをフィルタリングして libpq へ渡すパラメータリストを構築する。

呼び出し元と呼び出し先

  • **呼び出し元**: InitPgFdwOptions（構造体の初期化）、postgres_fdw_validator（検証）、ExtractConnectionOptions（抽出）などで参照される。
  • **呼び出し先**: 本構造体はデータ構造であり、関数を呼び出すことはない。

関連データ構造

  • non_libpq_options: postgres_fdw 固有のオプションを定義する静的配列。
  • postgres_fdw_options: libpq オプションと non_libpq_options を統合した実行時のオプションリスト。

失敗モードと境界条件

  • InitPgFdwOptions によるメモリ割り当て失敗時は、システム全体のエラーとして処理される。
  • 未定義のオプションや、不適切なコンテキスト（例：サーバー用オプションをテーブルに指定）で指定された場合は、postgres_fdw_validator が ereport(ERROR, ...) を発行する。

保守担当者向けメモ

  • 新しいオプションを追加する場合は、non_libpq_options 配列への追加が必要である。
  • is_libpq_opt フラグの誤設定は、接続パラメータの不整合や意図しないオプションの libpq への伝播を招くため注意すること。
