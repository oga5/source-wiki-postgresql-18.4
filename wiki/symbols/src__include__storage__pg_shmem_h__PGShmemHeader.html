PGShmemHeader

定義

  • 種別: struct
  • 定義箇所: src/include/storage/pg_shmem.h:29

目的

PostgreSQL の共有メモリセグメントの先頭に配置される標準ヘッダーであり、セグメントの識別、サイズ管理、および動的共有メモリ（DSM）やインデックス構造へのアクセスポイントを提供します。

シグネチャまたは形状

typedef struct PGShmemHeader {
    int32       magic;
    pid_t       creatorPID;
    Size        totalsize;
    Size        freeoffset;
    dsm_handle  dsm_control;
    void       *index;
#ifndef WIN32
    dev_t       device;
    ino_t       inode;
#endif
} PGShmemHeader;

入力と出力

本構造体は関数ではなくデータ構造であるため、入力と出力の概念は適用されません。メモリ上の特定のオフセットに配置されるメタデータとして機能します。

前提条件と不変条件

  • magic フィールドには定数 PGShmemMagic (679834894) が格納され、セグメントが PostgreSQL のものであることを識別するために使用されます。
  • WIN32 が定義されていない環境では、データディレクトリのデバイス ID (device) と inode 番号 (inode) を保持し、セグメントの整合性検証に使用されます。

アルゴリズムまたは振る舞い

  • サーバー起動時に共有メモリが確保された際、このヘッダーが領域の先頭に書き込まれます。
  • 子プロセスが共有メモリにアタッチする際、このヘッダーを読み取ることで、共有メモリ内のインデックスや DSM 制御ハンドルを特定します。
  • creatorPID は作成プロセスの PID を記録しますが、読み取りは行われません。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/port/sysv_shmem.c 内の共有メモリ確保およびアタッチ処理（shmat 等の戻り値をキャストして利用）から参照されています。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • dsm_handle: 動的共有メモリ制御セグメントの識別子。
  • ShmemIndex: 共有メモリ内のデータ構造を管理するためのハッシュテーブル（index ポインタが指す先）。

失敗モードと境界条件

  • size が sizeof(PGShmemHeader) を下回る場合、メモリ確保の整合性チェック（Assert）で失敗する可能性があります。
  • WIN32 環境下では device および inode フィールドが存在しないため、プラットフォーム間で構造体のバイナリレイアウトが異なります。

保守担当者向けメモ

  • 本構造体は共有メモリの物理的な基盤であるため、フィールドの追加や変更は、すべてのプラットフォーム（System V, mmap, Windows）および EXEC_BACKEND 環境での再アタッチ処理に影響を与えます。
  • creatorPID は現在使用されていないため、将来的なデバッグ目的以外での利用は推奨されません。
