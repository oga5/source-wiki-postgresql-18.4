XLogHaveInvalidPages

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogutils.c:223

目的

リカバリ処理中に、存在しないページへの参照が記録されたハッシュテーブル（invalid_page_tab）に未解決のエントリが存在するかどうかを判定します。

シグネチャまたは形状

bool XLogHaveInvalidPages(void)

入力と出力

  • **入力**: なし
  • **出力**:
  • true: invalid_page_tab が初期化されており、かつエントリが1つ以上存在する場合。
  • false: invalid_page_tab が NULL であるか、またはエントリが空の場合。

前提条件と不変条件

  • 本関数はリカバリプロセス中（InRecovery が真の状態）に呼び出されることを前提としています。
  • invalid_page_tab は、WALリプレイ中に存在しないページへのアクセスが発生した際に生成されるハッシュテーブルです。

アルゴリズムまたは振る舞い

1. invalid_page_tab が NULL でないかを確認します。 2. hash_get_num_entries を使用して、ハッシュテーブル内のエントリ数を取得します。 3. エントリ数が0より大きい場合は true を返し、それ以外の場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c:7599
  • **呼び出し先**:
  • hash_get_num_entries (ユーティリティ関数)

関連データ構造

  • invalid_page_tab: 無効なページ参照を追跡するために使用されるハッシュテーブル。

失敗モードと境界条件

  • invalid_page_tab が初期化される前（NULL の状態）であっても、安全に false を返すように設計されています。

保守担当者向けメモ

  • 本関数はリカバリ終了時の整合性チェック（XLogCheckInvalidPages）の一部として機能します。この関数が true を返す場合、リカバリプロセスはデータベースの整合性に問題があると判断し、PANICを引き起こす可能性があります。
