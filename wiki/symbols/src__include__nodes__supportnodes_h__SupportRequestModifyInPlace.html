SupportRequestModifyInPlace

定義

  • **種別**: struct
  • **定義箇所**: src/include/nodes/supportnodes.h:384

目的

関数呼び出しにおいて、入力値をコピーせずにメモリ上で直接更新（インプレース更新）が可能かどうかを判定するためのリクエスト構造体です。

シグネチャまたは形状

typedef struct SupportRequestModifyInPlace
{
    NodeTag     type;
    Oid         funcid;     /* 対象関数のPG_PROC OID */
    List       *args;       /* 関数への引数リスト */
    int         paramid;    /* 変数を表すParamのID */
} SupportRequestModifyInPlace;

入力と出力

  • **入力**: プランナーまたは実行エンジンが、最適化対象の関数情報（funcid、args）および変数識別子（paramid）を格納してサポート関数へ渡します。
  • **出力**: サポート関数は、インプレース更新が可能であれば情報を構造体に格納してポインタを返し、不可能であれば NULL を返します。

前提条件と不変条件

  • 本構造体は一時的な利用を目的としており、サポート関数内で永続的なメモリ確保は行われません。
  • サポート関数が処理できない場合は、エラーではなく NULL を返す必要があります。

アルゴリズムまたは振る舞い

1. プランナーまたは実行エンジンが、特定の関数に対してインプレース更新の可否を判断するために本構造体を生成します。 2. 対象関数に紐付けられたサポート関数が呼び出され、引数として本構造体が渡されます。 3. サポート関数は、渡された引数やパラメータIDに基づき、メモリ効率化のための最適化が可能か判定します。

呼び出し元と呼び出し先

  • **呼び出し元（確認された参照）**:
  • src/backend/utils/adt/array_userfuncs.c
  • src/backend/utils/adt/arraysubs.c
  • src/pl/plpgsql/src/pl_exec.c
  • **呼び出し先**:
  • 本構造体はデータ構造であり、具体的な呼び出し先（サポート関数）は pg_proc カタログの prosupport 属性に登録された関数に依存します。

関連データ構造

  • NodeTag: ノードの種類を識別するためのタグ。
  • List: 関数引数を保持するリスト。
  • FuncExpr / OpExpr: 本リクエストの対象となり得る式ノード（src/include/nodes/primnodes.h で定義）。

失敗モードと境界条件

  • サポート関数がインプレース更新をサポートしていない場合、NULL が返されます。
  • paramid が無効な場合や、引数リストが期待する形式と異なる場合の挙動は、各サポート関数の実装に依存します。

保守担当者向けメモ

  • 本構造体は「プランナーサポート関数」の一部ですが、実行時（Executor）にも利用されるため、プランナー専用のロジックと限定しないよう注意してください。
  • src/backend/utils/adt/ 配下のファイルで頻繁に参照されており、配列操作等の最適化に深く関与しています。
