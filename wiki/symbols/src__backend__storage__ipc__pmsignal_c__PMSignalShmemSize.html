PMSignalShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/pmsignal.c:129

目的

Postmasterとバックエンドプロセス間でシグナル状態を共有するために必要な共有メモリ領域のサイズを算出します。

シグネチャまたは形状

Size PMSignalShmemSize(void)

入力と出力

  • **入力**: なし
  • **出力**: Size 型のバイト数。PMSignalData 構造体の固定部分と、システムで許容される最大子プロセス数（MaxLivePostmasterChildren()）に応じた sig_atomic_t 配列領域の合計を返します。

前提条件と不変条件

  • MaxLivePostmasterChildren() が正しく設定されている必要があります。
  • 共有メモリの初期化（ShmemInitStruct）およびメモリクリア（MemSet）の前に呼び出される必要があります。

アルゴリズムまたは振る舞い

1. PMSignalData 構造体の PMChildFlags メンバまでのオフセットを取得します。 2. MaxLivePostmasterChildren() で取得した最大子プロセス数に sizeof(sig_atomic_t) を乗算します。 3. 上記の値を加算し、必要な合計サイズを算出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/ipci.c: 共有メモリ全体のサイズ計算時に呼び出し。
  • src/backend/storage/ipc/pmsignal.c: PMSignalState の初期化およびメモリクリア時に呼び出し。
  • **呼び出し先**:
  • MaxLivePostmasterChildren()
  • add_size() / mul_size() (PostgreSQLの安全な算術演算ユーティリティ)

関連データ構造

  • PMSignalData: 共有メモリ上に配置されるメイン構造体。
  • sig_atomic_t: プロセス間でのフラグ共有に使用される型。

失敗モードと境界条件

  • MaxLivePostmasterChildren() が 0 を返す場合、構造体の固定サイズのみが返されます。
  • add_size や mul_size 内部でオーバーフローが発生した場合、PostgreSQLの標準的なエラーハンドリングに従いプロセスが終了する可能性があります。

保守担当者向けメモ

  • PMSignalData 構造体のレイアウトが変更された場合、offsetof の対象が正しく維持されているか確認してください。
  • 本関数は共有メモリの確保サイズを決定するため、PMSignalShmemInit での確保サイズと常に一致させる必要があります。
