delete_function

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/funccache.c:432

目的

関数キャッシュエントリをハッシュテーブルから削除し、安全な場合に限り関連するメモリリソースを解放します。

シグネチャまたは形状

static void delete_function(CachedFunction *func)

入力と出力

  • **入力**: CachedFunction *func（削除対象のキャッシュエントリへのポインタ）
  • **出力**: なし（void）

前提条件と不変条件

  • func は有効な CachedFunction 構造体を指している必要があります。
  • cfunc_hashtable_delete は、既に削除済みであっても安全に呼び出せる必要があります。

アルゴリズムまたは振る舞い

1. cfunc_hashtable_delete(func) を呼び出し、ハッシュテーブルから該当エントリを削除します。 2. func->use_count が 0 かつ func->dcallback が NULL でない場合、dcallback を実行して関数固有のストレージを解放します。 3. 解放後、func->dcallback を NULL に設定し、二重解放を防止します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/funccache.c:419 (コメントによる言及)
  • src/backend/utils/cache/funccache.c:533
  • **呼び出し先**:
  • cfunc_hashtable_delete
  • func->dcallback (関数ポインタ経由のコールバック実行)

関連データ構造

  • CachedFunction: キャッシュされた関数の基底構造体。use_count および dcallback を保持します。

失敗モードと境界条件

  • use_count > 0 の場合、キャッシュエントリはハッシュテーブルから削除されますが、dcallback によるストレージ解放はスキップされます（実行中のクエリが存在する可能性があるため）。
  • dcallback が既に NULL の場合、ストレージ解放処理はスキップされます。

保守担当者向けメモ

  • 本関数は、キャッシュの無効化や破棄の最終段階で呼び出されます。
  • dcallback の実行は、メモリリークを防ぐために use_count が 0 であることを厳密に確認してから行う必要があります。
