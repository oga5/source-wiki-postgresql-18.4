stemmer_module

定義

  • 種別: struct
  • 定義箇所: src/backend/snowball/dict_snowball.c:90

目的

PostgreSQLの全文検索機能において、外部の libstemmer ライブラリが提供する各言語のステミング（語幹抽出）アルゴリズムを統合・管理するためのインターフェースを定義します。

シグネチャまたは形状

typedef struct stemmer_module
{
    const char *name;
    pg_enc      enc;
    struct SN_env *(*create) (void);
    void        (*close) (struct SN_env *);
    int         (*stem) (struct SN_env *);
} stemmer_module;

入力と出力

  • **入力**: 構造体の各フィールドは、特定の言語およびエンコーディングに対応するステマーのメタデータと関数ポインタを保持します。
  • **出力**: create 関数は SN_env 構造体を生成し、stem 関数はステミング処理の結果（成功時0以外、失敗時0など）を返します。

前提条件と不変条件

  • libstemmer の各ヘッダーファイル（stem_*.h）で定義された関数と互換性があること。
  • name は言語名を識別する文字列であること。

アルゴリズムまたは振る舞い

  • 各言語固有のステミングアルゴリズムを関数ポインタとして保持し、実行時に動的に呼び出すための抽象化レイヤーとして機能します。
  • stemmer_modules[] 配列内で、サポートされる言語とエンコーディングの組み合わせを静的に定義します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/dict_snowball.c 内の stemmer_modules[] 配列定義および、ステマーを選択するロジック（locate_stem_module 等）から参照されます。
  • **呼び出し先**: libstemmer ライブラリ内の各言語固有の create、close、stem 関数を指します。

関連データ構造

  • DictSnowball: このモジュールを使用して初期化されたステマー環境を保持する辞書構造体。
  • SN_env: libstemmer ライブラリが使用するステミング環境の内部構造体。

失敗モードと境界条件

  • 構造体内の関数ポインタが適切に初期化されていない場合、実行時にセグメンテーション違反が発生する可能性があります。
  • サポートされていないエンコーディングや言語が指定された場合、検索処理が失敗します。

保守担当者向けメモ

  • 新しい言語をサポートする場合は、libstemmer のヘッダーをインクルードし、stemmer_modules 配列に新しいエントリを追加してください。
  • pg_enc 型の整合性に注意し、サーバーエンコーディングとの変換処理が正しく行われるか確認してください。
