ValidateOperatorReference

定義

  • 種別: function
  • 定義箇所: src/backend/commands/operatorcmds.c:404

目的

指定された名前と引数型を持つ演算子の存在を確認し、その妥当性を検証した上で、対応する OID を返します。主に CREATE OPERATOR における COMMUTATOR や NEGATOR の参照解決に使用されます。

シグネチャまたは形状

static Oid ValidateOperatorReference(List *name, Oid leftTypeId, Oid rightTypeId)

入力と出力

  • **入力**:
  • name: 演算子名（List 形式）。
  • leftTypeId: 左辺の型 OID。
  • rightTypeId: 右辺の型 OID。
  • **出力**:
  • 成功時: 該当する演算子の Oid。
  • 失敗時: エラーをスローし、処理を中断します。

前提条件と不変条件

  • 演算子はシステムカタログ pg_operator に存在している必要があります。
  • 演算子は「シェル（未定義の演算子）」であってはなりません。
  • 呼び出し元ユーザーは、対象の演算子に対する所有権（Owner）を持っている必要があります。

アルゴリズムまたは振る舞い

1. OperatorLookup を呼び出し、名前と型から演算子の OID と定義済みフラグ（defined）を取得します。 2. 演算子が見つからない場合、ERRCODE_UNDEFINED_FUNCTION エラーを発生させます。 3. 演算子がシェル状態（defined が false）である場合、ERRCODE_UNDEFINED_FUNCTION エラーを発生させます。 4. object_ownercheck を使用し、現在のユーザーが演算子の所有者であることを確認します。所有者でない場合は aclcheck_error を呼び出します。 5. 検証を通過した場合、演算子の OID を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/operatorcmds.c 内の DefineOperator 関数から、commutator や negator の検証のために呼び出されます。
  • **呼び出し先**: OperatorLookup（演算子検索）、op_signature_string（エラーメッセージ生成）、object_ownercheck（権限確認）、aclcheck_error（権限エラー処理）。

関連データ構造

  • List: 演算子名を保持するリスト構造。
  • Oid: PostgreSQL のオブジェクト識別子。
  • pg_operator: 演算子情報を格納するシステムカタログ。

失敗モードと境界条件

  • **演算子未定義**: 存在しない演算子を指定するとエラー。
  • **シェル演算子**: プレースホルダーとして作成された未定義の演算子を指定するとエラー。
  • **権限不足**: 演算子の所有者でないユーザーが参照しようとするとエラー。

保守担当者向けメモ

  • 本関数は parse_oper.c のエラーメッセージ形式と整合性を保つように設計されています。修正時はメッセージの一貫性に注意してください。
  • 演算子作成時のシェル演算子拒否は、誤った設定を早期に検知するためのポリシーです。
