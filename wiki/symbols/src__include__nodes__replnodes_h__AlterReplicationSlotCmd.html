AlterReplicationSlotCmd

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/replnodes.h:79

目的

レプリケーションスロットの設定変更（ALTER REPLICATION SLOT）コマンドのパラメータを保持するためのパースノード構造体です。

シグネチャまたは形状

typedef struct AlterReplicationSlotCmd
{
    NodeTag     type;
    char       *slotname;
    List       *options;
} AlterReplicationSlotCmd;

入力と出力

  • **入力**: 構文解析器によって、フロントエンドから送信されたレプリケーションコマンドの引数（スロット名およびオプションリスト）が本構造体に格納されます。
  • **出力**: 実行エンジンが本構造体を参照し、対応するレプリケーション処理を実行します。

前提条件と不変条件

  • NodeTag 型の type フィールドを先頭に持ち、PostgreSQLのノードシステムの一部として扱われます。
  • options フィールドは List 構造体であり、コマンド固有の柔軟なオプションを保持します。

アルゴリズムまたは振る舞い

本構造体はデータ保持専用のノードであり、パース時にインスタンス化され、実行時にバックエンドのレプリケーション関数へ渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walsender.c 内の AlterReplicationSlot 関数が、本構造体を引数として受け取り処理を行います。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接関数を呼び出すことはありません。

関連データ構造

  • NodeTag: ノードの種類を識別するためのタグ。
  • List: src/include/nodes/pg_list.h で定義される、オプションを格納するためのリスト構造。

失敗モードと境界条件

  • 構造体内の各ポインタ（slotname, options）が NULL である場合の挙動は、本構造体ではなく、これを利用する実行側の関数（AlterReplicationSlot 等）のバリデーションロジックに依存します。

保守担当者向けメモ

  • 新しいレプリケーションスロット変更オプションを追加する場合は、本構造体の options リストを介して拡張可能です。
  • ノードのコピーやデバッグ出力が必要な場合は、src/backend/nodes/copyfuncs.c および outfuncs.c への追記を検討してください。
