EvaluateParams

定義

  • 種別: function
  • 定義箇所: src/backend/commands/prepare.c:280

目的

プリペアドステートメントの実行時に提供されたパラメータ式を評価し、計画が期待する型へ変換した上で ParamListInfo 構造体を構築する。

シグネチャまたは形状

static ParamListInfo EvaluateParams(ParseState *pstate, PreparedStatement *pstmt, List *params, EState *estate)

入力と出力

  • **入力**:
  • pstate: 解析状態。
  • pstmt: 実行対象のプリペアドステートメント。
  • params: 実行時に渡されたパラメータ式のリスト。
  • estate: 実行状態。
  • **出力**:
  • 評価済みパラメータを格納した ParamListInfo 構造体。パラメータがない場合は NULL を返す。

前提条件と不変条件

  • pstmt->plansource->num_params と params のリスト長が一致している必要がある。不一致の場合はエラーとなる。
  • パラメータの型変換（Coercion）が成功する必要がある。失敗した場合は ereport(ERROR) により処理が中断される。

アルゴリズムまたは振る舞い

1. パラメータ数の整合性を確認する。0個の場合は直ちに NULL を返す。 2. 入力されたパラメータリストを copyObject で複製する。 3. 各パラメータ式に対して transformExpr を実行し、coerce_to_target_type を用いてプリペアドステートメントが期待する型へ変換する。 4. assign_expr_collations で照合順序（Collation）を割り当てる。 5. ExecPrepareExprList を使用して、式を評価可能な状態（ExprState）に変換する。 6. makeParamList で ParamListInfo を確保し、ExecEvalExprSwitchContext を呼び出して各パラメータの値を評価・格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/prepare.c 内の ExecuteQuery（複数箇所）。
  • **呼び出し先**: copyObject, transformExpr, coerce_to_target_type, assign_expr_collations, ExecPrepareExprList, makeParamList, ExecEvalExprSwitchContext, GetPerTupleExprContext。

関連データ構造

  • PreparedStatement: プリペアドステートメントのメタデータ。
  • ParamListInfo: 実行時に渡されるパラメータの値を保持する構造体。
  • ExprState: 評価準備が完了した式。

失敗モードと境界条件

  • **パラメータ数不一致**: nparams != num_params の場合、ERRCODE_SYNTAX_ERROR でエラー終了。
  • **型変換失敗**: 期待する型へ変換できない場合、ERRCODE_DATATYPE_MISMATCH でエラー終了。

保守担当者向けメモ

  • 本関数はセッションローカルなプリペアドステートメントの実行に特化している。
  • パラメータの評価は estate のコンテキスト内で行われるため、メモリ管理は estate のライフサイクルに依存する。
