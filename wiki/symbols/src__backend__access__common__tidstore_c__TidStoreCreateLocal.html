TidStoreCreateLocal

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tidstore.c:161

目的

ローカルメモリ上で動作する TidStore（TID集合管理用データ構造）を初期化し、メモリコンテキストを割り当てる。

シグネチャまたは形状

TidStore *TidStoreCreateLocal(size_t max_bytes, bool insert_only)

入力と出力

  • **入力**:
  • max_bytes: ストレージの最大サイズ（メモリブロックサイズの計算に使用）。
  • insert_only: true の場合、挿入専用の BumpContext を使用し、false の場合は汎用的な AllocSetContext を使用する。
  • **出力**:
  • 初期化された TidStore 構造体へのポインタ。

前提条件と不変条件

  • max_bytes に基づき、内部のメモリブロックサイズが max_bytes の 1/16 を超えないよう調整される。
  • 最小ブロックサイズは ALLOCSET_DEFAULT_INITSIZE 以上であることが保証される。

アルゴリズムまたは振る舞い

1. TidStore 構造体をゼロ初期化して確保する。 2. max_bytes を基に、メモリコンテキストの maxBlockSize を計算する。 3. insert_only フラグに応じて、BumpContextCreate または AllocSetContextCreate を呼び出し、ts->rt_context を作成する。 4. local_ts_create を呼び出し、作成したコンテキストを用いてラディックツリーを初期化し、ts->tree.local に格納する。 5. 初期化済みの TidStore ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/vacuumlazy.c (VACUUM処理でのデッドアイテム管理)
  • src/test/modules/test_tidstore/test_tidstore.c (テストモジュール)
  • **呼び出し先**:
  • palloc0
  • BumpContextCreate / AllocSetContextCreate
  • local_ts_create

関連データ構造

  • TidStore: 本関数が作成するメイン構造体。
  • MemoryContext: 割り当てられたメモリ領域を管理するコンテキスト。

失敗モードと境界条件

  • メモリ不足時には palloc0 やコンテキスト作成関数がエラーを発生させる。
  • max_bytes が極端に小さい場合、ブロックサイズは ALLOCSET_DEFAULT_INITSIZE に固定される。

保守担当者向けメモ

  • 本関数はローカルメモリ専用のストレージを作成する。共有メモリを使用する場合は TidStoreCreateShared を参照すること。
  • insert_only を true に設定すると、メモリ効率の良い BumpContext が使用されるため、削除操作が不要な場合に推奨される。
