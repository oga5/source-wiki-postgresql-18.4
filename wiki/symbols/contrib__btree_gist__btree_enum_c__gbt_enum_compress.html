gbt_enum_compress

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_enum.c:106

目的

ENUM 型の値を GiST インデックスに挿入する際、インデックスキーとして適切に圧縮（表現）するために使用されます。

シグネチャまたは形状

Datum gbt_enum_compress(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体へのポインタを受け取ります。
  • **出力**: Datum 型として、圧縮された GISTENTRY を返します。

前提条件と不変条件

  • ENUM 型は内部的に OID として扱われるため、本関数は OID ベースの範囲検索ロジックに依存しています。
  • tinfo（gbtree_ninfo 構造体）が ENUM 型の比較関数やサイズ情報を保持している必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から GISTENTRY を取得します。 2. 取得した entry と、モジュール内で定義された tinfo を引数として、btree_utils_num.c で定義されている汎用関数 gbt_num_compress を呼び出します。 3. gbt_num_compress の戻り値を PG_RETURN_POINTER を介して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス挿入処理から呼び出されます（直接的な呼び出し元関数の詳細はソースコード内に明示されていません）。
  • **呼び出し先**: gbt_num_compress（contrib/btree_gist/btree_utils_num.c 内）。

関連データ構造

  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。
  • tinfo: gbtree_ninfo 構造体。ENUM 型の比較関数やメタ情報を保持する設定テーブル。

失敗モードと境界条件

  • 供給されたソースコードには、特定の失敗モードや境界条件に関する記述はありません。

保守担当者向けメモ

  • 本関数はアダプター層として機能しており、実際の圧縮ロジックは gbt_num_compress に委譲されています。
  • ENUM 型の順序付けは OID に基づくため、OID 型のインデックス実装（btree_oid.c）と類似の挙動をとります。
