ProcArrayRemove

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/procarray.c:564

目的

共有メモリ上のプロセス配列から指定された PGPROC を削除し、関連するトランザクション状態をクリーンアップする。

シグネチャまたは形状

void ProcArrayRemove(PGPROC *proc, TransactionId latestXid)

入力と出力

  • **入力**:
  • proc: 削除対象の PGPROC 構造体へのポインタ。
  • latestXid: 完了した最新のトランザクションID（無効な場合は InvalidTransactionId）。
  • **出力**: なし（副作用として共有メモリ上の状態を更新）。

前提条件と不変条件

  • ProcArrayLock および XidGenLock が排他モードで保持されていること。
  • proc->pgxactoff が有効な範囲内（0 以上 arrayP->numProcs 未満）であること。
  • 削除対象の PGPROC がプロセス配列内の正しい位置にあること。

アルゴリズムまたは振る舞い

1. ProcArrayLock と XidGenLock を排他モードで取得する。 2. latestXid が有効な場合、MaintainLatestCompletedXid を呼び出してグローバルな完了トランザクションIDを更新し、xactCompletionCount をインクリメントする。 3. 削除対象プロセスのトランザクションIDおよびサブトランザクション状態をリセットする。 4. memmove を使用して、プロセス配列（pgprocnos）、トランザクションID配列、サブトランザクション状態、ステータスフラグを前方へ詰め、配列をソート状態に保つ。 5. arrayP->numProcs をデクリメントする。 6. 削除位置より後ろにあるすべてのプロセスの pgxactoff を更新し、配列のインデックスと整合させる。 7. XidGenLock、ProcArrayLock の順でロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/twophase.c:1576
  • src/backend/storage/lmgr/proc.c:911
  • **呼び出し先**:
  • MaintainLatestCompletedXid
  • LWLockAcquire / LWLockRelease
  • memmove（標準ライブラリ）

関連データ構造

  • PGPROC: バックエンドプロセスの状態を保持する構造体。
  • ProcArrayStruct: 共有メモリ上のプロセス配列を管理するメイン構造体。
  • ProcGlobal: プロセス配列に関連するグローバルなトランザクション状態（xids, subxidStates, statusFlags など）。

失敗モードと境界条件

  • latestXid が無効な場合、実行中のトランザクションを削除しようとしていないことを Assert で検証する。
  • 配列のインデックス操作において、numProcs が正しく管理されていない場合、Assert により不整合を検知する。

保守担当者向けメモ

  • ProcArrayLock と XidGenLock の両方を保持する理由は、ProcGlobal のコメントを参照すること。
  • 配列の詰め合わせ（memmove）は、プロセス配列の順序を維持するために不可欠である。
  • 削除後の pgxactoff 更新ループは、配列の整合性を保つために必須である。
