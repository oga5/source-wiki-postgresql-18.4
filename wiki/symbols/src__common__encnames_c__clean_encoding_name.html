clean_encoding_name

定義

  • 種別: function
  • 定義箇所: src/common/encnames.c:526

目的

エンコーディング名を表す文字列を正規化し、検索キーとして利用可能な形式に変換します。

シグネチャまたは形状

static char *clean_encoding_name(const char *key, char *newkey);

入力と出力

  • **入力**:
  • key: 正規化対象のエンコーディング名文字列（null終端）。
  • newkey: 正規化後の文字列を格納するためのバッファ（呼び出し元で確保済みであること）。
  • **出力**:
  • newkey へのポインタを返します。

前提条件と不変条件

  • newkey は、入力 key の長さを十分に保持できる領域である必要があります。
  • 入力 key は \0 で終了している必要があります。

アルゴリズムまたは振る舞い

1. key の各文字を順次走査します。 2. isalnum() を使用し、英数字のみを抽出します。 3. 英字（A-Z）が含まれる場合は小文字（a-z）に変換します。 4. 抽出した文字を newkey に順次格納し、最後に \0 を付与して文字列を完成させます。 5. 記号や空白文字は無視されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/encnames.c 内の pg_char_to_encoding 関数から呼び出されます。
  • **呼び出し先**: 標準ライブラリの isalnum() を使用します。

関連データ構造

  • pg_encname_tbl: 本関数で正規化されたキーを用いて二分探索を行うためのテーブル。

失敗モードと境界条件

  • key が空文字列の場合、newkey には \0 のみが書き込まれます。
  • key に英数字が含まれない場合、結果は空文字列となります。
  • バッファオーバーフローの保護は呼び出し元に依存します。

保守担当者向けメモ

  • 本関数は pg_char_to_encoding における検索キーの正規化に特化しています。
  • isalnum を使用しているため、ロケール設定に依存する可能性があります。PostgreSQL の標準的な動作として、エンコーディング名が ASCII の範囲内であることを前提としています。
