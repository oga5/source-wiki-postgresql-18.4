定義

目的

PostgreSQL サーバーが接続を受け入れ可能な状態にあるかを判定し、その結果を終了ステータスとして返却するコマンドラインユーティリティのエントリポイントです。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**: コマンドライン引数（ホスト、ポート、ユーザー、データベース名、タイムアウト値）。
  • **出力**:
  • 標準出力: 接続確認結果（--quiet 指定時は抑制）。
  • 終了ステータス: libpq の PGPing 列挙型に基づいた値（サーバーの状態を示す）。

前提条件と不変条件

  • 実行環境において libpq が利用可能であること。
  • 引数解析において、不正な引数や過剰な引数が指定された場合は PQPING_NO_ATTEMPT で終了する。

アルゴリズムまたは振る舞い

1. **初期化**: ログ機能の初期化、ロケール設定、ヘルプ/バージョンオプションの処理を行う。 2. **引数解析**: getopt_long を使用してコマンドライン引数を解析し、接続パラメータを保持する。 3. **パラメータ構築**: 接続情報を keywords および values 配列に格納する。 4. **接続情報の解決**: 接続文字列が指定された場合は PQconninfoParse でパースし、デフォルト値と組み合わせてホスト・ポートを特定する。 5. **疎通確認**: PQpingParams を呼び出し、サーバーへの ping を実行する。 6. **結果出力**: --quiet が指定されていない場合、接続先情報と ping 結果を表示する。

呼び出し元と呼び出し先

  • **呼び出し元**: OS（プログラムのエントリポイント）。
  • **呼び出し先**: pg_logging_init, getopt_long, PQconninfoParse, PQconndefaults, PQpingParams 等。
  • **注意**: ツールによる解析結果において、contrib/amcheck 等のファイルで main という単語が言及されていますが、これらは本関数の呼び出し元ではありません。

関連データ構造

  • PGPing: 接続状態を示す列挙型。
  • PQconninfoOption: 接続オプションを保持する構造体。
  • keywords/values: PQpingParams に渡す接続パラメータの配列。

失敗モードと境界条件

  • 引数解析エラーや接続情報のパース失敗時は、PQPING_NO_ATTEMPT を返して終了する。
  • サーバーが応答しない場合や接続拒否された場合も、PQpingParams の結果に基づいた終了ステータスを返す。

保守担当者向けメモ

  • 本関数はデータベースへの接続を確立するのではなく、サーバーの稼働確認のみを行う点に留意してください。
  • 終了ステータスはシェルスクリプト等で直接利用されるため、変更時には互換性に注意してください。
