PQsetTraceFlags

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-trace.c:63

目的

PGconn オブジェクトに関連付けられたトレース設定（フラグ）を更新し、デバッグ出力の挙動を制御します。

シグネチャまたは形状

void PQsetTraceFlags(PGconn *conn, int flags)

入力と出力

  • conn: トレース設定を適用する対象の PGconn 構造体へのポインタ。
  • flags: 設定するトレースフラグのビットマスク。
  • 戻り値: なし。

前提条件と不変条件

  • conn が NULL の場合、関数は何もせず即座に復帰します。
  • conn->Pfdebug が NULL の場合（PQtrace が呼び出されていない、または失敗している状態）、フラグの更新は行われません。

アルゴリズムまたは振る舞い

1. 引数 conn が NULL かどうかを確認し、該当する場合は終了します。 2. conn->Pfdebug が NULL かどうかを確認し、トレース出力先が設定されていない場合は終了します。 3. 上記の条件を満たす場合、conn->traceFlags に引数 flags を代入します。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/libpq_pipeline/libpq_pipeline.c:2353 での呼び出しが確認されています。
  • 呼び出し先: 本関数は内部で他の関数を呼び出しません。

関連データ構造

  • PGconn: libpq の接続状態を保持する構造体。traceFlags フィールドおよび Pfdebug（出力先ストリーム）を保持します。

失敗モードと境界条件

  • conn が無効なポインタである場合、セグメンテーション違反が発生する可能性があります。
  • PQtrace を介してトレースが有効化されていない接続に対して本関数を呼び出しても、内部のガード節により設定は反映されません。

保守担当者向けメモ

  • 本関数は libpq-fe.h で公開されている API です。
  • フラグの具体的なビット定義については libpq-int.h および fe-trace.c 内のトレース出力ロジックを参照してください。
