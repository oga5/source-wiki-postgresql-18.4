exec_prog

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/exec.c:84

目的

外部コマンドを安全に実行し、その標準出力および標準エラー出力を指定されたログファイルへリダイレクトして記録する。

シグネチャまたは形状

bool exec_prog(const char *log_filename, const char *opt_log_file,
               bool report_error, bool exit_on_error, const char *fmt, ...);

入力と出力

  • **入力**:
  • log_filename: 出力先ログファイル名。
  • opt_log_file: 失敗時に併せて参照を促す補助的なログファイル名（任意）。
  • report_error: 実行失敗時にエラーメッセージを表示するかどうか。
  • exit_on_error: 実行失敗時に pg_fatal を呼び出しプロセスを終了させるかどうか。
  • fmt, ...: 実行するコマンド文字列のフォーマットと引数。
  • **出力**:
  • bool: コマンドが正常終了（終了コード 0）した場合は true、それ以外は false を返す。

前提条件と不変条件

  • コマンド文字列の長さは MAXCMDLEN (2 * MAXPGPATH) を超えてはならない。超えた場合は pg_fatal により終了する。
  • ログディレクトリ (log_opts.logdir) が適切に設定されている必要がある。

アルゴリズムまたは振る舞い

1. 可変長引数からコマンド文字列を構築し、標準出力と標準エラーをログファイルへリダイレクトするよう末尾に >> "log_file" 2>&1 を付与する。 2. Windows 環境では、スレッドごとのログ競合を避けるため、メインスレッド以外からの呼び出しでは system() を先行実行する。 3. ログファイルを開き、実行するコマンドを記録する。Windows 環境では、ファイルロックを避けるため、ログファイルを開く際にリトライ処理を行う。 4. system() を呼び出しコマンドを実行する。 5. 実行結果が非ゼロかつ report_error が真の場合、エラーログを出力する。exit_on_error が真であれば pg_fatal を呼び出す。 6. Windows 以外では、実行後にログファイルへ改行を追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/dump.c, src/bin/pg_upgrade/parallel.c, src/bin/pg_upgrade/pg_upgrade.c 等から呼び出される。
  • **呼び出し先**: system(), vsnprintf(), snprintf(), fopen(), fprintf(), fclose(), pg_log(), pg_fatal(), report_status(), pg_usleep() (Windowsのみ)。

関連データ構造

  • log_opts: ログ設定を保持するグローバル構造体（pg_upgrade.h 関連）。

失敗モードと境界条件

  • コマンド長が MAXCMDLEN を超過した場合、pg_fatal で異常終了する。
  • ログファイルが開けない場合、pg_fatal で異常終了する。
  • Windows 環境において、ログファイルが他プロセスによってロックされている場合、最大 4 秒間のリトライを行う。

保守担当者向けメモ

  • Windows 特有の #ifdef WIN32 ブロックは、ログファイルの共有違反やスレッド間の競合を回避するための複雑なロジックを含んでいる。修正時はログの書き込みタイミングと system() の実行順序に十分注意すること。
  • pg_ctl start のような長時間実行されるコマンドのログ出力については、Windows とそれ以外のプラットフォームで挙動が異なるため、プラットフォーム固有の制約を考慮すること。
