uuid_gt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/uuid.c:245

目的

2つの uuid 型の値が「より大きい」かどうかを比較し、SQL の > 演算子をサポートする。

シグネチャまたは形状

Datum uuid_gt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される2つの pg_uuid_t 型のポインタ。
  • **出力**: Datum 型として、第1引数が第2引数より大きい場合は true、それ以外は false を返す。

前提条件と不変条件

  • 入力される pg_uuid_t ポインタは有効なメモリ領域を指している必要がある。
  • 比較は uuid_internal_cmp を介して行われる。

アルゴリズムまたは振る舞い

1. PG_GETARG_UUID_P を使用して、2つの UUID 引数を取得する。 2. uuid_internal_cmp を呼び出し、2つの UUID のバイナリ値を比較する。 3. 比較結果が 0 より大きい（第1引数が大きい）場合に true を返し、そうでない場合に false を返す。

呼び出し元と呼び出し先

  • **呼び出し先**: uuid_internal_cmp
  • **呼び出し元**: 提供されたソース情報には、本関数を直接呼び出している特定の SQL 文や関数は記載されていない。

関連データ構造

  • pg_uuid_t: UUID を保持する 16 バイトの構造体。

失敗モードと境界条件

  • 引数が NULL の場合、PG_GETARG_UUID_P の挙動に依存する（通常はエラーまたは NULL 処理）。
  • UUID のバイナリ比較は memcmp に依存するため、エンディアンの影響を受けないバイト単位の比較が行われる。

保守担当者向けメモ

  • 本関数は uuid_internal_cmp の結果を評価するラッパーである。比較ロジックに変更を加える場合は uuid_internal_cmp を参照すること。
