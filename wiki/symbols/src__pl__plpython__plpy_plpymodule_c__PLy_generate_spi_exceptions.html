PLy_generate_spi_exceptions

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_plpymodule.c:236

目的

PostgreSQL の SQLSTATE 定義（exception_map）に基づき、対応する Python 例外クラスを動的に生成し、plpy.spiexceptions モジュールおよび内部ハッシュテーブル PLy_spi_exceptions に登録する。

シグネチャまたは形状

static void PLy_generate_spi_exceptions(PyObject *mod, PyObject *base)
  • mod: 例外クラスを登録する対象の Python モジュールオブジェクト。
  • base: 生成する例外クラスの基底クラス（通常は PLy_exc_spi_error）。

入力と出力

  • 入力: exception_map（静的定義）、mod（Python モジュール）、base（基底例外クラス）。
  • 出力: なし（副作用として Python モジュールへのクラス追加と PLy_spi_exceptions ハッシュテーブルの更新を行う）。

前提条件と不変条件

  • exception_map が有効なエントリで構成されていること。
  • PLy_spi_exceptions ハッシュテーブルが初期化済みであること。
  • Assert(!found) により、同一の SQLSTATE が二重に登録されないことが保証されている。

アルゴリズムまたは振る舞い

1. exception_map を反復処理する。 2. 各エントリに対し、SQLSTATE 文字列を保持する Python 辞書を作成する。 3. PLy_create_exception を呼び出し、Python の例外クラスを生成する。 4. hash_search を使用して PLy_spi_exceptions ハッシュテーブルにエントリを挿入し、生成した例外オブジェクトを格納する。 5. 辞書や文字列オブジェクトの参照カウントを適切に管理する。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpython/plpy_plpymodule.c 内の初期化処理（PLy_add_exceptions 等から間接的に呼び出される）。
  • 呼び出し先: PLy_create_exception（例外クラス生成）、hash_search（ハッシュテーブル登録）、PLy_elog（エラー報告）、PyDict_New 等の Python C API。

関連データ構造

  • exception_map: SQLSTATE と例外名、クラス名を保持する静的配列。
  • PLy_spi_exceptions: SQLSTATE をキーとして例外オブジェクトを管理するハッシュテーブル。
  • PLyExceptionEntry: ハッシュテーブルに格納されるデータ構造。

失敗モードと境界条件

  • PyDict_New が失敗した場合、PLy_elog(ERROR, ...) を呼び出し、PostgreSQL のエラーとして処理を中断する。
  • PLyUnicode_FromString が失敗した場合も同様にエラー報告を行う。
  • 既に登録済みの SQLSTATE が存在する場合、Assert によりデバッグビルド時に停止する。

保守担当者向けメモ

  • 本関数はビルド時に生成される spiexceptions.h に依存しているため、例外の追加・変更はソースコードではなく生成スクリプト側で行う必要がある。
  • Python オブジェクトの参照カウント（Py_DECREF）管理が重要であり、修正時はリークに注意すること。
