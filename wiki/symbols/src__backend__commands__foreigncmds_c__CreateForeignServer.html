CreateForeignServer

定義

  • 種別: function
  • 定義箇所: src/backend/commands/foreigncmds.c:853

目的

SQLの CREATE SERVER コマンドを処理し、外部サーバー定義をシステムカタログ pg_foreign_server に登録する。

シグネチャまたは形状

ObjectAddress CreateForeignServer(CreateForeignServerStmt *stmt)

入力と出力

  • **入力**: CreateForeignServerStmt *stmt（パーサーが生成したサーバー定義情報）
  • **出力**: ObjectAddress（作成されたサーバーオブジェクトの識別子）

前提条件と不変条件

  • pg_foreign_server システムカタログに対する RowExclusiveLock が保持される。
  • 指定された外部データラッパー（FDW）が存在し、実行ユーザーがそのFDWに対する USAGE 権限を有している必要がある。
  • 同名のサーバーが既に存在する場合、IF NOT EXISTS が指定されていない限りエラーとなる。

アルゴリズムまたは振る舞い

1. pg_foreign_server をロックし、同名サーバーの存在を確認する。 2. GetForeignDataWrapperByName を呼び出し、FDWの存在と権限を検証する。 3. 新規サーバーのOIDを生成し、pg_foreign_server 用のタプルを構築する。 4. transformGenericOptions を呼び出し、FDWのバリデーター関数を用いてオプションを検証・変換する。 5. CatalogTupleInsert でカタログに挿入する。 6. pg_depend にFDWへの依存関係、所有者情報、および現在の拡張機能（存在する場合）への依存関係を記録する。 7. InvokeObjectPostCreateHook を呼び出し、作成後のフック処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（SQLコマンド実行フロー）
  • **呼び出し先**:
  • table_open / table_close
  • get_foreign_server_oid
  • GetForeignDataWrapperByName
  • object_aclcheck
  • transformGenericOptions
  • heap_form_tuple / CatalogTupleInsert
  • recordDependencyOn / recordDependencyOnOwner / recordDependencyOnCurrentExtension
  • InvokeObjectPostCreateHook

関連データ構造

  • CreateForeignServerStmt: パーサーが生成するステートメント構造体。
  • pg_foreign_server: サーバー定義を保持するシステムカタログ。
  • ObjectAddress: 作成されたオブジェクトを特定するための構造体。

失敗モードと境界条件

  • 同名のサーバーが既に存在する場合、IF NOT EXISTS なしで ERRCODE_DUPLICATE_OBJECT を発生させる。
  • FDWが存在しない、または権限がない場合、エラーを発生させる。
  • オプションの検証（transformGenericOptions）でバリデーターが失敗した場合、エラーを発生させる。

保守担当者向けメモ

  • 本関数はDDL定義のみを扱い、実行時のデータ取得は行わない。
  • サーバー所有者は常に実行ユーザー（GetUserId()）として設定される。
  • 拡張機能内での実行時は、checkMembershipInCurrentExtension によりセキュリティ上の制約が適用される。
