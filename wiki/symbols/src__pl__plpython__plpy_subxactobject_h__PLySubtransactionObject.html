PLySubtransactionObject

定義

  • 種別: struct
  • 定義箇所: src/pl/plpython/plpy_subxactobject.h:16

目的

PL/Python において、PostgreSQL のサブトランザクションを Python のコンテキストマネージャ（with 文）として制御するための Python オブジェクト構造体です。

シグネチャまたは形状

typedef struct PLySubtransactionObject
{
	PyObject_HEAD
	bool		started;
	bool		exited;
} PLySubtransactionObject;

入力と出力

  • **入力**: Python インタプリタから plpy.subtransaction() を通じてインスタンス化されます。
  • **出力**: Python オブジェクトとして、サブトランザクションの開始・終了状態を保持します。

前提条件と不変条件

  • PyObject_HEAD を含み、Python のオブジェクトシステムと互換性があります。
  • started および exited フラグにより、サブトランザクションのライフサイクル（開始・終了）を管理します。

アルゴリズムまたは振る舞い

  • __enter__ メソッド（plpy_subxactobject.c）呼び出し時: BeginInternalSubTransaction を実行し、サブトランザクションを開始します。
  • __exit__ メソッド（plpy_subxactobject.c）呼び出し時: 例外の有無に応じてサブトランザクションのコミットまたはロールバックを行い、リソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_subxactobject.c 内の各関数（PyObject_New による生成や、__enter__/__exit__ メソッド内での型キャストなど）。
  • **呼び出し先**: Python C API（PyObject_New 等）および PostgreSQL 内部のサブトランザクション制御関数（BeginInternalSubTransaction 等）。

関連データ構造

  • PLySubtransactionData: PostgreSQL 内部でサブトランザクションのコンテキストを保持するために使用されます。
  • explicit_subtransactions: 現在アクティブなサブトランザクションを追跡するグローバルなリスト。

失敗モードと境界条件

  • 二重実行や不正な操作を防止するため、started および exited フラグによる状態チェックが行われます。

保守担当者向けメモ

  • 本構造体は Python のコンテキストマネージャとして設計されています。
  • explicit_subtransactions リストを直接操作する場合は、メモリコンテキストの切り替えに注意してください。
