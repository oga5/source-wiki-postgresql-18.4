moveToWaste

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_parse.c:141

目的

LexizeData 構造体内の作業用トークンリスト（towork）から、指定されたトークン（stop）に到達するまでの要素を削除し、処理済み状態へ移行させる。

シグネチャまたは形状

static void moveToWaste(LexizeData *ld, ParsedLex *stop)

入力と出力

  • **入力**:
  • ld: 辞書処理の状態を保持する LexizeData 構造体へのポインタ。
  • stop: 削除処理を停止する基準となる ParsedLex 構造体へのポインタ。
  • **出力**: なし（副作用として ld の状態を更新）。

前提条件と不変条件

  • ld->towork リストが有効であること。
  • stop は ld->towork リスト内に存在するか、あるいは NULL であることが想定される（NULL の場合はリスト全体が処理される）。

アルゴリズムまたは振る舞い

1. ld->towork.head が存在し、かつ処理継続フラグ go が真である間、ループを実行する。 2. 現在の先頭要素が stop と一致する場合：

  • ld->curSub を stop->next に更新する。
  • go を偽にしてループを終了する。

    • 3. RemoveHead(ld) を呼び出し、現在の先頭要素をリストから削除する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/ts_parse.c:319
  • src/backend/tsearch/ts_parse.c:324
  • **呼び出し先**:
  • RemoveHead (同ファイル内)

関連データ構造

  • LexizeData: 辞書処理の現在の状態を保持する構造体。
  • ParsedLex: トークン情報を保持する構造体。

失敗モードと境界条件

  • stop が ld->towork リスト内に存在しない場合、リスト全体が空になるまで RemoveHead が繰り返される。
  • ld->towork.head が NULL の場合、何も行わずに終了する。

保守担当者向けメモ

  • 本関数は辞書チェーンの処理において、正規化が完了したトークンを「作業中」から「処理済み」へ移動させるためのユーティリティである。
  • RemoveHead の副作用に依存しているため、リストのメモリ管理については RemoveHead の実装を参照すること。
