hash_update_hash_key

定義

  • 種別: function
  • 定義箇所: src/backend/utils/hash/dynahash.c:1145

目的

ハッシュテーブル内の既存エントリのキーを更新する。キーの変更に伴い、必要に応じてハッシュチェーン内でのエントリの再配置を行う。

シグネチャまたは形状

bool hash_update_hash_key(HTAB *hashp, void *existingEntry, const void *newKeyPtr)

入力と出力

  • **入力**:
  • hashp: 対象のハッシュテーブル制御構造体。
  • existingEntry: 更新対象の既存エントリへのポインタ。
  • newKeyPtr: 新しいキー値へのポインタ。
  • **出力**:
  • bool: 更新に成功した場合は true。新しいキーが既にハッシュテーブル内に存在し、衝突が発生した場合は false。

前提条件と不変条件

  • hashp が指すハッシュテーブルが frozen 状態（hashp->frozen == true）である場合、更新は許可されずエラーとなる。
  • existingEntry は、指定されたハッシュテーブル内に実際に存在していなければならない。存在しない場合はエラーとなる。

アルゴリズムまたは振る舞い

1. **既存エントリの検証**: existingEntry の保存されたハッシュ値を用いてハッシュテーブルを検索し、エントリが現在のハッシュチェーンに存在することを確認する。 2. **新しいキーのハッシュ計算**: newKeyPtr から新しいハッシュ値を計算し、挿入先のバケットを特定する。 3. **重複チェック**: 新しいキーが既にハッシュテーブル内に存在するかを走査し、存在する場合は false を返して終了する。 4. **再配置**:

  • 新旧のハッシュ値が異なるバケットに属する場合、古いチェーンからエントリを切り離し、新しいバケットのチェーンにリンクし直す。
  • 同一バケット内の場合はリンクの変更を行わない。

    • 5. **キーの更新**: エントリのハッシュ値を更新し、keycopy 関数を用いて新しいキーをエントリにコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/lock.c (直接的な呼び出しを確認)
  • **呼び出し先**:
  • hash_initial_lookup: ハッシュ値からバケットを特定するために使用。
  • hashp->hash: 新しいキーのハッシュ値計算に使用。
  • hashp->match: キーの比較に使用。
  • hashp->keycopy: キーのコピーに使用。
  • elog: エラー報告に使用。

関連データ構造

  • HTAB: ハッシュテーブルの制御構造体。
  • HASHELEMENT: ハッシュテーブル内の各要素を保持する構造体。
  • HASHHDR: 共有メモリ上のハッシュテーブルヘッダー（HASH_STATISTICS 有効時に参照）。

失敗モードと境界条件

  • **エラー**: hashp->frozen が真の場合、または existingEntry がハッシュテーブル内に見つからない場合は elog(ERROR, ...) により処理が中断される。
  • **衝突**: 新しいキーが既に存在する場合、false を返し、エントリの更新は行われない。

保守担当者向けメモ

  • 本関数は、ハッシュテーブルの構造を維持したままキーのみを書き換えるため、ハッシュ値の変更に伴うチェーンの再構築が不可欠である。
  • HASH_STATISTICS が有効な場合、アクセス数および衝突数がカウントされる。
  • 呼び出し元で existingEntry の有効性を保証する必要がある。
