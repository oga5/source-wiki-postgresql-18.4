hashadjustmembers

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hashvalidate.c:262

目的

ハッシュ演算子ファミリー（opfamily）に追加される演算子およびサポート関数に対し、適切な依存関係（ハード依存またはソフト依存）を割り当てる。

シグネチャまたは形状

void hashadjustmembers(Oid opfamilyoid, Oid opclassoid, List *operators, List *functions)

入力と出力

  • **入力**:
  • opfamilyoid: 対象の演算子ファミリーの OID。
  • opclassoid: 対象の演算子クラスの OID（指定がない場合は InvalidOid）。
  • operators: OpFamilyMember 構造体のリスト（演算子）。
  • functions: OpFamilyMember 構造体のリスト（サポート関数）。
  • **出力**: なし（引数として渡されたリスト内の OpFamilyMember 構造体のフィールドを直接更新する）。

前提条件と不変条件

  • opclassoid が有効な場合、CommandCounterIncrement() が実行され、システムカタログの最新状態が参照可能であること。
  • 演算子およびサポート関数は OpFamilyMember 構造体として表現されていること。

アルゴリズムまたは振る舞い

1. opclassoid が有効な場合、対応する入力型（opcintype）を取得する。 2. 演算子リストと関数リストを結合し、各メンバーに対して以下のルールで依存関係を決定する：

  • **オプションのサポート関数**（HASHSTANDARD_PROC 以外）: 常に演算子ファミリーへのソフト依存とする。
  • **クロスタイプ演算子**（左辺型と右辺型が異なる）: 常に演算子ファミリーへのソフト依存とする。
  • **上記以外（標準的な演算子）**:
  • 演算子の入力型が現在の opcintype と一致する場合、演算子クラスへのハード依存とする。
  • 一致しない場合、opclass_for_family_datatype を用いて適切な演算子クラスを検索し、存在すればハード依存、存在しなければ演算子ファミリーへのソフト依存とする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/hash/hash.c: amroutine->amadjustmembers に割り当てられている。
  • **呼び出し先**:
  • get_opclass_input_type
  • opclass_for_family_datatype
  • CommandCounterIncrement

関連データ構造

  • OpFamilyMember: 演算子や関数の依存関係情報を保持する構造体。ref_is_hard, ref_is_family, refobjid フィールドが更新される。

失敗モードと境界条件

  • 演算子クラスが存在しない場合、ハード依存を強制せず、演算子ファミリーへのソフト依存にフォールバックする。
  • ダンプ/リストア時のオブジェクト復元順序により、依存関係の選択が影響を受ける可能性がある（ダンプ/リストアのハザード）。

保守担当者向けメモ

  • 本関数は CREATE OPERATOR CLASS 実行時に呼び出される。
  • クロスタイプ演算子を演算子クラスに強くバインドすることは避けるべきであり、本関数はその整合性を維持する役割を持つ。
