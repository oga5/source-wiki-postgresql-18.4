EOH_init_header

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/expandeddatum.c:47

目的

展開済みオブジェクト（Expanded Object）のヘッダーを初期化し、メソッドテーブルとメモリコンテキストを紐付け、読み取り専用および読み取り書き込み可能なTOASTポインタを設定します。

シグネチャまたは形状

void EOH_init_header(ExpandedObjectHeader *eohptr,
                     const ExpandedObjectMethods *methods,
                     MemoryContext obj_context)

入力と出力

  • **入力**:
  • eohptr: 初期化対象となる ExpandedObjectHeader 構造体へのポインタ。
  • methods: オブジェクトの操作ロジックを定義する ExpandedObjectMethods 構造体へのポインタ。
  • obj_context: オブジェクトのデータが保持される MemoryContext。
  • **出力**: なし（eohptr が指すメモリ領域を直接更新します）。

前提条件と不変条件

  • eohptr は有効なメモリ領域を指している必要があります。
  • obj_context は、展開済みオブジェクトの全データが保持されるメモリコンテキストである必要があります。

アルゴリズムまたは振る舞い

1. eohptr->vl_len_ に EOH_HEADER_MAGIC を設定し、展開済みオブジェクトであることを識別可能にします。 2. eohptr->eoh_methods および eohptr->eoh_context に、それぞれ引数で渡されたメソッドテーブルとメモリコンテキストを格納します。 3. 内部変数 ptr に eohptr を保持し、以下のポインタを初期化します。

  • eoh_rw_ptr: VARTAG_EXPANDED_RW タグを付与し、ptr をコピーして書き込み可能ポインタとして設定します。
  • eoh_ro_ptr: VARTAG_EXPANDED_RO タグを付与し、ptr をコピーして読み取り専用ポインタとして設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/array_expanded.c
  • src/backend/utils/adt/expandedrecord.c
  • **呼び出し先**:
  • memcpy（標準ライブラリ）
  • SET_VARTAG_EXTERNAL（マクロ）
  • VARDATA_EXTERNAL（マクロ）

関連データ構造

  • ExpandedObjectHeader: 展開済みオブジェクトの共通ヘッダー。
  • ExpandedObjectMethods: オブジェクト固有の操作メソッド群。
  • varatt_expanded: 展開済みオブジェクトを指すTOASTポインタの内部表現。

失敗モードと境界条件

  • eohptr が NULL の場合、セグメンテーション違反が発生します。
  • obj_context が無効な場合、後続のメモリ操作で予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は、新しい展開済みデータ型を実装する際の必須初期化ステップです。
  • memcpy を使用しているのは、TOASTポインタがメモリ上で適切にアライメントされていない可能性があるためです。
  • 読み取り専用（R/O）と読み取り書き込み可能（R/W）の両方のポインタを初期化することで、型システムによるアクセス制御をサポートしています。
