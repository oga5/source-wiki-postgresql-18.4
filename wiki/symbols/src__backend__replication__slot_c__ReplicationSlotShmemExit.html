ReplicationSlotShmemExit

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slot.c:247

目的

共有メモリの終了処理時に、現在のバックエンドが保持するレプリケーションスロットの解放および一時スロットのクリーンアップを行う。

シグネチャまたは形状

static void ReplicationSlotShmemExit(int code, Datum arg)

入力と出力

  • code: 終了コード（before_shmem_exit から渡される）。
  • arg: 補助データ（before_shmem_exit から渡される）。
  • 戻り値: なし。

前提条件と不変条件

  • before_shmem_exit コールバックとして登録されていること。
  • MyReplicationSlot が現在のバックエンドに関連付けられている場合、そのスロットの所有権が適切に管理されていること。

アルゴリズムまたは振る舞い

1. MyReplicationSlot が NULL でない場合、ReplicationSlotRelease() を呼び出し、現在のアクティブなレプリケーションスロットを解放する。 2. ReplicationSlotCleanup(false) を呼び出し、一時的なレプリケーションスロットのクリーンアップを実行する。

呼び出し元と呼び出し先

  • 呼び出し元: before_shmem_exit により登録され、共有メモリ終了時に呼び出される。
  • 呼び出し先:
  • ReplicationSlotRelease()
  • ReplicationSlotCleanup()

関連データ構造

  • MyReplicationSlot: 現在のバックエンドが所有するスロットへのポインタ。

失敗モードと境界条件

  • MyReplicationSlot が NULL の場合、解放処理はスキップされる。
  • ReplicationSlotCleanup に渡される引数は false で固定されており、一時スロットの削除のみを対象とする。

保守担当者向けメモ

  • 本関数は before_shmem_exit を介して実行されるため、プロセス終了時のクリーンアップ順序に依存する。
  • ReplicationSlotCleanup の引数 false は、永続的なスロットまで削除しないことを意味する。
