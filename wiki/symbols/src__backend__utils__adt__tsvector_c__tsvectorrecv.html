tsvectorrecv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector.c:446

目的

ネットワーク経由で受信したバイナリ形式のデータから tsvector 型の内部構造を再構築する。

シグネチャまたは形状

Datum tsvectorrecv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_POINTER(0) を介して StringInfo バッファを受け取る。
  • **出力**: 構築された TSVector 構造体を Datum として返す。

前提条件と不変条件

  • 受信データは、tsvectorsend が生成する形式（エントリ数、語彙文字列、位置情報）に従っている必要がある。
  • tsvector 内の語彙は辞書順にソートされている必要がある。ソートされていない場合は、関数内で qsort_arg を用いて再ソートされる。
  • 位置情報（WordEntryPos）は、各語彙内で昇順に並んでいる必要がある。

アルゴリズムまたは振る舞い

1. 受信したエントリ数（nentries）を読み取り、妥当性を検証する。 2. TSVector 構造体用のメモリを確保し、各エントリをループ処理する。 3. 各語彙（lexeme）と位置情報の数（npos）をバッファから読み取る。 4. 語彙の長さや位置情報の数が制限値を超えていないか検証する。 5. 必要に応じてメモリを repalloc で拡張し、語彙データをコピーする。 6. 位置情報が存在する場合、2バイト境界にアライメント調整を行い、WordEntryPos の配列を読み取る。この際、位置情報が昇順であることを確認する。 7. 全エントリの処理後、SET_VARSIZE で全体のサイズを確定する。 8. 処理中にソート順の乱れが検出された場合（needSort フラグ）、qsort_arg を用いて compareentry 関数に基づきソートを行う。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（Fmgr）経由で呼び出される。具体的な呼び出し元コードの詳細は提供されていない。
  • **呼び出し先**:
  • pq_getmsgint, pq_getmsgstring: libpq/pqformat.h より。
  • compareentry: 語彙の比較に使用。
  • palloc0, repalloc: メモリ管理。
  • elog: エラー報告。

関連データ構造

  • TSVector: 全文検索用データ型のメイン構造体。
  • WordEntry: 語彙のメタデータ（位置情報の有無、長さ、オフセット）を保持。
  • WordEntryPos: 位置情報と重みをパックしたデータ型。
  • StringInfo: 受信バイナリデータのバッファ。

失敗モードと境界条件

  • nentries が負または許容最大値を超える場合、エラーとなる。
  • 語彙の長さが MAXSTRLEN を超える場合、エラーとなる。
  • 合計語彙長が MAXSTRPOS を超える場合、エラーとなる。
  • 位置情報の数が MAXNUMPOS を超える場合、エラーとなる。
  • 位置情報が昇順でない場合、エラーとなる。

保守担当者向けメモ

  • repalloc によるメモリ拡張時、palloc0 で初期化した領域以降はゼロ初期化されないため、アライメント調整時のパディングには注意が必要である。
  • tsvector の内部レイアウトは WordEntry 配列とそれに続く語彙文字列、位置情報データで構成される。ビット操作やアライメントについては tsearch/ts_type.h の定義を優先すること。
