pg_lsn_le

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_lsn.c:153

目的

2つのログシーケンス番号（pg_lsn）を比較し、第1引数が第2引数以下であるかどうかを判定する。

シグネチャまたは形状

Datum pg_lsn_le(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される2つの pg_lsn 型の値。
  • **出力**: bool 型（Datum として返却）。第1引数が第2引数以下の場合は true、そうでない場合は false。

前提条件と不変条件

  • 入力される pg_lsn は、内部的に XLogRecPtr（64ビット符号なし整数）として扱われる。
  • 比較は64ビット整数の大小比較として行われる。

アルゴリズムまたは振る舞い

1. PG_GETARG_LSN マクロを使用して、第1引数および第2引数を XLogRecPtr 型として取得する。 2. C言語の比較演算子 <= を用いて、第1引数（lsn1）が第2引数（lsn2）以下であるかを評価する。 3. PG_RETURN_BOOL マクロを使用して、評価結果を Datum 型に変換して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（演算子 <= の実行時）。
  • **呼び出し先**:
  • PG_GETARG_LSN（マクロ）
  • PG_RETURN_BOOL（マクロ）
  • 注: ソースコード上の直接的な関数呼び出し関係は上記マクロのみであり、他の関数への依存は確認されていない。

関連データ構造

  • XLogRecPtr: LSNの内部表現（uint64）。src/include/access/xlogdefs.h で定義。

失敗モードと境界条件

  • 引数が有効な pg_lsn 型であることを前提としており、不正なメモリ領域へのアクセスや型不整合については、PostgreSQLの関数呼び出し規約（fmgr）の枠組みで処理される。

保守担当者向けメモ

  • 本関数はB-treeインデックスの比較演算子として利用されることを想定している。
  • 比較ロジックは単純な64ビット整数の比較であるため、計算コストは極めて低い。
