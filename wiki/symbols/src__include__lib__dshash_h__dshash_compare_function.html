dshash_compare_function

定義

  • 種別: typedef
  • 定義箇所: src/include/lib/dshash.h:33

目的

dshash（動的共有ハッシュテーブル）において、2つのキーを比較するためのコールバック関数型を定義します。

シグネチャまたは形状

typedef int (*dshash_compare_function) (const void *a, const void *b,
                                        size_t size, void *arg);

入力と出力

  • a, b: 比較対象となるキーへのポインタ。
  • size: キーのサイズ。
  • arg: 比較関数に渡される追加のコンテキスト引数（dshash_parameters で指定）。
  • 戻り値: a と b が等しい場合は 0 を返し、それ以外の場合は非ゼロの値を返します。

前提条件と不変条件

  • 本関数は dshash_parameters 構造体を通じてハッシュテーブルに登録されます。
  • 関数ポインタ自体はプロセス間で共有できないため、各プロセスが dshash_attach を行う際に再設定する必要があります。

アルゴリズムまたは振る舞い

ハッシュテーブル内での検索、挿入、削除操作において、キーの衝突が発生した際や特定のエントリを特定する際に、この関数が呼び出されます。提供された2つのキーを比較し、一致するかどうかを判定します。

呼び出し元と呼び出し先

  • 呼び出し元: dshash の内部実装（src/backend/lib/dshash.c）から、キーの比較が必要なタイミングで呼び出されます。
  • 呼び出し先: ユーザー定義の比較関数が割り当てられます。
  • 補足: 具体的な呼び出し元関数の特定には、dshash.c の実装詳細の解析が必要です。

関連データ構造

  • dshash_parameters: この関数型をメンバ compare_function として保持します。

失敗モードと境界条件

  • a または b が NULL の場合の挙動は、実装される関数側に依存します。
  • 比較対象のキーサイズが size と一致しない場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数はプロセス間で共有されるメモリ領域には格納されません。各プロセスがハッシュテーブルにアタッチする際に、適切な関数ポインタを再設定することを忘れないでください。
