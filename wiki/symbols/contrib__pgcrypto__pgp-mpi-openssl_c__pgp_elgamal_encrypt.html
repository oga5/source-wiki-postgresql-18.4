pgp_elgamal_encrypt

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-mpi-openssl.c:100

目的

OpenPGP 公開鍵を用いた ElGamal 暗号化処理を実行します。入力されたメッセージ（MPI）を、指定された公開鍵のパラメータを用いて暗号化し、2つの MPI 成分（$c_1, c_2$）を生成します。

シグネチャまたは形状

int pgp_elgamal_encrypt(PGP_PubKey *pk, PGP_MPI *_m, PGP_MPI **c1_p, PGP_MPI **c2_p)

入力と出力

  • **入力**:
  • pk: 暗号化に使用する PGP_PubKey 構造体（ElGamal 公開鍵パラメータ $p, g, y$ を含む）。
  • _m: 暗号化対象のメッセージ（PGP_MPI）。
  • **出力**:
  • c1_p, c2_p: 暗号化された結果を格納する PGP_MPI へのポインタ。
  • **戻り値**: 成功時に 0、失敗時に PXE_PGP_MATH_FAILED を返します。

前提条件と不変条件

  • pk には有効な ElGamal 公開鍵パラメータが含まれている必要があります。
  • 内部で使用する OpenSSL の BIGNUM および BN_CTX は、関数終了時に必ず解放される必要があります。

アルゴリズムまたは振る舞い

1. mpi_to_bn を使用して、入力された PGP_MPI および公開鍵パラメータを OpenSSL の BIGNUM 形式に変換します。 2. decide_k_bits を呼び出し、公開鍵のビット長に基づいた乱数 $k$ のビット数を決定します。 3. BN_rand を用いて乱数 $k$ を生成します。 4. OpenSSL の BN_mod_exp を使用して以下の計算を行います。

  • $c_1 = g^k \pmod p$
  • $yk = y^k \pmod p$
  • $c_2 = m \cdot yk \pmod p$

    • 5. bn_to_mpi を使用して結果を PGP_MPI 形式に戻し、呼び出し元へ返します。 6. エラー発生時は goto err を経由し、確保したすべてのリソースを BN_clear_free で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-pubenc.c 内の関数から呼び出されます。
  • **呼び出し先**: mpi_to_bn, bn_to_mpi, decide_k_bits（いずれも同ファイル内のヘルパー関数）、および OpenSSL の BN_* 系関数。

関連データ構造

  • PGP_PubKey: 公開鍵情報を保持する構造体。
  • PGP_MPI: OpenPGP の多倍長整数表現。
  • BIGNUM, BN_CTX: OpenSSL の多倍長整数演算用構造体。

失敗モードと境界条件

  • メモリ不足や OpenSSL の演算失敗時には PXE_PGP_MATH_FAILED を返します。

保守担当者向けメモ

  • 本関数は GnuPG の暗号化ロジックを模倣しています。
  • 乱数 $k$ の生成には BN_rand を使用しており、セキュリティ強度は decide_k_bits の実装に依存します。
  • エラーハンドリングは goto err パターンによる厳格なリソース解放を行っているため、新しい BIGNUM を追加する際は必ず err ラベル以降の解放処理を確認してください。
