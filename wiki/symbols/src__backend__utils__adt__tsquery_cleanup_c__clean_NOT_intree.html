clean_NOT_intree

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_cleanup.c:135

目的

tsquery のツリー構造から NOT 演算子を含むノードを再帰的に探索し、除去することでクエリを簡略化します。

シグネチャまたは形状

static NODE *clean_NOT_intree(NODE *node);

入力と出力

  • **入力**: NODE *node（処理対象のクエリツリーのルートノード）
  • **出力**: NODE *（NOT 演算子が除去された後の新しいツリーのルート、またはツリー全体が削除された場合は NULL）

前提条件と不変条件

  • node は有効な NODE 構造体である必要があります。
  • 再帰呼び出しを行うため、スタックオーバーフローを防止する check_stack_depth() が呼び出されます。

アルゴリズムまたは振る舞い

1. **スタックチェック**: check_stack_depth() を実行し、スタックの深さを確認します。 2. **終端ノード**: QI_VAL 型のノードであれば、そのままノードを返します。 3. **NOT 演算子の処理**: OP_NOT 演算子の場合、そのノードと配下のツリーを freetree() で解放し、NULL を返します。 4. **OR 演算子の処理**: 左右の子ノードに対して再帰的に clean_NOT_intree を適用します。どちらか一方が NULL になった場合、OR ノード全体を解放して NULL を返します。 5. **AND / PHRASE 演算子の処理**: 左右の子ノードに対して再帰的に clean_NOT_intree を適用します。

  • 両方の枝が NULL になった場合、自身を解放して NULL を返します。
  • 片方の枝が NULL になった場合、残った枝を返し、自身を解放します。
  • 両方の枝が残る場合、更新されたノードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery_cleanup.c 内の clean_NOT_intree（再帰呼び出し）
  • src/backend/utils/adt/tsquery_cleanup.c 内の clean_NOT（外部からの呼び出し）
  • **呼び出し先**:
  • check_stack_depth()
  • freetree()
  • pfree()
  • clean_NOT_intree()（再帰）

関連データ構造

  • NODE: クエリツリーを構成するノード構造体。
  • QueryItem: tsquery のフラットな表現。

失敗モードと境界条件

  • **スタックオーバーフロー**: 深いネストを持つクエリツリーに対しては、check_stack_depth() によりエラーが発生する可能性があります。
  • **全ノード削除**: NOT 演算子の除去によりツリー全体が不要と判断された場合、NULL が返されます。

保守担当者向けメモ

  • 本関数は OP_AND および OP_PHRASE 演算子において、子ノードが NULL になった場合に自身のノードを pfree し、残った子ノードを昇格させることでツリーを再構築します。この際、メモリリークが発生しないよう freetree() と pfree() の使い分けに注意してください。
