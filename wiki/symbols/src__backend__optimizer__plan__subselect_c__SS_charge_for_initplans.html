SS_charge_for_initplans

定義

  • **種別**: function
  • **定義箇所**: src/backend/optimizer/plan/subselect.c:2247

目的

クエリ計画の各 Path に対して、InitPlan の実行コストを反映させ、必要に応じて並列実行の安全性（parallel_safe）を調整する。

シグネチャまたは形状

void SS_charge_for_initplans(PlannerInfo *root, RelOptInfo *final_rel)

入力と出力

  • **入力**:
  • root: クエリ全体の計画情報を保持する PlannerInfo 構造体。
  • final_rel: 最終的なリレーション情報を保持する RelOptInfo 構造体。
  • **出力**: なし（final_rel 内の pathlist および partial_pathlist が直接更新される）。

前提条件と不変条件

  • root->init_plans が NIL の場合、本関数は何も行わず即座に復帰する。
  • final_rel 内の各 Path は、本関数が呼び出される時点で既に生成されている必要がある。

アルゴリズムまたは振る舞い

1. SS_compute_initplan_cost を呼び出し、全 InitPlan の合計コストと並列実行の安全性（unsafe_initplans）を算出する。 2. final_rel->pathlist 内のすべての Path について、startup_cost および total_cost に算出したコストを加算する。 3. unsafe_initplans が真の場合、すべての Path の parallel_safe フラグを false に設定する。 4. unsafe_initplans が真の場合、final_rel->partial_pathlist をクリアし、final_rel->consider_parallel を false に設定する。 5. unsafe_initplans が偽の場合、final_rel->partial_pathlist 内の各 Path のコストにも同様にコストを加算する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planagg.c:431
  • src/backend/optimizer/plan/planner.c:1289
  • **呼び出し先**:
  • SS_compute_initplan_cost

関連データ構造

  • PlannerInfo: クエリ全体の計画状態を保持。
  • RelOptInfo: リレーションのパスリスト（pathlist, partial_pathlist）を保持。
  • Path: コスト情報および並列実行フラグを保持。

失敗モードと境界条件

  • root->init_plans が空の場合、処理はスキップされる。
  • unsafe_initplans が真の場合、部分パス（partial_pathlist）は破棄され、並列実行は無効化される。

保守担当者向けメモ

  • 本関数はコストの加算のみを行い、set_cheapest() によるパスの再評価は行わない（呼び出し元が担当する）。
  • InitPlan のコスト計算ロジックを変更する場合は、SS_compute_initplan_cost を参照すること。
