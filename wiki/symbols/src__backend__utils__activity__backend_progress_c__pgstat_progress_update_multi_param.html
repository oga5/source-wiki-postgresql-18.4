pgstat_progress_update_multi_param

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/backend_progress.c:120

目的

長時間実行コマンドの進捗状況を示す複数のパラメータを、共有メモリ上でアトミックに更新する。

シグネチャまたは形状

void pgstat_progress_update_multi_param(int nparam, const int *index, const int64 *val)

入力と出力

  • nparam (int): 更新するパラメータの個数。
  • index (const int*): 更新対象のパラメータインデックス（0〜19）を格納した配列。
  • val (const int64*): 各インデックスに対応する新しい値を格納した配列。
  • 戻り値: なし。

前提条件と不変条件

  • beentry（現在のバックエンドの共有メモリ状態）が有効であること。
  • pgstat_track_activities が有効であること。
  • index 配列内の各値は 0 以上 PGSTAT_NUM_PROGRESS_PARAM 未満であること（Assert により検証）。

アルゴリズムまたは振る舞い

1. beentry が無効、pgstat_track_activities が無効、または nparam が 0 の場合は即座にリターンする。 2. PGSTAT_BEGIN_WRITE_ACTIVITY マクロを使用して、共有メモリへの書き込みロック（またはメモリバリア）を確保する。 3. 指定された nparam 分だけループし、index[i] で指定された共有メモリ上の進捗パラメータを val[i] で更新する。 4. PGSTAT_END_WRITE_ACTIVITY マクロを使用して書き込みを完了する。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/access/gin/gininsert.c
  • src/backend/access/heap/heapam_handler.c
  • src/backend/access/heap/vacuumlazy.c
  • src/backend/access/nbtree/nbtsort.c
  • src/backend/backup/basebackup_progress.c
  • 呼び出し先:
  • PGSTAT_BEGIN_WRITE_ACTIVITY / PGSTAT_END_WRITE_ACTIVITY マクロ（共有メモリ操作用）。

関連データ構造

  • PgBackendStatus: 共有メモリ上のバックエンド状態構造体。
  • MyBEEntry: 現在のバックエンドの PgBackendStatus を指すポインタ。

失敗モードと境界条件

  • nparam が 0 の場合、何も更新せず正常終了する。
  • pgstat_track_activities が無効な場合、更新は行われない。
  • index が範囲外の場合、デバッグビルドでは Assert により停止する。

保守担当者向けメモ

  • 本関数は複数のパラメータを一度のロック期間で更新するため、読み取り側（ビューなど）から見て不整合な状態が発生するのを防ぐ。
  • 各インデックス（0〜19）の具体的な意味論は、本関数ではなく各コマンドの実装側に依存する。
