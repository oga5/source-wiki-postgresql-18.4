output_line_number

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/output.c:9

目的

生成されるCコード内に #line ディレクティブを出力し、コンパイル後のコードと元のECPGソースコードの行番号を同期させる。

シグネチャまたは形状

void output_line_number(void)

入力と出力

  • 入力: なし（内部的に hashline_number() を呼び出し、現在のソース位置を取得する）。
  • 出力: なし（副作用として、グローバルな出力ストリーム base_yyout へ文字列を書き出す）。

前提条件と不変条件

  • base_yyout が有効なファイルストリームとしてオープンされていること。
  • hashline_number() が現在のソースファイル名と行番号を含む有効な文字列を返すこと。

アルゴリズムまたは振る舞い

1. hashline_number() を呼び出し、現在のソース位置情報を含む #line ディレクティブ文字列を取得する。 2. 取得した文字列を fprintf を使用して base_yyout に書き出す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/interfaces/ecpg/preproc/ecpg.c:480
  • src/interfaces/ecpg/preproc/output.c 内の各出力関数（行番号 23, 68, 74, 80, 88）
  • 呼び出し先:
  • hashline_number() (同モジュール内)
  • fprintf (標準ライブラリ)

関連データ構造

  • base_yyout: 出力先となるファイルストリーム（preproc_extern.h で定義）。

失敗モードと境界条件

  • base_yyout が NULL または書き込み不可能な状態である場合、fprintf により実行時エラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は #line ディレクティブの生成を抽象化している。デバッグ時に生成されたCコードの行番号がずれている場合は、hashline_number() の実装を確認すること。
