pg_localtime

定義

  • 種別: function
  • 定義箇所: src/timezone/localtime.c:1343

目的

指定されたタイムゾーン定義（pg_tz）に基づき、UNIX タイムスタンプ（pg_time_t）を分解された時刻構造体（pg_tm）へ変換する外部インターフェースです。

シグネチャまたは形状

struct pg_tm *pg_localtime(const pg_time_t *timep, const pg_tz *tz);

入力と出力

  • **入力**:
  • timep: 変換対象の pg_time_t 型の時刻値へのポインタ。
  • tz: 変換に使用するタイムゾーン情報を含む pg_tz 構造体へのポインタ。
  • **出力**:
  • 変換結果を格納した pg_tm 構造体へのポインタを返します。

前提条件と不変条件

  • tz は有効な pg_tz 構造体を指している必要があります。
  • 内部的に静的領域（tm）を使用しているため、呼び出しごとに結果が上書きされる可能性があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った tz 構造体内の state メンバを参照します。 2. 内部関数 localsub を呼び出し、指定されたタイムゾーンルールに従って時刻変換を実行します。 3. 変換後の pg_tm 構造体へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogbackup.c
  • src/backend/postmaster/syslogger.c
  • src/backend/utils/adt/timestamp.c
  • src/backend/utils/error/elog.c
  • src/bin/initdb/findtimezone.c
  • **呼び出し先**:
  • localsub（src/timezone/localtime.c 内の内部関数）

関連データ構造

  • pg_time_t: UNIX タイムスタンプを表す型。
  • pg_tz: タイムゾーン定義を保持する構造体。
  • pg_tm: 分解された時刻（年・月・日・時・分・秒等）を保持する構造体。

失敗モードと境界条件

  • pg_time_t の範囲外の時刻が指定された場合の挙動は localsub の実装に依存します。
  • 内部で静的変数を使用しているため、マルチスレッド環境での同時呼び出しには注意が必要です。

保守担当者向けメモ

  • 本関数は localsub のラッパーです。詳細な変換ロジックや閏秒の扱いは localsub および timesub を参照してください。
  • PostgreSQL は原則として閏秒を無視する設計であるため、利用するタイムゾーンが閏秒を含む場合は pg_tz_acceptable による事前チェックが推奨されます。
