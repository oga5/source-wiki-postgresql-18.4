gistvacuum_delete_empty_pages

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/gist/gistvacuum.c:503

目的

VACUUM 処理において、vstate->empty_leaf_set に記録された空のリーフページを特定し、それらを参照している親ページ（内部ページ）のダウンリンクを削除してツリーから切り離す。

シグネチャまたは形状

static void gistvacuum_delete_empty_pages(IndexVacuumInfo *info, GistVacState *vstate)

入力と出力

  • **入力**:
  • info: インデックスの VACUUM 情報を保持する IndexVacuumInfo 構造体。
  • vstate: スキャン中の状態を保持する GistVacState 構造体（空リーフの集合および内部ページの集合を含む）。
  • **出力**: なし（副作用としてインデックスページが更新される）。

前提条件と不変条件

  • vstate->internal_page_set には、空のリーフページを指す可能性のある内部ページ番号が格納されている必要がある。
  • 親ページから最後の一つのダウンリンクを削除することは禁止されている（インデックス構造の維持のため）。

アルゴリズムまたは振る舞い

1. vstate->internal_page_set を走査し、各内部ページを読み込む。 2. 内部ページ内の各ダウンリンクを検査し、その参照先が vstate->empty_leaf_set に含まれるかを確認する。 3. 削除対象のリーフページが見つかった場合、デッドロックを避けるため以下の順序で処理を行う：

  • 親ページのロックを一時的に解放する。
  • 子（リーフ）ページを GIST_EXCLUSIVE モードでロックする。
  • 親ページを再ロックし、gistdeletepage を呼び出してダウンリンクを削除する。

    • 4. 削除完了後、子ページのロックを解放し、親ページのロックを解除する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistvacuum.c 内の gistvacuumcleanup から呼び出される。
  • **呼び出し先**:
  • ReadBufferExtended, LockBuffer, UnlockReleaseBuffer (ストレージ管理)
  • gistdeletepage (GiST ページ削除ロジック)
  • gistcheckpage (整合性チェック)

関連データ構造

  • GistVacState: 空リーフページ集合 (empty_leaf_set) と内部ページ集合 (internal_page_set) を保持。
  • IndexVacuumInfo: インデックスのメタデータと戦略情報。

失敗モードと境界条件

  • **ページ状態の変化**: 処理中にページが内部ページから別の種類へ変化した場合、Assert(false) により停止する。
  • **最後の一つのダウンリンク**: ページ内の唯一のダウンリンクは削除されない。
  • **並行性**: 親ページのロック解放中にダウンリンクが移動する可能性があるため、gistdeletepage 内で再チェックが行われる。

保守担当者向けメモ

  • 本関数は gistvacuumcleanup の一部として実行される。
  • デッドロック回避のため、親ページをロックしたまま子ページをロックすることは行わない設計となっている。
  • ページ削除の際は gistdeletepage に依存しており、同関数の再チェックロジックが並行性制御の要となっている。
