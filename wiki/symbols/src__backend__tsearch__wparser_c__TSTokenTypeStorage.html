TSTokenTypeStorage

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/wparser.c:27

目的

SQLインターフェースを通じてパーサーのトークン型定義を順次取得（Set-Returning Function）する際、その反復処理の状態を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	int			cur;      /* 現在処理中のトークン型のインデックス */
	LexDescr   *list;     /* パーサーがサポートするトークン型の配列 */
} TSTokenTypeStorage;

入力と出力

  • 本構造体は関数内部で palloc を介してメモリ確保され、FuncCallContext の user_fctx に格納されます。
  • 入力: パーサー定義から取得した LexDescr のリスト。
  • 出力: SRFの各呼び出しにおいて、cur が指す現在のトークン型情報。

前提条件と不変条件

  • list はパーサーから取得された有効なトークン型記述子の配列である必要があります。
  • cur は常に 0 から始まり、リストの終端に達するまでインクリメントされます。

アルゴリズムまたは振る舞い

1. **初期化**: tt_setup_firstcall 内で構造体が割り当てられ、パーサーのトークン型リストが list に設定されます。 2. **反復処理**: tt_process_call が呼び出されるたびに、cur が参照する list[cur] を返します。 3. **終了**: cur がリストの終端に達すると、SRFの終了条件（SRF_RETURN_DONE）がトリガーされます。

呼び出し元と呼び出し先

  • **呼び出し元**: ts_token_type_byid および ts_token_type_byname から間接的に利用されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、特定の関数を呼び出すことはありません。

関連データ構造

  • LexDescr: トークン型の詳細（ID、別名、説明）を保持する構造体。
  • FuncCallContext: PostgreSQLのSRF実装において、本構造体を保持するコンテキスト。

失敗モードと境界条件

  • メモリ不足: palloc 失敗時にエラーが発生します。
  • 空のリスト: パーサーがトークン型を定義していない場合、list が NULL または空となり、初回呼び出しで即座に終了します。

保守担当者向けメモ

  • 本構造体は ts_token_type_* 系関数専用のステート管理用です。
  • cur のインクリメントとリストの境界チェックは tt_process_call 内で行われるため、ロジック変更時は当該関数との整合性に注意してください。
