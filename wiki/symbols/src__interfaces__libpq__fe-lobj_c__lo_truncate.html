lo_truncate

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-lobj.c:130

目的

データベース内のラージオブジェクトを指定されたサイズに切り詰める（truncate）。

シグネチャまたは形状

int lo_truncate(PGconn *conn, int fd, size_t len);

入力と出力

  • conn: データベース接続ハンドル。
  • fd: 操作対象のラージオブジェクト記述子。
  • len: 切り詰め後のサイズ（バイト単位）。
  • 戻り値: 成功時は 0、失敗時は -1 を返す。

前提条件と不変条件

  • lo_initialize(conn) が成功していること。
  • サーバー側で lo_truncate 関数が利用可能であること（PostgreSQL 8.3以降）。
  • len は INT_MAX を超えてはならない（サーバー側の実装が int32 に制限されているため）。

アルゴリズムまたは振る舞い

1. lo_initialize を呼び出し、サーバー側関数のOIDを解決・キャッシュする。 2. サーバー側関数 fn_lo_truncate のOIDが PGconn に存在するか確認する。 3. len が INT_MAX を超える場合、エラーを記録して -1 を返す。 4. fd と len を PQArgBlock に格納する。 5. PQfn を介してサーバー側の lo_truncate を実行する。 6. 実行結果を確認し、PGRES_COMMAND_OK であれば PQclear を行い成功値を返す。それ以外はエラーとして -1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: 外部アプリケーション（libpq-fe.h を介して利用）。
  • 呼び出し先: lo_initialize, libpq_append_conn_error, PQfn, PQresultStatus, PQclear。

関連データ構造

  • PGconn: 接続状態およびキャッシュされたOID（lobjfuncs->fn_lo_truncate）を保持。
  • PQArgBlock: サーバー側関数へ渡す引数構造体。

失敗モードと境界条件

  • 接続が未初期化または初期化失敗時: -1 を返す。
  • サーバー側で lo_truncate がサポートされていない場合: -1 を返しエラーを記録。
  • len > INT_MAX: 整数オーバーフローを防ぐため -1 を返しエラーを記録。
  • サーバー側での実行エラー: PQresultStatus が PGRES_COMMAND_OK 以外となり、-1 を返す。

保守担当者向けメモ

  • size_t を引数に取りますが、内部で int32 にキャストされるため、非常に大きなサイズを扱う場合は lo_truncate64 の利用を検討してください。
  • サーバー側の関数OIDは lo_initialize でキャッシュされるため、接続のライフサイクルに依存します。
