list_head

定義

  • 種別: static inline function
  • 定義箇所: src/include/nodes/pg_list.h:127

目的

PostgreSQLの汎用リスト構造（List）の先頭要素を指す ListCell へのポインタを取得します。

シグネチャまたは形状

static inline ListCell *list_head(const List *l)

入力と出力

  • **入力**: const List *l （対象のリストへのポインタ）
  • **出力**: リストが空でない場合はその先頭要素の ListCell へのポインタ。リストが NIL（NULL）または空の場合は NULL を返します。

前提条件と不変条件

  • 入力 l は NULL（NIL）であっても安全です。
  • リストの要素配列 elements は、リストが空でない限り有効であることが保証されています。

アルゴリズムまたは振る舞い

1. 引数 l が NULL かどうかを判定します。 2. l が NULL であれば NULL を返します。 3. l が存在する場合、リスト構造体の elements 配列のインデックス 0 を参照し、そのアドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/deparse.c
  • src/backend/access/common/printtup.c
  • src/backend/access/spgist/spgutils.c
  • src/backend/catalog/index.c
  • src/backend/catalog/namespace.c
  • src/backend/catalog/partition.c
  • src/backend/commands/analyze.c
  • **呼び出し先**:
  • 本関数は単一のインライン関数であり、他の関数を呼び出しません。

関連データ構造

  • List: リストのヘッダ構造体。elements 配列を保持します。
  • ListCell: リスト内の各要素を保持する共用体。

失敗モードと境界条件

  • l が NIL の場合、本関数は NULL を返します。呼び出し側で戻り値の NULL チェックが必要です。

保守担当者向けメモ

  • 本関数はインライン化されており、パフォーマンス重視のコードパスで使用されます。
  • List 構造体の内部実装（elements 配列）に直接依存しているため、構造体の定義変更時には注意が必要です。
