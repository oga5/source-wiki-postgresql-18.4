xact_desc_commit

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/xactdesc.c:333

目的

トランザクションのコミットに関するWALレコードを人間が読める形式に変換し、指定されたバッファへ出力します。

シグネチャまたは形状

void xact_desc_commit(StringInfo buf, uint8 info, xl_xact_commit *xlrec, RepOriginId origin_id)

入力と出力

  • buf (入出力): 整形された文字列が追記される StringInfo バッファ。
  • info (入力): WALレコードの種別を示すビットフラグ。
  • xlrec (入力): コミットレコードのバイナリデータ構造体。
  • origin_id (入力): レプリケーションオリジンID。

前提条件と不変条件

  • ParseCommitRecord を介して xlrec から xl_xact_parsed_commit 構造体へ正しくパースできること。
  • buf は初期化済みであること。

アルゴリズムまたは振る舞い

1. ParseCommitRecord を呼び出し、xlrec から解析済みデータ parsed を取得する。 2. parsed.twophase_xid が有効な場合、そのXIDをバッファに書き込む。 3. xlrec->xact_time をタイムスタンプ文字列に変換してバッファに書き込む。 4. xact_desc_relations、xact_desc_subxacts、xact_desc_stats を順次呼び出し、関連リレーション、サブトランザクション、統計情報をバッファに追加する。 5. standby_desc_invalidations を呼び出し、無効化メッセージを処理する。 6. parsed.xinfo のフラグに基づき、apply_feedback や sync の情報を追記する。 7. XACT_XINFO_HAS_ORIGIN フラグが立っている場合、レプリケーションオリジン情報をバッファに追加する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/rmgrdesc/xactdesc.c 内のメインディスパッチャ（xact_desc）から呼び出されることが示唆されています。
  • 呼び出し先: ParseCommitRecord, appendStringInfo, appendStringInfoString, timestamptz_to_str, xact_desc_relations, xact_desc_subxacts, xact_desc_stats, standby_desc_invalidations, XactCompletionRelcacheInitFileInval, XactCompletionApplyFeedback, XactCompletionForceSyncCommit。

関連データ構造

  • StringInfo: 文字列構築用バッファ。
  • xl_xact_commit: WAL上のコミットレコード構造体。
  • xl_xact_parsed_commit: パース後のコミット情報保持用構造体。
  • RepOriginId: レプリケーションオリジン識別子。

失敗モードと境界条件

  • xlrec が不正な形式である場合、ParseCommitRecord 内で予期しない動作を引き起こす可能性があります。
  • parsed.xinfo のフラグ状態により、出力される文字列の内容が動的に変化します。

保守担当者向けメモ

  • 本関数はバックエンドとフロントエンド（pg_waldump）で共有されるため、依存関係の追加には注意が必要です。
  • WALレコードのパースは info ビットフラグに依存しており、構造変更時は ParseCommitRecord との整合性を維持してください。
