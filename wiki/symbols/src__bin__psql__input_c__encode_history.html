encode_history

定義

  • 種別: function
  • 定義箇所: src/bin/psql/input.c:298

目的

readline の履歴リストに含まれる各エントリ内の改行文字を、履歴ファイルへの保存に適した特殊文字（NL_IN_HISTORY）に変換します。

シグネチャまたは形状

static void encode_history(void)

入力と出力

  • 入力: グローバルな readline 履歴リスト（BEGIN_ITERATE_HISTORY マクロを通じてアクセス）。
  • 出力: なし（履歴リスト内の文字列を直接書き換える副作用を持つ）。

前提条件と不変条件

  • readline ライブラリが初期化され、履歴リストが利用可能であること。
  • BEGIN_ITERATE_HISTORY および END_ITERATE_HISTORY マクロが、環境に応じた適切なイテレーションを提供すること。

アルゴリズムまたは振る舞い

1. BEGIN_ITERATE_HISTORY を使用して、履歴リスト内の全エントリを走査します。 2. 各エントリの文字列（HIST_ENTRY.line）をポインタで走査します。 3. 文字列内の改行文字（\n）を検出し、NL_IN_HISTORY に置換します。 4. END_ITERATE_HISTORY で走査を終了します。

呼び出し元と呼び出し先

  • 呼び出し元: saveHistory（src/bin/psql/input.c:431）から呼び出されます。
  • 呼び出し先: readline ライブラリの履歴管理マクロ（BEGIN_ITERATE_HISTORY / END_ITERATE_HISTORY）。

関連データ構造

  • HIST_ENTRY: readline ライブラリの履歴エントリ構造体。

失敗モードと境界条件

  • HIST_ENTRY.line が const char * として宣言されているプラットフォームでは、キャストによる書き換えが未定義動作を引き起こす可能性があります（ソースコード上のコメントで示唆されています）。
  • 履歴リストが空の場合、ループは実行されません。

保守担当者向けメモ

  • 本関数は履歴をファイルへ永続化する直前に呼び出されることを想定しています。
  • decode_history と対になる処理です。改行のエンコード方式を変更する場合は、必ず両方の関数を更新してください。
