AsyncNotifyFreezeXids

定義

  • 種別: function
  • 定義箇所: src/backend/commands/async.c:2199

目的

非同期通知キュー内の古いトランザクションID（XID）を凍結（FrozenTransactionId）または無効化し、トランザクションIDの周回問題を防ぐためのクリーンアップ処理を行います。

シグネチャまたは形状

void AsyncNotifyFreezeXids(TransactionId newFrozenXid)

入力と出力

  • **入力**: newFrozenXid (凍結対象となる閾値のトランザクションID)
  • **出力**: なし（副作用として共有メモリ上の通知キューの内容を更新）

前提条件と不変条件

  • NotifyQueueTailLock および NotifyQueueLock の取得順序を厳守すること（デッドロック回避のため）。
  • 本関数実行中は NotifyQueueTailLock を保持し、キューのテール位置が移動しないことを保証する。

アルゴリズムまたは振る舞い

1. NotifyQueueTailLock (SHARED) および NotifyQueueLock (SHARED) を取得し、キューの現在位置（QUEUE_TAIL から QUEUE_HEAD まで）を特定する。 2. キューのテールからヘッドまでを順次スキャンする。 3. 各エントリのページ境界を跨ぐごとに、該当するSLRUバンクのロックを排他モードで取得し、ページを読み込む。 4. 各通知エントリの xid が newFrozenXid より古い場合：

  • コミット済みであれば FrozenTransactionId に置換する。
  • 未コミットであれば InvalidTransactionId に置換する。

    • 5. 変更があったページは page_dirty フラグを立て、SLRUの更新をマークする。 6. 最後にロックを解放し、処理を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/vacuum.c:1967 (VACUUM処理の一環として呼び出される)
  • **呼び出し先**: LWLockAcquire, LWLockRelease, SimpleLruGetBankLock, SimpleLruReadPage, TransactionIdIsNormal, TransactionIdPrecedes, TransactionIdDidCommit, asyncQueueAdvance

関連データ構造

  • NotifyCtl: 通知キューを管理するSLRU制御構造体。
  • AsyncQueueEntry: キュー内の個々の通知メッセージ。
  • QueuePosition: キュー内の位置情報。

失敗モードと境界条件

  • キューが空（QUEUE_POS_EQUAL(pos, head) が真）の場合、何も処理せず即座に終了する。
  • ページロック取得時に SimpleLruReadPage を使用しており、SLRUのバッファ管理に依存する。

保守担当者向けメモ

  • 本関数はVACUUMプロセスから呼び出されるため、パフォーマンスへの影響を考慮すること。
  • ロックの取得順序（TailLock -> QueueLock -> BankLock）は、他の非同期通知処理とのデッドロックを防ぐために極めて重要である。
