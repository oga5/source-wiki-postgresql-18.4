ExecGrant_Attribute

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/aclchk.c:1639

目的

特定のテーブル列（Attribute）に対する権限（ACL）の付与または取り消しを処理し、システムカタログ pg_attribute を更新する。

シグネチャまたは形状

static void ExecGrant_Attribute(InternalGrant *istmt, Oid relOid, const char *relname,
                                AttrNumber attnum, Oid ownerId, AclMode col_privileges,
                                Relation attRelation, const Acl *old_rel_acl)

入力と出力

  • **入力**:
  • istmt: 実行する GRANT/REVOKE 文の内部表現。
  • relOid, relname: 対象テーブルの OID と名称。
  • attnum: 対象列の番号。
  • ownerId: テーブル所有者の OID。
  • col_privileges: 適用する権限ビットマスク。
  • attRelation: 開かれている pg_attribute リレーション。
  • old_rel_acl: テーブルレベルの既存 ACL（列権限とのマージに使用）。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • attRelation は pg_attribute を指している必要がある。
  • SearchSysCache2(ATTNUM, ...) により対象列のタプルが取得可能であること。

アルゴリズムまたは振る舞い

1. **タプル取得**: ATTNUM システムキャッシュから対象列のタプルを取得。 2. **ACL 準備**: 既存の列 ACL を取得（存在しない場合は acldefault を使用）。 3. **権限マージ**: テーブルレベルの ACL と列レベルの ACL を連結し、select_best_grantor で権限付与者とオプションを決定。 4. **検証**: restrict_and_check_grant を呼び出し、権限付与の妥当性を確認。 5. **ACL 更新**: merge_acl_with_grant で新しい ACL を生成。 6. **カタログ更新**:

  • 新しい ACL が空でない場合は pg_attribute の attacl を更新。
  • 空の場合は attacl を NULL に設定。
  • CatalogTupleUpdate を実行し、拡張機能の初期権限および共有依存関係（pg_shdepend）を同期。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/aclchk.c 内の関数（ExecGrant_Relation 等から呼び出されることが示唆されている）。
  • **呼び出し先**: SearchSysCache2, SysCacheGetAttr, aclconcat, select_best_grantor, restrict_and_check_grant, merge_acl_with_grant, CatalogTupleUpdate, updateAclDependencies 等。

関連データ構造

  • InternalGrant: 権限操作のパラメータを保持。
  • pg_attribute: 列情報を保持するシステムカタログ。
  • Acl: アクセス制御リスト構造体。

失敗モードと境界条件

  • 対象列がシステムキャッシュに見つからない場合、elog(ERROR) を発生させる。
  • ACL が空になった場合、pg_attribute の attacl 列を NULL に更新する最適化を行う。

保守担当者向けメモ

  • 本関数は列レベルの権限変更に特化している。テーブルレベルの権限変更時に再帰的に呼び出される設計となっている。
  • pg_upgrade 等で利用される recordExtensionInitPriv を介して拡張機能の権限追跡を行っているため、変更時は依存関係の整合性に注意すること。
