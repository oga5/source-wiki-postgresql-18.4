RWConflictData

定義

  • **種別**: struct
  • **定義箇所**: src/include/storage/predicate_internals.h:193

目的

PostgreSQLのSSI（Serializable Snapshot Isolation）において、トランザクション間の読み書き依存関係（rw-conflict）を表現し、共有メモリ上でグラフ構造として保持するために使用されます。

シグネチャまたは形状

typedef struct RWConflictData
{
    dlist_node    outLink;        /* sxactからの出力競合リスト用リンク */
    dlist_node    inLink;         /* sxactへの入力競合リスト用リンク */
    SERIALIZABLEXACT *sxactOut;   /* 競合の発生元トランザクション */
    SERIALIZABLEXACT *sxactIn;    /* 競合の発生先トランザクション */
} RWConflictData;

入力と出力

  • **入力**: 競合する2つのトランザクション（SERIALIZABLEXACT）へのポインタ。
  • **出力**: 共有メモリ上のリスト（dlist）にリンクされた競合レコード。

前提条件と不変条件

  • dlist_node を使用しているため、共有メモリ上の dlist 管理ルールに従う必要があります。
  • sxactOut および sxactIn は、有効な SERIALIZABLEXACT 構造体を指している必要があります。

アルゴリズムまたは振る舞い

1. **生成**: トランザクション間のアクセス競合が検出された際に、RWConflictPool から割り当てられます。 2. **リンク**: outLink は sxactOut->outConflicts リストに、inLink は sxactIn->inConflicts リストにそれぞれ登録されます。 3. **追跡**: これにより、トランザクション間の依存関係グラフが構築され、直列化失敗の判定に利用されます。 4. **解放**: トランザクション終了時、関連する競合リストから取り外され、プールへ返却されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/lmgr/predicate.c 内の述語ロック管理ロジックが、競合の発生時に本構造体を操作します。
  • **呼び出し先**: 直接的な関数呼び出し先はありません（データ構造体のため）。

関連データ構造

  • SERIALIZABLEXACT: 競合に関与するトランザクション状態。
  • dlist_node: 共有メモリ上のリスト管理用ノード。
  • RWConflictPoolHeaderData: 競合レコードを管理するプール。

失敗モードと境界条件

  • **プール枯渇**: 共有メモリ上の RWConflictPool が空の場合、競合レコードの割り当てに失敗する可能性があります。
  • **メモリ整合性**: 共有メモリ上のデータ構造であるため、適切なロック（LWLock）なしでのアクセスはデータ破壊を招きます。

保守担当者向けメモ

  • dlist_container マクロを使用して、リストノードから RWConflictData 構造体を取得するパターンが predicate.c 内で多用されています。
  • 構造体のフィールド追加や変更を行う際は、predicate.c 内のリスト走査ロジックへの影響を考慮してください。
