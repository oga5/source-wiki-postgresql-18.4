swap_relation_files

定義

  • 種別: function
  • 定義箇所: src/backend/commands/cluster.c:1062

目的

2つのリレーション（テーブル）間で物理ファイル（relfilenode）および関連するメタデータを入れ替えます。主に CLUSTER や VACUUM FULL におけるテーブル再構築の最終段階で、新旧テーブルの物理ファイルを差し替えるために使用されます。

シグネチャまたは形状

static void swap_relation_files(Oid r1, Oid r2, bool target_is_pg_class,
                                bool swap_toast_by_content,
                                bool is_internal,
                                TransactionId frozenXid,
                                MultiXactId cutoffMulti,
                                Oid *mapped_tables)

入力と出力

  • **入力**:
  • r1, r2: 入れ替え対象のリレーションOID。
  • target_is_pg_class: 対象が pg_class 自体であるかを示すフラグ。
  • swap_toast_by_content: TOASTテーブルを内容に基づいて入れ替えるか（リンクの入れ替えか）。
  • frozenXid, cutoffMulti: 更新後の relfrozenxid および relminmxid。
  • **出力**:
  • mapped_tables: マッピングが更新されたテーブルのOIDを格納するポインタ。

前提条件と不変条件

  • pg_class の行に対する書き込みロック（RowExclusiveLock）が保持されていること。
  • マッピングされたリレーション（mapped_tables）の場合、テーブルスペース、永続性、アクセスメソッドの変更は許可されません。
  • 呼び出し元で適切な権限チェックが行われていること。

アルゴリズムまたは振る舞い

1. pg_class から対象リレーションのタプルをコピーし、relfilenode、reltablespace、relam、relpersistence などの属性を入れ替えます。 2. **非マッピングリレーションの場合**: pg_class タプルの値を直接書き換えます。 3. **マッピングリレーションの場合**: RelationMapUpdateMap を使用してマッピングを更新します。 4. RelationAssumeNewRelfilelocator を呼び出し、リレーションの新しいファイルロケータを認識させます。 5. relfrozenxid および relminmxid を更新します（インデックスを除く）。 6. 統計情報（ページ数、タプル数、可視性情報）を入れ替えます。

呼び出し元と呼び出し先

  • **呼び出し元**: rebuild_relation（src/backend/commands/cluster.c）など、テーブル再構築を行う関数群から呼び出されます。
  • **呼び出し先**: table_open, SearchSysCacheCopy1, RelationMapOidToFilenumber, RelationMapUpdateMap, RelationAssumeNewRelfilelocator 等。

関連データ構造

  • Form_pg_class: pg_class システムカタログの行構造。
  • RelFileNumber: 物理ファイル識別子。

失敗モードと境界条件

  • pg_class のキャッシュルックアップに失敗した場合、ERROR を発生させます。
  • マッピングされたリレーションに対して、許可されていない属性（テーブルスペース等）の変更を試みた場合、ERROR を発生させます。
  • マッピングが見つからない場合、ERROR を発生させます。

保守担当者向けメモ

  • 本関数はシステムカタログの整合性に直結するクリティカルな操作を行います。
  • 共有カタログやマッピングされたカタログに対する操作は、CommandCounterIncrement が実行されるまで反映されない点に注意してください。
  • pg_class 自体を入れ替える際は、統計情報の更新タイミングに注意が必要です。
