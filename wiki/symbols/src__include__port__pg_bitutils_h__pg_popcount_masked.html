pg_popcount_masked

定義

  • 種別: function
  • 定義箇所: src/include/port/pg_bitutils.h:393

目的

バッファ内の各バイトに対して特定のビットマスクを適用し、マスクされたビットが 1 である箇所の総数をカウントする。

シグネチャまたは形状

static inline uint64 pg_popcount_masked(const char *buf, int bytes, bits8 mask)

入力と出力

  • **入力**:
  • buf: カウント対象のデータバッファへのポインタ。
  • bytes: バッファのサイズ（バイト単位）。
  • mask: 各バイトに対して適用する 8 ビットのマスク。
  • **出力**:
  • uint64: マスクされたバッファ内の 1 のビット総数。

前提条件と不変条件

  • pg_number_of_ones テーブルが初期化されている必要がある。
  • bytes が負の値でないこと。

アルゴリズムまたは振る舞い

1. アーキテクチャのポインタサイズに基づき、閾値（64bit環境で8バイト、それ以外で4バイト）を決定する。 2. バッファサイズが閾値未満の場合:

  • ルックアップテーブル pg_number_of_ones を使用し、各バイトに対して (byte & mask) の計算結果をインデックスとして 1 の数を加算する。

    • 3. バッファサイズが閾値以上の場合:

  • pg_popcount_masked_optimized を呼び出し、ハードウェア命令等を利用した高速なカウント処理へ委譲する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/visibilitymap.c (複数箇所)
  • src/port/pg_bitutils.c
  • **呼び出し先**:
  • pg_popcount_masked_optimized (閾値以上の場合)

関連データ構造

  • pg_number_of_ones: 0-255 の各値に対する 1 のビット数を格納したルックアップテーブル。

失敗モードと境界条件

  • bytes が 0 の場合、ループは実行されず 0 を返す。
  • buf が NULL の場合、未定義動作となる可能性がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結している。
  • 閾値判定は関数呼び出しコストと命令実行コストのバランスを考慮しているため、変更時はパフォーマンスへの影響を測定すること。
