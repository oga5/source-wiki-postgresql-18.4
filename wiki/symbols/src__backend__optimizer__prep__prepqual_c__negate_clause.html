negate_clause

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepqual.c:72

目的

論理式（Qualification）を否定し、可能な限り NOT 演算子を排除して最適化器が扱いやすい形式へ変換します。

シグネチャまたは形状

Node *negate_clause(Node *node)

入力と出力

  • **入力**: 否定対象となる Node 構造体へのポインタ。
  • **出力**: 否定された論理式を表す新しい Node 構造体。

前提条件と不変条件

  • 入力 node が NULL であることは許容されません（エラーとなります）。
  • 入力式は既に簡略化されていることを前提としています（例: NOT の二重否定は単に解消される）。
  • 行型（rowtype）の NullTest は論理的な逆転ができないため、簡略化されません。

アルゴリズムまたは振る舞い

ノードの型に応じて以下の変換を行います。

  • **Const**: 定数値を反転させます（NOT NULL は NULL を維持）。
  • **OpExpr / ScalarArrayOpExpr**: get_negator を使用して演算子の反転が可能であれば、反転した演算子を持つ新しいノードを生成します。
  • **BoolExpr**: ド・モルガンの法則を適用します。
  • AND は OR に変換し、各子要素を再帰的に否定します。
  • OR は AND に変換し、各子要素を再帰的に否定します。
  • NOT は、その子要素をそのまま返して相殺します。
  • **NullTest**: スカラー型の場合、IS NULL と IS NOT NULL を相互に変換します。
  • **BooleanTest**: 各テストタイプ（IS TRUE, IS FALSE 等）を対応する否定形へ変換します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/prep/prepqual.c 内の再帰呼び出し。
  • src/backend/optimizer/util/clauses.c（定数畳み込みや式の簡略化プロセス）。
  • src/backend/optimizer/partitioning/partprune.c（パーティションプルーニング）。
  • **呼び出し先**:
  • get_negator（システムカタログ参照）。
  • make_andclause, make_orclause（ノード生成）。
  • negate_clause（自身を再帰的に呼び出し）。

関連データ構造

  • Node（汎用ノード）
  • OpExpr, ScalarArrayOpExpr, BoolExpr, NullTest, BooleanTest（各論理式ノード）

失敗モードと境界条件

  • 入力が NULL の場合、elog(ERROR) を発生させます。
  • 未知の boolop や booltesttype が渡された場合、elog(ERROR) を発生させます。
  • 行型（argisrow が真）の NullTest は変換されず、入力がそのまま返される可能性があります。

保守担当者向けメモ

  • 本関数は「正規形」への変換ではなく、最適化器が効率的に処理できる形式への整理を目的としています。
  • ド・モルガンの法則適用時、入力が既に平坦化されていれば、再帰呼び出し後も平坦な構造が維持されるため、追加の平坦化処理（pull_ors 等）は不要です。
