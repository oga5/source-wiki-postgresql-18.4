dotrim

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oracle_compat.c:394

目的

文字列の先頭および末尾から、指定された文字セットに含まれる文字を削除する共通実装を提供します。

シグネチャまたは形状

static text *dotrim(const char *string, int stringlen, const char *set, int setlen, bool doltrim, bool dortrim)

入力と出力

  • **入力**:
  • string: 対象文字列のポインタ。
  • stringlen: 対象文字列のバイト長。
  • set: 削除対象文字を含む文字列のポインタ。
  • setlen: 削除対象文字列のバイト長。
  • doltrim: 先頭をトリミングするかどうか。
  • dortrim: 末尾をトリミングするかどうか。
  • **出力**: トリミング後の文字列を格納した text 型へのポインタを返します。

前提条件と不変条件

  • stringlen または setlen が 0 の場合、トリミング処理は行われず、入力文字列がそのまま返されます。
  • マルチバイト文字環境では、pg_mblen_range を用いて文字境界を正しく判定します。

アルゴリズムまたは振る舞い

1. **エンコーディング判定**: データベースの最大文字長が 1 バイトを超える場合、マルチバイト対応の走査を行います。

  • 文字列およびセットを文字単位に分解し、ポインタと長さを配列に保持します。
  • doltrim が真の場合、先頭から順にセットに含まれるか判定し、一致する間ポインタを進めます。
  • dortrim が真の場合、末尾から順にセットに含まれるか判定し、一致する間長さを減らします。
  • 最後に一時メモリを解放します。

    • 2. **シングルバイト環境**: 1 バイト文字専用の最適化されたループにより、同様のトリミングを行います。 3. **結果生成**: 最終的な文字列のポインタと長さを cstring_to_text_with_len に渡し、text 型として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/oracle_compat.c 内の btrim, ltrim, rtrim 等の関数から呼び出されます。
  • **呼び出し先**: pg_database_encoding_max_length, pg_mblen_range, palloc, pfree, memcmp, cstring_to_text_with_len。

関連データ構造

  • text: PostgreSQL の可変長文字列型。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性があります。
  • マルチバイト文字の判定において、不正なエンコーディングが含まれる場合の挙動は pg_mblen_range の仕様に依存します。

保守担当者向けメモ

  • マルチバイト文字の走査において、stringchars 等の配列を動的に確保するため、文字列が非常に長い場合はメモリ消費に注意してください。
  • set が空の場合、ループはスキップされ、元の文字列が返されます。
