get_old_cluster_logical_slot_infos_query

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/info.c:651

目的

旧クラスタから論理レプリケーションスロットの情報を取得するための SQL クエリ文字列を生成する。

シグネチャまたは形状

char *get_old_cluster_logical_slot_infos_query(void)

入力と出力

  • **入力**: なし（グローバル変数 user_opts.live_check を参照）
  • **出力**: pg_catalog.pg_replication_slots からスロット名、プラグイン、二相コミットの可否、フェイルオーバーの可否、同期状態、無効化フラグを選択する SQL クエリ文字列。

前提条件と不変条件

  • 論理スロットのみを対象とする。
  • 一時的なスロット（temporary）は除外される。
  • 現在のデータベースに属するスロットのみを対象とする。

アルゴリズムまたは振る舞い

1. user_opts.live_check が真の場合、同期状態（caught_up）を FALSE としてクエリを生成する。 2. live_check が偽の場合、無効化されているスロットは FALSE とし、それ以外は binary_upgrade_logical_slot_has_caught_up() 関数を呼び出して同期状態を判定する。 3. psprintf を使用して動的にクエリ文字列を構築し、呼び出し元へ返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/info.c 内の get_db_rel_and_slot_infos 関数（310行目付近）。
  • **呼び出し先**: psprintf（文字列構築用）、pg_catalog.binary_upgrade_logical_slot_has_caught_up（SQL 実行時に評価される関数）。

関連データ構造

  • LogicalSlotInfo: 取得した情報を格納するために使用される（本関数自体はクエリ文字列のみを生成）。

失敗モードと境界条件

  • live_check 中は WAL の生成が続く可能性があるため、スロットが完全に同期しているかの保証はできない。
  • 無効化されたスロットについては、WAL が削除されている可能性があるため、同期チェックを意図的にスキップする。

保守担当者向けメモ

  • 本関数が生成するクエリは、アップグレードプロセス中にクラスタの起動・停止が繰り返されることを前提としており、一時的なスロットは自動的に削除されるため除外している。
  • 同期状態の判定ロジックは、PostgreSQL のバージョンアップに伴うアップグレード機能に依存している。
