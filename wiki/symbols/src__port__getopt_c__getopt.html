getopt

定義

  • 種別: function
  • 定義箇所: src/port/getopt.c:71

目的

POSIX 標準の getopt(3) 関数の移植用実装。プラットフォーム間で一貫したコマンドライン引数の解析を提供し、PostgreSQL のフロントエンドツールやユーティリティが OS 依存なくオプションを処理できるようにする。

シグネチャまたは形状

int getopt(int nargc, char *const *nargv, const char *ostr);

入力と出力

  • **入力**:
  • nargc: 引数の数。
  • nargv: 引数文字列の配列。
  • ostr: 許容するオプション文字のリスト（引数が必要な場合は : を後置）。
  • **出力**:
  • 成功時: 解析されたオプション文字（int）。
  • 引数終了時: -1。
  • エラー時: 不正なオプションの場合は BADCH、引数不足の場合は BADARG（または BADCH）。

前提条件と不変条件

  • 静的変数 place を使用するため、スレッドセーフではない。
  • optreset 変数は存在しない。解析を最初からやり直す場合は、呼び出し側で optind = 1 に設定する必要がある。
  • HAVE_INT_OPTERR マクロにより、OS が提供する getopt 変数と競合しないよう設計されている。

アルゴリズムまたは振る舞い

1. **走査**: 静的変数 place で現在の引数文字列内の位置を保持。optind が nargc に達するか、- で始まらない引数で終了。 2. **判定**: ostr を strchr で検索し、有効なオプションか確認。 3. **引数抽出**:

  • オプションに引数が必要な場合（ostr 内で : が続く場合）、place の残りまたは次の argv 要素を optarg に格納。
  • 不要な場合は optarg を NULL に設定。

    • 4. **状態更新**: optind を進め、解析したオプション文字を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/bootstrap/bootstrap.c
  • src/backend/postmaster/postmaster.c
  • src/backend/tcop/postgres.c
  • src/bin/pg_archivecleanup/pg_archivecleanup.c
  • src/bin/psql/startup.c
  • **呼び出し先**:
  • strchr (標準ライブラリ)
  • fprintf (標準ライブラリ)

関連データ構造

  • optarg: 現在のオプションに関連付けられた引数文字列へのポインタ。
  • optind: 次に処理すべき argv のインデックス。
  • optopt: 直近で処理されたオプション文字。
  • opterr: エラーメッセージ出力の制御フラグ。

失敗モードと境界条件

  • optind >= nargc または引数が - で始まらない場合、解析を終了し -1 を返す。
  • -- に遭遇した場合、解析を終了し -1 を返す。
  • ostr に含まれないオプション文字や、引数が必要なオプションで引数が欠落している場合、opterr が真であれば stderr にエラーを出力し、エラーコードを返す。

保守担当者向けメモ

  • 本実装は POSIX 互換レイヤーであり、GNU 形式のロングオプションが必要な場合は src/port/getopt_long.c を参照すること。
  • place 変数が静的であるため、同一プロセス内での複数回呼び出しや並列実行には注意が必要。
