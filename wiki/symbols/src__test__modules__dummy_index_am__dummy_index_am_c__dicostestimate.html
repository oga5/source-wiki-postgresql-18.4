dicostestimate

定義

  • 種別: function
  • 定義箇所: src/test/modules/dummy_index_am/dummy_index_am.c:201

目的

PostgreSQLのプランナに対し、このダミーインデックスの利用コストが極めて高いことを通知し、クエリ実行時にこのインデックスが選択されないように制御する。

シグネチャまたは形状

static void dicostestimate(PlannerInfo *root, IndexPath *path, double loop_count,
                           Cost *indexStartupCost, Cost *indexTotalCost,
                           Selectivity *indexSelectivity, double *indexCorrelation,
                           double *indexPages)

入力と出力

  • **入力**:
  • root, path, loop_count: プランナが提供するインデックスパス情報。
  • **出力**:
  • indexStartupCost, indexTotalCost: 1.0e10 に設定。
  • indexSelectivity: 1 に設定。
  • indexCorrelation: 0 に設定。
  • indexPages: 1 に設定。

前提条件と不変条件

  • 本関数は IndexAmRoutine 構造体の amcostestimate フィールドに割り当てられることを前提とする。
  • 戻り値としてコストを書き換える際、プランナがインデックスを選択しないよう、十分に大きな値（1.0e10）を割り当てる必要がある。

アルゴリズムまたは振る舞い

1. indexStartupCost および indexTotalCost に 1.0e10 を代入する。 2. indexSelectivity を 1、indexCorrelation を 0、indexPages を 1 に設定する。 3. 処理はこれら値を設定するのみであり、入力された root や path の内容は参照しない。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのプランナ（クエリ実行計画作成時）。
  • **呼び出し先**: なし。
  • **直接的な証拠**: src/test/modules/dummy_index_am/dummy_index_am.c 内の amroutine->amcostestimate = dicostestimate; により、AMルーチンの一部として登録されている。

関連データ構造

  • PlannerInfo: プランナの情報を保持する構造体。
  • IndexPath: インデックスパスの情報を保持する構造体。
  • IndexAmRoutine: インデックスアクセスメソッドの関数ポインタテーブル。

失敗モードと境界条件

  • 本関数は常に固定値を返すため、実行時エラーや失敗モードは存在しない。
  • loop_count 等の入力値に関わらず、常に同一のコストを返す。

保守担当者向けメモ

  • 本関数はテスト用のスタブ実装である。実用的なインデックスを実装する際は、実際のコスト計算ロジックに置き換える必要がある。
  • コスト値を変更する場合は、プランナが意図せずこのインデックスを選択しないよう注意すること。
