pg_range_sockaddr

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/ifaddr.c:48

目的

指定されたIPアドレス（addr）が、ネットワークアドレス（netaddr）とサブネットマスク（netmask）で定義される範囲内に含まれるかを判定します。

シグネチャまたは形状

int pg_range_sockaddr(const struct sockaddr_storage *addr,
                      const struct sockaddr_storage *netaddr,
                      const struct sockaddr_storage *netmask);

入力と出力

  • **入力**:
  • addr: 判定対象のアドレス。
  • netaddr: ネットワークのベースアドレス。
  • netmask: ネットワークのサブネットマスク。
  • **出力**:
  • 範囲内であれば非ゼロ（真）、範囲外または未対応のアドレスファミリーであれば 0（偽）を返します。

前提条件と不変条件

  • addr、netaddr、netmask は有効な sockaddr_storage 構造体を指している必要があります。
  • アドレスファミリー（ss_family）が AF_INET または AF_INET6 であることが期待されます。

アルゴリズムまたは振る舞い

1. addr->ss_family を確認します。 2. AF_INET の場合、引数を sockaddr_in にキャストし、range_sockaddr_AF_INET を呼び出します。 3. AF_INET6 の場合、引数を sockaddr_in6 にキャストし、range_sockaddr_AF_INET6 を呼び出します。 4. 上記以外のファミリーの場合は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/hba.c:1172（クライアント接続元のIP許可判定に使用）。
  • **呼び出し先**: range_sockaddr_AF_INET および range_sockaddr_AF_INET6。

関連データ構造

  • struct sockaddr_storage: 汎用的なソケットアドレス構造体。
  • struct sockaddr_in: IPv4用ソケットアドレス構造体。
  • struct sockaddr_in6: IPv6用ソケットアドレス構造体。

失敗モードと境界条件

  • AF_INET および AF_INET6 以外のファミリーが渡された場合、判定は行われず 0 が返されます。
  • 引数として渡される構造体の内容が不整合（例: addr と netaddr でファミリーが異なる等）である場合の挙動は、委譲先の関数に依存します。

保守担当者向けメモ

  • 本関数はOS依存のネットワーク処理を抽象化するレイヤーの一部です。
  • IPv4/IPv6の判定ロジックは ss_family に依存しているため、新しいアドレスファミリーをサポートする場合は本関数の条件分岐を拡張する必要があります。
