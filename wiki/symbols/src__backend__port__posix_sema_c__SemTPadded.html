SemTPadded

定義

  • **種別**: typedef (共用体)
  • **定義箇所**: src/backend/port/posix_sema.c:45

目的

POSIX セマフォ（sem_t）を CPU キャッシュラインサイズに合わせてパディングし、共有メモリ上での配置時に偽の共有（false sharing）を防止するためのラッパー構造体です。

シグネチャまたは形状

typedef union SemTPadded
{
    sem_t       pgsem;
    char        pad[PG_CACHE_LINE_SIZE];
} SemTPadded;

入力と出力

  • **入力**: なし（データ構造の定義）
  • **出力**: なし

前提条件と不変条件

  • PG_CACHE_LINE_SIZE が定義されている環境で、セマフォのサイズがキャッシュラインサイズ以下であることを前提としています。
  • 共用体として定義されており、pgsem メンバと pad メンバはメモリ領域を共有します。

アルゴリズムまたは振る舞い

  • sem_t 型のセマフォを保持しつつ、構造体全体が PG_CACHE_LINE_SIZE のサイズを占有するように強制します。これにより、配列として確保されたセマフォ群が隣接するキャッシュラインにまたがらないよう配置を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/port/posix_sema.c:54 にて SemTPadded sem_padded; としてインスタンス化されています。
  • **呼び出し先**: 構造体定義のため、直接的な関数呼び出しはありません。

関連データ構造

  • sem_t: POSIX セマフォの基底型。
  • PGSemaphoreData: 本モジュールにおけるセマフォの抽象データ構造。

失敗モードと境界条件

  • PG_CACHE_LINE_SIZE が sem_t のサイズよりも小さい場合、コンパイルエラーまたは意図しないメモリレイアウトになる可能性があります。

保守担当者向けメモ

  • 本構造体は、マルチプロセッサ環境におけるキャッシュラインの競合を避けるための最適化です。セマフォのサイズやキャッシュラインの定義が変更された場合は、パディングの整合性を確認してください。
