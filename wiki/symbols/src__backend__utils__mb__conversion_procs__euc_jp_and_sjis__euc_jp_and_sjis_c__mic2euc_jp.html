mic2euc_jp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_jp_and_sjis/euc_jp_and_sjis.c:469

目的

MULE_INTERNAL (MIC) 形式の文字列を EUC_JP 形式に変換する。

シグネチャまたは形状

static int mic2euc_jp(const unsigned char *mic, unsigned char *p, int len, bool noError)

入力と出力

  • **入力**:
  • mic: 変換元の MULE_INTERNAL 文字列へのポインタ。
  • p: 変換後の EUC_JP 文字列を格納するバッファへのポインタ。
  • len: 変換対象のバイト長。
  • noError: 変換不能な文字や不正なバイト列に遭遇した際、エラーを報告せずに処理を中断するかどうかを指定するフラグ。
  • **出力**:
  • 戻り値: 処理した入力バイト数。
  • p が指すバッファには、変換された EUC_JP 文字列が格納され、末尾にヌル文字 (\0) が付与される。

前提条件と不変条件

  • len が 0 より大きい間、ループ処理が継続される。
  • 変換後のバッファ p は、十分なサイズが確保されている必要がある。

アルゴリズムまたは振る舞い

1. 入力文字列を1文字ずつ走査する。 2. **ASCII文字**: IS_HIGHBIT_SET が偽の場合、そのまま p にコピーする。 3. **マルチバイト文字**: pg_encoding_verifymbchar で妥当性を検証する。

  • LC_JISX0201K (半角カナ): SS2 を付与し、後続の1バイトをコピーする。
  • LC_JISX0212 (補助漢字): SS3 を付与し、後続の2バイトをコピーする。
  • LC_JISX0208 (JIS X 0208): 後続の2バイトをそのままコピーする。

    • 4. 変換不能な文字や不正なバイト列の場合、noError が偽であればエラーを報告し、真であれば処理を中断する。 5. 最後に p の末尾にヌル文字を書き込み、処理した総バイト数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • 同一ファイル内の mic_to_euc_jp 関数（src/backend/utils/mb/conversion_procs/euc_jp_and_sjis/euc_jp_and_sjis.c:122）。
  • **呼び出し先**:
  • pg_encoding_verifymbchar
  • report_invalid_encoding
  • report_untranslatable_char

関連データ構造

  • PG_MULE_INTERNAL (エンコーディング識別子)
  • PG_EUC_JP (エンコーディング識別子)
  • SS2, SS3 (EUC_JP 制御文字)
  • LC_JISX0201K, LC_JISX0212, LC_JISX0208 (MULE_INTERNAL 文字コード識別子)

失敗モードと境界条件

  • 入力文字列が不正なバイト列である場合、noError が偽であれば report_invalid_encoding が呼び出される。
  • 変換対象外の文字コードが含まれている場合、noError が偽であれば report_untranslatable_char が呼び出される。
  • noError が真の場合、エラー発生時に処理を中断し、それまでに変換済みのバイト数を返す。

保守担当者向けメモ

  • 本関数は内部的な変換ロジックであり、PostgreSQLの関数マネージャから直接呼び出されるインターフェース（mic_to_euc_jp 等）から利用される。
  • 文字コードの追加やマッピング変更時は、LC_ 定数と EUC_JP への変換規則の整合性に注意すること。
