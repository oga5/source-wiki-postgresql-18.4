AllocDeadEndChild

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/pmchild.c:207

目的

接続失敗時など、制限のない「デッドエンド（dead-end）」バックエンドプロセスを追跡するための PMChild 構造体を動的に割り当て、管理リストに追加します。

シグネチャまたは形状

PMChild *AllocDeadEndChild(void)

入力と出力

  • **入力**: なし
  • **出力**: 成功時は初期化済みの PMChild 構造体へのポインタ、失敗時は NULL を返します。

前提条件と不変条件

  • palloc_extended を使用してメモリを確保します。
  • 割り当てられた PMChild は、bkend_type が B_DEAD_END_BACKEND に設定され、ActiveChildList に登録されます。

アルゴリズムまたは振る舞い

1. DEBUG2 レベルでログを出力します。 2. palloc_extended を用いて PMChild サイズのメモリを MCXT_ALLOC_NO_OOM フラグ付きで確保します。 3. メモリ確保に成功した場合、以下の初期化を行います。

  • pid を 0 に設定。
  • child_slot を 0 に設定。
  • bkend_type を B_DEAD_END_BACKEND に設定。
  • rw を NULL に設定。
  • bgworker_notify を false に設定。

    • 4. 構造体を ActiveChildList の先頭に挿入します。 5. 確保した構造体ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/postmaster.c:3552
  • **呼び出し先**: palloc_extended, dlist_push_head

関連データ構造

  • PMChild: 子プロセスのメタデータを保持する構造体。
  • ActiveChildList: 現在稼働中の全子プロセスを保持する双方向リスト。

失敗モードと境界条件

  • メモリ不足により palloc_extended が NULL を返した場合、関数はそのまま NULL を返します。このときリストへの追加は行われません。

保守担当者向けメモ

  • 本関数で割り当てられた構造体は、通常のプール管理（AssignPostmasterChildSlot）とは異なり、動的にメモリ確保される点に注意してください。
  • 終了時には適切にメモリ解放を行う必要があります。
