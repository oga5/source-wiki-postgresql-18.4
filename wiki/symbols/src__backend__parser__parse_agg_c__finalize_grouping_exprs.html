finalize_grouping_exprs

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_agg.c:1595

目的

クエリ内の式に含まれるグループ化関連の要素を最終処理し、必要に応じて列参照をグループ化された変数（RTE_GROUP）へ置換するためのコンテキストを初期化・実行します。

シグネチャまたは形状

static void finalize_grouping_exprs(Node *node, ParseState *pstate, Query *qry,
                                    List *groupClauses, bool hasJoinRTEs,
                                    bool have_non_var_grouping)

入力と出力

  • **入力**:
  • node: 処理対象の式ツリー（Node）。
  • pstate: 現在のパーサ状態（ParseState）。
  • qry: 現在のクエリ構造（Query）。
  • groupClauses: グループ化句のリスト（List）。
  • hasJoinRTEs: JOIN 式が含まれるかを示すフラグ。
  • have_non_var_grouping: 変数以外のグループ化要素が存在するかを示すフラグ。
  • **出力**: なし（副作用として node ツリー内のノードを書き換える）。

前提条件と不変条件

  • substitute_grouped_columns_context を使用して、再帰的なツリー走査を行うための状態を初期化します。
  • sublevels_up は 0 に初期化され、現在のクエリレベルでの処理を前提とします。

アルゴリズムまたは振る舞い

1. substitute_grouped_columns_context 構造体をローカルで宣言し、引数で渡された各パラメータをフィールドに割り当てます。 2. groupClauseCommonVars や gset_common などの内部リストを NIL で初期化します。 3. finalize_grouping_exprs_walker を呼び出し、node をルートとしてツリーを走査・変換します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_agg.c 内の複数箇所（1300行目、1313行目など）から呼び出されています。
  • **呼び出し先**: finalize_grouping_exprs_walker を呼び出します。

関連データ構造

  • substitute_grouped_columns_context: 変換処理中に保持される状態管理用構造体。
  • Query, ParseState, Node: PostgreSQL のパーサが使用する標準的なノード構造。

失敗モードと境界条件

  • ツリーの走査中に不正なノード構造や、解決不可能なグループ化式に遭遇した場合の挙動は、呼び出し先の finalize_grouping_exprs_walker の実装に依存します。

保守担当者向けメモ

  • 本関数は substitute_grouped_columns と密接に関連しています。sublevels_up の管理や RTE_GROUP への置換ロジックを変更する際は、コンテキストの初期化状態を確認してください。
  • finalize_grouping_exprs_walker がどのようにツリーを書き換えるかについては、当該関数の定義を参照してください。
