has_drive_prefix

定義

  • 種別: function
  • 定義箇所: src/port/path.c:93

目的

与えられたファイルパス文字列が、Windows におけるドライブレター（例: C:）で始まるかどうかを判定します。

シグネチャまたは形状

bool has_drive_prefix(const char *path);

入力と出力

  • **入力**: path (ヌル終端文字列へのポインタ)
  • **出力**:
  • true: パスがドライブレターで始まる場合。
  • false: パスがドライブレターで始まらない場合、または非 Windows 環境の場合。

前提条件と不変条件

  • path は有効なヌル終端文字列である必要があります。
  • Windows 環境以外では、常に false を返します。

アルゴリズムまたは振る舞い

  • Windows 環境 (WIN32 定義時): skip_drive(path) を呼び出し、戻り値が入力ポインタ path と一致するかを比較します。一致しない場合はドライブプレフィックスが存在すると判断します。
  • それ以外の環境: 常に false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c:4939 にて、パスが絶対パスかドライブプレフィックスを持つかを確認するために使用されています。
  • **呼び出し先**: Windows 環境では skip_drive を呼び出します。

関連データ構造

  • なし。

失敗モードと境界条件

  • path が NULL の場合の挙動は、skip_drive の実装に依存します。

保守担当者向けメモ

  • 本関数は src/include/port.h で宣言されています。
  • Windows 特有のパス形式を扱うための抽象化レイヤーの一部であり、プラットフォーム間のパス操作の一貫性を保つために使用されます。
