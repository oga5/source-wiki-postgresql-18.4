json_send

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/json.c:137

目的

json 型のデータを、PostgreSQLのフロントエンド/バックエンドプロトコルにおけるバイナリ形式（bytea）へ変換して送信するために使用されます。

シグネチャまたは形状

Datum json_send(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される json 型のデータ（text 型として取得）。
  • **出力**: bytea 型のバイナリデータ（Datum として返却）。

前提条件と不変条件

  • 入力データは有効な json 型の text データである必要があります。
  • pq_begintypsend および pq_endtypsend を使用したプロトコル準拠のバッファ操作が前提となります。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP(0) を呼び出し、入力された json データを text ポインタとして取得します。 2. pq_begintypsend(&buf) を呼び出し、送信用の StringInfo バッファを初期化します。 3. pq_sendtext(&buf, ...) を使用して、text データの本体（VARDATA_ANY）とサイズ（VARSIZE_ANY_EXHDR）をバッファにコピーします。 4. pq_endtypsend(&buf) を呼び出し、構築されたバッファを bytea 型に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（SQLインターフェース経由）。
  • **呼び出し先**:
  • pq_begintypsend
  • pq_sendtext
  • pq_endtypsend
  • PG_GETARG_TEXT_PP
  • PG_RETURN_BYTEA_P

    • (いずれも libpq/pqformat.h 等で定義されたプロトコル補助関数)

関連データ構造

  • StringInfoData: 送信データを一時的に保持するためのバッファ構造体。
  • text: PostgreSQLの可変長文字列型。

失敗モードと境界条件

  • 入力データが NULL の場合、PG_GETARG_TEXT_PP の仕様に従い例外が発生する可能性があります。
  • メモリ不足時には、バッファ確保時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は json 型のバイナリ転送を担う標準的な実装です。プロトコル形式の変更がない限り、修正の必要性は低いです。
  • jsonb 型のバイナリ送信（jsonb_send）とは実装が異なるため、混同しないよう注意してください。
