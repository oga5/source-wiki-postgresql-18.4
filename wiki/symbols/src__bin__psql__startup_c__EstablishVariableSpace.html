EstablishVariableSpace

定義

  • 種別: function
  • 定義箇所: src/bin/psql/startup.c:1217

目的

psql の内部変数空間を初期化し、各変数に対する置換フックおよび変更通知フックを登録することで、変数と内部状態の同期メカニズムを構築する。

シグネチャまたは形状

static void EstablishVariableSpace(void)

入力と出力

  • 入力: なし（グローバル変数 pset を参照）
  • 出力: なし（副作用として pset.vars を初期化し、フックを登録）

前提条件と不変条件

  • pset 構造体が初期化されていること。
  • CreateVariableSpace() が有効な変数空間を返すこと。

アルゴリズムまたは振る舞い

1. CreateVariableSpace() を呼び出し、pset.vars に新しい変数空間を割り当てる。 2. SetVariableHooks() を使用して、AUTOCOMMIT, ON_ERROR_STOP, PROMPT1 等の psql 制御用変数に対し、以下のフックを順次登録する。

  • **置換フック**: 変数設定時に値を検証・変換する関数（例: bool_substitute_hook）。
  • **変更通知フック**: 変数変更時に psql の内部状態を更新する関数（例: autocommit_hook）。

    • 3. 登録対象の変数は、動作制御（SINGLESTEP 等）、履歴管理（HISTFILE 等）、プロンプト設定（PROMPT1 等）など多岐にわたる。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/startup.c 内の main() 関数（初期化フェーズ）。
  • 呼び出し先: CreateVariableSpace() および SetVariableHooks()。

関連データ構造

  • pset: PsqlSettings 型のグローバル変数。vars メンバを通じて変数空間を保持する。
  • VariableSpace: psql の変数を管理する不透明なデータ構造。

失敗モードと境界条件

  • メモリ不足により CreateVariableSpace() が失敗した場合、プログラムは異常終了する可能性がある。
  • 存在しない変数名や無効なフック関数を登録しようとした場合の挙動は、SetVariableHooks の実装に依存する。

保守担当者向けメモ

  • 新しい設定項目を psql に追加する際は、必ず本関数内で SetVariableHooks を呼び出し、適切なフック関数を登録すること。
  • フック関数は src/bin/psql/variables.c 等で定義されているロジックと密接に関連しているため、変更時は影響範囲を確認すること。
