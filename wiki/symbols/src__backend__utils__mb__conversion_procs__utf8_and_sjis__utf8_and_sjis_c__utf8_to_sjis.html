utf8_to_sjis

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_sjis/utf8_and_sjis.c:62

目的

UTF-8エンコーディングの文字列をSJIS（Shift-JIS）エンコーディングに変換する、PostgreSQLの動的ロード可能な変換プロシージャです。

シグネチャまたは形状

Datum utf8_to_sjis(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の引数を受け取ります。
  • src (CSTRING): 変換元のUTF-8文字列。
  • dest (CSTRING): 変換後のSJIS文字列を格納するバッファ。
  • len (INT32): 変換対象のバイト長。
  • noError (BOOL): 変換エラー発生時の挙動を制御するフラグ。
  • **出力**: 変換されたバイト数を Datum 型（INT32）として返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_UTF8, PG_SJIS) マクロにより、ソースエンコーディングがUTF-8、ターゲットエンコーディングがSJISであることを検証します。

アルゴリズムまたは振る舞い

1. UtfToLocal 関数を呼び出し、変換処理を実行します。 2. 変換には sjis_from_unicode_tree（utf8_to_sjis.map 由来の静的データ）を参照します。 3. noError フラグを UtfToLocal に伝播させ、エラーハンドリングを委譲します。 4. 変換されたバイト数を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの変換エンジン（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: UtfToLocal（src/backend/utils/mb/wchar.c で定義される汎用変換エンジン）。

関連データ構造

  • sjis_from_unicode_tree: UTF-8からSJISへの変換に使用されるツリー構造の変換テーブル。

失敗モードと境界条件

  • エンコーディングの不一致が発生した場合、CHECK_ENCODING_CONVERSION_ARGS によりエラーとなります。
  • 変換エラー時の挙動は noError 引数の値に依存します。

保守担当者向けメモ

  • 本関数はラッパーであり、実際の変換ロジックは UtfToLocal および sjis_from_unicode_tree に依存しています。
  • 変換テーブルの修正が必要な場合は、直接ソースコードを編集せず、../../Unicode/utf8_to_sjis.map を更新してください。
