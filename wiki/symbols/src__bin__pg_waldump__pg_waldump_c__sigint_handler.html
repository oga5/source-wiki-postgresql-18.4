sigint_handler

定義

  • 種別: function
  • 定義箇所: src/bin/pg_waldump/pg_waldump.c:90

目的

ユーザーからの割り込み（SIGINT）を検知し、pg_waldump のメイン処理ループを安全に停止させるためのシグナルハンドラです。

シグネチャまたは形状

static void sigint_handler(SIGNAL_ARGS)

※ SIGNAL_ARGS は PostgreSQL のシグナルハンドラ用マクロです。

入力と出力

  • 入力: SIGNAL_ARGS（シグナル発生時の標準引数）
  • 出力: なし（戻り値 void）

前提条件と不変条件

  • グローバル変数 time_to_stop が定義されている必要があります。
  • 本関数はシグナルハンドラとして呼び出されるため、非同期シグナルセーフな操作のみを行うことが期待されます。

アルゴリズムまたは振る舞い

1. 呼び出されると、グローバル変数 time_to_stop を true に設定します。 2. これにより、メインループ内の処理が終了条件を検知し、WAL の読み込み処理を停止します。

呼び出し元と呼び出し先

  • 呼び出し元: OS（SIGINT シグナル受信時）
  • 呼び出し先: なし
  • 登録箇所: src/bin/pg_waldump/pg_waldump.c:831 にて pqsignal(SIGINT, sigint_handler) により登録されています。

関連データ構造

  • time_to_stop: 処理の継続可否を制御するグローバルなフラグ変数。

失敗モードと境界条件

  • 本関数自体は単純な代入操作であるため、関数内部での失敗モードはありません。
  • シグナルが連続して発生した場合でも、time_to_stop を繰り返し true に設定するのみであり、副作用はありません。

保守担当者向けメモ

  • 本関数は pg_waldump の終了処理のトリガーです。time_to_stop を参照しているメインループ側のロジックに変更を加える際は、本ハンドラとの整合性に注意してください。
