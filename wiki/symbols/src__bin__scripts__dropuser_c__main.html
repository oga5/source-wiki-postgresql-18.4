目的

PostgreSQL のロールを削除するためのコマンドラインユーティリティ dropuser のエントリポイントとして、引数解析、接続確立、および DROP ROLE SQL 文の実行を制御します。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。接続オプション（ホスト、ポート、ユーザー名等）、--interactive、--if-exists、および削除対象のロール名を受け付けます。
  • **出力**:
  • 正常終了時は 0 を返します。
  • 引数エラーや SQL 実行失敗時は 1 を返して終了します。

前提条件と不変条件

  • pg_logging_init およびロケール設定が初期化されていること。
  • 削除対象のロール名が引数として提供されるか、対話モードで入力されること。
  • 接続先データベースへの接続が connectMaintenanceDatabase により確立可能であること。

アルゴリズムまたは振る舞い

1. **初期化**: ログとロケールの設定を行い、getopt_long でコマンドラインオプションを解析します。 2. **引数検証**: 削除対象のロール名を確認します。指定がない場合は対話モード（--interactive）であれば入力を促し、そうでなければエラー終了します。 3. **確認**: --interactive 指定時は、ユーザーに削除の意思確認を行い、否定された場合は終了します。 4. **接続**: connectMaintenanceDatabase を使用してサーバーへ接続します。 5. **SQL 生成**: PQExpBuffer を用いて DROP ROLE [IF EXISTS] <name>; を構築します。ロール名は fmtIdEnc でエスケープされます。 6. **実行**: PQexec で SQL を送信し、結果を確認します。成功すれば接続を閉じて終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（プログラムのエントリポイント）。
  • **呼び出し先**:
  • getopt_long, pg_logging_init, get_progname, set_pglocale_pgservice
  • handle_help_version_opts, simple_prompt, yesno_prompt
  • connectMaintenanceDatabase
  • initPQExpBuffer, appendPQExpBuffer, fmtIdEnc
  • PQexec, PQresultStatus, PQerrorMessage, PQclear, PQfinish
  • ※ツールが提示した contrib/amcheck 等の呼び出し元情報は、本シンボルとは無関係な「main」という名称の偶然の一致であり、本関数とは無関係です。

関連データ構造

  • struct option: コマンドライン引数の定義に使用。
  • ConnParams: 接続パラメータを保持。
  • PQExpBufferData: SQL 文構築用バッファ。
  • PGconn, PGresult: libpq の接続および結果オブジェクト。

失敗モードと境界条件

  • **引数エラー**: 不正なオプションや過剰な引数が指定された場合、エラーメッセージを表示して終了します。
  • **ロール名欠落**: ロール名が指定されず、対話モードでもない場合はエラー終了します。
  • **接続失敗**: データベース接続に失敗した場合、connectMaintenanceDatabase 内で処理されます。
  • **SQL 実行失敗**: DROP ROLE が失敗した場合（例：ロールが存在しない、権限不足）、エラーメッセージを表示して終了します。

保守担当者向けメモ

  • fmtIdEnc による識別子のエスケープ処理は SQL インジェクション対策として必須です。
  • if_exists 変数は静的変数として定義されており、--if-exists オプションのフラグとして機能します。
  • 本ツールは DROP USER と同義の DROP ROLE を実行します。
