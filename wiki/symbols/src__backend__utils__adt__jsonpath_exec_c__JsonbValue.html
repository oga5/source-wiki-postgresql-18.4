JsonbValue

定義

  • 種別: typedef
  • 定義箇所: src/backend/utils/adt/jsonpath_exec.c:89

目的

jsonb データ型の各ノード（スカラー値、オブジェクト、配列など）を表現するための内部構造体です。

シグネチャまたは形状

ソースコード上の定義詳細は本抜粋に含まれていませんが、jsonb の各要素を保持するコンテナとして機能します。

入力と出力

  • 入力: jsonb データの解析過程で生成される各要素。
  • 出力: jsonpath 実行エンジンにおけるマッチング結果や中間計算値として利用されます。

前提条件と不変条件

  • jsonb 形式のデータ構造と整合性を保つ必要があります。
  • 実行エンジン内でのメモリ管理は、PostgreSQL のメモリコンテキストに依存します。

アルゴリズムまたは振る舞い

  • jsonpath の実行時、executeItem 等の関数を通じて、JSONB の各要素を走査・抽出・評価するために使用されます。
  • lax モードでは、配列の自動アンラップ処理において、個々の要素を JsonbValue として取り扱う振る舞いが見られます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/jsonb_plperl や contrib/hstore 等の拡張モジュール、および jsonpath_exec.c 内の各実行関数。
  • 呼び出し先: 本シンボル自体はデータ構造であるため、直接的な呼び出し先はありません。操作関数（jsonb 関連の API）が本構造体を引数として受け取ります。

関連データ構造

  • JsonValueList: マッチした JsonbValue のリストを管理する構造体。
  • JsonPathExecContext: 実行時の状態を保持するコンテキスト。

失敗モードと境界条件

  • メモリ不足時のエラーハンドリングは PostgreSQL の標準的なエラーメカニズムに依存します。
  • 不正な JSON 構造や型不整合が発生した場合、jsonpath 実行エンジンがエラーをスローします。

保守担当者向けメモ

  • 本構造体は jsonb 型の操作において極めて頻繁に使用されます。変更を加える際は、src/backend/utils/adt/jsonb.c および jsonpath_exec.c への影響を十分に考慮してください。
  • 構造体の定義詳細については、src/backend/utils/adt/jsonb.h を参照してください。
