CompareOpclassOptions

定義

  • 種別: function
  • 定義箇所: src/backend/commands/indexcmds.c:361

目的

2つの演算子クラスオプション（Datum 配列）が等価であるかを判定します。

シグネチャまたは形状

static bool CompareOpclassOptions(const Datum *opts1, const Datum *opts2, int natts)

入力と出力

  • **入力**:
  • opts1, opts2: 比較対象の Datum 配列。
  • natts: 比較する属性の数。
  • **出力**:
  • 両者が等価であれば true、そうでなければ false を返します。

前提条件と不変条件

  • opts1 と opts2 の両方が NULL の場合は true を返します。
  • オプションの比較には F_ARRAY_EQ 関数を使用し、照合順序には C_COLLATION_OID を強制します。これは演算子クラスオプションのセマンティクスが不明であるため、バイナリレベルでの等価性を保証するためです。

アルゴリズムまたは振る舞い

1. 両方のポインタが NULL かを確認し、真であれば即座に true を返します。 2. natts 分だけループし、各インデックスの Datum を比較します。 3. 片方が 0（NULL相当）で、もう片方が非 0 の場合は false を返します。 4. 両方が非 0 の場合、FunctionCall2Coll を用いて C ロケールで比較を行い、結果が false であれば不一致とみなします。 5. 全ての属性で一致が確認できれば true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/indexcmds.c 内の CheckIndexCompatible から呼び出されます。
  • **呼び出し先**: fmgr_info および FunctionCall2Coll を使用して、PostgreSQL の関数管理システムを介した比較を実行します。

関連データ構造

  • Datum: PostgreSQL の汎用データ型表現。
  • FmgrInfo: 関数呼び出し情報を保持する構造体。

失敗モードと境界条件

  • natts が 0 の場合、ループは実行されず true を返します。
  • opts1 または opts2 のいずれか一方が NULL で、もう一方が非 NULL の場合、比較ロジックにより false が返されます。

保守担当者向けメモ

  • 本関数は演算子クラスオプションのバイナリ比較に特化しています。将来的にオプションのセマンティクスが定義された場合、照合順序の扱いや比較ロジックの変更が必要になる可能性があります。
