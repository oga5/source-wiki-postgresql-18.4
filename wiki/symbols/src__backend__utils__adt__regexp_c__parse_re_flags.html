parse_re_flags

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/regexp.c:384

目的

SQLの正規表現関数（regexp_match 等）に渡されるフラグ文字列を解析し、POSIX正規表現エンジンが解釈可能な cflags（コンパイルフラグ）およびグローバルマッチングフラグへ変換します。

シグネチャまたは形状

void parse_re_flags(pg_re_flags *flags, text *opts)

入力と出力

  • **入力**:
  • flags: 変換結果を格納する pg_re_flags 構造体へのポインタ。
  • opts: フラグ文字を含む text 型の文字列（例: 'gi'）。NULLの場合はデフォルト値が適用されます。
  • **出力**: なし（flags 引数を直接更新します）。

前提条件と不変条件

  • デフォルト状態として、flags->cflags には REG_ADVANCED が設定され、flags->glob は false に初期化されます。
  • opts が提供された場合、その各文字を順次走査して flags を更新します。

アルゴリズムまたは振る舞い

1. 初期化: cflags を REG_ADVANCED に設定し、glob を false に設定します。 2. 文字列走査: opts 内の各文字に対して以下のフラグを適用します。

  • g: glob を true に設定。
  • b: REG_ADVANCED, REG_EXTENDED, REG_QUOTE を解除。
  • c: REG_ICASE を解除。
  • e: REG_EXTENDED を追加し、REG_ADVANCED, REG_QUOTE を解除。
  • i: REG_ICASE を追加。
  • m/n: REG_NEWLINE を追加。
  • p: REG_NLSTOP を追加し、REG_NLANCH を解除。
  • q: REG_QUOTE を追加し、REG_ADVANCED, REG_EXTENDED を解除。
  • s: REG_NEWLINE を解除。
  • t: REG_EXPANDED を解除。
  • w: REG_NLSTOP を解除し、REG_NLANCH を追加。
  • x: REG_EXPANDED を追加。

    • 3. エラー処理: 定義されていない文字が渡された場合、ereport(ERROR, ...) を呼び出し処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**: regexp_match, regexp_matches, regexp_split_to_table, regexp_split_to_array, regexp_replace 等のSQL関数実装（src/backend/utils/adt/regexp.c 内の各所）。
  • **呼び出し先**: ereport（エラー発生時）。

関連データ構造

  • pg_re_flags: 正規表現のコンパイルフラグとグローバルマッチングフラグを保持する構造体。

失敗モードと境界条件

  • 不正なフラグ文字が入力された場合、ERRCODE_INVALID_PARAMETER_VALUE を伴うエラーを発生させます。

保守担当者向けメモ

  • 本関数は正規表現エンジンのフラグをSQLインターフェースに適合させる重要な仲介役です。新しいフラグを追加する場合は、POSIX正規表現ライブラリの仕様と整合性を取る必要があります。
  • opts がNULLの場合の挙動はデフォルト値の適用のみであり、安全に処理されます。
