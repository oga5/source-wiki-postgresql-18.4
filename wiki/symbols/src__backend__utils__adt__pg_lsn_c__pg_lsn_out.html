pg_lsn_out

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_lsn.c:79

目的

pg_lsn 型の内部表現（XLogRecPtr）を、SQLの出力形式である XX/XXXXXXXX 文字列へ変換します。

シグネチャまたは形状

Datum pg_lsn_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_LSN(0) を介して XLogRecPtr 型のLSN値を受け取ります。
  • **出力**: PG_RETURN_CSTRING を介して、フォーマット済みの文字列（char *）を返却します。

前提条件と不変条件

  • 入力された Datum は有効な XLogRecPtr として解釈可能である必要があります。
  • 出力文字列は MAXPG_LSNLEN + 1 バイトのバッファに収まる形式（XX/XXXXXXXX）で生成されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_LSN(0) により、引数から64ビット整数のLSNを取得します。 2. LSN_FORMAT_ARGS マクロを使用して、LSNを上位32ビットと下位32ビットの16進数表現に分解します。 3. snprintf を用いて XX/XXXXXXXX 形式の文字列を一時バッファ buf に書き込みます。 4. pstrdup を使用して、メモリコンテキスト上に文字列のコピーを作成します。 5. 生成された文字列を PG_RETURN_CSTRING で返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースを介して呼び出されます（直接的なC言語レベルの呼び出し元は提供されたソース内にはありません）。
  • **呼び出し先**: pstrdup（メモリ確保）、snprintf（文字列整形）、PG_GETARG_LSN / PG_RETURN_CSTRING（PostgreSQL関数インターフェースマクロ）。

関連データ構造

  • XLogRecPtr: LSNの内部表現（64ビット整数）。
  • MAXPG_LSNLEN: LSNの文字列表現に必要な最大長を定義する定数。

失敗モードと境界条件

  • メモリ不足時には pstrdup がエラーを発生させ、処理が中断される可能性があります。
  • snprintf のバッファサイズは MAXPG_LSNLEN に基づいており、標準的なLSN形式に対して十分な領域が確保されています。

保守担当者向けメモ

  • 本関数は pg_lsn 型の出力関数として登録されています。
  • LSNのフォーマット変更が必要な場合は、LSN_FORMAT_ARGS マクロの定義を確認してください。
