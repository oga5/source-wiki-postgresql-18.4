TrgmGistOptions

定義

  • 種別: struct
  • 定義箇所: contrib/pg_trgm/trgm_gist.c:14

目的

PostgreSQL の pg_trgm 拡張における GiST インデックス作成時に、シグネチャ（ビットベクトル）の長さを保持・管理するためのオプション構造体です。

シグネチャまたは形状

typedef struct
{
	int32		vl_len_;		/* varlena header */
	int			siglen;			/* signature length in bytes */
} TrgmGistOptions;

入力と出力

  • **入力**: インデックス作成時のオプション設定値。
  • **出力**: siglen フィールドを通じて、インデックスのビットベクトル長を後続の処理（圧縮や検索）に提供します。

前提条件と不変条件

  • vl_len_ は PostgreSQL の可変長データ型（varlena）ヘッダーであり、直接操作してはなりません。
  • siglen はインデックスの物理的な構造を決定するため、インデックス作成後に変更することは想定されていません。

アルゴリズムまたは振る舞い

  • インデックスの初期化時に init_local_reloptions を通じてメモリが割り当てられ、設定値が格納されます。
  • 実行時には PG_GET_OPCLASS_OPTIONS() を介してこの構造体にアクセスし、siglen を取得することで、トリグラムのハッシュ化や比較に使用するビットベクトルのサイズを動的に決定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_trgm/trgm_gist.c 内のインデックスオプション初期化処理（init_local_reloptions）。
  • インデックスの演算子クラスオプション取得関数（PG_GET_OPCLASS_OPTIONS()）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • gtrgm_consistent_cache: 検索時に使用されるキャッシュ構造体。
  • varlena: PostgreSQL の可変長データ型ヘッダー。

失敗モードと境界条件

  • siglen の値が不正な場合、インデックスの構築や検索時にビットベクトル操作でメモリ境界外アクセスが発生する可能性があります。

保守担当者向けメモ

  • offsetof(TrgmGistOptions, siglen) を使用してフィールドにアクセスしている箇所があるため、構造体のメンバ順序を変更する際は注意してください。
  • vl_len_ は PostgreSQL の内部マクロで管理されるため、直接の読み書きは避けてください。
