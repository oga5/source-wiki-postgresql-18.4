ExecInitBitmapIndexScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapIndexscan.c:222

目的

BitmapIndexScan ノードの実行状態（BitmapIndexScanState）を初期化し、インデックススキャンの準備を行う。

シグネチャまたは形状

BitmapIndexScanState *ExecInitBitmapIndexScan(BitmapIndexScan *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の BitmapIndexScan プランノード。
  • estate: 実行状態（EState）。
  • eflags: 実行フラグ。EXEC_FLAG_BACKWARD および EXEC_FLAG_MARK はサポート外。
  • **出力**:
  • 初期化済みの BitmapIndexScanState 構造体へのポインタ。

前提条件と不変条件

  • 上位の BitmapHeapScan ノードが、実行ツリー全体を通じて対象ヒープリレーションに対する AccessShareLock 以上のロックを保持していることを前提とする。
  • 本関数内では、ベースとなるヒープリレーションのオープンやロックは行わない。

アルゴリズムまたは振る舞い

1. BitmapIndexScanState 構造体を生成し、ExecBitmapIndexScan を実行関数として設定する。 2. EXEC_FLAG_EXPLAIN_ONLY が指定されている場合は、インデックスのオープンを行わずに終了する。 3. index_open を呼び出し、対象インデックスをオープンする。 4. ExecIndexBuildScanKeys を呼び出し、インデックスの検索条件（indexqual）からスキャンキー、ランタイムキー、配列キーを構築する。 5. ランタイムキーまたは配列キーが存在する場合、評価用の ExprContext を割り当てる。 6. index_beginscan_bitmap を呼び出し、インデックススキャン記述子を準備する。 7. ランタイムキーや配列キーが存在しない場合、index_rescan を呼び出し、スキャンキーをインデックスアクセスメソッドに渡す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c（ExecInitNode 経由での呼び出しが示唆される）。
  • **呼び出し先**:
  • makeNode, exec_rt_fetch, index_open
  • ExecIndexBuildScanKeys
  • ExecAssignExprContext
  • index_beginscan_bitmap
  • index_rescan

関連データ構造

  • BitmapIndexScan: プランノード。
  • BitmapIndexScanState: 実行状態を保持するノード。
  • ExprContext: ランタイムキー評価用コンテキスト。

失敗モードと境界条件

  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK が含まれている場合、Assert により停止する。
  • インデックスが存在しない場合やロック取得に失敗した場合は、index_open 等の内部関数がエラーを発生させる。

保守担当者向けメモ

  • 本関数は ExecProcNode インターフェースを実装していない。実行時には必ず MultiExecBitmapIndexScan を使用すること。
  • ランタイムキーの評価には専用の ExprContext を使用する設計となっており、nodeIndexscan.c と同様のアプローチを採用している。
