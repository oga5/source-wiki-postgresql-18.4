TransactionIdDidCommit

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/transam.c:125

目的

指定されたトランザクションID（XID）が最終的にコミットされたかどうかを判定します。サブトランザクションの階層を再帰的に解決し、親トランザクションの状態に基づいて最終結果を返します。

シグネチャまたは形状

bool TransactionIdDidCommit(TransactionId transactionId)

入力と出力

  • **入力**: transactionId (判定対象のトランザクションID)
  • **出力**: bool (コミット済みであれば true、それ以外は false)

前提条件と不変条件

  • TransactionLogFetch を介して CLOG（Commit Log）から状態を取得します。
  • SUB_COMMITTED 状態のトランザクションについては、SubTransGetParent を用いて親トランザクションを辿る再帰処理が行われます。
  • TransactionXmin より古いトランザクションに対しては、pg_subtrans の情報が欠落している可能性があるため、安全側に倒して false を返します。

アルゴリズムまたは振る舞い

1. TransactionLogFetch(transactionId) を呼び出し、現在のステータスを取得します。 2. ステータスが TRANSACTION_STATUS_COMMITTED の場合、直ちに true を返します。 3. ステータスが TRANSACTION_STATUS_SUB_COMMITTED の場合:

  • transactionId が TransactionXmin より古い場合は false を返します。
  • SubTransGetParent で親XIDを取得します。
  • 親XIDが無効な場合は警告を出力し false を返します。
  • 有効な親XIDに対して、本関数を再帰的に呼び出し、その結果を返します。

    • 4. 上記以外（IN_PROGRESS や ABORTED 等）の場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_heapam.c
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/heapam_visibility.c
  • **呼び出し先**:
  • TransactionLogFetch
  • TransactionIdPrecedes
  • SubTransGetParent
  • TransactionIdIsValid
  • TransactionIdDidCommit (再帰呼び出し)
  • elog (警告出力時)

関連データ構造

  • XidStatus: トランザクションの状態（COMMITTED, SUB_COMMITTED 等）を保持する列挙型。
  • TransactionId: 32ビットのトランザクション識別子。

失敗モードと境界条件

  • **サブトランザクションの欠落**: データベース起動直後など、pg_subtrans に情報がない場合に警告 (WARNING) を出力し、false を返します。
  • **再帰の深さ**: 深いサブトランザクション階層を持つ場合、再帰呼び出しによるスタック消費が発生します。
  • **ラップアラウンド**: XID比較には TransactionIdPrecedes を使用しており、循環的な比較を正しく処理します。

保守担当者向けメモ

  • 本関数は CLOG へのアクセスを抽象化する高レベルAPIです。
  • SUB_COMMITTED 状態の解決には pg_subtrans を参照するため、サブトランザクションの親子関係が正しく維持されていることが前提となります。
  • キャッシュ機構（TransactionLogFetch 内）により、確定済みのトランザクション状態は最適化されます。
