SharedInvalRelcacheMsg

定義

  • 種別: struct
  • 定義箇所: src/include/storage/sinval.h:79

目的

リレーションキャッシュ（relcache）の無効化を他のバックエンドに通知するためのメッセージ構造体です。共有メモリ上の無効化キューを介して、特定のデータベースまたはリレーションに関連するキャッシュエントリの破棄を指示します。

シグネチャまたは形状

typedef struct {
    int8    id;        /* メッセージタイプ識別子 */
    Oid     dbId;      /* データベースID（共有リレーションの場合は0） */
    Oid     relId;     /* リレーションID（リレーション全体の場合は0） */
} SharedInvalRelcacheMsg;

入力と出力

  • **入力**: id（メッセージ種別）、dbId（対象DB）、relId（対象リレーション）。
  • **出力**: なし（本構造体は共有メモリキューへの書き込みデータとして使用されます）。

前提条件と不変条件

  • id フィールドは構造体の先頭に配置されなければなりません。
  • dbId が 0 の場合、共有リレーションを指します。
  • relId が 0 の場合、そのデータベース内のリレーションキャッシュ全体を無効化する対象となります。

アルゴリズムまたは振る舞い

バックエンドがリレーション定義を変更した際、この構造体を含むメッセージが共有メモリキューに投入されます。受信側のバックエンドは ReceiveSharedInvalidMessages を通じてこのメッセージを読み取り、id に基づいてローカルの relcache エントリを無効化または再ロードします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/storage/sinval.h:129 にて SharedInvalidationMessage 共用体の一部として参照されています。
  • **呼び出し先**: 直接的な関数呼び出し関係は本ソース抜粋からは不明です。

関連データ構造

  • SharedInvalidationMessage: 本構造体を内包する共用体。
  • RelFileLocator: リレーションの物理的識別に使用される関連構造体。

失敗モードと境界条件

  • relId に 0 を指定すると広範囲な無効化が発生するため、パフォーマンスに影響を与える可能性があります。
  • 共有メモリキューが溢れた場合、PROCSIG_CATCHUP_INTERRUPT による強制的な同期処理がトリガーされる可能性があります。

保守担当者向けメモ

  • 本構造体は SharedInvalidationMessage 共用体の一部として扱われるため、サイズ変更を行う際は共用体全体のメモリレイアウトに注意してください。
  • キャッシュ無効化はトランザクションコミット時に行われるため、メッセージの送信タイミングとトランザクションの整合性に留意してください。
