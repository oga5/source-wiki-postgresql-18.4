SPI_inside_nonatomic_context

定義

  • 種別: function
  • 定義箇所: src/backend/executor/spi.c:580

目的

現在の SPI コンテキストが「非アトミック（nonatomic）」であるかどうかを判定します。これは、トランザクション制御（コミットやロールバック）が許可されるコンテキストであるかを判断するために使用されます。

シグネチャまたは形状

bool SPI_inside_nonatomic_context(void)

入力と出力

  • **入力**: なし。内部状態 _SPI_current およびトランザクション状態を参照します。
  • **出力**: bool 型。非アトミックなコンテキストであれば true、そうでなければ false を返します。

前提条件と不変条件

  • _SPI_current が NULL の場合、SPI コンテキスト外とみなされ false を返します。
  • 以下のいずれかに該当する場合、アトミックなコンテキストとみなされ false を返します。
  • _SPI_current->atomic が真である場合（関数など）。
  • 現在サブトランザクション内にある場合（IsSubTransaction() が真）。

アルゴリズムまたは振る舞い

1. _SPI_current が NULL かを確認し、該当すれば false を返す。 2. _SPI_current->atomic を確認し、真であれば false を返す。 3. IsSubTransaction() を呼び出し、サブトランザクション内であれば false を返す。 4. 上記のいずれにも該当しない場合、非アトミックなコンテキストとして true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c:2190 にて、トランザクション制御の可否判定に使用されています。
  • **呼び出し先**: IsSubTransaction()（トランザクション状態確認用）。

関連データ構造

  • _SPI_current: 現在の SPI 実行コンテキストを保持する内部スタックのトップ要素。

失敗モードと境界条件

  • SPI コンテキスト外で呼び出された場合、安全に false を返します。
  • サブトランザクション内では、たとえプロシージャ内であってもアトミックとみなされ false を返します。

保守担当者向けメモ

  • 本関数は _SPI_commit がアトミックであると判断する基準と一致させる必要があります。ロジックを変更する際は、トランザクション制御の整合性に注意してください。
