conditional_active

定義

  • 種別: function
  • 定義箇所: src/fe_utils/conditional.c:139

目的

現在の条件付き実行スタックの状態に基づき、後続のコマンドを実行すべきか（アクティブか）、あるいはスキップすべきかを判定する。

シグネチャまたは形状

bool conditional_active(ConditionalStack cstack)

入力と出力

  • **入力**: cstack (ConditionalStack型) - 管理対象の条件スタックへのポインタ。
  • **出力**: bool - 現在のブロックが実行可能であれば true、スキップ対象であれば false を返す。

前提条件と不変条件

  • cstack は有効な ConditionalStack オブジェクトである必要がある。
  • スタックが空の場合、conditional_stack_peek の振る舞いに依存する（通常は IFSTATE_NONE が返される）。

アルゴリズムまたは振る舞い

1. conditional_stack_peek(cstack) を呼び出し、スタックのトップにある現在の状態 s を取得する。 2. 以下のいずれかの状態であれば true を返す：

  • IFSTATE_NONE: 条件分岐の外側。
  • IFSTATE_TRUE: \if 条件が真であり、実行中。
  • IFSTATE_ELSE_TRUE: \else ブロックに入り、かつ条件が真（実行中）。

    • 3. 上記以外（IFSTATE_FALSE 等）の場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pgbench/pgbench.c: pgbench のスクリプト実行制御にて使用。
  • src/bin/psql/command.c: psql のメタコマンド処理にて使用。
  • src/bin/psql/common.c: パススルー処理の判定に使用。
  • src/bin/psql/mainloop.c: メインループでのコマンド実行可否判定に使用。
  • **呼び出し先**:
  • conditional_stack_peek: スタックのトップ状態を取得するために呼び出す。

関連データ構造

  • ConditionalStack: 条件分岐のネストを管理するスタック構造体。
  • ifState: IFSTATE_NONE, IFSTATE_TRUE, IFSTATE_FALSE, IFSTATE_ELSE_TRUE 等を定義する列挙型。

失敗モードと境界条件

  • cstack が NULL の場合の挙動は、conditional_stack_peek の実装に依存する。
  • スタックが空の状態で呼び出された場合、IFSTATE_NONE とみなされ true を返す設計となっている。

保守担当者向けメモ

  • 本関数は述語関数であり、副作用を持たない。
  • \if 文のネストが深くなった場合でも、スタックのトップのみを参照することで現在の実行可否を決定する。
  • 状態の追加や変更を行う場合は、ifState 列挙型と本関数の判定ロジックを同期させること。
