pgp_cfb_create

定義

  • contrib/pgcrypto/pgp-cfb.c:51

目的

PGP暗号化処理で使用するCFB（Cipher Feedback）モードのコンテキストを初期化し、暗号器の準備を行う。

シグネチャまたは形状

int pgp_cfb_create(PGP_CFB **ctx_p, int algo, const uint8 *key, int key_len, int resync, uint8 *iv);

入力と出力

  • **入力**:
  • ctx_p: 生成された PGP_CFB 構造体へのポインタを格納するアドレス。
  • algo: 使用する暗号アルゴリズムの識別子。
  • key, key_len: 暗号化キーとその長さ。
  • resync: PGP特有の再同期処理を有効にするかどうかのフラグ。
  • iv: 初期化ベクトル（NULLの場合はコピーされない）。
  • **出力**:
  • 成功時は 0 を返し、ctx_p に初期化済みコンテキストを格納する。
  • 失敗時は負の値を返す。

前提条件と不変条件

  • pgp_load_cipher により指定されたアルゴリズムがロード可能であること。
  • px_cipher_init により暗号器の初期化が成功すること。
  • palloc0 によるメモリ確保が成功すること。

アルゴリズムまたは振る舞い

1. pgp_load_cipher を呼び出し、指定されたアルゴリズムの暗号器を取得する。 2. px_cipher_init を呼び出し、キーを用いて暗号器を初期化する。 3. PGP_CFB 構造体をメモリ確保し、暗号器、ブロックサイズ、再同期フラグをセットする。 4. iv が提供されている場合、その内容をコンテキスト内のフィードバックレジスタ（ctx->fr）にコピーする。 5. 初期化されたコンテキストを ctx_p に設定し、成功を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-decrypt.c
  • contrib/pgcrypto/pgp-encrypt.c
  • contrib/pgcrypto/pgp-pubkey.c
  • **呼び出し先**:
  • pgp_load_cipher
  • px_cipher_init
  • px_cipher_free
  • palloc0
  • px_cipher_block_size

関連データ構造

  • PGP_CFB: 暗号化コンテキストを保持する構造体。
  • PX_Cipher: 汎用暗号化インターフェース。

失敗モードと境界条件

  • 指定されたアルゴリズムがサポートされていない場合、pgp_load_cipher の戻り値（負の値）をそのまま返す。
  • 暗号器の初期化に失敗した場合、確保した暗号器を解放し、エラーコードを返す。

保守担当者向けメモ

  • resync フラグは、古いPGP形式の特殊な再同期ロジックを制御する。このフラグが有効な場合、後続の処理で mix_encrypt_resync 等が選択されるため、デバッグ時は注意が必要である。
