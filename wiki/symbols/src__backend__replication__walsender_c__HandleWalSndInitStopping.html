HandleWalSndInitStopping

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walsender.c:3619

目的

WAL 送信プロセス（walsender）に対する停止要求を処理します。レプリケーションの状態に応じて、即時終了するか、あるいは正常終了のための準備を行うかを判断します。

シグネチャまたは形状

void HandleWalSndInitStopping(void)

入力と出力

  • 入力: なし（グローバル変数 am_walsender および replication_active を参照）
  • 出力: なし（副作用としてプロセスへのシグナル送信、またはフラグの更新を行う）

前提条件と不変条件

  • am_walsender が真であること（Assert により検証）。

アルゴリズムまたは振る舞い

1. replication_active が偽の場合：

  • 現在レプリケーションが開始されていないと判断し、kill(MyProcPid, SIGTERM) を実行してプロセスを即時終了させます。

    • 2. replication_active が真の場合：

  • got_STOPPING フラグを true に設定します。これにより、メインループが残りの WAL 送信とスタンバイへの同期完了を待機した後に正常終了するよう制御します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/ipc/procsignal.c:687
  • 呼び出し先:
  • kill (システムコール)

関連データ構造

  • am_walsender (グローバル変数): 現在のプロセスが WAL 送信プロセスであるかを示す。
  • replication_active (グローバル変数): レプリケーションがアクティブ状態であるかを示す。
  • got_STOPPING (グローバル変数): 停止要求を受け取ったことを示すフラグ。
  • MyProcPid (グローバル変数): 現在のプロセスの PID。

失敗モードと境界条件

  • am_walsender が偽の状態で呼び出された場合、Assert により異常終了します。

保守担当者向けメモ

  • 本関数はシグナルハンドラ等のコンテキストから呼び出されることを想定しています。
  • got_STOPPING フラグが設定された後の終了シーケンスは、walsender.c 内のメインループ（WalSndLoop 等）で制御されます。
