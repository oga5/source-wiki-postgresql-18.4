TransState

定義

  • 種別: enum
  • 定義箇所: src/backend/access/transam/xact.c:141

目的

トランザクションの低レベルな実行状態を管理し、状態遷移を追跡するために使用されます。

シグネチャまたは形状

typedef enum TransState
{
    TRANS_DEFAULT,      /* idle */
    TRANS_START,        /* transaction starting */
    TRANS_INPROGRESS,   /* inside a valid transaction */
} TransState;

入力と出力

  • 入力: トランザクションのライフサイクルイベント（開始、コミット、アボート、プリペア）。
  • 出力: 現在のトランザクション状態を示す列挙値。

前提条件と不変条件

  • TransactionStateData 構造体内の state フィールドとして保持されます。
  • トランザクションの開始から終了までの間、この状態は xact.c 内のロジックによって厳密に遷移します。

アルゴリズムまたは振る舞い

  • TRANS_DEFAULT: トランザクションが開始されていない待機状態。
  • TRANS_START: トランザクションの初期化処理中。
  • TRANS_INPROGRESS: トランザクションが有効な実行状態。
  • TRANS_COMMIT / TRANS_ABORT / TRANS_PREPARE: それぞれの終了処理が進行中であることを示します。
  • TransStateAsString() 関数により、デバッグやログ出力用に文字列へ変換可能です。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c 内のトランザクション制御関数群。
  • 呼び出し先: 提供されたソース情報からは、特定の関数呼び出し関係は特定できません。

関連データ構造

  • TransactionStateData: この列挙型を保持するメインのトランザクション状態構造体。

失敗モードと境界条件

  • 状態遷移の順序が不正な場合（例: TRANS_DEFAULT から直接 TRANS_COMMIT への遷移など）、トランザクション管理の整合性が損なわれます。

保守担当者向けメモ

  • 本シンボルは低レベルな状態管理用です。SQLレベルの状態遷移については TBlockState を参照してください。
  • 状態遷移のデバッグには TransStateAsString を活用してください。
