test_timing

定義

  • 種別: function
  • 定義箇所: src/bin/pg_test_timing/pg_test_timing.c:121

目的

システムクロックの精度とオーバーヘッドを測定し、時刻の単調性を検証するメインループを実行する。

シグネチャまたは形状

static uint64 test_timing(unsigned int duration)

入力と出力

  • **入力**: duration (unsigned int) - テストを実行する秒数。
  • **出力**: uint64 - 計測期間中に実行された時刻取得ループの総回数。

前提条件と不変条件

  • duration が 0 より大きい場合、指定されたマイクロ秒数に達するまでループを継続する。
  • INSTR_TIME_* マクロがプラットフォームの時刻取得機能として利用可能であること。

アルゴリズムまたは振る舞い

1. start_time を基準点として設定し、ループを開始する。 2. ループ内で INSTR_TIME_SET_CURRENT を呼び出し、現在時刻を取得する。 3. 前回取得した時刻との差分（diff）を計算する。 4. diff が負の場合、時刻の逆転（Time warp）と判断し、エラーメッセージを出力してプロセスを終了する。 5. diff のビット長を計算し、グローバルな histogram[] 配列の該当バケットをインクリメントする。 6. 経過時間が duration に達するまでループを繰り返し、最後にループあたりの平均オーバーヘッドをナノ秒単位で標準出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/bin/pg_test_timing/pg_test_timing.c:35）。
  • **呼び出し先**: INSTR_TIME_* マクロ群（portability/instr_time.h）、fprintf、printf、exit。

関連データ構造

  • histogram[]: 時刻差分の分布を保持するグローバル配列。
  • instr_time: プラットフォーム非依存の時刻保持用構造体。

失敗モードと境界条件

  • **時刻の逆転**: diff < 0 となった場合、exit(1) により異常終了する。
  • **分解能の限界**: diff が 0 となる場合、システムクロックの分解能が測定ループの実行速度よりも粗いことを示す。

保守担当者向けメモ

  • 本関数は pg_test_timing の中核であり、高精度タイマーの挙動を直接測定する。
  • diff のビット長計算は、ヒストグラムのバケット分類（2のべき乗）に直結しているため、変更時は注意が必要である。
