bbsink_throttle_new

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_throttle.c:67

目的

転送速度制限（スロットリング）機能を備えた新しい bbsink インスタンスを生成し、初期化する。

シグネチャまたは形状

bbsink *bbsink_throttle_new(bbsink *next, uint32 maxrate)

入力と出力

  • **入力**:
  • next: データを転送する後続の bbsink オブジェクト。
  • maxrate: 最大転送レート（単位は KB/s）。
  • **出力**:
  • 初期化された bbsink 構造体へのポインタ。

前提条件と不変条件

  • next は NULL であってはならない。
  • maxrate は 0 より大きい値である必要がある。

アルゴリズムまたは振る舞い

1. bbsink_throttle 構造体用にメモリを確保する。 2. bbsink_throttle_ops を操作関数テーブルとして設定し、bbs_next に next を割り当てる。 3. THROTTLING_FREQUENCY に基づき、一度の判定単位となるバイト数 throttling_sample を計算する。 4. 判定単位あたりの最小経過時間 elapsed_min_unit を計算する。 5. 初期化済みの bbsink ベース構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c:1040
  • **呼び出し先**:
  • palloc0 (メモリ確保)

関連データ構造

  • bbsink: バックアップデータの出力先パイプラインの基底構造体。
  • bbsink_throttle: bbsink を継承し、スロットリング状態（throttling_sample, elapsed_min_unit 等）を保持する構造体。

失敗モードと境界条件

  • maxrate が 0 の場合、Assert により停止する。
  • next が NULL の場合、Assert により停止する。

保守担当者向けメモ

  • 本関数は bbsink のデコレータパターンを構成するコンストラクタである。
  • throttling_sample の計算には整数演算を使用しており、端数は次回の転送処理に持ち越される設計となっている。
