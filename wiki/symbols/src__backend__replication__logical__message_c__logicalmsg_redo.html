logicalmsg_redo

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/message.c:86

目的

WALリプレイ時に論理メッセージレコードを処理するためのリソースマネージャ（RMGR）関数です。物理的なデータ復旧ではなく、論理デコーディングエンジンがメッセージを認識するためのフックとして機能します。

シグネチャまたは形状

void logicalmsg_redo(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（解析対象のWALレコード）
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • 入力されたレコードのオペコードが XLOG_LOGICAL_MESSAGE である必要があります。

アルゴリズムまたは振る舞い

1. XLogRecGetInfo を使用してレコードの情報を取得し、XLR_INFO_MASK を除外したオペコードを抽出します。 2. オペコードが XLOG_LOGICAL_MESSAGE と一致するか検証します。一致しない場合は PANIC を発生させます。 3. 一致する場合、実質的な処理（NOOP）を行い終了します。論理デコーディングに必要な処理は、この関数内ではなく decode.c 等の論理デコーディングエンジン側で行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: WALリプレイ処理のメインループ（src/include/access/rmgrlist.h で RM_LOGICALMSG_ID に登録されているため、リプレイ時に間接的に呼び出されます）。
  • **呼び出し先**: XLogRecGetInfo（WALレコード情報取得）、elog（エラー報告）。

関連データ構造

  • XLogReaderState: WALレコードを読み取るための状態構造体。
  • xl_logical_message: WALに記録される論理メッセージの構造体。

失敗モードと境界条件

  • **PANIC**: XLogRecGetInfo で取得したオペコードが XLOG_LOGICAL_MESSAGE 以外である場合、予期せぬレコードタイプとしてシステムを停止させます。

保守担当者向けメモ

  • 本関数は意図的にNOOPとして実装されています。これは、論理メッセージが物理的なデータ復旧を必要とせず、論理デコーディングのストリーム上でのみ意味を持つためです。
  • 実際のデコード処理は src/backend/replication/logical/decode.c に委ねられています。変更を加える際は、論理デコーディングの整合性に影響がないか確認してください。
