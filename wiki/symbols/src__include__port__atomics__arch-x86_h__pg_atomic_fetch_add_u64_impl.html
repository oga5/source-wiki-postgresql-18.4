pg_atomic_fetch_add_u64_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/arch-x86.h:221

目的

x86アーキテクチャにおいて、64ビット符号なし整数に対するアトミックな加算操作を行い、加算前の値を返す。

シグネチャまたは形状

static inline uint64 pg_atomic_fetch_add_u64_impl(volatile pg_atomic_uint64 *ptr, int64 add_)

入力と出力

  • **入力**:
  • ptr: 操作対象となる pg_atomic_uint64 型のメモリ領域へのポインタ。
  • add_: 加算する値（int64）。
  • **出力**:
  • 加算が行われる直前の ptr->value の値（uint64）。

前提条件と不変条件

  • 本関数は __x86_64__ 環境での使用を前提としている（32bit環境では提供されない）。
  • ptr はアトミック操作に適したメモリ配置である必要がある。

アルゴリズムまたは振る舞い

1. インラインアセンブラを用いて lock xaddq 命令を実行する。 2. lock プレフィックスにより、マルチコア環境におけるメモリバスの排他制御とキャッシュコヒーレンシを保証する。 3. xaddq 命令により、ptr->value に add_ を加算し、加算前の値をレジスタ res に格納する。 4. memory および cc（条件コード）を破壊対象として指定し、コンパイラによる最適化を抑制する。 5. 加算前の値 res を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/atomics.h および src/include/port/atomics/generic.h 経由で、アトミック操作を必要とする上位モジュールから呼び出される。
  • **呼び出し先**: 本関数はインラインアセンブラを直接発行するため、他の関数呼び出しは行わない。

関連データ構造

  • pg_atomic_uint64: アトミック操作対象の64ビット整数を保持する構造体。

失敗モードと境界条件

  • 境界条件: add_ に負の値を指定することで、アトミックな減算として機能する。
  • 失敗モード: メモリポインタ ptr が無効な場合や、アライメントが不適切な場合はハードウェア例外が発生する可能性がある。

保守担当者向けメモ

  • 本実装はx86のTSO（Total Store Order）モデルに依存しており、lock 命令による同期のみで十分なセマンティクスを確保している。
  • 64bit環境専用であるため、移植性を考慮する場合は generic.h 経由のインターフェースを使用すること。
