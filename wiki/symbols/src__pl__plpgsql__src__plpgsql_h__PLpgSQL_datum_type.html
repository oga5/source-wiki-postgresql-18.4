PLpgSQL_datum_type

定義

  • 種別: enum
  • 定義箇所: src/pl/plpgsql/src/plpgsql.h:62

目的

PL/pgSQL におけるデータ保持要素（PLpgSQL_datum）の種類を識別するための列挙型です。実行エンジンが変数の型に応じて適切な処理を行うための判別子として機能します。

シグネチャまたは形状

typedef enum PLpgSQL_datum_type
{
    PLPGSQL_DTYPE_VAR,
    PLPGSQL_DTYPE_ROW,
    PLPGSQL_DTYPE_REC,
    PLPGSQL_DTYPE_RECFIELD,
    PLPGSQL_DTYPE_PROMISE,
} PLpgSQL_datum_type;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: PLPGSQL_DTYPE_VAR, PLPGSQL_DTYPE_ROW, PLPGSQL_DTYPE_REC, PLPGSQL_DTYPE_RECFIELD, PLPGSQL_PROMISE のいずれかの識別子。

前提条件と不変条件

  • PLpgSQL_datum 構造体（またはそれを継承・包含する構造体）の dtype フィールドとして使用されます。
  • 実行時、この値は PLpgSQL_execstate 内のデータ構造を解釈するための不変の型情報として参照されます。

アルゴリズムまたは振る舞い

  • 実行エンジンは、この列挙値に基づいて switch 文等で処理を分岐させ、変数の読み書きや評価を行います。
  • 各値は、PL/pgSQL が扱うスカラ変数、行型、レコード型、レコードフィールド、および遅延評価される値（promise）をそれぞれ表します。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpgsql/src/plpgsql.h 内の複数の構造体（PLpgSQL_datum を含む構造体）がこの型をメンバとして保持しています。
  • 呼び出し先: 具体的な関数呼び出し関係はソース抜粋からは不明です。

関連データ構造

  • PLpgSQL_datum: 本列挙型を dtype フィールドとして保持する基底構造体。
  • PLpgSQL_execstate: 実行時に datum を管理するコンテキスト。

失敗モードと境界条件

  • 未定義の列挙値が代入された場合、実行エンジン側の switch 文でデフォルトケース（エラー処理）に到達する可能性があります。

保守担当者向けメモ

  • 新しいデータ型を PL/pgSQL に追加する際は、この列挙型に新しいメンバを追加し、実行エンジン（pl_exec.c）側の処理ロジックを更新する必要があります。
