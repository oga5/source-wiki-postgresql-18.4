pg_char_and_wchar_strcmp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/wstrcmp.c:40

目的

マルチバイト文字列（char *）とワイド文字配列（pg_wchar *）を逐次比較し、辞書順の比較結果を返します。

シグネチャまたは形状

int pg_char_and_wchar_strcmp(const char *s1, const pg_wchar *s2)

入力と出力

  • **入力**:
  • s1: 比較対象のヌル終端マルチバイト文字列。
  • s2: 比較対象のヌル終端ワイド文字配列。
  • **出力**:
  • 両者が一致した場合は 0 を返します。
  • 不一致の場合は、s1 の文字と s2 の文字の差分（unsigned char と pg_wchar としての評価値の差）を返します。

前提条件と不変条件

  • s1 および s2 はヌル終端されている必要があります。
  • フロントエンドおよびバックエンドの両方から呼び出し可能です。

アルゴリズムまたは振る舞い

1. s1 の各文字を pg_wchar にキャストし、s2 の現在の要素と順次比較します。 2. s1 がヌル終端（0）に達した時点で比較を終了し、一致として 0 を返します。 3. 不一致が発生した場合、s1 の現在の文字（unsigned char にキャスト）と、不一致となった s2 の文字の差分を算出して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは、具体的な呼び出し元関数は特定されていません。
  • **呼び出し先**: pg_wchar 型の比較演算およびポインタ演算のみを使用しており、他の関数呼び出しは行いません。
  • **宣言**: src/include/mb/pg_wchar.h:696 にて外部宣言されています。

関連データ構造

  • pg_wchar: src/include/mb/pg_wchar.h で定義されるワイド文字型。

失敗モードと境界条件

  • 文字列の終端に達した際の挙動は、s1 のヌル終端文字を正しく判定することで安全に処理されます。
  • s1 の文字を unsigned char として評価することで、符号拡張による予期せぬ比較結果を回避しています。

保守担当者向けメモ

  • 本関数は BSD ライセンス由来の古典的な文字列比較ロジックをベースとしており、高速に動作するように設計されています。
  • src/backend/utils/mb/wstrncmp.c に、比較長を指定できる関連関数が存在します。
