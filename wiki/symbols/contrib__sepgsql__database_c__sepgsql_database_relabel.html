sepgsql_database_relabel

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/database.c:186

目的

データベースオブジェクトに対するセキュリティラベルの変更（リラベル）操作において、SELinuxポリシーに基づく権限検証を行う。

シグネチャまたは形状

void sepgsql_database_relabel(Oid databaseId, const char *seclabel)

入力と出力

  • databaseId (Oid): 対象となるデータベースのOID。
  • seclabel (const char *): 設定しようとしている新しいセキュリティラベル文字列。
  • 出力: なし（権限チェックに失敗した場合は例外をスローする）。

前提条件と不変条件

  • databaseId は有効なデータベースを指している必要がある。
  • 呼び出し元は、対象データベースのセキュリティラベルを変更する権限を保持している必要がある。

アルゴリズムまたは振る舞い

1. ObjectAddress 構造体を初期化し、対象データベースを特定する。 2. getObjectIdentity を使用して、監査ログ用の識別子文字列を取得する。 3. sepgsql_avc_check_perms を呼び出し、現在のセキュリティコンテキストに対して db_database:{setattr} および db_database:{relabelfrom} 権限を検証する。 4. sepgsql_avc_check_perms_label を呼び出し、新しいセキュリティラベル seclabel に対して db_database:{relabelto} 権限を検証する。 5. 最後に監査用文字列のメモリを解放する。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/sepgsql/label.c:500
  • 呼び出し先:
  • getObjectIdentity
  • sepgsql_avc_check_perms
  • sepgsql_avc_check_perms_label
  • pfree

関連データ構造

  • ObjectAddress: PostgreSQLのオブジェクトを識別するための構造体。
  • DatabaseRelationId: データベースカタログ（pg_database）を指す定数。

失敗モードと境界条件

  • 権限チェック（sepgsql_avc_check_perms または sepgsql_avc_check_perms_label）が失敗した場合、SELinuxのポリシーに従いエラーが発生する。
  • seclabel が無効な形式である場合、sepgsql_avc_check_perms_label 内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は二段階の権限チェック（変更元と変更先）を強制する。ラベル変更のロジックを変更する際は、必ず RELABELFROM と RELABELTO の両方の検証が維持されていることを確認すること。
