pgaio_io_acquire_nb

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio.c:187

目的

非同期I/O（AIO）ハンドルを非ブロッキングで取得します。空きハンドルが存在する場合にのみハンドルを返し、存在しない場合は NULL を返します。

シグネチャまたは形状

PgAioHandle *pgaio_io_acquire_nb(struct ResourceOwnerData *resowner, PgAioReturn *ret)

入力と出力

  • **入力**:
  • resowner: I/Oハンドルのクリーンアップを管理するリソースオーナー（任意）。
  • ret: I/O完了時の結果を格納する構造体へのポインタ（任意）。
  • **出力**:
  • 成功時: 取得した PgAioHandle へのポインタ。
  • 失敗時: NULL（空きハンドルがない場合）。

前提条件と不変条件

  • 呼び出し元バックエンドにおいて、既に handed_out_io が設定されている場合、API違反としてエラーを発生させます。
  • 状態遷移中は HOLD_INTERRUPTS() により割り込みが禁止されます。
  • 取得されたハンドルは PGAIO_HS_IDLE 状態であり、かつ現在のプロセス番号と一致している必要があります。

アルゴリズムまたは振る舞い

1. ステージングされたI/O数が上限（PGAIO_SUBMIT_BATCH_SIZE）に達している場合、pgaio_submit_staged() を呼び出してI/Oを発行します。 2. 割り込みを禁止します。 3. pgaio_my_backend->idle_ios リストから空きハンドルをポップします。 4. ハンドルが見つかった場合：

  • 状態を PGAIO_HS_HANDED_OUT に更新します。
  • handed_out_io にハンドルを記録します。
  • resowner が指定されていれば、pgaio_io_resowner_register を呼び出して登録します。
  • ret が指定されていれば、ハンドルに格納し、ステータスを PGAIO_RS_UNKNOWN に初期化します。

    • 5. 割り込みを許可し、ハンドルを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio.c (内部的な取得処理)
  • src/backend/storage/buffer/bufmgr.c:1847
  • **呼び出し先**:
  • pgaio_submit_staged()
  • pgaio_io_update_state()
  • pgaio_io_resowner_register()
  • dclist_pop_head_node() (リスト操作)

関連データ構造

  • PgAioHandle: I/O操作を管理するハンドル構造体。
  • PgAioReturn: I/O完了結果を保持する構造体。
  • ResourceOwnerData: リソースのライフサイクル管理用構造体。

失敗モードと境界条件

  • **API違反**: 既にハンドルを保持している状態で呼び出すと elog(ERROR) が発生します。
  • **リソース枯渇**: 空きハンドルがない場合、NULL を返します。呼び出し元はこの戻り値を適切に処理する必要があります。

保守担当者向けメモ

  • 本関数は非ブロッキングであるため、呼び出し元は NULL が返された場合のフォールバック処理（例：同期I/Oへの切り替えや待機）を実装する必要があります。
  • 状態遷移を伴うため、HOLD_INTERRUPTS() の範囲外でハンドルを操作しないよう注意してください。
