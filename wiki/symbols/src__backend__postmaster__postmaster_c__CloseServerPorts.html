CloseServerPorts

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/postmaster.c:1415

目的

Postmaster プロセスの終了時に、リッスン中のソケットおよび Unix ソケットファイルを明示的にクローズ・削除し、後続の Postmaster がポートやソケットファイルを再利用する際の競合を防止する。

シグネチャまたは形状

static void CloseServerPorts(int status, Datum arg)

入力と出力

  • **入力**:
  • status: on_proc_exit コールバック用引数（未使用）。
  • arg: on_proc_exit コールバック用引数（未使用）。
  • **出力**: なし（副作用としてソケットのクローズとファイル削除を行う）。

前提条件と不変条件

  • NumListenSockets が現在開いているソケット数を保持していること。
  • ListenSockets 配列に有効なソケット記述子が格納されていること。
  • 本関数は on_proc_exit を通じてプロセス終了時に呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. ListenSockets 配列を走査し、closesocket() を用いてすべてのリッスンソケットを閉じる。失敗した場合は elog(LOG, ...) でエラーを記録する。 2. NumListenSockets を 0 にリセットする。 3. RemoveSocketFiles() を呼び出し、Unix ソケットに関連するファイルシステム上のエントリを削除する。 4. ソケットロックファイルについては本関数では処理せず、後続の on_proc_exit コールバックに委ねる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/postmaster.c 内の on_proc_exit(CloseServerPorts, 0) により登録され、プロセス終了時に呼び出される。
  • **呼び出し先**:
  • closesocket() (システム関数)
  • RemoveSocketFiles() (同ファイル内)
  • elog() (エラーログ出力)

関連データ構造

  • ListenSockets: リッスン中のソケット記述子を保持する配列。
  • NumListenSockets: 現在開いているソケットの数。

失敗モードと境界条件

  • closesocket() が失敗した場合、ログにエラーを出力するが、処理は継続する。
  • NumListenSockets が 0 の場合、ループは実行されず、ソケットのクローズはスキップされる。

保守担当者向けメモ

  • 本関数は postmaster.pid ロックファイルの削除よりも前に実行される必要がある。これは、ロックファイル削除前にソケットを閉じることで、新しい Postmaster がポートを再利用する際の競合を防ぐためである。
  • ソケットロックファイルの削除は本関数の責務ではなく、別の on_proc_exit コールバックで行われる点に注意すること。
