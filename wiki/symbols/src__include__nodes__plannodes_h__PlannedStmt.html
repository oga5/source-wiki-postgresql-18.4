PlannedStmt

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/plannodes.h:46

目的

クエリ実行計画ツリーのルート構造体です。プランナによって生成され、エグゼキュータがクエリを実行するための「設計図」として利用されます。

シグネチャまたは形状

typedef struct PlannedStmt {
    NodeTag     type;
    CmdType     commandType;
    int64       queryId;
    int64       planId;
    bool        hasReturning;
    bool        hasModifyingCTE;
    bool        canSetTag;
    bool        transientPlan;
    bool        dependsOnRole;
    bool        parallelModeNeeded;
    int         jitFlags;
    struct Plan *planTree;
    List        *partPruneInfos;
    List        *rtable;
    Bitmapset   *unprunableRelids;
    List        *permInfos;
    List        *resultRelations;
    List        *appendRelations;
    List        *subplans;
    Bitmapset   *rewindPlanIDs;
    List        *rowMarks;
    List        *relationOids;
    List        *invalItems;
    List        *paramExecTypes;
    Node        *utilityStmt;
    ParseLoc    stmt_location;
    ParseLoc    stmt_len;
} PlannedStmt;

入力と出力

  • **入力**: プランナが Query 構造体から変換して生成します。
  • **出力**: エグゼキュータがこの構造体を参照し、planTree を再帰的に辿って実行処理を行います。

前提条件と不変条件

  • planTree は実行計画のルートノードを指します。
  • rtable（RangeTblEntryのリスト）はクエリ内の全リレーション情報を保持します。
  • commandType は実行するコマンドの種類（SELECT, INSERT, UPDATE, DELETE, MERGE, utility）を識別します。

アルゴリズムまたは振る舞い

1. **初期化**: プランナがクエリを最適化し、実行に必要なメタデータ（JITフラグ、パーティション情報、依存関係など）を本構造体に格納します。 2. **実行**: エグゼキュータが planTree を起点に各ノードの実行関数を呼び出します。 3. **射影とフィルタリング**: 各ノードは targetlist や qual を使用してタプルの処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/ 配下のプランナが生成します。
  • **呼び出し先**: src/backend/executor/ 配下のエグゼキュータが消費します。
  • **その他**: contrib/pg_overexplain がデバッグや説明目的で本構造体のフィールドを参照しています。

関連データ構造

  • Plan: 実行計画ツリーの基底構造体。
  • RangeTblEntry: クエリ内のリレーション定義。
  • List: 各種ノードや情報のリスト管理に使用。
  • Bitmapset: リレーションIDやプランIDの集合管理に使用。

失敗モードと境界条件

  • utilityStmt が非NULLの場合、通常のプラン実行ではなくユーティリティ文として扱われます。
  • rewindPlanIDs に含まれるサブプランは、実行中に再走査（REWIND）が必要となる境界条件を持ちます。

保守担当者向けメモ

  • Plan 構造体と同様に、C言語での継承を模した構造になっています。
  • outerPlan() / innerPlan() マクロを使用して planTree 内部のノードにアクセスすることが推奨されます。
  • no_equal および no_query_jumble 属性が付与されており、等価比較やクエリジャンブルの対象外であることを示しています。
