ProcessStartupPacket

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/backend_startup.c:491

目的

クライアントから送信されるスタートアップパケットを解析し、接続プロトコル（SSL/GSSAPI）のネゴシエーションおよび接続パラメータの抽出を行う。

シグネチャまたは形状

static int ProcessStartupPacket(Port *port, bool ssl_done, bool gss_done)

入力と出力

  • **入力**:
  • port: 接続情報を保持する Port 構造体へのポインタ。
  • ssl_done: SSLネゴシエーションが完了済みかを示すフラグ。
  • gss_done: GSSAPIネゴシエーションが完了済みかを示すフラグ。
  • **出力**:
  • 成功時は STATUS_OK（コード内では明示的な戻り値の定義に依存）、失敗またはキャンセル要求時は STATUS_ERROR を返す。

前提条件と不変条件

  • pq_startmsgread() が呼び出され、通信層がメッセージ読み取り可能な状態であること。
  • パケット長は MAX_STARTUP_PACKET_LENGTH 以下であること。
  • パケット内の文字列は、メモリ確保時に末尾に \0 を付与することで安全に終端される。

アルゴリズムまたは振る舞い

1. パケット長（4バイト）を読み取り、ネットワークバイトオーダーからホストバイトオーダーへ変換する。 2. パケット本体をメモリに確保し、読み込む。 3. パケットの先頭にあるプロトコルバージョンまたはリクエストコードを判定する。

  • CANCEL_REQUEST_CODE: ProcessCancelRequestPacket を呼び出し、処理を終了する。
  • NEGOTIATE_SSL_CODE: SSLの可否を判定し、クライアントに応答を返す。SSL有効時は secure_open_server を実行し、未暗号化データの混入をチェックして再試行（retry）する。
  • NEGOTIATE_GSS_CODE: GSSAPIの可否を判定し、クライアントに応答を返す。

    • 4. 正常なスタートアップパケットの場合は、接続パラメータを解析する（ソース抜粋以降の処理）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/backend_startup.c 内の BackendInitialize から呼び出される。
  • **呼び出し先**:
  • ProcessCancelRequestPacket
  • secure_write / secure_open_server
  • pq_getbytes / pq_startmsgread / pq_endmsgread / pq_buffer_remaining_data

関連データ構造

  • Port: 接続状態やソケット情報を保持する構造体。
  • ProtocolVersion: プロトコルバージョンを定義する型。

失敗モードと境界条件

  • **通信エラー**: pq_getbytes が EOF を返した場合、不完全なパケットとして STATUS_ERROR を返す。
  • **不正なパケット長**: 長さが ProtocolVersion 未満、または MAX_STARTUP_PACKET_LENGTH を超える場合は COMMERROR を記録し STATUS_ERROR を返す。
  • **SSL/GSSAPIネゴシエーション後の不正データ**: SSLハンドシェイク後に未暗号化データがバッファに残っている場合、FATAL エラーとして接続を終了する。

保守担当者向けメモ

  • 本関数は retry ラベルを用いたループ構造を持ち、SSL/GSSAPIネゴシエーション後に再度パケット読み取りを試みる設計となっている。
  • 共有メモリへのアクセスは行わないこと（BackendInitialize の制約に準拠）。
  • ssl_done および gss_done フラグの状態管理が、ネゴシエーションの順序制御において重要である。
