pgstat_fetch_stat_checkpointer

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_checkpointer.c:80

目的

チェックポイントおよび再起動ポイントに関する統計情報の最新スナップショットを取得し、呼び出し元へ提供する。

シグネチャまたは形状

PgStat_CheckpointerStats *pgstat_fetch_stat_checkpointer(void)

入力と出力

  • **入力**: なし。
  • **出力**: PgStat_CheckpointerStats 構造体へのポインタ。この構造体には、チェックポイントの回数、書き込みバッファ数、同期時間などの統計値が含まれる。

前提条件と不変条件

  • 呼び出し前に pgstat_snapshot_fixed(PGSTAT_KIND_CHECKPOINTER) が実行され、共有メモリ上の統計データがローカルのスナップショット領域に同期されている必要がある。

アルゴリズムまたは振る舞い

1. pgstat_snapshot_fixed(PGSTAT_KIND_CHECKPOINTER) を呼び出し、共有メモリからローカルの統計スナップショットを更新する。 2. pgStatLocal.snapshot.checkpointer （プロセスローカルなスナップショット領域）のアドレスを戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/pgstatfuncs.c 内の各種統計取得関数（例: チェックポイント関連の統計をSQL経由で参照する関数群）。
  • **呼び出し先**: pgstat_snapshot_fixed（pgstat サブシステム内の関数）。

関連データ構造

  • PgStat_CheckpointerStats: チェックポイント統計を保持する構造体。
  • pgStatLocal: プロセスローカルな統計管理用グローバル変数。

失敗モードと境界条件

  • 共有メモリが初期化されていない場合や、統計サブシステムが正しく動作していない場合の挙動については、本ソースコード単体では明示されていない。

保守担当者向けメモ

  • 本関数は、統計の「保存・アクセス」と「統計内容の詳細」を分離する設計に基づいている。
  • 戻り値の構造体は pgStatLocal 内の静的な領域を指しているため、呼び出し元でこのポインタを長期間保持することは避けること。
