visibilitymap_pin_ok

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/visibilitymap.c:216

目的

指定されたヒープブロックに対応する可視性マップ（Visibility Map）のページが、現在バッファにピン留めされているかどうかを確認します。

シグネチャまたは形状

bool visibilitymap_pin_ok(BlockNumber heapBlk, Buffer vmbuf)

入力と出力

  • **入力**:
  • heapBlk: 確認対象のヒープブロック番号。
  • vmbuf: 現在ピン留めされている可視性マップのバッファ。
  • **出力**:
  • bool: 指定されたバッファが対象のヒープブロックに対応する正しいマップページであれば true、そうでなければ false を返します。

前提条件と不変条件

  • HEAPBLK_TO_MAPBLOCK マクロにより、ヒープブロック番号から対応するマップブロック番号が一意に算出されます。
  • vmbuf が有効なバッファである必要があります。

アルゴリズムまたは振る舞い

1. heapBlk を HEAPBLK_TO_MAPBLOCK マクロに渡し、対応するマップブロック番号を計算します。 2. vmbuf が有効（BufferIsValid）であり、かつそのバッファが保持しているブロック番号が計算したマップブロック番号と一致するかを判定し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c:2643
  • src/backend/access/heap/hio.c:175, 178, 782
  • **呼び出し先**:
  • BufferIsValid (マクロ)
  • BufferGetBlockNumber (マクロ)
  • HEAPBLK_TO_MAPBLOCK (マクロ)

関連データ構造

  • Buffer: バッファプール内のページを識別するハンドル。
  • BlockNumber: ブロック番号を表す型。

失敗モードと境界条件

  • vmbuf が InvalidBuffer の場合、関数は false を返します。
  • ヒープブロック番号がマップの範囲外である場合、計算されるマップブロック番号もそれに準じます。

保守担当者向けメモ

  • 本関数は、可視性マップのページを再読み込みするコストを避けるための最適化チェックとして使用されます。
  • 呼び出し側は、この関数が false を返した場合、適切なマップページを再取得（visibilitymap_pin 等）する必要があります。
