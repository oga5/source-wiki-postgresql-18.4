TransInvalidationInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/inval.c:240

目的

トランザクションおよびサブトランザクション単位で、キャッシュ無効化メッセージのライフサイクルと階層構造を管理するために使用される構造体です。

シグネチャまたは形状

typedef struct TransInvalidationInfo
{
    struct InvalidationInfo ii;
    InvalidationMsgsGroup PriorCmdInvalidMsgs;
    struct TransInvalidationInfo *parent;
    int my_level;
} TransInvalidationInfo;

入力と出力

  • **入力**: トランザクションのネストレベル（my_level）および親トランザクションへの参照（parent）。
  • **出力**: 蓄積された無効化メッセージ群（PriorCmdInvalidMsgs）および基底クラス InvalidationInfo を介した管理情報。

前提条件と不変条件

  • parent ポインタにより、トランザクションのネスト構造を反映したチェーンを形成します。
  • PriorCmdInvalidMsgs は、現在の（サブ）トランザクションにおける先行コマンドによって生成された無効化メッセージを保持します。

アルゴリズムまたは振る舞い

  • トランザクションまたはサブトランザクションの開始時にインスタンス化され、キャッシュ無効化イベントを蓄積します。
  • コマンド境界において、蓄積されたメッセージを管理・移動させることで、コミット時まで他バックエンドへの通知を遅延させつつ、ローカルキャッシュの整合性を維持します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/inval.c 内のトランザクション管理関数（AtEOXact_Inval 等）により、メモリ確保および初期化が行われます。
  • **呼び出し先**: 構造体内部の InvalidationMsgsGroup を通じて、キャッシュ無効化メッセージの登録・送出処理が実行されます。
  • ※具体的な関数呼び出しの連鎖については、提供されたソース excerpt に詳細なスタックトレースが含まれていないため、これ以上の断定はできません。

関連データ構造

  • InvalidationInfo: 基底クラス。
  • InvalidationMsgsGroup: 蓄積された無効化メッセージのグループ。
  • TransInvalidationInfo *parent: 親トランザクションの情報を指す再帰的なポインタ。

失敗モードと境界条件

  • メモリ不足時、構造体の確保に失敗する可能性があります。
  • トランザクションのネストが深すぎる場合、parent チェーンの管理においてスタックやメモリの制約を受ける可能性があります。

保守担当者向けメモ

  • 本構造体は CurTransactionContext に関連付けられて管理されます。
  • Inplace 更新（relmapper 等）を行う際は、通常のトランザクション管理パスとは異なる処理フローとなるため、本構造体の状態と混同しないよう注意してください。
