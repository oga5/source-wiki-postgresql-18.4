pg_regmatch_t

定義

  • 種別: struct
  • 定義箇所: src/include/regex/regex.h:163

目的

正規表現の実行結果として、マッチした部分文字列の開始位置および終了位置を格納するために使用されます。

シグネチャまたは形状

typedef struct
{
	pg_regoff_t rm_so;			/* start of substring */
	pg_regoff_t rm_eo;			/* end of substring */
} pg_regmatch_t;

入力と出力

  • **入力**: pg_regexec 関数実行時に、マッチ結果を書き込むためのバッファとして提供されます。
  • **出力**: 正規表現エンジンによって、マッチした部分文字列のオフセット（rm_so および rm_eo）が書き込まれます。

前提条件と不変条件

  • rm_so は部分文字列の開始位置を示します。
  • rm_eo は部分文字列の終了位置を示します。
  • pg_regoff_t 型に依存しており、オフセット値の表現範囲は当該型に準拠します。

アルゴリズムまたは振る舞い

pg_regexec 関数において、正規表現のパターンマッチングが成功した際、対象文字列内の該当箇所の開始および終了インデックスが本構造体の各メンバに格納されます。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは、具体的な呼び出し元関数は特定できません。
  • **呼び出し先**: 本構造体はデータ保持用であり、関数を呼び出すものではありません。
  • **参照**: src/include/regex/regex.h 内で regmatch_t としてマクロ定義されています。また、REG_EXPECT フラグに関連して rm_extend メンバの型として使用される例があります。

関連データ構造

  • pg_regoff_t: オフセット値を保持するために使用される型。
  • pg_regex_t: 正規表現のコンパイル済み状態を保持する構造体。

失敗モードと境界条件

  • マッチしなかった場合の各メンバの値については、本ソース抜粋からは定義されていません。
  • pg_regoff_t の範囲を超える文字列長に対する挙動は、当該型の定義に依存します。

保守担当者向けメモ

  • 本構造体は POSIX の regmatch_t と互換性を持つ設計です。
  • src/include/regex/regex.h 内で regmatch_t にエイリアスされているため、外部インターフェースとの整合性に注意してください。
