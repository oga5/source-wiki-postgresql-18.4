hash_xlog_init_meta_page

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash_xlog.c:26

目的

ハッシュインデックスの作成時に記録されたWALレコードをリプレイし、メタページを初期化する。

シグネチャまたは形状

static void hash_xlog_init_meta_page(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）。レコード内には xl_hash_init_meta_page 構造体が含まれる。
  • **出力**: なし（副作用としてバッファおよびディスク上のメタページを更新）。

前提条件と不変条件

  • record は有効な XLogReaderState を指していること。
  • XLogRecGetData(record) が xl_hash_init_meta_page 型のデータとして正しく解釈できること。

アルゴリズムまたは振る舞い

1. XLogInitBufferForRedo を呼び出し、メタページ用のバッファを初期化する。 2. _hash_init_metabuffer を呼び出し、レコードから取得したタプル数、プロシージャID、フィルファクタを用いてメタページの内容を構築する。 3. PageSetLSN を用いて、ページのLSNをWALレコードの終了位置に更新する。 4. MarkBufferDirty を呼び出し、バッファをダーティ状態にする。 5. 対象が INIT_FORKNUM（初期化フォーク）である場合、FlushOneBuffer を呼び出してディスクへの書き込みを強制する。 6. UnlockReleaseBuffer を呼び出し、バッファのロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/hash/hash_xlog.c:1074（hash_redo 関数内でのディスパッチ）。
  • **呼び出し先**:
  • XLogRecGetData
  • XLogInitBufferForRedo
  • _hash_init_metabuffer
  • BufferGetPage
  • PageSetLSN
  • MarkBufferDirty
  • XLogRecGetBlockTag
  • FlushOneBuffer
  • UnlockReleaseBuffer

関連データ構造

  • xl_hash_init_meta_page: メタページ初期化に必要なパラメータ（タプル数、プロシージャID、フィルファクタ）を保持する構造体。
  • XLogReaderState: WALレコードの読み取り状態を保持する構造体。

失敗モードと境界条件

  • XLogInitBufferForRedo が失敗した場合、Assert により停止する可能性がある。
  • INIT_FORKNUM の判定により、インデックス作成時のメタページに対してのみ特別なディスク同期処理が行われる。

保守担当者向けメモ

  • 本関数はインデックス作成時のメタページ初期化専用である。
  • INIT_FORKNUM に対する FlushOneBuffer は、インデックス作成操作がメタページのフルページイメージをログに記録しないことに起因する整合性確保のための処理である。
