ECPGget_sqlca

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/misc.c:107

目的

現在のスレッドに対応する sqlca_t（SQL Communication Area）構造体を取得、または未作成の場合は生成して返します。

シグネチャまたは形状

struct sqlca_t *ECPGget_sqlca(void)

入力と出力

  • **入力**: なし
  • **出力**: 現在のスレッドに関連付けられた sqlca_t 構造体へのポインタ。メモリ確保に失敗した場合は NULL を返します。

前提条件と不変条件

  • pthread_once を使用して、スレッド固有のキー初期化関数 ecpg_sqlca_key_init が初回呼び出し時に一度だけ実行されることを保証します。
  • 取得される sqlca_t はスレッドローカルなストレージとして管理されます。

アルゴリズムまたは振る舞い

1. pthread_once を呼び出し、スレッド固有キーの初期化を完了させます。 2. pthread_getspecific を使用して、現在のスレッドに割り当てられた sqlca_t を検索します。 3. 構造体が未割り当て（NULL）の場合:

  • malloc で sqlca_t 用のメモリを確保します。
  • 確保に失敗した場合は NULL を返します。
  • ecpg_init_sqlca を呼び出して構造体を初期化します。
  • pthread_setspecific を使用して、確保したメモリを現在のスレッドに関連付けます。

    • 4. 確保済み、または新規作成された sqlca_t へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/ecpglib/connect.c
  • src/interfaces/ecpg/ecpglib/data.c
  • src/interfaces/ecpg/ecpglib/descriptor.c
  • src/interfaces/ecpg/ecpglib/error.c
  • **呼び出し先**:
  • pthread_once
  • pthread_getspecific
  • malloc
  • ecpg_init_sqlca
  • pthread_setspecific

関連データ構造

  • struct sqlca_t: SQL通信領域を定義する構造体（sqlca.h で定義）。

失敗モードと境界条件

  • malloc がメモリ不足により失敗した場合、関数は NULL を返します。呼び出し元はこの戻り値を確認する必要があります。

保守担当者向けメモ

  • 本関数はマルチスレッド環境でのスレッドセーフティを担保するため、pthread 関連の関数に依存しています。Windows環境では、ecpg-pthread-win32.h を介して提供される互換レイヤーが使用されます。
  • sqlca_t のライフサイクル管理は、スレッド終了時に ecpg_sqlca_key_destructor が適切に実行されることを前提としています。
