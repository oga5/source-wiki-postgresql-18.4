PG_STAT_GET_XACT_RELENTRY_INT64

定義

  • 種別: macro
  • 定義箇所: src/backend/utils/adt/pgstatfuncs.c:1775

目的

トランザクション単位の統計情報（PgStat_StatTabEntry）から、特定の int64 型の統計項目を抽出して SQL の Datum として返す関数を定義するためのテンプレートマクロです。

シグネチャまたは形状

PG_STAT_GET_XACT_RELENTRY_INT64(stat)
  • stat: 取得対象となる統計フィールド名。

入力と出力

  • **入力**: SQL 関数呼び出し時に渡される引数（対象テーブルの OID）。
  • **出力**: 指定された統計項目の値（int64）。対象が見つからない場合は NULL を返します。

前提条件と不変条件

  • 呼び出し元は、統計情報システムが提供するトランザクションレベルの統計データにアクセス可能である必要があります。
  • マクロ展開後の関数は、PG_FUNCTION_ARGS を引数に取る SQL 関数として定義されます。

アルゴリズムまたは振る舞い

1. 引数として渡された OID を基に、現在のトランザクションにおける対象テーブルの統計エントリを検索します。 2. エントリが存在しない場合は PG_RETURN_NULL() を実行します。 3. エントリが存在する場合、マクロ引数 stat で指定されたフィールドの値を PG_RETURN_INT64() を用いて返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下の統計取得関数が本マクロを使用して定義されています。
  • tuples_returned, tuples_fetched, tuples_hot_updated, tuples_newpage_updated, blocks_fetched, blocks_hit, tuples_inserted, tuples_updated, tuples_deleted
  • **呼び出し先**: 内部で統計情報検索用の関数（pgstat_fetch_stat_tabentry 等の統計システムAPI）を呼び出します（ソースコード上の直接的な呼び出し先詳細は抽出対象外）。

関連データ構造

  • PgStat_StatTabEntry: トランザクション単位のテーブル統計情報を保持する構造体。

失敗モードと境界条件

  • 指定された OID に対応する統計エントリが存在しない場合、関数は NULL を返します。
  • 統計情報システムが初期化されていない、または無効な状態での呼び出しについては、統計システムの仕様に依存します。

保守担当者向けメモ

  • 本マクロは pgstatfuncs.c 内で統計取得関数を量産するために使用されています。新しいトランザクション統計項目を追加する場合は、既存の定義パターンに従ってください。
  • 統計情報はスナップショットベースであるため、トランザクションの進行状況によって値が変化する点に留意してください。
