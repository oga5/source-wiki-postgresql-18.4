SetConstraintStateData

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/trigger.c:3626

目的

SET CONSTRAINTS コマンドの実行時において、対象となる制約トリガーの状態（遅延実行の可否など）を保持および管理するための構造体です。

シグネチャまたは形状

typedef struct SetConstraintStateData
{
    bool        all_isset;
    bool        all_isdeferred;
    int         numstates;      /* 使用中の trigstates[] エントリ数 */
    int         numalloc;       /* trigstates[] の確保済みサイズ */
    SetConstraintTriggerData trigstates[FLEXIBLE_ARRAY_MEMBER];
} SetConstraintStateData;

入力と出力

  • **入力**: SET CONSTRAINTS コマンド実行時に、対象となるトリガーのリストとそれぞれの状態（ALL 指定の有無、DEFERRED/IMMEDIATE の指定）が本構造体に格納されます。
  • **出力**: 処理対象のトリガー状態を保持し、後続のトリガー実行エンジンが参照可能な形式で提供します。

前提条件と不変条件

  • trigstates は FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体自体は可変長としてメモリ確保される必要があります。
  • numstates は numalloc を超えてはなりません。

アルゴリズムまたは振る舞い

  • SET CONSTRAINTS 実行時、指定されたトリガーの現在の状態をスキャンし、all_isset や all_isdeferred フラグを用いて一括設定の有無を管理します。
  • trigstates 配列を通じて、個別のトリガーに対する状態変更を追跡します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の offsetof(SetConstraintStateData, trigstates) を使用する箇所（src/backend/commands/trigger.c:5705, 5747）から、メモリレイアウトの計算やデータアクセスが行われています。
  • **呼び出し先**: 具体的な関数呼び出し関係は提供されたソースからは特定できません。

関連データ構造

  • SetConstraintTriggerData: trigstates 配列の各要素として使用される、個別のトリガー状態を保持する構造体。
  • SetConstraintState: SetConstraintStateData へのポインタ型（typedef SetConstraintStateData *SetConstraintState;）。

失敗モードと境界条件

  • メモリ確保時、numalloc を超えるアクセスが発生しないよう注意が必要です。
  • FLEXIBLE_ARRAY_MEMBER を含むため、静的なスタック確保ではなく、動的なメモリ割り当て（palloc 等）を前提としています。

保守担当者向けメモ

  • 本構造体は SET CONSTRAINTS の複雑な状態管理の中核を担います。trigstates の動的な拡張が必要な場合は、メモリ管理の整合性に注意してください。
  • 呼び出し元・呼び出し先の詳細なフローについては、trigger.c 内の SET CONSTRAINTS 関連関数群の解析が必要です。
