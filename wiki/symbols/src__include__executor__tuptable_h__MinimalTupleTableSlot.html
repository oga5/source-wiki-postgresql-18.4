MinimalTupleTableSlot

定義

  • 種別: struct
  • 定義箇所: src/include/executor/tuptable.h:286

目的

最小タプル（MinimalTuple）形式のデータを保持し、実行エンジン内で他の物理タプルと同様のインターフェースで属性抽出を可能にするための専用スロット構造体です。

シグネチャまたは形状

typedef struct MinimalTupleTableSlot {
    pg_node_attr(abstract)
    TupleTableSlot base;
    HeapTuple      tuple;
    MinimalTuple   mintuple;
    HeapTupleData  minhdr;
    uint32         off;
} MinimalTupleTableSlot;

入力と出力

  • **入力**: MinimalTuple 形式のデータ。
  • **出力**: slot_getattr 等の共通インターフェースを通じた属性値の提供。

前提条件と不変条件

  • minhdr.t_data は、mintuple の開始位置より MINIMAL_TUPLE_OFFSET バイト前の位置を指す必要があります。
  • この構造により、列抽出処理は通常の物理タプルと同一のロジックで処理されます。

アルゴリズムまたは振る舞い

  • mintuple を保持し、必要に応じて minhdr（HeapTupleData）をワークスペースとして利用します。
  • off フィールドは slot_deform_heap_tuple 実行時の状態保存に使用されます。
  • 実行エンジンは、このスロットを TupleTableSlot として扱い、TupleTableSlotOps を介して操作します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execTuples.c 内の各スロット操作関数（MinimalTupleTableSlot 用の TupleTableSlotOps 実装など）。
  • **呼び出し先**: 明示的な呼び出し先情報は提供されていません。

関連データ構造

  • TupleTableSlot: 基底構造体。
  • HeapTupleData: minhdr として内部で使用。
  • MinimalTuple: 格納対象のデータ形式。

失敗モードと境界条件

  • mintuple が NULL の場合、スロットは空の状態として扱われます。
  • 属性抽出時に minhdr の設定が不適切な場合、メモリ破壊や不正な値の読み取りが発生する可能性があります。

保守担当者向けメモ

  • src/backend/executor/execTuples.c にある TupleTableSlotOps の実装が、本構造体の具体的な操作ロジックを定義しています。
  • JIT コンパイル（src/backend/jit/llvm/llvmjit_types.c）においても型定義が参照されています。
