spgvalidate

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/spgist/spgvalidate.c:37

目的

SP-GiST インデックスの演算子クラス（opclass）および演算子ファミリー（opfamily）の定義が、PostgreSQL のアクセスメソッド要件を満たしているかを検証する。

シグネチャまたは形状

bool spgvalidate(Oid opclassoid)

入力と出力

  • **入力**: opclassoid (検証対象の演算子クラスの OID)
  • **出力**: bool (検証に合格した場合は true、不整合が検出された場合は false)

前提条件と不変条件

  • 指定された opclassoid がシステムカタログ pg_opclass に存在すること。
  • 検証中、不整合が見つかっても即座にエラー終了せず、ereport(INFO, ...) で情報を出力しつつ result を false に設定して処理を継続する。

アルゴリズムまたは振る舞い

1. **カタログ取得**: opclassoid を基に演算子クラスおよび関連する演算子ファミリーの情報を取得する。 2. **メンバー走査**: 演算子ファミリー内の全サポート関数および演算子を走査する。 3. **関数検証**:

  • 各サポート関数の左右の入力型が一致しているかを確認する。
  • SPGIST_CONFIG_PROC を実行し、インデックス構成情報を取得して、他の関数のシグネチャや戻り値型との整合性を検証する。
  • 各サポート番号（SPGIST_CHOOSE_PROC 等）に応じた期待されるシグネチャを check_amproc_signature 等で確認する。

    • 4. **演算子検証**: 演算子の戦略番号が有効範囲（1〜63）内にあるかを確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgutils.c 内で amroutine->amvalidate に割り当てられている。
  • **呼び出し先**: SearchSysCache1, SearchSysCacheList1, check_amproc_signature, check_amoptsproc_signature, OidFunctionCall2, ereport 等。

関連データ構造

  • Form_pg_opclass, Form_pg_amproc, Form_pg_amop: システムカタログの構造体。
  • spgConfigIn, spgConfigOut: SP-GiST の構成情報をやり取りするための構造体。
  • OpFamilyOpFuncGroup: 演算子ファミリーのグループ化情報。

失敗モードと境界条件

  • **キャッシュルックアップ失敗**: opclassoid が無効な場合、elog(ERROR) で停止する。
  • **型不一致**: サポート関数の左右の型が異なる場合、または config 関数の戻り値と宣言された型が一致しない場合に result が false となる。
  • **無効なサポート番号**: 定義されていないサポート番号が検出された場合、result が false となる。
  • **戦略番号範囲外**: 演算子の戦略番号が 1 未満または 63 を超える場合、検証が失敗する。

保守担当者向けメモ

  • ereport(INFO, ...) を使用しているため、検証結果が false であっても処理は最後まで継続される。
  • 新しいサポート関数を追加する際は、switch 文に該当する SPGIST_*_PROC ケースを追加し、適切なシグネチャチェックを実装すること。
