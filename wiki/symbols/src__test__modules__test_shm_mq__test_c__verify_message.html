verify_message

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_shm_mq/test.c:257

目的

共有メモリメッセージキューを介して送受信されたデータの整合性を検証し、不一致がある場合にエラーを報告する。

シグネチャまたは形状

static void verify_message(Size origlen, char *origdata, Size newlen, char *newdata)

入力と出力

  • origlen: 送信した元のメッセージの長さ。
  • origdata: 送信した元のメッセージデータ。
  • newlen: 受信したメッセージの長さ。
  • newdata: 受信したメッセージデータ。
  • 戻り値: なし。検証に失敗した場合は ereport(ERROR, ...) により処理を中断する。

前提条件と不変条件

  • origdata および newdata は、それぞれ少なくとも origlen および newlen バイトの有効なメモリ領域を指している必要がある。

アルゴリズムまたは振る舞い

1. **長さの比較**: origlen と newlen を比較し、一致しない場合は「message corrupted」としてエラーを発生させる。 2. **内容の比較**: origlen バイト分、origdata と newdata をバイト単位でループ走査する。 3. **不一致の検出**: 異なるバイトが見つかった場合、そのオフセット位置を添えて「message corrupted」としてエラーを発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_shm_mq/test.c 内の test_shm_mq 関数および test_shm_mq_pipelined 関数から呼び出される。
  • **呼び出し先**:
  • PostgreSQL のエラー報告関数 ereport を呼び出す。

関連データ構造

  • 特になし（プリミティブな型とバッファポインタのみを使用）。

失敗モードと境界条件

  • **長さの不一致**: 送受信データのサイズが異なる場合、即座にエラーとなる。
  • **内容の不一致**: データのいずれかのバイトが一致しない場合、最初に見つかった不一致箇所を報告してエラーとなる。

保守担当者向けメモ

  • 本関数は比較処理が重いため、パイプラインテスト等の高負荷な検証を行う際は、呼び出し側で実行頻度を制御するか、オプションで無効化することを検討すること。
