add_stringlist_item

定義

  • 種別: function
  • 定義箇所: src/test/regress/pg_regress.c:196

目的

文字列のリスト（連結リスト）の末尾に新しい要素を追加します。

シグネチャまたは形状

void add_stringlist_item(_stringlist **listhead, const char *str)

入力と出力

  • **入力**:
  • listhead: 文字列リストの先頭ポインタへのポインタ。
  • str: リストに追加する文字列（コピーされます）。
  • **出力**: なし（listhead が指すリストが更新されます）。

前提条件と不変条件

  • listhead は有効なポインタである必要があります。
  • リストの各要素は _stringlist 構造体であり、next ポインタで連結されます。
  • リストの末尾要素の next は必ず NULL である必要があります。

アルゴリズムまたは振る舞い

1. pg_malloc を使用して新しい _stringlist エントリをメモリ確保します。 2. pg_strdup を使用して入力文字列 str を複製し、エントリの str メンバに格納します。 3. 新しいエントリの next を NULL に設定します。 4. listhead が指すリストが空（NULL）であれば、新しいエントリを先頭にします。 5. リストが空でない場合、リストの末尾まで走査し、最後の要素の next に新しいエントリを接続します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/initdb/initdb.c
  • src/interfaces/ecpg/test/pg_regress_ecpg.c

    • （上記はツールによる参照候補であり、直接的な呼び出し関係の網羅性は保証されません）

  • **呼び出し先**:
  • pg_malloc
  • pg_strdup

関連データ構造

  • _stringlist: 文字列を保持する連結リスト構造体。

失敗モードと境界条件

  • メモリ不足時: pg_malloc および pg_strdup の内部実装に従い、通常はプロセスが終了します。
  • 空リストへの追加: listhead が NULL の場合、正しく先頭要素として追加されます。

保守担当者向けメモ

  • 本関数はリストの末尾を毎回走査するため、リストの要素数が増大すると計算コストが増加します。頻繁な追加が必要な場合は、末尾ポインタを保持する構造への変更を検討してください。
