PgStat_FunctionCallUsage

定義

  • 種別: struct
  • 定義箇所: src/include/pgstat.h:91

目的

関数実行時の統計情報を計測・追跡するためのコンテキストを保持する構造体です。関数呼び出しの開始から終了までの経過時間や、その関数に起因する実行時間を記録するために使用されます。

シグネチャまたは形状

typedef struct PgStat_FunctionCallUsage
{
    PgStat_FunctionCounts *fs;
    instr_time  save_f_total_time;
    instr_time  save_total;
    instr_time  start;
} PgStat_FunctionCallUsage;

入力と出力

  • **入力**: 関数呼び出しの開始時に本構造体を初期化し、fs に対応する関数の統計エントリを紐付けます。
  • **出力**: 実行終了時に fs が指す統計情報が更新され、累積時間が加算されます。

前提条件と不変条件

  • fs が NULL の場合、その関数呼び出しは統計対象外であることを示します。
  • fs は関数実行終了時まで有効なハッシュテーブルエントリを指している必要があります。
  • save_f_total_time および save_total は、計測開始時点の累積時間を保持します。

アルゴリズムまたは振る舞い

1. **開始**: 関数呼び出しの直前に構造体を準備し、現在のシステム時刻を start に記録します。 2. **計測**: 関数実行中、save_f_total_time と save_total を用いて、関数固有の実行時間とバックエンド全体の実行時間を算出します。 3. **終了**: pgstat_end_function_usage 等の関数により、start からの経過時間を計算し、fs が指す統計カウンタへ反映させます。

関連データ構造

  • PgStat_FunctionCounts: 関数ごとの統計情報を保持する構造体。本構造体の fs フィールドから参照されます。
  • instr_time: 高精度な時間計測に使用される型。

失敗モードと境界条件

  • fs が NULL の場合、統計更新処理はスキップされます。
  • 関数実行中に統計エントリが削除されると、fs が無効なポインタとなり不正なメモリ参照を引き起こす可能性があります（前提条件としてエントリの生存が必須）。

保守担当者向けメモ

  • 本構造体は関数実行のネストを考慮した計測を行うため、再帰的な関数呼び出しにおいても正しく時間が累積されるよう設計されています。
  • instr_time の精度に依存するため、プラットフォームごとの時間計測実装の影響を受けます。
