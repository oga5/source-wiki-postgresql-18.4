HeapTupleHeaderSetXmin

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/htup_details.h:335

目的

ヒープタプルのヘッダー構造において、挿入トランザクションID（t_xmin）を直接設定する。

シグネチャまたは形状

static inline void HeapTupleHeaderSetXmin(HeapTupleHeaderData *tup, TransactionId xid)

入力と出力

  • tup: 更新対象の HeapTupleHeaderData 構造体へのポインタ。
  • xid: 設定する TransactionId。
  • 出力: なし（tup が指すメモリ領域を直接変更する）。

前提条件と不変条件

  • tup は有効な HeapTupleHeaderData 構造体を指している必要がある。
  • 本関数は t_choice.t_heap.t_xmin フィールドを直接書き換えるため、呼び出し側はタプルの整合性（トランザクション状態の妥当性）を保証する必要がある。

アルゴリズムまたは振る舞い

1. 引数 tup が指すタプルヘッダーの t_choice.t_heap 共用体メンバにアクセスする。 2. t_xmin フィールドに引数 xid を代入する。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • contrib/pg_surgery/heap_surgery.c
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/heapam_xlog.c
  • src/backend/commands/sequence.c
  • src/backend/utils/cache/relcache.c
  • 呼び出し先: なし（インライン関数であり、他の関数を呼び出さない）。

関連データ構造

  • HeapTupleHeaderData: タプルの物理レイアウトを定義する構造体。
  • TransactionId: トランザクションを識別する型。

失敗モードと境界条件

  • tup が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • 呼び出し元で適切なロックや排他制御が行われていない場合、データ競合が発生する可能性がある。

保守担当者向けメモ

  • 本関数は低レベルなアクセサであり、t_infomask 等のフラグ更新は行わない。トランザクション状態の変更に伴うフラグ更新が必要な場合は、呼び出し側で別途処理すること。
