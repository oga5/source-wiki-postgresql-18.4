btree_xlog_mark_page_halfdead

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtxlog.c:716

目的

B-tree インデックスのページを「ハーフデッド（half-dead）」状態にマークする WAL リプレイ処理を実行します。これは、削除対象のサブツリーの親ページを更新し、対象のリーフページを再初期化してハーフデッドフラグを立てる一連の操作を再現します。

シグネチャまたは形状

static void btree_xlog_mark_page_halfdead(uint8 info, XLogReaderState *record)

入力と出力

  • info: WAL レコードのサブタイプ情報。
  • record: リプレイ対象の WAL レコードデータ。
  • 出力: なし（副作用としてバッファの更新とディスクへの書き込み準備を行う）。

前提条件と不変条件

  • リカバリ中であるため、インデックスに対する並行更新は発生しない前提で動作します。
  • ページごとのロック取得・解放で整合性が保たれるものとします。
  • 対象ページが空であるか、削除プロセスの一部であることを前提とします。

アルゴリズムまたは振る舞い

1. **親ページの更新**:

  • XLogReadBufferForRedo を使用して親ページを読み込みます。
  • 親ページ内のダウンリンクを右隣接ページへ付け替え、不要になったエントリを PageIndexTupleDelete で削除します。
  • 更新後のページをダーティとしてマークし、ロックを解放します。

    • 2. **リーフページの再初期化**:

  • XLogInitBufferForRedo を使用して対象のリーフページを初期化します。
  • ページフラグに BTP_HALF_DEAD | BTP_LEAF を設定し、左右の隣接ブロック情報を記録します。
  • BTreeTupleSetTopParent を用いて、トップ親ページを指すダミーのハイキー（high key）をページに追加します。
  • 最終的にページをダーティとしてマークし、ロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtxlog.c 内の btree_redo 関数から、WAL レコードタイプに応じて呼び出されます。
  • **呼び出し先**: XLogReadBufferForRedo, XLogInitBufferForRedo, _bt_pageinit, PageAddItem, PageSetLSN, MarkBufferDirty, UnlockReleaseBuffer 等のバッファ管理およびページ操作関数。

関連データ構造

  • xl_btree_mark_page_halfdead: WAL レコードの構造体。
  • BTPageOpaque: B-tree ページヘッダーのメタデータ。
  • IndexTupleData: インデックス内のタプル構造。

失敗モードと境界条件

  • PageAddItem が InvalidOffsetNumber を返した場合、ダミーハイキーの追加失敗として elog(ERROR) を発生させます。
  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返さない場合、親ページの更新処理はスキップされます。

保守担当者向けメモ

  • 本関数はリカバリ専用のロジックであり、通常のインデックス操作とは異なり、ページ間のロックを跨ぐ必要がないため、個別のロック・アンロック処理で完結しています。
  • BTP_HALF_DEAD フラグの付与は、インデックスの物理的な削除プロセスにおける重要な中間状態です。
