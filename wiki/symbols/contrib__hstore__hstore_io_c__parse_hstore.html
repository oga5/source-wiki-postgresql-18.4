parse_hstore

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_io.c:217

目的

key=>value 形式の文字列を解析し、Pairs 配列へ変換します。

シグネチャまたは形状

static bool parse_hstore(HSParser *state);

入力と出力

  • **入力**: HSParser *state（パース状態を保持する構造体。入力文字列の開始位置やメモリ管理情報を含む）。
  • **出力**: bool（成功時に true、エラーまたはパース完了時に false を返す）。

前提条件と不変条件

  • state->begin に解析対象の文字列が格納されていること。
  • state->escontext が適切に初期化されていること（ソフトエラー処理のため）。
  • Pairs 配列は palloc で確保され、必要に応じて repalloc で拡張される。

アルゴリズムまたは振る舞い

有限オートマトンを用いて文字列をトークン化します。 1. **WKEY**: キーを解析し、Pairs 配列に格納。 2. **WEQ**: => の = を確認。 3. **WGT**: => の > を確認。 4. **WVAL**: 値を解析。null という文字列（エスケープなし）は isnull フラグを立てる。 5. **WDEL**: , を確認し、次のキーへ遷移するか、終端（\0）で終了する。

  • 解析中に hstoreCheckKeyLength および hstoreCheckValLength を呼び出し、制限を超えた場合はエラーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/hstore/hstore_io.c 内の関数（例: hstore_in 等）。
  • **呼び出し先**: get_val（トークン取得）、hstoreCheckKeyLength / hstoreCheckValLength（バリデーション）、palloc / repalloc（メモリ確保）。

関連データ構造

  • HSParser: パース状態を保持。
  • Pairs: キーと値のペアを格納する構造体。

失敗モードと境界条件

  • **構文エラー**: PRSSYNTAXERROR マクロにより、予期しない文字や形式でエラーを報告。
  • **ソフトエラー**: SOFT_ERROR_OCCURRED が真の場合、即座に false を返して終了。
  • **メモリ不足**: palloc/repalloc 失敗時は PostgreSQL の標準的なメモリ管理によりエラーが発生。

保守担当者向けメモ

  • state->pairs の拡張ロジック（plen *= 2）は pcur が plen に達した際に行われます。
  • hstoreUniquePairs 等で後続処理を行うため、Pairs 配列のメモリ管理（needfree フラグ等）には注意してください。
