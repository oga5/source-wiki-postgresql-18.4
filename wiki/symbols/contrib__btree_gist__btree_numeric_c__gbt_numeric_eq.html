gbt_numeric_eq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_numeric.c:44

目的

PostgreSQL の numeric 型に対する等価比較（Equality）を GiST インデックスのコンテキストで提供します。

シグネチャまたは形状

static bool gbt_numeric_eq(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**: 比較対象となる2つの numeric 型データへのポインタ (a, b)、照合順序 (collation)、関数情報 (flinfo)。
  • **出力**: a と b が等しい場合に true、それ以外の場合に false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な numeric 型のデータ構造を指している必要があります。
  • DirectFunctionCall2 を使用するため、PostgreSQL の実行環境が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. 入力されたポインタ a と b を PointerGetDatum を用いて Datum 型に変換します。 2. PostgreSQL コアの numeric_eq 関数を DirectFunctionCall2 経由で呼び出し、比較を実行します。 3. 比較結果を DatumGetBool で bool 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_numeric.c 内の tinfo 構造体（gbtree_vinfo）の初期化時に参照されています（84行目）。
  • **呼び出し先**: PostgreSQL コアの numeric_eq 関数。

関連データ構造

  • gbtree_vinfo: numeric 型の比較関数群を保持する構造体。本関数はこの構造体を通じて GiST インデックスの操作に利用されます。

失敗モードと境界条件

  • numeric_eq は numeric 型の標準的な比較ロジックに従います。NaN（Not-a-Number）の比較については、コアの numeric_eq の仕様に依存します。

保守担当者向けメモ

  • 本関数は btree_gist モジュールにおけるアダプター関数です。ロジックの変更が必要な場合は、コアの numeric_eq の挙動を確認してください。
  • collation および flinfo 引数はシグネチャに含まれていますが、現在の実装では numeric_eq の呼び出しには使用されていません。
