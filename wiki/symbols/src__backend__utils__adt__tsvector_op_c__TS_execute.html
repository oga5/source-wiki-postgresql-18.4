TS_execute

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector_op.c:1853

目的

tsquery を tsvector に対して評価し、クエリが文書にマッチするかどうかを判定するメインのエントリポイントです。

シグネチャまたは形状

bool TS_execute(QueryItem *curitem, void *arg, uint32 flags, TSExecuteCallback chkcond);

入力と出力

  • **入力**:
  • curitem: 評価対象の tsquery ツリーのルートノード。
  • arg: 評価時に使用されるコンテキスト情報（コールバックへ渡される）。
  • flags: 実行オプション（例: TS_EXEC_PHRASE_NO_POS）。
  • chkcond: 語彙の照合を行うためのコールバック関数。
  • **出力**:
  • bool: クエリがマッチした場合は true、それ以外は false を返します。

前提条件と不変条件

  • TS_execute_recurse の戻り値が TS_NO ではない場合に true を返します。
  • 再帰的なクエリツリー処理を行うため、スタックオーバーフローを防ぐための check_stack_depth() が内部的に呼び出されます。

アルゴリズムまたは振る舞い

1. TS_execute_recurse を呼び出し、クエリツリーを再帰的に走査してマッチングを行います。 2. TS_execute_recurse が TS_MAYBE を返した場合、それは TS_EXEC_PHRASE_NO_POS フラグが指定されていることを意味するため、そのまま true として扱います。 3. 最終的な評価結果が TS_NO でないことを確認し、ブール値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/wparser_def.c (ヘッドライン生成、マッチング再確認)
  • src/backend/utils/adt/tsginidx.c (GIN インデックスデータ照合)
  • src/backend/utils/adt/tsgistidx.c (GiST インデックスデータ照合)
  • src/backend/utils/adt/tsrank.c (ランキング計算)
  • **呼び出し先**:
  • TS_execute_recurse (再帰的な評価ロジック)

関連データ構造

  • QueryItem: tsquery のツリー構造を構成するノード。
  • TSExecuteCallback: 語彙照合のための関数ポインタ型。

失敗モードと境界条件

  • クエリツリーが非常に深い場合、スタックオーバーフローが発生する可能性があります。
  • TS_EXEC_PHRASE_NO_POS フラグの有無により、位置情報の評価スキップや TS_MAYBE の解釈が変化します。

保守担当者向けメモ

  • 本関数は TS_execute_recurse へのラッパーとして機能しています。ロジックの変更が必要な場合は、再帰処理を行う TS_execute_recurse を参照してください。
  • TS_execute_locations 等の関連関数と併せて、全文検索の評価パイプラインの一部を構成しています。
