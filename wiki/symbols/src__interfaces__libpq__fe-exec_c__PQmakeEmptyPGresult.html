PQmakeEmptyPGresult

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-exec.c:159

目的

クエリ結果を保持するための PGresult 構造体を初期化し、空の状態として割り当てる。

シグネチャまたは形状

PGresult *PQmakeEmptyPGresult(PGconn *conn, ExecStatusType status);

入力と出力

  • **入力**:
  • conn: 接続情報を持つ PGconn 構造体へのポインタ（NULL 指定可能）。
  • status: 結果のステータスを示す ExecStatusType 値。
  • **出力**:
  • 初期化された PGresult 構造体へのポインタ。メモリ割り当てに失敗した場合は NULL を返す。

前提条件と不変条件

  • conn が提供される場合、その接続の noticeHooks および client_encoding が PGresult にコピーされる。
  • conn が提供されない場合、noticeHooks は NULL に、client_encoding は PG_SQL_ASCII に初期化される。
  • 成功時、PGresult の各フィールド（タプル、属性、イベント等）は空またはゼロ値で初期化される。

アルゴリズムまたは振る舞い

1. malloc を使用して PGresult 構造体分のメモリを確保する。 2. 構造体の各メンバを初期化（ntups=0, numAttributes=0, resultStatus=status 等）。 3. conn が存在する場合:

  • 接続の通知フックとエンコーディング設定をコピーする。
  • エラー系のステータスであれば、pqSetResultError を呼び出し接続側のエラーメッセージを結果に反映させる。
  • 接続にイベントが登録されている場合、dupEvents を使用してイベントを複製する。複製に失敗した場合は PQclear を呼び出してメモリを解放し NULL を返す。

    • 4. conn が存在しない場合、デフォルト値（NULL または PG_SQL_ASCII）を設定する。 5. 構築された PGresult ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/descriptor.c
  • src/interfaces/libpq/fe-exec.c（複数箇所）
  • src/interfaces/libpq/fe-protocol3.c（複数箇所）
  • **呼び出し先**:
  • malloc
  • pqSetResultError
  • dupEvents
  • PQclear

関連データ構造

  • PGresult: クエリ結果を保持する主要な構造体。
  • PGconn: データベース接続状態を保持する構造体。
  • ExecStatusType: クエリ実行結果のステータス列挙型。

失敗モードと境界条件

  • malloc が失敗した場合、NULL を返す。
  • conn からのイベント複製（dupEvents）に失敗した場合、内部で PQclear を実行してメモリを解放し、NULL を返す。

保守担当者向けメモ

  • PGresult は個別のフィールドごとに malloc するのではなく、独自のアロケータ（PQresultAlloc）による管理を前提としている。本関数は初期化のみを行うため、後続のデータ追加時には適切なメモリ管理関数を使用すること。
  • エラー処理において conn->errorMessage を参照する際、ステータスに応じてコピーの要否が分岐するため、新規ステータス追加時は switch 文の確認が必要。
