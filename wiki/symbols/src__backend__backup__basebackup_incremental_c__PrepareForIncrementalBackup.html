PrepareForIncrementalBackup

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_incremental.c:262

目的

増分バックアップの準備フェーズにおいて、バックアップマニフェスト内のWAL範囲とサーバーのタイムライン履歴を照合し、バックアップ開始に必要なメタデータを検証・設定する。

シグネチャまたは形状

void PrepareForIncrementalBackup(IncrementalBackupInfo *ib, BackupState *backup_state)

入力と出力

  • **入力**:
  • ib: 解析済みのマニフェスト情報を含む IncrementalBackupInfo 構造体。
  • backup_state: 現在のバックアップ状態を保持する BackupState 構造体。
  • **出力**: なし（副作用として backup_state の istartpoint および istarttli を更新）。

前提条件と不変条件

  • ib->buf.data が NULL であること（初期化済みであること）。
  • マニフェスト内に少なくとも1つのWAL範囲が存在すること。
  • マニフェストに含まれるすべてのWAL範囲のタイムラインID（TLI）が、サーバーのタイムライン履歴と一致すること。

アルゴリズムまたは振る舞い

1. ib->mcxt へメモリコンテキストを切り替える。 2. readTimeLineHistory を用いてサーバーのタイムライン履歴を取得する。 3. マニフェスト内の各 backup_wal_range を走査し、サーバーの履歴と照合する。 4. 履歴上の最も古いTLIと最新のTLIを特定し、backup_state の増分バックアップ開始点（istartpoint, istarttli）を更新する。 5. 各WAL範囲の開始・終了LSNが、サーバーのタイムライン履歴と矛盾していないか検証する。矛盾がある場合はエラーを発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup.c:289
  • **呼び出し先**: readTimeLineHistory（サーバーのタイムライン履歴取得）、list_nth（リスト操作）、ereport（エラー報告）。

関連データ構造

  • IncrementalBackupInfo: 増分バックアップのメタデータを保持。
  • BackupState: バックアップ実行時の状態管理。
  • backup_wal_range: マニフェストから抽出されたWAL範囲情報。
  • TimeLineHistoryEntry: サーバーのタイムライン履歴情報。

失敗モードと境界条件

  • **マニフェストにWAL範囲がない**: ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラー。
  • **サーバー履歴に存在しないTLI**: マニフェスト内のTLIがサーバー履歴で見つからない場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラー。
  • **LSNの不整合**: マニフェスト内の開始/終了LSNがサーバーのタイムライン開始点と一致しない場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラー。

保守担当者向けメモ

  • 本関数は増分バックアップの整合性チェックの要であり、タイムラインの切り替わりを含むバックアップの検証を行う。
  • 修正を行う際は、backup_wal_range の各フィールドと TimeLineHistoryEntry の境界条件（begin, end）の比較ロジックに注意すること。
