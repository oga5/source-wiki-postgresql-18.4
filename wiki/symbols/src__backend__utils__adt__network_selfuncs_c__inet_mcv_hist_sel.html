inet_mcv_hist_sel

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/network_selfuncs.c:719

目的

MCV（最頻値）リストとヒストグラムを組み合わせて、ネットワーク型（inet/cidr）の選択率を推定します。MCVの各要素に対してヒストグラムを用いた選択率を計算し、それらを重み付け加算することで全体の選択率を算出します。

シグネチャまたは形状

static Selectivity inet_mcv_hist_sel(Datum *mcv_values, float4 *mcv_numbers, int mcv_nvalues,
                                     Datum *hist_values, int hist_nvalues,
                                     int opr_codenum)

入力と出力

  • **入力**:
  • mcv_values: MCVのデータ値配列。
  • mcv_numbers: 各MCVの出現頻度（0.0〜1.0）。
  • mcv_nvalues: MCV配列の要素数。
  • hist_values: ヒストグラムの境界値配列。
  • hist_nvalues: ヒストグラムの要素数。
  • opr_codenum: 演算子を包含関係の数値（-2〜2）に変換したコード。
  • **出力**:
  • Selectivity (float): 推定された選択率（0.0〜1.0）。

前提条件と不変条件

  • inet_hist_value_sel を呼び出す際、ヒストグラムを左辺として扱うため、演算子を反転（opr_codenum = -opr_codenum）させる必要があります。

アルゴリズムまたは振る舞い

1. 演算子コード opr_codenum を反転させ、ヒストグラムとの比較に適した形式に変換します。 2. MCVリストをループし、各要素 mcv_values[i] に対して inet_hist_value_sel を呼び出します。 3. inet_hist_value_sel が返す選択率に、そのMCVの出現頻度 mcv_numbers[i] を乗算します。 4. 全MCVの計算結果を累積し、最終的な選択率として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network_selfuncs.c 内の結合選択率推定ロジック（networkjoinsel 関連の処理）から呼び出されます。
  • **呼び出し先**:
  • inet_hist_value_sel: ヒストグラム内のバケットと定数の包含関係から選択率を推定するために呼び出されます。

関連データ構造

  • Datum: PostgreSQLの汎用データ型表現。
  • Selectivity: 選択率を表す float 型の別名。

失敗モードと境界条件

  • mcv_nvalues が 0 の場合、ループは実行されず、初期値の 0.0 が返されます。
  • hist_nvalues が不適切な場合、inet_hist_value_sel の内部ロジックに依存します。

保守担当者向けメモ

  • 本関数は、MCVとヒストグラムの組み合わせによる選択率計算の「MCV側」の集計を担当します。
  • 演算子の反転は、inet_hist_value_sel の引数順序に合わせるための必須処理です。ロジックを変更する際は、演算子の包含関係（inet_opr_codenum で定義）との整合性に注意してください。
