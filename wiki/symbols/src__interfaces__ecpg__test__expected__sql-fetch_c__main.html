目的

ECPG（Embedded SQL for C）プリプロセッサの変換結果を検証するための回帰テスト用プログラム。特にSQLカーソル操作（DECLARE, FETCH, MOVE, CLOSE）のC言語への変換精度をテストする。

シグネチャまたは形状

int main()

入力と出力

  • 入力: なし（プログラム内でデータベース接続情報を保持）。
  • 出力: 標準出力（printf によるフェッチ結果の表示）、および標準エラー出力（ECPGdebug によるデバッグログ）。

前提条件と不変条件

  • データベース ecpg1_regression が接続可能であること。
  • My_Table テーブルの作成およびデータ挿入が成功すること。
  • 各SQL実行後の sqlca.sqlcode が正常（0以上）であること。負の場合は exit(1) により異常終了する。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、データベースへ接続。 2. **テーブル構築**: My_Table を作成し、4件のレコードを挿入。 3. **カーソル操作**:

  • DECLARE 文でカーソル C を定義。
  • ループ内で FETCH 1 を実行し、結果をホスト変数 i と str に格納して表示。ECPG_NOT_FOUND を検知するとループを終了。
  • MOVE コマンドでカーソル位置を調整。
  • パラメータ付きの FETCH を実行し、結果を表示。
  • CLOSE でカーソルを閉じる。

    • 4. **追加検証**: カーソル D を定義し、条件付きクエリのフェッチを検証。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）。
  • 呼び出し先: ECPGconnect, ECPGdo, sqlprint, exit, printf。

関連データ構造

  • sqlca: SQL通信領域。実行結果のステータスや警告を保持。
  • str (char[25]): SQL結果格納用ホスト変数。
  • i (int): SQL結果格納用ホスト変数。

失敗モードと境界条件

  • sqlca.sqlcode < 0: SQL実行エラー発生時に exit(1) で終了。
  • sqlca.sqlwarn[0] == 'W': SQL警告発生時に sqlprint() を呼び出し。
  • ECPG_NOT_FOUND: FETCH 中にデータが尽きた場合、ループを正常に終了させる。

保守担当者向けメモ

  • 本ファイルは手書きではなく ecpg プリプロセッサの生成物であるため、直接編集せず .pgc ソースを確認すること。
  • ECPGdo の引数はランタイムにSQL文とホスト変数の型情報を渡すための複雑な構造をしている。
