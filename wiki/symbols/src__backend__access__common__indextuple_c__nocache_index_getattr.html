nocache_index_getattr

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/indextuple.c:240

目的

インデックス・タプル（IndexTuple）から特定の属性値を抽出します。属性オフセットのキャッシュ（attcacheoff）を積極的に利用・更新し、アクセスを高速化することを目的としています。

シグネチャまたは形状

Datum nocache_index_getattr(IndexTuple tup, int attnum, TupleDesc tupleDesc)

入力と出力

  • **入力**:
  • tup: 対象の IndexTuple。
  • attnum: 抽出対象の属性番号（1始まり）。
  • tupleDesc: タプルの記述子。属性の長さやアライメント情報を含む。
  • **出力**:
  • 抽出された属性値（Datum）。

前提条件と不変条件

  • tupleDesc は対象の IndexTuple と整合している必要があります。
  • インデックス・タプルはヒープタプルと異なり、原則としてTOASTデータを含みません（TOAST_INDEX_HACKによりインライン化済みであること）。

アルゴリズムまたは振る舞い

1. **NULL判定**: タプルにNULLが含まれる場合、対象属性より前のビットを走査し、NULLの有無を確認します。 2. **キャッシュ利用**:

  • NULLや可変長属性が対象属性までに存在しない場合、TupleDesc 内の attcacheoff を確認し、有効であれば即座に値を返します。
  • キャッシュがない場合、固定長属性であればオフセットを計算してキャッシュを更新し、後続のアクセスに備えます。

    • 3. **低速パス**: NULLや可変長属性が存在する場合、タプルを先頭から順に走査して対象属性のオフセットを特定します。この際、可能な範囲でキャッシュの更新を試みます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/itup.h 内のインライン関数（index_getattr 等）から頻繁に呼び出されます。
  • **呼び出し先**: TupleDescCompactAttr マクロ、fetchatt マクロ、att_isnull マクロ、att_nominal_alignby 等の属性操作ユーティリティ。

関連データ構造

  • IndexTuple: インデックス行のメモリ表現。
  • TupleDesc: 属性定義情報。attcacheoff を保持し、高速化の鍵となります。
  • CompactAttribute: 属性の物理レイアウト情報。

失敗モードと境界条件

  • attnum が範囲外の場合の挙動は未定義（呼び出し元で保証される前提）。
  • 可変長属性（attlen == -1）が連続する場合、アライメントの制約によりキャッシュが利用できないケースがあります。

保守担当者向けメモ

  • 本関数は heaptuple.c にある同名のヒープ用関数とロジックが類似していますが、インデックス特有のヘッダー構造（IndexTupleData）を扱う点に注意してください。
  • キャッシュ更新ロジックはパフォーマンスに直結するため、変更時は attcacheoff の整合性に十分注意してください。
