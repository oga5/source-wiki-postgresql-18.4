citext_eq

定義

  • 種別: function
  • 定義箇所: contrib/citext/citext.c:188

目的

citext 型の2つの値が等しいかどうかを、大文字・小文字を区別せずに判定する。

シグネチャまたは形状

Datum citext_eq(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される2つの text 型データ。
  • **出力**: PG_RETURN_BOOL を介して、等価であれば true、そうでなければ false を返す。

前提条件と不変条件

  • 入力データは text 型として解釈可能であること。
  • str_tolower 関数により、比較前に両方の入力が小文字に変換される。
  • 変換後の文字列比較には DEFAULT_COLLATION_OID が使用される。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を使用して入力引数を取得する。 2. str_tolower を呼び出し、各入力文字列を小文字に変換した一時的な C 文字列（lcstr, rcstr）を生成する。 3. strcmp を使用して、変換後の文字列同士をビット単位で比較する。照合順序（strcoll）を介さないため、効率的な比較が可能である。 4. pfree を呼び出して一時的な文字列メモリを解放する。 5. PG_FREE_IF_COPY を使用して、必要に応じて入力データのコピーを解放する。 6. 比較結果をブール値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(citext_eq) マクロにより、PostgreSQL の関数マネージャから呼び出される。
  • **呼び出し先**: str_tolower, pfree, PG_GETARG_TEXT_PP, PG_FREE_IF_COPY, PG_RETURN_BOOL, strcmp。

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • 入力文字列が TOAST されている場合、PG_GETARG_TEXT_PP が適切に展開を行う。
  • メモリ不足時には pfree や変換処理でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は strcmp を使用して最適化されているため、照合順序に依存した複雑な比較は行わない。
  • str_tolower に渡す照合順序は DEFAULT_COLLATION_OID に固定されている。
