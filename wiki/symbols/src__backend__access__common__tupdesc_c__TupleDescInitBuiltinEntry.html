TupleDescInitBuiltinEntry

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tupdesc.c:917

目的

データベース接続が確立されていないプロセスやカタログアクセスが不可能な環境下でも、基本的な組み込み型を用いたタプル記述子（TupleDesc）の属性を初期化するために使用されます。

シグネチャまたは形状

void TupleDescInitBuiltinEntry(TupleDesc desc, AttrNumber attributeNumber, const char *attributeName, Oid oidtypeid, int32 typmod, int attdim)

入力と出力

  • **入力**:
  • desc: 初期化対象の TupleDesc。
  • attributeNumber: 属性番号（1始まり）。
  • attributeName: 属性名（NULL不可）。
  • oidtypeid: 属性の型ID（TEXTOID, BOOLOID, INT4OID, INT8OID, OIDOID のいずれか）。
  • typmod: 型修飾子。
  • attdim: 配列の次元数。
  • **出力**: なし（副作用として desc 内の属性情報が更新されます）。

前提条件と不変条件

  • desc が有効なポインタであること。
  • 1 <= attributeNumber <= desc->natts であること。
  • 0 <= attdim <= PG_INT16_MAX であること。
  • attributeName が NULL でないこと。

アルゴリズムまたは振る舞い

1. 指定された属性番号に対応する Form_pg_attribute 構造体を取得します。 2. 属性名、型修飾子、属性番号、次元数を設定します。 3. attnotnull、atthasdef、attisdropped 等のフラグをデフォルト値（主に false または空）に初期化します。 4. oidtypeid に基づき、attlen、attbyval、attalign、attstorage、attcollation などの型固有の属性情報を設定します。 5. 最後に populate_compact_attribute を呼び出し、最適化用の CompactAttribute を同期させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_copy.c
  • src/backend/replication/walsender.c
  • **呼び出し先**:
  • populate_compact_attribute（内部関数）
  • namestrcpy（マクロ）
  • elog（エラー発生時）

関連データ構造

  • TupleDesc: タプルの構造を保持する主要な構造体。
  • Form_pg_attribute: 属性のメタデータを保持する構造体。

失敗モードと境界条件

  • サポートされていない oidtypeid が渡された場合、elog(ERROR, ...) により処理が中断されます。
  • 属性番号が範囲外の場合、アサーション違反が発生します。

保守担当者向けメモ

  • 本関数はカタログアクセスを前提としないため、対応する型は switch 文で明示的に定義する必要があります。新しい型を追加する場合は、対応する属性情報を正確に設定してください。
  • FormData_pg_attribute のフィールドを直接操作しているため、構造体の定義変更時には本関数の初期化ロジックも確認してください。
