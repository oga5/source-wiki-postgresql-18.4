BackgroundWorker

定義

  • **種別**: struct
  • **定義箇所**: src/include/postmaster/bgworker.h:89

目的

PostgreSQLのプロセス管理下で実行されるバックグラウンドワーカープロセスの属性および起動設定を保持するための構造体です。

シグネチャまたは形状

typedef struct BackgroundWorker
{
	char		bgw_name[BGW_MAXLEN];
	char		bgw_type[BGW_MAXLEN];
	int			bgw_flags;
	BgWorkerStartTime bgw_start_time;
	int			bgw_restart_time;
	char		bgw_library_name[MAXPGPATH];
	char		bgw_function_name[BGW_MAXLEN];
	Datum		bgw_main_arg;
	char		bgw_extra[BGW_EXTRALEN];
	pid_t		bgw_notify_pid;
} BackgroundWorker;

入力と出力

  • **入力**: 拡張機能やモジュールが本構造体を初期化し、RegisterDynamicBackgroundWorker 等を通じてPostmasterに登録します。
  • **出力**: Postmasterはこの構造体の設定に基づき、指定されたライブラリ内の関数を呼び出し、ワーカープロセスを生成・管理します。

前提条件と不変条件

  • bgw_restart_time には秒数、または BGW_NEVER_RESTART を指定する必要があります。
  • 共有メモリへのアクセスが必要な場合は、bgw_flags に BGWORKER_SHMEM_ACCESS を含める必要があります。

アルゴリズムまたは振る舞い

1. **登録**: 構造体がPostmasterに渡されると、Postmasterは内部リストで管理します。 2. **起動**: bgw_start_time の条件を満たすと、Postmasterが fork() を実行します。 3. **実行**: ワーカープロセスは bgw_library_name で指定された共有ライブラリをロードし、bgw_function_name で指定された関数を bgw_main_arg を引数として実行します。 4. **終了と再起動**: プロセスの終了コードに基づき、Postmasterが再起動の要否を判断します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_prewarm/autoprewarm.c、src/backend/access/transam/parallel.c、src/backend/postmaster/bgworker.c 等でインスタンス化および参照が行われています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。ワーカープロセス内で bgw_function_name に指定された関数が実行されます。

関連データ構造

  • BgWorkerStartTime: 起動タイミングを定義する列挙型。
  • BackgroundWorkerHandle: ワーカーの状態監視に使用されるハンドル。

失敗モードと境界条件

  • bgw_name や bgw_type は BGW_MAXLEN を超えることはできません。
  • bgw_library_name は MAXPGPATH を超えることはできません。
  • SanityCheckBackgroundWorker 関数により、登録時に構造体の内容が検証されます。

保守担当者向けメモ

  • 本構造体はPostmasterとワーカープロセス間の契約を定義します。変更時はPostmaster側の管理ロジック（src/backend/postmaster/bgworker.c）との整合性に注意してください。
  • bgw_notify_pid はプロセス開始/終了時に SIGUSR1 を送信するために使用されます。
