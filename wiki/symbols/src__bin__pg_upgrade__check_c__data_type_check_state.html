data_type_check_state

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_upgrade/check.c:321

目的

pg_upgrade におけるデータ型の互換性チェック処理の実行状態を保持し、各データベースに対する検証結果を追跡するために使用されます。

シグネチャまたは形状

struct data_type_check_state
{
	DataTypesUsageChecks *check;	/* 実行中のチェック定義 */
	bool		result;			/* いずれかのデータベースでチェックが失敗した場合に true */
	PQExpBuffer *report;		/* 失敗したチェックのレポート用バッファ */
};

入力と出力

  • 入力: DataTypesUsageChecks 構造体へのポインタおよびレポート用の PQExpBuffer。
  • 出力: チェック実行の結果（result）および蓄積されたエラーレポート。

前提条件と不変条件

  • check メンバは有効な DataTypesUsageChecks 構造体を指している必要があります。
  • report メンバは、チェック失敗時の詳細を記録するために初期化された PQExpBuffer を参照する必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、check_for_data_types_usage 関数内で各チェックステップの状態管理に使用されます。
  • result は、対象となるデータベースのいずれかで互換性違反が検出された場合に true に設定され、アップグレードの可否判断に利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_upgrade/check.c 内の check_for_data_types_usage 関数において、pg_malloc0 を用いてメモリ確保および初期化が行われています。
  • 呼び出し先: 構造体自体はデータ保持用であり、関数呼び出しの引数として process_data_type_check 等へ渡されることが示唆されています。

関連データ構造

  • DataTypesUsageChecks: チェック対象のデータ型やクエリを定義する構造体。
  • PQExpBuffer: PostgreSQLの文字列バッファ構造体。

失敗モードと境界条件

  • メモリ確保失敗時: pg_malloc0 が失敗した場合、プログラムは終了します。
  • result の初期値は false ですが、チェック処理中にエラーが検出されると true に更新されます。

保守担当者向けメモ

  • 本構造体は check_for_data_types_usage のループ内で動的に配列として確保されます。配列のサイズは n_data_types_usage_checks に依存します。
