GMReaderTupleBuffer

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeGatherMerge.c:41

目的

並列クエリ実行において、各ワーカープロセスから送られてくるタプルを一時的に保持し、リーダープロセス側でのマージ処理を効率化するためのバッファ構造体です。

シグネチャまたは形状

typedef struct GMReaderTupleBuffer
{
	MinimalTuple *tuple;		/* array of length MAX_TUPLE_STORE */
	int			nTuples;		/* number of tuples currently stored */
	int			readCounter;	/* index of next tuple to extract */
	bool		done;			/* true if reader is known exhausted */
} GMReaderTupleBuffer;

入力と出力

  • **入力**: ワーカープロセスから受信した MinimalTuple の配列。
  • **出力**: readCounter を介して、順次取り出される個別の MinimalTuple。

前提条件と不変条件

  • tuple 配列の長さは MAX_TUPLE_STORE に固定されています。
  • nTuples は常に 0 以上 MAX_TUPLE_STORE 以下である必要があります。
  • done が true の場合、そのリーダーからのタプル供給は終了したことを示します。

アルゴリズムまたは振る舞い

1. **バッファリング**: ワーカーから一度に最大 MAX_TUPLE_STORE 個のタプルを読み込み、配列に格納します。 2. **順次読み出し**: readCounter をインクリメントしながら、バッファ内のタプルを順次消費します。 3. **補充**: バッファが空になった場合、再度ワーカーからタプルを取得し、nTuples を更新します。 4. **終了判定**: ワーカーからのストリームが枯渇した際、done フラグを立ててマージ処理に通知します。

呼び出し元と呼び出し先

  • **呼び出し元**: nodeGatherMerge.c 内の ExecGatherMerge や gather_merge_getnext 等が、状態管理のために本構造体を参照・操作します。
  • **呼び出し先**: 具体的な関数呼び出しの詳細は提供されていません。

関連データ構造

  • MinimalTuple: バッファに格納されるタプルの型。
  • GatherMergeState (推定): gm_tuple_buffers フィールドを通じて、本構造体の配列を保持します。

失敗モードと境界条件

  • **バッファ枯渇**: nTuples が 0 になった際、ワーカーからの入力が終了している場合は done が true となり、それ以上の読み出しは行われません。
  • **メモリ制約**: MAX_TUPLE_STORE を超えるタプルを一度に保持することはできません。

保守担当者向けメモ

  • 本構造体は nodeGatherMerge.c 内でプライベートに使用されることを意図しています。
  • MAX_TUPLE_STORE の値は、メモリ消費量とコンテキストスイッチ回数のトレードオフを決定するため、パフォーマンスチューニングの際は留意してください。
