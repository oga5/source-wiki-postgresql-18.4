toast_open_indexes

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/common/toast_internals.c:561

目的

TOASTテーブルに関連付けられたすべてのインデックスをオープンし、その中で最初に見つかった有効なインデックスのインデックス番号を返します。

シグネチャまたは形状

int toast_open_indexes(Relation toastrel, LOCKMODE lock, Relation **toastidxs, int *num_indexes)

入力と出力

  • **入力**:
  • toastrel: 対象のTOASTテーブルのリレーション。
  • lock: インデックスオープン時に適用するロックモード。
  • toastidxs: オープンされたインデックスリレーションの配列を格納するポインタ。
  • num_indexes: オープンされたインデックスの総数を格納するポインタ。
  • **出力**:
  • 戻り値: toastidxs 配列内における、最初に見つかった有効なインデックスのインデックス番号（int）。

前提条件と不変条件

  • toastrel は有効なTOASTテーブルである必要があります。
  • TOASTテーブルには少なくとも1つの有効なインデックスが存在しなければなりません。存在しない場合、エラーが発生します。

アルゴリズムまたは振る舞い

1. RelationGetIndexList を使用して、TOASTテーブルに関連するインデックスのOIDリストを取得します。 2. 取得したリストの長さ分だけ toastidxs 配列をメモリ確保（palloc）します。 3. リスト内の各インデックスを index_open で順次オープンし、配列に格納します。 4. 配列を走査し、rd_index->indisvalid が真である最初のインデックスを探します。 5. インデックスリストを解放し、見つかった有効なインデックスの番号を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/toast_internals.c (複数箇所)
  • src/backend/access/heap/heaptoast.c
  • contrib/amcheck/verify_heapam.c
  • **呼び出し先**:
  • RelationGetIndexList
  • palloc
  • index_open
  • list_free
  • elog (エラー発生時)

関連データ構造

  • Relation: リレーション記述子。
  • List: インデックスOIDのリスト。

失敗モードと境界条件

  • **エラー**: TOASTテーブルに有効なインデックスが1つも存在しない場合、elog(ERROR, ...) によりトランザクションが中断されます。

保守担当者向けメモ

  • 本関数でオープンしたインデックスは、呼び出し側で toast_close_indexes を使用して適切にクローズする必要があります。
  • toastidxs 配列のメモリは呼び出し側で管理されるため、不要になった時点で解放してください。
