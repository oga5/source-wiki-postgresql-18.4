copy_subdir_files

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.c:724

目的

旧クラスタの特定のサブディレクトリ（pg_multixact 等）の内容を、新クラスタの対応するディレクトリへ物理的にコピーする。

シグネチャまたは形状

static void copy_subdir_files(const char *old_subdir, const char *new_subdir)

入力と出力

  • **入力**:
  • old_subdir: 旧クラスタのデータディレクトリからの相対パス。
  • new_subdir: 新クラスタのデータディレクトリからの相対パス。
  • **出力**: なし（副作用としてファイルシステム上のコピーを実行）。

前提条件と不変条件

  • グローバル変数 old_cluster.pgdata および new_cluster.pgdata が初期化されていること。
  • 呼び出し前に、新クラスタ側の対象ディレクトリが remove_new_subdir によって削除可能であること。

アルゴリズムまたは振る舞い

1. remove_new_subdir(new_subdir, true) を呼び出し、新クラスタ側の既存ディレクトリを削除する。 2. old_cluster.pgdata と old_subdir を結合し、コピー元のフルパスを生成する。 3. new_cluster.pgdata と new_subdir を結合し、コピー先のフルパスを生成する。 4. prep_status で進捗を表示する。 5. OSに応じて以下のコマンドを実行する:

  • **非Windows**: cp -Rf を使用して再帰的にコピー。
  • **Windows**: xcopy /e /y /q /r を使用して再帰的にコピー。

    • 6. exec_prog を介して外部コマンドを実行し、check_ok() で成功を確認する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/pg_upgrade.c 内のアップグレード処理フロー（pg_multixact/offsets および pg_multixact/members の移行時など）。
  • **呼び出し先**:
  • remove_new_subdir
  • prep_status
  • exec_prog
  • check_ok

関連データ構造

  • old_cluster (グローバル変数)
  • new_cluster (グローバル変数)

失敗モードと境界条件

  • コピー元ディレクトリが存在しない場合、またはコピー先への書き込み権限がない場合、exec_prog がエラーを報告し、アップグレードプロセスが中断される可能性がある。
  • Windows環境では xcopy のフラグにより、読み取り専用属性のファイルも上書きされる。

保守担当者向けメモ

  • 本関数は物理的なファイルコピーを行うため、データ量が多い場合は時間がかかる可能性がある。
  • Windows環境でのパス区切り文字（\）の扱いに注意が必要。
  • cp -Rf や xcopy の挙動に依存しているため、プラットフォーム固有の制約を受ける。
