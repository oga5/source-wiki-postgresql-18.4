optimal_k

定義

  • 種別: function
  • 定義箇所: src/backend/lib/bloomfilter.c:228

目的

ブルームフィルタにおいて、偽陽性率を最小化するための最適なハッシュ関数の個数（k）を算出します。

シグネチャまたは形状

static int optimal_k(uint64 bitset_bits, int64 total_elems)

入力と出力

  • **入力**:
  • bitset_bits: ビットセットの総ビット数。
  • total_elems: フィルタに格納する要素の総数。
  • **出力**:
  • 計算された最適なハッシュ関数の数（int 型）。

前提条件と不変条件

  • MAX_HASH_FUNCS（10）がハッシュ関数の上限として適用されます。
  • 計算結果は必ず 1 以上 MAX_HASH_FUNCS 以下の範囲に収められます。

アルゴリズムまたは振る舞い

1. 理論上の最適値 $k = \ln(2) \times (\text{bitset\_bits} / \text{total\_elems})$ を計算します。 2. rint() 関数を用いて計算結果を整数に丸めます。 3. Max(1, Min(k, MAX_HASH_FUNCS)) を適用し、値を 1 から 10 の範囲にクランプして返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/lib/bloomfilter.c 内の bloom_create 関数（115行目）から呼び出されます。
  • **呼び出し先**: math.h の log および rint 関数、ならびに PostgreSQL の Max および Min マクロを使用します。

関連データ構造

  • bloom_filter: 本関数で算出された k は、この構造体の k_hash_funcs メンバに格納されます。

失敗モードと境界条件

  • total_elems が 0 または非常に小さい場合、浮動小数点演算の結果が極端に大きくなる可能性がありますが、Min マクロと MAX_HASH_FUNCS により 10 に制限されます。
  • bitset_bits が 0 の場合、計算結果は 0 となりますが、Max(1, ...) により最小値 1 が保証されます。

保守担当者向けメモ

  • 本関数は bloom_create によるフィルタ初期化時にのみ使用されます。
  • MAX_HASH_FUNCS の値を変更する場合は、この関数の制約条件も併せて確認してください。
