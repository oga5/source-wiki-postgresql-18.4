IsoLocaleName

定義

  • 種別: static function
  • 定義箇所: src/backend/utils/adt/pg_locale.c:1054

目的

Windows 環境におけるロケール名を、PostgreSQL が内部的に扱う標準的な ISO ロケール名形式へ変換します。

シグネチャまたは形状

static char *IsoLocaleName(const char *winlocname);

入力と出力

  • **入力**: winlocname (Windows 形式のロケール名文字列)
  • **出力**: 変換後のロケール名を示す char * 型のポインタ。

前提条件と不変条件

  • winlocname が "C" または "POSIX"（大文字小文字を区別しない）の場合、静的バッファ iso_lc_messages を使用して "C" を返します。
  • それ以外の場合は get_iso_localename(winlocname) を呼び出し、その結果を返します。

アルゴリズムまたは振る舞い

1. 入力された winlocname が "c" または "posix" と一致するかを pg_strcasecmp で判定します。 2. 一致する場合、静的領域 iso_lc_messages に "C" をコピーし、そのアドレスを返します。 3. 一致しない場合、get_iso_localename 関数へ処理を委譲します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale.c 内のロケール設定処理（行 262 付近）。
  • src/bin/initdb/initdb.c（コメント内での言及）。
  • **呼び出し先**:
  • pg_strcasecmp
  • get_iso_localename
  • strcpy

関連データ構造

  • iso_lc_messages: 関数内で定義された static char 配列。LOCALE_NAME_MAX_LENGTH のサイズを持ちます。

失敗モードと境界条件

  • winlocname が NULL の場合の挙動については、本ソースコードからは明示されていません。
  • get_iso_localename が失敗した場合の挙動は、当該関数の実装に依存します。

保守担当者向けメモ

  • 本関数は Windows 環境特有のロケール名変換を吸収するために存在します。
  • 静的バッファ iso_lc_messages を使用しているため、スレッドセーフな環境での呼び出しには注意が必要です。
