xmltype

定義

  • 種別: struct
  • 定義箇所: src/include/utils/xml.h:23

目的

PostgreSQLにおいてXMLデータを保持するためのデータ型定義です。

シグネチャまたは形状

typedef struct varlena xmltype;

varlena 型のエイリアスとして定義されています。

入力と出力

  • 入力: SQLの text 型や他の型から変換されたXMLデータ。
  • 出力: xmltype 型として保持されたXMLデータ。

前提条件と不変条件

  • varlena 型であるため、PostgreSQLの可変長データ型のメモリレイアウト規則に従います。
  • text 型と同一の内部構造を持つことが前提とされています。

アルゴリズムまたは振る舞い

  • XMLデータの格納コンテナとして機能します。
  • libxml2 を利用したパースや生成処理の対象データとして使用されます。
  • DatumGetXmlP や PG_GETARG_XML_P マクロを通じて、Datum 形式から安全に取り出されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/executor/execExprInterp.c
  • src/backend/utils/adt/xml.c（xml_out_internal 等の関数で引数や戻り値として使用）
  • 呼び出し先:
  • cstring_to_text や cstring_to_text_with_len 等の varlena 操作関数（xml.c 内の変換処理にて）。

関連データ構造

  • varlena: 基底となる可変長データ構造。
  • text: 内部構造が共通であるため、相互変換やキャストが頻繁に行われます。

失敗モードと境界条件

  • libxml2 が無効なビルド環境では、本型を扱う関数群がエラーを返す可能性があります。
  • メモリ管理において、PostgreSQLの palloc と libxml2 内部のメモリ割り当てが競合しないよう注意が必要です。

保守担当者向けメモ

  • 本型は text 型と構造的に同一であるため、型変換やポインタキャストを行う際はその前提が維持されていることを確認してください。
  • xml.c 内の変換ロジックを変更する場合は、varlena のヘッダー情報が正しく保持されるか注意してください。
