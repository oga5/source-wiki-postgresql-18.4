dshash_partition

定義

  • 種別: struct
  • 定義箇所: src/backend/lib/dshash.c:73

目的

動的共有メモリ（DSA）上に構築されたハッシュテーブルにおいて、並行アクセスを制御するためのパーティション単位の管理構造体です。

シグネチャまたは形状

typedef struct dshash_partition
{
    LWLock      lock;
    size_t      count;
} dshash_partition;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入出力はありません。各フィールドは以下の通りです。

  • lock: パーティション内の全バケットを保護する LWLock。
  • count: 当該パーティションに属するバケット内のエントリ総数。

前提条件と不変条件

  • dshash_table_control 内に DSHASH_NUM_PARTITIONS 個の配列として配置されます。
  • パーティション内の操作を行う際は、対応する lock を取得する必要があります。
  • 複数のロックを取得する必要がある場合は、デッドロックを回避するためパーティション番号の昇順で取得しなければなりません。

アルゴリズムまたは振る舞い

  • **並行制御**: ハッシュテーブル全体を単一のロックで保護するのではなく、パーティション単位でロックを細分化することで競合を最小限に抑えます。
  • **リサイズ時の挙動**: ハッシュテーブルのリサイズ時には、全パーティションのロックを一時的に取得してバケットの再配置を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/lib/dshash.c 内の各関数（dshash_find_or_insert 等）が、ハッシュテーブル操作時にこの構造体を参照・ロックします。
  • **呼び出し先**: LWLock 関連のライブラリ（storage/lwlock.h）を介してロック操作を行います。

関連データ構造

  • dshash_table_control: 本構造体を配列として保持する共有メモリ上の制御ブロック。
  • dshash_table: バックエンドローカルなハッシュテーブルの状態管理構造体。

失敗モードと境界条件

  • **ロック競合**: 頻繁なリサイズが発生すると全パーティションのロックが取得されるため、パフォーマンス低下を招きます。
  • **デッドロック**: 複数のパーティションを操作する際に昇順ルールを逸脱するとデッドロックが発生する可能性があります。

保守担当者向けメモ

  • 共有メモリ上に配置されるため、直接的なメモリポインタの保持は避けてください。
  • count フィールドはパーティション内のエントリ数を追跡するために使用されます。
