TouchSocketFiles

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqcomm.c:829

目的

Unixドメインソケットファイルに対して utime を実行し、最終アクセス時刻を更新することで、/tmp ディレクトリのクリーナー（自動削除プロセス）によるソケットファイルの削除を防止します。

シグネチャまたは形状

void TouchSocketFiles(void)

入力と出力

  • 入力: なし（内部的なグローバルリスト sock_paths を参照）
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • sock_paths リストに、監視対象となる有効なソケットファイルのパスが格納されている必要があります。

アルゴリズムまたは振る舞い

1. sock_paths リストを反復処理します。 2. 各要素（ソケットファイルのパス）に対して utime(sock_path, NULL) を呼び出します。 3. utime の実行中に発生したエラーはすべて無視されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/postmaster/postmaster.c:1798
  • 呼び出し先:
  • utime (システムコール)

関連データ構造

  • sock_paths: ソケットファイルのパスを保持する List 型のグローバル変数。

失敗モードと境界条件

  • utime が失敗した場合（ファイルが存在しない、権限がない等）でも、エラー報告を行わず処理を継続します。

保守担当者向けメモ

  • 本関数は、PostgreSQLのバックエンドが作成したUnixドメインソケットが、OS側の定期的なクリーンアップ処理によって誤って削除されるのを防ぐための生存維持メカニズムです。
  • sock_paths の管理状況については、pqcomm.c 内のソケット作成処理と併せて確認してください。
