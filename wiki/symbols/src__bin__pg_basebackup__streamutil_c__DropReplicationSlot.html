DropReplicationSlot

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/streamutil.c:696

目的

PostgreSQLサーバー上の指定されたレプリケーションスロットを削除します。

シグネチャまたは形状

bool DropReplicationSlot(PGconn *conn, const char *slot_name)

入力と出力

  • **入力**:
  • conn: サーバーへの有効な PGconn 接続ポインタ。
  • slot_name: 削除対象のスロット名（NULL不可）。
  • **出力**:
  • 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • slot_name は NULL であってはなりません（Assert により検証）。
  • 接続 conn はレプリケーションプロトコルが利用可能な状態である必要があります。

アルゴリズムまたは振る舞い

1. PQExpBuffer を用いて DROP_REPLICATION_SLOT "slot_name" コマンド文字列を構築します。 2. PQexec を介してサーバーへコマンドを送信します。 3. サーバーからの応答が PGRES_COMMAND_OK であることを確認します。 4. 応答結果にタプルやフィールドが含まれていないことを確認します（期待値は 0 行 0 フィールド）。 5. 処理終了後、バッファと結果セットを解放し、成功または失敗を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_receivewal.c
  • src/bin/pg_basebackup/pg_recvlogical.c
  • ※注: src/backend/replication/walsender.c に同名の関数が存在しますが、これはバックエンド側の実装であり、本関数（フロントエンド用）とは別物です。
  • **呼び出し先**:
  • libpq 関数群 (PQexec, PQresultStatus, PQerrorMessage, PQntuples, PQnfields, PQclear)
  • pqexpbuffer 関数群 (createPQExpBuffer, appendPQExpBuffer, destroyPQExpBuffer)
  • pg_log_error (エラーログ出力)

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • PQExpBuffer: コマンド文字列構築用バッファ。
  • PGresult: サーバーからの実行結果保持用構造体。

失敗モードと境界条件

  • PQexec が PGRES_COMMAND_OK 以外を返した場合、エラーログを出力し false を返します。
  • サーバーから予期せぬデータ（タプルやフィールド）が返された場合、エラーログを出力し false を返します。
  • slot_name が NULL の場合、アサーション違反により異常終了します。

保守担当者向けメモ

  • 本関数はフロントエンドツール用です。バックエンドの DropReplicationSlot と混同しないよう注意してください。
  • 削除対象のスロット名にはダブルクォーテーションによるエスケープ処理が含まれています。
