adjust_child_relids

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/appendinfo.c:577

目的

クエリ最適化において、リレーションIDの集合（Relids）に含まれる親テーブルのIDを、対応する子テーブルのIDに置換します。

シグネチャまたは形状

Relids adjust_child_relids(Relids relids, int nappinfos, AppendRelInfo **appinfos)

入力と出力

  • **入力**:
  • relids: 変換対象となるリレーションIDのビットマップセット。
  • nappinfos: appinfos 配列の要素数。
  • appinfos: 親子関係を定義する AppendRelInfo 構造体の配列。
  • **出力**:
  • 変換後の Relids。変更が発生した場合は新しいビットマップセットを返し、変更がない場合は元の relids をそのまま返します。

前提条件と不変条件

  • appinfos 配列内の各 AppendRelInfo 構造体は有効である必要があります。
  • relids は bms（Bitmapset）形式の集合です。

アルゴリズムまたは振る舞い

1. appinfos 配列を順次走査します。 2. 各 AppendRelInfo について、その parent_relid が入力 relids に含まれているかを確認します。 3. 含まれている場合、初めての変更であれば bms_copy で relids のコピーを作成します。 4. コピーされた集合から parent_relid を削除し、child_relid を追加します。 5. 走査終了後、変更が行われていれば新しい集合を、そうでなければ元の集合を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/joinrels.c: joinrel->relids や SpecialJoinInfo の各フィールドの調整に使用。
  • src/backend/optimizer/util/appendinfo.c: PlaceHolderVar の phrels や RestrictInfo の各種リレーションIDセットの調整に使用。
  • **呼び出し先**:
  • bms_is_member, bms_copy, bms_del_member, bms_add_member（いずれも Bitmapset 操作関数）。

関連データ構造

  • Relids: リレーションIDを保持するビットマップセット。
  • AppendRelInfo: 親子関係の変換情報を保持する構造体。

失敗モードと境界条件

  • relids が空の場合、ループ内で bms_is_member が偽となるため、そのまま空の集合が返されます。
  • appinfos に該当する親IDが含まれない場合、元の relids がそのまま返されます。

保守担当者向けメモ

  • 本関数は Relids の置換を行うためのユーティリティであり、副作用として新しい Bitmapset を割り当てる可能性があるため、呼び出し側でのメモリ管理に注意してください。
  • 複数レベルの継承関係を扱う場合は、adjust_child_relids_multilevel の使用を検討してください。
