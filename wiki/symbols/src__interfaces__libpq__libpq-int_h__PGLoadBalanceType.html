PGLoadBalanceType

定義

  • 種別: enum
  • 定義箇所: src/interfaces/libpq/libpq-int.h:247

目的

データベース接続時のホスト選択順序を制御するためのロードバランシング戦略を定義します。

シグネチャまたは形状

typedef enum
{
	LOAD_BALANCE_DISABLE = 0,
	LOAD_BALANCE_RANDOM,
} PGLoadBalanceType;

入力と出力

  • **入力**: 接続パラメータとして設定されるロードバランスの指定。
  • **出力**: PGconn 構造体内の load_balance_type フィールドとして保持されます。

前提条件と不変条件

  • LOAD_BALANCE_DISABLE がデフォルト値として使用されます。
  • 接続試行時に、この設定に基づいてホストリストの処理順序が決定されます。

アルゴリズムまたは振る舞い

  • LOAD_BALANCE_DISABLE: 接続文字列等で指定された既存のホスト順序を維持します。
  • LOAD_BALANCE_RANDOM: 接続対象のホストリストをランダムにシャッフルします。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/libpq-int.h:533 にて PGconn 構造体のメンバ load_balance_type として参照されています。
  • 呼び出し先: 提供されたソース excerpt 内には、この列挙型を使用する具体的な関数呼び出しの記述はありません。

関連データ構造

  • PGconn: 本列挙型を保持するメインの接続状態構造体。

失敗モードと境界条件

  • 該当なし（列挙型定義のため、直接的な失敗モードは存在しません）。

保守担当者向けメモ

  • 本シンボルは libpq 内部でのみ使用されるプライベートな定義です。
  • 接続時のホスト選択ロジックを変更または拡張する場合は、この列挙型に新しい戦略を追加することを検討してください。
