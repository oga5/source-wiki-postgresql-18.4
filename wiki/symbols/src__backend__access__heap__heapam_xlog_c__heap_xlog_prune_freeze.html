heap_xlog_prune_freeze

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_xlog.c:29

目的

ヒープページに対するタプルのプリューニング（不要領域の回収）およびフリーズ処理のWALリプレイを実行します。

シグネチャまたは形状

static void heap_xlog_prune_freeze(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）。
  • **出力**: なし（副作用としてバッファの更新、フリースペースマップの更新、Hot Standbyの競合解決を行う）。

前提条件と不変条件

  • xlrec.flags に XLHP_CLEANUP_LOCK が設定されていない場合、タプルデータの移動を伴う操作は許可されません。
  • XLHP_HAS_CONFLICT_HORIZON フラグがある場合、Hot Standbyモードであれば ResolveRecoveryConflictWithSnapshot を呼び出し、可視性に関する競合を解決する必要があります。

アルゴリズムまたは振る舞い

1. **メタデータ抽出**: WALレコードから xl_heap_prune 構造体および競合解消用のXIDを読み取ります。 2. **競合解決**: Hot Standby環境下で必要な場合、スナップショットの競合を解決します。 3. **バッファ取得**: XLogReadBufferForRedoExtended を使用して対象ページを読み込みます。 4. **プリューニング実行**: heap_xlog_deserialize_prune_and_freeze でデータを復元し、heap_page_prune_execute を呼び出してページ上のラインポインタを更新・断片化を解消します。 5. **フリーズ実行**: heap_execute_freeze_tuple を繰り返し呼び出し、指定されたタプルをフリーズします。 6. **後処理**: ページLSNを更新し、バッファをダーティとしてマークします。 7. **FSM更新**: プリューニングにより空き領域が変化した場合、XLogRecordPageWithFreeSpace を通じてフリースペースマップを更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_xlog.c 内の heap2_redo（推論される主要なディスパッチャ）。
  • **呼び出し先**:
  • heap_xlog_deserialize_prune_and_freeze
  • heap_page_prune_execute
  • heap_execute_freeze_tuple
  • ResolveRecoveryConflictWithSnapshot
  • XLogReadBufferForRedoExtended
  • XLogRecordPageWithFreeSpace

関連データ構造

  • xl_heap_prune: プリューニング操作のメタデータ。
  • xlhp_freeze_plan: タプルフリーズの計画情報。
  • XLogReaderState: WALレコード読み取り状態。

失敗モードと境界条件

  • XLogReadBufferForRedoExtended がページを読み込めない場合、または既に最新の状態である場合はリプレイをスキップします。
  • ページ内のデータとWALレコードの整合性が取れない場合、アサーションエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は heap2_redo から呼び出されることを前提としています。
  • ページ内のプリューナビリティヒントの更新は行われません。これは将来的なプリューニングサイクルで自然に解消されるためです。
  • XLHP_CLEANUP_LOCK フラグの有無により、ロックの取得モードとプリューニングの挙動が変化するため、変更時は注意が必要です。
