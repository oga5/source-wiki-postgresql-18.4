multirange_get_typcache

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/multirangetypes.c:549

目的

マルチレンジ型に関するキャッシュ情報を取得します。関数呼び出しの実行コンテキスト（fcinfo）にキャッシュを保持することで、型情報のルックアップを最適化します。

シグネチャまたは形状

TypeCacheEntry *multirange_get_typcache(FunctionCallInfo fcinfo, Oid mltrngtypid)

入力と出力

  • **入力**:
  • fcinfo: 現在の関数呼び出し情報。fn_extra フィールドをキャッシュの保存場所として使用します。
  • mltrngtypid: 取得対象のマルチレンジ型の OID。
  • **出力**:
  • TypeCacheEntry*: 指定されたマルチレンジ型に対応するキャッシュエントリ。

前提条件と不変条件

  • fcinfo->flinfo->fn_extra は、以前にキャッシュされた TypeCacheEntry へのポインタ、または NULL である必要があります。
  • 指定された mltrngtypid は、システムカタログ上で有効なマルチレンジ型である必要があります。

アルゴリズムまたは振る舞い

1. fcinfo->flinfo->fn_extra から既存のキャッシュエントリを取得します。 2. キャッシュが空であるか、キャッシュされている型 ID が要求された mltrngtypid と一致しない場合：

  • lookup_type_cache を呼び出し、TYPECACHE_MULTIRANGE_INFO フラグを立てて型情報を取得します。
  • 取得したエントリの rngtype が NULL の場合、指定された型がマルチレンジ型ではないと判断し、エラー（ERROR）を発生させます。
  • 新しいエントリを fcinfo->flinfo->fn_extra に保存します。

    • 3. キャッシュされたエントリを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/multirangetypes.c 内の複数の関数（例: 行 957, 1033, 1073, 1101, 1128, 1244, 1398, 1429, 1486, 1519）。
  • **呼び出し先**: lookup_type_cache（utils/lsyscache.h 経由）。

関連データ構造

  • TypeCacheEntry: utils/lsyscache.h で定義される型キャッシュ構造体。
  • FunctionCallInfo: utils/fmgr.h で定義される関数呼び出しコンテキスト。

失敗モードと境界条件

  • 指定された mltrngtypid がマルチレンジ型ではない場合、elog(ERROR, ...) によりトランザクションが中断されます。

保守担当者向けメモ

  • 本関数は fn_extra を利用したキャッシュ機構であるため、同一の fcinfo を使い回すコンテキストでのみ有効です。
  • lookup_type_cache の呼び出しコストを削減するための重要な最適化ポイントです。
