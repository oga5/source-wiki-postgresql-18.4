PgStat_BackendSubEntry

定義

  • 種別: struct
  • 定義箇所: src/include/pgstat.h:108

目的

サブスクリプションに関連する統計情報のうち、共有メモリへ未フラッシュのものを保持するためのデータ構造です。

シグネチャまたは形状

typedef struct PgStat_BackendSubEntry
{
	PgStat_Counter apply_error_count;
	PgStat_Counter sync_error_count;
	PgStat_Counter conflict_count[CONFLICT_NUM_TYPES];
} PgStat_BackendSubEntry;

入力と出力

  • **入力**: 各種統計カウンタ（apply_error_count, sync_error_count, conflict_count）の更新。
  • **出力**: 統計システムによる共有メモリへのフラッシュ対象となるデータ。

前提条件と不変条件

  • conflict_count 配列のサイズは CONFLICT_NUM_TYPES に依存します。
  • 本構造体は、統計情報のフラッシュが行われるまでの間、バックエンドローカルで保持される一時的な状態を表現します。

アルゴリズムまたは振る舞い

  • サブスクリプション処理中に発生したエラーや競合の回数をカウントします。
  • pgstat_subscription.c 内で、統計エントリの参照を通じてローカルのカウンタが更新され、最終的に共有メモリへ反映されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/utils/activity/pgstat.c: 統計エントリのサイズ定義に使用。
  • src/backend/utils/activity/pgstat_subscription.c: サブスクリプション統計のローカル保持および更新に使用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すものではありません。

関連データ構造

  • PgStat_Counter: 統計値の保持に使用される int64 型。
  • CONFLICT_NUM_TYPES: 競合の種類数を定義する定数。

失敗モードと境界条件

  • 統計情報のフラッシュ処理が失敗した場合、本構造体に保持された値が共有メモリへ正しく反映されない可能性があります。
  • CONFLICT_NUM_TYPES の定義変更に伴い、conflict_count 配列の整合性を維持する必要があります。

保守担当者向けメモ

  • 本構造体を変更した際は、PgStat_BackendSubEntry を参照している pgstat_subscription.c および pgstat.c でのサイズ計算やアクセスロジックに影響がないか確認してください。
  • 統計ファイルの互換性を維持するため、構造体定義の変更時には PGSTAT_FILE_FORMAT_ID の更新を検討してください。
