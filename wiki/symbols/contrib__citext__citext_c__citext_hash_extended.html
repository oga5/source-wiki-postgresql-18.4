citext_hash_extended

定義

  • 種別: function
  • 定義箇所: contrib/citext/citext.c:162

目的

citext 型のデータに対して、大文字・小文字を区別しないハッシュ値を計算します。ハッシュインデックス等の PostgreSQL 内部のハッシュ計算処理で使用されます。

シグネチャまたは形状

Datum citext_hash_extended(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_PP(0): ハッシュ対象の text 型データ。
  • PG_GETARG_INT64(1): ハッシュ計算に使用するシード値。
  • **出力**:
  • Datum: 計算された uint64 型のハッシュ値。

前提条件と不変条件

  • 入力された text データは、str_tolower を用いて DEFAULT_COLLATION_OID に基づき小文字に正規化されます。
  • 処理の過程で生成された一時的な文字列メモリは、必ず pfree で解放されます。
  • TOAST された入力データに対しては、PG_FREE_IF_COPY を使用してメモリリークを防止します。

アルゴリズムまたは振る舞い

1. 入力された text データを str_tolower で小文字に変換します。 2. 変換後の文字列に対して hash_any_extended を呼び出し、指定されたシード値を用いてハッシュ値を生成します。 3. 一時領域を解放し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/citext/citext.c:160 にて PG_FUNCTION_INFO_V1 マクロにより登録されています。
  • その他の具体的な呼び出し元については、提供されたソース情報からは確認できません。
  • **呼び出し先**:
  • str_tolower: 文字列の小文字変換。
  • hash_any_extended: ハッシュ値の計算。
  • pfree: メモリ解放。
  • PG_FREE_IF_COPY: TOAST データのメモリ管理。

関連データ構造

  • text: PostgreSQL の可変長文字列型。

失敗モードと境界条件

  • 入力データが TOAST されている場合、PG_FREE_IF_COPY を適切に呼び出さないとメモリリークが発生する可能性があります。本関数ではこの処理が実装されています。

保守担当者向けメモ

  • 本関数は DEFAULT_COLLATION_OID を固定で使用しています。これは大文字・小文字の変換ルールを照合順序に依存させないための設計です。
  • 今後の PostgreSQL コアで照合順序依存のハッシュ計算が標準化された場合、実装が簡素化される可能性があります。
