strlower_libc_mb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_libc.c:189

目的

マルチバイトエンコーディング環境において、OSのlibcロケール機能を利用して文字列を小文字に変換する。

シグネチャまたは形状

size_t strlower_libc_mb(char *dest, size_t destsize, const char *src, ssize_t srclen, pg_locale_t locale)

入力と出力

  • **入力**:
  • dest: 変換結果を格納するバッファ。
  • destsize: dest バッファのサイズ。
  • src: 変換対象の文字列。
  • srclen: src の長さ（負の場合は strlen(src) が使用される）。
  • locale: ロケール情報を含む pg_locale_t 構造体。
  • **出力**:
  • 変換後の文字列のバイト長を size_t で返す。

前提条件と不変条件

  • srclen が INT_MAX / sizeof(wchar_t) を超える場合、メモリ不足エラー（ERRCODE_OUT_OF_MEMORY）が発生する。
  • 変換処理には palloc を使用してヒープメモリを確保する。

アルゴリズムまたは振る舞い

1. srclen が負の場合、strlen で長さを取得する。 2. 入力文字列を char2wchar を用いてワイド文字配列（workspace）に変換する。 3. towlower_l を使用し、ロケールハンドル（locale->info.lt）に基づいてワイド文字配列の各要素を小文字に変換する。 4. 変換後の最大バイト数を計算し、palloc で結果用バッファを確保する。 5. wchar2char を用いてワイド文字配列を元のエンコーディングに戻す。 6. destsize が十分な場合、結果を dest にコピーし、NUL終端する。 7. 確保した workspace と result バッファを pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/pg_locale_libc.c 内の関数（例: strlower_libc）から呼び出される。
  • **呼び出し先**: strlen, palloc, char2wchar, towlower_l, pg_database_encoding_max_length, wchar2char, memcpy, pfree。

関連データ構造

  • pg_locale_t: ロケール情報を保持する構造体。info.lt（locale_t 型）を介してOSのロケールハンドルにアクセスする。

失敗モードと境界条件

  • 入力文字列が非常に長い場合、メモリ確保に失敗してエラーが発生する可能性がある。
  • destsize が変換後の文字列長より小さい場合、dest へのコピーは行われない。

保守担当者向けメモ

  • 本関数はマルチバイト文字専用の処理である。シングルバイト文字の場合は strlower_libc_sb が使用される。
  • pg_database_encoding_max_length() を使用して変換後の最大サイズを予測しているため、エンコーディングごとの特性に依存する。
