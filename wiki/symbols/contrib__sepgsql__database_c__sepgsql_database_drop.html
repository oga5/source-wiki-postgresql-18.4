sepgsql_database_drop

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/database.c:132

目的

データベース削除操作に対し、SELinuxのセキュリティポリシーに基づいた権限チェック（db_database:{drop}）を実行する。

シグネチャまたは形状

void sepgsql_database_drop(Oid databaseId)

入力と出力

  • **入力**: databaseId (Oid) - 削除対象となるデータベースのオブジェクト識別子。
  • **出力**: なし。権限チェックに失敗した場合は例外が発生し、処理が中断される。

前提条件と不変条件

  • databaseId は有効なデータベースの OID であること。
  • 呼び出し元は、PostgreSQLのデータベース削除処理のフックポイントであること。

アルゴリズムまたは振る舞い

1. ObjectAddress 構造体を初期化し、DatabaseRelationId と対象の databaseId を設定する。 2. getObjectIdentity を呼び出し、監査ログ用のオブジェクト識別文字列を取得する。 3. sepgsql_avc_check_perms を呼び出し、SEPG_CLASS_DB_DATABASE クラスに対する SEPG_DB_DATABASE__DROP 権限を検証する。 4. 監査ログ用文字列をメモリから解放（pfree）する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/hooks.c:168 から呼び出されることが確認されている。
  • **呼び出し先**:
  • getObjectIdentity (PostgreSQL内部関数)
  • sepgsql_avc_check_perms (SE-PostgreSQL AVCインターフェース)
  • pfree (PostgreSQLメモリ管理)

関連データ構造

  • ObjectAddress: データベースオブジェクトを識別するために使用される構造体。
  • DatabaseRelationId: pg_database カタログを指す定数。

失敗モードと境界条件

  • 権限チェック失敗時: sepgsql_avc_check_perms が内部でエラーを発生させ、トランザクションがアボートされる。
  • メモリ不足: getObjectIdentity がメモリを割り当てられない場合、PostgreSQLの標準的なエラー処理に従う。

保守担当者向けメモ

  • 本関数は sepgsql_avc_check_perms のラッパーとして機能する。
  • 権限チェックの引数 true は、監査ログの生成を有効にすることを意味する。
