hstore_fetchval

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_op.c:127

目的

hstore 型のデータから、指定されたキーに対応する値を検索して取得する。

シグネチャまたは形状

Datum hstore_fetchval(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_HSTORE_P(0): 検索対象の HStore 構造体。
  • PG_GETARG_TEXT_PP(1): 検索するキーを表す text 型。
  • **出力**:
  • キーが見つかり、かつ値が NULL でない場合: 対応する値を text 型として返す。
  • キーが見つからない、または値が NULL の場合: SQL の NULL を返す。

前提条件と不変条件

  • 入力される HStore は、内部的にキーがソートされている必要がある（hstoreFindKey のバイナリサーチが前提とするため）。

アルゴリズムまたは振る舞い

1. hstoreFindKey を呼び出し、対象キーのインデックスをバイナリサーチで特定する。 2. インデックスが負（キーが存在しない）または HSTORE_VALISNULL マクロにより値が NULL と判定された場合、PG_RETURN_NULL() を実行する。 3. キーが存在し値が有効な場合、HSTORE_VAL および HSTORE_VALLEN マクロを使用して値のポインタと長さを取得する。 4. cstring_to_text_with_len を用いて値を text 型に変換し、PG_RETURN_TEXT_P で返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_op.c 内で HSTORE_POLLUTE マクロにより fetchval として公開されている。
  • **呼び出し先**:
  • hstoreFindKey (検索処理)
  • cstring_to_text_with_len (型変換)
  • PG_GETARG_* / PG_RETURN_* (PostgreSQL FMGR インターフェース)

関連データ構造

  • HStore: 操作対象のデータ構造。
  • HEntry: HStore 内の各エントリを管理する構造体。

失敗モードと境界条件

  • キーが hstore 内に存在しない場合は NULL を返す。
  • キーに対応する値が NULL である場合は NULL を返す。

保守担当者向けメモ

  • 本関数は hstore の検索ロジックの基本形である。類似の関数（hstore_exists 等）も同様の検索パターンに従う。
  • hstore.h で定義された ARRPTR, STRPTR, HSTORE_VAL などのマクロを使用して内部レイアウトにアクセスしているため、構造体のメモリ配置を変更する際は注意が必要である。
