XLogCheckBufferNeedsBackup

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xloginsert.c:1026

目的

指定されたバッファに対して、フルページイメージ（FPI）を WAL に記録する必要があるかどうかを判定します。

シグネチャまたは形状

bool XLogCheckBufferNeedsBackup(Buffer buffer)

入力と出力

  • **入力**: buffer (判定対象のバッファ)
  • **出力**: bool (バックアップが必要な場合は true、不要な場合は false)

前提条件と不変条件

  • GetFullPageWriteInfo を介して取得したシステム全体のフルページ書き込み設定と、対象バッファの LSN を比較します。
  • 対象のバッファは、呼び出し前に適切にロックされていることが想定されます。

アルゴリズムまたは振る舞い

1. GetFullPageWriteInfo を呼び出し、現在のチェックポイントの Redo 挿入位置 (RedoRecPtr) と、フルページ書き込みが有効か (doPageWrites) を取得します。 2. 対象バッファのページを取得します。 3. 以下の条件を両方満たす場合、true を返します。

  • doPageWrites が有効である。
  • ページの LSN が RedoRecPtr 以下である（つまり、前回のチェックポイント以降、このページが一度も書き込まれていない）。

    • 4. 上記以外の場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/pruneheap.c
  • **呼び出し先**:
  • GetFullPageWriteInfo
  • BufferGetPage
  • PageGetLSN

関連データ構造

  • Buffer: バッファ管理用ハンドル。
  • XLogRecPtr: WAL 内のログ挿入位置を示すポインタ。

失敗モードと境界条件

  • buffer が無効な場合、BufferGetPage の挙動に依存します。
  • doPageWrites が無効な場合、常に false を返します。

保守担当者向けメモ

  • 本関数は、WAL 圧縮やフルページ書き込みの最適化において重要な判定基準となります。
  • PageGetLSN を使用するため、対象バッファが正しく初期化されていることが前提です。
