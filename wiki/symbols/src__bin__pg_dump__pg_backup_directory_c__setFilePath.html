setFilePath

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_directory.c:681

目的

ディレクトリ形式アーカイブにおける、ベースディレクトリと相対ファイル名を結合した絶対パス（または相対パス）を生成する。

シグネチャまたは形状

static void setFilePath(ArchiveHandle *AH, char *buf, const char *relativeFilename)

入力と出力

  • AH: アーカイブハンドル。内部の formatData からディレクトリパスを取得するために使用。
  • buf: 結合後のパス文字列を格納するバッファ。呼び出し元で十分なサイズを確保する必要がある。
  • relativeFilename: ディレクトリ配下のファイル名またはパス。
  • 出力: なし（buf に結果を書き込む）。

前提条件と不変条件

  • AH->formatData は lclContext 型としてキャスト可能であること。
  • buf は MAXPGPATH 以上のサイズを持つこと。

アルゴリズムまたは振る舞い

1. AH->formatData から現在のディレクトリパス (ctx->directory) を取得する。 2. ctx->directory と relativeFilename を結合した長さが MAXPGPATH を超える場合、pg_fatal を呼び出し異常終了する。 3. buf に ctx->directory、ディレクトリ区切り文字 /、relativeFilename を順に連結する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/pg_backup_directory.c 内の各所（toc.dat やデータファイルのパス解決に使用）。
  • 呼び出し先: strlen, strcpy, strcat, pg_fatal。

関連データ構造

  • ArchiveHandle: アーカイブ全体の管理構造体。
  • lclContext: ディレクトリ形式特有の実行時状態を保持する構造体。

失敗モードと境界条件

  • パス長が MAXPGPATH を超過する場合、pg_fatal によりプログラムが終了する。

保守担当者向けメモ

  • 本関数は pg_backup_directory.c 内でのみ使用される静的関数です。
  • パス結合時に MAXPGPATH を超えるかどうかのチェックを行っていますが、buf 自体のサイズが MAXPGPATH であることを前提としているため、呼び出し側のバッファサイズには注意してください。
