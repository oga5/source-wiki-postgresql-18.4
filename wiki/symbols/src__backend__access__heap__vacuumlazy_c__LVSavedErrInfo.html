LVSavedErrInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/access/heap/vacuumlazy.c:415

目的

VACUUM処理中にエラーが発生した際、その時点での処理コンテキスト（対象ブロック、オフセット、実行フェーズ）を保持し、エラー報告を補助するために使用されます。

シグネチャまたは形状

typedef struct LVSavedErrInfo
{
	BlockNumber blkno;
	OffsetNumber offnum;
	VacErrPhase phase;
} LVSavedErrInfo;

入力と出力

本構造体はデータ保持用であり、関数としての入出力は持ちません。各フィールドは vacuumlazy.c 内の各処理フェーズにおいて、現在の作業対象を記録するために代入されます。

前提条件と不変条件

  • blkno および offnum は、VACUUM処理中の特定のヒープページおよびタプル位置を指します。
  • phase は VacErrPhase 列挙型に基づき、現在のVACUUMフェーズを正確に反映している必要があります。

アルゴリズムまたは振る舞い

VACUUM処理の各主要ステップにおいて、この構造体のインスタンスがスタック上に確保されます。エラー発生時に update_vacuum_error_info を通じてエラーコンテキストを更新し、PostgreSQLの標準的なエラー報告メカニズムに現在の処理位置を伝達する役割を担います。

呼び出し元と呼び出し先

  • **呼び出し元**: lazy_scan_heap および lazy_vacuum 内の各処理ブロックで、エラーコンテキストを記録するためにインスタンス化されます。
  • **呼び出し先**: update_vacuum_error_info 関数が、本構造体を引数として受け取り、エラー情報を更新します。

関連データ構造

  • VacErrPhase: エラー発生時のフェーズを識別するために使用されます。
  • LVRelState: VACUUM実行中のリレーション状態を保持する構造体であり、エラー報告の文脈で本構造体と併用されます。

失敗モードと境界条件

  • 本構造体自体はメモリ上のデータ保持用であるため、直接的な失敗モードはありません。
  • 記録される blkno や offnum が無効な値である場合、エラー報告時に不正確な情報が出力される可能性があります。

保守担当者向けメモ

  • 本構造体は vacuumlazy.c 内でのエラー追跡に特化しています。新しいVACUUMフェーズを追加する場合は、VacErrPhase と併せて本構造体の利用箇所を適切に更新してください。
