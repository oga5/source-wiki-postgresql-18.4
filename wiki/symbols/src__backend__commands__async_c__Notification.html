Notification

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/async.c:381

目的

NOTIFY コマンドによって発行される非同期通知メッセージを、メモリ上で表現するためのデータ構造です。

シグネチャまたは形状

typedef struct Notification
{
    uint16      channel_len;    /* チャネル名の長さ */
    uint16      payload_len;    /* ペイロードの長さ */
    char        data[FLEXIBLE_ARRAY_MEMBER]; /* チャネル名とペイロードの連続領域 */
} Notification;

入力と出力

  • **入力**: palloc を用いて動的に確保されるメモリ領域。data メンバには、ヌル終端されたチャネル名と、それに続くヌル終端されたペイロードが格納されます。
  • **出力**: トランザクション内の保留中通知リスト（pendingNotifies）に格納されるか、asyncQueueNotificationToEntry を介して共有メモリ上の AsyncQueueEntry へ変換されます。

前提条件と不変条件

  • data 領域には、必ずチャネル名とペイロードがこの順序で、それぞれヌル終端された状態で格納されている必要があります。
  • channel_len および payload_len は、それぞれの文字列の長さを正確に反映している必要があります。

アルゴリズムまたは振る舞い

  • トランザクション実行中に NOTIFY が発行されると、この構造体が生成され、ローカルリストに蓄積されます。
  • コミット処理の過程で、この構造体の内容が共有メモリ上の AsyncQueueEntry にコピーされ、他のバックエンドへ通知が伝播されます。

呼び出し元と呼び出し先

  • **呼び出し元**: AddEventToPendingNotifies（保留リストへの追加時）、asyncQueueNotificationToEntry（キューへの書き込み時）などで参照されます。
  • **呼び出し先**: 直接的な関数呼び出し関係はありませんが、メモリ確保のために palloc が使用されます。

関連データ構造

  • AsyncQueueEntry: 共有メモリ上で通知を保持するための永続的な構造体。
  • pendingNotifies: 現在のトランザクションで保留中の Notification を保持するリスト。

失敗モードと境界条件

  • メモリ不足時: palloc によるメモリ確保に失敗した場合、エラーが発生します。
  • 文字列長: channel_len や payload_len が実際のデータ長と不整合を起こすと、後続の読み取り処理でバッファオーバーランや不正なメモリ参照を引き起こす可能性があります。

保守担当者向けメモ

  • data は可変長配列（FLEXIBLE_ARRAY_MEMBER）であるため、構造体のサイズ計算には offsetof(Notification, data) を使用してください。
  • この構造体はトランザクションローカルなメモリ上に存在するため、コミット完了後は解放されます。永続化が必要な場合は AsyncQueueEntry への変換が必須です。
