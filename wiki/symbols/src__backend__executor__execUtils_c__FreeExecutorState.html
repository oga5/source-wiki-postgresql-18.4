FreeExecutorState

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execUtils.c:191

目的

クエリ実行状態（EState）を破棄し、関連するリソースを解放する。

シグネチャまたは形状

void FreeExecutorState(EState *estate)

入力と出力

  • **入力**: estate (EState *) - 破棄対象の実行状態構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • estate は有効な EState 構造体を指している必要がある。
  • MemoryContextDelete(estate->es_query_cxt) が実行されるため、呼び出し後に estate ポインタおよびその配下のメンバへのアクセスは無効となる。

アルゴリズムまたは振る舞い

1. estate->es_exprcontexts に登録されたすべての ExprContext を順次 FreeExprContext で解放する。これにより、各コンテキストに紐付くシャットダウンコールバックが確実に実行される。 2. estate->es_jit が存在する場合、jit_release_context を呼び出して JIT コンテキストを解放し、ポインタを NULL に設定する。 3. estate->es_partition_directory が存在する場合、DestroyPartitionDirectory を呼び出してパーティションディレクトリを破棄し、ポインタを NULL に設定する。 4. 最後に MemoryContextDelete を使用して、EState 自体を含むクエリ単位のメモリコンテキスト全体を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c
  • src/backend/catalog/index.c
  • src/backend/commands/analyze.c
  • src/backend/commands/constraint.c
  • src/backend/commands/copyfrom.c
  • src/backend/commands/functioncmds.c
  • src/backend/commands/prepare.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • FreeExprContext
  • jit_release_context
  • DestroyPartitionDirectory
  • MemoryContextDelete

関連データ構造

  • EState: クエリ実行の全体状態を保持する構造体。
  • ExprContext: 式評価のためのメモリコンテキスト。

失敗モードと境界条件

  • estate が NULL の場合、MemoryContextDelete 等でクラッシュする可能性がある（本関数内での明示的な NULL チェックは存在しない）。

保守担当者向けメモ

  • estate->es_exprcontexts の解放において list_delete を繰り返す実装となっており、より効率的な手法への改善が検討されている。
  • シャットダウンコールバックの実行を保証するために、メモリコンテキストの削除前に明示的な FreeExprContext の呼び出しが必要である。
