inet_cidr_ntop_ipv6

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/inet_cidr_ntop.c:164

目的

IPv6アドレス（バイナリ形式）を、CIDRプレフィックス長を含む人間が読み取り可能な文字列形式（xxxx:xxxx::/bits 等）へ変換します。

シグネチャまたは形状

char *inet_cidr_ntop_ipv6(const u_char *src, int bits, char *dst, size_t size);

入力と出力

  • **入力**:
  • src: 16バイトのIPv6アドレス（ネットワークバイトオーダー）。
  • bits: CIDRプレフィックス長（0〜128）。
  • dst: 変換結果を格納するバッファ。
  • size: dst バッファのサイズ。
  • **出力**:
  • 成功時: dst へのポインタ。
  • 失敗時: NULL（errno が設定される）。

前提条件と不変条件

  • bits は 0 から 128 の範囲内である必要があります。
  • dst バッファは、結果文字列（最大46文字程度）を格納するのに十分なサイズである必要があります。
  • ホスト部は bits に基づいてゼロクリアされます。

アルゴリズムまたは振る舞い

1. **バリデーション**: bits が範囲外の場合、EINVAL を設定して終了します。 2. **正規化**: src を内部バッファにコピーし、bits より後のビットをゼロクリアします。 3. **ゼロ圧縮**: 連続するゼロのサブストリングを特定し、:: による短縮表現を適用します。 4. **文字列構築**: 16ビット単位で16進数文字列へ変換します。特定の条件下ではIPv4互換形式（x.x.x.x）への変換も行います。 5. **CIDR付与**: 最後に /bits を連結します。 6. **バッファ確認**: 結果が size を超える場合、EMSGSIZE を設定して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/inet_cidr_ntop.c 内の関数（pg_inet_cidr_ntop 等）から呼び出されることが示唆されています。
  • **呼び出し先**: SPRINTF マクロ、memcpy、memset、strlen、strcpy 等の標準的なメモリ・文字列操作関数。

関連データ構造

  • u_char: IPアドレスのバイナリ表現に使用される型。
  • outbuf: 変換過程で使用される固定長の一時バッファ。

失敗モードと境界条件

  • bits が 0〜128 の範囲外: errno = EINVAL を返し NULL を返却。
  • 出力文字列が size を超過: errno = EMSGSIZE を返し NULL を返却。

保守担当者向けメモ

  • IPv6のゼロ圧縮ロジックは複雑です。修正時は bits によるプレフィックス長制限が正しく適用されているか確認してください。
  • ISCライセンスに基づくコードが含まれており、SPRINTF マクロの使用など、他のPostgreSQLコードとは異なるスタイルが採用されています。
