setNamespaceVisibilityForRTE

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_merge.c:414

目的

指定された RangeTblEntry (RTE) に対応する名前空間項目を検索し、その可視性フラグ（リレーションおよび列の可視性）を更新します。主に MERGE 文の解析において、アクションごとにターゲットテーブルやソースリレーションの可視性を動的に切り替えるために使用されます。

シグネチャまたは形状

static void setNamespaceVisibilityForRTE(List *namespace, RangeTblEntry *rte, bool rel_visible, bool cols_visible)

入力と出力

  • namespace (List*): 検索対象となる ParseNamespaceItem のリスト。
  • rte (RangeTblEntry*): 可視性を変更したい対象の RTE。
  • rel_visible (bool): リレーション自体の可視性フラグ。
  • cols_visible (bool): 列の可視性フラグ。
  • 戻り値: なし（副作用として namespace 内の項目を直接更新）。

前提条件と不変条件

  • namespace リスト内の各要素は ParseNamespaceItem 型である必要があります。
  • 指定された rte が namespace 内に存在することが期待されます。

アルゴリズムまたは振る舞い

1. namespace リストを先頭から順に走査します。 2. 各要素（ParseNamespaceItem）の p_rte フィールドが引数の rte と一致するか確認します。 3. 一致した場合、その項目の p_rel_visible および p_cols_visible を引数で指定された値に更新し、ループを終了します。 4. 一致する項目が見つからない場合、何も行わずに終了します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/parser/parse_merge.c 内の複数箇所（transformMergeStmt 等の解析プロセスから、MERGE 文の各 WHEN 句処理時に呼び出されます）。
  • 呼び出し先: 本関数は末端のヘルパー関数であり、他の関数を呼び出しません。

関連データ構造

  • ParseNamespaceItem: 名前空間内の各項目を保持する構造体。p_rte（RTEへのポインタ）、p_rel_visible、p_cols_visible を含みます。
  • RangeTblEntry: 範囲テーブルエントリ。

失敗モードと境界条件

  • 指定された rte が namespace リスト内に存在しない場合、関数は何も変更を行わず静かに終了します（エラーは発生しません）。

保守担当者向けメモ

  • 本関数は ParseNamespaceItem を直接操作して可視フラグを書き換えるため、副作用を伴います。MERGE 文の解析フローにおいて、アクションごとの可視性制御が正しく行われているか確認する際は、本関数の呼び出し順序とフラグの状態に注意してください。
