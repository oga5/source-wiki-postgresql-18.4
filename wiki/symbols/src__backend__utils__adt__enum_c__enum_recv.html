enum_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/enum.c:178

目的

バイナリ形式の入力ストリームから列挙型（ENUM）の値を読み取り、対応する内部表現（OID）に変換します。

シグネチャまたは形状

Datum enum_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): StringInfo 型のバイナリデータバッファ。
  • PG_GETARG_OID(1): 対象となる列挙型の OID。
  • **出力**:
  • PG_RETURN_OID(enumoid): 変換された列挙値の OID を返します。

前提条件と不変条件

  • 入力されたラベル文字列の長さが NAMEDATALEN 未満である必要があります。
  • 指定された列挙型 OID とラベルの組み合わせがシステムカタログ pg_enum に存在する必要があります。
  • 取得した列挙値は、check_safe_enum_use により SQL での使用が安全であると検証される必要があります。

アルゴリズムまたは振る舞い

1. pq_getmsgtext を使用して、バッファから列挙値のラベル文字列を抽出します。 2. ラベルの長さが NAMEDATALEN 以上の場合、エラーを報告します。 3. SearchSysCache2 を使用し、ENUMTYPOIDNAME キャッシュから該当する列挙値のタプルを検索します。見つからない場合はエラーを報告します。 4. check_safe_enum_use を呼び出し、未コミットの列挙値でないことを確認します。 5. タプルから列挙値の OID を取得します。 6. システムキャッシュを解放し、抽出した文字列メモリを解放した後、OID を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は明示されていませんが、PostgreSQL のバイナリプロトコル経由で列挙型データを受信する際に呼び出されます。
  • **呼び出し先**:
  • pq_getmsgtext: バッファからの文字列抽出。
  • SearchSysCache2: pg_enum の検索。
  • check_safe_enum_use: 安全性検証。
  • ReleaseSysCache: キャッシュ解放。
  • pfree: メモリ解放。

関連データ構造

  • StringInfo: バイナリデータバッファ。
  • HeapTuple: pg_enum カタログのタプル。
  • Form_pg_enum: pg_enum カタログの構造体定義。

失敗モードと境界条件

  • **入力長超過**: ラベルが NAMEDATALEN 以上の場合、ERRCODE_INVALID_TEXT_REPRESENTATION でエラーとなります。
  • **値の不一致**: 指定された型に該当するラベルが存在しない場合、ERRCODE_INVALID_TEXT_REPRESENTATION でエラーとなります。
  • **安全性違反**: check_safe_enum_use が未コミットの列挙値と判断した場合、エラーが発生します。

保守担当者向けメモ

  • 本関数はバイナリ入力専用です。テキスト入力については enum_in が担当します。
  • check_safe_enum_use の呼び出しは、インデックス破損を防ぐために必須です。変更を加える際は、このガードが維持されていることを確認してください。
