pg_stat_get_activity

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pgstatfuncs.c:330

目的

現在稼働中の全バックエンドプロセスの状態（アクティビティ）を、SQLインターフェースを通じて取得・返却する。

シグネチャまたは形状

Datum pg_stat_get_activity(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_ARGISNULL(0): 特定のPIDを指定する場合に使用。指定がない場合は全バックエンドを対象とする。
  • **出力**:
  • ReturnSetInfo: tuplestore を用いたセット形式（SRF）で、バックエンドごとの統計情報（PID、データベースID、ユーザーID、状態、待機イベント、トランザクション開始時刻等）を返却する。

前提条件と不変条件

  • InitMaterializedSRF を呼び出し、結果セットの初期化を行う必要がある。
  • 統計情報の取得には pgstat_fetch_stat_numbackends および pgstat_get_local_beentry_by_index を使用する。
  • 権限チェック（HAS_PGSTAT_PERMISSIONS）を通過しない場合、機密性の高い情報（クエリ文字列や待機イベント等）はNULLまたはプレースホルダーとして扱われる。

アルゴリズムまたは振る舞い

1. 現在のバックエンド数を取得し、ループで各プロセスの情報を走査する。 2. PID指定がある場合、該当しないプロセスはスキップする。 3. データベースID、PID、ユーザーID、アプリ名などの基本情報を取得する。 4. HAS_PGSTAT_PERMISSIONS を確認し、権限がある場合に限り、詳細な状態（st_state）、クエリ文字列（st_activity_raw）、待機イベント、リーダーPID等の情報を抽出する。 5. 補助プロセス（B_BACKEND 以外）については、AuxiliaryPidGetProc を用いて共有メモリから情報を取得する。 6. 取得した値を tuplestore に格納し、結果セットとして返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLレイヤーから直接呼び出される。ソースコード内での明示的な呼び出し元は src/backend/utils/activity/backend_status.c:1146 に言及がある。
  • **呼び出し先**:
  • pgstat_fetch_stat_numbackends, pgstat_get_local_beentry_by_index
  • HAS_PGSTAT_PERMISSIONS
  • pgstat_clip_activity, pgstat_get_wait_event_type, pgstat_get_wait_event
  • BackendPidGetProc, AuxiliaryPidGetProc, GetLeaderApplyWorkerPid

関連データ構造

  • PgBackendStatus: バックエンドのステータス情報。
  • LocalPgBackendStatus: ローカルキャッシュされたバックエンドステータス。
  • PGPROC: プロセスごとの共有メモリ上の制御構造体。
  • ReturnSetInfo: SRF用の戻り値情報構造体。

失敗モードと境界条件

  • 権限不足: 呼び出し元が対象の統計情報を見る権限を持たない場合、一部のフィールドがNULLになる。
  • 整合性: ロックを取得せずに情報を読み取るため、複数行にわたる情報の整合性は保証されない。
  • 特定PID未検出: 指定したPIDが存在しない場合、結果セットは空となる。

保守担当者向けメモ

  • 本関数はSRFであるため、メモリ管理には InitMaterializedSRF を使用する定型パターンに従うこと。
  • 統計情報はスナップショットベースであるため、呼び出しタイミングにより値が変動する可能性がある。
  • 新しい統計項目を追加する場合は、PG_STAT_GET_ACTIVITY_COLS の定義と values/nulls 配列のインデックスを適切に更新すること。
