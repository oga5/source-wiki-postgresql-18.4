ReplicationOriginShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/origin.c:533

目的

論理レプリケーションの起点（Replication Origin）管理に必要な共有メモリ領域の総サイズを計算する。

シグネチャまたは形状

Size ReplicationOriginShmemSize(void)

入力と出力

  • **入力**: なし（GUC変数 max_active_replication_origins を参照）
  • **出力**: 必要な共有メモリサイズ（バイト単位の Size 型）

前提条件と不変条件

  • max_active_replication_origins が 0 の場合、サイズとして 0 を返す。
  • 計算には ReplicationStateCtl 構造体のヘッダー部分と、max_active_replication_origins 分の ReplicationState 配列のサイズが含まれる。

アルゴリズムまたは振る舞い

1. max_active_replication_origins が 0 であれば 0 を返す。 2. ReplicationStateCtl 構造体の states メンバーまでのオフセットサイズを取得する。 3. max_active_replication_origins と ReplicationState 構造体のサイズを乗算する。 4. 上記の値を加算し、合計サイズを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/ipci.c: システム起動時の共有メモリ割り当て計算の一部として呼び出される。
  • src/backend/replication/logical/origin.c: ReplicationOriginShmemInit 内で共有メモリの初期化時に呼び出される。
  • **呼び出し先**:
  • add_size (マクロ/関数): サイズ加算に使用。
  • mul_size (マクロ/関数): サイズ乗算に使用。

関連データ構造

  • ReplicationState: 各起点の進捗状態を保持する構造体。
  • ReplicationStateCtl: 共有メモリ上の起点管理用制御構造体。

失敗モードと境界条件

  • max_active_replication_origins が 0 の場合、共有メモリは割り当てられない。
  • add_size や mul_size の内部実装に依存するが、計算結果が Size 型の範囲を超える場合はエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウトを決定する重要な関数であるため、ReplicationState 構造体のサイズ変更や ReplicationStateCtl の構造変更を行った場合は、必ず本関数の計算ロジックを更新すること。
