seq_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/seqdesc.c:20

目的

シーケンス操作に関連するWAL（Write-Ahead Log）レコードの内容を解析し、人間が読み取り可能な形式でバッファへ出力する。

シグネチャまたは形状

void seq_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 出力文字列を格納する StringInfo バッファ。
  • record: 解析対象のWALレコード状態を保持する XLogReaderState 構造体。
  • **出力**: なし（buf を通じて副作用として文字列が追記される）。

前提条件と不変条件

  • record は有効な XLogReaderState を指している必要がある。
  • XLogRecGetData(record) が返すデータは xl_seq_rec 構造体としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. XLogRecGetData を用いてレコードの生データポインタを取得する。 2. XLogRecGetInfo で取得した情報から XLR_INFO_MASK を除外し、操作タイプを特定する。 3. 操作タイプが XLOG_SEQ_LOG と一致する場合、データを xl_seq_rec 型にキャストする。 4. xl_seq_rec 内の locator（spcOid, dbOid, relNumber）を抽出し、appendStringInfo を用いて buf に "rel OID/OID/OID" 形式で書き出す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h にて RmgrData 構造体の記述子ルーチンとして登録されており、PostgreSQLのWAL解析ツール（pg_waldump 等）から間接的に呼び出される。
  • **呼び出し先**:
  • XLogRecGetData（WALデータ取得）
  • XLogRecGetInfo（レコード情報取得）
  • appendStringInfo（バッファへの書き込み）

関連データ構造

  • xl_seq_rec: シーケンスのWALレコード構造体（commands/sequence.h で定義）。
  • StringInfo: 文字列バッファ管理用構造体。
  • XLogReaderState: WAL読み取り状態管理用構造体。

失敗モードと境界条件

  • info が XLOG_SEQ_LOG 以外の場合、バッファには何も追記されない。
  • record が不正なデータを含む場合、キャスト後の xlrec へのアクセスで予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は表示専用であり、シーケンスの更新ロジックには関与しない。
  • XLR_INFO_MASK によるビット演算は、レコードのフラグを除去し操作タイプを正しく識別するために必須である。
