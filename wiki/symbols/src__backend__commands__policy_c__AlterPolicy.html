AlterPolicy

定義

  • 種別: function
  • 定義箇所: src/backend/commands/policy.c:767

目的

既存の行レベルセキュリティ（RLS）ポリシー定義を更新し、システムカタログ pg_policy を変更する。

シグネチャまたは形状

ObjectAddress AlterPolicy(AlterPolicyStmt *stmt)

入力と出力

  • **入力**: AlterPolicyStmt *stmt（更新対象のポリシー名、テーブル名、および新しい USING / WITH CHECK 式やロール定義を含む構造体）
  • **出力**: ObjectAddress（更新されたポリシーのオブジェクト識別子）

前提条件と不変条件

  • 対象テーブルに対する適切な権限が必要（RangeVarGetRelidExtended による検証）。
  • 指定されたポリシー名が対象テーブルに存在すること。
  • SELECT または DELETE ポリシーに対して WITH CHECK 句を指定することは不可。
  • INSERT ポリシーに対して USING 句を指定することは不可。

アルゴリズムまたは振る舞い

1. **解析**: stmt->roles があればロール配列へ変換し、stmt->qual（USING句）および stmt->with_check（WITH CHECK句）を解析して式ツリーへ変換する。 2. **検証**: RangeVarGetRelidExtended を呼び出し、対象テーブルの存在確認と権限チェックを行う。 3. **検索**: pg_policy カタログを polrelid と polname でスキャンし、対象のポリシー行を特定する。 4. **制約チェック**: ポリシーのコマンド種別（polcmd）を確認し、指定された句（USING/WITH CHECK）がそのコマンド種別に対して妥当か検証する。 5. **更新準備**: カタログ更新用の値を準備し、必要に応じて既存のロール情報を取得する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（SQL解析後のDDL実行時）
  • **呼び出し先**: policy_role_list_to_array, RangeVarGetRelidExtended, relation_open, make_parsestate, transformWhereClause, systable_beginscan, systable_getnext 等。

関連データ構造

  • AlterPolicyStmt: DDL文の解析結果を保持するノード。
  • pg_policy: ポリシー定義を格納するシステムカタログ。
  • RowSecurityDesc: RLS記述子（本関数が更新するカタログ情報が後にロードされる先）。

失敗モードと境界条件

  • **ERRCODE_UNDEFINED_OBJECT**: 指定されたポリシー名がテーブルに見つからない場合。
  • **ERRCODE_SYNTAX_ERROR**: SELECT/DELETE に WITH CHECK を指定、または INSERT に USING を指定した場合。

保守担当者向けメモ

  • 本関数はカタログの更新までを担当する。更新後のキャッシュ無効化や依存関係の再構築については、後続の処理（本関数の戻り値以降のフロー）で適切に処理される必要がある。
  • pg_policy の polroles 配列の更新時には、依存関係の整合性を保つため既存のロール情報をカタログから読み出す必要がある点に注意。
