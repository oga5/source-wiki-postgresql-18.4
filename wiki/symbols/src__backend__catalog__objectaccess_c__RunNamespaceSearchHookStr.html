RunNamespaceSearchHookStr

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaccess.c:240

目的

名前空間（Namespace）検索時に、外部の拡張機能やプラグインが介入・検証を行うためのフックを実行します。

シグネチャまたは形状

bool RunNamespaceSearchHookStr(const char *objectName, bool ereport_on_violation)

入力と出力

  • **入力**:
  • objectName: 検索対象の名前空間名。
  • ereport_on_violation: 検索が許可されない場合にエラーを報告するかどうかを示すフラグ。
  • **出力**:
  • bool: 検索が許可された場合は true、拒否された場合は false を返します。

前提条件と不変条件

  • グローバル変数 object_access_hook_str が NULL でないことが前提です（Assert により検証されます）。

アルゴリズムまたは振る舞い

1. ObjectAccessNamespaceSearch 構造体を初期化し、引数 ereport_on_violation を設定します。 2. 検索結果のデフォルト値を true に設定します。 3. 登録されているフック関数 object_access_hook_str を呼び出し、イベントタイプ OAT_NAMESPACE_SEARCH、名前空間のOID NamespaceRelationId、および対象の objectName を渡します。 4. フック関数によって更新された ns_arg.result を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は、提供された情報からは特定できません。
  • **呼び出し先**: グローバル関数ポインタ object_access_hook_str が指す外部のフック関数を呼び出します。

関連データ構造

  • ObjectAccessNamespaceSearch: フックに渡される引数および結果格納用の構造体。

失敗モードと境界条件

  • object_access_hook_str が NULL の状態で呼び出された場合、アサーション違反によりプロセスが停止します。

保守担当者向けメモ

  • 本関数は名前ベースの識別が必要な場合に利用されます。OIDベースのフックが必要な場合は RunNamespaceSearchHook を使用してください。
  • 実際のアクセス制御ロジックは、ロードされた共有ライブラリ（プラグイン）側に実装されています。
