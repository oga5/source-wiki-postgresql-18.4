AddEnumLabel

定義

  • **種別**: function
  • **定義箇所**: src/backend/catalog/pg_enum.c:291

目的

既存の列挙型（ENUM）に対して新しいラベルを追加します。指定された順序位置（neighbor の前後）に基づき、適切な enumsortorder を割り当ててシステムカタログ pg_enum を更新します。

シグネチャまたは形状

void AddEnumLabel(Oid enumTypeOid,
                  const char *newVal,
                  const char *neighbor,
                  bool newValIsAfter,
                  bool skipIfExists)

入力と出力

  • **入力**:
  • enumTypeOid: 対象となる列挙型の OID。
  • newVal: 追加する新しいラベル名。
  • neighbor: 挿入位置の基準となる既存ラベル名（NULL の場合は末尾に追加）。
  • newValIsAfter: neighbor の後ろに挿入するかどうか。
  • skipIfExists: 既にラベルが存在する場合にエラーとせず通知のみで終了するかどうか。
  • **出力**: なし（成功時は pg_enum への挿入を実行）。

前提条件と不変条件

  • ラベル名は NAMEDATALEN - 1 バイト以下であること。
  • 対象の列挙型に対して ExclusiveLock を取得し、トランザクション終了まで保持する。
  • enumsortorder は float4 型であり、精度限界に達した場合は再採番が必要となる。

アルゴリズムまたは振る舞い

1. **検証**: ラベル名の長さを確認し、既に存在する場合は skipIfExists に応じてエラーまたは通知を出す。 2. **ロック**: LockDatabaseObject で対象型を排他ロックする。 3. **順序計算**: 既存のラベルを SearchSysCacheList1 で取得し、enumsortorder でソートする。 4. **位置決定**: neighbor が指定されている場合、その前後位置を計算する。float4 の精度不足で隣接値と重複する場合は RenumberEnumType を呼び出し、全ラベルの順序を再割り当てしてから再試行する。 5. **永続化**: CatalogTupleInsert を使用して新しいラベルをカタログに挿入する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/typecmds.c（ALTER TYPE ... ADD VALUE コマンド実行時）
  • src/backend/catalog/pg_enum.c 内の内部処理
  • **呼び出し先**:
  • LockDatabaseObject, SearchSysCache2, table_open, SearchSysCacheList1, qsort, RenumberEnumType, CatalogTupleInsert 等。

関連データ構造

  • pg_enum: 列挙型定義を保持するシステムカタログ。
  • CatCList: システムカタログのキャッシュリスト。
  • HeapTuple: カタログタプルの表現。

失敗モードと境界条件

  • **ラベル重複**: 既に存在するラベルを追加しようとした場合（skipIfExists が false ならエラー）。
  • **精度不足**: float4 の精度限界により新しい enumsortorder が一意に定まらない場合、RenumberEnumType による再採番が実行される。
  • **バイナリアップグレード**: IsBinaryUpgrade モードにおいて OID が設定されていない場合はエラーとなる。

保守担当者向けメモ

  • enumsortorder の計算は float4 の精度に依存するため、ラベル数が非常に多い場合に再採番コストが発生する点に注意してください。
  • トランザクション内での整合性を保つため、LockDatabaseObject による排他ロックが必須です。
