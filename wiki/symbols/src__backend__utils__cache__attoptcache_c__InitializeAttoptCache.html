InitializeAttoptCache

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/attoptcache.c:96

目的

属性オプション（attoptions）を管理するためのハッシュテーブル AttoptCacheHash を初期化し、システムキャッシュの無効化イベントに対するコールバックを登録する。

シグネチャまたは形状

static void InitializeAttoptCache(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 AttoptCacheHash を初期化）

前提条件と不変条件

  • CacheMemoryContext が存在しない場合は、本関数内で CreateCacheMemoryContext() を呼び出し生成する。
  • AttoptCacheHash は HASH_ELEM および HASH_FUNCTION フラグ付きで作成される。

アルゴリズムまたは振る舞い

1. HASHCTL 構造体を設定し、キーサイズを AttoptCacheKey、エントリサイズを AttoptCacheEntry に指定する。 2. ハッシュ関数として relatt_cache_syshash を指定し、システムキャッシュと同一のハッシュ計算ロジックを適用する。 3. hash_create を呼び出し、初期サイズ 256 で AttoptCacheHash を生成する。 4. CacheMemoryContext の存在を確認し、なければ作成する。 5. CacheRegisterSyscacheCallback を使用し、ATTNUM システムキャッシュの更新時に InvalidateAttoptCacheCallback が実行されるよう登録する。

呼び出し元と呼び出し先

  • 呼び出し元: get_attribute_options（キャッシュの遅延初期化時に呼び出される）
  • 呼び出し先: hash_create, CreateCacheMemoryContext, CacheRegisterSyscacheCallback

関連データ構造

  • AttoptCacheHash: 属性オプションを保持するハッシュテーブル。
  • AttoptCacheKey: ハッシュテーブルの検索キー。
  • AttoptCacheEntry: ハッシュテーブルに格納されるエントリ。
  • relatt_cache_syshash: ハッシュ計算用関数。

失敗モードと境界条件

  • メモリ不足時には hash_create または CreateCacheMemoryContext がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は get_attribute_options からの初回呼び出し時にのみ実行される遅延初期化の役割を持つ。
  • relatt_cache_syshash を使用することで、システムキャッシュとの検索効率の整合性を保っている。
  • キャッシュエントリの無効化は ATTNUM システムキャッシュの変更通知に依存しているため、関連するカタログ更新処理との整合性に注意すること。
