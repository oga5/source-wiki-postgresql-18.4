RelationDropStorage

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/storage.c:206

目的

リレーションの物理ストレージを、現在のトランザクションがコミットされた際に削除するよう予約する。

シグネチャまたは形状

void RelationDropStorage(Relation rel)

入力と出力

  • **入力**: rel (削除対象のリレーションを指す Relation 構造体)
  • **出力**: なし

前提条件と不変条件

  • TopMemoryContext が有効であること。
  • 呼び出し後、リレーションのストレージマネージャ（smgr）はクローズされる。

アルゴリズムまたは振る舞い

1. TopMemoryContext 上に PendingRelDelete 構造体を割り当てる。 2. リレーションのロケータ (rd_locator) とバックエンド番号 (rd_backend) を構造体に格納する。 3. atCommit フラグを true に設定し、現在のトランザクションのネストレベルを記録する。 4. 構造体をグローバルな pendingDeletes リストの先頭に追加する。 5. RelationCloseSmgr(rel) を呼び出し、リレーションに関連付けられたストレージマネージャを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c
  • src/backend/catalog/heap.c
  • src/backend/catalog/index.c
  • src/backend/commands/tablecmds.c
  • src/backend/utils/cache/relcache.c
  • **呼び出し先**:
  • MemoryContextAlloc
  • GetCurrentTransactionNestLevel
  • RelationCloseSmgr

関連データ構造

  • PendingRelDelete: 削除待ちリレーションの情報を保持するリスト構造。
  • Relation: リレーションのメタデータを保持する構造体。

失敗モードと境界条件

  • 同一トランザクション内で作成されたリレーションに対して本関数を呼ぶと、pendingDeletes リストに重複して登録される可能性がある。この場合、smgrDoPendingDeletes 実行時に重複エントリが無視されるため、エラーにはならない。

保守担当者向けメモ

  • 本関数は物理ファイルを即座に削除せず、トランザクション終了時の削除予約を行う。
  • pendingDeletes リストは TopMemoryContext に保持されるため、トランザクション終了後も生存する。
