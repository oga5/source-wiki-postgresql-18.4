ExecBitmapIndexScanInitializeWorker

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapIndexscan.c:405

目的

並列クエリ実行時、各ワーカープロセスにおいて BitmapIndexScan ノードの統計情報共有領域を初期化する。

シグネチャまたは形状

void ExecBitmapIndexScanInitializeWorker(BitmapIndexScanState *node, ParallelWorkerContext *pwcxt)

入力と出力

  • **入力**:
  • node: 初期化対象の BitmapIndexScanState 構造体。
  • pwcxt: 並列ワーカーのコンテキスト情報（共有メモリのTOCを含む）。
  • **出力**: なし（副作用として node->biss_SharedInfo が設定される）。

前提条件と不変条件

  • node->ss.ps.instrument が真である場合にのみ実行される。
  • pwcxt->toc に、現在のプランノードIDに対応する SharedIndexScanInstrumentation が存在している必要がある。

アルゴリズムまたは振る舞い

1. node->ss.ps.instrument を確認し、計測が有効でない場合は即座に終了する。 2. pwcxt->toc から、現在のプランノードID (node->ss.ps.plan->plan_node_id) をキーとして共有メモリ上の統計情報領域を検索する。 3. 取得したポインタを node->biss_SharedInfo に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:1356（並列実行の初期化プロセスから呼び出される）。
  • **呼び出し先**: shm_toc_lookup（共有メモリのTOC検索）。

関連データ構造

  • BitmapIndexScanState: スキャンノードの実行状態を保持する構造体。
  • ParallelWorkerContext: 並列ワーカーの共有メモリ管理コンテキスト。
  • SharedIndexScanInstrumentation: 並列ワーカー間で共有されるインデックススキャンの統計情報。

失敗モードと境界条件

  • shm_toc_lookup の第3引数が false で指定されているため、対応するノードIDがTOCに見つからない場合はエラーが発生する。

保守担当者向けメモ

  • 本関数は ExecProcNode インターフェースの一部ではなく、並列クエリのセットアップフェーズでのみ使用される。
  • 統計情報の計測が不要なプランでは本関数はスキップされるため、計測機能の拡張を行う際は node->ss.ps.instrument のチェックロジックに注意すること。
