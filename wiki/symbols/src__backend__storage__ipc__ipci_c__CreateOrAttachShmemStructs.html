CreateOrAttachShmemStructs

定義

  • **種別**: function
  • **定義箇所**: src/backend/storage/ipc/ipci.c:267

目的

PostgreSQLの各サブシステムが必要とする共有メモリ上のデータ構造を初期化、または既存の共有メモリ領域にアタッチする。

シグネチャまたは形状

static void CreateOrAttachShmemStructs(void)

入力と出力

  • **入力**: なし
  • **出力**: なし（副作用として共有メモリ上の各サブシステム用データ構造が初期化される）

前提条件と不変条件

  • 本関数が呼び出される前に、共有メモリセグメントの確保と基本的なメモリ管理機構（InitShmemAllocation）が完了している必要がある。
  • CreateLWLocks が最初に実行され、後続の InitShmemIndex 等に必要なロック機構が利用可能であること。

アルゴリズムまたは振る舞い

1. **基盤初期化**: CreateLWLocks で軽量ロックを生成し、InitShmemIndex で共有メモリインデックスハッシュテーブルを構築する。 2. **動的共有メモリ**: dsm_shmem_init および DSMRegistryShmemInit を実行。 3. **主要サブシステム初期化**: WAL、バッファマネージャ、ロックマネージャ、プロセス管理、共有インバリデーション、シグナル管理、および各拡張モジュール関連の ShmemInit 関数を順次呼び出す。 4. **プロセス管理**: IsUnderPostmaster が偽の場合のみ InitProcGlobal を実行し、その後 ProcArrayShmemInit 等のプロセス関連構造体を初期化する。 5. **その他**: 最後に BTreeShmemInit や StatsShmemInit 等の補助的なモジュールを初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/ipci.c 内の CreateSharedMemoryAndSemaphores（185行目および240行目付近）から呼び出される。
  • **呼び出し先**: 各サブシステムの ShmemInit 関数群（例: BufferManagerShmemInit, LockManagerShmemInit 等）および InitShmemIndex, InitProcGlobal。

関連データ構造

  • ShmemIndex（共有メモリ内のハッシュテーブル）
  • 各サブシステムが定義する共有メモリ制御構造体（ProcArrayStruct, LockManager 等）

失敗モードと境界条件

  • 共有メモリ領域が不足している場合、各 ShmemInit 関数内でパニックが発生する可能性がある。
  • IsUnderPostmaster の状態により InitProcGlobal の実行有無が分岐するため、呼び出し時のプロセス状態に依存する。

保守担当者向けメモ

  • 新しいサブシステムを追加し、共有メモリを使用する場合は、必ず本関数内で対応する ShmemInit 関数を呼び出す必要がある。
  • EXEC_BACKEND 環境（Windows等）では、子プロセスも本関数を通過するため、各 ShmemInit 関数は冪等性を保持するように実装されている必要がある。
