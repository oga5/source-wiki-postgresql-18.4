euc_jis_20042shift_jis_2004

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/mb/conversion_procs/euc2004_sjis2004/euc2004_sjis2004.c:77

目的

EUC_JIS_2004 エンコーディングの文字列を SHIFT_JIS_2004 エンコーディングへ変換する。

シグネチャまたは形状

int euc_jis_20042shift_jis_2004(const unsigned char *euc, unsigned char *p, int len, bool noError)

入力と出力

  • **入力**:
  • euc: 変換元文字列へのポインタ。
  • p: 変換先バッファへのポインタ。
  • len: 変換対象のバイト長。
  • noError: 不正なシーケンス検出時にエラーを報告せず中断するかを指定するフラグ。
  • **出力**: 変換処理したバイト数を返す。変換先バッファ p はヌル終端される。

前提条件と不変条件

  • euc は有効な EUC_JIS_2004 文字列であること。
  • 変換先バッファ p は、入力文字列を格納するのに十分なサイズを確保していること。

アルゴリズムまたは振る舞い

1. 入力文字列を 1 文字ずつ走査し、pg_encoding_verifymbchar で妥当性を検証する。 2. ASCII 文字はそのままコピーする。 3. SS2 (0x8e) で始まる文字は JIS X 0201 カナとして処理する。 4. SS3 (0x8f) で始まる文字は JIS X 0213 第 2 水準として、区点位置に基づきビット演算で SHIFT_JIS_2004 へ変換する。 5. それ以外（第 1 水準）は、同様に区点位置から SHIFT_JIS_2004 へ変換する。 6. 不正なバイト列が検出された場合、noError が false なら report_invalid_encoding を呼び出し、true なら処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**: euc_jis_2004_to_shift_jis_2004 (同ファイル内) から呼び出される。
  • **呼び出し先**: pg_encoding_verifymbchar および report_invalid_encoding を呼び出す。

関連データ構造

  • PG_EUC_JIS_2004 (エンコーディング ID)

失敗モードと境界条件

  • 入力文字列が不正なマルチバイトシーケンスを含む場合、noError フラグに従いエラー報告または処理中断を行う。
  • ku（区）や ten（点）が仕様の範囲外である場合、不正なエンコーディングとして処理される。

保守担当者向けメモ

  • JIS X 0213 の規格に基づいたマッピング実装である。
  • ku と ten の計算は、JIS コードと Shift_JIS のオフセットに基づいているため、変換ロジック変更時は規格書との照合が必須である。
