PgBenchValueType

定義

  • **種別**: enum
  • **定義箇所**: src/bin/pgbench/pgbench.h:34

目的

pgbench の式評価エンジンにおいて、評価結果である PgBenchValue が保持するデータの型を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
	PGBT_NO_VALUE = 0,
	PGBT_NULL,
	PGBT_INT,
	PGBT_DOUBLE,
	PGBT_BOOLEAN,
} PgBenchValueType;

入力と出力

  • **入力**: なし（列挙型定義）
  • **出力**: 式評価の結果として、PgBenchValue 構造体の type フィールドに格納される値。

前提条件と不変条件

  • PgBenchValue 構造体は共用体（union）を使用しているため、値にアクセスする際は必ずこの列挙型で型を確認する必要があります。

アルゴリズムまたは振る舞い

  • PGBT_NO_VALUE: 値が未設定であることを示します。
  • PGBT_NULL: SQL の NULL 値を表します。
  • PGBT_INT: 64ビット整数型。
  • PGBT_DOUBLE: 倍精度浮動小数点型。
  • PGBT_BOOLEAN: 真偽値。

呼び出し元と呼び出し先

  • **呼び出し元**: PgBenchValue 構造体の定義（src/bin/pgbench/pgbench.h:46）において、型識別子として参照されています。
  • **呼び出し先**: 情報なし。

関連データ構造

  • PgBenchValue: 本列挙型をメンバとして保持し、実際の値を格納する構造体。

失敗モードと境界条件

  • 型の不一致: PgBenchValue の type フィールドと、共用体内の実際のデータ型が一致しない場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 新しいデータ型を追加する場合は、本列挙型にシンボルを追加し、pgbench.c 側の評価ロジックおよび型変換処理を更新してください。
