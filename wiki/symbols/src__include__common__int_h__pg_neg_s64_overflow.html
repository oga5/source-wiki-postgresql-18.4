pg_neg_s64_overflow

定義

  • 種別: function
  • 定義箇所: src/include/common/int.h:335

目的

64ビット符号付き整数（int64）の符号反転時に発生するオーバーフローを検出し、安全に演算を行う。

シグネチャまたは形状

static inline bool pg_neg_s64_overflow(int64 a, int64 *result)

入力と出力

  • **入力**:
  • a: 符号反転対象の int64 値。
  • result: 演算結果を格納する int64 へのポインタ。
  • **出力**:
  • 戻り値: オーバーフローが発生した場合は true、それ以外は false を返す。
  • *result: 演算成功時は -a の値が格納される。オーバーフロー時はダミー値 0x5EED が格納される。

前提条件と不変条件

  • int64 型の範囲内での演算を前提とする。
  • 2の補数表現において、PG_INT64_MIN の符号反転は表現可能な範囲を超えるため、オーバーフローとみなされる。

アルゴリズムまたは振る舞い

1. HAVE__BUILTIN_OP_OVERFLOW が定義されている環境では、コンパイラ組み込み関数 __builtin_sub_overflow(0, a, result) を使用して演算とチェックを同時に行う。 2. 上記が利用できない環境では、以下のロジックを実行する:

  • a が PG_INT64_MIN と等しい場合、オーバーフローとして true を返し、result に 0x5EED を書き込む。
  • それ以外の場合、*result = -a を実行し、false を返す。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは具体的な呼び出し元は特定できません。
  • 呼び出し先: コンパイラ組み込み関数（__builtin_sub_overflow）を使用する場合があります。

関連データ構造

  • int64: PostgreSQL で定義される 64ビット符号付き整数型。

失敗モードと境界条件

  • a が PG_INT64_MIN の場合、符号反転は正の範囲に収まらないため、本関数は失敗（true）を返します。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、ヘッダーファイル src/include/common/int.h で提供されています。
  • 0x5EED という値は、オーバーフロー発生時のデバッグおよび警告回避のために使用される定数です。
