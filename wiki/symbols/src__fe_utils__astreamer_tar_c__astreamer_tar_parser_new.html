astreamer_tar_parser_new

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_tar.c:92

目的

tarアーカイブのストリーム解析を行うための astreamer_tar_parser インスタンスを生成し、初期化します。

シグネチャまたは形状

astreamer *astreamer_tar_parser_new(astreamer *next)

入力と出力

  • **入力**: next (解析結果を渡す後続の astreamer インスタンス)
  • **出力**: 初期化された astreamer 構造体へのポインタ。

前提条件と不変条件

  • next は有効な astreamer インスタンスである必要があります。
  • 内部状態 next_context は、解析開始時に ASTREAMER_MEMBER_HEADER に設定されます。

アルゴリズムまたは振る舞い

1. palloc0 を使用して astreamer_tar_parser 構造体分のメモリを確保します。 2. base.bbs_ops に astreamer_tar_parser_ops を割り当て、操作関数を紐付けます。 3. 引数 next を base.bbs_next に格納し、ストリーマーのパイプラインを構築します。 4. initStringInfo を呼び出し、内部バッファを初期化します。 5. 解析状態をヘッダー読み取りモード（ASTREAMER_MEMBER_HEADER）に設定して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c:1257
  • src/bin/pg_verifybackup/pg_verifybackup.c:1304
  • **呼び出し先**:
  • palloc0 (メモリ確保)
  • initStringInfo (バッファ初期化)

関連データ構造

  • astreamer_tar_parser: 本関数で生成される状態保持構造体。
  • astreamer: ストリーマーの基本インターフェース。
  • astreamer_ops: ストリーマーの操作関数テーブル。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • next が NULL の場合、後続の処理で不正なメモリアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本関数は astreamer パイプラインの構築時に呼び出されます。
  • astreamer_tar_parser_ops の定義変更時には、本関数での初期化ロジックとの整合性に注意してください。
