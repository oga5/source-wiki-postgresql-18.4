parse_error_callback_arg

定義

  • 種別: struct
  • 定義箇所: src/backend/catalog/pg_proc.c:47

目的

関数定義の解析中にエラーが発生した際、エラーコンテキストを保持し、エラーメッセージを元のクエリ文字列内の適切な位置へマッピングするための引数構造体です。

シグネチャまたは形状

typedef struct
{
	char	   *proname;
	char	   *prosrc;
} parse_error_callback_arg;

入力と出力

  • proname: エラーが発生した関数名（char *）。
  • prosrc: エラーが発生した関数本体のソースコード文字列（char *）。

前提条件と不変条件

  • proname および prosrc は、エラーコールバック関数が呼び出される間、有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

この構造体は、sql_function_parse_error_callback 等のエラーハンドラに渡されるコンテキストとして使用されます。エラー発生時に、この構造体に格納された情報を用いて、関数本体内のエラー位置を元のSQL文のコンテキストに変換（transpose）する処理を支援します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/pg_proc.c 内の関数定義処理（ProcedureCreate 等）において、エラーコールバックの設定時にインスタンス化されます。
  • 呼び出し先: function_parse_error_transpose 等のエラー処理ユーティリティによって、エラー位置の特定のために参照されます。

関連データ構造

  • sql_function_parse_error_callback: 本構造体を利用してエラーコンテキストを構築するコールバック関数。

失敗モードと境界条件

  • proname または prosrc が NULL の場合、エラーハンドラ内での参照時にクラッシュする可能性があるため、初期化時に適切な値が設定されている必要があります。

保守担当者向けメモ

  • 本構造体は pg_proc.c 内の局所的なエラーハンドリング専用です。
  • エラー位置の変換ロジックを変更する際は、prosrc が指す文字列の整合性に注意してください。
