transformContainerSubscripts

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_node.c:242

目的

配列やその他の添字可能なデータ型に対する添字操作を解析し、実行可能な SubscriptingRef ノードを生成する。

シグネチャまたは形状

SubscriptingRef *transformContainerSubscripts(
    ParseState *pstate,
    Node *containerBase,
    Oid containerType,
    int32 containerTypMod,
    List *indirection,
    bool isAssignment
)

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する ParseState 構造体。
  • containerBase: 添字対象となる式ノード。
  • containerType: コンテナの型 OID。
  • containerTypMod: コンテナの型修飾子。
  • indirection: 添字またはスライス指定のリスト。
  • isAssignment: 代入操作（更新）かどうかを示すフラグ。
  • **出力**: 構築された SubscriptingRef ノードへのポインタ。

前提条件と不変条件

  • isAssignment が false の場合、関数内で transformContainerType を呼び出し、ドメイン型を解決して実際のコンテナ型を特定する。
  • コンテナ型は getSubscriptingRoutines によってサポートされている必要がある。

アルゴリズムまたは振る舞い

1. 必要に応じて transformContainerType を呼び出し、型を解決する。 2. getSubscriptingRoutines を使用して、対象型が添字操作をサポートしているか検証し、サポート関数を取得する。 3. indirection リストを走査し、スライス指定（lower:upper）が含まれているか判定する。 4. SubscriptingRef ノードを生成し、型情報や式ノードを割り当てる。 5. コンテナ型固有の transform 関数を呼び出し、添字の変換と結果の型決定を行う。 6. 最終的に refrestype が有効であることを確認し、ノードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_expr.c (式解析時)
  • src/backend/parser/parse_target.c (ターゲットリスト解析時)
  • **呼び出し先**:
  • transformContainerType
  • getSubscriptingRoutines
  • sbsroutines->transform (コンテナ型固有の変換関数)
  • ereport (エラー報告)

関連データ構造

  • SubscriptingRef: 添字操作を表すノード。
  • SubscriptRoutines: コンテナ型ごとの添字操作を定義する構造体。
  • ParseState: 解析セッションの状態管理。
  • A_Indices: 構文解析器から渡される添字指定。

失敗モードと境界条件

  • **データ型不一致**: コンテナ型が添字操作をサポートしていない場合、ERRCODE_DATATYPE_MISMATCH でエラーを報告する。
  • **型解決失敗**: コンテナ固有の変換後に refrestype が無効な場合、同様にエラーを報告する。

保守担当者向けメモ

  • refcollid は parse_collate.c で設定されるため、本関数内では設定されない。
  • refassgnexpr は代入操作の場合に呼び出し元で設定される。
  • refrestype の検証は、不完全なハンドラ実装（typsubscript はあるが typelem がない等）に対する防御的措置である。
