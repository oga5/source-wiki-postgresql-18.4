CopyToRoutine

定義

  • 種別: struct
  • 定義箇所: src/include/commands/copyapi.h:23

目的

PostgreSQL の COPY TO コマンドにおけるデータ形式（CSV, Text, Binary 等）を抽象化し、プラグイン可能なインターフェースを提供するための構造体です。フォーマット固有のロジックをコアエンジンから分離します。

シグネチャまたは形状

typedef struct CopyToRoutine
{
    void (*CopyToOutFunc) (CopyToState cstate, Oid atttypid, FmgrInfo *finfo);
    void (*CopyToStart) (CopyToState cstate, TupleDesc tupDesc);
    void (*CopyToOneRow) (CopyToState cstate, TupleTableSlot *slot);
    void (*CopyToEnd) (CopyToState cstate);
} CopyToRoutine;

入力と出力

  • **CopyToOutFunc**: 出力関数の情報を設定します。cstate（状態）、atttypid（属性のOID）、finfo（出力関数のカタログ情報）を入力として受け取ります。
  • **CopyToStart**: COPY TO の開始処理を行います。cstate と tupDesc（読み込み対象リレーションのタプル記述子）を入力として受け取ります。
  • **CopyToOneRow**: slot に格納された行データを宛先へ書き出します。
  • **CopyToEnd**: COPY TO の終了処理を行い、リソースの解放や後処理を行います。

前提条件と不変条件

  • 本構造体はサーバーの生存期間中保持されるため、通常は static const として定義する必要があります。
  • 各コールバック関数に渡される cstate は、フォーマット固有のプライベートデータへのアクセスポイントとして機能します。

アルゴリズムまたは振る舞い

1. **初期化**: CopyToStart が呼び出され、フォーマット固有の初期化が行われます。 2. **ループ**: CopyToOneRow が繰り返し呼び出され、データがストリームまたはテーブルスロット間で転送されます。 3. **終了**: CopyToEnd により後処理が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyto.c 内の COPY TO 実行ロジックから参照されます。
  • **呼び出し先**: CopyToRoutineText、CopyToRoutineCSV、CopyToRoutineBinary などの具体的な実装インスタンスが定義されており、これらが各フォーマットの処理を担います。

関連データ構造

  • CopyToState: COPY TO の状態管理を行う構造体。
  • TupleDesc: タプルの記述子。
  • TupleTableSlot: 行データを保持するスロット。
  • FmgrInfo: 関数マネージャ情報。

失敗モードと境界条件

  • 構造体内の各関数ポインタが適切に初期化されていない場合、実行時に未定義の動作を引き起こす可能性があります。
  • cstate が無効な場合、各コールバック内でのメモリ参照エラーが発生する可能性があります。

保守担当者向けメモ

  • 新しいデータフォーマットを追加する際は、本構造体を実装し、コアの COPY ロジックから呼び出されるように登録してください。
  • CopyToRoutine はインターフェース定義であるため、実装側で各関数ポインタが適切に割り当てられていることを確認してください。
