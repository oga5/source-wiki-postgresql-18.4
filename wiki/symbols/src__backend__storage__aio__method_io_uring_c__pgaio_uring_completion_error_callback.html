pgaio_uring_completion_error_callback

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_io_uring.c:503

目的

エラー発生時に、現在処理中の I/O がどのプロセスに代わって実行されているかをエラーコンテキストへ追加するコールバック関数です。

シグネチャまたは形状

static void pgaio_uring_completion_error_callback(void *arg)

入力と出力

  • **入力**: arg (型: void *) - PgAioHandle 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • arg が NULL の場合、関数は即座に終了します。
  • ioh->owner_procno が現在のプロセス (MyProcNumber) と一致する場合、エラーコンテキストの追加は行われません。

アルゴリズムまたは振る舞い

1. 引数 arg を PgAioHandle * としてキャストします。 2. ioh が NULL であれば終了します。 3. ioh->owner_procno が MyProcNumber と等しい場合、自プロセスの I/O であるため終了します。 4. それ以外の場合、GetPGProcByNumber を使用して I/O 所有者の PGPROC 構造体を取得します。 5. 所有者の PID を特定し、errcontext を呼び出して「completing I/O on behalf of process [PID]」というメッセージをエラースタックに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていませんが、errcallback.callback フィールドに本関数が代入されている箇所 (src/backend/storage/aio/method_io_uring.c:534) が確認されています。
  • **呼び出し先**: GetPGProcByNumber および errcontext を呼び出します。

関連データ構造

  • PgAioHandle: I/O 操作の状態を保持する構造体。
  • PGPROC: PostgreSQL のバックエンドプロセス情報を保持する構造体。

失敗モードと境界条件

  • arg が PgAioHandle 以外の型を指している場合、または ioh->owner_procno が無効なプロセス番号である場合の挙動は未定義です。

保守担当者向けメモ

  • 本関数はエラー発生時のデバッグ情報を補完するためのものです。io_uring の完了処理中にエラーが発生した際、どのプロセスの I/O が原因かを特定するために使用されます。
