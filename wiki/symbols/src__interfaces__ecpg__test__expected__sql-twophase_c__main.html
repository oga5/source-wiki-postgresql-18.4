目的

ECPG（Embedded C for PostgreSQL）における二相コミット機能の動作を検証するための回帰テストを実行する。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（引数を受け取らない）。
  • 出力: 成功時に 0 を返す。SQL操作でエラーが発生した場合は sqlprint() により標準エラー出力へエラー情報が出力される。

前提条件と不変条件

  • ECPGdebug(1, stderr) により、実行時の詳細なデバッグログが標準エラー出力に有効化される。
  • 各SQL操作の直後に sqlca.sqlcode を評価し、負の値（エラー）である場合に sqlprint() を呼び出す必要がある。

アルゴリズムまたは振る舞い

1. **接続と設定**: ECPGconnect でデータベース ecpg1_regression に接続し、ECPGsetcommit で自動コミットを無効化する。 2. **テーブル作成**: create table t1 ( c int ) を実行する。 3. **トランザクション制御**:

  • commit を発行して初期状態を確定。
  • begin を発行してトランザクションを開始。
  • insert into t1 values ( 1 ) を実行。

    • 4. **二相コミットの実行**:

  • prepare transaction 'ecpg_twophase' を実行し、トランザクションを準備状態へ移行。
  • commit prepared 'ecpg_twophase' を実行し、準備済みトランザクションを確定。

    • 5. **終了処理**: drop table t1 を実行し、ECPGdisconnect で接続を解除する。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内に情報なし。
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGsetcommit, ECPGdo, ECPGtrans, ECPGdisconnect, sqlprint（すべてECPGランタイムライブラリ関数）。

関連データ構造

  • sqlca: SQL通信領域（SQL Communication Area）。各操作後の sqlcode を保持し、エラー判定に使用される。

失敗モードと境界条件

  • 各ステップで sqlca.sqlcode < 0 となった場合、エラーとして sqlprint() が実行される。
  • トランザクション名 'ecpg_twophase' の一貫性が失われた場合、PREPARE または COMMIT PREPARED が失敗する可能性がある。
