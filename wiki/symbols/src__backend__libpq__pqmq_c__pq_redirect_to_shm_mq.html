pq_redirect_to_shm_mq

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqmq.c:52

目的

バックエンドプロセスの通信メソッドを、通常のネットワークソケットから共有メモリメッセージキュー（shm_mq）へ切り替えるためのエントリポイントです。

シグネチャまたは形状

void pq_redirect_to_shm_mq(dsm_segment *seg, shm_mq_handle *mqh)

入力と出力

  • **入力**:
  • seg: 共有メモリセグメントへのポインタ。
  • mqh: 共有メモリメッセージキューのハンドル。
  • **出力**: なし（void）

前提条件と不変条件

  • PqCommMethods が共有メモリ通信用の PqCommMqMethods に置き換えられます。
  • whereToSendOutput が DestRemote に設定され、フロントエンドプロトコルが PG_PROTOCOL_LATEST に固定されます。

アルゴリズムまたは振る舞い

1. グローバル変数 PqCommMethods を PqCommMqMethods に書き換えます。 2. 共有メモリキューハンドル pq_mq_handle を引数 mqh で更新します。 3. 出力先を DestRemote に設定し、プロトコルバージョンを最新に設定します。 4. on_dsm_detach を呼び出し、共有メモリセグメントがデタッチされる際に pq_cleanup_redirect_to_shm_mq が実行されるよう登録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c:1382
  • src/backend/replication/logical/applyparallelworker.c:951
  • **呼び出し先**:
  • on_dsm_detach (PostgreSQL内部のDSM管理関数)

関連データ構造

  • dsm_segment: 共有メモリセグメント。
  • shm_mq_handle: 共有メモリメッセージキューの制御ハンドル。
  • PqCommMethods: 通信メソッドを定義する関数ポインタテーブル。

失敗モードと境界条件

  • 共有メモリセグメントが不正な場合、on_dsm_detach の登録処理でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は通信経路の切り替えを行うため、並列クエリや論理レプリケーションワーカーの初期化プロセスにおいて重要な役割を果たします。
  • pq_cleanup_redirect_to_shm_mq によるクリーンアップ処理が正しく登録されるため、メモリリークや不正な通信状態の永続化を防ぐ設計となっています。
