pg_get_wait_events

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/wait_event_funcs.c:45

目的

システム定義および動的に追加された待機イベントのメタデータ（型、名称、説明）を一覧として取得し、SQLインターフェースを通じて提供する。

シグネチャまたは形状

Datum pg_get_wait_events(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出し時の引数）
  • **出力**: Datum（戻り値は常に (Datum) 0 だが、内部的に tuplestore を介してセットリターン関数として行を返却する）

前提条件と不変条件

  • InitMaterializedSRF を呼び出し、結果を格納するためのタプルストアを初期化する必要がある。
  • 戻り値の各行は、型（type）、名称（name）、説明（description）の3列で構成される。

アルゴリズムまたは振る舞い

1. InitMaterializedSRF を呼び出し、結果セットを保持するタプルストアを準備する。 2. 静的配列 waitEventData を走査し、各要素をタプルストアに追加する。 3. GetWaitEventCustomNames を使用して、拡張機能（PG_WAIT_EXTENSION）由来のカスタム待機イベントを取得し、説明文を動的に生成してタプルストアに追加する。 4. 同様に、インジェクションポイント（PG_WAIT_INJECTIONPOINT）由来の待機イベントを取得し、説明文を生成してタプルストアに追加する。 5. 構築したタプルストアを返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（直接的な呼び出し元情報はソースコード内に明示されていない）。
  • **呼び出し先**:
  • InitMaterializedSRF
  • CStringGetTextDatum
  • tuplestore_putvalues
  • GetWaitEventCustomNames
  • initStringInfo
  • appendStringInfo

関連データ構造

  • waitEventData: システム定義の待機イベントを保持する静的配列。
  • ReturnSetInfo: セットリターン関数の結果情報を保持する構造体。
  • StringInfoData: 動的な説明文生成に使用されるバッファ。

失敗モードと境界条件

  • waitEventData の終端は type が NULL であることで判定される。
  • GetWaitEventCustomNames が返すリストが空の場合、そのカテゴリの処理はスキップされる。

保守担当者向けメモ

  • 本関数はセットリターン関数（SRF）として実装されているため、メモリ管理はPostgreSQLのタプルストア機構に依存している。
  • 新しい待機イベントカテゴリを追加する場合は、GetWaitEventCustomNames の呼び出しパターンを模倣すること。
