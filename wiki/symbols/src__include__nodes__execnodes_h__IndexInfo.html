IndexInfo

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/execnodes.h:193

目的

インデックスの構築、挿入、および検証処理において、インデックスの構造や制約、実行時状態を保持するために使用されるデータ構造です。

シグネチャまたは形状

typedef struct IndexInfo {
    NodeTag     type;
    int         ii_NumIndexAttrs;
    int         ii_NumIndexKeyAttrs;
    AttrNumber  ii_IndexAttrNumbers[INDEX_MAX_KEYS];
    List       *ii_Expressions;
    List       *ii_ExpressionsState;
    List       *ii_Predicate;
    ExprState  *ii_PredicateState;
    Oid        *ii_ExclusionOps;
    Oid        *ii_ExclusionProcs;
    uint16     *ii_ExclusionStrats;
    Oid        *ii_UniqueOps;
    Oid        *ii_UniqueProcs;
    uint16     *ii_UniqueStrats;
    bool        ii_Unique;
    bool        ii_NullsNotDistinct;
    bool        ii_ReadyForInserts;
    bool        ii_CheckedUnchanged;
    bool        ii_IndexUnchanged;
    bool        ii_Concurrent;
    bool        ii_BrokenHotChain;
    bool        ii_Summarizing;
    bool        ii_WithoutOverlaps;
    int         ii_ParallelWorkers;
    Oid         ii_Am;
    void       *ii_AmCache;
    MemoryContext ii_Context;
} IndexInfo;

入力と出力

本構造体は関数への引数として渡されることが多く、インデックスのメタデータ（属性番号、式、述語）を入力として保持し、実行時の状態（ii_AmCache や ii_Context）を管理します。

前提条件と不変条件

  • ii_Expressions と ii_ExpressionsState、および ii_Predicate と ii_PredicateState は、それぞれ式とその評価状態のペアとして整合性が保たれている必要があります。
  • ii_ExclusionOps 等の配列は、インデックスの列数に応じたサイズで確保されている必要があります。

アルゴリズムまたは振る舞い

  • インデックスの挿入や構築時に、インデックスキーの計算、述語の評価、一意性制約のチェック、排他制約の判定を行うための情報を供給します。
  • ii_AmCache を通じて、各インデックスアクセスメソッド（AM）固有のキャッシュデータを保持・再利用します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • contrib/amcheck/verify_nbtree.c
  • contrib/bloom/blinsert.c
  • src/backend/access/brin/brin.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、特定の関数を直接呼び出すものではありません。操作は BuildIndexInfo 等の構築関数や、各AMの挿入関数を介して行われます。

関連データ構造

  • ExprState: 式の評価状態を保持。
  • MemoryContext: インデックス関連情報のメモリ管理に使用。
  • List: インデックス式や述語のリスト管理に使用。

失敗モードと境界条件

  • INDEX_MAX_KEYS を超える属性数は扱えません。
  • メモリ不足時には ii_Context の割り当てやリスト操作でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は実行時専用のノードであり、copyObject や equal といったノードサポート関数はサポートしていません。
  • ii_AmCache のライフサイクルは ii_Context によって管理されることが想定されます。
