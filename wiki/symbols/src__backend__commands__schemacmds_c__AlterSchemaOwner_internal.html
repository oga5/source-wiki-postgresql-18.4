AlterSchemaOwner_internal

定義

  • **種別**: function
  • **定義箇所**: src/backend/commands/schemacmds.c:360

目的

スキーマの所有者を変更し、システムカタログの更新、ACL（アクセス制御リスト）の再計算、および所有権に関連する依存関係の更新を行う内部関数です。

シグネチャまたは形状

static void AlterSchemaOwner_internal(HeapTuple tup, Relation rel, Oid newOwnerId)

入力と出力

  • **入力**:
  • tup: pg_namespace カタログの対象スキーマタプル（HeapTuple）。
  • rel: pg_namespace リレーション（Relation）。
  • newOwnerId: 新しい所有者のロールID（Oid）。
  • **出力**: なし（副作用としてシステムカタログを更新）。

前提条件と不変条件

  • tup および rel は NamespaceRelationId（pg_namespace）を参照している必要があります。
  • 呼び出し元で適切なロックが取得されている必要があります。
  • 現在のユーザーが対象スキーマの所有者であるか、スーパーユーザーである必要があります。
  • 現在のユーザーが newOwnerId にロールを変更する権限を持っている必要があります。
  • 現在のデータベースに対して ACL_CREATE 権限が必要です。

アルゴリズムまたは振る舞い

1. **所有者チェック**: 現在の所有者と newOwnerId が同一であれば処理を終了します。 2. **権限検証**: object_ownercheck でスキーマ所有権を、check_can_set_role でロール変更権限を、object_aclcheck でデータベースへの作成権限を検証します。 3. **ACL更新**: スキーマにACLが設定されている場合、aclnewowner を使用して所有者変更に伴うACLの再計算を行います。 4. **カタログ更新**: heap_modify_tuple と CatalogTupleUpdate を使用して pg_namespace を更新します。 5. **依存関係更新**: changeDependencyOnOwner を呼び出し、所有者変更を依存関係システムに反映させます。 6. **フック実行**: 最後に InvokeObjectPostAlterHook を呼び出し、イベントトリガーやキャッシュ無効化を通知します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/schemacmds.c 内の AlterSchemaOwner および関連する所有者変更処理から呼び出されます。
  • **呼び出し先**: object_ownercheck, check_can_set_role, object_aclcheck, aclnewowner, CatalogTupleUpdate, changeDependencyOnOwner, InvokeObjectPostAlterHook 等。

関連データ構造

  • pg_namespace: スキーマ情報を保持するシステムカタログ。
  • Acl: アクセス制御リスト。

失敗モードと境界条件

  • 権限不足の場合、aclcheck_error によりエラーが発生します。
  • 新しい所有者が存在しない、またはロール変更が許可されていない場合、エラーが発生します。
  • データベースに対する ACL_CREATE 権限がない場合、エラーが発生します。

保守担当者向けメモ

  • 本関数は ALTER SCHEMA ... OWNER TO コマンドのコアロジックをカプセル化しています。
  • ACLの再計算は aclnewowner に依存しており、スキーマ固有の権限移行ルールに従います。
  • 依存関係の更新は changeDependencyOnOwner を介して行われるため、他のオブジェクトタイプとの整合性に注意してください。
