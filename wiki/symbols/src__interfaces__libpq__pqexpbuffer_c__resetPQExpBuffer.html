resetPQExpBuffer

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/pqexpbuffer.c:145

目的

PQExpBuffer バッファの内容をリセットし、空の状態に戻します。メモリ不足（OOM）により「broken」状態となったバッファの復旧も試みます。

シグネチャまたは形状

void resetPQExpBuffer(PQExpBuffer str)

入力と出力

  • **入力**: str (PQExpBuffer) - 操作対象のバッファ構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • str が NULL の場合、何も行われません。
  • バッファが正常な場合、len は 0 に設定され、data[0] にヌル終端文字が書き込まれます。
  • バッファが oom_buffer を指している（broken状態）場合、initPQExpBuffer を用いて再初期化が試みられます。

アルゴリズムまたは振る舞い

1. 引数 str が有効か確認します。 2. str->data が oom_buffer と等しいか判定します。

  • 等しくない場合（正常状態）: str->len を 0 にし、str->data[0] を '\0' に設定します。
  • 等しい場合（broken状態）: initPQExpBuffer(str) を呼び出し、バッファの再初期化を試みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/pg_basebackup/pg_createsubscriber.c
  • src/bin/pg_basebackup/pg_recvlogical.c
  • src/bin/pg_dump/dumputils.c
  • src/bin/pg_dump/parallel.c
  • src/bin/pg_dump/pg_backup_db.c
  • **呼び出し先**:
  • initPQExpBuffer (broken状態からの復旧時)

関連データ構造

  • PQExpBuffer: 動的文字列バッファを管理する構造体。
  • oom_buffer: メモリ不足時に使用される静的な空文字列バッファ。

失敗モードと境界条件

  • str が NULL の場合、関数は安全に早期リターンします。
  • broken状態から initPQExpBuffer を呼び出した際、内部的なメモリ確保に失敗した場合は再びbroken状態となる可能性があります。

保守担当者向けメモ

  • 本関数は、メモリ不足で停止したバッファを再利用可能な状態へ復帰させるための唯一の手段です。
  • PQExpBuffer の内部構造（data, len）に直接アクセスするため、構造体の定義変更時には注意が必要です。
