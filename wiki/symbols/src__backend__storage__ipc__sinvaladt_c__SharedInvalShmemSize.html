SharedInvalShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/sinvaladt.c:217

目的

共有キャッシュ無効化（Shared Cache Invalidation）機構で使用される共有メモリ領域の総サイズを計算する。

シグネチャまたは形状

Size SharedInvalShmemSize(void)

入力と出力

  • 入力: なし
  • 出力: Size 型の共有メモリ所要バイト数

前提条件と不変条件

  • NumProcStateSlots 定数に基づき、固定数のバックエンド状態を格納するための領域を算出する。

アルゴリズムまたは振る舞い

1. SISeg 構造体の先頭部分（procState フィールドの手前まで）のサイズを取得する。 2. ProcState 構造体の配列（NumProcStateSlots 分）のサイズを加算する。 3. int 型の配列（NumProcStateSlots 分、pgprocnos 用）のサイズを加算する。 4. 計算結果を返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/ipc/ipci.c:133 (共有メモリ初期化時のサイズ計算)
  • src/backend/storage/ipc/sinvaladt.c:241 (ShmemInitStruct 呼び出し時)
  • 呼び出し先:
  • offsetof
  • add_size
  • mul_size

関連データ構造

  • SISeg: 共有メモリ上のメイン構造体。
  • ProcState: 各バックエンドの進捗状態を保持する構造体。

失敗モードと境界条件

  • add_size および mul_size 内部でオーバーフローが発生した場合、PostgreSQL の標準的なエラー処理（elog(ERROR, ...)）が呼び出される可能性がある。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウトに直接依存している。SISeg 構造体の定義が変更された場合は、本関数の計算ロジックも更新する必要がある。
