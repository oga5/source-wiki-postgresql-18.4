initcm

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_color.c:48

目的

正規表現コンパイル時に使用するカラーマップ（colormap）構造体を初期化し、デフォルト色（WHITE）を割り当てる。

シグネチャまたは形状

static void initcm(struct vars *v, struct colormap *cm)

入力と出力

  • **入力**:
  • v: コンパイル状態を保持する vars 構造体へのポインタ。
  • cm: 初期化対象の colormap 構造体へのポインタ。
  • **出力**: なし（cm が指す構造体を直接書き換える）。エラー発生時は CERR マクロを通じて v にエラーを記録する。

前提条件と不変条件

  • WHITE が 0 であることに依存して memset を実行する。
  • cm->cdspace が colormap 構造体内に確保されている必要がある。

アルゴリズムまたは振る舞い

1. cm のマジックナンバーと vars への参照を設定し、内部カウンタをリセットする。 2. WHITE（デフォルト色）に対応する colordesc を初期化する。 3. locolormap 用のメモリを確保し、全要素を WHITE で初期化する。 4. classbits をクリアし、範囲管理用の cmranges を NULL に初期化する。 5. hicolormap 用のメモリを確保し、初期行を WHITE に設定する。 6. メモリ確保に失敗した場合は REG_ESPACE エラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regcomp.c 内の initcm（コンパイル開始時）。
  • **呼び出し先**: MALLOC, memset, CERR（マクロ）。

関連データ構造

  • colormap: カラーマップの管理構造体。
  • colordesc: 色の属性を保持する記述子。
  • vars: コンパイル全体のコンテキスト。

失敗モードと境界条件

  • MALLOC が NULL を返した場合、CERR(REG_ESPACE) を呼び出し、部分的に初期化された状態で終了する。
  • hicolormap の初期化失敗時は、cmranges を NULL に設定することで後続の freecm での二重解放や不正アクセスを防ぐ。

保守担当者向けメモ

  • locolormap の初期化は WHITE が 0 であるという前提に強く依存しているため、定数定義を変更する際は注意が必要。
  • maxarrayrows や hiarrayrows の初期値は任意に設定されており、必要に応じて動的に拡張される設計となっている。
