dummy_object_relabel

定義

  • 種別: function
  • 定義箇所: src/test/modules/dummy_seclabel/dummy_seclabel.c:24

目的

PostgreSQLのセキュリティラベルプロバイダー「dummy」において、指定されたセキュリティラベルの妥当性を検証し、権限に応じて設定の可否を決定するコールバック関数です。

シグネチャまたは形状

static void dummy_object_relabel(const ObjectAddress *object, const char *seclabel)

入力と出力

  • **入力**:
  • object: 対象となるデータベースオブジェクトのアドレス（本関数内では未使用）。
  • seclabel: 設定しようとするセキュリティラベル文字列。
  • **出力**: なし（戻り値は void）。
  • **副作用**: ラベルが不正な場合や権限不足の場合は ereport(ERROR, ...) を発行して処理を中断します。

前提条件と不変条件

  • seclabel が NULL の場合は、ラベルの削除（またはリセット）とみなされ、無条件で許可されます。
  • secret または top secret ラベルを設定するには、呼び出し元がスーパーユーザーである必要があります。

アルゴリズムまたは振る舞い

1. **許可リストの判定**: seclabel が NULL、unclassified、または classified のいずれかであれば、検証をパスして正常終了します。 2. **権限チェック**: seclabel が secret または top secret の場合、superuser() を呼び出して権限を確認します。権限がない場合は ERRCODE_INSUFFICIENT_PRIVILEGE でエラーを発生させます。 3. **バリデーションエラー**: 上記のいずれにも該当しないラベル文字列が渡された場合、ERRCODE_INVALID_NAME でエラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのコアシステム（register_label_provider を通じて登録されたコールバックとして、SECURITY LABEL コマンド実行時に呼び出されます）。
  • **呼び出し先**:
  • superuser(): 権限確認に使用。
  • ereport(): エラー通知に使用。
  • strcmp(): 文字列比較に使用。

関連データ構造

  • ObjectAddress: commands/seclabel.h で定義される、データベースオブジェクトを識別するための構造体。

失敗モードと境界条件

  • **ERRCODE_INSUFFICIENT_PRIVILEGE**: secret または top secret を非スーパーユーザーが設定しようとした場合に発生。
  • **ERRCODE_INVALID_NAME**: 定義されていないラベル文字列（例: "public" など）が指定された場合に発生。

保守担当者向けメモ

  • 本関数はテスト用スタブであり、実運用環境のセキュリティ要件を満たすものではありません。
  • 新しいラベルを追加する場合は、strcmp の条件分岐を拡張してください。
  • object 引数は将来的な拡張のために存在しますが、現在の実装では無視されています。
