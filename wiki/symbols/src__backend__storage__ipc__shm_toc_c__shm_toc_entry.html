shm_toc_entry

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/ipc/shm_toc.c:20

目的

共有メモリセグメント内の特定のデータ構造を識別し、その位置を特定するための目次（Table of Contents）の各エントリを保持する。

シグネチャまたは形状

typedef struct shm_toc_entry
{
    uint64      key;        /* 識別用キー */
    Size        offset;     /* TOC開始位置からのバイトオフセット */
} shm_toc_entry;

入力と出力

  • 本構造体はデータ保持用であり、関数ではないため入力・出力の概念は適用されない。
  • メンバ key は一意の識別子として使用される。
  • メンバ offset は、共有メモリセグメント内での相対的な位置を示す。

前提条件と不変条件

  • offset は絶対アドレスではなく、TOCの開始位置からの相対値である必要がある（プロセスごとに共有メモリの仮想アドレスが異なるため）。
  • shm_toc 構造体内の toc_entry 配列の一部として管理される。

アルゴリズムまたは振る舞い

  • shm_toc_insert 実行時に、キーと計算された相対オフセットが本構造体に格納される。
  • shm_toc_lookup 実行時に、キーの一致を確認するために走査される。
  • 検索時はロックフリーで動作するため、書き込み後のメモリバリア（pg_write_barrier）と読み込み時のバリア（pg_read_barrier）により整合性が保証される。

呼び出し元と呼び出し先

  • 呼び出し元/呼び出し先の直接的な関係は不明。
  • 参照箇所:
  • src/backend/storage/ipc/shm_toc.c 内の shm_toc 構造体定義（toc_entry 配列）。
  • shm_toc_create, shm_toc_allocate, shm_toc_insert, shm_toc_estimate 等の関数内で、サイズ計算やエントリ操作のために参照されている。

関連データ構造

  • shm_toc: 本構造体を配列として保持する親構造体。

失敗モードと境界条件

  • 共有メモリセグメントの容量を超えてエントリを追加しようとした場合、shm_toc_allocate 等の管理関数側で境界チェックが行われる。
  • offset の計算が誤っている場合、無効なメモリ領域を参照する可能性がある。

保守担当者向けメモ

  • 本構造体は多数のキーを扱うようには設計されていない。多数のポインタが必要な場合は、TOC には単一のデータ構造へのポインタのみを登録し、その構造体内部で管理すること。
  • 共有メモリ上の相対アドレスを使用しているため、ポインタの直接代入は避けること。
