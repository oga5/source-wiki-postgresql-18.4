StatEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsvector_op.c:45

目的

全文検索における語彙統計（ts_stat）を保持するための二分探索木ノードです。文書集合内での語彙の出現頻度や文書数を集計するために使用されます。

シグネチャまたは形状

typedef struct StatEntry
{
    uint32      ndoc;           /* 文書出現数（0は走査済みを示す） */
    uint32      nentry;         /* 総出現回数 */
    struct StatEntry *left;     /* 左部分木 */
    struct StatEntry *right;    /* 右部分木 */
    uint32      lenlexeme;      /* 語彙の長さ */
    char        lexeme[FLEXIBLE_ARRAY_MEMBER]; /* 語彙文字列 */
} StatEntry;

入力と出力

  • 本構造体は ts_stat 関数による統計集計プロセスにおいて、語彙をキーとして動的に生成・更新されます。
  • lexeme フィールドは可変長配列として定義されており、メモリ確保時には STATENTRYHDRSZ マクロがヘッダーサイズの計算に使用されます。

前提条件と不変条件

  • ndoc が 0 の場合、そのノードは既に統計走査の過程で処理済みであることを示します。
  • lexeme は FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体インスタンスは palloc 等を用いて適切なサイズで動的に確保する必要があります。

アルゴリズムまたは振る舞い

  • ts_stat 関数がテーブルをスキャンする際、ts_accum を通じてこの構造体のツリーを構築します。
  • 語彙の検索および挿入は、二分探索木のロジックに従って行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsvector_op.c 内の ts_stat 関連ロジック（ts_accum 等）から参照・操作されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • TSVectorStat: 統計情報のルートを保持する構造体。root（StatEntry へのポインタ）および stack（走査用スタック）を管理します。

失敗モードと境界条件

  • メモリ不足: palloc によるメモリ確保失敗時にエラーが発生します。
  • スタックオーバーフロー: 非常に深いツリー構造を走査する場合、再帰的な処理によってスタック制限に達する可能性があります。

保守担当者向けメモ

  • lexeme は可変長メンバーであるため、sizeof(StatEntry) をそのまま使用せず、必ず STATENTRYHDRSZ を用いてオフセット計算を行ってください。
  • ndoc の値が 0 になる条件は、統計集計のアルゴリズムに依存するため、ロジック変更時は ts_accum の実装を確認してください。
