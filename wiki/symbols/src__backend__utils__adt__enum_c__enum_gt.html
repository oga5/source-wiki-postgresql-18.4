enum_gt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/enum.c:350

目的

2つの列挙型（ENUM）値の「より大きい（greater than）」比較を行い、真偽値を返します。

シグネチャまたは形状

Datum enum_gt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの列挙型値（Oid）を受け取ります。
  • **出力**: 1番目の値が2番目の値より大きい場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力されたOIDは、システムカタログ pg_enum に存在する有効な列挙値である必要があります。
  • 比較ロジックは enum_cmp_internal に依存しており、列挙型の順序付けルールに従います。

アルゴリズムまたは振る舞い

1. PG_GETARG_OID を使用して、引数から2つの列挙型OID（a および b）を取得します。 2. enum_cmp_internal(a, b, fcinfo) を呼び出し、比較結果を取得します。 3. 比較結果が 0 より大きい（> 0）場合、true を返します。それ以外の場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_enum.c において、B-treeインデックスの比較操作として利用されています。
  • **呼び出し先**:
  • enum_cmp_internal（比較の共通エンジン）。

関連データ構造

  • Oid: 列挙型の内部表現として使用されるオブジェクト識別子。
  • pg_enum: 列挙型の定義と順序を保持するシステムカタログ。

失敗モードと境界条件

  • 無効なOIDが渡された場合、enum_cmp_internal 内部でエラーが発生する可能性があります。
  • 列挙型の順序が変更された場合、キャッシュの整合性維持のため TypeCache を介した比較が行われます。

保守担当者向けメモ

  • 本関数はSQLレベルの > 演算子に対応しています。
  • 比較ロジックの変更が必要な場合は、enum_cmp_internal を修正してください。
  • インデックスの整合性を保つため、未コミットの列挙値に対する安全チェック（check_safe_enum_use）が呼び出し元または比較エンジン側で適切に機能していることを前提としています。
