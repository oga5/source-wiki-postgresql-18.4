prune_element_hashtable

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_typanalyze.c:680

目的

Lossy Counting アルゴリズムに基づき、配列要素の頻度を追跡しているハッシュテーブルから、重要度の低い（頻度が閾値以下の）エントリを削除し、メモリ使用量を抑制する。

シグネチャまたは形状

static void prune_element_hashtable(HTAB *elements_tab, int b_current)

入力と出力

  • elements_tab (HTAB*): 追跡対象の要素を格納するハッシュテーブル。
  • b_current (int): 現在のバケット番号（Lossy Counting の閾値判定に使用）。
  • 戻り値: なし。

前提条件と不変条件

  • elements_tab は hash_seq_init および hash_search が可能な有効なハッシュテーブルであること。
  • グローバル変数 array_extra_data が初期化されており、typbyval プロパティが正しく設定されていること。

アルゴリズムまたは振る舞い

1. hash_seq_init を用いてハッシュテーブル全体を走査する。 2. 各 TrackItem について、frequency + delta <= b_current を満たすか判定する。 3. 条件を満たす（頻度が低い）場合、hash_search (HASH_REMOVE) を用いてエントリを削除する。 4. 削除対象の要素が typbyval でない（ポインタ渡しである）場合、pfree を呼び出してメモリを解放する。 5. 削除時にハッシュテーブルの整合性が失われた場合は elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/array_typanalyze.c 内の compute_array_stats 関数から呼び出される。
  • 呼び出し先: hash_seq_init, hash_seq_search, hash_search, pfree, elog。

関連データ構造

  • TrackItem: ハッシュテーブルに格納される要素の頻度とデルタ値を保持する構造体。
  • HTAB: PostgreSQL のハッシュテーブル構造体。

失敗モードと境界条件

  • ハッシュテーブルの走査中にエントリが見つからない場合、hash_search が NULL を返し、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は Lossy Counting アルゴリズムの剪定フェーズを担う。メモリ効率を維持するため、頻繁に呼び出される可能性がある。
  • typbyval が偽の場合のメモリ解放処理が含まれているため、ArrayAnalyzeExtraData の整合性が重要である。
