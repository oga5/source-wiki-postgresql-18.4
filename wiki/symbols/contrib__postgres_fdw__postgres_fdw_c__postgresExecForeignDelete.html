postgresExecForeignDelete

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/postgres_fdw.c:2124

目的

外部テーブルに対する DELETE 操作を実行し、削除された行のタプルをローカルの実行エンジンへ返却する。

シグネチャまたは形状

static TupleTableSlot *postgresExecForeignDelete(EState *estate,
                                                 ResultRelInfo *resultRelInfo,
                                                 TupleTableSlot *slot,
                                                 TupleTableSlot *planSlot)

入力と出力

  • **入力**:
  • estate: 実行状態（EState）。
  • resultRelInfo: ターゲットとなる外部テーブルの情報。
  • slot: 削除対象のタプルを保持するスロット。
  • planSlot: プランノードからのタプルを保持するスロット。
  • **出力**:
  • 削除された行を含む TupleTableSlot。削除が行われなかった場合は NULL を返す。

前提条件と不変条件

  • resultRelInfo は外部テーブルとして適切に初期化されている必要がある。
  • 内部で呼び出される execute_foreign_modify がリモートサーバーとの通信および削除処理を完遂できる状態であること。

アルゴリズムまたは振る舞い

1. execute_foreign_modify を呼び出し、CMD_DELETE コマンドを指定してリモートサーバーでの削除処理を実行する。 2. 実行結果として得られたスロット配列の先頭要素を保持する。 3. 処理結果が有効であればそのスロットを返し、無効であれば NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のエグゼキュータ（postgres_fdw_handler を介して ExecForeignDelete コールバックとして登録）。
  • **呼び出し先**: execute_foreign_modify（リモート操作の共通実行関数）。

関連データ構造

  • ResultRelInfo: ターゲットリレーションの情報を保持。
  • TupleTableSlot: タプルデータを保持する標準的な PostgreSQL 構造体。
  • PgFdwModifyState: 外部テーブル変更時の状態管理に使用される（execute_foreign_modify 内部で参照）。

失敗モードと境界条件

  • リモートサーバーへの接続が確立できない場合や、リモート側で削除が失敗した場合はエラーが発生する。
  • execute_foreign_modify が NULL を返した場合、本関数も NULL を返す。

保守担当者向けメモ

  • 本関数は postgres_fdw の ExecForeignDelete コールバックの実装である。
  • 削除処理のロジックは execute_foreign_modify に集約されているため、削除の挙動を変更する場合は同関数を確認すること。
