pg_rotate_logfile

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/signalfuncs.c:307

目的

Postmasterに対してログファイルのローテーションを要求するSQLインターフェースを提供します。

シグネチャまたは形状

Datum pg_rotate_logfile(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数として呼び出される際の引数群）。
  • **出力**: Datum 型として bool 値を返します。
  • 成功時: true
  • 失敗時: false

前提条件と不変条件

  • グローバル変数 Logging_collector が有効（true）である必要があります。無効な場合、ローテーションは実行されません。

アルゴリズムまたは振る舞い

1. Logging_collector の状態を確認します。 2. Logging_collector が無効な場合、WARNING レベルのログを出力し、false を返して終了します。 3. Logging_collector が有効な場合、SendPostmasterSignal(PMSIGNAL_ROTATE_LOGFILE) を呼び出し、Postmasterに対してローテーションのシグナルを送信します。 4. 成功時に true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます。
  • **呼び出し先**:
  • SendPostmasterSignal（src/backend/storage/ipc/pmsignal.c）
  • ereport（エラー報告用マクロ）

関連データ構造

  • PMSIGNAL_ROTATE_LOGFILE: Postmasterへのシグナル種別を示す定数。

失敗モードと境界条件

  • **ログ収集機能の無効化**: Logging_collector がオフの場合、ローテーションは実行されず false が返されます。この際、クライアントには WARNING が通知されます。

保守担当者向けメモ

  • 本関数は単なるシグナル送信の仲介役です。実際のログローテーション処理は、Postmasterがシグナルを受け取った後に syslogger プロセスによって実行されます。
