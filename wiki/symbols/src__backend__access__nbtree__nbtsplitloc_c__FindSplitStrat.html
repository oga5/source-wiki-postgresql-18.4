FindSplitStrat

定義

  • 種別: enum
  • 定義箇所: src/backend/access/nbtree/nbtsplitloc.c:20

目的

B-tree インデックスのページ分割時、インデックスのデータ特性（重複の多さや値の分布）に応じて、最適な分割点を選択するための戦略を定義する。

シグネチャまたは形状

typedef enum {
    SPLIT_DEFAULT,
    SPLIT_MANY_DUPLICATES,
    SPLIT_SINGLE_VALUE,
} FindSplitStrat;

入力と出力

  • 本シンボルは列挙型であり、関数への引数として渡されることで分割ロジックの挙動を切り替える。
  • _bt_bestsplitloc 等の関数において、FindSplitStrat 型の引数として受け取られる。

前提条件と不変条件

  • _bt_strategy 関数によって、インデックスの状況（重複の有無や右端ページか否か）に基づき、適切な戦略が選択されることが期待される。

アルゴリズムまたは振る舞い

  • SPLIT_DEFAULT: サフィックス切り詰め（Suffix Truncation）の効果を考慮し、バランスの良い分割を行う。
  • SPLIT_MANY_DUPLICATES: 重複が多いデータに対し、最小限の識別可能な分割点を探す。
  • SPLIT_SINGLE_VALUE: 単一の値がページを占有する場合など、左ページをほぼ満杯にする戦略をとる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/nbtree/nbtsplitloc.c 内の _bt_bestsplitloc や分割計算ロジック。
  • 呼び出し先: 本シンボルは列挙型であるため、直接的な関数呼び出しは行わない。

関連データ構造

  • SplitPoint: 分割候補の詳細を保持する構造体。
  • FindSplitData: 分割計算中のコンテキストを保持する構造体。

失敗モードと境界条件

  • 適切な戦略が選択されない場合、空間効率の低下や、サフィックス切り詰めによる最適化が不十分になる可能性がある。

保守担当者向けメモ

  • 本列挙型は _bt_strategy による分析結果を反映する。新しい分割戦略を追加する場合は、_bt_strategy のロジックと、それを利用する _bt_bestsplitloc の評価関数を更新する必要がある。
