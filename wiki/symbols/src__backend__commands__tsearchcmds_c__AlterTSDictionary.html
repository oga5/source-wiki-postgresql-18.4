AlterTSDictionary

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tsearchcmds.c:492

目的

SQL の ALTER TEXT SEARCH DICTIONARY コマンドを実行し、既存の全文検索辞書の初期化オプションを更新する。

シグネチャまたは形状

ObjectAddress AlterTSDictionary(AlterTSDictionaryStmt *stmt)

入力と出力

  • **入力**: AlterTSDictionaryStmt *stmt（辞書名および変更対象のオプションリストを含む構文解析木）。
  • **出力**: ObjectAddress（更新された辞書オブジェクトの識別子）。

前提条件と不変条件

  • 対象の辞書が存在すること。
  • 呼び出し元が対象辞書の所有者であること。
  • 指定されたオプションが、辞書テンプレートの検証関数（verify_dictoptions）によって妥当と判断されること。

アルゴリズムまたは振る舞い

1. get_ts_dict_oid を使用して辞書の OID を取得し、pg_ts_dict システムカタログをロックする。 2. SearchSysCache1 で辞書のタプルを取得し、所有権を確認する。 3. deserialize_deflist を用いて、現在の辞書オプションをリスト形式に変換する。 4. stmt->options を走査し、既存のオプションを削除または更新する。 5. verify_dictoptions を呼び出し、更新後のオプションセットを検証する。 6. serialize_deflist でオプションをシリアライズし、CatalogTupleUpdate を用いてシステムカタログを更新する。 7. InvokeObjectPostAlterHook を呼び出し、変更を通知する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（SQL 解析後の実行フェーズ）。
  • **呼び出し先**: get_ts_dict_oid, table_open, SearchSysCache1, object_ownercheck, deserialize_deflist, verify_dictoptions, serialize_deflist, heap_modify_tuple, CatalogTupleUpdate, InvokeObjectPostAlterHook 等。

関連データ構造

  • AlterTSDictionaryStmt: SQL 文の解析結果を保持する構造体。
  • pg_ts_dict: 全文検索辞書を定義するシステムカタログ。
  • ObjectAddress: オブジェクトの識別情報。

失敗モードと境界条件

  • 辞書名が見つからない場合、またはキャッシュルックアップに失敗した場合はエラーとなる。
  • 所有者でない場合、aclcheck_error が発生する。
  • オプションの検証に失敗した場合、エラーが発生する。

保守担当者向けメモ

  • 本関数は辞書の「オプション」のみを変更する。テンプレート自体の変更はサポートしておらず、依存関係の更新も不要である。将来的にオプションがデータベース内の他オブジェクトを参照するようになった場合は、依存関係の更新ロジックが必要となる可能性がある。
