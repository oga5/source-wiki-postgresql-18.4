StartupRereadConfig

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/startup.c:124

目的

起動プロセスにおいて、設定ファイル（postgresql.conf等）の再読み込みを行い、レプリケーション関連の設定変更を検知してWALレシーバーの再起動を要求する。

シグネチャまたは形状

static void StartupRereadConfig(void)

入力と出力

  • 入力: なし（グローバル変数 PrimaryConnInfo, PrimarySlotName, wal_receiver_create_temp_slot を参照）
  • 出力: なし（副作用として設定の再読み込みおよびWALレシーバーの再起動要求を実行）

前提条件と不変条件

  • ProcessConfigFile(PGC_SIGHUP) を呼び出し、設定ファイルを再読み込みする。
  • PrimaryConnInfo または PrimarySlotName が変更された場合、あるいはスロットが未設定の状態で wal_receiver_create_temp_slot が変更された場合にのみ、再起動要求が発生する。

アルゴリズムまたは振る舞い

1. 現在の PrimaryConnInfo と PrimarySlotName をメモリにコピーして保持する。 2. ProcessConfigFile(PGC_SIGHUP) を実行し、設定を更新する。 3. 更新前後の値を比較し、以下のいずれかが変更されたか判定する。

  • PrimaryConnInfo
  • PrimarySlotName
  • スロット名が空の場合の wal_receiver_create_temp_slot

    • 4. 変更が検知された場合、StartupRequestWalReceiverRestart() を呼び出す。 5. 保持していたコピーを解放する。

呼び出し元と呼び出し先

  • 呼び出し元: ソースコード内の記述から、シグナルハンドラ等のコンテキストで呼び出されることが示唆されるが、具体的な呼び出し箇所は提供された抜粋内では特定できない。
  • 呼び出し先:
  • ProcessConfigFile
  • StartupRequestWalReceiverRestart

関連データ構造

  • PrimaryConnInfo (グローバル変数)
  • PrimarySlotName (グローバル変数)
  • wal_receiver_create_temp_slot (グローバル変数)

失敗モードと境界条件

  • PrimarySlotName が空文字列の場合、wal_receiver_create_temp_slot の変更のみが再起動のトリガーとして考慮される。
  • メモリ割り当て（pstrdup）が失敗した場合の挙動は、PostgreSQLの標準的なエラーハンドリングに従う。

保守担当者向けメモ

  • 本関数は SIGHUP シグナル受信時の設定反映ロジックの一部である。
  • wal_receiver_create_temp_slot の変更追跡は、レプリケーションスロットが設定されていない場合に限定されている点に注意すること。
