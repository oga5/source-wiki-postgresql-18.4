xid_age

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xid.c:116

目的

指定されたトランザクションID（XID）の経過時間（age）を、現在の安定した最新トランザクションIDを基準に計算します。

シグネチャまたは形状

Datum xid_age(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_TRANSACTIONID(0) を介して TransactionId を受け取ります。
  • **出力**: int32 型の経過時間を返します。

前提条件と不変条件

  • TransactionIdIsNormal が偽（永続的なXIDなど）の場合、常に INT_MAX を返します。
  • 正常なXIDの場合、現在のトランザクションIDとの差分を計算します。

アルゴリズムまたは振る舞い

1. PG_GETARG_TRANSACTIONID で引数を取得します。 2. GetStableLatestTransactionId() を呼び出し、基準となる最新のトランザクションIDを取得します。 3. TransactionIdIsNormal を用いて、対象のXIDが通常のトランザクションIDか判定します。

  • 通常でない場合は INT_MAX を返して終了します。

    • 4. 通常の場合は、now - xid の差分を計算し、int32 にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • PG_GETARG_TRANSACTIONID
  • GetStableLatestTransactionId
  • TransactionIdIsNormal
  • PG_RETURN_INT32
  • **呼び出し元**: ソースコード内での明示的な呼び出し元情報は提供されていません。

関連データ構造

  • TransactionId: トランザクションを識別する基本型。
  • Datum: PostgreSQLの汎用データ表現。

失敗モードと境界条件

  • 永続的なXID（!TransactionIdIsNormal(xid)）が渡された場合、計算を行わず INT_MAX を返します。

保守担当者向けメモ

  • 本関数は xid の経過時間をSQLから取得するために使用されます。
  • 循環する可能性がある TransactionId の性質上、計算結果は現在のトランザクション状態に依存します。
