LockRelId

定義

  • 種別: struct
  • 定義箇所: src/include/utils/rel.h:38

目的

PostgreSQL のロックマネージャにおいて、特定のデータベース内のリレーションを一意に識別するための識別子を保持する。

シグネチャまたは形状

typedef struct LockRelId
{
    Oid    relId;    /* リレーション識別子 */
    Oid    dbId;     /* データベース識別子 */
} LockRelId;

入力と出力

  • 入力: relId および dbId（いずれも Oid 型）を保持する構造体。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • relId は対象となるリレーション（テーブルやインデックス等）の OID を指す。
  • dbId は対象リレーションが属するデータベースの OID を指す。

アルゴリズムまたは振る舞い

本構造体は、ロックマネージャがリレーションに対するロック状態を管理・検索する際のキーとして機能する。リレーション記述子（RelationData）内の rd_lockInfo フィールドの一部として保持される。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/access/common/relation.c
  • src/backend/access/index/indexam.c
  • src/backend/access/nbtree/nbtutils.c
  • src/backend/catalog/index.c
  • src/backend/commands/dbcommands.c
  • src/backend/commands/indexcmds.c
  • 呼び出し先: なし（データ構造定義のため）。

関連データ構造

  • RelationData: リレーション記述子。本構造体を rd_lockInfo フィールド内に含む。
  • LockInfoData: ロックマネージャが使用するリレーション識別情報。

失敗モードと境界条件

  • Oid が無効な値（InvalidOid）である場合、ロックマネージャによる識別が正しく行われない可能性がある。

保守担当者向けメモ

  • 本構造体はロックの識別に使用されるため、リレーションの再定義やデータベースの切り替え時に整合性が保たれるよう注意すること。
  • src/backend/commands/indexcmds.c 等で palloc_object(LockRelId) を用いた動的確保が行われている箇所がある。
