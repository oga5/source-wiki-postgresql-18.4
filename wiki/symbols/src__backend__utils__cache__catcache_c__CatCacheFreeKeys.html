CatCacheFreeKeys

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/catcache.c:2291

目的

キャッシュキーとして保持されている Datum 配列のうち、メモリ上に動的に確保された領域（attbyval が偽であるデータ）を解放する。

シグネチャまたは形状

static void CatCacheFreeKeys(TupleDesc tupdesc, int nkeys, int *attnos, Datum *keys)

入力と出力

  • tupdesc: キャッシュ対象のタプル記述子。属性の型情報（attbyval）の参照に使用。
  • nkeys: キーの数。
  • attnos: 各キーに対応する属性番号の配列。
  • keys: 解放対象の Datum 値の配列。
  • 戻り値: なし。

前提条件と不変条件

  • attnos に含まれる属性番号は 0 より大きいこと（システム属性はキャッシュ非対応のため）。
  • keys 配列内の各 Datum は、attbyval が偽である場合に限り、pfree で解放可能なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. nkeys 回のループ処理を行い、各キーを順次処理する。 2. attnos[i] から属性番号を取得し、TupleDescAttr を用いて対応する属性定義（Form_pg_attribute）を参照する。 3. 属性の attbyval フラグを確認する。

  • attbyval が偽（false）の場合、keys[i] をポインタとして解釈し、pfree を呼び出してメモリを解放する。
  • attbyval が真（true）の場合、値自体が Datum に格納されているため、解放処理は行わない。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/cache/catcache.c 内の複数箇所（キャッシュエントリの破棄や検索失敗時のクリーンアップ処理など）。
  • 呼び出し先:
  • pfree (メモリ解放)
  • TupleDescAttr (属性定義の取得)

関連データ構造

  • TupleDesc: タプルの構造定義。
  • Form_pg_attribute: 属性のメタデータ（attbyval を保持）。
  • Datum: キャッシュキーとして保持される値。

失敗モードと境界条件

  • attnum <= 0 の場合、Assert により異常終了する（システム属性はキャッシュ不可）。
  • attbyval が偽であるにもかかわらず keys[i] が無効なポインタである場合、pfree 呼び出し時にメモリ破壊やクラッシュが発生する可能性がある。

保守担当者向けメモ

  • 本関数は catcache.c 内のメモリ管理の基盤であり、キャッシュエントリのライフサイクル管理において重要な役割を果たす。
  • attbyval の判定ロジックは pg_attribute の定義に依存しているため、カタログ定義の変更時には注意が必要である。
