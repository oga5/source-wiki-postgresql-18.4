pg_restore_relation_stats

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/relation_stats.c:225

目的

ユーザーが指定した値を用いて、テーブル（リレーション）の統計情報を手動で上書き・復元するためのSQLインターフェース関数です。

シグネチャまたは形状

Datum pg_restore_relation_stats(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL経由で渡される引数群）
  • **出力**: Datum（内部的には bool 型。統計更新が成功した場合は true、失敗した場合は false を返す）

前提条件と不変条件

  • RecoveryInProgress() チェックにより、リカバリ中の実行は禁止されています。
  • 対象テーブルに対して ShareUpdateExclusiveLock を取得する必要があります。

アルゴリズムまたは振る舞い

1. NUM_RELATION_STATS_ARGS に基づき、関数呼び出し情報を初期化します。 2. stats_fill_fcinfo_from_arg_pairs を呼び出し、入力引数を内部構造体へ変換します。 3. relation_statistics_update を呼び出し、pg_class の統計カラム（relpages, reltuples 等）を更新します。 4. 処理の成否を bool 値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • SQLインターフェース経由での呼び出し。
  • src/bin/pg_dump/pg_dump.c において、統計情報のダンプ出力時に参照されています。
  • **呼び出し先**:
  • stats_fill_fcinfo_from_arg_pairs（引数解析）
  • relation_statistics_update（統計更新のコアロジック）

関連データ構造

  • relarginfo: 更新対象となる統計カラムとデータ型のマッピング定義。
  • pg_class: 本関数が更新対象とするシステムカタログ。

失敗モードと境界条件

  • 統計情報の更新処理（relation_statistics_update）が失敗した場合、関数は false を返します。
  • 同時実行時には ShareUpdateExclusiveLock によるデッドロックのリスクが存在します。

保守担当者向けメモ

  • 本関数は vac_update_relstats() と同様のロック戦略を採用しています。
  • 統計情報の整合性を維持するため、pg_class のシステムキャッシュおよびカタログ更新処理と密接に連携しています。
