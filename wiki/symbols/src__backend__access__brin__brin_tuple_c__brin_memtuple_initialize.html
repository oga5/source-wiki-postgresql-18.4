brin_memtuple_initialize

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_tuple.c:510

目的

既存の BrinMemTuple 構造体を初期化し、指定されたインデックス記述子 (BrinDesc) に基づいて各カラムのメタデータとメモリ領域を再設定する。

シグネチャまたは形状

BrinMemTuple *brin_memtuple_initialize(BrinMemTuple *dtuple, BrinDesc *brdesc);

入力と出力

  • **入力**:
  • dtuple: 初期化対象となる BrinMemTuple へのポインタ。
  • brdesc: インデックスの属性情報を含む BrinDesc 構造体。
  • **出力**:
  • 初期化済みの dtuple へのポインタを返す。

前提条件と不変条件

  • dtuple は有効なメモリ領域を指していること。
  • dtuple->bt_context が有効なメモリコンテキストとして初期化されていること。
  • brdesc が指すインデックス記述子が、対象のタプル構造と整合していること。

アルゴリズムまたは振る舞い

1. dtuple->bt_context に対して MemoryContextReset を実行し、既存のメモリ割り当てを解放する。 2. dtuple 構造体の直後に続くデータ領域の先頭アドレスを計算する。 3. インデックスの全属性 (natts) についてループ処理を行う:

  • bv_attno を 1 から始まる属性番号で設定する。
  • bv_allnulls を true、bv_hasnulls を false に初期化する。
  • bv_values に計算済みのデータ領域アドレスを割り当てる。
  • bv_mem_value を NULL に、bv_serialize を NULL に設定する。
  • bv_context に dtuple->bt_context を設定する。
  • 次の属性のためにデータ領域ポインタをオフセット分進める。

    • 4. dtuple->bt_empty_range を true に設定し、空の範囲であることを示す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c (複数箇所)
  • src/backend/access/brin/brin_tuple.c (内部および外部からの呼び出し)
  • **呼び出し先**:
  • MemoryContextReset (PostgreSQLメモリ管理API)

関連データ構造

  • BrinMemTuple: メモリ上で操作されるBRINタプル構造体。
  • BrinDesc: インデックスの属性定義を保持する構造体。
  • BrinValues: 各カラムの要約値を保持する構造体。

失敗モードと境界条件

  • dtuple が NULL の場合、メモリ参照エラーが発生する。
  • brdesc の属性数と dtuple が確保しているメモリ領域が不整合な場合、バッファオーバーランが発生する可能性がある。

保守担当者向けメモ

  • 本関数はメモリコンテキストをリセットするため、この関数を呼び出す前に dtuple 内の既存のデータへの参照が不要であることを確認すること。
  • bv_values の計算には MAXALIGN が使用されており、アライメント要件が厳格に守られている。変更時は注意が必要。
