ExecEndIndexOnlyScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexonlyscan.c:398

目的

Index-Only Scan実行ノードに関連するリソースを解放し、並列実行時の統計情報を集約する。

シグネチャまたは形状

void ExecEndIndexOnlyScan(IndexOnlyScanState *node)

入力と出力

  • **入力**: IndexOnlyScanState *node（実行中のインデックスのみスキャンの状態構造体）
  • **出力**: なし（副作用としてリソース解放と統計情報の更新を行う）

前提条件と不変条件

  • node は初期化済みであること。
  • 並列ワーカーとして実行されている場合、node->ioss_SharedInfo が有効であること。

アルゴリズムまたは振る舞い

1. **VMバッファの解放**: ioss_VMBuffer が InvalidBuffer でない場合、ReleaseBuffer を呼び出してピンを解放し、値を InvalidBuffer にリセットする。 2. **並列統計情報の集約**: 並列ワーカープロセスである場合、現在のノードで収集された nsearches を共有メモリ上の ioss_SharedInfo に加算する。 3. **インデックススキャンの終了**: indexScanDesc が存在する場合、index_endscan を呼び出してスキャンを終了する。 4. **リレーションのクローズ**: indexRelationDesc が存在する場合、index_close を呼び出してインデックスリレーションを閉じる（ロックは取得しない）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:644 にて呼び出される。
  • **呼び出し先**: ReleaseBuffer, IsParallelWorker, index_endscan, index_close。

関連データ構造

  • IndexOnlyScanState: スキャンノードの実行状態を保持する構造体。
  • IndexScanDesc: インデックススキャンの記述子。
  • IndexScanInstrumentation: 並列実行時の統計情報構造体。

失敗モードと境界条件

  • indexScanDesc や indexRelationDesc が NULL の場合、それぞれのクローズ処理はスキップされる（安全に処理される）。
  • ioss_VMBuffer が InvalidBuffer の場合、解放処理はスキップされる。

保守担当者向けメモ

  • 本関数は ExecEndIndexOnlyScan として nodeIndexonlyscan.c 内で定義されており、実行終了時のクリーンアップを一手に引き受ける。
  • 並列ワーカーの統計情報は memcpy ではなく加算（+=）で集約される点に注意すること。これは、Gatherノードがワーカーを再利用する際に統計情報がリセットされるためである。
