RestrictInfoIsTidQual

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/tidpath.c:234

目的

与えられた RestrictInfo が、TID（Tuple Identifier）スキャンに使用可能な条件（ctid 等価比較、ANY 比較、または CURRENT OF）であるかを判定します。

シグネチャまたは形状

static bool RestrictInfoIsTidQual(PlannerInfo *root, RestrictInfo *rinfo, RelOptInfo *rel)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rinfo: 評価対象の RestrictInfo 構造体。
  • rel: 対象となるリレーション情報。
  • **出力**:
  • bool: 条件がTIDスキャンに適していれば true、そうでなければ false を返します。

前提条件と不変条件

  • rinfo->pseudoconstant が true の場合、変数が含まれないためTID条件とは見なされず false を返します。
  • restriction_is_securely_promotable によるセキュリティチェックを通過する必要があります。

アルゴリズムまたは振る舞い

1. 擬似定数（pseudoconstant）であるかを確認し、該当する場合は即座に false を返します。 2. セキュリティ上の制約（restriction_is_securely_promotable）を確認し、不適格な場合は false を返します。 3. 以下のいずれかの条件に合致するかを順次判定します。

  • IsTidEqualClause: ctid = ... 形式の等価比較。
  • IsTidEqualAnyClause: ctid = ANY(...) 形式の比較。
  • IsCurrentOfClause: WHERE CURRENT OF 句。

    • 4. 上記のいずれかが true を返せば、この関数も true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/tidpath.c 内の TidQualFromRestrictInfoList 等から呼び出されます。
  • **呼び出し先**:
  • restriction_is_securely_promotable
  • IsTidEqualClause
  • IsTidEqualAnyClause
  • IsCurrentOfClause

関連データ構造

  • RestrictInfo: クエリの制約条件を保持する構造体。
  • PlannerInfo: クエリ全体のプランニング情報を保持する構造体。
  • RelOptInfo: リレーションの最適化情報を保持する構造体。

失敗モードと境界条件

  • rinfo が NULL の場合の挙動はソースコード上では明示されていません。
  • セキュリティ制約により、条件が有効であってもTIDスキャンに使用できないと判断される場合があります。

保守担当者向けメモ

  • 本関数はTIDスキャンパス生成におけるフィルタリングの要です。新しいTID関連の構文をサポートする場合は、判定ロジックの追加を検討してください。
