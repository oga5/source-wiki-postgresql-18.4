pgwin32_recv

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/socket.c:381

目的

Windows環境において、Winsockの WSARecv をラップし、PostgreSQLのシグナル処理およびブロッキング/非ブロッキングの挙動を統合したソケット受信関数を提供します。

シグネチャまたは形状

int pgwin32_recv(SOCKET s, char *buf, int len, int f)

入力と出力

  • **入力**:
  • s: 受信対象のソケットハンドル。
  • buf: 受信データを格納するバッファ。
  • len: バッファの長さ。
  • f: 受信フラグ。
  • **出力**:
  • 成功時: 受信したバイト数（int）。
  • 失敗時: -1 を返し、errno を設定。

前提条件と不変条件

  • 呼び出しの冒頭で pgwin32_poll_signals() が実行され、保留中のシグナルが優先的に処理されます。
  • pgwin32_noblock フラグが真の場合、関数は非ブロッキングモードとして振る舞います。
  • ブロッキングモード時は、pgwin32_waitforsinglesocket を用いてソケットの読み取り準備完了を待機します。

アルゴリズムまたは振る舞い

1. pgwin32_poll_signals() を呼び出し、シグナルがあれば即座に終了します。 2. WSARecv を実行し、成功すれば受信バイト数を返します。 3. WSAEWOULDBLOCK が発生した場合:

  • pgwin32_noblock が真なら errno = EWOULDBLOCK を設定して終了します。
  • ブロッキングモードの場合、pgwin32_waitforsinglesocket で最大5回まで再試行を行います。

    • 4. 再試行中に WSARecv が成功すればその値を返し、失敗し続けた場合は NOTICE を報告し EWOULDBLOCK を設定して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/win32_port.h にて recv マクロとして定義されており、PostgreSQLのネットワーク通信箇所から間接的に呼び出されます。
  • **呼び出し先**: pgwin32_poll_signals, WSARecv, WSAGetLastError, TranslateSocketError, pgwin32_waitforsinglesocket, pg_usleep, ereport。

関連データ構造

  • WSABUF: Winsockのデータバッファ構造体。
  • SOCKET: Windowsソケットハンドル。

失敗モードと境界条件

  • pgwin32_poll_signals がシグナルを検知した場合、-1 を返します。
  • WSARecv が WSAEWOULDBLOCK 以外のエラーを返した場合、TranslateSocketError を経由して errno を設定し -1 を返します。
  • ブロッキングモードで5回再試行してもデータが読み取れない場合、NOTICE をログ出力し、errno = EWOULDBLOCK として終了します。

保守担当者向けメモ

  • 本関数は recv の代替として機能します。Windows特有の挙動（WSAEWOULDBLOCK の誤検知など）に対する再試行ロジックが含まれているため、修正時は pgwin32_waitforsinglesocket との整合性に注意してください。
