pg_current_xact_id_if_assigned

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xid8funcs.c:351

目的

現在のトランザクションID（FullTransactionId）を、もし既に割り当てられている場合にのみ取得する。

シグネチャまたは形状

Datum pg_current_xact_id_if_assigned(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出し規約に基づく引数）
  • **出力**:
  • 成功時: 現在のトップレベルトランザクションID（FullTransactionId）
  • 失敗時（トランザクションIDが未割り当ての場合）: NULL

前提条件と不変条件

  • 本関数はトップレベルのトランザクションIDのみを対象とする。サブトランザクションIDは考慮されない。
  • FullTransactionIdIsValid() マクロにより、取得したIDの有効性が検証される。

アルゴリズムまたは振る舞い

1. GetTopFullTransactionIdIfAny() を呼び出し、現在のトップレベルトランザクションIDを取得する。 2. 取得したIDが有効でない場合、PG_RETURN_NULL() を実行して終了する。 3. IDが有効な場合、PG_RETURN_FULLTRANSACTIONID(topfxid) を実行してその値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（直接的なC言語コード内での呼び出し関係はソース excerpt に記載なし）。
  • **呼び出し先**:
  • GetTopFullTransactionIdIfAny()
  • FullTransactionIdIsValid()
  • PG_RETURN_NULL()
  • PG_RETURN_FULLTRANSACTIONID()

関連データ構造

  • FullTransactionId: 64ビットのトランザクションID。

失敗モードと境界条件

  • トランザクションが開始されていない、またはトランザクションIDがまだ割り当てられていない場合、本関数はエラーではなく NULL を返す。

保守担当者向けメモ

  • 本関数は、トランザクションIDの割り当て状況に応じて動的に結果を返すため、呼び出し側で NULL チェックを行う必要がある。
  • txid_current_if_assigned 等の古い関数からの移行先として利用される。
