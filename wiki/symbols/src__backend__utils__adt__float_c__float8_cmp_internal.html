float8_cmp_internal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/float.c:909

目的

2つの float8 値を比較し、標準的な比較結果（-1, 0, 1）を返します。

シグネチャまたは形状

int float8_cmp_internal(float8 a, float8 b)

入力と出力

  • **入力**: 比較対象となる2つの float8 型の値 a および b。
  • **出力**:
  • a > b の場合: 1
  • a < b の場合: -1
  • a == b の場合: 0

前提条件と不変条件

  • float8_gt および float8_lt の比較ロジックに依存します。
  • ソース内のコメントによれば、NaN は非 NaN 値よりも後にソートされるという不変条件が示唆されています。

アルゴリズムまたは振る舞い

1. float8_gt(a, b) を呼び出し、真であれば 1 を返します。 2. float8_lt(a, b) を呼び出し、真であれば -1 を返します。 3. 上記のいずれでもない場合（等価である場合）、0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/float.c 内の各種比較関数（PG_RETURN_INT32 を介してSQLインターフェースへ提供）。
  • src/backend/access/gist/gistproc.c（GiSTインデックス処理）。
  • src/backend/access/gist/gistscan.c（スキャン処理）。
  • contrib/btree_gist/btree_float8.c（btree_gist拡張）。
  • **呼び出し先**: float8_gt および float8_lt（src/include/utils/float.h で定義されたインライン関数と推測されます）。

関連データ構造

  • float8: PostgreSQLの8バイト浮動小数点型。

失敗モードと境界条件

  • NaN（非数）が含まれる場合の挙動は、float8_gt および float8_lt の実装に依存します。ソース内のコメントでは、NaN が非 NaN 値よりも後にソートされることが言及されています。

保守担当者向けメモ

  • 本関数はSQLレベルの比較演算子のバックエンド実装として頻繁に利用されます。
  • 比較ロジックを変更する場合は、float8_gt / float8_lt の定義および NaN のソート順序に関する仕様との整合性を確認してください。
