regexp_matches_ctx

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/regexp.c:52

目的

正規表現のマッチング結果（位置情報やサブパターン情報）および、SQL関数（テーブル関数や配列構築）へ結果を返すためのワークスペースを保持するコンテキスト構造体です。

シグネチャまたは形状

typedef struct regexp_matches_ctx
{
    text       *orig_str;       /* 元のTEXTデータ */
    int         nmatches;       /* マッチした箇所の数 */
    int         npatterns;      /* キャプチャ対象のサブパターン数 */
    int        *match_locs;     /* マッチ位置（0ベースの文字インデックス） */
    int         next_match;     /* 次に処理するマッチのインデックス */
    Datum      *elems;          /* 結果構築用ワークスペース */
    bool       *nulls;          /* 結果構築用ワークスペース */
    pg_wchar   *wide_str;       /* 元文字列のワイド文字版 */
    char       *conv_buf;       /* 必要に応じた変換バッファ */
    int         conv_bufsiz;    /* バッファサイズ */
} regexp_matches_ctx;

入力と出力

  • 本構造体自体は、正規表現エンジンによるマッチング処理の実行結果を格納し、後続の build_regexp_match_result 等の関数へ渡すためのデータコンテナとして機能します。

前提条件と不変条件

  • match_locs 配列のサイズは nmatches * npatterns * 2 である必要があります。
  • elems および nulls 配列は npatterns 個の要素を持つ必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: setup_regexp_matches 等により生成され、palloc0 でメモリが確保されます。 2. **マッチング**: 正規表現エンジンによるマッチング結果が match_locs に記録されます。 3. **結果構築**: next_match を進めながら、build_regexp_match_result や build_regexp_split_result がこのコンテキストを参照してSQL型の結果を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: setup_regexp_matches (生成元)、および正規表現関連のSQL関数実装（regexp_match 等）。
  • **呼び出し先**: build_regexp_match_result, build_regexp_split_result が本構造体を引数として受け取ります。

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • pg_wchar: ワイド文字表現。
  • Datum: PostgreSQLの汎用データ表現。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • nmatches や npatterns が0の場合、match_locs へのアクセスで境界条件に注意が必要です。

保守担当者向けメモ

  • 本構造体は、正規表現の実行結果を一時的に保持するライフサイクルを持ちます。SRF（Set Returning Function）として実装される関数では、funcctx->user_fctx に格納されて呼び出し間で維持されます。
  • match_locs のインデックス計算は nmatches * npatterns * 2 に依存しているため、構造体の拡張や変更時にはこの計算ロジックとの整合性に注意してください。
