check_foreign_key

定義

  • 種別: function
  • 定義箇所: contrib/spi/refint.c:249

目的

トリガーベースの参照整合性制約（RESTRICT, CASCADE, SET NULL）を維持するためのトリガー関数。削除または更新操作時に、参照元テーブルの整合性を検証・適用する。

シグネチャまたは形状

Datum check_foreign_key(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（TriggerData を含む）。トリガー引数として、参照数、アクション種別（'r', 'c', 's'）、キーカラム名、参照先テーブル名を受け取る。
  • **出力**: Datum（更新操作の場合は tg_newtuple、削除操作の場合は tg_trigtuple を返す）。

前提条件と不変条件

  • AFTER トリガーとして起動される必要がある。
  • 行レベル（ROW）トリガーである必要がある。
  • INSERT イベントでは使用できない。
  • 参照整合性アクション（RESTRICT, CASCADE, SET NULL）のいずれかが指定されていること。

アルゴリズムまたは振る舞い

1. **検証**: トリガーマネージャからの呼び出しであるか、イベント種別が適切かを確認する。 2. **引数解析**: CREATE TRIGGER で指定された引数を解析し、アクション種別とキーカラムを特定する。 3. **計画検索**: find_plan を呼び出し、トリガー名・リレーションID・操作種別に基づいた実行計画を検索する。 4. **値の抽出**: SPI_getbinval を使用して対象タプルからキー値を抽出する。NULL 値の場合は整合性チェックをスキップして終了する。 5. **更新時の比較**: UPDATE の場合、旧タプルと新タプルのキー値を比較し、変更がない場合は処理を終了する。 6. **計画準備**: 計画が存在しない場合、SPI_prepare を用いて SQL を構築し、TopMemoryContext にキャッシュする。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のトリガーマネージャ（CREATE TRIGGER 文により登録されたトリガー経由）。
  • **呼び出し先**: SPI_connect, find_plan, SPI_fnumber, SPI_getbinval, SPI_getvalue, SPI_finish 等の SPI インターフェース関数。

関連データ構造

  • TriggerData: トリガー起動時のコンテキスト情報。
  • EPlan: 実行計画（SPIPlanPtr）をキャッシュするための構造体。
  • Relation: トリガー対象のリレーション情報。

失敗モードと境界条件

  • **エラー**: 以下の条件で ereport(ERROR) を発行しトランザクションを中断する。
  • トリガーマネージャ以外からの呼び出し。
  • INSERT イベントでの実行。
  • 引数の数が不足している、または不正なアクションが指定されている場合。
  • 存在しないカラムが指定されている場合。
  • 実行計画の不整合（nplans の不一致）。

保守担当者向けメモ

  • 本関数は古いスタイルのトリガー実装である。現代の PostgreSQL では標準の FOREIGN KEY 制約の使用が推奨される。
  • SPI_keepplan を使用して計画を TopMemoryContext に保持しているため、メモリ管理には注意が必要。
  • DEBUG_QUERY マクロを有効にすると、実行される SQL がログに出力される。
