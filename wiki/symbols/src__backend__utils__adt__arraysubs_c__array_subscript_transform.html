array_subscript_transform

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arraysubs.c:55

目的

パーサーが配列の添字操作（SubscriptingRef）を解析する際、添字式を検証・変換し、実行計画に必要な情報をノードに格納する。

シグネチャまたは形状

void array_subscript_transform(SubscriptingRef *sbsref,
                               List *indirection,
                               ParseState *pstate,
                               bool isSlice,
                               bool isAssignment)

入力と出力

  • **入力**:
  • sbsref: 変換対象の SubscriptingRef ノード。
  • indirection: 添字式（A_Indices）のリスト。
  • pstate: パーサーの状態。
  • isSlice: スライス操作かどうかを示すフラグ。
  • isAssignment: 代入操作かどうかを示すフラグ。
  • **出力**: なし（sbsref の refupperindexpr および reflowerindexpr を更新）。

前提条件と不変条件

  • indirection 内の各添字式は A_Indices 型である必要がある。
  • 添字式は INT4 型に強制変換可能であること。
  • 配列の次元数は MAXDIM を超えてはならない。

アルゴリズムまたは振る舞い

1. indirection リストを走査し、各添字式を transformExpr で解析する。 2. 各添字式を INT4OID へ強制変換（coerce_to_target_type）する。変換失敗時はエラーを報告する。 3. isSlice が真の場合、下限値（lidx）が省略されていれば 1 を補完し、上限値（uidx）が省略されていれば NULL を保持する。 4. 変換後の式リストを sbsref->refupperindexpr および reflowerindexpr に格納する。 5. 次元数が MAXDIM を超えていないか検証する。 6. 操作の種類に応じて sbsref->refrestype を設定する（スライスの場合は配列型、要素アクセスの場合は要素型）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/arraysubs.c 内のハンドラ定義（array_subscript_handler 等の transform フィールド）。
  • **呼び出し先**:
  • transformExpr: 式の解析。
  • coerce_to_target_type: 型変換。
  • ereport: エラー報告。
  • lappend: リスト操作。

関連データ構造

  • SubscriptingRef: 添字操作を表現するノード。
  • A_Indices: パーサーが生成する添字情報の構造体。
  • ParseState: パーサーの状態管理。

失敗モードと境界条件

  • **型不一致**: 添字式が整数型に変換できない場合、ERRCODE_DATATYPE_MISMATCH でエラーとなる。
  • **次元数超過**: 配列の次元数が MAXDIM を超える場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラーとなる。

保守担当者向けメモ

  • 本関数は SubscriptingRef ノードの初期化において重要な役割を果たす。
  • isSlice が真の場合の挙動（下限値のデフォルト値補完など）は、SQL 標準の配列スライス構文に依存しているため、変更時は注意が必要。
