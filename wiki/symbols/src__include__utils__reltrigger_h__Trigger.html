Trigger

定義

  • 種別: struct
  • 定義箇所: src/include/utils/reltrigger.h:23

目的

リレーション（テーブル）に関連付けられた個々のトリガーの属性（OID、関数OID、トリガータイプ、引数など）を保持し、リレーションキャッシュ（relcache）内でトリガー情報を管理するために使用されます。

シグネチャまたは形状

typedef struct Trigger
{
	Oid			tgoid;
	char	   *tgname;
	Oid			tgfoid;
	int16		tgtype;
	char		tgenabled;
	bool		tgisinternal;
	bool		tgisclone;
	Oid			tgconstrrelid;
	Oid			tgconstrindid;
	Oid			tgconstraint;
	bool		tgdeferrable;
	bool		tginitdeferred;
	int16		tgnargs;
	int16		tgnattr;
	int16	   *tgattr;
	char	  **tgargs;
	char	   *tgqual;
	char	   *tgoldtable;
	char	   *tgnewtable;
} Trigger;

入力と出力

  • **入力**: pg_trigger システムカタログから読み込まれた各フィールド値。
  • **出力**: リレーションキャッシュの一部として、実行エンジンがトリガーの属性を参照するために提供されます。

前提条件と不変条件

  • relcache がリレーションをオープンする際、pg_trigger から情報を読み取り構築されます。
  • src/include/utils/rel.h で定義される RelationData 構造体から参照されることを前提としています。

アルゴリズムまたは振る舞い

本構造体はデータ保持に特化しており、ロジックは含みません。実行エンジンは、DML操作時に本構造体を参照してトリガーの属性を確認します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/spi/autoinc.c
  • contrib/spi/insert_username.c
  • contrib/spi/moddatetime.c
  • contrib/spi/refint.c
  • contrib/tcn/tcn.c
  • 上記のモジュールは主に trigger->tgname を参照するために本構造体を利用しています。
  • **呼び出し先**: 不明（本構造体はデータ定義のみを提供します）。

関連データ構造

  • TriggerDesc: リレーションに関連付けられた全トリガーの配列と、実行要否フラグを保持する構造体。
  • RelationData: TriggerDesc をメンバとして保持するリレーションの基底構造体。

失敗モードと境界条件

  • トリガーの追加・削除時には、TriggerDesc 内のフラグとの整合性を維持する必要があります。

保守担当者向けメモ

  • 本ファイルは rel.h からインクルードされるため、依存関係を最小限に抑える設計となっています。
  • フィールドの多くは pg_trigger システムカタログの定義と直接対応しています。変更時は pg_trigger.h との整合性に注意してください。
