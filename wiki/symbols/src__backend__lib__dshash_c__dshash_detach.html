dshash_detach

定義

  • 種別: function
  • 定義箇所: src/backend/lib/dshash.c:306

目的

プロセスが共有メモリ上のハッシュテーブルへのアクセスを終了し、ローカルの管理構造体に関連付けられたメモリを解放する。

シグネチャまたは形状

void dshash_detach(dshash_table *hash_table)

入力と出力

  • **入力**: hash_table - 解放対象の dshash_table 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • 呼び出し元は、当該ハッシュテーブルに関連するパーティションロックを一切保持していないこと（ASSERT_NO_PARTITION_LOCKS_HELD_BY_ME により検証される）。

アルゴリズムまたは振る舞い

1. 現在のプロセスがハッシュテーブルのパーティションロックを保持していないことを確認する。 2. ハッシュテーブル自体が他プロセスによって破棄されている可能性があるため、共有メモリ上のデータには触れず、ローカルの dshash_table 構造体に対して pfree を実行してメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/pgstat_shmem.c
  • src/backend/utils/cache/typcache.c
  • **呼び出し先**:
  • pfree (PostgreSQL メモリ管理関数)

関連データ構造

  • dshash_table: 本関数が解放するローカルのハッシュテーブル状態保持構造体。

失敗モードと境界条件

  • パーティションロックを保持した状態で呼び出した場合、アサーション違反によりプロセスが異常終了する可能性がある。
  • dshash_destroy を呼び出したプロセスは、本関数を呼び出してはならない。

保守担当者向けメモ

  • 本関数はローカルメモリの解放のみを行う。共有メモリ上のハッシュテーブル本体を削除する場合は dshash_destroy を使用すること。
