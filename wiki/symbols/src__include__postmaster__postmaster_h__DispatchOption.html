DispatchOption

定義

  • 種別: enum
  • 定義箇所: src/include/postmaster/postmaster.h:133

目的

PostgreSQLの起動時やプロセス生成時における、サブプログラムの実行モードやディスパッチの挙動を識別するために使用されます。

シグネチャまたは形状

typedef enum DispatchOption
{
	DISPATCH_CHECK,
	DISPATCH_BOOT,
	DISPATCH_FORKCHILD,
	DISPATCH_DESCRIBE_CONFIG,
	DISPATCH_SINGLE,
	DISPATCH_POSTMASTER,		/* must be last */
} DispatchOption;

入力と出力

  • 入力: parse_dispatch_option(const char *name) 関数により、文字列から対応する DispatchOption 値へ変換可能です。
  • 出力: プロセス起動時の制御フラグとして利用されます。

前提条件と不変条件

  • DISPATCH_POSTMASTER は列挙型の最後に配置される必要があります。

アルゴリズムまたは振る舞い

PostgreSQLのメインプロセス（main.c）において、起動時の引数やモードに応じて適切なディスパッチ先を決定するために使用されます。各値は、チェックモード、ブートストラップ、子プロセスのフォーク、設定情報の出力、シングルユーザーモード、あるいは通常のPostmaster起動といった実行形態を規定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/main/main.c において、起動時のオプション解析およびディスパッチ制御に使用されています。
  • 呼び出し先: 本列挙型自体は値の定義であり、関数呼び出しを伴うものではありません。

関連データ構造

  • parse_dispatch_option: 文字列から本列挙型への変換を行う関数が src/include/postmaster/postmaster.h で宣言されています。

失敗モードと境界条件

  • parse_dispatch_option に未知の文字列が渡された場合の挙動については、本ソース excerpt には記載がありません。

保守担当者向けメモ

  • DISPATCH_POSTMASTER が常に最後であることを維持してください。
  • 新しい起動モードを追加する場合は、本列挙型への追加と併せて parse_dispatch_option の実装（src/backend/postmaster/postmaster.c）を更新する必要があります。
