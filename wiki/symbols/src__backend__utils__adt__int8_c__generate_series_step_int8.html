generate_series_step_int8

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/int8.c:1382

目的

SQLの generate_series(start, finish, step) 関数（int8型用）のバックエンド実装として、指定された範囲とステップ数に基づき、逐次的に値を生成するセットリターニング関数（SRF）として機能します。

シグネチャまたは形状

Datum generate_series_step_int8(PG_FUNCTION_ARGS)
  • 引数: PG_FUNCTION_ARGS を介して start (int8), finish (int8), およびオプションの step (int8) を受け取ります。
  • 戻り値: Datum (int8型の値)。

入力と出力

  • **入力**:
  • 第1引数: 開始値 (start)
  • 第2引数: 終了値 (finish)
  • 第3引数 (任意): ステップサイズ (step)。省略時は 1 が使用されます。
  • **出力**:
  • 成功時: SRF_RETURN_NEXT により現在の値を返し、終了条件に達した場合は SRF_RETURN_DONE を返します。

前提条件と不変条件

  • step が 0 の場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告します。
  • 複数回の呼び出し間で状態を保持するため、FuncCallContext を使用します。
  • fctx->current は、現在の反復における値を保持します。

アルゴリズムまたは振る舞い

1. **初回呼び出し時**: SRF_IS_FIRSTCALL() によりコンテキストを初期化し、引数を generate_series_fctx 構造体に保存します。 2. **毎回の呼び出し**: SRF_PERCALL_SETUP() でコンテキストを取得します。 3. **値の生成**: fctx->current を現在の結果として返します。 4. **更新と終了判定**:

  • step > 0 かつ current <= finish、または step < 0 かつ current >= finish の間、処理を継続します。
  • 次の値を計算する際、pg_add_s64_overflow を使用してオーバーフローをチェックします。オーバーフローが発生した場合は fctx->step を 0 に設定し、次回の呼び出しで終了するように制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の src/backend/utils/adt/int8.c:1379 から参照されています。
  • **呼び出し先**: FuncCallContext 関連マクロ、MemoryContextSwitchTo、palloc、pg_add_s64_overflow 等のPostgreSQL内部APIを呼び出します。

関連データ構造

  • generate_series_fctx: 呼び出し間で状態（current, finish, step）を保持するために使用される構造体。
  • FuncCallContext: SRFの実行状態を管理するための標準的なコンテキスト構造体。

失敗モードと境界条件

  • **ステップサイズ0**: step が 0 の場合、エラーをスローします。
  • **オーバーフロー**: current + step が int64 の範囲を超える場合、オーバーフローを検知して安全に終了処理へ移行します。

保守担当者向けメモ

  • 本関数はステートマシンとして動作するため、generate_series_fctx 構造体の変更は、呼び出し間の状態整合性に直接影響します。
  • オーバーフローチェックには pg_add_s64_overflow を使用しており、このロジックは int8 型の境界値（INT64_MIN, INT64_MAX）を正しく扱うよう設計されています。
