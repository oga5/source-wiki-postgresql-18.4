check_locale_time

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale.c:389

目的

GUC（Grand Unified Configuration）パラメータとして設定されたロケール名が、システム上で LC_TIME カテゴリとして有効かどうかを検証します。

シグネチャまたは形状

bool check_locale_time(char **newval, void **extra, GucSource source);

入力と出力

  • **入力**:
  • newval: 設定しようとしているロケール名の文字列へのポインタ。
  • extra: 使用されていません（NULL が渡されます）。
  • source: 設定のソース（GUC の設定元）。
  • **出力**:
  • 戻り値: 指定されたロケールが有効であれば true、無効であれば false を返します。

前提条件と不変条件

  • newval には有効な文字列ポインタが渡される必要があります。
  • 内部で check_locale を呼び出し、LC_TIME カテゴリに対するロケールの妥当性を検証します。

アルゴリズムまたは振る舞い

1. check_locale 関数を呼び出し、第一引数に LC_TIME を指定します。 2. newval に格納されたロケール名を検証対象として渡します。 3. check_locale の検証結果（成功または失敗）をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/guc_tables.c: GUC パラメータのフックとして登録されています。
  • **呼び出し先**:
  • check_locale: ロケールの妥当性チェックを行う共通関数。

関連データ構造

  • GucSource: 設定の由来を示す列挙型。

失敗モードと境界条件

  • 指定されたロケール名が OS でサポートされていない場合、check_locale が false を返し、本関数も false を返します。

保守担当者向けメモ

  • 本関数は LC_TIME 専用のラッパーです。ロケールの検証ロジックそのものは check_locale に依存しているため、検証仕様の変更は check_locale を参照してください。
