range_in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes.c:89

目的

範囲型のテキスト表現（例: [a, b)）を解析し、内部的な RangeType 構造体へ変換する。

シグネチャまたは形状

Datum range_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_CSTRING(0): 範囲型のテキスト表現。
  • PG_GETARG_OID(1): 範囲型の OID。
  • PG_GETARG_INT32(2): 型修飾子（typmod）。
  • **出力**:
  • Datum: 変換された RangeType へのポインタ。解析失敗時は NULL を返す。

前提条件と不変条件

  • check_stack_depth() によりスタック深さを検証する（サブタイプが範囲型の場合の再帰呼び出しに対応）。
  • 内部処理において get_range_io_data を呼び出し、対象範囲型のキャッシュ情報を取得する必要がある。
  • 境界値の解析には range_parse を使用し、要素型の入力関数には InputFunctionCallSafe を使用する。

アルゴリズムまたは振る舞い

1. get_range_io_data を呼び出し、型キャッシュを取得する。 2. range_parse を実行し、フラグ（空、無限大、包含/排他）と境界値の文字列を抽出する。 3. 抽出された境界値文字列に対し、要素型の入力関数を適用して RangeBound 構造体に格納する。 4. 境界値のフラグ（infinite, inclusive, lower）を設定する。 5. make_range を呼び出し、正規化（canonicalization）を含めた RangeType の構築を行う。 6. 構築された RangeType を Datum として返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/multirangetypes.c:111（直接的な呼び出しの可能性あり）
  • **呼び出し先**:
  • check_stack_depth()
  • get_range_io_data()
  • range_parse()
  • InputFunctionCallSafe()
  • make_range()

関連データ構造

  • RangeType: 範囲型の内部表現。
  • RangeIOData: 範囲型の入出力に必要なキャッシュ情報。
  • RangeBound: 範囲の境界（下限・上限）を保持する構造体。

失敗モードと境界条件

  • range_parse が失敗した場合、NULL を返す。
  • InputFunctionCallSafe が失敗した場合、NULL を返す。
  • スタック深さが制限を超えた場合、エラーが発生する。

保守担当者向けメモ

  • 本関数は rangetypes.c のコア I/O 関数であり、要素型の入力関数に強く依存している。
  • 境界値の比較や正規化は make_range に委譲されているため、ロジック変更時は make_range の仕様を確認すること。
