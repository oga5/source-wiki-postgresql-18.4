ExtensionControlFile

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/extension.c:85

目的

PostgreSQLの拡張機能（Extension）の制御ファイル（.control）から読み込まれたメタデータを保持し、インストールや更新処理に必要な設定情報を管理する。

シグネチャまたは形状

typedef struct ExtensionControlFile
{
    char       *name;            /* 拡張機能名 */
    char       *basedir;         /* 制御ファイルとスクリプトの基底ディレクトリ */
    char       *control_dir;     /* 制御ファイルが存在するディレクトリ */
    char       *directory;       /* スクリプトファイル用ディレクトリ */
    char       *default_version; /* デフォルトのインストール対象バージョン */
    char       *module_pathname; /* MODULE_PATHNAME の置換文字列 */
    char       *comment;         /* コメント */
    char       *schema;          /* ターゲットスキーマ */
    bool        relocatable;     /* ALTER EXTENSION SET SCHEMA の可否 */
    bool        superuser;       /* インストールにスーパーユーザー権限が必要か */
    bool        trusted;         /* スーパーユーザー権限への昇格を許可するか */
    int         encoding;        /* スクリプトファイルのエンコーディング */
    List       *requires;        /* 前提となる拡張機能名のリスト */
    List       *no_relocate;     /* 再配置不可な前提拡張機能名のリスト */
} ExtensionControlFile;

入力と出力

  • **入力**: 外部の .control ファイルの内容が parse_extension_control_file を通じて本構造体にマッピングされる。
  • **出力**: 拡張機能の作成・更新プロセスにおいて、インストールパスの決定やスクリプト実行時の環境設定（schema や relocatable 等）の参照先として利用される。

前提条件と不変条件

  • relocatable が false の場合のみ、schema フィールドが有効なターゲットスキーマとして扱われる。
  • requires および no_relocate は List 型であり、拡張機能間の依存関係を保持する。

アルゴリズムまたは振る舞い

  • 拡張機能のインストール時、read_extension_control_file によりファイルシステムから情報を読み込み、本構造体に展開する。
  • 拡張機能の更新パス探索やスクリプト実行時に、本構造体の各フィールドが参照され、インストール環境の制約（権限やスキーマ要件）が検証される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/extension.c 内の get_available_versions_for_extension、new_ExtensionControlFile、find_extension_control_filename、get_extension_script_directory、get_extension_aux_control_filename、get_extension_script_filename、parse_extension_control_file 等から参照される。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行わない。

関連データ構造

  • List (PostgreSQLのリスト構造): requires および no_relocate フィールドで使用。

失敗モードと境界条件

  • 制御ファイルが存在しない、または構文エラーがある場合、parse_extension_control_file 実行時にエラーが発生する。
  • encoding が指定されていない場合、デフォルト値として -1 が設定される。

保守担当者向けメモ

  • 本構造体は src/include/commands/extension.h と対になって定義されている。
  • フィールドを追加・変更する場合は、制御ファイルのパーサー（parse_extension_control_file）および関連するカタログ更新ロジックとの整合性に注意すること。
