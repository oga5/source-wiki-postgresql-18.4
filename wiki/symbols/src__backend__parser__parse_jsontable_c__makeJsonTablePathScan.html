makeJsonTablePathScan

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_jsontable.c:496

目的

JSON_TABLE 構文解析において、JSONパススキャンを実行するための JsonTablePathScan ノードを生成します。

シグネチャまたは形状

static JsonTablePlan *makeJsonTablePathScan(JsonTablePathSpec *pathspec, bool errorOnError, int colMin, int colMax, JsonTablePlan *childplan)

入力と出力

  • **入力**:
  • pathspec: パス文字列と名前を含む JsonTablePathSpec 構造体。
  • errorOnError: エラー発生時の挙動を指定するフラグ。
  • colMin, colMax: このスキャンが対象とするカラム範囲のインデックス。
  • childplan: 子ノードとなる JsonTablePlan。
  • **出力**:
  • 生成された JsonTablePlan 型のポインタ（実体は JsonTablePathScan）。

前提条件と不変条件

  • pathspec->string は A_Const ノードである必要があります（Assert により検証）。

アルゴリズムまたは振る舞い

1. makeNode(JsonTablePathScan) を呼び出し、ノードを初期化します。 2. pathspec->string からパス文字列を抽出します。 3. jsonpath_in 関数を呼び出し、抽出した文字列を JSONPATHOID 型の定数（Const）に変換します。 4. ノードの各フィールド（path, errorOnError, child, colMin, colMax）に値を設定します。 5. 生成したノードを JsonTablePlan として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_jsontable.c 内の解析処理（transformJsonTableNestedColumns 等）から呼び出されます。
  • **呼び出し先**: makeNode、castNode、makeConst、DirectFunctionCall1（jsonpath_in を実行）、makeJsonTablePath。

関連データ構造

  • JsonTablePathScan: 本関数が生成する実行計画ノード。
  • JsonTablePathSpec: 入力として受け取るパス定義。
  • JsonTablePlan: 戻り値の基底型。

失敗モードと境界条件

  • pathspec->string が A_Const でない場合、アサーション失敗により異常終了します。
  • jsonpath_in が不正なパス文字列を受け取った場合、内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は JSON_TABLE の実行計画構築における主要なヘルパーです。JSON_TABLE のネスト構造を扱う際、パススキャンノードの生成ロジックに変更が必要な場合は本関数を確認してください。
