DecodingOutputState

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/logicalfuncs.c:40

目的

論理デコーディングの出力プラグインから受け取ったデータを一時的に保持し、SQLインターフェースの戻り値である Tuplestore へ変換・蓄積するための状態管理構造体です。

シグネチャまたは形状

typedef struct DecodingOutputState
{
	Tuplestorestate *tupstore;
	TupleDesc	tupdesc;
	bool		binary_output;
	int64		returned_rows;
} DecodingOutputState;

入力と出力

  • **入力**: pg_logical_slot_get_changes_guts 等の関数内で palloc0 を用いてメモリ確保され、論理デコーディングコンテキストの output_writer_private に格納されます。
  • **出力**: LogicalOutputWrite コールバックを通じて、デコードされたデータが tupstore に書き込まれます。

前提条件と不変条件

  • tupstore は、結果セットを構築するために適切に初期化されている必要があります。
  • binary_output フラグは、出力形式（テキストまたはバイナリ）を決定するために使用されます。

アルゴリズムまたは振る舞い

1. pg_logical_slot_get_changes_guts が実行される際、この構造体が初期化されます。 2. 論理デコーディングの処理中、出力プラグインがデータを生成すると LogicalOutputWrite が呼び出されます。 3. LogicalOutputWrite は、この構造体の tupstore メンバを参照し、データを Tuplestore に格納します。 4. returned_rows は、処理された行数を追跡するために更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/logicalfuncs.c 内の pg_logical_slot_get_changes_guts 等の関数が、この構造体のインスタンスを生成・管理します。
  • **呼び出し先**: LogicalOutputWrite がこの構造体を参照して書き込みを行います。

関連データ構造

  • Tuplestorestate: 抽出された変更内容を一時的に保持するストレージ。
  • TupleDesc: 返却されるタプルの記述子。

失敗モードと境界条件

  • メモリ不足時: palloc0 によるメモリ確保に失敗した場合、PostgreSQLの標準的なエラーハンドリング（elog 等）により処理が中断されます。
  • 構造体の初期化漏れ: output_writer_private が正しく設定されていない状態でコールバックが呼ばれた場合、不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は logicalfuncs.c 内のSQLインターフェース専用です。論理デコーディングのコアロジック（decode.c や reorderbuffer.c）には依存しません。
  • returned_rows の更新が正しく行われているか確認する際は、LogicalOutputWrite の実装を参照してください。
