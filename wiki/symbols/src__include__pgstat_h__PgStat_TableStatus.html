PgStat_TableStatus

定義

  • 種別: struct
  • 定義箇所: src/include/pgstat.h:174

目的

バックエンドプロセスがローカルでテーブル統計情報を蓄積・管理するための構造体です。共有メモリへ統計情報をフラッシュするまでの間、一時的な保持場所として機能します。

シグネチャまたは形状

typedef struct PgStat_TableStatus
{
	Oid			id;				/* table's OID */
	bool		shared;			/* is it a shared catalog? */
	struct PgStat_TableXactStatus *trans;	/* lowest subxact's counts */
	PgStat_TableCounts counts;	/* event counts to be sent */
	Relation	relation;		/* rel that is using this entry */
} PgStat_TableStatus;

入力と出力

  • **入力**: 各種統計更新関数（pgstat_count_* 系）を通じて、counts フィールドの各カウンタがインクリメントされます。
  • **出力**: pgstat_report_stat() が呼び出されると、counts に蓄積された情報が共有メモリ上の統計エントリへ反映されます。

前提条件と不変条件

  • id は対象テーブルの OID を保持します。
  • relation フィールドは、この統計エントリを使用しているリレーションへのポインタを保持します。
  • trans フィールドは、トランザクションのネストレベルに応じた統計状態（PgStat_TableXactStatus）を指します。

アルゴリズムまたは振る舞い

1. **初期化**: pgstat_prep_relation_pending 等により、統計対象のリレーションに対して本構造体が準備されます。 2. **蓄積**: バックエンドでのテーブル操作（挿入、更新、削除等）が発生するたびに、ローカルの counts が更新されます。 3. **フラッシュ**: pgstat_report_stat() 実行時、ローカルの counts が共有メモリへ転送されます。 4. **管理**: トランザクションのネストレベルに応じて trans が管理され、サブトランザクションの統計が適切に集約されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat_relation.c 内の関数群（pgstat_prep_relation_pending, add_tabstat_xact_level 等）が本構造体を生成・操作します。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を直接呼び出すことはありません。統計更新処理において、Relation 構造体から本構造体への参照が取得されます。

関連データ構造

  • PgStat_TableCounts: 実際の統計カウンタ（挿入、更新、削除数など）を保持します。
  • PgStat_TableXactStatus: トランザクションレベルの統計状態を保持します。
  • Relation: 統計対象となるリレーションのメタデータ。

失敗モードと境界条件

  • 統計情報が有効でない場合、本構造体の生成や更新はスキップされる可能性があります（pgstat_should_count_relation マクロによる判定）。
  • 共有メモリへのフラッシュ時にエラーが発生した場合の挙動については、本ソース excerpt からは判断できません。

保守担当者向けメモ

  • PgStat_TableStatus の構造を変更した場合は、統計ファイルの互換性を維持するために PGSTAT_FILE_FORMAT_ID の更新が必要となる場合があります。
  • 本構造体はバックエンドローカルなメモリ上に存在するため、プロセス終了時のクリーンアップ処理が正しく行われることを確認してください。
