pgstat_flush_backend_entry_wal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_backend.c:225

目的

プロセスローカルに蓄積されたWAL統計情報を、共有メモリ上のバックエンド統計エントリへ反映（フラッシュ）する。

シグネチャまたは形状

static void pgstat_flush_backend_entry_wal(PgStat_EntryRef *entry_ref)

入力と出力

  • **入力**: entry_ref (共有メモリ上の統計エントリへの参照)
  • **出力**: なし（副作用として共有メモリ上の統計値を更新し、ローカルの基準値を更新する）

前提条件と不変条件

  • entry_ref はロックされた状態で渡される必要がある（呼び出し元の文脈による）。
  • WAL統計の差分計算のため、グローバル変数 pgWalUsage と prevBackendWalUsage を使用する。

アルゴリズムまたは振る舞い

1. pgstat_backend_wal_have_pending() を呼び出し、保留中のWAL統計がない場合は早期リターンする。 2. WalUsageAccumDiff を使用し、現在の pgWalUsage と前回のフラッシュ時の値 prevBackendWalUsage との差分を算出する。 3. 算出された差分を、entry_ref が指す共有メモリ上の PgStatShared_Backend 構造体内の wal_counters に加算する。 4. 次回の差分計算に備え、prevBackendWalUsage を現在の pgWalUsage で更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: pgstat_flush_backend（src/backend/utils/activity/pgstat_backend.c 内）から呼び出される。
  • **呼び出し先**:
  • pgstat_backend_wal_have_pending()
  • WalUsageAccumDiff()

関連データ構造

  • PgStat_EntryRef: 共有メモリ上の統計エントリへのアクセスを管理する構造体。
  • PgStatShared_Backend: バックエンド統計を保持する共有メモリ上の構造体。
  • WalUsage: WAL統計カウンタを保持する構造体。

失敗モードと境界条件

  • 保留中のWAL統計がない場合、共有メモリへの書き込みは行われない。
  • 共有メモリへのアクセスは、呼び出し元が適切にロックを保持していることに依存する。

保守担当者向けメモ

  • 本関数は、メモリ割り当てを避ける設計となっており、クリティカルセクション内からの呼び出しを想定している。
  • prevBackendWalUsage はプロセスローカルな静的変数であり、本関数によってのみ更新される。
