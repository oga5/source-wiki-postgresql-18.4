ExecInitGather

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGather.c:52

目的

並列クエリ実行における Gather ノードの実行状態（GatherState）を初期化し、子プランの実行準備を行う。

シグネチャまたは形状

GatherState *ExecInitGather(Gather *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の Gather プランノード。
  • estate: 実行状態を保持する EState 構造体。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの GatherState 構造体へのポインタ。

前提条件と不変条件

  • Gather ノードは innerPlan を持たない（Assert(innerPlan(node) == NULL)）。
  • Gather ノードは qual を持たない（子ノード側で評価すべきであるため）。
  • GatherState の ps.ExecProcNode には ExecGather が割り当てられる。

アルゴリズムまたは振る舞い

1. GatherState 構造体を割り当て、基本情報を設定する。 2. parallel_leader_participation 設定と node->single_copy に基づき、リーダープロセスがローカルでスキャンを行う必要があるか（need_to_scan_locally）を判定する。 3. ExecAssignExprContext を呼び出し、式評価用のコンテキストを作成する。 4. ExecInitNode を再帰的に呼び出し、子プラン（outerPlan）を初期化する。 5. ExecInitResultTypeTL および ExecConditionalAssignProjectionInfo を使用して、結果の型と射影情報を設定する。 6. ExecInitExtraTupleSlot を使用し、子プランのタプル記述子に基づいた funnel_slot を作成する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execProcnode.c:356 (直接の呼び出しを確認)
  • **呼び出し先**:
  • ExecInitNode (子プランの初期化)
  • ExecAssignExprContext
  • ExecInitResultTypeTL
  • ExecConditionalAssignProjectionInfo
  • ExecInitExtraTupleSlot

関連データ構造

  • GatherState: 本関数で初期化される主要な状態構造体。
  • Gather: プランノード。
  • EState: 実行状態。
  • TupleDesc: タプルの記述子。

失敗モードと境界条件

  • 子プランの初期化が失敗した場合、ExecInitNode の挙動に従いエラーが発生する。
  • innerPlan が存在する場合、Assert によりデバッグビルドでは停止する。

保守担当者向けメモ

  • 本関数は Gather ノードの初期化のみを担当する。実行時のワーカー管理やタプル取得は ExecGather および gather_getnext が担う。
  • outeropsset および resultopsset が true に設定されているが、射影がない場合は resultopsfixed が false となる点に注意すること。
