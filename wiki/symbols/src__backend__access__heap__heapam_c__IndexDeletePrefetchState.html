IndexDeletePrefetchState

定義

  • 種別: struct
  • 定義箇所: src/backend/access/heap/heapam.c:177

目的

インデックス削除処理におけるプリフェッチ操作の状態を保持し、削除対象のタプル識別子（TID）のバッチ処理を管理する。

シグネチャまたは形状

typedef struct
{
	BlockNumber cur_hblkno;
	int			next_item;
	int			ndeltids;
	TM_IndexDelete *deltids;
} IndexDeletePrefetchState;

入力と出力

本構造体は関数間で状態を共有するために使用されます。

  • 入力: 削除対象の TM_IndexDelete 配列へのポインタおよびその件数。
  • 出力: 処理中のブロック番号 (cur_hblkno) や、配列内の現在のインデックス位置 (next_item) の更新。

前提条件と不変条件

  • deltids は ndeltids 分の要素を持つ有効なメモリ領域を指している必要があります。
  • next_item は 0 から ndeltids - 1 の範囲内である必要があります。

アルゴリズムまたは振る舞い

インデックス削除の際、ヒープブロック単位でプリフェッチを行うために使用されます。cur_hblkno で現在のブロックを追跡し、next_item を進めることで、削除対象の TID を順次処理します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/heapam.c 内のインデックス削除関連関数（8023 行目および 8137 行目付近で参照を確認）。
  • 呼び出し先: 具体的な関数呼び出し関係は提供されたソースからは特定できません。

関連データ構造

  • TM_IndexDelete: 削除対象の TID を保持する構造体。

失敗モードと境界条件

  • deltids が NULL の状態でアクセスした場合の動作は未定義です。
  • next_item が ndeltids を超えた場合の境界チェックについては、本構造体単体では規定されていません。

保守担当者向けメモ

  • 本構造体は heapam.c 内のインデックス削除ロジックに密結合しています。変更時は、関連する TM_IndexDelete の処理フロー全体に影響がないか確認してください。
