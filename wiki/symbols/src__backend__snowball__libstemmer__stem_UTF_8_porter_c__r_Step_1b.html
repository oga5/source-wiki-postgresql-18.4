r_Step_1b

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/snowball/libstemmer/stem_UTF_8_porter.c:267

目的

Porter Stemming アルゴリズムの第 1b 段階を実行し、単語末尾の特定の接尾辞（例: "-eed", "-ed", "-ing" 等）を条件付きで削除または置換する。

シグネチャまたは形状

static int r_Step_1b(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z（ステミング処理の環境状態を保持する構造体）
  • **出力**: int（成功時に 1、失敗または条件不一致時に 0、エラー時に負の値を返す）

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • 処理対象の文字列は UTF-8 エンコーディングである。
  • 内部の among 構造体（a_2）および g_v（母音グループ）が定義されている必要がある。

アルゴリズムまたは振る舞い

1. **接尾辞マッチング**: find_among_b を使用して、単語末尾が特定の接尾辞パターンに一致するかを確認する。 2. **条件分岐**:

  • **ケース 1**: 接尾辞が "-eed" に関連する場合、r_R1 を呼び出し、語幹領域 R1 内にあるかを確認した上で置換を行う。
  • **ケース 2**: 接尾辞が "-ed" または "-ing" に関連する場合、直前に母音が存在するかを確認し、条件を満たせば削除する。その後、さらに接尾辞（"-at", "-bl", "-iz" 等）の追加チェックを行い、必要に応じて語尾の調整（例: "e" の付加や二重子音の解消）を行う。

    • 3. **状態更新**: z->bra および z->ket を更新し、slice_from_s や slice_del を通じて文字列バッファを操作する。

関連データ構造

  • SN_env: ステミング環境（カーソル位置、文字列バッファ、領域境界など）。
  • among: 接尾辞マッチングのためのトライ木構造。

失敗モードと境界条件

  • z->c（カーソル位置）が文字列の境界（z->lb）を超えた場合、またはマッチングに失敗した場合は 0 を返す。
  • slice_from_s や slice_del 等の操作が失敗した場合、負の値を返す。
  • 特定の条件（r_R1 の判定や母音の有無）を満たさない場合、変更を行わずに 0 を返す。
