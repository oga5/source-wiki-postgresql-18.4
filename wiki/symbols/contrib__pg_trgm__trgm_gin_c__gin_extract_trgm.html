gin_extract_trgm

定義

  • 種別: function
  • 定義箇所: contrib/pg_trgm/trgm_gin.c:23

目的

GIN インデックスの構築および検索時において、引数の数に基づき適切なトリグラム抽出処理へルーティングを行うインターフェース関数です。

シグネチャまたは形状

Datum gin_extract_trgm(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（PostgreSQL の関数呼び出し引数）
  • **出力**: Datum 型（抽出されたトリグラムの配列、またはクエリ条件のトリグラム集合）
  • **副作用**: 引数の数が 3 または 7 以外の場合、elog(ERROR) によりエラーを発生させます。

前提条件と不変条件

  • GIN インデックスの演算クラス（opclass）から呼び出されることを前提としています。
  • 引数の数（PG_NARGS()）が 3 の場合はインデックス構築（値の抽出）として、7 の場合は検索（クエリの抽出）として処理されます。

アルゴリズムまたは振る舞い

1. PG_NARGS() を確認します。 2. 引数が 3 つの場合、gin_extract_value_trgm を呼び出し、結果を返します。 3. 引数が 7 つの場合、gin_extract_query_trgm を呼び出し、結果を返します。 4. 上記以外の場合は elog(ERROR) で処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GIN インデックス管理機能（SQL レベルの演算クラス定義を通じて呼び出されます）。
  • **呼び出し先**:
  • gin_extract_value_trgm (インデックス構築時)
  • gin_extract_query_trgm (検索時)

関連データ構造

  • Datum: PostgreSQL の汎用データ型。
  • PG_FUNCTION_ARGS: 関数引数コンテキスト。

失敗モードと境界条件

  • 引数の数が 3 でも 7 でもない場合、elog(ERROR) が発生し、トランザクションがアボートされます。

保守担当者向けメモ

  • 本関数は単なるディスパッチャーです。具体的なトリグラム抽出ロジックの変更は gin_extract_value_trgm または gin_extract_query_trgm に対して行ってください。
