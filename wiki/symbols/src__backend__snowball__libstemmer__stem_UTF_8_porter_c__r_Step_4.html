r_Step_4

定義

  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_porter.c:479

目的

Porter Stemming アルゴリズムの第 4 段階として、特定の接尾辞を単語から削除する。

シグネチャまたは形状

static int r_Step_4(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境の状態を保持する構造体）
  • 出力: 成功時に 1、失敗または条件不一致時に 0、エラー時に負の値を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • 処理対象の単語が R2 領域（r_R2 関数で定義される領域）内に接尾辞を持つ必要がある。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置を ket に設定する。 2. 文字列末尾の文字が特定の条件（a_5 に定義された接尾辞のいずれか）を満たすか確認する。 3. r_R2(z) を呼び出し、現在の位置が R2 領域内であることを検証する。 4. マッチした接尾辞の種類（among_var）に応じて以下の処理を行う:

  • case 1: slice_del(z) を実行し、接尾辞を削除する。
  • case 2: 接尾辞が 's' または 't' で終わる場合、それらを削除する。

呼び出し元と呼び出し先

  • 呼び出し元: porter_UTF_8_stem（直接的または間接的）、および stem_UTF_8_porter.c 内のメインフロー。
  • 呼び出し先: find_among_b（接尾辞検索）、r_R2（領域判定）、slice_del（文字列操作）。

関連データ構造

  • SN_env: ステミング環境を管理する構造体。
  • a_5: r_Step_4 で使用される接尾辞のリスト（among 構造体）。

失敗モードと境界条件

  • r_R2 が R2 領域外と判定した場合、処理を中断し 0 またはエラーコードを返す。
  • 接尾辞がマッチしない場合、何も変更せず 0 を返す。
  • slice_del 実行時にメモリ不足等のエラーが発生した場合、負の値を返す。
