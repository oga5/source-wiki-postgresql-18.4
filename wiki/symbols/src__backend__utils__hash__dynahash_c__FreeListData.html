FreeListData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/hash/dynahash.c:153

目的

ハッシュテーブルの各パーティションに関連付けられた空き要素リストと、それに対する並行アクセス制御用スピンロックを管理する。

シグネチャまたは形状

typedef struct
{
	slock_t		mutex;			/* spinlock for this freelist */
	long		nentries;		/* number of entries in associated buckets */
	HASHELEMENT *freeList;		/* chain of free elements */
} FreeListData;

入力と出力

本構造体はデータ保持用であり、関数ではないため入出力は定義されない。

前提条件と不変条件

  • mutex は、対応するパーティションの freeList および nentries へのアクセスを保護するために使用される。
  • nentries は、当該パーティションに関連付けられたバケット内の要素数を保持する。

アルゴリズムまたは振る舞い

  • パーティション分割されたハッシュテーブルにおいて、FREELIST_IDX マクロで決定されるインデックスに基づき、FreeListData の配列から特定のパーティションが選択される。
  • 要素の追加や削除の際、対応する FreeListData の mutex を取得することで、共有メモリ環境下での競合を回避する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/hash/dynahash.c 内のハッシュテーブル管理ロジック（hash_search 等）が、freeList 配列の要素として本構造体を参照する。
  • 呼び出し先: 構造体内部に直接的な関数呼び出しは含まれない。

関連データ構造

  • HASHELEMENT: freeList が指す要素の型。
  • HTAB: 本構造体を保持するハッシュテーブルの制御構造体。

失敗モードと境界条件

  • mutex の取得に失敗した場合や、デッドロックが発生するような不適切なロック順序でのアクセスは、システム全体のハングアップを招く可能性がある。

保守担当者向けメモ

  • src/backend/utils/hash/dynahash.c:180 にて FreeListData freeList[NUM_FREELISTS] として配列定義されている。
  • 個別のロックではなく FreeListData を配列として管理することで、パーティション単位の排他制御を実現している。
