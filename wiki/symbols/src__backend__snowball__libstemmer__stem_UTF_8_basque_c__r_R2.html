r_R2

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_basque.c:999

目的

現在のカーソル位置が、Snowball アルゴリズムで定義される領域「R2」の範囲内にあるかどうかを判定する。

シグネチャまたは形状

static int r_R2(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（ステミング環境構造体）。z->I[0] に R2 の開始位置が、z->c に現在のカーソル位置が格納されている。
  • **出力**: int 型。条件を満たす場合は 1、満たさない場合は 0 を返す。

前提条件と不変条件

  • r_mark_regions 関数が事前に実行され、z->I[0] に R2 の境界位置が正しく設定されている必要がある。

アルゴリズムまたは振る舞い

1. z->I[0]（R2 の開始位置）と z->c（現在のカーソル位置）を比較する。 2. z->I[0] <= z->c が真であれば、カーソルは R2 領域内（または境界上）にあると判断し、1 を返す。 3. それ以外の場合は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: stem_ISO_8859_1_basque.c 等の各言語ステミング実装ファイルから、接尾辞削除の条件判定として呼び出される。
  • **呼び出し先**: なし（算術比較のみを実行）。

関連データ構造

  • SN_env: ステミング処理の実行状態を保持する構造体。I 配列（整数型領域）および c（カーソル位置）を使用する。

失敗モードと境界条件

  • z が NULL の場合、プログラムは異常終了する可能性がある（Snowball ランタイムの標準的な挙動）。
  • r_mark_regions が未実行の場合、z->I[0] の値は不定であり、判定結果は信頼できない。
