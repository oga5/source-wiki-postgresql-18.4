DirectFunctionCall6Coll

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/fmgr.c:914

目的

6つの引数をとる関数を、オーバーヘッドを最小限に抑えて直接呼び出すためのラッパー関数です。

シグネチャまたは形状

Datum DirectFunctionCall6Coll(PGFunction func, Oid collation, Datum arg1, Datum arg2, Datum arg3, Datum arg4, Datum arg5, Datum arg6)

入力と出力

  • **入力**:
  • func: 呼び出す関数のポインタ (PGFunction)。
  • collation: 照合順序 (Oid)。
  • arg1 ～ arg6: 関数に渡す6つの引数 (Datum)。
  • **出力**:
  • 関数の実行結果 (Datum)。

前提条件と不変条件

  • 引数 arg1 ～ arg6 は非NULLとして扱われます。
  • 呼び出される関数 func は、結果としてNULLを返さないことが期待されます。

アルゴリズムまたは振る舞い

1. LOCAL_FCINFO マクロを使用して、6つの引数を持つ FunctionCallInfoBaseData をスタック上に確保します。 2. InitFunctionCallInfoData を呼び出し、照合順序情報等を設定して初期化します。 3. 6つの引数すべてを fcinfo->args 配列に格納し、isnull フラグを false に設定します。 4. 関数ポインタ func を介して直接関数を実行します。 5. 実行後、fcinfo->isnull を確認し、関数がNULLを返した場合は elog(ERROR) を発生させます。 6. 正常終了時に結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/fmgr.h:693 にてマクロ経由で呼び出されています。
  • **呼び出し先**: 引数として渡された PGFunction 型の関数ポインタを直接実行します。

関連データ構造

  • FunctionCallInfoBaseData: 関数呼び出しのコンテキストを保持する構造体。
  • Datum: PostgreSQLの汎用データ型。
  • PGFunction: 関数ポインタの型定義。

失敗モードと境界条件

  • 呼び出された関数がNULLを返した場合、ERROR レベルのログを出力して処理を中断します。
  • 本関数はポリモーフィック関数など、FmgrInfo の完全な初期化を必要とする関数には使用できません。

保守担当者向けメモ

  • 本関数はパフォーマンス重視のパスで使用されるため、実装の変更には慎重を期してください。
  • FmgrInfo を必要としない単純なC関数呼び出しに適しています。
