pg_popcount_optimized

定義

  • 種別: function
  • 定義箇所: src/port/pg_popcount_aarch64.c:280

目的

AArch64環境において、メモリバッファ内のビットカウント（popcount）を高速に実行するためのインターフェースを提供します。

シグネチャまたは形状

uint64 pg_popcount_optimized(const char *buf, int bytes)

入力と出力

  • **入力**:
  • buf: カウント対象のデータが格納されたバッファへのポインタ。
  • bytes: バッファのサイズ（バイト単位）。
  • **出力**: バッファ内に含まれるセットビット（1）の総数を uint64 型で返します。

前提条件と不変条件

  • 本関数は POPCNT_AARCH64 が定義されている環境でのみ有効です。
  • 実行時に適切な実装（NeonまたはSVE）へ動的にディスパッチされる前提で設計されています。

アルゴリズムまたは振る舞い

1. 呼び出されると、内部的に pg_popcount_neon を実行します。 2. SIMD命令セット（NeonまたはSVE）を活用し、バッファ単位で並列的にビットカウントを行います。 3. ベクトル化できない端数データについては、逐次処理によってカウントを完了させます。

呼び出し元と呼び出し先

  • **呼び出し先**: pg_popcount_neon を直接呼び出します。
  • **呼び出し元**: src/include/port/pg_bitutils.h および src/port/pg_bitutils.c 内のビット操作ユーティリティ層から参照されています。

関連データ構造

  • なし。

失敗モードと境界条件

  • bytes が 0 の場合、ビットカウントは 0 を返します。
  • 負の bytes 値が渡された場合の挙動は定義されていません。

保守担当者向けメモ

  • 本関数は関数ポインタとして保持され、初期化時に pg_popcount_choose によって適切な実装へ書き換えられる可能性があります。
  • SVEサポートの動的判定を行う場合は USE_SVE_POPCNT_WITH_RUNTIME_CHECK の定義状況を確認してください。
