TopologicalSort

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_cte.c:862

目的

WITH 句内の各 CTE（共通テーブル式）間の依存関係に基づき、解析順序を決定する。

シグネチャまたは形状

static void TopologicalSort(ParseState *pstate, CteItem *items, int numitems)

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する ParseState 構造体。
  • items: ソート対象となる CteItem の配列。各要素は depends_on（依存先IDのビットマップ）を保持する。
  • numitems: items 配列の要素数。
  • **出力**: なし（items 配列をインプレースで並び替える）。

前提条件と不変条件

  • items 配列内の各 CteItem は、依存関係が depends_on ビットマップとして正しく設定されている必要がある。
  • 循環参照が存在しないことが前提。

アルゴリズムまたは振る舞い

1. 配列の各位置 i に対して、まだ依存関係が解決されていない（depends_on が空の）アイテムを items[i...numitems-1] から探索する。 2. 見つかったアイテムを現在の位置 i に移動（スワップ）する。 3. 移動したアイテムの ID を、残りの未処理アイテムの depends_on ビットマップから削除する。 4. これを全アイテムに対して繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_cte.c 内の transformWithClause（cstate->items を渡して呼び出される）。
  • **呼び出し先**:
  • bms_is_empty (ビットマップ操作)
  • bms_del_member (ビットマップ操作)
  • ereport (エラー報告)
  • parser_errposition (エラー位置特定)

関連データ構造

  • CteItem: CTEの解析状態と依存関係を保持する構造体。
  • ParseState: クエリ解析のコンテキスト。

失敗モードと境界条件

  • **循環参照**: 依存関係を解決できない（depends_on が空のアイテムが見つからない）場合、ERRCODE_FEATURE_NOT_SUPPORTED を発生させエラー終了する。これは相互再帰がサポートされていないためである。

保守担当者向けメモ

  • 本関数は単純なトポロジカルソートアルゴリズムを実装している。
  • 依存関係の管理にはビットマップ（Bitmapset）を使用しているため、CTEの数が多い場合でも効率的に処理される。
