serializeAnalyzeShutdown

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_dr.c:243

目的

EXPLAIN (SERIALIZE) オプション実行時に使用される SerializeDestReceiver が保持するリソースを解放し、クリーンアップを行う。

シグネチャまたは形状

static void serializeAnalyzeShutdown(DestReceiver *self)

入力と出力

  • 入力: DestReceiver *self（SerializeDestReceiver にキャストして使用される計測用レシーバー）
  • 出力: なし（副作用としてメモリ解放を行う）

前提条件と不変条件

  • self は SerializeDestReceiver 型として安全にキャスト可能であること。
  • 呼び出し後、receiver->finfos、receiver->buf.data、および receiver->tmpcontext は NULL に設定される。

アルゴリズムまたは振る舞い

1. self を SerializeDestReceiver 構造体にキャストする。 2. finfos（型情報キャッシュ）が割り当てられている場合、pfree で解放しポインタを NULL にする。 3. buf.data（シリアライズ用バッファ）が割り当てられている場合、pfree で解放しポインタを NULL にする。 4. tmpcontext（行処理用メモリコンテキスト）が存在する場合、MemoryContextDelete で削除しポインタを NULL にする。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/explain_dr.c 内の DestReceiver インターフェース（rShutdown フィールド）を通じて呼び出される。
  • 呼び出し先: pfree、MemoryContextDelete（PostgreSQL メモリ管理 API）。

関連データ構造

  • SerializeDestReceiver: 本関数が管理するリソースを保持する構造体。

失敗モードと境界条件

  • finfos、buf.data、tmpcontext が既に NULL である場合、各ポインタのチェックにより安全にスキップされる。

保守担当者向けメモ

  • 本関数は DestReceiver のライフサイクル終了時に呼び出される。メモリリークを防ぐため、SerializeDestReceiver に新しい動的メモリ割り当てを追加した場合は、必ず本関数で解放処理を追加すること。
