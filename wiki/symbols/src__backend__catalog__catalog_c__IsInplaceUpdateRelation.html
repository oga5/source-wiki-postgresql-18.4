IsInplaceUpdateRelation

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/catalog.c:182

目的

指定されたリレーションが、通常のトランザクション制御を通さずに直接更新（インプレース更新）されるべきシステムカタログであるかを判定します。

シグネチャまたは形状

bool IsInplaceUpdateRelation(Relation relation)

入力と出力

  • **入力**: Relation relation（対象のリレーション記述子）
  • **出力**: bool（インプレース更新対象であれば true、そうでなければ false）

前提条件と不変条件

  • 本関数はカタログアクセスを行いません。ロック取得中や再帰的な呼び出しにおいても安全に呼び出せます。
  • 内部で RelationGetRelid(relation) を使用してOIDを取得し、判定を行います。

アルゴリズムまたは振る舞い

1. 引数 relation から RelationGetRelid を用いてリレーションのOIDを取得します。 2. 取得したOIDを引数として IsInplaceUpdateOid を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c
  • src/backend/access/index/genam.c
  • src/backend/executor/execMain.c
  • **呼び出し先**:
  • RelationGetRelid (マクロ)
  • IsInplaceUpdateOid (同ファイル内)

関連データ構造

  • Relation: リレーション記述子。

失敗モードと境界条件

  • relation が NULL の場合、RelationGetRelid の実装に依存した動作（通常はセグメンテーション違反）となります。

保守担当者向けメモ

  • 本関数は判定ロジックを集中管理するユーティリティの一部です。判定基準を変更する場合は、IsInplaceUpdateOid の定義を確認してください。
  • カタログアクセスを発生させない設計を維持してください。
