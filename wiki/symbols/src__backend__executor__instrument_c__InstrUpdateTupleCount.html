InstrUpdateTupleCount

定義

  • 種別: function
  • 定義箇所: src/backend/executor/instrument.c:131

目的

クエリ実行計画ノードが処理したタプル数を、計測用構造体（Instrumentation）に累積加算します。

シグネチャまたは形状

void InstrUpdateTupleCount(Instrumentation *instr, double nTuples)

入力と出力

  • instr: 更新対象の Instrumentation 構造体へのポインタ。
  • nTuples: 加算するタプル数（double 型）。
  • 出力: なし（instr->tuplecount が直接更新されます）。

前提条件と不変条件

  • instr は有効な Instrumentation 構造体を指している必要があります。
  • nTuples は加算される値であり、負の値が渡された場合の挙動は呼び出し側のロジックに依存します。

アルゴリズムまたは振る舞い

1. 引数として渡された nTuples を、instr が保持する tuplecount メンバに加算します。 2. この操作は単純な加算であり、他の統計値（時間やバッファ使用量など）には影響を与えません。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/executor/nodeModifyTable.c
  • **呼び出し先**:
  • なし（本関数は計算のみを行い、他の関数を呼び出しません）。

関連データ構造

  • Instrumentation: 実行計画ノードの統計情報を保持する構造体。tuplecount メンバを介して本関数と連携します。

失敗モードと境界条件

  • instr が NULL の場合、セグメンテーションフォールトが発生します。
  • tuplecount が double 型の最大値を超えた場合のオーバーフロー挙動については、標準的な浮動小数点演算の仕様に従います。

保守担当者向けメモ

  • 本関数は EXPLAIN ANALYZE 等で表示される「実際の行数」の集計に使用されます。
  • 並列クエリ実行時など、複数のワーカーから統計が集約される文脈では、呼び出し元で適切な同期や集約処理が行われていることを確認してください。
