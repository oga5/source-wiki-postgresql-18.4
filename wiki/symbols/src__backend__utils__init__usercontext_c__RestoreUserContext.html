RestoreUserContext

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/usercontext.c:86

目的

SwitchToUntrustedUser 等によって一時的に変更されたユーザーID、セキュリティコンテキスト、および GUC（設定パラメータ）の状態を、元の状態へ復元するために使用します。

シグネチャまたは形状

void RestoreUserContext(UserContext *context)

入力と出力

  • **入力**: UserContext *context - 復元対象の状態を保持する構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 context は、事前に GetUserIdAndSecContext 等で正しく初期化されている必要があります。
  • context->save_nestlevel が -1 の場合、GUC のロールバックはスキップされます。

アルゴリズムまたは振る舞い

1. context->save_nestlevel が -1 でない場合、AtEOXact_GUC(false, context->save_nestlevel) を呼び出し、指定されたネストレベル以降の GUC 変更をロールバックします。 2. SetUserIdAndSecContext を呼び出し、context 内に保存されていた save_userid および save_sec_context を適用してユーザー状態を復元します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c
  • src/backend/replication/logical/tablesync.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • AtEOXact_GUC
  • SetUserIdAndSecContext

関連データ構造

  • UserContext: ユーザーID、セキュリティコンテキスト、GUC ネストレベルを保持する構造体（src/include/utils/usercontext.h で定義）。

失敗モードと境界条件

  • 本関数を呼び出し忘れると、セッションのユーザー状態が不正なまま残るリスクがあります。
  • context が指すメモリ領域が不正な場合、セッション状態の復元に失敗し、予期せぬ権限で処理が継続される可能性があります。

保守担当者向けメモ

  • SwitchToUntrustedUser と対で使用することが必須です。
  • GUC のネストレベル管理（save_nestlevel）は、セキュリティ制限が適用された操作においてのみ有効である点に注意してください。
