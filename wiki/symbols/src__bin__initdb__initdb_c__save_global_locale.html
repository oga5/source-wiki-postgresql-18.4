save_global_locale

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/initdb.c:364

目的

現在のグローバルロケール設定を保存し、後で復元可能な形式で取得する。

シグネチャまたは形状

static save_locale_t save_global_locale(int category)

入力と出力

  • **入力**: category (int型) - setlocale に渡すロケールカテゴリ（例: LC_TIME）。
  • **出力**: save_locale_t 型の値を返す。これは現在のロケール文字列のコピーである。

前提条件と不変条件

  • 呼び出し側は、返された save_locale_t 型のメモリを適切に管理（解放）する責任を負う。
  • setlocale または _wsetlocale が失敗した場合、pg_fatal を呼び出しプログラムを終了させる。

アルゴリズムまたは振る舞い

1. WIN32 環境下では _wsetlocale(category, NULL) を使用して現在のロケールを取得し、wcsdup で複製する。 2. それ以外の環境では setlocale(category, NULL) を使用して現在のロケールを取得し、pg_strdup で複製する。 3. 取得したロケール文字列を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/initdb/initdb.c:2156 (LC_TIME を引数として呼び出し)
  • src/bin/initdb/initdb.c:2214 (任意の category を引数として呼び出し)
  • **呼び出し先**:
  • setlocale / _wsetlocale (標準ライブラリ)
  • pg_strdup / wcsdup (メモリ確保)
  • pg_fatal (エラーハンドリング)

関連データ構造

  • save_locale_t: ロケール情報を保持するための型。

失敗モードと境界条件

  • ロケールの取得に失敗した場合、pg_fatal によりエラーメッセージを出力してプロセスが終了する。
  • メモリ不足によりロケール文字列の複製に失敗した場合、pg_fatal によりプロセスが終了する。

保守担当者向けメモ

  • 本関数で取得したロケールは、対となる restore_global_locale を使用して復元することを想定している。
  • WIN32 とそれ以外でロケール取得関数が異なるため、プラットフォーム固有の挙動に注意すること。
