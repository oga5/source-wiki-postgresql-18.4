int_custom_in

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_custom_types/test_custom_types.c:44

目的

PostgreSQLのカスタム型（int_custom）に対する入力関数として、文字列形式の値を内部表現（int32）へ変換する。

シグネチャまたは形状

Datum int_custom_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して文字列（char *）を受け取る。
  • **出力**: PG_RETURN_INT32 を介して変換された int32 値を Datum 型として返す。

前提条件と不変条件

  • 入力文字列は pg_strtoint32_safe が許容する形式である必要がある。
  • fcinfo->context が有効なコンテキストとして pg_strtoint32_safe に渡される。

アルゴリズムまたは振る舞い

1. PG_GETARG_CSTRING(0) で入力文字列を取得する。 2. pg_strtoint32_safe を呼び出し、文字列を int32 に変換する。この際、エラーハンドリングのために fcinfo->context を渡す。 3. 変換結果を PG_RETURN_INT32 でラップして返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。具体的なSQL実行時の呼び出し経路はソースコード上では明示されていない。
  • **呼び出し先**: pg_strtoint32_safe（utils/builtins.h で定義）。

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • PG_FUNCTION_ARGS: 関数引数情報。

失敗モードと境界条件

  • 入力文字列が int32 の範囲外である場合、または数値として不正な場合、pg_strtoint32_safe の内部実装に従いエラーが発生する。

保守担当者向けメモ

  • 本関数はテスト用モジュールの一部であり、実運用環境の型定義ではなく、カスタム型のI/Oインターフェースの検証を目的としている。
  • pg_strtoint32_safe の挙動に依存しているため、変換ロジックの変更が必要な場合は同ユーティリティ関数の仕様を確認すること。
