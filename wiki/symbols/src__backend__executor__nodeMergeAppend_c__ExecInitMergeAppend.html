ExecInitMergeAppend

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMergeAppend.c:64

目的

MergeAppend ノードの実行時状態（MergeAppendState）を初期化し、複数のソート済み入力ストリームをマージするための準備を行う。

シグネチャまたは形状

MergeAppendState *ExecInitMergeAppend(MergeAppend *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の MergeAppend プランノード。
  • estate: 実行状態を保持する EState。
  • eflags: 実行フラグ（EXEC_FLAG_BACKWARD および EXEC_FLAG_MARK はサポート外）。
  • **出力**:
  • 初期化済みの MergeAppendState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK が含まれていないこと。
  • パーティション・プルーニングが有効な場合、part_prune_index が適切に設定されていること。
  • サブプランが少なくとも1つ存在すること。

アルゴリズムまたは振る舞い

1. **状態の生成**: MergeAppendState を割り当て、ExecMergeAppend を実行関数として登録する。 2. **パーティション・プルーニング**: node->part_prune_index に基づき、ExecInitPartitionExecPruning を呼び出して有効なサブプランを特定する。プルーニングが不要な場合は全サブプランを有効とする。 3. **リソース確保**: サブプラン数に応じた PlanState 配列、タプルスロット配列、およびマージ用の binaryheap を割り当てる。 4. **サブプラン初期化**: 有効な各サブプランに対して ExecInitNode を再帰的に呼び出す。 5. **スロット設定**: 子プランの出力スロット型が共通であればそれを採用し、そうでなければ TTSOpsVirtual を使用する。 6. **ソートキー設定**: SortSupport を使用して各ソートキーの照合順序や比較関数を準備する（省略キー変換は無効化）。 7. **初期化フラグ**: ms_initialized を false に設定して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内の ExecInitNode から呼び出される。
  • **呼び出し先**: ExecInitPartitionExecPruning, ExecInitNode, ExecGetCommonSlotOps, ExecInitResultTupleSlotTL, binaryheap_allocate, PrepareSortSupportFromOrderingOp 等。

関連データ構造

  • MergeAppendState: ノードの実行時状態を保持。
  • MergeAppend: プランノード定義。
  • binaryheap: マージ処理に使用する優先度付きキュー。
  • SortSupportData: ソートキーの比較ロジックを保持。

失敗モードと境界条件

  • サブプランが存在しない場合、アサーションエラーが発生する可能性がある。
  • サブプラン数が非常に多い場合、ヒープ操作のコストが増大する。

保守担当者向けメモ

  • 本関数は ExecMergeAppend で使用される binaryheap の比較関数として heap_compare_slots を登録する。
  • abbreviated key による最適化は、タプルがヒープへ順次投入される特性上、変換コストがメリットを上回るため意図的に無効化されている。
