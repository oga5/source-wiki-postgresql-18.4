relmap_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/relmapdesc.c:19

目的

リレーションマッピング（RelMapper）に関連するWALレコードの内容を解析し、人間が読み取り可能なテキスト形式に変換して出力する。

シグネチャまたは形状

void relmap_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 解析結果を格納する StringInfo バッファ。
  • record: 解析対象のWALレコード状態を保持する XLogReaderState 構造体。
  • **出力**: なし（buf に文字列が追記される）。

前提条件と不変条件

  • record は有効なWALレコードを指していること。
  • XLogRecGetInfo(record) で取得される情報が XLOG_RELMAP_UPDATE である場合にのみ、詳細な解析が行われる。

アルゴリズムまたは振る舞い

1. XLogRecGetData を呼び出し、WALレコードの生データポインタを取得する。 2. XLogRecGetInfo の結果に対して ~XLR_INFO_MASK を適用し、操作タイプを特定する。 3. 操作タイプが XLOG_RELMAP_UPDATE である場合、データを xl_relmap_update 構造体にキャストする。 4. 構造体から dbid（データベースID）、tsid（テーブルスペースID）、nbytes（データサイズ）を抽出し、appendStringInfo を用いて buf に書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h 内で PG_RMGR マクロを通じてリソースマネージャのコールバックとして登録されており、WAL解析ツール（pg_waldump 等）から間接的に呼び出される。
  • **呼び出し先**: XLogRecGetData, XLogRecGetInfo, appendStringInfo（いずれもPostgreSQLのWAL解析基盤関数）。

関連データ構造

  • xl_relmap_update: リレーションマッピング更新時のWALレコード構造体。
  • StringInfo: 文字列構築用のバッファ構造体。
  • XLogReaderState: WALレコード読み取り状態を管理する構造体。

失敗モードと境界条件

  • XLOG_RELMAP_UPDATE 以外の操作タイプが渡された場合、関数は何も出力せず終了する。
  • record が不正なデータを含む場合、キャスト後の構造体アクセスにより予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数はWALの「表示」のみを目的としており、更新ロジックは含まない。
  • relmapper.c に新しいWALレコードタイプが追加された場合は、本関数の if 文を拡張し、対応する構造体の解析処理を追加する必要がある。
