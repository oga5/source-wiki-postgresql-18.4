pgp_parse_pubenc_sesskey

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pubdec.c:149

目的

PGP公開鍵暗号化セッション鍵パケットを解析し、復号および検証を行って、後続のデータ復号に必要なセッション鍵情報を PGP_Context に格納する。

シグネチャまたは形状

int pgp_parse_pubenc_sesskey(PGP_Context *ctx, PullFilter *pkt)

入力と出力

  • **入力**:
  • ctx: 復号コンテキスト（公開鍵情報 pub_key を保持）。
  • pkt: パケットデータを読み出すための PullFilter。
  • **出力**:
  • 成功時: 0（または pgp_expect_packet_end の結果）。
  • 失敗時: 負の整数（エラーコード）。成功した場合、ctx 内の cipher_algo、sess_key_len、sess_key が更新される。

前提条件と不変条件

  • ctx->pub_key が存在すること。
  • パケットのバージョンが 3 であること。
  • 鍵IDが any_key または ctx->pub_key->key_id と一致すること。

アルゴリズムまたは振る舞い

1. **ヘッダ検証**: パケットバージョンと鍵IDを読み込み、検証する。 2. **復号**: アルゴリズム（RSAまたはElGamal）に応じて、対応する復号関数を呼び出し、MPIデータを取得する。 3. **パディング除去**: check_eme_pkcs1_v15 を使用してPKCS#1 v1.5パディングを除去する。 4. **整合性チェック**: control_cksum を使用して復号データのチェックサムを検証する。 5. **格納**: 抽出した暗号アルゴリズムIDとセッション鍵本体を ctx にコピーする。 6. **終了処理**: MPIメモリを解放し、パケットの終端を確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-decrypt.c:1123
  • **呼び出し先**:
  • decrypt_elgamal / decrypt_rsa（復号）
  • check_eme_pkcs1_v15（パディング除去）
  • control_cksum（チェックサム検証）
  • pgp_mpi_free（メモリ解放）
  • pgp_expect_packet_end（パケット終端確認）

関連データ構造

  • PGP_Context: 復号状態と結果を保持。
  • PGP_PubKey: 公開鍵情報。
  • PGP_MPI: 多倍長整数形式の復号データ。
  • PullFilter: パケット読み取り用ストリーム。

失敗モードと境界条件

  • PXE_BUG: 公開鍵が設定されていない場合。
  • PXE_PGP_CORRUPT_DATA: バージョンが3以外の場合。
  • PXE_PGP_WRONG_KEY: 鍵ID不一致、パディング検証失敗、またはチェックサム不一致の場合。
  • PXE_PGP_UNKNOWN_PUBALGO: 未知の公開鍵アルゴリズムの場合。
  • PXE_PGP_KEY_TOO_BIG: セッション鍵が PGP_MAX_KEY を超える場合。

保守担当者向けメモ

  • 本関数はPKCS#1 v1.5パディングに依存している。
  • any_key は鍵ID指定なし（0埋め）の場合のワイルドカードとして機能する。
  • 復号失敗時の PXE_PGP_WRONG_KEY は、パディングエラーやチェックサム不一致など複数の原因を包括する汎用的なエラーコードである。
