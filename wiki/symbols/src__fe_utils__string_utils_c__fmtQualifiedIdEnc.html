fmtQualifiedIdEnc

定義

  • 種別: function
  • 定義箇所: src/fe_utils/string_utils.c:262

目的

スキーマ名と識別子を結合し、指定されたエンコーディングに基づいて適切にクォートされた「スキーマ修飾名」を構築します。

シグネチャまたは形状

const char *fmtQualifiedIdEnc(const char *schema, const char *id, int encoding);

入力と出力

  • **入力**:
  • schema: スキーマ名（NULLまたは空文字列の場合、修飾なしで識別子のみを処理）。
  • id: 識別子名。
  • encoding: 文字列処理に使用するエンコーディングID。
  • **出力**:
  • クォート済みのスキーマ修飾名を含む静的バッファへのポインタ。

前提条件と不変条件

  • 戻り値は内部的な静的バッファを指しており、後続の fmtId 系関数の呼び出しによって内容が上書きされます。そのため、呼び出し直後に結果をコピーするか、使用を完了させる必要があります。
  • スレッドセーフではありません。

アルゴリズムまたは振る舞い

1. 一時的な PQExpBuffer を作成し、schema が存在する場合は fmtIdEnc を用いてクォートしたスキーマ名とドット（.）をバッファに追加します。 2. fmtIdEnc を用いてクォートした id をバッファに追加します。 3. getLocalPQExpBuffer で取得した永続的なバッファに一時バッファの内容をコピーします。 4. 一時バッファを破棄し、永続バッファ内の文字列ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/scripts/common.c
  • src/bin/scripts/reindexdb.c
  • src/bin/scripts/vacuumdb.c
  • src/fe_utils/string_utils.c (ラッパー関数 fmtQualifiedId より)
  • **呼び出し先**:
  • createPQExpBuffer
  • fmtIdEnc
  • appendPQExpBuffer / appendPQExpBufferStr
  • getLocalPQExpBuffer
  • destroyPQExpBuffer

関連データ構造

  • PQExpBuffer: 文字列構築に使用される動的バッファ構造体。

失敗モードと境界条件

  • schema が NULL または空文字列の場合、スキーマ修飾を行わず識別子のみを処理します。
  • メモリ不足時には PQExpBuffer 関連関数内でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は fmtQualifiedId の内部実装として利用されています。エンコーディングを明示的に制御する必要がある場合は、本関数を直接使用することが推奨されます。
