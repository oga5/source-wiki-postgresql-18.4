stop_streaming

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_receivewal.c:183

目的

WAL ストリーミング処理において、指定された終了位置（LSN）への到達、または終了シグナルの受信を判定し、ストリーミングを停止すべきかどうかを制御するコールバック関数です。

シグネチャまたは形状

static bool stop_streaming(XLogRecPtr xlogpos, uint32 timeline, bool segment_finished)

入力と出力

  • **入力**:
  • xlogpos: 現在の WAL 受信位置。
  • timeline: 現在のタイムライン ID。
  • segment_finished: セグメントの終了地点であるかを示すフラグ。
  • **出力**:
  • bool: ストリーミングを停止すべき場合は true、継続する場合は false を返します。

前提条件と不変条件

  • endpos（グローバル変数）が有効な LSN である場合、xlogpos が endpos を超えると停止判定が行われます。
  • time_to_stop（グローバル変数）が true の場合、本関数は true を返します。

アルゴリズムまたは振る舞い

1. segment_finished が真かつ verbose モードの場合、セグメント終了のログを出力します。 2. 現在の xlogpos が endpos を超えている場合、停止ログを出力し、time_to_stop を true に設定して true を返します。 3. タイムラインの切り替わりを検知した場合、前回のタイムラインと位置をログに記録します。 4. time_to_stop が既に真である場合、割り込み信号による終了ログを出力し、true を返します。 5. 上記いずれにも該当しない場合、false を返してストリーミングを継続します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/pg_receivewal.c 内のストリーミング処理（stream.stream_stop コールバックとして設定）。
  • **呼び出し先**: pg_log_info（ログ出力用）。

関連データ構造

  • XLogRecPtr: WAL の位置情報を保持する型。
  • endpos: 停止位置を保持するグローバル変数。
  • time_to_stop: 終了シグナル受信状態を保持するグローバル変数。

失敗モードと境界条件

  • タイムライン切り替え時、xlogpos がセグメントの先頭を指すため、報告される位置が厳密には前タイムラインの終了地点と一致しない場合があります。
  • endpos が無効な値（InvalidXLogRecPtr）である場合、LSN による停止判定はスキップされます。

保守担当者向けメモ

  • 本関数は receivelog.c が提供するストリーミングループからコールバックとして呼び出されます。
  • タイムライン切り替え時のログ出力は、正確な終了位置ではなく「前回の位置」に基づいている点に注意してください。
