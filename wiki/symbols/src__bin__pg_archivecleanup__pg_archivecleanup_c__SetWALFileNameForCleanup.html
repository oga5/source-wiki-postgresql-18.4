SetWALFileNameForCleanup

定義

  • 種別: function
  • 定義箇所: src/bin/pg_archivecleanup/pg_archivecleanup.c:183

目的

指定された基準ファイル名（restartWALFileName）を解析し、アーカイブから削除を開始する境界となる WAL ファイル名（exclusiveCleanupFileName）を決定する。

シグネチャまたは形状

static void SetWALFileNameForCleanup(void)

入力と出力

  • **入力**: グローバル変数 restartWALFileName（基準となるファイル名）および additional_ext（除去対象の拡張子）。
  • **出力**: グローバル変数 exclusiveCleanupFileName に、削除境界となる標準的な WAL ファイル名が格納される。戻り値はない。

前提条件と不変条件

  • restartWALFileName には、WAL ファイル名、.partial ファイル名、または .backup ファイル名のいずれかが格納されている必要がある。
  • ファイル名が上記形式のいずれにも合致しない場合、エラーを出力してプロセスを終了する。

アルゴリズムまたは振る舞い

1. TrimExtension を呼び出し、restartWALFileName から additional_ext を除去する。 2. ファイル名の形式を判定する:

  • **標準 WAL ファイル**: そのまま exclusiveCleanupFileName にコピーする。
  • **.partial ファイル**: sscanf でタイムライン、ログ、セグメント番号を抽出し、XLogFileNameById を用いて標準的な WAL ファイル名形式に変換して exclusiveCleanupFileName に格納する。
  • **.backup ファイル**: sscanf でタイムライン、ログ、セグメント番号、オフセットを抽出し、同様に標準的な WAL ファイル名形式に変換して exclusiveCleanupFileName に格納する。

    • 3. 判定の結果、有効なファイル名が生成されなかった場合は pg_log_error を出力し、終了コード 2 でプログラムを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/bin/pg_archivecleanup/pg_archivecleanup.c:386）。
  • **呼び出し先**: TrimExtension, IsXLogFileName, IsPartialXLogFileName, IsBackupHistoryFileName, XLogFileNameById, pg_log_error, pg_log_error_hint, exit。

関連データ構造

  • restartWALFileName: 処理の基準となるファイル名を保持するグローバル変数。
  • exclusiveCleanupFileName: 決定された削除境界のファイル名を保持するグローバル変数。

失敗モードと境界条件

  • **無効なファイル名**: restartWALFileName が WAL 関連の命名規則に従っていない場合、エラーメッセージを表示して exit(2) を実行する。
  • **解析失敗**: sscanf による数値抽出が期待する個数（3または4）に満たない場合、ファイル名が不正とみなされエラー終了する。

保守担当者向けメモ

  • 本関数は、.partial や .backup ファイルが指定された際に、それらを除去したプレフィックス部分のみを基準として扱うことで、誤った削除を防ぐ設計となっている。
  • ファイル名の解析ロジックは access/xlog_internal.h で定義された命名規則に強く依存しているため、将来的な WAL ファイル名の形式変更時には本関数の sscanf フォーマットを確認すること。
