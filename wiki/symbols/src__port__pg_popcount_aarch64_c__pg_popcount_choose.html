pg_popcount_choose

定義

  • 種別: function
  • 定義箇所: src/port/pg_popcount_aarch64.c:89

目的

AArch64環境において、実行時に最適なビットカウント（popcount）実装を選択し、その結果を用いてバッファのビット数を計算する。

シグネチャまたは形状

static uint64 pg_popcount_choose(const char *buf, int bytes)

入力と出力

  • 入力:
  • buf: カウント対象のデータが格納されたバッファへのポインタ。
  • bytes: バッファのサイズ（バイト単位）。
  • 出力:
  • uint64: バッファ内のセットされているビットの総数。

前提条件と不変条件

  • 本関数は、最適化された実装への関数ポインタ（pg_popcount_optimized）が初期化されていない、あるいは初回呼び出し時に実行される。
  • 実行時に choose_popcount_functions() が呼び出され、CPU機能に応じた実装への切り替えが完了している必要がある。

アルゴリズムまたは振る舞い

1. choose_popcount_functions() を呼び出し、CPUのSVEサポート状況等を判定して pg_popcount_optimized ポインタを適切な実装（NeonまたはSVE）へ書き換える。 2. 書き換え後の pg_popcount_optimized を呼び出し、実際のビットカウント処理を実行して結果を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/pg_bitutils.c および src/port/pg_popcount_aarch64.c 内の初期化処理（関数ポインタの初期値として参照）。
  • 呼び出し先: choose_popcount_functions()、および更新後の pg_popcount_optimized が指す関数。

関連データ構造

  • pg_popcount_optimized: 本関数によって初期化・更新される関数ポインタ。

失敗モードと境界条件

  • bytes が0以下の場合の挙動は、呼び出し先の pg_popcount_optimized 実装に依存する。
  • choose_popcount_functions() が実行環境のCPUフラグを正しく取得できない場合、フォールバック実装が選択される。

保守担当者向けメモ

  • 本関数はディスパッチャとして機能する。初回呼び出し以降は pg_popcount_optimized が直接各実装（pg_popcount_neon 等）を指すようになるため、本関数は実質的に一度だけ実行される。
