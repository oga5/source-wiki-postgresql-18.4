json_manifest_array_start

定義

  • 種別: function
  • 定義箇所: src/common/parse_manifest.c:350

目的

バックアップマニフェストのJSON解析中に配列の開始を検出し、現在の解析状態（JsonManifestParseState）を遷移させるためのコールバック関数です。

シグネチャまたは形状

static JsonParseErrorType json_manifest_array_start(void *state);

入力と出力

  • **入力**: void *state（実際には JsonManifestParseState 構造体へのポインタ）。
  • **出力**: 常に JSON_SUCCESS を返します。ただし、状態遷移に失敗した場合は内部で json_manifest_parse_failure を呼び出し、解析エラーを発生させます。

前提条件と不変条件

  • 引数 state は JsonManifestParseState 型としてキャスト可能である必要があります。
  • 解析状態 parse->state が JM_EXPECT_FILES_START または JM_EXPECT_WAL_RANGES_START のいずれかである必要があります。

アルゴリズムまたは振る舞い

1. state を JsonManifestParseState にキャストします。 2. 現在の parse->state を確認します。

  • JM_EXPECT_FILES_START の場合: JM_EXPECT_FILES_NEXT に遷移します。
  • JM_EXPECT_WAL_RANGES_START の場合: JM_EXPECT_WAL_RANGES_NEXT に遷移します。
  • それ以外の場合: json_manifest_parse_failure を呼び出し、エラーを通知します。

    • 3. JSON_SUCCESS を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: JSONパーサー（pg_parse_json 等）が、解析対象のJSON内で配列の開始 [ を検出した際にコールバックとして呼び出します。
  • **呼び出し先**:
  • json_manifest_parse_failure: 状態遷移が不正な場合に呼び出されます。

関連データ構造

  • JsonManifestParseState: 解析の進捗状況を保持する構造体。

失敗モードと境界条件

  • マニフェストの構造上、期待されていない場所で配列が出現した場合、json_manifest_parse_failure が呼び出され、解析が停止します。

保守担当者向けメモ

  • 本関数は JsonManifestParseContext のコールバックとして登録されます。
  • 状態遷移のロジックは、マニフェストのスキーマ定義（ファイルリストやWAL範囲の配置）に強く依存しています。スキーマが変更された場合は、本関数の switch 文も更新が必要です。
