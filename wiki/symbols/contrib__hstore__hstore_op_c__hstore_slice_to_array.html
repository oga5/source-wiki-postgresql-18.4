hstore_slice_to_array

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_op.c:568

目的

hstore オブジェクトから指定されたキー群に対応する値を抽出し、text 型の配列として返す。

シグネチャまたは形状

Datum hstore_slice_to_array(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • 第1引数: HStore * (対象の hstore データ)
  • 第2引数: ArrayType * (抽出対象のキーを格納した text[])
  • **出力**:
  • 戻り値: ArrayType * (抽出された値の配列。キーが存在しない、または値が NULL の場合は配列内の該当要素が NULL となる)

前提条件と不変条件

  • 入力 hstore は内部的にキーでソートされている必要がある。
  • key_array は TEXTOID 型の配列であること。

アルゴリズムまたは振る舞い

1. deconstruct_array_builtin を使用して、入力配列からキーの Datum 配列を構築する。 2. キー配列が空の場合は、空の text 配列を即座に返す。 3. キーの数分だけメモリを確保し、各キーに対して以下の処理を行う：

  • hstoreFindKey を呼び出し、hstore 内のインデックスを検索する。
  • キーが見つからない、または値が NULL の場合は、出力配列の該当位置を NULL に設定する。
  • キーが見つかった場合は、HSTORE_VAL マクロで値を取得し、cstring_to_text_with_len で text 型に変換して格納する。

    • 4. construct_md_array を使用して、入力配列の次元や境界情報を維持したまま、結果の text 配列を構築して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ経由で呼び出される（PG_FUNCTION_INFO_V1 マクロによる登録あり）。
  • **呼び出し先**: deconstruct_array_builtin, hstoreFindKey, palloc, cstring_to_text_with_len, construct_md_array, construct_empty_array。

関連データ構造

  • HStore: 操作対象の hstore 構造体。
  • HEntry: hstore 内の各要素のメタデータ。
  • ArrayType: 入出力される PostgreSQL 配列構造。

失敗モードと境界条件

  • key_count == 0: 空の配列を正しく処理し、空の text 配列を返す。
  • キーが存在しない場合: エラーにはならず、出力配列の該当要素が NULL となる。
  • 値が NULL の場合: 出力配列の該当要素が NULL となる。

保守担当者向けメモ

  • 本関数は hstore のソート済み特性を利用して hstoreFindKey で高速な検索を行っている。
  • メモリ割り当てには palloc を使用しているため、PostgreSQL のメモリコンテキスト管理に従う。
  • construct_md_array を使用しているため、多次元配列入力に対しても元の次元構造を保持して出力する。
