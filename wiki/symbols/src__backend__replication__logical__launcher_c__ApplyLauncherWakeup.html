ApplyLauncherWakeup

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/launcher.c:1121

目的

論理レプリケーションのランチャープロセスに対して、シグナルを送信して処理の再開（ウェイクアップ）を促す。

シグネチャまたは形状

static void ApplyLauncherWakeup(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてランチャープロセスへのシグナル送信を行う）

前提条件と不変条件

  • LogicalRepCtx が初期化されており、共有メモリ上でアクセス可能であること。
  • ランチャープロセスが起動済みであること（launcher_pid != 0）。

アルゴリズムまたは振る舞い

1. 共有メモリ上の LogicalRepCtx->launcher_pid を参照する。 2. PID が 0 でない（ランチャーが稼働中である）場合、当該プロセスに対して SIGUSR1 シグナルを送信する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/replication/logical/launcher.c:850
  • src/backend/replication/logical/launcher.c:1101
  • 呼び出し先:
  • kill() (システムコール)

関連データ構造

  • LogicalRepCtx: 共有メモリ上の論理レプリケーションコンテキスト。ランチャープロセスの PID を保持する。

失敗モードと境界条件

  • launcher_pid が 0 の場合、シグナルは送信されず、関数は何も行わずに終了する。
  • kill() が失敗した場合の挙動については、本関数内でのエラーハンドリングは定義されていない。

保守担当者向けメモ

  • 本関数は SIGUSR1 を使用してランチャーを叩き起こすための軽量なラッパーです。
  • ランチャープロセス側で SIGUSR1 を受信した際のハンドラが適切に設定されていることを前提としています。
