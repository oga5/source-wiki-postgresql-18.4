r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_english.c:419

目的

Porterステミングアルゴリズムで使用される単語内の領域（R1, R2）を特定し、環境構造体 SN_env の整数配列 I にその境界位置を記録する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z （ステミング対象の文字列と現在のカーソル位置、領域境界を保持する環境）
  • **出力**: int （常に 1 を返す）
  • **副作用**:
  • z->I[1] に R1 の開始位置を格納。
  • z->I[0] に R2 の開始位置を格納。

前提条件と不変条件

  • z は有効な SN_env 構造体を指していること。
  • g_v（母音グループ）が定義されていること。
  • 領域の計算は、単語の先頭から順次 out_grouping_U（非母音）と in_grouping_U（母音）を組み合わせて行われる。

アルゴリズムまたは振る舞い

1. 初期状態として、R1 と R2 の境界を単語の末尾（z->l）に設定する。 2. 特定の例外パターン（a_0）に一致する場合、その直後を R1 の開始点とする。 3. 例外に該当しない場合、以下の手順で領域を特定する：

  • 非母音の直後に母音が続く箇所を検索し、その直後を R1 の開始点とする。
  • R1 の開始点以降で、再度非母音の直後に母音が続く箇所を検索し、その直後を R2 の開始点とする。

    • 4. 処理の過程でカーソル位置（z->c）を一時的に変更するが、終了時には元の位置に復元する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/snowball/libstemmer/stem_ISO_8859_1_english.c 等の各言語実装ファイルから呼び出される（直接の呼び出し関係はソースコードの静的解析に基づく）。
  • **呼び出し先**:
  • find_among: 例外パターンの検索に使用。
  • out_grouping_U, in_grouping_U: 母音・非母音の判定に使用。

関連データ構造

  • struct SN_env: ステミング処理の実行環境を保持する構造体。
  • a_0: 例外パターンを定義した検索テーブル。
  • g_v: 母音の文字集合を定義したグループ。

失敗モードと境界条件

  • 文字列が短く、領域を特定するための母音・非母音のペアが見つからない場合、R1 および R2 は初期値（文字列の末尾）のままとなる。
  • out_grouping_U や in_grouping_U が失敗した場合、処理は lab0 にジャンプし、領域の更新を行わずに終了する。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによって自動生成されたコードである。ロジックの変更が必要な場合は、直接編集せず、対応する .sbl ファイルを修正すること。
  • z->I[1]（R1）と z->I[0]（R2）の計算ロジックは、Porterアルゴリズムの接尾辞削除ルールに直結するため、変更時にはステミングの精度に大きな影響を与える。
