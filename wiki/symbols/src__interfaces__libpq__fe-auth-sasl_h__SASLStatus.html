SASLStatus

定義

  • 種別: enum
  • 定義箇所: src/interfaces/libpq/fe-auth-sasl.h:28

目的

PostgreSQL クライアントにおける SASL 認証交換プロセスの進捗状態を表現し、認証ロジックの制御フローを決定するために使用されます。

シグネチャまたは形状

typedef enum
{
	SASL_COMPLETE = 0,
	SASL_FAILED,
	SASL_CONTINUE,
	SASL_ASYNC,
} SASLStatus;

入力と出力

  • 本シンボルは、認証メカニズム（pg_fe_sasl_mech）の exchange 関数等の戻り値として出力されます。
  • 呼び出し側（fe-auth.c 等）は、この戻り値を受け取り、認証フローの継続、終了、または非同期処理への移行を判断します。

前提条件と不変条件

  • SASL_COMPLETE は認証の正常終了を示します。
  • SASL_FAILED は認証の失敗を示します。
  • SASL_CONTINUE は認証プロセスが進行中であり、次の交換ステップが必要であることを示します。
  • SASL_ASYNC は非同期処理の待機状態を示します。この場合、conn->async_auth の適切な設定が前提となります。

アルゴリズムまたは振る舞い

認証交換の各ステップにおいて、exchange() 関数がサーバーからのチャレンジを処理し、本列挙型のいずれかを返します。呼び出し側はこの値に基づき、次のメッセージ送信を行うか、接続を終了するか、あるいは非同期イベントを待機するかを決定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq/fe-auth.c（認証フロー制御）
  • src/interfaces/libpq/fe-auth-oauth.c（OAuth 認証実装）
  • src/interfaces/libpq/fe-auth-scram.c（SCRAM 認証実装）
  • **呼び出し先**:
  • 本シンボルは列挙型であり、関数ではないため呼び出し先はありません。

関連データ構造

  • pg_fe_sasl_mech: SASL メカニズムを定義する構造体。exchange 関数ポインタの戻り値として本型を使用します。

失敗モードと境界条件

  • SASL_FAILED が返された場合、認証プロセスは直ちに停止する必要があります。
  • 非同期処理（SASL_ASYNC）を選択する場合、適切な状態管理が行われないと認証がデッドロックする可能性があります。

保守担当者向けメモ

  • 本シンボルは libpq 内部の認証コントラクトの一部です。新しい認証方式を追加する際は、必ずこの列挙型を用いて状態遷移を明示してください。
  • exchange() 関数内でのバイナリデータ処理時は、inputlen を必ず参照してください。
