conditional_stack_peek

定義

  • 種別: function
  • 定義箇所: src/fe_utils/conditional.c:105

目的

スタックの最上位（トップ）にある ifState を取得し、現在の \if ブロックの状態を判定するために使用されます。

シグネチャまたは形状

ifState conditional_stack_peek(ConditionalStack cstack)

入力と出力

  • **入力**: ConditionalStack cstack（操作対象のスタック構造体）
  • **出力**: ifState 型の値。スタックが空の場合は IFSTATE_NONE を返し、それ以外の場合はスタックのトップ要素の if_state を返します。

前提条件と不変条件

  • cstack は有効な ConditionalStack 構造体へのポインタである必要があります。
  • スタックの内部構造（head ポインタ）が整合性を保っていることを前提とします。

アルゴリズムまたは振る舞い

1. conditional_stack_empty(cstack) を呼び出し、スタックが空かどうかを確認します。 2. 空であれば IFSTATE_NONE を即座に返します。 3. 空でなければ、スタックの head が指す要素の if_state メンバーの値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pgbench/pgbench.c
  • src/bin/psql/command.c
  • src/fe_utils/conditional.c（内部の他の関数から参照）
  • **呼び出し先**:
  • conditional_stack_empty

関連データ構造

  • ConditionalStack: \if ブロックのネスト状態を管理するスタック構造体。
  • ifState: ブロックの状態（IFSTATE_TRUE, IFSTATE_FALSE 等）を定義する列挙型。

失敗モードと境界条件

  • **スタックが空の場合**: IFSTATE_NONE を返して安全に終了します。
  • **無効なポインタ**: cstack が NULL の場合の挙動は、conditional_stack_empty の実装に依存します（ソース excerpt には詳細なし）。

保守担当者向けメモ

  • 本関数は状態を読み取るのみであり、スタックの変更（push/pop）は行いません。
  • pgbench や psql のコマンド実行制御において、条件分岐の評価結果を判定する際の基点となります。
