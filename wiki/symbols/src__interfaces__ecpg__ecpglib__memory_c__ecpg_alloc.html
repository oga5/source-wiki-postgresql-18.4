ecpg_alloc

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/memory.c:18

目的

ECPGライブラリ内における標準的なメモリ確保のラッパー関数です。メモリ確保失敗時にECPG固有のエラー処理を統合し、一貫したエラー通知を行います。

シグネチャまたは形状

char *ecpg_alloc(long size, int lineno)

入力と出力

  • **入力**:
  • size (long): 確保するメモリサイズ（バイト単位）。
  • lineno (int): エラー発生時に報告されるソースコードの行番号。
  • **出力**:
  • 成功時: 確保されたメモリ領域へのポインタ（calloc によりゼロ初期化済み）。
  • 失敗時: NULL。

前提条件と不変条件

  • 確保されたメモリは calloc(1L, size) によってゼロで初期化されます。
  • 呼び出し元は、戻り値が NULL である可能性を考慮する必要があります。

アルゴリズムまたは振る舞い

1. calloc(1L, size) を呼び出し、メモリの確保を試みます。 2. 確保に成功した場合、そのポインタを返します。 3. 確保に失敗した場合、ecpg_raise を呼び出し、ECPG_OUT_OF_MEMORY エラーおよび ECPG_SQLSTATE_ECPG_OUT_OF_MEMORY SQLSTATE を記録します。 4. 失敗時は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/connect.c
  • src/interfaces/ecpg/ecpglib/data.c
  • src/interfaces/ecpg/ecpglib/descriptor.c
  • src/interfaces/ecpg/ecpglib/execute.c
  • **呼び出し先**:
  • calloc (標準ライブラリ)
  • ecpg_raise (エラーハンドリング用)

関連データ構造

  • 特になし（汎用的なメモリ確保関数として利用されます）。

失敗モードと境界条件

  • メモリ不足により calloc が NULL を返した場合、エラーハンドラが起動され、関数は NULL を返します。

保守担当者向けメモ

  • 本関数は ecpg_auto_alloc とは異なり、自動解放リストへの登録は行いません。手動で管理されるメモリの確保に使用してください。
  • エラー発生時の行番号 lineno は、ECPGの診断情報として重要であるため、呼び出し元で正確な値を渡す必要があります。
