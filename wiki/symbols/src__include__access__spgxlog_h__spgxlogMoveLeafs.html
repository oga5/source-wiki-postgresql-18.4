spgxlogMoveLeafs

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgxlog.h:64

目的

SP-GiST インデックスにおいて、リーフタプルを移動させる操作を記録するための WAL（Write-Ahead Log）レコード構造体です。リカバリ時にページ間のタプル移動を再実行するために使用されます。

シグネチャまたは形状

typedef struct spgxlogMoveLeafs
{
    uint16      nMoves;
    bool        newPage;
    bool        replaceDead;
    bool        storesNulls;
    OffsetNumber offnumParent;
    uint16      nodeI;
    spgxlogState stateSrc;
    OffsetNumber offsets[FLEXIBLE_ARRAY_MEMBER];
} spgxlogMoveLeafs;

入力と出力

  • **入力**: インデックス操作時に本構造体に値を詰め、XLogInsert を介して WAL に書き出されます。
  • **出力**: リカバリ時、spg_redo 関数が本構造体を読み取り、ページ上のアイテムを更新します。

前提条件と不変条件

  • offsets 配列以降には、可変長のデータ（削除されたタプル番号の配列、挿入されたタプル番号の配列、リーフタプルのリスト）が続きます。
  • replaceDead が true の場合、挿入されるタプル番号とリーフタプルはそれぞれ1つのみとなります。
  • 構造体直後に続くリーフタプルデータはアライメントされていません。

アルゴリズムまたは振る舞い

1. **記録**: インデックスのリーフ移動操作時に、移動数や親ノード情報、状態を構造体に格納します。 2. **再生**: リカバリ時、spg_redo がレコードを読み取り、XLogReadBufferForRedo で対象ページをロックします。その後、構造体内の offsets 配列および後続の可変長データを用いて、ページ上のタプルを適切に移動・配置します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgdoinsert.c（WAL 記録時）
  • **呼び出し先**: src/backend/access/spgist/spgxlog.c（spg_redo による再実行時）、src/backend/access/rmgrdesc/spgdesc.c（ログ出力時）

関連データ構造

  • spgxlogState: リカバリに必要なインデックス状態を保持します。
  • OffsetNumber: ページ内のオフセット管理に使用されます。
  • SizeOfSpgxlogMoveLeafs: 構造体の固定部分のサイズを計算するマクロ。

失敗モードと境界条件

  • 可変長データ（リーフタプル）はアライメントされていないため、直接アクセスせず memcpy 等で安全にコピーする必要があります。
  • replaceDead フラグの状態により、後続データのレイアウト（挿入タプル数やリーフタプル数）が変化するため、解析時にはこのフラグを考慮する必要があります。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を使用しているため、ポインタ演算を行う際は SizeOfSpgxlogMoveLeafs マクロを基準にしてください。
  • リカバリロジックを追う際は、spgxlog.c 内の spg_redo 実装と、各ブロック番号の対応関係を確認してください。
