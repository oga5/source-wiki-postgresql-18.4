mxid_age

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xid.c:132

目的

指定されたマルチトランザクションID（MultiXactId）の経過時間（age）を計算し、現在の最新のマルチトランザクションIDとの差分を返します。

シグネチャまたは形状

Datum mxid_age(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して TransactionId（実質的には MultiXactId として扱われる）を1つ受け取ります。
  • **出力**: int32 型の経過時間。無効なIDの場合は INT_MAX を返します。

前提条件と不変条件

  • 入力されたIDが MultiXactIdIsValid で検証されます。
  • ReadNextMultiXactId() により、現在の最新のマルチトランザクションIDが取得可能である必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_TRANSACTIONID(0) で引数を取得します。 2. ReadNextMultiXactId() を呼び出し、現在の最新のマルチトランザクションIDを取得します。 3. 入力されたIDが有効でない場合、INT_MAX を返します。 4. 有効な場合、現在のID - 入力ID の差分を計算し、int32 にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/scripts/vacuumdb.c（pg_catalog.mxid_age としてSQL経由で呼び出し）
  • **呼び出し先**:
  • ReadNextMultiXactId()（access/multixact.h で定義）
  • MultiXactIdIsValid()（access/multixact.h で定義）

関連データ構造

  • MultiXactId: マルチトランザクションを識別するためのID型。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • 入力IDが無効な場合、計算を行わず INT_MAX を返します。
  • MultiXactId の循環やオーバーフローに関する挙動は、ReadNextMultiXactId および MultiXactId の算術演算の定義に依存します。

保守担当者向けメモ

  • 本関数はSQLインターフェースとして公開されており、主にメンテナンス作業（vacuumdb 等）でテーブルのマルチトランザクションの古さを判定するために使用されます。
  • 内部で ReadNextMultiXactId を呼び出しているため、トランザクション管理のロックや状態に依存します。
