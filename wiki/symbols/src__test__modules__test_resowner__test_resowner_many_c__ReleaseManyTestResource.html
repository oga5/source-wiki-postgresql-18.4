ReleaseManyTestResource

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_resowner/test_resowner_many.c:66

目的

ResourceOwner 機構によって管理されるテスト用リソースの解放コールバックとして機能し、リソースの破棄と解放順序の整合性を検証します。

シグネチャまたは形状

static void ReleaseManyTestResource(Datum res);

入力と出力

  • **入力**: Datum res - 解放対象の ManyTestResource 構造体へのポインタを保持する Datum 型の値。
  • **出力**: なし（void）。

前提条件と不変条件

  • res は有効な ManyTestResource 構造体を指している必要があります。
  • 解放されるリソースの優先順位（mres->kind->desc.release_priority）は、直前に解放されたリソースの優先順位（last_release_priority）以上である必要があります（Assert による検証）。

アルゴリズムまたは振る舞い

1. Datum から ManyTestResource ポインタを復元します。 2. 解放対象のリソース情報を DEBUG1 レベルでログ出力します。 3. last_release_priority と現在のリソースの優先順位を比較し、順序が正しいことを検証します。 4. 内部管理リスト（dlist）から当該ノードを削除します。 5. リソース種別ごとの解放済みカウンタ（nreleased）をインクリメントします。 6. last_release_priority を現在のリソースの優先順位で更新します。 7. pfree を呼び出し、リソースメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: ResourceOwner 機構のコールバックとして登録され、ResourceOwnerRelease 経由で呼び出されます。
  • **呼び出し先**: dlist_delete（リスト操作）、pfree（メモリ解放）、および elog（ログ出力）。

関連データ構造

  • ManyTestResource: 解放対象となるリソースのインスタンス。
  • ManyTestResourceKind: リソースのメタデータ（優先順位や名前）を保持する構造体。

失敗モードと境界条件

  • last_release_priority の不整合が発生した場合、Assert によりプロセスが停止します。
  • res が無効なポインタである場合、メモリ破壊やセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は ResourceOwner の解放順序テストの核となるロジックです。優先順位のロジックを変更する場合は、last_release_priority の更新タイミングとアサーション条件に注意してください。
  • 呼び出し元に関する情報は、ResourceOwner API を介した間接的な呼び出しであるため、ソースコード上では直接的な関数呼び出しとして現れません。
