c_overpaid

定義

  • 種別: function
  • 定義箇所: src/tutorial/funcs.c:129

目的

複合型（行データ）の特定の属性値と指定された閾値を比較し、給与が閾値を超えているかを判定する PostgreSQL ユーザー定義関数です。

シグネチャまたは形状

Datum c_overpaid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • 第1引数: HeapTupleHeader 型（対象の行データ）
  • 第2引数: int32 型（比較対象の給与閾値）
  • **出力**: bool 型（PG_RETURN_BOOL を介して返却）

前提条件と不変条件

  • PostgreSQL の V1 呼び出し規約に従う必要があります。
  • 第1引数として渡される HeapTupleHeader には、"salary" という名前の属性が含まれている必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_HEAPTUPLEHEADER(0) で行データを取得します。 2. PG_GETARG_INT32(1) で閾値を取得します。 3. GetAttributeByName を使用して、行データから "salary" 属性を抽出します。 4. 抽出された値が NULL の場合、false を返します。 5. 抽出された値が NULL でない場合、その値が閾値（limit）より大きいかどうかを判定し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL エンジン（PG_FUNCTION_INFO_V1(c_overpaid) マクロにより登録）。
  • **呼び出し先**:
  • GetAttributeByName（PostgreSQL 実行エンジン）
  • DatumGetInt32（型変換マクロ）
  • PG_GETARG_... / PG_RETURN_...（関数マネージャインターフェース）

関連データ構造

  • HeapTupleHeader: 複合型の内部表現。
  • Datum: PostgreSQL の汎用データ型。

失敗モードと境界条件

  • "salary" 属性が NULL の場合、関数はエラーにならず false を返します。
  • 行データに "salary" 属性が存在しない場合、GetAttributeByName の挙動に依存します（通常はエラーが発生します）。

保守担当者向けメモ

  • 本関数は src/tutorial/funcs.c 内のサンプル実装です。
  • NULL 値の扱いについて、現在は false を返していますが、コメントにある通り PG_RETURN_NULL() を使用して NULL を返す設計に変更することも可能です。
