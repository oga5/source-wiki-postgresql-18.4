ProcGlobalShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/proc.c:138

目的

PostgreSQLの共有メモリ初期化時に、プロセス管理に関連するサブシステムが消費する共有メモリの総サイズを算出します。

シグネチャまたは形状

Size ProcGlobalShmemSize(void)

入力と出力

  • **入力**: なし
  • **出力**: Size 型（共有メモリの合計バイト数）

前提条件と不変条件

  • add_size 関数を用いて、各コンポーネントのサイズをオーバーフローを考慮して加算します。
  • 算出されるサイズには、PROC_HDR 構造体、スピンロック用領域、PGProc 配列、および FastPath ロック用領域が含まれます。

アルゴリズムまたは振る舞い

1. 初期サイズを 0 に設定します。 2. PROC_HDR 構造体のサイズを加算します。 3. スピンロック (slock_t) 用の領域サイズを加算します。 4. PGProcShmemSize() を呼び出し、プロセス配列に必要なサイズを加算します。 5. FastPathLockShmemSize() を呼び出し、高速パスロックに必要なサイズを加算します。 6. 合計サイズを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/ipci.c:119（共有メモリの全体サイズ計算時）
  • **呼び出し先**:
  • PGProcShmemSize()
  • FastPathLockShmemSize()
  • add_size()

関連データ構造

  • PROC_HDR: プロセス管理のグローバルヘッダー。
  • PGProc: 各バックエンドプロセスの状態を保持する構造体。

失敗モードと境界条件

  • add_size 内部でメモリサイズ計算のオーバーフローが発生した場合、PostgreSQLの標準的なエラー処理に従い、プロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数は共有メモリレイアウトの変更時に必ず更新する必要があります。
  • PGProcShmemSize や FastPathLockShmemSize が変更された場合、本関数が返す合計サイズも自動的に追従します。
