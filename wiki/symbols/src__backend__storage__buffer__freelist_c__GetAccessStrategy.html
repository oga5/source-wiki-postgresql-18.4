GetAccessStrategy

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/freelist.c:540

目的

指定されたバッファアクセス戦略タイプ（BufferAccessStrategyType）に基づき、適切なリングバッファサイズを決定し、GetAccessStrategyWithSize を呼び出して戦略オブジェクトを生成・返却する。

シグネチャまたは形状

BufferAccessStrategy GetAccessStrategy(BufferAccessStrategyType btype)

入力と出力

  • **入力**: btype (バッファアクセス戦略の種類: BAS_NORMAL, BAS_BULKREAD, BAS_BULKWRITE, BAS_VACUUM)
  • **出力**: 生成された BufferAccessStrategy オブジェクト。BAS_NORMAL の場合は NULL を返す。

前提条件と不変条件

  • BAS_NORMAL が指定された場合、リングバッファは不要とみなされ NULL が返される。
  • BAS_BULKREAD のリングサイズは、最小値（256KB）を確保しつつ、GetPinLimit() および effective_io_concurrency に基づいて動的に計算される。
  • 未知の btype が渡された場合、elog(ERROR) により処理が中断される。

アルゴリズムまたは振る舞い

1. btype に応じてリングバッファサイズ（ring_size_kb）を決定する。

  • BAS_BULKREAD: 最小256KBから開始し、GetPinLimit() による上限と、effective_io_concurrency に基づく追加分を考慮してサイズを算出する。
  • BAS_BULKWRITE: 16MB（16 * 1024KB）に固定。
  • BAS_VACUUM: 2MB（2048KB）に固定。

    • 2. 決定したサイズを引数として GetAccessStrategyWithSize(btype, ring_size_kb) を呼び出し、その結果を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c
  • contrib/amcheck/ 配下の各ファイル
  • contrib/bloom/blscan.c
  • contrib/pg_visibility/ および contrib/pgstattuple/ 配下の各ファイル
  • **呼び出し先**:
  • GetAccessStrategyWithSize (同一モジュール内)

関連データ構造

  • BufferAccessStrategy: バッファリング戦略を保持する構造体。
  • BufferAccessStrategyType: 戦略の種類を定義する列挙型。

失敗モードと境界条件

  • btype が定義外の値である場合、elog(ERROR) が発生する。
  • BAS_BULKREAD において、計算されたサイズが GetPinLimit() を超える場合は、上限値（ring_max_kb）に切り詰められる。

保守担当者向けメモ

  • BAS_BULKREAD のリングサイズを変更する場合、access/heap/syncscan.c 内の SYNC_SCAN_REPORT_INTERVAL との整合性に注意すること。
  • effective_io_concurrency が 0 の場合、AIO（非同期I/O）関連のサイズ加算は無効化される。
