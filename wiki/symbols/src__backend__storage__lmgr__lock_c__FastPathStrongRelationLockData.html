FastPathStrongRelationLockData

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/lmgr/lock.c:306

目的

高速パス（Fast-Path）ロック機構において、競合が発生しやすく高速パスでの処理を無効化すべき「強力なロック（Strong Relation Lock）」の保持数を管理するために使用されます。

シグネチャまたは形状

typedef struct
{
	slock_t		mutex;
	uint32		count[FAST_PATH_STRONG_LOCK_HASH_PARTITIONS];
} FastPathStrongRelationLockData;

入力と出力

  • 入力: 構造体内の count 配列は、各ハッシュパーティションに対応する強力なロックの数を保持します。
  • 出力: mutex を用いた排他制御により、ロックカウントの整合性が保護されます。

前提条件と不変条件

  • mutex は、count 配列へのアクセスを同期するために使用されます。
  • count 配列の各要素は、対応するハッシュパーティションにおける強力なロックの累積数を表します。

アルゴリズムまたは振る舞い

  • 高速パスロックの最適化において、特定のロックが「強力」であると判断された場合、この構造体を通じて共有メモリ上でカウントが増分されます。
  • このカウントが 0 より大きい場合、該当するリレーションに対する高速パスロックは無効化され、メインのロックテーブルへ移行する処理がトリガーされます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/lmgr/lock.c 内のロック管理ロジック（LockAcquire 等）から参照されます。
  • 呼び出し先: 具体的な関数呼び出しの証拠は提供されていませんが、共有メモリの初期化時に sizeof(FastPathStrongRelationLockData) を用いて領域確保が行われます。

関連データ構造

  • FAST_PATH_STRONG_LOCK_HASH_PARTITIONS: count 配列のサイズを決定する定数。
  • slock_t: スピンロック用の型。

失敗モードと境界条件

  • 共有メモリの初期化に失敗した場合、本構造体へのアクセスは未定義の動作を引き起こします。
  • count のオーバーフローについては、ソースコード上での明示的な保護ロジックは本抜粋からは確認できません。

保守担当者向けメモ

  • 本構造体は、高速パスロックの最適化とデッドロック検出器の整合性を保つための重要な同期ポイントです。
  • FastPathStrongRelationLocks という静的ポインタ変数を介して共有メモリ上のインスタンスが参照されます。
