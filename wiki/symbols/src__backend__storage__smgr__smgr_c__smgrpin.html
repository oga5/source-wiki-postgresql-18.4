smgrpin

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/smgr.c:295

目的

指定された SMgrRelation オブジェクトの参照カウント（pincount）をインクリメントし、トランザクション終了時などの自動的な破棄対象から除外する。

シグネチャまたは形状

void smgrpin(SMgrRelation reln)

入力と出力

  • **入力**: SMgrRelation reln（操作対象のリレーションハンドル）
  • **出力**: なし（副作用として reln の状態を更新）

前提条件と不変条件

  • reln は有効な SMgrRelation オブジェクトを指している必要がある。
  • pincount が 0 の場合、そのオブジェクトは現在リスト（dlist）から外される。

アルゴリズムまたは振る舞い

1. reln->pincount が 0 であるかを確認する。 2. 0 であれば、dlist_delete(&reln->node) を呼び出し、当該オブジェクトをリストから削除する。 3. reln->pincount を 1 増加させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/utils/rel.h:583 (rel->rd_smgr を引数として呼び出し)
  • **呼び出し先**:
  • dlist_delete（リスト操作用マクロ/関数）

関連データ構造

  • SMgrRelation: 物理ファイルへのハンドルを保持する構造体。pincount メンバおよび dlist_node を含む。

失敗モードと境界条件

  • reln が NULL の場合、または不正なメモリ領域を指している場合の動作は未定義。

保守担当者向けメモ

  • 本関数は、リレーションキャッシュが SMgrRelation を保持し続けるために使用される。
  • ピン留めされたオブジェクトは、smgrunpin() が呼び出されて pincount が 0 に戻るまで、AtEOXact_SMgr によるトランザクション終了時の自動破棄から保護される。
