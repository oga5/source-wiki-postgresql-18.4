minmax_qp_callback

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/planagg.c:479

目的

MIN/MAX 集約最適化のためのサブクエリ生成時、query_planner が使用するコールバック関数として、PlannerInfo 構造体のパスキー（pathkeys）を初期化・設定します。

シグネチャまたは形状

static void minmax_qp_callback(PlannerInfo *root, void *extra)

入力と出力

  • **入力**:
  • root: 最適化対象のサブクエリの PlannerInfo 構造体。
  • extra: 使用されていません（void *）。
  • **出力**: なし（void）。root 構造体のフィールドを直接書き換えます。

前提条件と不変条件

  • root->parse が有効な Query 構造体を参照している必要があります。
  • サブクエリの実行計画において、グループ化やウィンドウ関数、重複排除に関連するパスキーを無効化し、ソート句に基づくパスキーを適用する前提で実行されます。

アルゴリズムまたは振る舞い

1. root->group_pathkeys、root->window_pathkeys、root->distinct_pathkeys を NIL にリセットします。 2. make_pathkeys_for_sortclauses を呼び出し、root->parse->sortClause と root->parse->targetList に基づいてソート用のパスキーを生成します。 3. 生成したソートパスキーを root->sort_pathkeys および root->query_pathkeys に割り当てます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planagg.c 内の build_minmax_path 関数から、query_planner の引数として呼び出されます。
  • **呼び出し先**: make_pathkeys_for_sortclauses（PostgreSQL のパスキー生成ユーティリティ）。

関連データ構造

  • PlannerInfo: クエリのプランニング状態を保持する主要な構造体。
  • Query: パースツリーのルートノード。

失敗モードと境界条件

  • root->parse->sortClause が空の場合、make_pathkeys_for_sortclauses は NIL を返し、query_pathkeys も NIL となります。

保守担当者向けメモ

  • 本関数は query_planner のコールバックとして設計されており、サブクエリのパス生成時に特定のソート順序を強制するために使用されます。
  • extra 引数は現在の実装では無視されていますが、将来的な拡張のためにシグネチャの一部として保持されています。
