gbt_bool_fetch

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bool.c:96

目的

GiSTインデックス内のブール値エントリから、インデックス化された値をフェッチ（取得）します。

シグネチャまたは形状

Datum gbt_bool_fetch(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体へのポインタを受け取ります。
  • **出力**: Datum 型として、インデックスから復元されたブール値の表現を返します。

前提条件と不変条件

  • tinfo（gbtree_ninfo 構造体）が、bool 型の比較ロジックやサイズ情報を含めて適切に初期化されている必要があります。
  • GISTENTRY ポインタは有効なインデックスエントリを指している必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から GISTENTRY 構造体を取得します。 2. 共通ユーティリティ関数 gbt_num_fetch を呼び出し、tinfo を渡してインデックスエントリから値を抽出します。 3. 抽出結果を PG_RETURN_POINTER を介して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのGiSTインデックスエンジン（直接的な呼び出し元コードの詳細はソース抜粋に含まれていません）。
  • **呼び出し先**: gbt_num_fetch（btree_utils_num.h で定義された共通ユーティリティ関数）。

関連データ構造

  • GISTENTRY: GiSTインデックスの各エントリを保持する構造体。
  • tinfo: bool 型のインデックス操作を定義する gbtree_ninfo 構造体。

失敗モードと境界条件

  • 無効な GISTENTRY が渡された場合、gbt_num_fetch の内部挙動に依存します。
  • ソースコード上での明示的なエラーハンドリングは記述されていません。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で定義された汎用的な数値フェッチ関数を呼び出すだけの薄いラッパーです。
  • bool 型は内部的に数値として扱われるため、この実装は他の数値型（int4 等）のフェッチ関数と類似しています。
