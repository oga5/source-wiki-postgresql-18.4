json_parse_manifest_incremental_init

定義

  • src/common/parse_manifest.c:128

目的

バックアップマニフェストのインクリメンタル（逐次）解析を開始するための状態管理構造体を初期化します。

シグネチャまたは形状

JsonManifestParseIncrementalState *json_parse_manifest_incremental_init(JsonManifestParseContext *context);

入力と出力

  • **入力**: JsonManifestParseContext *context（解析結果を受け取るコールバック関数群を保持する構造体）。
  • **出力**: JsonManifestParseIncrementalState *（解析状態、JSON字句解析コンテキスト、およびSHA-256計算コンテキストを保持する構造体）。

前提条件と不変条件

  • context が有効であること。
  • SHA-256計算コンテキストの作成および初期化が成功すること。失敗した場合は context->error_cb を通じてエラーが通知されます。

アルゴリズムまたは振る舞い

1. JsonManifestParseIncrementalState および JsonManifestParseState 用のメモリを確保します。 2. JSON解析の初期状態を JM_EXPECT_TOPLEVEL_START に設定します。 3. makeJsonLexContextIncremental を呼び出し、UTF-8エンコーディングでインクリメンタルな字句解析器を初期化します。 4. マニフェスト解析用のコールバック関数群（json_manifest_object_start 等）を状態構造体に登録します。 5. pg_cryptohash_create(PG_SHA256) を使用してチェックサム計算コンテキストを作成し、初期化します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_incremental.c
  • src/bin/pg_combinebackup/load_manifest.c
  • src/bin/pg_verifybackup/pg_verifybackup.c
  • **呼び出し先**:
  • palloc
  • makeJsonLexContextIncremental
  • pg_cryptohash_create
  • pg_cryptohash_init
  • context->error_cb

関連データ構造

  • JsonManifestParseIncrementalState: インクリメンタル解析の全体状態。
  • JsonManifestParseState: 解析のセマンティック状態。
  • JsonManifestParseContext: 呼び出し元が提供するコールバックインターフェース。
  • pg_cryptohash_ctx: SHA-256計算用コンテキスト。

失敗モードと境界条件

  • メモリ不足時: palloc が失敗するか、pg_cryptohash_create が NULL を返すことでエラーコールバックが呼び出されます。
  • 初期化失敗時: pg_cryptohash_init が負の値を返すとエラーコールバックが呼び出されます。

保守担当者向けメモ

  • 本関数はフロントエンド向けに設計されています。
  • インクリメンタル解析の終了時には、対応する json_parse_manifest_incremental_shutdown を呼び出してリソースを解放する必要があります。
