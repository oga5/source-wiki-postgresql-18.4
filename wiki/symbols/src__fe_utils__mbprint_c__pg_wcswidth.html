pg_wcswidth

定義

  • 種別: function
  • 定義箇所: src/fe_utils/mbprint.c:176

目的

指定されたエンコーディングに基づき、マルチバイト文字列が端末上で占有する合計表示幅（カラム数）を算出します。

シグネチャまたは形状

int pg_wcswidth(const char *pwcs, size_t len, int encoding)

入力と出力

  • **入力**:
  • pwcs: 対象文字列へのポインタ。
  • len: 文字列のバイト長。
  • encoding: 文字列のエンコーディングID。
  • **出力**:
  • 戻り値: 文字列の合計表示幅（整数）。

前提条件と不変条件

  • encoding は libpq が認識可能な有効なエンコーディングIDである必要があります。
  • PQmblen および PQdsplen が指定された encoding に対して正しく動作することが前提です。

アルゴリズムまたは振る舞い

1. 文字列の先頭から len バイト分を走査します。 2. PQmblen を用いて現在の文字のバイト長を取得します。 3. 残りのバイト数が文字長より短い場合、不正な文字列とみなして走査を終了します。 4. PQdsplen を用いて現在の文字の表示幅を取得し、正の値であれば合計幅 width に加算します。 5. ポインタを文字長分進め、残りのバイト数を減算して繰り返します。 6. 走査終了後、累積した width を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/describe.c: psql の表示整形処理において利用されています。
  • **呼び出し先**:
  • PQmblen (libpq)
  • PQdsplen (libpq)

関連データ構造

  • 特になし（標準的な C 文字列と整数型を使用）。

失敗モードと境界条件

  • len が文字の境界と一致しない場合、その時点で走査を打ち切り、それまでに計算した幅を返します。
  • PQdsplen が 0 以下の値を返す文字（制御文字など）は、合計幅に加算されません。

保守担当者向けメモ

  • 本関数は「簡易的な表示幅計算関数」として定義されています。
  • pg_wcssize や pg_wcsformat とは異なり、エスケープ処理や改行制御は行いません。
  • libpq のエンコーディング依存関数に依存しているため、ロジック変更時は libpq の仕様変更に注意してください。
