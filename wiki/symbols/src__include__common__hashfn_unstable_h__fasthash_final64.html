fasthash_final64

定義

  • 種別: function
  • 定義箇所: src/include/common/hashfn_unstable.h:324

目的

ハッシュ計算の最終段階において、蓄積されたハッシュ値に「tweak」値を混合し、最終的な64ビットのハッシュ値を生成します。

シグネチャまたは形状

static inline uint64 fasthash_final64(fasthash_state *hs, uint64 tweak)

入力と出力

  • **入力**:
  • hs: ハッシュ計算の状態を保持する fasthash_state 構造体へのポインタ。
  • tweak: 最終計算に使用する64ビットの調整値（入力長など）。
  • **出力**:
  • uint64: 最終的なハッシュ値。

前提条件と不変条件

  • hs は fasthash_init によって初期化され、fasthash_accum 等でデータが蓄積されている必要があります。
  • 本関数は「不安定」なハッシュアルゴリズムの一部であり、プラットフォームやバージョン間で結果が一致する保証はありません。永続化（ディスク保存）には使用しないでください。

アルゴリズムまたは振る舞い

fasthash_state 構造体内の現在のハッシュ値 (hs->hash) と、引数として渡された tweak を fasthash_mix 関数に渡し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: fasthash_mix
  • **呼び出し元**:
  • src/include/common/hashfn_unstable.h 内のコードから直接呼び出されています。
  • fasthash_reduce32 の内部で呼び出されるケースがあります。
  • 内部的なハッシュ計算の完了処理として呼び出されます。

関連データ構造

  • fasthash_state: ハッシュ計算の進行状況を保持する構造体。

失敗モードと境界条件

  • hs が NULL の場合、未定義の動作を引き起こす可能性があります。
  • プラットフォームのエンディアンに依存するため、異なるアーキテクチャ間でのハッシュ値の互換性はありません。

保守担当者向けメモ

  • 本関数はインライン関数として定義されています。
  • SMHasher 等のテストをパスするため、tweak 値（通常は入力長）を適切に渡す設計となっています。
