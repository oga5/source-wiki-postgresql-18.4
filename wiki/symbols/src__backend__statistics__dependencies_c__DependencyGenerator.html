DependencyGenerator

定義

  • 種別: typedef
  • 定義箇所: src/backend/statistics/dependencies.c:65

目的

関数従属性（Functional Dependencies）の計算プロセスにおいて、属性の組み合わせを効率的に生成・管理するための不透明ポインタ型です。

シグネチャまたは形状

typedef DependencyGeneratorData *DependencyGenerator;

入力と出力

  • **入力**: DependencyGenerator_init(int n, int k) を介して、属性数 n と組み合わせサイズ k を受け取り初期化されます。
  • **出力**: DependencyGenerator_next(DependencyGenerator state) を呼び出すことで、次の属性組み合わせ（AttrNumber *）を順次取得します。

前提条件と不変条件

  • DependencyGenerator_init によってメモリが割り当てられ、初期化されている必要があります。
  • DependencyGenerator_free を呼び出すまで、内部状態は有効に保持されます。

アルゴリズムまたは振る舞い

  • 統計構築（ANALYZE）時に、列の組み合わせを再帰的または反復的に生成する状態管理を行います。
  • generate_dependencies_recurse や generate_dependencies 関数内で、属性の全組み合わせを網羅的に生成する際のイテレータとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: statext_dependencies_build がこの型を使用して統計構築プロセスを制御します。
  • **呼び出し先**: 以下の関数がこの型を引数として操作します。
  • DependencyGenerator_init
  • DependencyGenerator_free
  • DependencyGenerator_next
  • generate_dependencies
  • generate_dependencies_recurse

関連データ構造

  • DependencyGeneratorData: このポインタが指し示す実体となる構造体。

失敗モードと境界条件

  • メモリ不足時には DependencyGenerator_init が失敗する可能性があります。
  • 組み合わせの生成が完了した後の DependencyGenerator_next の戻り値については、実装依存（通常は NULL）となります。

保守担当者向けメモ

  • 本シンボルは DependencyGeneratorData へのポインタであり、内部実装の詳細を隠蔽しています。
  • 属性数に対して指数関数的に計算量が増加するため、大規模なテーブルに対する ANALYZE 実行時の負荷に注意してください。
