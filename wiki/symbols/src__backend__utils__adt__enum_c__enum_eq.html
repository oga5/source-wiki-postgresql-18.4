enum_eq

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/enum.c:323

目的

2つの列挙型（ENUM）値が等しいかどうかを判定する。

シグネチャまたは形状

Datum enum_eq(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される2つの Oid 型の値。
  • **出力**: 2つの値が等しい場合は true、そうでない場合は false を示す bool 値（PG_RETURN_BOOL 経由）。

前提条件と不変条件

  • 入力された Oid は、有効な列挙型の値としてシステムカタログ pg_enum に存在するものと想定される。
  • 列挙型の等価性判定は、内部表現である Oid の直接比較によって行われる。

アルゴリズムまたは振る舞い

1. PG_GETARG_OID(0) および PG_GETARG_OID(1) を使用して、比較対象の2つの Oid を取得する。 2. 取得した2つの Oid を比較演算子 == で評価する。 3. 比較結果を PG_RETURN_BOOL を通じて呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLレベルの等価比較演算子（=）から呼び出される。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • Oid: 列挙型の内部表現として使用されるオブジェクト識別子。

失敗モードと境界条件

  • 入力引数が不正な場合、PG_GETARG_OID マクロの仕様に従いエラーが発生する可能性がある。
  • 本関数自体は Oid の比較のみを行うため、列挙型の順序変更や型キャッシュの状態に依存せず、常に一貫した結果を返す。

保守担当者向けメモ

  • 本関数は列挙型の等価性判定を最適化するために、TypeCache を経由せず Oid の直接比較を行っている。
  • enum_ne と同様に、列挙型の比較において最も高速なパスとして機能する。
