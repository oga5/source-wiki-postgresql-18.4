GetVariable

定義

  • 種別: function
  • 定義箇所: src/bin/psql/variables.c:72

目的

VariableSpace から指定された名前の変数の値を検索し、取得します。

シグネチャまたは形状

const char *GetVariable(VariableSpace space, const char *name);

入力と出力

  • **入力**:
  • space: 変数リストを保持する VariableSpace 構造体。
  • name: 検索対象の変数名（文字列）。
  • **出力**:
  • 変数が見つかった場合はその値（const char *）。値が NULL の場合もその値を返します。
  • 変数が見つからない場合、または space が NULL の場合は NULL を返します。

前提条件と不変条件

  • VariableSpace 内の変数は、名前順（strcmp 順）にソートされた単方向連結リストとして保持されている必要があります。
  • この関数はリストを走査し、名前が一致するか、辞書順で名前を超えた時点で探索を終了します。

アルゴリズムまたは振る舞い

1. space が NULL であれば直ちに NULL を返します。 2. リストの先頭から順に current ノードを走査します。 3. current->name と name を strcmp で比較します。

  • 一致（cmp == 0）した場合、その変数の value を返します。
  • cmp > 0 の場合、リストはソート済みであるため、対象の変数は存在しないと判断し、ループを抜けて NULL を返します。

    • 4. リストの末尾まで到達した場合は NULL を返します。

関連データ構造

  • VariableSpace: 変数リストのヘッダー構造体。
  • struct _variable: 変数名と値を保持する連結リストのノード。

失敗モードと境界条件

  • space が NULL の場合、安全に NULL を返します。
  • 変数が存在しない場合、NULL を返します。
  • 変数は存在するが値が NULL（未設定状態）である場合、NULL を返します。

保守担当者向けメモ

  • 変数の値は常に文字列として返されます。数値やブール値として利用する場合は、呼び出し側で ParseVariable* 関数群を使用してください。
  • リストがソートされている前提に依存しているため、変数の追加・削除を行う関数（SetVariable 等）でソート順序が維持されていることを確認してください。
