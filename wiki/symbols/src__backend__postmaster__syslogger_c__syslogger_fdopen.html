syslogger_fdopen

定義

  • **種別**: function
  • **定義箇所**: src/backend/postmaster/syslogger.c:822

目的

ファイル記述子（fd）を標準入出力ライブラリの FILE ストリームに変換し、ログ書き込み用にバッファリング設定を行う。

シグネチャまたは形状

static FILE *syslogger_fdopen(int fd)

入力と出力

  • **入力**: int fd（オープン済みのファイル記述子）
  • **出力**: FILE *（成功時）、または NULL（失敗時）。

前提条件と不変条件

  • 非Windows環境では、fd が -1 でないことが期待される。
  • Windows環境では、fd が 0 でないことが期待される。
  • 戻り値の FILE ストリームは、常に PG_IOLBF（行バッファリング）モードで設定される。

アルゴリズムまたは振る舞い

1. **非Windows環境**:

  • fdopen(fd, "a") を呼び出し、追記モードでストリームを開く。
  • setvbuf を使用して、ストリームを PG_IOLBF（行単位のバッファリング）に設定する。

    • 2. **Windows環境**:

  • _open_osfhandle を使用して、OSのファイルハンドルをCランタイムのファイル記述子に変換する（_O_APPEND | _O_TEXT モード）。
  • 変換後の記述子が有効（> 0）であれば、fdopen(fd, "a") でストリームを開き、同様に行バッファリングを設定する。

    • 3. **共通**: 処理されたストリームを呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/syslogger.c 内の初期化処理（SysLoggerMain の一部と推測される箇所）から、syslogFile、csvlogFile、jsonlogFile の初期化のために呼び出されている。
  • **呼び出し先**: fdopen（標準Cライブラリ）、setvbuf（標準Cライブラリ）、およびWindows環境下では _open_osfhandle（Windows API）。

関連データ構造

  • FILE（標準Cライブラリのストリーム構造体）

失敗モードと境界条件

  • 入力 fd が無効な場合、NULL を返す。
  • Windows環境において _open_osfhandle が失敗した場合、ストリームは開かれず NULL が返される。

保守担当者向けメモ

  • 本関数はログ出力のバッファリング戦略を統一するために使用されている。
  • Windows特有のファイルハンドル変換処理が含まれているため、プラットフォーム固有のファイルI/O挙動を変更する際は注意が必要である。
