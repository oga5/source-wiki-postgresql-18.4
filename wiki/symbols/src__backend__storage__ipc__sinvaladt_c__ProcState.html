ProcState

定義

  • **種別**: struct
  • **定義箇所**: src/backend/storage/ipc/sinvaladt.c:137

目的

共有キャッシュ無効化メッセージの伝播において、各バックエンドプロセスの読み取り進捗状況と状態管理を行うための構造体です。

シグネチャまたは形状

typedef struct ProcState
{
    pid_t        procPid;
    int          nextMsgNum;
    bool         resetState;
    bool         signaled;
    bool         hasMessages;
    bool         sendOnly;
    LocalTransactionId nextLXID;
} ProcState;

入力と出力

本構造体は関数ではなくデータ構造です。各フィールドは共有メモリ上で管理され、SISeg 構造体内の配列として保持されます。

前提条件と不変条件

  • procPid が 0 の場合、そのエントリは非アクティブとみなされます。
  • procPid == 0 または resetState == true の場合、nextMsgNum は無効です。
  • nextLXID はアイドル状態のバックエンドスロットでのみ意味を持ち、アクティブなエントリでは無効です。

アルゴリズムまたは振る舞い

  • **進捗管理**: nextMsgNum により、各バックエンドが次に読み込むべきメッセージを追跡します。
  • **リセット制御**: resetState が真の場合、バックエンドはキャッシュを破棄して再構築する必要があります。
  • **シグナル制御**: signaled は、遅延しているバックエンドに対して PROCSIG_CATCHUP_INTERRUPT が送信済みであることを示します。
  • **送信専用モード**: sendOnly が真の場合、そのプロセスは無効化メッセージの送信のみを行い、受信は行いません（例: リカバリ中のスタートアッププロセス）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/sinvaladt.c 内の SIInsertDataEntries、SIGetDataEntries、SICleanupQueue 等が本構造体のフィールドを読み書きします。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • SISeg: ProcState の配列を保持する共有メモリ上のメイン構造体。
  • SharedInvalidationMessage: sinval.h で定義される、キューイングされるメッセージ本体。

失敗モードと境界条件

  • **バッファ溢れ**: バックエンドの処理が遅れ、共有メモリ上のメッセージキューから古いメッセージが削除されると、resetState が設定され、強制的なキャッシュ再構築が要求されます。
  • **非アクティブ状態**: procPid が 0 のエントリは、プロセスが終了または未割り当てであることを示します。

保守担当者向けメモ

  • 各バックエンドは自身の ProcState フィールドを主に操作しますが、SICleanupQueue などは全プロセスの状態をスキャンします。
  • nextLXID は ProcNumber でインデックス付けされており、ローカルメモリとのコピー時に利用されます。
