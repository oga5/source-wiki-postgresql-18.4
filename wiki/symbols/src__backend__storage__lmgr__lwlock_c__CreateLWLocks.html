CreateLWLocks

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lwlock.c:461

目的

PostgreSQL の起動プロセスにおいて、共有メモリ上に軽量ロック（LWLock）配列を割り当て、初期化し、拡張機能用のトランチ（Tranche）を登録する。

シグネチャまたは形状

void CreateLWLocks(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリの確保とグローバル変数 MainLWLockArray の設定を行う）

前提条件と不変条件

  • IsUnderPostmaster が偽（Postmaster 起動直後の初期化フェーズ）である必要がある。
  • 共有メモリの割り当てが成功すること。

アルゴリズムまたは振る舞い

1. **共有メモリ確保**: IsUnderPostmaster が偽の場合、LWLockShmemSize() で必要なサイズを計算し、ShmemAlloc で共有メモリを確保する。 2. **メモリ配置**: 確保した領域の先頭に動的トランチ割り当て用のカウンタを配置し、その直後に LWLOCK_PADDED_SIZE に基づいてアライメント調整を行った上で MainLWLockArray を配置する。 3. **初期化**: トランチカウンタを LWTRANCHE_FIRST_USER_DEFINED で初期化し、InitializeLWLocks() を呼び出して全 LWLock を初期化する。 4. **トランチ登録**: プロセスごとに NamedLWLockTrancheArray を走査し、各トランチを LWLockRegisterTranche で登録する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:274
  • 呼び出し先: LWLockShmemSize, ShmemAlloc, InitializeLWLocks, LWLockRegisterTranche

関連データ構造

  • MainLWLockArray: 共有メモリ上の LWLock 配列の先頭ポインタ。
  • LWLockPadded: LWLock のパディング付き構造体。
  • NamedLWLockTrancheArray: 登録対象のトランチリクエスト配列。

失敗モードと境界条件

  • 共有メモリが不足している場合、ShmemAlloc がエラーを発生させる可能性がある。
  • IsUnderPostmaster が真の場合、共有メモリの確保と初期化はスキップされ、トランチの登録のみが実行される。

保守担当者向けメモ

  • 本関数は PostgreSQL の共有メモリ初期化シーケンスの一部であるため、呼び出し順序の変更には注意が必要。
  • LWLockCounter の配置位置は MainLWLockArray の直前というメモリレイアウトに依存しているため、構造変更時はオフセット計算を確認すること。
