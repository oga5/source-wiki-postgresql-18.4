IndexClauseSet

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/path/indxpath.c:54

目的

インデックスの各列に対して適合するクエリ条件（IndexClause ノードのリスト）を保持し、インデックススキャンやビットマップスキャンのパス生成プロセスで利用するために使用されます。

シグネチャまたは形状

typedef struct {
    bool        nonempty;
    List       *indexclauses[INDEX_MAX_KEYS];
} IndexClauseSet;

入力と出力

  • **入力**: インデックスの各列に対応する List 型の indexclauses 配列。
  • **出力**: 構造体内の nonempty フラグ（リストが空でない場合に true となる）および各列に格納された条件リスト。

前提条件と不変条件

  • indexclauses 配列のサイズは INDEX_MAX_KEYS に固定されています。
  • nonempty フラグは、保持するリストのいずれかに要素が存在するかを反映している必要があります。

アルゴリズムまたは振る舞い

インデックスパス生成の過程で、クエリの制約条件（RestrictInfo）や結合条件がインデックスの各列に適合するかを判定し、その結果をこの構造体に収集します。収集された条件は、後のパス構築フェーズでインデックススキャンの実行条件として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/indxpath.c 内のパス生成関数群（create_index_paths 等）から、引数として参照・操作されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • List: 条件ノードを保持するために使用。
  • IndexOptInfo: インデックスの情報を保持する構造体。本構造体と組み合わせて使用されます。

失敗モードと境界条件

  • INDEX_MAX_KEYS を超える列を持つインデックスに対しては、配列の範囲外アクセスが発生する可能性があるため、呼び出し側での適切なガードが必要です。

保守担当者向けメモ

  • 本構造体は indxpath.c におけるインデックスマッチングの集約点です。条件の追加ロジックを変更する場合は、nonempty フラグの整合性が維持されているか確認してください。
