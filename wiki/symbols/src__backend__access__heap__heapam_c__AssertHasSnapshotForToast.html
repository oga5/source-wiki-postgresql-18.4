AssertHasSnapshotForToast

定義

  • 種別: static inline function
  • 定義箇所: src/backend/access/heap/heapam.c:222

目的

TOASTテーブルを持つリレーションに対して操作を行う際、適切なスナップショットが登録またはアクティブであることを検証するデバッグ用アサーションを提供します。

シグネチャまたは形状

static inline void AssertHasSnapshotForToast(Relation rel)

入力と出力

  • **入力**: rel (Relation) - 検証対象のリレーション記述子。
  • **出力**: なし。

前提条件と不変条件

  • USE_ASSERT_CHECKING が定義されている環境でのみ有効です。
  • 通常処理モード（IsNormalProcessingMode()）でない場合は、検証をスキップして正常終了します。
  • 対象のリレーションに TOAST テーブルが存在しない場合、検証はスキップされます。

アルゴリズムまたは振る舞い

1. USE_ASSERT_CHECKING が有効な場合のみ実行されます。 2. IsNormalProcessingMode() を確認し、ブートストラップモード等の場合は早期リターンします。 3. rel->rd_rel->reltoastrelid を参照し、TOAST テーブルが存在しない場合は早期リターンします。 4. HaveRegisteredOrActiveSnapshot() を呼び出し、スナップショットが有効でない場合にアサーション失敗を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam.c 内の複数箇所（行 2093, 2372, 2796, 3293 など）から呼び出されています。
  • **呼び出し先**: IsNormalProcessingMode(), OidIsValid(), HaveRegisteredOrActiveSnapshot() を呼び出します。

関連データ構造

  • Relation: リレーションのメタデータを保持する構造体。rd_rel->reltoastrelid を通じて TOAST テーブルの OID を参照します。

失敗モードと境界条件

  • USE_ASSERT_CHECKING が無効なビルドでは、本関数は何も行いません。
  • スナップショットが登録されていない状態で TOAST テーブルを必要とする操作が行われた場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数はデバッグビルド専用のガードレールです。本番環境の動作には影響しません。
  • TOAST 処理に関連するコードパスで、スナップショットの欠如による予期せぬ挙動を防ぐために使用してください。
