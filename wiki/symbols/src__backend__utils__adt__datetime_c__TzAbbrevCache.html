TzAbbrevCache

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/datetime.c:264

目的

タイムゾーンの略称（abbreviation）に関連する情報をキャッシュし、動的または固定のタイムゾーン解決を効率化するために使用されます。

シグネチャまたは形状

typedef struct TzAbbrevCache
{
    char        abbrev[TOKMAXLEN + 1]; /* 常にNUL終端 */
    char        ftype;                 /* TZ, DTZ, または DYNTZ */
    int         offset;                /* 固定オフセットの場合のGMTオフセット */
    pg_tz      *tz;                    /* 可変オフセットの場合の関連ゾーン */
} TzAbbrevCache;

入力と出力

本構造体はデータ保持用であり、関数ではありません。入力値は abbrev（文字列）、ftype（型識別子）、offset（整数）、tz（pg_tzへのポインタ）です。

前提条件と不変条件

  • abbrev フィールドは常に NUL 終端されている必要があります。
  • ftype はタイムゾーンのタイプ（固定か動的か）を識別するために使用されます。

アルゴリズムまたは振る舞い

タイムゾーンの略称を解決する際、この構造体の配列（tzabbrevcache）を参照することで、計算コストの高いタイムゾーン検索をスキップするキャッシュ機構の一部として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/datetime.c 内のタイムゾーン解決ロジック（tzabbrevcache 配列を介して参照）。
  • **呼び出し先**: 情報なし。

関連データ構造

  • tzabbrevcache: src/backend/utils/adt/datetime.c で定義される TzAbbrevCache 型の静的配列。
  • pg_tz: タイムゾーン情報を保持する構造体（tz フィールドで使用）。

失敗モードと境界条件

  • abbrev の長さが TOKMAXLEN を超える場合、バッファオーバーフローのリスクがあるため、格納時に制限が必要です。
  • キャッシュの有効期限や整合性に関する情報は、提供されたソースからは確認できません。

保守担当者向けメモ

  • tzabbrevcache 配列のサイズは MAXDATEFIELDS に依存しています。
  • タイムゾーンの解決は複雑であり、DST（夏時間）の切り替わりを考慮する必要があるため、このキャッシュの更新ロジックを変更する際は DetermineTimeZoneOffsetInternal の仕様を十分に確認してください。
