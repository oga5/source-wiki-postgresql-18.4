FunctionScanPerFuncState

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeFunctionscan.c:35

目的

FunctionScan ノードにおいて、クエリ内の各テーブル関数（RangeFunction）の実行状態および結果セットを個別に管理するための構造体です。

シグネチャまたは形状

typedef struct FunctionScanPerFuncState
{
    SetExprState *setexpr;      /* 評価対象の式状態 */
    TupleDesc    tupdesc;       /* 関数結果型の記述子 */
    int          colcount;      /* 結果カラム数 */
    Tuplestorestate *tstore;    /* 関数結果セットを保持するストア */
    int64        rowcount;      /* 結果セットの行数 (-1: 不明) */
    TupleTableSlot *func_slot;  /* 関数結果用スロット (または NULL) */
} FunctionScanPerFuncState;

入力と出力

  • **入力**: ExecInitFunctionScan により初期化され、関数実行に必要な式状態や型情報が設定されます。
  • **出力**: FunctionNext 等の処理において、tstore に格納されたタプルが読み出され、最終的なスキャン結果として提供されます。

前提条件と不変条件

  • tstore は関数実行結果をキャッシュするために使用されます。
  • rowcount は結果セットの行数を保持し、不明な場合は -1 となります。

アルゴリズムまたは振る舞い

  • **初期化**: ExecInitFunctionScan で FunctionScanState 内の配列として確保されます。
  • **実行**: 初回呼び出し時に ExecMakeTableFunctionResult を通じて関数を実行し、結果を tstore に格納します。
  • **再利用**: 2回目以降のアクセスや再スキャン時には、tstore からタプルを順次読み出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeFunctionscan.c 内の ExecInitFunctionScan（初期化）、FunctionNext（タプル取得）、ExecReScanFunctionScan（再スキャン）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すロジックは含みません。

関連データ構造

  • FunctionScanState: この構造体の配列（funcstates）を保持する上位の実行状態構造体。
  • Tuplestorestate: 関数結果を一時保存するための基盤構造体。
  • SetExprState: 評価対象の式を管理する構造体。

失敗モードと境界条件

  • 関数が結果を返さない場合、tstore は空の状態となります。
  • メモリ不足時、tstore への格納処理が失敗する可能性があります。

保守担当者向けメモ

  • FunctionScanState の funcstates フィールドを通じて配列として管理されています。
  • WITH ORDINALITY 指定の有無に関わらず、後方スキャンをサポートするために tstore を介したキャッシュが必須となっています。
