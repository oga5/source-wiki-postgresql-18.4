index_deform_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/indextuple.c:455

目的

インデックス・タプル（IndexTuple）を分解し、各属性値を Datum 配列および NULL 状態を示すブール配列へ展開します。

シグネチャまたは形状

void index_deform_tuple(IndexTuple tup, TupleDesc tupleDescriptor, Datum *values, bool *isnull)

入力と出力

  • **入力**:
  • tup: 分解対象の IndexTuple。
  • tupleDescriptor: タプルの構造を記述する TupleDesc。
  • values: 抽出された属性値を格納する Datum 配列（呼び出し元で確保）。
  • isnull: 各属性が NULL かどうかを示す bool 配列（呼び出し元で確保）。
  • **出力**: なし（引数の values および isnull 配列を更新）。

前提条件と不変条件

  • tup は有効な IndexTuple 構造体であること。
  • values および isnull 配列は、tupleDescriptor が示す属性数分以上の容量を持つこと。

アルゴリズムまたは振る舞い

1. タプルヘッダー直後の NULL ビットマップ位置を特定します。 2. IndexInfoFindDataOffset を使用して、タプル内のデータ開始位置を計算します。 3. index_deform_tuple_internal を呼び出し、NULL ビットマップとデータ領域から各属性を順次抽出して配列に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pageinspect/gistfuncs.c
  • src/backend/access/common/indextuple.c (内部再帰的利用)
  • src/backend/access/nbtree/nbtinsert.c
  • src/backend/executor/nodeIndexonlyscan.c
  • src/backend/utils/adt/selfuncs.c
  • src/backend/utils/sort/tuplesortvariants.c
  • **呼び出し先**:
  • index_deform_tuple_internal

関連データ構造

  • IndexTuple: インデックス行のメモリ表現。
  • TupleDesc: タプルの属性定義情報。
  • Datum: 属性値の汎用型。

失敗モードと境界条件

  • tupleDescriptor と tup の属性数が一致しない場合、メモリ破壊や不正な値の読み出しが発生する可能性があります。
  • NULL ビットマップの整合性が取れていない場合、isnull 配列に誤った情報が設定されます。

保守担当者向けメモ

  • 本関数は IndexTupleData ヘッダーの直後に NULL ビットマップが存在することを前提としています。
  • 属性の抽出ロジックは index_deform_tuple_internal に集約されているため、分解処理の変更時は同関数を確認してください。
