check_default_text_search_config

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/ts_cache.c:601

目的

GUC変数 default_text_search_config の設定値を検証し、必要に応じて完全修飾名（スキーマ名付き）に正規化します。

シグネチャまたは形状

bool check_default_text_search_config(char **newval, void **extra, GucSource source)

入力と出力

  • **入力**:
  • newval: 設定しようとしている全文検索設定名（文字列へのポインタ）。
  • extra: 未使用。
  • source: 設定のソース（GucSource型）。
  • **出力**:
  • 成功時 true、失敗時 false。
  • newval は、検証が成功した場合、システムカタログから取得した完全修飾名で上書きされます。

前提条件と不変条件

  • トランザクション状態（IsTransactionState()）であり、かつデータベースに接続している（MyDatabaseId != InvalidOid）場合のみ、カタログアクセスによる検証が行われます。
  • 上記条件を満たさない場合、検証は行われず常に true を返します。

アルゴリズムまたは振る舞い

1. **検証**: トランザクション内であれば、stringToQualifiedNameList で名前を解析し、get_ts_config_oid で該当する全文検索設定のOIDを検索します。 2. **エラーハンドリング**: 設定が存在しない場合、source が PGC_S_TEST であれば NOTICE を発行して true を返し、それ以外は false を返します。 3. **正規化**: 設定が存在する場合、SearchSysCache1 を用いて pg_ts_config カタログから情報を取得し、quote_qualified_identifier を使用してスキーマ名を含む完全修飾名に変換します。 4. **メモリ更新**: guc_free で古い値を解放し、guc_strdup で正規化された新しい値を割り当てます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/guc_tables.c におけるGUC設定のフックとして登録されています。
  • **呼び出し先**: stringToQualifiedNameList, get_ts_config_oid, SearchSysCache1, get_namespace_name, guc_free, guc_strdup 等。

関連データ構造

  • pg_ts_config (システムカタログ)
  • GucSource (GUC設定ソース)

失敗モードと境界条件

  • **カタログアクセス不可**: トランザクション外では検証がスキップされます。
  • **設定名不正**: 存在しない設定名が指定された場合、PGC_S_TEST 以外では false を返し、設定変更が拒否されます。
  • **メモリ不足**: guc_strdup が失敗した場合、false を返します。

保守担当者向けメモ

  • 本関数はGUCのフック関数であるため、newval のメモリ管理には palloc ではなく guc_malloc 系関数を使用する必要があります。
  • search_path の変更による影響を避けるため、ここで完全修飾名に変換して保持する仕様になっています。
