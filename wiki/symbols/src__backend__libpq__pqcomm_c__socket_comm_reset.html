socket_comm_reset

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqcomm.c:333

目的

エラーリカバリ処理の一環として、libpq の通信状態をリセットするために使用されます。

シグネチャまたは形状

static void socket_comm_reset(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数を更新）

前提条件と不変条件

  • 呼び出し時に保留中のデータ（pending data）を破棄してはならないという制約があります。

アルゴリズムまたは振る舞い

1. PqCommBusy フラグを false に設定します。 2. 保留中のデータは保持したまま、通信のビジー状態のみを解除します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/libpq/pqcomm.c 内の通信エラーリカバリ処理から参照されています。
  • 呼び出し先: 直接的な関数呼び出しは行いません。
  • 構造体への登録: PqCommMethods 構造体の .comm_reset フィールドに割り当てられています。

関連データ構造

  • PqCommBusy: 通信がビジー状態であるかを示すグローバル変数。

失敗モードと境界条件

  • 本関数自体はエラーを返しません。
  • 保留中のデータが存在する場合でも、それを破棄せずに状態をリセットする設計となっています。

保守担当者向けメモ

  • 本関数は libpq の通信層におけるエラーリカバリの重要な一部です。状態をリセットする際は、バッファ内のデータを不用意にクリアしないよう注意してください。
