xl_heap_multi_insert

定義

  • 種別: struct
  • 定義箇所: src/include/access/heapam_xlog.h:181

目的

ヒープテーブルへの複数タプル同時挿入操作をWAL（Write-Ahead Logging）に記録するためのヘッダー構造体です。

シグネチャまたは形状

typedef struct xl_heap_multi_insert
{
    uint8       flags;
    uint16      ntuples;
    OffsetNumber offsets[FLEXIBLE_ARRAY_MEMBER];
} xl_heap_multi_insert;

入力と出力

  • **入力**: 挿入操作のフラグ（flags）、挿入するタプルの数（ntuples）、および各タプルが配置されたページ内のオフセット番号配列（offsets）。
  • **出力**: WALレコードのメインデータとして、リカバリ時にヒープページを再構築するための情報を提供します。

前提条件と不変条件

  • offsets は FLEXIBLE_ARRAY_MEMBER として定義されており、ntuples の数に応じた可変長データとして扱われます。
  • 本構造体はWALレコードの先頭部分を構成します。

アルゴリズムまたは振る舞い

1. **記録**: heapam.c において、複数タプルの挿入時に本構造体が初期化され、WALレコードの一部として書き込まれます。 2. **再生**: リカバリ時、heapam_xlog.c のREDO処理において XLogRecGetData を通じて本構造体が読み取られます。 3. **適用**: 読み取られた offsets を基に、対象ページ内の該当位置へタプルデータが再配置されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c: WALレコード生成時に使用。
  • **呼び出し先**:
  • src/backend/access/heap/heapam_xlog.c: リカバリ（REDO）処理で参照。
  • src/backend/access/rmgrdesc/heapdesc.c: WAL内容のデコード（ログ出力）時に参照。
  • src/backend/replication/logical/decode.c: 論理デコーディング時に参照。

関連データ構造

  • xl_multi_insert_tuple: 挿入される個々のタプルデータに関連する構造体。
  • OffsetNumber: ページ内のタプル位置を指す型。

失敗モードと境界条件

  • ntuples が0の場合や、offsets 配列のサイズが実際のデータと不整合を起こす場合、WALレコードのデコードまたはREDO処理が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は xl_heap_insert と同様のフラグ値を使用します。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、メモリレイアウトを扱う際はポインタ演算に注意してください。
  • XLOG_HEAP2_ 系操作との関連性については、heapam_xlog.h の全体的な設計方針を参照してください。
