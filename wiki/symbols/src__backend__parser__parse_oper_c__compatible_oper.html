compatible_oper

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_oper.c:449

目的

指定された演算子名と引数型に対し、実行時の型キャストを必要としない（バイナリ互換性がある）演算子定義を検索・特定する。

シグネチャまたは形状

Operator compatible_oper(ParseState *pstate, List *op, Oid arg1, Oid arg2, bool noError, int location)

入力と出力

  • **入力**:
  • pstate: パーサーの状態（エラー報告用）。
  • op: 演算子名のリスト。
  • arg1, arg2: 演算子の左辺および右辺の型（Oid）。
  • noError: 真の場合、見つからない場合にエラーを投げず NULL を返す。
  • location: エラー発生時の構文解析位置。
  • **出力**:
  • 適合する演算子の Operator 構造体。見つからない場合は NULL。

前提条件と不変条件

  • oper() 関数によって、候補となる演算子が既に検索されていること。
  • 戻り値として返される Operator は、IsBinaryCoercible マクロに基づき、入力型から演算子の引数型へバイナリ互換性があるものに限られる。

アルゴリズムまたは振る舞い

1. oper() を呼び出し、候補となる演算子を検索する。 2. 候補が見つかった場合、その演算子の定義（pg_operator）を取得する。 3. IsBinaryCoercible を使用して、入力型 arg1, arg2 が演算子の定義上の引数型 oprleft, oprright とバイナリ互換であるかを確認する。 4. 互換性があればその演算子を返す。 5. 互換性がない場合は、システムキャッシュを解放し、noError が偽であれば ERRCODE_UNDEFINED_FUNCTION エラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_oper.c 内の演算子解決ロジック。
  • compatible_oper_opid（src/backend/parser/parse_oper.c:492）。
  • **呼び出し先**:
  • oper(): 候補検索の実行。
  • GETSTRUCT(): pg_operator 構造体の取得。
  • IsBinaryCoercible(): 型の互換性判定。
  • ReleaseSysCache(): キャッシュの解放。
  • ereport(): エラー報告。
  • op_signature_string(): エラーメッセージ用のシグネチャ生成。

関連データ構造

  • Operator: pg_operator システムカタログのキャッシュエントリ。
  • Form_pg_operator: 演算子定義の構造体。

失敗モードと境界条件

  • 演算子が見つからない場合、またはバイナリ互換性がない場合、noError が真であれば NULL を返す。
  • noError が偽の場合、適合する演算子が存在しない、あるいは実行時の型キャストが必要な場合には ERROR を発生させる。

保守担当者向けメモ

  • 本関数は「実行時の型キャストを許容しない」という厳格な制約を持つ。型キャストが必要なケースを扱う場合は、他の解決関数を検討すること。
  • IsBinaryCoercible の判定ロジックは pg_cast の定義に依存するため、システムカタログの変更時にはキャッシュの整合性に注意が必要である。
