OnConflictSetState

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/execnodes.h:424

目的

INSERT ... ON CONFLICT DO UPDATE 文の実行時において、競合発生時の更新処理（SET 句）の状態を保持するために使用されます。

シグネチャまたは形状

typedef struct OnConflictSetState
{
	NodeTag		type;
	TupleTableSlot *oc_Existing;
	TupleTableSlot *oc_ProjSlot;
	ProjectionInfo *oc_ProjInfo;
	ExprState  *oc_WhereClause;
} OnConflictSetState;

入力と出力

本構造体は実行時に makeNode(OnConflictSetState) によって生成され、ModifyTableState 等の実行ノード内で保持されます。直接的な関数引数としての入出力は定義されていません。

前提条件と不変条件

  • type フィールドには T_OnConflictSetState が設定される必要があります。
  • 実行エンジンが ON CONFLICT DO UPDATE を処理するコンテキストでのみ有効です。

アルゴリズムまたは振る舞い

  • oc_Existing: 競合が発生した既存のターゲットタプルを保持します。
  • oc_ProjSlot: SET 句の評価結果を格納するためのタプルスロットとして機能します。
  • oc_ProjInfo: SET 句の更新値を計算するための射影情報（Projection）を管理します。
  • oc_WhereClause: UPDATE を実行するかどうかを判定する WHERE 句の評価状態を保持します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execPartition.c
  • src/backend/executor/nodeModifyTable.c
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、特定の関数を直接呼び出すものではありません。各フィールド（ProjectionInfo や ExprState）が実行エンジン内の評価関数によって参照されます。

関連データ構造

  • TupleTableSlot: タプルの保持に使用。
  • ProjectionInfo: 更新値の計算に使用。
  • ExprState: WHERE 句の評価に使用。
  • ResultRelInfo: 本構造体をメンバ（ri_onConflict）として保持する可能性があります。

失敗モードと境界条件

  • メモリ不足時、makeNode による割り当てが失敗する可能性があります。
  • oc_WhereClause が NULL の場合、WHERE 句の制約なしとして処理されます。

保守担当者向けメモ

  • 本構造体は実行時専用のノードであり、copyObject や equal などのノードサポート関数には対応していません。
  • ModifyTable 実行ノードのロジック変更時には、本構造体の各フィールドが正しく初期化・解放されているか確認してください。
