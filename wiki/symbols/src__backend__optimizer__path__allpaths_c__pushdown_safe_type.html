pushdown_safe_type

定義

  • 種別: enum
  • 定義箇所: src/backend/optimizer/path/allpaths.c:70

目的

サブクエリに対するクエリ条件（Qual）のプッシュダウンの可否を分類するために使用されます。

シグネチャまたは形状

typedef enum pushdown_safe_type
{
    PUSHDOWN_UNSAFE,                /* プッシュダウン不可 */
    PUSHDOWN_SAFE,                  /* プッシュダウン可能 */
    PUSHDOWN_WINDOWCLAUSE_RUNCOND,  /* 不可だがWindowClauseの実行条件として機能する可能性あり */
} pushdown_safe_type;

入力と出力

  • 本シンボル自体は列挙型であり、関数等の戻り値としてプッシュダウン判定の結果を表現するために使用されます。

前提条件と不変条件

  • 特になし。

アルゴリズムまたは振る舞い

  • PUSHDOWN_UNSAFE: 条件をサブクエリ内に移動させることが論理的に不可能な場合に使用されます。
  • PUSHDOWN_SAFE: 条件をサブクエリ内に移動させても安全であると判断された場合に使用されます。
  • PUSHDOWN_WINDOWCLAUSE_RUNCOND: 一般的なプッシュダウンとしては安全ではないものの、特定のウィンドウ関数実行条件として利用可能な特殊ケースを示します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/path/allpaths.c 内の qual_is_pushdown_safe 関数および関連するプッシュダウン判定ロジック。
  • 呼び出し先: なし（列挙型定義のため）。

関連データ構造

  • pushdown_safety_info: プッシュダウンの可否を管理する構造体。

失敗モードと境界条件

  • 特になし。

保守担当者向けメモ

  • 本列挙型は、サブクエリ最適化における条件プッシュダウンの判定ロジックを拡張する際に、新しい判定状態が必要となった場合に更新してください。
