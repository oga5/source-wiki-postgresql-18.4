CreateIntoRelDestReceiver

定義

  • 種別: function
  • 定義箇所: src/backend/commands/createas.c:439

目的

CREATE TABLE AS (CTAS) コマンドにおいて、クエリ実行結果を新規作成したテーブルへ直接流し込むための DestReceiver オブジェクトを生成・初期化します。

シグネチャまたは形状

DestReceiver *CreateIntoRelDestReceiver(IntoClause *intoClause)

入力と出力

  • **入力**: IntoClause *intoClause（CTAS 文の INTO 句に関する情報）
  • **出力**: DestReceiver *（初期化された DR_intorel 構造体へのポインタ）

前提条件と不変条件

  • palloc0 により DR_intorel 構造体分のメモリがゼロクリアされた状態で確保されます。
  • 返される DestReceiver の mydest フィールドは DestIntoRel に設定されます。
  • receiveSlot, rStartup, rShutdown, rDestroy の各関数ポインタが、intorel_* 系の実装関数に正しく割り当てられます。

アルゴリズムまたは振る舞い

1. DR_intorel 構造体用のメモリを動的に確保します。 2. DestReceiver インターフェースの各メソッド（receiveSlot, rStartup, rShutdown, rDestroy）を、createas.c 内で定義された対応する関数にバインドします。 3. 引数として受け取った intoClause を構造体の into フィールドに格納します。 4. 初期化された構造体を DestReceiver 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/createas.c
  • src/backend/commands/explain.c
  • src/backend/tcop/dest.c
  • **呼び出し先**:
  • palloc0（メモリ確保）
  • intorel_receive, intorel_startup, intorel_shutdown, intorel_destroy（関数ポインタの割り当て）

関連データ構造

  • DR_intorel: 本関数が生成する DestReceiver の具象型。
  • IntoClause: CTAS のターゲットテーブル定義情報。
  • DestReceiver: クエリ結果の出力先を抽象化したインターフェース構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • intoClause が NULL の場合、呼び出し元（src/backend/tcop/dest.c）の文脈に従い動作しますが、後続の intorel_startup 実行時に into フィールドを参照するため、適切な初期化が前提となります。

保守担当者向けメモ

  • 本関数は DestReceiver の初期化のみを担当します。テーブルの物理的な作成やタプルの挿入といった具体的な処理は、割り当てられた関数ポインタ（intorel_*）が実行時に行います。
  • DR_intorel のプライベートフィールドの詳細は intorel_startup 実行時に設定されるため、本関数内では未設定のままとなります。
