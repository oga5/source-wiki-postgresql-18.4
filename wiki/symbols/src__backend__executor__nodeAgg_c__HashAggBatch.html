HashAggBatch

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeAgg.c:351

目的

ハッシュ集約（HashAgg）処理における、ディスクへのスピル（退避）が発生した際のバッチ処理単位を表現する。

シグネチャまたは形状

typedef struct HashAggBatch
{
	int			setno;			/* grouping set */
	int			used_bits;		/* number of bits of hash already used */
	LogicalTape *input_tape;	/* input partition tape */
	int64		input_tuples;	/* number of tuples in this batch */
	double		input_card;		/* estimated group cardinality */
} HashAggBatch;

入力と出力

  • **入力**: LogicalTape（ディスク上のパーティション化されたタプルデータ）を保持する。
  • **出力**: 処理対象のバッチ情報および、後続の集約処理に必要なメタデータ（タプル数やカーディナリティ推定値）を提供する。

前提条件と不変条件

  • ハッシュ集約がメモリ制限（work_mem）を超えた際に生成される。
  • input_tape は、特定のハッシュパーティションに対応する一時ファイルとして管理される。

アルゴリズムまたは振る舞い

1. **生成**: hashagg_batch_new により、特定のグループ化セット（setno）と入力テープを関連付けて初期化される。 2. **読み込み**: hashagg_batch_read を通じて、ディスク上の input_tape からタプルを順次読み込み、ハッシュテーブルの再構築や集約処理を行う。 3. **状態管理**: 処理済みのバッチは、必要に応じて再度スピル処理の対象となる可能性がある。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeAgg.c 内のハッシュ集約ロジック（hashagg_batch_new 等の関数群）。
  • **呼び出し先**: LogicalTape 関連の関数（logtape.h）を介してディスク I/O を実行する。
  • *注: 具体的な呼び出しスタックの全容は、提供されたソース抜粋からは特定できません。*

関連データ構造

  • LogicalTape: スピルされたタプルを保持する一時ファイル構造。
  • AggState: 集約ノードの実行状態を保持する上位構造体。

失敗モードと境界条件

  • ディスク容量不足: 一時ファイルへの書き出し時にエラーが発生する可能性がある。
  • カーディナリティ推定の不正確さ: input_card の推定値が不正確な場合、バッチ分割の効率が低下する可能性がある。

保守担当者向けメモ

  • 本構造体はハッシュ集約のメモリ管理戦略の核心部分である。
  • used_bits は、ハッシュ空間の再分割（リハッシュ）を行う際に、どのビットまでが現在のバッチで利用済みかを追跡するために使用される。
