base_yylex

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parser.c:110

目的

SQL構文解析における字句解析器（Flex）のラッパーとして機能します。LALR(1)文法では解決できない曖昧さを解消するため、トークンの先読み（Lookahead）と特定のキーワードの置換、およびUnicodeエスケープ処理を行います。

シグネチャまたは形状

int base_yylex(YYSTYPE *lvalp, YYLTYPE *llocp, core_yyscan_t yyscanner);

入力と出力

  • **入力**:
  • lvalp: トークンの値を格納する構造体へのポインタ。
  • llocp: トークンの位置情報を格納する構造体へのポインタ。
  • yyscanner: 字句解析器の状態を管理するハンドル。
  • **出力**:
  • 戻り値: 変換後のトークンID（整数）。
  • lvalp, llocp: 呼び出し元で参照可能なトークン情報が更新されます。

前提条件と不変条件

  • yyscanner は scanner_init によって初期化済みであること。
  • トークン置換処理において、core_yylex が一時的に挿入した終端文字（\0）を適切に管理し、エラー報告時に正しい位置を指し示すこと。
  • トランザクション状態に関わらず実行可能であること（テーブルアクセス禁止）。

アルゴリズムまたは振る舞い

1. **先読み確認**: yyextra->have_lookahead が真であれば、保持していた先読みトークンを返し、偽であれば core_yylex を呼び出します。 2. **トークン置換**: FORMAT, NOT, NULLS_P, WITH, WITHOUT などのキーワードに対し、後続のトークンを core_yylex で取得して確認します。条件に合致する場合、FORMAT_LA や NOT_LA などの専用トークンIDに置換します。 3. **Unicodeエスケープ**: UIDENT または USCONST が UESCAPE を伴う場合、さらに後続のトークンを読み込み、str_udeescape を用いてUnicode変換を適用します。 4. **状態復元**: 先読みのために一時的に変更したバッファ状態を元に戻し、次回の呼び出しに備えます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/gram.y（Bison生成コード）から呼び出されます。
  • **呼び出し先**: core_yylex（Flex生成コード）、str_udeescape、scanner_yyerror、check_uescapechar。

関連データ構造

  • base_yy_extra_type: 先読みトークン、位置情報、バッファ状態を保持する構造体。
  • YYSTYPE, YYLTYPE: Bisonが使用するトークン値および位置情報の型。

失敗モードと境界条件

  • UESCAPE の直後に SCONST が続かない場合、scanner_yyerror を呼び出しエラーを報告します。
  • 無効なUnicodeエスケープ文字が指定された場合、エラーとなります。
  • エラー報告時は、*llocp を適切に操作し、SQL文字列内の正しい位置を指し示すよう設計されています。

保守担当者向けメモ

  • 本関数は、SQL文法の制約を回避するための「先読み」を実装しています。複雑なバックトラックを避けるため、置換対象のキーワードは最小限に抑えられています。
  • *llocp の保存と復元は、Flexが内部で保持する位置情報ポインタと整合性を保つために必須です。修正時はエラー報告位置がずれないよう注意してください。
