pgreadlink

定義

  • 種別: function
  • 定義箇所: src/port/dirmod.c:308

目的

Windows 環境において、ジャンクションポイント（ディレクトリのシンボリックリンク代替）が指し示すターゲットパスを読み取り、POSIX 互換の形式に変換して取得する。

シグネチャまたは形状

int pgreadlink(const char *path, char *buf, size_t size)

入力と出力

  • **入力**:
  • path: 読み取り対象のジャンクションポイントへのパス。
  • buf: ターゲットパスを格納するバッファ。
  • size: buf のサイズ。
  • **出力**:
  • 成功時: 読み取ったパスの長さ（バイト数）。
  • 失敗時: -1 を返し、errno を設定する。

前提条件と不変条件

  • Windows 環境（WIN32）でのみ動作する。
  • 指定された path はリパースポイント（ジャンクションポイント）である必要がある。
  • ターゲットパスが \??\ で始まるドライブ絶対パス形式の場合、ユーザーが期待する形式（例: C:\...）へ変換される。

アルゴリズムまたは振る舞い

1. GetFileAttributes で対象がリパースポイントか確認する。 2. CreateFile を FILE_FLAG_OPEN_REPARSE_POINT フラグ付きで呼び出し、ハンドルを取得する。 3. DeviceIoControl に FSCTL_GET_REPARSE_POINT を渡し、ジャンクションポイントのメタデータを REPARSE_JUNCTION_DATA_BUFFER に取得する。 4. WideCharToMultiByte を使用して、取得したワイド文字列をマルチバイト文字列に変換する。 5. パスが \??\ プレフィックスで始まる場合、それを除去してパスを正規化する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は提供されていない。src/include/port.h および src/include/port/win32_port.h にて readlink としてマクロ定義されている。
  • **呼び出し先**: GetFileAttributes, CreateFile, DeviceIoControl, FormatMessage, LocalFree, CloseHandle, WideCharToMultiByte, _dosmaperr, ereport (または fprintf)。

関連データ構造

  • REPARSE_JUNCTION_DATA_BUFFER: ジャンクションポイントのメタデータを保持するために使用される構造体。

失敗モードと境界条件

  • 対象パスが存在しない、またはリパースポイントではない場合は EINVAL を返す。
  • DeviceIoControl が失敗した場合、Windows エラーコードを errno に変換し、エラーメッセージを出力して -1 を返す。
  • 変換後のパスがバッファサイズを超える場合や、変換処理自体が失敗した場合は EINVAL を返す。

保守担当者向けメモ

  • 本関数は pgsymlink と対になる処理を行う。pgsymlink で適用されたパス変換を逆転させるロジックが含まれているため、パス形式の変更時には両方の関数を確認すること。
  • \??\ プレフィックスの除去ロジックは、PGDATA 外部のパスに対しては汎用的な変換を行わない安全策をとっている。
