PLy_global_args_pop

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_exec.c:640

目的

PL/Python 関数の再帰呼び出し時に、プロシージャのグローバル辞書から退避されていた引数状態を復元し、呼び出しスタックを解放する。

シグネチャまたは形状

static void PLy_global_args_pop(PLyProcedure *proc)

入力と出力

  • **入力**: proc (PLyProcedure *) - 現在実行中のプロシージャ構造体。
  • **出力**: なし（副作用として proc 内のスタック状態を更新）。

前提条件と不変条件

  • proc->calldepth > 0 であること（アサーションにより検証）。
  • calldepth > 1 の場合、proc->argstack は NULL であってはならない。
  • calldepth == 1 の場合、proc->argstack は NULL である必要がある。

アルゴリズムまたは振る舞い

1. proc->calldepth をデクリメントする。 2. calldepth > 1 の場合:

  • proc->argstack の先頭要素をスタックから取り出す。
  • PLy_function_restore_args を呼び出し、退避されていた引数値をグローバル辞書に復元する。
  • 取り出した PLySavedArgs 構造体を解放する。

    • 3. calldepth == 1 の場合:

  • 最外層の呼び出し終了とみなし、スタックの空状態を確認する。
  • グローバル辞書内の引数エントリの削除は行わない（次回の呼び出し時に上書きされるため）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_exec.c 内の関数（PLy_exec_function 等の実行終了処理）。
  • **呼び出し先**: PLy_function_restore_args。

関連データ構造

  • PLyProcedure: プロシージャの実行状態および引数スタックを保持する。
  • PLySavedArgs: 再帰呼び出し時に退避された引数の値を格納するスタック要素。

失敗モードと境界条件

  • calldepth が 0 の状態で呼び出された場合、アサーション違反により異常終了する。
  • スタックの不整合（calldepth と argstack の不一致）がある場合、アサーションにより検出される。

保守担当者向けメモ

  • 本関数は PLy_global_args_push と対になる操作を行う。
  • Python のグローバル辞書を再利用する設計のため、最外層終了時に辞書をクリーンアップする必要はない。
  • メモリ管理については PLy_function_restore_args が行う引数構造体の解放と整合性を取ること。
