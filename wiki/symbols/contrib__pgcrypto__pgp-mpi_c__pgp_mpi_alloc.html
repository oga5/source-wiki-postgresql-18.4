pgp_mpi_alloc

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-mpi.c:36

目的

OpenPGP 形式の多倍長整数（MPI）を保持するための PGP_MPI 構造体に対し、指定されたビット数に応じたメモリ領域を動的に確保する。

シグネチャまたは形状

int pgp_mpi_alloc(int bits, PGP_MPI **mpi);

入力と出力

  • **入力**:
  • bits: 確保する MPI のビット長（整数）。
  • mpi: 確保された PGP_MPI 構造体へのポインタを格納するアドレス。
  • **出力**:
  • 成功時: 0 を返し、*mpi に確保済み構造体へのポインタを格納する。
  • 失敗時: PXE_PGP_CORRUPT_DATA を返す。

前提条件と不変条件

  • bits は 0 以上かつ 65535 (0xFFFF) 以下である必要がある。
  • 確保されるメモリは PGP_MPI 構造体本体と、データ格納用の (bits + 7) / 8 バイトの領域が連続して配置される。
  • n->data は構造体直後のメモリ領域を指す。

アルゴリズムまたは振る舞い

1. 引数 bits が範囲外（負数または 65535 超）でないか検証する。範囲外の場合はエラーを返す。 2. palloc を使用して、sizeof(PGP_MPI) と計算されたバイト長 len の合計サイズを確保する。 3. 確保したメモリの bits フィールドにビット数を、bytes フィールドにバイト数を設定する。 4. data ポインタを構造体直後のメモリ位置に設定する。 5. 確保したポインタを *mpi に代入し、成功を示す 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-mpi-openssl.c
  • contrib/pgcrypto/pgp-mpi.c（内部の読み込み処理など）
  • **呼び出し先**:
  • palloc（PostgreSQL メモリ管理関数）
  • px_debug（エラーログ出力）

関連データ構造

  • PGP_MPI: MPI データを保持する構造体。ビット数、バイト数、およびデータ本体へのポインタを持つ。

失敗モードと境界条件

  • bits < 0 または bits > 0xFFFF: 不正なビット数として PXE_PGP_CORRUPT_DATA を返し、メモリ確保を行わない。
  • メモリ不足時: palloc が内部的にエラーを発生させる（PostgreSQL の標準的な挙動）。

保守担当者向けメモ

  • 本関数で確保したメモリは、対応する pgp_mpi_free を用いて解放する必要がある。
  • data 領域は構造体と同一のメモリブロック内に確保されるため、個別の解放は不要である。
