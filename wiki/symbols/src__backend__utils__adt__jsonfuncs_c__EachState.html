EachState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonfuncs.c:109

目的

JSONオブジェクトまたは配列の要素を反復処理し、結果をセットとして返す関数（json_each 等）の実行状態を保持するために使用されます。

シグネチャまたは形状

typedef struct EachState
{
	JsonLexContext *lex;
	Tuplestorestate *tuple_store;
	TupleDesc	ret_tdesc;
	MemoryContext tmp_cxt;
	const char *result_start;
	bool		normalize_results;
	bool		next_scalar;
	char	   *normalized_scalar;
} EachState;

入力と出力

  • **入力**: palloc0 を介して初期化され、関数実行中の状態管理に使用されます。
  • **出力**: 処理されたJSON要素が tuple_store に格納され、最終的にSRF（Set-Returning Function）の結果として返却されます。

前提条件と不変条件

  • lex は有効な JsonLexContext を指している必要があります。
  • tuple_store は、結果を蓄積するために tuplestore_begin_heap 等で適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: palloc0 によりメモリが確保され、各フィールドが初期化されます。 2. **反復処理**: JSONの解析コンテキスト（lex）を利用して要素を走査します。 3. **状態管理**: next_scalar や normalized_scalar を用いて、スカラー値の正規化や次の要素への遷移を制御します。 4. **結果格納**: 抽出された要素は tuple_store に書き込まれ、呼び出し元へ返却されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonfuncs.c 内の json_each 関連関数（行番号 2064, 2100, 2122, 2170, 2184 付近）から参照・操作されています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありませんが、内部の lex フィールドを通じて JsonLexContext 関連のAPIが利用されます。

関連データ構造

  • JsonLexContext: JSON解析の基盤となるコンテキスト。
  • Tuplestorestate: SRFの結果を保持するためのストレージ。
  • TupleDesc: 返却されるタプルの型情報。
  • MemoryContext: 処理中のメモリ管理に使用されるコンテキスト。

失敗モードと境界条件

  • メモリ不足時に palloc0 が失敗する可能性があります。
  • 不正なJSON形式が入力された場合、lex を介した解析処理でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は json_each などの集合を返す関数に特化した状態保持用です。
  • normalized_scalar などのフィールドは、JSONの型変換や正規化処理に依存しているため、変更時は jsonfuncs.c 内の解析コールバックとの整合性に注意してください。
