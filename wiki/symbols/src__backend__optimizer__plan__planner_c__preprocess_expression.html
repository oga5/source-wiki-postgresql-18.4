preprocess_expression

定義

  • **種別**: function
  • **定義箇所**: src/backend/optimizer/plan/planner.c:1307

目的

クエリプランニングの初期段階において、式（Node）に対して定数畳み込み、サブリンクの変換、相関変数の置換などの正規化および最適化処理を適用する。

シグネチャまたは形状

Node *preprocess_expression(PlannerInfo *root, Node *expr, int kind)

入力と出力

  • **入力**:
  • root: 現在のクエリレベルの PlannerInfo 構造体。
  • expr: 処理対象の式ツリー。
  • kind: 式の種類を示す識別子（EXPRKIND_QUAL, EXPRKIND_TARGET 等）。
  • **出力**:
  • 変換後の Node ツリー。expr が NULL の場合は NULL を返す。

前提条件と不変条件

  • kind が EXPRKIND_QUAL の場合、処理後に式は「暗黙的AND形式（implicit-AND）」に変換される。
  • eval_const_expressions 実行後、AND/OR のネスト構造はフラット化（N引数形式）され、以降の処理でもこの不変条件を維持する必要がある。

アルゴリズムまたは振る舞い

1. **結合エイリアスの置換**: root->hasJoinRTEs が真かつ特定の kind でない場合、flatten_join_alias_vars を適用。 2. **定数畳み込み**: kind が EXPRKIND_RTFUNC でない場合、eval_const_expressions を適用（デフォルト引数の挿入や名前付き引数の位置指定への変換を含む）。 3. **正規化**: kind が EXPRKIND_QUAL の場合、canonicalize_qual を適用。 4. **ハッシュ化**: kind が EXPRKIND_QUAL または EXPRKIND_TARGET の場合、convert_saop_to_hashed_saop を適用。 5. **サブリンク展開**: root->parse->hasSubLinks が真の場合、SS_process_sublinks を適用。 6. **相関変数置換**: root->query_level > 1 の場合、SS_replace_correlation_vars を適用。 7. **AND形式変換**: kind が EXPRKIND_QUAL の場合、make_ands_implicit を適用。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planner.c 内の各所（ターゲットリスト、HAVING 句、LIMIT 句、WCO 句などの処理時）。
  • **呼び出し先**: flatten_join_alias_vars, eval_const_expressions, canonicalize_qual, convert_saop_to_hashed_saop, SS_process_sublinks, SS_replace_correlation_vars, make_ands_implicit。

関連データ構造

  • PlannerInfo: クエリ全体のプランニング状態を保持。
  • Node: 式ツリーの基底型。
  • Expr: 式ノードの型。

失敗モードと境界条件

  • expr が NULL の場合、即座に NULL を返す。
  • EXPRKIND_QUAL 以外の kind では、canonicalize_qual や make_ands_implicit などの特定の変換がスキップされる。

保守担当者向けメモ

  • SS_replace_correlation_vars の前後には、依存関係を壊さないよう注意が必要である。
  • EXPRKIND_QUAL 以外の式に対しても、定数畳み込みは必須であるため、スキップしないこと。
  • AND/OR のフラット化ルール（AND の下に AND を置かない等）を維持すること。
