relopt_real

定義

  • 種別: struct
  • 定義箇所: src/include/access/reloptions.h:105

目的

リレーションオプション（pg_class.reloptions 等）において、浮動小数点数（double）型のパラメータを定義・管理するための構造体です。

シグネチャまたは形状

typedef struct relopt_real
{
    relopt_gen  gen;
    double      default_val;
    double      min;
    double      max;
} relopt_real;

入力と出力

  • **入力**: オプション定義時に default_val（デフォルト値）、min（最小値）、max（最大値）を指定します。
  • **出力**: 解析処理において、指定された範囲内にあるか検証された double 型の値を提供します。

前提条件と不変条件

  • gen フィールドは relopt_gen 型であり、すべてのリレーションオプション型で共通のヘッダーとして機能します。
  • min および max は、オプション値の妥当性を検証するための境界値として使用されます。

アルゴリズムまたは振る舞い

  • reloptions.c 内の関数によって、ユーザーが指定したオプション文字列を double 型に変換し、min から max の範囲内にあるか検証するために使用されます。
  • init_real_reloption 等の初期化関数を通じて、新しいオプション定義の登録に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/reloptions.c 内の各関数が、オプションの初期化や値の取得時に本構造体を操作します。
  • **呼び出し先**: 具体的な呼び出し先（関数呼び出しの連鎖）に関する情報は、提供されたソース抜粋からは特定できません。

関連データ構造

  • relopt_gen: 本構造体の先頭メンバであり、オプションの共通基盤です。
  • relopt_value: 解析済みのオプション値を保持する際に参照されます。

失敗モードと境界条件

  • min を超える、または max を下回る値が指定された場合、検証処理においてエラーが発生します。
  • 構造体の初期化時に適切な範囲が設定されていない場合、予期せぬ検証結果を招く可能性があります。

保守担当者向けメモ

  • 本構造体は reloptions.c 内の realRelOpts 配列などで静的に定義されるほか、動的なオプション追加時にも使用されます。
  • 浮動小数点数の精度や比較に関する制約については、reloptions.c の実装ロジックを参照してください。
