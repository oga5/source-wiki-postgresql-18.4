SlabAllocInvalidSize

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/slab.c:606

目的

SLAB アロケータにおいて、要求された割り当てサイズがコンテキストの定義サイズと一致しない場合に、エラーを報告して処理を停止する。

シグネチャまたは形状

static void SlabAllocInvalidSize(MemoryContext context, Size size)

入力と出力

  • context: 対象の MemoryContext（内部で SlabContext にキャストされる）。
  • size: 割り当て要求されたサイズ。
  • 出力: なし（elog(ERROR, ...) により制御が戻らない）。

前提条件と不変条件

  • context は SlabContext 型としてキャスト可能であること。
  • 本関数が呼び出される時点で、要求サイズが slab->chunkSize と不一致であることが確定していること。

アルゴリズムまたは振る舞い

1. 引数の context を SlabContext 構造体にキャストする。 2. elog 関数を呼び出し、ERROR レベルでメッセージを出力する。 3. メッセージには、要求されたサイズと、コンテキストが期待する chunkSize の両方が含まれる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/mmgr/slab.c 内の SlabAlloc（サイズ不一致を検出した箇所）。
  • 呼び出し先: elog（PostgreSQL のエラー報告関数）。

関連データ構造

  • SlabContext: SLAB アロケータの管理構造体。chunkSize を保持する。

失敗モードと境界条件

  • 本関数自体がエラー報告を目的としているため、正常なリターンは行わない。
  • context が SlabContext 以外のメモリコンテキストである場合、キャスト後の slab->chunkSize 参照により不正なメモリ読み取りが発生する可能性がある。

保守担当者向けメモ

  • 本関数は SlabAlloc のサイズチェック失敗時にのみ使用される。
  • SlabAlloc は pg_noinline なヘルパー関数へロジックを分離する方針があるため、本関数もその一環として独立している。
