spgxlogPickSplit

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgxlog.h:165

目的

SP-GiST インデックスのページ分割（PickSplit）操作を WAL（Write-Ahead Log）に記録するための構造体です。リカバリ時に、ソースページおよび宛先ページの状態を再構築するために使用されます。

シグネチャまたは形状

typedef struct spgxlogPickSplit
{
    bool        isRootSplit;
    uint16      nDelete;
    uint16      nInsert;
    bool        initSrc;
    bool        initDest;
    OffsetNumber offnumInner;
    bool        initInner;
    bool        storesNulls;
    bool        innerIsParent;
    OffsetNumber offnumParent;
    uint16      nodeI;
    spgxlogState stateSrc;
    OffsetNumber offsets[FLEXIBLE_ARRAY_MEMBER];
} spgxlogPickSplit;

入力と出力

  • **入力**: インデックス操作時に spgdoinsert.c 等で設定される分割パラメータ。
  • **出力**: WAL レコードとしてディスクへ書き出され、リカバリ時に spgxlog.c の Redo ロジックがページ状態を復元するために読み取ります。

前提条件と不変条件

  • 構造体末尾の offsets 配列以降には、可変長のデータ（削除タプル番号、挿入タプル番号、ページセレクタバイト、新規内部タプル、リーフタプルリスト）が順に続きます。
  • 構造体内のタプルデータはアライメントされていません。アクセス時は memcpy 等を用いて安全にコピーする必要があります。

アルゴリズムまたは振る舞い

1. **記録**: インデックスの分割発生時、この構造体に分割元（Src）および分割先（Dest）の初期化フラグ、削除・挿入するタプルの数、親ノードへのダウンリンク情報などを格納します。 2. **再生**: リカバリ時、spg_redo 関数がこの構造体を読み込みます。initSrc や initDest 等のフラグに基づき、対象ページを初期化するかどうかを判断し、offsets 配列および後続の可変長データを用いてページ上のアイテムを更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgdoinsert.c（WAL 記録時）
  • **呼び出し先**: src/backend/access/spgist/spgxlog.c（Redo 処理時）、src/backend/access/rmgrdesc/spgdesc.c（ログ出力時）

関連データ構造

  • spgxlogState: リカバリに必要なインデックスの最小限の状態を保持します。
  • FLEXIBLE_ARRAY_MEMBER: 可変長データ領域の定義に使用されます。

失敗モードと境界条件

  • 可変長データが構造体直後に配置されるため、ポインタ演算の誤りはメモリ破壊を引き起こします。SizeOfSpgxlogPickSplit マクロを使用して固定部分のサイズを正しく計算する必要があります。

保守担当者向けメモ

  • 本構造体は spgxlog.c の Redo ロジックと密接に結合しています。構造体のフィールドを変更する場合は、必ず spgxlog.c 内の対応する復元ロジックを更新してください。
  • 構造体内のタプルデータはアライメントされていないため、直接キャストしてアクセスすることは避けてください。
