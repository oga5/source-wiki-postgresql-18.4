pthread_mutex_unlock

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/pthread-win32.c:59

目的

Windows環境において、libpq が内部的に使用するミューテックスのロックを解放します。

シグネチャまたは形状

int pthread_mutex_unlock(pthread_mutex_t *mp);

入力と出力

  • **入力**: mp (ミューテックス構造体 pthread_mutex_t へのポインタ)
  • **出力**: 成功時は 0 を返します。初期化されていないミューテックスが渡された場合は EINVAL を返します。

前提条件と不変条件

  • mp が指すミューテックスは、事前に pthread_mutex_init または pthread_mutex_lock による遅延初期化を経て、initstate が 1（初期化済み）の状態である必要があります。

アルゴリズムまたは振る舞い

1. 引数 mp の initstate を確認します。 2. initstate が 1 でない場合、エラーとして EINVAL を返します。 3. initstate が 1 の場合、Windows API の LeaveCriticalSection を呼び出し、OSレベルで管理されているクリティカルセクションを解放します。 4. 正常終了として 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/connect.c および src/interfaces/ecpg/ecpglib/misc.c 内の各関数から、排他制御の終了時に呼び出されます。
  • **呼び出し先**: Windows API の LeaveCriticalSection を呼び出します。

関連データ構造

  • pthread_mutex_t: src/port/pthread-win32.h で定義される構造体。initstate（初期化状態）と csection（CRITICAL_SECTION オブジェクト）を保持します。

失敗モードと境界条件

  • initstate が 1 以外の場合、ロックの解放は行われず EINVAL が返されます。これは未初期化のミューテックスに対する操作を防止します。

保守担当者向けメモ

  • 本関数は Windows 固有の CRITICAL_SECTION をラップしたものです。
  • libpq 内部の同期ニーズに特化しており、汎用的な pthread 実装としての完全性は備えていません。
