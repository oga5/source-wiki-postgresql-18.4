pgaio_worker_choose_idle

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_worker.c:164

目的

共有メモリ上の io_worker_control から、現在アイドル状態にあるワーカーを一つ選択し、その割り当てを確保する。

シグネチャまたは形状

static int pgaio_worker_choose_idle(void)

入力と出力

  • **入力**: なし（グローバルな io_worker_control を参照）。
  • **出力**:
  • 成功時: アイドル状態のワーカーインデックス（int 型）。
  • 失敗時: -1（アイドル状態のワーカーが存在しない場合）。

前提条件と不変条件

  • io_worker_control が初期化済みであること。
  • 選択されたワーカーの in_use フラグが真であること（Assert により検証）。

アルゴリズムまたは振る舞い

1. io_worker_control->idle_worker_mask が 0 かどうかを確認し、0 であれば -1 を返す。 2. pg_rightmost_one_pos64 を使用して、マスク内の最も右側（最下位）にあるビット位置（ワーカーインデックス）を特定する。 3. 特定したビットをマスクからクリアし、該当ワーカーを「使用中」状態へ遷移させる。 4. 選択したワーカーのインデックスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/method_worker.c 内のワーカー管理ロジック（行 270 および 493 付近）。
  • **呼び出し先**:
  • pg_rightmost_one_pos64（ビット操作ユーティリティ）。

関連データ構造

  • PgAioWorkerControl: 全ワーカーの稼働状況（idle_worker_mask）を保持する共有メモリ構造体。
  • PgAioWorkerSlot: 各ワーカーの状態（in_use）を保持する配列。

失敗モードと境界条件

  • アイドルワーカーが枯渇している場合、即座に -1 を返し、呼び出し側でフォールバック処理や待機が行われる。

保守担当者向けメモ

  • 本関数は idle_worker_mask を直接操作するため、呼び出し側で適切なロック（AioWorkerSubmissionQueueLock 等）が保持されていることを前提としている可能性がある。実装変更時は排他制御の整合性に注意すること。
