oauth_free

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth-oauth.c:83

目的

OAuth 認証セッションの状態を保持する fe_oauth_state 構造体のメモリを解放し、リソースをクリーンアップします。

シグネチャまたは形状

static void oauth_free(void *opaq);

入力と出力

  • **入力**: void *opaq - fe_oauth_state 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • opaq は fe_oauth_state 型としてキャスト可能である必要があります。
  • 非同期認証の状態（state->async_ctx）は、本関数が呼び出される前に既にクリーンアップされている必要があります（Assert による検証あり）。

アルゴリズムまたは振る舞い

1. 引数 opaq を fe_oauth_state *state にキャストします。 2. state->async_ctx が空であることを確認します。 3. free(state) を呼び出し、構造体メモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は特定されていませんが、fe-auth-oauth.c 内の pg_oauth_mech 構造体（SASL メカニズムの関数テーブル）から参照されることが示唆されています。
  • **呼び出し先**: 標準ライブラリの free()。

関連データ構造

  • fe_oauth_state: 認証セッションの状態を保持する構造体。

失敗モードと境界条件

  • opaq が NULL の場合、free の仕様に従い動作します。
  • state->async_ctx が解放されていない状態で呼び出された場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は SASL 認証メカニズムの終了処理の一部として機能します。メモリリークを防ぐため、fe_oauth_state に新しいメンバを追加した場合は、本関数内での解放処理を検討してください。
