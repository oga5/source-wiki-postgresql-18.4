be_tls_init

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-openssl.c:97

目的

PostgreSQL バックエンドにおける TLS 通信用の OpenSSL コンテキスト（SSL_CTX）を生成し、証明書、秘密鍵、およびプロトコルバージョン等のセキュリティ設定を初期化する。

シグネチャまたは形状

int be_tls_init(bool isServerStart)

入力と出力

  • **入力**: isServerStart (bool) - サーバー起動時の初期化か否かを示すフラグ。エラー発生時のログ出力レベル（FATAL または LOG）の決定に使用される。
  • **出力**: 成功時は 0 を返す。失敗時はエラーを報告し、負の値を返す（呼び出し元へ制御を戻す）。

前提条件と不変条件

  • グローバル変数 ssl_cert_file および ssl_key_file が適切に設定されていること。
  • openssl_tls_init_hook が初期化されていること。
  • 秘密鍵ファイルがパスフレーズで保護されている場合、適切なコールバックが設定されていること。

アルゴリズムまたは振る舞い

1. SSLv23_method() を用いて SSL_CTX を生成する。 2. 非ブロッキング I/O での誤検知を防ぐため SSL_MODE_ACCEPT_MOVING_WRITE_BUFFER を設定する。 3. openssl_tls_init_hook を呼び出し、環境固有の初期化（パスワードコールバック設定等）を行う。 4. サーバー証明書チェーンファイル（ssl_cert_file）を読み込む。 5. 秘密鍵ファイル（ssl_key_file）を読み込み、証明書との整合性を検証する。 6. GUC 設定（ssl_min_protocol_version, ssl_max_protocol_version）に基づき、許可する TLS バージョン範囲を制限する。 7. セッションチケット、セッションキャッシュ、および圧縮を無効化する。 8. TLS 1.2 以前の再ネゴシエーションを無効化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/be-secure.c (関数 be_tls_init を呼び出し)
  • **呼び出し先**: OpenSSL API（SSL_CTX_new, SSL_CTX_use_certificate_chain_file, SSL_CTX_use_PrivateKey_file 等）、および PostgreSQL 内部関数（ereport, SSLerrmessage, check_ssl_key_file_permissions 等）。

関連データ構造

  • SSL_CTX: OpenSSL のセキュア接続設定を保持する構造体。
  • ssl_cert_file, ssl_key_file: サーバー証明書および秘密鍵のパスを保持するグローバル変数。

失敗モードと境界条件

  • SSL_CTX の生成失敗、証明書や秘密鍵の読み込み失敗、または鍵の整合性チェック失敗時に ereport を発行し終了する。
  • 指定されたプロトコルバージョンがビルド環境でサポートされていない場合、または min が max を上回る設定の場合はエラーとなる。

保守担当者向けメモ

  • OpenSSL のエラーキューはスレッドローカルであるため、エラー発生時には ERR_get_error() を使用して詳細を取得すること。
  • dummy_ssl_passwd_cb_called フラグは、パスフレーズが必要な鍵ファイル読み込み時のエラーハンドリングに使用される。
