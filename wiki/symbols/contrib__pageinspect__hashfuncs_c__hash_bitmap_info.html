hash_bitmap_info

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/hashfuncs.c:390

目的

指定されたハッシュインデックスのオーバーフローページが、どのビットマップページによって管理されているか、およびそのビットマップ上の状態（使用中か否か）を特定して返します。

シグネチャまたは形状

Datum hash_bitmap_info(PG_FUNCTION_ARGS)

  • SQL呼び出し例: SELECT * FROM hash_bitmap_info('index_name'::regclass, block_number);

入力と出力

  • **入力**:
  • indexRelid (Oid): 対象ハッシュインデックスのOID。
  • ovflblkno (int64): 調査対象のオーバーフローブロック番号。
  • **出力**: 以下の3カラムを持つ行を返します。
  • bitmapblkno (int64): 管理しているビットマップページのブロック番号。
  • bitmapbit (int32): ビットマップ内のビット位置。
  • bit (bool): ビットの状態（true は使用中、false は未使用）。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要です。
  • 対象リレーションはハッシュインデックスである必要があります。
  • ovflblkno は有効な範囲内である必要があります。
  • メタページ、ビットマップページ、およびプライマリバケットページは対象外であり、指定するとエラーになります。

アルゴリズムまたは振る舞い

1. インデックスリレーションを開き、権限と型を検証します。 2. メタページを読み込み、hashm_nmaps を確認します。 3. 指定された ovflblkno がメタページや既存のビットマップページ自身を指していないか検証します。 4. _hash_ovflblkno_to_bitno を使用して、該当するビットマップページ番号とビット位置を算出します。 5. 該当するビットマップページを読み込み、HashPageGetBitmap でビットマップを取得します。 6. ISSET マクロを用いて対象ビットの状態を確認します。 7. 取得した情報を HeapTuple に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます（PG_FUNCTION_INFO_V1 マクロによる登録あり）。
  • **呼び出し先**:
  • _hash_getbuf, _hash_relbuf: バッファ管理。
  • _hash_ovflblkno_to_bitno: ブロック番号からビット位置への変換。
  • HashPageGetMeta, HashPageGetBitmap: ページ構造の解釈。
  • heap_form_tuple: 結果の構築。

関連データ構造

  • HashMetaPage: ハッシュインデックスのメタデータ。
  • Buffer: ページ読み込み用バッファ。
  • TupleDesc: 結果セットの定義。

失敗モードと境界条件

  • スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE を送出。
  • ハッシュインデックス以外を指定した場合、ERRCODE_WRONG_OBJECT_TYPE を送出。
  • 範囲外のブロック番号を指定した場合、ERRCODE_INVALID_PARAMETER_VALUE を送出。
  • メタページやビットマップページ自身を対象とした場合、ERRCODE_INVALID_PARAMETER_VALUE を送出。

保守担当者向けメモ

  • 本関数は物理ページを直接読み取るため、インデックスの破損状態によっては予期せぬ動作をする可能性があります。
  • _hash_ovflblkno_to_bitno 等の内部関数への依存が強いため、src/backend/access/hash/ 側の変更に注意してください。
