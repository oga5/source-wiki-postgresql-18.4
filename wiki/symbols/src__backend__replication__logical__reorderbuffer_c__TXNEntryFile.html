TXNEntryFile

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/reorderbuffer.c:151

目的

論理デコーディングにおいて、メモリ制限を超えたトランザクションデータをディスクへ退避（スプール）する際、その一時ファイルを管理するための構造体です。

シグネチャまたは形状

typedef struct TXNEntryFile
{
	File		vfd;			/* -1 when the file is closed */
	off_t		curOffset;		/* offset for next write or read. Reset to 0
								 * when vfd is opened. */
} TXNEntryFile;

入力と出力

  • vfd: PostgreSQL の仮想ファイル記述子。ファイルが閉じている場合は -1。
  • curOffset: 次の読み書き操作を行うためのファイルオフセット。ファイルが開かれるたびに 0 にリセットされます。

前提条件と不変条件

  • vfd が -1 の場合、ファイルはクローズ状態であることを示します。
  • curOffset は常に現在のファイルポインタ位置を追跡し、読み書きの整合性を維持します。

アルゴリズムまたは振る舞い

本構造体は、大規模トランザクションのシリアライズ処理において、ディスク上のデータ位置を保持するために使用されます。ファイルが開かれると curOffset が初期化され、以降の読み書き操作に応じて更新されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/logical/reorderbuffer.c 内のトランザクションシリアライズおよび復元処理に関連する関数群。
  • 呼び出し先: storage/fd.h で定義されるファイル操作関数（vfd を介した操作）。
  • 注記: 具体的な関数呼び出し関係については、提供されたソース excerpt には詳細なコールスタックが含まれていないため、これ以上の推論は行いません。

関連データ構造

  • ReorderBufferTXN: トランザクション状態を保持する構造体であり、本構造体を内部で保持してディスク退避を管理します。

失敗モードと境界条件

  • vfd が -1 の状態でファイル操作を試みた場合、エラーが発生する可能性があります。
  • curOffset の管理が不整合を起こすと、ディスク上のデータ読み取り時に破損したデータが復元されるリスクがあります。

保守担当者向けメモ

  • 本構造体は reorderbuffer.c におけるディスクスプール処理の基盤です。デバッグ時には、pg_logical/ 配下に生成される .spill ファイルと、この構造体が保持する curOffset の値が一致しているかを確認してください。
