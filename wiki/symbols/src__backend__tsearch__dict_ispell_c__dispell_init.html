dispell_init

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/dict_ispell.c:30

目的

Ispell形式の辞書ファイル（.dict）、接尾辞ファイル（.aff）、およびストップワードリストを読み込み、DictISpell 構造体を初期化して検索可能な状態にする。

シグネチャまたは形状

Datum dispell_init(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_POINTER(0) を介して List *dictoptions（辞書設定オプションのリスト）を受け取る。
  • **出力**: 初期化された DictISpell 構造体へのポインタを Datum 型として返す。

前提条件と不変条件

  • dictoptions には dictfile と afffile が必須であり、これらが欠如している場合はエラーとなる。
  • 各パラメータ（dictfile, afffile, stopwords）は重複して指定できない。

アルゴリズムまたは振る舞い

1. palloc0 で DictISpell 構造体を確保し、NIStartBuild で構築を開始する。 2. dictoptions を走査し、各パラメータに応じて以下の処理を行う：

  • dictfile: NIImportDictionary で辞書ファイルを読み込む。
  • afffile: NIImportAffixes で接尾辞ファイルを読み込む。
  • stopwords: readstoplist でストップワードリストを読み込む。

    • 3. 必須ファイル（dictfile および afffile）の読み込みが完了していることを確認し、NISortDictionary および NISortAffixes でインデックスを構築する。 4. NIFinishBuild で構築を終了し、初期化済みオブジェクトを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は明示されていない（PostgreSQLの辞書インターフェースから動的に呼び出される）。
  • **呼び出し先**: NIStartBuild, NIImportDictionary, NIImportAffixes, readstoplist, NISortDictionary, NISortAffixes, NIFinishBuild 等の spell.c で定義された低レベル関数。

関連データ構造

  • DictISpell: Ispell辞書の状態を保持するメイン構造体。

失敗モードと境界条件

  • 必須パラメータ（dictfile または afffile）が欠如している場合、ERROR を発生させる。
  • パラメータが重複して指定された場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーとなる。
  • 未知のパラメータが指定された場合、エラーとなる。

保守担当者向けメモ

  • 本関数は spell.c が提供する低レベル API のラッパーである。詳細な解析ロジックについては spell.c を参照すること。
  • 設定ファイルの読み込みには get_tsearch_config_filename を使用しており、ファイルパスの解決はPostgreSQLの標準的な設定ディレクトリに依存する。
