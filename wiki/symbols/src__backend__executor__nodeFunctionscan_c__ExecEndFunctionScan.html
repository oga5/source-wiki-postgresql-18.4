ExecEndFunctionScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeFunctionscan.c:529

目的

FunctionScanState ノードに関連付けられたリソースを解放し、クエリ実行終了時にクリーンアップを行う。

シグネチャまたは形状

void ExecEndFunctionScan(FunctionScanState *node)

入力と出力

  • **入力**: FunctionScanState *node - 実行中の関数スキャンノードの状態構造体。
  • **出力**: なし（副作用としてリソースを解放）。

前提条件と不変条件

  • node は有効な FunctionScanState を指している必要がある。
  • node->funcstates 内の各 tstore は、tuplestore_end を呼び出すか、NULL である必要がある。

アルゴリズムまたは振る舞い

1. ノードが保持する関数数（node->nfuncs）分だけループする。 2. 各関数ごとの状態（FunctionScanPerFuncState）を確認する。 3. tstore（Tuplestore）が割り当てられている場合、tuplestore_end を呼び出してリソースを解放し、ポインタを NULL に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:668（クエリ実行終了処理の一部として呼び出される）。
  • **呼び出し先**: tuplestore_end（src/backend/executor/nodeFunctionscan.c 内で直接呼び出し）。

関連データ構造

  • FunctionScanState: スキャンノードの実行時状態を保持。
  • FunctionScanPerFuncState: 各関数ごとの Tuplestore 状態を管理。

失敗モードと境界条件

  • tstore が既に NULL の場合、ループ内でスキップされるため安全である。

保守担当者向けメモ

  • 本関数は FunctionScan ノードのライフサイクル管理の最終段階を担う。追加のリソース（メモリコンテキストや他の動的割り当て）が FunctionScanState に追加された場合は、本関数内で適切に解放する必要がある。
