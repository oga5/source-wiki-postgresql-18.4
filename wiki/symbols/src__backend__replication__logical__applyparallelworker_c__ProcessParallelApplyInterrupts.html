ProcessParallelApplyInterrupts

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/applyparallelworker.c:711

目的

論理レプリケーションの並列適用ワーカーにおいて、保留中の割り込み要求（シャットダウンや設定再読み込み）を処理する。

シグネチャまたは形状

static void ProcessParallelApplyInterrupts(void)

入力と出力

  • 入力: なし（グローバルな割り込みフラグを参照）
  • 出力: なし（シャットダウン時は proc_exit(0) によりプロセス終了）

前提条件と不変条件

  • 本関数は、並列適用ワーカーの実行コンテキスト内で呼び出されることを前提とする。
  • MySubscription が有効なサブスクリプション情報を保持している必要がある。

アルゴリズムまたは振る舞い

1. CHECK_FOR_INTERRUPTS() を呼び出し、標準的な割り込みチェックを行う。 2. ShutdownRequestPending が真の場合:

  • サブスクリプション名をログ出力し、proc_exit(0) でプロセスを終了する。

    • 3. ConfigReloadPending が真の場合:

  • フラグを偽にリセットし、ProcessConfigFile(PGC_SIGHUP) を呼び出して設定を再読み込みする。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/replication/logical/applyparallelworker.c:712 (内部参照)
  • src/backend/replication/logical/applyparallelworker.c:761 (内部参照)
  • 呼び出し先:
  • CHECK_FOR_INTERRUPTS()
  • ereport()
  • proc_exit()
  • ProcessConfigFile()

関連データ構造

  • MySubscription (グローバル変数、サブスクリプション情報)
  • ShutdownRequestPending (グローバルフラグ)
  • ConfigReloadPending (グローバルフラグ)

失敗モードと境界条件

  • ShutdownRequestPending が設定されている場合、本関数は呼び出し元へ復帰せずプロセスを終了させる。
  • ConfigReloadPending 処理中に設定ファイルの内容が不正な場合、ProcessConfigFile がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は並列適用ワーカーのメインループ内で定期的に呼び出されることを想定している。
  • 割り込み処理の追加が必要な場合は、既存のフラグチェックと同様のパターンに従うこと。
