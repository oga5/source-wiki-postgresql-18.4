acquire_sample_rows

定義

  • 種別: function
  • 定義箇所: src/backend/commands/analyze.c:1198

目的

テーブルから統計情報生成用のランダムな行サンプルを抽出する。

シグネチャまたは形状

int acquire_sample_rows(Relation onerel, int elevel, HeapTuple *rows, int targrows, double *totalrows, double *totaldeadrows)

入力と出力

  • **入力**:
  • onerel: 対象のリレーション。
  • elevel: ログ出力レベル。
  • rows: サンプル行を格納する配列（呼び出し元で確保）。
  • targrows: 取得対象のサンプル行数。
  • **出力**:
  • 戻り値: 実際に取得できたサンプル行数。
  • rows: 抽出された行データ（HeapTuple）。
  • totalrows: 推定される全生存行数。
  • totaldeadrows: 推定される全死滅行数。

前提条件と不変条件

  • targrows > 0 であること。
  • 抽出されたサンプル行は、リレーション内の物理的な位置（itempointer）順にソートされて返される。

アルゴリズムまたは振る舞い

1. **初期化**: BlockSampler を用いてサンプリング対象のブロックを決定し、reservoir_init_selection_state でリザーバーサンプリングの準備を行う。 2. **スキャン**: read_stream_begin_relation を使用してテーブルをスキャンし、各ブロックからタプルを取得する。 3. **サンプリング**: Jeff Vitter のアルゴリズムに基づき、リザーバーサンプリングを実行する。

  • 最初の targrows 個まではそのまま格納。
  • それ以降は、確率的に既存のサンプルと入れ替える。

    • 4. **推定**: スキャンしたブロックの密度に基づき、テーブル全体の生存行数および死滅行数を外挿する。 5. **ソート**: 取得したサンプル行を compare_rows を用いて物理位置順にソートする。 6. **報告**: ereport によりスキャン結果をログ出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/analyze.c 内の統計情報生成プロセスから呼び出される（直接の関数名はソース抜粋からは特定不可）。
  • **呼び出し先**: BlockSampler_Init, reservoir_init_selection_state, table_scan_analyze_next_block, table_scan_analyze_next_tuple, qsort_interruptible 等。

関連データ構造

  • Relation: 対象テーブルのメタデータ。
  • HeapTuple: 抽出された行データ。
  • BlockSamplerData: ブロックサンプリングの状態管理。
  • ReservoirStateData: リザーバーサンプリングの状態管理。
  • TableScanDesc: テーブルスキャンの状態管理。

失敗モードと境界条件

  • リレーションの行数が targrows 未満の場合、取得できた分のみを返し、ソートは行われない。
  • スキャン対象のブロックが 0 の場合、推定行数は 0 となる。

保守担当者向けメモ

  • 本関数は analyze.c の統計生成の核心であり、メモリ管理は anl_context に依存している。
  • サンプリングアルゴリズムの詳細は utils/misc/sampling.c の Vitter の論文を参照のこと。
  • 呼び出し元情報の詳細は、src/backend/commands/analyze.c 内の他の関数との依存関係を追跡する必要がある。
