stringToNodeInternal

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/read.c:48

目的

シリアライズされた文字列からPostgreSQLのノードツリーを再構築するための内部的な制御関数です。グローバルな解析状態を一時的に退避・復元することで、再入可能なノード読み込み処理を実現します。

シグネチャまたは形状

void *stringToNodeInternal(const char *str, bool restore_loc_fields)

入力と出力

  • **入力**:
  • str: 解析対象のシリアライズされた文字列。
  • restore_loc_fields: ノードの場所情報（location fields）を復元するかどうかを制御するフラグ。
  • **出力**:
  • 構築されたノードツリーのルートポインタ（void *）。

前提条件と不変条件

  • pg_strtok_ptr が解析対象の文字列を指すように一時的に書き換えられます。
  • 処理終了時には、pg_strtok_ptr および（DEBUG_NODE_TESTS_ENABLED 有効時）restore_location_fields の状態が呼び出し前の値に復元されます。

アルゴリズムまたは振る舞い

1. 現在のグローバル変数 pg_strtok_ptr を save_strtok に退避します。 2. pg_strtok_ptr を入力文字列 str に設定します。 3. DEBUG_NODE_TESTS_ENABLED が定義されている場合、現在の場所情報フラグを退避し、引数 restore_loc_fields を適用します。 4. nodeRead(NULL, 0) を呼び出し、再帰的にノードツリーを構築します。 5. 退避していた pg_strtok_ptr および場所情報フラグを元の値に戻します。 6. 構築されたノードツリーのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • stringToNode (引数 false で呼び出し)
  • stringToNodeWithLocations (引数 true で呼び出し)
  • **呼び出し先**:
  • nodeRead

関連データ構造

  • Node: 再構築されるツリーの基本構造体。
  • pg_strtok_ptr: 解析位置を保持する静的変数。

失敗モードと境界条件

  • 入力文字列 str が不正なフォーマットである場合、nodeRead 経由でエラーが発生する可能性があります。
  • pg_strtok_ptr を介した再入処理を行いますが、マルチスレッド環境での同時実行は想定されていません。

保守担当者向けメモ

  • 本関数は pg_strtok_ptr を一時的に書き換えることで、readfuncs.c 内の各関数に解析位置ポインタを引数として渡すオーバーヘッドを回避しています。
  • DEBUG_NODE_TESTS_ENABLED が有効な場合、デバッグ用の場所情報復元ロジックが介入するため、テスト環境と本番環境で動作が一部異なります。
