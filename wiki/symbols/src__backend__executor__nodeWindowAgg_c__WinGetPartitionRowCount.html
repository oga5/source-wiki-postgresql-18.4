WinGetPartitionRowCount

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeWindowAgg.c:3313

目的

現在のウィンドウパーティションに含まれる総行数を取得する。

シグネチャまたは形状

int64 WinGetPartitionRowCount(WindowObject winobj)

入力と出力

  • **入力**: WindowObject winobj（ウィンドウ関数の実行コンテキスト）
  • **出力**: int64（パーティション内の総行数）

前提条件と不変条件

  • winobj は WindowObjectIsValid マクロにより有効であることが保証されていなければならない。

アルゴリズムまたは振る舞い

1. winobj を介して spool_tuples を呼び出し、パーティション内の全行を tuplestore に読み込む（引数 -1 は全行の読み込みを指示）。 2. winobj->winstate->spooled_rows に格納されている、読み込み済みの行数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/windowfuncs.c（複数の箇所で集約計算等のために呼び出されている）
  • **呼び出し先**:
  • spool_tuples（nodeWindowAgg.c 内の内部関数）

関連データ構造

  • WindowObject: ウィンドウ関数の実行状態を保持する構造体。
  • WindowAggState: winobj->winstate を通じてアクセスされる、ウィンドウ集約ノードの実行時状態。

失敗モードと境界条件

  • winobj が無効な場合、Assert により異常終了する。
  • パーティションが非常に大きい場合、spool_tuples によるメモリ消費やディスクへのスピルが発生する可能性がある。

保守担当者向けメモ

  • 本関数は windowapi.h で公開されており、ウィンドウ関数を自作する際の標準的なAPIとして提供されている。
  • spool_tuples を呼び出すため、この関数を呼ぶとパーティション全体がメモリまたはディスク上に展開される点に注意すること。
