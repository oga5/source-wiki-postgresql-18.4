init_dsm_registry

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/dsm_registry.c:90

目的

名前付き動的共有メモリ（DSM）レジストリを管理するための共有ハッシュテーブルを初期化、または既存のテーブルへアタッチする。

シグネチャまたは形状

static void init_dsm_registry(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 dsm_registry_table および dsm_registry_dsa を設定）

前提条件と不変条件

  • DSMRegistryCtx が初期化済みであること。
  • DSMRegistryLock を使用して排他制御を行うため、呼び出し時にこのロックが保持されていないこと。

アルゴリズムまたは振る舞い

1. **早期リターン**: dsm_registry_table が既に設定されている場合は即座に終了する。 2. **排他制御**: DSMRegistryLock を LW_EXCLUSIVE モードで取得する。 3. **初期化判定**: DSMRegistryCtx->dshh が DSHASH_HANDLE_INVALID かどうかを確認する。

  • **未初期化の場合**:
  • dsa_create で動的共有メモリ領域（DSA）を作成する。
  • dshash_create で共有ハッシュテーブルを作成する。
  • dsa_pin および dsa_pin_mapping を実行し、DSAを固定する。
  • 作成したDSAとハッシュテーブルのハンドルを DSMRegistryCtx に保存する。
  • **初期化済みの場合**:
  • dsa_attach で既存のDSAにアタッチする。
  • dshash_attach で既存の共有ハッシュテーブルにアタッチする。

    • 4. **終了処理**: LWLockRelease でロックを解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/dsm_registry.c 内の関数（GetNamedDSMSegment 等から呼び出される）。
  • 呼び出し先: LWLockAcquire, LWLockRelease, dsa_create, dsa_attach, dsa_pin, dsa_pin_mapping, dsa_get_handle, dshash_create, dshash_attach, dshash_get_hash_table_handle。

関連データ構造

  • DSMRegistryCtx: 共有メモリ上のレジストリメタデータ。
  • dsm_registry_table: 共有ハッシュテーブル（dshash）。
  • dsm_registry_dsa: 動的共有メモリ領域（dsa）。

失敗モードと境界条件

  • 共有メモリ不足やロック取得失敗時には、PostgreSQLの標準的なエラーハンドリング（elog 等）によりプロセスが中断される可能性がある。

保守担当者向けメモ

  • 本関数は、複数のバックエンドが同時にレジストリへアクセスする際の競合を DSMRegistryLock で解決する重要な同期ポイントである。
  • dsh_params を介してハッシュテーブルのパラメータが制御されている点に留意すること。
