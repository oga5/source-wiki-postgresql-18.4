pqsecure_open_gss

定義

  • **種別**: function
  • **定義箇所**: src/interfaces/libpq/fe-secure-gssapi.c:480

目的

GSSAPI を用いたクライアント・サーバー間の認証ネゴシエーションおよびセキュリティコンテキストの確立を行う。

シグネチャまたは形状

PostgresPollingStatusType pqsecure_open_gss(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn（接続状態を保持する構造体）
  • **出力**: PostgresPollingStatusType（PGRES_POLLING_OK, PGRES_POLLING_READING, PGRES_POLLING_WRITING, PGRES_POLLING_FAILED のいずれか）

前提条件と不変条件

  • 初回呼び出し時に、認証用バッファ（PqGSSSendBuffer, PqGSSRecvBuffer, PqGSSResultBuffer）が PQ_GSS_AUTH_BUFFER_SIZE サイズで確保される。
  • 通信は非同期 I/O を前提としており、部分的な読み書きが発生した場合は呼び出し元が再試行する必要がある。
  • GSSAPI パケットには、ネットワークバイトオーダーの uint32 長さヘッダーが付与される。

アルゴリズムまたは振る舞い

1. **バッファ初期化**: 初回実行時に通信用バッファを確保し、状態変数をリセットする。 2. **送信処理**: PqGSSSendLength にデータがある場合、pqsecure_raw_write を使用して送信を試みる。 3. **受信処理**: conn->gctx が存在する場合、サーバーからのパケットを読み込む。

  • まず uint32 の長さヘッダーを読み取り、パケットサイズを確定させる。
  • パケットがエラー（'E' で始まる）の場合、エラーメッセージを解析して PGRES_POLLING_FAILED を返す。
  • ペイロード全体を受信完了するまで gss_read を繰り返す。

    • 4. **認証ネゴシエーション**: サービス名のロードおよび資格情報の委任設定（gssdelegation）を確認し、gss_init_sec_context を呼び出してセキュリティコンテキストを構築する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-connect.c
  • **呼び出し先**: pqsecure_raw_write, gss_read, pg_GSS_load_servicename, pg_GSS_have_cred_cache, gss_init_sec_context

関連データ構造

  • PGconn: 接続状態および GSSAPI コンテキスト（gctx）を保持。
  • PqGSSSendBuffer, PqGSSRecvBuffer, PqGSSResultBuffer: 通信用の一時バッファ。

失敗モードと境界条件

  • **メモリ不足**: バッファ確保失敗時に PGRES_POLLING_FAILED を返す。
  • **パケットサイズ超過**: サーバーから送信されたパケットが PQ_GSS_AUTH_BUFFER_SIZE を超える場合、エラーとして PGRES_POLLING_FAILED を返す。
  • **I/O 待機**: ソケットがブロックされた場合、EAGAIN 等を検知して PGRES_POLLING_READING または PGRES_POLLING_WRITING を返す。

保守担当者向けメモ

  • 本関数は認証フェーズで使用される。認証完了後、通信用バッファは PQ_GSS_MAX_PACKET_SIZE に切り替えられる設計となっている。
  • パケットの長さヘッダーはネットワークバイトオーダーであるため、pg_ntoh32 による変換が必須である。
