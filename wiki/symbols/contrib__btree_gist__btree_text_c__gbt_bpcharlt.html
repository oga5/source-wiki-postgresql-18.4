gbt_bpcharlt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_text.c:133

目的

bpchar（空白埋め固定長文字）型に対する「より小さい（less than）」比較をGiSTインデックスのコンテキストで実行する。

シグネチャまたは形状

static bool gbt_bpcharlt(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の bpchar 型データへのポインタ。
  • collation: 比較に使用する照合順序（Collation）のOID。
  • flinfo: 関数マネージャ情報（本関数内では直接使用されていない）。
  • **出力**:
  • a が b より小さい場合は true、それ以外は false を返す。

前提条件と不変条件

  • 入力ポインタ a および b は、PostgreSQLの bpchar 型として解釈可能な有効なデータであること。
  • DirectFunctionCall2Coll を介して標準の bpcharlt 関数を呼び出すため、データベースの照合順序設定に依存する。

アルゴリズムまたは振る舞い

1. a と b のポインタを Datum 型に変換する。 2. DirectFunctionCall2Coll を使用して、PostgreSQL標準の bpcharlt 関数を呼び出す。 3. 比較結果を DatumGetBool で bool 型に変換して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_text.c 内の bptinfo 構造体（gbtree_vinfo 型）の初期化時に参照される。
  • **呼び出し先**: bpcharlt（PostgreSQL標準の比較関数）。

関連データ構造

  • gbtree_vinfo: btree_text.c 内で定義される、GiSTインデックス操作に必要な比較関数群を保持する構造体。本関数はこの構造体の一部として登録される。

失敗モードと境界条件

  • a または b が NULL の場合、標準の bpcharlt 関数の振る舞いに従う。
  • 照合順序（collation）が不正な場合、DirectFunctionCall2Coll 経由でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は bpchar 型専用の比較ラッパーである。text 型の比較が必要な場合は gbt_textlt を参照すること。
  • DirectFunctionCall2Coll を使用しているため、インデックスの比較ロジックは標準のB-tree比較ルールと整合性が保たれる。
