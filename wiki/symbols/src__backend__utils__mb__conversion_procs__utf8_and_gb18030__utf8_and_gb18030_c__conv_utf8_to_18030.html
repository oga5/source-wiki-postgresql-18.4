conv_utf8_to_18030

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_gb18030/utf8_and_gb18030.c:157

目的

UTF-8 エンコーディングのコードポイントを、GB18030 エンコーディングの 4 バイト表現へ変換する。主にマッピングテーブルでカバーされない特殊な範囲の文字を処理する。

シグネチャまたは形状

uint32 conv_utf8_to_18030(uint32 code)

入力と出力

  • **入力**: code (UTF-8 バイト列を表現する uint32 値)
  • **出力**: 変換後の GB18030 4 バイト表現（uint32）。変換不能な場合は 0 を返す。

前提条件と不変条件

  • 入力 code は utf8word_to_unicode を介して Unicode コードポイントに変換可能であること。
  • 変換ロジックは GB18030-2000 仕様に基づく線形空間計算に依存している。

アルゴリズムまたは振る舞い

1. utf8word_to_unicode(code) を呼び出し、UTF-8 表現を Unicode コードポイント（ucs）に変換する。 2. 定義された複数の Unicode 範囲（minunicode から maxunicode）に対して、ucs が該当するかを順次判定する。 3. 該当する場合、gb_linear(mincode) で基準となる GB18030 コードを線形空間上の値に変換し、ucs との差分を加算した後、gb_unlinear で再び GB18030 の 4 バイト形式に戻して返す。 4. いずれの範囲にも該当しない場合は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • utf8word_to_unicode
  • gb_linear
  • gb_unlinear
  • **呼び出し元**: ソースコード内の参照（src/backend/utils/mb/conversion_procs/utf8_and_gb18030/utf8_and_gb18030.c:231）が確認されているが、具体的な関数名はソースから直接特定できない。

関連データ構造

  • 特になし（マクロ convutf8 を使用して変換範囲を定義している）。

失敗モードと境界条件

  • 定義された範囲外の Unicode コードポイントが入力された場合、関数は 0 を返す。
  • 範囲内の計算において、GB18030 の 4 バイト構造に適合しない値が算出される可能性については、本関数内では明示的なエラーハンドリングを行っていない。

保守担当者向けメモ

  • 本関数はマッピングテーブル外の文字を扱うための補助的なロジックである。
  • 新たな Unicode 範囲の追加が必要な場合は、convutf8 マクロを使用して範囲と開始コードを指定する。
  • gb_linear および gb_unlinear の実装変更が本関数の出力に直接影響を与えるため、変更時は注意すること。
