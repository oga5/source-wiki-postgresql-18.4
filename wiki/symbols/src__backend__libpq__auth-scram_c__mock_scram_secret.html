mock_scram_secret

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-scram.c:696

目的

ユーザーが存在しない場合でも、認証プロセスを模倣（モック）することで、タイミング攻撃やユーザー列挙攻撃を防ぐためのダミーのSCRAMシークレット情報を生成します。

シグネチャまたは形状

static void mock_scram_secret(const char *username, pg_cryptohash_type *hash_type,
                              int *iterations, int *key_length, char **salt,
                              uint8 *stored_key, uint8 *server_key)

入力と出力

  • **入力**:
  • username: 認証対象のユーザー名。
  • **出力**:
  • hash_type: PG_SHA256 に設定。
  • iterations: SCRAM_SHA_256_DEFAULT_ITERATIONS に設定。
  • key_length: SCRAM_SHA_256_KEY_LEN に設定。
  • salt: ユーザー名とクラスタのnonceから生成されたBase64エンコード済み文字列。
  • stored_key, server_key: ゼロクリアされたバッファ。

前提条件と不変条件

  • 認証が「doomed（失敗確定）」状態であること。
  • scram_mock_salt が成功すること（失敗時は elog(ERROR) で停止）。

アルゴリズムまたは振る舞い

1. ハッシュアルゴリズム、反復回数、鍵長を固定値に設定します。 2. scram_mock_salt を呼び出し、ユーザー名に基づいた決定論的なソルトを生成します。 3. 生成されたソルトをBase64エンコードし、salt 引数に格納します。 4. stored_key および server_key をゼロで埋めます（モック認証では使用されないため）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-scram.c 内の認証フロー（scram_init 等の文脈で、ユーザーが存在しないと判断された際に呼び出される）。
  • **呼び出し先**: scram_mock_salt（ソルト生成）、pg_b64_encode（エンコード）、palloc（メモリ確保）、memset（メモリ初期化）。

関連データ構造

  • pg_cryptohash_type: ハッシュアルゴリズムの指定に使用。
  • SCRAM_SHA_256_KEY_LEN, SCRAM_SHA_256_DEFAULT_ITERATIONS: 定数定義。

失敗モードと境界条件

  • scram_mock_salt が NULL を返した場合、elog(ERROR) によりプロセスが中断されます。
  • Base64エンコードに失敗した場合、elog(ERROR) によりプロセスが中断されます。

保守担当者向けメモ

  • 本関数は攻撃者にユーザーの存在を悟らせないためのセキュリティ対策です。計算コストを意図的に発生させる設計であるため、ロジックの変更時にはタイミング攻撃耐性に影響がないか注意してください。
  • stored_key と server_key はモック認証では使用されませんが、呼び出し元がバッファを正しく確保していることを前提としています。
