TranslateSocketError

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/socket.c:55

目的

Winsock特有のエラーコード（WSAxxx）を、PostgreSQLが期待する標準的なPOSIX準拠の errno に変換し、グローバル変数 errno を更新する。

シグネチャまたは形状

static void TranslateSocketError(void)

入力と出力

  • 入力: WSAGetLastError() が返す直近のWinsockエラーコード。
  • 出力: なし（副作用として errno を更新）。

前提条件と不変条件

  • Windows環境下での実行を前提とする。
  • ソケット操作でエラーが発生した直後に呼び出される必要がある。

アルゴリズムまたは振る舞い

1. WSAGetLastError() を呼び出し、現在のエラーコードを取得する。 2. switch 文を用いて、Winsockエラーコードを対応する標準的な errno 値（EINVAL, ECONNRESET, EWOULDBLOCK 等）にマッピングする。 3. マッピングされた値を errno に代入する。 4. 定義されていない未知のエラーコードが渡された場合、NOTICE レベルでログを出力し、errno を EINVAL に設定する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/port/win32/socket.c 内の各ソケットラッパー関数（pgwin32_socket, pgwin32_bind, pgwin32_listen, pgwin32_accept, pgwin32_connect, pgwin32_recv, pgwin32_send, pgwin32_select 等）。
  • 呼び出し先: WSAGetLastError()（Winsock API）、ereport()（PostgreSQLエラー報告機能）。

関連データ構造

  • errno: 標準Cライブラリのエラー変数。

失敗モードと境界条件

  • 未知のWinsockエラーコードが渡された場合、ereport により通知を行い、errno を EINVAL として処理を継続する。

保守担当者向けメモ

  • 新しいWinsockエラーコードが導入された場合、switch 文にケースを追加する必要がある。
  • WSAGetLastError() はエラーが発生した直後の状態を反映するため、本関数の呼び出しタイミングが重要である。
