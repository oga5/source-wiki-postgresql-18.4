DecodedXLogRecord

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/xlogreader.h:156

目的

WAL（Write-Ahead Log）のバイナリデータからデコードされた、論理的なレコード構造を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct DecodedXLogRecord
{
    size_t      size;
    bool        oversized;
    struct DecodedXLogRecord *next;
    XLogRecPtr  lsn;
    XLogRecPtr  next_lsn;
    XLogRecord  header;
    RepOriginId record_origin;
    TransactionId toplevel_xid;
    char       *main_data;
    uint32      main_data_len;
    int         max_block_id;
    DecodedBkpBlock blocks[FLEXIBLE_ARRAY_MEMBER];
} DecodedXLogRecord;

入力と出力

  • **入力**: WAL リーダーが読み込んだ物理的な WAL バイト列。
  • **出力**: デコードされた各フィールド（LSN、ヘッダー情報、メインデータ、ブロック参照など）。

前提条件と不変条件

  • blocks フィールドは FLEXIBLE_ARRAY_MEMBER を使用しており、構造体末尾に可変長のブロック情報が続くことを前提としています。
  • max_block_id は、使用されているブロックの最大 ID を示し、ブロックがない場合は -1 となります。

アルゴリズムまたは振る舞い

1. WAL リーダーが物理レコードを読み込み、この構造体に展開します。 2. next ポインタにより、デコード済みレコードのキューを形成します。 3. main_data はレコードのメインデータ部分を指し、main_data_len でその長さを管理します。 4. oversized フラグにより、通常のデコードバッファ外に配置されたレコードであるかを識別します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlogreader.c（デコード処理）、src/backend/access/transam/xlog.c、src/backend/access/transam/xlogprefetcher.c 等で参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • XLogRecord: レコードの物理ヘッダー定義。
  • DecodedBkpBlock: レコードに含まれるブロック参照情報。
  • XLogReaderState: 本構造体を管理するリーダーの状態構造体。

失敗モードと境界条件

  • oversized が true の場合、通常のバッファ管理とは異なるメモリ領域で扱われる可能性があります。
  • max_block_id が -1 の場合、blocks 配列へのアクセスは無効です。

保守担当者向けメモ

  • 本構造体はフロントエンド（pg_waldump 等）でも利用されるため、バックエンド固有の依存関係を混入させないよう注意してください。
  • フィールドへの直接アクセスよりも、XLogReader が提供するアクセサマクロの使用が推奨されます。
