pg_atomic_clear_flag_impl

定義

  • 種別: static inline function
  • 定義箇所: src/include/port/atomics/arch-x86.h:149

目的

x86アーキテクチャにおいて、アトミックフラグを安全にクリア（0に設定）する。

シグネチャまたは形状

void pg_atomic_clear_flag_impl(volatile pg_atomic_flag *ptr)

入力と出力

  • **入力**: ptr - クリア対象となる pg_atomic_flag 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • ptr は有効なメモリ領域を指している必要がある。
  • x86のTSO（Total Store Order）メモリモデルに依存している。

アルゴリズムまたは振る舞い

1. コンパイラバリア（__asm__ __volatile__("" ::: "memory")）を発行し、コンパイラによる命令の並べ替えを防止する。 2. ptr->value に 0 を代入する。

  • x86ではTSOモデルにより、この順序でストア操作を行うことでリリースセマンティクスが保証される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/port/atomics.h:209
  • src/include/port/atomics/generic-gcc.h:151
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わない。インラインアセンブラによるコンパイラバリアのみを実行する。

関連データ構造

  • pg_atomic_flag: アトミック操作対象のフラグ構造体。

失敗モードと境界条件

  • ptr が NULL の場合、未定義動作となる。
  • メモリ上の値が volatile 修飾されているため、コンパイラによる最適化で代入が省略されることはない。

保守担当者向けメモ

  • 本実装はx86固有のTSO特性を利用している。他アーキテクチャへの移植時には、適切なメモリフェンス命令（mfence等）が必要になる可能性があるため注意すること。
