array_iter_next

定義

  • 種別: function
  • 定義箇所: src/include/utils/arrayaccess.h:80

目的

配列（フラットな配列または展開された配列）のイテレータから、指定されたインデックスの要素を順次取得し、NULL状態を判定します。

シグネチャまたは形状

static inline Datum array_iter_next(array_iter *it, bool *isnull, int i, int elmlen, bool elmbyval, char elmalign)

入力と出力

  • **入力**:
  • it: 走査状態を保持する array_iter 構造体へのポインタ。
  • i: 取得対象の要素インデックス。
  • elmlen, elmbyval, elmalign: 要素の型情報（サイズ、値渡し可否、アライメント）。
  • **出力**:
  • 戻り値: 要素の Datum 値。
  • isnull: 要素がNULLである場合に true が設定されるフラグ。

前提条件と不変条件

  • array_iter_setup() によってイテレータが初期化済みであること。
  • 呼び出し側は、配列の要素型に対応する正確な elmlen, elmbyval, elmalign を提供しなければなりません。

アルゴリズムまたは振る舞い

1. **展開済み配列の場合**: it->datumptr が存在すれば、そこからインデックス i で直接値を取得し、it->isnullptr があればNULL状態を反映します。 2. **フラットな配列の場合**:

  • ビットマップを確認し、現在の要素がNULLであれば isnull = true とし、値を (Datum) 0 とします。
  • NULLでない場合、fetch_att を用いて現在の it->dataptr から値を取得します。その後、att_addlength_pointer と att_align_nominal を用いて it->dataptr を次の要素位置へ進めます。

    • 3. **状態更新**: ビットマップのマスクをシフトし、8ビット（1バイト）進むごとに it->bitmapptr をインクリメントして it->bitmask をリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/arrayfuncs.c 内の複数の関数（例: array_iter_next を使用して配列要素を走査する処理）。
  • **呼び出し先**: fetch_att, att_addlength_pointer, att_align_nominal（いずれも access/tupmacs.h 等で定義されたマクロ）。

関連データ構造

  • array_iter: 配列走査の状態を保持する構造体。

失敗モードと境界条件

  • インデックス i が配列の範囲外である場合の挙動は、呼び出し側の管理に依存します。
  • フラットな配列において、提供された型情報（elmlen 等）が実際の配列データと不一致の場合、メモリ破壊や不正な値の読み取りが発生する可能性があります。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、パフォーマンスが重視されています。
  • 展開済み配列とフラットな配列のメモリレイアウトの違いを隠蔽する重要な抽象化レイヤーです。変更時は両方のパスに影響がないか確認してください。
