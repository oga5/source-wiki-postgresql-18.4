pgstat_wal_have_pending

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_wal.c:77

目的

現在のプロセスにおけるWAL使用量（pgWalUsage）が、前回の報告時点（prevWalUsage）から更新されているかどうかを判定します。

シグネチャまたは形状

static inline bool pgstat_wal_have_pending(void)

入力と出力

  • **入力**: なし（プロセスローカルな静的変数 pgWalUsage および prevWalUsage を参照）。
  • **出力**: bool 型。変更がある場合は true、ない場合は false を返します。

前提条件と不変条件

  • 本関数は pgstat_wal_flush_cb 内で呼び出されることを前提としています。
  • pgWalUsage および prevWalUsage は WalUsage 構造体として定義されている必要があります。

アルゴリズムまたは振る舞い

1. 現在のWALレコード数 pgWalUsage.wal_records と、前回の報告時の値 prevWalUsage.wal_records を比較します。 2. 両者が一致しない場合、統計情報の更新が必要であると判断し true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: pgstat_wal_flush_cb（src/backend/utils/activity/pgstat_wal.c:104）から呼び出されます。
  • **呼び出し先**: 外部関数は呼び出しません。

関連データ構造

  • WalUsage: executor/instrument.h で定義される、WAL使用量統計を保持する構造体。
  • pgWalUsage: プロセスローカルな現在のWAL使用量。
  • prevWalUsage: プロセスローカルな前回の報告時点のWAL使用量。

失敗モードと境界条件

  • 本関数自体は単純な比較を行うため、失敗モードはありません。
  • pgWalUsage が更新されていない場合、常に false を返します。

保守担当者向けメモ

  • 本関数はパフォーマンス最適化のために static inline として定義されています。
  • WAL統計の更新頻度が高いため、ロック取得前に変更の有無を判定することで、不要な排他制御を回避する役割を担っています。
