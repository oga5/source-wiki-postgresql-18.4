ltree_level

定義

  • 種別: struct
  • 定義箇所: contrib/ltree/ltree.h:33

目的

ltree 構造体における個々のラベル（パスの構成要素）を表現するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	uint16		len;			/* label string length in bytes */
	char		name[FLEXIBLE_ARRAY_MEMBER];
} ltree_level;

入力と出力

  • **入力**: ltree 構造体内のメモリ領域から、LTREE_FIRST マクロ等を通じてポインタとして取得されます。
  • **出力**: len フィールドによりラベル文字列のバイト長を提供し、name フィールドにより実際のラベル文字列（可変長）を提供します。

前提条件と不変条件

  • name フィールドは FLEXIBLE_ARRAY_MEMBER として定義されており、構造体自体のサイズは固定ではありません。
  • メモリ上の配置は MAXALIGN に準拠している必要があります。
  • len は name に格納された文字列のバイト長と一致している必要があります。

アルゴリズムまたは振る舞い

  • ltree 構造体は、この ltree_level が連続して配置された配列としてメモリ上に存在します。
  • 次のレベルへ移動する際は、LEVEL_NEXT マクロを使用して、現在のラベル長とアライメントを考慮したオフセット計算を行います。
  • lquery_op.c において、checkLevel 関数などが lquery_level との照合を行う際の比較対象として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • contrib/ltree/_ltree_gist.c: インデックス構築時のノード走査に使用。
  • contrib/ltree/lquery_op.c: クエリマッチングおよびサブノード比較に使用。
  • **呼び出し先**:
  • 本構造体はデータ構造であり、関数を直接呼び出すことはありません。

関連データ構造

  • ltree: ltree_level を複数保持する親構造体。
  • lquery_level: クエリ実行時に ltree_level と比較される対応構造体。

失敗モードと境界条件

  • **アライメント違反**: sizeof(ltree_level) を直接使用してサイズ計算を行うと、FLEXIBLE_ARRAY_MEMBER の性質上、メモリレイアウトが崩れアライメント違反が発生します。必ず LEVEL_HDRSIZE や MAXALIGN を含むマクロを使用してください。
  • **メモリ破壊**: len フィールドの値を不正に書き換えると、LEVEL_NEXT による後続のレベルへのポインタ計算が誤り、メモリ破壊を引き起こします。

保守担当者向けメモ

  • 本構造体は ltree モジュールの基盤であり、変更はシリアライズ形式（ディスク上のデータ）に直接影響します。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体の拡張やフィールドの追加を行う際は、既存のオフセット計算マクロとの整合性を十分に検証してください。
