pgp_mpi_read

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-mpi.c:79

目的

OpenPGP 形式のストリームから多倍長整数（MPI）を読み込み、メモリ上に PGP_MPI 構造体として構築する。

シグネチャまたは形状

int pgp_mpi_read(PullFilter *src, PGP_MPI **mpi)

入力と出力

  • **入力**:
  • src: 読み込み元の PullFilter ストリーム。
  • mpi: 読み込んだ MPI データを格納するための PGP_MPI 構造体へのポインタのポインタ。
  • **出力**:
  • 成功時は 0 を返し、*mpi に確保された構造体が設定される。
  • 失敗時は負の整数（エラーコード）を返す。

前提条件と不変条件

  • src は有効な PullFilter であること。
  • ストリームの先頭 2 バイトは、MPI のビット長を示すビッグエンディアン形式の整数であること。

アルゴリズムまたは振る舞い

1. ストリームから 2 バイトのヘッダーを読み取り、ビット長 bits を算出する。 2. pgp_mpi_alloc(bits, &n) を呼び出し、必要なメモリを確保する。 3. 確保した構造体の n->bytes に基づき、ストリームからデータ本体を n->data へ読み込む。 4. 読み込みに成功した場合、*mpi に n を代入して終了する。 5. 読み込み中にエラーが発生した場合、確保したメモリを pgp_mpi_free(n) で解放し、エラーコードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-pubdec.c
  • contrib/pgcrypto/pgp-pubkey.c
  • **呼び出し先**:
  • pullf_read_fixed (ストリームからのデータ読み込み)
  • pgp_mpi_alloc (メモリ確保)
  • pgp_mpi_free (エラー時のメモリ解放)

関連データ構造

  • PGP_MPI: MPI データを保持する構造体（ビット数、バイト数、データ本体へのポインタを含む）。
  • PullFilter: データの読み込み元となるストリームインターフェース。

失敗モードと境界条件

  • ストリームの読み込み（ヘッダーまたは本体）が失敗した場合、負の値を返す。
  • pgp_mpi_alloc によるメモリ確保が失敗した場合、そのエラーコードを伝播する。
  • 読み込み失敗時には、確保済みのメモリが適切に解放される。

保守担当者向けメモ

  • 本関数はデータ構造の構築に特化している。数学的な演算が必要な場合は pgp-mpi-openssl.c を参照すること。
  • エラー発生時のメモリリークを防ぐため、pgp_mpi_free による解放処理が組み込まれている。
