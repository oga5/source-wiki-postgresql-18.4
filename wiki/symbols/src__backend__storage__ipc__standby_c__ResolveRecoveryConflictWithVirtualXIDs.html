ResolveRecoveryConflictWithVirtualXIDs

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/standby.c:359

目的

ホットスタンバイ環境において、リカバリプロセスが要求するリソースと競合している特定の仮想トランザクション（VirtualXID）が終了するまで待機、または強制終了させることで競合を解決します。

シグネチャまたは形状

static void ResolveRecoveryConflictWithVirtualXIDs(
    VirtualTransactionId *waitlist,
    ProcSignalReason reason,
    uint32 wait_event_info,
    bool report_waiting
)

入力と出力

  • **入力**:
  • waitlist: 待機対象の VirtualTransactionId 配列（終端は無効なID）。
  • reason: 競合理由（ProcSignalReason）。
  • wait_event_info: 待機イベント情報。
  • report_waiting: 待機状況をログやプロセスステータスに報告するかどうかのフラグ。
  • **出力**: なし（副作用として対象トランザクションの終了待機またはキャンセルを実行）。

前提条件と不変条件

  • waitlist が指す配列は、有効な VirtualTransactionId のリストであること。
  • 呼び出し元は、リカバリプロセスが保持するロックやバッファピンと競合が発生した状態にあること。

アルゴリズムまたは振る舞い

1. **早期終了**: waitlist が空であれば即座にリターンします。 2. **待機ループ**: waitlist 内の各 VirtualTransactionId に対して、VirtualXactLock が取得可能になるまで（＝対象トランザクションが終了するまで）ループします。 3. **タイムアウト判定**: WaitExceedsMaxStandbyDelay を呼び出し、設定された待機時間を超過した場合は CancelVirtualTransaction を実行して対象バックエンドを強制終了させます。 4. **報告**: report_waiting が真の場合、待機時間が deadlock_timeout を超えると LogRecoveryConflict を呼び出し、ログ出力やプロセスステータスの更新を行います。 5. **終了処理**: 全てのトランザクションが終了した後に、必要に応じてログの事後報告やプロセスステータスのリセットを行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/standby.c 内の各競合解決処理（ResolveRecoveryConflictWithSnapshot 等）から呼び出されます。
  • **呼び出し先**: VirtualTransactionIdIsValid, GetCurrentTimestamp, VirtualXactLock, WaitExceedsMaxStandbyDelay, CancelVirtualTransaction, pg_usleep, TimestampDifferenceExceeds, set_ps_display_suffix, LogRecoveryConflict, set_ps_display_remove_suffix。

関連データ構造

  • VirtualTransactionId: 仮想トランザクションを識別する構造体。
  • ProcSignalReason: 競合理由を示す列挙型。

失敗モードと境界条件

  • **タイムアウト**: max_standby_streaming_delay を超えると、対象バックエンドが強制終了されます。
  • **空リスト**: waitlist が無効なIDで始まる場合、何もせず即座に終了します。

保守担当者向けメモ

  • 本関数は、スタンバイ側でプライマリのロック状態を模倣する際の重要な同期ポイントです。
  • 待機中の pg_usleep(5000L) は、システム負荷が高い状況下で応答のないバックエンドへの過度な負荷を避けるための調整値です。
