uuid_lt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/uuid.c:209

目的

2つの uuid 型の値（pg_uuid_t）を比較し、第1引数が第2引数より小さいかどうかを判定する。

シグネチャまたは形状

Datum uuid_lt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される2つの pg_uuid_t 構造体へのポインタ。
  • **出力**: PG_RETURN_BOOL を介して、arg1 < arg2 であれば true、そうでなければ false を返す。

前提条件と不変条件

  • 入力される2つの引数は、有効な pg_uuid_t 構造体を指している必要がある。
  • uuid_internal_cmp が提供する比較ロジックに依存する。

アルゴリズムまたは振る舞い

1. PG_GETARG_UUID_P マクロを使用して、引数から pg_uuid_t へのポインタを取得する。 2. uuid_internal_cmp(arg1, arg2) を呼び出し、16バイトのバイナリ比較結果を取得する。 3. 比較結果が 0 未満（arg1 < arg2）である場合に true を返し、それ以外の場合に false を返す。

呼び出し元と呼び出し先

  • **呼び出し先**: uuid_internal_cmp
  • **呼び出し元**: ソースコード内に直接的な呼び出し元の記述なし。SQL レベルの演算子定義から間接的に呼び出されることが想定される。

関連データ構造

  • pg_uuid_t: UUID を保持する 16 バイトの構造体。

失敗モードと境界条件

  • 引数が NULL の場合、PG_GETARG_UUID_P の挙動に従い、呼び出し元で適切に処理される必要がある。
  • 比較対象の UUID が同一である場合、uuid_internal_cmp は 0 を返し、本関数は false を返す。

保守担当者向けメモ

  • 本関数は uuid_internal_cmp のラッパーである。比較ロジックの変更が必要な場合は uuid_internal_cmp を参照すること。
  • B-tree インデックスの比較演算子として使用されるため、変更時はソート順序の整合性に注意が必要である。
