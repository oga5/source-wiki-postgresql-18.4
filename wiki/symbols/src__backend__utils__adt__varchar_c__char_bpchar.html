char_bpchar

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varchar.c:352

目的

単一の文字（char 型）を、PostgreSQL の固定長文字列型である bpchar（char(n)）形式のデータに変換します。

シグネチャまたは形状

Datum char_bpchar(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CHAR(0) を介して単一の文字（char）を受け取ります。
  • **出力**: BpChar * 型の Datum を返します。これは varlena 構造体としてメモリ確保された文字列データです。

前提条件と不変条件

  • palloc を使用して VARHDRSZ + 1 バイトのメモリを確保します。
  • 戻り値は SET_VARSIZE マクロにより、ヘッダーサイズと文字データ（1バイト）の合計サイズで初期化されます。

アルゴリズムまたは振る舞い

1. 引数から文字 c を取得します。 2. palloc を用いて、可変長ヘッダー（VARHDRSZ）と文字データ（1バイト）を格納するためのメモリを確保します。 3. SET_VARSIZE を使用して、確保したメモリ領域のサイズを適切に設定します。 4. VARDATA マクロで取得したデータ領域の先頭に、入力された文字 c を書き込みます。 5. PG_RETURN_BPCHAR_P マクロを介して、構築した BpChar データを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内のコメントに言及がありますが、具体的な呼び出し箇所は提供された情報からは特定できません。
  • **呼び出し先**: palloc（メモリ確保）、SET_VARSIZE、VARDATA、PG_RETURN_BPCHAR_P などの PostgreSQL 内部マクロを呼び出します。

関連データ構造

  • BpChar: char(n) 型の内部表現。
  • Datum: PostgreSQL の汎用データ型表現。
  • varlena: 可変長データ型の基底構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • 入力された文字がマルチバイト文字である場合、本関数は単一バイトとして処理するため、意図しない挙動となる可能性があります（char 型の仕様に依存）。

保守担当者向けメモ

  • 本関数は char 型から bpchar 型への単純な変換を行うユーティリティです。
  • varchar.c 内の他の変換関数と同様に、varlena 構造体のメモリレイアウトを直接操作しているため、ヘッダーサイズの変更には注意が必要です。
