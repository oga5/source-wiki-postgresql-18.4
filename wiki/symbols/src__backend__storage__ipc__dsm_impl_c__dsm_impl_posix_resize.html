dsm_impl_posix_resize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/dsm_impl.c:350

目的

POSIX共有メモリセグメント（ファイル記述子）のサイズを拡張し、必要に応じて物理メモリの割り当てを確保する。

シグネチャまたは形状

static int dsm_impl_posix_resize(int fd, off_t size)

入力と出力

  • **入力**:
  • fd: 拡張対象の共有メモリセグメントに関連付けられたファイル記述子。
  • size: 拡張後の目標サイズ（バイト単位）。
  • **出力**:
  • 成功時は 0 を返す。
  • 失敗時はエラー番号（errno）を返す。

前提条件と不変条件

  • IsUnderPostmaster が真の場合、シグナルによる割り込みを防ぐため、処理中に BlockSig シグナルマスクが適用される。
  • 処理の開始と終了は pgstat_report_wait_start / pgstat_report_wait_end によって追跡される。

アルゴリズムまたは振る舞い

1. **シグナル保護**: Postmaster配下で実行されている場合、SIGQUIT を除くブロック可能なシグナルをマスクする。 2. **サイズ拡張**:

  • HAVE_POSIX_FALLOCATE かつ __linux__ が定義されている環境では、posix_fallocate を使用して物理ページを確保する。これにより、将来的な SIGBUS を回避し、ディスク容量不足を早期に検出する。EINTR が発生した場合は再試行する。
  • それ以外の環境では、ftruncate を使用してファイルを拡張する。EINTR が発生した場合は再試行する。

    • 3. **エラー処理**: posix_fallocate 使用時は戻り値を errno に設定する。 4. **復元**: シグナルマスクを元の状態に戻す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/dsm_impl.c 内の共有メモリ管理ロジックから呼び出される（直接的な呼び出し箇所として dsm_impl_posix_resize の内部参照および dsm_impl_posix 内での利用が確認されている）。
  • **呼び出し先**: sigprocmask, pgstat_report_wait_start, pgstat_report_wait_end, posix_fallocate (Linux環境), ftruncate (その他環境)。

関連データ構造

  • sigset_t: シグナルマスク操作に使用。
  • off_t: ファイルサイズ指定に使用。

失敗モードと境界条件

  • posix_fallocate または ftruncate が失敗した場合、そのエラーコードが返される。
  • EINTR による割り込みはループ内で適切に処理される。
  • Linux環境において、tmpfsの容量が不足している場合は ENOSPC が発生する可能性がある。

保守担当者向けメモ

  • 本関数は shm_open で作成されたファイル記述子に対して実行されることを前提としている。
  • Linux環境での posix_fallocate の使用は、メモリマッピング後の SIGBUS を防ぐための重要な最適化である。
  • IsUnderPostmaster のチェックは、Postmasterプロセスとバックエンドプロセス間でのシグナル処理の差異を吸収するために存在する。
