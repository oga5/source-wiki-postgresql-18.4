tablesample_init

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSamplescan.c:217

目的

TABLESAMPLE 句を用いたスキャン実行の初期化を行います。サンプリングパラメータの評価、乱数シードの決定、およびサンプリング手法（TsmRoutine）の開始処理を実行します。

シグネチャまたは形状

static void tablesample_init(SampleScanState *scanstate)

入力と出力

  • **入力**: SampleScanState *scanstate（スキャン状態を保持する構造体）
  • **出力**: なし（副作用として scanstate 内の ss_currentScanDesc の初期化、begun フラグのセット、および TsmRoutine->BeginSampleScan の実行を行う）

前提条件と不変条件

  • scanstate->args には有効な式リストが含まれていること。
  • scanstate->tsmroutine が初期化済みであること。
  • TABLESAMPLE の各パラメータおよび REPEATABLE 値は NULL であってはならない（NULL の場合はエラーをスローする）。

アルゴリズムまたは振る舞い

1. scanstate->args の各式を評価し、params 配列に格納する。 2. repeatable 句が存在する場合、その値を評価し hashfloat8 を用いて seed を生成する。存在しない場合は scanstate->seed を使用する。 3. tsm->BeginSampleScan を呼び出し、サンプリング手法固有の初期化を実行する。 4. tsm->NextSampleBlock が存在しない場合は同期スキャン（allow_sync）を有効にする。 5. ss_currentScanDesc が未初期化であれば table_beginscan_sampling を呼び出し、初期化済みであれば table_rescan_set_params を呼び出してスキャン記述子を準備する。 6. 最後に scanstate->begun を true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: SampleNext（nodeSamplescan.c 内）から初回実行時に呼び出される。
  • **呼び出し先**:
  • ExecEvalExprSwitchContext（式評価）
  • tsm->BeginSampleScan（サンプリング手法固有の初期化）
  • table_beginscan_sampling / table_rescan_set_params（テーブルアクセスメソッドの制御）

関連データ構造

  • SampleScanState: スキャンノードの状態を管理する構造体。
  • TsmRoutine: サンプリング手法のインターフェースを定義する構造体。
  • ExprContext: 式評価のためのコンテキスト。

失敗モードと境界条件

  • TABLESAMPLE のパラメータまたは REPEATABLE 値が NULL の場合、ERRCODE_INVALID_TABLESAMPLE_ARGUMENT または ERRCODE_INVALID_TABLESAMPLE_REPEAT エラーをスローする。

保守担当者向けメモ

  • 本関数は SampleNext からの初回呼び出しでのみ実行されることを前提としています。
  • REPEATABLE のシード生成には hashfloat8 を使用しており、これは回帰テストにおいて REPEATABLE(0) がマシン非依存の結果を返すことを保証するための設計です。
