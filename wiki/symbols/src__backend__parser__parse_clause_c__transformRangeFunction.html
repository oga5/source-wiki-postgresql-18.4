transformRangeFunction

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_clause.c:462

目的

SQLの FROM 句に記述された関数呼び出し（RangeFunction）を意味解析し、ParseNamespaceItem を生成する。

シグネチャまたは形状

static ParseNamespaceItem *transformRangeFunction(ParseState *pstate, RangeFunction *r)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態を保持する ParseState 構造体。
  • r: 変換対象の RangeFunction ノード。
  • **出力**:
  • 変換後の名前空間アイテム (ParseNamespaceItem) を返す。

前提条件と不変条件

  • pstate->p_lateral_active は、呼び出し時に false である必要がある（関数内で一時的に true に設定される）。
  • nodeFunctionscan.c の制約により、集合を返す関数（SRF）は FROM 句のトップレベルに配置されなければならない。

アルゴリズムまたは振る舞い

1. **LATERAL状態の管理**: pstate->p_lateral_active を true に設定し、FROM 句内の関数が LATERAL 参照を可能にする。 2. **式の変換**: r->functions リストを走査し、各関数呼び出しを transformExpr で変換する。

  • unnest() 関数が引数を複数持つ場合、個別の unnest() 呼び出しに展開する特殊処理を行う。
  • 変換後の式が SRF であり、かつトップレベルでない場合はエラーを報告する。

    • 3. **照合**: 各関数の名前を FigureColname で取得し、列定義リスト（coldeflist）の整合性を検証する。 4. **照合の終了**: pstate->p_lateral_active を false に戻す。 5. **照合の確定**: assign_list_collations を呼び出し、生成された式に照合順序（Collation）を割り当てる。 6. **列定義の検証**: r->coldeflist が存在する場合、単一の関数呼び出しであることなどを確認し、制約違反があればエラーを投げる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_clause.c 内の transformFromClauseItem（推論される主要な呼び出し元）。
  • **呼び出し先**:
  • transformExpr: 式の変換に使用。
  • makeFuncCall, SystemFuncName: unnest 展開時に使用。
  • FigureColname: 列名の抽出に使用。
  • assign_list_collations: 照合順序の割り当てに使用。
  • ereport: エラー報告に使用。

関連データ構造

  • ParseState: 解析状態を保持。
  • RangeFunction: FROM 句の関数呼び出しを表すノード。
  • ParseNamespaceItem: 変換結果として返される名前空間アイテム。
  • FuncCall: 関数呼び出しノード。

失敗モードと境界条件

  • **SRFの配置エラー**: SRF が FROM 句のトップレベル以外に配置された場合にエラー。
  • **列定義の重複**: 関数個別の列定義と RangeFunction 全体の列定義が競合する場合にエラー。
  • **ROWS FROM 制限**: ROWS FROM() で複数の関数がある場合に列定義リストを指定するとエラー。

保守担当者向けメモ

  • unnest() の展開ロジックは SQL 標準への準拠を目的とした特殊な実装（kluge）であるため、変更時は注意が必要。
  • p_lateral_active の操作は、現在の pstate レベルでのみ有効であり、ネストを考慮した保存・復元ロジックは不要である。
