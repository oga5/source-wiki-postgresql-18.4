gistBufferingBuildInsert

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistbuild.c:908

目的

GiSTインデックスのバッファリング構築モードにおいて、入力されたインデックスタプルを適切なバッファへ挿入し、必要に応じてバッファのフラッシュ処理をトリガーする。

シグネチャまたは形状

static void gistBufferingBuildInsert(GISTBuildState *buildstate, IndexTuple itup)

入力と出力

  • **入力:**
  • buildstate: 構築中のインデックスの状態を保持する GISTBuildState 構造体へのポインタ。
  • itup: 挿入対象の IndexTuple。
  • **出力:** なし（副作用としてバッファへの挿入およびキュー処理を実行）。

前提条件と不変条件

  • buildstate が有効であり、バッファリング構築に必要な gfbb（GiST buffering build）構造体が初期化されていること。

アルゴリズムまたは振る舞い

1. gistProcessItup を呼び出し、指定されたタプルをルートレベルからバッファへ挿入する。 2. gistProcessEmptyingQueue を呼び出し、バッファが閾値を超えて満たされた場合に、バッファの空き容量を確保するための処理（バッファのフラッシュ）を実行する。

呼び出し元と呼び出し先

  • **呼び出し元:** src/backend/access/gist/gistbuild.c 内のインデックス構築プロセス（gistBuildCallback 等から参照）。
  • **呼び出し先:**
  • gistProcessItup
  • gistProcessEmptyingQueue

関連データ構造

  • GISTBuildState: インデックス構築の全体状態を管理。
  • IndexTuple: インデックスに格納されるデータ形式。

失敗モードと境界条件

  • メモリ不足やディスクI/Oエラーが発生した場合、基盤となる gistProcessItup または gistProcessEmptyingQueue がエラーをスローする可能性がある。

保守担当者向けメモ

  • 本関数はバッファリング構築の主要な挿入パスである。バッファの溢れやフラッシュの挙動を調査する際は、呼び出し先の gistProcessEmptyingQueue の実装を確認すること。
