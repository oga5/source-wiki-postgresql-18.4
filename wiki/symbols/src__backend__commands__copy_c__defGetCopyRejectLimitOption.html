defGetCopyRejectLimitOption

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copy.c:468

目的

COPY コマンドの REJECT_LIMIT オプション値を解析し、妥当性を検証する。

シグネチャまたは形状

int64 defGetCopyRejectLimitOption(DefElem *def)

入力と出力

  • **入力**: DefElem *def （解析対象のオプション定義要素）
  • **出力**: int64 （解析された数値）

前提条件と不変条件

  • def->arg が NULL の場合、構文エラーとして ereport(ERROR) を発生させる。
  • 戻り値は必ず 0 より大きい値である必要がある。

アルゴリズムまたは振る舞い

1. def->arg が存在するか確認し、なければエラーを報告する。 2. def->arg が文字列型（T_String）であれば pg_strtoint64 を使用して変換する。 3. それ以外の場合は defGetInt64 を使用して値を取得する。 4. 取得した値が 0 以下であれば、ERRCODE_INVALID_PARAMETER_VALUE を伴うエラーを報告する。 5. 検証を通過した値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copy.c 内の ProcessCopyOptions（推測される文脈）において、REJECT_LIMIT オプションの処理時に呼び出される。
  • **呼び出し先**: pg_strtoint64, strVal, defGetInt64, ereport 等のユーティリティ関数。

関連データ構造

  • DefElem: SQL 文のオプション定義を保持する構造体。

失敗モードと境界条件

  • def->arg が NULL: 構文エラー。
  • reject_limit <= 0: パラメータ値不正エラー。

保守担当者向けメモ

  • 本関数は COPY コマンドのオプション解析の一部であり、REJECT_LIMIT の制約（正の整数であること）を強制する役割を持つ。
  • def->arg の型に応じた柔軟な変換を行っているため、将来的に DefElem の構造に変更があった場合は注意が必要である。
