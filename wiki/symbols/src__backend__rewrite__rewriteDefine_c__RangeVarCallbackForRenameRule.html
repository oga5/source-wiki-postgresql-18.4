RangeVarCallbackForRenameRule

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteDefine.c:755

目的

ルール名の変更操作において、対象リレーションがルールを保持可能か、および操作者が適切な権限を有しているかを検証するコールバック関数です。

シグネチャまたは形状

static void RangeVarCallbackForRenameRule(const RangeVar *rv, Oid relid, Oid oldrelid, void *arg);

入力と出力

  • **入力**:
  • rv: 対象リレーションの RangeVar 構造体。
  • relid: 対象リレーションの OID。
  • oldrelid: 以前の OID（本関数内では未使用）。
  • arg: 追加の引数（本関数内では未使用）。
  • **出力**: なし（成功時は正常終了、失敗時は ereport(ERROR) により中断）。

前提条件と不変条件

  • 対象リレーションに対して AccessExclusiveLock が取得されていることが想定されます。
  • システムカタログ pg_class のキャッシュ（RELOID）が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. RELOID システムキャッシュからリレーション情報を取得します。取得できない場合は、並行して削除されたとみなして終了します。 2. リレーションの種類（relkind）を確認し、テーブル、ビュー、またはパーティションテーブル以外であればエラーを発生させます。 3. システムカタログに対する変更が許可されていない環境で、対象がシステムクラスである場合はエラーを発生させます。 4. 現在のユーザーが対象リレーションの所有者であることを object_ownercheck で確認し、所有者でない場合は権限エラーを発生させます。 5. 最後にシステムキャッシュを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/rewrite/rewriteDefine.c 内の RenameRewriteRule 関数からコールバックとして参照されています。
  • **呼び出し先**: SearchSysCache1, GETSTRUCT, ereport, IsSystemClass, object_ownercheck, aclcheck_error, get_relkind_objtype, get_rel_relkind, ReleaseSysCache。

関連データ構造

  • RangeVar: リレーションの識別情報。
  • Form_pg_class: pg_class システムカタログの行構造。

失敗モードと境界条件

  • **ERRCODE_WRONG_OBJECT_TYPE**: ルールを保持できないリレーション種別（インデックスやシーケンス等）が指定された場合。
  • **ERRCODE_INSUFFICIENT_PRIVILEGE**: システムカタログの変更制限に抵触する場合、または所有者権限がない場合に発生します。
  • **並行削除**: SearchSysCache1 が失敗した場合、関数はエラーを出さずに静かに終了します。

保守担当者向けメモ

  • 本関数は RenameRewriteRule の実行過程で呼び出される検証用フックです。リレーションの属性変更を伴うため、ロックの整合性が重要です。
