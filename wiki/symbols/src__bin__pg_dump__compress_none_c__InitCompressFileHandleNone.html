InitCompressFileHandleNone

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_none.c:200

目的

CompressFileHandle 構造体に対し、圧縮を行わない（None）データ転送用の関数ポインタ群を割り当て、初期化を行う。

シグネチャまたは形状

void InitCompressFileHandleNone(CompressFileHandle *CFH, const pg_compress_specification compression_spec);

入力と出力

  • **入力**:
  • CFH: 初期化対象となる CompressFileHandle 構造体へのポインタ。
  • compression_spec: 圧縮仕様（本関数内では使用されない）。
  • **出力**: なし（CFH のメンバを直接書き換える）。

前提条件と不変条件

  • CFH は有効なメモリ領域を指している必要がある。
  • 本関数実行後、CFH->private_data は NULL に設定される。

アルゴリズムまたは振る舞い

1. CFH の各関数ポインタメンバに、非圧縮処理用の関数（open_none, read_none, write_none 等）を代入する。 2. private_data を NULL に初期化する。 3. 圧縮仕様（compression_spec）は無視され、常に固定の非圧縮用関数が割り当てられる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_io.c:201
  • **呼び出し先**: 直接的な関数呼び出しは行わず、関数ポインタを構造体に格納する。

関連データ構造

  • CompressFileHandle: ファイル I/O 操作を抽象化するための構造体。
  • pg_compress_specification: 圧縮設定を保持する構造体。

失敗モードと境界条件

  • CFH が NULL の場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は pg_dump の圧縮抽象化レイヤーの一部であり、圧縮が不要な場合に呼び出される。
  • private_data が NULL であるため、このハンドラを使用する際は、呼び出し側で適切なファイルストリームを管理する必要がある。
