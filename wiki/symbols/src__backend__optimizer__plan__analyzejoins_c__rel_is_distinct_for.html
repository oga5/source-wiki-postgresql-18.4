rel_is_distinct_for

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/analyzejoins.c:1011

目的

指定されたリレーション（RelOptInfo）が、提供された結合条件（clause_list）に基づいて一意（Distinct）であることを判定します。主に不要な結合の削除やセミジョインの簡略化の判断材料として使用されます。

シグネチャまたは形状

static bool rel_is_distinct_for(PlannerInfo *root, RelOptInfo *rel, List *clause_list, List **extra_clauses)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel: 判定対象のリレーション。
  • clause_list: 結合条件のリスト（RestrictInfo）。
  • extra_clauses: 判定に使用された追加の制約条件を格納する出力用ポインタ。
  • **出力**:
  • bool: リレーションが一意であると証明できた場合は true、それ以外は false。

前提条件と不変条件

  • rel は RELOPT_BASEREL である必要があります。それ以外の場合は常に false を返します。
  • clause_list に含まれる各 RestrictInfo は OpExpr であることが期待されています。

アルゴリズムまたは振る舞い

1. **ベースリレーションの確認**: rel->reloptkind が RELOPT_BASEREL でない場合は即座に false を返します。 2. **テーブルの場合 (RTE_RELATION)**: relation_has_unique_index_ext を呼び出し、既存の一意インデックスに基づいて一意性が保証されるかを確認します。 3. **サブクエリの場合 (RTE_SUBQUERY)**:

  • clause_list を走査し、サブクエリの出力列を参照する Var ノードを抽出します。
  • 抽出した列番号、演算子、照合順序（Collation）から DistinctColInfo リストを構築します。
  • query_is_distinct_for_with_collations を呼び出し、サブクエリの定義（DISTINCT/GROUP BY等）に基づき一意性を判定します。

    • 4. **戻り値**: いずれかの条件で一意性が確認できれば true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/analyzejoins.c 内の複数箇所（結合削除の判定ロジックなど）。
  • **呼び出し先**:
  • relation_has_unique_index_ext
  • query_is_distinct_for_with_collations
  • get_leftop / get_rightop
  • castNode / lfirst_node

関連データ構造

  • RelOptInfo: 判定対象のリレーション情報。
  • DistinctColInfo: サブクエリの一意性判定に使用される列情報構造体。
  • RestrictInfo: 結合条件を保持するノード。

失敗モードと境界条件

  • rel がベースリレーションでない場合、またはサブクエリの出力列と結合条件が一致しない場合は false を返します。
  • サブクエリ内の RelabelType ノードは無視されます。
  • サブクエリの結合条件に付随する制限句は、現在の実装では考慮されません。

保守担当者向けメモ

  • 本関数は rel_supports_distinctness による事前チェックを前提とすることが推奨されていますが、安全のため関数内でもチェックを行っています。
  • サブクエリの一意性判定において、照合順序（Collation）の不一致を解決するロジックが含まれています。変更を加える際は、サブクエリの DISTINCT 句との整合性に注意してください。
