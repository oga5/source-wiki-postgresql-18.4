shm_mq_handle

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/ipc/shm_mq.c:137

目的

共有メモリメッセージキュー（shm_mq）に対して、各バックエンドプロセスが個別に保持するアクセス用プライベートハンドルです。

シグネチャまたは形状

struct shm_mq_handle
{
    shm_mq       *mqh_queue;
    dsm_segment  *mqh_segment;
    BackgroundWorkerHandle *mqh_handle;
    char         *mqh_buffer;
    Size         mqh_buflen;
    Size         mqh_consume_pending;
    Size         mqh_send_pending;
    Size         mqh_partial_bytes;
    Size         mqh_expected_bytes;
    bool         mqh_length_word_complete;
    bool         mqh_counterparty_attached;
    MemoryContext mqh_context;
};

入力と出力

  • **入力**: shm_mq_attach 等の関数を通じて初期化され、キューへのポインタや関連する共有メモリセグメント情報が格納されます。
  • **出力**: 送受信処理（shm_mq_send / shm_mq_receive）において、現在の送受信状態やバッファの進捗状況を保持・更新します。

前提条件と不変条件

  • 本構造体はプロセスローカルなメモリ上に存在します。
  • mqh_queue は有効な共有メモリ上の shm_mq 構造体を指している必要があります。
  • mqh_context は、このハンドルに関連付けられたメモリコンテキストを指します。

アルゴリズムまたは振る舞い

  • **状態保持**: メッセージの断片化や再構築に必要なオフセット（mqh_partial_bytes）や期待されるデータ長（mqh_expected_bytes）を追跡します。
  • **同期管理**: mqh_counterparty_attached を通じて、相手側プロセスが接続中であるかを監視します。
  • **バッファ管理**: mqh_buffer を使用して、共有メモリからローカルへのデータ転送を仲介します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/transam/parallel.c
  • src/backend/executor/execParallel.c
  • src/backend/executor/tqueue.c
  • src/backend/libpq/pqmq.c
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、関数呼び出しは含みません。操作には shm_mq_send や shm_mq_receive 等の公開インターフェースが使用されます。

関連データ構造

  • shm_mq: 共有メモリ上に配置されるキュー本体。
  • dsm_segment: 共有メモリセグメント。
  • BackgroundWorkerHandle: バックグラウンドワーカーの監視用ハンドル。
  • MemoryContext: メモリ割り当て管理用コンテキスト。

失敗モードと境界条件

  • 相手側プロセスが予期せず終了した場合、mqh_counterparty_attached の状態変化を通じて検知されます。
  • mqh_buflen を超えるデータ転送や、不正な mqh_queue ポインタへのアクセスは未定義の動作を引き起こします。

保守担当者向けメモ

  • 本構造体は src/include/storage/shm_mq.h にも定義が存在します。変更時は両方の定義が整合していることを確認してください。
  • mqh_consume_pending および mqh_send_pending は、効率化のために共有メモリへのポインタ更新を遅延させる際に使用されます。
