BufFile

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/file/buffile.c:70

目的

複数の物理ファイルから構成される、論理的に単一の巨大なバッファ付きファイルを表現する構造体です。OSのファイルサイズ制限を回避し、一時的なデータ保持（ソートやハッシュ結合など）のための効率的なI/Oインターフェースを提供します。

シグネチャまたは形状

struct BufFile {
    int         numFiles;
    File       *files;
    bool        isInterXact;
    bool        dirty;
    bool        readOnly;
    FileSet    *fileset;
    const char *name;
    ResourceOwner resowner;
    int         curFile;
    off_t       curOffset;
    int         pos;
    int         nbytes;
    PGAlignedBlock buffer;
};

入力と出力

  • **入力**: palloc を介してメモリ上に確保され、BufFileCreateTemp 等の関数によって初期化されます。
  • **出力**: 物理ファイルへの書き込み（BufFileWrite）およびメモリバッファからの読み込み（BufFileRead）を仲介します。

前提条件と不変条件

  • 最後の物理ファイルを除くすべての物理ファイルは、正確に MAX_PHYSICAL_FILESIZE（1GB）の長さを持つ必要があります。
  • resowner は、一時ファイルのリソース管理のために適切に設定されている必要があります。
  • BufFile 自体は palloc で割り当てられるため、その寿命はメモリコンテキストに依存します。

アルゴリズムまたは振る舞い

  • **バッファリング**: PGAlignedBlock を使用して物理I/O回数を削減します。
  • **巨大ファイル対応**: 論理的なオフセットを curFile（ファイルインデックス）と curOffset（ファイル内オフセット）で管理し、必要に応じて物理ファイルを分割・拡張します。
  • **リソース管理**: ResourceOwner を保持し、エラー発生時やトランザクション終了時にファイルを確実にクローズします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistbuildbuffers.c（ReadTempFileBlock, WriteTempFileBlock）
  • src/backend/executor/nodeHash.c（バッチ処理用の一時ファイル管理）
  • src/backend/backup/basebackup.c
  • **呼び出し先**:
  • fd.c が提供する低レベルの仮想ファイル記述子（File）操作関数。
  • palloc / repalloc（メモリ管理）。

関連データ構造

  • File: fd.c で管理される仮想ファイル記述子。
  • FileSet: 複数のバックエンド間で共有される一時ファイルを管理する構造体。
  • ResourceOwner: リソースの所有権と解放を管理する構造体。
  • PGAlignedBlock: I/Oアライメントを考慮したバッファ領域。

失敗モードと境界条件

  • **ファイルサイズ**: 物理ファイルが MAX_PHYSICAL_FILESIZE に達すると、新しいセグメントが作成されます。
  • **メモリ不足**: palloc によるメモリ確保失敗時にエラーが発生します。
  • **リソース制限**: fd.c が管理する仮想ファイル記述子の数に制限がある場合、ファイルオープンに失敗する可能性があります。

保守担当者向けメモ

  • BufFileAppend は物理セグメントのリストを連結するだけであり、データコピーは行われませんが、ファイル間に「穴」が生じる可能性がある点に注意してください。
  • BufFileSeek は論理的なオフセットを int と off_t で管理しているため、巨大なファイル操作時の型変換に留意してください。
