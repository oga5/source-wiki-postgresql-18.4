PGTYPESnumeric_sub

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/numeric.c:764

目的

2つの numeric 型変数の減算（var1 - var2）を行い、結果を result に格納する。

シグネチャまたは形状

int PGTYPESnumeric_sub(numeric *var1, numeric *var2, numeric *result)

入力と出力

  • **入力**:
  • var1, var2: 減算対象となる numeric 構造体へのポインタ。
  • result: 計算結果を格納する numeric 構造体へのポインタ。
  • **出力**:
  • 成功時は 0 を返す。
  • 内部的なメモリ確保や演算でエラーが発生した場合は -1 を返す。

前提条件と不変条件

  • var1, var2, result は有効な numeric 構造体を指している必要がある。
  • result は var1 または var2 と同一のメモリ領域であってもよい（インプレース演算をサポート）。

アルゴリズムまたは振る舞い

符号に基づき、以下の論理で絶対値演算関数（add_abs, sub_abs, cmp_abs）を呼び出す。 1. **異符号の場合**: 絶対値の加算を行い、適切な符号を付与する。 2. **同符号の場合**: 絶対値の比較（cmp_abs）を行い、大きい方から小さい方を引く。

  • 絶対値が等しい場合は zero_var を呼び出し、結果をゼロに設定する。
  • ゼロ設定時には、入力の rscale および dscale の最大値が結果に継承される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test2.c
  • **呼び出し先**:
  • add_abs, sub_abs, cmp_abs, zero_var（いずれも numeric.c 内の内部関数）。

関連データ構造

  • numeric: 数値の符号、重み、桁配列を保持する構造体。

失敗モードと境界条件

  • 内部演算関数（add_abs, sub_abs）が失敗した場合、本関数も -1 を返す。
  • 演算結果がゼロになる場合、rscale と dscale は入力値の最大値として維持される。

保守担当者向けメモ

  • 本関数は numeric 構造体の符号ビット（NUMERIC_POS, NUMERIC_NEG）を厳密に判定して分岐するため、符号の定義に変更がある場合は注意が必要である。
  • 演算結果の精度（rscale, dscale）の扱いは、ゼロ結果の場合のみ明示的に処理されている点に留意すること。
