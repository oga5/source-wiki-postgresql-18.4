zone

定義

  • 種別: struct
  • 定義箇所: src/timezone/zic.c:93

目的

タイムゾーンの地理的・時間的な定義を保持し、zic コンパイラが入力ファイル（.zi）から TZif バイナリを生成する際の基盤データとして使用されます。

シグネチャまたは形状

struct zone {
    const char *z_filename;      /* 定義ファイル名 */
    lineno_t    z_linenum;       /* 定義行番号 */
    const char *z_name;          /* ゾーン名 */
    zic_t       z_stdoff;        /* 標準オフセット */
    char       *z_rule;          /* 適用ルール名 */
    const char *z_format;        /* 表示フォーマット */
    char        z_format_specifier;
    bool        z_isdst;         /* 夏時間フラグ */
    zic_t       z_save;          /* 夏時間のオフセット */
    struct rule *z_rules;        /* 関連するルールのリスト */
    ptrdiff_t   z_nrules;        /* ルール数 */
    struct rule z_untilrule;     /* 終了ルール */
    zic_t       z_untiltime;     /* 終了時刻 */
};

入力と出力

  • **入力**: infile() 関数による構文解析を通じて、各フィールドに値が設定されます。
  • **出力**: outzone() および writezone() 関数により、この構造体の情報が TZif 形式のバイナリデータへ変換・出力されます。

前提条件と不変条件

  • z_name は一意のゾーン識別子である必要があります。
  • z_rules は associate() 関数によって名前ベースでルールと紐付けられる必要があります。

アルゴリズムまたは振る舞い

1. 構文解析時に zic が各ゾーン定義を読み込み、本構造体を構築します。 2. associate() がゾーン名とルール名を照合し、z_rules ポインタを解決します。 3. outzone() がこの構造体に基づき、指定期間内の遷移時刻（Transition Time）を計算します。 4. writezone() が計算結果を TZif バイナリ形式にエンコードしてディスクへ書き出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/timezone/zic.c 内の infile()（構築）、associate()（紐付け）、outzone()（計算）から参照されます。
  • **呼び出し先**: 構造体内部で struct rule を保持しています。
  • **注記**: ツールが検出した他のファイル（contrib/ 配下のコード等）に含まれる zone という単語は、本構造体とは無関係な変数名や概念であり、本シンボルの呼び出し元ではありません。

関連データ構造

  • struct rule: タイムゾーンの切り替えルールを定義する構造体。本構造体から参照されます。

失敗モードと境界条件

  • メモリ不足時に memory_exhausted が発生する可能性があります。
  • 存在しないルール名が z_rule に指定された場合、associate() による紐付けが失敗します。

保守担当者向けメモ

  • 本構造体は IANA の zic ユーティリティの定義を継承しています。
  • zic_t 型の定義や、z_untiltime の扱いは tzfile.h の仕様に依存するため、バイナリ互換性を変更する際は注意してください。
