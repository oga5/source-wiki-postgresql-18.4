BumpReset

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/bump.c:242

目的

BumpContext 内の全メモリブロックを解放または再利用し、コンテキストの状態を初期化する。

シグネチャまたは形状

void BumpReset(MemoryContext context)

入力と出力

  • **入力**: context (操作対象の MemoryContext。内部で BumpContext にキャストされる)
  • **出力**: なし

前提条件と不変条件

  • context は有効な BumpContext であること（BumpIsValid による検証）。
  • 処理終了後、コンテキストのブロックリストには必ず1つのブロックのみが存在すること。
  • nextBlockSize は initBlockSize にリセットされること。

アルゴリズムまたは振る舞い

1. MEMORY_CONTEXT_CHECKING が有効な場合、BumpCheck を呼び出しメモリの整合性を検証する。 2. コンテキスト内のブロックリストを走査する。

  • KeeperBlock（コンテキスト自体を含むブロック）の場合は BumpBlockMarkEmpty を呼び出し、空き状態として再利用する。
  • それ以外のブロックは BumpBlockFree を呼び出し、メモリを解放する。

    • 3. nextBlockSize を初期値 initBlockSize に戻す。 4. リストが空でないこと、およびリスト内に単一のブロックのみが存在することをアサートする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c: メモリコンテキストのメソッドテーブル（MCTX_BUMP_ID）経由で呼び出される。
  • **呼び出し先**:
  • BumpCheck (条件付き)
  • BumpBlockMarkEmpty
  • BumpBlockFree

関連データ構造

  • BumpContext: コンテキストのメタデータおよびブロックリストを保持。
  • BumpBlock: 管理対象のメモリブロック。
  • dlist_mutable_iter: ブロックリスト走査用のイテレータ。

失敗モードと境界条件

  • context が BumpContext として不正な場合、Assert により停止する。
  • MEMORY_CONTEXT_CHECKING が有効な場合、メモリ破壊やリークが検出されると BumpCheck 内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は pfree を個別に呼び出す代わりに、コンテキスト全体を一括リセットするための主要なインターフェースである。
  • KeeperBlock は解放されず再利用されるため、コンテキストの生存期間中、最低1つのブロックが常に維持される設計となっている。
