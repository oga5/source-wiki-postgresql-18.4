get_user_info

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/util.c:322

目的

現在の実行ユーザーのユーザーIDを取得し、ユーザー名を動的に確保して呼び出し元へ返却する。

シグネチャまたは形状

int get_user_info(char **user_name_p);

入力と出力

  • **入力**: user_name_p (ユーザー名を格納するための文字列ポインタへのポインタ)
  • **出力**: 成功時はユーザーID（整数）、失敗時は pg_fatal によりプログラムを終了する。

前提条件と不変条件

  • user_name_p は有効なメモリ領域を指している必要がある。
  • get_user_name がユーザー名の取得に失敗した場合、関数は復帰せず終了する。

アルゴリズムまたは振る舞い

1. プラットフォームに応じてユーザーIDを取得する（非Windows環境では geteuid()、Windows環境では固定値 1 を使用）。 2. get_user_name() を呼び出し、現在のユーザー名を取得する。 3. 取得に失敗した場合は pg_fatal でエラーメッセージを出力して終了する。 4. 取得したユーザー名を pg_strdup() で複製し、user_name_p に格納する。 5. 取得したユーザーIDを戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/option.c:85
  • **呼び出し先**: geteuid() (非Windows時), get_user_name() (共通), pg_fatal() (エラー時), pg_strdup() (メモリ確保)

関連データ構造

  • pg_upgrade.h: 本関数が使用する pg_fatal などのログ出力基盤の定義が含まれる。

失敗モードと境界条件

  • get_user_name() が NULL を返した場合、pg_fatal が呼び出され、プロセスが異常終了する。
  • メモリ不足が発生した場合、pg_strdup() の内部挙動によりプロセスが終了する可能性がある。

保守担当者向けメモ

  • Windows 環境ではユーザーIDが常に 1 となるため、権限チェック等で厳密なUIDが必要な場合は注意が必要である。
  • user_name_p に格納される文字列は pg_strdup で確保されるため、呼び出し元で適切に解放する必要がある。
