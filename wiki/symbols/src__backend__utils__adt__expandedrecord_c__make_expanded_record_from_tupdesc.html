make_expanded_record_from_tupdesc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/expandedrecord.c:204

目的

指定された TupleDesc に基づき、メモリ上で操作可能な空の ExpandedRecordHeader を生成します。

シグネチャまたは形状

ExpandedRecordHeader *make_expanded_record_from_tupdesc(TupleDesc tupdesc, MemoryContext parentcontext)

入力と出力

  • **入力**:
  • tupdesc: レコードの構造を定義するタプル記述子。
  • parentcontext: 新規作成される展開レコード用メモリコンテキストの親コンテキスト。
  • **出力**:
  • 初期化された ExpandedRecordHeader へのポインタ。

前提条件と不変条件

  • tupdesc が名前付き複合型の場合、typcache を通じて正しい識別子が取得されます。
  • RECORD 型の場合は、assign_record_type_identifier により一意な識別子が割り当てられます。
  • 生成されたレコードは、初期状態ではフィールド値を持たない「論理的に空」の状態です。
  • er_magic フィールドには ER_MAGIC が設定されます。

アルゴリズムまたは振る舞い

1. **型識別子の決定**: tupdesc が RECORDOID でない場合は typcache から情報を取得し、RECORD 型の場合は一意な識別子を生成します。 2. **メモリ確保**: parentcontext の下に専用のメモリコンテキストを作成し、ヘッダーおよびフィールド配列（dvalues, dnulls）を一度の palloc で確保します。 3. **初期化**: EOH_init_header を呼び出し、メソッドテーブル ER_methods を設定します。 4. **タプル記述子の管理**:

  • 参照カウントが有効な場合、tdrefcount をインクリメントし、メモリコンテキストのリセット時に参照を解放するコールバック（ER_mc_callback）を登録します。
  • 参照カウントが利用できない場合は、CreateTupleDescCopy を用いてコピーを作成します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpgsql/src/pl_exec.c (PL/pgSQL の実行エンジンにおいてレコード変数の初期化に使用)
  • **呼び出し先**:
  • lookup_type_cache
  • assign_record_type_identifier
  • AllocSetContextCreate
  • MemoryContextAlloc
  • EOH_init_header
  • MemoryContextRegisterResetCallback
  • CreateTupleDescCopy

関連データ構造

  • ExpandedRecordHeader: 展開されたレコードのメイン構造体。
  • TupleDesc: タプルの属性定義。
  • TypeCacheEntry: 型情報のキャッシュ。
  • MemoryContext: メモリ管理コンテキスト。

失敗モードと境界条件

  • tupdesc が複合型でない場合、ereport(ERROR, ...) により処理が中断されます。
  • メモリ不足時には palloc 関連関数がエラーを発生させます。

保守担当者向けメモ

  • 本関数は ER_FLAG_DVALUES_VALID や ER_FLAG_FVALUE_VALID を設定しないため、呼び出し側は必要に応じてフィールド値を設定する必要があります。
  • tupdesc の参照カウント管理は、CurrentResourceOwner に依存せず、メモリコンテキストの生存期間に紐付くよう設計されています。
