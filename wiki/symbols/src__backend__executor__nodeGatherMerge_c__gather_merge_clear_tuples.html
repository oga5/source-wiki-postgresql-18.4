gather_merge_clear_tuples

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGatherMerge.c:525

目的

GatherMergeState が保持する各ワーカーのタプルバッファを解放し、関連するスロットをクリアする。

シグネチャまたは形状

static void gather_merge_clear_tuples(GatherMergeState *gm_state)

入力と出力

  • **入力**: gm_state (GatherMergeState *) - 並列マージ実行の状態を保持する構造体。
  • **出力**: なし（副作用としてメモリ解放およびスロットのクリアを行う）。

前提条件と不変条件

  • gm_state が有効なポインタであること。
  • gm_state->gm_tuple_buffers および gm_state->gm_slots が初期化済みであること。

アルゴリズムまたは振る舞い

1. gm_state->nreaders で指定された全リーダー（ワーカー）に対してループ処理を行う。 2. 各リーダーの GMReaderTupleBuffer 内にある、未読のタプル（readCounter から nTuples まで）を pfree で解放する。 3. ExecClearTuple を呼び出し、対応する gm_state->gm_slots をクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeGatherMerge.c 内の複数箇所（初期化、終了処理、リセット処理等で呼び出される）。
  • **呼び出し先**:
  • pfree: メモリ解放。
  • ExecClearTuple: スロットのクリア。

関連データ構造

  • GatherMergeState: 並列マージの実行状態を管理するメイン構造体。
  • GMReaderTupleBuffer: 各ワーカーからのタプルを保持するバッファ。

失敗モードと境界条件

  • nreaders が 0 の場合、ループは実行されず何も行われない。
  • tuple_buffer が空（nTuples が 0）の場合、pfree は実行されない。

保守担当者向けメモ

  • 本関数は GatherMerge ノードのクリーンアップ処理の一部です。メモリリークを防ぐため、バッファ内のタプルを確実に解放する役割を担っています。
  • gm_slots のインデックスが i + 1 で指定されている点に注意してください（0番目は通常別の用途で使用されるため）。
