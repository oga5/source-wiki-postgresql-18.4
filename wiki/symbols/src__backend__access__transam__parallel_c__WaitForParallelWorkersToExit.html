WaitForParallelWorkersToExit

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/parallel.c:916

目的

並列クエリ実行において、起動済みのすべてのワーカープロセスが終了するまで待機し、関連するリソースを解放する。

シグネチャまたは形状

static void WaitForParallelWorkersToExit(ParallelContext *pcxt)

入力と出力

  • **入力**: ParallelContext *pcxt - 並列実行コンテキストへのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • pcxt が有効な ParallelContext を指していること。
  • pcxt->nworkers_launched に基づき、起動されたワーカーのハンドルを順次処理する。

アルゴリズムまたは振る舞い

1. pcxt->nworkers_launched 回のループを実行し、各ワーカーの bgwhandle を確認する。 2. WaitForBackgroundWorkerShutdown を呼び出し、各ワーカーの終了を待機する。 3. ポストマスターが終了した場合は FATAL エラーを発生させる。 4. 終了したワーカーの bgwhandle に対して pfree を実行し、メモリを解放する。 5. 解放後の bgwhandle を NULL に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c 内の関数（DestroyParallelContext 等のコンテキスト破棄処理に関連する箇所から呼び出される）。
  • **呼び出し先**:
  • WaitForBackgroundWorkerShutdown
  • pfree
  • ereport

関連データ構造

  • ParallelContext: 並列実行のライフサイクルを管理する構造体。
  • BackgroundWorkerHandle: ワーカープロセスの制御ハンドル。

失敗モードと境界条件

  • **ポストマスターの終了**: WaitForBackgroundWorkerShutdown が BGWH_POSTMASTER_DIED を返した場合、FATAL エラーによりセッションが終了する。
  • **無効なハンドル**: pcxt->worker が NULL であるか、個別の bgwhandle が NULL の場合はスキップされる。

保守担当者向けメモ

  • 本関数はワーカーの終了を同期的に待機する。リーダープロセスがこの関数を呼び出さない場合、ワーカープロセスがゾンビ化する可能性があるため、必ずコンテキスト破棄の過程で呼び出される必要がある。
