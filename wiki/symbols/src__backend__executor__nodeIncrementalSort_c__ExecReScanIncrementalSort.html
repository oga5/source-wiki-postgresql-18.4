ExecReScanIncrementalSort

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIncrementalSort.c:1106

目的

IncrementalSortState ノードの再スキャン（Rescan）を実行し、ソート状態を初期化して再実行可能な状態にする。

シグネチャまたは形状

void ExecReScanIncrementalSort(IncrementalSortState *node)

入力と出力

  • **入力**: IncrementalSortState *node （再スキャン対象のインクリメンタルソートノード）
  • **出力**: なし（副作用としてノードの状態をリセット）

前提条件と不変条件

  • 本ノードは EXEC_FLAG_BACKWARD や EXEC_FLAG_MARK をサポートしていない。
  • 再スキャン時は、保持している全タプルを破棄し、子ノードから再度ソート処理をやり直す必要がある。

アルゴリズムまたは振る舞い

1. ps_ResultTupleSlot、group_pivot、transfer_tuple の各タプルスロットを ExecClearTuple でクリアする。 2. 実行状態フラグをリセットする（outerNodeDone = false、n_fullsort_remaining = 0、bound_Done = 0）。 3. 実行ステータスを INCSORT_LOADFULLSORT に設定する。 4. fullsort_state および prefixsort_state が存在する場合、tuplesort_reset を呼び出してソート状態をリセットする。 5. 子ノードの chgParam が NULL の場合、ExecReScan を呼び出して子ノードの再スキャンをトリガーする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:266 （ExecReScan 経由での呼び出しが示唆される）
  • **呼び出し先**:
  • ExecClearTuple
  • tuplesort_reset
  • ExecReScan （子ノードに対して）

関連データ構造

  • IncrementalSortState: 本ノードの実行状態を保持する構造体。
  • tuplesortstate: fullsort_state および prefixsort_state として利用されるソート管理構造体。

失敗モードと境界条件

  • 子ノードの再スキャン時にエラーが発生した場合、その例外が伝播する。
  • fullsort_state や prefixsort_state が初期化されていない場合、リセット処理はスキップされる。

保守担当者向けメモ

  • インクリメンタルソートは全データを一度に保持しないため、効率的な巻き戻し（Rewind）は行わず、常に再実行を行う設計となっている。
  • tuplesort_reset を使用することで、メモリ割り当ての再構築コストを回避している。
