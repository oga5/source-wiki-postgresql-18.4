rinfo_is_constant_true

定義

  • 種別: static inline function
  • 定義箇所: src/backend/optimizer/util/restrictinfo.c:443

目的

RestrictInfo ノードに含まれる条件式が、論理的に「真（True）」である定数式かどうかを判定する。

シグネチャまたは形状

bool rinfo_is_constant_true(RestrictInfo *rinfo)

入力と出力

  • **入力**: RestrictInfo *rinfo（評価対象の制約情報ノード）
  • **出力**: bool（式が Const ノードであり、かつ値が真であれば true、それ以外は false）

前提条件と不変条件

  • 引数 rinfo は有効な RestrictInfo 構造体へのポインタであること。
  • rinfo->clause が NULL でないことが前提となる（Const 型チェックの前にアクセスするため）。

アルゴリズムまたは振る舞い

以下の3つの条件をすべて満たす場合に true を返す： 1. rinfo->clause が Const ノードであること。 2. その Const ノードが NULL 値ではないこと（constisnull が偽）。 3. その Const ノードの値が真であること（DatumGetBool による判定）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/restrictinfo.c 内の複数の箇所で、制約条件の最適化やフィルタリングの判定に使用されている。
  • **呼び出し先**: IsA マクロ、DatumGetBool マクロ。

関連データ構造

  • RestrictInfo: クエリプランナが使用する制約条件のラッパー構造体。
  • Const: PostgreSQL の定数ノード。

失敗モードと境界条件

  • rinfo->clause が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • Const 以外のノード型（OpExpr 等）が clause に格納されている場合、IsA チェックにより安全に false が返される。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスを考慮して頻繁に呼び出される箇所での利用を想定している。
  • RestrictInfo の clause フィールドが定数かどうかを判定する唯一の手段ではないが、定数「真」を識別するための標準的なユーティリティである。
