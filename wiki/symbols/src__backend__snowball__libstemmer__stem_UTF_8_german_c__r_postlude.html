r_postlude

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_german.c:285

目的

ステミング処理の最終段階として、単語内の特殊文字や特定の接尾辞パターンを正規化・調整する。

シグネチャまたは形状

static int r_postlude(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z （ステミング環境の状態を保持する構造体）
  • **出力**: 成功時は 1 を返し、失敗時（slice_from_s や skip_utf8 のエラー）は負の値を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • find_among を用いて a_1 テーブル（6要素）に基づいたマッチングを行う。

アルゴリズムまたは振る舞い

1. 無限ループ内で find_among を呼び出し、現在のカーソル位置から一致するパターンを検索する。 2. 一致した among_var に応じて以下の処理を行う：

  • 1〜4: slice_from_s を使用して、対応する文字列（s_6〜s_9）で置換する。
  • 5: skip_utf8 を使用して、UTF-8 文字を 1 つスキップする。

    • 3. マッチしなくなった時点でループを終了し、1 を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内では明示されていないが、german_UTF_8_stem の後処理として呼び出される設計である。
  • **呼び出し先**: find_among, slice_from_s, skip_utf8（いずれも header.h 経由のランタイム関数）。

関連データ構造

  • struct SN_env: ステミングの実行状態（カーソル位置、文字列バッファ等）を管理する。
  • a_1: r_postlude 内で使用されるマッチングテーブル。

失敗モードと境界条件

  • slice_from_s が負の値を返した場合、その値をそのまま呼び出し元へ伝播させる。
  • skip_utf8 が失敗した場合、lab0 ラベルへジャンプし、カーソル位置を復元してループを終了する。

  • among_var の各ケースは、ドイツ語のステミングにおける最終的な文字変換ルールに対応している。
