PrepareSortSupportFromIndexRel

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/sortsupport.c:160

目的

インデックスリレーション（indexRel）の演算子クラス情報に基づき、ソートサポート構造体（SortSupport）を初期化する。

シグネチャまたは形状

void PrepareSortSupportFromIndexRel(Relation indexRel, bool reverse, SortSupport ssup)

入力と出力

  • **入力**:
  • indexRel: ソート対象のインデックスリレーション。
  • reverse: ソート順を反転させるかどうかを示すフラグ。
  • ssup: 初期化対象の SortSupportData 構造体（ssup_attno が設定済みであること）。
  • **出力**: なし（ssup 構造体の内部フィールドが更新される）。

前提条件と不変条件

  • ssup->comparator が NULL であること（Assert により検証）。
  • indexRel がソート可能なアクセスメソッド（amcanorder が真）であること。

アルゴリズムまたは振る舞い

1. ssup->ssup_attno を用いて、indexRel から該当属性の演算子ファミリー（opfamily）と入力型（opcintype）を取得する。 2. indexRel がソートをサポートしていない場合、エラーを発生させる。 3. ssup->ssup_reverse に reverse 引数を設定する。 4. FinishSortSupportFunction を呼び出し、演算子ファミリーから最適な比較関数を検索・設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtsort.c
  • src/backend/utils/sort/tuplesortvariants.c
  • **呼び出し先**:
  • FinishSortSupportFunction

関連データ構造

  • Relation: インデックスのメタデータ保持に使用。
  • SortSupport: ソート処理の状態と比較ロジックを保持する構造体。

失敗モードと境界条件

  • indexRel->rd_indam->amcanorder が偽の場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は ssup->ssup_attno が既に設定されていることを前提としているため、呼び出し前に必ず属性番号が正しくセットされているか確認すること。
