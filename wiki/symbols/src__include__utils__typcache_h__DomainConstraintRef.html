DomainConstraintRef

定義

  • 種別: struct
  • 定義箇所: src/include/utils/typcache.h:165

目的

ドメイン型に対する制約チェックに必要な情報を、バックエンドの生存期間中または特定のメモリコンテキスト内で保持するための参照構造体です。

シグネチャまたは形状

typedef struct DomainConstraintRef
{
	List	   *constraints;	/* DomainConstraintState ノードのリスト */
	MemoryContext refctx;		/* 本構造体を保持するメモリコンテキスト */
	TypeCacheEntry *tcache;		/* ドメイン型の型キャッシュエントリ */
	bool		need_exprstate; /* check_exprstate が必要か否か */

	/* 管理用データ（typcache.c 内部でのみ使用） */
	DomainConstraintCache *dcc; /* 現在の制約、存在しない場合は NULL */
	MemoryContextCallback callback; /* 終了時に参照カウントを解放するためのコールバック */
} DomainConstraintRef;

入力と出力

  • 入力: InitDomainConstraintRef を呼び出す際、対象の型ID（Oid）と、本構造体を割り当てる MemoryContext が必要です。
  • 出力: 初期化された DomainConstraintRef 構造体。

前提条件と不変条件

  • Management data として定義されたフィールド（dcc, callback）は、typcache.c の内部実装専用であり、外部から直接操作してはなりません。
  • 本構造体は、指定された MemoryContext の生存期間に依存します。

アルゴリズムまたは振る舞い

1. **初期化**: InitDomainConstraintRef により、指定された型IDに対応するドメイン制約情報を取得し、構造体に格納します。 2. **更新**: UpdateDomainConstraintRef を通じて、キャッシュされた制約情報の更新が行われます。 3. **ライフサイクル**: MemoryContextCallback を利用して、コンテキストの削除やリセット時に参照カウントの解放処理が自動的に実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExpr.c: ドメイン制約の評価時に使用。
  • src/backend/utils/adt/domains.c: ドメイン関連の処理で使用。
  • **呼び出し先**:
  • src/backend/utils/cache/typcache.c: InitDomainConstraintRef および UpdateDomainConstraintRef が実装されており、本構造体の管理ロジックを提供します。

関連データ構造

  • TypeCacheEntry: ドメイン型のキャッシュエントリを保持します。
  • DomainConstraintCache: 実際の制約情報を保持する内部構造体。
  • DomainConstraintState: constraints リストに含まれる個別の制約状態。

失敗モードと境界条件

  • 指定された Oid がドメイン型でない場合や、制約が存在しない場合の挙動は typcache.c の実装に依存します。
  • 参照カウントの解放処理が適切に行われない場合、メモリリークが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は typcache.c の内部管理データを含んでいるため、フィールドの変更を行う際は typcache.c 側のコールバック処理（callback）との整合性に注意してください。
  • src/backend/utils/adt/domains.c には、参照解放の仕組みが未対応である旨のコメントが存在するため、利用時には最新のライフサイクル管理方針を確認してください。
