SPI_commit_and_chain

定義

  • 種別: function
  • 定義箇所: src/backend/executor/spi.c:325

目的

現在のトランザクションをコミットし、即座に新しいトランザクションを開始してトランザクションチェーンを継続する。

シグネチャまたは形状

void SPI_commit_and_chain(void)

入力と出力

  • 入力: なし
  • 出力: なし（成功時）

前提条件と不変条件

  • SPI コンテキスト内から呼び出される必要がある。
  • トランザクション制御が許可されているコンテキスト（例: プロシージャ内）であること。

アルゴリズムまたは振る舞い

内部関数 _SPI_commit(true) を呼び出す。引数 true は、コミット後に新しいトランザクションを開始する（チェーンする）ことを示す。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpgsql/src/pl_exec.c:4959
  • 呼び出し先: _SPI_commit（内部関数）

関連データ構造

  • src/backend/executor/spi.c 内の SPI 内部スタック管理構造（_SPI_stack 等）がトランザクション境界で更新される。

失敗モードと境界条件

  • トランザクション制御が許可されていないコンテキストで呼び出された場合、PostgreSQL のトランザクション管理機構によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は _SPI_commit のラッパーであり、チェーン機能の有効化をハードコードしている。
  • 類似の関数として SPI_rollback_and_chain が存在する。
