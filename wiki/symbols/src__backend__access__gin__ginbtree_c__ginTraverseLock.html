ginTraverseLock

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginbtree.c:38

目的

GINインデックスのページトラバーサルにおいて、リーフページ到達時に必要に応じて共有ロックから排他ロックへ昇格させ、最終的なロック状態を管理する。

シグネチャまたは形状

int ginTraverseLock(Buffer buffer, bool searchMode)

入力と出力

  • **入力**:
  • buffer: 操作対象のページバッファ。
  • searchMode: 検索モードか否かを示すフラグ。
  • **出力**:
  • int: 最終的に適用されたロックモード（GIN_SHARE または GIN_EXCLUSIVE）。

前提条件と不変条件

  • 呼び出し時点で、対象の buffer はロックされていない状態であること。
  • searchMode が false の場合、リーフページに対して排他ロックの取得を試みる。
  • ロック昇格中にページがリーフから非リーフへ変化する可能性があるため、昇格後に再チェックを行う。

アルゴリズムまたは振る舞い

1. 初期状態で buffer に対して GIN_SHARE ロックを取得する。 2. ページがリーフページ (GinPageIsLeaf) かつ searchMode が false の場合：

  • 一度 GIN_UNLOCK でロックを解除し、GIN_EXCLUSIVE で再ロックする。
  • 再ロック後、ページが依然としてリーフであることを確認する。
  • もしリーフでなくなっていた場合（ルートの分割等による）、再度 GIN_UNLOCK し、GIN_SHARE に戻す。

    • 3. 最終的なロックモードを戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginbtree.c:107 (推測される文脈: ツリー降下処理中)
  • **呼び出し先**:
  • LockBuffer (PostgreSQLストレージマネージャ)
  • BufferGetPage
  • GinPageIsLeaf

関連データ構造

  • Buffer: PostgreSQLのバッファ管理構造体。
  • Page: ページヘッダ情報。

失敗モードと境界条件

  • **ルートの非リーフ化**: ロック昇格の試行中にページ構造が変化し、リーフではなくなるケース。この場合、排他ロックを諦め共有ロックにフォールバックする。

保守担当者向けメモ

  • 本関数はロックカップリングの一環として使用される。
  • GIN_EXCLUSIVE への昇格は、リーフページに対する書き込み操作を前提としている。
  • 昇格後の再チェックは、並行処理によるツリー構造の変化に対する整合性維持のために不可欠である。
