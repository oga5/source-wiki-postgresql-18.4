lz4_decompress_datum

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/common/toast_compression.c:181

目的

LZ4アルゴリズムで圧縮されたTOASTデータ（varlena構造体）を伸長し、元のデータを復元する。

シグネチャまたは形状

struct varlena *lz4_decompress_datum(const struct varlena *value)

入力と出力

  • **入力**: value - 圧縮されたデータを含むvarlena構造体へのポインタ。
  • **出力**: 伸長されたデータを含むvarlena構造体へのポインタ。メモリはpallocによって確保される。

前提条件と不変条件

  • ビルド時に USE_LZ4 が定義されている必要がある。未定義の場合、本関数は NO_LZ4_SUPPORT() を呼び出しエラーとなる。
  • 入力 value はLZ4形式で圧縮されていることが前提である。

アルゴリズムまたは振る舞い

1. VARDATA_COMPRESSED_GET_EXTSIZE(value) を用いて伸長後のサイズを特定し、必要なメモリを palloc で確保する。 2. LZ4_decompress_safe を呼び出し、圧縮データから伸長を実行する。 3. 伸長結果が負の値（エラー）である場合、ERRCODE_DATA_CORRUPTED を伴うエラーを発生させる。 4. 正常終了時、SET_VARSIZE を用いて伸長後のサイズをヘッダーに設定し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/detoast.c:487
  • src/backend/access/common/toast_compression.c:226
  • **呼び出し先**:
  • LZ4_decompress_safe (外部ライブラリ関数)
  • palloc (PostgreSQLメモリ管理)
  • ereport (エラー報告)

関連データ構造

  • varlena: PostgreSQLの可変長データ型を保持する構造体。

失敗モードと境界条件

  • **LZ4未サポート**: USE_LZ4 が無効な環境で呼び出された場合、NO_LZ4_SUPPORT() によりエラーとなる。
  • **データ破損**: LZ4_decompress_safe が負の値を返した場合、圧縮データが破損していると判断し、ERROR レベルで処理を中断する。

保守担当者向けメモ

  • 本関数は varlena ヘッダーの圧縮メタデータに依存している。
  • LZ4のバージョンやビルド設定に強く依存するため、環境移行時には注意が必要である。
