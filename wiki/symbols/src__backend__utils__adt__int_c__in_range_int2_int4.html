in_range_int2_int4

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/int.c:732

目的

ウィンドウ関数のフレーム境界計算において、int2 型の値を int4 型のオフセットを用いて範囲内にあるか判定する。

シグネチャまたは形状

Datum in_range_int2_int4(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_INT16(0): 比較対象の値 (val)
  • PG_GETARG_INT16(1): 基準値 (base)
  • PG_GETARG_INT32(2): オフセット値 (offset)
  • PG_GETARG_BOOL(3): 減算フラグ (sub)
  • PG_GETARG_BOOL(4): 比較方向フラグ (less)
  • **出力**:
  • bool: 範囲内であれば true、そうでなければ false を Datum として返す。

前提条件と不変条件

  • すべての算術演算は int32 型にキャストして実行される。
  • offset が負の値である場合、ERRCODE_INVALID_PRECEDING_OR_FOLLOWING_SIZE エラーを発生させる。
  • sub が真の場合、offset は負数として扱われる。

アルゴリズムまたは振る舞い

1. 引数を int32 に変換し、offset の正当性を検証する。 2. sub が真であれば offset を反転させる。 3. pg_add_s32_overflow を使用して base + offset を計算する。 4. オーバーフローが発生した場合:

  • sub が偽なら val は常に sum より小さいとみなし、less の値を返す。
  • sub が真なら val は常に sum より大きいとみなし、!less の値を返す。

    • 5. オーバーフローが発生しない場合:

  • less が真なら val <= sum を判定する。
  • less が偽なら val >= sum を判定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/int.c:771 (DirectFunctionCall5 経由)
  • **呼び出し先**:
  • pg_add_s32_overflow (算術オーバーフローチェック)
  • ereport (エラー通知)

関連データ構造

  • Datum: PostgreSQL の汎用データ型表現。
  • PG_FUNCTION_ARGS: SQL 関数呼び出し規約のための引数コンテナ。

失敗モードと境界条件

  • offset < 0: ウィンドウ関数の仕様違反としてエラー終了。
  • 算術オーバーフロー: pg_add_s32_overflow により検出され、論理的な比較結果（less または !less）を返して安全に処理される。

保守担当者向けメモ

  • 本関数は int2 と int4 の組み合わせ専用である。他の型との組み合わせについては、対応する in_range_* 関数を参照すること。
  • 算術演算時のオーバーフローは未定義動作を避けるため、必ず common/int.h の関数を経由すること。
