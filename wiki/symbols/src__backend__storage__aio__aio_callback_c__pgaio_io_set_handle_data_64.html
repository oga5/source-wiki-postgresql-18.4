pgaio_io_set_handle_data_64

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_callback.c:121

目的

非同期I/O操作（PgAioHandle）に関連付けられた補助データを、共有メモリ領域に書き込みます。

シグネチャまたは形状

void pgaio_io_set_handle_data_64(PgAioHandle *ioh, uint64 *data, uint8 len)

入力と出力

  • **入力**:
  • ioh: 対象となる PgAioHandle 構造体へのポインタ。
  • data: 書き込む64ビットデータの配列。
  • len: 書き込むデータの要素数。
  • **出力**: なし（副作用として共有メモリ pgaio_ctl->handle_data を更新）。

前提条件と不変条件

  • ioh->state が PGAIO_HS_HANDED_OUT であること。
  • ioh->handle_data_len が 0 であること（1つのI/Oにつき1回のみ設定可能）。
  • len が PG_IOV_MAX 以下であること。
  • len が io_max_combine_limit 以下であること。

アルゴリズムまたは振る舞い

1. 引数で指定された len 分のデータを、ioh->iovec_off をオフセットとして共有メモリ pgaio_ctl->handle_data にコピーします。 2. ioh->handle_data_len に len を代入し、データが設定済みであることを記録します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は特定されていませんが、src/include/storage/aio.h で外部公開されています。
  • **呼び出し先**: 内部で呼び出している関数はありません。

関連データ構造

  • PgAioHandle: I/O操作の状態とメタデータを保持する構造体。
  • pgaio_ctl: 共有メモリ上のAIO制御構造体。

失敗モードと境界条件

  • 前提条件（Assert）に違反した場合、プロセスはアボートします。
  • メモリ消費を抑えるため、一度設定したデータの再設定は許可されません。

保守担当者向けメモ

  • 本関数は START_CRIT_SECTION() 内で実行される可能性があるため、処理は軽量である必要があります。
  • pgaio_io_set_handle_data_32 は本関数のラッパーとして実装されています。
