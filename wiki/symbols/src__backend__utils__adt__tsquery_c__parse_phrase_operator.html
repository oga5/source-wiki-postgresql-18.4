parse_phrase_operator

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery.c:164

目的

tsquery の解析中に、<N> 形式のフレーズ演算子（距離指定）を構文解析し、その距離値を抽出します。

シグネチャまたは形状

static bool parse_phrase_operator(TSQueryParserState pstate, int16 *distance)

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する TSQueryParserState 構造体。buf メンバから現在の解析位置を読み取ります。
  • distance: 抽出された距離値を格納するための int16 型ポインタ。
  • **出力**:
  • 成功時は true を返し、distance に解析した値を設定し、pstate->buf を演算子終了位置まで進めます。
  • 失敗時は false を返します。

前提条件と不変条件

  • pstate->buf は、フレーズ演算子の開始位置（<）を指している必要があります。
  • 距離値は 0 から MAXENTRYPOS の範囲内である必要があります。この範囲外の場合はエラーが報告されます。

アルゴリズムまたは振る舞い

1. 状態遷移マシン（PHRASE_OPEN → PHRASE_DIST → PHRASE_CLOSE → PHRASE_FINISH）を用いて文字列を走査します。 2. < で開始し、オプションの整数（距離）を読み取ります。整数が省略された場合のデフォルト値は 1 です。 3. - を経由して > で終了する形式を検証します。 4. 正常終了時に distance へ値を書き込み、pstate->buf を更新します。 5. 解析中に構文エラーや範囲外の数値が検出された場合、false を返すか、escontext を介してエラーを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsquery.c 内のクエリ解析ロジック（364行目付近）から呼び出されます。
  • **呼び出し先**: strtol（標準ライブラリ）、t_iseq（マクロ）、ereturn（エラーハンドリング用マクロ）。

関連データ構造

  • TSQueryParserState: 解析中のバッファ位置やエラーコンテキストを保持する構造体。

失敗モードと境界条件

  • **構文エラー**: < や > の欠落、または不正な文字が含まれる場合は false を返します。
  • **数値範囲外**: 距離が負の値、または MAXENTRYPOS を超える場合は ERRCODE_INVALID_PARAMETER_VALUE エラーを発生させます。
  • **解析失敗**: strtol が数値を認識できない場合は false を返します。

保守担当者向けメモ

  • 本関数は tsquery の構文解析における再帰的な処理の一部として機能します。
  • escontext を使用したエラー報告を行っているため、呼び出し元でソフトエラーとしてハンドリング可能です。
  • 距離値の最大値は MAXENTRYPOS に依存するため、将来的な仕様変更時はこの定数を確認してください。
