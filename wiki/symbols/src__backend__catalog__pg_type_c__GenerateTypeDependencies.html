GenerateTypeDependencies

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_type.c:556

目的

PostgreSQLのデータ型（pg_type）に関連する依存関係をカタログ（pg_depend 等）に記録または再構築します。

シグネチャまたは形状

void GenerateTypeDependencies(HeapTuple typeTuple,
                              Relation typeCatalog,
                              Node *defaultExpr,
                              void *typacl,
                              char relationKind,
                              bool isImplicitArray,
                              bool isDependentType,
                              bool makeExtensionDep,
                              bool rebuild)

入力と出力

  • **入力**:
  • typeTuple: 対象型の pg_type 行タプル。
  • typeCatalog: pg_type カタログのリレーション。
  • defaultExpr, typacl: 省略可能なデフォルト式およびACL情報。未指定時はタプルから抽出。
  • relationKind, isImplicitArray, isDependentType, makeExtensionDep: 依存関係の性質を決定するフラグ。
  • rebuild: 既存の依存関係を削除して再構築するかを指定するフラグ。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • rebuild が true の場合、既存の依存関係レコード（共有依存関係を含む）が削除されます。
  • isDependentType が指定された場合、名前空間や所有者への依存関係記録はスキップされます（マルチレンジ型を除く）。

アルゴリズムまたは振る舞い

1. **初期化**: defaultExpr や typacl が未提供の場合、タプルから取得します。 2. **クリーンアップ**: rebuild が真であれば、deleteDependencyRecordsFor 等を用いて既存の依存関係を削除します。 3. **依存関係の収集**:

  • 名前空間、所有者、ACLへの依存関係を記録。
  • I/O関数（typinput, typoutput 等）、サブスクリプト関数、基底型、照合順序（collation）への依存関係を addrs_normal に収集。

    • 4. **記録**: 収集した依存関係を record_object_address_dependencies で一括記録します。 5. **式とリレーション**: デフォルト式への依存を記録し、型がリレーションに関連する場合は typrelid を通じて内部依存関係を構築します。 6. **配列型**: 暗黙的に作成された配列型の場合、要素型への内部依存関係を記録します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/pg_type.c 内の型作成処理、および src/backend/commands/typecmds.c の ALTER TYPE 等のコマンド処理。
  • **呼び出し先**: dependency.c（recordDependencyOn 等）、heap.c（heap_getattr）、catalog.c 等のカタログ操作関数群。

関連データ構造

  • Form_pg_type: pg_type カタログの構造体。
  • ObjectAddress, ObjectAddresses: 依存関係対象を保持する構造体。

失敗モードと境界条件

  • rebuild フラグの誤用により、必要な依存関係が消失するリスクがあります。
  • typrelid が有効な場合、relationKind に応じて依存の向き（型→リレーション、またはリレーション→型）が切り替わります。

保守担当者向けメモ

  • 依存関係の再構築（rebuild）は、ALTER TYPE 等で定義が変更された際の整合性維持に不可欠です。
  • 拡張機能（Extension）への依存関係は、makeExtensionDep が真の場合にのみ記録されます。
