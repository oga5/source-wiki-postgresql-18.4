replace_outer_placeholdervar

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/paramassign.c:197

目的

上位レベルのクエリで定義された PlaceHolderVar を、現在のプランレベルで参照可能な PARAM_EXEC 型のパラメータノードに変換します。

シグネチャまたは形状

Param *replace_outer_placeholdervar(PlannerInfo *root, PlaceHolderVar *phv)

入力と出力

  • **入力**:
  • root: 現在のクエリレベルの PlannerInfo 構造体。
  • phv: 置換対象の PlaceHolderVar 構造体。
  • **出力**:
  • Param: PARAM_EXEC として設定された新しいパラメータノード。

前提条件と不変条件

  • phv->phlevelsup > 0 かつ phv->phlevelsup < root->query_level であること（上位レベルの変数であること）。
  • assign_param_for_placeholdervar を介して、対象の PlaceHolderVar が root->glob->paramExecTypes に登録済み、または登録可能であること。

アルゴリズムまたは振る舞い

1. assign_param_for_placeholdervar を呼び出し、対象の PlaceHolderVar に対応する PARAM_EXEC ID を取得（または新規割り当て）します。 2. makeNode(Param) で新しい Param ノードを生成します。 3. paramkind を PARAM_EXEC に設定します。 4. paramid に手順1で取得した ID を格納します。 5. phv->phexpr の型情報（型、typmod、照合順序）を抽出し、Param ノードの paramtype, paramtypmod, paramcollid に設定します。 6. location を -1 に設定し、生成した Param ノードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/subselect.c (サブクエリ処理に関連する箇所で呼び出されています)
  • **呼び出し先**:
  • assign_param_for_placeholdervar (パラメータIDの割り当て)
  • makeNode (ノード生成)
  • exprType, exprTypmod, exprCollation (式情報の抽出)

関連データ構造

  • PlaceHolderVar: 置換対象のプレースホルダ変数。
  • Param: 生成される実行時パラメータノード。
  • PlannerInfo: クエリ全体のプランニング情報を保持する構造体。

失敗モードと境界条件

  • phv が現在のクエリレベル（phlevelsup == 0）である場合、Assert により停止します。
  • assign_param_for_placeholdervar が失敗した場合、その呼び出し元でエラーが発生します。

保守担当者向けメモ

  • 本関数は PlaceHolderVar 専用です。通常の Var の置換には replace_outer_var を使用してください。
  • PARAM_EXEC ID の割り当ては root->glob->paramExecTypes に依存しており、プラン全体で一貫性が保たれる必要があります。
