r_prelude

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_french.c:444

目的

フランス語のステミング処理における前処理として、単語内の特定の文字パターン（母音に挟まれた 'u', 'i', 'y' や、'qu' など）を置換し、正規化を行う。

シグネチャまたは形状

static int r_prelude(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境構造体。文字列バッファ p とカーソル位置 c を含む）
  • 出力: 成功時に 1、失敗（メモリ不足や不正な操作）時に負の整数を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • g_v（母音グループ）が定義されている必要がある。

アルゴリズムまたは振る舞い

1. 文字列全体を走査し、以下のパターンを順次検索・置換する：

  • 母音に挟まれた 'u' を 'U' に置換。
  • 母音に挟まれた 'i' を 'I' に置換。
  • 母音に挟まれた 'y' を 'Y' に置換。
  • 特定の文字列（s_4, s_6）を別の文字列（s_5, s_7）に置換。
  • 母音に続く 'y' を 'Y' に置換。
  • 'qu' を 'QU' に置換。

    • 2. マッチしない文字は skip_utf8 を用いてカーソルを前進させ、文字列の末尾に到達するまで繰り返す。

呼び出し元と呼び出し先

  • 呼び出し元: french_UTF_8_stem（直接の言及はないが、ファイル概要の制御フローに基づく）。
  • 呼び出し先: in_grouping_U, slice_from_s, eq_s, skip_utf8（いずれも header.h で提供されるランタイム関数）。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • among 構造体配列（a_0 ～ a_8）: 接尾辞マッチングに使用されるデータ構造。

失敗モードと境界条件

  • slice_from_s が負の値を返した場合、そのエラーコードをそのまま呼び出し元へ伝播させる。
  • 文字列の末尾（z->l）に達した際の境界チェックが各ステップで行われる。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによって自動生成されたコードである。ロジックの変更が必要な場合は、直接編集せず Snowball のソース言語（.sbl）を修正すること。
  • デバッグ時は SN_env の p（文字列）と c（カーソル）の状態を追跡すること。
