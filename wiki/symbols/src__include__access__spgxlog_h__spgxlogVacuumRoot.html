spgxlogVacuumRoot

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgxlog.h:225

目的

SP-GiST インデックスにおいて、ルートページがリーフページを兼ねている場合に、そのページ内のタプルを削除する操作を記録するための WAL レコード構造体です。

シグネチャまたは形状

typedef struct spgxlogVacuumRoot
{
    uint16      nDelete;
    spgxlogState stateSrc;
    OffsetNumber offsets[FLEXIBLE_ARRAY_MEMBER];
} spgxlogVacuumRoot;

入力と出力

  • **入力**: 削除対象のタプル数 (nDelete)、インデックス状態 (stateSrc)、および削除対象タプルのオフセット配列 (offsets)。
  • **出力**: WAL ログへの書き込みデータ。可変長配列 offsets を含む構造体としてシリアライズされます。

前提条件と不変条件

  • offsets は FLEXIBLE_ARRAY_MEMBER として定義されており、構造体の末尾に配置されます。
  • 構造体の固定サイズ部分は SizeOfSpgxlogVacuumRoot マクロで算出可能です。
  • 本構造体は、リカバリ時に spg_redo 関数によって読み取られ、対象ページ上のアイテムを更新するために使用されます。

アルゴリズムまたは振る舞い

1. インデックスのバキューム処理中に、ルートかつリーフであるページから不要なタプルを削除する際に本構造体がインスタンス化されます。 2. nDelete に削除するタプル数を設定し、offsets 配列に各タプルのオフセット番号を格納します。 3. XLogInsert を介して WAL に書き出されます。 4. リカバリ時には、spg_redo がこの構造体を読み取り、指定されたオフセットのタプルをページから削除することで物理的な状態を復元します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgvacuum.c 内のバキューム処理ロジックから初期化されます。
  • **呼び出し先**: src/backend/access/rmgrdesc/spgdesc.c（ログ出力用）および src/backend/access/spgist/spgxlog.c（Redo 処理用）で参照されます。

関連データ構造

  • spgxlogState: リカバリに必要なインデックスの最小限の状態を保持します。
  • OffsetNumber: storage/off.h で定義される、ページ内のタプル位置を指す型です。

失敗モードと境界条件

  • 可変長データを含むため、ポインタ演算や memcpy を行う際は SizeOfSpgxlogVacuumRoot マクロを基準にする必要があります。
  • アライメントされていないデータが含まれる可能性があるため、アクセス時には注意が必要です。

保守担当者向けメモ

  • SizeOfSpgxlogVacuumRoot マクロが offsetof(spgxlogVacuumRoot, offsets) として定義されているため、構造体のサイズ計算には必ずこれを使用してください。
  • 本構造体は spgxlog.c の Redo ロジックと密接に結合しているため、構造体のメンバを変更する場合は spg_redo 側の実装も更新する必要があります。
