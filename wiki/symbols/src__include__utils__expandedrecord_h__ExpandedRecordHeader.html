ExpandedRecordHeader

定義

  • 種別: struct
  • 定義箇所: src/include/utils/expandedrecord.h:42

目的

複合型（Composite Type）やレコード型の値をメモリ上で効率的に操作・変更するためのデータ構造を提供します。フラットなタプル形式と、個別に管理されるフィールド配列（Datum[]）の双方向変換を管理します。

シグネチャまたは形状

ExpandedObjectHeader を継承した構造体であり、以下の主要フィールドを持ちます：

  • er_magic: デバッグ用識別子
  • flags: ER_FLAG_* ビットフラグ（状態管理）
  • er_decltypeid, er_typeid, er_typmod: 型情報
  • er_tupdesc: タプル記述子
  • dvalues, dnulls: フィールド値とNULLフラグの配列
  • fvalue: フラットなタプル表現（HeapTuple）
  • er_short_term_cxt: 短期メモリコンテキスト

入力と出力

  • **入力**: 物理的なタプル（HeapTuple）または個別のフィールド値。
  • **出力**: Datum 型としてのレコード表現、または ER_flatten_into を介した物理タプル形式への変換。

前提条件と不変条件

  • er_magic はデバッグ用であり、整合性検証に使用されます。
  • dvalues 配列が有効な場合、ER_FLAG_DVALUES_VALID フラグがセットされます。
  • er_tupdesc_id は INVALID_TUPLEDESC_IDENTIFIER にはなりません。
  • ER_FLAGS_NON_DATA に含まれるフラグは、タプルデータ置換時にもクリアされません。

アルゴリズムまたは振る舞い

1. **遅延デコード**: フィールドアクセス時、dvalues が無効な場合はタプルのデコードが発生します。 2. **メモリ管理**: 独自のメモリコンテキスト内で管理され、MemoryContextCallback を用いてリソースを解放します。 3. **ドメインチェック**: ER_FLAG_IS_DOMAIN がセットされている場合、er_dummy_header や er_domaininfo を使用してドメイン制約の検証を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execExprInterp.c 等で Datum からのキャストにより参照されます。
  • **呼び出し先**: src/backend/utils/adt/expandedrecord.c 内の各種内部関数（get_short_term_cxt, build_dummy_expanded_header 等）が本構造体を操作します。

関連データ構造

  • ExpandedObjectHeader: 基底構造体。
  • TupleDesc: タプルの構造定義。
  • HeapTuple: 物理的なタプル表現。
  • MemoryContext: メモリ管理コンテキスト。

失敗モードと境界条件

  • dvalues が未確保の場合、dvalues および dnulls は NULL となる可能性があります。
  • er_tupdesc が NULL の場合、タプル記述子が存在しない状態を示します。
  • flat_size が 0 の場合、フラットなサイズは計算されていません。

保守担当者向けメモ

  • er_magic はデバッグ用マーカーです。
  • ER_FLAG_FVALUE_VALID が立っていない場合でも、システム列の取得には fvalue が利用可能な場合があります。
  • TransferExpandedRecord を使用してメモリコンテキストの親を切り替えることが可能です。
