array_subscript_fetch_slice

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arraysubs.c:264

目的

配列型に対するスライス取得操作を実行し、結果を式評価コンテキストに格納する。

シグネチャまたは形状

static void array_subscript_fetch_slice(ExprState *state, ExprEvalStep *op, ExprContext *econtext)

入力と出力

  • **入力**:
  • op->d.sbsref.state: 実行時の添字操作状態（SubscriptingRefState）。
  • op->resvalue: 取得対象の配列データ（入力）および結果の格納先（出力）。
  • op->resnull: ヌルフラグ（入力）。
  • **出力**: なし（*op->resvalue を更新）。

前提条件と不変条件

  • *op->resnull が真である場合、本関数は呼び出されない（アサーションにより検証）。
  • スライス操作の結果は決して NULL にならないため、*op->resnull の更新は不要。

アルゴリズムまたは振る舞い

1. op->d.sbsref.state から ArraySubWorkspace を取得する。 2. array_get_slice を呼び出し、配列から指定された範囲のスライスを抽出する。 3. array_get_slice には、ワークスペース内の境界値（upperindex, lowerindex）および型情報（長さ、アライメント等）を渡す。 4. 抽出されたスライス結果を *op->resvalue に書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/arraysubs.c:519（SubscriptingRefMethods 構造体の sbs_fetch フィールドへの代入）。
  • **呼び出し先**:
  • array_get_slice（src/backend/utils/adt/arrayfuncs.c で定義される低レベル関数）。

関連データ構造

  • ExprState, ExprEvalStep, ExprContext: 式評価エンジンで使用される構造体。
  • SubscriptingRefState: 添字操作の実行時状態を保持。
  • ArraySubWorkspace: 添字の境界値や要素の型情報を保持するワークスペース。

失敗モードと境界条件

  • 添字が配列の範囲外である場合や、無効な次元指定が行われた場合の挙動は array_get_slice の実装に依存する。
  • 入力配列が NULL の場合はアサーション違反となる。

保守担当者向けメモ

  • 本関数は SubscriptingRefMethods を介して呼び出されるため、直接的な呼び出し箇所は限定的である。
  • ArraySubWorkspace の各フィールド（refattrlength 等）が正しく初期化されていることを前提としているため、実行計画構築時のセットアップロジックと密接に関連している。
