cfunc_resolve_polymorphic_argtypes

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/funccache.c:347

目的

多相関数（polymorphic function）の引数型を、実行時のコンテキストに基づいて解決または検証する。

シグネチャまたは形状

void cfunc_resolve_polymorphic_argtypes(int numargs, Oid *argtypes, char *argmodes, Node *call_expr, bool forValidator, const char *proname)

入力と出力

  • **入力**:
  • numargs: 引数の数。
  • argtypes: 引数のOID配列（解決対象）。
  • argmodes: 引数のモード（IN/OUT等）の配列。
  • call_expr: 呼び出し式のノード。
  • forValidator: 検証用モードか否かを示すフラグ。
  • proname: エラー報告用関数名。
  • **出力**: なし（argtypes 配列を直接書き換える）。

前提条件と不変条件

  • forValidator が false の場合、標準の多相型解決が成功する必要がある。失敗した場合は ERRCODE_FEATURE_NOT_SUPPORTED でエラーを投げる。

アルゴリズムまたは振る舞い

1. **通常モード (forValidator == false)**:

  • resolve_polymorphic_argtypes を呼び出し、標準的な多相型（ANYELEMENT 等）を解決する。
  • 続いて RECORD 型および RECORDARRAY 型の入力引数を走査し、get_call_expr_argtype を用いて実際の型を特定し、argtypes を更新する。

    • 2. **検証モード (forValidator == true)**:

  • 多相型（ANYELEMENT, ANYARRAY, ANYRANGE 等）を、プレースホルダーとして INT4 系の型に置換する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/funccache.c
  • src/pl/plpgsql/src/pl_comp.c
  • **呼び出し先**:
  • resolve_polymorphic_argtypes
  • get_call_expr_argtype
  • ereport

関連データ構造

  • Oid: PostgreSQLの型識別子。
  • Node: 式ツリーのノード。

失敗モードと境界条件

  • 標準的な多相型解決が失敗した場合、ereport によりエラーが発生する。
  • RECORD 型の解決において、get_call_expr_argtype が有効な型を返さない場合は更新されない。

保守担当者向けメモ

  • 本関数は RECORD 型を多相型と同様に扱うための拡張ロジックを含んでいる。
  • ANYENUMOID の処理には XXX dubious（疑わしい）とのコメントがあるため、変更時は注意が必要。
