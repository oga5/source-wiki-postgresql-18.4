sqlca_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-dynalloc.c:31

目的

ECPG（Embedded SQL for C）において、SQL文の実行結果やエラー状態を保持・伝達するための通信領域（SQL Communication Area）を定義する構造体です。

シグネチャまたは形状

struct sqlca_t {
    char    sqlcaid[8];
    long    sqlabc;
    long    sqlcode;
    struct {
        int     sqlerrml;
        char    sqlerrmc[SQLERRMC_LEN];
    } sqlerrm;
    char    sqlerrp[8];
    long    sqlerrd[6];
    char    sqlwarn[8];
    char    sqlstate[5];
};

入力と出力

  • **入力**: ECPGランタイムライブラリがSQL文の実行結果を各フィールドに書き込みます。
  • **出力**: アプリケーション側が sqlcode や sqlerrd（処理行数など）を参照し、エラーハンドリングや後続処理の制御に使用します。

前提条件と不変条件

  • sqlerrd[2] は、INSERT、UPDATE、DELETE 文の実行後に処理された行数を保持します。
  • sqlwarn[0] は、他の警告フラグが一つでもセットされた場合に 'W' に設定されます。
  • sqlwarn[1] は、ホスト変数への格納時に文字列の切り詰めが発生した場合に 'W' に設定されます。

アルゴリズムまたは振る舞い

各SQL文の実行後、ECPGランタイムは本構造体の各フィールドを更新します。アプリケーションは sqlcode をチェックして成功・失敗を判断し、必要に応じて sqlprint() 等のヘルパー関数を用いてエラー情報を出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPGget_sqlca() を介して、src/interfaces/ecpg/ecpglib/ 配下の各関数（connect.c, descriptor.c, data.c 等）および src/interfaces/ecpg/compatlib/informix.c から参照・更新されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • sqlda_t: SQL記述子領域。動的SQLにおいて列情報や値を保持するために併用されます。
  • sqlvar_t: sqlda_t 内で使用される列定義構造体。

失敗モードと境界条件

  • sqlcode が 0 未満の場合、SQL実行エラーを示します。
  • sqlwarn の各要素は、非致命的な通知やデータの切り詰めが発生した際の境界条件を通知します。
  • sqlerrmc は SQLERRMC_LEN で定義されたサイズ制限があり、エラーメッセージが長すぎる場合は切り詰められる可能性があります。

保守担当者向けメモ

  • 本構造体はECPGのランタイムと密接に結合しています。定義を変更する場合は、ecpglib および関連する互換ライブラリとのバイナリ互換性に注意してください。
  • sqlerrd や sqlwarn の各要素のインデックスには特定の意味が割り当てられているため、ソースコード内のコメントを参照して整合性を維持してください。
