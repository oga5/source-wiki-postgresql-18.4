plpgsql_compile_error_callback

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_comp.c:884

目的

PL/pgSQL 関数のコンパイル中にエラーが発生した際、エラーコンテキスト情報（関数名や行番号）を PostgreSQL のエラー報告システムに追加するコールバック関数です。

シグネチャまたは形状

static void plpgsql_compile_error_callback(void *arg)

入力と出力

  • **入力**: arg - struct compile_error_callback_arg 型へのポインタ。スキャナ状態 (yyscanner) およびソーステキスト情報 (proc_source) を保持します。
  • **出力**: なし（副作用として errcontext を呼び出します）。

前提条件と不変条件

  • arg は compile_error_callback_arg 構造体としてキャスト可能である必要があります。
  • plpgsql_error_funcname および plpgsql_latest_lineno() が有効なコンパイル状態を指している必要があります。

アルゴリズムまたは振る舞い

1. 引数 arg を compile_error_callback_arg にキャストし、スキャナを取得します。 2. proc_source が存在する場合、function_parse_error_transpose を呼び出し、構文エラーの位置を元のソーステキスト上の位置へ変換を試みます。変換に成功した場合はそのまま終了します。 3. 構文エラー位置の変換が行われなかった場合、plpgsql_error_funcname と plpgsql_latest_lineno() を使用して、エラーコンテキストに「PL/pgSQL 関数 "..." のコンパイル中、行 N 付近」という情報を追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/pl_comp.c 内のコンパイル処理（plpgsql_compile 等）において、エラーハンドリング用の plerrcontext.callback に設定されます。
  • **呼び出し先**:
  • function_parse_error_transpose
  • errcontext (PostgreSQL エラー報告マクロ)
  • plpgsql_latest_lineno

関連データ構造

  • compile_error_callback_arg: コンパイルエラー時に必要なスキャナ状態とソース情報を保持する構造体。

失敗モードと境界条件

  • proc_source が NULL の場合、構文エラー位置の変換はスキップされ、行番号ベースの報告のみが行われます。
  • plpgsql_error_funcname が設定されていない場合、関数名を含まない形式でエラーコンテキストが報告される可能性があります。

保守担当者向けメモ

  • 本関数は PostgreSQL のエラー報告メカニズムのコールバックとして登録されるため、直接呼び出すのではなく、error_context_stack を通じて間接的に実行されます。
  • 構文エラーの行番号変換ロジック (function_parse_error_transpose) に依存しているため、パーサーの変更時にはこの連携が維持されているか確認してください。
