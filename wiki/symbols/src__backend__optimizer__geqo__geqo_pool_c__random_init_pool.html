random_init_pool

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_pool.c:90

目的

遺伝的アルゴリズム（GEQO）で使用する個体群（Pool）をランダムな初期個体で満たします。評価不能な個体を排除し、有効な個体のみでプールを構成します。

シグネチャまたは形状

void random_init_pool(PlannerInfo *root, Pool *pool)

入力と出力

  • **入力**:
  • root: クエリ最適化のコンテキスト情報。
  • pool: 初期化対象の Pool 構造体（個体群データとサイズ情報を含む）。
  • **出力**: なし（pool->data 内の各染色体の string および worth が更新されます）。

前提条件と不変条件

  • pool->data は既にメモリ確保済みであること。
  • geqo_eval が返すコスト値において、DBL_MAX は評価不能（無効）な個体を意味します。
  • 10,000回連続して有効な個体が生成できない場合、システムエラーとして処理を中断します。

アルゴリズムまたは振る舞い

1. プールのサイズ分だけループし、各個体に対して init_tour でランダムな初期値を生成します。 2. geqo_eval を呼び出し、個体の適応度（コスト）を算出します。 3. コストが DBL_MAX 未満であれば有効な個体として採用し、インデックスを進めます。 4. DBL_MAX であれば無効な個体としてカウントし、再試行します。 5. 10,000回試行しても有効な個体が生成できない場合は elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_main.c
  • **呼び出し先**:
  • init_tour (染色体の初期化)
  • geqo_eval (適応度の評価)
  • elog (エラー報告)

関連データ構造

  • Pool: 個体群を管理する構造体。
  • Chromosome: 個体情報を保持する構造体。
  • PlannerInfo: クエリプランナーのコンテキスト。

失敗モードと境界条件

  • **無限ループ防止**: 10,000回連続で無効な個体が生成された場合、ERROR をスローして停止します。
  • **無効な個体の排除**: 評価値が DBL_MAX となる個体はプールに格納されず、破棄されます。

保守担当者向けメモ

  • 本関数は D. Whitley の Genitor アルゴリズムに基づいています。
  • GEQO_DEBUG が定義されている場合、無効な個体が生成された回数がデバッグログに出力されます。
