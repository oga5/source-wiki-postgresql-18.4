GetQuitSignalReason

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/pmsignal.c:212

目的

Postmasterから送信されたシステムシャットダウンの理由（QuitSignalReason）を取得します。

シグネチャまたは形状

QuitSignalReason GetQuitSignalReason(void);

入力と出力

  • **入力**: なし
  • **出力**: QuitSignalReason 型の値。シャットダウン理由が設定されていない場合は PMQUIT_NOT_SENT を返します。

前提条件と不変条件

  • シグナルハンドラ内から呼び出されることを想定しており、安全性を考慮した実装になっています。
  • IsUnderPostmaster が偽であるか、共有メモリ上の PMSignalState が初期化されていない（NULL）場合は、理由を特定できないものとして PMQUIT_NOT_SENT を返します。

アルゴリズムまたは振る舞い

1. プロセスがPostmaster配下にあるか、および PMSignalState が有効かを確認します。 2. 条件を満たさない場合は PMQUIT_NOT_SENT を返します。 3. 条件を満たす場合、共有メモリ上の PMSignalState->sigquit_reason に格納されている値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/postgres.c:2977
  • **呼び出し先**:
  • 共有メモリ上の PMSignalState 構造体（直接参照）

関連データ構造

  • PMSignalData: 共有メモリ上に配置される構造体で、sigquit_reason を保持します。
  • QuitSignalReason: シャットダウン理由を表す列挙型。

失敗モードと境界条件

  • PMSignalState が未初期化の場合、安全に PMQUIT_NOT_SENT を返すことでクラッシュを回避します。

保守担当者向けメモ

  • 本関数はシグナルハンドラ内での実行が想定されているため、ロックの取得や複雑な処理は避けてください。
  • PMSignalState の更新は SetQuitSignalReason によって行われます。
