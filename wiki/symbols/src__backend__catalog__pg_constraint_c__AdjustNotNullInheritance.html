AdjustNotNullInheritance

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_constraint.c:741

目的

指定されたテーブル列に対する既存の NOT NULL 制約の継承状態（coninhcount）やローカル定義フラグ（conislocal）を更新する。

シグネチャまたは形状

bool AdjustNotNullInheritance(Oid relid, AttrNumber attnum, const char *new_conname,
                              bool is_local, bool is_no_inherit, bool is_notvalid)

入力と出力

  • **入力**:
  • relid: 対象テーブルの OID。
  • attnum: 対象列の属性番号。
  • new_conname: 新規定義時に指定された制約名（検証用）。
  • is_local: ローカル定義として追加されるか否か。
  • is_no_inherit: NO INHERIT フラグの要求状態。
  • is_notvalid: NOT VALID フラグの要求状態。
  • **出力**:
  • bool: 対象の NOT NULL 制約が見つかり、処理が行われた場合は true、見つからなかった場合は false を返す。

前提条件と不変条件

  • 対象の pg_constraint 行に対して RowExclusiveLock を取得する。
  • 既存の制約の connoinherit 属性と is_no_inherit 引数が一致しない場合、エラーを発生させる。
  • 既存の制約が NOT VALID であり、かつ呼び出し元が有効な制約を求めている場合、エラーを発生させる。

アルゴリズムまたは振る舞い

1. findNotNullConstraintAttnum を使用して、指定された列の NOT NULL 制約を検索する。 2. 制約が存在しない場合は false を返して終了する。 3. 存在する場合、pg_constraint カタログを RowExclusiveLock で開く。 4. is_local が false（継承による追加）の場合、coninhcount をインクリメントする。 5. is_local が true で、かつ既存の制約がローカル定義でない場合、conislocal を true に設定する。 6. 変更が発生した場合、CatalogTupleUpdate を用いてカタログを更新し、テーブルを閉じて true を返す。

失敗モードと境界条件

  • **エラー**:
  • NO INHERIT 状態の不一致。
  • NOT VALID 制約に対する有効性要求の競合。
  • ローカル定義時の制約名不一致。
  • coninhcount のオーバーフロー（too many inheritance parents）。

保守担当者向けメモ

  • 本関数は NOT NULL 制約が pg_attribute ではなく pg_constraint で管理されていることに依存している。
  • 継承関係の複雑な更新を伴うため、カタログ更新時のロック順序と整合性に注意すること。
