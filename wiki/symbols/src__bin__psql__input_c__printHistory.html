printHistory

定義

  • 種別: function
  • 定義箇所: src/bin/psql/input.c:493

目的

readline の履歴リストに保持されているコマンド履歴を、コンソール（ページャ経由）または指定されたファイルへ出力する。

シグネチャまたは形状

bool printHistory(const char *fname, unsigned short int pager)

入力と出力

  • **入力**:
  • fname: 出力先ファイルパス。NULL の場合はコンソールへ出力する。
  • pager: ページャを使用するかどうかのフラグ（0 または 1）。
  • **出力**:
  • 成功時に true、失敗時または履歴機能が無効な場合に false を返す。

前提条件と不変条件

  • USE_READLINE が定義されている環境でのみ機能する。
  • グローバル変数 useHistory が真である必要がある。

アルゴリズムまたは振る舞い

1. useHistory が偽であれば直ちに false を返す。 2. fname が NULL の場合、PageOutput を介してページャを初期化し、コンソール出力の準備を行う。 3. fname が指定されている場合、該当ファイルを書き込みモードで開く。ファイルオープンに失敗した場合はエラーログを出力し false を返す。 4. BEGIN_ITERATE_HISTORY マクロを使用して履歴リストを走査し、各行を fprintf で出力先に書き出す。 5. 出力先がページャであれば ClosePager を呼び出し、ファイルであれば fclose を呼び出してリソースを解放する。 6. 正常終了時に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c:2821 （\s コマンド実行時など）
  • **呼び出し先**:
  • PageOutput / ClosePager (コンソール出力時)
  • fopen / fclose (ファイル出力時)
  • pg_log_error (エラー発生時)
  • BEGIN_ITERATE_HISTORY / END_ITERATE_HISTORY (履歴走査用マクロ)

関連データ構造

  • pset: グローバル設定構造体（pset.popt.topt を通じてページャ設定を参照）。
  • cur_hist: 履歴走査時に使用される内部的な履歴エントリポインタ。

失敗モードと境界条件

  • USE_READLINE が未定義の環境では、履歴機能がサポートされていない旨のエラーを出力し false を返す。
  • useHistory が無効な場合、何も出力せず false を返す。
  • ファイル出力時に書き込み権限がない等の理由で fopen が失敗した場合、エラーログを出力し false を返す。

保守担当者向けメモ

  • BEGIN_ITERATE_HISTORY マクロは libedit と readline の差異を吸収するため、環境によって展開内容が異なる点に注意すること。
  • ページャの制御は pset.popt.topt に依存しているため、設定変更時には整合性に留意が必要。
