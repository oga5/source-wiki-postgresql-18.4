get_rel_infos_query

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/info.c:470

目的

pg_upgrade において、アップグレード対象となるテーブル、TOAST テーブル、およびインデックスのメタデータを PostgreSQL クラスタから取得するための SQL クエリ文字列を生成します。

シグネチャまたは形状

char *get_rel_infos_query(void)

入力と出力

  • **入力**: なし（ただし、グローバル変数 user_opts.transfer_mode を参照してクエリ内容を動的に変更します）。
  • **出力**: 実行可能な SQL クエリを含む char * 型の文字列。

前提条件と不変条件

  • 呼び出し側は、返された文字列を適切に解放する責任を負います。
  • クエリは pg_catalog を対象とし、FirstNormalObjectId 以上の OID を持つユーザーオブジェクトを抽出するように設計されています。
  • 結果セットは OID 順にソートされることが保証されています。

アルゴリズムまたは振る舞い

1. PQExpBuffer を初期化し、以下の 3 つの CTE を構築します。

  • regular_heap: ユーザーテーブルおよびマテリアライズドビューを収集（pg_largeobject を含む）。TRANSFER_MODE_SWAP の場合はシーケンスも対象に含めます。
  • toast_heap: regular_heap に関連付けられた TOAST テーブルを収集。
  • all_index: 有効かつ準備完了状態（indisvalid かつ indisready）のインデックスを収集。

    • 2. 最終的にこれら 3 つの CTE を UNION ALL で結合し、pg_class、pg_namespace、pg_tablespace と結合して、リレーション名、ファイルノード、テーブルスペース情報などを取得するクエリを生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/info.c 内の get_db_rel_and_slot_infos 関数から呼び出されます。
  • **呼び出し先**: initPQExpBuffer、appendPQExpBuffer、appendPQExpBufferStr などの libpq のバッファ操作関数を使用します。

関連データ構造

  • PQExpBufferData: クエリ文字列を構築するための動的バッファ。
  • RelInfo: このクエリの結果を格納するために使用される構造体（本関数自体はクエリ文字列を生成するのみ）。

失敗モードと境界条件

  • メモリ不足により PQExpBuffer の拡張が失敗した場合、libpq の内部ルールに従い終了する可能性があります。
  • user_opts.transfer_mode が TRANSFER_MODE_SWAP であるか否かでクエリの対象範囲（シーケンスの有無）が変化します。

保守担当者向けメモ

  • クエリ内の pg_largeobject の扱いは、pg_dump の挙動に合わせるために特別扱いされています。変更時は注意してください。
  • OID 順のソートは、後続の gen_db_file_maps での効率的な照合処理（$O(N)$）に不可欠です。クエリを変更する際は ORDER BY 1 を維持してください。
