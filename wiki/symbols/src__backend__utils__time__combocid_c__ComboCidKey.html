ComboCidKey

定義

  • 種別: typedef
  • 定義箇所: src/backend/utils/time/combocid.c:62

目的

ComboCidKeyData 構造体へのポインタ型であり、同一トランザクション内でのタプルの挿入と削除を追跡するためのハッシュキーとして機能します。

シグネチャまたは形状

typedef ComboCidKeyData *ComboCidKey;

入力と出力

  • **入力**: ComboCidKeyData 構造体への参照。
  • **出力**: ComboCidKey 型（ポインタ）。

前提条件と不変条件

  • ComboCidKeyData 構造体にはパディングが存在しないことが前提とされており、ハッシュ操作においてこの不変条件が利用されます。

アルゴリズムまたは振る舞い

  • ComboCidKey は、comboHash ハッシュテーブル内での検索やエントリ作成時のキーとして使用されます。
  • cmin と cmax のペアを抽象化し、Combo CID を介してタプルヘッダーの領域制約を回避するためのインデックスとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/time/combocid.c 内の静的変数 comboCids の定義において参照されています。
  • **呼び出し先**: 構造体定義である ComboCidKeyData に依存しています。

関連データ構造

  • ComboCidKeyData: 本型が指し示す実体。
  • comboHash: 本型をキーとして使用するハッシュテーブル。
  • comboCids: 本型をインデックスとして管理される配列。

失敗モードと境界条件

  • 構造体のパディングに関する前提が崩れた場合、ハッシュ計算の整合性が失われる可能性があります。

保守担当者向けメモ

  • 本型は TopTransactionContext に確保されるメモリ領域を指します。トランザクション終了時にコンテキストが破棄されるため、明示的なメモリ解放は不要です。
  • 構造体のレイアウトを変更する際は、ハッシュ操作への影響を考慮してください。
