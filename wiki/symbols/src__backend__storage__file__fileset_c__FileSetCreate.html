FileSetCreate

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/fileset.c:91

目的

指定された名前の一時ファイルを FileSet 内に作成します。必要に応じて、ファイル格納用のディレクトリをオンデマンドで生成します。

シグネチャまたは形状

File FileSetCreate(FileSet *fileset, const char *name)

入力と出力

  • **入力**:
  • fileset: 対象となる FileSet 構造体へのポインタ。
  • name: 作成するファイルの名前。
  • **出力**:
  • 成功時: 作成されたファイルの File 記述子。
  • 失敗時: 0 以下の値（PathNameCreateTemporaryFile の仕様に準拠）。

前提条件と不変条件

  • fileset は事前に FileSetInit によって初期化されている必要があります。
  • ファイルの配置先テーブルスペースは、ChooseTablespace により fileset と name に基づいて決定されます。

アルゴリズムまたは振る舞い

1. FilePath を使用して、指定された name に対応する物理パスを構築します。 2. PathNameCreateTemporaryFile を呼び出し、ファイルの作成を試みます。 3. ファイル作成に失敗した場合（戻り値が 0 以下の場合）:

  • ChooseTablespace で配置先テーブルスペースを特定します。
  • TempTablespacePath および FileSetPath を用いて、一時ディレクトリのパスを解決します。
  • PathNameCreateTemporaryDir を呼び出し、ディレクトリをオンデマンドで作成します。
  • 再度 PathNameCreateTemporaryFile を呼び出し、ファイルの作成を試みます。

    • 4. 最終的な File 記述子を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/file/buffile.c:247
  • **呼び出し先**:
  • FilePath
  • PathNameCreateTemporaryFile
  • ChooseTablespace
  • TempTablespacePath
  • FileSetPath
  • PathNameCreateTemporaryDir

関連データ構造

  • FileSet: 一時ファイルの集合を管理する構造体。
  • File: fd.c で定義されるファイル記述子。

失敗モードと境界条件

  • PathNameCreateTemporaryFile がファイル作成に失敗した場合、ディレクトリ作成を試みますが、ディレクトリ作成も失敗した場合はその後の PathNameCreateTemporaryFile の結果がそのまま返されます。
  • ファイル名やパスの長さが MAXPGPATH を超える場合の挙動は、基盤となる fd.c の実装に依存します。

保守担当者向けメモ

  • 本関数は FileSet のディレクトリが存在しない場合に自動生成する「オンデマンド作成」のロジックを含んでいます。
  • 呼び出し側は、戻り値が有効なファイル記述子であるかを適切にチェックする必要があります。
  • FileSet はトランザクション終了時に自動削除されないため、リソースリークを防ぐために FileSetDelete 等による明示的な削除が必要です。
