convert_and_check_filename

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/genfile.c:53

目的

SQLインターフェース経由でサーバー上のファイルにアクセスする際、指定されたファイルパスが許可された範囲内にあるかを検証し、正規化されたパスを返します。

シグネチャまたは形状

static char *convert_and_check_filename(text *arg);

入力と出力

  • **入力**: text *arg（ファイルパスを表す文字列）
  • **出力**: char *（正規化されたファイルパスの文字列）
  • **戻り値**: 検証に成功した場合、正規化されたパス文字列を返します。

前提条件と不変条件

  • 呼び出し元は、PostgreSQLのメモリ管理コンテキスト内で有効な text 型の引数を提供する必要があります。
  • pg_read_server_files ロールを持つユーザーは、パスの制限を受けずにアクセス可能です。
  • それ以外のユーザーの場合、パスは DataDir 内、または Log_directory 内に限定されます。

アルゴリズムまたは振る舞い

1. text_to_cstring を使用して引数をC文字列に変換します。 2. canonicalize_path を呼び出し、パスを正規化します（パスの長さが変更される可能性があります）。 3. 現在のユーザーが pg_read_server_files ロールの権限を持つ場合、検証をスキップしてパスを返します。 4. 権限がない場合:

  • 絶対パスの場合: DataDir または Log_directory の配下にあるかを確認します。
  • 相対パスの場合: カレントワーキングディレクトリ（cwd）の配下にあるかを確認します。

    • 5. 条件を満たさない場合、ERRCODE_INSUFFICIENT_PRIVILEGE エラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/genfile.c 内のファイル読み込み関数群（read_text_file, read_binary_file 等）およびファイル属性取得関数。
  • **呼び出し先**:
  • text_to_cstring
  • canonicalize_path
  • has_privs_of_role
  • is_absolute_path
  • path_is_prefix_of_path
  • path_is_relative_and_below_cwd
  • ereport

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • DataDir, Log_directory: サーバー設定で定義されたディレクトリパス。

失敗モードと境界条件

  • **権限不足**: 指定されたパスが許可されたディレクトリ外にある場合、ERROR をスローします。
  • **パスの正規化**: canonicalize_path によりパスが変更されるため、呼び出し元は戻り値の文字列を適切に管理する必要があります。

保守担当者向けメモ

  • 本関数はセキュリティの要です。パス検証ロジック（特に path_is_prefix_of_path の使用箇所）を変更する際は、ディレクトリトラバーサル攻撃の脆弱性が混入しないよう細心の注意を払ってください。
  • Log_directory が DataDir の外部にあるケースも考慮されているため、検証ロジックを簡略化する際は注意が必要です。
