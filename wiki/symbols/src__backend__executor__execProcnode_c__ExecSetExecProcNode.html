ExecSetExecProcNode

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execProcnode.c:429

目的

実行ノードのタプル取得処理（ExecProcNode）を初期化または再設定し、初回実行時にスタックチェックやインストルメンテーションのセットアップを行うラッパーを適用します。

シグネチャまたは形状

void ExecSetExecProcNode(PlanState *node, ExecProcNodeMtd function)
  • node: 処理対象の PlanState 構造体へのポインタ。
  • function: ノード固有のタプル取得関数（ExecProcNodeMtd 型）。

入力と出力

  • 入力: PlanState ノードおよび実行関数ポインタ。
  • 出力: なし（副作用として node 内部の関数ポインタを更新）。

前提条件と不変条件

  • node は有効な PlanState 構造体を指している必要があります。
  • 実行開始後に本関数が呼び出された場合、初回実行時のラッパー（ExecProcNodeFirst）が再度適用されます。

アルゴリズムまたは振る舞い

1. 引数 function を node->ExecProcNodeReal に格納します。 2. node->ExecProcNode に ExecProcNodeFirst を代入します。

  • これにより、次に ExecProcNode が呼び出された際、ExecProcNodeFirst がスタック深さのチェックやインストルメンテーションの初期化を行い、その後 ExecProcNodeReal を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execProcnode.c (内部再帰呼び出し)
  • src/backend/executor/nodeHashjoin.c (ExecParallelHashJoin の設定時)
  • **呼び出し先**:
  • ExecProcNodeFirst (代入先として参照)

関連データ構造

  • PlanState: 実行ノードの状態を保持する構造体。ExecProcNode および ExecProcNodeReal フィールドを保持します。

失敗モードと境界条件

  • node が NULL の場合、セグメンテーションフォールトが発生する可能性があります。
  • 実行開始後に本関数を呼び出すと、既に初期化済みのノードであっても ExecProcNodeFirst が再度実行されるため、冗長な処理が発生します。

保守担当者向けメモ

  • 本関数は、実行エンジンにおける動的なディスパッチ切り替えの要です。新しいノードタイプを追加する際、実行関数の割り当てが適切に行われているか確認してください。
