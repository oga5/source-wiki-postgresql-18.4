index_deform_tuple_internal

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/common/indextuple.c:478

目的

インデックス・タプルの物理的なデータ領域から各属性値を抽出し、Datum 配列および NULL フラグ配列に展開する内部処理関数です。

シグネチャまたは形状

void index_deform_tuple_internal(TupleDesc tupleDescriptor,
                                 Datum *values, bool *isnull,
                                 char *tp, bits8 *bp, int hasnulls)

入力と出力

  • **入力**:
  • tupleDescriptor: タプルの属性定義情報。
  • values: 抽出した値を格納する Datum 配列（呼び出し元で確保）。
  • isnull: 各属性が NULL かどうかを格納する bool 配列（呼び出し元で確保）。
  • tp: インデックス・タプルのデータ開始位置を指すポインタ。
  • bp: NULL ビットマップの先頭ポインタ。
  • hasnulls: NULL ビットマップが存在するかを示すフラグ。
  • **出力**: なし（values および isnull 配列を直接更新）。

前提条件と不変条件

  • tupleDescriptor->natts が INDEX_MAX_KEYS 以下であること（アサーションにより保証）。
  • values および isnull 配列は、少なくとも tupleDescriptor->natts 分の要素を保持できるサイズであること。

アルゴリズムまたは振る舞い

1. 各属性を順次走査し、NULL ビットマップを確認して NULL であれば値をスキップします。 2. attcacheoff を利用して属性のオフセットを特定します。キャッシュが有効でない場合は、データ型のアライメント規則（attalignby）に従ってオフセットを計算し、可能であればキャッシュを更新します。 3. fetchatt を使用して tp + off から属性値を抽出します。 4. att_addlength_pointer を用いて、次の属性の開始位置を計算します。 5. 可変長属性や NULL の存在によりオフセット計算が複雑になる場合、slow フラグを立ててキャッシュ利用を抑制します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/indextuple.c (内部関数 index_deform_tuple からの呼び出し)
  • src/backend/access/spgist/spgutils.c
  • **呼び出し先**:
  • att_isnull (マクロ)
  • TupleDescCompactAttr (マクロ)
  • att_nominal_alignby / att_pointer_alignby (アライメント計算)
  • fetchatt (値の抽出)
  • att_addlength_pointer (オフセット加算)

関連データ構造

  • TupleDesc: 属性定義を保持。
  • CompactAttribute: 属性のメタデータ（アライメントやキャッシュオフセットを含む）。
  • IndexTuple: 操作対象の物理データ構造。

失敗モードと境界条件

  • natts が INDEX_MAX_KEYS を超える場合、アサーション違反により停止します。
  • 可変長属性（attlen == -1）が含まれる場合、アライメントの計算が複雑になり、キャッシュの利用が制限される場合があります。

保守担当者向けメモ

  • 本関数は index_deform_tuple の主要なロジックを担っています。
  • attcacheoff の更新は、同一の TupleDesc を使用する連続的なアクセスにおいてパフォーマンスを向上させます。
  • slow フラグが true になった場合、以降の属性抽出ではキャッシュを利用せず、動的なオフセット計算が行われます。
