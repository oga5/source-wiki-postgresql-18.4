sepgsql_proc_relabel

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/proc.c:197

目的

PostgreSQLの関数オブジェクトに対する SECURITY LABEL コマンド実行時に、SELinuxポリシーに基づいた権限チェックを行う。

シグネチャまたは形状

void sepgsql_proc_relabel(Oid functionId, const char *seclabel)

入力と出力

  • **入力**:
  • functionId (Oid): 対象関数のオブジェクト識別子。
  • seclabel (const char *): 新たに適用しようとするセキュリティラベル文字列。
  • **出力**: なし（権限チェックに失敗した場合は例外をスローする）。

前提条件と不変条件

  • functionId は pg_proc カタログに存在する有効な関数を指している必要がある。
  • 呼び出し元は、対象オブジェクトに対するラベル変更操作のコンテキストを保持していること。

アルゴリズムまたは振る舞い

1. getObjectIdentity を使用して、対象関数を識別するための文字列（audit_name）を生成する。 2. sepgsql_avc_check_perms を呼び出し、現在の関数に対して db_procedure:{setattr relabelfrom} 権限があるか検証する。 3. sepgsql_avc_check_perms_label を呼び出し、指定された seclabel に対して db_procedure:{relabelto} 権限があるか検証する。 4. 生成した audit_name を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/label.c:518 から呼び出されることが確認されている。
  • **呼び出し先**:
  • getObjectIdentity
  • sepgsql_avc_check_perms
  • sepgsql_avc_check_perms_label
  • pfree

関連データ構造

  • ObjectAddress: 対象関数を特定するために使用。
  • ProcedureRelationId: pg_proc カタログの識別子。

失敗モードと境界条件

  • 権限チェック（sepgsql_avc_check_perms または sepgsql_avc_check_perms_label）が失敗した場合、SELinuxポリシーに従ってアクセス拒否の例外が発生する。
  • getObjectIdentity が失敗した場合、またはメモリ不足が発生した場合は、PostgreSQLの標準的なエラーハンドリングに従う。

保守担当者向けメモ

  • 本関数は SECURITY LABEL コマンドのフックとして機能する。
  • SEPG_DB_PROCEDURE__SETATTR と SEPG_DB_PROCEDURE__RELABELFROM の両方が要求される点に注意すること。
