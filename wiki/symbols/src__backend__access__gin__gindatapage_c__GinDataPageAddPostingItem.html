GinDataPageAddPostingItem

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/gindatapage.c:379

目的

GINポスティングツリーの内部ノード（非リーフページ）に対し、新しい PostingItem を挿入する。

シグネチャまたは形状

void GinDataPageAddPostingItem(Page page, PostingItem *data, OffsetNumber offset)

入力と出力

  • **入力**:
  • page: 操作対象のページ（Page 型）。
  • data: 挿入する PostingItem 構造体へのポインタ。
  • offset: 挿入位置を示す OffsetNumber。InvalidOffsetNumber を指定した場合は末尾に追加される。
  • **出力**: なし（ページ内容を直接更新）。

前提条件と不変条件

  • data が指す PostingItem のブロック番号が InvalidBlockNumber でないこと。
  • 操作対象の page がリーフページではないこと（!GinPageIsLeaf(page)）。

アルゴリズムまたは振る舞い

1. ページ内の現在の最大オフセット（maxoff）を取得する。 2. offset が InvalidOffsetNumber の場合、末尾（maxoff + 1）を挿入先とする。 3. offset が指定されている場合、その位置に既存のアイテムがあれば memmove で後方にずらし、挿入領域を確保する。 4. memcpy を使用して data をページ内の指定位置へ書き込む。 5. ページ内の最大オフセットを更新する。 6. GinDataPageSetDataSize を呼び出し、ページ内のデータサイズを更新して pd_lower を調整する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gindatapage.c 内の複数箇所（ページ分割や初期化処理など）。
  • src/backend/access/gin/ginxlog.c: WALリプレイ時に呼び出される。
  • **呼び出し先**:
  • GinPageGetOpaque: ページヘッダーの取得。
  • PostingItemGetBlockNumber: アイテムの検証。
  • GinDataPageGetPostingItem: ページ内のアイテム位置計算。
  • GinPageIsLeaf: ページ種別の判定。
  • GinDataPageSetDataSize: ページメタデータの更新。

関連データ構造

  • Page: PostgreSQLの標準的なページ構造。
  • PostingItem: ポスティングツリーの内部ノードに格納されるアイテム構造体。

失敗モードと境界条件

  • ページ内に十分な空き容量がない場合、メモリ破壊やパニックが発生する可能性がある（本関数内では容量チェックを行わず、呼び出し元で事前に確保されていることが前提）。
  • offset が現在の maxoff を超える値を指定した場合の挙動は未定義。

保守担当者向けメモ

  • 本関数は非リーフページ専用である。リーフページへの挿入には別のロジックが必要。
  • pd_lower の更新は、フルページイメージ取得時の効率化（未使用領域の圧縮）を目的としている。
