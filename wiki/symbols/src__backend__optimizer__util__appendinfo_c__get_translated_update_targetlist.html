get_translated_update_targetlist

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/appendinfo.c:713

目的

UPDATE クエリにおいて、親テーブル向けに定義されたターゲットリストおよび更新列情報を、指定されたリレーション（子テーブルまたは親自身）向けに変換・複製する。

シグネチャまたは形状

void get_translated_update_targetlist(PlannerInfo *root, Index relid,
                                      List **processed_tlist, List **update_colnos)

入力と出力

  • root: プランナのグローバル情報。
  • relid: 変換対象のリレーションID。
  • processed_tlist (出力): 変換後のターゲットリスト。
  • update_colnos (出力): 変換後の更新列番号リスト（NULL可）。

前提条件と不変条件

  • root->parse->commandType が CMD_UPDATE であること（アサーションにより強制）。
  • relid が root->all_result_relids に含まれていること（継承関係にある場合）。

アルゴリズムまたは振る舞い

1. **非継承ケース**: relid が root->parse->resultRelation と一致する場合、root->processed_tlist および root->update_colnos を copyObject で複製して返す。 2. **継承ケース**:

  • processed_tlist に対して adjust_appendrel_attrs_multilevel を呼び出し、親から子への属性変換を行う。
  • update_colnos が指定されている場合、adjust_inherited_attnums_multilevel を呼び出し、更新対象の列番号を子テーブルのスキーマに合わせて変換する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/postgres_fdw.c:2521
  • **呼び出し先**:
  • copyObject
  • adjust_appendrel_attrs_multilevel
  • adjust_inherited_attnums_multilevel
  • find_base_rel

関連データ構造

  • PlannerInfo: プランナのコンテキスト。
  • List: ターゲットリストおよび列番号リストの保持に使用。

失敗モードと境界条件

  • UPDATE 以外のコマンドで呼び出された場合はアサーション失敗となる。
  • 継承関係にないリレーションに対して relid が resultRelation と一致しない場合、bms_is_member のアサーションにより停止する可能性がある。

保守担当者向けメモ

  • 本関数は UPDATE 専用のユーティリティである。
  • processed_tlist のコピーは、呼び出し元がツリーを破壊的に変更する可能性があることを考慮している。
