GetForeignDataWrapperByName

定義

  • 種別: function
  • 定義箇所: src/backend/foreign/foreign.c:96

目的

指定された名前の外部データラッパー（FDW）の定義情報をシステムカタログから取得する。

シグネチャまたは形状

ForeignDataWrapper *GetForeignDataWrapperByName(const char *fdwname, bool missing_ok)

入力と出力

  • **入力**:
  • fdwname: 検索対象の外部データラッパー名（const char *）。
  • missing_ok: true の場合、対象が存在しなくてもエラーにせず NULL を返す。false の場合はエラーを発生させる。
  • **出力**:
  • 成功時: ForeignDataWrapper 構造体へのポインタ。
  • 失敗時（missing_ok=true かつ対象なし）: NULL。

前提条件と不変条件

  • get_foreign_data_wrapper_oid を介してシステムカタログ（pg_foreign_data_wrapper）を検索する。
  • 返される ForeignDataWrapper 構造体は、呼び出し側のメモリコンテキストで適切に管理される必要がある。

アルゴリズムまたは振る舞い

1. get_foreign_data_wrapper_oid を呼び出し、指定された名前の FDW に対応する OID を取得する。 2. OID が無効（!OidIsValid）であれば、NULL を返す。 3. OID が有効であれば、GetForeignDataWrapper(fdwId) を呼び出し、詳細な定義情報を取得して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/foreigncmds.c: CREATE FOREIGN DATA WRAPPER 等のコマンド実行時に存在確認や定義取得のために呼び出される。
  • **呼び出し先**:
  • get_foreign_data_wrapper_oid
  • GetForeignDataWrapper

関連データ構造

  • ForeignDataWrapper: FDW の定義情報を保持する構造体。

失敗モードと境界条件

  • missing_ok が false の場合、指定された名前の FDW が存在しないと get_foreign_data_wrapper_oid 内でエラーが発生する。
  • データベース接続やシステムカタログへのアクセス権限がない場合、例外が発生する可能性がある。

保守担当者向けメモ

  • 本関数は名前ベースの検索を行うラッパーであり、実質的なデータ取得は GetForeignDataWrapper に依存している。
  • foreign.h にて宣言されており、外部モジュールからも利用可能。
