目的

PostgreSQLのクエリ計画作成時において、外部テーブル（Foreign Table）に対する更新操作（INSERT, UPDATE, DELETE）の計画を識別するためのインデックス値を保持するために使用されます。

シグネチャまたは形状

typedef Oid Index;

*(注: ソースコード上の定義は typedef ですが、PostgreSQLの慣習として Oid 型の別名として定義されています)*

入力と出力

  • **入力**: 外部テーブルの更新計画を生成する関数（例: PlanForeignModify_function）において、resultRelation 引数として渡されます。
  • **出力**: 該当するリレーションを特定するための識別子として機能します。

前提条件と不変条件

  • PlannerInfo 構造体内の simple_rel_array におけるインデックスとして有効である必要があります。
  • 外部テーブルに対する操作が計画される際、クエリ内のリレーションを一意に特定するために使用されます。

アルゴリズムまたは振る舞い

  • クエリ計画フェーズにおいて、ModifyTable ノードがどの外部テーブルを操作対象とするかを指定するために使用されます。
  • PlanForeignModify_function の引数として渡され、FDWが対象の外部テーブルに関する情報を取得する際のキーとなります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/foreign/fdwapi.h 内の PlanForeignModify_function シグネチャにおいて引数として利用されています。
  • **呼び出し先**: ツールによる解析では、特定の関数呼び出しの直接的な証拠は得られていません。
  • **備考**: contrib/amcheck 等のソースコードで言及されている Index は、本シンボル（FDW API用の型定義）とは異なる文脈（B-Treeインデックス等）である可能性が高いため、混同に注意してください。

関連データ構造

  • PlannerInfo: クエリ計画全体の情報を保持する構造体。
  • ModifyTable: 更新操作を表す計画ノード。
  • PlanForeignModify_function: 本型を引数に取る関数ポインタ型。

失敗モードと境界条件

  • 無効な Index 値が渡された場合、PlannerInfo からリレーション情報を取得する際にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本シンボルは fdwapi.h において typedef として定義されています。
  • contrib モジュール内のインデックス関連コードと名前が重複していますが、FDW APIの文脈では「リレーションの識別子」を指すことに留意してください。
