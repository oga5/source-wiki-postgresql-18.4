gen_db_file_maps

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/info.c:44

目的

新旧 PostgreSQL クラスタ間で、データベース内のリレーション（テーブルやインデックス）の OID を照合し、ファイル移行のためのマッピング情報を生成します。

シグネチャまたは形状

FileNameMap *gen_db_file_maps(DbInfo *old_db, DbInfo *new_db, int *nmaps, const char *old_pgdata, const char *new_pgdata);

入力と出力

  • **入力**:
  • old_db, new_db: 比較対象となる新旧データベースのメタデータ構造体。
  • old_pgdata, new_pgdata: 新旧クラスタのデータディレクトリパス。
  • **出力**:
  • 戻り値: 生成された FileNameMap 配列へのポインタ。
  • nmaps: 生成されたマッピングの個数が格納される整数ポインタ。

前提条件と不変条件

  • old_db および new_db 内の RelInfo 配列は、OID 順にソートされている必要があります。
  • マッピング生成時にすべてのリレーションが一致しない場合、関数は pg_fatal を呼び出しプロセスを終了させます。

アルゴリズムまたは振る舞い

1. old_db のリレーション数に基づき、FileNameMap 配列をメモリ確保します。 2. 新旧の RelInfo 配列を OID 順に線形走査（$O(N)$）します。 3. OID が一致する場合、create_rel_filename_map を呼び出しマッピングを作成します。 4. OID が不一致の場合、report_unmatched_relation を呼び出し警告またはエラーを記録します。ただし、pg_toast 名前空間の不一致は無視される場合があります。 5. 名前空間（nspname）やリレーション名（relname）が一致しない場合は警告を出力します。 6. 最終的に一致しなかったリレーションがある場合、致命的エラーとして終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/relfilenumber.c:203
  • **呼び出し先**:
  • create_rel_filename_map
  • report_unmatched_relation
  • pg_malloc, pg_log, pg_fatal (ユーティリティ関数)

関連データ構造

  • DbInfo: データベース情報を保持。
  • RelInfo: 個別のリレーション情報を保持。
  • FileNameMap: 新旧のファイルパスと OID を結びつける移行用構造体。

失敗モードと境界条件

  • **OID 不一致**: 新旧で OID が一致しない場合、report_unmatched_relation によりエラーが報告されます。
  • **名前の不一致**: OID は一致しても名前が異なる場合、警告が出力されます。
  • **メモリ不足**: pg_malloc が失敗した場合、プロセスは終了します。
  • **TOAST テーブルの不一致**: 新しいクラスタで不要な TOAST テーブルが生成されている場合、特定の条件下で無視されます。

保守担当者向けメモ

  • 本関数は OID 順序に強く依存しています。get_rel_infos_query によるクエリ結果がソートされていることが前提です。
  • pg_toast 名前空間の扱いは、新旧クラスタ間での TOAST テーブル生成ロジックの差異を吸収するための例外処理を含んでいます。
