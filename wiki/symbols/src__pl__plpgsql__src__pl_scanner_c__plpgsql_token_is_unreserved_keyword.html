plpgsql_token_is_unreserved_keyword

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_scanner.c:417

目的

指定されたトークンが PL/pgSQL における「非予約キーワード（unreserved keyword）」であるかどうかを判定します。

シグネチャまたは形状

bool plpgsql_token_is_unreserved_keyword(int token)

入力と出力

  • **入力**: token (int型) - 判定対象のトークンコード。
  • **出力**: bool - 引数のトークンが UnreservedPLKeywordTokens 配列に含まれていれば true、そうでなければ false を返します。

前提条件と不変条件

  • UnreservedPLKeywordTokens 配列が定義されている必要があります。
  • 比較対象の token は、pl_gram.h 等で定義された有効なトークンコードであると想定されます。

アルゴリズムまたは振る舞い

1. UnreservedPLKeywordTokens 配列の全要素をループで走査します。 2. 現在の要素が引数の token と一致するかを比較します。 3. 一致した時点で直ちに true を返し、ループ終了まで一致が見つからなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 外部からの具体的な呼び出し箇所は提供されたソース情報からは特定できません。
  • **呼び出し先**: lengthof マクロ（配列サイズ取得）および UnreservedPLKeywordTokens 配列を参照します。

関連データ構造

  • UnreservedPLKeywordTokens: 非予約キーワードのトークンコードを保持する配列（pl_unreserved_kwlist_d.h 等で定義）。

失敗モードと境界条件

  • 引数 token が無効な値であっても、配列の範囲内であれば単に false を返すのみで、異常終了はしません。
  • 配列が空の場合、常に false を返します。

保守担当者向けメモ

  • 本関数は pl_unreserved_kwlist.h に依存しています。新しい非予約キーワードを追加する場合は、対応するヘッダーファイルを更新してください。
