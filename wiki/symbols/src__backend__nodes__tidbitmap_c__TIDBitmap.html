TIDBitmap

定義

  • 種別: struct
  • 定義箇所: src/backend/nodes/tidbitmap.c:140

目的

PostgreSQLのクエリ実行エンジンにおいて、タプル識別子（TID）の集合をメモリ効率よく管理するためのデータ構造です。ビットマップスキャンにおいて、ヒープアクセスを最適化するために使用されます。

シグネチャまたは形状

struct TIDBitmap {
    NodeTag type;
    MemoryContext mcxt;
    TBMStatus status;
    struct pagetable_hash *pagetable;
    int nentries;
    int maxentries;
    int npages;
    int nchunks;
    TBMIteratingState iterating;
    uint32 lossify_start;
    PagetableEntry entry1;
    PagetableEntry **spages;
    PagetableEntry **schunks;
    dsa_pointer dsapagetable;
    dsa_pointer dsapagetableold;
    dsa_pointer ptpages;
    dsa_pointer ptchunks;
    dsa_area *dsa;
};

入力と出力

  • **入力**: tbm_add_tuples 等の関数を通じて TID が追加されます。
  • **出力**: tbm_iterate を通じて、ソートされたページ単位またはタプル単位の反復結果が提供されます。

前提条件と不変条件

  • status が TBM_ONE_PAGE の場合、entry1 フィールドが使用されます。
  • iterating が真の場合、spages および schunks が有効なソート済みリストとして保持されます。
  • メモリ制限（maxentries）を超過した場合、正確なページ情報は損失あり（lossy）チャンクへ変換されます。

アルゴリズムまたは振る舞い

1. **初期化**: TBM_ONE_PAGE モードで開始し、単一ページを効率的に保持します。 2. **蓄積**: 複数のページが追加されるとハッシュテーブル（pagetable）へ移行します。 3. **損失化（Lossify）**: メモリ制限に達すると、正確なタプル情報をページ単位の存在フラグへ変換し、メモリ消費を抑制します。 4. **反復**: tbm_begin_iterate 呼び出しにより読み取り専用モードへ移行し、ページ番号順にソートされた結果を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: blgetbitmap (bloom), bringetbitmap (brin), gingetbitmap (gin), gistgetbitmap (gist), hashgetbitmap (hash) 等のインデックススキャン関数。
  • **呼び出し先**: 内部的に pagetable_hash（ハッシュテーブル）や dsa_area（共有メモリ）を操作します。

関連データ構造

  • PagetableEntry: ハッシュテーブルの各エントリ。
  • TBMStatus: ビットマップの状態（TBM_ONE_PAGE 等）。
  • TBMIteratingState: 反復処理の状態。

失敗モードと境界条件

  • メモリ不足時: tbm_lossify が実行され、正確な情報が失われる（損失あり状態への移行）。
  • 並列クエリ時: dsa_area を介した共有メモリ管理が必須となり、正しく初期化されていない場合はエラーとなります。

保守担当者向けメモ

  • 本構造体は Node として定義されており、PostgreSQL のノードコピーや出力関数と互換性があります。
  • dsa_pointer を含むフィールドは、並列クエリ実行時のワーカー間共有を目的としています。
  • 損失あり（lossy）状態への変換ロジックは、メモリ消費量とスキャン精度のトレードオフを制御する重要な箇所です。
