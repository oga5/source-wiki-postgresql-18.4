get_crosstab_tuplestore

定義

  • 種別: function
  • 定義箇所: contrib/tablefunc/tablefunc.c:790

目的

SQL クエリの結果セットを SPI (Server Programming Interface) 経由で取得し、指定されたハッシュテーブルに基づいてクロス集計（横持ち変換）を行い、結果を Tuplestore に格納して返却する。

シグネチャまたは形状

static Tuplestorestate *get_crosstab_tuplestore(
    char *sql,
    HTAB *crosstab_hash,
    TupleDesc tupdesc,
    bool randomAccess
)

入力と出力

  • **入力**:
  • sql: 実行するソースデータ取得用クエリ。
  • crosstab_hash: カテゴリ名と属性インデックスを保持するハッシュテーブル。
  • tupdesc: 出力タプルの構造定義。
  • randomAccess: Tuplestore のランダムアクセス可否。
  • **出力**:
  • 構築された Tuplestorestate へのポインタ。

前提条件と不変条件

  • 入力 SQL は最低 3 列（rowname, category, value）を返す必要がある。
  • 4 列以上の場合は、第 1 列が rowname、最後から 2 番目が category、最後が value として扱われる。
  • crosstab_hash にはカテゴリ情報が格納されている必要がある。
  • 出力タプルの列数は (ソース列数 - 2) + カテゴリ数 と一致しなければならない。

アルゴリズムまたは振る舞い

1. tuplestore_begin_heap でストレージを初期化し、SPI_connect でクエリ実行環境を確立する。 2. SPI_execute で SQL を実行し、結果セットを走査する。 3. rowname が変化するたびに、蓄積された values 配列から BuildTupleFromCStrings を用いてタプルを生成し、tuplestore_puttuple で格納する。 4. 各行の category をハッシュテーブルで検索し、対応する列インデックスに value を配置する。 5. 最終行の処理後、SPI_finish を呼び出し、構築した Tuplestorestate を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/tablefunc/tablefunc.c 内の crosstab_hash 関数（683行目付近）。
  • **呼び出し先**: SPI_connect, SPI_execute, SPI_getvalue, SPI_finish, tuplestore_begin_heap, tuplestore_puttuple, BuildTupleFromCStrings, crosstab_HashTableLookup 等。

関連データ構造

  • Tuplestorestate: 結果セットを保持するメモリ/ディスク上の構造。
  • HTAB: カテゴリ管理用ハッシュテーブル。
  • TupleDesc: タプルのメタデータ。
  • crosstab_cat_desc: カテゴリ名と出力インデックスのマッピング情報。

失敗モードと境界条件

  • **ERRCODE_CARDINALITY_VIOLATION**: カテゴリハッシュが空の場合。
  • **ERRCODE_INVALID_PARAMETER_VALUE**: ソースクエリの列数が 3 未満の場合。
  • **ERRCODE_DATATYPE_MISMATCH**: 出力タプルの列数が計算値と一致しない場合。
  • **SPI エラー**: SPI_finish が失敗した場合に elog(ERROR) を発生させる。

保守担当者向けメモ

  • 本関数は SPI を使用するため、呼び出し元のクエリコンテキストに依存する。
  • values 配列のメモリ管理には palloc0 を使用しており、行のフラッシュ時に xpfree で個別に解放している。
  • 入力 SQL の順序（ORDER BY）が正しくない場合、クロス集計結果が不完全になる可能性がある。
