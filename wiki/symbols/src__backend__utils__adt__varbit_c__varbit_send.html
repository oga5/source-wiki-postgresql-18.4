varbit_send

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varbit.c:680

目的

VarBit 型のデータを、クライアントとの通信に適したバイナリ形式に変換して送信する。

シグネチャまたは形状

Datum varbit_send(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して VarBit 型のデータを受け取る。
  • **出力**: bytea 型のバイナリデータ（PG_RETURN_BYTEA_P）。

前提条件と不変条件

  • 入力される VarBit 構造体は、VARBIT_PAD マクロに従い、最後のバイトの未使用ビットがゼロクリアされている必要がある。

アルゴリズムまたは振る舞い

1. pq_begintypsend を呼び出し、送信用のバッファ StringInfoData を初期化する。 2. VARBITLEN(s) を使用してビット長を 32 ビット整数としてバッファに書き込む。 3. VARBITS(s) から VARBITBYTES(s) バイト分をバッファにコピーする。 4. pq_endtypsend を呼び出し、構築されたバイナリデータを bytea 形式で返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/varbit.c:379 (bit_send 関数がコード共有のために本関数を呼び出す)
  • **呼び出し先**:
  • pq_begintypsend
  • pq_sendint32
  • pq_sendbytes
  • pq_endtypsend
  • PG_GETARG_VARBIT_P
  • PG_RETURN_BYTEA_P

関連データ構造

  • VarBit: ビット列を保持する構造体。
  • StringInfoData: 通信用のバイナリバッファ。

失敗モードと境界条件

  • メモリ不足時、palloc 関連の内部処理によりエラーが発生する可能性がある。
  • 入力データが不正な形式（VarBit 構造として不整合）である場合、後続の処理で予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • bit_send 関数とロジックが同一であるため、変更時は両方の関数に影響が及ぶ点に留意すること。
  • ビット列のバイト境界を跨ぐデータ構造を扱うため、VARBITBYTES マクロによるバイト数計算の正確性が重要である。
