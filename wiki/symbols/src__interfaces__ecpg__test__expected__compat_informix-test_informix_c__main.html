目的

ECPG（Embedded SQL for C）における Informix 互換機能の回帰テストを実行するメインエントリポイント。データベース接続からテーブル操作、カーソル処理、および Informix 固有のデータ型（decimal）の動作検証を行う。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は使用しない）
  • 出力: 正常終了時に 0 を返し、エラー発生時に 1 を返して終了する。標準出力および標準エラー出力にテスト結果やデバッグログを出力する。

前提条件と不変条件

  • 実行環境において ecpg1_regression データベースへの接続が可能であること。
  • sqlca 構造体が SQL 操作の結果を保持するグローバル変数として利用可能であること。
  • 各 SQL 文の実行後、sqlca.sqlcode を確認し、負の値の場合は dosqlprint() を呼び出してエラーを表示する不変のフローを持つ。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、データベースへ接続する。 2. **テーブル作成**: test テーブルを作成する。 3. **データ操作**: decimal 型を用いた INSERT を実行し、制約違反のテストやトランザクション制御（commit/rollback）を行う。 4. **カーソル処理**: openit() を呼び出してカーソルを宣言し、fetch ループで結果セットを走査する。risnull で NULL 判定を行い、dectoint で decimal 型を整数へ変換して表示する。 5. **演算と削除**: decadd を用いて decimal 型の加算を行い、その結果に基づいた DELETE 文を実行する。 6. **終了**: 最後にテーブルの存在確認を行い、プログラムを終了する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先:
  • ECPGconnect, ECPGdo, ECPGtrans, ECPGset_var: ECPG ライブラリ関数。
  • dosqlprint(): エラーハンドリング用ヘルパー関数。
  • openit(): カーソル宣言用補助関数。
  • rsetnull, risnull, deccvint, dectoint, decadd: Informix 互換ライブラリ関数。

関連データ構造

  • sqlca: SQL 通信領域。実行結果のステータスやエラーメッセージを保持。
  • decimal: Informix 互換の数値型。

失敗モードと境界条件

  • データベース接続失敗時: exit(1) で終了。
  • SQL エラー発生時: sqlca.sqlcode < 0 の場合に dosqlprint() を呼び出し、必要に応じてロールバックを実行。
  • 一意制約違反やサブクエリの行数超過など、意図的なエラーケースがテストに含まれており、それらが正しく検出されることを検証する。

保守担当者向けメモ

  • 本関数は自動生成されたテストコードであるため、直接編集しないこと。
  • decimal 型の演算は PostgreSQL ネイティブではなく、Informix 互換ライブラリを介して処理される点に留意すること。
  • 実行時の詳細なログは ECPGdebug(1, stderr) により標準エラー出力へ出力される。
