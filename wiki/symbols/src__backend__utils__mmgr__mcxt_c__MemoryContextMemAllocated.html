MemoryContextMemAllocated

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/mcxt.c:767

目的

指定されたメモリコンテキスト、およびオプションでそのすべての子孫コンテキストが現在割り当てているメモリの総量を計算します。

シグネチャまたは形状

Size MemoryContextMemAllocated(MemoryContext context, bool recurse)

入力と出力

  • **入力**:
  • context: 対象となる MemoryContext 構造体へのポインタ。
  • recurse: true の場合、子孫コンテキストのメモリ量も合計に含めます。
  • **出力**:
  • Size: 割り当て済みメモリの合計バイト数。

前提条件と不変条件

  • context は MemoryContextIsValid(context) を満たす有効なポインタである必要があります。

アルゴリズムまたは振る舞い

1. 指定された context の mem_allocated フィールドの値を初期合計値として取得します。 2. recurse が true の場合、MemoryContextTraverseNext を用いて、コンテキストツリーの直下の子から順に非再帰的なトラバーサルを行います。 3. トラバーサル中に見つかった各子孫コンテキストの mem_allocated を合計値に加算します。 4. 最終的な合計値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAgg.c (集計処理におけるメモリ統計取得)
  • src/include/lib/radixtree.h
  • **呼び出し先**:
  • MemoryContextIsValid (マクロ)
  • MemoryContextTraverseNext (コンテキストツリーのトラバーサル用関数)

関連データ構造

  • MemoryContext: メモリコンテキストの基底構造体。mem_allocated フィールドおよび firstchild ポインタを保持します。

失敗モードと境界条件

  • 無効な context ポインタが渡された場合、Assert によりデバッグビルドで停止します。
  • recurse が false の場合、子孫コンテキストのメモリ量は無視され、対象コンテキスト単体の値のみが返されます。

保守担当者向けメモ

  • 本関数は再帰を避けるため MemoryContextTraverseNext を使用しています。コンテキストツリーの構造を変更する際は、このトラバーサルロジックが整合性を保っているか確認してください。
