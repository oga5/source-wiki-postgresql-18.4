testleftright

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_rbtree/test_rbtree.c:163

目的

PostgreSQL の赤黒木ライブラリにおける LeftRightWalk（昇順走査）の正当性を検証する。

シグネチャまたは形状

static void testleftright(int size)

入力と出力

  • **入力**: size (int) - 挿入するノードの総数。
  • **出力**: なし。検証失敗時には elog(ERROR, ...) を通じて例外を発生させる。

前提条件と不変条件

  • create_int_rbtree() によって初期化された空の赤黒木が使用される。
  • rbt_populate(tree, size, 1) により、1 から size までの連続した整数がキーとして挿入される。
  • 走査結果は、キーが昇順（node->key > lastKey）で、かつ全要素が網羅されている必要がある。

アルゴリズムまたは振る舞い

1. 空の木に対して rbt_begin_iterate を実行し、要素が取得されないことを確認する。 2. rbt_populate を用いて木を構築する。 3. rbt_begin_iterate に LeftRightWalk を指定してイテレータを初期化する。 4. rbt_iterate をループで呼び出し、以下の不変条件を検証する。

  • 各ノードのキーが直前のキーより大きいこと。

    • 5. ループ終了後、最終キーが size - 1 であること、および走査した要素数が size と一致することを確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_rbtree/test_rbtree.c 内のテスト実行ロジック（test_rb_tree 等から呼び出される）。
  • **呼び出し先**:
  • create_int_rbtree()
  • rbt_begin_iterate()
  • rbt_iterate()
  • rbt_populate()
  • elog()

関連データ構造

  • RBTree: 検証対象の赤黒木構造体。
  • RBTreeIterator: 走査用イテレータ。
  • IntRBTreeNode: テスト用のノード構造体。

失敗モードと境界条件

  • **空の木**: rbt_iterate が要素を返した場合にエラーとなる。
  • **順序の不整合**: 走査中にキーが昇順でない（node->key <= lastKey）場合にエラーとなる。
  • **要素の欠落**: 最終キーが期待値と異なる、または走査した要素数が size と一致しない場合にエラーとなる。

保守担当者向けメモ

  • 本関数は LeftRightWalk のロジックのみを検証する。逆順走査の検証には testrightleft を参照すること。
  • 乱数生成器の状態には依存せず、常に連続した整数値でテストを行う設計となっている。
