ginxlogInsertListPage

定義

  • 種別: struct
  • 定義箇所: src/include/access/ginxlog.h:182

目的

GINインデックスのリストページ（List Page）へのタプル挿入操作をWAL（Write-Ahead Log）に記録するためのデータ構造です。

シグネチャまたは形状

typedef struct ginxlogInsertListPage
{
	BlockNumber rightlink;
	int32		ntuples;
	/* array of inserted tuples follows */
} ginxlogInsertListPage;

入力と出力

  • **入力**: 挿入対象のタプル数（ntuples）およびリンク先ブロック番号（rightlink）。
  • **出力**: 構造体直後に続く可変長のタプルデータ配列。

前提条件と不変条件

  • 構造体の直後に、ntuples で指定された数のインデックスタプルがメモリ上で連続して配置されている必要があります。
  • WALレコードとして書き込まれる際、アライメントおよびパディングがバイナリ互換性を損なわないよう管理される必要があります。

アルゴリズムまたは振る舞い

1. **記録時**: ginfast.c において、挿入するタプル情報とともに本構造体がWALレコードとして登録されます。 2. **再生時**: ginxlog.c 内で XLogRecGetData を通じて本構造体が読み取られ、リカバリ処理において対象ページへのタプル挿入と rightlink の更新が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginfast.c（WALレコードの作成および登録）。
  • **呼び出し先**: src/backend/access/gin/ginxlog.c（WALレコードの読み取りおよびREDO処理）。

関連データ構造

  • BlockNumber: ページ識別子。
  • XLogRecGetData: WALレコードから本構造体を取り出すために使用される関数。

失敗モードと境界条件

  • 構造体定義のフィールド順序やサイズを変更した場合、既存のWALログとのバイナリ互換性が失われ、リカバリ不能になるリスクがあります。
  • ntuples と実際のデータサイズが一致しない場合、メモリ破壊や不正なWAL再生を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は可変長データを含むため、フィールドの追加や変更を行う際は、後続するタプル配列のオフセット計算に影響がないか注意してください。
  • pg_waldump 等で内容を確認する際は、gin_desc 関数が正しく本構造体を解釈しているか確認してください。
