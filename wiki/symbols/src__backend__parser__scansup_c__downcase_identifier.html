downcase_identifier

定義

  • 種別: function
  • 定義箇所: src/backend/parser/scansup.c:45

目的

SQL 識別子を小文字に変換し、必要に応じて NAMEDATALEN に基づく切り詰め処理を行うことで、データベース内部の標準形式へ正規化します。

シグネチャまたは形状

char *downcase_identifier(const char *ident, int len, bool warn, bool truncate)

入力と出力

  • **入力**:
  • ident: 変換対象の文字列ポインタ。
  • len: 文字列の長さ。
  • warn: 切り詰め発生時に警告を発するかどうか。
  • truncate: NAMEDATALEN を超える場合に切り詰めを行うかどうか。
  • **出力**: 変換および必要に応じた切り詰めが適用された、palloc 済みの新しい文字列（呼び出し側で解放が必要）。

前提条件と不変条件

  • 戻り値の文字列は palloc によってメモリ確保されるため、呼び出し側で管理する必要があります。
  • ロケール依存の tolower() を使用するため、Turkish 等の特殊なロケールでは期待と異なる挙動を示す可能性があります。

アルゴリズムまたは振る舞い

1. palloc で len + 1 バイトのメモリを確保します。 2. 文字列の各文字をループで走査します。

  • ASCII の 'A' から 'Z' は直接小文字化します。
  • シングルバイトエンコーディング環境かつ上位ビットが立っている文字は、isupper() および tolower() を使用してロケール依存の変換を行います。

    • 3. 処理後の文字列をヌル終端します。 4. truncate が真かつ文字列長が NAMEDATALEN 以上の場合、truncate_identifier を呼び出して切り詰めと警告（warn が真の場合）を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/scansup.c (内部ラッパー)
  • src/backend/utils/adt/misc.c
  • src/bin/psql/tab-complete.in.c
  • **呼び出し先**:
  • palloc (メモリ確保)
  • pg_database_encoding_max_length
  • isupper, tolower (文字変換)
  • truncate_identifier (条件付き切り詰め)

関連データ構造

  • NAMEDATALEN: 識別子の最大長を定義する定数。

失敗モードと境界条件

  • len が NAMEDATALEN を超える場合、truncate 引数が真であれば truncate_identifier により切り詰めが発生します。
  • マルチバイト文字環境では、文字の境界を壊さないよう truncate_identifier 内で pg_mbcliplen が使用されます。

保守担当者向けメモ

  • SQL 標準が求める Unicode 準拠の正規化には対応していません。
  • tolower() の使用はロケールに依存するため、特定の言語環境下での識別子比較に影響を与える可能性があります。
