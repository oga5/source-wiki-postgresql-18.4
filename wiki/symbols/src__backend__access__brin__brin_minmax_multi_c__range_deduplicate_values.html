range_deduplicate_values

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_minmax_multi.c:515

目的

Ranges 構造体内の未ソート領域にある値をソートし、重複を除去することで、データセットを正規化された状態に保つ。

シグネチャまたは形状

static void range_deduplicate_values(Ranges *range)

入力と出力

  • **入力**: Ranges *range - 処理対象の Ranges 構造体へのポインタ。
  • **出力**: なし（引数 range の values 配列、nvalues、nsorted フィールドを直接更新する）。

前提条件と不変条件

  • range->values のうち、2 * range->nranges 以降の領域がソート対象として扱われる。
  • 処理終了後、range->nvalues と range->nsorted は重複除去後の値の数に更新される。
  • 終了時に AssertCheckRanges が呼び出され、データ構造の整合性（ソート済みであること等）が検証される。

アルゴリズムまたは振る舞い

1. range->nsorted == range->nvalues の場合、処理を即座に終了する。 2. qsort_arg を使用し、range->values の未ソート領域を compare_values 関数に基づいて昇順にソートする。 3. ソートされた配列を走査し、隣接する値が等しい場合はスキップすることで重複を除去する。 4. 重複除去後の値を配列の先頭（未ソート領域の開始位置）に詰め直す。 5. range->nvalues および range->nsorted を新しいカウント値に更新する。

関連データ構造

  • Ranges: 値の保持および管理を行う主要構造体。
  • compare_context: ソート時に使用される比較コンテキスト。

失敗モードと境界条件

  • nsorted == nvalues の場合、何も行わず正常終了する。
  • すべての値が重複している場合、nvalues は 1 になる。

保守担当者向けメモ

  • 現在のソート処理は qsort_arg を使用しているが、ソース内のコメントにある通り、既存のソート済み領域を活用するマージソートへの最適化の余地がある。
  • compare_values の結果に依存するため、データ型ごとの比較ロジックが正しく実装されていることが前提となる。
