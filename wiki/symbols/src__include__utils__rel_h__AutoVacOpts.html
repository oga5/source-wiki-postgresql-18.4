AutoVacOpts

定義

  • 種別: struct
  • 定義箇所: src/include/utils/rel.h:311

目的

リレーション（テーブル等）に対する自動バキューム（autovacuum）および自動アナライズ（autoanalyze）の動作パラメータを保持する構造体です。

シグネチャまたは形状

typedef struct AutoVacOpts
{
    bool        enabled;
    int         vacuum_threshold;
    int         vacuum_max_threshold;
    int         vacuum_ins_threshold;
    int         analyze_threshold;
    int         vacuum_cost_limit;
    int         freeze_min_age;
    int         freeze_max_age;
    int         freeze_table_age;
    int         multixact_freeze_min_age;
    int         multixact_freeze_max_age;
    int         multixact_freeze_table_age;
    int         log_min_duration;
    float8      vacuum_cost_limit;
    float8      vacuum_scale_factor;
    float8      vacuum_ins_scale_factor;
    float8      analyze_scale_factor;
} AutoVacOpts;

入力と出力

本構造体は、リレーションのオプション設定としてメモリ上に保持されます。直接的な関数入出力ではなく、StdRdOptions 構造体の一部として格納され、リレーションのメタデータとして参照されます。

前提条件と不変条件

  • rd_options フィールド内に格納される bytea 形式のデータからキャストして利用されることが想定されています。
  • 各フィールドの値は、対応するリレーションのストレージ設定を反映します。

アルゴリズムまたは振る舞い

自動バキュームの実行判定やコスト制御を行う際、本構造体の各閾値（vacuum_threshold 等）やスケーリング係数（vacuum_scale_factor 等）が参照されます。これにより、テーブルごとの個別の自動バキューム設定が適用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/reloptions.c において、StdRdOptions 構造体内のオフセット計算を通じて、リレーションのオプション解析時に参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • StdRdOptions: 本構造体をメンバとして保持する、リレーションのオプション設定用構造体。
  • RelationData: リレーション記述子の本体。rd_options を通じて本構造体へアクセスします。

失敗モードと境界条件

  • rd_options が適切に初期化されていない場合、または不正なメモリレイアウトでキャストされた場合に予期せぬ値が読み取られる可能性があります。

保守担当者向けメモ

  • 本構造体のフィールドを追加・変更する場合は、src/backend/access/common/reloptions.c のオプション定義テーブルも併せて更新する必要があります。
  • フィールドへの直接アクセスは避け、必要に応じてアクセサマクロや関数を介して利用してください。
