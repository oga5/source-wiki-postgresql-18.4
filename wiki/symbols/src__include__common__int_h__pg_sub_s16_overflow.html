pg_sub_s16_overflow

定義

  • 種別: function
  • 定義箇所: src/include/common/int.h:84

目的

int16 型の減算において、結果が型の表現範囲（PG_INT16_MIN から PG_INT16_MAX）を超えるか否かを判定し、安全に演算を行う。

シグネチャまたは形状

static inline bool pg_sub_s16_overflow(int16 a, int16 b, int16 *result)

入力と出力

  • **入力**:
  • a: 被減数 (int16)
  • b: 減数 (int16)
  • result: 演算結果を格納するポインタ (int16 *)
  • **出力**:
  • 戻り値: オーバーフローが発生した場合は true、それ以外は false を返す。
  • *result: 演算が成功した場合は計算結果、オーバーフローした場合はダミー値 0x5EED が格納される。

前提条件と不変条件

  • int16 型の演算において、コンパイラ依存の未定義動作を回避する。
  • HAVE__BUILTIN_OP_OVERFLOW が定義されている環境では、コンパイラ組み込み関数を利用する。

アルゴリズムまたは振る舞い

1. HAVE__BUILTIN_OP_OVERFLOW が定義されている場合、__builtin_sub_overflow を呼び出して結果を返す。 2. 未定義の場合、int16 を int32 にキャストして減算を行い、その結果が PG_INT16_MAX を超えるか、PG_INT16_MIN 未満であるかを判定する。 3. 範囲外であれば *result に 0x5EED を代入し true を返す。 4. 範囲内であれば *result にキャストした計算結果を代入し false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_int2.c:103
  • src/backend/utils/adt/int.c:955
  • **呼び出し先**:
  • コンパイラ組み込み関数 __builtin_sub_overflow（利用可能な場合）。

関連データ構造

  • int16, int32: PostgreSQL で定義された固定長整数型。

失敗モードと境界条件

  • a - b の結果が int16 の範囲（-32768 ～ 32767）を逸脱する場合にオーバーフローとして検出される。

保守担当者向けメモ

  • 本関数はインライン関数であり、ヘッダーファイル内で定義されている。
  • *result に書き込まれる 0x5EED は、デバッグ時の警告回避を目的としたダミー値である。
