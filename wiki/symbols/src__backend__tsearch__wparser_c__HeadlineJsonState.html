HeadlineJsonState

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/wparser.c:34

目的

JSONまたはJSONBデータ内の文字列値に対して、全文検索のハイライト処理を再帰的に適用する際の状態を保持します。

シグネチャまたは形状

typedef struct HeadlineJsonState
{
	HeadlineParsedText *prs;
	TSConfigCacheEntry *cfg;
	TSParserCacheEntry *prsobj;
	TSQuery		query;
	List	   *prsoptions;
	bool		transformed;
} HeadlineJsonState;

入力と出力

本構造体は関数間の状態受け渡しに使用されます。

  • **入力**: ハイライト生成に必要な設定（cfg）、パーサー（prsobj）、クエリ（query）、および解析済みテキスト（prs）を保持します。
  • **出力**: transformed フラグにより、JSON内の文字列がハイライト処理によって変更されたかどうかの状態を追跡します。

前提条件と不変条件

  • palloc0 を使用してゼロ初期化された状態でインスタンス化されることが想定されています。
  • ハイライト処理の実行中、cfg および prsobj はキャッシュから取得された有効なエントリを指している必要があります。

アルゴリズムまたは振る舞い

JSON/JSONBのハイライト処理において、headline_json_value 等のコールバック関数がこの構造体を介してコンテキストを共有します。再帰的なJSON走査中に、現在の解析状態とクエリ情報を保持し、各文字列値に対して一貫したハイライト適用を可能にします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/wparser.c 内の ts_headline_jsonb_byid_opt 等のハイライト生成関数が、この構造体を割り当てて初期化します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しの引数として headline_json_value 等の処理関数へ渡されます。

関連データ構造

  • HeadlineParsedText: 解析済みテキストの保持に使用。
  • TSConfigCacheEntry, TSParserCacheEntry: 全文検索の設定およびパーサーのキャッシュ情報。
  • TSQuery: ハイライト対象となる検索クエリ。

失敗モードと境界条件

  • メモリ不足時: palloc0 によるメモリ確保が失敗した場合、PostgreSQLの標準的なエラー処理によりアボートします。
  • 無効なキャッシュ: cfg や prsobj が NULL の状態で処理が進行した場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は ts_headline 系のJSON対応関数群において、再帰的な状態管理を行うための中心的な役割を果たします。
  • transformed フラグは、JSON構造の変更有無を判定するために使用されており、不要なコピーを避けるための最適化に寄与しています。
