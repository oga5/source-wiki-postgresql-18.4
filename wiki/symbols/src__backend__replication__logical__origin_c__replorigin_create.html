replorigin_create

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/origin.c:256

目的

論理レプリケーションにおける新しい「レプリケーション起点（Replication Origin）」を作成し、カタログ pg_replication_origin に登録して、割り当てられた一意な16ビットIDを返します。

シグネチャまたは形状

RepOriginId replorigin_create(const char *roname)

入力と出力

  • **入力**: roname (起点名。最大512バイトまで)
  • **出力**: RepOriginId (割り当てられた16ビットの識別子)
  • **例外**: 名前が長すぎる場合、または利用可能なIDが見つからない場合にエラーを報告します。

前提条件と不変条件

  • トランザクション状態であること (Assert(IsTransactionState()))。
  • pg_replication_origin カタログにTOASTテーブルが存在しないこと。
  • 起点名は MAX_RONAME_LEN (512バイト) 以下であること。

アルゴリズムまたは振る舞い

1. 名前長を検証し、超過していればエラーを発生させます。 2. pg_replication_origin テーブルを ExclusiveLock でロックします。 3. DirtySnapshot を使用して、カタログ内を走査し、未使用の16ビットID（1〜65535）を探索します。 4. 未使用IDを発見した場合、そのIDと名前をカタログに挿入し、CommandCounterIncrement() を実行して可視化します。 5. テーブルのロックを解放し、割り当てたIDを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/subscriptioncmds.c
  • src/backend/replication/logical/origin.c

関連データ構造

  • pg_replication_origin (カタログ)
  • RepOriginId (16ビットの識別子型)

失敗モードと境界条件

  • **名前長超過**: 512バイトを超える名前を指定すると ERRCODE_PROGRAM_LIMIT_EXCEEDED で失敗します。
  • **ID枯渇**: 16ビットの範囲内で未使用のIDが見つからない場合、エラーとなります。
  • **同時実行**: ExclusiveLock により、探索中の競合を防いでいます。

保守担当者向けメモ

  • 本関数は pg_replication_origin のIDを16ビットに制限しているため、最大ノード数は65,535に制限されます。
  • カタログにTOASTテーブルを追加する場合は、スナップショット管理の要件が変更されるため注意が必要です。
  • ID探索は全走査を行うため、頻繁な呼び出しには適していません。
