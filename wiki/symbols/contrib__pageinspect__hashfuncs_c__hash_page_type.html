hash_page_type

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/hashfuncs.c:191

目的

PostgreSQL ハッシュインデックスの物理ページ（bytea）を解析し、そのページタイプを文字列として返します。

シグネチャまたは形状

Datum hash_page_type(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_BYTEA_P(0) を介して取得される、ハッシュインデックスの物理ページデータ（bytea）。
  • **出力**: ページタイプを表す text 型の文字列（metapage, overflow, bucket, bitmap, unused のいずれか）。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要です（superuser() チェック）。
  • 入力データは verify_hash_page による検証を通過する必要があります。

アルゴリズムまたは振る舞い

1. スーパーユーザー権限を確認し、不足している場合はエラーを発生させます。 2. verify_hash_page を呼び出し、入力された bytea が有効なハッシュインデックスページであることを検証します。 3. PageIsNew を使用してページが初期化済みか判定し、未初期化の場合は "unused" を返します。 4. 初期化済みの場合、HashPageGetOpaque を介してページヘッダーの不透明領域（opaque）を取得します。 5. hasho_flag に対して LH_PAGE_TYPE マスクを適用し、ビットフラグに基づいてページタイプ（メタ、オーバーフロー、バケット、ビットマップ）を特定します。 6. 特定したタイプを text 型に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されます（PG_FUNCTION_INFO_V1 マクロによる登録あり）。
  • **呼び出し先**: superuser(), verify_hash_page(), PageIsNew(), HashPageGetOpaque(), PG_RETURN_TEXT_P(), cstring_to_text()。

関連データ構造

  • HashPageOpaqueData（access/hash.h で定義）: ページヘッダーの不透明領域。hasho_flag を保持します。

失敗モードと境界条件

  • スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE エラーが発生します。
  • verify_hash_page がページヘッダーの整合性エラーを検出した場合、エラーが発生します。
  • ページタイプが既知のフラグと一致しない場合、デフォルトで "unused" が返されます。

保守担当者向けメモ

  • 本関数は物理的なページレイアウトに依存しているため、PostgreSQL のハッシュインデックス内部構造（access/hash.h）に変更があった場合は追従が必要です。
  • ページタイプ判定ロジックは hasho_flag のビットマスクに依存しています。
