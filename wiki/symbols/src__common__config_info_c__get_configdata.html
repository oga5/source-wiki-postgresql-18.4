get_configdata

定義

  • 種別: function
  • 定義箇所: src/common/config_info.c:32

目的

PostgreSQLのビルド時設定情報（インストールディレクトリやコンパイルオプションなど）を収集し、ConfigData 構造体の配列として提供する。

シグネチャまたは形状

ConfigData *get_configdata(const char *my_exec_path, size_t *configdata_len);

入力と出力

  • **入力**:
  • my_exec_path: 実行ファイルのパス。各ディレクトリパスの算出基点として使用される。
  • configdata_len: 取得した設定項目の数を格納するための出力用ポインタ。
  • **出力**:
  • 戻り値: 設定名と値のペアを保持する ConfigData 構造体の配列へのポインタ。

前提条件と不変条件

  • 戻り値の配列は palloc_array で確保されるため、呼び出し元が pfree を使用してメモリを解放する必要がある。
  • configdata_len には常に 23 が設定される。
  • 内部のインデックス i は、最終的に *configdata_len と一致しなければならない（Assert による検証あり）。

アルゴリズムまたは振る舞い

1. configdata_len を 23 に設定し、メモリを確保する。 2. my_exec_path を基に、get_doc_path や get_lib_path などのヘルパー関数を呼び出し、各ディレクトリパスを算出・正規化して格納する。 3. PGXS パスを get_pkglib_path の結果を基に構築する。 4. CONFIGURE 引数やコンパイル時フラグ（CFLAGS, LDFLAGS 等）を順次格納する。 5. マクロが未定義の場合は「not recorded」という文字列を格納する。 6. 最後に VERSION を格納し、配列を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/pg_config.c:33
  • src/bin/pg_config/pg_config.c:157
  • **呼び出し先**:
  • get_doc_path, get_html_path, get_include_path, get_pkginclude_path, get_includeserver_path, get_lib_path, get_pkglib_path, get_locale_path, get_man_path, get_share_path, get_etc_path
  • palloc_array, pstrdup, cleanup_path, strlcpy, strrchr, strlcat

関連データ構造

  • ConfigData: 設定名（name）と値（setting）のペアを保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc_array がエラーを発生させる可能性がある。
  • my_exec_path が無効なパスである場合、算出される各ディレクトリパスが不正確になる可能性がある。

保守担当者向けメモ

  • configdata_len を変更する場合は、関数内のインデックス i のカウントと必ず整合させること。
  • 本関数はフロントエンドおよびバックエンドの両方から利用されるため、変更時は両環境への影響を考慮すること。
