pg_wal_summary_contents

定義

  • 種別: function
  • 定義箇所: src/backend/backup/walsummaryfuncs.c:68

目的

指定されたWALサマリーファイルの内容（リレーション、フォーク、変更されたブロック番号）を解析し、SQLから参照可能な形式で出力するSet Returning Function (SRF) です。

シグネチャまたは形状

Datum pg_wal_summary_contents(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • tli (int64): タイムラインID（1以上 PG_INT32_MAX 以下）。
  • start_lsn (pg_lsn): サマリーの開始LSN。
  • end_lsn (pg_lsn): サマリーの終了LSN。
  • **出力**:
  • tuplestore を介して、リレーション情報、フォーク番号、ブロック番号、およびそれが制限ブロック（limit_block）であるかを示すフラグを含む行セットを返します。

前提条件と不変条件

  • InitMaterializedSRF を呼び出し、結果セットを初期化する必要があります。
  • タイムラインIDが範囲外（1未満または PG_INT32_MAX 超）の場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告します。
  • 処理中、長時間実行されるクエリのキャンセル要求に応答するため、ループ内で CHECK_FOR_INTERRUPTS() を実行します。

アルゴリズムまたは振る舞い

1. 引数からタイムラインIDとLSN範囲を取得し、検証します。 2. OpenWalSummaryFile で対象のサマリーファイルを開き、CreateBlockRefTableReader で読み取り用リーダーを初期化します。 3. BlockRefTableReaderNextRelation を使用して、リレーション単位でループ処理を行います。 4. リレーションごとに limit_block が有効な場合、そのブロック番号と limit_block = true を示す行を出力します。 5. BlockRefTableReaderGetBlocks を使用して、変更されたブロック番号を最大 MAX_BLOCKS_PER_CALL (256) 個ずつ取得し、limit_block = false として出力します。 6. 最後に DestroyBlockRefTableReader および FileClose を呼び出し、リソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます（直接的なC言語レベルの呼び出し元はソース内に明示されていません）。
  • **呼び出し先**:
  • InitMaterializedSRF
  • OpenWalSummaryFile
  • CreateBlockRefTableReader
  • BlockRefTableReaderNextRelation
  • BlockRefTableReaderGetBlocks
  • DestroyBlockRefTableReader
  • FileClose
  • heap_form_tuple
  • tuplestore_puttuple

関連データ構造

  • WalSummaryFile: サマリーファイルのメタデータ（TLI、LSN範囲）。
  • WalSummaryIO: ファイル読み込み用のI/O状態。
  • BlockRefTableReader: ブロック参照テーブルの読み込み用リーダー。
  • RelFileLocator: リレーションの識別情報。

失敗モードと境界条件

  • タイムラインIDが不正な場合、エラーをスローします。
  • limit_block が InvalidBlockNumber の場合、その行は出力されません。
  • 大量のブロック情報を含むサマリーを処理する場合、メモリ消費量が増大する可能性があります。

保守担当者向けメモ

  • 本関数はSRFであるため、tuplestore を使用したメモリ管理が行われています。
  • MAX_BLOCKS_PER_CALL は一度に処理するブロック数の上限であり、変更が必要な場合はこの定数を確認してください。
  • 読み込みエラーが発生した場合は ReportWalSummaryError が使用されます。
