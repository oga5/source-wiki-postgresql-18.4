pgp_mpi_free

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-mpi.c:69

目的

PGP_MPI 構造体によって確保されたメモリを安全に解放し、機密情報がメモリ上に残存しないようゼロクリアを行う。

シグネチャまたは形状

int pgp_mpi_free(PGP_MPI *mpi)

入力と出力

  • **入力**: PGP_MPI *mpi （解放対象の MPI 構造体へのポインタ）
  • **出力**: int （常に 0 を返す）

前提条件と不変条件

  • mpi が NULL の場合、関数は何もしない（安全に終了する）。
  • mpi が指す構造体は、pgp_mpi_alloc 等によって動的に確保されたものである必要がある。

アルゴリズムまたは振る舞い

1. mpi が NULL かどうかを判定し、該当する場合は直ちに 0 を返す。 2. px_memset を使用し、PGP_MPI 構造体本体およびその内部のデータ領域（mpi->bytes 分）をゼロで上書きする。 3. pfree を呼び出し、メモリを解放する。 4. 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-mpi-openssl.c
  • contrib/pgcrypto/pgp-mpi.c
  • contrib/pgcrypto/pgp-pubdec.c
  • contrib/pgcrypto/pgp-pubenc.c
  • **呼び出し先**:
  • px_memset (機密情報の消去用)
  • pfree (PostgreSQL のメモリ解放関数)

関連データ構造

  • PGP_MPI: MPI データを保持する構造体。ビット数、バイト数、データ本体へのポインタを含む。

失敗モードと境界条件

  • mpi が NULL の場合、エラーにはならず正常終了する。
  • mpi が指すメモリ領域が不正な場合、px_memset または pfree の実行時に未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は機密情報の漏洩を防ぐため、解放前に px_memset によるゼロクリアを必須としている。メモリ管理を行う際は必ず本関数を経由すること。
