wait_result_to_exit_code

定義

  • 種別: function
  • 定義箇所: src/common/wait_error.c:137

目的

wait(2) や waitpid(2) 等から得られるプロセス終了ステータスを、シェルが期待する標準的な 0-255 の終了コードに正規化する。

シグネチャまたは形状

int wait_result_to_exit_code(int exit_status);

入力と出力

  • **入力**: exit_status (int) - wait システムコール等から返されるプロセス終了ステータス。
  • **出力**: (int) - 変換後の終了コード（0-255）、またはエラー時は -1。

前提条件と不変条件

  • 入力値は wait 系システムコールが返す有効なステータス値であること。
  • POSIX 準拠の環境を想定しており、シグナル終了時は 128 + シグナル番号 を返す慣習に従う。

アルゴリズムまたは振る舞い

1. exit_status が -1 の場合、エラーとして -1 を返す。 2. WIFEXITED(exit_status) が真の場合、WEXITSTATUS(exit_status) を用いて通常の終了コードを抽出して返す。 3. WIFSIGNALED(exit_status) が真の場合、128 + WTERMSIG(exit_status) を計算して返す。 4. 上記のいずれにも該当しない場合、-1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/common.c:522 にて、wait_result を変換するために使用されている。
  • **呼び出し先**: WIFEXITED, WEXITSTATUS, WIFSIGNALED, WTERMSIG マクロ（<sys/wait.h>）。

関連データ構造

  • 特になし（標準的な整数型とシステム定義のマクロを使用）。

失敗モードと境界条件

  • exit_status が -1 の場合、システムコール失敗として -1 を返す。
  • 正常に終了ステータスを判定できない場合、-1 を返す（多くのシステムでは到達不能）。

保守担当者向けメモ

  • Windows 環境では WTERMSIG 等の扱いが異なるため、プラットフォーム依存の差異に注意すること。
  • 本関数は src/include/port.h で宣言されており、フロントエンド・バックエンド双方から利用可能である。
