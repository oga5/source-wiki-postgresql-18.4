BackendMain

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/backend_startup.c:75

目的

PostgreSQLのバックエンドプロセスにおける初期化フェーズを統括し、クライアントとの接続確立からクエリ処理ループへの移行までを実行するエントリポイントです。

シグネチャまたは形状

void BackendMain(const void *startup_data, size_t startup_data_len)

入力と出力

  • **入力**:
  • startup_data: BackendStartupData 構造体へのポインタ。
  • startup_data_len: startup_data のサイズ。sizeof(BackendStartupData) と一致する必要があります。
  • **出力**: なし（プロセスは PostgresMain へ移行するか、エラーにより終了します）。

前提条件と不変条件

  • MyClientSocket が有効（NULLではない）であること。
  • startup_data_len が sizeof(BackendStartupData) と等しいこと。

アルゴリズムまたは振る舞い

1. **SSL初期化**: EXEC_BACKEND が定義されている環境下で、SSLが有効な場合は secure_initialize を呼び出しSSLコンテキストを再構築します。 2. **接続初期化**: BackendInitialize を呼び出し、通信層の準備およびクライアントからのスタートアップパケットの収集を行います。 3. **プロセス登録**: InitProcess を呼び出し、共有メモリ上にバックエンド固有の PGPROC 構造体を作成します（LWLocksや共有メモリへのアクセスに必須）。 4. **メモリ管理**: MemoryContextSwitchTo を使用して、メモリコンテキストを TopMemoryContext に切り替えます。 5. **メイン処理への移行**: PostgresMain を呼び出し、データベース名とユーザー名を渡してクエリ処理フェーズへ制御を移します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/launch_backend.c (バックエンド起動時のエントリとして参照)
  • **呼び出し先**:
  • BackendInitialize
  • InitProcess
  • MemoryContextSwitchTo
  • PostgresMain
  • secure_initialize (条件付き)

関連データ構造

  • BackendStartupData: 起動時に必要なパラメータを保持する構造体。
  • PGPROC: バックエンドプロセスが共有メモリ上で管理されるための構造体。

失敗モードと境界条件

  • startup_data_len が期待値と異なる場合、Assert により停止します。
  • SSLの初期化に失敗した場合、ログを出力しつつSSLなしで処理を継続します。

保守担当者向けメモ

  • InitProcess() を呼び出すまでは共有メモリへのアクセスが禁止されています。
  • PostmasterContext は InitPostgres がHBAデータを使用するため、この時点ではまだ削除できません。
