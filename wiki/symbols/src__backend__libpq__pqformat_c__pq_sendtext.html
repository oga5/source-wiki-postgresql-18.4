pq_sendtext

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqformat.c:171

目的

テキスト文字列をエンコーディング変換した上で、StringInfo バッファに追加する。

シグネチャまたは形状

void pq_sendtext(StringInfo buf, const char *str, int slen)

入力と出力

  • buf: 追加先の StringInfo バッファ（入力および出力）。
  • str: 追加するテキスト文字列のポインタ（入力）。
  • slen: 文字列の長さ（入力）。

前提条件と不変条件

  • str は少なくとも slen バイトの有効なメモリ領域を指していること。
  • buf は初期化済みの StringInfo であること。

アルゴリズムまたは振る舞い

1. pg_server_to_client を呼び出し、サーバーからクライアントへの文字エンコーディング変換を試みる。 2. 変換が発生した場合（戻り値のポインタが元の str と異なる場合）:

  • 変換後の文字列長を strlen で取得する。
  • appendBinaryStringInfo を使用して変換後の文字列を buf に追加する。
  • 変換時に確保されたメモリを pfree で解放する。

    • 3. 変換が発生しなかった場合:

  • 元の str をそのまま appendBinaryStringInfo で buf に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_io.c
  • contrib/ltree/ltree_io.c
  • contrib/ltree/ltxtquery_io.c
  • src/backend/utils/adt/enum.c
  • src/backend/utils/adt/json.c
  • src/backend/utils/adt/jsonb.c
  • src/backend/utils/adt/jsonpath.c
  • **呼び出し先**:
  • pg_server_to_client
  • appendBinaryStringInfo
  • pfree
  • strlen

関連データ構造

  • StringInfo: メッセージ構築に使用される動的バッファ構造体。

失敗モードと境界条件

  • pg_server_to_client が内部でエラーを発生させた場合、処理は中断される可能性がある。
  • slen が実際の文字列長と一致しない場合、バッファへのコピー時に予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は自動的なエンコーディング変換を行う。変換が不要な場合は pq_send_ascii_string や pq_sendbytes を検討すること。
  • 変換後の文字列に対して pfree を行うため、変換が発生した場合はメモリ管理が適切に行われる。
