bbsink_lz4_manifest_contents

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_lz4.c:273

目的

LZ4圧縮シンクにおいて、圧縮対象外であるマニフェストデータを後続のシンクへ透過的に転送する。

シグネチャまたは形状

static void bbsink_lz4_manifest_contents(bbsink *sink, size_t len)

入力と出力

  • sink: 現在のLZ4シンク構造体へのポインタ。
  • len: 転送するマニフェストデータのサイズ（バイト単位）。
  • 出力: なし（副作用として後続シンクへデータを転送）。

前提条件と不変条件

  • sink は有効な bbsink 構造体であり、bbs_next が初期化されていること。
  • sink->bbs_buffer には転送すべきマニフェストデータが格納されていること。

アルゴリズムまたは振る舞い

1. sink->bbs_buffer から sink->bbs_next->bbs_buffer へ、指定されたサイズ len 分のデータを memcpy でコピーする。 2. bbsink_manifest_contents を呼び出し、後続のシンクに対してマニフェストデータの転送を委譲する。

呼び出し元と呼び出し先

  • 呼び出し元: bbsink_lz4_ops 構造体（関数ポインタとして登録）。
  • 呼び出し先: bbsink_manifest_contents（後続のシンク）。

関連データ構造

  • bbsink: バックアップシンクのパイプライン構造体。
  • bbsink_lz4: LZ4圧縮状態を保持する構造体。

失敗モードと境界条件

  • len が sink->bbs_buffer のサイズを超過する場合、メモリ破壊の可能性がある（呼び出し元での保証に依存）。

保守担当者向けメモ

  • 本関数はマニフェストデータが圧縮不要であることを前提としている。圧縮処理を行う archive_contents とは異なり、データ変換を行わずにそのままパススルーする実装である。
