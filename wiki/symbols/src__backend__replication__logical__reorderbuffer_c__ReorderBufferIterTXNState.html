ReorderBufferIterTXNState

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/reorderbuffer.c:168

目的

複数のトランザクションにまたがる変更レコードを、LSN（Log Sequence Number）順にマージしてイテレートするための状態管理構造体です。

シグネチャまたは形状

typedef struct ReorderBufferIterTXNState
{
	binaryheap *heap;
	Size		nr_txns;
	dlist_head	old_change;
	ReorderBufferIterTXNEntry entries[FLEXIBLE_ARRAY_MEMBER];
} ReorderBufferIterTXNState;

入力と出力

  • **入力**: ReorderBufferIterTXNInit 等により、対象となるトランザクション群と変更レコードが本構造体に格納されます。
  • **出力**: ReorderBufferIterTXNNext を通じて、LSN 順にソートされた ReorderBufferChange が順次返されます。

前提条件と不変条件

  • entries は FLEXIBLE_ARRAY_MEMBER を使用しており、動的に確保されたメモリ領域の末尾に配置されます。
  • heap（バイナリヒープ）は、イテレーション中に変更レコードを LSN 順に保持するために使用されます。

アルゴリズムまたは振る舞い

1. **初期化**: ReorderBufferIterTXNInit にて、対象トランザクションの変更リストをバイナリヒープに投入します。 2. **イテレーション**: ReorderBufferIterTXNNext が呼び出されるたびに、ヒープから最小の LSN を持つ変更レコードを取り出し、必要に応じて次のレコードをヒープに補充します。 3. **終了**: 全ての変更レコードが処理されると、イテレータは終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: ReorderBufferProcessTXN 等のトランザクション処理関数から利用されます。
  • **呼び出し先**: ReorderBufferIterTXNNext が本構造体を引数として受け取り、変更レコードの取り出しを行います。

関連データ構造

  • ReorderBufferIterTXNEntry: イテレータが管理する個別のトランザクションエントリ。
  • binaryheap: LSN 順序を維持するために使用される優先度付きキュー。
  • ReorderBufferChange: イテレータが最終的に出力する変更レコード。

失敗モードと境界条件

  • メモリ不足: ReorderBufferIterTXNInit 内での構造体確保時にメモリが不足した場合、エラーが発生します。
  • 空のトランザクション: 変更レコードを含まないトランザクションが渡された場合、イテレータは即座に終了します。

保守担当者向けメモ

  • entries が可変長配列であるため、構造体のサイズ計算には sizeof(ReorderBufferIterTXNState) + (n * sizeof(ReorderBufferIterTXNEntry)) のような形式が使用されています。メモリ確保時の計算ロジックを変更する際は注意してください。
