pgoutput_send_begin

定義

  • 種別: function
  • 定義箇所: src/backend/replication/pgoutput/pgoutput.c:607

目的

論理レプリケーションにおいて、トランザクションの開始（BEGIN）をサブスクライバーへ送信する。

シグネチャまたは形状

static void pgoutput_send_begin(LogicalDecodingContext *ctx, ReorderBufferTXN *txn)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • txn: 送信対象のトランザクション情報（ReorderBufferTXN）。
  • **出力**: なし（副作用としてストリームへデータを書き出す）。

前提条件と不変条件

  • txn->output_plugin_private に PGOutputTxnData が格納されていること。
  • 当該トランザクションに対して、まだ BEGIN メッセージが送信されていないこと（txndata->sent_begin_txn が false であること）。

アルゴリズムまたは振る舞い

1. トランザクションのレプリケーション起点（origin）情報を確認し、送信が必要か判定する。 2. OutputPluginPrepareWrite を呼び出し、書き込みバッファを準備する。 3. logicalrep_write_begin を呼び出し、プロトコルに従って BEGIN メッセージを書き込む。 4. txndata->sent_begin_txn を true に更新し、送信済み状態を記録する。 5. send_repl_origin を呼び出し、レプリケーション起点の情報を送信する。 6. OutputPluginWrite を呼び出し、バッファの内容をストリームへフラッシュする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/pgoutput/pgoutput.c 内のトランザクション処理フロー（pgoutput_change 等の文脈から呼び出される）。
  • **呼び出し先**:
  • OutputPluginPrepareWrite
  • logicalrep_write_begin
  • send_repl_origin
  • OutputPluginWrite

関連データ構造

  • LogicalDecodingContext: デコーディングセッションの状態管理。
  • ReorderBufferTXN: トランザクションのメタデータ（LSN、origin情報など）。
  • PGOutputTxnData: プラグイン固有のトランザクション状態（sent_begin_txn フラグを含む）。

失敗モードと境界条件

  • txndata が NULL の場合、または既に BEGIN が送信済みである場合は Assert により停止する。

保守担当者向けメモ

  • 本関数はトランザクションの開始を通知する重要な役割を担う。レプリケーション起点の送信ロジックと連動しているため、プロトコル仕様の変更時には logicalrep_write_begin との整合性に注意すること。
