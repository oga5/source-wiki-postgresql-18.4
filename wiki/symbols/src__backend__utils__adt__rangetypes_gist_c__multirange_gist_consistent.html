multirange_gist_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_gist.c:269

目的

マルチ範囲型（Multirange）に対するGiSTインデックスの検索整合性判定を行う。インデックス内のノード（リーフまたは内部）とクエリ条件を比較し、検索対象が含まれる可能性があるかを判定する。

シグネチャまたは形状

Datum multirange_gist_consistent(PG_FUNCTION_ARGS)
  • 引数: GISTENTRY *entry, Datum query, StrategyNumber strategy, Oid subtype, bool *recheck

入力と出力

  • **入力**:
  • entry: インデックス内のエントリ（キーを含む）。
  • query: 検索対象のクエリデータ。
  • strategy: 適用する演算子戦略番号。
  • subtype: クエリの型（範囲、マルチ範囲、または要素）。
  • **出力**:
  • recheck: 常に true を設定（マルチ範囲がギャップのない範囲の和集合として近似されるため）。
  • 戻り値: 整合性判定結果（bool）。

前提条件と不変条件

  • マルチ範囲型は、インデックス上では「ギャップのない範囲の和集合」として近似的に表現される。
  • この関数が返す結果は常に不完全であるため、呼び出し元は必ず recheck フラグを確認する必要がある。

アルゴリズムまたは振る舞い

1. recheck を true に設定する。 2. range_get_typcache を使用して型情報を取得する。 3. GIST_LEAF(entry) を判定し、リーフノードか内部ノードかを分岐する。 4. subtype に基づき、以下のいずれかの関数へ処理を委譲する：

  • マルチ範囲型 (ANYMULTIRANGEOID): range_gist_consistent_leaf_multirange または range_gist_consistent_int_multirange
  • 範囲型 (ANYRANGEOID): range_gist_consistent_leaf_range または range_gist_consistent_int_range
  • 要素型: range_gist_consistent_leaf_element または range_gist_consistent_int_element

呼び出し元と呼び出し先

  • **呼び出し元**: GiSTインデックスの検索プロセス（詳細はソースコード内に明示的な記述なし）。
  • **呼び出し先**:
  • range_get_typcache
  • range_gist_consistent_leaf_multirange / range_gist_consistent_int_multirange
  • range_gist_consistent_leaf_range / range_gist_consistent_int_range
  • range_gist_consistent_leaf_element / range_gist_consistent_int_element

関連データ構造

  • GISTENTRY: GiSTインデックスのエントリ構造体。
  • RangeType: 範囲型の内部表現。
  • TypeCacheEntry: 型情報のキャッシュ構造体。

失敗モードと境界条件

  • subtype が無効な場合、クエリ型はマルチ範囲型として扱われる。
  • 範囲の近似表現により、インデックス検索後に必ず再チェック（recheck）が発生する。

保守担当者向けメモ

  • 本関数はGiSTインデックスの検索におけるディスパッチャとして機能する。
  • 範囲型とマルチ範囲型の両方を扱うため、subtype の判定ロジックに変更を加える際は、リーフノードと内部ノードの両方のパスを考慮すること。
