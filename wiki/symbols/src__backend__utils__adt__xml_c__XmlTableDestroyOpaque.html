XmlTableDestroyOpaque

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xml.c:5077

目的

XMLTABLE 演算子の実行状態（TableFuncScanState）に関連付けられた libxml2 関連のリソースを解放し、メモリリークを防ぐ。

シグネチャまたは形状

static void XmlTableDestroyOpaque(TableFuncScanState *state)

入力と出力

  • **入力**: state (TableFuncScanState *) - 実行状態を保持する構造体。
  • **出力**: なし（void）。副作用として state->opaque が NULL に設定される。

前提条件と不変条件

  • USE_LIBXML が定義されている環境でのみ有効。
  • state->opaque には GetXmlTableBuilderPrivateData で取得可能な XmlTableBuilderData 構造体が格納されている必要がある。
  • 呼び出し後、xtCxt->magic は 0 に設定され、当該コンテキストは無効化される。

アルゴリズムまたは振る舞い

1. GetXmlTableBuilderPrivateData を介して XmlTableBuilderData (xtCxt) を取得する。 2. xmlSetStructuredErrorFunc を呼び出し、エラーコンテキストを xtCxt->xmlerrcxt に設定する。 3. xtCxt->xpathscomp 配列内の各コンパイル済みXPath式を xmlXPathFreeCompExpr で解放する。 4. xpathobj (XPathオブジェクト)、xpathcomp (XPath式)、xpathcxt (XPathコンテキスト)、doc (XMLドキュメント)、ctxt (パーサコンテキスト) を順次解放する。 5. pg_xml_done を呼び出し、PostgreSQL側のXMLエラーハンドリングを終了させる。 6. xtCxt->magic を 0 に設定し、state->opaque を NULL にクリアする。 7. USE_LIBXML が未定義の場合は NO_XML_SUPPORT() マクロを実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/xml.c 内の TableFuncScanState の破棄処理（DestroyOpaque コールバックとして登録されている）。
  • **呼び出し先**: GetXmlTableBuilderPrivateData, xmlSetStructuredErrorFunc, xmlXPathFreeCompExpr, xmlXPathFreeObject, xmlXPathFreeContext, xmlFreeDoc, xmlFreeParserCtxt, pg_xml_done。

関連データ構造

  • TableFuncScanState: 実行ノードの状態を保持する構造体。
  • XmlTableBuilderData: XMLTABLE 実行に必要な libxml2 のリソースを保持するプライベート構造体。

失敗モードと境界条件

  • xtCxt が NULL の場合、GetXmlTableBuilderPrivateData 内部でエラーが発生する可能性がある。
  • 各ポインタ（doc, ctxt 等）が NULL の場合は解放処理をスキップする安全な実装となっている。

保守担当者向けメモ

  • 本関数は XMLTABLE のライフサイクル終了時に呼び出されるため、リソース解放の順序は libxml2 の依存関係（ドキュメントやコンテキストの解放順）に注意すること。
  • pg_xml_done の呼び出しは、エラーハンドラを復元するために必須である。
