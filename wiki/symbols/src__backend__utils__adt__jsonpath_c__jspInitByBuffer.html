jspInitByBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonpath.c:989

目的

バイナリバッファ内の指定された位置から JsonPathItem 構造体を復元し、初期化します。

シグネチャまたは形状

void jspInitByBuffer(JsonPathItem *v, char *base, int32 pos)

入力と出力

  • **入力**:
  • v: 初期化対象の JsonPathItem 構造体へのポインタ。
  • base: バイナリデータが格納されているバッファの先頭アドレス。
  • pos: バッファ内における対象アイテムの開始オフセット。
  • **出力**: なし（v が指す構造体が更新されます）。

前提条件と不変条件

  • base は有効な JsonPath バイナリバッファを指している必要があります。
  • pos はバッファ内の有効なアイテム開始位置である必要があります。
  • バイナリデータは4バイトアライメントされていることを前提としています。

アルゴリズムまたは振る舞い

1. v->base に base + pos を設定します。 2. base + pos からアイテムの type を読み取ります。 3. pos を4バイト境界にアライメント調整します。 4. アライメント後の位置から nextPos（次のアイテムへのオフセット）を読み取ります。 5. v->type に基づき、以下の通り v->content を設定します。

  • **定数・単純型**: 処理なし。
  • **文字列・キー・変数**: datalen を読み取り、data ポインタをバッファ内の該当位置に設定。
  • **二項演算子**: left および right オフセットを読み取り。
  • **単項演算子**: arg オフセットを読み取り。
  • **配列インデックス**: 要素数 nelems を読み取り、要素配列を読み込み。
  • **Any**: first および last 境界値を読み取り。
  • **正規表現**: flags, expr, patternlen を読み取り、pattern ポインタを設定。

    • 6. 未知の型が指定された場合は elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonpath.c 内の各所（jspInitByBuffer 自身を再帰的に呼び出すケースを含む）および src/backend/utils/adt/jsonpath_exec.c:1863。
  • **呼び出し先**: read_byte, read_int32, read_int32_n マクロ（ソースコード上の直接的な呼び出し先）。

関連データ構造

  • JsonPathItem: パス式のノードを表現する構造体。
  • JsonPathItemType: ノードの種類を定義する列挙型。

失敗モードと境界条件

  • pos がバッファの範囲外を指している場合、メモリ破壊や不正な読み取りが発生する可能性があります。
  • 未定義の JsonPathItemType がバッファ内に存在する場合、elog(ERROR) によりトランザクションがアボートされます。

保守担当者向けメモ

  • 本関数はバイナリ表現のデコードを担うため、JsonPathItemType に新しい型を追加した際は、必ず switch 文に該当ケースを追加してください。
  • read_int32 等のマクロは base と pos を用いてバッファから値を抽出するため、バイナリレイアウトの変更時にはこれらのマクロの整合性を確認してください。
