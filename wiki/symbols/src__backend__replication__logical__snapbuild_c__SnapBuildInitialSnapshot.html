SnapBuildInitialSnapshot

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/replication/logical/snapbuild.c:439

目的

論理デコーディングにおいて、カタログ読み取り用の初期MVCCスナップショットを構築する。既存の SnapBuild 状態から、進行中のトランザクションを特定して標準的なスナップショット形式へ変換する。

シグネチャまたは形状

Snapshot SnapBuildInitialSnapshot(SnapBuild *builder)

入力と出力

  • **入力**: SnapBuild *builder（スナップショット構築状態を保持する構造体）
  • **出力**: Snapshot（構築された SNAPSHOT_MVCC 型のスナップショット）

前提条件と不変条件

  • トランザクション分離レベルが XACT_REPEATABLE_READ であること。
  • builder->building_full_snapshot が真であること。
  • builder->state が SNAPBUILD_CONSISTENT であること。
  • builder->committed.includes_all_transactions が真であること（すべてのトランザクションが監視対象であること）。
  • 既存のスナップショットが登録・アクティブでないこと。
  • MyProc->xmin が無効であること。

アルゴリズムまたは振る舞い

1. **検証**: 前提条件を満たしているか確認し、満たさない場合はエラーを投げる。 2. **スナップショット生成**: SnapBuildBuildSnapshot を呼び出し、カタログ用スナップショットを取得する。 3. **安全性の確認**: ProcArrayLock を取得し、GetOldestSafeDecodingTransactionId を用いて、現在のスナップショットの xmin が安全な範囲内にあるか検証する。 4. **ProcArrayへの反映**: MyProc->xmin にスナップショットの xmin を設定する。 5. **変換処理**: SnapBuild が保持する「コミット済みリスト」を反転させ、xmin から xmax までの範囲でコミットされていないトランザクションを「実行中」として newxip 配列に格納する。 6. **完了**: スナップショットの型を SNAPSHOT_MVCC に設定し、構築した配列を割り当てて返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/snapbuild.c (内部呼び出し)
  • src/backend/replication/walsender.c:1330
  • **呼び出し先**:
  • SnapBuildBuildSnapshot
  • GetOldestSafeDecodingTransactionId
  • LWLockAcquire / LWLockRelease
  • palloc
  • bsearch

関連データ構造

  • SnapBuild: スナップショット構築のメイン状態構造体。
  • Snapshot: PostgreSQLのMVCCスナップショット構造体。
  • MyProc: 現在のプロセスの制御構造体。

失敗モードと境界条件

  • **エラー**: 既存のスナップショットが存在する場合、一貫性状態に達していない場合、または監視対象外のトランザクションがある場合はエラーとなる。
  • **メモリ制限**: GetMaxSnapshotXidCount() を超えるトランザクションが存在する場合、initial slot snapshot too large エラーが発生する。
  • **安全性の欠如**: safeXid がスナップショットの xmin より後方にある場合、データ整合性保護のためエラーとなる。

保守担当者向けメモ

  • 本関数は、論理デコーディングの開始時に一度だけ実行される高コストな処理である。
  • snapbuild.c はコミット済みトランザクションを保持する性質上、標準的なスナップショット（実行中トランザクションを保持）への変換には bsearch を用いた反転処理が必要となる。
