appendQualifiedRelation

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/common.c:67

目的

指定されたリレーション名（テーブル名等）を検索パスに基づいて解決し、スキーマ修飾された完全な形式でバッファに追記する。

シグネチャまたは形状

void appendQualifiedRelation(PQExpBuffer buf, const char *spec, PGconn *conn, bool echo);

入力と出力

  • buf: 追記先の PQExpBuffer。
  • spec: 解析対象のテーブル指定文字列（例: table や table(col)）。
  • conn: データベース接続。
  • echo: クエリ実行時にログ出力を行うかのフラグ。
  • 出力: なし（buf に結果が追記される）。

前提条件と不変条件

  • conn は有効なデータベース接続であること。
  • 内部で RESET search_path を実行するため、呼び出し元の接続状態が一時的に変更される。
  • 最終的に ALWAYS_SECURE_SEARCH_PATH_SQL が適用され、検索パスが安全な状態にリセットされる。

アルゴリズムまたは振る舞い

1. splitTableColumnsSpec を呼び出し、入力 spec をテーブル名とカラム指定に分離する。 2. pg_class と pg_namespace を参照し、regclass を用いてテーブルのスキーマ名とリレーション名を特定するクエリを構築する。 3. RESET search_path を実行して環境を初期化する。 4. クエリを実行し、結果がちょうど1行であることを確認する（それ以外はエラーとして終了）。 5. 取得したスキーマ名とリレーション名を fmtQualifiedIdEnc で修飾し、buf に追記する。 6. 分離しておいたカラム指定を buf に追記する。 7. ALWAYS_SECURE_SEARCH_PATH_SQL を実行して検索パスを再設定する。

関連データ構造

  • PQExpBuffer: 文字列バッファ。
  • PGconn: データベース接続ハンドル。
  • PGresult: クエリ実行結果。

失敗モードと境界条件

  • spec が存在しないリレーションを指す場合や、クエリ結果が1行でない場合は pg_log_error を出力して exit(1) で終了する。
  • regclass が解決できない場合、クエリ結果が0行となりエラーとなる。

保守担当者向けメモ

  • 本関数はセキュリティ上の理由から ALWAYS_SECURE_SEARCH_PATH_SQL を使用して検索パスをリセットします。呼び出し側で現在の接続状態が変更されることを許容する必要があります。
  • splitTableColumnsSpec が引用符のネストを処理するため、複雑なテーブル指定の解析ロジックを変更する際は注意してください。
