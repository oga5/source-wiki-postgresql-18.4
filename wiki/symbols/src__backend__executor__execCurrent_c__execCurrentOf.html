execCurrentOf

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execCurrent.c:43

目的

SQLの WHERE CURRENT OF cursor_name 句を処理し、指定されたカーソルが現在指し示している行の物理的な位置（TID）を特定して更新・削除操作を可能にする。

シグネチャまたは形状

bool execCurrentOf(CurrentOfExpr *cexpr, ExprContext *econtext, Oid table_oid, ItemPointer current_tid)

入力と出力

  • **入力**:
  • cexpr: WHERE CURRENT OF 句の式情報。
  • econtext: 式評価コンテキスト。
  • table_oid: 対象テーブルのOID。
  • current_tid: 特定されたTIDを格納する出力用ポインタ。
  • **出力**:
  • bool: 成功かつ対象テーブルの行を指している場合は true、対象テーブルがアクティブでない（継承テーブル等）場合は false を返す。

前提条件と不変条件

  • 指定されたカーソルは SELECT クエリである必要がある。
  • カーソルは現在有効な行を指している必要がある（atStart または atEnd であってはならない）。
  • トランザクションを跨いで保持されているカーソルはエラーとなる。

アルゴリズムまたは振る舞い

1. **カーソル解決**: cexpr からカーソル名を取得し、GetPortalByName で Portal を特定する。 2. **戦略の分岐**:

  • FOR UPDATE/SHARE を使用している場合: es_rowmarks を走査し、対象テーブルに対応する ExecRowMark から curCtid を抽出する。
  • それ以外の場合: search_plan_tree を用いてプランツリーから該当する ScanState を探索し、物理タプルからTIDを取得する。

    • 3. **TID抽出**:

  • IndexOnlyScanState の場合は xs_heaptid を使用。
  • それ以外はスキャンノードの現在のタプルからTIDを取得する。

    • 4. **継承テーブルの考慮**: 対象テーブルがアクティブでない場合は false を返し、呼び出し元が次のテーブルを試行できるようにする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeTidscan.c
  • **呼び出し先**:
  • fetch_cursor_param_value
  • GetPortalByName
  • search_plan_tree
  • RowMarkRequiresRowShareLock
  • slot_getsysattr (実装の続きで参照)

関連データ構造

  • CurrentOfExpr: WHERE CURRENT OF 句の構文解析結果。
  • Portal: カーソルの状態を保持する構造体。
  • ExecRowMark: FOR UPDATE/SHARE 句の行ロック情報。
  • ScanState: プランツリーのスキャン状態。

失敗モードと境界条件

  • **エラー**:
  • カーソルが存在しない場合 (ERRCODE_UNDEFINED_CURSOR)。
  • カーソルが SELECT ではない、または保持されたカーソルである場合 (ERRCODE_INVALID_CURSOR_STATE)。
  • カーソルが特定の行を指していない場合 (ERRCODE_INVALID_CURSOR_STATE)。
  • FOR UPDATE/SHARE 参照が重複または存在しない場合。
  • スキャンノードが見つからない場合。

保守担当者向けメモ

  • 継承テーブルの扱いにおいて、特定のテーブルが非アクティブな場合はエラーではなく false を返す設計となっている。
  • search_plan_tree は複雑なプラン（MergeAppend 等）を透過的に探索できない制限がある。
  • USE_ASSERT_CHECKING が有効な場合、取得したTIDとテーブルOIDの厳密な整合性チェックが行われる。
