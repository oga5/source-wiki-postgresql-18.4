PartitionTupleRouting

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/execPartition.c:91

目的

パーティションテーブルに対するデータ操作（INSERT/UPDATE/MERGE）において、タプルを適切なパーティションへルーティングするために必要な情報を一元管理する構造体です。

シグネチャまたは形状

struct PartitionTupleRouting
{
    Relation    partition_root;
    PartitionDispatch *partition_dispatch_info;
    ResultRelInfo **nonleaf_partitions;
    int         num_dispatch;
    int         max_dispatch;
    ResultRelInfo **partitions;
    bool       *is_borrowed_rel;
    int         num_partitions;
    int         max_partitions;
    MemoryContext memcxt;
};

入力と出力

本構造体はデータそのものではなく、ルーティング処理の実行環境を保持します。

  • **入力**: ExecSetupPartitionTupleRouting 等の初期化関数により、ルートテーブル情報やメモリコンテキストが格納されます。
  • **出力**: ルーティング過程で特定された ResultRelInfo（パーティションごとの実行情報）が partitions 配列等に動的に蓄積されます。

前提条件と不変条件

  • PartitionDispatch 情報は必ず本構造体にカプセル化されます。
  • ResultRelInfo はルーティング時に必要となった時点でオンデマンドに初期化されます。
  • 保持するメモリコンテキスト (memcxt) は、ルーティングに関連する動的割り当ての生存期間を管理します。

アルゴリズムまたは振る舞い

1. **初期化**: ExecSetupPartitionTupleRouting により、ルーティングに必要な階層構造が構築されます。 2. **ルーティング**: ExecFindPartition が本構造体を参照し、タプルのパーティションキーを評価して対象パーティションを特定します。 3. **管理**: 未初期化のパーティションが特定された場合、ExecInitPartitionInfo 等を用いて ResultRelInfo を生成し、本構造体の配列に登録します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfrom.c（COPYコマンド実行時）や nodeModifyTable.c 等の実行ノードから参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありませんが、ExecFindPartition などのルーティング関数に引数として渡されます。

関連データ構造

  • PartitionDispatch: パーティション階層の各レベルにおけるルーティング情報を保持。
  • ResultRelInfo: 各パーティションの実行時情報（リレーション記述子やトリガー情報など）。

失敗モードと境界条件

  • パーティション階層が深い場合、partition_dispatch_info を介した再帰的な探索が発生します。
  • max_partitions を超えるパーティションが特定された場合、配列の拡張が必要です（実装上の境界条件）。

保守担当者向けメモ

  • ResultRelInfo のオンデマンド初期化は、パーティション数が多いテーブルでのメモリ消費を抑えるための重要な最適化です。
  • 連続するタプルが同一パーティションに属する場合のキャッシュ機構が get_partition_for_tuple 内で利用されるため、本構造体を通じたアクセス効率が全体の性能に直結します。
