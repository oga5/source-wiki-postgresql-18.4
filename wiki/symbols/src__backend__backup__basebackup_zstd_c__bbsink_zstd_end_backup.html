bbsink_zstd_end_backup

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_zstd.c:281

目的

Zstandard圧縮シンクの終了処理を行い、圧縮コンテキストの解放と後続のシンクへの終了通知を行う。

シグネチャまたは形状

static void bbsink_zstd_end_backup(bbsink *sink, XLogRecPtr endptr, TimeLineID endtli)

入力と出力

  • sink: 処理対象の bbsink 構造体（実際には bbsink_zstd 型としてキャストされる）。
  • endptr: バックアップ終了時の WAL 位置。
  • endtli: バックアップ終了時のタイムライン ID。
  • 出力: なし（副作用としてコンテキストの解放と後続シンクへの転送を行う）。

前提条件と不変条件

  • sink は bbsink_zstd 型のインスタンスであること。
  • mysink->cctx が存在する場合、それは有効な ZSTD_CCtx であること。

アルゴリズムまたは振る舞い

1. sink を bbsink_zstd 型にキャストする。 2. mysink->cctx が存在する場合、ZSTD_freeCCtx を呼び出して圧縮コンテキストを解放し、ポインタを NULL に設定する。 3. bbsink_forward_end_backup を呼び出し、endptr と endtli を引数として後続のシンクへ終了処理を転送する。

呼び出し元と呼び出し先

  • 呼び出し元: bbsink_zstd_ops 構造体（basebackup_zstd.c:52）を通じて、バックアップパイプラインの終了時に呼び出される。
  • 呼び出し先: ZSTD_freeCCtx（libzstd）、bbsink_forward_end_backup（basebackup_sink インターフェース）。

関連データ構造

  • bbsink_zstd: bbsink を継承したZstd圧縮用状態構造体。
  • ZSTD_CCtx: libzstd の圧縮コンテキスト。

失敗モードと境界条件

  • mysink->cctx が既に NULL の場合、解放処理はスキップされる。

保守担当者向けメモ

  • 本関数は bbsink インターフェースの end_backup メソッドとして登録されている。
  • 圧縮コンテキストの解放後に bbsink_forward_end_backup を呼び出す順序が重要である。
