QTN2QTState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsquery_util.c:311

目的

QTNode ツリー構造からフラットな TSQuery 形式へ変換する際、変換処理の状態を保持するための作業用構造体です。

シグネチャまたは形状

typedef struct {
    QueryItem *curitem;    /* 現在処理中の QueryItem */
    char      *operand;    /* オペランド格納領域の先頭ポインタ */
    char      *curoperand; /* 現在書き込み中のオペランド位置 */
} QTN2QTState;

入力と出力

本構造体自体はデータ保持用ですが、QTN2QT 関数内で初期化され、fillQT 関数へ渡されることで、ツリーの走査結果をフラットな配列およびバッファへ書き出すためのコンテキストとして機能します。

前提条件と不変条件

  • fillQT 関数に渡される前に、curitem および curoperand が適切なメモリ領域を指している必要があります。
  • curoperand は、operand で指定されたバッファの範囲内で更新される必要があります。

アルゴリズムまたは振る舞い

QTN2QT 関数がツリーを再帰的に走査する際、この構造体を通じて現在の書き込み位置（curitem）とオペランドの格納位置（curoperand）を追跡します。fillQT はこの状態を参照しながら、ツリーノードの内容を順次シリアライズします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsquery_util.c 内の QTN2QT 関数でインスタンス化されます。
  • **呼び出し先**: src/backend/utils/adt/tsquery_util.c 内の fillQT 関数が、この構造体を引数として受け取り、内部のポインタを更新しながら書き込みを行います。

関連データ構造

  • QTNode: 変換元のツリー構造。
  • QueryItem: 変換先のフラットな配列要素。

失敗モードと境界条件

  • curoperand が割り当てられたバッファサイズを超えて書き込みを行った場合、メモリ破壊が発生する可能性があります（QTN2QT 側での事前のサイズ計算 cntsize が前提となります）。

保守担当者向けメモ

  • 本構造体は QTN2QT の再帰処理における状態管理に特化しています。変更を加える際は、fillQT におけるポインタのインクリメントロジックと整合性が取れていることを確認してください。
