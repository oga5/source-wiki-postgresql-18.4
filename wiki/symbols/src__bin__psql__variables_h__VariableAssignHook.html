VariableAssignHook

定義

  • 種別: typedef
  • 定義箇所: src/bin/psql/variables.h:31

目的

psqlの変数に対して値が割り当てられる際に、その値の妥当性を検証または処理するためのコールバック関数型を定義する。

シグネチャまたは形状

typedef bool (*VariableAssignHook) (const char *newval);

入力と出力

  • 入力: newval (割り当てようとしている新しい値を示す文字列へのポインタ)
  • 出力: bool (割り当てが許可される場合は true、拒否される場合は false)

前提条件と不変条件

  • newval は NULL である可能性がある（実装依存）。
  • 戻り値が false の場合、変数の値は更新されないことが期待される。

アルゴリズムまたは振る舞い

この関数ポインタが設定されている場合、変数の更新処理中に呼び出され、新しい値がその変数にとって適切かどうかを判定する。判定結果に基づいて、呼び出し側は割り当ての成否を決定する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/variables.c 内の変数割り当てロジック（直接的な証拠として src/bin/psql/variables.c:386 に言及あり）。
  • 呼び出し先: この型を実装する具体的な関数（ソース抜粋からは特定不可）。

関連データ構造

  • src/bin/psql/variables.h 内の変数管理構造体において、assign_hook フィールドとして保持される（src/bin/psql/variables.h:67）。

失敗モードと境界条件

  • 割り当てが拒否された場合、呼び出し側はエラーメッセージを表示するか、割り当てを中断する挙動をとる必要がある。
  • 渡された文字列の解析に失敗した場合、このフックは false を返すことで安全に割り当てを阻止できる。

保守担当者向けメモ

  • 本シンボルは関数ポインタ型であるため、実装関数を追加する際は、必ず bool を返し、副作用として変数の状態を破壊しないように注意すること。
  • src/bin/psql/variables.c の実装詳細を確認し、フックが呼び出される正確なタイミングとエラーハンドリングの仕様を把握すること。
