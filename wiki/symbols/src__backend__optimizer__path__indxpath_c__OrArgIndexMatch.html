OrArgIndexMatch

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/path/indxpath.c:1180

目的

OR条件に含まれる各引数（句）が、どのインデックスおよびどの列に適合するかを追跡し、グループ化するための情報を保持する。

シグネチャまたは形状

typedef struct {
    int     indexnum;      /* マッチしたインデックスのインデックス、不適合時は -1 */
    int     colnum;        /* マッチした列のインデックス、不適合時は -1 */
    Oid     opno;          /* OpExprの演算子OID、OpExprでない場合は InvalidOid */
    Oid     inputcollid;   /* OpClauseの入力照合順序OID */
    int     argindex;      /* 引数リスト内での当該句のインデックス */
    int     groupindex;    /* 類似句グループの先頭句のargindex */
} OrArgIndexMatch;

入力と出力

本構造体は、クエリ内のOR条件を解析する過程で生成される配列の要素として使用される。入力となるOR条件の各引数に対し、適合するインデックス情報が各フィールドに格納される。

前提条件と不変条件

  • indexnum または colnum が -1 の場合、その句はインデックスに適合しないことを示す。
  • groupindex は、同一のインデックスおよび列に適合する句のグループを識別するために使用される。

アルゴリズムまたは振る舞い

  • OR条件の各引数に対して本構造体のインスタンスが生成され、配列として保持される。
  • qsort を用いて、or_arg_index_match_cmp または or_arg_index_match_cmp_group 関数によりソートされ、インデックス適合性やグループ単位での整理が行われる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/path/indxpath.c 内のOR条件処理ロジック。
  • 呼び出し先: qsort（標準ライブラリ）および比較関数 or_arg_index_match_cmp / or_arg_index_match_cmp_group。

関連データ構造

  • IndexClauseSet: インデックスの列に対する条件適合情報を保持する構造体。

失敗モードと境界条件

  • インデックスに適合しない句については、indexnum および colnum が -1 に設定される。
  • palloc によるメモリ確保が失敗した場合、システム全体としてメモリ不足エラーが発生する可能性がある。

保守担当者向けメモ

  • 本構造体はOR条件の最適化プロセスにおいて、どのインデックスが利用可能かを判定するための重要な中間データである。
  • 比較関数（or_arg_index_match_cmp 等）を変更する場合は、ソート順序が後続のパス生成ロジックに与える影響を考慮すること。
