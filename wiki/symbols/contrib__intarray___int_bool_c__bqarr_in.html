bqarr_in

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_bool.c:514

目的

ユーザーが入力したクエリ文字列を解析し、内部的な QUERYTYPE 構造体（ポーランド記法形式の ITEM 配列）へ変換する。

シグネチャまたは形状

Datum bqarr_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_POINTER(0) を介して取得されるクエリ文字列（char *）。
  • **出力**: 構築された QUERYTYPE 構造体へのポインタを Datum として返す。解析失敗時は NULL を返す。

前提条件と不変条件

  • makepol による構文解析が成功すること。
  • クエリの項目数が QUERYTYPEMAXITEMS を超えないこと。
  • findoprnd による演算子の左オペランド解決が完了し、pos が -1 になること。

アルゴリズムまたは振る舞い

1. WORKSTATE を初期化し、makepol を呼び出して文字列を逆ポーランド記法のリスト構造に変換する。 2. 項目数が 0 の場合はエラーを報告する。 3. 項目数が上限を超えている場合はエラーを報告する。 4. QUERYTYPE 構造体に必要なメモリを確保し、リスト構造から ITEM 配列へデータをコピーする。 5. findoprnd を呼び出し、各演算子ノードの left フィールド（左オペランドへのオフセット）を計算する。 6. 構築された QUERYTYPE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の型入力関数として登録されている（PG_FUNCTION_INFO_V1(bqarr_in)）。
  • **呼び出し先**: makepol, findoprnd, palloc, pfree, ereturn 等。

関連データ構造

  • QUERYTYPE: クエリを保持するメイン構造体。
  • ITEM: クエリツリーの各ノード（値または演算子）。
  • WORKSTATE: 解析中の状態を保持する一時構造体。
  • NODE: 解析中に使用される一時的なリストノード。

失敗モードと境界条件

  • **空のクエリ**: state.num が 0 の場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー終了。
  • **項目数超過**: state.num が QUERYTYPEMAXITEMS を超える場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラー終了。
  • **解析エラー**: makepol または findoprnd が失敗した場合、NULL を返す。

保守担当者向けメモ

  • 本関数は _int_bool.c 内のクエリ処理の入り口です。
  • findoprnd が pos == -1 を満たさない場合、ツリー構築の論理的な不整合を意味します。
  • デバッグ用に BS_DEBUG マクロが定義されており、有効時にはポーランド記法の構造がログ出力されます。
