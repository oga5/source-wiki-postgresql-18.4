rot13_passphrase

定義

  • 種別: function
  • 定義箇所: src/test/modules/ldap_password_func/ldap_password_func.c:37

目的

LDAP認証プロセスにおいて、入力されたパスワード文字列に対してROT13変換を適用し、変換後のパスワードを生成する。

シグネチャまたは形状

static char *rot13_passphrase(char *pw)

入力と出力

  • **入力**: pw (変換対象の文字列ポインタ)
  • **出力**: 変換後の文字列を格納した、palloc によってメモリ確保された新しい文字列ポインタ。

前提条件と不変条件

  • 入力文字列 pw はヌル終端されている必要がある。
  • palloc を使用するため、PostgreSQLのメモリコンテキスト内で呼び出される必要がある。
  • アルファベット（A-Z, a-z）のみが変換対象であり、数字や記号は変更されない。

アルゴリズムまたは振る舞い

1. 入力文字列の長さに1を加えたメモリ領域を palloc で確保する。 2. strlcpy を使用して入力文字列を確保した領域にコピーする。 3. 文字列の各文字を走査し、以下の変換を行う：

  • 'a'-'m' または 'A'-'M' の場合、13文字分加算する。
  • 'n'-'z' または 'N'-'Z' の場合、13文字分減算する。

    • 4. 変換後の文字列ポインタを返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: _PG_init 関数内にて ldap_password_hook に関数ポインタとして代入される。コアシステム（src/backend/libpq/auth.c）が認証時にこのフックを介して呼び出す。
  • **呼び出し先**: palloc（メモリ確保）、strlen、strlcpy（文字列操作）。

関連データ構造

  • ldap_password_hook: 本関数が登録されるグローバルな関数ポインタ。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる可能性がある。
  • アルファベット以外の文字（数字、記号、マルチバイト文字等）が含まれる場合、それらは変換されずそのまま維持される。

保守担当者向けメモ

  • 本関数はテスト目的のモジュールであり、実運用環境での使用は想定されていない。
  • src/test/modules/ssl_passphrase_callback/ssl_passphrase_func.c に同名の関数が存在するが、シグネチャと目的が異なるため混同しないこと。
