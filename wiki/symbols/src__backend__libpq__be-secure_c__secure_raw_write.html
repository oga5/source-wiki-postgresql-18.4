secure_raw_write

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/libpq/be-secure.c:376

目的

暗号化層を介さず、基盤となるソケットに対して直接データを書き込むための低レベルなI/Oインターフェースを提供します。

シグネチャまたは形状

ssize_t secure_raw_write(Port *port, const void *ptr, size_t len)

入力と出力

  • **入力**:
  • port: 通信状態を保持する Port 構造体へのポインタ。
  • ptr: 送信するデータが格納されたバッファへのポインタ。
  • len: 送信するデータのバイト数。
  • **出力**:
  • 送信に成功した場合は送信バイト数を返し、失敗した場合は -1 を返します。

前提条件と不変条件

  • port 引数は有効な Port 構造体を指している必要があります。
  • Windows環境では、実行中にグローバル変数 pgwin32_noblock が一時的に true に設定され、操作終了後に false へ戻されます。

アルゴリズムまたは振る舞い

1. Windows環境の場合、pgwin32_noblock を true に設定します。 2. port->sock を使用して、標準の send() システムコールを呼び出し、データを送信します。 3. Windows環境の場合、pgwin32_noblock を false に戻します。 4. send() の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-secure-gssapi.c (GSSAPI送信処理)
  • src/backend/libpq/be-secure-openssl.c (OpenSSL BIOコールバック)
  • src/backend/libpq/be-secure.c (内部の書き込み処理)
  • **呼び出し先**:
  • send() (システムコール)

関連データ構造

  • Port: 通信ソケット (sock) を保持する構造体。

失敗モードと境界条件

  • send() システムコールが失敗した場合、-1 が返されます。エラーの詳細は errno に依存します。
  • len が 0 の場合の挙動は、基盤となる send() システムコールの仕様に従います。

保守担当者向けメモ

  • 本関数は暗号化を一切行わない「生の」書き込み関数です。TLSやGSSAPIによる暗号化が必要な場合は、secure_write を経由してください。
  • Windowsにおける pgwin32_noblock の操作は、非ブロッキングI/Oを正しく扱うためのプラットフォーム固有の回避策です。
