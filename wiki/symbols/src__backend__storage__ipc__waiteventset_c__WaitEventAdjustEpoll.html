WaitEventAdjustEpoll

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/waiteventset.c:737

目的

WaitEventSet 内のイベント設定を、OS の epoll インターフェースに対して反映（追加、変更、削除）させる。

シグネチャまたは形状

static void WaitEventAdjustEpoll(WaitEventSet *set, WaitEvent *event, int action)

入力と出力

  • **入力**:
  • set: 監視対象の epoll_fd を保持する WaitEventSet 構造体。
  • event: 監視対象のイベント情報（FD、イベント種別など）。
  • action: epoll_ctl に渡す操作（EPOLL_CTL_ADD, EPOLL_CTL_MOD, EPOLL_CTL_DEL）。
  • **出力**: なし（成功時は副作用としてカーネル内の epoll 状態が更新される）。

前提条件と不変条件

  • set->epoll_fd が有効な epoll インスタンスであること。
  • event が WL_LATCH_SET または WL_POSTMASTER_DEATH の場合、あるいは有効なファイル記述子（fd）を保持していること。
  • epoll_ctl が失敗した場合は ereport(ERROR, ...) によりプロセスが中断される。

アルゴリズムまたは振る舞い

1. epoll_event 構造体を初期化し、data.ptr に event ポインタを格納する。 2. 常に EPOLLERR および EPOLLHUP を監視対象に含める。 3. event->events の種別に応じて、以下の通り epoll イベントフラグをマッピングする：

  • WL_LATCH_SET または WL_POSTMASTER_DEATH: EPOLLIN を追加。
  • ソケット関連: WL_SOCKET_READABLE → EPOLLIN、WL_SOCKET_WRITEABLE → EPOLLOUT、WL_SOCKET_CLOSED → EPOLLRDHUP をそれぞれ追加。

    • 4. epoll_ctl を呼び出し、指定された action を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/waiteventset.c 内の AddWaitEventToSet (イベント追加時)
  • src/backend/storage/ipc/waiteventset.c 内の ModifyWaitEvent (イベント変更時)
  • **呼び出し先**:
  • epoll_ctl (システムコール)

関連データ構造

  • WaitEventSet: 監視セット全体を管理する構造体。
  • WaitEvent: 個別の監視対象イベントを定義する構造体。
  • struct epoll_event: epoll システムコールで使用される標準構造体。

失敗モードと境界条件

  • epoll_ctl がエラーを返した場合、errcode_for_socket_access() を用いてエラー報告を行い、処理を終了する。

保守担当者向けメモ

  • EPOLL_CTL_DEL を実行する場合でも、過去の epoll のバグ対応およびコードの簡素化のため、epoll_ev を引数として渡す実装となっている。
  • 本関数は WAIT_USE_EPOLL が有効な環境でのみ使用される。
