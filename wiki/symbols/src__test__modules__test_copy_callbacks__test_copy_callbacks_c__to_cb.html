to_cb

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_copy_callbacks/test_copy_callbacks.c:24

目的

PostgreSQLの COPY TO コマンド実行時に、データ出力先として登録されるカスタムコールバック関数です。受信したデータを NOTICE レベルのログとして出力し、テスト目的でデータフローを可視化します。

シグネチャまたは形状

static void to_cb(void *data, int len)

入力と出力

  • **入力**:
  • data: 出力対象のデータバッファへのポインタ。
  • len: data が指すデータの長さ（バイト単位）。
  • **出力**: なし（void）。副作用として ereport によるログ出力を行います。

前提条件と不変条件

  • COPY TO の実行コンテキストにおいて、CopyToState にコールバックとして登録されている必要があります。
  • data は char * としてキャスト可能であると想定されています。

アルゴリズムまたは振る舞い

1. 受け取った data と len を引数として ereport(NOTICE, ...) を呼び出します。 2. ログメッセージには、受信したデータの内容（文字列として解釈）とデータ長が含まれます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_copy_callbacks/test_copy_callbacks.c 内の test_copy_to_callback 関数から、BeginCopyTo の引数として参照されています。
  • **呼び出し先**: PostgreSQLの ereport マクロを呼び出します。

関連データ構造

  • CopyToState: COPY TO コマンドの実行状態を保持する構造体。本関数はこの構造体に登録されます。

失敗モードと境界条件

  • data が指すメモリ領域がヌル終端されていない場合、errmsg 内での文字列出力時に予期せぬ動作をする可能性があります。
  • COPY の出力形式（CSV, Text, Binary）によっては、data にバイナリデータが含まれる場合があり、文字列としての表示には制限があります。

保守担当者向けメモ

  • 本関数はテスト用モジュールの一部です。実際の運用環境でのカスタム出力先実装の雛形として利用可能です。
  • data の解釈については、呼び出し元の COPY 設定に依存するため、変更を加える際は commands/copy.h の仕様を確認してください。
