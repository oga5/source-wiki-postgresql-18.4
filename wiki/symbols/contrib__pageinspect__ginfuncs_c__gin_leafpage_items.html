gin_leafpage_items

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/ginfuncs.c:178

目的

GIN インデックスのリーフページ内に格納された圧縮済みポストリストをデコードし、含まれるアイテムポインタ（TID）を列挙するセット返却関数（SRF）です。

シグネチャまたは形状

Datum gin_leafpage_items(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: bytea 型の raw ページデータ（PG_GETARG_BYTEA_P(0)）。
  • **出力**: 以下のカラムを持つ行を順次返します。
  • first_tid (ItemPointer): ポストリストセグメントの最初の TID。
  • nbytes (int2): セグメントのバイト数。
  • tids (tid[]): デコードされた TID の配列。

前提条件と不変条件

  • 呼び出し元はスーパーユーザー権限を持つ必要があります。
  • 入力ページは GIN データリーフページであり、かつ GIN_DATA | GIN_LEAF | GIN_COMPRESSED フラグが設定されている必要があります。
  • ページが空（PageIsNew）の場合は NULL を返します。

アルゴリズムまたは振る舞い

1. **初期化**: SRF_IS_FIRSTCALL() にて、入力ページを検証し、gin_leafpage_items_state 構造体を初期化します。 2. **走査**: GinDataLeafPageGetPostingList を用いてポストリストの先頭セグメントを取得し、lastseg に到達するまで反復します。 3. **デコード**: 各セグメントに対して ginPostingListDecode を実行し、圧縮された TID を展開します。 4. **構築**: 展開された TID を construct_array_builtin で配列化し、heap_form_tuple で行形式に変換して返却します。 5. **更新**: GinNextPostingListSegment を使用して次のセグメントへ移動します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: get_page_from_raw, ginPostingListDecode, GinNextPostingListSegment, heap_form_tuple 等。

関連データ構造

  • gin_leafpage_items_state: 呼び出し間で走査状態（現在のセグメント、終了位置、タプル記述子）を保持するコンテキスト。
  • GinPageOpaqueData: GIN ページのフラグ確認に使用。
  • GinPostingList: 圧縮されたポストリストのデータ構造。

失敗モードと境界条件

  • **権限エラー**: スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー終了します。
  • **形式エラー**: ページが GIN リーフページでない、またはフラグが不一致の場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー終了します。
  • **型エラー**: 戻り値の型が複合型でない場合、elog でエラー終了します。

保守担当者向けメモ

  • 本関数は FuncCallContext を使用した SRF 実装です。メモリリークを防ぐため、fctx->multi_call_memory_ctx を適切に使用しています。
  • ginPostingListDecode で割り当てられたメモリは、各反復の終了時に pfree される必要があります。
