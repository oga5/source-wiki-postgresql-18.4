parseNodeString

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/readfuncs.c:572

目的

シリアライズされた文字列形式のノードツリーを、メモリ上の Node 構造体へ復元するメインエントリーポイントです。

シグネチャまたは形状

Node *parseNodeString(void)

入力と出力

  • **入力**: なし（内部的に pg_strtok を介してグローバルな文字列バッファからトークンを読み取ります）。
  • **出力**: 復元された Node 構造体へのポインタ。

前提条件と不変条件

  • pg_strtok が有効な文字列バッファを指している必要があります。
  • スタックオーバーフローを防ぐため、内部で check_stack_depth() が呼び出されます。

アルゴリズムまたは振る舞い

1. READ_TEMP_LOCALS() マクロで必要なローカル変数を初期化します。 2. check_stack_depth() を実行し、再帰的なノード読み込みによるスタック溢れを防止します。 3. pg_strtok を使用して、シリアライズ文字列からノードの型名（トークン）を取得します。 4. readfuncs.switch.c をインクルードし、取得したトークン名に基づいて適切な _read<NodeName> 関数へディスパッチします。 5. 対応するノード型が見つからない場合、elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/read.c 内の nodeRead 等から呼び出されます。
  • **呼び出し先**: readfuncs.switch.c 内の各 _read<NodeName> 関数（動的なディスパッチ先）。

関連データ構造

  • Node: 復元対象となる PostgreSQL のノード構造体。
  • readfuncs.switch.c: ノード型と読み込み関数を紐付けるスイッチ文を含むソースファイル。

失敗モードと境界条件

  • **スタックオーバーフロー**: 複雑すぎる式が入力された場合、check_stack_depth() によりエラーとなります。
  • **フォーマットエラー**: 認識できないノード型名や不正な形式の文字列が渡された場合、elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数は readfuncs.switch.c に強く依存しています。新しいノード型を追加した際は、対応する _read 関数を実装し、スイッチ文に登録する必要があります。
  • pg_strtok を使用しているため、呼び出し元で文字列バッファの状態が正しく管理されていることを前提としています。
