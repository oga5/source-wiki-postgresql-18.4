strtitle_builtin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_builtin.c:88

目的

PostgreSQLの組み込みロケールプロバイダにおいて、Unicode規格に基づいた文字列のタイトルケース変換を行う。

シグネチャまたは形状

size_t strtitle_builtin(char *dest, size_t destsize, const char *src, ssize_t srclen, pg_locale_t locale)

入力と出力

  • **入力**:
  • dest: 変換結果を格納するバッファ。
  • destsize: dest バッファのサイズ。
  • src: 変換対象のソース文字列。
  • srclen: ソース文字列の長さ（負の場合は strlen(src) が使用される）。
  • locale: ロケール情報を含む pg_locale_t 構造体。
  • **出力**:
  • 変換後の文字列のバイト長を size_t で返す。

前提条件と不変条件

  • locale 引数は有効な pg_locale_t 構造体を指している必要がある。
  • locale->info.builtin.casemap_full が、変換に使用するUnicodeケースマッピングの挙動を決定する。

アルゴリズムまたは振る舞い

1. WordBoundaryState 構造体を初期化し、文字列の走査状態（現在の位置、単語境界の判定フラグ等）を保持する。 2. locale->info.builtin.casemap_full の値に基づき、POSIX互換モードか否かを wbstate.posix に設定する。 3. unicode_strtitle 関数を呼び出し、initcap_wbnext をイテレータとして渡すことで、単語境界を検出しながらタイトルケース変換を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/pg_locale.c 内のロケール抽象化レイヤーから呼び出される。
  • **呼び出し先**:
  • unicode_strtitle (Unicode変換の実行)
  • initcap_wbnext (単語境界判定のコールバック)

関連データ構造

  • WordBoundaryState: 文字列走査中の状態を保持する内部構造体。
  • pg_locale_t: ロケール固有の設定情報（casemap_full 等）を保持する構造体。

失敗モードと境界条件

  • srclen が負の場合、strlen を使用するため、src がヌル終端されている必要がある。
  • destsize が不足している場合の挙動は unicode_strtitle の実装に依存する。

保守担当者向けメモ

  • 本関数は initcap_wbnext を介して単語境界を判定する。複雑な言語ルールには対応しておらず、pg_u_isalnum の変化に基づくシンプルな判定ロジックである点に注意すること。
