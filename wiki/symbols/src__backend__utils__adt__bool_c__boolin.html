boolin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/bool.c:126

目的

SQLの boolean 型の入力関数として、文字列形式のデータを内部的な bool 型へ変換します。

シグネチャまたは形状

Datum boolin(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して文字列（const char *）を受け取ります。
  • **出力**: 成功時は bool 型の値を Datum 形式で返します。失敗時はエラーを発生させます。

前提条件と不変条件

  • 入力文字列は、先頭および末尾の空白が無視されます。
  • 変換には parse_bool_with_len が使用され、この関数が許容する形式（true, false, yes, no, on, off, 1, 0 等）である必要があります。

アルゴリズムまたは振る舞い

1. 入力文字列の先頭および末尾の空白をスキップし、有効な文字列範囲を特定します。 2. parse_bool_with_len を呼び出し、文字列を bool 値に変換します。 3. 変換に成功した場合、PG_RETURN_BOOL を使用して結果を返します。 4. 変換に失敗した場合、ERRCODE_INVALID_TEXT_REPRESENTATION エラーを発生させ、処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの型システム（SQLの boolean 型への入力処理時）。
  • **呼び出し先**: parse_bool_with_len（文字列解析の実行）。

関連データ構造

  • Datum: PostgreSQLの汎用データ型表現。
  • PG_FUNCTION_ARGS: 関数呼び出し規約のための引数構造体。

失敗モードと境界条件

  • 入力文字列がブール値として解釈できない場合、invalid input syntax for type boolean というエラーメッセージと共に ERRCODE_INVALID_TEXT_REPRESENTATION が発生します。

保守担当者向けメモ

  • 本関数は parse_bool_with_len に依存しており、解析ロジックの変更はそちらに影響します。
  • 空白処理は isspace を使用して行われており、ロケール設定に依存する可能性があります。
