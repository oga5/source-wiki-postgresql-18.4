pg_atomic_uint32

定義

  • 種別: struct
  • 定義箇所: src/include/port/atomics/arch-ppc.h:29

目的

PowerPC アーキテクチャにおいて、アトミック操作の対象となる 32 ビット符号なし整数のメモリ領域を定義します。

シグネチャまたは形状

typedef struct pg_atomic_uint32
{
	volatile uint32 value;
} pg_atomic_uint32;

入力と出力

  • **入力**: uint32 型の値を保持する構造体。
  • **出力**: なし（データ構造の定義）。

前提条件と不変条件

  • value メンバは volatile 修飾されており、コンパイラによる最適化でメモリ読み込みが省略されないことが保証されます。
  • 本構造体に対する操作は、src/include/port/atomics/arch-ppc.h で定義される lwarx/stwcx. 命令を用いたアトミックプリミティブを介して行われることが前提です。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のためのコンテナです。この構造体に対するアトミックな更新は、以下の手順で行われます： 1. lwarx (Load Word and Reserve) 命令により、メモリをロードし予約をセット。 2. 比較または演算を実行。 3. stwcx. (Store Word Conditional) 命令により、予約が維持されていれば書き込みを実行。 4. 予約が解除されていた場合はループして再試行。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/commands/vacuum.c
  • src/backend/commands/vacuumparallel.c
  • src/backend/nodes/tidbitmap.c
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/pgarch.c
  • src/backend/storage/buffer/freelist.c
  • src/backend/storage/ipc/procsignal.c
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数呼び出しは発生しません。

関連データ構造

  • pg_atomic_uint64: 64 ビット版のアトミック整数構造体。
  • pg_atomic_flag: フラグ用のアトミック構造体。

失敗モードと境界条件

  • 本構造体は 32 ビット幅であるため、値のオーバーフローは型定義の範囲内に依存します。
  • メモリ上の配置がアライメントされていない場合、CPU アーキテクチャの制約によりアトミック性が保証されない可能性があります。

保守担当者向けメモ

  • 本構造体は src/include/port/atomics/generic.h を通じて上位層から利用されます。
  • 構造体のメンバを変更する際は、arch-ppc.h 内のインラインアセンブラ（lwarx/stwcx.）がこの構造体のメモリレイアウトに依存している点に注意してください。
