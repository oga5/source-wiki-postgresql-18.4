sepgsql_context_info_t

定義

  • **種別**: struct
  • **定義箇所**: contrib/sepgsql/hooks.c:47

目的

DDL コマンド実行時に、コマンド固有のメタデータやコンテキスト情報を保持するための構造体です。

シグネチャまたは形状

typedef struct
{
	NodeTag		cmdtype;
	const char *createdb_dtemplate;
} sepgsql_context_info_t;

入力と出力

  • **入力**: cmdtype（コマンドの種類を示すタグ）、createdb_dtemplate（CREATE DATABASE 実行時のテンプレートデータベース名）。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • createdb_dtemplate は、CREATE DATABASE コマンド実行時以外（デフォルトを含む）は NULL である必要があります。

アルゴリズムまたは振る舞い

  • コマンド実行中のコンテキストを追跡し、必要に応じて sepgsql_context_info 静的変数に格納されます。
  • エラー発生時や処理終了時に、PG_TRY/PG_FINALLY ブロックを通じてコンテキストの復元が行われることを想定した設計となっています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/hooks.c 内の静的変数 sepgsql_context_info として定義。
  • contrib/sepgsql/hooks.c:326 にて、saved_context_info として現在のコンテキストを退避する際に参照されています。
  • **呼び出し先**: 情報なし。

関連データ構造

  • NodeTag: PostgreSQL のノード型識別子。

失敗モードと境界条件

  • createdb_dtemplate が適切に管理されない場合、後続の権限チェックにおいて誤ったコンテキストが参照される可能性があります。

保守担当者向けメモ

  • 本構造体は hooks.c 内でグローバルなコンテキスト状態を管理するために使用されます。他のモジュールから参照する場合は、hooks.c での管理方針に従ってください。
