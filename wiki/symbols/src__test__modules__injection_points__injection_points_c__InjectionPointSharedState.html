InjectionPointSharedState

定義

  • **種別**: struct
  • **定義箇所**: src/test/modules/injection_points/injection_points.c:78

目的

PostgreSQLのテスト用インジェクションポイントにおいて、プロセス間で共有される状態（待機カウンタ、ポイント名、同期用変数）を管理するための共有メモリ構造体です。

シグネチャまたは形状

typedef struct InjectionPointSharedState
{
    slock_t        lock;
    uint32         wait_counts[INJ_MAX_WAIT];
    char           name[INJ_MAX_WAIT][INJ_NAME_MAXLEN];
    ConditionVariable wait_point;
} InjectionPointSharedState;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。ただし、共有メモリ上のインスタンスとして、injection_points_wakeup によるカウンタ更新や、injection_wait による ConditionVariable を介した待機状態の制御に使用されます。

前提条件と不変条件

  • 共有メモリ上に配置され、lock フィールドによって内部フィールドへのアクセスが保護される必要があります。
  • wait_counts と name は INJ_MAX_WAIT で定義された上限まで管理されます。

アルゴリズムまたは振る舞い

  • **同期**: wait_point（ConditionVariable）を用いて、特定のインジェクションポイントで待機中のプロセスを管理します。
  • **状態管理**: injection_points_wakeup が呼び出されると、対応するインジェクションポイントのカウンタがインクリメントされ、待機中のプロセスに対して通知（Broadcast）が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: injection_points.c 内の共有メモリ初期化処理（injection_init_shmem 等）および状態アクセス関数。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ConditionVariable: プロセス間同期に使用されるプリミティブ。
  • slock_t: 共有メモリ上のデータ保護に使用されるスピンロック。

失敗モードと境界条件

  • INJ_MAX_WAIT を超える数のインジェクションポイントを同時に待機させることはできません。
  • 共有メモリの初期化（inj_state）が完了していない状態でアクセスした場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体はテスト専用モジュールの一部であり、本番環境での使用は想定されていません。
  • 共有メモリのサイズ計算には sizeof(InjectionPointSharedState) が使用されているため、構造体のフィールドを変更した場合は関連するメモリ確保ロジックの整合性を確認してください。
