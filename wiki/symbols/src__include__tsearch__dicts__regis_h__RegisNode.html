RegisNode

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/dicts/regis.h:17

目的

PostgreSQL の全文検索（tsearch）における、高速な正規表現サブセットのパターン要素を表現するリンクリスト構造体です。

シグネチャまたは形状

typedef struct RegisNode
{
    uint32
        type:2,
        len:16,
        unused:14;
    struct RegisNode *next;
    unsigned char data[FLEXIBLE_ARRAY_MEMBER];
} RegisNode;

入力と出力

  • **入力**: RS_compile 時に文字列パターンから生成され、data メンバに文字集合が格納されます。
  • **出力**: RS_execute が本構造体のリストを走査し、対象文字列とのマッチング結果を返します。

前提条件と不変条件

  • type メンバは、マッチングの種類（RSF_ONEOF または RSF_NONEOF）を識別します。
  • data はフレキシブル配列メンバであり、len で指定された長さのデータが格納されます。
  • RS_compile で生成されたリストは、RS_free を用いて明示的にメモリ解放する必要があります。

アルゴリズムまたは振る舞い

  • リンクリスト形式でパターンを保持し、next ポインタにより後続のノードへ連結されます。
  • RS_execute は、各ノードの type に基づき、指定された文字集合の包含または除外判定を順次行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/regis.c 内の newRegisNode 関数が本構造体のインスタンスを生成します。また、RS_compile や RS_execute 等の関数がリストを操作します。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接関数を呼び出すことはありません。

関連データ構造

  • Regis: パターン全体を管理する構造体であり、RegisNode のリストを保持します。

失敗モードと境界条件

  • パターンが長すぎる場合、リンクリストの走査によりマッチングコストが増大します。
  • data 領域のメモリ割り当ては RNHDRSZ（offsetof(RegisNode, data)）を基準に行われるため、この計算と実際のデータ長が一致しない場合、メモリ破壊の恐れがあります。

保守担当者向けメモ

  • 本構造体は汎用的な正規表現エンジンではなく、[...] および [^...] 形式の限定的なパターンマッチング専用です。
  • unused ビットフィールドは現在使用されていません。
