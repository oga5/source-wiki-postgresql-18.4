RunObjectPostAlterHookStr

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaccess.c:217

目的

データベースオブジェクトの変更（ALTER）操作後に、名前ベースで識別されるオブジェクトに対して外部フックを呼び出し、通知を行うためのエントリーポイントを提供します。

シグネチャまたは形状

void RunObjectPostAlterHookStr(Oid classId, const char *objectName, int subId, Oid auxiliaryId, bool is_internal)

入力と出力

  • **入力**:
  • classId: 対象オブジェクトのカタログOID。
  • objectName: 対象オブジェクトの名前。
  • subId: サブオブジェクトID（該当しない場合は0）。
  • auxiliaryId: 補助的なOID。
  • is_internal: 内部操作かどうかを示すフラグ。
  • **出力**: なし（void）。

前提条件と不変条件

  • グローバル変数 object_access_hook_str が非NULLである必要があります（Assertにより検証）。

アルゴリズムまたは振る舞い

1. ObjectAccessPostAlter 構造体をスタック上に確保し、ゼロクリアします。 2. 引数 auxiliaryId および is_internal を構造体に格納します。 3. 登録されているフック関数 object_access_hook_str を呼び出します。その際、イベントタイプとして OAT_POST_ALTER を指定し、オブジェクト識別情報および構築した構造体へのポインタを渡します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコードの文脈からは具体的な呼び出し元は特定されていません。
  • **呼び出し先**: グローバル関数ポインタ object_access_hook_str が指す関数（外部プラグイン等で登録されたもの）。

関連データ構造

  • ObjectAccessPostAlter: フック関数に渡される引数構造体。
  • OAT_POST_ALTER: フックに通知されるイベントタイプ。

失敗モードと境界条件

  • object_access_hook_str が NULL の状態で呼び出された場合、アサーション違反によりプロセスが停止します。

保守担当者向けメモ

  • 本関数は名前ベースの識別が必要なケースで使用されます。OIDが確定している場合は RunObjectPostAlterHook の使用を検討してください。
  • 実際の処理はロードされた共有ライブラリ（プラグイン）側に依存します。
