Gzip_eof

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_gzip.c:349

目的

GZIP 圧縮されたファイルハンドルが EOF（ファイル終端）に達しているかを判定する。

シグネチャまたは形状

static bool Gzip_eof(CompressFileHandle *CFH)

入力と出力

  • **入力**: CompressFileHandle *CFH（GZIP 圧縮状態を保持するファイルハンドル構造体）
  • **出力**: bool（EOF に達している場合は true、それ以外は false）

前提条件と不変条件

  • CFH->private_data に有効な gzFile ハンドルが格納されていること。

アルゴリズムまたは振る舞い

1. CFH->private_data を gzFile 型にキャストして取得する。 2. zlib の gzeof() 関数を呼び出し、戻り値が 1（EOF）であるかを判定して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 直接的な呼び出し元はソースコードの文脈からは特定できません。
  • **呼び出し先**: zlib ライブラリの gzeof() を呼び出します。
  • **参照**: src/bin/pg_dump/compress_gzip.c:429 にて、CFH->eof_func フィールドに関数ポインタとして代入されています。

関連データ構造

  • CompressFileHandle: 圧縮 I/O を抽象化する構造体。private_data メンバを介して gzFile を保持します。

失敗モードと境界条件

  • CFH が NULL の場合、あるいは private_data が初期化されていない場合の動作は未定義です。
  • gzeof() の仕様に基づき、エラー発生時や EOF 未到達時には 0 が返されます。

保守担当者向けメモ

  • 本関数は CompressFileHandle のインターフェースの一部として実装されています。
  • zlib の gzeof は、読み込み操作中に EOF に達したかどうかを報告します。
