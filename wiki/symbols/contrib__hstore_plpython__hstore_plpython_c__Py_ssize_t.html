Py_ssize_t

定義

  • 種別: typedef
  • 定義箇所: contrib/hstore_plpython/hstore_plpython.c:16

目的

Python C API において、コンテナのサイズ、インデックス、または文字列の長さを表現するために使用される符号付き整数型です。

シグネチャまたは形状

Python C API の標準的な型定義であり、プラットフォームのポインタサイズに応じて ssize_t または同等の符号付き整数型に解決されます。

入力と出力

  • **入力**: 整数値（サイズ、インデックス、長さ）。
  • **出力**: 符号付き整数値。

前提条件と不変条件

  • 負の値は、Python C API の一部の関数においてエラー状態や特殊なインデックス（末尾からのオフセットなど）を示すために使用されます。
  • コンテナのサイズやインデックスとして使用する場合、そのコンテナの許容範囲内である必要があります。

アルゴリズムまたは振る舞い

本モジュール内では、主に以下の用途で使用されます。

  • hstore データの要素数（pcount）の保持。
  • ループ処理におけるインデックス（i）の管理。
  • PLyUnicode_FromStringAndSize 等の関数において、文字列の長さを指定する引数として利用。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/hstore_plpython/hstore_plpython.c 内の変換関数（hstore_to_plpython 等）において、ループカウンタやサイズ保持用変数として直接参照されています。
  • **呼び出し先**: Python C API の関数（PyList_New, PyList_SetItem, PLyUnicode_FromStringAndSize 等）に引数として渡されます。

関連データ構造

  • PyObject: Python オブジェクトの基底構造体。
  • hstore: PostgreSQL 側のキーバリュー格納構造体。

失敗モードと境界条件

  • **オーバーフロー**: 非常に大きなデータセットを扱う際、プラットフォームの Py_ssize_t の最大値を超えるインデックスやサイズを指定すると、Python C API 側でエラーが発生する可能性があります。
  • **負の値の誤用**: インデックスとして負の値を渡す際、API がそれをサポートしていない関数であれば、予期せぬ動作やクラッシュを招く可能性があります。

保守担当者向けメモ

  • 本シンボルは Python C API の標準型であるため、PostgreSQL 固有の型ではなく Python の実行環境に依存します。
  • contrib/jsonb_plpython や src/pl/plpython など、他の plpython 関連モジュールでも共通して使用されています。
