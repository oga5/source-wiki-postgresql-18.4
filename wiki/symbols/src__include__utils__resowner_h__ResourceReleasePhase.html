ResourceReleasePhase

定義

  • 種別: enum
  • 定義箇所: src/include/utils/resowner.h:52

目的

リソースオーナー（ResourceOwner）が保持するリソースを解放する際、依存関係を考慮した順序付けを行うためのフェーズを定義します。

シグネチャまたは形状

typedef enum
{
    RESOURCE_RELEASE_BEFORE_LOCKS = 1,
    RESOURCE_RELEASE_LOCKS,
    RESOURCE_RELEASE_AFTER_LOCKS,
} ResourceReleasePhase;

入力と出力

  • 本シンボル自体は列挙型であり、関数への引数として渡されることで、対象となる解放処理の範囲を制御します。

前提条件と不変条件

  • ResourceOwnerRelease 関数において、指定されたフェーズに従って登録済みのリソース解放コールバックが実行されます。
  • ロック管理は専用のキャッシュで最適化されているため、本フェーズの定義は汎用的なリソース追跡メカニズムとロック解放の整合性を保つために重要です。

アルゴリズムまたは振る舞い

  • RESOURCE_RELEASE_BEFORE_LOCKS: ロック解放前に実行すべきクリーンアップ（バッファピンの解放など）。
  • RESOURCE_RELEASE_LOCKS: ロック自体の解放。
  • RESOURCE_RELEASE_AFTER_LOCKS: ロック解放後に実行すべきクリーンアップ。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/resowner/resowner.c 内の ResourceOwnerReleaseAll 等の関数が、リソース解放処理の引数として利用しています。
  • 呼び出し先: 本シンボルは列挙型であり、直接の呼び出し先はありません。ただし、ResourceReleaseCallback 型の関数ポインタの引数として使用されます。

関連データ構造

  • ResourceOwner: リソースを追跡する不透明なデータ構造。
  • ResourceReleaseCallback: 本フェーズを引数に取るコールバック関数型。

失敗モードと境界条件

  • 解放コールバックはコミットまたはアボート後に実行されるため、失敗が許されない非クリティカルな処理のみを実装する必要があります。

保守担当者向けメモ

  • 拡張機能で新しいリソース型を追加する場合は、既存の優先度（ResourceReleasePriority）と併せて、適切なフェーズを選択してください。
