inet_out

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network.c:164

目的

inet 型の内部表現を、人間が読み取り可能な文字列形式に変換して出力します。

シグネチャまたは形状

Datum inet_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される inet 型のポインタ。
  • **出力**: PG_RETURN_CSTRING を介して、IP アドレスの文字列表現（char *）を返します。

前提条件と不変条件

  • 入力引数は inet 型の有効なポインタである必要があります。
  • PG_GETARG_INET_PP(0) を使用して、TOAST 展開済みの inet 構造体を取得します。

アルゴリズムまたは振る舞い

1. PG_GETARG_INET_PP(0) を呼び出し、入力された inet 型のデータを取得します。 2. network_out(src, false) を呼び出します。第2引数に false を渡すことで、cidr 型ではなく inet 型としての出力形式（マスク長を省略しない等の挙動）を指定します。 3. network_out が生成した文字列を呼び出し元に返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: network_out
  • **呼び出し元**: ソースコード内での直接的な呼び出し元は確認されていません（SQL インターフェース経由で呼び出されることが想定されます）。

関連データ構造

  • inet: PostgreSQL のネットワーク型を保持する内部構造体。

失敗モードと境界条件

  • メモリ不足等のシステムエラーが発生した場合、PostgreSQL のエラーハンドリング機構により中断されます。
  • 入力が不正な inet 構造体である場合、network_out 内部で予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は inet 型専用のラッパーです。cidr 型の出力には cidr_out が使用されます。
  • 文字列変換のコアロジックは network_out に集約されているため、出力形式の変更が必要な場合は network_out を参照してください。
