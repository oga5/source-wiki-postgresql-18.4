WaitEventCustomNew

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/wait_event.c:174

目的

拡張モジュールやインジェクションポイントが使用するカスタム待機イベントを動的に登録し、一意な wait_event_info を生成・取得する。

シグネチャまたは形状

static uint32 WaitEventCustomNew(uint32 classId, const char *wait_event_name)

入力と出力

  • **入力**:
  • classId: 待機イベントのクラスID（上位ビット）。
  • wait_event_name: 登録する待機イベント名（文字列）。
  • **出力**:
  • 成功時: 登録された待機イベントを一意に識別する uint32 型の wait_event_info。
  • **例外**:
  • wait_event_name が NAMEDATALEN 以上の場合、エラー。
  • 同一名で異なるクラスIDが指定された場合、ERRCODE_DUPLICATE_OBJECT でエラー。
  • 登録数が WAIT_EVENT_CUSTOM_HASH_MAX_SIZE を超えた場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラー。

前提条件と不変条件

  • wait_event_name は NAMEDATALEN 未満であること。
  • 共有メモリ上のハッシュテーブル WaitEventCustomHashByName および WaitEventCustomHashByInfo が初期化済みであること。

アルゴリズムまたは振る舞い

1. WaitEventCustomLock を共有モードで取得し、指定された名前が既に登録済みか確認する。存在すればその wait_event_info を返す。 2. 未登録の場合、WaitEventCustomLock を排他モードで再取得し、競合プロセスによる重複登録を再チェックする。 3. WaitEventCustomCounter->mutex（スピンロック）を使用して、共有メモリ上のカウンタから新しい eventId を取得・インクリメントする。 4. classId と eventId を組み合わせて wait_event_info を生成する。 5. WaitEventCustomHashByInfo および WaitEventCustomHashByName の両方に新しいエントリを登録する。 6. WaitEventCustomLock を解放し、生成した wait_event_info を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • WaitEventExtensionNew (推論: src/backend/utils/activity/wait_event.c:165)
  • WaitEventInjectionPointNew (推論: src/backend/utils/activity/wait_event.c:171)
  • **呼び出し先**:
  • hash_search, LWLockAcquire, LWLockRelease, SpinLockAcquire, SpinLockRelease, pgstat_get_wait_event_type, strlcpy, elog/ereport

関連データ構造

  • WaitEventCustomEntryByName: 名前から情報を引くためのハッシュエントリ。
  • WaitEventCustomEntryByInfo: IDから名前を引くためのハッシュエントリ。
  • WaitEventCustomCounterData: 次のIDを保持する共有メモリ上のカウンタ。

失敗モードと境界条件

  • **名前の長さ**: NAMEDATALEN を超える名前は拒否される。
  • **リソース上限**: WAIT_EVENT_CUSTOM_HASH_MAX_SIZE（128）を超えて登録しようとするとエラーとなる。
  • **重複定義**: 既存のイベント名に対して異なるクラスIDで登録しようとするとエラーとなる。

保守担当者向けメモ

  • 本関数は共有メモリ上のハッシュテーブルを操作するため、排他制御（LWLock および SpinLock）が必須である。
  • 頻繁に呼び出されるパスではないが、拡張モジュールが初期化時に呼び出すことを想定している。
