RecoveryTargetType

定義

  • 種別: enum
  • 定義箇所: src/include/access/xlogrecovery.h:23

目的

WAL（Write-Ahead Log）リカバリ処理において、データベースの復旧を停止させるための条件種別を定義します。

シグネチャまたは形状

typedef enum
{
	RECOVERY_TARGET_UNSET,
	RECOVERY_TARGET_XID,
	RECOVERY_TARGET_TIME,
	RECOVERY_TARGET_NAME,
	RECOVERY_TARGET_LSN,
	RECOVERY_TARGET_IMMEDIATE,
} RecoveryTargetType;

入力と出力

  • 本シンボル自体は列挙型であり、入力や出力は持ちません。
  • 関連するグローバル変数 recoveryTarget（RecoveryTargetType型）が、リカバリプロセスの停止条件を保持します。

前提条件と不変条件

  • リカバリ開始時、recoveryTarget はデフォルトで RECOVERY_TARGET_UNSET に初期化されます。
  • リカバリプロセスは、設定されたターゲット条件に到達した時点で停止または一時停止の判定を行います。

アルゴリズムまたは振る舞い

  • RECOVERY_TARGET_UNSET: リカバリターゲットが設定されていません。
  • RECOVERY_TARGET_XID: 特定のトランザクションID（XID）に到達した時点で停止します。
  • RECOVERY_TARGET_TIME: 特定のタイムスタンプに到達した時点で停止します。
  • RECOVERY_TARGET_NAME: 名前付きリストアポイントに到達した時点で停止します。
  • RECOVERY_TARGET_LSN: 特定のログシーケンス番号（LSN）に到達した時点で停止します。
  • RECOVERY_TARGET_IMMEDIATE: リカバリ可能な最も早い一貫性のある時点（即時）で停止します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlogrecovery.c にて、リカバリターゲットの初期化および判定ロジックで使用されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • recoveryTarget: src/include/access/xlogrecovery.h で宣言されている RecoveryTargetType 型の外部変数。

失敗モードと境界条件

  • ターゲットが設定されていない状態でリカバリが完了した場合、WALの末尾まで再生が継続されます。
  • ターゲット条件が過去の時点や到達不可能なLSNを指定した場合の挙動は、リカバリプロセスの制御ロジックに依存します。

保守担当者向けメモ

  • 本列挙型に新しい停止条件を追加する場合は、src/backend/access/transam/xlogrecovery.c 内の判定ロジック（PerformWalRecovery 等）に該当する処理を追加する必要があります。
