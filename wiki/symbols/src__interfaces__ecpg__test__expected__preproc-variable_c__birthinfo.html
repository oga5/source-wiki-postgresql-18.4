目的

ECPG（Embedded SQL for C）のテストにおいて、データベースの行データに含まれる誕生年および年齢情報を保持するためのデータ構造です。

シグネチャまたは形状

struct birthinfo {
    long born;
    short age;
};

入力と出力

  • 入力: データベースから取得した born（long型）および age（short型）の値。
  • 出力: ECPGのホスト変数として、SQL文の実行結果を格納するコンテナとして機能します。

前提条件と不変条件

  • 本構造体はプリプロセッサによって生成されたコードの一部であり、手動編集は想定されていません。
  • ECPGdo 関数呼び出しにおいて、対応するインジケータ変数構造体（ind_birth）とペアでバインドされることが前提となります。

アルゴリズムまたは振る舞い

  • データベースから取得したデータを、C言語の構造体メンバにマッピングします。
  • ECPGdo 実行時に、sizeof(struct birthinfo) を用いてメモリレイアウトが指定され、ホスト変数およびインジケータ変数のオフセットが管理されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/test/expected/preproc-variable.c 内の SQL 実行処理。
  • 呼び出し先: ECPGdo（ECPGライブラリ関数）。
  • 参照箇所:
  • birth および ind_birth 変数の型として使用。
  • ECPGdo の引数リストにおいて、p->birth.born および p->birth.age がホスト変数として、i->ind_birth.born および i->ind_birth.age がインジケータ変数として参照されています。

関連データ構造

  • personal_struct: birthinfo 型のメンバ birth を保持する親構造体。
  • personal_indicator: birthinfo 型のメンバに対応するインジケータ変数を保持する構造体。

失敗モードと境界条件

  • データベース側の型と long/short 型の不一致が発生した場合、ECPGのバインド処理でエラーが発生する可能性があります。
  • 構造体のメモリ配置が ECPGdo が期待するサイズと一致しない場合、データ取得時に不正なメモリ参照が発生するリスクがあります。
