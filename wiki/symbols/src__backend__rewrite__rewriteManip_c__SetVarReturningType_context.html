SetVarReturningType_context

定義

  • **種別**: struct
  • **定義箇所**: src/backend/rewrite/rewriteManip.c:940

目的

クエリツリーの走査中に、特定の範囲テーブルエントリ（RTE）に関連する変数の戻り値型（VarReturningType）を更新するための状態を保持するコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    int                result_relation;
    int                sublevels_up;
    VarReturningType   returning_type;
} SetVarReturningType_context;

入力と出力

  • **入力**:
  • result_relation: 処理対象とする範囲テーブルのインデックス。
  • sublevels_up: 現在のネストレベル。
  • returning_type: 設定対象となる VarReturningType 値。
  • **出力**: 構造体自体が走査関数（SetVarReturningType_walker）の引数として渡され、走査中の状態管理に使用されます。

前提条件と不変条件

  • sublevels_up は、サブクエリに潜る際にインクリメントされ、戻る際にデクリメントされる必要があります。
  • 走査対象のノードが Var ノードである場合、varno と varlevelsup がこのコンテキスト内の値と一致するかどうかが判定基準となります。

アルゴリズムまたは振る舞い

  • 本構造体は SetVarReturningType_walker 関数によって利用されます。
  • 走査中に遭遇した Var ノードに対し、varno が result_relation と一致し、かつ varlevelsup が sublevels_up と一致する場合に、対象ノードの型情報を更新するロジックの制御に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/rewrite/rewriteManip.c:981 にてインスタンス化されます。
  • **呼び出し先**: SetVarReturningType_walker 関数がこの構造体を引数として受け取ります。

関連データ構造

  • VarReturningType: 更新対象となる型情報。
  • SetVarReturningType_walker: このコンテキストを利用してツリーを走査する関数。

失敗モードと境界条件

  • 構造体内の sublevels_up の管理が不適切である場合、誤ったネストレベルの変数が更新される可能性があります。

保守担当者向けメモ

  • 本構造体は rewriteManip.c 内の特定の走査ロジックに特化しています。他の走査関数と同様に、expression_tree_walker を介した再帰的な処理を前提としています。
