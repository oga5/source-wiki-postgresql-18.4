conditional_stack_poke

定義

  • 種別: function
  • 定義箇所: src/fe_utils/conditional.c:117

目的

スタックの最上位（現在のアクティブな \if ブロック）の状態を、指定された新しい状態に更新する。

シグネチャまたは形状

bool conditional_stack_poke(ConditionalStack cstack, ifState new_state)

入力と出力

  • **入力**:
  • cstack: 操作対象の ConditionalStack 構造体。
  • new_state: 設定する ifState 型の新しい状態。
  • **出力**:
  • bool: スタックが空でなく更新に成功した場合は true、スタックが空で更新できなかった場合は false を返す。

前提条件と不変条件

  • cstack は conditional_stack_create によって初期化されている必要がある。
  • スタックが空（conditional_stack_empty が true）の場合、状態の更新は行われない。

アルゴリズムまたは振る舞い

1. conditional_stack_empty(cstack) を呼び出し、スタックが空かどうかを確認する。 2. 空であれば false を返して終了する。 3. 空でなければ、スタックの先頭要素（cstack->head）の if_state メンバに new_state を代入する。 4. true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pgbench/pgbench.c: \if ブロックの条件判定結果に基づき状態を更新するために使用。
  • src/bin/psql/command.c: \if, \elif, \else, \endif コマンドの処理において、現在のブロック状態を遷移させるために使用。
  • **呼び出し先**:
  • conditional_stack_empty: スタックの空判定に使用。

関連データ構造

  • ConditionalStack: スタック全体を管理する構造体。
  • ifState: IFSTATE_TRUE, IFSTATE_FALSE, IFSTATE_IGNORED, IFSTATE_ELSE_TRUE, IFSTATE_ELSE_FALSE などの状態を定義する列挙型。

失敗モードと境界条件

  • スタックが空の状態で呼び出された場合、副作用なしで false を返す。

保守担当者向けメモ

  • 本関数はスタックの「トップ」のみを書き換える。ネストされたブロックの階層構造自体には影響を与えない。
  • pgbench および psql の両方から利用されているため、状態遷移のロジックを変更する際は両方のクライアントへの影響を考慮すること。
