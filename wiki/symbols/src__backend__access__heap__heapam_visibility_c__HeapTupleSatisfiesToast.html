HeapTupleSatisfiesToast

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_visibility.c:361

目的

TOASTテーブルに格納されたタプルが、現在のコンテキストにおいて有効（アクセス可能）であるかを判定します。

シグネチャまたは形状

static bool HeapTupleSatisfiesToast(HeapTuple htup, Snapshot snapshot, Buffer buffer)

入力と出力

  • **入力**:
  • htup: 判定対象のタプル。
  • snapshot: 可視性判定に使用するスナップショット（本関数内では直接参照されません）。
  • buffer: タプルが格納されているバッファ。
  • **出力**:
  • bool: タプルが有効であれば true、無効であれば false を返します。

前提条件と不変条件

  • htup->t_self が有効なアイテムポインタであること。
  • htup->t_tableOid が InvalidOid ではないこと。
  • 呼び出し元は、対象のバッファに対して少なくとも共有ロックを保持している必要があります。

アルゴリズムまたは振る舞い

1. **コミット済みチェック**: HeapTupleHeaderXminCommitted が真であれば、即座に true を返します。 2. **無効チェック**: HeapTupleHeaderXminInvalid が真であれば、false を返します。 3. **レガシーフラグ処理**: 9.0以前のバイナリアップグレードで使用される HEAP_MOVED_OFF または HEAP_MOVED_IN フラグを確認します。

  • HEAP_MOVED_OFF: xvac が現在のトランザクションであれば false。それ以外でコミット済みであれば SetHintBits で HEAP_XMIN_INVALID を設定し false を返します。
  • HEAP_MOVED_IN: xvac が現在のトランザクションでなく、かつコミット済みであれば HEAP_XMIN_COMMITTED を設定します。アボート済みであれば HEAP_XMIN_INVALID を設定し false を返します。

    • 4. **推測的挿入の判定**: Xmin が無効な場合は false を返します。 5. 上記のいずれにも該当しない場合、TOASTタプルとして有効とみなして true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_visibility.c 内の他の可視性判定ロジックから参照されています。
  • **呼び出し先**:
  • SetHintBits: ヒントビットの更新に使用。
  • TransactionIdIsCurrentTransactionId, TransactionIdIsInProgress, TransactionIdDidCommit: トランザクション状態の確認に使用。

関連データ構造

  • HeapTuple: 判定対象のタプル構造体。
  • HeapTupleHeader: タプルヘッダー情報（t_infomask 等）。
  • Buffer: バッファ管理構造体。

失敗モードと境界条件

  • Xmin が無効な場合や、特定の移行フラグ（HEAP_MOVED_OFF/HEAP_MOVED_IN）によって無効と判定された場合に false を返します。
  • 競合回避のため、TransactionIdIsInProgress を TransactionIdDidCommit より先にチェックする順序が維持されています。

保守担当者向けメモ

  • 本関数は主にTOASTタプルの特殊な可視性判定に特化しています。
  • snapshot 引数はシグネチャに含まれていますが、現在の実装では判定ロジック内で使用されていません。
