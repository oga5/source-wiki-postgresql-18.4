LogicalRepInfos

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_basebackup/pg_createsubscriber.c:70

目的

論理レプリケーションのサブスクライバー変換プロセスにおいて、データベースごとの情報管理およびクリーンアップ対象のオブジェクト状態を保持するために使用されます。

シグネチャまたは形状

struct LogicalRepInfos
{
	struct LogicalRepInfo *dbinfo;
	bool		two_phase;
	bits32		objecttypes_to_clean;
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。各フィールドは以下の役割を持ちます。

  • dbinfo: データベースごとのパブリケーション/サブスクリプション情報を格納する LogicalRepInfo 構造体へのポインタ。
  • two_phase: 2フェーズコミットを有効にするかを示すフラグ。
  • objecttypes_to_clean: サブスクライバー側でクリーンアップすべきオブジェクトの種類を示すビットフラグ。

前提条件と不変条件

  • dbinfo が有効な LogicalRepInfo 構造体を指している必要があります。
  • objecttypes_to_clean は、クリーンアップ処理の実行時に適切なビットマスクとして評価される必要があります。

アルゴリズムまたは振る舞い

本構造体は、pg_createsubscriber 実行中に変換対象となるデータベース群のメタデータと、変換失敗時や終了時にクリーンアップすべきリソースの追跡を管理します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_basebackup/pg_createsubscriber.c:141 にて、静的変数 dbinfos として定義されています。
  • 呼び出し先: 構造体であるため、直接的な呼び出し先はありません。

関連データ構造

  • LogicalRepInfo: データベースごとの詳細情報を保持する構造体。

失敗モードと境界条件

  • dbinfo が NULL の状態でアクセスした場合、セグメンテーションフォールトが発生する可能性があります。
  • objecttypes_to_clean に不正なビット値が設定された場合、クリーンアップ処理が意図しない挙動を示す可能性があります。

保守担当者向けメモ

  • 本構造体は pg_createsubscriber のグローバルな状態管理の一部です。変更を加える際は、cleanup_objects_atexit 等のクリーンアップ処理との整合性に注意してください。
