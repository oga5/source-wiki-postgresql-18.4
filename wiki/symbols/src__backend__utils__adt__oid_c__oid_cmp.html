oid_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oid.c:286

目的

qsort 等の標準ライブラリ関数で使用可能な、Oid 型の昇順比較を行うための比較関数を提供します。

シグネチャまたは形状

int oid_cmp(const void *p1, const void *p2)

入力と出力

  • **入力**: 比較対象となる2つの Oid 値へのポインタ (p1, p2)。
  • **出力**: p1 が p2 より小さい場合は負の整数、等しい場合は 0、大きい場合は正の整数を返します。

前提条件と不変条件

  • p1 および p2 は、有効な Oid 値を指すポインタである必要があります。
  • Oid 型の比較は、内部的に pg_cmp_u32 を介して符号なし32ビット整数として行われます。

アルゴリズムまたは振る舞い

1. ポインタ p1 および p2 が指す値を Oid 型としてキャストし、それぞれ v1, v2 に格納します。 2. pg_cmp_u32(v1, v2) を呼び出し、その結果をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_enum.c
  • src/backend/catalog/pg_inherits.c
  • src/backend/commands/subscriptioncmds.c
  • src/backend/utils/adt/acl.c
  • **呼び出し先**: pg_cmp_u32（common/int.h で定義される比較関数）。

関連データ構造

  • Oid: PostgreSQL のオブジェクト識別子（uint32 の別名）。

失敗モードと境界条件

  • 入力ポインタが NULL の場合の動作は定義されていません。
  • Oid は符号なし整数であるため、オーバーフローを考慮した比較が pg_cmp_u32 によって行われます。

保守担当者向けメモ

  • 本関数は qsort のコールバックシグネチャに適合するように設計されています。
  • 比較ロジックを変更する場合は、pg_cmp_u32 の仕様を確認してください。
