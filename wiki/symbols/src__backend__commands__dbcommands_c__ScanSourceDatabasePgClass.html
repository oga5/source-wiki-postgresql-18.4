ScanSourceDatabasePgClass

定義

  • 種別: function
  • 定義箇所: src/backend/commands/dbcommands.c:249

目的

ソースデータベース内の pg_class システムカタログをスキャンし、関連するリレーションのリストを抽出する。主にデータベース複製処理において、コピー対象となるリレーションを特定するために使用される。

シグネチャまたは形状

List *ScanSourceDatabasePgClass(Oid tbid, Oid dbid, char *srcpath)

入力と出力

  • **入力**:
  • tbid: ソースデータベースのテーブルスペースOID。
  • dbid: ソースデータベースのOID。
  • srcpath: ソースデータベースの物理パス。
  • **出力**:
  • List *: 抽出されたリレーション情報（RelFileLocator 等を含むリスト）を返す。

前提条件と不変条件

  • 呼び出し元は、対象のデータベースに対して適切なロックを保持している必要がある。
  • 本関数内部で pg_class に対して AccessShareLock を取得し、処理終了後に解放する。
  • GetLatestSnapshot() を使用してスナップショットを登録し、処理後に必ず登録解除を行う必要がある。

アルゴリズムまたは振る舞い

1. RelationMapOidToFilenumberForDatabase を用いて pg_class のファイル番号を取得する。 2. pg_class に対して AccessShareLock を取得する。 3. smgropen で対象リレーションを開き、全ブロック数を取得する。 4. BAS_BULKREAD 戦略でバッファアクセスを最適化する。 5. 全ブロックを順次走査し、ReadBufferWithoutRelcache で読み込む。 6. 各ページが空でない場合、ScanSourceDatabasePgClassPage を呼び出してタプルを抽出し、リストに追加する。 7. 割り当てたスナップショットを解放し、ロックを解除して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/dbcommands.c 内のデータベース複製ロジックから呼び出される。
  • **呼び出し先**:
  • RelationMapOidToFilenumberForDatabase, LockRelationId, smgropen, smgrnblocks, smgrclose
  • GetAccessStrategy, RegisterSnapshot, UnregisterSnapshot
  • ReadBufferWithoutRelcache, LockBuffer, BufferGetPage, PageIsNew, PageIsEmpty, UnlockReleaseBuffer
  • ScanSourceDatabasePgClassPage (内部ヘルパー)

関連データ構造

  • RelFileLocator: リレーションの物理的な位置を特定する構造体。
  • BufferAccessStrategy: バッファ管理の最適化に使用。
  • Snapshot: トランザクションの可視性制御に使用。

失敗モードと境界条件

  • CHECK_FOR_INTERRUPTS() がループ内に配置されており、ユーザーによる中断要求に応答する。
  • ページが空または新規の場合、そのページはスキップされる。
  • データベースの物理ファイルが存在しない、または読み取り権限がない場合はエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は pg_class を直接スキャンするため、カタログ構造の変更時には ScanSourceDatabasePgClassPage と併せて影響を確認すること。
  • 大規模なデータベースではスキャンに時間がかかるため、BAS_BULKREAD の使用は必須である。
