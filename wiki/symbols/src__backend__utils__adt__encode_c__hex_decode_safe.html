hex_decode_safe

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/encode.c:216

目的

16進数文字列をバイナリデータ（bytea）へ安全にデコードします。エラー発生時に escontext を介した柔軟なエラー報告を可能にします。

シグネチャまたは形状

uint64 hex_decode_safe(const char *src, size_t len, char *dst, Node *escontext)

入力と出力

  • **入力**:
  • src: デコード対象の16進数文字列。
  • len: 入力文字列の長さ。
  • dst: デコード結果を格納するバッファ（十分なサイズが確保されていること）。
  • escontext: エラー報告用コンテキスト（Node *）。
  • **出力**:
  • 戻り値: デコードされたバイナリデータの実際のバイト数（uint64）。

前提条件と不変条件

  • dst バッファは、入力文字列の有効な16進数ペアの数（空白等を除く）以上のサイズが必要です。
  • escontext が提供されている場合、エラー発生時に ereturn マクロにより制御が呼び出し元へ戻ります。

アルゴリズムまたは振る舞い

1. 入力文字列 src を先頭から走査します。 2. 空白文字（スペース、改行、タブ、復帰）はスキップします。 3. get_hex を使用して2文字ずつ16進数として読み取り、1バイトのバイナリ値に変換して dst に書き込みます。 4. 変換中に不正な文字や、奇数個の16進数桁数（データ不足）を検出した場合、escontext を用いてエラーを報告し、処理を中断します。 5. 最終的に書き込んだバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/encode.c:213 (ラッパー関数)
  • src/backend/utils/adt/varlena.c:315
  • **呼び出し先**:
  • get_hex: 16進数文字の数値変換に使用。
  • pg_mblen_range: エラーメッセージ生成時の文字長計算に使用。
  • ereturn: エラー発生時の制御フロー制御に使用。

関連データ構造

  • Node: エラーコンテキストとして使用。

失敗モードと境界条件

  • **不正な文字**: 16進数として解釈できない文字が含まれる場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーとなります。
  • **奇数桁**: 16進数ペアが完結しない場合、エラーとなります。
  • **メモリ**: dst バッファのオーバーフロー保護は呼び出し元に依存します。

保守担当者向けメモ

  • 本関数は hex_decode の安全な代替として設計されています。
  • 新しいエンコーディング方式を追加する際は、同様の _safe パターンに従うことが推奨されます。
