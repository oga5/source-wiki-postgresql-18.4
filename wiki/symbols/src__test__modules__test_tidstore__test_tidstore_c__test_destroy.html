test_destroy

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_tidstore/test_tidstore.c:346

目的

テスト用モジュールにおいて、TidStore インスタンスおよび検証用データ構造のメモリを解放し、状態をリセットする。

シグネチャまたは形状

Datum test_destroy(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS (SQL インターフェース経由の呼び出し)
  • 出力: PG_RETURN_VOID() (戻り値なし)

前提条件と不変条件

  • check_tidstore_available() により、tidstore が初期化済みであることを確認する必要がある。

アルゴリズムまたは振る舞い

1. check_tidstore_available() を呼び出し、テスト対象の TidStore が存在するか検証する。 2. TidStoreDestroy(tidstore) を呼び出し、TidStore インスタンスを破棄する。 3. 静的変数 tidstore を NULL に設定する。 4. 検証用構造体 items の保持する TID 数を 0 にリセットする。 5. items 内の動的に確保された配列 (insert_tids, lookup_tids, iter_tids) を pfree で解放する。

呼び出し元と呼び出し先

  • 呼び出し元: SQL インターフェース経由で呼び出される（PG_FUNCTION_INFO_V1(test_destroy) により登録）。
  • 呼び出し先: check_tidstore_available, TidStoreDestroy, pfree。

関連データ構造

  • tidstore (静的変数): テスト対象の TidStore インスタンス。
  • items (ItemArray 型): テストの整合性検証に使用されるデータ保持用構造体。

失敗モードと境界条件

  • tidstore が初期化されていない状態で呼び出された場合、check_tidstore_available() 内でエラーが発生する。

保守担当者向けメモ

  • 本関数は単一プロセスでのテスト実行を前提としている。
  • items 構造体の各配列ポインタが NULL でないことを前提に pfree を実行しているため、test_create 等の初期化フローと対になっていることを確認すること。
