pgwin32_listen

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/socket.c:325

目的

Windows環境において、Winsockの listen 関数をラップし、PostgreSQLが期待するエラーハンドリングを統合する。

シグネチャまたは形状

int pgwin32_listen(SOCKET s, int backlog)

入力と出力

  • **入力**:
  • s: 待機状態にするソケット（SOCKET型）。
  • backlog: 接続待ちキューの最大長。
  • **出力**:
  • 成功時は 0 を返す。
  • 失敗時は -1 を返し、内部的に errno を設定する。

前提条件と不変条件

  • postgres.h がインクルードされている環境で実行される。
  • Winsock2 APIが利用可能であること。

アルゴリズムまたは振る舞い

1. Winsockの標準関数 listen(s, backlog) を呼び出す。 2. 戻り値が 0 未満（エラー）の場合、TranslateSocketError() を呼び出し、Winsock特有のエラーコードをPOSIX準拠の errno に変換する。 3. 最終的な結果を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/win32_port.h にて listen マクロとして定義されており、PostgreSQLのネットワーク処理コードから透過的に呼び出される。
  • **呼び出し先**:
  • listen (Winsock API)
  • TranslateSocketError (エラー変換ユーティリティ)

関連データ構造

  • SOCKET: Windowsのソケットハンドル型。

失敗モードと境界条件

  • listen が失敗した場合、TranslateSocketError() により errno が適切に設定される。
  • 境界条件に関する具体的な制約はWinsockの listen 実装に依存する。

保守担当者向けメモ

  • 本関数は src/include/port/win32_port.h で listen をオーバーライドするために使用されている。
  • エラー発生時の挙動は TranslateSocketError の実装に依存するため、エラーコードの変換ロジックを変更する場合は同関数を確認すること。
