WriteDataCallback

定義

  • 種別: typedef
  • 定義箇所: src/bin/pg_basebackup/pg_basebackup.c:79

目的

PostgreSQL のストリーミングレプリケーションプロトコルを通じて受信したデータチャンクを処理するためのコールバック関数型です。

シグネチャまたは形状

typedef void (*WriteDataCallback) (size_t nbytes, char *buf, void *callback_data);

入力と出力

  • nbytes: 受信したデータのバイト数。
  • buf: 受信したデータが格納されているバッファへのポインタ。
  • callback_data: コールバック関数に渡される任意のコンテキストデータ（状態保持用）。
  • 戻り値: なし (void)。

前提条件と不変条件

  • buf は nbytes 分の有効なメモリ領域を指している必要があります。
  • callback_data は、実装側で期待される型にキャストして使用されることが前提となります。

アルゴリズムまたは振る舞い

この関数型は、サーバーから受信したデータストリームを逐次処理するために使用されます。具体的な処理内容は、この型を実装する関数に依存します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_basebackup/pg_basebackup.c:1015 の ReceiveCopyData 関数が、このコールバックを引数として受け取り、データ受信時に実行します。
  • 呼び出し先: 実装側の関数に依存します。

関連データ構造

  • ArchiveStreamState や WriteTarState などの状態管理構造体が、callback_data として渡される可能性があります。

失敗モードと境界条件

  • nbytes が 0 の場合の挙動は実装に依存します。
  • buf が NULL の場合、実装側で適切にガードする必要があります。

保守担当者向けメモ

  • 本シンボルは pg_basebackup におけるデータ受信パイプラインの抽象化の一部です。
  • ReceiveCopyData がどのようにこのコールバックを呼び出しているかを確認することで、具体的なデータ処理フローを追跡可能です。
