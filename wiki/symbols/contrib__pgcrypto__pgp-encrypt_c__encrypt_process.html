encrypt_process

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-encrypt.c:188

目的

CFB（Cipher Feedback）モードを用いて、入力データを暗号化し、後続のフィルタへ書き出す。

シグネチャまたは形状

static int encrypt_process(PushFilter *next, void *priv, const uint8 *data, int len)

入力と出力

  • next: 書き込み先の PushFilter オブジェクト。
  • priv: EncStat 構造体へのポインタ（暗号化コンテキストを保持）。
  • data: 暗号化対象の平文データ。
  • len: 入力データの長さ。
  • 戻り値: 成功時は 0、失敗時は負の値を返す。

前提条件と不変条件

  • priv は EncStat 型としてキャスト可能であること。
  • EncStat 内の ciph（暗号化コンテキスト）および buf（一時バッファ）が初期化済みであること。

アルゴリズムまたは振る舞い

1. 入力データ len を ENCBUF サイズごとに分割して処理するループを実行する。 2. pgp_cfb_encrypt を呼び出し、st->ciph を使用して data を st->buf へ暗号化する。 3. 暗号化されたデータを pushf_write を介して next フィルタへ転送する。 4. 処理した分だけ data ポインタを進め、残りの avail を減算する。

呼び出し元と呼び出し先

  • 呼び出し元: ソースコード上の直接的な呼び出し元は不明だが、encrypt_init 等と組み合わせてフィルタチェーンの一部として登録される。
  • 呼び出し先: pgp_cfb_encrypt（暗号化実行）、pushf_write（フィルタへの書き込み）。

関連データ構造

  • PushFilter: 暗号化パイプラインのフィルタ構造体。
  • EncStat: 暗号化状態を保持する構造体（ciph および buf を含む）。

失敗モードと境界条件

  • pgp_cfb_encrypt が負の値を返した場合、そのエラーを伝播する。
  • pushf_write が負の値を返した場合、そのエラーを伝播する。
  • len が 0 の場合、ループは実行されず 0 を返す。

保守担当者向けメモ

  • 本関数は PushFilter インターフェースの一部として実装されている。
  • ENCBUF サイズを超えるデータは、ループ内で分割処理されるため、メモリ使用量は ENCBUF に依存する。
