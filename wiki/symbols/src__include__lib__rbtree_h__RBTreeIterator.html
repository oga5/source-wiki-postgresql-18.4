RBTreeIterator

定義

  • 種別: struct
  • 定義箇所: src/include/lib/rbtree.h:46

目的

赤黒木（RBTree）を走査する際の現在の状態を保持するために使用されます。

シグネチャまたは形状

typedef struct RBTreeIterator RBTreeIterator;

struct RBTreeIterator {
    RBTNode *(*iterate) (RBTreeIterator *iter);
    /* 内部状態フィールドは非公開 */
};

入力と出力

  • **初期化**: rbt_begin_iterate() に渡され、走査順序（RBTOrderControl）に基づいて初期化されます。
  • **出力**: rbt_iterate() を呼び出すことで、順次 RBTNode* が返されます。走査が完了すると NULL を返します。

前提条件と不変条件

  • 走査を開始する前に rbt_begin_iterate() による初期化が必須です。
  • 走査中に木構造が変更（挿入や削除）された場合、イテレータの動作は保証されません。
  • 内部フィールドを直接操作してはなりません。

アルゴリズムまたは振る舞い

  • rbt_begin_iterate() が指定された順序（昇順または降順）に応じた内部関数（rbt_left_right_iterator または rbt_right_left_iterator）を iterate 関数ポインタにセットします。
  • rbt_iterate() は、この関数ポインタを介して次のノードを特定し、イテレータ内部の状態を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/lib/rbtree.c 内の rbt_begin_iterate および rbt_iterate
  • src/include/access/gin_private.h
  • src/test/modules/test_rbtree/test_rbtree.c
  • **呼び出し先**:
  • rbt_left_right_iterator (昇順走査用)
  • rbt_right_left_iterator (降順走査用)

関連データ構造

  • RBTree: 走査対象となる赤黒木本体。
  • RBTNode: 走査によって取得されるノード。
  • RBTOrderControl: 走査順序を制御する列挙型。

失敗モードと境界条件

  • 木が空の場合、rbt_iterate() は即座に NULL を返します。
  • 走査中に木が変更された場合の挙動は未定義です。

保守担当者向けメモ

  • 本構造体は侵入型設計の一部であり、RBTNode を含むユーザー定義構造体へのポインタ変換を前提としています。
  • 内部フィールドの詳細は src/backend/lib/rbtree.c の実装に依存しており、ヘッダーファイル側では不透明な型として扱われます。
