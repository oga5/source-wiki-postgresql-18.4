PLy_cursor

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_cursorobject.c:76

目的

Python から PostgreSQL のカーソルを作成するためのエントリポイントを提供します。SQL クエリ文字列またはプリペアドプランオブジェクトを受け取り、対応するカーソルオブジェクトを生成します。

シグネチャまたは形状

PyObject *PLy_cursor(PyObject *self, PyObject *args)

入力と出力

  • **入力**:
  • self: Python モジュールオブジェクト。
  • args: Python の引数タプル。以下のいずれかの形式を期待します。
  • (char *query): SQL クエリ文字列。
  • (PyObject *plan, [PyObject *planargs]): プリペアドプランオブジェクトおよびオプションの引数。
  • **出力**: 成功時は生成されたカーソルを表す PyObject* を返し、失敗時は NULL を返します。

前提条件と不変条件

  • PyArg_ParseTuple を使用して引数の型を検証します。
  • クエリまたはプランのいずれかが提供される必要があります。

アルゴリズムまたは振る舞い

1. 引数が文字列（クエリ）として解析できる場合、PLy_cursor_query(query) を呼び出してカーソルを作成します。 2. 文字列として解析できない場合、PyErr_Clear() でエラー状態をリセットし、プランオブジェクトとして解析を試みます。 3. プランとして解析できる場合、PLy_cursor_plan(plan, planargs) を呼び出してカーソルを作成します。 4. いずれの形式にも合致しない場合、PLy_exception_set を使用して plpy.cursor にクエリまたはプランが必要である旨の例外を設定し、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_plpymodule.c 内のモジュールメソッド定義（{"cursor", PLy_cursor, ...}）から参照されています。
  • **呼び出し先**:
  • PLy_cursor_query
  • PLy_cursor_plan
  • PLy_exception_set

関連データ構造

  • PLyCursorObject: カーソルの状態を保持する構造体。
  • PLyPlanObject: プリペアドプランを保持する構造体。

失敗モードと境界条件

  • 引数が指定された形式（クエリ文字列またはプランオブジェクト）に適合しない場合、PLy_exc_error 例外が発生します。
  • 内部の PLy_cursor_query または PLy_cursor_plan が失敗した場合、そのエラーが伝播されます。

保守担当者向けメモ

  • 本関数は Python の C API を使用したラッパー関数です。引数解析の順序（クエリ優先）は固定されています。
  • 呼び出し先関数（PLy_cursor_query, PLy_cursor_plan）がメモリコンテキスト管理や SPI 操作の責任を負います。
