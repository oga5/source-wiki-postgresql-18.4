AggClauseCosts

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/pathnodes.h:58

目的

集約演算（Aggregate）の実行コストおよびメモリ使用量を見積もるための情報を保持する構造体です。プランナがクエリ最適化の過程で、集約処理のコスト計算を行う際に使用します。

シグネチャまたは形状

typedef struct AggClauseCosts
{
	QualCost	transCost;		/* 入力行ごとの合計実行コスト */
	QualCost	finalCost;		/* 集約行ごとの合計コスト */
	Size		transitionSpace;	/* 参照渡しされる遷移データのメモリ空間 */
} AggClauseCosts;

入力と出力

  • **入力**: 集約関数に関連するコスト情報（transCost, finalCost）および遷移状態に必要なメモリサイズ（transitionSpace）。
  • **出力**: プランナのコスト見積もりロジックにおいて、集約ノードの総コストを算出するためのパラメータとして利用されます。

前提条件と不変条件

  • transCost および finalCost は QualCost 型であり、CPUコストと演算コストの合計を保持します。
  • transitionSpace は、集約の遷移状態（transition state）を保持するために必要なバイト数を示します。

アルゴリズムまたは振る舞い

プランナは、クエリ内の集約関数を解析する際にこの構造体を初期化し、コスト見積もり関数（costsize.c 内の関数など）に渡します。これにより、集約処理がどの程度の計算資源を消費するかを定量化し、最適な実行計画を選択するための判断材料とします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planner.c (複数の箇所でコスト計算の引数として使用)
  • contrib/postgres_fdw/postgres_fdw.c (外部テーブルの集約コスト計算)
  • **呼び出し先**:
  • src/backend/optimizer/path/costsize.c (この構造体を受け取り、コスト計算を実行する関数群)

関連データ構造

  • QualCost: コスト見積もりに使用される構造体。
  • AggStrategy: 集約戦略（planner.c 等で本構造体と併用される）。

失敗モードと境界条件

  • transitionSpace が 0 の場合、集約状態の保持に特別なメモリ確保が不要であることを示します。
  • 構造体全体が 0 で初期化されている場合（MemSet 等）、コスト計算の初期状態として扱われます。

保守担当者向けメモ

  • 本構造体はプランナのコストモデルに直結しているため、集約関数の実行コスト見積もりロジックを変更する際は、transCost と finalCost の計算内訳を整合させる必要があります。
  • transitionSpace は、特にカスタム集約関数や複雑なデータ型を扱う場合に重要となるため、メモリ見積もりの精度に注意してください。
