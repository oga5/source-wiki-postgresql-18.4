GetAttrDefaultColumnAddress

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_attrdef.c:319

目的

指定された pg_attrdef エントリの OID に対応する、列の ObjectAddress を取得する。

シグネチャまたは形状

ObjectAddress GetAttrDefaultColumnAddress(Oid attrdefoid)

入力と出力

  • **入力**: attrdefoid (Oid) - 検索対象の pg_attrdef エントリの OID。
  • **出力**: ObjectAddress - 該当する列の classId (RelationRelationId)、objectId (テーブルのOID)、objectSubId (列番号) を含む構造体。該当なしの場合は InvalidObjectAddress を返す。

前提条件と不変条件

  • pg_attrdef システムカタログに対する AccessShareLock を取得して処理を行う。
  • 検索には AttrDefaultOidIndexId インデックスを使用する。

アルゴリズムまたは振る舞い

1. pg_attrdef テーブルを AccessShareLock で開く。 2. AttrDefaultOidIndexId を使用し、指定された attrdefoid をキーとして systable_beginscan を開始する。 3. systable_getnext でタプルを取得し、有効であれば Form_pg_attrdef 構造体へキャストする。 4. タプルから adrelid (テーブルOID) と adnum (列番号) を抽出し、ObjectAddress 構造体に格納する。 5. スキャンを終了し、テーブルを閉じて結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c (複数箇所)
  • src/backend/commands/event_trigger.c:1313
  • src/backend/commands/tablecmds.c:15193
  • **呼び出し先**:
  • table_open, table_close
  • ScanKeyInit, systable_beginscan, systable_getnext, systable_endscan
  • GETSTRUCT

関連データ構造

  • ObjectAddress: データベースオブジェクトを一意に識別する構造体。
  • Form_pg_attrdef: pg_attrdef カタログのタプル形式。

失敗モードと境界条件

  • 指定された attrdefoid が存在しない場合、systable_getnext は NULL を返し、関数は InvalidObjectAddress を返す。

保守担当者向けメモ

  • 本関数は pg_attrdef の OID から、そのデフォルト値が属するテーブルと列を逆引きするために使用される。
  • pg_attrdef のスキーマ変更やインデックス定義の変更時には、本関数のスキャンロジックが影響を受ける可能性がある。
