pg_GSS_error_int

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-gssapi-common.c:25

目的

GSSAPIのステータスコード（メジャー/マイナー）を人間が読める文字列に変換し、指定されたバッファへ追記する内部ユーティリティ関数です。

シグネチャまたは形状

static void pg_GSS_error_int(PQExpBuffer str, OM_uint32 stat, int type);

入力と出力

  • str (入力/出力): エラーメッセージを格納する PQExpBuffer へのポインタ。
  • stat (入力): 解析対象のGSSAPIステータスコード。
  • type (入力): ステータスの種類（GSS_C_GSS_CODE または GSS_C_MECH_CODE）。
  • 戻り値: なし。

前提条件と不変条件

  • str は初期化済みの PQExpBuffer である必要があります。
  • gss_display_status が正常にステータスを返せる環境である必要があります。

アルゴリズムまたは振る舞い

1. msg_ctx を 0 で初期化し、ループを開始します。 2. gss_display_status を呼び出し、ステータス情報を取得します。 3. 取得に成功した場合、appendPQExpBufferChar でスペースを挿入し、appendBinaryPQExpBuffer でメッセージ内容を str に追記します。 4. gss_release_buffer で一時バッファを解放します。 5. msg_ctx が 0 になるまで（すべてのステータス情報が取得されるまで）ループを繰り返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/interfaces/libpq/fe-gssapi-common.c 内の pg_GSS_error 関数から、メジャーおよびマイナーコードの解析のために呼び出されます。
  • src/backend/libpq/be-gssapi-common.c からも呼び出されていることがツールにより示唆されています。
  • 呼び出し先:
  • gss_display_status (GSSAPIライブラリ)
  • appendPQExpBufferChar (libpq内部)
  • appendBinaryPQExpBuffer (libpq内部)
  • gss_release_buffer (GSSAPIライブラリ)

関連データ構造

  • PQExpBuffer: エラーメッセージを蓄積するための可変長文字列バッファ。
  • OM_uint32: GSSAPIステータスコードの型。
  • gss_buffer_desc: GSSAPIがステータス文字列を返すために使用する構造体。

失敗モードと境界条件

  • gss_display_status が GSS_S_COMPLETE を返さない場合、ループを即座に終了します。
  • msg_ctx が 0 の場合はループが実行されず、何も追記されません。

保守担当者向けメモ

  • 本関数は ENABLE_GSS マクロが定義されている環境でのみコンパイルされます。
  • 複数のエラー情報がスタックされている場合、すべて回収される設計になっています。
