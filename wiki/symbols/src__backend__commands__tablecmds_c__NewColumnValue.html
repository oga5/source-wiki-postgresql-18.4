NewColumnValue

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/tablecmds.c:234

目的

テーブルの列に対する新しい値を計算するための式と、その実行状態を保持する構造体です。主に ALTER TABLE 等の処理において、列の更新や生成列の評価に使用されます。

シグネチャまたは形状

typedef struct NewColumnValue
{
	AttrNumber	attnum;			/* 対象の列番号 */
	Expr	   *expr;			/* 計算用式 */
	ExprState  *exprstate;		/* 実行状態 */
	bool		is_generated;	/* 生成列（GENERATED）かどうか */
} NewColumnValue;

入力と出力

  • 入力: attnum（列識別子）、expr（評価式）、is_generated（フラグ）を初期化時に設定します。
  • 出力: 実行時に exprstate が設定され、式評価のコンテキストとして利用されます。

前提条件と不変条件

  • attnum は有効なテーブルの列番号である必要があります。
  • exprstate は、実行前に適切なコンテキストで初期化されている必要があります。

アルゴリズムまたは振る舞い

本構造体はリスト形式（List *newvals）で管理され、テーブルの列更新処理において以下の役割を果たします。 1. 更新対象の列番号と、値を算出するための式を保持します。 2. 実行フェーズにおいて、exprstate を用いて式を評価し、新しい列値を生成します。 3. is_generated フラグにより、通常の列更新と生成列の更新を区別します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/tablecmds.c 内の複数の箇所で、palloc0 を用いてメモリ確保され、リストに追加されます（例: 行 7518, 8732, 14535）。
  • 呼び出し先: 構造体内部の exprstate を通じて、PostgreSQL の式評価エンジンが呼び出されます。

関連データ構造

  • List: NewColumnValue のインスタンスを保持するリストとして使用されます。
  • Expr, ExprState: 式の定義と実行状態を保持するために使用されます。

失敗モードと境界条件

  • メモリ不足: palloc0 失敗時にエラーが発生します。
  • 式の評価失敗: exprstate による評価中にエラーが発生した場合、トランザクションが中断される可能性があります。

保守担当者向けメモ

  • 本構造体は ALTER TABLE の複雑な処理フローの一部として使用されます。特に ATRewriteTable 等のデータ書き換えフェーズでの挙動に注意してください。
  • is_generated フラグの扱いは、生成列の制約や依存関係と密接に関連しているため、変更時はカタログ操作との整合性に留意してください。
