cash_send

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/cash.c:602

目的

money 型の値を、クライアントとの通信に適したバイナリ形式（ネットワークバイトオーダー）に変換します。

シグネチャまたは形状

Datum cash_send(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して money 型の値（Cash 型）を受け取ります。
  • **出力**: bytea 型のバイナリデータとして返却されます。

前提条件と不変条件

  • Cash 型は内部的に int64 として保持されています。
  • pq_begintypsend および pq_endtypsend を使用して、PostgreSQL のバイナリプロトコルに従ったデータ構築を行う必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_CASH(0) により、入力された money 値を取得します。 2. pq_begintypsend を呼び出し、送信用のバッファを初期化します。 3. pq_sendint64 を使用して、64ビット整数値をネットワークバイトオーダーでバッファに書き込みます。 4. pq_endtypsend を呼び出し、構築されたバッファを bytea 型として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の型システム（バイナリ送信時）。具体的な呼び出し元コードの詳細は提供されていません。
  • **呼び出し先**: libpq/pqformat.h で定義されている pq_begintypsend, pq_sendint64, pq_endtypsend を呼び出します。

関連データ構造

  • Cash: int64 の別名であり、通貨型の内部表現。
  • StringInfoData: バイナリ送信用のバッファ構造体。

失敗モードと境界条件

  • メモリ不足時やバッファ操作に関連するエラーが発生した場合、libpq 側の関数がエラーを報告します。
  • int64 の範囲を超える値は考慮されていません（内部表現が int64 であるため）。

保守担当者向けメモ

  • 本関数は money 型のバイナリ送信専用です。ロケール依存のフォーマット（cash_out）とは異なり、バイナリ形式はロケールに依存せず、内部表現の int64 をそのまま転送します。
