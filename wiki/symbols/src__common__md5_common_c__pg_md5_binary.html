pg_md5_binary

定義

  • 種別: function
  • 定義箇所: src/common/md5_common.c:107

目的

入力バッファからMD5ハッシュを計算し、その結果を16バイトのバイナリデータとして出力します。

シグネチャまたは形状

bool pg_md5_binary(const void *buff, size_t len, uint8 *outbuf, const char **errstr)

入力と出力

  • buff: ハッシュ計算対象のデータポインタ。
  • len: buff のバイト長。
  • outbuf: 計算結果（16バイト）を格納するバッファ。呼び出し元で確保されている必要があります。
  • errstr: エラー発生時にエラーメッセージへのポインタが代入されます。成功時は NULL が設定されます。
  • 戻り値: 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • outbuf は少なくとも MD5_DIGEST_LENGTH (16バイト) の領域を保持している必要があります。
  • pg_cryptohash_create がメモリ不足等で失敗した場合、関数は直ちに false を返し、errstr にエラー情報が格納されます。

アルゴリズムまたは振る舞い

1. pg_cryptohash_create(PG_MD5) を呼び出し、ハッシュ計算コンテキストを初期化します。 2. pg_cryptohash_init、pg_cryptohash_update、pg_cryptohash_final を順次実行し、MD5ハッシュを計算します。 3. 計算過程でエラーが発生した場合、pg_cryptohash_error を取得し、コンテキストを解放して false を返します。 4. 正常終了時は pg_cryptohash_free を呼び出してコンテキストを解放し、true を返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/libpq/auth.c (認証処理におけるRADIUSベクターの計算等)
  • 呼び出し先:
  • pg_cryptohash_create
  • pg_cryptohash_init
  • pg_cryptohash_update
  • pg_cryptohash_final
  • pg_cryptohash_error
  • pg_cryptohash_free

関連データ構造

  • pg_cryptohash_ctx: ハッシュ計算の内部状態を保持する不透明な構造体。

失敗モードと境界条件

  • コンテキスト生成失敗: pg_cryptohash_create が NULL を返した場合、メモリ不足として処理されます。
  • 計算処理失敗: init, update, final のいずれかが負の値を返した場合、エラーとして処理されます。
  • セキュリティ上の注意: MD5は衝突耐性が低いため、認証以外の用途での利用には注意が必要です。

保守担当者向けメモ

  • 本関数は cryptohash インターフェースのラッパーであり、具体的なMD5アルゴリズムの実装は cryptohash.c および関連するバックエンド実装に依存します。
  • errstr 引数は呼び出し元でエラーログを出力するために使用されます。
