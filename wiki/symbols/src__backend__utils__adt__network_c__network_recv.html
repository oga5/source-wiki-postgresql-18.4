network_recv

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/adt/network.c:191

目的

ネットワーク型（inet または cidr）の外部バイナリ表現を、PostgreSQL 内部の inet 構造体に変換します。

シグネチャまたは形状

inet *network_recv(StringInfo buf, bool is_cidr)

入力と出力

  • **入力**:
  • buf: ネットワークデータを含む StringInfo バッファ。
  • is_cidr: cidr 型として処理するかを示すフラグ。
  • **出力**: 変換された inet 構造体へのポインタ。

前提条件と不変条件

  • cidr 型として処理する場合、マスク長より右側のビットはすべて 0 である必要があります（addressOK による検証）。
  • 入力バッファ内のアドレスファミリーは PGSQL_AF_INET または PGSQL_AF_INET6 である必要があります。

アルゴリズムまたは振る舞い

1. palloc0 を使用して inet 構造体をメモリ確保します。 2. pq_getmsgbyte を用いて、バッファからアドレスファミリー、マスク長、およびアドレス長を順次読み取ります。 3. 読み取った値が妥当か（ファミリーの正当性、マスク長の範囲、アドレス長の一致）を検証し、不正な場合はエラーを報告します。 4. バッファからアドレス本体を inet 構造体のデータ領域へコピーします。 5. is_cidr が真の場合、addressOK を呼び出し、マスク長より右側のビットが 0 であることを確認します。 6. SET_INET_VARSIZE マクロで構造体のサイズを設定し、ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network.c 内の inet_recv および cidr_recv（推測されるラッパー関数）。
  • **呼び出し先**:
  • pq_getmsgbyte: バッファからのバイト読み取り。
  • addressOK: cidr 型のビット妥当性検証。
  • palloc0: メモリ確保。
  • ereport: エラー報告。

関連データ構造

  • inet: ネットワーク型を保持する内部構造体。
  • StringInfo: PostgreSQL のバイナリ通信用バッファ構造体。

失敗モードと境界条件

  • **エラー発生時**: 以下の条件で ERROR を発生させます。
  • サポートされていないアドレスファミリー。
  • マスク長が許容範囲外。
  • バッファから読み取ったアドレス長が期待値と不一致。
  • cidr 型において、マスク長より右側にビットが立っている場合。

保守担当者向けメモ

  • 本関数は inet と cidr の両方で共通のバイナリデシリアライズロジックを提供します。
  • cidr 型の不変条件（マスク長以降のビットが 0）は、この関数で強制されます。
  • pq_getmsgbyte の呼び出し順序は、バイナリプロトコルの仕様と厳密に一致させる必要があります。
