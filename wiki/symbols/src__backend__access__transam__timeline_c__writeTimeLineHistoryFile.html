writeTimeLineHistoryFile

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/timeline.c:462

目的

指定されたタイムラインID（tli）に対応する履歴ファイルを、指定された内容（content）で安全に作成または上書きする。

シグネチャまたは形状

void writeTimeLineHistoryFile(TimeLineID tli, char *content, int size)

入力と出力

  • **入力**:
  • tli: 書き込み対象のタイムラインID。
  • content: 履歴ファイルに書き込む文字列データ。
  • size: 書き込むデータのバイト数。
  • **出力**: なし。成功時はファイルシステム上に履歴ファイルが生成される。失敗時はエラーを報告する。

前提条件と不変条件

  • 書き込みは一時ファイル経由で行われ、最後に durable_rename を使用してアトミックに置き換えられる。
  • 呼び出し元は、content が適切な形式（タブ区切りの履歴テキスト）であることを保証する必要がある。

アルゴリズムまたは振る舞い

1. XLOGDIR 内にプロセスIDを含む一時ファイル（xlogtemp.<pid>）を作成する。 2. content を一時ファイルに書き込む。 3. pg_fsync を呼び出し、ディスクへの物理的な書き込みを保証する。 4. 一時ファイルをクローズする。 5. TLHistoryFilePath を用いて最終的なファイルパスを生成し、durable_rename を使用して一時ファイルを正式な履歴ファイル名にリネームする。これにより、既存のファイルが存在する場合は上書きされる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/timeline.c (内部呼び出し)
  • src/backend/replication/walreceiver.c
  • src/bin/pg_basebackup/receivelog.c
  • **呼び出し先**:
  • OpenTransientFile, CloseTransientFile (ファイル操作)
  • write (システムコール)
  • pg_fsync (同期)
  • TLHistoryFilePath (パス生成)
  • durable_rename (アトミックなファイル移動)
  • pgstat_report_wait_start/end (I/O待機計測)

関連データ構造

  • TimeLineID: タイムラインを識別する整数型。
  • MAXPGPATH: ファイルパスの最大長を定義する定数。

失敗モードと境界条件

  • 一時ファイルの作成、書き込み、または fsync に失敗した場合、ereport(ERROR, ...) を通じて処理を中断する。
  • 書き込み失敗時には、ディスク領域不足（ENOSPC）等のエラーが報告される可能性がある。
  • durable_rename の失敗は、ファイルシステムの整合性を維持するためエラーとして扱われる。

保守担当者向けメモ

  • 本関数は、クラッシュリカバリ時のデータ破損を防ぐため、常に一時ファイル経由の書き込みと fsync を行う設計となっている。
  • durable_rename を使用しているため、履歴ファイルの更新はアトミックであると見なせる。
