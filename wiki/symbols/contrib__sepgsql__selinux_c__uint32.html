uint32

定義

  • 種別: struct（ソースコード上の定義に基づく）
  • 定義箇所: contrib/sepgsql/selinux.c:34

目的

SELinux のアクセス権限コードを保持するためのデータ型として使用されます。

シグネチャまたは形状

struct
{
    const char *av_name;
    uint32      av_code;
} av[32];

※上記は contrib/sepgsql/selinux.c 内でこの型が使用されている構造体の抜粋です。

入力と出力

  • 入力: SELinux ポリシーに関連付けられた数値コード。
  • 出力: 構造体内のメンバとして保持される 32 ビット符号なし整数。

前提条件と不変条件

  • 32 ビットの符号なし整数値として定義されています。
  • contrib/sepgsql モジュール内において、PostgreSQL 内部の権限ビットと SELinux 外部コードをマッピングする際の型として機能します。

アルゴリズムまたは振る舞い

  • 本シンボル自体は型定義であり、特定のアルゴリズムは持ちません。
  • contrib/sepgsql/selinux.c 内では、av_code として権限ビットマスクを格納し、sepgsql_compute_avd 等の関数によるアクセス判定処理において、PostgreSQL 内部形式と SELinux 外部形式の変換に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数関係については、提供されたソース excerpt に情報が含まれていないため不明です。
  • ツールによるヒューリスティックな検索結果では、contrib/amcheck や contrib/bloom 等の他モジュールでも uint32 型が使用されていますが、これらは本ファイル（contrib/sepgsql/selinux.c）の定義を直接参照しているとは限りません。

関連データ構造

  • av[32]: contrib/sepgsql/selinux.c 内で定義されている、権限名とコードをペアにするための配列構造体。

失敗モードと境界条件

  • 32 ビットの範囲を超える値の代入については、標準的な C 言語の型定義に従います。
  • 構造体内の配列サイズ（32）を超える権限定義が必要な場合、この構造体の拡張が必要です。

保守担当者向けメモ

  • 本シンボルは PostgreSQL の標準的な uint32 型の別名、あるいは同等の型定義として扱われています。
  • contrib/sepgsql モジュール内での利用に際しては、sepgsql_catalog を介した動的な権限変換が行われるため、ポリシー変更に伴う型定義の整合性に注意してください。
