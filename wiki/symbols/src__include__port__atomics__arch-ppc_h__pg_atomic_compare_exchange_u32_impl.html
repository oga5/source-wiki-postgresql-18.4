pg_atomic_compare_exchange_u32_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/arch-ppc.h:79

目的

PowerPCアーキテクチャにおいて、lwarx/stwcx.命令を用いた32ビットのアトミックな比較交換（CAS）操作を実行する。

シグネチャまたは形状

static inline bool pg_atomic_compare_exchange_u32_impl(volatile pg_atomic_uint32 *ptr, uint32 *expected, uint32 newval)

入力と出力

  • ptr: 操作対象のアトミック変数へのポインタ。
  • expected: 期待値へのポインタ。成功時は更新されない。失敗時は現在のメモリ値で上書きされる。
  • newval: 書き込む新しい値。
  • 戻り値: 比較が成功し、値が更新された場合は true、失敗した場合は false を返す。

前提条件と不変条件

  • ptr は pg_atomic_uint32 型の有効なメモリ領域を指していること。
  • GCCインラインアセンブラ環境下で動作する。
  • HAVE_I_CONSTRAINT__BUILTIN_CONSTANT_P が定義されている場合、expected が定数であれば最適化された命令（cmpwi）が使用される。

アルゴリズムまたは振る舞い

1. sync 命令によりメモリバリアを張る。 2. lwarx 命令でメモリをロードし、予約をセットする。 3. cmpw（または定数最適化時は cmpwi）で現在の値と expected を比較する。 4. 一致しない場合、lwsync を経て終了する。 5. 一致する場合、stwcx. 命令で newval の書き込みを試みる。 6. stwcx. が失敗した場合（予約が解除された場合）、ステップ2に戻り再試行する。 7. mfcr 命令で条件レジスタを取得し、成功・失敗を判定する。 8. 失敗時は *expected に現在のメモリ値（found）を格納し、結果を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/port/atomics/generic.h 等の汎用アトミックインターフェースから呼び出される。
  • 呼び出し先: 本関数はインラインアセンブラで完結しており、外部関数呼び出しは行わない。

関連データ構造

  • pg_atomic_uint32: 32ビットアトミック操作用の構造体。

失敗モードと境界条件

  • stwcx. が失敗した場合、ループにより再試行が行われるため、競合が激しい環境では処理時間が長くなる可能性がある。
  • expected が指す値が現在のメモリ値と異なる場合、関数は false を返し、expected を更新する。

保守担当者向けメモ

  • インラインアセンブラ内の memory および cc 制約は、コンパイラの最適化を制御するために不可欠である。変更時は PowerPC のメモリモデルへの影響を考慮すること。
  • bne 命令による分岐先オフセット（$+12, $-16）は、アセンブラの命令長に依存するため、命令セットの変更時には注意が必要である。
