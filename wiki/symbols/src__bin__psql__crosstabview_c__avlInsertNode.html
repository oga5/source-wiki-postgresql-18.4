avlInsertNode

定義

  • 種別: function
  • 定義箇所: src/bin/psql/crosstabview.c:528

目的

AVL木に対して新しい pivot_field を挿入し、必要に応じて木を再平衡化します。

シグネチャまたは形状

static void avlInsertNode(avl_tree *tree, avl_node **node, pivot_field field)

入力と出力

  • tree: 操作対象の avl_tree 構造体へのポインタ。
  • node: 挿入位置となる avl_node へのポインタのポインタ。
  • field: 挿入する pivot_field データ。
  • 出力: なし（副作用として tree の構造を更新）。

前提条件と不変条件

  • tree は初期化済みであること。
  • node が指す先が tree->end（番兵）である場合、新しいノードが割り当てられ挿入される。
  • 挿入後、avlAdjustBalance を呼び出し、AVL木の平衡条件を維持する。

アルゴリズムまたは振る舞い

1. 現在のノードが tree->end であれば、新しいノードをメモリ確保し、field を格納して挿入する。 2. tree->count をインクリメントする。 3. 現在のノードが既存のノードであれば、pivotFieldCompare を使用して field を比較する。 4. 比較結果が 0 でない場合、再帰的に左または右の子ノードへ挿入を試みる。 5. 再帰呼び出しの戻り時に avlAdjustBalance を実行し、平衡を保つ。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/crosstabview.c 内の関数（avlInsertNode 自身による再帰呼び出し、および外部からの呼び出し）。
  • 呼び出し先: pivotFieldCompare, avlAdjustBalance, pg_malloc。

関連データ構造

  • avl_tree: 木のルート、カウント、番兵（end）を保持。
  • avl_node: ノードの高さ、データ（field）、左右の子ノードを保持。
  • pivot_field: 挿入されるデータ構造。

失敗モードと境界条件

  • メモリ不足時: pg_malloc が失敗した場合、プロセスは終了する。
  • 重複値: pivotFieldCompare が 0 を返す場合、ノードは挿入されず、木は変更されない。

保守担当者向けメモ

  • 本関数は挿入専用の AVL 木実装の一部です。削除機能は提供されていません。
  • tree->end は空のノードを表す番兵として使用されているため、NULL チェックではなく tree->end との比較が必須です。
