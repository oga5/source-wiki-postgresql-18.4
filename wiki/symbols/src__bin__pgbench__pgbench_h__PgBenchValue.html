PgBenchValue

定義

  • **種別**: struct
  • **定義箇所**: src/bin/pgbench/pgbench.h:44

目的

pgbench の式評価エンジンにおいて、計算結果や変数の値を保持するための汎用的なコンテナとして機能します。

シグネチャまたは形状

typedef struct
{
	PgBenchValueType type;
	union
	{
		int64		ival;
		double		dval;
		bool		bval;
	}			u;
} PgBenchValue;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。値の読み書きには PgBenchValueType に基づく共用体 u の適切なフィールドへのアクセスが必要です。

前提条件と不変条件

  • type フィールドは、共用体 u 内のどのメンバが有効であるかを示します。
  • 構造体へのアクセス時には、必ず type を確認して型安全性を確保する必要があります。

アルゴリズムまたは振る舞い

本構造体は、ベンチマークスクリプト内の式評価結果を格納します。評価エンジンは、この構造体を通じて整数、浮動小数点数、真偽値の各型を統一的に扱います。

呼び出し元と呼び出し先

  • **呼び出し元**: pgbench.c 内の式評価ロジックおよび変数管理ロジック。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接的な呼び出し先はありません。操作関数として setNullValue, setBoolValue, setIntValue, setDoubleValue, coerceToBool, coerceToInt などが pgbench.c で定義されています。

関連データ構造

  • PgBenchValueType: PgBenchValue の型を識別するために使用される列挙型。
  • PgBenchExpr: 式の抽象構文木ノード。評価結果として PgBenchValue を生成します。

失敗モードと境界条件

  • 共用体の型と type フィールドの不一致によるデータ破損。
  • 未初期化の PgBenchValue へのアクセス。

保守担当者向けメモ

  • 新しいデータ型を追加する場合は、共用体 u にメンバを追加し、PgBenchValueType 列挙型を更新してください。
  • 評価ロジック（pgbench.c）側で、型変換や値の取得を行う関数群が適切に更新されていることを確認してください。
