AggStatePerTransData

定義

  • 種別: struct
  • 定義箇所: src/include/executor/nodeAgg.h:30

目的

集約関数（Aggref）の実行において、入力行から中間状態を更新するための遷移関数（transfn）や結合関数、および関連するメタデータを保持する。

シグネチャまたは形状

Aggref へのポインタ、遷移関数・シリアライズ関数の FmgrInfo、ソート状態管理用の Tuplesortstate 配列、および入力引数評価用の TupleTableSlot 等を含む構造体。

入力と出力

  • **入力**: ExecInitAgg() 時にプランノードから初期化される。実行時には sortslot 等を介して入力タプルを受け取る。
  • **出力**: 遷移関数（transfn）の実行結果として、中間状態（Datum）を更新する。

前提条件と不変条件

  • ExecInitAgg() で設定された値は、その後の実行フェーズでは変更されない。
  • 複数の Aggref が同一の遷移状態を共有する場合、入力と遷移関数が同一であり、かつ最終関数が読み書き両用でないことが条件となる。

アルゴリズムまたは振る舞い

  • **単純集約**: 入力値を直接遷移関数に渡して状態を更新する。
  • **DISTINCT / ORDER BY 集約**: 入力値を Tuplesortstate に蓄積し、グループ終了時にソート・重複排除を行った上で遷移関数を適用する。
  • **最適化**: transfn_fcinfo 等の FunctionCallInfo 構造体を事前初期化することで、行ごとの関数呼び出しオーバーヘッドを削減する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeAgg.c 内の ExecInitAgg にてメモリ確保および初期化が行われる。
  • **呼び出し先**: 遷移関数、シリアライズ/デシリアライズ関数が FmgrInfo を介して呼び出される。

関連データ構造

  • Aggref: 関連付けられた集約式。
  • AggStatePerAggData: 最終計算（finalfn）を担当する構造体。
  • Tuplesortstate: ソートが必要な集約で使用されるソート状態。
  • TupleTableSlot: 入力タプルの保持に使用。

失敗モードと境界条件

  • DISTINCT 指定時、単一列か複数列かによって equalfnOne または equalfnMulti が使い分けられる。
  • haslast フラグにより、DISTINCT 処理における最後の値の有無を追跡する。

保守担当者向けメモ

  • AggStatePerTransData と AggStatePerAggData の分離は、メモリ効率と計算効率の最適化を目的としている。
  • 実際のロジック（特に transfn の呼び出しやメモリ管理）については nodeAgg.c の ExecAgg 関数を参照すること。
