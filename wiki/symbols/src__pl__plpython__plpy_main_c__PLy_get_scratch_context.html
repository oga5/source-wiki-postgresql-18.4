PLy_get_scratch_context

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_main.c:376

目的

PL/Python プロシージャ実行中に一時的なメモリ割り当てが必要となった際、遅延初期化されたメモリコンテキストを提供します。

シグネチャまたは形状

MemoryContext PLy_get_scratch_context(PLyExecutionContext *context)

入力と出力

  • **入力**: PLyExecutionContext *context - 現在の実行コンテキストへのポインタ。
  • **出力**: MemoryContext - 一時作業用メモリコンテキストへのハンドル。

前提条件と不変条件

  • 引数 context は有効な PLyExecutionContext を指している必要があります。
  • context->scratch_ctx が NULL の場合、本関数は TopTransactionContext を親として新しいメモリコンテキストを生成します。

アルゴリズムまたは振る舞い

1. context->scratch_ctx が既に割り当てられているか確認します。 2. 未割り当て（NULL）の場合、AllocSetContextCreate を呼び出し、TopTransactionContext を親として「PL/Python scratch context」という名称のメモリコンテキストを生成・代入します。 3. 割り当て済み、または生成後の context->scratch_ctx を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_typeio.c:85
  • src/pl/plpython/plpy_typeio.c:138
  • **呼び出し先**:
  • AllocSetContextCreate (PostgreSQL メモリ管理 API)

関連データ構造

  • PLyExecutionContext: 実行中の手続きやメモリコンテキストを保持するスタック構造体。
  • MemoryContext: PostgreSQL のメモリ管理単位。

失敗モードと境界条件

  • メモリ不足時には AllocSetContextCreate がエラーを発生させ、PostgreSQL のエラー処理メカニズムへ制御が移ります。

保守担当者向けメモ

  • 本関数で生成される scratch_ctx は、PLy_pop_execution_context を通じて適切に破棄される必要があります。メモリリークを防ぐため、コンテキストのライフサイクル管理に注意してください。
