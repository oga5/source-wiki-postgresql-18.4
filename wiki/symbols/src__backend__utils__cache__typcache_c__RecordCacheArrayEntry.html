RecordCacheArrayEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/typcache.c:297

目的

レコード型（複合型）のキャッシュにおいて、特定の識別子とタプル記述子（TupleDesc）を関連付けて保持するためのエントリ構造体です。

シグネチャまたは形状

typedef struct RecordCacheArrayEntry
{
	uint64		id;
	TupleDesc	tupdesc;
} RecordCacheArrayEntry;

入力と出力

  • id: レコード型を識別するための uint64 型の識別子。
  • tupdesc: 該当するレコード型の構造を定義する TupleDesc へのポインタ。

前提条件と不変条件

  • 本構造体は RecordCacheArray と呼ばれる配列内で管理されます。
  • tupdesc は、キャッシュの生存期間中、有効なタプル記述子を指している必要があります。

アルゴリズムまたは振る舞い

  • typcache.c 内で、レコード型のキャッシュ検索や管理を行う際に、識別子と構造定義のペアとして利用されます。
  • RecordCacheArray の初期化時に、64 * sizeof(RecordCacheArrayEntry) のサイズでメモリが割り当てられます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/typcache.c 内のキャッシュ初期化および管理ロジック（1803行目付近）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • TupleDesc: レコードの列定義を保持する構造体。
  • RecordCacheArray: 本構造体を要素として保持する静的配列。

失敗モードと境界条件

  • メモリ割り当て失敗: RecordCacheArray の初期化時にメモリ不足が発生した場合。
  • 識別子の衝突: id が一意でない場合、キャッシュの整合性が損なわれる可能性があります。

保守担当者向けメモ

  • 本構造体は typcache.c 内の静的配列 RecordCacheArray で使用されています。
  • キャッシュの無効化や更新ロジックを修正する際は、この配列内の tupdesc が適切に解放または更新されるか確認してください。
