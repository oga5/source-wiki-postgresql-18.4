ResourceOwnerCreate

定義

  • 種別: function
  • 定義箇所: src/backend/utils/resowner/resowner.c:420

目的

新しいリソースオーナー（ResourceOwner）インスタンスを生成し、階層構造に組み込む。

シグネチャまたは形状

ResourceOwner ResourceOwnerCreate(ResourceOwner parent, const char *name)

入力と出力

  • **入力**:
  • parent: 親となる ResourceOwner（NULL の場合はトップレベルオーナー）。
  • name: デバッグ用の識別名。
  • **出力**:
  • 生成された ResourceOwner 構造体へのポインタ。

前提条件と不変条件

  • TopMemoryContext が初期化されていること。
  • parent が指定された場合、その parent は有効な ResourceOwner であること。

アルゴリズムまたは振る舞い

1. TopMemoryContext 上に ResourceOwnerData 構造体分のメモリをゼロクリアして確保する。 2. 引数 name を構造体に設定する。 3. parent が指定されている場合:

  • 新しいオーナーの parent フィールドに設定する。
  • 親の firstchild リストの先頭に新しいオーナーを挿入する。

    • 4. AIO ハンドル管理用の双方向リスト（aio_handles）を初期化する。 5. 生成したオーナーポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c（トランザクション開始時）
  • src/backend/utils/mmgr/portalmem.c（ポータル作成時）
  • src/backend/utils/resowner/resowner.c（補助プロセス用）
  • src/pl/plpgsql/src/pl_exec.c / pl_handler.c
  • src/test/modules/ 配下のテストコード
  • **呼び出し先**:
  • MemoryContextAllocZero
  • dlist_init

関連データ構造

  • ResourceOwnerData: リソースオーナーの内部状態を保持する構造体。
  • ResourceOwner: ResourceOwnerData へのポインタ型。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAllocZero によりエラーが発生する。

保守担当者向けメモ

  • 本関数で生成されたリソースオーナーは TopMemoryContext に割り当てられるため、不要になった際は ResourceOwnerDelete() を呼び出して明示的にメモリを解放する必要がある。
