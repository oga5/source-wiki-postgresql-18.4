pg_cryptohash_error

定義

  • 種別: function
  • 定義箇所: src/common/cryptohash.c:253

目的

暗号学的ハッシュ計算コンテキストに関連付けられたエラーメッセージを取得する。

シグネチャまたは形状

const char *pg_cryptohash_error(pg_cryptohash_ctx *ctx);

入力と出力

  • **入力**: pg_cryptohash_ctx *ctx（ハッシュ計算コンテキストへのポインタ）。
  • **出力**: エラー内容を示す静的な文字列（const char *）。

前提条件と不変条件

  • ctx が NULL の場合、メモリ不足エラーを示す文字列を返す。
  • 正常終了時は "success" を返す。

アルゴリズムまたは振る舞い

1. ctx が NULL かどうかを判定し、真であれば "out of memory" を返す。 2. ctx->error の値を評価する。

  • PG_CRYPTOHASH_ERROR_NONE: "success" を返す。
  • PG_CRYPTOHASH_ERROR_DEST_LEN: "destination buffer too small" を返す。

    • 3. 上記以外の予期せぬエラー値に対しては Assert(false) を実行し、フォールバックとして "success" を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/uuid-ossp/uuid-ossp.c
  • src/backend/backup/backup_manifest.c
  • src/backend/utils/adt/cryptohashfuncs.c
  • **呼び出し先**:
  • 外部関数への直接の呼び出しはなし。

関連データ構造

  • pg_cryptohash_ctx: ハッシュ計算の状態を保持する不透明な構造体。

失敗モードと境界条件

  • ctx が NULL の場合、メモリ確保失敗として扱われる。
  • 未定義の ctx->error 値が渡された場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は pg_cryptohash_final 等で発生したエラーを診断するために使用される。
  • 新しいエラーコードを pg_cryptohash_ctx に追加した場合は、本関数の switch 文にもケースを追加すること。
