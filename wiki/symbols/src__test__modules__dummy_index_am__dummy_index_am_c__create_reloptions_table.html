create_reloptions_table

定義

  • 種別: function
  • 定義箇所: src/test/modules/dummy_index_am/dummy_index_am.c:73

目的

dummy_index_am モジュールで使用されるカスタムリレーションオプションをシステムに登録し、それらのメタデータ（名前、型、構造体内のオフセット）を di_relopt_tab 配列に格納する。

シグネチャまたは形状

static void create_reloptions_table(void)

入力と出力

  • 入力: なし（グローバル変数 di_relopt_kind および di_relopt_tab を操作する）。
  • 出力: なし（副作用としてシステムへのオプション登録と配列の初期化を行う）。

前提条件と不変条件

  • 本関数はモジュール初期化時に一度だけ実行されることを想定している。
  • di_relopt_kind が add_reloption_kind() によって正しく初期化されている必要がある。
  • di_relopt_tab 配列の各要素は、DummyIndexOptions 構造体のメンバと整合するオフセットを持つ必要がある。

アルゴリズムまたは振る舞い

1. add_reloption_kind() を呼び出し、新しいリレーションオプション種別を取得する。 2. add_int_reloption, add_real_reloption, add_bool_reloption, add_enum_reloption, add_string_reloption を順次呼び出し、各オプションをシステムに登録する。 3. 各登録と並行して、対応する di_relopt_tab のインデックスに対して optname（名前）、opttype（型）、offset（DummyIndexOptions 構造体内のオフセット）を割り当てる。

呼び出し元と呼び出し先

  • 呼び出し元: _PG_init（ソースコード内の文脈より、モジュールロード時に実行される）。
  • 呼び出し先: add_reloption_kind, add_int_reloption, add_real_reloption, add_bool_reloption, add_enum_reloption, add_string_reloption, offsetof マクロ。

関連データ構造

  • DummyIndexOptions: オプション値を保持する構造体。
  • di_relopt_tab: オプションのメタデータを管理する配列。
  • di_relopt_kind: 登録されたオプション種別を識別するID。

失敗モードと境界条件

  • add_..._reloption 関数群は、内部でメモリ不足や不正な引数に対してエラーを発生させる可能性がある。
  • offsetof の指定が DummyIndexOptions 構造体の定義と一致しない場合、実行時に不正なメモリ参照を引き起こす可能性がある。

保守担当者向けメモ

  • 新しいオプションを追加する場合は、di_relopt_tab のサイズとインデックスが整合していることを確認すること。
  • AccessExclusiveLock が各オプション登録時に指定されており、これはオプション定義の変更が排他的であることを示している。
