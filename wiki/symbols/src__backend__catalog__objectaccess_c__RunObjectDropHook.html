RunObjectDropHook

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaccess.c:53

目的

データベースオブジェクトの削除操作が発生した際、登録されている外部フック（object_access_hook）を呼び出し、セキュリティ拡張や監査ログプラグイン等にイベントを通知する。

シグネチャまたは形状

void RunObjectDropHook(Oid classId, Oid objectId, int subId, int dropflags)

入力と出力

  • **入力**:
  • classId: 削除対象オブジェクトのカタログOID。
  • objectId: 削除対象オブジェクトのOID。
  • subId: サブオブジェクトID（列番号など）。
  • dropflags: 削除操作に関するフラグ（dropflags）。
  • **出力**: なし（void）。

前提条件と不変条件

  • グローバル変数 object_access_hook が非NULLであること（Assertにより検証）。
  • 呼び出し元は、フックが登録されていることを確認することが推奨される。

アルゴリズムまたは振る舞い

1. ObjectAccessDrop 構造体をスタック上に確保し、memset でゼロ初期化する。 2. 入力された dropflags を drop_arg.dropflags に格納する。 3. グローバル関数ポインタ object_access_hook を介して、イベントタイプ OAT_DROP と共に引数をフック関数へ渡す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/dependency.c 等のオブジェクト削除処理を行うコアコード（ソースコード上の直接的な呼び出し元は src/backend/catalog/objectaccess.c 内のラッパー等）。
  • **呼び出し先**: グローバル変数 object_access_hook に登録された外部プラグインの関数。

関連データ構造

  • ObjectAccessDrop: 削除操作に関連する引数を保持する構造体。
  • OAT_DROP: ObjectAccessType 列挙型に含まれる、削除イベントを示す定数。

失敗モードと境界条件

  • object_access_hook が NULL の状態で呼び出された場合、Assert によりプログラムが停止する（デバッグビルド時）。

保守担当者向けメモ

  • 本関数はフックの実行エントリーポイントであり、実際のロジックはロードされた共有ライブラリ側に依存する。
  • 名前ベースの削除フックが必要な場合は、対となる RunObjectDropHookStr を使用すること。
