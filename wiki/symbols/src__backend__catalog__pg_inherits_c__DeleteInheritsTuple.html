DeleteInheritsTuple

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_inherits.c:551

目的

システムカタログ pg_inherits から、指定された子テーブル（inhrelid）に関連する継承エントリを削除します。

シグネチャまたは形状

bool DeleteInheritsTuple(Oid inhrelid, Oid inhparent, bool expect_detach_pending, const char *childname)

入力と出力

  • **入力**:
  • inhrelid: 削除対象の継承関係を持つ子テーブルの OID。
  • inhparent: 削除対象の親テーブルの OID（InvalidOid を指定すると、該当する子テーブルの全親エントリが対象となる）。
  • expect_detach_pending: パーティションのデタッチ保留状態に関する期待値。
  • childname: エラーメッセージ表示用のテーブル名。
  • **出力**:
  • bool: 少なくとも1つのエントリが削除された場合に true を返す。

前提条件と不変条件

  • 呼び出し元は、対象のリレーションに対して適切なロック（RowExclusiveLock 等）を保持している必要があります。
  • pg_inherits カタログに対するスキャンは InheritsRelidSeqnoIndexId を使用して行われます。

アルゴリズムまたは振る舞い

1. pg_inherits カタログを RowExclusiveLock でオープンします。 2. inhrelid をキーとしてインデックススキャンを開始します。 3. 該当する各タプルについて、inhparent が一致するか（または inhparent が InvalidOid か）を確認します。 4. 一致した場合、タプルの inhdetachpending フラグと expect_detach_pending 引数を比較し、状態が不整合であればエラーを発生させます。 5. 整合性が確認されたタプルを CatalogTupleDelete で削除します。 6. スキャンを終了し、カタログをクローズして結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/index.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • table_open, table_close
  • systable_beginscan, systable_getnext, systable_endscan
  • CatalogTupleDelete
  • ereport (エラー発生時)

関連データ構造

  • pg_inherits: 継承関係を保持するシステムカタログ。
  • Form_pg_inherits: pg_inherits のタプル構造体（inhparent, inhdetachpending フィールドを参照）。

失敗モードと境界条件

  • **デタッチ状態の不整合**:
  • inhdetachpending が真で expect_detach_pending が偽の場合、またはその逆の場合に ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE エラーが発生します。
  • **該当なし**: 指定された inhrelid や inhparent に一致するエントリがない場合、何も削除されず false が返されます。

保守担当者向けメモ

  • 本関数はパーティションのデタッチ処理と密接に関連しています。expect_detach_pending の扱いは、パーティションの整合性を維持するための重要なガードレールです。
  • 呼び出し元で適切なロックが確保されていない場合、並行処理による競合が発生する可能性があるため注意してください。
