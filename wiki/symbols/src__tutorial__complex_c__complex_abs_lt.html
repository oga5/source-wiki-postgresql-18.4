complex_abs_lt

定義

  • 種別: function
  • 定義箇所: src/tutorial/complex.c:147

目的

2つの Complex 型の値を絶対値に基づいて比較し、第1引数が第2引数より小さいかどうかを判定する。

シグネチャまたは形状

Datum complex_abs_lt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される2つの Complex 構造体へのポインタ。
  • **出力**: Datum 型として返されるブール値（true または false）。

前提条件と不変条件

  • 引数は PG_GETARG_POINTER を介して取得されるため、有効な Complex 構造体を指している必要がある。
  • 比較ロジックは complex_abs_cmp_internal に依存しており、この関数が返す比較結果（負、ゼロ、正）に基づき判定を行う。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) および PG_GETARG_POINTER(1) を使用して、比較対象の2つの複素数 a と b を取得する。 2. complex_abs_cmp_internal(a, b) を呼び出し、絶対値の比較結果を取得する。 3. 戻り値が 0 未満（a の絶対値が b より小さい）であれば true を、そうでなければ false を PG_RETURN_BOOL で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（PG_FUNCTION_INFO_V1(complex_abs_lt) マクロにより登録）。
  • **呼び出し先**: complex_abs_cmp_internal。

関連データ構造

  • Complex: 複素数を表す構造体（double x, y）。

失敗モードと境界条件

  • 引数が NULL の場合の挙動については、本関数の実装内では明示的なチェックを行っていない。PostgreSQLの呼び出し規約に従い、呼び出し元で適切に処理されることが前提となる。

保守担当者向けメモ

  • 本関数はB-treeインデックスの演算子クラスの一部として使用されることを想定している。
  • 比較ロジックの整合性を保つため、比較演算の変更が必要な場合は complex_abs_cmp_internal を修正すること。
