PgAioOp

定義

  • 種別: enum
  • 定義箇所: src/include/storage/aio.h:87

目的

PostgreSQL の非同期 I/O (AIO) サブシステムにおいて、実行される I/O 操作の種類を識別するために使用されます。

シグネチャまたは形状

typedef enum PgAioOp
{
    PGAIO_OP_INVALID = 0,
    PGAIO_OP_READV,
    PGAIO_OP_WRITEV,
} PgAioOp;

入力と出力

  • 入力: PgAioHandle を介して操作の種類として設定されます。
  • 出力: pgaio_io_get_op() 関数により、ハンドルに関連付けられた現在の操作種別を取得可能です。

前提条件と不変条件

  • PGAIO_OP_INVALID は値 0 として定義されており、メモリがゼロ初期化された状態を検知するために使用されます。

アルゴリズムまたは振る舞い

  • pgaio_io_stage() を通じて操作をステージングする際に指定されます。
  • 各種 I/O 実装（io_uring 等）やバックエンドの処理ロジックにおいて、switch 文を用いて操作種別に応じた分岐処理が行われます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/storage/aio/aio.c: pgaio_io_stage()
  • src/backend/storage/aio/aio_funcs.c: switch 文による判定
  • src/backend/storage/aio/aio_io.c: 各種 switch 文による判定
  • src/backend/storage/aio/method_io_uring.c: switch 文による判定
  • 呼び出し先:
  • 本シンボル自体は列挙型であり、関数呼び出しは行いません。

関連データ構造

  • PgAioHandle: I/O 操作のコンテキストを保持する構造体。
  • PgAioOpData: PgAioOp に関連するサポートデータ構造。

失敗モードと境界条件

  • 未定義の操作種別が渡された場合、各実装の switch 文においてデフォルトケース（またはエラーハンドリング）に到達する可能性があります。

保守担当者向けメモ

  • 将来的に fsync, fdatasync, flush_range およびネットワーク関連（send, recv, accept）の操作が追加される予定です。
