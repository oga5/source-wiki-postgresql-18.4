MdPathStr

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/smgr/md.c:128

目的

リレーションの物理ファイルパスを保持するための固定長文字列バッファを提供します。

シグネチャまたは形状

typedef struct MdPathStr
{
	char		str[MD_PATH_STR_MAXLEN + 1];
} MdPathStr;

入力と出力

  • 入力: MD_PATH_STR_MAXLEN に基づく固定長の文字配列。
  • 出力: 構造体として、リレーションのパス文字列を格納した状態で返されます。

前提条件と不変条件

  • str メンバは MD_PATH_STR_MAXLEN + 1 バイトのサイズを持ち、ヌル終端文字列を格納するために十分な領域を確保しています。

アルゴリズムまたは振る舞い

  • 本構造体は、リレーションのパス生成や操作において、スタック領域等でパス文字列を安全に保持・受け渡しするために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • _mdfd_segpath (src/backend/storage/smgr/md.c:147)
  • mdunlink (src/backend/storage/smgr/md.c:1680)
  • mdexists (src/backend/storage/smgr/md.c:431)
  • 呼び出し先: 本構造体はデータ保持用であり、関数呼び出しを行いません。

関連データ構造

  • MdfdVec: 個々のセグメントファイル記述子を管理する構造体。本構造体で生成されたパスがファイルオープン時に利用されます。

失敗モードと境界条件

  • MD_PATH_STR_MAXLEN を超えるパスを格納しようとした場合、バッファオーバーフローのリスクがあるため、使用時にはパスの長さがこの定数内に収まることが前提となります。

保守担当者向けメモ

  • 本構造体は md.c 内のローカルなパス操作の利便性のために定義されています。パスの最大長を変更する場合は MD_PATH_STR_MAXLEN の定義を確認してください。
