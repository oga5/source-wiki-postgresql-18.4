Node

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/nodes.h:134

目的

PostgreSQLのクエリ処理におけるすべてのノード構造体の基盤となる共通ヘッダーを提供します。システム全体でノードの型識別、メモリレイアウトの統一、および動的な型キャストを可能にします。

シグネチャまたは形状

typedef struct Node
{
	NodeTag		type;
} Node;

入力と出力

  • **入力**: ノード生成時（makeNode マクロ等）に、適切な NodeTag が割り当てられます。
  • **出力**: 任意のノード構造体へのポインタを Node* にキャストすることで、共通の type フィールドを参照可能です。

前提条件と不変条件

  • すべてのノード構造体は、必ず Node 型のフィールドを構造体の先頭に配置しなければなりません。
  • NodeTag の順序はABIに影響するため、リリース済みのブランチでは変更不可です。

アルゴリズムまたは振る舞い

  • **型識別**: 先頭の type フィールドにより、実行時に IsA マクロ等を用いてノードの種類を判定します。
  • **メモリ管理**: makeNode マクロを通じて動的に割り当てられ、NodeTag が初期化されます。
  • **型キャスト**: castNode マクロにより、アサーション付きの安全な型キャストが提供されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/file_fdw/file_fdw.c や contrib/hstore/hstore_io.c など、クエリ処理やデータ操作を行う多数のモジュールから参照されています。
  • **呼び出し先**: NodeTag を定義する nodes/nodetags.h に依存しています。

関連データ構造

  • NodeTag: ノードの種類を識別する列挙型。

失敗モードと境界条件

  • Node 構造体の先頭に NodeTag が配置されていない場合、IsA や castNode による型判定・キャストが不正なメモリ領域を参照し、クラッシュや未定義動作を引き起こします。

保守担当者向けメモ

  • 新しいノードを追加する際は、必ず Node を先頭に含め、pg_node_attr マクロを使用してコピーや比較の挙動を定義してください。
  • NodeTag の追加・変更時は、ABI互換性に十分注意してください。
