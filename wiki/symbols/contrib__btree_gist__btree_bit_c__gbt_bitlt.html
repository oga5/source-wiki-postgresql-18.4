gbt_bitlt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bit.c:57

目的

BIT 型および VARBIT 型のデータに対して「より小さい（less than）」比較を行い、その結果をブール値で返します。

シグネチャまたは形状

bool gbt_bitlt(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象のビット列データへのポインタ。
  • collation: 照合順序（本関数では使用されません）。
  • flinfo: 関数呼び出し情報（本関数では使用されません）。
  • **出力**:
  • a が b より小さい場合に true、それ以外の場合に false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、PostgreSQL の BIT/VARBIT 型として有効なデータ構造を指している必要があります。
  • DirectFunctionCall2 を介して標準の bitlt 関数を呼び出すため、標準のビット列比較規則に従います。

アルゴリズムまたは振る舞い

1. a と b を PointerGetDatum を用いて Datum 型に変換します。 2. PostgreSQL の標準関数 bitlt を DirectFunctionCall2 で呼び出し、比較を実行します。 3. 比較結果を DatumGetBool でブール値に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_bit.c 内の tinfo 構造体（gbtree_vinfo）の初期化時に参照されています（117行目）。
  • **呼び出し先**: PostgreSQL のコア関数 bitlt を呼び出します。

関連データ構造

  • gbtree_vinfo: btree_gist がインデックス操作に使用する設定テーブル。本関数はこのテーブルの一部として登録されます。

失敗モードと境界条件

  • a または b が無効なポインタである場合、メモリ保護違反が発生する可能性があります。
  • bitlt 関数が内部的にエラーを発生させた場合、その例外が伝播します。

保守担当者向けメモ

  • 本関数は bitlt の単なるラッパーです。比較ロジックの変更が必要な場合は、コア側の bitlt 実装を確認してください。
  • collation および flinfo 引数は現在の実装では使用されていませんが、btree_gist のインターフェース仕様に合わせて保持されています。
