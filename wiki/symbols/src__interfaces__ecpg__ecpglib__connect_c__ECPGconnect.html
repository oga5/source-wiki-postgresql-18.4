ECPGconnect

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/ecpg/ecpglib/connect.c:259

目的

データベース接続を初期化し、ECPGの接続管理リストに追加する。

シグネチャまたは形状

bool ECPGconnect(int lineno, int c, const char *name, const char *user, const char *passwd, const char *connection_name, int autocommit)

入力と出力

  • **入力**:
  • lineno: 呼び出し元の行番号。
  • c: 互換モード（COMPAT_MODE）。
  • name: 接続先データベース名（接続文字列）。
  • user, passwd: 認証情報。
  • connection_name: 接続を識別する名前。
  • autocommit: オートコミット設定。
  • **出力**: 成功時に true、失敗時に false を返す。

前提条件と不変条件

  • sqlca 構造体が取得可能であること。
  • connection_name が既にリストに存在しないこと（一意性の保証）。
  • INFORMIX_MODE が有効な場合、環境変数 PG_DBPATH が name を上書きする。

アルゴリズムまたは振る舞い

1. sqlca の初期化と ecpg_clear_auto_mem() によるメモリ状態のクリアを行う。 2. ecpg_pthreads_init() を呼び出し、スレッド環境を初期化する。 3. connection_name の重複チェックを行い、既存であればエラーを返す。 4. connection 構造体用のメモリを確保する。 5. 接続文字列（name）を解析する。

  • tcp: または unix: で始まる新形式、あるいは dbname[@server][:port] の旧形式を判別し、ホスト、ポート、データベース名を抽出する。
  • unix: 接続の場合、localhost または 127.0.0.1 以外へのアクセスを拒否する。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPGで生成されたCコード（テストスイート等で確認）。
  • **呼び出し先**: ECPGget_sqlca, ecpg_init_sqlca, ecpg_clear_auto_mem, ecpg_pthreads_init, ecpg_get_connection, ecpg_alloc, ecpg_free, ecpg_log, ecpg_raise, last_dir_separator 等。

関連データ構造

  • struct connection: 接続状態を保持する構造体。
  • struct sqlca_t: SQL通信領域。

失敗モードと境界条件

  • メモリ不足（ECPG_OUT_OF_MEMORY）: sqlca 取得失敗時や connection 構造体確保失敗時に発生。
  • 接続名重複: 既に存在する識別子を指定した場合に失敗。
  • 不正なソケットアクセス: unix: 形式で localhost 以外を指定した場合にエラーを発生。

保守担当者向けメモ

  • 接続文字列の解析ロジックは、新旧の形式（postgresql:// 形式と dbname@host 形式）が混在しているため、修正時は注意が必要。
  • INFORMIX_MODE などの互換性フラグによる挙動の変化を考慮すること。
