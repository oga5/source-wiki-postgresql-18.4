tblspc_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/tblspcdesc.c:40

目的

WAL（Write-Ahead Log）レコードの info フィールドから、テーブルスペース操作の種類（CREATE または DROP）を識別し、対応する文字列を返します。

シグネチャまたは形状

const char *tblspc_identify(uint8 info)

入力と出力

  • **入力**: info (uint8) - WALレコードの info ビットフィールド。
  • **出力**: const char * - 操作の種類を示す文字列（"CREATE" または "DROP"）。該当しない場合は NULL。

前提条件と不変条件

  • 入力された info は、XLR_INFO_MASK を用いて補助フラグが除去された状態で評価されます。

アルゴリズムまたは振る舞い

1. info に対してビット演算 info & ~XLR_INFO_MASK を行い、操作タイプを抽出します。 2. 抽出された値が XLOG_TBLSPC_CREATE であれば "CREATE" を返します。 3. 抽出された値が XLOG_TBLSPC_DROP であれば "DROP" を返します。 4. 上記いずれにも該当しない場合、初期値である NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h において、Tablespace リソースマネージャの識別子として登録されています。
  • **呼び出し先**: ソースコード上での直接的な呼び出し先は記述されていません。

関連データ構造

  • xl_tblspc_create_rec および xl_tblspc_drop_rec（本関数が識別する操作に関連するデータ構造）。

失敗モードと境界条件

  • info が XLOG_TBLSPC_CREATE または XLOG_TBLSPC_DROP のいずれにも一致しない場合、NULL を返します。

保守担当者向けメモ

  • 新しいテーブルスペース操作が追加された場合は、本関数の switch 文に新しいケースを追加する必要があります。
  • XLR_INFO_MASK によるビットマスクは、WALレコードの補助フラグを無視して純粋な操作タイプを特定するために不可欠です。
