drop_assignments

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/descriptor.c:31

目的

グローバルな assignments リストを走査し、保持されているすべての assignment 構造体とその内部の variable メンバーを解放することで、メモリリークを防止します。

シグネチャまたは形状

static void drop_assignments(void)

入力と出力

  • 入力: なし（グローバル変数 assignments を参照）
  • 出力: なし（副作用としてメモリを解放）

前提条件と不変条件

  • assignments リストは、push_assignment によって構築された一時的なスタックとして機能していること。
  • 各 assignment 構造体内の variable メンバーは、動的に割り当てられていること。

アルゴリズムまたは振る舞い

1. assignments リストが空になるまでループを実行します。 2. 現在のリストの先頭（old_head）を保持します。 3. リストのポインタを次の要素へ進めます。 4. old_head->variable を解放します。 5. old_head 自体を解放します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/interfaces/ecpg/preproc/descriptor.c 内のコード生成関数（output_get_descr や output_set_descr 等）の終了処理として呼び出されます。
  • 呼び出し先:
  • 標準ライブラリの free()。

関連データ構造

  • struct assignment: 記述子操作項目を保持する連結リスト構造体。

失敗モードと境界条件

  • assignments が NULL の場合、ループは実行されず何も行われません。
  • リスト内のポインタが不正な場合、free() 呼び出し時に未定義の動作が発生する可能性があります。

保守担当者向けメモ

  • 本関数は、記述子操作のコード生成が完了するたびに必ず呼び出す必要があります。
  • assignments リストは一時的なスタックとして設計されているため、本関数によるクリーンアップを忘れるとメモリリークの原因となります。
