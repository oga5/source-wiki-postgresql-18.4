InvalidationInfo

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/cache/inval.c:230

目的

現在のコマンド実行中に発生したキャッシュ無効化イベントを保持し、リレーションキャッシュの初期化ファイル（relcache init file）の無効化要否を管理するための構造体です。

シグネチャまたは形状

typedef struct InvalidationInfo
{
    InvalidationMsgsGroup CurrentCmdInvalidMsgs;
    bool        RelcacheInitFileInval;
} InvalidationInfo;

入力と出力

本構造体は、キャッシュ無効化イベントを蓄積するためのコンテナとして機能します。

  • **入力**: RegisterCatalogInvalidation 等の関数を介して、特定の無効化イベントが CurrentCmdInvalidMsgs に追加されます。
  • **出力**: トランザクション終了時やコマンド境界において、蓄積されたメッセージが共有メモリの SI メッセージキューへ伝播されます。

前提条件と不変条件

  • CurrentCmdInvalidMsgs は、現在のコマンド内で発生した無効化メッセージを保持します。
  • RelcacheInitFileInval が true の場合、リレーションキャッシュの初期化ファイルを無効化する必要があります。

アルゴリズムまたは振る舞い

1. **イベント蓄積**: CacheInvalidateHeapTuple 等の呼び出しにより、変更内容が CurrentCmdInvalidMsgs に追加されます。 2. **状態管理**: トランザクションやサブトランザクションの文脈で InvalidationInfo が割り当てられ、キャッシュの整合性を維持するためのライフサイクルを管理します。 3. **伝播**: コマンド終了時やトランザクションコミット時に、保持しているメッセージがローカルキャッシュの更新や共有メモリへの書き出しに使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/inval.c 内の各登録関数（RegisterCatalogInvalidation, RegisterRelcacheInvalidation 等）およびトランザクション管理関数が本構造体を操作します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • InvalidationMsgsGroup: 無効化メッセージのグループを保持するメンバ。
  • TransInvalidationInfo: トランザクションごとの無効化状態を管理する上位構造体。

失敗モードと境界条件

  • メモリ不足: CurrentMemoryContext 上に割り当てられるため、メモリ枯渇時に割り当て失敗の可能性があります。
  • 整合性: トランザクションのコミット/アボート処理と正しく同期されない場合、キャッシュの不整合が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は src/backend/utils/cache/inval.c 内で動的に割り当てられ、トランザクションのライフサイクルに依存します。
  • inplaceInvalInfo などの静的変数を通じて、トランザクション外の更新（relmapper 等）にも利用される点に注意してください。
