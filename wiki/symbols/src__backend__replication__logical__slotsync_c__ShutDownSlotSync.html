ShutDownSlotSync

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/slotsync.c:1690

目的

スタンバイサーバーの昇格時などに、実行中の論理レプリケーションスロット同期プロセスを安全に停止させる。

シグネチャまたは形状

void ShutDownSlotSync(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として同期プロセスの停止と共有メモリ状態の更新を行う）

前提条件と不変条件

  • SlotSyncCtx 共有メモリ構造体が初期化されていること。
  • 呼び出し元は、同期プロセスが停止するまで待機する準備ができていること。

アルゴリズムまたは振る舞い

1. SlotSyncCtx->mutex を取得し、stopSignaled フラグを true に設定する。 2. 同期が実行中でない場合、update_synced_slots_inactive_since() を呼び出して終了する。 3. 同期が実行中の場合、対象プロセスの PID を取得し、SendProcSignal を用いて PROCSIG_SLOTSYNC_MESSAGE シグナルを送信する。 4. 同期プロセスが停止するまで、WaitLatch を用いてループ内で待機する。 5. 停止が確認されたら、update_synced_slots_inactive_since() を呼び出して終了する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/xlogrecovery.c:1505 (スタンバイ昇格時のスタートアッププロセス)
  • 呼び出し先:
  • SpinLockAcquire / SpinLockRelease
  • SendProcSignal
  • WaitLatch
  • CHECK_FOR_INTERRUPTS
  • update_synced_slots_inactive_since

関連データ構造

  • SlotSyncCtx: 同期プロセスの状態（PID、停止フラグ、実行中フラグ）を保持する共有メモリ構造体。

失敗モードと境界条件

  • 同期プロセスが既に終了している場合、待機ループに入らず即座に終了する。
  • 待機中にポストマスターが終了した場合、WL_EXIT_ON_PM_DEATH により待機ループから脱出する。

保守担当者向けメモ

  • 本関数は、スタンバイ昇格時の安全な停止を保証するために不可欠です。
  • 待機ループ内での CHECK_FOR_INTERRUPTS() は、停止処理中に割り込みが発生した場合の応答性を確保するために重要です。
