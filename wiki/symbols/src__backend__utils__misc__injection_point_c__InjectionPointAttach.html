InjectionPointAttach

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/injection_point.c:272

目的

共有メモリ上に新しいインジェクションポイントを登録し、指定された共有ライブラリ内の関数を動的に呼び出せるように設定します。

シグネチャまたは形状

void InjectionPointAttach(const char *name,
                          const char *library,
                          const char *function,
                          const void *private_data,
                          int private_data_size)

入力と出力

  • **入力**:
  • name: インジェクションポイントの識別名。
  • library: 呼び出す関数を含む共有ライブラリ名。
  • function: 呼び出す関数名。
  • private_data: 関数に渡す任意のデータ（ポインタ）。
  • private_data_size: private_data のサイズ（バイト単位）。
  • **出力**: なし（成功時は共有メモリを更新、失敗時はエラーをスロー）。

前提条件と不変条件

  • ビルド時に --enable-injection-points が有効である必要があります。
  • 各引数の文字列長は、それぞれ INJ_NAME_MAXLEN、INJ_LIB_MAXLEN、INJ_FUNC_MAXLEN を超えてはなりません。
  • private_data_size は INJ_PRIVATE_MAXLEN を超えてはなりません。
  • 同名のインジェクションポイントが既に登録されている場合はエラーとなります。

アルゴリズムまたは振る舞い

1. 引数の長さおよびデータサイズを検証し、超過している場合はエラーをスローします。 2. InjectionPointLock を排他モードで取得します。 3. 共有メモリ上の ActiveInjectionPoints をスキャンし、空きスロット（世代カウンタが偶数）を探します。同時に、同名のポイントが既に存在しないか確認します。 4. 空きスロットが見つからない場合、現在の使用数（max_inuse）をインクリメントして新しいスロットを確保します（上限 MAX_INJECTION_POINTS に達している場合はエラー）。 5. 指定された情報をスロットに書き込みます。 6. メモリバリアを挿入後、世代カウンタをインクリメント（奇数へ変更）して登録を確定させます。 7. InjectionPointLock を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/injection_points/injection_points.c
  • src/test/modules/test_aio/test_aio.c
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease (共有メモリ保護)
  • pg_atomic_read_u32 / pg_atomic_write_u32 / pg_atomic_read_u64 / pg_atomic_write_u64 (アトミック操作)
  • elog (エラー報告)

関連データ構造

  • InjectionPointEntry: 共有メモリ上の各インジェクションポイントの定義を保持する構造体。
  • InjectionPointsCtl: 全インジェクションポイントを管理する共有メモリ上の制御構造体。

失敗モードと境界条件

  • **エラー**:
  • 引数が定義された最大長を超過している場合。
  • 同名のインジェクションポイントが既に存在する場合。
  • 登録可能な最大数（MAX_INJECTION_POINTS）を超過した場合。
  • ビルド時に機能が無効化されている場合。

保守担当者向けメモ

  • 本関数はデバッグおよびテスト専用です。本番環境での使用は想定されていません。
  • 世代カウンタの偶奇判定によるロックフリーな読み取りをサポートするため、書き込み完了時にカウンタを更新する順序が重要です。
