tarFileType

定義

  • 種別: enum
  • 定義箇所: src/include/pgtar.h:59

目的

tar アーカイブ内の各エントリがどのような種類のファイル（通常ファイル、ディレクトリ、シンボリックリンク等）であるかを識別するためのフラグを定義します。

シグネチャまたは形状

enum tarFileType
{
	TAR_FILETYPE_PLAIN = '0',
	TAR_FILETYPE_PLAIN_OLD = '\0',
	TAR_FILETYPE_SYMLINK = '2',
	TAR_FILETYPE_DIRECTORY = '5',
	TAR_FILETYPE_PAX_EXTENDED = 'x',
	TAR_FILETYPE_PAX_EXTENDED_GLOBAL = 'g',
};

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: 各列挙子は char 型の文字値（またはヌル文字）として表現されます。

前提条件と不変条件

  • TAR_FILETYPE_PLAIN_OLD は POSIX 互換性のために提供されています。
  • 各値は tar ヘッダーの特定のフィールド（ファイルタイプフラグ）に格納されることを想定しています。

アルゴリズムまたは振る舞い

tar アーカイブのヘッダー構築時や解析時に、ファイル種別を識別するための定数として使用されます。各値は標準的な tar 形式の仕様に基づいています。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報には、本シンボルを直接参照している具体的な関数名は記載されていません。
  • 呼び出し先: なし。

関連データ構造

  • src/include/pgtar.h 内で定義される他の tar 関連定数および構造体。

失敗モードと境界条件

  • 未定義の文字コードをファイルタイプとして指定した場合の動作は、本定義のみでは規定されていません。

保守担当者向けメモ

  • 本列挙型は tar 形式の仕様に依存しています。新しいファイルタイプを追加する場合は、POSIX 規格との整合性を確認してください。
  • TAR_FILETYPE_PLAIN_OLD は古い実装との互換性維持が目的であるため、削除や変更には注意が必要です。
