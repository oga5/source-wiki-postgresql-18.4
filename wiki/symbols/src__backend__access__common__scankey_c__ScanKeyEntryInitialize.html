ScanKeyEntryInitialize

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/scankey.c:31

目的

インデックススキャンやテーブルスキャンで使用される ScanKeyData 構造体の各フィールドを初期化し、比較演算子の実行に必要な情報を設定します。

シグネチャまたは形状

void ScanKeyEntryInitialize(ScanKey entry,
                            int flags,
                            AttrNumber attributeNumber,
                            StrategyNumber strategy,
                            Oid subtype,
                            Oid collation,
                            RegProcedure procedure,
                            Datum argument)

入力と出力

  • **入力**:
  • entry: 初期化対象の ScanKey 構造体へのポインタ。
  • flags: スキャン動作を制御するフラグ（SK_SEARCHNULL 等）。
  • attributeNumber: 検索対象の属性番号。
  • strategy: 比較戦略番号。
  • subtype: 比較対象のサブタイプ OID。
  • collation: 照合順序 OID。
  • procedure: 比較関数を特定する RegProcedure (OID)。
  • argument: 比較に使用する値 (Datum)。
  • **出力**: なし（entry が指す構造体が直接更新されます）。

前提条件と不変条件

  • entry は有効なメモリ領域を指している必要があります。
  • FmgrInfo の初期化が行われるため、呼び出し元の CurrentMemoryContext が ScanKey の生存期間中有効である必要があります。
  • procedure が無効な場合、flags に SK_SEARCHNULL または SK_SEARCHNOTNULL が含まれている必要があります。

アルゴリズムまたは振る舞い

1. 引数で渡された flags、attributeNumber、strategy、subtype、collation、argument を entry 構造体の各フィールドに代入します。 2. procedure が有効（RegProcedureIsValid）な場合、fmgr_info を呼び出し、指定された OID に対応する関数情報を entry->sk_func に格納します。 3. procedure が無効な場合、flags に SK_SEARCHNULL または SK_SEARCHNOTNULL が含まれていることをアサートし、entry->sk_func をゼロクリアします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtpreprocesskeys.c
  • src/backend/access/nbtree/nbtsearch.c
  • src/backend/executor/execIndexing.c
  • src/backend/executor/nodeIndexscan.c
  • **呼び出し先**:
  • fmgr_info (比較関数のルックアップ)
  • MemSet (関数情報のクリア)
  • RegProcedureIsValid (マクロ)
  • Assert (マクロ)

関連データ構造

  • ScanKeyData: 初期化対象の構造体。
  • FmgrInfo: sk_func フィールドの型。
  • Datum: 検索値の型。

失敗モードと境界条件

  • procedure が無効かつ SK_SEARCHNULL / SK_SEARCHNOTNULL フラグが設定されていない場合、Assert により停止します。
  • fmgr_info は内部でメモリ割り当てを行う可能性があるため、メモリ不足時にはエラーが発生します。

保守担当者向けメモ

  • 頻繁に呼び出される箇所では、fmgr_info のオーバーヘッドを避けるために ScanKeyEntryInitializeWithInfo の使用を検討してください。
  • ScanKey の生存期間とメモリコンテキストの整合性に注意してください。
