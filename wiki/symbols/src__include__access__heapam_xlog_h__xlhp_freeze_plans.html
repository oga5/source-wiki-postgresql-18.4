xlhp_freeze_plans

定義

  • 種別: struct
  • 定義箇所: src/include/access/heapam_xlog.h:365

目的

VACUUM処理中にヒープページ内のタプルに対して行われる「凍結（Freeze）」操作の計画を、WALレコードの一部として保持するためのデータ構造です。

シグネチャまたは形状

typedef struct xlhp_freeze_plans
{
    uint16      nplans;
    xlhp_freeze_plan plans[FLEXIBLE_ARRAY_MEMBER];
} xlhp_freeze_plans;

入力と出力

  • 入力: nplans（凍結計画の数）、および可変長配列 plans（各タプルの凍結詳細）。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • plans は FLEXIBLE_ARRAY_MEMBER を使用しており、メモリレイアウト上は nplans の直後に連続して配置されます。
  • 本構造体は xl_heap_prune WALレコードのサブレコードとして使用されることが想定されています。

アルゴリズムまたは振る舞い

  • WALリカバリ時、heapdesc.c 等でWALレコードをデコードする際、この構造体を用いて cursor を進め、各凍結計画を順次読み取ります。
  • offsetof(xlhp_freeze_plans, plans) を使用して、構造体の先頭から配列開始位置までのオフセットを計算し、メモリ上のデータにアクセスします。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/access/heap/pruneheap.c: 凍結計画の構築およびWALへの書き込み時に使用。
  • src/backend/access/rmgrdesc/heapdesc.c: WALレコードのデコードおよび内容表示時に使用。
  • 呼び出し先:
  • xlhp_freeze_plan: 本構造体の配列要素として利用。

関連データ構造

  • xlhp_freeze_plan: 個別の凍結計画を定義する構造体。
  • xl_heap_prune: 本構造体を含む親WALレコード構造体。

失敗モードと境界条件

  • nplans が 0 の場合、plans 配列には要素が含まれません。
  • 可変長配列のため、メモリコピーやポインタ演算を行う際は、必ず offsetof を用いた適切なオフセット計算が必要です。

保守担当者向けメモ

  • 本構造体は xl_heap_prune の一部として可変長で続くため、解析時には heap_xlog_deserialize_prune_and_freeze の実装を参照し、データがどのようにシリアライズ/デシリアライズされているかを確認してください。
