index_compute_xid_horizon_for_tuples

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/genam.c:294

目的

削除対象としてマークされたインデックスエントリ群に基づき、テーブルアクセスメソッド（Table AM）を介して、トランザクションの XID ホライゾン（snapshotConflictHorizon）を計算する。

シグネチャまたは形状

TransactionId index_compute_xid_horizon_for_tuples(Relation irel,
                                                   Relation hrel,
                                                   Buffer ibuf,
                                                   OffsetNumber *itemnos,
                                                   int nitems)

入力と出力

  • **入力**:
  • irel: インデックスリレーション。
  • hrel: ヒープリレーション。
  • ibuf: インデックスページを含むバッファ。
  • itemnos: 削除対象のインデックスエントリを示すオフセット番号の配列。
  • nitems: itemnos 配列の要素数。
  • **出力**:
  • 戻り値: 計算された TransactionId（XID ホライゾン）。

前提条件と不変条件

  • nitems > 0 であること。
  • 指定された itemnos に対応するインデックスエントリは、すべて ItemIdIsDead（LP_DEAD）としてマークされている必要がある。

アルゴリズムまたは振る舞い

1. TM_IndexDeleteOp 構造体を初期化し、削除対象のインデックス情報を格納する領域を確保する。 2. 指定された itemnos をループで走査し、各インデックスエントリから対応する ItemPointer（TID）を抽出して delstate.deltids に格納する。 3. table_index_delete_tuples(hrel, &delstate) を呼び出し、テーブル AM に削除対象の TID を渡して、スナップショットの競合を回避するための XID ホライゾンを算出する。 4. 処理完了後、確保したメモリ領域を解放し、算出された snapshotConflictHorizon を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gist.c:1705
  • src/backend/access/hash/hashinsert.c:397
  • **呼び出し先**:
  • table_index_delete_tuples（テーブル AM インターフェース）

関連データ構造

  • TM_IndexDeleteOp: インデックス削除操作の状態を保持する構造体。
  • TM_IndexDelete: 削除対象の TID を保持する構造体。
  • TM_IndexStatus: 各インデックスエントリの削除ステータスを保持する構造体。

失敗モードと境界条件

  • nitems が 0 の場合、Assert により停止する。
  • itemnos で指定されたエントリが LP_DEAD でない場合、Assert により停止する。

保守担当者向けメモ

  • 本関数は MVCC の整合性を保つための重要な計算を行う。table_index_delete_tuples の戻り値が InvalidTransactionId を返す可能性がある点に留意すること。
  • メモリ割り当て（palloc）を行っているため、呼び出し側でのメモリリークがないことを確認すること。
