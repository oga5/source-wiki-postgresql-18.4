tarRead

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_tar.c:510

目的

TARアーカイブ内の特定のメンバー（TAR_MEMBER）からデータを読み出し、読み出したバイト数を返します。

シグネチャまたは形状

size_t tarRead(void *buf, size_t len, TAR_MEMBER *th)

入力と出力

  • **入力**:
  • buf: 読み込んだデータを格納するバッファ。
  • len: 読み込みを要求するバイト数。
  • th: 対象となるTARメンバーのコンテキスト情報。
  • **出力**:
  • 実際に読み込まれたバイト数（size_t）。

前提条件と不変条件

  • th->pos は現在の読み込み位置を保持し、読み込みごとに更新されます。
  • 読み込み要求サイズ len は、メンバーの残りサイズ（th->fileLen - th->pos）を超えないよう調整されます。

アルゴリズムまたは振る舞い

1. 要求された len がメンバーの残りデータ量を超える場合、読み込み上限を残りデータ量に制限します。 2. 読み込みサイズが 0 以下であれば、直ちに 0 を返します。 3. _tarReadRaw を呼び出し、アーカイブからデータを抽出します。 4. 抽出されたバイト数分だけ th->pos を進めます。 5. 最終的に読み込んだバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_tar.c 内の各所（データ読み込みループやヘッダー解析処理など）。
  • **呼び出し先**: _tarReadRaw（内部関数）。

関連データ構造

  • TAR_MEMBER: TARアーカイブ内のメンバー情報を保持する構造体。

失敗モードと境界条件

  • 読み込み要求サイズが 0 の場合、処理を行わず 0 を返します。
  • メンバーの終端に達している場合、len の調整により 0 が返されます。

保守担当者向けメモ

  • 本関数は _tarReadRaw を介して低レベルの読み込みを行います。アーカイブのシークや位置管理は th->pos に依存しているため、この値を直接操作する際は注意してください。
