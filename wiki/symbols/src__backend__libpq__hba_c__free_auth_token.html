free_auth_token

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/hba.c:279

目的

認証トークンに関連付けられた正規表現リソースを解放します。

シグネチャまたは形状

static void free_auth_token(AuthToken *token);

入力と出力

  • **入力**: AuthToken *token - 解放対象の認証トークン構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 token は有効な AuthToken 構造体を指している必要があります。
  • token が正規表現を保持している場合、その正規表現は pg_regcomp 等で初期化されている必要があります。

アルゴリズムまたは振る舞い

1. token_has_regexp(token) を呼び出し、対象トークンが正規表現を保持しているかを確認します。 2. 正規表現を保持している場合、pg_regfree(token->regex) を呼び出して正規表現に関連するメモリリソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/hba.c:280 (内部的な解放処理)
  • src/backend/libpq/hba.c:2922 (expanded_pg_user_token の解放)
  • **呼び出し先**:
  • token_has_regexp
  • pg_regfree

関連データ構造

  • AuthToken: 認証設定ファイル（pg_hba.conf 等）の各トークンを保持する構造体。

失敗モードと境界条件

  • token が NULL の場合の挙動については、本ソース抜粋からは判断できません。
  • token_has_regexp が偽を返す場合、関数は何も行わずに終了します。

保守担当者向けメモ

  • 本関数は正規表現リソースの解放のみを担当します。AuthToken 構造体自体のメモリ解放は呼び出し元で行う必要があります。
