ExecScanExtended

定義

  • 種別: function
  • 定義箇所: src/include/executor/execScan.h:159

目的

スキャンノードにおけるタプル取得、フィルタリング（Qual）、および射影（Projection）の定型的なループ処理を抽象化し、実行フローを最適化する。

シグネチャまたは形状

static pg_attribute_always_inline TupleTableSlot *
ExecScanExtended(ScanState *node,
                 ExecScanAccessMtd accessMtd,
                 ExecScanRecheckMtd recheckMtd,
                 EPQState *epqstate,
                 ExprState *qual,
                 ProjectionInfo *projInfo)

入力と出力

  • **入力**:
  • node: スキャンノードの状態管理構造体。
  • accessMtd: タプルを取得するための関数ポインタ。
  • recheckMtd: タプルを再検証するための関数ポインタ。
  • epqstate: EvalPlanQual 状態（NULL可）。
  • qual: フィルタリング条件式（NULL可）。
  • projInfo: 射影情報（NULL可）。
  • **出力**: 処理後の TupleTableSlot。スキャン終了時は空のスロットを返す。

前提条件と不変条件

  • node->ps.ps_ExprContext が初期化されていること。
  • qual または projInfo が存在しない場合、オーバーヘッドを避けるため ExecScanFetch を直接呼び出す最適化パスが実行される。
  • フィルタリング失敗時には、ResetExprContext によりメモリコンテキストがリセットされる。

アルゴリズムまたは振る舞い

1. qual と projInfo が共に NULL の場合、ExecScanFetch を呼び出し即座に結果を返す。 2. それ以外の場合、以下のループを実行する:

  • ExecScanFetch でタプルを取得する。
  • タプルが NULL（スキャン終了）であれば、projInfo に応じて適切なスロットを返す。
  • タプルを econtext->ecxt_scantuple にセットする。
  • qual が NULL または ExecQual が真を返すまでループを継続する。
  • 条件を満たした場合、projInfo があれば ExecProject を実行し、なければそのままタプルを返す。
  • 条件を満たさない場合、InstrCountFiltered1 を更新し、メモリコンテキストをリセットして次のタプルへ進む。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execScan.c および src/backend/executor/nodeSeqscan.c からの呼び出しが確認されている。
  • **呼び出し先**: ExecScanFetch, ResetExprContext, TupIsNull, ExecClearTuple, ExecQual, ExecProject, InstrCountFiltered1 を呼び出す。

関連データ構造

  • ScanState, EPQState, ExprState, ProjectionInfo, TupleTableSlot, ExprContext

失敗モードと境界条件

  • スキャン対象が空の場合、TupIsNull が真となり、適切に空スロットが返される。
  • qual 評価で失敗したタプルはメモリが解放され、次のタプル取得へ遷移する。

保守担当者向けメモ

  • pg_attribute_always_inline が指定されており、コンパイル時に EPQ 関連の分岐が最適化されるため、通常の実行パスへの影響は最小限である。
  • 割り込みチェックは ExecScanFetch 内で行われるため、本関数内では実施されない。
