pg_b64_dec_len

定義

  • 種別: function
  • 定義箇所: src/common/base64.c:238

目的

Base64エンコードされた文字列をデコードする際に必要となる、出力バッファの最大サイズを算出します。

シグネチャまたは形状

int pg_b64_dec_len(int srclen);

入力と出力

  • **入力 (srclen)**: デコード対象となるBase64文字列の長さ（バイト数）。
  • **出力**: デコード後のバイナリデータが格納されるために必要なバッファサイズ（バイト数）。

前提条件と不変条件

  • 入力 srclen は非負の整数である必要があります。
  • 本関数は計算のみを行い、入力文字列の内容（パディングの有無や不正な文字）は検証しません。

アルゴリズムまたは振る舞い

入力された文字列長 srclen に対して (srclen * 3) >> 2 を計算します。これはBase64の4文字が3バイトのバイナリデータに対応するという特性に基づいた、理論上の最大サイズ算出式です。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth-scram.c (SCRAM認証関連のデータ長計算)
  • src/interfaces/libpq/fe-auth-scram.c (クライアント側のSCRAM認証)
  • src/interfaces/libpq/fe-connect.c (接続時のSCRAMキー長計算)
  • **呼び出し先**: なし（算術演算のみ）。

関連データ構造

  • なし。

失敗モードと境界条件

  • srclen が 0 の場合、戻り値は 0 となります。
  • 本関数自体はエラーを返しませんが、呼び出し側は算出されたサイズをバッファ確保の目安として使用し、実際のデコード処理 (pg_b64_decode) におけるバッファオーバーフローを防ぐ必要があります。

保守担当者向けメモ

  • 本関数は厳密な変換後のバイト数ではなく、デコード結果を格納するために十分な「最大サイズ」を返します。
  • Base64の仕様上、パディング文字 = が含まれる場合でも、この計算式で確保したバッファサイズは安全です。
