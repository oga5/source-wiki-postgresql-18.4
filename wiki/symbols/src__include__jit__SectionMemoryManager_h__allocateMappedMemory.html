allocateMappedMemory

定義

  • 種別: virtual function
  • 定義箇所: src/include/jit/SectionMemoryManager.h:80

目的

オペレーティングシステムから仮想メモリページを要求するための抽象インターフェースです。SectionMemoryManager が JIT コンパイルされたコードやデータを格納する領域を確保する際に使用されます。

シグネチャまたは形状

virtual sys::MemoryBlock allocateMappedMemory(
    AllocationPurpose Purpose, 
    size_t NumBytes,
    const sys::MemoryBlock *const NearBlock,
    unsigned Flags, 
    std::error_code &EC) = 0;

入力と出力

  • **入力**:
  • Purpose: 確保するメモリの用途（Code, ROData, RWData）。
  • NumBytes: 要求するバイト数。
  • NearBlock: 既存のメモリブロックへのポインタ（近接配置のヒント）。
  • Flags: 初期保護フラグ。
  • EC: エラー発生時に詳細を格納する出力用参照。
  • **出力**:
  • 成功時: 確保されたメモリ領域を記述する sys::MemoryBlock。
  • 失敗時: EC にエラー情報が設定された空の MemoryBlock。

前提条件と不変条件

  • 確保されるメモリブロックの開始アドレスは、システムの割り当て粒度（Linux ではページサイズ、Windows では 64K）にアライメントされている必要があります。
  • NearBlock が指定された場合、その直後のアドレスがアライメント境界に合致しないときは、次の境界まで切り上げられます。
  • 戻り値の MemoryBlock が示す実際のサイズは、要求した NumBytes 以上になる場合があります。

アルゴリズムまたは振る舞い

1. 指定された用途とサイズに基づき、OS から仮想メモリを確保します。 2. NearBlock の近傍への配置を試みますが、物理的な近接性は保証されません。 3. 指定された Flags で初期保護状態を設定します。 4. 処理結果を MemoryBlock として返し、エラーが発生した場合は EC を通じて通知します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/jit/llvm/SectionMemoryManager.cpp 内の各メソッド（allocateCodeSection 等の内部実装）。
  • **呼び出し先**: OS のメモリ管理機能（デフォルト実装の場合）。

関連データ構造

  • AllocationPurpose: メモリ確保の目的を定義する列挙型。
  • sys::MemoryBlock: 確保されたメモリの範囲を保持する構造体。
  • MemoryMapper: 本メソッドを定義するインターフェースクラス。

失敗モードと境界条件

  • 要求されたサイズが確保できない場合、またはシステム制限に達した場合にエラーが発生します。
  • NearBlock の指定が不適切な場合でも、アライメント調整が行われた上でメモリ確保が試行されます。

保守担当者向けメモ

  • 本メソッドは MemoryMapper インターフェースの一部であり、SectionMemoryManager のサブクラスやカスタム実装でオーバーライド可能です。
  • finalizeMemory が呼ばれるまで、確保されたメモリは通常 RW 権限で保持されます。
  • releaseMappedMemory を使用して、本メソッドで確保したメモリのみを解放してください。
