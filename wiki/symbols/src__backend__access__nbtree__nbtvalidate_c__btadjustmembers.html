btadjustmembers

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtvalidate.c:287

目的

B-tree 演算子族（opfamily）に追加される演算子およびサポート関数に対し、適切な依存関係（演算子クラスへのハード依存、または演算子族へのソフト依存）を割り当てる。

シグネチャまたは形状

void btadjustmembers(Oid opfamilyoid, Oid opclassoid, List *operators, List *functions)

入力と出力

  • **入力**:
  • opfamilyoid: 対象の演算子族の OID。
  • opclassoid: 関連付けの起点となる演算子クラスの OID（指定がない場合は InvalidOid）。
  • operators: 演算子のリスト。
  • functions: サポート関数のリスト。
  • **出力**: なし（引数として渡された OpFamilyMember 構造体の ref_is_hard, ref_is_family, refobjid フィールドを直接更新する）。

前提条件と不変条件

  • opclassoid が有効な場合、CommandCounterIncrement() が実行され、システムカタログの最新状態が参照可能であること。
  • 演算子や関数が「必須サポート関数（BTORDER_PROC）」または「クロス型演算子」である場合、常に演算子族へのソフト依存となる。

アルゴリズムまたは振る舞い

1. operators と functions を結合し、各メンバーに対して以下の判定を行う。 2. **オプションのサポート関数**（BTORDER_PROC 以外）または **クロス型演算子**（lefttype != righttype）の場合:

  • 演算子族へのソフト依存（ref_is_hard = false, ref_is_family = true）を設定。

    • 3. **同一型演算子**の場合:

  • 適切な演算子クラスが存在するかを確認し、存在すればその演算子クラスへのハード依存（ref_is_hard = true, ref_is_family = false）を設定。
  • 適切な演算子クラスが見つからない場合は、演算子族へのソフト依存にフォールバックする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtree.c: amroutine->amadjustmembers フィールドに登録されている。
  • **呼び出し先**:
  • get_opclass_input_type: 演算子クラスの入力型を取得。
  • opclass_for_family_datatype: 演算子族内の特定のデータ型に対応する演算子クラスを検索。

関連データ構造

  • OpFamilyMember: 演算子族の各メンバー情報を保持する構造体。
  • List, ListCell: PostgreSQL のリスト構造。

失敗モードと境界条件

  • 適切な演算子クラスが存在しない場合、エラーを発生させるのではなく、演算子族へのソフト依存として処理を継続する。
  • 演算子クラスの検索コストを抑えるため、直近の opcintype をキャッシュして再利用する。

保守担当者向けメモ

  • 本関数はダンプ/リストア時の依存関係の決定に影響を与える。クロス型演算子が誤って演算子クラスに強くバインドされている場合、本関数によって演算子族への依存に修正される可能性がある。
