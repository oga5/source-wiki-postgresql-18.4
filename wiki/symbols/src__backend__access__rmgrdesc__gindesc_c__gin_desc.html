gin_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/gindesc.c:71

目的

GIN（Generalized Inverted Index）インデックスに関連するWALレコードの内容を人間が読める形式に変換し、StringInfo バッファへ出力します。

シグネチャまたは形状

void gin_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 変換後の文字列を格納する StringInfo バッファ。
  • record: 解析対象のWALレコード状態を保持する XLogReaderState 構造体。
  • **出力**: なし（buf を通じて文字列が書き出される）。

前提条件と不変条件

  • record は有効なWALレコードを指している必要があります。
  • XLogRecGetInfo で取得したレコードタイプに基づき、適切なデータ構造へのキャストが行われます。

アルゴリズムまたは振る舞い

1. XLogRecGetInfo からレコードの操作タイプ（info）を抽出します。 2. switch 文により操作タイプごとに分岐します。

  • XLOG_GIN_INSERT: ginxlogInsert 構造体からフラグを読み取り、リーフノードか否か、データか否かに応じて詳細（子ブロック番号、削除フラグ、ポスティングアイテム等）を buf に追加します。
  • XLOG_GIN_SPLIT: ginxlogSplit 構造体から分割フラグを読み取り出力します。
  • XLOG_GIN_DELETE_LISTPAGE: 削除されたページ数を出力します。
  • XLOG_GIN_VACUUM_DATA_LEAF_PAGE: desc_recompress_leaf を呼び出し、再圧縮データを解析します。

    • 3. フルページイメージが含まれる場合は、その旨を buf に追記します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h で定義される RmgrData 構造体を通じて、PostgreSQLのリカバリプロセスや pg_waldump から呼び出されます。
  • **呼び出し先**:
  • desc_recompress_leaf（GINリーフページの再圧縮解析用）
  • appendStringInfo 等の文字列操作ユーティリティ

関連データ構造

  • XLogReaderState: WALレコードの読み取り状態。
  • StringInfo: 文字列構築用バッファ。
  • ginxlogInsert, ginxlogSplit, ginxlogVacuumDataLeafPage: GINのWALレコード構造体。

失敗モードと境界条件

  • XLogRecGetBlockData が取得できない場合や、レコードのバイナリレイアウトが期待と異なる場合、ポインタ演算により不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本関数は読み取り専用のデコード処理です。
  • desc_recompress_leaf 内のポインタ演算は、ginxlogRecompressDataLeaf 構造体と密結合しているため、構造体定義の変更時には必ず本関数の解析ロジックも確認してください。
