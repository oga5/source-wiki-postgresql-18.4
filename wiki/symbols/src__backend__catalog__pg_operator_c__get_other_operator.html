get_other_operator

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_operator.c:621

目的

演算子の定義時、関連する他の演算子（交換子や否定子など）の OID を取得または生成します。対象の演算子が未定義の場合は「シェル（プレースホルダー）」を作成し、カタログ上の整合性を確保します。

シグネチャまたは形状

Oid get_other_operator(List *otherOp, Oid otherLeftTypeId, Oid otherRightTypeId,
                       const char *operatorName, Oid operatorNamespace,
                       Oid leftTypeId, Oid rightTypeId)

入力と出力

  • **入力**:
  • otherOp: 検索対象の演算子名（リスト形式）。
  • otherLeftTypeId, otherRightTypeId: 検索対象の演算子の引数型。
  • operatorName, operatorNamespace: 現在作成中の演算子の名前と名前空間。
  • leftTypeId, rightTypeId: 現在作成中の演算子の引数型。
  • **出力**:
  • 成功時: 既存または新規作成された演算子の Oid。
  • 自己参照時: InvalidOid。

前提条件と不変条件

  • OperatorLookup を使用してカタログを検索し、存在する場合はその OID を返します。
  • 検索対象の演算子が現在作成中の演算子自身と一致する場合、シェルは作成されず InvalidOid が返されます。

アルゴリズムまたは振る舞い

1. OperatorLookup を呼び出し、指定された名前と型を持つ演算子が既に存在するか確認します。存在すればその OID を返します。 2. 存在しない場合、QualifiedNameGetCreationNamespace で対象の演算子を作成すべき名前空間を特定します。 3. 検索対象の演算子が作成中の演算子自身（名前、名前空間、型がすべて一致）である場合、InvalidOid を返します。 4. それ以外の場合、対象の名前空間に対する ACL_CREATE 権限を object_aclcheck で検証します。 5. 権限が確認できれば OperatorShellMake を呼び出し、仮の OID を持つシェル演算子を作成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_operator.c 内の OperatorCreate（交換子や否定子の解決時に呼び出し）。
  • **呼び出し先**:
  • OperatorLookup
  • QualifiedNameGetCreationNamespace
  • object_aclcheck
  • OperatorShellMake

関連データ構造

  • List: 演算子名を保持するリスト。
  • Oid: データベースオブジェクト識別子。

失敗モードと境界条件

  • **権限エラー**: 対象の名前空間に対して CREATE 権限がない場合、aclcheck_error によりエラーが発生します。
  • **自己参照**: 演算子が自分自身を交換子や否定子として指定した場合、InvalidOid が返されます。呼び出し元はこの戻り値を適切に処理する必要があります。

保守担当者向けメモ

  • 本関数は、相互参照が必要な演算子を先行作成するための重要なメカニズムです。
  • OperatorShellMake で作成されたシェルは、後に実体化されることが期待されています。
  • 呼び出し元が InvalidOid を受け取った際の挙動については、OperatorCreate の実装ロジックを確認してください。
