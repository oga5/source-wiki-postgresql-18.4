PartitionListValue

定義

  • **種別**: struct
  • **定義箇所**: src/backend/partitioning/partbounds.c:57

目的

リストパーティションにおいて、個々の値とそれが属するパーティションのインデックスを関連付けるためのデータ構造です。

シグネチャまたは形状

typedef struct PartitionListValue
{
    int         index;  /* パーティションのインデックス */
    Datum       value;  /* パーティションキーの値 */
} PartitionListValue;

入力と出力

本構造体は、リストパーティションの境界定義を構築する際に、入力された値と対応するパーティション番号を保持するために使用されます。

前提条件と不変条件

  • index は、対応するパーティションを識別するための非負の整数です。
  • value は、Datum 型として保持されるパーティションキーの値です。

アルゴリズムまたは振る舞い

  • リストパーティションの境界構築時、複数の PartitionListValue を含む配列が生成されます。
  • この配列は、qsort_arg を用いて value を基準にソートされ、効率的な検索や重複チェックに利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/partitioning/partbounds.c 内のリストパーティション境界生成ロジック（create_list_bounds 等）からメモリ確保およびソート処理のために参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • PartitionBoundInfoData: リストパーティションを含む、パーティション境界情報を管理する上位構造体。

失敗モードと境界条件

  • メモリ不足時: palloc による配列確保時に失敗する可能性があります。
  • ソート時: qsort_arg 呼び出しにおいて、比較関数が value を正しく処理できない場合、予期せぬ順序やエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は partbounds.c 内でローカルに利用されることが主ですが、リストパーティションの正規化プロセスにおいて中心的な役割を果たします。
  • value の比較ロジックを変更する場合は、qsort_arg に渡される比較関数（src/backend/partitioning/partbounds.c:3795 付近）の整合性を確認してください。
