BufferAccessStrategyType

定義

  • 種別: enum
  • 定義箇所: src/include/storage/bufmgr.h:34

目的

バッファマネージャにおけるページ置換アルゴリズムの最適化戦略を識別するために使用されます。スキャンや書き込み操作の特性に応じて適切な戦略を選択することで、キャッシュ効率を向上させます。

シグネチャまたは形状

typedef enum BufferAccessStrategyType
{
    BAS_NORMAL,    /* 通常のランダムアクセス */
    BAS_BULKREAD,  /* 大規模な読み取り専用スキャン */
    BAS_BULKWRITE, /* 大規模なマルチブロック書き込み（例: COPY IN） */
    BAS_VACUUM     /* VACUUM処理 */
} BufferAccessStrategyType;

入力と出力

  • 入力: GetAccessStrategy および GetAccessStrategyWithSize 関数へ引数として渡されます。
  • 出力: 指定された戦略に基づき、バッファ管理用の BufferAccessStrategy オブジェクトが生成・返却されます。

前提条件と不変条件

  • 新しい BufferAccessStrategyType を追加する場合、対応する IOContext も追加する必要があります。

アルゴリズムまたは振る舞い

  • BAS_NORMAL: 標準的なバッファ置換アルゴリズム（LRUに近い挙動）を適用します。
  • BAS_BULKREAD: 大規模スキャン時にバッファプール全体を汚染しないよう、限定されたバッファリングを行います。
  • BAS_BULKWRITE: 大量書き込み時にバッファを効率的に再利用するための戦略を適用します。
  • BAS_VACUUM: VACUUM処理専用のバッファリング制御を行い、システム全体のパフォーマンス低下を防ぎます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/buffer/freelist.c 内の GetAccessStrategy および GetAccessStrategyWithSize がこの型を引数として受け取ります。
  • 呼び出し先: 本シンボル自体は列挙型であり、直接の呼び出し先はありません。

関連データ構造

  • BufferAccessStrategy: この列挙型によって生成される、具体的な戦略を保持する構造体。

失敗モードと境界条件

  • 未定義の BufferAccessStrategyType が渡された場合、freelist.c 内で elog(ERROR, ...) が発生し、処理が中断されます。

保守担当者向けメモ

  • 新しい戦略を追加する際は、src/include/storage/bufmgr.h のコメントにある通り、IOContext との整合性を確認してください。
  • freelist.c 内の GetAccessStrategy 実装が、この列挙型の全ケースを網羅しているか確認してください。
