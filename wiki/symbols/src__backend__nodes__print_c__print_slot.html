print_slot

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/print.c:495

目的

TupleTableSlot の内容をデバッグ目的で標準出力にダンプする。

シグネチャまたは形状

void print_slot(TupleTableSlot *slot)

入力と出力

  • 入力: TupleTableSlot *slot（表示対象のタプルスロット）
  • 出力: なし（標準出力への書き出しのみ）

前提条件と不変条件

  • slot が NULL であるか、tts_tupleDescriptor が設定されていない場合、エラーメッセージを出力して処理を終了する。

アルゴリズムまたは振る舞い

1. TupIsNull(slot) を用いてスロットが空か判定し、空であれば "tuple is null." と出力して戻る。 2. slot->tts_tupleDescriptor が存在するか確認し、存在しなければ "no tuple descriptor." と出力して戻る。 3. 上記の条件を満たす場合、debugtup(slot, NULL) を呼び出してスロットの内容を表示する。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは特定できません。
  • 呼び出し先: TupIsNull（マクロ）、printf、debugtup。

関連データ構造

  • TupleTableSlot: 表示対象となるタプル保持構造体。

失敗モードと境界条件

  • スロットが NULL の場合: "tuple is null." を出力。
  • タプル記述子（tts_tupleDescriptor）が欠如している場合: "no tuple descriptor." を出力。

保守担当者向けメモ

  • 本関数はデバッグ専用のユーティリティです。
  • debugtup を内部で利用しているため、debugtup の出力仕様に依存します。
