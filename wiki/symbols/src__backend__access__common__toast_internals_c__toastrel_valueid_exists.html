toastrel_valueid_exists

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/common/toast_internals.c:458

目的

指定されたTOASTテーブル内に、特定の valueid（TOASTチャンクの識別子）を持つデータが存在するかを確認する。

シグネチャまたは形状

static bool toastrel_valueid_exists(Relation toastrel, Oid valueid)

入力と出力

  • **入力**:
  • toastrel: 対象のTOASTテーブルのリレーション（Relation）。
  • valueid: 検索対象のチャンク識別子（Oid）。
  • **出力**:
  • 該当するチャンクが1つでも存在すれば true、存在しなければ false を返す。

前提条件と不変条件

  • toastrel は有効なTOASTテーブルのリレーションであること。
  • 呼び出し元は適切なロックを保持している必要がある（内部で RowExclusiveLock を使用）。
  • トランザクション内で呼び出されること（SnapshotAny を使用したスキャンを行うため）。

アルゴリズムまたは振る舞い

1. toast_open_indexes を呼び出し、対象TOASTテーブルのインデックス群を取得する。 2. ScanKeyInit を使用し、第1属性（va_valueid）が指定された valueid と一致する条件を作成する。 3. systable_beginscan を用いて、有効なインデックス経由でテーブルをスキャンする。 4. systable_getnext で最初のレコードを読み取り、結果が存在すれば true をセットする。 5. systable_endscan でスキャンを終了し、toast_close_indexes でインデックスを閉じてリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/toast_internals.c 内の複数箇所（toast_delete_datum 等の内部ロジックから参照されている）。
  • **呼び出し先**:
  • toast_open_indexes
  • ScanKeyInit
  • systable_beginscan
  • systable_getnext
  • systable_endscan
  • toast_close_indexes

関連データ構造

  • Relation: TOASTテーブルおよびインデックスの管理に使用。
  • ScanKeyData: 検索条件の保持に使用。
  • SysScanDesc: システムテーブルスキャンの状態管理に使用。

失敗モードと境界条件

  • TOASTテーブルにインデックスが存在しない場合、toast_open_indexes の挙動に依存する。
  • 該当する valueid が存在しない場合、systable_getnext は NULL を返し、関数は false を正しく返す。

保守担当者向けメモ

  • 本関数は toastrel に対して RowExclusiveLock を取得・解放するため、呼び出し元で既に同等のロックを保持している場合は競合やデッドロックの可能性に注意すること。
  • SnapshotAny を使用しているため、可視性に関わらず物理的に存在するチャンクを検索する。
