consider_new_or_clause

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/orclauses.c:253

目的

抽出されたOR条件のサブ節をベースリレーションの制限条件（baserestrictinfo）として追加し、元の結合条件の選択率を補正することで、プランナによる行数見積もりの歪みを防ぐ。

シグネチャまたは形状

static void consider_new_or_clause(PlannerInfo *root, RelOptInfo *rel, Expr *orclause, RestrictInfo *join_or_rinfo)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel: 条件を追加する対象のベースリレーション。
  • orclause: 追加対象のOR条件式。
  • join_or_rinfo: 元となる結合条件の RestrictInfo。
  • **出力**: なし（副作用として rel および join_or_rinfo を更新）。

前提条件と不変条件

  • orclause は対象リレーションのみを参照する有効な制限条件であると仮定される。
  • 選択率が 0.9 を超える場合、冗長な計算コストを避けるため条件は追加されない。

アルゴリズムまたは振る舞い

1. make_restrictinfo を用いて orclause から RestrictInfo を生成する。 2. clause_selectivity で当該条件の選択率を算出する。 3. 選択率が 0.9 以下であれば、rel->baserestrictinfo に追加し、rel->baserestrict_min_security を更新する。 4. 選択率が 0 より大きい場合、元の結合条件 join_or_rinfo->norm_selec を orig_selec / or_selec で補正し、冗長な条件追加による見積もりの過小評価を相殺する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/orclauses.c 内の関数（詳細はソースコードの文脈上、extract_or_clause 等から呼び出されることが示唆されている）。
  • **呼び出し先**: make_restrictinfo, clause_selectivity, init_dummy_sjinfo, lappend, bms_difference, Min。

関連データ構造

  • PlannerInfo: プランナの全体状態。
  • RelOptInfo: リレーションの最適化情報。
  • RestrictInfo: 制限条件および結合条件のメタデータ。
  • SpecialJoinInfo: 結合の特殊な性質を保持する構造体。

失敗モードと境界条件

  • **選択率の閾値**: 選択率が 0.9 を超える場合、条件は追加されず早期リターンする。
  • **選択率の補正**: 選択率が 0 の場合、補正処理はスキップされる。
  • **ハックの限界**: 外部結合（Outer Join）の場合、norm_selec のみの補正では不十分であり、outer_selec は調整されない。

保守担当者向けメモ

  • 本関数は「Major Hack」と呼ばれる選択率補正ロジックを含みます。キャッシュされた選択率を直接書き換えるため、複雑な結合条件（特に外部結合）では見積もりが非線形になり、完全な精度は保証されません。
  • 補正ロジックは norm_selec（JOIN_INNER セマンティクス）のみを対象としています。
