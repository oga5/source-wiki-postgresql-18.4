sendFileWithContent

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup.c:1074

目的

メモリ上のコンテンツをファイルとして扱い、tar形式のバックアップストリームに挿入する。主に backup_label や tablespace_map などのメタデータファイルをバックアップに含めるために使用される。

シグネチャまたは形状

static void sendFileWithContent(bbsink *sink, const char *filename, const char *content, int len, backup_manifest_info *manifest)

入力と出力

  • **入力**:
  • sink: データ送信先となる bbsink オブジェクト。
  • filename: 作成するファイル名。
  • content: 書き込むデータの内容。
  • len: データの長さ（負の場合は strlen(content) が使用される）。
  • manifest: バックアップマニフェスト情報。
  • **出力**: なし（副作用として sink へのデータ書き込みと manifest へのファイル登録を行う）。

前提条件と不変条件

  • sink は初期化済みであること。
  • manifest は有効なポインタであること。
  • len が負の場合、content はヌル終端文字列である必要がある。

アルゴリズムまたは振る舞い

1. pg_checksum_init を呼び出し、指定されたアルゴリズムでチェックサム計算を初期化する。 2. stat 構造体を構築し、現在のUID/GID、現在時刻、ファイルモードを設定する。 3. _tarWriteHeader を呼び出し、tarヘッダーを sink に書き込む。 4. pg_checksum_update を呼び出し、コンテンツのチェックサムを計算する。 5. sink->bbs_buffer を使用して、コンテンツをチャンク単位で bbsink_archive_contents に渡す。 6. _tarWritePadding を呼び出し、tar形式に必要なパディングを書き込む。 7. AddFileToBackupManifest を呼び出し、作成したファイルをマニフェストに登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c 内の複数箇所（BACKUP_LABEL_FILE や TABLESPACE_MAP の生成時など）。
  • **呼び出し先**:
  • pg_checksum_init, pg_checksum_update
  • _tarWriteHeader, _tarWritePadding
  • bbsink_archive_contents
  • AddFileToBackupManifest

関連データ構造

  • bbsink: バックアップデータの出力先を抽象化した構造体。
  • backup_manifest_info: バックアップマニフェストの生成状態を保持する構造体。
  • pg_checksum_context: チェックサム計算のコンテキスト。

失敗モードと境界条件

  • pg_checksum_init または pg_checksum_update が失敗した場合、elog(ERROR) によりバックアップ処理全体が中断される。
  • len が 0 の場合、空ファイルとして処理される。

保守担当者向けメモ

  • 本関数は bbsink インターフェースを介してデータをストリーミングするため、メモリ消費量は sink->bbs_buffer_length に依存する。
  • Windows環境ではUID/GIDが0に固定される点に注意すること。
