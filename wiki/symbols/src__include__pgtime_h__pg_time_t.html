pg_time_t

定義

  • 種別: typedef
  • 定義箇所: src/include/pgtime.h:23

目的

PostgreSQL 内部で時刻を表現するための 64 ビット符号付き整数型です。プラットフォームごとの time_t の差異を排除し、長期的な時刻計算の整合性と移植性を保証します。

シグネチャまたは形状

typedef int64 pg_time_t;

入力と出力

  • 入力: 通常、time(NULL) 等のシステム時刻や、時刻計算の結果がキャストされて代入されます。
  • 出力: 時刻変換関数（pg_localtime や pg_gmtime）の引数として渡され、構造化された時刻データ（pg_tm）への変換に使用されます。

前提条件と不変条件

  • 64 ビット幅の符号付き整数として定義されており、システム依存の time_t よりも広い範囲の時刻を表現可能です。

アルゴリズムまたは振る舞い

  • データベースエンジン全体で時刻を保持・計算するための標準的な型として機能します。
  • pg_localtime や pg_gmtime を介して、この型から pg_tm 構造体への変換が行われます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/access/transam/xlog.c: チェックポイント時刻やログセグメント切り替え時刻の保持に使用。
  • src/backend/backup/backup_manifest.c: バックアップマニフェストのファイル更新時刻（mtime）として使用。
  • 呼び出し先:
  • 本シンボル自体は型定義であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • pg_tm: pg_time_t を分解して年・月・日・時・分・秒を保持するための構造体。
  • pg_tz: タイムゾーン情報を保持する不透明な構造体。

失敗モードと境界条件

  • 整数オーバーフロー: 64 ビット幅であるため、標準的な time_t（32 ビット環境）よりも大幅に長い期間を扱えますが、極端な未来や過去の時刻を扱う際には int64 の範囲制限を受けます。

保守担当者向けメモ

  • pg_time_t は PostgreSQL の時刻管理の基盤です。時刻計算ロジックを変更する際は、この型が 64 ビットであることを前提として設計してください。
  • pg_tm 構造体との相互運用時には、IANA タイムゾーンライブラリの慣習（月が 0 起点など）に従う必要がある点に注意してください。
