varchar_5

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-variable.c:105

目的

ECPG（Embedded SQL for C）プリプロセッサのテストにおいて、SQLの VARCHAR 型に対応するC言語のホスト変数構造体を定義するために使用されます。

シグネチャまたは形状

struct varchar_5 {
    int len;
    char arr[50];
};

入力と出力

  • **入力**: SQL文の実行結果（文字列データ）を格納するためのバッファとして機能します。
  • **出力**: ECPGライブラリを介して、データベースから取得した文字列データが arr に、その長さが len に書き込まれます。

前提条件と不変条件

  • len メンバは、arr に格納された有効な文字列の長さを保持する必要があります。
  • arr は固定長配列（50バイト）であるため、格納されるデータはこのサイズを超えてはなりません。

アルゴリズムまたは振る舞い

ECPGプリプロセッサによって生成されたコード内で、データベースとのデータ交換用ホスト変数として利用されます。ECPGdo 関数等の呼び出し時に、この構造体のアドレスが渡され、SQLの VARCHAR 型とマッピングされます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/preproc-array_of_struct.c:258 において、ECPGt_varchar 型指定子とともに参照されている可能性が高いです。
  • **呼び出し先**: ECPGライブラリ（ecpglib）の関数群が、この構造体のメモリ領域に対して読み書きを行います。

関連データ構造

  • varchar_4, varchar_6: 同様の目的で定義された、異なるバッファサイズを持つ構造体。
  • sqlca: SQL実行結果のステータスを保持する構造体。

失敗モードと境界条件

  • **バッファオーバーフロー**: データベースから取得する文字列が50バイトを超える場合、arr 配列の境界を越えるリスクがあります。
  • **型不一致**: ECPGの型指定子（ECPGt_varchar）と実際の構造体定義が一致しない場合、ランタイムエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はプリプロセッサが生成するテストコードの一部です。手動で定義を変更する際は、対応する .pgc ファイルの定義を確認してください。
  • 構造体のサイズ（50バイト）は、テストケースの要件に基づいています。変更が必要な場合は、関連するテストの期待値も更新する必要があります。
