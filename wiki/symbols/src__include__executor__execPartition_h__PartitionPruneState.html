PartitionPruneState

定義

  • 種別: struct
  • 定義箇所: src/include/executor/execPartition.h:121

目的

クエリ実行時において、パラメータ値等に基づき不要なパーティション（サブプラン）を動的に除外（プルーニング）するための状態を保持する。

シグネチャまたは形状

typedef struct PartitionPruneState
{
	ExprContext *econtext;
	Bitmapset  *execparamids;
	Bitmapset  *other_subplans;
	MemoryContext prune_context;
	bool		do_initial_prune;
	bool		do_exec_prune;
	int			num_partprunedata;
	PartitionPruningData *partprunedata[FLEXIBLE_ARRAY_MEMBER];
} PartitionPruneState;

入力と出力

  • **入力**: econtext（式評価用コンテキスト）、execparamids（実行時パラメータIDの集合）、partprunedata（パーティションプルーニング用データ配列）。
  • **出力**: other_subplans（除外対象外のサブプラン集合）、prune_context（メモリ管理用コンテキスト）。

前提条件と不変条件

  • partprunedata は FLEXIBLE_ARRAY_MEMBER を使用しており、num_partprunedata で指定された要素数を持つ。
  • プルーニング処理は、初期化時（do_initial_prune）および実行時（do_exec_prune）の両方で呼び出される可能性がある。

アルゴリズムまたは振る舞い

1. ExecInitPartitionExecPruning 等により生成され、実行計画の初期化および実行時のプルーニング状態を管理する。 2. ExecFindMatchingSubPlans が本構造体を参照し、実行時に評価を行って不要なサブプランをビットマップとして特定・除外する。 3. 実行時パラメータ（execparamids）が変更された場合、関連するマップが更新される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execPartition.c 内の CreatePartitionPruneState 等により生成される。また、ExecDoInitialPruning や ExecFindMatchingSubPlans 等から参照される。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すものではない。

関連データ構造

  • PartitionPruningData: パーティションごとのプルーニング情報を保持する。
  • ExprContext: 式評価のための実行コンテキスト。
  • Bitmapset: 除外対象やパラメータIDを管理するビットマップ。

失敗モードと境界条件

  • パーティションの階層構造が深い場合、再帰的な処理が必要となる。
  • execparamids の変更に伴う状態の不整合に注意が必要。

保守担当者向けメモ

  • 本構造体は execPartition.c の実装と密接に関連している。
  • プルーニングのロジックは partitioning/partprune.h に依存している。
  • 実行計画の初期化時と実行時の両方で状態が更新されるため、メモリ管理（prune_context）のライフサイクルに注意すること。
