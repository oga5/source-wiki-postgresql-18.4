timingsafe_bcmp

定義

  • 種別: function
  • 定義箇所: src/port/timingsafe_bcmp.c:29

目的

メモリ領域の比較処理において、処理時間を入力データの内容に依存させない（定数時間で実行する）ことで、タイミング攻撃を防止する。

シグネチャまたは形状

int timingsafe_bcmp(const void *b1, const void *b2, size_t n);

入力と出力

  • b1, b2: 比較対象のメモリ領域へのポインタ。
  • n: 比較するバイト数。
  • 戻り値: 比較した領域が完全に一致する場合は 0、不一致の場合は非 0 を返す。

前提条件と不変条件

  • 比較時間は入力データの内容に依存しない。

アルゴリズムまたは振る舞い

  • USE_SSL マクロが定義されている環境では、OpenSSL の CRYPTO_memcmp を呼び出す。
  • それ以外の環境では、以下のフォールバック実装を使用する：

    • 1. 入力ポインタを unsigned char として扱う。 2. 各バイトの XOR (^) を取り、その結果を累積変数 ret にビット単位の論理和 (|=) で格納する。 3. 全バイトの走査完了後、ret が 0 であれば一致、そうでなければ不一致と判定する。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • src/backend/libpq/auth-scram.c
  • src/backend/libpq/auth.c
  • src/backend/libpq/crypt.c
  • src/backend/storage/ipc/procsignal.c
  • src/interfaces/libpq/fe-auth-scram.c
  • 呼び出し先:
  • USE_SSL 定義時: CRYPTO_memcmp (OpenSSL)
  • USE_SSL 未定義時: なし（ループ処理のみ）

関連データ構造

  • 特になし。

失敗モードと境界条件

  • n が 0 の場合、ループは実行されず ret は初期値の 0 を返す（一致とみなされる）。
  • ポインタが指すメモリ領域が n バイト未満である場合、未定義動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数はパフォーマンスよりも処理時間の一定性を優先している。
  • ret |= *p1++ ^ *p2++ というイディオムは、分岐命令を発生させずに不一致を累積させるための定石である。
  • 暗号学的に重要なデータ（認証トークンやハッシュ等）の比較には必ず本関数を使用すること。
