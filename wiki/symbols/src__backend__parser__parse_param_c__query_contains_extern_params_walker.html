query_contains_extern_params_walker

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_param.c:338

目的

クエリツリーまたは式ツリーを走査し、PARAM_EXTERN 種類のパラメータが含まれているかを判定します。

シグネチャまたは形状

static bool query_contains_extern_params_walker(Node *node, void *context)

入力と出力

  • **入力**:
  • node: 走査対象の Node ツリー。
  • context: 走査の文脈（本関数内では使用されていません）。
  • **出力**:
  • bool: 外部パラメータが見つかった場合は true、それ以外は false を返します。

前提条件と不変条件

  • node が NULL の場合は false を返します。
  • 走査中に Param ノードに遭遇した場合、その paramkind が PARAM_EXTERN であれば即座に true を返します。

アルゴリズムまたは振る舞い

1. ノードが NULL なら false を返します。 2. ノードが Param 型の場合、paramkind が PARAM_EXTERN かを確認し、一致すれば true を返します。 3. ノードが Query 型の場合、query_tree_walker を使用してサブクエリ内を再帰的に走査します。 4. 上記以外の場合、expression_tree_walker を使用して式ツリーを再帰的に走査します。

呼び出し元と呼び出し先

  • **呼び出し元**: query_contains_extern_params（ソースコード上の直接的な呼び出し元は、本関数自身による再帰呼び出しおよび query_contains_extern_params 内での参照が確認されています）。
  • **呼び出し先**:
  • query_tree_walker（Query ノード走査時）
  • expression_tree_walker（その他のノード走査時）

関連データ構造

  • Node: 走査対象の汎用ノード構造体。
  • Param: パラメータ情報を保持する構造体。
  • Query: クエリツリーのルート構造体。

失敗モードと境界条件

  • 循環参照を持つツリー構造が渡された場合、walker 関数群の仕様に依存します。
  • node が NULL の場合は安全に false を返します。

保守担当者向けメモ

  • 本関数は query_tree_walker および expression_tree_walker のコールバックとして設計されています。
  • context 引数は現在の実装では使用されていませんが、将来的な拡張のためにシグネチャに含まれています。
