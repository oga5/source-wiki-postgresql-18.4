astreamer_zstd_decompressor_content

定義

  • **種別**: 関数
  • **定義箇所**: src/fe_utils/astreamer_zstd.c:296

目的

Zstandard形式で圧縮された入力データストリームを伸長し、その結果を後続のストリーマーへ転送する。

シグネチャまたは形状

static void astreamer_zstd_decompressor_content(
    astreamer *streamer,
    astreamer_member *member,
    const char *data,
    int len,
    astreamer_archive_context context
)

入力と出力

  • **入力**:
  • streamer: 伸長処理を行う astreamer_zstd_frame 構造体。
  • member: 現在処理中のアーカイブメンバー情報。
  • data, len: 伸長対象の圧縮データバッファとその長さ。
  • context: アーカイブ処理のコンテキスト。
  • **出力**: なし（副作用として後続のストリーマーへ伸長済みデータを渡す）。

前提条件と不変条件

  • streamer は astreamer_zstd_frame 型としてキャスト可能であること。
  • mystreamer->dctx（ZSTD伸長コンテキスト）が初期化済みであること。
  • mystreamer->zstd_outBuf が有効な出力バッファを保持していること。

アルゴリズムまたは振る舞い

1. 入力データ data を ZSTD_inBuffer に格納する。 2. 入力データがすべて処理されるまでループする。 3. 出力バッファ mystreamer->zstd_outBuf が満杯の場合、astreamer_content を呼び出して後続のストリーマーへデータを転送し、出力バッファをリセットする。 4. ZSTD_decompressStream を呼び出し、入力データを伸長して出力バッファへ書き込む。 5. ZSTD_isError でエラーを判定し、エラー発生時は pg_fatal で処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer インフラストラクチャのパイプライン（直接の呼び出し元はソースコード上では明示されていないが、astreamer 構造体の .content フィールドに登録される）。
  • **呼び出し先**:
  • astreamer_content（後続のストリーマーへのデータ転送）
  • ZSTD_decompressStream（libzstdによる伸長）
  • pg_fatal（エラー発生時）

関連データ構造

  • astreamer_zstd_frame: 伸長コンテキスト（dctx）および出力バッファ（zstd_outBuf）を保持する構造体。
  • ZSTD_inBuffer, ZSTD_outBuffer: libzstdのストリーム処理用バッファ構造体。

失敗モードと境界条件

  • **失敗モード**: ZSTD_decompressStream がエラーコードを返した場合、pg_fatal によりプログラムが異常終了する。
  • **境界条件**: 出力バッファが満杯になった際のフラッシュ処理およびバッファのリセットが正しく行われる必要がある。

保守担当者向けメモ

  • 本関数は USE_ZSTD が定義されている環境でのみ有効です。
  • ZSTD_decompressStream の戻り値チェックは必須であり、エラー発生時のログ出力には ZSTD_getErrorName を使用しています。
