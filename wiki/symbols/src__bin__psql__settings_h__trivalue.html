trivalue

定義

  • 種別: enum
  • 定義箇所: src/bin/psql/settings.h:94

目的

設定値が「デフォルト」「有効」「無効」の3つの状態を持つ場合に、その状態を表現するために使用されます。

シグネチャまたは形状

enum trivalue
{
    TRI_DEFAULT,
    TRI_NO,
    TRI_YES,
};

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: TRI_DEFAULT（0）、TRI_NO（1）、TRI_YES（2）のいずれかの整数値。

前提条件と不変条件

  • TRI_DEFAULT がデフォルト値として使用されることが想定されています。
  • 状態はこれら3つの値のいずれかに限定されます。

アルゴリズムまたは振る舞い

  • ユーザーによる明示的な設定がない場合に TRI_DEFAULT を保持し、必要に応じて TRI_YES（有効）または TRI_NO（無効）へ遷移させるロジックの基盤となります。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • contrib/vacuumlo/vacuumlo.c
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/pg_dump/connectdb.c / connectdb.h
  • src/bin/pg_dump/pg_backup.h / pg_backup_db.c
  • src/bin/pg_dump/pg_dumpall.c
  • src/bin/psql/command.c
  • src/bin/psql/settings.h (PsqlSettings 構造体内の getPassword フィールド)
  • 呼び出し先: なし（列挙型のため、関数呼び出しは発生しません）。

関連データ構造

  • PsqlSettings: psql の全状態を保持する構造体であり、この中で設定状態の管理に使用されます。

失敗モードと境界条件

  • 境界条件: 定義された3つの値以外の整数値が代入された場合の動作は未定義です。

保守担当者向けメモ

  • pset 構造体内のフィールドで使用される際、変数の変更に伴うフック関数経由で更新されるケースがあるため、直接代入を行う際は注意してください。
  • pg_dump や pg_amcheck など、psql 以外のツールでも共通して利用されているため、変更時は影響範囲を広範に確認してください。
