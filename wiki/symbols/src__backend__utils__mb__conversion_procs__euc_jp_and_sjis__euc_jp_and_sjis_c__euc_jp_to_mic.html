euc_jp_to_mic

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_jp_and_sjis/euc_jp_and_sjis.c:95

目的

EUC_JP エンコーディングの文字列を、PostgreSQL 内部表現である MULE_INTERNAL (MIC) に変換する。

シグネチャまたは形状

Datum euc_jp_to_mic(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を取得
  • src: 変換元文字列 (CSTRING)
  • dest: 変換先バッファ (CSTRING)
  • len: 変換対象の長さ (INT32)
  • noError: エラーハンドリングフラグ (BOOL)
  • **出力**: 変換されたバイト数 (INT32) を PG_RETURN_INT32 で返す。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_EUC_JP, PG_MULE_INTERNAL) により、エンコーディングの整合性が検証される。

アルゴリズムまたは振る舞い

1. PG_FUNCTION_ARGS から引数を抽出する。 2. CHECK_ENCODING_CONVERSION_ARGS を実行し、変換元が EUC_JP、変換先が MULE_INTERNAL であることを確認する。 3. 内部関数 euc_jp2mic を呼び出し、実際の変換処理を行う。 4. 変換されたバイト数を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1(euc_jp_to_mic) による登録経由）。
  • **呼び出し先**: euc_jp2mic（同ファイル内の静的関数）。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQL の関数引数インターフェース。

失敗モードと境界条件

  • noError フラグが false の場合、不正なバイト列や変換不能文字に対してエラーが報告される可能性がある（詳細は euc_jp2mic の実装に依存）。

保守担当者向けメモ

  • 本関数はインターフェース層であり、実際の変換ロジックは euc_jp2mic に委譲されている。
  • 変換エラーの挙動を調査する場合は、euc_jp2mic 内の report_invalid_encoding や report_untranslatable_char の呼び出し箇所を参照すること。
