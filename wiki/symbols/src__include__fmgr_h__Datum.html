Datum

定義

  • 種別: typedef
  • 定義箇所: src/include/fmgr.h:26

目的

PostgreSQL における汎用的なデータ表現型です。SQL レベルのデータ値（スカラ値やポインタ）を C 言語の関数間で受け渡すための標準的なコンテナとして機能します。

シグネチャまたは形状

typedef uintptr_t Datum; （※c.h 等の基盤定義に依存）

入力と出力

  • **入力**: SQL 関数呼び出し時に FunctionCallInfo を介して渡される引数、または PG_GETARG_xxx マクロによる抽出結果。
  • **出力**: PG_RETURN_xxx マクロを介して関数から返される戻り値。

前提条件と不変条件

  • Datum は、値そのもの（整数など）またはデータへのポインタを保持します。
  • ポインタとして使用される場合、そのデータは適切なアライメントを保持している必要があります。
  • PG_DETOAST_DATUM 等のデトースト処理を介さない限り、可変長データ（TOAST）の内部構造を直接操作してはなりません。

アルゴリズムまたは振る舞い

  • 関数マネージャ（fmgr）において、SQL 関数と C 言語関数を橋渡しする際の共通型として使用されます。
  • 呼び出し側は Datum を通じて型を抽象化し、受け取り側は PG_GETARG_xxx マクロを使用して特定の C 型へ変換します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_gin.c、verify_heapam.c、verify_nbtree.c 等のインデックス検証ロジックや、PostgreSQL バックエンドの各 SQL 関数実装。
  • **呼び出し先**: fmgr.h で定義される PG_GETARG_xxx や PG_RETURN_xxx マクロ群。
  • ※具体的な関数呼び出しの連鎖については、本シンボルが型定義であるため、特定の呼び出し先関数は存在しません。

関連データ構造

  • FunctionCallInfo: Datum を保持する引数配列を含む構造体。
  • FmgrInfo: 関数呼び出しのメタデータを管理する構造体。

失敗モードと境界条件

  • **NULL 処理**: Datum 自体は NULL を表現できません。NULL の有無は FunctionCallInfo 内の isnull 配列で管理されます。
  • **型不整合**: 期待される型と異なる Datum を PG_GETARG_xxx で取得した場合、予期せぬメモリ参照やクラッシュを招く可能性があります。

保守担当者向けメモ

  • Datum は PostgreSQL の ABI の根幹です。定義を変更するとシステム全体のバイナリ互換性に影響するため、慎重に扱う必要があります。
  • 64bit 環境では 8 バイトの整数として扱われますが、ポインタを格納する際はプラットフォーム依存のキャストに注意してください。
