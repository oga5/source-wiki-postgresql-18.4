ExecResult

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeResult.c:66

目的

Result ノードのメイン実行関数。定数条件（One-Time Filter）の評価、子ノードからのタプル取得、およびターゲットリストに基づく投影（Projection）を行う。

シグネチャまたは形状

TupleTableSlot *ExecResult(PlanState *pstate)

入力と出力

  • **入力**: pstate (PlanState *) - ResultState 構造体を含むプラン状態。
  • **出力**: TupleTableSlot * - 投影後のタプル、または条件不一致や処理終了時の NULL。

前提条件と不変条件

  • pstate は ResultState 型にキャスト可能であること。
  • CHECK_FOR_INTERRUPTS() が呼び出され、割り込みがチェックされる。
  • rs_checkqual が真の場合、resconstantqual が評価され、結果が偽であれば rs_done が真に設定される。

アルゴリズムまたは振る舞い

1. **定数評価**: rs_checkqual が真であれば resconstantqual を評価する。偽であれば rs_done を真にして NULL を返す。 2. **メモリリセット**: ResetExprContext を呼び出し、式評価用メモリを解放する。 3. **終了判定**: rs_done が真であれば NULL を返す。 4. **子ノード処理**:

  • 子ノードが存在する場合: ExecProcNode でタプルを取得。NULL なら終了。取得したタプルを ecxt_outertuple にセットする。
  • 子ノードが存在しない場合: 定数のみの生成とみなし、rs_done を真にする。

    • 5. **投影**: ExecProject を呼び出し、計算結果のタプルを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン内のプランノード実行フロー（詳細はソースコード上の明示的な呼び出し元情報が不足しているため、これ以上の特定は不可）。
  • **呼び出し先**: ExecQual, ResetExprContext, outerPlanState, ExecProcNode, ExecProject。

関連データ構造

  • ResultState: ノードの実行時状態を保持。
  • PlanState: 共通のプラン実行状態。
  • ExprContext: 式評価用のコンテキスト。
  • TupleTableSlot: タプルを保持するスロット。

失敗モードと境界条件

  • resconstantqual が偽の場合: 処理を中断し NULL を返す。
  • 子ノードが NULL を返した場合: 処理を終了する。
  • 子ノードがない場合: 一度だけ実行され、rs_done が真となる。

保守担当者向けメモ

  • rs_done フラグは、定数のみの生成や定数条件の失敗を追跡するために使用される。
  • Mark/Restore 操作は本関数では直接サポートしておらず、子ノードへ転送される設計となっている。
