uuid_out

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/uuid.c:88

目的

pg_uuid_t 構造体として保持されている 16 バイトの UUID バイナリデータを、標準的な 36 文字のハイフン区切り文字列（例: a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11）に変換します。

シグネチャまたは形状

Datum uuid_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される pg_uuid_t 型のポインタ。
  • **出力**: Datum 型として返される、UUID の文字列表現（C 文字列）。

前提条件と不変条件

  • 入力される pg_uuid_t は 16 バイトのデータ長を持つ必要があります。
  • 出力バッファは palloc により 2 * UUID_LEN + 5 バイト（ハイフン 4 つと終端ヌル文字を含む）が確保されます。

アルゴリズムまたは振る舞い

1. 16 バイトの UUID データを順次走査します。 2. 各バイトを上位 4 ビットと下位 4 ビットに分割し、hex_chars 配列を用いて 16 進数文字に変換します。 3. UUID の標準的なフォーマットに従い、インデックスが 4, 6, 8, 10 番目の位置にハイフン (-) を挿入します。 4. 最後に終端ヌル文字を付与し、文字列として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/uuid-ossp/uuid-ossp.c:239 (string_to_uuid 関数内)
  • **呼び出し先**:
  • palloc (メモリ確保)
  • PG_RETURN_CSTRING (結果の返却)

関連データ構造

  • pg_uuid_t: UUID を保持する 16 バイトの構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • 入力ポインタが不正な場合、メモリ保護違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は SQL レベルの型変換ハンドラとして機能します。
  • 出力フォーマットは RFC 9562 等の標準的な UUID 表記に固定されており、変更の際は SQL インターフェースとの整合性に注意してください。
