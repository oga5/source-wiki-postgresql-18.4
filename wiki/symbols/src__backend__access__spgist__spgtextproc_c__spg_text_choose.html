spg_text_choose

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgtextproc.c:183

目的

SP-GiST 索引において、挿入対象の文字列データに基づき、木構造のどのノードへ進むか、あるいはタプルの分割が必要かを決定する。

シグネチャまたは形状

Datum spg_text_choose(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgChooseIn（挿入対象データ、現在のレベル、接頭辞情報など）および spgChooseOut（結果格納用）を受け取る。
  • **出力**: spgChooseOut 構造体の resultType に応じて、以下のいずれかを設定する。
  • spgMatchNode: 既存ノードへの降下。
  • spgAddNode: 新規ノードの追加。
  • spgSplitTuple: タプルの分割。

前提条件と不変条件

  • 入力データは text 型として扱われる。
  • in->hasPrefix が真の場合、現在の接頭辞と入力文字列の比較が行われる。
  • 照合順序（Collation）の扱いは、spgtextproc.c 全体の設計に依存する。

アルゴリズムまたは振る舞い

1. **接頭辞照合**: 接頭辞がある場合、commonPrefix を用いて入力文字列と比較する。

  • 一致しない場合、spgSplitTuple を選択し、共通部分を抽出してタプルを再構成する。

    • 2. **ノード選択**: 接頭辞以降のバイト（nodeChar）を特定する。

  • searchChar で既存のノードラベルを検索し、見つかれば spgMatchNode を設定して降下する。
  • 見つからない場合、allTheSame フラグの状態に応じて spgSplitTuple または spgAddNode を選択する。

    • 3. **終了**: PG_RETURN_VOID() で終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiST のコアエンジン（spgdoinsert.c 等の挿入処理から呼び出されることが想定されるが、ソースコード上での直接的な呼び出し元は明示されていない）。
  • **呼び出し先**: commonPrefix, formTextDatum, searchChar, palloc 等のユーティリティ関数。

関連データ構造

  • spgChooseIn, spgChooseOut: SP-GiST インターフェースで定義される入出力構造体。
  • text: PostgreSQL の可変長文字列型。

失敗モードと境界条件

  • **タプル分割**: 接頭辞が一致しない場合や、allTheSame 状態でノード追加が不可能な場合に発生する。
  • **終端**: 文字列の終端は nodeChar = -1 として扱われる。
  • **メモリ**: palloc を使用するため、メモリ不足時にはエラーが発生する可能性がある。

保守担当者向けメモ

  • allTheSame 状態での分割時に使用されるダミーラベル -2 は、文字列の復元には関与しない特殊な値である。
  • nodeChar の決定ロジックは、インデックスの木構造を維持するための重要な分岐点となるため、変更時は spg_text_inner_consistent との整合性に注意すること。
