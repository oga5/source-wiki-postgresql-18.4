pgstat_delete_pending_entry

定義

  • src/backend/utils/activity/pgstat.c:1317

目的

バックエンドローカルな統計情報の保留エントリ（pending entry）を破棄し、関連するメモリを解放して管理リストから削除します。

シグネチャまたは形状

void pgstat_delete_pending_entry(PgStat_EntryRef *entry_ref)

入力と出力

  • **入力**: PgStat_EntryRef *entry_ref - 削除対象の統計エントリ参照。
  • **出力**: なし（副作用としてメモリ解放とリスト操作を実行）。

前提条件と不変条件

  • entry_ref->pending が NULL であってはならない（Assert により検証）。
  • 対象の統計種別（kind）は fixed_amount（固定量）であってはならない（Assert により検証）。

アルゴリズムまたは振る舞い

1. entry_ref から統計種別を取得し、対応する PgStat_KindInfo を特定します。 2. 種別固有の削除コールバック delete_pending_cb が定義されている場合、それを実行します。 3. entry_ref->pending が指すメモリ領域を pfree で解放し、ポインタを NULL に設定します。 4. dlist_delete を呼び出し、当該エントリを保留リスト（pending list）から削除します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/pgstat.c 内の統計管理処理。
  • src/backend/utils/activity/pgstat_shmem.c:609。
  • **呼び出し先**:
  • pgstat_get_kind_info（種別情報の取得）。
  • kind_info->delete_pending_cb（種別固有のクリーンアップ処理）。
  • pfree（メモリ解放）。
  • dlist_delete（リスト操作）。

関連データ構造

  • PgStat_EntryRef: 統計エントリの参照と保留状態を保持する構造体。
  • PgStat_KindInfo: 統計種別ごとの振る舞いを定義する構造体。

失敗モードと境界条件

  • entry_ref が指す pending データが既に NULL の場合、Assert により異常終了します。
  • fixed_amount が真である統計種別に対して呼び出した場合、Assert により異常終了します。

保守担当者向けメモ

  • 本関数は、動的に割り当てられた保留統計データのクリーンアップを担います。fixed_amount 統計については別の管理パスが存在するため、本関数は使用されません。
