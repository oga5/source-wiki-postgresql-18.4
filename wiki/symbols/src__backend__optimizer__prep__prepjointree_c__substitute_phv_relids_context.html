substitute_phv_relids_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/prep/prepjointree.c:4138

目的

PlaceHolderVar（PHV）内のリレーション識別子（relids）を置換する処理において、置換対象の特定と状態保持を行うためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct
{
	int			varno;
	int			sublevels_up;
	Relids		subrelids;
} substitute_phv_relids_context;

入力と出力

本構造体は関数への引数として渡されるコンテキストとして機能します。

  • varno: 置換対象となる変数のリレーション番号。
  • sublevels_up: サブクエリの階層レベル。
  • subrelids: 置換後の新しいリレーション識別子の集合。

前提条件と不変条件

  • subrelids は Relids 型であり、ビットマップセットとして管理されます。
  • 本構造体は、特定の走査処理（PlaceHolderVar の探索・置換）の実行中にのみ有効な一時的な状態を保持します。

アルゴリズムまたは振る舞い

ジョインツリーの最適化過程において、特定の PlaceHolderVar が参照するリレーション ID を別の集合へ置き換える際に使用されます。この構造体を参照する関数（src/backend/optimizer/prep/prepjointree.c:4147 付近）が、ツリーを再帰的に探索し、条件に合致するノードに対して subrelids への更新を適用します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/prep/prepjointree.c 内の関数（4191 行目などでインスタンス化）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、これを引数として受け取る走査関数が呼び出されます。
  • 詳細なコールグラフは提供されたソース抜粋からは特定できません。

関連データ構造

  • Relids: リレーション ID の集合を保持する型。
  • PlaceHolderVar: 本コンテキストが操作対象とする最適化ノード。

失敗モードと境界条件

  • sublevels_up が不適切な値である場合、誤った階層の変数が置換される可能性があります。
  • subrelids が空集合の場合、置換後の参照先が消失する可能性があります。

保守担当者向けメモ

  • 本構造体は prepjointree.c 内の局所的な最適化ロジックに密結合しています。
  • PlaceHolderVar の伝播ルールに変更を加える際は、本構造体のフィールドが適切に更新されているか確認してください。
