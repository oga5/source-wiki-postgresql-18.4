shift_jis_2004_to_utf8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_sjis2004/utf8_and_sjis2004.c:41

目的

PostgreSQLの文字コード変換サブシステムにおいて、SHIFT_JIS_2004 エンコーディングから UTF-8 エンコーディングへの変換を実行するエントリポイントを提供します。

シグネチャまたは形状

Datum shift_jis_2004_to_utf8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の引数を受け取ります。
  • src (第3引数): 変換元文字列へのポインタ。
  • dest (第4引数): 変換先バッファへのポインタ。
  • len (第5引数): 変換元文字列の長さ。
  • noError (第6引数): 変換不能文字に対するエラーハンドリングフラグ。
  • **出力**: PG_RETURN_INT32 を介して、変換されたバイト数を返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_SHIFT_JIS_2004, PG_UTF8) により、エンコーディングIDが正しいことが検証されます。
  • 変換には shift_jis_2004_to_unicode_tree および LUmapSHIFT_JIS_2004_combined マッピングデータが使用されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得します。 2. エンコーディングIDの検証を行います。 3. LocalToUtf 関数を呼び出し、マッピングテーブルを用いて SHIFT_JIS_2004 から UTF-8 への変換を実行します。 4. noError が true の場合、変換不能な文字があってもエラーを発生させずに処理を継続します。 5. 変換されたバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの文字コード変換サブシステム（mb）から、動的ロード可能な変換プロシージャとして呼び出されます。
  • **呼び出し先**: LocalToUtf（mb/pg_wchar.h で定義）を呼び出します。

関連データ構造

  • shift_jis_2004_to_unicode_tree: 変換用ツリー構造。
  • LUmapSHIFT_JIS_2004_combined: 変換用マッピングテーブル。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、エラーが発生します。
  • noError フラグが false の場合、変換不能な文字が含まれているとエラーが発生します。

保守担当者向けメモ

  • 本関数は変換の「接着剤」であり、実際の変換ロジックはインクルードされている .map ファイルおよび LocalToUtf 関数に依存しています。
  • マッピングテーブルの構造については src/backend/utils/Unicode/ 以下の生成スクリプトを参照してください。
