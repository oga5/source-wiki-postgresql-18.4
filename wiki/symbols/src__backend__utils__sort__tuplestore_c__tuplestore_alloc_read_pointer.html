tuplestore_alloc_read_pointer

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplestore.c:394

目的

Tuplestorestate に対して、独立した新しい読み取りポインタ（TSReadPointer）を割り当て、初期化する。

シグネチャまたは形状

int tuplestore_alloc_read_pointer(Tuplestorestate *state, int eflags)

入力と出力

  • **入力**:
  • state: 操作対象の Tuplestorestate 構造体へのポインタ。
  • eflags: 新しい読み取りポインタに適用する実行フラグ（EXEC_FLAG_*）。
  • **出力**:
  • 戻り値: 新しく割り当てられた読み取りポインタのインデックス（int）。

前提条件と不変条件

  • すでにタプルの読み書きが進行している場合、新しい読み取りポインタに対して既存の eflags よりも厳しい要求（新しいフラグの追加）を行うことはできない。これに違反すると elog(ERROR) が発生する。

アルゴリズムまたは振る舞い

1. **制約チェック**: state がメモリ内（TSS_INMEM）かつ空でない場合、または既に読み書きが進行している場合、要求された eflags が既存の state->eflags の範囲内であることを検証する。 2. **領域確保**: readptrcount が readptrsize に達している場合、repalloc を用いて readptrs 配列のサイズを2倍に拡張する。 3. **初期化**: 新しい読み取りポインタをインデックス 0 のポインタの状態（現在の読み取り位置など）で初期化し、指定された eflags を設定する。 4. **状態更新**: state->eflags に新しい eflags を統合し、readptrcount をインクリメントして返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • nodeCtescan.c: ExecCteScan
  • nodeMaterial.c: ExecMaterial
  • nodeNamedtuplestorescan.c: ExecNamedTuplestoreScan
  • nodeWindowAgg.c: ExecWindowAgg
  • **呼び出し先**:
  • repalloc (メモリ管理)
  • elog (エラー報告)

関連データ構造

  • Tuplestorestate: 状態管理構造体。
  • TSReadPointer: 読み取り位置やフラグを保持する構造体。

失敗モードと境界条件

  • **メモリ不足**: repalloc が失敗した場合は標準的なメモリ不足エラーが発生する。
  • **フラグ違反**: 既に進行中の処理に対して、互換性のない eflags を要求すると elog(ERROR) により中断される。

保守担当者向けメモ

  • 本関数は、複数の読み取りポインタを動的に追加する仕組みを提供します。readptrs 配列の拡張は repalloc で行われるため、ポインタの再配置が発生する点に注意してください。
