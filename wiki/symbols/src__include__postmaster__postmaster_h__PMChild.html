PMChild

定義

  • 種別: struct
  • 定義箇所: src/include/postmaster/postmaster.h:40

目的

Postmasterが管理するアクティブな子プロセスの状態を追跡し、ライフサイクルを制御するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    pid_t pid;                      /* プロセスのPID */
    int child_slot;                 /* 割り当てられたスロット番号 */
    BackendType bkend_type;         /* プロセスの種類 */
    struct RegisteredBgWorker *rw;  /* バックグラウンドワーカー情報（該当する場合） */
    bool bgworker_notify;           /* ワーカーの開始/停止通知フラグ */
    dlist_node elem;                /* ActiveChildListへのリンク */
} PMChild;

入力と出力

本構造体はデータ保持用であり、関数ではありません。src/backend/postmaster/pmchild.c において、スロット割り当て時に初期化され、プロセス終了時に無効化されます。

前提条件と不変条件

  • ActiveChildList にリンクされている間は、有効な子プロセスとして追跡されます。
  • child_slot は、固定プールまたは動的割り当てのいずれかによって管理されます。

アルゴリズムまたは振る舞い

  • Postmasterは、新しい子プロセス生成時に PMChild エントリを割り当てます。
  • 子プロセスの終了検知時、対応する PMChild エントリを参照してスロットの解放や状態更新を行います。
  • bgworker_notify が真の場合、バックグラウンドワーカーのライフサイクルイベントに応じた通知処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/pmchild.c 内のプロセス管理関数群が、この構造体の割り当ておよび管理を行います。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出しません。

関連データ構造

  • ActiveChildList: 実行中の全 PMChild を管理する二重連結リスト。
  • RegisteredBgWorker: バックグラウンドワーカーの詳細情報。
  • BackendType: プロセスの種類を識別する列挙型。

失敗モードと境界条件

  • pmchild.c において、指定された BackendType に対してスロットが確保できない場合、elog(ERROR) が発生します。
  • クラッシュリカバリ時には、通常のプロセス終了時とは異なるスロット管理ロジックが適用される場合があります。

保守担当者向けメモ

  • 本構造体は共有メモリ外で管理されるプロセス管理用データです。
  • EXEC_BACKEND が定義されている環境（Windows等）では、プロセス生成の仕組みが異なるため、本構造体の利用方法に注意してください。
  • dlist_node を含んでいるため、lib/ilist.h のリスト操作関数と組み合わせて使用する必要があります。
