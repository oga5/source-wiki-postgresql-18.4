BackendStatusShmemInit

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/activity/backend_status.c:113

目的

PostgreSQLのバックエンドプロセスの活動状況を追跡するための共有メモリ領域を確保および初期化します。

シグネチャまたは形状

void BackendStatusShmemInit(void)

入力と出力

  • **入力**: なし（グローバル変数 NumBackendStatSlots および pgstat_track_activity_query_size を参照）。
  • **出力**: なし。共有メモリ上のグローバルポインタ変数（BackendStatusArray 等）を初期化します。

前提条件と不変条件

  • ポストマスター起動時に一度だけ呼び出される必要があります。
  • ShmemInitStruct を使用して共有メモリを割り当てるため、共有メモリシステムが初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. **共有メモリの確保**: ShmemInitStruct を使用し、以下の各領域を確保または既存領域へのアタッチを行います。

  • BackendStatusArray: 全バックエンドのステータス構造体配列。
  • BackendAppnameBuffer: アプリケーション名格納用バッファ。
  • BackendClientHostnameBuffer: クライアントホスト名格納用バッファ。
  • BackendActivityBuffer: クエリ文字列格納用バッファ。
  • BackendSslStatusBuffer / BackendGssStatusBuffer: (有効時) SSL/GSSAPI状態格納用バッファ。

    • 2. **初期化**: 領域が新規作成された場合（found が false）、MemSet でゼロクリアし、BackendStatusArray 内の各エントリが対応するバッファ領域を指すようにポインタをセットアップします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/ipci.c:313（CreateSharedMemoryAndSemaphores 内から呼び出される）。
  • **呼び出し先**: ShmemInitStruct（共有メモリ管理用API）。

関連データ構造

  • PgBackendStatus: 各バックエンドの状態を保持する構造体。
  • BackendStatusArray: PgBackendStatus の共有メモリ配列。
  • BackendAppnameBuffer, BackendClientHostnameBuffer, BackendActivityBuffer: 各種文字列情報を保持する共有メモリバッファ。

失敗モードと境界条件

  • 共有メモリが不足している場合、ShmemInitStruct がエラーを発生させ、プロセスが終了する可能性があります。
  • NumBackendStatSlots が正しく設定されていない場合、メモリ確保サイズが不正になります。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウトを決定する重要な初期化処理です。構造体のメンバやバッファサイズを変更する場合は、本関数内のポインタ初期化ロジックと整合性を取る必要があります。
  • USE_SSL や ENABLE_GSS の条件付きコンパイルにより、共有メモリの構成が動的に変化することに注意してください。
