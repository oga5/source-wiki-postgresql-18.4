sepgsql_relation_setattr_extra

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/relation.c:720

目的

特定のシステムカタログからオブジェクトの関連付け（リレーションID）を検索し、そのリレーションに対して sepgsql_relation_setattr を呼び出すことで、セキュリティ属性の変更処理を委譲する。

シグネチャまたは形状

static void sepgsql_relation_setattr_extra(
    Relation catalog,
    Oid catindex_id,
    Oid extra_oid,
    AttrNumber anum_relation_id,
    AttrNumber anum_extra_id
)

入力と出力

  • **入力**:
  • catalog: 検索対象のシステムカタログリレーション。
  • catindex_id: 検索に使用するインデックスのOID。
  • extra_oid: 検索キーとなるオブジェクトのOID。
  • anum_relation_id: 取得対象のリレーションIDが格納されている属性番号。
  • anum_extra_id: 検索キーとなる属性番号。
  • **出力**: なし（副作用として sepgsql_relation_setattr を実行）。

前提条件と不変条件

  • extra_oid に対応するタプルが catalog 内に存在すること。存在しない場合は ERROR が発生する。
  • anum_relation_id で指定された属性値は NULL であってはならない（Assert により検証）。

アルゴリズムまたは振る舞い

1. ScanKeyInit を使用し、anum_extra_id と extra_oid を用いた検索キーを初期化する。 2. systable_beginscan を呼び出し、SnapshotSelf を用いてカタログスキャンを開始する。 3. systable_getnext で対象タプルを取得する。 4. heap_getattr を使用し、タプルから anum_relation_id に該当するリレーションOIDを抽出する。 5. 抽出したOIDを引数として sepgsql_relation_setattr を呼び出す。 6. systable_endscan でスキャンを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の参照情報に基づくと、contrib/sepgsql/relation.c 内の他の関数から呼び出されている可能性があるが、具体的な関数名は提供されていない。
  • **呼び出し先**:
  • systable_beginscan, systable_getnext, systable_endscan (PostgreSQLカタログアクセスAPI)
  • heap_getattr
  • sepgsql_relation_setattr

関連データ構造

  • Relation: システムカタログの構造体。
  • ScanKeyData: カタログスキャン用の検索条件。
  • SysScanDesc: システムカタログスキャンの状態管理。
  • HeapTuple: カタログから取得されるタプル。

失敗モードと境界条件

  • 指定された extra_oid がカタログ内に見つからない場合、elog(ERROR, ...) によりトランザクションが中断される。
  • 取得したリレーションIDが NULL の場合、Assert によりデバッグビルド環境で停止する。

保守担当者向けメモ

  • 本関数は、インデックスやその他の補助オブジェクトに対する操作を、親リレーションのセキュリティ属性変更として扱うための汎用的なブリッジとして機能する。
  • SnapshotSelf を使用しているため、DDL実行直後の未コミット状態のカタログレコードを正しく参照できる。
