SSLerrfree

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure-openssl.c:1611

目的

OpenSSL エラーメッセージ文字列のメモリを解放する。

シグネチャまたは形状

static void SSLerrfree(char *buf)

入力と出力

  • **入力**: buf (解放対象の文字列ポインタ)
  • **出力**: なし

前提条件と不変条件

  • buf が ssl_nomem と等しい場合、メモリ解放は行われない。
  • buf が ssl_nomem 以外を指している場合、そのメモリは free() によって解放される。

アルゴリズムまたは振る舞い

1. 引数 buf が静的定数 ssl_nomem と一致するか比較する。 2. 一致しない場合のみ、標準ライブラリの free() を呼び出してメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-secure-openssl.c 内の複数の箇所（行 202, 308, 777, 829, 855, 894, 911, 1019, 1042, 1091, 1108）から呼び出されている。
  • **呼び出し先**: free()（標準ライブラリ）。

関連データ構造

  • ssl_nomem: メモリ不足時に使用される静的なエラーメッセージ用定数。

失敗モードと境界条件

  • buf が NULL の場合、free() の仕様に依存する（通常は安全だが、本関数内では ssl_nomem との比較が先行する）。
  • ssl_nomem 以外のポインタが free() できない領域を指している場合、未定義動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は SSLerrmessage 等で生成されたエラーメッセージバッファのライフサイクル管理に使用される。
  • ssl_nomem は静的メモリを指しているため、誤って解放しないようガードされている。
