StartTransaction

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_db.c:467

目的

データベース接続に対して BEGIN コマンドを発行し、トランザクションを開始する。

シグネチャまたは形状

void StartTransaction(Archive *AHX)

入力と出力

  • **入力**: Archive *AHX - アーカイブハンドルへのポインタ。
  • **出力**: なし（成功時）。失敗時は内部でエラー処理を行いプロセスを終了する。

前提条件と不変条件

  • AHX は有効な Archive 構造体である必要がある。
  • 内部で ArchiveHandle 型へキャストされるため、pg_backup_archiver.h で定義された構造体レイアウトと整合している必要がある。

アルゴリズムまたは振る舞い

1. 引数 AHX を ArchiveHandle *AH にキャストする。 2. ExecuteSqlCommand 関数を呼び出し、SQL コマンド "BEGIN" を実行する。 3. 実行失敗時には、エラーメッセージ "could not start database transaction" を伴って終了処理を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内には、本関数を呼び出している具体的な箇所は記載されていない。
  • **呼び出し先**: ExecuteSqlCommand を呼び出す。
  • 注意: ツールが検出した src/backend/access/transam/xact.c 内の StartTransaction は、バックエンド側の同名関数であり、本関数（フロントエンド用）とは別物である。

関連データ構造

  • Archive / ArchiveHandle: データベース接続状態やアーカイブ設定を保持する構造体。

失敗モードと境界条件

  • データベース接続が確立されていない場合や、サーバー側でトランザクション開始が拒否された場合、ExecuteSqlCommand 経由でプロセスが終了する。

保守担当者向けメモ

  • 本関数は pg_dump 等のフロントエンドツール用であり、バックエンドのトランザクション管理機能とは直接関係がない。
  • ExecuteSqlCommand に依存しているため、SQL 実行時のエラーハンドリング方針（即時終了）を継承している。
