vm_readbuf

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/visibilitymap.c:559

目的

可視性マップ（Visibility Map）の指定されたブロックをバッファプールから読み込み、必要に応じて初期化を行う。

シグネチャまたは形状

static Buffer vm_readbuf(Relation rel, BlockNumber blkno, bool extend)

入力と出力

  • **入力**:
  • rel: 対象のリレーション。
  • blkno: 読み込む可視性マップのブロック番号。
  • extend: 指定ブロックが現在のマップサイズを超えている場合に、拡張を試みるかどうか。
  • **出力**:
  • 成功時: 対象ブロックの Buffer。
  • 失敗時: extend が false でブロックが存在しない場合、InvalidBuffer。

前提条件と不変条件

  • VISIBILITYMAP_FORKNUM を使用してアクセスする。
  • 読み込み時に RBM_ZERO_ON_ERROR モードを使用するため、破損ページはエラーではなくゼロクリアされる。
  • ページが新規（PageIsNew）である場合、排他ロックを取得して PageInit を実行する。

アルゴリズムまたは振る舞い

1. リレーションの可視性マップフォークのサイズをキャッシュから確認し、必要に応じて smgrnblocks で更新する。 2. 指定ブロックが現在のサイズを超えている場合:

  • extend が true なら vm_extend を呼び出し、そうでなければ InvalidBuffer を返す。

    • 3. ReadBufferExtended を使用してバッファを読み込む。 4. 読み込んだページが新規（PageIsNew）であれば:

  • 排他ロックを取得し、再度新規性を確認した上で PageInit を実行して初期化する。
  • ロックを解除してバッファを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • visibilitymap.c 内の各関数（visibilitymap_pin 等）から、マップ操作の基盤として呼び出される。
  • **呼び出し先**:
  • RelationGetSmgr
  • smgrexists / smgrnblocks
  • vm_extend
  • ReadBufferExtended
  • PageIsNew / BufferGetPage / PageInit
  • LockBuffer

関連データ構造

  • Relation: リレーション情報。
  • Buffer: バッファプール内のページハンドル。
  • SMgrRelation: ストレージマネージャのキャッシュ情報。

失敗モードと境界条件

  • extend が false の状態でマップの末尾を超えたブロックを要求すると InvalidBuffer を返す。
  • 並行して他プロセスがページを初期化している場合、PageIsNew の再チェックにより二重初期化を防ぐ設計となっている。

保守担当者向けメモ

  • 本関数はロックなしで PageIsNew をチェックするパスを含むが、これは PageGetContents() がページヘッダーの正確な内容に依存しないため許容されている。
  • smgr_cached_nblocks のキャッシュはリレーションのクローズ時に無効化される可能性があるため、RelationGetSmgr の使用には注意が必要である。
