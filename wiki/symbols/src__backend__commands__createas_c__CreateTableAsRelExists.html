CreateTableAsRelExists

定義

  • 種別: function
  • 定義箇所: src/backend/commands/createas.c:392

目的

CREATE TABLE AS (CTAS) 文において、ターゲットとなるリレーションが既に存在するかを確認し、IF NOT EXISTS 句の指定に基づいた制御を行う。

シグネチャまたは形状

bool CreateTableAsRelExists(CreateTableAsStmt *ctas)

入力と出力

  • **入力**: CreateTableAsStmt *ctas（CTAS 文の構文解析ツリー）
  • **出力**: bool
  • true: リレーションが既に存在し、かつ IF NOT EXISTS が指定されている場合。
  • false: リレーションが存在しない場合。

前提条件と不変条件

  • ctas->into->rel を通じてターゲットリレーションの識別情報が提供されていること。
  • RangeVarGetCreationNamespace により、リレーションを作成する名前空間が特定可能であること。

アルゴリズムまたは振る舞い

1. RangeVarGetCreationNamespace を呼び出し、対象リレーションの名前空間を特定する。 2. get_relname_relid を使用して、当該名前空間内に同名のリレーションが存在するかを確認する。 3. リレーションが存在する場合:

  • ctas->if_not_exists が偽であれば、ERRCODE_DUPLICATE_TABLE エラーを発生させる。
  • if_not_exists が真であれば、checkMembershipInCurrentExtension を呼び出し、拡張スクリプト内でのセキュリティチェックを行う。
  • NOTICE を発行し、true を返して処理をスキップさせる。

    • 4. リレーションが存在しない場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/createas.c:236
  • src/backend/commands/explain.c:411
  • **呼び出し先**:
  • RangeVarGetCreationNamespace
  • get_relname_relid
  • checkMembershipInCurrentExtension
  • ereport (エラーおよび通知の発行)

関連データ構造

  • CreateTableAsStmt: CTAS 文の構造体。
  • IntoClause: ターゲットリレーションの指定を含む構造体。

失敗モードと境界条件

  • 同名のリレーションが既に存在し、かつ IF NOT EXISTS が指定されていない場合、ERRCODE_DUPLICATE_TABLE でエラー終了する。
  • 拡張スクリプト内で既存オブジェクトが拡張のメンバーでない場合、セキュリティ上の理由でエラーとなる可能性がある。

保守担当者向けメモ

  • 本関数は CTAS 実行の初期段階で、重複作成を回避するために使用される。
  • IF NOT EXISTS の挙動は、ダンプ/リストア時の依存関係解決において重要な役割を果たす。
