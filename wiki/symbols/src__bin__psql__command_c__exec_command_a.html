exec_command_a

定義

  • 種別: function
  • 定義箇所: src/bin/psql/command.c:500

目的

psqlの出力フォーマットを aligned と unaligned の間でトグル（切り替え）します。

シグネチャまたは形状

static backslashResult exec_command_a(PsqlScanState scan_state, bool active_branch)

入力と出力

  • **入力**:
  • scan_state: 字句解析器の状態（本関数内では未使用）。
  • active_branch: 現在の条件付き実行ブロックがアクティブかどうかを示すフラグ。
  • **出力**:
  • backslashResult: 実行結果。成功時は PSQL_CMD_SKIP_LINE、失敗時は PSQL_CMD_ERROR を返します。

前提条件と不変条件

  • active_branch が true の場合のみ、出力フォーマットの変更処理が実行されます。
  • pset（グローバル設定）が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. active_branch が true の場合：

  • 現在のフォーマットが PRINT_ALIGNED であれば、do_pset を呼び出してフォーマットを unaligned に設定します。
  • それ以外の場合は、do_pset を呼び出してフォーマットを aligned に設定します。

    • 2. do_pset の戻り値を success に格納します。 3. success が true ならば PSQL_CMD_SKIP_LINE を、false ならば PSQL_CMD_ERROR を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c 内のコマンドディスパッチャ（exec_command）から呼び出されます。
  • **呼び出し先**: do_pset（出力設定変更関数）。

関連データ構造

  • backslashResult: コマンド実行結果の列挙型。
  • pset: psql のグローバル設定を保持する構造体。

失敗モードと境界条件

  • do_pset が失敗した場合、本関数は PSQL_CMD_ERROR を返します。
  • active_branch が false の場合は何も変更せず、常に PSQL_CMD_SKIP_LINE を返します。

保守担当者向けメモ

  • 本関数は \a コマンドの実装です。
  • do_pset の第4引数に pset.quiet を渡しているため、設定変更時のメッセージ出力は現在の静音設定に従います。
