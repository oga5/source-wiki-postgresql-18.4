BloomOpaque

定義

  • **種別**: struct
  • **定義箇所**: src/backend/access/brin/brin_bloom.c:433

目的

BRIN（Block Range Index）のブルームフィルタ演算子クラスにおいて、インデックス構築やスキャンに必要な関数情報（FmgrInfo）を保持するための不透明な（opaque）データ構造です。

シグネチャまたは形状

typedef struct BloomOpaque
{
    FmgrInfo    extra_procinfos[BLOOM_MAX_PROCNUMS];
} BloomOpaque;

入力と出力

  • **入力**: 構造体内部の extra_procinfos 配列を通じて、ハッシュ関数等の関数呼び出し情報が格納されます。
  • **出力**: インデックス記述子（BrinDesc）の bd_info を介して、演算子クラスの実行時に必要な関数情報を提供します。

前提条件と不変条件

  • BLOOM_MAX_PROCNUMS で定義された数の FmgrInfo を保持します。
  • 他のBRIN演算子クラス（InclusionやMinMax）との整合性を保つため、将来的な拡張を見越して配列形式で保持されています。

アルゴリズムまたは振る舞い

  • インデックスの初期化時にメモリが割り当てられ、演算子クラスの関数情報が格納されます。
  • 実行時には、特定の属性番号（attno）に対応する bd_info からキャストして取得され、ハッシュ計算等の処理に利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_bloom.c 内のインデックス初期化およびスキャン処理（brin_bloom_opcinfo 等）から、sizeof(BloomOpaque) を用いたメモリ確保や、oi_opaque へのキャストを通じて参照されます。
  • **呼び出し先**:
  • 直接的な関数呼び出しは行いませんが、保持している FmgrInfo を通じて、PostgreSQLの関数マネージャが提供するハッシュ関数等が実行されます。

関連データ構造

  • BrinDesc: インデックス記述子。bd_info 配列を通じて本構造体を保持します。
  • FmgrInfo: 関数マネージャの情報構造体。本構造体の主要な構成要素です。

失敗モードと境界条件

  • メモリ確保失敗時（palloc 失敗時）にエラーが発生する可能性があります。
  • attno - 1 を用いた配列アクセスが行われるため、インデックス定義と整合しない属性番号でのアクセスは境界外参照のリスクがあります。

保守担当者向けメモ

  • 現在はハッシュ計算用の単一プロシージャのみが必要ですが、将来的な機能拡張（追加のプロシージャ利用）に備えて配列構造が維持されています。変更時は BLOOM_MAX_PROCNUMS の定義に注意してください。
