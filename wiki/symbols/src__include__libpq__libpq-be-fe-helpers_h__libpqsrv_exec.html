libpqsrv_exec

定義

  • 種別: function
  • 定義箇所: src/include/libpq/libpq-be-fe-helpers.h:255

目的

PostgreSQL バックエンド内で libpq を使用してクエリを実行する際のラッパー関数です。クエリ実行中の割り込み処理を安全に行い、バックエンドの FD 管理ルールに適合させます。

シグネチャまたは形状

PGresult *libpqsrv_exec(PGconn *conn, const char *query, uint32 wait_event_info);

入力と出力

  • conn: 接続済みの PGconn オブジェクト。
  • query: 実行する SQL クエリ文字列。
  • wait_event_info: 待機イベントの識別子（utils/wait_event.h で定義）。
  • 戻り値: クエリ結果を保持する PGresult へのポインタ。失敗時は NULL を返します。

前提条件と不変条件

  • conn は libpqsrv_connect 等で正しく確立された有効な接続である必要があります。
  • バックエンドの FD 制限内で動作するため、呼び出し元は適切なリソース管理下にあることが前提です。

アルゴリズムまたは振る舞い

1. PQsendQuery(conn, query) を呼び出し、クエリをサーバーへ送信します。送信に失敗した場合は直ちに NULL を返します。 2. 送信成功後、libpqsrv_get_result_last を呼び出します。 3. libpqsrv_get_result_last は、内部で PQisBusy が偽になるまで WaitLatchOrSocket を用いて待機し、割り込み（CHECK_FOR_INTERRUPTS()）を処理しながら結果を収集します。 4. 最終的な PGresult を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • src/backend/replication/libpqwalreceiver/libpqwalreceiver.c
  • **呼び出し先**:
  • PQsendQuery (libpq)
  • libpqsrv_get_result_last (本ヘッダー内のヘルパー関数)

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • PGresult: クエリ実行結果を格納する構造体。

失敗モードと境界条件

  • PQsendQuery が失敗した場合、関数は NULL を返します。
  • 実行中にバックエンドで割り込み（キャンセル要求など）が発生した場合、libpqsrv_get_result_last 内の処理により適切に中断される可能性があります。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーオンリーライブラリの一部として提供されています。
  • 拡張機能開発時のみ利用可能です（バックエンド本体のビルド時は BUILDING_DLL ガードにより制限されます）。
  • 接続の非ブロッキング処理については歴史的経緯があり、将来的な改善が推奨されています。
