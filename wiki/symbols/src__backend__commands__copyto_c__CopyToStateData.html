CopyToStateData

定義

  • **種別**: struct
  • **定義箇所**: src/backend/commands/copyto.c:65

目的

COPY TO コマンドの実行中における全状態を保持・管理するためのメイン構造体です。出力形式の抽象化、出力先管理、メモリコンテキストの制御、および進捗状況の追跡を統合します。

シグネチャまたは形状

typedef struct CopyToStateData {
    const CopyToRoutine *routine;
    CopyDest    copy_dest;
    FILE       *copy_file;
    StringInfo  fe_msgbuf;
    int         file_encoding;
    bool        need_transcoding;
    bool        encoding_embeds_ascii;
    Relation    rel;
    QueryDesc  *queryDesc;
    List       *attnumlist;
    char       *filename;
    bool        is_program;
    copy_data_dest_cb data_dest_cb;
    CopyFormatOptions opts;
    Node       *whereClause;
    MemoryContext copycontext;
    FmgrInfo   *out_functions;
    MemoryContext rowcontext;
    uint64      bytes_processed;
} CopyToStateData;

入力と出力

  • **入力**: COPY コマンドの構文解析結果（rel, queryDesc, opts, whereClause 等）を保持します。
  • **出力**: fe_msgbuf を介してクライアントまたはファイルへデータを転送し、bytes_processed で処理済みバイト数を記録します。

前提条件と不変条件

  • copycontext は COPY 操作全体で有効なメモリを保持し、rowcontext はタプルごとの一時的な評価に使用されます。
  • need_transcoding が真の場合、サーバーエンコーディングからファイルエンコーディングへの変換が自動的に行われます。

アルゴリズムまたは振る舞い

1. **初期化**: BeginCopyTo により構造体が構築され、出力先（ファイル/パイプ/フロントエンド）が決定されます。 2. **実行**: CopyToRoutine に定義された関数ポインタを用いて、タプルを外部形式へ変換・出力します。 3. **メモリ管理**: タプル処理ごとに rowcontext をリセットすることで、大量データ処理時のメモリリークを防止します。 4. **終了**: CopySendEndOfRow を経てバッファをフラッシュし、リソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyto.c 内の DoCopyTo 等でインスタンス化（palloc0）されます。
  • **呼び出し先**: CopyToRoutine を介して各出力形式（Text, CSV, Binary）の実装関数を呼び出します。

関連データ構造

  • CopyToRoutine: 出力形式ごとの関数ポインタをまとめたインターフェース。
  • CopyDest: 出力先（ファイル、パイプ、フロントエンド）を識別する列挙型。
  • CopyFormatOptions: COPY コマンドのオプション設定。

失敗モードと境界条件

  • is_program が真の場合、popen による外部プログラム実行の成否に依存します。
  • エンコーディング変換が必要な場合、need_transcoding の設定が不適切だとデータ破損の可能性があります。

保守担当者向けメモ

  • 新しい出力形式を追加する場合は、CopyToRoutine に実装を追加し、CopyToGetRoutine で選択可能にしてください。
  • rowcontext はタプル処理のループ内で適切にリセットされていることを確認してください。
