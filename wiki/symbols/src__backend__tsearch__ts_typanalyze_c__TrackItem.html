TrackItem

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/ts_typanalyze.c:33

目的

tsvector 型の統計情報収集（ANALYZE）において、Lossy Counting アルゴリズムを用いて語彙（lexeme）の出現頻度と誤差を追跡するためのハッシュテーブルエントリとして機能します。

シグネチャまたは形状

typedef struct {
    LexemeHashKey key;      /* 語彙の識別子 (LCアルゴリズムの 'e') */
    int           frequency; /* 出現頻度 (LCアルゴリズムの 'f') */
    int           delta;     /* 最大誤差 (LCアルゴリズムの 'delta') */
} TrackItem;

入力と出力

本構造体はデータ構造であり、関数ではありません。compute_tsvector_stats 内でハッシュテーブルの要素として生成・更新され、最終的に統計情報のソートおよび pg_statistic への格納に利用されます。

前提条件と不変条件

  • key は語彙を一意に識別します。
  • frequency は当該語彙の推定出現回数を示します。
  • delta は当該頻度推定値に含まれる最大誤差を示します。

アルゴリズムまたは振る舞い

Lossy Counting アルゴリズムに基づき、語彙の出現時に frequency を更新します。メモリ制限を維持するため、prune_lexemes_hashtable によって frequency + delta が閾値以下のエントリが定期的に削除されます。

呼び出し元と呼び出し先

  • **呼び出し元**: compute_tsvector_stats がハッシュテーブルの要素として本構造体を割り当て、hash_search を通じて操作します。また、qsort_interruptible を用いたソート処理の対象となります。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • LexemeHashKey: TrackItem のキー部分を構成する構造体。
  • HTAB (ハッシュテーブル): TrackItem を格納するコンテナ。

失敗モードと境界条件

  • メモリ不足: palloc によるメモリ確保時に発生する可能性があります。
  • ハッシュテーブルの肥大化: prune_lexemes_hashtable による剪定が不十分な場合、メモリ消費が制限を超える可能性があります。

保守担当者向けメモ

  • 本構造体は ts_typanalyze.c 内の統計計算ロジックに密結合しています。
  • qsort_interruptible を使用してソートを行う際は、TrackItem へのポインタ配列を介して操作される点に注意してください。
