xl_btree_split

定義

  • 種別: struct
  • 定義箇所: src/include/access/nbtxlog.h:153

目的

B-treeインデックスのページ分割操作をWAL（Write-Ahead Log）に記録し、クラッシュリカバリやレプリケーション時に当該操作を再実行（REDO）するための情報を保持する。

シグネチャまたは形状

typedef struct xl_btree_split
{
    uint32      level;            /* 分割対象ページのツリーレベル */
    OffsetNumber firstrightoff;   /* 右側ページにおける元のページからの最初のアイテムオフセット */
    OffsetNumber newitemoff;      /* 新規アイテムのオフセット */
    uint16      postingoff;       /* 元のPostingタプル内のオフセット */
} xl_btree_split;

入力と出力

  • **入力**: B-treeのページ分割時に、分割後の状態（レベル、アイテム配置、Postingタプルのオフセット）を本構造体に格納し、XLogInsert を介してWALへ書き出す。
  • **出力**: REDO処理時、XLogRecGetData を通じてWALレコードから本構造体を読み出し、ページ内容の復元に使用する。

前提条件と不変条件

  • 右側のページは、WALレコードから完全に復元可能である必要がある。
  • REDO処理においては、LSN（Log Sequence Number）チェックを行い、ページが既に更新済みである場合に備えて冪等性を維持すること。

アルゴリズムまたは振る舞い

1. **記録**: nbtree の挿入処理（_bt_insert 等）でページ分割が発生した際、本構造体に分割状態を記録する。 2. **再生**: クラッシュリカバリ時、btree_redo がWALレコードを読み込み、xl_info フラグに基づいて本構造体を参照する。 3. **復元**: 記録されたオフセット情報を用いて、_bt_restore_page 等の関数によりインデックスページを再構築する。

呼び出し元と呼び出し先

  • **呼び出し元（記録側）**: src/backend/access/nbtree/nbtinsert.c
  • **呼び出し先（解析・再生側）**: src/backend/access/nbtree/nbtxlog.c（REDO処理）、src/backend/access/rmgrdesc/nbtdesc.c（ログ解析・デバッグ出力）

関連データ構造

  • XLOG_BTREE_SPLIT: 本構造体に関連付けられた操作種別フラグ。
  • SizeOfBtreeSplit: 本構造体のサイズを算出するためのマクロ（offsetof(xl_btree_split, postingoff) + sizeof(uint16)）。

失敗モードと境界条件

  • ページ分割時のデータ不整合：REDO時にWALレコード内のオフセット情報が物理ページと矛盾する場合、インデックスの破損を招く可能性がある。
  • 冪等性の欠如：LSNチェックを適切に行わない場合、既に適用済みの分割操作を二重に適用し、ページ構造を破壊するリスクがある。

保守担当者向けメモ

  • 本構造体は、通常の増分更新よりも効率的なWAL記録を行うために設計されている。
  • 重複キーの圧縮（Deduplication）に関連する postingoff の扱いに注意すること。
  • 構造体のフィールド追加や変更を行う際は、SizeOfBtreeSplit マクロの定義が適切か確認すること。
