gbt_bytea_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bytea.c:98

目的

GiSTインデックス検索において、bytea型のインデックスキーが指定されたクエリ条件を満たすかどうかを判定します。

シグネチャまたは形状

Datum gbt_bytea_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY * (インデックス内のキー)
  • PG_GETARG_DATUM(1): bytea * (検索条件となる値)
  • PG_GETARG_UINT16(2): StrategyNumber (比較演算の種類)
  • PG_GETARG_POINTER(4): bool * (再チェックフラグの出力先)
  • **出力**: bool (条件に一致する場合は true、それ以外は false)

前提条件と不変条件

  • 本関数は完全一致（exact）検索を前提としており、呼び出し時に *recheck を常に false に設定します。
  • tinfo（gbtree_vinfo構造体）がモジュール内で定義されており、比較ロジックの提供に使用されます。

アルゴリズムまたは振る舞い

1. entry->key から GBT_VARKEY を取得し、gbt_var_key_readable を用いて読み取り可能な形式 GBT_VARKEY_R に変換します。 2. *recheck を false に設定します。 3. gbt_var_consistent を呼び出し、tinfo に保持された比較関数と fcinfo->flinfo を使用して、インデックスキーとクエリ条件の評価を委譲します。 4. 評価結果を PG_RETURN_BOOL で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのGiSTインデックス検索エンジン（直接の呼び出し元コードはソース抜粋に含まれていません）。
  • **呼び出し先**:
  • gbt_var_key_readable
  • gbt_var_consistent

関連データ構造

  • GISTENTRY: GiSTインデックスのキーエントリ。
  • GBT_VARKEY: bytea型をGiSTで扱うための内部キー表現。
  • tinfo: bytea型の比較関数を保持するgbtree_vinfo構造体。

失敗モードと境界条件

  • bytea型特有の比較ロジックは tinfo を介して gbt_var_consistent に依存しているため、比較関数の不整合がある場合はその挙動に従います。

保守担当者向けメモ

  • 本関数は btree_utils_var.c で提供される汎用関数へのアダプターとして機能しています。
  • 検索ロジックの変更が必要な場合は、本関数ではなく tinfo に設定されている比較関数または gbt_var_consistent の実装を確認してください。
