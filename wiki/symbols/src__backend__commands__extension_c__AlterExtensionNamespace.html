AlterExtensionNamespace

定義

  • 種別: function
  • 定義箇所: src/backend/commands/extension.c:3192

目的

拡張機能（Extension）を別のスキーマへ移動する（ALTER EXTENSION ... SET SCHEMA）ためのメインロジックを実行します。

シグネチャまたは形状

ObjectAddress AlterExtensionNamespace(const char *extensionName, const char *newschema, Oid *oldschema)

入力と出力

  • **入力**:
  • extensionName: 対象の拡張機能名。
  • newschema: 移動先のスキーマ名。
  • oldschema: 移動前のスキーマOIDを格納するポインタ（出力用）。
  • **出力**:
  • 移動した拡張機能の ObjectAddress を返します。既にターゲットスキーマにある場合は InvalidObjectAddress を返します。

前提条件と不変条件

  • 呼び出し元は拡張機能の所有者である必要があります。
  • ターゲットスキーマに対する CREATE 権限が必要です。
  • 拡張機能が extrelocatable（再配置可能）として定義されている必要があります。
  • 拡張機能が自身を含むスキーマへ移動することはできません（依存関係ループの防止）。

アルゴリズムまたは振る舞い

1. 拡張機能のOIDとターゲットスキーマのOIDを特定します。 2. 所有権および権限チェックを行い、依存関係ループを検証します。 3. pg_extension カタログから対象のタプルを取得し、既にターゲットスキーマにある場合は早期終了します。 4. pg_depend をスキャンし、当該拡張機能に依存する全オブジェクトを特定します。 5. 各依存オブジェクトに対し、no_relocate 制約がないか確認し、問題なければ AlterObjectNamespace_oid を呼び出してスキーマを移動します。 6. 全ての依存オブジェクトが正しく移動されたことを確認し、処理を完了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/alter.c:543 (ExecAlterExtensionStmt からの呼び出し)
  • **呼び出し先**:
  • get_extension_oid, LookupCreationNamespace, object_ownercheck, object_aclcheck, getExtensionOfObject, table_open, systable_beginscan, heap_copytuple, read_extension_control_file, AlterObjectNamespace_oid 等。

関連データ構造

  • Form_pg_extension: 拡張機能のメタデータ構造。
  • ObjectAddresses: 移動対象オブジェクトのリスト管理用。
  • ScanKeyData: カタログスキャン用キー。

失敗モードと境界条件

  • **権限エラー**: 所有者でない、またはターゲットスキーマへの作成権限がない場合にエラー。
  • **再配置不可**: extrelocatable が偽の場合にエラー。
  • **依存関係エラー**: 拡張機能が移動を禁止している場合、または移動先が拡張機能自身を含む場合にエラー。
  • **不整合**: 依存オブジェクトの旧スキーマが一致しない場合にエラー。

保守担当者向けメモ

  • pg_depend のスキャン中にエラーが発生した場合、それまでの変更はトランザクションによりロールバックされます。
  • no_relocate のチェックはスキャンループ内で行われるため、大規模な拡張機能ではパフォーマンスに影響する可能性があります。
