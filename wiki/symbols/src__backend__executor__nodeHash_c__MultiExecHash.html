MultiExecHash

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHash.c:104

目的

ハッシュ結合の構築フェーズにおいて、入力リレーションからメモリ内ハッシュテーブルを生成するメインエントリーポイントです。

シグネチャまたは形状

Node *MultiExecHash(HashState *node)

入力と出力

  • **入力**: HashState *node（ハッシュ結合の構築状態を保持するノード状態構造体）
  • **出力**: NULL（ハッシュテーブル自体は HashState 構造体内に保持されるため、API仕様上 NULL を返します）

前提条件と不変条件

  • HashState ノードが初期化済みであること。
  • 実行時計測（instrumentation）が有効な場合、ノードの計測が開始・停止されること。

アルゴリズムまたは振る舞い

1. **計測開始**: node->ps.instrument が設定されていれば InstrStartNode を呼び出します。 2. **構築処理の分岐**:

  • node->parallel_state が存在する場合、MultiExecParallelHash(node) を呼び出し、並列環境でのハッシュテーブル構築を行います。
  • それ以外の場合、MultiExecPrivateHash(node) を呼び出し、単一プロセスでのハッシュテーブル構築を行います。

    • 3. **計測終了**: node->ps.instrument が設定されていれば、構築された部分タプル数（node->hashtable->partialTuples）を引数に InstrStopNode を呼び出します。 4. **戻り値**: NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:525
  • **呼び出し先**:
  • MultiExecParallelHash（並列実行時）
  • MultiExecPrivateHash（非並列実行時）
  • InstrStartNode / InstrStopNode（計測有効時）

関連データ構造

  • HashState: 実行中のハッシュノードの状態を保持。
  • HashJoinTable: ハッシュ結合の全状態を保持する主要なデータ構造。

失敗モードと境界条件

  • メモリ制限（work_mem）を超過した場合、内部で ExecHashIncreaseNumBatches が呼び出され、バッチ数の増加とタプルの再パーティショニングが発生します。
  • 並列実行時には、共有メモリ（DSA）の確保やバリア同期が失敗する可能性があります。

保守担当者向けメモ

  • 本関数は MultiExecProcNode API に準拠するため、ハッシュテーブルを直接返しません。親ノードである Hashjoin が HashState を介してハッシュテーブルを取得する設計となっています。
  • 並列処理と非並列処理のコードパスが分離されているため、修正時は両方のパスへの影響を考慮してください。
