makeSublist

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginfast.c:144

目的

GINインデックスのペンディングリストへ挿入するタプル群を、物理的なページサイズに合わせて分割し、新しいサブリストとして構築する。

シグネチャまたは形状

static void makeSublist(Relation index, IndexTuple *tuples, int32 ntuples, GinMetaPageData *res)

入力と出力

  • **入力**:
  • index: 対象のGINインデックスリレーション。
  • tuples: 挿入対象の IndexTuple 配列。
  • ntuples: 配列内のタプル数。
  • **出力**:
  • res: 構築されたサブリストのメタデータ（ヘッドページ、テールページ、ページ数、空き容量、ヒープタプル数）が格納される GinMetaPageData 構造体。

前提条件と不変条件

  • ntuples > 0 であること。
  • GinNewBuffer により新しいページが割り当て可能であること。

アルゴリズムまたは振る舞い

1. 入力されたタプル群を GinListPageSize に収まるようにページ単位で分割する。 2. ページ内にタプルが収まらない場合は、新しいページを割り当て、現在のページを確定させる。 3. writeListPage を呼び出し、各ページにタプルを書き込む。 4. ページ間のリンク（次ページ番号）を維持しつつ、リストのヘッドとテールを res に記録する。 5. 最終ページを書き込み、res にテール位置と残りの空き容量を反映させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginfast.c 内の関数（ginHeapTupleFastInsert 等から参照されている）。
  • **呼び出し先**:
  • GinNewBuffer: 新規ページ割り当て。
  • writeListPage: ページへのタプル書き込み。
  • BufferGetBlockNumber: バッファのブロック番号取得。

関連データ構造

  • Relation: インデックスリレーション。
  • IndexTuple: 挿入されるデータ。
  • GinMetaPageData: ペンディングリストの管理情報。

失敗モードと境界条件

  • ntuples が0の場合、Assert により停止する。
  • ページ容量を超えるタプルが渡された場合、自動的にページ分割が行われる。
  • GinNewBuffer が失敗した場合（ディスクフル等）、例外が発生する可能性がある。

保守担当者向けメモ

  • 本関数はペンディングリストの物理的なレイアウトを決定する。
  • writeListPage が返す空き容量が res->tailFreeSize に正しく反映されるため、後続の挿入処理の効率に直結する。
  • ページ分割のロジックは GinListPageSize に依存しているため、ページサイズ変更時には注意が必要。
