ExecCustomScanEstimate

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeCustom.c:160

目的

並列クエリ実行時において、カスタムスキャンが共有メモリ（DSM）に必要とする領域サイズを見積もり、共有メモリのレイアウト計画に登録する。

シグネチャまたは形状

void ExecCustomScanEstimate(CustomScanState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: カスタムスキャンの実行状態を保持する CustomScanState 構造体。
  • pcxt: 並列クエリ実行のコンテキストを保持する ParallelContext 構造体。
  • **出力**: なし（副作用として node->pscan_len への代入および pcxt->estimator への見積もり値の追加を行う）。

前提条件と不変条件

  • node->methods が有効な CustomExecMethods を指していること。
  • 拡張機能側で EstimateDSMCustomScan コールバックが定義されている場合のみ、見積もり処理が実行される。

アルゴリズムまたは振る舞い

1. node->methods から拡張機能固有のメソッドテーブルを取得する。 2. methods->EstimateDSMCustomScan が定義されている場合、それを呼び出して必要な共有メモリサイズ（バイト数）を取得し、node->pscan_len に格納する。 3. shm_toc_estimate_chunk を呼び出し、取得したサイズ分を共有メモリのチャンクとして見積もりに加算する。 4. shm_toc_estimate_keys を呼び出し、カスタムスキャン用のキー領域（1つ分）を確保する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:275（並列実行計画の準備プロセス内）。
  • **呼び出し先**: 拡張機能が提供する CustomExecMethods 内の EstimateDSMCustomScan 関数。

関連データ構造

  • CustomScanState: カスタムスキャンの実行状態を管理する構造体。
  • ParallelContext: 並列クエリ実行の共有メモリ管理用コンテキスト。
  • CustomExecMethods: 拡張機能が実装するコールバック関数群。

失敗モードと境界条件

  • methods->EstimateDSMCustomScan が定義されていない場合、本関数は何もしない（エラーにはならない）。

保守担当者向けメモ

  • 本関数は ParallelContext の見積もりフェーズでのみ呼び出される。
  • 拡張機能側で EstimateDSMCustomScan を実装する場合、ここで見積もったサイズと InitializeDSM で実際に割り当てるサイズが一致するように注意すること。
