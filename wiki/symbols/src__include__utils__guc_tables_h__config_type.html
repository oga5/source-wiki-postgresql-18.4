config_type

定義

  • 種別: enum
  • 定義箇所: src/include/utils/guc_tables.h:23

目的

PostgreSQLのGUC（Grand Unified Configuration）システムにおいて、設定変数が保持する値のデータ型を識別するために使用されます。

シグネチャまたは形状

enum config_type
{
    PGC_BOOL,
    PGC_INT,
    PGC_REAL,
    PGC_STRING,
    PGC_ENUM,
};

入力と出力

  • 入力: GUC変数の定義時に、その変数がどの型に属するかを指定するために使用されます。
  • 出力: システム内部で変数の型を判定し、適切な処理（検証や表示）を行うための識別子として機能します。

前提条件と不変条件

  • 設定変数の型は起動時にのみ設定され、実行中に変更されることはありません。

アルゴリズムまたは振る舞い

  • GUCシステムは、この列挙型に基づいて各変数の構造体（config_bool, config_int 等）を特定し、値の読み込みや検証を行います。
  • src/backend/utils/misc/guc_tables.c において、各型に対応する表示用名称の管理に使用されています。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/include/utils/guc_tables.h:181: config_generic 構造体内の vartype フィールドとして定義。
  • src/backend/utils/misc/guc.c:4881: 変数定義の型指定に使用。
  • src/backend/utils/misc/guc_tables.c:755: 表示用名称の定義において参照。

関連データ構造

  • config_generic: 全てのGUC変数の基底構造体。vartype フィールドを通じて本列挙型を保持します。
  • config_bool, config_int, config_real, config_string, config_enum: 各型に対応する具体的な構造体定義。

失敗モードと境界条件

  • 列挙型に含まれない値を設定しようとした場合、コンパイルエラーまたはシステム定義の不整合を引き起こします。

保守担当者向けメモ

  • 新しい設定型を追加する場合は、本列挙型への追加に加え、対応する config_xxx 構造体の定義と、guc.c 内の処理ロジック（検証・表示関数）の更新が必要です。
