check_agg_arguments_walker

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_agg.c:750

目的

集約関数の引数式を走査し、変数、集約関数、CTE参照のネストレベルを追跡するとともに、集約関数の引数として許可されない式（集合を返す関数やウィンドウ関数）を検出・検証する。

シグネチャまたは形状

static bool check_agg_arguments_walker(Node *node, check_agg_arguments_context *context)

入力と出力

  • **入力**:
  • node: 走査対象の構文木ノード。
  • context: 走査状態（sublevels_up、最小レベルの追跡用変数など）を保持する check_agg_arguments_context 構造体。
  • **出力**:
  • 戻り値: 走査を継続する場合は false、停止する場合は true。

前提条件と不変条件

  • context->sublevels_up は現在のクエリレベルからの相対的な深さを表す。
  • Var, Aggref, GroupingFunc, RangeTblEntry の各ノードにおいて、sublevels_up を考慮したレベル計算が正しく行われること。

アルゴリズムまたは振る舞い

1. **ノード種別ごとの処理**:

  • Var, Aggref, GroupingFunc: 現在の sublevels_up を加味してレベルを正規化し、context 内の最小レベル値を更新する。
  • RangeTblEntry (RTE_CTE): CTEのレベルを正規化し、最小レベルと対象CTEを context に記録する。
  • Query: sublevels_up をインクリメントして再帰的に query_tree_walker を呼び出し、終了後にデクリメントする。

    • 2. **検証ロジック**:

  • context->sublevels_up == 0 の場合、集約関数の引数内に集合を返す関数（FuncExpr, OpExpr）やウィンドウ関数が含まれていればエラーを投げる。

    • 3. **再帰**:

  • 上記以外のノードについては expression_tree_walker を使用してツリーを走査する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_agg.c 内の関数（check_agg_arguments 等から引数やフィルタ式に対して呼び出される）。
  • **呼び出し先**:
  • query_tree_walker
  • expression_tree_walker
  • ereport（エラー発生時）

関連データ構造

  • check_agg_arguments_context: 走査中の状態を保持する構造体。
  • Var, Aggref, GroupingFunc, WindowFunc, RangeTblEntry, Query: 走査対象となるノード群。

失敗モードと境界条件

  • 集約関数の引数内に集合を返す関数が含まれる場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラー。
  • 集約関数の引数内にウィンドウ関数が含まれる場合、ERRCODE_GROUPING_ERROR でエラー。

保守担当者向けメモ

  • sublevels_up の管理は、サブクエリのネストを正しく扱うために不可欠である。変更を加える際は、レベル計算のオフセットが適切か確認すること。
  • expression_tree_walker を利用しているため、新しいノードタイプを追加した場合はこの関数でのハンドリングが必要になる可能性がある。
