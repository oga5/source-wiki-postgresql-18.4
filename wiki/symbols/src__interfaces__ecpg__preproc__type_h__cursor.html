cursor

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/preproc/type.h:137

目的

ECPGプリプロセッサにおいて、SQLカーソルの状態および定義情報を保持するための構造体です。

シグネチャまたは形状

struct cursor
{
    char       *name;
    char       *function;
    char       *command;
    char       *connection;
    bool        opened;
    struct arguments *argsinsert;
    struct arguments *argsinsert_oos;
    struct arguments *argsresult;
    struct arguments *argsresult_oos;
    struct cursor *next;
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。各フィールドはカーソルの識別子（name）、実行されるSQL（command）、接続先（connection）、および入出力引数のリストを保持します。

前提条件と不変条件

  • next フィールドにより、カーソル定義の連結リストを構成します。
  • opened フラグは、カーソルの現在のオープン状態を示します。

アルゴリズムまたは振る舞い

プリプロセッサがソースコードを解析する際、カーソルの宣言や操作を追跡するために使用されます。argsinsert や argsresult などのフィールドを通じて、カーソルに関連付けられたホスト変数の引数リストを管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは、具体的な呼び出し元関数は特定できません。
  • **呼び出し先**: 構造体内部で struct arguments を参照しています。
  • **注記**: contrib/dblink/dblink.c 等で「cursor」という用語が使用されていますが、これらは本構造体への直接的な参照であるという証拠はありません。

関連データ構造

  • struct arguments: カーソルの入出力引数を管理するために使用されます。

失敗モードと境界条件

  • メモリ管理: 本構造体は動的に確保されることが想定されており、適切な解放処理が必要です。
  • 連結リストの終端: next が NULL であることでリストの終端を示します。

保守担当者向けメモ

  • 本構造体は再帰的なリスト構造の一部として機能します。解析時には next ポインタを辿る処理が必要です。
  • _oos サフィックスを持つフィールド（argsinsert_oos 等）の具体的な役割については、type.c の実装を参照してください。
