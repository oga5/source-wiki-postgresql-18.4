spgPrepareScanKeys

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgscan.c:207

目的

SP-GiST インデックススキャンにおいて、実行エンジンから渡された検索条件（Quals）および順序付け条件（Order-by）を解析し、スキャン状態（SpGistScanOpaque）に最適化された形式で格納する。

シグネチャまたは形状

static void spgPrepareScanKeys(IndexScanDesc scan)

入力と出力

  • **入力**: IndexScanDesc scan（スキャン記述子。keyData および orderByData を含む）。
  • **出力**: なし（副作用として scan->opaque が指す SpGistScanOpaque 構造体の各フィールドを更新する）。

前提条件と不変条件

  • scan->opaque は SpGistScanOpaque 型としてキャスト可能であること。
  • scan->numberOfOrderBys が 0 以下の場合、numberOfNonNullOrderBys は 0 に設定される。
  • IS NULL 条件と他の条件が混在する場合、スキャンは「不満足（unsatisfiable）」と判定され、検索対象は空になる。

アルゴリズムまたは振る舞い

1. **Order-by の処理**:

  • NULL 値を含む Order-by キーを除外し、非 NULL キーのみを so->orderByData に詰め直す。
  • 各キーの元のインデックスと新しいインデックスの対応を so->nonNullOrderByOffsets に記録する。

    • 2. **Quals の解析**:

  • scan->keyData を走査し、SK_SEARCHNULL（IS NULL）および SK_SEARCHNOTNULL（IS NOT NULL）フラグを抽出する。
  • 通常の検索条件は so->keyData にコピーし、haveNotNull を真とする。

    • 3. **整合性チェック**:

  • IS NULL と IS NOT NULL（または通常条件）が同時に存在する場合、qual_ok を false に設定する。

    • 4. **結果の反映**:

  • 解析結果に基づき、so->searchNulls、so->searchNonNulls、so->numberOfKeys を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • spgbeginscan (src/backend/access/spgist/spgscan.c:208)
  • spgrescan (src/backend/access/spgist/spgscan.c:417)
  • **呼び出し先**:
  • 外部関数呼び出しなし（構造体フィールドへの代入およびループ処理のみ）。

関連データ構造

  • IndexScanDesc: 実行エンジンから渡されるスキャン情報。
  • SpGistScanOpaque: SP-GiST 固有のスキャン状態保持構造体。

失敗モードと境界条件

  • **検索不能条件**: IS NULL と他の条件が同時に指定された場合、qual_ok が false となり、スキャンは何も返さない状態（numberOfKeys = 0）に設定される。
  • **空の Quals**: numberOfKeys <= 0 の場合、全件走査（searchNulls = true, searchNonNulls = true）として処理される。

保守担当者向けメモ

  • 本関数はスキャン開始時および再スキャン時に呼び出されるため、メモリコンテキストの整合性に注意すること。
  • nonNullOrderByOffsets 配列のサイズが scan->numberOfOrderBys 分確保されていることを前提としている。
