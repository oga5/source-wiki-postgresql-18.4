mic_to_iso

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/cyrillic_and_mic/cyrillic_and_mic.c:357

目的

MULE_INTERNAL エンコーディングから ISO-8859-5 エンコーディングへ文字列を変換する。

シグネチャまたは形状

Datum mic_to_iso(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を取得
  • src: 変換元文字列（PG_GETARG_CSTRING(2)）
  • dest: 変換先バッファ（PG_GETARG_CSTRING(3)）
  • len: 変換対象の長さ（PG_GETARG_INT32(4)）
  • noError: エラーハンドリングフラグ（PG_GETARG_BOOL(5)）
  • **出力**: 変換されたバイト数を int32 として返す（PG_RETURN_INT32）

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_MULE_INTERNAL, PG_ISO_8859_5) により、ソースが MULE_INTERNAL、宛先が ISO-8859-5 であることが検証される。

アルゴリズムまたは振る舞い

1. 引数からソース、宛先、長さ、エラーフラグを取得する。 2. mic2latin_with_table 関数を呼び出し、変換を実行する。

  • 変換テーブルとして koi2iso を使用。
  • 内部的な文字セットとして LC_KOI8_R を指定。

    • 3. 変換されたバイト数を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1(mic_to_iso) マクロにより登録）。
  • **呼び出し先**: mic2latin_with_table（変換処理の実行）。

関連データ構造

  • koi2iso: 変換に使用される static const unsigned char 型のマッピングテーブル。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、PostgreSQL のエラー処理メカニズムにより中断される。
  • noError フラグが false の場合、変換不能な文字が含まれるとエラーが発生する可能性がある（mic2latin_with_table の仕様に依存）。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_ARGS を引数に取るため、PostgreSQL の関数呼び出し規約に従って利用すること。
  • 変換ロジックを変更する場合は、koi2iso テーブルの整合性を確認すること。
