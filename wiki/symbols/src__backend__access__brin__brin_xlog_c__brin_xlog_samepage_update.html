brin_xlog_samepage_update

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_xlog.c:169

目的

BRINインデックスの同一ページ内におけるタプル更新操作のWALリプレイ（Redo）を実行します。

シグネチャまたは形状

static void brin_xlog_samepage_update(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）。
  • **出力**: なし。副作用としてバッファの更新およびLSNの更新を行います。

前提条件と不変条件

  • record から取得される xl_brin_samepage_update 構造体が、有効なオフセット番号（offnum）を含んでいる必要があります。
  • 対象ページが XLogReadBufferForRedo によって正常に読み込まれている必要があります。

アルゴリズムまたは振る舞い

1. XLogRecGetData を介してWALレコードから xl_brin_samepage_update データを取得します。 2. XLogReadBufferForRedo を呼び出し、対象ページをバッファに読み込みます。 3. BLK_NEEDS_REDO が返された場合のみ、以下の処理を実行します。

  • XLogRecGetBlockData を使用して、更新用の新しいタプルデータを取得します。
  • PageIndexTupleOverwrite を呼び出し、指定されたオフセット番号のタプルを新しいデータで上書きします。失敗した場合は PANIC で停止します。
  • PageSetLSN でページのLSNを更新し、MarkBufferDirty でバッファをダーティ状態にします。

    • 4. バッファが有効な場合、UnlockReleaseBuffer を呼び出してロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: brin_redo（src/backend/access/brin/brin_xlog.c:325 での呼び出しを確認）。
  • **呼び出し先**: XLogRecGetData, XLogReadBufferForRedo, XLogRecGetBlockData, BufferGetPage, PageIndexTupleOverwrite, PageSetLSN, MarkBufferDirty, BufferIsValid, UnlockReleaseBuffer, elog。

関連データ構造

  • xl_brin_samepage_update: 更新対象のオフセット番号を保持するWALレコード構造体。
  • BrinTuple: ページ内に格納されるBRINインデックスのタプル形式。

失敗モードと境界条件

  • PageIndexTupleOverwrite が失敗した場合、インデックスの整合性が損なわれたと判断し PANIC を発生させます。
  • FSM（空き領域マップ）の更新は本関数内では行われません。

保守担当者向けメモ

  • 本関数はページ内のタプル置換のみを扱います。REVMAPの更新が必要な場合は、別のWALレコードタイプおよび処理関数を使用してください。
