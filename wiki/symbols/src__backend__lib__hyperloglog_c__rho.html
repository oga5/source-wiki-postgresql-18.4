rho

定義

  • 種別: static inline 関数
  • 定義箇所: src/backend/lib/hyperloglog.c:241

目的

ハッシュ値の先頭から最初のビットが立つ位置（ランク）を特定し、HyperLogLog アルゴリズムにおけるレジスタ更新用の値を算出する。

シグネチャまたは形状

static inline uint8 rho(uint32 x, uint8 b)

入力と出力

  • **入力**:
  • x (uint32): ランク計算対象のハッシュ値。
  • b (uint8): 許容されるランクの最大値（ビット幅に関連する閾値）。
  • **出力**:
  • uint8: 最初のビット位置、または x が 0 の場合や位置が b を超える場合は b + 1 を返す。

前提条件と不変条件

  • x が 0 の場合、常に b + 1 を返す。
  • 戻り値は常に 1 から b + 1 の範囲内に収まる。

アルゴリズムまたは振る舞い

1. x が 0 の場合、直ちに b + 1 を返す。 2. pg_leftmost_one_pos32(x) を使用して、最上位ビットの位置を特定する。 3. 32 からその位置を引くことで、最初のビットが立つまでのゼロの数（ランク）を算出する。 4. 算出されたランク j が b を超える場合、b + 1 を返す。 5. それ以外の場合は j を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/lib/hyperloglog.c 内の addHyperLogLog 関数から呼び出される。
  • **呼び出し先**: port/pg_bitutils.h で定義されている pg_leftmost_one_pos32 を利用する。

関連データ構造

  • hyperLogLogState: 本関数が算出するランク値を用いてレジスタ配列が更新される。

失敗モードと境界条件

  • x = 0: 戻り値 b + 1 となる。
  • j > b: ランクが閾値を超えた場合、上限値 b + 1 に切り詰められる。

保守担当者向けメモ

  • 本関数は Hideaki Ohno 氏の C++ 実装に基づく。
  • pg_leftmost_one_pos32 に依存しているため、プラットフォーム固有のビット操作最適化が適用される。
