GISTDeletedPageContents

定義

  • 種別: struct
  • 定義箇所: src/include/access/gist.h:200

目的

GiSTインデックスにおいて、削除されたページがスキャン処理から参照可能であった最後のトランザクションID（deleteXid）を保持するために使用されます。

シグネチャまたは形状

typedef struct GISTDeletedPageContents
{
    FullTransactionId deleteXid;
} GISTDeletedPageContents;

入力と出力

  • **入力**: ページ削除時に deleteXid が設定されます。
  • **出力**: ページスキャン時に、当該ページが参照可能か判定するために deleteXid が読み取られます。

前提条件と不変条件

  • ページが削除状態にある場合、ページヘッダーの pd_lower は MAXALIGN(SizeOfPageHeaderData) + sizeof(GISTDeletedPageContents) に設定されます。
  • deleteXid は FullTransactionId 型であり、並行スキャンにおける可視性判定の基準となります。

アルゴリズムまたは振る舞い

1. **書き込み**: ページ削除時、PageGetContents(page) を介してこの構造体にアクセスし、deleteXid を更新します。 2. **読み取り**: スキャン処理において、PageGetContents(page) をキャストして deleteXid を取得し、現在のトランザクション状態と比較することで、削除済みページへのアクセス可否を判断します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/gist.h 内のインライン関数（GistPageSetDeleted, GistPageGetDeleteXid 等）がこの構造体を操作します。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • FullTransactionId: 削除トランザクションIDを保持する型。
  • PageHeader: ページレイアウトの管理に使用。

失敗モードと境界条件

  • ページが削除済みとしてマークされていない状態でこの構造体を参照した場合、メモリ上の不正な領域を読み取る可能性があります。
  • PageGetContents が返すポインタが正しく初期化されていない場合、deleteXid の値は未定義となります。

保守担当者向けメモ

  • 本構造体は PageGetContents を通じてページ末尾の領域に配置されるため、構造体のサイズ変更はページレイアウトに直接影響します。
  • offsetof(GISTDeletedPageContents, deleteXid) を使用したオフセット計算が行われているため、メンバの追加や順序変更には注意が必要です。
