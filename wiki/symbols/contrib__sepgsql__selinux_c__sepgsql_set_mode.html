sepgsql_set_mode

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/selinux.c:633

目的

SE-PostgreSQL の動作モード（sepgsql_mode）を更新し、変更前のモードを呼び出し元へ返却する。

シグネチャまたは形状

int sepgsql_set_mode(int new_mode)

入力と出力

  • **入力**: new_mode (設定する新しい動作モードを表す整数値)
  • **出力**: 変更前の sepgsql_mode の値（整数）

前提条件と不変条件

  • sepgsql_mode はモジュール内でグローバルな動作状態を保持する変数である。
  • 本関数は、呼び出し時に sepgsql_mode の値を new_mode で上書きする。

アルゴリズムまたは振る舞い

1. 現在の sepgsql_mode の値をローカル変数 old_mode に退避する。 2. グローバル変数 sepgsql_mode に引数 new_mode を代入する。 3. 退避しておいた old_mode を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/hooks.c:421
  • contrib/sepgsql/label.c:255
  • contrib/sepgsql/label.c:257
  • **呼び出し先**:
  • 本関数は他の関数を呼び出さない。

関連データ構造

  • sepgsql_mode: モジュール内で定義された、現在の動作モードを保持する変数。

失敗モードと境界条件

  • 特筆すべき失敗モードはない。引数 new_mode の妥当性チェックは本関数内では行われない。

保守担当者向けメモ

  • 本関数は単なるセッターであり、モード変更に伴う副作用（キャッシュのクリアや再初期化など）は含まれていない。モード変更時に追加の処理が必要な場合は、呼び出し元で制御するか、本関数を拡張する必要がある。
