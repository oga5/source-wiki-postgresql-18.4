CreateExtension

定義

  • 種別: function
  • 定義箇所: src/backend/commands/extension.c:2093

目的

SQLコマンド CREATE EXTENSION を処理し、PostgreSQL拡張機能のインストールプロセスを開始する。

シグネチャまたは形状

ObjectAddress CreateExtension(ParseState *pstate, CreateExtensionStmt *stmt)

入力と出力

  • **入力**:
  • pstate: パーサーの状態。
  • stmt: CREATE EXTENSION 文の構文解析結果（拡張機能名、オプション等）。
  • **出力**:
  • ObjectAddress: 作成された拡張機能のオブジェクト識別子。
  • 失敗時はエラーをスローする。

前提条件と不変条件

  • グローバル変数 creating_extension が true の場合、ネストされた CREATE EXTENSION は許可されずエラーとなる。
  • 拡張機能の作成は一度に一つのみ可能である。
  • 拡張機能名の妥当性は、ファイルシステムへのアクセス前に check_valid_extension_name で検証される。

アルゴリズムまたは振る舞い

1. check_valid_extension_name を呼び出し、拡張機能名の妥当性を確認する。 2. get_extension_oid を使用して既存の拡張機能を確認する。重複がある場合、IF NOT EXISTS が指定されていれば通知を出して終了し、そうでなければエラーをスローする。 3. creating_extension フラグを確認し、ネストされた呼び出しを禁止する。 4. stmt->options を走査し、schema、new_version、cascade の各オプションを解析する。 5. 解析したパラメータを引数として CreateExtensionInternal を呼び出し、実際のインストール処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c:1577 (ProcessUtility 経由)
  • **呼び出し先**:
  • check_valid_extension_name
  • get_extension_oid
  • errorConflictingDefElem
  • defGetString / defGetBoolean
  • CreateExtensionInternal

関連データ構造

  • CreateExtensionStmt: CREATE EXTENSION 文の構文解析ツリー。
  • DefElem: 文のオプション（schema, new_version, cascade）を保持する構造体。
  • ObjectAddress: 作成されたオブジェクトの識別情報。

失敗モードと境界条件

  • **重複エラー**: 同名の拡張機能が既に存在する場合（IF NOT EXISTS なし）。
  • **ネストエラー**: 拡張機能のインストール中に別の CREATE EXTENSION が呼び出された場合。
  • **オプションエラー**: 不明なオプションが指定された場合、または重複して指定された場合。

保守担当者向けメモ

  • 本関数はフロントエンド的な役割であり、実際のカタログ操作やスクリプト実行は CreateExtensionInternal に委譲されている。
  • creating_extension フラグは、インストール中に作成されるオブジェクトを拡張機能のメンバとして自動的に紐付けるための重要な不変条件である。
