expand_dbname_patterns

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dumpall.c:1574

目的

データベース名のパターンリストを解決し、一致する実際のデータベース名の一覧を生成します。

シグネチャまたは形状

static void expand_dbname_patterns(PGconn *conn, SimpleStringList *patterns, SimpleStringList *names);

入力と出力

  • **入力**:
  • conn: データベース接続ハンドル。
  • patterns: 検索対象のデータベース名パターンを保持する SimpleStringList。
  • **出力**:
  • names: 解決されたデータベース名が追加される SimpleStringList。

前提条件と不変条件

  • patterns が空（head == NULL）の場合、関数は何もせず即座に復帰します。
  • データベース名のパターンにはドット（.）を含めることはできません。

アルゴリズムまたは振る舞い

1. patterns リスト内の各パターンをループ処理します。 2. pg_catalog.pg_database に対して SELECT datname を実行し、processSQLNamePattern を使用してパターンマッチングを行います。 3. マッチしたデータベース名を names リストに追加します。 4. 処理の過程で PQExpBuffer を使用してクエリを構築し、各ループの終了時にバッファをリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dumpall.c 内の複数箇所（行76, 544, 1575）。
  • **呼び出し先**: createPQExpBuffer, appendPQExpBufferStr, processSQLNamePattern, executeQuery, PQntuples, simple_string_list_append, PQgetvalue, PQclear, resetPQExpBuffer, destroyPQExpBuffer, pg_log_error, PQfinish, exit_nicely。

関連データ構造

  • PGconn: PostgreSQL 接続オブジェクト。
  • SimpleStringList: 文字列のリストを管理する構造体。
  • PQExpBuffer: SQL クエリ構築用のバッファ。

失敗モードと境界条件

  • パターンにドットが含まれている場合（dotcnt > 0）、pg_log_error を出力し、PQfinish で接続を閉じた後、exit_nicely(1) でプロセスを終了します。

保守担当者向けメモ

  • 本関数は、重複するデータベース名がリストに追加されることを許容しています。これは、呼び出し側が後続の処理でメンバーシップテストを行うことを前提としているためです。
