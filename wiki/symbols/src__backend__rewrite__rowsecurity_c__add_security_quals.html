add_security_quals

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rowsecurity.c:699

目的

行レベルセキュリティ（RLS）ポリシーに基づき、クエリのリライト時に適用すべき securityQuals（読み取り制限条件）を生成し、クエリツリーに統合する。

シグネチャまたは形状

static void add_security_quals(int rt_index,
                               List *permissive_policies,
                               List *restrictive_policies,
                               List **securityQuals,
                               bool *hasSubLinks)

入力と出力

  • rt_index: 対象リレーションのレンジテーブルインデックス。
  • permissive_policies: 適用可能な Permissive ポリシーのリスト。
  • restrictive_policies: 適用可能な Restrictive ポリシーのリスト。
  • securityQuals (入出力): 生成されたセキュリティ制約が追加されるリスト。
  • hasSubLinks (入出力): ポリシー内にサブリンクが含まれる場合に true に設定されるフラグ。

前提条件と不変条件

  • permissive_policies が空の場合、行は一切表示されないものとみなされ、常に false を返す制約が生成される。
  • permissive_policies が存在する場合、Permissive ポリシーは OR で結合され、Restrictive ポリシーは個別に AND 条件として追加される。
  • ChangeVarNodes を使用して、ポリシー内の変数ノードを対象リレーションの rt_index に適合させる必要がある。

アルゴリズムまたは振る舞い

1. **Permissive ポリシーの収集**: permissive_policies を走査し、各ポリシーの qual を抽出して permissive_quals リストを作成する。 2. **Restrictive ポリシーの適用**: permissive_policies が存在する場合、restrictive_policies を走査し、各 qual を securityQuals に追加する。 3. **Permissive ポリシーの結合**: permissive_quals を OR 条件で結合し、単一の式として securityQuals に追加する。 4. **デフォルト拒否**: permissive_policies が空の場合、false を表す定数式を securityQuals に追加する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/rewrite/rowsecurity.c 内の get_row_security_policies 等から呼び出される。
  • 呼び出し先: copyObject, lappend, list_append_unique, ChangeVarNodes, makeBoolExpr, makeConst 等の PostgreSQL 内部ユーティリティ関数。

関連データ構造

  • RowSecurityPolicy: ポリシー定義を保持する構造体。
  • List: ポリシーや制約を管理するリスト構造。
  • Expr: セキュリティ制約を表す式ノード。

失敗モードと境界条件

  • ポリシーが一つも存在しない（かつ Permissive がない）場合、アクセスを完全に遮断する false 制約が生成される。
  • qual が NULL のポリシーは無視される。

保守担当者向けメモ

  • 本関数は USING 句の処理を担当する。WITH CHECK 句の処理については add_with_check_options を参照すること。
  • ChangeVarNodes によるノード書き換えは、クエリツリーの整合性を保つために不可欠である。
