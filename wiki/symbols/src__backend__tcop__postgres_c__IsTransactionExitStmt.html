IsTransactionExitStmt

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/postgres.c:2857

目的

与えられた構文解析木（Node *parsetree）が、トランザクションを終了させるコマンド（COMMIT, PREPARE, ROLLBACK 等）であるかどうかを判定する。

シグネチャまたは形状

static bool IsTransactionExitStmt(Node *parsetree)

入力と出力

  • **入力**: Node *parsetree（解析対象のノード）
  • **出力**: bool（トランザクション終了コマンドであれば true、それ以外は false）

前提条件と不変条件

  • parsetree が NULL の場合、false を返す。
  • parsetree が TransactionStmt 型でない場合、false を返す。

アルゴリズムまたは振る舞い

1. 引数 parsetree が有効かつ TransactionStmt 型であることを確認する。 2. TransactionStmt の kind フィールドを評価する。 3. kind が以下のいずれかである場合に true を返す。

  • TRANS_STMT_COMMIT
  • TRANS_STMT_PREPARE
  • TRANS_STMT_ROLLBACK
  • TRANS_STMT_ROLLBACK_TO

    • 4. 上記以外、またはノード型が一致しない場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/postgres.c 内の複数箇所（行 1135, 1492, 1743, 2882）で、クエリ実行フローにおけるトランザクション制御の判断に使用されている。
  • **呼び出し先**: IsA マクロ（ノード型判定）。

関連データ構造

  • Node: PostgreSQL の構文解析木の基本構造体。
  • TransactionStmt: トランザクション制御コマンドを表すノード構造体。

失敗モードと境界条件

  • parsetree が NULL の場合、安全に false を返す。
  • TransactionStmt 以外のノードが渡された場合、型チェックにより安全に false を返す。

保守担当者向けメモ

  • 本関数は postgres.c 内でのみ使用される static 関数である。
  • トランザクション終了コマンドの定義に変更があった場合（新しい終了コマンドの追加など）、本関数の条件式を更新する必要がある。
