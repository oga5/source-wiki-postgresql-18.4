r_postlude

目的

ドイツ語のステミング処理の最終段階として、単語に対して文字の置換や削除といった正規化処理を適用する。

シグネチャまたは形状

static int r_postlude(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z （ステミング環境を保持する構造体）
  • 出力: 成功時に 1 を返し、slice_from_s 等の内部処理でエラーが発生した場合は負の値を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • find_among 関数が使用する a_1 構造体が定義されていること。

アルゴリズムまたは振る舞い

1. 無限ループ内で find_among を呼び出し、現在のカーソル位置から a_1 に定義されたパターンを検索する。 2. マッチした among_var に応じて以下の処理を行う：

  • ケース 1〜4: slice_from_s を用いて、マッチした部分を特定の文字列に置換する。
  • ケース 5: 現在の文字をスキップする（z->c++）。

    • 3. マッチしない場合は lab0 にジャンプし、ループを終了して 1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: german_ISO_8859_1_stem （src/backend/snowball/libstemmer/stem_ISO_8859_1_german.c 内）
  • 呼び出し先: find_among, slice_from_s （Snowball ランタイム関数）

関連データ構造

  • struct SN_env: ステミングの実行状態（カーソル位置、文字列バッファ等）を管理する。
  • a_1: r_postlude 内で検索対象となる among 構造体の配列。

失敗モードと境界条件

  • slice_from_s が失敗した場合、その戻り値（負の値）をそのまま呼び出し元に伝播させる。
  • 文字列の末尾に達した場合、z->c >= z->l のチェックによりループを抜ける。
