free_token_error

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq-oauth/oauth-curl.c:175

目的

OAuth 2.0 トークンリクエスト時に発生したエラー情報を保持する token_error 構造体のメンバーが動的に確保したメモリを解放する。

シグネチャまたは形状

static void free_token_error(struct token_error *err)

入力と出力

  • 入力: struct token_error *err - 解放対象の token_error 構造体へのポインタ。
  • 出力: なし（void）。

前提条件と不変条件

  • 引数 err は有効な token_error 構造体を指している必要がある。
  • err->error および err->error_description は、malloc 等で確保されたメモリ領域であるか、あるいは NULL である必要がある（free の仕様に準拠）。

アルゴリズムまたは振る舞い

1. err->error を解放する。 2. err->error_description を解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/interfaces/libpq-oauth/oauth-curl.c 内の各所（tok->err の解放や、エラーハンドリング時のローカル変数 err の解放に使用）。
  • 呼び出し先:
  • 標準ライブラリの free()。

関連データ構造

  • struct token_error: OAuth 2.0 のエラーレスポンス（error および error_description フィールド）を保持する構造体。

失敗モードと境界条件

  • err が NULL の場合、呼び出し元で適切に制御されていない限り未定義動作となる可能性がある。
  • 各フィールドが NULL の場合、free() は何もしないため安全である。

保守担当者向けメモ

  • 本関数は token_error 構造体自体のメモリは解放しない。構造体自体を解放する前に、内部の文字列ポインタをクリーンアップするために使用すること。
