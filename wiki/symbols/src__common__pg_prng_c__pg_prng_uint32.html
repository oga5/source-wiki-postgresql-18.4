pg_prng_uint32

定義

  • **種別**: 関数
  • **定義箇所**: src/common/pg_prng.c:226

目的

擬似乱数生成器（PRNG）の状態から、32ビットの符号なし整数を生成して返します。

シグネチャまたは形状

uint32 pg_prng_uint32(pg_prng_state *state)

入力と出力

  • **入力**: pg_prng_state *state（乱数生成器の状態を保持する構造体へのポインタ）
  • **出力**: uint32（生成された32ビットの擬似乱数値）

前提条件と不変条件

  • state は初期化済みである必要があります。
  • 内部で使用される xoroshiro128ss アルゴリズムの特性上、状態がすべて0になると0を返し続けるため、初期化時に pg_prng_seed_check 等でガードされていることが前提となります。

アルゴリズムまたは振る舞い

1. 内部関数 xoroshiro128ss(state) を呼び出し、64ビットの乱数値を生成します。 2. 生成された64ビット値の上位32ビットを抽出（右シフト）して返します。

  • 低位ビットのランダム性に対する懸念を回避するため、上位ビットを使用する設計となっています。

呼び出し元と呼び出し先

  • **呼び出し先**: xoroshiro128ss
  • **呼び出し元**:
  • src/backend/access/heap/vacuumlazy.c
  • src/backend/access/nbtree/nbtinsert.c
  • src/backend/commands/analyze.c
  • src/backend/executor/nodeSamplescan.c
  • src/backend/postmaster/postmaster.c
  • src/backend/storage/ipc/dsm.c
  • src/backend/utils/cache/catcache.c
  • src/backend/utils/misc/sampling.c

関連データ構造

  • pg_prng_state: 乱数生成器の状態を保持する構造体。

失敗モードと境界条件

  • 本関数自体はエラーを返しません。
  • 状態がすべて0の初期値で初期化された場合、一貫して0を返します。

保守担当者向けメモ

  • 本関数は暗号学的に安全ではありません。パスワードや暗号鍵の生成には使用しないでください。
  • 64ビットの生成結果から上位32ビットのみを抽出しているため、生成される値の範囲は 0 から UINT32_MAX です。
