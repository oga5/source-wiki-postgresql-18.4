dropstate

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_nfa.c:225

目的

NFA（非決定性有限オートマトン）から特定の状態を削除し、その状態に関連するすべての遷移（入力および出力アーク）を解放する。

シグネチャまたは形状

static void dropstate(struct nfa *nfa, struct state *s)

入力と出力

  • nfa: 状態が所属する nfa 構造体へのポインタ。
  • s: 削除対象の state 構造体へのポインタ。
  • 出力: なし（void）。

前提条件と不変条件

  • nfa および s は有効なポインタであること。
  • freearc および freestate が内部で呼び出されるため、これらが nfa のメモリ管理ルールに従って動作可能であること。

アルゴリズムまたは振る舞い

1. 状態 s の入力アークリスト（s->ins）が空になるまで、freearc を呼び出して各アークを解放する。 2. 状態 s の出力アークリスト（s->outs）が空になるまで、freearc を呼び出して各アークを解放する。 3. 最後に freestate を呼び出し、状態 s 自体を解放する。

呼び出し元と呼び出し先

  • 呼び出し先: freearc（アーク解放用）、freestate（状態解放用）。
  • 呼び出し元: src/backend/regex/regc_nfa.c 内の各最適化処理（行 1675, 1846, 2107, 2127, 2280, 2404, 2442, 2848, 2981）および src/backend/regex/regcomp.c:149 で宣言。

関連データ構造

  • struct nfa: NFA 全体を管理する構造体。
  • struct state: グラフのノードを表現する構造体。
  • struct arc: 状態間の遷移を表現する構造体。

失敗モードと境界条件

  • 状態 s が既に解放済みである場合や、無効なポインタである場合は未定義の動作を引き起こす。
  • freearc がアークリストを正しく更新しない場合、無限ループが発生する可能性がある。

保守担当者向けメモ

  • 本関数は状態を削除する際の標準的なクリーンアップ手順である。状態を削除する際は、孤立したアークを残さないよう必ず本関数を経由すること。
