acquire_inherited_sample_rows

定義

  • 種別: function
  • 定義箇所: src/backend/commands/analyze.c:1390

目的

継承ツリー（親テーブルおよびその子テーブル）全体から統計情報生成用のサンプル行を収集する。

シグネチャまたは形状

static int acquire_inherited_sample_rows(
    Relation onerel, 
    int elevel,
    HeapTuple *rows, 
    int targrows,
    double *totalrows, 
    double *totaldeadrows
)

入力と出力

  • **入力**:
  • onerel: 対象となる親リレーション。
  • elevel: エラー報告レベル。
  • targrows: 収集対象のサンプル行数。
  • **出力**:
  • rows: 収集されたサンプル行が格納される配列。
  • totalrows: 継承ツリー全体の推定総行数。
  • totaldeadrows: 継承ツリー全体の推定死滅行数。
  • **戻り値**: 実際に収集されたサンプル行数。

前提条件と不変条件

  • 呼び出し元で ShareUpdateExclusiveLock を保持している必要がある。
  • find_all_inheritors を使用して継承関係を特定する。
  • 継承ツリーに子テーブルが存在しない場合、エラー報告を行い 0 を返す。

アルゴリズムまたは振る舞い

1. find_all_inheritors で全子テーブルの OID を取得する。 2. 各子テーブルを順次開き、RELKIND_RELATION または RELKIND_MATVIEW であれば acquire_sample_rows を、外部テーブルであれば FDW の AnalyzeForeignTable フックを使用してサンプリング関数を特定する。 3. 収集対象外のテーブル（一時テーブルやパーティションテーブル等）はスキップする。 4. 各リレーションのブロック数に基づいて、全体に対するサンプリング比率を考慮しつつ行を収集する。 5. 処理中、pgstat_progress_update_multi_param を呼び出し、進捗状況（現在のテーブル、処理済みブロック数等）を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: do_analyze_rel (src/backend/commands/analyze.c)
  • **呼び出し先**:
  • find_all_inheritors
  • table_open / table_close
  • acquire_sample_rows (直接参照)
  • GetFdwRoutineForRelation
  • pgstat_progress_update_multi_param

関連データ構造

  • Relation: リレーション記述子。
  • AcquireSampleRowsFunc: サンプリング関数への関数ポインタ型。
  • FdwRoutine: 外部テーブルの解析用ルーチン。

失敗モードと境界条件

  • 子テーブルが存在しない場合、SetRelationHasSubclass(..., false) を実行し、エラー報告を行って終了する。
  • 外部テーブルで AnalyzeForeignTable が定義されていない、または解析をサポートしない場合はそのテーブルをスキップする。
  • 継承ツリーに解析可能なテーブルが一つも存在しない場合、エラー報告を行い 0 を返す。

保守担当者向けメモ

  • サンプリングは各リレーションのブロック数に比例して行われる。リレーション間で空き領域率（fillfactor）が大きく異なる場合、サンプリングの偏りが生じる可能性がある。
  • totalrows および totaldeadrows は、早期終了時にも 0 で初期化されるよう設計されている。
