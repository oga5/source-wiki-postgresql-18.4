int32KEY

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_int4.c:10

目的

PostgreSQL の GiST インデックスにおいて、int4 型の範囲（下限値と上限値）を保持および表現するためのデータ構造です。

シグネチャまたは形状

typedef struct int32key
{
	int32		lower;
	int32		upper;
} int32KEY;

入力と出力

  • 入力: int4 型の値を lower および upper メンバに格納します。
  • 出力: GiST インデックスのノード間で受け渡されるキーとして機能します。

前提条件と不変条件

  • lower は範囲の下限、upper は範囲の上限を表します。
  • GiST インデックスの特性上、この構造体は int4 型のデータ範囲を圧縮・表現するために使用されます。

アルゴリズムまたは振る舞い

  • int4 型の値を GiST インデックスのキー形式に変換（圧縮）する際に、この構造体にラップされます。
  • インデックスの検索（consistent）や比較（gbt_int4key_cmp）の際、この構造体の lower および upper メンバが参照され、範囲の包含関係や距離計算が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/btree_gist/btree_int4.c 内の各 GiST サポート関数（gbt_int4_consistent, gbt_int4_compress, gbt_int4_fetch 等）からキャストを介して頻繁に参照されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。int32KEY はこの key フィールドに格納されます。
  • Nsrt: ソートサポート時に使用される構造体。int32KEY へのポインタとしてキャストされる箇所があります。

失敗モードと境界条件

  • int4 型の範囲外の値や、lower > upper となるような不正な範囲設定については、本構造体自体は制約を強制しません（ロジック側での制御に依存します）。
  • メモリ割り当て時（palloc）の失敗は PostgreSQL の標準的なエラー処理に従います。

保守担当者向けメモ

  • 本構造体は btree_utils_num.h で定義された汎用的な数値型 GiST 処理ロジックと密接に連携しています。
  • 構造体のサイズは sizeof(int32KEY) で取得され、インデックスのメタデータとして利用されます。変更を加える際は、関連する gbt_int4_* 関数群のキャスト処理に影響がないか確認してください。
