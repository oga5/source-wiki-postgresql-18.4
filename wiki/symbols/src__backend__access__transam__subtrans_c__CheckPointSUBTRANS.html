CheckPointSUBTRANS

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/subtrans.c:354

目的

サブトランザクションログ（SUBTRANS）のダーティページをディスクへ書き出します。

シグネチャまたは形状

void CheckPointSUBTRANS(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてディスクへの書き込みを実行）

前提条件と不変条件

  • SubTransCtl が初期化されている必要があります。
  • 本関数は正確性の観点からは必須ではありません。

アルゴリズムまたは振る舞い

1. TRACE_POSTGRESQL_SUBTRANS_CHECKPOINT_START を実行し、チェックポイント開始をトレースします。 2. SimpleLruWriteAll(SubTransCtl, true) を呼び出し、SubTransCtl が管理するすべてのダーティページをディスクへフラッシュします。 3. TRACE_POSTGRESQL_SUBTRANS_CHECKPOINT_DONE を実行し、チェックポイント終了をトレースします。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/xlog.c:7563
  • 呼び出し先:
  • SimpleLruWriteAll (SLRU 共有メモリ制御構造体 SubTransCtl を引数として実行)

関連データ構造

  • SubTransCtl: サブトランザクション管理用の SLRU 制御構造体。

失敗モードと境界条件

  • ページ書き込み時に I/O エラーが発生した場合、SimpleLruWriteAll 経由でエラーが報告されます。

保守担当者向けメモ

  • 本関数は、チェックポイント処理においてバックエンドプロセスではなくチェックポイントプロセス自身にダーティページの書き込みを行わせることで、負荷分散を図る目的で実装されています。
  • データの整合性維持のために必須の処理ではありません。
