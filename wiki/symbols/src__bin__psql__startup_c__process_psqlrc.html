process_psqlrc

定義

  • 種別: function
  • 定義箇所: src/bin/psql/startup.c:778

目的

psql の起動時に、システム全体の設定ファイルおよびユーザー固有の設定ファイル（psqlrc）を読み込み、環境をカスタマイズする。

シグネチャまたは形状

static void process_psqlrc(char *argv0)

入力と出力

  • **入力**: argv0（プログラムの実行パス）。
  • **出力**: なし（void）。内部で process_psqlrc_file を呼び出し、設定を適用する。

前提条件と不変条件

  • argv0 は find_my_exec に渡され、実行ファイルの場所を特定するために使用される。
  • SYSPSQLRC（システム設定）および PSQLRC（ユーザー設定）のファイル名定数が定義されている必要がある。

アルゴリズムまたは振る舞い

1. find_my_exec を使用して実行ファイルのパスを特定する。失敗した場合は pg_fatal で終了する。 2. get_etc_path を使用してシステム設定ファイルのディレクトリを特定し、SYSPSQLRC を読み込む。 3. 環境変数 PSQLRC が設定されている場合:

  • その値を pstrdup で複製し、expand_tilde でチルダ展開を行った後、process_psqlrc_file で読み込む。

    • 4. 環境変数 PSQLRC が設定されていない場合:

  • get_home_path でユーザーのホームディレクトリを特定し、PSQLRC ファイルを読み込む。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/startup.c 内の main 関数（引数解析後の初期化フェーズ）。
  • **呼び出し先**:
  • find_my_exec
  • get_etc_path
  • process_psqlrc_file
  • getenv
  • pstrdup
  • expand_tilde
  • get_home_path
  • pg_fatal

関連データ構造

  • MAXPGPATH: ファイルパスを格納するバッファサイズ。

失敗モードと境界条件

  • find_my_exec が実行ファイルのパス特定に失敗した場合、pg_fatal によりプログラムが異常終了する。
  • 環境変数 PSQLRC が空文字列の場合、ユーザーのホームディレクトリにあるデフォルトの PSQLRC ファイルを読み込むフォールバック処理が行われる。

保守担当者向けメモ

  • process_psqlrc_file は、ファイルが存在しない場合や読み込みに失敗した場合の挙動を内部で制御しているため、本関数側ではファイル有無の明示的なチェックは行わない。
  • 環境変数 PSQLRC を処理する際、pstrdup でメモリを確保しているため、必要に応じてメモリ管理に注意すること。
