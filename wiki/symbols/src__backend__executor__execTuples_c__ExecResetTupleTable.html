ExecResetTupleTable

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execTuples.c:1379

目的

実行エンジンが保持するタプルテーブル（TupleTableSlot のリスト）内の全スロットをクリアし、必要に応じてリソースおよびメモリを解放する。

シグネチャまたは形状

void ExecResetTupleTable(List *tupleTable, bool shouldFree)

入力と出力

  • tupleTable: TupleTableSlot を要素として含むリスト。
  • shouldFree: true の場合、スロット自体およびリスト構造のメモリを解放する。false の場合はスロットの保持するリソースのみを解放する。
  • 戻り値: なし。

前提条件と不変条件

  • tupleTable 内の各要素は TupleTableSlot である必要がある。
  • shouldFree が true の場合、TTS_FIXED(slot) が真であるスロットに対しては、tts_values および tts_isnull 配列の pfree は行われない。

アルゴリズムまたは振る舞い

1. tupleTable リストを走査し、各 TupleTableSlot に対して以下を実行する：

  • ExecClearTuple(slot) を呼び出し、スロットを空にする。
  • slot->tts_ops->release(slot) を呼び出し、スロット固有のリソースを解放する。
  • tts_tupleDescriptor が存在する場合、ReleaseTupleDesc で参照カウントを減らし、ポインタを NULL にする。
  • shouldFree が true の場合：
  • TTS_FIXED(slot) でないスロットについては、tts_values と tts_isnull 配列を pfree する。
  • スロット構造体自体を pfree する。

    • 2. shouldFree が true の場合、最後に list_free(tupleTable) を呼び出し、リスト構造自体を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfrom.c
  • src/backend/commands/trigger.c
  • src/backend/executor/execMain.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • ExecClearTuple
  • slot->tts_ops->release
  • ReleaseTupleDesc
  • pfree
  • list_free

関連データ構造

  • TupleTableSlot: タプルを保持するコンテナ。
  • TupleTableSlotOps: スロットの操作関数テーブル（release メソッドを含む）。
  • TupleDesc: タプルの記述子。

失敗モードと境界条件

  • tupleTable が空の場合、何も処理を行わず正常終了する。
  • TTS_FIXED フラグが設定されているスロットに対しては、内部配列の解放をスキップすることでメモリ破壊を防ぐ。

保守担当者向けメモ

  • 本関数は実行終了時やエラーリカバリ時に呼び出される。スロットのメモリ管理方針（shouldFree）が呼び出し元によって適切に制御されていることを確認すること。
