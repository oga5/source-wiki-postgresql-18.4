create_toy_buffer

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_aio/test_aio.c:306

目的

テスト対象のブロックに対して、共有バッファまたはローカルバッファを確保し、初期化（ゼロクリア）および状態の強制リセットを行うためのヘルパー関数です。

シグネチャまたは形状

static Buffer create_toy_buffer(Relation rel, BlockNumber blkno)

入力と出力

  • **入力**:
  • rel: 対象のリレーション。
  • blkno: 確保対象のブロック番号。
  • **出力**:
  • 確保されたバッファの識別子（Buffer）。

前提条件と不変条件

  • 本関数はテスト専用であり、実データが存在する環境での使用は想定されていません。
  • 呼び出し側は、対象のバッファが他のバックエンドによって使用されていないことを前提としています。

アルゴリズムまたは振る舞い

1. ReadBufferExtended を呼び出し、指定ブロックを RBM_ZERO_AND_LOCK モードで読み込みます。 2. 直後に LockBuffer でロックを解除します。 3. リレーションの種類に応じてローカルバッファまたは共有バッファのディスクリプタを取得します。 4. バッファの参照カウント（refcount）を確認し、1より大きい場合は既にピン留めされていると見なしエラーを発生させます。 5. 参照カウントが1以下の場合、バッファの状態から BM_VALID および BM_DIRTY フラグをクリアし、バッファを初期状態にリセットします。 6. 最後に、確保したバッファ識別子を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_aio/test_aio.c 内のテスト関数（read_rel_block_ll 等）から呼び出されます。
  • **呼び出し先**:
  • ReadBufferExtended, LockBuffer, GetLocalBufferDescriptor, GetBufferDescriptor, LockBufHdr, UnlockBufHdr, pg_atomic_read_u32, pg_atomic_unlocked_write_u32, elog 等のPostgreSQL内部API。

関連データ構造

  • Buffer: バッファ識別子。
  • BufferDesc: バッファディスクリプタ。
  • Relation: リレーション情報。

失敗モードと境界条件

  • BUF_STATE_GET_REFCOUNT(buf_state) > 1 の場合、elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数は共有バッファの状態を直接操作する「ハック」を含みます。バッファ管理の標準的な挙動をバイパスするため、テスト環境以外での利用は厳禁です。
