replace_empty_jointree

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepjointree.c:409

目的

クエリのジョインツリーが空である場合に、ダミーの RTE_RESULT を挿入してジョインツリーを補完する。これにより、プランナが空の FROM 句を持つクエリを適切に処理できるようにする。

シグネチャまたは形状

void replace_empty_jointree(Query *parse)

入力と出力

  • **入力**: Query *parse - 処理対象のクエリ構造体。
  • **出力**: なし（副作用として parse->rtable および parse->jointree->fromlist を直接変更する）。

前提条件と不変条件

  • parse->jointree が初期化されていること。
  • parse->jointree->fromlist が空（NIL）である場合にのみ動作する。
  • parse->setOperations が設定されている（UNION 等の集合演算ツリーである）場合は、何も行わず終了する。

アルゴリズムまたは振る舞い

1. parse->jointree->fromlist が空でない場合、または parse->setOperations が存在する場合は即座にリターンする。 2. RTE_RESULT 型の新しい RangeTblEntry を作成し、エイリアスを *RESULT* に設定する。 3. 作成した rte を parse->rtable の末尾に追加する。 4. 追加された rte のインデックスを参照する RangeTblRef を作成し、parse->jointree->fromlist に格納する。

関連データ構造

  • Query: クエリの解析結果を保持する構造体。
  • RangeTblEntry: 範囲テーブルの各エントリを定義する構造体。
  • RangeTblRef: ジョインツリー内で範囲テーブルを参照するためのノード。

失敗モードと境界条件

  • parse が NULL の場合、あるいは parse->jointree が NULL の場合はセグメンテーションフォールトが発生する可能性がある（ソースコード上での明示的なチェックは存在しない）。

保守担当者向けメモ

  • 本関数はクエリ最適化の初期段階で呼び出される。RTE_RESULT を挿入することで、後のプランニング工程で FROM 句が空であることを意識せずに処理を行えるようにする設計である。
  • 集合演算（setOperations）が存在する場合に処理をスキップするのは、それらが既に独自の構造を持っているためである。
