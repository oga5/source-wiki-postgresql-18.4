decoct

定義

  • 種別: function
  • 定義箇所: src/port/inet_net_ntop.c:154

目的

バイナリ形式のネットワークアドレス（オクテット列）を、ドット区切りの 10 進数文字列形式に変換します。主に IPv6 内部で IPv4 互換アドレスを扱う際の補助関数として使用されます。

シグネチャまたは形状

static int decoct(const u_char *src, int bytes, char *dst, size_t size)

入力と出力

  • src: 変換対象のバイナリデータ（u_char 配列）。
  • bytes: 処理するバイト数。
  • dst: 変換結果を格納するバッファ。
  • size: dst バッファの利用可能なサイズ。
  • 戻り値: 成功時は書き込んだバイト数、バッファ不足時は 0 を返します。

前提条件と不変条件

  • src は少なくとも bytes 分の有効なメモリ領域を指している必要があります。
  • dst は size 分の書き込み可能なバッファである必要があります。

アルゴリズムまたは振る舞い

1. bytes 回のループで各オクテットを処理します。 2. 各オクテットを SPRINTF マクロを用いて 10 進数文字列に変換し、dst に書き込みます。 3. 各オクテットの間に . を挿入します（最後のオクテットを除く）。 4. 処理の都度、残りのバッファサイズ size を減算し、不足した場合は即座に 0 を返します。 5. 最終的に書き込まれた総バイト数を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/inet_net_ntop.c 内の inet_net_ntop_ipv6 関数から呼び出されます（src + 12 の位置にある 4 バイトの IPv4 互換アドレスを処理するため）。
  • 呼び出し先: SPRINTF マクロ（標準的な sprintf のラッパー）。

関連データ構造

  • u_char: ネットワークアドレスの各オクテットを保持するために使用されます。

失敗モードと境界条件

  • バッファサイズ不足: size が 255. を書き込むのに十分でない場合（sizeof "255." 以下）、0 を返して処理を中断します。

保守担当者向けメモ

  • 本関数は IPv6 変換ロジックの一部として特化しています。
  • SPRINTF マクロの挙動に依存しているため、移植先環境で sprintf の戻り値の定義が異なる場合の影響を受ける可能性があります。
