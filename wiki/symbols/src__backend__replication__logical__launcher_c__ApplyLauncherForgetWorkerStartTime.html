ApplyLauncherForgetWorkerStartTime

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/launcher.c:1084

目的

指定されたサブスクリプションIDに関連付けられた、論理レプリケーションワーカーの「最終起動時刻」情報を共有メモリ上のハッシュテーブルから削除します。

シグネチャまたは形状

void ApplyLauncherForgetWorkerStartTime(Oid subid)

入力と出力

  • **入力**: subid (Oid) - 情報を削除する対象のサブスクリプション識別子。
  • **出力**: なし。

前提条件と不変条件

  • 共有メモリ上の last_start_times ハッシュテーブルが初期化されている必要があります（logicalrep_launcher_attach_dshmem() により保証）。

アルゴリズムまたは振る舞い

1. logicalrep_launcher_attach_dshmem() を呼び出し、共有メモリ内のハッシュテーブルへのアクセスを確立します。 2. dshash_delete_key() を使用し、引数 subid をキーとしてハッシュテーブルから該当エントリを削除します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/subscriptioncmds.c
  • src/backend/replication/logical/tablesync.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • logicalrep_launcher_attach_dshmem (内部関数)
  • dshash_delete_key (共有ハッシュテーブル操作関数)

関連データ構造

  • last_start_times: 共有メモリ上に配置された dshash 型のハッシュテーブル。ワーカーの再起動間隔を制御するために使用されます。

失敗モードと境界条件

  • 指定した subid がハッシュテーブル内に存在しない場合、dshash_delete_key は単に何も行わず正常終了します。

保守担当者向けメモ

  • 本関数は、ワーカーが正常に終了した際やサブスクリプションの設定変更時に、古い起動時刻情報をクリアするために呼び出されます。
  • dshash の操作を伴うため、共有メモリの競合やロック状態については dshash の仕様に依存します。
