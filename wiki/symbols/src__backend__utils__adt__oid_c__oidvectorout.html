oidvectorout

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oid.c:181

目的

oidvector 型の内部表現を、スペース区切りの数値文字列（例: "123 456 789"）に変換します。

シグネチャまたは形状

Datum oidvectorout(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して oidvector 構造体へのポインタを受け取ります。
  • **出力**: 変換された文字列を cstring（Datum 型）として返却します。

前提条件と不変条件

  • 入力される oidvector は、check_valid_oidvector によって次元数やNULLの有無が検証済みである必要があります。
  • oidvector は1次元配列であり、インデックスは0から始まります。

アルゴリズムまたは振る舞い

1. check_valid_oidvector を呼び出し、入力データの妥当性を確認します。 2. 配列の要素数（dim1）を取得します。 3. 各要素を文字列化するために必要なメモリを palloc で確保します（1要素あたり最大12バイトを想定）。 4. ループ内で各 Oid 値を sprintf を用いて文字列に変換し、要素間にスペースを挿入します。 5. 最後に終端文字 \0 を付与し、結果の文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • check_valid_oidvector: 入力の検証に使用。
  • palloc: メモリ確保に使用。
  • sprintf: 数値の文字列変換に使用。
  • **呼び出し元**: ソースコード内の明示的な呼び出し元は確認されていませんが、PostgreSQLの型システム（pg_type）を通じて、SQLレベルでの出力処理時に自動的に呼び出されます。

関連データ構造

  • oidvector: Oid の固定長配列を保持する構造体。

失敗モードと境界条件

  • check_valid_oidvector が無効な配列を検出した場合、エラーが発生します。
  • メモリ不足時には palloc がエラーをスローします。

保守担当者向けメモ

  • 本関数は oidvector のテキスト表現を生成します。バイナリ形式の出力が必要な場合は oidvectorsend を参照してください。
  • sprintf のバッファサイズ計算（nnums * 12 + 1）は、Oid が符号なし32ビット整数であることを前提とした安全な見積もりです。
