DisownLatch

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/latch.c:143

目的

共有ラッチの所有権を解放し、ラッチに関連付けられたプロセスIDをクリアする。

シグネチャまたは形状

void DisownLatch(Latch *latch)

入力と出力

  • **入力**: latch (操作対象の Latch 構造体へのポインタ)
  • **出力**: なし

前提条件と不変条件

  • 対象のラッチが共有ラッチ（latch->is_shared が真）であること。
  • 呼び出し元プロセスが当該ラッチの現在の所有者（latch->owner_pid == MyProcPid）であること。

アルゴリズムまたは振る舞い

1. latch->is_shared が真であることをアサートする。 2. latch->owner_pid が現在のプロセスID（MyProcPid）と一致することをアサートする。 3. latch->owner_pid を 0 に設定し、所有権を解除する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogrecovery.c:1662
  • src/backend/storage/lmgr/proc.c:1000
  • src/backend/storage/lmgr/proc.c:1069
  • **呼び出し先**:
  • 外部関数呼び出しなし（アサートマクロを除く）。

関連データ構造

  • Latch: ラッチの状態を保持する構造体。owner_pid および is_shared フィールドを含む。

失敗モードと境界条件

  • 前提条件を満たさない場合（共有されていないラッチに対して呼び出した場合、または所有者ではないプロセスが呼び出した場合）、アサート失敗によりプロセスが異常終了する。

保守担当者向けメモ

  • 本関数は所有権の移譲や解放を管理するものであり、ロックを伴わない。そのため、呼び出し側で適切な排他制御が行われていることを確認すること。
