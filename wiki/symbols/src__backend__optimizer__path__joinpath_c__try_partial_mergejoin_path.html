try_partial_mergejoin_path

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/joinpath.c:1144

目的

結合リレーションに対して、並列実行可能なマージ結合（Mergejoin）パスを生成し、コスト評価を経て有効であればパスリストに追加する。

シグネチャまたは形状

static void try_partial_mergejoin_path(PlannerInfo *root,
                                       RelOptInfo *joinrel,
                                       Path *outer_path,
                                       Path *inner_path,
                                       List *pathkeys,
                                       List *mergeclauses,
                                       List *outersortkeys,
                                       List *innersortkeys,
                                       JoinType jointype,
                                       JoinPathExtraData *extra)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • joinrel: 結合対象のリレーション情報。
  • outer_path, inner_path: 結合する左右のパス。
  • pathkeys, mergeclauses: 結合に必要なキーと句。
  • outersortkeys, innersortkeys: ソートが必要な場合のキー。
  • jointype: 結合の種類（INNER, LEFT 等）。
  • extra: 結合処理に必要なメタデータ。
  • **出力**: なし（成功時に joinrel->partial_pathlist を更新）。

前提条件と不変条件

  • joinrel に lateral_relids が含まれていないこと。
  • outer_path が他のリレーションにパラメータ化されていないこと。
  • inner_path が他のリレーションにパラメータ化されていないこと。

アルゴリズムまたは振る舞い

1. **ソートの最適化**: outersortkeys および innersortkeys が入力パスの pathkeys に含まれている場合、明示的なソートを回避するためにソートキーを NIL に設定する。 2. **コスト計算**: initial_cost_mergejoin を呼び出し、マージ結合のコストを算出する。 3. **事前チェック**: add_partial_path_precheck を呼び出し、コストやノードの有効性を検証する。 4. **パス生成と登録**: create_mergejoin_path でパスを作成し、add_partial_path を通じて joinrel に登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/joinpath.c 内の結合パス生成ロジックから呼び出される（行1047, 1536等）。
  • **呼び出し先**:
  • initial_cost_mergejoin
  • add_partial_path_precheck
  • create_mergejoin_path
  • add_partial_path

関連データ構造

  • PlannerInfo: プランナの全体状態。
  • RelOptInfo: 結合対象リレーション。
  • Path: 結合パスの基本構造。
  • JoinPathExtraData: 結合に必要な追加情報（restrictlist 等）。
  • JoinCostWorkspace: コスト計算結果を保持する作業領域。

失敗モードと境界条件

  • inner_path がパラメータ化されている場合、即座に終了する。
  • add_partial_path_precheck の判定でコストが劣ると判断された場合、パスは追加されない。

保守担当者向けメモ

  • 本関数は並列クエリ実行用であるため、inner_path に対するインクリメンタルソートのサポート状況（現状未対応）に注意すること。
  • try_partial_hashjoin_path や try_partial_nestloop_path と同様の設計パターンに従っている。
