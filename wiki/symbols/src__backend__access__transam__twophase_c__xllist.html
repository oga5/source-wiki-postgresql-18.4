xllist

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/twophase.c:1003

目的

準備済みトランザクションのデータをディスクへ書き出す際に、メモリ上で一時的に保持するためのデータチャンクのリスト（チェーン）を管理します。

シグネチャまたは形状

static struct xllist
{
    StateFileChunk *head;       /* チェーンの先頭データブロック */
    StateFileChunk *tail;       /* チェーンの末尾ブロック */
    uint32      num_chunks;     /* チャンク数 */
    uint32      bytes_free;     /* 末尾ブロックの空きバイト数 */
    uint32      total_len;      /* チェーン内の総データバイト数 */
} records;

入力と出力

  • 入力: StateFileChunk 構造体によるデータブロックの追加。
  • 出力: 構造体内の各フィールド（head, tail, num_chunks, bytes_free, total_len）の更新。

前提条件と不変条件

  • head が NULL の場合、リストは空とみなされます。
  • tail は常にリストの末尾を指し、新しいデータは tail の空き領域（bytes_free）に書き込まれます。
  • total_len はリストに含まれる全チャンクのデータ長の合計と一致する必要があります。

アルゴリズムまたは振る舞い

  • 準備済みトランザクションの情報をシリアライズする際、データを StateFileChunk 単位で動的に確保し、このリストに連結します。
  • 新しいチャンクが必要な場合は tail を更新し、既存のチャンクに空きがある場合は bytes_free を減らしてデータを追記します。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数名は、提供されたソース抜粋からは特定できません。

関連データ構造

  • StateFileChunk: リストを構成する個々のデータブロック。
  • TwoPhaseFileHeader: 準備済みトランザクションのファイルヘッダー。

失敗モードと境界条件

  • メモリ不足: StateFileChunk の割り当てに失敗した場合、処理が中断される可能性があります。
  • チャンク数やデータ長のオーバーフロー: uint32 の範囲を超えるデータ量が発生した場合の挙動については、提供されたソースからは不明です。

保守担当者向けメモ

  • 本構造体は static な変数 records として定義されており、モジュール内でのみ使用されます。
  • 2フェーズコミットの永続化処理において、メモリ上の状態をディスク書き出し用バッファへ変換する際の中間表現として機能しています。
