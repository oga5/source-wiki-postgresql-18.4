escape_string_conn

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_escape/test_escape.c:296

目的

PQescapeStringConn をラップし、エスケープ対象の文字列をシングルクォートで囲んだ形式で PQExpBuffer に格納するテスト用ユーティリティ関数です。

シグネチャまたは形状

static bool escape_string_conn(PGconn *conn, PQExpBuffer target, const char *unescaped, size_t unescaped_len, PQExpBuffer escape_err);

入力と出力

  • **入力**:
  • conn: PostgreSQL接続ハンドル。
  • target: エスケープ結果を格納するバッファ。
  • unescaped: エスケープ対象のソース文字列。
  • unescaped_len: ソース文字列の長さ。
  • escape_err: エラー発生時にエラーメッセージを格納するバッファ。
  • **出力**:
  • 成功時に true、失敗時に false を返します。
  • target には ' + エスケープ済み文字列 + ' が追記されます。
  • 失敗時は escape_err に PQerrorMessage が格納されます。

前提条件と不変条件

  • target バッファは事前に初期化されている必要があります。
  • unescaped は少なくとも unescaped_len バイトの読み取りが可能である必要があります。

アルゴリズムまたは振る舞い

1. target バッファの末尾に開始クォート ' を追加します。 2. enlargePQExpBuffer を呼び出し、エスケープ後の最大長（unescaped_len * 2 + 1）を確保します。 3. PQescapeStringConn を実行し、target の現在の末尾位置にエスケープ結果を書き込みます。 4. target の長さを更新し、終了クォート ' を追加します。 5. PQescapeStringConn がエラーを報告した場合、PQerrorMessage を escape_err にコピーし、末尾の改行を除去して false を返します。 6. 正常終了時は true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_escape/test_escape.c 内の pe_test_escape_func 構造体の初期化箇所（414行目）で参照されています。
  • **呼び出し先**: appendPQExpBufferChar, enlargePQExpBuffer, PQescapeStringConn, appendPQExpBufferStr, PQerrorMessage。

関連データ構造

  • PGconn: libpqの接続オブジェクト。
  • PQExpBuffer: PostgreSQLの動的文字列バッファ。

失敗モードと境界条件

  • PQescapeStringConn 自体が失敗した場合、エラーメッセージが escape_err に格納され、関数は false を返します。
  • バッファの拡張に失敗した場合、PQExpBuffer の内部ロジックによりプログラムが終了する可能性があります。

保守担当者向けメモ

  • 本関数はテストスイートの一部であり、PQescapeStringConn の挙動を検証するために使用されます。
  • エラーメッセージの末尾にある改行文字を削除する処理が含まれているため、エラーログのフォーマット変更時には注意してください。
