PrintTOCSummary

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_archiver.c:1301

目的

アーカイブファイル内の目次（TOC）情報を要約し、指定された出力先（ファイルまたは標準出力）に人間が読み取り可能な形式で書き出す。

シグネチャまたは形状

void PrintTOCSummary(Archive *AHX)

入力と出力

  • **入力**: Archive *AHX（アーカイブハンドルへのポインタ）。
  • **出力**: なし（戻り値なし）。副作用として、AH->public.ropt->filename で指定されたファイル、または標準出力に対して、アーカイブのメタデータおよび TOC エントリのリストを書き出す。

前提条件と不変条件

  • AHX は有効な ArchiveHandle 構造体としてキャスト可能であること。
  • AH->toc が初期化されており、TOC エントリがリスト構造として存在すること。
  • ropt->filename が指定されている場合、そのファイルへの書き込み権限があること。

アルゴリズムまたは振る舞い

1. **出力設定**: SaveOutput を使用して現在の出力状態を保存し、ropt->filename があれば SetOutput で出力先を切り替える。 2. **ヘッダー出力**: アーカイブ作成日時、データベース名、TOC エントリ数、圧縮アルゴリズム、アーカイブ形式、バージョン情報、整数/オフセットサイズなどを ahprintf で出力する。 3. **TOC エントリの走査**: AH->toc リストを順次走査する。

  • 各エントリのセクションを判定し、_tocEntryRequired を呼び出してエントリの要求フラグ（reqs）を更新する。
  • ropt->verbose が有効、またはエントリがスキーマ・データ・統計のいずれかに関連する場合、エントリの詳細（ID、OID、説明、名前空間、所有者など）を整形して出力する。
  • ropt->verbose が有効かつ依存関係がある場合、依存先の ID リストを出力する。

    • 4. **終了処理**: ropt->strict_names が有効な場合、StrictNamesCheck を実行する。最後に RestoreOutput で出力先を復元する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_restore.c:513 から呼び出される。
  • **呼び出し先**: SaveOutput, SetOutput, ahprintf, get_compress_algorithm_name, _tocEntryRequired, sanitize_line, free, StrictNamesCheck, RestoreOutput。

関連データ構造

  • ArchiveHandle: アーカイブ全体の状態を保持する。
  • RestoreOptions: 復元オプション（出力ファイル名や詳細表示フラグなど）。
  • TocEntry: 個別のダンプ対象オブジェクトのメタデータ。

失敗モードと境界条件

  • strftime による日時変換が失敗した場合、タイムスタンプは [unknown] と表示される。
  • ropt->filename が指定されている場合にファイルオープンや書き込みに失敗すると、SetOutput 経由でエラーが発生する可能性がある。
  • AH->toc が空の場合、ヘッダー情報のみが出力される。

保守担当者向けメモ

  • 本関数は pg_restore の -l (list) オプションの実装として機能する。
  • _tocEntryRequired のロジックと同期している必要があるため、フィルタリング条件を変更する際は注意が必要である。
  • sanitize_line を使用して出力文字列をエスケープしているため、TOC 名に含まれる特殊文字による出力の崩れを防いでいる。
