AlterEventTriggerOwner_oid

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/commands/event_trigger.c:513

目的

指定されたOIDを持つイベントトリガーの所有者を変更します。

シグネチャまたは形状

void AlterEventTriggerOwner_oid(Oid trigOid, Oid newOwnerId)

入力と出力

  • **入力**:
  • trigOid (Oid): 変更対象のイベントトリガーのOID。
  • newOwnerId (Oid): 新しい所有者のロールID。
  • **出力**: なし（成功時はカタログを更新し、失敗時はエラーをスロー）。

前提条件と不変条件

  • pg_event_trigger システムカタログに対する RowExclusiveLock を取得します。
  • 指定された trigOid が存在する必要があります。存在しない場合は ERRCODE_UNDEFINED_OBJECT エラーが発生します。

アルゴリズムまたは振る舞い

1. table_open を呼び出し、pg_event_trigger リレーションを RowExclusiveLock モードで開きます。 2. SearchSysCacheCopy1 を使用して、指定された trigOid に対応するイベントトリガーのタプルをシステムキャッシュから取得します。 3. タプルが存在しない場合、エラーを報告します。 4. AlterEventTriggerOwner_internal を呼び出し、所有者変更の内部処理を実行します。 5. heap_freetuple でタプルのメモリを解放します。 6. table_close を呼び出し、リレーションを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_shdepend.c:1691
  • **呼び出し先**:
  • table_open / table_close
  • SearchSysCacheCopy1
  • AlterEventTriggerOwner_internal
  • heap_freetuple
  • ereport

関連データ構造

  • pg_event_trigger (システムカタログ)
  • HeapTuple

失敗モードと境界条件

  • 指定されたOIDのイベントトリガーが見つからない場合、ereport によりエラー終了します。

保守担当者向けメモ

  • 本関数は、所有者変更のメインロジックを AlterEventTriggerOwner_internal に委譲しています。
  • pg_shdepend.c からの呼び出しは、共有依存関係の更新に伴う所有者変更の文脈で行われています。
