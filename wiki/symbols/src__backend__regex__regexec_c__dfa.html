dfa

定義

  • 種別: struct
  • 定義箇所: src/backend/regex/regexec.c:63

目的

正規表現マッチングにおける「遅延評価DFA（決定性有限オートマトン）」の状態管理を行うための構造体です。マッチング中に必要に応じて状態セットを生成・キャッシュし、効率的な探索を実現します。

シグネチャまたは形状

struct dfa {
    int nssets;             /* キャッシュサイズ */
    int nssused;            /* 使用中のエントリ数 */
    int nstates;            /* 状態数 */
    int ncolors;            /* outarc/inchainベクトルの長さ */
    int wordsper;           /* 状態セットビットベクトルの長さ */
    struct sset *ssets;     /* 状態セットキャッシュ */
    unsigned *statesarea;   /* ビットベクトル格納領域 */
    unsigned *work;         /* 作業用領域ポインタ */
    struct sset **outsarea; /* outarcベクトル格納領域 */
    struct arcp *incarea;   /* inchain格納領域 */
    struct cnfa *cnfa;      /* コンパイル済みNFA */
    struct colormap *cm;    /* カラーマップ */
    chr *lastpost;          /* 最後にキャッシュフラッシュされた成功位置 */
    chr *lastnopr;          /* 最後にキャッシュフラッシュされたNOPROGRESS位置 */
    struct sset *search;    /* 置換探索ポインタメモリ */
    int backno;             /* バックリファレンス用サブ式番号 */
    short backmin;          /* バックリファレンスの最小繰り返し回数 */
    short backmax;          /* バックリファレンスの最大繰り返し回数 */
    bool ismalloced;        /* 構造体自体の解放フラグ */
    bool arraysmalloced;    /* 内部配列の解放フラグ */
};

入力と出力

  • **入力**: regcomp.c で生成された cnfa および colormap を保持します。
  • **出力**: マッチングプロセスにおいて、ssets（状態セットキャッシュ）を介して遷移結果を提供します。

前提条件と不変条件

  • ismalloced および arraysmalloced フラグにより、メモリ管理（スタック上の smalldfa か、ヒープ上の malloc か）が制御されます。
  • ssets は遅延生成されるため、マッチングの進行に伴い nssused が増加します。

アルゴリズムまたは振る舞い

  • **遅延評価**: 事前に全状態を計算せず、入力文字列の走査に合わせて必要な状態遷移のみを ssets にキャッシュします。
  • **バックリファレンス対応**: backno 等のフィールドにより、バックリファレンスを含む正規表現の制約を保持します。
  • **メモリ最適化**: 小規模な正規表現ではスタック領域を、大規模な場合は MALLOC を使用して動的に領域を確保します。

呼び出し元と呼び出し先

  • **呼び出し元**: 主に src/backend/regex/rege_dfa.c 内の関数群（freedfa 等）から参照・操作されます。
  • **呼び出し先**: 構造体内部のポインタを通じて sset や arcp などの関連データ構造を管理します。

関連データ構造

  • struct sset: キャッシュされた状態セットを保持する構造体。
  • struct cnfa: コンパイル済みのNFA。
  • struct colormap: 文字クラスのカラーマッピング。
  • struct arcp: インチェーン（inchain）ストレージ。
  • struct smalldfa: 本構造体を内包し、スタックメモリ利用を最適化するためのラッパー構造体。

失敗モードと境界条件

  • キャッシュ領域（nssets）が不足した場合、キャッシュのフラッシュが発生します。
  • ismalloced フラグの管理を誤ると、メモリリークまたは二重解放が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は rege_dfa.c のロジックと密接に結合しています。変更を加える際は、rege_dfa.c 内のメモリ確保・解放ロジックとの整合性に注意してください。
  • lastpost および lastnopr はキャッシュフラッシュ時の状態復元に使用されます。
