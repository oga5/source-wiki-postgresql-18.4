inclusion_get_strategy_procinfo

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_inclusion.c:607

目的

BRINインデックスの「包含（Inclusion）」演算子クラスにおいて、指定された戦略番号に対応する関数情報を取得・キャッシュする。

シグネチャまたは形状

FmgrInfo *inclusion_get_strategy_procinfo(BrinDesc *bdesc, uint16 attno, Oid subtype, uint16 strategynum)

入力と出力

  • **入力**:
  • bdesc: BRINインデックスの記述子。
  • attno: 対象の属性番号。
  • subtype: 比較対象のサブタイプ（データ型）。
  • strategynum: 取得する演算子の戦略番号（1〜RTMaxStrategyNumber）。
  • **出力**:
  • 指定された演算子を実行するための FmgrInfo 構造体へのポインタ。

前提条件と不変条件

  • strategynum は 1 以上 RTMaxStrategyNumber 以下であること。
  • bdesc->bd_info[attno - 1] に InclusionOpaque が格納されていること。

アルゴリズムまたは振る舞い

1. bdesc から InclusionOpaque を取得する。 2. キャッシュされている cached_subtype が現在の subtype と異なる場合、既存のキャッシュをすべて無効化（fn_oid を InvalidOid に設定）し、新しいサブタイプを記録する。 3. 指定された strategynum に対応するキャッシュが存在しない場合：

  • AMOPSTRATEGY システムカタログを検索し、演算子OIDを取得する。
  • 該当する演算子が見つからない場合はエラーを投げる。
  • fmgr_info_cxt を使用して関数情報を取得し、キャッシュに保存する。

    • 4. キャッシュされた FmgrInfo 構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: brin_inclusion_consistent（ソースコード内の参照より）。
  • **呼び出し先**: SearchSysCache4, SysCacheGetAttrNotNull, ReleaseSysCache, fmgr_info_cxt, get_opcode。

関連データ構造

  • InclusionOpaque: インデックス列ごとのメタデータと関数キャッシュを保持する構造体。
  • BrinDesc: BRINインデックスの記述子。
  • FmgrInfo: 関数マネージャの情報構造体。

失敗モードと境界条件

  • 指定された戦略番号に対応する演算子がシステムカタログ（pg_amop）に存在しない場合、elog(ERROR) により処理が中断される。
  • subtype が変更されるたびにキャッシュがリセットされるため、頻繁な型変更はパフォーマンスに影響を与える可能性がある。

保守担当者向けメモ

  • minmax_get_strategy_procinfo と設計が共通しているため、修正時は両方の整合性を確認すること。
  • キャッシュの無効化ロジックは subtype の変更に依存しているため、この前提が崩れるような変更を行う場合は注意が必要。
