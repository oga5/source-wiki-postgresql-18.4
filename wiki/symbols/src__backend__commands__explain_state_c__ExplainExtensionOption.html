ExplainExtensionOption

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/explain_state.c:43

目的

拡張機能が独自の EXPLAIN オプションを登録し、その処理ハンドラを関連付けるためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	const char *option_name;
	ExplainOptionHandler option_handler;
} ExplainExtensionOption;

入力と出力

  • option_name: 拡張機能が定義するオプションの名称（文字列）。
  • option_handler: オプションが指定された際に呼び出される関数ポインタ（ExplainOptionHandler 型）。

前提条件と不変条件

  • ExplainExtensionOption のインスタンスは、ExplainExtensionOptionArray という静的配列内で管理されます。
  • 拡張機能は RegisterExtensionExplainOption を通じてこの構造体を登録します。

アルゴリズムまたは振る舞い

  • EXPLAIN コマンドの解析時、組み込みオプションに該当しない未知のオプションが検出されると、登録された ExplainExtensionOption のリストが走査されます。
  • 一致する option_name が見つかった場合、対応する option_handler が実行されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/explain_state.c 内のオプション登録および解析ロジック。
  • 呼び出し先: 拡張機能側で実装された ExplainOptionHandler 型の関数。
  • 注: 具体的な呼び出し元・呼び出し先の関数名は、提供されたソース抜粋からは特定できません。

関連データ構造

  • ExplainExtensionOptionArray: 登録されたすべての ExplainExtensionOption を保持する静的配列。
  • ExplainState: EXPLAIN コマンドの実行状態を保持する構造体。

失敗モードと境界条件

  • ExplainExtensionOptionArray は repalloc を使用して動的に拡張されるため、メモリ不足時にはエラーが発生する可能性があります。
  • 登録されていないオプション名が指定された場合、解析処理は失敗します。

保守担当者向けメモ

  • ExplainExtensionOptionArray はバックエンドの生存期間中のみ有効です。
  • 拡張機能がこの構造体を利用する際は、GetExplainExtensionId を介して取得した ID との整合性に注意してください。
