ComputeQueryIdType

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/queryjumble.h:81

目的

PostgreSQL のクエリ識別子（Query ID）計算機能を制御する GUC パラメータ compute_query_id の設定値を表現するために使用されます。

シグネチャまたは形状

enum ComputeQueryIdType
{
    COMPUTE_QUERY_ID_OFF,
    COMPUTE_QUERY_ID_ON,
    COMPUTE_QUERY_ID_AUTO,
    COMPUTE_QUERY_ID_REGRESS,
};

入力と出力

  • 本列挙型は、システム設定値の入力を受け取り、クエリのフィンガープリント生成処理の挙動を決定するフラグとして出力（利用）されます。

前提条件と不変条件

  • compute_query_id GUC パラメータの値と直接対応しています。
  • 値は OFF, ON, AUTO, REGRESS のいずれかをとります。

アルゴリズムまたは振る舞い

  • COMPUTE_QUERY_ID_OFF: Query ID の計算を無効化します。
  • COMPUTE_QUERY_ID_ON: Query ID の計算を強制的に有効化します。
  • COMPUTE_QUERY_ID_AUTO: 拡張機能（pg_stat_statements 等）が必要とする場合にのみ計算を有効化します。
  • COMPUTE_QUERY_ID_REGRESS: 回帰テスト用の特殊なモードとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/misc/guc.c にて本パラメータが定義・管理されています。
  • 呼び出し先: IsQueryIdEnabled() インライン関数が本列挙型の値を参照し、計算の要否を判定します。

関連データ構造

  • JumbleState: クエリのハッシュ化プロセスで使用される構造体。
  • IsQueryIdEnabled(): 本列挙型を利用して計算の有効性を判定する関数。

失敗モードと境界条件

  • 提供されたソース情報には、本列挙型自体に起因する失敗モードや境界条件に関する記述はありません。

保守担当者向けメモ

  • src/backend/utils/misc/guc.c でのパラメータ定義と同期を維持してください。
  • 新しい計算モードを追加する場合は、IsQueryIdEnabled 関数のロジックも併せて更新する必要があります。
