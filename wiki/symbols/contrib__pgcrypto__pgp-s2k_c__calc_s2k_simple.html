calc_s2k_simple

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-s2k.c:37

目的

OpenPGP仕様（RFC 4880）に基づく「Simple S2K（String-to-Key）」鍵導出アルゴリズムを実装し、パスフレーズから指定された長さのバイナリ鍵を生成します。

シグネチャまたは形状

static int calc_s2k_simple(PGP_S2K *s2k, PX_MD *md, const uint8 *key, unsigned key_len);

入力と出力

  • **入力**:
  • s2k: 鍵長情報（s2k->key_len）および出力先バッファ（s2k->key）を含む構造体。
  • md: ハッシュ計算に使用する PX_MD インターフェース。
  • key: パスフレーズ文字列。
  • key_len: パスフレーズの長さ。
  • **出力**:
  • 成功時は 0 を返します。
  • s2k->key に生成されたバイナリ鍵が格納されます。

前提条件と不変条件

  • s2k->key は、s2k->key_len 分の書き込みが可能なメモリ領域である必要があります。
  • md は初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. 必要な鍵長（s2k->key_len）が満たされるまで、ハッシュ計算を繰り返します。 2. 各反復において、ハッシュ計算の先頭に「ゼロ埋めされたプレフィックス」を付加します（1回目の反復は0バイト、2回目は1バイト、3回目は2バイト…と増加）。 3. プレフィックスの後にパスフレーズ（key）を連結してハッシュを計算します。 4. 計算されたハッシュ値を s2k->key バッファへ順次コピーし、必要な長さに達するまで繰り返します。 5. 最後に、計算に使用した一時バッファを px_memset でゼロクリアします。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-s2k.c 内の pgp_s2k_process から呼び出されます。
  • **呼び出し先**: px_md_result_size, px_md_reset, px_md_update, px_md_finish, px_memset（いずれも px.h で定義された汎用暗号インターフェース）。

関連データ構造

  • PGP_S2K: S2K設定および出力鍵を保持する構造体。
  • PX_MD: ハッシュ関数コンテキスト。

失敗モードと境界条件

  • 鍵長がハッシュ関数の出力サイズ（md_rlen）より小さい場合、ハッシュ値の先頭から必要な分だけがコピーされます。
  • 内部バッファ buf は PGP_MAX_DIGEST サイズに制限されています。

保守担当者向けメモ

  • 本関数はRFC 4880のSimple S2K仕様に準拠しています。
  • メモリ安全性のため、計算終了後に一時バッファを確実にゼロクリアする処理が含まれています。
