date_test_defmt

定義

  • **種別**: function
  • **定義箇所**: src/interfaces/ecpg/test/expected/compat_informix-rfmtdate.c:51

目的

Informix 互換ライブラリの rdefmtdate 関数を用いた日付文字列の解析および変換処理を検証するためのテスト関数です。

シグネチャまたは形状

static void date_test_defmt(const char *fmt, const char *input)
  • fmt: 日付フォーマットを指定する文字列。
  • input: 解析対象の日付文字列。

入力と出力

  • **入力**: フォーマット文字列 fmt および入力日付文字列 input。
  • **出力**: なし（標準出力へテスト結果を表示）。

前提条件と不変条件

  • rdefmtdate が正常終了（戻り値 0）した場合、内部日付型 date が生成されること。
  • rdatestr を用いて、生成された date 型が文字列へ再変換可能であること。

アルゴリズムまたは振る舞い

1. rdefmtdate を呼び出し、input を fmt に基づいて date 型へ変換する。 2. 変換結果（r）が 0 の場合:

  • rdatestr を呼び出し、date 型を文字列 dbuf へ変換する。
  • 変換結果（q）が 0 の場合、変換後の日付文字列を標準出力へ表示する。

    • 3. 変換に失敗した場合（r != 0）:

  • check_return(r) を呼び出し、エラーコードを処理する。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内のテスト実行箇所（main 関数から呼び出される一連のテストケース）。
  • **呼び出し先**:
  • rdefmtdate: 日付変換の実行。
  • rdatestr: 日付から文字列への変換。
  • check_return: エラー発生時の診断出力。
  • printf: 結果の表示。

関連データ構造

  • date: ECPG で使用される内部日付型。

失敗モードと境界条件

  • rdefmtdate が 0 以外の値を返す場合、check_return が呼び出され、エラーコードに応じた診断が行われる。
  • rdatestr が 0 以外の値を返す場合、出力は行われない。

保守担当者向けメモ

  • 本関数は回帰テスト用であり、パーサーの堅牢性を確認するために意図的に冗長なフォーマットや入力文字列がテストケースとして渡される。
  • 修正や機能追加を行う際は、src/interfaces/ecpg/pgtypeslib/ の実装との整合性に注意すること。
