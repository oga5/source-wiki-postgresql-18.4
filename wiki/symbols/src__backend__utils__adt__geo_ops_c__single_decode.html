single_decode

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_ops.c:193

目的

文字列形式の入力から単一の浮動小数点数（float8）を解析し、エラーチェックを統合して抽出する。

シグネチャまたは形状

static bool single_decode(char *num, float8 *x, char **endptr_p,
                          const char *type_name, const char *orig_string,
                          Node *escontext);

入力と出力

  • **入力**:
  • num: 解析対象の文字列。
  • x: 解析結果を格納する float8 へのポインタ。
  • endptr_p: 解析終了位置を指すポインタのポインタ。
  • type_name: エラーメッセージに使用する型名。
  • orig_string: エラー発生時に参照される元の文字列全体。
  • escontext: エラー処理用コンテキスト（Node）。
  • **出力**:
  • 成功時に true を返し、*x に解析値を格納する。失敗時は false を返す。

前提条件と不変条件

  • escontext が有効であること。
  • float8in_internal が内部で呼び出されるため、浮動小数点数として妥当な形式の文字列が num に渡される必要がある。

アルゴリズムまたは振る舞い

1. float8in_internal を呼び出し、文字列 num を float8 に変換する。 2. 変換結果を *x に代入する。 3. SOFT_ERROR_OCCURRED(escontext) マクロを使用して、変換中にエラーが発生しなかったかを判定する。 4. エラーがなければ true を、あれば false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/geo_ops.c 内の各幾何データ型（point, line, circle 等）の入力解析処理から呼び出される。
  • **呼び出し先**:
  • float8in_internal
  • SOFT_ERROR_OCCURRED (マクロ)

関連データ構造

  • float8: PostgreSQLの内部的な倍精度浮動小数点型。
  • Node: エラー処理コンテキストを保持する汎用ノード構造体。

失敗モードと境界条件

  • 文字列が数値として解釈できない場合、float8in_internal がエラーを報告し、SOFT_ERROR_OCCURRED がそれを検知して false を返す。
  • escontext がエラーを捕捉するように設定されている場合、プロセス全体を停止させずに呼び出し元へ制御を戻す。

保守担当者向けメモ

  • 本関数は geo_ops.c 内の複数の幾何データ型解析で共通利用されている。
  • エラーハンドリングの挙動は escontext の状態に依存するため、呼び出し側で適切にコンテキストを管理すること。
