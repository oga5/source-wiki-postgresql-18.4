NamespaceCreate

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_namespace.c:42

目的

システムカタログ pg_namespace に新しいスキーマ（名前空間）のレコードを挿入し、所有者、ACL、拡張機能との依存関係を登録して、作成後のフックを呼び出す。

シグネチャまたは形状

Oid NamespaceCreate(const char *nspName, Oid ownerId, bool isTemp)

入力と出力

  • **入力**:
  • nspName: 作成するスキーマ名。
  • ownerId: スキーマ所有者のOID。
  • isTemp: 一時スキーマかどうかを示すフラグ。
  • **出力**:
  • 作成されたスキーマのOID（Oid 型）。

前提条件と不変条件

  • nspName が NULL であってはならない。
  • 同名のスキーマが既に存在する場合、ERRCODE_DUPLICATE_SCHEMA エラーが発生する。
  • pg_namespace テーブルに対する RowExclusiveLock が保持される。

アルゴリズムまたは振る舞い

1. **重複確認**: SearchSysCacheExists1 を使用し、指定された名前のスキーマが既に存在しないか検証する。 2. **ACL取得**: isTemp が false の場合、get_user_default_acl を呼び出し、所有者のデフォルト権限を取得する。 3. **カタログ挿入**:

  • table_open で pg_namespace を開く。
  • GetNewOidWithIndex で一意なOIDを生成する。
  • heap_form_tuple でタプルを構築し、CatalogTupleInsert で挿入する。

    • 4. **依存関係の記録**:

  • recordDependencyOnOwner で所有者との依存関係を記録。
  • recordDependencyOnNewAcl でACL内のロールとの依存関係を記録。
  • isTemp が false の場合、recordDependencyOnCurrentExtension で拡張機能との依存関係を記録。

    • 5. **通知**: InvokeObjectPostCreateHook を呼び出し、作成完了を通知する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/namespace.c
  • src/backend/commands/schemacmds.c
  • **呼び出し先**:
  • SearchSysCacheExists1, get_user_default_acl, table_open, GetNewOidWithIndex, heap_form_tuple, CatalogTupleInsert, table_close, recordDependencyOnOwner, recordDependencyOnNewAcl, recordDependencyOnCurrentExtension, InvokeObjectPostCreateHook

関連データ構造

  • pg_namespace: スキーマ情報を保持するシステムカタログ。
  • ObjectAddress: 依存関係記録に使用されるオブジェクト識別構造体。

失敗モードと境界条件

  • nspName が NULL の場合、elog(ERROR) で停止する。
  • 同名のスキーマが存在する場合、ereport(ERROR) で停止する。
  • 一時スキーマ（isTemp == true）の場合、デフォルトACLの取得および拡張機能への依存関係登録はスキップされる。

保守担当者向けメモ

  • isTemp フラグの扱いに注意すること。一時スキーマは拡張機能の依存関係から除外される特殊な挙動を持つ。
  • カタログ操作は RowExclusiveLock 下で行われるため、同時実行時のデッドロックに留意すること。
