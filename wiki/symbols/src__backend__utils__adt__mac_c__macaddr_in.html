macaddr_in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mac.c:54

目的

文字列形式のMACアドレス（EUI-48）を解析し、PostgreSQL内部の macaddr 構造体に変換します。

シグネチャまたは形状

Datum macaddr_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して取得される、MACアドレスを表す文字列。
  • **出力**: 解析された macaddr 構造体へのポインタを Datum 型として返却します。

前提条件と不変条件

  • 入力文字列は、以下のいずれかの形式である必要があります。
  • XX:XX:XX:XX:XX:XX
  • XX-XX-XX-XX-XX-XX
  • XXXXXX:XXXXXX
  • XXXXXX-XXXXXX
  • XXXX.XXXX.XXXX
  • XXXX-XXXX-XXXX
  • XXXXXXXXXXXX
  • 各オクテット（a〜f）は 0 から 255 の範囲内である必要があります。

アルゴリズムまたは振る舞い

1. sscanf を使用し、サポートされている複数のMACアドレス表記パターンを順次試行します。 2. 6つのオクテットが正しく抽出できた場合、各値が 0〜255 の範囲内にあるか検証します。 3. 検証に成功した場合、palloc で macaddr 構造体用のメモリを確保し、各オクテットを格納します。 4. 最後に PG_RETURN_MACADDR_P マクロを使用して結果を返却します。 5. 解析失敗時や範囲外の値が検出された場合は、ereturn を使用してエラーを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報には、本関数を直接呼び出す特定の関数名は記載されていません。
  • **呼び出し先**: palloc（メモリ確保）、ereturn（エラー報告）、PG_RETURN_MACADDR_P（結果返却）などのPostgreSQL内部マクロおよび関数を呼び出します。

関連データ構造

  • macaddr: 6バイトのMACアドレスを保持する構造体。

失敗モードと境界条件

  • **構文エラー**: 指定されたパターンに一致しない文字列が渡された場合、ERRCODE_INVALID_TEXT_REPRESENTATION を発生させます。
  • **範囲外エラー**: 各オクテットが 0〜255 の範囲外である場合、ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE を発生させます。

保守担当者向けメモ

  • 新しいMACアドレス表記形式を追加する場合は、sscanf のパターンリストを拡張してください。
  • sscanf の末尾にある %1s は、入力文字列の末尾に余分な文字（ゴミ）が含まれていないかをチェックするために使用されています。
