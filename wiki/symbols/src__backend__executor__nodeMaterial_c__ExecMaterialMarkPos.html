ExecMaterialMarkPos

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMaterial.c:261

目的

Material ノードにおいて、現在のタプル読み取り位置を「マーク」として保存し、後続の ExecMaterialRestrPos による復元を可能にする。

シグネチャまたは形状

void ExecMaterialMarkPos(MaterialState *node)

入力と出力

  • **入力**: MaterialState *node - マテリアライズ処理の状態を保持する構造体。
  • **出力**: なし（副作用として Tuplestore の状態を更新）。

前提条件と不変条件

  • node->eflags に EXEC_FLAG_MARK フラグが設定されている必要がある（Assert による検証）。

アルゴリズムまたは振る舞い

1. node->tuplestorestate が初期化されていない場合、何もせず終了する。 2. tuplestore_copy_read_pointer を呼び出し、現在の読み取り位置をマーク位置としてコピーする。 3. tuplestore_trim を呼び出し、マーク位置より前の不要になったタプルを解放（切り詰め）する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:344
  • **呼び出し先**:
  • tuplestore_copy_read_pointer (PostgreSQL 内部ライブラリ)
  • tuplestore_trim (PostgreSQL 内部ライブラリ)

関連データ構造

  • MaterialState: ノードの実行状態を管理する構造体。tuplestorestate を保持する。

失敗モードと境界条件

  • tuplestorestate が NULL の場合は安全に早期リターンする。
  • EXEC_FLAG_MARK が設定されていない状態で呼び出された場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は Tuplestore の読み取りポインタ操作をラップするものです。Tuplestore のメモリ管理（work_mem 関連）は tuplestore.c 側に依存しているため、メモリ消費に関する問題調査時は tuplestore_trim の挙動を確認してください。
