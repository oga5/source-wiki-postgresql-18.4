pg_eucjp_dsplen

定義

  • 種別: function
  • 定義箇所: src/common/wchar.c:195

目的

EUC-JPエンコーディングにおける、指定されたマルチバイト文字の表示幅（カラム数）を計算します。

シグネチャまたは形状

static int pg_eucjp_dsplen(const unsigned char *s)

入力と出力

  • **入力**: s - 評価対象のマルチバイト文字列の先頭ポインタ。
  • **出力**: 文字の表示幅を整数（int）で返します。

前提条件と不変条件

  • 入力ポインタ s は有効なメモリ領域を指している必要があります。
  • EUC-JPの制御文字（SS2, SS3）および高ビットセットの有無に基づき、エンコーディング規則に従って幅を判定します。

アルゴリズムまたは振る舞い

1. *s が SS2 の場合、幅として 1 を返します。 2. *s が SS3 の場合、幅として 2 を返します。 3. IS_HIGHBIT_SET(*s) が真（高ビットがセットされている）の場合、幅として 2 を返します。 4. 上記いずれにも該当しない場合、pg_ascii_dsplen(s) を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/wchar.c 内の pg_wchar_table 配列において、PG_EUC_JP および PG_EUC_JIS_2004 エンコーディングの表示幅計算関数として登録されています。
  • **呼び出し先**:
  • pg_ascii_dsplen（ASCII範囲内の文字幅計算用）。

関連データ構造

  • pg_wchar_table: エンコーディングごとの関数ポインタを保持するテーブル。

失敗モードと境界条件

  • 入力ポインタが指す先が不正なエンコーディングシーケンスである場合の挙動については、本関数単体では定義されていません。
  • pg_ascii_dsplen の境界条件に依存します。

保守担当者向けメモ

  • 本関数は pg_wchar_table を介してディスパッチされるため、直接呼び出すのではなく、エンコーディング抽象化レイヤーを通じて利用してください。
  • EUC-JPの拡張文字セット（JIS X 0213等）の表示幅要件に変更があった場合は、本関数の条件分岐を見直す必要があります。
