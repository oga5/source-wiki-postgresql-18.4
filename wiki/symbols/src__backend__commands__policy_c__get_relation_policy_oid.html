get_relation_policy_oid

定義

  • 種別: function
  • 定義箇所: src/backend/commands/policy.c:1203

目的

指定されたテーブルOIDとポリシー名に基づき、システムカタログ pg_policy から該当するポリシーのOIDを検索・取得します。

シグネチャまたは形状

Oid get_relation_policy_oid(Oid relid, const char *policy_name, bool missing_ok)

入力と出力

  • **入力**:
  • relid (Oid): 対象テーブルのOID。
  • policy_name (const char *): 検索するポリシー名。
  • missing_ok (bool): true の場合、ポリシーが見つからなくてもエラーにせず InvalidOid を返します。
  • **出力**:
  • 成功時: 該当ポリシーのOID。
  • 失敗時（missing_ok が true の場合）: InvalidOid。

前提条件と不変条件

  • pg_policy カタログに対する AccessShareLock を取得してスキャンを行います。
  • PolicyPolrelidPolnameIndexId インデックスを使用して効率的に検索します。

アルゴリズムまたは振る舞い

1. pg_policy テーブルを AccessShareLock モードで開きます。 2. ScanKey を使用して、polrelid（テーブルOID）と polname（ポリシー名）を検索条件として設定します。 3. systable_beginscan を開始し、systable_getnext でタプルを取得します。 4. タプルが見つからない場合:

  • missing_ok が false ならば ERRCODE_UNDEFINED_OBJECT でエラーを報告します。
  • missing_ok が true ならば InvalidOid を返します。

    • 5. タプルが見つかった場合、GETSTRUCT を介して Form_pg_policy 構造体から oid フィールドを抽出します。 6. systable_endscan を呼び出し、table_close でロックを解放して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c:1472
  • **呼び出し先**:
  • table_open, ScanKeyInit, systable_beginscan, systable_getnext, systable_endscan, table_close, ereport, get_rel_name 等のPostgreSQL内部API。

関連データ構造

  • pg_policy: ポリシー情報を保持するシステムカタログ。
  • Form_pg_policy: pg_policy カタログのタプル構造体。

失敗モードと境界条件

  • 指定されたテーブルに該当する名前のポリシーが存在しない場合、missing_ok が false であればエラーが発生します。
  • relid が無効な場合、get_rel_name が失敗する可能性があります。

保守担当者向けメモ

  • 本関数はポリシーの存在確認およびOID取得の標準的な手段です。ポリシー定義の変更や削除を行う際は、この関数で取得したOIDを基点とすることが推奨されます。
