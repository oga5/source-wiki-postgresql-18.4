set_opfuncid

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/nodeFuncs.c:1867

目的

OpExpr ノード内の演算子関数 ID (opfuncid) が未設定である場合に、システムカタログから対応する関数 OID を検索して設定します。

シグネチャまたは形状

void set_opfuncid(OpExpr *opexpr)

入力と出力

  • **入力**: OpExpr *opexpr - 処理対象の演算子式ノードへのポインタ。
  • **出力**: なし（引数の opexpr 構造体を直接変更します）。

前提条件と不変条件

  • opexpr は有効な OpExpr 構造体を指している必要があります。
  • opexpr->opno には有効な演算子 OID が格納されている必要があります。

アルゴリズムまたは振る舞い

1. opexpr->opfuncid が InvalidOid であるかを確認します。 2. InvalidOid の場合、get_opcode(opexpr->opno) を呼び出し、演算子に対応する関数 OID を取得します。 3. 取得した OID を opexpr->opfuncid に代入します。 4. すでに opfuncid が設定されている場合は、何も行いません。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/nodeFuncs.c (複数箇所)
  • src/backend/optimizer/path/allpaths.c
  • src/backend/optimizer/path/costsize.c
  • src/backend/optimizer/path/equivclass.c
  • src/backend/optimizer/path/indxpath.c
  • src/backend/optimizer/plan/setrefs.c
  • **呼び出し先**:
  • get_opcode (システムカタログから演算子関数 OID を取得する関数)

関連データ構造

  • OpExpr: 演算子式を表すノード構造体。opno（演算子 OID）と opfuncid（関数 OID）を保持します。

失敗モードと境界条件

  • opexpr->opno が無効な場合、get_opcode の挙動に依存します。
  • opexpr が NULL の場合の挙動は定義されていません。

保守担当者向けメモ

  • 本関数は、ノードツリーの走査中に演算子関数の解決を確実に行うために使用されます。
  • fix_opfuncids などの関連関数と併せて、演算子ノードのメタデータ整合性を維持する役割を担います。
