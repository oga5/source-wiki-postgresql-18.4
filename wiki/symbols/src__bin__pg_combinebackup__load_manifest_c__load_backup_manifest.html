load_backup_manifest

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/load_manifest.c:104

目的

指定されたバックアップディレクトリから backup_manifest ファイルを読み込み、その内容を解析してメモリ上の manifest_data 構造体に展開する。

シグネチャまたは形状

manifest_data *load_backup_manifest(char *backup_directory);

入力と出力

  • **入力**: backup_directory (バックアップデータが格納されているディレクトリパス)
  • **出力**: 解析結果を保持する manifest_data 構造体へのポインタ。ファイルが存在しない場合は NULL を返す。

前提条件と不変条件

  • 指定されたディレクトリ内に backup_manifest ファイルが存在すること。
  • manifest_files_create によりハッシュテーブルが正常に初期化可能であること。
  • バージョン1のマニフェストは増分バックアップをサポートしていないため、パース時にエラーとして扱われる。

アルゴリズムまたは振る舞い

1. backup_directory/backup_manifest を開く。 2. ファイルサイズを取得し、それに基づき manifest_files_create でハッシュテーブルを初期化する。 3. JsonManifestParseContext を設定し、各種コールバック関数（combinebackup_per_file_cb 等）を登録する。 4. ファイルサイズに応じて以下のいずれかのパース処理を行う:

  • **小規模ファイル**: ファイル全体をメモリに読み込み json_parse_manifest で一括パースする。
  • **大規模ファイル**: json_parse_manifest_incremental_init を使用し、チャンク単位で読み込みながら json_parse_manifest_incremental_chunk で逐次パースする。

    • 5. パース完了後、メモリを解放し、構築された manifest_data を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_combinebackup/load_manifest.c 内の load_backup_manifests 関数。
  • **呼び出し先**: manifest_files_create, pg_malloc0, json_parse_manifest, json_parse_manifest_incremental_init, json_parse_manifest_incremental_chunk, json_parse_manifest_incremental_shutdown, pg_fatal, pg_log_warning 等。

関連データ構造

  • manifest_data: マニフェスト情報を保持するメイン構造体。
  • manifest_files_hash: ファイル情報を格納するハッシュテーブル。
  • JsonManifestParseContext: パース処理の状態とコールバックを保持するコンテキスト。

失敗モードと境界条件

  • ファイルが存在しない場合: ENOENT を検知し、警告を出力して NULL を返す。
  • ファイルのオープンや読み込みに失敗した場合: pg_fatal を呼び出しプロセスを終了する。
  • パースエラー: report_manifest_error を介してエラーが報告される。

保守担当者向けメモ

  • simplehash.h を使用しているため、ハッシュテーブルの型定義や関数名はマクロ生成されている点に留意すること。
  • 大規模なマニフェストを扱う際のメモリ効率のため、インクリメンタルなパース処理が実装されている。
  • ファイルの末尾にチェックサムが含まれるよう、チャンク読み込み時にサイズ調整が行われている。
