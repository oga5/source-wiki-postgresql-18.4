tuplestore_trim

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplestore.c:1423

目的

メモリ内に保持されているタプルのうち、どの読み取りポインタからも参照されなくなった古いタプルを解放し、メモリ使用量を削減する。

シグネチャまたは形状

void tuplestore_trim(Tuplestorestate *state)

入力と出力

  • **入力**: Tuplestorestate *state（対象となるタプルストアの状態構造体）
  • **出力**: なし（副作用としてメモリ解放および状態更新を行う）

前提条件と不変条件

  • state->eflags に EXEC_FLAG_REWIND が含まれている場合、本関数は何も行わない。
  • state->status が TSS_INMEM ではない場合、一時ファイルへの書き出しは行わず、何も行わない。
  • 呼び出し元が保持している可能性のある直前のタプルを保護するため、最も古い読み取りポインタの指す位置から1つ手前までのタプルを保持する。

アルゴリズムまたは振る舞い

1. **対象範囲の特定**: 全ての読み取りポインタ (readptrs) を走査し、最も古い current 位置を特定する。 2. **削除対象の決定**: oldest - 1 を削除対象数 (nremove) とする。 3. **メモリ解放**: memtuples 配列の memtupdeleted から nremove までの各タプルを pfree し、メモリを解放する。 4. **配列の再配置**:

  • 削除数が全タプル数の 1/8 未満であれば、配列の詰め直しは行わず終了する。
  • それ以外の場合、memmove を用いて配列を前方に詰め、memtupcount を更新する。

    • 5. **ポインタ調整**: 各読み取りポインタの current 位置を nremove 分だけ減算する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeMaterial.c
  • src/backend/executor/nodeWindowAgg.c
  • **呼び出し先**:
  • tuplestore_updatemax
  • pfree
  • GetMemoryChunkSpace
  • memmove

関連データ構造

  • Tuplestorestate: タプルストアの全体状態を管理する構造体。
  • TSReadPointer: 個別の読み取り位置を管理する構造体。

失敗モードと境界条件

  • nremove <= 0 の場合は何も行わずに終了する。
  • TSS_INMEM 以外の状態では処理をスキップする。
  • EXEC_FLAG_REWIND が設定されている場合、データの再読み取りが必要なためトリミングは無効化される。

保守担当者向けメモ

  • 本関数は memtuples 配列を直接操作するため、呼び出しタイミングによっては呼び出し元が保持しているタプルポインタが無効化される可能性がある点に注意すること。
  • nremove < memtupcount / 8 の場合に詰め直しをスキップするロジックは、memmove による計算コスト（O(N^2)）を回避するための最適化である。
