read_bytes

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/reconstruct.c:534

目的

指定された rfile から正確に指定バイト数を読み込み、バッファへ格納する。読み込みが完了しない場合やエラーが発生した場合にはプロセスを終了させる。

シグネチャまたは形状

static void read_bytes(rfile *rf, void *buffer, unsigned length)

入力と出力

  • rf: 読み込み対象のファイル情報を保持する rfile 構造体へのポインタ。
  • buffer: 読み込んだデータを格納するメモリ領域へのポインタ。
  • length: 読み込むバイト数。
  • 出力: なし（成功時はバッファにデータが書き込まれる）。

前提条件と不変条件

  • rf は有効な rfile 構造体を指していること。
  • rf->fd は読み込み可能なファイル記述子であること。

アルゴリズムまたは振る舞い

1. read(rf->fd, buffer, length) を呼び出し、指定されたバイト数の読み込みを試みる。 2. 読み込んだバイト数（rb）が要求した length と一致するか確認する。 3. rb != length の場合:

  • rb < 0 ならば、システムエラーとして pg_fatal を呼び出し、ファイル名と共にエラーメッセージを出力して終了する。
  • rb >= 0 ならば、期待したバイト数に満たなかったとして、読み込めたバイト数と要求バイト数を報告し、pg_fatal で終了する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_combinebackup/reconstruct.c 内の make_incremental_rfile 関数から呼び出されていることが確認されている。
  • 呼び出し先: システムコール read およびエラー報告用の pg_fatal。

関連データ構造

  • rfile: ファイル記述子 (fd) とファイル名 (filename) を保持する構造体。

失敗モードと境界条件

  • ファイルの終端に達して要求バイト数が読み込めない場合、pg_fatal によりプログラムが異常終了する。
  • ファイル読み込み中にシステムエラーが発生した場合、%m を含むエラーメッセージと共に異常終了する。

保守担当者向けメモ

  • 本関数は、読み込みが「完全に成功すること」を前提としたユーティリティである。部分的な読み込みを許容するロジックは含まれていないため、呼び出し側はファイルサイズが十分であることを事前に保証する必要がある。
