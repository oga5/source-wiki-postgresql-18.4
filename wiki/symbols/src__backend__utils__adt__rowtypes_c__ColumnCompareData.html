ColumnCompareData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/rowtypes.c:54

目的

複合型（Composite Types）や匿名レコードの比較処理において、各列（カラム）の型情報を保持し、比較演算を効率化するためのキャッシュ構造体です。

シグネチャまたは形状

typedef struct ColumnCompareData
{
    TypeCacheEntry *typentry;
} ColumnCompareData;

入力と出力

  • **入力**: TypeCacheEntry へのポインタ。これは utils/typcache.h で定義される型キャッシュエントリであり、比較に必要な型情報や関数ポインタを保持します。
  • **出力**: なし（データ構造体）。

前提条件と不変条件

  • RecordCompareData 構造体内の配列要素として使用されます。
  • 比較対象となる列の型情報が TypeCacheEntry に正しく格納されている必要があります。

アルゴリズムまたは振る舞い

  • 本構造体自体はデータ保持用です。
  • 実際の比較処理では、保持された typentry を介して、各列の型に対応する比較関数（等価性、大小比較など）をルックアップし、再帰的に実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/rowtypes.c 内のレコード比較関数群（record_cmp, record_eq 等）が、実行時に RecordCompareData を初期化する際に本構造体の配列を割り当てます。
  • **呼び出し先**: 本構造体は TypeCacheEntry を参照します。

関連データ構造

  • RecordCompareData: 本構造体を可変長配列として保持する親構造体。
  • TypeCacheEntry: 型のメタデータおよび比較関数を保持するキャッシュエントリ。

失敗モードと境界条件

  • 削除された列（attisdropped）は比較ロジックにおいてスキップされるため、本構造体は有効な列に対してのみ適切に初期化される必要があります。
  • メモリ割り当て時、ncols * sizeof(ColumnCompareData) として計算されるため、列数に応じた適切なメモリ確保が前提となります。

保守担当者向けメモ

  • typentry メンバは比較に必要な情報をすべて含んでいるため、追加のフィールドは不要と判断されています。
  • RecordCompareData の columns 配列として FLEXIBLE_ARRAY_MEMBER 形式で利用されているため、構造体のサイズ変更やメモリレイアウトの変更には注意してください。
