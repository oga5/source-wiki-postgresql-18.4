PgXmlErrorContext

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/adt/xml.c:117
  • **公開ヘッダー**: src/include/utils/xml.h

目的

libxml2 ライブラリで発生したエラーを捕捉し、PostgreSQL の ereport メカニズムへ安全に引き渡すためのコンテキストを保持します。

シグネチャまたは形状

struct PgXmlErrorContext {
    int magic;
    PgXmlStrictness strictness;
    bool err_occurred;
    StringInfoData err_buf;
    xmlStructuredErrorFunc saved_errfunc;
    void *saved_errcxt;
    xmlExternalEntityLoader saved_entityfunc;
};

入力と出力

  • **入力**: pg_xml_init 呼び出し時に PgXmlStrictness が設定されます。
  • **出力**: 蓄積されたエラーメッセージは err_buf (StringInfoData) に格納され、最終的に xml_ereport 等を通じて PostgreSQL のエラーログとして出力されます。

前提条件と不変条件

  • magic フィールドは、構造体の識別用として使用されます。
  • libxml2 のグローバルなエラーハンドラおよびエンティティローダーを一時的に退避し、処理終了後に復元することが前提となります。

アルゴリズムまたは振る舞い

1. **初期化**: pg_xml_init が呼び出されると、現在の libxml2 のエラーハンドラとエンティティローダーを saved_ フィールドに退避します。 2. **捕捉**: libxml2 実行中にエラーが発生すると、カスタムエラーハンドラが err_occurred を true に設定し、メッセージを err_buf に蓄積します。 3. **復元**: 処理完了後、退避していた元のハンドラを libxml2 に戻します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/xml2/xpath.c
  • contrib/xml2/xslt_proc.c
  • src/backend/utils/adt/xml.c (内部関数 xml_errsave, xml_xmlnodetoxmltype 等)
  • **呼び出し先**:
  • libxml2 のエラーハンドリング関数群（xmlStructuredErrorFunc 等）。

関連データ構造

  • StringInfoData: エラーメッセージを構築するための可変長文字列バッファ。
  • PgXmlStrictness: XML 解析の厳密さを定義する列挙型。

失敗モードと境界条件

  • libxml2 のグローバルな状態に依存するため、マルチスレッド環境や複雑なネスト構造でのエラーハンドラ切り替え時に競合が発生する可能性があります。
  • メモリ不足により err_buf の拡張が失敗した場合、PostgreSQL のメモリ管理機構によりエラーがスローされます。

保守担当者向けメモ

  • magic フィールドは構造体の整合性チェックに使用されます。
  • libxml2 のエラーハンドラはセッション単位で管理される必要があるため、本構造体を用いたハンドラの退避・復元処理は厳密に行う必要があります。
