scan_directory_ci

定義

  • 種別: function
  • 定義箇所: src/timezone/pgtz.c:150

目的

指定されたディレクトリ内を走査し、大文字小文字を区別しない比較（case-insensitive）によって、ターゲットとなるファイル名と一致するエントリを検索・正規化します。

シグネチャまたは形状

static bool scan_directory_ci(const char *dirname, const char *fname, int fnamelen, char *canonname, int canonnamelen);

入力と出力

  • **入力**:
  • dirname: 検索対象のディレクトリパス。
  • fname: 検索するファイル名。
  • fnamelen: fname の長さ。
  • canonname: 一致したファイル名を格納するバッファ。
  • canonnamelen: canonname バッファのサイズ。
  • **出力**:
  • bool: 一致するファイルが見つかった場合は true、それ以外は false。
  • canonname: 一致した場合、ファイルシステム上の実際のファイル名が格納されます。

前提条件と不変条件

  • dirname は AllocateDir で開くことが可能な有効なディレクトリパスである必要があります。
  • . で始まるファイルやディレクトリは、セキュリティ上の理由からスキップされます。

アルゴリズムまたは振る舞い

1. AllocateDir を使用して dirname を開きます。 2. ReadDirExtended を用いてディレクトリ内のエントリを順次読み込みます。 3. 各エントリに対し、d_name[0] == '.' であればスキップします。 4. pg_strncasecmp を使用し、fname とディレクトリ内のエントリ名を大文字小文字を区別せずに比較します。 5. 一致した場合、strlcpy で canonname にファイル名をコピーし、found を true に設定してループを終了します。 6. FreeDir でディレクトリハンドルを解放し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/timezone/pgtz.c 内の関数（pg_open_tzfile 等での利用が示唆されています）。
  • **呼び出し先**: AllocateDir, ReadDirExtended, pg_strncasecmp, strlcpy, FreeDir。

関連データ構造

  • DIR: ディレクトリ操作用ハンドル。
  • struct dirent: ディレクトリ内のエントリ情報。

失敗モードと境界条件

  • ディレクトリが開けない場合、AllocateDir の挙動に依存します。
  • canonnamelen が一致したファイル名に対して不足している場合、strlcpy の仕様に従い切り詰めが発生する可能性があります。
  • 該当するファイルが存在しない場合、false を返します。

保守担当者向けメモ

  • 本関数はタイムゾーンファイル検索のセキュリティ対策として、隠しファイル（. で始まるもの）へのアクセスを明示的に除外しています。
  • 大文字小文字を区別しない比較を行っているため、ファイルシステムがケースセンシティブであっても、PostgreSQL 側で一貫したタイムゾーン名解決を可能にしています。
