IsBinaryCoercibleWithCast

定義

  • src/backend/parser/parse_coerce.c:3044

目的

2つの型の間で、バイナリ互換性のあるキャスト（暗黙的な変換）が可能かどうかを判定し、可能な場合はそのキャスト定義のOIDを取得する。

シグネチャまたは形状

bool IsBinaryCoercibleWithCast(Oid srctype, Oid targettype, Oid *castoid)

入力と出力

  • **入力**:
  • srctype (Oid): 変換元の型。
  • targettype (Oid): 変換先の型。
  • **出力**:
  • 戻り値 (bool): 変換可能であれば true。
  • *castoid (Oid): 変換に使用する pg_cast エントリのOID。変換が不要またはバイナリ互換でない場合は InvalidOid。

前提条件と不変条件

  • srctype がドメイン型の場合、内部で getBaseType により基底型に解決される。
  • srctype と targettype が同一の場合、常に true を返す。
  • 多相型（ANYELEMENT 等）や複合型（RECORD 等）に対する特定のルールが適用される。

アルゴリズムまたは振る舞い

1. **同一型チェック**: srctype == targettype ならば true を返す。 2. **多相型チェック**: targettype が ANY 系（ANYOID, ANYELEMENTOID 等）の場合、常に true を返す。 3. **ドメイン解決**: srctype を基底型に変換し、再度同一型チェックを行う。 4. **特殊型チェック**: targettype が ANYARRAYOID などの特殊な型である場合、srctype が対応するカテゴリ（配列、列挙型、範囲型等）に属するかを判定する。 5. **カタログ検索**: 上記に該当しない場合、pg_cast システムカタログを CASTSOURCETARGET インデックスで検索する。 6. **判定**: 検索されたキャストの castmethod が COERCION_METHOD_BINARY かつ castcontext が COERCION_CODE_IMPLICIT である場合にのみ true とし、そのOIDを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/functioncmds.c
  • src/backend/parser/parse_coerce.c
  • **呼び出し先**:
  • getBaseType
  • type_is_array, type_is_enum, type_is_range, type_is_multirange
  • ISCOMPLEX, is_complex_array
  • SearchSysCache2 (CASTSOURCETARGET)
  • ReleaseSysCache

関連データ構造

  • pg_cast (システムカタログ)
  • Oid (型およびキャストの識別子)
  • HeapTuple (システムカタログ検索結果)

失敗モードと境界条件

  • pg_cast に該当するエントリが存在しない場合、false を返す。
  • castmethod が COERCION_METHOD_BINARY でない、または castcontext が暗黙的（IMPLICIT）でない場合、バイナリ互換とみなされず false を返す。

保守担当者向けメモ

  • 本関数は「バイナリ互換かつ暗黙的」なキャストのみを対象とする。明示的なキャスト（CAST 構文が必要なもの）は対象外である点に注意すること。
  • pg_cast の検索結果は ReleaseSysCache で必ず解放される設計となっている。
