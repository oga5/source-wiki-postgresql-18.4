truncate_check_rel

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tablecmds.c:2371

目的

TRUNCATE コマンドの実行対象となるリレーションが、切り詰め操作に対して妥当かつ許可されているかを検証する。

シグネチャまたは形状

static void truncate_check_rel(Oid relid, Form_pg_class reltuple)

入力と出力

  • **入力**:
  • relid: 対象リレーションの OID。
  • reltuple: 対象リレーションの pg_class 行データ。
  • **出力**: なし（正常終了時）。エラー発生時は ereport により処理を中断する。

前提条件と不変条件

  • 対象が外部テーブルの場合、使用している外部データラッパー（FDW）が ExecForeignTruncate をサポートしている必要がある。
  • システムカタログの切り詰めは原則禁止されている（allow_system_table_mods が有効な場合を除く）。
  • 例外として、pg_largeobject は pg_upgrade 中（IsBinaryUpgrade が真）であれば切り詰めが許可される。

アルゴリズムまたは振る舞い

1. **リレーション種別の検証**:

  • 通常のテーブルまたはパーティションテーブル以外であれば、エラーを投げる。

    • 2. **システムカタログ保護の検証**:

  • システムカタログである場合、allow_system_table_mods が無効かつ pg_largeobject へのバイナリアップグレード時ではない場合にエラーを投げる。

    • 3. **フックの実行**:

  • InvokeObjectTruncateHook(relid) を呼び出し、登録されたオブジェクト切り詰めフックを実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/tablecmds.c 内の TRUNCATE 関連処理（TRUNCATE コマンド実行時の各リレーションに対する事前チェックとして利用）。
  • **呼び出し先**:
  • GetForeignServerIdByRelId, GetForeignServerIdByRelId (FDW 関連)
  • IsSystemClass (システムカタログ判定)
  • InvokeObjectTruncateHook (フック実行)
  • ereport (エラー報告)

関連データ構造

  • Form_pg_class: pg_class システムカタログの行構造体。
  • FdwRoutine: 外部データラッパーの関数ポインタ群を保持する構造体。

失敗モードと境界条件

  • ERRCODE_FEATURE_NOT_SUPPORTED: FDW が TRUNCATE をサポートしていない場合。
  • ERRCODE_WRONG_OBJECT_TYPE: テーブルやパーティションテーブル以外のリレーションを対象とした場合。
  • ERRCODE_INSUFFICIENT_PRIVILEGE: 権限のないシステムカタログを切り詰めようとした場合。

保守担当者向けメモ

  • 本関数は TRUNCATE コマンドの実行前チェックの要所です。新しいリレーション種別や制約を追加する際は、本関数での検証ロジックが漏れていないか確認してください。
  • pg_largeobject の例外処理は pg_upgrade に強く依存しているため、変更時は注意が必要です。
