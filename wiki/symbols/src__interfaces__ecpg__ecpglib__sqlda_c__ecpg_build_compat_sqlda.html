ecpg_build_compat_sqlda

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/ecpg/ecpglib/sqlda.c:204

目的

Informix 互換モードにおける SQLDA（SQL Descriptor Area）構造体を、PGresult から取得したメタデータに基づいて初期化および構築します。

シグネチャまたは形状

struct sqlda_compat *ecpg_build_compat_sqlda(int line, PGresult *res, int row, enum COMPAT_MODE compat);

入力と出力

  • **入力**:
  • line: 呼び出し元の行番号（ログ出力用）。
  • res: 変換元の PGresult オブジェクト。
  • row: 対象とする行インデックス。
  • compat: 互換モードの指定。
  • **出力**:
  • 成功時: 初期化済みの struct sqlda_compat へのポインタ。
  • 失敗時: NULL（メモリ確保失敗時）。

前提条件と不変条件

  • SQLDA 構造体は、メタデータとデータ領域が単一の連続したメモリブロックとして確保されます。
  • 戻り値の sqlda->desc_occ には、確保されたメモリの全サイズが格納されます。
  • 呼び出し側は、返されたポインタに対して free() を実行することでメモリを解放できます。

アルゴリズムまたは振る舞い

1. sqlda_compat_total_size を呼び出し、構造体本体、フィールド定義、列名文字列に必要な合計サイズを算出します。 2. ecpg_alloc を用いてメモリを確保し、memset でゼロ初期化します。 3. sqlda 構造体の sqld（フィールド数）および sqlvar（フィールド配列へのポインタ）を設定します。 4. PGresult の各列に対してループ処理を行い、以下をマッピングします。

  • sqltype: sqlda_dynamic_type による型変換。
  • sqlname: PGresult から列名文字列をコピー。
  • sqlxid: PostgreSQL の型 ID。
  • sqltypelen: 列のサイズ。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/descriptor.c:941
  • src/interfaces/ecpg/ecpglib/execute.c:1751
  • **呼び出し先**:
  • sqlda_compat_total_size
  • ecpg_alloc
  • ecpg_log
  • sqlda_dynamic_type
  • PQnfields, PQftype, PQfname, PQfsize (libpq 関数)

関連データ構造

  • struct sqlda_compat: Informix 互換の SQLDA 構造体。
  • struct sqlvar_compat: 各列の情報を保持する構造体。

失敗モードと境界条件

  • ecpg_alloc がメモリ確保に失敗した場合、NULL を返します。
  • PGresult が空の場合、sqld は 0 となり、ループは実行されません。

保守担当者向けメモ

  • 本関数は Informix 互換モード専用です。ネイティブモードの場合は ecpg_build_native_sqlda を使用してください。
  • sqlvar[i].sqlformat は将来の拡張用に予約されており、現在はコメントアウトされています。
  • メモリレイアウトは sqlda_compat_total_size の計算ロジックと密接に結合しているため、構造体定義を変更する際は両方の整合性を確認してください。
