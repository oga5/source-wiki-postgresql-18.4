hk_breadth_search

定義

  • 種別: function
  • 定義箇所: src/backend/lib/bipartite_match.c:92

目的

Hopcroft-Karp アルゴリズムにおいて、BFS（幅優先探索）を用いて増大路の距離を計算し、探索の基盤を構築する。

シグネチャまたは形状

static bool hk_breadth_search(BipartiteMatchState *state)

入力と出力

  • **入力**: BipartiteMatchState *state（マッチング計算の状態を保持する構造体）。
  • **出力**: bool 型。増大路が見つかった場合（distance[0] != HK_INFINITY）に true を返し、それ以外は false を返す。

前提条件と不変条件

  • state->adjacency は short ** 型の隣接リストであり、各要素の先頭は隣接ノード数であること。
  • ノード数は SHRT_MAX 未満であること（distance 配列の初期値として SHRT_MAX を無限大として利用するため）。
  • distance[0] はダミーノード（未マッチノードの終端）の距離として使用される。

アルゴリズムまたは振る舞い

1. distance[0] を HK_INFINITY に初期化する。 2. 未マッチのノード（pair_uv[u] == 0）をキューに入れ、距離を 0 に設定する。それ以外のノードの距離は HK_INFINITY とする。 3. キューが空になるまで以下を繰り返す：

  • キューからノード u を取り出す。
  • distance[u] < distance[0] の場合、u に隣接するノードを辿り、未訪問のノード u_next があれば距離を 1 + distance[u] に更新してキューに追加する。

    • 4. 最終的に distance[0] が更新されていれば、増大路が存在すると判断する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/lib/bipartite_match.c 内のメインループ（while (hk_breadth_search(state))）。
  • **呼び出し先**: 直接的な関数呼び出しは行わないが、state 構造体の各フィールド（pair_uv, pair_vu, adjacency 等）を参照する。

関連データ構造

  • BipartiteMatchState: 計算状態、距離配列、キュー、マッチングペアを保持する。

失敗モードと境界条件

  • adjacency[u] が NULL の場合、隣接ノード数は 0 として扱われる。
  • 増大路が存在しない場合、distance[0] は HK_INFINITY のままとなり、関数は false を返す。
  • Assert(qhead < usize + 2) により、キューのオーバーフローをチェックしている。

保守担当者向けメモ

  • 本関数は BipartiteMatch アルゴリズムの反復ステップの一部である。
  • distance 配列の更新ロジックは、グラフの最短増大路を特定するために不可欠である。
  • 呼び出し元で CHECK_FOR_INTERRUPTS 等の安全策が講じられていることを前提としている。
