replace_percent_placeholders

定義

  • 種別: function
  • 定義箇所: src/common/percentrepl.c:58

目的

文字列内の % で始まるプレースホルダーを、可変長引数で提供される値に置換し、新しい文字列を生成します。主に GUC パラメータの文字列展開に使用されます。

シグネチャまたは形状

char *replace_percent_placeholders(const char *instr, const char *param_name, const char *letters, ...);

入力と出力

  • **入力**:
  • instr: 置換対象の文字列。
  • param_name: エラー報告時に使用するパラメータ名。
  • letters: プレースホルダーとして認識する文字のリスト（例: "frp"）。
  • ...: letters に対応する置換後の文字列（char *）。
  • **出力**:
  • 置換後の文字列（char *）。StringInfo API を使用して構築されます。

前提条件と不変条件

  • すべての置換値は char * 型である必要があります。
  • letters に含まれる各文字に対応する値が、可変長引数として順番に提供されている必要があります。
  • NULL 値が渡された場合、置換は行われずエラーとなります。

アルゴリズムまたは振る舞い

1. initStringInfo で結果格納用バッファを初期化します。 2. instr を一文字ずつ走査します。 3. % を検出した場合:

  • 次の文字が % なら、単一の % としてバッファに追加します。
  • 次の文字が終端なら、エラーを報告して終了します。
  • それ以外の場合、letters を走査して一致する文字を探し、対応する可変長引数の値（char *）をバッファに追加します。

    • 4. % 以外はそのままバッファに追加します。 5. 最終的な文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:

  • src/backend/access/transam/xlogarchive.c
  • src/backend/archive/shell_archive.c

  • **呼び出し先**:

  • ereport (バックエンド) または pg_log_error / exit (フロントエンド)

関連データ構造

  • StringInfoData: 動的文字列構築に使用される構造体。

失敗モードと境界条件

  • **不正なプレースホルダー**: letters に定義されていない文字が % の後に続いた場合、エラーを報告して終了します。
  • **不完全なエスケープ**: 文字列の末尾が % で終わっている場合、エラーを報告して終了します。
  • **NULL 値**: letters に対応する引数が NULL の場合、エラーを報告して終了します。

保守担当者向けメモ

  • FRONTEND マクロの定義有無により、エラー報告の仕組み（ereport vs pg_log_error）が切り替わります。
  • 本関数はすべての置換値が事前に準備できている場合に適しています。複雑な条件分岐や型混在が必要なケースには対応していません。
