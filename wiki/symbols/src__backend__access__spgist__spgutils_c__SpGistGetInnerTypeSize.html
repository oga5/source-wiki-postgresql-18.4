SpGistGetInnerTypeSize

定義

  • src/backend/access/spgist/spgutils.c:778

目的

SP-GiSTインデックスの内部タプル（Inner Tuple）を構成するデータ要素の、アライメント調整済みのサイズを算出します。

シグネチャまたは形状

unsigned int SpGistGetInnerTypeSize(SpGistTypeDesc *att, Datum datum)

入力と出力

  • **入力**:
  • att: 型情報を保持する SpGistTypeDesc 構造体。
  • datum: サイズを測定対象とするデータ値。
  • **出力**:
  • unsigned int: MAXALIGN 適用後のバイトサイズ。

前提条件と不変条件

  • att が指す構造体には、attbyval（値渡しフラグ）および attlen（固定長型の長さ）が正しく設定されている必要があります。
  • 可変長型（attlen <= 0）の場合、datum は有効なポインタとして参照可能である必要があります。

アルゴリズムまたは振る舞い

1. att->attbyval が真の場合、サイズを sizeof(Datum) とします。 2. attbyval が偽で att->attlen > 0 の場合、固定長として attlen をサイズとします。 3. 上記以外（可変長型）の場合、VARSIZE_ANY(datum) を使用して実際のデータサイズを取得します。 4. 最終的に MAXALIGN() マクロを適用し、メモリ配置上の境界に合わせたサイズを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgutils.c 内の内部タプル構築処理（ラベルやプレフィックスのサイズ計算）。
  • **呼び出し先**:
  • VARSIZE_ANY マクロ（可変長データの場合）。
  • MAXALIGN マクロ。

関連データ構造

  • SpGistTypeDesc: 型の属性（長さ、値渡しフラグ）を定義する構造体。
  • Datum: PostgreSQLの汎用データ表現型。

失敗モードと境界条件

  • attlen <= 0 かつ datum が無効なポインタである場合、VARSIZE_ANY の呼び出しによりセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数はSP-GiSTの物理的なタプルレイアウトに依存しています。新しいデータ型をサポートする際、attlen や attbyval の設定が適切でないと、インデックスの破損やメモリ破壊を招く恐れがあるため注意してください。
