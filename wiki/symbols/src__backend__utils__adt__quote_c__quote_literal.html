quote_literal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/quote.c:77

目的

SQL文を動的に構築する際、入力された文字列を単一引用符（'）で囲み、内部の引用符を適切にエスケープすることで、SQLインジェクションを防ぎ安全な文字列リテラルとして利用可能にする。

シグネチャまたは形状

Datum quote_literal(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される text 型の文字列。
  • **出力**: エスケープ処理が施され、単一引用符で囲まれた text 型の文字列を Datum として返す。

前提条件と不変条件

  • 入力文字列は text 型として正しく初期化されている必要がある。
  • standard_conforming_strings の設定に関わらず、常に安全なエスケープ形式（単一引用符の二重化）を適用する。

アルゴリズムまたは振る舞い

1. 入力文字列の長さを取得する。 2. 最悪のケース（全文字が引用符である場合）を想定し、十分なメモリ領域を palloc で確保する。 3. quote_literal_internal を呼び出し、以下の変換を行う：

  • 文字列を単一引用符で囲む。
  • 文字列内の単一引用符（'）を二重化（''）する。
  • バックスラッシュが含まれる場合は ESCAPE_STRING_SYNTAX を付与する。

    • 4. 変換後の文字列サイズを SET_VARSIZE で設定し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_subscription.c
  • src/bin/pg_dump/pg_dump.c
  • SQLクエリ内からの直接呼び出し（DirectFunctionCall1 等）
  • **呼び出し先**:
  • quote_literal_internal（内部的なエスケープ処理の実行）
  • palloc（メモリ確保）

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • PG_FUNCTION_ARGS: SQL関数呼び出し規約に基づく引数コンテナ。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • 入力文字列が非常に長い場合、確保されるバッファサイズが大きくなる可能性があるが、最悪ケースを想定した固定的な計算により安全性を優先している。

保守担当者向けメモ

  • 本関数は quote_literal_internal に処理を委譲しているため、エスケープルールの変更が必要な場合は quote_literal_internal を参照すること。
  • メモリ効率よりも安全性を優先した実装となっている。
