ReplaceVarsFromTargetList_context

定義

  • 種別: struct
  • 定義箇所: src/backend/rewrite/rewriteManip.c:1753

目的

ターゲットリスト（targetlist）内の項目を使用して、クエリツリー内の特定の範囲テーブルエントリ（RTE）を参照する変数を置換する際の処理状態を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    RangeTblEntry *target_rte;
    List          *targetlist;
    int            result_relation;
    ReplaceVarsNoMatchOption nomatch_option;
    int            nomatch_varno;
} ReplaceVarsFromTargetList_context;

入力と出力

本構造体は関数への引数として渡されるコンテキストオブジェクトです。

  • **入力**: 置換対象の target_rte、置換元となる targetlist、および置換ルールを制御するオプション群。
  • **出力**: なし（構造体自体は状態保持用）。

前提条件と不変条件

  • targetlist は、置換対象となる式を含むリストである必要があります。
  • nomatch_option は、一致する変数が見つからなかった場合の挙動を決定します。

アルゴリズムまたは振る舞い

この構造体は、replace_rte_variables 等の走査関数において、コールバック関数へ渡される context->callback_arg として利用されます。走査中に遭遇した変数が target_rte に一致する場合、targetlist 内の対応する項目を用いて変数を置換するロジックの制御に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/rewrite/rewriteManip.c:1959 にてインスタンス化されます。
  • **呼び出し先**: src/backend/rewrite/rewriteManip.c:1766 にて、コールバック関数内で context->callback_arg からキャストして参照されます。

関連データ構造

  • RangeTblEntry: 置換対象を特定するための範囲テーブルエントリ。
  • ReplaceVarsNoMatchOption: 一致しない場合の動作を定義する列挙型。
  • List: 置換元となるターゲットリスト。

失敗モードと境界条件

  • target_rte が NULL の場合や、targetlist が空の場合の挙動は、呼び出し側の走査ロジックに依存します。
  • nomatch_option の設定により、一致しない変数に遭遇した際にエラーを発生させるか、そのまま維持するかが決定されます。

保守担当者向けメモ

  • 本構造体は rewriteManip.c 内の変数置換ロジック専用です。変更を加える際は、replace_rte_variables のコールバック実装との整合性に注意してください。
