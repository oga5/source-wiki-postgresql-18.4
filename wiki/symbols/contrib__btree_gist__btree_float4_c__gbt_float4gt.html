gbt_float4gt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_float4.c:28

目的

GiSTインデックスにおける float4 型の「より大きい（greater than）」比較演算を提供します。

シグネチャまたは形状

static bool gbt_float4gt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の float4 型値へのポインタ（void * として渡される）。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • bool: *a > *b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は、有効な float4 型のメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. void * 型の引数 a および b を float4 * にキャストします。 2. ポインタが指す値をデリファレンスし、標準の比較演算子 > を用いて大小比較を行います。 3. 比較結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_float4.c 内の静的変数 tinfo の初期化時に参照されています。
  • **呼び出し先**: 直接的な関数呼び出しはありません（標準の比較演算子を使用）。

関連データ構造

  • float4: PostgreSQLの単精度浮動小数点数型。
  • tinfo: gbtree_ninfo 型の静的変数。本関数を比較関数として保持し、btree_utils_num.c の汎用関数へ提供します。

失敗モードと境界条件

  • a または b が NULL ポインタである場合、セグメンテーション違反が発生する可能性があります。
  • float4 型の特性上、NaN（非数）が含まれる場合の比較結果は、標準的な浮動小数点演算のルールに従います。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で定義された汎用的なGiSTインデックスロジックから呼び出されるラッパーです。
  • ロジックを変更する場合は、他の比較関数（gbt_float4lt, gbt_float4eq 等）との一貫性を維持してください。
