InitWalSender

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walsender.c:296

目的

WAL 送信プロセス（WalSender）の初期化を行い、共有メモリへの登録、リソース所有者の作成、および Postmaster へのプロセス状態の通知を行う。

シグネチャまたは形状

void InitWalSender(void)

入力と出力

  • 入力: なし（グローバル変数 MyDatabaseId や MyProc を参照）
  • 出力: なし（副作用として共有メモリおよびプロセス状態を更新）

前提条件と不変条件

  • 本関数呼び出し以降、当該プロセスは WAL レコードを書き込んではならない。
  • Postmaster への通知後は、シャットダウンシーケンスにおいて最後に終了するプロセスとして扱われる。

アルゴリズムまたは振る舞い

1. RecoveryInProgress() を用いて am_cascading_walsender を設定する。 2. InitWalSenderSlot() を呼び出し、共有メモリ上の WAL 送信スロットを確保する。 3. CreateAuxProcessResourceOwner() を呼び出し、ベースバックアップ等のリソース管理用オーナーを作成する。 4. MarkPostmasterChildWalSender() および SendPostmasterSignal(PMSIGNAL_ADVANCE_STATE_MACHINE) を呼び出し、Postmaster に WAL 送信プロセスであることを通知する。 5. データベース未指定（MyDatabaseId == InvalidOid）の場合、ProcArrayLock を取得し、当該プロセスが全データベースの Vacuum ホライゾンに影響を与えるよう PROC_AFFECTS_ALL_HORIZONS フラグをセットする。 6. TopMemoryContext 上にラグ追跡用の LagTracker 構造体を割り当てる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tcop/postgres.c:4327
  • 呼び出し先: InitWalSenderSlot, CreateAuxProcessResourceOwner, MarkPostmasterChildWalSender, SendPostmasterSignal, MemoryContextAllocZero, LWLockAcquire, LWLockRelease

関連データ構造

  • WalSnd: 共有メモリ上の制御構造体。
  • LagTracker: ラグ追跡用のメモリバッファ。
  • ProcGlobal: プロセス状態を保持するグローバル配列。

失敗モードと境界条件

  • 共有メモリの確保やロック取得に失敗した場合、PostgreSQL の標準的なエラーハンドリング（elog 等）によりプロセスが異常終了する可能性がある。

保守担当者向けメモ

  • 本関数はプロセス初期化のクリティカルパスである。Postmaster への通知後はシャットダウン時の挙動が変更されるため、安易な呼び出し順序の変更は避けること。
  • PROC_AFFECTS_ALL_HORIZONS の設定は、物理レプリケーションにおける Hot Standby フィードバックの有効化に直結する。
