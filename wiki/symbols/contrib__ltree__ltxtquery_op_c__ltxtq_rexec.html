ltxtq_rexec

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltxtquery_op.c:103

目的

ltree 型と ltxtquery 型の照合演算子 @ において、引数の順序を逆転させた形式（ltxtquery @ ltree）をサポートするためのラッパー関数です。

シグネチャまたは形状

Datum ltxtq_rexec(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（PostgreSQL の標準的な関数引数インターフェース）。第1引数に ltxtquery、第2引数に ltree を受け取ります。
  • 出力: Datum 型。照合結果（真偽値）を返します。

前提条件と不変条件

  • ltxtq_exec 関数が利用可能である必要があります。
  • PostgreSQL の関数呼び出し規約に従い、引数は PG_GETARG_DATUM マクロで取得されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_DATUM(1)（第2引数）と PG_GETARG_DATUM(0)（第1引数）を取得します。 2. DirectFunctionCall2 を使用して、引数を入れ替えた状態で ltxtq_exec を呼び出します。 3. ltxtq_exec の戻り値をそのまま返します。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQL の SQL エンジン（演算子 @ の定義を通じて呼び出されます）。
  • 呼び出し先: ltxtq_exec（直接の呼び出し）。

関連データ構造

  • ltree: 評価対象のパスデータ。
  • ltxtquery: 評価対象のブール検索クエリ。

失敗モードと境界条件

  • 内部で呼び出される ltxtq_exec の失敗モードに依存します。
  • 引数が正しく渡されない場合、PG_GETARG_DATUM マクロの挙動によりエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は単なる引数入れ替えのラッパーであり、ロジックの本体は ltxtq_exec に存在します。
  • PG_FUNCTION_INFO_V1(ltxtq_rexec) が定義されており、PostgreSQL の関数マネージャに登録されています。
