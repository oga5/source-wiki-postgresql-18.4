RE_execute

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/regexp.c:323

目的

入力文字列をワイド文字形式に変換し、POSIX正規表現エンジンを用いてマッチングを実行するラッパー関数です。

シグネチャまたは形状

static bool RE_execute(regex_t *re, char *dat, int dat_len, int nmatch, regmatch_t *pmatch)

入力と出力

  • **入力**:
  • re: コンパイル済みの正規表現オブジェクト (regex_t*)。
  • dat: 対象となる文字列 (char*)。
  • dat_len: 文字列の長さ。
  • nmatch: マッチング結果を格納する pmatch 配列のサイズ。
  • pmatch: マッチング位置情報を格納する構造体配列。
  • **出力**:
  • bool: マッチした場合は true、それ以外は false を返します。

前提条件と不変条件

  • dat は有効な文字列ポインタである必要があります。
  • re は事前に pg_regcomp 等で初期化された有効な正規表現オブジェクトである必要があります。
  • 内部で palloc を使用するため、メモリコンテキストが有効である必要があります。

アルゴリズムまたは振る舞い

1. 入力文字列 dat を pg_wchar 型のワイド文字配列に変換します（pg_mb2wchar_with_len を使用）。 2. 変換されたワイド文字データを用いて RE_wchar_execute を呼び出し、正規表現マッチングを実行します。 3. 変換に使用した一時メモリを pfree で解放します。 4. マッチング結果を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/regexp.c 内の複数の箇所（正規表現関数実装部）。
  • **呼び出し先**:
  • pg_mb2wchar_with_len (文字列変換)
  • RE_wchar_execute (正規表現実行)
  • palloc / pfree (メモリ管理)

関連データ構造

  • regex_t: POSIX正規表現エンジンのコンパイル済みパターン構造体。
  • regmatch_t: マッチングの開始・終了位置を保持する構造体。
  • pg_wchar: PostgreSQL内部で使用されるワイド文字型。

失敗モードと境界条件

  • メモリ不足時: palloc がエラーを発生させます。
  • 不正なマルチバイト文字: pg_mb2wchar_with_len の挙動に依存します。
  • nmatch が不適切な場合: RE_wchar_execute 経由で正規表現エンジンがエラーを返す可能性があります。

保守担当者向けメモ

  • 本関数は、マルチバイト文字列を正規表現エンジンが処理可能なワイド文字形式へ変換するブリッジの役割を果たします。
  • パフォーマンス向上のため、頻繁に呼び出される場合はメモリ割り当てのオーバーヘッドに留意してください。
