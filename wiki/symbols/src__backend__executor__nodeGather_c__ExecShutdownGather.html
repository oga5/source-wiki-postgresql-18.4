ExecShutdownGather

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGather.c:417

目的

Gatherノードに関連する並列実行リソース（バックグラウンドワーカーおよび共有メモリコンテキスト）を解放し、クリーンアップを行う。

シグネチャまたは形状

void ExecShutdownGather(GatherState *node)

入力と出力

  • **入力**: GatherState *node - 終了対象のGatherノードの状態構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • node は有効な GatherState 構造体を指している必要がある。
  • node->pei が非NULLの場合、それは有効な並列実行コンテキストを指している必要がある。

アルゴリズムまたは振る舞い

1. ExecShutdownGatherWorkers(node) を呼び出し、関連するすべての並列ワーカーを停止させる。 2. node->pei（並列実行情報）がNULLでない場合、ExecParallelCleanup(node->pei) を呼び出して共有メモリ上の並列実行コンテキストを破棄する。 3. 破棄後、node->pei を NULL に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execProcnode.c:803
  • src/backend/executor/nodeGather.c:254
  • **呼び出し先**:
  • ExecShutdownGatherWorkers
  • ExecParallelCleanup

関連データ構造

  • GatherState: Gatherノードの実行状態を保持する構造体。
  • ParallelExecutorInfo (pei): 並列実行に関連する共有メモリやワーカーの状態を管理する構造体。

失敗モードと境界条件

  • node がNULLの場合の挙動は定義されていない（セグメンテーション違反の可能性がある）。
  • node->pei が既にNULLである場合、ExecParallelCleanup はスキップされるため安全である。

保守担当者向けメモ

  • 本関数は ExecEndGather や ExecReScanGather 等の終了処理フローの一部として呼び出される。
  • 並列実行リソースのリークを防ぐため、Gatherノードのライフサイクル終了時には必ず本関数が実行されることを保証する必要がある。
