GetSQLCurrentTime

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/date.c:349

目的

SQLの CURRENT_TIME および CURRENT_TIME(n) 式を実装し、現在の時刻とタイムゾーンオフセットを含む TimeTzADT 構造体を生成する。

シグネチャまたは形状

TimeTzADT *GetSQLCurrentTime(int32 typmod)

入力と出力

  • **入力**:
  • typmod (int32): 時刻の精度（小数点以下の桁数）を指定する修飾子。
  • **出力**:
  • TimeTzADT *: 現在時刻とタイムゾーンオフセットを格納した、メモリ割り当て済みの構造体へのポインタ。

前提条件と不変条件

  • palloc を使用してメモリを確保するため、呼び出し元は適切なメモリコンテキストで実行されている必要がある。
  • typmod は anytime_typmod_check 等で検証済みの値であることが想定される。

アルゴリズムまたは振る舞い

1. GetCurrentTimeUsec を呼び出し、現在の時刻（pg_tm）、マイクロ秒単位の端数（fsec）、およびタイムゾーンオフセット（tz）を取得する。 2. palloc を使用して TimeTzADT 構造体用のメモリを確保する。 3. tm2timetz を使用して、取得した時刻情報を TimeTzADT 構造体に変換・格納する。 4. AdjustTimeForTypmod を呼び出し、指定された typmod に基づいて時刻の精度を調整する。 5. 構築された TimeTzADT 構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c: SQL式の実行時に呼び出される。
  • **呼び出し先**:
  • GetCurrentTimeUsec: 現在時刻の取得。
  • palloc: メモリ確保。
  • tm2timetz: 時刻情報の構造体への変換。
  • AdjustTimeForTypmod: 精度調整。

関連データ構造

  • TimeTzADT: 時刻とタイムゾーンオフセットを保持する構造体。
  • pg_tm: 日時情報を保持する標準的な構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • typmod が不正な値の場合、AdjustTimeForTypmod 内で予期しない動作やエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は CURRENT_TIME の実装であるため、システムクロックの変更やタイムゾーン設定の影響を受ける。
  • TimeTzADT は palloc されたメモリを返すため、呼び出し側でのメモリリークに注意すること。
