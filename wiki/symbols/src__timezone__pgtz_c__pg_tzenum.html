pg_tzenum

定義

  • 種別: struct
  • 定義箇所: src/timezone/pgtz.c:385

目的

利用可能なタイムゾーンをファイルシステムから再帰的に列挙するためのイテレータ状態を保持する構造体です。

シグネチャまたは形状

struct pg_tzenum
{
    int         baselen;
    int         depth;
    DIR        *dirdesc[MAX_TZDIR_DEPTH];
    char       *dirname[MAX_TZDIR_DEPTH];
    struct pg_tz tz;
};

入力と出力

  • **入力**: pg_tzenumerate_start() によってメモリが割り当てられ、初期化されます。
  • **出力**: pg_tzenumerate_next() を通じて、走査中の各タイムゾーンに対応する pg_tz 構造体へのポインタが返されます。

前提条件と不変条件

  • dirdesc および dirname 配列は MAX_TZDIR_DEPTH までの深さでディレクトリ階層を管理します。
  • pg_tzenumerate_end() を呼び出すことで、割り当てられたリソース（ディレクトリ記述子など）を適切に解放する必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: pg_tzenumerate_start() が palloc0 を使用して構造体を確保し、走査を開始します。 2. **走査**: pg_tzenumerate_next() が呼び出されるたびに、dirdesc に保持されたディレクトリハンドルを使用してファイルシステムを再帰的に走査します。 3. **状態保持**: 現在のディレクトリ深さ (depth) とベースパスの長さ (baselen) を維持し、タイムゾーンファイルが見つかるまで探索を継続します。 4. **終了**: pg_tzenumerate_end() が呼び出されると、保持しているディレクトリハンドルをクローズし、メモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/datetime.c
  • **呼び出し先**:
  • pg_tzenumerate_start() (コンストラクタ)
  • pg_tzenumerate_next() (イテレータ)
  • pg_tzenumerate_end() (デストラクタ)

関連データ構造

  • pg_tz: タイムゾーン定義を保持する構造体。
  • DIR: ディレクトリ操作のためのシステム定義構造体。

失敗モードと境界条件

  • **メモリ不足**: palloc0 が失敗した場合、プロセスは終了します。
  • **ディレクトリ深さ**: MAX_TZDIR_DEPTH を超える階層を持つディレクトリ構造は走査できません。
  • **権限**: 読み取り権限のないディレクトリに遭遇した場合、走査がスキップされる可能性があります。

保守担当者向けメモ

  • pg_tzenumerate_next() は、この構造体内部の tz フィールドへのポインタを返すため、イテレータが終了または破棄された後のポインタ参照は無効となります。
  • palloc を使用しているため、呼び出し側のメモリコンテキストに依存します。
