remove_gene

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_erx.c:239

目的

エッジ組み換え交叉（ERX）アルゴリズムにおいて、ツアーに組み込まれた遺伝子（都市）をエッジテーブルから削除し、残りの遺伝子の接続関係を更新する。

シグネチャまたは形状

static void remove_gene(PlannerInfo *root, Gene gene, Edge edge, Edge *edge_table)

入力と出力

  • root: クエリのプランナ情報（本関数内では直接使用されない）。
  • gene: ツアーに新しく追加され、削除対象となる遺伝子。
  • edge: 削除対象遺伝子に関連するエッジ情報。
  • edge_table: 更新対象となるエッジテーブル全体。

前提条件と不変条件

  • edge_table は gimme_edge_table によって適切に初期化されていること。
  • edge.edge_list に含まれる各都市が edge_table 内に存在すること。

アルゴリズムまたは振る舞い

1. edge.edge_list を走査し、削除対象の gene と接続を持つすべての都市（possess_edge）を特定する。 2. 各 possess_edge のエッジリストから gene を検索する。 3. gene が見つかった場合、以下の処理を行う：

  • その都市の unused_edges カウントをデクリメントする。
  • エッジリスト内の該当要素を、リストの末尾にある要素で上書きし、リストを詰め直す（削除操作）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_erx.c 内の gimme_tour 関数から呼び出される。
  • **呼び出し先**: 本関数は標準ライブラリの abs() を除き、他の関数を呼び出さない。

関連データ構造

  • Edge: 都市間の接続関係を保持する構造体。unused_edges（未使用エッジ数）と edge_list（接続先リスト）を持つ。
  • Gene: 遺伝子（都市）を表す型。

失敗モードと境界条件

  • edge.unused_edges が 0 の場合、ループは実行されず何も行われない。
  • edge_list 内に gene が存在しない場合、リストの更新は行われない。

保守担当者向けメモ

  • 本関数はエッジテーブルの整合性を維持するための低レベルな更新処理である。
  • edge_list の要素を末尾の要素で置き換えることで削除を実現しているため、リスト内の順序は保持されない。順序が重要でない場合にのみ有効な実装である。
