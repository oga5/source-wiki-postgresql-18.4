GetSessionDsmHandle

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/common/session.c:69

目的

セッションスコープの動的共有メモリ（DSM）セグメントを生成または取得し、そのハンドルを返します。このセグメントは、並列クエリ実行時などにリーダープロセスとワーカープロセス間でセッション状態を共有するために使用されます。

シグネチャまたは形状

dsm_handle GetSessionDsmHandle(void)

入力と出力

  • **入力**: なし
  • **出力**: dsm_handle 型のセグメントハンドル。生成に失敗した場合は DSM_HANDLE_INVALID を返します。

前提条件と不変条件

  • CurrentSession グローバル変数が初期化されている必要があります。
  • 既にセッションDSMが作成されている場合、そのハンドルを再利用します。
  • 新規作成時は TopMemoryContext に切り替えてメモリ確保を行います。
  • 成功時は dsm_pin_mapping および dsa_pin_mapping により、バックエンドの生存期間中メモリマッピングが保持されます。

アルゴリズムまたは振る舞い

1. **既存確認**: CurrentSession->segment が存在すれば、そのハンドルを即座に返します。 2. **見積もり**: shm_toc_estimator を使用し、DSA領域とレコード型修飾子レジストリに必要なサイズを見積もります。 3. **セグメント作成**: dsm_create で共有メモリセグメントを確保し、shm_toc を初期化します。 4. **領域確保**: shm_toc_allocate を用いてDSA領域とレジストリ領域を確保し、それぞれ SESSION_KEY_DSA および SESSION_KEY_RECORD_TYPMOD_REGISTRY キーで登録します。 5. **初期化**: dsa_create_in_place および SharedRecordTypmodRegistryInit を実行します。 6. **永続化**: dsm_pin_mapping と dsa_pin_mapping を呼び出し、CurrentSession にセグメントと領域を格納してハンドルを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c (並列クエリ実行時)
  • **呼び出し先**:
  • dsm_create, dsm_segment_handle, dsm_pin_mapping
  • shm_toc_initialize_estimator, shm_toc_create, shm_toc_allocate, shm_toc_insert
  • dsa_create_in_place, dsa_pin_mapping
  • SharedRecordTypmodRegistryEstimate, SharedRecordTypmodRegistryInit
  • MemoryContextSwitchTo

関連データ構造

  • CurrentSession: セッション状態を保持するグローバル構造体。
  • shm_toc: 共有メモリ内のデータ配置を管理するテーブル・オブ・コンテンツ。
  • dsa_area: 動的共有メモリ領域。
  • SharedRecordTypmodRegistry: 共有される型修飾子レジストリ。

失敗モードと境界条件

  • dsm_create が NULL を返した場合（メモリ不足やセグメント上限到達時）、DSM_HANDLE_INVALID を返します。この際、メモリコンテキストは呼び出し前の状態に復元されます。

保守担当者向けメモ

  • 本関数で確保されたリソースは、バックエンドの生存期間中保持される設計です。
  • SESSION_MAGIC や SESSION_KEY_* 定数の変更は、リーダーとワーカー間の通信整合性に影響するため注意してください。
  • SharedRecordTypmodRegistry の初期化失敗時のクリーンアップは、CurrentResourceOwner によるDSMデタッチ時に自動的に行われる仕組みになっています。
