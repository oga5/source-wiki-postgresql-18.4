PLy_cursor_init_type

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_cursorobject.c:68

目的

PL/Python モジュール内でのカーソル操作を可能にするため、PLyCursor 型オブジェクトを初期化し、Python インタプリタに登録するための準備を行う。

シグネチャまたは形状

void PLy_cursor_init_type(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 PLy_CursorType を初期化する）

前提条件と不変条件

  • Python インタプリタが初期化されている環境で呼び出される必要がある。
  • PLyCursor_spec が定義されており、有効な Python 型定義情報を含んでいること。

アルゴリズムまたは振る舞い

1. PyType_FromSpec を使用して PLyCursor_spec から Python 型オブジェクトを生成する。 2. 生成された型オブジェクトをグローバル変数 PLy_CursorType に格納する。 3. 型オブジェクトの生成に失敗した場合、elog(ERROR, ...) を呼び出し、PostgreSQL のエラーメカニズムを通じて処理を中断する。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpython/plpy_plpymodule.c:152（PL/Python モジュールの初期化プロセスの一部として呼び出される）
  • 呼び出し先: PyType_FromSpec（Python C API）

関連データ構造

  • PLyCursor_spec: PLyCursor 型の構造を定義する PyType_Spec 構造体。
  • PLy_CursorType: 初期化された型オブジェクトを保持するグローバル変数。

失敗モードと境界条件

  • PyType_FromSpec が NULL を返した場合、elog によりエラーが発生し、トランザクションがアボートされる。

保守担当者向けメモ

  • 本関数はモジュールロード時に一度だけ実行されることを想定している。
  • PLyCursor_spec の定義を変更した場合は、本関数による初期化が正しく行われるか確認が必要である。
