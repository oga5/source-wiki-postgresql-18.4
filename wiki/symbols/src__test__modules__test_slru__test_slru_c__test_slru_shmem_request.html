test_slru_shmem_request

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_slru/test_slru.c:197

目的

PostgreSQLの共有メモリ初期化プロセスにおいて、テスト用SLRU（Simple Least Recently Used）キャッシュに必要な共有メモリ領域を確保する。

シグネチャまたは形状

static void test_slru_shmem_request(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリの予約を行う）

前提条件と不変条件

  • 本関数は shmem_request_hook として登録されることを前提としている。
  • NUM_TEST_BUFFERS（16）分のSLRUバッファ領域が確保可能である必要がある。

アルゴリズムまたは振る舞い

1. 既存の prev_shmem_request_hook が存在する場合、それを呼び出してフックの連鎖を維持する。 2. SimpleLruShmemSize(NUM_TEST_BUFFERS, 0) を使用して、テスト用SLRUに必要な共有メモリサイズを計算する。 3. RequestAddinShmemSpace を呼び出し、計算したサイズをPostgreSQLの共有メモリ要求として登録する。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLの共有メモリ要求プロセス（shmem_request_hook 経由）。
  • 呼び出し先:
  • prev_shmem_request_hook（存在する場合）
  • SimpleLruShmemSize
  • RequestAddinShmemSpace

関連データ構造

  • NUM_TEST_BUFFERS: SLRUバッファ数（16）。
  • prev_shmem_request_hook: 以前に登録されていたフック関数を保持する変数。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、RequestAddinShmemSpace がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は _PG_init 内で shmem_request_hook に代入されることで有効化される。
  • フックの連鎖を正しく行うため、prev_shmem_request_hook の呼び出しを省略してはならない。
