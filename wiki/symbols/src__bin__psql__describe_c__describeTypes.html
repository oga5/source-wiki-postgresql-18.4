describeTypes

定義

  • 種別: function
  • 定義箇所: src/bin/psql/describe.c:638

目的

データベース内のデータ型情報を取得し、整形して表示する。psql の \dT メタコマンドの実装として機能する。

シグネチャまたは形状

bool describeTypes(const char *pattern, bool verbose, bool showSystem)

入力と出力

  • **入力**:
  • pattern: 検索対象の型名パターン（NULL の場合は全件）。
  • verbose: 詳細表示フラグ（true の場合、内部名、サイズ、列挙型要素、所有者、ACL を追加）。
  • showSystem: システムカタログの型を含めるかどうかのフラグ。
  • **出力**:
  • 成功時に true、失敗時に false を返す。結果は pset.queryFout に出力される。

前提条件と不変条件

  • pset（グローバル設定）が初期化されていること。
  • データベースへの接続が確立されていること。

アルゴリズムまたは振る舞い

1. pg_catalog.pg_type を基点に SQL クエリを構築する。 2. verbose モードに応じて、内部名、サイズ、列挙型要素、所有者、ACL 情報をクエリに追加する。 3. 複合型（typrelid != 0）のうち、スタンドアロンの複合型以外を除外するフィルタを適用する。 4. pattern に [] が含まれない限り、配列型を除外する。 5. showSystem が false かつ pattern が指定されていない場合、pg_catalog および information_schema スキーマを除外する。 6. validateSQLNamePattern を使用してパターンを検証・適用する。 7. PSQLexec でクエリを実行し、printQuery で結果を整形出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c:1185（\dT コマンド実行時）
  • **呼び出し先**: initPQExpBuffer, printfPQExpBuffer, appendPQExpBuffer, printACLColumn, validateSQLNamePattern, map_typename_pattern, PSQLexec, termPQExpBuffer, printQuery, PQclear

関連データ構造

  • PQExpBufferData: SQL クエリ構築用バッファ。
  • PGresult: クエリ実行結果の保持。
  • printQueryOpt: 出力オプション。

失敗モードと境界条件

  • PSQLexec が失敗した場合（クエリ実行エラー等）、false を返して終了する。
  • validateSQLNamePattern がパターンを解決できない場合、false を返して終了する。
  • メモリ不足等でバッファ操作が失敗した場合の挙動は libpq の仕様に依存する。

保守担当者向けメモ

  • サーバーバージョンによるクエリの差異は、主に pg_catalog の関数や列の利用を通じて吸収されている。
  • 新しい型属性を追加する場合は、verbose 分岐内のクエリ構築ロジックを更新する必要がある。
