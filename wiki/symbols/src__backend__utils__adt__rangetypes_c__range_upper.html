range_upper

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes.c:468

目的

指定された範囲型（RangeType）の上限値を取得します。

シグネチャまたは形状

Datum range_upper(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して RangeType オブジェクトを受け取ります。
  • **出力**: 範囲の上限値を表す Datum を返します。上限が存在しない（空の範囲、または上限が無限大）場合は NULL を返します。

前提条件と不変条件

  • 入力された RangeType は、操作前にデトースト（detoast）されている必要があります。
  • 内部的な境界値の比較や取得には、range_get_typcache を通じて取得した型キャッシュ情報が必要です。

アルゴリズムまたは振る舞い

1. range_get_typcache を呼び出し、対象範囲型のメタデータを取得します。 2. range_deserialize を使用して、RangeType を RangeBound 構造体（下限・上限・空フラグ）に展開します。 3. 範囲が空であるか、または上限が無限大（upper.infinite）である場合は、PG_RETURN_NULL() を実行します。 4. 上記以外の場合、upper.val に格納されている上限値を PG_RETURN_DATUM() で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェースから直接呼び出される関数です。
  • **呼び出し先**:
  • range_get_typcache
  • range_deserialize
  • PG_GETARG_RANGE_P (マクロ)
  • PG_RETURN_NULL / PG_RETURN_DATUM (マクロ)

関連データ構造

  • RangeType: 範囲型の内部表現（varlena）。
  • RangeBound: 範囲の境界情報を保持する構造体。
  • TypeCacheEntry: 範囲型の要素型に関するキャッシュ情報。

失敗モードと境界条件

  • **空の範囲**: empty が真の場合、上限値は定義されないため NULL を返します。
  • **無限大**: 上限が無限大（upper.infinite）の場合、有限の上限値が存在しないため NULL を返します。

保守担当者向けメモ

  • 本関数は rangetypes.c 内の標準的な範囲型アクセサです。
  • 境界値の抽出には range_deserialize を介した RangeBound 構造体の利用が必須です。直接メモリレイアウトを解析しないでください。
