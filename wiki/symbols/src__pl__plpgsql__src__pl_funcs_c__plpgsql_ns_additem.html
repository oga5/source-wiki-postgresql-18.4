plpgsql_ns_additem

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_funcs.c:91

目的

現在の名前空間チェーン（ns_top）に新しい項目（変数、レコード、ラベル等）を追加します。

シグネチャまたは形状

void plpgsql_ns_additem(PLpgSQL_nsitem_type itemtype, int itemno, const char *name)

入力と出力

  • itemtype (入力): 追加する項目の型（PLpgSQL_nsitem_type）。
  • itemno (入力): 項目に関連付けられた識別番号。
  • name (入力): 項目の名称。
  • 出力: なし（副作用としてグローバルな名前空間スタックを更新）。

前提条件と不変条件

  • name は NULL であってはならない。
  • 名前空間チェーンが空の場合、最初に追加される項目は PLPGSQL_NSTYPE_LABEL 型でなければならない。

アルゴリズムまたは振る舞い

1. name の長さに基づき、PLpgSQL_nsitem 構造体に必要なメモリを palloc で確保する。 2. 確保した構造体に itemtype、itemno、および name をコピーする。 3. 新しい項目の prev ポインタに現在の ns_top を設定し、スタックを連結する。 4. ns_top を新しく作成した項目に更新し、スタックの先頭を差し替える。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/pl/plpgsql/src/pl_comp.c (複数箇所)
  • src/pl/plpgsql/src/pl_funcs.c
  • 呼び出し先:
  • palloc (メモリ確保)
  • strcpy (文字列コピー)

関連データ構造

  • PLpgSQL_nsitem: 名前空間チェーンを構成するリンクリスト要素。
  • ns_top: 名前空間スタックの先頭を保持するグローバル変数。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • name が NULL の場合、Assert によりデバッグビルドで停止する。
  • 最初の項目がラベルでない場合、Assert によりデバッグビルドで停止する。

保守担当者向けメモ

  • 本関数は名前空間スタックの先頭を破壊的に更新するため、呼び出し順序が重要です。
  • PLpgSQL_nsitem は可変長構造体（末尾に name 配列を持つ）として設計されているため、palloc 時のサイズ計算には注意が必要です。
