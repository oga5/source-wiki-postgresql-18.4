datumTransfer

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/datum.c:193

目的

Datum を現在のメモリコンテキスト（CurrentMemoryContext）へ転送します。拡張オブジェクト（Expanded Object）に対しては再配置（reparent）による最適化を行い、それ以外の場合はコピーを作成します。

シグネチャまたは形状

Datum datumTransfer(Datum value, bool typByVal, int typLen)

入力と出力

  • **入力**:
  • value: 転送対象の Datum。
  • typByVal: 値渡し（by-value）かどうかを示すフラグ。
  • typLen: 型の長さ（バイト数）。可変長型の場合は -1。
  • **出力**:
  • 転送後の Datum 値。

前提条件と不変条件

  • value が NULL でないことを前提としています。
  • 呼び出し元は、対象の型に対して正しい typByVal および typLen を提供する必要があります。誤った値はメモリ破壊や不正なポインタ参照を引き起こします。

アルゴリズムまたは振る舞い

1. typByVal が偽かつ typLen が -1（可変長）であり、かつ value が読み書き可能な拡張オブジェクト（VARATT_IS_EXTERNAL_EXPANDED_RW）である場合：

  • TransferExpandedObject を呼び出し、現在のメモリコンテキストへ再配置します。

    • 2. それ以外の場合：

  • datumCopy を呼び出し、メモリコンテキスト内に独立したコピーを作成します。

    • 3. 処理後の Datum を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/spi.c
  • src/pl/plpgsql/src/pl_exec.c
  • **呼び出し先**:
  • TransferExpandedObject
  • datumCopy

関連データ構造

  • Datum: PostgreSQL の汎用データ型表現。
  • ExpandedObject: メモリ上で操作可能な拡張データ構造。

失敗モードと境界条件

  • typByVal や typLen が実際の型定義と不一致である場合、未定義の動作が発生します。
  • datumCopy 内部でのメモリ不足時には palloc によるエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は、メモリコンテキストの寿命を意識せずに Datum を安全に保持したい場合に利用されます。
  • 拡張オブジェクトの再配置は、コピーを避けるための重要な最適化です。修正を加える際は、TransferExpandedObject のメモリコンテキスト管理との整合性に注意してください。
