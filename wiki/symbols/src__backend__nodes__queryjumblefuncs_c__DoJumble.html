DoJumble

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/queryjumblefuncs.c:208

目的

JumbleState に蓄積されたクエリのシリアライズデータから、最終的な 64 ビットの queryId を計算する。

シグネチャまたは形状

int64 DoJumble(JumbleState *jstate, Node *node)

入力と出力

  • **入力**:
  • jstate: ハッシュ計算用のバッファや状態を保持する JumbleState 構造体へのポインタ。
  • node: フィンガープリント対象となるクエリツリーのルートノード。
  • **出力**:
  • 計算された 64 ビットのハッシュ値（queryId）。

前提条件と不変条件

  • jstate は初期化済みであること。
  • _jumbleNode によるノードの走査が完了した後に呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. _jumbleNode(jstate, node) を呼び出し、ノードツリーを再帰的に走査してバッファへシリアライズする。 2. バッファ内に保留中の NULL 値がある場合、FlushPendingNulls(jstate) を呼び出してフラッシュする。 3. リストが「Squashed（圧縮）」されている場合、jstate->highest_extern_param_id を 0 にリセットする。 4. jstate->jumble バッファの内容を hash_any_extended に渡し、最終的なハッシュ値を生成して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/queryjumblefuncs.c:145 (JumbleQuery 関数内)
  • **呼び出し先**:
  • _jumbleNode
  • FlushPendingNulls
  • hash_any_extended

関連データ構造

  • JumbleState: ハッシュ計算のバッファおよび状態管理用構造体。
  • Node: フィンガープリント対象のクエリツリーノード。

失敗モードと境界条件

  • バッファのオーバーフローについては AppendJumbleInternal が管理しており、本関数は最終的なハッシュ計算を行う。
  • pending_nulls が 0 より大きい場合は、ハッシュ計算前に必ずフラッシュ処理が行われる。

保守担当者向けメモ

  • 本関数はクエリのフィンガープリント生成プロセスの最終段階を担う。
  • 新しいノード型を追加した際は、_jumbleNode 経由で正しくシリアライズされることを確認すること。
