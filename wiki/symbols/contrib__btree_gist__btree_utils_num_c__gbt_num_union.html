gbt_num_union

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_utils_num.c:171

目的

GiST インデックスにおいて、複数の数値範囲エントリを包含する最小の範囲（Union）を計算します。

シグネチャまたは形状

void *gbt_num_union(GBT_NUMKEY *out, const GistEntryVector *entryvec, const gbtree_ninfo *tinfo, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • out: 結果を格納するメモリ領域。
  • entryvec: 結合対象となる複数の GiST エントリを含むベクトル。
  • tinfo: データ型固有のサイズおよび比較関数（f_gt, f_lt）を保持する構造体。
  • flinfo: 関数呼び出し情報。
  • **出力**:
  • 結合後の範囲を格納した out へのポインタを返します。

前提条件と不変条件

  • entryvec には少なくとも1つ以上のエントリが存在する必要があります。
  • tinfo で指定される比較関数は、対象データ型に対して適切に定義されている必要があります。
  • out は、結合された下限値と上限値を格納するために十分なメモリ領域（2 * tinfo->size）を確保している必要があります。

アルゴリズムまたは振る舞い

1. entryvec の最初の要素を初期の範囲として out にコピーします。 2. 残りの各エントリを順次走査します。 3. 各エントリの下限値が現在の out の下限値より小さい場合、その値を新しい下限値として更新します。 4. 各エントリの上限値が現在の out の上限値より大きい場合、その値を新しい上限値として更新します。 5. 比較には tinfo 内の f_gt および f_lt 関数を使用します。

呼び出し元と呼び出し先

  • **呼び出し元**: btree_bool.c, btree_cash.c, btree_date.c, btree_enum.c, btree_float4.c, btree_float8.c, btree_inet.c, btree_int2.c, btree_int4.c, btree_int8.c, btree_interval.c, btree_macaddr.c 等の各データ型実装ファイル。
  • **呼び出し先**: tinfo 構造体に含まれる f_gt および f_lt 関数。

関連データ構造

  • GBT_NUMKEY: 数値範囲を表現する構造体。
  • gbtree_ninfo: データ型固有の比較ロジックを抽象化するための設定構造体。
  • GistEntryVector: GiST インデックスのノードに含まれるエントリの集合。

失敗モードと境界条件

  • entryvec が空の場合、entryvec->vector[0] へのアクセスにより不正なメモリ参照が発生する可能性があります。
  • tinfo が提供する比較関数が NULL である場合、実行時にクラッシュします。

保守担当者向けメモ

  • 本関数は数値型全般で共通利用されるため、変更時は全データ型への影響を考慮してください。
  • unconstify マクロを使用して const 修飾を解除し、メモリコピーを行っています。これは GBT_NUMKEY の構造が固定長であることを前提としています。
