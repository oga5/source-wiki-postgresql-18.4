enum_out

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/enum.c:154

目的

列挙型（ENUM）の内部表現であるOIDを、対応する人間が読み取り可能なラベル（文字列）に変換します。

シグネチャまたは形状

Datum enum_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介して列挙型のOIDを受け取ります。
  • **出力**: PG_RETURN_CSTRING を介して、対応する列挙ラベルの文字列（char *）を返します。

前提条件と不変条件

  • 入力されたOIDがシステムカタログ pg_enum に存在する必要があります。
  • 処理終了時に ReleaseSysCache を呼び出し、取得したシステムキャッシュタプルを解放する必要があります。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用し、ENUMOID キャッシュから指定されたOIDに対応する pg_enum のタプルを検索します。 2. タプルが見つからない場合、ERRCODE_INVALID_BINARY_REPRESENTATION エラーを発生させます。 3. GETSTRUCT を用いてタプルから Form_pg_enum 構造体を取り出し、enumlabel フィールドを pstrdup でコピーします。 4. ReleaseSysCache でタプルを解放します。 5. コピーした文字列を戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pseudotypes.c:199（enum_out を直接呼び出し）
  • その他、列挙型の出力処理が必要な箇所から間接的に参照されます。
  • **呼び出し先**:
  • SearchSysCache1
  • ReleaseSysCache
  • pstrdup
  • ereport

関連データ構造

  • pg_enum: 列挙型の定義を保持するシステムカタログ。
  • Form_pg_enum: pg_enum タプルの構造体。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • 指定されたOIDが pg_enum に存在しない場合、エラーが発生します。
  • メモリ不足により pstrdup が失敗した場合、PostgreSQLのメモリ管理機構によりエラーがスローされます。

保守担当者向けメモ

  • 本関数は列挙型のテキスト出力用です。バイナリ形式の出力には enum_send が使用されます。
  • pg_enum のキャッシュを利用しているため、カタログの更新が即座に反映される設計となっています。
