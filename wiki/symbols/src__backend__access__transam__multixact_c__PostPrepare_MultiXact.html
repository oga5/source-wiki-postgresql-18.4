PostPrepare_MultiXact

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/multixact.c:1999

目的

トランザクションのプリペア（準備）完了時に、当該トランザクションが保持していた MultiXact 関連のメタデータを、実行中のバックエンドからプリペア済みトランザクション用のスロットへ移行し、ローカルキャッシュを破棄する。

シグネチャまたは形状

void PostPrepare_MultiXact(TransactionId xid)

入力と出力

  • **入力**: xid (プリペアされるトランザクションの ID)
  • **出力**: なし（副作用として共有メモリ上のスロット更新およびローカルメモリの解放を行う）

前提条件と不変条件

  • 呼び出し元は、トランザクションのプリペア処理中であること。
  • MyOldestMemberMXactIdSlot() が指すスロットに有効な値がある場合、MultiXactGenLock を排他モードで取得して更新を行う。
  • MyOldestVisibleMXactIdSlot() の更新はアトミックであると仮定されるため、ロックを必要としない。

アルゴリズムまたは振る舞い

1. 現在のバックエンドの OldestMemberMXactId を取得する。 2. 値が有効な場合:

  • TwoPhaseGetDummyProcNumber を用いて、プリペア済みトランザクション用のダミープロセス番号を取得する。
  • MultiXactGenLock を排他取得し、値をプリペア済みトランザクション用のスロットへコピーする。
  • 現在のバックエンドのスロットを InvalidMultiXactId でリセットする。

    • 3. MyOldestVisibleMXactIdSlot を InvalidMultiXactId にリセットする。 4. ローカルの MultiXact キャッシュ（MXactCache）を初期化し、MXactContext を NULL に設定してメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c:2741 (プリペア処理の最終段階)
  • **呼び出し先**:
  • MyOldestMemberMXactIdSlot()
  • TwoPhaseGetDummyProcNumber()
  • LWLockAcquire() / LWLockRelease()
  • PreparedXactOldestMemberMXactIdSlot()
  • MyOldestVisibleMXactIdSlot()
  • dclist_init()

関連データ構造

  • MultiXactGenLock: 共有メモリ上の MultiXact 状態を保護するロック。
  • MXactCache: ローカルの MultiXact キャッシュリスト。
  • MXactContext: キャッシュ用メモリコンテキスト。

失敗モードと境界条件

  • myOldestMember が InvalidMultiXactId の場合、スロットの移行処理はスキップされる。
  • プリペア済みトランザクション用のスロット取得に失敗した場合の挙動については、本ソース内では明示されていない。

保守担当者向けメモ

  • 本関数は AtEOXact_MultiXact と同様のキャッシュ破棄ロジックを共有している。
  • MultiXactGenLock の取得は、プリペア済みトランザクションへの移行時に他のバックエンドから一貫した状態が見えることを保証するために行われる。
