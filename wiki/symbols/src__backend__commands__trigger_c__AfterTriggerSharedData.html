AfterTriggerSharedData

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/trigger.c:3694

目的

AFTER トリガーの実行に必要な共有状態を保持するためのデータ構造です。トランザクション内で発生したトリガーイベントに関連付けられ、遅延実行されるトリガーのメタデータや実行コンテキストを管理します。

シグネチャまたは形状

typedef struct AfterTriggerSharedData
{
	TriggerEvent ats_event;             /* イベント種別 */
	Oid          ats_tgoid;             /* トリガーID */
	Oid          ats_relid;             /* 対象リレーションID */
	Oid          ats_rolid;             /* 実行ロールID */
	CommandId    ats_firing_id;         /* 発火サイクルID */
	struct AfterTriggersTableData *ats_table; /* 遷移テーブルへのアクセス */
	Bitmapset   *ats_modifiedcols;      /* 変更された列の集合 */
} AfterTriggerSharedData;

入力と出力

  • 本構造体は、トリガーイベント発生時にメモリ上に割り当てられ、AFTER トリガーのキューイングに使用されます。
  • ats_table を介して遷移テーブル（Transition Tables）へのアクセスを提供します。

前提条件と不変条件

  • ats_tgoid および ats_relid は有効なシステムカタログ上の識別子である必要があります。
  • トランザクションのライフサイクルを通じて、遅延実行されるトリガーの整合性を維持するために使用されます。

アルゴリズムまたは振る舞い

  • AFTER トリガーのイベント記録（AfterTriggerEvent）と対になって管理されます。
  • トランザクション終了時や SET CONSTRAINTS 実行時に、この構造体に含まれる情報に基づいてトリガー関数が呼び出されます。
  • 実行サイクルごとに ats_firing_id を用いて、トリガーの実行順序や重複排除を制御します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/trigger.c 内のトリガーイベント生成処理（AfterTriggerSaveEvent 等）から参照されます。
  • 呼び出し先: 具体的な関数呼び出し関係はソース抜粋からは特定できません。

関連データ構造

  • AfterTriggerEvent: この構造体と対になるイベント記録。
  • AfterTriggersTableData: 遷移テーブルの管理用構造体。
  • TriggerEvent: トリガーのイベント種別を定義する型。

失敗モードと境界条件

  • メモリ不足: 大量にトリガーイベントが発生した場合、本構造体を格納するメモリ領域の確保に失敗する可能性があります。
  • 識別子の無効化: トリガーやリレーションが削除された後に本構造体が参照されると、実行時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は可変状態を保持するため、トランザクションの整合性に直結します。変更を加える際は、AfterTriggerFireDeferred 等の実行エンジン側のロジックへの影響を十分に考慮してください。
  • ats_table はポインタであるため、メモリ管理（ライフサイクル）に注意が必要です。
