check_return

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/compat_informix-rfmtlong.c:62

目的

ECPG Informix 互換ライブラリの関数実行時に発生したエラーコードを、人間が判読可能な文字列として標準出力へ出力するヘルパー関数です。

シグネチャまたは形状

static void check_return(int ret)

入力と出力

  • **入力**: int ret - ECPG の Informix 互換エラーコード。
  • **出力**: なし（標準出力へエラー内容を書き出す）。

前提条件と不変条件

  • ret には ecpgerrno.h 等で定義された Informix 関連のエラーコードが渡されることを想定しています。
  • 呼び出し後、必ず改行コードが出力されます。

アルゴリズムまたは振る舞い

1. switch 文により ret の値を評価します。 2. 既知のエラーコード（ECPG_INFORMIX_ENOTDMY, ECPG_INFORMIX_ENOSHORTDATE, ECPG_INFORMIX_BAD_DAY, ECPG_INFORMIX_BAD_MONTH）に合致する場合、対応する文字列を printf で出力します。 3. いずれにも該当しない場合は (unknown ret: %d) として数値を表示します。 4. 最後に改行を出力して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/test/expected/compat_informix-rfmtdate.c (複数箇所)
  • src/interfaces/ecpg/test/expected/compat_informix-rfmtlong.c (複数箇所)
  • **呼び出し先**: 標準ライブラリの printf を呼び出します。

関連データ構造

  • ecpgerrno.h で定義されるエラー定数群。

失敗モードと境界条件

  • 未定義の ret 値が渡された場合、default 節により数値がそのまま出力されます。

保守担当者向けメモ

  • 本関数はテストの期待値生成用コードの一部です。エラーコードの定義が変更された場合は、本関数の switch 文も更新する必要があります。
