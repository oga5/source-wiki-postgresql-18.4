bms_is_subset

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/bitmapset.c:411

目的

集合 a が集合 b の部分集合であるか（a ⊆ b）を判定します。

シグネチャまたは形状

bool bms_is_subset(const Bitmapset *a, const Bitmapset *b)

入力と出力

  • **入力**:
  • a: 比較対象の Bitmapset ポインタ（NULL 許容）。
  • b: 比較対象の Bitmapset ポインタ（NULL 許容）。
  • **出力**:
  • true: a が b の部分集合である場合。
  • false: そうでない場合。

前提条件と不変条件

  • a および b は bms_is_valid_set によって検証可能である必要があります。
  • 空集合（NULL）は、任意の集合の部分集合とみなされます。

アルゴリズムまたは振る舞い

1. a が NULL の場合、true を返します。 2. b が NULL の場合、false を返します。 3. a のワード数（nwords）が b より大きい場合、a は b の部分集合になり得ないため false を返します。 4. a の各ワードを順次走査し、a->words[i] & ~b->words[i] が 0 以外であれば、b に含まれないビットが a に存在すると判断し false を返します。 5. 全てのワードで条件を満たせば true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/catalog/pg_constraint.c
  • src/backend/optimizer/path/allpaths.c
  • src/backend/optimizer/path/costsize.c
  • **呼び出し先**:
  • bms_is_valid_set（アサーション内）

関連データ構造

  • Bitmapset: 非負整数の集合を保持する構造体。nwords と words 配列で構成されます。

失敗モードと境界条件

  • a が NULL の場合は常に true を返します。
  • b が NULL で a が非空の場合、false を返します。
  • a のワード数が b より大きい場合、ビット比較を行わずに早期終了します。

保守担当者向けメモ

  • 本関数は、bms_subset_compare よりも特定の包含関係のみを判定する場合に効率的です。
  • Bitmapset の NULL 規約に基づき、入力ポインタの NULL チェックが必須です。
