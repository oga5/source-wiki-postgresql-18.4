compute_function_hashkey

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/funccache.c:246

目的

関数実行データのキャッシュ検索に使用する CachedFunctionHashKey を生成します。関数OID、引数型、照合順序、トリガー情報などを集約し、実行コンテキストに応じた一意なキーを作成します。

シグネチャまたは形状

void compute_function_hashkey(FunctionCallInfo fcinfo,
                              Form_pg_proc procStruct,
                              CachedFunctionHashKey *hashkey,
                              Size cacheEntrySize,
                              bool includeResultType,
                              bool forValidator)

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報。
  • procStruct: pg_proc の関数定義データ。
  • cacheEntrySize: キャッシュエントリのサイズ。
  • includeResultType: 戻り値の型をキーに含めるかどうかのフラグ。
  • forValidator: バリデーションモードかどうかを示すフラグ。
  • **出力**:
  • hashkey: 生成されたキーが格納される構造体。

前提条件と不変条件

  • hashkey 構造体の固定部分のパディングバイトはゼロクリアされる必要があります。
  • 多相型（polymorphic types）の引数は、呼び出し時に確定した実型に解決される必要があります。

アルゴリズムまたは振る舞い

1. hashkey をゼロクリアし、関数OID、トリガーフラグ、イベントトリガーフラグ、キャッシュエントリサイズをセットします。 2. DMLトリガーの場合、バリデーションモードでなければ trigOid を設定します。 3. 入力照合順序（inputCollation）をセットします。 4. procStruct に基づき引数型をコピーし、cfunc_resolve_polymorphic_argtypes を呼び出して多相型を解決します。 5. includeResultType が真の場合、get_call_result_type を実行し、複合型（TYPEFUNC_COMPOSITE 等）であれば callResultType に TupleDesc を格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/funccache.c 内の cached_function_compile（行512および行568）。
  • **呼び出し先**: cfunc_resolve_polymorphic_argtypes、get_call_result_type。

関連データ構造

  • CachedFunctionHashKey: キャッシュ検索キーの構造体。
  • FunctionCallInfo: 関数呼び出しの実行コンテキスト。
  • Form_pg_proc: pg_proc システムカタログの行定義。
  • TriggerData: トリガー実行時のデータ構造。

失敗モードと境界条件

  • forValidator が真の場合、トリガーOIDはゼロのままとなり、実際の呼び出しには使用されないキーが生成されます。
  • includeResultType が偽の場合、戻り値の型によるキャッシュの区別は行われません。
  • 戻り値がスカラー型または不確定な行型の場合、callResultType は更新されません。

保守担当者向けメモ

  • 本関数は、異なる言語間でのキャッシュ共有を可能にするため、cacheEntrySize をキーに含めています。
  • 戻り値の型（RECORD 等）が動的に変化する関数を扱う際は、includeResultType を適切に制御する必要があります。
