var_list

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/ecpglib/ecpglib_extern.h:152

目的

ECPGライブラリ内において、ホスト変数や関連するデータへのポインタを保持するための連結リスト構造を提供します。

シグネチャまたは形状

struct var_list
{
    int             number;
    void           *pointer;
    struct var_list *next;
};

入力と出力

  • **入力**: リストの各ノードは、整数型の識別子（number）と、任意のデータ型を指すポインタ（pointer）を保持します。
  • **出力**: next メンバにより、後続の var_list ノードへの参照を提供します。

前提条件と不変条件

  • next メンバは、リストの終端において NULL である必要があります。
  • pointer は、ECPGのメモリ管理ポリシーに従って確保されたメモリ領域を指すことが想定されます。

アルゴリズムまたは振る舞い

  • 連結リストとして機能し、misc.c 内の関数によって動的にノードが生成（calloc）され、リストの構築や走査が行われます。
  • 外部変数 ivlist がこのリストの先頭を保持する役割を担います。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/interfaces/ecpg/ecpglib/connect.c:148
  • src/interfaces/ecpg/ecpglib/misc.c:535, 540, 564, 595
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • struct variable: ECPGにおけるホスト変数とSQL変数のマッピングを保持する構造体。
  • ivlist: misc.c で定義される var_list 型の外部変数（リストのヘッド）。

失敗モードと境界条件

  • calloc によるメモリ確保失敗時にリストの拡張が停止します。
  • リストの走査時に next ポインタが正しく終端（NULL）を指していない場合、メモリ破壊やセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は ecpglib 内部の管理用リストとして使用されています。メモリ管理については、標準の malloc ではなく ecpg_alloc 等のライブラリ内規約に従うことが推奨されます。
  • misc.c 内の ivlist 操作ロジックを変更する際は、リストのメモリリークが発生しないよう注意してください。
