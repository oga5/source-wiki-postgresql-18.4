AtStart_Cache

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xact.c:1166

目的

トランザクション開始時に、共有メモリ上の無効化メッセージ（Invalidation Messages）を処理し、ローカルキャッシュを最新の状態に同期させる。

シグネチャまたは形状

static void AtStart_Cache(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてローカルキャッシュの更新を行う）

前提条件と不変条件

  • 本関数はトランザクション開始処理の過程で呼び出されることを想定している。
  • AcceptInvalidationMessages() が正常に完了し、保留中の無効化メッセージが適用される必要がある。

アルゴリズムまたは振る舞い

1. AcceptInvalidationMessages() を呼び出し、共有メモリから受信した無効化メッセージを処理する。 2. これにより、現在のトランザクションが参照するカタログキャッシュやリレーションキャッシュが、他セッションによる変更を反映した状態に更新される。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/xact.c:2205 (推測される文脈: トランザクション開始シーケンス内)
  • 呼び出し先:
  • AcceptInvalidationMessages()

関連データ構造

  • utils/inval.h で定義される無効化メッセージ管理機構に関連する。

失敗モードと境界条件

  • 共有メモリへのアクセスや無効化メッセージの処理中にエラーが発生した場合、PostgreSQLの標準的なエラーハンドリング（elog / ereport）によりトランザクションがアボートされる可能性がある。

保守担当者向けメモ

  • 本関数はトランザクション開始時のキャッシュ整合性を担保する重要なステップである。キャッシュの無効化ロジックに変更を加える際は、本関数が適切にメッセージを処理しているか確認すること。
