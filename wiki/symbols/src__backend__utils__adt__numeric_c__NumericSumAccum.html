NumericSumAccum

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/numeric.c:380

目的

SUM() などの集約関数において、計算過程の中間状態を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct NumericSumAccum
{
	int			ndigits;
	int			weight;
	int			dscale;
	int			num_uncarried;
	bool		have_carry_space;
	int32	   *pos_digits;
	int32	   *neg_digits;
} NumericSumAccum;

入力と出力

本構造体は関数への引数として渡され、集約計算の累積状態を保持・更新します。直接的な入出力関数は定義されていませんが、関連する操作関数が本構造体を介して状態を読み書きします。

前提条件と不変条件

  • pos_digits および neg_digits は、集約計算に必要な桁数分が動的に割り当てられる必要があります。
  • 累積計算の過程で、数値の重み（weight）が大きく変動する可能性があります。

アルゴリズムまたは振る舞い

集約計算における加算、リスケール、桁上げ（キャリー）処理、および最終的な Numeric 値への変換をサポートします。

  • accum_sum_add: 値の加算。
  • accum_sum_rescale: スケールの調整。
  • accum_sum_carry: 桁上げ処理。
  • accum_sum_final: 最終結果の生成。
  • accum_sum_reset: 状態の初期化。
  • accum_sum_copy: 状態のコピー。
  • accum_sum_combine: 2つの累積状態の結合。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/numeric.c 内の集約処理ロジック（例: SUM() 実装）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、操作は accum_sum_* 系関数によって行われます。
  • 補足: 具体的な呼び出し元の関数名はソースコードの断片からは特定できません。

関連データ構造

  • NumericVar: 算術演算に使用されるメモリ上の作業用構造体。
  • NumericData: NUMERIC 型のオンディスク表現。

失敗モードと境界条件

  • メモリ不足: pos_digits や neg_digits の割り当て時に発生する可能性があります。
  • 桁あふれ: 非常に大きな数値や重みを持つ場合に発生する可能性があります。

保守担当者向けメモ

  • 本構造体は SUM() などの集約関数の中間状態管理に特化しています。
  • pos_digits と neg_digits は動的メモリ管理を伴うため、使用後の解放や適切なサイズ管理に注意してください。
  • 構造体の各フィールドは、NumericVar と比較してより大きな重みを扱えるように設計されています。
