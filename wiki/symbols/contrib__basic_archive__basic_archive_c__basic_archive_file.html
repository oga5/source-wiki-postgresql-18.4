basic_archive_file

定義

  • 種別: function
  • 定義箇所: contrib/basic_archive/basic_archive.c:140

目的

PostgreSQLのアーカイブモジュールとして、WALファイルを指定されたディレクトリへ安全にコピー・永続化する。

シグネチャまたは形状

static bool basic_archive_file(ArchiveModuleState *state, const char *file, const char *path)

入力と出力

  • **入力**:
  • state: アーカイブモジュールの状態（未使用）。
  • file: アーカイブ対象のファイル名。
  • path: アーカイブ対象ファイルのフルパス。
  • **出力**:
  • 成功時は true を返す。
  • 失敗時はエラーを発生させ、処理を中断する。

前提条件と不変条件

  • archive_directory GUC変数が設定されていること。
  • path で指定されたファイルが読み取り可能であること。
  • アーカイブ先ディレクトリへの書き込み権限があること。

アルゴリズムまたは振る舞い

1. **重複確認**: 宛先に同名ファイルが存在する場合、compare_files で内容を比較する。内容が一致すれば、永続化（fsync）を確認して成功として終了する。不一致の場合はエラーとする。 2. **一時ファイル作成**: プロセスIDとタイムスタンプを用いた一意な一時ファイル名（archtemp...）を生成する。 3. **コピー**: copy_file を使用して、ソースファイルを一時ファイルへコピーする。 4. **永続化と移動**: durable_rename を使用して、一時ファイルを宛先パスへアトミックに移動し、ディスクへの書き込みを保証する。

呼び出し元と呼び出し先

  • **呼び出し元**: basic_archive_callbacks 構造体を通じて、PostgreSQLのアーカイブプロセスからコールバックとして呼び出される。
  • **呼び出し先**:
  • compare_files: 内容比較用。
  • fsync_fname: ファイルおよびディレクトリの永続化用。
  • copy_file: ファイルコピー用。
  • durable_rename: ファイルの安全な移動用。
  • gettimeofday: 一時ファイル名生成用。

関連データ構造

  • ArchiveModuleState: アーカイブモジュールの状態管理構造体。
  • archive_directory: アーカイブ先ディレクトリを保持するGUC変数。

失敗モードと境界条件

  • **ファイル存在**: 宛先に同名ファイルがあり、内容が異なる場合はエラーとなる。
  • **権限・アクセス**: stat やファイル操作が失敗した場合はエラーとなる。
  • **一時ファイル生成**: タイムスタンプ計算でオーバーフローが発生した場合はエラーとなる。
  • **パス長**: MAXPGPATH を超えるパスは扱えない。

保守担当者向けメモ

  • 本関数は durable_rename を使用しており、システムクラッシュ時にもファイルが壊れないよう設計されている。
  • 重複排除ロジックは、クラッシュリカバリ時に .ready ファイルが残存しているケースを考慮している。
