InitializeSessionUserId

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/miscinit.c:760

目的

セッションユーザーの識別情報を初期化し、認証されたユーザーIDの設定、GUCパラメータの更新、およびログイン権限や接続制限の検証を行う。

シグネチャまたは形状

void InitializeSessionUserId(const char *rolename, Oid roleid, bool bypass_login_check)

入力と出力

  • **入力**:
  • rolename: ロール名（指定がある場合）。
  • roleid: ロールOID（rolename がNULLの場合に使用）。
  • bypass_login_check: ログイン権限チェックをスキップするかどうかのフラグ。
  • **出力**: なし（成功時は内部状態を更新し、失敗時は FATAL エラーでプロセスを終了する）。

前提条件と不変条件

  • IsBootstrapProcessingMode() が偽であること。
  • 並列ワーカー（InitializingParallelWorker）の場合は、何もせず即座に復帰する。
  • システムカタログ（pg_authid）がアクセス可能であること。

アルゴリズムまたは振る舞い

1. **無効化メッセージの処理**: AcceptInvalidationMessages() を呼び出し、最新のカタログ変更を反映させる。 2. **ロール情報の検索**: rolename または roleid をキーに pg_authid システムカタログを検索。存在しない場合は FATAL エラー。 3. **ユーザーIDの設定**: SetAuthenticatedUserId() を呼び出し、認証済みユーザーIDを記録。 4. **GUC設定**: session_authorization パラメータをロール名で更新。 5. **ログイン・接続制限の検証** (IsUnderPostmaster が真の場合):

  • bypass_login_check が偽かつ rolcanlogin が偽の場合、FATAL エラー。
  • 通常のバックエンドプロセスかつ非スーパーユーザーの場合、CountUserBackends() を用いて rolconnlimit を超えていないか確認し、超過時は FATAL エラー。

    • 6. **リソース解放**: ReleaseSysCache() で検索したタプルを解放。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c
  • src/backend/utils/init/postinit.c
  • **呼び出し先**:
  • AcceptInvalidationMessages
  • SearchSysCache1
  • SetAuthenticatedUserId
  • SetConfigOption
  • CountUserBackends
  • ReleaseSysCache

関連データ構造

  • pg_authid: ロール情報を保持するシステムカタログ。
  • Form_pg_authid: pg_authid の構造体定義。

失敗モードと境界条件

  • **ロール不在**: 指定されたロール名またはOIDが pg_authid に存在しない場合、FATAL エラー。
  • **ログイン禁止**: rolcanlogin が偽のロールでログインしようとした場合、FATAL エラー。
  • **接続数超過**: rolconnlimit を超える接続が発生した場合、FATAL エラー。
  • **競合**: 接続制限チェックには競合状態（Race Condition）が存在し、制限値は近似的なものとなる。

保守担当者向けメモ

  • 本関数は postmaster 配下での動作を前提としたチェック（ログイン可否、接続数制限）を含んでいる。
  • session_authorization の設定は PGC_S_OVERRIDE を使用しており、pg_db_role_setting による動的な上書きは現在の実装では意図的に無効化されている。変更にはメジャーリリースレベルの検討が必要。
