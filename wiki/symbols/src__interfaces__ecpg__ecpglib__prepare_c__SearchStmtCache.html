SearchStmtCache

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/prepare.c:420

目的

ECPGの自動プリペア機能において、指定されたSQLクエリがステートメントキャッシュ内に存在するかを検索し、そのインデックスを返します。

シグネチャまたは形状

static int SearchStmtCache(const char *ecpgQuery)

入力と出力

  • **入力**: ecpgQuery (検索対象のSQL文字列)
  • **出力**: キャッシュ内で一致したエントリのインデックス（int）。見つからない場合やキャッシュが未初期化の場合は 0 を返します。

前提条件と不変条件

  • stmtCacheEntries が初期化されていること。未初期化（NULL）の場合は即座に 0 を返します。
  • キャッシュは stmtCacheNBuckets 個のバケットを持ち、各バケットは stmtCacheEntPerBucket 個のエントリで構成される固定長構造です。

アルゴリズムまたは振る舞い

1. stmtCacheEntries が NULL であれば 0 を返す。 2. HashStmt(ecpgQuery) を呼び出し、ハッシュ値から開始インデックス entNo を算出する。 3. stmtCacheEntPerBucket の範囲内でループし、以下の条件で検索を行う：

  • stmtID[0] が非ゼロ（エントリが使用中）であること。
  • strcmp により ecpgQuery がキャッシュ内のクエリ文字列と一致すること。

    • 4. 一致した場合はそのインデックスを返し、ループ終了まで見つからない場合は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/prepare.c 内の ecpg_auto_prepare 関数から呼び出されることが確認されています。
  • **呼び出し先**: HashStmt を呼び出します。

関連データ構造

  • stmtCacheEntries: キャッシュエントリを保持するグローバルなハッシュテーブル構造体。
  • stmtCacheEntPerBucket: 1バケットあたりのエントリ数（定数）。

失敗モードと境界条件

  • **キャッシュ未初期化**: stmtCacheEntries が NULL の場合、安全に 0 を返します。
  • **キャッシュミス**: 指定されたクエリがキャッシュ内に存在しない場合、0 を返します。
  • **ハッシュ衝突**: 同一バケット内で複数のエントリがハッシュ化される場合、ループによる線形探索で解決されます。

保守担当者向けメモ

  • 本関数は stmtID[0] を使用してエントリの有効性を判定しています。キャッシュの置換アルゴリズム（LRU等）との整合性に注意してください。
  • 戻り値の 0 は「見つからない」ことを意味しますが、インデックス 0 が有効なキャッシュエントリとして使用される可能性があるか、データ構造の定義と照らし合わせて確認が必要です。
