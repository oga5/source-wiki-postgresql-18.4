ExecInitResultSlot

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execTuples.c:1967

目的

プランノードの実行結果を保持するための TupleTableSlot を初期化し、指定された PlanState に割り当てる。

シグネチャまたは形状

void ExecInitResultSlot(PlanState *planstate, const TupleTableSlotOps *tts_ops)

入力と出力

  • **入力**:
  • planstate: 対象となるプランノードの実行状態構造体。
  • tts_ops: スロットの振る舞いを定義する関数テーブル（TupleTableSlotOps）。
  • **出力**: なし（planstate 構造体のフィールドを直接更新する）。

前提条件と不変条件

  • planstate->ps_ResultTupleDesc が有効な記述子であるか、あるいは NULL であること。
  • planstate->state->es_tupleTable が初期化済みであること。

アルゴリズムまたは振る舞い

1. ExecAllocTableSlot を呼び出し、planstate->state->es_tupleTable に新しいスロットを割り当てる。その際、ps_ResultTupleDesc と tts_ops を使用する。 2. 割り当てられたスロットを planstate->ps_ResultTupleSlot に格納する。 3. planstate->resultopsfixed を、記述子の有無（ps_ResultTupleDesc != NULL）に基づいて設定する。 4. planstate->resultops に tts_ops を設定し、planstate->resultopsset を true に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execTuples.c 内の初期化処理。
  • src/backend/executor/execUtils.c:620（TTSOpsVirtual を指定して呼び出し）。
  • **呼び出し先**:
  • ExecAllocTableSlot（スロットの確保）。

関連データ構造

  • PlanState: 実行中のプランノードの状態を保持する構造体。
  • TupleTableSlot: タプルを保持するコンテナ。
  • TupleTableSlotOps: スロットの操作インターフェースを定義する構造体。

失敗モードと境界条件

  • メモリ不足時には ExecAllocTableSlot 経由でエラーが発生する可能性がある。
  • planstate が NULL の場合の挙動は未定義（ソースコード上でのガードなし）。

保守担当者向けメモ

  • 本関数はプランノードの初期化フェーズで呼び出される。
  • resultopsset フラグは、スロットの操作関数テーブルが確定したことを示すために使用される。
