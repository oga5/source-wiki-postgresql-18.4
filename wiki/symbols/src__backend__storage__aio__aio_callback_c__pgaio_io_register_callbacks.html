pgaio_io_register_callbacks

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_callback.c:85

目的

非同期I/Oハンドル（PgAioHandle）に対して、I/O完了時に実行されるコールバック関数を登録します。

シグネチャまたは形状

void pgaio_io_register_callbacks(PgAioHandle *ioh, PgAioHandleCallbackID cb_id, uint8 cb_data);

入力と出力

  • **入力**:
  • ioh: 登録対象の PgAioHandle 構造体へのポインタ。
  • cb_id: 登録するコールバックを識別する PgAioHandleCallbackID。
  • cb_data: コールバックに関連付ける8ビットの補助データ。
  • **出力**: なし（副作用として ioh の内部状態を更新）。

前提条件と不変条件

  • cb_id は PGAIO_HCB_MAX 以下であること。
  • 指定された cb_id に対応するコールバック定義（aio_handle_cbs）が、complete_shared または complete_local の少なくとも一方を実装していること。
  • ioh に登録済みのコールバック数が PGAIO_HANDLE_MAX_CALLBACKS 未満であること。

アルゴリズムまたは振る舞い

1. aio_handle_cbs 配列から cb_id に対応するエントリを取得します。 2. cb_id の範囲チェックおよびコールバック実装の有無を検証し、不正な場合はエラーを発生させます。 3. ioh の現在のコールバック数が上限を超えていないか確認し、超えている場合は PANIC を発生させます。 4. ioh->callbacks 配列の末尾に cb_id を、ioh->callbacks_data 配列に cb_data を格納します。 5. デバッグログ（DEBUG3）を出力します。 6. ioh->num_callbacks をインクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio.c
  • src/backend/storage/buffer/bufmgr.c
  • src/backend/storage/smgr/md.c
  • src/test/modules/test_aio/test_aio.c
  • **呼び出し先**:
  • elog / Assert（エラーハンドリング）
  • pgaio_debug_io（ログ出力）

関連データ構造

  • PgAioHandle: I/O操作の状態と登録されたコールバックを保持する構造体。
  • PgAioHandleCallbacksEntry: コールバック関数ポインタと識別子を保持する構造体。
  • aio_handle_cbs: 利用可能な全コールバックを定義する静的配列。

失敗モードと境界条件

  • cb_id が aio_handle_cbs の範囲外の場合: ERROR をスロー。
  • 指定された cb_id のコールバックに実行可能な関数（complete_shared または complete_local）が定義されていない場合: ERROR をスロー。
  • 登録数が PGAIO_HANDLE_MAX_CALLBACKS を超える場合: PANIC をスロー。

保守担当者向けメモ

  • コールバックは登録された順序の逆順で実行されるため、登録順序が処理の優先度に影響します。
  • 本関数は PgAioHandle の内部配列を直接操作するため、マルチスレッド環境での競合が発生しないよう、呼び出し側で適切な排他制御が行われていることを前提としています。
