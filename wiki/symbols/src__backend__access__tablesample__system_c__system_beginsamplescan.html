system_beginsamplescan

定義

  • 種別: function
  • 定義箇所: src/backend/access/tablesample/system.c:138

目的

TABLESAMPLE SYSTEM メソッドによるスキャンを開始するための初期化処理を行います。サンプリング率に基づいたハッシュのカットオフ値を算出し、スキャン状態を初期化します。

シグネチャまたは形状

static void system_beginsamplescan(SampleScanState *node, Datum *params, int nparams, uint32 seed)

入力と出力

  • **入力**:
  • node: スキャン状態を保持する SampleScanState 構造体。
  • params: サンプリング率（0〜100のパーセンテージ）を含む Datum 配列。
  • nparams: パラメータ数。
  • seed: ハッシュ計算に使用するシード値。
  • **出力**: なし（node->tsm_state に格納された SystemSamplerData を更新）。

前提条件と不変条件

  • params[0] に指定されたサンプリング率が 0 から 100 の範囲内であること。
  • node->tsm_state が SystemSamplerData 型として適切に割り当てられていること。

アルゴリズムまたは振る舞い

1. サンプリング率が 0〜100 の範囲外または NaN の場合、エラーを報告します。 2. 指定されたパーセンテージからハッシュのカットオフ値（dcutoff）を計算します。 3. SystemSamplerData の cutoff に計算値を、seed に引数のシード値を設定します。 4. スキャン開始位置として nextblock を 0 に、lt（オフセット）を InvalidOffsetNumber に初期化します。 5. サンプリング率が 1% 以上の場合、node->use_bulkread を有効にします。 6. 全タプルをスキャンするため、node->use_pagemode を常に true に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: tsm_system_handler を介して SampleScan ノードから呼び出されます。
  • **呼び出し先**: DatumGetFloat4（マクロ）、rint（標準ライブラリ）、ereport（エラー報告）。

関連データ構造

  • SystemSamplerData: スキャン中の状態（カットオフ値、シード、ブロック位置）を保持します。
  • SampleScanState: スキャン実行時のノード状態を保持します。

失敗モードと境界条件

  • サンプリング率が 0 未満または 100 を超える場合、ERRCODE_INVALID_TABLESAMPLE_ARGUMENT でエラー終了します。
  • NaN が渡された場合も同様にエラーとなります。

保守担当者向けメモ

  • node->use_bulkread の 1% という閾値は経験的な推定値です。
  • hash_any を使用したハッシュ計算を行うため、プラットフォーム間で一貫したサンプリング結果が得られる設計となっています。
