lclContext

定義

  • **種別**: struct
  • **定義箇所**: src/bin/pg_dump/pg_backup_custom.c:68

目的

pg_dump のカスタム形式アーカイブにおける、アーカイブ全体の実行時コンテキストを保持する構造体です。ファイルハンドル、圧縮状態、およびシーク位置の追跡を管理します。

シグネチャまたは形状

typedef struct {
    CompressorState *cs;
    int             hasSeek;
    pgoff_t         lastFilePos;
} lclContext;

入力と出力

  • **入力**: pg_malloc0 を介してメモリが割り当てられ、ArchiveHandle->formatData に格納されます。
  • **出力**: アーカイブの読み込みおよび書き込み処理において、現在のファイル位置や圧縮状態を保持・更新するために参照されます。

前提条件と不変条件

  • lastFilePos は読み込み時にのみ使用されます。
  • hasSeek が偽（シーク不可）の場合、lastFilePos は無効な値となる可能性があります。

アルゴリズムまたは振る舞い

  • 本構造体は、カスタム形式アーカイブの各操作（データブロックの読み書きやシーク）において、現在のストリーム状態を追跡します。
  • _getFilePos 関数などで参照され、アーカイブ内のデータブロック位置を特定するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_custom.c 内の複数の関数（_getFilePos 等）が ArchiveHandle から本構造体を取得して利用しています。
  • **呼び出し先**: 構造体内部で CompressorState を保持しており、圧縮・解凍処理の委譲先として機能します。

関連データ構造

  • CompressorState: 圧縮ストリームの抽象化状態を保持します。
  • ArchiveHandle: 本構造体を formatData メンバとして保持する上位のアーカイブ管理構造体です。

失敗モードと境界条件

  • シーク不可なストリーム（パイプなど）を使用する場合、hasSeek が偽となり、ランダムアクセスが制限されます。この場合、データブロックの検索には順次スキャンが必要となります。

保守担当者向けメモ

  • Windows環境でのファイルハンドル操作や、マルチスレッド環境における lclContext へのアクセスについては、ソースコード内のコメントにプラットフォーム固有の制約が記載されているため、修正時は参照してください。
