validate_token_format

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-oauth.c:555

目的

クライアントから提供された認証ヘッダー文字列を解析し、OAuth 2.0 ベアラートークンの形式が妥当であるかを確認する。

シグネチャまたは形状

static const char *validate_token_format(const char *header);

入力と出力

  • **入力**: header (認証ヘッダー文字列)
  • **出力**:
  • 妥当な場合: トークン部分の先頭を指すポインター。
  • 不正または空の場合: NULL。

前提条件と不変条件

  • header は NULL であってはならない（Assert により検証）。
  • トークンは Bearer スキームで開始する必要がある。
  • トークン本体は、Base64 関連文字（英数字、-, ., _, ~, +, /）および末尾のパディング文字 = のみで構成される必要がある。

アルゴリズムまたは振る舞い

1. header が空文字列の場合、NULL を返す。 2. BEARER_SCHEME（"Bearer "）で開始するかを大文字小文字を区別せずに確認し、一致しない場合はエラーを報告して NULL を返す。 3. スキーム部分をスキップし、後続の空白文字を読み飛ばす。 4. トークン本体が空であればエラーを報告して NULL を返す。 5. トークン本体の各文字が許可された文字セットに含まれるかを確認する。 6. 妥当な形式であれば、トークン開始位置のポインターを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-oauth.c 内の認証処理フロー（validate 関数などから呼び出される）。
  • **呼び出し先**:
  • pg_strncasecmp, strlen, strspn (標準ライブラリ)
  • ereport (PostgreSQL エラー報告マクロ)

関連データ構造

  • BEARER_SCHEME: トークン形式を識別するための定数文字列。
  • b64token_allowed_set: トークンとして許可される文字セットを定義した静的文字列。

失敗モードと境界条件

  • **空のヘッダー**: 認証パラメータの問い合わせとみなされ、ログを出力せずに NULL を返す。
  • **不正なスキーム**: COMMERROR レベルで ERRCODE_PROTOCOL_VIOLATION を報告。
  • **空のトークン本体**: COMMERROR レベルで ERRCODE_PROTOCOL_VIOLATION を報告。
  • **不正な文字**: トークン内に許可外の文字が含まれる場合、COMMERROR レベルで ERRCODE_PROTOCOL_VIOLATION を報告。セキュリティ上の理由から、問題の文字はログに出力されない。

保守担当者向けメモ

  • 本関数はトークンの構文チェックのみを行い、認証・認可の検証は行わない。
  • ログ出力時にトークン内容が漏洩しないよう、エラーメッセージにはトークン文字列を含めないこと。
