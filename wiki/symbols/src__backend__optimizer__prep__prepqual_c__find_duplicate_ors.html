find_duplicate_ors

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepqual.c:405

目的

論理式（AND/OR）ツリーを再帰的に走査し、冗長な定数項の削除、ネストの平坦化、および OR 句内の共通項抽出（分配法則の逆適用）を行うことで、最適化器が処理しやすい形式へ整理します。

シグネチャまたは形状

static Expr *find_duplicate_ors(Expr *qual, bool is_check)

入力と出力

  • **入力**:
  • qual: 処理対象の式ツリー（Expr）。
  • is_check: CHECK 制約のコンテキストか否かを示すフラグ。NULL 値の扱い（WHERE では FALSE 扱い、CHECK では TRUE 扱い）を制御します。
  • **出力**:
  • 簡略化・正規化された Expr。

前提条件と不変条件

  • is_check フラグにより、定数畳み込み時の NULL の評価ルールが切り替わります。
  • OR 句または AND 句の引数リストが空になった場合、論理的な恒等値（TRUE または FALSE）へ還元されます。

アルゴリズムまたは振る舞い

1. **再帰的走査**: 式ツリーを深さ優先で走査します。 2. **定数削除**:

  • OR 句内: FALSE を削除。TRUE または NULL（CHECK 制約時）に遭遇した場合は TRUE を返します。
  • AND 句内: TRUE を削除。FALSE または NULL（WHERE 句時）に遭遇した場合は FALSE を返します。

    • 3. **平坦化**: pull_ors または pull_ands を呼び出し、ネストされた同一論理演算子を単一リストに統合します。 4. **共通項抽出**: OR 句に対して process_duplicate_ors を呼び出し、分配法則の逆を適用して共通項を抽出します。 5. **再構築**: AND 句の場合、要素数に応じて TRUE、単一式、または make_andclause によるノードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/prep/prepqual.c 内の canonicalize_qual（推論）および再帰呼び出し。
  • **呼び出し先**: pull_ors, pull_ands, process_duplicate_ors, makeBoolConst, make_andclause。

関連データ構造

  • Expr: 式ツリーのノード。
  • BoolExpr: AND/OR 演算子を保持するノード構造。
  • Const: 定数ノード。

失敗モードと境界条件

  • 引数が空の AND 句は TRUE に還元されます。
  • OR 句内に TRUE が含まれる場合、式全体が TRUE に簡略化されます。
  • AND 句内に FALSE が含まれる場合、式全体が FALSE に簡略化されます。

保守担当者向けメモ

  • 本関数は is_check フラグを使用して、WHERE 句と CHECK 制約間での NULL 評価の差異を吸収しています。
  • process_duplicate_ors は OR 句の共通項抽出のみを担当するため、AND 句に対する同様の最適化は本関数内での平坦化と定数削除に依存しています。
