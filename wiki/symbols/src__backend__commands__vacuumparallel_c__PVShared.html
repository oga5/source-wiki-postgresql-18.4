PVShared

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/vacuumparallel.c:57

目的

並列 VACUUM 処理において、リーダープロセスとワーカープロセス間で共有される制御情報および統計情報を保持する。

シグネチャまたは形状

typedef struct PVShared {
    Oid relid;
    int elevel;
    int64 queryid;
    double reltuples;
    bool estimated_count;
    int maintenance_work_mem_worker;
    int ring_nbuffers;
    pg_atomic_uint32 cost_balance;
    pg_atomic_uint32 active_nworkers;
    pg_atomic_uint32 idx;
    dsa_handle dead_items_dsa_handle;
    dsa_pointer dead_items_handle;
    VacDeadItemsInfo dead_items_info;
} PVShared;

入力と出力

  • **入力**: parallel_vacuum_init 時に初期化され、DSM（動的共有メモリ）上に配置される。
  • **出力**: 各ワーカープロセスがこの構造体を通じて、共有メモリ上の TidStore へのアクセスや、コストベースの遅延制御、インデックス処理の進捗管理を行う。

前提条件と不変条件

  • relid, elevel, queryid は並列処理中に変更されない。
  • maintenance_work_mem_worker は、全ワーカーの合計メモリ消費量が単一プロセス時の maintenance_work_mem を超えないよう調整される。

アルゴリズムまたは振る舞い

  • **コスト管理**: cost_balance と active_nworkers を使用し、並列ワーカー全体でのコストベースの遅延制御を調整する。
  • **インデックス処理**: idx をアトミックカウンタとして使用し、複数のワーカー間で未処理のインデックスを動的に割り当てる。
  • **メモリ共有**: dead_items_dsa_handle および dead_items_handle を介して、削除対象タプル情報（TidStore）を共有する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/vacuumparallel.c 内の parallel_vacuum_init（領域確保）、parallel_vacuum_main（参照）、および parallel_vacuum_bulkdel_all_indexes 等の各処理関数。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはない。

関連データ構造

  • TidStore: dead_items_handle を介して参照される削除対象タプル情報の格納先。
  • VacDeadItemsInfo: 共有される削除対象タプルの統計情報。
  • ParallelVacuumState: 本構造体を管理する上位の並列 VACUUM 状態構造体。

失敗モードと境界条件

  • ワーカープロセス内でエラーが発生した場合、parallel_vacuum_error_callback が呼び出され、本構造体の情報を用いて失敗したインデックス処理が特定される。
  • reltuples が -1 の場合、タプル数は不明（estimated_count が真）として扱われる。

保守担当者向けメモ

  • 本構造体は DSM 上に配置されるため、フィールドの追加や変更を行う際は、共有メモリレイアウトおよびアトミック操作の整合性に注意すること。
  • maintenance_work_mem_worker の計算ロジックは、並列実行時のメモリ消費量を単一プロセス時と同等に抑えるための重要な制御点である。
