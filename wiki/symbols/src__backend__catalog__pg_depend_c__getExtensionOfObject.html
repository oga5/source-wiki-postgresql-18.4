getExtensionOfObject

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_depend.c:733

目的

指定されたオブジェクト（classId および objectId）がどの拡張機能（Extension）に属しているかを特定し、その拡張機能の OID を返します。

シグネチャまたは形状

Oid getExtensionOfObject(Oid classId, Oid objectId)

入力と出力

  • **入力**:
  • classId: 対象オブジェクトのカタログ ID。
  • objectId: 対象オブジェクトの OID。
  • **出力**:
  • 拡張機能に属している場合はその拡張機能の OID。
  • どの拡張機能にも属していない場合は InvalidOid。

前提条件と不変条件

  • pg_depend システムカタログに対する AccessShareLock を取得して読み取りを行います。
  • DependDependerIndexId インデックスを使用して検索を行います。

アルゴリズムまたは振る舞い

1. pg_depend テーブルを AccessShareLock で開きます。 2. classId と objectId をキーとして DependDependerIndexId インデックスをスキャンします。 3. スキャン結果の各レコードに対し、以下の条件を確認します。

  • refclassid が ExtensionRelationId であること。
  • deptype が DEPENDENCY_EXTENSION であること。

    • 4. 条件に合致するレコードが見つかった場合、その refobjid（拡張機能の OID）を戻り値として保持し、スキャンを終了します。 5. 最後にテーブルをクローズし、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/shippable.c
  • src/backend/catalog/pg_depend.c
  • src/backend/commands/extension.c
  • **呼び出し先**:
  • table_open, table_close
  • systable_beginscan, systable_getnext, systable_endscan
  • ScanKeyInit
  • GETSTRUCT

関連データ構造

  • pg_depend: 依存関係を保持するシステムカタログ。
  • Form_pg_depend: pg_depend のタプル構造体。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • 対象オブジェクトがどの拡張機能にも依存していない場合、InvalidOid を返します。
  • 複数の拡張機能に関連付けられている可能性がある場合、最初に見つかった DEPENDENCY_EXTENSION レコードの OID を返して終了します。

保守担当者向けメモ

  • 本関数は pg_depend のインデックススキャンに依存しています。インデックス定義に変更があった場合は注意が必要です。
  • 拡張機能の管理に関するロジック（DEPENDENCY_EXTENSION の判定など）は catalog/dependency.h の定義と密接に関連しています。
