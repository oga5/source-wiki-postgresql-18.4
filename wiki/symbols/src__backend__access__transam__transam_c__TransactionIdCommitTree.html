TransactionIdCommitTree

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/transam.c:239

目的

指定されたトランザクションID（XID）およびそのサブトランザクション群のステータスを「コミット済み（TRANSACTION_STATUS_COMMITTED）」として一括設定します。

シグネチャまたは形状

void TransactionIdCommitTree(TransactionId xid, int nxids, TransactionId *xids)

入力と出力

  • **入力**:
  • xid: 親トランザクションのID。
  • nxids: サブトランザクションの数。
  • xids: サブトランザクションIDの配列。
  • **出力**: なし（void）。

前提条件と不変条件

  • TransactionIdSetTreeStatus を介して、CLOG（Commit Log）および関連する共有メモリ領域の状態を更新します。
  • ログシーケンス番号（LSN）は InvalidXLogRecPtr として渡され、この時点では特定のWALレコードとは紐付けられません。

アルゴリズムまたは振る舞い

内部関数 TransactionIdSetTreeStatus を呼び出し、親トランザクション xid とその子トランザクション配列 xids のステータスを TRANSACTION_STATUS_COMMITTED に更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/twophase.c: 二相コミット処理時。
  • src/backend/access/transam/xact.c: トランザクションのコミット処理時。
  • **呼び出し先**:
  • TransactionIdSetTreeStatus（内部関数）

関連データ構造

  • TransactionId: トランザクションを識別する32ビットの循環値。
  • CLOG (Commit Log): トランザクションのコミット状態を永続化する領域。

失敗モードと境界条件

  • 共有メモリのロック競合や、CLOGのページ書き込みエラーが発生する可能性があります。
  • nxids が0の場合、親トランザクションのみが更新対象となります。

保守担当者向けメモ

  • 本関数はトランザクションツリーのステータス更新を行う高レベルAPIです。
  • 状態更新の詳細は TransactionIdSetTreeStatus の実装に依存します。
  • 頻繁に呼び出されるため、パフォーマンスへの影響を考慮する必要があります。
