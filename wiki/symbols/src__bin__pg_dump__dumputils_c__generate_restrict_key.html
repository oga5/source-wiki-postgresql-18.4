generate_restrict_key

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/dumputils.c:968

目的

ランダムな文字列（制限キー）を生成し、ダンプ処理における識別や制限用途に供する。

シグネチャまたは形状

char *generate_restrict_key(void)

入力と出力

  • 入力: なし
  • 出力: 64バイトのバッファから生成された、ヌル終端の文字列（char *）。メモリは palloc で確保される。失敗時は NULL を返す。

前提条件と不変条件

  • pg_strong_random が利用可能であること。
  • 生成される文字列は restrict_chars 配列に含まれる文字のみで構成される。
  • 戻り値の文字列長は常に63文字（＋ヌル終端）である。

アルゴリズムまたは振る舞い

1. 64バイトのスタックバッファ buf を確保する。 2. pg_strong_random を呼び出し、バッファを暗号論的に安全な乱数で埋める。失敗した場合は NULL を返す。 3. バッファ内の各バイト値を restrict_chars の長さで剰余演算し、対応するインデックスの文字を ret バッファに格納する。 4. 最後にヌル終端を付与して文字列を完成させ、呼び出し元に返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_dump/pg_dump.c
  • src/bin/pg_dump/pg_dumpall.c
  • src/bin/pg_dump/pg_restore.c
  • 呼び出し先:
  • pg_strong_random (外部)
  • palloc (外部)
  • strlen (外部)

関連データ構造

  • restrict_chars: 本関数が参照する文字セットを定義するグローバル配列（ソースコード内の定義に依存）。

失敗モードと境界条件

  • pg_strong_random が乱数を生成できない場合、関数は NULL を返す。呼び出し元でのエラーハンドリングが必要。

保守担当者向けメモ

  • restrict_chars の内容が変更された場合、生成されるキーの文字種が変化する。
  • buf のサイズ（64バイト）を変更する場合は、戻り値のバッファサイズとループ範囲の整合性に注意すること。
