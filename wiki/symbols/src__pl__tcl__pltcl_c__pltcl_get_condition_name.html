pltcl_get_condition_name

定義

  • 種別: function
  • 定義箇所: src/pl/tcl/pltcl.c:2074

目的

与えられた SQLSTATE（整数値）に対応するエラー条件名（文字列）を検索し、返却する。

シグネチャまたは形状

static const char *pltcl_get_condition_name(int sqlstate)

入力と出力

  • **入力**: sqlstate (int) - 検索対象の SQLSTATE コード。
  • **出力**: const char * - 対応する条件名の文字列。一致するものがない場合は "unrecognized_sqlstate" を返す。

前提条件と不変条件

  • exception_name_map という名前の配列がスコープ内に存在し、label（文字列）と sqlerrstate（整数）を持つ構造体のリストとして定義されている必要がある。
  • 配列は label が NULL である要素を終端として持つこと。

アルゴリズムまたは振る舞い

1. exception_name_map を先頭から順に走査する。 2. 現在の要素の sqlerrstate が引数の sqlstate と一致するか確認する。 3. 一致した場合は、その要素の label を即座に返す。 4. 配列の終端（label == NULL）に達するまで一致が見つからない場合、デフォルト値 "unrecognized_sqlstate" を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/tcl/pltcl.c:1945（エラー処理時に SQLSTATE を Tcl オブジェクトへ変換する際）
  • **呼び出し先**:
  • 外部関数呼び出しなし（配列の走査のみ）。

関連データ構造

  • TclExceptionNameMap: SQLSTATE と条件名のマッピングを保持する構造体（ソース内の exception_name_map で使用）。

失敗モードと境界条件

  • **境界条件**: 既知の SQLSTATE に該当しない値が渡された場合、エラーにはならず "unrecognized_sqlstate" が返される。
  • **失敗モード**: 特になし。

保守担当者向けメモ

  • 本関数は exception_name_map の内容に完全に依存している。新しい SQLSTATE をサポートする必要がある場合は、exception_name_map の定義を更新すること。
