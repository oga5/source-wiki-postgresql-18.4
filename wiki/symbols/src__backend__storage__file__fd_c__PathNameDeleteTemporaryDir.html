PathNameDeleteTemporaryDir

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/fd.c:1694

目的

指定されたパスの一時ディレクトリを再帰的に削除します。

シグネチャまたは形状

void PathNameDeleteTemporaryDir(const char *dirname)

入力と出力

  • **入力**: dirname (削除対象のディレクトリパスを示す文字列)
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • 対象のディレクトリが存在しない場合、エラーを発生させず静かに終了します。
  • 削除処理は walkdir を介して行われます。

アルゴリズムまたは振る舞い

1. stat を用いてディレクトリの存在を確認します。存在しない場合は ENOENT を確認し、処理を終了します。 2. walkdir 関数を呼び出し、ディレクトリ内の各エントリに対して unlink_if_exists_fname を適用して削除を試みます。 3. 削除失敗時はログ出力が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/file/fileset.c:163
  • **呼び出し先**:
  • stat (システムコール)
  • walkdir (ディレクトリ走査関数)
  • unlink_if_exists_fname (各ファイル削除用コールバック)

関連データ構造

  • struct stat (ディレクトリの存在確認に使用)

失敗モードと境界条件

  • ディレクトリが存在しない場合: stat が失敗し errno == ENOENT であれば正常終了します。
  • 削除失敗時: walkdir の仕様により、現在の実装では呼び出し元に成功・失敗を通知する手段がなく、ログ出力のみが行われます。

保守担当者向けメモ

  • walkdir が状態保持をサポートしていないため、操作の成否を呼び出し元へ伝達できません。クリーンアップパスでの使用を前提としています。
