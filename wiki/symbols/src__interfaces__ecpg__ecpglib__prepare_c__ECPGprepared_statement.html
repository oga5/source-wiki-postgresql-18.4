ECPGprepared_statement

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/prepare.c:370

目的

指定された接続において、名前で識別されるプリペアドステートメントのSQL文字列を取得します。

シグネチャまたは形状

char *ECPGprepared_statement(const char *connection_name, const char *name, int lineno);

入力と出力

  • **入力**:
  • connection_name: 接続名（NULL の場合はデフォルト接続）。
  • name: 取得対象のプリペアドステートメント名。
  • lineno: 呼び出し元の行番号（エラーログ用）。
  • **出力**:
  • 成功時: プリペアドステートメントに対応するSQL文字列へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • ecpg_init による接続の初期化が成功している必要があります。
  • 接続が確立されていない、または指定された名前のステートメントが存在しない場合は NULL を返します。

アルゴリズムまたは振る舞い

1. ecpg_get_connection を呼び出し、connection_name に対応する接続オブジェクトを取得します。 2. ecpg_init を呼び出し、接続の有効性を確認します。失敗した場合は NULL を返します。 3. ecpg_prepared を呼び出し、接続オブジェクト内のリストから指定された name に一致するステートメントを検索し、そのSQL文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPGのテストスイート（compat_informix-sqlda.c, preproc-autoprep.c, sql-declare.c 等）から、動的SQLの検証やデバッグ目的で頻繁に呼び出されています。
  • **呼び出し先**:
  • ecpg_get_connection (接続取得)
  • ecpg_init (接続初期化)
  • ecpg_prepared (ステートメント検索)

関連データ構造

  • connection: 接続状態を保持する構造体。
  • prepared_statement: プリペアドステートメントの情報を保持する構造体（ecpg_prepared 内で使用）。

失敗モードと境界条件

  • 指定した connection_name が存在しない場合、ecpg_init が失敗し NULL を返します。
  • 指定した name のプリペアドステートメントが接続内に存在しない場合、ecpg_prepared は NULL を返します。

保守担当者向けメモ

  • 本関数は主にテストやデバッグインターフェースとして利用されています。
  • 返される文字列ポインタの所有権はライブラリ側にあり、呼び出し側で解放してはなりません。
