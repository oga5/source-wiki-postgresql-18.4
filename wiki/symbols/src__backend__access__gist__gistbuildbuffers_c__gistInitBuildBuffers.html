gistInitBuildBuffers

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistbuildbuffers.c:43

目的

GiST インデックスのバッファリングビルドプロセスで使用される、ノードバッファ管理構造体（GISTBuildBuffers）を初期化し、メモリおよび一時ファイル管理のためのリソースを割り当てる。

シグネチャまたは形状

GISTBuildBuffers *gistInitBuildBuffers(int pagesPerBuffer, int levelStep, int maxLevel)

入力と出力

  • **入力**:
  • pagesPerBuffer: 各ノードバッファが保持するページ数。
  • levelStep: バッファリングのレベルステップ数。
  • maxLevel: インデックスのルートレベル。
  • **出力**:
  • 初期化された GISTBuildBuffers 構造体へのポインタ。

前提条件と不変条件

  • CurrentMemoryContext が、ビルドプロセス全体で永続的なメモリ割り当てに適したコンテキストであることを前提とする。
  • 一時ファイル操作には BufFile を使用する。

アルゴリズムまたは振る舞い

1. GISTBuildBuffers 構造体をメモリ上に確保し、引数を設定する。 2. BufFileCreateTemp を呼び出し、メモリから溢れたバッファページを退避させるための一時ファイルを作成する。 3. 空きブロック管理用の配列（freeBlocks）を初期化する。 4. インデックスブロックとノードバッファを関連付けるためのハッシュテーブル（nodeBuffersTab）を作成する。 5. レベルごとのバッファリスト（buffersOnLevels）および、メモリ上にロードされたバッファを追跡するための配列（loadedBuffers）を初期化する。 6. 設定された maxLevel をルートレベルとして保持し、構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistbuild.c:770
  • **呼び出し先**: palloc (メモリ確保), BufFileCreateTemp (一時ファイル作成), hash_create (ハッシュテーブル作成)

関連データ構造

  • GISTBuildBuffers: バッファリングビルドの状態を保持するメイン構造体。
  • GISTNodeBuffer: ノードごとのバッファ情報を保持する構造体。
  • BufFile: 一時ファイル操作用インターフェース。

失敗モードと境界条件

  • メモリ不足時や一時ファイルの作成失敗時には、PostgreSQL のエラーハンドリング機構により elog(ERROR) が発生し、処理が停止する。

保守担当者向けメモ

  • 本関数は GiST バッファリングビルドの初期化フェーズを担当する。
  • nodeBuffersTab のハッシュサイズは 1024 で固定されている。
  • freeBlocks や loadedBuffers の初期サイズは 32 であり、必要に応じて拡張される設計となっている。
