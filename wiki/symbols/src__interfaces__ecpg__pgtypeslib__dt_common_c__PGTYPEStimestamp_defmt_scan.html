PGTYPEStimestamp_defmt_scan

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/ecpg/pgtypeslib/dt_common.c:2520

目的

指定されたフォーマット文字列に従って入力文字列を解析し、タイムスタンプ構成要素（年、月、日、時、分、秒、タイムゾーン）を抽出して各引数に格納します。

シグネチャまたは形状

int PGTYPEStimestamp_defmt_scan(char **str, char *fmt, timestamp *d,
                                int *year, int *month, int *day,
                                int *hour, int *minute, int *second,
                                int *tz);

入力と出力

  • **入力**:
  • str: 解析対象文字列へのポインタ（解析が進むと更新される）。
  • fmt: 解析ルールを定義するフォーマット文字列。
  • **出力**:
  • d, year, month, day, hour, minute, second, tz: 抽出された値が格納される各ポインタ。
  • **戻り値**: 成功時に 0、失敗時に 1 を返します。

前提条件と不変条件

  • fmt に含まれる % で始まる各指定子に対応する値が str 内に存在する必要があります。
  • str が指す文字列は、解析の進行に伴い消費されます。
  • %D などの一部のフォーマット指定子は、内部で再帰的に本関数を呼び出します。

アルゴリズムまたは振る舞い

1. fmt を走査し、空白をスキップしながら str と照合します。 2. % に遭遇した場合、続く文字（指定子）に応じて以下の処理を行います。

  • **日付関連**: a, A（曜日）、b, h, B（月名）は定義済みのテーブルと照合します。d, e, m は pgtypes_defmt_scan を介して数値を抽出します。
  • **年**: y, g は2桁の年を解析し、必要に応じて1900年を加算します。C, G は年を抽出します。
  • **再帰**: %D のような複合フォーマットは、一時的なフォーマット文字列を生成して自身を再帰呼び出しします。

    • 3. 一致しない場合や解析エラーが発生した場合は err = 1 となり、処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/pgtypeslib/dt_common.c 内（再帰呼び出し）
  • src/interfaces/ecpg/pgtypeslib/timestamp.c:841
  • **呼び出し先**:
  • pgtypes_defmt_scan（数値抽出用）
  • pgtypes_alloc, free（メモリ管理）
  • strncmp, strlen, strcpy, strcat（文字列操作）

関連データ構造

  • timestamp: タイムスタンプ型。
  • pgtypes_date_weekdays_short, days, months, pgtypes_date_months: 日付解析用のキーワードテーブル。

失敗モードと境界条件

  • フォーマットと入力文字列が一致しない場合、エラー 1 を返します。
  • メモリ割り当て（pgtypes_alloc）に失敗した場合、エラー 1 を返します。
  • 2桁の年（%y）が100未満の場合、自動的に1900年が加算されます。

保守担当者向けメモ

  • 本関数は strptime に似た動作をしますが、ECPG 独自の解析ルールに従います。
  • %a, %A, %B などの解析では、対応するテーブルの末尾まで検索を行うため、テーブルの整合性に注意してください。
  • 再帰呼び出し（%D 等）を行う際は、動的にメモリを確保するため、メモリリークが発生しないよう free の呼び出しを確認してください。
