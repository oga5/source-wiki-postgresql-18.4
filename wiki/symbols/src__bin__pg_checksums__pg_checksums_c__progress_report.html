progress_report

定義

  • 種別: function
  • 定義箇所: src/bin/pg_checksums/pg_checksums.c:122

目的

データディレクトリのチェックサム処理における進捗状況を標準エラー出力（stderr）へ表示します。

シグネチャまたは形状

static void progress_report(bool finished);

入力と出力

  • **入力**:
  • finished (bool): 処理が完了したかどうかを示すフラグ。
  • **出力**: なし（標準エラー出力への書き込みのみ）。

前提条件と不変条件

  • グローバル変数 showprogress が true である必要があります（Assert により検証）。
  • current_size および total_size が適切に更新されている必要があります。

アルゴリズムまたは振る舞い

1. showprogress が有効か確認します。 2. 最後に報告してから1秒経過していない場合、かつ finished が false の場合は何もせず終了します。 3. current_size が total_size を超えている場合は total_size を補正します。 4. 現在の進捗率（%）を計算します。 5. current_size と total_size（MB単位）および進捗率を stderr に出力します。 6. 処理が完了していないかつ端末（isatty）であれば \r を出力して行を維持し、完了時または非端末であれば \n を出力して改行します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース excerpt 内には明示的な呼び出し元は記載されていません。
  • **呼び出し先**: time(), fprintf(), fputc(), fileno(), isatty()。
  • **注意**: ツールが検出した pg_amcheck.c や pg_basebackup.c 内の progress_report は同名の別関数であり、本シンボルとは直接の関係はありません。

関連データ構造

  • current_size (グローバル変数): 現在の処理済みサイズ。
  • total_size (グローバル変数): 処理対象の総サイズ。
  • last_progress_report (グローバル変数): 最後に報告した時刻。

失敗モードと境界条件

  • total_size が 0 の場合、進捗率は 0% として計算されます。
  • 端末以外（パイプ等）への出力時は、\r ではなく \n が使用されます。

保守担当者向けメモ

  • 本関数は pg_checksums.c 内のローカルな進捗表示ロジックです。他ツール（pg_basebackup 等）の同名関数と混同しないよう注意してください。
  • 進捗表示を有効にする場合、処理前に全ファイルサイズの事前計算が必要となるため、大規模環境では実行時間に影響を与えます。
