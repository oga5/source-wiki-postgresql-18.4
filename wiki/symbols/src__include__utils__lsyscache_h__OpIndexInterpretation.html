OpIndexInterpretation

定義

  • 種別: struct
  • 定義箇所: src/include/utils/lsyscache.h:25

目的

演算子がインデックス内でどのように解釈されるか（演算子族、比較型、および左右の入力データ型）を保持するための構造体です。

シグネチャまたは形状

typedef struct OpIndexInterpretation
{
	Oid			opfamily_id;	/* 演算子を含む演算子族 */
	CompareType cmptype;		/* 汎用的な比較型 */
	Oid			oplefttype;		/* 宣言された左側入力データ型 */
	Oid			oprighttype;	/* 宣言された右側入力データ型 */
} OpIndexInterpretation;

入力と出力

本構造体は主にシステムカタログキャッシュから情報を抽出した結果を格納するために使用されます。関数がこの構造体のリストを palloc で動的に割り当てて返却するケースがあります。

前提条件と不変条件

  • 演算子族（opfamily_id）およびデータ型（oplefttype, oprighttype）は Oid で管理されます。
  • cmptype は CompareType 列挙型に依存します。

アルゴリズムまたは振る舞い

システムカタログ（pg_proc 等）から取得した演算子情報を、プランナやパーサがインデックス利用の可否を判断するために扱いやすい形式へ変換する際の中間表現として機能します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • src/backend/optimizer/path/allpaths.c
  • src/backend/optimizer/util/predtest.c
  • src/backend/parser/parse_expr.c
  • **呼び出し先**:
  • src/backend/utils/cache/lsyscache.c 内の関数が、この構造体を palloc し、値を設定してリストとして返却する処理を行っています。

関連データ構造

  • CompareType: 比較の種別を定義する型。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • palloc を使用してメモリを確保しているため、メモリ不足時にはエラーが発生する可能性があります。
  • 呼び出し側でリストとして受け取った場合、適切なメモリ管理（解放）が必要です。

保守担当者向けメモ

  • 本構造体は lsyscache.c で生成されるリストの要素として頻繁に利用されます。
  • 構造体のフィールドを変更する場合は、これを利用しているオプティマイザおよびパーサ側のコードへの影響を考慮してください。
