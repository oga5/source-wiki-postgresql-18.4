inv_create

定義

  • 種別: function
  • 定義箇所: src/backend/storage/large_object/inv_api.c:172

目的

新しいラージオブジェクト（LO）を作成し、所有権の依存関係登録および作成後のフック処理を実行して、システムカタログ上で利用可能な状態にする。

シグネチャまたは形状

Oid inv_create(Oid lobjId)

入力と出力

  • **入力**: lobjId (Oid) - 作成するLOのOID。InvalidOid を指定した場合はシステムが自動的にOIDを割り当てる。
  • **出力**: (Oid) - 作成されたラージオブジェクトのOID。

前提条件と不変条件

  • 現在のトランザクション内で実行される必要がある。
  • LargeObjectRelationId（pg_largeobject）に対する適切な権限が必要。

アルゴリズムまたは振る舞い

1. LargeObjectCreate(lobjId) を呼び出し、空のデータページを持つ新しいLOをカタログ上に作成する。 2. recordDependencyOnOwner を呼び出し、現在のユーザー（GetUserId()）と作成されたLOの間に所有権の依存関係を記録する。 3. InvokeObjectPostCreateHook を呼び出し、LO作成後のフック処理を実行する。 4. CommandCounterIncrement() を呼び出し、作成されたタプルを後続の操作から可視化する。 5. 作成されたLOのOIDを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-fsstubs.c (複数の箇所で呼び出しを確認)
  • **呼び出し先**:
  • LargeObjectCreate
  • recordDependencyOnOwner
  • InvokeObjectPostCreateHook
  • CommandCounterIncrement
  • GetUserId

関連データ構造

  • LargeObjectRelationId: LOの依存関係記録に使用されるカタログID。

失敗モードと境界条件

  • lobjId が既に存在する場合、LargeObjectCreate の内部実装に依存してエラーが発生する可能性がある。
  • トランザクションがアボート状態にある場合、カタログ更新に失敗する。

保守担当者向けメモ

  • LOの依存関係記録には、後方互換性の理由から LargeObjectMetadataRelationId ではなく LargeObjectRelationId が使用されている。この仕様変更は pg_dump 等の外部ツールに影響を与えるため注意が必要である。
