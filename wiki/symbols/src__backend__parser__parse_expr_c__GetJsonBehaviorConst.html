GetJsonBehaviorConst

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_expr.c:4903

目的

SQL/JSON関数において、指定された JsonBehaviorType に対応する定数ノード（Const）を生成して返します。

シグネチャまたは形状

Node *GetJsonBehaviorConst(JsonBehaviorType btype, int location)

入力と出力

  • **入力**:
  • btype: 生成する定数の種類を示す JsonBehaviorType 列挙型。
  • location: パーサー内のトークン位置を示す整数。
  • **出力**:
  • 生成された定数値を保持する Node*（実際には Const 構造体）。

前提条件と不変条件

  • btype が JSON_BEHAVIOR_DEFAULT または JSON_BEHAVIOR_ERROR の場合、呼び出し側で処理されるべきものとして Assert(false) が発生します。
  • 未知の btype が渡された場合は elog(ERROR) により処理が中断されます。

アルゴリズムまたは振る舞い

1. btype に応じて、定数の型（typid）、値（val）、および属性（len, isbyval, isnull）を決定します。

  • EMPTY_ARRAY / EMPTY_OBJECT: jsonb_in を使用して jsonb 型の定数を生成。
  • TRUE / FALSE: bool 型の定数を生成。
  • NULL / UNKNOWN / EMPTY: int4 型の NULL 定数を生成。

    • 2. 決定した属性を用いて makeConst を呼び出し、Const ノードを生成します。 3. location を設定し、ノードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_expr.c 内のコード（行4800付近）から呼び出されています。
  • **呼び出し先**: makeConst（ノード生成）、DirectFunctionCall1（jsonb_in 呼び出し）、BoolGetDatum、CStringGetDatum。

関連データ構造

  • JsonBehaviorType: 振る舞いの種類を定義する列挙型。
  • Const: PostgreSQLの定数ノード構造体。
  • Node: 構文解析ツリーの汎用ノード型。

失敗モードと境界条件

  • JSON_BEHAVIOR_DEFAULT および JSON_BEHAVIOR_ERROR は本関数内では処理されず、アサーションエラーとなります。
  • 定義されていない btype が渡された場合、elog(ERROR) が発生します。

保守担当者向けメモ

  • 本関数はSQL/JSONの特定の振る舞いに対する定数生成に特化しています。新しい振る舞いタイプを追加する場合は、switch 文にケースを追加し、適切な型情報（typid 等）を定義してください。
