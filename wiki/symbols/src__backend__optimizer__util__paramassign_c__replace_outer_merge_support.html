replace_outer_merge_support

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/paramassign.c:317

目的

上位レベルの MERGE クエリで定義された MergeSupportFunc を、サブクエリ等の下位レベルから参照可能にするため、PARAM_EXEC スロットへ変換・登録します。

シグネチャまたは形状

Param *replace_outer_merge_support(PlannerInfo *root, MergeSupportFunc *msf)

入力と出力

  • **入力**:
  • root: 現在のクエリレベルの PlannerInfo。
  • msf: 変換対象の MergeSupportFunc ノード。
  • **出力**:
  • Param: PARAM_EXEC として設定された新しいパラメータノード。

前提条件と不変条件

  • 呼び出し元の root->parse->commandType は CMD_MERGE であってはならない（MERGE クエリ自体の中では使用されない）。
  • MergeSupportFunc は、上位レベルの MERGE クエリの RETURNING リストに存在する必要がある。

アルゴリズムまたは振る舞い

1. root->parent_root を辿り、commandType が CMD_MERGE である上位クエリを特定する。見つからない場合はエラーを投げる。 2. msf を copyObject で複製する。 3. 新しい PlannerParamItem を作成し、複製した msf を格納する。 4. root->glob->paramExecTypes の長さを基に新しい paramId を割り当て、型情報をリストに追加する。 5. root->plan_params に pitem を追加する。 6. PARAM_EXEC 種別の Param ノードを生成し、割り当てた paramId と型情報を設定して返却する。

  • 重複排除は行わず、呼び出しごとに常に新しいスロットを割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/subselect.c:2006
  • **呼び出し先**:
  • copyObject (PostgreSQL 内部ユーティリティ)
  • makeNode (PostgreSQL 内部ユーティリティ)
  • lappend / lappend_oid (PostgreSQL 内部ユーティリティ)
  • list_length (PostgreSQL 内部ユーティリティ)
  • exprType (PostgreSQL 内部ユーティリティ)

関連データ構造

  • PlannerInfo: クエリのプランニング状態を保持。
  • MergeSupportFunc: MERGE クエリのサポート関数。
  • PlannerParamItem: plan_params リストに格納されるパラメータ定義。
  • Param: 実行時に参照されるパラメータノード。

失敗モードと境界条件

  • root を遡っても CMD_MERGE クエリが見つからない場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は MergeSupportFunc の重複排除を行わない設計です。これは、各参照が個別の実行時スロットを必要とするという判断に基づいています。
  • root->glob->paramExecTypes はプラン全体で共有されるため、ここでの変更はプラン全体に影響します。
