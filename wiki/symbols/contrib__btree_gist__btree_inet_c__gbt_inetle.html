gbt_inetle

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_inet.c:43

目的

inet 型の GiST インデックスにおいて、2つの値（double に変換済み）を比較し、左辺が右辺以下であるか（<=）を判定する。

シグネチャまたは形状

static bool gbt_inetle(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の左辺値（double 型へのポインタ）。
  • b: 比較対象の右辺値（double 型へのポインタ）。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • bool: *a <= *b が真であれば true、そうでなければ false を返す。

前提条件と不変条件

  • 引数 a および b は、double 型の値を指す有効なポインタである必要がある。
  • inet 型の値は、事前に convert_network_to_scalar 等を用いて double 型に変換されていることが前提である。

アルゴリズムまたは振る舞い

1. ポインタ a および b が指す値を double 型としてキャストし、デリファレンスする。 2. 両者の値を比較演算子 <= で評価し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_inet.c 内の静的配列定義（gbtree_ninfo 構造体の初期化時）において、比較関数として参照されている。
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わない。

関連データ構造

  • inetKEY: inet 型のインデックス格納用構造体。本関数は、この構造体に含まれる double 型の境界値比較に使用される。

失敗モードと境界条件

  • ポインタが NULL の場合、セグメンテーション違反が発生する可能性がある。
  • double 型への変換精度に依存するため、非常に大きなネットワーク範囲や複雑なサブネットマスクでは、比較結果が厳密な inet 型の順序と一致しない可能性がある。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で定義される汎用的な数値比較ロジックを補完する役割を持つ。
  • flinfo 引数は現在の実装では使用されていないが、将来的な拡張やインターフェースの互換性のために保持されている。
