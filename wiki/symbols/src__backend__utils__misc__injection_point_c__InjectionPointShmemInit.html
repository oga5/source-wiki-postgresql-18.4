InjectionPointShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/injection_point.c:248

目的

インジェクションポイント機能を管理するための共有メモリ領域を確保および初期化します。

シグネチャまたは形状

void InjectionPointShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上の ActiveInjectionPoints を初期化）

前提条件と不変条件

  • 本機能は --enable-injection-points を指定してビルドされている必要があります。
  • Postmaster プロセスで実行される場合、共有メモリの初期化が行われます。
  • Postmaster 以外（子プロセス）で実行される場合、共有メモリは既に初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を呼び出し、InjectionPoint hash という名前で InjectionPointsCtl 構造体分の共有メモリを確保します。 2. IsUnderPostmaster が偽（Postmaster プロセス）の場合:

  • 共有メモリが新規作成されたことを確認します。
  • max_inuse カウンタを 0 に初期化します。
  • 全てのインジェクションポイントエントリの generation カウンタを 0 に初期化します。

    • 3. IsUnderPostmaster が真（子プロセス）の場合:

  • 共有メモリが既に存在していることを確認します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:346
  • 呼び出し先: ShmemInitStruct, pg_atomic_init_u32, pg_atomic_init_u64

関連データ構造

  • InjectionPointsCtl: 共有メモリ上の管理構造体。
  • InjectionPointEntry: 各インジェクションポイントの定義を保持する構造体。

失敗モードと境界条件

  • IsUnderPostmaster が偽の状態で共有メモリが既に存在していた場合、または真の状態で共有メモリが存在しない場合、Assert により停止します。

保守担当者向けメモ

  • 本関数は PostgreSQL の共有メモリ初期化シーケンスの一部として呼び出されます。
  • USE_INJECTION_POINTS マクロが定義されていない場合、本関数は空の操作となります。
