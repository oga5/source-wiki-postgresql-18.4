GroupVarInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/selfuncs.c:3322

目的

クエリプランナが統計情報に基づいた選択率推定を行う際、特定の変数（または式）に関するメタデータと、その一意な値の数（ndistinct）を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct {
    Node       *var;        /* 式または変数 */
    RelOptInfo *rel;        /* 属するリレーション */
    double      ndistinct;  /* 一意な値の数 */
    bool        isdefault;  /* デフォルト値が使用されたか否か */
} GroupVarInfo;

入力と出力

  • 本構造体は、主に palloc または palloc0 を介してメモリ上に生成されます。
  • メンバ変数は、プランナの統計情報収集プロセスにおいて初期化され、後続の選択率計算ロジックへ渡されます。

前提条件と不変条件

  • var は単なる Var ノードだけでなく、複雑な式（Node）を含む可能性があります。
  • isdefault が true の場合、統計情報が欠落しており DEFAULT_NUM_DISTINCT が適用されたことを示します。

アルゴリズムまたは振る舞い

  • 統計情報取得の過程で、リスト（List）として管理されることが多く、lfirst 等を用いてリストから取り出され、各要素の統計情報が参照されます。
  • 主に、複数の変数や式をグループ化して統計的な特性を評価する際のコンテナとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/selfuncs.c 内の統計情報処理ロジック（3334行目以降の各箇所）から、リスト操作や初期化を通じて参照・生成されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • Node: 式の表現に使用。
  • RelOptInfo: 関連するリレーションの最適化情報。
  • List: GroupVarInfo のインスタンスを複数管理するために使用されます。

失敗モードと境界条件

  • 統計情報が不足している場合、isdefault フラグによりデフォルト値が使用されるため、計算結果は近似値となります。
  • メモリ割り当て失敗時は palloc の挙動に従い、PostgreSQL のエラーハンドリングが起動します。

保守担当者向けメモ

  • 本構造体は selfuncs.c 内の特定の統計推定ロジックに密結合しています。変更を加える際は、selfuncs.c 内のリスト操作箇所（3352, 3622, 3929 行目など）への影響を確認してください。
