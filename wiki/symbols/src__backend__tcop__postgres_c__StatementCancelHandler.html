StatementCancelHandler

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/postgres.c:3056

目的

クエリのキャンセル要求（SIGINT）を処理し、実行中のクエリを中断するためのフラグ設定とプロセスへの通知を行う。

シグネチャまたは形状

void StatementCancelHandler(SIGNAL_ARGS)

※ SIGNAL_ARGS は libpq/pqsignal.h 等で定義されるシグナルハンドラの引数。

入力と出力

  • 入力: シグナルハンドラとして呼び出される（引数はプラットフォーム依存）。
  • 出力: なし（副作用としてグローバル変数を更新）。

前提条件と不変条件

  • proc_exit_inprogress が false であること（プロセス終了処理中ではないこと）。

アルゴリズムまたは振る舞い

1. プロセス終了処理中（proc_exit_inprogress）でないことを確認する。 2. InterruptPending および QueryCancelPending を true に設定する。 3. SetLatch(MyLatch) を呼び出し、当該プロセスが待機状態であれば即座に起床させる。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/bgworker.c
  • src/backend/replication/logical/slotsync.c
  • src/backend/replication/walsender.c
  • src/backend/tcop/postgres.c

    • （これらは pqsignal(SIGINT, StatementCancelHandler) を通じてシグナルハンドラとして登録している）

  • 呼び出し先:
  • SetLatch（storage/ipc.h）

関連データ構造

  • InterruptPending（グローバルフラグ）
  • QueryCancelPending（グローバルフラグ）
  • MyLatch（プロセス固有のラッチ）

失敗モードと境界条件

  • プロセス終了処理中（proc_exit_inprogress が true）の場合、フラグの更新はスキップされる。
  • シグナルハンドラ内での実行であるため、非同期安全（async-signal-safe）な操作のみに限定されている。

保守担当者向けメモ

  • 本関数はシグナルハンドラとして登録されるため、呼び出し元は pqsignal を介したシグナル登録箇所となる。
  • QueryCancelPending が設定された後、メインループ内の CHECK_FOR_INTERRUPTS() マクロによって実際のキャンセル処理が実行される。
