ndistinct_for_combination

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/mvdistinct.c:424

目的

指定された列の組み合わせに基づき、サンプルデータから一意な値の数（NDistinct）を推定します。

シグネチャまたは形状

static double ndistinct_for_combination(double totalrows, StatsBuildData *data, int k, int *combination)

入力と出力

  • **入力**:
  • totalrows: テーブルの全行数。
  • data: 統計収集用のサンプルデータ構造体。
  • k: 組み合わせに含まれる列数。
  • combination: 統計対象とする列のインデックス配列。
  • **出力**:
  • 推定されたNDistinct値（double型）。

前提条件と不変条件

  • data内の各列の型に対して、lookup_type_cacheによりソート演算子（lt_opr）が取得可能であること。
  • numrows（サンプル数）が0より大きいこと。

アルゴリズムまたは振る舞い

1. **初期化**: multi_sort_initを用いて、指定されたk個の列に対するソートサポートを準備します。 2. **データ準備**: サンプルデータから対象列の値を抽出し、SortItem配列に格納します。 3. **ソート**: qsort_interruptibleとmulti_sort_compareを用いて、列の組み合わせをキーとして全サンプルデータをソートします。 4. **カウント**: ソート済みデータから、重複を除いた組み合わせの数（d）と、1回だけ出現する組み合わせの数（f1）を算出します。 5. **推定**: estimate_ndistinctを呼び出し、Duj1推定器を用いて全行数に対するNDistinctを推定して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/mvdistinct.c内の統計構築ロジックから呼び出されます。
  • **呼び出し先**: multi_sort_init, lookup_type_cache, multi_sort_add_dimension, qsort_interruptible, multi_sort_compare, estimate_ndistinct。

関連データ構造

  • StatsBuildData: 統計収集時のサンプルデータおよびメタデータを保持する構造体。
  • SortItem: ソート対象の値を保持する内部構造体。
  • MultiSortSupport: 複数列のソート状態を管理する構造体。

失敗モードと境界条件

  • lookup_type_cacheがソート演算子を見つけられない場合、elog(ERROR)により処理を中断します。
  • サンプルデータが空の場合、numrowsが0となり、ソートおよびカウント処理がスキップまたは不正な動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は列の組み合わせごとにメモリを動的に確保（palloc）するため、k（列数）が非常に大きい場合はメモリ消費量に注意が必要です。
  • ソート演算子には列のデフォルトの照合順序（Collation）が使用されます。必要に応じて列固有の照合順序を考慮する拡張が検討可能です。
