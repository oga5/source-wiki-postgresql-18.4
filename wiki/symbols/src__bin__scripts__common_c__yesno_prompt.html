yesno_prompt

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/common.c:135

目的

ユーザーに対して対話的に「はい/いいえ」の確認を求め、標準的な入力を受け付けるための共通ユーティリティ関数です。

シグネチャまたは形状

bool yesno_prompt(const char *question);

入力と出力

  • **入力**: question (表示する質問文の文字列)
  • **出力**: ユーザーが「はい」に対応する文字を入力した場合は true を、それ以外（「いいえ」）の場合は false を返します。

前提条件と不変条件

  • PG_YESLETTER および PG_NOLETTER で定義されたローカライズ済みの文字が入力として期待されます。
  • 内部で simple_prompt を呼び出すため、標準入力が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. 質問文と「(y/n)」形式の選択肢を組み合わせたプロンプト文字列を生成します。 2. 無限ループ内で simple_prompt を呼び出し、ユーザーの入力を取得します。 3. 入力された文字列が PG_YESLETTER と一致すれば true を、PG_NOLETTER と一致すれば false を返します。 4. 一致しない場合は、正しい入力を促すメッセージを表示し、入力を再試行します。 5. 取得した入力文字列は、各判定後に free() で解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/scripts/createuser.c (ロール作成時の権限確認)
  • src/bin/scripts/dropdb.c (データベース削除の確認)
  • src/bin/scripts/dropuser.c (ロール削除の確認)
  • **呼び出し先**:
  • simple_prompt (ユーザー入力の取得)
  • snprintf, strcmp, printf, free (標準ライブラリ関数)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • simple_prompt が失敗した場合の挙動は、当該関数の実装に依存します。
  • 入力文字列が PG_YESLETTER または PG_NOLETTER に一致しない限り、ループから抜け出せません。

保守担当者向けメモ

  • 本関数は src/bin/scripts/common.h で宣言されており、クライアントツール間で共通の対話インターフェースを提供します。
  • 翻訳対象の文字列（_() マクロ）を使用しているため、ロケール設定に応じた挙動となります。
