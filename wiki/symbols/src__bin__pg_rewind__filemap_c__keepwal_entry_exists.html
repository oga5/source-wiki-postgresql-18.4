keepwal_entry_exists

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/filemap.c:266

目的

指定されたファイルパスが、保持対象の WAL エントリ（keepwal）として登録されているかどうかを確認する。

シグネチャまたは形状

static bool keepwal_entry_exists(const char *path)

入力と出力

  • **入力**: path (const char *) - 確認対象のファイルパス。
  • **出力**: bool - エントリが存在する場合は true、存在しない場合は false を返す。

前提条件と不変条件

  • グローバル変数または静的変数 keepwal が初期化済みであること。
  • keepwal_lookup 関数が利用可能であること。

アルゴリズムまたは振る舞い

1. keepwal ハッシュテーブルに対して keepwal_lookup を呼び出し、指定された path に対応するエントリを検索する。 2. 検索結果が NULL でなければ true を、NULL であれば false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/filemap.c:754 にて呼び出しを確認。
  • **呼び出し先**: keepwal_lookup を呼び出す。

関連データ構造

  • keepwal: ファイルパスをキーとして保持するハッシュテーブル（filemap.c 内で管理）。
  • keepwal_entry: 保持対象の WAL ファイル情報を格納する構造体。

失敗モードと境界条件

  • keepwal が初期化されていない状態で呼び出された場合、未定義の動作を引き起こす可能性がある。
  • path が NULL の場合の挙動は keepwal_lookup の実装に依存する。

保守担当者向けメモ

  • 本関数は keepwal 関連の検索インターフェースをラップする簡易的なヘルパーである。
  • keepwal の構造や検索ロジックに変更があった場合は、本関数の戻り値の整合性を確認すること。
