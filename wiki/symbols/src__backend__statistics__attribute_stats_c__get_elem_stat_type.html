get_elem_stat_type

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/attribute_stats.c:676

目的

指定された属性型に対応する要素型（element type）およびその要素型に対する等価演算子（equality operator）を特定します。

シグネチャまたは形状

static bool get_elem_stat_type(Oid atttypid, char atttyptype, Oid *elemtypid, Oid *elem_eq_opr)

入力と出力

  • **入力**:
  • atttypid: 属性のデータ型OID。
  • atttyptype: 属性の型カテゴリ（pg_type.typtype）。
  • elemtypid: 要素型のOIDを格納する出力ポインタ。
  • elem_eq_opr: 等価演算子のOIDを格納する出力ポインタ。
  • **出力**:
  • 成功時に true、失敗時に false を返します。成功時はポインタ引数に結果が格納されます。

前提条件と不変条件

  • atttypid が TSVECTOROID の場合、要素型は強制的に TEXTOID とみなされます。
  • それ以外の場合は get_base_element_type を通じて基底要素型を解決します。
  • 解決された要素型に対して lookup_type_cache を実行し、TYPECACHE_EQ_OPR フラグを用いて等価演算子をキャッシュから取得します。

アルゴリズムまたは振る舞い

1. atttypid が TSVECTOROID か判定し、該当すれば elemtypid を TEXTOID に設定します。 2. それ以外の場合、get_base_element_type(atttypid) を呼び出し、ドメイン型を考慮した基底要素型を取得します。 3. elemtypid が無効なOIDであれば false を返して終了します。 4. lookup_type_cache を使用して、要素型に対応する等価演算子（eq_opr）を検索します。 5. 演算子が見つからない場合は false を返し、見つかった場合は elem_eq_opr に値を格納して true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/attribute_stats.c 内の統計情報更新処理（行109, 299付近）。
  • **呼び出し先**:
  • get_base_element_type
  • lookup_type_cache
  • OidIsValid（マクロ）

関連データ構造

  • TypeCacheEntry: 型キャッシュ情報。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • get_base_element_type が有効な要素型を返さない場合、false を返します。
  • 要素型に対応する等価演算子がシステムカタログ上に存在しない（lookup_type_cache で取得できない）場合、false を返します。

保守担当者向けメモ

  • 本関数は、統計情報の計算において配列や tsvector などのコンテナ型を扱う際に、その内部要素の比較方法を決定するために使用されます。
  • TSVECTOROID の扱いは compute_tsvector_stats() のロジックと整合させる必要があります。
