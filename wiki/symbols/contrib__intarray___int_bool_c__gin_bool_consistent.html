gin_bool_consistent

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_bool.c:333

目的

GIN インデックスを用いた整数配列のクエリ評価において、ブール演算条件が一致するかを判定する。

シグネチャまたは形状

bool gin_bool_consistent(QUERYTYPE *query, bool *check)

入力と出力

  • **入力**:
  • query: 評価対象の QUERYTYPE 構造体（ポーランド記法で格納されたクエリ）。
  • check: GIN インデックスの各キーに対する一致状況を示すブール配列。
  • **出力**:
  • bool: クエリ条件が満たされる場合は true、それ以外は false。

前提条件と不変条件

  • query->size が 0 以下の場合は常に false を返す。
  • check 配列は、ginint4_queryextract で抽出されたクエリのキー順序と一致している必要がある。

アルゴリズムまたは振る舞い

1. GinChkVal 構造体を初期化し、クエリ内の VAL 型ノードに対して check 配列の値をマッピングする。 2. execute 関数を呼び出し、クエリツリーを末尾から走査する。 3. 走査の際、checkcondition_gin コールバックを使用して各ノードの真偽を再帰的に評価する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/intarray/_int_gin.c:171
  • **呼び出し先**: execute（_int_bool.c 内）、checkcondition_gin（_int_bool.c 内）

関連データ構造

  • QUERYTYPE: クエリを保持する構造体。
  • ITEM: クエリツリーの各ノード（値または演算子）。
  • GinChkVal: GIN インデックス評価用のコンテキスト構造体。

失敗モードと境界条件

  • query->size <= 0 の場合、処理を中断し false を返す。
  • メモリ不足時には palloc がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は GIN インデックスの整合性チェック用であり、_int_gin.c と密接に連携している。
  • checkcondition_gin との整合性を維持するため、クエリ抽出ロジックに変更を加える際は注意が必要である。
