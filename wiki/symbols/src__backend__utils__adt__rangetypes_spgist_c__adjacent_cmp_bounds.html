adjacent_cmp_bounds

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_spgist.c:784

目的

SP-GiSTインデックスにおいて、範囲型の隣接関係（adjacency）を検索する際、現在のノード（重心）に対して探索対象の範囲が左側にあるか右側にあるかを判定し、探索方向を決定する。

シグネチャまたは形状

int adjacent_cmp_bounds(TypeCacheEntry *typcache, const RangeBound *arg, const RangeBound *centroid)

入力と出力

  • **入力**:
  • typcache: 型キャッシュ情報。
  • arg: 検索対象の範囲境界。
  • centroid: 現在のノードの重心となる範囲境界。
  • **出力**:
  • 1: 探索対象が重心の右側にあることを示す。
  • -1: 探索対象が重心の左側にあることを示す。

前提条件と不変条件

  • arg->lower と centroid->lower は一致してはならない（Assertにより検証）。

アルゴリズムまたは振る舞い

range_cmp_bounds を用いて arg と centroid を比較し、その結果と bounds_adjacent による隣接判定を組み合わせて探索方向を決定する。

1. **重心が下限（lower）の場合**:

  • 検索対象は隣接する上限境界。
  • arg が centroid より小さいか等しい場合、左側を探索（-1）。それ以外は右側（1）。

    • 2. **重心が上限（upper）の場合**:

  • 検索対象は隣接する下限境界。
  • arg が centroid より小さく、かつ隣接していない場合に左側を探索（-1）。それ以外は右側（1）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/rangetypes_spgist.c 内の adjacent_inner_consistent 関数から呼び出される。
  • **呼び出し先**:
  • range_cmp_bounds: 境界値の比較を実行。
  • bounds_adjacent: 境界同士が隣接しているかを確認。

関連データ構造

  • RangeBound: 範囲の境界（下限または上限）を保持する構造体。
  • TypeCacheEntry: データ型のキャッシュ情報。

失敗モードと境界条件

  • arg->lower == centroid->lower の場合、アサーション違反によりプロセスが停止する可能性がある。
  • 比較ロジックは bounds_adjacent の結果に依存するため、型固有の隣接定義が正しく実装されている必要がある。

保守担当者向けメモ

  • 本関数は adjacent_inner_consistent における枝刈りロジックの核となる。
  • 検索方向の判定ロジックは、重心が下限か上限かで分岐する。コメント内の表（ARGUMENT/CENTROIDの比較表）が、各ケースの探索方向を決定する際の論理的根拠となっているため、修正時はこの表を参照すること。
