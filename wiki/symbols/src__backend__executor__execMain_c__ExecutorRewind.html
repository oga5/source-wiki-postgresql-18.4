ExecutorRewind

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execMain.c:535

目的

クエリ実行状態を初期状態に巻き戻し、プランツリーの再スキャン（Rescan）を可能にする。

シグネチャまたは形状

void ExecutorRewind(QueryDesc *queryDesc)

入力と出力

  • **入力**: QueryDesc *queryDesc（実行対象のクエリ記述子）
  • **出力**: なし（副作用としてプランの状態をリセットする）

前提条件と不変条件

  • queryDesc は NULL であってはならない。
  • queryDesc->estate は NULL であってはならない。
  • クエリの操作種別（queryDesc->operation）は CMD_SELECT であることが期待される（更新を伴うクエリの再スキャンは非推奨）。

アルゴリズムまたは振る舞い

1. queryDesc および estate の妥当性を検証する。 2. estate->es_query_cxt（クエリごとのメモリコンテキスト）へ切り替える。 3. ExecReScan(queryDesc->planstate) を呼び出し、プランツリー全体を再スキャン可能な状態にリセットする。 4. 元のメモリコンテキストへ復帰する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/portalcmds.c:400
  • src/backend/tcop/pquery.c:1702
  • **呼び出し先**:
  • ExecReScan（プランツリーの再スキャンを実行）
  • MemoryContextSwitchTo（メモリコンテキストの切り替え）

関連データ構造

  • QueryDesc: クエリ実行に必要な情報（プラン、実行状態、スナップショット等）を保持する構造体。
  • EState: クエリ実行中の状態を管理する構造体。
  • PlanState: プランツリーの実行状態を保持するノード。

失敗モードと境界条件

  • queryDesc や estate が NULL の場合、Assert により異常終了する。
  • CMD_SELECT 以外の操作に対して呼び出した場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数は主にポータル（Portal）の再実行やカーソル操作に関連して呼び出されます。
  • ExecReScan はプランノードごとの再スキャン処理をトリガーするため、新しいプランノードを追加した場合は、そのノードが ExecReScan に正しく対応しているか確認してください。
