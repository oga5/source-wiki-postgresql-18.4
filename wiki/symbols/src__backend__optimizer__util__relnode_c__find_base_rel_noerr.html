find_base_rel_noerr

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/relnode.c:435

目的

PlannerInfo が保持するベースリレーションの配列から、指定されたリレーションID（relid）に対応する RelOptInfo を安全に取得する。

シグネチャまたは形状

RelOptInfo *find_base_rel_noerr(PlannerInfo *root, int relid)

入力と出力

  • **入力**:
  • root: プランナの情報を保持する PlannerInfo 構造体へのポインタ。
  • relid: 検索対象のリレーションID（整数）。
  • **出力**:
  • 指定された relid が有効な範囲内であれば、対応する RelOptInfo 構造体へのポインタを返す。
  • 範囲外の場合は NULL を返す。

前提条件と不変条件

  • root->simple_rel_array が初期化されている必要がある。
  • relid は uint32 にキャストして比較されるため、負の値を渡しても配列外アクセスは発生しない。

アルゴリズムまたは振る舞い

1. relid を uint32 にキャストし、root->simple_rel_array_size と比較する。 2. relid が配列サイズ未満であれば、root->simple_rel_array[relid] を返す。 3. relid が範囲外であれば、エラーを発生させずに NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/selfuncs.c:5978
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行わない。

関連データ構造

  • PlannerInfo: プランナのグローバル状態を保持する構造体。simple_rel_array を介してベースリレーションにアクセスする。
  • RelOptInfo: プランナが扱うリレーションの基本構造体。

失敗モードと境界条件

  • relid が負の値、または simple_rel_array_size 以上である場合、関数は安全に NULL を返す。
  • root が NULL の場合、ポインタ参照によりクラッシュする可能性がある（呼び出し元で保証する必要がある）。

保守担当者向けメモ

  • 本関数はエラーを報告せずに NULL を返すため、呼び出し側では戻り値が NULL である可能性を適切に処理すること。
  • find_base_rel と異なり、リレーションが見つからないことが正常なケース（例：特定の条件での検索）で使用される。
