apply_child_basequals

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/inherit.c:841

目的

親テーブルの制約（baserestrictinfo）を子テーブルの属性に変換・適用し、定数評価を通じて不要な子テーブルを排除（プルーニング）する。

シグネチャまたは形状

bool apply_child_basequals(PlannerInfo *root, RelOptInfo *parentrel,
                           RelOptInfo *childrel, RangeTblEntry *childRTE,
                           AppendRelInfo *appinfo)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • parentrel: 親テーブルの RelOptInfo。
  • childrel: 変換対象となる子テーブルの RelOptInfo。
  • childRTE: 子テーブルの範囲テーブルエントリ。
  • appinfo: 親から子への属性変換情報。
  • **出力**:
  • bool: 子テーブルが制約により完全に排除される場合は false、それ以外は true。

前提条件と不変条件

  • parentrel->baserestrictinfo に含まれる各制約は、adjust_appendrel_attrs を介して子テーブルのコンテキストに変換される。
  • 変換後の制約が定数 FALSE または NULL に評価された場合、その子テーブルはクエリ対象から除外される。

アルゴリズムまたは振る舞い

1. 親の baserestrictinfo を順次走査し、adjust_appendrel_attrs で子テーブルの属性に変換する。 2. eval_const_expressions を適用し、定数評価を行う。

  • 定数 FALSE または NULL になった場合、即座に false を返し、子テーブルを排除する。
  • 定数 TRUE になった場合、その制約は無視する。

    • 3. 残りの制約を make_restrictinfo で再構成し、childrel->baserestrictinfo に格納する。 4. childRTE->securityQuals が存在する場合、それらを子テーブルの制約として追加する。 5. 最終的な子テーブルの制約リストと、最小セキュリティレベルを childrel に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/relnode.c 内の処理から呼び出されることが確認されている。
  • **呼び出し先**: adjust_appendrel_attrs, eval_const_expressions, make_restrictinfo, restriction_is_always_false, restriction_is_always_true 等の最適化ユーティリティ関数。

関連データ構造

  • PlannerInfo: プランニング全体のコンテキスト。
  • RelOptInfo: リレーションの最適化情報。
  • AppendRelInfo: 継承関係の変換情報。
  • RestrictInfo: 制約条件のメタデータ。

失敗モードと境界条件

  • 制約が定数 FALSE に評価された場合、関数は false を返し、呼び出し元は当該子テーブルをプランニング対象から除外する。
  • securityQuals が存在する場合、それらは制約として追加されるが、親の制約とは異なり定数評価による排除は行われない。

保守担当者向けメモ

  • 本関数は、継承階層の展開において「不要な子テーブルの排除」という重要な最適化を担う。
  • securityQuals の扱いは、expand_single_inheritance_child で生成される通常の継承テーブルと、UNION ALL 由来の appendrel で挙動が異なる点に注意すること。
