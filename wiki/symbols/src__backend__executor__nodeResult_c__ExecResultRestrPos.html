ExecResultRestrPos

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeResult.c:160

目的

Result ノードにおいて、子プランの実行位置を以前にマークされた位置へ復元（Restore）する処理を仲介します。

シグネチャまたは形状

void ExecResultRestrPos(ResultState *node)

入力と出力

  • **入力**: ResultState *node （Result ノードの実行時状態を保持する構造体へのポインタ）
  • **出力**: なし（副作用として子プランの状態を更新）

前提条件と不変条件

  • Result ノード自体は Mark/Restore 操作をサポートしていません。
  • 子プランが存在する場合のみ、その子プランに対して ExecRestrPos を委譲します。

アルゴリズムまたは振る舞い

1. outerPlanState(node) を介して、現在の Result ノードの子プラン状態を取得します。 2. 子プランが存在する場合、ExecRestrPos(outerPlan) を呼び出し、子プラン側の復元処理を実行します。 3. 子プランが存在しない場合、elog(ERROR, ...) を発行し、エラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:401 から呼び出されていることが確認されています。
  • **呼び出し先**: ExecRestrPos（子プランの状態に対して呼び出される）。

関連データ構造

  • ResultState: Result ノードの実行状態を管理する構造体。
  • PlanState: 子プランの実行状態を管理する構造体。

失敗モードと境界条件

  • **エラー**: 子プランが存在しない状態で本関数が呼び出された場合、Result ノードは Mark/Restore をサポートしていないため、ERROR レベルのログを出力して処理を中断します。

保守担当者向けメモ

  • 本関数は Result ノードの透過的な動作を保証するためのラッパーです。Result ノード自体がデータソースを持たない場合、この操作は無効であるという設計上の制約を反映しています。
