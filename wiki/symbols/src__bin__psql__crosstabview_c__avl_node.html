avl_node

定義

  • 種別: struct
  • 定義箇所: src/bin/psql/crosstabview.c:50

目的

\crosstabview コマンドにおけるクエリ結果の重複排除とソート済み配列生成のための、平衡二分探索木（AVL木）のノードを表現します。

シグネチャまたは形状

typedef struct _avl_node
{
    pivot_field field;
    int         height;
    struct _avl_node *children[2];
} avl_node;

入力と出力

  • **入力**: pivot_field 型のデータ（ヘッダー値、ソート値、ランク）を保持します。
  • **出力**: children[0]（左部分木）および children[1]（右部分木）へのポインタを保持します。

前提条件と不変条件

  • children 配列は決して NULL になりません。子ノードが存在しない場合は、avl_tree 構造体が保持する空ノード（end）を指す必要があります。
  • height は、そのノードから葉ノードまでの最長パス上のノード数として維持されます。

アルゴリズムまたは振る舞い

  • 本構造体は、avlInsertNode による挿入操作を通じて木を構成します。
  • avlRotate および avlBalance 関数により、挿入後の平衡状態が維持されます。
  • avlUpdateHeight により、ノードの高さが再計算されます。

呼び出し元と呼び出し先

  • **呼び出し元**: avl_tree の初期化時（avl_tree.end の割り当て）、avlInsertNode、avlRotate、avlFree 等の木操作関数から参照されます。
  • **呼び出し先**: pivot_field 型を保持し、子ノードとして自身（avl_node）を再帰的に参照します。

関連データ構造

  • avl_tree: 本ノードを管理する木構造のルートおよび終端ノードを保持します。
  • pivot_field: 各ノードが格納する実データ構造です。

失敗モードと境界条件

  • CROSSTABVIEW_MAX_COLUMNS（1600）を超えるノードが生成される場合、システム全体としてエラーを返します。
  • 削除操作はサポートされていません。

保守担当者向けメモ

  • 本実装は挿入専用の最小限の AVL 木です。検索や削除機能は含まれていません。
  • children が NULL になることはないため、ポインタのデリファレンス時には avl_tree.end との比較を前提としてください。
