PgAioBackend

定義

  • 種別: struct
  • 定義箇所: src/include/storage/aio_internal.h:191

目的

PostgreSQL の非同期 I/O (AIO) サブシステムにおいて、各バックエンドプロセスごとの I/O 状態、ステージング領域、および I/O キューを管理する制御構造体です。

シグネチャまたは形状

typedef struct PgAioBackend
{
    uint32      io_handle_off;
    dclist_head idle_ios;
    PgAioHandle *handed_out_io;
    bool        in_batchmode;
    uint16      num_staged_ios;
    PgAioHandle *staged_ios[PGAIO_SUBMIT_BATCH_SIZE];
    dclist_head in_flight_ios;
} PgAioBackend;

入力と出力

  • **入力**: pgaio_io_acquire() 等により取得された PgAioHandle が handed_out_io に保持されます。
  • **出力**: staged_ios 配列および in_flight_ios リストを通じて、実行待ちまたは実行中の I/O 操作を管理します。

前提条件と不変条件

  • handed_out_io は、定義（I/O への関連付け）または解放（pgaio_io_release）が行われるまで、一度に一つしか保持できません。
  • staged_ios は PGAIO_SUBMIT_BATCH_SIZE（32）を上限とします。
  • in_flight_ios は、提出時刻順にソートされたリンクドリストとして維持されます。

アルゴリズムまたは振る舞い

1. **取得**: pgaio_io_acquire() を呼び出し、handed_out_io を介してハンドルを確保します。 2. **ステージング**: pgaio_io_stage() によりハンドルを staged_ios に格納し、num_staged_ios を更新します。 3. **提出**: in_batchmode が有効な場合、バッチ処理を経て in_flight_ios へ移行します。 4. **完了と回収**: 他のバックエンドによって完了された I/O も in_flight_ios に含まれる可能性があり、本バックエンドがそれを検知した際に回収（reclaim）を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/aio/aio.c および src/backend/storage/aio/aio_init.c で参照・初期化されます。
  • **呼び出し先**: PgAioHandle 構造体および dclist（リンクドリスト）操作関数群を利用します。

関連データ構造

  • PgAioHandle: 個別の I/O 操作を追跡する構造体。
  • PgAioCtl: システム全体で共有される AIO 制御構造体（backend_state 配列として保持）。
  • dclist_head: lib/ilist.h で定義される双方向リスト。

失敗モードと境界条件

  • staged_ios が PGAIO_SUBMIT_BATCH_SIZE を超える場合、バッチ処理の制限に抵触します。
  • handed_out_io が適切に解放されない場合、後続の pgaio_io_acquire() が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は PgAioCtl の一部としてプロセスごとに割り当てられます。
  • in_flight_ios のリスト操作は、I/O 完了時の競合を避けるため、適切なロックまたはアトミックな状態遷移が必要です。
