gbt_boolgt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bool.c:26

目的

GiSTインデックスにおける boolean 型の「より大きい（greater than）」比較演算を提供します。

シグネチャまたは形状

static bool gbt_boolgt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の bool 型値へのポインタ。
  • flinfo: 関数マネージャ情報（本関数内では未使用）。
  • **出力**:
  • bool: *a > *b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な bool 型の値を指している必要があります。
  • PostgreSQLの bool 型の内部表現（0または1）に基づいた比較が行われます。

アルゴリズムまたは振る舞い

1. 引数 a と b を const bool * にキャストします。 2. それぞれの指す値をデリファレンスし、C言語の > 演算子を用いて比較します。 3. 比較結果をそのまま呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_bool.c 内の静的テーブル定義（gbt_bool_consistent 等の内部ロジックで使用される比較関数群の一部として参照）。
  • **呼び出し先**: 直接的な関数呼び出しはありません。

関連データ構造

  • bool: PostgreSQLの標準ブール型。
  • FmgrInfo: PostgreSQLの関数呼び出し情報構造体。

失敗モードと境界条件

  • a または b が NULL ポインタである場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.c が提供する数値系GiSTユーティリティから呼び出される比較関数の一つです。
  • bool 型は実質的に 0/1 の数値として扱われるため、この単純な比較ロジックで機能します。
