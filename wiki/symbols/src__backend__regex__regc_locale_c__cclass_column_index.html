cclass_column_index

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_locale.c:670

目的

指定された文字 c がどのロケール依存の文字クラスに属するかを判定し、対応するビットマスク（カラーマップの列インデックス）を返します。

シグネチャまたは形状

static int cclass_column_index(struct colormap *cm, chr c)

入力と出力

  • **入力**:
  • cm: カラーマップ構造体へのポインタ。
  • c: 判定対象の文字（chr 型）。
  • **出力**:
  • int: 判定された文字クラスに対応するビットマスクの論理和。

前提条件と不変条件

  • c > MAX_SIMPLE_CHR であること（単純な文字は本関数を通さない前提）。
  • cm->classbits の各要素が、pg_wc_* 系関数による判定結果と整合していること。
  • CC_ASCII, CC_BLANK, CC_CNTRL, CC_XDIGIT に対応するビットは常に0であること（アサーションで検証済み）。

アルゴリズムまたは振る舞い

1. 初期値 colnum を 0 に設定します。 2. cm->classbits にビットが設定されている文字クラスについて、pg_wc_* 関数（pg_wc_isprint, pg_wc_isalnum 等）を用いて文字 c が該当するかを順次確認します。 3. 該当する場合、対応するビットマスクを colnum にビット単位の OR で加算します。 4. 最終的な colnum を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_color.c:158
  • **呼び出し先**:
  • pg_wc_isprint, pg_wc_isalnum, pg_wc_isalpha, pg_wc_isword, pg_wc_isdigit, pg_wc_ispunct, pg_wc_isspace, pg_wc_islower, pg_wc_isupper, pg_wc_isgraph（すべてPostgreSQLのロケール依存文字判定関数）。

関連データ構造

  • struct colormap: 正規表現エンジンが使用する文字分類マップ。
  • chr: 正規表現エンジン内で使用される文字型。

失敗モードと境界条件

  • MAX_SIMPLE_CHR 以下の文字が渡された場合、アサーション違反が発生します。
  • ロケール設定が正しくない場合、pg_wc_* 関数群の戻り値が期待通りにならない可能性があります。

保守担当者向けメモ

  • 本関数は src/backend/regex/regc_locale.c 内の他のロケール処理ロジックと同期を保つ必要があります。特に regc_locale.c:591 付近のコードと整合性を維持してください。
  • regguts.h の enum char_classes に変更を加える際は、本関数の if 文の構成も併せて確認してください。
