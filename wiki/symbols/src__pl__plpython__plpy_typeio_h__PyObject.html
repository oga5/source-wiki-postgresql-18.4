PyObject

定義

  • 種別: typedef
  • 定義箇所: src/pl/plpython/plpy_typeio.h:89

目的

Python C API におけるすべての Python オブジェクトを表現する基底型です。PL/Python においては、PostgreSQL のデータ型（Datum）と Python インタプリタ間でやり取りされるデータのコンテナとして機能します。

シグネチャまたは形状

Python C API の定義に準拠する不透明な構造体です。

入力と出力

  • **入力**: Python インタプリタから生成されるオブジェクト、または PLy_input_convert を介して PostgreSQL から変換されたデータ。
  • **出力**: PLy_output_convert を介して PostgreSQL の Datum へ変換される対象、または Python インタプリタへ渡される戻り値。

前提条件と不変条件

  • Python C API のメモリ管理規則（参照カウント）に従う必要があります。
  • PLy_output_convert に渡される際、Py_None はドメイン制約チェックのために変換関数へ渡されることが許可されています。

アルゴリズムまたは振る舞い

PL/Python 内では、PLyDatumToOb や PLyObToDatum 構造体に含まれる変換関数を通じて、PyObject への変換および PyObject からの抽出が行われます。変換処理は再帰的に実行されることがあり、その過程で MemoryContext を用いたメモリ管理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • contrib/hstore_plpython/hstore_plpython.c: dict、key、value 等の型として使用。
  • contrib/jsonb_plpython/jsonb_plpython.c: decimal_constructor や PLyObject_FromJsonbContainer の戻り値として使用。
  • **呼び出し先**:
  • Python C API の各種関数（参照カウント操作や型変換関数など）。

関連データ構造

  • PLyDatumToOb: PostgreSQL から Python への変換状態を保持。
  • PLyObToDatum: Python から PostgreSQL への変換状態を保持。
  • Datum: PostgreSQL 側のデータ表現。

失敗モードと境界条件

  • Python インタプリタが例外を発生させた場合、PyObject を介した変換処理は中断され、PLy_elog を通じて PostgreSQL のエラー報告システムへ伝播します。
  • メモリ不足時には Python の例外メカニズムにより処理が停止します。

保守担当者向けメモ

  • 本シンボルは Python C API の一部であり、PL/Python 内部では plpython.h を通じて利用されます。
  • 変換関数を実装する際は、メモリリークを防ぐため、PyObject の参照カウント管理（Py_INCREF / Py_DECREF）を適切に行う必要があります。
