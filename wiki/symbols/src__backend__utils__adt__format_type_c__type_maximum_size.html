type_maximum_size

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/format_type.c:411

目的

指定されたデータ型（OID）と型修飾子（typmod）に基づき、可変長カラムが取り得る最大サイズ（バイト単位）を算出します。

シグネチャまたは形状

int32 type_maximum_size(Oid type_oid, int32 typemod)

入力と出力

  • **入力**:
  • type_oid: 対象となるデータ型のOID。
  • typemod: 型修飾子（例: varchar(n) の n）。
  • **出力**:
  • 最大サイズをバイト数で返します。不明な型やサイズ制限のない型の場合は -1 を返します。

前提条件と不変条件

  • typemod が負の値である場合、サイズ制限がないものと見なされ -1 を返します。
  • BPCHAROID および VARCHAROID の計算では、typemod に含まれる varlena ヘッダーサイズを考慮し、データベースのエンコーディングに基づいた最大文字バイト数を乗算します。

アルゴリズムまたは振る舞い

1. typemod < 0 の場合、直ちに -1 を返します。 2. type_oid に基づき以下の処理を行います：

  • BPCHAROID, VARCHAROID: (typemod - VARHDRSZ) * pg_encoding_max_length(GetDatabaseEncoding()) + VARHDRSZ を計算します。
  • NUMERICOID: numeric_maximum_size(typemod) を呼び出します。
  • VARBITOID, BITOID: ビット数からバイト数を算出し、2 * sizeof(int32)（ヘッダーおよび長さ情報）を加算します。

    • 3. 上記以外の型については -1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c:2064
  • src/backend/utils/cache/lsyscache.c:2771
  • **呼び出し先**:
  • numeric_maximum_size (NUMERIC型の場合)
  • pg_encoding_max_length (文字型の場合)
  • GetDatabaseEncoding (文字型の場合)

関連データ構造

  • Oid: PostgreSQLのシステムカタログにおけるオブジェクト識別子。
  • int32: PostgreSQLの標準的な32ビット整数型。

失敗モードと境界条件

  • 未知の型や、最大サイズが定義されていない型（text型など）が渡された場合、関数は -1 を返します。
  • typemod が不正な値である場合の挙動は、各型の計算ロジックに依存します。

保守担当者向けメモ

  • 本関数は特定の型定義をハードコードしているため、新しい型を追加したり既存の型の内部表現を変更したりする際は、本関数のロジックも併せて更新する必要があります。
  • typemod の解釈は型ごとに異なるため、修正時は src/backend/utils/cache/lsyscache.c 等の関連箇所との整合性に注意してください。
