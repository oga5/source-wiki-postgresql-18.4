CreateComments

定義

  • 種別: function
  • 定義箇所: src/backend/commands/comment.c:142

目的

データベースオブジェクトに対するコメントの挿入、更新、または削除を行う。

シグネチャまたは形状

void CreateComments(Oid oid, Oid classoid, int32 subid, const char *comment)

入力と出力

  • oid: 対象オブジェクトのOID。
  • classoid: 対象オブジェクトが属するカタログのOID。
  • subid: サブオブジェクトID（該当しない場合は0）。
  • comment: 設定するコメント文字列。NULL または空文字列が渡された場合は、既存のコメントを削除する。
  • 出力: なし（副作用として pg_description カタログを更新）。

前提条件と不変条件

  • pg_description テーブルに対する RowExclusiveLock を取得する。
  • 呼び出し元で適切な権限チェック（所有権の確認等）が完了していること。
  • 同一オブジェクトに対するコメントは、カタログ内に最大で1つのみ存在することを前提とする。

アルゴリズムまたは振る舞い

1. **前処理**: comment が空文字列の場合、NULL に変換して削除処理として扱う。 2. **検索**: DescriptionObjIndexId インデックスを使用し、指定された oid, classoid, subid に一致する既存レコードを検索する。 3. **更新/削除**: 既存レコードが見つかった場合、comment が NULL なら CatalogTupleDelete を実行し、そうでなければ CatalogTupleUpdate で内容を更新する。 4. **挿入**: 既存レコードが見つからず、かつ comment が NULL でない場合、CatalogTupleInsert を用いて新規レコードを作成する。 5. **終了処理**: 使用したスキャンを終了し、テーブルをクローズする。

呼び出し元と呼び出し先

  • 呼び出し元: CommentObject (同ファイル内), collationcmds.c, extension.c, indexcmds.c, statscmds.c 等から呼び出される。
  • 呼び出し先: table_open, systable_beginscan, systable_getnext, CatalogTupleUpdate, CatalogTupleDelete, CatalogTupleInsert, heap_modify_tuple, heap_form_tuple, heap_freetuple, table_close 等のカタログアクセスAPI。

関連データ構造

  • pg_description: コメントを保持するシステムカタログ。
  • ScanKeyData: インデックススキャン用の検索キー。
  • HeapTuple: カタログ操作対象のタプル。

失敗モードと境界条件

  • 対象オブジェクトがカタログ内に存在しない場合、何も挿入・更新されず終了する。
  • 複数のレコードが一致する状況は想定されていない（break により最初の1件のみ処理）。

保守担当者向けメモ

  • 本関数は通常オブジェクト用である。共有オブジェクト（データベース、ロール等）には CreateSharedComments を使用すること。
  • subid が 0 の場合はオブジェクト全体に対するコメントを指す。
