DWORD

定義

  • 種別: struct（Windows API の型定義に基づく）
  • 定義箇所: src/port/win32error.c:20

目的

Windows API が返す 32 ビット符号なし整数型のエラーコードを保持するために使用されます。src/port/win32error.c においては、Windows エラーコードと POSIX errno を対応付けるためのマッピング構造体の一部として定義されています。

シグネチャまたは形状

static const struct
{
    DWORD       winerr;
    int         doserr;
} doserrors[] = ...

入力と出力

  • 入力: Windows API から取得したエラーコード（unsigned long）。
  • 出力: マッピングされた POSIX 準拠の errno 値。

前提条件と不変条件

  • 本シンボルは Windows 環境でのみ有効です。
  • doserrors 配列内において、winerr は Windows のエラーコード体系に従う必要があります。

アルゴリズムまたは振る舞い

_dosmaperr 関数が呼び出された際、この構造体内の winerr が入力値と比較されます。一致した場合、対応する doserr がグローバルな errno に設定されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/win32error.c 内の _dosmaperr 関数がこの構造体を参照します。
  • 呼び出し先: 構造体自体はデータ保持用であり、関数呼び出しは行いません。
  • 注記: DWORD 型自体は Windows SDK の型であり、PostgreSQL の各所（src/backend/port/win32/signal.c や src/backend/libpq/auth.c など）で広範に使用されていますが、本ファイルにおける DWORD はマッピングテーブルの定義要素として機能しています。

関連データ構造

  • doserrors: DWORD 型の winerr と int 型の doserr をペアで保持する静的構造体配列。

失敗モードと境界条件

  • 未定義のエラーコードが渡された場合、マッピングテーブルの探索が失敗し、デフォルト値として EINVAL が設定されます。

保守担当者向けメモ

  • 本ファイルは Windows 固有の処理を扱うため、プラットフォーム非依存のコードと混同しないよう注意してください。
  • マッピングテーブルは網羅的ではないため、新しいエラーコードの追加が必要な場合は doserrors 配列を更新してください。
