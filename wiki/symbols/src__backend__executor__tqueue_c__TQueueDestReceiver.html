TQueueDestReceiver

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/tqueue.c:30

目的

並列クエリ実行において、実行結果のタプルを共有メモリキュー（shm_mq）へ転送するための送信側インターフェースを提供します。

シグネチャまたは形状

typedef struct TQueueDestReceiver
{
    DestReceiver pub;           /* public fields */
    shm_mq_handle *queue;       /* shm_mq to send to */
} TQueueDestReceiver;

入力と出力

  • **入力**: DestReceiver インターフェースを介して受け取ったタプルを、内部の shm_mq_handle を通じて共有メモリへ書き出します。
  • **出力**: 共有メモリキューへのシリアライズされたタプルデータ。

前提条件と不変条件

  • pub フィールドは DestReceiver として初期化されている必要があります。
  • queue は有効な shm_mq_handle を指している必要があります。

アルゴリズムまたは振る舞い

1. DestReceiver の実装として、実行エンジンからタプルを受け取ります。 2. 保持している shm_mq_handle を使用し、shm_mq_send を呼び出してタプルを共有メモリへ書き込みます。 3. タプルのシリアライズには MinimalTuple 形式が使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: CreateTupleQueueDestReceiver によりインスタンス化されます。また、tqueueReceiveSlot 等の DestReceiver コールバック関数内でキャストして利用されます。
  • **呼び出し先**: shm_mq_send（共有メモリへの書き込み）。

関連データ構造

  • DestReceiver: 本構造体が継承する基底インターフェース。
  • shm_mq_handle: 共有メモリキューへの接続状態を保持する構造体。
  • TupleQueueReader: 対となる受信側の構造体。

失敗モードと境界条件

  • 共有メモリキューが満杯の場合、shm_mq_send の振る舞いに依存した待機またはエラーが発生する可能性があります。
  • 接続が切断された場合、書き込み操作は失敗します。

保守担当者向けメモ

  • 本構造体は shm_mq のラッパーです。タプルのシリアライズ形式は MinimalTuple に依存しているため、関連する変更時は access/htup_details.h の仕様を確認してください。
  • palloc0 でメモリ確保されるため、ライフサイクル終了時には適切に解放する必要があります。
