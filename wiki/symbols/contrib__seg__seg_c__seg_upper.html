seg_upper

定義

  • 種別: function
  • 定義箇所: contrib/seg/seg.c:180

目的

SEG 構造体から区間の上限値（upper bound）を取得し、PostgreSQL の float4 型として返します。

シグネチャまたは形状

Datum seg_upper(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される SEG 型のポインタ。
  • **出力**: PG_RETURN_FLOAT4 を介して返される float4 型の数値。

前提条件と不変条件

  • 入力された SEG ポインタが有効なメモリ領域を指していること。
  • SEG 構造体内の upper メンバが初期化済みであること。

アルゴリズムまたは振る舞い

1. PG_GETARG_SEG_P(0) マクロを使用して、引数から SEG 構造体へのポインタを取得します。 2. 構造体メンバ seg->upper にアクセスします。 3. 取得した値を PG_RETURN_FLOAT4 マクロでラップして返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1(seg_upper) により登録）。具体的な SQL クエリからの呼び出しについては、提供されたソースからは直接確認できません。
  • **呼び出し先**: なし（標準的なマクロのみを使用）。

関連データ構造

  • SEG: segdata.h で定義される、区間の境界値や有効桁数を保持する構造体。

失敗モードと境界条件

  • 入力引数が NULL の場合、PG_GETARG_SEG_P の挙動に依存します。
  • seg ポインタが無効な場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は単純なアクセサです。SEG 構造体の定義に変更があった場合は、メンバ名や型に注意してください。
