InitCompressFileHandleLZ4

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_lz4.c:781

目的

CompressFileHandle 構造体を初期化し、LZ4 圧縮・解凍ストリームを扱うための関数ポインタとコンテキストを割り当てる。

シグネチャまたは形状

void InitCompressFileHandleLZ4(CompressFileHandle *CFH, const pg_compress_specification compression_spec)

入力と出力

  • **入力**:
  • CFH: 初期化対象の CompressFileHandle 構造体へのポインタ。
  • compression_spec: 圧縮設定（レベル等を含む）。
  • **出力**: なし（CFH の各フィールドおよび private_data が更新される）。

前提条件と不変条件

  • CFH は有効なメモリ領域を指している必要がある。
  • pg_malloc0 を使用して LZ4State を確保するため、メモリ不足時はプロセスが終了する可能性がある。

アルゴリズムまたは振る舞い

1. CFH の各関数ポインタ（open_func, read_func, write_func 等）に、LZ4Stream_ で始まる対応する LZ4 実装関数を割り当てる。 2. CFH->compression_spec を引数の値で更新する。 3. LZ4State 構造体用のメモリをゼロクリアで確保し、CFH->private_data に格納する。 4. compression_spec.level が 0 以上の場合、確保した state->prefs.compressionLevel にその値を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_io.c:205 からの呼び出しが確認されている。
  • **呼び出し先**: pg_malloc0（メモリ確保）、および LZ4Stream_ 系関数群（関数ポインタの代入先）。

関連データ構造

  • CompressFileHandle: 圧縮ストリームの抽象インターフェース。
  • LZ4State: LZ4 圧縮・解凍のコンテキストとバッファを保持する構造体。
  • pg_compress_specification: 圧縮設定を保持する構造体。

失敗モードと境界条件

  • compression_spec.level が負の値の場合、state->prefs.compressionLevel はデフォルト値のままとなる。
  • メモリ確保失敗時は pg_malloc0 の挙動に従い、通常はエラー終了する。

保守担当者向けメモ

  • 本関数は USE_LZ4 が定義されている環境でのみ有効な実装である。
  • LZ4State の初期化は最小限に留められており、実際の LZ4 コンテキスト生成は各ストリーム操作関数内での遅延初期化に依存している。
