spgChooseResultType

定義

  • 種別: enum
  • 定義箇所: src/include/access/spgist.h:67

目的

SP-GiSTインデックスへの値挿入時、現在の内部タプルに対してどのような操作を行うべきかを決定するためのアクションコードを定義します。

シグネチャまたは形状

typedef enum spgChooseResultType
{
    spgMatchNode = 1,
    spgAddNode,
    spgSplitTuple,
} spgChooseResultType;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: spgChooseOut 構造体の resultType フィールドとして使用されます。

前提条件と不変条件

  • SP-GiSTの挿入処理において、演算子クラスが提供する choose サポート関数が返す値として使用されます。

アルゴリズムまたは振る舞い

  • spgMatchNode: 既存のノードへ降下することを指示します。
  • spgAddNode: 内部タプルに新しいノードを追加することを指示します。
  • spgSplitTuple: 内部タプルのプレフィックスを変更し、タプルを分割することを指示します。

呼び出し元と呼び出し先

  • 呼び出し元: SP-GiSTの挿入ロジック（spgChoose 関連処理）。
  • 呼び出し先: 情報なし。

関連データ構造

  • spgChooseOut: この列挙型を保持する出力用構造体。

失敗モードと境界条件

  • 定義された3つの値以外の値が設定された場合の挙動は、呼び出し側の実装に依存します。

保守担当者向けメモ

  • 新しい挿入戦略を追加する場合は、この列挙型に値を追加し、対応する処理を spgChoose の呼び出し元（src/backend/access/spgist/ 配下の実装）に追加する必要があります。
