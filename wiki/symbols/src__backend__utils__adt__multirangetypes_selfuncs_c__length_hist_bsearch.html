length_hist_bsearch

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/multirangetypes_selfuncs.c:767

目的

レンジ長ヒストグラム（length_hist_values）から、指定された値（value）がどのビンに該当するかを二分探索で特定する。

シグネチャまたは形状

static int length_hist_bsearch(Datum *length_hist_values, int length_hist_nvalues, double value, bool equal)

入力と出力

  • **入力**:
  • length_hist_values: ヒストグラムの境界値配列（Datum形式）。
  • length_hist_nvalues: 配列の要素数。
  • value: 検索対象の数値。
  • equal: 比較時に等価条件を含めるかどうかのフラグ。
  • **出力**:
  • 検索対象の値が該当するヒストグラムのインデックス（int）。

前提条件と不変条件

  • length_hist_values は昇順にソートされている必要がある。
  • 戻り値は -1 から length_hist_nvalues - 1 の範囲となる。

アルゴリズムまたは振る舞い

1. lower を -1、upper を length_hist_nvalues - 1 に初期化する。 2. lower < upper である間、以下の処理を繰り返す。

  • 中間インデックス middle を (lower + upper + 1) / 2 で算出する。
  • length_hist_values[middle] を double に変換し、value と比較する。
  • equal が true の場合、middleval <= value を条件として判定する。
  • 条件を満たす場合は lower = middle とし、それ以外の場合は upper = middle - 1 とする。

    • 3. 最終的な lower を返す。

関連データ構造

  • Datum: PostgreSQLの汎用データ表現。ヒストグラム値の保持に使用。

失敗モードと境界条件

  • length_hist_nvalues が 0 の場合、lower は初期値の -1 を返す。
  • 検索値がヒストグラムの全範囲外にある場合、境界値（-1 または最大インデックス）を返す。
