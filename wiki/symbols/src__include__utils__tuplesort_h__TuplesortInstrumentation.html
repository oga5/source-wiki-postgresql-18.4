TuplesortInstrumentation

定義

  • 種別: struct
  • 定義箇所: src/include/utils/tuplesort.h:111

目的

タプルソート処理における実行統計情報を保持するための構造体です。主に EXPLAIN コマンド等でソートの実行状況（使用されたアルゴリズムやメモリ/ディスク消費量）を報告するために使用されます。

シグネチャまたは形状

typedef struct TuplesortInstrumentation
{
	TuplesortMethod sortMethod; /* 使用されたソートアルゴリズム */
	TuplesortSpaceType spaceType;	/* spaceUsed の単位/種類 */
	int64		spaceUsed;		/* 消費量 (単位: kB) */
} TuplesortInstrumentation;

入力と出力

  • **入力**: ソート処理の実行中に、tuplesort.c 内の各処理フェーズで値が更新されます。
  • **出力**: EXPLAIN 実行時に、ソートノードの統計情報として外部へ提供されます。

前提条件と不変条件

  • ポインタを含まない構造である必要があります（src/include/utils/tuplesort.h:66 のコメントによる）。
  • spaceUsed はキロバイト (kB) 単位で記録されます。

アルゴリズムまたは振る舞い

ソート処理の過程で、使用されたソート手法（TuplesortMethod）と、メモリまたは一時ファイル（TuplesortSpaceType）のどちらが使用されたか、およびその消費量（spaceUsed）を記録します。この構造体は、ソート実行ノード（nodeSort.c や nodeIncrementalSort.c）において、並列ワーカー数に応じた配列として確保されることがあります。

呼び出し元と呼び出し先

  • **呼び出し元（参照/操作）**:
  • src/backend/commands/explain.c: 統計情報の出力処理。
  • src/backend/executor/nodeSort.c: 並列ソート時の共有メモリ領域確保および統計管理。
  • src/backend/executor/nodeIncrementalSort.c: インクリメンタルソートの統計管理。
  • **呼び出し先**:
  • src/backend/utils/sort/tuplesort.c: ソート実行中にこの構造体へ統計値を書き込む関数が存在します。

関連データ構造

  • TuplesortMethod: 使用されたソートアルゴリズムの列挙型。
  • TuplesortSpaceType: リソース消費の種類（メモリまたはディスク）を示す列挙型。
  • execnodes.h 内の SortState 等の実行ノード構造体から参照されます。

失敗モードと境界条件

  • 並列ソート時にワーカー数分だけ配列を確保する際、mul_size を用いたサイズ計算が行われており、メモリ確保失敗の可能性があります。
  • 統計情報の更新はソート処理の完了前後に行われるため、ソートが正常に終了しない場合、不完全な統計情報が保持される可能性があります。

保守担当者向けメモ

  • 本構造体はポインタを含まない設計が必須であるため、フィールドを追加する際は注意してください。
  • 並列ソート環境下では、num_workers に基づく配列アクセスが発生するため、インデックスの範囲外アクセスに注意が必要です。
