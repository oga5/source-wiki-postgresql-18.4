get_parallel_tabidx_list

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/reindexdb.c:743

目的

並列実行時に、指定されたインデックスをテーブル単位でグループ化し、再構築順序を最適化するためのリストを構築する。

シグネチャまたは形状

static void get_parallel_tabidx_list(PGconn *conn, SimpleStringList *index_list, SimpleOidList **table_list, bool echo)

入力と出力

  • **入力**:
  • conn: データベース接続ハンドル。
  • index_list: 再構築対象のインデックス名リスト（入力時に消費され、出力時には再構築される）。
  • echo: SQL クエリを表示するかどうかのフラグ。
  • **出力**:
  • table_list: 取得したインデックスに対応するテーブル OID のリスト。
  • index_list: クエリ結果に基づいて再構築・ソートされたインデックス名のリスト。

前提条件と不変条件

  • index_list は NULL であってはならない。
  • データベース接続 conn が有効であること。
  • 同一テーブルに属するインデックスは、並列ジョブ内でまとめて処理される必要がある。

アルゴリズムまたは振る舞い

1. index_list に含まれるインデックスを対象に、pg_index および pg_class カタログからテーブル OID とインデックス名を取得するクエリを構築する。 2. クエリ結果を「テーブルごとの最大インデックスサイズ」および「テーブル OID」、「インデックスサイズ」の順でソートし、同一テーブルのインデックスが連続するように配置する。 3. 元の index_list を破棄し、クエリ結果から得られたデータで再構築する。 4. table_list を初期化し、クエリ結果から得られたテーブル OID を格納する。 5. 最終的に PQclear を呼び出し、クエリ結果を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/scripts/reindexdb.c 内の reindex_one_database 関数から呼び出される。
  • **呼び出し先**: initPQExpBuffer, appendPQExpBufferStr, appendQualifiedRelation, simple_string_list_destroy, executeQuery, termPQExpBuffer, PQntuples, PQclear, pg_malloc0, simple_oid_list_append, simple_string_list_append, fmtQualifiedIdEnc, PQclientEncoding 等。

関連データ構造

  • PGconn: PostgreSQL 接続オブジェクト。
  • SimpleStringList: インデックス名を保持するリスト構造。
  • SimpleOidList: テーブル OID を保持するリスト構造。
  • PQExpBufferData: SQL クエリ構築用バッファ。

失敗モードと境界条件

  • index_list が空の場合、クエリは実行されず、リストの構築も行われない。
  • クエリ結果が 0 件の場合、関数は早期リターンする。
  • メモリ不足時には pg_malloc0 等によりプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は REINDEX を並列実行する際の接続管理を最適化するために重要である。
  • クエリのソート順序（ORDER BY 句）を変更する場合、並列実行の効率に直接影響するため注意すること。
