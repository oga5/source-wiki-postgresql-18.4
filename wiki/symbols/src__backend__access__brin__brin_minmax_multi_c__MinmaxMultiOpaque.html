MinmaxMultiOpaque

定義

  • 種別: struct
  • 定義箇所: src/backend/access/brin/brin_minmax_multi.c:111

目的

BRIN インデックスの Multi Min/Max 演算クラスにおいて、インデックス操作に必要な関数情報およびサブタイプ情報を保持するための不透明（opaque）なデータ構造です。

シグネチャまたは形状

typedef struct MinmaxMultiOpaque
{
    FmgrInfo    extra_procinfos[MINMAX_MAX_PROCNUMS];
    Oid         cached_subtype;
    FmgrInfo    strategy_procinfos[BTMaxStrategyNumber];
} MinmaxMultiOpaque;

入力と出力

  • **入力**: BRIN インデックス記述子（BrinDesc）の oi_opaque フィールドを介してアクセスされます。
  • **出力**: 保持している FmgrInfo を通じて、インデックスの整合性チェックや値の比較処理に必要な関数呼び出しが行われます。

前提条件と不変条件

  • extra_procinfos は MINMAX_MAX_PROCNUMS 分の要素を持ち、特定のプロシージャ番号からインデックスを算出するために使用されます。
  • strategy_procinfos は B-Tree 戦略数（BTMaxStrategyNumber）に基づいた関数情報を保持します。

アルゴリズムまたは振る舞い

  • インデックスの初期化時にメモリが割り当てられ、BrinDesc に格納されます。
  • インデックススキャンや値の追加時に、この構造体からキャッシュされた FmgrInfo を取り出し、PostgreSQL の関数呼び出し基盤を利用して比較演算を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: brin_minmax_multi.c 内の関数（brin_minmax_multi_consistent 等）が、bdesc->bd_info[attno - 1]->oi_opaque をキャストして本構造体を参照します。
  • **呼び出し先**: 保持している FmgrInfo を介して、PostgreSQL の関数マネージャ（fmgr）が呼び出されます。

関連データ構造

  • BrinDesc: 本構造体を保持する親構造体。
  • FmgrInfo: 関数呼び出しを高速化するためのキャッシュ構造体。

失敗モードと境界条件

  • attno が不適切な場合、bd_info 配列へのアクセスで境界外参照が発生する可能性があります。
  • cached_subtype が正しく設定されていない場合、型依存の比較処理で予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は brin_minmax_multi.c 内でのみ使用されるプライベートな構造体です。
  • MINMAX_MAX_PROCNUMS や BTMaxStrategyNumber の定義変更に伴い、配列サイズが影響を受けるため注意してください。
