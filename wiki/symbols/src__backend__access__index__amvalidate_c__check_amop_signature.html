check_amop_signature

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/amvalidate.c:205

目的

指定された演算子が、アクセスメソッドの要件として期待される戻り値型および引数型と一致するかを検証します。

シグネチャまたは形状

bool check_amop_signature(Oid opno, Oid restype, Oid lefttype, Oid righttype)

入力と出力

  • **入力**:
  • opno: 検証対象の演算子の OID。
  • restype: 期待される戻り値の型 OID。
  • lefttype: 期待される左辺の型 OID。
  • righttype: 期待される右辺の型 OID。
  • **出力**:
  • bool: 演算子の定義が指定された型制約および二項演算子であるという条件を満たす場合は true、それ以外は false を返します。

前提条件と不変条件

  • opno に対応する演算子がシステムカタログ pg_operator に存在する必要があります。
  • 演算子は「二項演算子（binary operator）」である必要があります。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用して、OPEROID キャッシュから演算子の情報を取得します。 2. 取得した Form_pg_operator 構造体を参照し、以下の条件をすべて満たすか確認します。

  • 戻り値型 (oprresult) が restype と一致すること。
  • 演算子種別 (oprkind) が 'b'（二項演算子）であること。
  • 左辺の型 (oprleft) が lefttype と一致すること。
  • 右辺の型 (oprright) が righttype と一致すること。

    • 3. ReleaseSysCache を呼び出してキャッシュエントリを解放します。 4. 条件を満たせば true、満たさなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blvalidate.c

  • **呼び出し先**:

  • ReleaseSysCache
  • elog (エラー発生時)

関連データ構造

  • Form_pg_operator: pg_operator システムカタログの行構造体。

失敗モードと境界条件

  • opno が無効な場合、elog(ERROR, ...) により処理が中断されます。
  • 演算子が二項演算子でない場合、型が一致していても false を返します。

保守担当者向けメモ

  • 本関数はメタデータ検証用であり、インデックスの実行時操作には関与しません。
  • 演算子番号の検証は行わず、あくまで型シグネチャの整合性のみをチェックします。
