gist_indexsortbuild

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/gist/gistbuild.c:399

目的

ソート済みの入力タプルを使用して、GiSTインデックスをボトムアップ方式で効率的に構築します。

シグネチャまたは形状

static void gist_indexsortbuild(GISTBuildState *state)

入力と出力

  • **入力**: GISTBuildState *state（構築中のインデックス状態、ソート済みタプルソースを含む）。
  • **出力**: なし（副作用としてインデックスファイルへの書き込みを行う）。

前提条件と不変条件

  • state->sortstate にソート済みのタプルが準備されていること。
  • インデックスのルートページとしてブロック0が予約される。

アルゴリズムまたは振る舞い

1. **初期化**: smgr_bulk_start_rel を呼び出し、リーフレベルの初期ページを割り当てて初期化する。 2. **タプル挿入**: tuplesort_getindextuple でソート済みタプルを順次取得し、gist_indexsortbuild_levelstate_add を用いてページへ追加する。各ステップでメモリコンテキストをリセットする。 3. **フラッシュ**: リーフからルートに向かって、満杯になったページを gist_indexsortbuild_levelstate_flush で順次書き出す。 4. **ルート書き込み**: 最終的なルートページを GIST_ROOT_BLKNO に書き込み、バルク書き込みを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: gistbuild（src/backend/access/gist/gistbuild.c:283）。
  • **呼び出し先**:
  • tuplesort_getindextuple
  • gist_indexsortbuild_levelstate_add
  • gist_indexsortbuild_levelstate_flush
  • smgr_bulk_* 系関数（ストレージ管理）

関連データ構造

  • GISTBuildState: 構築全体の管理状態。
  • GistSortedBuildLevelState: 構築中の各階層のページ状態を保持。
  • BulkWriteBuffer: バルク書き込み用バッファ。

失敗モードと境界条件

  • インデックスが単一ページ（ルートのみ）で構成される場合、ループ処理により適切にルートが書き出される。
  • メモリ不足時は palloc 等によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、GiSTインデックスの「ソート済み構築」パスの核心部です。
  • gist_indexsortbuild_levelstate_add および flush が階層構造の構築を担っているため、ロジックの変更時はこれらの関数との連携を確認してください。
  • GIST_SORTED_BUILD_PAGE_NUM に基づくページ管理のライフサイクルに注意してください。
