HeapTupleIsSurelyDead

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_visibility.c:1473

目的

指定されたヒープタプルが、現在のグローバルな可視性状態において「確実に削除済み（Dead）」であるかを判定します。主にVACUUM処理などで、タプルが物理的に不要であるかを高速に判断するために使用されます。

シグネチャまたは形状

bool HeapTupleIsSurelyDead(HeapTuple htup, GlobalVisState *vistest)

入力と出力

  • **入力**:
  • htup: 判定対象の HeapTuple 構造体。
  • vistest: 可視性判定のための GlobalVisState 構造体。
  • **出力**:
  • bool: タプルが確実に削除済みであれば true、そうでなければ false を返します。

前提条件と不変条件

  • htup->t_self が有効なアイテムポインタであること。
  • htup->t_tableOid が InvalidOid ではないこと。
  • 呼び出し元は、対象タプルを含むバッファに対して適切なロック（通常は共有ロック以上）を保持している必要があります。

アルゴリズムまたは振る舞い

1. **挿入トランザクションの確認**: タプルの Xmin がコミット済みでない場合、Xmin が無効（Invalid）であればタプルは確実に削除済みと判定します。 2. **削除トランザクションの確認**:

  • Xmax が無効（HEAP_XMAX_INVALID）な場合、タプルは生存しているとみなします。
  • Xmax がロック専用（HEAP_XMAX_IS_LOCKED_ONLY）の場合、タプルは生存しているとみなします。
  • Xmax がマルチトランザクション（HEAP_XMAX_IS_MULTI）の場合、pg_multixact の確認が必要となるため、ここでは生存しているとみなします。
  • Xmax がコミット済みでない場合、削除トランザクションが実行中であると仮定し、生存しているとみなします。

    • 3. **最終判定**: 削除トランザクションがコミット済みであることが確認できた場合、GlobalVisTestIsRemovableXid を呼び出し、その Xmax が十分に古い（どのスナップショットからも参照されない）かどうかを判定して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c:1834
  • **呼び出し先**:
  • GlobalVisTestIsRemovableXid (外部モジュール関数)
  • HeapTupleHeaderXminCommitted / HeapTupleHeaderXminInvalid (マクロ)
  • HeapTupleHeaderGetRawXmax (マクロ)

関連データ構造

  • HeapTuple: タプルのヘッダーおよびデータを含む構造体。
  • GlobalVisState: グローバルな可視性判定状態を保持する構造体。
  • HeapTupleHeaderData: タプルヘッダーのフラグ（t_infomask）を保持。

失敗モードと境界条件

  • Xmax がマルチトランザクションである場合、この関数は保守的に false を返します。
  • 挿入トランザクションの状態が不明な場合、ヒントビットの前提に基づき生存しているとみなします。

保守担当者向けメモ

  • 本関数は、タプルが「確実に死んでいる」ことの証明を試みるものであり、false が返されたからといって必ずしもタプルが生存しているとは限りません（判定不能なケースを含むため）。
  • t_infomask のビット操作は、PostgreSQLのMVCC可視性判定の根幹であるため、変更時は src/backend/access/heap/heapam_visibility.c 全体の整合性に注意してください。
