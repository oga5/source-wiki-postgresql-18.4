Datum

定義

  • 種別: typedef
  • 定義箇所: src/include/commands/user.h:25

目的

PostgreSQL における汎用的なデータ値を表現するための型です。

シグネチャまたは形状

typedef uintptr_t Datum;

*(注: 本ソース抜粋には型定義の詳細は含まれていませんが、PostgreSQL の標準的な定義に基づきます)*

入力と出力

  • **入力**: 任意のデータ型（整数、ポインタ、または特定の構造体）を格納可能。
  • **出力**: 格納されたデータ値。

前提条件と不変条件

  • Datum は、整数値またはポインタを保持できる十分なサイズ（通常は uintptr_t と同等）を持つ必要があります。
  • 値がポインタである場合、そのメモリ領域は有効である必要があります。

アルゴリズムまたは振る舞い

  • Datum は、PostgreSQL の実行エンジン全体で、型を問わず値を渡すための汎用コンテナとして機能します。
  • 値が「値渡し」可能な型（整数など）か、「参照渡し」が必要な型（構造体など）かによって、解釈方法が異なります。

呼び出し元と呼び出し先

  • **呼び出し元**: データベースのクエリ実行エンジン、インデックスチェックツール（contrib/amcheck）、およびシステムカタログ操作関数。
  • **呼び出し先**: Datum を引数として受け取る各種関数（例: check_password_hook_type の validuntil_time 引数）。
  • *注: 提供されたソース抜粋には、具体的な呼び出し関係のフローは記述されていません。*

関連データ構造

  • check_password_hook_type: パスワード検証フックにおいて、有効期限（validuntil_time）を渡すために使用されます。
  • PasswordType: パスワードの暗号化方式を定義する列挙型。

失敗モードと境界条件

  • **型不整合**: Datum に格納された型と、それを取り出す際の期待される型が一致しない場合、メモリ破壊や不正な値の読み取りが発生します。
  • **NULL 許容**: Datum 自体は NULL を保持できないため、別途 bool 型のフラグ（例: validuntil_null）と組み合わせて NULL を表現する必要があります。

保守担当者向けメモ

  • Datum は PostgreSQL のコア全体で使用される基盤型です。変更を加える際は、システム全体のバイナリ互換性に影響を与える可能性があるため注意してください。
  • user.h 内では、主にパスワードの有効期限などのメタデータを柔軟に渡すために利用されています。
