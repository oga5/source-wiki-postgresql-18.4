MemoryChunk

定義

  • 種別: struct
  • 定義箇所: src/include/utils/memutils_memorychunk.h:124

目的

PostgreSQL のメモリコンテキストによって割り当てられたメモリチャンクの先頭に付与されるヘッダー構造体です。チャンクのメタデータ保持および親ブロックの特定を可能にする軽量なインターフェースを提供します。

シグネチャまたは形状

typedef struct MemoryChunk
{
#ifdef MEMORY_CONTEXT_CHECKING
    Size        requested_size;
#endif
    uint64      hdrmask;
} MemoryChunk;

入力と出力

  • **入力**: メモリ割り当て時に MemoryChunkSetHdrMask 等を通じてメタデータ（コンテキストID、ブロックオフセット、フラグ等）が hdrmask に書き込まれます。
  • **出力**: MemoryChunkGetBlock を通じて、チャンクポインタから親ブロックのアドレスが復元されます。

前提条件と不変条件

  • hdrmask のビット配置は、ブロックオフセットの最下位ビットが常に 0 であること（MAXALIGN 制約）に依存しています。
  • MEMORY_CONTEXT_CHECKING が有効な場合、ヘッダーサイズが 8 バイトから 16 バイトに増加します。
  • 外部管理チャンクの場合、hdrmask はマジックナンバーを保持し、コンテキスト固有の管理ロジックが適用されます。

アルゴリズムまたは振る舞い

1. **割り当て**: メモリコンテキストが hdrmask にオフセットとサイズ情報をパックします。 2. **逆算**: MemoryChunkGetBlock が hdrmask からオフセットを抽出し、現在のチャンクポインタから減算することで、元のブロックアドレスを特定します。 3. **外部管理**: 大規模割り当て時は「外部」フラグが立てられ、標準的なオフセット計算ではなく個別の管理スキームが実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAgg.c: CHUNKHDRSZ 定義およびメモリオーバーヘッドの計算に使用。
  • src/backend/utils/mmgr/alignedalloc.c: アライメント調整されたメモリの管理および Valgrind によるメモリ追跡に使用。
  • **呼び出し先**:
  • 直接的な関数呼び出し関係はソース抜粋からは特定できませんが、utils/memutils_internal.h に依存しています。

関連データ構造

  • hdrmask: メタデータをパックする 64 ビットフィールド。
  • MemoryContextMethodID: memutils_internal.h で定義されるコンテキスト識別子。

失敗モードと境界条件

  • MEMORY_CONTEXT_CHECKING の有無によるヘッダーサイズの変動を考慮しない場合、メモリレイアウトの破損を招く恐れがあります。
  • hdrmask のビット共有規約（末尾 4 ビットの MemoryContextMethodID 保持など）に違反すると、メモリ管理サブシステムが正しく動作しません。

保守担当者向けメモ

  • 本構造体はメモリ管理の低レイヤー基盤であるため、変更時は MemoryChunkGetBlock や MemoryChunkIsExternal などの関連インライン関数への影響を必ず確認してください。
  • Valgrind 等のデバッグツールを使用する際は、alignedalloc.c の例のように VALGRIND_MAKE_MEM_DEFINED を適切に適用してください。
