SPLITCOST

定義

  • 種別: struct
  • 定義箇所: contrib/pg_trgm/trgm_gist.c:765

目的

GiST インデックスの分割処理（picksplit）において、各ノードの分割コストを管理するために使用される構造体です。

シグネチャまたは形状

typedef struct
{
	OffsetNumber pos;
	int32		cost;
} SPLITCOST;

入力と出力

  • pos: 分割対象となるインデックス項目のオフセット番号。
  • cost: その項目を特定のグループに割り当てた際のペナルティコスト。

前提条件と不変条件

  • pos は OffsetNumber 型であり、有効なインデックス内の位置を指す必要があります。
  • cost は比較演算（qsort 等）の基準として使用されます。

アルゴリズムまたは振る舞い

GiST インデックスの分割アルゴリズムにおいて、複数の項目に対してコストを計算し、この構造体の配列に格納した上で、コストに基づいてソートを行うことで最適な分割位置を決定するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_trgm/trgm_gist.c 内の gtrgm_picksplit 関数（直接的なソースコード上の明示的な呼び出し関係は、提供された抜粋内には含まれていません）。
  • **呼び出し先**: なし。
  • **備考**: contrib/hstore/hstore_gist.c および contrib/intarray/_int_gist.c においても同名の構造体が定義・利用されていますが、これらは別モジュールの実装であり、本シンボルとの直接的な依存関係はありません。

関連データ構造

  • OffsetNumber: PostgreSQL のインデックスページ内での項目位置を示す型。

失敗モードと境界条件

  • メモリ不足: palloc を使用して配列を確保する際、maxoff が極端に大きい場合にメモリ不足が発生する可能性があります。
  • ソートの安定性: cost が同一の場合の順序は、比較関数の実装に依存します。

保守担当者向けメモ

  • 本構造体は hstore や intarray 拡張と共通の設計パターンを採用しています。変更を加える際は、他の GiST 実装との整合性に注意してください。
  • contrib/pg_trgm/trgm_gist.c 内での具体的な使用箇所については、gtrgm_picksplit の実装詳細を確認してください。
