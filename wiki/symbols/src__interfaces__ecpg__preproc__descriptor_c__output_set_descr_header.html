output_set_descr_header

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/descriptor.c:207

目的

SQL記述子領域のヘッダー設定を行うためのCコードを生成します。具体的には、SET DESCRIPTOR 文のヘッダー操作（COUNT 等）をECPGランタイムライブラリの関数呼び出しに変換します。

シグネチャまたは形状

void output_set_descr_header(const char *desc_name)

入力と出力

  • **入力**:
  • desc_name: 設定対象となる記述子の名前（文字列）。
  • assignments (グローバル変数): 記述子に対して設定される項目（ECPGd_count 等）と変数のリスト。
  • **出力**:
  • base_yyout: 生成されたCコード（ECPGset_desc_header 関数の呼び出し）が書き込まれます。

前提条件と不変条件

  • グローバル変数 assignments に、設定対象の項目が格納されている必要があります。
  • 呼び出し終了時に drop_assignments() が実行され、assignments リストが解放されます。

アルゴリズムまたは振る舞い

1. ECPGset_desc_header 関数の開始コードを base_yyout に出力します。 2. assignments リストを走査し、各項目を処理します。

  • 項目が ECPGd_count の場合、ECPGnumeric_lvalue を呼び出して変数の型情報を出力します。
  • それ以外の項目が指定された場合、mmerror を通じて警告を発行します。

    • 3. drop_assignments() を呼び出し、リストをクリーンアップします。 4. 関数の閉じ括弧とセミコロンを出力し、whenever_action(3) を実行してエラーハンドリングを適用します。

呼び出し元と呼び出し先

  • **呼び出し元**: プリプロセッサの構文解析フェーズ（詳細はソースコード上の直接的な呼び出し元情報に依存）。
  • **呼び出し先**:
  • ECPGnumeric_lvalue
  • mmerror
  • drop_assignments
  • whenever_action

関連データ構造

  • struct assignment: 記述子への代入操作を保持するリスト構造。
  • base_yyout: コード生成先の出力ストリーム。

失敗モードと境界条件

  • ECPGd_count 以外の項目が assignments に含まれている場合、ET_WARNING レベルの警告が生成されます。

保守担当者向けメモ

  • 本関数は assignments リストを消費して空にするため、一度しか呼び出せません。
  • 記述子ヘッダーの項目追加が必要な場合は、assignments の走査ロジックを拡張してください。
