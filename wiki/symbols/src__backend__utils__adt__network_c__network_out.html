network_out

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network.c:140

目的

inet または cidr 型の内部表現を、PostgreSQL のクライアントへ送信可能な文字列形式に変換します。

シグネチャまたは形状

static char *network_out(inet *src, bool is_cidr)

入力と出力

  • **入力**:
  • src: 変換対象の inet 構造体ポインタ。
  • is_cidr: cidr 型として扱うかを示すフラグ。
  • **出力**: 変換された IP アドレス文字列（pstrdup によりメモリ確保済み）。

前提条件と不変条件

  • src は有効な inet 構造体を指している必要があります。
  • cidr 型の場合、マスク長より右側のビットが 0 であるという不変条件が前提となります。

アルゴリズムまたは振る舞い

1. pg_inet_net_ntop を使用し、src のファミリー、アドレス、ビット長を基に文字列へ変換します。 2. 変換に失敗した場合は ERROR を発生させます。 3. is_cidr が true であり、かつ文字列内に / が含まれていない場合、snprintf を用いてマスク長（/n）を文字列の末尾に付加します。 4. 最終的な文字列を pstrdup で複製して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network.c 内の inet_out および cidr_out 関数から呼び出されます。
  • **呼び出し先**:
  • pg_inet_net_ntop: 文字列変換を実行。
  • ereport: エラー報告。
  • pstrdup: メモリ確保。

関連データ構造

  • inet: ネットワーク型を保持する内部構造体。

失敗モードと境界条件

  • pg_inet_net_ntop が NULL を返した場合、ERRCODE_INVALID_BINARY_REPRESENTATION エラーが発生します。
  • is_cidr が指定されている場合、マスク長が欠落していれば自動的に補完されます。

保守担当者向けメモ

  • 本関数は inet_out および cidr_out の共通ロジックです。
  • tmp バッファサイズは IPv6 アドレスとマスク長を格納するのに十分なサイズ（xxxx:xxxx.../128）で定義されています。
