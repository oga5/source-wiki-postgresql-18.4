latin2_to_mic

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/latin2_and_win1250/latin2_and_win1250.c:85

目的

LATIN2 (ISO-8859-2) エンコーディングから PostgreSQL 内部表現 (MULE_INTERNAL) への文字コード変換を実行します。

シグネチャまたは形状

Datum latin2_to_mic(PG_FUNCTION_ARGS)

PostgreSQL の関数マネージャインターフェース（fmgr.h）に準拠した形式です。

入力と出力

  • **入力**:
  • PG_GETARG_CSTRING(2): 変換元文字列（unsigned char *src）
  • PG_GETARG_CSTRING(3): 変換先バッファ（unsigned char *dest）
  • PG_GETARG_INT32(4): 文字列長（len）
  • PG_GETARG_BOOL(5): エラーハンドリングフラグ（noError）
  • **出力**:
  • PG_RETURN_INT32(converted): 変換されたバイト数。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_LATIN2, PG_MULE_INTERNAL) により、呼び出し時にエンコーディングIDが LATIN2 および MULE_INTERNAL であることが検証されます。

アルゴリズムまたは振る舞い

1. 引数から変換元ポインタ、変換先ポインタ、長さ、エラー制御フラグを取得します。 2. エンコーディング引数の妥当性を検証します。 3. mb/wchar.c で定義された汎用ユーティリティ関数 latin2mic を呼び出し、変換を実行します。 4. 変換されたバイト数を戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の変換マネージャ（動的ロード経由）。PG_FUNCTION_INFO_V1(latin2_to_mic) マクロにより登録されています。
  • **呼び出し先**: latin2mic（src/backend/utils/mb/wchar.c 内のユーティリティ関数）。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQL の関数引数コンテナ。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、PostgreSQL のエラー処理機構により処理が中断されます。
  • noError フラグの設定に基づき、変換不能な文字が含まれる場合の挙動が latin2mic 内で制御されます。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の変換ロジックは latin2mic に委譲されています。
  • 変換ロジックの変更が必要な場合は、src/backend/utils/mb/wchar.c を参照してください。
