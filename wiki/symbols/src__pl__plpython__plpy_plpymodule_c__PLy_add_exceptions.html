PLy_add_exceptions

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_plpymodule.c:171

目的

PL/Python モジュール (plpy) 内に spiexceptions サブモジュールを生成し、PostgreSQL のエラー状態を Python 例外として扱うための基盤を初期化します。

シグネチャまたは形状

static void PLy_add_exceptions(PyObject *plpy)

入力と出力

  • **入力**: plpy (Python モジュールオブジェクトへのポインタ)
  • **出力**: なし（副作用として Python インタプリタ内のモジュール構造を更新）

前提条件と不変条件

  • Python インタプリタが初期化済みであること。
  • PLy_exc_module が定義されていること。
  • PLy_create_exception および PLy_generate_spi_exceptions が利用可能であること。

アルゴリズムまたは振る舞い

1. PyModule_Create を使用して spiexceptions モジュールを作成します。 2. PyModule_AddObject を用いて、作成したモジュールを plpy モジュールに登録します（参照カウントを適切に管理）。 3. PLy_create_exception を呼び出し、以下の基本例外クラスを生成・登録します。

  • plpy.Error
  • plpy.Fatal
  • plpy.SPIError

    • 4. hash_create を呼び出し、SQLSTATE をキーとする例外マッピング用のハッシュテーブル PLy_spi_exceptions を作成します。 5. PLy_generate_spi_exceptions を呼び出し、SQLSTATE に対応する個別の例外クラスを spiexceptions モジュール内に動的に生成・登録します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_plpymodule.c 内の PyInit_plpy（モジュール初期化時）。
  • **呼び出し先**:
  • PyModule_Create, PyModule_AddObject, Py_INCREF (Python C API)
  • PLy_elog (エラー報告)
  • PLy_create_exception (例外クラス生成)
  • hash_create (PostgreSQL ハッシュテーブル作成)
  • PLy_generate_spi_exceptions (例外クラスの動的登録)

関連データ構造

  • PLy_spi_exceptions: SQLSTATE と Python 例外オブジェクトを紐付けるハッシュテーブル。
  • PLyExceptionEntry: ハッシュテーブルの各エントリの構造体。

失敗モードと境界条件

  • PyModule_Create が失敗した場合、PLy_elog(ERROR, ...) により PostgreSQL のエラーとして報告されます。
  • PyModule_AddObject が失敗した場合も同様にエラー報告されます。

保守担当者向けメモ

  • PyModule_AddObject はオブジェクトの参照カウントをインクリメントしない仕様であるため、明示的に Py_INCREF を呼び出しています。この挙動を変更する際は注意してください。
  • PLy_spi_exceptions は HASH_ELEM | HASH_BLOBS フラグで作成されており、キーサイズは int です。
