plperl_create_sub

定義

  • 種別: function
  • 定義箇所: src/pl/plperl/plperl.c:2095

目的

PostgreSQL の関数ソースコードを Perl の無名サブルーチンとしてコンパイルし、その参照を plperl_proc_desc 構造体に格納する。

シグネチャまたは形状

static void plperl_create_sub(plperl_proc_desc *prodesc, const char *s, Oid fn_oid)

入力と出力

  • **入力**:
  • prodesc: コンパイル結果を格納する関数記述子。
  • s: コンパイル対象となる Perl コードのソース文字列。
  • fn_oid: 関数の OID（サブルーチン名の生成に使用）。
  • **出力**: なし（副作用として prodesc->reference にサブルーチン参照をセットする）。

前提条件と不変条件

  • Perl インタプリタがアクティブであること。
  • PostgreSQL::InServer::mkfunc が Perl 側で定義されていること。
  • コンパイルエラーが発生した場合、PostgreSQL のエラーとして報告される。

アルゴリズムまたは振る舞い

1. subname を生成し、pragma_hv（strict モード設定用）を初期化する。 2. Perl スタックを準備し、サブルーチン名、プラグマ、ソースコードを引数として PostgreSQL::InServer::mkfunc を呼び出す。 3. G_EVAL および G_KEEPERR を指定して実行し、Perl 側のコンパイルエラーを捕捉する。 4. 戻り値がコード参照（SVt_PVCV）であることを確認し、prodesc->reference に格納する。 5. コンパイル失敗時やエラー発生時は ereport(ERROR, ...) を呼び出し、処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plperl/plperl.c 内の関数（plperl_inline_handler およびキャッシュミス時の関数コンパイル処理から呼び出される）。
  • **呼び出し先**:
  • call_pv（Perl C API）
  • ereport（PostgreSQL エラー報告）

関連データ構造

  • plperl_proc_desc: コンパイルされた関数情報を保持する構造体。
  • HV, SV, AV: Perl のデータ型（ハッシュ、スカラー、配列）。

失敗モードと境界条件

  • **構文エラー**: Perl 側でコンパイルエラーが発生した場合、ERRSV を通じてエラーメッセージを取得し、PostgreSQL の ERROR として報告する。
  • **型不一致**: mkfunc がコード参照を返さない場合、エラーとして報告する。

保守担当者向けメモ

  • G_KEEPERR の使用は、mkfunc 内部での eval との干渉を防ぐために必須である。
  • subname の生成には NAMEDATALEN を使用しており、バッファオーバーフローに注意が必要。
  • prolog 引数として &PL_sv_no を渡しているが、これは将来的な拡張性（NYTprof 等）のための互換性維持目的である。
