pg_strtoint64_safe

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/numutils.c:649

目的

文字列を int64 型の整数へ安全に変換します。エラー発生時には ereport を直接呼び出す代わりに、提供された escontext を介してエラー情報を伝播させます。

シグネチャまたは形状

int64 pg_strtoint64_safe(const char *s, Node *escontext);

入力と出力

  • **入力**:
  • s: 解析対象の文字列（NUL終端）。
  • escontext: エラー報告用コンテキスト（Node *）。NULL の場合、エラー時は即座に中断されます。
  • **出力**: 変換後の int64 値。

前提条件と不変条件

  • 入力文字列は、10進数、16進数（0x）、8進数（0o）、または2進数（0b）の形式を許容します。
  • 数値の可読性を高めるためのアンダースコア（_）を数値の間に含めることができます。
  • 変換結果が int64 の範囲（PG_INT64_MIN ～ PG_INT64_MAX）を超えた場合はエラーとみなされます。

アルゴリズムまたは振る舞い

1. **高速パス**: まず、アンダースコアを含まない単純な10進数として解析を試みます。 2. **低速パス**: 高速パスで解析できない場合（基数指定がある、アンダースコアが含まれる、または先頭に空白がある場合）、slow ラベルへ遷移し、基数（10, 16, 8, 2）を判定して詳細なパースを行います。 3. **エラー処理**: オーバーフローや構文エラーを検出した場合、escontext が提供されていればそこへエラーを記録し、そうでなければ ereport を発行して処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_node.c
  • src/backend/utils/adt/int8.c
  • src/backend/utils/adt/numutils.c（自身をラップする関数など）
  • **呼び出し先**:
  • isspace, isxdigit（標準ライブラリ）
  • pg_neg_u64_overflow（オーバーフローチェック）
  • ereport（エラー発生時）

関連データ構造

  • Node: エラー報告コンテキストとして使用。
  • int64, uint64: PostgreSQL の固定長整数型。

失敗モードと境界条件

  • **オーバーフロー**: 変換後の値が int64 の範囲外である場合。
  • **構文エラー**:
  • アンダースコアの直後に数字以外の文字が続く場合。
  • 基数指定（0x等）の直後に有効な数字が続かない場合。
  • 不正な文字が含まれる場合。
  • **空入力**: 数字が含まれない文字列は構文エラーとなります。

保守担当者向けメモ

  • 本関数はパフォーマンスを重視しており、単純な10進数は高速パスで処理されます。修正を加える際は、高速パスの条件分岐（likely/unlikely マクロの使用箇所）が意図通りに機能しているか確認してください。
  • escontext の利用により、呼び出し元でエラー制御が可能になっています。新しいエラーケースを追加する際は、ereport を直接呼ぶのではなく、escontext を適切に利用する設計を維持してください。
