get_tsearch_config_filename

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_utils.c:33

目的

PostgreSQLの全文検索機能で使用される設定ファイルに対し、share/tsearch_data ディレクトリ配下の安全な絶対パスを生成します。

シグネチャまたは形状

char *get_tsearch_config_filename(const char *basename, const char *extension);

入力と出力

  • **入力**:
  • basename: ファイルのベース名（文字列）。
  • extension: ファイルの拡張子（文字列）。
  • **出力**:
  • 成功時: 生成された絶対パスを含む palloc された文字列（char *）。
  • 失敗時: ereport(ERROR, ...) により処理が中断されます。

前提条件と不変条件

  • basename は a-z、0-9、およびアンダースコア（_）のみで構成されている必要があります。これ以外の文字が含まれる場合、セキュリティ上の理由（ディレクトリトラバーサル攻撃の防止）からエラーとなります。

アルゴリズムまたは振る舞い

1. strspn を使用して basename の文字種を検証し、許可されていない文字が含まれていればエラーを報告します。 2. get_share_path を呼び出し、実行環境の共有ディレクトリパスを取得します。 3. palloc を使用してパス格納用のメモリを確保します。 4. snprintf を使用し、{sharepath}/tsearch_data/{basename}.{extension} という形式で絶対パスを構築します。 5. 生成したパス文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dict_xsyn/dict_xsyn.c
  • contrib/unaccent/unaccent.c
  • src/backend/tsearch/dict_ispell.c
  • src/backend/tsearch/dict_synonym.c
  • src/backend/tsearch/dict_thesaurus.c
  • src/backend/tsearch/ts_utils.c（内部のストップワード読み込み処理など）
  • **呼び出し先**:
  • get_share_path（PostgreSQL内部関数）
  • palloc（メモリ割り当て）
  • snprintf（文字列整形）
  • strspn, strlen（標準Cライブラリ）
  • ereport（エラー報告）

関連データ構造

  • MAXPGPATH: パス文字列の最大長を定義する定数。

失敗モードと境界条件

  • basename に許可外の文字（/, :, \ 等）が含まれる場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー終了します。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数はセキュリティ境界として機能します。パス生成ロジックを変更する際は、ディレクトリトラバーサル攻撃の脆弱性が生じないよう厳重に検証してください。
  • 大文字や非ASCII文字を許可すると、ファイルシステムのケースセンシティビティの違いによりプラットフォーム間で互換性の問題が生じる可能性があるため、現在の制限は意図的なものです。
