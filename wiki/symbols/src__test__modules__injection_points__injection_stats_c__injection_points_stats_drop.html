injection_points_stats_drop

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_stats.c:210

目的

インジェクションポイントに関連する統計エントリを削除するためのSQL呼び出し可能な関数です。

シグネチャまたは形状

Datum injection_points_stats_drop(PG_FUNCTION_ARGS);

入力と出力

  • 入力: PG_FUNCTION_ARGS（SQLインターフェース経由で呼び出される標準的な引数）
  • 出力: void（PG_RETURN_VOID() を介して返却）

前提条件と不変条件

  • PostgreSQLの統計サブシステムが初期化されている必要があります。
  • match_inj_entries 関数が、削除対象を特定するための条件として機能します。

アルゴリズムまたは振る舞い

1. pgstat_drop_matching_entries を呼び出し、統計サブシステム内のインジェクションポイント統計エントリを検索します。 2. match_inj_entries をフィルタとして使用し、一致するエントリを削除します。 3. 処理完了後、値を返さずに終了します。

呼び出し元と呼び出し先

  • 呼び出し元: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • 呼び出し先: pgstat_drop_matching_entries（PostgreSQL統計サブシステム関数）、match_inj_entries（一致判定用コールバック）。

関連データ構造

  • PgStat_StatInjEntry: 統計エントリのデータ構造。
  • match_inj_entries: 削除対象を判定するためのマッチング関数。

失敗モードと境界条件

  • 統計サブシステムがロックされている場合や、メモリ不足等のシステムエラーが発生した場合は、基盤となる pgstat 関数群の挙動に従いエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はSQLから直接呼び出される管理用関数です。
  • 削除ロジックを変更する場合は、match_inj_entries の実装と整合性が取れていることを確認してください。
