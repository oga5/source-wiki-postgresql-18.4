BumpAlloc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/bump.c:490

目的

バンプアロケータ（BumpContext）において、指定されたサイズのメモリチャンクを高速に割り当てるためのメインエントリポイントです。

シグネチャまたは形状

void *BumpAlloc(MemoryContext context, Size size, int flags)

入力と出力

  • **入力**:
  • context: 割り当て対象の MemoryContext（BumpContext としてキャストして使用）。
  • size: 要求するメモリサイズ（バイト単位）。
  • flags: 割り当てオプション。
  • **出力**:
  • 割り当てられたメモリ領域へのポインタ。

前提条件と不変条件

  • context は BumpIsValid を満たす有効な BumpContext であること。
  • この関数で確保したメモリに対して pfree を呼び出すことは禁止されており、実行するとエラーとなります。

アルゴリズムまたは振る舞い

1. **サイズ調整**: MEMORY_CONTEXT_CHECKING が有効な場合は、センチネルバイト用に size + 1 をアライメントし、無効な場合は size をアライメントします。 2. **巨大割り当ての判定**: 調整後のサイズが set->allocChunkLimit を超える場合、BumpAllocLarge を呼び出して処理を委譲します。 3. **ブロック内割り当て**:

  • 現在の最新ブロック（dlist_head_node）を確認します。
  • ブロック内の空き容量が required_size（チャンクサイズ + ヘッダーサイズ）未満の場合、BumpAllocFromNewBlock を呼び出して新しいブロックを確保します。
  • 十分な空きがある場合、BumpAllocChunkFromBlock を呼び出し、ブロック内のポインタを前進させてメモリを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c: MCTX_BUMP_ID の alloc メソッドとして登録。
  • **呼び出し先**:
  • BumpAllocLarge: 巨大な割り当て要求時。
  • BumpAllocFromNewBlock: 現在のブロックに空きがない場合。
  • BumpAllocChunkFromBlock: 現在のブロックに空きがある場合。

関連データ構造

  • BumpContext: コンテキストのメタデータおよびブロックリストを保持。
  • BumpBlock: メモリ割り当ての物理的な単位。

失敗モードと境界条件

  • chunk_size > set->allocChunkLimit: 巨大な割り当てとして処理されます。
  • ブロックの空き容量不足: 新しいブロックの確保が試行されます。

保守担当者向けメモ

  • 本関数は pfree をサポートしないため、短命なオブジェクトの大量割り当てに最適化されています。
  • デバッグ時には MEMORY_CONTEXT_CHECKING を有効にすることで、メモリ破壊の検知が可能です。
