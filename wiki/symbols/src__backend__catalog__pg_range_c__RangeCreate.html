RangeCreate

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_range.c:35

目的

新しい範囲型（Range Type）をシステムカタログ pg_range に登録し、その型が依存するサブタイプや演算クラス等の依存関係を pg_depend に記録します。

シグネチャまたは形状

void RangeCreate(Oid rangeTypeOid, Oid rangeSubType, Oid rangeCollation,
                 Oid rangeSubOpclass, RegProcedure rangeCanonical,
                 RegProcedure rangeSubDiff, Oid multirangeTypeOid)

入力と出力

  • **入力**:
  • rangeTypeOid: 作成する範囲型のOID。
  • rangeSubType: 範囲の要素となるサブタイプのOID。
  • rangeCollation: 照合順序のOID（有効な場合）。
  • rangeSubOpclass: 演算クラスのOID。
  • rangeCanonical: 正規化関数のOID（有効な場合）。
  • rangeSubDiff: 差分計算関数のOID（有効な場合）。
  • multirangeTypeOid: 対応するマルチレンジ型のOID。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • pg_range テーブルに対する RowExclusiveLock を保持している必要があります。
  • 範囲型が依存するサブタイプや演算クラスなどのオブジェクトは、呼び出し時点で存在している必要があります。

アルゴリズムまたは振る舞い

1. pg_range テーブルを開き、引数からタプルを構築して CatalogTupleInsert で挿入します。 2. new_object_addresses を使用して、範囲型が依存する以下のオブジェクトをリスト化します。

  • サブタイプ、演算クラス、照合順序（有効な場合）、正規化関数（有効な場合）、差分計算関数（有効な場合）。

    • 3. record_object_address_dependencies を呼び出し、範囲型から上記リストへの DEPENDENCY_NORMAL 依存関係を一括登録します。 4. マルチレンジ型から範囲型への DEPENDENCY_INTERNAL 依存関係を recordDependencyOn で記録します。 5. 最後にテーブルを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_type.c
  • src/backend/commands/typecmds.c
  • **呼び出し先**:
  • table_open, table_close
  • heap_form_tuple, heap_freetuple
  • CatalogTupleInsert
  • new_object_addresses, add_exact_object_address, record_object_address_dependencies, free_object_addresses
  • recordDependencyOn

関連データ構造

  • pg_range: 範囲型のメタデータを保持するシステムカタログ。
  • ObjectAddresses: 依存関係を管理するための一時的なリスト構造。
  • ObjectAddress: 依存関係の対象を特定するための構造体。

失敗モードと境界条件

  • OidIsValid チェックにより、照合順序や関数が指定されていない（InvalidOid）場合は依存関係の登録をスキップします。
  • カタログ挿入時に一意性制約違反が発生した場合、トランザクションがアボートされます。

保守担当者向けメモ

  • 本関数は範囲型の作成時にのみ呼び出される低レベルなカタログ操作関数です。
  • 依存関係の登録は、範囲型が削除される際にサブタイプ等が誤って削除されることを防ぐための重要な保護メカニズムです。
  • マルチレンジ型との依存関係は DEPENDENCY_INTERNAL として記録されており、範囲型が削除されるとマルチレンジ型も自動的に削除される設計となっています。
