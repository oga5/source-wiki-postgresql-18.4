Relids

定義

  • 種別: typedef
  • 定義箇所: src/include/nodes/pathnodes.h:30

目的

クエリプランナにおいて、リレーションの集合を効率的に表現および管理するために使用される型です。

シグネチャまたは形状

typedef Bitmapset *Relids;

入力と出力

  • 入力: Bitmapset へのポインタ。
  • 出力: リレーション ID の集合を保持するビットマップ構造体。

前提条件と不変条件

  • Bitmapset 構造体として定義されているため、ビット演算による集合操作（和集合、積集合、差集合など）が可能です。
  • リレーションの集合を扱うため、各ビット位置が特定のリレーションに対応します。

アルゴリズムまたは振る舞い

  • プランナがベースリレーションや結合リレーションを識別する際に、ビットマップ形式でリレーションの包含関係を保持します。
  • 複数のリレーションを一度に表現できるため、結合条件の評価やパスの依存関係チェックにおいて高速な集合演算を提供します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • contrib/postgres_fdw/deparse.c
  • contrib/postgres_fdw/postgres_fdw.c
  • contrib/postgres_fdw/postgres_fdw.h
  • src/backend/optimizer/path/allpaths.c
  • 呼び出し先:
  • Bitmapset 型を操作する関数群（src/include/nodes/bitmapset.h で定義される関数群と推測されますが、本ソース excerpt には詳細な呼び出し先関数は含まれていません）。

関連データ構造

  • Bitmapset: Relids の実体となる構造体。
  • RelOptInfo: リレーションの最適化情報を保持する構造体であり、Relids をメンバとして保持します。

失敗モードと境界条件

  • Bitmapset のメモリ管理に依存するため、メモリ不足時には割り当て失敗が発生する可能性があります。
  • 集合の要素数が Bitmapset の許容範囲を超えた場合の挙動は、Bitmapset の実装仕様に依存します。

保守担当者向けメモ

  • Relids は Bitmapset の別名です。ビット演算による集合操作が多用されるため、変更を加える際は src/include/nodes/bitmapset.h の仕様を十分に確認してください。
  • プランナの再帰的な処理で頻繁に参照されるため、パフォーマンスに直結します。
