datumRestore

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/datum.c:556

目的

シリアライズされたバイトストリームから Datum を復元します。datumSerialize によって書き込まれた形式を読み取り、元のデータ型に応じた適切な Datum 表現を再構築します。

シグネチャまたは形状

Datum datumRestore(char **start_address, bool *isnull)

入力と出力

  • **入力**:
  • start_address: 復元対象のバイトストリームの現在位置を指すポインタのポインタ。関数内で読み取り分だけ進められます。
  • isnull: NULL フラグを格納するためのポインタ。
  • **出力**:
  • 戻り値: 復元された Datum 値。
  • isnull: 復元されたデータが NULL であれば true、そうでなければ false が設定されます。

前提条件と不変条件

  • start_address が指す先には、datumSerialize で書き込まれた有効なヘッダー情報とデータが存在する必要があります。
  • ヘッダー値が -2 の場合は NULL として扱われます。
  • ヘッダー値が -1 の場合は値渡し（pass-by-value）のデータとして扱われます。
  • ヘッダー値が正の数の場合は、そのバイト数分のメモリを palloc で確保し、参照渡し（pass-by-reference）のデータとして復元します。

アルゴリズムまたは振る舞い

1. ストリームから int 型のヘッダーを読み取り、start_address を進めます。 2. ヘッダーが -2 の場合、isnull を true に設定し、(Datum) 0 を返します。 3. ヘッダーが -1 の場合、isnull を false に設定し、続く sizeof(Datum) バイトを Datum 値として読み取り、start_address を進めてその値を返します。 4. ヘッダーが正の数の場合、isnull を false に設定し、その値（バイト数）分だけ palloc でメモリを確保します。ストリームからデータをコピーし、start_address を進めた後、確保したメモリへのポインタを Datum に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtree.c
  • src/backend/executor/execParallel.c
  • src/backend/nodes/params.c
  • **呼び出し先**:
  • palloc (メモリ確保)
  • memcpy (メモリコピー)

関連データ構造

  • Datum: PostgreSQL の汎用データ型表現。
  • char **start_address: シリアライズされたデータのバッファ。

失敗モードと境界条件

  • start_address が指すメモリ領域が不足している場合、memcpy によるメモリ破壊や不正なポインタ参照が発生する可能性があります。
  • palloc がメモリ不足により失敗した場合、PostgreSQL のエラーハンドリング機構によりアボートされます。

保守担当者向けメモ

  • 本関数は datumSerialize と対になる処理です。シリアライズ形式を変更する場合は、必ず両方の関数を更新してください。
  • 参照渡し（header > 0）の場合、palloc を使用しているため、呼び出し元は適切なメモリコンテキストがアクティブであることを保証する必要があります。
