first_path_var_separator

定義

  • 種別: function
  • 定義箇所: src/port/path.c:126

目的

パスリスト文字列（環境変数 PATH など）に含まれる最初の区切り文字を検索し、その位置を特定します。

シグネチャまたは形状

char *first_path_var_separator(const char *pathlist)

入力と出力

  • **入力**: pathlist (検索対象のパスリスト文字列へのポインタ)
  • **出力**: 見つかった区切り文字へのポインタ。見つからない場合は NULL を返します。

前提条件と不変条件

  • pathlist はヌル終端文字列である必要があります。
  • 戻り値は const 修飾が外された char * 型として返されます。

アルゴリズムまたは振る舞い

1. 入力された pathlist の先頭から順に各文字を走査します。 2. IS_PATH_VAR_SEP マクロを使用して、現在の文字がプラットフォーム固有のパス変数区切り文字（例: Unix系では :、Windowsでは ;）であるかを判定します。 3. 区切り文字が見つかった時点で、その位置へのポインタを返します。 4. 文字列の終端に達しても区切り文字が見つからない場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/extension.c:497
  • src/backend/utils/fmgr/dfmgr.c:602
  • src/common/exec.c:200
  • **呼び出し先**:
  • IS_PATH_VAR_SEP マクロ（プラットフォーム依存の判定）

関連データ構造

  • 特になし。

失敗モードと境界条件

  • 空文字列が渡された場合、ループは実行されず NULL を返します。
  • 区切り文字が含まれない文字列の場合、NULL を返します。

保守担当者向けメモ

  • 本関数は unconstify を使用して const 修飾を外して返します。呼び出し側でこのポインタを介して文字列を変更しないよう注意してください。
  • IS_PATH_VAR_SEP マクロの定義はプラットフォーム（pg_config_paths.h 等）に依存するため、移植性を考慮する際はその定義を確認してください。
