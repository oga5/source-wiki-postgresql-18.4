Oid

定義

  • 種別: struct
  • 定義箇所: src/include/executor/execExpr.h:335

目的

PostgreSQLのシステムカタログにおけるオブジェクト識別子（Object Identifier）を保持するための型です。式評価エンジン内では、主に変数のデータ型を特定するために使用されます。

シグネチャまたは形状

execExpr.h 内の var 構造体の一部として定義されています。

struct {
    int attnum;
    Oid vartype;
    VarReturningType varreturningtype;
} var;

入力と出力

  • 入力: 式評価の初期化時、プランナーが生成した式ツリーから vartype として設定されます。
  • 出力: 実行時に変数の型情報を判定するために参照されます。

前提条件と不変条件

  • vartype は、PostgreSQLのシステムカタログ（pg_type）に存在する有効な OID である必要があります。

アルゴリズムまたは振る舞い

式評価ステップにおいて、特定の変数がどのデータ型であるかを識別するために使用されます。これにより、実行エンジンは適切な型変換や関数呼び出しのディスパッチを行うことが可能になります。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報には、この構造体メンバーを直接操作する具体的な関数呼び出しの記述はありません。
  • 呼び出し先: 提供されたソース情報には、このメンバーを引数として渡す具体的な関数呼び出しの記述はありません。
  • 補足: contrib/amcheck 等の外部モジュールで Oid 型自体は広く利用されていますが、本シンボル（execExpr.h 内の var.vartype）との直接的な関連を示す証拠は不足しています。

関連データ構造

  • Var: 式評価における変数ノードの構造体。
  • ExprEvalStep: 式評価の各ステップを保持する構造体。

失敗モードと境界条件

  • 無効な OID が設定された場合、実行時の型判定や関数解決において予期せぬエラーが発生する可能性があります。

保守担当者向けメモ

  • 本シンボルは ExprEvalStep の一部としてメモリレイアウトに影響を与えます。構造体のサイズ変更を行う際は、キャッシュライン（64バイト）への適合性に注意してください。
