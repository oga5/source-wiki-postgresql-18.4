DefineTSDictionary

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tsearchcmds.c:396

目的

SQL の CREATE TEXT SEARCH DICTIONARY コマンドを実行し、新しい全文検索辞書をシステムカタログ pg_ts_dict に登録する。

シグネチャまたは形状

ObjectAddress DefineTSDictionary(List *names, List *parameters)

入力と出力

  • **入力**:
  • names: 辞書名を指定する修飾名リスト。
  • parameters: 辞書の定義オプション（テンプレート名や初期化オプションなど）を含む DefElem のリスト。
  • **出力**:
  • 作成された辞書の ObjectAddress を返す。

前提条件と不変条件

  • 呼び出し元は、対象スキーマに対する CREATE 権限を持っている必要がある。
  • parameters 内に template オプションが必須であり、指定されたテンプレートが存在しなければならない。

アルゴリズムまたは振る舞い

1. **名前解決と権限チェック**: QualifiedNameGetCreationNamespace で名前空間を特定し、object_aclcheck で権限を検証する。 2. **パラメータ解析**: parameters リストを走査し、template を抽出して OID を取得する。それ以外は辞書オプションとして保持する。 3. **検証**: verify_dictoptions を呼び出し、テンプレートに対してオプションが妥当かを確認する。 4. **カタログ挿入**: pg_ts_dict を開き、新しい OID を割り当ててタプルを構築する。オプションが存在する場合は serialize_deflist でシリアライズして格納する。 5. **依存関係とフック**: makeDictionaryDependencies で依存関係を構築し、InvokeObjectPostCreateHook で作成後フックを実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c (SQL 解析後のディスパッチ)
  • **呼び出し先**: QualifiedNameGetCreationNamespace, object_aclcheck, get_ts_template_oid, verify_dictoptions, table_open, GetNewOidWithIndex, serialize_deflist, heap_form_tuple, CatalogTupleInsert, makeDictionaryDependencies, InvokeObjectPostCreateHook

関連データ構造

  • pg_ts_dict: 辞書定義を保持するシステムカタログ。
  • DefElem: SQL 文から渡されるパラメータ要素。
  • ObjectAddress: 作成されたオブジェクトの識別子。

失敗モードと境界条件

  • 権限不足の場合、aclcheck_error によりエラーが発生する。
  • template オプションが欠落している場合、ERRCODE_INVALID_OBJECT_DEFINITION でエラーとなる。
  • verify_dictoptions による検証に失敗した場合、エラーが発生する。

保守担当者向けメモ

  • 本関数は DDL 処理の一部であり、pg_dump による再構築をサポートするため、オプションのシリアライズ形式（serialize_deflist）には注意が必要である。
  • 依存関係の管理は makeDictionaryDependencies に委譲されているため、カタログスキーマ変更時は同関数も確認すること。
