get_dtype

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/type.c:671

目的

ECPGプリプロセッサにおいて、列挙型 ECPGdtype の値を対応する文字列リテラルに変換し、デバッグ出力やコード生成時の識別子として利用可能にする。

シグネチャまたは形状

const char *get_dtype(enum ECPGdtype type)

入力と出力

  • **入力**: enum ECPGdtype 型の列挙値。
  • **出力**: 対応する列挙値の名称を表す const char * 型の文字列。一致する値がない場合は NULL を返し、エラーを報告する。

前提条件と不変条件

  • 入力される type は ECPGdtype 列挙型で定義された有効な値であること。
  • 予期しない値が渡された場合、mmerror を通じて PARSE_ERROR が発生する。

アルゴリズムまたは振る舞い

1. switch 文を用いて、入力された type を評価する。 2. ECPGd_count, ECPGd_data, ECPGd_indicator 等、定義された各列挙値に対して対応する文字列（例: "ECPGd_count"）を返す。 3. default ケースでは、mmerror 関数を呼び出し、認識できない記述子アイテムコードであることをエラーログとして出力する。 4. 正常終了時は文字列を返し、エラー時は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/descriptor.c:196
  • src/interfaces/ecpg/preproc/descriptor.c:306
  • **呼び出し先**:
  • mmerror (エラー発生時)

関連データ構造

  • enum ECPGdtype (src/interfaces/ecpg/preproc/type.h で定義)

失敗モードと境界条件

  • ECPGdtype に定義されていない整数値が渡された場合、mmerror により解析エラーとして処理され、関数は NULL を返す。

保守担当者向けメモ

  • 新しい記述子アイテムコードを ECPGdtype に追加した場合は、必ず本関数の switch 文にもケースを追加すること。
  • 戻り値の文字列は、ECPGライブラリが実行時に解釈する識別子と一致している必要があるため、変更時はライブラリ側の実装との整合性に注意すること。
