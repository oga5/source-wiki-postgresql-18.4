pvsnprintf

定義

  • 種別: function
  • 定義箇所: src/common/psprintf.c:102

目的

vsnprintf のラッパーとして、バッファサイズ不足時に必要なメモリ量を推定し、呼び出し側に再試行のためのサイズを通知します。

シグネチャまたは形状

size_t pvsnprintf(char *buf, size_t len, const char *fmt, va_list args);

入力と出力

  • **入力**:
  • buf: フォーマット済み文字列を格納するバッファ。
  • len: buf のサイズ。
  • fmt: フォーマット文字列。
  • args: 可変長引数リスト。
  • **出力**:
  • 成功時: 書き込まれたバイト数（終端文字を含まない）。
  • バッファ不足時: 必要なバッファサイズ（終端文字を含む）。

前提条件と不変条件

  • vsnprintf が C99 準拠であることを前提としています。
  • 呼び出し側は errno を保持しておく必要があります（fmt 内の %m 処理のため）。
  • MaxAllocSize を超えるメモリ確保が必要な場合はエラーとして扱われます。

アルゴリズムまたは振る舞い

1. vsnprintf を実行し、結果を nprinted に格納します。 2. nprinted < 0 の場合、フォーマットエラーとして環境に応じた終了処理（elog(ERROR) または exit(EXIT_FAILURE)）を行います。 3. nprinted < len の場合、書き込み成功として nprinted を返します。 4. nprinted >= len の場合、必要なサイズが MaxAllocSize を超えていないか確認し、超えていればエラー終了します。 5. 正常な不足サイズとして nprinted + 1 を返します。

関連データ構造

  • MaxAllocSize (src/include/c.h で定義)

失敗モードと境界条件

  • vsnprintf が負の値を返した場合、フォーマット文字列が不正とみなされプロセスが終了します。
  • 必要なバッファサイズが MaxAllocSize - 1 を超える場合、メモリ不足エラーとしてプロセスが終了します。

保守担当者向けメモ

  • 本関数はエラー時にプロセスを終了させる可能性があるため、ライブラリ（libpq 等）内での利用には注意が必要です。
  • psprintf からの呼び出しが主ですが、StringInfo の構築時にも利用されています。
