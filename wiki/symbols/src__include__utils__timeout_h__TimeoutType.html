TimeoutType

定義

  • 種別: enum
  • 定義箇所: src/include/utils/timeout.h:51

目的

タイムアウトイベントのスケジューリング方式を識別するために使用されます。

シグネチャまたは形状

typedef enum TimeoutType
{
	TMPARAM_AFTER,
	TMPARAM_AT,
	TMPARAM_EVERY,
} TimeoutType;

入力と出力

  • 入力: タイムアウト設定関数（enable_timeout_after 等）の引数として指定されます。
  • 出力: タイムアウトの動作モードを決定する列挙値。

前提条件と不変条件

  • TMPARAM_AFTER: 現在時刻からの相対時間でタイムアウトを設定します。
  • TMPARAM_AT: 特定の絶対時刻でタイムアウトを設定します。
  • TMPARAM_EVERY: 定期的な間隔でタイムアウトを繰り返します。

アルゴリズムまたは振る舞い

バックエンドのタイムアウト管理において、指定された TimeoutType に基づき、src/backend/utils/misc/timeout.c 内で終了時刻の計算やスケジューリングが行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/utils/timeout.h 内の EnableTimeoutParams 構造体でメンバとして使用されています。
  • 呼び出し先: 提供されたソース情報からは、具体的な関数呼び出し関係は特定できません。

関連データ構造

  • EnableTimeoutParams: タイムアウト設定時に本型を保持します。
  • TimeoutId: タイムアウトの理由を識別する列挙型。

失敗モードと境界条件

  • 適切な TimeoutType が指定されない場合、タイムアウトの計算やスケジューリングが意図通りに動作しない可能性があります。

保守担当者向けメモ

  • 本列挙型は src/backend/utils/misc/timeout.c におけるタイムアウト処理のロジック分岐の基盤となります。変更時は、関連するすべての enable_timeout_* 系関数の実装を確認してください。
