pg_send_history

定義

  • 種別: function
  • 定義箇所: src/bin/psql/input.c:134

目的

入力されたクエリを履歴バッファから取り出し、readline の履歴リストへ追加（コミット）する。

シグネチャまたは形状

void pg_send_history(PQExpBuffer history_buf)

入力と出力

  • **入力**: history_buf (PQExpBuffer) - 履歴として保存すべき文字列データを保持するバッファ。
  • **出力**: なし（副作用として readline の履歴リストを更新し、history_buf をリセットする）。

前提条件と不変条件

  • USE_READLINE が定義されている環境でのみ動作する。
  • history_buf は有効な PQExpBuffer 構造体へのポインタであること。
  • pset.histcontrol 設定に基づき、重複排除やスペース開始行の無視が行われる。

アルゴリズムまたは振る舞い

1. history_buf 内の文字列末尾にある改行文字（\n）を削除する。 2. useHistory が有効かつ文字列が空でない場合、以下のフィルタリングを行う：

  • hctl_ignorespace が設定されており、文字列がスペースで始まる場合は無視する。
  • hctl_ignoredups が設定されており、直前の履歴と同一の場合は無視する。

    • 3. フィルタを通過した場合、add_history() を呼び出して履歴リストへ追加し、history_lines_added をインクリメントする。 4. 最後に resetPQExpBuffer() を呼び出し、渡されたバッファを空にする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/mainloop.c 内の複数箇所（クエリ実行完了時など）。
  • **呼び出し先**: add_history (readline ライブラリ), resetPQExpBuffer (libpq), pg_strdup, free。

関連データ構造

  • PQExpBuffer: 履歴文字列を保持する可変長バッファ。
  • pset: psql のグローバル設定を保持する構造体（histcontrol を参照）。

失敗モードと境界条件

  • history_buf が空の場合、履歴への追加は行われない。
  • readline が無効な環境では、この関数は何も行わない（空の関数として定義される）。
  • prev_hist は静的変数として保持され、履歴の重複チェックに使用される。

保守担当者向けメモ

  • readline の履歴リストへの追加処理は USE_READLINE マクロで囲まれているため、ビルド環境によって挙動が異なる点に注意すること。
  • history_buf の内容を直接書き換えて末尾の改行を削除しているため、呼び出し元でバッファの内容を再利用する際は注意が必要。
