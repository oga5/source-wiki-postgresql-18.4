r_elisions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_french.c:421

目的

フランス語のステミング処理において、単語内の母音省略（エリジオン）箇所を特定し、アポストロフィ以降の接頭辞部分を削除する。

シグネチャまたは形状

static int r_elisions(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境構造体）
  • 出力: 成功時に 1、失敗または条件不一致時に 0、エラー発生時に負の値を返す。

前提条件と不変条件

  • z が有効な SN_env 構造体を指していること。
  • z->p（文字列バッファ）および z->c（カーソル位置）が適切に初期化されていること。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置を z->bra に設定する。 2. g_elision_char グループに含まれる文字を確認するか、特定の文字列（s_0）との一致を確認する。 3. 条件を満たした場合、カーソル位置が文字列の末尾でないこと、かつ現在の文字がアポストロフィ（'）であることを確認する。 4. アポストロフィの直後を z->ket に設定し、slice_del(z) を呼び出して該当箇所を削除する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_french.c および src/backend/snowball/libstemmer/stem_ISO_8859_1_french.c 内のメイン処理フローから参照されている。
  • 呼び出し先: in_grouping_U、eq_s、slice_del（いずれも header.h 経由のランタイム関数）。

関連データ構造

  • struct SN_env: ステミングの実行状態（バッファ、カーソル、領域境界など）を保持する。

失敗モードと境界条件

  • カーソルが文字列の末尾に達している場合、または現在の文字がアポストロフィでない場合は 0 を返す。
  • slice_del がメモリ不足等の理由で負の値を返した場合、その値をそのまま伝播させる。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによって自動生成されたコードである。ロジックの変更が必要な場合は、直接編集せずソースとなる .sbl ファイルを修正すること。
  • デバッグ時は SN_env の c（カーソル）と p（文字列）の状態を監視することで、どの位置でエリジオンが判定されたかを確認できる。
