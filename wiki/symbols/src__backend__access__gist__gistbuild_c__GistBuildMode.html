GistBuildMode

定義

  • 種別: enum
  • 定義箇所: src/backend/access/gist/gistbuild.c:67

目的

GiSTインデックス構築時における、アルゴリズム戦略の選択状態を管理するために使用されます。

シグネチャまたは形状

typedef enum {
    GIST_SORTED_BUILD,
    GIST_BUFFERING_DISABLED,
    GIST_BUFFERING_AUTO,
    GIST_BUFFERING_STATS,
    GIST_BUFFERING_ACTIVE
} GistBuildMode;

入力と出力

  • 本シンボルは列挙型であり、関数への入力や出力として GISTBuildState 構造体の一部（buildMode メンバ）として保持されます。

前提条件と不変条件

  • インデックス構築開始時に、演算子クラスの特性やインデックスサイズに基づいていずれかのモードが選択されます。
  • GIST_BUFFERING_AUTO モードは、インデックスが特定のサイズを超えた場合に自動的に GIST_BUFFERING_ACTIVE へ遷移する可能性があります。

アルゴリズムまたは振る舞い

  • GIST_SORTED_BUILD: 入力タプルをソートし、ボトムアップでインデックスページを構築します。
  • GIST_BUFFERING_DISABLED: バッファリングを行わない通常の構築モード。
  • GIST_BUFFERING_AUTO: 通常モードで開始し、必要に応じてバッファリング構築へ切り替える準備状態。
  • GIST_BUFFERING_STATS: バッファリング構築への切り替え判断のため、タプルサイズの統計を収集する状態。
  • GIST_BUFFERING_ACTIVE: バッファリング構築を実行中の状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/gist/gistbuild.c 内の gistbuild 関数が、構築戦略の決定および管理のためにこの型を参照しています。
  • 呼び出し先: 本シンボル自体は列挙型であり、関数呼び出しは発生しません。

関連データ構造

  • GISTBuildState: 構築中の状態を保持する構造体で、この列挙型をメンバとして含みます。

失敗モードと境界条件

  • 構築戦略の切り替え（AUTO から ACTIVE への遷移）は、インデックスの成長度合いに依存します。これに関連する具体的なエラー条件は、本シンボル定義自体には含まれていません。

保守担当者向けメモ

  • 本列挙型は GiST インデックス構築の制御フローを決定する重要なフラグです。新しい構築戦略を追加する際は、gistbuild 内の条件分岐ロジックと整合性を取る必要があります。
