contain_placeholder_references_walker

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/placeholder.c:507

目的

式ツリーを走査し、指定されたリレーション（relid）を参照する PlaceHolderVar (PHV) が存在するかを判定します。

シグネチャまたは形状

static bool contain_placeholder_references_walker(Node *node, contain_placeholder_references_context *context)

入力と出力

  • **入力**:
  • node: 走査対象のノード。
  • context: 検索対象のリレーションID (relid) と現在のクエリレベル (sublevels_up) を保持する構造体。
  • **出力**:
  • bool: 指定されたリレーションを参照する PHV が見つかった場合は true、それ以外は false。

前提条件と不変条件

  • node が NULL の場合は常に false を返します。
  • context->sublevels_up は、サブクエリへの再帰時に適切に増減される必要があります。

アルゴリズムまたは振る舞い

1. **PHV の判定**: ノードが PlaceHolderVar の場合、現在のクエリレベル (phlevelsup) が context->sublevels_up と一致するか確認します。

  • 一致する場合、phrels ビットマップに context->relid が含まれていれば true を返します。
  • phnullingrels は無視し、PHV 内部の式への再帰も行いません（phrels が依存関係を十分に要約していると見なすため）。

    • 2. **サブクエリの処理**: ノードが Query の場合、sublevels_up をインクリメントして query_tree_walker を呼び出し、再帰的に探索します。 3. **その他のノード**: expression_tree_walker を使用して、子ノードを再帰的に探索します。

呼び出し元と呼び出し先

  • **呼び出し元**: contain_placeholder_references_to (ソースコード内の直接的な参照より)。
  • **呼び出し先**: query_tree_walker および expression_tree_walker。

関連データ構造

  • PlaceHolderVar: 評価対象のノード。
  • contain_placeholder_references_context: 検索条件を保持するコンテキスト構造体。

失敗モードと境界条件

  • node が NULL の場合は安全に false を返します。
  • サブクエリ内の PHV を探索する際、sublevels_up の管理が正しく行われないと、誤ったクエリレベルの PHV を参照する可能性があります。

保守担当者向けメモ

  • 本関数は phrels のビットマップのみを信頼して判定を行うため、PHV 内部の式を再帰的に探索する必要はありません。
  • phnullingrels を評価対象から除外している点は、この関数の仕様として意図されたものです。
