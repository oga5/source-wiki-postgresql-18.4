TrackFunctionsLevel

定義

  • 種別: enum
  • 定義箇所: src/include/pgstat.h:38

目的

PostgreSQL における関数実行統計の収集レベルを制御するための列挙型です。

シグネチャまたは形状

typedef enum TrackFunctionsLevel
{
    TRACK_FUNC_OFF,
    TRACK_FUNC_PL,
    TRACK_FUNC_ALL,
} TrackFunctionsLevel;

入力と出力

  • 入力: なし（設定値として使用される列挙型）
  • 出力: なし

前提条件と不変条件

  • TRACK_FUNC_OFF: 関数統計の収集を無効化します。
  • TRACK_FUNC_PL: プロシージャ言語（PL）で記述された関数のみ統計を収集します。
  • TRACK_FUNC_ALL: すべての関数（SQL関数およびPL関数）の統計を収集します。

アルゴリズムまたは振る舞い

統計サブシステムにおいて、どの範囲の関数呼び出しを計測対象とするかを決定するフラグとして機能します。この値に基づいて、バックエンドプロセスは関数実行時の統計カウンタ更新を行うか否かを判断します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース excerpt には具体的な呼び出し元は記載されていません。
  • 呼び出し先: なし（データ定義型のため）。

関連データ構造

  • PgStat_FunctionCounts: 関数統計を保持する構造体。
  • PgStat_FunctionCallUsage: 関数呼び出しの利用状況を追跡する構造体。

失敗モードと境界条件

  • 列挙型以外の値が設定された場合の動作は定義されていません。

保守担当者向けメモ

  • 本シンボルは統計収集の有効範囲を決定する重要な設定値です。新しい統計収集対象の関数種別を追加する場合は、この列挙型に新しいメンバを追加し、統計収集ロジック側で条件分岐を更新する必要があります。
