buildACLCommands

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/dumputils.c:103

目的

データベースオブジェクトの現在の ACL（アクセス制御リスト）とベース ACL を比較し、必要な GRANT および REVOKE SQL 文を生成して PQExpBuffer に格納する。

シグネチャまたは形状

bool buildACLCommands(const char *name, const char *subname, const char *nspname,
                      const char *type, const char *acls, const char *baseacls,
                      const char *owner, const char *prefix, int remoteVersion,
                      PQExpBuffer sql);

入力と出力

  • **入力**:
  • name, subname, nspname: オブジェクトの識別情報。
  • type: オブジェクトの種類（例: "TABLE"）。
  • acls, baseacls: 比較対象の ACL 文字列。
  • owner: オブジェクトの所有者。
  • prefix: 生成される SQL 文の接頭辞。
  • remoteVersion: 接続先データベースのバージョン。
  • **出力**:
  • sql: 生成された SQL 文が追記されるバッファ。
  • 戻り値: 成功時に true、解析失敗時に false を返す。

前提条件と不変条件

  • acls が NULL または空文字列の場合、デフォルト権限とみなされ、何も処理せず true を返す。
  • owner が空文字列の場合は NULL として扱う。
  • ACL 文字列は aclitemout() の出力形式であることを前提とする。

アルゴリズムまたは振る舞い

1. parsePGArray を用いて acls と baseacls を解析し、配列化する。 2. 両者の差分を抽出し、grantitems（追加分）と revokeitems（削除分）を特定する。 3. revokeitems に対して parseAclItem を呼び出し、REVOKE 文を構築する。 4. grantitems に対して parseAclItem を呼び出し、GRANT 文を構築する。 5. 所有者の権限付与を他の権限より優先して出力するよう制御する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c および src/bin/pg_dump/pg_dumpall.c 内の各ダンプ処理関数。
  • **呼び出し先**: parsePGArray, parseAclItem, createPQExpBuffer, fmtId, appendPQExpBuffer 等。

関連データ構造

  • PQExpBuffer: SQL 文構築用バッファ。

失敗モードと境界条件

  • parsePGArray が失敗した場合、メモリを解放して false を返す。
  • parseAclItem が失敗した場合、処理を中断し false を返す。
  • ACL の比較は文字列一致で行われるため、空白等の微細な差異により冗長な SQL が生成される可能性がある。

保守担当者向けメモ

  • 本関数内の ACL 解析ロジックは、PostgreSQL のバージョン間で ACL 形式が変更された場合に調整が必要となる。
  • grantor が所有者でない場合、SET SESSION AUTHORIZATION を用いた権限委譲の処理が含まれる。
