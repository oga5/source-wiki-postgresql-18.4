gistSplitHalf

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistsplit.c:584

目的

GiSTインデックスのページ分割において、指定されたタプル群を強制的に2つのグループへ均等に分割するフォールバック処理を行います。

シグネチャまたは形状

static void gistSplitHalf(GIST_SPLITVEC *v, int len)

入力と出力

  • **入力**:
  • v: 分割結果を格納する GIST_SPLITVEC 構造体へのポインタ。
  • len: 分割対象となるタプルの総数。
  • **出力**:
  • なし（v が指す構造体の spl_left, spl_right, spl_nleft, spl_nright フィールドが更新されます）。

前提条件と不変条件

  • v は有効な GIST_SPLITVEC 構造体を指している必要があります。
  • len は正の整数である必要があります。
  • 本関数はユニオンキーの計算を行いません（呼び出し元が責任を持つ前提です）。

アルゴリズムまたは振る舞い

1. v->spl_left および v->spl_right に対して、len 分の OffsetNumber 配列をメモリ確保します。 2. 1 から len までのインデックスをループ処理します。 3. インデックスが len / 2 未満であれば spl_right に、それ以外であれば spl_left にタプルを割り当てます。 4. 各グループのカウント（spl_nright, spl_nleft）を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistsplit.c 内の genericPickSplit 等から呼び出されます。
  • **呼び出し先**: palloc（PostgreSQLのメモリ割り当て関数）を呼び出します。

関連データ構造

  • GIST_SPLITVEC: GiSTの分割結果（左右のタプル集合とカウント）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc によりエラーが発生します。
  • len が極端に小さい場合、分割結果が偏る可能性がありますが、アルゴリズム上は正常に動作します。

保守担当者向けメモ

  • 本関数は、ユーザー定義の picksplit 関数が適切な分割を提供できなかった場合の安全装置として機能します。
  • 割り当てられたメモリは呼び出し元で解放される必要があります。
