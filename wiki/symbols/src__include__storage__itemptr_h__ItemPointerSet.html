ItemPointerSet

定義

  • 種別: static inline function
  • 定義箇所: src/include/storage/itemptr.h:134

目的

ItemPointerData 構造体に対して、ブロック番号とオフセット番号をセットし、有効なタプル識別子（TID）として初期化または更新を行う。

シグネチャまたは形状

static inline void ItemPointerSet(ItemPointerData *pointer, BlockNumber blockNumber, OffsetNumber offNum)

入力と出力

  • **入力**:
  • pointer: 設定対象となる ItemPointerData 構造体へのポインタ。
  • blockNumber: 設定するブロック番号（BlockNumber）。
  • offNum: 設定するオフセット番号（OffsetNumber）。
  • **出力**: なし（副作用として pointer が指すメモリの内容が更新される）。

前提条件と不変条件

  • pointer は有効なメモリ領域を指している必要がある（Assert(PointerIsValid(pointer)) により検証される）。
  • blockNumber および offNum は、PostgreSQL のストレージレイアウトにおける有効な範囲内であること。

アルゴリズムまたは振る舞い

1. pointer が有効なポインタであるか検証する。 2. BlockIdSet マクロを使用して、pointer 内の ip_blkid フィールドに blockNumber を書き込む。 3. pointer 内の ip_posid フィールドに offNum を直接代入する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_surgery/heap_surgery.c
  • contrib/pg_visibility/pg_visibility.c
  • contrib/pgstattuple/pgstatapprox.c
  • src/backend/access/brin/brin_pageops.c
  • src/backend/access/brin/brin_revmap.c
  • src/backend/access/brin/brin_xlog.c
  • src/backend/access/gin/ginget.c
  • **呼び出し先**:
  • BlockIdSet（マクロ）

関連データ構造

  • ItemPointerData: TID を保持する 6 バイトの構造体。
  • BlockNumber: ブロック番号を表現する型。
  • OffsetNumber: ページ内のオフセットを表現する型。

失敗モードと境界条件

  • pointer が NULL または無効なアドレスである場合、Assert が有効なビルドではプロセスが停止する。
  • offNum に MaxOffsetNumber を超える特殊値（SpecTokenOffsetNumber 等）を渡した場合、通常のタプル位置ではなく特殊な状態として扱われる。

保守担当者向けメモ

  • 本関数はインライン化されており、パフォーマンスが重視されるパスで使用される。
  • ItemPointerData のメモリレイアウトは 6 バイトに固定されているため、構造体のフィールド配置を変更する際はパディングが発生しないよう注意が必要である。
