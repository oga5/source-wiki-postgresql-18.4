cube_out

定義

  • 種別: function
  • 定義箇所: contrib/cube/cube.c:295

目的

PostgreSQL の cube 型（多次元ボックス）を、ユーザーが読み取り可能な文字列形式に変換（シリアライズ）します。

シグネチャまたは形状

Datum cube_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_NDBOX_P(0) を介して取得される NDBOX 構造体へのポインタ。
  • **出力**: PG_RETURN_CSTRING を介して返される、ボックスの座標を表現した文字列（char *）。

前提条件と不変条件

  • 入力された NDBOX は有効なメモリ領域にあること。
  • DIM(cube) マクロにより次元数が正しく取得できること。
  • float8out_internal 関数が各座標値の文字列変換に使用可能であること。

アルゴリズムまたは振る舞い

1. StringInfoData バッファを初期化します。 2. LL_COORD マクロを使用して左下隅の座標を順次取得し、(x1, x2, ...) の形式でバッファに書き込みます。 3. cube_is_point_internal を呼び出し、対象が点（point）であるか判定します。 4. 点でない場合、さらに UR_COORD マクロを使用して右上隅の座標を取得し、,(y1, y2, ...) の形式でバッファに追加します。 5. 最後に、構築された文字列を PG_RETURN_CSTRING で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の型システム（PG_FUNCTION_INFO_V1(cube_out) による登録経由）。
  • **呼び出し先**: initStringInfo, appendStringInfoChar, appendStringInfoString, float8out_internal, LL_COORD, UR_COORD, cube_is_point_internal, PG_FREE_IF_COPY。

関連データ構造

  • NDBOX: 多次元ボックスを保持する主要なデータ構造。
  • StringInfoData: 文字列構築用の一時バッファ。

失敗モードと境界条件

  • メモリ不足時には initStringInfo 等の内部関数がエラーを発生させる可能性があります。
  • cube が NULL の場合、PG_GETARG_NDBOX_P の仕様に従い動作します。

保守担当者向けメモ

  • 本関数は cube 型の出力関数として登録されています。
  • 座標の出力形式を変更する場合は、float8out_internal の呼び出し部分と文字列連結ロジックを調整してください。
  • cube_is_point_internal の判定ロジックは、cube 型の定義（点かボックスか）に依存しているため、データ構造の変更時には注意が必要です。
