OutputPluginPrepareWrite

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/replication/logical/logical.c:693

目的

論理デコーディングの出力プラグインに対し、データの書き込み準備を通知します。

シグネチャまたは形状

void OutputPluginPrepareWrite(struct LogicalDecodingContext *ctx, bool last_write)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト構造体。
  • last_write: 今回の書き込みがトランザクションの最後であるかを示すフラグ。
  • **出力**: なし（副作用として ctx の状態を更新）。

前提条件と不変条件

  • ctx->accept_writes が真である必要があります。偽の場合、エラーが発生します。
  • 呼び出し後、ctx->prepared_write が true に設定されます。

アルゴリズムまたは振る舞い

1. ctx->accept_writes を確認し、許可されていない場合はエラーを投げます。 2. ctx->prepare_write コールバックを呼び出し、現在の書き込み位置 (ctx->write_location)、トランザクションID (ctx->write_xid)、および last_write フラグを渡します。 3. ctx->prepared_write を true に設定し、書き込み準備が完了したことを記録します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/test_decoding/test_decoding.c 内の各コールバック関数（test_begin_cb, test_change_cb 等）から頻繁に呼び出されています。
  • **呼び出し先**: ctx->prepare_write（LogicalDecodingContext 内で定義された関数ポインタ）。

関連データ構造

  • LogicalDecodingContext: 論理デコーディングの状態を保持する主要な構造体。

失敗モードと境界条件

  • accept_writes が偽の状態で呼び出された場合、elog(ERROR, ...) により処理が中断されます。これは、書き込みが許可されていないコールバック（begin, commit, change以外）から呼び出された場合に発生します。

保守担当者向けメモ

  • 本関数は、出力プラグインがデータを書き出す直前に必ず呼び出す必要があります。
  • ctx->prepared_write フラグは、後続の OutputPluginWrite 関数で書き込みが適切に行われたかを検証するために使用されます。
