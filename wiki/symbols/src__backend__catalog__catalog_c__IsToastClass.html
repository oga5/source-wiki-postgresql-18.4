IsToastClass

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/catalog.c:225

目的

指定されたリレーションのタプル情報に基づき、そのリレーションがTOASTテーブルであるかどうかを判定します。

シグネチャまたは形状

bool IsToastClass(Form_pg_class reltuple)

入力と出力

  • **入力**: Form_pg_class reltuple（pg_class システムカタログのタプル構造体へのポインタ）
  • **出力**: bool 型。対象がTOAST名前空間に属していれば true、そうでなければ false を返します。

前提条件と不変条件

  • reltuple は有効な Form_pg_class 構造体を指している必要があります。
  • 本関数はカタログアクセスを行わない設計となっており、ロック取得中や再帰的な呼び出しにおいても安全に実行可能です。

アルゴリズムまたは振る舞い

1. 入力された reltuple から relnamespace（名前空間OID）を取得します。 2. IsToastNamespace 関数を呼び出し、その名前空間がTOAST用であるかを判定して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/catalog.c:89（IsSystemRelation 内での判定の一部として使用）
  • **呼び出し先**:
  • IsToastNamespace（名前空間の判定に使用）

関連データ構造

  • Form_pg_class: pg_class システムカタログの行データを保持する構造体。

失敗モードと境界条件

  • reltuple が NULL の場合、ポインタ参照によりクラッシュする可能性があります。

保守担当者向けメモ

  • 本関数は、システムカタログの分類ロジックの一部です。判定基準を変更する場合は、IsToastNamespace の定義と整合性を保つ必要があります。
