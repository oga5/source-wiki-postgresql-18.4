OSAPerGroupState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/orderedsetaggs.c:92

目的

順序集合集約関数（Ordered-set aggregate functions）における、グループ単位の実行状態を保持する構造体です。

シグネチャまたは形状

typedef struct OSAPerGroupState
{
    OSAPerQueryState *qstate;    /* クエリ全体のメタデータへのリンク */
    MemoryContext gcontext;      /* グループ単位のデータを保持するメモリコンテキスト */
    Tuplesortstate *sortstate;   /* データを蓄積するソートオブジェクト */
    int64        number_of_rows; /* sortstate に挿入された通常行の数 */
    bool         sort_done;      /* tuplesort_performsort 実行済みフラグ */
} OSAPerGroupState;

入力と出力

  • **入力**: 集約関数実行時に palloc を介してインスタンス化され、遷移関数（ordered_set_transition 等）を通じてデータが投入されます。
  • **出力**: 最終計算関数（finalfn）が sortstate を利用して統計量を算出し、結果を返します。

前提条件と不変条件

  • sortstate は tuplesort を用いてデータを整列させるために使用されます。
  • gcontext はグループのライフサイクルに紐付いており、集約終了時に適切に解放される必要があります。
  • sort_done は tuplesort_performsort の呼び出し状態を追跡し、重複実行を防ぎます。

アルゴリズムまたは振る舞い

1. **初期化**: ordered_set_startup により生成され、qstate（クエリ共有メタデータ）と関連付けられます。 2. **データ蓄積**: 遷移関数が入力値を sortstate に追加し、number_of_rows を更新します。 3. **ソートと抽出**: 最終計算関数が呼び出された際、sort_done が偽であれば tuplesort_performsort を実行してソートを確定させ、順位に基づいた値を抽出します。 4. **終了**: ordered_set_shutdown コールバックにより、tuplesort_end を介してリソースが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/orderedsetaggs.c 内の各集約関数（ordered_set_transition、percentile_disc_final 等）が、PG_GETARG_POINTER 等を通じて本構造体を取得します。
  • **呼び出し先**: tuplesort API（tuplesort_performsort 等）を呼び出し、データを処理します。

関連データ構造

  • OSAPerQueryState: クエリ全体で共有されるメタデータ（ソート順、型情報など）。
  • Tuplesortstate: utils/tuplesort.h で定義されるソートエンジン。

失敗モードと境界条件

  • **メモリ不足**: tuplesort は work_mem を使用するため、大量のデータを扱う際に一時ファイルが生成される可能性があります。
  • **リソースリーク**: AggRegisterCallback を介したクリーンアップが正しく行われない場合、メモリリークが発生します。

保守担当者向けメモ

  • 本構造体はグループのライフサイクルに依存します。
  • 仮説的集約（AGGKIND_HYPOTHETICAL）の場合、ソート対象にフラグカラムが含まれる点に留意してください。
