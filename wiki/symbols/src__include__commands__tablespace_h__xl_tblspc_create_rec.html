xl_tblspc_create_rec

定義

  • 種別: struct
  • 定義箇所: src/include/commands/tablespace.h:30

目的

テーブルスペース作成操作をWAL（Write-Ahead Logging）に記録するための物理レコード構造体です。クラッシュリカバリ時に、物理ディレクトリの作成操作を再実行するために使用されます。

シグネチャまたは形状

typedef struct xl_tblspc_create_rec
{
    Oid         ts_id;
    char        ts_path[FLEXIBLE_ARRAY_MEMBER];
} xl_tblspc_create_rec;

入力と出力

  • **入力**: テーブルスペースのOID (ts_id) およびパス文字列 (ts_path) を保持します。
  • **出力**: WALログの一部として書き込まれ、リカバリ時に tblspc_redo 関数によって読み取られます。

前提条件と不変条件

  • ts_path はヌル終端文字列である必要があります。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズは可変であり、メモリ確保時には ts_path の長さを考慮する必要があります。

アルゴリズムまたは振る舞い

1. テーブルスペース作成時、この構造体にOIDとパスを格納し、WALレコードとして書き出します。 2. リカバリ時、XLogRecGetData を通じてWALレコードから本構造体へキャストされ、ts_id と ts_path が抽出されます。 3. 抽出された情報に基づき、物理的なディレクトリ作成操作が再現されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablespace.c: テーブルスペース作成処理内でレコードの構築および書き込みに使用。
  • **呼び出し先**:
  • src/backend/access/rmgrdesc/tblspcdesc.c: WALレコードのデバッグ出力（記述）に使用。
  • src/backend/commands/tablespace.c: リカバリ時の tblspc_redo 処理内でレコードの読み取りに使用。

関連データ構造

  • xl_tblspc_drop_rec: テーブルスペース削除操作用の対となるWALレコード構造体。

失敗モードと境界条件

  • ts_path の長さがファイルシステムの制限を超えている場合、ディレクトリ作成時に失敗する可能性があります。
  • WALレコードの読み取り時に、データが破損している場合はキャスト後の値が不正になる可能性があります。

保守担当者向けメモ

  • 本構造体は可変長配列を含むため、sizeof(xl_tblspc_create_rec) を使用する際は注意してください。offsetof(xl_tblspc_create_rec, ts_path) を用いたオフセット計算が src/backend/commands/tablespace.c で行われています。
