format_operator_parts

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/regproc.c:805

目的

指定された演算子OIDに対応する名前空間、演算子名、および引数の型情報を抽出し、オブジェクト識別用のリスト形式に変換します。

シグネチャまたは形状

void format_operator_parts(Oid operator_oid, List **objnames, List **objargs, bool missing_ok)

入力と出力

  • **入力**:
  • operator_oid: 対象となる演算子のOID。
  • missing_ok: true の場合、OIDが見つからなくてもエラーを発生させず終了します。
  • **出力**:
  • *objnames: {スキーマ名, 演算子名} を含むリストが格納されます。
  • *objargs: 演算子の左オペランド型および右オペランド型の修飾名リストが格納されます（存在しない場合は NIL）。

前提条件と不変条件

  • pg_operator システムカタログへのアクセスが必要です。
  • missing_ok が false の場合、指定されたOIDがシステムカタログに存在しないと elog(ERROR) が発生します。

アルゴリズムまたは振る舞い

1. OPEROID システムキャッシュを使用して、指定された operator_oid に対応する pg_operator のタプルを検索します。 2. タプルが見つからない場合、missing_ok に応じてエラーを投げるか、処理を終了します。 3. タプルから演算子の名前空間（oprnamespace）と演算子名（oprname）を取得し、objnames に設定します。 4. 演算子の左オペランド（oprleft）および右オペランド（oprright）が存在する場合、それぞれ format_type_be_qualified を用いて型名を解決し、objargs リストに追加します。 5. 最後に ReleaseSysCache を呼び出し、取得したタプルを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c:5129
  • **呼び出し先**:
  • SearchSysCache1
  • get_namespace_name_or_temp
  • format_type_be_qualified
  • ReleaseSysCache

関連データ構造

  • pg_operator: 演算子の定義を保持するシステムカタログ。
  • List: PostgreSQLのリスト構造（objnames および objargs 用）。

失敗モードと境界条件

  • 指定されたOIDがカタログに存在せず、かつ missing_ok が false の場合、キャッシュルックアップ失敗としてエラーが発生します。
  • オペランドが存在しない（単項演算子など）場合、対応する objargs の要素はスキップされます。

保守担当者向けメモ

  • 本関数は pg_operator の構造に依存しています。カタログ定義に変更があった場合は、oprleft や oprright の取得ロジックを確認してください。
  • format_type_be_qualified を使用しているため、型名の出力形式は現在の検索パスやスキーマ設定に依存します。
