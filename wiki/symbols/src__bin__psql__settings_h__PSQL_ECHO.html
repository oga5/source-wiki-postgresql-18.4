PSQL_ECHO

定義

  • 種別: enum
  • 定義箇所: src/bin/psql/settings.h:41

目的

psql におけるクエリやコマンドの出力（エコー）レベルを制御するための列挙型です。

シグネチャまたは形状

typedef enum
{
	PSQL_ECHO_NONE,
	PSQL_ECHO_QUERIES,
	PSQL_ECHO_ERRORS,
	PSQL_ECHO_ALL,
} PSQL_ECHO;

入力と出力

  • 入力: なし（列挙定数）
  • 出力: なし（列挙定数）

前提条件と不変条件

  • PsqlSettings 構造体内の echo フィールドの型として使用されます。
  • 4つの値（NONE, QUERIES, ERRORS, ALL）のいずれかをとります。

アルゴリズムまたは振る舞い

  • PSQL_ECHO_NONE: エコーを行いません。
  • PSQL_ECHO_QUERIES: 実行されるクエリをエコーします。
  • PSQL_ECHO_ERRORS: エラーが発生した場合のみエコーします。
  • PSQL_ECHO_ALL: すべての入力をエコーします。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/settings.h 内の PsqlSettings 構造体定義（176行目）でフィールド型として参照されています。
  • 呼び出し先: 提供されたソース情報には、この列挙型を利用して具体的なエコー処理を行う関数や呼び出し先に関する情報は含まれていません。

関連データ構造

  • PsqlSettings: psql の全状態を保持する構造体であり、この列挙型をメンバとして保持します。

失敗モードと境界条件

  • 列挙型定義のため、定義外の値が代入された場合の挙動は、利用側の実装に依存します。

保守担当者向けメモ

  • psql の設定変更（\set ECHO 等）に伴い、この列挙値がどのように更新・参照されるかについては、variables.c および command.c の実装を確認してください。
