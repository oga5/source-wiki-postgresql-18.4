io_stat_col

定義

  • 種別: enum
  • 定義箇所: src/backend/utils/adt/pgstatfuncs.c:1324

目的

IO統計情報を出力する際の列（カラム）を識別するための列挙型です。pg_stat_io ビュー等の統計情報取得処理において、各データ項目をマッピングするために使用されます。

シグネチャまたは形状

typedef enum io_stat_col {
    IO_COL_INVALID = -1,
    IO_COL_BACKEND_TYPE,
    IO_COL_OBJECT,
    IO_COL_CONTEXT,
    IO_COL_READS,
    IO_COL_READ_BYTES,
    IO_COL_READ_TIME,
    IO_COL_WRITES,
    IO_COL_WRITE_BYTES,
    IO_COL_WRITE_TIME,
    IO_COL_WRITEBACKS,
    IO_COL_WRITEBACK_TIME,
    IO_COL_EXTENDS,
    IO_COL_EXTEND_BYTES,
    IO_COL_EXTEND_TIME,
    IO_COL_HITS,
    IO_COL_EVICTIONS,
    IO_COL_REUSES,
    IO_COL_FSYNCS,
    IO_COL_FSYNC_TIME,
    IO_COL_RESET_TIME,
    IO_NUM_COLUMNS,
} io_stat_col;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし（列挙型定義）

前提条件と不変条件

  • IO_COL_INVALID は無効な列を示し、-1 で初期化されます。
  • IO_NUM_COLUMNS は列挙型の末尾に配置され、列の総数を表すために使用されます。

アルゴリズムまたは振る舞い

統計情報の構築プロセスにおいて、特定のIO操作やメトリクスをこの列挙値にマッピングすることで、SQLインターフェースへ返す結果セットの列順序や識別を制御します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/pgstatfuncs.c 内の統計情報取得関数群（static な関数内で参照されています）。
  • 呼び出し先: なし（データ型定義のため）。

関連データ構造

  • PgStat_StatTabEntry 等の統計情報構造体（本列挙型を用いて各フィールドへアクセスする実装が想定されます）。

失敗モードと境界条件

  • IO_COL_INVALID を使用して、未定義または無効な列へのアクセスを表現します。
  • 新しいIO操作を追加する際は、本列挙型に新しいメンバを追加し、対応する処理関数に case 文を追加する必要があります。

保守担当者向けメモ

  • 本列挙型は pg_stat_io ビューの列構成と密接に関連しています。列の追加や順序変更を行う際は、関連するSQLビュー定義および pgstatfuncs.c 内の変換ロジックとの整合性に注意してください。
