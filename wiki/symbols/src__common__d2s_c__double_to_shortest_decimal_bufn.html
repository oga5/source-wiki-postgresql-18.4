double_to_shortest_decimal_bufn

定義

  • 種別: function
  • 定義箇所: src/common/d2s.c:1014

目的

IEEE 754 倍精度浮動小数点数（double）を、精度を損なわない最短の 10 進数文字列へ変換し、指定されたバッファに書き込みます。

シグネチャまたは形状

int double_to_shortest_decimal_bufn(double f, char *result)

入力と出力

  • **入力**:
  • f: 変換対象の double 型浮動小数点数。
  • result: 変換後の文字列を格納するためのバッファ（呼び出し元で十分な領域を確保する必要がある）。
  • **出力**:
  • 戻り値: 書き込まれた文字列の長さ（int 型）。

前提条件と不変条件

  • double_to_bits を用いて f をビット表現に分解します。
  • 特殊値（NaN, Infinity, 0）は、copy_special_str を介して即座に処理されます。
  • 変換には Ryu アルゴリズムが使用され、128 ビット整数演算またはその代替実装が内部的に利用されます。

アルゴリズムまたは振る舞い

1. **デコード**: double_to_bits で符号、仮数、指数を抽出します。 2. **特殊値判定**: 指数が最大値（無限大/NaN）または 0 かつ仮数が 0 の場合、copy_special_str を呼び出して終了します。 3. **変換**:

  • d2d_small_int を試し、小さな整数として表現可能か判定します。
  • 不可能な場合、d2d を呼び出して floating_decimal_64 構造体へ変換します。

    • 4. **フォーマット**: to_chars を呼び出し、結果を result バッファへ書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は src/common/d2s.c 内の他の関数である可能性がありますが、提供された情報には具体的な呼び出し元の関数名は含まれていません。
  • **呼び出し先**: double_to_bits, copy_special_str, d2d_small_int, d2d, to_chars を呼び出します。

関連データ構造

  • floating_decimal_64: 変換過程で仮数と指数を保持するために使用されます。

失敗モードと境界条件

  • 特殊値（NaN, Infinity, 0）は copy_special_str により処理されます。
  • バッファオーバーフローの保護については、本関数単体ではバッファサイズの検証を行わないため、呼び出し元で十分な領域を確保する必要があります。

保守担当者向けメモ

  • 本関数は Ryu アルゴリズムの PostgreSQL 移植版の一部です。
  • 128 ビット演算が利用できない環境では RYU_ONLY_64_BIT_OPS マクロによりパフォーマンスが低下する可能性があります。
  • 変更を加える際は、d2d の丸め処理や to_chars のフォーマットが IEEE 754 仕様と整合しているか確認してください。
