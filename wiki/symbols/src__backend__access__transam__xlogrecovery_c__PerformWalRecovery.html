PerformWalRecovery

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogrecovery.c:1670

目的

WAL（Write-Ahead Log）の適用（Redo）ループを実行し、データベースの状態を整合性のあるポイントまで復元する。

シグネチャまたは形状

void PerformWalRecovery(void)

入力と出力

  • **入力**: 引数なし。グローバル変数 RedoStartLSN, CheckPointLoc, xlogreader, xlogprefetcher 等を参照。
  • **出力**: なし（副作用としてデータベース状態の更新およびリカバリの進行）。

前提条件と不変条件

  • InitWalRecovery が完了し、リカバリモードが確定していること。
  • xlogreader および xlogprefetcher が初期化済みであること。
  • 共有メモリ上の XLogRecoveryCtl がアクセス可能であること。

アルゴリズムまたは振る舞い

1. **初期化**: XLogRecoveryCtl のリカバリ進捗情報を初期化し、Postmasterへリカバリ開始を通知する。 2. **整合性チェック**: CheckRecoveryConsistency を呼び出し、読み取り専用接続の可否を判定する。 3. **開始地点の特定**: RedoStartLSN に基づき、チェックポイント以降の最初のWALレコードを読み込む。 4. **Redoループ**:

  • ProcessStartupProcInterrupts で割り込みを処理。
  • recoveryPauseState を確認し、必要に応じて一時停止する。
  • recoveryStopsBefore でリカバリターゲット到達を判定し、到達していればループを終了。
  • recoveryApplyDelay でラグ設定がある場合は待機。
  • ApplyWalRecord を呼び出し、レコードを適用。
  • recoveryStopsAfter でリカバリターゲット到達を判定し、到達していればループを終了。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlog.c 内のスタートアップ処理から呼び出される。
  • **呼び出し先**: ReadRecord, ApplyWalRecord, RmgrStartup, CheckRecoveryConsistency, ProcessStartupProcInterrupts, recoveryPausesHere, recoveryStopsBefore, recoveryStopsAfter, recoveryApplyDelay 等。

関連データ構造

  • XLogRecoveryCtl: リカバリ状態を保持する共有メモリ構造体。
  • XLogRecord: 適用対象のWALレコード。
  • XLogReaderState: WAL読み込み状態。

失敗モードと境界条件

  • **FATAL**: RedoStartLSN で期待されるレコードタイプ（XLOG_CHECKPOINT_REDO）が見つからない場合、エラー終了する。
  • **リカバリターゲット**: 指定されたターゲット（XID、時刻、LSN）に到達した時点でループを正常終了する。

保守担当者向けメモ

  • 本関数はスタートアッププロセスでのみ実行される。
  • recoveryPauseState のチェックは、パフォーマンスへの影響を抑えるためロックを取得せずに行われる。
  • WAL_DEBUG が有効な場合、適用されるレコードの詳細がログ出力される。
