uuid_send

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/uuid.c:191

目的

PostgreSQL の uuid 型データを、クライアントとの通信に適したバイナリ形式（ネットワークバイトオーダー）に変換して送信する。

シグネチャまたは形状

Datum uuid_send(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを介して渡される uuid 型のデータ（pg_uuid_t 構造体へのポインタ）。
  • **出力**: bytea 型としてシリアライズされたバイナリデータ。

前提条件と不変条件

  • 入力引数として有効な pg_uuid_t 構造体が渡されること。
  • UUID_LEN（16バイト）のデータが正しく保持されていること。

アルゴリズムまたは振る舞い

1. pq_begintypsend を呼び出し、送信用の StringInfoData バッファを初期化する。 2. pq_sendbytes を使用し、uuid->data から 16 バイトのバイナリデータをバッファにコピーする。 3. pq_endtypsend を呼び出してバッファを bytea 型に変換し、その結果を PG_RETURN_BYTEA_P マクロで返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（Fmgr）経由で、SQL レベルのバイナリ出力要求に応じて呼び出される。
  • **呼び出し先**:
  • pq_begintypsend
  • pq_sendbytes
  • pq_endtypsend
  • PG_RETURN_BYTEA_P

    • ※いずれも libpq/pqformat.h で定義されたバイナリプロトコル処理関数。

関連データ構造

  • pg_uuid_t: UUID を保持する 16 バイトの構造体。
  • StringInfoData: 送信データ構築用の一時バッファ。

失敗モードと境界条件

  • メモリ不足時、pq_begintypsend 等の内部関数がエラーを発生させる可能性がある。
  • 入力ポインタが NULL の場合、PG_GETARG_UUID_P マクロの仕様に従い例外が発生する。

保守担当者向けメモ

  • 本関数は RFC 9562 等の UUID 規格に関わらず、単に 16 バイトのバイナリ列をそのまま送信する役割を持つ。
  • バイナリプロトコルの変更が必要な場合は libpq/pqformat.h の規約を確認すること。
