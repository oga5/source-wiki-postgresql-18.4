RecoveryState

定義

  • 種別: enum
  • 定義箇所: src/include/access/xlog.h:89

目的

PostgreSQL の起動およびリカバリプロセスにおける現在のシステム状態を識別するために使用されます。

シグネチャまたは形状

typedef enum RecoveryState
{
    RECOVERY_STATE_CRASH = 0,   /* クラッシュリカバリ中 */
    RECOVERY_STATE_ARCHIVE,     /* アーカイブリカバリ中 */
    RECOVERY_STATE_DONE,        /* 通常稼働中 */
} RecoveryState;

入力と出力

  • **入力**: 状態遷移はシステム起動時のリカバリ処理や、リカバリ完了時の内部状態更新によって行われます。
  • **出力**: GetRecoveryState() 関数を通じて、現在のリカバリ状態を RecoveryState 型の値として取得可能です。

前提条件と不変条件

  • システム起動時、リカバリの必要性に応じて適切な状態に初期化されます。
  • RECOVERY_STATE_DONE は、リカバリ処理が完了し、システムが通常のトランザクション処理を受け入れ可能な状態であることを示します。

アルゴリズムまたは振る舞い

  • RECOVERY_STATE_CRASH: 予期せぬ停止後の再起動時に、WAL を適用して一貫性を回復するプロセスを示します。
  • RECOVERY_STATE_ARCHIVE: 外部アーカイブから WAL を読み込み、特定の時点まで復旧するプロセスを示します。
  • RECOVERY_STATE_DONE: リカバリが完了し、通常の運用モードへ移行したことを示します。

呼び出し元と呼び出し先

  • **参照元**: src/backend/access/transam/xlog.c 内のグローバル変数 SharedRecoveryState として保持されています。
  • **呼び出し先**: src/include/access/xlog.h で宣言されている GetRecoveryState() 関数がこの値を返します。

関連データ構造

  • SharedRecoveryState: xlog.c 内で定義されている、この列挙型を保持する共有状態変数。

失敗モードと境界条件

  • 状態遷移の整合性は xlog.c 内のリカバリロジックに依存します。不適切なタイミングでの状態取得や遷移は、システムの一貫性に影響を及ぼす可能性があります。

保守担当者向けメモ

  • 本列挙型は、リカバリの進行状況を判定するための重要な指標です。新しいリカバリモードを追加する場合は、xlog.c 内の関連する状態遷移ロジックをすべて更新する必要があります。
