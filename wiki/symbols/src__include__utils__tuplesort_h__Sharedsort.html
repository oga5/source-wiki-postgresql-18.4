Sharedsort

定義

  • 種別: struct
  • 定義箇所: src/include/utils/tuplesort.h:39

目的

並列処理環境において、複数のプロセス間でソート状態を共有および調整するための構造体です。

シグネチャまたは形状

typedef struct Sharedsort Sharedsort;

（構造体の内部メンバはソース抜粋に含まれていないため不明です）

入力と出力

  • 入力: shm_toc_allocate を介して共有メモリ領域から割り当てられます。
  • 出力: 共有メモリ上のポインタとして、並列クエリ実行時のリーダーおよびワーカープロセス間で参照されます。

前提条件と不変条件

  • 並列クエリ実行コンテキスト（ParallelContext）内での使用を前提としています。
  • 共有メモリ上の領域（shm_toc）から適切に割り当てられている必要があります。

アルゴリズムまたは振る舞い

  • 並列ソートにおいて、リーダーとワーカーがソートの進捗や状態を同期するために利用されます。
  • 具体的なソートロジックは tuplesort.c に依存し、本構造体はそのための共有状態を保持します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • src/backend/access/brin/brin.c: brin.c 内のインデックス構築処理で、共有メモリの割り当ておよび参照に使用。
  • src/backend/access/gin/gininsert.c: gininsert.c 内のインデックス構築処理で、共有メモリの割り当ておよび参照に使用。
  • **呼び出し先**:
  • ソース抜粋からは、本構造体が直接呼び出す関数や、本構造体を操作する具体的な関数は特定できません。

関連データ構造

  • BrinShared: brin.c における共有状態。
  • GinBuildShared: gininsert.c における共有状態。
  • shm_toc: 共有メモリのテーブル・オブ・コンテンツ。

失敗モードと境界条件

  • 共有メモリの割り当て失敗（shm_toc_allocate が NULL を返す場合など）が発生する可能性があります。
  • 共有メモリのサイズ見積もり（estsort）が不十分な場合、実行時エラーとなる可能性があります。

保守担当者向けメモ

  • 本構造体は不透明な型（opaque type）として定義されています。詳細なメンバ構成を確認するには src/backend/utils/sort/tuplesort.c を参照してください。
  • 並列ソートの実装において、リーダーとワーカー間の同期ロジックが変更された場合、本構造体の定義も影響を受ける可能性があります。
