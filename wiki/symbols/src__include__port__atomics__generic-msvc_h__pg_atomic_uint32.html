pg_atomic_uint32

定義

  • 種別: struct
  • 定義箇所: src/include/port/atomics/generic-msvc.h:34

目的

PostgreSQL のアトミック操作抽象化レイヤーにおいて、32ビット符号なし整数に対するアトミックな読み書きおよび演算操作をサポートするためのデータ構造です。

シグネチャまたは形状

typedef struct pg_atomic_uint32
{
    volatile uint32 value;
} pg_atomic_uint32;

入力と出力

  • 入力: 構造体内部の value フィールドは uint32 型。
  • 出力: なし（データ構造体のため）。

前提条件と不変条件

  • volatile 修飾子により、コンパイラによる最適化（レジスタへのキャッシュ等）を抑制し、メモリ上の値を直接参照することを保証します。
  • 本構造体に対する操作は、src/include/port/atomics/generic-msvc.h で定義された Interlocked ファミリー関数を介して行われる必要があります。

アルゴリズムまたは振る舞い

本構造体は単なるデータコンテナであり、それ自体にロジックは含まれません。実際の振る舞いは、pg_atomic_compare_exchange_u32_impl や pg_atomic_fetch_add_u32_impl などの関数が、本構造体のポインタを引数として受け取り、Windows の Interlocked API を呼び出すことで実現されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/commands/vacuum.c
  • src/backend/commands/vacuumparallel.c
  • src/backend/nodes/tidbitmap.c
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/pgarch.c
  • src/backend/storage/buffer/freelist.c
  • src/backend/storage/ipc/procsignal.c
  • 呼び出し先: なし（データ構造のため）。

関連データ構造

  • pg_atomic_uint64: 64ビット版のアトミック操作用構造体。
  • pg_atomic_flag: フラグ操作用のアトミック構造体。

失敗モードと境界条件

  • 本構造体自体に失敗モードはありません。
  • アトミック操作の成否は、使用する Interlocked API の実装およびメモリ配置に依存します。

保守担当者向けメモ

  • 本構造体は atomics.h を経由して利用されることを前提としています。
  • MSVC 環境下では intrin.h が提供する組み込み関数と組み合わせて使用されます。
  • 構造体内の value に直接アクセスすることは、アトミック性が保証されないため避けてください。必ず提供されている pg_atomic_* 関数群を使用してください。
