pairingheap_dump_recurse

定義

  • 種別: function
  • 定義箇所: src/backend/lib/pairingheap.c:295

目的

ペアリングヒープの構造を再帰的に走査し、各ノードの内容を StringInfo バッファにダンプする。

シグネチャまたは形状

static void pairingheap_dump_recurse(
    StringInfo buf,
    pairingheap_node *node,
    void (*dumpfunc) (pairingheap_node *node, StringInfo buf, void *opaque),
    void *opaque,
    int depth,
    pairingheap_node *prev_or_parent
)

入力と出力

  • buf: 出力先の StringInfo。
  • node: 現在走査中の pairingheap_node。
  • dumpfunc: 個々のノードの情報を文字列化するためのコールバック関数。
  • opaque: dumpfunc に渡される任意のデータポインタ。
  • depth: 現在の再帰深さ（インデント制御に使用）。
  • prev_or_parent: 現在のノードが期待する prev_or_parent ポインタの値。

前提条件と不変条件

  • node->prev_or_parent が prev_or_parent と一致している必要がある（Assert で検証）。

アルゴリズムまたは振る舞い

1. node が非 NULL である限り、兄弟ノードを順次走査する。 2. 現在の深さに応じたインデント（depth * 4）を buf に追加する。 3. dumpfunc を呼び出し、現在のノード情報を buf に書き込む。 4. node->first_child が存在する場合、depth + 1 を指定して再帰的に呼び出す。 5. prev_or_parent を現在のノードに更新し、node を next_sibling に進めてループを継続する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/lib/pairingheap.c 内の pairingheap_dump 関数。
  • pairingheap_dump_recurse 自身（再帰呼び出し）。
  • **呼び出し先**:
  • dumpfunc（引数として渡された関数ポインタ）。
  • appendStringInfoSpaces / appendStringInfoChar（PostgreSQL の文字列操作ユーティリティ）。

関連データ構造

  • pairingheap_node: ヒープの各要素。
  • StringInfo: PostgreSQL の動的文字列バッファ。

失敗モードと境界条件

  • node が NULL の場合、何も行わず即座に終了する。
  • node->prev_or_parent が期待値と異なる場合、アサーション違反によりプロセスが停止する可能性がある（デバッグビルド時）。

保守担当者向けメモ

  • 本関数は PAIRINGHEAP_DEBUG が定義されている場合にのみ有効な pairingheap_dump から利用されるデバッグ用ユーティリティである。
  • dumpfunc は呼び出し側で提供されるため、ヒープ内のノードが保持する具体的なデータ型に応じて適切な表示ロジックを実装すること。
