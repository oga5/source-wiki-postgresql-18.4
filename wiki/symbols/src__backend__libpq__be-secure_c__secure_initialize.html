secure_initialize

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure.c:74

目的

PostgreSQLバックエンドの起動時またはバックエンドプロセス生成時に、セキュア通信（SSL/TLS）のためのライブラリコンテキストを初期化する。

シグネチャまたは形状

int secure_initialize(bool isServerStart);

入力と出力

  • **入力**:
  • isServerStart (bool): サーバー起動時の初期化であれば true、個別のバックエンドプロセス生成時であれば false を指定する。
  • **出力**:
  • int: 成功時は 0 を返し、失敗時は非ゼロの値を返す。SSLが有効でない環境では常に 0 を返す。

前提条件と不変条件

  • USE_SSL マクロが定義されている場合、内部で be_tls_init が呼び出される。
  • 本関数は、通信機能を利用する前に呼び出される必要がある。

アルゴリズムまたは振る舞い

1. USE_SSL が定義されている場合、be_tls_init(isServerStart) を呼び出し、その戻り値を返す。 2. USE_SSL が定義されていない場合、何も行わず 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/postmaster.c: サーバー起動時およびバックエンド生成時に呼び出し。
  • src/backend/tcop/backend_startup.c: バックエンド起動時に呼び出し。
  • **呼び出し先**:
  • be_tls_init (SSL有効時のみ)

関連データ構造

  • なし（本関数はライブラリの初期化を行うラッパーである）。

失敗モードと境界条件

  • USE_SSL が有効な環境において、be_tls_init が内部で失敗した場合、そのエラーコードがそのまま返される。
  • USE_SSL が無効な環境では、常に成功（0）とみなされる。

保守担当者向けメモ

  • 本関数は be-secure.c における抽象化レイヤーの一部であり、具体的なTLS実装は be-secure-openssl.c 等の別ファイルに委譲されている。
  • isServerStart フラグは、SSLコンテキストの初期化において、グローバルな設定とプロセス固有の設定を区別するために使用される。
