astreamer_verify_content

定義

  • 種別: function
  • 定義箇所: src/bin/pg_verifybackup/astreamer_verify.c:90

目的

tarアーカイブのストリーム処理において、各メンバー（ファイル）のヘッダー、コンテンツ、トレーラーの各フェーズに応じた検証処理を制御するメインエントリポイントです。

シグネチャまたは形状

static void astreamer_verify_content(astreamer *streamer, astreamer_member *member, const char *data, int len, astreamer_archive_context context)

入力と出力

  • streamer: 検証状態を保持する astreamer_verify 構造体へのポインタ。
  • member: 現在処理中のアーカイブメンバー情報。
  • data: コンテンツデータ（ASTREAMER_MEMBER_CONTENTS 時のみ有効）。
  • len: data の長さ。
  • context: 現在の処理フェーズを示す astreamer_archive_context 列挙値。
  • 出力: なし（副作用として検証状態の更新およびエラー時の終了）。

前提条件と不変条件

  • context は ASTREAMER_UNKNOWN 以外であること。
  • streamer は astreamer_verify 型にキャスト可能であること。

アルゴリズムまたは振る舞い

context に基づき以下の処理を分岐実行します： 1. **ASTREAMER_MEMBER_HEADER**: member_verify_header を呼び出し、マニフェストとの照合および検証フラグの設定を行う。 2. **ASTREAMER_MEMBER_CONTENTS**:

  • verify_checksum が有効な場合、member_compute_checksum でチェックサムを計算。
  • verify_control_data が有効な場合、member_copy_control_data で制御ファイルをバッファリング。

    • 3. **ASTREAMER_MEMBER_TRAILER**:

  • verify_checksum が有効な場合、member_verify_checksum で最終比較。
  • verify_control_data が有効な場合、member_verify_control_data でCRCおよび識別子を検証。
  • member_reset_info を呼び出し、次のメンバー処理へ向けて状態をリセット。

    • 4. **ASTREAMER_ARCHIVE_TRAILER**: 何も行わない。

呼び出し元と呼び出し先

  • 呼び出し元: astreamer_ops 構造体を通じて、アーカイブストリーミング処理のパイプラインから呼び出されます。
  • 呼び出し先: member_verify_header, member_compute_checksum, member_copy_control_data, member_verify_checksum, member_verify_control_data, member_reset_info, pg_fatal。

関連データ構造

  • astreamer_verify: 検証コンテキストを保持する構造体。
  • astreamer_member: tarアーカイブのメンバー情報。
  • astreamer_archive_context: 処理フェーズを示す列挙型。

失敗モードと境界条件

  • 未定義の context が渡された場合、pg_fatal を呼び出しプログラムを異常終了させます。

保守担当者向けメモ

  • 本関数は astreamer インターフェースの一部であり、astreamer_ops を介して呼び出されるため、直接的な呼び出し元は抽象化されています。
  • 制御ファイルの検証は、マニフェストのバージョンが1より大きい場合にのみ実行される設計となっています。
