RoleMembershipCacheCallback

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/acl.c:5070

目的

システムカタログの変更に伴い、ロールメンバーシップのキャッシュを無効化（クリア）するために使用されるコールバック関数です。

シグネチャまたは形状

static void RoleMembershipCacheCallback(Datum arg, int cacheid, uint32 hashvalue)

入力と出力

  • arg: 未使用。
  • cacheid: 変更されたシステムカタログのキャッシュID。
  • hashvalue: 変更されたエントリのハッシュ値。
  • 戻り値: なし。

前提条件と不変条件

  • 本関数は、システムカタログの更新時にキャッシュ無効化メカニズムから呼び出されることを前提としています。
  • DATABASEOID キャッシュの変更時、現在のデータベースと無関係な変更であれば処理をスキップします。

アルゴリズムまたは振る舞い

1. cacheid が DATABASEOID であり、かつ hashvalue が現在のデータベースのハッシュ値と一致しない場合、処理を中断して戻ります。 2. 上記以外の場合、ロールメンバーシップのキャッシュ状態を保持するグローバル変数 cached_role の各要素（ROLERECURSE_MEMBERS, ROLERECURSE_PRIVS, ROLERECURSE_SETROLE）を InvalidOid に設定します。 3. これにより、次回の権限チェック時にキャッシュが再計算されるよう強制します。

呼び出し元と呼び出し先

  • 呼び出し元: システムカタログのキャッシュ無効化サブシステム（直接の呼び出し元はソースコード上では明示されていませんが、acl.c 内のキャッシュ登録処理から参照されています）。
  • 呼び出し先: なし。

関連データ構造

  • cached_role: ロールメンバーシップの再帰的解決結果を保持する配列。
  • DATABASEOID: データベースのシステムカタログキャッシュID。

失敗モードと境界条件

  • hashvalue が 0 の場合、データベースの変更として無視されず、キャッシュがクリアされます。
  • データベース固有のキャッシュ変更でない場合、無条件にキャッシュがクリアされるため、頻繁なカタログ更新はパフォーマンスに影響を与える可能性があります。

保守担当者向けメモ

  • 本関数は TopMemoryContext に保持されるキャッシュの整合性を維持するために不可欠です。
  • 新たなキャッシュタイプを cached_role に追加した場合は、本関数内で適切に InvalidOid を代入して無効化対象に含めてください。
