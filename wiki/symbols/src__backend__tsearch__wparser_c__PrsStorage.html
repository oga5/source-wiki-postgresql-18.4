PrsStorage

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/wparser.c:153

目的

全文検索の解析処理において、解析済みのトークンリストとその現在の走査位置を保持するための状態管理構造体です。

シグネチャまたは形状

typedef struct
{
	int			cur;            /* 現在の走査位置 */
	int			len;            /* リストの総数 */
	LexemeEntry *list;          /* 解析されたトークンの配列 */
} PrsStorage;

入力と出力

本構造体は関数間で状態を共有するために使用されます。

  • 入力: palloc を用いてメモリ確保され、解析結果のリストが格納されます。
  • 出力: SRF（Set-Returning Function）のコンテキスト（funcctx->user_fctx）を通じて、後続のタプル生成処理に渡されます。

前提条件と不変条件

  • list は LexemeEntry 型の配列を指す必要があります。
  • cur は 0 以上 len 以下の値を保持し、cur < len の間は有効なトークンを指します。

アルゴリズムまたは振る舞い

1. **初期化**: prs_setup_firstcall 等でメモリ確保され、解析結果が list に格納されます。 2. **状態保持**: SRFの呼び出し間で cur をインクリメントすることで、解析結果を一つずつ順次出力します。 3. **終了**: cur が len に達した時点で、処理が完了したとみなされます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tsearch/wparser.c 内の prs_setup_firstcall（初期化時）および prs_process_call（タプル生成時）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • LexemeEntry: PrsStorage が保持するトークン情報の各要素。

失敗モードと境界条件

  • メモリ不足: palloc 失敗時にエラーが発生します。
  • 空の解析結果: len が 0 の場合、cur は即座に終了条件を満たします。

保守担当者向けメモ

  • 本構造体は FuncCallContext の user_fctx にキャストされて格納されます。型安全性を維持するため、アクセス時は必ず PrsStorage * へのキャストを行ってください。
  • list のメモリ管理は prs_setup_firstcall のライフサイクルに依存します。
