ParamExternData

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/params.h:90

目的

クエリ実行時に使用される外部パラメータ（PARAM_EXTERN）の個々の値、型情報、および属性フラグを保持するために使用されます。

シグネチャまたは形状

typedef struct ParamExternData
{
	Datum		value;			/* パラメータ値 */
	bool		isnull;			/* NULLか否か */
	uint16		pflags;			/* フラグビット（PARAM_FLAG_CONST等） */
	Oid			ptype;			/* パラメータのデータ型、または0 */
} ParamExternData;

入力と出力

  • **入力**: 構造体の各フィールド（value, isnull, pflags, ptype）に値を設定することで、実行エンジンへパラメータ情報を渡します。
  • **出力**: 実行エンジン（エグゼキュータ）は、この構造体を参照してパラメータ値を取得し、式評価やプラン実行に利用します。

前提条件と不変条件

  • isnull が true の場合、value の内容は無視されるか、特定のデフォルト値である必要があります。
  • ptype が 0 の場合、型情報が未定義であることを示します。

アルゴリズムまたは振る舞い

  • ParamListInfo 構造体内の配列要素として保持され、静的なパラメータ参照に使用されます。
  • pflags に PARAM_FLAG_CONST が設定されている場合、プランナーは当該パラメータを定数として扱い、最適化を適用することが許可されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/commands/prepare.c
  • src/backend/executor/execCurrent.c
  • src/backend/executor/execExprInterp.c
  • src/backend/executor/functions.c
  • src/backend/executor/spi.c
  • src/backend/nodes/params.c
  • **呼び出し先**: 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ParamListInfo: 本構造体の配列を保持する主要なコンテナ。
  • ParamExecData: 実行時内部計算用パラメータを保持する構造体（用途が異なる）。

失敗モードと境界条件

  • ParamListInfo の配列サイズを超えたインデックスへのアクセスは、メモリ破壊を引き起こす可能性があります。
  • ptype に無効な OID が指定された場合、型変換や式評価時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は ParamListInfo の一部として使用されます。ParamListInfo が paramFetch フックを使用している場合、この配列が必ずしも有効なデータを含んでいるとは限らない点に注意してください。
  • pflags のビット定義については、src/include/nodes/params.h 内の関連定義を参照してください。
