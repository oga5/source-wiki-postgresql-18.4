get_tablespace_maintenance_io_concurrency

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/spccache.c:228

目的

指定されたテーブルスペースにおけるメンテナンス用I/O並行性設定値を取得します。テーブルスペース固有の設定が存在しない、または無効な場合は、システム全体のデフォルト値を返します。

シグネチャまたは形状

int get_tablespace_maintenance_io_concurrency(Oid spcid)

入力と出力

  • **入力**: spcid (Oid) - 対象テーブルスペースのOID。
  • **出力**: int - 適用されるメンテナンス用I/O並行性の値。

前提条件と不変条件

  • get_tablespace(spcid) が有効な TableSpaceCacheEntry を返すことを前提とします。
  • テーブルスペース固有のオプション（opts）が未設定、または maintenance_io_concurrency が負の値である場合、グローバル変数 maintenance_io_concurrency がフォールバックとして使用されます。

アルゴリズムまたは振る舞い

1. get_tablespace(spcid) を呼び出し、キャッシュから該当テーブルスペースの情報を取得します。 2. 取得したエントリの opts が存在し、かつ opts->maintenance_io_concurrency が 0 以上であるかを確認します。 3. 条件を満たす場合は、そのテーブルスペース固有の設定値を返します。 4. 条件を満たさない（opts が NULL または値が負）場合は、システム全体のデフォルト値である maintenance_io_concurrency を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c:8184
  • src/backend/storage/aio/read_stream.c:575
  • **呼び出し先**:
  • get_tablespace(Oid spcid)

関連データ構造

  • TableSpaceCacheEntry: キャッシュされたテーブルスペース情報およびオプションを保持する構造体。

失敗モードと境界条件

  • spcid が無効な場合、get_tablespace の振る舞いに依存します（通常はエラーまたはデフォルト値の解決が行われます）。
  • テーブルスペースオプションが未定義の場合、安全にグローバルデフォルト値へフォールバックします。

保守担当者向けメモ

  • 本関数はプランナやI/Oストリーム制御において、テーブルスペースごとのI/O特性を反映するために使用されます。
  • キャッシュの無効化は InvalidateTableSpaceCacheCallback によって行われるため、本関数が返す値は常に最新のカタログ状態を反映します。
