ReadArrayDimensions

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arrayfuncs.c:401

目的

配列リテラルの文字列形式から、次元数、各次元のサイズ、および下限値（lower bound）を解析して抽出する。

シグネチャまたは形状

static bool ReadArrayDimensions(char **srcptr, int *ndim_p, int *dim, int *lBound, const char *origStr, Node *escontext)

入力と出力

  • **入力**:
  • srcptr: 解析対象文字列の現在位置を指すポインタ。
  • origStr: エラーメッセージ生成用の元の文字列全体。
  • escontext: エラー報告用ノード。
  • **出力**:
  • ndim_p: 抽出された次元数。
  • dim: 各次元のサイズを格納する配列。
  • lBound: 各次元の下限値を格納する配列。
  • 戻り値: 成功時に true、失敗時に false。

前提条件と不変条件

  • ndim は MAXDIM を超えてはならない。
  • ub (上限値) は lBound 以上である必要がある。
  • ub は INT_MAX であってはならない。
  • 計算される配列サイズは MaxArraySize を超えてはならない。

アルゴリズムまたは振る舞い

1. [ で始まる次元指定（[n] または [m:n]）をループで順次解析する。 2. [ が見つからない場合はループを終了する。 3. [m:n] 形式の場合、m を下限値、n を上限値として設定する。 4. [n] 形式の場合、下限値を 1、上限値を n として設定する。 5. 各次元のサイズ（ub - lBound + 1）を計算し、オーバーフローチェックを行う。 6. 解析した次元情報を dim および lBound 配列に格納し、srcptr を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/arrayfuncs.c 内の配列入力処理関数（array_in 等）から呼び出される。
  • **呼び出し先**: ReadDimensionInt を呼び出して数値部分を解析する。

関連データ構造

  • MAXDIM: 配列の最大次元数。
  • MaxArraySize: 配列の最大要素数。

失敗モードと境界条件

  • **次元数超過**: ndim >= MAXDIM の場合にエラーを報告。
  • **構文エラー**: [ の後の数値欠落や、] の欠落、[m:n] の形式不正時にエラーを報告。
  • **範囲エラー**: 上限値が下限値より小さい場合、または上限値が INT_MAX の場合にエラーを報告。
  • **サイズ超過**: 計算された配列サイズが MaxArraySize を超える場合にエラーを報告。

保守担当者向けメモ

  • 本関数は array_in 等の文字列解析フローの一部であり、escontext を用いたエラーハンドリングを行っている。
  • [1:0] のようなゼロ長次元はサポートされていない。
  • ReadDimensionInt の結果が p == q（数値が読み取れなかった）の場合、不正なリテラルとして処理される。
