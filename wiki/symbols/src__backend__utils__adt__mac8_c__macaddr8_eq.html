macaddr8_eq

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mac8.c:355

目的

2つの macaddr8 型の値が等しいかどうかを判定し、SQLの等価演算子（=）の結果を返します。

シグネチャまたは形状

Datum macaddr8_eq(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの macaddr8 型へのポインタを受け取ります。
  • **出力**: bool 型の Datum を返します。両者が等しい場合は true、そうでない場合は false です。

前提条件と不変条件

  • 入力される macaddr8 ポインタは有効である必要があります。
  • 比較ロジックは macaddr8_cmp_internal に依存しており、内部的な8バイト表現のビット単位の比較に基づきます。

アルゴリズムまたは振る舞い

1. PG_GETARG_MACADDR8_P マクロを使用して、引数から2つの macaddr8 構造体へのポインタを取得します。 2. macaddr8_cmp_internal を呼び出し、2つの値を比較します。 3. 比較結果が 0（等しい）であれば true を、それ以外であれば false を PG_RETURN_BOOL を通じて返します。

呼び出し元と呼び出し先

  • **呼び出し先**: macaddr8_cmp_internal
  • **呼び出し元**:
  • contrib/btree_gist/btree_macaddr8.c:43 にて、DirectFunctionCall2 を介して呼び出されている証拠があります。

関連データ構造

  • macaddr8: 8バイト（EUI-64）のMACアドレスを保持するデータ型。

失敗モードと境界条件

  • 入力が NULL の場合、PG_GETARG_MACADDR8_P の仕様に従い、呼び出し元で適切に処理される必要があります（通常は関数マネージャが NULL を伝播させます）。

保守担当者向けメモ

  • 本関数は macaddr8_cmp_internal の結果に完全に依存しています。比較ロジックの変更が必要な場合は、macaddr8_cmp_internal を参照してください。
  • btree_gist などのインデックス拡張から直接呼び出されるため、シグネチャの変更には注意が必要です。
