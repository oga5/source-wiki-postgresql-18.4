WaitEventCustomEntryByInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/activity/wait_event.c:70

目的

カスタム待機イベントの情報を、待機イベントID（wait_event_info）をキーとして検索・管理するためのハッシュテーブル用エントリ構造体です。

シグネチャまたは形状

typedef struct WaitEventCustomEntryByInfo
{
    uint32      wait_event_info;    /* hash key */
    char        wait_event_name[NAMEDATALEN];    /* custom wait event name */
} WaitEventCustomEntryByInfo;

入力と出力

  • 入力: 待機イベントID（uint32）およびイベント名（char配列）。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • wait_event_info はハッシュテーブルの検索キーとして機能します。
  • wait_event_name は NAMEDATALEN バイトの固定長配列であり、カスタム待機イベントの名称を保持します。

アルゴリズムまたは振る舞い

  • 本構造体は、共有メモリ上のハッシュテーブルにおいて、IDから名前を逆引きするために使用されます。
  • 待機イベントの登録時や検索時に、この構造体を通じてメモリ上のデータにアクセスします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/activity/wait_event.c 内のハッシュテーブル操作関数群（WaitEventCustomNew 等）から参照されます。
  • 呼び出し先: 情報なし。

関連データ構造

  • WaitEventCustomEntryByName: 名前をキーとして検索するための対となる構造体。
  • WaitEventCustomCounterData: カスタムイベントのID割り当てを管理するデータ構造。

失敗モードと境界条件

  • wait_event_name が NAMEDATALEN を超える場合、切り詰められる可能性があります。
  • ハッシュテーブルの容量制限（WAIT_EVENT_CUSTOM_HASH_MAX_SIZE）に達した場合、新しいエントリの追加は失敗します。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されるため、構造体のサイズ変更を行う際は、関連するハッシュテーブルの初期化ロジック（WaitEventCustomShmemInit）との整合性に注意してください。
