key_cmp

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_radixtree/test_radixtree.c:288

目的

uint64 型の値を比較し、qsort 等の標準ライブラリ関数で使用可能な比較結果（整数）を返すための比較関数です。

シグネチャまたは形状

static int key_cmp(const void *a, const void *b)

入力と出力

  • **入力**: 比較対象の uint64 値を指す const void * 型のポインタ a および b。
  • **出力**: a が b より小さい場合は負の数、等しい場合は 0、大きい場合は正の数を返します。

前提条件と不変条件

  • 引数 a および b は、有効な uint64 値を指している必要があります。

アルゴリズムまたは振る舞い

内部で pg_cmp_u64 を呼び出し、ポインタが指す uint64 値の大小比較を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_radixtree/test_radixtree.c 内の qsort 呼び出し（351行目）において、キー配列のソート用比較関数として使用されています。
  • **呼び出し先**: pg_cmp_u64（common/int.h で定義）。

関連データ構造

  • uint64: 比較対象となるデータ型。

失敗モードと境界条件

  • ポインタが NULL である場合や、有効な uint64 を指していない場合の動作は未定義です。

保守担当者向けメモ

  • 本関数は qsort のコールバックとして設計されています。比較ロジックを変更する場合は、pg_cmp_u64 の仕様を確認してください。
