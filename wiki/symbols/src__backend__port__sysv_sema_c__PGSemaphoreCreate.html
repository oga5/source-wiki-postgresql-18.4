PGSemaphoreCreate

定義

  • 種別: function
  • 定義箇所: src/backend/port/sysv_sema.c:387

目的

新しいセマフォを割り当て、必要に応じてカーネルレベルのセマフォセットを新規作成して初期化する。

シグネチャまたは形状

PGSemaphore PGSemaphoreCreate(void)

入力と出力

  • **入力**: なし
  • **出力**: 割り当てられたセマフォへのポインタ (PGSemaphore)

前提条件と不変条件

  • IsUnderPostmaster が false であること（Postmaster プロセス以外での実行はアサートにより禁止）。
  • システム全体で許可されたセマフォセット数およびセマフォ数を超過していないこと。

アルゴリズムまたは振る舞い

1. 現在のセマフォセットが満杯（nextSemaNumber >= SEMAS_PER_SET）の場合:

  • maxSemaSets を確認し、上限に達していれば PANIC で終了。
  • IpcSemaphoreCreate を呼び出し、新しいセマフォセットを確保して mySemaSets に追加。
  • セマフォインデックスをリセット。

    • 2. 共有メモリ上のセマフォ領域（sharedSemas）から未使用の PGSemaphore 構造体を取得。 3. 取得した構造体に現在のセマフォセット ID とインデックスを割り当て。 4. IpcSemaphoreInitialize を呼び出し、当該セマフォの初期値を 1 に設定。 5. 割り当てたセマフォ構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/proc.c (直接的な呼び出しを確認)
  • その他、posix_sema.c や win32_sema.c 等のコメント内で言及あり。
  • **呼び出し先**:
  • IpcSemaphoreCreate
  • IpcSemaphoreInitialize

関連データ構造

  • PGSemaphore: セマフォを管理する構造体へのポインタ。
  • PGSemaphoreData: セマフォセット ID (semId) とセット内インデックス (semNum) を保持する構造体。
  • mySemaSets: 確保済みのセマフォセット ID を保持する配列。
  • sharedSemas: 共有メモリ上に確保されたセマフォの配列。

失敗モードと境界条件

  • **PANIC**:
  • numSemaSets >= maxSemaSets: セマフォセットの最大数を超過した場合。
  • numSharedSemas >= maxSharedSemas: 確保可能なセマフォの総数を超過した場合。
  • **アサート**: IsUnderPostmaster が真の場合、プロセス異常としてアサート失敗。

保守担当者向けメモ

  • 本関数は SysV セマフォ実装の一部であり、カーネルパラメータ（SEMMSL 等）の制限に依存する。
  • セマフォセットの末尾には識別用のマジックナンバーが保持されるため、SEMAS_PER_SET の設定には注意が必要。
