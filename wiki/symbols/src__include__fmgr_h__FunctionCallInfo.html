FunctionCallInfo

定義

目的

PostgreSQL の関数マネージャ（fmgr）を介して呼び出される関数に対し、引数、実行コンテキスト、および戻り値の受け渡しを行うための標準的なインターフェースを提供します。

シグネチャまたは形状

typedef struct FunctionCallInfoBaseData *FunctionCallInfo;

入力と出力

  • **入力**: 関数呼び出し時に、呼び出し元から実行対象の関数へ渡される引数群およびメタデータ。
  • **出力**: 関数実行後、Datum 型の戻り値や NULL 状態が本構造体を通じて呼び出し元へ伝達されます。

前提条件と不変条件

  • FunctionCallInfo は FunctionCallInfoBaseData 構造体を指す有効なポインタである必要があります。
  • 本構造体は、関数マネージャの呼び出し規約（Version 1）に従って初期化されている必要があります。

アルゴリズムまたは振る舞い

  • 関数呼び出しの抽象化レイヤーとして機能し、SQL レベルの関数と C 言語実装を橋渡しします。
  • 呼び出された関数内では、PG_GETARG_xxx マクロを使用して引数にアクセスし、PG_RETURN_xxx マクロを使用して結果を返します。
  • 実行時の引数数や NULL 判定などのメタデータ管理を統括します。

呼び出し元と呼び出し先

  • **呼び出し元（例）**:
  • contrib/btree_gin/btree_gin.c (例: gin_btree_extract_value, gin_btree_compare_prefix)
  • contrib/dblink/dblink.c (例: dblink_record_internal, prepTuplestoreResult)
  • **呼び出し先**:
  • 提供されたソース情報からは、特定の関数実装への直接的な呼び出し先は特定できません。

関連データ構造

  • FunctionCallInfoBaseData: 本型が指す実体となる構造体。
  • FmgrInfo: 関数呼び出しに必要なメタデータ（関数ポインタ、引数数など）を保持する構造体。

失敗モードと境界条件

  • **NULL 処理**: 引数が NULL の場合、PG_ARGISNULL マクロによる明示的なチェックが必要です。
  • **メモリ管理**: 構造体に関連付けられたデータは、指定された MemoryContext に依存します。長期的なキャッシュが必要な場合は fmgr_info_cxt の使用が推奨されます。
