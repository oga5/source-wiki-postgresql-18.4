hash_page_stats

定義

  • **種別**: 関数
  • **定義箇所**: contrib/pageinspect/hashfuncs.c:236

目的

ハッシュインデックスのバケットページまたはオーバーフローページの内容を解析し、ライブ/デッドタプル数や空き容量、ページメタデータを含む統計情報を SQL で取得可能にする。

シグネチャまたは形状

Datum hash_page_stats(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: bytea 型の生ページデータ（PG_GETARG_BYTEA_P(0)）。
  • **出力**: 以下の9項目を含む複合型（HeapTuple）を返す。
  • live_items (int32), dead_items (int32), page_size (int32), free_size (int32), hasho_prevblkno (int64), hasho_nextblkno (int64), hasho_bucket (int64), hasho_flag (int32), hasho_page_id (int32)

前提条件と不変条件

  • 呼び出し元がスーパーユーザーであること（superuser() チェック）。
  • 入力ページが LH_BUCKET_PAGE または LH_OVERFLOW_PAGE であること（verify_hash_page による検証）。
  • 戻り値の型が複合型（TYPEFUNC_COMPOSITE）であること。

アルゴリズムまたは振る舞い

1. スーパーユーザー権限を確認し、失敗時はエラーを投げる。 2. verify_hash_page を呼び出し、ページヘッダーの整合性を検証する。 3. HashPageStat 構造体を初期化し、GetHashPageStatistics を呼び出してページ内の統計情報を算出する。 4. get_call_result_type を用いて呼び出し元の期待するタプル記述子を取得する。 5. 算出された統計値を values 配列に格納し、heap_form_tuple でタプルを生成して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（SELECT * FROM hash_page_stats(...)）。
  • **呼び出し先**: superuser, verify_hash_page, GetHashPageStatistics, get_call_result_type, BlessTupleDesc, heap_form_tuple。

関連データ構造

  • HashPageStat: ページ統計情報を保持する構造体。
  • HeapTuple, TupleDesc: PostgreSQL の行データおよびその記述子。

失敗モードと境界条件

  • スーパーユーザー権限がない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラーとなる。
  • ページがハッシュインデックスのバケットまたはオーバーフローページでない場合、verify_hash_page がエラーを投げる。
  • 戻り値の型が複合型でない場合、elog(ERROR) が発生する。

保守担当者向けメモ

  • 本関数は pageinspect モジュールの一部であり、物理ページを直接操作するため、インデックス破損時には注意が必要である。
  • 統計情報のフィールドを追加・変更する場合は、values 配列のインデックスと tupleDesc の整合性に注意すること。
