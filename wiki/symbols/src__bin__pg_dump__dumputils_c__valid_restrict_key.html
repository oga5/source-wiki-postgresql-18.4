valid_restrict_key

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/dumputils.c:992

目的

提供された文字列（restrict_key）が、許可された文字のみで構成されているか、かつ空ではないかを検証する。

シグネチャまたは形状

bool valid_restrict_key(const char *restrict_key)

入力と出力

  • **入力**: restrict_key (検証対象の文字列ポインタ)
  • **出力**: true（文字列がNULLではなく、空ではなく、かつ許可された文字のみで構成されている場合）、それ以外は false。

前提条件と不変条件

  • restrict_chars という名前の静的定数（許可文字セット）がスコープ内に存在することを前提としている。
  • restrict_key が NULL の場合、関数は安全に false を返す。

アルゴリズムまたは振る舞い

1. restrict_key が NULL でないことを確認する。 2. restrict_key が空文字列（'\0'）でないことを確認する。 3. strspn を使用して、restrict_key 内のすべての文字が restrict_chars に含まれているかを確認する。 4. strspn の戻り値が restrict_key の長さと一致する場合にのみ true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c:913
  • src/bin/pg_dump/pg_dumpall.c:505
  • src/bin/pg_dump/pg_restore.c:376
  • **呼び出し先**:
  • 標準Cライブラリ関数: strspn, strlen

関連データ構造

  • なし（文字列操作のみ）。

失敗モードと境界条件

  • restrict_key が NULL の場合: false を返す。
  • restrict_key が空文字列の場合: false を返す。
  • restrict_key に restrict_chars に含まれない文字が含まれる場合: false を返す。

保守担当者向けメモ

  • 本関数は pg_dump、pg_dumpall、pg_restore における制限キーの妥当性チェックを一元管理している。
  • 許可される文字セット（restrict_chars）を変更する場合は、本関数の挙動に直接影響するため注意すること。
