SetPostRewriteHook

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/plancache.c:504

目的

CachedPlanSource に対して、クエリの書き換え（Rewrite）処理が完了した直後に実行されるフック関数とその引数を設定します。

シグネチャまたは形状

void SetPostRewriteHook(CachedPlanSource *plansource,
                        PostRewriteHook postRewrite,
                        void *postRewriteArg)

入力と出力

  • plansource: 操作対象の CachedPlanSource 構造体へのポインタ。
  • postRewrite: 設定するフック関数（PostRewriteHook 型）。
  • postRewriteArg: フック関数に渡される任意の引数。
  • 戻り値: なし。

前提条件と不変条件

  • plansource は有効な CachedPlanSource 構造体であり、plansource->magic が CACHEDPLANSOURCE_MAGIC と一致している必要があります（Assert による検証）。

アルゴリズムまたは振る舞い

1. plansource のマジックナンバーを検証し、構造体の整合性を確認します。 2. 引数として渡された postRewrite 関数ポインタを plansource->postRewrite に格納します。 3. 引数として渡された postRewriteArg ポインタを plansource->postRewriteArg に格納します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/functions.c:998 からの呼び出しが確認されています。
  • 呼び出し先: 本関数はフックを登録するのみであり、直接的な呼び出し先はありません（フック関数自体は後続の処理で実行されます）。

関連データ構造

  • CachedPlanSource: クエリのソースや解析済みツリーを保持する構造体。本関数はこの構造体のメンバーを更新します。

失敗モードと境界条件

  • plansource が不正なポインタである場合、または magic フィールドが正しくない場合、Assert によりプログラムが停止します（デバッグビルド時）。

保守担当者向けメモ

  • 本関数は、クエリの書き換え後に特定のカスタマイズを行いたい場合に利用されます。
  • PostRewriteHook の具体的な定義や、フックが実際に呼び出されるタイミングについては、src/include/utils/plancache.h および plancache.c 内の関連する実行ロジックを参照してください。
