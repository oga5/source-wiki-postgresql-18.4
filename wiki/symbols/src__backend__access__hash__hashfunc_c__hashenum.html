hashenum

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hashfunc.c:128

目的

列挙型（Enum）の値をハッシュインデックスやハッシュ結合で使用可能な32ビットのハッシュ値に変換します。

シグネチャまたは形状

Datum hashenum(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される列挙型の値（OIDとして取得）。
  • **出力**: Datum 型の32ビットハッシュ値。

前提条件と不変条件

  • 入力値は列挙型の内部表現である OID として正しく取得できる必要があります。
  • 等価な列挙値に対しては、常に同一のハッシュ値を返す必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_OID(0) を使用して、引数から列挙型の OID を抽出します。 2. 抽出した OID を uint32 型にキャストします。 3. hash_uint32 関数を呼び出し、その結果を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのハッシュアクセスメソッド（pg_amproc を介した演算子クラスからの呼び出し）。
  • **呼び出し先**: hash_uint32（src/include/common/hashfn.h で定義）。

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • OID: 列挙型の内部識別子。

失敗モードと境界条件

  • 入力引数が存在しない場合や不正な場合は、PG_GETARG_OID マクロの仕様に従いエラーが発生します。

保守担当者向けメモ

  • 本関数は列挙型専用のハッシュ関数です。ハッシュ結合のパフォーマンス最適化が必要な場合は、シード値を受け取る hashenumextended の実装と併せて保守してください。
