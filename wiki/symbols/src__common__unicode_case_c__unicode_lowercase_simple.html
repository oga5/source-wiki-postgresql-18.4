unicode_lowercase_simple

定義

  • 種別: function
  • 定義箇所: src/common/unicode_case.c:49

目的

単一の Unicode 文字（pg_wchar）を、対応する小文字へ変換します。

シグネチャまたは形状

pg_wchar unicode_lowercase_simple(pg_wchar code);

入力と出力

  • **入力**: code (変換対象の pg_wchar 型 Unicode コードポイント)
  • **出力**: 変換後の pg_wchar。小文字へのマッピングが存在しない場合は、入力された code がそのまま返されます。

前提条件と不変条件

  • 入力 code は有効な Unicode コードポイントである必要があります。
  • 内部的に case_map_lower テーブルを参照するため、当該テーブルが初期化されている必要があります。

アルゴリズムまたは振る舞い

1. find_case_map 関数を呼び出し、case_map_lower テーブルを用いて code に対応する小文字を検索します。 2. 検索結果が 0 以外（マッピングが見つかった場合）であれば、その値を返します。 3. 検索結果が 0 の場合（マッピングが存在しない場合）、入力された code をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_pg_locale.c:594
  • src/common/unicode/case_test.c:82
  • **呼び出し先**:
  • find_case_map (内部関数)

関連データ構造

  • pg_wchar: PostgreSQL で使用されるワイド文字型。
  • case_map_lower: src/common/unicode_case_table.h で定義される小文字変換用テーブル。

失敗モードと境界条件

  • マッピングが存在しない文字（すでに小文字である文字や、ケースを持たない文字）が入力された場合、エラーにはならず、入力値をそのまま返します。

保守担当者向けメモ

  • 本関数は「単純な」変換のみを扱います。ギリシャ語の最終シグマのような文脈依存の変換（特殊ケースマッピング）は考慮しません。
  • パフォーマンス向上のため、テーブル参照による高速な変換が行われる設計となっています。
