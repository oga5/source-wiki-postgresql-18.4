get_rte_attribute_is_dropped

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_relation.c:3395

目的

指定された RangeTblEntry (RTE) において、特定の属性番号（列）が削除済み（dropped）であるかどうかを判定する。

シグネチャまたは形状

bool get_rte_attribute_is_dropped(RangeTblEntry *rte, AttrNumber attnum)

入力と出力

  • **入力**:
  • rte: 検査対象の RangeTblEntry 構造体へのポインタ。
  • attnum: 検査対象の属性番号（AttrNumber）。
  • **出力**:
  • 削除済みであれば true、そうでなければ false を返す。

前提条件と不変条件

  • rte は有効なポインタであること。
  • RTE_RELATION の場合、attnum はシステムカタログ pg_attribute に存在する有効な番号である必要がある。
  • RTE_JOIN や RTE_NAMEDTUPLESTORE の場合、attnum はリストの範囲内である必要がある。

アルゴリズムまたは振る舞い

rte->rtekind に基づいて分岐し、以下のロジックで判定を行う。

  • **RTE_RELATION**: SearchSysCache2 を使用して pg_attribute を検索し、attisdropped フラグを確認する。
  • **RTE_SUBQUERY / TABLEFUNC / VALUES / CTE / GROUP**: 削除された列を持つことはないため、常に false を返す。
  • **RTE_NAMEDTUPLESTORE**: rte->coltypes リストの該当インデックスが OidIsValid でない場合に true と判定する。
  • **RTE_JOIN**: rte->joinaliasvars リストの該当要素が NULL である場合に true と判定する。
  • **RTE_FUNCTION**: 関数が返す複合型の定義を get_expr_result_tupdesc で取得し、その属性の attisdropped を確認する。ただし、funccolnames が存在する場合や、funcordinality 列の場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/rewrite/rewriteHandler.c において、ルール処理中の列削除検出に使用されている。
  • **呼び出し先**: SearchSysCache2, ReleaseSysCache, get_expr_result_tupdesc, list_nth, list_nth_oid 等のユーティリティ関数。

関連データ構造

  • RangeTblEntry: クエリ内のリレーション情報を保持する構造体。
  • Form_pg_attribute: pg_attribute システムカタログの行構造。
  • Var: RTE_JOIN の判定に使用されるノード。

失敗モードと境界条件

  • **エラー**:
  • RTE_RELATION でキャッシュ検索に失敗した場合、elog(ERROR) を発生させる。
  • RTE_NAMEDTUPLESTORE や RTE_JOIN で attnum が範囲外の場合、elog(ERROR) を発生させる。
  • RTE_FUNCTION や RTE_RESULT で列が見つからない場合、ereport(ERROR, ...) により ERRCODE_UNDEFINED_COLUMN を報告する。
  • 未知の rtekind が渡された場合、elog(ERROR) を発生させる。

保守担当者向けメモ

  • RTE_JOIN における削除列の判定は、joinaliasvars の NULL ポインタに依存している。ルール書き換え処理との整合性に注意すること。
  • RTE_FUNCTION の判定ロジックは、関数が返す型情報に依存するため、型定義の変更が影響を与える可能性がある。
