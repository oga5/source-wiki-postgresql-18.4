transformJsonTableColumns

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_jsontable.c:248

目的

JSON_TABLE 構文におけるカラム定義リストを走査し、TableFunc 構造体への列情報（型、照合順序、式）の登録および、ネストされた構造を含む実行計画ツリー（JsonTablePlan）の構築を行います。

シグネチャまたは形状

static JsonTablePlan *transformJsonTableColumns(
    JsonTableParseContext *cxt, 
    List *columns,
    List *passingArgs,
    JsonTablePathSpec *pathspec
)

入力と出力

  • **入力**:
  • cxt: 解析コンテキスト（TableFunc やパーサ状態を含む）。
  • columns: 解析対象の JsonTableColumn ノードリスト。
  • passingArgs: JSONパス評価に渡される引数リスト。
  • pathspec: 現在のパス指定。
  • **出力**:
  • 構築された JsonTablePlan（makeJsonTablePathScan の戻り値）。

前提条件と不変条件

  • cxt->tf（TableFunc）が初期化されていること。
  • NESTED 以外のカラムは name を持つこと。
  • FOR ORDINALITY カラムはリスト内に最大1つであること。

アルゴリズムまたは振る舞い

1. **カラム走査**: columns リストを順次処理し、型に応じて変換を行う。

  • JTC_FOR_ORDINALITY: INT4 型として登録。重複をチェック。
  • JTC_REGULAR/FORMATTED/EXISTS: transformJsonTableColumn を呼び出し、JsonFuncExpr を生成。その後 transformExpr で式を変換し、型・照合順序を TableFunc に追加。
  • JTC_NESTED: 本関数内ではスキップ（後続の再帰処理で扱う）。

    • 2. **型判定**: isCompositeType を用いて、REGULAR カラムを FORMATTED（JSON_QUERY）へ昇格させる判定を行う。 3. **再帰処理**: transformJsonTableNestedColumns を呼び出し、ネストされたカラムを処理して子プランを生成。 4. **プラン構築**: makeJsonTablePathScan を呼び出し、現在の階層の列情報と子プランを統合して JsonTablePlan を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_jsontable.c 内の transformJsonTable（メインエントリ）。
  • src/backend/parser/parse_jsontable.c 内の transformJsonTableNestedColumns（再帰呼び出し）。
  • **呼び出し先**:
  • transformJsonTableColumn
  • transformJsonTableNestedColumns
  • makeJsonTablePathScan
  • transformExpr（式変換）

関連データ構造

  • JsonTableParseContext: 解析状態を保持。
  • TableFunc: 変換結果が格納されるターゲット構造体。
  • JsonTableColumn: 入力となる構文解析ノード。
  • JsonTablePlan: 構築される実行計画ノード。

失敗モードと境界条件

  • FOR ORDINALITY カラムが複数定義されている場合、ERRCODE_SYNTAX_ERROR でエラー終了。
  • 未知の coltype が指定された場合、elog(ERROR) で停止。
  • typenameTypeIdAndMod による型解決失敗時はエラー。

保守担当者向けメモ

  • 本関数は LATERAL 結合の文脈で実行されることを前提としています。
  • colMin と colMax は、TableFunc 内の列リストにおける現在の階層が担当する範囲を指します。
  • NESTED 句の再帰的な処理は transformJsonTableNestedColumns に委譲されているため、階層構造の変更時はそちらも併せて確認してください。
