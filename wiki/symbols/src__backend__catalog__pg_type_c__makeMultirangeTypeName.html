makeMultirangeTypeName

定義

  • **種別**: function
  • **定義箇所**: src/backend/catalog/pg_type.c:949

目的

指定されたレンジ型名に基づき、対応するマルチレンジ型の名前を生成する。

シグネチャまたは形状

char *makeMultirangeTypeName(const char *rangeTypeName, Oid typeNamespace)

入力と出力

  • **入力**:
  • rangeTypeName: 元となるレンジ型の名前。
  • typeNamespace: 型が存在する名前空間のOID。
  • **出力**:
  • 生成されたマルチレンジ型の名前（palloc された文字列）。

前提条件と不変条件

  • 生成される名前は NAMEDATALEN - 1 バイト以内に切り詰められる。
  • 指定された名前空間において、生成された名前が既に存在しないこと。

アルゴリズムまたは振る舞い

1. rangeTypeName に "range" という文字列が含まれている場合、それを "multirange" に置換する。 2. 含まれていない場合、rangeTypeName の末尾に "_multirange" を付加する。 3. 生成された文字列を NAMEDATALEN - 1 バイトで切り詰める。 4. SearchSysCacheExists2 を使用して、指定された名前空間に同名の型が存在するか確認する。 5. 存在する場合は ERRCODE_DUPLICATE_OBJECT でエラーを発生させる。 6. 存在しない場合、生成した名前を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/typecmds.c:1633 において、マルチレンジ型作成時に呼び出される。
  • **呼び出し先**: strstr, pnstrdup, psprintf, pg_mbcliplen, SearchSysCacheExists2, ereport, pstrdup 等のユーティリティ関数。

関連データ構造

  • pg_type: 本関数が名前の妥当性を検証する対象のシステムカタログ。

失敗モードと境界条件

  • **名前の重複**: 生成された名前が既に pg_type に存在する場合、ereport によりエラー終了する。
  • **名前の長さ**: NAMEDATALEN を超える名前は切り詰められる。

保守担当者向けメモ

  • 本関数はマルチレンジ型の命名規則を強制する。命名規則を変更する場合は、range から multirange への置換ロジックおよびサフィックス付加ロジックの両方を考慮すること。
