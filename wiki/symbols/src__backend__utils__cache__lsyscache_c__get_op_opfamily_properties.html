get_op_opfamily_properties

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/lsyscache.c:136

目的

指定された演算子（opno）が特定の演算子ファミリー（opfamily）に属しているかを確認し、そのファミリー内での演算子のプロパティ（ストラテジ番号および左右の入力型）を取得します。

シグネチャまたは形状

void get_op_opfamily_properties(Oid opno, Oid opfamily, bool ordering_op,
                                int *strategy, Oid *lefttype, Oid *righttype)

入力と出力

  • **入力**:
  • opno: 検索対象の演算子OID。
  • opfamily: 検索対象の演算子ファミリーOID。
  • ordering_op: 順序付け演算子（AMOP_ORDER）を検索する場合は true、検索演算子（AMOP_SEARCH）の場合は false。
  • **出力**:
  • *strategy: 演算子のストラテジ番号が格納されます。
  • *lefttype: 演算子の左辺入力型OIDが格納されます。
  • *righttype: 演算子の右辺入力型OIDが格納されます。

前提条件と不変条件

  • 呼び出し元は、出力引数として有効な int および Oid 型のポインタを提供する必要があります。
  • 検索対象の演算子が指定されたファミリー内に存在しない場合、関数はエラーを発生させます。

アルゴリズムまたは振る舞い

1. SearchSysCache3 を使用し、AMOPOPID キャッシュを検索します。検索キーは opno、ordering_op に基づくフラグ（AMOP_ORDER または AMOP_SEARCH）、および opfamily です。 2. キャッシュヒットしない（HeapTuple が無効）場合は elog(ERROR) を呼び出し、処理を中断します。 3. GETSTRUCT() マクロを用いてタプルを Form_pg_amop 構造体にキャストします。 4. 構造体から amopstrategy、amoplefttype、amoprighttype を抽出し、各ポインタ引数に代入します。 5. ReleaseSysCache() を呼び出し、参照カウントを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExpr.c
  • src/backend/executor/nodeIndexscan.c
  • src/backend/executor/nodeMergejoin.c
  • src/backend/optimizer/path/indxpath.c
  • src/backend/partitioning/partprune.c
  • src/backend/utils/adt/selfuncs.c
  • **呼び出し先**:
  • SearchSysCache3 (システムカタログ検索)
  • ReleaseSysCache (キャッシュ解放)
  • elog (エラー報告)

関連データ構造

  • Form_pg_amop: pg_amop システムカタログの行構造体。演算子ファミリーのメンバーシップ情報を保持します。

失敗モードと境界条件

  • 演算子が指定されたファミリーのメンバーでない場合、elog(ERROR) によりトランザクションがアボートされます。

保守担当者向けメモ

  • 本関数は pg_amop カタログのキャッシュに直接依存しています。カタログ定義に変更があった場合は、キャッシュキーの整合性に注意してください。
  • 戻り値がないため、成功時は必ずポインタ経由で値が書き込まれることを前提としています。
