print_ssl_library

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/test/libpq_testclient.c:15

目的

現在リンクされている SSL ライブラリの名称を取得し、標準出力へ表示する。

シグネチャまたは形状

static void print_ssl_library(void);

入力と出力

  • 入力: なし（引数を受け取らない）
  • 出力: なし（戻り値は void）
  • 副作用:
  • SSL ライブラリ名が取得できた場合、その名称を標準出力 (stdout) に書き出す。
  • SSL が有効でない場合、エラーメッセージを標準エラー出力 (stderr) に書き出す。

前提条件と不変条件

  • libpq-fe.h がインクルードされている環境で実行されること。
  • PQsslAttribute 関数が利用可能であること。

アルゴリズムまたは振る舞い

1. PQsslAttribute(NULL, "library") を呼び出し、SSL ライブラリ名を取得する。 2. 戻り値が NULL の場合、SSL が無効であると判断し、stderr に "SSL is not enabled" と出力する。 3. 戻り値が有効な文字列である場合、その文字列を stdout に出力する。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/test/libpq_testclient.c 内の main 関数。
  • 呼び出し先: libpq ライブラリの PQsslAttribute 関数。

関連データ構造

  • なし（libpq が提供する不透明な属性値を使用）。

失敗モードと境界条件

  • PQsslAttribute が NULL を返す場合（SSL がビルド時に無効化されている、またはリンクされていない場合）、エラーメッセージが出力される。

保守担当者向けメモ

  • 本関数は PQsslAttribute に NULL を渡すことで、特定の接続に依存しないグローバルな SSL ライブラリ情報を取得する手法を採用している。
  • 疎通確認用のユーティリティであるため、複雑なエラーハンドリングは含まれていない。
