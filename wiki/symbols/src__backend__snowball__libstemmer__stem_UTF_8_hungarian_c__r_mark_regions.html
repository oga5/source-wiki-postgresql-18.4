r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_hungarian.c:479

目的

ハンガリー語のステミング処理において、語幹抽出の有効範囲（R1領域）を決定し、Snowball 環境のインデックス z->I[0] に格納する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z（ステミング実行環境）
  • **出力**: 成功時に 1、失敗時に 0 を返す。
  • **副作用**: z->I[0] を更新し、内部カーソル z->c を移動させる。

前提条件と不変条件

  • z は有効な SN_env 構造体であること。
  • g_v（母音グループ）が定義されていること。

アルゴリズムまたは振る舞い

1. 初期状態として z->I[0] を文字列の末尾 z->l に設定する。 2. 母音の出現位置を探索する。

  • 母音が見つかった場合、その直後の子音の直後を R1 の開始位置として z->I[0] に設定する。
  • 特定の接尾辞（a_0 配列）が続く場合は、その長さを考慮して調整する。
  • 母音が見つからない、または条件を満たさない場合は、別の母音探索ルール（out_grouping_U）を適用し、その結果に基づいて z->I[0] を更新する。

    • 3. 最終的に決定した位置を z->I[0] に保持して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: hungarian_UTF_8_stem（ステミング処理のメインエントリーポイント）。
  • **呼び出し先**: in_grouping_U, out_grouping_U, find_among, skip_utf8（いずれも Snowball ランタイム関数）。

関連データ構造

  • struct SN_env: ステミングの実行状態を保持する構造体。
  • a_0: 接尾辞マッチングに使用される among 構造体の配列。

失敗モードと境界条件

  • 文字列が空、または母音が含まれない場合、z->I[0] は初期値（文字列長）のままとなる。
  • skip_utf8 や out_grouping_U が範囲外を検出した場合、処理を中断し 0 を返す。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードである。ロジックの変更が必要な場合は、直接編集せずソース定義ファイルを修正すること。
  • z->I[0] は後続のステミングルールにおける境界条件として使用されるため、本関数の計算結果はステミング精度に直結する。
