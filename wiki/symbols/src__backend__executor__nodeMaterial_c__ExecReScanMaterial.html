ExecReScanMaterial

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMaterial.c:312

目的

Material ノードの再スキャン処理を実行し、蓄積されたタプルセットの再利用または破棄と再構築を制御する。

シグネチャまたは形状

void ExecReScanMaterial(MaterialState *node)

入力と出力

  • **入力**: MaterialState *node (マテリアライズノードの状態構造体)
  • **出力**: なし（副作用としてノードの状態を更新）

前提条件と不変条件

  • node->ss.ps.ps_ResultTupleSlot は本関数の開始時にクリアされる。
  • node->eflags が 0 の場合、本ノードは透過的なパススルーとして動作する。
  • tuplestorestate が存在する場合、eof_underlying フラグは再スキャン開始時に false にリセットされる。

アルゴリズムまたは振る舞い

1. ExecClearTuple を呼び出し、結果タプルスロットをクリアする。 2. eflags != 0 の場合（マテリアライズが有効な場合）:

  • tuplestorestate が未初期化なら即座に終了する。
  • outerPlan->chgParam が存在するか、EXEC_FLAG_REWIND が指定されていない場合、現在の tuplestorestate を破棄し、必要に応じて子ノードの ExecReScan を実行して再構築に備える。
  • それ以外の場合、tuplestore_rescan を呼び出し、既存の tuplestore の読み取り位置を先頭に戻す。

    • 3. eflags == 0 の場合（パススルーモード）:

  • outerPlan->chgParam が NULL ならば、子ノードに対して ExecReScan を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:254 (確認済み)
  • **呼び出し先**:
  • ExecClearTuple
  • outerPlanState
  • tuplestore_end
  • tuplestore_rescan
  • ExecReScan (子ノードに対して)

関連データ構造

  • MaterialState: ノードの状態を保持。tuplestorestate や eflags を含む。
  • PlanState: 子ノードの状態。chgParam を通じて再スキャンの要否を判断する。
  • Tuplestore: タプルのバッファリングに使用される構造体。

失敗モードと境界条件

  • eflags が適切に設定されていない場合、再スキャン時に tuplestore の再構築が必要となるが、呼び出し元が EXEC_FLAG_REWIND を渡していないと効率が低下する可能性がある。
  • outerPlan->chgParam が NULL でない場合、子ノードの再スキャンは ExecProcNode 呼び出し時に自動的に行われるため、本関数内ではスキップされる。

保守担当者向けメモ

  • tuplestore の破棄と再構築はメモリおよびディスク I/O に影響を与えるため、chgParam の判定ロジックを変更する際は注意が必要。
  • eof_underlying フラグの管理は、子ノードの終了状態を正しく追跡するために不可欠である。
