array_subscript_check_subscripts

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arraysubs.c:180

目的

配列の添字（サブスクリプト）式を評価し、その妥当性を検証した上で、実行用のワークスペースへ整数値として格納します。

シグネチャまたは形状

static bool array_subscript_check_subscripts(ExprState *state, ExprEvalStep *op, ExprContext *econtext);

入力と出力

  • **入力**:
  • state: 式評価の状態管理構造体。
  • op: 現在の評価ステップ。sbsref_subscript.state を介して SubscriptingRefState にアクセスします。
  • econtext: 式の評価コンテキスト。
  • **出力**:
  • 戻り値: 添字が有効であれば true、NULL でありかつ代入操作でない場合は false を返します。
  • 副作用: op->resnull が true に設定される場合があります。また、workspace->upperindex および workspace->lowerindex が更新されます。

前提条件と不変条件

  • op->d.sbsref_subscript.state が有効な SubscriptingRefState を指していること。
  • workspace が sbsrefstate->workspace から正しくキャスト可能であること。

アルゴリズムまたは振る舞い

1. **上界（upper）添字の処理**: numupper 分だけループし、提供されている添字をチェックします。 2. **NULL チェック**: 添字が NULL の場合、代入操作であればエラーを報告し、そうでなければ op->resnull を true にして false を返します。 3. **値の格納**: NULL でない場合、Datum 型の添字を int32 に変換し、workspace->upperindex に格納します。 4. **下界（lower）添字の処理**: 上界と同様のロジックを numlower 分だけ繰り返します。 5. **正常終了**: すべての添字処理が完了すれば true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/arraysubs.c 内で methods->sbs_check_subscripts に割り当てられており、実行エンジンから間接的に呼び出されます。
  • **呼び出し先**:
  • DatumGetInt32 (マクロ)
  • ereport (エラー報告用)

関連データ構造

  • SubscriptingRefState: 添字操作の実行時状態を保持します。
  • ArraySubWorkspace: 添字の境界値や型情報を保持する作業用構造体です。
  • ExprEvalStep: 式評価のステップ定義。

失敗モードと境界条件

  • **NULL 添字**: 添字が NULL の場合、代入操作（isassignment が true）であれば ERRCODE_NULL_VALUE_NOT_ALLOWED エラーが発生します。
  • **非代入時の NULL**: 代入でない場合はエラーにはならず、結果が NULL となります。

保守担当者向けメモ

  • 本関数は SubscriptingRef ノードの評価プロセスの一部です。
  • ArraySubWorkspace の次元数制限（MAXDIM）に関するチェックは、この関数を呼び出す前段のセットアップ処理で保証されている前提です。
