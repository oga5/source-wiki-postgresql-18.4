ListDictionary

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/ts_cache.h:65

目的

全文検索設定（TSConfigCacheEntry）において、特定のトークン型に関連付けられた辞書リストを保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	int			len;
	Oid		   *dictIds;
} ListDictionary;

入力と出力

  • len: リストに含まれる辞書の数。
  • dictIds: 辞書オブジェクトを識別する Oid の配列。

前提条件と不変条件

  • dictIds は、len で指定された要素数を持つ Oid 配列を指す必要があります。
  • TSConfigCacheEntry 内でトークン型ごとの辞書リストとして管理されます。

アルゴリズムまたは振る舞い

  • 本構造体は、特定のトークン型に対して適用すべき辞書の順序付きリストを保持します。
  • 全文検索の実行時、このリストを順次参照することで、トークンに対する辞書処理（lexize）が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/ts_cache.c において、キャッシュエントリの構築時や設定情報の読み込み時に参照・割り当てが行われます。
  • 呼び出し先: src/backend/tsearch/ts_parse.c 等の全文検索処理において、トークン解析結果に対する辞書適用順序の制御に使用されます。

関連データ構造

  • TSConfigCacheEntry: 本構造体をメンバとして保持し、設定ごとのトークン型マッピングを管理します。

失敗モードと境界条件

  • dictIds が指すメモリ領域の管理は、キャッシュエントリのライフサイクルに依存します。キャッシュ再構築時のメモリリークや不正なポインタ参照に注意が必要です。

保守担当者向けメモ

  • TSConfigCacheEntry の一部として動的に確保されることが多いため、メモリコンテキストの管理には注意してください。
  • 構造体のメンバ順序やサイズは、キャッシュのシリアライズやメモリレイアウトに影響を与える可能性があります。
