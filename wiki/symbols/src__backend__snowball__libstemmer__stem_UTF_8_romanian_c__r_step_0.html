目的

ルーマニア語のステミング処理における初期段階として、特定の接尾辞パターンを識別し、条件を満たす場合に削除または置換を行う。

シグネチャまたは形状

static int r_step_0(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境構造体。現在のカーソル位置 z->c や文字列データ z->p を含む）
  • 出力: 成功時に 1、失敗または処理対象外の場合に 0、エラー発生時に負の値を返す。

前提条件と不変条件

  • z->c は処理対象の文字列内を指している必要がある。
  • r_R1(z) を呼び出し、対象が R1 領域内にあることを検証する。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置 z->c を z->ket に設定する。 2. 文字列の末尾付近の文字をチェックし、find_among_b を用いて a_2 テーブル（16個のパターン）と照合する。 3. マッチした場合、z->bra を現在のカーソル位置に設定し、r_R1 を実行して領域制約を確認する。 4. マッチしたパターン（among_var）に応じて以下の処理を行う：

  • case 1: slice_del で削除。
  • case 2-4, 6-7: slice_from_s で指定文字列に置換。
  • case 5: 特定の条件（eq_s_b）を確認し、条件に応じて置換を行う。

    • 5. 処理成功時に 1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_romanian.c 内のメイン処理フローから呼び出される。
  • 呼び出し先: find_among_b, r_R1, slice_del, slice_from_s, eq_s_b（いずれも Snowball ランタイム関数）。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • a_2: 接尾辞マッチング用の among 構造体テーブル。

失敗モードと境界条件

  • z->c - 1 <= z->lb: 文字列の境界外へのアクセスを防ぐガード条件。
  • r_R1 が失敗（<= 0）した場合、その戻り値をそのまま返す。
  • slice_del や slice_from_s が失敗（< 0）した場合、エラーコードを伝播させる。
