tuplesort_begin_index_hash

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplesortvariants.c:440

目的

ハッシュインデックス構築のためのソート処理を初期化し、ハッシュ計算に必要なパラメータ（マスク値やバケット数）と、インデックス専用の比較・入出力関数を Tuplesortstate に登録する。

シグネチャまたは形状

Tuplesortstate *tuplesort_begin_index_hash(
    Relation heapRel,
    Relation indexRel,
    uint32 high_mask,
    uint32 low_mask,
    uint32 max_buckets,
    int workMem,
    SortCoordinate coordinate,
    int sortopt
)

入力と出力

  • **入力**:
  • heapRel, indexRel: 対象となるヒープおよびインデックスのリレーション。
  • high_mask, low_mask, max_buckets: ハッシュインデックスの構築に必要な制御パラメータ。
  • workMem: ソートに使用可能なメモリ量。
  • coordinate, sortopt: ソートの実行環境およびオプションフラグ。
  • **出力**:
  • 初期化済みの Tuplesortstate 構造体へのポインタ。

前提条件と不変条件

  • tuplesort_begin_common が正常に終了し、メモリコンテキストが準備されていること。
  • TuplesortIndexHashArg 構造体用のメモリ確保が成功すること。
  • ソートキーはハッシュコードの1列のみに固定される（base->nKeys = 1）。

アルゴリズムまたは振る舞い

1. tuplesort_begin_common を呼び出し、共通のソート状態を初期化する。 2. TuplesortIndexHashArg 構造体を割り当て、ハッシュインデックス固有のパラメータを格納する。 3. Tuplesortstate の関数ポインタに、ハッシュインデックス専用の関数を割り当てる：

  • comparetup: comparetup_index_hash
  • comparetup_tiebreak: comparetup_index_hash_tiebreak
  • writetup: writetup_index
  • readtup: readtup_index

    • 4. trace_sort が有効な場合、設定されたパラメータをログに出力する。 5. 構築した Tuplesortstate を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/hash/hashsort.c (ハッシュインデックスのソート処理開始時)
  • **呼び出し先**:
  • tuplesort_begin_common
  • palloc
  • MemoryContextSwitchTo
  • elog

関連データ構造

  • Tuplesortstate: ソート処理の全状態を保持するメイン構造体。
  • TuplesortIndexHashArg: ハッシュインデックス固有のパラメータ（マスク値、バケット数、リレーション情報）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時、palloc がエラーを発生させる可能性がある。
  • workMem の値が極端に小さい場合、後続のソート処理でディスクへのスピルが発生する（writetup_index が使用される）。

保守担当者向けメモ

  • 本関数はハッシュインデックス専用の比較ロジックを注入するプラグインとして機能する。
  • ハッシュコードの比較において、主要キーが一致した場合のタイブレーク処理（comparetup_index_hash_tiebreak）が正しく設定されていることを確認すること。
