TwoPhaseShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/twophase.c:252

目的

2フェーズコミット（2PC）で使用される共有メモリ領域の初期化および準備済みトランザクション管理用構造体のセットアップを行う。

シグネチャまたは形状

void TwoPhaseShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上の TwoPhaseState を初期化）

前提条件と不変条件

  • TwoPhaseShmemSize() で計算されたサイズの共有メモリが確保可能であること。
  • IsUnderPostmaster が偽（Postmaster起動時）の場合、TwoPhaseState は未初期化である必要がある。
  • IsUnderPostmaster が真（バックエンド起動時）の場合、TwoPhaseState は既に初期化済みである必要がある。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を呼び出し、共有メモリ上の TwoPhaseState 構造体を確保または取得する。 2. IsUnderPostmaster が偽の場合（初期化時）:

  • TwoPhaseState->freeGXacts を空に初期化する。
  • max_prepared_xacts 分の GlobalTransactionData 構造体を共有メモリ領域内に配置する。
  • 各 GlobalTransactionData を freeGXacts リストに連結し、InitProcGlobal で割り当てられた PreparedXactProcs の pgprocno を各構造体に紐付ける。

    • 3. IsUnderPostmaster が真の場合（バックエンド接続時）:

  • 既に初期化済みであることを Assert(found) で確認する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c 内の CreateSharedMemoryAndSemaphores から呼び出される。
  • 呼び出し先: ShmemInitStruct, TwoPhaseShmemSize, GetNumberFromPGProc。

関連データ構造

  • TwoPhaseStateData: 共有メモリ上のメイン管理構造体。
  • GlobalTransactionData: 個々の準備済みトランザクションの状態を保持する構造体。
  • PreparedXactProcs: 準備済みトランザクション用に予約された PGPROC 配列。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、ShmemInitStruct 内でパニックが発生する可能性がある。
  • max_prepared_xacts が0に設定されている場合、リストの初期化はスキップされる。

保守担当者向けメモ

  • 本関数はPostgreSQLの起動シーケンスにおいて、共有メモリの初期化フェーズで一度だけ実行される。
  • GlobalTransactionData の配置は TwoPhaseStateData の直後にメモリをオフセット計算して配置する設計となっているため、構造体のレイアウト変更時には TwoPhaseShmemSize との整合性に注意すること。
