test_random

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_radixtree/test_radixtree.c:294

目的

適応型基数木（ART）に対して、擬似乱数を用いた大規模なデータセットの挿入・検索・削除・イテレーションを行い、その整合性と堅牢性を検証する。

シグネチャまたは形状

static void test_random(void)

入力と出力

  • 入力: なし（内部で GetCurrentTimestamp() をシードとして使用）
  • 出力: なし（検証失敗時は EXPECT_* マクロにより elog(ERROR) を発行）

前提条件と不変条件

  • TEST_SHARED_RT マクロの定義有無により、メモリ管理方式（DSA または SlabContext）が切り替わる。
  • 挿入したキーと値は、テスト終了までにすべて削除され、木が空（エントリ数 0）になる必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: 基数木インスタンスを作成し、pg_prng を初期化する。 2. **挿入**: 100,000 個のランダムなキーを生成し、木に挿入すると同時に配列に保持する。 3. **検索検証**: 挿入した全キーが rt_find で正しく取得できるか確認する。 4. **不在検証**: キーのソート後、挿入されていない範囲（キー間、最小値未満、最大値超過）の検索結果が NULL であることを確認する。 5. **イテレーション検証**: rt_begin_iterate を使用し、キーが昇順で正しく列挙されるか確認する。 6. **削除**: 挿入時と同じシードで乱数を再生成し、全キーを削除する。 7. **終了**: rt_num_entries が 0 であることを確認し、メモリを解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/test_radixtree/test_radixtree.c 内のメインテスト関数（test_radixtree）から呼び出される。
  • 呼び出し先: rt_create, rt_set, rt_find, rt_begin_iterate, rt_iterate_next, rt_end_iterate, rt_delete, rt_num_entries, rt_free 等の基数木 API。

関連データ構造

  • rt_radix_tree: テスト対象の基数木インスタンス。
  • rt_iter: イテレーション用ハンドル。
  • pg_prng_state: 乱数生成器の状態。
  • TestValueType: 格納される値の型。

失敗モードと境界条件

  • EXPECT_* マクロによる検証失敗時、トランザクションがアボートされる。
  • 乱数生成器のシードによりキーが重複する場合、イテレーションや削除処理で重複をスキップするロジックが適用される。
  • 境界条件として、キー空間のフィルタリング（filter）によりメモリ使用量を制限している。

保守担当者向けメモ

  • TEST_SHARED_RT マクロの切り替えにより、共有メモリ環境での動作もテスト対象となるため、変更時は両方のパスを確認すること。
