xstrcat

定義

  • 種別: function
  • 定義箇所: src/bin/psql/copy.c:77

目的

既存の文字列ポインタが指すメモリ領域を拡張し、新しい文字列を末尾に連結して更新する。

シグネチャまたは形状

static void xstrcat(char **var, const char *more)

入力と出力

  • var: 更新対象の文字列ポインタへのポインタ。関数終了後、新しいメモリ領域を指すように更新される。
  • more: *var の末尾に追加するソース文字列。
  • 戻り値: なし（void）。

前提条件と不変条件

  • *var は malloc 等で確保された有効なメモリ領域を指しているか、あるいは NULL である必要がある（psprintf の仕様に依存）。
  • 関数実行後、古い *var は free されるため、呼び出し元は古いポインタを参照し続けてはならない。

アルゴリズムまたは振る舞い

1. psprintf を使用して、現在の *var と more を連結した新しい文字列を生成する。 2. free(*var) を呼び出し、古いメモリ領域を解放する。 3. *var に新しい文字列のポインタを代入する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/copy.c 内の parse_slash_copy 関数から複数回呼び出され、result->before_tofrom 構造体メンバの構築に使用されている。
  • 呼び出し先: psprintf（メモリ確保とフォーマット）、free（標準ライブラリ）。

関連データ構造

  • copy_options: parse_slash_copy 内で xstrcat を用いて文字列が構築される構造体。

失敗モードと境界条件

  • psprintf がメモリ不足により失敗した場合、プログラムは通常終了する（psprintf の内部実装による）。
  • *var が NULL の場合、psprintf の実装がそれを許容するかどうかに依存する。

保守担当者向けメモ

  • 本関数は static であり、copy.c 内でのみ使用される。
  • 文字列の連結を繰り返す際にメモリリークを防ぐためのユーティリティである。頻繁な連結を行う場合は、パフォーマンスの観点から PQExpBuffer の使用を検討すること。
