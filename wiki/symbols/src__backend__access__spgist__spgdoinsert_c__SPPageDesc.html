SPPageDesc

定義

  • 種別: struct
  • 定義箇所: src/backend/access/spgist/spgdoinsert.c:36

目的

SP-GiSTインデックスへのデータ挿入処理において、現在操作対象としているページ、タプル、およびノードに関する情報を保持・追跡するための一時的な構造体です。

シグネチャまたは形状

typedef struct SPPageDesc
{
    BlockNumber blkno;          /* ブロック番号、または InvalidBlockNumber */
    Buffer      buffer;         /* バッファ番号、または InvalidBuffer */
    Page        page;           /* ページバッファへのポインタ、または NULL */
    OffsetNumber offnum;        /* タプルのオフセット、または InvalidOffsetNumber */
    int         node;           /* インナータプル内のノード番号、または -1 */
} SPPageDesc;

入力と出力

本構造体は関数間でポインタとして受け渡され、挿入アルゴリズムの各ステップ（探索、分割、移動など）における状態を共有するために使用されます。

前提条件と不変条件

  • buffer が有効な場合、対応する page ポインタが正しく設定されている必要があります。
  • blkno はインデックス内の有効なブロック番号を指すか、あるいは InvalidBlockNumber である必要があります。

アルゴリズムまたは振る舞い

SP-GiSTの挿入プロセスにおいて、ツリーを下降する際の「現在地」および「親ページ」の情報を保持します。挿入アルゴリズム（spgdoinsert）は、この構造体を通じてページ内のタプル位置やノード番号を更新し、必要に応じてページ分割やリーフチェーンの移動を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: spgdoinsert を中心に、addLeafTuple、doPickSplit、moveLeafs、setRedirectionTuple、saveNodeLink などの挿入関連関数で引数として使用されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • Buffer: storage/bufmgr.h で定義されるバッファ管理用型。
  • Page: storage/bufpage.h 等で定義されるページレイアウト用型。
  • BlockNumber, OffsetNumber: インデックス構造を特定するための識別子。

失敗モードと境界条件

  • buffer が InvalidBuffer の場合、ページへのアクセスは行えません。
  • node が -1 の場合、インナータプル内の特定のノードを指していない状態を示します。
  • 挿入処理中にデッドロックを回避するため、ロック取得に失敗した場合は本構造体の情報を保持したまま処理を中断し、再試行する設計となっています。

保守担当者向けメモ

  • 本構造体は挿入処理中のスタック上の変数として頻繁に利用されます。構造体のフィールドを追加・変更する場合は、spgdoinsert.c 内のすべての参照箇所で整合性が保たれていることを確認してください。
