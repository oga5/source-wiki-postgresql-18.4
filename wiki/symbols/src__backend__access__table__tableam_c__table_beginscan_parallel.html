table_beginscan_parallel

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/tableam.c:165

目的

並列スキャンを実行するためのテーブルスキャン記述子（TableScanDesc）を初期化し、共有メモリからスナップショットを復元または取得します。

シグネチャまたは形状

TableScanDesc table_beginscan_parallel(Relation relation, ParallelTableScanDesc pscan)

入力と出力

  • **入力**:
  • relation: スキャン対象のテーブルリレーション。
  • pscan: 並列スキャン状態を保持する共有メモリ上の記述子。
  • **出力**:
  • 初期化された TableScanDesc 構造体。

前提条件と不変条件

  • relation のリレーションロケータと pscan 内のロケータ（phs_locator）が一致している必要があります（Assert による検証）。

アルゴリズムまたは振る舞い

1. **フラグ設定**: シーケンシャルスキャン、戦略的スキャン、同期スキャン、ページモードを許可するフラグを初期化します。 2. **スナップショットの解決**:

  • pscan->phs_snapshot_any が偽の場合、共有メモリ上のシリアライズされたスナップショットを RestoreSnapshot で復元し、RegisterSnapshot で登録します。この際、一時スナップショットフラグを付与します。
  • pscan->phs_snapshot_any が真の場合、SnapshotAny を使用します。

    • 3. **AMへの委譲**: relation->rd_tableam->scan_begin を呼び出し、各テーブルアクセスメソッド固有の初期化処理を実行して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/nbtree/nbtsort.c
  • src/backend/executor/nodeSeqscan.c
  • **呼び出し先**:
  • RestoreSnapshot
  • RegisterSnapshot
  • relation->rd_tableam->scan_begin（各AM固有の実装）

関連データ構造

  • TableScanDesc: スキャン状態を保持する構造体。
  • ParallelTableScanDesc: 並列スキャン用の共有状態。
  • Relation: テーブルのメタデータ。

失敗モードと境界条件

  • RelFileLocatorEquals の不一致が発生した場合、アサーションエラーとなります。
  • スナップショットの復元やAM固有の scan_begin 処理内でエラーが発生した場合、PostgreSQLの標準的なエラーハンドリング（elog(ERROR) 等）によりトランザクションが中断される可能性があります。

保守担当者向けメモ

  • 本関数は rd_tableam を介したディスパッチを行うため、特定のストレージエンジン（Heapなど）の挙動を追う際は、該当する TableAmRoutine の scan_begin 実装を参照してください。
  • スナップショットのライフサイクル管理（RegisterSnapshot）が含まれているため、スキャン終了時には適切なクリーンアップが必要です。
