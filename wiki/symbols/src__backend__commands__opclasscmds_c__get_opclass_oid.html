get_opclass_oid

定義

  • 種別: function
  • 定義箇所: src/backend/commands/opclasscmds.c:219

目的

指定されたアクセスメソッド（AM）と演算子クラス名から、システムカタログ上の演算子クラスのOIDを取得します。

シグネチャまたは形状

Oid get_opclass_oid(Oid amID, List *opclassname, bool missing_ok)

入力と出力

  • **入力**:
  • amID: 演算子クラスが属するアクセスメソッドのOID。
  • opclassname: 演算子クラス名を指定するリスト（スキーマ修飾名を含む場合がある）。
  • missing_ok: true の場合、演算子クラスが見つからなくてもエラーにせず InvalidOid を返します。
  • **出力**:
  • 成功時: 対応する演算子クラスの Oid。
  • 失敗時: missing_ok が true の場合は InvalidOid。

前提条件と不変条件

  • OpClassCacheLookup を介してシステムキャッシュを利用するため、トランザクションコンテキスト内で呼び出される必要があります。
  • 取得したタプルは関数内で ReleaseSysCache されるため、呼び出し元でタプルを保持することはできません。

アルゴリズムまたは振る舞い

1. OpClassCacheLookup を呼び出し、指定されたAMと名前で演算子クラスのシステムカタログタプルを検索します。 2. タプルが無効な場合、InvalidOid を返します。 3. タプルが有効な場合、GETSTRUCT マクロを使用して Form_pg_opclass 構造体へキャストし、oid フィールドを抽出します。 4. ReleaseSysCache を呼び出してキャッシュ参照を解放します。 5. 抽出した opcID を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c
  • src/backend/commands/typecmds.c
  • src/backend/parser/parse_clause.c
  • **呼び出し先**:
  • OpClassCacheLookup (内部ヘルパー)
  • ReleaseSysCache (システムユーティリティ)

関連データ構造

  • Form_pg_opclass: pg_opclass システムカタログのタプル構造体。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • 指定された演算子クラスが存在しない場合、missing_ok が false であれば OpClassCacheLookup 内部でエラーが発生します。
  • amID が無効な場合、検索は失敗します。

保守担当者向けメモ

  • 本関数は pg_opclass カタログへの読み取り専用アクセスを提供します。
  • 演算子クラスの定義変更や削除を行う際は、本関数で取得したOIDを基点として依存関係の検証を行う必要があります。
