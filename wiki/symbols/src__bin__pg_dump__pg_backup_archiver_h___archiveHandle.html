_archiveHandle

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_backup_archiver.h:215

目的

pg_dump および pg_restore におけるアーカイブ処理の全状態を保持・管理するメイン構造体です。アーカイブの物理構造と論理的なダンプ内容を抽象化し、フォーマット固有の処理を統合します。

シグネチャまたは形状

struct _archiveHandle {
    Archive public;
    int version;
    /* ... バージョン情報、フォーマット、TOC、関数ポインタ、接続状態等 ... */
    struct _tocEntry *toc;
    int tocCount;
    /* ... */
};

入力と出力

  • **入力**: アーカイブファイル、データベース接続（PGconn）、およびダンプ/復元設定。
  • **出力**: アーカイブファイルへの書き込み、またはデータベースへの復元実行。

前提条件と不変条件

  • ArchiveHandle は、選択されたアーカイブ形式（Custom, Directory, Tar等）に応じた関数ポインタ群で初期化されている必要があります。
  • toc は循環リストのヘッダーであり、tocCount はそのエントリ数と一致している必要があります。
  • 並列復元時には restorePass が現在の復元フェーズを正しく追跡している必要があります。

アルゴリズムまたは振る舞い

  • **フォーマット抽象化**: ArchiveEntryPtr や WriteDataPtr などの関数ポインタを通じて、実行時にアーカイブ形式に応じた処理を動的に切り替えます。
  • **TOC 管理**: TocEntry を用いてダンプ対象オブジェクトの依存関係や順序を管理します。
  • **並列処理**: ParallelState と連携し、WorkerJob 関数を通じて並列復元を実行します。
  • **状態管理**: lookahead バッファを使用してアーカイブ形式を自動判別し、readHeader で「PGDMP」マーカーの確認状態を保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_dump および pg_restore のメイン処理フロー。
  • **呼び出し先**: pg_backup_archiver.c 内の各実装関数、および各アーカイブ形式固有のモジュール（pg_backup_custom.c 等）。
  • 注: 具体的な関数呼び出しの連鎖については、ソースコードの動的解析が必要です。

関連データ構造

  • Archive: 公開インターフェース部分。
  • TocEntry: ダンプ対象オブジェクトのメタデータ。
  • RestorePass: 復元順序の制御用列挙型。
  • PGconn: データベース接続用。
  • ArchiverStage: アーカイブ処理の現在のステージ。

失敗モードと境界条件

  • lookahead バッファのサイズ制限や、アーカイブ形式の不一致による初期化失敗。
  • データベース接続の切断や、connCancel による SIGINT 割り込み時の処理中断。
  • tocCount と実際のリスト構造の不整合によるメモリ破壊の可能性。

保守担当者向けメモ

  • 本構造体はプライベートインターフェースであり、外部ツールからの直接操作は想定されていません。
  • 関数ポインタの追加や変更を行う際は、すべてのアーカイブ形式（Custom, Directory, Tar）で実装が更新されているか確認してください。
  • lookahead 関連のフィールドは、アーカイブ形式判別時のみ使用されるため、他のフェーズでの誤用を避けてください。
