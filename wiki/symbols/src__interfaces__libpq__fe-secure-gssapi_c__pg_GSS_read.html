pg_GSS_read

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure-gssapi.c:265

目的

GSSAPI で暗号化された通信路からデータを読み取り、復号して呼び出し元に提供する。

シグネチャまたは形状

ssize_t pg_GSS_read(PGconn *conn, void *ptr, size_t len)

入力と出力

  • **入力**:
  • conn: 接続状態を保持する PGconn 構造体。
  • ptr: 復号後のデータを格納するバッファ。
  • len: 読み取りを要求するバイト数。
  • **出力**:
  • 成功時: 実際に読み取られたバイト数（ssize_t）。
  • 失敗時: -1 を返し、errno を設定する。

前提条件と不変条件

  • conn->gctx が有効な GSSAPI セキュリティコンテキストであること。
  • 通信路はメッセージ指向であり、各パケットには uint32 の長さヘッダーが付与されていること。
  • 復号されたデータは PqGSSResultBuffer に一時保持され、呼び出し元へ順次コピーされる。

アルゴリズムまたは振る舞い

1. PqGSSResultBuffer に未読データがある場合、それを優先的に ptr へコピーして戻る。 2. バッファが空の場合、pqsecure_raw_read を用いてネットワークからパケット長（uint32）を読み取る。 3. パケット長を確認後、ペイロード全体が揃うまで pqsecure_raw_read を繰り返す。 4. gss_unwrap を呼び出し、受信データを復号する。 5. 復号結果が機密性（confidentiality）を欠く場合はエラーとする。 6. 復号されたデータを PqGSSResultBuffer に格納し、呼び出し元へコピーして終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-secure.c（pg_GSS_read を呼び出す上位のセキュリティ抽象化層）。
  • **呼び出し先**: pqsecure_raw_read（低レベルソケット読み取り）、gss_unwrap（GSSAPI 復号）、gss_release_buffer（GSSAPI バッファ解放）、pg_GSS_error（エラー報告）。

関連データ構造

  • PGconn: 接続状態および GSSAPI コンテキスト（gctx）を保持。
  • PqGSSRecvBuffer: 受信した暗号化パケットを一時保持するバッファ。
  • PqGSSResultBuffer: 復号後のデータを保持するバッファ。

失敗モードと境界条件

  • **パケットサイズ超過**: 受信したパケット長が PQ_GSS_MAX_PACKET_SIZE を超える場合、EIO を設定し -1 を返す。
  • **復号失敗**: gss_unwrap が GSS_S_COMPLETE 以外を返した場合、エラー報告を行い EIO を設定して -1 を返す。
  • **機密性欠如**: conf_state が 0 の場合、EIO を設定して -1 を返す。
  • **部分読み取り**: ネットワークからデータが完全に届いていない場合、EWOULDBLOCK を設定して -1 を返す。

保守担当者向けメモ

  • 本関数は非同期 I/O を前提としており、部分的な読み取りが発生した場合は呼び出し元が再試行を行う必要がある。
  • PqGSSResultBuffer の管理は本関数内で行われるため、バッファポインタの更新（PqGSSResultNext 等）に注意すること。
  • gss_unwrap 後の gss_release_buffer によるメモリ解放は必須であり、エラーパスを含めて適切に処理されていることを確認すること。
