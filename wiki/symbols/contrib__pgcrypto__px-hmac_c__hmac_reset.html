hmac_reset

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/px-hmac.c:83

目的

HMAC 計算の状態を初期状態（キー適用直後の状態）にリセットし、新しいメッセージのハッシュ計算を開始できるようにする。

シグネチャまたは形状

static void hmac_reset(PX_HMAC *h)

入力と出力

  • **入力**: PX_HMAC *h - リセット対象の HMAC コンテキスト構造体へのポインタ。
  • **出力**: なし（副作用としてコンテキスト内部の状態を更新）。

前提条件と不変条件

  • 引数 h は有効な PX_HMAC 構造体を指している必要がある。
  • h->md には初期化済みのダイジェストコンテキストが保持されていること。
  • h->p.ipad には、RFC 2104 に基づく適切な Inner Pad 値が格納されていること。

アルゴリズムまたは振る舞い

1. h->md からハッシュアルゴリズムのブロックサイズを取得する。 2. px_md_reset(md) を呼び出し、内部のダイジェストコンテキストを初期化する。 3. px_md_update(md, h->p.ipad, bs) を呼び出し、保存されている ipad をダイジェストコンテキストに適用する。これにより、次の update 呼び出しからメッセージデータの処理が開始される状態となる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/px-hmac.c 内で PX_HMAC 構造体の reset 関数ポインタとして割り当てられている。
  • **呼び出し先**:
  • px_md_reset (基盤レイヤーのダイジェストリセット関数)
  • px_md_update (基盤レイヤーのダイジェスト更新関数)
  • px_md_block_size (基盤レイヤーのブロックサイズ取得関数)

関連データ構造

  • PX_HMAC: HMAC 操作の状態を保持する構造体。md（ダイジェストコンテキスト）および p.ipad（内部パディング）を含む。

失敗モードと境界条件

  • h が NULL の場合、あるいは h->md が不正な状態である場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は PX_HMAC 構造体のメソッドとして登録されるため、直接呼び出すよりも h->reset(h) の形式で利用されることが想定されている。
  • ipad の適用は RFC 2104 の HMAC 仕様に準拠しており、変更の際は当該仕様を参照すること。
