system_nextsampleblock

定義

  • 種別: function
  • 定義箇所: src/backend/access/tablesample/system.c:177

目的

TABLESAMPLE SYSTEM メソッドにおいて、次に読み込むべきテーブルブロックを決定します。ハッシュ値とカットオフ値を比較することで、指定されたサンプリング率に基づいたブロック選択を行います。

シグネチャまたは形状

static BlockNumber system_nextsampleblock(SampleScanState *node, BlockNumber nblocks)

入力と出力

  • **入力**:
  • node: SampleScanState 構造体へのポインタ。内部の tsm_state に SystemSamplerData を保持。
  • nblocks: テーブルの総ブロック数。
  • **出力**:
  • 選択された BlockNumber。条件を満たすブロックがない場合は InvalidBlockNumber を返します。

前提条件と不変条件

  • node->tsm_state は SystemSamplerData 型としてキャスト可能であること。
  • SystemSamplerData 内の seed および cutoff は、system_beginsamplescan により初期化済みであること。
  • ハッシュ計算には hash_any を使用しており、プラットフォーム間で一貫した結果が得られること。

アルゴリズムまたは振る舞い

1. SystemSamplerData から現在のスキャン位置 (nextblock) とシード値を取得します。 2. nextblock から nblocks に達するまでループし、各ブロック番号とシード値を組み合わせて hash_any でハッシュ値を算出します。 3. 算出されたハッシュ値が sampler->cutoff 未満であれば、そのブロックを選択してループを終了します。 4. ブロックが見つかった場合、次回の開始位置として nextblock + 1 を保存し、そのブロック番号を返します。 5. 全ブロックを走査しても条件を満たすブロックが見つからない場合、sampler->nextblock を 0 にリセットし、InvalidBlockNumber を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/tablesample/system.c: tsm_system_handler の初期化処理において、TsmRoutine 構造体の NextSampleBlock フィールドに本関数が登録されます。
  • **呼び出し先**:
  • hash_any: ハッシュ値の計算に使用。

関連データ構造

  • SampleScanState: スキャン全体の実行状態を保持。
  • SystemSamplerData: 本メソッドが使用するサンプリング状態（現在のブロック位置、シード値、カットオフ値）を保持。

失敗モードと境界条件

  • **終了条件**: nextblock が nblocks に達した場合、サンプリング終了とみなされ InvalidBlockNumber を返します。
  • **安全性**: 終了時には sampler->nextblock を 0 にリセットし、次回のスキャン開始に備えます。

保守担当者向けメモ

  • 本関数は TABLESAMPLE SYSTEM の再現性を保証するため、hash_any を使用しています。ロジックを変更する際は、サンプリングの一貫性が損なわれないよう注意してください。
  • hashinput 配列はブロック番号とシード値の 2 つの uint32 で構成されており、この構造は hash_any の入力として最適化されています。
