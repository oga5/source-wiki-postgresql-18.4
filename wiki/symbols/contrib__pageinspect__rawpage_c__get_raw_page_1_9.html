get_raw_page_1_9

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/rawpage.c:48

目的

指定されたリレーションのメインフォークから、特定のブロック番号に対応する物理ページを読み出し、bytea 形式で取得する。

シグネチャまたは形状

Datum get_raw_page_1_9(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • relname (text): 対象リレーション名。
  • blkno (int64): 読み出し対象のブロック番号。
  • **出力**:
  • bytea: 取得したページイメージ。

前提条件と不変条件

  • blkno は 0 以上かつ MaxBlockNumber 以下である必要がある。
  • 実行にはスーパーユーザー権限が必要（get_raw_page_internal の内部仕様に依存）。

アルゴリズムまたは振る舞い

1. 引数からリレーション名とブロック番号を取得する。 2. ブロック番号が有効範囲内か検証し、範囲外の場合は ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告する。 3. get_raw_page_internal を呼び出し、MAIN_FORKNUM を指定してページデータを取得する。 4. 取得した bytea データを戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: get_raw_page_internal。

関連データ構造

  • bytea: ページイメージの格納に使用。
  • MAIN_FORKNUM: 読み出し対象のフォーク種別。

失敗モードと境界条件

  • blkno が負の値、または MaxBlockNumber を超える場合にエラーが発生する。
  • リレーションが存在しない、またはアクセス権限がない場合、内部の get_raw_page_internal がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は pageinspect 拡張機能のバージョン 1.9 向けインターフェースである。
  • 物理ストレージへの直接アクセスを行うため、セキュリティ上の制約（スーパーユーザー権限）が適用される点に留意すること。
