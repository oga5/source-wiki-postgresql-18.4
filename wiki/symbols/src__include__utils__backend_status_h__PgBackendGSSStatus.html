PgBackendGSSStatus

定義

  • 種別: struct
  • 定義箇所: src/include/utils/backend_status.h:75

目的

バックエンドプロセスにおけるGSSAPI接続に関連する状態情報を保持し、共有メモリを介して監視可能にするための構造体です。

シグネチャまたは形状

typedef struct PgBackendGSSStatus
{
    char        gss_princ[NAMEDATALEN]; /* GSSAPI Principal used to auth */
    bool        gss_auth;               /* If GSSAPI authentication was used */
    bool        gss_enc;                /* If encryption is being used */
    bool        gss_delegation;         /* If credentials delegated */
} PgBackendGSSStatus;

入力と出力

  • **入力**: GSSAPI認証プロセスおよび接続確立時に設定される認証情報、暗号化フラグ、委任フラグ。
  • **出力**: 共有メモリ上の状態エントリとして、外部からの監視プロセスに対して現在のGSSAPI接続状態を提供します。

前提条件と不変条件

  • gss_princ は NAMEDATALEN バイト以内の文字列である必要があります。
  • 共有メモリ上のデータ更新時は、backend_status.h で定義されるチェンジカウンタ方式（PGSTAT_BEGIN_WRITE_ACTIVITY / PGSTAT_END_WRITE_ACTIVITY）に従い、不整合を防ぐ必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、バックエンドの活動状況を追跡する PgBackendStatus の一部として利用されます。
  • 接続確立時に memset で初期化され、認証成功後に各フィールドが更新されます。
  • 読み取り側は、メモリバリアを伴うチェンジカウンタの検証を通じて、更新中の不整合なデータを読み取らないよう制御されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/backend_status.c 内のバックエンド状態管理関数群。
  • **呼び出し先**: 直接的な関数呼び出し先は定義されていません。データ構造として src/backend/utils/activity/backend_status.c 内の BackendGssStatusBuffer や、各バックエンドのローカルエントリを通じて操作されます。

関連データ構造

  • PgBackendStatus: 本構造体を含む、バックエンド全体の活動状況を保持する親構造体。
  • PgBackendSSLStatus: 対となるSSL接続状態を保持する構造体。

失敗モードと境界条件

  • NAMEDATALEN を超えるプリンシパル名は切り詰められる可能性があります。
  • 共有メモリの初期化や割り当て（mul_size(sizeof(PgBackendGSSStatus), NumBackendStatSlots)）に失敗した場合、バックエンドの起動や状態監視機能が正常に動作しません。

保守担当者向けメモ

  • 本構造体は pgstat が管理する累積統計とは異なり、現在の接続状態を保持するリアルタイム監視用です。
  • フィールドを追加・変更する場合は、backend_status.c 内のコピー処理（memcpy）や初期化処理が正しく追従しているか確認してください。
