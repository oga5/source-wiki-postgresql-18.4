create_range_bounds

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partbounds.c:676

目的

Rangeパーティションの定義（PartitionBoundSpec）を、実行時に効率的な検索が可能な内部表現（PartitionBoundInfo）へ変換します。

シグネチャまたは形状

static PartitionBoundInfo create_range_bounds(PartitionBoundSpec **boundspecs, int nparts, PartitionKey key, int **mapping)

入力と出力

  • **入力**:
  • boundspecs: パーティション境界定義の配列。
  • nparts: パーティション数。
  • key: パーティションキー定義。
  • mapping: パーティションインデックスのマッピング情報（出力用）。
  • **出力**:
  • 構築された PartitionBoundInfo 構造体。

前提条件と不変条件

  • key->strategy が PARTITION_STRATEGY_RANGE であること。
  • boundspecs に含まれる各 PartitionBoundSpec の戦略も PARTITION_STRATEGY_RANGE であること。
  • Rangeパーティションには、HashやListのような「NULL値専用」のパーティションは存在しない。

アルゴリズムまたは振る舞い

1. **初期化**: PartitionBoundInfoData を割り当て、戦略を設定。 2. **境界の収集**: 各パーティションの境界（下限・上限）を all_bounds に収集。デフォルトパーティションが存在する場合はそのインデックスを記録。 3. **ソート**: qsort_arg を使用し、qsort_partition_rbound_cmp に基づいて全境界を昇順にソート。 4. **重複排除**: ソートされた境界を走査し、重複する境界を除去して rbounds に格納。 5. **構造体構築**:

  • 抽出された境界値を boundinfo->datums にコピー。
  • boundinfo->indexes 配列を構築。Rangeパーティションでは ndatums + 1 のサイズを持ち、パーティションが存在しない範囲（ギャップ）には -1 を設定。
  • 下限境界（lower bound）に対応するインデックスには -1 を割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**: partition_bounds_create（src/backend/partitioning/partbounds.c 内）から呼び出される。
  • **呼び出し先**:
  • make_one_partition_rbound
  • qsort_arg
  • qsort_partition_rbound_cmp
  • datumCopy
  • FunctionCall2Coll

関連データ構造

  • PartitionBoundInfo: 変換後の境界情報を保持するメイン構造体。
  • PartitionRangeBound: 個別のRange境界を表現する構造体。
  • PartitionKey: パーティションキーのメタデータ。

失敗モードと境界条件

  • PARTITION_STRATEGY_RANGE 以外の戦略が指定された場合、elog(ERROR) を発生させる。
  • 境界値のソートおよび重複排除において、key->partsupfunc を用いた比較が失敗した場合、エラーが発生する可能性がある。

保守担当者向けメモ

  • indexes 配列のサイズは ndatums + 1 であり、Rangeパーティションの特性上、ギャップを -1 で表現する点に注意すること。
  • boundDatums および boundKinds は、メモリ割り当てを最適化するために一括で確保されている。
