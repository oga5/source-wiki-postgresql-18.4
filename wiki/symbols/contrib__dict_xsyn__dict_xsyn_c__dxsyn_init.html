dxsyn_init

定義

  • 種別: function
  • 定義箇所: contrib/dict_xsyn/dict_xsyn.c:145

目的

PostgreSQLの全文検索用拡張シノニム辞書（xsyn）の初期化を行います。SQLで指定された設定オプションを解析し、辞書データ構造を構築して外部ファイルからシノニムを読み込みます。

シグネチャまたは形状

Datum dxsyn_init(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して、辞書設定オプションのリスト（List *dictoptions）を受け取ります。
  • **出力**: 初期化された DictSyn 構造体へのポインタを Datum 型として返します。

前提条件と不変条件

  • dictoptions に含まれる各要素は DefElem 構造体である必要があります。
  • rules パラメータで指定される辞書ファイルは、read_dictionary が処理可能な形式である必要があります。
  • 辞書オプションの解析が完了した後にのみ、ファイル読み込みが実行されます。

アルゴリズムまたは振る舞い

1. DictSyn 構造体をメモリ確保し、デフォルト値（matchorig=true, keeporig=true, matchsynonyms=false, keepsynonyms=true）で初期化します。 2. dictoptions リストを反復処理し、各パラメータ（matchorig, keeporig, matchsynonyms, keepsynonyms, rules）を解析して構造体に反映させます。 3. 未知のパラメータが指定された場合は ERROR を発生させます。 4. rules パラメータでファイル名が指定されている場合、read_dictionary を呼び出して辞書データをロードします。 5. 構築した DictSyn 構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの全文検索フレームワーク（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: read_dictionary（辞書ファイル読み込み用）、defGetBoolean / defGetString（オプション解析用）、palloc0（メモリ確保用）。

関連データ構造

  • DictSyn: 辞書の設定とシノニムデータを保持するメイン構造体。
  • DefElem: SQLから渡される設定オプションを保持する構造体。

失敗モードと境界条件

  • 未知のパラメータが渡された場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー終了します。
  • read_dictionary 内部でファイル読み込みに失敗した場合、その関数内でエラーが発生します。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_INFO_V1 によりエクスポートされています。
  • rules パラメータの処理は、すべてのオプション解析が完了した後に実行されるよう設計されています。これは、ファイル読み込み時に他のオプション設定が確定している必要があるためです。
