ApplyLauncherSetWorkerStartTime

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/launcher.c:1042

目的

指定されたサブスクリプション（subid）に関連付けられた論理レプリケーションワーカーの最終起動時刻を、共有メモリ上のハッシュテーブルに記録する。

シグネチャまたは形状

static void ApplyLauncherSetWorkerStartTime(Oid subid, TimestampTz start_time)

入力と出力

  • **入力**:
  • subid (Oid): 対象となるサブスクリプションの識別子。
  • start_time (TimestampTz): 記録する起動時刻。
  • **出力**: なし（副作用として共有メモリの状態を更新）。

前提条件と不変条件

  • 呼び出し前に logicalrep_launcher_attach_dshmem() が実行され、共有メモリへのアタッチが保証されている必要がある。
  • last_start_times ハッシュテーブルが初期化済みであること。

アルゴリズムまたは振る舞い

1. logicalrep_launcher_attach_dshmem() を呼び出し、共有メモリへのアクセスを確保する。 2. dshash_find_or_insert を使用して、subid に対応する LauncherLastStartTimesEntry を検索または新規作成する。 3. 取得したエントリの last_start_time フィールドに start_time を代入する。 4. dshash_release_lock を呼び出し、該当エントリのロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/launcher.c:1209（サブスクリプションのワーカー起動処理内）
  • **呼び出し先**:
  • logicalrep_launcher_attach_dshmem
  • dshash_find_or_insert
  • dshash_release_lock

関連データ構造

  • LauncherLastStartTimesEntry: 起動時刻を保持する共有ハッシュテーブルのエントリ構造体。
  • last_start_times: 起動時刻を管理する共有ハッシュテーブル（dshash）。

失敗モードと境界条件

  • 共有メモリが初期化されていない状態で呼び出された場合、dshash 関連関数内でエラーが発生する可能性がある。
  • dshash_find_or_insert がメモリ不足等で失敗した場合、例外が発生する。

保守担当者向けメモ

  • 本関数はワーカーの再起動間隔を制御し、エラー発生時の過度なリソース消費を防ぐための重要な役割を担う。
  • ロックの取得・解放は dshash の内部メカニズムに依存しているため、この関数を呼び出す際はデッドロックの可能性を考慮する必要がある。
