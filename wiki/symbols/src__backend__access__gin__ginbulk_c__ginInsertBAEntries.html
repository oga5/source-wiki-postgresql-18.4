ginInsertBAEntries

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginbulk.c:209

目的

GINインデックス構築時、複数のキーとTID（タプル識別子）のペアを効率的に BuildAccumulator（赤黒木）へ挿入する。入力順序を最適化することで、赤黒木の再平衡化コストを最小化する。

シグネチャまたは形状

void ginInsertBAEntries(BuildAccumulator *accum, ItemPointer heapptr, OffsetNumber attnum, Datum *entries, GinNullCategory *categories, int32 nentries)

入力と出力

  • **入力**:
  • accum: 構築状態を保持する BuildAccumulator 構造体へのポインタ。
  • heapptr: 挿入対象の ItemPointer。
  • attnum: 属性番号。
  • entries: 挿入するキー（Datum）の配列。
  • categories: 各キーのカテゴリ（GinNullCategory）の配列。
  • nentries: 挿入するエントリ数。
  • **出力**: なし（副作用として accum 内の赤黒木が更新される）。

前提条件と不変条件

  • nentries > 0 であること。
  • heapptr が有効な ItemPointer であること。
  • attnum >= FirstOffsetNumber であること。

アルゴリズムまたは振る舞い

1. 入力された nentries に対して、それ以下の最大の2のべき乗を計算し、初期 step とする。 2. step を用いて、入力配列の「中央値」付近から順に ginInsertBAEntry を呼び出す。 3. step を半分に減らしながら、配列全体を網羅するまでこの処理を繰り返す。 4. この挿入順序により、赤黒木のバランスを保ちつつ、ソート済み入力に近い効率でノードを構築する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginfast.c
  • src/backend/access/gin/gininsert.c
  • **呼び出し先**:
  • ginInsertBAEntry（各エントリの個別の挿入処理）

関連データ構造

  • BuildAccumulator: 構築中の状態を管理するメイン構造体。
  • GinEntryAccumulator: 赤黒木の各ノード。

失敗モードと境界条件

  • nentries <= 0 の場合、何もせず即座にリターンする。
  • メモリ不足時は ginInsertBAEntry 経由で ereport(ERROR) が発生する可能性がある。

保守担当者向けメモ

  • step の計算におけるビット演算は、入力データがソートされていることを前提とした最適化である。
  • ginInsertBAEntry を呼び出す際のループ間隔（step << 1）は、二分探索木の構築順序を制御するための重要なロジックである。
