exclude_list_item

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_rewind/filemap.c:102

目的

同期処理（pg_rewind 等）において、データディレクトリ内の特定のファイルやディレクトリを対象から除外するための定義を保持する。

シグネチャまたは形状

struct exclude_list_item
{
	const char *name;
	bool		match_prefix;
};

入力と出力

  • name: 除外対象となるファイル名またはディレクトリ名の文字列。
  • match_prefix: true の場合、name をプレフィックスとして前方一致で判定する。false の場合は完全一致で判定する。

前提条件と不変条件

  • name はヌル終端文字列である必要がある。
  • この構造体の配列は、通常 static const として定義され、除外リストを構成する。

アルゴリズムまたは振る舞い

  • この構造体自体はデータ保持用であり、ロジックは含まない。
  • 外部の判定関数（check_file_excluded 等）がこの構造体の配列を走査し、対象ファイルが除外リストに含まれるかを判定するために使用される。

呼び出し元と呼び出し先

  • 呼び出し元（定義の参照元）:
  • src/bin/pg_rewind/filemap.c:158 (excludeFiles 配列の定義)
  • src/backend/backup/basebackup.c:191
  • src/bin/pg_checksums/pg_checksums.c:107
  • 呼び出し先: なし（データ構造のため）。

関連データ構造

  • excludeFiles（filemap.c 内の静的配列）: 本構造体のインスタンスを格納し、除外リストを構成する。

失敗モードと境界条件

  • name が NULL の場合の挙動は、この構造体を使用する側の実装に依存する。
  • プレフィックス一致（match_prefix = true）を使用する場合、意図しないファイルまで除外されないよう、name の指定には注意が必要である。

保守担当者向けメモ

  • pg_rewind における除外リストは basebackup.c と同期させる必要があるため、修正時は両方の実装を確認すること。
  • 本構造体は pg_rewind 以外のツール（pg_checksums 等）でも同様の目的で利用されている。
