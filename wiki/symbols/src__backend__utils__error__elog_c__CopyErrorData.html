CopyErrorData

定義

  • 種別: function
  • 定義箇所: src/backend/utils/error/elog.c:1750

目的

現在スタックの最上位にあるエラー情報（ErrorData）のコピーを作成します。トランザクション境界を越えてエラー情報を保持する必要がある場合や、エラー処理後に情報を保存しておくために使用されます。

シグネチャまたは形状

ErrorData *CopyErrorData(void)

入力と出力

  • **入力**: なし（内部的な errordata スタックの最上位エントリを参照）。
  • **出力**: 複製された ErrorData 構造体へのポインタ。

前提条件と不変条件

  • CurrentMemoryContext が ErrorContext と等しくないこと（Assert により検証）。
  • 呼び出し元は、返された ErrorData を最終的に FreeErrorData で解放する責任を負います。

アルゴリズムまたは振る舞い

1. 現在のスタック深さを確認します。 2. palloc を使用して ErrorData 構造体自体のメモリを確保し、memcpy で内容をコピーします。 3. 構造体内の文字列ポインタ（message, filename, funcname, detail, hint 等）について、それぞれ pstrdup を実行して個別にメモリを確保し、コピーします。これは、元の文字列がJITコンパイルされたコードセグメントや、トランザクション終了時に解放される領域を指している可能性があるためです。 4. assoc_context に現在の CurrentMemoryContext を設定し、文字列の割り当て元コンテキストを記録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/connection.c
  • contrib/unaccent/unaccent.c
  • src/backend/executor/spi.c
  • src/backend/replication/logical/reorderbuffer.c
  • src/pl/plperl/plperl.c
  • **呼び出し先**:
  • palloc
  • memcpy
  • pstrdup

関連データ構造

  • ErrorData: エラーの詳細情報を保持する構造体。

失敗モードと境界条件

  • メモリ不足（OOM）が発生した場合、palloc または pstrdup がエラーをスローします。この関数内では recursion_depth をインクリメントしないため、メモリ不足によるエラーはエラーサブシステム内の問題とは見なされません。

保守担当者向けメモ

  • コピーされた文字列は、呼び出し時の CurrentMemoryContext に割り当てられます。
  • filename や funcname などの定数文字列もコピーされます。これは、それらがJIT生成コードセグメントを指している場合、トランザクション終了時にアンロードされる可能性があるためです。
