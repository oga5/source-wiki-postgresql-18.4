row_security_policy_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rowsecurity.c:673

目的

行レベルセキュリティ（RLS）ポリシーのリストを、ポリシー名（policy_name）に基づいて辞書順にソートするための比較関数です。

シグネチャまたは形状

static int row_security_policy_cmp(const ListCell *a, const ListCell *b)

入力と出力

  • **入力**:
  • a, b: RowSecurityPolicy 構造体を指す ListCell へのポインタ。
  • **出力**:
  • int: 比較結果。a が b より小さい場合は負の値、等しい場合は 0、大きい場合は正の値を返します。

前提条件と不変条件

  • 入力される ListCell は RowSecurityPolicy 構造体へのポインタを保持している必要があります。
  • RowSecurityPolicy 構造体の policy_name フィールドは NULL である可能性があります（拡張モジュールによるポリシー定義を想定）。

アルゴリズムまたは振る舞い

1. lfirst マクロを使用して ListCell から RowSecurityPolicy 構造体を取得します。 2. policy_name が NULL である場合のガード処理を行います。

  • 両方の policy_name が NULL の場合は 0 を返します。
  • a のみが NULL の場合は 1 を返し、b のみが NULL の場合は -1 を返します。

    • 3. 両方の policy_name が有効な場合、strcmp を使用して文字列の辞書順比較を行い、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/rewrite/rowsecurity.c 内の list_sort(policies, row_security_policy_cmp); で呼び出されます。
  • **呼び出し先**: strcmp（標準Cライブラリ）。

関連データ構造

  • RowSecurityPolicy: src/include/rewrite/rowsecurity.h で定義される、RLSポリシー情報を保持する構造体。

失敗モードと境界条件

  • policy_name が NULL の場合、プログラムがクラッシュしないよう明示的に処理されています。
  • ListCell が指すデータが RowSecurityPolicy ではない場合、型キャストにより未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は list_sort のコールバックとしてのみ使用されます。
  • ポリシー名の比較ロジックを変更する場合は、NULL 許容の順序付けルールが既存のポリシー管理と整合しているか確認してください。
