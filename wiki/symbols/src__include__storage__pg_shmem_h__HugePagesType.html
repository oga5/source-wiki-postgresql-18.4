HugePagesType

定義

  • 種別: enum
  • 定義箇所: src/include/storage/pg_shmem.h:51

目的

PostgreSQL における Huge Pages（巨大ページ）の有効化状態を管理・表現するために使用されます。

シグネチャまたは形状

typedef enum
{
    HUGE_PAGES_OFF,
    HUGE_PAGES_ON,
    HUGE_PAGES_TRY,
    HUGE_PAGES_UNKNOWN,
} HugePagesType;

入力と出力

  • 本シンボル自体は列挙型であり、入力や出力を持つ関数ではありません。
  • 設定値として huge_pages GUC 変数や、現在の状態を示す huge_pages_status の値として利用されます。

前提条件と不変条件

  • HUGE_PAGES_TRY は huge_pages 設定でのみ使用されます。
  • HUGE_PAGES_UNKNOWN は huge_pages_status でのみ使用されます。

アルゴリズムまたは振る舞い

  • HUGE_PAGES_OFF: Huge Pages を使用しません。
  • HUGE_PAGES_ON: Huge Pages の使用を強制します。
  • HUGE_PAGES_TRY: 可能であれば Huge Pages を使用し、失敗した場合は通常のメモリ割り当てにフォールバックします。
  • HUGE_PAGES_UNKNOWN: 状態が未確定であることを示します。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数情報は、提供されたソース抜粋からは特定できません。

関連データ構造

  • PGShmemHeader: 共有メモリセグメントのメタデータを保持する構造体であり、本列挙型と関連して共有メモリの構成管理に使用されます。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 本列挙型は src/include/storage/pg_shmem.h で定義されており、共有メモリの初期化プロセスにおいて OS へのメモリ要求方法を決定する際の判断基準となります。
  • HUGE_PAGES_TRY と HUGE_PAGES_UNKNOWN の用途が限定されている点に注意してください。
