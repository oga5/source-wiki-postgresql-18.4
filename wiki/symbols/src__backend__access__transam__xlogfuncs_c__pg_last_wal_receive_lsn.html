pg_last_wal_receive_lsn

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogfuncs.c:337

目的

スタンバイサーバーにおいて、WALレシーバーが受信し、ディスクにフラッシュした最新のWAL位置（LSN）を取得する。

シグネチャまたは形状

Datum pg_last_wal_receive_lsn(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数インターフェース経由の引数）
  • **出力**: Datum 型として、受信した最新のLSNを返す。受信したWALがない場合は NULL を返す。

前提条件と不変条件

  • 本関数はPostgreSQLの関数呼び出し規約に従う。
  • 内部的に GetWalRcvFlushRecPtr を呼び出し、WAL受信状態を参照する。

アルゴリズムまたは振る舞い

1. GetWalRcvFlushRecPtr(NULL, NULL) を呼び出し、現在受信済みのWALフラッシュ位置を取得する。 2. 取得したLSNが 0（初期値）である場合、WALがまだ受信されていないと判断し、PG_RETURN_NULL() を実行する。 3. それ以外の場合、PG_RETURN_LSN(recptr) を用いて取得したLSNをSQL層へ返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（ユーザーまたは管理ツールからのクエリ）。
  • **呼び出し先**: GetWalRcvFlushRecPtr（src/backend/access/transam/xlog_internal.h 等で定義されるWALレシーバーの内部API）。

関連データ構造

  • XLogRecPtr: LSN（Log Sequence Number）を保持するデータ型。

失敗モードと境界条件

  • WALレシーバーが動作していない、またはWALを一度も受信していない場合、GetWalRcvFlushRecPtr は 0 を返し、本関数は NULL を返す。

保守担当者向けメモ

  • 本関数はスタンバイサーバーでの利用を想定している。
  • GetWalRcvFlushRecPtr の挙動に依存しているため、WALレシーバーの内部実装に変更があった場合は本関数の動作も影響を受ける可能性がある。
