relopt_gen

定義

  • 種別: struct
  • 定義箇所: src/include/access/reloptions.h:64

目的

PostgreSQL のリレーションやテーブルスペースに対して設定可能なオプションを定義するための、共通ヘッダー構造体です。

シグネチャまたは形状

typedef struct relopt_gen
{
    const char *name;       /* リスト終了マーカーとして使用されるため先頭に配置 */
    const char *desc;       /* オプションの説明 */
    bits32      kinds;      /* 適用対象の種別（ビットマスク） */
    LOCKMODE    lockmode;   /* 変更時に必要なロックレベル */
    int         namelen;    /* 名前文字列の長さ */
    relopt_type type;       /* オプションのデータ型 */
} relopt_gen;

入力と出力

本構造体は、add_reloption や add_local_reloption 関数を通じてシステムに登録されます。入力として各フィールドの値を受け取り、リレーションオプションのメタデータとして保持されます。

前提条件と不変条件

  • name フィールドは構造体の先頭に配置する必要があります（リスト終了マーカーとしての判定に使用されるため）。
  • kinds は relopt_kind に基づくビットマスクである必要があります。

アルゴリズムまたは振る舞い

本構造体は、個別のオプション型（relopt_bool, relopt_int 等）の先頭メンバとして埋め込まれます。これにより、型を問わず共通のヘッダー情報としてオプションの検証や解析処理を行うことが可能になります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/reloptions.c 内の add_reloption および add_local_reloption 関数が、本構造体を引数として受け取り、グローバルまたはローカルなオプションリストに追加します。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接的な関数呼び出しは行いません。

関連データ構造

  • relopt_type: オプションの型を識別する列挙型。
  • relopt_kind: オプションの適用対象を分類するビットマスク。
  • relopt_value: 解析済みの値を保持する構造体。

失敗モードと境界条件

  • name が NULL の場合、オプションリストの終了とみなされます。
  • namelen が実際の文字列長と一致しない場合、オプションの検索や照合で不整合が生じる可能性があります。

保守担当者向けメモ

  • 新しいオプション型を追加する際は、必ずこの relopt_gen を先頭メンバとして定義してください。
  • lockmode の設定は、そのオプションがクエリ実行計画やデータ読み取りに与える影響を考慮し、適切に選択してください。
