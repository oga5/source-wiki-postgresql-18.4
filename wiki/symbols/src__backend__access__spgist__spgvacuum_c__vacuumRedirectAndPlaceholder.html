vacuumRedirectAndPlaceholder

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/spgist/spgvacuum.c:493

目的

SP-GiST インデックスページ内の不要な REDIRECT タプルを PLACEHOLDER に変換し、ページ末尾に連続する PLACEHOLDER を削除して領域を解放する。

シグネチャまたは形状

static void vacuumRedirectAndPlaceholder(Relation index, Relation heaprel, Buffer buffer)

入力と出力

  • **入力**:
  • index: 対象の SP-GiST インデックスリレーション。
  • heaprel: 対応するヒープリレーション。
  • buffer: 処理対象のインデックスページを含むバッファ。
  • **出力**: なし（副作用としてバッファの内容を更新）。

前提条件と不変条件

  • buffer はピン留めされ、ロックされている必要がある。
  • REDIRECT タプルの変換は、その XID が現在実行中のどのインデックススキャンからも参照されていない（GlobalVisTestIsRemovableXid が真）場合にのみ行われる。
  • 非 PLACEHOLDER タプルのオフセット番号は変更できないため、ページ末尾の PLACEHOLDER のみ削除可能。

アルゴリズムまたは振る舞い

1. ページ内のタプルを末尾から逆順に走査する。 2. REDIRECT タプルに対し、GlobalVisTest を用いて削除可能か判定し、可能であれば SPGIST_PLACEHOLDER に変換する。 3. 変換した REDIRECT の XID を追跡し、xlrec.snapshotConflictHorizon を更新する。 4. ページ末尾から連続する PLACEHOLDER を特定し、PageIndexMultiDelete を用いて一括削除する。 5. 変更が発生した場合、バッファをダーティにし、必要に応じて WAL (XLOG_SPGIST_VACUUM_REDIRECT) を記録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgvacuum.c 内の spgvacuumpage 等から複数回呼び出される。
  • **呼び出し先**: GlobalVisTestFor, GlobalVisTestIsRemovableXid, PageGetItem, PageIndexMultiDelete, XLogInsert 等の内部関数およびマクロ。

関連データ構造

  • SpGistPageOpaque: ページ内の REDIRECT および PLACEHOLDER の数を保持。
  • spgxlogVacuumRedirect: WAL 記録用の構造体。
  • SpGistDeadTuple: ページ内のタプル状態（tupstate）を保持する構造体。

失敗モードと境界条件

  • ページ内に REDIRECT や PLACEHOLDER が存在しない場合、処理は何も行わない。
  • ページ末尾に PLACEHOLDER が存在しない場合、削除処理はスキップされる。
  • 実行中のインデックススキャンが存在する場合、REDIRECT は削除されず保持される。

保守担当者向けメモ

  • 本関数は CRIT_SECTION 内で実行されるため、メモリ割り当てやエラー発生には注意が必要。
  • WAL 記録時に itemToPlaceholder 配列を登録しているが、firstPlaceholder 以降の削除対象タプルは再構築可能なため WAL には含めていない。
