BF_decode

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/crypt-blowfish.c:371

目的

bcrypt 形式の Base64 エンコードされた文字列をバイナリデータにデコードします。

シグネチャまたは形状

static int BF_decode(BF_word *dst, const char *src, int size);

入力と出力

  • dst: デコード結果を格納する BF_word 型のバッファへのポインタ。
  • src: デコード対象の Base64 文字列。
  • size: デコードするバイト数。
  • 戻り値: 常に 0 を返します。

前提条件と不変条件

  • src は有効な Base64 文字列である必要があります。
  • dst が指すメモリ領域は、size バイト以上の書き込みが可能である必要があります。

アルゴリズムまたは振る舞い

1. BF_safe_atoi64 マクロを使用して、Base64 文字を 6 ビットの値に変換します。 2. 4 文字の Base64 入力から 3 バイトのバイナリデータを復元し、dst に順次格納します。 3. dptr が end（dst + size）に達するまで、この変換処理をループします。 4. 各 1 バイトの書き込み後に dptr >= end をチェックし、指定サイズに達した時点で処理を中断します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/crypt-blowfish.c 内の _crypt_blowfish_rn 関数（634行目付近）から呼び出されます。
  • 呼び出し先: BF_safe_atoi64 マクロを使用します。

関連データ構造

  • BF_word: デコード先のデータ型として使用されます。

失敗モードと境界条件

  • src が不正な文字を含む場合、BF_safe_atoi64 の挙動に依存します。
  • size が 3 の倍数でない場合でも、ループ内の条件分岐により dptr が end に達した時点で安全に終了します。

保守担当者向けメモ

  • 本関数は bcrypt 特有の Base64 エンコーディングを前提としています。標準的な Base64 とは変換テーブルが異なる可能性があるため、変更時は注意してください。
  • 戻り値は常に 0 ですが、将来的なエラーハンドリングの拡張に備えてこのシグネチャが維持されています。
