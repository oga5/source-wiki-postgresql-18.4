replace_nestloop_param_var

定義

  • src/backend/optimizer/util/paramassign.c:413

目的

ネストループ結合において、上位レベルの Var を実行時に参照可能な PARAM_EXEC パラメータへ置換し、必要に応じて PlannerInfo の curOuterParams リストへ登録する。

シグネチャまたは形状

Param *replace_nestloop_param_var(PlannerInfo *root, Var *var)

入力と出力

  • **入力**:
  • root: 現在のクエリの PlannerInfo 構造体。
  • var: パラメータ化対象の Var ノード。
  • **出力**:
  • 置換後の Param ノード（paramkind は PARAM_EXEC）。

前提条件と不変条件

  • root->curOuterParams は、現在処理中のネストループ結合で必要とされる NestLoopParam のリストを保持している必要がある。
  • 同一の Var が既に curOuterParams に存在する場合、新規のパラメータIDは発行されず、既存のIDが再利用される。

アルゴリズムまたは振る舞い

1. root->curOuterParams を走査し、引数の var と一致する NestLoopParam が存在するか確認する。 2. 一致するものがあれば、その NestLoopParam が保持する paramno を使用して Param ノードを作成し、即座に返す。 3. 一致するものがない場合:

  • generate_new_exec_param を呼び出し、新しい PARAM_EXEC ID を発行する。
  • 新しい NestLoopParam ノードを作成し、paramval に var のコピーを格納する。
  • 作成した NestLoopParam を root->curOuterParams に追加する。
  • 新規作成した Param ノードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/createplan.c:5058
  • **呼び出し先**:
  • generate_new_exec_param (新規ID発行)
  • equal (リスト内の既存パラメータ検索)
  • makeNode, copyObject, lappend (ノード操作およびリスト管理)

関連データ構造

  • PlannerInfo: クエリ全体の最適化情報を保持。
  • NestLoopParam: ネストループ結合で受け渡されるパラメータの定義。
  • Param: 実行時に参照されるパラメータノード。

失敗モードと境界条件

  • var が curOuterParams に含まれていない場合、必ず新しい PARAM_EXEC ID が消費される。
  • copyObject(var) を使用しているため、メモリ不足時にはエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は Var 専用である。PlaceHolderVar を扱う場合は replace_nestloop_param_placeholdervar を使用すること。
  • curOuterParams はサブクエリやネストループの処理単位でリセットされるため、プラン作成の文脈を考慮して呼び出す必要がある。
