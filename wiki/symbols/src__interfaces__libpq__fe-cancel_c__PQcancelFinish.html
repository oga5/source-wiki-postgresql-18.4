PQcancelFinish

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-cancel.c:352

目的

キャンセル要求に使用した PGcancelConn オブジェクトを破棄し、関連するリソースを解放する。

シグネチャまたは形状

void PQcancelFinish(PGcancelConn *cancelConn);

入力と出力

  • **入力**: cancelConn - 破棄対象の PGcancelConn 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • cancelConn は PQcancelCreate 等によって初期化された有効なポインタであること。
  • cancelConn が NULL の場合の挙動については、内部で呼び出される PQfinish の仕様に依存する。

アルゴリズムまたは振る舞い

1. 引数として受け取った cancelConn 構造体内部の conn メンバ（PGconn 型）を参照する。 2. PQfinish(&cancelConn->conn) を呼び出し、接続に関連するソケットのクローズやメモリの解放を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/fe_utils/connect_utils.c:167
  • src/include/libpq/libpq-be-fe-helpers.h:450
  • src/test/isolation/isolationtester.c:963
  • src/test/modules/libpq_pipeline/libpq_pipeline.c:290, 402
  • **呼び出し先**:
  • PQfinish (libpq内部関数)

関連データ構造

  • PGcancelConn: キャンセル要求用の接続状態を保持する構造体。

失敗モードと境界条件

  • cancelConn が NULL の場合、PQfinish の実装に依存した動作となる。

保守担当者向けメモ

  • 本関数は PQcancelCreate で生成されたリソースのライフサイクルを終了させるための標準的なインターフェースである。
  • 内部で PQfinish を呼び出しているため、PGconn の終了処理に変更があった場合は本関数も影響を受ける可能性がある。
