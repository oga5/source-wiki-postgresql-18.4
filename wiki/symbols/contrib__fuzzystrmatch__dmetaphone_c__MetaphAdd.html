MetaphAdd

定義

  • 種別: function
  • 定義箇所: contrib/fuzzystrmatch/dmetaphone.c:377

目的

metastring 構造体が保持する文字列バッファに対し、指定された文字列を追加します。

シグネチャまたは形状

static void MetaphAdd(metastring *s, const char *new_str)

入力と出力

  • **入力**:
  • s: 操作対象の metastring 構造体へのポインタ。
  • new_str: 追加する文字列（const char *）。
  • **出力**: なし（void）。

前提条件と不変条件

  • new_str が NULL の場合、関数は何も行わずに即座に復帰します。
  • s が指す metastring は、NewMetaString 等によって適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. new_str が NULL でないことを確認します。 2. 追加する文字列の長さを計算します。 3. 現在のバッファ容量（s->bufsize）が不足している場合、IncreaseBuffer を呼び出してバッファを拡張します。 4. strcat を使用して、既存の文字列の末尾に new_str を連結します。 5. s->length を追加した文字列の分だけ更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: DoubleMetaphone 関数内の各発音ルール処理において、プライマリまたはオルタネートのコードを構築する際に頻繁に呼び出されます。
  • **呼び出し先**: IncreaseBuffer（バッファ拡張が必要な場合）。

関連データ構造

  • metastring: 文字列データ、現在の長さ、およびバッファサイズを管理する構造体。

失敗モードと境界条件

  • new_str が NULL の場合は安全に無視されます。
  • バッファ容量を超える追加が発生した場合、IncreaseBuffer が内部でメモリ再割り当てを行います。

保守担当者向けメモ

  • 本関数は PostgreSQL のメモリコンテキスト内で動作するため、IncreaseBuffer によるメモリ拡張は repalloc を介して行われます。
  • strcat を使用しているため、バッファの整合性が metastring 構造体の length と bufsize によって正しく維持されていることが前提となります。
