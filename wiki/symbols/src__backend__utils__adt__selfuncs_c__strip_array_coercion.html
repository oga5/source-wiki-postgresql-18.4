strip_array_coercion

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/selfuncs.c:1790

目的

式ツリーから、バイナリ互換性のある型変換（ArrayCoerceExpr や RelabelType）を再帰的に除去し、元のノードを抽出します。

シグネチャまたは形状

Node *strip_array_coercion(Node *node)

入力と出力

  • **入力**: Node *node（解析対象の式ツリーノード）
  • **出力**: Node *（変換が除去された後のノード、または変換が適用されていない場合は元のノード）

前提条件と不変条件

  • 入力ノードが ArrayCoerceExpr の場合、その要素式（elemexpr）が RelabelType であり、かつその引数が CaseTestExpr である場合に限り、バイナリ互換とみなして除去を行います。
  • RelabelType ノードは直接除去されます。

アルゴリズムまたは振る舞い

1. ループ内で以下の条件を順次判定し、該当するノードを剥がします。

  • ArrayCoerceExpr の場合: 要素式が RelabelType かつ CaseTestExpr をラップしている場合、acoerce->arg に進みます。それ以外はループを終了します。
  • RelabelType の場合: 無条件にその引数（arg）に進みます。

    • 2. 上記のいずれにも該当しないノードに到達した時点でループを終了し、現在のノードを返します。

呼び出し元と呼び出し先

  • **呼び出し先**:

関連データ構造

  • Node: PostgreSQL の解析ツリーの基本構造体。
  • ArrayCoerceExpr: 配列の型変換を表すノード。
  • RelabelType: 型の再ラベル付けを表すノード。
  • CaseTestExpr: CASE 式の評価中に使用されるプレースホルダーノード。

失敗モードと境界条件

  • node が NULL の場合、ループ条件により即座に終了し NULL を返します。
  • 認識できないノード型に遭遇した場合、変換を行わずにそのノードをそのまま返します。

保守担当者向けメモ

  • 本関数は、プランナが統計情報を取得する際に、不要な型変換ノードを無視して元の変数や式を特定するために使用されます。
  • RelabelType の除去は予期せぬケースとしてコメントされていますが、安全策として実装されています。変更を加える際は、プランナの選択率推定ロジックへの影響を考慮してください。
