CheckLogicalDecodingRequirements

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/logical.c:110

目的

論理デコーディングを開始またはレプリケーションスロットを作成する前に、現在のシステム状態が論理デコーディングを実行可能な要件を満たしているかを検証する。

シグネチャまたは形状

void CheckLogicalDecodingRequirements(void)

入力と出力

  • 入力: なし（グローバルなシステム状態を参照）
  • 出力: なし
  • 副作用: 要件を満たさない場合、ereport(ERROR, ...) を呼び出し処理を中断する。

前提条件と不変条件

  • wal_level が WAL_LEVEL_LOGICAL 以上であること。
  • 現在のセッションがデータベースに接続されていること（MyDatabaseId != InvalidOid）。
  • スタンバイサーバー上で実行中の場合、プライマリサーバー側の wal_level が WAL_LEVEL_LOGICAL 以上であること。

アルゴリズムまたは振る舞い

1. CheckSlotRequirements() を呼び出し、スロット関連の基本要件を検証する。 2. wal_level が論理デコーディングをサポートしていない場合、エラーを発生させる。 3. データベース接続が確立されていない場合、エラーを発生させる。 4. リカバリ中（スタンバイ）である場合、GetActiveWalLevelOnStandby() を使用してプライマリ側の wal_level を確認し、要件を満たさない場合はエラーを発生させる。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/replication/logical/logical.c (コンテキスト生成時)
  • src/backend/replication/logical/logicalfuncs.c (SQLインターフェース)
  • src/backend/replication/slotfuncs.c (スロット操作)
  • src/backend/replication/walsender.c (WAL送信プロセス)
  • 呼び出し先:
  • CheckSlotRequirements()
  • RecoveryInProgress()
  • GetActiveWalLevelOnStandby()
  • ereport()

関連データ構造

  • wal_level (グローバル変数)
  • MyDatabaseId (グローバル変数)

失敗モードと境界条件

  • wal_level が不十分な場合: ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラー終了。
  • データベース未接続時: ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラー終了。
  • スタンバイ側でプライマリの wal_level が不十分な場合: ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラー終了。

保守担当者向けメモ

  • 本関数に要件を追加する場合、RestoreSlotFromDisk() においても同様のチェックが必要になる可能性がある点に留意すること。
  • スタンバイ時のチェックには競合状態（race condition）の可能性があるが、wal_level 変更時の検証およびスロット作成・起動時の二重チェックによりリスクを低減している。
