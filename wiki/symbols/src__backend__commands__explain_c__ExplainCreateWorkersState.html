ExplainCreateWorkersState

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain.c:4838

目的

並列クエリ実行時の各ワーカーの状態を保持するための ExplainWorkersState 構造体を初期化し、メモリを割り当てる。

シグネチャまたは形状

static ExplainWorkersState *ExplainCreateWorkersState(int num_workers)

入力と出力

  • **入力**: num_workers (int) - 追跡対象とするワーカーの数。
  • **出力**: 初期化済みの ExplainWorkersState 構造体へのポインタ。

前提条件と不変条件

  • num_workers は非負の整数であること。
  • メモリ割り当て（palloc）が成功すること。

アルゴリズムまたは振る舞い

1. ExplainWorkersState 構造体サイズ分のメモリを割り当てる。 2. num_workers を構造体に格納する。 3. 以下の各配列に対して num_workers 分のメモリを割り当て、初期化する。

  • worker_inited: 各ワーカーの初期化状態（palloc0 でゼロ初期化）。
  • worker_str: 各ワーカーの出力文字列バッファ（palloc0 でゼロ初期化）。
  • worker_state_save: 各ワーカーの状態保存用整数配列。

    • 4. 構築した構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/explain.c:1372 において、プランステートのワーカー数に基づいて呼び出されている。
  • **呼び出し先**: palloc および palloc0（PostgreSQL のメモリ管理関数）。

関連データ構造

  • ExplainWorkersState: ワーカーごとの実行統計や状態を管理する構造体。
  • StringInfoData: worker_str で使用される文字列バッファ構造体。

失敗モードと境界条件

  • num_workers が 0 の場合、各配列にはサイズ 0 のメモリが割り当てられる。
  • メモリ不足時には palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は EXPLAIN コマンドの実行中に並列ワーカーの統計情報を収集する際に使用される。
  • worker_str や worker_state_save のメモリレイアウトを変更する場合は、本関数内の割り当てロジックを更新すること。
