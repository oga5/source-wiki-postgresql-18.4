AlignedAllocGetChunkSpace

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/alignedalloc.c:157

目的

アライメント調整されたメモリチャンクが占有する総メモリ量（オーバーヘッドを含む）を算出する。

シグネチャまたは形状

Size AlignedAllocGetChunkSpace(void *pointer)

入力と出力

  • **入力**: pointer - アライメント調整されたメモリ領域を指すポインタ。
  • **出力**: Size - チャンクが占有する総バイト数。

前提条件と不変条件

  • pointer は palloc_aligned() によって確保された有効なメモリ領域を指している必要がある。
  • MemoryChunk ヘッダーがポインタの直前に配置されていることを前提とする。

アルゴリズムまたは振る舞い

1. PointerGetMemoryChunk(pointer) を呼び出し、アライメント調整されたポインタから MemoryChunk ヘッダーを取得する。 2. VALGRIND_MAKE_MEM_DEFINED を使用して、ヘッダー領域を読み取り可能にする。 3. MemoryChunkGetBlock(redirchunk) を介して、アライメント調整前の「真の」メモリブロックを取得する。 4. GetMemoryChunkSpace(unaligned) を呼び出し、取得したブロックの占有サイズを算出する。 5. VALGRIND_MAKE_MEM_NOACCESS を使用して、ヘッダー領域を再びアクセス不可にする。 6. 算出されたサイズを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c: MCTX_ALIGNED_REDIRECT_ID 構造体の get_chunk_space フィールドから参照されている。
  • **呼び出し先**:
  • PointerGetMemoryChunk
  • MemoryChunkGetBlock
  • GetMemoryChunkSpace
  • VALGRIND_MAKE_MEM_DEFINED / VALGRIND_MAKE_MEM_NOACCESS

関連データ構造

  • MemoryChunk: アライメント調整されたメモリのメタデータを保持する構造体。

失敗モードと境界条件

  • pointer が無効な場合や、MemoryChunk ヘッダーが破損している場合の動作は未定義。

保守担当者向けメモ

  • 本関数は palloc_aligned で確保されたメモリに対するデコレータとして機能する。
  • MemoryChunk ヘッダーの配置位置に依存しているため、メモリレイアウトの変更時には注意が必要。
