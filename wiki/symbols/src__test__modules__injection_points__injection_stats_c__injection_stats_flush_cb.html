injection_stats_flush_cb

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_stats.c:70

目的

PostgreSQLの統計サブシステムにおいて、ローカルのインジェクションポイント統計（pending）を共有メモリ上の統計データ（shared_stats）へ同期（フラッシュ）するために使用されます。

シグネチャまたは形状

static bool injection_stats_flush_cb(PgStat_EntryRef *entry_ref, bool nowait);

入力と出力

  • **入力**:
  • entry_ref: 統計エントリへの参照。pending（ローカル）および shared_stats（共有メモリ）へのアクセスに使用。
  • nowait: ロック取得時に待機するかどうかを指定するフラグ。
  • **出力**:
  • bool: 統計のフラッシュに成功した場合は true、ロック取得失敗等により処理できなかった場合は false を返します。

前提条件と不変条件

  • entry_ref が有効な統計エントリを指している必要があります。
  • 共有メモリ上の統計データへの書き込みには、pgstat_lock_entry による排他制御が必須です。

アルゴリズムまたは振る舞い

1. entry_ref からローカルの統計データ（PgStat_StatInjEntry）と共有メモリ上の統計データ（PgStatShared_InjectionPoint）を取得します。 2. pgstat_lock_entry を呼び出し、共有メモリへのアクセス権を確保します。nowait が true でロックが取得できない場合は、処理を中断し false を返します。 3. ローカルの統計データに蓄積された numcalls を、共有メモリ上の統計エントリの numcalls に加算します。 4. pgstat_unlock_entry を呼び出してロックを解放し、true を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: pgstat サブシステムの統計フラッシュ処理から呼び出されます（injection_stats 構造体の .flush_pending_cb フィールドに登録）。
  • **呼び出し先**:
  • pgstat_lock_entry
  • pgstat_unlock_entry

関連データ構造

  • PgStat_EntryRef: 統計エントリの参照管理構造体。
  • PgStat_StatInjEntry: ローカル側の統計保持構造体。
  • PgStatShared_InjectionPoint: 共有メモリ側の統計保持構造体。

失敗モードと境界条件

  • nowait が true の場合、他のプロセスが既に当該エントリをロックしていると、加算処理を行わずに false を返します。

保守担当者向けメモ

  • 本関数は pgstat のカスタム統計種別実装の一部です。統計の加算はアトミックに行われる必要があるため、ロックの取得・解放の順序を厳守してください。
