TS_phrase_execute

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector_op.c:1608

目的

全文検索クエリ（tsquery）を tsvector に対して評価し、フレーズ検索や論理演算（AND, OR, NOT）に基づくマッチング結果と位置情報を算出する。

シグネチャまたは形状

static TSTernaryValue TS_phrase_execute(
    QueryItem *curitem, 
    void *arg, 
    uint32 flags,
    TSExecuteCallback chkcond,
    ExecPhraseData *data
)

入力と出力

  • **入力**:
  • curitem: 評価対象のクエリツリーノード。
  • arg: コールバック関数に渡すコンテキスト。
  • flags: 実行オプション（TS_EXEC_SKIP_NOT 等）。
  • chkcond: 語彙の存在を確認するコールバック関数。
  • data: マッチング結果（位置情報、幅、否定フラグ）を格納する構造体。
  • **出力**:
  • 戻り値: TS_YES（マッチ）、TS_NO（不一致）、TS_MAYBE（判定不能）。

前提条件と不変条件

  • 再帰呼び出しを行うため、スタックオーバーフローを防ぐための check_stack_depth() が呼び出される。
  • 割り込みチェックのため CHECK_FOR_INTERRUPTS() が実行される。
  • TS_MAYBE が返される場合、位置情報は信頼できないものとして扱われる。

アルゴリズムまたは振る舞い

1. **ノード判定**: curitem が QI_VAL（単一語）であれば、chkcond を呼び出して結果を返す。 2. **演算子処理**:

  • OP_NOT: 否定演算を実行。TS_EXEC_SKIP_NOT フラグがある場合は「どこでもマッチ」として扱う。
  • OP_PHRASE / OP_AND: 左右のサブツリーを再帰的に評価。OP_PHRASE の場合は距離（distance）に基づき data->width を計算する。
  • OP_OR: 左右のサブツリーを評価し、いずれかがマッチすれば TS_YES を返す。

    • 3. **位置情報の統合**: TS_phrase_output を呼び出し、演算子に応じた位置情報のマージを行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsvector_op.c 内の複数の箇所（TS_execute 関連のロジックや、OP_PHRASE を含むクエリ処理）。
  • **呼び出し先**: 自身（再帰）、chkcond（コールバック）、TS_phrase_output。

関連データ構造

  • QueryItem: クエリツリーのノード。
  • ExecPhraseData: マッチング時の位置情報、幅、否定状態を保持する構造体。
  • TSTernaryValue: 3値論理（YES/NO/MAYBE）の結果型。

失敗モードと境界条件

  • lmatch または rmatch が TS_MAYBE の場合、位置情報の算出が困難なため、結果として TS_MAYBE を返す。
  • OP_NOT 処理において、位置情報が空の場合の挙動が定義されている。
  • スタック深度が限界を超えた場合、エラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はクエリツリーを再帰的に走査するため、複雑なクエリに対してはスタック消費量に注意すること。
  • data 引数が NULL の場合、位置情報の計算をスキップする可能性があるため、呼び出し側での data の初期化状態に留意すること。
