SerializeMetrics

定義

  • 種別: struct
  • 定義箇所: src/include/commands/explain_dr.h:22

目的

EXPLAIN (SERIALIZE) コマンド実行時における、データシリアライズ処理のオーバーヘッド（バイト数、所要時間、バッファ使用量）を計測・保持するために使用される構造体です。

シグネチャまたは形状

typedef struct SerializeMetrics
{
	uint64		bytesSent;		/* # of bytes serialized */
	instr_time	timeSpent;		/* time spent serializing */
	BufferUsage bufferUsage;	/* buffers accessed during serialization */
} SerializeMetrics;

入力と出力

本構造体は関数への引数として渡されるか、あるいは DestReceiver 構造体の一部として保持されます。

  • 入力: EXPLAIN 実行中に各タプルが処理される際、各フィールドが累積的に更新されます。
  • 出力: GetSerializationMetrics() 関数を介して、収集された統計情報が EXPLAIN の出力結果生成のために取得されます。

前提条件と不変条件

  • EXPLAIN (SERIALIZE) が有効なコンテキストでのみ使用されます。
  • 統計情報は memset 等で初期化された後に累積されます。

アルゴリズムまたは振る舞い

1. CreateExplainSerializeDestReceiver により初期化されます。 2. クエリ実行中、タプルが受信機に渡されるたびに、シリアライズ処理のコストが計算され、本構造体の各フィールドに加算されます。 3. 最終的に GetSerializationMetrics を通じて、蓄積された値が読み取られます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/explain.c および src/backend/commands/explain_dr.c 内の各関数が、計測値の初期化、更新、取得のために本構造体を参照しています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • instr_time: executor/instrument.h で定義される時間計測用型。
  • BufferUsage: executor/instrument.h で定義されるバッファアクセス統計用構造体。
  • DestReceiver: tcop/dest.h で定義されるクエリ結果受信機の基底構造体。

失敗モードと境界条件

  • 統計情報のオーバーフローについては、uint64 型の範囲に依存します。
  • EXPLAIN (SERIALIZE) 以外のコンテキストで本構造体を使用した場合の動作は定義されていません。

保守担当者向けメモ

  • 本構造体はネットワーク送信をシミュレートする特殊な DestReceiver の一部として機能します。実際のネットワーク送信は行われません。
  • フィールドの追加や変更を行う際は、src/backend/commands/explain_dr.c 内の初期化処理（memset）および EXPLAIN 出力ロジックとの整合性に注意してください。
