relopt_int

定義

  • 種別: struct
  • 定義箇所: src/include/access/reloptions.h:97

目的

リレーションやテーブルスペースのオプション設定において、整数型（int）のパラメータ定義を保持するための構造体です。

シグネチャまたは形状

typedef struct relopt_int
{
    relopt_gen  gen;          /* 共通ヘッダー */
    int         default_val;  /* デフォルト値 */
    int         min;          /* 許容最小値 */
    int         max;          /* 許容最大値 */
} relopt_int;

入力と出力

  • **入力**: オプション定義時に default_val, min, max が指定されます。
  • **出力**: build_reloptions 等の処理を通じて、解析済みの整数値がシステム内で利用されます。

前提条件と不変条件

  • gen フィールドは、すべてのオプション型で共通のヘッダーとして機能します。
  • min および max は、ユーザーが指定した値の妥当性を検証するための境界値として使用されます。

アルゴリズムまたは振る舞い

  • reloptions.c 内の init_int_reloption 等の関数により初期化されます。
  • ユーザーが ALTER TABLE SET (...) 等で値を設定する際、この構造体に定義された min および max を用いて値の範囲チェックが行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/reloptions.c 内の intRelOpts 配列定義や、init_int_reloption を呼び出す各アクセスメソッドの初期化処理。
  • **呼び出し先**: relopt_gen を介して汎用的なオプション管理関数から参照されます。
  • ※具体的な呼び出し関係の全容は、ソースコードの静的解析結果に基づいています。

関連データ構造

  • relopt_gen: 本構造体の先頭メンバであり、オプションの共通属性を保持します。

失敗モードと境界条件

  • 設定値が min 未満または max を超える場合、検証処理においてエラーが発生します。

保守担当者向けメモ

  • 新しい整数型オプションを追加する際は、init_int_reloption を使用して定義してください。
  • min と max の設定が、対象となるパラメータの物理的・論理的な制約と一致していることを確認してください。
