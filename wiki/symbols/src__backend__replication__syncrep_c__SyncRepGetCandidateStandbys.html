SyncRepGetCandidateStandbys

定義

  • 種別: function
  • 定義箇所: src/backend/replication/syncrep.c:753

目的

同期レプリケーションの候補となるスタンバイサーバーのリストを収集し、現在の設定（PriorityまたはQuorum）に基づいて適切な候補数を抽出する。

シグネチャまたは形状

int SyncRepGetCandidateStandbys(SyncRepStandbyData **standbys)

入力と出力

  • **入力**: SyncRepStandbyData 構造体へのポインタのポインタ。
  • **出力**: 候補として選定されたスタンバイの数（int）。*standbys には割り当てられた配列が格納される。

前提条件と不変条件

  • SyncRepConfig が初期化されていること。
  • max_wal_senders 分のメモリ割り当てが成功すること。
  • 候補となるスタンバイは、WALSNDSTATE_STREAMING または WALSNDSTATE_STOPPING 状態であり、かつ sync_standby_priority > 0 である必要がある。

アルゴリズムまたは振る舞い

1. max_wal_senders 分の SyncRepStandbyData 配列をメモリ確保する。 2. SyncRepConfig が NULL の場合は即座に 0 を返す。 3. 全ての WalSnd プロセスを走査し、以下の条件を満たすものを候補として抽出する:

  • プロセスがアクティブ（pid != 0）。
  • 状態がストリーミングまたは停止中。
  • 同期スタンバイとして設定されている（priority > 0）。
  • 有効な flush 位置を持っている。

    • 4. SyncRepConfig->syncrep_method が SYNC_REP_PRIORITY であり、候補数 n が num_sync を超える場合、優先度順にソートし、上位 num_sync 個のみを返す。 5. SYNC_REP_QUORUM の場合は、全ての候補をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/syncrep.c:604
  • src/backend/replication/walsender.c:3937
  • **呼び出し先**:
  • palloc (メモリ確保)
  • SpinLockAcquire / SpinLockRelease (共有メモリ保護)
  • qsort (優先度によるソート)
  • standby_priority_comparator (比較関数)

関連データ構造

  • SyncRepStandbyData: スタンバイの進捗情報（write, flush, apply）や優先度を保持する構造体。
  • WalSnd: WAL送信プロセスの共有メモリ状態。
  • SyncRepConfig: 同期レプリケーションの動作モード（Priority/Quorum）を保持するグローバル設定。

失敗モードと境界条件

  • SyncRepConfig が NULL の場合、候補なしとして 0 を返す。
  • 候補となるスタンバイが存在しない場合、0 を返す。
  • XLogRecPtrIsInvalid(stby->flush) の場合、そのスタンバイは候補から除外される。

保守担当者向けメモ

  • 本関数は WalSndCtl->walsnds を走査するため、各要素のアクセスには SpinLock が必須である。
  • SyncRepStandbyData の配列は呼び出し元で解放（pfree）する必要がある点に注意すること。
