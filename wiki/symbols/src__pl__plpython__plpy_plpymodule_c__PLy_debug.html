PLy_debug

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_plpymodule.c:277

目的

Python スクリプトから呼び出された際に、PostgreSQL の DEBUG2 ログレベルでメッセージを出力するためのインターフェースを提供します。

シグネチャまたは形状

static PyObject *PLy_debug(PyObject *self, PyObject *args, PyObject *kw);

入力と出力

  • **入力**:
  • self: Python モジュールオブジェクトまたはメソッドのインスタンス。
  • args: Python から渡された位置引数（タプル）。
  • kw: Python から渡されたキーワード引数（辞書）。
  • **出力**:
  • PLy_output の戻り値である PyObject*（通常は Py_None）。

前提条件と不変条件

  • 本関数は Python の C API 規約に従い、METH_VARARGS | METH_KEYWORDS 形式で呼び出されることを前提としています。
  • 内部で呼び出される PLy_output が PostgreSQL のエラーハンドリング機構（ereport）を介するため、PostgreSQL のメモリコンテキストが有効である必要があります。

アルゴリズムまたは振る舞い

1. Python から受け取った引数 args および kw を PLy_output 関数に渡します。 2. PLy_output に対してログレベルとして DEBUG2 を指定します。 3. PLy_output が実行した ereport の結果をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Python インタプリタ（plpy.debug メソッド経由）。
  • **呼び出し先**: PLy_output（src/pl/plpython/plpy_elog.c に定義）。
  • **参照**: src/pl/plpython/plpy_plpymodule.c 内の PLy_methods テーブルにて、"debug" という名前で公開されています。

関連データ構造

  • PyObject: Python のオブジェクト型。
  • DEBUG2: PostgreSQL のログレベル定数。

失敗モードと境界条件

  • PLy_output 内部で PostgreSQL のエラーが発生した場合、PG_TRY/PG_CATCH ブロックによりキャッチされ、Python 例外として再送出されます。
  • 引数の解析に失敗した場合、PLy_output が適切な Python 例外を発生させます。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的なログ出力ロジックは PLy_output に依存しています。ログ出力の挙動を変更する場合は PLy_output を参照してください。
  • PLy_methods テーブルへの登録により Python から利用可能になっているため、シグネチャを変更する場合は PLy_methods の定義と整合性を取る必要があります。
