getTokenTypes

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tsearchcmds.c:1228

目的

指定されたテキスト検索パーサーがサポートするトークン型を検証し、名前とIDのペアのリストとして取得します。

シグネチャまたは形状

List *getTokenTypes(Oid prsId, List *tokennames)

入力と出力

  • **入力**:
  • prsId: テキスト検索パーサーの OID。
  • tokennames: 検証対象のトークン名（String ノードのリスト）。
  • **出力**:
  • List *: TSTokenTypeItem 構造体のリスト。成功時は指定されたトークン名に対応する ID と名前のペアを返し、空の入力に対しては NIL を返します。

前提条件と不変条件

  • prsId に対応するパーサーキャッシュが存在すること。
  • パーサーの lextype メソッド（lextypeOid）が定義されていること。
  • tokennames に含まれる各トークン名が、パーサーの lextype メソッドが返すトークン型定義リスト内に存在すること。

アルゴリズムまたは振る舞い

1. lookup_ts_parser_cache を用いてパーサー情報を取得します。 2. tokennames が空であれば NIL を返します。 3. パーサーの lextypeOid を呼び出し、パーサーがサポートするトークン型のリスト（LexDescr）を取得します。 4. 指定された tokennames を順に走査します。

  • 既に結果リストに含まれている場合はスキップします。
  • LexDescr リストを検索し、名前が一致するトークン型が見つかれば、TSTokenTypeItem を割り当てて結果リストに追加します。
  • 一致するものが見つからない場合はエラーを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/tsearchcmds.c 内の DefineTSConfiguration および AlterTSConfiguration から呼び出されます。
  • **呼び出し先**: lookup_ts_parser_cache, OidFunctionCall1, tstoken_list_member, palloc0, pstrdup, lfirst_node, lappend, ereport 等。

関連データ構造

  • TSTokenTypeItem: トークン ID (num) と名前 (name) を保持する構造体。
  • LexDescr: パーサーが提供するトークン型の定義情報。
  • TSParserCacheEntry: パーサーのキャッシュ情報。

失敗モードと境界条件

  • lextype メソッドが定義されていない場合、ERROR をスローします。
  • 指定されたトークン名がパーサーの定義に含まれていない場合、ERRCODE_INVALID_PARAMETER_VALUE を伴う ERROR をスローします。

保守担当者向けメモ

  • 本関数は pg_ts_config の設定変更時に、トークンと辞書のマッピングを検証するために使用されます。
  • lextype メソッドの戻り値である LexDescr 配列の終端は lexid が 0 であることで判定されます。
