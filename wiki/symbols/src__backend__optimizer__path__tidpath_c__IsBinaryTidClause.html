IsBinaryTidClause

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/tidpath.c:76

目的

与えられた RestrictInfo が、特定のテーブルの ctid 列と定数（または擬似定数）を比較する二項演算子式であるかを判定します。

シグネチャまたは形状

static bool IsBinaryTidClause(RestrictInfo *rinfo, RelOptInfo *rel)

入力と出力

  • **入力**:
  • rinfo: 評価対象の RestrictInfo 構造体。
  • rel: 比較対象の ctid 列を持つ RelOptInfo 構造体。
  • **出力**:
  • bool: 条件が ctid を含む二項演算子式であり、かつ他方の引数が定数（または擬似定数）である場合に true を返します。

前提条件と不変条件

  • rinfo->clause は有効な式である必要があります。
  • rel は ctid 列を含むリレーションである必要があります。

アルゴリズムまたは振る舞い

1. rinfo->clause が二項演算子式（OpExpr）であり、引数が2つであることを確認します。 2. 引数のいずれか一方が IsCTIDVar を通じて対象リレーションの ctid 列であるかを確認します。 3. ctid ではない方の引数が、対象リレーションの列を含んでいないこと（bms_is_member で判定）、および揮発性関数（contain_volatile_functions）を含んでいないことを確認します。 4. 上記すべてを満たす場合に true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/tidpath.c 内の関数（TidQualFromRestrictInfoList 等から間接的に参照される可能性あり）。
  • **呼び出し先**:
  • is_opclause
  • IsCTIDVar
  • bms_is_member
  • contain_volatile_functions

関連データ構造

  • RestrictInfo: 述語条件を保持する構造体。
  • RelOptInfo: リレーションの最適化情報を保持する構造体。
  • OpExpr: 演算子式ノード。
  • Var: 変数ノード。

失敗モードと境界条件

  • rinfo->clause が OpExpr ではない場合、または引数が2つではない場合は false を返します。
  • ctid を含む式であっても、他方の引数が対象リレーションの列に依存している場合や、揮発性関数を含む場合は false を返します。

保守担当者向けメモ

  • 本関数は TidPath 生成の適格性判定において、条件式が定数比較であることを保証する重要な役割を担います。
  • IsCTIDVar の判定ロジックに変更があった場合、本関数の挙動も影響を受ける可能性があります。
