XidCacheStatus

定義

  • 種別: struct
  • 定義箇所: src/include/storage/proc.h:41

目的

バックエンドプロセスが保持するサブトランザクションID（subxid）のキャッシュ状態を管理するための構造体です。

シグネチャまたは形状

typedef struct XidCacheStatus
{
    uint8       count;      /* キャッシュされたsubxidの数 */
    bool        overflowed; /* subxidsがオーバーフローしたか否か */
} XidCacheStatus;

入力と出力

  • 本構造体は ProcGlobal->subxidStates 配列の一部として共有メモリ上に配置されます。
  • count は PGPROC_MAX_CACHED_SUBXIDS を上限とします。

前提条件と不変条件

  • count は常に PGPROC_MAX_CACHED_SUBXIDS 以下である必要があります。
  • overflowed が true の場合、キャッシュされたサブトランザクションIDのリストは完全ではないことを示します。

アルゴリズムまたは振る舞い

  • プロセスがサブトランザクションを開始・終了する際、この構造体を通じてキャッシュの有効性とサイズが追跡されます。
  • ProcGlobal を介して全バックエンドの状態がミラーリングされており、ProcArray スキャン処理などで参照されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照・更新）**:
  • src/backend/access/transam/varsup.c
  • src/backend/storage/ipc/procarray.c
  • src/backend/storage/lmgr/proc.c
  • **呼び出し先**:
  • 本構造体はデータ構造であり、特定の関数を呼び出すものではありません。

関連データ構造

  • PGPROC: 各バックエンドの主要なプロセス制御構造体。
  • PROC_HDR: subxidStates 配列を保持するグローバルヘッダー。

失敗モードと境界条件

  • count が上限に達した際に overflowed フラグが適切にセットされない場合、サブトランザクションの可視性判定に不整合が生じる可能性があります。

保守担当者向けメモ

  • PGPROC 内の subxidStatus フィールドと ProcGlobal->subxidStates 配列は同期して管理される必要があります。
  • ProcArrayLock 等の適切なロックなしに本構造体のフィールドを更新することは避けてください。
