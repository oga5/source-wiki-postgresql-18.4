CheckAndCreateToastTable

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/toasting.c:77

目的

指定されたリレーションに対してTOASTテーブルの作成処理を仲介し、リレーションのオープンとクローズを管理するラッパー関数です。

シグネチャまたは形状

static void CheckAndCreateToastTable(Oid relOid, Datum reloptions, LOCKMODE lockmode, bool check, Oid OIDOldToast)

入力と出力

  • **入力**:
  • relOid: 対象リレーションのOID。
  • reloptions: TOASTテーブルに適用するオプション（Datum形式）。
  • lockmode: リレーションを開く際に使用するロックモード。
  • check: TOASTテーブルが必要かどうかの判定を行うかを示すフラグ。
  • OIDOldToast: 既存のTOASTテーブルOID（存在する場合）。
  • **出力**: なし（void）。

前提条件と不変条件

  • relOid で指定されたリレーションは、呼び出し元で適切にロックされている必要があります。
  • 内部で table_open を実行するため、呼び出し元でリレーションが既に開かれている場合は注意が必要です。

アルゴリズムまたは振る舞い

1. table_open を使用して、指定された relOid と lockmode でリレーションを開きます。 2. create_toast_table を呼び出し、TOASTテーブルの構築、インデックス作成、カタログ更新、依存関係の登録といった一連の作業を委譲します。 3. table_close を使用して、リレーションをクローズします（ロックは保持されます）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/toasting.c 内の各所（NewRelationCreateToastTable 等の公開インターフェースから呼び出されます）。
  • **呼び出し先**:
  • table_open
  • create_toast_table
  • table_close

関連データ構造

  • Relation: table_open によって取得されるリレーション記述子。

失敗モードと境界条件

  • table_open が失敗した場合、例外が発生し処理が中断されます。
  • create_toast_table 内で発生するカタログ操作やメモリ不足等のエラーは、本関数を通じて伝播します。

保守担当者向けメモ

  • 本関数は create_toast_table の単純なラッパーです。ロジックの変更が必要な場合は、主に create_toast_table 側を確認してください。
  • table_close に NoLock を渡しているため、本関数終了後も呼び出し元で指定したロックは維持されます。
