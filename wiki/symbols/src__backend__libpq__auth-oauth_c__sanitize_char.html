sanitize_char

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-oauth.c:309

目的

認証ログ出力等のために、任意の文字を安全に表示可能な文字列形式へ変換する。

シグネチャまたは形状

static char *sanitize_char(char c);

入力と出力

  • **入力**: char c (変換対象の文字)
  • **出力**: char * (変換結果を保持する静的バッファへのポインタ)

前提条件と不変条件

  • 戻り値は静的バッファ (static char buf[5]) を指すため、呼び出しごとに内容が上書きされる。
  • 戻り値の文字列は常にヌル終端される。

アルゴリズムまたは振る舞い

1. 入力文字 c が表示可能範囲（ASCII 0x21 から 0x7E まで）にあるか判定する。 2. 範囲内の場合、'c' の形式（シングルクォートで囲った文字）でバッファに書き込む。 3. 範囲外の場合、0xXX の形式（16進数表記）でバッファに書き込む。 4. バッファへのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-oauth.c 内の各所（トークン解析やフラグ検証時のログ出力用）。
  • **呼び出し先**: 標準ライブラリの snprintf を使用。

関連データ構造

  • なし（静的バッファ buf[5] を内部的に使用）。

失敗モードと境界条件

  • 入力文字が 0x21 未満または 0x7E より大きい場合は、常に16進数表記へフォールバックする。
  • 静的バッファを使用しているため、マルチスレッド環境や同一行での複数回呼び出しには注意が必要（本関数は libpq の認証プロセス内で使用される）。

保守担当者向けメモ

  • src/backend/libpq/auth-scram.c にも同名の関数が存在する。修正時は両方の実装の一貫性に留意すること。
  • バッファサイズ 5 は、0x + 2桁の16進数 + ヌル終端文字で構成されるため、現在の実装に対して十分である。
