fix_infomask_from_infobits

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_xlog.c:316

目的

WALレコードに記録された圧縮された情報ビット（infobits）を基に、タプルヘッダーの infomask および infomask2 フィールドを復元・更新します。

シグネチャまたは形状

static void fix_infomask_from_infobits(uint8 infobits, uint16 *infomask, uint16 *infomask2)

入力と出力

  • **入力**:
  • infobits: WALレコードから読み取られた状態ビット（XLHL_* 定数）。
  • infomask: 更新対象のタプルヘッダー t_infomask へのポインタ。
  • infomask2: 更新対象のタプルヘッダー t_infomask2 へのポインタ。
  • **出力**: なし（ポインタ経由で infomask および infomask2 を直接更新）。

前提条件と不変条件

  • infomask および infomask2 は有効なメモリ領域を指している必要があります。
  • HEAP_XMAX_SHR_LOCK ビットはこの関数では考慮されません。

アルゴリズムまたは振る舞い

1. infomask から既存のロック関連フラグ（HEAP_XMAX_IS_MULTI, HEAP_XMAX_LOCK_ONLY, HEAP_XMAX_KEYSHR_LOCK, HEAP_XMAX_EXCL_LOCK）を一度クリアします。 2. infomask2 から HEAP_KEYS_UPDATED フラグをクリアします。 3. infobits の各ビットを検査し、対応するフラグを infomask および infomask2 に再設定します。

  • XLHL_XMAX_IS_MULTI → HEAP_XMAX_IS_MULTI
  • XLHL_XMAX_LOCK_ONLY → HEAP_XMAX_LOCK_ONLY
  • XLHL_XMAX_EXCL_LOCK → HEAP_XMAX_EXCL_LOCK
  • XLHL_XMAX_KEYSHR_LOCK → HEAP_XMAX_KEYSHR_LOCK
  • XLHL_KEYS_UPDATED → HEAP_KEYS_UPDATED

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c:2725
  • src/backend/access/heap/heapam_xlog.c 内の各WALリプレイ関数（heap_xlog_insert, heap_xlog_update, heap_xlog_lock 等）。
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行いません。

関連データ構造

  • HeapTupleHeaderData（t_infomask, t_infomask2 を含む）

失敗モードと境界条件

  • ポインタが NULL の場合、セグメンテーション違反が発生します。
  • infobits に未定義のビットが含まれていても、本関数はそれを無視して処理を継続します。

保守担当者向けメモ

  • 本関数はWALリプレイ時にタプルのロック状態を正確に再現するために不可欠です。新しいロックタイプやフラグをヒープタプルに追加する場合は、本関数のビットマスク処理も併せて更新する必要があります。
