AddStmtToCache

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/prepare.c:504

目的

ECPGの自動プリペア機能において、新しいSQLステートメントをキャッシュエントリに登録する。

シグネチャまたは形状

static int AddStmtToCache(int lineno, const char *stmtID, const char *connection, int compat, const char *ecpgQuery)

入力と出力

  • **入力**:
  • lineno: 呼び出し元の行番号。
  • stmtID: ステートメント識別子。
  • connection: 接続名。
  • compat: 互換性レベル。
  • ecpgQuery: キャッシュ対象のSQLクエリ文字列。
  • **出力**:
  • 成功時: キャッシュ内のエントリ番号（int）。
  • 失敗時: -1。

前提条件と不変条件

  • stmtCacheEntries が未初期化の場合は、内部で ecpg_alloc を用いてメモリ確保を行う。
  • キャッシュはバケット単位で管理され、各バケットは固定数のエントリを持つ。

アルゴリズムまたは振る舞い

1. HashStmt を呼び出し、クエリに対応するバケットの開始インデックスを算出する。 2. 指定されたバケット内で空きエントリを探索する。 3. 空きエントリが見つからない場合、実行回数（execs）が最も少ないエントリを置換対象として選定する。 4. ecpg_freeStmtCacheEntry を呼び出し、選定したエントリの既存データを解放する。 5. 新しいクエリ情報（ecpgQuery の複製、stmtID 等）をエントリに書き込み、エントリ番号を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/prepare.c 内の関数（ecpg_auto_prepare 等からの呼び出しが示唆される）。
  • **呼び出し先**:
  • ecpg_alloc / ecpg_freeStmtCacheEntry / ecpg_strdup（メモリ管理）
  • HashStmt（ハッシュ計算）

関連データ構造

  • stmtCacheEntries: キャッシュエントリを保持するグローバルな配列。
  • stmtCacheEntry: 各ステートメントの情報を保持する構造体。

失敗モードと境界条件

  • ecpg_alloc によるメモリ確保失敗時に -1 を返す。
  • ecpg_freeStmtCacheEntry が失敗した場合に -1 を返す。
  • キャッシュが満杯の場合、最も使用頻度の低いエントリが強制的に上書きされる。

保守担当者向けメモ

  • stmtCacheEntries の初期化は初回呼び出し時に行われるため、スレッドセーフティについては呼び出し元のコンテキストを確認すること。
  • ecpg_strdup で複製されたクエリ文字列の解放は、ecpg_freeStmtCacheEntry の責務であるため、メモリリークを防ぐには両関数の整合性が重要である。
