pgp_mpi_hash

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-mpi.c:118

目的

OpenPGP 形式の多倍長整数（MPI）をハッシュ計算対象のデータストリームに含めるためのユーティリティ関数です。

シグネチャまたは形状

int pgp_mpi_hash(PX_MD *md, PGP_MPI *n)

入力と出力

  • **入力**:
  • md: ハッシュ計算の状態を保持する PX_MD 構造体へのポインタ。
  • n: ハッシュ対象の PGP_MPI 構造体へのポインタ。
  • **出力**:
  • 常に 0 を返します。

前提条件と不変条件

  • n が指す PGP_MPI 構造体は有効であり、data フィールドには少なくとも bytes フィールドで指定された長さのメモリが確保されている必要があります。
  • md は px_md_update を呼び出し可能な有効な状態である必要があります。

アルゴリズムまたは振る舞い

1. n->bits（ビット長）をビッグエンディアン形式の 2 バイトデータとして抽出します。 2. px_md_update を呼び出し、抽出した 2 バイトのヘッダーをハッシュコンテキスト md に追加します。 3. px_md_update を呼び出し、n->data が保持する MPI の本体データ（n->bytes バイト分）をハッシュコンテキスト md に追加します。 4. 処理完了後、0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-pubkey.c 内の各関数（ElGamal、RSA、DSA の公開鍵および秘密鍵の処理において、鍵パラメータのハッシュ計算に使用）。
  • **呼び出し先**:
  • px_md_update（px.h で定義されたハッシュ更新インターフェース）。

関連データ構造

  • PGP_MPI: MPI データを保持する構造体（ビット数、バイト数、データ本体へのポインタ）。
  • PX_MD: ハッシュ計算のコンテキストを保持する構造体。

失敗モードと境界条件

  • 本関数自体はエラーを返しません。
  • n が NULL の場合、あるいは n->data が不正なポインタである場合はセグメンテーションフォールトが発生する可能性があります。

保守担当者向けメモ

  • 本関数は MPI のシリアライズ形式（2 バイトのビット長 + データ本体）に厳密に従ってハッシュ対象を構築します。MPI のデータ構造に変更を加える際は、このシリアライズ順序との整合性に注意してください。
