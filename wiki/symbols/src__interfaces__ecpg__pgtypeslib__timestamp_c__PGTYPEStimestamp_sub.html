PGTYPEStimestamp_sub

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/timestamp.c:792

目的

2つの timestamp 値の差分を計算し、その結果を interval 構造体に格納します。

シグネチャまたは形状

int PGTYPEStimestamp_sub(timestamp * ts1, timestamp * ts2, interval * iv)

入力と出力

  • **入力**:
  • ts1: 減算される側のタイムスタンプ（timestamp *）
  • ts2: 減算する側のタイムスタンプ（timestamp *）
  • iv: 結果を格納するインターバル構造体（interval *）
  • **出力**:
  • 成功時は 0 を返します。
  • エラー時は PGTYPES_TS_ERR_EINFTIME を返します。

前提条件と不変条件

  • ts1 および ts2 は、有限な値である必要があります。
  • iv は書き込み可能な有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. TIMESTAMP_NOT_FINITE マクロを用いて、ts1 または ts2 が有限値（infinity 等ではない）かを確認します。 2. 有限でない場合は PGTYPES_TS_ERR_EINFTIME を返して終了します。 3. 有限である場合、*ts1 - *ts2 の差分を計算し、iv->time に代入します。 4. iv->month を 0 に初期化します。 5. 0 を返して正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c:652 からの呼び出しが確認されています。
  • **呼び出し先**:
  • 内部マクロ TIMESTAMP_NOT_FINITE を使用しています。

関連データ構造

  • timestamp: マイクロ秒単位の int64 として定義されています。
  • interval: 期間を保持する構造体で、本関数では time フィールドと month フィールドを操作します。

失敗モードと境界条件

  • 入力されたタイムスタンプのいずれかが infinity または -infinity である場合、計算を行わずエラーコード PGTYPES_TS_ERR_EINFTIME を返します。

保守担当者向けメモ

  • 本関数は月単位の差分を考慮せず、iv->month を常に 0 に設定します。月を跨ぐ期間計算が必要な場合は、別のロジックを検討する必要があります。
