init_dummy_sjinfo

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/joinrels.c:660

目的

SpecialJoinInfo 構造体を、結合制約を持たない単純な内部結合（Inner Join）として初期化する。

シグネチャまたは形状

void init_dummy_sjinfo(SpecialJoinInfo *sjinfo, Relids left_relids, Relids right_relids)

入力と出力

  • **入力**:
  • sjinfo: 初期化対象の SpecialJoinInfo 構造体へのポインタ。
  • left_relids: 左辺のリレーションIDセット。
  • right_relids: 右辺のリレーションIDセット。
  • **出力**: なし（sjinfo が指すメモリ領域が直接書き換えられる）。

前提条件と不変条件

  • sjinfo は有効なメモリ領域を指している必要がある。
  • left_relids および right_relids は、結合に関与するリレーションを正しく保持している必要がある。

アルゴリズムまたは振る舞い

1. sjinfo->type を T_SpecialJoinInfo に設定する。 2. min_lefthand, min_righthand, syn_lefthand, syn_righthand にそれぞれ指定された left_relids と right_relids を代入する。 3. jointype を JOIN_INNER に設定する。 4. ojrelid を 0 に設定し、関連するポインタ（commute_above_l 等）を NULL に初期化する。 5. 複雑な結合制約に関連するフラグ（lhs_strict 等）を false に、リスト（semi_operators 等）を NIL に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/costsize.c
  • src/backend/optimizer/path/joinrels.c
  • src/backend/optimizer/util/orclauses.c
  • **呼び出し先**:
  • 本関数は特定の関数を呼び出さない（構造体のフィールドへの代入のみを行う）。

関連データ構造

  • SpecialJoinInfo: 結合の制約情報を保持する構造体。
  • Relids: リレーションIDの集合を表す型。

失敗モードと境界条件

  • sjinfo が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • 本関数は構造体の全フィールドを網羅的に初期化するわけではなく、一部のフィールドは未設定のままとなる（ソースコード上の明示的な意図による）。

保守担当者向けメモ

  • 本関数は、オプティマイザが結合の正当性チェックを行う際に、制約が存在しないケースを便宜的に扱うために使用される。
  • 構造体のすべてのフィールドを初期化するわけではないため、この関数で初期化した SpecialJoinInfo を使用する際は、未設定のフィールドに依存しないよう注意すること。
