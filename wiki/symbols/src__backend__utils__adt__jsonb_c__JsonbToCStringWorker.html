JsonbToCStringWorker

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb.c:490

目的

JsonbContainer 構造を走査し、JSON 文字列形式に変換して StringInfo バッファへ書き出す汎用ワーカ関数。

シグネチャまたは形状

char *JsonbToCStringWorker(StringInfo out, JsonbContainer *in, int estimated_len, bool indent)

入力と出力

  • **入力**:
  • out: 出力先バッファ（NULL の場合は新規作成）。
  • in: 変換対象の JsonbContainer。
  • estimated_len: バッファの初期確保サイズ（負の場合は 64 バイト）。
  • indent: 整形出力（インデント）を行うかどうかのフラグ。
  • **出力**: 変換後の文字列データ（out->data）。

前提条件と不変条件

  • JsonbIterator を使用してバイナリツリーを走査する。
  • 処理終了時に level が 0 に戻ることを Assert で保証している。
  • jsonb_put_escaped_value を介して値のエスケープ処理を行う。

アルゴリズムまたは振る舞い

1. JsonbIteratorInit でイテレータを初期化し、JsonbIteratorNext をループで呼び出す。 2. トークン種別（WJB_BEGIN_ARRAY, WJB_KEY 等）に応じて StringInfo に文字列を追記する。 3. indent が有効な場合、add_indent を呼び出し、階層に応じたインデントと改行を挿入する。 4. WJB_KEY 処理時、値がオブジェクトや配列の場合は再帰的な構造を扱うために redo_switch を用いてイテレータの再評価を行う。 5. WJB_END_ARRAY および WJB_END_OBJECT で階層レベルを減算する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • JsonbToCString (整形なし)
  • JsonbToCStringIndent (整形あり)
  • **呼び出し先**:
  • JsonbIteratorInit, JsonbIteratorNext
  • jsonb_put_escaped_value
  • add_indent
  • makeStringInfo, enlargeStringInfo, appendStringInfoCharMacro, appendBinaryStringInfo

関連データ構造

  • JsonbContainer: 変換対象のバイナリ表現。
  • JsonbIterator: 走査用イテレータ。
  • StringInfo: 出力用バッファ構造体。

失敗モードと境界条件

  • 不明なイテレータトークンが検出された場合、elog(ERROR) で停止する。
  • level が 0 以外で終了した場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数は jsonb 型のシリアライズにおける中心的なロジックである。
  • インデント処理において、カンマ後のスペース挿入ルール（ispaces）が整形フラグに依存している点に注意すること。
  • redo_switch は、キーに対応する値がネストされた構造である場合に、イテレータを適切に制御するためのフラグとして機能している。
