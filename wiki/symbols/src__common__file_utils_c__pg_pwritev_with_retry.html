pg_pwritev_with_retry

定義

  • 種別: function
  • 定義箇所: src/common/file_utils.c:658

目的

pwritev システムコールをラップし、部分的な書き込みが発生した場合でも、すべてのデータが書き込まれるまで自動的に再試行を行うことで、堅牢なI/O操作を実現します。

シグネチャまたは形状

ssize_t pg_pwritev_with_retry(int fd, const struct iovec *iov, int iovcnt, off_t offset);

入力と出力

  • **入力**:
  • fd: 書き込み対象のファイルディスクリプタ。
  • iov: 書き込むデータのベクトル配列。
  • iovcnt: iov 配列の要素数。
  • offset: 書き込みを開始するファイルオフセット。
  • **出力**:
  • 成功時: 書き込まれた合計バイト数を返します。
  • 失敗時: -1 を返し、errno が設定されます。

前提条件と不変条件

  • iovcnt は PG_IOV_MAX 以下である必要があります。これを超える場合、errno に EINVAL が設定され、即座に -1 を返します。
  • 内部で iov のコピーを保持するため、呼び出し元が渡した iov 配列の内容は再試行ループ中に変更されません。

アルゴリズムまたは振る舞い

1. iovcnt が PG_IOV_MAX を超えていないか検証します。 2. do-while ループ内で以下の処理を繰り返します。

  • pg_pwritev を呼び出し、データを書き込みます。
  • 書き込み結果が負であればエラーとして終了します。
  • 書き込み成功分だけ sum（合計バイト数）と offset を更新します。
  • compute_remaining_iovec を呼び出し、未書き込みのデータ量を計算して iov_copy に格納し、次のループで参照される iov を更新します。

    • 3. すべてのデータが書き込まれるまでループを継続し、完了後に合計バイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/smgr/md.c 内の pg_pwrite_zeros（FileZero経由）から呼び出されます。
  • **呼び出し先**: pg_pwritev（プラットフォーム依存のI/O）、compute_remaining_iovec。

関連データ構造

  • struct iovec: ベクトルI/Oを定義する標準構造体。
  • PG_IOV_MAX: iovec 配列の最大要素数。

失敗モードと境界条件

  • iovcnt > PG_IOV_MAX: EINVAL を返し失敗します。
  • pg_pwritev がエラーを返した場合: そのままエラーを伝播します。
  • SIMULATE_SHORT_WRITE が定義されている場合: デバッグ目的で書き込みサイズが強制的に制限される可能性があります。

保守担当者向けメモ

  • 本関数は FRONTEND 定義時にのみコンパイルされるコードパスに含まれる可能性があります。
  • compute_remaining_iovec は、部分書き込み後の iovec 状態を正しく再構築するために不可欠です。修正時はこの依存関係に注意してください。
