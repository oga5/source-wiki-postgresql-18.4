IsBinaryCoercible

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_coerce.c:3029

目的

あるデータ型（srctype）から別のデータ型（targettype）へ、実行時に変換関数を介さずバイナリレベルで互換性を持って変換可能かどうかを判定します。

シグネチャまたは形状

bool IsBinaryCoercible(Oid srctype, Oid targettype)

入力と出力

  • **入力**:
  • srctype (Oid): 変換元のデータ型OID。
  • targettype (Oid): 変換先のデータ型OID。
  • **出力**:
  • bool: 変換可能であれば true、そうでなければ false を返します。

前提条件と不変条件

  • srctype と targettype は有効な型OIDである必要があります。
  • 内部的に IsBinaryCoercibleWithCast を呼び出し、その結果に依存します。

アルゴリズムまたは振る舞い

1. 内部関数 IsBinaryCoercibleWithCast を呼び出します。 2. その際、キャストOIDを格納するためのスタック変数 castoid を渡します。 3. IsBinaryCoercibleWithCast の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**: IsBinaryCoercibleWithCast

関連データ構造

  • Oid: PostgreSQLのシステムカタログにおけるオブジェクト識別子。

失敗モードと境界条件

  • srctype と targettype が同一である場合、通常はバイナリ互換とみなされます（IsBinaryCoercibleWithCast の実装に依存）。
  • 変換パスが存在しない場合や、バイナリ互換ではない変換（関数呼び出しが必要な場合など）は false を返します。
