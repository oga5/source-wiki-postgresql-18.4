SH_PREV

定義

  • 種別: static inline function
  • 定義箇所: src/include/lib/simplehash.h:374

目的

ハッシュテーブル内において、現在のバケットインデックスから論理的に一つ前のバケットインデックスを計算して返します。

シグネチャまたは形状

static inline uint32 SH_PREV(SH_TYPE * tb, uint32 curelem, uint32 startelem)

入力と出力

  • **入力**:
  • tb: ハッシュテーブル構造体へのポインタ。
  • curelem: 現在のバケットインデックス。
  • startelem: 探索開始位置（ループ終了判定用）。
  • **出力**:
  • uint32: 更新された一つ前のバケットインデックス。

前提条件と不変条件

  • tb->sizemask はハッシュテーブルのサイズ（2のべき乗）に基づいたマスク値であること。
  • curelem は startelem と一致してはならない（Assert による検証）。

アルゴリズムまたは振る舞い

1. curelem から 1 を減算し、tb->sizemask とのビット論理積をとることで、循環バッファ上の前方のインデックスを算出します。 2. 算出されたインデックスが startelem と等しい場合、アサーションエラーを発生させます。 3. 計算結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/lib/simplehash.h:733 において、要素のシフト処理中に呼び出されています。
  • **呼び出し先**: なし（標準的な算術演算のみ）。

関連データ構造

  • SH_TYPE: simplehash.h で生成されるハッシュテーブル構造体。sizemask メンバを保持します。

失敗モードと境界条件

  • curelem が startelem に到達した場合、Assert によりデバッグビルドでは停止します。
  • ハッシュテーブルのサイズが2のべき乗でない場合、sizemask を用いたインデックス計算が正しく機能しません。

保守担当者向けメモ

  • 本関数はマクロ展開によって生成されるコードの一部であり、SH_TYPE に依存します。
  • 循環的な探索を行う際のインデックス制御に使用されるため、変更時はハッシュテーブルのサイズ計算ロジックとの整合性に注意してください。
