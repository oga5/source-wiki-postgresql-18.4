SPLITCOST

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsquery_gist.c:151

目的

GiST インデックスのページ分割処理において、各エントリの分割コストと位置情報を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct
{
	OffsetNumber pos;
	int32		cost;
} SPLITCOST;

入力と出力

本構造体はデータ保持用であり、関数ではありません。pos はインデックス内のオフセット位置を、cost は分割に伴うペナルティやコストの計算値を格納します。

前提条件と不変条件

  • pos は OffsetNumber 型であり、インデックスページ内の有効なエントリ位置を指す必要があります。
  • cost は int32 型であり、比較関数（例: comparecost）によってソート可能な値である必要があります。

アルゴリズムまたは振る舞い

GiST インデックスの picksplit 処理において、エントリを左右のページに振り分ける際の判断基準として利用されます。具体的には、各エントリに対してコストを計算し、その結果に基づいて qsort 等で並べ替えることで、分割の最適化を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsquery_gist.c 内の gtsquery_picksplit 関数が、インデックスページ分割時に本構造体の配列を割り当て、コスト計算に使用します。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接的な呼び出し先はありません。ただし、contrib/hstore/hstore_gist.c および contrib/intarray/_int_gist.c においても、同様の目的で本構造体と類似のロジックが利用されています。

関連データ構造

  • OffsetNumber: インデックスページ内のエントリ位置を特定するために使用。
  • comparecost: SPLITCOST 配列をソートするために使用される比較関数。

失敗モードと境界条件

  • メモリ不足: palloc を使用して配列を確保する際、メモリが不足するとエラーが発生します。
  • オーバーフロー: cost の計算において int32 の範囲を超える値が算出された場合、予期せぬソート順序になる可能性があります。

保守担当者向けメモ

  • 本構造体は tsquery_gist.c だけでなく、他の拡張モジュール（hstore, intarray）でも同様の定義が使われています。変更を加える際は、これら関連モジュールへの影響を考慮してください。
  • cost の計算ロジックはインデックスの検索効率に直結するため、変更時にはハミング距離等の指標が適切に反映されているか確認が必要です。
