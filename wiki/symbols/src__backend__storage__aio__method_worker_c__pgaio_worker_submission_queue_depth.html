pgaio_worker_submission_queue_depth

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_worker.c:217

目的

共有メモリ上のI/O投入用リングバッファ（io_worker_submission_queue）に現在蓄積されている、未処理のI/Oリクエスト数を算出します。

シグネチャまたは形状

static uint32 pgaio_worker_submission_queue_depth(void)

入力と出力

  • **入力**: なし（グローバルな io_worker_submission_queue を参照）
  • **出力**: uint32 型のキュー内のリクエスト数

前提条件と不変条件

  • io_worker_submission_queue が初期化済みであること。
  • head および tail の値がリングバッファのサイズ範囲内であること。
  • Assert(head >= tail) により、計算後のヘッド位置がテール位置を下回らないことが保証されています（リングバッファのラップアラウンド考慮後）。

アルゴリズムまたは振る舞い

1. グローバルなキュー構造体から現在の head と tail を取得します。 2. tail > head の場合、リングバッファが一周していると判断し、head にキューの size を加算して線形的な位置関係に補正します。 3. 補正後の head から tail を減算し、現在のキューの深さを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/method_worker.c:489: nwakeups の算出に使用。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • PgAioWorkerSubmissionQueue: 本関数が参照する共有メモリ上のリングバッファ構造体。

失敗モードと境界条件

  • キューが空の場合、head と tail が等しいため 0 を返します。
  • キューが満杯の場合、リングバッファの仕様に従った差分が返されます。

保守担当者向けメモ

  • 本関数はロックを取得せずにキューのポインタを参照します。呼び出し側で適切な同期（AioWorkerSubmissionQueueLock 等）が確保されていることを前提としています。
