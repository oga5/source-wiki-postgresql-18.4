pgstat_fetch_stat_backend

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_backend.c:92

目的

指定されたプロセス番号（ProcNumber）に対応するバックエンド統計エントリを取得する。

シグネチャまたは形状

PgStat_Backend *pgstat_fetch_stat_backend(ProcNumber procNumber)

入力と出力

  • **入力**: procNumber (統計を取得する対象プロセスの番号)
  • **出力**: PgStat_Backend 構造体へのポインタ。該当するエントリが存在しない場合の挙動については、呼び出し先の pgstat_fetch_entry の仕様に依存する。

前提条件と不変条件

  • 統計サブシステムが初期化されている必要がある。
  • PGSTAT_KIND_BACKEND 種別の統計エントリが共有メモリ上に存在することを前提としている。

アルゴリズムまたは振る舞い

1. pgstat_fetch_entry 関数を呼び出し、PGSTAT_KIND_BACKEND 種別および指定された procNumber をキーとして統計エントリを検索する。 2. 取得したエントリへのポインタを呼び出し元へ返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/pgstat_backend.c:150
  • **呼び出し先**:
  • pgstat_fetch_entry (統計サブシステムの基盤関数)

関連データ構造

  • PgStat_Backend: バックエンド統計を保持する構造体。
  • ProcNumber: プロセスを識別するための数値型。

失敗モードと境界条件

  • 無効な procNumber が指定された場合の挙動は pgstat_fetch_entry の実装に依存する。
  • 共有メモリ上の統計領域が未初期化またはアクセス不能な場合、呼び出しは失敗する可能性がある。

保守担当者向けメモ

  • 本関数は pgstat_fetch_entry のラッパーであり、特定の統計種別（PGSTAT_KIND_BACKEND）に特化したインターフェースを提供している。
  • 統計情報の取得は共有メモリへのアクセスを伴うため、呼び出し頻度が高い場合はパフォーマンスへの影響を考慮すること。
