calc_hist

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_selfuncs.c:920

目的

要素数ヒストグラムの境界値に基づき、各整数値（0からnまで）が占める確率密度を計算し、配列の包含関係の選択率推定を補正するための確率分布配列を生成します。

シグネチャまたは形状

static float *calc_hist(const float4 *hist, int nhist, int n)

入力と出力

  • **入力**:
  • hist: float4 型のヒストグラム境界値配列。
  • nhist: ヒストグラムの要素数。
  • n: 計算対象とする最大整数値。
  • **出力**:
  • float *: サイズ n + 1 の確率分布配列。呼び出し元で pfree する必要があります。

前提条件と不変条件

  • hist は昇順にソートされている必要があります。
  • ヒストグラムの各区間は 1 / (nhist - 1) の確率寄与を持つと仮定されます。
  • hist 内の浮動小数点値は、端数がある場合、次の整数値に切り上げて扱われます。

アルゴリズムまたは振る舞い

1. palloc により n + 1 個の float 配列を確保します。 2. 各区間 [k, k+1] に対して、ヒストグラム境界値がどのように分布しているかを走査します。 3. ヒストグラムの境界値が k と一致する場合、その前後の区間長（prev_interval, next_interval）を用いて確率密度を補正します。 4. 境界値が k を跨ぐ場合、その区間内の寄与を frac（区間あたりの確率）に基づいて加算します。 5. 最終的に、各整数値 k に対する確率密度を格納した配列を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/array_selfuncs.c 内の mcelem_array_contained_selec（882行目付近）。
  • **呼び出し先**: palloc（メモリ確保）。

関連データ構造

  • float4: PostgreSQL の単精度浮動小数点型。
  • ArrayType: 本関数が間接的にサポートする配列型の内部表現。

失敗モードと境界条件

  • nhist が 1 以下の場合、frac の計算でゼロ除算が発生する可能性があります。
  • next_interval や prev_interval が 0 の場合、確率密度の加算処理をスキップまたは制限するロジックが含まれています。
  • n が極端に大きい場合、メモリ不足（palloc 失敗）が発生する可能性があります。

保守担当者向けメモ

  • 本関数は、<@（包含）演算子の選択率推定において、独立性の仮定を補正するために使用されます。
  • ヒストグラムの境界値が浮動小数点であるため、丸め誤差が累積しないよう、k との比較には注意が必要です。
