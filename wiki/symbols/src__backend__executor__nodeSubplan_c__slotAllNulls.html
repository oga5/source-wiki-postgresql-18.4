slotAllNulls

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSubplan.c:761

目的

指定された TupleTableSlot 内のすべての属性が NULL であるかどうかを判定する。

シグネチャまたは形状

static bool slotAllNulls(TupleTableSlot *slot)

入力と出力

  • **入力**: TupleTableSlot *slot（検査対象のタプルスロット）
  • **出力**: bool（すべての属性が NULL であれば true、一つでも非 NULL の属性があれば false を返す）

前提条件と不変条件

  • slot は有効な TupleTableSlot 構造体を指している必要がある。
  • slot->tts_tupleDescriptor が初期化されており、属性数（natts）が正しく設定されていること。

アルゴリズムまたは振る舞い

1. スロット内の属性数（ncols）を取得する。 2. 1番目の属性から最後の属性まで順次ループする。 3. slot_attisnull(slot, i) を呼び出し、各属性が NULL かどうかを確認する。 4. NULL ではない属性が見つかった時点で直ちに false を返す。 5. すべての属性の確認が完了し、一度も false が返されなかった場合に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeSubplan.c 内のロジック（行 175 付近）にて、サブプランの評価結果がすべて NULL かどうかの判定に使用されている。
  • **呼び出し先**: slot_attisnull マクロ（属性の NULL 判定用）。

関連データ構造

  • TupleTableSlot: PostgreSQL の実行エンジンでタプルを保持するために使用される標準的な構造体。

失敗モードと境界条件

  • 属性数が 0 の場合、ループは実行されず true を返す。
  • slot が NULL ポインタである場合の挙動は定義されていない（セグメンテーション違反の可能性がある）。

保守担当者向けメモ

  • 本関数は nodeSubplan.c 内の静的ヘルパー関数である。
  • SQL の三値論理における NULL 判定ロジックの一部として機能している。
