pg_log_v

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/util.c:175

目的

pg_upgrade におけるログ出力の基盤関数であり、指定されたログ種別に応じて、標準出力（TTY）および内部ログファイル（log_opts.internal）への整形されたメッセージ出力を行う。

シグネチャまたは形状

static void pg_log_v(eLogType type, const char *fmt, va_list ap)

入力と出力

  • **入力**:
  • type: 出力形式を決定する eLogType 列挙値。
  • fmt: メッセージのフォーマット文字列。
  • ap: 可変長引数リスト。
  • **出力**: なし（副作用として標準出力およびログファイルへ書き出し）。

前提条件と不変条件

  • fmt は NULL であってはならない。
  • fmt の末尾に改行コードを含めてはならない（関数内で付与されるため）。
  • log_opts グローバル変数が初期化されていること。

アルゴリズムまたは振る舞い

1. vsnprintf を使用してフォーマット済みメッセージを生成する。 2. log_opts.internal が有効な場合、ログ種別に応じてログファイルへ書き出す。 3. type に応じた switch 文で標準出力への表示を制御する：

  • PG_STATUS: TTY の場合は MESSAGE_WIDTH に基づいて左詰め・切り詰めを行い、\r を付与して同一行での更新を可能にする。
  • PG_REPORT_NONL: 改行なしで出力する。
  • PG_FATAL: メッセージ出力後に exit(1) で終了する。
  • その他: 通常のメッセージとして出力する。

    • 4. 最後に fflush(stdout) を実行し、出力を確定させる。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_log および pg_fatal（src/bin/pg_upgrade/util.c 内）。
  • **呼び出し先**: vsnprintf, fprintf, printf, fflush, exit, strlen 等の標準ライブラリ関数。

関連データ構造

  • eLogType: ログ種別を定義する列挙型。
  • LogOpts (log_opts): ログ出力設定（出力先、冗長モード、TTY判定など）を保持する構造体。

失敗モードと境界条件

  • log_opts.internal が NULL の場合、ログファイルへの書き出しはスキップされる。
  • PG_STATUS 出力時、メッセージが MESSAGE_WIDTH を超える場合は左側が切り詰められ、... が付与される。
  • PG_FATAL 発生時はプログラムが即座に終了する。

保守担当者向けメモ

  • 本関数はメッセージの末尾に改行を付与する設計であるため、呼び出し側で改行を含めないよう注意すること。
  • PG_STATUS の進捗表示ロジックは、TTY か否かによって挙動が大きく異なるため、変更時は log_opts.isatty の判定に留意すること。
