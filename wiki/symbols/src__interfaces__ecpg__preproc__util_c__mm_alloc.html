mm_alloc

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/util.c:84

目的

標準の malloc をラップし、メモリ確保失敗時に致命的エラーを報告してプロセスを終了させることで、呼び出し側でのエラーチェックを省略可能にする。

シグネチャまたは形状

void *mm_alloc(size_t size);

入力と出力

  • **入力**: size_t size（確保するバイト数）
  • **出力**: 確保されたメモリ領域へのポインタ（void *）

前提条件と不変条件

  • 確保に成功した場合、戻り値は NULL ではない。
  • 確保に失敗した場合、関数は戻らずに mmfatal を経由してプロセスを終了させる。

アルゴリズムまたは振る舞い

1. malloc(size) を呼び出し、メモリ確保を試みる。 2. 戻り値が NULL の場合、mmfatal(OUT_OF_MEMORY, "out of memory") を呼び出し、プログラムを異常終了させる。 3. 成功した場合は、確保したメモリ領域のポインタを呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し先**: malloc (標準ライブラリ), mmfatal (同モジュール内)
  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/descriptor.c
  • src/interfaces/ecpg/preproc/ecpg.c
  • src/interfaces/ecpg/preproc/type.c

    • (詳細はツールによる参照箇所を参照)

関連データ構造

  • 特になし（汎用的なメモリ確保関数）。

失敗モードと境界条件

  • size が 0 の場合、malloc の実装依存の挙動（NULL または有効なポインタ）に従う。
  • システムのメモリが枯渇した場合、mmfatal によりプログラムが終了する。

保守担当者向けメモ

  • 本関数は malloc の単純なラッパーであり、メモリ解放機能は持たない。
  • 一時的なメモリ管理が必要な場合は、本関数ではなく loc_alloc の使用を検討すること。
