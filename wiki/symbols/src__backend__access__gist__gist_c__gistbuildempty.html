gistbuildempty

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gist.c:138

目的

GiST インデックスの初期化時に、空のルートページを作成する。

シグネチャまたは形状

void gistbuildempty(Relation index)

入力と出力

  • **入力**: index (初期化対象の Relation 構造体)
  • **出力**: なし（副作用としてインデックスの初期化フォークにルートページが作成される）

前提条件と不変条件

  • 指定された index は、GiST インデックスとして適切にオープンされていること。
  • 本関数はインデックスの初期化フェーズで呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. ExtendBufferedRel を呼び出し、インデックスの初期化フォークに新しいページを割り当てる。 2. クリティカルセクションを開始する。 3. GISTInitBuffer を使用して、割り当てたページを F_LEAF フラグ付きの GiST ルートページとして初期化する。 4. ページをダーティとしてマークし、log_newpage_buffer により WAL ログを記録する。 5. クリティカルセクションを終了する。 6. UnlockReleaseBuffer を呼び出し、バッファのロックを解除して解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gist.c 内の IndexAmRoutine 構造体（ambuildempty フィールド）を通じて、インデックス作成プロセスから呼び出される。
  • **呼び出し先**: ExtendBufferedRel, GISTInitBuffer, MarkBufferDirty, log_newpage_buffer, UnlockReleaseBuffer

関連データ構造

  • Relation: インデックスの物理的・論理的情報を保持する構造体。
  • Buffer: バッファマネージャが管理するページへのハンドル。

失敗モードと境界条件

  • バッファの割り当てや WAL ログの記録に失敗した場合、PostgreSQL のエラーハンドリング機構によりトランザクションがアボートされる。

保守担当者向けメモ

  • 本関数は WAL ログを記録するため、クラッシュリカバリ時にルートページの存在が保証される。
  • GISTInitBuffer に渡される F_LEAF フラグは、新規作成されたルートがリーフページであることを示している。
