pg_unicode_decomposition

定義

  • 種別: struct
  • 定義箇所: src/include/common/unicode_norm_table.h:19

目的

Unicode 正規化処理において、特定のコードポイントに対する分解情報（結合クラスや分解後のコードポイント列への参照など）を保持するための静的データ構造です。

シグネチャまたは形状

typedef struct
{
    uint32      codepoint;        /* Unicode コードポイント */
    uint8       comb_class;       /* 文字の結合クラス */
    uint8       dec_size_flags;   /* 分解コードリストのサイズとフラグ */
    uint16      dec_index;        /* UnicodeDecomp_codepoints へのインデックス、またはインライン分解値 */
} pg_unicode_decomposition;

入力と出力

本構造体はデータ定義であり、関数ではありません。正規化処理の過程で、対象のコードポイントに対応する本構造体のインスタンスが検索され、そのフィールド値が分解処理の入力として利用されます。

前提条件と不変条件

  • 本構造体は src/common/unicode/generate-unicode_norm_table.pl によって自動生成される静的データの一部です。
  • dec_index フィールドは、dec_size_flags に DECOMP_INLINE フラグが設定されているかどうかによって、配列インデックスとして解釈されるか、直接的な分解値として解釈されるかが決定されます。

アルゴリズムまたは振る舞い

1. 正規化処理関数が UnicodeDecompMain テーブルから対象コードポイントに対応する本構造体を取得します。 2. dec_size_flags を参照し、分解の有無、および分解データがインラインかインデックス参照かを判定します。 3. インデックス参照の場合は UnicodeDecomp_codepoints 配列から分解後のコードポイント列を読み取ります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/unicode_norm.c 内の各関数（get_code_entry 等）が本構造体を参照しています。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接関数を呼び出すことはありません。

関連データ構造

  • UnicodeDecompMain: 本構造体のインスタンスを格納するメインの静的配列。
  • UnicodeDecomp_codepoints: 分解後のコードポイント列を保持する配列。
  • DECOMP_INLINE / DECOMP_COMPAT: dec_size_flags で使用されるフラグ定数。

失敗モードと境界条件

  • 本構造体は静的データであるため、実行時の失敗モードはありません。
  • 境界条件として、dec_index が UnicodeDecomp_codepoints の有効範囲外を指さないことが、生成スクリプトによって保証されている必要があります。
