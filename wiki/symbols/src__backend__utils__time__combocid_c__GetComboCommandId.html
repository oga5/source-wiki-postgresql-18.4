GetComboCommandId

定義

  • 種別: function
  • 定義箇所: src/backend/utils/time/combocid.c:203

目的

同一トランザクション内で挿入と削除が連続して行われたタプルの cmin と cmax のペアを、単一の CommandId（Combo CID）にマッピングして管理する。

シグネチャまたは形状

static CommandId GetComboCommandId(CommandId cmin, CommandId cmax)

入力と出力

  • **入力**:
  • cmin: 挿入時のコマンドID
  • cmax: 削除時のコマンドID
  • **出力**:
  • CommandId: マッピングされたCombo CID（comboCids 配列のインデックス）

前提条件と不変条件

  • ComboCidKeyData 構造体にパディングが存在しないことを前提としてハッシュ操作を行う。
  • comboHash が存在する場合、comboCids 配列も必ず存在している必要がある。
  • メモリ確保は TopTransactionContext に対して行われる。

アルゴリズムまたは振る舞い

1. **初期化**: comboHash が未作成の場合、TopTransactionContext 上にハッシュテーブルと comboCids 配列を初期化する。 2. **容量確保**: usedComboCids が sizeComboCids に達した場合、comboCids 配列を倍のサイズに拡張する。 3. **検索**: 指定された cmin/cmax をキーとして comboHash を検索する。

  • 存在する場合: 既存の combocid を返す。
  • 存在しない場合: 新規エントリを作成し、comboCids 配列の末尾に cmin/cmax を格納して、そのインデックスを combocid として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/time/combocid.c 内の HeapTupleHeaderAdjustCmax 等から呼び出される。
  • **呼び出し先**:
  • MemoryContextAlloc / repalloc (メモリ管理)
  • hash_create / hash_search (ハッシュテーブル操作)

関連データ構造

  • ComboCidKeyData: cmin と cmax を保持するキー構造体。
  • ComboCidEntryData: ハッシュテーブルのエントリ構造体。
  • comboHash: cmin/cmax ペアを検索するためのハッシュテーブル。
  • comboCids: Combo CIDをインデックスとして実値を保持する配列。

失敗モードと境界条件

  • repalloc によるメモリ確保失敗時はシステムエラーとなる。
  • ComboCidKeyData に構造体パディングが含まれる場合、ハッシュ検索が正しく機能しない可能性がある。

保守担当者向けメモ

  • 本関数は static であり、モジュール外からは直接呼び出されない。
  • トランザクション終了時のメモリ解放は AtEOXact_ComboCid が担うため、本関数内での明示的な解放は不要である。
