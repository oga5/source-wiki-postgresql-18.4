BlockIdData

定義

  • 種別: struct
  • 定義箇所: src/include/storage/block.h:53

目的

ディスク上のデータ構造において、ブロック番号を保持するための識別子を提供します。BlockNumber（32ビット）を2つの16ビット値に分割することで、構造体のアライメントによるパディングを抑制し、メモリおよびディスク上の占有サイズを最小化します。

シグネチャまたは形状

typedef struct BlockIdData
{
    uint16      bi_hi;
    uint16      bi_lo;
} BlockIdData;

入力と出力

  • **入力**: BlockNumber を BlockIdSet マクロ等を通じて本構造体に格納します。
  • **出力**: BlockIdGetBlockNumber を通じて、格納された2つの16ビット値から元の BlockNumber を復元します。

前提条件と不変条件

  • bi_hi に上位16ビット、bi_lo に下位16ビットが格納されることが期待されます。
  • 構造体サイズは sizeof(uint16) * 2 であり、パディングを含まないことが前提です。

アルゴリズムまたは振る舞い

  • 物理的なI/Oやディスク上の構造体（ItemPointerData など）にブロック番号を保存する際、直接 BlockNumber を使用する代わりに本構造体を経由します。これにより、データ構造のパディングを最適化し、タプルヘッダやインデックスエントリのサイズを削減します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/gin/ginbtree.c
  • src/backend/access/gin/ginxlog.c
  • src/backend/access/rmgrdesc/gindesc.c
  • src/backend/utils/adt/tid.c
  • src/include/access/ginblock.h
  • src/include/access/ginxlog.h
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、特定の関数を呼び出すことはありません。値の変換には src/include/storage/block.h 内のインライン関数が使用されます。

関連データ構造

  • BlockNumber: 本構造体が保持する元の32ビットブロック番号。
  • BlockId: BlockIdData へのポインタ型。
  • ItemPointerData: 本構造体を内部に保持し、ディスク上のタプル識別子として使用されます。

失敗モードと境界条件

  • 32ビットの BlockNumber を超える値を格納しようとした場合、ビットの切り捨てやオーバーフローが発生する可能性があります。
  • InvalidBlockNumber（0xFFFFFFFF）を格納した場合、復元時にも同様の値が得られる必要があります。

保守担当者向けメモ

  • 本構造体は SHORTALIGN を可能にするための最適化手段です。構造体のメンバ順序や型を変更する際は、ディスク上のレイアウトに影響を与えるため注意してください。
  • ginblock.h 等で言及されている通り、パディングによるスペースの浪費を避けるための重要な設計要素です。
