float8_dist

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_float8.c:102

目的

2つの float8 値の間の絶対距離を計算し、PostgreSQLの関数インターフェースとして提供する。

シグネチャまたは形状

Datum float8_dist(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS を介して2つの float8 値を受け取る。
  • 出力: Datum 型として、入力値の差の絶対値（float8）を返す。

前提条件と不変条件

  • 入力値は PG_GETARG_FLOAT8 マクロにより取得される。
  • 演算結果がオーバーフローし、かつ元の入力値が無限大（inf）でない場合、float_overflow_error() を呼び出してエラーを発生させる。

アルゴリズムまたは振る舞い

1. 第1引数 a と第2引数 b を取得する。 2. r = a - b を計算する。 3. r が無限大であり、かつ a と b がいずれも無限大でない場合、数値オーバーフローと判定してエラーを投げる。 4. fabs(r) を計算し、その結果を PG_RETURN_FLOAT8 で返す。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLの関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • 呼び出し先: PG_GETARG_FLOAT8, isinf, float_overflow_error, fabs, PG_RETURN_FLOAT8。
  • 注: 特定のSQLクエリや内部関数からの直接的な呼び出し関係については、提供されたソース excerpt に情報がありません。

関連データ構造

  • float8: PostgreSQLの倍精度浮動小数点数型。

失敗モードと境界条件

  • 数値オーバーフロー: 差分計算時に inf が発生し、かつ入力値が有限である場合にエラーとなる。
  • 入力値が inf の場合: isinf のチェックにより、オーバーフローエラーを回避する。

保守担当者向けメモ

  • 本関数は btree_gist モジュールにおける float8 型の距離計算ロジックを担う。
  • float_overflow_error() は utils/float.h で定義された標準的なエラーハンドリング関数である。
