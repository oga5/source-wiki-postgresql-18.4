gbt_enum_penalty

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_enum.c:155

目的

GiST インデックスの挿入時において、既存のインデックスエントリに新しい値を挿入する際の「ペナルティ（コスト）」を計算します。

シグネチャまたは形状

Datum gbt_enum_penalty(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): 既存のインデックスエントリ（oidKEY 構造体へのポインタ）。
  • PG_GETARG_POINTER(1): 新しく挿入する値（oidKEY 構造体へのポインタ）。
  • PG_GETARG_POINTER(2): 計算結果を格納する float 型のポインタ。
  • **出力**:
  • PG_RETURN_POINTER(result): 計算されたペナルティ値（float）を返します。

前提条件と不変条件

  • 入力される key は oidKEY 構造体としてキャスト可能である必要があります。
  • ENUM 型は内部的に OID として扱われるため、OID の範囲比較ロジックが適用されます。

アルゴリズムまたは振る舞い

1. 引数から既存エントリ（origentry）と新規エントリ（newentry）の範囲情報を取得します。 2. penalty_num 関数を呼び出し、既存の範囲を新しい値を含めるために拡張する際のコストを計算します。 3. 計算結果を result ポインタに書き込み、そのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: GiST インデックスの挿入処理（PostgreSQL コアの GiST インデックス管理機構から呼び出されます）。
  • **呼び出し先**: penalty_num（btree_utils_num.c で定義された汎用ユーティリティ関数）。

関連データ構造

  • oidKEY: ENUM 型の範囲（lower, upper）を保持する構造体。
  • GISTENTRY: GiST インデックスの各ノードエントリを保持する構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、セグメンテーション違反が発生する可能性があります。
  • penalty_num 内部での計算において、範囲のオーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_enum.c におけるアダプター層であり、実際の計算ロジックは btree_utils_num.c に依存しています。
  • ENUM 型の順序付けは OID に依存しているため、OID 型のインデックス実装と挙動が共通化されています。
