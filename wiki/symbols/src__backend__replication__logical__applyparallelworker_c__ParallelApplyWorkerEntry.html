ParallelApplyWorkerEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/applyparallelworker.c:215

目的

論理レプリケーションの並列適用において、トランザクションID（xid）とそれに対応する並列適用ワーカー情報（ParallelApplyWorkerInfo）をハッシュテーブル内で関連付けるためのエントリ構造体です。

シグネチャまたは形状

typedef struct ParallelApplyWorkerEntry
{
	TransactionId xid;			/* Hash key -- must be first */
	ParallelApplyWorkerInfo *winfo;
} ParallelApplyWorkerEntry;

入力と出力

  • **入力**: 構造体生成時に TransactionId および ParallelApplyWorkerInfo へのポインタが設定されます。
  • **出力**: ハッシュテーブルの検索結果として、特定のトランザクションに関連付けられたワーカー情報を提供します。

前提条件と不変条件

  • xid フィールドはハッシュキーとして機能するため、構造体の先頭に配置される必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、並列適用ワーカーを管理するハッシュテーブルのエントリとして使用されます。
  • xid をキーとして検索を行うことで、特定のトランザクションを担当するワーカーの制御情報（winfo）を高速に取得します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/applyparallelworker.c 内のハッシュテーブル操作関数（entry 変数として参照）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ParallelApplyWorkerInfo: ワーカーの状態（DSMセグメント、キュー等）を保持する構造体。
  • TransactionId: PostgreSQLのトランザクション識別子。

失敗モードと境界条件

  • ハッシュテーブルのサイズ設定（ctl.entrysize）において、本構造体のサイズ（sizeof(ParallelApplyWorkerEntry)）と不整合が生じるとメモリ破壊の原因となります。

保守担当者向けメモ

  • 本構造体はハッシュテーブルのキーとして使用されるため、メンバの順序変更や追加を行う際は、ハッシュテーブルの定義および検索ロジックとの整合性に注意してください。
