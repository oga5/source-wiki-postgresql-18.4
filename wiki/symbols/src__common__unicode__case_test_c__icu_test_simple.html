icu_test_simple

定義

  • 種別: function
  • 定義箇所: src/common/unicode/case_test.c:79

目的

単一の Unicode コードポイントに対し、PostgreSQL 内部の単純変換関数と ICU ライブラリの変換結果を比較し、整合性を検証する。

シグネチャまたは形状

static void icu_test_simple(pg_wchar code)

入力と出力

  • 入力: pg_wchar code（検証対象の Unicode コードポイント）
  • 出力: なし（戻り値なし）
  • 副作用: 比較結果が一致しない場合、標準出力にエラー詳細を表示し、exit(1) を呼び出してプログラムを終了する。

前提条件と不変条件

  • USE_ICU が定義されている環境で実行されることを想定している。
  • PostgreSQL の変換関数（unicode_lowercase_simple 等）と ICU の関数（u_tolower 等）が、同一のコードポイントに対して同一の変換結果を返すことが期待される。

アルゴリズムまたは振る舞い

1. 引数 code に対して、PostgreSQL の各変換関数（lower, title, upper, fold）を呼び出し、結果を保持する。 2. 同一の code に対して、ICU の対応する関数（u_tolower, u_totitle, u_toupper, u_foldCase）を呼び出し、結果を保持する。 3. 両者の結果を比較する。 4. 一つでも不一致がある場合、両方の変換結果を標準出力に書き出し、プロセスを異常終了させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/unicode/case_test.c 内の test_icu 関数（全コードポイント走査時）。
  • 呼び出し先:
  • PostgreSQL 内部関数: unicode_lowercase_simple, unicode_titlecase_simple, unicode_uppercase_simple, unicode_casefold_simple
  • ICU ライブラリ関数: u_tolower, u_totitle, u_toupper, u_foldCase

関連データ構造

  • pg_wchar: Unicode コードポイントを保持する型。

失敗モードと境界条件

  • ICU のバージョンと PostgreSQL が内蔵する Unicode データのバージョン差異により、特定のコードポイントで不一致が発生する可能性がある。
  • 不一致発生時は即座に exit(1) で終了するため、テストスイート全体が停止する。

保守担当者向けメモ

  • 本関数は単一文字の変換を検証する。文字列全体の変換テストについては icu_test_full を参照のこと。
  • ICU が存在しない環境での動作については、ソースコードの条件付きコンパイル設定を確認すること。
