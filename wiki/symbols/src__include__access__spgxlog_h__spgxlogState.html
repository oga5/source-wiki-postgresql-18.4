spgxlogState

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgxlog.h:36

目的

SP-GiST インデックスのリカバリ（Redo）処理において、インデックスの最小限の状態を保持し、WAL レコードの再実行をサポートするために使用されます。

シグネチャまたは形状

typedef struct spgxlogState
{
	TransactionId redirectXid;
	bool		isBuild;
} spgxlogState;

入力と出力

  • **入力**: WAL レコードに含まれる状態情報。
  • **出力**: spg_redo 関数等のリカバリ処理において、インデックスの整合性を復元するためのパラメータとして利用されます。

前提条件と不変条件

  • リカバリ時に必要な情報が有効である必要があります。
  • spgxlogState は、WAL レコード内の情報を保持するコンテナとして機能します。

アルゴリズムまたは振る舞い

1. インデックス操作時に、現在の状態を spgxlogState に格納し、WAL レコードの一部として書き出します。 2. リカバリ時、spg_redo 関数が WAL レコードからこの構造体を読み取ります。 3. 読み取られた redirectXid や isBuild フラグに基づき、対象ページの状態を適切に更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgxlog.c 内の fillFakeState 関数が、この構造体を使用して SpGistState を初期化します。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • SpGistState: fillFakeState を通じて本構造体の情報が反映されるインデックスの内部状態管理構造体。
  • 各種 WAL レコード構造体（spgxlogAddLeaf, spgxlogPickSplit 等）: 本構造体をメンバとして保持、あるいはリカバリ処理の文脈で併用されます。

失敗モードと境界条件

  • 構造体内のデータが破損している場合、リカバリ後のインデックス整合性が損なわれます。
  • 構造体直後に続く可変長データ（タプル等）のポインタ演算を行う際、アライメントが考慮されていないとメモリ不正アクセスの原因となります。

保守担当者向けメモ

  • 本構造体は spgxlog.h 内の複数の WAL レコード構造体で stateSrc として定義されています。
  • リカバリロジックを追う際は、spg_redo がどのように redirectXid を使用してページ上のアイテムを更新しているかを確認してください。
  • 可変長データを含む構造体であるため、サイズ計算には必ず SizeOf... マクロを使用してください。
