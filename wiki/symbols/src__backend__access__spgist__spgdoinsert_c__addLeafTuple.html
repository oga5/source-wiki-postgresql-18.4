addLeafTuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgdoinsert.c:202

目的

SP-GiSTインデックスのリーフページに対して、新しいリーフタプルを挿入または既存のリーフチェーンを更新します。

シグネチャまたは形状

static void addLeafTuple(Relation index, SpGistState *state, SpGistLeafTuple leafTuple,
                         SPPageDesc *current, SPPageDesc *parent, bool isNulls, bool isNew)

入力と出力

  • **入力**:
  • index: 対象のインデックスリレーション。
  • state: SP-GiSTの操作状態。
  • leafTuple: 挿入するリーフタプル。
  • current: 現在のリーフページ情報。
  • parent: 親ページ情報（ダウンリンク更新が必要な場合に使用）。
  • isNulls: NULL値を含むかどうかのフラグ。
  • isNew: 新規ページかどうかを示すフラグ。
  • **出力**: なし（副作用としてページ内容およびWALを更新）。

前提条件と不変条件

  • current->buffer は書き込みロックされている必要があります。
  • 構造変更は START_CRIT_SECTION() で保護され、エラー発生時にはインデックスの整合性が維持されます。

アルゴリズムまたは振る舞い

1. **初期化**: WAL記録用の spgxlogAddLeaf 構造体を初期化します。 2. **挿入処理**:

  • **チェーン外**: current->offnum が無効またはルートページの場合、ページに新しいタプルを追加し、必要に応じて saveNodeLink を呼び出して親のダウンリンクを更新します。
  • **チェーン内**:
  • SPGIST_LIVE タプルの場合: 新しいタプルをチェーンの2番目に追加し、リンクを更新します。
  • SPGIST_DEAD タプルの場合: 既存のDEADタプルを削除し、その位置に新しいタプルを配置します。

    • 3. **WAL記録**: RelationNeedsWAL が真かつビルド中でない場合、XLogInsert を使用して変更をログに記録し、ページLSNを更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgdoinsert.c 内の挿入ロジックから呼び出されます。
  • **呼び出し先**: SpGistPageAddNewItem, saveNodeLink, PageGetItem, PageIndexTupleDelete, PageAddItem, MarkBufferDirty, XLogInsert 等。

関連データ構造

  • SpGistLeafTuple: 挿入対象のタプル構造。
  • SPPageDesc: ページおよびタプルの位置情報を保持する記述子。
  • spgxlogAddLeaf: WAL記録用の構造体。

失敗モードと境界条件

  • head->tupstate が SPGIST_LIVE でも SPGIST_DEAD でもない場合、elog(ERROR) を発生させます。
  • PageAddItem が期待したオフセット番号を返さない場合、elog(ERROR) を発生させます。

保守担当者向けメモ

  • 本関数はクリティカルセクション内で実行されるため、メモリ割り当てや複雑なロジックの追加には注意が必要です。
  • WALリプレイ時の整合性を保つため、xlrec の各フィールド（offnumLeaf, offnumHeadLeaf, offnumParent）の更新ロジックは厳密に維持してください。
