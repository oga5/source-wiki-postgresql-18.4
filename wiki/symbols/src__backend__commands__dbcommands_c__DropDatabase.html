DropDatabase

定義

  • 種別: function
  • 定義箇所: src/backend/commands/dbcommands.c:2341

目的

DROP DATABASE SQLコマンドの解析結果を受け取り、データベース削除処理の実行を仲介します。

シグネチャまたは形状

void DropDatabase(ParseState *pstate, DropdbStmt *stmt)

入力と出力

  • **入力**:
  • pstate: パーサーの状態情報。
  • stmt: DropdbStmt 構造体（削除対象のデータベース名、missing_ok フラグ、およびオプションリストを含む）。
  • **出力**: なし（成功時はデータベースを削除し、失敗時はエラーを報告）。

前提条件と不変条件

  • stmt->options に含まれるオプションは force のみをサポートします。それ以外のオプションが指定された場合はエラーとなります。

アルゴリズムまたは振る舞い

1. stmt->options を走査し、force オプションの有無を確認します。 2. 未知のオプションが指定された場合は ereport(ERROR, ...) を呼び出し、構文エラーとして処理を中断します。 3. 抽出したオプションと stmt 内の情報を引数として、内部関数 dropdb() を呼び出し、実際の削除処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（SQLコマンド実行のディスパッチャ）。
  • **呼び出し先**: dropdb()（src/backend/commands/dbcommands.c 内の削除ロジック本体）。

関連データ構造

  • DropdbStmt: 削除コマンドの構文解析結果を保持する構造体。
  • ParseState: パーサーのコンテキスト情報。

失敗モードと境界条件

  • **構文エラー**: force 以外のオプションが指定された場合、ERRCODE_SYNTAX_ERROR でエラー終了します。
  • **削除失敗**: 内部の dropdb() が失敗した場合、そのエラーがそのまま伝播します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の物理削除やカタログ操作は dropdb() に依存しています。
  • データベース削除に伴う排他制御やエラー時のクリーンアップ（PG_ENSURE_ERROR_CLEANUP）は、呼び出し先の dropdb() 内で管理されています。
