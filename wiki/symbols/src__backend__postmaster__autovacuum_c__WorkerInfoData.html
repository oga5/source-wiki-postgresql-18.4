WorkerInfoData

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/autovacuum.c:231

目的

PostgreSQLの自動バキューム（Autovacuum）システムにおいて、個々のワーカープロセスの状態を追跡および管理するためのデータ構造です。

シグネチャまたは形状

typedef struct WorkerInfoData
{
    dlist_node    wi_links;        /* ワーカーリスト管理用ノード */
    Oid           wi_dboid;        /* 対象データベースのOID */
    Oid           wi_tableoid;     /* 対象テーブルのOID */
    PGPROC       *wi_proc;         /* 対応するプロセス構造体へのポインタ */
    TimestampTz   wi_launchtime;   /* 起動時刻 */
    pg_atomic_flag wi_dobalance;   /* バキュームコストバランス調整フラグ */
    bool          wi_sharedrel;    /* 共有リレーションか否か */
} WorkerInfoData;

入力と出力

  • **入力**: ワーカー起動時に各フィールドが初期化されます。
  • **出力**: 共有メモリ上の状態として保持され、ランチャープロセスがワーカーの稼働状況やコスト制限を計算するために参照します。

前提条件と不変条件

  • dlist_node を介してリスト管理されるため、メモリ配置の整合性が求められます。
  • wi_proc は有効な PGPROC を指している必要があります。

アルゴリズムまたは振る舞い

  • ワーカーの起動・終了時に dlist 操作を通じてリストへの追加・削除が行われます。
  • wi_dobalance はアトミック操作により、複数のワーカー間でのコスト制限の動的な再計算を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/autovacuum.c 内のワーカー管理ロジック（dlist_container を用いたリスト走査など）。
  • 共有メモリ初期化処理（sizeof(WorkerInfoData) を使用）。
  • **呼び出し先**:
  • 特定の関数呼び出し関係はソース抜粋からは不明です。

関連データ構造

  • WorkerInfo: WorkerInfoData へのポインタ型（typedef struct WorkerInfoData *WorkerInfo;）。
  • AutoVacuumShmemStruct: 本構造体を保持する共有メモリ上の制御構造体。
  • dlist_node: リスト管理に使用されるリンク構造。

失敗モードと境界条件

  • 共有メモリ上の配列として確保されるため、最大ワーカー数を超えた割り当てはできません。
  • 参照先 wi_proc が無効化された後のアクセスは未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は autovacuum.c 内の共有メモリ管理と密接に関連しています。
  • dlist を使用しているため、リスト操作時のロック（AutovacuumLock 等）の取得状況に注意してください。
