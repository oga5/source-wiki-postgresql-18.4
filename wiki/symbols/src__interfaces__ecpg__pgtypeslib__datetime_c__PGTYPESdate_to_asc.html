PGTYPESdate_to_asc

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/datetime.c:100

目的

内部的な date 型の値を、人間が読み取り可能なASCII文字列形式に変換します。

シグネチャまたは形状

char *PGTYPESdate_to_asc(date dDate);

入力と出力

  • **入力**: date dDate（2000年1月1日を基準としたオフセット値）。
  • **出力**: 変換された日付文字列を保持する動的に確保されたメモリ領域（char *）。呼び出し元で解放する必要があります。

前提条件と不変条件

  • 入力された date 値は、date2j(2000, 1, 1) を基準とした有効なオフセットである必要があります。
  • 内部的に DateStyle は 1（ISO形式）、EuroDates は false に固定されています。

アルゴリズムまたは振る舞い

1. date2j(2000, 1, 1) を加算してユリウス日へ変換し、さらに j2date を用いて年・月・日の各要素を tm 構造体に分解します。 2. EncodeDateOnly 関数を呼び出し、固定されたスタイル設定（ISO形式、非欧州日付形式）で buf に文字列を書き込みます。 3. pgtypes_strdup を使用して、バッファの内容をヒープメモリに複製して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/ecpglib/execute.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-dt_test.c
  • **呼び出し先**:
  • j2date
  • date2j
  • EncodeDateOnly
  • pgtypes_strdup

関連データ構造

  • date: ECPG内で使用される日付型（内部的には整数）。
  • struct tm: 日付要素を保持する標準構造体。

失敗モードと境界条件

  • pgtypes_strdup がメモリ確保に失敗した場合、NULLが返される可能性があります。
  • MAXDATELEN を超えるような異常な日付値が入力された場合の挙動は、EncodeDateOnly の実装に依存します。

保守担当者向けメモ

  • 本関数は固定されたフォーマット（ISO）で出力するため、ロケールやユーザー設定に応じた柔軟なフォーマットが必要な場合は PGTYPESdate_fmt_asc の使用を検討してください。
  • 戻り値は pgtypes_strdup で確保されているため、メモリリークを防ぐために呼び出し元で必ず解放してください。
