have_unsafe_outer_join_ref

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/joinpath.c:389

目的

結合パスの生成において、外部結合（Outer Join）の制約を侵害するようなパラメータ化されたパスが生成されていないかを判定します。

シグネチャまたは形状

static inline bool have_unsafe_outer_join_ref(PlannerInfo *root, Relids outerrelids, Relids inner_paramrels)

入力と出力

  • **入力**:
  • root: クエリのプランナ情報。
  • outerrelids: 結合の外部リレーションセット。
  • inner_paramrels: 内部リレーションが依存するパラメータリレーションセット。
  • **出力**:
  • bool: 外部結合の制約に違反する参照が含まれている場合は true、そうでない場合は false を返します。

前提条件と不変条件

  • inner_paramrels と outerrelids の差分集合（unsatisfied）が、クエリ内の外部結合リレーション（root->outer_join_rels）と重なる場合にのみ、詳細なチェックが行われます。

アルゴリズムまたは振る舞い

1. inner_paramrels を outerrelids との交差集合（satisfied）と、差分集合（unsatisfied）に分割します。 2. unsatisfied が root->outer_join_rels と重なる場合、root->join_info_list を走査します。 3. 各 SpecialJoinInfo（sjinfo）について、unsatisfied に含まれる外部結合リレーションであるかを確認します。 4. 以下のいずれかに該当する場合、安全ではないと判断し true を返します。

  • satisfied が sjinfo->min_righthand と重なる場合。
  • sjinfo が JOIN_FULL であり、かつ satisfied が sjinfo->min_lefthand と重なる場合。

    • 5. 最後に、使用したビットマップセットを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/joinpath.c 内の結合パス生成ロジック（例: try_nestloop_path 等の周辺処理）。
  • **呼び出し先**: bms_difference, bms_intersect, bms_overlap, bms_is_member, bms_free などのビットマップセット操作関数。

関連データ構造

  • PlannerInfo: クエリ全体のプランナ情報。
  • SpecialJoinInfo: 外部結合やセミ結合などの特殊な結合に関する制約情報。
  • Relids: リレーションIDの集合（ビットマップ）。

失敗モードと境界条件

  • inner_paramrels が空の場合、unsatisfied も空となり、常に false を返します。
  • root->join_info_list が空の場合、ループは実行されず false を返します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、結合パス生成時のコスト計算やパス検証のホットパスで使用されます。
  • 外部結合の制約違反判定ロジックを変更する場合は、SpecialJoinInfo の min_righthand および min_lefthand の定義と整合性を取る必要があります。
