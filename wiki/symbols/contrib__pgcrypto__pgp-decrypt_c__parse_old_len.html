parse_old_len

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-decrypt.c:94

目的

OpenPGP パケットヘッダー内の「古い形式（old format）」の長さ情報を読み取り、パケット長を算出する。

シグネチャまたは形状

static int parse_old_len(PullFilter *src, int *len_p, int lentype);

入力と出力

  • **入力**:
  • src: データを読み出すための PullFilter。
  • lentype: 長さのバイト数を示すフラグ（0, 1, 2）。
  • **出力**:
  • len_p: 算出された長さが格納されるポインタ。
  • 戻り値: 成功時は PKT_NORMAL、不正な長さの場合は PXE_PGP_CORRUPT_DATA。

前提条件と不変条件

  • src は有効な PullFilter であること。
  • lentype は 0, 1, 2 のいずれかであること。
  • 算出された長さは MAX_CHUNK 以下である必要がある。

アルゴリズムまたは振る舞い

1. lentype に応じて src からバイトを読み取る。

  • lentype == 0: 1 バイトを読み取り、長さを決定。
  • lentype == 1: 2 バイトを読み取り、ビッグエンディアンとして結合。
  • lentype == 2: 4 バイトを読み取り、ビッグエンディアンとして結合。

    • 2. 算出された len が 0 未満または MAX_CHUNK を超える場合、エラーを返す。 3. 正常な場合、len_p に値を書き込み PKT_NORMAL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-decrypt.c 内のパケット解析処理から呼び出される（ソースコード上の直接的な言及あり）。
  • **呼び出し先**: GETBYTE マクロ（src からのバイト読み取り）。

関連データ構造

  • PullFilter: データのストリーム読み取りに使用される構造体。

失敗モードと境界条件

  • GETBYTE が失敗した場合（ストリーム終端など）。
  • len が MAX_CHUNK を超える場合、PXE_PGP_CORRUPT_DATA を返す。

保守担当者向けメモ

  • 本関数は RFC 4880 の古い形式のパケット長エンコーディングを処理する。
  • lentype の値が 0, 1, 2 以外の場合の挙動は定義されていない。
