ResetProcSignalBarrierBits

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/procsignal.c:636

目的

現在のプロセスに対して、指定されたバリア関連のシグナルビットをセットし、割り込み処理を保留状態に設定します。

シグネチャまたは形状

static void ResetProcSignalBarrierBits(uint32 flags)

入力と出力

  • **入力**: flags (uint32) - セットするバリアビットのマスク。
  • **出力**: なし。

前提条件と不変条件

  • MyProcSignalSlot が初期化されており、有効な共有メモリ上のスロットを指していること。
  • 本関数は、プロセスが自身のシグナル状態を更新する際に呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. MyProcSignalSlot->pss_barrierCheckMask に対して、引数 flags をアトミックな OR 操作で適用します。 2. グローバル変数 ProcSignalBarrierPending を true に設定し、バリア処理が保留中であることを示します。 3. グローバル変数 InterruptPending を true に設定し、後続の CHECK_FOR_INTERRUPTS() 等で割り込み処理が実行されるようにします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/procsignal.c:595
  • src/backend/storage/ipc/procsignal.c:607
  • **呼び出し先**:
  • pg_atomic_fetch_or_u32 (アトミック操作)

関連データ構造

  • ProcSignalSlot: プロセスごとのシグナル状態を保持する構造体。pss_barrierCheckMask を含みます。

失敗モードと境界条件

  • MyProcSignalSlot が NULL の場合、セグメンテーション違反が発生します。
  • アトミック操作自体は失敗しませんが、flags の値が不正な場合、意図しないビットがセットされる可能性があります。

保守担当者向けメモ

  • 本関数は static であり、procsignal.c 内でのみ使用されます。
  • InterruptPending をセットすることで、PostgreSQL のメインループや CHECK_FOR_INTERRUPTS() が呼び出される箇所で、バリア処理が確実にトリガーされる仕組みになっています。
