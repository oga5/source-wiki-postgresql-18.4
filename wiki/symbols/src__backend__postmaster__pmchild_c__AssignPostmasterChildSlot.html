AssignPostmasterChildSlot

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/pmchild.c:161

目的

新規子プロセスを追跡するために、指定された BackendType に対応するプールから空きスロット（PMChild 構造体）を確保し、アクティブリストへ登録する。

シグネチャまたは形状

PMChild *AssignPostmasterChildSlot(BackendType btype)

入力と出力

  • **入力**: btype (BackendType) - 割り当てるプロセスのタイプ。
  • **出力**: 確保された PMChild 構造体へのポインタ。空きがない場合は NULL を返す。

前提条件と不変条件

  • 指定された btype のプールサイズが 0 であってはならない（0 の場合はエラー）。
  • 確保された PMChild の child_slot は、対応するプールの有効範囲内（first_slotno 以上かつ first_slotno + size 未満）でなければならない。

アルゴリズムまたは振る舞い

1. 指定された btype の freelist が空か確認し、空であれば NULL を返す。 2. freelist からノードを取り出し、PMChild 構造体を取得する。 3. 取得した PMChild の各フィールド（pid, bkend_type, rw, bgworker_notify）を初期化する。 4. 整合性チェックを行い、スロット番号が妥当であることを確認する。 5. ActiveChildList に当該 PMChild を追加する。 6. MarkPostmasterChildSlotAssigned を呼び出し、共有メモリ上の状態を更新する。 7. 成功時に確保した PMChild 構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/postmaster.c 内の各プロセス起動処理（B_BACKEND, B_LOGGER, B_BG_WORKER 等の起動時）。
  • **呼び出し先**: MarkPostmasterChildSlotAssigned（共有メモリ状態更新）、dlist 操作関数群。

関連データ構造

  • PMChild: 子プロセスのメタデータを保持する構造体。
  • pmchild_pools: 各 BackendType ごとのスロットプールを保持する配列。
  • ActiveChildList: 稼働中の全子プロセスを管理する双方向リスト。

失敗モードと境界条件

  • **エラー**: pmchild_pools[btype].size == 0 の場合、割り当て不能として elog(ERROR) を発生させる。
  • **エラー**: 確保したスロットの child_slot が範囲外の場合、freelist の破損とみなして elog(ERROR) を発生させる。
  • **境界条件**: freelist が空の場合、NULL を返して呼び出し元に制御を戻す。

保守担当者向けメモ

  • 本関数は Postmaster プロセス内でのみ実行されることを前提としている。
  • MarkPostmasterChildSlotAssigned による共有メモリ更新は、Postmaster がシグナル送信等の判断を行うための重要な同期ポイントである。
