xideq

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xid.c:80

目的

2つのトランザクション識別子（TransactionId）が等しいかどうかを判定する。

シグネチャまたは形状

Datum xideq(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る2つの TransactionId 型引数。
  • **出力**: bool 型（Datum 形式）。両者が等しい場合は true、そうでない場合は false を返す。

前提条件と不変条件

  • 入力引数は PG_GETARG_TRANSACTIONID マクロで正しく取得可能であること。
  • 比較ロジックは TransactionIdEquals マクロ（または関数）の定義に依存する。

アルゴリズムまたは振る舞い

1. PG_GETARG_TRANSACTIONID(0) および (1) を使用して、2つのトランザクションIDを抽出する。 2. TransactionIdEquals を呼び出し、抽出した2つのIDの等価性を評価する。 3. 評価結果を PG_RETURN_BOOL を介して呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（SQLレベルでの比較演算子として呼び出される）。
  • **呼び出し先**: TransactionIdEquals（access/transam.h で定義されたプリミティブ）。

関連データ構造

  • TransactionId: PostgreSQLのトランザクション識別子。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • 引数が無効な場合、PG_GETARG_TRANSACTIONID マクロの挙動に依存する。
  • TransactionId は循環する可能性があるが、本関数は単純な等価比較を行うため、循環による大小関係の考慮は不要である。

保守担当者向けメモ

  • 本関数は単なる等価比較用である。トランザクションの前後関係（論理的な順序）を判定する場合は、TransactionIdPrecedes 等の専用関数を使用すること。
