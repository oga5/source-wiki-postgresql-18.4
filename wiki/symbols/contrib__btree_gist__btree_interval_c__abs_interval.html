abs_interval

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_interval.c:112

目的

interval 型の値を絶対値に正規化します。主に距離計算の過程で、負のインターバルを正の値へ変換するために使用されます。

シグネチャまたは形状

Interval *abs_interval(Interval *a)

入力と出力

  • **入力**: Interval *a （対象となるインターバル値）
  • **出力**: Interval * （入力値が負の場合は符号反転した値、それ以外は入力値そのもの）

前提条件と不変条件

  • 入力ポインタ a は有効な Interval 構造体を指している必要があります。
  • 内部で interval_lt および interval_um 関数を呼び出すため、PostgreSQL の関数マネージャ（fmgr）が利用可能な環境である必要があります。

アルゴリズムまたは振る舞い

1. 静的なゼロ値 zero を定義します。 2. interval_lt を使用して、入力 a が zero より小さいか判定します。 3. a < 0 の場合、interval_um（単項マイナス演算子）を適用して符号を反転させ、その結果を返します。 4. a >= 0 の場合は、入力 a をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_interval.c
  • contrib/btree_gist/btree_time.c
  • contrib/btree_gist/btree_ts.c
  • **呼び出し先**:
  • interval_lt (PostgreSQL 内部関数)
  • interval_um (PostgreSQL 内部関数)

関連データ構造

  • Interval: PostgreSQL の時間間隔を表す構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、PostgreSQL の関数呼び出し規約に従い、セグメンテーション違反が発生する可能性があります。
  • interval_um がオーバーフローを引き起こすような極端な値（最小値など）が入力された場合の挙動は、基盤となる interval_um の実装に依存します。

保守担当者向けメモ

  • 本関数は btree_utils_num.h で extern 宣言されており、複数の btree_gist 実装ファイルから共有されています。
  • 距離計算ロジックの正規化ステップとして機能しているため、変更時は interval 型の比較ロジックとの整合性に注意してください。
