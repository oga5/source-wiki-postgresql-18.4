PLy_spi_exception_set

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_spi.c:616

目的

PostgreSQL の SPI 実行中に発生したエラー（ErrorData）を、Python の例外オブジェクトへ変換して送出する。

シグネチャまたは形状

static void PLy_spi_exception_set(PyObject *excclass, ErrorData *edata)

入力と出力

  • **入力**:
  • excclass: 送出する Python 例外クラス。
  • edata: PostgreSQL のエラー情報を含む ErrorData 構造体。
  • **出力**: なし（戻り値は void）。成功時は Python インタプリタの例外状態を更新し、失敗時は PostgreSQL の elog(ERROR, ...) を呼び出す。

前提条件と不変条件

  • edata は有効な ErrorData 構造体を指している必要がある。
  • Python インタプリタが初期化されている環境で呼び出されること。

アルゴリズムまたは振る舞い

1. edata->message を用いて Python 例外オブジェクトを生成する。 2. edata 内の各種エラー詳細情報（SQL エラーコード、詳細、ヒント、クエリ位置、スキーマ/テーブル/カラム名等）をタプルにパックし、spidata として生成する。 3. 生成した spidata を例外オブジェクトの属性 spidata として設定する。 4. PyErr_SetObject を呼び出し、Python 側に例外をセットする。 5. 処理の各段階でメモリ確保やオブジェクト生成に失敗した場合は、failure ラベルへ遷移し、確保済みのリソースを解放した上で PostgreSQL の ERROR を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_spi.c 内の各 SPI 実行関数（PG_CATCH ブロック内から呼び出される）。
  • **呼び出し先**:
  • Py_BuildValue, PyObject_CallObject, PyObject_SetAttrString, PyErr_SetObject, Py_DECREF, Py_XDECREF（Python C API）。
  • elog（PostgreSQL エラー報告関数）。

関連データ構造

  • ErrorData: PostgreSQL のエラー情報を保持する構造体。
  • PyObject: Python のオブジェクト表現。

失敗モードと境界条件

  • Python オブジェクトの生成（Py_BuildValue 等）が失敗した場合、elog(ERROR, ...) により PostgreSQL 側のトランザクションがアボートされる。
  • spidata の属性設定に失敗した場合も同様にエラーとして扱われる。

保守担当者向けメモ

  • 本関数は PG_CATCH ブロック内で呼び出されることを想定している。Python の例外状態をセットした後、呼び出し元で適切に制御を戻す必要がある。
  • spidata に格納されるフィールドは edata の構造に依存しているため、PostgreSQL 側の ErrorData 定義が変更された場合は本関数のタプル生成ロジックも更新が必要。
