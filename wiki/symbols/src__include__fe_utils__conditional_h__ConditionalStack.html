ConditionalStack

定義

  • 種別: typedef (ポインタ型)
  • 定義箇所: src/include/fe_utils/conditional.h:71
  • 実体: struct ConditionalStackData へのポインタ

目的

クライアントアプリケーション（psql, pgbench 等）において、ネストされた \if ブロックの実行状態を管理するためのスタック構造を提供します。

シグネチャまたは形状

typedef struct ConditionalStackData *ConditionalStack;

入力と出力

  • 本シンボル自体はスタックへのハンドルとして機能します。
  • conditional_stack_create() 等の関数により生成され、スタック操作関数（push, pop, peek 等）の引数として渡されます。

前提条件と不変条件

  • ConditionalStackData を指す有効なポインタである必要があります。
  • スタックの各要素は IfStackElem で構成され、連結リストとして管理されます。
  • 親ブロックの状態は子ブロックに継承される設計となっています。

アルゴリズムまたは振る舞い

  • インタプリタが \if に遭遇した際にスタックへ状態を積み、\endif で取り出します。
  • conditional_active() 関数がスタックのトップを参照し、現在のブロックが実行対象か否かを判定します。
  • 非アクティブなブロックでは、解析中のクエリ長や括弧の深さなどのメタデータを保持し、バッファ制御を支援します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pgbench/pgbench.c
  • src/bin/psql/command.c（exec_command_if, exec_command_elif, exec_command_else, exec_command_endif 等）
  • **呼び出し先**:
  • 本シンボルはデータ型であるため、直接の呼び出し先はありません。操作関数（src/fe_utils/conditional.c 実装）が本型を引数として受け取ります。

関連データ構造

  • ConditionalStackData: スタックの実体構造体。
  • IfStackElem: スタック内の各要素（連結リストのノード）。
  • ifState: 条件分岐の状態（IFSTATE_TRUE, IFSTATE_FALSE, IFSTATE_IGNORED 等）を定義する列挙型。

失敗モードと境界条件

  • スタックが空の状態で pop や peek を行った場合の挙動は、実装側の関数定義に依存します。
  • 連結リストのメモリ確保失敗については、標準的な C のメモリ管理ルールに従います。

保守担当者向けメモ

  • 本シンボルは ConditionalStackData への不透明ポインタとして定義されています。内部構造を変更する場合は src/fe_utils/conditional.c との整合性に注意してください。
  • 非アクティブなブロックのテキスト破棄処理と密接に関連しているため、解析器（psqlscan）との連携ロジックを修正する際は本スタックの状態遷移を考慮する必要があります。
