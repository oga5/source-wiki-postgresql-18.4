CoercionPathType

定義

  • 種別: enum
  • 定義箇所: src/include/parser/parse_coerce.h:24

目的

PostgreSQLのパーサーにおいて、あるデータ型から別のデータ型へ変換する際に、どのような手法（パス）を用いるべきかを分類するために使用されます。

シグネチャまたは形状

typedef enum CoercionPathType
{
    COERCION_PATH_NONE,         /* 変換パスが見つからない */
    COERCION_PATH_FUNC,         /* 指定された変換関数を適用 */
    COERCION_PATH_RELABELTYPE,  /* バイナリ互換キャスト（関数不要） */
    COERCION_PATH_ARRAYCOERCE,  /* ArrayCoerceExprノードが必要 */
    COERCION_PATH_COERCEVIAIO,  /* CoerceViaIOノードが必要 */
} CoercionPathType;

入力と出力

  • 本シンボルは列挙型であり、関数や式の評価結果として返される値です。
  • 主に find_coercion_pathway 等の変換判定関数が、変換の可否と手法を決定した結果としてこの値を返します。

前提条件と不変条件

  • COERCION_PATH_NONE は、変換が不可能であることを示します。
  • その他の値は、対応する変換ノードの生成や処理が必要であることを示唆します。

アルゴリズムまたは振る舞い

  • パーサーは、型変換が必要な場面でシステムカタログ（pg_cast等）を参照し、変換パスを決定します。
  • 決定された CoercionPathType に基づき、後続の処理で適切なASTノード（CoerceViaIO, ArrayCoerceExpr 等）が生成されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_coerce.c 内の型変換ロジック、および src/backend/commands/tablecmds.c における外部キー制約等の型チェック処理から参照されています。
  • **呼び出し先**: 本シンボル自体は列挙型であり、直接的な呼び出し先はありません。

関連データ構造

  • CoerceViaIO: COERCION_PATH_COERCEVIAIO の際に生成されるノード。
  • ArrayCoerceExpr: COERCION_PATH_ARRAYCOERCE の際に生成されるノード。
  • pg_cast: 型変換ルールを保持するシステムカタログ。

失敗モードと境界条件

  • COERCION_PATH_NONE が返された場合、型変換のパスが見つからなかったことを意味し、パーサーはエラーを発生させるか、別の解決策を試みる必要があります。

保守担当者向けメモ

  • 新しい変換手法を追加する場合は、この列挙型に値を追加し、parse_coerce.c 内の変換判定ロジックおよびノード生成ロジックを更新する必要があります。
  • COERCION_PATH_RELABELTYPE はバイナリ互換であることを前提としているため、物理的なデータ表現が異なる型に対して誤って適用しないよう注意してください。
