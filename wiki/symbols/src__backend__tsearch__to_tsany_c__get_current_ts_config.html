get_current_ts_config

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/to_tsany.c:47

目的

現在のセッションで設定されている全文検索設定（Text Search Configuration）の OID を取得し、SQL レイヤーへ返却する。

シグネチャまたは形状

Datum get_current_ts_config(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL 関数呼び出しとしての標準引数）
  • **出力**: Datum 型として、現在の全文検索設定の OID を返す。

前提条件と不変条件

  • getTSCurrentConfig(true) が呼び出されるため、設定が未定義の場合はエラーが発生する可能性がある（true 引数は設定の存在を強制する）。

アルゴリズムまたは振る舞い

1. getTSCurrentConfig(true) を呼び出し、現在の全文検索設定を取得する。 2. 取得した OID を PG_RETURN_OID マクロを介して呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェースから呼び出される（直接的な C 言語レベルの呼び出し元情報は提供されていない）。
  • **呼び出し先**: getTSCurrentConfig（tsearch/ts_cache.h で定義されていると推測される関数）。

関連データ構造

  • Oid: 全文検索設定を識別するためのオブジェクト識別子。

失敗モードと境界条件

  • 現在のセッションで全文検索設定が設定されていない場合、getTSCurrentConfig(true) の内部動作によりエラーが発生する。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的なロジックは getTSCurrentConfig に依存している。
  • 動作の詳細は tsearch/ts_cache.h および関連するキャッシュ管理の実装を参照すること。
