cfunc_hashtable_delete

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/funccache.c:206

目的

グローバルな関数キャッシュハッシュテーブル（cfunc_hashtable）から、指定された CachedFunction エントリを削除し、関連するリソースを解放する。

シグネチャまたは形状

static void cfunc_hashtable_delete(CachedFunction *function)

入力と出力

  • **入力**: function (削除対象の CachedFunction 構造体へのポインタ)
  • **出力**: なし（副作用としてハッシュテーブルおよびメモリ状態を更新）

前提条件と不変条件

  • function がハッシュテーブルに登録されていない場合（fn_hashkey が NULL）、本関数は何も行わず即座に復帰する。
  • 削除対象の CachedFunction が保持する callResultType（TupleDesc）は、ハッシュテーブルからの削除後に FreeTupleDesc を通じて解放される。

アルゴリズムまたは振る舞い

1. function->fn_hashkey が NULL であれば終了する。 2. function->fn_hashkey から callResultType を一時的に保持する。 3. hash_search を HASH_REMOVE モードで呼び出し、cfunc_hashtable から該当エントリを削除する。 4. 削除に失敗した場合（hentry が NULL）、警告ログを出力する。 5. function->fn_hashkey を NULL に設定し、バックリンクを無効化する。 6. 保持していた callResultType が存在する場合、FreeTupleDesc を呼び出してメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/funccache.c:436 (キャッシュ無効化時などの削除処理)
  • **呼び出し先**:
  • hash_search (ハッシュテーブル操作)
  • elog (エラー/警告ログ出力)
  • FreeTupleDesc (タプル記述子の解放)

関連データ構造

  • CachedFunction: キャッシュされた関数の基底構造体。
  • CachedFunctionHashEntry: ハッシュテーブル内のエントリ構造体。
  • cfunc_hashtable: キャッシュを保持するグローバルなハッシュテーブル。

失敗モードと境界条件

  • **ハッシュテーブル不整合**: 削除対象の function がハッシュテーブル内に見つからない場合、elog(WARNING, ...) が発生する。
  • **NULL ポインタ**: function 自体が NULL の場合の挙動は定義されていない（ソースコード上は function->fn_hashkey へのアクセスでクラッシュする可能性がある）。

保守担当者向けメモ

  • 本関数は CachedFunction のメモリ管理の一部を担う。fn_hashkey のバックリンクを適切にクリアすることで、二重解放や不正なメモリ参照を防いでいる。
  • callResultType の解放は、ハッシュテーブル検索後に安全に行われるよう順序が制御されている。
