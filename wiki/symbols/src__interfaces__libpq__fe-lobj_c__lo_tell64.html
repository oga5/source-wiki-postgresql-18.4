lo_tell64

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-lobj.c:547

目的

データベース内のラージオブジェクトにおける現在の読み書き位置（オフセット）を64ビット整数で取得します。

シグネチャまたは形状

int64_t lo_tell64(PGconn *conn, int fd);

入力と出力

  • **入力**:
  • conn: データベース接続を管理する PGconn 構造体へのポインタ。
  • fd: 対象となるラージオブジェクトのファイルディスクリプタ。
  • **出力**:
  • 成功時: 現在のオフセット位置（int64_t）。
  • 失敗時: -1。

前提条件と不変条件

  • lo_initialize(conn) が正常に完了している必要があります。
  • サーバー側で lo_tell64 関数がサポートされている必要があります（conn->lobjfuncs->fn_lo_tell64 が有効であること）。

アルゴリズムまたは振る舞い

1. lo_initialize を呼び出し、サーバー側関数のOIDキャッシュを準備します。 2. fn_lo_tell64 がキャッシュされているか確認し、未定義の場合はエラーを記録して -1 を返します。 3. PQArgBlock を使用してファイルディスクリプタ fd を引数として設定します。 4. PQnfn を介してサーバー側の lo_tell64 関数を呼び出します。 5. サーバーからの応答が PGRES_COMMAND_OK であり、結果長が8バイトであることを確認します。 6. 成功した場合、lo_ntoh64 を用いてネットワークバイトオーダーからホストバイトオーダーへ変換した値を返します。 7. 失敗した場合は PQclear で結果を解放し、-1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/examples/testlo64.c での使用が確認されています。
  • **呼び出し先**: lo_initialize, libpq_append_conn_error, PQnfn, PQresultStatus, PQclear, lo_ntoh64 を呼び出します。

関連データ構造

  • PGconn: 接続状態およびラージオブジェクト関数OIDのキャッシュを保持します。
  • PQArgBlock: サーバー関数呼び出し時の引数構造体。
  • PGresult: サーバーからの実行結果を保持します。

失敗モードと境界条件

  • 接続が初期化できない場合、またはサーバー側で関数がサポートされていない場合は -1 を返します。
  • サーバー側での実行が失敗した場合（PGRES_COMMAND_OK 以外）、-1 を返します。
  • サーバーからの応答サイズが期待値（8バイト）と異なる場合、-1 を返します。

保守担当者向けメモ

  • 64ビット対応の関数であるため、古いサーバーバージョンとの互換性に注意してください。
  • lo_ntoh64 によるバイトオーダー変換が必須です。
  • 呼び出し元でエラーが発生した際は、PQerrorMessage(conn) を通じて詳細なエラー情報を取得可能です。
