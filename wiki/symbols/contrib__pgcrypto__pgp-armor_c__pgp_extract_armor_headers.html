pgp_extract_armor_headers

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-armor.c:389

目的

PGP ASCII アーマー形式のデータから、メタデータヘッダー（キー・値ペア）を抽出します。

シグネチャまたは形状

int pgp_extract_armor_headers(const uint8 *src, unsigned len,
                              int *nheaders, char ***keys, char ***values);

入力と出力

  • **入力**:
  • src: アーマー化されたデータを含むバッファの先頭ポインタ。
  • len: バッファの長さ。
  • **出力**:
  • nheaders: 抽出されたヘッダーの個数が格納されます。
  • keys: 抽出されたキー文字列の配列へのポインタ。
  • values: 抽出された値文字列の配列へのポインタ。
  • **戻り値**: 成功時は 0、エラー時は PXE_PGP_CORRUPT_ARMOR を返します。

前提条件と不変条件

  • keys および values に格納される文字列は、関数内で palloc されたメモリ領域を指します。
  • 呼び出し側は、palloc されたメモリを適切に解放する責任があります。

アルゴリズムまたは振る舞い

1. find_header を使用してアーマーの開始位置と終了位置を特定します。 2. アーマー内のヘッダー行数をカウントし、その数に基づいて keys および values 配列を確保します。 3. ヘッダー部分をメモリにコピーし、各行を走査します。 4. 各行を : を区切り文字として分割し、キーと値のポインタを配列に格納します。 5. 行末の改行コード（CRLF/LF）を適切に処理します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-pgsql.c:944
  • **呼び出し先**: find_header（アーマー境界の特定）、palloc（メモリ確保）、memcpy、strchr、strstr、elog（エラー報告）。

関連データ構造

  • StringInfo（間接的に利用されるバッファ管理構造）

失敗モードと境界条件

  • アーマーの開始・終了ヘッダーが見つからない場合、またはヘッダー行に : が含まれない場合は PXE_PGP_CORRUPT_ARMOR を返します。
  • 予期せぬ行数が発生した場合は elog(ERROR) により処理を中断します。

保守担当者向けメモ

  • 本関数は palloc を使用するため、メモリコンテキストの管理に注意してください。
  • pgp-armor.c 内の Base64 実装は独自のものであるため、変更時は整合性に注意が必要です。
