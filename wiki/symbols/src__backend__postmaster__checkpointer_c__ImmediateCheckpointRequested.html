ImmediateCheckpointRequested

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/checkpointer.c:744

目的

共有メモリ上のチェックポイントフラグを確認し、即時実行（Immediate Checkpoint）が要求されているかどうかを判定する。

シグネチャまたは形状

static bool ImmediateCheckpointRequested(void)

入力と出力

  • **入力**: なし（共有メモリ CheckpointerShmem を直接参照）
  • **出力**: 即時チェックポイントが要求されている場合は true、それ以外は false を返す。

前提条件と不変条件

  • CheckpointerShmem が初期化済みであること。
  • 共有メモリ上の ckpt_flags ビットフィールドを読み取るため、ロック取得は不要である（単一ビットの読み取りであるため）。

アルゴリズムまたは振る舞い

1. 共有メモリ構造体 CheckpointerShmemStruct へのポインタを取得する。 2. ckpt_flags フィールドに対して CHECKPOINT_IMMEDIATE ビットの論理積をとる。 3. ビットが立っていれば true を、立っていなければ false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/checkpointer.c:787（チェックポイント実行ループ内での判定）
  • **呼び出し先**: なし

関連データ構造

  • CheckpointerShmemStruct: チェックポインターの状態を保持する共有メモリ構造体。ckpt_flags を含む。

失敗モードと境界条件

  • 共有メモリが正しくアタッチされていない場合、プロセスがクラッシュする可能性がある。
  • ckpt_flags は volatile 修飾されているため、常に最新のメモリ状態が読み取られる。

保守担当者向けメモ

  • 本関数はロックを取得せずにフラグを読み取るため、非常に軽量である。
  • 即時チェックポイントの要求は、主にバックエンドプロセスからのシグナルや要求によって ckpt_flags が更新されることで発生する。
