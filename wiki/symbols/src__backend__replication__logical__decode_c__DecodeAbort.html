DecodeAbort

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/decode.c:837

目的

WAL（Write-Ahead Log）のトランザクションアボートレコードを処理し、論理デコーディングのコンテキストにおいて該当トランザクションの破棄またはロールバックを実行する。

シグネチャまたは形状

static void DecodeAbort(LogicalDecodingContext *ctx, XLogRecordBuffer *buf,
                        xl_xact_parsed_abort *parsed, TransactionId xid,
                        bool two_phase)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • buf: WAL レコードと LSN 範囲を保持するバッファ。
  • parsed: 解析済みのトランザクションアボート情報。
  • xid: アボート対象のトランザクション ID。
  • two_phase: 2フェーズコミット（2PC）のトランザクションかどうかを示すフラグ。
  • **出力**: なし（副作用として reorderbuffer の状態を更新）。

前提条件と不変条件

  • parsed には有効なアボート時刻およびサブトランザクション情報が含まれている必要がある。
  • ctx->reorder が初期化されていること。

アルゴリズムまたは振る舞い

1. **メタデータ抽出**: レコードからオリジン情報（origin_id, origin_lsn）およびアボート時刻を特定する。 2. **フィルタリング判定**: DecodeTXNNeedSkip を呼び出し、当該トランザクションを処理対象とするか判定する。 3. **処理の分岐**:

  • two_phase が真かつスキップ対象外の場合: ReorderBufferFinishPrepared を呼び出し、準備済みトランザクションのロールバックを完了させる。
  • それ以外の場合:
  • parsed->subxacts に含まれるすべてのサブトランザクションに対して ReorderBufferAbort を実行。
  • メイントランザクション（xid）に対して ReorderBufferAbort を実行。

    • 4. **統計更新**: UpdateDecodingStats を呼び出し、デコーディング統計を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/decode.c:270（LogicalDecodingProcessRecord 内のディスパッチ処理から呼び出される）。
  • **呼び出し先**:
  • DecodeTXNNeedSkip
  • ReorderBufferFinishPrepared
  • ReorderBufferAbort
  • UpdateDecodingStats

関連データ構造

  • LogicalDecodingContext: デコーディングの全体状態。
  • XLogRecordBuffer: WAL レコードの物理構造。
  • xl_xact_parsed_abort: アボートレコードの解析済みデータ。
  • ReorderBuffer: 変更を蓄積するバッファ。

失敗モードと境界条件

  • two_phase が真の場合、準備済みトランザクションの終了処理が行われるが、スキップ対象の場合は何も行われない。
  • サブトランザクションが存在する場合、ループ処理によりすべてのアボートが順次適用される。

保守担当者向けメモ

  • 本関数は xact_decode の一部として呼び出されることを想定している。
  • 2フェーズコミットの処理パスと通常のアボート処理パスが明確に分かれているため、変更時は両方の整合性に注意すること。
