shmem_request_hook_type

定義

  • 種別: typedef
  • 定義箇所: src/include/miscadmin.h:533

目的

共有メモリの要求（Shared Memory Request）フェーズにおいて、拡張モジュールやサブシステムが独自の共有メモリ割り当てを登録するためのフック関数の型を定義します。

シグネチャまたは形状

typedef void (*shmem_request_hook_type) (void);

入力と出力

  • 入力: なし（void）
  • 出力: なし（void）

前提条件と不変条件

  • このフックは、PostgreSQLの共有メモリ初期化プロセスにおいて、各モジュールが RequestAddinShmemSpace() を呼び出すタイミングで実行されることが想定されています。
  • 実行時に shmem_request_hook が NULL でない場合、その関数ポインタが呼び出されます。

アルゴリズムまたは振る舞い

1. システムの共有メモリ要求フェーズにおいて、登録されたフック関数が実行されます。 2. フック関数内では、通常 RequestAddinShmemSpace() を呼び出し、必要な共有メモリサイズをシステムに通知します。 3. 複数のモジュールがフックを利用する場合、既存のフックを退避（保存）し、新しいフックを登録する「チェイン」形式の実装が一般的です。

呼び出し元と呼び出し先

  • 呼び出し元: 共有メモリの初期化を行うバックエンドの起動プロセス（src/backend/utils/init/miscinit.c 等の初期化フロー）。
  • 呼び出し先: 拡張モジュールやテストモジュール（contrib/pg_stat_statements, src/test/modules/injection_points 等）で定義された、共有メモリサイズを要求する関数。

関連データ構造

  • shmem_request_hook: miscadmin.h で宣言されているグローバル変数。この型の関数ポインタを保持します。

失敗モードと境界条件

  • フック関数内で共有メモリの要求量を超過するような処理を行った場合、システム起動時の共有メモリ割り当てに失敗する可能性があります。
  • フックが NULL の場合、呼び出し側は何も実行しません。

保守担当者向けメモ

  • このフックは、拡張モジュールが動的に共有メモリを確保するための重要な接点です。
  • 複数のモジュールがフックを上書きしないよう、実装時には必ず prev_shmem_request_hook のような変数に既存のフックを保存し、自身の処理後に呼び出すようにしてください。
