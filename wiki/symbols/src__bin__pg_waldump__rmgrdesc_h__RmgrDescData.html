RmgrDescData

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_waldump/rmgrdesc.h:14

目的

PostgreSQL の WAL（Write-Ahead Log）レコードを解析・表示するためのリソースマネージャ（rmgr）固有のメソッドと情報を保持するデータ構造です。pg_waldump が WAL レコードを人間が読める形式に変換する際のインターフェースとして機能します。

シグネチャまたは形状

typedef struct RmgrDescData
{
	const char *rm_name;
	void		(*rm_desc) (StringInfo buf, XLogReaderState *record);
	const char *(*rm_identify) (uint8 info);
} RmgrDescData;

入力と出力

  • rm_name: リソースマネージャの名前（例: "Heap"）。
  • rm_desc: StringInfo バッファにレコードの詳細な記述を書き出す関数。
  • rm_identify: レコードのサブタイプ（info）から識別文字列を返す関数。

前提条件と不変条件

  • rm_desc 関数は、StringInfo バッファへの書き込みを通じてレコード内容をデコードします。
  • rm_identify 関数は、WAL レコードの info フィールドを引数として受け取り、対応する識別子を返します。

アルゴリズムまたは振る舞い

pg_waldump は、WAL レコードから取得した RmgrId を用いて GetRmgrDesc() を呼び出し、対応する RmgrDescData を取得します。その後、構造体内の関数ポインタを介して、レコード固有のデコード処理や識別処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_waldump/pg_waldump.c 内のメインループにおいて、WAL レコードの解析時に GetRmgrDesc() を経由して参照されます。
  • **呼び出し先**: src/bin/pg_waldump/rmgrdesc.c 内の RmgrDescTable および CustomRmgrDesc 配列で実体が定義されています。

関連データ構造

  • StringInfo: デコード結果を格納するための動的文字列バッファ。
  • XLogReaderState: WAL レコードの読み取り状態を保持する構造体。
  • RmgrId: リソースマネージャを識別するための ID。

失敗モードと境界条件

  • 組み込みのリソースマネージャに該当しない場合、rmgrdesc.c の実装により "custom###" という形式のフォールバック名が使用される仕組みになっています。

保守担当者向けメモ

  • 本構造体はフロントエンド（pg_waldump）専用であり、バックエンドの WAL 処理ロジックとは分離されています。
  • 新しいリソースマネージャを追加する場合は、rmgrdesc.c のテーブル定義を更新する必要があります。
