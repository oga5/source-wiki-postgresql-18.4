ReindexIndexCallbackState

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/indexcmds.c:123

目的

REINDEX コマンド実行時、特に並行処理（CONCURRENTLY）において、コマンドのオプション情報と、既にロックを取得済みのテーブルの OID を保持し、コールバック間で状態を共有するために使用されます。

シグネチャまたは形状

struct ReindexIndexCallbackState
{
    ReindexParams params;       /* ステートメントからのオプション */
    Oid         locked_table_oid;   /* 以前にロックされたテーブルの追跡 */
};

入力と出力

  • 本構造体は、REINDEX 処理のコンテキスト内でインスタンス化され、コールバック関数へ引数として渡されます。
  • params フィールドには実行時の再構築オプションが格納され、locked_table_oid にはデッドロック防止等のためにロック済みのテーブル識別子が格納されます。

前提条件と不変条件

  • locked_table_oid は、処理の整合性を保つために、対応するテーブルが適切にロックされていることを前提とします。

アルゴリズムまたは振る舞い

  • REINDEX コマンドの実行フローにおいて、状態管理オブジェクトとして機能します。
  • src/backend/commands/indexcmds.c 内の関数で、この構造体を介して ReindexParams を参照し、並行再構築の各フェーズで必要なパラメータを維持します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/indexcmds.c 内の REINDEX 関連処理（2924行目付近でインスタンス化）。
  • 呼び出し先: コールバック関数内で引数として参照されます（2979行目付近）。

関連データ構造

  • ReindexParams: REINDEX コマンドのオプションを保持する構造体。

失敗モードと境界条件

  • 構造体自体に固有の失敗モードはありませんが、保持する locked_table_oid が無効な場合や、ロックの順序が不適切な場合に、後続のカタログ操作やロック取得でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は REINDEX の状態遷移を跨いで情報を保持するため、メモリコンテキストの生存期間と、トランザクションの境界を意識して管理してください。
