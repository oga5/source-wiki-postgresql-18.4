write_duplicated_handle

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/launch_backend.c:814

目的

Windows環境において、親プロセス（Postmaster）が保持するハンドルを子プロセスへ継承可能な形式で複製し、指定された格納先へ書き出します。

シグネチャまたは形状

static bool write_duplicated_handle(HANDLE *dest, HANDLE src, HANDLE childProcess)

入力と出力

  • **入力**:
  • dest: 複製されたハンドルを格納するポインタ。
  • src: 複製元のハンドル。
  • childProcess: 複製先となる子プロセスのハンドル。
  • **出力**:
  • 成功時は true を返し、dest に複製されたハンドルが格納されます。
  • 失敗時は false を返します。

前提条件と不変条件

  • Windows環境（EXEC_BACKEND モード）での実行を前提としています。
  • childProcess は有効なプロセスハンドルである必要があります。

アルゴリズムまたは振る舞い

1. DuplicateHandle 関数を呼び出し、src ハンドルを childProcess のプロセス空間で利用可能な形式で複製します。 2. 複製時に DUPLICATE_CLOSE_SOURCE フラグを指定し、親プロセス側のソースハンドルを閉じます。 3. DUPLICATE_SAME_ACCESS フラグにより、元のハンドルと同じアクセス権限を維持します。 4. 複製に成功した場合、結果のハンドルを *dest に代入して true を返します。 5. 失敗した場合は ereport(LOG, ...) でエラーを記録し、false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c 内の save_backend_variables 関数等から呼び出されます。
  • **呼び出し先**: Windows API の DuplicateHandle および PostgreSQL のエラー報告関数 ereport。

関連データ構造

  • HANDLE: Windows API で定義されるハンドル型。

失敗モードと境界条件

  • DuplicateHandle が失敗した場合（例: 無効なハンドルや権限不足）、エラーログを出力して処理を中断します。

保守担当者向けメモ

  • 本関数は EXEC_BACKEND モードにおけるプロセス間リソース引き継ぎの基盤です。
  • DuplicateHandle のフラグ設定（DUPLICATE_CLOSE_SOURCE）により、呼び出し元ではハンドルがクローズされる点に注意してください。
