CreateCachedPlanForQuery

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/plancache.c:262

目的

解析済みのクエリツリーから CachedPlanSource を初期化し、キャッシュエントリを作成する。

シグネチャまたは形状

CachedPlanSource *CreateCachedPlanForQuery(Query *analyzed_parse_tree,
                                           const char *query_string,
                                           CommandTag commandTag)

入力と出力

  • **入力**:
  • analyzed_parse_tree: 解析済みの Query 構造体。
  • query_string: クエリのソース文字列。
  • commandTag: クエリのコマンドタグ。
  • **出力**:
  • 初期化された CachedPlanSource 構造体へのポインタ。

前提条件と不変条件

  • analyzed_parse_tree は有効な解析済みツリーであること。
  • plansource->context（キャッシュ用メモリコンテキスト）が CreateCachedPlan によって適切に割り当てられていること。

アルゴリズムまたは振る舞い

1. CreateCachedPlan を呼び出し、基本的な CachedPlanSource 構造体を生成する。 2. plansource->context にメモリコンテキストを切り替える。 3. copyObject を使用して、入力された analyzed_parse_tree をキャッシュのメモリコンテキスト内に複製して格納する。 4. 元のメモリコンテキストに戻し、生成された plansource を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/functions.c:928 での呼び出しが確認されている。
  • **呼び出し先**: CreateCachedPlan（plancache.c 内）、copyObject（ノード複製用）。

関連データ構造

  • CachedPlanSource: クエリのソースや解析済みツリーを保持する主要構造体。
  • Query: 解析済みクエリツリー。

失敗モードと境界条件

  • メモリ不足時には copyObject または CreateCachedPlan 内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は CreateCachedPlan のラッパーとして機能しており、解析済みツリーをキャッシュに保持する点に特化している。
  • copyObject を使用しているため、analyzed_parse_tree に含まれるすべてのノードがディープコピーされる。
