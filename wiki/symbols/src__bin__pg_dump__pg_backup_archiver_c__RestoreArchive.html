RestoreArchive

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_archiver.c:342

目的

アーカイブデータの復元処理におけるメインループを制御し、環境の初期化、接続確立、および復元対象の準備を行う。

シグネチャまたは形状

void RestoreArchive(Archive *AHX)

入力と出力

  • **入力**: Archive *AHX (アーカイブハンドルへのポインタ)
  • **出力**: なし（副作用としてデータベースへのデータ復元やファイル出力を行う）

前提条件と不変条件

  • AHX は ArchiveHandle 構造体としてキャスト可能であること。
  • 並列復元を行う場合、アーカイブフォーマットが ClonePtr および ReopenPtr をサポートしていること。
  • 8.0 以前の古いアーカイブ形式では並列復元は不可。

アルゴリズムまたは振る舞い

1. **初期化**: アーカイブの状態を STAGE_INITIALIZING に設定。 2. **並列モード判定**: 並列復元が要求されている場合、フォーマットの互換性を検証し、入力ファイルを再オープンする。 3. **圧縮チェック**: 復元に必要な圧縮形式がサポートされているか検証。 4. **TOC 構築**: インデックス配列が未作成であれば buildTocEntryArrays を呼び出し構築。 5. **DB 接続**: ropt->useDB が有効な場合、データベースへ接続し、バージョン互換性を設定。 6. **スキーマ復元判定**: スキーマ復元が要求されている場合、TOC 内にスキーマエントリが存在するか確認し、存在しない場合は dumpSchema フラグを無効化。 7. **出力設定**: SaveOutput および SetOutput を用いて出力先を構成。 8. **ヘッダー出力**: ダンプ情報やバージョン情報を出力。 9. **トランザクション開始**: single_txn が有効な場合、トランザクションを開始。 10. **状態遷移**: STAGE_PROCESSING へ移行し、dropSchema が有効な場合は逆順で削除処理を開始。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_restore.c
  • src/bin/pg_dump/pg_dump.c
  • src/bin/pg_dump/pg_backup_tar.c
  • src/bin/pg_dump/pg_backup_null.c
  • **呼び出し先**:
  • AH->ReopenPtr
  • buildTocEntryArrays
  • ConnectDatabaseAhx
  • SaveOutput / SetOutput
  • ahprintf
  • StartTransaction
  • _doSetFixedOutputState

関連データ構造

  • ArchiveHandle: アーカイブの全体状態を保持。
  • RestoreOptions: 復元時のオプション設定。
  • TocEntry: ダンプ対象オブジェクトのメタデータ。

失敗モードと境界条件

  • **並列復元不可**: サポート外のフォーマットや古いアーカイブ形式の場合、pg_fatal で終了。
  • **圧縮エラー**: サポートされていない圧縮形式が指定されている場合、pg_fatal で終了。
  • **接続エラー**: データベース接続失敗時に pg_fatal で終了。

保守担当者向けメモ

  • 本関数は復元処理の入り口であり、STAGE_PROCESSING への移行前後で処理が大きく分かれる。
  • ropt->createDB モード時の DROP 処理は、接続先データベースの整合性を保つため、特定のオブジェクト（DATABASE/DATABASE PROPERTIES）のみに限定される点に注意。
