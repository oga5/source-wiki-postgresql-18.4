shell_run_command

定義

  • 種別: function
  • 定義箇所: contrib/basebackup_to_shell/basebackup_to_shell.c:255

目的

バックアップデータ転送用の外部シェルコマンドを構築し、パイプ経由で実行を開始する。

シグネチャまたは形状

static void shell_run_command(bbsink_shell *sink, const char *filename)

入力と出力

  • **入力**:
  • sink: 外部コマンド実行状態を保持する bbsink_shell 構造体へのポインタ。
  • filename: コマンド文字列内の %f プレースホルダーを置換するために使用するファイル名。
  • **出力**: なし（成功時は sink->pipe および sink->current_command が設定される）。

前提条件と不変条件

  • 実行開始時に sink->current_command および sink->pipe が NULL であること（既にコマンドが実行中でないこと）。

アルゴリズムまたは振る舞い

1. shell_construct_command を呼び出し、設定されたシェルコマンド文字列内のプレースホルダー（%f, %d）を filename および sink->target_detail で置換してコマンドを生成する。 2. 生成されたコマンドを OpenPipeStream に渡し、書き込みモード（PG_BINARY_W）でパイプを開く。 3. パイプのオープンに失敗した場合は、ereport(ERROR, ...) を発行してバックアップ処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/basebackup_to_shell/basebackup_to_shell.c:314 (bbsink_shell_begin_archive 内)
  • contrib/basebackup_to_shell/basebackup_to_shell.c:350 (bbsink_shell_begin_manifest 内)
  • **呼び出し先**:
  • shell_construct_command
  • OpenPipeStream
  • ereport

関連データ構造

  • bbsink_shell: コマンド実行状態（パイプ、現在のコマンド文字列、ターゲット詳細）を保持する構造体。

失敗モードと境界条件

  • OpenPipeStream が失敗した場合、エラーが発生しバックアップ処理全体が停止する。
  • 既にコマンドが実行中の状態で呼び出された場合、Assert によりデバッグビルドで停止する。

保守担当者向けメモ

  • 本関数は bbsink フレームワークのライフサイクル管理の一部として呼び出されます。
  • 外部コマンドの実行にはセキュリティリスクが伴うため、target_detail のバリデーション（shell_check_detail）が先行して行われていることを前提としています。
