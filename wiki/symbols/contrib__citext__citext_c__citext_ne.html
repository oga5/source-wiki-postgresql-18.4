citext_ne

定義

  • 種別: function
  • 定義箇所: contrib/citext/citext.c:218

目的

2つの citext 型の値が等しくないかどうかを判定し、その結果をブール値として返します。

シグネチャまたは形状

Datum citext_ne(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される2つの text 型（citext として扱われる）の引数。
  • **出力**: PG_RETURN_BOOL を介して、両者が等しくない場合に true、等しい場合に false を返します。

前提条件と不変条件

  • 入力された text 型データは、PG_GETARG_TEXT_PP により適切に展開またはコピーされます。
  • 比較の前に str_tolower を使用して、DEFAULT_COLLATION_OID に基づき両方の文字列を小文字に正規化します。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を使用して2つの入力引数を取得します。 2. str_tolower を呼び出し、両方の文字列を小文字に変換した一時的な C 文字列（lcstr, rcstr）を生成します。 3. strcmp を使用して、正規化された文字列同士を比較します。照合順序（strcoll）を介さないビット単位の比較を行うことで、処理コストを削減しています。 4. 比較結果が 0 でない（等しくない）場合に true を、等しい場合に false を保持します。 5. pfree で一時的な文字列メモリを解放し、PG_FREE_IF_COPY で入力引数のメモリを適切に解放します。 6. 最終的なブール値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(citext_ne) マクロにより、PostgreSQL の関数マネージャから呼び出されることが示唆されています。
  • **呼び出し先**:
  • PG_GETARG_TEXT_PP
  • str_tolower
  • strcmp
  • pfree
  • PG_FREE_IF_COPY
  • PG_RETURN_BOOL

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • 入力データが TOAST されている場合、PG_GETARG_TEXT_PP および PG_FREE_IF_COPY が適切に処理を行います。
  • 文字列の長さが異なる場合、strcmp は正しく不等価を判定します。

保守担当者向けメモ

  • 本関数は等価性判定のみを目的としているため、strcoll を使用する比較関数よりも効率的です。
  • str_tolower に渡される DEFAULT_COLLATION_OID は固定されており、大文字・小文字変換のルールが照合順序に依存しないように設計されています。
