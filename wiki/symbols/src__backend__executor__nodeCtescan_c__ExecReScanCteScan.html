ExecReScanCteScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeCtescan.c:306

目的

CTE（共通テーブル式）スキャンノードの再スキャン（Rescan）処理を実行し、Tuplestore の状態をリセットまたは巻き戻しすることで、CTE の再評価または再読み取りを可能にする。

シグネチャまたは形状

void ExecReScanCteScan(CteScanState *node)

入力と出力

  • **入力**: CteScanState *node（再スキャン対象の CTE スキャンノードの状態構造体）
  • **出力**: なし（副作用として node 内の Tuplestore 状態およびスキャンポインタを更新）

前提条件と不変条件

  • node は初期化済みであること。
  • node->leader が有効であり、共有の cte_table（Tuplestorestate）にアクセス可能であること。

アルゴリズムまたは振る舞い

1. 現在の ResultTupleSlot をクリアする。 2. ExecScanReScan を呼び出し、共通のスキャン基盤をリセットする。 3. CTE のサブプランに変更があるか（chgParam が非 NULL か）を確認する。

  • **変更がある場合**: tuplestore_clear を呼び出して Tuplestore を空にし、eof_cte フラグを false にリセットする。これにより、次回スキャン時にサブプランが再実行される。
  • **変更がない場合**: tuplestore_select_read_pointer を使用して自身の読み取りポインタを選択し、tuplestore_rescan を呼び出して既存の Tuplestore 内のデータを先頭から再読み取り可能にする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:222（ExecReScan 経由での呼び出しが確認されている）
  • **呼び出し先**:
  • ExecClearTuple
  • ExecScanReScan
  • tuplestore_clear
  • tuplestore_select_read_pointer
  • tuplestore_rescan

関連データ構造

  • CteScanState: CTE スキャンノードの実行時状態を保持。
  • Tuplestorestate: CTE の結果を保持するストレージ。

失敗モードと境界条件

  • 複数の CTE ノードが冗長に tuplestore_clear を呼び出す可能性があるが、これは許容される動作である。
  • chgParam がクリアされると、以降の再スキャンでは tuplestore_rescan のパスが選択される。

保守担当者向けメモ

  • Tuplestore は work_mem を使用するため、巨大な CTE の場合はディスクへのスピルが発生する可能性がある。
