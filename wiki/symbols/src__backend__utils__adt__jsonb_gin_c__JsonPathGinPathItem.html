JsonPathGinPathItem

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonb_gin.c:117

目的

JSONBのGINインデックス構築時において、JSONパスの階層構造を表現するためのリンクリスト要素として機能します。

シグネチャまたは形状

typedef struct JsonPathGinPathItem
{
	struct JsonPathGinPathItem *parent;
	Datum		keyName;
	JsonPathItemType type;
} JsonPathGinPathItem;

入力と出力

  • parent: 親となるパスアイテムへのポインタ。ルート要素の場合は NULL。
  • keyName: パスアイテムがキー（.key）である場合、そのキー名を表す Datum。それ以外の場合は NULL。
  • type: JsonPathItemType 列挙型による、このパスアイテムのノード種別。

前提条件と不変条件

  • jsonb_ops インデックス構築処理において、パスの走査状態を保持するために使用されます。
  • 構造体内の parent ポインタにより、再帰的なパス構造を逆方向に辿ることが可能です。

アルゴリズムまたは振る舞い

  • インデックス抽出処理において、JSONBドキュメントを走査する際、現在のパス位置を追跡するためにこの構造体が生成・連結されます。
  • gin_extract_jsonb 関数等のコンテキスト内で、パスの深さや種類に応じたインデックスキーの生成を補助します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/jsonb_gin.c 内のインデックス抽出ロジック（gin_extract_jsonb 等）。
  • 呼び出し先: 直接的な関数呼び出しではなく、データ構造としてパスの連結や参照に使用されます。
  • 補足: 具体的な関数呼び出しの連鎖については、提供されたソース excerpt には詳細な記述がありません。

関連データ構造

  • JsonPathGinPath: 本構造体をリストとして保持する上位構造体。
  • JsonPathItemType: jsonpath のノード種別を定義する型。
  • Datum: PostgreSQLの汎用データ表現。

失敗モードと境界条件

  • メモリ不足時: 構造体の割り当てに失敗する可能性があります。
  • 階層の深さ: JSONBドキュメントが極端に深いネストを持つ場合、スタックやメモリの制限に抵触する可能性があります。

保守担当者向けメモ

  • 本構造体は jsonb_ops のインデックス抽出プロセスに特化しています。
  • keyName は Datum 型であるため、メモリ管理や型変換の際には jsonb_gin.c 内のユーティリティ関数（make_text_key 等）との整合性に注意してください。
