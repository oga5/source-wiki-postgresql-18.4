pgstat_get_stat_snapshot_timestamp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat.c:1028

目的

現在の統計スナップショットのタイムスタンプを取得し、スナップショットが有効であるかを示すフラグを返します。

シグネチャまたは形状

TimestampTz pgstat_get_stat_snapshot_timestamp(bool *have_snapshot)

入力と出力

  • **入力**: have_snapshot (bool型へのポインタ): スナップショットの有効性を格納する出力用引数。
  • **出力**: TimestampTz 型のタイムスタンプ。スナップショットが存在しない場合は 0 を返します。

前提条件と不変条件

  • force_stats_snapshot_clear が真の場合、本関数呼び出しの副作用として pgstat_clear_snapshot() が実行され、既存のスナップショットが破棄されます。

アルゴリズムまたは振る舞い

1. force_stats_snapshot_clear が真であれば、pgstat_clear_snapshot() を呼び出してスナップショットをクリアします。 2. pgStatLocal.snapshot.mode が PGSTAT_FETCH_CONSISTENCY_SNAPSHOT であるかを確認します。 3. 一致する場合、have_snapshot に true を設定し、pgStatLocal.snapshot.snapshot_timestamp を返します。 4. 一致しない場合、have_snapshot に false を設定し、0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pgstatfuncs.c:1840
  • **呼び出し先**:
  • pgstat_clear_snapshot() (条件付き)

関連データ構造

  • pgStatLocal: バックエンドローカルな統計状態を保持するグローバル変数。
  • TimestampTz: PostgreSQLのタイムスタンプ型。

失敗モードと境界条件

  • スナップショットが保持されていない（mode が PGSTAT_FETCH_CONSISTENCY_SNAPSHOT ではない）場合、have_snapshot は false となり、戻り値は 0 となります。これはエラーではなく、有効なスナップショットが存在しない状態を示します。

保守担当者向けメモ

  • 本関数は pgStatLocal のスナップショットモードに依存しています。統計の整合性レベル（stats_fetch_consistency GUC）の変更がこのモードに影響を与える点に留意してください。
