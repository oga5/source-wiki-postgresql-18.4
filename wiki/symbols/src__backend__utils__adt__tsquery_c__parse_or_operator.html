parse_or_operator

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery.c:243

目的

tsquery の解析過程において、現在のバッファ位置が OR 演算子として有効な構文であるかを判定し、該当する場合は解析状態を進める。

シグネチャまたは形状

static bool parse_or_operator(TSQueryParserState pstate)

入力と出力

  • **入力**: TSQueryParserState pstate（解析中の状態を保持する構造体）。
  • **出力**: bool 型。OR 演算子として有効な構文であれば true を返し、そうでなければ false を返す。

前提条件と不変条件

  • pstate->buf は解析対象の文字列を指している必要がある。
  • OR 演算子として認識されるためには、文字列が "OR" で始まり、その直後にオペランド（またはオペランドの開始）が存在しなければならない。

アルゴリズムまたは振る舞い

1. バッファの先頭 2 文字が OR（大文字小文字を区別しない）であるかを確認する。 2. OR の直後にオペランドが存在するかを確認する。

  • 文字列の終端である場合は false を返す。
  • OR が単語の一部（-, _, 英数字）として認識される場合は false を返す。

    • 3. OR の後に続く空白以外の文字（オペランドの開始）を探索する。

  • オペランドが見つかる前に文字列が終了した場合は false を返す。

    • 4. 条件をすべて満たした場合、pstate->buf を 2 文字分進め、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsquery.c 内の解析ロジック（else if (parse_or_operator(state)) として呼び出される）。
  • **呼び出し先**: pg_strncasecmp, t_iseq, t_isalnum_cstr, pg_mblen_cstr, isspace（いずれも標準または PostgreSQL ユーティリティ関数）。

関連データ構造

  • TSQueryParserState: 解析中のトークンやバッファ位置を管理する構造体。

失敗モードと境界条件

  • OR 文字列の直後にオペランドが存在しない場合、または OR が他の単語の一部と見なされる場合は false を返す。
  • マルチバイト文字を含む文字列において、pg_mblen_cstr を用いて正しく文字境界をスキップする。

保守担当者向けメモ

  • 本関数は tsquery の構文解析における演算子判定の一部であり、OR 演算子を lexeme（単語）としてではなく演算子として扱うためのガード条件を実装している。
  • 呼び出し元との関係については、ソースコード内の src/backend/utils/adt/tsquery.c:483 からの参照が確認されている。
