hashint4

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hashfunc.c:71

目的

int4 型（32ビット整数）の値をハッシュインデックスやハッシュ結合で使用可能なハッシュ値に変換します。

シグネチャまたは形状

Datum hashint4(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを介して受け取る SQL レベルの引数。
  • **出力**: Datum 型として返される 32 ビットのハッシュ値。

前提条件と不変条件

  • 入力値は int4 型として解釈されます。
  • 等価な入力値に対しては、常に同一のハッシュ値を生成する必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT32(0) を使用して、第 1 引数から int32 値を取得します。 2. 取得した値を hash_uint32 関数に渡し、ハッシュ値を計算します。 3. 計算されたハッシュ値を Datum 型として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のハッシュアクセスメソッド（pg_amproc システムカタログ経由で呼び出されます）。
  • **呼び出し先**: hash_uint32（src/common/hashfn.h で定義されるハッシュ計算関数）。

関連データ構造

  • Datum: PostgreSQL の汎用データ型表現。
  • PG_FUNCTION_ARGS: SQL 関数インターフェース用の引数コンテナ。

失敗モードと境界条件

  • 境界条件に関する特記事項はありません。入力された int32 値の全範囲に対してハッシュ計算を行います。

保守担当者向けメモ

  • 本関数は int4 型専用です。ハッシュ結合のパフォーマンス向上のためにシード値を受け取る必要がある場合は、対応する hashint4extended を使用してください。
  • hashint2 と同様のハッシュアルゴリズムを使用しており、論理的に等価な入力に対して一貫したハッシュ値を生成することが保証されています。
