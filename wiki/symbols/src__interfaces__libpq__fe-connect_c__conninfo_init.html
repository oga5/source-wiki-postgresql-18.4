conninfo_init

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/libpq/fe-connect.c:6196

目的

接続パラメータを保持するための PQconninfoOption 配列を動的に確保し、静的なデフォルト定義（PQconninfoOptions）から値をコピーして初期化する。

シグネチャまたは形状

static PQconninfoOption *conninfo_init(PQExpBuffer errorMessage);

入力と出力

  • **入力**: errorMessage (エラー発生時にメッセージを格納するバッファ)
  • **出力**: 初期化された PQconninfoOption 配列へのポインタ。メモリ確保に失敗した場合は NULL を返し、errorMessage にエラー内容を記録する。

前提条件と不変条件

  • PQconninfoOptions が有効な静的配列として定義されていること。
  • 戻り値の配列は、呼び出し元で適切に解放（PQconninfoFree 等）される必要がある。
  • 配列の末尾は、MemSet によりゼロクリアされた要素で終端される。

アルゴリズムまたは振る舞い

1. PQconninfoOptions の全要素を格納するために必要なメモリを malloc で確保する。 2. 確保に失敗した場合、errorMessage に "out of memory" を設定し NULL を返す。 3. PQconninfoOptions を反復し、各要素を確保した領域へ memcpy でコピーする。 4. 最後に、配列の終端を示すために MemSet でゼロクリアされた要素を配置する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq/fe-connect.c 内の複数箇所（行 463, 2203, 6301, 6504, 6765, 7431）から参照されている。
  • **呼び出し先**:
  • malloc (システム関数)
  • libpq_append_error
  • MemSet

関連データ構造

  • PQconninfoOption: 接続パラメータの定義を保持する構造体。
  • internalPQconninfoOption: PQconninfoOptions の元となる内部定義構造体。
  • PQExpBuffer: エラーメッセージ格納用バッファ。

失敗モードと境界条件

  • **メモリ不足**: malloc が失敗した場合、NULL を返し、エラーメッセージをバッファに書き込む。

保守担当者向けメモ

  • 本関数は PQconninfoOptions の構造をコピーするため、PQconninfoOption の定義に変更を加える際は、コピー処理が適切に行われているか確認すること。
  • 戻り値のメモリ管理は呼び出し側に委ねられているため、メモリリークを防ぐために PQconninfoFree の呼び出しが必須である。
