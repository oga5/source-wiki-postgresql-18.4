BTPendingFSM

定義

  • 種別: struct
  • 定義箇所: src/include/access/nbtree.h:325

目的

VACUUM 処理中に削除された B-tree インデックスページの状態を一時的に保持するための構造体です。

シグネチャまたは形状

typedef struct BTPendingFSM
{
    BlockNumber target;         /* 削除対象のページ番号 */
    FullTransactionId safexid;  /* ページ内の BTDeletedPageData.safexid */
} BTPendingFSM;

入力と出力

  • 入力: VACUUM 処理中に新しく削除されたページの情報。
  • 出力: src/backend/access/nbtree/nbtpage.c 内で、メモリ確保やページ状態の管理に使用されます。

前提条件と不変条件

  • nbtree.c 内部でのみ使用されるプライベートな状態構造体です。
  • VACUUM 処理中にのみ有効なデータです。

アルゴリズムまたは振る舞い

  • src/backend/access/nbtree/nbtpage.c において、work_mem に基づいて計算されたバッファサイズ分だけ palloc でメモリを確保し、削除されたページ情報を格納します。
  • 削除されたページごとに 1 つのエントリが割り当てられます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/nbtree/nbtpage.c (メモリ確保および配列への格納処理)
  • 呼び出し先: 情報なし

関連データ構造

  • BTVacState: VACUUM 処理中の状態を管理する構造体。BTPendingFSM の配列（pendingpages）を保持します。
  • BTDeletedPageData: 削除されたページが保持するメタデータ。safexid の参照元となります。

失敗モードと境界条件

  • maxbufsize の計算において、work_mem の制約および MaxAllocSize による制限を受けます。

保守担当者向けメモ

  • 本構造体は nbtree.c の内部的な VACUUM 処理専用です。変更時は nbtpage.c でのメモリ管理ロジックへの影響を確認してください。
