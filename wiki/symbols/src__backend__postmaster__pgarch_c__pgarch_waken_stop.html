pgarch_waken_stop

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/pgarch.c:297

目的

WALアーカイバプロセスに対し、現在のアーカイブ処理サイクルを完了させた後に終了するよう通知するシグナルハンドラです。

シグネチャまたは形状

static void pgarch_waken_stop(SIGNAL_ARGS)

入力と出力

  • 入力: SIGNAL_ARGS（シグナルハンドラとしての標準引数）
  • 出力: なし（副作用としてグローバル変数およびラッチを更新）

前提条件と不変条件

  • 本関数は SIGUSR2 シグナルのハンドラとして登録されることを前提としています。
  • ready_to_stop 変数がスコープ内でアクセス可能である必要があります。

アルゴリズムまたは振る舞い

1. グローバル変数 ready_to_stop を true に設定し、終了フラグを立てます。 2. SetLatch(MyLatch) を呼び出し、待機中のアーカイバプロセスを起床させます。これにより、プロセスはメインループの次の反復で終了処理へ移行します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/pgarch.c 内で pqsignal(SIGUSR2, pgarch_waken_stop) によりシグナルハンドラとして登録されています。
  • 呼び出し先: SetLatch（PostgreSQLのラッチ機構）。

関連データ構造

  • ready_to_stop: アーカイバプロセスの終了状態を制御するグローバルフラグ。
  • MyLatch: 現在のプロセスが待機状態から脱出するために使用するラッチ。

失敗モードと境界条件

  • 本関数はシグナルハンドラ内から呼び出されるため、非同期シグナル安全（async-signal-safe）な操作のみを行う必要があります。SetLatch はこの要件を満たすよう設計されています。

保守担当者向けメモ

  • 本関数は、アーカイバプロセスが即座に停止するのではなく、現在のアーカイブ作業を完遂させる「グレースフル・シャットダウン」をトリガーするために使用されます。
