CopyLoadInputBuf

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfromparse.c:649

目的

COPY FROM コマンドにおいて、入力バッファ (input_buf) に処理可能なデータが不足している場合に、ソースからデータを読み込み、必要に応じてエンコーディング変換を行ってバッファを補充する。

シグネチャまたは形状

static void CopyLoadInputBuf(CopyFromState cstate)

入力と出力

  • **入力**: CopyFromState cstate（COPY 操作の現在の状態を保持する構造体）。
  • **出力**: なし（副作用として cstate 内のバッファ状態を更新する）。

前提条件と不変条件

  • cstate->raw_buf と cstate->input_buf が同一メモリ領域を指す場合、エンコーディング変換 (need_transcoding) は行われない。
  • cstate->input_buf_index は、呼び出し元によって消費済みとしてマークされた位置を指している必要がある。
  • cstate->raw_buf == cstate->input_buf の場合、cstate->input_buf_index >= cstate->raw_buf_index であること。

アルゴリズムまたは振る舞い

1. **インデックス同期**: input_buf と raw_buf が同一領域の場合、raw_buf_index を input_buf_index に同期させる。 2. **ループ処理**: 以下の手順を繰り返す。

  • CopyConvertBuf(cstate) を呼び出し、未変換データを変換する。
  • 変換の結果、input_buf 内の有効データ量が増加していればリターンする。
  • 変換中にエラー（不正なバイト列や不完全なマルチバイト文字）が検出された場合、CopyConversionError を呼び出す。
  • 入力が終了 (input_reached_eof) していればループを抜ける。
  • 上記以外の場合、CopyLoadRawBuf(cstate) を呼び出してソースから生データを読み込み、再度変換を試みる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfromparse.c 内の COPY 解析処理（CopyReadLine 等の内部）。
  • **呼び出し先**:
  • CopyConvertBuf
  • CopyConversionError
  • CopyLoadRawBuf

関連データ構造

  • CopyFromState: COPY 操作の実行状態、バッファポインタ、インデックス、および EOF/エラーフラグを保持する構造体。

失敗モードと境界条件

  • **変換エラー**: input_reached_error が真の場合、CopyConversionError を経由して処理が中断される。
  • **EOF**: 入力ソースが枯渇し、かつ変換すべきデータが残っていない場合はループを終了する。

保守担当者向けメモ

  • 本関数は COPY のデータ読み込みパイプラインの低レベル層に位置する。
  • raw_buf と input_buf の関係性（同一領域か否か）に依存したインデックス管理が行われているため、バッファ管理ロジックを変更する際は注意が必要。
