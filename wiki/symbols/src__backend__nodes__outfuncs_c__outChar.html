outChar

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/outfuncs.c:195

目的

ノード構造体のフィールドに含まれる単一の文字（char）を、シリアライズ可能なASCII文字列形式に変換して StringInfo バッファへ書き込みます。

シグネチャまたは形状

static void outChar(StringInfo str, char c)

入力と出力

  • **入力**:
  • str: 書き込み先の StringInfo バッファ。
  • c: 出力対象の文字。
  • **出力**: なし（str が指すバッファを更新）。

前提条件と不変条件

  • str は初期化済みの有効な StringInfo である必要があります。

アルゴリズムまたは振る舞い

1. 文字 c がヌル文字（'\0'）である場合、文字列 "<>" を str に追加して終了します。 2. それ以外の場合、文字 c を1文字の文字列として扱い、outToken 関数を呼び出してエスケープ処理を行った上で str に書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/outfuncs.c 内のノード出力処理（例: WRITE_CHAR_FIELD マクロ経由での呼び出しが想定されます）。
  • **呼び出し先**:
  • appendStringInfoString (lib/stringinfo.h)
  • outToken (src/backend/nodes/outfuncs.c)

関連データ構造

  • StringInfo: PostgreSQLの動的文字列構築用データ構造。

失敗モードと境界条件

  • c が '\0' の場合に特殊な表現（"<>"）を用いる点が境界条件です。
  • outToken の内部実装に依存するため、エスケープが必要な文字が含まれる場合はそのルールに従います。

保守担当者向けメモ

  • 本関数は outfuncs.c 内のフィールド出力用ユーティリティです。新しいノード型を追加する際、char 型フィールドの出力には本関数を利用してください。
  • ヌル文字の表現として "<>" を使用する慣習は、readfuncs.c 側での復元処理と対になっているため、変更の際は注意が必要です。
