pg_atomic_compare_exchange_u32_impl

定義

  • 種別: static inline function
  • 定義箇所: src/include/port/atomics/arch-x86.h:161

目的

x86アーキテクチャにおいて、pg_atomic_uint32 型のメモリ値に対してアトミックな比較および交換操作（CAS）を実行します。

シグネチャまたは形状

bool pg_atomic_compare_exchange_u32_impl(volatile pg_atomic_uint32 *ptr, uint32 *expected, uint32 newval);

入力と出力

  • **入力**:
  • ptr: 操作対象となる pg_atomic_uint32 へのポインタ。
  • expected: 期待値が格納されたアドレス。操作失敗時には、メモリの現在の値で更新されます。
  • newval: 交換後の新しい値。
  • **出力**:
  • 戻り値: 比較が成功（メモリ値が expected と一致）した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • ptr はアトミック操作のために適切にアライメントされている必要があります。
  • x86の lock プレフィックス付き命令を使用するため、マルチコア環境でのキャッシュコヒーレンシが保証されます。

アルゴリズムまたは振る舞い

1. インラインアセンブラを用いて lock cmpxchgl 命令を実行します。 2. メモリ上の値と expected（eax レジスタ）を比較します。 3. 一致した場合、メモリに newval を書き込みます。 4. 不一致の場合、メモリの現在の値を expected にロードします。 5. setz 命令により、比較結果（ゼロフラグ）を ret 変数に格納し、最終的に bool 型として返します。 6. memory および cc（条件フラグ）を破壊対象として指定し、コンパイラの最適化による順序入れ替えを防止します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/atomics/generic.h 内の各種アトミック操作関数（pg_atomic_compare_exchange_u32 等）から呼び出されます。
  • **呼び出し先**: CPUのハードウェア命令（cmpxchgl, setz）を直接発行します。

関連データ構造

  • pg_atomic_uint32: アトミック操作対象の32ビット整数型。

失敗モードと境界条件

  • 比較が失敗した場合、関数は false を返し、expected 引数が指すメモリ領域は、操作時点での実際のメモリ値で更新されます。

保守担当者向けメモ

  • 本関数は __x86_64__ および IA-32 環境でのインラインアセンブラ実装です。
  • volatile 修飾子により、コンパイラがメモリ読み込みを最適化で省略することを防いでいます。
  • lock プレフィックスは、x86のTSOモデルにおいてもアトミック性を保証するために必須です。
