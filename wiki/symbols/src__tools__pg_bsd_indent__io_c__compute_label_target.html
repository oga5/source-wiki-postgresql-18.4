compute_label_target

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/io.c:251

目的

ラベル（case 文、プリプロセッサディレクティブ、または通常のラベル）を出力する際の適切なインデントカラム位置を計算する。

シグネチャまたは形状

int compute_label_target(void)

入力と出力

  • **入力**: グローバル変数 ps（parser_state）、case_ind、s_lab、label_offset を参照。
  • **出力**: 計算されたインデント位置（カラム番号）を int 型で返す。

前提条件と不変条件

  • ps が現在の構文解析状態を正しく保持していること。
  • s_lab がラベルバッファを指していること。

アルゴリズムまたは振る舞い

以下の優先順位でインデント位置を決定する： 1. ps.pcase が真の場合: (case_ind * ps.ind_size) + 1 を返す。 2. s_lab の先頭文字が # の場合: 1 を返す（プリプロセッサディレクティブ用）。 3. それ以外の場合: ps.ind_size * (ps.ind_level - label_offset) + 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/tools/pg_bsd_indent/io.c:116 (pad_output への引数として使用)
  • src/tools/pg_bsd_indent/pr_comment.c:137 (count_spaces への引数として使用)
  • **呼び出し先**: 外部関数呼び出しなし。

関連データ構造

  • parser_state (ps): インデントレベルやケース文の状態を保持。
  • s_lab: ラベル文字列を保持するバッファ。

失敗モードと境界条件

  • label_offset の値が ps.ind_level を上回る場合、計算結果が負または 1 未満になる可能性がある。

保守担当者向けメモ

  • 本関数は indent_globs.h で定義されたグローバル状態に強く依存している。
  • case_ind や label_offset は indent の実行時オプションによって制御されるため、変更時は関連する引数解析ロジックを確認すること。
