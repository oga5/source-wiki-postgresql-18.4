check_tidstore_available

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_tidstore/test_tidstore.c:161

目的

テスト対象の TidStore インスタンスが初期化済みであることを検証し、未初期化状態での操作を防止する。

シグネチャまたは形状

static void check_tidstore_available(void)

入力と出力

  • 入力: なし
  • 出力: なし（正常時は何も返さない）

前提条件と不変条件

  • 静的変数 tidstore が NULL でないこと。

アルゴリズムまたは振る舞い

1. 静的変数 tidstore の値を評価する。 2. tidstore が NULL の場合、elog(ERROR, ...) を呼び出し、エラーメッセージを出力して処理を中断する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/test/modules/test_tidstore/test_tidstore.c 内の各テスト操作関数（do_set_block_offsets, check_set_block_offsets, test_is_full 等）から呼び出されている。
  • 呼び出し先:
  • elog (PostgreSQL のエラー報告関数)

関連データ構造

  • tidstore: 本モジュール内で管理される TidStore 型の静的ポインタ変数。

失敗モードと境界条件

  • tidstore が NULL の場合、ERROR レベルのログを発行してプロセスを停止させる。

保守担当者向けメモ

  • 本関数は単一プロセスでのテスト実行を前提としています。TidStore のライフサイクル管理（test_create / test_destroy）と密接に関連しているため、テストの実行順序を変更する際は注意してください。
