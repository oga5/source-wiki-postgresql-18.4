SpGistTypeDesc

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/spgist_private.h:135

目的

SP-GiSTインデックスにおいて、特定のデータ型に関する物理的な属性情報（サイズ、アライメント、格納方式など）を保持し、インデックス操作やタプル構築時に型情報を一元的に扱うために使用されます。

シグネチャまたは形状

typedef struct SpGistTypeDesc
{
    Oid         type;         /* データ型のOID */
    int16       attlen;       /* 型の長さ（バイト数） */
    bool        attbyval;     /* 値渡し可能か */
    char        attalign;     /* アライメント要件 */
    char        attstorage;   /* 格納方式（PLAIN, EXTENDED等） */
} SpGistTypeDesc;

入力と出力

本構造体は、主に fillTypeDesc 関数によって初期化され、インデックスのメタデータやスキャン状態の一部として保持されます。直接的な関数呼び出しの入出力というよりは、データ型情報のコンテナとして機能します。

前提条件と不変条件

  • SP-GiSTのタプル操作（SpGistGetInnerTypeSize や memcpyInnerDatum など）において、対象となるデータ型の属性が正しく設定されている必要があります。
  • attlen、attbyval、attalign は、PostgreSQLのシステムカタログ（pg_type）の定義と一致している必要があります。

アルゴリズムまたは振る舞い

  • インデックス構築やスキャン時に、特定のデータ型（インデックス対象、リーフ値、プレフィックス、ラベルなど）の物理的特性をキャッシュすることで、都度カタログを参照するオーバーヘッドを排除します。
  • memcpyInnerDatum 等の関数に渡されることで、メモリ上のデータコピーやサイズ計算の基準として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/access/spgist/spgutils.c: fillTypeDesc, getSpGistTupleDesc, SpGistGetInnerTypeSize, memcpyInnerDatum
  • src/include/access/spgist_private.h: SpGistState 構造体内の各フィールド（attType, attLeafType, attPrefixType, attLabelType）
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • SpGistState: インデックス操作の共通状態を保持する構造体であり、複数の SpGistTypeDesc をメンバとして保持します。

失敗モードと境界条件

  • 不正な Oid を指定して初期化を試みた場合、fillTypeDesc の実装に依存しますが、無効な属性値が設定される可能性があります。
  • 可変長データ型（attlen == -1）の場合、attbyval は必ず false である必要があります。

保守担当者向けメモ

  • 本構造体は、SP-GiSTのインデックス定義と実際のデータ型の物理レイアウトを橋渡しする重要な役割を担っています。
  • 新しいデータ型をSP-GiSTでサポートする場合、fillTypeDesc が正しく属性を抽出できているか確認してください。
  • 32bit/64bit環境間での互換性を維持するため、attalign の扱いは MAXALIGN マクロと整合している必要があります。
