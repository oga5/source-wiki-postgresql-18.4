pgwin32_bind

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/socket.c:314

目的

Windows環境において、標準的な bind() システムコールをWinsock2 API上でエミュレートし、エラー発生時にPostgreSQLが期待する形式へ変換する。

シグネチャまたは形状

int pgwin32_bind(SOCKET s, struct sockaddr *addr, int addrlen);

入力と出力

  • **入力**:
  • s: バインド対象のソケットハンドル（SOCKET）。
  • addr: アドレス情報を格納した sockaddr 構造体へのポインタ。
  • addrlen: addr が指す構造体のサイズ。
  • **出力**:
  • 成功時は 0 を返す。
  • 失敗時は -1 を返し、内部的に TranslateSocketError() を呼び出してエラーコードを errno に設定する。

前提条件と不変条件

  • winsock2.h が適切に初期化されている環境下で呼び出されること。
  • src/include/port/win32_port.h にて bind マクロとして定義されており、Windowsビルド時に標準の bind の代わりに使用される。

アルゴリズムまたは振る舞い

1. Winsockの bind(s, addr, addrlen) を実行する。 2. 戻り値が負（エラー）の場合、TranslateSocketError() を呼び出し、Winsock特有のエラーコード（WSAxxx）をPOSIX準拠の errno に変換する。 3. 最終的な結果（成功時は0、失敗時は-1）を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/win32_port.h のマクロ定義を通じて、PostgreSQLのネットワーク関連コードから透過的に呼び出される。
  • **呼び出し先**:
  • bind() (Winsock2 API)
  • TranslateSocketError() (エラー変換用ユーティリティ)

関連データ構造

  • SOCKET: Winsock2のソケットハンドル型。
  • struct sockaddr: ソケットアドレス情報を保持する標準構造体。

失敗モードと境界条件

  • bind() が失敗した場合、TranslateSocketError() により errno が設定される。
  • 境界条件に関する具体的な追加情報はソース抜粋からは確認できない。

保守担当者向けメモ

  • 本関数は bind のラッパーであり、ロジックの本体はWinsockの bind に依存している。
  • エラーハンドリングの挙動を変更する場合は、TranslateSocketError の実装を確認すること。
