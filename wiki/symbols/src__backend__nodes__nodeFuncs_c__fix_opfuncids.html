fix_opfuncids

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/nodeFuncs.c:1836

目的

ノードツリー内を走査し、演算子ノード（OpExpr, DistinctExpr, NullIfExpr, ScalarArrayOpExpr 等）に含まれる演算子IDから、対応する関数ID（opfuncid）を解決・設定する。

シグネチャまたは形状

void fix_opfuncids(Node *node)

入力と出力

  • **入力**: node (処理対象となるノードツリーのルートポインタ)
  • **出力**: なし（副作用として、ツリー内の演算子ノードの関数IDフィールドが更新される）

前提条件と不変条件

  • 入力 node は有効なノードツリーであること。
  • 内部で fix_opfuncids_walker を呼び出すため、ツリー構造が expression_tree_walker で走査可能であること。

アルゴリズムまたは振る舞い

1. fix_opfuncids_walker を呼び出し、ツリー全体を再帰的に走査する。 2. 走査中に演算子関連のノードを検出した場合、適切な関数ID解決ロジック（set_opfuncid 等）を適用してノードを更新する。 3. 特殊なセットアップを必要とせず、即座に走査を開始する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planner.c
  • src/backend/optimizer/util/clauses.c
  • src/backend/optimizer/util/plancat.c
  • src/backend/optimizer/util/predtest.c
  • src/backend/partitioning/partbounds.c
  • src/backend/statistics/extended_stats.c
  • src/backend/utils/cache/partcache.c
  • src/backend/utils/cache/relcache.c
  • **呼び出し先**:
  • fix_opfuncids_walker (直接の呼び出し先)

関連データ構造

  • Node: 走査対象の汎用ノード構造。
  • OpExpr, DistinctExpr, NullIfExpr, ScalarArrayOpExpr: 関数IDの解決対象となる演算子ノード。

失敗モードと境界条件

  • 走査中にスタックの深さが制限を超えた場合、PostgreSQLのスタックチェック機構によりエラーが発生する可能性がある。
  • 演算子IDに対応する関数がシステムカタログ上で見つからない場合、set_opfuncid 等の内部処理でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的な走査ロジックは fix_opfuncids_walker に依存している。
  • 新しい演算子ノード型を追加した場合は、fix_opfuncids_walker 側の switch 文を更新する必要がある。
