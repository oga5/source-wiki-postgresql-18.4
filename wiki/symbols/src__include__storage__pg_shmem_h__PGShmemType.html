PGShmemType

定義

  • 種別: enum
  • 定義箇所: src/include/storage/pg_shmem.h:60

目的

PostgreSQL がサポートする共有メモリのプラットフォーム固有実装を識別するための列挙型です。

シグネチャまたは形状

typedef enum
{
	SHMEM_TYPE_WINDOWS,
	SHMEM_TYPE_SYSV,
	SHMEM_TYPE_MMAP,
} PGShmemType;

入力と出力

  • 入力: なし（列挙定数）
  • 出力: 共有メモリの作成や管理を行う際の、実装方式の指定値として使用されます。

前提条件と不変条件

  • 各列挙値は、実行環境の OS がサポートする共有メモリのメカニズムに対応しています。
  • 実行時に選択された shared_memory_type GUC 変数の値と対応します。

アルゴリズムまたは振る舞い

  • SHMEM_TYPE_WINDOWS: Windows 環境での共有メモリ実装を指定します。
  • SHMEM_TYPE_SYSV: System V 共有メモリ実装を指定します。
  • SHMEM_TYPE_MMAP: メモリマップドファイルによる共有メモリ実装を指定します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース抜粋からは、具体的な呼び出し元は特定できません。
  • 呼び出し先: なし（データ型定義のため）。

関連データ構造

  • PGShmemHeader: 共有メモリセグメントのメタデータを保持する構造体。
  • HugePagesType: Huge Pages の状態を管理する列挙型。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 本シンボルは、プラットフォーム抽象化レイヤーの基盤です。新しい共有メモリ実装を追加する場合は、この列挙型に値を追加し、src/backend/storage/ipc/pg_shmem.c 内の対応するロジックを更新する必要があります。
