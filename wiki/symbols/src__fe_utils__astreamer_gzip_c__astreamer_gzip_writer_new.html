astreamer_gzip_writer_new

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_gzip.c:98

目的

gzip圧縮を行うための astreamer インターフェースを初期化し、圧縮書き込み器（writer）を生成する。

シグネチャまたは形状

astreamer *astreamer_gzip_writer_new(char *pathname, FILE *file, pg_compress_specification *compress)

入力と出力

  • **入力**:
  • pathname: ファイルパス（file が NULL の場合に使用）。
  • file: 出力先のファイルハンドル（指定がある場合）。
  • compress: 圧縮レベルを含む設定構造体。
  • **出力**: 初期化された astreamer 構造体へのポインタ。

前提条件と不変条件

  • HAVE_LIBZ が定義されている環境で動作する。
  • file が提供された場合、内部で dup() を実行してファイル記述子を複製し、元の FILE ハンドルが gzclose によって閉じられないように保護する。

アルゴリズムまたは振る舞い

1. astreamer_gzip_writer 構造体をメモリ確保し、メソッドテーブル astreamer_gzip_writer_ops を割り当てる。 2. file が NULL なら gzopen でパスからファイルを開き、指定があれば fileno(file) を dup() した上で gzdopen を使用して開く。 3. gzsetparams を呼び出し、compress->level に基づいて圧縮レベルを設定する。 4. 失敗時は pg_fatal を呼び出し、プロセスを終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/pg_basebackup.c:1197 での呼び出しが確認されている。
  • **呼び出し先**: palloc0, pstrdup, gzopen, dup, fileno, gzdopen, gzsetparams, pg_fatal, get_gz_error。

関連データ構造

  • astreamer_gzip_writer: 圧縮器の状態を保持する構造体。
  • astreamer: 基底となるストリーミングインターフェース。
  • pg_compress_specification: 圧縮設定を保持する構造体。

失敗モードと境界条件

  • HAVE_LIBZ が未定義の場合、呼び出しは致命的エラーとなる。
  • dup() に失敗した場合、または gzopen/gzdopen によるファイルオープンに失敗した場合、pg_fatal により異常終了する。
  • gzsetparams が Z_OK を返さない場合、エラーメッセージを出力して異常終了する。

保守担当者向けメモ

  • 本関数は gzFile を直接管理するため、他の圧縮モジュール（LZ4/Zstd）と異なり、圧縮結果を別の astreamer に渡すことはできない。
  • dup() の使用は、gzclose が内部で close() を実行してしまう仕様から呼び出し元の FILE を守るための必須措置である。
