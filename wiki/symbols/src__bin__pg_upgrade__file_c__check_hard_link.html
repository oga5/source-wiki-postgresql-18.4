check_hard_link

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/file.c:436

目的

旧データディレクトリと新データディレクトリの間でハードリンクが作成可能かどうかを検証し、ファイルシステムが同一であることを確認する。

シグネチャまたは形状

void check_hard_link(transferMode transfer_mode)

入力と出力

  • **入力**:
  • transfer_mode: transferMode 型。リンク操作を要求しているモード（TRANSFER_MODE_LINK または TRANSFER_MODE_SWAP）を識別するために使用される。
  • **出力**: なし（成功時は正常終了）。失敗時は pg_fatal によりプロセスを終了する。

前提条件と不変条件

  • old_cluster.pgdata および new_cluster.pgdata が初期化されていること。
  • 呼び出し元は、ハードリンクによるデータ移行を意図していること。

アルゴリズムまたは振る舞い

1. 旧データディレクトリ内の PG_VERSION ファイルをソースとして指定する。 2. 新データディレクトリ内に PG_VERSION.linktest という一時ファイルパスを生成する。 3. 既存の PG_VERSION.linktest があれば削除を試みる。 4. link() システムコールを使用して、ソースから一時ファイルへのハードリンク作成を試みる。 5. link() が失敗した場合、transfer_mode に応じたエラーメッセージを伴い pg_fatal を呼び出す。 6. 成功した場合は、作成した一時ファイルを削除して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/check.c:729 (TRANSFER_MODE_LINK を指定)
  • src/bin/pg_upgrade/check.c:742 (TRANSFER_MODE_SWAP を指定)
  • **呼び出し先**:
  • link() (システムコール)
  • unlink() (システムコール)
  • pg_fatal() (エラーハンドリング)

関連データ構造

  • transferMode (列挙型: 転送モードの定義)
  • old_cluster / new_cluster (グローバルなクラスタ情報構造体)

失敗モードと境界条件

  • **ファイルシステム境界**: 旧データディレクトリと新データディレクトリが異なるファイルシステム上にある場合、link() が失敗し、pg_fatal により移行が中断される。
  • **権限不足**: プロセスが新データディレクトリへの書き込み権限を持たない場合、link() が失敗する。
  • **一時ファイルの競合**: PG_VERSION.linktest が既に存在し、削除できない場合に失敗する可能性がある。

保守担当者向けメモ

  • 本関数は診断目的のテストであり、実際のデータ移行処理ではない。
  • pg_fatal を使用しているため、この関数が失敗すると pg_upgrade 全体の処理が即座に停止する。
  • link() の戻り値が負であることのみをチェックしており、具体的なエラーコード（errno）の詳細は pg_fatal 内の %m によって出力される。
