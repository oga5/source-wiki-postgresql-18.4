try_nestloop_path

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/joinpath.c:830

目的

2 つの入力パス（outer_path, inner_path）からネステッドループ結合パスを生成し、コスト評価を経て有効であれば結合リレーション（joinrel）のパスリストに追加する。

シグネチャまたは形状

static void try_nestloop_path(PlannerInfo *root, RelOptInfo *joinrel, Path *outer_path, Path *inner_path, List *pathkeys, JoinType jointype, JoinPathExtraData *extra);

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • joinrel: 結合対象のリレーション情報。
  • outer_path, inner_path: 結合する左右の入力パス。
  • pathkeys: 結合結果のソート順序。
  • jointype: 結合の種類（INNER, LEFT, SEMI 等）。
  • extra: 結合に必要なメタデータ（restrictlist, sjinfo 等）。
  • **出力**: なし（成功時は joinrel->pathlist を更新）。

前提条件と不変条件

  • 入力パスのパラメータ化が結合条件に適合していること。
  • 外部結合（Outer Join）において、入力パスがその外部結合自体をパラメータとして使用していないこと。
  • inner_path が outer_path の親リレーションによってパラメータ化されている場合、再パラメータ化が可能であること。

アルゴリズムまたは振る舞い

1. **パラメータ化の検証**: calc_nestloop_required_outer を使用し、結合に必要なパラメータを算出。スター型スキーマ結合の例外を除き、無効なパラメータ化であれば終了。 2. **安全性の確認**: have_unsafe_outer_join_ref により、外部結合参照の安全性を確認。 3. **再パラメータ化の確認**: inner_path が outer_path の親でパラメータ化されている場合、path_is_reparameterizable_by_child で変換可能性をチェック。 4. **コスト評価と選別**: initial_cost_nestloop で概算コストを算出し、add_path_precheck で既存パスと比較。 5. **パスの生成と登録**: 評価を通過した場合、create_nestloop_path でパスを作成し、add_path で joinrel に登録。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/joinpath.c 内の add_paths_to_joinrel 等から複数回呼び出される。
  • **呼び出し先**: calc_nestloop_required_outer, allow_star_schema_join, have_unsafe_outer_join_ref, PATH_PARAM_BY_PARENT, path_is_reparameterizable_by_child, initial_cost_nestloop, add_path_precheck, create_nestloop_path, add_path。

関連データ構造

  • RelOptInfo: 結合対象リレーション。
  • Path: 結合の入力および生成されるパス。
  • JoinPathExtraData: 結合処理に必要な補助情報。
  • JoinCostWorkspace: コスト計算結果を保持する作業領域。

失敗モードと境界条件

  • 外部結合の制約違反や、パラメータ化の不整合が発生した場合は早期リターンする。
  • add_path_precheck により、既存のパスよりコストが高いと判断された場合はパスの生成を行わない。

保守担当者向けメモ

  • 本関数は add_paths_to_joinrel における主要なパス生成ロジックの一部である。
  • メモリリークを防ぐため、早期リターン時には required_outer の解放を確実に行うこと。
  • create_plan() 時の再パラメータ化処理との整合性を維持するため、PATH_PARAM_BY_PARENT 等のチェックロジックを変更する際は注意が必要。
