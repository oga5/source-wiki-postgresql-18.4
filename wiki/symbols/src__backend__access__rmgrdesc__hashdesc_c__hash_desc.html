hash_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/hashdesc.c:19

目的

PostgreSQLのハッシュインデックスに関連するWAL（Write-Ahead Log）レコードの内容を解析し、人間が読める形式の文字列として出力します。主に pg_waldump 等のツールによるデバッグやリカバリ監視を目的としています。

シグネチャまたは形状

void hash_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 解析結果を書き込むための StringInfo バッファ。
  • record: 解析対象のWALレコードを保持する XLogReaderState 構造体。
  • **出力**: なし（buf に文字列が追記されます）。

前提条件と不変条件

  • record は有効なWALレコードを指している必要があります。
  • XLogRecGetInfo(record) で取得した情報から XLR_INFO_MASK を除外した値が、ハッシュインデックスの有効なWAL操作タイプと一致する必要があります。

アルゴリズムまたは振る舞い

1. XLogRecGetData を使用してレコードの生データを取得します。 2. XLogRecGetInfo からフラグを除去し、操作タイプ（info）を特定します。 3. switch 文により操作タイプを判定し、対応する構造体（例: xl_hash_insert）にキャストします。 4. キャストした構造体の各フィールドを appendStringInfo を用いて buf に整形して書き出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h で定義されているリソースマネージャ（RM_HASH_ID）の記述子ルーチンとして、WAL読み込み基盤から呼び出されます。
  • **呼び出し先**: XLogRecGetData、XLogRecGetInfo、appendStringInfo（PostgreSQL内部API）。

関連データ構造

  • XLogReaderState: WALレコードの読み込み状態を管理する構造体。
  • StringInfo: 文字列バッファ。
  • xl_hash_* 各種構造体: ハッシュインデックスの各WAL操作に対応するデータ構造（access/hash_xlog.h で定義）。

失敗モードと境界条件

  • 未知の info 値が渡された場合、何も出力されずに終了します。
  • レコードデータが期待される構造体サイズと一致しない場合、キャスト後のフィールド参照で不正な値が出力される可能性があります。

保守担当者向けメモ

  • 本関数は表示専用であり、リカバリ等のロジック実行には関与しません。
  • 新しいハッシュインデックスのWAL操作を追加した際は、必ず hash_desc および hash_identify の両方にケースを追加してください。
