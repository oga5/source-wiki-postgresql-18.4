pg_popcount32_fast

定義

  • src/port/pg_bitutils.c:219

目的

ハードウェア命令を利用して、32ビット整数内の「1」のビット数（ポピュレーションカウント）を高速に算出する。

シグネチャまたは形状

static inline int pg_popcount32_fast(uint32 word)

入力と出力

  • **入力**: uint32 word（カウント対象の32ビット整数）
  • **出力**: int（word 内の1のビット数）

前提条件と不変条件

  • 本関数は、実行環境が POPCNT 命令をサポートしていることを前提として呼び出される必要がある。
  • コンパイル環境に応じて、MSVCの組み込み関数またはGCC/Clang互換のインラインアセンブラが選択される。

アルゴリズムまたは振る舞い

1. **MSVC環境 (_MSC_VER)**: __popcnt 組み込み関数を呼び出し、結果を返す。 2. **その他環境**: popcntl 命令を含むインラインアセンブラを実行する。

  • word を入力とし、res に結果を格納する。
  • cc（条件コード）レジスタが変更されることを明示している。

    • 3. 算出された結果を int 型にキャストして返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/port/pg_bitutils.c 内の初期化処理（pg_popcount32 関数ポインタへの代入）。
  • src/port/pg_bitutils.c 内のバッファ走査処理（pg_popcount_fast 等）。
  • **呼び出し先**:
  • ハードウェア命令（__popcnt または popcntl）。

関連データ構造

  • なし。

失敗モードと境界条件

  • CPUが POPCNT 命令をサポートしていない環境で本関数を直接実行した場合、不正命令例外が発生する可能性がある（そのため、通常は pg_popcount_available による事前チェックを経て呼び出される）。

保守担当者向けメモ

  • 本関数は static inline で定義されており、コンパイル時の環境（_MSC_VER の有無）によって実装が切り替わる。
  • インラインアセンブラ部分は、ターゲットアーキテクチャが x86_64 であることを前提としている。
