astreamer_tar_terminator_new

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_tar.c:497

目的

tarアーカイブのストリーム処理パイプラインにおいて、アーカイブの末尾に不足しがちな2ブロックのNULバイトを付与し、整合性を確保するための終端処理用ストリーマーインスタンスを作成する。

シグネチャまたは形状

astreamer *astreamer_tar_terminator_new(astreamer *next)

入力と出力

  • **入力**: next (後続の処理を行う astreamer 構造体へのポインタ)
  • **出力**: 初期化された astreamer 構造体へのポインタ。失敗時は palloc0 の挙動に従いメモリ不足で終了する。

前提条件と不変条件

  • next は有効な astreamer インスタンスであること。
  • 返されるストリーマーは、astreamer_tar_terminator_ops を操作関数として保持する。

アルゴリズムまたは振る舞い

1. palloc0 を使用して astreamer 構造体分のメモリをゼロ初期化して確保する。 2. 確保した構造体の bbs_ops に astreamer_tar_terminator_ops を割り当てる。 3. 引数で渡された next を構造体の bbs_next に格納し、パイプラインを接続する。 4. 初期化済みのストリーマーポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/pg_basebackup.c:1259 にて呼び出されている。
  • **呼び出し先**: palloc0（メモリ確保）。

関連データ構造

  • astreamer: ストリーム処理の基盤となる構造体。
  • astreamer_ops: ストリーマーの操作関数テーブル（本関数では astreamer_tar_terminator_ops を使用）。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させ、プロセスが終了する。

保守担当者向けメモ

  • 本関数は astreamer インターフェースの具象実装の一つである。
  • 終端処理の具体的なロジック（NULブロックの注入）は、astreamer_tar_terminator_ops に定義された finalize 関数等で実装されている。
