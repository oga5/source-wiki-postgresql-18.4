pairingheap_allocate

定義

  • 種別: function
  • 定義箇所: src/backend/lib/pairingheap.c:41

目的

新しいペアリングヒープ（pairingheap）構造体を動的に確保し、初期化します。

シグネチャまたは形状

pairingheap *pairingheap_allocate(pairingheap_comparator compare, void *arg)

入力と出力

  • **入力**:
  • compare: ヒープ内のノード順序を決定するための比較関数。
  • arg: 比較関数に渡される任意の補助データ（コンテキスト）。
  • **出力**:
  • 初期化済みの pairingheap 構造体へのポインタ。

前提条件と不変条件

  • palloc を使用するため、PostgreSQL のメモリコンテキストが有効である必要があります。
  • 戻り値の ph_root は NULL で初期化され、空のヒープとして作成されます。

アルゴリズムまたは振る舞い

1. palloc を用いて pairingheap 構造体分のメモリを確保します。 2. 引数として受け取った比較関数 compare と補助データ arg を構造体に格納します。 3. ヒープのルートノード ph_root を NULL に設定します。 4. 初期化されたヒープ構造体へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistscan.c
  • src/backend/access/spgist/spgscan.c
  • src/backend/executor/nodeIndexscan.c
  • src/backend/replication/logical/reorderbuffer.c
  • **呼び出し先**:
  • palloc (PostgreSQL メモリ管理関数)

関連データ構造

  • pairingheap: ヒープ全体を管理する構造体。
  • pairingheap_comparator: ノード比較のための関数ポインタ型。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させ、処理が中断されます。

保守担当者向けメモ

  • 本関数で確保されたメモリは、pairingheap_free を呼び出すことで解放可能です。
  • ヒープ内の各ノードのメモリ管理は呼び出し側に委ねられており、pairingheap_free はノード自体を解放しません。
