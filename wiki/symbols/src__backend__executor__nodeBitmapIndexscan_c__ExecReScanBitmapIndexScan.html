ExecReScanBitmapIndexScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapIndexscan.c:130

目的

BitmapIndexScan ノードのインデックススキャンをリセットし、ランタイムキーを再評価してスキャンを再開可能にする。

シグネチャまたは形状

void ExecReScanBitmapIndexScan(BitmapIndexScanState *node)

入力と出力

  • **入力**: BitmapIndexScanState *node （スキャン状態を保持するノード構造体）
  • **出力**: なし（副作用としてノード内の状態が更新される）

前提条件と不変条件

  • node->biss_RuntimeContext が存在する場合、メモリリークを防ぐために ResetExprContext が呼び出される。
  • すべてのランタイムキーは、この関数が呼び出されるたびに再計算されることを前提としている。

アルゴリズムまたは振る舞い

1. **コンテキストのリセット**: biss_RuntimeContext をリセットし、メモリを解放する。 2. **ランタイムキーの再評価**:

  • biss_NumRuntimeKeys が存在する場合、ExecIndexEvalRuntimeKeys を実行して新しいキー値を計算する。
  • biss_NumArrayKeys が存在する場合、ExecIndexEvalArrayKeys を実行してキーを更新し、その結果を biss_RuntimeKeysReady に格納する。存在しない場合は true を設定する。

    • 3. **スキャンのリセット**: biss_RuntimeKeysReady が真であれば、index_rescan を呼び出し、インデックススキャン記述子を新しいキーで初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:190
  • **呼び出し先**: ResetExprContext, ExecIndexEvalRuntimeKeys, ExecIndexEvalArrayKeys, index_rescan

関連データ構造

  • BitmapIndexScanState: スキャン状態を管理する構造体。
  • ExprContext: ランタイムキー評価用のコンテキスト。

失敗モードと境界条件

  • biss_NumArrayKeys が存在し、ExecIndexEvalArrayKeys が空の配列キーを返した場合、biss_RuntimeKeysReady は false となり、index_rescan はスキップされる。

保守担当者向けメモ

  • 本関数は ExecProcNode インターフェースの一部ではない。
  • 並列クエリ実行時には、統計情報の共有状態に注意が必要である。
