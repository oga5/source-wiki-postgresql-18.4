check_prepare_conn

定義

  • 種別: function
  • 定義箇所: src/test/examples/testlibpq4.c:24

目的

データベース接続の確立状況を検証し、セキュリティ確保のために search_path を空に設定する初期化関数。

シグネチャまたは形状

static void check_prepare_conn(PGconn *conn, const char *dbName)

入力と出力

  • 入力:
  • conn: 検証対象の PGconn オブジェクトへのポインタ。
  • dbName: 接続先データベース名（関数内では未使用）。
  • 出力: なし（戻り値は void）。接続失敗時や設定失敗時は exit(1) によりプロセスを終了する。

前提条件と不変条件

  • conn は PQsetdb 等で初期化済みであること。
  • 接続が確立されていない場合、または search_path の設定に失敗した場合はプログラムが終了する。

アルゴリズムまたは振る舞い

1. PQstatus(conn) を呼び出し、接続状態が CONNECTION_OK であるかを確認する。失敗時はエラーメッセージを出力して終了する。 2. PQexec を使用して SELECT pg_catalog.set_config('search_path', '', false) を実行し、検索パスを空に設定する。 3. PQresultStatus で実行結果が PGRES_TUPLES_OK であるかを確認する。失敗時はエラーメッセージを出力し、PQclear で結果オブジェクトを解放して終了する。 4. 正常終了時に PQclear(res) を呼び出し、結果オブジェクトを解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/test/examples/testlibpq4.c 内の main 関数（conn1 および conn2 に対して呼び出し）。
  • 呼び出し先:
  • PQstatus, PQerrorMessage, PQexec, PQresultStatus, PQclear（すべて libpq ライブラリ関数）。

関連データ構造

  • PGconn: PostgreSQL 接続ハンドル。
  • PGresult: クエリ実行結果を保持する構造体。

失敗モードと境界条件

  • 接続が確立されていない場合、stderr にエラーを出力して終了する。
  • search_path の設定クエリが失敗した場合、stderr にエラーを出力し、メモリリークを防ぐために PQclear を実行した上で終了する。

保守担当者向けメモ

  • 本関数は libpq の古い API を使用するサンプルの一部である。
  • dbName 引数は現在の実装では使用されていない。
  • 異常終了時に exit(1) を呼び出すため、呼び出し側でリソース（他の接続など）の解放が必要な場合は注意が必要である。
