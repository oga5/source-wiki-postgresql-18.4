internalPQconninfoOption

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/fe-connect.c:178

目的

libpq の接続パラメータを定義し、環境変数やデフォルト値、および PGconn 構造体へのマッピング情報を保持するための内部構造体です。

シグネチャまたは形状

typedef struct _internalPQconninfoOption
{
    char       *keyword;        /* オプションのキーワード */
    char       *envvar;         /* フォールバック用環境変数名 */
    char       *compiled;       /* コンパイル時のデフォルト値 */
    char       *val;            /* 現在の値、または NULL */
    char       *label;          /* 接続ダイアログ用ラベル */
    char       *dispchar;       /* 表示形式（"":通常, "*":パスワード, "D":デバッグ） */
    int         dispsize;       /* ダイアログ用フィールドサイズ */
    off_t       connofs;        /* PGconn 構造体内のオフセット（該当なしの場合は -1） */
} internalPQconninfoOption;

入力と出力

  • 本構造体自体はデータ保持用であり、関数ではありません。
  • libpq-fe.h で定義される PQconninfoOption とメモリレイアウトが互換である必要があり、memcpy によるデータ転送が行われます。

前提条件と不変条件

  • connofs フィールドより上のメンバ構成は、libpq-fe.h 内の PQconninfoOption と完全に同期している必要があります。
  • connofs は PGconn 構造体内のメンバへのオフセットを指し、対応するメンバがない場合は -1 となります。

アルゴリズムまたは振る舞い

  • PQconninfoOptions 静的配列の要素として使用され、接続パラメータの初期化、環境変数からの値解決、および接続文字列のパース処理において参照されます。
  • dispchar フィールドにより、クライアント側の接続ダイアログにおける入力フィールドの表示制御（パスワードの隠蔽など）を決定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-connect.c 内の接続処理関数群（PQconndefaults 等）がこの構造体の配列を参照します。
  • 呼び出し先: 本構造体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • PQconninfoOption: libpq-fe.h で定義される公開用構造体。本構造体とメモリレイアウトが共有されます。
  • PGconn: 接続状態を保持する主要構造体。connofs を通じて本構造体から参照されます。
  • PQconninfoOptions: 本構造体のインスタンスを保持する静的配列。

失敗モードと境界条件

  • connofs が -1 の場合、そのオプションは PGconn 構造体に直接マッピングされません。
  • memcpy を使用して PQconninfoOption との間でデータをコピーするため、構造体のパディングやアライメントが環境間で一致しない場合、未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • internalPQconninfoOption のメンバ順序を変更する場合は、必ず libpq-fe.h の PQconninfoOption との整合性を確認してください。
  • connofs の値は offsetof() マクロを使用して計算されることが想定されています。
