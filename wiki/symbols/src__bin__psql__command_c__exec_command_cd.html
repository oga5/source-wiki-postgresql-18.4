exec_command_cd

定義

  • 種別: function
  • 定義箇所: src/bin/psql/command.c:690

目的

psql の \cd コマンドを実行し、カレントワーキングディレクトリを変更する。

シグネチャまたは形状

static backslashResult exec_command_cd(PsqlScanState scan_state, bool active_branch, const char *cmd)

入力と出力

  • **入力**:
  • scan_state: コマンド引数を読み取るためのスキャン状態。
  • active_branch: 現在の条件付き実行ブロックがアクティブかどうかを示すフラグ。
  • cmd: 実行されたコマンド名。
  • **出力**:
  • backslashResult: 成功時は PSQL_CMD_SKIP_LINE、失敗時は PSQL_CMD_ERROR を返す。

前提条件と不変条件

  • active_branch が true の場合のみディレクトリ変更が試行される。
  • active_branch が false の場合は、引数を消費するのみでディレクトリ変更は行われない。

アルゴリズムまたは振る舞い

1. active_branch が true の場合:

  • psql_scan_slash_option を使用して引数（ディレクトリパス）を取得する。
  • 引数が指定されていない場合、ホームディレクトリを特定する（非Windows環境では HOME 環境変数または getpwuid を使用、Windows環境では / をデフォルトとする）。
  • chdir() を呼び出し、ディレクトリ変更を試みる。失敗した場合はエラーログを出力する。
  • 取得した引数メモリを解放する。

    • 2. active_branch が false の場合:

  • ignore_slash_options を呼び出し、引数を読み飛ばす。

    • 3. 最終的な実行結果を backslashResult 型で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c 内のコマンドディスパッチャ（exec_command 関数内での呼び出しが確認されている）。
  • **呼び出し先**: psql_scan_slash_option, getenv, getpwuid, geteuid, chdir, pg_log_error, ignore_slash_options, free。

関連データ構造

  • PsqlScanState: コマンド引数の解析状態を保持する構造体。
  • backslashResult: コマンド実行後のステータスを示す列挙型。

失敗モードと境界条件

  • ホームディレクトリの取得失敗（ユーザーが存在しない等）時にエラーログを出力し、コマンド実行を失敗させる。
  • chdir() が失敗した場合（権限不足やディレクトリ不存在等）にエラーログを出力し、コマンド実行を失敗させる。
  • Windows環境では cd 引数なしの挙動が他プラットフォームと異なり、ルートディレクトリへの移動として処理される。

保守担当者向けメモ

  • 本関数は active_branch の状態管理に依存しているため、条件付き実行ロジックを変更する際は注意が必要。
  • ホームディレクトリ取得ロジックは get_home_path() と整合性を保つ必要がある。
