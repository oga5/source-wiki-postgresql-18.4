DatumGetIntervalP

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/timestamp.h:39

目的

Datum 型として保持されている interval 型のポインタを、内部表現である Interval * へ安全にキャストして取得する。

シグネチャまたは形状

static inline Interval *DatumGetIntervalP(Datum X)

入力と出力

  • **入力**: Datum X（interval 型のデータを含む Datum）
  • **出力**: Interval *（interval 構造体へのポインタ）

前提条件と不変条件

  • 入力 Datum X は、interval 型のデータへのポインタを保持している必要がある。
  • 本関数は DatumGetPointer マクロを利用して変換を行うため、X が有効なメモリ領域を指していることが前提となる。

アルゴリズムまたは振る舞い

1. DatumGetPointer(X) を呼び出し、Datum から汎用ポインタ型へ変換する。 2. 変換されたポインタを Interval * 型にキャストして返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_interval.c
  • contrib/btree_gist/btree_time.c
  • contrib/btree_gist/btree_ts.c
  • src/backend/commands/variable.c
  • **呼び出し先**:
  • DatumGetPointer（マクロ）

関連データ構造

  • Interval: PostgreSQLの interval 型を表現する内部構造体。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • X が NULL ポインタを指す Datum である場合、戻り値も NULL となる。
  • Datum が interval 型以外のデータを指している場合、型安全性の保証はない（呼び出し側の責任）。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、オーバーヘッドを最小限に抑える設計となっている。
  • interval 型の操作において、Datum との相互変換を行う際の標準的なインターフェースであるため、変更時は他の DatumGet* 系関数との整合性に注意すること。
