EndCompressFileHandle

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_io.c:288

目的

圧縮ファイルハンドル（CompressFileHandle）に関連付けられたリソースを解放し、ストリームを終了させる。

シグネチャまたは形状

bool EndCompressFileHandle(CompressFileHandle *CFH)

入力と出力

  • **入力**: CFH - 終了対象の CompressFileHandle 構造体へのポインタ。
  • **出力**: bool - 正常終了した場合は true、それ以外は false を返す。

前提条件と不変条件

  • CFH は有効な CompressFileHandle 構造体を指している必要がある。
  • errno は本関数の開始時に 0 にリセットされる。

アルゴリズムまたは振る舞い

1. errno を 0 に初期化する。 2. CFH->private_data が存在する場合、CFH->close_func を呼び出してストリームのクローズ処理を実行し、その戻り値を ret に格納する。 3. free_keep_errno を呼び出し、CFH 自体のメモリを解放する（この際、errno は保持される）。 4. ret を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_archiver.c
  • src/bin/pg_dump/pg_backup_directory.c
  • **呼び出し先**:
  • CFH->close_func (関数ポインタ経由の呼び出し)
  • free_keep_errno

関連データ構造

  • CompressFileHandle: ファイル I/O を抽象化するための関数ポインタ群と状態を保持する構造体。

失敗モードと境界条件

  • CFH->private_data が NULL の場合、close_func は呼び出されず、ret は初期値の false のままメモリ解放が行われる。
  • close_func 内でエラーが発生した場合、そのエラー状態が ret に反映される。

保守担当者向けメモ

  • 本関数は free_keep_errno を使用しており、メモリ解放処理が errno を破壊しないよう設計されている。
  • 新しい圧縮アルゴリズムを追加する際は、対応する close_func が適切に実装されていることを確認すること。
