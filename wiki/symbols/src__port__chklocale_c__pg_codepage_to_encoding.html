pg_codepage_to_encoding

定義

  • 種別: function
  • 定義箇所: src/port/chklocale.c:266

目的

Windows のコードページ番号（UINT）を、PostgreSQL 内部で使用されるエンコーディング識別子（pg_enc）に変換します。

シグネチャまたは形状

int pg_codepage_to_encoding(UINT cp);

入力と出力

  • **入力**: cp (Windows のコードページ番号を表す UINT)
  • **出力**: 一致した場合は対応する pg_enc コード（整数）、一致しない場合は -1 を返します。

前提条件と不変条件

  • encoding_match_list がシステムエンコーディング名と PostgreSQL エンコーディングコードの対応テーブルとして存在している必要があります。

アルゴリズムまたは振る舞い

1. 入力されたコードページ番号 cp を "CP%u" という形式の文字列に変換します。 2. encoding_match_list を順次走査し、pg_strcasecmp を用いて大文字小文字を区別せずにシステムエンコーディング名との照合を行います。 3. 一致するものが見つかった場合、対応する pg_enc_code を返します。 4. テーブルの末尾まで一致するものがない場合、WARNING レベルのログを出力し、-1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/error/elog.c:2489 で GetACP() の結果を引数として呼び出されています。
  • **呼び出し先**: pg_strcasecmp（文字列比較用）、ereport（警告出力用）。

関連データ構造

  • encoding_match_list: システムのエンコーディング名と PostgreSQL のエンコーディングコードを保持する静的テーブル。

失敗モードと境界条件

  • 指定されたコードページに対応するエントリが encoding_match_list に存在しない場合、関数は -1 を返し、警告ログを出力します。

保守担当者向けメモ

  • 新しいコードページや別名を追加する必要がある場合は、encoding_match_list にエントリを追加してください。
  • 本関数は Windows 環境での使用を想定しています。
