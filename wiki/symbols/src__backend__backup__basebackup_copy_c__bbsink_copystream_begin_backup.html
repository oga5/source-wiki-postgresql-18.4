bbsink_copystream_begin_backup

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_copy.c:125

目的

バックアップ開始時のメタデータ送信および、以降のアーカイブ・マニフェスト転送用COPYストリームの初期化を行う。

シグネチャまたは形状

static void bbsink_copystream_begin_backup(bbsink *sink)

入力と出力

  • 入力: bbsink *sink （bbsink_copystream 構造体としてキャストして使用）
  • 出力: なし（副作用としてクライアントへメッセージを送信）

前提条件と不変条件

  • sink は bbsink_copystream 型として初期化されていること。
  • sink->bbs_state に有効なバックアップ状態（XLog位置やテーブルスペースリスト）が格納されていること。

アルゴリズムまたは振る舞い

1. **バッファの確保**: bbs_buffer_length に MAXIMUM_ALIGNOF を加えた領域を確保する。 2. **アライメント調整**: 呼び出し元が期待するアライメントを維持しつつ、先頭にタイプバイト（'d'）を挿入可能な領域を確保する。 3. **メタデータ送信**:

  • SendXlogRecPtrResult でバックアップ開始位置を送信。
  • SendTablespaceList でテーブルスペースリストを送信。
  • pq_puttextmessage で CommandComplete ("SELECT") を送信。

    • 4. **ストリーム開始**: SendCopyOutResponse を呼び出し、COPY OUTモードへ移行する。

呼び出し元と呼び出し先

  • 呼び出し元: bbsink_copystream_ops 構造体を通じて、バックアップ処理のメインループから呼び出される。
  • 呼び出し先: SendXlogRecPtrResult, SendTablespaceList, pq_puttextmessage, SendCopyOutResponse。

関連データ構造

  • bbsink_copystream: 本関数が操作するメイン構造体。
  • bbsink_state: バックアップのメタデータ（開始位置、テーブルスペース等）を保持。

失敗モードと境界条件

  • メモリ不足時: palloc がエラーを発生させる。
  • 通信エラー時: pq_puttextmessage 等の内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • msgbuffer の先頭にタイプバイトを埋め込む際、アライメントを維持するために MAXIMUM_ALIGNOF を使用している。このパディングにより、データコピーのオーバーヘッドを回避している。
