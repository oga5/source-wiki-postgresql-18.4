tqueueReceiveSlot

定義

  • 種別: function
  • 定義箇所: src/backend/executor/tqueue.c:53

目的

並列クエリ実行において、実行エンジンから受け取ったタプルを共有メモリキュー（shm_mq）へ書き出し、他のバックエンドへ転送する。

シグネチャまたは形状

static bool tqueueReceiveSlot(TupleTableSlot *slot, DestReceiver *self)

入力と出力

  • **入力**:
  • slot: 送信対象のタプルを保持する TupleTableSlot。
  • self: TQueueDestReceiver 構造体へのポインタ（DestReceiver としてキャストされている）。
  • **出力**:
  • 成功時は true を返す。
  • 共有メモリキューがデタッチされた場合は false を返す。

前提条件と不変条件

  • self は TQueueDestReceiver 型として安全にキャスト可能であること。
  • slot から取得した MinimalTuple は、shm_mq_send 呼び出し後に必要に応じて解放される。

アルゴリズムまたは振る舞い

1. ExecFetchSlotMinimalTuple を呼び出し、スロットから MinimalTuple を抽出する。 2. shm_mq_send を使用し、共有メモリキューへタプルを書き込む。 3. 抽出したタプルが一時的なメモリ割り当てによるものであれば pfree で解放する。 4. shm_mq_send の結果を判定する:

  • SHM_MQ_DETACHED の場合、処理を中断し false を返す。
  • SHM_MQ_SUCCESS 以外の場合、ereport(ERROR) を発生させる。

    • 5. 正常終了時は true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（DestReceiver インターフェース経由で呼び出される）。
  • **呼び出し先**:
  • ExecFetchSlotMinimalTuple
  • shm_mq_send
  • pfree
  • ereport

関連データ構造

  • TQueueDestReceiver: 送信側の状態を保持する構造体。
  • TupleTableSlot: タプルを保持する標準的な実行用構造体。
  • shm_mq: 共有メモリキューの基盤構造体。

失敗モードと境界条件

  • **キューのデタッチ**: 受信側が終了している場合、SHM_MQ_DETACHED が返され、関数は安全に終了する。
  • **送信エラー**: キューへの書き込みが成功しなかった場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE エラーが発生する。

保守担当者向けメモ

  • 本関数は DestReceiver インターフェースの一部として登録される。
  • shm_mq_send の第5引数が false に設定されており、ブロッキング動作が指定されている。
  • タプルのシリアライズ形式は MinimalTuple に依存しているため、関連するデータ構造の変更時には注意が必要である。
