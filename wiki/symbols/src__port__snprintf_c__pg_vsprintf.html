pg_vsprintf

定義

  • 種別: function
  • 定義箇所: src/port/snprintf.c:213

目的

プラットフォーム依存の vsprintf を置き換え、PostgreSQL の要件（POSIX 準拠のフォーマット制御や %m 展開など）を満たすポータブルな文字列フォーマット出力機能を提供します。

シグネチャまたは形状

int pg_vsprintf(char *str, const char *fmt, va_list args);

入力と出力

  • **入力**:
  • str: フォーマットされた文字列を格納するバッファへのポインタ。
  • fmt: フォーマット指定文字列。
  • args: 可変長引数リスト (va_list)。
  • **出力**:
  • 成功時: 書き込まれた文字数（終端の \0 を除く）。
  • 失敗時: -1。

前提条件と不変条件

  • str は十分な長さを持つバッファを指している必要があります（本関数はバッファサイズ制限を行わないため、呼び出し側でオーバーランを防ぐ必要があります）。
  • 内部で PrintfTarget 構造体を使用して出力状態を管理します。

アルゴリズムまたは振る舞い

1. PrintfTarget 構造体を初期化し、出力先を str に設定します。 2. dopr() 関数を呼び出し、フォーマット文字列の解析とバッファへの書き込みを実行します。 3. 書き込み終了後、バッファの末尾にヌル終端文字 \0 を付与します。 4. dopr() がエラーを報告した場合は -1 を返し、成功した場合は書き込んだ文字数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port.h で vsprintf のマクロとして定義されており、システム全体から参照される可能性があります。
  • **呼び出し先**: dopr() を呼び出してフォーマット処理のバックエンドを実行します。

関連データ構造

  • PrintfTarget: 出力先バッファや現在の状態を保持する構造体。

失敗モードと境界条件

  • dopr() 内部でエラーが発生した場合、target.failed が true となり、本関数は -1 を返します。
  • バッファサイズ制限がないため、出力がバッファを超過した場合のメモリ破壊に対する保護は呼び出し側に依存します。

保守担当者向けメモ

  • 本関数は snprintf ではなく sprintf の挙動を模倣するため、バッファオーバーフローの危険性があります。可能な限り pg_vsnprintf の使用を推奨します。
  • dopr() は PostgreSQL の printf ファミリー共通のバックエンドであるため、修正時は他の pg_printf 系関数への影響を考慮してください。
