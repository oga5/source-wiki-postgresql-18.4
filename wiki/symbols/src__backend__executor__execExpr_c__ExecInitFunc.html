ExecInitFunc

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execExpr.c:2703

目的

式評価における関数呼び出し（FuncExpr）のための実行ステップ（ExprEvalStep）を初期化し、関数実行に必要なメタデータと引数評価ロジックを構築する。

シグネチャまたは形状

static void ExecInitFunc(ExprEvalStep *scratch, Expr *node, List *args, Oid funcid, Oid inputcollid, ExprState *state)

入力と出力

  • **入力**:
  • scratch: 初期化対象の ExprEvalStep 構造体。
  • node: 評価対象の式ノード。
  • args: 関数引数のリスト。
  • funcid: 呼び出す関数の OID。
  • inputcollid: 入力照合順序。
  • state: 親となる ExprState。
  • **出力**: なし（scratch 構造体のフィールドを直接更新する）。

前提条件と不変条件

  • 関数呼び出しに対する実行権限（ACL_EXECUTE）が検証済みであること。
  • 引数の数が FUNC_MAX_ARGS を超えていないこと。
  • 呼び出される関数が集合を返す関数（Set-valued function）ではないこと。

アルゴリズムまたは振る舞い

1. **権限確認**: object_aclcheck を実行し、失敗時はエラーを報告する。 2. **引数数チェック**: nargs が FUNC_MAX_ARGS を超える場合はエラーとする。 3. **メモリ確保**: FmgrInfo および FunctionCallInfo を palloc0 で確保する。 4. **関数情報設定**: fmgr_info を用いて関数情報を取得し、InitFunctionCallInfoData で呼び出し情報を初期化する。 5. **引数評価の準備**: 引数リストを走査し、定数の場合は値を直接設定し、それ以外は ExecInitExprRec を再帰的に呼び出して評価ステップを構築する。 6. **オペコードの決定**: 関数の厳密性（strictness）および統計追跡設定（pgstat_track_functions）に基づき、最適な EEOP_FUNCEXPR_* 系オペコードを scratch->opcode に割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execExpr.c 内の ExecInitExpr 等から呼び出される（詳細はソース内の ExecInitFunc 参照箇所を参照）。
  • **呼び出し先**: object_aclcheck, fmgr_info, InitFunctionCallInfoData, ExecInitExprRec 等。

関連データ構造

  • ExprEvalStep: 式評価の各ステップを保持する構造体。
  • FmgrInfo: 関数マネージャの呼び出し情報。
  • FunctionCallInfo: 関数呼び出し時の引数や状態を保持する構造体。

失敗モードと境界条件

  • **権限エラー**: 実行権限がない場合、aclcheck_error を呼び出す。
  • **引数過多**: 引数が FUNC_MAX_ARGS を超える場合、ERRCODE_TOO_MANY_ARGUMENTS でエラーとなる。
  • **集合関数**: 集合を返す関数が指定された場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラーとなる。

保守担当者向けメモ

  • 本関数は式評価の「コンパイル」フェーズを担当する。実行時のオーバーヘッドを最小化するため、引数の評価順序や厳密性に応じたオペコードの選択が重要である。
  • EEOP_FUNCEXPR 系のオペコードを変更・追加する場合は、execExprInterp.c のインタプリタ側の対応も必須となる。
