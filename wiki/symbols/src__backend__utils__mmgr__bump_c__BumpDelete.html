BumpDelete

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/bump.c:277

目的

バンプアロケータのメモリコンテキストを破棄し、関連するすべてのメモリブロックを解放する。

シグネチャまたは形状

void BumpDelete(MemoryContext context)

入力と出力

  • **入力**: context (破棄対象の MemoryContext)
  • **出力**: なし

前提条件と不変条件

  • 指定された context は BumpContext 型である必要がある。
  • 本関数を呼び出した後、context ポインタは無効となり、アクセスしてはならない。

アルゴリズムまたは振る舞い

1. BumpReset(context) を呼び出し、コンテキスト内の解放可能なすべての BumpBlock を解放する。 2. コンテキストヘッダーおよび保持ブロック（keeper block）自体を free() で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c: MCTX_BUMP_ID の delete_context メソッドとして登録されている。
  • **呼び出し先**:
  • BumpReset: コンテキスト内のブロック解放を実行。
  • free: コンテキストヘッダーのメモリ解放を実行。

関連データ構造

  • MemoryContext: メモリ管理コンテキストの抽象型。
  • BumpContext: バンプアロケータのメタデータ構造体。
  • BumpBlock: バンプアロケータが管理するメモリブロック単位。

失敗モードと境界条件

  • context が NULL の場合、標準的な free() の挙動に従い未定義の動作を引き起こす可能性がある。
  • BumpReset 内部で発生するエラーは本関数に伝播する。

保守担当者向けメモ

  • 本関数は MemoryContext インターフェースの一部として実装されており、直接呼び出すよりも MemoryContextDelete マクロ経由での使用が想定されている。
  • BumpReset を呼び出すことで、個別の pfree を必要とせずに一括解放を実現している。
