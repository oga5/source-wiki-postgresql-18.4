bbsink_forward_begin_backup

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_sink.c:23

目的

bbsink（バックアップシンク）チェーンにおいて、バックアップ開始処理を後続のシンクへ転送し、バッファの共有設定を行う。

シグネチャまたは形状

void bbsink_forward_begin_backup(bbsink *sink)

入力と出力

  • **入力**: sink (bbsink *) - 現在のシンクオブジェクト。
  • **出力**: なし。副作用として、後続シンクの初期化および現在のシンクのバッファポインタ更新が行われる。

前提条件と不変条件

  • sink->bbs_next が NULL であってはならない。
  • sink->bbs_state が NULL であってはならない。

アルゴリズムまたは振る舞い

1. sink->bbs_next（後続のシンク）に対して bbsink_begin_backup を呼び出し、現在のシンクの bbs_state と bbs_buffer_length を引き継ぐ。 2. 後続シンクのバッファ (sink->bbs_next->bbs_buffer) を、現在のシンクのバッファ (sink->bbs_buffer) として参照するように設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/basebackup_to_shell/basebackup_to_shell.c
  • src/backend/backup/basebackup_progress.c
  • src/backend/backup/basebackup_server.c
  • src/backend/backup/basebackup_throttle.c
  • **呼び出し先**:
  • bbsink_begin_backup（sink->bbs_next のメソッド）

関連データ構造

  • bbsink: バックアップデータの転送先を抽象化した構造体。

失敗モードと境界条件

  • sink->bbs_next が初期化されていない場合、アサーション違反により異常終了する。

保守担当者向けメモ

  • 本関数は転送専用のヘルパーである。独自のシンクを実装する際、begin_backup コールバックにこの関数を直接割り当てることが可能である。
  • バッファの共有が前提となっているため、後続シンクのバッファ管理状態に依存する。
