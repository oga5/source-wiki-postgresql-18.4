setNamespaceForMergeWhen

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_merge.c:51

目的

MERGE 文の各 WHEN 句の処理において、ターゲットテーブルおよびソースリレーションの可視性を、アクションの種類に応じて動的に切り替えるために使用されます。

シグネチャまたは形状

static void setNamespaceForMergeWhen(ParseState *pstate, MergeWhenClause *mergeWhenClause, Index targetRTI, Index sourceRTI)

入力と出力

  • pstate: 解析状態を保持する ParseState 構造体へのポインタ。
  • mergeWhenClause: 現在処理中の WHEN 句情報。
  • targetRTI: ターゲットリレーションのレンジテーブルインデックス。
  • sourceRTI: ソースリレーションのレンジテーブルインデックス。
  • 出力: なし（pstate->p_namespace 内の可視性フラグを直接更新する副作用を持つ）。

前提条件と不変条件

  • pstate は有効な解析状態であること。
  • targetRTI および sourceRTI は pstate->p_rtable 内の有効なエントリを指していること。
  • mergeWhenClause->matchKind に応じて、許可される commandType が以下の通り制約される：
  • MERGE_WHEN_MATCHED: UPDATE, DELETE, NOTHING
  • MERGE_WHEN_NOT_MATCHED_BY_SOURCE: UPDATE, DELETE, NOTHING
  • MERGE_WHEN_NOT_MATCHED_BY_TARGET: INSERT, NOTHING

アルゴリズムまたは振る舞い

1. rt_fetch を用いて、指定されたインデックスからターゲットおよびソースの RangeTblEntry を取得します。 2. mergeWhenClause->matchKind を判定し、以下の可視性設定を行います：

  • MERGE_WHEN_MATCHED: ターゲットとソースの両方を可視（true, true）に設定。
  • MERGE_WHEN_NOT_MATCHED_BY_SOURCE: ターゲットを可視、ソースを不可視（false, false）に設定。
  • MERGE_WHEN_NOT_MATCHED_BY_TARGET: ターゲットを不可視、ソースを可視に設定。

    • 3. setNamespaceVisibilityForRTE を呼び出し、名前空間内の各リレーションの可視性フラグを更新します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/parser/parse_merge.c 内の MERGE 文解析ループ（transformMergeStmt 内の WHEN 句処理箇所）。
  • 呼び出し先: setNamespaceVisibilityForRTE（名前空間内の可視性フラグ更新用）。

関連データ構造

  • ParseState: 解析状態を管理。
  • MergeWhenClause: MERGE 文の各アクション句を定義。
  • RangeTblEntry: リレーションの情報を保持。

失敗モードと境界条件

  • matchKind が想定外の値である場合、あるいは commandType が各 matchKind の制約に違反する場合、Assert により異常終了します。
  • targetRTI または sourceRTI が無効な場合、rt_fetch 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • setNamespaceVisibilityForRTE は名前空間内の ParseNamespaceItem を直接操作するため、呼び出し順序や状態の復元が必要な場合は注意してください。
  • MERGE 文特有の「アクションごとの可視性切り替え」を担う唯一の関数であるため、新しい WHEN 句のバリエーションを追加する際は、本関数の条件分岐を更新する必要があります。
