XmlTableSetNamespace

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xml.c:4787

目的

XMLTABLE 演算子の実行コンテキストに対し、XPath評価で使用する名前空間（接頭辞とURIのペア）を登録します。

シグネチャまたは形状

static void XmlTableSetNamespace(TableFuncScanState *state, const char *name, const char *uri)

入力と出力

  • state: TableFuncScanState 構造体へのポインタ。内部の XmlTableBuilderData を取得するために使用されます。
  • name: 登録する名前空間の接頭辞。
  • uri: 登録する名前空間のURI。
  • 戻り値: なし。成功時は内部状態を更新し、失敗時は ereport によりエラーを発生させます。

前提条件と不変条件

  • USE_LIBXML が定義されている環境でのみ動作します。
  • state から取得される XmlTableBuilderData 内の xpathcxt が有効である必要があります。

アルゴリズムまたは振る舞い

1. name が NULL の場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラーを報告します。 2. GetXmlTableBuilderPrivateData を呼び出し、state から XmlTableBuilderData を取得します。 3. pg_xmlCharStrndup を用いて name と uri を libxml2 形式の文字列に変換します。 4. xmlXPathRegisterNs を呼び出し、XPathコンテキストに名前空間を登録します。 5. 登録に失敗した場合は、xml_ereport を介して ERRCODE_INVALID_ARGUMENT_FOR_XQUERY エラーを発生させます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/xml.c 内の XmlTableRoutine 構造体の初期化箇所（.SetNamespace = XmlTableSetNamespace）から参照されています。
  • 呼び出し先: GetXmlTableBuilderPrivateData, pg_xmlCharStrndup, xmlXPathRegisterNs, xml_ereport。

関連データ構造

  • TableFuncScanState: 実行ノードの状態を保持する構造体。
  • XmlTableBuilderData: XMLTABLE の実行に必要なプライベートデータ（XPathコンテキスト等）を保持する構造体。

失敗モードと境界条件

  • name が NULL の場合: ERROR を発生させます。
  • xmlXPathRegisterNs が失敗した場合: ERRCODE_INVALID_ARGUMENT_FOR_XQUERY を伴う ERROR を発生させます。
  • USE_LIBXML が未定義の場合: NO_XML_SUPPORT() マクロによりエラーとなります。

保守担当者向けメモ

  • 本関数は libxml2 のグローバルな状態に依存するため、pg_xml_init 等による適切なエラーハンドリング環境下で呼び出されることを前提としています。
  • デフォルト名前空間（name が NULL のケース）のサポートが必要な場合は、libxml2 の仕様と照らし合わせて実装を検討してください。
