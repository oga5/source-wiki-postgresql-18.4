ScalarArrayOpExprHashTable

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/execExprInterp.c:225

目的

ScalarArrayOpExpr（配列に対する演算子式）の評価において、定数配列の要素を高速に検索するためのハッシュテーブル構造を保持する。

シグネチャまたは形状

typedef struct ScalarArrayOpExprHashTable
{
    saophash_hash *hashtab;              /* 基盤となるハッシュテーブル */
    struct ExprEvalStep *op;             /* 関連する式評価ステップ */
    FmgrInfo    hash_finfo;              /* ハッシュ関数のルックアップ情報 */
    FunctionCallInfoBaseData hash_fcinfo_data; /* 関数呼び出し引数データ */
} ScalarArrayOpExprHashTable;

入力と出力

  • 入力: 式評価実行時に ExprEvalStep から参照される。
  • 出力: ハッシュテーブルを用いた検索結果（ScalarArrayOpExprHashEntry 経由）。

前提条件と不変条件

  • lib/simplehash.h を使用して実装されている。
  • palloc0 を用いてメモリ確保される際、hash_fcinfo_data のサイズを考慮したオフセット計算が行われる。

アルゴリズムまたは振る舞い

  • 式評価エンジンにおいて、配列内の要素検索を線形探索からハッシュ検索へ最適化するために使用される。
  • ExecInterpExpr 等の実行コンテキスト内で、op->d.hashedscalararrayop.elements_tab を通じてアクセスされる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execExprInterp.c 内の式評価ロジック（4206, 4227, 4256, 4306 行付近）。
  • 呼び出し先: lib/simplehash.h が提供するハッシュテーブル操作関数。

関連データ構造

  • ExprEvalStep: この構造体を保持する親ステップ。
  • ScalarArrayOpExprHashEntry: ハッシュテーブルに格納される個々のエントリ。

失敗モードと境界条件

  • メモリ不足時: palloc0 によるメモリ確保失敗。
  • ハッシュ関数未定義時: hash_finfo の初期化不備による実行時エラー。

保守担当者向けメモ

  • 本構造体は src/include/executor/execExpr.h にも前方宣言が存在する。
  • hash_fcinfo_data は可変長データを含む可能性があるため、メモリ確保時には offsetof を使用したサイズ計算が必須である。
