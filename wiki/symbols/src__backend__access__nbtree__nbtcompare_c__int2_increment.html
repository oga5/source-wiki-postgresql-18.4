int2_increment

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtcompare.c:171

目的

B-tree インデックスのスキップスキャンにおいて、int16 型のキー値を 1 増加させるための補助関数です。

シグネチャまたは形状

static Datum int2_increment(Relation rel, Datum existing, bool *overflow)

入力と出力

  • **入力**:
  • rel: 対象のリレーション（本関数内では未使用）。
  • existing: 増加対象の int16 値（Datum 型）。
  • overflow: オーバーフロー状態を格納するフラグへのポインタ。
  • **出力**:
  • 戻り値: 増加後の int16 値（Datum 型）。オーバーフロー時は 0。
  • *overflow: 増加後の値が int16 の範囲を超える場合は true、それ以外は false。

前提条件と不変条件

  • existing は int16 型として解釈可能であること。
  • overflow ポインタは有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. existing を int16 型に変換します。 2. 値が PG_INT16_MAX と等しい場合：

  • *overflow を true に設定します。
  • 0 を返します（戻り値は未定義）。

    • 3. それ以外の場合：

  • *overflow を false に設定します。
  • 値に 1 を加算し、Datum 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtcompare.c 内で、SkipSupport 構造体の increment フィールドに本関数が登録されています。
  • **呼び出し先**: 直接的な関数呼び出しは行いませんが、DatumGetInt16 および Int16GetDatum マクロを使用します。

関連データ構造

  • SkipSupport: インデックススキップスキャン用の増減ロジックを保持する構造体。

失敗モードと境界条件

  • **オーバーフロー**: 入力値が PG_INT16_MAX の場合、増加処理は行われず overflow フラグが立てられます。

保守担当者向けメモ

  • 本関数は SkipSupport インターフェースの一部として提供されています。
  • rel 引数は現在の実装では使用されていませんが、将来的な拡張のためにシグネチャに含まれています。
