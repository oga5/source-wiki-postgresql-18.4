seg_out

定義

  • 種別: function
  • 定義箇所: contrib/seg/seg.c:123

目的

PostgreSQL の SEG 構造体（区間データ）を、人間が読み取り可能な文字列形式に変換して出力する。

シグネチャまたは形状

Datum seg_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_SEG_P(0) を介して取得される SEG 構造体へのポインタ。
  • **出力**: PG_RETURN_CSTRING を介して返される、区間を表現する C 文字列。

前提条件と不変条件

  • SEG 構造体は segdata.h で定義されている。
  • palloc(40) により 40 バイトのメモリが確保される。
  • 数値の書式化には補助関数 restore を使用し、有効桁数を維持する。

アルゴリズムまたは振る舞い

1. 40 バイトのメモリを確保する。 2. 境界値の型（l_ext）が <, >, ~ のいずれかであれば、その文字を先頭に出力する。 3. 下限と上限が等しく、かつ境界型も同一である場合、単一の点として restore を呼び出し出力する。 4. それ以外の場合、以下の形式で出力する：

  • 下限が存在すれば restore で出力し、スペースを挟む。
  • .. を出力する。
  • 上限が存在すれば、必要に応じて境界型を出力し、restore で上限値を出力する。

    • 5. 構築した文字列を Datum として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1(seg_out) により登録）。
  • **呼び出し先**: palloc、sprintf、restore。

関連データ構造

  • SEG 構造体（segdata.h で定義）

失敗モードと境界条件

  • 確保した 40 バイトを超える文字列が生成される可能性については、ソースコード内に明示的なチェックはない。

保守担当者向けメモ

  • restore 関数が有効桁数を保持するロジックを担っているため、出力形式を変更する場合は restore の仕様を確認すること。
  • seg_in と対になる関数であり、seg_in でパース可能な形式で出力されることが期待される。
