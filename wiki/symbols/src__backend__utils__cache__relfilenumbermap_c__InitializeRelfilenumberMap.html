InitializeRelfilenumberMap

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/relfilenumbermap.c:85

目的

(tablespace, relfilenumber) から relid を逆引きするためのハッシュテーブル RelfilenumberMapHash を初期化し、リレーションキャッシュの無効化イベントを監視するためのコールバックを登録します。

シグネチャまたは形状

static void InitializeRelfilenumberMap(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 RelfilenumberMapHash を初期化し、コールバックを登録する）

前提条件と不変条件

  • CacheMemoryContext が存在すること（存在しない場合は CreateCacheMemoryContext() により作成される）。
  • fmgr_info_cxt の実行中にメモリ不足等のエラーが発生した場合、ハッシュテーブルが部分的に初期化された状態にならないよう、ハッシュテーブルの作成は最後に行われる。

アルゴリズムまたは振る舞い

1. CacheMemoryContext の存在を確認し、必要に応じて作成する。 2. pg_class のインデックススキャン用スキャンキー（relfilenumber_skey）を初期化する。

  • Anum_pg_class_reltablespace と Anum_pg_class_relfilenode を対象に F_OIDEQ を設定。

    • 3. hash_create を呼び出し、RelfilenumberMapHash を作成する。

  • キーサイズ: RelfilenumberMapKey
  • エントリサイズ: RelfilenumberMapEntry
  • フラグ: HASH_ELEM | HASH_BLOBS | HASH_CONTEXT

    • 4. CacheRegisterRelcacheCallback を呼び出し、RelfilenumberMapInvalidateCallback を無効化イベントに登録する。

呼び出し元と呼び出し先

  • 呼び出し元: RelidByRelfilenumber（キャッシュの遅延初期化時に呼び出される）
  • 呼び出し先: CreateCacheMemoryContext, MemSet, fmgr_info_cxt, hash_create, CacheRegisterRelcacheCallback

関連データ構造

  • RelfilenumberMapHash: キャッシュの実体となるハッシュテーブル。
  • RelfilenumberMapKey: ハッシュテーブルの検索キー。
  • RelfilenumberMapEntry: ハッシュテーブルのエントリ構造体。
  • relfilenumber_skey: pg_class 検索用のスキャンキー。

失敗モードと境界条件

  • メモリ不足: fmgr_info_cxt や hash_create 内部でメモリ不足が発生した場合、PostgreSQL のエラー処理機構により中断される。

保守担当者向けメモ

  • 本関数は RelidByRelfilenumber からの初回呼び出し時にのみ実行される遅延初期化の仕組みをとっています。
  • RelfilenumberMapHash の作成順序は、エラー発生時の整合性を保つため意図的に最後に行われています。
