MVDependency

定義

  • 種別: struct
  • 定義箇所: src/include/statistics/statistics.h:50

目的

列間の関数従属性（Functional Dependency）を表現するためのデータ構造です。クエリプランナが、ある列の集合の値が別の列の値を決定する度合いを評価し、選択率（Selectivity）の推定精度を向上させるために使用されます。

シグネチャまたは形状

typedef struct MVDependency
{
	double		degree;
	AttrNumber	nattributes;
	AttrNumber	attributes[FLEXIBLE_ARRAY_MEMBER];
} MVDependency;

入力と出力

  • **入力**: 構造体生成時に degree（従属性の強さ）、nattributes（属性数）、および attributes（属性番号の配列）が設定されます。
  • **出力**: プランナが統計情報として本構造体を参照し、クエリ内の句に対する選択率を算出します。

前提条件と不変条件

  • degree は 0 から 1 の範囲の値をとります。
  • attributes は FLEXIBLE_ARRAY_MEMBER を使用した可変長配列であり、メモリ確保時には offsetof(MVDependency, attributes) を用いたサイズ計算が必要です。

アルゴリズムまたは振る舞い

  • 本構造体は、ANALYZE 処理によって構築された統計情報の一部として保持されます。
  • プランナは、クエリ内の句に含まれる列がこの構造体の attributes に含まれるかを確認し、degree を用いて選択率を補正します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/dependencies.c 内の各関数（dependency_is_fully_matched, find_strongest_dependency 等）が、統計情報の照合や検索のために本構造体を操作しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • MVDependencies: 複数の MVDependency を保持するコンテナ構造体です。

失敗モードと境界条件

  • 可変長配列であるため、メモリ確保時にサイズ計算を誤るとバッファオーバーランを引き起こす可能性があります。
  • nattributes が STATS_MAX_DIMENSIONS を超えるような不正なデータがロードされた場合、後続の処理で予期せぬ動作をする可能性があります。

保守担当者向けメモ

  • 本構造体はシリアライズ可能な形式であるため、フィールドの追加や変更を行う際は、マジックナンバーやバージョン管理との整合性に注意してください。
  • attributes 配列へのアクセスは、nattributes の値に基づいた境界チェックを確実に行ってください。
