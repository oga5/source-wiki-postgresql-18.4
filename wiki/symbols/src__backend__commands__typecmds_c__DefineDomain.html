DefineDomain

定義

  • 種別: function
  • 定義箇所: src/backend/commands/typecmds.c:696

目的

SQLの CREATE DOMAIN コマンドを処理し、既存のデータ型に基づいた新しいドメイン型をシステムカタログに登録します。

シグネチャまたは形状

ObjectAddress DefineDomain(ParseState *pstate, CreateDomainStmt *stmt)

入力と出力

  • **入力**:
  • pstate: パーサーの状態。
  • stmt: CREATE DOMAIN 文の構文木。
  • **出力**:
  • ObjectAddress: 作成されたドメイン型のオブジェクトアドレス。

前提条件と不変条件

  • 対象スキーマに対する CREATE 権限が必要です。
  • ベースとなる型に対する USAGE 権限が必要です。
  • ベース型は BASE, COMPOSITE, DOMAIN, ENUM, RANGE, MULTIRANGE のいずれかである必要があります（擬似型は不可）。
  • 既存の型名と衝突する場合、それが自動生成された配列型であればリネームを試みますが、それ以外はエラーとなります。

アルゴリズムまたは振る舞い

1. **名前解決**: 指定された名前とスキーマを特定し、権限を確認します。 2. **型名衝突チェック**: 同名の型が存在しないか確認し、必要に応じて既存の配列型を退避させます。 3. **ベース型検証**: 指定されたベース型のメタデータを取得し、ドメインの基底として適切か検証します。 4. **プロパティ継承**: ベース型の属性（アライメント、ストレージ、I/O関数等）を継承します。 5. **照合順序の決定**: COLLATE 句があれば適用し、未指定の場合はベース型の照合順序を継承します。 6. **制約処理**: stmt->constraints リストを走査し、ドメイン制約を個別に処理します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c (SQLコマンド実行時)
  • **呼び出し先**:
  • QualifiedNameGetCreationNamespace, object_aclcheck, GetSysCacheOid2, moveArrayTypeName, typenameType, get_collation_oid 等のユーティリティ関数。

関連データ構造

  • CreateDomainStmt: SQL構文木。
  • Form_pg_type: pg_type カタログの行構造。
  • ObjectAddress: 作成されたオブジェクトの識別子。

失敗モードと境界条件

  • **権限不足**: スキーマ作成権限や型使用権限がない場合にエラーを発生させます。
  • **型不整合**: 許可されていない型（擬似型など）をベースに指定した場合にエラーとなります。
  • **名前衝突**: 同名の型が既に存在し、かつリネームできない場合にエラーとなります。
  • **照合順序エラー**: 照合順序をサポートしない型に対して COLLATE を指定した場合にエラーとなります。

保守担当者向けメモ

  • 本関数で継承するプロパティを変更した場合は、必ず AlterTypeRecurse() のロジックも同期させて更新してください。
  • ドメインは独自の typmod を受け付けないため、typmodin/out 関数は設定されません。
