CaseMapResult

定義

  • 種別: enum
  • 定義箇所: src/common/unicode_case.c:23

目的

Unicode 文字のケース変換処理において、適用すべき変換ルールの状態を分類するために使用されます。

シグネチャまたは形状

enum CaseMapResult
{
    CASEMAP_SELF,
    CASEMAP_SIMPLE,
    CASEMAP_SPECIAL,
};

入力と出力

  • 本シンボルは列挙型であり、関数 casemap の戻り値として使用されます。
  • 各値は、変換ロジックが「変換不要（自己）」「単純な1対1変換」「文脈依存の特殊変換」のいずれであるかを示します。

前提条件と不変条件

  • casemap 関数が文字単位の変換ルールを決定する際に、この列挙型のいずれかを返します。

アルゴリズムまたは振る舞い

  • CASEMAP_SELF: 入力文字に対して変換が定義されていない、または変換結果が入力と同一であることを示します。
  • CASEMAP_SIMPLE: 単純なテーブル参照による1対1のケース変換が適用可能であることを示します。
  • CASEMAP_SPECIAL: ギリシャ語の最終シグマ処理など、文脈や特殊条件を考慮した変換が必要であることを示します。

呼び出し元と呼び出し先

  • 呼び出し元: casemap 関数（src/common/unicode_case.c:45）がこの型を返します。
  • 呼び出し先: convert_case 関数（src/common/unicode_case.c:238, 396）が、casemap の戻り値としてこの型を受け取り、後続の変換処理を分岐させます。

関連データ構造

  • pg_wchar: Unicode 文字を扱うための型。
  • CaseKind: 変換の種類（大文字、小文字、タイトルケース等）を定義する列挙型。

失敗モードと境界条件

  • 特記事項なし。列挙型であるため、定義された3つの状態以外の値は想定されていません。

保守担当者向けメモ

  • 新たな Unicode ケース変換ルールが追加された場合、この列挙型に状態を追加する必要があるか検討してください。
