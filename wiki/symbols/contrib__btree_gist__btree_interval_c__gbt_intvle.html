gbt_intvle

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_interval.c:49

目的

2つの interval 型の値が「以下（less than or equal to）」の関係にあるかを判定し、GiST インデックスの比較処理をサポートします。

シグネチャまたは形状

static bool gbt_intvle(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の interval 値へのポインタ。
  • b: 比較対象の interval 値へのポインタ。
  • flinfo: 関数呼び出し情報（使用されません）。
  • **出力**:
  • bool: a <= b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は有効な interval 型のデータである必要があります。
  • 内部的に interval_le 関数を呼び出すため、PostgreSQL の標準的な interval 比較ロジックに依存します。

アルゴリズムまたは振る舞い

1. IntervalPGetDatum マクロを使用して、入力されたポインタを PostgreSQL の Datum 形式に変換します。 2. DirectFunctionCall2 を介して、PostgreSQL のコア関数である interval_le を呼び出します。 3. 戻り値を DatumGetBool で論理値に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し先**: interval_le（PostgreSQL コア関数）
  • **呼び出し元**:
  • contrib/btree_gist/btree_interval.c 内の tinfo 構造体（gbtree_ninfo）の初期化時に参照されています。

関連データ構造

  • interval: PostgreSQL の時間間隔型。
  • FmgrInfo: 関数呼び出し情報（本関数では未使用）。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、IntervalPGetDatum の挙動により未定義の動作を引き起こす可能性があります。
  • interval_le がエラーを発生させた場合、本関数もそのエラーを伝播させます。

保守担当者向けメモ

  • 本関数は btree_gist モジュールにおける interval 型の比較演算のラッパーです。ロジックの変更が必要な場合は、コアの interval_le 実装を確認してください。
  • flinfo 引数は現在の実装では使用されていませんが、インターフェースの整合性のために保持されています。
