MemoryContextCheckSize

定義

  • 種別: function (static inline)
  • 定義箇所: src/include/utils/memutils_internal.h:166

目的

要求されたメモリ割り当てサイズが、現在のメモリコンテキストおよびフラグ設定において妥当であるかを検証します。

シグネチャまたは形状

static inline void MemoryContextCheckSize(MemoryContext context, Size size, int flags)

入力と出力

  • context: 検証対象のメモリコンテキスト。
  • size: 割り当て要求サイズ。
  • flags: 割り当ての特性を示すビットフラグ（MCXT_ALLOC_HUGE 等）。
  • 出力: なし。検証失敗時は MemoryContextSizeFailure を呼び出し、エラーを発生させます。

前提条件と不変条件

  • size が AllocSizeIsValid を満たさない場合、flags に MCXT_ALLOC_HUGE が含まれており、かつ AllocHugeSizeIsValid(size) が真である必要があります。

アルゴリズムまたは振る舞い

1. AllocSizeIsValid(size) をチェックし、サイズが標準的な範囲内であれば正常終了します。 2. サイズが標準範囲外の場合、flags に MCXT_ALLOC_HUGE が含まれているかを確認します。 3. MCXT_ALLOC_HUGE が指定されており、かつ AllocHugeSizeIsValid(size) が真であれば、巨大な割り当てとして許可し正常終了します。 4. 上記のいずれの条件も満たさない場合、MemoryContextSizeFailure を呼び出してエラー処理へ移行します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/mmgr/aset.c
  • src/backend/utils/mmgr/bump.c
  • src/backend/utils/mmgr/generation.c
  • src/backend/utils/mmgr/slab.c
  • 呼び出し先:
  • AllocSizeIsValid (マクロ)
  • AllocHugeSizeIsValid (マクロ)
  • MemoryContextSizeFailure (関数)

関連データ構造

  • MemoryContext: メモリ管理の基底構造体。
  • Size: メモリサイズを表す型。

失敗モードと境界条件

  • size がシステムで許容される最大値を超えている場合、MemoryContextSizeFailure が呼び出されます。
  • MCXT_ALLOC_HUGE フラグなしで巨大なサイズが要求された場合、AllocSizeIsValid のチェックで失敗します。

保守担当者向けメモ

  • 本関数はインライン関数であり、メモリ割り当てのホットパスで実行されます。検証ロジックを変更する場合は、パフォーマンスへの影響を考慮してください。
  • MemoryContextSizeFailure は通常、エラーをスローして制御フローを中断させるため、本関数以降の処理は実行されません。
