ConnStatusType

定義

  • 種別: enum
  • 定義箇所: src/interfaces/libpq/libpq-fe.h:82

目的

PostgreSQL データベース接続の現在の状態を表現するために使用されます。

シグネチャまたは形状

typedef enum {
    CONNECTION_OK,
    CONNECTION_BAD,
    CONNECTION_STARTED,
    CONNECTION_MADE,
    CONNECTION_AWAITING_RESPONSE,
    CONNECTION_AUTH_OK,
    CONNECTION_SETENV,
    CONNECTION_SSL_STARTUP,
    CONNECTION_NEEDED,
    CONNECTION_CHECK_WRITABLE,
    CONNECTION_CONSUME,
    CONNECTION_GSS_STARTUP,
    CONNECTION_CHECK_TARGET,
    CONNECTION_CHECK_STANDBY,
    CONNECTION_ALLOCATED,
    CONNECTION_AUTHENTICATING
} ConnStatusType;

入力と出力

  • 本シンボルは列挙型であり、関数ではありません。
  • PQstatus(const PGconn *conn) 関数の戻り値として、接続の現在の状態を返します。

前提条件と不変条件

  • CONNECTION_OK および CONNECTION_BAD 以外の値は、主に非ブロッキングモードでの接続処理中に発生します。
  • CONNECTION_SETENV は現在使用されていません。
  • CONNECTION_STARTED 以降の各状態は、ユーザーへのフィードバックや進捗確認の目的でのみ使用すべきであり、プログラムのロジックでこれらの存在に強く依存してはなりません。

アルゴリズムまたは振る舞い

  • 接続確立プロセスにおける各フェーズ（SSLハンドシェイク、認証、バックエンド起動待機など）を識別します。
  • 内部状態として libpq-int.h 内の PGconn 構造体で保持されます。

呼び出し元と呼び出し先

  • 呼び出し元: PQstatus や PQcancelStatus などの公開 API がこの型を返します。
  • 呼び出し先: 本シンボルは列挙型であるため、直接の呼び出し先はありません。

関連データ構造

  • PGconn: 接続状態を保持する不透明な構造体。
  • PGcancelConn: キャンセル接続の状態を保持する構造体。

失敗モードと境界条件

  • CONNECTION_BAD: 接続が確立されていない、または切断された状態を示します。
  • 状態遷移の詳細は実装に依存しており、将来的に新しい状態が追加される可能性がありますが、既存の定数の削除や変更は行われません。

保守担当者向けメモ

  • PGconn のメンバに直接アクセスせず、必ず PQstatus() などのアクセサ関数経由でこの値を取得してください。
  • 非ブロッキングモードの接続処理フローを変更する際は、新しい状態を本列挙型に追加する必要があるか検討してください。
