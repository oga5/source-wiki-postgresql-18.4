hex_decode

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/encode.c:210

目的

16進数形式の文字列（src）をバイナリデータ（dst）にデコードします。

シグネチャまたは形状

uint64 hex_decode(const char *src, size_t len, char *dst)

入力と出力

  • **入力**:
  • src: デコード対象の16進数文字列。
  • len: src の長さ（バイト単位）。
  • dst: デコード結果を格納するバッファ。
  • **出力**:
  • 戻り値: デコードされたバイナリデータの実際の長さ（uint64）。

前提条件と不変条件

  • dst は、デコード後のデータを受け取るために十分なメモリ領域が確保されている必要があります。
  • 内部的に hex_decode_safe を呼び出しており、エラーコンテキスト（escontext）には NULL が渡されます。

アルゴリズムまたは振る舞い

1. hex_decode_safe(src, len, dst, NULL) を呼び出します。 2. hex_decode_safe の戻り値（デコードされたバイト数）をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/encode.c 内の内部処理。
  • src/interfaces/ecpg/ecpglib/data.c: ECPGライブラリ内でのデータ変換処理。
  • **呼び出し先**:
  • hex_decode_safe (直接の委譲先)。

関連データ構造

  • なし（基本型のみを使用）。

失敗モードと境界条件

  • 入力文字列に16進数として不正な文字が含まれている場合、hex_decode_safe の仕様に従いエラーが発生します。
  • NULL コンテキストが渡されているため、エラー発生時はトランザクションの中断を伴う標準的なエラー報告が行われます。

保守担当者向けメモ

  • 本関数は hex_decode_safe のラッパーです。デコード処理のロジックを変更する場合は hex_decode_safe を参照してください。
  • ECPG等の外部インターフェースからも利用されているため、シグネチャの変更には注意が必要です。
