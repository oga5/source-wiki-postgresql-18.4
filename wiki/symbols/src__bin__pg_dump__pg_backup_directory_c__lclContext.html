lclContext

定義

  • **種別**: struct
  • **定義箇所**: src/bin/pg_dump/pg_backup_directory.c:48

目的

PostgreSQL のディレクトリ形式アーカイブにおける、実行時の状態管理を行うための構造体です。ダンプ先ディレクトリのパス、現在処理中のファイルハンドル、および並列処理の状態を保持します。

シグネチャまたは形状

typedef struct {
    char               *directory;
    CompressFileHandle *dataFH;
    CompressFileHandle *LOsTocFH;
    ParallelState      *pstate;
} lclContext;

入力と出力

  • **入力**: pg_dump 実行時に指定されたディレクトリパスが directory メンバに格納されます。
  • **出力**: 本構造体を通じて、個別のデータファイル（dataFH）やラージオブジェクトの TOC ファイル（LOsTocFH）へのアクセスが提供されます。

前提条件と不変条件

  • directory は有効なディレクトリパスを指す必要があります。
  • dataFH および LOsTocFH は、ファイルが開かれている間のみ有効なハンドルを保持します。

アルゴリズムまたは振る舞い

  • pg_backup_directory.c 内の各関数（_StartData, _EndData 等）において、ArchiveHandle->formatData を通じて本構造体が参照されます。
  • 並列処理時には pstate を用いてジョブの制御とファイルアクセス管理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_custom.c 内の複数の関数（_getFilePos 等）から、AH->formatData をキャストして参照されています。
  • **呼び出し先**: pg_malloc0 によるメモリ確保が行われています。

関連データ構造

  • CompressFileHandle: 圧縮・解凍を伴うファイル操作の抽象化。
  • ParallelState: 並列ダンプ・リストアの制御状態。
  • ArchiveHandle: アーカイブ全体の管理構造体。

失敗モードと境界条件

  • メモリ不足時に pg_malloc0 が失敗する可能性があります。
  • 指定されたディレクトリが存在しない、または書き込み権限がない場合、ファイルハンドル操作時にエラーが発生します。

保守担当者向けメモ

  • 本構造体は pg_backup_directory.c の内部状態を保持するプライベートな構造体です。
  • pg_backup_tar.c と構造が類似しているため、変更時は整合性に注意してください。
  • dataFH は現在開いているデータファイルのみを追跡するため、並列処理時の排他制御については pstate の実装に依存します。
