bms_is_valid_set

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/bitmapset.c:78

目的

Bitmapset 構造体が内部的な整合性ルールを満たしているかを検証します。主にデバッグアサーション用に使用されます。

シグネチャまたは形状

static bool bms_is_valid_set(const Bitmapset *a)

入力と出力

  • **入力**: const Bitmapset *a （検証対象のビットマップセット）
  • **出力**: bool （有効な場合は true、無効な場合は false）

前提条件と不変条件

  • NULL ポインタは空集合として有効とみなされます。
  • 非 NULL の場合、Bitmapset ノードタグが正しく設定されている必要があります。
  • メモリ効率のため、末尾のワード（words[nwords - 1]）はゼロであってはなりません。

アルゴリズムまたは振る舞い

1. 入力が NULL の場合、直ちに true を返します。 2. IsA(a, Bitmapset) を使用して、ノード型が正しいかを確認します。不正な場合は false を返します。 3. a->words 配列の最後の要素（nwords - 1）をチェックし、値が 0 であれば末尾のゼロワード排除ルールに違反していると判断し、false を返します。 4. 上記すべてを通過した場合、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/bitmapset.c 内の各集合演算関数（bms_copy, bms_equal, bms_union, bms_intersect, bms_difference, bms_is_subset 等）において、Assert(bms_is_valid_set(a)) として頻繁に呼び出されています。
  • **呼び出し先**: IsA マクロ（ノード型チェック）。

関連データ構造

  • Bitmapset: 本関数が検証対象とする構造体。nwords（ワード数）と words（ビット配列）を保持します。

失敗モードと境界条件

  • **NULL入力**: 成功（空集合として許容）。
  • **不正なノード型**: false を返します。
  • **末尾ゼロワード**: false を返します（メモリ最適化不備の検知）。

保守担当者向けメモ

  • 本関数は cassert ビルド環境での整合性チェックを主目的としています。
  • pfree されたメモリ領域へのアクセスや、不正なポインタが渡された場合の検知に役立ちます。
