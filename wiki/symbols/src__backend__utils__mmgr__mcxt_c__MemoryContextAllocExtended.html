MemoryContextAllocExtended

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/mcxt.c:1247

目的

指定されたメモリコンテキストに対して、フラグ制御付きのメモリ割り当てを行う汎用インターフェースです。

シグネチャまたは形状

void *MemoryContextAllocExtended(MemoryContext context, Size size, int flags)

入力と出力

  • **入力**:
  • context: 割り当て対象の MemoryContext。
  • size: 割り当てるバイト数。
  • flags: 動作を制御するビットマスク（MCXT_ALLOC_HUGE, MCXT_ALLOC_ZERO 等）。
  • **出力**:
  • 割り当てられたメモリ領域へのポインタ。割り当て失敗時は NULL を返す。

前提条件と不変条件

  • context が有効であること（MemoryContextIsValid による検証）。
  • メモリ割り当てが禁止されているクリティカルセクション内ではないこと（AssertNotInCriticalSection）。
  • size が flags に応じた有効な範囲内であること（AllocHugeSizeIsValid または AllocSizeIsValid）。

アルゴリズムまたは振る舞い

1. 引数の妥当性を検証し、無効なサイズであればエラーを投げる。 2. コンテキストの isReset フラグを false に設定する。 3. コンテキスト固有の methods->alloc を呼び出し、メモリを確保する。 4. 割り当てが失敗した場合は直ちに NULL を返す。 5. Valgrind 用のメモリプール追跡を通知する。 6. MCXT_ALLOC_ZERO フラグが指定されている場合、確保した領域を 0 で初期化する。 7. 確保した領域のポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/tidbitmap.c
  • src/backend/postmaster/bgworker.c
  • src/backend/utils/hash/dynahash.c
  • src/backend/utils/misc/guc.c
  • src/backend/utils/mmgr/mcxt.c (自身を呼び出すケースあり)
  • **呼び出し先**:
  • context->methods->alloc: コンテキスト型に応じた具体的な割り当て実装。

関連データ構造

  • MemoryContext: メモリコンテキストの基底構造体。
  • MemoryContextMethods: 各コンテキスト型が実装するメソッドテーブル。

失敗モードと境界条件

  • size が不正な場合: elog(ERROR) によりプロセスが中断される。
  • methods->alloc が NULL を返した場合: 呼び出し元へ NULL を伝播させる。

保守担当者向けメモ

  • 本関数はメモリ割り当てのフロントエンドであり、実際の確保ロジックは context->methods->alloc に委譲されます。
  • MCXT_ALLOC_ZERO によるゼロクリアは、割り当て成功後に実行されます。
  • palloc 等の他の割り当て関数と挙動を合わせるため、フラグの扱いに注意してください。
