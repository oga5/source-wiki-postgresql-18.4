EphemeralNameRelationType

定義

  • 種別: enum
  • 定義箇所: src/include/utils/queryenvironment.h:20

目的

クエリ実行中に一時的に管理されるリレーション（Ephemeral Named Relations: ENR）の種類を識別するために使用されます。

シグネチャまたは形状

typedef enum EphemeralNameRelationType
{
	ENR_NAMED_TUPLESTORE,		/* named tuplestore relation; e.g., deltas */
} EphemeralNameRelationType;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし

前提条件と不変条件

  • 現在、ENR_NAMED_TUPLESTORE のみが定義されており、トリガーの遷移テーブルなどの名前付きタプルストアリレーションを識別します。

アルゴリズムまたは振る舞い

  • 本シンボルは、EphemeralNamedRelationMetadata 構造体内の enrtype フィールドとして保持され、リレーションの型を判別するフラグとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/utils/queryenvironment.h 内の EphemeralNamedRelationMetadata 構造体定義で型として参照されています。
  • 呼び出し先: なし（列挙型のため、直接的な関数呼び出しは発生しません）。

関連データ構造

  • EphemeralNamedRelationMetadata: この列挙型をメンバとして保持します。
  • QueryEnvironment: ENR を管理するコンテナ構造体。

失敗モードと境界条件

  • 該当なし。

保守担当者向けメモ

  • 新しい種類の一時リレーションを追加する必要がある場合は、この列挙型に新しいメンバを追加してください。
  • 現在の実装はリストベースの管理を想定しているため、列挙型の追加に伴う検索ロジックへの影響を src/backend/utils/misc/queryenvironment.c で確認してください。
