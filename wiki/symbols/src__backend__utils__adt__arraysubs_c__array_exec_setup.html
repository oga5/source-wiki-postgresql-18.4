array_exec_setup

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arraysubs.c:473

目的

配列の添字操作（サブスクリプト）を実行するための環境を構築し、操作の種類に応じた関数ポインタをセットアップします。

シグネチャまたは形状

static void array_exec_setup(const SubscriptingRef *sbsref, SubscriptingRefState *sbsrefstate, SubscriptExecSteps *methods)

入力と出力

  • **入力**:
  • sbsref: 構文解析済みの SubscriptingRef ノード。
  • sbsrefstate: 実行時の状態を保持する構造体。
  • methods: 実行ステップ（fetch/assign関数）を格納する構造体。
  • **出力**: なし（sbsrefstate->workspace の割り当ておよび methods 構造体の更新による副作用）。

前提条件と不変条件

  • sbsrefstate->numupper が MAXDIM を超える場合、エラーが発生します。
  • スライス操作（numlower != 0）の場合、numupper と numlower の長さが一致している必要があります。

アルゴリズムまたは振る舞い

1. **次元数チェック**: 配列の次元数が MAXDIM を超えていないか検証します。 2. **整合性チェック**: スライス操作時に上下限のリスト長が一致しているか確認します。 3. **ワークスペース確保**: ArraySubWorkspace をメモリ割り当てし、sbsrefstate に格納します。 4. **型情報の収集**: get_typlen および get_typlenbyvalalign を使用して、要素の型情報（長さ、値渡しフラグ、アライメント）をワークスペースに保存します。 5. **メソッドの割り当て**: 操作がスライスか単一要素かにより、以下の関数ポインタを methods に設定します。

  • スライス時: array_subscript_fetch_slice, array_subscript_assign_slice, array_subscript_fetch_old_slice
  • 単一要素時: array_subscript_fetch, array_subscript_assign, array_subscript_fetch_old

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/arraysubs.c 内の array_subscript_handler および raw_array_subscript_handler において、SubscriptingRefState の exec_setup フィールドとして参照されています。
  • **呼び出し先**:
  • palloc
  • get_typlen
  • get_typlenbyvalalign
  • ereport / elog

関連データ構造

  • SubscriptingRef: 添字操作の構文ノード。
  • SubscriptingRefState: 実行時の状態管理構造体。
  • SubscriptExecSteps: 実行関数ポインタを保持する構造体。
  • ArraySubWorkspace: 実行時に必要な型情報や境界値を保持する構造体。

失敗モードと境界条件

  • numupper > MAXDIM: ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラー終了します。
  • numlower != 0 かつ numupper != numlower: 不整合な添字リストとしてエラー終了します。

保守担当者向けメモ

  • 本関数は実行計画の初期化時に呼び出されます。
  • ArraySubWorkspace は MAXDIM に依存した固定長配列を含むため、次元数制限のチェックは必須です。
  • raw_array_subscript_handler と array_subscript_handler の両方から参照されているため、変更時は両方の挙動に影響がないか確認してください。
