inet_mcv_join_sel

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_selfuncs.c:687

目的

2つのカラムの最頻値（MCV）リスト間における結合選択率を推定します。

シグネチャまたは形状

static Selectivity inet_mcv_join_sel(Datum *mcv1_values, float4 *mcv1_numbers, int mcv1_nvalues,
                                     Datum *mcv2_values, float4 *mcv2_numbers, int mcv2_nvalues,
                                     Oid operator)

入力と出力

  • **入力**:
  • mcv1_values, mcv2_values: 各カラムのMCV値の配列。
  • mcv1_numbers, mcv2_numbers: 各MCV値の出現頻度（0.0〜1.0）の配列。
  • mcv1_nvalues, mcv2_nvalues: 各MCVリストの要素数。
  • operator: 結合に使用される演算子のOID。
  • **出力**:
  • Selectivity (float8): 推定された結合選択率。

前提条件と不変条件

  • operator は get_opcode を介して実行可能な関数として解決可能である必要があります。
  • mcv1_numbers および mcv2_numbers は、それぞれの母集団における割合を正しく保持している必要があります。

アルゴリズムまたは振る舞い

1. operator に対応する関数情報を fmgr_info で取得します。 2. MCVリストの全組み合わせ（デカルト積）に対して二重ループを実行します。 3. 各ペア (mcv1_values[i], mcv2_values[j]) に対して演算子関数を適用し、真（TRUE）を返す場合に限り、そのペアの出現確率 mcv1_numbers[i] * mcv2_numbers[j] を合計値 selec に加算します。 4. 最終的な合計値 selec を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/network_selfuncs.c:351 にて呼び出されています。
  • **呼び出し先**:
  • get_opcode (演算子のOID解決)
  • fmgr_info (関数情報の取得)
  • FunctionCall2 (演算子関数の実行)

関連データ構造

  • Datum: PostgreSQLの汎用データ表現。
  • Selectivity: 選択率を表す float8 型の別名。
  • FmgrInfo: 関数呼び出しのための管理構造体。

失敗モードと境界条件

  • MCVリストが空（mcv_nvalues == 0）の場合、ループは実行されず選択率は 0.0 となります。
  • 演算子関数がエラーをスローした場合、処理は中断されます。

保守担当者向けメモ

  • 本関数は $O(N \times M)$ の計算量を持つため、MCVリストのサイズが大きい場合はプランナのコスト計算に影響を与える可能性があります。
  • 包含関係の判定は operator に依存するため、演算子のセマンティクスが変更された場合は本ロジックの妥当性を再確認してください。
