pgp_get_digest_code

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp.c:103

目的

OpenPGP のハッシュアルゴリズム名（文字列）を、内部で使用される数値コードへ変換します。

シグネチャまたは形状

int pgp_get_digest_code(const char *name);

入力と出力

  • **入力**: name (ハッシュアルゴリズム名を表す文字列)
  • **出力**: 成功時は対応する内部コード（int）、一致するアルゴリズムが見つからない場合は PXE_PGP_UNSUPPORTED_HASH を返します。

前提条件と不変条件

  • digest_list が有効な digest_info 構造体の配列として定義されており、末尾が name フィールドの NULL チェックで判定可能である必要があります。
  • 入力 name は NULL であってはなりません。

アルゴリズムまたは振る舞い

1. 静的に定義された digest_list を順次走査します。 2. pg_strcasecmp を使用し、大文字小文字を区別せずに name とリスト内のアルゴリズム名を比較します。 3. 一致するものが見つかった場合、そのアルゴリズムに関連付けられた code を返します。 4. リストの最後まで一致が見つからなかった場合、エラーコード PXE_PGP_UNSUPPORTED_HASH を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-pgsql.c:230
  • contrib/pgcrypto/pgp.c:331
  • **呼び出し先**:
  • pg_strcasecmp (PostgreSQL 標準ライブラリ関数)

関連データ構造

  • digest_info: アルゴリズム名と内部コードを保持する構造体。

失敗モードと境界条件

  • 指定された名前がサポートされていない場合、PXE_PGP_UNSUPPORTED_HASH が返されます。
  • name に NULL を渡した場合の挙動は未定義です（ソースコード上でのガードは存在しません）。

保守担当者向けメモ

  • 新しいハッシュアルゴリズムを追加する場合は、digest_list を更新してください。
  • 比較には pg_strcasecmp を使用しているため、アルゴリズム名の指定はケースインセンシティブです。
