spg_kd_inner_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgkdtreeproc.c:159

目的

SP-GiSTインデックスのk-d木において、検索クエリ条件に基づき、探索対象のサブツリーを枝刈り（pruning）し、次に進むべきノードを判定する。

シグネチャまたは形状

Datum spg_kd_inner_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgInnerConsistentIn（検索条件、プレフィックス、レベル情報等）を受け取る。
  • **出力**: spgInnerConsistentOut を介して、探索を継続すべきノード番号、および ORDER BY 句がある場合は各ノードの境界ボックスと距離情報を返す。

前提条件と不変条件

  • in->hasPrefix が真であること。
  • in->nNodes == 2 であること。
  • in->allTheSame は発生しない（k-d木では中央値分割が前提のため）。
  • 探索対象のノードはビットマスク which で管理される。

アルゴリズムまたは振る舞い

1. **初期化**: 探索対象ノードを両方（1 << 1 | 1 << 2）に設定する。 2. **枝刈り**: 各 scankeys に対して、現在のレベル（in->level）とプレフィックス（coord）に基づき、クエリ条件を満たさないノードを which ビットマスクから除外する。 3. **早期終了**: which == 0 となった場合、探索を打ち切る。 4. **距離計算（ORDER BY時）**:

  • 現在のノードの境界ボックスを計算し、traversalValues に保持する。
  • spg_key_orderbys_distances を呼び出し、各ノードの距離を算出する。

    • 5. **結果格納**: which で指定されたノード番号を out->nodeNumbers に格納し、必要に応じて out->traversalValues と out->distances を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: 供給されたソース情報からは特定不可。
  • **呼び出し先**: DatumGetPointP, DatumGetBoxP, FPlt, FPgt, get_float8_infinity, palloc, MemoryContextSwitchTo, box_copy, spg_key_orderbys_distances 等のユーティリティ関数を呼び出す。

関連データ構造

  • spgInnerConsistentIn, spgInnerConsistentOut: SP-GiSTの内部整合性チェック用構造体。
  • BOX: 空間分割および境界ボックス計算に使用。
  • Point: 検索クエリの座標データ。

失敗モードと境界条件

  • allTheSame が検出された場合、エラーをスローする。
  • 未知の sk_strategy が渡された場合、エラーをスローする。
  • which == 0 の場合、探索対象なしとして即座に終了する。

保守担当者向けメモ

  • in->level % 2 の偶奇によって、分割軸が y か x かが切り替わる。
  • ORDER BY 句がある場合、再帰的に境界ボックスを計算して traversalValues に保持する必要がある。
  • RTContainedByStrategyNumber 処理時は、クエリが Point ではなく BOX として扱われる点に注意。
