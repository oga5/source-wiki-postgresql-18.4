OprProofCacheKey

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/util/predtest.c:2074

目的

演算子間の論理包含関係を証明する際のキャッシュ検索用キーとして機能します。

シグネチャまたは形状

typedef struct OprProofCacheKey
{
    Oid         pred_op;        /* 述語演算子 */
    Oid         clause_op;      /* 節演算子 */
} OprProofCacheKey;

入力と出力

  • 入力: 比較対象となる2つの演算子の Oid。
  • 出力: ハッシュテーブル検索のための構造体インスタンス。

前提条件と不変条件

  • OprProofCacheHash ハッシュテーブルのキーとして使用されます。
  • 構造体のサイズは sizeof(OprProofCacheKey) で定義されます。

アルゴリズムまたは振る舞い

  • 演算子ペアをキーとして、OprProofCacheHash 内のキャッシュエントリを特定するために使用されます。
  • 述語の証明プロセスにおいて、特定の演算子組み合わせに対する推論結果を再利用可能にします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/util/predtest.c 内の演算子推論ロジック。
  • 呼び出し先: 情報なし。

関連データ構造

  • OprProofCacheEntry: このキーに対応するキャッシュ値。
  • OprProofCacheHash: このキーを格納するハッシュテーブル。

失敗モードと境界条件

  • キャッシュの無効化: pg_amop カタログが変更された場合、InvalidateOprProofCacheCallBack によってキャッシュ全体がフラッシュされます。

保守担当者向けメモ

  • 本構造体は OprProofCacheHash のキーとして直接メモリレイアウトに依存するため、メンバの追加や変更を行う際はハッシュ関数の整合性に注意してください。
