GetHashPageStatistics

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/hashfuncs.c:155

目的

ハッシュインデックスの物理ページから統計情報を抽出し、指定された HashPageStat 構造体に格納する。

シグネチャまたは形状

static void GetHashPageStatistics(Page page, HashPageStat *stat)

入力と出力

  • **入力**:
  • page: 解析対象のハッシュインデックスページ（Page 型）。
  • stat: 統計情報を書き込む先の HashPageStat 構造体へのポインタ。
  • **出力**: なし（stat を通じて結果を返す）。

前提条件と不変条件

  • 入力された page は、呼び出し元で verify_hash_page 等により有効性が検証されていることが期待される。
  • stat ポインタは有効なメモリ領域を指している必要がある。

アルゴリズムまたは振る舞い

1. ページ内の最大オフセット番号を取得し、ページ内の不透明データ（HashPageOpaque）にアクセスする。 2. ページサイズ、および不透明データ内のメタ情報（hasho_prevblkno, hasho_nextblkno, hasho_bucket, hasho_flag, hasho_page_id）を stat にコピーする。 3. FirstOffsetNumber から最大オフセットまでループし、各アイテムの ItemId を確認する。

  • ItemIdIsDead が偽であれば live_items をインクリメントする。
  • ItemIdIsDead が真であれば dead_items をインクリメントする。

    • 4. PageGetFreeSpace を使用して、ページ内の空き容量を stat->free_size に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pageinspect/hashfuncs.c 内の関数（例: hash_page_stats 等）から呼び出される。
  • **呼び出し先**:
  • PageGetMaxOffsetNumber
  • HashPageGetOpaque
  • PageGetPageSize
  • PageGetItemId
  • ItemIdIsDead
  • PageGetFreeSpace

関連データ構造

  • HashPageStat: 統計情報を保持する構造体。
  • HashPageOpaqueData: ハッシュインデックスページの末尾に配置される不透明データ構造。

失敗モードと境界条件

  • ページが破損している場合、PageGetMaxOffsetNumber や PageGetItemId の内部で不正なメモリ参照が発生する可能性がある。
  • 本関数自体はエラーハンドリングを行わず、呼び出し元での事前検証に依存する。

保守担当者向けメモ

  • 本関数は static であり、モジュール内でのみ使用される。
  • HashPageStat 構造体の定義が変更された場合、本関数内のコピー処理も更新する必要がある。
