SPLITCOST

定義

  • 種別: struct
  • 定義箇所: contrib/intarray/_intbig_gist.c:307

目的

GiST インデックスのノード分割（picksplit）処理において、各エントリの移動コストを保持およびソートするために使用されるデータ構造です。

シグネチャまたは形状

typedef struct
{
	OffsetNumber pos;
	int32		cost;
} SPLITCOST;

入力と出力

  • pos: インデックス内のエントリ位置（OffsetNumber）。
  • cost: そのエントリを特定のノードへ移動させる際のコスト値（int32）。

前提条件と不変条件

  • picksplit 処理中に、インデックス内の全エントリに対してコストを計算し、本構造体の配列に格納した上でソートを行うことが想定されています。

アルゴリズムまたは振る舞い

GiST インデックスのノード分割アルゴリズムにおいて、エントリを左右のノードに振り分ける際、コストに基づいて優先順位を決定するために利用されます。具体的には、palloc で確保された SPLITCOST の配列に対し、qsort を適用してコスト順に並べ替えることで、分割の最適化を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/intarray/_intbig_gist.c および contrib/intarray/_int_gist.c 内の picksplit 実装関数。
  • **呼び出し先**: qsort（標準ライブラリ）によるソート処理。
  • 注: contrib/hstore/hstore_gist.c でも同名の構造体が定義・利用されていますが、本ページは _intbig_gist.c の定義を対象としています。

関連データ構造

  • OffsetNumber: PostgreSQL のインデックスページ内でのエントリ位置を示す型。

失敗モードと境界条件

  • メモリ不足時: palloc によるメモリ確保に失敗した場合、PostgreSQL のエラーハンドリングにより処理が中断されます。
  • maxoff（エントリ数）が 0 の場合、配列確保時に問題が発生する可能性があるため、呼び出し側の実装で適切にガードされている必要があります。

保守担当者向けメモ

  • 本構造体は _int_gist.c と _intbig_gist.c で重複定義されています。修正を行う際は、両方のファイルで整合性が取れているか確認してください。
  • hstore モジュールにも同名の構造体が存在するため、コード検索時にはファイルパスに注意してください。
