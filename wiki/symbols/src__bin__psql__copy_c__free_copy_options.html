free_copy_options

定義

  • 種別: function
  • 定義箇所: src/bin/psql/copy.c:64

目的

copy_options 構造体およびその内部で動的に確保された文字列メンバを解放し、メモリリークを防ぐ。

シグネチャまたは形状

static void free_copy_options(struct copy_options *ptr);

入力と出力

  • **入力**: struct copy_options *ptr（解放対象の構造体ポインタ）
  • **出力**: なし（void）

前提条件と不変条件

  • 引数 ptr が NULL の場合、関数は何もせず正常に終了する。
  • ptr が指す構造体内の各メンバ（before_tofrom, after_tofrom, file）は、malloc 等で動的に確保されているか、あるいは NULL である必要がある。

アルゴリズムまたは振る舞い

1. ptr が NULL かどうかを判定し、NULL であれば即座にリターンする。 2. 構造体内の各メンバ before_tofrom、after_tofrom、file に対して free() を呼び出す。 3. 最後に構造体自身である ptr を free() する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/copy.c 内の do_copy 関数（複数箇所でエラーハンドリングや処理終了時に呼び出し）。
  • **呼び出し先**:
  • 標準ライブラリの free()。

関連データ構造

  • struct copy_options（src/bin/psql/copy.c で定義）

失敗モードと境界条件

  • ptr が無効なメモリアドレスを指している場合、free() の呼び出しにより未定義動作が発生する。
  • 構造体内の各メンバが動的に確保されていない（スタック上のメモリを指している等）場合、free() の呼び出しによりプログラムが異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は copy_options 構造体の所有権を管理する唯一の解放手段であるため、copy_options を動的に生成する箇所では必ず本関数を呼び出すこと。
