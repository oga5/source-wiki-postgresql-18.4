datumSerialize

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/datum.c:494

目的

Datum 値をバイトストリームへシリアライズし、プロセス間通信や永続化に適した形式に変換します。

シグネチャまたは形状

void datumSerialize(Datum value, bool isnull, bool typByVal, int typLen, char **start_address)

入力と出力

  • value: シリアライズ対象の Datum。
  • isnull: 値が NULL かどうかを示すフラグ。
  • typByVal: 型が値渡し（by-value）かどうか。
  • typLen: 型の長さ（バイト単位）。
  • start_address: シリアライズ先のアドレスを指すポインタ。書き込み後、書き込んだバイト数分だけ進められます。

前提条件と不変条件

  • start_address が指すバッファは、シリアライズされるデータ量（ヘッダー + ペイロード）を格納するのに十分なサイズである必要があります。
  • 呼び出し元は、対象の型に応じた正しい typByVal と typLen を提供しなければなりません。

アルゴリズムまたは振る舞い

1. **ヘッダー書き込み**:

  • NULL の場合は -2、値渡し型の場合は -1 をヘッダーとして書き込みます。
  • 拡張オブジェクト（Expanded Object）の場合はそのフラットサイズを、それ以外は datumGetSize で取得したサイズをヘッダーとして書き込みます。

    • 2. **ペイロード書き込み**:

  • isnull が真の場合は終了します。
  • typByVal が真の場合、Datum 値を直接コピーします。
  • 拡張オブジェクトの場合、一時メモリ領域へフラット化してからバッファへコピーし、一時領域を解放します。
  • 上記以外の場合、Datum が指すポインタから header バイト分をコピーします。

    • 3. **ポインタ更新**: 処理の都度、*start_address を書き込んだデータ分だけ進めます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtree.c
  • src/backend/executor/execParallel.c
  • src/backend/nodes/params.c
  • **呼び出し先**:
  • datumGetSize
  • EOH_get_flat_size
  • EOH_flatten_into
  • palloc / pfree
  • memcpy

関連データ構造

  • Datum: PostgreSQL の汎用データ型表現。
  • ExpandedObjectHeader: 拡張オブジェクトのヘッダー構造体。

失敗モードと境界条件

  • typByVal や typLen が実際の型定義と一致しない場合、メモリ破壊や不正なポインタ参照が発生します。
  • start_address が指す領域が不足している場合、バッファオーバーランが発生します。

保守担当者向けメモ

  • 本関数は拡張オブジェクトを自動的にフラット化するため、呼び出し元はメモリコンテキストの寿命を意識せずにシリアライズ可能です。
  • EOH_flatten_into はターゲットアドレスの整列（alignment）を要求するため、一時バッファを経由する実装になっています。
