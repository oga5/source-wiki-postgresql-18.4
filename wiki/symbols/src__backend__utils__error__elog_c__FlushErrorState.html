FlushErrorState

定義

  • 種別: function
  • 定義箇所: src/backend/utils/error/elog.c:1871

目的

エラー処理中に中断されたメッセージ構築状態を破棄し、エラー管理スタックおよび関連するメモリコンテキストを初期状態にリセットする。

シグネチャまたは形状

void FlushErrorState(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルなエラー状態変数を更新し、メモリを解放する）

前提条件と不変条件

  • エラー処理中に別のメッセージ構築が中断された場合、制御がその構築処理から脱出しており、二度と戻らないことを前提とする。
  • ErrorContext はエラー処理専用のメモリコンテキストとして管理されている必要がある。

アルゴリズムまたは振る舞い

1. errordata_stack_depth を -1 に設定し、エラーデータスタックを空にする。 2. recursion_depth を 0 にリセットし、再帰的なエラー処理状態を解除する。 3. MemoryContextReset(ErrorContext) を呼び出し、エラー処理用メモリコンテキスト内の全データを解放する。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • contrib/postgres_fdw/connection.c
  • contrib/unaccent/unaccent.c
  • src/backend/executor/spi.c
  • src/backend/postmaster/autovacuum.c

  • src/backend/postmaster/walwriter.c
  • 呼び出し先:
  • MemoryContextReset（utils/memutils.h 経由）

関連データ構造

  • ErrorContext: エラー処理中に使用されるメモリコンテキスト。
  • errordata_stack_depth: エラーデータスタックの深さを保持するグローバル変数。
  • recursion_depth: エラー処理の再帰深度を保持するグローバル変数。

失敗モードと境界条件

  • 本関数自体はエラーを返さない。
  • スタックが既に空である場合や、ErrorContext が既に空である場合でも、リセット処理は安全に実行される。

保守担当者向けメモ

  • errordata_stack_depth が 0 より大きい場合、スタックの不整合が発生している可能性があるが、本関数は強制的にリセットを行う設計となっている。
