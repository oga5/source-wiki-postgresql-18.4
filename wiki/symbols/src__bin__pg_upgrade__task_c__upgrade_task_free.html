upgrade_task_free

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/task.c:132

目的

UpgradeTask 構造体に関連付けられたリソースを解放し、メモリリークを防ぐ。

シグネチャまたは形状

void upgrade_task_free(UpgradeTask *task)

入力と出力

  • **入力**: task (UpgradeTask *) - 解放対象のタスク構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • task は upgrade_task_create 等で適切に初期化されている必要がある。
  • 本関数呼び出し後、task ポインタへのアクセスは無効となる。

アルゴリズムまたは振る舞い

1. task->queries が保持する PQExpBuffer を destroyPQExpBuffer で破棄する。 2. task->steps 配列を pg_free で解放する。 3. task 構造体自体を pg_free で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/check.c (複数箇所)
  • src/bin/pg_upgrade/function.c:103
  • src/bin/pg_upgrade/info.c:318
  • **呼び出し先**:
  • destroyPQExpBuffer
  • pg_free

関連データ構造

  • UpgradeTask: 本関数が管理するメインのデータ構造。

失敗モードと境界条件

  • task が NULL の場合、pg_free および destroyPQExpBuffer の実装依存の挙動となる（通常はセグメンテーション違反を引き起こす可能性がある）。

保守担当者向けメモ

  • 本関数は UpgradeTask のライフサイクルの終了時に必ず呼び出すこと。
  • task->steps や task->queries の内部要素が個別に動的確保されている場合、それらの解放順序や依存関係に変更がないか注意が必要。
