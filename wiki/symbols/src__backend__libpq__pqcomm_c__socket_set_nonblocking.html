socket_set_nonblocking

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqcomm.c:880

目的

現在のクライアント接続（MyProcPort）のノンブロッキング状態を切り替える。

シグネチャまたは形状

static void socket_set_nonblocking(bool nonblocking)

入力と出力

  • **入力**: nonblocking (bool) - true を指定するとノンブロッキングモードに、false を指定するとブロッキングモードに設定する。
  • **出力**: なし（void）。

前提条件と不変条件

  • MyProcPort が初期化されていること。初期化されていない場合、エラーが発生する。

アルゴリズムまたは振る舞い

1. MyProcPort が NULL かどうかを確認し、NULL であれば ERRCODE_CONNECTION_DOES_NOT_EXIST エラーを発生させる。 2. MyProcPort->noblock フィールドに引数 nonblocking の値を代入する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/pqcomm.c 内の複数箇所（行 915, 1017, 1287, 1301, 1336, 1449）から呼び出されている。
  • **呼び出し先**: 直接的な関数呼び出しはない。エラー発生時に ereport を使用する。

関連データ構造

  • Port (グローバル変数 MyProcPort が参照): クライアント接続情報を保持する構造体。noblock メンバを更新する。

失敗モードと境界条件

  • MyProcPort が NULL の場合、ereport(ERROR, ...) により処理が中断される。

保守担当者向けメモ

  • 本関数は Port 構造体のフラグを更新するのみであり、実際のソケットに対する fcntl 等のシステムコールは呼び出し元や他の層で管理されている可能性があるため注意すること。
