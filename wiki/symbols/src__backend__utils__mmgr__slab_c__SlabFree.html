SlabFree

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/slab.c:701

目的

SLAB アロケータによって割り当てられたメモリチャンクを解放し、管理ブロックの空きリストへ戻す。ブロックが完全に空になった場合は、再利用のために emptyblocks リストへ移動するか、メモリをシステムへ返却する。

シグネチャまたは形状

void SlabFree(void *pointer)

入力と出力

  • **入力**: pointer - 解放対象のメモリ領域を指すポインタ。
  • **出力**: なし。

前提条件と不変条件

  • pointer は、当該 SLAB コンテキストから割り当てられた有効なチャンクを指している必要がある。
  • block->nfree は常に 0 より大きく、slab->chunksPerBlock 以下であること。
  • MEMORY_CONTEXT_CHECKING が有効な場合、二重解放（requested_size == InvalidAllocSize）はエラーとして報告される。

アルゴリズムまたは振る舞い

1. **チャンク特定**: pointer からチャンクヘッダーと所属する SlabBlock を特定する。 2. **解放処理**: チャンクをブロックの freehead リストの先頭に挿入し、nfree をインクリメントする。 3. **ブロックリスト更新**: 空き数（nfree）の変化に基づき、ブロックを適切な blocklist インデックスへ移動させる。必要に応じて slab->curBlocklistIndex を再計算する。 4. **空きブロック処理**: ブロックが完全に空（nfree == chunksPerBlock）になった場合:

  • emptyblocks リストのサイズが上限（SLAB_MAXIMUM_EMPTY_BLOCKS）未満であれば、再利用のためにリストへ移動する。
  • 上限を超えている場合は、free() を呼び出してメモリを解放し、コンテキストの mem_allocated を減算する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c: メモリコンテキストの free_p 関数ポインタとして登録されている。
  • **呼び出し先**:
  • SlabBlockIsValid (マクロ/関数)
  • SlabBlocklistIndex
  • dlist_delete_from / dlist_push_head
  • dclist_push_head / dclist_count
  • SlabFindNextBlockListIndex
  • free() (標準ライブラリ)

関連データ構造

  • SlabContext: SLAB アロケータのメイン構造体。
  • SlabBlock: メモリブロックの管理構造体。
  • MemoryChunk: チャンクヘッダー。

失敗モードと境界条件

  • **二重解放**: MEMORY_CONTEXT_CHECKING 有効時、既に解放済みのチャンクに対して elog(ERROR) を発生させる。
  • **境界外書き込み**: MEMORY_CONTEXT_CHECKING 有効時、チャンク末尾のセンチネル値が破壊されている場合に elog(WARNING) を発生させる。
  • **メモリ解放**: emptyblocks が上限に達している場合、ブロック単位で free() が実行される。

保守担当者向けメモ

  • 本関数は SlabAlloc と対になる。SlabRealloc はサポートされていないため、本関数は固定サイズの解放のみを想定している。
  • CLOBBER_FREED_MEMORY が定義されている場合、解放されたメモリ領域はデバッグ用に上書きされる。
  • curBlocklistIndex の更新ロジックは、常に最も埋まっているブロックを優先的に使用するという SLAB の最適化方針を維持するために重要である。
