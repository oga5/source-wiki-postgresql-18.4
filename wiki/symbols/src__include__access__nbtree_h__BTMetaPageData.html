BTMetaPageData

定義

  • 種別: struct
  • 定義箇所: src/include/access/nbtree.h:104

目的

PostgreSQL B-tree インデックスの先頭ページ（メタページ）の構造を定義し、ルートページの位置、ツリーの高さ、およびバージョン管理情報を保持する。

シグネチャまたは形状

typedef struct BTMetaPageData
{
    uint32      btm_magic;
    uint32      btm_version;
    BlockNumber btm_root;
    uint32      btm_level;
    BlockNumber btm_fastroot;
    uint32      btm_fastlevel;
    uint32      btm_last_cleanup_num_delpages;
    float8      btm_last_cleanup_num_heap_tuples;
    bool        btm_allequalimage;
} BTMetaPageData;

入力と出力

本構造体はデータ保持用であり、関数ではない。各フィールドはインデックスのメタデータとして読み書きされる。

前提条件と不変条件

  • btm_magic は BTREE_MAGIC を保持する必要がある。
  • btm_version は常に BTREE_VERSION 以下であること。
  • btm_version >= BTREE_NOVAC_VERSION の場合のみ、btm_last_cleanup_num_delpages および btm_last_cleanup_num_heap_tuples が有効な値を持つ。

アルゴリズムまたは振る舞い

  • インデックスのルートページおよび「高速ルート（fast root）」ページを追跡し、ツリー降下時の開始点を提供する。
  • 最終クリーンアップ時の統計情報を保持し、メンテナンス処理の判断材料として利用される。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • contrib/amcheck/verify_nbtree.c
  • contrib/pageinspect/btreefuncs.c
  • contrib/pgstattuple/pgstatindex.c
  • src/backend/access/nbtree/nbtinsert.c
  • src/backend/access/nbtree/nbtpage.c
  • **呼び出し先**:
  • 本構造体はデータ構造であり、直接的な関数呼び出しは行わない。

関連データ構造

  • BTPageOpaqueData: B-tree ページの物理レイアウトを定義する構造体。
  • BTScanInsertData: ツリー降下時に使用される挿入用スキャンキー。

失敗モードと境界条件

  • btm_version が現在の実装でサポートされていない場合、インデックスの読み込みや操作に失敗する可能性がある。
  • btm_root や btm_fastroot が無効なブロック番号を指している場合、インデックス破損とみなされる。

保守担当者向けメモ

  • btm_version の変更時には、古いバージョンとの互換性を維持するためのロジックが必要となる。
  • btm_last_cleanup_num_heap_tuples は現在非推奨（deprecated）である点に留意すること。
