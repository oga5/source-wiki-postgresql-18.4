CreateCast

定義

  • **種別**: function
  • **定義箇所**: src/backend/commands/functioncmds.c:1538

目的

SQLの CREATE CAST 文を処理し、データ型間のキャスト定義をシステムカタログに登録するための準備および検証を行う。

シグネチャまたは形状

ObjectAddress CreateCast(CreateCastStmt *stmt)

入力と出力

  • **入力**: CreateCastStmt *stmt（解析済みの CREATE CAST 構文木）
  • **出力**: ObjectAddress（作成されたキャストオブジェクトの識別子）

前提条件と不変条件

  • ソース型およびターゲット型は擬似型（TYPTYPE_PSEUDO）であってはならない。
  • 実行ユーザーはソース型またはターゲット型のいずれかの所有者である必要がある。
  • 両方の型に対して ACL_USAGE 権限が必要。
  • ドメイン型が指定された場合、警告が発行される。

アルゴリズムまたは振る舞い

1. **型解決**: typenameTypeId を使用してソース型とターゲット型のOIDを取得。 2. **権限検証**: 所有者チェックおよび ACL_USAGE チェックを実行。 3. **キャストメソッドの決定**:

  • 関数指定がある場合: COERCION_METHOD_FUNCTION
  • INOUT 指定がある場合: COERCION_METHOD_INOUT
  • それ以外: COERCION_METHOD_BINARY

    • 4. **関数キャストの検証**:

  • LookupFuncWithArgs で関数を特定。
  • 引数の数（1〜3個）および型がソース型と互換性があるか、戻り値がターゲット型と互換性があるかを IsBinaryCoercibleWithCast で確認。
  • 関数が通常の関数であること、および集合を返さないことを検証。

    • 5. **バイナリキャストの検証**:

  • スーパーユーザー権限が必要。
  • get_typlenbyvalalign を使用し、型サイズ、アライメント、値渡し属性が一致することを確認。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（parsetree からのディスパッチ）
  • **呼び出し先**: ソースコード内で typenameTypeId, get_typtype, object_ownercheck, object_aclcheck, LookupFuncWithArgs, SearchSysCache1, IsBinaryCoercibleWithCast, get_typlenbyvalalign 等のユーティリティ関数を呼び出す。

関連データ構造

  • CreateCastStmt: SQL構文木を保持する構造体。
  • ObjectAddress: 作成されたオブジェクトの識別情報。
  • pg_proc: キャスト関数を検索するためのシステムカタログ。

失敗モードと境界条件

  • 擬似型が指定された場合: ERRCODE_WRONG_OBJECT_TYPE でエラー。
  • 権限不足: ERRCODE_INSUFFICIENT_PRIVILEGE でエラー。
  • 関数キャストで引数や戻り値の互換性が取れない場合: ERRCODE_INVALID_OBJECT_DEFINITION でエラー。
  • バイナリキャストで物理的な互換性（サイズ・アライメント等）がない場合: ERRCODE_INVALID_OBJECT_DEFINITION でエラー。

保守担当者向けメモ

  • NOT_USED マクロで囲まれたブロックにある通り、キャスト関数の揮発性（volatility）チェックは過去の互換性維持のため無効化されている。
  • バイナリキャストはバックエンドのクラッシュを招く可能性があるため、スーパーユーザーのみに制限されている。
