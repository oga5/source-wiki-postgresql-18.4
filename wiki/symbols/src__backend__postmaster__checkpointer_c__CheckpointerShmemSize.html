CheckpointerShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/checkpointer.c:937

目的

チェックポインタープロセスが共有メモリ上で使用する CheckpointerShmemStruct 構造体の合計サイズを計算する。

シグネチャまたは形状

Size CheckpointerShmemSize(void)

入力と出力

  • **入力**: なし
  • **出力**: Size 型の共有メモリ所要バイト数

前提条件と不変条件

  • NBuffers（バッファ数）および MAX_CHECKPOINT_REQUESTS（リクエストの最大数）に基づき計算される。
  • 共有メモリの割り当て前に呼び出される必要がある。

アルゴリズムまたは振る舞い

1. CheckpointerShmemStruct の固定部分のサイズ（offsetof）を取得する。 2. requests[] 配列のサイズを計算する。この際、NBuffers と MAX_CHECKPOINT_REQUESTS の小さい方の値を選択し、CheckpointerRequest 構造体のサイズを乗算する。 3. 固定部分と配列部分のサイズを add_size で加算して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/ipci.c: 共有メモリ初期化時に呼び出される。
  • src/backend/postmaster/checkpointer.c: CheckpointerShmemInit 内で呼び出される。
  • **呼び出し先**:
  • add_size (ユーティリティ関数)
  • mul_size (ユーティリティ関数)
  • Min (マクロ)

関連データ構造

  • CheckpointerShmemStruct: チェックポインターが使用する共有メモリ構造体。
  • CheckpointerRequest: fsync 要求を保持する構造体。

失敗モードと境界条件

  • add_size および mul_size は、計算結果が Size 型の範囲を超える場合にエラーを発生させる可能性がある。
  • MAX_CHECKPOINT_REQUESTS によるキャップにより、NBuffers が非常に大きい場合でも共有メモリ消費量が制限される。

保守担当者向けメモ

  • requests[] 配列のサイズ決定ロジックを変更する場合は、CheckpointerShmemInit での初期化ロジックと整合性が取れていることを確認すること。
