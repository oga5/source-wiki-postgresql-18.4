ASN1_STRING_to_text

定義

  • 種別: function
  • 定義箇所: contrib/sslinfo/sslinfo.c:150

目的

OpenSSLの ASN1_STRING 形式のデータを、PostgreSQLの text 型へ変換します。

シグネチャまたは形状

static Datum ASN1_STRING_to_text(ASN1_STRING *str)

入力と出力

  • **入力**: ASN1_STRING *str （変換対象のASN.1文字列）
  • **出力**: Datum （PostgreSQLの text 型として表現された変換後の文字列）

前提条件と不変条件

  • OpenSSLの BIO 構造体の作成および解放が成功する必要があります。
  • 変換後の文字列は、PostgreSQLのサーバーエンコーディングに適合している必要があります。

アルゴリズムまたは振る舞い

1. BIO_new(BIO_s_mem()) でメモリバッファを作成します。 2. ASN1_STRING_print_ex を使用し、RFC 2253形式かつUTF-8変換フラグを付与してバッファへ書き出します。 3. バッファの末尾にヌル文字を追加し、BIO_get_mem_data で内容を取得します。 4. pg_any_to_server を呼び出し、UTF-8からデータベースのサーバーエンコーディングへ変換します。 5. cstring_to_text でPostgreSQLの text 型に変換し、変換時にメモリが割り当てられた場合は pfree で解放します。 6. BIO_free でバッファを解放し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sslinfo/sslinfo.c 内の証明書属性抽出関数（ssl_client_dn_field 等）から呼び出されています。
  • **呼び出し先**:
  • OpenSSL API: BIO_new, BIO_set_close, ASN1_STRING_print_ex, BIO_write, BIO_get_mem_data, BIO_free
  • PostgreSQL API: ereport, pg_any_to_server, cstring_to_text, pfree, elog, PG_RETURN_TEXT_P

関連データ構造

  • ASN1_STRING (OpenSSL)
  • BIO (OpenSSL)
  • text (PostgreSQL)

失敗モードと境界条件

  • BIO_new が失敗した場合、ERRCODE_OUT_OF_MEMORY でエラーを報告します。
  • BIO_free が失敗した場合、elog(ERROR) でトランザクションを中断します。
  • 変換後の文字列が元のバッファと異なる場合（dp != sp）、一時メモリの解放を行います。

保守担当者向けメモ

  • BIO のメモリ管理と、pg_any_to_server によるエンコーディング変換後のメモリ解放（pfree）が正しく行われているか注意してください。
  • OpenSSLのAPIエラーはPostgreSQLのトランザクション中断を伴うため、呼び出し側でのエラーハンドリングを考慮する必要はありません。
