SharedIndexScanInstrumentation

定義

  • 種別: struct
  • 定義箇所: src/include/access/genam.h:44

目的

並列インデックススキャンにおいて、各ワーカープロセスの実行統計情報を共有メモリ上で保持するための構造体です。

シグネチャまたは形状

typedef struct SharedIndexScanInstrumentation
{
    int            num_workers;
    IndexScanInstrumentation winstrument[FLEXIBLE_ARRAY_MEMBER];
} SharedIndexScanInstrumentation;

入力と出力

  • **入力**: 並列クエリ実行時に、ワーカー数に基づいて共有メモリ領域が割り当てられます。
  • **出力**: 実行計画の表示（EXPLAIN）や統計収集のために、各ワーカーの計測結果が winstrument 配列に格納されます。

前提条件と不変条件

  • 本構造体は共有メモリ上に配置されるため、ポインタを含めてはなりません。
  • winstrument は可変長配列（FLEXIBLE_ARRAY_MEMBER）として定義されており、メモリ割り当て時には num_workers に応じたオフセット計算が必要です。

アルゴリズムまたは振る舞い

1. **初期化**: 並列スキャン開始時に、必要なワーカー数分を考慮したサイズで共有メモリが確保されます。 2. **計測**: 各ワーカーは自身の担当するインデックススキャンの統計を winstrument の対応するインデックス位置に書き込みます。 3. **集約**: 実行終了後、または EXPLAIN 実行時に、各ワーカーの計測データが読み取られ、全体の統計情報として集約されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/index/indexam.c: 並列スキャン時の共有メモリサイズ見積もりおよび初期化。
  • src/backend/executor/nodeBitmapIndexscan.c: ビットマップインデックススキャンにおける共有メモリの割り当ておよびアクセス。
  • src/backend/commands/explain.c: 並列スキャンの実行統計表示。
  • **呼び出し先**: 構造体定義のため、特定の関数を呼び出すことはありません。

関連データ構造

  • IndexScanInstrumentation: 各ワーカーの個別の計測データを保持する構造体。
  • ParallelIndexScanDesc: 並列インデックススキャンの状態を管理する構造体。

失敗モードと境界条件

  • 共有メモリの割り当てサイズ計算（offsetof を使用）が誤っている場合、メモリ破壊や不正アクセスが発生します。
  • ワーカー数が num_workers を超えてアクセスしようとした場合、境界外参照が発生します。

保守担当者向けメモ

  • 本構造体は並列スキャン時に共有メモリへコピーされるため、フィールドの追加や変更を行う際は、ポインタ型が含まれないよう厳重に注意してください。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、サイズ計算には必ず offsetof(SharedIndexScanInstrumentation, winstrument) を使用してください。
