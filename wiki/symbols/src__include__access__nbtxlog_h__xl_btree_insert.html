xl_btree_insert

定義

  • 種別: struct
  • 定義箇所: src/include/access/nbtxlog.h:79

目的

B-treeインデックスへのタプル挿入操作をWAL（Write-Ahead Log）に記録するためのデータ構造です。クラッシュリカバリ時にインデックスページを再構築するために使用されます。

シグネチャまたは形状

typedef struct xl_btree_insert
{
    OffsetNumber offnum;
    /* POSTING SPLIT OFFSET FOLLOWS (INSERT_POST case) */
    /* NEW TUPLE ALWAYS FOLLOWS AT THE END */
} xl_btree_insert;

入力と出力

  • **入力**: 挿入対象のページ内オフセット（offnum）。
  • **出力**: 構造体の末尾に続く可変長データとして、挿入される新しいタプルデータが格納されます。また、INSERT_POST（Posting List分割）時には、追加のオフセット情報が構造体の直後に配置されます。

前提条件と不変条件

  • 構造体のサイズは SizeOfBtreeInsert マクロ（offsetof(xl_btree_insert, offnum) + sizeof(OffsetNumber)）によって計算可能です。
  • 挿入される新しいタプルは、常に構造体の末尾に配置される必要があります。

アルゴリズムまたは振る舞い

1. **記録**: B-treeへの挿入処理（_bt_insert等）において、この構造体に挿入位置を記録し、XLogInsert を通じてWALへ書き出します。 2. **再生**: btree_redo 関数がWALレコードを読み込み、XLogRecGetData を介して本構造体を取得します。 3. **復元**: 記録された offnum を基に、対象ページ内の適切な位置へ新しいタプルを挿入し、ページ内容を再構築します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtinsert.c（挿入操作時）
  • **呼び出し先**: src/backend/access/nbtree/nbtxlog.c（REDO処理時）、src/backend/access/rmgrdesc/nbtdesc.c（ログ解析・デバッグ出力時）

関連データ構造

  • OffsetNumber: ページ内のアイテムオフセットを管理する型。
  • xl_btree_split: ページ分割時に使用される関連構造体。

失敗モードと境界条件

  • 構造体定義自体に境界条件はありませんが、REDO処理時には対象ページが既に更新済みである可能性があるため、LSN（Log Sequence Number）による冪等性のチェックが必須となります。

保守担当者向けメモ

  • 本構造体は可変長データ（新しいタプル）を伴うため、メモリレイアウトを操作する際は SizeOfBtreeInsert マクロを正しく使用してください。
  • Posting Listの分割処理（INSERT_POST）は、この構造体のREDOロジックと密接に関連しています。変更を加える際は nbtxlog.c のREDO実装を確認してください。
