getRestoreCommand

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/pg_rewind.c:1057

目的

ターゲットクラスタの postgresql.conf 等で設定されている restore_command を取得し、グローバル変数 restore_command に格納する。

シグネチャまたは形状

static void getRestoreCommand(const char *argv0)

入力と出力

  • **入力**: argv0（実行プログラムのパス）。また、グローバル変数 restore_wal（フラグ）、datadir_target（ターゲットデータディレクトリ）、config_file（設定ファイルパス）を参照する。
  • **出力**: なし（副作用としてグローバル変数 restore_command を更新する）。

前提条件と不変条件

  • restore_wal が真であること。
  • ターゲットクラスタの postgres 実行ファイルが、pg_rewind と同じディレクトリまたはパス内で発見可能であること。
  • ターゲットクラスタにおいて restore_command が設定されていること。

アルゴリズムまたは振る舞い

1. restore_wal が偽であれば即座に終了する。 2. find_other_exec を使用して、ターゲットクラスタの postgres 実行ファイルのパスを特定する。見つからない場合やバージョンが一致しない場合はエラー終了する。 3. postgres -D <datadir_target> [-c config_file=<config_file>] -C restore_command というコマンド文字列を構築する。 4. pipe_read_line を介して当該コマンドを実行し、標準出力から restore_command の値を取得する。 5. 取得した文字列から改行コードを除去し、空文字列でないことを確認する。 6. 成功した場合、取得した値をグローバル変数 restore_command に保持する。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数 (src/bin/pg_rewind/pg_rewind.c:298)。
  • **呼び出し先**: find_other_exec, find_my_exec, createPQExpBuffer, appendShellString, appendPQExpBufferStr, pipe_read_line, pg_strip_crlf, destroyPQExpBuffer 等。

関連データ構造

  • PQExpBuffer: コマンド文字列構築用バッファ。

失敗モードと境界条件

  • postgres 実行ファイルが見つからない、またはバージョン不一致の場合: pg_fatal で終了。
  • restore_command の読み取りに失敗した場合: pg_fatal で終了。
  • restore_command が設定されていない（空文字列）場合: pg_fatal で終了。

保守担当者向けメモ

  • 本関数は postgres コマンドを外部プロセスとして呼び出すため、実行環境のパス設定や権限に依存する。
  • restore_command の取得に失敗した際の挙動は、pg_rewind のリカバリ計画全体に影響を与えるため、エラーハンドリングは厳格に行われている。
