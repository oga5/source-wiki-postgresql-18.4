page_header

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/rawpage.c:248

目的

PostgreSQL の物理ページデータ（bytea）を解析し、そのページヘッダー情報を SQL から参照可能なタプルとして抽出する。

シグネチャまたは形状

Datum page_header(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される bytea 型のページデータ。
  • **出力**: ページヘッダーの各フィールド（LSN、チェックサム、フラグ、ポインタ位置、ページサイズ、バージョン、pd_prune_xid）を含む複合型（レコード）。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要。
  • 入力された bytea データは、get_page_from_raw を通じてメモリ上で適切にアライメントされた Page 構造体として扱われる必要がある。
  • 戻り値の型は複合型（row type）である必要がある。

アルゴリズムまたは振る舞い

1. **権限確認**: superuser() を呼び出し、非スーパーユーザーからのアクセスを拒否する。 2. **データ変換**: get_page_from_raw を用いて bytea を Page 構造体に変換し、PageHeader としてキャストする。 3. **型定義の取得**: get_call_result_type を使用して、呼び出し元の期待するタプル記述子（TupleDesc）を取得する。 4. **フィールド抽出**:

  • LSN を取得し、拡張機能のバージョンに応じて text または pg_lsn 型として格納する。
  • チェックサム、フラグ、pd_lower、pd_upper、pd_special、ページサイズ、バージョン、pd_prune_xid を抽出する。
  • 拡張機能のバージョン（int2 vs int4）に応じて、ポインタ関連フィールドの型を動的に判定する。

    • 5. **タプル構築**: heap_form_tuple を使用して結果を構築し、Datum として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • get_page_from_raw（ページデータの変換）
  • PageGetLSN, PageGetPageSize, PageGetPageLayoutVersion（マクロによるヘッダー情報取得）
  • heap_form_tuple（結果セットの構築）
  • superuser, ereport, elog（システム関数）

関連データ構造

  • PageHeaderData: ページヘッダーの物理レイアウトを定義する構造体。
  • TupleDesc: 戻り値のタプル構造を定義する記述子。
  • HeapTuple: 構築される結果タプル。

失敗モードと境界条件

  • **権限不足**: スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー終了する。
  • **型不一致**: 戻り値の型が複合型でない場合、または出力フィールドの型が期待と異なる場合、elog(ERROR) で終了する。

保守担当者向けメモ

  • 本関数は pageinspect 拡張機能のバージョンアップに伴う型変更（text から pg_lsn への移行、int2 から int4 への移行）を TupleDesc のチェックを通じて吸収している。修正時は TupleDescAttr による型判定ロジックに注意すること。
