OpFamilyOpFuncGroup

定義

  • 種別: struct
  • 定義箇所: src/include/access/amvalidate.h:20

目的

演算子族（Operator Family）内において、特定の左辺型と右辺型のペアに関連付けられた演算子およびサポート関数のセットを保持・管理するために使用されます。

シグネチャまたは形状

typedef struct OpFamilyOpFuncGroup
{
    Oid         lefttype;       /* amoplefttype/amproclefttype */
    Oid         righttype;      /* amoprighttype/amprocrighttype */
    uint64      operatorset;    /* 演算子のビットマスク */
    uint64      functionset;    /* サポート関数のビットマスク */
} OpFamilyOpFuncGroup;

入力と出力

  • 本構造体は、identify_opfamily_groups() 関数によって生成されるリストの要素として使用されます。
  • operatorset および functionset は、それぞれ最大63番までの戦略番号や関数番号をビットマスクとして保持します。

前提条件と不変条件

  • identify_opfamily_groups() を呼び出す際、入力となるカタログキャッシュ（CatCList）は適切にソートされている必要があります。
  • ビットマスクを使用しているため、戦略番号や関数番号が64以上となる拡張には対応していません。

アルゴリズムまたは振る舞い

  • 演算子族内の演算子とサポート関数を、型ペア（lefttype, righttype）ごとにグループ化します。
  • 各グループ内の演算子および関数は、ビットマスクとして集約され、インデックスの整合性検証時に各AMの検証ロジックによって参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下の各インデックスアクセスメソッドの検証ロジックから参照されています。
  • contrib/bloom/blvalidate.c
  • src/backend/access/brin/brin_validate.c
  • src/backend/access/gin/ginvalidate.c
  • src/backend/access/gist/gistvalidate.c
  • src/backend/access/hash/hashvalidate.c
  • **呼び出し先**: identify_opfamily_groups()（src/backend/access/index/amvalidate.c）が本構造体のリストを構築します。

関連データ構造

  • CatCList: カタログキャッシュから取得される入力データ。
  • List: identify_opfamily_groups が返す OpFamilyOpFuncGroup の集合を保持するリスト構造。

失敗モードと境界条件

  • 戦略番号または関数番号が64以上の場合、ビットマスクによる管理ができないため、正しく検証が行われません。

保守担当者向けメモ

  • 本構造体は、インデックス作成時の CREATE OPERATOR FAMILY 等の検証フェーズにおいて、メタデータの論理的整合性を確認するために不可欠です。
  • 構造体の拡張やビットマスクの制限変更が必要な場合は、amvalidate.c 内のグループ化ロジックと併せて検討してください。
