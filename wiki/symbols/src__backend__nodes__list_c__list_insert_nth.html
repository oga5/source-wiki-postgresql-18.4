list_insert_nth

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/list.c:438

目的

ポインタを保持するリストの指定された位置（インデックス）に新しい要素を挿入します。

シグネチャまたは形状

List *list_insert_nth(List *list, int pos, void *datum)

入力と出力

  • **入力**:
  • list: 操作対象の List 構造体へのポインタ。
  • pos: 要素を挿入する位置（0から始まるインデックス）。
  • datum: リストに格納する要素（ポインタ）。
  • **出力**:
  • 挿入後のリストを指す List *。リストが新規作成された場合や、メモリ再配置によりリストのポインタが変更された場合に備え、戻り値を使用する必要があります。

前提条件と不変条件

  • list が NIL の場合、pos は 0 でなければなりません（Assert による検証）。
  • list が NIL でない場合、そのリストはポインタを保持する型（IsPointerList）である必要があります。
  • 操作の完了後、check_list_invariants(list) が呼び出され、リストの内部整合性が検証されます。

アルゴリズムまたは振る舞い

1. list が NIL の場合、list_make1(datum) を呼び出して新しいリストを作成し、それを返します。 2. list が存在する場合、insert_new_cell(list, pos) を呼び出して指定位置に新しいセルを確保します。この際、必要に応じて配列のシフトやメモリの再割り当てが行われます。 3. 確保されたセルに datum を格納します。 4. リストの不変条件をチェックし、リストのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/geqo/geqo_eval.c
  • src/backend/optimizer/util/pathnode.c
  • **呼び出し先**:
  • list_make1
  • insert_new_cell
  • check_list_invariants
  • IsPointerList (マクロ)

関連データ構造

  • List: 動的配列ベースの汎用リスト構造体。
  • ListCell: リストの各要素を保持する構造体。

失敗モードと境界条件

  • pos がリストの現在の長さの範囲外である場合、insert_new_cell 内でエラーが発生する可能性があります。
  • メモリ不足時には palloc を通じてエラーが発生します。
  • リストの要素を指すポインタを保持している場合、本関数による挿入で配列が再配置されると、そのポインタは無効になる可能性があります。

保守担当者向けメモ

  • 本関数はポインタを保持するリスト専用です。整数やOIDを保持するリストには、それぞれの専用関数（list_insert_nth_int 等）を使用してください。
  • リストの挿入操作は配列のシフトを伴うため、計算量は O(N) です。頻繁な挿入が必要な場合はデータ構造の選択を再検討してください。
