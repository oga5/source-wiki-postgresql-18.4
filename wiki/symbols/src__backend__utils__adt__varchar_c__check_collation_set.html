check_collation_set

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varchar.c:726

目的

文字列比較において、有効な照合順序（Collation）が指定されているかを検証し、無効な場合はエラーを報告する。

シグネチャまたは形状

static void check_collation_set(Oid collid)

入力と出力

  • **入力**: collid (Oid) - 検証対象の照合順序ID。
  • **出力**: なし（正常時は何も返さない）。

前提条件と不変条件

  • collid が OidIsValid マクロで真と判定される（有効なOidである）必要がある。

アルゴリズムまたは振る舞い

1. OidIsValid(collid) を評価する。 2. collid が無効な場合、ereport を呼び出し、ERRCODE_INDETERMINATE_COLLATION エラーを発生させる。 3. エラーメッセージには「照合順序を決定できない」旨と、「COLLATE句を使用して明示的に設定せよ」というヒントが含まれる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/varchar.c 内の比較関数等。
  • src/backend/utils/adt/varlena.c 内の複数の関数（varlena.c:150, 1232, 1278, 1637, 1671, 1745, 1800, 1913, 1991 等）。
  • **呼び出し先**:
  • ereport（PostgreSQLのエラー報告メカニズム）。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • collid が無効（InvalidOid）である場合、常に ERRCODE_INDETERMINATE_COLLATION エラーで終了する。これはパーサが暗黙的な照合順序の競合を解決できなかった状況を想定している。

保守担当者向けメモ

  • 本関数は varchar.c と varlena.c の両方から参照されている。共通のユーティリティとして機能しているため、変更時は両ファイルに関連する比較ロジックへの影響を考慮すること。
