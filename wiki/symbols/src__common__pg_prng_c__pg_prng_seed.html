pg_prng_seed

定義

  • 種別: function
  • 定義箇所: src/common/pg_prng.c:88

目的

pg_prng_state 構造体を初期化し、擬似乱数生成器（PRNG）の内部状態を設定する。

シグネチャまたは形状

void pg_prng_seed(pg_prng_state *state, uint64 seed)

入力と出力

  • **入力**:
  • state: 初期化対象となる pg_prng_state 構造体へのポインタ。
  • seed: 乱数生成の基点となる 64 ビットのシード値。
  • **出力**: なし（state が指す構造体が更新される）。

前提条件と不変条件

  • state は有効な pg_prng_state 構造体を指している必要がある。
  • 初期化後、生成器の状態（s0, s1）がすべて 0 になることは許容されない。

アルゴリズムまたは振る舞い

1. splitmix64 関数を2回呼び出し、提供された seed から生成した値を state->s0 および state->s1 に格納する。 2. pg_prng_seed_check(state) を呼び出し、内部状態がすべて 0 になっていないことを検証する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/postmaster.c
  • src/backend/storage/lmgr/s_lock.c
  • src/backend/utils/adt/pseudorandomfuncs.c
  • src/backend/utils/misc/sampling.c
  • src/bin/initdb/initdb.c
  • src/bin/pg_basebackup/pg_createsubscriber.c
  • src/bin/pg_test_fsync/pg_test_fsync.c
  • src/bin/pgbench/pgbench.c
  • **呼び出し先**:
  • splitmix64（内部的な状態展開用）
  • pg_prng_seed_check（不動点回避の検証用）

関連データ構造

  • pg_prng_state: 乱数生成器の状態を保持する構造体。

失敗モードと境界条件

  • seed に 0 を指定した場合でも、splitmix64 による展開と pg_prng_seed_check による検証を経て、安全な初期状態が確保される。

保守担当者向けメモ

  • 本関数は暗号学的に安全な乱数生成を目的としていない。暗号鍵やパスワード生成には使用しないこと。
  • pg_prng_state の全ビットが 0 になると生成器が不動点（0を返し続ける状態）に陥るため、pg_prng_seed_check の呼び出しは必須である。
