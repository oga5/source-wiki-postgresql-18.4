sync_queue_destroy

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/relfilenumber.c:86

目的

pg_upgrade の --swap モードで使用される同期キューを破棄し、割り当てられたメモリを解放する。

シグネチャまたは形状

static inline void sync_queue_destroy(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルな同期キューの状態をリセットする）

前提条件と不変条件

  • sync_queue_inited フラグが true の場合のみ、キューのメモリ解放処理が実行される。
  • 実行後、sync_queue_inited は false に設定され、キューの長さは 0 にリセットされる。

アルゴリズムまたは振る舞い

1. sync_queue_inited が false であれば、何もせず直ちに終了する。 2. sync_queue_inited を false に設定する。 3. sync_queue_len を 0 に設定する。 4. SYNC_QUEUE_MAX_LEN に基づきループし、各要素に対して pfree() を呼び出してメモリを解放し、ポインタを NULL に設定する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_upgrade/relfilenumber.c 内の複数箇所（行 39 および 218）から呼び出されていることが確認されている。
  • 呼び出し先: pfree（メモリ解放関数）。

関連データ構造

  • sync_queue: 同期対象のファイルパスを保持するグローバルな配列。
  • sync_queue_inited: キューの初期化状態を示すフラグ。
  • sync_queue_len: 現在キューに保持されている要素数。
  • SYNC_QUEUE_MAX_LEN: キューの最大容量を定義する定数。

失敗モードと境界条件

  • sync_queue_inited が false の状態で呼び出された場合、安全に早期リターンする。
  • pfree は NULL ポインタに対して安全である必要がある（PostgreSQL のメモリ管理規約に依存）。

保守担当者向けメモ

  • 本関数は static inline で定義されており、relfilenumber.c 内でのみ使用される。
  • sync_queue 関連の操作は --swap モードの整合性に直結するため、メモリ解放後の再初期化フローが正しく維持されていることを確認すること。
