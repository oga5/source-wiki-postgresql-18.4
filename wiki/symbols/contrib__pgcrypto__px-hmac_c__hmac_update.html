hmac_update

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/px-hmac.c:93

目的

HMAC 計算プロセスにおいて、入力データをハッシュ関数へ逐次的に渡して更新を行う。

シグネチャまたは形状

static void hmac_update(PX_HMAC *h, const uint8 *data, unsigned dlen)

入力と出力

  • h: HMAC コンテキスト構造体へのポインタ。
  • data: ハッシュ対象となる入力データのバッファ。
  • dlen: data のバイト長。
  • 戻り値: なし。

前提条件と不変条件

  • h は px_find_hmac によって適切に初期化されている必要がある。
  • 内部のダイジェストコンテキスト（h->md）が有効であること。

アルゴリズムまたは振る舞い

1. 内部のダイジェスト関数インターフェースである px_md_update を呼び出す。 2. h->md が保持するハッシュコンテキストに対して、指定された data と dlen を適用する。

呼び出し元と呼び出し先

  • 呼び出し先: px_md_update（px レイヤーのダイジェスト更新関数）。
  • 呼び出し元: contrib/pgcrypto/px-hmac.c 内の PX_HMAC 構造体の関数ポインタ update として割り当てられている。直接的な外部からの呼び出し元はソースコード上では特定されていない。

関連データ構造

  • PX_HMAC: HMAC 操作の状態を保持する構造体。md メンバを通じてダイジェストコンテキストを管理する。

失敗モードと境界条件

  • h が NULL の場合、あるいは h->md が初期化されていない場合は未定義の動作となる。

保守担当者向けメモ

  • 本関数は PX_HMAC 構造体の update フィールドに代入されるアダプター関数である。
  • 内部で呼び出される px_md_update は、pgcrypto の抽象化レイヤーを通じて具体的なハッシュアルゴリズム（OpenSSL 等）へ処理を委譲する。
