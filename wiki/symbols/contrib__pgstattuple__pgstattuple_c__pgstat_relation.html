pgstat_relation

定義

  • 種別: function
  • 定義箇所: contrib/pgstattuple/pgstattuple.c:242

目的

リレーション（テーブルまたはインデックス）の種別を判定し、統計情報を算出するための適切なスキャン関数へ処理を振り分けるエントリポイントです。

シグネチャまたは形状

static Datum pgstat_relation(Relation rel, FunctionCallInfo fcinfo);

入力と出力

  • **入力**:
  • rel: 統計対象となるリレーションの Relation 構造体。
  • fcinfo: SQL 関数呼び出しのコンテキスト情報。
  • **出力**:
  • 成功時: 統計結果を含む Datum（pgstattuple_type 構造体に基づく）。
  • 失敗時: エラーをスローし、呼び出し元へは戻りません。

前提条件と不変条件

  • 対象リレーションに対して AccessShareLock が保持されている必要があります。
  • 呼び出し側でリレーションの有効性が確認されていることが期待されます。

アルゴリズムまたは振る舞い

1. **一時テーブルチェック**: 他セッションの一時テーブルである場合、誤ったデータを防ぐためエラーを発生させます。 2. **リレーション種別判定**:

  • **テーブル/シーケンス**: pgstat_heap() を呼び出し、ヒープスキャンを実行します。
  • **インデックス**:
  • インデックスが有効（indisvalid）であることを確認します。
  • アクセスメソッド（relam）に基づき、以下のいずれかへディスパッチします：
  • B-tree: pgstat_btree_page を使用して pgstat_index を実行。
  • Hash: pgstat_hash_page を使用して pgstat_index を実行。
  • GiST: pgstat_gist_page を使用して pgstat_index を実行。
  • GIN, SP-GiST, BRIN 等、未サポートのインデックス種別の場合はエラーを発生させます。

    • 3. **その他**: サポート対象外のリレーション種別の場合はエラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: pgstattuple 関連の SQL 関数（pgstattuple_v1_5 等）から呼び出されます。
  • **呼び出し先**:
  • pgstat_heap()
  • pgstat_index()
  • pgstat_btree_page / pgstat_hash_page / pgstat_gist_page

関連データ構造

  • Relation: リレーションのメタデータ。
  • pgstattuple_type: 統計結果を格納する構造体。
  • FunctionCallInfo: SQL 関数呼び出しの引数情報。

失敗モードと境界条件

  • **エラー**:
  • 他セッションの一時テーブルへのアクセス（ERRCODE_FEATURE_NOT_SUPPORTED）。
  • 無効なインデックスへのアクセス（ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE）。
  • 未サポートのインデックス種別やリレーション種別（ERRCODE_FEATURE_NOT_SUPPORTED）。

保守担当者向けメモ

  • 本関数は pgstattuple モジュールのディスパッチャとして機能します。新しいアクセスメソッドのサポートを追加する場合は、switch 文にケースを追加し、対応するページ走査関数を実装してください。
  • pgstat_relation.c（src/backend/utils/activity/）というファイル名が索引に含まれていますが、本シンボルは contrib/pgstattuple/pgstattuple.c 内のローカル関数であり、混同しないよう注意してください。
