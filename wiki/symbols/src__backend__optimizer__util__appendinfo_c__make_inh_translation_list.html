make_inh_translation_list

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/appendinfo.c:79

目的

親リレーションの属性を子リレーションの属性へ変換するためのマッピング情報（translated_vars）および逆変換用配列（parent_colnos）を生成し、AppendRelInfo 構造体に格納する。

シグネチャまたは形状

static void make_inh_translation_list(Relation oldrelation, Relation newrelation,
                                      Index newvarno, AppendRelInfo *appinfo)

入力と出力

  • **入力**:
  • oldrelation: 親リレーションの Relation オブジェクト。
  • newrelation: 子リレーションの Relation オブジェクト。
  • newvarno: 変換後の Var ノードで使用されるリレーションインデックス（varno）。
  • appinfo: 変換結果を格納する AppendRelInfo 構造体へのポインタ。
  • **出力**:
  • appinfo の translated_vars（List）および parent_colnos（AttrNumber 配列）が更新される。

前提条件と不変条件

  • oldrelation と newrelation は有効なリレーションであること。
  • appinfo は呼び出し元でメモリ確保されていること。
  • 継承関係において、対応する属性の型、型修飾子（atttypmod）、および照合順序（attcollation）が一致していること。

アルゴリズムまたは振る舞い

1. newrelation の属性数に基づいて parent_colnos 配列を初期化する。 2. oldrelation の各属性を走査する。

  • 属性が削除済み（attisdropped）の場合は NULL をリストに追加する。
  • oldrelation と newrelation が同一の場合、属性番号をそのまま使用して Var ノードを生成する。
  • それ以外の場合、属性名で newrelation を検索し、対応する属性番号を特定する。

    • 3. 特定した属性の型・照合順序が親と一致するか検証し、一致しない場合はエラーを投げる。 4. 変換後の Var ノードを translated_vars リストに追加し、逆変換用の parent_colnos を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: make_append_rel_info（src/backend/optimizer/util/appendinfo.c 内の関数）。
  • **呼び出し先**: RelationGetDescr, RelationGetRelid, lappend, makeVar, SearchSysCacheAttName, ReleaseSysCache, elog, ereport 等の PostgreSQL 内部 API。

関連データ構造

  • AppendRelInfo: 変換情報を保持する主要な構造体。
  • Relation: リレーションのメタデータ。
  • Var: 変換対象となる変数ノード。

失敗モードと境界条件

  • **エラー**: 継承された属性が newrelation に存在しない場合、または型・照合順序が親と一致しない場合に ERROR を発生させる。
  • **境界条件**: 属性が削除されている場合、その位置には NULL が挿入される。

保守担当者向けメモ

  • 本関数は、ALTER TABLE ADD COLUMN 等による列順序の不一致を考慮し、名前ベースの検索をフォールバックとして備えている。
  • 属性の型チェックは厳格に行われるため、スキーマ変更後の整合性維持に寄与する。
