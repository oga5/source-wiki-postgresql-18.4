gistxlogPageUpdate

定義

  • 種別: struct
  • 定義箇所: src/include/access/gistxlog.h:36

目的

GiSTインデックスページにおけるタプルの挿入および削除操作をWAL（Write-Ahead Logging）に記録するためのデータ構造です。

シグネチャまたは形状

typedef struct gistxlogPageUpdate
{
    uint16      ntodelete;
    uint16      ntoinsert;
} gistxlogPageUpdate;

入力と出力

  • 入力: GiSTインデックスの更新操作時に、削除対象のオフセット数（ntodelete）と挿入対象のタプル数（ntoinsert）が設定されます。
  • 出力: WALレコードの一部として書き込まれます。この構造体の直後に、削除対象の OffsetNumber 配列および挿入対象のタプルデータがペイロードとして続きます。

前提条件と不変条件

  • 本構造体は、GiSTインデックスのページ更新操作の記録に使用されます。
  • 構造体メンバの変更は、WALのバイナリレイアウトに直接影響するため、対応する gist_redo および gist_desc 関数の更新が必須です。

アルゴリズムまたは振る舞い

1. GiSTインデックスのページ更新時、本構造体に削除・挿入数を格納します。 2. XLogRegisterData を介してWALレコードに登録されます。 3. リカバリ時には、XLogRecGetData で本構造体を取得し、後続のペイロードから具体的な削除オフセットと挿入タプルを読み取ってページを再構築します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/gist/gistxlog.c 内のGiSTインデックス更新処理（gistxlogPageUpdate 型の変数が使用されています）。
  • 呼び出し先: src/backend/access/rmgrdesc/gistdesc.c 内の out_gistxlogPageUpdate 関数が、デバッグ出力のために本構造体を参照しています。

関連データ構造

  • gistxlogPageSplit, gistxlogPageDelete, gistxlogPageReuse: 他のGiST WAL操作用構造体。
  • XLogReaderState: WALリプレイ時に本構造体を含むレコードを読み取るために使用されます。

失敗モードと境界条件

  • WALレコードのペイロードとして定義された順序（削除オフセット、次に挿入タプル）と、構造体メンバの整合性が取れていない場合、リプレイ時にデータ破損を引き起こします。

保守担当者向けメモ

  • 本構造体はバイナリレイアウトと密接に関連しています。メンバの追加や変更を行う際は、src/backend/access/gist/gistxlog.c のリプレイロジックおよび src/backend/access/rmgrdesc/gistdesc.c の記述ロジックを必ず同期させてください。
