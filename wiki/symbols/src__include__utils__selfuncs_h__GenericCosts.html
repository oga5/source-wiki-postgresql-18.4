GenericCosts

定義

  • 種別: struct
  • 定義箇所: src/include/utils/selfuncs.h:126

目的

クエリプランナがインデックススキャンコストを見積もる際、計算結果を保持および伝達するために使用される構造体です。

シグネチャまたは形状

typedef struct {
    Cost        indexStartupCost;
    Cost        indexTotalCost;
    Selectivity indexSelectivity;
    double      indexCorrelation;
    double      numIndexPages;
    double      numIndexTuples;
    double      spc_random_page_cost;
    double      num_sa_scans;
} GenericCosts;

入力と出力

  • **入力**: コスト見積もり関数（genericcostestimate() 等）が計算過程で算出する中間値および最終的なコスト指標。
  • **出力**: プランナに対して返されるインデックスの起動コスト、総コスト、選択率、および順序相関。

前提条件と不変条件

  • 呼び出し元は、使用前に構造体の全フィールドをゼロ初期化する必要があります。
  • indexSelectivity は 0.0 から 1.0 の範囲である必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: 呼び出し元が構造体をゼロで初期化します。 2. **計算**: コスト見積もり関数が、インデックスの特性（ページ数、タプル数）や統計情報に基づき、中間値（numIndexPages 等）を算出します。 3. **結果格納**: 最終的なコスト見積もり値（indexStartupCost, indexTotalCost 等）を構造体に書き込み、プランナへ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/selfuncs.c 内の各コスト見積もり関数、および contrib/bloom/blcost.c。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • Cost (型定義)
  • Selectivity (型定義)

失敗モードと境界条件

  • 統計情報が不足している場合、デフォルト値が使用されます。
  • ゼロ初期化を怠った場合、未定義のコスト値がプランナに渡され、誤ったプラン選択を招く可能性があります。

保守担当者向けメモ

  • 本構造体は genericcostestimate() 等のインターフェースと密接に関連しています。フィールドを追加・変更する際は、関連するコスト見積もり関数の実装をすべて確認してください。
