cstr2sv

定義

  • 種別: static inline function
  • 定義箇所: src/pl/plperl/plperl.h:146

目的

PostgreSQL のデータベースエンコーディングで表現された C 文字列を、Perl が扱える UTF-8 フラグ付きの SV (Scalar Value) に変換します。

シグネチャまたは形状

SV *cstr2sv(const char *str)

入力と出力

  • **入力**: const char *str (PostgreSQL データベースエンコーディングのヌル終端文字列)
  • **出力**: SV * (Perl の UTF-8 フラグが有効化されたスカラー値)

前提条件と不変条件

  • dTHX マクロにより、Perl インタプリタコンテキストが適切に設定されている必要があります。
  • データベースエンコーディングが SQL_ASCII 以外の場合、内部で utf_e2u を呼び出し、変換後のメモリを pfree で解放します。

アルゴリズムまたは振る舞い

1. 現在のデータベースエンコーディングを確認します。 2. エンコーディングが PG_SQL_ASCII の場合、変換を行わず newSVpv でそのまま SV を作成して返します。 3. それ以外の場合:

  • utf_e2u を呼び出し、文字列を UTF-8 に変換します。
  • newSVpv で SV を作成します。
  • SvUTF8_on を呼び出し、Perl 側に当該 SV が UTF-8 であることを通知します。
  • 変換用の一時バッファを pfree で解放します。

    • 4. 生成された SV を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore_plperl/hstore_plperl.c
  • contrib/jsonb_plperl/jsonb_plperl.c
  • src/pl/plperl/plperl.c
  • **呼び出し先**:
  • GetDatabaseEncoding()
  • newSVpv()
  • utf_e2u()
  • SvUTF8_on()
  • pfree()

関連データ構造

  • SV (Perl のスカラー値構造体)

失敗モードと境界条件

  • 入力文字列 str が NULL の場合の挙動は、newSVpv および utf_e2u の実装に依存します。
  • メモリ不足時には palloc (内部で使用) により PostgreSQL のエラーハンドラが呼び出されます。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、plperl.c 以外のモジュールからも利用されます。
  • SQL_ASCII 時の最適化により、エンコーディング変換コストを回避しています。
  • utf_e2u が返すメモリは palloc 経由で確保されるため、必ず pfree で解放する必要があります。
