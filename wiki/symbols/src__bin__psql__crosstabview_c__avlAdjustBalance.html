avlAdjustBalance

定義

  • 種別: function
  • 定義箇所: src/bin/psql/crosstabview.c:505

目的

AVL木のノードのバランス係数を評価し、必要に応じて回転操作（単回転または二重回転）を実行して木の平衡を維持する。

シグネチャまたは形状

static void avlAdjustBalance(avl_tree *tree, avl_node **node)

入力と出力

  • tree: 操作対象の avl_tree 構造体へのポインタ。
  • node: バランス調整を行う対象ノードへのポインタのポインタ。

前提条件と不変条件

  • node が指すノードは、avl_tree 内の有効なノードであること。
  • avlBalance 関数がノードのバランス係数を正しく算出できること。

アルゴリズムまたは振る舞い

1. avlBalance を用いて現在のノードのバランス係数 b を取得し、2で除算する。 2. b が 0 でない場合（不均衡な場合）:

  • 不均衡の方向 dir を決定する。
  • 子ノードのバランス係数が b と逆符号であれば、まず子ノードに対して avlRotate を実行する。
  • 現在のノードに対して avlRotate を実行し、回転後のノードを更新する。

    • 3. 現在のノードが tree->end（番兵）でない場合、avlUpdateHeight を呼び出してノードの高さを再計算する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/crosstabview.c 内の avlInsertNode 等から呼び出される（ソースコード上の直接的な参照箇所として src/bin/psql/crosstabview.c:553 が確認されている）。
  • **呼び出し先**: avlBalance, avlRotate, avlUpdateHeight。

関連データ構造

  • avl_tree: AVL木の管理構造体。
  • avl_node: 木を構成するノード構造体。

失敗モードと境界条件

  • node が指す先が NULL の場合、avlBalance 呼び出し時に未定義の動作を引き起こす可能性がある。
  • tree->end との比較により、番兵ノードに対する高さ更新を回避している。

保守担当者向けメモ

  • 本関数は挿入専用の AVL 実装の一部である。削除操作には対応していない。
  • avlRotate の戻り値を使用して current ポインタを更新し、その後の高さ更新に反映させている点に注意すること。
