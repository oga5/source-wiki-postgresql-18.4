tsvector_update_trigger_bycolumn

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector_op.c:2736

目的

テーブルの特定のカラム群を基に、tsvector 型の値を自動生成または更新するためのトリガー関数を提供します。

シグネチャまたは形状

Datum tsvector_update_trigger_bycolumn(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（トリガー実行コンテキスト fcinfo を含む）。
  • **出力**: Datum（更新後の tsvector 値、またはトリガーの戻り値として適切な形式）。

前提条件と不変条件

  • 本関数はトリガーとして呼び出されることを前提としています。
  • 内部的に tsvector_update_trigger を呼び出すため、同関数の制約（tsvector の位置情報制限など）に準拠します。

アルゴリズムまたは振る舞い

1. fcinfo を引数として tsvector_update_trigger を呼び出します。 2. 第2引数に true を渡し、カラム指定による更新モードであることを明示して処理を委譲します。 3. tsvector_update_trigger の戻り値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のトリガーマネージャ（SQL レベルでのトリガー定義経由）。
  • **呼び出し先**: tsvector_update_trigger（src/backend/utils/adt/tsvector_op.c 内）。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約で使用される引数コンテキスト。
  • tsvector: 全文検索用の可変長データ型。

失敗モードと境界条件

  • トリガーとして正しく設定されていない場合や、対象カラムのデータ型が tsvector 生成に適さない場合はエラーが発生する可能性があります。
  • 詳細は tsvector_update_trigger の実装に依存します。

保守担当者向けメモ

  • 本関数は tsvector_update_trigger への単なるラッパーです。ロジックの変更が必要な場合は、呼び出し先の tsvector_update_trigger を確認してください。
