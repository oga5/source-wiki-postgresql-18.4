unaccent_init

定義

  • 種別: function
  • 定義箇所: contrib/unaccent/unaccent.c:337

目的

PostgreSQL テキスト検索辞書 unaccent の初期化を行い、設定ファイルから置換ルールを読み込んで Trie 木構造を構築する。

シグネチャまたは形状

Datum unaccent_init(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される List *dictoptions（辞書設定パラメータ）。
  • **出力**: Datum 型として、構築された TrieChar 構造体のポインタを返す。

前提条件と不変条件

  • dictoptions 内には "rules" パラメータが必須であり、これが存在しない場合はエラーとなる。
  • "rules" パラメータは重複して指定できない。
  • 未知のパラメータが指定された場合はエラーとなる。

アルゴリズムまたは振る舞い

1. dictoptions リストを走査し、各要素を確認する。 2. "rules" パラメータが見つかった場合、その値を引数として initTrie() を呼び出し、Trie 木を生成する。 3. 既に "rules" が読み込まれている状態で再度 "rules" が指定された場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告する。 4. "rules" 以外のパラメータが指定された場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告する。 5. 全ての走査終了後、"rules" が読み込まれていなければエラーを報告する。 6. 構築した rootTrie を PG_RETURN_POINTER で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のテキスト検索システム（辞書初期化時）。
  • **呼び出し先**: initTrie()。

関連データ構造

  • TrieChar: 置換ルールを保持する Trie 木のノード。
  • List: 設定オプションを保持する PostgreSQL のリスト構造。
  • DefElem: 設定オプションの各要素。

失敗モードと境界条件

  • **ルールファイル欠落**: "rules" パラメータが指定されていない場合、エラー終了する。
  • **重複定義**: "rules" パラメータが複数回指定された場合、エラー終了する。
  • **不正なパラメータ**: "rules" 以外の未知のパラメータが指定された場合、エラー終了する。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_INFO_V1(unaccent_init) マクロにより、PostgreSQL の関数インターフェースとして登録されている。
  • initTrie() が内部でファイル読み込みと解析を行うため、ルールファイルのパスや形式に関する変更は initTrie() 側を確認すること。
