newnfa

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_nfa.c:46

目的

正規表現コンパイルプロセスにおいて、NFA（非決定性有限オートマトン）の初期構造を生成・初期化する。

シグネチャまたは形状

struct nfa *newnfa(struct vars *v, struct colormap *cm, struct nfa *parent)

入力と出力

  • **入力**:
  • v: コンパイル状態を保持する vars 構造体。
  • cm: カラーマップ。
  • parent: 親となる NFA（サブパターン等の場合）。プライマリ NFA の場合は NULL。
  • **出力**: 初期化された struct nfa へのポインタ。メモリ確保失敗時やエラー発生時は NULL を返す。

前提条件と不変条件

  • MALLOC によるメモリ確保が成功すること。
  • freenfa() が安全に呼び出せるよう、構造体の各ポインタフィールドは初期化時に NULL または有効な状態に設定される。

アルゴリズムまたは振る舞い

1. nfa 構造体用のメモリを確保し、各フィールドを初期化する。 2. 必須のインフラストラクチャとして、post（終了用）、pre（開始用）、init（初期）、final（最終）の各状態を作成する。 3. pre から init へ、および final から post へ、PLAIN 遷移（rainbow）および制約遷移（^ および $）を追加する。 4. 処理中にエラー（ISERR()）が発生した場合は、freenfa(nfa) を呼び出してリソースを解放し、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regcomp.c（プライマリ NFA の作成およびサブパターン用 NFA の作成時に呼び出し）
  • **呼び出し先**:
  • newfstate()
  • newstate()
  • rainbow()
  • newarc()
  • freenfa()
  • ERR() / ISERR()（マクロ）

関連データ構造

  • struct nfa: NFA 全体を管理する構造体。
  • struct vars: コンパイル時のコンテキスト情報。
  • struct colormap: 遷移条件（色）の管理。

失敗モードと境界条件

  • メモリ不足時: MALLOC が失敗した場合、REG_ESPACE エラーを記録し NULL を返す。
  • 構築中のエラー: newfstate や newstate 等の内部処理でエラーが発生した場合、freenfa でクリーンアップを行い NULL を返す。

保守担当者向けメモ

  • newfstate は newstate よりも先に呼び出され、parent が正しく設定される必要がある。
  • nfa->pre と nfa->post はそれぞれ状態番号 1 と 0 に割り当てられる。
  • 状態や遷移のメモリ管理はバッチ割り当て方式であり、個別の free は行わない設計となっている。
