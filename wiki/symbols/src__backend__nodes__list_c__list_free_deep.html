list_free_deep

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/list.c:1559

目的

リスト構造体自体と、そのリストが保持する各要素（ポインタ）が指し示すメモリ領域の両方を解放する。

シグネチャまたは形状

void list_free_deep(List *list)

入力と出力

  • **入力**: list (解放対象の List 構造体へのポインタ)
  • **出力**: なし（void）

前提条件と不変条件

  • list はポインタを保持するリスト（IsPointerList(list) が真）でなければならない。これに違反する場合、アサーションエラーが発生する。

アルゴリズムまたは振る舞い

1. IsPointerList(list) を用いて、対象がポインタリストであることを検証する。 2. list_free_private(list, true) を呼び出す。この際、第2引数に true を渡すことで、リスト内の各要素に対して再帰的な解放処理（深い解放）を指示する。

  • **呼び出し先**:
  • list_free_private

関連データ構造

  • List: PostgreSQLの配列ベースの動的リスト構造体。

失敗モードと境界条件

  • list が NULL の場合の挙動は list_free_private の実装に依存する。
  • ポインタリスト以外のリスト（整数リストやOIDリストなど）に対して呼び出した場合、アサーションが有効なビルドではプログラムが停止する。

保守担当者向けメモ

  • 本関数はポインタのリスト専用である。リスト内の各要素が指すメモリ領域が、現在のメモリコンテキストにおいて解放可能であることを前提としている。
