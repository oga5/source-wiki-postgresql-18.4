stats_check_arg_type

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/stat_utils.c:268

目的

統計情報の更新関数において、指定された引数のデータ型が期待される型と一致しているかを検証します。

シグネチャまたは形状

static bool stats_check_arg_type(const char *argname, Oid argtype, Oid expectedtype)

入力と出力

  • **入力**:
  • argname: 検証対象の引数名（エラーメッセージ用）。
  • argtype: 実際の引数の型OID。
  • expectedtype: 期待される型OID。
  • **出力**:
  • bool: 型が一致すれば true、一致しなければ false を返します。

前提条件と不変条件

  • argtype および expectedtype は有効な型OIDである必要があります。

アルゴリズムまたは振る舞い

1. argtype と expectedtype を比較します。 2. 両者が一致しない場合、WARNING レベルの ereport を発行し、引数名と実際の型、期待される型をログに出力します。 3. 一致しない場合は false を返し、一致する場合は true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/stat_utils.c 内の統計情報更新処理（引数の型検証ロジック）から呼び出されます。
  • **呼び出し先**: format_type_be（型OIDを文字列に変換するために使用）、ereport。

関連データ構造

  • Oid: PostgreSQLのシステムカタログにおけるオブジェクト識別子。

失敗モードと境界条件

  • 型が一致しない場合、処理を中断させるエラー（ERROR）ではなく、警告（WARNING）を発行して呼び出し元へ制御を戻します。

保守担当者向けメモ

  • 本関数は統計情報のバリデーション層の一部であり、型不一致を検出した際にシステム全体を停止させるのではなく、警告ログを出力して検証を継続する設計となっています。
