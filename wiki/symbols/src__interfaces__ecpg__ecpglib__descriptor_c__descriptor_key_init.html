descriptor_key_init

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/descriptor.c:33

目的

スレッドローカルストレージ（TLS）で記述子リストを管理するための pthread_key_t を初期化します。

シグネチャまたは形状

static void descriptor_key_init(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として descriptor_key が初期化される）

前提条件と不変条件

  • pthread_once を介して一度だけ実行されることが前提です。
  • 実行後、descriptor_key はスレッド終了時に descriptor_destructor を呼び出すように設定されます。

アルゴリズムまたは振る舞い

1. pthread_key_create を呼び出し、グローバル変数 descriptor_key を初期化します。 2. その際、スレッド終了時に記述子のメモリを解放するためのデストラクタとして descriptor_destructor を登録します。

呼び出し元と呼び出し先

  • 呼び出し元: pthread_once(&descriptor_once, descriptor_key_init); （src/interfaces/ecpg/ecpglib/descriptor.c:42）
  • 呼び出し先: pthread_key_create

関連データ構造

  • descriptor_key: スレッドごとの記述子リストを保持するための pthread_key_t 型のグローバル変数。
  • descriptor_destructor: スレッド終了時に呼び出されるクリーンアップ関数。

失敗モードと境界条件

  • pthread_key_create が失敗した場合の挙動は、POSIX スレッドライブラリの仕様に依存します。

保守担当者向けメモ

  • 本関数はスレッドセーフな記述子管理の基盤です。pthread_once による初期化パターンが崩れないよう注意してください。
