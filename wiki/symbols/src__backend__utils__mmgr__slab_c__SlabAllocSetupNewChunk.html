SlabAllocSetupNewChunk

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/slab.c:497

目的

SLAB アロケータにおいて、新しく割り当てられたチャンクのヘッダーを初期化し、メモリコンテキストのメタデータとして登録する。

シグネチャまたは形状

static inline void *SlabAllocSetupNewChunk(MemoryContext context, SlabBlock *block, MemoryChunk *chunk, Size size)

入力と出力

  • **入力**:
  • context: 対象のメモリコンテキスト。
  • block: チャンクが属する SlabBlock。
  • chunk: 初期化対象の MemoryChunk ポインタ。
  • size: 要求されたメモリサイズ。
  • **出力**:
  • 初期化されたチャンクのデータ領域を指すポインタ（void *）。

前提条件と不変条件

  • chunk が指定された block 内の有効な範囲に存在し、かつアライメントが正しいこと（Assert による検証）。
  • slab->chunkSize がチャンクの全サイズからヘッダーサイズを引いた値よりも小さいこと（MEMORY_CONTEXT_CHECKING 有効時）。

アルゴリズムまたは振る舞い

1. **検証**: チャンクがブロック内の境界内にあり、適切にアライメントされているかを確認する。 2. **ヘッダー初期化**: MemoryChunkSetHdrMask を使用して、チャンクヘッダーにブロック情報、サイズ、SLAB ID を設定する。 3. **デバッグ支援**:

  • MEMORY_CONTEXT_CHECKING が有効な場合、要求サイズを記録し、未使用領域に番兵（sentinel）を配置する。
  • RANDOMIZE_ALLOCATED_MEMORY が有効な場合、割り当て領域をジャンクデータで埋める。

    • 4. **アクセス制御**: Valgrind を使用して、チャンクヘッダー領域へのアクセスを禁止する。 5. **返却**: チャンクのデータ領域へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/slab.c:594 (SlabAlloc 内)
  • src/backend/utils/mmgr/slab.c:694 (SlabAlloc 内)
  • **呼び出し先**:
  • SlabBlockGetChunk (マクロ)
  • SlabChunkMod (マクロ)
  • MemoryChunkSetHdrMask
  • MemoryChunkGetPointer
  • set_sentinel (デバッグ時)
  • randomize_mem (デバッグ時)
  • VALGRIND_MAKE_MEM_UNDEFINED / VALGRIND_MAKE_MEM_NOACCESS

関連データ構造

  • SlabContext: SLAB アロケータの管理構造体。
  • SlabBlock: メモリブロックの管理構造体。
  • MemoryChunk: チャンクのヘッダー構造体。

失敗モードと境界条件

  • チャンクがブロックの境界外にある場合や、アライメントが不正な場合は Assert により異常終了する。

保守担当者向けメモ

  • 本関数は static inline で定義されており、SlabAlloc のパフォーマンスを最適化するためにインライン化を意図している。
  • MEMORY_CONTEXT_CHECKING が有効な場合、メモリの二重解放や境界外書き込みを検出するための追加処理が実行される。
