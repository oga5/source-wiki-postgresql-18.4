InitializeBackupManifest

定義

  • 種別: function
  • 定義箇所: src/backend/backup/backup_manifest.c:56

目的

バックアップ・マニフェスト生成プロセスの初期化を行い、JSON形式のヘッダー情報を一時ファイルへ書き込みます。

シグネチャまたは形状

void InitializeBackupManifest(backup_manifest_info *manifest,
                              backup_manifest_option want_manifest,
                              pg_checksum_type manifest_checksum_type)

入力と出力

  • **入力**:
  • manifest: 初期化対象の backup_manifest_info 構造体へのポインタ。
  • want_manifest: マニフェスト生成の要否およびエンコードオプション。
  • manifest_checksum_type: ファイルチェックサムに使用するアルゴリズム。
  • **出力**: なし（manifest 構造体の各フィールドが初期化されます）。

前提条件と不変条件

  • manifest ポインタは有効なメモリ領域を指している必要があります。
  • want_manifest が MANIFEST_OPTION_NO でない場合、一時ファイル（BufFile）およびSHA-256コンテキストが作成されます。

アルゴリズムまたは振る舞い

1. manifest 構造体をゼロクリアし、指定されたチェックサムタイプを設定します。 2. マニフェスト生成が有効な場合:

  • BufFileCreateTemp で一時ファイルを作成します。
  • pg_cryptohash_create でSHA-256コンテキストを初期化します。

    • 3. マニフェストのメタデータ（manifest_size, first_file 等）を初期化します。 4. マニフェスト生成が有効な場合、JSONの開始部分（バージョン、システム識別子、Files 配列の開始）を一時ファイルに書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup.c:259
  • **呼び出し先**:
  • BufFileCreateTemp
  • pg_cryptohash_create / pg_cryptohash_init
  • GetSystemIdentifier
  • AppendToManifest

関連データ構造

  • backup_manifest_info: マニフェスト生成の状態を保持する構造体。
  • backup_manifest_option: マニフェスト生成のオプション（列挙型）。

失敗モードと境界条件

  • pg_cryptohash_init が失敗した場合、elog(ERROR) により処理が中断されます。
  • want_manifest が MANIFEST_OPTION_NO の場合、一時ファイルやハッシュコンテキストは作成されず、マニフェストの書き出しも行われません。

保守担当者向けメモ

  • マニフェストのJSON構造は手動で構築されているため、AppendToManifest を介した書き込み順序がJSONの妥当性に直結します。
  • マニフェスト全体の整合性を保証するため、SHA-256コンテキストが初期化されている点に注意してください。
