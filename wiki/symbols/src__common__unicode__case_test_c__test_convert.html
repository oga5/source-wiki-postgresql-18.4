test_convert

定義

  • 種別: function
  • 定義箇所: src/common/unicode/case_test.c:207

目的

Unicode 大文字・小文字変換関数の堅牢性を検証するため、ソース文字列と宛先バッファの NULL 終端の有無を組み合わせた 4 通りの境界条件で変換結果をテストする。

シグネチャまたは形状

static void test_convert(TestFunc tfunc, const char *test_string, const char *expected)

入力と出力

  • tfunc: 実行する変換関数（TestFunc 型）。
  • test_string: テスト対象の入力文字列。
  • expected: 期待される変換後の文字列。
  • 出力: 戻り値なし。検証失敗時は printf でエラーを出力し exit(1) で終了する。

前提条件と不変条件

  • tfunc は、dst バッファ、dst サイズ、src 文字列、src 長さを引数に取り、必要なバッファサイズを返す関数であること。
  • test_string および expected は有効な文字列であること。

アルゴリズムまたは振る舞い

以下の 4 つのバッファ状態パターンを順次実行し、変換関数 tfunc の戻り値（必要なサイズ）と変換結果を検証する。 1. **非 NULL 終端ソース / 非 NULL 終端宛先**: src1len を指定し、dst1 に変換。 2. **非 NULL 終端ソース / NULL 終端宛先**: src1len を指定し、dst2 に変換。 3. **NULL 終端ソース / 非 NULL 終端宛先**: src2len (-1) を指定し、dst1 に変換。 4. **NULL 終端ソース / NULL 終端宛先**: src2len (-1) を指定し、dst2 に変換。

各ステップで needed 値が strlen(expected) と一致するかを確認し、一致しない場合は異常終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: test_convert_case 関数（src/common/unicode/case_test.c 内）。
  • **呼び出し先**: 引数として渡された tfunc（tfunc_lower, tfunc_upper, tfunc_title, tfunc_fold など）。

関連データ構造

  • TestFunc: 変換関数を抽象化する関数ポインタ型。

失敗モードと境界条件

  • 変換結果が期待値と異なる場合、または必要なバッファサイズが strlen(expected) と異なる場合に exit(1) で終了する。
  • バッファの初期化に 0x7F を使用しており、変換関数がバッファを正しく上書きしているかを検証する。

保守担当者向けメモ

  • 本関数はメモリを動的に確保・解放するため、テストケース追加時はメモリリークに注意すること。
  • src2len = -1 は、関数内部で strlen を使用して長さを決定させるための慣習的な指定である。
