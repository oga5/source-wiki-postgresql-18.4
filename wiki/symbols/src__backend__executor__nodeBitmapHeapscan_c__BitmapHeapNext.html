BitmapHeapNext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapHeapscan.c:125

目的

ビットマップヒープスキャンにおいて、TIDBitmap を走査し、ヒープから条件に合致する次のタプルをフェッチして返す。

シグネチャまたは形状

static TupleTableSlot *BitmapHeapNext(BitmapHeapScanState *node)

入力と出力

  • **入力**: BitmapHeapScanState *node （スキャンノードの状態構造体）
  • **出力**: 条件を満たす TupleTableSlot *。スキャン終了時は ExecClearTuple(slot) の結果を返す。

前提条件と不変条件

  • node->initialized が偽の場合、内部で BitmapTableScanSetup が呼び出され、スキャンが初期化される。
  • MVCC 準拠のスナップショットが使用されている必要がある。

アルゴリズムまたは振る舞い

1. 未初期化であれば BitmapTableScanSetup を実行する。 2. table_scan_bitmap_next_tuple を呼び出し、ビットマップに従ってヒープからタプルを順次取得する。 3. 取得ごとに CHECK_FOR_INTERRUPTS() を実行し、割り込みを確認する。 4. ビットマップが「損失あり（lossy）」である場合（node->recheck が真）、ExecQualAndReset を用いてタプルが条件を満たすか再検証する。

  • 再検証に失敗した場合は InstrCountFiltered2 を更新し、タプルを破棄して次の反復へ進む。

    • 5. 条件を満たすタプルが見つかった場合はそのスロットを返し、ビットマップの走査が完了した場合は空のスロットを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていませんが、ExecScanAccessMtd 経由で実行エンジンから呼び出されます。
  • **呼び出し先**:
  • BitmapTableScanSetup
  • table_scan_bitmap_next_tuple
  • ExecQualAndReset
  • InstrCountFiltered2
  • ExecClearTuple

関連データ構造

  • BitmapHeapScanState: スキャンノードの実行状態を保持する構造体。
  • TIDBitmap: インデックススキャンによって生成されたタプルIDの集合。
  • TupleTableSlot: タプルデータを保持するスロット。

失敗モードと境界条件

  • ビットマップが空、または走査対象のヒープページに条件を満たすタプルが存在しない場合、スキャンは終了し空のスロットを返す。
  • 損失のあるビットマップにおいて、すべてのタプルが再検証（recheck）に失敗した場合、スキャンは継続して次のタプルを探す。

保守担当者向けメモ

  • 本関数は table_scan_bitmap_next_tuple を介してテーブルアクセスメソッド（AM）と対話します。
  • 並列実行時、共有状態の同期は BitmapTableScanSetup 内で管理されますが、本関数は取得したビットマップの反復処理に専念します。
