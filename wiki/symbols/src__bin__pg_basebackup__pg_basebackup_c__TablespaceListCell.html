TablespaceListCell

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_basebackup/pg_basebackup.c:43

目的

ユーザーが指定したテーブルスペースの配置変更（マッピング）情報を保持するための連結リストのノードとして機能します。

シグネチャまたは形状

typedef struct TablespaceListCell
{
	struct TablespaceListCell *next;
	char		old_dir[MAXPGPATH];
	char		new_dir[MAXPGPATH];
} TablespaceListCell;

入力と出力

  • 入力: なし（データ構造定義）
  • 出力: なし

前提条件と不変条件

  • old_dir および new_dir は MAXPGPATH のサイズ制限に従う必要があります。
  • 連結リストの末尾は next が NULL である必要があります。

アルゴリズムまたは振る舞い

  • 連結リストのノードとして、old_dir（元のパス）から new_dir（新しいパス）へのマッピング情報を格納します。
  • pg_malloc0 を使用してメモリ確保されることが想定されています。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_basebackup/pg_basebackup.c 内のテーブルスペースマッピング処理箇所（pg_malloc0 によるインスタンス化など）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • TablespaceList: 本構造体をリストとして管理するための構造体（head および tail ポインタを保持）。

失敗モードと境界条件

  • メモリ不足時: pg_malloc0 が失敗した場合、プログラムは終了します。
  • パス長制限: MAXPGPATH を超えるパスが指定された場合、バッファオーバーフローのリスクがあります。

保守担当者向けメモ

  • 本構造体は pg_basebackup におけるテーブルスペースの再配置機能の基盤です。リストの走査を行う際は、next ポインタのヌルチェックを確実に行ってください。
