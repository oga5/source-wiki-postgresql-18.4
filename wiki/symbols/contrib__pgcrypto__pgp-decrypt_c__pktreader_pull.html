pktreader_pull

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-decrypt.c:174

目的

PGP パケットストリームからデータを読み取り、必要に応じてパケットヘッダーの解析とストリームの切り替えを行うフィルタ関数です。

シグネチャまたは形状

static int pktreader_pull(void *priv, PullFilter *src, int len, uint8 **data_p, uint8 *buf, int buflen);

入力と出力

  • **入力**:
  • priv: PktData 構造体へのポインタ。現在のパケット状態を保持。
  • src: 読み取り元の PullFilter。
  • len: 要求するデータ長。
  • data_p, buf, buflen: 読み取りバッファおよびデータポインタ。
  • **出力**:
  • 成功時: 読み取ったバイト数（正の整数）。
  • 終了時: 0。
  • 失敗時: 負の整数（エラーコード）。

前提条件と不変条件

  • priv は struct PktData 型としてキャスト可能であること。
  • pkt->type が PKT_CONTEXT の場合、パケット境界を意識せず src から直接読み取りを行う。
  • pkt->len が 0 の場合、ストリームの次のチャンクを解析して長さを更新する。

アルゴリズムまたは振る舞い

1. pkt->type が PKT_CONTEXT ならば、pullf_read を呼び出して終了。 2. pkt->len が 0 の場合、ループ内で parse_new_len を呼び出し、次のチャンクの長さを取得する。

  • pkt->type が PKT_NORMAL であれば、ストリームの終端として 0 を返す。

    • 3. 要求された len が現在のパケット長 pkt->len を超える場合、len を pkt->len に切り詰める。 4. pullf_read を呼び出し、データを読み取る。 5. 読み取り成功時、読み取った分だけ pkt->len を減算し、読み取ったバイト数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: pgp_create_pkt_reader 内でフィルタ構造体の一部として登録される。
  • **呼び出し先**: pullf_read（フィルタチェーンの読み取り）、parse_new_len（パケット長解析）。

関連データ構造

  • PktData: パケットの型と残りの長さを保持する構造体。
  • PullFilter: データの逐次読み取りを行うための抽象化構造体。

失敗モードと境界条件

  • parse_new_len が負の値を返した場合、そのエラーコードをそのまま伝播させる。

保守担当者向けメモ

  • 本関数は PullFilter のコールバックとして機能する。
  • pkt->len の管理は、パケットの断片化（チャンク）を透過的に扱うための重要なロジックである。
  • PKT_CONTEXT モードは、パケット構造を無視してストリームをそのまま通す特殊なケースとして利用される。
