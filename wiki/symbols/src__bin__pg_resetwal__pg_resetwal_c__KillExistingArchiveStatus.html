KillExistingArchiveStatus

定義

  • 種別: function
  • 定義箇所: src/bin/pg_resetwal/pg_resetwal.c:1038

目的

WAL アーカイブ状態ディレクトリ（pg_wal/archive_status）内の古いアーカイブステータスファイルを削除し、リセット後の整合性を確保する。

シグネチャまたは形状

static void KillExistingArchiveStatus(void)

入力と出力

  • 入力: なし（XLOGDIR "/archive_status" を固定パスとして使用）。
  • 出力: なし（成功時は終了、失敗時は pg_fatal によりプロセスを終了）。

前提条件と不変条件

  • XLOGDIR が正しく定義されており、対象ディレクトリが存在すること。
  • プロセスが対象ディレクトリに対する読み取りおよび削除権限を有していること。

アルゴリズムまたは振る舞い

1. pg_wal/archive_status ディレクトリを開く。 2. ディレクトリ内の全エントリを走査する。 3. ファイル名が WAL セグメント形式（16桁の16進数）であり、かつ拡張子が .ready, .done, .partial.ready, .partial.done のいずれかであるファイルを特定する。 4. 特定したファイルを unlink で削除する。 5. 走査完了後、ディレクトリを閉じる。いずれの操作においてもエラーが発生した場合は pg_fatal で異常終了する。

呼び出し元と呼び出し先

  • 呼び出し元: main 関数（src/bin/pg_resetwal/pg_resetwal.c:523）から直接呼び出される。
  • 呼び出し先: opendir, readdir, closedir, unlink, pg_fatal, snprintf, strspn, strcmp 等の標準ライブラリおよびユーティリティ関数。

関連データ構造

  • DIR, struct dirent: ディレクトリ走査に使用。
  • MAXPGPATH: パス文字列のバッファサイズ定義。

失敗モードと境界条件

  • ディレクトリが開けない場合、読み込めない場合、またはファイルの削除に失敗した場合は pg_fatal によりエラーメッセージを出力して終了する。
  • 該当するファイルが存在しない場合は、何も削除せず正常に終了する。

保守担当者向けメモ

  • 本関数は pg_resetwal の破壊的更新フェーズの一部として実行される。
  • 削除対象の拡張子リストに変更がある場合は、strcmp の条件式を更新する必要がある。
