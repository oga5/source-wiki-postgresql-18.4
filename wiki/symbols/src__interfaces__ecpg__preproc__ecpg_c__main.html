main

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/ecpg.c:129

目的

PostgreSQL 埋め込み SQL プリプロセッサ（ecpg）のメインエントリポイントであり、コマンドライン引数の解析、環境設定、および入力ファイルに対するプリプロセス処理の制御を行う。

シグネチャまたは形状

int main(int argc, char *const argv[])

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。
  • オプション: -c (Cコード自動生成), -C (互換モード), -d (デバッグ), -D (定義追加), -h (ヘッダーモード), -i (システムインクルード), -I (インクルードパス), -o (出力先), -r (実行時オプション), -t (自動コミット), -v (詳細表示)。
  • **出力**:
  • 戻り値: 成功時は 0、エラー時は ILLEGAL_OPTION 等のステータスコード。
  • 副作用: 指定された入力ファイルを解析し、C ソースコードを生成（または標準出力へ出力）。

前提条件と不変条件

  • 実行環境において find_my_exec が実行可能であること。
  • optind 以降の引数は入力ファイル名として解釈される。
  • エラー発生時、生成中の出力ファイルは unlink により削除される（不完全なファイルの残存防止）。

アルゴリズムまたは振る舞い

1. **初期化**: ロケール設定、実行パスの特定、プログラム名の取得を行う。 2. **引数解析**: getopt_long を使用し、オプションをグローバル変数（compat, include_paths, auto_create_c 等）に反映する。 3. **パス設定**: デフォルトのインクルードパス（., /usr/local/include, /usr/include 等）を登録する。 4. **詳細モード**: -v 指定時は設定されたインクルードパスを表示して終了する。 5. **ファイル処理ループ**:

  • 引数で指定された各ファイルに対し、base_yyin（入力）と base_yyout（出力）を準備する。
  • base_yyparse() を呼び出し、構文解析を実行する。
  • 解析終了後、ファイルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: OS（プログラム実行時）。
  • **呼び出し先**: getopt_long, help, add_include_path, add_preprocessor_define, base_yyparse, fopen, fprintf, exit 等。
  • 注: ツールが提示した contrib/amcheck 等の参照は、本シンボルとは無関係な同名の別関数への言及であり、本関数の呼び出し関係を示すものではない。

関連データ構造

  • struct option ecpg_options: getopt_long 用のオプション定義。
  • struct _include_path: インクルードパスのリスト構造。
  • グローバル変数: compat, include_paths, auto_create_c, base_yyin, base_yyout 等。

失敗モードと境界条件

  • 実行パスの特定失敗: ILLEGAL_OPTION を返し終了。
  • 不正なオプション: ILLEGAL_OPTION を返し終了。
  • 出力ファイルオープン失敗: エラーメッセージを出力し、処理を継続または終了。
  • 入力ファイル未指定: ILLEGAL_OPTION を返し終了。

保守担当者向けメモ

  • 複数のファイルを処理する際、cur（カーソル）や types（型定義）などのグローバル状態が適切にリセットされることを確認すること。
  • -C オプションによる互換モードの分岐は、compat 変数を通じて各モジュールへ伝播するため、互換性挙動の変更時はこの変数の参照先を追跡すること。
