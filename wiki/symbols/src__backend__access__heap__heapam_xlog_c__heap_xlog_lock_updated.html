heap_xlog_lock_updated

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/heap/heapam_xlog.c:1070

目的

更新されたタプルのロック状態に関するWALレコードをリプレイし、ヒープページ上のタプルヘッダー情報および可視性マップを更新する。

シグネチャまたは形状

static void heap_xlog_lock_updated(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）
  • **出力**: なし（副作用としてバッファおよび可視性マップを更新）

前提条件と不変条件

  • XLogReaderState は有効なWALレコードを指している必要がある。
  • xl_heap_lock_updated 構造体としてレコードデータを解釈可能であること。
  • ページ上のタプルが正常な状態（ItemIdIsNormal）であることを前提とする。

アルゴリズムまたは振る舞い

1. **可視性マップの更新**: レコードのフラグに XLH_LOCK_ALL_FROZEN_CLEARED が設定されている場合、対応するブロックの可視性マップから VISIBILITYMAP_ALL_FROZEN ビットをクリアする。 2. **バッファの読み込み**: XLogReadBufferForRedo を呼び出し、ページが最新でない（BLK_NEEDS_REDO）場合にのみ以下の更新を行う。

  • タプルヘッダーの t_infomask から HEAP_XMAX_BITS および HEAP_MOVED を除去。
  • t_infomask2 から HEAP_KEYS_UPDATED を除去。
  • fix_infomask_from_infobits を用いて、レコード内の情報に基づきマスクを再設定。
  • HeapTupleHeaderSetXmax を用いて xmax を更新。
  • ページのLSNを更新し、バッファをダーティとしてマークする。

    • 3. **後処理**: バッファが取得されている場合はロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_xlog.c:1245（WALリプレイのディスパッチ処理内）
  • **呼び出し先**:
  • XLogRecGetData, XLogRecGetBlockTag, XLogReadBufferForRedo
  • visibilitymap_pin, visibilitymap_clear
  • CreateFakeRelcacheEntry, FreeFakeRelcacheEntry
  • PageGetMaxOffsetNumber, PageGetItemId, PageGetItem, PageSetLSN
  • fix_infomask_from_infobits, HeapTupleHeaderSetXmax
  • MarkBufferDirty, UnlockReleaseBuffer

関連データ構造

  • xl_heap_lock_updated: WALレコードのデータ構造。
  • XLogReaderState: WAL読み取り状態。
  • Buffer: ヒープページを保持するバッファ。
  • HeapTupleHeader: タプルの物理ヘッダー。

失敗モードと境界条件

  • PageGetMaxOffsetNumber(page) < offnum または !ItemIdIsNormal(lp) の場合、PANIC を発生させてリカバリを停止する。

保守担当者向けメモ

  • 本関数は、ロック更新に伴う可視性マップのクリアと、タプルヘッダーのビット操作をアトミックに行う必要がある。
  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返さない場合でも、可視性マップの更新は必要となる可能性があるため、条件分岐の順序に注意すること。
