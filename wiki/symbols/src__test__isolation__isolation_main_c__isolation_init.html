isolation_init

定義

  • 種別: function
  • 定義箇所: src/test/isolation/isolation_main.c:110

目的

分離テスト（Isolation Tests）の実行に必要な初期設定を行う。具体的には、isolationtester バイナリの検索に必要な実行パスの保存と、デフォルトのデータベース名の登録を行う。

シグネチャまたは形状

static void isolation_init(int argc, char **argv)

入力と出力

  • **入力**:
  • argc: コマンドライン引数の数。
  • argv: コマンドライン引数の配列。
  • **出力**: なし（副作用としてグローバル変数 saved_argv0 へのパス保存および dblist への項目追加を行う）。

前提条件と不変条件

  • argv[0] が有効なパス文字列であること。
  • MAXPGPATH を超える長さのパスが渡された場合、エラーメッセージを出力してプロセスを終了する。

アルゴリズムまたは振る舞い

1. argv[0] を静的バッファ saved_argv0 にコピーする。 2. コピーしたパスが MAXPGPATH を超える場合は、エラーを出力して終了コード 2 で終了する。 3. add_stringlist_item を呼び出し、デフォルトの回帰テスト用データベース名 "isolation_regression" を dblist に追加する。 4. isolationtester バイナリの検索（find_other_exec）は、ライブラリ検索パスが未設定である可能性があるため、本関数内では行わず、後続の isolation_start_test に遅延させる。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（pg_regress の初期化プロセスの一部として呼び出される）。
  • **呼び出し先**: add_stringlist_item（pg_regress フレームワークの関数）。

関連データ構造

  • saved_argv0: isolationtester のパスを保持する静的バッファ。
  • dblist: テスト対象のデータベース名を管理するリスト。

失敗モードと境界条件

  • argv[0] の長さが MAXPGPATH 以上の場合、stderr にエラーを出力し exit(2) を実行する。

保守担当者向けメモ

  • 本関数でバイナリ検索を行わない理由は、regression_main() がコマンドライン引数を解析する前に本関数を呼び出すためである。この時点ではライブラリ検索パスが正しく設定されておらず、find_other_exec が内部で行う isolationtester -V の実行が失敗する可能性がある。
