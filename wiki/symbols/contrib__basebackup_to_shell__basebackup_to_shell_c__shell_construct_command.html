shell_construct_command

定義

  • 種別: function
  • 定義箇所: contrib/basebackup_to_shell/basebackup_to_shell.c:211

目的

バックアップ処理で使用するシェルコマンド文字列内のプレースホルダーを、具体的なファイル名およびターゲット詳細情報で置換し、実行可能なコマンド文字列を生成する。

シグネチャまたは形状

static char *shell_construct_command(const char *base_command, const char *filename, const char *target_detail);

入力と出力

  • **入力**:
  • base_command: 置換対象のテンプレートとなるコマンド文字列。
  • filename: %f プレースホルダーに代入するファイル名。
  • target_detail: %d プレースホルダーに代入するターゲット詳細情報。
  • **出力**:
  • プレースホルダーが置換された新しい文字列（呼び出し元でメモリ解放が必要）。

前提条件と不変条件

  • base_command には %f および %d プレースホルダーが含まれていることが想定される。
  • common/percentrepl.h で定義された replace_percent_placeholders 関数が利用可能であること。

アルゴリズムまたは振る舞い

1. replace_percent_placeholders を呼び出す。 2. 第2引数にエラーメッセージ用の設定名 "basebackup_to_shell.command" を指定。 3. 第3引数に置換対象のプレースホルダー文字 "df" を指定し、target_detail と filename をそれぞれ対応させる。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/basebackup_to_shell/basebackup_to_shell.c 内の shell_run_command（推測される文脈）および bbsink_shell_begin_archive 等の初期化処理（ソース内の直接的な参照箇所: 263行目）。
  • **呼び出し先**: replace_percent_placeholders（common/percentrepl.h）。

関連データ構造

  • bbsink_shell: コマンド文字列を保持する構造体。

失敗モードと境界条件

  • replace_percent_placeholders が内部的にエラーを発生させた場合、バックアップ処理全体に影響が及ぶ可能性がある。
  • 入力文字列が NULL の場合の挙動は replace_percent_placeholders の実装に依存する。

保守担当者向けメモ

  • 本関数は単なる文字列置換のラッパーである。コマンドのセキュリティバリデーションは本関数の前段（shell_check_detail 等）で行う必要がある。
