ExecParallelHashJoinNewBatch

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHashjoin.c:1270

目的

並列ハッシュ結合において、未処理のバッチを検索し、必要に応じてハッシュテーブルの構築やロードを行い、プローブ可能な状態へ遷移させる。

シグネチャまたは形状

static bool ExecParallelHashJoinNewBatch(HashJoinState *hjstate)

入力と出力

  • **入力**: HashJoinState *hjstate（結合状態を保持する構造体）
  • **出力**: bool（新しいバッチの準備が完了し、プローブを開始できる場合は true、すべてのバッチが処理済みであれば false）

前提条件と不変条件

  • hjstate->hj_HashTable が有効であること。
  • 並列実行環境下で呼び出されること。
  • デッドロック回避のため、PHJ_BATCH_PROBE フェーズではバリア待機を行わないこと。

アルゴリズムまたは振る舞い

1. **バッチの切り替え**: 現在のバッチがあれば完了状態にし、デタッチする。 2. **バッチの選択**: parallel_state->distributor を用いて、参加プロセス間で負荷分散するように開始バッチ番号を決定する。

  • BarrierAttach によりバッチのフェーズを判定し、以下の処理を行う。
  • PHJ_BATCH_ELECT: 1つのバックエンドがハッシュテーブルを割り当てる。
  • PHJ_BATCH_ALLOCATE: 割り当て完了を待機する。
  • PHJ_BATCH_LOAD: 共有タプルストアからタプルを読み込み、ハッシュテーブルへ挿入する。
  • PHJ_BATCH_PROBE: プローブフェーズへ移行し、true を返して呼び出し元へ制御を戻す。
  • PHJ_BATCH_SCAN: 完了済みとしてマークし、デタッチする。
  • PHJ_BATCH_FREE: メモリ解放等の後処理を行い、デタッチする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHashjoin.c 内の並列実行ロジック（ExecParallelHashJoin 等から間接的に参照される）。
  • **呼び出し先**: ExecHashTableDetachBatch, BarrierAttach, BarrierArriveAndWait, ExecParallelHashTableAlloc, ExecParallelHashTableSetCurrentBatch, sts_begin_parallel_scan, sts_parallel_scan_next, ExecForceStoreMinimalTuple, ExecParallelHashTableInsertCurrentBatch, sts_end_parallel_scan, BarrierDetach。

関連データ構造

  • HashJoinState: 結合ノードの状態管理。
  • HashJoinTable: ハッシュテーブルの共有状態。
  • Barrier: 並列処理の同期用バリア。
  • SharedTuplestoreAccessor: 共有メモリ上のタプルストアへのアクセス用。

失敗モードと境界条件

  • 未定義のバッチフェーズに遭遇した場合、elog(ERROR) を発生させる。
  • すべてのバッチが処理済みの場合、false を返して終了する。

保守担当者向けメモ

  • 本関数は並列実行時のデッドロック回避のため、PHJ_BATCH_PROBE フェーズでバリア待機を行わない設計となっている。
