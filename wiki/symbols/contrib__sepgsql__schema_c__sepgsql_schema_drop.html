sepgsql_schema_drop

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/schema.c:113

目的

データベーススキーマ（pg_namespace）の削除操作に対し、SELinux ポリシーに基づいた権限チェックを実行します。

シグネチャまたは形状

void sepgsql_schema_drop(Oid namespaceId)

入力と出力

  • **入力**: namespaceId (Oid) - 削除対象となるスキーマのオブジェクト識別子。
  • **出力**: なし。権限チェックに失敗した場合はエラーを発生させ、トランザクションを中断します。

前提条件と不変条件

  • namespaceId は有効な pg_namespace カタログ内の OID である必要があります。
  • 内部で getObjectIdentity を使用して対象の識別名を取得するため、システムカタログへのアクセスが可能な状態である必要があります。

アルゴリズムまたは振る舞い

1. NamespaceRelationId と指定された namespaceId を用いて ObjectAddress 構造体を初期化します。 2. getObjectIdentity を呼び出し、監査ログ出力用のスキーマ識別名を取得します。 3. sepgsql_avc_check_perms を呼び出し、SEPG_CLASS_DB_SCHEMA クラスに対する SEPG_DB_SCHEMA__DROP 権限を検証します。

  • 第5引数に true を指定しているため、権限違反時は監査ログを生成し、エラーを発生させます。

    • 4. 取得した識別名文字列を pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/hooks.c:172 から呼び出されます。
  • **呼び出し先**:
  • getObjectIdentity (PostgreSQL 内部関数)
  • sepgsql_avc_check_perms (SE-PostgreSQL AVC インターフェース)
  • pfree (PostgreSQL メモリ管理)

関連データ構造

  • ObjectAddress: 削除対象のスキーマを特定するために使用。
  • NamespaceRelationId: pg_namespace カタログを参照するための定数。

失敗モードと境界条件

  • 指定された namespaceId が存在しない場合、getObjectIdentity が失敗する可能性があります。
  • SELinux ポリシーにより DROP 権限が拒否された場合、sepgsql_avc_check_perms がエラーを発生させ、処理が中断されます。

保守担当者向けメモ

  • 本関数は sepgsql_hooks.c を経由して PostgreSQL のフックポイントから呼び出される設計です。
  • 権限チェックの成否は AVC（Access Vector Cache）の状態に依存します。
