pseudocolor

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_color.c:312

目的

正規表現エンジンにおいて、文字コードの等価類（色）とは別に管理される「擬似的な色（pseudocolor）」を割り当てます。これは主に、行頭・行末などの特殊な遷移条件を表現するために使用されます。

シグネチャまたは形状

color pseudocolor(struct colormap *cm)

入力と出力

  • **入力**: cm (カラーマップ構造体へのポインタ)
  • **出力**: 新たに割り当てられた color 型の値。エラー発生時は COLORLESS を返します。

前提条件と不変条件

  • newcolor(cm) が正常に実行可能であること。
  • 呼び出し後に CISERR() マクロでエラー状態を確認する必要があります。

アルゴリズムまたは振る舞い

1. newcolor(cm) を呼び出し、新しい色インデックスを取得します。 2. エラーが発生した場合は直ちに COLORLESS を返します。 3. 取得した色に対応する colordesc 構造体を初期化します。

  • nuchrs を 1 に設定し、上位マップに存在するものとして扱います。
  • sub を NOSUB に設定します。
  • arcs を NULL に初期化します。
  • flags に PSEUDO フラグを付与します。

    • 4. 割り当てた色インデックスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regc_nfa.c 内の nfa 構造体初期化処理（nfa->bos および nfa->eos の設定時）。
  • **呼び出し先**: newcolor（src/backend/regex/regc_color.c 内）。

関連データ構造

  • colormap: カラーマップ全体を管理する構造体。
  • colordesc: 個別の色に関する属性（フラグ、エッジリスト等）を保持する構造体。

失敗モードと境界条件

  • newcolor がメモリ不足などで失敗した場合、COLORLESS が返されます。呼び出し側でのエラーチェックが必須です。

保守担当者向けメモ

  • 本関数で生成された色は、通常の文字クラスによる遷移とは区別されます。regc_nfa.c 内の遷移条件判定ロジックにおいて、PSEUDO フラグを持つ色は特別な扱いを受けるため、変更時は注意してください。
