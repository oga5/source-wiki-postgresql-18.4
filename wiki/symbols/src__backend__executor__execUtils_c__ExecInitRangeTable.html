ExecInitRangeTable

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execUtils.c:772

目的

クエリ実行状態（EState）に対して、レンジテーブルおよび関連するパーミッション情報、スキャン対象リレーションのビットマップセットを初期化し、リレーション格納用配列を準備する。

シグネチャまたは形状

void ExecInitRangeTable(EState *estate, List *rangeTable, List *permInfos, Bitmapset *unpruned_relids)

入力と出力

  • **入力**:
  • estate: 初期化対象の実行状態構造体。
  • rangeTable: レンジテーブルエントリのリスト。
  • permInfos: レンジテーブルエントリに対応するパーミッション情報のリスト。
  • unpruned_relids: スキャン対象となるリレーションIDのビットマップセット。
  • **出力**: なし（estate 構造体のメンバを直接更新）。

前提条件と不変条件

  • estate は既に割り当て済みであること。
  • estate->es_range_table_size は rangeTable のリスト長と一致する。
  • estate->es_relations は es_range_table_size 分の Relation ポインタを保持する配列として確保される。

アルゴリズムまたは振る舞い

1. estate に rangeTable および permInfos を設定する。 2. rangeTable のリスト長を計算し、es_range_table_size に格納する。 3. unpruned_relids を estate->es_unpruned_relids に設定する。 4. es_range_table_size に基づき、Relation ポインタを格納する配列を palloc0 で確保し、estate->es_relations に割り当てる（初期値はすべて NULL）。 5. estate->es_result_relations および estate->es_rowmarks を NULL に初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfrom.c
  • src/backend/executor/execMain.c
  • src/backend/replication/logical/worker.c
  • src/backend/replication/pgoutput/pgoutput.c
  • **呼び出し先**:
  • list_length (PostgreSQL リスト操作関数)
  • palloc0 (メモリ割り当て関数)

関連データ構造

  • EState: クエリ実行状態を保持する主要構造体。
  • Relation: リレーション記述子。
  • List: レンジテーブルおよびパーミッション情報を保持するリスト。
  • Bitmapset: スキャン対象リレーションの識別に使用。

失敗モードと境界条件

  • メモリ不足時、palloc0 によりエラーが発生する可能性がある。
  • rangeTable が空の場合、es_range_table_size は 0 となり、es_relations 配列は空（または NULL）として確保される。

保守担当者向けメモ

  • es_relations は必要に応じて遅延オープンされるため、本関数実行直後はすべて NULL である。
  • es_result_relations と es_rowmarks は本関数では初期化されず、必要に応じて別途割り当てられる。
  • ExecDoInitialPruning() によって es_unpruned_relids が後から拡張される可能性がある点に留意すること。
