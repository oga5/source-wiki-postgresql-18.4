RELSYNCCALLBACK

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/inval.c:296

目的

リレーション同期（RelSync）に関連するコールバック関数とその引数を保持し、キャッシュ無効化イベント発生時に実行される処理を管理するための構造体です。

シグネチャまたは形状

struct RELSYNCCALLBACK {
    RelSyncCallbackFunction function;
    Datum                   arg;
};

この構造体は、relsync_callback_list[MAX_RELSYNC_CALLBACKS] という静的配列の要素として定義されています。

入力と出力

  • **入力**: RelSyncCallbackFunction 型の関数ポインタおよび Datum 型の引数。
  • **出力**: なし（構造体として状態を保持）。

前提条件と不変条件

  • relsync_callback_list 配列のサイズは MAX_RELSYNC_CALLBACKS によって制限されています。
  • 登録されたコールバックは、キャッシュ無効化イベントの伝播プロセスにおいて呼び出されます。

アルゴリズムまたは振る舞い

  • 本構造体は、特定のキャッシュ無効化イベントが発生した際に実行すべきカスタムロジックを登録するために使用されます。
  • relsync_callback_list 配列を走査することで、登録済みのコールバックを順次参照・実行する仕組みとなっています。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/inval.c 内の関数（809行目および1924行目付近）において、relsync_callback_list 配列の要素へのポインタとして参照されています。
  • **呼び出し先**: 保持している function フィールドが指す関数が実行されます（具体的な関数名はソース抜粋からは特定できません）。

関連データ構造

  • relsync_callback_list: RELSYNCCALLBACK 型の要素を格納する静的配列。
  • RelSyncCallbackFunction: コールバック関数の型定義。

失敗モードと境界条件

  • MAX_RELSYNC_CALLBACKS を超える数のコールバックを登録しようとした場合の挙動については、本ソース抜粋からは不明です。

保守担当者向けメモ

  • 本構造体は inval.c 内で管理されるコールバックリストの一部です。
  • 呼び出し元・呼び出し先の詳細なフローについては、relsync_callback_list を操作する関数群（登録および実行ロジック）の解析が必要です。
