gistfitpage

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistutil.c:78

目的

指定されたインデックスタプルの集合が、GiSTインデックスの1ページ内に収まるかどうかを判定します。

シグネチャまたは形状

bool gistfitpage(IndexTuple *itvec, int len)

入力と出力

  • **入力**:
  • itvec: 評価対象となる IndexTuple の配列。
  • len: 配列内のタプル数。
  • **出力**:
  • bool: 全タプルが GiSTPageSize 以内に収まる場合は true、それ以外は false を返します。

前提条件と不変条件

  • itvec には有効な IndexTuple が格納されている必要があります。
  • ページサイズ制限の計算には IndexTupleSize および sizeof(ItemIdData) が使用されます。

アルゴリズムまたは振る舞い

1. 配列内の各タプルについて、IndexTupleSize(itvec[i]) に sizeof(ItemIdData)（ページ内のアイテムポインタ分）を加算し、合計サイズを算出します。 2. 算出された合計サイズが GiSTPageSize 以下であるかを比較し、その結果を返します。 3. 現在の実装では、インデックスのフィルファクタ（fillfactor）は考慮されていません。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gist.c: ページ分割後の左右のページにタプルが収まるかを確認するために呼び出されます。
  • **呼び出し先**:
  • IndexTupleSize マクロ（詳細な呼び出し先はソースコードのインクルード階層に依存）。

関連データ構造

  • IndexTuple: GiSTインデックスに格納されるデータ構造。
  • ItemIdData: ページ内のアイテムポインタを管理する構造体。

失敗モードと境界条件

  • len が 0 の場合、サイズは 0 となり true を返します。
  • タプルの合計サイズが GiSTPageSize を超える場合、false を返します。

保守担当者向けメモ

  • 現在、フィルファクタ（fillfactor）を考慮していないため、将来的にインデックスの充填率を制御する必要がある場合は、この関数にロジックを追加してください。
