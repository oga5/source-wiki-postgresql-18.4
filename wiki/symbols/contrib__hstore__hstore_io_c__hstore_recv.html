hstore_recv

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_io.c:503

目的

PostgreSQL のバイナリ転送プロトコル経由で受信したデータストリームから、hstore 型の内部表現（HStore 構造体）を構築する。

シグネチャまたは形状

Datum hstore_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_POINTER(0) を介して取得される StringInfo 型のバッファ。このバッファには、ペア数（int32）、各キーの長さと内容、各値の長さと内容が順次格納されている。
  • **出力**: 構築された HStore 構造体へのポインタを Datum として返す。

前提条件と不変条件

  • 入力バッファは libpq のバイナリ形式に従っていること。
  • ペア数が MaxAllocSize / sizeof(Pairs) を超える場合、エラーが発生する。
  • キーは NULL を許容しない（rawlen < 0 の場合エラー）。値は NULL を許容する。

アルゴリズムまたは振る舞い

1. バッファからペア数 pcount を読み取る。pcount が 0 の場合は空の HStore を作成して終了する。 2. pcount 分の Pairs 配列をメモリ確保する。 3. 各ペアについて以下の処理を繰り返す：

  • キーの長さを読み取り、pq_getmsgtext でキー文字列を取得し、hstoreCheckKeyLen で長さを検証する。
  • 値の長さを読み取り、負の値であれば NULL として扱い、正の値であれば pq_getmsgtext で値文字列を取得し、hstoreCheckValLen で長さを検証する。

    • 4. hstoreUniquePairs を呼び出し、キーのソートと重複排除を行う。 5. hstorePairs を呼び出し、最終的な HStore 構造体を生成して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: pq_getmsgint, pq_getmsgtext, hstoreCheckKeyLen, hstoreCheckValLen, hstoreUniquePairs, hstorePairs, palloc, ereport。

関連データ構造

  • HStore: hstore 型の内部表現。
  • Pairs: パース中の一時的なキーと値のペアを保持する構造体。
  • StringInfo: 入力データストリームを保持する構造体。

失敗モードと境界条件

  • pcount が許容範囲外の場合: ERRCODE_PROGRAM_LIMIT_EXCEEDED エラー。
  • キーが NULL の場合: ERRCODE_NULL_VALUE_NOT_ALLOWED エラー。
  • メモリ不足時: palloc によるエラー。

保守担当者向けメモ

  • 本関数はバイナリプロトコルに依存しているため、hstore の内部バイナリ形式を変更する際は hstore_send との整合性に注意すること。
  • hstoreUniquePairs が内部でメモリを解放する場合があるため、Pairs 配列のライフサイクル管理に留意すること。
