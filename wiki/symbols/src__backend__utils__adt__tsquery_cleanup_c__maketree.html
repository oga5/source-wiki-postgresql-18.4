maketree

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_cleanup.c:32

目的

フラットな QueryItem 配列形式の tsquery を、再帰的な NODE 構造体のツリー形式へ変換します。

シグネチャまたは形状

static NODE *maketree(QueryItem *in)

入力と出力

  • **入力**: QueryItem *in - 変換対象となる tsquery のフラットな配列の先頭ポインタ。
  • **出力**: NODE * - 構築されたツリーのルートノード。

前提条件と不変条件

  • in は有効な QueryItem 配列を指している必要があります。
  • 再帰的な処理を行うため、スタックオーバーフローを防止する check_stack_depth() が呼び出されます。

アルゴリズムまたは振る舞い

1. palloc を使用して新しい NODE 構造体をメモリ確保します。 2. check_stack_depth() を実行し、スタックの深さを確認します。 3. 現在の QueryItem を node->valnode に割り当てます。 4. in->type が QI_OPR（演算子）である場合、再帰的にツリーを構築します。

  • node->right には in + 1 を渡して再帰呼び出しを行います。
  • OP_NOT 以外の演算子の場合、node->left には in + in->qoperator.left を渡して再帰呼び出しを行います。

    • 5. 構築したノードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery_cleanup.c 内の各箇所（root = maketree(ptr) や clean_stopword_intree への引数として利用）。
  • **呼び出し先**:
  • maketree (自身を再帰的に呼び出し)
  • palloc (メモリ確保)
  • check_stack_depth (スタック保護)

関連データ構造

  • NODE: クエリツリーを表現する構造体。valnode（QueryItem へのポインタ）、left、right を保持します。
  • QueryItem: tsquery のフラットな要素。

失敗モードと境界条件

  • スタックの深さが制限を超えた場合、check_stack_depth() によりエラーが発生します。
  • メモリ不足時には palloc がエラーをスローします。

保守担当者向けメモ

  • 本関数は再帰的なツリー構築を行うため、非常に深いクエリが入力された場合にスタックを消費します。
  • OP_NOT 演算子の場合は右側のみを処理し、それ以外の演算子の場合は左右両方を処理するという、tsquery の構造に依存したロジックになっています。
