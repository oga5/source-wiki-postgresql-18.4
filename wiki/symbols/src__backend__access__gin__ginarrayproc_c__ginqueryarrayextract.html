ginqueryarrayextract

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginarrayproc.c:78

目的

GINインデックスを用いた配列型の検索において、検索条件となる配列を解析し、演算子に応じた検索モード（searchMode）の決定と、インデックスキーの抽出を行います。

シグネチャまたは形状

Datum ginqueryarrayextract(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_ARRAYTYPE_P_COPY(0): 検索対象の配列データ。
  • PG_GETARG_POINTER(1): 抽出されたキーの数を格納する int32 へのポインタ。
  • PG_GETARG_UINT16(2): 適用する演算子の StrategyNumber。
  • PG_GETARG_POINTER(5): NULLフラグ配列を格納する bool** へのポインタ。
  • PG_GETARG_POINTER(6): 決定された検索モードを格納する int32* へのポインタ。
  • **出力**:
  • Datum: 抽出された要素の配列（Datum*）。

前提条件と不変条件

  • 入力された配列データは、関数実行中に消失しないようコピーが保持されます。
  • 抽出された要素の配列は、元の配列メモリ領域を指しているため、呼び出し側で元の配列を解放してはなりません。

アルゴリズムまたは振る舞い

1. get_typlenbyvalalign を用いて配列要素の型情報を取得します。 2. deconstruct_array を呼び出し、配列を個々の要素（elems）とNULLフラグ（nulls）に分解します。 3. 演算子（strategy）に基づき、以下の通り searchMode を設定します。

  • GinOverlapStrategy (&&): GIN_SEARCH_MODE_DEFAULT
  • GinContainsStrategy (@>): 要素数が0より大きければ DEFAULT、そうでなければ全件検索 (GIN_SEARCH_MODE_ALL)。
  • GinContainedStrategy (<@): GIN_SEARCH_MODE_INCLUDE_EMPTY
  • GinEqualStrategy (=): 要素数が0より大きければ DEFAULT、そうでなければ GIN_SEARCH_MODE_INCLUDE_EMPTY。

    • 4. 抽出した要素の配列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は明示されていませんが、GINインデックスの演算子クラスのサポート関数としてデータベースエンジンから呼び出されます。
  • **呼び出し先**: deconstruct_array, get_typlenbyvalalign, elog。

関連データ構造

  • ArrayType: 入力配列の構造体。
  • StrategyNumber: 検索演算子を識別する数値。
  • GIN_SEARCH_MODE_*: GINインデックスの検索挙動を制御する定数。

失敗モードと境界条件

  • 未知の strategy が渡された場合、elog(ERROR) により処理が中断されます。
  • 空配列が入力された場合、演算子に応じて GIN_SEARCH_MODE_ALL や GIN_SEARCH_MODE_INCLUDE_EMPTY に切り替わり、検索範囲が調整されます。

保守担当者向けメモ

  • 本関数は deconstruct_array で生成されたメモリ領域を返すため、メモリ管理には注意が必要です。
  • strategy の追加や変更を行う際は、switch 文の網羅性を確認してください。
