consume_xids_common

定義

  • 種別: function
  • 定義箇所: src/test/modules/xid_wraparound/xid_wraparound.c:70

目的

指定された数、またはターゲットとなるトランザクションID（XID）に到達するまで、システム上のXIDを強制的に消費する。

シグネチャまたは形状

static FullTransactionId consume_xids_common(FullTransactionId untilxid, uint64 nxids)

入力と出力

  • **入力**:
  • untilxid: 到達目標となる FullTransactionId（nxids が 0 の場合に使用）。
  • nxids: 消費するXIDの数（0 より大きい場合に使用）。
  • **出力**:
  • 最後に消費された FullTransactionId を返す。

前提条件と不変条件

  • 本関数を呼び出すトランザクションは、トップレベルのXIDを保持している必要がある（GetTopTransactionId() を内部で呼び出し確保する）。
  • CHECK_FOR_INTERRUPTS() をループ内で実行するため、割り込みによる中断が可能である。

アルゴリズムまたは振る舞い

1. ReadNextFullTransactionId() で現在の次期XIDを取得し初期化する。 2. ループ内で以下の処理を繰り返す：

  • 残り消費数を確認し、目標に達していれば終了する。
  • **高速パス**: 残りXIDが2000を超え、かつサブトランザクションキャッシュがオーバーフローしている場合、consume_xids_shortcut() を呼び出して nextXid カウンタを直接進める。
  • **低速パス**: 上記条件を満たさない場合、GetNewTransactionId(true) を呼び出し、サブトランザクションとして1つずつXIDを消費する。
  • 1,000万XID消費ごとに NOTICE メッセージで進捗をログ出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • consume_xids (src/test/modules/xid_wraparound/xid_wraparound.c:43)
  • consume_xids_until (src/test/modules/xid_wraparound/xid_wraparound.c:62)
  • **呼び出し先**:
  • GetTopTransactionId()
  • CHECK_FOR_INTERRUPTS()
  • consume_xids_shortcut()
  • GetNewTransactionId()
  • elog()

関連データ構造

  • FullTransactionId: 64bitトランザクションID。
  • PGPROC (内部の subxidStatus.overflowed): サブトランザクションキャッシュの状態。

失敗モードと境界条件

  • XidGenLock を排他ロックするため、実行中は他のトランザクションの進行をブロックする。
  • 本番環境で実行するとXID周回を早め、データベース停止のリスクがある。
  • nxids が 0 の場合、untilxid に到達するまで消費を続ける。

保守担当者向けメモ

  • REPORT_INTERVAL 定数は 10,000,000 に固定されている。
  • ショートカットロジックは XidGenLock に依存しており、PostgreSQLの内部実装変更の影響を受ける可能性がある。
