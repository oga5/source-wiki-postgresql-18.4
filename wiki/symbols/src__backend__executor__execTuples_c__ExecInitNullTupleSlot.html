ExecInitNullTupleSlot

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execTuples.c:2035

目的

指定されたタプル記述子（TupleDesc）に基づき、すべての属性が NULL であるタプルを保持する TupleTableSlot を初期化して返します。

シグネチャまたは形状

TupleTableSlot *ExecInitNullTupleSlot(EState *estate, TupleDesc tupType, const TupleTableSlotOps *tts_ops)

入力と出力

  • **入力**:
  • estate: 実行状態（EState）。
  • tupType: スロットが扱うタプルの構造を定義する TupleDesc。
  • tts_ops: スロットの操作セットを定義する TupleTableSlotOps。
  • **出力**:
  • 初期化され、全属性が NULL に設定された TupleTableSlot へのポインタ。

前提条件と不変条件

  • estate および tupType は有効なポインタである必要があります。
  • tts_ops は、スロットの型（TTSOpsVirtual 等）を決定するために必要です。

アルゴリズムまたは振る舞い

1. ExecInitExtraTupleSlot を呼び出し、指定された estate、tupType、tts_ops を用いて新しい TupleTableSlot を割り当てます。 2. 割り当てられたスロットに対し ExecStoreAllNullTuple を呼び出し、スロット内の全属性を NULL に設定します。 3. 初期化および NULL 設定が完了したスロットを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeHashjoin.c
  • src/backend/executor/nodeMergejoin.c
  • src/backend/executor/nodeNestloop.c
  • **呼び出し先**:
  • ExecInitExtraTupleSlot (直接の依存)
  • ExecStoreAllNullTuple (直接の依存)

関連データ構造

  • TupleTableSlot: タプルを保持するコンテナ。
  • TupleDesc: タプルの属性定義。
  • TupleTableSlotOps: スロットの操作関数テーブル。
  • EState: 実行時の状態管理構造体。

失敗モードと境界条件

  • メモリ不足時には palloc 等の内部関数経由でエラーが発生する可能性があります。
  • tupType が NULL の場合、ExecInitExtraTupleSlot の内部で未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は主に結合ノード（Hash Join, Merge Join, Nest Loop）において、外部結合（Outer Join）の際に「一致する行がない場合」のプレースホルダーとして NULL タプルを生成するために使用されます。
  • tts_ops に TTSOpsVirtual を指定することが一般的ですが、呼び出し元での用途に応じて適切な操作セットを選択してください。
