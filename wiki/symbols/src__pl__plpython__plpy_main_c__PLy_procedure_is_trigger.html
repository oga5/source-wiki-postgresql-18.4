PLy_procedure_is_trigger

定義

  • 種別: static function
  • 定義箇所: src/pl/plpython/plpy_main.c:339

目的

与えられた PostgreSQL プロシージャがトリガー関数であるかどうかを判定する。

シグネチャまたは形状

static bool PLy_procedure_is_trigger(Form_pg_proc procStruct)

入力と出力

  • **入力**: procStruct (Form_pg_proc) - 検査対象のプロシージャのシステムカタログ情報。
  • **出力**: bool - procStruct->prorettype が TRIGGEROID と一致する場合は true、それ以外は false。

前提条件と不変条件

  • procStruct は有効な Form_pg_proc 構造体を指している必要がある。

アルゴリズムまたは振る舞い

1. 引数として受け取った procStruct の戻り値型フィールド (prorettype) を参照する。 2. その値が TRIGGEROID と等しいかを比較し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_main.c 内の関数（例: 行183付近）から呼び出されている。
  • **呼び出し先**: 外部関数への呼び出しは行わない。

関連データ構造

  • Form_pg_proc: PostgreSQL の pg_proc システムカタログの行構造体。
  • TRIGGEROID: トリガー関数の戻り値型を表す OID。

失敗モードと境界条件

  • procStruct が NULL の場合、セグメンテーション違反が発生する可能性がある（ソースコード上に明示的なガードは存在しない）。

保守担当者向けメモ

  • 本関数は prorettype の比較のみを行う単純なヘルパー関数である。
  • 今後、トリガー以外の特殊な戻り値型（例: EVENT_TRIGGEROID）の判定が必要になった場合、本関数の拡張または類似関数の追加を検討すること。
