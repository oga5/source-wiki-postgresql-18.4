gistxlogPageDelete

定義

  • 種別: struct
  • 定義箇所: src/include/access/gistxlog.h:86

目的

GiSTインデックスのページ削除操作をWAL（Write-Ahead Logging）に記録するためのデータ構造です。削除されたページと、それを参照していた親ページのダウンリンク情報を保持します。

シグネチャまたは形状

typedef struct gistxlogPageDelete
{
    FullTransactionId deleteXid;    /* ページをスキャンで見ることができた最後のXid */
    OffsetNumber downlinkOffset;    /* このページを参照するダウンリンクのオフセット */
} gistxlogPageDelete;

入力と出力

  • 入力: GiSTインデックスのページ削除処理時に、この構造体に現在のトランザクションIDとダウンリンク位置を格納してWALレコードとして書き込みます。
  • 出力: リカバリ（リプレイ）時に、gist_redo 関数がこの構造体を読み取り、インデックスの整合性を復元するために使用します。

前提条件と不変条件

  • downlinkOffset は、削除対象ページを指す親ページ内の有効なオフセットである必要があります。
  • deleteXid は、ページ削除の可視性を判断するための基準として使用されます。

アルゴリズムまたは振る舞い

1. **記録**: GiSTインデックスのページ削除時、本構造体に情報を詰め XLogInsert を介してWALに書き込まれます。 2. **リプレイ**: リカバリ時、gist_redo がWALレコードから本構造体を復元し、該当するページ削除操作を再実行してインデックス構造を同期させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistxlog.c 内のGiST WALリプレイ処理およびログ記録処理。
  • **呼び出し先**: src/backend/access/rmgrdesc/gistdesc.c の out_gistxlogPageDelete 関数が、デバッグ出力のために本構造体を参照します。

関連データ構造

  • SizeOfGistxlogPageDelete: 本構造体のサイズを計算するためのマクロ（src/include/access/gistxlog.h:93）。

失敗モードと境界条件

  • WALレコードの読み込み時にデータが破損している場合、XLogReaderState を介した解析でエラーが発生する可能性があります。
  • 構造体のメンバ変更時には、対応する gist_redo および gist_desc の実装を更新しないと、リカバリ失敗やデバッグ情報の不整合を招きます。

保守担当者向けメモ

  • 本構造体はバイナリレイアウトに直結しているため、メンバの追加や順序変更はWALの互換性を損なう可能性があります。
  • ページ削除は親ページへの影響を伴うため、リプレイ時には block_id を用いた適切なバッファ管理が必要です。
