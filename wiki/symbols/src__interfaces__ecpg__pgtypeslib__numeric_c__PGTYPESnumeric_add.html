PGTYPESnumeric_add

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/numeric.c:636

目的

2つの numeric 型変数の加算を行い、結果を第3の変数に格納する。

シグネチャまたは形状

int PGTYPESnumeric_add(numeric *var1, numeric *var2, numeric *result)

入力と出力

  • **入力**: var1, var2 (加算対象の numeric 構造体へのポインタ)
  • **出力**: result (計算結果を格納する numeric 構造体へのポインタ)
  • **戻り値**: 成功時は 0、失敗時は -1 を返す。

前提条件と不変条件

  • var1, var2, result は有効な numeric 構造体を指している必要がある。
  • result は計算結果を保持するために十分なメモリが確保されている必要がある。

アルゴリズムまたは振る舞い

1. var1 と var2 の符号（sign）に基づき、以下のいずれかの処理を行う。

  • 両者が同符号の場合: add_abs を呼び出し、絶対値の和を計算して符号を適用する。
  • 異符号の場合: cmp_abs で絶対値を比較する。
  • 絶対値が等しい場合: zero_var で結果をゼロに初期化し、rscale と dscale を最大値に設定する。
  • 絶対値が異なる場合: sub_abs を呼び出し、大きい方の絶対値から小さい方を減算し、適切な符号を適用する。

    • 2. 内部演算関数（add_abs, sub_abs）がエラーを返した場合は、即座に -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test2.c
  • **呼び出し先**:
  • add_abs
  • sub_abs
  • cmp_abs
  • zero_var

関連データ構造

  • numeric: 数値の符号、スケール、桁配列を保持する構造体。

失敗モードと境界条件

  • add_abs または sub_abs が失敗した場合、関数は -1 を返す。
  • 異符号かつ絶対値が等しい場合、結果はゼロとして扱われ、スケールは入力値の最大値に調整される。

保守担当者向けメモ

  • 本関数は符号判定と絶対値演算のラッパーである。詳細な数値演算ロジックは add_abs および sub_abs に依存している。
  • メモリ管理は呼び出し元に依存するため、result の初期化状態に注意すること。
