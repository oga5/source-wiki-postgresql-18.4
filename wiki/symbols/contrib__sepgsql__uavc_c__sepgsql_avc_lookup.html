sepgsql_avc_lookup

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/uavc.c:296

目的

SELinux のアクセスベクタキャッシュ（AVC）を検索し、指定されたセキュリティコンテキストとオブジェクトクラスに対応する権限判定結果を取得します。キャッシュミス時には新規エントリの計算と挿入を行います。

シグネチャまたは形状

static avc_cache *sepgsql_avc_lookup(const char *scontext, const char *tcontext, uint16 tclass)

入力と出力

  • **入力**:
  • scontext (const char *): 主体（Subject）のセキュリティコンテキスト。
  • tcontext (const char *): 対象（Target）のセキュリティコンテキスト。
  • tclass (uint16): オブジェクトクラス。
  • **出力**:
  • avc_cache *: 該当するキャッシュエントリへのポインタ。

前提条件と不変条件

  • sepgsql_avc_hash によって算出されたハッシュ値と、scontext、tcontext、tclass の完全一致がキャッシュヒットの条件です。
  • キャッシュヒット時、該当エントリの hot_cache フラグが true に設定されます。

アルゴリズムまたは振る舞い

1. sepgsql_avc_hash を用いて入力値からハッシュ値を計算し、avc_slots 配列のインデックスを特定します。 2. 指定されたスロット内のリストを走査し、ハッシュ値および各コンテキスト文字列、クラスが一致するエントリを探します。 3. 一致するエントリが見つかった場合、そのエントリを返し、hot_cache を更新します。 4. 一致するエントリが見つからない場合、sepgsql_avc_compute を呼び出してカーネルから情報を取得し、新しいキャッシュエントリを生成・挿入してその結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/uavc.c 内の各権限チェック関数（sepgsql_avc_check_perms 等）。
  • **呼び出し先**: sepgsql_avc_hash（ハッシュ計算）、sepgsql_avc_compute（キャッシュミス時の計算）、sepgsql_avc_unlabeled（引数として使用される場合あり）。

関連データ構造

  • avc_cache: キャッシュエントリを保持する構造体。
  • avc_slots: キャッシュエントリを格納するハッシュテーブル（リストの配列）。

失敗モードと境界条件

  • キャッシュミスが発生した場合、sepgsql_avc_compute の挙動（カーネルへの問い合わせ成功可否）に依存します。
  • メモリ不足等のシステムエラーが発生した場合の挙動は、sepgsql_avc_compute の実装に依存します。

保守担当者向けメモ

  • 本関数はキャッシュの検索と更新を単一のフローで処理します。
  • hot_cache フラグの更新は、LRU 等のキャッシュ管理アルゴリズムのヒントとして利用されます。
  • 権限チェックのループ内での呼び出しが想定されているため、ポリシー更新によるキャッシュの無効化（sepgsql_avc_check_valid）と併用される必要があります。
