GetDbnameFromConnectionOptions

定義

  • 種別: function
  • 定義箇所: src/fe_utils/recovery_gen.c:203

目的

接続文字列または環境変数からデータベース名（dbname）を抽出します。

シグネチャまたは形状

char *GetDbnameFromConnectionOptions(const char *connstr)

入力と出力

  • **入力**: connstr（接続文字列、NULL許容）
  • **出力**: 抽出されたデータベース名を含む文字列（呼び出し元で解放が必要）、または見つからない場合はNULL。

前提条件と不変条件

  • connstr が提供された場合、PQconninfoParse が成功する必要があります。
  • PQconndefaults がメモリ不足により失敗した場合、pg_fatal を呼び出しプロセスを終了します。

アルゴリズムまたは振る舞い

1. connstr が指定されている場合、PQconninfoParse で解析し、FindDbnameInConnOpts を用いて dbname を検索します。見つかればその値を返します。 2. connstr が指定されていない、または上記で見つからなかった場合、PQconndefaults を呼び出し、環境変数からデフォルトの接続オプションを取得します。 3. デフォルトオプションに対して FindDbnameInConnOpts を実行し、結果を返します。 4. 内部で使用した PQconninfoOption は、各ステップの終了時に PQconninfoFree で解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_rewind/pg_rewind.c
  • **呼び出し先**:
  • PQconninfoParse
  • PQconndefaults
  • FindDbnameInConnOpts
  • PQconninfoFree
  • pg_fatal

関連データ構造

  • PQconninfoOption: libpqの接続オプション構造体。

失敗モードと境界条件

  • PQconninfoParse が失敗した場合、エラーメッセージとともに pg_fatal で終了します。
  • PQconndefaults がメモリ不足でNULLを返した場合、pg_fatal で終了します。
  • どちらの検索方法でも dbname が見つからない場合、NULLを返します。

保守担当者向けメモ

  • 本関数は FindDbnameInConnOpts をラップするヘルパーです。
  • 接続文字列の解析と環境変数のフォールバックという二段階の探索順序は固定されています。
