pgstat_io_reset_all_cb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_io.c:286

目的

共有メモリ上に保持されているすべてのバックエンドタイプ別のI/O統計情報をリセットし、統計リセット時刻を更新します。

シグネチャまたは形状

void pgstat_io_reset_all_cb(TimestampTz ts)

入力と出力

  • **入力**: ts (TimestampTz型) - 統計リセット時刻として設定するタイムスタンプ。
  • **出力**: なし（副作用として共有メモリ上の統計構造体がゼロクリアされます）。

前提条件と不変条件

  • pgStatLocal.shmem が初期化済みであること。
  • 共有メモリ上のI/O統計領域（io.stats.stats）および対応するロック（io.locks）が有効であること。

アルゴリズムまたは振る舞い

1. BACKEND_NUM_TYPES 分のループを実行し、各バックエンドタイプに対応する統計領域を順次処理します。 2. 各バックエンドタイプに対応する LWLock を LW_EXCLUSIVE モードで取得します。 3. ループの初回（i == 0）のみ、共有メモリ内の統計リセット時刻（stat_reset_timestamp）を引数 ts で更新します。 4. memset を使用して、対象の PgStat_BktypeIO 構造体をゼロクリアします。 5. LWLock を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c:445 にて、統計リセット用コールバックとして登録されています。
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease (排他制御)
  • memset (メモリクリア)

関連データ構造

  • PgStat_BktypeIO: 各バックエンドタイプのI/O統計を保持する構造体。
  • pgStatLocal: 統計サブシステムのローカル状態を保持するグローバル変数。

失敗モードと境界条件

  • 共有メモリが初期化されていない状態で呼び出された場合、セグメンテーション違反が発生する可能性があります。
  • LWLock の取得に失敗した場合（デッドロック等）、処理が停止します。

保守担当者向けメモ

  • 本関数は統計リセット処理の一部として呼び出されます。統計構造体の定義に変更があった場合は、memset によるクリアが適切か確認してください。
  • i == 0 の条件分岐により、リセット時刻の更新を最初のロック取得時に集約しています。
