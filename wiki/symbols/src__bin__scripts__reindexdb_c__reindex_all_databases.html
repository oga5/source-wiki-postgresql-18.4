reindex_all_databases

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/reindexdb.c:830

目的

PostgreSQL インスタンス内の全データベースを反復処理し、指定された条件（システムカタログ、スキーマ、テーブル、インデックス等）に基づいて各データベースに対して REINDEX 操作を実行する。

シグネチャまたは形状

static void reindex_all_databases(
    ConnParams *cparams,
    const char *progname, bool echo, bool quiet, bool verbose,
    bool concurrently, int concurrentCons,
    const char *tablespace, bool syscatalog,
    SimpleStringList *schemas, SimpleStringList *tables,
    SimpleStringList *indexes
)

入力と出力

  • **入力**: 接続パラメータ (cparams)、実行オプション（echo, quiet, verbose, concurrently 等）、および再構築対象のリスト（schemas, tables, indexes）。
  • **出力**: なし（副作用として各データベースへの REINDEX コマンド発行と標準出力へのログ出力を行う）。

前提条件と不変条件

  • cparams を使用してメンテナンス用データベースに接続可能であること。
  • pg_database システムカタログから接続可能なデータベースの一覧を取得できること。
  • syscatalog が指定されている場合、または対象リストが空の場合、適切な REINDEX モードが選択される。

アルゴリズムまたは振る舞い

1. connectMaintenanceDatabase を呼び出し、メンテナンス用データベースに接続する。 2. pg_database から接続可能な全データベース名を取得し、結果を保持する。 3. 取得した各データベースに対して以下の処理を順次行う：

  • cparams->override_dbname を対象データベース名に更新する。
  • 指定されたフラグに基づき、reindex_one_database を呼び出して再構築を実行する。
  • 優先順位は syscatalog > schemas > indexes > tables の順で判定される。
  • いずれの対象も指定されていない場合、データベース全体 (REINDEX_DATABASE) を対象とする。

    • 4. 全データベースの処理終了後、結果セットを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/scripts/reindexdb.c 内の main 関数（reindexdb.c:226）。
  • **呼び出し先**:
  • connectMaintenanceDatabase
  • executeQuery
  • PQfinish, PQntuples, PQgetvalue, PQclear
  • reindex_one_database

関連データ構造

  • ConnParams: 接続設定を保持する構造体。
  • SimpleStringList: 再構築対象のスキーマ、テーブル、インデックス名を保持するリスト構造。
  • PGconn, PGresult: libpq の接続および結果保持用構造体。

失敗モードと境界条件

  • データベース接続に失敗した場合、connectMaintenanceDatabase 内でエラー終了する可能性がある。
  • syscatalog 指定時は並列実行が制限される（reindex_one_database へ 1 を渡す）。
  • データベース名が取得できない、またはクエリ実行に失敗した場合は処理が中断される。

保守担当者向けメモ

  • 本関数は全データベースを対象とするため、実行時間が長くなる可能性がある。
  • reindex_one_database への引数渡しにおいて、並列実行数 (concurrentCons) の制御が対象種別ごとにハードコードされている箇所があるため、変更時は注意が必要。
