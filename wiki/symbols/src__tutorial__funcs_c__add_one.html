add_one

定義

  • 種別: function
  • 定義箇所: src/tutorial/funcs.c:24

目的

入力された int32 型の整数値に 1 を加算して返す、PostgreSQL のユーザー定義関数（UDF）のサンプル実装です。

シグネチャまたは形状

Datum add_one(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される PostgreSQL の関数引数。第 0 引数として int32 型の値を受け取ります。
  • **出力**: Datum 型として、入力値に 1 を加算した結果を返します。

前提条件と不変条件

  • PostgreSQL の V1 呼び出し規約に従う必要があります。
  • PG_FUNCTION_INFO_V1(add_one) マクロによる関数情報の宣言が必須です。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT32(0) を使用して、第 0 引数を int32 型として取得します。 2. 取得した値に 1 を加算します。 3. PG_RETURN_INT32() マクロを使用して、計算結果を Datum 型に変換して呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL エンジンから呼び出されます（ソースコード上の直接的な呼び出し元は特定されていません）。
  • **呼び出し先**:
  • PG_GETARG_INT32: 引数取得用マクロ。
  • PG_RETURN_INT32: 戻り値設定用マクロ。

関連データ構造

  • Datum: PostgreSQL における汎用的なデータ表現型。
  • int32: PostgreSQL の 32 ビット整数型。

失敗モードと境界条件

  • 入力値が int32 の範囲外である場合、加算処理によるオーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 本関数は src/tutorial/funcs.c 内で定義されており、V1 呼び出し規約の基本的な実装例として機能します。
  • メモリ管理や複雑な型変換は含まれていないため、単純な値操作のテンプレートとして参照してください。
