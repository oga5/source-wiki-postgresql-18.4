unregister_ENR

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/queryenvironment.c:81

目的

指定された名前を持つ一時的な名前付きリレーション（ENR: Ephemeral Named Relation）を、クエリ環境（QueryEnvironment）から削除します。

シグネチャまたは形状

void unregister_ENR(QueryEnvironment *queryEnv, const char *name)

入力と出力

  • queryEnv: ENR を保持するクエリ環境へのポインタ。
  • name: 削除対象の ENR を特定するための名前（文字列）。
  • 戻り値: なし。

前提条件と不変条件

  • queryEnv は有効な QueryEnvironment 構造体を指している必要があります。
  • 内部的に get_ENR を使用して検索を行うため、指定した名前の ENR が存在しない場合は何も行われません。

アルゴリズムまたは振る舞い

1. get_ENR を呼び出し、queryEnv 内から指定された name に一致する EphemeralNamedRelation を検索します。 2. 一致する ENR が見つかった場合（match が非 NULL の場合）、queryEnv->namedRelList から該当する要素を削除します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/spi.c 内のコードから呼び出されていることが確認されています。
  • 呼び出し先: get_ENR（同ファイル内）および list_delete（PostgreSQL のリスト操作関数）。

関連データ構造

  • QueryEnvironment: ENR を管理するコンテナ。
  • EphemeralNamedRelation: 削除対象となるリレーションのメタデータ構造体。

失敗モードと境界条件

  • 指定した名前の ENR が存在しない場合、get_ENR は NULL を返し、本関数は何の操作も行わず正常に終了します。

保守担当者向けメモ

  • 現在の実装では list_delete を使用した線形探索を行っています。ENR の登録数が少ないことを前提とした設計ですが、将来的に登録数が増加した場合はデータ構造の変更が必要になる可能性があります。
