目的

ECPG（Embedded SQL for C）の Informix 互換モードにおける文字列操作関数（rupshift, ldchar, byleng）の動作を検証する回帰テスト用プログラム。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム内部で定義された固定文字列 t1 を使用）。
  • 出力: 標準出力へテスト対象文字列の変換結果および各関数による計算結果を出力。正常終了時に 0 を返す。

前提条件と不変条件

  • Informix 互換ライブラリがリンクされていること。
  • t1 は "abc def ghi "（長さ 15）で初期化される。

アルゴリズムまたは振る舞い

1. 初期状態の t1 を標準出力に表示。 2. rupshift(t1) を呼び出し、t1 内の小文字を大文字に変換。 3. 変換後の t1 を標準出力に表示。 4. k の値を 2, 5, 9, 15 と変化させながら以下の処理を繰り返す：

  • ldchar(t1, k, buf): t1 から末尾の空白を除去して buf にコピー。
  • byleng(t1, k): t1 の末尾の空白を除いた長さを計算。
  • 計算結果と buf の内容を標準出力に表示。

    • 5. 処理完了後、0 を返して終了。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内に情報なし。
  • 呼び出し先: printf, rupshift, ldchar, byleng。

関連データ構造

  • char t1[]: テスト対象の文字列バッファ。
  • char buf[50]: ldchar の結果を格納する出力用バッファ。

失敗モードと境界条件

  • k の値が t1 の有効範囲を超えた場合の挙動は、各ライブラリ関数の実装に依存する。
  • 本関数自体はエラーハンドリングを行わず、正常終了コード 0 を返す設計となっている。

保守担当者向けメモ

  • 本ファイルは ECPG プリプロセッサの出力結果を検証するための「期待値」ファイルである。
  • Informix 互換モード特有の空白処理（ldchar や byleng の挙動）を確認する目的で作成されているため、標準的な C 言語の文字列操作とは異なる点に注意すること。
