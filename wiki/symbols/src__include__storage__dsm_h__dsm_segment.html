dsm_segment

定義

  • 種別: struct
  • 定義箇所: src/include/storage/dsm.h:18

目的

動的共有メモリ（Dynamic Shared Memory: DSM）セグメントの状態を保持する不透明な構造体です。実行時に確保・解放可能な共有メモリ領域のライフサイクル管理とリソース追跡を担います。

シグネチャまたは形状

typedef struct dsm_segment dsm_segment;

※構造体の内部メンバは公開されていません。

入力と出力

  • **入力**: dsm_create() により新規作成されるか、dsm_attach() により既存セグメントへアタッチされる際に生成されます。
  • **出力**: dsm_segment_address() を通じて、共有メモリ領域へのポインタが提供されます。

前提条件と不変条件

  • dsm_shmem_init() による共有メモリ制御領域の初期化が完了している必要があります。
  • プロセス終了時、dsm_detach_all() を通じて自動的にクリーンアップされるか、ResourceOwner と連携してリソースが解放されます。

アルゴリズムまたは振る舞い

1. **作成/アタッチ**: プロセスローカルなリストに登録され、参照カウントによって管理されます。 2. **利用**: 取得したアドレスを介して、並列クエリやバックグラウンドワーカー間でのデータ共有に使用されます。 3. **デタッチ**: dsm_detach() が呼ばれると参照カウントが減少し、必要に応じてメモリ領域が解放されます。 4. **ピン留め**: dsm_pin_mapping() を使用することで、リソースオーナーの寿命を超えてセッション終了まで保持可能です。

関連データ構造

  • dsm_handle: セグメントを一意に識別する識別子。
  • shm_toc: DSM 内に配置されるデータ構造のテーブル・オブ・コンテント。
  • ResourceOwner: リソースの所有権と自動クリーンアップを管理する構造体。

失敗モードと境界条件

  • セグメント作成時に上限に達した場合、DSM_CREATE_NULL_IF_MAXSEGMENTS フラグが指定されていれば NULL が返されます。
  • プロセスが異常終了した場合、on_dsm_detach コールバックを通じてクリーンアップが試行されます。

保守担当者向けメモ

  • 本構造体は不透明（opaque）であるため、直接メンバにアクセスせず、必ず dsm.h で提供される API を使用してください。
  • dsm_detach は明示的な呼び出しが可能ですが、リソースオーナーの終了時に自動実行される設計となっています。
