handle_auth_data

定義

  • 種別: function
  • 定義箇所: src/test/modules/oauth_validator/oauth_hook_client.c:172

目的

libpq の認証プロセスをインターセプトし、OAuth 認証データの注入、検証、または意図的なエラー発生を行うテスト用フック関数です。

シグネチャまたは形状

int handle_auth_data(PGauthData type, PGconn *conn, void *data);

入力と出力

  • **入力**:
  • type: 認証データの種類（PGauthData）。
  • conn: PostgreSQL 接続オブジェクト（PGconn *）。
  • data: 認証リクエストデータ（PGoauthBearerRequest *）。
  • **出力**:
  • 1: 認証処理が成功または非同期処理へ移行したことを示す。
  • 0: フックが適用対象外であることを示す。
  • -1: 検証エラーが発生したことを示す。

前提条件と不変条件

  • data 引数は PGoauthBearerRequest 型としてキャスト可能であること。
  • no_hook が真、または type が PQAUTHDATA_OAUTH_BEARER_TOKEN 以外の場合は即座に 0 を返す。

アルゴリズムまたは振る舞い

1. **非同期テスト**: hang_forever が有効な場合、req->async に async_cb を設定し 1 を返す。 2. **異常系テスト**: misbehave_mode が有効な場合、req->async に misbehave_cb を設定し 1 を返す。 3. **検証**: expected_uri および expected_scope が設定されている場合、req 内の対応するフィールドと比較し、不一致があればエラーメッセージを出力して -1 を返す。 4. **正常系**: 上記のいずれにも該当しない場合、req->token にグローバル変数 token を代入し 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: libpq の内部認証プロセス（PQsetAuthDataHook を介して登録）。
  • **呼び出し先**:
  • async_cb（hang_forever 時）
  • misbehave_cb（misbehave_mode 時）
  • strcmp, fprintf（標準ライブラリ）

関連データ構造

  • PGoauthBearerRequest: 認証リクエスト情報（トークン、スコープ、URI、非同期コールバック等）を保持する構造体。

失敗モードと境界条件

  • expected_uri や expected_scope が期待値と異なる場合、標準エラー出力に詳細を出力し -1 を返す。
  • req->openid_configuration や req->scope が NULL の状態で検証が行われた場合、エラーとして -1 を返す。

保守担当者向けメモ

  • 本関数はテストツールの一部であり、実運用環境のコードではありません。
  • misbehave_mode の値によって挙動が大きく変わるため、テストケース追加時は misbehave_cb との連携に注意してください。
