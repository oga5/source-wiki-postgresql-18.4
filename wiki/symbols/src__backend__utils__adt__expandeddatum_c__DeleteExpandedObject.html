DeleteExpandedObject

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/expandeddatum.c:135

目的

展開済みオブジェクト（Expanded Object）に関連付けられたメモリコンテキストを削除し、オブジェクトを破棄します。

シグネチャまたは形状

void DeleteExpandedObject(Datum d)

入力と出力

  • **入力**: d (展開済みオブジェクトの Datum 表現)
  • **出力**: なし

前提条件と不変条件

  • 入力 d は、読み取り書き込み可能（R/W）なTOASTポインタである必要があります。この条件は VARATT_IS_EXTERNAL_EXPANDED_RW マクロによって実行時に検証されます。

アルゴリズムまたは振る舞い

1. DatumGetEOHP(d) を呼び出し、Datum から ExpandedObjectHeader を取得します。 2. VARATT_IS_EXTERNAL_EXPANDED_RW を使用して、入力がR/Wポインタであることをアサートします。 3. ヘッダー内の eoh_context（メモリコンテキスト）に対して MemoryContextDelete を実行し、オブジェクトが保持するメモリ領域を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c
  • src/backend/executor/nodeWindowAgg.c
  • src/pl/plpgsql/src/pl_exec.c
  • **呼び出し先**:
  • DatumGetEOHP (マクロ)
  • VARATT_IS_EXTERNAL_EXPANDED_RW (マクロ)
  • MemoryContextDelete

関連データ構造

  • ExpandedObjectHeader: 展開済みオブジェクトの共通ヘッダー。eoh_context を保持します。

失敗モードと境界条件

  • 入力がR/Wポインタでない場合、Assert によりプログラムが停止します（デバッグビルド時）。
  • 既に削除済みのメモリコンテキストに対して呼び出した場合、メモリ破壊やクラッシュの原因となります。

保守担当者向けメモ

  • 本関数は、オブジェクトのライフサイクル管理における「破棄」の最終手段です。
  • 展開済みオブジェクトのデータはすべて eoh_context 内に存在することが前提となっているため、このコンテキストを削除することで安全にメモリを解放できます。
