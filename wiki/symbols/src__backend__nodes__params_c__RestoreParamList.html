RestoreParamList

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/params.c:291

目的

シリアライズされたバイナリデータから ParamListInfo 構造体を再構築します。

シグネチャまたは形状

ParamListInfo RestoreParamList(char **start_address)

入力と出力

  • **入力**: char **start_address - シリアライズされたデータが格納されているメモリ領域へのポインタ。関数実行後、ポインタは読み取ったデータの分だけ進められます。
  • **出力**: 再構築された ParamListInfo 構造体へのポインタ。

前提条件と不変条件

  • start_address が指すメモリ領域には、SerializeParamList 等で生成された有効なバイナリデータが格納されている必要があります。
  • 内部で makeParamList を呼び出すため、呼び出し側のメモリコンテキストが適切である必要があります。

アルゴリズムまたは振る舞い

1. start_address からパラメータ数 (nparams) を読み取り、ポインタを進めます。 2. makeParamList(nparams) を呼び出し、必要なメモリを確保した ParamListInfo を作成します。 3. パラメータ数分だけループし、各 ParamExternData に対して以下を順次読み取ります。

  • 型 OID (ptype)
  • フラグ (pflags)
  • 値 (value) と NULL 判定 (isnull) を datumRestore を用いて復元。

    • 4. 復元した ParamListInfo を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:1279
  • **呼び出し先**: makeParamList, datumRestore

関連データ構造

  • ParamListInfo: パラメータリストを保持する主要構造体。
  • ParamExternData: 個々のパラメータの型、フラグ、値を保持する構造体。

失敗モードと境界条件

  • start_address が指すデータが破損している場合、メモリの不正アクセスや誤ったデータ読み込みが発生する可能性があります。
  • makeParamList がメモリ不足で失敗した場合、例外が発生します。

保守担当者向けメモ

  • 本関数は SerializeParamList と対になる関数です。シリアライズ形式を変更する場合は、必ず本関数の読み込みロジックも同期して更新してください。
  • paramValuesStr はシリアライズ対象外であるため、復元後の ParamListInfo には含まれません。
