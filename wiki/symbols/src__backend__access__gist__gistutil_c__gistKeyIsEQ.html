gistKeyIsEQ

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistutil.c:280

目的

GiSTインデックスにおいて、指定された属性番号に対応する等価性比較関数（equalFn）を呼び出し、2つの Datum 値が等しいかどうかを判定します。

シグネチャまたは形状

bool gistKeyIsEQ(GISTSTATE *giststate, int attno, Datum a, Datum b)

入力と出力

  • **入力**:
  • giststate: GiSTインデックスの状態管理構造体。
  • attno: 比較対象の属性番号。
  • a, b: 比較する2つの Datum 値。
  • **出力**:
  • bool: a と b が等しい場合は true、それ以外は false を返します。

前提条件と不変条件

  • giststate は初期化済みであり、指定された attno に対応する equalFn および supportCollation が有効である必要があります。

アルゴリズムまたは振る舞い

1. giststate->equalFn[attno] に格納された関数を、giststate->supportCollation[attno] を照合順序として使用して呼び出します。 2. 比較結果を result 変数に格納し、その値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistsplit.c:497
  • src/backend/access/gist/gistutil.c:352
  • **呼び出し先**:
  • FunctionCall3Coll（PostgreSQLの関数呼び出しインターフェース）

関連データ構造

  • GISTSTATE: GiSTインデックスの操作に必要なサポート関数や照合順序情報を保持する構造体。

失敗モードと境界条件

  • attno が giststate の範囲外である場合、メモリ破壊やセグメンテーション違反が発生する可能性があります。
  • 比較関数自体がエラーを投げる場合、本関数も中断されます。

保守担当者向けメモ

  • 本関数は giststate 内の equalFn 配列を直接参照します。インデックスの opclass 定義が正しく行われていることが前提となります。
