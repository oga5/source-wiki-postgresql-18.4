brin_summarize_new_values

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin.c:1365

目的

BRINインデックスにおいて、まだ要約されていないすべてのページ範囲を対象として、要約処理を実行するためのインターフェースを提供します。

シグネチャまたは形状

Datum brin_summarize_new_values(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して、対象となるリレーション（インデックス）の Datum を受け取ります。
  • **出力**: DirectFunctionCall2 の戻り値である Datum を返します。

前提条件と不変条件

  • 本関数は PostgreSQL の関数インターフェースとして呼び出されることを前提としています。
  • 内部的に BRIN_ALL_BLOCKRANGES を指定して brin_summarize_range を呼び出すため、インデックス内の全未要約範囲が処理対象となります。

アルゴリズムまたは振る舞い

1. 引数からリレーションの Datum を取得します。 2. brin_summarize_range 関数を呼び出します。その際、第2引数に BRIN_ALL_BLOCKRANGES を渡すことで、特定の範囲ではなく全範囲を対象とするよう指示します。 3. brin_summarize_range の実行結果をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の記述から、SQLインターフェース経由での呼び出しが想定されます。
  • **呼び出し先**: brin_summarize_range を直接呼び出します。

関連データ構造

  • Datum: PostgreSQL の汎用データ型。
  • BRIN_ALL_BLOCKRANGES: 全てのブロック範囲を処理対象とするための定数。

失敗モードと境界条件

  • brin_summarize_range が失敗した場合、そのエラーがそのまま伝播します。
  • 対象のリレーションが BRIN インデックスでない場合や、アクセス権限がない場合は、基盤となる関数によってエラーが発生します。

保守担当者向けメモ

  • 本関数は brin_summarize_range のラッパーです。ロジックの変更が必要な場合は、主に brin_summarize_range 側の実装を確認してください。
  • BRIN_ALL_BLOCKRANGES の定義や挙動に変更があった場合、本関数の動作にも影響が及びます。
