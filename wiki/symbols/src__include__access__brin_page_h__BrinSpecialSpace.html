BrinSpecialSpace

定義

  • 種別: struct
  • 定義箇所: src/include/access/brin_page.h:29

目的

BRIN（Block Range Index）インデックスの各ページ末尾に配置され、ページタイプやフラグ情報を保持するための構造体です。

シグネチャまたは形状

typedef struct BrinSpecialSpace
{
	uint16		vector[MAXALIGN(1) / sizeof(uint16)];
} BrinSpecialSpace;

入力と出力

  • 入力: なし（データ構造定義）
  • 出力: なし

前提条件と不変条件

  • ページ末尾に配置されるため、アクセスには PageGetSpecialPointer を使用する必要があります。
  • 構造体のサイズは MAXALIGN によってパディングされ、アライメントが保証されています。

アルゴリズムまたは振る舞い

  • 本構造体は、BRINインデックスのページタイプ（メタページ、リバースマップ、データページ）を識別するための基盤として機能します。
  • BrinPageType(page) マクロや BRIN_IS_*_PAGE(page) マクロを通じて、ページヘッダーの特殊領域から情報を抽出・判定するために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • contrib/pageinspect/brinfuncs.c: ページサイズ検証に使用。
  • src/backend/access/brin/brin_bloom.c: ページレイアウト計算に使用。
  • src/backend/access/brin/brin_pageops.c: ページ初期化およびサイズ検証に使用。
  • src/include/access/brin_page.h: ページタイプ判定マクロ内で使用。
  • 呼び出し先: なし（データ構造定義のため）

関連データ構造

  • BrinMetaPageData: インデックスのメタデータ構造。
  • RevmapContents: リバースマップのデータ構造。

失敗モードと境界条件

  • ページサイズが MAXALIGN(sizeof(BrinSpecialSpace)) 未満である場合、PageGetSpecialSize による検証で不整合として扱われます。

保守担当者向けメモ

  • 本構造体は可変長データを含む可能性があるため、メモリレイアウトを計算する際は必ず MAXALIGN を介したマクロを使用してください。
  • ページタイプフラグの追加や変更を行う際は、src/include/access/brin_page.h 内の関連マクロ定義と整合性を取ってください。
