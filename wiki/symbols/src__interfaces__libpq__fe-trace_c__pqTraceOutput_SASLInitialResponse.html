pqTraceOutput_SASLInitialResponse

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-trace.c:381

目的

SASL初期応答メッセージ（SASLInitialResponse）の内容をデバッグ用トレース出力ストリームへ書き出す。

シグネチャまたは形状

static void pqTraceOutput_SASLInitialResponse(FILE *f, const char *message, int *cursor, bool regress)

入力と出力

  • f: 出力先のファイルポインタ。
  • message: 解析対象のメッセージデータを含むバッファ。
  • cursor: 現在の読み取り位置を示すポインタ（関数内で更新される）。
  • regress: 回帰テストモードフラグ。true の場合、可変情報がサニタイズされる。

前提条件と不変条件

  • cursor は有効なメモリ領域を指していること。
  • f は書き込み可能な状態であること。

アルゴリズムまたは振る舞い

1. 文字列 "SASLInitialResponse\t" を出力する。 2. pqTraceOutputString を呼び出し、メッセージ内の文字列フィールドを出力する。 3. pqTraceOutputInt32 を呼び出し、続く4バイトの整数（initialResponse）を読み取り出力する。 4. initialResponse が -1 でない場合、pqTraceOutputNchar を呼び出し、指定された長さのデータを regress モードに従って出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-trace.c 内のメッセージ解析ルーチンから呼び出される（直接の呼び出し箇所として fe-trace.c:733 が確認されている）。
  • **呼び出し先**: pqTraceOutputString, pqTraceOutputInt32, pqTraceOutputNchar を呼び出す。

関連データ構造

  • FILE: 標準ライブラリのファイルストリーム。

失敗モードと境界条件

  • initialResponse が -1 の場合、後続のデータ出力はスキップされる。
  • regress フラグが true の場合、pqTraceOutputNchar 内で出力内容が抑制または置換される。

保守担当者向けメモ

  • 本関数はプロトコルメッセージのデコードを行うため、プロトコル仕様に変更があった場合は pqTraceOutputInt32 や pqTraceOutputNchar の呼び出し順序や引数を確認すること。
