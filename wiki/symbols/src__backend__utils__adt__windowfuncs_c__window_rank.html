window_rank

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/windowfuncs.c:137

目的

SQLの RANK() ウィンドウ関数を実装し、現在のパーティション内における行の順位を計算します。

シグネチャまたは形状

Datum window_rank(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して WindowObject を受け取ります。
  • **出力**: int64 型の順位（Datum）を返します。

前提条件と不変条件

  • WinGetPartitionLocalMemory を使用して、パーティションごとの rank_context を保持・更新する必要があります。
  • 順位の更新は、rank_up 関数によるピア（同順位）判定の結果に基づいて行われます。

アルゴリズムまたは振る舞い

1. PG_WINDOW_OBJECT() を呼び出し、ウィンドウ操作用のオブジェクトを取得します。 2. rank_up(winobj) を呼び出し、現在の行が前の行と異なる順位グループに属するかを判定します。 3. WinGetPartitionLocalMemory を介して、パーティション固有の rank_context 構造体を取得します。 4. rank_up が真（up == true）の場合、現在の行位置（WinGetCurrentPosition）に1を加算した値を新しい順位として context->rank に格納します。 5. 保持されている context->rank を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのウィンドウ関数実行エンジン（src/backend/executor/nodeWindowAgg.c 等から間接的に呼び出されます）。
  • **呼び出し先**:
  • rank_up(): 順位更新の判定。
  • WinGetPartitionLocalMemory(): パーティションごとの状態保持。
  • WinGetCurrentPosition(): 現在の行位置の取得。
  • PG_RETURN_INT64(): 結果の返却。

関連データ構造

  • rank_context: パーティションごとの順位状態を保持する構造体。
  • WindowObject: ウィンドウ関数実行用のAPIコンテキスト。

失敗モードと境界条件

  • パーティションの先頭行では rank_up が常に真となり、順位は1となります。
  • ピア（同順位）の行が連続する場合、rank_up は偽を返し、順位は更新されません。

保守担当者向けメモ

  • 本関数は window_rank_support 関数と対になっており、クエリ計画時に最適化ヒントを提供します。
  • ピア判定のロジックを変更する場合は、rank_up の実装を確認してください。
