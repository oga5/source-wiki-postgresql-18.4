DictSyn

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/dict_synonym.c:31

目的

PostgreSQLの全文検索における「シノニム（同義語）辞書」の全体状態を保持する構造体です。辞書ファイルから読み込まれた同義語ペアの配列と、検索時の動作設定を管理します。

シグネチャまたは形状

typedef struct {
    int         len;            /* syn配列の要素数 */
    Syn        *syn;            /* 同義語ペアの配列 */
    bool        case_sensitive; /* 大文字小文字の区別フラグ */
} DictSyn;

入力と出力

本構造体はデータ保持用であり、直接的な関数入出力は持ちません。初期化関数 dsynonym_init によってメモリ上に構築され、置換関数 dsynonym_lexize によって検索対象として参照されます。

前提条件と不変条件

  • syn 配列は、初期化時に qsort によってソートされている必要があります。
  • len は syn 配列の有効な要素数と一致している必要があります。

アルゴリズムまたは振る舞い

  • 辞書検索時には、保持している syn 配列に対して bsearch を実行し、高速な単語置換を実現します。
  • case_sensitive フラグに基づき、入力単語の小文字化の要否を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/dict_synonym.c 内の dsynonym_init および dsynonym_lexize。
  • contrib/dict_xsyn/dict_xsyn.c において、外部モジュールとして参照・操作されています。
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • Syn: DictSyn が保持する個別の同義語ペア（入力語と出力語）を定義する構造体。

失敗モードと境界条件

  • メモリ不足時: palloc0 によるメモリ確保に失敗する可能性があります。
  • 辞書ファイルが空の場合: len が 0 となり、検索は常に不一致となります。

保守担当者向けメモ

  • palloc0 を使用してメモリ管理を行っているため、大規模な辞書を読み込む際はメモリ消費量に留意してください。
  • contrib/dict_xsyn からも参照されているため、構造体の変更は互換性に影響を与える可能性があります。
