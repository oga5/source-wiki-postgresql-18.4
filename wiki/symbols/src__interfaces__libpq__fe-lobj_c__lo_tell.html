lo_tell

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-lobj.c:514

目的

開いているラージオブジェクトの現在の読み書き位置（オフセット）を取得します。

シグネチャまたは形状

int lo_tell(PGconn *conn, int fd);

入力と出力

  • **入力**:
  • conn: データベース接続を管理する PGconn 構造体へのポインタ。
  • fd: 対象となるラージオブジェクトのファイルディスクリプタ。
  • **出力**:
  • 成功時: 現在のオフセット位置（非負の整数）。
  • 失敗時: -1。

前提条件と不変条件

  • 呼び出し前に lo_initialize(conn) が実行され、必要なサーバー側関数のOIDがキャッシュされている必要があります。
  • fd は、有効なラージオブジェクトのディスクリプタである必要があります。

アルゴリズムまたは振る舞い

1. lo_initialize を呼び出し、接続の初期化状態を確認します。失敗した場合は -1 を返します。 2. fd を PQArgBlock に格納し、サーバー側の lo_tell 関数を PQfn（Fastpath）経由で実行します。 3. サーバーからの応答が PGRES_COMMAND_OK であれば、取得したオフセット値を返します。 4. 処理の成否に関わらず、PQfn が生成した PGresult を PQclear で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: 外部アプリケーションから呼び出される公開APIです。
  • **呼び出し先**:
  • lo_initialize: 接続の初期化。
  • PQfn: サーバー側関数 lo_tell の実行。
  • PQresultStatus, PQclear: 実行結果の確認とメモリ解放。

関連データ構造

  • PGconn: 接続状態およびキャッシュされた関数OID（fn_lo_tell）を保持します。
  • PQArgBlock: PQfn に渡す引数構造体。

失敗モードと境界条件

  • lo_initialize が失敗した場合、-1 を返します。
  • サーバー側でエラーが発生した場合（例: 無効なディスクリプタ）、PQresultStatus が PGRES_COMMAND_OK 以外となり、-1 を返します。
  • 戻り値の型が int であるため、非常に大きなオフセット値（2GBを超える場合など）を扱う際には注意が必要です。

保守担当者向けメモ

  • 本関数は int を返すため、64ビットオフセットが必要な場合は lo_tell64 の使用を検討してください。
  • サーバー側の lo_tell 関数OIDは conn->lobjfuncs->fn_lo_tell にキャッシュされます。
