目的

ECPG（Embedded SQL for C）における動的SQL実行、記述子（Descriptor）を用いたデータ取得、およびNULL値ハンドリングの回帰テストを実行する。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は使用しない）
  • 出力: 標準出力にテスト結果（列データおよびNULL判定）を書き出す。終了コード 0 を返す。

前提条件と不変条件

  • ECPGライブラリが初期化され、データベース ecpg1_regression への接続が可能であること。
  • sqlca（SQL通信領域）が各SQL実行後のエラーチェックに使用される。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGconnect でデータベースに接続し、test テーブルを作成してデータを投入する（NULL値を含む）。 2. **記述子確保**: ECPGallocate_desc で mydesc を確保する。 3. **クエリ実行**: ECPGdo で SELECT * を実行し、結果を記述子にバインドする。 4. **データ抽出**: ECPGget_desc_header で列数を取得し、ECPGget_desc を用いて各列のデータおよびインジケータ（NULL判定用）をホスト変数に格納する。 5. **検証**: ループ内でインジケータを確認し、NULLであれば "NULL" を、値があればその内容を標準出力へ出力する。 6. **終了処理**: ECPGfree_auto_mem でメモリを解放し、記述子の破棄と接続解除を行う。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先: ECPGconnect, ECPGdo, ECPGallocate_desc, ECPGget_desc_header, ECPGget_desc, ECPGfree_auto_mem, ECPGdeallocate_desc, ECPGtrans, ECPGdisconnect, sqlprint, printf

関連データ構造

  • sqlca: SQL実行状態を保持する通信領域。
  • mydesc: SQL記述子（Descriptor）。

失敗モードと境界条件

  • 各SQL実行後に sqlca.sqlcode < 0 をチェックしており、エラー発生時は sqlprint() を呼び出して詳細を出力する。
  • NULL値を含むレコードの取得時に、インジケータ変数（ipointer1, ipointer2）が正しくNULLを検知できるかを境界条件として検証する。

保守担当者向けメモ

  • 本関数はプリプロセッサ生成コードであるため、直接編集しないこと。
  • ECPGdebug(1, stderr) が有効であり、実行時の詳細ログが標準エラー出力に流れる。
  • NULL値のハンドリング（ECPGd_indicator）の挙動を確認するための重要なテストケースである。
