DbLocaleInfo

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.h:209

目的

PostgreSQLのデータベースにおけるロケール設定およびエンコーディング情報を保持し、アップグレード前後でこれらの属性が整合しているかを確認するために使用されます。

シグネチャまたは形状

typedef struct
{
	char	   *db_collate;
	char	   *db_ctype;
	char		db_collprovider;
	char	   *db_locale;
	int			db_encoding;
} DbLocaleInfo;

入力と出力

本構造体はデータ保持用であり、関数ではありません。info.c 等でメモリ確保（pg_malloc）が行われ、データベースのメタデータから取得した値が各フィールドに格納されます。

前提条件と不変条件

  • db_collate, db_ctype, db_locale は文字列へのポインタであり、有効なロケール名である必要があります。
  • db_encoding はPostgreSQLで定義されたエンコーディングIDを保持します。

アルゴリズムまたは振る舞い

本構造体自体はロジックを持ちませんが、pg_upgrade の実行過程において、旧クラスタと新クラスタの template0 データベースのロケール情報を比較する際のデータコンテナとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/info.c: データベース情報の収集時にインスタンス化されます。
  • src/bin/pg_upgrade/pg_upgrade.c: old_cluster.template0 を通じて参照されます。
  • **呼び出し先**:
  • 提供されたソース情報からは、本構造体から直接呼び出される関数や、本構造体を引数として呼び出される特定の関数関係は特定できません。

関連データ構造

  • ClusterInfo: アップグレード対象のクラスタ情報を保持し、そのメンバとして template0（DbLocaleInfo 型）を保持します。

失敗モードと境界条件

  • メモリ不足: pg_malloc によるメモリ確保時に失敗する可能性があります。
  • 不整合: 旧クラスタと新クラスタでロケール設定が一致しない場合、アップグレード処理が中断される可能性があります。

保守担当者向けメモ

  • 本構造体は pg_upgrade における互換性チェックの要です。ロケール関連のパラメータがPostgreSQLのバージョン間で変更された場合、本構造体の定義を見直す必要があります。
  • db_collprovider の扱いや、新しいロケール関連の属性が追加された場合は、この構造体へのフィールド追加を検討してください。
