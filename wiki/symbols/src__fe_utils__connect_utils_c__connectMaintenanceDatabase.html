connectMaintenanceDatabase

定義

  • 種別: function
  • 定義箇所: src/fe_utils/connect_utils.c:133

目的

PostgreSQLのメンテナンス用データベース（postgres または template1）への接続を確立するためのラッパー関数です。

シグネチャまたは形状

PGconn *connectMaintenanceDatabase(ConnParams *cparams, const char *progname, bool echo);

入力と出力

  • **入力**:
  • cparams: 接続パラメータを保持する ConnParams 構造体へのポインタ。
  • progname: ログ出力に使用されるプログラム名。
  • echo: クエリの実行状況を表示するかどうかを示すフラグ。
  • **出力**:
  • 接続成功時は PGconn *（データベース接続ハンドル）を返します。
  • 接続失敗時は NULL を返します。

前提条件と不変条件

  • cparams->dbname が指定されている場合、そのデータベースへの接続を優先します。
  • cparams->dbname が指定されていない場合、postgres データベースへの接続を試み、失敗した場合は template1 データベースへの接続を試みます。

アルゴリズムまたは振る舞い

1. cparams->dbname が設定されている場合、connectDatabase を呼び出して接続を試行し、その結果を返します。 2. cparams->dbname が設定されていない場合:

  • cparams->dbname を "postgres" に設定し、connectDatabase を呼び出します。
  • 接続が成功すればそのハンドルを返します。
  • 接続が失敗した場合、cparams->dbname を "template1" に書き換え、再度 connectDatabase を呼び出してその結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/scripts/clusterdb.c
  • src/bin/scripts/createdb.c
  • src/bin/scripts/createuser.c
  • src/bin/scripts/dropdb.c
  • src/bin/scripts/dropuser.c
  • src/bin/scripts/reindexdb.c
  • src/bin/scripts/vacuumdb.c
  • **呼び出し先**:
  • connectDatabase (同一ファイル内)

関連データ構造

  • ConnParams: 接続先ホスト、ポート、ユーザー名、DB名などを保持する構造体。

失敗モードと境界条件

  • postgres および template1 の両方への接続が失敗した場合、関数は NULL を返します。
  • 呼び出し元は、戻り値が NULL である可能性を考慮してエラーハンドリングを行う必要があります。

保守担当者向けメモ

  • 本関数は postgres を優先し、失敗時に template1 を試すというPostgreSQLの標準的な慣習を実装しています。
  • cparams 構造体の dbname フィールドは、関数内で書き換えられる可能性があるため注意してください。
