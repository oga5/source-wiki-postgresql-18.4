TokenAuxData

定義

  • 種別: struct
  • 定義箇所: src/pl/plpgsql/src/pl_scanner.c:91

目的

PL/pgSQL の字句解析において、トークンに関連するセマンティック情報、ソースコード上の位置情報、およびバイト長を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	YYSTYPE		lval;			/* セマンティック情報 */
	YYLTYPE		lloc;			/* スキャンバッファ内のオフセット */
	int			leng;			/* バイト単位の長さ */
} TokenAuxData;

入力と出力

本構造体はデータ保持用であり、関数ではありません。主に internal_yylex などの字句解析関数において、トークン情報の受け渡しに使用されます。

前提条件と不変条件

  • leng はトークンのバイト長を正しく反映している必要があります。
  • lloc はソースコード内の有効なオフセット範囲を指している必要があります。

アルゴリズムまたは振る舞い

字句解析器がトークンを生成する際、そのトークンに関連するメタデータを一括して管理するために使用されます。特に、構文解析の先読み（プッシュバック）が必要な際、トークン本体とこの補助データをセットでスタックに保持することで、解析状態の復元を可能にします。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • pl_scanner.c 内の pushback_auxdata 配列の要素として定義。
  • internal_yylex 関数の引数として使用。
  • push_back_token 関数の引数として使用。
  • plpgsql_yylex 内のローカル変数として使用。
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • YYSTYPE: トークンのセマンティック値を保持する型。
  • YYLTYPE: トークンの位置情報を保持する型。
  • plpgsql_yy_extra_type: スキャナの状態を保持する構造体（本構造体を含むプッシュバックスタックを管理）。

失敗モードと境界条件

  • MAX_PUSHBACKS を超えるトークンのプッシュバックはサポートされていません。
  • leng が負の値になるような不正なトークン生成は、後続の構文解析エラーを引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は plpgsql_yylex によるトークン化プロセスの中核をなすデータ保持ユニットです。修正を行う際は、pl_gram.y（構文解析器）が期待する lval および lloc の形式と整合性が取れていることを確認してください。
