finish_spin_delay

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/s_lock.c:185

目的

スピンロック取得後の試行結果に基づき、プロセスローカルなスピン回数の閾値（spins_per_delay）を調整し、次回のロック取得試行における待機戦略を最適化します。

シグネチャまたは形状

void finish_spin_delay(SpinDelayStatus *status)

入力と出力

  • **入力**: SpinDelayStatus *status - ロック取得試行のステータスを保持する構造体。cur_delay フィールドが 0 か否かで、待機（pg_usleep）が発生したかを判定します。
  • **出力**: なし（副作用としてグローバル変数 spins_per_delay を更新）。

前提条件と不変条件

  • spins_per_delay は MIN_SPINS_PER_DELAY から MAX_SPINS_PER_DELAY の範囲内に維持されます。
  • status ポインタは有効である必要があります。

アルゴリズムまたは振る舞い

1. **待機が発生しなかった場合 (status->cur_delay == 0)**:

  • ロックが即座に取得できたと判断し、spins_per_delay を 100 増加させます（上限 MAX_SPINS_PER_DELAY まで）。これにより、次回以降はより長くスピンを継続するようになります。

    • 2. **待機が発生した場合 (status->cur_delay != 0)**:

  • 競合が激しいと判断し、spins_per_delay を 1 減少させます（下限 MIN_SPINS_PER_DELAY まで）。これにより、次回以降はより早い段階で pg_usleep による待機へ移行し、CPU 負荷を軽減します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c
  • src/backend/storage/lmgr/lwlock.c
  • src/backend/storage/lmgr/s_lock.c
  • **呼び出し先**:
  • 直接的な関数呼び出しはありませんが、Min および Max マクロを使用して spins_per_delay の値を制御します。

関連データ構造

  • SpinDelayStatus: スピンロックの待機状態を追跡する構造体。
  • spins_per_delay: プロセス全体で共有されるスピン回数の閾値。

失敗モードと境界条件

  • spins_per_delay が MIN_SPINS_PER_DELAY または MAX_SPINS_PER_DELAY に達している場合、それ以上の増減は行われず、境界値に固定されます。

保守担当者向けメモ

  • 本関数は適応型スピン待機アルゴリズムの核心部です。spins_per_delay の調整幅（+100 / -1）は、競合の激しさと CPU 負荷のトレードオフを考慮して設定されています。変更時はマルチプロセッサ環境での性能への影響を十分に評価してください。
