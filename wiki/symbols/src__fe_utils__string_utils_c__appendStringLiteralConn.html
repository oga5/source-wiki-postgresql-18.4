appendStringLiteralConn

定義

  • 種別: function
  • 定義箇所: src/fe_utils/string_utils.c:445

目的

文字列をSQLリテラル形式に変換し、指定された PQExpBuffer に安全に追加します。データベース接続情報を利用して、適切なエスケープ処理を行います。

シグネチャまたは形状

void appendStringLiteralConn(PQExpBuffer buf, const char *str, PGconn *conn)

入力と出力

  • buf (入力/出力): 文字列を追加する対象のバッファ。
  • str (入力): SQLリテラルとして追加する対象の文字列。
  • conn (入力): データベース接続ハンドル。エンコーディングの判定やエスケープ処理に使用されます。
  • 戻り値: なし。

前提条件と不変条件

  • buf は初期化済みである必要があります。
  • conn は有効なデータベース接続である必要があります。
  • str が NULL の場合、未定義の動作を引き起こす可能性があります。

アルゴリズムまたは振る舞い

1. **エスケープ文字列構文の処理**: 文字列内にバックスラッシュ (\) が含まれ、かつサーバーバージョンが 8.1 以上の場合、ESCAPE_STRING_SYNTAX (E'...') を使用します。この際、バッファの末尾がスペースでない場合はスペースを挿入して識別子との隣接を防ぎます。 2. **標準的なリテラル処理**: 上記以外の場合、バッファを 2 * length + 2 分拡張し、シングルクォートで囲んだ上で PQescapeStringConn を用いてエスケープされた文字列をバッファに書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_amcheck.c、dumputils.c、pg_dump.c、pg_dumpall.c など、クライアント側のクエリ構築処理から広く利用されています。
  • **呼び出し先**: strlen、strchr、PQserverVersion、PQclientEncoding、appendPQExpBufferChar、appendStringLiteral、enlargePQExpBuffer、PQescapeStringConn を呼び出します。

関連データ構造

  • PQExpBuffer: 文字列を動的に保持・拡張するための構造体。
  • PGconn: libpq のデータベース接続状態を保持する構造体。

失敗モードと境界条件

  • enlargePQExpBuffer がメモリ確保に失敗した場合、関数は何も追加せずに早期リターンします。
  • PQescapeStringConn の結果によりバッファサイズが不足する場合、メモリ確保の成否が処理の継続を左右します。

保守担当者向けメモ

  • 本関数内の ESCAPE_STRING_SYNTAX を使用する条件分岐は、ユーティリティプログラムにおける escape_string_warning を抑制するための暫定的な措置（kluge）です。将来的なリファクタリングの対象です。
