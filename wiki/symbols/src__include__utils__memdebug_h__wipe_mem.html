wipe_mem

定義

  • 種別: function
  • 定義箇所: src/include/utils/memdebug.h:38

目的

解放されたメモリ領域を特定のパターン（0x7F）で上書きし、メモリの再利用時に発生する誤検知や不正なデータ参照を防止・検出する。

シグネチャまたは形状

static inline void wipe_mem(void *ptr, size_t size)

入力と出力

  • ptr: 消去対象となるメモリ領域の先頭ポインタ。
  • size: 消去対象のバイト数。
  • 戻り値: なし。

前提条件と不変条件

  • ptr が指すメモリ領域は、呼び出し元において書き込み可能であること。
  • Valgrind環境下では、対象領域がアクセス可能な状態である必要がある（内部で VALGRIND_MAKE_MEM_UNDEFINED を呼び出すため）。

アルゴリズムまたは振る舞い

1. VALGRIND_MAKE_MEM_UNDEFINED(ptr, size) を呼び出し、Valgrindに対して対象領域を「定義済みだが値は不定」の状態として通知する。 2. memset(ptr, 0x7F, size) を実行し、指定された領域を 0x7F で埋める。 3. VALGRIND_MAKE_MEM_NOACCESS(ptr, size) を呼び出し、Valgrindに対して対象領域を「アクセス不可」としてマークし、以降の不正な読み書きを検知可能にする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/list.c
  • src/backend/storage/aio/read_stream.c
  • src/backend/utils/mmgr/aset.c
  • src/backend/utils/mmgr/bump.c
  • **呼び出し先**:
  • memset (標準ライブラリ)
  • VALGRIND_MAKE_MEM_UNDEFINED (Valgrindマクロ)
  • VALGRIND_MAKE_MEM_NOACCESS (Valgrindマクロ)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • ptr が NULL の場合、あるいは size が確保された領域を超えている場合、memset によるセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数はデバッグ支援用であり、主にメモリコンテキストの解放処理に関連して使用される。
  • VALGRIND_... マクロは valgrind/memcheck.h に依存しており、ビルド環境によって動作が切り替わる。
  • 本番環境ではオーバーヘッドを避けるため、適切なビルドフラグにより無効化される設計となっている。
