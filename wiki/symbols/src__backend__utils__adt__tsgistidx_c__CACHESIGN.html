CACHESIGN

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsgistidx.c:560

目的

GiST インデックスの分割処理（picksplit）において、ハミング距離計算の効率化を図るためのキャッシュ構造体です。

シグネチャまたは形状

typedef struct
{
	bool		allistrue;
	BITVECP		sign;
} CACHESIGN;

入力と出力

  • **入力**: fillcache 関数により、SignTSVector から本構造体へデータが抽出・格納されます。
  • **出力**: hemdistcache 関数が、本構造体ペアのハミング距離を計算するために使用します。

前提条件と不変条件

  • sign メンバは BITVECP 型であり、ビットベクトルへのポインタを保持します。
  • allistrue が true の場合、対象のインデックスキーがすべてのビットを立てている状態（全包含）であることを示します。

アルゴリズムまたは振る舞い

  • インデックスのページ分割時に、エントリごとのシグネチャを毎回再計算するコストを避けるため、事前に CACHESIGN 配列としてキャッシュします。
  • hemdistcache は、このキャッシュされた構造体を用いて、インデックスのオーバーラップを最小化するための距離計算を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsgistidx.c 内の gtsvector_picksplit 関数が、キャッシュ生成および距離計算のために利用します。
  • **呼び出し先**: fillcache および hemdistcache 関数が本構造体を引数として操作します。
  • **その他**: contrib/pg_trgm/trgm_gist.c においても同名の構造体が定義・利用されていますが、本ページは tsgistidx.c の定義に基づいています。

関連データ構造

  • SignTSVector: インデックスキーの元となる構造体。
  • BITVECP: ビットベクトルを指すポインタ型。

失敗モードと境界条件

  • palloc によるメモリ確保失敗時にエラーが発生する可能性があります。
  • siglen（シグネチャ長）が不整合な場合、hemdistcache での距離計算結果が不正になる可能性があります。

保守担当者向けメモ

  • 本構造体は tsgistidx.c と trgm_gist.c の両方で定義されています。修正を行う際は、両方の実装に影響がないか確認してください。
  • allistrue フラグの扱いは、検索時の適合性判定ロジックと密接に関係しているため、変更時は gtsvector_consistent への影響を考慮してください。
