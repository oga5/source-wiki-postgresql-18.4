RI_Initial_Check

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/ri_triggers.c:1518

目的

ALTER TABLE ADD FOREIGN KEY 実行時などにおいて、既存のテーブルデータが外部キー制約を満たしているかを検証するための初期チェックを実行します。

シグネチャまたは形状

bool RI_Initial_Check(Trigger *trigger, Relation fk_rel, Relation pk_rel)

入力と出力

  • **入力**:
  • trigger: 実行対象のトリガー定義。
  • fk_rel: 外部キーを持つテーブル（参照側）。
  • pk_rel: 参照先のテーブル（被参照側）。
  • **出力**:
  • bool: 制約違反がない場合は true、検証クエリの実行権限不足やRLS制約により検証をスキップ（または失敗）した場合は false を返します。

前提条件と不変条件

  • 呼び出し元は、検証対象の制約情報を ri_FetchConstraintInfo を通じて取得します。
  • 検証クエリを実行するために、対象テーブルに対する ACL_SELECT 権限が必要です。
  • 行レベルセキュリティ（RLS）が有効な場合、呼び出し元が BYPASSRLS 権限を持つか、テーブル所有者である必要があります。

アルゴリズムまたは振る舞い

1. **権限確認**: ExecCheckPermissions を使用し、両テーブルに対する SELECT 権限を検証します。 2. **RLSチェック**: has_bypassrls_privilege および object_ownercheck を用いて、RLSによるアクセス制限を評価します。 3. **クエリ構築**: 外部キー制約のキー列に基づき、LEFT OUTER JOIN を用いた検証用SQLを動的に生成します。

  • MATCH SIMPLE または MATCH FULL の設定に応じ、WHERE 句の条件（IS NULL / IS NOT NULL）を切り替えます。
  • 異なる照合順序（Collation）を持つ列の比較には COLLATE 句を付与します。

    • 4. **実行準備**: クエリ実行効率化のため、一時的に work_mem を調整する処理（ソースコード上では guc.c への設定変更指示）が含まれます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c:13733
  • **呼び出し先**:
  • ri_FetchConstraintInfo
  • makeNode / lappend (リスト操作)
  • ExecCheckPermissions
  • has_bypassrls_privilege / object_ownercheck
  • initStringInfo / appendStringInfo (クエリ構築)
  • ri_GenerateQual / ri_GenerateQualCollation

関連データ構造

  • RI_ConstraintInfo: 外部キー制約のメタデータ（キー列番号、マッチングタイプ等）。
  • RangeTblEntry / RTEPermissionInfo: 検証クエリの権限チェックに使用されるノード。
  • StringInfoData: 動的クエリ文字列を保持するバッファ。

失敗モードと境界条件

  • 権限不足やRLS制約により検証が実行できない場合、false を返して呼び出し元に制御を戻します。
  • パーティションテーブルの場合、ONLY キーワードの有無を制御してクエリを生成します。

保守担当者向けメモ

  • 本関数はメンテナンス操作として扱われるため、実行時に maintenance_work_mem を利用する設計となっています。
  • hash_mem_multiplier を一時的に 1 に設定する処理が含まれており、GUC設定の副作用に注意してください。
  • 実行されるクエリは SPI を介して処理されます。
