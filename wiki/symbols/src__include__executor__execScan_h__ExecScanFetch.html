ExecScanFetch

定義

  • 種別: static inline function
  • 定義箇所: src/include/executor/execScan.h:31

目的

スキャンノードにおけるタプル取得処理を抽象化し、通常の実行パスと EvalPlanQual (EPQ) による再評価パスを統合して提供する。

シグネチャまたは形状

static pg_attribute_always_inline TupleTableSlot *
ExecScanFetch(ScanState *node,
              EPQState *epqstate,
              ExecScanAccessMtd accessMtd,
              ExecScanRecheckMtd recheckMtd)

入力と出力

  • **入力**:
  • node: 現在のスキャン状態 (ScanState)。
  • epqstate: EPQ 実行状態（NULL の場合は通常スキャン）。
  • accessMtd: 通常のタプル取得を行う関数ポインタ。
  • recheckMtd: EPQ 時の条件再検証を行う関数ポインタ。
  • **出力**: 取得したタプルを格納する TupleTableSlot。条件を満たさない場合やタプルがない場合は NULL または空のスロットを返す。

前提条件と不変条件

  • CHECK_FOR_INTERRUPTS() が呼び出され、割り込みチェックが保証される。
  • epqstate が有効な場合、scanrelid に基づいて適切な EPQ 処理（置換タプルの取得や行マークのフェッチ）が実行される。

アルゴリズムまたは振る舞い

1. **割り込みチェック**: CHECK_FOR_INTERRUPTS() を実行。 2. **EPQ モードの判定**: epqstate が存在する場合、以下のいずれかの処理を行う。

  • scanrelid == 0 (Foreign/Custom Scan): recheckMtd を呼び出し、結果を返す。
  • relsubs_done が真: 空のスロットを返す。
  • relsubs_slot が存在: 提供された置換タプルを recheckMtd で検証し、結果を返す。
  • relsubs_rowmark が存在: EvalPlanQualFetchRowMark でタプルを取得し、recheckMtd で検証して返す。

    • 3. **通常モード**: 上記のいずれにも該当しない場合、accessMtd を呼び出して次のタプルを取得し、返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の記述から、src/include/executor/execScan.h 内の他の関数（ExecScanExtended 等）から呼び出されていることが示唆されている。
  • **呼び出し先**: 引数として渡された accessMtd および recheckMtd 関数ポインタ。また、EPQ 関連の補助関数（EvalPlanQualFetchRowMark 等）。

関連データ構造

  • ScanState: スキャンノードの実行状態。
  • EPQState: EvalPlanQual の実行状態。
  • TupleTableSlot: タプルを保持する構造体。
  • ExecScanAccessMtd / ExecScanRecheckMtd: スキャン手法固有の関数ポインタ型。

失敗モードと境界条件

  • epqstate が NULL の場合、通常のアクセス手法に依存する。
  • recheckMtd が失敗（条件不一致）した場合、ExecClearTuple によりスロットがクリアされ、タプルは返されない。
  • TupIsNull(slot) の場合、適切に NULL または空スロットが返される。

保守担当者向けメモ

  • 本関数は static inline であり、コンパイル時に epqstate の有無による分岐が最適化される。
  • 新規スキャンノードを追加する際は、この関数を介してアクセス手法を統合することで、EPQ への対応を一貫させることができる。
