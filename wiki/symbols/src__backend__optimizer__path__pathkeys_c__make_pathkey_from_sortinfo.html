make_pathkey_from_sortinfo

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/pathkeys.c:197

目的

ソート情報（式、演算子族、照合順序など）に基づき、正規化された PathKey を生成または取得します。

シグネチャまたは形状

PathKey *make_pathkey_from_sortinfo(PlannerInfo *root, Expr *expr, Oid opfamily, Oid opcintype, Oid collation, bool reverse_sort, bool nulls_first, Index sortref, Relids rel, bool create_it);

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • expr: ソート対象の式。
  • opfamily: 演算子族の OID。
  • opcintype: 演算子クラスの入力型 OID。
  • collation: 照合順序。
  • reverse_sort: 降順ソートかどうか。
  • nulls_first: NULL を先頭に配置するかどうか。
  • sortref: ソート参照インデックス。
  • rel: 関連するリレーション ID。
  • create_it: 既存の EquivalenceClass がない場合に新規作成するかどうか。
  • **出力**:
  • 成功時: 正規化された PathKey へのポインタ。
  • 失敗時: create_it が false でかつ該当する EquivalenceClass が存在しない場合、NULL を返します。

前提条件と不変条件

  • opfamily に対応する等価演算子（COMPARE_EQ）がシステムカタログに存在する必要があります。
  • 取得した等価演算子から、マージ結合可能な演算子族リストを特定できる必要があります。

アルゴリズムまたは振る舞い

1. reverse_sort フラグに基づき、比較タイプ（COMPARE_LT または COMPARE_GT）を決定します。 2. opfamily から等価演算子を検索し、それに関連する演算子族リスト（opfamilies）を取得します。 3. get_eclass_for_sort_expr を呼び出し、指定された式と演算子族に適合する EquivalenceClass を検索または作成します。 4. EquivalenceClass が取得できた場合、make_canonical_pathkey を呼び出して正規化された PathKey を生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/pathkeys.c 内の複数箇所（make_pathkey_from_sortop 等のラッパー関数や、パス生成ロジックから直接呼び出されます）。
  • **呼び出し先**:
  • get_opfamily_member_for_cmptype
  • get_mergejoin_opfamilies
  • get_eclass_for_sort_expr
  • make_canonical_pathkey

関連データ構造

  • PathKey: 順序を表現する構造体。
  • EquivalenceClass: 意味的に等価な式の集合。
  • PlannerInfo: クエリ計画の全体状態。

失敗モードと境界条件

  • get_opfamily_member_for_cmptype が等価演算子を見つけられない場合、elog(ERROR) で停止します。
  • get_mergejoin_opfamilies が演算子族を見つけられない場合、elog(ERROR) で停止します。
  • create_it が false の場合、適合する EquivalenceClass が存在しなければ NULL を返します。

保守担当者向けメモ

  • 本関数は EquivalenceClass を介した正規化を前提としています。
  • make_canonical_pathkey を呼び出すことで、同一の順序表現が重複して生成されることを防いでいます。
  • 演算子族の検索ロジックは、マージ結合の可否判定と密接に関連しているため、変更時は equivclass.c の挙動と整合性を取る必要があります。
