ExplainFormat

定義

  • 種別: enum
  • 定義箇所: src/include/commands/explain_state.h:27

目的

EXPLAIN コマンドの実行結果を出力する際のフォーマットを指定するために使用されます。

シグネチャまたは形状

typedef enum ExplainFormat
{
    EXPLAIN_FORMAT_TEXT,
    EXPLAIN_FORMAT_XML,
    EXPLAIN_FORMAT_JSON,
    EXPLAIN_FORMAT_YAML,
} ExplainFormat;

入力と出力

  • 入力: EXPLAIN コマンドのオプション解析時に指定されるフォーマット値。
  • 出力: ExplainState 構造体内の format フィールドとして保持され、後続の出力処理の分岐条件として利用されます。

前提条件と不変条件

  • ExplainState 構造体の一部として、クエリ実行のライフサイクルを通じて保持されます。
  • 定義された 4 つの列挙値（TEXT, XML, JSON, YAML）のいずれかをとります。

アルゴリズムまたは振る舞い

  • EXPLAIN コマンド実行エンジンは、本列挙型の値に基づいてプランツリーの整形および出力形式を切り替えます。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/commands/explain_state.h 内の ExplainState 構造体（format フィールド）で参照されています。
  • 呼び出し先: 提供されたソース情報からは、具体的な関数レベルの呼び出し先は特定できません。

関連データ構造

  • ExplainState: EXPLAIN 実行中の状態を管理するメイン構造体であり、本型をメンバとして保持します。

失敗モードと境界条件

  • 未定義のフォーマット値が設定された場合の挙動については、本ソース excerpt からは判断できません。

保守担当者向けメモ

  • 出力形式を追加・変更する場合は、本列挙型に値を追加し、それに対応する整形ロジックを src/backend/commands/explain_state.c 等の出力処理側に実装する必要があります。
