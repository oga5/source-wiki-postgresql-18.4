SwitchToUntrustedUser

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/usercontext.c:32

目的

現在のユーザーコンテキストを一時的に別のユーザー（ロール）へ切り替えます。切り替え先ユーザーが現在のユーザーに対して権限を持たない場合、セキュリティ制限を適用し、GUC（設定パラメータ）の変更を隔離するサンドボックス環境を提供します。

シグネチャまたは形状

void SwitchToUntrustedUser(Oid userid, UserContext *context)

入力と出力

  • **入力**:
  • userid: 切り替え先のユーザーID（Oid）。
  • context: 現在の状態を保存し、切り替え後の管理情報（GUCネストレベル等）を格納するための UserContext 構造体へのポインタ。
  • **出力**: なし（void）。context 引数を通じて状態が更新されます。

前提条件と不変条件

  • 現在のユーザーが userid に対して SET ROLE する権限を持っている必要があります。権限がない場合はエラーが発生します。
  • 呼び出し側は、後続の処理終了後に必ず RestoreUserContext を呼び出し、状態を復元する必要があります。

アルゴリズムまたは振る舞い

1. GetUserIdAndSecContext を呼び出し、現在のユーザーIDとセキュリティコンテキストを context に保存します。 2. member_can_set_role を使用し、現在のユーザーが対象の userid に切り替える権限があるか検証します。権限がない場合は ERRCODE_INSUFFICIENT_PRIVILEGE でエラーを報告します。 3. 相互に SET ROLE が可能な場合は、単純に SetUserIdAndSecContext で切り替えを行い、context->save_nestlevel を -1 に設定します（GUCのロールバック不要）。 4. 片方向のみの権限である場合は、SECURITY_RESTRICTED_OPERATION をセキュリティコンテキストに付与して切り替えを行い、NewGUCNestLevel() を呼び出して context->save_nestlevel に保存します（GUCの変更を追跡可能にする）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c
  • src/backend/replication/logical/tablesync.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • GetUserIdAndSecContext
  • member_can_set_role
  • SetUserIdAndSecContext
  • NewGUCNestLevel
  • ereport

関連データ構造

  • UserContext: 切り替え前のユーザーID、セキュリティコンテキスト、およびGUCのネストレベルを保持する構造体。

失敗モードと境界条件

  • **権限不足**: 現在のユーザーが対象ロールへ切り替える権限を持たない場合、ereport(ERROR, ...) により処理が中断されます。
  • **GUCネストレベル**: save_nestlevel が -1 の場合、GUCのロールバック処理は不要とみなされます。

保守担当者向けメモ

  • 本関数は RestoreUserContext と対で使用されることを前提としています。呼び出し忘れはセッション状態の不整合を招くため注意してください。
  • SECURITY_RESTRICTED_OPERATION が適用されるケースでは、ターゲットユーザーによる悪意のあるGUC変更を防ぐための保護が有効になります。
