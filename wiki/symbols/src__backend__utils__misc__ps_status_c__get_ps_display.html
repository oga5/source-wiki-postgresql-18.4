get_ps_display

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/ps_status.c:531

目的

現在プロセスが保持している表示用文字列（プロセス名や状態）のうち、固定部分を除いた可変部分のポインタと長さを取得します。

シグネチャまたは形状

const char *get_ps_display(int *displen);

入力と出力

  • **入力**: displen (intへのポインタ) - 可変部分の文字列長を格納する領域。
  • **出力**: 可変部分の文字列を指す const char *。

前提条件と不変条件

  • PS_USE_CLOBBER_ARGV が定義されている環境では、ps_buffer が初期化されていない場合、displen に 0 を設定し空文字列を返します。
  • PS_USE_NONE が定義されている環境では、常に displen に 0 を設定し空文字列を返します。

アルゴリズムまたは振る舞い

1. PS_USE_CLOBBER_ARGV 環境下で ps_buffer が未初期化であれば、即座に空文字列を返します。 2. PS_USE_NONE 以外の環境では、現在のバッファ長 (ps_buffer_cur_len) から固定部分のサイズ (ps_buffer_fixed_size) を差し引いた値を displen に代入します。 3. バッファの先頭から固定部分のサイズ分オフセットした位置にある文字列ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/error/csvlog.c:139
  • src/backend/utils/error/elog.c:3061
  • src/backend/utils/error/jsonlog.c:185
  • **呼び出し先**:
  • 本関数内での直接的な関数呼び出しはありません。

関連データ構造

  • ps_buffer: プロセス表示用文字列を保持する内部バッファ。
  • ps_buffer_cur_len: 現在のバッファ内の総データ長。
  • ps_buffer_fixed_size: プロセス名の固定部分（プログラム名等）のサイズ。

失敗モードと境界条件

  • ps_buffer が NULL の場合、PS_USE_CLOBBER_ARGV 定義環境下では安全に空文字列を返します。
  • PS_USE_NONE 定義環境下では、常に空文字列を返します。

保守担当者向けメモ

  • 本関数はログ出力処理（CSV, JSON, elog）において、現在のプロセス状態をログに記録するために利用されています。
  • OS固有のプロセス名更新メカニズム（PS_USE_... マクロ群）に強く依存しているため、プラットフォーム固有の挙動を調査する際は ps_status.c 内の定義を確認してください。
