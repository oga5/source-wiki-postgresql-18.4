ControlData

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.h:229

目的

PostgreSQL の pg_control ファイルの内容を保持するための構造体です。バックエンドの定義に依存せず、異なるメジャーバージョン間での構造体定義の不一致を吸収し、アップグレード時の互換性検証を可能にします。

シグネチャまたは形状

typedef struct {
    uint32      ctrl_ver;
    uint32      cat_ver;
    char        nextxlogfile[25];
    uint32      chkpnt_nxtxid;
    uint32      chkpnt_nxtepoch;
    uint32      chkpnt_nxtoid;
    uint32      chkpnt_nxtmulti;
    uint32      chkpnt_nxtmxoff;
    uint32      chkpnt_oldstMulti;
    uint32      chkpnt_oldstxid;
    uint32      align;
    uint32      blocksz;
    uint32      largesz;
    uint32      walsz;
    uint32      walseg;
    uint32      ident;
    uint32      index;
    uint32      toast;
    uint32      large_object;
    bool        date_is_int;
    bool        float8_pass_by_value;
    uint32      data_checksum_version;
    bool        default_char_signedness;
} ControlData;

入力と出力

  • 入力: pg_upgrade 実行時に旧クラスタおよび新クラスタから読み込まれる pg_control ファイルのデータ。
  • 出力: check_control_data 関数等を通じて、アップグレードの可否判断やクラスタ状態の検証に使用されます。

前提条件と不変条件

  • pg_upgrade ツール内で独自に定義されており、PostgreSQL バックエンドの内部構造体とは分離されています。
  • バージョン間の差異を吸収するため、特定のカタログバージョン（cat_ver）や制御バージョン（ctrl_ver）がアップグレードの可否を判断する基準となります。

アルゴリズムまたは振る舞い

本構造体は、新旧クラスタのメタデータを保持するコンテナとして機能します。pg_upgrade の各フェーズにおいて、この構造体に格納された値が比較・検証され、互換性が保たれているかを確認します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/bin/pg_upgrade/controldata.c: check_control_data 関数にて旧クラスタと新クラスタの ControlData を比較。
  • src/bin/pg_upgrade/pg_upgrade.h: ClusterInfo 構造体の一部として controldata メンバを定義。
  • 呼び出し先: 構造体定義のため、直接的な関数呼び出しはありません。

関連データ構造

  • ClusterInfo: アップグレード対象となるクラスタの全情報を格納する構造体であり、本構造体をメンバとして保持します。

失敗モードと境界条件

  • 異なるメジャーバージョン間での pg_control のフォーマット変更により、本構造体で定義されたフィールドと実際のファイル内容が一致しない場合、アップグレード処理が中断される可能性があります。

保守担当者向けメモ

  • 本構造体はバックエンドの定義に依存しないため、PostgreSQL のバージョンアップに伴い、pg_control のフォーマットが変更された場合は、本構造体の定義も適切に更新する必要があります。
  • WIN32 環境下での動作については、プラットフォーム固有の制約を考慮してください。
