executeQuery

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/connectdb.c:277

目的

データベースに対してクエリを実行し、結果が正常（PGRES_TUPLES_OK）でない場合にエラーログを出力してプログラムを終了させるラッパー関数です。

シグネチャまたは形状

PGresult *executeQuery(PGconn *conn, const char *query)

入力と出力

  • **入力**:
  • conn: データベース接続を保持する PGconn オブジェクト。
  • query: 実行する SQL クエリ文字列。
  • **出力**:
  • 成功時: クエリ結果を保持する PGresult オブジェクト。
  • 失敗時: 戻り値は返されず、プログラムが終了します。

前提条件と不変条件

  • conn は有効なデータベース接続である必要があります。
  • クエリ実行の結果が PGRES_TUPLES_OK 以外である場合、本関数は PQfinish(conn) を呼び出して接続を閉じ、exit_nicely(1) によりプロセスを終了させます。

アルゴリズムまたは振る舞い

1. 実行するクエリの内容を pg_log_info でログ出力します。 2. PQexec を呼び出し、データベースに対してクエリを実行します。 3. 結果オブジェクトが空であるか、ステータスが PGRES_TUPLES_OK でない場合：

  • PQerrorMessage を用いてエラー内容をログ出力します。
  • 実行しようとしたクエリ文字列を詳細ログとして出力します。
  • PQfinish で接続を解放します。
  • exit_nicely(1) を呼び出し、プログラムを異常終了させます。

    • 4. 正常終了時は、取得した PGresult ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/connectdb.c
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/pg_dump/pg_dumpall.c
  • **呼び出し先**:
  • PQexec (libpq)
  • pg_log_info, pg_log_error, pg_log_error_detail (共通ログ出力)
  • PQfinish (libpq)
  • exit_nicely (pg_dump ユーティリティ)

関連データ構造

  • PGconn: libpq のデータベース接続ハンドル。
  • PGresult: libpq のクエリ結果保持構造体。

失敗モードと境界条件

  • クエリが構文エラーや権限不足などで失敗した場合、本関数は即座にプログラムを終了させるため、呼び出し側でエラーハンドリングを行うことはできません。
  • PQexec が NULL を返した場合も同様に終了処理が行われます。

保守担当者向けメモ

  • 本関数は pg_dump 等のツールにおいて、致命的なエラーを伴うクエリ実行を簡略化するために設計されています。
  • 戻り値の PGresult は、呼び出し側で不要になった際に PQclear を使用して解放する必要があります。
