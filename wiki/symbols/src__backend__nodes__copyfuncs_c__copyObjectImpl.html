copyObjectImpl

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/copyfuncs.c:176

目的

PostgreSQLのノード構造体（クエリツリーやプランツリーなど）をメモリ上で再帰的に複製（ディープコピー）する中心的なディスパッチャーです。

シグネチャまたは形状

void *copyObjectImpl(const void *from);

入力と出力

  • **入力**: from (コピー対象のノード構造体へのポインタ)
  • **出力**: 複製されたノード構造体へのポインタ。from が NULL の場合は NULL を返します。

前提条件と不変条件

  • check_stack_depth() により、再帰呼び出しによるスタックオーバーフローが防止されます。
  • 循環参照を持つデータ構造には使用できません。

アルゴリズムまたは振る舞い

1. 入力が NULL であれば即座に NULL を返します。 2. スタック深度をチェックします。 3. nodeTag(from) を判定し、以下のいずれかの処理を行います。

  • copyfuncs.switch.c に定義された各ノード型専用のコピー関数へディスパッチ。
  • T_List: list_copy_deep() を使用して再帰的にコピー。
  • T_IntList, T_OidList, T_XidList: list_copy() を使用して浅いコピーを実行。

    • 4. 未知のノード型が渡された場合は elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/copyfuncs.c 内の各コピー関数（COPY_NODE_FIELD マクロ経由）
  • src/backend/nodes/list.c:1652
  • **呼び出し先**:
  • check_stack_depth()
  • nodeTag()
  • list_copy_deep()
  • list_copy()
  • copyfuncs.switch.c 内の各 _copy 関数

関連データ構造

  • Node (ノードタグを持つ構造体全般)
  • List, IntList, OidList, XidList

失敗モードと境界条件

  • **スタックオーバーフロー**: 非常に複雑なノードツリーを処理する際に発生する可能性があります。
  • **未知のノード型**: nodeTag が認識できない値の場合、ERROR で終了します。
  • **NULL入力**: 安全に処理され NULL が返されます。

保守担当者向けメモ

  • 本関数は copyObject() マクロを通じて呼び出されることが想定されています。
  • 新しいノード型を追加した場合は、copyfuncs.switch.c 側での対応が必要です。
  • COPY_NODE_FIELD マクロは本関数を再帰的に呼び出すため、ノード定義の変更時には再帰構造に注意してください。
