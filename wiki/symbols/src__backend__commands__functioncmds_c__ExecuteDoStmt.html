ExecuteDoStmt

定義

  • 種別: function
  • 定義箇所: src/backend/commands/functioncmds.c:2083

目的

SQLの DO 文を実行し、指定された言語のインラインハンドラを呼び出すことで、匿名コードブロックを実行する。

シグネチャまたは形状

void ExecuteDoStmt(ParseState *pstate, DoStmt *stmt, bool atomic)

入力と出力

  • **入力**:
  • pstate: パーサー状態。
  • stmt: DO 文の構文木（args リストに as や language オプションを含む）。
  • atomic: トランザクションの原子性フラグ。
  • **出力**: なし（副作用としてコードブロックを実行）。

前提条件と不変条件

  • stmt->args に as オプション（実行するコード）が必須。
  • 指定された言語がシステムカタログ pg_language に存在すること。
  • 信頼された言語の場合は USAGE 権限が必要、信頼されていない言語の場合はスーパーユーザー権限が必要。
  • 指定された言語がインライン実行（laninline）をサポートしていること。

アルゴリズムまたは振る舞い

1. stmt->args を走査し、as（ソースコード）と language（デフォルトは plpgsql）を抽出する。 2. SearchSysCache1 を使用して pg_language から言語情報を取得する。 3. 言語の信頼性（lanpltrusted）に基づき、object_aclcheck または superuser() で権限を検証する。 4. 言語の laninline フィールドからハンドラ関数の OID を取得する。 5. InlineCodeBlock ノードを作成し、ソースコード、言語 OID、信頼性フラグ、原子性フラグを格納する。 6. OidFunctionCall1 を介して、取得したハンドラ関数を InlineCodeBlock を引数として実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c（DoStmt 実行時）
  • **呼び出し先**:
  • OidFunctionCall1（言語ハンドラの起動）
  • SearchSysCache1 / ReleaseSysCache（pg_language 検索）
  • object_aclcheck（権限確認）

関連データ構造

  • DoStmt: DO 文の構文木。
  • InlineCodeBlock: 実行時に言語ハンドラへ渡されるデータ構造。
  • Form_pg_language: pg_language カタログの行構造。

失敗モードと境界条件

  • as オプションが指定されていない場合: ERRCODE_SYNTAX_ERROR でエラー。
  • 言語が存在しない場合: ERRCODE_UNDEFINED_OBJECT でエラー。
  • 権限不足の場合: aclcheck_error を呼び出しエラー。
  • 言語がインライン実行をサポートしていない場合: ERRCODE_FEATURE_NOT_SUPPORTED でエラー。

保守担当者向けメモ

  • 本関数は DO 文の実行におけるゲートウェイです。言語固有の実行ロジックは laninline で指定されたハンドラ関数側に委譲されます。
  • atomic 引数は、プロシージャ実行時と同様にトランザクション制御のコンテキストをハンドラに伝えるために重要です。
