AlignedAllocFree

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/alignedalloc.c:28

目的

palloc_aligned() によって確保されたアライメント調整済みメモリブロックを解放し、関連するメタデータを適切に処理する。

シグネチャまたは形状

void AlignedAllocFree(void *pointer)

入力と出力

  • **入力**: pointer (アライメント調整されたメモリ領域を指すポインタ)
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • pointer は palloc_aligned() を通じて確保された有効なメモリ領域を指していること。
  • MemoryChunk ヘッダーがポインタの直前に存在すること。
  • MemoryChunkIsExternal(chunk) が偽であること（外部メモリチャンクではないこと）。

アルゴリズムまたは振る舞い

1. PointerGetMemoryChunk() を使用して、渡されたポインタから MemoryChunk ヘッダーを取得する。 2. VALGRIND_MAKE_MEM_DEFINED を呼び出し、チャンクのメタデータ領域をアクセス可能にする。 3. MemoryChunkGetBlock() を呼び出し、アライメント調整前の「真の」メモリブロック（unaligned）を取得する。 4. MEMORY_CONTEXT_CHECKING が有効な場合、sentinel_ok() を用いてメモリ破壊（バッファオーバーラン）の有無を検証し、異常があれば警告を出力する。 5. 取得した unaligned ポインタを pfree() に渡し、メモリの解放を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c: MCTX_ALIGNED_REDIRECT_ID の free_p 関数ポインタとして登録されている。
  • **呼び出し先**:
  • PointerGetMemoryChunk (マクロ)
  • MemoryChunkGetBlock (マクロ)
  • VALGRIND_MAKE_MEM_DEFINED (マクロ)
  • sentinel_ok (マクロ、MEMORY_CONTEXT_CHECKING 有効時)
  • GetMemoryChunkContext (マクロ、MEMORY_CONTEXT_CHECKING 有効時)
  • elog (マクロ、MEMORY_CONTEXT_CHECKING 有効時)
  • pfree (関数)

関連データ構造

  • MemoryChunk: メモリブロックのメタデータを保持する構造体。

失敗モードと境界条件

  • MEMORY_CONTEXT_CHECKING が有効な環境において、メモリチャンクの末尾を越えた書き込みが検出された場合、elog により警告がログ出力される。

保守担当者向けメモ

  • 本関数は palloc_aligned で確保されたメモリのライフサイクルを終了させるためのブリッジ関数です。
  • MemoryChunk ヘッダーの配置構造に依存しているため、utils/memutils_memorychunk.h の変更時には本関数のロジックも再確認してください。
