brin_evacuate_page

定義

  • src/backend/access/brin/brin_pageops.c:563

目的

BRINインデックスページを再利用または削除する準備として、当該ページ内に存在するすべてのタプルを他の適切なページへ移動（退避）させる。

シグネチャまたは形状

void brin_evacuate_page(Relation idxRel, BlockNumber pagesPerRange, BrinRevmap *revmap, Buffer buf)

入力と出力

  • idxRel: 対象のインデックスリレーション。
  • pagesPerRange: 1つのBRIN範囲に含まれるブロック数。
  • revmap: インデックスのRevmap構造体。
  • buf: 退避対象となるページのバッファ。

前提条件と不変条件

  • 呼び出し時点で、対象ページには BRIN_EVACUATE_PAGE フラグが設定されている必要がある（Assertにより検証）。
  • 処理中、ページは共有ロック状態で保持される。

アルゴリズムまたは振る舞い

1. 対象ページ内の各オフセットを走査し、使用中のタプルを特定する。 2. 各タプルを brin_copy_tuple でコピーし、一時的に保持する。 3. ページに対するロックを一時的に解除し、brin_doupdate を呼び出してタプルを別のページへ移動させる。 4. brin_doupdate が失敗した場合は、該当オフセットを再試行する。 5. 移動完了後、ページに対する共有ロックを再取得し、ページが依然として退避対象（非レギュラーページ）であるかを確認する。 6. 全タプルの処理終了後、バッファを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin_revmap.c:592
  • **呼び出し先**:
  • brin_copy_tuple
  • brin_doupdate
  • LockBuffer / UnlockReleaseBuffer
  • PageGetMaxOffsetNumber / PageGetItemId / PageGetItem

関連データ構造

  • BrinTuple: インデックスに格納されるタプル構造。
  • BrinRevmap: ヒープブロックとインデックスの対応を管理する構造体。
  • Buffer: PostgreSQLのバッファ管理構造。

失敗モードと境界条件

  • brin_doupdate がタプルの移動に失敗した場合、ループ内でオフセットをデクリメントし、再試行を行う。
  • 処理中にページの状態が変化し、BRIN_IS_REGULAR_PAGE が真となった場合（退避の必要がなくなった場合）、ループを中断する。

保守担当者向けメモ

  • 本関数はクリティカルセクション外で実行されるが、内部で brin_doupdate を呼び出すことで整合性を維持している。
  • CHECK_FOR_INTERRUPTS() がループ内に配置されており、長時間実行時の割り込みに対応している。
