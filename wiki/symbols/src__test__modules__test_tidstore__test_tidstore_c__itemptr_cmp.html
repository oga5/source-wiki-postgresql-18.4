itemptr_cmp

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_tidstore/test_tidstore.c:51

目的

ItemPointer（TID）の比較を行い、qsort 等のソート処理や、期待値との整合性検証における順序比較を可能にする。

シグネチャまたは形状

static int itemptr_cmp(const void *left, const void *right)

入力と出力

  • **入力**: 比較対象となる2つの ItemPointer へのポインタ（const void * 型）。
  • **出力**: 比較結果を示す整数。
  • left < right の場合: -1
  • left > right の場合: 1
  • left == right の場合: 0

前提条件と不変条件

  • 入力ポインタは、有効な ItemPointerData 構造体を指している必要がある。
  • ブロック番号およびオフセット番号の比較において、PostgreSQL の標準的な ItemPointer 操作関数（ItemPointerGetBlockNumber, ItemPointerGetOffsetNumber）が利用可能であること。

アルゴリズムまたは振る舞い

1. ItemPointerGetBlockNumber を使用して、両者のブロック番号を取得し比較する。 2. ブロック番号が異なる場合、その大小関係に基づき -1 または 1 を即座に返す。 3. ブロック番号が等しい場合、ItemPointerGetOffsetNumber を使用してオフセット番号を取得し比較する。 4. オフセット番号の大小関係に基づき -1 または 1 を返す。 5. 両者が完全に一致する場合、0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_tidstore/test_tidstore.c 内の check_set_block_offsets 関数。
  • qsort の比較関数として、items.insert_tids および items.lookup_tids のソートに使用。
  • items.insert_tids と items.iter_tids、および items.insert_tids と items.lookup_tids の比較に使用。
  • **呼び出し先**:
  • ItemPointerGetBlockNumber
  • ItemPointerGetOffsetNumber

関連データ構造

  • ItemPointerData (およびそのポインタ型 ItemPointer)

失敗モードと境界条件

  • 入力ポインタが NULL の場合、関数内部で ItemPointer へのキャストが行われるため、未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は qsort の比較関数シグネチャに準拠している。
  • ItemPointer の比較順序は、まずブロック番号、次にオフセット番号という PostgreSQL の標準的な順序に従っている。
