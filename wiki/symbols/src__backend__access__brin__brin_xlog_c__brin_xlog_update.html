brin_xlog_update

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_xlog.c:134

目的

BRINインデックスにおける更新操作のWALリプレイを実行し、古いタプルの削除と新しいタプルの挿入およびREVMAPの更新を統合的に処理します。

シグネチャまたは形状

static void brin_xlog_update(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）。
  • **出力**: なし（副作用としてバッファの更新およびディスクへの反映が行われます）。

前提条件と不変条件

  • record は xl_brin_update 構造体を含む有効なWALレコードである必要があります。
  • 呼び出し元は、WALレコードの解析および適切なディスパッチを完了している必要があります。

アルゴリズムまたは振る舞い

1. XLogRecGetData を介してWALレコードから xl_brin_update データを取得します。 2. XLogReadBufferForRedo を使用して、対象のインデックスページ（ブロック番号2）を読み込みます。 3. BLK_NEEDS_REDO が返された場合、以下の処理を行います。

  • PageIndexTupleDeleteNoCompact を呼び出し、xlrec->oldOffnum で指定された古いタプルをページから削除します。
  • PageSetLSN でページLSNを更新し、MarkBufferDirty でバッファをダーティ状態にします。

    • 4. brin_xlog_insert_update を呼び出し、新しいタプルの挿入とREVMAPの更新を実行します。 5. 最後に、読み込んだバッファが有効であれば UnlockReleaseBuffer を呼び出してロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: brin_redo（src/backend/access/brin/brin_xlog.c:322 での参照を確認）。
  • **呼び出し先**: XLogRecGetData, XLogReadBufferForRedo, BufferGetPage, PageIndexTupleDeleteNoCompact, PageSetLSN, MarkBufferDirty, brin_xlog_insert_update, BufferIsValid, UnlockReleaseBuffer。

関連データ構造

  • xl_brin_update: 更新操作に必要な情報（古いタプルのオフセット番号および挿入用データ）を保持する構造体。
  • XLogReaderState: WALレコードの読み取り状態を管理する構造体。

失敗モードと境界条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO 以外を返した場合、古いタプルの削除処理はスキップされますが、brin_xlog_insert_update は引き続き実行されます。
  • バッファが無効な場合、ロック解放処理はスキップされます。

保守担当者向けメモ

  • 本関数は、インデックスページ上の物理的な削除と、REVMAPを含む論理的な挿入を組み合わせた複合的な操作です。
  • brin_xlog_insert_update が内部でREVMAPの更新を管理しているため、本関数側ではインデックスページ自体の操作に集中しています。
