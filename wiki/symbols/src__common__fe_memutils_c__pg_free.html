pg_free

定義

  • 種別: function
  • 定義箇所: src/common/fe_memutils.c:110

目的

PostgreSQLのフロントエンドプログラムにおいて、動的に確保されたメモリ領域を解放するための標準的なラッパー関数です。

シグネチャまたは形状

void pg_free(void *ptr);

入力と出力

  • **入力**: ptr (解放対象のメモリ領域を指すポインタ。NULL も許容される)
  • **出力**: なし

前提条件と不変条件

  • ptr は、malloc またはそれと同等のメモリ確保関数によって確保された有効なポインタ、あるいは NULL である必要があります。

アルゴリズムまたは振る舞い

標準ライブラリの free(ptr) を直接呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/initdb/initdb.c
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_basebackup/pg_createsubscriber.c

    • (上記はツールによる参照のヒントであり、網羅的ではありません)

  • **呼び出し先**:
  • free (標準Cライブラリ)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • ptr が NULL の場合、標準の free の仕様に従い、何も行われません。
  • 無効なポインタや既に解放済みのポインタを渡した場合の動作は、標準の free と同様に未定義です。

保守担当者向けメモ

  • 本関数はバックエンドの pfree との互換性を維持するために提供されています。
  • FRONTEND マクロが定義されている環境でのみコンパイルされます。
