IsTransactionStmtList

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/postgres.c:2889

目的

与えられた文リストが、単一のトランザクション制御コマンド（TransactionStmt）で構成されているかどうかを判定する。

シグネチャまたは形状

static bool IsTransactionStmtList(List *pstmts)

入力と出力

  • **入力**: pstmts (実行対象の PlannedStmt を含むリスト)
  • **出力**: bool 型。リストが単一のユーティリティコマンドであり、かつそれが TransactionStmt である場合に true を返す。それ以外は false。

前提条件と不変条件

  • pstmts は PlannedStmt ノードのリストである必要がある。
  • linitial_node(PlannedStmt, pstmts) を呼び出すため、リストが空でないことが前提となる（空リストの場合の挙動は未定義）。

アルゴリズムまたは振る舞い

1. リストの長さが 1 であるかを確認する。 2. リスト内の唯一の要素を PlannedStmt として取得する。 3. コマンドタイプが CMD_UTILITY であり、かつそのユーティリティ文が TransactionStmt 型であるかを確認する。 4. 上記条件をすべて満たす場合に true を返し、それ以外は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/postgres.c 内のクエリ実行処理（2152 行目付近）にて、トランザクションコマンドであるかを判定するために使用されている。
  • **呼び出し先**: list_length, linitial_node, IsA マクロを呼び出す。

関連データ構造

  • List: PostgreSQL の汎用リスト構造。
  • PlannedStmt: クエリ計画を表すノード。
  • TransactionStmt: トランザクション制御（BEGIN, COMMIT, ROLLBACK 等）を表すユーティリティ文ノード。

失敗モードと境界条件

  • リストが空の場合、linitial_node マクロの内部実装に依存するが、通常はアサーション失敗やクラッシュの可能性がある。
  • リストの長さが 2 以上の場合、常に false を返す。

保守担当者向けメモ

  • 本関数は postgres.c 内の静的関数であり、主にトランザクション状態の遷移判断に使用される。
  • TransactionStmt 以外のユーティリティコマンドや、複数の文を含むリストは対象外となる点に注意すること。
