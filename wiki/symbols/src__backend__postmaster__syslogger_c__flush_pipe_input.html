flush_pipe_input

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/syslogger.c:1041

目的

Syslogger プロセス終了時やエラー発生時に、パイプ内に残存しているログデータおよびバッファリング中の不完全なプロトコルメッセージを強制的にフラッシュ（書き出し）する。

シグネチャまたは形状

static void flush_pipe_input(char *logbuffer, int *bytes_in_logbuffer)

入力と出力

  • logbuffer: パイプから読み込まれた未処理の生データバッファ。
  • bytes_in_logbuffer: logbuffer 内の有効なデータ長（入力および出力）。処理完了後に 0 にリセットされる。

前提条件と不変条件

  • buffer_lists に格納された save_buffer 構造体は、処理後に pid が 0 にリセットされ、関連するメモリが解放される。
  • 処理終了後、*bytes_in_logbuffer は必ず 0 となる。

アルゴリズムまたは振る舞い

1. **不完全メッセージのフラッシュ**: buffer_lists を走査し、pid が有効（非ゼロ）な save_buffer をすべて検索する。見つかった場合、その内容を write_syslogger_file を介して LOG_DESTINATION_STDERR へ書き出し、メモリを解放する。 2. **残存パイプデータのフラッシュ**: logbuffer 内にデータが残っている場合、プロトコルヘッダーの有無を問わず、そのまま write_syslogger_file を介して書き出す。 3. **状態のリセット**: *bytes_in_logbuffer を 0 に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/syslogger.c:546
  • src/backend/postmaster/syslogger.c:1200
  • **呼び出し先**:
  • write_syslogger_file
  • pfree

関連データ構造

  • save_buffer: プロトコルメッセージを一時保持する構造体。
  • buffer_lists: save_buffer を管理するリスト配列。
  • StringInfo: save_buffer 内のデータ保持に使用される文字列バッファ。

失敗モードと境界条件

  • bytes_in_logbuffer が 0 の場合、パイプデータのフラッシュ処理はスキップされる。
  • buffer_lists が空の場合、不完全メッセージのフラッシュ処理はスキップされる。

保守担当者向けメモ

  • 本関数は、ログの取りこぼしを防ぐための最終的なクリーンアップ処理である。
  • パイプデータのフラッシュ時にプロトコルヘッダーの解析を行わないため、出力先ファイルには不完全なヘッダーが含まれる可能性がある。
