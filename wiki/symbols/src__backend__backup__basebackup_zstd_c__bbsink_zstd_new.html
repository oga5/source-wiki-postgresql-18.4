bbsink_zstd_new

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_zstd.c:60

目的

Zstandard (zstd) 圧縮を適用するための新しい bbsink インスタンスを生成し、初期化するコンストラクタ関数です。

シグネチャまたは形状

bbsink *bbsink_zstd_new(bbsink *next, pg_compress_specification *compress)

入力と出力

  • **入力**:
  • next: 圧縮後のデータを受け取る後続の bbsink オブジェクト。
  • compress: 圧縮設定（レベルやスレッド数など）を保持する pg_compress_specification 構造体。
  • **出力**:
  • 成功時: 初期化された bbsink 構造体へのポインタ。
  • 失敗時: USE_ZSTD が未定義の場合、エラーを報告して終了します。

前提条件と不変条件

  • next 引数は NULL であってはなりません（Assert による検証）。
  • ビルド時に USE_ZSTD マクロが定義されている必要があります。

アルゴリズムまたは振る舞い

1. USE_ZSTD が未定義の場合、ERRCODE_FEATURE_NOT_SUPPORTED エラーを発生させます。 2. USE_ZSTD が定義されている場合:

  • bbsink_zstd 構造体分のメモリを palloc0 で確保します。
  • 確保した領域の bbs_ops に bbsink_zstd_ops を割り当てます。
  • bbs_next に引数 next を設定し、圧縮設定を保持します。
  • 初期化済みの bbsink 構造体へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup.c:1048 にて、バックアップオプションに基づき呼び出されます。
  • **呼び出し先**: palloc0（PostgreSQL メモリ管理関数）。

関連データ構造

  • bbsink: 継承元の基底構造体。
  • bbsink_zstd: 本関数で生成される、Zstd 圧縮状態を保持する構造体。
  • bbsink_zstd_ops: bbsink インターフェースの関数ポインタテーブル。
  • pg_compress_specification: 圧縮パラメータを保持する構造体。

失敗モードと境界条件

  • USE_ZSTD が定義されていない環境で呼び出された場合、ereport(ERROR, ...) により処理が中断されます。
  • next が NULL の場合、アサーション違反によりプロセスが停止します。

保守担当者向けメモ

  • 本関数は bbsink パイプラインの構築時に呼び出されます。
  • 圧縮コンテキスト（ZSTD_CCtx）の実際の初期化は、本関数ではなく、パイプライン開始時に呼ばれる bbsink_zstd_begin_backup 等で行われます。
