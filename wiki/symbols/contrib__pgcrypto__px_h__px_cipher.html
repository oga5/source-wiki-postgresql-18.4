px_cipher

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/px.h:148

目的

暗号化アルゴリズムを抽象化し、共通のインターフェースを通じて暗号操作（初期化、暗号化、復号）を実行するための構造体です。

シグネチャまたは形状

struct px_cipher {
    unsigned (*block_size) (PX_Cipher *c);
    unsigned (*key_size) (PX_Cipher *c);
    unsigned (*iv_size) (PX_Cipher *c);
    int      (*init) (PX_Cipher *c, const uint8 *key, unsigned klen, const uint8 *iv);
    int      (*encrypt) (PX_Cipher *c, int padding, const uint8 *data, unsigned dlen, uint8 *res, unsigned *rlen);
    int      (*decrypt) (PX_Cipher *c, int padding, const uint8 *data, unsigned dlen, uint8 *res, unsigned *rlen);
    void     (*free) (PX_Cipher *c);
    void    *ptr;
    int      pstat;
};

入力と出力

  • **入力**: 各関数ポインタは PX_Cipher（px_cipher の別名）へのポインタを第一引数に取ります。init は鍵と初期化ベクトル（IV）を、encrypt/decrypt はデータ本体とパディングフラグを受け取ります。
  • **出力**: block_size, key_size, iv_size はサイズを返します。init, encrypt, decrypt は成功時に 0 を返し、エラー時には PXE_* 定数に基づいたエラーコードを返します。

前提条件と不変条件

  • ptr メンバは、具体的な暗号実装（OpenSSL等）のコンテキストを保持するために使用されます。
  • init を呼び出した後に encrypt または decrypt を実行する必要があります。
  • 処理終了後は free を呼び出し、リソースを解放する必要があります。

アルゴリズムまたは振る舞い

  • 関数ポインタによるポリモーフィズムを提供し、具体的な暗号アルゴリズムの実装詳細を隠蔽します。
  • encrypt および decrypt は、padding 引数を用いてパディングの有無を制御します。
  • pstat は特定のバックエンド（mcrypt）の状態管理に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/px.h 内で typedef struct px_cipher PX_Cipher; として定義されており、上位レイヤーがこの型を通じて操作を行います。
  • 呼び出し先: 具体的な実装（openssl.c 等）が各関数ポインタの実体を提供します。

関連データ構造

  • PX_Cipher: px_cipher 構造体の別名。

失敗モードと境界条件

  • 各関数ポインタは、エラー発生時に px.h で定義された PXE_* エラーコードを返します。
  • ptr が正しく初期化されていない場合や、メモリ不足時には予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は関数ポインタの集合体であるため、デバッグ時には ptr が指す具体的な実装コンテキストを確認してください。
  • 新しい暗号アルゴリズムを追加する場合は、この構造体の全関数ポインタを実装したバックエンドを作成する必要があります。
