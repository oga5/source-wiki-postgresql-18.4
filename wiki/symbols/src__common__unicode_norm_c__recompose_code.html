recompose_code

定義

  • 種別: function
  • 定義箇所: src/common/unicode_norm.c:218

目的

分解された2つのUnicodeコードポイントを結合し、可能な場合に合成文字（NFC/NFKC形式）を生成します。

シグネチャまたは形状

static bool recompose_code(uint32 start, uint32 code, uint32 *result)

入力と出力

  • **入力**:
  • start: 合成対象の先行コードポイント。
  • code: 合成対象の次続コードポイント。
  • **出力**:
  • result: 合成に成功した場合、合成後のコードポイントが格納される。
  • 戻り値: 合成が行われた場合は true、それ以外は false。

前提条件と不変条件

  • start は既に部分的に正規化（合成）済みであると仮定されます。
  • ハングル文字の合成は、Unicode仕様に基づきアルゴリズム的に処理されます。
  • ハングル以外の文字は、静的データテーブル（UnicodeDecompMain 等）を用いた逆引きによって合成されます。

アルゴリズムまたは振る舞い

1. **ハングル合成**:

  • start が L（初声）、code が V（中声）の場合、LV音節を生成します。
  • start が LV音節、code が T（終声）の場合、LVT音節を生成します。

    • 2. **その他文字の合成**:

  • ハングル以外の場合、分解テーブルの逆引きを行います。
  • **バックエンド環境**: 完全ハッシュ関数を用いて高速に検索します。
  • **フロントエンド環境**: テーブルを線形走査し、分解サイズが2かつ合成可能なエントリを検索します。

    • 3. **結果**: 合成可能な組み合わせが見つかった場合、result に値を書き込み true を返します。

呼び出し元と呼び出し先

  • **呼び出し先**:

関連データ構造

  • pg_unicode_decomposition: 分解情報を保持する構造体。
  • UnicodeDecompMain: 分解情報のメインテーブル。
  • UnicodeRecompInfo: バックエンド用の再合成情報（ハッシュ関数等を含む）。

失敗モードと境界条件

  • ハングル以外の文字で、テーブル内に対応する合成ペアが存在しない場合は false を返します。
  • バックエンドにおいて、ハッシュ値が範囲外の場合は一致なしとみなされます。
  • 分解サイズが2でない、または合成禁止フラグが立っているエントリは無視されます。

保守担当者向けメモ

  • FRONTEND マクロにより、検索アルゴリズム（ハッシュ vs 線形探索）が切り替わるため、修正時は両方のパスを確認してください。
  • ハングル処理は計算式に依存しているため、Unicode仕様の変更時には定数（LBASE, VCOUNT 等）の整合性に注意してください。
