pg_get_line

定義

  • 種別: function
  • 定義箇所: src/common/pg_get_line.c:58

目的

入力ストリームから1行を読み込み、動的に確保したメモリ領域に格納して返す。fgets() の拡張版として、バッファサイズを気にせず任意の長さの行を読み取るために使用される。

シグネチャまたは形状

char *pg_get_line(FILE *stream, PromptInterruptContext *prompt_ctx)

入力と出力

  • **入力**:
  • stream: 読み込み対象の FILE ポインタ。
  • prompt_ctx: 割り込み制御用のコンテキスト（NULL 指定可能）。
  • **出力**:
  • 読み取った文字列（末尾に改行文字を含む）。
  • 読み込み失敗時または EOF 到達時は NULL を返す。

前提条件と不変条件

  • 内部で palloc を使用してメモリを確保するため、呼び出し元は返された文字列に対して pfree を実行する責任がある。
  • 改行文字は削除されず、そのまま保持される。

アルゴリズムまたは振る舞い

1. StringInfoData を初期化し、動的バッファを準備する。 2. pg_get_line_append を呼び出し、ストリームからバッファへデータを読み込む。 3. 読み込みに失敗した場合、確保したバッファを解放し、errno を保存した上で NULL を返す。 4. 成功した場合、バッファ内の文字列データ（buf.data）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/initdb/initdb.c:1694
  • src/common/exec.c:369

  • **呼び出し先**:

関連データ構造

  • StringInfoData: 動的文字列バッファを管理するための構造体。
  • PromptInterruptContext: 割り込みシグナルを処理するためのコンテキスト構造体。

失敗モードと境界条件

  • **メモリ不足**: 内部の enlargeStringInfo がメモリ不足を検知した場合、エラーが発生する。
  • **I/O エラー**: fgets がエラーを返した場合、バッファは解放され NULL が返される。
  • **EOF**: 入力が空の状態で EOF に到達した場合、NULL が返される。

保守担当者向けメモ

  • 本関数は palloc を使用するため、メモリ管理のライフサイクルに注意すること。
  • 大量の行を処理する場合、メモリ確保のオーバーヘッドを避けるため、バッファを再利用する pg_get_line_buf の利用を検討すること。
  • 改行文字の除去が必要な場合は、別途 pg_strip_crlf() 等の併用が必要である。
