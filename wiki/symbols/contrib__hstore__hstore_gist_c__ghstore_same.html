ghstore_same

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_gist.c:218

目的

2つの GISTTYPE（hstore の GiST インデックス用シグネチャ）が同一であるかを判定します。

シグネチャまたは形状

Datum ghstore_same(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): 比較対象の GISTTYPE *a
  • PG_GETARG_POINTER(1): 比較対象の GISTTYPE *b
  • PG_GETARG_POINTER(2): 結果を格納する bool *result
  • **出力**:
  • PG_RETURN_POINTER(result): 比較結果（true または false）を指すポインタを返します。

前提条件と不変条件

  • a および b は有効な GISTTYPE 構造体である必要があります。
  • siglen は GET_SIGLEN() マクロにより現在のインデックス設定から取得されます。

アルゴリズムまたは振る舞い

1. 両方のシグネチャが「すべて真（ALLTRUE）」である場合、result を true に設定します。 2. 片方のみが「すべて真」である場合、result を false に設定します。 3. いずれでもない場合、GETSIGN マクロで取得したビットベクトルを siglen バイト分ループし、各バイトを比較します。 4. 1バイトでも不一致があれば result を false にし、ループを終了します。 5. 全バイトが一致すれば result は true のまま維持されます。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックスインターフェースから呼び出されます（直接的な呼び出し元コードの詳細はソース内に明示されていません）。
  • **呼び出し先**:
  • GET_SIGLEN()
  • ISALLTRUE()
  • GETSIGN()
  • LOOPBYTE()

関連データ構造

  • GISTTYPE: hstore の GiST インデックス用シグネチャを保持する構造体。
  • BITVECP: ビットベクトルを操作するためのポインタ型。

失敗モードと境界条件

  • siglen が 0 の場合、ループは実行されず result は true となります。
  • メモリ上の不正なポインタが渡された場合、GETSIGN や ISALLTRUE の実行時にセグメンテーションフォールトが発生する可能性があります。

保守担当者向けメモ

  • 本関数は GiST インデックスの same メソッドとして機能します。
  • ISALLTRUE は、インデックスの圧縮時にシグネチャがオーバーフローした場合などに設定されるフラグをチェックします。
  • 比較はバイト単位で行われるため、siglen の値がインデックス作成時と検索時で一致していることが前提となります。
