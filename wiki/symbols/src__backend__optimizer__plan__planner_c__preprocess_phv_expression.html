preprocess_phv_expression

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/planner.c:1642

目的

PlaceHolderVar (PHV) に含まれる式を、プランニングの各フェーズで処理可能な形式に正規化・変換します。

シグネチャまたは形状

Expr *preprocess_phv_expression(PlannerInfo *root, Expr *expr)

入力と出力

  • **入力**:
  • root: プランニングのグローバル状態を保持する PlannerInfo 構造体。
  • expr: 処理対象となる式（Expr）。
  • **出力**:
  • 変換済みの式（Expr）。

前提条件と不変条件

  • root は有効な PlannerInfo 構造体を指している必要があります。
  • expr は EXPRKIND_PHV として処理されることが前提となります。

アルゴリズムまたは振る舞い

内部的に preprocess_expression 関数を呼び出し、EXPRKIND_PHV という式種別を指定して式の前処理を実行します。この処理により、式ツリー内の定数畳み込みやサブリンクの変換など、プランニングに必要な正規化が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/initsplan.c:818 にて、phv->phexpr の更新時に呼び出されています。
  • **呼び出し先**:
  • preprocess_expression (同ファイル内)

関連データ構造

  • PlannerInfo: プランニングのコンテキスト情報。
  • Expr: PostgreSQL の式ツリーの基底ノード。

失敗モードと境界条件

  • 入力 expr が NULL の場合や、preprocess_expression 内で処理できない不正なノード構造が含まれている場合、基盤となる式処理関数に依存したエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は preprocess_expression のラッパーです。PHV 特有の処理が必要になった場合は、EXPRKIND_PHV の定義を確認してください。
