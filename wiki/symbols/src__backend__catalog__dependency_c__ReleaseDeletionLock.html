ReleaseDeletionLock

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/catalog/dependency.c:1527

目的

オブジェクトの削除処理に関連して取得された AccessExclusiveLock を解放します。

シグネチャまたは形状

void ReleaseDeletionLock(const ObjectAddress *object)

入力と出力

  • **入力**: object (解放対象のオブジェクトを指す ObjectAddress 構造体へのポインタ)
  • **出力**: なし（副作用としてロックを解放）

前提条件と不変条件

  • 入力された object は有効な ObjectAddress である必要があります。
  • 呼び出し前に、対応する AccessExclusiveLock が当該オブジェクトに対して取得されている必要があります。

アルゴリズムまたは振る舞い

1. object->classId が RelationRelationId（リレーション）であるかを確認します。 2. リレーションの場合は UnlockRelationOid を呼び出し、対象の OID に対して AccessExclusiveLock を解放します。 3. それ以外の場合は、UnlockDatabaseObject を呼び出し、対象のクラス ID と OID に対して AccessExclusiveLock を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/dependency.c 内の複数箇所（削除処理の制御フロー中）
  • src/backend/catalog/pg_shdepend.c 内の複数箇所
  • **呼び出し先**:
  • UnlockRelationOid
  • UnlockDatabaseObject

関連データ構造

  • ObjectAddress: 削除対象オブジェクトを識別するための構造体。

失敗モードと境界条件

  • ロックが取得されていない状態で呼び出した場合の挙動は、基盤となるロックマネージャの仕様に依存します。
  • サブオブジェクトに対するロック解放は明示的にサポートされておらず、オブジェクト全体に対するロック解放を前提としています。

保守担当者向けメモ

  • 本関数は AcquireDeletionLock と対になる操作です。新しいオブジェクトタイプを追加する際は、ロックの取得・解放ロジックが整合しているか確認してください。
