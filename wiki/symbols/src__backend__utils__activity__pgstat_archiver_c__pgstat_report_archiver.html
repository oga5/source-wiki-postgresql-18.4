pgstat_report_archiver

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/activity/pgstat_archiver.c:27

目的

WAL アーカイバプロセスによる WAL ファイルのアーカイブ結果（成功または失敗）を統計情報として記録し、共有メモリ上の統計構造体を更新する。

シグネチャまたは形状

void pgstat_report_archiver(const char *xlog, bool failed)

入力と出力

  • **入力**:
  • xlog: アーカイブ対象の WAL ファイル名。
  • failed: アーカイブが失敗した場合は true、成功した場合は false。
  • **出力**: なし（副作用として共有メモリ上の統計情報を更新）。

前提条件と不変条件

  • pgStatLocal.shmem->archiver が初期化されていること。
  • 共有メモリへの書き込み整合性を保つため、changecount を用いたロックフリーな更新プロトコルに従うこと。

アルゴリズムまたは振る舞い

1. GetCurrentTimestamp() を呼び出し、現在のタイムスタンプを取得する。 2. pgstat_begin_changecount_write を呼び出し、共有メモリ更新の開始を宣言する。 3. failed フラグに基づき、以下のいずれかの統計を更新する：

  • **失敗時**: failed_count をインクリメントし、last_failed_wal にファイル名をコピーし、last_failed_timestamp を現在時刻に更新する。
  • **成功時**: archived_count をインクリメントし、last_archived_wal にファイル名をコピーし、last_archived_timestamp を現在時刻に更新する。

    • 4. pgstat_end_changecount_write を呼び出し、更新を完了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/pgarch.c:484 (アーカイブ成功時)
  • src/backend/postmaster/pgarch.c:494 (アーカイブ失敗時)
  • **呼び出し先**:
  • GetCurrentTimestamp()
  • pgstat_begin_changecount_write()
  • pgstat_end_changecount_write()

関連データ構造

  • PgStatShared_Archiver: 共有メモリ上に配置されたアーカイバ統計の本体。
  • pgStatLocal.shmem: 統計サブシステムが使用する共有メモリ領域。

失敗モードと境界条件

  • xlog が NULL または無効なポインタである場合、memcpy によるメモリ破壊の可能性がある（呼び出し元での保証が必要）。
  • 共有メモリが未初期化の状態で呼び出された場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は pgstat_archiver_snapshot_cb と対になる更新処理です。統計の整合性は changecount によって保護されているため、更新ロジックを変更する際は必ず begin/end のペアで囲む必要があります。
