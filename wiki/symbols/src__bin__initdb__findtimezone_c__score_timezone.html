score_timezone

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/findtimezone.c:233

目的

指定されたタイムゾーン名が、システムの現在のタイムゾーン設定とどの程度一致するかをスコアリングします。

シグネチャまたは形状

int score_timezone(const char *tzname, struct tztry *tt);

入力と出力

  • **入力**:
  • tzname: 評価対象のタイムゾーン名（文字列）。
  • tt: 比較に使用するテスト時刻データを含む tztry 構造体。
  • **出力**:
  • int: 一致したテスト時刻の数（スコア）。完全に一致した場合は tt->n_test_times を返します。失敗時や無効なゾーンの場合は -1 を返します。

前提条件と不変条件

  • tzname は pg_load_tz で読み込み可能であること。
  • タイムゾーン定義がうるう秒を含む場合、この関数は -1 を返し、そのゾーンを拒否します。

アルゴリズムまたは振る舞い

1. pg_load_tz を用いてタイムゾーン定義を読み込みます。 2. pg_tz_acceptable でうるう秒の有無を確認し、不適切な場合は終了します。 3. tt->test_times に格納された各時刻について、以下の比較を行います。

  • pg_localtime（PostgreSQL内部ロジック）と localtime（システム標準）の結果を比較します。
  • compare_tm を使用して、年月日時分秒およびDSTフラグが一致するか確認します。
  • 一致する場合、strftime で取得したタイムゾーン略称（%Z）が pgtm->tm_zone と一致するかを確認します。

    • 4. 比較に失敗した時点でのインデックス i を返し、すべて一致した場合は最終的なカウントを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/initdb/findtimezone.c 内の複数箇所（identify_system_timezone 等）から、タイムゾーンの候補を検証するために呼び出されます。
  • **呼び出し先**: pg_load_tz, pg_tz_acceptable, pg_localtime, localtime, compare_tm, strftime, strcmp, memset。

関連データ構造

  • tztry: 比較用のテスト時刻セットを保持する構造体。
  • pg_tz: PostgreSQLのタイムゾーン定義構造体。

失敗モードと境界条件

  • pg_load_tz が失敗した場合（ゾーン名が不明）は -1 を返します。
  • うるう秒を含むタイムゾーンは拒否されます。
  • システムの localtime がデータを返さない場合、その時点までのスコアを返します。
  • タイムゾーン略称が一致しない場合、その時点までのスコアを返します。

保守担当者向けメモ

  • 本関数は「ブルートフォース」的なサンプリング比較を行うため、呼び出し頻度が高い場合はパフォーマンスに影響を与える可能性があります。
  • DEBUG_IDENTIFY_TIMEZONE マクロが定義されている場合、比較失敗時の詳細なログが stderr に出力されます。
