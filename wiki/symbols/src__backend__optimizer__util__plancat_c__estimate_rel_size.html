estimate_rel_size

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/plancat.c:1074

目的

リレーション（テーブルまたはインデックス）のページ数、タプル数、および全可視フラクション（all-visible fraction）を推定し、プランナのコスト計算用データを提供します。

シグネチャまたは形状

void estimate_rel_size(Relation rel, int32 *attr_widths, BlockNumber *pages, double *tuples, double *allvisfrac)

入力と出力

  • **入力**:
  • rel: 対象リレーションの Relation 構造体。
  • attr_widths: 属性幅の配列（get_rel_data_width 等で使用）。
  • **出力**:
  • pages: 推定ページ数（BlockNumber へのポインタ）。
  • tuples: 推定タプル数（double へのポインタ）。
  • allvisfrac: 全可視ページの割合（double へのポインタ）。

前提条件と不変条件

  • rel は既にオープンされており、適切なロックが取得されている必要があります。
  • RELKIND_HAS_TABLE_AM が真の場合、テーブルアクセスメソッド（AM）の推定ロジックに委譲されます。

アルゴリズムまたは振る舞い

1. **テーブルAM**: table_relation_estimate_size を呼び出し、AM固有の推定を実行します。 2. **インデックス**:

  • RelationGetNumberOfBlocks で物理ページ数を取得します。
  • pg_class の統計情報（relpages, reltuples）に基づき、タプル密度を計算します。
  • 統計情報が不足している場合、属性データ型からタプル幅を算出し、ページが満杯であると仮定して密度を推定します。
  • メタページ分を考慮してページ数を調整し、最終的なタプル数を算出します。
  • relallvisible を基に全可視フラクションを計算します。

    • 3. **その他（外部テーブル等）**: pg_class の値をそのまま使用します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/hash/hash.c
  • src/backend/optimizer/plan/planner.c
  • src/backend/optimizer/util/plancat.c 内の各所
  • **呼び出し先**:
  • table_relation_estimate_size
  • RelationGetNumberOfBlocks
  • get_rel_data_width

関連データ構造

  • Relation: リレーションのメタデータおよび物理特性を保持。
  • Form_pg_class: rel->rd_rel を通じてアクセスされる統計情報。

失敗モードと境界条件

  • curpages が 0 の場合、タプル数と全可視フラクションを 0 として即時終了します。
  • 統計情報（reltuples）が負の値の場合、データ型に基づく推定ロジックへフォールバックします。
  • インデックスのメタページ考慮は、btree, hash, GIN には適していますが、GiST 等では不正確な可能性があります。

保守担当者向けメモ

  • 本関数はプランナのコスト見積もりの基礎となるため、変更時はプランナの挙動への影響を考慮してください。
  • インデックスのタプル密度推定ロジックは簡略化されており、将来的にはインデックスタイプごとのコールバックへ移行することが望ましいとされています。
