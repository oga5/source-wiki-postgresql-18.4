openFlagsToCreateFileFlags

定義

  • 種別: function
  • 定義箇所: src/port/open.c:28

目的

POSIX 準拠のファイルオープンフラグ（O_CREAT, O_TRUNC, O_EXCL）を、Windows API の CreateFile 関数で利用される dwCreationDisposition パラメータへ変換する。

シグネチャまたは形状

static int openFlagsToCreateFileFlags(int openFlags)

入力と出力

  • **入力**: openFlags (POSIX のファイルオープンフラグのビットマスク)
  • **出力**: Windows の dwCreationDisposition に対応する整数値（OPEN_EXISTING, OPEN_ALWAYS, TRUNCATE_EXISTING, CREATE_ALWAYS, CREATE_NEW のいずれか）

前提条件と不変条件

  • 入力されるフラグは O_CREAT, O_TRUNC, O_EXCL の組み合わせを想定している。
  • O_EXCL は O_CREAT と併用されない場合、無視される。
  • O_TRUNC は O_CREAT と併用される場合、O_TRUNC の意味が優先される（CREATE_ALWAYS 等のロジックに反映される）。

アルゴリズムまたは振る舞い

openFlags のビットマスクに対し switch 文を用いて以下の変換を行う：

  • 0 または O_EXCL: OPEN_EXISTING
  • O_CREAT: OPEN_ALWAYS
  • O_TRUNC または O_TRUNC | O_EXCL: TRUNCATE_EXISTING
  • O_CREAT | O_TRUNC: CREATE_ALWAYS
  • O_CREAT | O_EXCL または O_CREAT | O_TRUNC | O_EXCL: CREATE_NEW

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/open.c 内のファイルオープン処理（pgwin32_open 等）から参照されている。
  • **呼び出し先**: 本関数は変換ロジックのみを持ち、外部関数を呼び出さない。

関連データ構造

  • fcntl.h で定義される POSIX フラグ定数。
  • Windows API の dwCreationDisposition 定数。

失敗モードと境界条件

  • 定義されたフラグの組み合わせ以外が渡された場合、デフォルトとして 0 を返すが、現在の実装ロジック上、網羅的なケース分けにより到達不能である。

保守担当者向けメモ

  • 本関数は Windows 環境専用の移植層の一部である。
  • O_EXCL や O_TRUNC の組み合わせによる挙動は、POSIX のセマンティクスを Windows の CreateFile で正確に再現するために調整されている。変更時は CreateFile の dwCreationDisposition の仕様を再確認すること。
