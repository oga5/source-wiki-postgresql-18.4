ShutdownMode

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_ctl/pg_ctl.c:37

目的

PostgreSQL サーバー（postmaster）を停止する際のシャットダウンモードを定義します。

シグネチャまたは形状

typedef enum
{
	SMART_MODE,
	FAST_MODE,
	IMMEDIATE_MODE,
} ShutdownMode;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: SMART_MODE, FAST_MODE, IMMEDIATE_MODE のいずれかの定数

前提条件と不変条件

  • サーバー停止操作において、いずれか一つのモードが選択される必要があります。
  • デフォルト値として FAST_MODE が使用されます（src/bin/pg_ctl/pg_ctl.c:79）。

アルゴリズムまたは振る舞い

  • SMART_MODE: 既存の接続が終了するのを待機してから停止します。
  • FAST_MODE: 接続を強制的に切断し、チェックポイントを実行して停止します。
  • IMMEDIATE_MODE: チェックポイントを実行せず、即座にプロセスを終了させます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_ctl/pg_ctl.c 内の shutdown_mode 変数定義にて参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • CtlCommand: pg_ctl の実行コマンドを定義する列挙型。

失敗モードと境界条件

  • 該当なし。

保守担当者向けメモ

  • 本列挙型は pg_ctl の停止コマンド実行時の挙動を制御します。新しいシャットダウンモードを追加する場合は、この列挙型とそれを利用する停止ロジックの両方を更新する必要があります。
