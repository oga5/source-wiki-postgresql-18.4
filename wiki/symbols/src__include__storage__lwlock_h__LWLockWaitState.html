LWLockWaitState

定義

  • 種別: enum
  • 定義箇所: src/include/storage/lwlock.h:28

目的

軽量ロック（LWLock）の競合発生時における、プロセスの待機状態を追跡するために使用されます。

シグネチャまたは形状

typedef enum LWLockWaitState
{
    LW_WS_NOT_WAITING,      /* 待機中ではない、または起床済み */
    LW_WS_WAITING,          /* 現在待機中 */
} LWLockWaitState;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし

前提条件と不変条件

  • プロセスがロックを待機する際、状態は LW_WS_WAITING に遷移します。
  • LW_WS_PENDING_WAKEUP は、ロックの解放処理によって待機リストから外された後、実際にプロセスが起床するまでの過渡的な状態を表します。

アルゴリズムまたは振る舞い

LWLock の待機管理において、プロセスがロック取得を試みて失敗した場合、自身の状態を LW_WS_WAITING に設定して待機リストに加わります。ロック解放時に待機者が起床させられる際、状態は LW_WS_PENDING_WAKEUP を経由して LW_WS_NOT_WAITING へと遷移します。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/storage/proc.h 内の PGPROC 構造体（lwWaiting メンバ）がこの状態を参照しています。
  • 呼び出し先: 情報なし。

関連データ構造

  • PGPROC: プロセスごとの情報を保持する構造体。lwWaiting メンバを通じてこの列挙型を利用します。
  • LWLock: 軽量ロック本体。

失敗モードと境界条件

  • 該当なし。

保守担当者向けメモ

  • 本シンボルは PGPROC 構造体と密接に関連しています。状態遷移のロジックを変更する場合は、src/backend/storage/lmgr/lwlock.c 内のロック取得・解放処理におけるアトミックな状態更新手順を確認してください。
