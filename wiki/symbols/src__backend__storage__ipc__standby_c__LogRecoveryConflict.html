LogRecoveryConflict

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/standby.c:273

目的

ホットスタンバイモードにおいて、リカバリプロセスがクエリ実行バックエンドとの競合により待機している状態、または待機が終了した状態をログに記録する。

シグネチャまたは形状

void LogRecoveryConflict(ProcSignalReason reason, TimestampTz wait_start,
                         TimestampTz now, VirtualTransactionId *wait_list,
                         bool still_waiting)

入力と出力

  • reason: 競合の原因（ProcSignalReason）。
  • wait_start: 待機開始時刻。
  • now: 現在時刻。
  • wait_list: 競合しているバックエンドの VirtualTransactionId 配列。
  • 出力: なし（副作用としてログ出力を行う）。

前提条件と不変条件

  • still_waiting が false の場合、wait_list は NULL でなければならない。

アルゴリズムまたは振る舞い

1. wait_start と now の差分から待機時間を算出する。 2. wait_list が提供されている場合、各 VirtualTransactionId に対応するプロセス（PGPROC）を検索し、アクティブなプロセスの PID を文字列バッファに構築する。 4. still_waiting が false の場合、待機が終了した旨を LOG レベルで出力する。 5. 構築した文字列バッファがあればメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c (バッファピン競合時)
  • src/backend/storage/ipc/standby.c (リカバリ競合解決処理内)
  • src/backend/storage/lmgr/proc.c (ロック競合時)
  • **呼び出し先**:
  • TimestampDifference
  • ProcNumberGetProc
  • initStringInfo, appendStringInfo
  • get_recovery_conflict_desc
  • ereport
  • pfree

関連データ構造

  • VirtualTransactionId: 競合するバックエンドの識別子。
  • PGPROC: バックエンドプロセスの状態管理構造体。
  • StringInfoData: ログ出力用文字列構築に使用。

失敗モードと境界条件

  • wait_list 内の VirtualTransactionId に対応する PGPROC が存在しない（既に終了している）場合、その PID はログ出力対象から除外される。
  • 競合プロセスが一つもアクティブでない場合、詳細メッセージ（errdetail_log_plural）は出力されない。

保守担当者向けメモ

  • 本関数は競合解決の進捗を可視化するためのものであり、状態遷移そのものには影響を与えない。
  • ログ出力のフォーマットを変更する場合は、get_recovery_conflict_desc が返す説明文との整合性に注意すること。
