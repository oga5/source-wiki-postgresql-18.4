concat_text

定義

  • 種別: function
  • 定義箇所: src/tutorial/funcs.c:90

目的

2つの text 型引数を連結し、新しい text 型として返す PostgreSQL ユーザー定義関数です。

シグネチャまたは形状

Datum concat_text(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される2つの text 型引数。
  • **出力**: 連結された結果を格納した text 型の Datum。

前提条件と不変条件

  • PostgreSQL の V1 呼び出し規約に従う必要があります。
  • PG_GETARG_TEXT_PP を使用して引数を取得するため、入力データは TOAST 展開済み（またはショートヘッダー対応）の状態で処理されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を用いて2つの入力引数を取得します。 2. VARSIZE_ANY_EXHDR を使用して、各引数のヘッダーを除いたデータサイズを算出します。 3. 連結後の合計サイズ（各データサイズ + VARHDRSZ）を計算します。 4. palloc を使用して、新しい text 型のためのメモリを確保します。 5. SET_VARSIZE で新しいテキストのサイズを設定します。 6. memcpy を使用して、1番目の引数、続いて2番目の引数のデータを新しいメモリ領域へ順次コピーします。 7. PG_RETURN_TEXT_P を介して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されます（直接的な C 言語レベルの呼び出し元はソースコード上では特定されていません）。
  • **呼び出し先**:
  • palloc: メモリ確保に使用。
  • memcpy: データコピーに使用。
  • PG_GETARG_TEXT_PP / PG_RETURN_TEXT_P: PostgreSQL の関数インターフェースマクロ。
  • VARSIZE_ANY_EXHDR / VARDATA / VARDATA_ANY / SET_VARHDRSZ: 可変長データ操作マクロ。

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させ、トランザクションがアボートされます。
  • 入力引数が NULL の場合、PG_GETARG_TEXT_PP の仕様に従い、NULL ポインタのデリファレンスを避けるための適切なハンドリングが必要です（本実装では NULL チェックの明示的な記述はありません）。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_INFO_V1(concat_text) マクロにより V1 規約として登録されています。
  • palloc で確保したメモリは PostgreSQL のメモリコンテキストによって管理されるため、手動の free は不要です。
  • 可変長データの操作には、必ず VARSIZE_ANY_EXHDR や VARDATA_ANY などのマクロを使用し、ヘッダー構造の差異（圧縮やショートヘッダー）を吸収するようにしてください。
