AggCheckCallContext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAgg.c:4613

目的

SQL関数が集約関数またはウィンドウ関数として呼び出されているかを判定し、その実行コンテキスト（メモリコンテキスト）を取得する。

シグネチャまたは形状

int AggCheckCallContext(FunctionCallInfo fcinfo, MemoryContext *aggcontext)

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報。
  • aggcontext: 呼び出し元が提供する MemoryContext へのポインタ（出力用）。
  • **出力**:
  • 戻り値: 呼び出しコンテキストの種類を示す整数値（AGG_CONTEXT_AGGREGATE、AGG_CONTEXT_WINDOW、または 0）。
  • aggcontext: 呼び出しが有効な場合、対応するメモリコンテキストが格納される。

前提条件と不変条件

  • fcinfo->context が AggState または WindowAggState のいずれかである場合にのみ、有効なコンテキストが返される。
  • aggcontext が NULL でない場合のみ、メモリコンテキストの書き込みが行われる。

アルゴリズムまたは振る舞い

1. fcinfo->context が AggState 型か確認する。該当する場合、AggState 内の curaggcontext->ecxt_per_tuple_memory を aggcontext に設定し、AGG_CONTEXT_AGGREGATE を返す。 2. fcinfo->context が WindowAggState 型か確認する。該当する場合、WindowAggState 内の curaggcontext を aggcontext に設定し、AGG_CONTEXT_WINDOW を返す。 3. いずれにも該当しない場合、aggcontext を NULL に初期化し、0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAgg.c (複数箇所)
  • src/backend/executor/nodeWindowAgg.c
  • src/backend/utils/adt/array_userfuncs.c
  • **呼び出し先**:
  • IsA マクロ（型判定）

関連データ構造

  • FunctionCallInfo: 関数呼び出しのメタデータ。
  • AggState: 集約実行ノードの状態。
  • WindowAggState: ウィンドウ関数実行ノードの状態。
  • MemoryContext: メモリ管理コンテキスト。

失敗モードと境界条件

  • fcinfo->context が NULL の場合、または期待される型（AggState / WindowAggState）でない場合、戻り値は 0 となり、aggcontext は NULL に設定される。

保守担当者向けメモ

  • 本関数は、集約関数やウィンドウ関数が実行中にメモリコンテキストを安全に取得するための標準的なAPIである。
  • 戻り値の定数（AGG_CONTEXT_AGGREGATE 等）は、呼び出し側で集約の種類を判別するために使用される。
