generateSerialExtraStmts

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_utilcmd.c:389

目的

SERIAL 型または IDENTITY 列に関連付けられるシーケンスオブジェクトを生成し、テーブル作成や列追加の前後で実行すべきコマンド（CREATE SEQUENCE および ALTER SEQUENCE ... OWNED BY）を構築する。

シグネチャまたは形状

static void generateSerialExtraStmts(CreateStmtContext *cxt, ColumnDef *column,
                                     Oid seqtypid, List *seqoptions,
                                     bool for_identity, bool col_exists,
                                     char **snamespace_p, char **sname_p)

入力と出力

  • **入力**:
  • cxt: 変換状態を保持する CreateStmtContext。
  • column: 対象の列定義。
  • seqtypid: シーケンスのデータ型 OID。
  • seqoptions: シーケンス生成オプションのリスト。
  • for_identity: IDENTITY 列かどうかを示すフラグ。
  • **出力**:
  • cxt->blist に CreateSeqStmt を追加。
  • column->identitySequence に生成されたシーケンス名を格納。
  • snamespace_p, sname_p: 生成されたシーケンスのスキーマ名とリレーション名を返す（出力パラメータ）。

前提条件と不変条件

  • cxt->rel（既存テーブル）または cxt->relation（新規作成テーブル）が有効であること。
  • シーケンスの永続性は、対象テーブルの永続性（RELPERSISTENCE）に準拠する。

アルゴリズムまたは振る舞い

1. **オプション解析**: seqoptions から sequence_name や logged/unlogged 指定を抽出し、標準的なシーケンスオプションから分離する。 2. **名前解決**: 指定があればその名前を、なければ ChooseRelationName を用いて自動生成する。 3. **シーケンス作成コマンド構築**: CreateSeqStmt を生成し、cxt->blist に追加する。 4. **所有権設定**: ALTER SEQUENCE ... OWNED BY を構築し、シーケンスを列に関連付ける。 5. **メタデータ更新**: column->identitySequence にシーケンス情報を保持する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_utilcmd.c 内の transformColumnDefinition 等から複数回呼び出される。
  • **呼び出し先**: makeRangeVarFromNameList, RelationGetNamespace, RangeVarGetCreationNamespace, ChooseRelationName, makeNode, lappend 等。

関連データ構造

  • CreateStmtContext: 変換処理の文脈を保持する構造体。
  • ColumnDef: 列定義ノード。
  • CreateSeqStmt, AlterSeqStmt: シーケンス操作用ノード。

失敗モードと境界条件

  • **競合**: sequence_name と logged/unlogged が重複指定された場合にエラーを報告。
  • **一時テーブル制約**: 一時テーブルに対して logged 属性を指定した場合にエラーを報告。
  • **名前の衝突**: 自動生成されるシーケンス名が既存のオブジェクトと衝突する可能性がある（実装上の注意点として言及あり）。

保守担当者向けメモ

  • 本関数は CREATE TABLE や ALTER TABLE の解析フェーズで実行される。
  • blist（前処理リスト）への追加が主目的であり、シーケンスの所有権設定は ALTER TABLE の実行順序に依存する。
  • snamespace_p と sname_p の出力パラメータは、呼び出し元でシーケンス名の追跡に使用される。
