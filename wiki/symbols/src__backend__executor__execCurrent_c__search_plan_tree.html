search_plan_tree

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execCurrent.c:313

目的

プランツリーを再帰的に探索し、指定されたテーブル（table_oid）をスキャンしている ScanState ノードを特定します。これは WHERE CURRENT OF 句による位置指定更新・削除操作において、対象行を特定するために使用されます。

シグネチャまたは形状

static ScanState *search_plan_tree(PlanState *node, Oid table_oid, bool *pending_rescan)

入力と出力

  • **入力**:
  • node: 探索を開始するプランツリーのルートノード（PlanState）。
  • table_oid: 検索対象となるテーブルの OID。
  • pending_rescan: 探索中に chgParam が設定されたノードが見つかった場合に true が書き込まれるフラグへのポインタ。
  • **出力**:
  • 対象テーブルをスキャンしている ScanState へのポインタ。見つからない場合は NULL。

前提条件と不変条件

  • node が NULL の場合は NULL を返します。
  • AppendState において、対象テーブルをスキャンするノードが複数見つかった場合は、曖昧さを避けるため NULL を返します。
  • MergeAppendState は、その内部状態を透過的に探索できないため、本関数ではサポート対象外です。

アルゴリズムまたは振る舞い

1. **ノード種別の判定**:

  • ScanState 系（SeqScan, IndexScan 等）: ss_currentRelation が table_oid と一致するか確認し、一致すればそのノードを返します。
  • AppendState: 各子ノードを再帰的に探索します。
  • ResultState, LimitState, SubqueryScanState: 子ノードを再帰的に探索します。
  • その他: 探索を停止します。

    • 2. **再スキャンフラグの更新**: 探索過程で node->chgParam が NULL でないノードを通過した場合、*pending_rescan を true に設定します。 3. **再帰的探索**: ツリーを深さ優先で探索し、条件に合致する単一のノードを特定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execCurrent.c 内の execCurrentOf 関数から呼び出されます。
  • 同ファイル内で再帰的に呼び出されます。
  • **呼び出し先**:
  • RelationGetRelid（マクロ）
  • nodeTag（マクロ）
  • outerPlanState（マクロ）
  • search_plan_tree（自身を再帰呼び出し）

関連データ構造

  • PlanState: 実行プランのノード状態を保持する基底構造体。
  • ScanState: スキャン操作の状態を保持する構造体。
  • AppendState, SubqueryScanState: 特定のプランノードの状態構造体。

失敗モードと境界条件

  • **複数マッチ**: Append ノード配下で対象テーブルが複数回スキャンされている場合、NULL を返します。
  • **探索不可**: MergeAppend など、本関数が対応していないノード構造に遭遇した場合、探索はそこで打ち切られ、結果として NULL が返されます。
  • **非アクティブなスキャン**: 継承テーブルなどで対象テーブルがスキャンされていても、それが現在アクティブでない場合、呼び出し元がそのノードの出力結果を検証することで適切に処理されます。

保守担当者向けメモ

  • 本関数は MergeAppend をサポートしていません。ORDER BY を伴うクエリで WHERE CURRENT OF が利用できない制限の主因となっています。
  • ForeignScan や CustomScan は ss_currentRelation が設定されていない場合、探索対象から除外されます。
