pg_gssinfo

定義

  • 種別: struct
  • 定義箇所: src/include/libpq/libpq-be.h:61

目的

PostgreSQL バックエンドにおいて、GSSAPI 認証および暗号化に関連する接続状態を保持するための構造体です。

シグネチャまたは形状

typedef struct
{
	gss_buffer_desc outbuf;
#ifdef ENABLE_GSS
	gss_cred_id_t cred;
	gss_ctx_id_t ctx;
	gss_name_t	name;
	char	   *princ;
	bool		auth;
	bool		enc;
	bool		delegated_creds;
#endif
} pg_gssinfo;

入力と出力

  • **入力**: GSSAPI 認証プロセス中に生成されるトークン、資格情報、コンテキスト情報。
  • **出力**: 認証の成否（auth）、暗号化の有効状態（enc）、および認証に使用されたプリンシパル名（princ）。

前提条件と不変条件

  • ENABLE_GSS が定義されていないビルド環境では、outbuf のみが構造体に含まれます。
  • 構造体内の各フィールドは、GSSAPI 認証フローの進行に応じて初期化されます。
  • princ フィールドは、GSSAPI 認証が使用されなかった場合は NULL となります。

アルゴリズムまたは振る舞い

  • GSSAPI 認証および暗号化のセッション状態を管理します。
  • outbuf を使用して、GSSAPI トークンの送受信バッファを保持します。
  • auth および enc フラグにより、後続の通信処理において GSSAPI 固有のセキュリティ層を適用するかどうかを制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth.c および src/backend/libpq/be-secure-gssapi.c において、Port 構造体の gss フィールドとして動的に割り当てられ、初期化されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数呼び出しの対象ではありません。

関連データ構造

  • Port: バックエンドの接続状態を管理する構造体であり、pg_gssinfo *gss フィールドを保持します。
  • gss_buffer_desc, gss_cred_id_t, gss_ctx_id_t, gss_name_t: GSSAPI ライブラリが提供する標準的な型定義。

失敗モードと境界条件

  • メモリ割り当て失敗: MemoryContextAllocZero 等による割り当てが失敗した場合、バックエンドプロセスは異常終了する可能性があります。
  • GSSAPI 未サポート環境: ENABLE_GSS が未定義の場合、認証や暗号化に関するフィールドにはアクセスできません。

保守担当者向けメモ

  • Port 構造体の一部として TopMemoryContext 上で割り当てられることを前提としています。
  • 拡張機能や内部コードでこの構造体を操作する際は、ENABLE_GSS の有無による構造体のサイズ変化に注意してください。
