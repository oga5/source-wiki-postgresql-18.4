PGSharedMemoryIsInUse

定義

  • 種別: function
  • 定義箇所: src/backend/port/sysv_shmem.c:316

目的

指定された共有メモリセグメントが、現在他のプロセスによって使用中であるかどうかを判定します。

シグネチャまたは形状

bool PGSharedMemoryIsInUse(unsigned long id1, unsigned long id2)

入力と出力

  • **入力**:
  • id1: 使用されていません（将来的な拡張用または互換性維持のための引数）。
  • id2: 共有メモリセグメントの識別子（IpcMemoryId としてキャストして使用）。
  • **出力**:
  • bool: セグメントが使用中であれば true、そうでなければ false を返します。

前提条件と不変条件

  • PGSharedMemoryAttach が正常に呼び出せる環境であること。
  • 呼び出し後に shmdt を実行して、一時的にアタッチしたメモリをデタッチする不変条件を持ちます。

アルゴリズムまたは振る舞い

1. PGSharedMemoryAttach を呼び出し、指定された id2 に対応する共有メモリセグメントの状態を取得します。 2. セグメントのアタッチに成功した場合、直ちに shmdt を呼び出してデタッチします。デタッチ失敗時はログに記録します。 3. PGSharedMemoryAttach から返された IpcMemoryState に基づき判定します。

  • SHMSTATE_ENOENT, SHMSTATE_FOREIGN, SHMSTATE_UNATTACHED の場合は false を返します。
  • SHMSTATE_ANALYSIS_FAILURE, SHMSTATE_ATTACHED の場合は true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/init/miscinit.c:1404
  • **呼び出し先**:
  • PGSharedMemoryAttach
  • shmdt (システムコール)
  • elog

関連データ構造

  • IpcMemoryState (列挙型)
  • PGShmemHeader

失敗モードと境界条件

  • shmdt が失敗した場合、ログにエラーを出力しますが、関数の戻り値には影響を与えません。
  • PGSharedMemoryAttach が SHMSTATE_ANALYSIS_FAILURE を返した場合、安全側に倒して true（使用中）と判定します。

保守担当者向けメモ

  • id1 引数は現在の実装では使用されていません。
  • 本関数は src/backend/port/win32_shmem.c にも同名の関数が存在し、プラットフォーム間でインターフェースが共通化されています。
