JsonManifestFileField

定義

  • 種別: enum
  • 定義箇所: src/common/parse_manifest.c:44

目的

バックアップマニフェスト（JSON形式）内の各ファイルエントリに含まれるフィールドを識別するための列挙型です。

シグネチャまたは形状

typedef enum
{
    JMFF_PATH,
    JMFF_ENCODED_PATH,
    JMFF_SIZE,
    JMFF_LAST_MODIFIED,
    JMFF_CHECKSUM_ALGORITHM,
    JMFF_CHECKSUM,
} JsonManifestFileField;

入力と出力

  • 入力: JSON解析プロセスにおけるフィールド名。
  • 出力: 解析中のフィールドを特定するための列挙値。

前提条件と不変条件

  • 本列挙型は、マニフェスト解析時の状態遷移において、現在処理中のキーがどのファイル属性に対応するかを追跡するために使用されます。

アルゴリズムまたは振る舞い

  • JSON解析中に特定のキー（例: "Path", "Size" 等）が検出された際、対応する JsonManifestFileField 値が状態管理に使用されます。
  • JMFF_ENCODED_PATH は、16進数エンコードされたパスを扱う際に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/parse_manifest.c 内のJSON解析ハンドラ。
  • 呼び出し先: 情報なし。

関連データ構造

  • JsonManifestParseState: 解析中の内部状態を保持する構造体。この構造体内で JsonManifestFileField 型のメンバ（file_field）が使用されています。

失敗モードと境界条件

  • マニフェスト内に定義外のフィールドが存在する場合、解析ロジックが適切に無視またはエラーを返す必要があります。

保守担当者向けメモ

  • 新しいファイル属性をマニフェストに追加する場合は、本列挙型に新しいメンバを追加し、解析ロジック（json_manifest_object_field_start 等）を更新してください。
