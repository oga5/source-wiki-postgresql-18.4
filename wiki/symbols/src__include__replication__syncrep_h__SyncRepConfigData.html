SyncRepConfigData

定義

  • **種別**: struct
  • **定義箇所**: src/include/replication/syncrep.h:63

目的

同期レプリケーションの設定情報を保持するための構造体です。synchronous_standby_names GUCパラメータの解析結果を格納し、プライマリサーバーがどのスタンバイサーバーを、どのような手法（PriorityベースまたはQuorumベース）で待機すべきかを定義します。

シグネチャまたは形状

typedef struct SyncRepConfigData
{
    int         config_size;    /* 構造体全体のサイズ（バイト単位） */
    int         num_sync;       /* 待機対象とする同期スタンバイの数 */
    uint8       syncrep_method; /* スタンバイ選択手法 */
    int         nmembers;       /* メンバー名の数 */
    char        member_names[FLEXIBLE_ARRAY_MEMBER]; /* メンバー名のリスト */
} SyncRepConfigData;

入力と出力

  • **入力**: synchronous_standby_names の構文解析プロセス（syncrep_yyparse）を通じて生成されます。
  • **出力**: 同期レプリケーションの待機ロジックにおいて、待機条件の判定基準として参照されます。

前提条件と不変条件

  • member_names は nmembers 個のヌル終端文字列が連続して格納される可変長配列です。
  • 構造体はフラットなメモリレイアウトとして設計されており、共有メモリ等での利用を想定しています。

アルゴリズムまたは振る舞い

  • syncrep_method に基づき、WalSender プロセスがスタンバイからの応答を評価する際の基準（FIRST や ANY など）を決定します。
  • 待機中のバックエンドプロセスは、この構造体で定義された条件が満たされるまで待機を継続します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/replication/syncrep.c: SyncRepConfig グローバル変数として保持され、待機判定に使用されます。
  • syncrep_yyparse: 構文解析の結果として本構造体を生成します。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数呼び出しは含みません。

関連データ構造

  • SyncRepStandbyData: 各スタンバイの進捗状況を保持する構造体。本構造体と組み合わせて同期判定に使用されます。

失敗モードと境界条件

  • member_names が可変長であるため、メモリ確保時に config_size を正しく計算する必要があります。
  • 構文解析に失敗した場合、syncrep_parse_result は NULL となる可能性があります。

保守担当者向けメモ

  • 本構造体は synchronous_standby_names の「extra」データとして扱われます。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のコピーやメモリ確保を行う際は config_size を参照してください。
