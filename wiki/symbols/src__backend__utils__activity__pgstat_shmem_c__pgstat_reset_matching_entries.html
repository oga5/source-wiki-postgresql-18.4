pgstat_reset_matching_entries

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_shmem.c:1120

目的

共有メモリ上の統計情報エントリのうち、指定された条件に合致するものを検索し、内容をリセットする。

シグネチャまたは形状

void pgstat_reset_matching_entries(bool (*do_reset) (PgStatShared_HashEntry *, Datum),
                                  Datum match_data, TimestampTz ts)

入力と出力

  • do_reset: 判定用コールバック関数。エントリがリセット対象であれば true を返す。
  • match_data: do_reset に渡される補助データ。
  • ts: リセット時刻（TimestampTz）。
  • 戻り値: なし。

前提条件と不変条件

  • pgStatLocal.shared_hash が初期化されていること。
  • do_reset コールバックは、PgStatShared_HashEntry の内容を読み取る際に共有ロックを必要としない（または内部で適切に処理する）こと。

アルゴリズムまたは振る舞い

1. pgStatLocal.shared_hash に対して dshash_seq_init を呼び出し、全エントリを走査する。 2. 各エントリについて以下を実行する:

  • dropped フラグが立っているエントリはスキップする。
  • do_reset を呼び出し、リセット対象か判定する。
  • 対象であれば、dsa_get_address で共有メモリ上の実体（PgStatShared_Common）を取得する。
  • header->lock を LW_EXCLUSIVE モードで獲得する。
  • shared_stat_reset_contents を呼び出し、統計情報をリセットする。
  • ロックを解放する。

    • 3. 走査終了後、dshash_seq_term でシーケンスを終了する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/activity/pgstat.c:838
  • src/backend/utils/activity/pgstat_shmem.c:1159
  • 呼び出し先:
  • dshash_seq_init, dshash_seq_next, dshash_seq_term
  • dsa_get_address
  • LWLockAcquire, LWLockRelease
  • shared_stat_reset_contents
  • 引数として渡された do_reset 関数

関連データ構造

  • PgStatShared_HashEntry: 共有ハッシュテーブルのエントリ。
  • PgStatShared_Common: 統計情報の共通ヘッダー（ロックを保持）。
  • dshash_seq_status: ハッシュテーブル走査用状態。

失敗モードと境界条件

  • 共有メモリ上のハッシュテーブルが空の場合、何も行わずに終了する。
  • do_reset が false を返すエントリは無視される。
  • 削除済み（dropped）エントリはリセット対象から除外される。

保守担当者向けメモ

  • 本関数は共有メモリ上の統計情報を直接操作するため、LW_EXCLUSIVE ロックによる排他制御が不可欠である。
  • do_reset コールバックの実行コストが全体のパフォーマンスに影響するため、判定ロジックは軽量であるべきである。
