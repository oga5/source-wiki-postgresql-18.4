pg_GSS_error

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-gssapi-common.c:74

目的

GSSAPI認証処理中に発生したエラーを、PostgreSQLのログシステム（ereport）へ統合して出力する。メジャーコードとマイナーコードを人間が判読可能な文字列に変換し、詳細情報として記録する。

シグネチャまたは形状

void pg_GSS_error(const char *errmsg, OM_uint32 maj_stat, OM_uint32 min_stat)

入力と出力

  • **入力**:
  • errmsg: エラーの概要を示す文字列。
  • maj_stat: GSSAPIのメジャー状態コード。
  • min_stat: GSSAPIのメカニズム固有のマイナー状態コード。
  • **出力**: なし（void）。ereport を通じてログ出力を行う。

前提条件と不変条件

  • errmsg は呼び出し元で翻訳済みであること。
  • 内部でスタック上に128バイトの固定長バッファを使用するため、エラーメッセージがこれを超える場合は切り詰められる可能性がある。

アルゴリズムまたは振る舞い

1. pg_GSS_error_int を呼び出し、maj_stat を GSS_C_GSS_CODE として文字列化し msg_major に格納する。 2. pg_GSS_error_int を呼び出し、min_stat を GSS_C_MECH_CODE として文字列化し msg_minor に格納する。 3. COMMERROR レベルで ereport を発行する。

  • errmsg_internal を使用して errmsg を出力。
  • errdetail_internal を使用して、変換されたメジャーおよびマイナーメッセージを詳細情報として出力。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth.c
  • src/backend/libpq/be-gssapi-common.c
  • src/backend/libpq/be-secure-gssapi.c
  • src/interfaces/libpq/fe-auth.c
  • src/interfaces/libpq/fe-gssapi-common.c
  • **呼び出し先**:
  • pg_GSS_error_int
  • ereport (PostgreSQLログ機能)

関連データ構造

  • OM_uint32: GSSAPIステータスコードを保持する型。

失敗モードと境界条件

  • COMMERROR を使用することで、エラー報告自体がGSSAPIエラーを誘発し無限再帰に陥ることを防いでいる。
  • メモリ割り当てを避ける設計のため、メッセージバッファのサイズ（128バイト）が境界条件となる。

保守担当者向けメモ

  • 本関数はバックエンドプロセスモデル（forkベース）を前提としており、マルチスレッド環境での動作は考慮されていない。
  • errmsg には翻訳済みの文字列を渡す必要があるため、呼び出し側で _() マクロ等を使用すること。
