help

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/createdb.c:296

目的

createdb コマンドの利用方法、利用可能なコマンドラインオプション、および接続オプションを標準出力に表示する。

シグネチャまたは形状

static void help(const char *progname)

入力と出力

  • **入力**: progname (実行ファイル名を示す文字列)
  • **出力**: なし（標準出力へのメッセージ出力のみ）

前提条件と不変条件

  • progname は有効な文字列ポインタであること。
  • printf が標準出力に書き込み可能であること。

アルゴリズムまたは振る舞い

1. progname を使用して、コマンドの概要と使用法（Usage）を表示する。 2. データベース作成に関連するオプション（テーブルスペース、エンコーディング、ロケール、所有者、戦略、テンプレート等）の一覧を表示する。 3. サーバー接続に関連するオプション（ホスト、ポート、ユーザー名、パスワード関連等）の一覧を表示する。 4. デフォルトの動作（データベース名の決定ルール）について補足説明を表示する。 5. バグ報告先およびプロジェクトのホームページ URL を表示する。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内では、main 関数からの呼び出しが想定されるが、本ソース excerpt 内では明示的な呼び出し元は特定されていない。
  • **呼び出し先**: 標準ライブラリの printf を使用する。
  • **注意**: ツールが検出した contrib/oid2name/oid2name.c 等の他ファイルにおける help 関数は、同名の別関数であり、本シンボルとは無関係である。

関連データ構造

  • なし

失敗モードと境界条件

  • 標準出力がクローズされている場合、printf の呼び出しは失敗する可能性があるが、本関数内で明示的なエラーハンドリングは行わない。

保守担当者向けメモ

  • ロケール関連のオプション（--locale, --icu-locale 等）を追加・変更した際は、本関数の出力メッセージも同期して更新する必要がある。
  • PACKAGE_BUGREPORT および PACKAGE_NAME などのマクロ定義に依存している。
