format_type_be

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/format_type.c:342

目的

データ型OIDを、人間が読みやすい標準的なSQL形式の文字列に変換します。型修飾子（typmod）を考慮しないデフォルトの型名取得に使用されます。

シグネチャまたは形状

char *format_type_be(Oid type_oid)

入力と出力

  • **入力**: type_oid (変換対象のデータ型OID)
  • **出力**: 変換された型名を含む文字列（pallocによりメモリ確保された領域）。

前提条件と不変条件

  • 内部で format_type_extended を呼び出し、typmod を -1（未指定）、フラグを 0 として処理します。
  • 戻り値の文字列は呼び出し元で解放する必要があります。

アルゴリズムまたは振る舞い

1. format_type_extended(type_oid, -1, 0) を呼び出します。 2. システムカタログ pg_type を参照し、型名を取得します。 3. 必要に応じてスキーマ修飾や引用符付けを行い、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/tablefunc/tablefunc.c
  • contrib/test_decoding/test_decoding.c
  • src/backend/access/brin/brin_validate.c
  • src/backend/access/common/attmap.c
  • **呼び出し先**:
  • format_type_extended (直接の委譲先)

関連データ構造

  • Oid: PostgreSQLのシステムカタログにおけるオブジェクト識別子。

失敗モードと境界条件

  • type_oid が無効な場合、format_type_extended の仕様に従い、エラーが発生するか、あるいは ??? のような代替文字列が返される可能性があります。

保守担当者向けメモ

  • 本関数は format_type_extended の単純なラッパーです。型修飾子を伴う出力が必要な場合は format_type_with_typemod 等の利用を検討してください。
