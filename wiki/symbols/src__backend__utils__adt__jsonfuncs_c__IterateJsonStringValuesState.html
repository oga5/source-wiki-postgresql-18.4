IterateJsonStringValuesState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonfuncs.c:65

目的

JSONデータ内の文字列値を反復処理する際の状態を保持するために使用されます。

シグネチャまたは形状

typedef struct IterateJsonStringValuesState
{
	JsonLexContext *lex;
	JsonIterateStringValuesAction action;
	void	   *action_state;
	uint32		flags;
} IterateJsonStringValuesState;

入力と出力

  • **入力**:
  • lex: JSON解析用のコンテキスト。
  • action: 各JSON値に対して適用されるコールバック関数。
  • action_state: 反復処理中に保持する任意のコンテキストデータ。
  • flags: 反復対象とするJSON要素の種類を指定するビットフラグ。
  • **出力**: なし（構造体として状態を保持）。

前提条件と不変条件

  • lex は有効な JsonLexContext を指している必要があります。
  • action は反復処理の各ステップで呼び出される関数ポインタです。

アルゴリズムまたは振る舞い

JSONドキュメントを走査し、flags で指定された条件に合致する要素に対して action を実行します。反復処理の進捗や中間結果は action_state に格納されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonfuncs.c 内の関数（行5737, 5759, 5787付近）でインスタンス化およびキャストが行われています。
  • **呼び出し先**: action フィールドに設定された関数がコールバックとして呼び出されます。

関連データ構造

  • JsonLexContext: JSON解析の基盤となる構造体。
  • JsonIterateStringValuesAction: 反復処理時に実行されるアクションの型定義。

失敗モードと境界条件

  • lex が初期化されていない場合や、不正なJSONデータが渡された場合の挙動は、解析器（JsonLexContext）の仕様に依存します。

保守担当者向けメモ

  • 本構造体は jsonfuncs.c 内の特定の反復処理ロジックに特化しています。
  • flags の具体的な値や action のシグネチャについては、src/include/utils/jsonfuncs.h の定義を参照してください。
