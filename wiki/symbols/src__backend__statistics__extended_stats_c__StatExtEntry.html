StatExtEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/statistics/extended_stats.c:64

目的

CREATE STATISTICS コマンドによって作成された統計オブジェクトの定義情報を保持し、ANALYZE 処理やプランナによる選択率推定時に、対象となる列、式、統計の種類、および精度設定を管理するために使用されます。

シグネチャまたは形状

typedef struct StatExtEntry
{
	Oid			statOid;		/* pg_statistic_ext エントリのOID */
	char	   *schema;			/* 統計オブジェクトのスキーマ名 */
	char	   *name;			/* 統計オブジェクト名 */
	Bitmapset  *columns;		/* 対象となる属性番号の集合 */
	List	   *types;			/* 有効な統計種別のリスト ('char') */
	int			stattarget;		/* 統計ターゲット (-1 はデフォルト) */
	List	   *exprs;			/* 統計対象の式リスト */
} StatExtEntry;

入力と出力

  • 本構造体は、fetch_statentries_for_relation 関数によってカタログから読み取られ、リストとして返されます。
  • make_build_data 関数などの統計構築プロセスにおいて、統計計算のパラメータとして入力されます。

前提条件と不変条件

  • stattarget が -1 の場合、システムデフォルトの統計ターゲットが適用されます。
  • columns は Bitmapset であり、対象テーブルの属性番号を保持します。

アルゴリズムまたは振る舞い

  • 統計オブジェクトのメタデータ（名前、スキーマ、OID）と、計算に必要な構成要素（列、式、ターゲット）をメモリ上に集約します。
  • ANALYZE 実行時に、この構造体に基づいて各統計種別（MCV, 依存関係, 多変量NDistinct）の計算関数が呼び出されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/extended_stats.c 内の make_build_data
  • fetch_statentries_for_relation (統計オブジェクトリストの生成)
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありませんが、make_build_data 等の関数が本構造体のメンバを参照して計算処理を行います。

関連データ構造

  • pg_statistic_ext: カタログ上の統計オブジェクト定義。
  • Bitmapset: columns メンバで使用。
  • List: types および exprs メンバで使用。

失敗モードと境界条件

  • 構造体内のポインタ（schema, name）やリストが適切に初期化されていない場合、メモリ参照エラーが発生する可能性があります。
  • palloc0 を用いてメモリ確保が行われるため、メモリ不足時にはエラーが発生します。

保守担当者向けメモ

  • 新しい統計種別を追加する場合は、types リストの処理ロジックを更新する必要があります。
  • exprs に含まれる式は ExecPrepareExpr を介して評価されるため、式評価の変更時には本構造体の利用箇所を確認してください。
