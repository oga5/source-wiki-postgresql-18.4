getBaseTypeAndTypmod

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/lsyscache.c:2715

目的

ドメイン型を再帰的に辿り、その基底となる型（非ドメイン型）のOIDと、対応する型修飾子（typmod）を取得します。

シグネチャまたは形状

Oid getBaseTypeAndTypmod(Oid typid, int32 *typmod)

入力と出力

  • **入力**:
  • typid: 調査対象の型のOID。
  • typmod: 型修飾子へのポインタ。初期値として -1 が期待されます。
  • **出力**:
  • 戻り値: ドメインの連鎖を辿り着いた先の基底型のOID。
  • *typmod: 最終的な基底型に対応する型修飾子で更新されます。

前提条件と不変条件

  • *typmod は呼び出し時に -1 である必要があります（Assertによる検証あり）。
  • 指定された typid がシステムカタログ pg_type に存在しない場合、エラーが発生します。

アルゴリズムまたは振る舞い

1. TYPEOID システムキャッシュを使用して、現在の typid に対応する pg_type 行を検索します。 2. 取得した行の typtype を確認します。

  • TYPTYPE_DOMAIN であれば、そのドメインの基底型（typbasetype）と型修飾子（typtypmod）を読み取り、ループを継続します。
  • ドメインでなければ、ループを終了します。

    • 3. ループの各ステップで ReleaseSysCache を呼び出し、参照カウントを適切に解放します。 4. 最終的な基底型のOIDを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • printtup.c, tablecmds.c, preptlist.c, parse_coerce.c, parse_expr.c, parse_node.c, parse_target.c, jsonfuncs.c など、型変換や型情報の解決を行う複数のモジュールから呼び出されています。
  • **呼び出し先**:
  • SearchSysCache1: システムカタログの検索。
  • ReleaseSysCache: キャッシュの解放。
  • GETSTRUCT: タプルから構造体へのキャスト。

関連データ構造

  • Form_pg_type: pg_type システムカタログの行構造体。
  • HeapTuple: システムキャッシュから取得されるタプル。

失敗モードと境界条件

  • SearchSysCache1 が有効なタプルを返さない場合、elog(ERROR) により処理が中断されます。
  • ドメインの連鎖が深い場合でも、ループ構造により正しく終端まで到達します。

保守担当者向けメモ

  • 本関数はドメインの解決に特化しています。型に関するより複雑なキャッシュ情報が必要な場合は typcache.c の利用を検討してください。
  • typmod のポインタ渡しによる更新は、呼び出し元で変数が正しく初期化されていることに依存しているため注意してください。
