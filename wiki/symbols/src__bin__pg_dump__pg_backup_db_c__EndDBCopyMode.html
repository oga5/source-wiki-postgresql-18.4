EndDBCopyMode

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_db.c:438

目的

データベース接続における COPY モードを終了し、libpq の状態をアイドル状態へ復帰させる。

シグネチャまたは形状

void EndDBCopyMode(Archive *AHX, const char *tocEntryTag)

入力と出力

  • **入力**:
  • AHX: Archive 型へのポインタ（内部で ArchiveHandle にキャストされる）。
  • tocEntryTag: COPY 失敗時にエラーメッセージへ出力されるテーブル名等の識別子。
  • **出力**: なし（副作用としてデータベース接続状態を更新）。

前提条件と不変条件

  • ArchiveHandle の pgCopyIn フラグが true である場合にのみ、COPY 終了処理が実行される。
  • 呼び出し後、AH->pgCopyIn は false に設定される。

アルゴリズムまたは振る舞い

1. AH->pgCopyIn が真の場合、PQputCopyEnd を呼び出し、サーバーへ COPY 終了を通知する。 2. PQgetResult を呼び出し、コマンド実行結果が PGRES_COMMAND_OK であることを確認する。失敗した場合は warn_or_exit_horribly を通じてエラーを報告し終了する。 3. PQclear で結果オブジェクトを解放する。 4. PQgetResult を再度呼び出し、libpq がアイドル状態に戻ったことを確認する。予期せぬ結果が残っている場合は警告を出力する。 5. AH->pgCopyIn を false に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_archiver.c:1050
  • **呼び出し先**: libpq 関数群 (PQputCopyEnd, PQgetResult, PQresultStatus, PQerrorMessage, PQclear)

関連データ構造

  • ArchiveHandle (キャスト元: Archive): データベース接続情報および pgCopyIn フラグを保持する。

失敗モードと境界条件

  • PQputCopyEnd が 0 以下の値を返した場合、pg_fatal によりプロセスが終了する。
  • COPY 終了後の PQgetResult が PGRES_COMMAND_OK 以外を返した場合、warn_or_exit_horribly によりプロセスが終了する。
  • COPY 終了後に予期せぬ結果が libpq に残っている場合、警告ログが出力される。

保守担当者向けメモ

  • 本関数は ArchiveHandle の内部状態を直接操作するため、Archive 構造体の定義変更時には注意が必要である。
  • libpq のステートマシンをアイドル状態に戻すための重要な処理であり、COPY 操作を伴うバックアップ・リストア処理の整合性に直結する。
