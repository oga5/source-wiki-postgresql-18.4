SharedFileSetInit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/sharedfileset.c:37

目的

共有一時ファイルセットの初期化を行い、参照カウンタの設定およびDSM（動的共有メモリ）デタッチ時の自動クリーンアップ処理を登録します。

シグネチャまたは形状

void SharedFileSetInit(SharedFileSet *fileset, dsm_segment *seg)

入力と出力

  • **入力**:
  • fileset: 初期化対象となる SharedFileSet 構造体へのポインタ。
  • seg: 関連付ける dsm_segment へのポインタ（NULL の場合、デタッチコールバックは登録されません）。
  • **出力**: なし。

前提条件と不変条件

  • fileset は呼び出し元によってメモリが確保されている必要があります。
  • 参照カウンタ（refcnt）は初期値として 1 に設定されます。
  • fileset->mutex はスピンロックとして初期化され、以降の参照カウンタ操作を保護します。

アルゴリズムまたは振る舞い

1. SpinLockInit を呼び出し、fileset->mutex を初期化します。 2. fileset->refcnt を 1 に設定します。 3. FileSetInit を呼び出し、内部の FileSet 構造体を初期化します。 4. seg が指定されている場合、on_dsm_detach を使用して、DSM デタッチ時に SharedFileSetOnDetach が実行されるようコールバックを登録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeHashjoin.c
  • src/backend/utils/sort/tuplesort.c
  • **呼び出し先**:
  • SpinLockInit
  • FileSetInit
  • on_dsm_detach
  • SharedFileSetOnDetach（コールバックとして登録）

関連データ構造

  • SharedFileSet: 共有ファイルセットを管理する構造体。
  • dsm_segment: 共有メモリセグメント。
  • FileSet: SharedFileSet が内部で利用するファイルセット管理構造体。

失敗モードと境界条件

  • seg が NULL の場合、DSM デタッチ時の自動クリーンアップは登録されません。この場合、呼び出し元が手動でクリーンアップを管理する必要があります。

保守担当者向けメモ

  • 本関数は SharedFileSet のライフサイクル管理の起点となります。
  • SharedFileSetOnDetach はエラー発生時のクリーンアップパスでも呼び出されるため、コールバック内でエラーを発生させないよう注意してください。
