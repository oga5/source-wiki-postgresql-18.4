generic_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/genericdesc.c:23

目的

汎用WAL（Write-Ahead Logging）レコードのデータ部を走査し、ページ内の変更箇所（オフセットと長さ）を人間が読める形式の文字列として出力バッファに書き出す。

シグネチャまたは形状

void generic_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 出力文字列を格納する StringInfo 構造体へのポインタ。
  • record: 解析対象のWALレコードを保持する XLogReaderState 構造体へのポインタ。
  • **出力**: なし（buf の内容が更新される）。

前提条件と不変条件

  • record は有効なWALレコードを指していること。
  • WALレコードのデータ部は、offset（OffsetNumber）と length（OffsetNumber）のペアが連続し、その後に実際の変更データが続く構造であること。

アルゴリズムまたは振る舞い

1. XLogRecGetData を用いてレコードのデータ領域の開始位置と終了位置を取得する。 2. データ領域をポインタで走査し、以下の手順を繰り返す。

  • offset と length を読み取り、ポインタをそれぞれ sizeof(offset) および sizeof(length) 分進める。
  • 読み取った length 分だけポインタをスキップする（実際の変更データ自体は出力しない）。
  • 読み取った offset と length を buf に追記する。
  • 最後の要素でない場合は、区切り文字として "; " を付与する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h で定義されているリソースマネージャテーブル（RM_GENERIC_ID）から参照される。
  • **呼び出し先**:
  • XLogRecGetData: レコードデータ取得用。
  • XLogRecGetDataLen: データ長取得用。
  • appendStringInfo: 文字列バッファへの書き込み用。
  • memcpy: メモリからの値コピー用。

関連データ構造

  • StringInfo: 文字列操作用バッファ。
  • XLogReaderState: WALレコード読み取り状態。
  • OffsetNumber: PostgreSQLのページ内オフセット型。

失敗モードと境界条件

  • データ領域の末尾に達した際、最後の要素には "; " を付与しない制御を行っている。
  • レコードデータが不正な形式（offset/length の読み取り中に end を超えるなど）である場合の明示的なエラーハンドリングはソースに含まれていない。

保守担当者向けメモ

  • 本関数は汎用WALレコードの構造をダンプするのみであり、非常に単純な実装である。
  • 変更データの内容自体はダンプせず、メタデータ（オフセットと長さ）のみを対象としている点に注意すること。
