rbound_bsearch

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/multirangetypes_selfuncs.c:738

目的

レンジ境界値のヒストグラム配列に対して二分探索を行い、指定された値（value）が該当する位置（インデックス）を特定します。

シグネチャまたは形状

static int rbound_bsearch(TypeCacheEntry *typcache, const RangeBound *value, const RangeBound *hist, int hist_length, bool equal)

入力と出力

  • **入力**:
  • typcache: 型情報のキャッシュ。
  • value: 検索対象の RangeBound。
  • hist: 比較対象となる RangeBound のヒストグラム配列。
  • hist_length: ヒストグラムの要素数。
  • equal: 比較において等価な値を含めるかどうかを指定するフラグ。
  • **出力**:
  • 検索結果のインデックス（int）。該当する範囲の末尾位置を返します。

前提条件と不変条件

  • hist 配列は昇順にソートされている必要があります。
  • typcache は有効な型情報を含んでいる必要があります。

アルゴリズムまたは振る舞い

1. 探索範囲を lower = -1 から upper = hist_length - 1 に初期化します。 2. lower < upper である間、中央値 middle を計算し、range_cmp_bounds を用いて hist[middle] と value を比較します。 3. 比較結果が cmp < 0（または equal が真かつ cmp == 0）の場合、lower を middle に更新します。 4. それ以外の場合、upper を middle - 1 に更新します。 5. 最終的な lower の値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/multirangetypes_selfuncs.c 内の複数箇所（統計計算ロジック）。
  • src/backend/utils/adt/rangetypes_selfuncs.c 内の複数箇所。
  • **呼び出し先**:
  • range_cmp_bounds（境界値の比較関数）。

関連データ構造

  • RangeBound: 範囲型の境界値を保持する構造体。
  • TypeCacheEntry: 型の比較関数等を含むキャッシュ構造体。

失敗モードと境界条件

  • hist_length が 0 の場合、lower は初期値の -1 を返します。
  • 探索対象がヒストグラムの全要素よりも小さい場合、-1 が返されます。
