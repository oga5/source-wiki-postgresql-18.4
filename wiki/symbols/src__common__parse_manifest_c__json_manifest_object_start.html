json_manifest_object_start

定義

  • 種別: function
  • 定義箇所: src/common/parse_manifest.c:275

目的

バックアップマニフェストのJSON解析において、オブジェクトの開始を検知し、現在の解析状態（JsonManifestParseState）に基づいて後続のフィールド解析に向けた状態遷移を行う。

シグネチャまたは形状

static JsonParseErrorType json_manifest_object_start(void *state);

入力と出力

  • **入力**: void *state （実際には JsonManifestParseState* 型のポインタ）
  • **出力**: JsonParseErrorType 型の戻り値。成功時は JSON_SUCCESS を返す。

前提条件と不変条件

  • 引数 state は JsonManifestParseState 構造体として初期化されている必要がある。
  • parse->state が定義された状態遷移（JM_EXPECT_TOPLEVEL_START 等）のいずれかにあること。

アルゴリズムまたは振る舞い

1. state を JsonManifestParseState としてキャストする。 2. 現在の parse->state に応じて以下の遷移を行う：

  • JM_EXPECT_TOPLEVEL_START: JM_EXPECT_TOPLEVEL_FIELD へ遷移。
  • JM_EXPECT_FILES_NEXT: JM_EXPECT_THIS_FILE_FIELD へ遷移し、ファイル関連のフィールドポインタ（pathname, size 等）を NULL にリセットする。
  • JM_EXPECT_WAL_RANGES_NEXT: JM_EXPECT_THIS_WAL_RANGE_FIELD へ遷移し、WAL関連のフィールドポインタ（timeline, start_lsn 等）を NULL にリセットする。

    • 3. 上記以外の状態の場合は json_manifest_parse_failure を呼び出し、エラーを通知する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/parse_manifest.c 内の解析初期化処理（json_parse_manifest 等）において、JsonManifestParseContext のコールバックとして登録され、JSONパーサーから呼び出される。
  • **呼び出し先**: json_manifest_parse_failure（エラー発生時）。

関連データ構造

  • JsonManifestParseState: 解析の進捗と現在のコンテキストを保持する構造体。

失敗モードと境界条件

  • 予期しないタイミングでオブジェクトが開始された場合、json_manifest_parse_failure が呼び出され、解析が中断される。

保守担当者向けメモ

  • 本関数は JsonManifestParseState の状態マシンを制御する重要な役割を担う。新しいマニフェストフィールドを追加する場合は、状態遷移のケースを適切に追加する必要がある。
