DecodeCommit

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/decode.c:666

目的

WALのコミットレコードを解析し、論理デコーディングコンテキストにおいてトランザクションの確定処理（reorderbuffer への反映およびスナップショットの更新）を行う。

シグネチャまたは形状

static void DecodeCommit(LogicalDecodingContext *ctx, XLogRecordBuffer *buf,
                         xl_xact_parsed_commit *parsed, TransactionId xid,
                         bool two_phase)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングの現在のコンテキスト。
  • buf: 解析対象のWALレコードバッファ。
  • parsed: コミットレコードの解析済みデータ構造。
  • xid: コミット対象のトランザクションID。
  • two_phase: 2フェーズコミット（2PC）かどうかを示すフラグ。
  • **出力**: なし（副作用として reorderbuffer および snapshot_builder を更新）。

前提条件と不変条件

  • parsed は有効なコミットレコード情報を含んでいること。
  • ctx->snapshot_builder および ctx->reorder が初期化されていること。

アルゴリズムまたは振る舞い

1. **メタデータ抽出**: レコードからレプリケーションオリジンのIDとLSN、コミット時刻を取得する。 2. **スナップショット更新**: SnapBuildCommitTxn を呼び出し、スナップショットの状態を更新する。 3. **フィルタリング判定**: DecodeTXNNeedSkip を実行。対象外のデータベースやオリジンの場合は、ReorderBufferForget を用いて当該トランザクションおよびサブトランザクションの情報を破棄し、早期リターンする。 4. **サブトランザクション登録**: 生存しているサブトランザクションを ReorderBufferCommitChild で登録する。 5. **コミット処理**:

  • two_phase が真の場合: ReorderBufferFinishPrepared を呼び出し、準備済みトランザクションを完了させる。
  • それ以外の場合: ReorderBufferCommit を呼び出し、トランザクションを確定させる。

    • 6. **統計更新**: UpdateDecodingStats を呼び出し、デコーディング統計を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/decode.c 内のレコード処理ループ（LogicalDecodingProcessRecord からのディスパッチ経由）。
  • **呼び出し先**:
  • SnapBuildCommitTxn
  • DecodeTXNNeedSkip
  • ReorderBufferForget
  • ReorderBufferCommitChild
  • ReorderBufferFinishPrepared
  • ReorderBufferCommit
  • UpdateDecodingStats

関連データ構造

  • LogicalDecodingContext: デコーディングの全体状態。
  • XLogRecordBuffer: WALレコードの保持構造。
  • xl_xact_parsed_commit: コミットレコードの解析結果。
  • ReorderBuffer: 変更を蓄積するバッファ。

失敗モードと境界条件

  • **スキップ処理**: DecodeTXNNeedSkip が真を返す場合、トランザクションの内容は破棄される。この際、共有カタログの無効化が必要なケースでは注意が必要（ソースコードのコメント参照）。
  • **2PC**: two_phase フラグにより、通常のコミットと準備済みトランザクションの完了処理が分岐する。

保守担当者向けメモ

  • 本関数は DecodePrepare と類似のロジックを共有している。
  • ReorderBufferForget を呼び出す際は、トランザクションの無効化処理が適切に行われるか確認すること。
  • 統計情報の更新は、デコーディングの中断時に情報を失わないよう、コミット等のタイミングで明示的に行われている。
