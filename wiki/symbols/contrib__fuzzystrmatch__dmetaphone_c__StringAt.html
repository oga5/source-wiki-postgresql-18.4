StringAt

定義

  • 種別: function
  • 定義箇所: contrib/fuzzystrmatch/dmetaphone.c:347

目的

metastring 内の指定位置から始まる文字列が、可変長引数として渡された複数の文字列パターンのいずれかと一致するかを判定する。

シグネチャまたは形状

int StringAt(metastring *s, int start, int length, ...)

入力と出力

  • **入力**:
  • s: 走査対象の metastring 構造体。
  • start: 比較を開始するインデックス。
  • length: 比較する文字列の長さ。
  • ...: 比較対象となる文字列のリスト（末尾は空文字列 "" で終端する必要がある）。
  • **出力**:
  • 1: いずれかのパターンと一致した場合。
  • 0: 一致しなかった場合、または start が範囲外の場合。

前提条件と不変条件

  • s->str は有効なヌル終端文字列であること。
  • 可変長引数のリストは、必ず空文字列 "" で終了していること。

アルゴリズムまたは振る舞い

1. start が 0 未満、または s->length 以上であれば直ちに 0 を返す。 2. s->str の start 位置を起点とするポインタを取得する。 3. 可変長引数を順次取得し、strncmp を用いて length 分の文字列比較を行う。 4. 一致するパターンが見つかった場合は 1 を返し、空文字列 "" に到達した場合は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: DoubleMetaphone 関数内（contrib/fuzzystrmatch/dmetaphone.c）の各所。
  • 例: GN, KN, PN, WR, PS の先頭一致判定や、ACH, BACHER, CAESAR 等の特定の綴りパターン判定に使用。
  • **呼び出し先**: 標準ライブラリ関数 strncmp, strcmp, va_start, va_arg, va_end。

関連データ構造

  • metastring: 文字列バッファと現在の長さを保持する構造体。

失敗モードと境界条件

  • start が文字列の範囲外である場合、安全に 0 を返す。
  • 可変長引数の末尾に "" がない場合、va_arg が不正なメモリ領域を参照する可能性がある。

保守担当者向けメモ

  • 本関数は DoubleMetaphone アルゴリズムにおける複雑な綴りルールを簡潔に記述するために多用されている。
  • パフォーマンス向上のため、頻繁に呼び出される箇所では比較対象の文字列長が length と一致しているか確認すること。
