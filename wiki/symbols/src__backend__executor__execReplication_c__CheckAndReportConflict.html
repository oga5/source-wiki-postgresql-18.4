CheckAndReportConflict

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execReplication.c:521

目的

論理レプリケーションの適用処理中に発生した競合を検出し、詳細情報を収集してエラーを報告する。

シグネチャまたは形状

static void CheckAndReportConflict(ResultRelInfo *resultRelInfo, EState *estate,
                                   ConflictType type, List *recheckIndexes,
                                   TupleTableSlot *searchslot, TupleTableSlot *remoteslot)

入力と出力

  • **入力**:
  • resultRelInfo: 対象リレーションの情報。
  • estate: 実行状態。
  • type: 発生した競合の型（ConflictType）。
  • recheckIndexes: 競合チェック対象のインデックスリスト。
  • searchslot, remoteslot: 競合確認に使用するタプルスロット。
  • **出力**: なし（競合が検出された場合、ReportApplyConflict を経由して ERROR を発生させる）。

前提条件と不変条件

  • resultRelInfo->ri_onConflictArbiterIndexes に定義されたユニークインデックスが競合チェックの対象となる。
  • recheckIndexes に含まれるインデックスのみがチェック対象として考慮される。

アルゴリズムまたは振る舞い

1. resultRelInfo->ri_onConflictArbiterIndexes を反復し、recheckIndexes に含まれるインデックスに対して FindConflictTuple を呼び出す。 2. 競合タプルが見つかった場合、ConflictTupleInfo 構造体を割り当て、対象タプルのトランザクション情報（xmin, origin, ts）を GetTupleTransactionInfo で取得してリストに格納する。 3. 競合タプルが一つでも存在する場合、ReportApplyConflict を呼び出す。

  • 競合が複数ある場合は CT_MULTIPLE_UNIQUE_CONFLICTS として報告する。
  • 報告時のログレベルは ERROR である。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execReplication.c 内のレプリケーション適用ロジック（INSERT/UPDATE実行時）。
  • **呼び出し先**:
  • FindConflictTuple
  • GetTupleTransactionInfo
  • ReportApplyConflict
  • palloc0_object
  • lappend

関連データ構造

  • ResultRelInfo: リレーションの実行時情報。
  • ConflictTupleInfo: 競合したタプルの詳細情報（スロット、インデックスOID、トランザクション情報）。
  • ConflictType: 競合の種類を定義する列挙型。

失敗モードと境界条件

  • 競合が検出されない場合、何も報告せず正常終了する。
  • 複数のユニークインデックスで競合が発生した場合、ReportApplyConflict に CT_MULTIPLE_UNIQUE_CONFLICTS が渡される。

保守担当者向けメモ

  • 本関数は ERROR を発生させるため、呼び出し側はトランザクションのロールバックやエラーハンドリングが適切に行われるコンテキストであることを確認すること。
  • recheckIndexes のフィルタリングにより、不要なインデックススキャンを回避している点に注意。
