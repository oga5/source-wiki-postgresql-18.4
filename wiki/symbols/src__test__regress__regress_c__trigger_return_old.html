trigger_return_old

定義

  • 種別: function
  • 定義箇所: src/test/regress/regress.c:257

目的

PostgreSQLのトリガー関数として動作し、トリガー起動時に渡された元のタプル（tg_trigtuple）をそのまま呼び出し元へ返却するテスト用関数です。

シグネチャまたは形状

Datum trigger_return_old(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（fcinfo を介してトリガー実行コンテキストを受け取る）。
  • **出力**: Datum 型（トリガーが処理対象としている元のタプル tg_trigtuple へのポインタ）。

前提条件と不変条件

  • 本関数はトリガーマネージャから呼び出されることを前提としています。
  • CALLED_AS_TRIGGER(fcinfo) マクロによる検証が必須であり、トリガー以外からの呼び出しはエラーとなります。

アルゴリズムまたは振る舞い

1. fcinfo->context を TriggerData 構造体にキャストし、トリガー情報を取得します。 2. CALLED_AS_TRIGGER(fcinfo) を使用して、実行コンテキストがトリガーによるものかを確認します。トリガーでない場合は elog(ERROR) を発生させます。 3. trigdata->tg_trigtuple から元のタプルを取得します。 4. 取得したタプルを PointerGetDatum で変換し、戻り値として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL の CREATE TRIGGER 文を通じてトリガーマネージャから呼び出されます。
  • **呼び出し先**:
  • CALLED_AS_TRIGGER マクロ（検証用）
  • PointerGetDatum マクロ（戻り値変換用）
  • elog（エラー発生時）

関連データ構造

  • TriggerData: トリガー実行時のコンテキスト情報を保持する構造体。
  • HeapTuple: PostgreSQL のヒープタプル構造体。

失敗モードと境界条件

  • **トリガー以外からの呼び出し**: CALLED_AS_TRIGGER が偽を返す場合、elog(ERROR) によりトランザクションが中断されます。

保守担当者向けメモ

  • 本関数は回帰テスト専用です。本番環境のトリガーロジックには使用しないでください。
  • tg_trigtuple を直接返却するため、トリガーの変更操作を行わずに元の状態を維持したいテストケースで使用されます。
