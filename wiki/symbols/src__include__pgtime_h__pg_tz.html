pg_tz

定義

  • 種別: struct
  • 定義箇所: src/include/pgtime.h:50

目的

タイムゾーン情報を保持するための不透明な（opaque）構造体です。PostgreSQL における時刻計算やタイムゾーン変換の基盤として使用されます。

シグネチャまたは形状

typedef struct pg_tz pg_tz;

内部構造は公開されておらず、ポインタ経由で操作されます。

入力と出力

  • 入力: pg_tzset() 等の API を介して、タイムゾーン名（文字列）から本構造体へのポインタが取得されます。
  • 出力: DetermineTimeZoneOffsetInternal() 等の時刻変換関数へ引数として渡され、オフセット計算等に利用されます。

前提条件と不変条件

  • 本構造体の内部メンバに直接アクセスすることは禁止されています。操作には必ず提供されている API 関数を経由する必要があります。
  • 使用前に pg_timezone_initialize() による環境のセットアップが必要です。

アルゴリズムまたは振る舞い

  • タイムゾーンのロード、列挙、およびオフセット計算のインターフェースを提供します。
  • session_timezone や log_timezone といったグローバルなタイムゾーン設定の保持に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/commands/variable.c: セッションおよびログのタイムゾーン設定管理。
  • src/backend/utils/adt/date.c: 日付関連の処理。
  • src/backend/utils/adt/datetime.c: 時刻変換およびオフセット計算。
  • 呼び出し先:
  • 内部実装は src/timezone/pgtz.c に存在しますが、本ヘッダーファイル内での呼び出し先関係は定義されていません。

関連データ構造

  • pg_time_t: 64 ビット時刻型。
  • pg_tm: PostgreSQL 固有の構造化時刻表現。
  • TimeZoneAbbrevTable: タイムゾーン短縮形テーブル。

失敗モードと境界条件

  • 構造体が不透明であるため、無効なポインタや初期化されていないポインタを API に渡した場合の挙動は、各操作関数の実装に依存します。

保守担当者向けメモ

  • 本構造体の定義は src/include/pgtime.h にありますが、実体は src/timezone/ 配下の実装に隠蔽されています。
  • タイムゾーンの操作ロジックを変更する場合は、src/timezone/pgtz.c を参照してください。
