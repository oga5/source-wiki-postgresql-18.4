uniquePos

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector.c:51

目的

tsvector 内の語彙に関連付けられた位置情報（WordEntryPos）の配列をソートし、重複を排除して正規化する。

シグネチャまたは形状

int uniquePos(WordEntryPos *a, int l)

入力と出力

  • **入力**:
  • a: WordEntryPos 型の配列（位置情報のリスト）。
  • l: 配列 a の要素数。
  • **出力**:
  • 重複排除後の有効な要素数（int）。配列 a はインプレースで書き換えられる。

前提条件と不変条件

  • 入力配列 a は WordEntryPos 型の要素で構成される。
  • 処理後、配列内の位置情報は昇順にソートされ、同一位置を持つ要素は重みが大きいものに統合される。
  • MAXNUMPOS（最大位置数）および MAXENTRYPOS（最大位置値）の制約を超えない範囲で処理される。

アルゴリズムまたは振る舞い

1. 要素数が1以下の場合はそのまま返す。 2. qsort と compareWordEntryPos を使用して、配列 a を位置情報の昇順でソートする。 3. ソート済み配列を走査し、隣接する要素の位置（WEP_GETPOS）を比較する。

  • 位置が異なる場合: 新しい要素として保持する。
  • 位置が同じ場合: WEP_GETWEIGHT を比較し、より大きい重みを保持する（WEP_SETWEIGHT）。

    • 4. 最終的な有効要素数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsvector.c 内の tsvectorin 等の処理において、WordEntry の位置情報リストを整理する際に呼び出される。
  • **呼び出し先**:
  • qsort（標準ライブラリ）
  • compareWordEntryPos（比較関数）

関連データ構造

  • WordEntryPos: 位置と重みを保持するデータ構造。
  • WordEntry: tsvector 内の語彙エントリ。

失敗モードと境界条件

  • l <= 1: 処理を行わず即座に終了する。
  • MAXNUMPOS または MAXENTRYPOS の上限に達した場合: 処理を中断し、その時点までの結果を返す。

保守担当者向けメモ

  • 本関数は tsvector の内部バイナリ表現の整合性を維持するために不可欠である。
  • 重みの統合ロジック（WEP_SETWEIGHT）は、tsearch/ts_type.h で定義されたビット操作マクロに依存しているため、構造体の変更時には注意が必要である。
