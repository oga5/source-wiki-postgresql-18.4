btmarkpos

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtree.c:505

目的

現在のB-treeインデックススキャン位置を保存し、後で btrestrpos によってその位置へ復帰できるようにする。

シグネチャまたは形状

void btmarkpos(IndexScanDesc scan)

入力と出力

  • **入力**: IndexScanDesc scan - 現在実行中のインデックススキャンの記述子。
  • **出力**: なし（副作用として scan->opaque 内のマーク用状態が更新される）。

前提条件と不変条件

  • scan->opaque が BTScanOpaque 型としてキャスト可能であること。
  • 呼び出し時に so->markPos が保持している古いピンは、この関数内で解放される。

アルゴリズムまたは振る舞い

1. BTScanPosUnpinIfPinned を呼び出し、既存のマーク位置に関連付けられたピンを解放する。 2. 現在のスキャン位置 so->currPos が有効かを確認する。

  • 有効な場合: so->markItemIndex に現在の itemIndex を記録する。
  • 無効な場合: so->markPos を無効化し、so->markItemIndex を -1 に設定する。

    • 3. ページを跨ぐ移動が発生した場合、_bt_steppage が markPos 構造体全体をコピーする遅延評価の仕組みをとる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtree.c 内の IndexAmRoutine 構造体（ammarkpos フィールド）を通じて呼び出される。
  • **呼び出し先**:
  • BTScanPosUnpinIfPinned
  • BTScanPosIsValid
  • BTScanPosInvalidate
  • _bt_steppage（条件付きで呼び出される可能性がある）

関連データ構造

  • IndexScanDesc: インデックススキャンの共通記述子。
  • BTScanOpaque: B-tree固有のスキャン状態（markPos, currPos, markItemIndex を保持）。

失敗モードと境界条件

  • currPos が無効な状態（スキャンが終了している、または開始前）で呼び出された場合、マーク位置は無効として扱われる。
  • ページを跨ぐ移動が発生する前にマーク位置が変更された場合、markPos への完全なコピーは行われない（最適化）。

保守担当者向けメモ

  • 本関数は IndexAmRoutine の一部として登録されている。
  • markPos への完全なコピーは _bt_steppage に依存しているため、スキャン位置の管理ロジックを変更する際は nbtsearch.c の関連処理と整合性を取る必要がある。
