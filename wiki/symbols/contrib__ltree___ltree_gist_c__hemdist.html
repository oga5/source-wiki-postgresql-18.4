hemdist

定義

  • 種別: function
  • 定義箇所: contrib/ltree/_ltree_gist.c:202

目的

2つの ltree_gist 構造体間のハミング距離（またはそれに準ずる差異）を計算し、GiST インデックスにおけるペナルティ計算や分割コスト評価の指標を提供します。

シグネチャまたは形状

static int hemdist(ltree_gist *a, ltree_gist *b, int siglen)

入力と出力

  • **入力**:
  • a, b: 比較対象の ltree_gist 構造体へのポインタ。
  • siglen: シグネチャの長さ（バイト単位）。
  • **出力**:
  • 2つの構造体間の距離を表す整数値を返します。

前提条件と不変条件

  • a および b は有効な ltree_gist 構造体である必要があります。
  • siglen はシグネチャのサイズと整合している必要があります。

アルゴリズムまたは振る舞い

1. a が「すべて真（ALLTRUE）」の状態である場合:

  • b も「すべて真」であれば 0 を返します。
  • それ以外の場合、シグネチャの全ビット数から b のシグネチャに含まれるビット数を引いた値を返します。

    • 2. b が「すべて真」である場合:

  • シグネチャの全ビット数から a のシグネチャに含まれるビット数を引いた値を返します。

    • 3. 上記以外の場合:

  • hemdistsign を呼び出し、両者のシグネチャ間のハミング距離を計算して返します。

呼び出し元と呼び出し先

  • **呼び出し先**: sizebitvec（内部ヘルパー）、hemdistsign。
  • **呼び出し元**: ツールによる解析では、contrib/hstore/hstore_gist.c および contrib/intarray/_intbig_gist.c 内の関数から、同名の関数が呼び出されていることが示唆されています。ただし、これらは ltree モジュール外のファイルであるため、本関数が直接呼び出されているか、あるいは各モジュールで同名の静的関数が定義されているかの確認が必要です。

関連データ構造

  • ltree_gist: GiST インデックスで使用されるシグネチャ構造体。

失敗モードと境界条件

  • LTG_ISALLTRUE マクロによる判定が優先され、特殊なケース（全ビットが立っている状態）が個別に処理されます。
  • siglen が不適切な場合、sizebitvec や hemdistsign 内部で予期せぬ動作をする可能性があります。

保守担当者向けメモ

  • 本関数は static で定義されており、contrib/ltree/_ltree_gist.c 内でのみ使用されることが意図されています。
  • ツールが指摘する hstore や intarray からの呼び出しは、同名の関数が各モジュールで個別に定義されている可能性が高いため、混同しないよう注意してください。
