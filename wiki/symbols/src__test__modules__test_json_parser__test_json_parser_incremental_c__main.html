目的

PostgreSQL のインクリメンタル JSON パーサーの堅牢性を検証するためのテストプログラムのエントリポイント。指定されたファイルをチャンク単位で読み込み、パーサーへ逐次供給することで、断片的なデータ入力や境界条件の処理をテストする。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**:
  • コマンドライン引数: -r (チャンク範囲テスト), -c (チャンクサイズ指定), -o (トークン所有権の切り替え), -s (セマンティック処理の有効化) および対象ファイルパス。
  • **出力**:
  • 標準出力: パース成功時の "SUCCESS!" メッセージ、または -s 指定時の再構成された JSON。
  • 標準エラー: パースエラー発生時の詳細情報 (json_errdetail)。
  • 戻り値: 正常終了時は 0、エラー発生時は 1。

前提条件と不変条件

  • 入力ファイルは PG_BINARY_R モードで読み取り可能であること。
  • チャンクサイズは BUFSIZE を超えてはならない。
  • -s オプション使用時は、DoState 構造体が適切に初期化され、JsonSemAction が設定されること。

アルゴリズムまたは振る舞い

1. **初期化**: getopt で引数を解析し、JsonLexContext を準備する。 2. **ループ処理**:

  • ファイルを chunk_size 分だけ読み込み、appendBinaryStringInfo でバッファに追加する。
  • バッファの末尾に意図的に "1+23 trailing junk" を付加し、パーサーが不要なデータを無視できるか確認する。
  • pg_parse_json_incremental を呼び出し、JSON_INCOMPLETE が返る限り入力を継続する。
  • 最終チャンクでは is_last = true を指定してパースを完了させる。

    • 3. **終了処理**: freeJsonLexContext を呼び出し、リソースを解放する。 4. **反復**: -r オプションが指定されている場合、チャンクサイズを減らしながら上記プロセスを繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（実行ファイルとして起動）。
  • **呼び出し先**:
  • pg_parse_json_incremental (パース実行)
  • makeJsonLexContextIncremental (コンテキスト初期化)
  • getopt, fread, fstat, fopen (標準ライブラリ)
  • appendBinaryStringInfo, resetStringInfo (lib/stringinfo.h)
  • json_errdetail (common/jsonapi.h)

関連データ構造

  • JsonLexContext: パース状態を保持する。
  • JsonSemAction: パースイベントのコールバックテーブル。
  • DoState: セマンティック処理時に使用される状態保持用構造体。
  • StringInfoData: 入力チャンクを保持する動的バッファ。

失敗モードと境界条件

  • chunk_size が BUFSIZE を超える場合、pg_fatal で終了する。
  • ファイルのオープンや stat 取得に失敗した場合、pg_fatal で終了する。
  • パース結果が JSON_SUCCESS 以外の場合、エラー詳細を出力して終了する。
  • 読み込みバイト数が期待値と異なる場合、エラーとして処理される。

保守担当者向けメモ

  • ツールによる呼び出し元情報の検索結果（amcheck 等）は、本シンボルとは無関係な PostgreSQL 内部の同名関数を指している可能性が高いため、無視すること。
  • -s モードにおける DoState のメモリ割り当ては palloc を使用しているため、終了時の解放フローに注意が必要。
