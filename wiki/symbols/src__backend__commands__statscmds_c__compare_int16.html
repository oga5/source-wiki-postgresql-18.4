compare_int16

定義

  • 種別: function
  • 定義箇所: src/backend/commands/statscmds.c:48

目的

int16 型の値のペアを比較し、qsort 等のソート関数で使用可能な比較結果を返す。

シグネチャまたは形状

int compare_int16(const void *a, const void *b)

入力と出力

  • **入力**: 比較対象となる2つの int16 値へのポインタ (a, b)。
  • **出力**: a と b の差分（int 型）。a < b なら負、a == b なら 0、a > b なら正の値を返す。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な int16 値を指している必要がある。
  • 実行環境において int 型が int16 よりも広いビット幅を持つことを前提としており、減算結果のオーバーフローは発生しない。

アルゴリズムまたは振る舞い

1. a および b が指す値を int16 として読み取り、それぞれ av, bv に代入する。 2. av - bv を計算し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/statscmds.c:446 にて qsort の比較関数として呼び出されている。
  • **呼び出し先**: なし。

関連データ構造

  • int16: 比較対象のデータ型。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、未定義の動作となる。
  • int 型が int16 と同じビット幅である環境では、減算時にオーバーフローが発生する可能性がある（PostgreSQLの現在の移植性要件ではこのリスクは考慮済み）。

保守担当者向けメモ

  • 本関数は qsort 用の汎用的な比較関数として実装されている。
  • 統計情報の属性番号（attnum）のソートなど、int16 配列の順序付けが必要な箇所でのみ使用すること。
