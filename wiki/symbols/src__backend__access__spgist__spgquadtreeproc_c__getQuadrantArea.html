getQuadrantArea

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgquadtreeproc.c:82

目的

SP-GiST四分木において、指定された境界ボックス（bbox）を重心（centroid）に基づいて4つの象限に分割し、特定の象限に対応する新しい境界ボックスを生成する。

シグネチャまたは形状

BOX *getQuadrantArea(BOX *bbox, Point *centroid, int quadrant)

入力と出力

  • **入力**:
  • bbox: 分割対象となる元の境界ボックス（BOX型）。
  • centroid: 分割の基準点となる重心（Point型）。
  • quadrant: 取得対象の象限番号（1〜4）。
  • **出力**:
  • 指定された象限の範囲を定義する新しい BOX 構造体へのポインタ。メモリは palloc により動的に確保される。

前提条件と不変条件

  • quadrant は 1 から 4 のいずれかである必要がある。
  • bbox および centroid は有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. palloc を使用して新しい BOX 構造体のメモリを確保する。 2. quadrant の値に基づき、以下の通り重心を境界として bbox を切り出す：

  • **1 (右上)**: low を centroid に、high を bbox->high に設定。
  • **2 (右下)**: low.x を centroid.x、low.y を bbox->low.y、high.x を bbox->high.x、high.y を centroid.y に設定。
  • **3 (左下)**: low を bbox->low に、high を centroid に設定。
  • **4 (左上)**: low.x を bbox->low.x、low.y を centroid.y、high.x を centroid.x、high.y を bbox->high.y に設定。

    • 3. 構築した BOX 構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgquadtreeproc.c 内の spg_quad_inner_consistent 関数から呼び出される（388行目）。
  • **呼び出し先**: メモリ確保のために palloc を呼び出す。

関連データ構造

  • BOX: 幾何学的な矩形領域を定義する構造体（utils/geo_decls.h）。
  • Point: 2次元座標を保持する構造体（utils/geo_decls.h）。

失敗モードと境界条件

  • quadrant が 1〜4 以外の値である場合、result のメンバは初期化されず、不定な値が返される可能性がある。
  • メモリ不足時には palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は spg_quad_inner_consistent におけるKNN検索（近傍検索）の境界ボックス計算に使用されている。
  • 象限の定義は反時計回り（1:右上、2:右下、3:左下、4:左上）に基づいている。変更時は getQuadrant 関数との整合性に注意すること。
