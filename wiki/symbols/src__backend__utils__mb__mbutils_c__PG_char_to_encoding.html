PG_char_to_encoding

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/mbutils.c:1410

目的

SQLレベルから呼び出し可能な関数として、エンコーディング名（文字列）を内部的なエンコーディングID（整数）に変換します。

シグネチャまたは形状

Datum PG_char_to_encoding(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る Name 型の引数（エンコーディング名）。
  • **出力**: int32 型のエンコーディングIDを Datum として返します。

前提条件と不変条件

  • 入力された Name 型の引数は、有効なエンコーディング名として解釈可能である必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_NAME(0) を使用して、引数から Name 型の構造体を取得します。 2. NameStr(*s) により、Name 構造体からC文字列を取り出します。 3. 内部関数 pg_char_to_encoding() を呼び出し、文字列をエンコーディングIDに変換します。 4. PG_RETURN_INT32() を介して、取得したIDを戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースを介して呼び出されます（具体的なSQL文脈はソースコード上では明示されていません）。
  • **呼び出し先**: 内部関数 pg_char_to_encoding() を呼び出します。

関連データ構造

  • Name: PostgreSQLのシステムカタログ等で使用される固定長文字列型。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • 入力されたエンコーディング名がシステムで認識されない場合、内部の pg_char_to_encoding() の実装に依存したエラー処理（通常はエラー発生）が行われます。

保守担当者向けメモ

  • 本関数はSQLラッパーであり、実際の変換ロジックは pg_char_to_encoding() に委譲されています。ロジックの変更が必要な場合は、内部関数側を確認してください。
