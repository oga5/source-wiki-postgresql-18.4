RevmapContents

定義

  • 種別: struct
  • 定義箇所: src/include/access/brin_page.h:78

目的

BRIN（Block Range Index）インデックスにおいて、ページ範囲とインデックス内のタプルを対応付けるリバースマップ（Revmap）のデータ構造を定義する。

シグネチャまたは形状

typedef struct RevmapContents
{
    ItemPointerData rm_tids[1];
} RevmapContents;

入力と出力

  • **入力**: 本構造体はページ内のコンテンツ領域として配置される。
  • **出力**: rm_tids 配列を通じて、特定のページ範囲に対応する ItemPointerData（タプル識別子）のリストを提供する。

前提条件と不変条件

  • 本構造体はページ内の利用可能な全領域を占有するように設計されている。
  • rm_tids は可変長配列として扱われるため、メモリレイアウトの計算には REVMAP_CONTENT_SIZE マクロを使用する必要がある。
  • 構造体定義上は rm_tids[1] となっているが、実際にはページ境界まで拡張される。

アルゴリズムまたは振る舞い

  • BRINインデックスのページがリバースマップページである場合、PageGetContents(page) を介して本構造体へキャストし、rm_tids 配列にアクセスすることで、インデックス内のタプル位置を特定する。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • contrib/pageinspect/brinfuncs.c: PageGetContents を経由して rm_tids を取得。
  • src/backend/access/brin/brin_revmap.c: リバースマップの読み書き操作において、ページ内容を本構造体にキャストして使用。
  • **呼び出し先**:
  • storage/itemptr.h: ItemPointerData 型の定義を利用。

関連データ構造

  • ItemPointerData: rm_tids 配列の要素型。
  • BrinSpecialSpace: BRINページの末尾に配置されるページタイプ識別用構造体。

失敗モードと境界条件

  • 構造体が可変長配列として扱われるため、静的なサイズ計算（sizeof(RevmapContents)）は実際のデータ保持量と一致しない。
  • ページ内の空き領域を超えてアクセスしないよう、REVMAP_PAGE_MAXITEMS による境界チェックが必須である。

保守担当者向けメモ

  • 本構造体は、C言語の仕様上、空の構造体内に柔軟な配列メンバー（Flexible Array Member）を宣言できない制約を回避するために rm_tids[1] と定義されている。
  • メモリレイアウトを扱う際は、offsetof(RevmapContents, rm_tids) を用いた計算がソースコード内で参照されている点に注意すること。
