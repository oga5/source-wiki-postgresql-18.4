IntervalAggState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/timestamp.c:80

目的

interval 型の集約関数（sum 等）を実行する際に、集約状態を保持するための構造体です。有限な間隔の合計と、無限大（正および負）の出現回数を個別に管理します。

シグネチャまたは形状

typedef struct IntervalAggState
{
    int64       N;              /* 有限な間隔の処理数 */
    Interval    sumX;           /* 有限な間隔の合計 */
    int64       pInfcount;      /* +infinity の出現回数 */
    int64       nInfcount;      /* -infinity の出現回数 */
} IntervalAggState;

入力と出力

  • **入力**: 集約対象となる Interval 値。
  • **出力**: 集約状態の更新（do_interval_accum 等による状態の蓄積）。

前提条件と不変条件

  • N は有限な間隔の数のみをカウントします。
  • pInfcount および nInfcount は N に含まれません。
  • 合計値の計算には IA_TOTAL_COUNT() マクロを使用して、有限数と無限大のカウントを統合する必要があります。

アルゴリズムまたは振る舞い

  • do_interval_accum を通じて、新しい Interval 値が有限か無限か判定され、対応するフィールド（sumX または各 Infcount）が更新されます。
  • do_interval_discard を通じて、集約状態から特定の値を減算する処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/timestamp.c 内の集約関数実装（do_interval_accum, do_interval_discard 等）。
  • **呼び出し先**: palloc0（メモリ確保）が使用されています。

関連データ構造

  • Interval: src/include/datatype/timestamp.h で定義される、time、day、month を保持する構造体。

失敗モードと境界条件

  • N が 0 の場合、sumX は未定義または初期状態となります。
  • 無限大のカウントは int64 の範囲内で管理されます。

保守担当者向けメモ

  • N と pInfcount/nInfcount を混同しないよう注意してください。集約結果を算出する際は、必ず IA_TOTAL_COUNT() マクロを介して合計件数を取得してください。
