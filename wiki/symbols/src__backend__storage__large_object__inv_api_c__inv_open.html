inv_open

定義

  • 種別: function
  • 定義箇所: src/backend/storage/large_object/inv_api.c:214

目的

既存のラージオブジェクト（LO）をオープンし、操作に必要な権限チェックと状態管理を行うための LargeObjectDesc 構造体を生成・初期化します。

シグネチャまたは形状

LargeObjectDesc *inv_open(Oid lobjId, int flags, MemoryContext mcxt)

入力と出力

  • **入力**:
  • lobjId: オープン対象のラージオブジェクトのOID。
  • flags: アクセスモード（INV_READ, INV_WRITE）。
  • mcxt: 記述子を割り当てるメモリコンテキスト。
  • **出力**:
  • 成功時: 初期化された LargeObjectDesc 構造体へのポインタ。
  • 失敗時: エラー（ereport）によりトランザクションが中断される。

前提条件と不変条件

  • flags に INV_READ または INV_WRITE が含まれている必要があります。
  • 指定された lobjId がシステムカタログ上に存在する必要があります。
  • 読み込みまたは書き込みの権限が、現在のユーザーに対して許可されている必要があります。

アルゴリズムまたは振る舞い

1. **フラグ変換**: flags を内部的な descflags（IFS_RDLOCK, IFS_WRLOCK）に変換します。 2. **スナップショット取得**: 書き込み権限が必要な場合は NULL（最新状態）、読み込みのみの場合は GetActiveSnapshot() を使用します。 3. **存在確認**: LargeObjectExistsWithSnapshot を呼び出し、対象LOの存在を検証します。 4. **権限チェック**: pg_largeobject_aclcheck_snapshot を使用し、現在のユーザーが対象LOに対して必要な権限（ACL_SELECT または ACL_UPDATE）を持っているか確認します。 5. **記述子生成**: 指定された mcxt 上に LargeObjectDesc を割り当て、ID、オフセット、フラグ、スナップショットを格納して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-fsstubs.c（複数の箇所で INV_READ または INV_WRITE モードにて呼び出し）
  • **呼び出し先**:
  • GetActiveSnapshot()
  • LargeObjectExistsWithSnapshot()
  • pg_largeobject_aclcheck_snapshot()
  • MemoryContextAlloc()

関連データ構造

  • LargeObjectDesc: オープンされたLOの状態を保持する構造体。
  • Oid: ラージオブジェクトを一意に識別する型。

失敗モードと境界条件

  • **無効なフラグ**: flags が INV_READ も INV_WRITE も含まない場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー。
  • **オブジェクト不在**: 指定されたOIDが存在しない場合、ERRCODE_UNDEFINED_OBJECT でエラー。
  • **権限不足**: 必要なACL権限がない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー。

保守担当者向けメモ

  • 本関数で割り当てられた LargeObjectDesc は、呼び出し元が適切なメモリコンテキストで管理する必要があります。
  • snapshot は現在の有効なスナップショットが設定されますが、呼び出し元が必要に応じてより長寿命なコピーに置き換えることが想定されています。
  • lo_compat_privileges が有効な場合、権限チェックはスキップされます。
