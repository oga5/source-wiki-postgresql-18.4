prune_append_rel_partitions

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partprune.c:779

目的

プランニング段階において、クエリの制約条件（baserestrictinfo）に基づき、不要なパーティションを静的に除外（プルーニング）し、スキャン対象となるパーティションの集合を特定する。

シグネチャまたは形状

Bitmapset *prune_append_rel_partitions(RelOptInfo *rel)

入力と出力

  • **入力**: RelOptInfo *rel（対象となるパーティションテーブルのプランニング情報）
  • **出力**: Bitmapset *（生存するパーティションのインデックス集合。条件が矛盾する場合は NULL を返す）

前提条件と不変条件

  • rel->part_scheme が NULL であってはならない。
  • enable_partition_pruning が無効、または制約条件が存在しない場合は、全パーティションのインデックスを返す。
  • プランニング時の呼び出しであるため、PartitionPruneContext の planstate、exprcontext、exprstates は NULL に設定される。

アルゴリズムまたは振る舞い

1. パーティションが存在しない場合、NULL を返す。 2. gen_partprune_steps を呼び出し、クエリの baserestrictinfo からプランニング時に評価可能なプルーニングステップを生成する。 3. ステップ生成の結果、条件が矛盾（gcontext.contradictory）していれば NULL を返す。 4. 有効なステップが存在しない場合は、全パーティションを対象とする Bitmapset を返す。 5. PartitionPruneContext を初期化し、get_matching_partitions を呼び出して生存パーティションを算出する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/inherit.c:358
  • **呼び出し先**:
  • gen_partprune_steps
  • get_matching_partitions
  • bms_add_range

関連データ構造

  • RelOptInfo: パーティション情報（part_scheme, boundinfo, nparts）を保持。
  • GeneratePruningStepsContext: ステップ生成中の状態を保持。
  • PartitionPruneContext: プルーニング実行に必要なメタデータ（戦略、境界情報、関数情報など）を保持。

失敗モードと境界条件

  • rel->nparts == 0: 空の Bitmapset（NULL）を返す。
  • gcontext.contradictory: 条件が矛盾していると判断し、NULL を返す。
  • pruning_steps == NIL: プルーニング可能な条件がないと判断し、全パーティションを返す。

保守担当者向けメモ

  • 本関数はプランニング時の静的プルーニング専用である。実行時の動的プルーニングについては、PartitionPruneContext の planstate 等が設定される別のフロー（execPartition.c 等からの呼び出し）を参照すること。
  • context.stepcmpfuncs のメモリ割り当ては palloc0 を使用しており、partnatts とステップ数に依存する。
