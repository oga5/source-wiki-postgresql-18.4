coerce_to_domain

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_coerce.c:674

目的

式をドメイン型へ変換するためのノード（CoerceToDomain）を生成し、実行時にドメイン制約のチェックが行われるように式ツリーを修飾する。

シグネチャまたは形状

Node *coerce_to_domain(Node *arg, Oid baseTypeId, int32 baseTypeMod, Oid typeId,
                       CoercionContext ccontext, CoercionForm cformat, int location,
                       bool hideInputCoercion)

入力と出力

  • **入力**:
  • arg: 変換対象の式ノード。
  • baseTypeId, baseTypeMod: 変換元となるベース型のOIDと型修飾子。
  • typeId: 変換先のドメイン型のOID。
  • ccontext, cformat: 変換のコンテキストと表示形式。
  • location: パーサー内の位置情報。
  • hideInputCoercion: ネストされた変換ステップの表示を抑制するかどうか。
  • **出力**:
  • 変換後の式ノード（CoerceToDomain ノードでラップされたもの）。

前提条件と不変条件

  • baseTypeId は有効なOIDである必要がある（Assert による検証）。
  • ドメイン型への変換において、resulttypmod は常に -1 に設定される。

アルゴリズムまたは振る舞い

1. **型の一致確認**: baseTypeId と typeId が同一であれば、変換を行わず arg をそのまま返す。 2. **表示制御**: hideInputCoercion が真の場合、hide_coercion_node を呼び出してネストされた変換ステップの表示を抑制する。 3. **型修飾子の適用**: coerce_type_typmod を呼び出し、ベース型に対する型修飾子（長さ制限等）を適用する。この際、COERCE_IMPLICIT_CAST を指定する。 4. **ノード生成**: CoerceToDomain ノードを生成し、制約チェックに必要な情報を格納する。 5. **結果の返却**: 生成した CoerceToDomain ノードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_coerce.c 内の各変換処理（coerce_type 等）、および src/backend/parser/parse_target.c。
  • **呼び出し先**: hide_coercion_node, coerce_type_typmod, makeNode。

関連データ構造

  • CoerceToDomain: ドメイン制約チェックを表現するノード構造体。
  • Node, Oid, CoercionContext, CoercionForm: PostgreSQLのパーサーで使用される基本型および列挙型。

失敗モードと境界条件

  • baseTypeId が無効な場合、アサーションエラーが発生する。
  • baseTypeId == typeId の場合は変換処理をスキップする。

保守担当者向けメモ

  • 本関数は実行時の制約チェックを目的としている。
  • ALTER DOMAIN で typtypmod を変更することは、本関数の固定的な式構造の性質上、安全ではない可能性がある。
  • resultcollid は本関数内では設定されず、後続の parse_collate.c で処理される。
