euc_jp2sjis

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_jp_and_sjis/euc_jp_and_sjis.c:536

目的

EUC_JP形式の文字列をSJIS形式に変換します。

シグネチャまたは形状

static int euc_jp2sjis(const unsigned char *euc, unsigned char *p, int len, bool noError)

入力と出力

  • **入力**:
  • euc: 変換元EUC_JP文字列へのポインタ。
  • p: 変換先バッファへのポインタ。
  • len: 変換対象のバイト長。
  • noError: 変換エラー時に例外を発生させず中断するかを指定するフラグ。
  • **出力**:
  • 戻り値: 処理したバイト数。
  • pが指すバッファに変換後のSJIS文字列が格納され、末尾にヌル文字が付与されます。

前提条件と不変条件

  • eucは有効なEUC_JPバイト列である必要があります。
  • pは変換結果を格納するのに十分な領域を確保している必要があります。

アルゴリズムまたは振る舞い

1. lenが0になるまで、ソース文字列を1文字ずつ走査します。 2. ASCII文字（高ビットが立っていない）はそのままコピーします。 3. マルチバイト文字の場合、pg_encoding_verifymbcharで妥当性を検証します。 4. 文字種に応じて変換を行います：

  • **SS2**: 半角カナとして処理し、後続バイトをコピーします。
  • **SS3**: JIS X0212漢字またはIBM漢字として処理します。IBM漢字の場合はibmkanjiテーブルを参照し、一致しない場合はPGSJISALTCODEを使用します。
  • **その他**: JIS X0208漢字として計算式に基づき変換します。

    • 5. 変換したバイト数分だけポインタを進め、最後にヌル終端します。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の変換インターフェース関数（例: euc_jp_to_sjis）から呼び出されます。
  • **呼び出し先**: pg_encoding_verifymbchar（検証用）、report_invalid_encoding（エラー報告用）。

関連データ構造

  • ibmkanji: SJISとEUC_JP間の特殊な文字変換テーブル。
  • PGSJISALTCODE: マッピング未定義時の代替コード。

失敗モードと境界条件

  • 不正なバイト列が検出された場合、noErrorがfalseであればreport_invalid_encodingを呼び出し、trueであれば処理を中断します。

保守担当者向けメモ

  • 本関数は内部的な変換ロジックを担う静的関数です。
  • IBM漢字の変換テーブル（ibmkanji）はsjis.mapに依存しているため、マッピングの変更が必要な場合は当該ファイルを確認してください。
