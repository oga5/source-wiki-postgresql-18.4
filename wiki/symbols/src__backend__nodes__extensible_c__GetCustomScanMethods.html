GetCustomScanMethods

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/extensible.c:136

目的

登録済みのカスタムスキャンメソッド構造体（CustomScanMethods）を、指定された名前をキーとして検索・取得します。

シグネチャまたは形状

const CustomScanMethods *GetCustomScanMethods(const char *CustomName, bool missing_ok)

入力と出力

  • **入力**:
  • CustomName: 検索対象のカスタムスキャン名（文字列）。
  • missing_ok: true の場合、対象が見つからなくてもエラーを発生させず NULL を返します。false の場合はエラーを報告します。
  • **出力**:
  • 成功時: 該当する CustomScanMethods 構造体へのポインタ。
  • 失敗時: missing_ok が true であれば NULL。

前提条件と不変条件

  • custom_scan_methods ハッシュテーブルが初期化されている必要があります。
  • CustomName は EXTNODENAME_MAX_LEN（64バイト）以内の文字列である必要があります。

アルゴリズムまたは振る舞い

1. 内部ハッシュテーブル custom_scan_methods を対象として GetExtensibleNodeEntry を呼び出します。 2. 指定された CustomName に一致するエントリを検索します。 3. 取得したエントリを CustomScanMethods 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていませんが、カスタムスキャン計画を処理するコアシステム（copyfuncs.c や outfuncs.c 等）から利用されることが想定されています。
  • **呼び出し先**: GetExtensibleNodeEntry（src/backend/nodes/extensible.c 内）。

関連データ構造

  • CustomScanMethods: カスタムスキャンノードの操作メソッドを定義する構造体。
  • custom_scan_methods: 登録されたメソッドを保持する内部ハッシュテーブル。

失敗モードと境界条件

  • missing_ok が false の状態で存在しない名前を指定した場合、GetExtensibleNodeEntry 経由でエラーが発生します。
  • CustomName が NULL または不正なポインタである場合の挙動は、GetExtensibleNodeEntry の実装に依存します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実体は GetExtensibleNodeEntry によるハッシュテーブル検索です。
  • 拡張機能がカスタムスキャンを定義する際、この関数を通じてメソッドが取得されるため、登録時の名前の一致が重要です。
