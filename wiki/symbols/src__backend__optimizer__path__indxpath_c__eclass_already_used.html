eclass_already_used

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/indxpath.c:683

目的

指定された EquivalenceClass (EC) が、既に indexjoinclauses リスト内のいずれかの制約条件によって使用されているかを判定する。

シグネチャまたは形状

static bool eclass_already_used(EquivalenceClass *parent_ec, Relids oldrelids, List *indexjoinclauses)

入力と出力

  • **入力**:
  • parent_ec: 検査対象の EquivalenceClass。
  • oldrelids: 比較対象となるリレーション ID の集合。
  • indexjoinclauses: IndexClause 構造体のリスト。
  • **出力**:
  • bool: parent_ec と一致し、かつ clause_relids が oldrelids のサブセットである IndexClause がリスト内に存在する場合は true、それ以外は false。

前提条件と不変条件

  • indexjoinclauses リスト内の各要素は IndexClause 型であり、その rinfo フィールド（RestrictInfo）が有効であること。

アルゴリズムまたは振る舞い

1. indexjoinclauses リストを反復処理する。 2. 各 IndexClause から RestrictInfo を取得する。 3. RestrictInfo の parent_ec が引数の parent_ec と一致し、かつ rinfo->clause_relids が oldrelids のサブセット（bms_is_subset）であるかを確認する。 4. 条件を満たす要素が見つかった時点で true を返し、リストの最後まで見つからなければ false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/indxpath.c 内の複数箇所（行 102, 563, 679 付近）から呼び出されている。
  • **呼び出し先**:
  • bms_is_subset (PostgreSQL のビットマップセット操作関数) を呼び出している。

関連データ構造

  • EquivalenceClass: 等価クラスを保持する構造体。
  • IndexClause: インデックス適合条件を保持する構造体。
  • RestrictInfo: クエリの制約条件を保持する構造体。

失敗モードと境界条件

  • indexjoinclauses が空リストの場合、常に false を返す。
  • rinfo->parent_ec が NULL の場合、比較は失敗する（parent_ec と一致しないため）。

保守担当者向けメモ

  • 本関数は、インデックスパス生成時に重複する等価クラスの適用を避けるためのフィルタリングに使用される。
  • bms_is_subset によるリレーション ID の包含関係チェックは、パラメータ化されたパスの生成において重要な役割を果たす。
