ExplainOpenSetAsideGroup

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_format.c:388

目的

EXPLAIN 出力において、現在の階層構造とは別に、特定の情報を独立したグループとして扱うための準備を行う。主に JSON や YAML 形式において、ネストレベルの調整やグループ状態のスタック管理を行う。

シグネチャまたは形状

void ExplainOpenSetAsideGroup(const char *objtype, const char *labelname,
                              bool labeled, int depth, ExplainState *es)

入力と出力

  • objtype: オブジェクトの型（現在は未使用）。
  • labelname: グループのラベル名。
  • labeled: ラベルの有無を示すフラグ。
  • depth: インデントを増やす深さ。
  • es: ExplainState 構造体へのポインタ。出力状態を保持する。
  • 出力: なし（es 構造体の内部状態を更新する）。

前提条件と不変条件

  • es が有効な ExplainState を指していること。
  • es->format に基づいて適切な状態遷移が行われること。

アルゴリズムまたは振る舞い

es->format に応じて以下の処理を行う：

  • EXPLAIN_FORMAT_TEXT: 何も行わない。
  • EXPLAIN_FORMAT_XML: es->indent を depth 分増加させる。
  • EXPLAIN_FORMAT_JSON: es->grouping_stack に 0 をプッシュし、es->indent を depth 分増加させる。
  • EXPLAIN_FORMAT_YAML: labelname が存在する場合 1 を、存在しない場合 0 を es->grouping_stack にプッシュし、es->indent を depth 分増加させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/explain.c 内の EXPLAIN 実行ロジック（例: EXPLAIN のワーカー出力処理など）。
  • 呼び出し先: lcons_int（リスト操作関数）。

関連データ構造

  • ExplainState: 出力フォーマット、インデント、グループ状態（grouping_stack）を管理する中心的な構造体。

失敗モードと境界条件

  • es->format が未定義または不正な値の場合の挙動は定義されていない。
  • depth に負の値を指定した場合のインデントへの影響は呼び出し側に依存する。

保守担当者向けメモ

  • 本関数は ExplainSaveGroup と対で使用されることが多い。
  • grouping_stack へのプッシュ操作は、後続の ExplainCloseGroup 等での階層管理と整合している必要がある。
