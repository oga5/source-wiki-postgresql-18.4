find_next_mcelem

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_selfuncs.c:1129

目的

最頻要素（MCELEM）の配列に対して二分探索を行い、指定された値の存在確認または挿入位置の特定を行います。

シグネチャまたは形状

static bool find_next_mcelem(Datum *mcelem, int nmcelem, Datum value, int *index, TypeCacheEntry *typentry)

入力と出力

  • **入力**:
  • mcelem: 比較対象となる最頻要素の配列（Datum 配列）。
  • nmcelem: mcelem 配列の要素数。
  • value: 検索対象の値。
  • index: 検索開始位置を示すポインタ。
  • typentry: 型情報のキャッシュ。
  • **出力**:
  • 戻り値: 値が見つかった場合は true、見つからない場合は false。
  • index: 値が見つかった場合はそのインデックス、見つからない場合は挿入すべき位置（l）が格納されます。

前提条件と不変条件

  • mcelem 配列は、element_compare 関数によって定義される順序でソートされている必要があります。
  • *index は、探索を開始する有効なインデックスである必要があります。

アルゴリズムまたは振る舞い

1. *index から nmcelem - 1 の範囲で二分探索を実行します。 2. element_compare を用いて mcelem[i] と value を比較します。 3. 一致した場合は *index にその位置を代入し true を返します。 4. 一致しない場合は、探索範囲を狭めながら繰り返し、最終的に値が存在すべき位置を *index に代入して false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/array_selfuncs.c:593
  • **呼び出し先**:
  • element_compare（比較関数）

関連データ構造

  • Datum: PostgreSQLの汎用データ表現。
  • TypeCacheEntry: 型の比較関数などを保持するキャッシュ構造体。

失敗モードと境界条件

  • nmcelem が 0 の場合、ループは実行されず *index は初期値のまま false を返します。
  • 探索範囲外のインデックスが指定された場合、二分探索のロジックに従い適切な挿入位置が算出されます。

保守担当者向けメモ

  • 本関数は mcelem 配列がソートされていることを前提としています。統計情報の更新プロセスでソート順が維持されているか確認してください。
  • element_compare の挙動が型によって異なるため、新しい型をサポートする際は typentry の設定に注意してください。
