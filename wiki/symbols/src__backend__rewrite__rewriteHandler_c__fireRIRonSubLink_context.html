fireRIRonSubLink_context

定義

  • 種別: struct
  • 定義箇所: src/backend/rewrite/rewriteHandler.c:61

目的

クエリ書き換え処理において、サブリンク（SubLink）内のRIR（Retrieve-Instead-Retrieve）ルール適用状況と行レベルセキュリティ（RLS）の状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct fireRIRonSubLink_context
{
	List	   *activeRIRs;
	bool		hasRowSecurity;
} fireRIRonSubLink_context;

入力と出力

本構造体は関数 fireRIRonSubLink の引数として渡され、再帰的なツリー走査を通じて状態を共有します。

  • activeRIRs: 現在処理中のRIRルールに関連するリレーションのリスト。再帰的なビュー参照による無限ループを検知するために使用されます。
  • hasRowSecurity: 処理対象のクエリまたはサブクエリに行レベルセキュリティポリシーが適用されているかを示すフラグ。

前提条件と不変条件

  • activeRIRs は、クエリ書き換えの再帰呼び出しにおいて、現在展開中のビューを追跡するために使用される必要があります。
  • hasRowSecurity は、RLSポリシーの適用判定結果を反映します。

アルゴリズムまたは振る舞い

クエリツリーの走査中に fireRIRonSubLink 関数へ渡され、サブリンク内のノードに対してRIRルールの適用やRLSのチェックを行う際の制御状態を維持します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/rewrite/rewriteHandler.c 内の fireRIRonSubLink を呼び出す箇所（例: 2209 行目、2262 行目付近）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しの引数として fireRIRonSubLink(Node *node, fireRIRonSubLink_context *context) に渡されます。

関連データ構造

  • List: PostgreSQLの汎用リスト構造。activeRIRs の保持に使用。

失敗モードと境界条件

  • activeRIRs の管理が不適切な場合、再帰的なビュー参照により無限ループが発生する可能性があります。

保守担当者向けメモ

  • "RIR" は "Retrieve-Instead-Retrieve" の略称です。
  • 本構造体は rewriteHandler.c 内のクエリ書き換えロジックに特化しています。変更時は fireRIRonSubLink 関数との整合性に注意してください。
