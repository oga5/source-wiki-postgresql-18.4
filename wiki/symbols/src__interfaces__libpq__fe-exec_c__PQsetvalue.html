PQsetvalue

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-exec.c:452

目的

PGresult オブジェクト内の特定の行および列に対して、値を設定または更新します。

シグネチャまたは形状

int PQsetvalue(PGresult *res, int tup_num, int field_num, char *value, int len);

入力と出力

  • **入力**:
  • res: 対象の PGresult オブジェクト。
  • tup_num: 設定対象の行番号（0から開始）。
  • field_num: 設定対象の列番号。
  • value: 格納する文字列データ。
  • len: value の長さ。NULL_LEN または value が NULL の場合は NULL 値として扱われます。
  • **出力**: 成功時に true (1)、失敗時に false (0) を返します。

前提条件と不変条件

  • res が NULL または OOM_result である場合、操作は失敗します。
  • field_num は有効な列範囲内である必要があります。
  • tup_num は 0 以上かつ res->ntups 以下である必要があります。

アルゴリズムまたは振る舞い

1. 引数の妥当性を検証します。 2. tup_num == res->ntups の場合、新しい行を割り当て、各列を NULL で初期化した上で pqAddTuple を呼び出して結果セットに追加します。 3. len が NULL_LEN または value が NULL の場合、対象フィールドを NULL としてマークします。 4. len <= 0 の場合、対象フィールドを空文字列としてマークします。 5. それ以外の場合、pqResultAlloc を使用してメモリを確保し、value をコピーして終端文字 \0 を付加します。 6. 失敗時は pqInternalNotice を通じてエラーメッセージを通知します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-exec.c 内で参照されています。
  • **呼び出し先**: check_field_number, pqResultAlloc, pqAddTuple, pqInternalNotice, libpq_gettext を呼び出します。

関連データ構造

  • PGresult: クエリ結果を保持する構造体。
  • PGresAttValue: 行内の各フィールド値を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: pqResultAlloc が失敗し、false を返します。
  • 範囲外アクセス: tup_num が res->ntups を超える場合、または field_num が無効な場合に false を返します。

保守担当者向けメモ

  • 本関数は PGresult の内部メモリ管理（pqResultAlloc）に強く依存しています。
  • NULL 値の扱いは res->null_field を介して行われるため、このフィールドの整合性に注意してください。
