calculate_database_size

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/dbsize.c:117

目的

指定されたデータベース（OID）が占有する物理ディスクサイズをバイト単位で集計する。

シグネチャまたは形状

int64 calculate_database_size(Oid dbOid)

入力と出力

  • **入力**: dbOid (対象データベースのOID)
  • **出力**: int64 (全テーブルスペース上の合計サイズ)

前提条件と不変条件

  • 実行ユーザーは、対象データベースに対する CONNECT 権限を持つか、pg_read_all_stats ロールの権限を有している必要がある。権限がない場合はエラーが発生する。
  • pg_global（共有ストレージ）は集計対象外である。
  • ファイル走査中にクエリのキャンセル要求を検知するため、ループ内で CHECK_FOR_INTERRUPTS() を呼び出す。

アルゴリズムまたは振る舞い

1. **権限チェック**: object_aclcheck を使用してアクセス権を確認する。 2. **デフォルト領域の集計**: base/<dbOid> ディレクトリのサイズを db_dir_size で取得する。 3. **追加テーブルスペースの走査**: pg_tblspc ディレクトリを走査し、各テーブルスペース内に存在する当該データベースのディレクトリ（.../<tablespace_oid>/<version>/<dbOid>）のサイズを順次加算する。 4. **リソース解放**: ディレクトリ記述子を FreeDir で閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/dbsize.c 内の pg_database_size_oid および pg_database_size_name から呼び出される。
  • **呼び出し先**:
  • object_aclcheck, has_privs_of_role, aclcheck_error, get_database_name (権限・メタデータ関連)
  • db_dir_size (ディレクトリサイズ計算)
  • AllocateDir, ReadDir, FreeDir (ディレクトリ操作)
  • CHECK_FOR_INTERRUPTS (割り込みチェック)

関連データ構造

  • Oid: データベース識別子。
  • DIR, dirent: ディレクトリ操作に使用される構造体。

失敗モードと境界条件

  • 権限不足: aclcheck_error を呼び出し、エラーをスローする。
  • ディレクトリ不在: db_dir_size が適切に処理できない場合、集計値に影響する可能性がある。
  • 割り込み: CHECK_FOR_INTERRUPTS() により、ユーザーによるキャンセル要求時に処理が中断される。

保守担当者向けメモ

  • pg_global を除外する仕様はハードコードされている。
  • テーブルスペースのディレクトリ構造（PG_TBLSPC_DIR および TABLESPACE_VERSION_DIRECTORY）に変更があった場合は、パス生成ロジックの更新が必要。
