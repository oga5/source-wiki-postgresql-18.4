PgArchShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/pgarch.c:157

目的

WALアーカイバプロセスが共有メモリ上で使用するデータ構造（PgArchData）の合計サイズを計算する。

シグネチャまたは形状

Size PgArchShmemSize(void)

入力と出力

  • 入力: なし
  • 出力: Size 型の共有メモリ所要バイト数

前提条件と不変条件

  • add_size 関数を使用して計算を行うため、オーバーフローが発生した場合は適切な最大値が返される。
  • 返されるサイズは PgArchData 構造体のサイズと一致する。

アルゴリズムまたは振る舞い

1. 初期サイズ size を 0 に設定する。 2. sizeof(PgArchData) を size に加算する。 3. 計算された合計サイズを返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/postmaster/pgarch.c: PgArchShmemInit 内で共有メモリ確保および初期化のために呼び出される。
  • src/backend/storage/ipc/ipci.c: CreateSharedMemoryAndSemaphores 内でシステム全体の共有メモリサイズ計算の一部として呼び出される。
  • 呼び出し先:
  • add_size (PostgreSQLのメモリ計算ユーティリティ)

関連データ構造

  • PgArchData: アーカイバプロセスの状態を保持する共有メモリ構造体。

失敗モードと境界条件

  • 共有メモリのサイズ計算において、add_size の制約に従う。

保守担当者向けメモ

  • 本関数は PgArchData 構造体の定義が変更された際に、必ず整合性を保つ必要がある。
  • src/include/postmaster/pgarch.h に外部宣言が存在する。
