expanded_record_fetch_tupdesc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/expandedrecord.c:823

目的

展開されたレコード（ExpandedRecordHeader）に関連付けられたタプル記述子（TupleDesc）を取得します。キャッシュ済みであればそれを返し、未キャッシュであれば型キャッシュから検索して取得・保持します。

シグネチャまたは形状

TupleDesc expanded_record_fetch_tupdesc(ExpandedRecordHeader *erh)

入力と出力

  • **入力**: erh - 処理対象の展開レコードヘッダーへのポインタ。
  • **出力**: 該当する TupleDesc 構造体へのポインタ。

前提条件と不変条件

  • erh は有効な ExpandedRecordHeader を指している必要があります。
  • 呼び出し元が既に erh->er_tupdesc が NULL でないことを確認している場合、即座にその値が返されます。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: erh->er_tupdesc が設定済みであれば、それを返します。 2. **検索**: lookup_rowtype_tupdesc を使用して、erh が保持する型IDと型修飾子から TupleDesc を取得します。 3. **参照カウント管理**:

  • 取得した TupleDesc が参照カウントを持つ場合（tdrefcount >= 0）:
  • メモリコンテキストのクリーンアップ時に参照を解放するため、ER_mc_callback をリセットコールバックとして登録します（未登録の場合）。
  • erh->er_tupdesc に取得した記述子を格納し、参照カウントをインクリメントします。
  • lookup_rowtype_tupdesc が取得した初期ピンを ReleaseTupleDesc で解放します。
  • 参照カウントを持たない場合:
  • 単に erh->er_tupdesc にポインタを格納します。

    • 4. **識別子割り当て**: assign_record_type_identifier を呼び出し、プロセスグローバルな er_tupdesc_id を更新します。 5. 取得した TupleDesc を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/utils/expandedrecord.h 内のインライン関数から参照されています。
  • その他の具体的な呼び出し元については、提供されたソース範囲内では特定できません。
  • **呼び出し先**:
  • lookup_rowtype_tupdesc
  • MemoryContextRegisterResetCallback
  • ER_mc_callback
  • ReleaseTupleDesc
  • assign_record_type_identifier

関連データ構造

  • ExpandedRecordHeader: レコードのメタデータと状態を保持するメイン構造体。
  • TupleDesc: タプルの構造を定義する記述子。

失敗モードと境界条件

  • lookup_rowtype_tupdesc が型情報を解決できない場合、システムカタログの不整合等によりエラーが発生する可能性があります。
  • メモリコンテキストのコールバック登録時にメモリ不足が発生する可能性があります。

保守担当者向けメモ

  • 本関数は、展開レコードのライフサイクルと TupleDesc の参照カウントを同期させる重要な役割を担っています。
  • tdrefcount の有無による分岐処理は、静的割り当ての記述子と動的割り当ての記述子を区別するために必須です。
