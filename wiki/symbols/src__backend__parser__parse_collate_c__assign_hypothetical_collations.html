assign_hypothetical_collations

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_collate.c:954

目的

集約関数（Aggref）における「仮説的（hypothetical）」引数と、それに対応する集約引数の間で照合順序（Collation）を解決し、必要に応じてノードの照合順序を調整します。

シグネチャまたは形状

static void assign_hypothetical_collations(Aggref *aggref, assign_collations_context *loccontext);

入力と出力

  • **入力**:
  • aggref: 処理対象の集約関数ノード。
  • loccontext: 照合順序の伝播状態を保持するコンテキスト。
  • **出力**: なし（副作用として aggref 内の式ノードを直接変更します）。

前提条件と不変条件

  • aggref->aggdirectargs と aggref->args の対応する要素間で照合順序の競合がないこと。
  • 照合順序が競合した場合はエラーを発生させること。

アルゴリズムまたは振る舞い

1. **初期化**: aggdirectargs のうち、仮説的引数ではない部分を先に処理します。 2. **並行走査**: 仮説的引数と集約引数をペアで走査します。 3. **照合順序の解決**: 各ペアに対して assign_collations_walker を呼び出し、共通の照合順序を特定します。 4. **競合検出**: 照合順序が一致しない場合は COLLATE_CONFLICT としてエラーを報告します。 5. **ノードの更新**: 照合順序が変更される場合、RelabelType ノードを挿入して新しい照合順序を適用します。 6. **状態の伝播**: 条件（引数が1つのみ等）を満たす場合、merge_collation_state を呼び出し、親コンテキストへ照合順序をマージします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_collate.c 内の集約関数処理ロジック（assign_aggregate_collations 等から呼び出される）。
  • **呼び出し先**: assign_collations_walker, merge_collation_state, makeRelabelType, ereport 等。

関連データ構造

  • Aggref: 集約関数を表すノード。
  • assign_collations_context: 照合順序の解決状態を保持する構造体。
  • RelabelType: 照合順序を強制的に変更するために挿入されるノード。

失敗モードと境界条件

  • **照合順序の不一致**: 異なる暗黙的照合順序を持つ引数が混在する場合、ERRCODE_COLLATION_MISMATCH エラーを発生させます。
  • **非照合型**: 型が照合順序を持たない場合、解決処理はスキップされます。

保守担当者向けメモ

  • 本関数は RelabelType を使用してパース時に照合順序をマークしていますが、これは本来プランナ/エグゼキュータ向けの仕組みを流用した暫定的な実装です。
  • assign_collations_walker が式構造を直接変更する設計は脆弱であるため、修正時は ruleutils.c 等への影響に注意してください。
