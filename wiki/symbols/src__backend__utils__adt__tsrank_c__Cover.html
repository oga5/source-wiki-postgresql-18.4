Cover

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsrank.c:650

目的

文書（DocRepresentation）内から、指定されたクエリ（QueryRepresentation）を完全に包含する最小の範囲（Cover）を探索し、その位置情報を算出します。

シグネチャまたは形状

static bool Cover(DocRepresentation *doc, int len, QueryRepresentation *qr, CoverExt *ext);

入力と出力

  • **入力**:
  • doc: 文書の表現データ配列。
  • len: 文書の長さ。
  • qr: クエリの表現データ。
  • ext: 探索範囲および結果を保持する CoverExt 構造体。
  • **出力**:
  • 戻り値: クエリをカバーする範囲が見つかった場合は true、見つからなかった場合は false。
  • ext: 探索結果（begin, end）および次回の探索開始位置（pos）が更新されます。

前提条件と不変条件

  • check_stack_depth() により、再帰呼び出し時のスタック深さが監視されます。
  • ext->pos は、探索を開始する文書内のインデックスを指している必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: resetQueryRepresentation を呼び出し、クエリ状態をリセットします。 2. **上限探索**: 現在の ext->pos から文書の末尾に向かって走査し、クエリ条件を満たす最初の位置（ext->q）を特定します。 3. **下限探索**: 見つかった上限位置から逆方向（前方）へ走査し、クエリ条件を満たす最も近い位置（ext->p）を特定します。 4. **再帰的探索**:

  • ext->p <= ext->q の場合、有効なカバーが見つかったと判断し、次回の探索開始位置を更新して終了します。
  • 条件を満たさない場合、ext->pos を進めて自身を再帰呼び出しします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsrank.c 内のランキング処理ロジック（calc_rank_cd 等から呼び出されるループ内）。
  • **呼び出し先**:
  • resetQueryRepresentation
  • fillQueryRepresentationData
  • TS_execute
  • Cover (再帰呼び出し)

関連データ構造

  • DocRepresentation: 文書の単語位置情報を含む構造体。
  • QueryRepresentation: クエリの実行状態を保持する構造体。
  • CoverExt: 探索範囲（begin, end）および位置情報（p, q, pos）を管理する構造体。

失敗モードと境界条件

  • **スタックオーバーフロー**: 再帰呼び出しを行うため、非常に複雑なクエリや長い文書ではスタック深さに注意が必要です。
  • **探索失敗**: 文書内にクエリをカバーする範囲が存在しない場合、false を返します。
  • **境界**: ptr - doc < len の範囲外へのアクセスは発生しないよう制御されています。

保守担当者向けメモ

  • 本関数は ts_rank_cd（近接度ランキング）のコアロジックです。
  • 探索は右方向への走査（上限特定）と左方向への走査（下限特定）の二段階で行われます。
  • 再帰呼び出しの終了条件は ext->p <= ext->q です。
