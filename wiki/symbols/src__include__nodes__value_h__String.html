String

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/value.h:63

目的

SQLクエリの構文解析時や字句解析時に、文字列リテラルを保持・伝達するためのノード構造体です。

シグネチャまたは形状

typedef struct String
{
	pg_node_attr(special_read_write)

	NodeTag		type;
	char	   *sval;
} String;

入力と出力

  • **入力**: sval フィールドに文字列ポインタを保持します。
  • **出力**: strVal() マクロ（または同等のアクセサ）を通じて、内部の char* 値が抽出されます。

前提条件と不変条件

  • type フィールドは T_String である必要があります。
  • sval が指す文字列は、PostgreSQLのメモリ管理システム（palloc）によって割り当てられたメモリ領域である必要があります。

アルゴリズムまたは振る舞い

  • パーサーが文字列リテラルを検出した際に makeString 関数を介して生成されます。
  • 構文解析ツリー（Parse Tree）の一部として List 構造等に格納され、後続のフェーズ（意味解析や計画作成）まで保持されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照の可能性）**:
  • contrib/pg_overexplain/pg_overexplain.c
  • contrib/postgres_fdw/postgres_fdw.c
  • contrib/test_decoding/test_decoding.c
  • src/backend/access/common/tupdesc.c
  • src/backend/catalog/namespace.c
  • src/backend/catalog/objectaccess.c
  • src/backend/catalog/objectaddress.c
  • **呼び出し先**:
  • 本構造体自体はデータコンテナであり、特定の関数を呼び出すロジックは含みません。生成には makeString（src/backend/nodes/value.c）が使用されます。

関連データ構造

  • NodeTag: ノードの型識別子。
  • List: String ノードを保持するために頻繁に使用されるリスト構造。

失敗モードと境界条件

  • sval が NULL の場合、後続の処理でデリファレンスが発生するリスクがあります。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • String ノードに格納される文字列は、呼び出し側で palloc されたものである必要があります。
  • src/include/nodes/value.h で定義されているアクセスマクロ strVal() を使用して値を取り出すことが推奨されます。
