astreamer_lz4_compressor_content

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_lz4.c:117

目的

入力データストリームをLZ4形式で圧縮し、バッファ管理を行いながら後続のストリーマーへ転送する。

シグネチャまたは形状

static void astreamer_lz4_compressor_content(
    astreamer *streamer,
    astreamer_member *member,
    const char *data,
    int len,
    astreamer_archive_context context
)

入力と出力

  • **入力**:
  • streamer: 圧縮状態を保持する astreamer_lz4_frame 構造体。
  • data, len: 圧縮対象の入力データとその長さ。
  • member, context: ストリーム処理のメタデータおよびアーカイブコンテキスト。
  • **出力**: なし（副作用として内部バッファへの書き込み、または後続ストリーマーへのデータ転送を行う）。

前提条件と不変条件

  • streamer は astreamer_lz4_frame 型としてキャスト可能であること。
  • LZ4F_compressBegin が未実行の場合、初回呼び出し時にヘッダーが書き込まれる。
  • 出力バッファの容量が不足した場合は、enlargeStringInfo により動的に拡張される。

アルゴリズムまたは振る舞い

1. **ヘッダー処理**: 初回呼び出し時、LZ4F_compressBegin を用いてLZ4フレームヘッダーをバッファに書き込む。 2. **バッファ確認**: 現在の出力バッファの空き容量が、入力データに対する最大圧縮サイズ（LZ4F_compressBound）を下回る場合、既存のバッファ内容を astreamer_content を介して後続ストリーマーへフラッシュし、バッファを空にする。 3. **バッファ拡張**: 必要に応じて enlargeStringInfo でバッファを拡張する。 4. **圧縮実行**: LZ4F_compressUpdate を呼び出し、入力データを圧縮してバッファへ書き込む。 5. **エラーハンドリング**: LZ4ライブラリの各関数がエラーを返した場合、pg_fatal を呼び出しプロセスを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer フレームワークのパイプライン処理（直接的な呼び出し元関数名はソース内に明示されていないが、astreamer_lz4_compressor_new 内で関数ポインタとして登録される）。
  • **呼び出し先**:
  • LZ4F_compressBegin, LZ4F_compressUpdate, LZ4F_compressBound, LZ4F_isError, LZ4F_getErrorName (LZ4ライブラリ)
  • astreamer_content (後続ストリーマーへの転送)
  • enlargeStringInfo (バッファ拡張)
  • pg_fatal (エラー報告)

関連データ構造

  • astreamer_lz4_frame: 圧縮状態、LZ4コンテキスト、および出力バッファを保持する構造体。

失敗モードと境界条件

  • **LZ4エラー**: 圧縮処理中にライブラリがエラーを返した場合、pg_fatal により異常終了する。
  • **メモリ不足**: バッファ拡張時にメモリが不足した場合、enlargeStringInfo の挙動に従い異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は USE_LZ4 が定義されている環境でのみコンパイルされる。
  • bbs_buffer の管理は astreamer フレームワークの規約に従っており、bytes_written が現在の書き込み位置を正確に追跡していることを前提としている。
