ProcessStartupProcInterrupts

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/startup.c:153

目的

リカバリ処理（WALリプレイ）のループ中に定期的に呼び出され、保留中のシグナルや割り込み、およびPostmasterプロセスの生存状態を監視・処理する。

シグネチャまたは形状

void ProcessStartupProcInterrupts(void)

入力と出力

  • **入力**: 引数なし。グローバル変数（got_SIGHUP, shutdown_requested, ProcSignalBarrierPending, LogMemoryContextPending 等）を参照する。
  • **出力**: なし（副作用としてプロセス終了や設定再読み込みを行う）。

前提条件と不変条件

  • 本関数はリカバリ処理中の適切なタイミングで呼び出される必要がある。
  • POSTMASTER_POLL_RATE_LIMIT が定義されている場合、PostmasterIsAlive() のチェックは間引いて実行される。

アルゴリズムまたは振る舞い

1. **設定再読み込み**: got_SIGHUP が真の場合、フラグをリセットし StartupRereadConfig() を実行する。 2. **終了要求の確認**: shutdown_requested が真の場合、proc_exit(1) を呼び出しプロセスを終了する。 3. **Postmaster生存確認**: IsUnderPostmaster が真の場合、PostmasterIsAlive() を確認する。生存していない場合は exit(1) で緊急停止する。このチェックは POSTMASTER_POLL_RATE_LIMIT に基づき頻度が制限される。 4. **バリア処理**: ProcSignalBarrierPending が真の場合、ProcessProcSignalBarrier() を実行する。 5. **メモリコンテキストログ**: LogMemoryContextPending が真の場合、ProcessLogMemoryContextInterrupt() を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlogrecovery.c 内のリカバリ処理ループ（複数箇所）。
  • **呼び出し先**:
  • StartupRereadConfig()
  • proc_exit() / exit()
  • PostmasterIsAlive()
  • ProcessProcSignalBarrier()
  • ProcessLogMemoryContextInterrupt()

関連データ構造

  • got_SIGHUP (グローバルフラグ)
  • shutdown_requested (グローバルフラグ)
  • ProcSignalBarrierPending (グローバルフラグ)
  • LogMemoryContextPending (グローバルフラグ)

失敗モードと境界条件

  • **Postmasterの異常終了**: PostmasterIsAlive() が偽を返すと、リカバリ完了を待たずに exit(1) でプロセスが終了する。

保守担当者向けメモ

  • POSTMASTER_POLL_RATE_LIMIT は、システムコール（PostmasterIsAlive）のオーバーヘッドを抑えるための最適化である。
  • 本関数はリカバリ処理の進捗を妨げないよう、軽量に保つ必要がある。
