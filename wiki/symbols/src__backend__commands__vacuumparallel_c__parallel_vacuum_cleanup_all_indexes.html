parallel_vacuum_cleanup_all_indexes

定義

  • 種別: function
  • 定義箇所: src/backend/commands/vacuumparallel.c:518

目的

並列VACUUM処理におけるインデックスのクリーンアップフェーズを実行し、共有メモリ上の統計情報を更新する。

シグネチャまたは形状

void parallel_vacuum_cleanup_all_indexes(ParallelVacuumState *pvs, long num_table_tuples, int num_index_scans, bool estimated_count)

入力と出力

  • pvs: 並列VACUUMの状態を保持する構造体。
  • num_table_tuples: テーブルの生存タプル数の推定値。
  • num_index_scans: 実行するインデックススキャンの回数。
  • estimated_count: num_table_tuples が推定値であるかを示すフラグ。
  • 出力: なし（副作用として pvs->shared の統計情報を更新し、インデックス処理を実行する）。

前提条件と不変条件

  • 本関数はリーダープロセスによって呼び出される必要がある（IsParallelWorker() が偽であることをアサートしている）。

アルゴリズムまたは振る舞い

1. 引数として渡されたテーブルの生存タプル数（num_table_tuples）および推定フラグを、共有メモリ上の pvs->shared 構造体に書き込む。 2. parallel_vacuum_process_all_indexes を呼び出し、インデックスのクリーンアップ処理を開始する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/vacuumlazy.c
  • 呼び出し先: parallel_vacuum_process_all_indexes

関連データ構造

  • ParallelVacuumState: 並列VACUUMの制御状態を保持する構造体。

失敗モードと境界条件

  • 並列ワーカープロセスから呼び出された場合、アサート失敗により異常終了する。

保守担当者向けメモ

  • 本関数はインデックスのクリーンアップフェーズの開始点であり、リーダープロセスが共有メモリを更新した後にワーカーへ処理を委譲する役割を持つ。
