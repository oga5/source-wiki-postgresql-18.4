ResourceOwnerRememberJIT

定義

  • 種別: static inline function
  • 定義箇所: src/backend/jit/llvm/llvmjit.c:130

目的

JIT コンテキスト（LLVMJitContext）を PostgreSQL のリソースオーナー（ResourceOwner）に登録し、トランザクション終了時の自動的なクリーンアップを保証する。

シグネチャまたは形状

void ResourceOwnerRememberJIT(ResourceOwner owner, LLVMJitContext *handle)

入力と出力

  • **入力**:
  • owner: リソースを管理する ResourceOwner 構造体へのポインタ。
  • handle: 登録対象の LLVMJitContext 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • owner は有効な ResourceOwner であること。
  • handle は PointerGetDatum を介して ResourceOwnerRemember に渡されるため、リソースオーナーの管理下で適切に追跡可能な状態であること。

アルゴリズムまたは振る舞い

1. handle を Datum 型に変換する。 2. jit_resowner_desc を記述子として使用し、ResourceOwnerRemember を呼び出すことで、指定された owner に handle を登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit.c:241（llvm_create_context 内で CurrentResourceOwner を引数として呼び出される）。
  • **呼び出し先**:
  • ResourceOwnerRemember（PostgreSQL のリソース管理 API）。

関連データ構造

  • LLVMJitContext: JIT コンパイル状態を保持する構造体。
  • ResourceOwner: トランザクション単位のリソース管理を行う構造体。
  • jit_resowner_desc: JIT リソースの解放処理を定義する記述子。

失敗モードと境界条件

  • owner が NULL の場合、ResourceOwnerRemember の内部実装に依存する（通常はエラーとなる）。
  • メモリ不足等の理由でリソースオーナーへの登録が失敗した場合、ResourceOwnerRemember がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、主に llvm_create_context におけるコンテキスト生成直後の登録処理で使用される。
  • リソースの解放が必要な場合は、対応する ResourceOwnerForgetJIT および ResOwnerReleaseJitContext が適切に実装されていることを確認すること。
