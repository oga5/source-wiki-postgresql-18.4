SortShimExtra

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/sort/sortsupport.c:27

目的

従来のSQL呼び出し可能な比較関数（BTORDER_PROC）を、SortSupport インターフェースで利用可能な形式に適合させるためのコンテキスト情報を保持します。

シグネチャまたは形状

typedef struct
{
    FmgrInfo    flinfo;            /* 比較関数のルックアップデータ */
    FunctionCallInfoBaseData fcinfo; /* 再利用可能な呼び出し情報構造体 */
} SortShimExtra;

入力と出力

本構造体自体はデータ保持用であり、直接的な入力・出力関数はありません。ssup->ssup_extra を通じて SortSupportData に付随します。

前提条件と不変条件

  • ssup_extra フィールドとして使用する場合、メモリリークを防ぐため、必ず ssup_cxt（メモリコンテキスト）上で割り当てる必要があります。
  • fcinfo のサイズは可変であるため、割り当て時には SizeForSortShimExtra(nargs) マクロを使用して必要なメモリ量を算出する必要があります。

アルゴリズムまたは振る舞い

  • 比較関数呼び出しに必要な FmgrInfo をキャッシュし、FunctionCallInfoBaseData を保持することで、ソートループ内での関数呼び出しオーバーヘッドを削減します。
  • comparison_shim 関数がこの構造体を参照し、ラップされた従来の比較関数を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/sortsupport.c 内の PrepareSortSupportComparisonShim 関数などが、この構造体を割り当てて初期化します。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。ただし、内部の flinfo を介して比較関数が実行されます。

関連データ構造

  • SortSupportData: 本構造体が格納される親構造体。
  • comparison_shim: 本構造体を利用して比較を行うラッパー関数。

失敗モードと境界条件

  • メモリ割り当て失敗: MemoryContextAlloc が失敗した場合、システムはエラーをスローします。
  • 不適切な nargs 指定: SizeForSortShimExtra マクロで誤った引数数を指定した場合、メモリ破壊が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は SortSupport フレームワークの内部実装詳細です。
  • fcinfo は可変長構造体であるため、構造体の末尾に配置されています。この構造体を拡張または変更する際は、メモリレイアウトに注意してください。
