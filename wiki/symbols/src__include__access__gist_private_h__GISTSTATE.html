GISTSTATE

定義

  • 種別: struct
  • 定義箇所: src/include/access/gist_private.h:75

目的

GiST (Generalized Search Tree) インデックスの操作（検索、挿入、構築）において、演算子クラスが提供するサポート関数へのアクセス情報や、インデックス操作中のメモリコンテキスト、タプル記述子を一元管理するために使用されます。

シグネチャまたは形状

typedef struct GISTSTATE
{
    MemoryContext scanCxt;
    MemoryContext tempCxt;
    TupleDesc   leafTupdesc;
    TupleDesc   nonLeafTupdesc;
    TupleDesc   fetchTupdesc;
    FmgrInfo    consistentFn[INDEX_MAX_KEYS];
    FmgrInfo    unionFn[INDEX_MAX_KEYS];
    FmgrInfo    compressFn[INDEX_MAX_KEYS];
    FmgrInfo    decompressFn[INDEX_MAX_KEYS];
    FmgrInfo    penaltyFn[INDEX_MAX_KEYS];
    FmgrInfo    picksplitFn[INDEX_MAX_KEYS];
    FmgrInfo    equalFn[INDEX_MAX_KEYS];
    FmgrInfo    distanceFn[INDEX_MAX_KEYS];
    FmgrInfo    fetchFn[INDEX_MAX_KEYS];
    Oid         supportCollation[INDEX_MAX_KEYS];
} GISTSTATE;

入力と出力

  • **入力**: インデックス操作開始時に、対象インデックスのメタデータ（TupleDesc やサポート関数情報）を保持するために初期化されます。
  • **出力**: 保持された FmgrInfo を通じて、GiST インデックスの各操作（consistent, union, penalty 等）に必要な関数呼び出しを仲介します。

前提条件と不変条件

  • INDEX_MAX_KEYS までの各サポート関数が適切に初期化されている必要があります。
  • scanCxt はスキャン寿命のデータ、tempCxt は関数呼び出し時の短期データ保持に使用されるというメモリ管理上の役割分担があります。

アルゴリズムまたは振る舞い

GiST インデックスのエンジン（gist.c 等）がツリーを降下または操作する際、この構造体を参照して適切なサポート関数を呼び出します。特に、インデックスの構築や挿入時のページ分割、検索時のタプル比較において、演算子クラス固有のロジックを動的に実行するためのハブとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gist.c 内の関数群（gistfixsplit, gistplacetopage, gistformdownlink など）が、インデックス操作のコンテキストとして本構造体を利用しています。
  • **呼び出し先**: FmgrInfo フィールドを介して、インデックスの演算子クラスで定義された各サポート関数（consistent, union, penalty 等）を呼び出します。

関連データ構造

  • TupleDesc: インデックスのタプル構造を記述。
  • FmgrInfo: PostgreSQL の関数マネージャ情報。
  • GISTInsertState: 挿入処理の状態管理に使用される構造体。

失敗モードと境界条件

  • サポート関数が未定義または不正な場合、FmgrInfo を介した呼び出しでエラーが発生する可能性があります。
  • INDEX_MAX_KEYS を超えるキー数を持つインデックス定義はサポートされません。

保守担当者向けメモ

  • 各フィールドはインデックスの演算子クラス（opclass）と密接に結びついています。サポート関数の追加や変更を行う際は、本構造体の定義と初期化ロジックを同期させる必要があります。
  • GISTSTATE の初期化は、インデックス操作の最初の呼び出し時に indexInfo->ii_AmCache を介して行われるのが一般的です。
