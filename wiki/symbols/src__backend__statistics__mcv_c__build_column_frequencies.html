build_column_frequencies

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/mcv.c:489

目的

多変量MCV（Most Common Values）リスト構築プロセスにおいて、各列ごとの値の出現頻度を集計し、重複排除されたリストを生成する。

シグネチャまたは形状

SortItem **build_column_frequencies(SortItem *groups, int ngroups, MultiSortSupport mss, int *ncounts)

入力と出力

  • **入力**:
  • groups: 集計対象となる SortItem の配列。
  • ngroups: groups 配列の要素数。
  • mss: 複数列のソートと比較をサポートする MultiSortSupport 構造体。
  • ncounts: 各列ごとのユニークな値の数を格納するための出力用配列（呼び出し元で確保）。
  • **出力**:
  • 各列ごとの頻度情報を含む SortItem の二次元配列（SortItem **）。

前提条件と不変条件

  • groups および ncounts は NULL であってはならない（Assert による検証）。
  • mss->ndims は処理対象の列数を表し、メモリ確保の基準となる。

アルゴリズムまたは振る舞い

1. **メモリ確保**: 全列分を単一のメモリチャンクとして確保し、ポインタ配列と各列のデータ配列を配置する。 2. **データ抽出**: 各列（次元）ごとに、入力された groups から値、NULLフラグ、カウント情報を抽出する。 3. **ソートと集計**:

  • qsort_interruptible を使用して、各列の値をソートする。
  • ソートされたリストを走査し、同一の値を持つアイテムをマージしてカウントを合算する。
  • 重複排除後の結果を ncounts に記録し、圧縮されたリストを構築する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/mcv.c 内の関数から呼び出されている（行274など）。
  • **呼び出し先**: qsort_interruptible（標準的なソート関数）、sort_item_compare（比較関数）。

関連データ構造

  • SortItem: 値、NULLフラグ、カウントを保持する構造体。
  • MultiSortSupport: 複数列のソート順序や比較ロジックを保持する構造体。

失敗モードと境界条件

  • ngroups が 0 の場合、ループは実行されず空のリストが返される。
  • メモリ不足時には palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数はメモリを単一の大きなブロックとして確保するため、pfree を行う際は result ポインタを解放するだけで全領域が解放される設計となっている。
  • sort_item_compare の結果が 0 の場合にカウントを合算するロジックは、値の重複排除において重要である。
