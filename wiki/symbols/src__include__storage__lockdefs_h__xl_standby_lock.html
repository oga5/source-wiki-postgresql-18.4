xl_standby_lock

定義

  • 種別: struct
  • 定義箇所: src/include/storage/lockdefs.h:54

目的

スタンバイサーバーにおいて、WAL（Write-Ahead Logging）記録時にテーブルに対する排他ロック（AccessExclusiveLock）の状態を保持・伝播するために使用される構造体です。

シグネチャまたは形状

typedef struct xl_standby_lock
{
	TransactionId xid;
	Oid			dbOid;
	Oid			relOid;
} xl_standby_lock;

入力と出力

  • **入力**: 構造体の各フィールド（xid, dbOid, relOid）は、ロックを保持しているトランザクションと対象テーブルを特定するために使用されます。
  • **出力**: WALレコードの一部として書き出されるか、スタンバイサーバー側のロック管理用ハッシュテーブルのキーとして使用されます。

前提条件と不変条件

  • xid は AccessExclusiveLock を保持しているトランザクションIDである必要があります。
  • dbOid および relOid は、ロック対象のデータベースおよびテーブルを正しく指し示している必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、主に LogAccessExclusiveLocks 関数を通じてWALへの書き込みに使用されます。
  • スタンバイサーバー側では、この構造体をキーとしてハッシュテーブルを構築し、リカバリ中のロック状態を再現するために利用されます。
  • XLogRegisterData を介して、複数の xl_standby_lock 構造体が連続したメモリ領域としてWALに登録されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/storage/ipc/standby.c: WAL記録およびスタンバイ側のロック管理ロジック。
  • src/backend/storage/lmgr/lock.c: ロック状態の取得およびメモリ確保。
  • src/include/storage/lock.h: GetRunningTransactionLocks 関数による外部インターフェース提供。
  • **呼び出し先**:
  • 本構造体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • TransactionId: トランザクション識別子。
  • Oid: オブジェクト識別子。
  • LOCKMODE: 本ファイルで定義されるロックモード（本構造体は AccessExclusiveLock に特化）。

失敗モードと境界条件

  • メモリ確保失敗: palloc を使用する箇所（lock.c 等）でメモリ不足が発生する可能性があります。
  • 不整合なWALデータ: リカバリ時にWALから読み取った dbOid や relOid が現在のシステムカタログと一致しない場合、スタンバイ側のロック管理に影響を与える可能性があります。

保守担当者向けメモ

  • 本構造体はWALフォーマットの一部であるため、フィールドの順序やサイズを変更するとWALの互換性が損なわれます。
  • src/backend/storage/ipc/standby.c 内のハッシュテーブルキーとして使用されているため、構造体のサイズ変更はハッシュ計算ロジックに影響を与える可能性があります。
