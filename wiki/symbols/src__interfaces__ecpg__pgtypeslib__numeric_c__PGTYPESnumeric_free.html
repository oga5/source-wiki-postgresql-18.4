PGTYPESnumeric_free

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/numeric.c:384

目的

numeric 構造体によって動的に確保されたメモリ領域を解放し、リソースリークを防ぐ。

シグネチャまたは形状

void PGTYPESnumeric_free(numeric *var)

入力と出力

  • **入力**: var (numeric *) - 解放対象の numeric 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 var は、PGTYPESnumeric_new 等の関数によって動的に確保された有効な numeric 構造体を指している必要がある。
  • var->buf は、内部的に確保されたバッファを指している必要がある。

アルゴリズムまたは振る舞い

1. var->buf を引数として digitbuf_free を呼び出し、数値データ用のバッファを解放する。 2. free(var) を呼び出し、numeric 構造体自体のメモリ領域を解放する。

呼び出し元と呼び出し先

  • **呼び出し先**: digitbuf_free（バッファ解放用）、標準ライブラリの free。
  • **呼び出し元**: src/interfaces/ecpg/compatlib/informix.c 内の複数の箇所から呼び出されている。

関連データ構造

  • numeric: 数値の桁、重み、符号、およびバッファポインタを保持する構造体。

失敗モードと境界条件

  • var が NULL の場合、標準ライブラリの free の仕様に依存する（通常は未定義動作またはクラッシュ）。
  • var->buf が不正なポインタである場合、digitbuf_free 内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は PGTYPESnumeric_new で確保されたメモリの対となる解放関数である。メモリリークを避けるため、numeric 型のライフサイクル終了時には必ず本関数を呼び出すこと。
