libpq_source

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_rewind/libpq_source.c:36

目的

リモートの PostgreSQL サーバーからデータファイルやメタデータを取得するための通信レイヤーを保持する構造体です。rewind_source インターフェースを継承し、libpq を介したファイル転送とリクエストのバッチ処理を管理します。

シグネチャまたは形状

typedef struct {
    rewind_source common;
    PGconn     *conn;
    int         num_requests;
    fetch_range_request request_queue[MAX_CHUNKS_PER_QUERY];
    StringInfoData paths;
    StringInfoData offsets;
    StringInfoData lengths;
} libpq_source;

入力と出力

  • **入力**: init_libpq_source() により初期化され、PGconn 接続を保持します。
  • **出力**: process_queued_fetch_requests() を通じて、リモートサーバーから取得したデータがローカルのターゲットファイルへ書き込まれます。

前提条件と不変条件

  • request_queue に格納されるリクエスト数は MAX_CHUNKS_PER_QUERY を超えてはなりません。
  • conn は有効な PGconn オブジェクトを指している必要があります。

アルゴリズムまたは振る舞い

1. **リクエスト蓄積**: libpq_queue_fetch_range() を介して、取得対象のファイル範囲が request_queue に追加されます。 2. **バッチ処理**: process_queued_fetch_requests() が呼び出されると、蓄積されたリクエストを StringInfoData 型のバッファに変換し、単一の libpq クエリとして送信します。 3. **ストリーミング**: PQsetSingleRowMode を使用して結果をストリーミング形式で受け取り、各チャンクを処理します。

呼び出し元と呼び出し先

  • **呼び出し元**: init_libpq_source, libpq_queue_fetch_range, libpq_finish_fetch, process_queued_fetch_requests 等の libpq_source.c 内の関数。
  • **呼び出し先**: libpq ライブラリ関数（PQsendQueryPrepared, PQsetSingleRowMode 等）。

関連データ構造

  • rewind_source: 継承元の抽象インターフェース。
  • fetch_range_request: request_queue を構成する個別のリクエスト要素。
  • StringInfoData: バッチ処理時にクエリ引数を構築するための一時バッファ。

失敗モードと境界条件

  • リモートサーバーでファイルが欠落している場合、process_queued_fetch_requests 内で remove_target_file が呼び出され、ローカルとの同期が図られます。
  • 接続の所有権は呼び出し元にあるため、libpq_destroy は接続を閉じません。

保守担当者向けメモ

  • MAX_CHUNKS_PER_QUERY の値は通信スループットに直結するため、大規模同期時のパフォーマンス調整の際はここを検討してください。
  • 接続管理の責務が呼び出し元にある点に注意してください。
