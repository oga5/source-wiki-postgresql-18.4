cash_eq

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/cash.c:617

目的

2つの通貨型（money）の値が等しいかどうかを判定する。

シグネチャまたは形状

Datum cash_eq(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの Cash 型（内部的には int64）の値を受け取る。
  • **出力**: bool 型の値を Datum として返す（等しい場合は true、それ以外は false）。

前提条件と不変条件

  • Cash 型は int64 として定義されている。
  • 入力値は PG_GETARG_CASH マクロにより適切に取得される必要がある。

アルゴリズムまたは振る舞い

1. PG_GETARG_CASH(0) および PG_GETARG_CASH(1) を使用して、引数として渡された2つの Cash 値を取得する。 2. 取得した2つの値を比較演算子 == で比較する。 3. 比較結果を PG_RETURN_BOOL マクロを介して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL レベルの等価比較演算子から呼び出される（直接的な呼び出し元情報はソースコード上では明示されていない）。
  • **呼び出し先**: PG_GETARG_CASH および PG_RETURN_BOOL マクロを使用。

関連データ構造

  • Cash: int64 の別名。通貨型の内部表現。

失敗モードと境界条件

  • 特筆すべき失敗モードはない。int64 同士の単純な比較であるため、オーバーフローや例外は発生しない。

保守担当者向けメモ

  • 本関数は money 型の比較演算における標準的な実装である。
  • ロケール設定（lc_monetary）は値の表示やパースには影響するが、本関数は内部表現である int64 同士を直接比較するため、ロケールに依存せずバイナリ値の一致を判定する。
