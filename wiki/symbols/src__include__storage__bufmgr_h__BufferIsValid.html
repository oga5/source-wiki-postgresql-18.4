BufferIsValid

定義

  • 種別: static inline function
  • 定義箇所: src/include/storage/bufmgr.h:367

目的

指定されたバッファ識別子が有効なバッファを参照しているかどうかを判定します。

シグネチャまたは形状

bool BufferIsValid(Buffer bufnum)

入力と出力

  • **入力**: bufnum (Buffer型) - 検査対象のバッファ識別子。
  • **出力**: bool - bufnum が InvalidBuffer ではなく、かつバッファの範囲内であれば true を返します。

前提条件と不変条件

  • bufnum は NBuffers（共有バッファ数）以下であること。
  • bufnum は -NLocBuffer（ローカルバッファ数の負値）以上であること。
  • 上記の条件は Assert によって実行時に検証されます。

アルゴリズムまたは振る舞い

1. bufnum が NBuffers 以下かつ -NLocBuffer 以上であることをアサートします。 2. bufnum が InvalidBuffer と等しくないことを確認し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_heapam.c
  • contrib/amcheck/verify_nbtree.c
  • contrib/pgstattuple/pgstatapprox.c
  • src/backend/access/brin/brin.c
  • src/backend/access/brin/brin_pageops.c
  • **呼び出し先**:
  • 本関数はインライン関数であり、特定の関数を呼び出すことはありません。

関連データ構造

  • Buffer: バッファを識別する整数型。正の値は共有バッファ、負の値はローカルバッファを指します。
  • InvalidBuffer: 無効なバッファを示す定数。

失敗モードと境界条件

  • bufnum が NBuffers を超える、または -NLocBuffer 未満の場合、Assert が有効なビルドではプログラムが停止します。
  • InvalidBuffer を渡した場合は false を返します。

保守担当者向けメモ

  • 本関数は共有バッファとローカルバッファの両方を扱うため、バッファ管理の境界チェックとして機能します。
  • FRONTEND マクロによる条件分岐は含まれていませんが、インライン関数として定義されているため、ヘッダーをインクルードする各ソースで展開されます。
