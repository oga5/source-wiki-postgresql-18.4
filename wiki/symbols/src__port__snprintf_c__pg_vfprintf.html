pg_vfprintf

定義

  • 種別: function
  • 定義箇所: src/port/snprintf.c:241

目的

PostgreSQL 独自の printf ファミリー実装の一部として、プラットフォーム間の差異を吸収し、POSIX 準拠のフォーマット機能を備えた vfprintf を提供する。

シグネチャまたは形状

int pg_vfprintf(FILE *stream, const char *fmt, va_list args)

入力と出力

  • **入力**:
  • stream: 出力先の FILE ポインタ。
  • fmt: フォーマット文字列。
  • args: 可変長引数リスト (va_list)。
  • **出力**:
  • 成功時: 出力された文字数（target.nchars）。
  • 失敗時: -1。

前提条件と不変条件

  • stream が NULL の場合、errno を EINVAL に設定し、即座に -1 を返す。
  • 内部バッファ（1024バイト）を使用してフォーマット処理を行い、必要に応じて flushbuffer を介してストリームへ書き出す。

アルゴリズムまたは振る舞い

1. PrintfTarget 構造体を初期化し、ローカルの 1024 バイトバッファを割り当てる。 2. dopr 関数を呼び出し、フォーマット文字列の解析と引数の展開を実行する。 3. dopr 完了後、flushbuffer を呼び出してバッファに残っている内容をストリームへ書き出す。 4. target.failed フラグの状態に基づき、エラー発生時は -1 を、成功時は書き込み文字数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port.h で vfprintf としてマクロ定義されており、PostgreSQL 内の各所から呼び出される可能性がある。
  • **呼び出し先**:
  • dopr(): フォーマット処理のバックエンド。
  • flushbuffer(): バッファの内容をストリームへ出力。

関連データ構造

  • PrintfTarget: 出力先バッファ、ストリーム、現在の状態（文字数、エラーフラグ）を保持する構造体。

失敗モードと境界条件

  • stream が NULL の場合、エラーとして -1 を返す。
  • dopr 内部でエラーが発生した場合、target.failed が true となり、最終的に -1 を返す。

保守担当者向けメモ

  • 本関数は標準ライブラリの vfprintf を置き換えるため、再帰呼び出しや標準関数との混同に注意すること。
  • 内部バッファサイズは 1024 バイトで固定されている。
