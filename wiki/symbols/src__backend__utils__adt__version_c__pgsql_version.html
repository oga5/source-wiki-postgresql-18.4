pgsql_version

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/version.c:20

目的

SQL関数 version() の実装として、PostgreSQLのビルド情報を含むバージョン文字列をシステムから取得し、SQLエンジンが扱える text 型として返却する。

シグネチャまたは形状

Datum pgsql_version(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出し規約に基づく引数群）。
  • **出力**: Datum 型（PostgreSQL内部の text 型として解釈されるバージョン文字列）。

前提条件と不変条件

  • PG_VERSION_STR マクロがビルド時に定義されていること。
  • cstring_to_text 関数が利用可能であること。

アルゴリズムまたは振る舞い

1. PG_VERSION_STR マクロからビルド時のバージョン文字列定数を取得する。 2. cstring_to_text を呼び出し、C言語の文字列をPostgreSQLの text 型へ変換する。 3. PG_RETURN_TEXT_P マクロを使用して、変換された text 型のデータを呼び出し元へ返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（version() 関数経由）。
  • **呼び出し先**: cstring_to_text（文字列変換用組み込み関数）。

関連データ構造

  • Datum: PostgreSQLの汎用データ表現。
  • text: PostgreSQLの可変長文字列型。
  • PG_VERSION_STR: バージョン情報を含むマクロ定数。

失敗モードと境界条件

  • メモリ不足時: cstring_to_text が内部でメモリ割り当てに失敗した場合、PostgreSQLの標準的なエラー処理（elog 等）によりアボートされる可能性がある。

保守担当者向けメモ

  • 本関数は PG_VERSION_STR に依存しているため、バージョン文字列の変更が必要な場合はソースコードではなくビルド設定（pg_config.h 等）を確認すること。
  • SQLインターフェースと内部定数を結びつけるシンプルなラッパーである。
