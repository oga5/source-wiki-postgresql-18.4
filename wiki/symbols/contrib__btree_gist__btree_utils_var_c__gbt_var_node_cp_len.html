gbt_var_node_cp_len

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_utils_var.c:122

目的

GiST インデックスノード内の可変長キー（lower と upper）間で共通するプレフィックスの長さをバイト単位で計算します。

シグネチャまたは形状

int32 gbt_var_node_cp_len(const GBT_VARKEY *node, const gbtree_vinfo *tinfo)

入力と出力

  • **入力**:
  • node: 比較対象の lower および upper 境界を持つ GBT_VARKEY 構造体。
  • tinfo: エンコーディング最大長 (eml) 等を含む gbtree_vinfo 構造体。
  • **出力**:
  • int32: 共通プレフィックスのバイト長。

前提条件と不変条件

  • node は gbt_var_key_readable で読み取り可能な形式である必要があります。
  • tinfo->eml が 1 より大きい場合、マルチバイト文字の境界を考慮した計算が行われます。

アルゴリズムまたは振る舞い

1. gbt_var_key_readable を用いて node から lower と upper のデータポインタと長さを取得します。 2. 両者の最小長 (ml) までバイト単位で比較を繰り返します。 4. 全バイトが一致した場合は ml を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_utils_var.c 内の複数箇所（gbt_var_node_truncate 等のインデックス操作ロジックから呼び出されています）。
  • **呼び出し先**:
  • gbt_var_key_readable
  • pg_mblen_range（tinfo->eml > 1 の場合）

関連データ構造

  • GBT_VARKEY: 可変長キーのコンテナ。
  • gbtree_vinfo: データ型固有のメタデータ（エンコーディング情報など）。
  • GBT_VARKEY_R: gbt_var_key_readable によって返される、読み取り用構造体。

失敗モードと境界条件

  • ml == 0（いずれかのキーが空）の場合、直ちに 0 を返します。

保守担当者向けメモ

  • eml（エンコーディング最大長）が 1 より大きい場合にのみマルチバイト対応の複雑なロジックが実行されます。
  • 戻り値はバイト数である点に注意してください。文字数ではありません。
