lookup_external_function

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/dfmgr.c:170

目的

ロード済みの共有ライブラリハンドルから、指定された名前の関数シンボルのアドレスを取得します。

シグネチャまたは形状

void *lookup_external_function(void *filehandle, const char *funcname);

入力と出力

  • **入力**:
  • filehandle: dlopen 等で取得された共有ライブラリのハンドル。
  • funcname: 検索対象の関数名を示すヌル終端文字列。
  • **出力**:
  • 成功時: 検索対象関数のメモリ上のアドレス（void *）。
  • 失敗時: NULL（dlsym の仕様に準拠）。

前提条件と不変条件

  • filehandle は、既に dlopen 等で正常にロードされた有効なライブラリハンドルである必要があります。
  • funcname は有効なシンボル名である必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った filehandle と funcname をそのまま dlsym 関数に渡します。 2. dlsym の戻り値（シンボルのアドレスまたはエラーを示す NULL）をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/fmgr/dfmgr.c:91 (ライブラリ内の追加関数検索)
  • src/backend/utils/fmgr/fmgr.c:464 (関数マネージャによる外部関数解決)
  • **呼び出し先**:
  • dlsym (システム提供の動的リンク用 API)

関連データ構造

  • 特になし（標準的な void * ハンドルと文字列を使用）。

失敗モードと境界条件

  • 指定した funcname がライブラリ内に存在しない場合、dlsym は NULL を返します。
  • filehandle が無効な場合、システム依存の動作（通常はエラーまたはクラッシュ）が発生する可能性があります。

保守担当者向けメモ

  • 本関数は dlsym の単純なラッパーです。
  • Windows 環境など、dlfcn.h が提供されないプラットフォームでは、dfmgr.c 内で定義されたマクロや抽象化レイヤーを通じて OS 固有の API に置換される点に注意してください。
