ExecCustomRestrPos

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeCustom.c:149

目的

カスタムスキャン実行ノードにおいて、以前にマークされた位置へスキャン位置を復元（RestrPos）するためのインターフェースを提供します。

シグネチャまたは形状

void ExecCustomRestrPos(CustomScanState *node)

入力と出力

  • **入力**: CustomScanState *node - カスタムスキャンの実行状態を保持する構造体。
  • **出力**: なし（副作用としてスキャン位置を復元）。

前提条件と不変条件

  • node->methods が有効であり、かつ RestrPosCustomScan コールバック関数が実装されている必要があります。

アルゴリズムまたは振る舞い

1. node->methods->RestrPosCustomScan が存在するかを確認します。 2. 存在しない場合は、ERRCODE_FEATURE_NOT_SUPPORTED エラーを発生させます。 3. 存在する場合は、拡張機能側で定義された RestrPosCustomScan 関数を呼び出し、スキャン位置の復元処理を委譲します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:389
  • **呼び出し先**: node->methods->RestrPosCustomScan（拡張機能により提供される関数ポインタ）

関連データ構造

  • CustomScanState: カスタムスキャンの実行状態を管理する構造体。
  • CustomExecMethods: 拡張機能が実装するコールバック関数のテーブル。

失敗モードと境界条件

  • 拡張機能が RestrPosCustomScan を実装していない場合、ereport(ERROR, ...) により処理が中断されます。

保守担当者向けメモ

  • 本関数は拡張機能への単なるディスパッチャです。ロジックの詳細は各拡張機能の実装に依存します。
  • MarkPos（位置の記録）と RestrPos（位置の復元）の両方がサポートされている必要がある点に注意してください。
