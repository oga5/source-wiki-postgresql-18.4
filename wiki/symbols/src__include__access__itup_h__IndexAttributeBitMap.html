IndexAttributeBitMap

定義

  • 種別: typedef
  • 定義箇所: src/include/access/itup.h:60

目的

インデックスタプル（IndexTuple）内に格納される属性値の NULL 状態を管理するためのビットマップ構造体へのポインタ型です。

シグネチャまたは形状

typedef IndexAttributeBitMapData * IndexAttributeBitMap;

入力と出力

  • **入力**: IndexAttributeBitMapData 構造体へのポインタ。
  • **出力**: なし（型定義のため）。

前提条件と不変条件

  • IndexTupleData の直後に配置されるメモリレイアウトを前提としています。
  • INDEX_MAX_KEYS に基づく固定長のビットマップ領域を消費します。

アルゴリズムまたは振る舞い

  • インデックスタプルのヘッダー情報（t_info）において NULL ビットマップの存在が示されている場合、この型を通じてビットマップ領域にアクセスし、各属性が NULL かどうかを判定します。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数情報は、提供されたソース抜粋からは特定できません。

関連データ構造

  • IndexAttributeBitMapData: 本型が指し示す実体となる構造体。
  • IndexTupleData: インデックスタプルの基底構造体。
  • IndexTupleHasNulls: タプルが NULL ビットマップを持つかを判定するマクロ。

失敗モードと境界条件

  • NULL ビットマップが存在しないタプルに対して本型を使用してアクセスを試みた場合、メモリ上の不正な位置を参照するリスクがあります。必ず IndexTupleHasNulls 等でフラグを確認する必要があります。

保守担当者向けメモ

  • 本型はポインタ型であるため、メモリレイアウトの変更時には IndexTupleData との相対位置関係に注意してください。
  • 属性数に関わらずビットマップサイズは固定であるため、将来的な属性数上限の変更時には関連する定数の整合性に留意してください。
