varchar_support

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varchar.c:564

目的

varchar 型の式に対するプランナの最適化（簡略化）をサポートします。具体的には、型キャストに伴う不要な処理を排除し、型修飾子（typmod）の変更のみで済む場合に式を再構築します。

シグネチャまたは形状

Datum varchar_support(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して SupportRequestSimplify 構造体へのポインタを受け取ります。
  • **出力**: Datum 型。最適化が可能な場合は簡略化された Node（relabel_to_typmod の結果）、不可能な場合は NULL を返します。

前提条件と不変条件

  • 入力された rawreq が SupportRequestSimplify 型である必要があります。
  • 式の引数リスト（expr->args）は少なくとも2つ以上の要素を持つ必要があります。
  • typmod 引数は定数（Const）であり、かつ NULL であってはなりません。

アルゴリズムまたは振る舞い

1. SupportRequestSimplify リクエストを検証し、対象の関数式を取得します。 2. 第2引数から新しい typmod を抽出します。 3. 以下の条件を満たす場合、relabel_to_typmod を用いて式を簡略化します。

  • 新しい typmod が負（制限なし）である場合。
  • 元の型が既知の typmod を持ち、かつ元の最大長（old_max）が新しい最大長（new_max）以下である場合（つまり、切り詰めが不要な場合）。

    • 4. 条件に合致しない場合は NULL を返し、最適化を行いません。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のプランナ（src/backend/utils/adt/varchar.c 内のサポート関数登録経由で呼び出されます）。
  • **呼び出し先**:
  • IsA, list_length, lsecond, linitial, exprTypmod, DatumGetInt32（マクロおよびユーティリティ関数）
  • relabel_to_typmod（式ノードの再構築）

関連データ構造

  • SupportRequestSimplify: プランナによる式簡略化リクエストを保持する構造体。
  • FuncExpr: 関数呼び出しを表現するノード。
  • Const: 定数値を保持するノード。

失敗モードと境界条件

  • typmod が定数でない場合や NULL の場合は最適化されません。
  • 元の型が typmod を持たない（old_typmod < 0）場合、安全側に倒して最適化をスキップする可能性があります。

保守担当者向けメモ

  • 本関数は varchar 型のキャスト処理を最適化するためのものです。bpchar（char(n)）のサポート関数とはロジックが異なる可能性があるため、変更時は varchar 特有のセマンティクスに注意してください。
