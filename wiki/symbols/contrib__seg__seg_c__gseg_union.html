gseg_union

定義

  • 種別: function
  • 定義箇所: contrib/seg/seg.c:226

目的

GiST インデックス構築時において、複数の区間（SEG）を包含する最小の境界を持つ単一の区間を計算する。

シグネチャまたは形状

Datum gseg_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GistEntryVector *（結合対象の区間エントリ群）。
  • PG_GETARG_POINTER(1): int *（結果のサイズを格納するポインタ）。
  • **出力**:
  • Datum: 結合結果である SEG 構造体へのポインタ。
  • *sizep: sizeof(SEG) が設定される。

前提条件と不変条件

  • 入力される GistEntryVector は少なくとも1つ以上のエントリを持つ必要がある。
  • sizep が指すメモリ領域は書き込み可能であること。

アルゴリズムまたは振る舞い

1. entryvec から最初の区間を初期値として取得する。 2. sizep に sizeof(SEG) を設定する。 3. 2番目以降のすべてのエントリに対し、gseg_binary_union を繰り返し呼び出し、累積的に結合を行う。 4. 最終的な結合結果を Datum として返す。 5. GIST_DEBUG が定義されている場合、標準エラー出力に "union" と出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス管理機構（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: gseg_binary_union（ループ内で各要素を結合するために呼び出される）。

関連データ構造

  • SEG: 区間データを保持する構造体。
  • GistEntryVector: GiST インデックスのノードに含まれるエントリの配列。

失敗モードと境界条件

  • entryvec が空の場合の挙動については、ソースコード上では明示的なチェックがないため、呼び出し側の GiST 実装に依存する。

保守担当者向けメモ

  • 本関数は GiST の union メソッドとして機能する。
  • 結合ロジックの詳細は gseg_binary_union に依存しているため、結合アルゴリズムの変更が必要な場合はそちらを参照すること。
