json_parse_manifest_incremental_chunk

定義

  • 種別: function
  • 定義箇所: src/common/parse_manifest.c:184

目的

バックアップマニフェスト（JSON形式）を分割されたチャンク単位でインクリメンタルに解析し、同時にSHA-256チェックサムの更新および検証を行う。

シグネチャまたは形状

void json_parse_manifest_incremental_chunk(
    JsonManifestParseIncrementalState *incstate,
    const char *chunk,
    size_t size,
    bool is_last
)

入力と出力

  • incstate: 解析状態を保持する構造体。
  • chunk: 解析対象のJSONデータ断片。
  • size: chunk のバイト数。
  • is_last: 最終チャンクであるかを示すフラグ。
  • 出力: なし（エラー時はコールバック経由で通知または異常終了）。

前提条件と不変条件

  • incstate は事前に json_parse_manifest_incremental_init で初期化されていること。
  • is_last が true の場合、解析終了後に内部状態が JM_EXPECT_EOF に到達している必要がある。

アルゴリズムまたは振る舞い

1. pg_parse_json_incremental を呼び出し、JSONの構文解析を実行する。 2. 解析結果が期待値（is_last なら JSON_SUCCESS、それ以外なら JSON_INCOMPLETE）と一致しない場合、json_manifest_parse_failure を呼び出す。 3. is_last が true で、かつ解析状態が JM_EXPECT_EOF でない場合、エラーとして処理する。 4. is_last が false の場合、pg_cryptohash_update を用いて現在のチャンクでチェックサムを更新する。 5. is_last が true の場合、verify_manifest_checksum を呼び出し、蓄積されたデータとマニフェスト内のチェックサムを照合する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/backup/basebackup_incremental.c
  • src/bin/pg_combinebackup/load_manifest.c
  • src/bin/pg_verifybackup/pg_verifybackup.c
  • 呼び出し先:
  • pg_parse_json_incremental
  • json_manifest_parse_failure
  • pg_cryptohash_update
  • verify_manifest_checksum

関連データ構造

  • JsonManifestParseIncrementalState: インクリメンタル解析のコンテキスト。
  • JsonManifestParseState: 解析中のセマンティック状態。
  • JsonManifestParseContext: コールバック関数群を保持する構造体。

失敗モードと境界条件

  • JSONの構文エラーが発生した場合、json_manifest_parse_failure により解析が中断される。
  • 最終チャンクでチェックサムの不一致が検出された場合、検証エラーとなる。
  • マニフェストが予期せず終了した場合（JM_EXPECT_EOF に達しない場合）はエラーとなる。

保守担当者向けメモ

  • 本関数はフロントエンド向けに設計されている。
  • json_manifest_parse_failure は pg_unreachable() を呼び出すため、エラー発生時は呼び出し元へ復帰しない。
  • 最終チャンクには必ずチェックサムが含まれている必要がある。
