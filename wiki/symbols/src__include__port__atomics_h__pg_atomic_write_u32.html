pg_atomic_write_u32

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics.h:275

目的

pg_atomic_uint32 型の変数に対して、アトミックに値を書き込みます。

シグネチャまたは形状

static inline void pg_atomic_write_u32(volatile pg_atomic_uint32 *ptr, uint32 val)

入力と出力

  • ptr: 書き込み対象となる pg_atomic_uint32 型変数へのポインタ（volatile 修飾あり）。
  • val: 書き込む 32bit 符号なし整数値。
  • 戻り値: なし。

前提条件と不変条件

  • ptr は 4 バイト境界にアライメントされている必要があります（AssertPointerAlignment により検証されます）。

アルゴリズムまたは振る舞い

1. AssertPointerAlignment を呼び出し、ポインタのアライメントを検証します。 2. プラットフォーム固有の実装である pg_atomic_write_u32_impl(ptr, val) を呼び出し、アトミックな書き込みを実行します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/clog.c
  • src/backend/commands/vacuumparallel.c
  • src/backend/postmaster/autovacuum.c
  • src/backend/storage/ipc/procarray.c
  • src/backend/storage/ipc/procsignal.c
  • 呼び出し先:
  • pg_atomic_write_u32_impl（プラットフォーム固有の実装へ委譲）

関連データ構造

  • pg_atomic_uint32: アトミック操作対象の 32bit 整数型。

失敗モードと境界条件

  • アライメントが 4 バイトでない場合、アサーションエラーが発生します。
  • ptr が指すメモリ領域が有効でない場合、未定義動作となります。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結します。
  • 実際の書き込み処理は pg_atomic_write_u32_impl に依存しているため、アーキテクチャ固有の挙動を確認する場合は各 arch-*.h ファイルを参照してください。
