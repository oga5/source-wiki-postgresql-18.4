s_unlock

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/s_lock.c:115

目的

スピンロックを解放し、ロック状態を解除します。

シグネチャまたは形状

void s_unlock(volatile slock_t *lock)

入力と出力

  • **入力**: lock - 解放対象となる slock_t 型のポインタ。volatile 修飾されており、コンパイラによる最適化を抑制します。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • 引数 lock は有効なメモリ領域を指している必要があります。
  • 本関数は、対応する s_lock 等によって既にロックが取得されている状態に対して呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. 引数 lock が指すメモリ位置に 0 を書き込みます。 2. これにより、ハードウェアレベルのロック状態がリセットされ、他のプロセスによるロック取得が可能になります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/storage/s_lock.h で定義される S_UNLOCK(lock) マクロを通じて呼び出されます。具体的な呼び出し箇所はソースコード全体に分散しています。
  • **呼び出し先**: なし。

関連データ構造

  • slock_t: src/include/storage/s_lock.h で定義される、プラットフォーム依存のスピンロック型。

失敗モードと境界条件

  • lock が NULL ポインタである場合、セグメンテーション違反が発生します。
  • ロックが保持されていない状態で呼び出された場合の動作は未定義です。

保守担当者向けメモ

  • 本関数は非常に単純な操作であるため、インライン化が推奨される場合があります。src/include/storage/s_lock.h 内の S_UNLOCK マクロ定義と整合性を保つ必要があります。
