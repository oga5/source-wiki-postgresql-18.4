winsock_strerror

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/win32.c:276

目的

Windows環境において、Winsockエラーコードを人間が読み取れる文字列に変換する。標準の strerror では解決できない WSA* 系エラーを補完する。

シグネチャまたは形状

const char *winsock_strerror(int err, char *strerrbuf, size_t buflen);

入力と出力

  • **入力**:
  • err: 変換対象のWinsockエラーコード。
  • strerrbuf: メッセージを格納する呼び出し元提供のバッファ。
  • buflen: strerrbuf のサイズ。
  • **出力**:
  • 変換されたエラーメッセージを含む strerrbuf へのポインタを返す。

前提条件と不変条件

  • strerrbuf は、エラーメッセージと付加情報を格納するために十分なサイズ（少なくとも64バイト以上の余裕）を確保している必要がある。
  • dlls 配列内のDLLは、一度ロードされるとプロセス終了まで保持される。

アルゴリズムまたは振る舞い

1. LookupWSErrorMessage を呼び出し、静的テーブル WSErrors からエラーメッセージの検索を試みる。 2. テーブルで見つからない場合、dlls 配列内の各DLLを LoadLibraryEx（LOAD_LIBRARY_AS_DATAFILE）で順次ロードし、FormatMessage API を用いてシステムからメッセージ取得を試みる。 3. 成功した場合、メッセージをバッファに書き込み、末尾にエラーコード（16進数および10進数）を付加する。 4. 全ての試行が失敗した場合は、「unrecognized socket error」という汎用メッセージをバッファに書き込む。 5. 最終的に strerrbuf を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/libpq-int.h にて SOCK_STRERROR マクロとして定義されており、libpq 内のソケットエラー処理で使用される。
  • **呼び出し先**: LookupWSErrorMessage, LoadLibraryEx, FormatMessage。

関連データ構造

  • WSErrors: 一般的なWinsockエラーコードを保持する静的テーブル。
  • dlls: エラーメッセージ取得のために検索対象となるWindows DLLのリスト。

失敗モードと境界条件

  • **変換失敗**: テーブルおよびシステムDLLのいずれからもメッセージを取得できない場合、unrecognized socket error メッセージが生成される。
  • **バッファ不足**: buflen が極端に小さい場合、FormatMessage の呼び出しや末尾のコード付加時に切り詰めが発生する可能性がある。

保守担当者向けメモ

  • 本関数はエラー発生時のみ実行されるため、線形探索やDLLロードのオーバーヘッドは許容範囲内とみなされている。
  • FormatMessage の呼び出し時に buflen - 64 を指定しており、エラーコード付加用の領域を確保している点に注意すること。
