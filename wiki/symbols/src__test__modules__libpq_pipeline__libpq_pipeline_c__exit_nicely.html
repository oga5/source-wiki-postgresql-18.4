exit_nicely

定義

  • 種別: function
  • 定義箇所: src/test/modules/libpq_pipeline/libpq_pipeline.c:58

目的

データベース接続を適切にクローズし、エラー終了コード（1）でプロセスを終了させるためのユーティリティ関数です。

シグネチャまたは形状

static void exit_nicely(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn - 終了前にクローズ対象となる libpq のデータベース接続ポインタ。
  • **出力**: なし（関数内で exit(1) を呼び出すため、呼び出し元には戻りません）。

前提条件と不変条件

  • 引数 conn は PQfinish に渡すことが可能な有効なポインタである必要があります。

アルゴリズムまたは振る舞い

1. PQfinish(conn) を呼び出し、データベース接続を解放します。 2. exit(1) を呼び出し、異常終了ステータスでプログラムを終了します。

呼び出し元と呼び出し先

  • **呼び出し先**: PQfinish (libpq API), exit (標準ライブラリ)。
  • **呼び出し元**: 提供されたソースコード内での直接的な呼び出し元は特定されていません。
  • **注意**: ツールが検出した src/bin/pg_dump 等の参照は、本ファイル内の exit_nicely とは名前が同一の別実装である可能性が高く、本シンボルの呼び出し元とは断定できません。

関連データ構造

  • PGconn: libpq の接続状態を保持する構造体。

失敗モードと境界条件

  • conn が NULL の場合、PQfinish の仕様に従い、何も行わずに終了処理へ進みます。

保守担当者向けメモ

  • 本関数は libpq_pipeline.c 内のテスト実行中に致命的なエラーが発生した際のクリーンアップ用として定義されています。
  • 他のモジュール（pg_dump 等）にも同名の関数が存在しますが、それらとは実装が異なるため混同しないよう注意してください。
