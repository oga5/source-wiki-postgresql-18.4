GlobalVisState

定義

  • **種別**: struct
  • **定義箇所**: src/backend/storage/ipc/procarray.c:167
  • **関連定義**: utils/snapmgr.h:98 にも同名の構造体定義が存在します。

目的

MVCC（多版同時実行制御）において、特定のトランザクションID（XID）がどのバックエンドから必要とされているか、あるいは不要であるかを判定するための可視性境界を保持します。

シグネチャまたは形状

struct GlobalVisState
{
    FullTransactionId definitely_needed;
    FullTransactionId maybe_needed;
};

入力と出力

本構造体は、可視性判定の境界値として FullTransactionId 型の値を保持します。

  • definitely_needed: この値以上のXIDは、いずれかのバックエンドによって実行中とみなされます。
  • maybe_needed: この値未満のXIDは、どのバックエンドからも実行中とはみなされません。

前提条件と不変条件

  • definitely_needed および maybe_needed は FullTransactionId 型であり、トランザクションの進行に伴い更新されます。
  • 共有リレーション、カタログリレーション、通常リレーション、一時リレーションごとに個別の状態管理が想定されています。

アルゴリズムまたは振る舞い

本構造体は、HeapTupleIsSurelyDead などの可視性判定関数において、タプルが削除可能かどうかを判断するための閾値として利用されます。VACUUM処理やヒープの剪定（pruning）プロセスにおいて、現在の可視性状態を照合するために参照されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/heapam_visibility.c (HeapTupleIsSurelyDead)
  • src/backend/access/heap/pruneheap.c
  • src/backend/access/heap/vacuumlazy.c
  • src/backend/access/spgist/spgvacuum.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • FullTransactionId: 可視性判定に使用される64ビットトランザクションID。
  • GlobalVisSharedRels / GlobalVisCatalogRels: procarray.c 内で定義されている本構造体の静的インスタンス。

失敗モードと境界条件

  • VACUUM開始後に GlobalVisState が更新された場合、古い状態に基づいた判定は不正確になる可能性があります。
  • 共有メモリ上の状態と同期していない場合、誤った可視性判定が行われるリスクがあります。

保守担当者向けメモ

  • 本構造体は procarray.c と snapmgr.h の両方で定義されています。変更を加える際は両方の定義の一貫性に注意してください。
  • HeapTupleIsSurelyDead 等の関数で利用される際、最新の可視性境界が反映されていることを確認してください。
