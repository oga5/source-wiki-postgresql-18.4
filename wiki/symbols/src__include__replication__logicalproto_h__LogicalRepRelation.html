LogicalRepRelation

定義

  • **種別**: struct
  • **定義箇所**: src/include/replication/logicalproto.h:104

目的

論理レプリケーションにおいて、パブリッシャーから送信されたリモート側のテーブル定義情報を保持するための構造体です。サブスクライバー側で受信したデータをローカルのテーブル定義と照合するために使用されます。

シグネチャまたは形状

typedef struct LogicalRepRelation
{
    LogicalRepRelId remoteid;   /* リモート側のリレーションID */
    char       *nspname;        /* スキーマ名 */
    char       *relname;        /* テーブル名 */
    int         natts;          /* カラム数 */
    char      **attnames;       /* カラム名の配列 */
    Oid        *atttyps;        /* カラム型のOID配列 */
    char        replident;      /* レプリカアイデンティティ */
    char        relkind;        /* リモート側のリレーション種別 */
    Bitmapset  *attkeys;        /* キーカラムのビットマップ */
} LogicalRepRelation;

入力と出力

本構造体は、主に logicalrep_read_rel 関数（src/backend/replication/logical/proto.c）によってストリームからデコードされ、生成されます。

前提条件と不変条件

  • remoteid はリモート側で一意である必要があります。
  • attnames および atttyps は natts の数だけ要素を持つ配列である必要があります。
  • 本構造体に含まれる情報はリモート側の定義に基づいているため、ローカル側のテーブル定義と一致するとは限りません。

アルゴリズムまたは振る舞い

論理レプリケーションのワーカープロセスが、パブリッシャーから送られてくるリレーション定義メッセージを読み取る際に、この構造体をメモリ上に確保し、各フィールドを埋めます。その後、この情報を基にローカルのテーブルとのマッピング（logicalrep_relmap_update）や、カラム名の解決、属性の検証が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/proto.c: ストリームからの読み取り時に使用。
  • src/backend/replication/logical/relation.c: リレーションマッピングの更新や属性解決時に使用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • LogicalRepRelId: リレーションの識別子。
  • Bitmapset: キーカラムの特定に使用。
  • Relation (utils/rel.h): ローカル側のテーブル定義情報。

失敗モードと境界条件

  • リモート側とローカル側でカラム数（natts）や型（atttyps）が不一致の場合、レプリケーション適用時にエラーが発生する可能性があります。
  • メモリ不足時には palloc による構造体確保が失敗します。

保守担当者向けメモ

  • 本構造体はワイヤプロトコルに直結しています。フィールドの追加や変更を行う際は、プロトコルバージョン（LOGICALREP_PROTO_VERSION_NUM）の整合性に注意してください。
  • attnames や atttyps などのポインタフィールドは、動的に割り当てられるため、使用後のメモリ解放（pfree）が適切に行われているか確認してください。
