LockViewRecurse_walker

定義

  • 種別: function
  • 定義箇所: src/backend/commands/lockcmds.c:176

目的

ビューのクエリツリーを走査し、ビュー定義内で参照されているリレーション（テーブル、ビュー）を特定して、それらに対して再帰的にロックを適用する。

シグネチャまたは形状

static bool LockViewRecurse_walker(Node *node, LockViewRecurse_context *context)

入力と出力

  • **入力**:
  • node: 走査対象のノード（Query または式ツリー）。
  • context: ロックモード、ユーザー権限、再帰済みビューのリスト、nowait フラグを含む LockViewRecurse_context 構造体。
  • **出力**:
  • bool: 走査を継続する場合は false、停止する場合は true を返す（walker パターンの慣習に従う）。

前提条件と不変条件

  • context->ancestor_views には、循環参照を防ぐために既にロック済みのビューの OID が保持されている必要がある。
  • 走査対象のリレーションがテーブル、パーティションテーブル、またはビューである場合にのみロック処理が行われる。

アルゴリズムまたは振る舞い

1. **ノード判定**: Query ノードの場合、その rtable（Range Table）を反復処理する。 2. **リレーション確認**: 各 RangeTblEntry に対して、対象がテーブルまたはビューかを確認する。 3. **循環チェック**: ancestor_views に含まれる OID であればスキップする。 4. **権限検証**: LockTableAclCheck を呼び出し、現在のユーザー（またはビュー所有者）が対象リレーションをロックする権限を持つか検証する。 5. **ロック適用**:

  • nowait が偽であれば LockRelationOid を実行。
  • nowait が真であれば ConditionalLockRelationOid を実行し、失敗時はエラーを投げる。

    • 6. **再帰呼び出し**:

  • 対象がビューであれば LockViewRecurse を呼び出す。
  • 対象が継承テーブルであれば LockTableRecurse を呼び出す。

    • 7. **ツリー走査**: query_tree_walker または expression_tree_walker を用いて、クエリツリー全体を再帰的に探索する。

呼び出し元と呼び出し先

  • **呼び出し元**: LockViewRecurse から呼び出される（src/backend/commands/lockcmds.c:269）。また、自身を再帰的に呼び出す。
  • **呼び出し先**: LockTableAclCheck, LockRelationOid, ConditionalLockRelationOid, LockViewRecurse, LockTableRecurse, query_tree_walker, expression_tree_walker。

関連データ構造

  • LockViewRecurse_context: 走査状態を保持する構造体。
  • Query: PostgreSQL のクエリ表現。
  • RangeTblEntry: クエリ内のリレーション参照。

失敗モードと境界条件

  • **権限不足**: LockTableAclCheck が ACLCHECK_OK 以外を返した場合、aclcheck_error によりエラーが発生する。
  • **ロック競合**: nowait 指定時にロックが取得できない場合、ERRCODE_LOCK_NOT_AVAILABLE エラーが発生する。
  • **対象外リレーション**: テーブル、パーティションテーブル、ビュー以外のリレーションは無視される。

保守担当者向けメモ

  • 本関数は query_tree_walker のコールバックとして機能する。
  • 循環参照の防止は ancestor_views リストへの依存に完全に依存しているため、このリストの管理ロジックを変更する際は注意が必要である。
