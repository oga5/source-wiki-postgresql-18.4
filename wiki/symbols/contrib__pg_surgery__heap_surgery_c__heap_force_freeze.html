heap_force_freeze

定義

  • 種別: function
  • 定義箇所: contrib/pg_surgery/heap_surgery.c:72

目的

SQL インターフェースを通じて、指定されたヒープタプルを強制的にフリーズ（Frozen）状態に変更する。

シグネチャまたは形状

Datum heap_force_freeze(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL から渡されるリレーション OID および TID 配列）。
  • **出力**: Datum（heap_force_common の戻り値）。

前提条件と不変条件

  • 実行者が対象リレーションの所有者であるか、スーパーユーザーであること。
  • RecoveryInProgress() が偽であること（リカバリ中は実行不可）。
  • 対象リレーションに対して RowExclusiveLock が取得されていること。

アルゴリズムまたは振る舞い

1. fcinfo を引数として heap_force_common を呼び出す。 2. その際、操作種別として HEAP_FORCE_FREEZE を指定する。 3. heap_force_common 内で TID 配列の検証、バッファのロック、タプルヘッダーへの FrozenTransactionId の書き込み、および WAL 記録が行われる。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（SELECT heap_force_freeze(...)）。
  • **呼び出し先**: heap_force_common（実処理の共通関数）。

関連データ構造

  • HeapTupleForceOption: 操作種別を定義する列挙型（HEAP_FORCE_FREEZE を使用）。

失敗モードと境界条件

  • 対象がヒープテーブルでない場合、または権限がない場合にエラーが発生する。
  • 破損が深刻でページ構造自体が読み込めない場合、バッファ管理層でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は整合性チェックをバイパスする低レベル操作である。誤用はデータベースの深刻な破損を招くため、緊急時の最終手段としてのみ使用すること。
  • 変更は WAL に記録されるため、レプリケーションや PITR に影響を与える。
