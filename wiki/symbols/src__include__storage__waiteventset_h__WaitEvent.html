WaitEvent

定義

  • 種別: struct
  • 定義箇所: src/include/storage/waiteventset.h:59

目的

WaitEventSet を介した待機操作において、発生したイベントの詳細情報を呼び出し元へ伝達するために使用される構造体です。

シグネチャまたは形状

typedef struct WaitEvent
{
	int			pos;			/* イベントデータ構造体内の位置 */
	uint32		events;			/* 発生したイベントのビットマスク */
	pgsocket	fd;				/* イベントに関連付けられたソケット記述子 */
	void	   *user_data;		/* AddWaitEventToSet で指定されたポインタ */
#ifdef WIN32
	bool		reset;			/* イベントのリセットが必要か否か */
#endif
} WaitEvent;

入力と出力

  • 本構造体は WaitEventSetWait() の呼び出し結果として、発生したイベントの情報を格納する出力用コンテナとして利用されます。
  • user_data フィールドは、イベント登録時に呼び出し元が提供した任意のデータを保持します。

前提条件と不変条件

  • WaitEventSetWait() を呼び出す前に、適切なサイズの WaitEvent 配列を確保しておく必要があります。
  • pos はイベントセット内での一意なインデックスを指します。

アルゴリズムまたは振る舞い

1. WaitEventSetWait() がイベントを検知すると、対応する WaitEvent 構造体の各フィールドが更新されます。 2. 呼び出し元は、戻り値として得られた WaitEvent 配列を走査し、events ビットマスクを確認して、ソケットの読み書きやラッチのセットといった必要な処理を分岐実行します。 3. Windows 環境では、reset フラグが true の場合にイベントのリセット処理が必要となる場合があります。

  • **呼び出し先**:
  • 本構造体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • WaitEventSet: 待機対象を管理する不透明な構造体。
  • WL_* マクロ: events フィールドに格納されるビットマスクの定義群。

失敗モードと境界条件

  • 構造体自体に失敗モードはありませんが、WaitEventSetWait() が返す events が 0 の場合、タイムアウトまたは無効なイベントとして扱われる可能性があります。
  • Windows 固有の reset フィールドは、プラットフォーム依存の挙動を制御するため、クロスプラットフォーム実装時には注意が必要です。

保守担当者向けメモ

  • WaitEvent は WaitEventSet の内部実装と密接に結合しています。waiteventset.c 内の WaitEventAdjustEpoll 等の内部関数でどのように更新されるかを確認してください。
  • user_data は型安全ではないため、キャスト時の整合性に注意してください。
