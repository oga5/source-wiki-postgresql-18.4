CreateInitDecodingContext

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/logical.c:331

目的

論理デコーディングを開始するための LogicalDecodingContext を初期化し、レプリケーションスロットの設定および安全なトランザクション開始点の計算を行う。

シグネチャまたは形状

LogicalDecodingContext *CreateInitDecodingContext(
    const char *plugin,
    List *output_plugin_options,
    bool need_full_snapshot,
    XLogRecPtr restart_lsn,
    XLogReaderRoutine *xl_routine,
    LogicalOutputPluginWriterPrepareWrite prepare_write,
    LogicalOutputPluginWriterWrite do_write,
    LogicalOutputPluginWriterUpdateProgress update_progress
)

入力と出力

  • **入力**: 出力プラグイン名、オプションリスト、スナップショット要否フラグ、開始LSN、WAL読み取りルーチン、および出力用コールバック関数群。
  • **出力**: 初期化済みの LogicalDecodingContext 構造体へのポインタ。

前提条件と不変条件

  • MyReplicationSlot が取得済みであること。
  • 物理レプリケーションスロットではないこと。
  • スロットが現在のデータベースで作成されていること。
  • 書き込みを伴うアクティブなトランザクション内ではないこと。
  • CheckLogicalDecodingRequirements() による要件チェックを通過すること。

アルゴリズムまたは振る舞い

1. **検証**: スロットの有効性とプラグインの指定を確認。 2. **スロット更新**: 指定されたプラグイン名をスロットに登録し、restart_lsn を設定（未指定時は ReplicationSlotReserveWal を実行）。 3. **xmin 境界計算**: ReplicationSlotControlLock と ProcArrayLock を排他制御下で取得し、GetOldestSafeDecodingTransactionId を用いて安全な xmin を算出。スロットの catalog_xmin 等を更新し、永続化する。 4. **コンテキスト生成**: StartupDecodingContext を呼び出し、デコーディング基盤を構築。 5. **プラグイン初期化**: 出力プラグインの startup_cb が定義されていれば実行。 6. **状態調整**: スロットの二相コミット対応状況に基づき、コンテキストの twophase フラグを調整。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/slotfuncs.c
  • src/backend/replication/walsender.c
  • **呼び出し先**:
  • CheckLogicalDecodingRequirements
  • namestrcpy, SpinLockAcquire/Release
  • ReplicationSlotReserveWal, ReplicationSlotSave, ReplicationSlotMarkDirty
  • LWLockAcquire/Release (ReplicationSlotControlLock, ProcArrayLock)
  • GetOldestSafeDecodingTransactionId
  • StartupDecodingContext
  • startup_cb_wrapper

関連データ構造

  • LogicalDecodingContext: 本関数が生成する主要構造体。
  • ReplicationSlot: 現在のレプリケーションスロット状態。
  • XLogReaderRoutine: WAL読み取りインターフェース。

失敗モードと境界条件

  • スロット未取得、プラグイン未指定、物理スロットの使用、データベース不一致、またはトランザクション内での実行時に ERROR を発生させる。
  • restart_lsn が無効な場合はWALの予約を試みる。

保守担当者向けメモ

  • 本関数はスロットの xmin 境界を計算する際、ProcArrayLock を保持して競合を防ぐ重要なセクションを含む。
  • startup_cb の実行時はメモリコンテキストを切り替えており、プラグイン側のメモリ管理と整合性を保つ必要がある。
