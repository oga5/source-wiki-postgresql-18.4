float4in_internal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/float.c:182

目的

文字列形式の入力を float4（単精度浮動小数点数）型に変換する、プラットフォーム非依存の内部解析ルーチンです。

シグネチャまたは形状

float4 float4in_internal(char *num, char **endptr_p, const char *type_name, const char *orig_string, struct Node *escontext)

入力と出力

  • **入力**:
  • num: 解析対象の文字列。
  • endptr_p: 解析終了位置を格納するポインタ（NULL可）。
  • type_name: エラーメッセージ用型名。
  • orig_string: エラーメッセージ用元の入力文字列。
  • escontext: エラー報告用コンテキスト。
  • **出力**:
  • 変換された float4 値。

前提条件と不変条件

  • 入力文字列の先頭の空白は自動的にスキップされます。
  • strtof() を使用して変換を試みますが、プラットフォームによる挙動の差異を吸収するため、特定の特殊値（NaN, Infinity等）は手動で判定します。

アルゴリズムまたは振る舞い

1. 先頭の空白をスキップします。 2. 空文字列の場合はエラーを報告します。 3. strtof() を呼び出し、変換を試みます。 4. strtof() が失敗した場合（endptr == num または errno != 0）、以下のケースを順次判定します：

  • "NaN", "Infinity", "inf"（符号付きを含む）の文字列パターンを pg_strncasecmp で検出し、対応する float4 値を割り当てます。
  • ERANGE エラー発生時、値が 0.0 または無限大であれば範囲外エラーとして処理します。

    • 5. 変換後、末尾の空白をスキップします。 6. endptr_p が指定されていれば終了位置を返し、指定がない場合は文字列の末尾まで解析されたことを確認します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/float.c 内の float4in（PG_RETURN_FLOAT4 マクロ経由）。
  • **呼び出し先**:
  • strtof() (標準Cライブラリ)
  • pg_strncasecmp()
  • get_float4_nan(), get_float4_infinity()
  • ereturn() (エラー報告)

関連データ構造

  • float4: PostgreSQLの単精度浮動小数点型。
  • Node: エラーコンテキスト構造体。

失敗モードと境界条件

  • **無効な構文**: 数値として解釈できない文字列は ERRCODE_INVALID_TEXT_REPRESENTATION でエラーとなります。
  • **範囲外**: float4 の表現範囲を超える数値は ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE でエラーとなります。
  • **空入力**: 空文字列はエラーとなります。

保守担当者向けメモ

  • 本関数は float8in_internal と同様のロジックを共有しています。修正時は両方の整合性を確認してください。
  • strtof() のプラットフォーム依存性を排除するため、特殊値の判定ロジックは厳格に維持されています。
