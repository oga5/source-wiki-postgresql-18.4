ecpg_realloc

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/memory.c:32

目的

標準の realloc をラップし、メモリ割り当て失敗時にECPG固有のエラー処理を統合して実行する。

シグネチャまたは形状

char *ecpg_realloc(void *ptr, long size, int lineno)

入力と出力

  • **入力**:
  • ptr: 再割り当て対象のメモリブロックへのポインタ。
  • size: 要求する新しいサイズ（バイト単位）。
  • lineno: エラー発生時に報告されるソースコードの行番号。
  • **出力**:
  • 成功時: 再割り当てされたメモリブロックへのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • ptr は malloc 等で確保された有効なメモリブロック、または NULL である必要がある。
  • 割り当て失敗時には、内部で ecpg_raise が呼び出され、エラー状態が設定される。

アルゴリズムまたは振る舞い

1. 標準の realloc(ptr, size) を呼び出し、結果を保持する。 2. 戻り値が NULL（メモリ不足）の場合:

  • ecpg_raise を呼び出し、ECPG_OUT_OF_MEMORY エラーと ECPG_SQLSTATE_ECPG_OUT_OF_MEMORY SQLSTATE を記録する。
  • NULL を返す。

    • 3. 戻り値が有効なポインタである場合、そのポインタをそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/execute.c: SQL実行時のバッファ拡張やパラメータ配列の再確保に使用。
  • **呼び出し先**:
  • realloc (標準ライブラリ)
  • ecpg_raise (エラーハンドリング)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • size が 0 の場合、realloc の標準的な挙動に従う（実装依存）。
  • メモリ不足時には NULL を返し、呼び出し元が適切にエラーハンドリングを行う必要がある。

保守担当者向けメモ

  • 本関数は ecpglib 内のメモリ管理の基盤であり、直接的なメモリ追跡（ecpg_auto_alloc 等）とは異なり、単なるラッパーとして機能する。
  • 呼び出し元で lineno を正確に渡すことで、エラー発生時のデバッグ情報を維持している。
