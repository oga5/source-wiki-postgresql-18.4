populate_with_dummy_strings

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_bloomfilter/test_bloomfilter.c:31

目的

ブルームフィルタのテストにおいて、指定された個数のダミー文字列をフィルタへ投入し、検証用のデータセットを構築する。

シグネチャまたは形状

static void populate_with_dummy_strings(bloom_filter *filter, int64 nelements)

入力と出力

  • **入力**:
  • filter: 投入対象となる bloom_filter 構造体へのポインタ。
  • nelements: 投入する要素の総数（int64）。
  • **出力**: なし（void）。

前提条件と不変条件

  • filter は bloom_create 等で適切に初期化されている必要がある。
  • 投入される文字列は "i" + 数値 の形式で生成される。

アルゴリズムまたは振る舞い

1. nelements 回のループを実行する。 2. 各ループの開始時に CHECK_FOR_INTERRUPTS() を呼び出し、割り込み要求を監視する。 3. snprintf を使用して、現在のインデックス i を含む文字列 "i<i_value>" を生成する。 4. bloom_add_element を呼び出し、生成した文字列をブルームフィルタに追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_bloomfilter/test_bloomfilter.c 内の create_and_test_bloom 関数から呼び出される（92行目）。
  • **呼び出し先**:
  • CHECK_FOR_INTERRUPTS()
  • snprintf()
  • bloom_add_element()
  • strlen()

関連データ構造

  • bloom_filter: lib/bloomfilter.h で定義されるブルームフィルタの管理構造体。

失敗モードと境界条件

  • nelements が極端に大きい場合、ループ処理が長時間に及ぶ可能性があるが、CHECK_FOR_INTERRUPTS() によりSQL側からのキャンセルが可能。
  • element バッファのサイズ（MAX_ELEMENT_BYTES）を超える文字列が生成される場合、snprintf の仕様に従い切り詰めが発生する可能性がある。

保守担当者向けメモ

  • 本関数はテスト専用のユーティリティである。
  • 偽陽性率の測定を行う nfalsepos_for_missing_strings 関数と対になる存在であり、テストの整合性を保つために文字列生成フォーマット（"i" プレフィックス）を変更する際は両方の関数を更新すること。
