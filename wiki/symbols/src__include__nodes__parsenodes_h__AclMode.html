AclMode

定義

  • 種別: typedef uint64
  • 定義箇所: src/include/nodes/parsenodes.h:74

目的

PostgreSQLにおける権限（Privilege）を表現するためのビットマスク型です。

シグネチャまたは形状

uint64 型のビットフィールドとして定義されています。

入力と出力

  • **入力**: 権限チェックや付与を行う関数において、要求される権限ビットの組み合わせとして入力されます。
  • **出力**: 権限の有無や種類を判定する関数において、現在の権限状態を示すビットマスクとして返されます。

前提条件と不変条件

  • N_ACL_RIGHTS（15）は、定義されている権限ビットの最大数（1 + 最後のビットシフト値）を示します。
  • ACL_NO_RIGHTS は 0 であり、権限がない状態を指します。
  • ACL_SELECT_FOR_UPDATE は ACL_UPDATE と同値として扱われます。

アルゴリズムまたは振る舞い

各ビットは特定の操作に対応しています。

  • ACL_INSERT (1<<0) ～ ACL_MAINTAIN (1<<14) までのビットが定義されており、リレーション、関数、データベース、設定パラメータ等のオブジェクトに対する操作権限を表現します。
  • 権限の判定や付与は、これらのビットに対するビット演算によって行われます。

呼び出し元と呼び出し先

  • **呼び出し元（例）**:
  • contrib/dblink/dblink.c: get_rel_from_relname 関数が引数として使用。
  • src/backend/catalog/aclchk.c: ExecGrant_common や restrict_and_check_grant 等の権限チェック関数で使用。
  • **呼び出し先**:
  • 本シンボル自体は型定義であり、直接的な呼び出し先はありません。関連する操作関数（string_to_privilege や privilege_to_string 等）が aclchk.c 内で定義されています。

関連データ構造

  • InternalGrant: 権限付与操作を保持する構造体（aclchk.c で使用）。
  • Bitmapset: 権限管理に関連して使用されるビットマップ構造体。

失敗モードと境界条件

  • 定義されているビット範囲（0〜14）を超えた値の使用は、予期しない動作を引き起こす可能性があります。
  • ACL_NO_RIGHTS（0）は、いかなる権限も持たない状態を明示します。

保守担当者向けメモ

  • 新しい権限タイプを追加する場合は、ビット位置の重複を避け、N_ACL_RIGHTS を適切に更新してください。
  • uint64 を使用しているため、最大64ビットまでの拡張が可能ですが、現在の実装では15ビットまでが使用されています。
