UnlockRelationId

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lmgr.c:213

目的

指定されたリレーション識別子（LockRelId）に対応するロックを解放します。

シグネチャまたは形状

void UnlockRelationId(LockRelId *relid, LOCKMODE lockmode)

入力と出力

  • **入力**:
  • relid: 解放対象のリレーションを特定する LockRelId 構造体へのポインタ。
  • lockmode: 解放するロックのモード（LOCKMODE）。
  • **出力**: なし（void）。

前提条件と不変条件

  • relid が指す構造体は有効である必要があります。
  • 指定された lockmode で、呼び出し元が既にロックを保持していることが前提です。

アルゴリズムまたは振る舞い

1. SET_LOCKTAG_RELATION マクロを使用して、relid（データベースIDとリレーションID）から LOCKTAG 構造体を構築します。 2. LockRelease 関数を呼び出し、構築したタグと指定されたロックモードでロックを解放します。第3引数には false が渡され、セッションロックではなく通常のロック解放として処理されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/relation.c
  • src/backend/access/index/indexam.c
  • src/backend/commands/dbcommands.c
  • **呼び出し先**:
  • LockRelease（src/backend/storage/lmgr/lock.c 内の関数と推測される）

関連データ構造

  • LockRelId: リレーションを識別するための構造体。
  • LOCKTAG: ロック対象を一意に特定するための構造体。

失敗モードと境界条件

  • 保持していないロックを解放しようとした場合、LockRelease の内部ロジックによりエラーが発生する可能性があります。

保守担当者向けメモ

  • リレーションの OID のみが既知の場合は、UnlockRelationOid の使用が推奨されますが、LockRelId が利用可能な場合は本関数を使用してください。
