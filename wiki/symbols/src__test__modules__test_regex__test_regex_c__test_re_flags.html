test_re_flags

定義

  • 種別: struct
  • 定義箇所: src/test/modules/test_regex/test_regex.c:26

目的

PostgreSQL の正規表現エンジン（Spencer's regex）に対するコンパイルおよび実行時のテスト用オプションを保持する構造体です。

シグネチャまたは形状

typedef struct test_re_flags
{
    int         cflags;     /* コンパイルフラグ */
    int         eflags;     /* 実行フラグ */
    long        info;       /* 期待される re_info ビット */
    bool        glob;       /* 全出現箇所を対象とするか */
    bool        indices;    /* 文字列ではなくインデックスを報告するか */
    bool        partial;    /* 部分一致を期待するか */
} test_re_flags;

入力と出力

本構造体は、parse_test_flags 関数によって入力文字列（text *opts）からフラグ設定が読み込まれ、その後、正規表現のコンパイル（test_re_compile）や実行（test_re_execute）の制御パラメータとして利用されます。

前提条件と不変条件

  • cflags および eflags は、PostgreSQL の regex/regex.h で定義された正規表現エンジンが解釈可能なビットフラグである必要があります。
  • info フィールドは、正規表現のメタ情報検証に使用されます。

アルゴリズムまたは振る舞い

テストハーネスにおいて、正規表現の挙動をカスタマイズするために使用されます。特に glob が真の場合、マッチングを一度で終了せず、対象文字列内のすべての出現箇所を反復処理する制御フローを決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: test_regex（メイン関数）、parse_test_flags、test_re_compile、test_re_execute、build_test_info_result 等の関数内で参照・更新されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • regex_t: cflags を用いてコンパイルされた正規表現パターンを保持します。
  • test_regex_ctx: テスト実行時のコンテキストとして、本構造体を内部に保持します。

失敗モードと境界条件

  • parse_test_flags において、サポートされていないフラグ文字が指定された場合、エラーが発生する可能性があります。
  • cflags や eflags に無効なビット組み合わせを指定した場合、正規表現エンジン側で予期せぬ挙動やエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は Tcl の正規表現テストスイートとの互換性を維持するように設計されています。フラグの追加や変更を行う際は、parse_test_flags の実装と整合性を取る必要があります。
