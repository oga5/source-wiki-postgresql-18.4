insert_username

定義

  • 種別: function
  • 定義箇所: contrib/spi/insert_username.c:24

目的

PostgreSQL のトリガー関数として、行の挿入または更新時に、指定されたカラムへ現在のセッションユーザー名を自動的に格納する。

シグネチャまたは形状

Datum insert_username(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（トリガー実行コンテキスト TriggerData を含む）。トリガー定義の引数として、更新対象のカラム名を1つ受け取る。
  • **出力**: Datum（更新された HeapTuple を指すポインタ）。

前提条件と不変条件

  • BEFORE トリガーとして実行される必要がある。
  • 行レベルのトリガーである必要がある。
  • 対象カラムは TEXT 型でなければならない。
  • トリガー定義には正確に1つの引数（カラム名）が必要である。

アルゴリズムまたは振る舞い

1. **コンテキスト検証**: CALLED_AS_TRIGGER 等を用いて、トリガーマネージャから正しく呼び出されたか、行レベルかつ BEFORE イベントであるかを検証する。 2. **イベント判定**: INSERT または UPDATE イベントであることを確認し、対象タプルを取得する。DELETE イベントはエラーとする。 3. **引数解析**: トリガー引数から対象カラム名を特定し、テーブル定義からその属性番号を取得する。 4. **型チェック**: 対象カラムが TEXT 型であることを確認する。 5. **値の生成**: GetUserNameFromId で現在のユーザー名を取得し、CStringGetTextDatum で変換する。 6. **タプル構築**: heap_modify_tuple_by_cols を使用して、対象カラムを新しいユーザー名で書き換えたタプルを生成し、返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のトリガーマネージャ（直接の呼び出し元情報はソースコード上では明示されていない）。
  • **呼び出し先**: SPI_getrelname, SPI_fnumber, SPI_gettypeid, GetUserNameFromId, GetUserId, CStringGetTextDatum, heap_modify_tuple_by_cols, pfree 等の PostgreSQL 内部関数。

関連データ構造

  • TriggerData: トリガー実行時のコンテキスト情報。
  • Trigger: トリガー定義情報。
  • HeapTuple: 操作対象の行データ。
  • TupleDesc: テーブルの属性記述子。

失敗モードと境界条件

  • **エラー**:
  • トリガーとして呼び出されていない場合。
  • 行レベル以外のトリガーとして呼び出された場合。
  • AFTER トリガーとして呼び出された場合。
  • DELETE イベントで呼び出された場合。
  • 引数の数が1つでない場合。
  • 指定されたカラム名がテーブルに存在しない場合。
  • 指定されたカラムが TEXT 型でない場合。

保守担当者向けメモ

  • 本関数は BEFORE トリガー専用である。AFTER トリガーで使用してもタプルの書き換えは行われない。
  • autoinc.c と同様のトリガー検証ロジックを採用しているため、修正時は整合性に注意すること。
