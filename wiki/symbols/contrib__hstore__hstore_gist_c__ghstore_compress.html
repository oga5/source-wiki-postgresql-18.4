ghstore_compress

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_gist.c:147

目的

hstore データを GiST インデックスに格納するためのシグネチャ（ビットベクトル）へ変換します。

シグネチャまたは形状

Datum ghstore_compress(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体を受け取ります。
  • **出力**: 圧縮されたシグネチャを含む新しい GISTENTRY へのポインタを Datum として返します。

前提条件と不変条件

  • GET_SIGLEN() により取得されるシグネチャ長（siglen）が、インデックス全体で一貫している必要があります。
  • リーフノードの場合、hstore 内の各キーおよび値（存在する場合）が CRC-32 でハッシュ化され、ビットベクトルにマッピングされます。

アルゴリズムまたは振る舞い

1. **リーフノード処理**: entry->leafkey が真の場合、hstore の各キーと値を crc32_sz でハッシュ化し、HASH マクロを用いて GISTTYPE 内のビットベクトルに該当ビットをセットします。その後、gistentryinit を用いて新しい GISTENTRY を作成します。 2. **非リーフノード処理**: entry->leafkey が偽の場合、既存のシグネチャが既に全ビットセット（ISALLTRUE）されているかを確認します。全ビットがセットされていない場合は元のエントリを返し、セットされている場合は全ビットが有効な新しい GISTTYPE を作成して返します。 3. **戻り値**: 処理された GISTENTRY を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス構築プロセスから呼び出されます（ソースコード内の PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: ghstore_alloc, crc32_sz, HASH, gistentryinit, DatumGetHStoreP 等の内部関数およびマクロを呼び出します。

関連データ構造

  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。
  • GISTTYPE: インデックスに格納されるシグネチャを保持する構造体。
  • HStore: 圧縮対象の hstore データ構造。

失敗モードと境界条件

  • hstore の値が NULL の場合、その値のハッシュ化はスキップされます。
  • siglen の設定により衝突率が変化します。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数は GiST インデックスの圧縮フェーズを担当します。
  • recheck が必要なインデックスであるため、この関数による圧縮はあくまでフィルタリングのための近似表現であることを考慮してください。
