ForEachState

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/pg_list.h:73

目的

PostgreSQLのリスト構造を走査する際のイテレータ状態を保持するために使用されます。

シグネチャまたは形状

typedef struct ForEachState
{
	const List *l;				/* 走査対象のリスト */
	int			i;				/* 現在の要素インデックス */
} ForEachState;

入力と出力

  • **入力**: 走査対象の List ポインタおよび開始位置（インデックスまたはセル）。
  • **出力**: 走査状態を保持する構造体インスタンス。

前提条件と不変条件

  • l は有効な List 構造体、または NIL を指す必要があります。
  • i はリストのインデックスとして有効な範囲内である必要があります。

アルゴリズムまたは振る舞い

foreach マクロおよび関連する走査マクロの内部状態として機能します。リストの走査時に i を順次インクリメントし、現在の ListCell を特定するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: foreach、for_each_from_setup、for_each_cell_setup などのリスト走査用マクロから直接インスタンス化されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • List: 走査対象となる汎用リスト構造体。
  • ListCell: リスト内の各要素を保持する共用体。

失敗モードと境界条件

  • 空リスト（NIL）に対して走査を開始した場合、i がリストの長さを超えるため、ループ処理は即座に終了します。
  • 走査中にリストの要素を削除する場合、通常の削除関数を使用するとイテレータの整合性が崩れるため、foreach_delete_current を使用する必要があります。

保守担当者向けメモ

  • 本構造体は src/include/nodes/pg_list.h 内の反復処理マクロ群の基盤です。変更を加える際は、関連するすべての foreach 系マクロの動作に影響を与えるため注意してください。
