trackitem_compare_frequencies_desc

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_typanalyze.c:517

目的

TrackItem 構造体のポインタ配列を、各アイテムの出現頻度（frequency）に基づいて降順にソートするための比較関数を提供します。

シグネチャまたは形状

static int trackitem_compare_frequencies_desc(const void *e1, const void *e2, void *arg)

入力と出力

  • **入力**:
  • e1, e2: ソート対象の TrackItem 構造体へのポインタを指すポインタ (const TrackItem *const *)。
  • arg: 使用されない引数（void *）。
  • **出力**:
  • int: e1 の頻度が e2 より小さい場合は正の値、大きい場合は負の値、等しい場合は 0 を返します。

前提条件と不変条件

  • e1 および e2 は、有効な TrackItem 構造体を指すポインタの配列要素である必要があります。
  • TrackItem 構造体は frequency メンバを持つことが前提です。

アルゴリズムまたは振る舞い

1. 引数 e1 と e2 を TrackItem * 型のポインタへキャストします。 2. (*t2)->frequency - (*t1)->frequency を計算します。 3. この計算結果により、頻度が高い順（降順）に並ぶようソート順序を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/ts_typanalyze.c 内の統計計算処理（compute_tsvector_stats）から、qsort 等のソート関数の引数として呼び出されます。
  • **呼び出し先**: 本関数は比較ロジックのみであり、他の関数を呼び出しません。
  • **注記**: src/backend/utils/adt/array_typanalyze.c にも同名の関数が存在しますが、本ページは ts_typanalyze.c 内の定義を対象としています。

関連データ構造

  • TrackItem: 語彙の出現頻度と誤差を保持する構造体。

失敗モードと境界条件

  • e1 または e2 が NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • 頻度の差が int 型の範囲を超える場合、オーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 本関数は qsort 互換の比較関数です。頻度順のソートが必要な箇所でのみ使用してください。
  • 語彙の辞書順ソートが必要な場合は、別途定義されている trackitem_compare_lexemes を使用してください。
