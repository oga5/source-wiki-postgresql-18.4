SetMatViewPopulatedState

定義

  • 種別: function
  • 定義箇所: src/backend/commands/matview.c:78

目的

マテリアライズド・ビュー（MV）の relispopulated フラグを更新し、その状態変更をシステム全体に通知する。

シグネチャまたは形状

void SetMatViewPopulatedState(Relation relation, bool newstate)

入力と出力

  • **入力**:
  • relation: 対象となるマテリアライズド・ビューの Relation 構造体。
  • newstate: 設定する状態（true でデータが存在する状態、false で空の状態）。
  • **出力**: なし（副作用としてシステムカタログを更新）。

前提条件と不変条件

  • relation は RELKIND_MATVIEW である必要がある（Assert で検証済み）。
  • 呼び出し元は、対象リレーションに対して適切なロックを保持している必要がある。

アルゴリズムまたは振る舞い

1. pg_class システムカタログを RowExclusiveLock で開く。 2. SearchSysCacheCopy1 を使用して、対象リレーションの pg_class タプルをコピーとして取得する。 3. タプルの relispopulated フィールドを newstate に更新する。 4. CatalogTupleUpdate を呼び出し、システムカタログの変更を永続化する。 5. heap_freetuple でコピーしたタプルを解放し、table_close で pg_class を閉じる。 6. CommandCounterIncrement() を実行し、変更を現在のトランザクション内で可視化する。

  • 副作用として、この処理により他のバックエンドおよび自プロセスに対して、リレーションキャッシュ（relcache）を再構築させるための SI メッセージが送信される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/createas.c: CREATE MATERIALIZED VIEW 実行時。
  • src/backend/commands/matview.c: REFRESH MATERIALIZED VIEW 実行時。
  • **呼び出し先**:
  • table_open, table_close
  • SearchSysCacheCopy1, CatalogTupleUpdate
  • CommandCounterIncrement

関連データ構造

  • Relation: 操作対象のMV定義。
  • Form_pg_class: pg_class システムカタログの構造体（relispopulated フィールドを含む）。

失敗モードと境界条件

  • SearchSysCacheCopy1 がタプルを見つけられない場合、elog(ERROR) によりトランザクションがアボートされる。

保守担当者向けメモ

  • 本関数は pg_class を直接更新するため、呼び出し元で適切なロックが確保されていることを確認すること。
  • CommandCounterIncrement を呼び出しているため、本関数の実行後に同一トランザクション内でリレーションを参照すると、更新後の状態が即座に反映される。
