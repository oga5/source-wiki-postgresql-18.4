FixedParallelExecutorState

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/execParallel.c:74

目的

並列クエリ実行において、リーダープロセスとワーカープロセス間で共有される固定的な実行状態を保持する。

シグネチャまたは形状

typedef struct FixedParallelExecutorState
{
    int64       tuples_needed;  /* tuple bound, see ExecSetTupleBound */
    dsa_pointer param_exec;
    int         eflags;
    int         jit_flags;
} FixedParallelExecutorState;

入力と出力

  • **入力**: 並列実行の初期化時にリーダープロセスによって設定される。
  • **出力**: ワーカープロセスが ExecutorStart を実行する際に、共有メモリからこの構造体を読み取り、実行環境を再構築するために使用する。

前提条件と不変条件

  • 動的共有メモリ（DSM）上に配置される必要がある。
  • shm_toc_allocate を用いて割り当てられる。

アルゴリズムまたは振る舞い

1. リーダープロセスが ExecInitParallelPlan 等の過程で本構造体を初期化し、DSM領域に配置する。 2. ワーカープロセスは起動後、DSMから本構造体のアドレスを取得する。 3. ワーカーは構造体内の tuples_needed や eflags、jit_flags を参照し、自身の実行プランの初期化に反映させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c 内の並列実行初期化および実行ルーチン（ExecInitParallelPlan 等）から参照される。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはない。

関連データ構造

  • ParallelExecutorInfo: リーダー側で並列実行全体を管理する構造体。
  • dsa_pointer: 動的共有メモリ内の領域を指すポインタ。

失敗モードと境界条件

  • DSMの割り当て失敗時、shm_toc_allocate がエラーを発生させる可能性がある。
  • 構造体内の各フィールドが適切に初期化されていない場合、ワーカープロセスでのプラン実行時に予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本構造体は並列実行の基盤となるため、フィールドの追加や変更を行う際は、リーダー側での初期化処理とワーカー側での読み取り処理の両方に影響が及ぶことに注意すること。
  • tuples_needed は ExecSetTupleBound に関連しており、並列クエリの最適化において重要な役割を果たす。
