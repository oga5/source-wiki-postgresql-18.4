appendJSONKeyValueFmt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/error/jsonlog.c:69

目的

printf 形式でフォーマットされた値を JSON の値として生成し、指定された StringInfo バッファにキーと値のペアとして追加する。

シグネチャまたは形状

static void appendJSONKeyValueFmt(StringInfo buf, const char *key, bool escape_key, const char *fmt, ...);

入力と出力

  • buf: 出力先の StringInfo バッファ。
  • key: JSON オブジェクトのキー文字列。
  • escape_key: true の場合、キーを JSON エスケープする。
  • fmt: 値を生成するための printf 形式のフォーマット文字列。
  • ...: フォーマット文字列に渡す可変長引数。
  • 出力: なし（buf を直接更新する）。

前提条件と不変条件

  • buf は初期化済みであること。
  • fmt に続く引数は、fmt の指定と型が一致していること。
  • palloc および pfree によるメモリ管理が行われるため、呼び出し元でメモリ不足時の例外処理が考慮されていること。

アルゴリズムまたは振る舞い

1. 初期バッファサイズ 128 バイトで palloc を行い、pvsnprintf を使用してフォーマット済み文字列を生成する。 2. 生成された文字列がバッファサイズを超える場合、必要なサイズまで拡張して再試行する。 3. 生成された文字列を appendJSONKeyValue に渡し、JSON のキー・値ペアとして buf に追加する。 4. 最後に、一時的に確保したバッファを pfree で解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/error/jsonlog.c 内の各箇所（PID、セッションID、トランザクションID、エラー位置などのログ出力時）。
  • 呼び出し先: pvsnprintf（フォーマット処理）、appendJSONKeyValue（バッファへの書き込み）、palloc/pfree（メモリ管理）。

関連データ構造

  • StringInfo: PostgreSQL の可変長文字列バッファ構造体。

失敗モードと境界条件

  • メモリ不足時: palloc がエラーを発生させる。
  • フォーマット失敗: pvsnprintf の挙動に依存する。
  • エスケープ処理: appendJSONKeyValue の仕様に基づき、値のエスケープ有無は escape_key 引数および appendJSONKeyValue 内部のロジックに依存する。

保守担当者向けメモ

  • 本関数は動的にバッファサイズを調整するため、非常に長い文字列をフォーマットする場合でも安全に動作する。
  • appendJSONKeyValue との役割分担を維持し、フォーマットが不要な場合は直接 appendJSONKeyValue を使用することが推奨される。
