ApplySetting

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_db_role_setting.c:219

目的

システムカタログ pg_db_role_setting から特定のデータベースおよびロールに関連付けられた設定値（GUCパラメータ）を読み込み、現在のプロセスに適用する。

シグネチャまたは形状

void ApplySetting(Snapshot snapshot, Oid databaseid, Oid roleid, Relation relsetting, GucSource source)

入力と出力

  • **入力**:
  • snapshot: 読み取りに使用するスナップショット。
  • databaseid: 対象のデータベースOID。
  • roleid: 対象のロールOID。
  • relsetting: pg_db_role_setting カタログの開かれたリレーション。
  • source: 設定の適用元（GucSource）。
  • **出力**: なし（副作用として現在のプロセスに設定を適用）。

前提条件と不変条件

  • relsetting は pg_db_role_setting カタログを指している必要がある。
  • カタログに格納されている設定値は、挿入時に適切な権限チェックが完了しているものと仮定する。

アルゴリズムまたは振る舞い

1. 指定された databaseid と roleid をキーとして、DbRoleSettingDatidRolidIndexId インデックスを用いたスキャンを開始する。 2. スキャンで取得した各タプルから setconfig カラム（GUC設定配列）を抽出する。 3. setconfig が NULL でない場合、DatumGetArrayTypeP で配列に変換する。 4. ProcessGUCArray を呼び出し、PGC_SUSET レベルで設定を現在のプロセスに適用する。 5. スキャンを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/init/postinit.c: セッション初期化時に、グローバル、データベース単位、ロール単位、およびその組み合わせの設定を適用するために呼び出される。
  • **呼び出し先**:
  • systable_beginscan / systable_getnext / systable_endscan: カタログ走査用。
  • ScanKeyInit: 検索キーの初期化。
  • heap_getattr: タプルからの属性抽出。
  • DatumGetArrayTypeP: Datum からの配列変換。
  • ProcessGUCArray: GUC 設定の適用。

関連データ構造

  • pg_db_role_setting: 設定を保持するシステムカタログ。
  • DbRoleSettingDatidRolidIndexId: 検索に使用するインデックス。
  • GucSource: 設定の適用元を示す列挙型。

失敗モードと境界条件

  • 指定された databaseid や roleid に該当する設定が存在しない場合、ループは実行されず何も適用されない。
  • setconfig カラムが NULL のタプルはスキップされる。

保守担当者向けメモ

  • 本関数は特定の組み合わせに対してのみ動作するため、複数の階層（グローバル、DB、ロール等）の設定を適用する場合は、呼び出し側で InvalidOid を適切に指定して複数回呼び出す必要がある。
  • setconfig は text 型の配列であり、TOAST テーブルが自動的に使用される点に留意すること。
