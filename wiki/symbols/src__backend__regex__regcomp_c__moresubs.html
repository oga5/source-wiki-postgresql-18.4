moresubs

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regcomp.c:554

目的

正規表現のコンパイル中に使用されるサブ式（subre）ベクタを拡張し、指定された数以上の要素を保持できるようにする。

シグネチャまたは形状

static void moresubs(struct vars *v, int wanted)

入力と出力

  • **入力**:
  • v: コンパイル状態を保持する vars 構造体へのポインタ。
  • wanted: 確保が必要な最小のサブ式数。
  • **出力**: なし（副作用として v->subs を更新）。

前提条件と不変条件

  • wanted > 0 かつ (size_t) wanted >= v->nsubs であること。
  • 処理終了後、v->nsubs は拡張後のサイズと一致し、wanted < v->nsubs が成立すること。

アルゴリズムまたは振る舞い

1. 新しいサイズ n を wanted * 3 / 2 + 1 として計算する。 2. 現在の v->subs が初期バッファ v->sub10 を指している場合は MALLOC で新規確保し、既存の内容をコピーする。 3. それ以外の場合は REALLOC で既存の領域を拡張する。 4. メモリ確保に失敗した場合は ERR(REG_ESPACE) を呼び出し、処理を中断する。 5. 確保した領域の新しい末尾部分を NULL で初期化する。 6. v->subs を新しいポインタで更新し、v->nsubs を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regcomp.c 内のコード（行1018など）から呼び出される。
  • **呼び出し先**: MALLOC, REALLOC, memcpy, ERR マクロ。

関連データ構造

  • struct vars: コンパイル状態を保持するコンテキスト。
  • struct subre: 正規表現の構造を表現するツリーノード。

失敗モードと境界条件

  • メモリ不足時: REALLOC または MALLOC が NULL を返した場合、ERR(REG_ESPACE) を経由してエラー状態に遷移し、関数を終了する。

保守担当者向けメモ

  • 本関数は subre ベクタのサイズを動的に管理する。wanted を引数に取ることで、必要な容量を保証する設計となっている。
  • v->sub10 からの移行ロジックが含まれているため、初期化時のメモリレイアウトに依存している点に注意すること。
