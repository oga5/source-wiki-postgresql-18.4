gbt_float4key_cmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_float4.c:54

目的

GiSTインデックスにおける float4KEY 型の比較を行い、ソートやインデックス構築時の順序付けを決定する。

シグネチャまたは形状

int gbt_float4key_cmp(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の Nsrt 構造体へのポインタ。内部の t メンバが float4KEY としてキャストされる。
  • flinfo: 関数マネージャ情報（本関数内では未使用）。
  • **出力**:
  • 0: a と b が等しい場合。
  • 1: a が b より大きい場合。
  • -1: a が b より小さい場合。

前提条件と不変条件

  • 入力ポインタ a および b は、Nsrt 構造体を介して float4KEY 型のデータにアクセス可能であること。
  • float4KEY 構造体は lower および upper メンバを持つこと。

アルゴリズムまたは振る舞い

1. a と b の lower 値を比較する。

  • ia->lower != ib->lower の場合、その大小関係を返す。

    • 2. lower 値が等しい場合、upper 値を比較する。

  • ia->upper == ib->upper ならば 0 を返す。
  • ia->upper > ib->upper ならば 1 を返し、そうでなければ -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_float4.c 内の静的初期化テーブル（tinfo）から参照されている。
  • **呼び出し先**: 直接的な関数呼び出しは行わず、ポインタ演算と構造体メンバの比較のみを行う。

関連データ構造

  • float4KEY: float4 型の範囲（lower, upper）を保持する構造体。
  • Nsrt: ソート処理のために使用されるラッパー構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合の動作は定義されていない。
  • 浮動小数点数の比較において、NaN（Not a Number）が含まれる場合の挙動は、標準的な比較演算子（>）の動作に依存する。

保守担当者向けメモ

  • 本関数は btree_utils_num の汎用ロジックではなく、float4 特有の比較順序を定義している。
  • float4KEY の構造に変更を加える場合は、本関数の比較ロジックも整合性を取る必要がある。
