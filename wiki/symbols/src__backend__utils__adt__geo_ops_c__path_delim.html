path_delim

定義

  • 種別: enum
  • 定義箇所: src/backend/utils/adt/geo_ops.c:73

目的

幾何データ型である「パス（PATH）」の境界状態（開いているか、閉じているか）を識別するために使用されます。

シグネチャまたは形状

enum path_delim
{
    PATH_NONE,
    PATH_OPEN,
    PATH_CLOSED
};

入力と出力

  • 入力: path_encode 関数等の引数として、パスの形状を決定する列挙値を受け取ります。
  • 出力: 該当なし（列挙型定義）。

前提条件と不変条件

  • PATH_NONE は初期状態や未定義状態を示唆します。
  • PATH_OPEN および PATH_CLOSED は、パスの描画や演算において、始点と終点を接続するかどうかの論理的なフラグとして機能します。

アルゴリズムまたは振る舞い

  • path_encode 関数内において、この列挙値に基づいた switch 文により、パスの文字列表現（括弧の種類など）を決定する分岐条件として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/geo_ops.c 内の path_encode 関数が、引数としてこの型を受け取ります。
  • 呼び出し先: 該当なし（列挙型定義のため）。

関連データ構造

  • PATH: src/include/utils/geo_decls.h で定義される幾何データ型。

失敗モードと境界条件

  • 列挙型として定義されているため、定義外の値が渡された場合の挙動は path_encode 内の switch 文の実装に依存します。

保守担当者向けメモ

  • 本シンボルは path_encode におけるパスの書式化ロジックと密接に関連しています。パスの新しい境界タイプを追加する場合は、path_encode 内の switch 文の網羅性を確認してください。
