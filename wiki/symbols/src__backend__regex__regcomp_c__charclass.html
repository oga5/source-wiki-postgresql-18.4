charclass

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regcomp.c:1494

目的

正規表現のコンパイル過程において、指定された文字クラス（char_classes）に対応する NFA（非決定性有限オートマトン）の遷移アークを生成する。

シグネチャまたは形状

static void charclass(struct vars *v, enum char_classes cls, struct state *lp, struct state *rp)

入力と出力

  • **入力**:
  • v: コンパイル状態を保持する vars 構造体へのポインタ。
  • cls: 生成対象の文字クラスを示す列挙型。
  • lp, rp: 遷移の始点および終点となる NFA の状態（state）。
  • **出力**: なし（副作用として NFA にアークを追加する）。

前提条件と不変条件

  • v が有効なコンパイルコンテキストを指していること。
  • lp および rp が NFA 内の有効な状態であること。
  • 内部で呼び出される cclasscvec および subcolorcvec が成功すること（失敗時は NOERR() マクロにより処理が中断される）。

アルゴリズムまたは振る舞い

1. cclasscvec を呼び出し、指定された文字クラス cls に対応する文字集合（cvec）を取得する。この際、v->cflags の REG_ICASE フラグに基づき大文字小文字の区別を考慮する。 2. NOERR() マクロにより、ここまでの処理でエラーが発生していないか確認する。 3. subcolorcvec を呼び出し、取得した cvec に基づいて lp から rp への NFA アークを構築する。この過程で、必要に応じてカラーマッピングの分割が行われる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regcomp.c 内の複数箇所（995行目、1821行目、1825行目など）。
  • **呼び出し先**:
  • cclasscvec
  • subcolorcvec
  • NOERR() マクロ

関連データ構造

  • struct vars: コンパイル中の状態を保持するコンテキスト。
  • struct state: NFA の状態を表現する構造体。
  • struct cvec: 文字クラスに含まれる文字集合を保持する構造体。
  • enum char_classes: 文字クラスの種類を定義する列挙型。

失敗モードと境界条件

  • cclasscvec がメモリ不足や不正なクラス指定によりエラーを返した場合、NOERR() マクロによって処理が停止する。
  • subcolorcvec によるアーク構築時にカラー分割が発生し、リソース制限に達した場合はエラーとして扱われる。

保守担当者向けメモ

  • 本関数は charclasscomplement と対になる概念であり、ブラケット式内での利用を想定している。
  • REG_ULOCALE ノートが記録されるため、ロケール依存の動作変更が必要な場合はこの点に留意すること。
