PQconnectStartParams

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-connect.c:866

目的

キーワードと値のペア配列からデータベース接続プロセスを開始する。

シグネチャまたは形状

PGconn *PQconnectStartParams(const char *const *keywords,
                             const char *const *values,
                             int expand_dbname);

入力と出力

  • **入力**:
  • keywords: 接続パラメータ名の配列（NULL終端）。
  • values: 対応するパラメータ値の配列（NULL終端）。
  • expand_dbname: データベース名を展開するかどうかを示すフラグ。
  • **出力**:
  • 接続状態を保持する PGconn 構造体へのポインタ。失敗時は CONNECTION_BAD 状態の PGconn を返すか、メモリ確保失敗時に NULL を返す。

前提条件と不変条件

  • 返される PGconn 構造体の errorMessage は初期化済みであること。
  • 接続プロセス中のエラーメッセージは、既存のバッファに追記される。

アルゴリズムまたは振る舞い

1. pqMakeEmptyPGconn() で PGconn 構造体を確保。 2. conninfo_array_parse() で入力配列を解析し、オプション情報を生成。 3. fillPGconn() で解析結果を PGconn 構造体に反映。 4. PQconninfoFree() で一時的なオプション情報を解放。 5. pqConnectOptions2() でホストやポートなどの派生オプションを計算。 6. pqConnectDBStart() を呼び出し、接続の初期段階（ソケット作成等）を実行。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c および src/include/libpq/libpq-be-fe-helpers.h 等から参照されている。
  • **呼び出し先**: pqMakeEmptyPGconn, conninfo_array_parse, fillPGconn, PQconninfoFree, pqConnectOptions2, pqConnectDBStart を呼び出す。

関連データ構造

  • PGconn: 接続状態を管理する主要な構造体。
  • PQconninfoOption: 接続パラメータを保持する中間構造体。

失敗モードと境界条件

  • pqMakeEmptyPGconn が失敗した場合は NULL を返す。
  • conninfo_array_parse が失敗した場合は status を CONNECTION_BAD に設定して PGconn を返す。
  • pqConnectOptions2 または pqConnectDBStart が失敗した場合も、同様に status が CONNECTION_BAD に設定された PGconn が返される。

保守担当者向けメモ

  • 本関数は非同期接続の開始点である。接続の進行状況は後続の PQconnectPoll で管理される。
  • PQconninfoOption 配列は関数内で適切に解放されるため、メモリリークの懸念は低い。
