examine_attribute

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/extended_stats.c:524

目的

統計情報の構築（ANALYZE）プロセスにおいて、特定の式（Node *expr）に対する統計解析の準備を行う。型固有の解析関数を呼び出し、統計計算に必要なメタデータを初期化する。

シグネチャまたは形状

static VacAttrStats *examine_attribute(Node *expr)

入力と出力

  • **入力**: Node *expr（解析対象となる式）
  • **出力**: VacAttrStats *（初期化された統計構造体、または解析不要と判断された場合は NULL）

前提条件と不変条件

  • expr は有効な Node であること。
  • システムカタログ（TYPEOID）から対象型の情報を取得できること。
  • VacAttrStats 構造体のメモリは palloc0 で確保され、attstattarget は -1 に初期化される。

アルゴリズムまたは振る舞い

1. VacAttrStats 構造体を割り当て、式の型情報（attrtypid, attrtypmod, attrcollid）を抽出する。 2. TYPEOID システムキャッシュから型情報を取得し、構造体に格納する。 3. 統計スロット（STATISTIC_NUM_SLOTS）の型情報を、対象の型属性に基づいて初期化する。 4. 型固有の typanalyze 関数を呼び出す。定義されていない場合は std_typanalyze を使用する。 5. 解析結果（ok）、計算関数（compute_stats）、最小行数（minrows）を検証し、条件を満たさない場合はメモリを解放して NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/extended_stats.c:737（BuildRelationExtStatistics 内のループ処理から呼び出される）。
  • **呼び出し先**:
  • exprType, exprTypmod, exprCollation（式情報の抽出）
  • SearchSysCacheCopy1（型情報の取得）
  • OidFunctionCall1（型固有の typanalyze 実行）
  • std_typanalyze（デフォルトの解析関数）

関連データ構造

  • VacAttrStats: 統計解析の状態を保持する構造体。
  • Form_pg_type: 型の定義情報。

失敗モードと境界条件

  • 型情報のキャッシュルックアップに失敗した場合、elog(ERROR) により処理を中断する。
  • typanalyze 関数が false を返した場合、または compute_stats が設定されていない場合、あるいは minrows <= 0 の場合は、解析対象外として NULL を返す。

保守担当者向けメモ

  • 本関数は extended_stats.c 内の式解析用であり、src/backend/commands/analyze.c に存在する同名の関数（列解析用）とはシグネチャおよび目的が異なる点に注意すること。
  • anl_context は NULL に設定されており、個別の属性解析とは異なるメモリ管理方針をとっている。
