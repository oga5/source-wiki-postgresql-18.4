inv_read

定義

  • 種別: function
  • 定義箇所: src/backend/storage/large_object/inv_api.c:449

目的

ラージオブジェクト（LO）の現在のオフセット位置から指定されたバイト数を読み込み、バッファに格納します。pg_largeobject カタログからデータを取得し、欠落しているページ（穴）をゼロで埋める処理を行います。

シグネチャまたは形状

int inv_read(LargeObjectDesc *obj_desc, char *buf, int nbytes)

入力と出力

  • **入力**:
  • obj_desc: オープンされたラージオブジェクトの状態を保持する記述子。
  • buf: 読み込んだデータを格納するバッファ。
  • nbytes: 読み込む最大バイト数。
  • **出力**:
  • 実際に読み込まれたバイト数を返します。

前提条件と不変条件

  • obj_desc が有効なポインタであること。
  • buf が非NULLであること。
  • obj_desc->flags に IFS_RDLOCK が設定されていること（権限チェック）。
  • pg_largeobject テーブルおよびインデックスが open_lo_relation() によってオープン可能であること。

アルゴリズムまたは振る舞い

1. 権限チェックを行い、読み取り権限がない場合はエラーを発生させます。 2. pg_largeobject テーブルに対して、対象LOのIDと現在のページ番号以降を対象とした順序付きスキャンを開始します。 3. スキャン結果をループ処理し、以下の順序でデータをバッファへコピーします。

  • **穴の処理**: 取得したページ番号が現在のオフセットより先にある場合、その間の領域をゼロで埋めます。
  • **データコピー**: ページ内のデータを getdatafield で取得し、TOAST展開後に現在のオフセットから nbytes に達するまでバッファへコピーします。

    • 4. 読み込み完了後、obj_desc->offset を進め、スキャンを終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-fsstubs.c (複数の箇所で利用)
  • **呼び出し先**:
  • open_lo_relation()
  • systable_beginscan_ordered()
  • systable_getnext_ordered()
  • getdatafield()
  • systable_endscan_ordered()

関連データ構造

  • LargeObjectDesc: LOのID、オフセット、スナップショット等を保持。
  • pg_largeobject: データが格納されるシステムカタログ。

失敗モードと境界条件

  • **権限不足**: IFS_RDLOCK フラグがない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー終了します。
  • **データ破損**: pg_largeobject 内にNULLフィールドが存在する場合、elog(ERROR) で停止します。
  • **穴の存在**: ページが連続していない場合、欠落部分はゼロとして読み込まれます。
  • **nbytes <= 0**: 0を返し、読み込みを行いません。

保守担当者向けメモ

  • pg_largeobject.data はTOASTされている可能性があるため、getdatafield によるデトースト処理が必須です。
  • obj_desc->offset は読み込みの進行に合わせて更新されるため、呼び出し元は現在のオフセットを意識する必要はありません。
  • ページングは LOBLKSIZE 単位で行われます。
