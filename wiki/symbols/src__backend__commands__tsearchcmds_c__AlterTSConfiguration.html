AlterTSConfiguration

定義

  • **種別**: function
  • **定義箇所**: src/backend/commands/tsearchcmds.c:1155

目的

SQL の ALTER TEXT SEARCH CONFIGURATION コマンドを実行し、既存の全文検索設定に対するトークンと辞書のマッピング変更（追加または削除）をシステムカタログに反映します。

シグネチャまたは形状

ObjectAddress AlterTSConfiguration(AlterTSConfigurationStmt *stmt)

入力と出力

  • **入力**: AlterTSConfigurationStmt *stmt（解析された ALTER 文の構文木）
  • **出力**: ObjectAddress（変更された全文検索設定オブジェクトの識別子）

前提条件と不変条件

  • 対象となる全文検索設定がシステムカタログ pg_ts_config に存在すること。
  • 呼び出し元が対象設定の所有者であること（object_ownercheck による検証）。
  • pg_ts_config_map カタログに対して RowExclusiveLock を保持すること。

アルゴリズムまたは振る舞い

1. GetTSConfigTuple を使用して対象設定のタプルを取得し、存在確認を行います。 2. object_ownercheck により、実行ユーザーが設定の所有者であることを検証します。 3. table_open で pg_ts_config_map を開きます。 4. stmt->dicts が存在する場合は MakeConfigurationMapping を呼び出し、マッピングの追加を行います。 5. stmt->tokentype が存在する場合は DropConfigurationMapping を呼び出し、マッピングの削除を行います。 6. makeConfigurationDependencies を呼び出し、依存関係を更新します。 7. InvokeObjectPostAlterHook を呼び出し、イベントトリガーやキャッシュ無効化を通知します。 8. ObjectAddressSet で戻り値を設定し、リソースを解放して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（AlterTSConfigurationStmt を処理するユーティリティ層）
  • **呼び出し先**: GetTSConfigTuple, object_ownercheck, table_open, MakeConfigurationMapping, DropConfigurationMapping, makeConfigurationDependencies, InvokeObjectPostAlterHook, ObjectAddressSet, table_close, ReleaseSysCache

関連データ構造

  • AlterTSConfigurationStmt: SQL 文の解析結果を保持する構造体。
  • pg_ts_config: 全文検索設定を格納するシステムカタログ。
  • pg_ts_config_map: トークンと辞書のマッピングを格納するシステムカタログ。

失敗モードと境界条件

  • **エラー**: 対象の設定名が存在しない場合、ERRCODE_UNDEFINED_OBJECT でエラー終了します。
  • **エラー**: 実行ユーザーが所有者でない場合、aclcheck_error が発生します。

保守担当者向けメモ

  • 本関数は pg_ts_config_map の更新に特化しています。依存関係の整合性は makeConfigurationDependencies に依存しているため、カタログ構造の変更時には同関数も併せて確認してください。
