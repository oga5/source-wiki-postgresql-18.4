gbt_vsrt_arg

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_utils_var.c:22

目的

可変長データ型の GiST インデックス操作（特にソート処理）において、比較関数や関数呼び出しに必要なコンテキスト情報を保持するための構造体です。

シグネチャまたは形状

typedef struct
{
	const gbtree_vinfo *tinfo;
	Oid			collation;
	FmgrInfo   *flinfo;
} gbt_vsrt_arg;

入力と出力

本構造体はデータではなく、以下の設定値を保持するコンテナとして機能します。

  • tinfo: 可変長型固有のメソッド（比較関数等）を定義する gbtree_vinfo へのポインタ。
  • collation: 照合順序（Collation）の OID。
  • flinfo: 関数呼び出し情報（FmgrInfo）へのポインタ。

前提条件と不変条件

  • tinfo は有効な gbtree_vinfo 構造体を指している必要があります。
  • flinfo は PostgreSQL の関数マネージャが提供する有効な情報である必要があります。

アルゴリズムまたは振る舞い

主に qsort_arg を使用したソート処理において、比較関数へ追加のコンテキスト（照合順序や関数情報）を渡すための引数として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/btree_gist/btree_utils_var.c 内のソート処理（qsort_arg を呼び出す箇所）にてインスタンス化されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありませんが、本構造体を引数として受け取る比較関数が flinfo や collation を参照します。

関連データ構造

  • gbtree_vinfo: 可変長型ごとの比較ロジックを抽象化する構造体。
  • FmgrInfo: PostgreSQL の関数呼び出し情報を管理する標準構造体。

失敗モードと境界条件

  • tinfo や flinfo が NULL の状態で比較関数に渡された場合、参照時にセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は qsort_arg の引数として渡されることを前提としています。ソートロジックを修正する際は、この構造体に保持される情報が比較関数側で正しく利用されているか確認してください。
