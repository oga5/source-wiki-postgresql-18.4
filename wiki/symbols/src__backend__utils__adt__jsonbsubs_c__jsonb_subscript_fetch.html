jsonb_subscript_fetch

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonbsubs.c:234

目的

jsonb 型のデータに対して、添字（subscript）を用いた要素の取得（fetch）を実行する。

シグネチャまたは形状

static void jsonb_subscript_fetch(ExprState *state, ExprEvalStep *op, ExprContext *econtext)

入力と出力

  • **入力**:
  • state: 式評価の状態。
  • op: 現在の評価ステップ。op->resvalue に対象の jsonb データが格納されている。
  • econtext: 式の評価コンテキスト。
  • **出力**:
  • op->resvalue: 取得した要素の Datum。
  • op->resnull: 取得結果が NULL かどうかを示すフラグ。

前提条件と不変条件

  • op->resnull が指す値が NULL でないこと（アサーションにより保証）。
  • sbsrefstate->workspace が JsonbSubWorkspace 型としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. op->resvalue から jsonb データを抽出する。 2. JsonbSubWorkspace 内の index（パス情報）と sbsrefstate->numupper を取得する。 3. jsonb_get_element を呼び出し、指定されたパスに対応する要素を取得する。 4. 取得結果を op->resvalue に格納し、NULL 状態を op->resnull に反映させる。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の式評価エンジン（ExprEvalStep を介した実行時）。
  • **呼び出し先**:
  • jsonb_get_element（jsonb 構造からの要素抽出）。

関連データ構造

  • JsonbSubWorkspace: 添字の型情報や変換後の値を保持する構造体。
  • SubscriptingRefState: 添字操作の実行状態を保持する構造体。
  • Jsonb: jsonb 型の内部表現。

失敗モードと境界条件

  • 対象の jsonb データが指定されたパス（キーまたはインデックス）を含まない場合、jsonb_get_element の仕様に従い NULL が返される。
  • 添字の型が jsonb の構造と一致しない場合、実行時エラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は jsonb_subscript_handler の sbs_fetch フィールドに登録されるエントリポイントである。
  • スライス操作はサポートしていないため、単一要素の取得のみを想定している。
