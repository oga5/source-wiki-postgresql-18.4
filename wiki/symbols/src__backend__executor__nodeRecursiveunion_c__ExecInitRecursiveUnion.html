ExecInitRecursiveUnion

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeRecursiveunion.c:179

目的

WITH RECURSIVE クエリ実行のための RecursiveUnionState ノードを初期化し、再帰処理に必要なメモリコンテキスト、タプルストア、およびハッシュテーブル（重複排除用）をセットアップする。

シグネチャまたは形状

RecursiveUnionState *ExecInitRecursiveUnion(RecursiveUnion *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の RecursiveUnion プランノード。
  • estate: 実行状態を保持する EState。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの RecursiveUnionState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK が含まれていないこと（Assert で検証）。
  • node->plan.qual が NIL であること（再帰ノードは式評価を行わないため）。
  • WorkTableScan ノードが本ノードの状態を参照できるよう、estate->es_param_exec_vals に rustate が格納される。

アルゴリズムまたは振る舞い

1. RecursiveUnionState を割り当て、実行関数ポインタを ExecRecursiveUnion に設定する。 2. working_table および intermediate_table 用のタプルストアを初期化する。 3. 重複排除（UNION）が必要な場合、tempContext および tableContext を作成し、execTuplesHashPrepare でハッシュ関数を準備後、build_hash_table を呼び出す。 4. ExecInitResultTypeTL を呼び出し、結果のタプル型を初期化する。 5. 子ノード（outerPlan および innerPlan）に対して ExecInitNode を再帰的に呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内の ExecInitNode から呼び出される。
  • **呼び出し先**:
  • tuplestore_begin_heap
  • AllocSetContextCreate
  • ExecInitResultTypeTL
  • ExecInitNode (子ノード用)
  • execTuplesHashPrepare
  • build_hash_table

関連データ構造

  • RecursiveUnionState: 本ノードの実行状態を保持する主要構造体。
  • RecursiveUnion: プランノード定義。
  • EState: クエリ全体の実行状態。
  • ParamExecData: WorkTableScan とのデータ共有に使用。

失敗モードと境界条件

  • node->numCols > 0 の場合、ハッシュテーブル構築のためのメモリ割り当てが発生する。
  • eflags に無効なフラグが渡された場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数は WorkTableScan ノードが依存する working_table を作成するため、子ノードの初期化前に rustate の設定を完了させる必要がある。
  • UNION ALL の場合はハッシュテーブルが不要となるため、node->numCols の値による条件分岐が最適化の鍵となる。
