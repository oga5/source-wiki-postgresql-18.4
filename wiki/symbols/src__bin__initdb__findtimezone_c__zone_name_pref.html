zone_name_pref

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/findtimezone.c:614

目的

タイムゾーン名に対して優先度スコアを割り当て、システムが最適なタイムゾーンを選択する際の判断基準を提供します。

シグネチャまたは形状

static int zone_name_pref(const char *zonename);

入力と出力

  • **入力**: zonename (検証対象のタイムゾーン名文字列)
  • **出力**: 優先度を示す整数値。高い値ほど優先される。

前提条件と不変条件

  • 入力された zonename は NULL でないことを前提としています。
  • 戻り値は、特定のタイムゾーン名に対して以下の固定値を返します。
  • "UTC": 50
  • "Etc/UTC": 40
  • "localtime" または "posixrules": -50
  • その他: 0

アルゴリズムまたは振る舞い

1. "UTC" を最も優先し、スコア 50 を返します。 2. "Etc/UTC" を次に優先し、スコア 40 を返します。 3. "localtime" および "posixrules" は実在のタイムゾーン名ではないため、負のスコア -50 を返し、選択を避けるように誘導します。 4. 上記以外の名前には、中立的なスコア 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/initdb/findtimezone.c 内のタイムゾーン探索ロジックにおいて、候補となるタイムゾーン名の比較に使用されています。
  • **呼び出し先**:
  • 標準ライブラリ関数 strcmp を呼び出しています。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • 入力文字列が上記リストのいずれにも該当しない場合、デフォルト値 0 が返されます。
  • 比較対象の文字列が NULL の場合、strcmp により未定義の動作（セグメンテーション違反等）が発生する可能性があります。

保守担当者向けメモ

  • 本関数は、タイムゾーンの自動検出時に「どの名前を優先的に採用するか」を決定するヒューリスティックな役割を担っています。
  • 新しいタイムゾーンの優先順位付けが必要になった場合は、この関数に条件を追加してください。
