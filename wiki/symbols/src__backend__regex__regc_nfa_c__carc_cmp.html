carc_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_nfa.c:3621

目的

NFAの遷移構造体（struct carc）の配列を、遷移の色（co）および遷移先状態（to）に基づいてソートするための比較関数。

シグネチャまたは形状

static int carc_cmp(const void *a, const void *b)

入力と出力

  • **入力**: 比較対象となる2つの struct carc へのポインタ（const void *）。
  • **出力**:
  • a < b の場合: 負の整数
  • a > b の場合: 正の整数
  • a == b の場合: 0

前提条件と不変条件

  • a および b は struct carc 型へのポインタとしてキャスト可能であること。
  • 比較対象の struct carc は、co（色）および to（遷移先）メンバを持つこと。

アルゴリズムまたは振る舞い

1. a と b を struct carc 型にキャストする。 2. まず co（色）メンバを比較し、順序を決定する。 3. co が等しい場合、to（遷移先）メンバを比較し、順序を決定する。 4. 両者が等しい場合は 0 を返す（重複する遷移は存在しない前提）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regc_nfa.c 内の qsort 呼び出し（carc 配列のソート用）。
  • **呼び出し先**: なし。

関連データ構造

  • struct carc: NFAの遷移を表現する構造体。co（色）および to（遷移先状態）を保持する。

失敗モードと境界条件

  • 重複する遷移が存在しないことを前提としており、重複がある場合の挙動は定義されていない（コード内のコメントによれば、この時点では重複は排除されているはずである）。

保守担当者向けメモ

  • 本関数は qsort の比較関数としてのみ使用される。
  • 比較ロジックを変更する場合は、struct carc のソート順序が NFA の最適化処理に与える影響を考慮すること。
