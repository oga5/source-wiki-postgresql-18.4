ltree_le

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_op.c:101

目的

2つの ltree 型の値が「以下（less than or equal to）」の関係にあるかを判定し、SQL の比較演算子 <= をサポートする。

シグネチャまたは形状

Datum ltree_le(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される2つの ltree 型のポインタ。
  • **出力**: bool 型（Datum 形式）。第1引数が第2引数以下であれば true、そうでなければ false を返す。

前提条件と不変条件

  • RUNCMP マクロが定義されており、引数として渡された ltree 型の比較結果が変数 res に格納されること。
  • res は ltree_compare 等の比較ロジックにより、第1引数が小さい場合に負、等しい場合に0、大きい場合に正の値をとる。

アルゴリズムまたは振る舞い

1. RUNCMP マクロを実行し、内部的な比較関数を呼び出して結果を res に代入する。 2. res <= 0 の評価結果を PG_RETURN_BOOL を介して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の SQL エンジン（演算子 <= の実行時）。
  • **呼び出し先**: RUNCMP マクロ内で呼び出される比較ロジック（ltree_compare 等）。
  • **注記**: ソースコード上では PG_FUNCTION_INFO_V1(ltree_le) による登録が確認できる。

関連データ構造

  • ltree: 比較対象となるラベルパス構造体。
  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約に従う引数コンテナ。

失敗モードと境界条件

  • 入力が NULL の場合、PostgreSQL の関数呼び出し規約に従い、呼び出し元で適切に処理される必要がある。
  • 比較対象の ltree が空である場合や、ラベルの長さが異なる場合の挙動は ltree_compare の実装に依存する。

保守担当者向けメモ

  • 本関数は RUNCMP マクロに強く依存している。比較ロジックの変更が必要な場合は、ltree_compare 側の実装を確認すること。
  • PG_RETURN_BOOL を使用しているため、メモリ管理は PostgreSQL の標準的な関数呼び出し規約に従う。
