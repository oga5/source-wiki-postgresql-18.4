cfunc_match

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/funccache.c:108

目的

関数キャッシュ用ハッシュテーブルにおいて、2つの CachedFunctionHashKey が同一の関数実行コンテキストを指しているかを判定する比較関数。

シグネチャまたは形状

static int cfunc_match(const void *key1, const void *key2, Size keysize)

入力と出力

  • **入力**: 比較対象の key1, key2 (いずれも CachedFunctionHashKey へのポインタ)、および keysize。
  • **出力**: 2つのキーが一致する場合は 0、一致しない場合は 1 を返す。

前提条件と不変条件

  • keysize は sizeof(CachedFunctionHashKey) と等しい必要がある（Assert で検証）。

アルゴリズムまたは振る舞い

1. callResultType フィールドまでの固定長フィールドを memcmp で比較する。 2. nargs > 0 の場合、argtypes 配列の内容を memcmp で比較する。 3. callResultType が両方のキーに存在する場合、equalRowTypes を用いて行型の一致を確認する。 4. いずれかのステップで不一致が検出された場合は直ちに 1 を返し、すべての比較を通過した場合は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/funccache.c 内のハッシュテーブル初期化処理において、ctl.match フィールドに本関数が設定される。
  • **呼び出し先**: equalRowTypes (行型の比較に使用)。

関連データ構造

  • CachedFunctionHashKey: キャッシュ検索のキーとなる構造体。
  • cfunc_hashtable: 本関数を利用するグローバルなハッシュテーブル。

失敗モードと境界条件

  • callResultType の有無がキー間で異なる場合、不一致として扱われる。
  • nargs が 0 の場合、argtypes の比較はスキップされる。

保守担当者向けメモ

  • 本関数は cfunc_hash と対で使用される。キー構造体に変更を加えた場合は、本関数内の memcmp 範囲や比較ロジックを更新する必要がある。
