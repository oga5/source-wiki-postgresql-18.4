translate_col_privs

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/inherit.c:709

目的

親テーブルの列権限（Bitmapset）を、継承関係にある子テーブルの列属性番号に変換します。

シグネチャまたは形状

static Bitmapset *translate_col_privs(const Bitmapset *parent_privs, List *translated_vars)

入力と出力

  • **入力**:
  • parent_privs: 親テーブルの列権限を示す Bitmapset。
  • translated_vars: 親テーブルの列から子テーブルの列へのマッピングを保持する Var ノードのリスト。
  • **出力**:
  • 子テーブルの列属性番号に対応する Bitmapset。

前提条件と不変条件

  • システム属性（attno < 0）は、すべてのテーブルで共通の番号を持つものとして扱われます。
  • translated_vars リスト内の NULL 要素は、削除された列として無視されます。

アルゴリズムまたは振る舞い

1. **システム属性の処理**: 親の権限セットに含まれるシステム属性を、そのまま子テーブルの権限セットに追加します。 2. **行全体参照の確認**: 親の権限セットに「行全体（whole-row）」参照が含まれているかを確認します。 3. **ユーザー属性の変換**: translated_vars を走査し、親の列番号に対応する子テーブルの varattno を特定します。

  • 親が「行全体」参照を持つ場合、または特定の列に対する権限を持つ場合、その列に対応する子テーブルの varattno を結果の Bitmapset に追加します。

    • 4. 最終的に構築された child_privs を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/inherit.c 内の複数箇所（child_updatedCols の計算や、継承関係の列変換処理など）。
  • **呼び出し先**:
  • bms_is_member, bms_add_member（Bitmapset 操作用ユーティリティ）。
  • lfirst_node（リスト走査用マクロ）。

関連データ構造

  • Bitmapset: 権限や属性番号の集合を保持するために使用。
  • Var: クエリ内の列参照を表現するノード。

失敗モードと境界条件

  • translated_vars に NULL が含まれる場合、その列はスキップされます。
  • parent_privs が NULL の場合、結果も NULL となります。

保守担当者向けメモ

  • 本関数は、継承階層における権限の伝播を正しく行うために不可欠です。
  • translate_col_privs_multilevel は本関数を基盤としており、多段階の継承関係を扱う際に参照されます。
