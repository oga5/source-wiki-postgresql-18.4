pg_digest

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgcrypto.c:88

目的

SQL インターフェースを通じて、指定されたアルゴリズムでデータのハッシュ値を計算する。

シグネチャまたは形状

Datum pg_digest(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_BYTEA_PP(0): ハッシュ対象のデータ (bytea)
  • PG_GETARG_TEXT_PP(1): ハッシュアルゴリズム名 (text)
  • **出力**:
  • 計算されたハッシュ値 (bytea) を PG_RETURN_BYTEA_P で返却。

前提条件と不変条件

  • find_provider が指定されたアルゴリズムを解決できない場合、エラーが発生し処理が中断される。
  • 結果格納用のメモリは palloc により確保され、SET_VARSIZE で適切にサイズ設定される。

アルゴリズムまたは振る舞い

1. find_provider を呼び出し、px_find_digest を用いてハッシュプロバイダー (PX_MD) を取得する。 2. px_md_result_size でハッシュ結果のサイズを取得し、戻り値用のメモリを確保する。 3. px_md_update で入力データをハッシュコンテキストに読み込む。 4. px_md_finish でハッシュ計算を完了し、結果を確保したメモリ領域に書き込む。 5. px_md_free でハッシュコンテキストを解放する。 6. PG_FREE_IF_COPY を用いて、入力引数のコピーが存在する場合にメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1(pg_digest) により登録）。
  • **呼び出し先**: find_provider, px_md_result_size, px_md_update, px_md_finish, px_md_free（いずれも px レイヤーの関数）。

関連データ構造

  • PX_MD: ハッシュ計算用のコンテキスト構造体。
  • bytea, text: PostgreSQL のデータ型。

失敗モードと境界条件

  • アルゴリズム名が不正な場合、find_provider 内でエラーが発生する。
  • メモリ確保失敗時は PostgreSQL の標準的なエラーハンドリングに従う。

保守担当者向けメモ

  • 本関数は pgcrypto モジュールの SQL フロントエンドであり、実際の暗号処理は px レイヤーに委譲している。
  • PG_FREE_IF_COPY の使用により、PostgreSQL のメモリ管理ルール（TOAST 展開済みデータの解放）を遵守している。
