setTargetTable

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_clause.c:177

目的

UPDATE、DELETE、MERGE 文における対象テーブル（ターゲットリレーション）を特定し、ロックの取得および ParseState への登録を行う。

シグネチャまたは形状

int setTargetTable(ParseState *pstate, RangeVar *relation, bool inh, bool alsoSource, AclMode requiredPerms)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態。
  • relation: 対象テーブルを指定する RangeVar。
  • inh: 継承テーブルを含めるかどうかのフラグ。
  • alsoSource: テーブルをソース（FROM 句等）としても使用するかどうかのフラグ。
  • requiredPerms: テーブルに対して要求されるアクセス権限。
  • **出力**:
  • 戻り値: 登録されたレンジテーブルエントリ（RTE）のインデックス（int）。

前提条件と不変条件

  • relation が一時的な名前付きリレーション（ENR）である場合、変更対象とすることはできずエラーとなる。
  • 取得したテーブルロックはトランザクション終了まで保持される。
  • pstate->p_target_relation が既に設定されている場合、既存のテーブルはクローズされる。

アルゴリズムまたは振る舞い

1. **ENRチェック**: relation が ENR であればエラーを報告する。 2. **既存ターゲットのクローズ**: pstate->p_target_relation が存在すれば table_close を実行する。 3. **テーブルオープン**: parserOpenTable を呼び出し、RowExclusiveLock を取得して pstate->p_target_relation に格納する。 4. **RTE作成**: addRangeTableEntryForRelation を用いて RTE と ParseNamespaceItem を作成し、pstate->p_target_nsitem に保持する。 5. **権限設定**: nsitem->p_perminfo->requiredPerms を指定された requiredPerms で上書きする。 6. **ソース追加**: alsoSource が真の場合、addNSItemToQuery を呼び出し、対象テーブルを名前空間および結合リストに追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/analyze.c (複数箇所)
  • src/backend/parser/parse_merge.c
  • **呼び出し先**:
  • scanNameSpaceForENR
  • table_close
  • parserOpenTable
  • addRangeTableEntryForRelation
  • addNSItemToQuery

関連データ構造

  • ParseState: 解析状態を保持する構造体。
  • RangeVar: テーブル参照情報を保持するノード。
  • ParseNamespaceItem: 名前空間内のアイテム。
  • RowExclusiveLock: 取得するロックモード。

失敗モードと境界条件

  • 指定されたリレーションが ENR である場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラーとなる。
  • parserOpenTable が失敗した場合、例外が発生する可能性がある。

保守担当者向けメモ

  • 本関数は p_target_relation のロックをトランザクション終了まで保持する。free_parsestate() はテーブルをクローズするが、ロックの解放は行わない点に注意すること。
  • requiredPerms の上書きは、後続の解析フェーズで markVarForSelectPriv 等により必要に応じて ACL_SELECT が再付与されることを前提としている。
