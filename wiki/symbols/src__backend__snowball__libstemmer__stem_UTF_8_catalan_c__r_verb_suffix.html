r_verb_suffix

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_catalan.c:1346

目的

カタルーニャ語のステミング処理において、単語末尾の動詞接尾辞を特定し、適切な領域（R1またはR2）の条件を満たす場合に削除を行う。

シグネチャまたは形状

static int r_verb_suffix(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境構造体）
  • 出力: 処理が成功または該当なしの場合は 1、失敗した場合は 0 または負の整数（エラーコード）。

前提条件と不変条件

  • z->c が現在の処理位置を指していること。
  • find_among_b を用いて、定義済みの接尾辞配列 a_3（283要素）から後方一致検索を行う。
  • 削除対象の接尾辞が特定された場合、その種類に応じて r_R1 または r_R2 による領域判定が必須となる。

アルゴリズムまたは振る舞い

1. z->ket を現在位置 z->c に設定する。 2. find_among_b で接尾辞を検索し、結果（among_var）を得る。一致しない場合は 0 を返して終了する。 3. z->bra を現在位置に設定する。 4. among_var の値に基づき分岐する：

  • case 1: r_R1(z) を実行し、成功すれば slice_del(z) で接尾辞を削除する。
  • case 2: r_R2(z) を実行し、成功すれば slice_del(z) で接尾辞を削除する。

    • 5. 削除処理が完了すれば 1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_catalan.c 内のメインステミング処理フローから参照されている。
  • 呼び出し先: find_among_b, r_R1, r_R2, slice_del（いずれもSnowballランタイム関数）。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • a_3: 動詞接尾辞のパターンを格納した among 構造体の配列。

失敗モードと境界条件

  • find_among_b が一致を見つけられない場合、即座に 0 を返す。
  • r_R1 または r_R2 が失敗（戻り値が 0 以下）した場合、その戻り値をそのまま伝播させて終了する。
  • slice_del が失敗した場合、負の戻り値を伝播させる。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによる自動生成コードである。ロジックの変更が必要な場合は、直接編集せず元のSnowballスクリプトを修正すること。
  • among_var のケース分岐は、ステミングルールセットの定義に依存している。
