XLogReaderResetError

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogreader.c:1375

目的

XLogReaderState に保持されているエラーメッセージバッファをクリアし、遅延エラーフラグをリセットします。

シグネチャまたは形状

void XLogReaderResetError(XLogReaderState *state)

入力と出力

  • **入力**: state (XLogReaderState *) - 操作対象のWALリーダー状態構造体。
  • **出力**: なし（副作用として state の内容を更新）。

前提条件と不変条件

  • state は有効な XLogReaderState 構造体を指している必要があります。

アルゴリズムまたは振る舞い

1. state->errormsg_buf の先頭にヌル文字 ('\0') を書き込み、文字列を空にします。 2. state->errormsg_deferred を false に設定し、エラー状態を解除します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogrecovery.c:3511
  • src/backend/access/transam/xlogreader.c:1376 (自己参照の可能性あり)
  • **呼び出し先**:
  • 提供されたソース情報には、本関数から呼び出している関数は含まれていません。

関連データ構造

  • XLogReaderState: WAL読み取りの状態を保持するメイン構造体。errormsg_buf および errormsg_deferred フィールドを管理します。

失敗モードと境界条件

  • state が NULL の場合、未定義の動作（セグメンテーション違反）が発生します。

保守担当者向けメモ

  • 本関数は、エラーが発生した後にリーダーを再利用したり、エラー状態をクリアして次の処理へ進む際に使用されます。
  • FRONTEND マクロが定義されている環境では、ereport が使用できないため、このエラーバッファ経由でのエラー報告が重要となります。
