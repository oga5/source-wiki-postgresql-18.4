pg_visibility_map_rel

定義

  • 種別: function
  • 定義箇所: contrib/pg_visibility/pg_visibility.c:179

目的

指定されたリレーションの可視性マップ（VM）情報を取得し、各ブロックの「すべて可視（all-visible）」および「すべて凍結（all-frozen）」ビットの状態をSQLインターフェースを通じて返します。

シグネチャまたは形状

Datum pg_visibility_map_rel(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: SQL引数としてリレーションのOID（PG_GETARG_OID(0)）を受け取ります。
  • **出力**: (blkno int8, all_visible bool, all_frozen bool) の形式を持つタプルの集合（SRF）を返します。

前提条件と不変条件

  • SRF_IS_FIRSTCALL() の初回呼び出し時に、リレーションの可視性データが collect_visibility_data によってメモリ上に収集されます。
  • 戻り値の型定義（tuple_desc）は pg_visibility_tupdesc(true, false) によって生成されます。

アルゴリズムまたは振る舞い

1. **初期化**: 初回呼び出し時に collect_visibility_data を呼び出し、リレーション全体のVM状態を vbits 構造体に格納します。 2. **反復処理**: funcctx->user_fctx に保持された vbits 構造体を参照し、info->next が info->count に達するまで各ブロックのビット情報を順次処理します。 3. **データ変換**: vbits 内のビットフラグを bool 値に変換し、heap_form_tuple を用いて行を作成します。 4. **終了**: 全ブロックの処理完了後、SRF_RETURN_DONE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのSQLエンジン（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • collect_visibility_data（データ収集）
  • pg_visibility_tupdesc（タプル定義取得）
  • heap_form_tuple（タプル構築）
  • SRF_FIRSTCALL_INIT, SRF_PERCALL_SETUP, SRF_RETURN_NEXT, SRF_RETURN_DONE（SRF制御マクロ）

関連データ構造

  • vbits: VM情報を保持する構造体。
  • FuncCallContext: SRFの呼び出し間で状態を保持するためのコンテキスト。

失敗モードと境界条件

  • 指定されたOIDが有効なリレーションでない場合、collect_visibility_data 内でエラーが発生する可能性があります。
  • メモリ不足時には MemoryContext の切り替えやデータ収集プロセスで例外が発生する可能性があります。

保守担当者向けメモ

  • 本関数は全ページを走査するため、巨大なテーブルに対して実行するとI/O負荷が高まります。
  • collect_visibility_data がリレーションのロックを管理しているため、実行中の整合性は保証されます。
