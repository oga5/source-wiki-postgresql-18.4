proclist_is_empty

定義

  • 種別: function
  • 定義箇所: src/include/storage/proclist.h:37

目的

指定された proclist_head が保持するプロセスリストが空であるかどうかを判定します。

シグネチャまたは形状

static inline bool proclist_is_empty(const proclist_head *list)

入力と出力

  • **入力**: list - 判定対象となる proclist_head 構造体へのポインタ。
  • **出力**: リストが空であれば true、そうでなければ false を返します。

前提条件と不変条件

  • list は有効な proclist_head 構造体を指している必要があります。
  • リストの空状態は、head メンバが INVALID_PROC_NUMBER であることによって定義されます。

アルゴリズムまたは振る舞い

list->head の値が INVALID_PROC_NUMBER と等しいかどうかを比較し、その結果を真偽値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/condition_variable.c (複数箇所)
  • src/backend/storage/lmgr/lwlock.c (複数箇所)
  • src/include/storage/proclist.h (内部アサーション)
  • **呼び出し先**: 呼び出し先に関する情報は提供されていません。

関連データ構造

  • proclist_head: リストの先頭と末尾の procno を保持する構造体。
  • INVALID_PROC_NUMBER: リストが空であることを示す定数。

失敗モードと境界条件

  • list が NULL の場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、共有メモリ上のプロセスリストの状態を高速に確認するために使用されます。
  • リストの整合性を確認する際のアサーション（Assert(!proclist_is_empty(list)) 等）にも利用されています。
