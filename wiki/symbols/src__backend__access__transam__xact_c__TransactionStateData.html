TransactionStateData

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/xact.c:193

目的

PostgreSQLのトランザクションおよびサブトランザクションの現在の実行状態を保持し、階層的なトランザクション管理を実現するためのデータ構造です。

シグネチャまたは形状

typedef struct TransactionStateData {
    FullTransactionId fullTransactionId;
    SubTransactionId subTransactionId;
    char *name;
    int savepointLevel;
    TransState state;
    TBlockState blockState;
    int nestingLevel;
    int gucNestLevel;
    MemoryContext curTransactionContext;
    ResourceOwner curTransactionOwner;
    MemoryContext priorContext;
    TransactionId *childXids;
    int nChildXids;
    int maxChildXids;
    Oid prevUser;
    int prevSecContext;
    bool prevXactReadOnly;
    bool startedInRecovery;
    bool didLogXid;
    int parallelModeLevel;
    bool parallelChildXact;
    bool chain;
    bool topXidLogged;
    struct TransactionStateData *parent;
} TransactionStateData;

入力と出力

  • 入力: トランザクション開始時やセーブポイント作成時に、親の状態から継承または新規作成されます。
  • 出力: トランザクション終了時やアボート時に、リソース解放や状態遷移の判断材料として参照されます。

前提条件と不変条件

  • parent ポインタにより、トップレベルトランザクションを根とするスタック構造を形成します。
  • nestingLevel はトランザクションの深さを表し、トップレベルは0です。
  • curTransactionContext および curTransactionOwner は、当該トランザクションの生存期間に紐付くリソースを管理します。

アルゴリズムまたは振る舞い

  • トランザクションの開始、コミット、アボートに伴い、TBlockState（高レベル状態）と TransState（低レベル状態）が更新されます。
  • サブトランザクションが作成されると、新しい TransactionStateData がスタックに積まれ、parent が現在の状態を指すように設定されます。
  • 並列クエリ実行時には、parallelModeLevel を用いて並列実行の制約を管理します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c 内のトランザクション制御関数（StartTransaction, CommitTransaction 等）がこの構造体を操作します。
  • 呼び出し先: 具体的な関数呼び出し関係はソース抜粋からは特定できません。

関連データ構造

  • TransState: 低レベルのトランザクション状態。
  • TBlockState: SQLレベルのトランザクションブロック状態。
  • TransactionState: TransactionStateData へのポインタ型。
  • TopTransactionStateData: 静的に定義されたトップレベルトランザクションの状態データ。

失敗モードと境界条件

  • maxChildXids を超えるサブトランザクションが発生した場合、childXids 配列の拡張が必要です。
  • メモリ不足時には curTransactionContext の割り当てが失敗する可能性があります。

保守担当者向けメモ

  • 本構造体はトランザクションのライフサイクル全体に影響するため、フィールドの追加や変更は xact.c 内のすべての状態遷移ロジックに影響を与えます。
  • 並列クエリ（Parallel Query）導入以降、parallelModeLevel 等の並列処理関連フィールドが追加されており、並列ワーカーでの動作時には特に注意が必要です。
  • parent ポインタによるスタック管理は、サブトランザクションのネストが深くなる場合に再帰的な処理やスタックオーバーフローのリスクがないか考慮してください。
