utf_u2e

定義

  • 種別: static inline function
  • 定義箇所: src/pl/plperl/plperl.h:50

目的

UTF-8 エンコーディングの文字列を、現在のデータベースエンコーディングに変換します。

シグネチャまたは形状

static inline char *utf_u2e(char *utf8_str, size_t len)

入力と出力

  • **入力**:
  • utf8_str: 変換対象となる UTF-8 文字列へのポインタ。
  • len: 文字列の長さ。
  • **出力**:
  • 変換後のデータベースエンコーディングでエンコードされた文字列へのポインタを返します。

前提条件と不変条件

  • pg_any_to_server を利用するため、mb/pg_wchar.h がインクルードされている必要があります。
  • 戻り値は常に新しいメモリ領域（palloc 経由）を指すことが保証されます。

アルゴリズムまたは振る舞い

1. pg_any_to_server を呼び出し、utf8_str を PG_UTF8 から現在のデータベースエンコーディングへ変換します。 2. 変換結果が元のポインタと同一（変換が発生しなかった場合）であっても、呼び出し側が安全にメモリを管理できるよう、pstrdup を用いて必ずコピーを作成します。 3. 変換後の文字列ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plperl/plperl.c:346
  • src/pl/plperl/plperl.h:134
  • src/pl/tcl/pltcl.c (複数の箇所で参照あり)
  • **呼び出し先**:
  • pg_any_to_server (PostgreSQL 内部関数)
  • pstrdup (PostgreSQL メモリ管理関数)

関連データ構造

  • PG_UTF8: エンコーディング指定定数。

失敗モードと境界条件

  • pg_any_to_server が内部的にエラーを発生させた場合、PostgreSQL のエラーハンドリングメカニズムにより制御が戻らない可能性があります。
  • データベースエンコーディングが SQL_ASCII の場合、変換はスキップされますが、pstrdup によりコピーは生成されます。

保守担当者向けメモ

  • 本関数は palloc を使用してメモリを確保するため、呼び出し側は PostgreSQL のメモリコンテキストのライフサイクルを考慮する必要があります。
  • src/pl/tcl/pltcl.c からの参照が見られますが、メモリリークの可能性について注意が必要です。
