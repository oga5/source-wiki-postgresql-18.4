test_sync

定義

  • 種別: function
  • 定義箇所: src/bin/pg_test_fsync/pg_test_fsync.c:287

目的

PostgreSQLがサポートする複数のファイル同期手法（open_datasync, fdatasync, fsync, fsync_writethrough, open_sync）のパフォーマンスを比較・測定する。

シグネチャまたは形状

static void test_sync(int writes_per_op)

入力と出力

  • **入力**: writes_per_op（1回の同期操作あたりに行う書き込み回数。1または2を指定）。
  • **出力**: なし（標準出力へ計測結果を表示）。

前提条件と不変条件

  • buf に書き込み用のランダムデータが準備されていること。
  • filename が有効な一時ファイルパスとして定義されていること。
  • alarm_triggered フラグが計測期間の終了を制御すること。

アルゴリズムまたは振る舞い

1. 指定された writes_per_op に基づき、テスト対象の同期手法を順次実行する。 2. 各手法について、START_TIMER から STOP_TIMER までの期間中、以下のループを繰り返す：

  • pg_pwrite を用いて指定回数（writes_per_op）の書き込みを行う。
  • 各手法に応じた同期関数（fdatasync, fsync, pg_fsync_writethrough 等）を呼び出す。

    • 3. open_datasync および open_sync は open_direct を介してファイルを開き、プラットフォームでサポートされていない場合は n/a を表示する。 4. 最後に、ファイルシステムが直接I/Oをサポートしていない場合に警告メッセージを表示する。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/bin/pg_test_fsync/pg_test_fsync.c 内で test_sync(1) および test_sync(2) として呼び出し）。
  • **呼び出し先**: open_direct, pg_pwrite, fdatasync, fsync, pg_fsync_writethrough, open, close, printf, die。

関連データ構造

  • XLOG_BLCKSZ: 書き込み単位のブロックサイズ。
  • buf: 書き込み用データバッファ。
  • alarm_triggered: 計測終了を判定するグローバルなシグナルフラグ。

失敗モードと境界条件

  • **書き込み失敗**: pg_pwrite が期待したバイト数を書き込めない場合、die によりプログラムが終了する。
  • **同期失敗**: fsync や pg_fsync_writethrough がエラーを返した場合、die によりプログラムが終了する。
  • **ファイルオープン失敗**: open が失敗した場合、die によりプログラムが終了する。
  • **プラットフォーム制限**: O_DSYNC や O_SYNC が未定義の環境では、該当するテストはスキップされ n/a と表示される。
  • **I/O制限**: ファイルシステムが直接I/Oをサポートしない場合、open_direct が失敗し、警告フラグ fs_warning がセットされる。

保守担当者向けメモ

  • 本関数は wal_sync_method の優先順位に基づいた比較を行う。
  • pg_fsync_writethrough のテストは HAVE_FSYNC_WRITETHROUGH マクロの定義に依存する。
  • writes_per_op が2の場合、ファイル記述子を跨いだ同期挙動の検証が含まれる。
