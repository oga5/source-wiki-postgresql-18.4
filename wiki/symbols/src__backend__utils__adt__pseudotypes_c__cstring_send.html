cstring_send

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pseudotypes.c:133

目的

cstring 型の値を、PostgreSQL のバイナリ通信プロトコル形式に変換して送信するための関数です。

シグネチャまたは形状

Datum cstring_send(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して cstring 型の値を受け取ります。
  • **出力**: bytea 型（bytea *）として、バイナリ形式に変換された文字列を返します。

前提条件と不変条件

  • 入力値は PG_GETARG_CSTRING(0) により正しく取得可能であること。
  • libpq/pqformat.h で定義される StringInfo インターフェースが利用可能であること。

アルゴリズムまたは振る舞い

1. pq_begintypsend を呼び出し、バイナリ送信用のバッファ（StringInfoData）を初期化します。 2. strlen で文字列の長さを取得し、pq_sendtext を使用してバッファにテキストデータを書き込みます。 3. pq_endtypsend を呼び出し、バッファの内容を bytea 形式に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（fmgr）経由で呼び出されます。具体的な呼び出し元コードの詳細は提供されていません。
  • **呼び出し先**:
  • pq_begintypsend
  • pq_sendtext
  • pq_endtypsend
  • strlen

関連データ構造

  • StringInfoData: バイナリ送信用のバッファ構造体。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • 入力文字列が NULL の場合、PG_GETARG_CSTRING の挙動に依存します。
  • メモリ不足が発生した場合、pq_begintypsend 等の内部関数がエラーを報告する可能性があります。

保守担当者向けメモ

  • 本関数は cstring 型のバイナリ送信用インターフェースとして機能します。
  • libpq のプロトコル仕様変更時には、pq_sendtext の使用箇所を確認してください。
