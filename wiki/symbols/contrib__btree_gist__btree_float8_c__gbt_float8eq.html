gbt_float8eq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_float8.c:39

目的

GiSTインデックスにおいて、2つの float8（倍精度浮動小数点数）値が等しいかどうかを判定する比較関数です。

シグネチャまたは形状

static bool gbt_float8eq(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の float8 値を指すポインタ。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • bool: *a == *b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は、有効な float8 型の値を指している必要があります。
  • 本関数は btree_gist モジュール内での比較ロジックとして使用されます。

アルゴリズムまたは振る舞い

1. 引数 a と b を float8 * 型にキャストします。 2. それぞれが指す値をデリファレンスし、C言語の標準的な等価演算子 == を用いて比較結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_float8.c 内の静的配列（tinfo 構造体の初期化箇所）から参照されています。
  • **呼び出し先**: 直接的な関数呼び出しはありません。

関連データ構造

  • float8: PostgreSQLの倍精度浮動小数点型。
  • FmgrInfo: PostgreSQLの関数管理構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、セグメンテーション違反が発生する可能性があります。
  • 浮動小数点数の比較であるため、NaN（非数）同士の比較など、IEEE 754の仕様に基づく挙動に従います。

保守担当者向けメモ

  • 本関数は btree_utils_num が提供する汎用的な数値処理ロジックの一部として、tinfo 構造体を通じて利用されます。
  • 比較ロジックは非常に単純ですが、インデックスの整合性に直結するため、変更時は btree_float8 型の他の比較関数（gbt_float8lt 等）との一貫性を維持してください。
