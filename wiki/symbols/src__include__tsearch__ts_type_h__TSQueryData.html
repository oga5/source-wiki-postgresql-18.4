TSQueryData

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/ts_type.h:220

目的

PostgreSQL の全文検索機能において、検索条件（tsquery）のバイナリ表現を保持するための構造体です。

シグネチャまたは形状

typedef struct
{
	int32		vl_len_;
	int32		size;
	char		data[FLEXIBLE_ARRAY_MEMBER];
} TSQueryData;

入力と出力

  • vl_len_: PostgreSQL の varlena ヘッダー。直接操作は不可。
  • size: クエリツリーを構成する QueryItem の総数。
  • data: 可変長配列。演算子およびオペランドが連続して格納される領域。

前提条件と不変条件

  • 4 バイトアライメントを前提としています。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズ計算には CALCDATASIZE や COMPUTESIZE マクロを使用する必要があります。
  • 8 バイトアライメントを必要とする型（int64 等）を構造体に追加してはなりません。

アルゴリズムまたは振る舞い

tsquery の内部表現として、演算子とオペランドからなるクエリツリーをメモリ上に連続して保持します。data 領域には QueryItem が格納され、検索実行時にこのツリーを走査してマッチングを行います。

呼び出し元と呼び出し先

  • 呼び出し元: TSQueryGetDatum マクロ（src/include/tsearch/ts_type.h:263）が本構造体へのポインタを引数として受け取ります。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • TSQuery: TSQueryData * の型定義。
  • QueryItem: data 配列内に格納されるノード（演算子またはオペランド）。

失敗モードと境界条件

  • メモリレイアウトが厳密に定義されているため、手動で構造体を構築する際はアライメントと varlena ヘッダーの整合性を維持する必要があります。
  • 不適切なサイズ計算はメモリ破壊を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は varlena 型であるため、PostgreSQL のメモリ管理ルールに従い、palloc 等で確保された領域で扱う必要があります。
  • 構造体の変更時は、tsquery のバイナリ互換性に影響を与えるため、アライメント要件を厳守してください。
