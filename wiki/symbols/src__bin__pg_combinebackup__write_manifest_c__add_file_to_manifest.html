add_file_to_manifest

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/write_manifest.c:75

目的

バックアップマニフェスト（JSON形式）に、個別のファイル情報（パス、サイズ、更新日時、チェックサム）をエントリとして追加する。

シグネチャまたは形状

void add_file_to_manifest(manifest_writer *mwriter, const char *manifest_path,
                          uint64 size, time_t mtime,
                          pg_checksum_type checksum_type,
                          int checksum_length,
                          uint8 *checksum_payload)

入力と出力

  • **入力**:
  • mwriter: マニフェスト生成状態を保持する構造体。
  • manifest_path: マニフェストに記録するファイルパス。
  • size: ファイルサイズ。
  • mtime: ファイルの最終更新日時。
  • checksum_type / checksum_length / checksum_payload: チェックサムのアルゴリズム、長さ、およびデータ。
  • **出力**: なし（副作用として mwriter->buf に JSON 文字列が追記される）。

前提条件と不変条件

  • mwriter は create_manifest_writer() によって初期化済みであること。
  • manifest_path が UTF-8 でない場合、Encoded-Path フィールドとして 16 進数エンコードされる。

アルゴリズムまたは振る舞い

1. **区切り文字の挿入**: マニフェスト内の最初のファイルでない場合、先行するエントリとの間にカンマを挿入する。 2. **パスの書き込み**: UTF-8 検証を行い、有効であれば Path として、無効であれば Encoded-Path として 16 進数文字列をバッファに書き込む。 3. **メタデータの書き込み**: Size および Last-Modified（ISO 8601 形式）をバッファに追記する。 4. **バッファ管理**: バッファサイズが 128KB を超えた場合、flush_manifest() を呼び出してディスクへ書き出す。 5. **チェックサムの追加**: checksum_length > 0 の場合、Checksum-Algorithm と 16 進数エンコードされた Checksum を追記する。 6. **終了処理**: JSON オブジェクトを閉じ、再度バッファサイズが 128KB を超えていれば flush_manifest() を呼び出す。

関連データ構造

  • manifest_writer: マニフェストの書き込み状態（バッファ、ファイル記述子等）を保持する。

失敗モードと境界条件

  • **バッファ溢れ**: 128KB を超えるごとに自動的にフラッシュされるため、メモリ不足のリスクは低い。
  • **エンコーディング**: 非 UTF-8 パスは Encoded-Path に変換されるため、パスの破損は発生しない。

保守担当者向けメモ

  • 本関数はフロントエンドツール用であり、バックエンドの AddFileToBackupManifest とは異なり、StringInfo を介したバッファリングと手動の JSON エスケープを行っている点に注意すること。
