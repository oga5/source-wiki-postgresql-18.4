oauth_json_object_end

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth-oauth.c:220

目的

JSON 解析プロセスにおいて、オブジェクトの終了（}）を検出し、解析コンテキストのネストレベルをデクリメントする。

シグネチャまたは形状

JsonParseErrorType oauth_json_object_end(void *state)

入力と出力

  • **入力**: void *state - json_ctx 構造体へのポインタ。
  • **出力**: JsonParseErrorType - 常に JSON_SUCCESS を返す。

前提条件と不変条件

  • state は json_ctx 構造体としてキャスト可能であること。
  • 解析対象の JSON 構造において、オブジェクトの開始と終了が正しく対応していること。

アルゴリズムまたは振る舞い

1. 引数 state を struct json_ctx * にキャストする。 2. ctx->nested（ネストレベル）を 1 減らす。 3. JSON_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq-oauth/oauth-curl.c
  • src/interfaces/libpq/fe-auth-oauth.c
  • **呼び出し先**:
  • 直接的な関数呼び出しはなし。

関連データ構造

  • json_ctx: 解析状態を保持する構造体。nested メンバを通じて現在のネスト深さを管理する。

失敗モードと境界条件

  • ctx が NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • 本関数自体はエラーを返さない設計となっている。

保守担当者向けメモ

  • 本関数は json_ctx の nested カウンタを操作するのみであり、JSON の文法的な妥当性チェックは呼び出し側のパーサーに依存する。
