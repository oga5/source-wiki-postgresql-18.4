euc_cn_to_utf8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_euc_cn/utf8_and_euc_cn.c:41

目的

PostgreSQLのマルチバイト文字処理サブシステムにおいて、EUC_CNエンコーディングからUTF-8エンコーディングへの文字列変換を実行する。

シグネチャまたは形状

Datum euc_cn_to_utf8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の引数を受け取る。
  • src (CSTRING): 変換元のEUC_CN文字列。
  • dest (CSTRING): 変換後のUTF-8文字列を格納するバッファ。
  • len (INT32): ソース文字列の長さ。
  • noError (BOOL): 変換エラー時の挙動フラグ（true の場合、エラーを投げずに処理を継続）。
  • **出力**: 正常に変換されたバイト数を Datum 型（INT32）として返す。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_EUC_CN, PG_UTF8) により、呼び出し時のエンコーディングIDがEUC_CNからUTF-8への変換として妥当であることが検証される。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得する。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングの整合性を確認する。 3. LocalToUtf 汎用関数を呼び出し、変換を実行する。この際、変換ロジックとして euc_cn_to_unicode_tree を引数として渡す。 4. 変換結果のバイト数を PG_RETURN_INT32 で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数管理インターフェース（fmgr）から動的に呼び出される。
  • **呼び出し先**: LocalToUtf（src/backend/utils/mb/wchar.c で定義される汎用変換関数）。
  • 注: ソースコードの範囲内では、具体的な呼び出し元の関数名は特定されていない。

関連データ構造

  • euc_cn_to_unicode_tree: 変換マッピングを保持する木構造（../../Unicode/euc_cn_to_utf8.map よりインクルード）。

失敗モードと境界条件

  • noError が false の場合、変換エラー発生時にPostgreSQLのエラーハンドリング機構により処理が中断される可能性がある。
  • noError が true の場合、エラー発生時は変換可能な部分まで処理を行い、そのバイト数を返す。

保守担当者向けメモ

  • 本関数は LocalToUtf を呼び出す薄いラッパーである。
  • 変換ロジックの変更が必要な場合は、euc_cn_to_unicode_tree が定義されている .map ファイルを確認すること。
  • 新しいエンコーディング変換を追加する際のテンプレートとして利用可能である。
