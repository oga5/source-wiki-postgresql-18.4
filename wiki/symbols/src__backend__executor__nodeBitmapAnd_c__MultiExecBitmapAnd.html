MultiExecBitmapAnd

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapAnd.c:109

目的

複数の子ノードから取得した TIDBitmap を論理積（AND）で結合し、最終的なビットマップを生成する。

シグネチャまたは形状

Node *MultiExecBitmapAnd(BitmapAndState *node)

入力と出力

  • **入力**: BitmapAndState *node （実行状態を保持するノード構造体）
  • **出力**: Node * （結合結果である TIDBitmap へのポインタ）

前提条件と不変条件

  • node->bitmapplans に有効なサブプランが格納されていること。
  • MultiExecProcNode を介して呼び出されること（通常の ExecProcNode 呼び出しはエラーとなる）。
  • 少なくとも1つのサブプランが存在すること。

アルゴリズムまたは振る舞い

1. node->bitmapplans に含まれる全サブプランを順次実行する。 2. 最初のサブプランの結果をベースとし、以降の結果を tbm_intersect で順次結合する。 3. 結合に使用したサブプランのビットマップは tbm_free で解放する。 5. 計測用インストゥルメンテーションが有効な場合、InstrStartNode および InstrStopNode を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:533
  • **呼び出し先**:
  • MultiExecProcNode (各サブプランの実行)
  • tbm_intersect (ビットマップの論理積演算)
  • tbm_free (ビットマップのメモリ解放)
  • tbm_is_empty (早期終了判定)
  • InstrStartNode / InstrStopNode (計測用)

関連データ構造

  • BitmapAndState: 本ノードの実行状態を保持する構造体。
  • TIDBitmap: 結合対象および結果となるビットマップ構造体。

失敗モードと境界条件

  • **エラー**: サブプランの結果が TIDBitmap 型でない場合、またはサブプランが0個の場合は elog(ERROR) を発生させる。
  • **境界条件**: 最初のサブプランでビットマップが空になった場合、即座に処理を終了する。

保守担当者向けメモ

  • 本ノードは ExecProcNode インターフェースをサポートしていないため、呼び出し規約の変更時には注意が必要。
  • サブプランの順序はプランナーによって選択度に基づいて最適化されているため、早期終了の効率が維持されるよう留意すること。
