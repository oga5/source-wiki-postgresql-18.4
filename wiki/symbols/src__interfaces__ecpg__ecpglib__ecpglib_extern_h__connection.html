connection

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/ecpglib/ecpglib_extern.h:101

目的

ECPGライブラリにおけるデータベース接続状態を管理し、関連するプリペアドステートメントやキャッシュ情報を保持するための構造体です。

シグネチャまたは形状

struct connection
{
	char	   *name;
	PGconn	   *connection;
	bool		autocommit;
	struct ECPGtype_information_cache *cache_head;
	struct prepared_statement *prep_stmts;
	struct connection *next;
};

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。各フィールドは接続名、libpq の接続ハンドル、自動コミット設定、型情報キャッシュ、プリペアドステートメントのリスト、および接続リストの次要素へのポインタを保持します。

前提条件と不変条件

  • next フィールドにより、接続のリンクリストを構成します。
  • connection フィールドは libpq の PGconn を参照します。

アルゴリズムまたは振る舞い

ECPGライブラリ内部で、複数のデータベース接続を追跡・管理するために使用されます。接続の確立、ステートメントの準備、およびトランザクション制御の際に、この構造体を通じて状態が参照・更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは、特定の関数による直接的な呼び出し元は特定できません。
  • **呼び出し先**: libpq ライブラリ（PGconn を介して）および ECPGtype_information_cache や prepared_statement 構造体を参照します。
  • **注意**: ツールが提示した contrib/dblink/dblink.c の言及は、本構造体とは別の文脈（PostgreSQLサーバー側の拡張機能）であり、本シンボルへの直接的な参照ではありません。

関連データ構造

  • PGconn (libpq)
  • ECPGtype_information_cache
  • prepared_statement

失敗モードと境界条件

  • 接続リストの末尾では next が NULL となります。
  • connection フィールドが NULL の場合、有効なデータベース接続が存在しない状態を示します。

保守担当者向けメモ

  • 本構造体は ecpglib の内部管理用です。メモリ管理には標準の malloc ではなく、ecpg_alloc 等のライブラリ専用関数を使用してください。
  • 接続の追加・削除ロジックは src/interfaces/ecpg/ecpglib/connect.c に集約されています。
