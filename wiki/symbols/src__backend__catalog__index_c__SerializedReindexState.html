SerializedReindexState

定義

  • 種別: struct
  • 定義箇所: src/backend/catalog/index.c:93

目的

REINDEX処理の状態をシリアライズし、プロセス間やトランザクション間で共有または永続化するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
    Oid            currentlyReindexedHeap;
    Oid            currentlyReindexedIndex;
    int            numPendingReindexedIndexes;
    Oid            pendingReindexedIndexes[FLEXIBLE_ARRAY_MEMBER];
} SerializedReindexState;

入力と出力

  • **入力**: REINDEX対象のヒープおよびインデックスのOID、および保留中のインデックスリスト。
  • **出力**: 構造体内の各フィールドに格納された状態情報。

前提条件と不変条件

  • pendingReindexedIndexes は FLEXIBLE_ARRAY_MEMBER を使用しており、構造体の末尾に可変長配列として配置されます。
  • 構造体サイズを計算する際は offsetof(SerializedReindexState, pendingReindexedIndexes) を使用する必要があります。

アルゴリズムまたは振る舞い

  • REINDEX処理中に現在処理中のオブジェクト（ヒープおよびインデックス）を追跡します。
  • numPendingReindexedIndexes により、処理待ちのインデックス数を管理します。
  • シリアライズされた状態からポインタキャストを通じてアクセスされることが想定されています。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/index.c 内のREINDEX関連処理。
  • 呼び出し先: 直接的な関数呼び出し関係はソース抜粋からは不明です。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • 可変長配列のサイズが確保されたメモリ領域を超えないよう、呼び出し側で適切にメモリ管理を行う必要があります。

保守担当者向けメモ

  • pendingReindexedIndexes が可変長であるため、この構造体をスタック上に直接確保することは避けてください。
  • src/backend/catalog/index.c 内の offsetof を使用した計算ロジックを変更する際は、メモリレイアウトの整合性に注意してください。
