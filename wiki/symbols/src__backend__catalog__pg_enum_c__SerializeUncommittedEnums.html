SerializeUncommittedEnums

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_enum.c:826

目的

トランザクション内で作成された未コミットの列挙型（ENUM）およびその値の OID を、指定されたメモリ領域にシリアライズする。これは主に並列クエリ実行時などに、未コミット状態を共有メモリへ伝播させるために使用される。

シグネチャまたは形状

void SerializeUncommittedEnums(void *space, Size size)

入力と出力

  • space: シリアライズ先となるメモリ領域へのポインタ。
  • size: space に割り当てられたバイトサイズ。
  • 出力: なし（space が指すメモリ領域に OID のリストが書き込まれる）。

前提条件と不変条件

  • size は EstimateUncommittedEnumsSpace() が返す値と一致しなければならない（Assert により検証）。
  • 処理終了時、実際に書き込まれたデータサイズが size と一致しなければならない（Assert により検証）。

アルゴリズムまたは振る舞い

1. uncommitted_enum_types ハッシュテーブル内の全 OID を space に書き込み、終端として InvalidOid を付与する。 2. uncommitted_enum_values ハッシュテーブル内の全 OID を同様に書き込み、終端として InvalidOid を付与する。 3. 各ハッシュテーブルが空の場合はスキップされる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/parallel.c:445
  • 呼び出し先: hash_seq_init, hash_seq_search（ハッシュテーブル走査用）

関連データ構造

  • uncommitted_enum_types: 未コミットの列挙型 OID を保持するハッシュテーブル。
  • uncommitted_enum_values: 未コミットの列挙値 OID を保持するハッシュテーブル。
  • InvalidOid: リストの区切り文字として使用される。

失敗モードと境界条件

  • size の不一致: 呼び出し側が確保したメモリサイズと、実際にシリアライズされたデータサイズが一致しない場合、Assert により異常終了する。
  • ハッシュテーブルの不在: uncommitted_enum_types または uncommitted_enum_values が NULL の場合、そのセクションはスキップされる。

保守担当者向けメモ

  • 本関数は EstimateUncommittedEnumsSpace と対になって動作する。シリアライズ対象のデータ構造に変更を加える場合は、必ず両方の関数を更新すること。
  • 終端の InvalidOid は、復元側（RestoreUncommittedEnums）がリストの境界を判定するために不可欠である。
