SyncRepCancelWait

定義

  • 種別: function
  • 定義箇所: src/backend/replication/syncrep.c:405

目的

同期レプリケーションの完了を待機しているバックエンドプロセスを、待機キューから安全に離脱させ、待機状態を解除する。

シグネチャまたは形状

static void SyncRepCancelWait(void)

入力と出力

  • 入力: なし（グローバルな MyProc を参照）
  • 出力: なし（副作用として共有メモリ上のキュー状態とプロセス状態を更新）

前提条件と不変条件

  • SyncRepLock を排他モードで取得する必要がある。
  • MyProc が有効なプロセス構造体として初期化されていること。

アルゴリズムまたは振る舞い

1. SyncRepLock を排他モードで取得する。 2. MyProc->syncRepLinks がキューから離脱済みでないことを確認する。 3. 離脱済みでない場合、dlist_delete_thoroughly を使用して待機キューから当該プロセスを削除する。 4. MyProc->syncRepState を SYNC_REP_NOT_WAITING に設定する。 5. SyncRepLock を解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/replication/syncrep.c 内の同期待機処理に関連する複数の箇所（ソースコード上の直接的な参照として確認）。
  • 呼び出し先:
  • LWLockAcquire / LWLockRelease
  • dlist_node_is_detached / dlist_delete_thoroughly

関連データ構造

  • SyncRepLock: 待機キューを保護するロック。
  • PGPROC (MyProc): 待機中のバックエンドプロセス情報。
  • syncRepLinks: PGPROC 内の待機キュー用リストノード。
  • syncRepState: プロセスの同期待機状態を示すフラグ。

失敗モードと境界条件

  • MyProc->syncRepLinks が既に離脱している場合、dlist_delete_thoroughly は呼び出されないため、二重削除は発生しない。
  • ロック取得失敗時の挙動については、LWLockAcquire の標準的な動作に従う。

保守担当者向けメモ

  • 本関数は、トランザクションの終了やエラー発生時に、不要になった同期待機をクリーンアップするために使用される。
  • SyncRepLock を保持したまま実行されるため、本関数内での重い処理は避けること。
