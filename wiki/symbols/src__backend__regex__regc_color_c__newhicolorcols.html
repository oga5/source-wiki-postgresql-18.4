newhicolorcols

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_color.c:468

目的

高位カラーマップ（hicolormap）の列数を倍に拡張し、既存の列データを複製してカラーマップの容量を増やす。

シグネチャまたは形状

static void newhicolorcols(struct colormap *cm)

入力と出力

  • **入力**: struct colormap *cm （カラーマップ構造体へのポインタ）
  • **出力**: なし（副作用として cm 内の hicolormap 配列および hiarraycols が更新される）

前提条件と不変条件

  • cm->hiarraycols が INT_MAX / (cm->maxarrayrows * 2) を超えていないこと。
  • REALLOC_ARRAY によるメモリ確保が成功すること。

アルゴリズムまたは振る舞い

1. **容量チェック**: 拡張後のサイズが INT_MAX を超える場合は REG_ESPACE エラーを報告して終了する。 2. **メモリ再確保**: hicolormap 配列を現在の2倍のサイズに再確保する。 3. **データ複製**: 既存の列データを新しい領域の後半部分にもコピーする。

  • 配列の末尾から先頭に向かって処理を行い、既存の列データを新しい列位置（newrowptr および newrowptr2）に書き込む。
  • 複製された各色について、対応するカラー記述子（cm->cd[co]）の参照カウント nuchrs をインクリメントする。

    • 4. **メタデータ更新**: cm->hiarraycols を2倍に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_color.c:586
  • **呼び出し先**:
  • REALLOC_ARRAY（マクロ）
  • CERR（エラー報告マクロ）

関連データ構造

  • struct colormap: 正規表現の文字彩色を管理する主要構造体。
  • color: 色を表現する整数型。
  • colordesc: 色の属性を保持する構造体（nuchrs 参照カウントを含む）。

失敗モードと境界条件

  • **メモリ不足**: REALLOC_ARRAY が失敗した場合、CERR(REG_ESPACE) を呼び出し、処理を中断する。
  • **サイズ制限**: 拡張後の列数が INT_MAX を超える場合、同様に REG_ESPACE を報告する。

保守担当者向けメモ

  • 本関数は hicolormap の再配置を行うため、呼び出し後に古いポインタを参照しないよう注意が必要である。
  • 参照カウント（nuchrs）の更新は、列の複製に伴う整合性維持のために必須である。
