mix_encrypt_resync

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-cfb.c:124

目的

古い PGP 仕様で必要とされる、CFB モードにおける特殊な再同期（resync）処理を伴う暗号化を実行します。

シグネチャまたは形状

static int mix_encrypt_resync(PGP_CFB *ctx, const uint8 *data, int len, uint8 *dst);

入力と出力

  • ctx: 暗号化コンテキスト（PGP_CFB）。
  • data: 暗号化対象の入力データ。
  • len: 入力データの長さ。
  • dst: 暗号化結果を格納するバッファ。
  • 戻り値: 処理したバイト数。

前提条件と不変条件

  • ctx->resync が有効な場合に呼び出されることを想定しています。
  • ctx->block_no == 2 の場合、ブロック境界での特殊なレジスタ更新ロジックが適用されます。

アルゴリズムまたは振る舞い

1. **ブロック番号2の特殊処理**: ctx->block_no が 2 の場合、最初の 2 バイトに対して特殊な処理を行います。

  • 入力データと ctx->fre（暗号化されたフィードバックレジスタ）の XOR を取り、結果を dst および ctx->encbuf に格納します。
  • 2 バイトの処理が完了した時点で、ctx->fr（フィードバックレジスタ）を ctx->encbuf の内容に基づいて再構成（再同期）し、ctx->pos をリセットします。

    • 2. **通常処理**: 上記以外のケース、またはブロック番号2の処理完了後は、入力データと ctx->fre の XOR を取り、結果を dst および ctx->encbuf に格納します。 3. **状態更新**: 処理したバイト数分だけ ctx->pos を進めます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/pgp-cfb.c 内で、ctx->resync が真の場合に mix 関数ポインタとして代入され、cfb_process 等から間接的に呼び出されます。
  • 呼び出し先: 直接的な関数呼び出しはありませんが、memcpy を使用して内部状態を更新します。

関連データ構造

  • PGP_CFB: 暗号化コンテキスト。fr（フィードバックレジスタ）、fre（暗号化済みレジスタ）、encbuf（内部バッファ）、pos（現在の位置）、block_no（ブロック番号）を保持します。

失敗モードと境界条件

  • len が 0 の場合、処理を行わず 0 を返します。
  • ctx->block_no == 2 の際、入力 len が 2 バイトに満たない場合、部分的な処理を行い、残りのバイトは次回の呼び出しで処理されます。

保守担当者向けメモ

  • 本関数は PGP の古い仕様に依存した複雑なロジックを含みます。resync フラグが有効な場合のレジスタ更新順序は仕様に厳密に従う必要があるため、変更時は注意してください。
  • memcpy による ctx->fr の再構成箇所は、特にデバッグ時に注意を要する領域です。
