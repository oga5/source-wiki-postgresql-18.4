sqlca_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-copystdout.c:32

目的

ECPG（Embedded SQL for C）において、SQL文の実行結果やステータス情報を保持するSQL通信領域（SQL Communication Area）のデータ構造を定義します。

シグネチャまたは形状

struct sqlca_t {
    char        sqlcaid[8];
    long        sqlabc;
    long        sqlcode;
    struct {
        int     sqlerrml;
        char    sqlerrmc[SQLERRMC_LEN];
    }           sqlerrm;
    char        sqlerrp[8];
    long        sqlerrd[6];
    char        sqlwarn[8];
    char        sqlstate[5];
};

入力と出力

  • **入力**: SQL文の実行結果に基づき、ECPGランタイムライブラリによって各フィールドが更新されます。
  • **出力**: アプリケーションは sqlcode や sqlstate を参照することで、直前のSQL操作の成否や詳細なエラー情報を取得できます。

前提条件と不変条件

  • sqlwarn[0] は、他の警告フラグが一つでも設定されている場合に 'W' となります。
  • sqlerrd[2] は、INSERT、UPDATE、DELETE 文の実行後に処理された行数を保持します。

アルゴリズムまたは振る舞い

ECPGプリプロセッサが生成するコードにおいて、SQL文の実行後にこの構造体の内容が更新されます。アプリケーションはこの構造体を確認することで、エラーハンドリング（sqlprint 等の呼び出し）を判断します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/ecpglib/connect.c
  • src/interfaces/ecpg/ecpglib/data.c
  • src/interfaces/ecpg/ecpglib/descriptor.c
  • **呼び出し先**:
  • 構造体定義であるため、関数呼び出しは発生しません。ただし、ECPGget_sqlca() を介してこの構造体へのポインタが取得されます。

関連データ構造

  • sqlda_t: 動的SQLで使用される記述子領域。
  • sqlvar_t: sqlda_t 内で使用される変数定義。

失敗モードと境界条件

  • sqlcode が 0 以外の場合、エラーまたは警告が発生したことを示します。
  • sqlwarn[1] が 'W' の場合、ホスト変数への格納時に文字列の切り捨てが発生したことを示します。

保守担当者向けメモ

  • 本構造体はECPGの標準的な通信領域であり、自動生成されるテストコードにおいてもエラーチェックの基盤として使用されます。
  • フィールドのレイアウトはSQL標準およびECPGの仕様に依存しているため、変更には注意が必要です。
