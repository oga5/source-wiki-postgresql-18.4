AtPrepare_Locks

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lock.c:3445

目的

トランザクションの PREPARE 処理中に、現在のトランザクションが保持しているロック情報を 2 フェーズコミット（2PC）用の永続的レコードとして登録する。

シグネチャまたは形状

void AtPrepare_Locks(void)

入力と出力

  • 入力: なし（ローカルのロックハッシュテーブル LockMethodLocalHash を参照）
  • 出力: なし（副作用として 2PC 状態へのレコード登録を行う）

前提条件と不変条件

  • セッションレベルのロックとトランザクションレベルのロックを同時に保持していないこと（CheckForSessionAndXactLocks により検証済み）。
  • LOCKTAG_VIRTUALTRANSACTION 型のロックは無視される。
  • トランザクションレベルのロックを保持していないロックエントリは処理対象外となる。

アルゴリズムまたは振る舞い

1. LockMethodLocalHash を走査し、各ロックエントリを確認する。 2. 仮想トランザクションID（VXID）ロックおよび、トランザクションレベルのロックを保持していないエントリをスキップする。 3. セッションレベルとトランザクションレベルのロックを混在して保持している場合はエラーを発生させる。 4. 高速パス（Fast-Path）で獲得されたロックについては、メインのロックテーブルへ移行させる。 5. holdsStrongLockCount を false に設定し、準備完了後のロック保持を維持する。 6. ロックタグとロックモードを TwoPhaseLockRecord に格納し、RegisterTwoPhaseRecord を通じて 2PC 状態へ登録する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c:2670
  • 呼び出し先: CheckForSessionAndXactLocks, hash_seq_init, hash_seq_search, FastPathGetRelationLockEntry, RegisterTwoPhaseRecord

関連データ構造

  • LOCALLOCK: プロセスローカルなロック管理構造体。
  • TwoPhaseLockRecord: 2PC 状態に保存されるロック情報の構造体。
  • LockMethodLocalHash: プロセスごとのロック状態を保持するハッシュテーブル。

失敗モードと境界条件

  • セッションレベルとトランザクションレベルのロックを同一オブジェクトに対して保持している場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラー終了する。

保守担当者向けメモ

  • 本関数は PREPARE TRANSACTION の一部として実行される。ここで登録されたロックは、準備されたトランザクションがコミットまたはロールバックされるまで保持される必要がある。
  • locallock->proclock == NULL の場合にメインテーブルへ移行する処理は、高速パスの最適化と整合性を保つために不可欠である。
