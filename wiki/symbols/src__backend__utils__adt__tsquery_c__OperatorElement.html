OperatorElement

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsquery.c:629

目的

tsquery の解析プロセスにおいて、演算子（Operator）とその付随する距離情報（Distance）をスタック上で保持するために使用されます。

シグネチャまたは形状

typedef struct OperatorElement
{
	int8		op;
	int16		distance;
} OperatorElement;

入力と出力

  • op: 演算子の識別子（int8）。
  • distance: 演算に関連する距離情報（int16）。

前提条件と不変条件

  • tsquery の解析スタック操作において、演算子スタックの要素として利用されます。

アルゴリズムまたは振る舞い

  • tsquery 解析時の演算子スタック（opstack）の各要素として格納されます。
  • pushOpStack 関数などを通じて、演算子と距離情報をスタックにプッシュする際のデータ単位として機能します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/tsquery.c 内の pushOpStack 関数および演算子スタックを操作する関数群。
  • 呼び出し先: 情報なし。

関連データ構造

  • opstack: OperatorElement 型の配列として定義され、解析中の演算子スタックとして利用されます。

失敗モードと境界条件

  • opstack のサイズは STACKDEPTH 定数によって制限されており、これを超えるプッシュ操作はスタックオーバーフローを引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は tsquery のポーランド記法変換ロジックにおいて、演算子の優先順位管理と構造構築に不可欠です。変更時は pushOpStack 等のスタック操作関数への影響を確認してください。
