multirange_typanalyze

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_typanalyze.c:71

目的

マルチ範囲型（Multirange Types）カラムに対する統計情報収集（ANALYZE）の準備を行う。プランナが選択率を推定するために必要な統計計算関数を登録し、型固有のキャッシュ情報を設定する。

シグネチャまたは形状

Datum multirange_typanalyze(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（VacAttrStats 構造体へのポインタを含む）。
  • **出力**: bool 型の true を返す。副作用として、渡された VacAttrStats 構造体のフィールド（compute_stats, extra_data, attstattarget, minrows）が更新される。

前提条件と不変条件

  • stats->attrtypid が有効なマルチ範囲型であることを前提とする。
  • attstattarget が負の場合、default_statistics_target で初期化される。
  • minrows は 300 * attstattarget に設定される。

アルゴリズムまたは振る舞い

1. multirange_get_typcache を呼び出し、対象カラムの型情報（ドメインの場合はベース型）を取得する。 2. 統計ターゲットが未設定（負）であれば、デフォルト値で補完する。 3. 統計計算のメイン関数として compute_range_stats を stats->compute_stats に割り当てる。 4. 取得した型キャッシュを stats->extra_data に格納し、後続の統計計算処理で利用可能にする。 5. minrows を設定し、ANALYZEに必要な最小サンプル行数を指定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/attribute_stats.c（ANALYZE処理の統計収集フェーズから呼び出される）。
  • **呼び出し先**: multirange_get_typcache（型情報の取得）、compute_range_stats（統計計算関数の登録先）。

関連データ構造

  • VacAttrStats: ANALYZE処理中に統計情報を保持する構造体。
  • TypeCacheEntry: 型のプロパティをキャッシュする構造体。

失敗モードと境界条件

  • multirange_get_typcache が失敗した場合、例外が発生する可能性がある。
  • カラムがマルチ範囲型でない場合、型キャッシュの取得時にエラーとなる可能性がある。

保守担当者向けメモ

  • 本関数は std_typanalyze と同様の初期化パターンに従っている。
  • compute_range_stats は範囲型とマルチ範囲型で共通の統計計算ロジックを使用しているため、変更時は両方の型への影響を考慮すること。
