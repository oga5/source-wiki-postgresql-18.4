ss_report_location

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/syncscan.c:288

目的

同期スキャン（Synchronous Scan）において、現在スキャン中のテーブルのブロック位置を共有メモリへ報告し、他のバックエンドとスキャン位置を共有可能にする。

シグネチャまたは形状

void ss_report_location(Relation rel, BlockNumber location)

入力と出力

  • rel: スキャン対象の Relation 構造体。
  • location: 現在スキャン中の BlockNumber。
  • 戻り値: なし。

前提条件と不変条件

  • SyncScanLock が初期化されていること。
  • location が SYNC_SCAN_REPORT_INTERVAL の倍数である場合にのみ、共有メモリへの書き込みが試行される。

アルゴリズムまたは振る舞い

1. location が SYNC_SCAN_REPORT_INTERVAL の倍数であるかを確認する。 2. 倍数である場合、LWLockConditionalAcquire を用いて SyncScanLock の排他獲得を試みる。 3. ロックが獲得できた場合、ss_search を呼び出して共有メモリ上のスキャン位置を更新し、その後ロックを解放する。 4. ロックが獲得できない場合、更新をスキップする（競合回避のため）。 5. TRACE_SYNCSCAN が定義されている場合、デバッグログを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/heapam_handler.c
  • src/backend/access/table/tableam.c
  • **呼び出し先**:
  • ss_search (共有メモリ上の位置更新)
  • LWLockConditionalAcquire / LWLockRelease (同期制御)
  • RelationGetRelationName (ログ出力用)

関連データ構造

  • Relation: スキャン対象のテーブル情報。
  • SyncScanLock: 同期スキャン用共有メモリを保護する LWLock。

失敗モードと境界条件

  • **ロック競合**: LWLockConditionalAcquire が失敗した場合、更新は行われない。これは設計上の意図であり、多少の更新漏れは許容される。
  • **間引き**: SYNC_SCAN_REPORT_INTERVAL ごとにしか更新されないため、報告される位置は実際の現在位置よりわずかに遅れる可能性がある。

保守担当者向けメモ

  • 本関数はパフォーマンスを優先し、ロックの待機を避ける設計となっている。
  • SYNC_SCAN_REPORT_INTERVAL の値は、スキャン位置の共有精度とロック競合のトレードオフを決定する。
