JsonPathGinNode

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonb_gin.c:94

目的

jsonpathクエリをGINインデックスで検索可能な論理ツリー構造（AND/OR条件など）に変換した際の中間表現として使用されます。

シグネチャまたは形状

struct JsonPathGinNode {
    JsonPathGinNodeType type;
    /* 柔軟な配列メンバーとして、OR/ANDノードの子ノードを保持 */
    JsonPathGinNode *args[FLEXIBLE_ARRAY_MEMBER];
};

入力と出力

  • **入力**: extract_jsp_bool_expr 等の関数により、JsonPathGinContextを介してjsonpath式から構築されます。
  • **出力**: GINインデックスの整合性判定関数（execute_jsp_gin_node）において、インデックスから得られたビットマップと照合するために使用されます。

前提条件と不変条件

  • argsメンバーは、typeがORまたはANDである場合に有効な子ノードを保持します。
  • ノードのメモリ割り当てにはpallocが使用され、offsetof(JsonPathGinNode, args)を基準としたサイズ計算が行われます。

アルゴリズムまたは振る舞い

  • jsonpathクエリの解析フェーズで、クエリ条件を論理ツリーに分解するために再帰的に生成されます。
  • 検索実行時には、execute_jsp_gin_nodeがこのツリーを走査し、インデックスの検索結果（check配列）がクエリ条件を満たすかを論理的に評価します。

呼び出し元と呼び出し先

  • **呼び出し元**: extract_jsp_bool_expr、およびmake_jsp_expr_node等のノード生成関数から呼び出されます。
  • **呼び出し先**: execute_jsp_gin_nodeによって、クエリ実行時にツリーの評価が行われます。

関連データ構造

  • JsonPathGinNodeType: ノードの種類（AND, OR, 等）を定義する列挙型。
  • JsonPathGinContext: クエリ解析時のコンテキスト情報。

失敗モードと境界条件

  • 複雑なjsonpath式を解析する際、再帰的な処理が発生するため、スタックオーバーフローを防ぐためのcheck_stack_depth()が適切に配置されている必要があります。

保守担当者向けメモ

  • GINインデックスは構造情報を完全には保持しないため、このノードを用いた判定結果は「損失あり」とみなされ、最終的にヒープ上の実データに対するrecheckが必須となります。
