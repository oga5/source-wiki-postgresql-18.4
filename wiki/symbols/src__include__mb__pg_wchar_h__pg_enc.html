pg_enc

定義

  • 種別: enum
  • 定義箇所: src/include/mb/pg_wchar.h:240

目的

PostgreSQLがサポートする文字エンコーディングを識別するための列挙型です。バックエンドおよびフロントエンド（libpqを除く）において、文字変換、検証、およびエンコーディング固有の操作を抽象化するために使用されます。

シグネチャまたは形状

typedef enum pg_enc {
    PG_SQL_ASCII = 0,
    PG_EUC_JP,
    /* ... (各エンコーディング定義) ... */
    _PG_LAST_ENCODING_
} pg_enc;

入力と出力

  • **入力**: 本シンボル自体は型定義であり、関数ではありません。
  • **出力**: エンコーディングを識別する整数値（列挙子）として機能します。

前提条件と不変条件

  • PG_SQL_ASCII は 0 に固定されています。
  • 列挙型の順序は libpq の ABI に影響を与えるため、変更には細心の注意が必要です。
  • _PG_LAST_ENCODING_ は列挙の末尾を示すマーカーです。

アルゴリズムまたは振る舞い

  • pg_wchar_tbl などの変換テーブルにおいて、特定のエンコーディングに対応する関数ポインタを選択するためのインデックスとして利用されます。
  • InitializeClientEncoding() 等の関数を通じて、セッションごとの現在のエンコーディング状態を保持・参照するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/utils/adt/xml.c: XML処理におけるエンコーディング指定。
  • src/backend/utils/mb/mbutils.c: bind_textdomain_codeset 呼び出し時の変換。
  • src/common/encnames.c: エンコーディング名テーブルの参照。
  • src/port/chklocale.c: ロケールチェック処理。
  • **呼び出し先**: 本シンボルは型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • pg_wchar_tbl: 各エンコーディングに対応する変換・検証関数を保持するテーブル。
  • pg_enc2name_tbl: エンコーディングIDと名称をマッピングするテーブル。

失敗モードと境界条件

  • 未定義のエンコーディングIDを使用した場合、変換テーブル参照時に不正なアクセスが発生する可能性があります。
  • libpq とバックエンド間でエンコーディングIDの不一致が生じると、通信時の文字化けや検証エラーの原因となります。

保守担当者向けメモ

  • 本列挙型に新しいエンコーディングを追加する場合は、src/common/encnames.c のテーブル更新も併せて行う必要があります。
  • libpq との互換性を維持するため、既存の列挙子の順序を変更しないでください。
