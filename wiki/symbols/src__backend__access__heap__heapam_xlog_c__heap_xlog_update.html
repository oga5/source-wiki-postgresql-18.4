heap_xlog_update

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_xlog.c:682

目的

ヒープテーブルに対する更新操作（UPDATE）のWALレコードをリプレイし、対象ページのタプル状態を更新する。

シグネチャまたは形状

static void heap_xlog_update(XLogReaderState *record, bool hot_update)

入力と出力

  • record: 処理対象のWALレコード（XLogReaderState）。
  • hot_update: HOT（Heap Only Tuple）更新であるかを示すフラグ。
  • 出力: なし（副作用としてバッファおよび可視性マップを更新）。

前提条件と不変条件

  • XLogReadBufferForRedo により、対象ページが最新でない場合のみ更新が適用される。
  • 更新対象が別ページにまたがる場合、デッドロック回避のためページ番号順のロック取得が推奨されるが、WALリプレイ中は単一スレッド動作のため競合は発生しない。
  • Hot Standby環境において、更新後のページが反映されるまで元のページをアンロックしてはならない。

アルゴリズムまたは振る舞い

1. **可視性マップの更新**: XLH_UPDATE_OLD_ALL_VISIBLE_CLEARED フラグがある場合、旧ページの可視性マップビットをクリアする。 2. **旧タプルの更新**:

  • XLogReadBufferForRedo で旧ページを取得。
  • タプルの infomask を更新し、t_ctid に新しいタプルの位置（newtid）を書き込む。
  • ページを「プルーニング候補」としてマークし、LSNを更新してダーティにする。

    • 3. **新タプルの挿入**:

  • oldblk == newblk の場合は旧ページを再利用。
  • XLOG_HEAP_INIT_PAGE フラグがある場合はページを初期化。
  • XLH_UPDATE_NEW_ALL_VISIBLE_CLEARED フラグがある場合、新ページの可視性マップビットをクリア。
  • 新しいタプルデータをページに書き込む（ソースコードはここで終了）。

呼び出し元と呼び出し先

  • 呼び出し元: heap_redo（src/backend/access/heap/heapam_xlog.c 内のディスパッチャ）。
  • 呼び出し先: XLogRecGetData, XLogRecGetBlockTag, visibilitymap_clear, XLogReadBufferForRedo, PageSetPrunable, MarkBufferDirty 等。

関連データ構造

  • xl_heap_update: 更新操作のWALレコード構造体。
  • XLogReaderState: WALレコード読み取り状態。
  • Buffer: ページバッファ。

失敗モードと境界条件

  • PageGetMaxOffsetNumber が期待値と一致しない場合、PANIC を発生させる。
  • hot_update が真の場合、ページをまたぐ更新は発生しない（Assert で検証）。

保守担当者向けメモ

  • 本関数は heap_redo から呼び出される。
  • 新しいWALレコードタイプを追加した際は、heap_mask 関数で非決定的なビットをマスクし、リカバリテストの偽陽性を防ぐ必要がある。
  • XLogReadBufferForRedo の戻り値が BLK_NEEDS_REDO でない場合、ページ更新処理はスキップされる。
