check_required_directory

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/option.c:384

目的

指定されたディレクトリパスが未設定または空の場合に、環境変数またはカレントディレクトリから値を補完し、パスを正規化する。

シグネチャまたは形状

static void check_required_directory(char **dirpath, const char *envVarName, bool useCwd,
                                     const char *cmdLineOption, const char *description,
                                     bool missingOk)

入力と出力

  • dirpath: 検査・補完対象のパスを保持するポインタ。
  • envVarName: 補完に使用する環境変数名。
  • useCwd: true の場合、環境変数も未設定ならカレントディレクトリを使用する。
  • cmdLineOption: エラーメッセージで使用するコマンドラインオプション名。
  • description: エラーメッセージで使用するディレクトリの説明文。
  • missingOk: true の場合、パスが未設定でもエラーとせず終了する。

前提条件と不変条件

  • dirpath が指すポインタは有効であること。
  • canonicalize_path により、戻り値のパス末尾のセパレータは除去される。

アルゴリズムまたは振る舞い

1. *dirpath が NULL または空文字列の場合:

  • 環境変数 envVarName が設定されていれば、その値を pg_strdup して *dirpath に代入する。
  • 環境変数がなく useCwd が true の場合、getcwd でカレントディレクトリを取得し *dirpath に代入する。失敗時は pg_fatal で終了する。
  • 上記いずれでもなく missingOk が true ならそのままリターンする。
  • 上記いずれでもなく missingOk が false なら、description、cmdLineOption、envVarName を含むエラーメッセージを出力し pg_fatal で終了する。

    • 2. 最後に canonicalize_path(*dirpath) を呼び出し、パスを正規化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/option.c 内の以下の箇所から呼び出される。
  • old_cluster.bindir (PGBINOLD)
  • new_cluster.bindir (PGBINNEW)
  • old_cluster.pgdata (PGDATAOLD)
  • new_cluster.pgdata (PGDATANEW)
  • user_opts.socketdir (PGSOCKETDIR)
  • **呼び出し先**: getenv, pg_strdup, getcwd, pg_fatal, canonicalize_path

関連データ構造

  • ClusterInfo (パス格納用)
  • UserOpts (ソケットディレクトリ格納用)

失敗モードと境界条件

  • getcwd が失敗した場合、pg_fatal によりプログラムが異常終了する。
  • 必須のディレクトリが特定できず、かつ missingOk が false の場合、pg_fatal によりプログラムが異常終了する。

保守担当者向けメモ

  • パスを連結して使用する前提があるため、canonicalize_path による末尾セパレータの除去は必須の処理となっている。
