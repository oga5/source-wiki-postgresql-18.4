ExecAsyncRequestPending

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execAsync.c:148

目的

シグネチャまたは形状

void ExecAsyncRequestPending(AsyncRequest *areq)

入力と出力

  • **入力**: areq (AsyncRequest *) - 状態を更新する対象の非同期リクエスト構造体。
  • **出力**: なし（副作用として areq のメンバを更新）。

前提条件と不変条件

  • areq は有効な AsyncRequest 構造体を指している必要があります。

アルゴリズムまたは振る舞い

以下の順序で areq のメンバを更新します。 1. callback_pending を true に設定し、コールバック待ち状態であることを示します。 3. result を NULL に初期化し、現時点で取得された結果がないことを示します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c 内の関数から呼び出されていることが確認されています。
  • **呼び出し先**:
  • 提供されたソース情報には、本関数から呼び出される特定の関数は含まれていません。

関連データ構造

  • AsyncRequest: 非同期実行ノード間で受け渡されるリクエスト情報を保持する構造体（src/include/nodes/execnodes.h で定義）。

失敗モードと境界条件

  • areq が NULL の場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は非同期処理のステータス管理を行うユーティリティです。新しい非同期対応ノードを実装する際、リクエストが完了していないことを明示的に示すために利用してください。
