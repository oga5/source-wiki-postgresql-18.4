BMS_Comparison

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/bitmapset.h:60

目的

2つの Bitmapset 間の包含関係および等価性を判定した結果を表現するために使用されます。

シグネチャまたは形状

typedef enum
{
    BMS_EQUAL,
    BMS_SUBSET1,
    BMS_SUBSET2,
    BMS_DIFFERENT,
} BMS_Comparison;

入力と出力

  • 本シンボルは列挙型であり、関数 bms_subset_compare(const Bitmapset *a, const Bitmapset *b) の戻り値として出力されます。

前提条件と不変条件

  • BMS_EQUAL: 両方の集合が同一の要素を持つ。
  • BMS_SUBSET1: 第1引数の集合が第2引数の集合のサブセットである。
  • BMS_SUBSET2: 第2引数の集合が第1引数の集合のサブセットである。
  • BMS_DIFFERENT: どちらの集合も他方のサブセットではない。

アルゴリズムまたは振る舞い

  • bms_subset_compare 関数において、集合間のビットパターンを比較し、上記いずれかの状態を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: 本シンボル自体は型定義であり、呼び出し先はありません。
  • **呼び出し元**:
  • src/backend/nodes/bitmapset.c
  • src/backend/optimizer/util/pathnode.c
  • src/include/nodes/bitmapset.h (関数宣言内)

関連データ構造

  • Bitmapset: 本列挙型が比較対象とするデータ構造。

失敗モードと境界条件

  • 入力として NULL（空集合）が渡された場合の挙動は、bms_subset_compare 関数の実装に依存します。

保守担当者向けメモ

  • bms_subset_compare の戻り値として使用されるため、新しい比較状態を追加する場合は同関数のロジック更新が必要です。
