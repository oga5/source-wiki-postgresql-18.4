crosstab_hash

定義

  • 種別: function
  • 定義箇所: contrib/tablefunc/tablefunc.c:635

目的

SQLクエリの結果をクロス集計（縦持ちから横持ちへの変換）し、ハッシュテーブルを用いてカテゴリを管理することで、欠損値を適切に処理した結果セットを返します。

シグネチャまたは形状

Datum crosstab_hash(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_PP(0): ソースデータ取得用のSQLクエリ文字列。
  • PG_GETARG_TEXT_PP(1): カテゴリ定義取得用のSQLクエリ文字列。
  • **出力**:
  • Datum: 0 (実際のデータは rsinfo->setResult に格納される Tuplestore を介して返却される)。

前提条件と不変条件

  • 呼び出し元が ReturnSetInfo をサポートし、かつ SFRM_Materialize モードが許可されている必要があります。
  • 戻り値のタプル記述子（expectedDesc）は、少なくとも2列以上である必要があります。

アルゴリズムまたは振る舞い

1. **コンテキスト検証**: 呼び出しコンテキストが ReturnSetInfo をサポートしているか確認し、エラー時は例外をスローします。 2. **メモリ管理**: per_query_memory コンテキストへ切り替え、タプル記述子をコピーします。 3. **カテゴリ構築**: load_categories_hash を呼び出し、カテゴリ情報をハッシュテーブル（HTAB）にロードします。 4. **データ生成**: get_crosstab_tuplestore を呼び出し、SQL実行結果とハッシュテーブルを用いて Tuplestore を構築します。 5. **結果設定**: rsinfo->returnMode を SFRM_Materialize に設定し、構築した Tuplestore を rsinfo->setResult に格納します。 6. **終了処理**: メモリコンテキストを復元し、0 を返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • load_categories_hash: カテゴリ情報のハッシュテーブル生成。
  • get_crosstab_tuplestore: 最終的な結果セットの構築。
  • **呼び出し元**:
  • 提供されたソースコード内には、本関数を直接呼び出している具体的な関数名は記載されていません。

関連データ構造

  • ReturnSetInfo: セットを返す関数の実行情報と結果格納用構造体。
  • HTAB: カテゴリ管理に使用されるハッシュテーブル。
  • TupleDesc: 戻り値のタプル構造定義。

失敗モードと境界条件

  • **コンテキストエラー**: ReturnSetInfo が無効な場合や、Materialize モードが許可されていない場合はエラーとなります。
  • **型不整合**: 戻り値の列数が2未満の場合、ERRCODE_DATATYPE_MISMATCH でエラーとなります。

保守担当者向けメモ

  • 本関数は SFRM_Materialize モードで動作するため、戻り値の Datum は常に 0 です。実際のデータは rsinfo->setResult を通じて渡されます。
  • crosstab_hash は crosstab 関数と比較して、カテゴリをハッシュで管理するため、カテゴリの欠損や順序の不一致に対してより柔軟な処理が可能です。
