geo_distance_internal

定義

  • 種別: function
  • 定義箇所: contrib/earthdistance/earthdistance.c:52

目的

地球を完全な球体と仮定し、2点間の大圏距離（Great-circle distance）をマイル単位で算出する。

シグネチャまたは形状

static double geo_distance_internal(Point *pt1, Point *pt2)

入力と出力

  • **入力**:
  • pt1, pt2: Point 構造体へのポインタ（x は経度、y は緯度）。
  • **出力**:
  • 2点間の距離を double 型（マイル単位）で返す。

前提条件と不変条件

  • 入力される Point の座標は度（degrees）単位であること。
  • 地球を半径 EARTH_RADIUS（約3958.75マイル）の球体として扱う。
  • 経度差が180度を超える場合は、最短経路となるよう補正される。

アルゴリズムまたは振る舞い

1. degtorad を使用し、入力された緯度・経度をラジアンに変換する。 2. 経度の差 longdiff を計算し、180度（M_PI）を超える場合は TWO_PI から減算して最短の経度差を求める。 3. Haversine公式に近い手法を用い、中心角の正弦（sino）を算出する。 4. 計算誤差による sino の範囲外（> 1.0）をクリップする。 5. 2 * EARTH_RADIUS * asin(sino) により距離を算出する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/earthdistance/earthdistance.c 内の geo_distance 関数から呼び出される。
  • **呼び出し先**:
  • degtorad（ラジアン変換）
  • math.h の数学関数（fabs, sin, cos, sqrt, asin）

関連データ構造

  • Point: utils/geo_decls.h で定義される、座標を保持する構造体。

失敗モードと境界条件

  • sino が計算誤差により 1.0 を超える場合、強制的に 1.0 に設定される。
  • 厳密な楕円体計算ではなく球体近似であるため、極地や長距離において誤差が生じる可能性がある。

保守担当者向けメモ

  • 本関数は静的関数であり、モジュール内部でのみ使用される。
  • 単位はマイル固定であるため、キロメートル単位が必要な場合は呼び出し側で変換を行うこと。
  • M_PI が未定義の環境へのフォールバックが考慮されている。
