gbt_bool_union

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bool.c:126

目的

GiSTインデックスの構築時、複数のブール値キーを統合して単一の範囲キー（boolKEY）を生成します。

シグネチャまたは形状

Datum gbt_bool_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_POINTER(0) から GistEntryVector を取得。PG_GETARG_POINTER(1) は出力サイズ格納用ポインタ。
  • **出力**: 統合された boolKEY 構造体へのポインタを Datum として返却。

前提条件と不変条件

  • tinfo（gbtree_ninfo 構造体）が bool 型の比較ロジックを保持している必要があります。
  • 内部的に boolKEY 構造体のサイズが PG_GETARG_POINTER(1) に書き込まれます。

アルゴリズムまたは振る舞い

1. palloc を使用して boolKEY サイズのメモリ領域を確保します。 2. 出力サイズ格納用ポインタに sizeof(boolKEY) を設定します。 3. gbt_num_union を呼び出し、entryvec 内の全エントリを tinfo に基づいて統合します。 4. gbt_num_union の戻り値をそのまま返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのGiSTインデックス構築プロセス（直接的な呼び出し元関数名はソース内に明示されていません）。
  • **呼び出し先**: gbt_num_union（contrib/btree_gist/btree_utils_num.c で定義された共通ユーティリティ関数）。

関連データ構造

  • boolKEY: bool 型の範囲を表現する構造体。
  • GistEntryVector: 統合対象となる複数のインデックスエントリを保持する構造体。
  • gbtree_ninfo (tinfo): bool 型の比較関数やサイズ情報を定義する構造体。

失敗モードと境界条件

  • メモリ不足時に palloc がエラーを発生させる可能性があります。
  • entryvec が空または不正な場合、gbt_num_union の挙動に依存します。

保守担当者向けメモ

  • 本関数は btree_utils_num.c の共通ロジックを呼び出す薄いラッパーです。ロジックの変更が必要な場合は、まず gbt_num_union の仕様を確認してください。
  • PG_FUNCTION_INFO_V1(gbt_bool_union) により、PostgreSQLの関数マネージャに登録されています。
