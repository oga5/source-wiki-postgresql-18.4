LsnReadQueue

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/xlogprefetcher.c:103

目的

WALリカバリ処理における先読み（Prefetching）のI/Oリクエストを管理するための循環キューです。

シグネチャまたは形状

typedef struct LsnReadQueue
{
    LsnReadQueueNextFun next;
    uintptr_t    lrq_private;
    uint32       max_inflight;
    uint32       inflight;
    uint32       completed;
    uint32       head;
    uint32       tail;
    uint32       size;
    struct
    {
        bool     io;
        XLogRecPtr lsn;
    }            queue[FLEXIBLE_ARRAY_MEMBER];
} LsnReadQueue;

入力と出力

  • **入力**: lrq_alloc を介して初期化され、lrq_prefetch によってI/Oリクエストが追加されます。
  • **出力**: lrq_complete_lsn を通じて、完了したLSNに関連付けられたI/Oリクエストが解放されます。

前提条件と不変条件

  • queue は FLEXIBLE_ARRAY_MEMBER を使用しており、構造体生成時に size に基づいてメモリが動的に割り当てられる必要があります。
  • inflight は現在実行中のI/O数、completed は完了済みのI/O数を示します。
  • lrq_complete_lsn は、指定されたLSNより前のI/Oリクエストを考慮対象とします。

アルゴリズムまたは振る舞い

  • **循環キュー管理**: head と tail を用いて、先読みリクエストをFIFO形式で管理します。
  • **I/O追跡**: 各エントリは io フラグと lsn を保持し、リプレイ済みのWALと先読みの進捗を同期させます。
  • **動的割り当て**: palloc を使用して、指定されたサイズに応じたメモリを確保します。

呼び出し元と呼び出し先

  • **呼び出し元**: XLogPrefetcher 関連の関数（lrq_alloc, lrq_free, lrq_inflight, lrq_completed, lrq_prefetch, lrq_complete_lsn）から参照されます。
  • **呼び出し先**: 構造体内部の next フィールドは LsnReadQueueNextFun 型のコールバックを指します。

関連データ構造

  • XLogPrefetcher: 本構造体を保持し、先読み状態を管理するメイン構造体。
  • XLogRecPtr: WALの論理位置を示す型。

失敗モードと境界条件

  • **メモリ不足**: lrq_alloc 時の palloc 失敗。
  • **キューの溢れ**: max_inflight を超えるリクエストの追加（実装上の制限）。

保守担当者向けメモ

  • 本構造体は xlogprefetcher.c 内で完結しており、外部モジュールからの直接操作は推奨されません。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、サイズ計算には offsetof を使用してください。
