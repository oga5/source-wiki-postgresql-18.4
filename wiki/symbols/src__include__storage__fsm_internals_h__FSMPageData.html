FSMPageData

定義

  • 種別: struct
  • 定義箇所: src/include/storage/fsm_internals.h:24

目的

PostgreSQLの空き領域マップ（FSM）における、ページ内の物理的なメモリレイアウトを定義する構造体です。

シグネチャまたは形状

typedef struct {
    int    fp_next_slot;
    uint8  fp_nodes[FLEXIBLE_ARRAY_MEMBER];
} FSMPageData;

入力と出力

  • fp_next_slot: ラウンドロビン方式で次に割り当てるスロットを指すポインタ（入力/出力）。
  • fp_nodes: ページ内の空き容量を保持するバイナリツリー（入力/出力）。

前提条件と不変条件

  • fp_next_slot は排他ロックなしで更新されるため、アトミックな操作が可能な int 型として定義されています。
  • fp_nodes はバイナリツリー構造を保持し、前半の NonLeafNodesPerPage 要素が上位ノード、後半の LeafNodesPerPage 要素がリーフノードとして構成されます。
  • 未使用ノードの値は 0 です。

アルゴリズムまたは振る舞い

  • **負荷分散**: fp_next_slot を用いて、複数のバックエンドが同一ページに集中しないようラウンドロビンでスロットを割り当てます。
  • **ツリー管理**: fp_nodes は各リーフノードにページごとの空き容量を保持し、親ノードにはその子ノードの最大値が格納されます。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数名は、提供されたソース抜粋からは特定できません。
  • 関連する操作関数として fsm_search_avail() および fsm_set_avail() が存在することが示唆されています。

関連データ構造

  • FSMPage: FSMPageData * として定義されるポインタ型。

失敗モードと境界条件

  • ページ内のノード数は FLEXIBLE_ARRAY_MEMBER により可変であり、ページヘッダーサイズに依存します。
  • fp_nodes 内の各ノードは uint8 型であり、0〜255の範囲で空き容量のカテゴリまたは割合を示します。

保守担当者向けメモ

  • fp_next_slot はアトミックなフェッチ/ストアが期待されるため、型変更を行う際は注意が必要です。
  • ページ内のツリー構造を直接操作する場合、fsmpage.c の実装ロジックと整合性を保つ必要があります。
