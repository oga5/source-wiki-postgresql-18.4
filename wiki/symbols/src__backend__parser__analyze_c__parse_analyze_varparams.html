parse_analyze_varparams

定義

  • 種別: function
  • 定義箇所: src/backend/parser/analyze.c:144

目的

生の解析ツリー（RawStmt）を、オプティマイザーやエグゼキューターが処理可能なクエリツリー（Query）へ変換します。特に、パラメーターの型が動的に決定されるケースの解析を担います。

シグネチャまたは形状

Query *parse_analyze_varparams(RawStmt *parseTree, const char *sourceText,
                               Oid **paramTypes, int *numParams,
                               QueryEnvironment *queryEnv)

入力と出力

  • **入力**:
  • parseTree: 構文解析済みの生の解析ツリー。
  • sourceText: SQL 文のソース文字列（必須）。
  • paramTypes: パラメーター型の配列へのポインタ。
  • numParams: パラメーター数のポインタ。
  • queryEnv: クエリ実行環境。
  • **出力**:
  • 変換済みの Query 構造体へのポインタ。

前提条件と不変条件

  • sourceText は NULL であってはなりません。
  • ParseState が内部で生成され、解析終了時に解放されます。

アルゴリズムまたは振る舞い

1. make_parsestate で解析状態を初期化します。 2. setup_parse_variable_parameters を呼び出し、パラメーターの型解決を準備します。 3. transformTopLevelStmt を実行して、解析ツリーをクエリツリーへ変換します。 4. check_variable_parameters を呼び出し、パラメーター型の整合性を検証します。 5. クエリ ID の生成が有効な場合、JumbleQuery を実行します。 6. post_parse_analyze_hook が定義されていれば実行します。 7. free_parsestate で解析状態を破棄し、pgstat_report_query_id でクエリ ID を報告した後に Query を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/postgres.c:722
  • **呼び出し先**:
  • make_parsestate, setup_parse_variable_parameters, transformTopLevelStmt, check_variable_parameters, JumbleQuery, post_parse_analyze_hook, free_parsestate, pgstat_report_query_id

関連データ構造

  • RawStmt: 入力となる解析ツリー。
  • Query: 出力となるクエリツリー。
  • ParseState: 解析中に保持される状態管理構造体。
  • JumbleState: クエリのハッシュ化に使用される構造体。

失敗モードと境界条件

  • sourceText が NULL の場合、Assert により停止します。
  • パラメーター型の解決に失敗した場合、check_variable_parameters 内でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は動的パラメーターを扱うためのエントリーポイントです。固定パラメーターを扱う parse_analyze_fixedparams との使い分けに注意してください。
  • post_parse_analyze_hook を介した外部プラグインの介入があるため、フック実行時の状態管理に留意してください。
