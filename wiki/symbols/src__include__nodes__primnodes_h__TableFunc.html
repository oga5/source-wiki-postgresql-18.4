TableFunc

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/primnodes.h:110

目的

XMLTABLE や JSON_TABLE といった、外部データソースからテーブル形式の出力を生成する関数呼び出しを表現するためのノード構造体です。

シグネチャまたは形状

typedef struct TableFunc {
    NodeTag     type;
    TableFuncType functype;
    List       *ns_uris;
    List       *ns_names;
    Node       *docexpr;
    Node       *rowexpr;
    List       *colnames;
    List       *coltypes;
    List       *coltypmods;
    List       *colcollations;
    List       *colexprs;
    List       *coldefexprs;
    List       *colvalexprs;
    List       *passingvalexprs;
    Bitmapset  *notnulls;
    Node       *plan;
    int         ordinalitycol;
    ParseLoc    location;
} TableFunc;

入力と出力

  • **入力**: docexpr（対象ドキュメント）、rowexpr（行フィルタ）、colexprs（列フィルタ）、passingvalexprs（JSON_TABLE用引数）などの式ノード。
  • **出力**: 定義された列名、型、照合順序に基づき、テーブル形式の行セットを生成します。

前提条件と不変条件

  • pg_node_attr(query_jumble_ignore) が付与されたフィールドは、クエリのハッシュ化や等価性判定の対象外となります。
  • ordinalitycol は 0 からカウントされ、指定がない場合は -1 となります。
  • location はトークンの位置を示し、不明な場合は -1 となります。

アルゴリズムまたは振る舞い

実行時には、ExecEvalExpr 等の関数を通じて、指定された docexpr に対して rowexpr を適用し、各列の式（colexprs 等）を評価して結果セットを構築します。JSON_TABLE の場合は plan ノードが実行計画の一部として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/dependency.c (依存関係の解決)
  • src/backend/commands/explain.c (実行計画の表示)
  • src/backend/executor/nodeTableFuncscan.c (実行時処理)
  • src/backend/nodes/nodeFuncs.c (ノード走査)
  • **呼び出し先**:
  • 提供されたソースからは、具体的な関数呼び出し先は特定できません。

関連データ構造

  • TableFuncType: 関数種別（XMLTABLE/JSON_TABLE）を定義。
  • List: 各種リスト（列名、型、式など）を保持。
  • Bitmapset: 列の非NULL制約を管理。
  • Node: 汎用ノード型。

失敗モードと境界条件

  • location が -1 の場合、解析位置情報が欠落していることを示します。
  • ordinalitycol が -1 の場合、序数列は生成されません。
  • ns_names が NULL の場合、名前空間の指定がないことを意味します。

保守担当者向けメモ

  • 本構造体はクエリの解析・計画・実行の各段階で利用されます。
  • query_jumble_ignore 属性が付与されたフィールドは、クエリの正規化やハッシュ計算において無視されるため、変更時は注意が必要です。
