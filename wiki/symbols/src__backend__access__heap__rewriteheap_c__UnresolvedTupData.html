UnresolvedTupData

定義

  • 種別: struct
  • 定義箇所: src/backend/access/heap/rewriteheap.c:170

目的

ヒープ再構築（VACUUM FULL や CLUSTER 等）の過程で、更新チェーンの解決が一時的に保留されているタプルを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    TidHashKey    key;          /* 期待されるxmin/Bタプルの旧位置 */
    ItemPointerData old_tid;    /* Aタプルの旧ヒープ上の位置 */
    HeapTuple     tuple;        /* Aタプルの内容 */
} UnresolvedTupData;

入力と出力

  • 本構造体はデータ保持用であり、関数ではないため直接的な入力・出力はありません。
  • ハッシュテーブル（rs_unresolved_tups）の要素として格納されます。

前提条件と不変条件

  • key フィールドは、更新チェーンの解決に必要な識別子として機能します。
  • tuple は、再構築処理中に一時的にメモリ上に保持されるタプルデータです。

アルゴリズムまたは振る舞い

  • rewrite_heap_tuple 実行時、更新チェーンが即座に解決できない場合に本構造体がインスタンス化され、ハッシュテーブルに保存されます。
  • チェーンが解決可能になった時点で、保存された情報を用いて新しいヒープへの書き込みが行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/rewriteheap.c 内の rewrite_heap_tuple が、未解決タプルの保存先として本構造体を利用します。
  • 呼び出し先: 構造体自体はデータ保持用であり、関数呼び出しは含みません。

関連データ構造

  • TidHashKey: ハッシュテーブルのキーとして使用されます。
  • UnresolvedTup: UnresolvedTupData へのポインタ型です。
  • RewriteStateData: 本構造体を管理するハッシュテーブルを保持する親構造体です。

失敗モードと境界条件

  • メモリ制限: 大量の更新が未解決のまま滞留すると、インメモリのハッシュテーブルがメモリを圧迫する可能性があります。
  • 境界条件: 構造体サイズは sizeof(UnresolvedTupData) としてハッシュテーブルの初期化時に参照されます。

保守担当者向けメモ

  • 本構造体は rewriteheap.c 内の論理デコーディングおよび更新チェーン解決ロジックに深く依存しています。
  • 構造体のフィールドを変更する場合は、ハッシュテーブルのキー定義（TidHashKey）との整合性に注意してください。
