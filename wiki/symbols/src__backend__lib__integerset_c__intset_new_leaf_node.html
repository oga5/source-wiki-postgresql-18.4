intset_new_leaf_node

定義

  • 種別: function
  • 定義箇所: src/backend/lib/integerset.c:329

目的

IntegerSet 構造体に関連付けられたメモリコンテキスト内に、新しいリーフノード（intset_leaf_node）を割り当てて初期化する。

シグネチャまたは形状

static intset_leaf_node *intset_new_leaf_node(IntegerSet *intset)

入力と出力

  • **入力**: IntegerSet *intset （ノードを割り当てる対象の整数集合構造体）
  • **出力**: 初期化済みの intset_leaf_node へのポインタ

前提条件と不変条件

  • intset->context が有効なメモリコンテキストであること。
  • 割り当てられたノードの level は 0 に設定される。
  • 割り当てられたノードの num_items は 0 に設定される。
  • 割り当てられたノードの next ポインタは NULL に設定される。

アルゴリズムまたは振る舞い

1. intset->context から intset_leaf_node サイズのメモリを割り当てる。 2. GetMemoryChunkSpace を用いて割り当てたメモリサイズを計算し、intset->mem_used に加算する。 3. ノードの各メンバ（level, num_items, next）を初期値で設定する。 4. 初期化したノードポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/lib/integerset.c 内の複数箇所（intset_flush_buffered_values 等でのノード分割時など）。
  • **呼び出し先**:
  • MemoryContextAlloc
  • GetMemoryChunkSpace

関連データ構造

  • IntegerSet: 集合のルート構造体。
  • intset_leaf_node: 本関数が生成するリーフノード構造体。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAlloc がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は static であり、モジュール内部でのみ使用される。
  • mem_used の追跡は GetMemoryChunkSpace に依存しているため、メモリコンテキストの割り当てオーバーヘッドを含めた正確なサイズ管理が行われる。
