AlterEventTriggerOwner_internal

定義

  • 種別: function
  • 定義箇所: src/backend/commands/event_trigger.c:538

目的

イベントトリガーの所有者を変更し、システムカタログの更新および依存関係の再構築を行う内部関数です。

シグネチャまたは形状

static void AlterEventTriggerOwner_internal(Relation rel, HeapTuple tup, Oid newOwnerId)

入力と出力

  • rel: pg_event_trigger システムカタログのリレーション（開いている状態）。
  • tup: 変更対象のイベントトリガーを表す HeapTuple。
  • newOwnerId: 新しい所有者の Oid。
  • 出力: なし（成功時はカタログを更新し、失敗時はエラーをスロー）。

前提条件と不変条件

  • tup は pg_event_trigger カタログの有効なタプルであること。
  • 呼び出し元は適切なロックを保持していること。
  • イベントトリガーの所有者はスーパーユーザーである必要がある。

アルゴリズムまたは振る舞い

1. GETSTRUCT を用いてタプルから Form_pg_event_trigger を取得する。 2. 現在の所有者と newOwnerId が同一であれば、何もせず終了する。 3. 現在のユーザーが対象イベントトリガーの所有者であるかを確認し、権限がなければエラーをスローする。 4. newOwnerId がスーパーユーザーであることを確認し、そうでなければエラーをスローする。 5. タプルの evtowner フィールドを更新し、CatalogTupleUpdate でカタログを書き換える。 6. changeDependencyOnOwner を呼び出し、所有者に関する依存関係を更新する。 7. InvokeObjectPostAlterHook を呼び出し、変更をフックに通知する。

呼び出し元と呼び出し先

  • 呼び出し元: AlterEventTriggerOwner および AlterEventTriggerOwner_oid （src/backend/commands/event_trigger.c 内）。
  • 呼び出し先: GETSTRUCT, object_ownercheck, aclcheck_error, superuser_arg, ereport, CatalogTupleUpdate, changeDependencyOnOwner, InvokeObjectPostAlterHook。

関連データ構造

  • Form_pg_event_trigger: イベントトリガーのカタログ定義構造体。
  • Relation: システムカタログへのアクセスに使用。
  • HeapTuple: カタログタプルの操作に使用。

失敗モードと境界条件

  • **権限不足**: 実行ユーザーが所有者でない場合、ACLCHECK_NOT_OWNER エラーが発生する。
  • **スーパーユーザー要件**: 新しい所有者がスーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE エラーが発生する。

保守担当者向けメモ

  • 本関数は AlterEventTriggerOwner 系の関数から呼び出される内部処理です。所有者変更に伴う依存関係の整合性は changeDependencyOnOwner に依存しています。
