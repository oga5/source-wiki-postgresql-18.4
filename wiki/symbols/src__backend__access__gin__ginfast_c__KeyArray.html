KeyArray

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gin/ginfast.c:44

目的

GINインデックスのペンディングリスト処理において、抽出されたキー（Datum）とそれに対応するNULLカテゴリを一時的に保持・管理するための動的配列構造体です。

シグネチャまたは形状

typedef struct KeyArray
{
    Datum           *keys;          /* 拡張可能な配列 */
    GinNullCategory *categories;    /* 拡張可能な配列 */
    int32           nvalues;        /* 現在の有効なエントリ数 */
    int32           maxvalues;      /* 配列の割り当て済みサイズ */
} KeyArray;

入力と出力

  • 本構造体自体はデータ保持用であり、関数へのポインタ渡しを通じて状態が更新されます。
  • addDatum 関数により、keys および categories 配列に要素が追加され、必要に応じて maxvalues が拡張されます。

前提条件と不変条件

  • nvalues は常に maxvalues 以下であること。
  • keys および categories は、nvalues が maxvalues を超える場合に再割り当て（リサイズ）可能なメモリ領域を指していること。

アルゴリズムまたは振る舞い

  • **初期化**: initKeyArray により、指定されたサイズでメモリが割り当てられ、空の状態として初期化されます。
  • **動的拡張**: addDatum が呼び出される際、nvalues が maxvalues に達している場合は配列を拡張し、新しい要素を格納します。
  • **データ収集**: processPendingPage 等の処理において、ペンディングリストから読み取ったキー情報を一時的に蓄積するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • ginfast.c 内の ginInsertCleanup や processPendingPage 等の関数が、ペンディングリストの処理過程で本構造体をインスタンス化または操作します。
  • **呼び出し先**:
  • initKeyArray（初期化）
  • addDatum（要素追加）

関連データ構造

  • Datum: GINインデックスで扱われるキー値。
  • GinNullCategory: NULL値のカテゴリ分類。
  • BuildAccumulator: processPendingPage を通じて本構造体と連携し、データを集約します。

失敗モードと境界条件

  • メモリ不足: 配列の拡張時にメモリ割り当てが失敗した場合、システムエラーが発生します。
  • maxvalues の上限: 整数型（int32）の範囲を超える要素数は保持できません。

保守担当者向けメモ

  • 本構造体は ginfast.c 内のペンディングリスト処理専用のユーティリティです。
  • 配列の再割り当てロジック（addDatum 内）を変更する際は、メモリリークが発生しないよう注意してください。
