recompute_limits

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeLimit.c:352

目的

LIMIT および OFFSET 句の式を再評価し、LimitState ノードの内部状態をリセットして、下位ノードに対して必要なタプル数を通知する。

シグネチャまたは形状

static void recompute_limits(LimitState *node)

入力と出力

  • **入力**: LimitState *node（LIMIT/OFFSET 処理の状態を保持するノード構造体）
  • **出力**: なし（副作用として node のメンバおよび下位ノードの状態を更新）

前提条件と不変条件

  • node->ps.ps_ExprContext が有効な式コンテキストを指していること。
  • limitOffset または limitCount が設定されている場合、それらは評価可能な式であること。

アルゴリズムまたは振る舞い

1. **OFFSET の評価**: node->limitOffset が存在する場合、式を評価し node->offset に格納する。NULL の場合は 0 とする。負の値はエラーとする。 2. **LIMIT の評価**: node->limitCount が存在する場合、式を評価し node->count に格納する。NULL の場合は LIMIT ALL と見なし node->noCount を true に設定する。負の値はエラーとする。 3. **状態のリセット**: 走査位置 node->position を 0 に、node->subSlot を NULL にリセットする。 4. **状態遷移**: node->lstate を LIMIT_RESCAN に設定する。 5. **通知**: compute_tuples_needed(node) を呼び出し、その結果を ExecSetTupleBound を通じて下位ノード（outerPlanState(node)）に伝達する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeLimit.c 内の ExecLimit および ExecReScanLimit から呼び出される。
  • **呼び出し先**:
  • ExecEvalExprSwitchContext（式評価）
  • ereport（エラー報告）
  • compute_tuples_needed（必要なタプル数の計算）
  • ExecSetTupleBound（下位ノードへの通知）

関連データ構造

  • LimitState: LIMIT/OFFSET 実行ノードの実行時状態を保持する構造体。

失敗モードと境界条件

  • **負の値**: OFFSET または LIMIT に負の値が指定された場合、それぞれ ERRCODE_INVALID_ROW_COUNT_IN_RESULT_OFFSET_CLAUSE および ERRCODE_INVALID_ROW_COUNT_IN_LIMIT_CLAUSE でエラー終了する。
  • **NULL 値**: OFFSET が NULL の場合は 0 として扱い、LIMIT が NULL の場合は LIMIT ALL（制限なし）として扱う。

保守担当者向けメモ

  • 本関数は再スキャン時にも呼び出されるため、最適化を目的とした ExecSetTupleBound の呼び出し省略は避けること（前回の実行結果と異なる可能性があるため）。
