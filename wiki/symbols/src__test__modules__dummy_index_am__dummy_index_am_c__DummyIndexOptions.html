DummyIndexOptions

定義

  • 種別: struct
  • 定義箇所: src/test/modules/dummy_index_am/dummy_index_am.c:37

目的

dummy_index_am において、インデックス作成時に指定可能なカスタムリレーションオプション（int, real, bool, enum, string）を保持するための構造体です。

シグネチャまたは形状

typedef struct DummyIndexOptions
{
    int32       vl_len_;        /* varlena header */
    int         option_int;
    double      option_real;
    bool        option_bool;
    DummyAmEnum option_enum;
    int         option_string_val_offset;
    int         option_string_null_offset;
} DummyIndexOptions;

入力と出力

  • 入力: インデックス作成時のリレーションオプション設定。
  • 出力: dioptions 関数等を通じて解析されたオプション値が本構造体に格納され、インデックスのメタデータとして利用されます。

前提条件と不変条件

  • vl_len_ は可変長データ（varlena）のヘッダーであり、直接操作してはなりません。
  • 各フィールドは di_relopt_tab テーブル内のオフセット定義と一致している必要があります。

アルゴリズムまたは振る舞い

本構造体は、PostgreSQLの reloptions 機構と連携します。_PG_init で登録されたオプション定義に基づき、解析された値がこの構造体の各フィールドにマッピングされます。

呼び出し元と呼び出し先

  • 呼び出し元: dioptions 関数（リレーションオプションの解析時）および di_relopt_tab の定義箇所。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • DummyAmEnum: option_enum フィールドで使用される列挙型。
  • di_relopt_tab: 本構造体の各フィールドへのオフセットを保持する解析テーブル。

失敗モードと境界条件

  • offsetof を使用してフィールド位置を特定しているため、構造体のメンバ順序を変更する場合は di_relopt_tab の定義も同時に更新する必要があります。

保守担当者向けメモ

  • 本構造体は dummy_index_am のテンプレートとしての役割を果たすため、新しいオプションを追加する際は、本構造体へのメンバ追加と di_relopt_tab への登録をセットで行ってください。
