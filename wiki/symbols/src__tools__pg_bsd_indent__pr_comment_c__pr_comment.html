pr_comment

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/pr_comment.c:78

目的

C言語ソースコード内のコメントを解析し、整形ルール（ボックスコメントか否か）を判定した上で、出力バッファへ適切に書き出す。

シグネチャまたは形状

void pr_comment(void)

入力と出力

  • **入力**: グローバル変数 buf_ptr が指す入力バッファ内のコメント文字列。
  • **出力**: グローバルバッファ e_com を通じて整形済みコメントを出力。dump_line() を呼び出し、必要に応じて出力バッファをフラッシュする。

前提条件と不変条件

  • ps (parser_state) グローバル構造体が現在のパーサー状態を保持していること。
  • コメントの開始位置が字句解析器によって検知されていること。
  • CHECK_SIZE_COM マクロにより、コメントバッファのオーバーフローが防止されていること。

アルゴリズムまたは振る舞い

1. **初期化と分類**: ps.box_com を初期化し、コメントの内容（- や * の有無、改行位置）に基づき、整形を抑制する「ボックスコメント」か否かを判定する。 2. **配置計算**: コードのインデントレベルや宣言の有無に基づき、コメントの開始カラム（ps.com_col）を計算する。 3. **バッファ書き込み**: /* を出力バッファに書き込み、必要に応じて空白をスキップする。 4. **メインループ**: buf_ptr を走査し、以下の文字を処理する。

  • 014 (フォームフィード): ボックスコメント以外では改行を強制し、整形を行う。
  • \n (改行): ボックスコメントの場合は維持し、それ以外では行の継続を考慮して整形する。
  • * (アスタリスク): コメント終了 (*/) を判定する。
  • default: テキストをバッファへコピーし、行長制限を超えた場合は dump_line() を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/tools/pg_bsd_indent/indent.c (1187行目付近)
  • **呼び出し先**: dump_line(), fill_buffer(), count_spaces(), count_spaces_until(), compute_code_target(), compute_label_target()

関連データ構造

  • parser_state (ps): コメントの整形状態（box_com, com_col, n_comment_delta 等）を保持。
  • buf_ptr: 入力バッファの現在位置。
  • e_com: 出力コメントバッファの末尾ポインタ。

失敗モードと境界条件

  • **未終了コメント**: had_eof が真の状態で改行に遭遇した場合、「Unterminated comment」を出力して終了する。
  • **バッファオーバーフロー**: CHECK_SIZE_COM マクロによる保護が不十分な場合、バッファ破壊の可能性がある。
  • **ボックスコメント**: ps.box_com が真の場合、空白や改行の整形は抑制される。

保守担当者向けメモ

  • 本関数は1980年代のコードベースに由来し、複雑なポインタ操作と goto 文が含まれる。
  • ps.box_com フラグが true の場合、コメント内の空白や改行は極力維持される。
  • 修正時は CHECK_SIZE_COM マクロによるバッファサイズ管理を厳守すること。
