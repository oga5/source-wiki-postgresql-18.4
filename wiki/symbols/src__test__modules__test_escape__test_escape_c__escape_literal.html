escape_literal

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_escape/test_escape.c:250

目的

PQescapeLiteral を呼び出し、その結果を PQExpBuffer に格納またはエラー情報を記録するラッパー関数です。

シグネチャまたは形状

static bool escape_literal(PGconn *conn, PQExpBuffer target, const char *unescaped, size_t unescaped_len, PQExpBuffer escape_err)

入力と出力

  • **入力**:
  • conn: PostgreSQL接続オブジェクト。
  • target: エスケープ後の文字列を格納するバッファ。
  • unescaped: エスケープ対象の入力文字列。
  • unescaped_len: 入力文字列の長さ。
  • escape_err: エラー発生時にエラーメッセージを格納するバッファ。
  • **出力**:
  • 成功時に true を返し、target にエスケープ済み文字列を追記。
  • 失敗時に false を返し、escape_err にエラーメッセージを追記。

前提条件と不変条件

  • conn は有効な接続状態である必要があります。
  • PQescapeLiteral がメモリを割り当てた場合、本関数内で PQfreemem を呼び出し解放します。

アルゴリズムまたは振る舞い

1. PQescapeLiteral を呼び出し、エスケープ処理を実行します。 2. 結果が NULL の場合（失敗）:

  • PQerrorMessage を取得し、escape_err バッファに追記します。
  • バッファ末尾の改行文字を除去し、false を返します。

    • 3. 結果が NULL でない場合（成功）:

  • target バッファにエスケープ済み文字列を追記します。
  • PQfreemem でメモリを解放し、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_escape/test_escape.c 内の pe_test_escape_func 構造体定義（402行目）にて参照されています。
  • **呼び出し先**: PQescapeLiteral, appendPQExpBufferStr, PQerrorMessage, PQfreemem（すべて libpq ライブラリ関数）。

関連データ構造

  • PGconn: PostgreSQL接続ハンドル。
  • PQExpBuffer: 文字列バッファ管理構造体。

失敗モードと境界条件

  • PQescapeLiteral がメモリ不足等で NULL を返した場合、エラーメッセージの取得とバッファ調整が行われます。
  • escape_err の末尾から改行文字を削除する際、バッファの長さが正であることを前提としています。

保守担当者向けメモ

  • 本関数はテストスイートの一部であり、PQescapeLiteral の挙動を検証するためのインターフェースとして機能します。
  • escape_err の末尾文字削除処理は、PQerrorMessage が改行を含むことを前提とした実装です。
