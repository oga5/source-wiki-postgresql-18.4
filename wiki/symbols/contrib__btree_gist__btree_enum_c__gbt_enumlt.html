gbt_enumlt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_enum.c:58

目的

PostgreSQL の列挙型（ENUM）に対する GiST インデックス操作において、2つの ENUM 値の「より小さい（less than）」比較を行うためのアダプター関数です。

シグネチャまたは形状

static bool gbt_enumlt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の ENUM 値を指すポインタ（内部的に Oid として扱われる）。
  • flinfo: PostgreSQL の関数マネージャー情報。
  • **出力**:
  • bool: a < b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は有効な Oid を指している必要があります。
  • flinfo は enum_lt 関数を呼び出すために適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. a と b が指す値を Oid として取得します。 2. ObjectIdGetDatum を使用して、これらを PostgreSQL の Datum 形式に変換します。 3. CallerFInfoFunctionCall2 を介して、標準の ENUM 比較関数である enum_lt を呼び出します。 4. 呼び出し結果を DatumGetBool で論理値に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_enum.c 内の tinfo 構造体定義（GiST インデックスの比較ロジックとして参照）。
  • **呼び出し先**: enum_lt（PostgreSQL 標準の ENUM 比較関数）。

関連データ構造

  • tinfo: gbtree_ninfo 構造体。本関数を比較演算子として保持します。
  • Oid: ENUM 型の内部表現。

失敗モードと境界条件

  • a または b が無効なポインタである場合、セグメンテーション違反が発生する可能性があります。
  • flinfo が不適切な場合、CallerFInfoFunctionCall2 内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.c の汎用関数からコールバックとして呼び出されることを前提としています。
  • ENUM 型の順序は OID の順序に依存するため、OID の比較ロジックと整合性を保つ必要があります。
