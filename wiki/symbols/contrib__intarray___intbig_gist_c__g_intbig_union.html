g_intbig_union

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_intbig_gist.c:267

目的

GiST インデックスにおける複数の intbig エントリを統合し、それらを包含する単一のビットベクトル（シグネチャ）を生成します。

シグネチャまたは形状

Datum g_intbig_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GistEntryVector *entryvec（統合対象のインデックスエントリ群）。
  • PG_GETARG_POINTER(1): int *size（結果のサイズを格納する出力用ポインタ）。
  • **出力**:
  • 戻り値: Datum（統合された GISTTYPE 構造体へのポインタ）。
  • *size: 統合されたエントリのバイトサイズが書き込まれます。

前提条件と不変条件

  • GET_SIGLEN() により、現在のインデックス設定に基づいたシグネチャ長が取得可能であること。
  • _intbig_alloc によりメモリが確保されること。

アルゴリズムまたは振る舞い

1. _intbig_alloc を用いて、新しい GISTTYPE 構造体を初期化します。 2. entryvec 内の各エントリを順次走査し、unionkey を用いてビットベクトルを統合します。 3. 統合の過程で ALLISTRUE フラグが立つ（すべてのビットがセットされる）場合、即座に処理を中断し、その状態を反映したサイズを計算して終了します。 4. 最終的なサイズを *size に格納し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス管理機構（直接的な呼び出し元コードはソース内に明示されていませんが、PG_FUNCTION_INFO_V1 マクロにより登録されています）。
  • **呼び出し先**: _intbig_alloc, GET_SIGLEN, GETSIGN, GETENTRY, unionkey, SET_VARSIZE, CALCGTSIZE, VARSIZE, PG_RETURN_POINTER。

関連データ構造

  • GistEntryVector: 統合対象となる複数のインデックスエントリを保持する構造体。
  • GISTTYPE: intbig 型の GiST インデックスエントリを表現する構造体。
  • BITVECP: ビットベクトルを操作するためのポインタ型。

失敗モードと境界条件

  • entryvec が空の場合の挙動については、本ソースコード内では明示的なガード句は確認できません。
  • unionkey の結果として ALLISTRUE が設定された場合、ループを早期終了する最適化が行われます。

保守担当者向けメモ

  • 本関数は intbig 型のインデックス統合を担うため、siglen の変更がインデックスの物理構造に直接影響します。
  • ALLISTRUE フラグの扱いは、インデックスの「損失あり（lossy）」特性を維持するために重要です。
