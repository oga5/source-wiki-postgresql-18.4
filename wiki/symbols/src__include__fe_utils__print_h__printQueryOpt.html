printQueryOpt

定義

  • 種別: struct
  • 定義箇所: src/include/fe_utils/print.h:183

目的

クエリ結果の整形出力を行う際、標準的なテーブル出力設定（printTableOpt）を拡張し、クエリ固有の表示オプション（NULL値の表現、タイトル、フッター、翻訳設定など）を保持するために使用されます。

シグネチャまたは形状

typedef struct printQueryOpt
{
	printTableOpt topt;             /* 基本的な出力設定 */
	char         *nullPrint;        /* NULL値の表示文字列 */
	char         *title;            /* タイトルのオーバーライド */
	char        **footers;          /* フッターのオーバーライド */
	bool         translate_header;  /* カラムヘッダーの翻訳フラグ */
	const bool  *translate_columns; /* 各カラムの翻訳フラグ配列 */
	int          n_translate_columns; /* translate_columnsの長さ */
} printQueryOpt;

入力と出力

本構造体は、クエリ結果を出力する関数（printQuery 等）に対する設定入力として機能します。出力自体は行いませんが、出力処理の挙動を制御するパラメータ群を保持します。

前提条件と不変条件

  • topt フィールドは、テーブル出力の基本設定（罫線やページャ設定など）を保持する printTableOpt 型である必要があります。
  • n_translate_columns は translate_columns 配列の要素数と一致している必要があります。

アルゴリズムまたは振る舞い

  • psql 等のフロントエンドにおいて、ユーザー設定の保存・復元（savePsetInfo / restorePsetInfo）や、特定のクエリ実行時の表示設定のオーバーライドに使用されます。
  • translate_columns を通じて、特定のカラムに対して個別に gettext による翻訳を適用するかどうかを制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c 内の do_pset、printPsetInfo、pset_value_string、savePsetInfo、restorePsetInfo などから参照・操作されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • printTableOpt: printQueryOpt の先頭メンバとして含まれる、基本的な出力設定構造体。

失敗モードと境界条件

  • memcpy を用いた構造体のコピーが行われる箇所（src/bin/psql/command.c:5629 等）があるため、ポインタメンバ（nullPrint, title, footers 等）のメモリ管理には注意が必要です。浅いコピー（shallow copy）となるため、元の構造体が解放された後のポインタ参照は無効となります。

保守担当者向けメモ

  • 本構造体は psql の設定管理と密接に関連しています。フィールドを追加・変更する場合は、command.c 内のコピー処理やメモリ管理ロジックに影響がないか確認してください。
