shm_mq_set_handle

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shm_mq.c:318

目的

共有メモリメッセージキューのハンドル (shm_mq_handle) に、対応するバックグラウンドワーカーのハンドル (BackgroundWorkerHandle) を関連付けます。

シグネチャまたは形状

void shm_mq_set_handle(shm_mq_handle *mqh, BackgroundWorkerHandle *handle)

入力と出力

  • mqh: 設定対象となる shm_mq_handle 構造体へのポインタ。
  • handle: 関連付ける BackgroundWorkerHandle 構造体へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • mqh->mqh_handle が NULL であること（アサーションにより検証）。

アルゴリズムまたは振る舞い

1. mqh->mqh_handle が NULL であることを確認する。 2. 引数として渡された handle を mqh->mqh_handle に代入する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/parallel.c:630
  • src/backend/executor/execParallel.c:904
  • 呼び出し先:
  • 本関数内での関数呼び出しはありません。

関連データ構造

  • shm_mq_handle: キューへのアクセス用プライベートハンドル。
  • BackgroundWorkerHandle: バックグラウンドワーカーを管理するためのハンドル。

失敗モードと境界条件

  • mqh->mqh_handle が既に設定されている（NULL ではない）場合、アサーション失敗によりプロセスが異常終了します。

保守担当者向けメモ

  • 本関数は、キューの所有者がバックグラウンドワーカーのライフサイクルを監視できるようにするために使用されます。
  • 呼び出し元で BackgroundWorkerHandle が適切に初期化されていることを確認してください。
