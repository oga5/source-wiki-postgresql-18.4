CreateTableSpace

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tablespace.c:207

目的

SQL コマンド CREATE TABLESPACE を実行し、データベースオブジェクトを格納するための新しいテーブルスペースをシステムに登録および物理作成する。

シグネチャまたは形状

Oid CreateTableSpace(CreateTableSpaceStmt *stmt)

入力と出力

  • **入力**: CreateTableSpaceStmt *stmt（テーブルスペース名、場所、所有者、オプションを含む構文解析木）。
  • **出力**: 作成されたテーブルスペースの Oid を返す。

前提条件と不変条件

  • 実行ユーザーはスーパーユーザーである必要がある。
  • 指定された場所（パス）は絶対パスである必要がある（allow_in_place_tablespaces が有効かつパスが空の場合を除く）。
  • テーブルスペース名はシステム予約語（pg_ で始まる名前）であってはならない。
  • 同名のテーブルスペースが既に存在してはならない。
  • 指定されたパスがデータディレクトリ内にある場合は警告が発行される。

アルゴリズムまたは振る舞い

1. **権限および引数検証**: スーパーユーザー権限の確認、パスの正規化、パスの長さ制限チェック、および予約語チェックを行う。 2. **カタログ登録**: pg_tablespace システムカタログに新しいタプルを挿入し、所有者との依存関係を記録する。 3. **物理ディレクトリ作成**: create_tablespace_directories を呼び出し、指定されたパスに物理的なディレクトリ構造を作成する。 4. **WAL 記録**: ファイルシステムの変更を XLOG に記録し、スタンバイ側での再現を可能にする。 5. **同期**: ForceSyncCommit を呼び出し、ディスクへの書き込みを強制する。 6. **終了処理**: pg_tablespace へのロックを保持したまま、作成された OID を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（SQL 解析後の実行フェーズ）。
  • **呼び出し先**: create_tablespace_directories（物理ディレクトリ作成）、table_open/table_close（カタログ操作）、XLogInsert（WAL 記録）、recordDependencyOnOwner（依存関係記録）。

関連データ構造

  • CreateTableSpaceStmt: SQL 文の解析結果を保持する構造体。
  • pg_tablespace: テーブルスペース情報を管理するシステムカタログ。
  • xl_tblspc_create_rec: WAL 記録用のレコード構造体。

失敗モードと境界条件

  • **権限エラー**: スーパーユーザー以外が実行した場合。
  • **重複エラー**: 同名のテーブルスペースが既に存在する場合。
  • **パスエラー**: パスが長すぎる、または絶対パスではない場合。
  • **バイナリアップグレード**: IsBinaryUpgrade モード時に OID が設定されていない場合。

保守担当者向けメモ

  • 本関数は pg_tablespace へのロックを保持したまま終了するため、呼び出し側でトランザクションのコミットが完了するまでロックが維持される点に注意すること。
  • create_tablespace_directories の失敗はトランザクションのロールバックを誘発する。
