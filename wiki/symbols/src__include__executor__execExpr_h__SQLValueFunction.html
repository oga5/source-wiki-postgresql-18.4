SQLValueFunction

定義

  • 種別: struct
  • 定義箇所: src/include/executor/execExpr.h:465

目的

SQL標準で定義される値関数（CURRENT_USER, CURRENT_DATE 等）の式評価をサポートするためのデータ構造です。

シグネチャまたは形状

ExprEvalStep 構造体内の union d メンバとして保持されます。

struct {
    SQLValueFunction *svf;
} sqlvaluefunction;

入力と出力

  • **入力**: 実行時に ExprEvalStep から取得される SQLValueFunction 構造体へのポインタ。
  • **出力**: 評価結果は ExprEvalStep の resvalue および resnull に格納されます。

前提条件と不変条件

  • SQLValueFunction は式評価のコンテキストにおいて安定（stable）であると見なされます。
  • 実行エンジンは、この構造体を通じて適切な SQL 値関数を特定し、評価を実行します。

アルゴリズムまたは振る舞い

1. 実行エンジンは ExprEvalStep の opcode を参照し、SQLValueFunction 評価ステップを識別します。 2. op->d.sqlvaluefunction.svf から対象の関数情報を取得します。 3. 取得した情報に基づき、対応する SQL 値関数を評価し、結果を後続のステップへ渡します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execExprInterp.c 内の式評価ループから参照されます。
  • **呼び出し先**: 実行エンジンが内部的に保持する SQL 値関数評価ロジック。
  • 補足: 呼び出し元・呼び出し先の詳細な関数コールグラフは提供されていません。

関連データ構造

  • ExprEvalStep: 本構造体を保持する評価ステップの単位。
  • ExprState: 式評価の実行状態を管理する構造体。

失敗モードと境界条件

  • 構造体内の type や typmod が不正な場合、ノード走査時にアサーションエラーが発生する可能性があります。
  • 評価対象の関数がサポートされていない場合、実行時にエラーとなる可能性があります。

保守担当者向けメモ

  • SQLValueFunction は src/backend/nodes/nodeFuncs.c 等のノード操作関数で頻繁に参照されます。
  • 最適化フェーズ（costsize.c や clauses.c）において、このノードが安定であるという前提が利用されているため、変更時は注意してください。
