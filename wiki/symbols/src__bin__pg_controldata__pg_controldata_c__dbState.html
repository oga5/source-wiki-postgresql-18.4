dbState

定義

  • 種別: function
  • 定義箇所: src/bin/pg_controldata/pg_controldata.c:49

目的

PostgreSQLのデータベース状態を示す DBState 列挙型の値を、人間が判読可能な文字列に変換するヘルパー関数です。

シグネチャまたは形状

static const char *dbState(DBState state)

入力と出力

  • **入力**: DBState 型の列挙値（データベースの現在の状態）。
  • **出力**: 状態に対応する翻訳済みの文字列（const char *）。認識できない値の場合は "unrecognized status code" を返します。

前提条件と不変条件

  • 入力値は DBState 列挙型で定義された有効な値であることが期待されます。
  • _() マクロにより、実行環境のロケールに応じた翻訳が行われます。

アルゴリズムまたは振る舞い

1. switch 文を使用して、入力された state を評価します。 2. 以下の各状態に対して対応する文字列を返します：

  • DB_STARTUP: "starting up"
  • DB_SHUTDOWNED: "shut down"
  • DB_SHUTDOWNED_IN_RECOVERY: "shut down in recovery"
  • DB_SHUTDOWNING: "shutting down"
  • DB_IN_CRASH_RECOVERY: "in crash recovery"
  • DB_IN_ARCHIVE_RECOVERY: "in archive recovery"
  • DB_IN_PRODUCTION: "in production"

    • 3. 上記のいずれにも該当しない場合、デフォルトとして "unrecognized status code" を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_controldata/pg_controldata.c 内のメイン処理（main 関数内）から、制御ファイル内の状態を表示する際に呼び出されます。
  • **呼び出し先**: 翻訳マクロ _() を呼び出します。

関連データ構造

  • DBState: データベースの状態を定義する列挙型（catalog/pg_control.h で定義）。
  • ControlFileData: 制御ファイルの内容を保持する構造体。本関数はこの構造体内の state フィールドを処理するために使用されます。

失敗モードと境界条件

  • 定義されていない DBState 値が渡された場合、エラーを発生させるのではなく、"unrecognized status code" という文字列を返して処理を継続します。

保守担当者向けメモ

  • 新しいデータベース状態が DBState に追加された場合は、本関数の switch 文にも必ずケースを追加してください。
  • 翻訳対象の文字列を変更する場合は、PostgreSQLの国際化ガイドラインに従ってください。
