sha1_step

定義

  • 種別: function
  • 定義箇所: src/common/sha1.c:89

目的

64バイトのデータブロックに対して SHA-1 アルゴリズムの変換関数を適用し、内部ハッシュ状態を更新します。

シグネチャまたは形状

static void sha1_step(pg_sha1_ctx *ctx)

入力と出力

  • **入力**: pg_sha1_ctx *ctx（ハッシュ計算のコンテキスト。処理対象の64バイトデータおよび現在のハッシュ状態 H を保持）
  • **出力**: なし（副作用として ctx 内のハッシュ状態 H を更新し、処理済みブロックバッファをゼロクリアする）

前提条件と不変条件

  • ctx->m には処理対象となる64バイトのデータが格納されている必要があります。
  • リトルエンディアン環境では、内部でバイト順序の入れ替えが行われ、ビッグエンディアンとして処理されます。

アルゴリズムまたは振る舞い

1. **エンディアン変換**: WORDS_BIGENDIAN が定義されていない場合、64バイトのブロックに対してバイト順序の入れ替えを行い、ビッグエンディアン形式に正規化します。 2. **状態初期化**: コンテキスト内の現在のハッシュ値 H(0)〜H(4) をローカル変数 a〜e にコピーします。 3. **変換ループ**: 80回の反復処理を行い、以下のステップを繰り返します。

  • メッセージスケジュール W の生成。
  • 非線形関数（F0〜F3）、ビット回転シフト（S）、および定数 K を用いた状態更新。

    • 4. **状態加算**: 計算結果を元のハッシュ値 H に加算します。 5. **クリーンアップ**: 処理後のブロックバッファを memset でゼロクリアします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/sha1.c 内の pg_sha1_update およびパディング処理に関連する箇所から呼び出されます。
  • **呼び出し先**: 内部マクロ H(), W(), S(), F0()〜F3(), K() を使用します。これらマクロの定義元は sha1_int.h です。

関連データ構造

  • pg_sha1_ctx: SHA-1 計算の状態を保持する構造体（sha1_int.h で定義）。

失敗モードと境界条件

  • 入力データが64バイト未満の場合の挙動は、本関数の呼び出し元である pg_sha1_update 等の制御に依存します。
  • エンディアン変換ロジックは、プラットフォームのバイトオーダーに強く依存します。

保守担当者向けメモ

  • 本実装は RFC 3174 に基づくフォールバック実装です。
  • WORDS_BIGENDIAN マクロによる条件分岐が含まれており、移植性を確認する際はバイトオーダーの処理に注意してください。
  • ハードウェアアクセラレーション（SHA-NI等）は含まれていません。
