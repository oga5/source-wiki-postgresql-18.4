pgssStoreKind

定義

  • 種別: enum
  • 定義箇所: contrib/pg_stat_statements/pg_stat_statements.c:119

目的

pg_stat_statements において、統計情報の対象がプランニング（PGSS_PLAN）か実行（PGSS_EXEC）かを識別するために使用されます。

シグネチャまたは形状

typedef enum pgssStoreKind
{
	PGSS_INVALID = -1,
	PGSS_PLAN = 0,
	PGSS_EXEC,
} pgssStoreKind;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: Counters 構造体内の配列インデックスとして利用される。

前提条件と不変条件

  • PGSS_PLAN は 0、PGSS_EXEC は 1 である必要があります。
  • この順序は Counters 構造体内の配列参照および pg_stat_statements_internal() 関数での処理において必須の不変条件です。

アルゴリズムまたは振る舞い

  • 統計情報の種別を明示的に指定することで、同一クエリに対するプランニング時間と実行時間を区別して集計することを可能にします。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • contrib/pg_stat_statements/pg_stat_statements.c:351
  • contrib/pg_stat_statements/pg_stat_statements.c:1284
  • 呼び出し先: なし（列挙型のため、関数呼び出しは発生しません）。

関連データ構造

  • Counters: この列挙値を使用して、プランニング用と実行用の統計値を配列から選択します。

失敗モードと境界条件

  • PGSS_INVALID（値: -1）は、無効な状態を示すために定義されています。これ以外の値を Counters のインデックスとして使用した場合、メモリ破壊や未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • PGSS_PLAN と PGSS_EXEC の数値は、Counters 構造体の配列レイアウトと密接に結合しています。将来的に新しい種別を追加する場合、既存のインデックス値（0, 1）を変更しないよう注意してください。
