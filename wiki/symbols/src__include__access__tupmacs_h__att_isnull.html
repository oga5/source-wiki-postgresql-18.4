att_isnull

定義

  • 種別: function
  • 定義箇所: src/include/access/tupmacs.h:25

目的

タプルのNULLビットマップを参照し、指定された属性番号（ATT）に対応する属性がNULLであるかどうかを判定する。

シグネチャまたは形状

static inline bool att_isnull(int ATT, const bits8 *BITS)

入力と出力

  • **入力**:
  • ATT: 判定対象の属性番号（0始まり）。
  • BITS: タプルのNULLビットマップを指すポインタ（bits8 *）。
  • **出力**:
  • 属性がNULLであれば true、そうでなければ false を返す。

前提条件と不変条件

  • BITS は、対象タプルのNULLビットマップの先頭を指している必要がある。
  • ATT は、タプル内の有効な属性番号であること。

アルゴリズムまたは振る舞い

1. ATT >> 3 により、属性番号に対応するビットマップ内のバイトオフセットを算出する。 2. ATT & 0x07 により、そのバイト内でのビット位置を算出する。 3. ビット演算 (BITS[byte_offset] & (1 << bit_position)) を行い、対象ビットが立っているかを確認する。 4. ビットが立っていない（0である）場合にNULLと見なし、論理否定 ! を適用して true を返す。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • contrib/amcheck/verify_heapam.c
  • contrib/pageinspect/heapfuncs.c
  • src/backend/access/brin/brin_tuple.c
  • src/backend/access/common/heaptuple.c
  • src/backend/access/common/indextuple.c
  • **呼び出し先**:
  • 外部関数呼び出しなし（ビット演算のみ）。

関連データ構造

  • bits8: NULLビットマップを構成する基本型。
  • HeapTupleHeaderData（t_bits メンバ）: この関数が参照するビットマップの典型的な格納先。

失敗モードと境界条件

  • ATT が負の値や、タプルの属性数を超過する値である場合、メモリ破壊や誤った判定を招く可能性がある。本関数自体には境界チェックは含まれない。

保守担当者向けメモ

  • 本関数は非常に頻繁に呼び出される低レベルなユーティリティであるため、インライン化によるパフォーマンス維持が重要である。
  • brin_tuple.c の一部の箇所では、この関数の戻り値を反転させて使用するロジックが存在するため、修正時には注意が必要である。
