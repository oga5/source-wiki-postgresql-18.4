pglz_find_match

定義

  • 種別: function
  • 定義箇所: src/common/pg_lzcompress.c:398

目的

LZ77圧縮アルゴリズムにおいて、入力データと過去の履歴データの間で最長一致（Longest Match）を検索し、圧縮のためのオフセットと長さを特定します。

シグネチャまたは形状

static inline int pglz_find_match(
    int16 *hstart, 
    const char *input, 
    const char *end,
    int *lenp, 
    int *offp, 
    int good_match, 
    int good_drop, 
    int mask
)

入力と出力

  • **入力**:
  • hstart: ハッシュテーブルの開始ポインタ。
  • input: 現在の入力位置。
  • end: 入力データの終端。
  • good_match, good_drop: 検索を早期終了するための閾値と減衰率。
  • mask: ハッシュインデックス計算用のマスク。
  • **出力**:
  • lenp: 見つかった一致の長さ（戻り値が1の場合に設定）。
  • offp: 見つかった一致のオフセット（戻り値が1の場合に設定）。
  • **戻り値**: 2バイトを超える一致が見つかった場合は 1、それ以外は 0。

前提条件と不変条件

  • 検索対象のオフセットは 0x0fff（4095バイト）未満である必要があります。
  • 一致長が2バイト以下の場合は、圧縮効率が上がらないため一致なしとみなされます。

アルゴリズムまたは振る舞い

1. hstart と mask を用いてハッシュインデックスを計算し、履歴リスト（PGLZ_HistEntry）を辿ります。 2. 各履歴エントリに対し、現在の入力位置との比較を行います。

  • 一致長が16バイト以上の場合は memcmp を使用して高速に比較し、その後1バイトずつ詳細に比較します。
  • 一致長が16バイト未満の場合は、1バイトずつ比較します。

    • 3. 最も長い一致が見つかった場合、その長さとオフセットを保持します。 4. 検索中、good_match 閾値に達した場合は早期終了します。リストを辿るごとに good_match は good_drop に基づいて減少します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/pg_lzcompress.c 内の圧縮処理（pglz_compress 等）から呼び出されます。
  • **呼び出し先**: memcmp（標準ライブラリ）、pglz_hist_idx（マクロ）。

関連データ構造

  • PGLZ_HistEntry: 履歴検索のためのリンクリスト構造体。

失敗モードと境界条件

  • **オフセット超過**: thisoff >= 0x0fff となった場合、そのエントリ以降の検索を中断します。
  • **一致なし**: 3バイト以上の重複が見つからない場合、関数は 0 を返し、圧縮タグは生成されません。
  • **終端到達**: input が end に達した時点で比較を終了します。

保守担当者向けメモ

  • pglz_hist_add などのマクロは引数を複数回評価するため、副作用のある式を渡さないよう注意してください。
  • 伸長時の memcpy と同様、圧縮時の比較ロジックもパフォーマンスと正確性のバランスが重要です。
