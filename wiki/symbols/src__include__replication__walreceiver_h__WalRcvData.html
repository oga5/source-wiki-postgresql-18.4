WalRcvData

定義

  • 種別: struct
  • 定義箇所: src/include/replication/walreceiver.h:57

目的

スタンバイサーバーにおけるWALレシーバープロセスの状態を共有メモリ上で管理するための構造体です。プロセス間での受信位置の共有、状態遷移の制御、およびプライマリサーバーとの通信パラメータの保持を担います。

シグネチャまたは形状

typedef struct {
    ProcNumber procno;
    pid_t pid;
    WalRcvState walRcvState;
    ConditionVariable walRcvStoppedCV;
    pg_time_t startTime;
    XLogRecPtr receiveStart;
    TimeLineID receiveStartTLI;
    XLogRecPtr flushedUpto;
    TimeLineID receivedTLI;
    XLogRecPtr latestChunkStart;
    TimestampTz lastMsgSendTime;
    TimestampTz lastMsgReceiptTime;
    XLogRecPtr latestWalEnd;
    TimestampTz latestWalEndTime;
    char conninfo[MAXCONNINFO];
    char sender_host[NI_MAXHOST];
    int sender_port;
    char slotname[NAMEDATALEN];
    bool is_temp_slot;
    bool ready_to_display;
    slock_t mutex;
    pg_atomic_uint64 writtenUpto;
    sig_atomic_t force_reply;
} WalRcvData;

入力と出力

  • **入力**: スタートアッププロセスが receiveStart や receiveStartTLI を設定し、ストリーミングの開始位置を指示します。
  • **出力**: WALレシーバーが受信したWALをディスクへ書き込むたびに flushedUpto が更新され、スタートアッププロセスがリプレイ可能な位置を把握するために参照されます。

前提条件と不変条件

  • 共有メモリ上に配置されるため、構造体内の変数（writtenUpto を除く）へのアクセスには mutex によるロックが必須です。
  • writtenUpto はアトミック操作によりロックなしで読み取り可能ですが、データ整合性の保証には使用できません。

アルゴリズムまたは振る舞い

1. **起動**: スタートアッププロセスが共有メモリを更新し、WALレシーバーを起動します。 2. **受信と更新**: WALレシーバーはプライマリからWALを取得し、flushedUpto を更新します。 3. **進捗監視**: latestChunkStart を通じて、スタートアッププロセスは受信の遅延状況を監視可能です。 4. **終了**: SIGTERM 等のシグナルにより walRcvState が STOPPING に遷移し、プロセスが終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walreceiver.c および src/backend/replication/walreceiverfuncs.c 内の各関数が、共有メモリ上の WalRcvData インスタンス（WalRcv）を参照・更新しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • WalRcvState: プロセスのライフサイクル状態（STREAMING, WAITING, STOPPED 等）。
  • XLogRecPtr: WALの論理位置情報。
  • ConditionVariable: プロセス間同期に使用。

失敗モードと境界条件

  • conninfo は接続後にセキュリティ保護のため上書き（clobber）されます。
  • writtenUpto は書き込み後に進められますが、データ整合性目的の読み取りには適さないため、厳密な整合性が必要な場合は mutex で保護された flushedUpto を使用する必要があります。

保守担当者向けメモ

  • 本構造体は共有メモリ上に存在するため、フィールドの追加や変更を行う際はアライメントとロックの粒度に注意してください。
  • writtenUpto はロックなしで読み取れる設計ですが、あくまで進捗の目安として利用してください。
