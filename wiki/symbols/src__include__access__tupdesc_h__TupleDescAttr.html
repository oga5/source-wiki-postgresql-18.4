TupleDescAttr

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/tupdesc.h:159

目的

タプル記述子（TupleDesc）が保持する属性定義配列（FormData_pg_attribute）から、指定されたインデックス i に対応する属性情報を取得します。

シグネチャまたは形状

static inline FormData_pg_attribute *TupleDescAttr(TupleDesc tupdesc, int i)

入力と出力

  • **入力**:
  • tupdesc: 対象のタプル記述子。
  • i: 取得したい属性のインデックス（0 始まり）。
  • **出力**:
  • 指定された属性に対応する FormData_pg_attribute 構造体へのポインタ。

前提条件と不変条件

  • tupdesc は有効なポインタである必要があります。
  • インデックス i は、tupdesc->natts（属性数）の範囲内である必要があります。
  • 内部で TupleDescAttrAddress を呼び出し、メモリ上の属性配列の先頭アドレスを算出します。

アルゴリズムまたは振る舞い

1. TupleDescAttrAddress(tupdesc) を呼び出し、タプル記述子内の属性配列のベースアドレスを取得します。 2. 取得したベースアドレスを起点として、インデックス i 番目の要素へのポインタを計算し、返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_nbtree.c
  • contrib/dblink/dblink.c
  • contrib/file_fdw/file_fdw.c
  • **呼び出し先**:
  • TupleDescAttrAddress（インライン関数）

関連データ構造

  • TupleDesc: タプル記述子のメイン構造体。
  • FormData_pg_attribute: システムカタログ pg_attribute の行データ構造。

失敗モードと境界条件

  • インデックス i が範囲外（負の値、または natts 以上）の場合、メモリ破壊や不正なメモリアクセスが発生する可能性があります。本関数自体には境界チェックは含まれていません。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、インライン化されています。変更を加える際は、パフォーマンスへの影響を考慮してください。
  • CompactAttribute を使用する TupleDescCompactAttr と混同しないよう注意してください。こちらはカタログ定義の完全な構造体（FormData_pg_attribute）を返します。
