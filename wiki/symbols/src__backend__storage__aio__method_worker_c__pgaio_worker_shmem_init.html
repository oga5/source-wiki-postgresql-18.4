pgaio_worker_shmem_init

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_worker.c:132

目的

非同期I/O（AIO）ワーカー方式で使用される共有メモリ領域（投入キューおよび制御構造体）の初期化またはアタッチを行う。

シグネチャまたは形状

static void pgaio_worker_shmem_init(bool first_time)

入力と出力

  • **入力**: first_time (bool) - 共有メモリの初期化が初回であるかを示すフラグ。
  • **出力**: なし（副作用として共有メモリ上のグローバル変数 io_worker_submission_queue および io_worker_control を初期化）。

前提条件と不変条件

  • ShmemInitStruct を介して共有メモリ領域が確保または取得可能であること。
  • 共有メモリのサイズ計算を行う pgaio_worker_queue_shmem_size および pgaio_worker_control_shmem_size が利用可能であること。

アルゴリズムまたは振る舞い

1. AioWorkerSubmissionQueue 用の共有メモリ領域を確保または取得する。

  • 初回作成時（!found）は、キューのサイズ、ヘッド、テールを初期化する。

    • 2. AioWorkerControl 用の共有メモリ領域を確保または取得する。

  • 初回作成時（!found）は、アイドルワーカーマスクを 0 にリセットし、各ワーカーのスロット（latch および in_use）を初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/method_worker.c 内の構造体初期化子（.shmem_init）から参照されている。
  • **呼び出し先**:
  • ShmemInitStruct
  • pgaio_worker_queue_shmem_size
  • pgaio_worker_control_shmem_size

関連データ構造

  • PgAioWorkerSubmissionQueue
  • PgAioWorkerControl
  • PgAioWorkerSlot

失敗モードと境界条件

  • ShmemInitStruct が失敗した場合、PostgreSQLの共有メモリ管理規則に従いプロセスが終了する可能性がある。
  • MAX_IO_WORKERS を超えるワーカーの初期化は行われない。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウトを定義する重要な初期化処理である。構造体のメンバー追加やサイズ変更を行う際は、対応する _shmem_size 関数も併せて更新する必要がある。
