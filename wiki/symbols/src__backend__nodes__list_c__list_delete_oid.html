list_delete_oid

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/list.c:909

目的

OID型のリストから、指定された値と一致する最初の要素を削除する。

シグネチャまたは形状

List *list_delete_oid(List *list, Oid datum)

入力と出力

  • **入力**:
  • list: 操作対象の List 構造体へのポインタ。
  • datum: 削除対象の Oid 値。
  • **出力**:
  • 削除後のリストを指すポインタ。要素が見つからなかった場合は元のリストがそのまま返される。

前提条件と不変条件

  • list は IsOidList マクロにより、OID型を保持するリストであることが保証されていなければならない。
  • 操作の前後で check_list_invariants(list) が呼び出され、リストの整合性が検証される。

アルゴリズムまたは振る舞い

1. リストの整合性を確認する。 2. foreach マクロを使用してリスト内の各セルを走査する。 3. lfirst_oid を用いて現在のセルの値を取得し、datum と比較する。 4. 一致する値が見つかった場合、list_delete_cell を呼び出してそのセルを削除し、結果を返す。 5. リストの末尾まで一致するものが見つからなかった場合、変更を加えずに元のリストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/index.c:4204
  • **呼び出し先**:
  • check_list_invariants (内部検証)
  • list_delete_cell (要素削除)

関連データ構造

  • List: リストのヘッダ構造体。
  • ListCell: リストの各要素を保持する構造体。

失敗モードと境界条件

  • リストが空の場合、または指定した datum がリスト内に存在しない場合は、何も削除されず元のリストが返される。
  • リストの伸長や削除に伴い内部配列が再配置される可能性があるため、呼び出し元は操作後の戻り値をリストの新しいポインタとして扱う必要がある。

保守担当者向けメモ

  • 本関数は配列ベースのリスト操作を行うため、要素削除時には後続の要素がシフトされる。頻繁な削除操作はパフォーマンスに影響を与える可能性がある。
