perform_rewind

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/pg_rewind.c:554

目的

同期計画（filemap）に基づき、ソースシステムからターゲットシステムへデータファイルおよび変更されたページを転送・適用し、ターゲットの制御ファイルを更新してリカバリ可能な状態にする。

シグネチャまたは形状

static void perform_rewind(filemap_t *filemap, rewind_source *source,
                           XLogRecPtr chkptrec, TimeLineID chkpttli,
                           XLogRecPtr chkptredo)

入力と出力

  • **入力**:
  • filemap: 同期対象のファイル操作リスト。
  • source: データ取得元（ローカルまたは libpq）。
  • chkptrec, chkpttli, chkptredo: 共通チェックポイントの WAL 位置情報。
  • **出力**: なし（副作用としてターゲットディレクトリのファイル更新および制御ファイルの書き換えを行う）。

前提条件と不変条件

  • ターゲットシステムはクリーンに停止していること。
  • filemap には実行すべきアクションが決定済みであること。
  • source は有効な接続状態であること。

アルゴリズムまたは振る舞い

1. **ファイル操作の実行**: filemap 内の全エントリを走査し、変更されたページブロックの転送（queue_fetch_range）およびファイル単位のアクション（コピー、切り詰め、削除、作成）をキューイングする。 2. **データ転送**: source->finish_fetch を呼び出し、キューイングされた範囲転送を完了させる。 3. **制御ファイルの同期**: ソースから最新の制御ファイルを取得し、整合性を検証する。 4. **リカバリ設定**: backup_label を生成し、ターゲットの制御ファイルを DB_IN_ARCHIVE_RECOVERY 状態に更新して、適切な minRecoveryPoint を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/pg_rewind.c 内の main 関数から呼び出される。
  • **呼び出し先**: source インターフェース（queue_fetch_range, queue_fetch_file, finish_fetch, fetch_file）、file_ops.c（truncate_target_file, remove_target, create_target, close_target_file, update_controlfile）、progress_report、createBackupLabel、digestControlFile。

関連データ構造

  • filemap_t: 同期計画を保持する構造体。
  • rewind_source: ソースシステムへのアクセス抽象化レイヤー。
  • ControlFileData: PostgreSQL の制御ファイル構造体。

失敗モードと境界条件

  • FILE_ACTION_UNDECIDED に遭遇した場合、pg_fatal で異常終了する。
  • ソースシステムが実行中に変更された場合（ローカルソース時）、pg_fatal で異常終了する。
  • ターゲットの制御ファイル更新時に dry_run が有効な場合は書き込みをスキップする。

保守担当者向けメモ

  • 本関数はターゲットのデータディレクトリを直接変更する破壊的な操作を含む。
  • ソースがスタンバイサーバーの場合、minRecoveryPoint の計算ロジックがプロダクションサーバーと異なるため注意が必要。
