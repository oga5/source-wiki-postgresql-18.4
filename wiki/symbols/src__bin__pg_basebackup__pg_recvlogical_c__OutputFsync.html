OutputFsync

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_recvlogical.c:186

目的

論理デコーディングストリームの受信データをディスクへ永続化（fsync）し、フラッシュ済み LSN を更新する。

シグネチャまたは形状

static bool OutputFsync(TimestampTz now)

入力と出力

  • **入力**: now (現在のタイムスタンプ)
  • **出力**: 常に true を返す。

前提条件と不変条件

  • output_written_lsn が最新の書き込み位置を保持していること。
  • outfd が有効なファイル記述子であること。
  • fsync_interval が正の値である場合にのみ、物理的な fsync が実行される。

アルゴリズムまたは振る舞い

1. output_last_fsync を引数 now で更新する。 2. output_fsync_lsn を現在の output_written_lsn に同期させる。 3. fsync_interval <= 0 の場合、または output_needs_fsync が偽の場合は、物理的な fsync をスキップして true を返す。 4. output_needs_fsync を false にリセットする。 5. 出力先が通常ファイル (output_isfile が真) であることを確認する。 6. fsync(outfd) を実行し、失敗した場合は pg_fatal でエラー終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/pg_recvlogical.c 内の複数箇所（メインループおよび終了処理周辺）。
  • **呼び出し先**: fsync() (システムコール)、pg_fatal() (エラーハンドリング)。

関連データ構造

  • output_last_fsync: 最後に fsync を実行した時刻を保持。
  • output_fsync_lsn: 最後に fsync が完了した LSN。
  • output_written_lsn: 現在書き込み済みの LSN。
  • output_needs_fsync: fsync が必要かどうかを示すフラグ。
  • outfd: 書き込み対象のファイル記述子。
  • output_isfile: 出力先が通常ファイルかどうかを示すフラグ。

失敗モードと境界条件

  • fsync(outfd) が失敗した場合、pg_fatal によりプロセスが異常終了する。
  • output_isfile が偽（標準出力など）の場合、fsync は実行されず、成功として扱われる。
  • fsync_interval が 0 以下の場合は、常に fsync を実行せずに成功を返す。

保守担当者向けメモ

  • 本関数は pg_recvlogical におけるデータ永続化の要です。fsync の頻度は fsync_interval オプションに依存します。
  • output_needs_fsync フラグの管理が、不要な fsync 呼び出しを抑制する設計となっています。
