grow_rel

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_aio/test_aio.c:158

目的

指定されたリレーション（テーブル）を、指定したブロック数分だけ強制的に拡張するためのテスト用関数です。

シグネチャまたは形状

Datum grow_rel(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • relid (Oid): 拡張対象のリレーションのOID。
  • nblocks (uint32): 拡張するブロック数。
  • **出力**:
  • なし (PG_RETURN_VOID)。

前提条件と不変条件

  • AccessExclusiveLock を取得してリレーションをオープンするため、対象リレーションに対する排他アクセス権が必要です。
  • 拡張処理は MAX_BUFFERS_TO_EXTEND_BY（64ブロック）単位でループ処理されます。

アルゴリズムまたは振る舞い

1. 指定された relid を AccessExclusiveLock モードでオープンします。 2. nblocks が 0 になるまで以下の処理を繰り返します。

  • 残りのブロック数と MAX_BUFFERS_TO_EXTEND_BY の小さい方を extend_by_pages とします。
  • ExtendBufferedRelBy を呼び出し、リレーションを拡張します。
  • 確保されたバッファを victim_buffers 配列に格納します。
  • 拡張したブロック数分だけ nblocks を減算します。
  • 確保した各バッファに対して ReleaseBuffer を呼び出し、ピンを解放します。

    • 3. リレーションをクローズし、処理を終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由での呼び出しを想定（PG_FUNCTION_INFO_V1 による登録あり）。
  • **呼び出し先**:
  • relation_open / relation_close
  • ExtendBufferedRelBy
  • ReleaseBuffer

関連データ構造

  • Relation: PostgreSQL のリレーション記述子。
  • Buffer: バッファ管理用の識別子。

失敗モードと境界条件

  • nblocks が 0 の場合、ループは実行されず即座に終了します。
  • ExtendBufferedRelBy が失敗した場合、PostgreSQL のエラーハンドリング機構によりトランザクションがアボートする可能性があります。

保守担当者向けメモ

  • 本関数はテスト専用のヘルパーであり、共有バッファの状態を直接操作します。本番環境のコードには流用しないでください。
  • MAX_BUFFERS_TO_EXTEND_BY は 64 に固定されています。
