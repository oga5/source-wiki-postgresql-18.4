GetIncrementalFilePath

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_incremental.c:624

目的

増分バックアップにおいて、特定のデータベースファイルに対応する増分データファイルのパスを生成します。

シグネチャまたは形状

char *GetIncrementalFilePath(Oid dboid, Oid spcoid, RelFileNumber relfilenumber, ForkNumber forknum, unsigned segno)

入力と出力

  • **入力**:
  • dboid, spcoid, relfilenumber, forknum: 対象リレーションを特定するための識別子。
  • segno: セグメント番号（0以上の整数）。
  • **出力**:
  • 生成されたファイルパス文字列（palloc されたメモリ領域）。

前提条件と不変条件

  • GetRelationPath が有効なパスを返すことを前提としています。
  • パス文字列内に少なくとも一つのディレクトリ区切り文字（/）が含まれている必要があります（Assert により検証）。

アルゴリズムまたは振る舞い

1. GetRelationPath を使用して、対象リレーションの標準的なパスを取得します。 2. 取得したパスの最後のディレクトリ区切り文字（/）を検索し、パスをディレクトリ部分とファイル名部分に分割します。 3. segno が 0 より大きい場合、ディレクトリ/INCREMENTAL.ファイル名.セグメント番号 という形式のパスを生成します。 4. segno が 0 の場合、ディレクトリ/INCREMENTAL.ファイル名 という形式のパスを生成します。 5. 生成した文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup_incremental.c 内の増分バックアップ処理ロジックから呼び出されます。
  • **呼び出し先**: GetRelationPath（PostgreSQLの標準的なパス生成関数）。

関連データ構造

  • RelFileNumber, Oid, ForkNumber: リレーションを特定するための基本型。

失敗モードと境界条件

  • GetRelationPath が返すパスに / が含まれない場合、Assert により異常終了します。
  • メモリ不足時には psprintf 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は、増分バックアップ用のファイル命名規則を集中管理しています。命名規則を変更する場合は、本関数とそれを読み込む側のロジックの両方を考慮してください。
