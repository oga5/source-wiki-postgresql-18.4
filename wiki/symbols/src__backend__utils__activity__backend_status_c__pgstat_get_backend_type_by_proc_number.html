pgstat_get_backend_type_by_proc_number

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/backend_status.c:1180

目的

指定されたプロセス番号（ProcNumber）に対応するバックエンドプロセスの種別（BackendType）を取得します。

シグネチャまたは形状

BackendType pgstat_get_backend_type_by_proc_number(ProcNumber procNumber)

入力と出力

  • **入力**: procNumber (ProcNumber型) - 対象となるバックエンドのプロセス番号。
  • **出力**: BackendType型 - 該当バックエンドの種別を返します。

前提条件と不変条件

  • BackendStatusArray が初期化済みである必要があります。
  • procNumber は BackendStatusArray の有効なインデックス範囲内である必要があります。

アルゴリズムまたは振る舞い

1. BackendStatusArray 内の指定された procNumber に対応する PgBackendStatus 構造体へのポインタを取得します。 2. st_backendType フィールドの値を読み取ります。 3. 本関数は、st_changecount による楽観的並行制御（非ブロッキングプロトコル）をバイパスします。これは、整数型の読み取りおよび書き込みがアトミックであるという前提に基づいています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/signalfuncs.c:91
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません（共有メモリ上の配列への直接アクセスのみ）。

関連データ構造

  • BackendStatusArray: 全バックエンドのステータスを保持する共有メモリ上の配列。
  • PgBackendStatus: 各バックエンドの状態を保持する構造体。
  • BackendType: バックエンドの種別を定義する列挙型。

失敗モードと境界条件

  • procNumber が配列の範囲外である場合、メモリ破壊や不正なアクセスが発生する可能性があります（境界チェックは本関数内では行われていません）。

保守担当者向けメモ

  • 本関数は st_changecount を使用しないため、非常に軽量ですが、呼び出し側は procNumber が有効であることを保証する必要があります。
  • 共有メモリ上のデータに直接アクセスするため、将来的に BackendType のサイズや構造が変更され、アトミック性が保証されなくなった場合には、st_changecount を使用する方式への修正が必要です。
