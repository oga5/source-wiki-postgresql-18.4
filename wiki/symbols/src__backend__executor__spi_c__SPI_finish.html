SPI_finish

定義

  • 種別: function
  • 定義箇所: src/backend/executor/spi.c:181

目的

SPI（Server Programming Interface）セッションを終了し、関連するリソースを解放して実行コンテキストを復元する。

シグネチャまたは形状

int SPI_finish(void)

入力と出力

  • **入力**: なし。
  • **出力**: 成功時は SPI_OK_FINISH を返す。接続されていない場合は _SPI_begin_call が返すエラーコードを返す。

前提条件と不変条件

  • SPI_connect によって開始された SPI セッション内から呼び出される必要がある。
  • 呼び出しにより、現在の SPI スタックレベルがポップされる。

アルゴリズムまたは振る舞い

1. _SPI_begin_call(false) を呼び出し、SPI セッションに接続中であることを確認する。 2. MemoryContextSwitchTo を使用し、SPI 呼び出し前のメモリコンテキストへ切り替える。 3. MemoryContextDelete を呼び出し、現在の SPI セッションで使用していた execCxt および procCxt を解放する。 4. グローバル変数 SPI_processed、SPI_tuptable、SPI_result を、スタックに保存されていた外側の呼び出し元の値に復元する。 5. 内部スタックポインタ _SPI_connected をデクリメントし、_SPI_current を一つ前のスタックレベルへ更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • contrib/spi/refint.c
  • contrib/tablefunc/tablefunc.c
  • (注: 上記はツールによる発見であり、網羅的ではない可能性がある)
  • **呼び出し先**:
  • _SPI_begin_call
  • MemoryContextSwitchTo
  • MemoryContextDelete

関連データ構造

  • _SPI_stack: SPI 呼び出しのネストを管理する内部スタック。
  • _SPI_current: 現在の SPI コンテキストを指す内部ポインタ。

失敗モードと境界条件

  • 接続されていない状態で呼び出した場合、_SPI_begin_call の戻り値（負の値）がそのまま返される。
  • _SPI_connected が 0 未満になった場合、_SPI_current は NULL に設定される。

保守担当者向けメモ

  • 本関数は SPI_connect と対になる。メモリリークを防ぐため、SPI を使用する処理の終了時には必ず呼び出す必要がある。
  • SPI_tuptable 等のグローバル変数は、本関数によってスタックから復元されるため、ネストした SPI 呼び出しにおいても整合性が保たれる。
