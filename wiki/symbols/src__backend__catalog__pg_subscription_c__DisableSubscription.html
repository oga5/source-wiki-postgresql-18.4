DisableSubscription

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_subscription.c:199

目的

指定されたサブスクリプション（subid）を無効化し、システムカタログ pg_subscription の状態を更新します。

シグネチャまたは形状

void DisableSubscription(Oid subid)

入力と出力

  • **入力**: subid (Oid) - 無効化対象のサブスクリプションのOID。
  • **出力**: なし。カタログ更新の結果として pg_subscription が変更されます。

前提条件と不変条件

  • subid に対応するサブスクリプションが pg_subscription カタログに存在すること。
  • 呼び出し元は適切な権限を持っている必要があります（カタログ更新操作のため）。

アルゴリズムまたは振る舞い

1. pg_subscription リレーションを RowExclusiveLock で開きます。 2. SUBSCRIPTIONOID システムキャッシュを使用して、対象サブスクリプションのタプルをコピー取得します。 3. キャッシュ内にタプルが見つからない場合はエラーを発生させます。 4. LockSharedObject を使用して、対象サブスクリプションに対する AccessShareLock を取得し、競合を防止します。 5. subenabled 属性を false に設定するためのタプル変更用データを準備します。 6. heap_modify_tuple を用いてタプルを更新し、CatalogTupleUpdate でカタログへ書き込みます。 7. 取得したタプルを解放し、リレーションを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/worker.c:4865
  • **呼び出し先**:
  • table_open, SearchSysCacheCopy1, LockSharedObject, heap_modify_tuple, CatalogTupleUpdate, heap_freetuple, table_close（すべてPostgreSQL内部のカタログ操作関数）。

関連データ構造

  • pg_subscription: サブスクリプション情報を保持するシステムカタログ。
  • SubscriptionRelationId: pg_subscription カタログのOID。

失敗モードと境界条件

  • 指定された subid が存在しない場合、elog(ERROR) により処理が中断されます。
  • カタログロックの競合が発生した場合、PostgreSQLの標準的なロック待機動作に従います。

保守担当者向けメモ

  • 本関数は SysCache を介してタプルを取得・更新します。カタログの直接的な変更を行うため、排他制御（LockSharedObject）が適切に行われていることを確認してください。
  • pg_subscription のスキーマ変更（属性の追加等）が発生した場合は、values や replaces 配列の初期化処理に影響が出る可能性があります。
