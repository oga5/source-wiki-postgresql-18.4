create_tidscan_paths

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/tidpath.c:497

目的

リレーションに対して、CTID を用いたスキャンパス（TidPath および TidRangePath）を生成し、プランナのパスリストに追加する。

シグネチャまたは形状

bool create_tidscan_paths(PlannerInfo *root, RelOptInfo *rel)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel: 対象となるリレーション情報。
  • **出力**:
  • bool: true の場合、CURRENT OF 句が検出され、TIDスキャンが強制されることを示す。それ以外は false。

前提条件と不変条件

  • enable_tidscan が false の場合、CURRENT OF 句によるスキャンを除き、TIDスキャンパスの生成はスキップされる。
  • TidRangePath の生成は、対象のテーブルアクセスメソッドが範囲スキャンをサポートしている場合にのみ行われる。

アルゴリズムまたは振る舞い

1. **TID等価条件の処理**: TidQualFromRestrictInfoList を呼び出し、baserestrictinfo から ctid 等価条件を抽出する。条件が存在すれば TidPath を生成して追加する。

  • isCurrentOf が true の場合、他のパス生成をスキップして即座に true を返す。

    • 2. **TID範囲条件の処理**: enable_tidscan が有効な場合、TidRangeQualFromRestrictInfoList を使用して範囲条件を抽出する。条件が存在すれば TidRangePath を生成して追加する。 3. **パラメータ化パスの生成**:

  • EquivalenceClass から ctid 関連の等価条件を生成し、BuildParameterizedTidPaths を呼び出す。
  • rel->joininfo に含まれる結合条件に対しても BuildParameterizedTidPaths を呼び出し、パラメータ化されたTIDパスを検討する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/allpaths.c（プランナのパス生成プロセスから呼び出される）。
  • **呼び出し先**:
  • TidQualFromRestrictInfoList, TidRangeQualFromRestrictInfoList
  • create_tidscan_path, create_tidrangescan_path
  • add_path
  • generate_implied_equalities_for_column
  • BuildParameterizedTidPaths

関連データ構造

  • PlannerInfo: プランナのコンテキスト。
  • RelOptInfo: スキャン対象のリレーション情報。
  • RestrictInfo: フィルタ条件のリスト。
  • Path: 生成されるアクセスパスの基底構造。

失敗モードと境界条件

  • enable_tidscan が無効な場合、CURRENT OF 句以外のパス生成は行われない。
  • 適切な ctid 関連条件が見つからない場合、パスは追加されない。

保守担当者向けメモ

  • CURRENT OF 句は特殊なケースであり、この関数で検出された場合は他のパス生成を抑制する優先的なロジックが含まれている。
  • パラメータ化パスの生成において、LATERAL 参照による依存関係が考慮されている。
