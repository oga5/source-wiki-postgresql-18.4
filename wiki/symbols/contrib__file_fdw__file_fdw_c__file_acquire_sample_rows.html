file_acquire_sample_rows

定義

  • 種別: function
  • 定義箇所: contrib/file_fdw/file_fdw.c:1187

目的

外部テーブルからランダムな行サンプルを取得し、統計情報収集（ANALYZE）をサポートする。

シグネチャまたは形状

int file_acquire_sample_rows(Relation onerel, int elevel, HeapTuple *rows, int targrows, double *totalrows, double *totaldeadrows);

入力と出力

  • **入力**:
  • onerel: 対象の外部テーブルリレーション。
  • elevel: ログ出力レベル。
  • targrows: 取得するサンプル行数。
  • **出力**:
  • rows: 取得したサンプルの HeapTuple 配列。
  • totalrows: 読み込んだ総行数（出力パラメータ）。
  • totaldeadrows: 常に 0 が設定される（出力パラメータ）。
  • **戻り値**: 実際に取得されたサンプル行数。

前提条件と不変条件

  • onerel は有効であること。
  • targrows > 0 であること。
  • 外部テーブルのオプション（ファイルパス等）が取得可能であること。

アルゴリズムまたは振る舞い

1. BeginCopyFrom を使用して外部ソースからのストリームを開く。 2. reservoir_init_selection_state でリザーバーサンプリングの準備を行う。 3. NextCopyFrom をループで呼び出し、各行を読み込む。

  • on_error = 'ignore' が設定されている場合、データ型不一致によるエラーをスキップする。
  • Jeff Vitter のアルゴリズムに基づき、最初の targrows 行を無条件に保存し、以降は確率的に既存のサンプルと置き換える。

    • 4. vacuum_delay_point を呼び出し、ユーザーによる中断要求を監視する。 5. 終了時に EndCopyFrom でリソースを解放し、処理した行数とサンプル数をログ出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: file_fdw_handler 内で FdwRoutine の AcquireSampleRows フィールドに登録され、PostgreSQL の ANALYZE 処理から呼び出される。
  • **呼び出し先**: fileGetOptions, BeginCopyFrom, NextCopyFrom, EndCopyFrom, reservoir_init_selection_state, reservoir_get_next_S, heap_form_tuple, vacuum_delay_point 等。

関連データ構造

  • CopyFromState: COPY コマンドの実行状態を保持。
  • ReservoirStateData: リザーバーサンプリングの統計状態を保持。
  • TupleDesc, HeapTuple: タプルの定義と実体。

失敗モードと境界条件

  • 外部ファイルが存在しない、または読み取り権限がない場合は BeginCopyFrom がエラーを発生させる。
  • targrows が 0 以下の場合、Assert により停止する。
  • 読み込み中にエラーが発生した場合、CopyFromErrorCallback がエラーコンテキストを補足する。

保守担当者向けメモ

  • 本関数は COPY コマンドのインフラストラクチャに強く依存している。COPY の仕様変更や NextCopyFrom の挙動変更が直接影響するため、修正時は src/backend/commands/copyfrom.c を参照すること。
  • サンプリングアルゴリズムの詳細は commands/analyze.c にある Vitter の論文に関する記述を参照すること。
