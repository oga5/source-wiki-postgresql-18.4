BootStrapCLOG

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/clog.c:832

目的

データベースの初期化（initdb）時において、コミットログ（CLOG）の最初のページを作成および初期化する。

シグネチャまたは形状

void BootStrapCLOG(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として CLOG の最初のページがディスクに書き込まれる）

前提条件と不変条件

  • XactCtl が初期化済みであること。
  • 排他ロック（LW_EXCLUSIVE）を保持した状態で実行されること。
  • 実行後、CLOG の最初のページ（ページ番号 0）がディスク上に存在し、ダーティ状態ではないことが保証される。

アルゴリズムまたは振る舞い

1. XactCtl のバンク 0 に対する排他ロックを取得する。 2. ZeroCLOGPage(0, false) を呼び出し、CLOG の最初のページをゼロクリアして初期化する。 3. SimpleLruWritePage を呼び出し、初期化したページをディスクへ書き出す。 4. Assert により、ページがダーティ状態でないことを確認する。 5. ロックを解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlog.c:5222
  • 呼び出し先:
  • SimpleLruGetBankLock
  • LWLockAcquire
  • ZeroCLOGPage
  • SimpleLruWritePage
  • LWLockRelease

関連データ構造

  • XactCtl: CLOG の共有メモリ状態を管理する SlruCtlData 構造体。

失敗モードと境界条件

  • 排他ロックの取得に失敗した場合、またはディスク I/O エラーが発生した場合は、基盤となる SLRU 関数によりエラーがスローされる。

保守担当者向けメモ

  • 本関数はデータベースのブートストラップ時にのみ使用される。
  • ZeroCLOGPage の第 2 引数に false を渡しており、WAL ログの生成を伴わない初期化を行っている点に注意すること。
