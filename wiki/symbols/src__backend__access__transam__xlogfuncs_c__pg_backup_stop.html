pg_backup_stop

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogfuncs.c:123

目的

オンラインバックアップを終了し、バックアップ完了に必要なメタデータ（LSN、ラベルファイル、テーブルスペースマップ）を生成して呼び出し元へ返却する。

シグネチャまたは形状

Datum pg_backup_stop(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • waitforarchive (bool): WALのアーカイブ完了を待機するかどうかを指定する引数。
  • **出力**:
  • Datum: 以下の3列からなるレコードを返す。

    • 1. lsn (LSN): バックアップ終了時点のログシーケンス番号。 2. backup_label (text): バックアップラベルの内容。 3. tablespace_map (text): テーブルスペースマップの内容。

前提条件と不変条件

  • セッション内でオンラインバックアップが実行中であること（SESSION_BACKUP_RUNNING 状態であること）。
  • backup_state および tablespace_map が非NULLであること。
  • 呼び出し前に pg_backup_start() が実行されている必要がある。

アルゴリズムまたは振る舞い

1. 戻り値の型（レコード型）を検証する。 2. 現在のバックアップ状態を確認し、実行中でなければエラーを投げる。 3. do_pg_backup_stop を呼び出し、バックアップ終了処理を実行する。 4. build_backup_content を使用してバックアップラベルを生成する。 5. 取得したLSN、ラベル、テーブルスペースマップをレコードの各列に格納する。 6. backup_label を解放し、セッションレベルのバックアップ状態変数（backup_state, tablespace_map）をリセットする。 7. バックアップ専用のメモリコンテキスト backupcontext を削除する。 8. 構築したレコードを Datum として返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由でユーザーから呼び出される。
  • **呼び出し先**:
  • get_backup_status()
  • get_call_result_type()
  • do_pg_backup_stop()
  • build_backup_content()
  • MemoryContextDelete()
  • heap_form_tuple()

関連データ構造

  • backup_state: セッション内で実行中のバックアップ状態を保持する静的変数。
  • tablespace_map: テーブルスペースの配置情報を保持する静的変数。
  • backupcontext: バックアップ関連情報を保持する専用メモリコンテキスト。

失敗モードと境界条件

  • バックアップが開始されていない状態で呼び出した場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE エラーが発生する。
  • 戻り値の型が期待するレコード型と一致しない場合、エラーが発生する。

保守担当者向けメモ

  • 本関数はセッションレベルのバックアップ状態を管理しており、終了時に backupcontext を削除することでメモリリークを防いでいる。
  • backup_state や tablespace_map はセッションを跨がないため、エラー発生時や終了時のクリーンアップ処理が重要である。
