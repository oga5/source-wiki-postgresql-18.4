tfunc_title

定義

  • 種別: function
  • 定義箇所: src/common/unicode/case_test.c:300

目的

Unicode 文字列のタイトルケース変換（Title Case）を検証するためのラッパー関数です。unicode_strtitle を呼び出す際に必要な単語境界判定用の状態管理とイテレータを提供します。

シグネチャまたは形状

static size_t tfunc_title(char *dst, size_t dstsize, const char *src, ssize_t srclen)

入力と出力

  • **入力**:
  • dst: 変換結果を格納するバッファ。
  • dstsize: dst バッファのサイズ。
  • src: 変換対象のソース文字列。
  • srclen: ソース文字列の長さ（負の場合は strlen(src) が使用される）。
  • **出力**:
  • 変換後の文字列長（size_t）。

前提条件と不変条件

  • WordBoundaryState 構造体は、src 文字列の先頭から走査を開始するように初期化されます。
  • init フラグは false に設定され、単語境界判定の初期状態が保証されます。

アルゴリズムまたは振る舞い

1. WordBoundaryState 構造体を初期化し、ソース文字列の長さやオフセットをセットアップします。 2. unicode_strtitle を呼び出し、以下の引数を渡します。

  • 変換先バッファとサイズ。
  • ソース文字列と長さ。
  • true（タイトルケース変換の有効化）。
  • initcap_wbnext（単語境界判定用イテレータ関数）。
  • wbstate（状態保持構造体へのポインタ）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/unicode/case_test.c 内の test_convert 関数から、タイトルケース変換のテストケースとして複数回呼び出されています。
  • **呼び出し先**: unicode_strtitle および initcap_wbnext。

関連データ構造

  • WordBoundaryState: タイトルケース変換時の単語境界を追跡するための状態保持構造体。

失敗モードと境界条件

  • srclen が負の値の場合、strlen(src) に依存するため、src が NULL 終端されていない場合は未定義動作となります。
  • dstsize が変換結果を格納するのに不十分な場合、unicode_strtitle の仕様に従った切り詰めやエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はテスト専用のラッパーです。実運用コードでのタイトルケース変換には、直接 unicode_strtitle を使用してください。
  • initcap_wbnext のロジック変更は、本関数のテスト結果に直接影響を与えます。
