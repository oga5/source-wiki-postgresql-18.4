dsynonym_init

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/dict_synonym.c:93

目的

全文検索用シノニム辞書の設定ファイルを読み込み、メモリ上に同義語ペアの配列を構築およびソートして初期化する。

シグネチャまたは形状

Datum dsynonym_init(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して List *dictoptions を受け取る。
  • synonyms: 辞書ファイル名（必須）。
  • casesensitive: 大文字小文字を区別するかどうかの真偽値（デフォルトは false）。
  • **出力**: DictSyn 構造体へのポインタを Datum 型として返す。

前提条件と不変条件

  • synonyms パラメータが指定されていない場合、エラーが発生する。
  • 辞書ファイルは get_tsearch_config_filename により解決される。
  • 構築された DictSyn 内の syn 配列は、compareSyn 関数を用いてソート済みであること。

アルゴリズムまたは振る舞い

1. dictoptions を走査し、設定値を取得する。 2. tsearch_readline_begin で辞書ファイルを開く。 3. tsearch_readline で行ごとに読み込み、findwrd を用いて各行から入力語と出力語を抽出する。 4. 抽出した単語を DictSyn 構造体の syn 配列に格納する。必要に応じて palloc / repalloc でメモリを拡張する。 5. case_sensitive フラグが false の場合、str_tolower を用いて単語を小文字化する。 6. 全行の読み込み終了後、qsort を使用して syn 配列をソートする。 7. 最終的な辞書状態を保持する DictSyn を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの辞書管理システム（詳細は提供されたソース範囲外）。
  • **呼び出し先**: findwrd, tsearch_readline_begin, tsearch_readline, tsearch_readline_end, str_tolower, qsort, compareSyn。

関連データ構造

  • DictSyn: 辞書全体の状態を保持する構造体。
  • Syn: 個別の同義語ペア（入力語、出力語、フラグ等）を保持する構造体。

失敗モードと境界条件

  • 未知のパラメータが指定された場合、エラーを報告する。
  • ファイルが開けない場合、エラーを報告する。
  • 1単語のみの行や空行は無視される。
  • メモリ不足時は palloc / repalloc がエラーを発生させる。

保守担当者向けメモ

  • 辞書データは palloc で確保されるため、大規模な辞書を読み込む際はメモリ消費量に留意すること。
  • findwrd は単語末尾の * をプレフィックスフラグとして解釈するロジックを含むため、辞書ファイルのフォーマット変更時には注意が必要。
