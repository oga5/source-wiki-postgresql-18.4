remove_tablespace_symlink

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tablespace.c:882

目的

指定されたパスにあるテーブルスペースのシンボリックリンクまたはディレクトリを削除します。

シグネチャまたは形状

void remove_tablespace_symlink(const char *linkloc)

入力と出力

  • **入力**: linkloc (削除対象のパスを示す文字列)
  • **出力**: なし（成功時は正常終了、失敗時はエラーを報告）

前提条件と不変条件

  • 対象パスがディレクトリまたはシンボリックリンクである必要があります。それ以外の場合はエラーとなります。
  • linkloc が存在しない場合、関数は何もせず正常に終了します。

アルゴリズムまたは振る舞い

1. lstat を使用して対象パスの情報を取得します。存在しない場合は即座に復帰します。 2. 対象がディレクトリ (S_ISDIR) の場合、rmdir を試みます。 3. 対象がシンボリックリンク (S_ISLNK) の場合、unlink を試みます。 4. 上記いずれでもない場合、エラーを報告します。 5. 各削除操作において、ENOENT（既に存在しない）以外のエラーが発生した場合は ereport(ERROR, ...) を発行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogrecovery.c:692
  • src/backend/commands/tablespace.c:656
  • **呼び出し先**:
  • lstat, rmdir, unlink (標準ライブラリ)
  • ereport (PostgreSQL エラー報告マクロ)

関連データ構造

  • struct stat (ファイル属性取得用)

失敗モードと境界条件

  • **ENOENT**: 対象が既に存在しない場合は無視されます。
  • **ディレクトリが空でない**: rmdir が失敗し、エラーが報告されます。
  • **権限不足または不正なパス**: lstat, rmdir, unlink の各システムコールが失敗した場合、errcode_for_file_access() を用いてエラーが報告されます。
  • **対象がディレクトリ/リンク以外**: ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE エラーが発生します。

保守担当者向けメモ

  • Windows 環境ではシンボリックリンクの代わりにジャンクションポイントが使用される場合がありますが、本関数は rmdir を使用してディレクトリとして削除を試みるロジックを含んでいます。
