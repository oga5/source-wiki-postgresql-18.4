ParallelSlotsSetup

定義

  • 種別: function
  • 定義箇所: src/fe_utils/parallel_slot.c:427

目的

データベース接続を並列管理するための ParallelSlotArray 構造体を初期化し、メモリを確保します。

シグネチャまたは形状

ParallelSlotArray *ParallelSlotsSetup(int numslots, ConnParams *cparams, const char *progname, bool echo, const char *initcmd);

入力と出力

  • **入力**:
  • numslots: 作成するスロット数（正の整数）。
  • cparams: 接続パラメータへのポインタ。
  • progname: プログラム名。
  • echo: SQLコマンドのエコー出力フラグ。
  • initcmd: 各接続確立後に実行する初期化コマンド（NULL可）。
  • **出力**:
  • 初期化済みの ParallelSlotArray 構造体へのポインタ。

前提条件と不変条件

  • numslots > 0 であること。
  • cparams および progname は NULL であってはならない。
  • 内部で palloc0 を使用するため、メモリ不足時はエラー終了します。

アルゴリズムまたは振る舞い

1. ParallelSlotArray 構造体と、それに続く numslots 分の ParallelSlot 配列を保持するために必要なメモリを palloc0 で確保します。 2. 引数で受け取った設定値（接続パラメータ、プログラム名、エコー設定、初期化コマンド）を構造体の各フィールドにコピーします。 3. 初期化された構造体ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/scripts/reindexdb.c
  • src/bin/scripts/vacuumdb.c
  • **呼び出し先**:
  • palloc0 (PostgreSQLメモリ管理関数)

関連データ構造

  • ParallelSlotArray: 並列接続管理のメイン構造体。
  • ParallelSlot: 個別の接続状態を保持する構造体。
  • ConnParams: データベース接続に必要なパラメータ群。

失敗モードと境界条件

  • numslots が 0 以下の場合、Assert により異常終了します。
  • cparams または progname が NULL の場合、Assert により異常終了します。

保守担当者向けメモ

  • 本関数はスロット配列のメモリ確保と設定値の格納のみを行います。実際のデータベース接続確立は、後続の ParallelSlotsGetIdle 等の呼び出し時に行われます。
  • 本モジュールはシングルスレッド環境での使用を前提としています。
