tarClose

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_tar.c:397

目的

TAR形式アーカイブにおけるファイルメンバーのクローズ処理およびリソース解放を行う。

シグネチャまたは形状

static void tarClose(ArchiveHandle *AH, TAR_MEMBER *th)

入力と出力

  • AH: アーカイブ全体のハンドル。圧縮アルゴリズムの検証に使用される。
  • th: クローズ対象の TAR_MEMBER 構造体。
  • 出力: なし（副作用としてリソースを解放する）。

前提条件と不変条件

  • AH->compression_spec.algorithm が PG_COMPRESSION_NONE であること（TAR形式は圧縮をサポートしないため、それ以外の場合は致命的エラーとなる）。

アルゴリズムまたは振る舞い

1. 圧縮アルゴリズムが設定されているか確認し、有効な場合は pg_fatal を呼び出して終了する。 2. th->mode が書き込みモード（'w'）である場合、_tarAddFile を呼び出して一時ファイルをメインアーカイブへ統合する（この過程で一時ファイルはクローズされる）。 3. th->targetFile を free する。 4. th->nFH を NULL に設定し、ハンドルを無効化する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/pg_backup_tar.c 内の複数箇所（tarOpen 後のクローズ処理や、アーカイブ操作の終了時など）。
  • 呼び出し先: _tarAddFile（書き込みモード時）、free（標準ライブラリ）、pg_fatal（エラー時）。

関連データ構造

  • ArchiveHandle: アーカイブ全体のコンテキスト。
  • TAR_MEMBER: TARアーカイブ内の個々のメンバーを管理する構造体。

失敗モードと境界条件

  • 圧縮が有効な状態で呼び出された場合、pg_fatal によりプロセスが終了する。

保守担当者向けメモ

  • 本関数は読み込みモード時にはファイルハンドルのクローズを行わない（dup() を使用していないため）。書き込みモード時のみ _tarAddFile を通じて一時ファイルのクローズとアーカイブへの統合を行う点に注意すること。
