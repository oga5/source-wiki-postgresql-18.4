getDumpableObjects

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/common.c:796

目的

現在メモリ上に保持されているすべてのダンプ対象オブジェクト（DumpableObject）のリストを生成し、呼び出し元に返します。

シグネチャまたは形状

void getDumpableObjects(DumpableObject ***objs, int *numObjs)

入力と出力

  • **入力**:
  • objs: DumpableObject ポインタの配列へのポインタ（出力先）。
  • numObjs: 取得されたオブジェクトの総数を格納する整数へのポインタ。
  • **出力**:
  • *objs: pg_malloc_array で確保された、有効な DumpableObject へのポインタ配列。
  • *numObjs: 配列内の有効な要素数。

前提条件と不変条件

  • グローバル変数 allocedDumpIds が初期化されており、有効な範囲を示していること。
  • グローバル配列 dumpIdMap が存在し、ダンプ対象オブジェクトが格納されていること。

アルゴリズムまたは振る舞い

1. allocedDumpIds のサイズに基づいて、objs 用のメモリ領域を pg_malloc_array で確保します。 2. dumpIdMap をインデックス 1 から allocedDumpIds - 1 まで走査します。 3. dumpIdMap[i] が非 NULL（有効なオブジェクト）である場合、そのポインタを *objs 配列にコピーし、カウンタ j をインクリメントします。 4. 最終的なオブジェクト数 j を *numObjs に書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c:1150
  • src/bin/pg_dump/pg_dump.c:3194
  • **呼び出し先**:
  • pg_malloc_array（メモリ確保用）

関連データ構造

  • DumpableObject: ダンプ対象となるデータベースオブジェクトの抽象構造体。
  • dumpIdMap: DumpId をキーとして DumpableObject を保持するグローバル配列。

失敗モードと境界条件

  • allocedDumpIds が 0 または未初期化の場合、メモリ確保やループ処理で予期せぬ動作をする可能性があります。
  • pg_malloc_array がメモリ不足で失敗した場合、プログラムは終了します。

保守担当者向けメモ

  • 本関数は dumpIdMap のスパースな構造を詰め直してリスト化する役割を持ちます。
  • dumpIdMap のインデックス 0 は使用されない前提（i = 1 から開始）で実装されています。
