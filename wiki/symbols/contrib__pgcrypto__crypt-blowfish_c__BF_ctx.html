BF_ctx

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/crypt-blowfish.c:60

目的

Blowfish 暗号アルゴリズムの実行に必要な内部状態（S-box および P-box）を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	BF_word		S[4][0x100];
	BF_key		P;
} BF_ctx;

入力と出力

本構造体はデータ保持用であり、関数ではありません。_BF_body_r などの関数に対して、暗号化状態を保持するコンテキストとしてポインタ経由で渡されます。

前提条件と不変条件

  • bcrypt アルゴリズムの実行において、初期化された状態を保持する必要があります。
  • 処理終了後は、機密情報保護のため px_memset 等を用いてメモリ上の内容を消去することが推奨されます。

アルゴリズムまたは振る舞い

Blowfish 暗号の鍵拡張および暗号化ラウンド処理において、S-box（置換ボックス）および P-box（置換配列）の値を保持・更新します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/crypt-blowfish.c 内の各関数（_crypt_blowfish_rn 等）でローカル変数として宣言または初期化されます。
  • 呼び出し先: _BF_body_r(BF_ctx *ctx) 等の Blowfish 処理関数へ引数として渡されます。

関連データ構造

  • BF_word: S-box の要素型。
  • BF_key: P-box の型。

失敗モードと境界条件

  • スタック上に確保されるため、メモリ不足による直接的な失敗はありませんが、機密情報がスタックに残存するリスクがあります。
  • 構造体内の各要素は bcrypt の仕様に基づいた固定サイズであり、境界外アクセスはアルゴリズムの誤実装を招きます。

保守担当者向けメモ

  • 本構造体は再入可能（reentrant）な設計をサポートするために使用されています。
  • ビッグエンディアンおよびリトルエンディアンの両環境で動作するよう設計されています。エンディアン変換が必要な場合は BF_swap を参照してください。
