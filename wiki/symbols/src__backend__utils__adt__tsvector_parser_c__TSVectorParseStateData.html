TSVectorParseStateData

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/adt/tsvector_parser.c:37

目的

全文検索（tsvector および tsquery）の入力文字列を解析する際、現在の解析位置、トークンバッファ、およびエラー処理コンテキストを保持するための状態管理構造体です。

シグネチャまたは形状

struct TSVectorParseStateData {
    char   *prsbuf;      /* 次の入力文字 */
    char   *bufstart;    /* 入力文字列全体（エラー報告用） */
    char   *word;        /* 現在の単語を保持するバッファ */
    int     len;         /* 'word' に割り当てられたバイトサイズ */
    int     eml;         /* 1文字あたりの最大バイト数 */
    bool    oprisdelim;  /* ! | * ( ) を区切り文字として扱うか */
    bool    is_tsquery;  /* エラーメッセージで "tsquery" を使用するか */
    bool    is_web;      /* websearch_to_tsquery() 内であるか */
    Node   *escontext;   /* ソフトエラー報告用コンテキスト */
};

入力と出力

本構造体自体はデータ保持用ですが、パーサーの各関数（init_tsvector_parser 等）を通じて以下の入出力に関与します。

  • **入力**: 解析対象の文字列、および解析ルールを決定するフラグ群。
  • **出力**: 解析されたトークン（word バッファ経由）およびエラー発生時のコンテキスト情報。

前提条件と不変条件

  • word バッファは、解析中に必要に応じて動的に拡張される可能性があります。
  • bufstart はエラー発生時の位置特定のために保持され、解析中も変更されません。
  • escontext が設定されている場合、構文エラーは例外を投げる代わりにこのコンテキストを通じて報告されます。

アルゴリズムまたは振る舞い

  • パーサーのメインループ（gettoken_tsvector）において、prsbuf を進めながら状態遷移を行い、抽出したトークンを word に格納します。
  • oprisdelim フラグにより、特定の記号を単語の一部とみなすか、区切り文字とみなすかの挙動が切り替わります。
  • RESIZEPRSBUF マクロ（関連コード参照）により、word バッファの容量不足時には自動的にメモリが再確保されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsvector_parser.c 内の init_tsvector_parser にて palloc を通じてインスタンス化されます。
  • **呼び出し先**: 本構造体は tsvector および tsquery の解析処理全体で共有されるため、gettoken_tsvector 等のパーサー関数が本構造体のメンバを直接参照・更新します。

関連データ構造

  • TSVectorParseState: 本構造体へのポインタ型（typedef struct TSVectorParseStateData *TSVectorParseState）。
  • Node: エラー報告用コンテキストとして使用。

失敗モードと境界条件

  • **メモリ不足**: word バッファの拡張時にメモリ不足が発生した場合、PostgreSQL の標準的なエラー処理により例外が発生します。
  • **構文エラー**: is_tsquery フラグの状態に応じて、エラーメッセージの主語が切り替わります。escontext が提供されている場合、エラーは例外を投げずに通知されます。

保守担当者向けメモ

  • 本構造体は tsvector と tsquery の両方で共有されています。新しい構文ルールを追加する際は、既存のフラグ（is_tsquery, is_web 等）が他の解析モードに悪影響を及ぼさないか注意してください。
  • bufstart はエラー報告専用であるため、解析ロジック内でポインタを移動させないようにしてください。
