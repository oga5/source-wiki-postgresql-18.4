brin_minmax_add_value

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_minmax.c:63

目的

BRINインデックスのページ範囲における最小値・最大値を更新し、新しい値が既存の範囲外である場合に境界を拡張します。

シグネチャまたは形状

Datum brin_minmax_add_value(PG_FUNCTION_ARGS)
  • 引数: BrinDesc*, BrinValues*, Datum (新値), bool (nullフラグ)
  • 戻り値: bool (値が更新された場合は true、それ以外は false)

入力と出力

  • 入力:
  • bdesc: インデックス記述子。
  • column: 更新対象の BrinValues 構造体。
  • newval: 追加する値。
  • isnull: 値がNULLかを示すフラグ（本関数内では false であることが前提）。
  • 出力:
  • column->bv_values[0] (最小値) または column->bv_values[1] (最大値) が更新されます。
  • 戻り値として、境界が変更されたかどうかの真偽値を返します。

前提条件と不変条件

  • isnull 引数は false である必要があります（Assert により検証）。
  • column->bv_allnulls が true の場合、newval が最小値と最大値の両方に設定され、bv_allnulls は false に更新されます。

アルゴリズムまたは振る舞い

1. bv_allnulls が真の場合、newval を最小値・最大値としてコピーし、処理を終了します。 2. bv_allnulls が偽の場合、minmax_get_strategy_procinfo を使用して比較関数を取得します。 3. BTLessStrategyNumber を用いて newval が現在の最小値より小さいか判定し、小さい場合は最小値を更新します。 4. BTGreaterStrategyNumber を用いて newval が現在の最大値より大きいか判定し、大きい場合は最大値を更新します。 5. 値が更新された場合、datumCopy を用いてメモリを確保し、必要に応じて古い値を pfree します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • 呼び出し先: minmax_get_strategy_procinfo (戦略関数取得), datumCopy (値のコピー), pfree (メモリ解放), FunctionCall2Coll (比較関数の実行)。

関連データ構造

  • BrinValues: ページ範囲の要約情報を保持する構造体。
  • BrinDesc: インデックスのメタデータ。
  • MinmaxOpaque: 比較関数をキャッシュする構造体。

失敗モードと境界条件

  • メモリ不足: datumCopy 実行時に発生する可能性があります。
  • データ型不一致: minmax_get_strategy_procinfo が適切な比較関数を返せない場合にエラーとなる可能性があります。

保守担当者向けメモ

  • bv_allnulls が true の状態から値が追加されると、その値が最小値と最大値の両方に設定される仕様です。
  • 比較関数は FmgrInfo としてキャッシュされるため、修正時は minmax_get_strategy_procinfo の設計と整合性を保ってください。
