apw_read_stream_next_block

定義

  • 種別: function
  • 定義箇所: contrib/pg_prewarm/autoprewarm.c:452

目的

read_stream API を用いたプリウォーム処理において、次に読み込むべきブロック番号を決定し、ストリームに提供する。

シグネチャまたは形状

static BlockNumber apw_read_stream_next_block(ReadStream *stream, void *callback_private_data, void *per_buffer_data)

入力と出力

  • **入力**:
  • stream: 現在の読み込みストリーム。
  • callback_private_data: AutoPrewarmReadStreamData 構造体へのポインタ。現在の読み込み位置や対象のファイル情報を保持。
  • per_buffer_data: 使用されていない。
  • **出力**:
  • 成功時: 次に読み込むべき BlockNumber。
  • 失敗または終了時: InvalidBlockNumber。

前提条件と不変条件

  • callback_private_data は AutoPrewarmReadStreamData 型としてキャスト可能であること。
  • 共有バッファの空き状況を確認する have_free_buffer() が利用可能であること。

アルゴリズムまたは振る舞い

1. CHECK_FOR_INTERRUPTS() を呼び出し、割り込み要求を確認する。 2. p->pos が apw_state->prewarm_stop_idx に達するまでループする。 3. have_free_buffer() を確認し、空きがない場合は p->pos を終了位置まで進め、InvalidBlockNumber を返して処理を中断する。 4. 現在のブロック情報（blk）が、現在のストリームの対象（テーブルスペース、ファイル番号、フォーク番号）と一致するか確認し、不一致なら InvalidBlockNumber を返す。 5. p->pos をインクリメントする。 6. blk.blocknum が対象ファイルのサイズ（p->nblocks）を超えている場合は、無効なブロックとしてスキップし、次の反復へ進む。 7. 有効なブロック番号を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_prewarm/autoprewarm.c:630 にて read_stream_begin_relation 等の引数として参照されている。
  • **呼び出し先**: have_free_buffer()（共有バッファの空き確認）、CHECK_FOR_INTERRUPTS()（割り込みチェック）。

関連データ構造

  • AutoPrewarmReadStreamData: プリウォームの進捗状況や対象ファイルのメタデータを保持する構造体。
  • BlockInfoRecord: ダンプされたブロックのメタデータ。

失敗モードと境界条件

  • **空きバッファ不足**: have_free_buffer() が偽を返すと、プリウォーム処理を即座に停止する。
  • **ファイル不一致**: 現在のストリーム対象と異なるファイルやフォークに遭遇した場合、ストリームを終了する。
  • **範囲外ブロック**: ファイルサイズ（p->nblocks）を超えるブロック番号はスキップされる。

保守担当者向けメモ

  • 本関数は read_stream コールバックとして動作するため、状態管理は callback_private_data を介して行われる。
  • p->pos の更新タイミングが、ストリームの継続条件と密接に関わっている点に注意すること。
