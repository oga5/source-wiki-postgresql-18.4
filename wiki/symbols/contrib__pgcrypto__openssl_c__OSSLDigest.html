OSSLDigest

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/openssl.c:57

目的

OpenSSL の EVP API を使用したハッシュ計算において、計算コンテキストと PostgreSQL のリソース管理機構を紐付けるためのラッパー構造体です。

シグネチャまたは形状

typedef struct OSSLDigest
{
	const EVP_MD *algo;
	EVP_MD_CTX *ctx;
	ResourceOwner owner;
} OSSLDigest;

入力と出力

  • algo: 使用するハッシュアルゴリズム（EVP_MD）。
  • ctx: OpenSSL のハッシュ計算コンテキスト（EVP_MD_CTX）。
  • owner: PostgreSQL のリソース所有者（ResourceOwner）。

前提条件と不変条件

  • ctx は OpenSSL の EVP_MD_CTX として有効である必要があります。
  • owner は、トランザクション終了時やエラー発生時にリソースを解放するために使用されます。

アルゴリズムまたは振る舞い

  • pgcrypto モジュール内でハッシュ計算を行う際、この構造体がコンテキストとして保持されます。
  • ResourceOwner を介して、PostgreSQL のエラーハンドリング（elog(ERROR) 等）によるスタック巻き戻し時に、OpenSSL のコンテキストがリークしないよう管理されます。
  • free_openssl_digest 等の関数により、OpenSSL 側のコンテキスト破棄とリソース所有者からの登録解除が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/openssl.c 内のハッシュ初期化および管理関数（px_find_digest 経由の生成処理など）。
  • **呼び出し先**: ResourceOwnerRememberOSSLDigest, ResourceOwnerForgetOSSLDigest, ResOwnerReleaseOSSLDigest などのリソース管理関数。

関連データ構造

  • EVP_MD: OpenSSL のハッシュアルゴリズム定義。
  • EVP_MD_CTX: OpenSSL のハッシュ計算状態。
  • ResourceOwner: PostgreSQL のリソース管理用構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時や、OpenSSL の初期化に失敗した場合、構造体の生成が中断されます。
  • ResourceOwner への登録が正しく行われない場合、エラー発生時にメモリリークが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は pgcrypto の OpenSSL バックエンドにおける必須コンポーネントです。
  • 新規のハッシュアルゴリズム対応や OpenSSL のバージョンアップを行う際は、ResourceOwner のライフサイクル管理が適切に維持されていることを確認してください。
