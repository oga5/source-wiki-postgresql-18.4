OpFamilyMember

定義

  • 種別: struct
  • 定義箇所: src/include/access/amapi.h:86

目的

演算子族（Operator Family）の構築時において、演算子やサポート関数を管理・保持するためのメタデータ構造体です。

シグネチャまたは形状

typedef struct OpFamilyMember
{
	bool		is_func;		/* 演算子かサポート関数か */
	Oid			object;			/* 演算子またはサポート関数のOID */
	int			number;			/* 戦略番号またはサポート関数番号 */
	Oid			lefttype;		/* 左辺の型 */
	Oid			righttype;		/* 右辺の型 */
	Oid			sortfamily;		/* ソート演算子族のOID（または0） */
	bool		ref_is_hard;	/* 依存関係の強弱（ハード/ソフト） */
	bool		ref_is_family;	/* 演算子クラスか演算子族への依存か */
	Oid			refobjid;		/* 演算子クラスまたは演算子族のOID */
} OpFamilyMember;

入力と出力

本構造体は、演算子族の定義をリストとして管理する際の各要素として利用されます。直接的な関数入出力ではなく、データ保持のためのコンテナとして機能します。

前提条件と不変条件

  • is_func が true の場合、object はサポート関数を指し、number はサポート関数番号を示します。
  • is_func が false の場合、object は演算子を指し、number は戦略番号を示します。

アルゴリズムまたは振る舞い

演算子族のバリデーションや定義の追加プロセスにおいて、各メンバーの属性を保持するために使用されます。src/backend/commands/opclasscmds.c 内の関数群により、型の割り当てやリストへの追加が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginvalidate.c
  • src/backend/access/gist/gistvalidate.c
  • src/backend/access/hash/hashvalidate.c
  • src/backend/access/nbtree/nbtvalidate.c
  • src/backend/access/spgist/spgvalidate.c
  • src/backend/commands/opclasscmds.c
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • List: 複数の OpFamilyMember を保持するために使用されます。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • sortfamily が 0 の場合、ソート演算子族が指定されていないことを示します。
  • ref_is_hard および ref_is_family は、システムカタログにおける依存関係の整合性を維持するために使用されます。

保守担当者向けメモ

  • 各インデックス AM のバリデーションロジック（*validate.c）において、リストから取り出された OpFamilyMember の各フィールドが、その AM の要件を満たしているか確認する際に参照されます。
