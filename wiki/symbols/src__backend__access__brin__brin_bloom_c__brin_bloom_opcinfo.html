brin_bloom_opcinfo

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_bloom.c:447

目的

BRINインデックスのブルームフィルタ演算子クラスにおけるメタデータ構造（BrinOpcInfo）を初期化し、インデックス構築やスキャンに必要な設定を提供します。

シグネチャまたは形状

Datum brin_bloom_opcinfo(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（標準的なPostgreSQL関数引数）
  • **出力**: BrinOpcInfo 構造体へのポインタを Datum 型として返却。

前提条件と不変条件

  • BrinOpcInfo 構造体と BloomOpaque 構造体を保持するために必要なメモリ領域を確保します。
  • oi_nstored は 1 に設定されます（ブルームフィルタは単一の BYTEA カラムとして格納されるため）。
  • oi_regular_nulls は true に設定されます。

アルゴリズムまたは振る舞い

1. palloc0 を使用して、BrinOpcInfo と BloomOpaque を格納するのに十分なメモリをゼロ初期化して確保します。 2. oi_nstored を 1 に設定します。 3. oi_regular_nulls を true に設定します。 4. oi_opaque メンバを、確保したメモリ内の BloomOpaque 領域を指すように設定します。 5. oi_typcache[0] に対して PG_BRIN_BLOOM_SUMMARYOID を用いて型キャッシュのルックアップを行います。 6. 構築した result を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は特定されていません（PostgreSQLのインデックスAMインターフェースから間接的に呼び出されます）。
  • **呼び出し先**: palloc0、lookup_type_cache、SizeofBrinOpcInfo（マクロ）。

関連データ構造

  • BrinOpcInfo: BRIN演算子クラスのメタデータ。
  • BloomOpaque: ブルームフィルタ固有の不透明なデータ構造。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • lookup_type_cache が失敗した場合、システムカタログの状態に依存してエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はブルームフィルタインデックスのメタデータ初期化を担当します。BloomOpaque の構造変更や、格納するデータ型（BYTEA）の変更が必要な場合は、本関数の設定値を確認してください。
