calc_s2k_salted

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-s2k.c:81

目的

OpenPGP仕様（RFC 4880）に基づき、パスフレーズとソルトを用いて暗号化鍵を生成する「Salted S2K」アルゴリズムを実行します。

シグネチャまたは形状

int calc_s2k_salted(PGP_S2K *s2k, PX_MD *md, const uint8 *key, unsigned key_len)

入力と出力

  • **入力**:
  • s2k: 鍵生成に必要なソルト（s2k->salt）および出力先バッファ（s2k->key）と長さ（s2k->key_len）を保持する構造体。
  • md: ハッシュ計算に使用するメッセージダイジェストコンテキスト。
  • key: パスフレーズ文字列。
  • key_len: パスフレーズの長さ。
  • **出力**:
  • 成功時は 0 を返します。
  • s2k->key に生成されたバイナリ鍵が書き込まれます。

前提条件と不変条件

  • s2k->key は、要求された鍵長を格納するのに十分なメモリ領域を確保している必要があります。
  • md は初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. ハッシュ関数の出力サイズ（md_rlen）を取得します。 2. 必要な鍵長（remain）が満たされるまで、以下のループを繰り返します。

  • px_md_reset(md) でハッシュ状態をリセットします。
  • ループ回数に応じて、先頭にゼロパディング（preload）を追加します。
  • s2k->salt とパスフレーズ（key）をハッシュ関数に入力します。
  • ハッシュ計算結果を buf に格納します。
  • buf の内容を s2k->key の現在位置へコピーし、残りの必要長を減らします。

    • 3. 最後に px_memset を使用して、一時バッファ buf をゼロクリアします。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-s2k.c:298（pgp_s2k_process 関数内から呼び出されることが確認されています）。
  • **呼び出し先**: px_md_result_size, px_md_reset, px_md_update, px_md_finish, px_memset（いずれも px.h で定義された汎用暗号インターフェース）。

関連データ構造

  • PGP_S2K: S2Kパラメータ（ソルト、鍵長、鍵バッファ）を保持する構造体。
  • PX_MD: ハッシュ計算のための抽象インターフェース。

失敗モードと境界条件

  • remain が md_rlen より小さい場合、ハッシュ結果の先頭部分のみがコピーされます。
  • 内部で使用する buf は PGP_MAX_DIGEST サイズに制限されており、これを超えるハッシュアルゴリズムには対応していません。

保守担当者向けメモ

  • 本関数はRFC 4880のSalted S2K仕様を実装しています。
  • メモリ安全性のため、計算終了後に一時バッファを確実にゼロクリアする処理が含まれています。
  • 呼び出し元で s2k 構造体が適切に初期化されていることを前提としています。
