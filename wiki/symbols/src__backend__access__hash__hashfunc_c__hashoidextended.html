hashoidextended

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hashfunc.c:122

目的

OID（Object Identifier）型の値を、指定されたシード値を用いてハッシュ化する。主にハッシュ結合のパフォーマンス向上のために使用される。

シグネチャまたは形状

Datum hashoidextended(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の2つの引数を受け取る。
  • 第1引数: OID 型の値（PG_GETARG_OID(0)）。
  • 第2引数: int64 型のシード値（PG_GETARG_INT64(1)）。
  • **出力**: 計算されたハッシュ値を Datum 型として返す。

前提条件と不変条件

  • 入力された OID 値は uint32 にキャストされて処理される。
  • ハッシュ関数は、同一の入力とシード値に対して常に決定論的な結果を返す必要がある。

アルゴリズムまたは振る舞い

1. PG_GETARG_OID(0) で取得した OID を uint32 にキャストする。 2. PG_GETARG_INT64(1) で取得したシード値とともに hash_uint32_extended を呼び出す。 3. hash_uint32_extended の戻り値をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し先**: hash_uint32_extended
  • **呼び出し元**: ソースコード内に明示的な呼び出し元の記述なし。

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • OID: オブジェクト識別子型。

失敗モードと境界条件

  • PG_FUNCTION_ARGS マクロが正しく初期化されていない場合、引数の取得に失敗する可能性がある。
  • 内部で呼び出される hash_uint32_extended の振る舞いに依存する。

保守担当者向けメモ

  • 本関数は hashoid の拡張版であり、ハッシュ結合のシード値を受け取るインターフェースを持つ。
  • 新しい型を追加する際は、同様に ...extended バージョンを実装することが推奨される。
