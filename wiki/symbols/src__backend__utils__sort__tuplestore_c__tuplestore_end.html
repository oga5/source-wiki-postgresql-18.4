tuplestore_end

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplestore.c:491

目的

Tuplestorestate 構造体に関連付けられたリソースを解放し、タプル格納処理を終了させる。

シグネチャまたは形状

void tuplestore_end(Tuplestorestate *state)

入力と出力

  • **入力**: state (Tuplestorestate *) - 終了対象となるタプルストアの状態構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • state は有効な Tuplestorestate 構造体を指している必要がある。
  • 本関数を呼び出した後、state ポインタおよびその内部のメンバへのアクセスは無効となる。

アルゴリズムまたは振る舞い

1. state->myfile が存在する場合、BufFileClose を呼び出して一時ファイルを閉じる。 2. state->context で管理されているメモリコンテキストを MemoryContextDelete で破棄する。 3. state->memtuples および state->readptrs 配列を pfree で解放する。 4. 最後に state 構造体自体を pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • src/backend/commands/trigger.c
  • src/backend/executor/execSRF.c
  • src/backend/executor/functions.c
  • src/backend/executor/nodeCtescan.c
  • src/backend/executor/nodeFunctionscan.c
  • src/backend/executor/nodeMaterial.c
  • **呼び出し先**:
  • BufFileClose
  • MemoryContextDelete
  • pfree

関連データ構造

  • Tuplestorestate: 本関数が管理・解放する主要な状態構造体。

失敗モードと境界条件

  • state が NULL の場合、pfree やメンバアクセスによりセグメンテーション違反が発生する可能性がある（ソースコード上でのガードは存在しない）。

保守担当者向けメモ

  • 本関数はメモリコンテキストごと削除するため、呼び出し後に state を参照するコードが残っていないことを確認すること。
  • BufFileClose は一時ファイルが存在する場合のみ実行されるため、メモリのみで完結した処理の後でも安全に呼び出せる。
