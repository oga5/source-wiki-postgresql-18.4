avg_pool

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_misc.c:33

目的

遺伝的アルゴリズム（GEQO）の個体群（Pool）における、全個体の適応度（worth）の平均値を算出する。主にデバッグ出力用の補助ルーチンとして使用される。

シグネチャまたは形状

static double avg_pool(Pool *pool)

入力と出力

  • **入力**: Pool *pool（評価対象の個体群構造体へのポインタ）
  • **出力**: double 型の平均適応度値

前提条件と不変条件

  • pool->size が 0 より大きいこと。0 の場合はエラーが発生する。
  • GEQO_DEBUG マクロが定義されている環境でのみ有効化される。

アルゴリズムまたは振る舞い

1. pool->size が 0 以下でないかを確認し、0 の場合は elog(ERROR) を送出する。 2. 各個体の適応度（pool->data[i].worth）を個体数（pool->size）で除算した値を累積加算する。 3. 最後に累積値を返す。

  • **注意**: オーバーフローを回避するため、合計してから除算するのではなく、各要素を個別に除算してから加算する手法を採用している。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_misc.c 内の print_gen 関数から呼び出される。
  • **呼び出し先**: PostgreSQL のエラー報告機能である elog を呼び出す可能性がある。

関連データ構造

  • Pool: 個体群を管理する構造体。data 配列（個体データ）と size（個体数）を保持する。

失敗モードと境界条件

  • pool->size が 0 の場合、elog(ERROR) により処理が中断される。
  • 個体の適応度値に DBL_MAX が含まれる場合でも、除算を先に行うことでオーバーフローを防ぐ設計となっている。

保守担当者向けメモ

  • 本関数は精度よりも安全性を優先した実装である。
  • デバッグ目的の関数であるため、パフォーマンスへの影響は考慮されていない。
  • GEQO_DEBUG が定義されていないビルドでは本関数はコンパイルされない点に注意すること。
