get_language_oid

定義

  • 種別: function
  • 定義箇所: src/backend/commands/proclang.c:225

目的

指定された言語名に対応するシステムカタログ pg_language 上の OID を取得する。

シグネチャまたは形状

Oid get_language_oid(const char *langname, bool missing_ok)

入力と出力

  • **入力**:
  • langname (const char *): 検索対象の言語名。
  • missing_ok (bool): true の場合、言語が見つからなくてもエラーを発生させず InvalidOid を返す。false の場合はエラーを投げる。
  • **出力**:
  • Oid: 見つかった言語の OID。見つからない場合は InvalidOid。

前提条件と不変条件

  • LANGNAME システムキャッシュが初期化されている必要がある。
  • langname は有効な文字列ポインタであること。

アルゴリズムまたは振る舞い

1. GetSysCacheOid1 を使用し、LANGNAME キャッシュから langname に一致する pg_language の OID を検索する。 2. 検索結果が有効な OID でなく、かつ missing_ok が false である場合、ERRCODE_UNDEFINED_OBJECT エラーを発生させて処理を中断する。 3. 検索結果（有効な OID または InvalidOid）を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c
  • src/backend/commands/functioncmds.c
  • src/backend/utils/adt/acl.c
  • **呼び出し先**:
  • GetSysCacheOid1 (システムキャッシュ検索)
  • ereport (エラー報告)

関連データ構造

  • pg_language: 言語情報を保持するシステムカタログ。
  • Oid: PostgreSQL のオブジェクト識別子型。

失敗モードと境界条件

  • 指定された言語名が存在しない場合、missing_ok が false であれば ERROR レベルで終了する。
  • missing_ok が true の場合、存在しない言語名に対しては InvalidOid を返す。

保守担当者向けメモ

  • 本関数は src/include/commands/proclang.h で宣言されており、外部モジュールからも利用可能である。
  • 言語名の検索はシステムキャッシュ経由で行われるため、カタログへの直接アクセスよりも高速である。
