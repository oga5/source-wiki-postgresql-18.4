attribute_statistics_update

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/attribute_stats.c:139

目的

リレーションの属性（カラム）に対する統計情報を、外部から提供されたデータに基づいて pg_statistic システムカタログへ更新（Upsert）する。

シグネチャまたは形状

static bool attribute_statistics_update(FunctionCallInfo fcinfo)

入力と出力

  • **入力**: FunctionCallInfo 型の引数 fcinfo。スキーマ名、リレーション名、属性名（または属性番号）、および各種統計データ（MCV、ヒストグラム等）を保持する。
  • **出力**: bool 型。統計情報の更新が成功した場合は true、一部の引数が無効で警告が発生した場合は false を返す。

前提条件と不変条件

  • データベースがリカバリ中ではないこと（リカバリ中はエラーとなる）。
  • 対象リレーションに対して ShareUpdateExclusiveLock を取得する。
  • システム列（attnum < 0）に対する統計更新は禁止されている。
  • 属性名と属性番号を同時に指定することはできない。

アルゴリズムまたは振る舞い

1. **引数検証**: stats_check_required_arg を使用し、必須引数の存在を確認する。 2. **リレーション解決**: RangeVarGetRelidExtended で対象テーブルを特定し、ロックを取得する。 3. **属性特定**: 属性名または属性番号から attnum を特定し、存在確認を行う。 4. **引数サニティチェック**: 各統計データ（MCV、ヒストグラム等）の整合性を stats_check_arg_array 等で検証する。無効なデータがある場合は警告を発行し、該当する統計フラグをオフにする。 5. **型情報の導出**: get_attr_stat_type を呼び出し、属性の型、照合順序、演算子情報を取得する。 6. **要素型導出**: MCV要素やデコード済みヒストグラムが必要な場合、get_elem_stat_type で要素型を特定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/attribute_stats.c:1029 (内部的な呼び出し)
  • src/backend/statistics/stat_utils.c:286 (位置引数による呼び出しの言及あり)
  • **呼び出し先**:
  • stats_check_required_arg, stats_check_arg_array, stats_check_arg_pair
  • RangeVarGetRelidExtended, get_attnum, get_attname, SearchSysCacheExistsAttName
  • get_attr_stat_type, get_elem_stat_type

関連データ構造

  • FunctionCallInfo: 呼び出し時の引数コンテナ。
  • pg_statistic: 更新対象のシステムカタログ。
  • attarginfo: 統計更新関数が受け取る引数の定義テーブル。

失敗モードと境界条件

  • **エラー**: リカバリ中の実行、システム列の指定、属性名と番号の同時指定、存在しない列の指定は ERROR を発生させる。
  • **警告**: 統計データの変換失敗や、必要な演算子（< 等）が欠如している場合は WARNING を発行し、処理を継続（または一部スキップ）する。

保守担当者向けメモ

  • 本関数は pg_statistic のスロット管理（set_stats_slot）や永続化（upsert_pg_statistic）の前処理を担う。
  • 統計情報の変換失敗は WARNING として報告し、可能な限り他の統計情報の更新を継続する設計となっている。
  • 拡張統計（extended_stats.c）とは異なり、単一属性の統計のみを対象とする。
