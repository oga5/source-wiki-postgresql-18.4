pg_strip_crlf

定義

  • 種別: function
  • 定義箇所: src/common/string.c:153

目的

文字列の末尾にある改行コード（\n）および復帰コード（\r）を削除し、正規化された文字列の長さを返します。

シグネチャまたは形状

int pg_strip_crlf(char *str);

入力と出力

  • **入力**: str (ヌル終端文字列へのポインタ)
  • **出力**: 処理後の文字列の新しい長さ（int 型）

前提条件と不変条件

  • 引数 str は有効なヌル終端文字列である必要があります。
  • インプレース操作を行うため、str が指すメモリ領域は書き込み可能である必要があります。

アルゴリズムまたは振る舞い

1. strlen を使用して文字列の現在の長さを取得します。 2. 文字列の末尾から逆方向に走査し、文字が \n または \r である限り、その位置にヌル文字（\0）を代入して文字列を短縮します。 3. 最終的な文字列の長さを返します。

  • **呼び出し元**:
  • src/backend/libpq/be-secure-common.c

  • src/bin/psql/prompt.c
  • **呼び出し先**: 標準 C ライブラリの strlen を使用しています。

関連データ構造

  • 特になし（標準的な char * 文字列を操作します）。

失敗モードと境界条件

  • 空文字列が渡された場合、len は 0 となり、ループは実行されず 0 が返されます。
  • 改行コードを含まない文字列が渡された場合、元の長さがそのまま返されます。
  • \r\n のような連続した改行コードは、末尾から順にすべて削除されます。

保守担当者向けメモ

  • 本関数はインプレースで文字列を書き換えるため、呼び出し元で元の文字列を保持する必要がある場合は、事前にコピーを作成してください。
  • PostgreSQL のバックエンドとフロントエンドの両方から利用されるため、変更時は両環境への影響を考慮してください。
