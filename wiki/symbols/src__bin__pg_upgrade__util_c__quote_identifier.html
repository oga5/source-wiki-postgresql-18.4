quote_identifier

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/util.c:298

目的

SQL識別子を二重引用符で囲み、識別子内部に含まれる二重引用符をエスケープすることで、SQL文中で安全に使用可能な文字列を生成します。

シグネチャまたは形状

char *quote_identifier(const char *s);

入力と出力

  • **入力**: s (const char *) - エスケープ対象の識別子文字列。
  • **出力**: (char *) - 二重引用符で囲まれ、内部の引用符がエスケープされた新しい文字列。

前提条件と不変条件

  • 入力文字列 s はヌル終端である必要があります。
  • 戻り値として返される文字列は pg_malloc によって動的に確保されます。
  • pg_upgrade の設計上、この関数が確保したメモリの解放は呼び出し側で行われない場合があり、メモリリークは許容される設計となっています。

アルゴリズムまたは振る舞い

1. 入力文字列の長さの2倍に、開始・終了の引用符およびヌル終端文字分を加えたサイズ（strlen(s) * 2 + 3）のメモリを確保します。 2. 結果文字列の先頭に " を配置します。 3. 入力文字列 s を走査し、各文字を結果文字列にコピーします。 4. 文字が " である場合、エスケープとして追加の " を挿入します。 5. 最後に終了の " とヌル終端文字を付与して返します。

呼び出し元と呼び出し先

  • **呼び出し先**: pg_malloc（メモリ確保用）。
  • **呼び出し元**: 提供されたソース情報には src/bin/pg_upgrade 内での直接的な呼び出し元は記載されていません。なお、ツールによるヒューリスティックな検索では contrib/pg_overexplain や contrib/postgres_fdw 等の外部モジュールからの参照が確認されていますが、これらが本関数を直接呼び出しているかはソースの文脈からは確定できません。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • 入力文字列が空の場合、"" という文字列が返されます。
  • メモリ確保に失敗した場合、pg_malloc の内部実装に従いプログラムが終了する可能性があります。

保守担当者向けメモ

  • 本関数はメッセージの末尾に改行を含めない設計です。
  • pg_upgrade 内の他のユーティリティ関数と同様、メモリ管理は簡略化されています。
