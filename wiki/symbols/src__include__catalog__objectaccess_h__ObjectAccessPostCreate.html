ObjectAccessPostCreate

定義

  • 種別: struct
  • 定義箇所: src/include/catalog/objectaccess.h:61

目的

SQLオブジェクト作成後のフック処理において、その作成がユーザーによる直接的な操作か、あるいはシステム内部的な処理（TOASTテーブルや型変更に伴うインデックス作成など）かを識別するためのコンテキスト情報を提供します。

シグネチャまたは形状

typedef struct {
    bool is_internal;
} ObjectAccessPostCreate;

入力と出力

  • **入力**: 構造体メンバ is_internal（bool）。
  • **出力**: なし（本構造体はフック関数へ渡されるコンテキスト情報として利用されます）。

前提条件と不変条件

  • is_internal が true の場合、そのオブジェクト作成はシステム内部的な処理として扱われるべきです。
  • 拡張機能はこのフラグを参照し、監査対象とするか否かの判断基準として利用できます。

アルゴリズムまたは振る舞い

コアコード（src/backend/catalog/objectaccess.c 等）において、オブジェクト作成イベント発生時に本構造体が初期化され、フック関数へ渡されます。フック関数側では、この構造体をキャストして受け取り、is_internal の値に基づいて処理を分岐させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaccess.c（pc_arg の初期化およびフック呼び出し）
  • **呼び出し先**:
  • contrib/sepgsql/hooks.c
  • src/test/modules/test_oat_hooks/test_oat_hooks.c

    • （上記は本構造体を引数として受け取るフック実装の例です）

関連データ構造

  • ObjectAccessType: 本構造体が関連付けられるイベントの種類（OAT_POST_CREATE）を定義します。

失敗モードと境界条件

  • 構造体の初期化時に memset 等でゼロクリアされていない場合、is_internal が不定値となるリスクがあります。ソースコード上では memset による初期化が確認されています。

保守担当者向けメモ

  • 本構造体は拡張機能のフックインターフェースの一部です。メンバの追加は、既存のプラグインとのバイナリ互換性に影響を与える可能性があるため注意してください。
