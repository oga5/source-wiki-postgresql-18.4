mic_to_euc_jp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_jp_and_sjis/euc_jp_and_sjis.c:111

目的

PostgreSQLの内部文字コードであるMULE_INTERNAL（MIC）からEUC_JPへの変換を行う、関数マネージャ経由の公開インターフェースです。

シグネチャまたは形状

Datum mic_to_euc_jp(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を取得します。
  • src (CSTRING): 変換元のMULE_INTERNAL文字列。
  • dest (CSTRING): 変換結果を格納するバッファ。
  • len (INT32): 変換対象の長さ。
  • noError (BOOL): 変換エラー発生時の挙動フラグ。
  • **出力**: Datum 型として、変換されたバイト数を返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_MULE_INTERNAL, PG_EUC_JP) により、ソースおよびターゲットのエンコーディングが正しく指定されている必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を抽出します。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングの整合性を検証します。 3. 内部関数 mic2euc_jp を呼び出し、実際の変換処理を実行します。 4. 変換されたバイト数を PG_RETURN_INT32 で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（PG_FUNCTION_INFO_V1 マクロによる登録経由）。
  • **呼び出し先**: mic2euc_jp（内部変換ロジック）。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQLの関数引数コンテナ。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • noError フラグが false の場合、変換不能な文字や不正なバイト列に対してエラーを報告する可能性があります（詳細は mic2euc_jp の実装に依存）。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、変換ロジックの詳細は mic2euc_jp に集約されています。
  • 変換エラーの挙動を変更する場合は、mic2euc_jp の実装を確認してください。
