encoding_conflicts_ascii

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_escape/test_escape.c:155

目的

指定されたエンコーディングがASCIIと競合する可能性があるか（クライアント専用のエンコーディングであるか）を判定する。

シグネチャまたは形状

static bool encoding_conflicts_ascii(int encoding)

入力と出力

  • **入力**: int encoding（PostgreSQLのエンコーディングID）
  • **出力**: bool（encoding が PG_ENCODING_BE_LAST を超える場合に true、それ以外は false）

前提条件と不変条件

  • encoding 引数は、PostgreSQLの有効なエンコーディングIDであることを前提とする。
  • この関数は、エンコーディングが「クライアント専用」であるかどうかを、ASCIIとの競合可能性を示すプロキシとして利用する。

アルゴリズムまたは振る舞い

1. 引数 encoding が PG_ENCODING_BE_LAST より大きいかどうかを比較する。 2. 大きい場合は true を返し、そうでない場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_escape/test_escape.c:656（PQclientEncoding の戻り値を引数として呼び出し）
  • **呼び出し先**:
  • 直接的な関数呼び出しはなし。

関連データ構造

  • mb/pg_wchar.h で定義されるエンコーディング定数（PG_ENCODING_BE_LAST 等）。

失敗モードと境界条件

  • encoding 値が負数や未定義のIDである場合の挙動については、ソースコード内に明示的なガード条件は存在しない。

保守担当者向けメモ

  • 本関数はエンコーディングのプロパティを直接保持しているわけではなく、あくまでクライアント専用エンコーディングかどうかの判定結果を競合の指標として使用している点に留意すること。
