intarray_del_elem

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_op.c:349

目的

PostgreSQLの整数配列から、指定された特定の整数要素をすべて削除した新しい配列を生成します。

シグネチャまたは形状

Datum intarray_del_elem(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_ARRAYTYPE_P_COPY(0): 操作対象の整数配列（コピー）。
  • PG_GETARG_INT32(1): 削除対象の整数値。
  • **出力**:
  • PG_RETURN_POINTER(a): 指定要素が削除された後の ArrayType ポインタ。

前提条件と不変条件

  • 入力配列 a は CHECKARRVALID(a) マクロにより整合性が検証されている必要があります。
  • 配列が空でない場合、内部で resize_intArrayType を呼び出し、メモリサイズを調整します。

アルゴリズムまたは振る舞い

1. 入力配列のコピーを取得し、空でないことを確認します。 2. 配列内の全要素を走査します。 3. 削除対象の要素と一致しない要素のみを、配列の先頭側に詰め直します。 4. 詰め直した後の要素数に基づいて resize_intArrayType を呼び出し、配列のサイズを縮小します。 5. 処理済みの配列ポインタを返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1(intarray_del_elem) により登録）。
  • **呼び出し先**:
  • CHECKARRVALID (マクロ)
  • ARRISEMPTY (マクロ)
  • ARRNELEMS (マクロ)
  • ARRPTR (マクロ)
  • resize_intArrayType (関数)
  • PG_RETURN_POINTER (マクロ)

関連データ構造

  • ArrayType: PostgreSQLの標準的な配列データ構造。

失敗モードと境界条件

  • **空配列**: ARRISEMPTY(a) が真の場合、何も削除処理を行わずそのまま配列を返します。
  • **対象要素なし**: 配列内に削除対象の要素が存在しない場合、元の配列と同じ内容の配列が返されます。

保守担当者向けメモ

  • 本関数は入力配列をコピーして操作するため、元のデータには影響を与えません。
  • resize_intArrayType によるメモリ再割り当てが行われるため、呼び出し側でのメモリ管理に注意してください。
