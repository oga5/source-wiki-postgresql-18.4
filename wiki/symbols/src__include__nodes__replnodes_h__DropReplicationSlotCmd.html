DropReplicationSlotCmd

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/replnodes.h:67

目的

レプリケーションスロットを削除するコマンド（DROP_REPLICATION_SLOT）のパラメータを保持するためのパースノードです。

シグネチャまたは形状

typedef struct DropReplicationSlotCmd
{
	NodeTag		type;
	char	   *slotname;
	bool		wait;
} DropReplicationSlotCmd;

入力と出力

  • **入力**: 構文解析器によって、フロントエンドから送信されたレプリケーションコマンドの引数が本構造体に格納されます。
  • **出力**: 実行エンジンが本構造体を参照し、指定されたスロットの削除処理を実行します。

前提条件と不変条件

  • type フィールドには、PostgreSQLのノードシステムにおける適切なタグが設定されている必要があります。
  • slotname は削除対象のスロット名を指す文字列へのポインタです。

アルゴリズムまたは振る舞い

本構造体はデータ保持専用のノードです。構文解析器がレプリケーションプロトコルのコマンドを解釈する際にインスタンス化され、その後、実行エンジンがこの情報を基にバックエンドのレプリケーション管理関数を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walsender.c 内の関数群が、本構造体を引数として受け取り処理を行います。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接関数を呼び出すことはありません。実行エンジン側で DropReplicationSlot(DropReplicationSlotCmd *cmd) 関数が本構造体を参照します。

関連データ構造

  • NodeTag: PostgreSQLのノードシステムでノードの種類を識別するために使用されます。

失敗モードと境界条件

  • slotname が無効な場合や、存在しないスロットを指定した場合の挙動は、本構造体ではなく、これを受け取る実行エンジン側の実装（walsender.c 等）に依存します。

保守担当者向けメモ

  • 新しいレプリケーションコマンドのオプションを追加する場合は、本構造体にフィールドを追加し、src/backend/nodes/copyfuncs.c および outfuncs.c での対応が必要になる可能性があります。
