shiftList

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginfast.c:553

目的

GINインデックスのペンディングリストから、処理済みのページを削除し、メタページの管理情報を更新する。

シグネチャまたは形状

static void shiftList(Relation index, Buffer metabuffer, BlockNumber newHead,
                      bool fill_fsm, IndexBulkDeleteResult *stats)

入力と出力

  • index: 対象のインデックスリレーション。
  • metabuffer: GINメタページのバッファ。
  • newHead: 削除対象リストの終了地点（このブロック番号に到達するまで削除を行う）。
  • fill_fsm: true の場合、削除したページをFSM（Free Space Map）へ返却する。
  • stats: 削除されたページ数を記録する統計構造体（NULL可）。

前提条件と不変条件

  • metabuffer は排他ロックされている必要がある。
  • newHead は現在のペンディングリストのヘッドから到達可能なブロック番号であること。
  • 削除対象の各ページは、処理中に排他ロックされる。

アルゴリズムまたは振る舞い

1. 現在のメタページのヘッドから newHead に到達するまで、GIN_NDELETE_AT_ONCE（定数）単位でページを読み込み、排他ロックを取得する。 2. 各ページのタプル数を集計し、メタページのペンディングリスト管理情報（head, nPendingPages, nPendingHeapTuples）を更新する。 3. 削除対象ページのフラグを GIN_DELETED に設定する。 4. WALが必要な場合、メタページと削除対象ページ群の変更を記録する。 5. クリティカルセクション内でメタページと各ページを更新し、LSNを設定する。 6. 処理終了後、各バッファをアンロック・解放する。 7. fill_fsm が真であれば、削除したページを RecordFreeIndexPage を通じてFSMへ返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginfast.c 内の ginInsertCleanup から呼び出される。
  • **呼び出し先**: ReadBuffer, LockBuffer, BufferGetPage, GinPageGetMeta, GinPageGetOpaque, GinPageIsDeleted, XLogEnsureRecordSpace, START_CRIT_SECTION, MarkBufferDirty, XLogBeginInsert, XLogRegisterBuffer, XLogRegisterData, XLogInsert, PageSetLSN, UnlockReleaseBuffer, END_CRIT_SECTION, RecordFreeIndexPage。

関連データ構造

  • GinMetaPageData: メタページ内のペンディングリスト管理構造体。
  • ginxlogDeleteListPages: 削除操作を記録するためのWALログ構造体。
  • Buffer: インデックスページを保持するバッファ。

失敗モードと境界条件

  • newHead が InvalidBlockNumber の場合、リスト全体が空になったとみなし、メタページのテール情報やカウントをリセットする。
  • ページ削除中にクラッシュした場合、WALによりメタページと各ページの整合性が保たれる。

保守担当者向けメモ

  • 本関数は一度に大量のページを操作するため、WAL記録前に XLogEnsureRecordSpace を呼び出し、必要な領域を確保している。
  • pd_lower の更新は、古いPostgreSQLバージョンとの互換性維持およびメタページ圧縮時のデータ消失防止のために必須である。
