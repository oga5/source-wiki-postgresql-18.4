GinBufferReset

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/gininsert.c:1551

目的

GinBuffer 構造体の内容をリセットし、保持しているメモリリソースを解放します。

シグネチャまたは形状

static void GinBufferReset(GinBuffer *buffer)

入力と出力

  • **入力**: GinBuffer *buffer（リセット対象のバッファへのポインタ）
  • **出力**: なし（副作用として buffer のメンバが初期化される）

前提条件と不変条件

  • buffer は GinBufferIsEmpty(buffer) が偽であること（空ではない状態であること）を前提とします。

アルゴリズムまたは振る舞い

1. Assert によりバッファが空でないことを確認します。 2. バッファ内のキーが GIN_CAT_NORM_KEY カテゴリであり、かつ typbyval が偽（参照渡し）である場合、pfree を呼び出して key が指すメモリを解放します。 3. buffer->key を 0 に設定します。 4. attnum, category, keylen, nitems, nfrozen, typlen, typbyval の各メンバを 0 にリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/gininsert.c 内の以下の行から呼び出されています。
  • 1727行目
  • 1782行目

  • **呼び出し先**: pfree（メモリ解放用）および GinBufferIsEmpty（アサーション用）。

関連データ構造

  • GinBuffer: GINインデックス構築時にキーごとのTIDリストを一時的に蓄積するために使用される構造体。

失敗モードと境界条件

  • buffer が指すメモリ領域が不正な場合、セグメンテーション違反が発生する可能性があります。
  • GinBufferIsEmpty が真の状態で呼び出された場合、Assert によりプロセスが停止します（デバッグビルド時）。

保守担当者向けメモ

  • 本関数は GinBuffer の再利用前に状態をクリアするために使用されます。参照渡し型のキーを扱う際はメモリリークを防ぐため、必ず typbyval のチェックを維持してください。
