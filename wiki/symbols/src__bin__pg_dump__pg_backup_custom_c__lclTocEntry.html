lclTocEntry

定義

  • **種別**: struct
  • **定義箇所**: src/bin/pg_dump/pg_backup_custom.c:76

目的

カスタム形式アーカイブにおける各TOC（目次）エントリに関連付けられたデータ位置情報を保持するための構造体です。

シグネチャまたは形状

typedef struct
{
	int			dataState;
	pgoff_t		dataPos;
} lclTocEntry;

入力と出力

  • **入力**: pg_malloc0 を介してメモリが割り当てられ、te->formatData（TocEntry 構造体のメンバ）に格納されます。
  • **出力**: dataPos メンバは、アーカイブ内のデータブロックへのオフセットを保持します。

前提条件と不変条件

  • dataPos は、dataState が K_OFFSET_POS_SET である場合にのみ有効な値として扱われます。

アルゴリズムまたは振る舞い

  • 本構造体は、カスタム形式のアーカイブにおいて、特定のテーブルデータやラージオブジェクトがファイル内のどこに配置されているかを追跡します。
  • pg_backup_custom.c 内の各関数（_PrintTocData 等）は、te->formatData を lclTocEntry 型にキャストしてアクセスし、シーク操作やデータ読み込みの制御に使用します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_custom.c 内の複数の関数（_PrintTocData など）が、te->formatData を通じて本構造体を参照・更新します。
  • **呼び出し先**: 情報なし。

関連データ構造

  • TocEntry (外部): te->formatData を通じて本構造体を保持します。
  • lclContext: アーカイブ全体のコンテキストを保持する構造体。

失敗モードと境界条件

  • シーク不可能なストリーム（パイプなど）を使用している場合、dataPos を用いた直接的なシークができないため、順次スキャンが必要となります。
  • マルチスレッド環境において、同一の lclTocEntry インスタンスへ同時にアクセスする際の排他制御については、実装上の注意が必要です。

保守担当者向けメモ

  • dataState の具体的な値（K_OFFSET_POS_SET 等）の定義については、pg_backup_custom.c のソースコード全体を参照してください。
  • Windows環境におけるファイルハンドル操作やマルチスレッド時の挙動については、ソース内のコメントにプラットフォーム固有の制約が記載されています。
