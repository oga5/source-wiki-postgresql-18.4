get_view_query

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteHandler.c:2548

目的

ビューのリレーションキャッシュから、そのビューを定義する _RETURN ルール（CMD_SELECT）のクエリツリーを取得する。

シグネチャまたは形状

Query *get_view_query(Relation view)

入力と出力

  • **入力**: view (Relation) - 対象となるビューのリレーション構造体。
  • **出力**: Query * - ビューの _RETURN ルールに含まれるクエリツリーへのポインタ。

前提条件と不変条件

  • 入力された view は RELKIND_VIEW である必要がある（Assert により検証）。
  • ビューには必ず単一のアクションを持つ CMD_SELECT ルール（_RETURN ルール）が存在しなければならない。

アルゴリズムまたは振る舞い

1. view->rd_rules を走査し、CMD_SELECT イベントを持つルールを検索する。 2. 対象ルールが見つかった場合、その actions リストの長さを確認する。 3. アクションが単一であれば、そのクエリツリーを返す。 4. ルールが見つからない、またはアクションの仕様が不正な場合は elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/lockcmds.c:254
  • src/backend/commands/tablecmds.c:16726
  • src/backend/rewrite/rewriteHandler.c:3042
  • src/backend/rewrite/rewriteHandler.c:3306
  • **呼び出し先**:
  • list_length (マクロ)
  • linitial (マクロ)
  • elog (エラー報告)

関連データ構造

  • Relation: リレーションキャッシュ構造体。
  • RewriteRule: ルール定義を保持する構造体。
  • Query: 書き換え後のクエリツリー。

失敗モードと境界条件

  • view->rd_rel->relkind が RELKIND_VIEW でない場合、アサーション失敗により異常終了する。
  • _RETURN ルールが存在しない場合、elog(ERROR) により処理が中断される。
  • _RETURN ルールのアクション数が 1 以外の場合、elog(ERROR) により処理が中断される。
  • 返されるポインタはリレーションキャッシュ内のメモリを指しているため、呼び出し側で変更を加える場合は copyObject() 等による複製が必要となる。

保守担当者向けメモ

  • 本関数はリレーションキャッシュ内のポインタを直接返すため、呼び出し側でクエリツリーを改変する場合は注意が必要である（rewriteHandler.c:3306 では copyObject を併用している）。
  • "RIR"（Retrieve-Instead-Retrieve）ルールの実装の一部であり、ビューの展開処理において不可欠な関数である。
