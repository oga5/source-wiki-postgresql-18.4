crc32c_bytea

定義

  • 種別: function
  • 定義箇所: src/backend/utils/hash/pg_crc.c:119

目的

SQLインターフェースを通じて、bytea 型のデータに対する CRC-32C（Castagnoli）チェックサムを計算します。

シグネチャまたは形状

Datum crc32c_bytea(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される bytea 型のデータ。
  • **出力**: PG_RETURN_INT64 を介して返される int64 型の CRC-32C 値。

前提条件と不変条件

  • 入力データは PG_GETARG_BYTEA_PP で取得可能であること。
  • pg_crc32c 型の変数が初期化され、計算後に最終処理（ビット反転等）が行われること。

アルゴリズムまたは振る舞い

1. INIT_CRC32C(crc) マクロにより、CRCアキュムレータを初期化します。 2. VARDATA_ANY(in) および VARSIZE_ANY_EXHDR(in) を使用して、入力データのポインタとサイズを取得します。 3. COMP_CRC32C マクロにより、入力データに対して CRC-32C アルゴリズムを適用します。 4. FIN_CRC32C(crc) マクロにより、最終的な計算処理を完了させます。 5. 計算された値を int64 として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に直接的な呼び出し元の記述はありません。SQL関数として外部から呼び出されることを想定しています。
  • **呼び出し先**: INIT_CRC32C, COMP_CRC32C, FIN_CRC32C マクロ（port/pg_crc32c.h および utils/pg_crc.h で定義）を呼び出します。

関連データ構造

  • bytea: PostgreSQLの可変長バイト列型。
  • pg_crc32c: CRC-32C計算用の型。

失敗モードと境界条件

  • 入力データが NULL の場合や、bytea 型として不正な形式である場合の挙動については、PG_GETARG_BYTEA_PP の仕様に依存します。

保守担当者向けメモ

  • 本関数は現代的な CRC-32C を使用しており、レガシーな CRC-32 とは異なります。新規開発ではこちらを推奨します。
  • ハードウェアアクセラレーションが利用可能な環境では、COMP_CRC32C マクロ内で CPU 命令が活用されます。
