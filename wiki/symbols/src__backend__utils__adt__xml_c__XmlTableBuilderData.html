XmlTableBuilderData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/xml.c:196

目的

XMLTABLE 演算子の実行状態を保持するためのコンテキスト構造体です。XMLドキュメントの解析、XPathの評価、および行単位のデータ抽出に必要なリソースを管理します。

シグネチャまたは形状

typedef struct XmlTableBuilderData
{
    int            magic;
    int            natts;
    long int       row_count;
    PgXmlErrorContext *xmlerrcxt;
    xmlParserCtxtPtr ctxt;
    xmlDocPtr      doc;
    xmlXPathContextPtr xpathcxt;
    xmlXPathCompExprPtr xpathcomp;
    xmlXPathObjectPtr xpathobj;
    xmlXPathCompExprPtr *xpathscomp;
} XmlTableBuilderData;

入力と出力

  • **入力**: XMLTABLE 演算子の実行時に、palloc0 を介してメモリが確保され、各フィールドが初期化されます。
  • **出力**: 演算子の実行終了時、またはエラー発生時に、保持している libxml2 関連のリソース（doc, xpathcxt 等）が解放されます。

前提条件と不変条件

  • libxml2 ライブラリが初期化されている必要があります。
  • magic フィールドは、構造体の整合性チェックに使用される可能性があります。
  • PgXmlErrorContext を通じて、PostgreSQL のエラーハンドリング機構と統合されている必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: palloc0 によりゼロクリアされた状態で生成されます。 2. **解析と評価**: libxml2 のパーサコンテキスト（ctxt）および XPath コンテキスト（xpathcxt）を保持し、XMLドキュメントの解析および XPath 式のコンパイル・評価を行います。 3. **状態管理**: row_count により処理済みの行数を追跡し、xpathscomp を通じて各カラムに対応する XPath 式を管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/xml.c 内の XMLTABLE 関連関数（GetXmlTableBuilderPrivateData, XmlTableInitOpaque 等）から参照・操作されます。
  • **呼び出し先**: libxml2 ライブラリの各関数（xmlParserCtxtPtr, xmlXPathContextPtr 等を引数に取る関数）を呼び出します。

関連データ構造

  • PgXmlErrorContext: XML処理中のエラーをPostgreSQLの ereport に変換するために使用されます。
  • xmlParserCtxtPtr, xmlDocPtr, xmlXPathContextPtr: libxml2 が提供するXML処理用の主要な構造体です。

失敗モードと境界条件

  • libxml2 のメモリ不足や不正なXML形式により、xmlerrcxt を介したエラー報告が発生する可能性があります。
  • XPath 式のコンパイル失敗や評価エラーが発生した場合、libxml2 のエラーハンドラが呼び出されます。

保守担当者向けメモ

  • 本構造体は libxml2 のグローバルな状態に依存するため、メモリコンテキストの管理には注意が必要です。
  • XMLTABLE 演算子の実行ノード（nodeTableFuncscan.c）と密接に関連しているため、変更時は当該ノードへの影響を確認してください。
  • magic フィールドの具体的な検証ロジックについては、ソースコード内の GetXmlTableBuilderPrivateData 等のアクセサ関数を参照してください。
