PLy_cursor_fetch

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_cursorobject.c:397

目的

Python から PostgreSQL のカーソルに対して指定行数のデータフェッチを行い、結果を PLyResultObject として返却する。

シグネチャまたは形状

static PyObject *PLy_cursor_fetch(PyObject *self, PyObject *args)
  • self: PLyCursorObject インスタンス。
  • args: フェッチする行数（整数）を含む Python タプル。

入力と出力

  • **入力**: Python の引数タプル（行数 count を指定）。
  • **出力**: 取得した行を格納した PLyResultObject。失敗時は NULL を返し、Python 例外をセットする。

前提条件と不変条件

  • カーソルがクローズされていないこと。
  • カーソルが有効なポータルを参照していること（アボートされたサブトランザクション内のカーソルは無効）。
  • SPI_processed が PY_SSIZE_T_MAX を超えないこと。

アルゴリズムまたは振る舞い

1. 引数から count を解析する。 2. カーソルの状態を確認し、クローズ済みまたは無効なポータルの場合はエラーを投げる。 3. PLy_result_new() で結果オブジェクトを生成する。 4. PLy_spi_subtransaction_begin() を呼び出し、SPI 操作のためのサブトランザクションを開始する。 5. PG_TRY ブロック内で SPI_cursor_fetch を実行し、結果を PLyResultObject に格納する。

  • 取得したタプルを PLy_input_from_tuple を用いて Python オブジェクトに変換し、リストへ追加する。

    • 6. SPI_freetuptable でタプルテーブルを解放し、サブトランザクションをコミットする。 7. PG_CATCH ブロックでエラー発生時にサブトランザクションをアボートする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_cursorobject.c 内のメソッドテーブル（{"fetch", PLy_cursor_fetch, ...}）を介して Python インタプリタから呼び出される。
  • **呼び出し先**: SPI_cursor_fetch, PLy_result_new, PLy_spi_subtransaction_begin/commit/abort, PLy_input_from_tuple 等。

関連データ構造

  • PLyCursorObject: カーソル状態（ポータル名、クローズ状態等）を保持。
  • PLyResultObject: フェッチ結果（行データ、ステータス等）を保持。
  • Portal: PostgreSQL のカーソル管理用構造体。

失敗モードと境界条件

  • **ValueError**: カーソルが既にクローズされている、またはサブトランザクションのアボートによりポータルが無効な場合に発生。
  • **MemoryError**: PyList_New 等のメモリ確保失敗時に発生。
  • **Program Limit Exceeded**: 取得行数が PY_SSIZE_T_MAX を超える場合に発生。
  • **SPI エラー**: SPI_cursor_fetch が失敗した場合、サブトランザクションのアボートを経て NULL を返す。

保守担当者向けメモ

  • 本関数は PG_TRY/PG_CATCH を使用してサブトランザクションの整合性を保証している。
  • SPI_tuptable の解放漏れがないよう注意すること。
  • Python の参照カウント（Py_DECREF）管理が適切に行われているか、変更時には特に注意が必要。
