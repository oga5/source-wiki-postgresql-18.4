SearchSysCache4

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/syscache.c:253

目的

4つのキーを条件として、システムカタログキャッシュから特定のタプルを検索します。

シグネチャまたは形状

HeapTuple SearchSysCache4(int cacheId, Datum key1, Datum key2, Datum key3, Datum key4)

入力と出力

  • **入力**:
  • cacheId: 検索対象のキャッシュID（SysCache 配列のインデックス）。
  • key1 ～ key4: 検索に使用する4つの Datum 型のキー。
  • **出力**:
  • 成功時: 見つかったタプルへのポインタ (HeapTuple)。
  • 失敗時: NULL。

前提条件と不変条件

  • cacheId は有効な範囲内（0 以上 SysCacheSize 未満）であること。
  • 指定された cacheId に対応するキャッシュが存在すること。
  • 指定されたキャッシュのキー数が4つであること（cc_nkeys == 4）。

アルゴリズムまたは振る舞い

1. cacheId が有効かつキー数が4であることを Assert で検証します。 2. SysCache 配列から該当するキャッシュ構造体を取得します。 3. SearchCatCache4 を呼び出し、指定された4つのキーを用いてキャッシュ内の検索を実行し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_inclusion.c
  • src/backend/access/brin/brin_minmax.c
  • src/backend/access/brin/brin_minmax_multi.c
  • src/backend/catalog/namespace.c
  • src/backend/catalog/objectaddress.c
  • src/backend/catalog/pg_operator.c
  • src/backend/utils/cache/lsyscache.c
  • **呼び出し先**:
  • SearchCatCache4 (内部的な検索実行関数)

関連データ構造

  • SysCache[]: システムキャッシュを保持する CatCache 構造体の配列。
  • HeapTuple: 検索結果として返されるタプル構造体。

失敗モードと境界条件

  • cacheId が無効な場合、またはキー数が4でない場合は Assert により異常終了します。
  • 検索条件に合致するタプルが存在しない場合、NULL を返します。

保守担当者向けメモ

  • 本関数はキャッシュ内部のポインタを返すため、呼び出し側でタプルを直接変更してはいけません。変更が必要な場合は SearchSysCacheCopy を使用してください。
  • 新しいキャッシュを追加する場合は syscache_info.h を編集し、DECLARE_UNIQUE_INDEX が定義されていることを確認してください。
