RelInfoArr

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.h:151

目的

データベース内のリレーション情報（RelInfo）の配列を保持し、アップグレード対象となるリレーションの集合を管理するために使用されます。

シグネチャまたは形状

typedef struct
{
	RelInfo    *rels;
	int			nrels;
} RelInfoArr;

入力と出力

  • rels: RelInfo 構造体へのポインタ（配列の先頭）。
  • nrels: 配列に含まれるリレーションの総数。

前提条件と不変条件

  • nrels が 0 より大きい場合、rels は有効なメモリ領域を指している必要があります。
  • nrels が 0 の場合、rels は NULL であるか、空の配列として扱われます。

アルゴリズムまたは振る舞い

  • pg_upgrade において、特定のデータベース内の全リレーション情報を一括管理するために使用されます。
  • info.c 内の関数によって、リレーション情報のメモリ解放（free_rel_infos）やデバッグ出力（print_rel_infos）が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/check.c: new_cluster.dbarr.dbs[dbnum].rel_arr として参照。
  • src/bin/pg_upgrade/pg_upgrade.h: ClusterInfo 構造体の一部として定義。
  • **呼び出し先**:
  • src/bin/pg_upgrade/info.c: free_rel_infos および print_rel_infos 関数が引数として受け取り処理を実行。

関連データ構造

  • RelInfo: 本構造体が保持する個々のリレーション情報。
  • DbInfo: RelInfoArr をメンバとして保持し、データベース単位の情報を管理する構造体。

失敗モードと境界条件

  • nrels が負の値になることは想定されていません。
  • メモリ割り当てに失敗した場合、rels が正しく初期化されない可能性があります。

保守担当者向けメモ

  • RelInfoArr のメモリ管理は info.c 内の関数に依存しているため、配列の拡張や解放を行う際はこれらの関数との整合性に注意してください。
