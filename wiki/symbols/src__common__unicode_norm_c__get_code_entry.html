get_code_entry

定義

  • 種別: function
  • 定義箇所: src/common/unicode_norm.c:72

目的

指定された Unicode コードポイントに対応する分解情報を、環境（フロントエンド/バックエンド）に応じた最適な検索手法を用いて取得する。

シグネチャまたは形状

static const pg_unicode_decomposition *get_code_entry(pg_wchar code)

入力と出力

  • **入力**: pg_wchar code（検索対象の Unicode コードポイント）
  • **出力**: pg_unicode_decomposition 構造体へのポインタ。該当するエントリが見つからない場合は NULL を返す。

前提条件と不変条件

  • バックエンド環境では、UnicodeDecompInfo が提供する完全ハッシュ関数が有効であること。
  • フロントエンド環境では、UnicodeDecompMain 配列がソート済みであること。

アルゴリズムまたは振る舞い

  • **バックエンド (!FRONTEND)**:

    • 1. コードポイントをネットワークバイトオーダーに変換し、ハッシュ値を計算する。 2. ハッシュ値を用いて UnicodeDecompInfo からエントリを特定する。 3. 範囲外のハッシュ値またはコードポイントの不一致がある場合は NULL を返す。

  • **フロントエンド (FRONTEND)**:

    • 1. bsearch を使用し、UnicodeDecompMain 配列に対して conv_compare 関数を用いたバイナリサーチを実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/unicode_norm.c 内の複数箇所（行66, 73, 115, 187, 362）で参照されている。
  • **呼び出し先**:
  • バックエンド: decompinfo.hash
  • フロントエンド: bsearch（標準ライブラリ関数）、conv_compare

関連データ構造

  • pg_unicode_decomposition: 分解情報を保持する構造体。
  • pg_unicode_decompinfo: バックエンド用のハッシュテーブル情報。
  • UnicodeDecompMain: フロントエンド用の静的データ配列。

失敗モードと境界条件

  • 指定されたコードポイントがテーブル内に存在しない場合、NULL を返す。
  • バックエンドにおいて、ハッシュ計算結果がテーブルの範囲外である場合、NULL を返す。

保守担当者向けメモ

  • 本関数はプリプロセッサマクロ FRONTEND により実装が完全に分岐する。変更時は両方のパスを確認すること。
  • バックエンドのハッシュテーブルは「完全ハッシュ」であるため、ハッシュ衝突後の再検索は考慮されていない。
