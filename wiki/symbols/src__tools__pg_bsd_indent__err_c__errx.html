errx

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/err.c:57

目的

システムエラー（errno）を伴わない致命的なエラーメッセージを標準エラー出力（stderr）へ出力し、指定された終了コードでプロセスを終了させる。

シグネチャまたは形状

void errx(int eval, const char *fmt, ...);

入力と出力

  • eval (int): プロセス終了時に使用する終了ステータスコード。
  • fmt (const char *): vfprintf に渡されるフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • 出力: 成功時、フォーマットされたメッセージと改行が stderr に書き込まれる。戻り値はない。

前提条件と不変条件

  • pg_noreturn 属性が付与されており、本関数呼び出し後のコードは到達不能（unreachable）である。
  • fmt が NULL の場合、メッセージ出力はスキップされるが、改行の出力とプロセスの終了は実行される。

アルゴリズムまたは振る舞い

1. va_start を使用して可変長引数リストを初期化する。 2. fmt が NULL でない場合、vfprintf を用いて stderr にメッセージを出力する。 3. stderr に改行文字を出力する。 4. va_end で引数リストを終了する。 5. exit(eval) を呼び出し、プロセスを即座に終了させる。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/tools/pg_bsd_indent/args.c (引数解析エラー)
  • src/tools/pg_bsd_indent/indent.c (ファイル操作や入力エラー)
  • src/tools/pg_bsd_indent/io.c (入出力制限エラー)
  • 呼び出し先:
  • vfprintf (標準ライブラリ)
  • fprintf (標準ライブラリ)
  • exit (標準ライブラリ)

関連データ構造

  • va_list (可変長引数処理用)

失敗モードと境界条件

  • fmt が NULL の場合、メッセージは出力されず改行のみが出力される。
  • exit を呼び出すため、呼び出し元以降の処理は一切実行されない。

保守担当者向けメモ

  • BSD 由来の err(3) ファミリーの最小限の実装である。
  • システムエラー（errno）を伴うエラー報告が必要な場合は、対となる err 関数を使用すること。
