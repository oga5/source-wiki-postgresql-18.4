ExecAppendInitializeWorker

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAppend.c:554

目的

並列クエリ実行において、ワーカープロセス側で Append ノードの状態を初期化し、共有メモリ上の並列実行状態をアタッチする。

シグネチャまたは形状

void ExecAppendInitializeWorker(AppendState *node, ParallelWorkerContext *pwcxt)

入力と出力

  • **入力**:
  • node: 初期化対象の AppendState 構造体。
  • pwcxt: 並列ワーカーのコンテキスト情報（共有メモリの TOC を含む）。
  • **出力**: なし（node 構造体のフィールドを更新する）。

前提条件と不変条件

  • pwcxt->toc が有効であり、node->ps.plan->plan_node_id に対応するエントリが共有メモリ上に存在すること。

アルゴリズムまたは振る舞い

1. shm_toc_lookup を使用し、共有メモリ上の TOC から node->ps.plan->plan_node_id に紐づく並列実行状態（ParallelAppendState）を検索し、node->as_pstate に格納する。 2. ワーカープロセス用のサブプラン選択関数として choose_next_subplan_for_worker を node->choose_next_subplan に割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:1366（並列ワーカーの初期化プロセスから呼び出される）。
  • **呼び出し先**: shm_toc_lookup（共有メモリ管理関数）、choose_next_subplan_for_worker（関数ポインタへの代入先）。

関連データ構造

  • AppendState: Append ノードの実行時状態を保持。
  • ParallelWorkerContext: 並列ワーカーの実行コンテキスト。
  • ParallelAppendState: 並列実行時にワーカー間で共有される状態。

失敗モードと境界条件

  • shm_toc_lookup が指定された ID を見つけられない場合、false 引数によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はワーカープロセスでのみ実行される。リーダープロセス側の初期化については ExecAppendInitializeDSM を参照のこと。
  • choose_next_subplan の切り替えにより、ワーカーは共有メモリ上の pa_next_plan を参照して動的にサブプランを選択するようになる。
