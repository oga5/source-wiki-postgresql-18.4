pg_enc2name

定義

  • 種別: struct
  • 定義箇所: src/include/mb/pg_wchar.h:355

目的

PostgreSQLのエンコーディング識別子（pg_enc）と、それに対応する人間が読み取り可能なエンコーディング名（文字列）を紐付けるためのデータ構造です。

シグネチャまたは形状

typedef struct pg_enc2name
{
	const char *name;
	pg_enc		encoding;
#ifdef WIN32
	unsigned	codepage;
#endif
} pg_enc2name;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。入力としてエンコーディング名やIDを保持し、出力としてそれらの属性を提供します。

前提条件と不変条件

  • name フィールドは、エンコーディングを識別する文字列へのポインタです。
  • encoding フィールドは、pg_enc 列挙型で定義された有効なエンコーディングIDである必要があります。
  • WIN32環境下では、Windows固有のコードページ番号を保持します。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のみを行います。この構造体の配列である pg_enc2name_tbl を通じて、エンコーディングIDから名前への変換や、その逆引きを行うためのルックアップテーブルとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/utils/mb/mbutils.c: セッションのクライアント、データベース、メッセージエンコーディングの初期状態保持に使用。
  • src/common/encnames.c: エンコーディング名テーブル（pg_enc2name_tbl）の定義および検索処理で使用。
  • **呼び出し先**:
  • 本構造体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • pg_enc: エンコーディングを識別するための列挙型。
  • pg_enc2name_tbl: pg_enc2name 構造体の配列であり、システムでサポートされる全エンコーディングの定義を保持します。

失敗モードと境界条件

  • pg_enc2name_tbl のインデックスとして無効な pg_enc 値を使用した場合、メモリの不正参照が発生する可能性があります。
  • WIN32環境以外では codepage フィールドが存在しないため、プラットフォーム間での構造体サイズの違いに注意が必要です。

保守担当者向けメモ

  • pg_enc 列挙型の順序を変更すると、pg_enc2name_tbl のインデックスと整合性が取れなくなるため、libpqのABI破壊を避けるために慎重な修正が求められます。
  • 新しいエンコーディングを追加する際は、src/common/encnames.c 内のテーブル定義を更新する必要があります。
