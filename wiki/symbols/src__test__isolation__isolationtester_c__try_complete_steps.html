try_complete_steps

定義

  • 種別: function
  • 定義箇所: src/test/isolation/isolationtester.c:754

目的

複数のステップ（PermutationStep）の実行完了を待機し、ブロック条件が満たされるまでポーリングを行うことで、テストの進行を制御します。

シグネチャまたは形状

static int try_complete_steps(TestSpec *testspec, PermutationStep **waiting, int nwaiting, int flags);

入力と出力

  • **入力**:
  • testspec: テスト仕様全体を保持する構造体。
  • waiting: 完了待ちのステップを格納した配列。
  • nwaiting: waiting 配列内のステップ数。
  • flags: 実行時の動作を制御するフラグ。
  • **出力**:
  • int: 処理終了後にまだ完了していない（待機中の）ステップの数。

前提条件と不変条件

  • waiting 配列内の各ステップは、実行中または完了待ちの状態である必要があります。
  • any_new_notice グローバル変数は、ループ内での通知受信状態を追跡するために使用されます。

アルゴリズムまたは振る舞い

1. **ループ開始**: ステップの完了または通知の受信がある限り、ループを繰り返します。 2. **通知リセット**: ループごとに any_new_notice を false にリセットします。 3. **ステップ走査**: waiting 配列内の各ステップに対し、try_complete_step を呼び出します。

  • try_complete_step が成功（完了）した場合、配列からそのステップを除去し、nwaiting を減らします。
  • 失敗（ブロック中）した場合、配列に保持し続けます。

    • 4. **再試行判定**: ブロック条件を持つステップが存在し、かつステップの完了または新規通知が発生した場合は、ステップ間の依存関係を解消するために再度ループを実行します。 5. **終了**: 完了しなかったステップの数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/isolation/isolationtester.c 内の run_permutation 等から複数回呼び出されています。
  • **呼び出し先**:
  • try_complete_step（各ステップの個別の完了判定およびポーリングを実行）。

関連データ構造

  • TestSpec: テスト全体の定義。
  • PermutationStep: 実行される個別のステップ情報。

失敗モードと境界条件

  • ステップがタイムアウトした場合や、デッドロックが発生した際の挙動は try_complete_step の実装に依存します。
  • nwaiting が 0 の場合、ループは即座に終了します。

保守担当者向けメモ

  • 本関数は、ステップの完了順序が他のステップのブロック解除に影響を与える可能性があるため、any_new_notice や nwaiting の変化を監視して再試行する設計になっています。
  • memmove を使用して配列を詰め直しているため、ステップ数が多い場合のパフォーマンスに注意してください。
