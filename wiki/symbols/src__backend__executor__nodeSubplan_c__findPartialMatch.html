findPartialMatch

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSubplan.c:726

目的

指定された TupleHashTable 内に、入力 slot と等価ではない（execTuplesUnequal が偽を返す）エントリが存在するかを判定します。

シグネチャまたは形状

static bool findPartialMatch(TupleHashTable hashtable, TupleTableSlot *slot, FmgrInfo *eqfunctions)

入力と出力

  • **入力**:
  • hashtable: 検索対象のハッシュテーブル。
  • slot: 比較基準となるタプルスロット。
  • eqfunctions: 比較に使用する等価性関数群。
  • **出力**:
  • bool: ハッシュテーブル内に条件を満たすエントリが見つかった場合は true、それ以外は false を返します。

前提条件と不変条件

  • hashtable は有効な TupleHashTable 構造体である必要があります。
  • eqfunctions は hashtable のキー列数に対応した関数群である必要があります。

アルゴリズムまたは振る舞い

1. InitTupleHashIterator を用いてハッシュテーブルのイテレータを初期化します。 2. ScanTupleHashTable を使用して、ハッシュテーブル内の全エントリを順次走査します。 3. 各エントリに対し CHECK_FOR_INTERRUPTS() を実行し、割り込み要求を確認します。 4. ExecStoreMinimalTuple を呼び出し、現在のエントリを hashtable->tableslot に展開します。 5. execTuplesUnequal を呼び出し、入力 slot と hashtable->tableslot を比較します。 6. 比較の結果、等価でない（execTuplesUnequal が false を返す）エントリが見つかった場合、イテレータを終了し true を返します。 7. 全エントリを走査しても条件を満たすものが見つからなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeSubplan.c 内のサブプラン実行ロジック（ExecHashSubPlan 等）から呼び出されています。
  • **呼び出し先**:
  • InitTupleHashIterator, ScanTupleHashTable, TermTupleHashIterator
  • CHECK_FOR_INTERRUPTS
  • ExecStoreMinimalTuple
  • execTuplesUnequal
  • TupleHashEntryGetTuple

関連データ構造

  • TupleHashTable: ハッシュテーブルの管理構造。
  • TupleTableSlot: タプルデータを保持するスロット。
  • FmgrInfo: 関数呼び出し情報。

失敗モードと境界条件

  • ハッシュテーブルが空の場合、ループは実行されず false を返します。
  • 比較関数 eqfunctions が予期しないエラーを発生させた場合、呼び出し元に伝播します。

保守担当者向けメモ

  • 本関数は、ハッシュテーブル内のエントリが特定の条件（等価性）を満たさないことを確認するために使用されます。
  • TermTupleHashIterator は、一致が見つかった場合にのみ呼び出され、ループを正常終了した場合には呼び出されない点に注意してください。
