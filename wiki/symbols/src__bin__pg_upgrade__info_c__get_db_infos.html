get_db_infos

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/info.c:403

目的

指定された PostgreSQL クラスタからデータベースのメタデータ（OID、名前、テーブルスペースの場所）を収集し、ClusterInfo 構造体に格納する。

シグネチャまたは形状

static void get_db_infos(ClusterInfo *cluster)

入力と出力

  • **入力**: ClusterInfo *cluster（対象クラスタの情報および結果格納先）
  • **出力**: なし（副作用として cluster->dbarr にデータベース情報の配列と件数が設定される）

前提条件と不変条件

  • cluster 引数は有効な ClusterInfo 構造体を指している必要がある。
  • connectToServer 関数が正常に接続を確立できること。
  • 接続先のクラスタにおいて pg_catalog.pg_database および pg_catalog.pg_tablespace へのアクセス権限が必要。

アルゴリズムまたは振る舞い

1. template1 データベースへ接続する。 2. PostgreSQL のバージョンに応じて datlocprovider や datlocale を含むクエリを動的に構築する。 3. pg_database を pg_tablespace と結合し、接続可能なデータベースを OID 順に取得する。 4. 取得した結果セットを走査し、DbInfo 構造体の配列を動的に割り当てて情報をコピーする。 5. 接続を終了し、結果セットを解放する。 6. 最終的に cluster->dbarr にデータベース情報の配列と件数を格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/info.c 内の get_db_rel_and_slot_infos（290行目）から呼び出される。
  • **呼び出し先**: connectToServer, executeQueryOrDie, PQfnumber, PQntuples, pg_malloc0, atooid, pg_strdup, PQgetvalue, PQclear, PQfinish。

関連データ構造

  • ClusterInfo: クラスタ全体の情報を保持する構造体。
  • DbInfo: 個別のデータベース情報を保持する構造体。

失敗モードと境界条件

  • executeQueryOrDie が失敗した場合、プログラムは終了する。
  • データベースが存在しない、または接続不可の場合は ntups が 0 となり、空の配列が設定される。
  • メモリ不足時は pg_malloc0 によりエラーが発生する可能性がある。

保守担当者向けメモ

  • クエリの構築において、PostgreSQL のメジャーバージョン（1500, 1700）に基づいた条件分岐が含まれている。バージョンアップに伴うシステムカタログの変更時はこのクエリの調整が必要。
  • pg_tablespace_location を使用してテーブルスペースのパスを取得しているため、テーブルスペースの構成変更時には注意が必要。
