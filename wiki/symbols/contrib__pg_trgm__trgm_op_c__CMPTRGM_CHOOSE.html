CMPTRGM_CHOOSE

定義

  • 種別: function
  • 定義箇所: contrib/pg_trgm/trgm_op.c:218

目的

システム環境の文字符号（signed/unsigned）に基づき、トリグラム比較関数 CMPTRGM を動的に初期化および切り替えるためのラッパー関数です。

シグネチャまたは形状

static int CMPTRGM_CHOOSE(const void *a, const void *b)

入力と出力

  • **入力**: 比較対象となる2つのメモリ領域へのポインタ a および b。
  • **出力**: 比較結果（a < b なら負、a == b なら 0、a > b なら正の整数）。

前提条件と不変条件

  • CMPTRGM は関数ポインタとして定義されており、本関数が初回呼び出しされることで、環境に応じた適切な比較関数（CMPTRGM_SIGNED または CMPTRGM_UNSIGNED）が代入されます。
  • GetDefaultCharSignedness() の結果が実行中に変化しない限り、以降の比較は最適化された関数ポインタ経由で行われます。

アルゴリズムまたは振る舞い

1. GetDefaultCharSignedness() を呼び出し、システムのデフォルト文字符号を確認します。 2. 符号付きであれば CMPTRGM に CMPTRGM_SIGNED を、符号なしであれば CMPTRGM_UNSIGNED を代入します。 3. 代入された関数ポインタを使用して、引数 a と b を比較し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_trgm/trgm_op.c 内の初期化処理（CMPTRGM の初期値として設定）。
  • **呼び出し先**: GetDefaultCharSignedness()、および CMPTRGM_SIGNED または CMPTRGM_UNSIGNED。

関連データ構造

  • CMPTRGM: 比較ロジックを保持するグローバルな関数ポインタ。

失敗モードと境界条件

  • 引数 a または b が NULL の場合の挙動は、代入先の比較関数（CMPTRGM_SIGNED / CMPTRGM_UNSIGNED）の仕様に依存します。

保守担当者向けメモ

  • 本関数は CMPTRGM ポインタの初回実行時にのみ実行される初期化ロジックを含みます。
  • 文字列比較の挙動を修正・調査する場合は、本関数ではなく CMPTRGM_SIGNED または CMPTRGM_UNSIGNED を確認してください。
