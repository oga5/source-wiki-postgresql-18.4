EA_flatten_into

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/array_expanded.c:292

目的

展開済み配列（ExpandedArrayHeader）を、PostgreSQLの標準的なフラットな配列形式（ArrayType）へシリアライズ（変換）します。

シグネチャまたは形状

static void EA_flatten_into(ExpandedObjectHeader *eohptr, void *result, Size allocated_size)

入力と出力

  • **入力**:
  • eohptr: 展開済み配列のヘッダーへのポインタ。
  • result: シリアライズ結果を格納するメモリ領域へのポインタ。
  • allocated_size: result に割り当てられたサイズ（バイト単位）。
  • **出力**: なし（result が指すメモリ領域にフラットな配列データが書き込まれます）。

前提条件と不変条件

  • eohptr が指す構造体は EA_MAGIC で検証可能であること。
  • allocated_size は、事前に EA_get_flat_size で算出されたサイズと一致していること。
  • result は allocated_size 分のメモリが確保されていること。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: eah->fvalue（既にフラット化された値）が存在する場合、それを result にコピーして終了します。 2. **初期化**: result 領域をゼロクリアし、ArrayType のヘッダー情報（次元数、要素型、次元配列、下限値配列）をコピーします。 3. **NULLビットマップ設定**: NULL値が含まれる場合は dataoffset を計算し、NULLビットマップ用の領域を確保します。 4. **データ構築**: CopyArrayEls を使用して、展開済み配列内の dvalues（値）および dnulls（NULLフラグ）から、フラットな配列のデータ領域へ要素を書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は明示されていませんが、ExpandedObjectMethods の一部として、展開済みオブジェクトを扱うフレームワークから呼び出されます。
  • **呼び出し先**: memcpy, memset, SET_VARSIZE, ARR_SIZE, ARR_OVERHEAD_WITHNULLS, ARR_DIMS, ARR_LBOUND, CopyArrayEls などのマクロおよび関数。

関連データ構造

  • ExpandedArrayHeader: 展開済み配列の内部表現。
  • ArrayType: PostgreSQLの標準的なフラットな配列構造体。
  • ExpandedObjectHeader: 展開済みオブジェクトの基底ヘッダー。

失敗モードと境界条件

  • Assert により、マジックナンバーの不一致や、割り当てサイズと期待されるサイズの不一致が検出された場合に異常終了します。

保守担当者向けメモ

  • 本関数は ExpandedObjectMethods の実装の一部です。配列の要素型や次元数に変更を加える際は、CopyArrayEls に渡す引数の整合性に注意してください。
  • eah->fvalue が存在する場合の高速パスと、dvalues/dnulls から再構築するパスの2系統がある点に留意してください。
