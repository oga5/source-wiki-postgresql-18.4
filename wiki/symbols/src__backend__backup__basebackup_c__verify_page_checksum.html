verify_page_checksum

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup.c:1993

目的

バックアップ対象のデータページが、バックアップ開始時点から変更されていないことを前提に、ページチェックサムの整合性を検証する。

シグネチャまたは形状

static bool verify_page_checksum(Page page, XLogRecPtr start_lsn, BlockNumber blkno, uint16 *expected_checksum)

入力と出力

  • **入力**:
  • page: 検証対象のページデータ。
  • start_lsn: バックアップ開始時の LSN。
  • blkno: ページ番号。
  • expected_checksum: チェックサム不一致時に期待される値を格納するポインタ。
  • **出力**:
  • bool: チェックサムが一致した場合は true、不一致または検証スキップの場合は false（ただし、検証対象外の場合は常に true を返す）。

前提条件と不変条件

  • PageIsNew(page) であるか、ページ内の LSN が start_lsn 以上である場合、ページは検証対象外とみなされ、常に true を返す。
  • ページが検証対象である場合、pg_checksum_page を用いて計算された値と、ページヘッダー内の pd_checksum が一致する必要がある。

アルゴリズムまたは振る舞い

1. ページが新規作成されたもの、あるいはバックアップ開始後に変更されたものであるかを確認し、該当する場合は検証をスキップして true を返す。 2. pg_checksum_page を呼び出し、現在のページ内容からチェックサムを算出する。 3. 算出したチェックサムとページヘッダーの pd_checksum を比較する。 4. 一致すれば true を返す。 5. 不一致の場合、算出したチェックサムを expected_checksum に書き込み、false を返す。

関連データ構造

  • Page: ページデータ構造。
  • XLogRecPtr: WAL の位置情報。
  • PageHeader: ページヘッダー構造体（pd_checksum フィールドを参照）。

失敗モードと境界条件

  • **検証スキップ**: バックアップ開始後に更新されたページや、新規ページはチェックサム検証の対象外となる。
  • **チェックサム不一致**: ページヘッダーのチェックサムと計算値が異なる場合、false を返し、呼び出し元に期待値を通知する。

保守担当者向けメモ

  • 本関数は、バックアップ中のデータ整合性を担保するための重要なチェックポイントである。
  • ページがバックアップ開始後に更新されている場合、チェックサムが不完全である可能性があるため、検証をスキップするロジックが組み込まれている。この場合、WAL リプレイによって整合性が確保される前提となっている。
