ginFreeScanKeys

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginscan.c:238

目的

GINインデックススキャンに関連するメモリリソースおよびバッファを解放し、スキャン状態をリセットする。

シグネチャまたは形状

void ginFreeScanKeys(GinScanOpaque so)

入力と出力

  • **入力**: so (GinScanOpaque) - GINインデックススキャンの状態を保持する不透明な構造体ポインタ。
  • **出力**: なし（副作用として so が指す構造体の内容がクリアされる）。

前提条件と不変条件

  • so が有効な GinScanOpaque 構造体を指していること。
  • so->keyCtx が有効なメモリコンテキストであること。

アルゴリズムまたは振る舞い

1. so->keys が NULL の場合は即座に終了する。 2. so->entries 配列を走査し、各 GinScanEntry に対して以下の処理を行う：

  • buffer が InvalidBuffer でなければ ReleaseBuffer を呼び出す。
  • list が存在すれば pfree で解放する。
  • matchIterator が存在すれば tbm_end_private_iterate を呼び出す。
  • matchBitmap が存在すれば tbm_free を呼び出す。

    • 3. so->keyCtx メモリコンテキストを MemoryContextReset でリセットする。 4. so 構造体のメンバ（keys, nkeys, entries, totalentries）を初期値（NULL または 0）に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginget.c:1941
  • src/backend/access/gin/ginscan.c:498
  • src/backend/access/gin/ginscan.c:510
  • **呼び出し先**:
  • ReleaseBuffer
  • pfree
  • tbm_end_private_iterate
  • tbm_free
  • MemoryContextReset

関連データ構造

  • GinScanOpaque: GINスキャンの状態を保持する構造体。
  • GinScanEntry: インデックスエントリの検索状態を保持する構造体。

失敗モードと境界条件

  • so->keys が既に NULL の場合、何もせず正常終了する。
  • 各エントリのポインタやバッファが NULL または InvalidBuffer の場合、それらの解放処理はスキップされる。

保守担当者向けメモ

  • 本関数は ginrescan や ginendscan の過程で呼び出され、スキャンキーの生存期間を管理する keyCtx をリセットすることでメモリリークを防ぐ。
  • tbm_ 系関数を使用しているため、ビットマップ処理の変更時には本関数の解放ロジックも確認すること。
