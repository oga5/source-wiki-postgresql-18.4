SlabBlocklistIndex

定義

  • 種別: static inline function
  • 定義箇所: src/backend/utils/mmgr/slab.c:210

目的

SLAB アロケータにおいて、ブロック内の空きチャンク数（nfree）に基づき、そのブロックを管理すべき適切なリストインデックスを算出する。

シグネチャまたは形状

static inline int32 SlabBlocklistIndex(SlabContext *slab, int nfree)

入力と出力

  • **入力**:
  • slab: 対象の SlabContext 構造体へのポインタ。
  • nfree: 現在のブロックにおける空きチャンク数。
  • **出力**:
  • int32: 0 から SLAB_BLOCKLIST_COUNT - 1 の範囲のインデックス値。

前提条件と不変条件

  • nfree は 0 以上かつ slab->chunksPerBlock 以下であること。
  • nfree == 0 の場合、戻り値は必ず 0 となる。
  • nfree > 0 の場合、戻り値は 1 以上かつ SLAB_BLOCKLIST_COUNT 未満となる。

アルゴリズムまたは振る舞い

1. slab->blocklist_shift を用いて、空きチャンク数からインデックスを計算する。 2. 2の補数表現を利用したビット演算（-((-nfree) >> blocklist_shift)）により、0 を 0 に維持しつつ、それ以外の値を効率的にスケーリングする。 3. 計算結果に対してアサーションによる検証を行い、インデックスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/slab.c 内の各所（SlabAlloc、SlabFree 等のブロック管理ロジックにおいて、ブロックのリスト移動や状態確認のために頻繁に呼び出される）。
  • **呼び出し先**:
  • 特になし（純粋な計算関数）。

関連データ構造

  • SlabContext: blocklist_shift を保持する。
  • SLAB_BLOCKLIST_COUNT: インデックスの最大範囲を定義する定数。

失敗モードと境界条件

  • nfree が負の値、または chunksPerBlock を超える値である場合、Assert によりプログラムが停止する（デバッグビルド時）。
  • blocklist_shift が不適切な値である場合、計算結果が期待する範囲外となる可能性がある。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、static inline として定義されている。
  • ビット演算による最適化が行われているため、ロジックを変更する際は nfree == 0 の不変条件が維持されることを確認すること。
