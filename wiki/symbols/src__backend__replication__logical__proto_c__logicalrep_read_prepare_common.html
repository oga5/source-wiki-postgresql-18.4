logicalrep_read_prepare_common

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/proto.c:198

目的

論理レプリケーションにおける「準備済みトランザクション（Prepared Transaction）」関連メッセージの共通フィールドを、受信バッファからデシリアライズして構造体に格納します。

シグネチャまたは形状

static void logicalrep_read_prepare_common(StringInfo in, char *msgtype, LogicalRepPreparedTxnData *prepare_data)

入力と出力

  • **入力**:
  • in: 受信したメッセージデータを含む StringInfo バッファ。
  • msgtype: エラーログ出力時に使用するメッセージ種別名（文字列）。
  • prepare_data: デシリアライズ結果を格納する LogicalRepPreparedTxnData 構造体へのポインタ。
  • **出力**: なし（prepare_data を直接更新）。

前提条件と不変条件

  • in バッファには、プロトコル定義に従った順序でデータが格納されている必要があります。
  • prepare_data->gid は、コピー先として十分なサイズが確保されている必要があります。
  • prepare_lsn および end_lsn は InvalidXLogRecPtr であってはならず、xid は InvalidTransactionId であってはなりません。

アルゴリズムまたは振る舞い

1. **フラグの読み込み**: 1バイトのフラグを読み込みます。0以外の場合はエラーを発生させます。 2. **LSNの読み込み**: prepare_lsn と end_lsn を64ビット整数として読み込みます。無効な値の場合はエラーとなります。 3. **時刻とXIDの読み込み**: prepare_time（64ビット）と xid（32ビット）を読み込みます。xid が無効な場合はエラーとなります。 4. **GIDの読み込み**: 最後に、グローバル識別子（GID）を文字列として読み込み、prepare_data->gid にコピーします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • logicalrep_read_prepare (src/backend/replication/logical/proto.c:230)
  • logicalrep_read_stream_prepare (src/backend/replication/logical/proto.c:367 ※ソース内では "stream prepare" として呼び出し)
  • **呼び出し先**:
  • pq_getmsgbyte
  • pq_getmsgint64
  • pq_getmsgint
  • pq_getmsgstring
  • elog

関連データ構造

  • StringInfo: 通信バッファ。
  • LogicalRepPreparedTxnData: 準備済みトランザクションのメタデータを保持する構造体。

失敗モードと境界条件

  • **エラー発生**:
  • フラグが0以外の場合。
  • prepare_lsn または end_lsn が InvalidXLogRecPtr の場合。
  • xid が InvalidTransactionId の場合。
  • **境界条件**: StringInfo の読み込み位置がプロトコル定義と一致しない場合、以降のデータ読み込みが不正になります。

保守担当者向けメモ

  • 本関数は prepare 関連メッセージの共通処理を抽出したものです。プロトコル定義（logicalproto.h）に変更を加える際は、読み込み順序が logicalrep_write_prepare_common と完全に一致していることを確認してください。
