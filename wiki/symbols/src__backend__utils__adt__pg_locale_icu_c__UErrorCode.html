UErrorCode

定義

  • 種別: typedef
  • 定義箇所: src/backend/utils/adt/pg_locale_icu.c:69

目的

ICU（International Components for Unicode）ライブラリの関数実行時におけるエラー状態を保持および伝播するための型です。

シグネチャまたは形状

ICUライブラリで定義される列挙型または整数型（typedef）です。

入力と出力

  • **入力**: ICU関連関数の引数として、エラー状態を記録するためのポインタ（UErrorCode *）が渡されます。
  • **出力**: 関数実行後、成功時は U_ZERO_ERROR、失敗時は特定のエラーコードが格納されます。

前提条件と不変条件

  • ICU関数を呼び出す際、呼び出し元は初期値として U_ZERO_ERROR を設定した変数のアドレスを渡す必要があります。
  • エラーが発生した場合、PostgreSQL側では ereport(ERROR, ...) を用いてトランザクションを中断する設計となっています。

アルゴリズムまたは振る舞い

ICUライブラリの各関数は、処理の成否をこの型を通じて通知します。PostgreSQLのICUラッパー関数（pg_ucol_open 等）は、この値を監視し、エラーが検出された場合にPostgreSQLの例外メカニズムへ制御を移します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • src/backend/utils/adt/pg_locale_icu.c 内の各関数（pg_ucol_open 等）
  • src/backend/utils/adt/pg_locale.c
  • src/backend/commands/collationcmds.c
  • **呼び出し先**: ICUライブラリの各関数（ucol_open 等）。

関連データ構造

  • UCollator: ICUの照合順序オブジェクト。
  • UConverter: ICUのエンコーディング変換オブジェクト。

失敗モードと境界条件

  • U_ZERO_ERROR 以外の値が設定された場合、ICU関数は通常処理をスキップします。
  • ICUのバージョン差異により、特定のエラーコードの解釈が異なる場合があります。

保守担当者向けメモ

  • ICUの UErrorCode を監視し、エラー発生時は必ずPostgreSQLの ereport を介してトランザクションを安全に終了させてください。
  • ICU 53未満の環境では、一部の関数でエラーハンドリングが不安定になる可能性があるため、マクロによる条件分岐を確認してください。
