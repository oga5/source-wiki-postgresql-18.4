TableFuncNext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTableFuncscan.c:53

目的

TableFuncScan ノードにおいて、テーブル関数（XMLTABLE や JSON_TABLE 等）の実行結果である tuplestore から次のタプルを取得し、スキャンノードの出力スロットへ提供する。

シグネチャまたは形状

TupleTableSlot *TableFuncNext(TableFuncScanState *node)

入力と出力

  • **入力**: TableFuncScanState *node （テーブル関数スキャンの実行状態を保持する構造体）
  • **出力**: TupleTableSlot * （取得したタプルを格納したスキャンタプルスロット）

前提条件と不変条件

  • node->ss.ss_ScanTupleSlot が有効なスロットとして初期化されていること。
  • 初回呼び出し時に node->tupstore が NULL であれば、tfuncFetchRows を介してデータがロードされる。

アルゴリズムまたは振る舞い

1. node->ss.ss_ScanTupleSlot を取得する。 2. node->tupstore が NULL の場合、tfuncFetchRows を呼び出し、入力ドキュメントの解析および全行の tuplestore へのロードを実行する。 3. tuplestore_gettupleslot を呼び出し、tuplestore から次のタプルをスロットに読み込む。 4. 読み込んだ scanslot を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecScan インターフェースを介して実行エンジンから呼び出される（src/backend/executor/nodeTableFuncscan.c:102 に ExecScanAccessMtd として登録）。
  • **呼び出し先**:
  • tfuncFetchRows: 初回実行時に全行をロードするために呼び出す。
  • tuplestore_gettupleslot: tuplestore からタプルを取得するために呼び出す。

関連データ構造

  • TableFuncScanState: 本関数の引数であり、実行状態や tuplestore を保持する。
  • TupleTableSlot: タプルを保持する実行時のデータ構造。

失敗モードと境界条件

  • tuplestore が空の場合、tuplestore_gettupleslot の挙動に従い、スロットは空（または終了状態）として返される。
  • 入力ドキュメントが巨大な場合、tfuncFetchRows による全行ロード時にメモリ消費が増大する可能性がある。

保守担当者向けメモ

  • 本関数は「マテリアライズ型」のスキャンを実装している。初回実行時に全行をメモリ上の tuplestore にロードするため、パフォーマンス特性は初回呼び出しに偏る。
  • tfuncFetchRows が node->tupstore を初期化する役割を担っているため、この依存関係を変更する際は注意が必要。
