GetAllTablesPublicationRelations

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_publication.c:863

目的

システム内の全リレーションを走査し、パブリケーションの対象として適格なリレーションのOIDリストを取得します。

シグネチャまたは形状

List *GetAllTablesPublicationRelations(bool pubviaroot)

入力と出力

  • **入力**:
  • pubviaroot (bool): パーティションテーブルをルートとして扱うか否かを指定するフラグ。
  • **出力**:
  • List *: パブリケーション対象となるリレーションのOIDリスト。

前提条件と不変条件

  • pg_class システムカタログへのアクセス権限が必要です。
  • AccessShareLock を取得してカタログを読み取ります。

アルゴリズムまたは振る舞い

1. pg_class を RELKIND_RELATION でスキャンし、is_publishable_class を満たすリレーションを収集します。 2. pubviaroot が true の場合、パーティションテーブル（RELKIND_PARTITIONED_TABLE）もスキャン対象に加え、relispartition が false である（つまりルートテーブルである）ものをリストに追加します。 3. pubviaroot が true の場合、最初のスキャンで relispartition が true であるリレーションは除外されます。 4. 最終的に収集されたOIDのリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_publication.c:777
  • src/backend/catalog/pg_publication.c:1163
  • **呼び出し先**:
  • table_open, table_beginscan_catalog, heap_getnext, table_endscan, table_close

関連データ構造

  • List: OIDを保持するリスト構造。
  • pg_class: スキャン対象のシステムカタログ。

失敗モードと境界条件

  • カタログスキャン中にメモリ不足が発生した場合、PostgreSQLの標準的なエラーハンドリングによりアボートします。
  • is_publishable_class の判定基準により、システムテーブルや一時テーブルは自動的に除外されます。

保守担当者向けメモ

  • パーティション階層の扱い（pubviaroot）がロジックの分岐点です。将来的に新しいリレーションタイプが追加された場合、スキャン条件の更新が必要になる可能性があります。
