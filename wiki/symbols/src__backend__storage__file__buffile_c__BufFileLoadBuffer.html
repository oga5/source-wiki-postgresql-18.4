BufFileLoadBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/buffile.c:433

目的

BufFile 構造体が管理するバッファへ、物理ファイルからデータを読み込みます。

シグネチャまたは形状

static void BufFileLoadBuffer(BufFile *file)

入力と出力

  • **入力**: BufFile *file （読み込み対象のバッファ付きファイル構造体）
  • **出力**: なし（副作用として file->buffer.data にデータが格納され、file->nbytes が更新されます）

前提条件と不変条件

  • file->curOffset が MAX_PHYSICAL_FILESIZE に達している場合、自動的に次の物理ファイルセグメントへ切り替わります。
  • track_io_timing が有効な場合、I/O 時間が計測され pgBufferUsage に累積されます。

アルゴリズムまたは振る舞い

1. 現在のオフセットが物理ファイルの最大サイズを超えており、かつ次のセグメントが存在する場合、curFile をインクリメントし curOffset を 0 にリセットします。 2. FileRead を呼び出し、物理ファイルからバッファサイズ分を読み込みます。 3. 読み込みに失敗した場合（nbytes < 0）、エラーを報告します。 4. 読み込み成功時、pgBufferUsage.temp_blks_read をインクリメントします。 5. 読み込み後の curOffset は本関数内では進められません。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/file/buffile.c 内の読み込み処理（例: BufFileReadCommon 等）。
  • **呼び出し先**: FileRead（低レベルファイル読み込み）、ereport（エラー報告）、INSTR_TIME_* マクロ（計測）。

関連データ構造

  • BufFile: バッファ状態、ファイルセグメント配列、現在のオフセットを保持する構造体。
  • pgBufferUsage: I/O 統計情報を保持するグローバル変数。

失敗モードと境界条件

  • 物理ファイルからの読み込みが失敗した場合、ereport(ERROR, ...) によりトランザクションが中断されます。
  • 最終セグメントの末尾に達した場合、FileRead は読み込めたバイト数（0を含む）を返します。

保守担当者向けメモ

  • 本関数はバッファのロードのみを行い、論理的なファイルポインタ（curOffset）の更新は呼び出し側で行う設計になっています。
  • MAX_PHYSICAL_FILESIZE（1GB）ごとのファイル分割境界での挙動に注意してください。
