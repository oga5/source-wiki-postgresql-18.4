ModifyTable

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/plannodes.h:281

目的

テーブルに対するデータ変更操作（INSERT, UPDATE, DELETE, MERGE）を実行するためのプランノードを定義します。

シグネチャまたは形状

Plan 構造体を先頭メンバとして保持し、操作対象のリレーション情報、RETURNING 句の処理、ON CONFLICT や MERGE アクションに関連するメタデータを格納します。

入力と出力

  • **入力**: 子プランノードから供給されるタプル。
  • **出力**: 操作対象テーブルへの変更適用、および RETURNING 句が指定されている場合は処理後のタプル。

前提条件と不変条件

  • Plan 構造体を継承しており、プランノードとしてキャスト可能です。
  • 複数のターゲットテーブルを扱う場合、resultRelations 等のリストは対応する順序で保持されます。

アルゴリズムまたは振る舞い

1. operation フィールドに基づき、INSERT, UPDATE, DELETE, MERGE のいずれかの操作を決定します。 2. canSetTag が真の場合、コマンドタグや処理行数（es_processed）を更新します。 3. パーティションや継承テーブルが関与する場合、rootRelation を使用して階層を特定します。 4. ON CONFLICT や MERGE が指定されている場合、関連する条件式やターゲットリストを用いて競合解決や条件付きアクションを実行します。 5. RETURNING 句がある場合、指定されたエイリアス（returningOldAlias, returningNewAlias）を用いて結果を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: プランナ（src/backend/optimizer/）が生成し、エグゼキュータ（src/backend/executor/）が消費します。
  • **呼び出し先**: contrib/postgres_fdw/postgres_fdw.c 等の外部データラッパーが、FDW 固有の処理のために本構造体を参照・操作する事例が確認されています。

関連データ構造

  • Plan: 基底構造体。
  • CmdType: 操作種別（INSERT/UPDATE/DELETE/MERGE）。
  • List: ターゲットリレーション、RETURNING リスト、FDW 関連データ等の保持に使用。
  • Bitmapset: FDW 関連の直接変更プランのインデックス管理に使用。

失敗モードと境界条件

  • 外部データラッパー（FDW）を使用する場合、fdwPrivLists や fdwDirectModifyPlans が適切に設定されていないと実行時エラーの原因となります。
  • epqParam が正しく設定されていない場合、EvalPlanQual（再評価）が必要な状況で正しく動作しません。

保守担当者向けメモ

  • 本構造体は Plan を継承しているため、メンバ追加時はバイナリ互換性に注意してください。
  • outerPlan / innerPlan マクロの使用が推奨されますが、本構造体は複数のターゲットテーブルを扱うため、リスト形式のメンバ（resultRelations 等）との整合性に留意してください。
