check_ok

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/util.c:283

目的

処理が正常に完了したことを示すメッセージをログおよび標準出力に報告する。

シグネチャまたは形状

void check_ok(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてログ出力を行う）

前提条件と不変条件

  • 呼び出し時点で、先行する処理が正常に終了していることが期待される。

アルゴリズムまたは振る舞い

1. report_status 関数を呼び出し、PG_REPORT レベルで文字列 "ok" を出力する。

呼び出し元と呼び出し先

  • 呼び出し先: report_status
  • 呼び出し元: 提供されたソース情報には src/bin/pg_upgrade/util.c 内の呼び出し元は記載されていません。
  • 注: ツールにより src/bin/initdb/initdb.c からの参照が検出されていますが、これは別プロジェクトの同名関数である可能性が高く、本モジュール内での直接的な呼び出し関係は確認されていません。

関連データ構造

  • なし

失敗モードと境界条件

  • report_status の内部実装に依存する。

保守担当者向けメモ

  • 本関数は pg_upgrade における進捗報告の一環として使用される。
  • src/bin/initdb/initdb.c にも同名の関数が存在するため、コード検索時にはディレクトリパスに注意すること。
