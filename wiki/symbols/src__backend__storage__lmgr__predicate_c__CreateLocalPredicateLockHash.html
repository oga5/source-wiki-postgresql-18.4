CreateLocalPredicateLockHash

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/predicate.c:1939

目的

バックエンドローカルなハッシュテーブル LocalPredicateLockHash を初期化し、現在のトランザクションが保持する述語ロック（親ロック）を追跡するためのメモリ領域を確保します。

シグネチャまたは形状

static void CreateLocalPredicateLockHash(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として LocalPredicateLockHash グローバル変数を初期化）

前提条件と不変条件

  • LocalPredicateLockHash が NULL であること（初期化済みでないこと）を Assert で検証します。

アルゴリズムまたは振る舞い

1. HASHCTL 構造体を設定します。

  • キーサイズ: PREDICATELOCKTARGETTAG
  • エントリサイズ: LOCALPREDICATELOCK

    • 2. hash_create を呼び出し、max_predicate_locks_per_xact をサイズ上限としてハッシュテーブルを生成します。 3. 生成したハッシュテーブルを LocalPredicateLockHash に代入します。フラグとして HASH_ELEM および HASH_BLOBS を使用します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/lmgr/predicate.c 内の複数箇所から参照されています。
  • 呼び出し先: hash_create（PostgreSQL のハッシュテーブル管理関数）。

関連データ構造

  • LocalPredicateLockHash: バックエンドローカルなハッシュテーブル。
  • PREDICATELOCKTARGETTAG: ハッシュテーブルのキー。
  • LOCALPREDICATELOCK: ハッシュテーブルのエントリ。
  • max_predicate_locks_per_xact: ハッシュテーブルのサイズを決定する GUC パラメータ。

失敗モードと境界条件

  • LocalPredicateLockHash が既に初期化されている場合、Assert により異常終了します。
  • メモリ不足時には hash_create がエラーを発生させます。

保守担当者向けメモ

  • 本関数はバックエンドプロセスごとのローカルメモリを管理するものであり、共有メモリ上の PredicateLockHash とは異なります。
  • 述語ロックのメモリ制限（max_predicate_locks_per_xact）に達した場合の挙動は、このハッシュテーブルの管理方針に依存します。
