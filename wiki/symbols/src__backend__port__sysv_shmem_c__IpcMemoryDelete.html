IpcMemoryDelete

定義

  • 種別: function
  • 定義箇所: src/backend/port/sysv_shmem.c:297

目的

System V 共有メモリセグメントをシステムから削除（マーク）します。

シグネチャまたは形状

static void IpcMemoryDelete(int status, Datum shmId)

入力と出力

  • **入力**:
  • status: on_shmem_exit コールバック用のステータスコード（未使用）。
  • shmId: 削除対象の共有メモリセグメント ID（Datum 型）。
  • **出力**: なし。

前提条件と不変条件

  • shmId は有効な System V 共有メモリセグメント ID である必要があります。

アルゴリズムまたは振る舞い

1. shmctl システムコールを IPC_RMID コマンドで呼び出し、指定された共有メモリセグメントの削除を要求します。 2. shmctl が負の値を返した場合、elog を使用して LOG レベルでエラーメッセージを出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/port/sysv_shmem.c 内の on_shmem_exit に登録されるコールバックとして呼び出されます（行252）。
  • **呼び出し先**: shmctl（システムコール）、elog。

関連データ構造

  • Datum: PostgreSQL の汎用データ型。
  • shmid_ds（shmctl 経由で間接的に関連）。

失敗モードと境界条件

  • shmctl が失敗した場合（例: ID が無効、権限不足）、elog によりログ出力が行われますが、プロセスは終了しません。

保守担当者向けメモ

  • 本関数は on_shmem_exit を介してプロセス終了時に実行されることを想定しています。
  • status 引数は on_shmem_exit のシグネチャに合わせるために存在しますが、本関数内では利用されていません。
