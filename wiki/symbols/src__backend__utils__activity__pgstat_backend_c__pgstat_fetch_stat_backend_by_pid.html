pgstat_fetch_stat_backend_by_pid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_backend.c:110

目的

指定されたプロセスID（PID）に対応するバックエンドプロセスの統計情報を取得します。

シグネチャまたは形状

PgStat_Backend *pgstat_fetch_stat_backend_by_pid(int pid, BackendType *bktype)

入力と出力

  • **入力**:
  • pid: 検索対象のプロセスID。
  • bktype: バックエンド種別を格納するためのポインタ（NULL可）。
  • **出力**:
  • 成功時: 対象プロセスの PgStat_Backend 構造体へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • 指定されたPIDが有効なバックエンドまたは補助プロセス（Auxiliary process）に関連付けられている必要があります。
  • 統計収集対象外のバックエンド種別である場合、取得は失敗します。

アルゴリズムまたは振る舞い

1. BackendPidGetProc および AuxiliaryPidGetProc を用いて、PIDから PGPROC を特定します。 2. GetNumberFromPGProc で取得した ProcNumber を使用し、pgstat_get_beentry_by_proc_number でバックエンド状態エントリを取得します。 3. pgstat_tracks_backend_bktype を呼び出し、当該バックエンドが統計収集対象かを確認します。 4. PIDの一致を確認し、一致すれば bktype 引数にバックエンド種別を書き込みます。 5. pgstat_fetch_stat_backend を呼び出し、統計情報を取得して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pgstatfuncs.c (SQL関数経由での統計取得)
  • **呼び出し先**:
  • BackendPidGetProc, AuxiliaryPidGetProc, GetNumberFromPGProc
  • pgstat_get_beentry_by_proc_number
  • pgstat_tracks_backend_bktype
  • pgstat_fetch_stat_backend

関連データ構造

  • PgStat_Backend: 取得される統計情報の構造体。
  • PGPROC: プロセス管理用構造体。
  • PgBackendStatus: バックエンドの実行状態を保持する構造体。

失敗モードと境界条件

  • PIDに対応するプロセスが存在しない場合、NULL を返します。
  • 統計収集対象外のバックエンド種別の場合、NULL を返します。
  • pgstat_fetch_stat_backend が統計情報を取得できない場合、NULL を返します。
  • beentry は stats_fetch_consistency の設定により解放される可能性があるため、取得後のアクセスは禁止されています。

保守担当者向けメモ

  • 本関数は pgstat_fetch_stat_backend を内部で呼び出しており、その戻り値の有効性は呼び出し側の統計整合性設定に依存します。
  • 補助プロセスも対象に含まれるため、PID検索時には通常プロセスと補助プロセスの両方を考慮する必要があります。
