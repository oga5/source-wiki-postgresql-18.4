inet_gist_fetch

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_gist.c:589

目的

GiSTインデックスに格納された圧縮形式の GistInetKey を、元の inet 型データに復元（フェッチ）します。

シグネチャまたは形状

Datum inet_gist_fetch(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体を受け取ります。
  • **出力**: 復元された inet 型データを保持する GISTENTRY 構造体へのポインタを Datum として返します。

前提条件と不変条件

  • 入力された entry->key は、有効な GistInetKey 構造体として解釈可能である必要があります。
  • 復元された inet 型は、palloc0 によってメモリ確保され、SET_INET_VARSIZE マクロによって適切な可変長サイズが設定されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) で GISTENTRY を取得し、DatumGetInetKeyP を用いてインデックスキーを GistInetKey 構造体に変換します。 2. 新しい inet 型のメモリ領域を確保します。 3. GistInetKey からアドレスファミリー、最小ネットマスク長（minbits）、およびアドレス本体をコピーして inet 構造体に格納します。 4. SET_INET_VARSIZE を呼び出し、inet 型のサイズを確定させます。 5. gistentryinit を使用して、復元した inet 値を含む新しい GISTENTRY を初期化し、返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に明示的な呼び出し元は記載されていません。GiSTインデックスのフェッチ処理として、PostgreSQLのインデックスアクセス層から間接的に呼び出されることが想定されます。
  • **呼び出し先**: palloc0, palloc, gistentryinit, DatumGetInetKeyP, ip_family, ip_bits, ip_addr, ip_addrsize, SET_INET_VARSIZE, InetPGetDatum を呼び出します。

関連データ構造

  • GistInetKey: GiSTインデックス内でネットワーク情報を保持する構造体。
  • inet: PostgreSQLのネットワーク型。
  • GISTENTRY: GiSTインデックスの各エントリを表現する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • 入力された entry->key が不正な形式である場合、DatumGetInetKeyP による変換時に予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は、GiSTインデックスの「フェッチ」サポート関数として機能します。インデックス内の圧縮されたキーを元の型に戻す必要がある場合にのみ使用されます。
  • GistInetKey の内部構造が変更された場合、本関数内のコピーロジックも追従して更新する必要があります。
