printtup_startup

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/printtup.c:110

目的

DestReceiver を使用したクエリ結果出力処理の初期化を行います。タプル送信用のバッファ確保と、行処理ごとのメモリ管理用コンテキストの作成を担います。

シグネチャまたは形状

static void printtup_startup(DestReceiver *self, int operation, TupleDesc typeinfo)

入力と出力

  • **入力**:
  • self: DR_printtup 構造体へのポインタ（DestReceiver としてキャスト）。
  • operation: 実行操作の種類（未使用）。
  • typeinfo: 出力対象タプルの記述子。
  • **出力**: なし（副作用として self 内のバッファおよびメモリコンテキストを初期化）。

前提条件と不変条件

  • self は DR_printtup 構造体として正しく初期化されている必要があります。
  • self->portal が有効なポータルを参照している必要があります。

アルゴリズムまたは振る舞い

1. initStringInfo を呼び出し、メッセージ送信用バッファ（myState->buf）を初期化します。このバッファは行を跨いで再利用されます。 2. AllocSetContextCreate を使用し、行ごとのメモリ解放を効率化するための専用メモリコンテキスト（tmpcontext）を作成します。 3. myState->sendDescrip が真の場合、SendRowDescriptionMessage を呼び出し、カラム定義情報をクライアントへ送信します。 4. 属性情報のキャッシュ（PrinttupAttrInfo）の構築は、最初のタプル処理時まで遅延されます。

呼び出し元と呼び出し先

  • **呼び出し元**: printtup_create_DR 内で DestReceiver の rStartup フィールドに登録され、実行エンジンから呼び出されます。
  • **呼び出し先**:
  • initStringInfo
  • AllocSetContextCreate
  • SendRowDescriptionMessage
  • FetchPortalTargetList

関連データ構造

  • DR_printtup: 本関数が状態を更新する対象の構造体。
  • DestReceiver: 本関数が実装するインターフェース。
  • TupleDesc: タプルの構造情報。

失敗モードと境界条件

  • メモリ不足時には AllocSetContextCreate がエラーを発生させます。
  • portal が無効な場合、SendRowDescriptionMessage 実行時に異常終了する可能性があります。

保守担当者向けメモ

  • 本関数は rStartup として登録されます。
  • 属性情報のキャッシュ構築を遅延させているのは、タプルが存在しない場合のオーバーヘッド回避と、実行中にタプル型が変更される可能性への対応のためです。
  • tmpcontext は行ごとのリセットを前提としているため、このコンテキスト内に永続的なデータを保持してはいけません。
