find_active_timeout

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/timeout.c:95

目的

現在有効なタイムアウトのリスト（active_timeouts）から、指定された TimeoutId に一致するタイムアウトのインデックスを検索します。

シグネチャまたは形状

static int find_active_timeout(TimeoutId id)

入力と出力

  • **入力**: id (TimeoutId) - 検索対象のタイムアウト識別子。
  • **出力**: int - 見つかった場合は active_timeouts 配列内のインデックス（0以上）、見つからない場合は -1 を返します。

前提条件と不変条件

  • active_timeouts 配列は、現在有効なタイムアウトを保持している必要があります。
  • num_active_timeouts は、active_timeouts 配列内の有効な要素数を正しく反映している必要があります。

アルゴリズムまたは振る舞い

1. num_active_timeouts を上限として、active_timeouts 配列を先頭から順に走査します。 2. 各要素の index フィールドが引数の id と一致するか比較します。 3. 一致した時点でそのインデックスを即座に返します。 4. ループ終了まで一致するものが見つからなかった場合、-1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • remove_timeout_index を呼び出す箇所（src/backend/utils/misc/timeout.c 内の複数箇所）から、削除対象のインデックスを特定するために呼び出されます。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • TimeoutId: タイムアウトの種類を識別する列挙型。
  • active_timeouts: 現在有効なタイムアウトを保持する静的配列。

失敗モードと境界条件

  • 指定された id が現在アクティブでない場合、-1 を返します。
  • active_timeouts が空の場合、ループは実行されず -1 を返します。

保守担当者向けメモ

  • 本関数は線形探索を行うため、active_timeouts の要素数が増加した場合には性能への影響を考慮する必要があります。
  • 呼び出し元で戻り値が -1 である可能性を考慮したエラーハンドリングが行われているか確認してください。
