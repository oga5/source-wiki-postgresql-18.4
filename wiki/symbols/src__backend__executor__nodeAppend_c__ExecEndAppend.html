ExecEndAppend

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAppend.c:400

目的

Append ノードの実行を終了し、関連するすべてのサブプラン（子ノード）をクリーンアップする。

シグネチャまたは形状

void ExecEndAppend(AppendState *node)

入力と出力

  • **入力**: AppendState *node - 終了対象となる Append ノードの実行時状態構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • node は有効な AppendState 構造体を指している必要がある。
  • node->appendplans に格納された各サブプランの状態は、ExecEndNode を呼び出すことで安全に解放可能であること。

アルゴリズムまたは振る舞い

1. node からサブプランの配列 (appendplans) とサブプランの総数 (nplans) を取得する。 2. 0 から nplans - 1 までのインデックスでループを実行する。 3. 各反復において、appendplans[i] を引数として ExecEndNode を呼び出し、各サブプランの終了処理を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:601 において、実行ツリーの終了処理の一部として呼び出される。
  • **呼び出し先**: ExecEndNode（各サブプランの終了処理を実行）。

関連データ構造

  • AppendState: Append ノードの実行時状態を保持する構造体。appendplans（サブプランの配列）および as_nplans（サブプラン数）を含む。

失敗モードと境界条件

  • nplans が 0 の場合、ループは実行されず、何も行わずに正常終了する。
  • appendplans 内の各要素が NULL でないことが前提となる。

保守担当者向けメモ

  • 本関数は Append ノードの終了処理のみを担う。並列実行に関連する共有メモリの解放や非同期実行のクリーンアップが必要な場合は、別途 ExecEndAppend の前後または内部で適切な処理が呼び出されているか確認すること。
