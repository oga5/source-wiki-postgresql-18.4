定義

目的

Informix 互換ライブラリにおける日付フォーマット変換機能（rfmtdate, rdefmtdate, rstrdate）の動作を検証する回帰テストの実行。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（引数なし）
  • 出力: 正常終了時に 0 を返却。標準出力へテスト結果および変換された日付文字列を出力。

前提条件と不変条件

  • ECPGdebug(1, stderr) により、デバッグ情報が標準エラーに出力される設定で実行される。
  • rmdyjul 関数が正常に動作し、基準となる日付が生成されることを前提とする。

アルゴリズムまたは振る舞い

1. **初期化**: rmdyjul を用いて基準日付を生成し、rdatestr で文字列変換して結果を表示する。 2. **文字列変換テスト**: date_test_strdate を呼び出し、特定の文字列形式から日付への変換を検証する。 3. **フォーマット変換テスト**: date_test_fmt を呼び出し、内部日付を多様なフォーマット文字列（mmddyy, yy/mm/dd 等）で変換し、結果を表示する。 4. **逆フォーマット変換テスト**: date_test_defmt を呼び出し、複雑な入力文字列とフォーマット指定の組み合わせを検証する。これには異常系（不正な日付、形式）も含まれ、エラーハンドリングの確認を行う。 5. **終了**: すべてのテストケース実行後、0 を返して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先: ECPGdebug, rmdyjul, rdatestr, printf, date_test_strdate, date_test_fmt, date_test_defmt
  • 注記: ツールによる静的解析では、本シンボルを呼び出している他のソースコード内の関数は特定されていません。

関連データ構造

  • date: 日付の内部表現型。
  • short mdy[3]: 月・日・年を保持する配列。
  • char dbuf[11]: 日付文字列を格納するバッファ。

失敗モードと境界条件

  • rdefmtdate 関連のテストにおいて、以下のエラーケースを網羅するよう設計されている:
  • ECPG_INFORMIX_ENOTDMY: 日付構成要素（dd, mm, yy）の不足。
  • ECPG_INFORMIX_ENOSHORTDATE: 不正な長さの入力。
  • ECPG_INFORMIX_BAD_DAY: 不正な日。
  • ECPG_INFORMIX_BAD_MONTH: 不正な月。

保守担当者向けメモ

  • 本関数はビジネスロジックではなく、ECPG プリプロセッサの回帰テスト用コードです。
  • date_test_defmt に渡されるフォーマット文字列には、パーサーの堅牢性を確認するための意図的な冗長文字が含まれています。
  • 修正や機能追加を行う際は、src/interfaces/ecpg/pgtypeslib/ にある変換ロジックの実装と整合性を確認してください。
