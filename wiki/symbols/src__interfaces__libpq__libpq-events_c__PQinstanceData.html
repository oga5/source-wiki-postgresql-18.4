PQinstanceData

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/libpq-events.c:120

目的

PGconn オブジェクトに関連付けられた、特定のイベントプロシージャに対応するインスタンスデータを取得します。

シグネチャまたは形状

void *PQinstanceData(const PGconn *conn, PGEventProc proc);

入力と出力

  • **入力**:
  • conn: データを取得する対象の PGconn オブジェクトへのポインタ。
  • proc: データを識別するためのイベントプロシージャ（関数ポインタ）。
  • **出力**:
  • 成功時: 登録されているインスタンスデータへの void * ポインタ。
  • 失敗時: NULL（conn または proc が NULL の場合、あるいは該当するプロシージャが登録されていない場合）。

前提条件と不変条件

  • proc は、事前に PQregisterEventProc を通じて conn に登録されている必要があります。
  • イベントの識別には関数ポインタのアドレスが使用されるため、同一のプロシージャを複数回登録することはできません。

アルゴリズムまたは振る舞い

1. conn または proc が NULL であれば直ちに NULL を返します。 2. conn->events 配列を走査し、登録されている各イベントエントリの proc フィールドを引数の proc と比較します。 3. 一致するエントリが見つかった場合、そのエントリに保持されている data ポインタを返します。 4. 配列の最後まで一致するものが見つからなかった場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内では、特定の呼び出し元は明示されていませんが、libpq のイベント API を利用する外部ライブラリやアプリケーションから呼び出されることが想定されています。
  • **呼び出し先**: 内部的な関数呼び出しは行いません。

関連データ構造

  • PGconn: 接続情報を保持する構造体。nEvents（イベント数）と events（イベント配列）を保持します。
  • PGEventProc: イベント処理関数の型定義。

失敗モードと境界条件

  • conn が NULL の場合: NULL を返します。
  • proc が NULL の場合: NULL を返します。
  • 登録されていない proc を指定した場合: NULL を返します。

保守担当者向けメモ

  • 本関数は検索処理のみを行うため、副作用はありません。
  • 検索は線形探索（O(n)）で行われます。nEvents は通常非常に小さいため、パフォーマンス上の問題は想定されていません。
