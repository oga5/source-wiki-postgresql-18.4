config_bool

定義

  • 種別: struct
  • 定義箇所: src/include/utils/guc_tables.h:216

目的

PostgreSQLのGUC（Grand Unified Configuration）システムにおいて、真偽値（boolean）をとる設定変数の構造を定義します。

シグネチャまたは形状

struct config_bool
{
    struct config_generic gen;
    bool       *variable;
    bool        boot_val;
    GucBoolCheckHook check_hook;
    GucBoolAssignHook assign_hook;
    GucShowHook show_hook;
    bool        reset_val;
    void       *reset_extra;
};

入力と出力

  • **入力**: gen（基底構造体）、variable（値へのポインタ）、boot_val（初期値）、各種フック関数。
  • **出力**: reset_val および reset_extra を通じて、実行時の設定リセットや状態管理が行われます。

前提条件と不変条件

  • gen フィールドは config_generic 型であり、GUCシステム共通のメタデータを保持します。
  • variable は、設定値が格納されるメモリ上のアドレスを指す必要があります。
  • boot_val は、システム起動時のデフォルト値を保持します。

アルゴリズムまたは振る舞い

  • GUCシステムは、本構造体を用いて設定値の検証（check_hook）、割り当て（assign_hook）、および表示（show_hook）を制御します。
  • reset_val は、設定変更が取り消された際やリセット時に適用される値を保持します。
  • reset_extra は、check_hook で生成された複雑な状態を保持し、assign_hook へ引き継ぐためのブリッジとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/guc.c 内の各関数（call_bool_check_hook、ShowGUCOption 等）が、本構造体を参照・操作します。
  • **呼び出し先**: check_hook、assign_hook、show_hook に登録された関数が、設定変更のライフサイクルに応じて呼び出されます。

関連データ構造

  • config_generic: 本構造体が継承する基底構造体。
  • GucBoolCheckHook, GucBoolAssignHook, GucShowHook: フック関数の型定義。

失敗モードと境界条件

  • variable ポインタが正しく初期化されていない場合、設定値の読み書き時にメモリ破壊や不正な動作を引き起こす可能性があります。
  • check_hook が失敗を返した場合、新しい設定値の適用は拒否されます。

保守担当者向けメモ

  • 本構造体は src/backend/utils/misc/guc_tables.c 内の ConfigureNamesBool 配列で実体が定義されています。
  • extra ポインタ（reset_extra）の管理は、設定変更の整合性を保つために重要です。変更を加える際は、guc.c におけるメモリ管理とフックの呼び出し順序を確認してください。
