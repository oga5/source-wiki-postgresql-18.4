gistxlogPageReuse

定義

  • 種別: struct
  • 定義箇所: src/include/access/gistxlog.h:99

目的

GiSTインデックスにおいて、FSM（Free Space Map）によりページが再利用された際の情報をWAL（Write-Ahead Logging）に記録するためのデータ構造です。

シグネチャまたは形状

typedef struct gistxlogPageReuse
{
	RelFileLocator locator;
	BlockNumber block;
	FullTransactionId snapshotConflictHorizon;
	bool		isCatalogRel;
} gistxlogPageReuse;

入力と出力

  • **入力**: GiSTインデックスのページ再利用時に、対象リレーションの識別子、ブロック番号、スナップショット競合の境界値、およびカタログリレーションかどうかのフラグが格納されます。
  • **出力**: WALレコードの一部として書き込まれ、リカバリ時に gist_redo がこの構造体を読み取って再利用処理を再現します。

前提条件と不変条件

  • SizeOfGistxlogPageReuse マクロにより、構造体のサイズ（isCatalogRel フィールドまで）が定義されています。
  • WALリプレイ時には、XLogRecGetData を介してこの構造体へキャストしてアクセスされます。

アルゴリズムまたは振る舞い

1. **記録**: GiSTインデックスのページ再利用が発生した際、本構造体に情報を詰め、XLogInsert を通じてWALへ書き出します。 2. **リプレイ**: リカバリ処理中、gist_redo がWALレコードを解析し、本構造体から再利用対象のページ情報を取得してインデックス状態を復元します。 3. **競合処理**: snapshotConflictHorizon と isCatalogRel は、スタンバイサーバーでの論理デコード実行時に発生するリカバリ競合を適切に処理するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistxlog.c 内のWAL記録およびリプレイ処理。
  • **呼び出し先**: src/backend/access/rmgrdesc/gistdesc.c の out_gistxlogPageReuse 関数が、デバッグ出力のために本構造体を参照しています。

関連データ構造

  • RelFileLocator: 対象リレーションの識別。
  • BlockNumber: 再利用されるページ番号。
  • FullTransactionId: スナップショット競合の境界。

失敗モードと境界条件

  • 構造体のメンバ構成を変更する場合、gist_redo および gist_desc の実装と整合性を保つ必要があります。
  • isCatalogRel の設定が誤っていると、スタンバイサーバーでの論理デコード時に競合処理が正しく行われない可能性があります。

保守担当者向けメモ

  • 本構造体はWALのバイナリレイアウトと直結しています。フィールドの追加や変更を行う際は、SizeOfGistxlogPageReuse マクロの定義も併せて更新してください。
