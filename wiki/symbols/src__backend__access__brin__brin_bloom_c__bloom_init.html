bloom_init

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_bloom.c:308

目的

BRINインデックスのページ範囲要約に使用する新しいブルームフィルタ構造体を初期化および割り当てます。

シグネチャまたは形状

static BloomFilter *bloom_init(int ndistinct, double false_positive_rate)

入力と出力

  • **入力**:
  • ndistinct: ページ範囲内に含まれると推定される一意な値の数。
  • false_positive_rate: 許容される誤判定率（0より大きく1未満）。
  • **出力**:
  • 初期化済みの BloomFilter 構造体へのポインタ。

前提条件と不変条件

  • ndistinct > 0 であること。
  • 0 < false_positive_rate < 1 であること。
  • 割り当てられるフィルタサイズが BloomMaxFilterSize を超えないこと。

アルゴリズムまたは振る舞い

1. bloom_filter_size を呼び出し、指定されたパラメータに基づき必要なバイト数、ビット数、ハッシュ関数数を算出します。 2. 算出されたサイズが BloomMaxFilterSize を超える場合はエラーを発生させます。 3. offsetof(BloomFilter, data) + nbytes に基づきメモリを確保し、palloc0 でゼロ初期化します。 4. 構造体のメンバ（flags, nhashes, nbits）を設定し、SET_VARSIZE で可変長データとしてのサイズを記録して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin_bloom.c 内の関数（brin_bloom_add_value 等から呼び出される可能性あり）。
  • **呼び出し先**: bloom_filter_size（パラメータ算出用）。

関連データ構造

  • BloomFilter: ブルームフィルタのビットマップとメタデータを保持する構造体。

失敗モードと境界条件

  • nbytes > BloomMaxFilterSize: フィルタサイズがインデックスページに収まらない場合に elog(ERROR) を発生させます。

保守担当者向けメモ

  • 本関数は palloc0 を使用するため、メモリコンテキストの管理に注意してください。
  • フィルタサイズがページ容量ギリギリの場合、将来的に他のデータと結合した際にページサイズ制限を超えるリスクがある点に留意してください。
