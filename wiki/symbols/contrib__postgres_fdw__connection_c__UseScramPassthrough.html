UseScramPassthrough

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/connection.c:683

目的

外部サーバーまたはユーザーマッピングのオプション設定から、SCRAM認証のパススルー（use_scram_passthrough）が有効かどうかを判定します。

シグネチャまたは形状

static bool UseScramPassthrough(ForeignServer *server, UserMapping *user);

入力と出力

  • **入力**:
  • server: 外部サーバー定義（ForeignServer）へのポインタ。
  • user: ユーザーマッピング定義（UserMapping）へのポインタ。
  • **出力**:
  • bool: use_scram_passthrough オプションが true と評価される場合は true、それ以外は false を返します。

前提条件と不変条件

  • server および user は有効なポインタである必要があります。
  • オプションの検索順序は、まず server のオプションを走査し、次に user のオプションを走査します。

アルゴリズムまたは振る舞い

1. server->options リストを走査し、use_scram_passthrough という名前のオプションを探します。見つかった場合はその値を defGetBoolean で取得して返します。 2. server に設定がない場合、user->options リストを走査し、同様に use_scram_passthrough を探します。見つかった場合はその値を返します。 3. どちらのオプションにも存在しない場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/connection.c: 接続確立時の条件判定で使用。
  • contrib/dblink/dblink.c: 接続時のSCRAMキー検証で使用。
  • **呼び出し先**:
  • defGetBoolean: オプション値を真偽値に変換するために呼び出されます。

関連データ構造

  • ForeignServer: 外部サーバーの定義情報を保持する構造体。
  • UserMapping: ユーザーと外部サーバーの紐付け情報を保持する構造体。
  • DefElem: オプションの定義要素を保持する構造体。

失敗モードと境界条件

  • オプションリストが空の場合、または対象のオプション名が含まれていない場合は、デフォルト値として false を返します。
  • defGetBoolean が無効な値を処理する場合の挙動は、当該関数の実装に依存します。

保守担当者向けメモ

  • 本関数は postgres_fdw と dblink の両方で利用されています。
  • サーバーレベルの設定がユーザーレベルの設定よりも優先されます。
