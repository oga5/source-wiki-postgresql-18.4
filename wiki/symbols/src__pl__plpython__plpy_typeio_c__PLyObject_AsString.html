PLyObject_AsString

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_typeio.c:1026

目的

PythonオブジェクトをPostgreSQLで利用可能なC文字列（char *）に変換します。

シグネチャまたは形状

char *PLyObject_AsString(PyObject *plrv)

入力と出力

  • **入力**: plrv (変換対象の PyObject*)
  • **出力**: 変換された文字列を指す char*（呼び出し元で解放が必要なメモリ領域）。

前提条件と不変条件

  • 入力されたPythonオブジェクトから文字列形式への変換が可能であること。
  • 変換後の文字列にNULLバイトが含まれていないこと（含まれる場合はエラーとなる）。

アルゴリズムまたは振る舞い

1. **型に応じた文字列化**:

  • PyUnicode の場合はそのままバイト列へ変換。
  • float の場合は精度損失を防ぐため PyObject_Repr を使用。
  • それ以外は PyObject_Str を使用。

    • 2. **メモリ確保**: pstrdup を用いてPythonのバイト列からPostgreSQLのメモリコンテキストへコピーを作成。 3. **検証**: 変換後の文字列長と元のバイト列のサイズを比較し、NULLバイトの混入や不整合をチェック。 4. **エンコーディング検証**: pg_verifymbstr を呼び出し、PostgreSQLの現在のエンコーディングとして妥当かを確認。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore_plpython/hstore_plpython.c
  • contrib/jsonb_plpython/jsonb_plpython.c
  • src/pl/plpython/plpy_typeio.c
  • **呼び出し先**:
  • Python C API (PyUnicode_Check, PyFloat_Check, PyObject_Repr, PyObject_Str, PLyUnicode_Bytes, PyBytes_AsString, PyBytes_Size, Py_XDECREF)
  • PostgreSQL内部関数 (pstrdup, PLy_elog, ereport, elog, pg_verifymbstr)

関連データ構造

  • PyObject (Python C API)

失敗モードと境界条件

  • **変換失敗**: Pythonオブジェクトの文字列化に失敗した場合、PLy_elog(ERROR, ...) によりPostgreSQLエラーが発生。
  • **NULLバイト**: 変換後の文字列にNULLバイトが含まれる場合、ERRCODE_DATATYPE_MISMATCH でエラー終了。
  • **長さ不整合**: Python側が報告するサイズと実際の文字列長が一致しない場合、elog(ERROR, ...) でエラー終了。

保守担当者向けメモ

  • 本関数は pstrdup を使用しているため、戻り値のメモリ管理は呼び出し側の責任となります。
  • float 型に対して str() ではなく repr() を使用しているのは、str() が精度を落とす可能性があるためです。この仕様変更を行う際は注意してください。
