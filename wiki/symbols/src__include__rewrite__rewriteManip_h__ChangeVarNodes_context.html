ChangeVarNodes_context

定義

  • **種別**: struct
  • **定義箇所**: src/include/rewrite/rewriteManip.h:44

目的

クエリツリーやパースツリー内の Var ノードを走査し、その rt_index（範囲テーブルインデックス）を変更する際の処理状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct ChangeVarNodes_context ChangeVarNodes_context;

※構造体の内部フィールド定義は、提供されたソース抜粋には含まれていません。

入力と出力

本構造体は、ChangeVarNodes_walker や ChangeVarNodesWalkExpression などの走査関数において、引数として渡され、現在の走査状態を保持・伝播するために使用されます。

前提条件と不変条件

  • rewriteManip.c で実装される走査関数において、ツリーの走査中に rt_index の置換や調整を行うための状態管理に使用されます。
  • 破壊的な変更を伴う操作に使用される可能性があるため、呼び出し元でツリーのコピーが必要かどうかの判断が求められます。

アルゴリズムまたは振る舞い

  • walker 関数（expression_tree_walker 等）と組み合わせて使用され、ツリーを再帰的に探索します。
  • コンテキストを通じて、現在のネストレベル（sublevels_up）やターゲットとなる varno を管理し、スコープを制御します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な言及）**:
  • src/backend/optimizer/plan/analyzejoins.c
  • src/backend/rewrite/rewriteManip.c
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、ChangeVarNodes_walker や ChangeVarNodesWalkExpression などの関数によって操作されます。

関連データ構造

  • Var: rt_index を変更対象とするノード。
  • nodes/parsenodes.h: Query や RangeTblEntry など、走査対象となるノード定義が含まれます。
  • nodes/pathnodes.h: 集合操作に関連するデータ構造。

失敗モードと境界条件

  • 構造体内部のフィールド定義が不明なため、特定の失敗モードは定義されていません。
  • 一般的なツリー走査と同様、サブクエリを跨ぐ際の sublevels_up の管理が不適切な場合、誤ったノードが置換されるリスクがあります。

保守担当者向けメモ

  • 構造体の詳細なフィールド定義については、src/include/rewrite/rewriteManip.h の該当行以降を参照してください。
  • sublevels_up のインクリメント/デクリメント処理は、このコンテキストを利用する関数群で一貫して実装されている必要があります。
