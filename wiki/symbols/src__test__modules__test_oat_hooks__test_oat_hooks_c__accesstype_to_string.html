accesstype_to_string

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_oat_hooks/test_oat_hooks.c:417

目的

ObjectAccessType 列挙型とサブID（subId）を、監査ログ出力に適した人間が読みやすい文字列形式に変換します。

シグネチャまたは形状

static char *accesstype_to_string(ObjectAccessType access, int subId);

入力と出力

  • **入力**:
  • access: ObjectAccessType 型のアクセス種別。
  • subId: 権限フラグ等を含む整数値。
  • **出力**:
  • 変換された文字列を保持する char * 型のポインタ。psprintf を使用してメモリが割り当てられるため、呼び出し元で解放する必要があります。

前提条件と不変条件

  • access が既知の ObjectAccessType 値（OAT_POST_CREATE 等）でない場合、デフォルトの警告文字列が返されます。
  • subId のビットフラグ（ACL_SET, ACL_ALTER_SYSTEM）に応じて、出力文字列に詳細な権限情報が付加されます。

アルゴリズムまたは振る舞い

1. switch 文により access に対応する基本文字列（"create", "drop" 等）を決定します。 2. subId のビットマスクを確認し、以下の優先順位で文字列を構築します：

  • ACL_SET と ACL_ALTER_SYSTEM の両方が設定されている場合、"all privileges" を付加。
  • ACL_SET のみが設定されている場合、"set" を付加。
  • ACL_ALTER_SYSTEM のみが設定されている場合、"alter system" を付加。

    • 3. psprintf を用いて、決定した種別名と subId の16進数表記を含む文字列を生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_oat_hooks/test_oat_hooks.c 内の監査ログ出力処理（audit_attempt 等の関連関数）から複数回呼び出されています。
  • **呼び出し先**:
  • psprintf（PostgreSQLのメモリ管理された文字列フォーマット関数）。

関連データ構造

  • ObjectAccessType: catalog/objectaccess.h で定義される列挙型。
  • ACL_SET, ACL_ALTER_SYSTEM: 権限フラグ定数。

失敗モードと境界条件

  • psprintf がメモリ不足に陥った場合、PostgreSQLの標準的なエラーハンドリングによりプロセスが終了する可能性があります。
  • 未定義の ObjectAccessType が渡された場合、"UNRECOGNIZED ObjectAccessType" という文字列が返されます。

保守担当者向けメモ

  • 本関数は psprintf を使用しているため、呼び出し側で返り値のメモリリークが発生しないよう注意してください。
  • 新しい ObjectAccessType が追加された場合は、switch 文にケースを追加する必要があります。
