DeserialIOData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/array_userfuncs.c:44

目的

並列クエリ実行時における配列集約状態のデシリアライズ処理において、型固有の受信関数（typreceive）とそのパラメータ（typioparam）を保持するために使用されます。

シグネチャまたは形状

typedef struct DeserialIOData
{
	FmgrInfo	typreceive;
	Oid			typioparam;
} DeserialIOData;

入力と出力

  • 本構造体は、関数マネージャの fn_extra フィールドにキャッシュされるデータとして利用されます。
  • 入力: FmgrInfo 構造体および Oid 型の型パラメータ。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • fn_extra を介してキャッシュされるため、メモリコンテキストの生存期間に依存します。
  • typreceive は、対象となる配列要素型の受信関数を指す必要があります。

アルゴリズムまたは振る舞い

1. 集約関数の実行中、fn_extra が未初期化であれば、システムカタログから型情報を検索し、DeserialIOData を割り当てて初期化します。 2. 以降の呼び出しでは、キャッシュされた DeserialIOData を再利用して、シリアライズされたデータから配列要素を復元します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/array_userfuncs.c 内の array_agg_deserialize 関数が、fn_extra から本構造体を取得して利用します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありませんが、保持している typreceive フィールドが実行時に呼び出されます。

関連データ構造

  • FmgrInfo: PostgreSQL の関数マネージャ情報。
  • ArrayType: 配列の内部表現。
  • ArrayBuildState: 集約処理中に使用される配列構築状態。

失敗モードと境界条件

  • メモリ不足: fn_extra の初期化時にメモリ割り当てが失敗する可能性があります。
  • 型情報欠落: システムカタログから対象型の受信関数が取得できない場合、実行時エラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は array_agg_deserialize におけるキャッシュ戦略の要です。fn_extra のライフサイクル管理（メモリコンテキストの適切な割り当て）が正しく行われているか注意してください。
  • typioparam は、受信関数が型固有の情報を必要とする場合に備えて保持されています。
