QuerySource

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/parsenodes.h:34

目的

クエリツリーがどのような経緯で生成されたか（元のクエリか、あるいはルール展開によるものか）を識別するために使用されます。

シグネチャまたは形状

typedef enum QuerySource
{
    QSRC_ORIGINAL,
    QSRC_PARSER,
    QSRC_INSTEAD_RULE,
    QSRC_QUAL_INSTEAD_RULE,
    QSRC_NON_INSTEAD_RULE,
} QuerySource;

入力と出力

  • 本シンボルは列挙型であり、関数ではないため入力・出力の概念は適用されません。
  • Query 構造体の querySource フィールドとして保持されます。

前提条件と不変条件

  • QSRC_PARSER は現在使用されていません。
  • クエリの同一性判定（Jumbling）においては、このフィールドは無視される設定（query_jumble_ignore）となっています。

アルゴリズムまたは振る舞い

  • QSRC_ORIGINAL: ユーザーが直接発行したクエリ（パースツリー）であることを示します。
  • QSRC_INSTEAD_RULE: 無条件の INSTEAD ルールによって追加されたクエリであることを示します。
  • QSRC_QUAL_INSTEAD_RULE: 条件付きの INSTEAD ルールによって追加されたクエリであることを示します。
  • QSRC_NON_INSTEAD_RULE: INSTEAD ではないルールによって追加されたクエリであることを示します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/rewrite/rewriteHandler.c において、ルール適用時のアクションに対する QuerySource 値の決定に使用されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • Query: parsenodes.h で定義されるクエリツリーのルートノード。この構造体内に querySource フィールドとして含まれます。

失敗モードと境界条件

  • 特筆すべき失敗モードや境界条件の定義はありません。

保守担当者向けメモ

  • QSRC_PARSER は現在未使用であるため、将来的なコード整理の対象となる可能性があります。
  • ルールシステム（rewriteHandler.c）によるクエリ変換の追跡において重要な役割を果たすため、新しいルール種別が導入される際は本列挙型の拡張が必要になる場合があります。
