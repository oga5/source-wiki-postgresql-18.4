PortalHashEnt

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/mmgr/portalmem.c:48

目的

名前付きポータルを管理するためのハッシュテーブル（PortalHashTable）におけるエントリ構造体です。ポータル名とポータルオブジェクトの対応関係を保持します。

シグネチャまたは形状

typedef struct portalhashent
{
    char        portalname[MAX_PORTALNAME_LEN];
    Portal      portal;
} PortalHashEnt;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。hash_search 等のハッシュテーブル操作関数を通じて、キー（portalname）による検索や挿入の対象となります。

前提条件と不変条件

  • portalname は MAX_PORTALNAME_LEN に収まる文字列である必要があります。
  • PortalHashTable の初期化時に ctl.entrysize として本構造体のサイズが指定されている必要があります。

アルゴリズムまたは振る舞い

ハッシュテーブルの各エントリとして、特定の名前を持つポータルへのポインタを格納します。hash_search を用いて名前をキーに検索を行い、該当する Portal オブジェクトを取得または更新するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mmgr/portalmem.c 内のポータル管理関数群（CreatePortal、PortalDrop、GetPortalByName 等）が、ハッシュテーブル操作の際に本構造体を利用しています。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • Portal: 本構造体が保持するポータルオブジェクトの型。
  • PortalHashTable: 本構造体を要素として管理するハッシュテーブル。

失敗モードと境界条件

  • portalname が MAX_PORTALNAME_LEN を超える場合、切り詰めやバッファオーバーフローのリスクがあります。
  • ハッシュテーブルが初期化されていない状態で本構造体へアクセスすることはできません。

保守担当者向けメモ

  • 本構造体は portalmem.c 内のハッシュテーブル操作と密接に結合しています。フィールドの追加や変更を行う際は、PortalHashTable の初期化ロジック（ctl.entrysize の設定箇所）に影響がないか確認してください。
