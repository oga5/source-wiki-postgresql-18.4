hashing

定義

  • 種別: function
  • 定義箇所: contrib/ltree/_ltree_gist.c:34

目的

ltree 構造内の各レベル名に対して CRC32 ハッシュを計算し、その結果をビットベクトル（シグネチャ）にマッピングすることで、GiST インデックス用のシグネチャを生成する。

シグネチャまたは形状

static void hashing(BITVECP sign, ltree *t, int siglen)

入力と出力

  • **入力**:
  • sign (BITVECP): 更新対象のビットベクトル。
  • t (ltree *): ハッシュ化対象の ltree 構造体。
  • siglen (int): ビットベクトルの長さ。
  • **出力**: なし（sign が直接更新される）。

前提条件と不変条件

  • t は有効な ltree 構造体であること。
  • sign は少なくとも siglen バイト以上のメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. ltree 内の各レベルを順次走査する。 2. 各レベルの名前 (cur->name) と長さ (cur->len) を用いて ltree_crc32_sz を呼び出し、ハッシュ値を算出する。 3. 算出されたハッシュ値を AHASH マクロに渡し、sign ビットベクトル上の対応するビットをセットする。 4. 全てのレベルを処理するまで繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/_ltree_gist.c:77
  • contrib/ltree/ltree_gist.c:213, 345, 366
  • **呼び出し先**:
  • ltree_crc32_sz (ハッシュ計算)
  • AHASH (ビットベクトルへのマッピング)

関連データ構造

  • ltree: ツリー構造を保持するデータ型。
  • ltree_level: ltree 内の各ノード情報。
  • BITVECP: ビットベクトルを指すポインタ型。

失敗モードと境界条件

  • t->numlevel が 0 の場合、ループは実行されず sign は変更されない。
  • siglen が不適切な場合、AHASH マクロ内でのメモリ境界外アクセスが発生する可能性がある。

保守担当者向けメモ

  • 本関数は ltree の GiST インデックス作成におけるシグネチャ生成の核となる。
  • AHASH マクロの定義および ltree_crc32_sz の実装が、インデックスの衝突率や検索性能に直接影響する。
