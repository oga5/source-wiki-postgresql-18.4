AttachIndexCallbackState

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/tablecmds.c:21573

目的

パーティションテーブルに対するインデックス操作（アタッチ処理）において、コールバック関数間で共有される状態を保持するために使用されます。

シグネチャまたは形状

struct AttachIndexCallbackState
{
    Oid         partitionOid;    /* 対象パーティションのOID */
    Oid         parentTblOid;    /* 親テーブルのOID */
    bool        lockedParentTbl; /* 親テーブルがロック済みかを示すフラグ */
};

入力と出力

  • 本構造体は、コールバック関数へ渡される引数（void *arg）としてキャストされて入力されます。
  • 構造体内の各フィールドは、インデックスアタッチ処理の実行コンテキストを特定するために参照されます。

前提条件と不変条件

  • partitionOid および parentTblOid は有効なリレーションOIDである必要があります。
  • lockedParentTbl は、現在のトランザクション内で親テーブルに対する適切なロックが取得されているかどうかの状態を反映します。

アルゴリズムまたは振る舞い

  • tablecmds.c 内のインデックスアタッチ処理において、コールバック関数に渡される状態オブジェクトとしてインスタンス化されます。
  • 処理の過程で、この構造体を通じてパーティションと親テーブルの関係性およびロック状態が伝播されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/tablecmds.c 内のインデックスアタッチ関連関数（詳細はソースコードの文脈に依存）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。
  • tablecmds.c 内の他の状態管理構造体（AlteredTableInfo 等）と並び、DDL実行時のコンテキスト管理に使用されます。

失敗モードと境界条件

  • 無効なOIDが設定された場合、後続のカタログ操作やロック取得処理でエラーが発生する可能性があります。
  • lockedParentTbl の値が実際のロック状態と不整合を起こした場合、デッドロックやロック競合の予期せぬ挙動を招く恐れがあります。

保守担当者向けメモ

  • 本構造体は tablecmds.c 内の特定のコールバック処理に特化しています。変更を加える際は、当該コールバック関数がどのようにこの構造体を解釈しているかを確認してください。
