MultirangeParseState

定義

  • 種別: enum
  • 定義箇所: src/backend/utils/adt/multirangetypes.c:56

目的

マルチレンジ型のテキスト形式入力をパースする際の状態遷移を管理するために使用されます。

シグネチャまたは形状

typedef enum {
    MULTIRANGE_BEFORE_RANGE,
    MULTIRANGE_IN_RANGE,
    MULTIRANGE_IN_RANGE_ESCAPED,
    MULTIRANGE_IN_RANGE_QUOTED,
    MULTIRANGE_IN_RANGE_QUOTED_ESCAPED,
    MULTIRANGE_AFTER_RANGE,
    MULTIRANGE_FINISHED,
} MultirangeParseState;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし（列挙型定義）

前提条件と不変条件

  • テキスト形式のマルチレンジ入力文字列を走査する際、この状態遷移に従って現在の解析位置（範囲外、範囲内、エスケープ中、クォート中など）を追跡する必要があります。

アルゴリズムまたは振る舞い

  • 入力文字列の各文字を走査し、現在の MultirangeParseState に基づいて次の状態へ遷移します。
  • MULTIRANGE_FINISHED に到達することで、パース処理の正常終了を判定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/multirangetypes.c 内のパース処理関数（multirange_in 等）にて、ローカル変数 parse_state として利用されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • MultirangeType: マルチレンジのメモリ上の表現。
  • MultirangeIOData: I/O処理中に使用されるキャッシュ構造体。

失敗モードと境界条件

  • 状態遷移の定義外の文字や、不正なエスケープシーケンスに遭遇した場合の挙動については、本ソース excerpt には記載がありません。

保守担当者向けメモ

  • 本列挙型は multirange_in 関数等のテキスト解析ロジックと密結合しています。解析ロジックを変更する際は、この状態遷移図が網羅的であることを確認してください。
