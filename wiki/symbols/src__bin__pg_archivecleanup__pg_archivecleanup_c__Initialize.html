Initialize

定義

  • 種別: function
  • 定義箇所: src/bin/pg_archivecleanup/pg_archivecleanup.c:57

目的

指定されたアーカイブディレクトリ（archiveLocation）が存在し、かつディレクトリとしてアクセス可能であることを検証する。

シグネチャまたは形状

static void Initialize(void)

入力と出力

  • 入力: グローバル変数 archiveLocation（アーカイブディレクトリのパス）。
  • 出力: なし（正常終了時）。異常時はエラーメッセージを出力し、プロセスを終了する。

前提条件と不変条件

  • archiveLocation が有効なパス文字列として設定されていること。
  • プロセスが当該パスに対して stat() を実行する権限を有していること。

アルゴリズムまたは振る舞い

1. stat() システムコールを使用して、archiveLocation のファイル状態を取得する。 2. stat() が失敗した場合、または取得した情報がディレクトリ（S_ISDIR）を示していない場合、pg_log_error を通じてエラーを報告する。 3. 検証に失敗した場合は exit(2) を呼び出し、プログラムを終了させる。

呼び出し元と呼び出し先

  • 呼び出し元: main 関数（提供されたソース excerpt およびファイル概要より）。
  • 呼び出し先: stat()（標準ライブラリ）、pg_log_error()（PostgreSQL 共通ログ機能）、exit()（標準ライブラリ）。
  • 注記: ツールが提示した contrib/amcheck や contrib/bloom 等の呼び出し元情報は、本シンボルとは無関係な別モジュールのコードであり、本シンボルの呼び出し元ではありません。

関連データ構造

  • struct stat: ファイルシステム上のファイル情報を保持するために使用。

失敗モードと境界条件

  • 指定されたパスが存在しない場合、エラーを出力して終了する。
  • 指定されたパスがディレクトリではなくファイルやデバイスである場合、エラーを出力して終了する。

保守担当者向けメモ

  • 本関数はローカルファイルシステム上のディレクトリを前提としています。ネットワークストレージやリモートパスを扱うように拡張する場合は、この検証ロジックの修正が必要です。
