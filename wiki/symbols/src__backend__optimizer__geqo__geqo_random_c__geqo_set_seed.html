geqo_set_seed

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_random.c:19

目的

GEQO（遺伝的クエリ最適化）セッションにおける乱数生成器の状態を、指定されたシード値に基づいて初期化します。

シグネチャまたは形状

void geqo_set_seed(PlannerInfo *root, double seed)

入力と出力

  • **入力**:
  • root: PlannerInfo 構造体へのポインタ。セッション固有の GeqoPrivateData を保持するために使用されます。
  • seed: 乱数生成器の初期化に使用する double 型のシード値。
  • **出力**: なし（void）。

前提条件と不変条件

  • root->join_search_private が GeqoPrivateData 構造体として適切に初期化されている必要があります。
  • 内部的に PostgreSQL 標準の擬似乱数生成器（pg_prng）を使用しており、private->random_state が更新対象となります。

アルゴリズムまたは振る舞い

1. root->join_search_private から GeqoPrivateData 構造体を取得します。 2. pg_prng_fseed を呼び出し、取得した構造体内の random_state に対して指定された seed を適用します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_main.c:106 にて呼び出されています。
  • **呼び出し先**: pg_prng_fseed（PostgreSQL 内部の擬似乱数生成関数）を呼び出します。

関連データ構造

  • PlannerInfo: クエリ最適化の情報を保持する構造体。
  • GeqoPrivateData: GEQO モジュール固有のデータを保持する構造体。random_state を含みます。

失敗モードと境界条件

  • root が NULL である場合、または root->join_search_private が適切にキャストできない場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は GEQO の実行結果を再現可能にするために使用されます。テストやデバッグ時に固定値を渡すことで、確率的な最適化プロセスを制御可能です。
