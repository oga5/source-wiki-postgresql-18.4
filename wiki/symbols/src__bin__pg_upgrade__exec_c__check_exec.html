check_exec

定義

  • **種別**: 関数
  • **定義箇所**: src/bin/pg_upgrade/exec.c:428

目的

指定されたディレクトリ内のPostgreSQL関連バイナリの存在を確認し、オプションでそのバージョンがpg_upgradeと一致するかを検証する。

シグネチャまたは形状

static void check_exec(const char *dir, const char *program, bool check_version)

入力と出力

  • **入力**:
  • dir: バイナリが存在するディレクトリパス。
  • program: 検証対象の実行ファイル名。
  • check_version: バージョンチェックを実行するかどうかのフラグ。
  • **出力**: なし（成功時は正常終了、失敗時はpg_fatalによりプロセスを終了）。

前提条件と不変条件

  • dirとprogramを結合したパスが、実行可能なファイルとして存在すること。
  • check_versionが真の場合、対象プログラムが-Vオプションをサポートし、PG_VERSIONと一致する文字列を標準出力に返すこと。

アルゴリズムまたは振る舞い

1. validate_execを用いて、指定パスの実行ファイルが有効か確認する。 2. "{dir}/{program}" -V コマンドを実行し、出力を取得する。 3. check_versionが真の場合、取得した出力から改行を除去し、"{program} (PostgreSQL) {PG_VERSION}" という期待値と比較する。 4. 比較結果が一致しない場合、またはコマンド実行に失敗した場合はpg_fatalでエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**: check_bin_dir（src/bin/pg_upgrade/exec.c内）から、postgres, pg_controldata, pg_ctl, pg_resetwal, initdb, pg_dump 等の各バイナリに対して呼び出される。
  • **呼び出し先**: validate_exec, pipe_read_line, pg_strip_crlf, pg_fatal, pg_free。

関連データ構造

  • MAXPGPATH: パス文字列の最大長を定義する定数。

失敗モードと境界条件

  • **ファイル不在**: validate_execが失敗した場合、致命的エラーとなる。
  • **実行失敗**: pipe_read_lineがNULLを返した場合（コマンド実行不可）、致命的エラーとなる。
  • **バージョン不一致**: 取得したバージョン文字列が期待値と異なる場合、致命的エラーとなる。

保守担当者向けメモ

  • 本関数はpg_upgradeが使用するバイナリの整合性を保証する重要な役割を担う。
  • Windows環境では、exec_prog等と同様にログ出力やプロセス実行時の排他制御が重要となるため、修正時はexec.c内の他の関数との一貫性に注意すること。
