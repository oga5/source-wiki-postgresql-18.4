bh_node_type

定義

  • 種別: typedef
  • 定義箇所: src/include/lib/binaryheap.h:21

目的

二分ヒープ（binaryheap）に格納されるノードのデータ型を抽象化するために使用されます。

シグネチャまたは形状

typedef void *bh_node_type;

※環境や設定により typedef Datum bh_node_type; と定義される場合もあります（src/include/lib/binaryheap.h:23）。

入力と出力

  • 入力: ポインタ型（または Datum 型）の値を保持します。
  • 出力: ヒープ操作関数（binaryheap_add 等）において、ノード値として受け渡しされます。

前提条件と不変条件

  • binaryheap 構造体内部の bh_nodes 配列の要素として使用されます。
  • 直接的な使用は推奨されておらず、ヒープのインターフェースを介した操作が想定されています。

アルゴリズムまたは振る舞い

ヒープ内の各ノードの値を保持する汎用的な型として機能します。メモリ確保時には sizeof(bh_node_type) を用いて必要な領域が計算されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/common/binaryheap.c: binaryheap_add_unordered, binaryheap_add, binaryheap_replace_first 等の関数引数および内部変数として使用。
  • src/include/lib/binaryheap.h: 構造体定義および型定義。
  • 呼び出し先: 特定の関数を呼び出すシンボルではありません。

関連データ構造

  • binaryheap: この型を要素として保持する二分ヒープ構造体。

失敗モードと境界条件

  • 型の解決（void * か Datum か）が環境間で不一致になると、コンパイルエラーやメモリレイアウトの不整合を引き起こす可能性があります。

保守担当者向けメモ

  • 本シンボルは抽象化レイヤーであるため、直接操作するよりも binaryheap の API を通じて利用してください。
  • src/include/lib/binaryheap.h 内で条件分岐により定義が切り替わる点に注意してください。
