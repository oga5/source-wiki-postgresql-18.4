SortByDir

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/parsenodes.h:44

目的

SQLクエリにおけるソート順序（昇順、降順、デフォルト、または演算子指定）を表現するために使用されます。

シグネチャまたは形状

typedef enum SortByDir
{
	SORTBY_DEFAULT,
	SORTBY_ASC,
	SORTBY_DESC,
	SORTBY_USING,
} SortByDir;

入力と出力

  • 本シンボルは列挙型であり、関数ではありません。
  • パーサーがSQL文を解析する際、ソート指定の種別を保持するために使用されます。

前提条件と不変条件

  • SORTBY_USING は CREATE INDEX 文では使用できません。

アルゴリズムまたは振る舞い

  • SORTBY_DEFAULT: 明示的な指定がない場合のデフォルトのソート順。
  • SORTBY_ASC: 昇順（ASC）。
  • SORTBY_DESC: 降順（DESC）。
  • SORTBY_USING: 演算子（USING 句）を使用したソート指定。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数情報は提供されていません。
  • 参照箇所:
  • src/include/nodes/parsenodes.h:573: SortBy 構造体内の sortby_dir フィールドとして使用。
  • src/include/nodes/parsenodes.h:815: WindowDef 構造体内の ordering フィールドとして使用。

関連データ構造

  • SortBy: ソート指定を保持する構造体。
  • WindowDef: ウィンドウ関数の定義を保持する構造体。

失敗モードと境界条件

  • SORTBY_USING を CREATE INDEX 文で使用しようとした場合、構文解析または意味解析の段階で制約違反として扱われる可能性があります。

保守担当者向けメモ

  • 本シンボルはパーサーが生成するノードの一部です。ソート順序の解釈ロジックを変更する場合は、この列挙型だけでなく、これを利用する SortBy や WindowDef 構造体、およびそれらを処理する解析器（analyze.c）の変更が必要です。
