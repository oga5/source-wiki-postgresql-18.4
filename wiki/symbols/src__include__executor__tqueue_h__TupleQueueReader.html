TupleQueueReader

定義

  • 種別: struct
  • 定義箇所: src/include/executor/tqueue.h:21

目的

並列クエリ実行において、共有メモリキュー（shm_mq）からタプルを非同期的に読み取るための不透明な構造体です。

シグネチャまたは形状

不透明な構造体（typedef struct TupleQueueReader TupleQueueReader;）として定義されており、内部メンバは公開されていません。

入力と出力

  • 入力: 共有メモリ上のメッセージキュー。
  • 出力: TupleQueueReaderNext() を介して取得される MinimalTuple。

前提条件と不変条件

  • shm_mq インターフェースを介して共有メモリと接続されている必要があります。
  • 送信側が shm_mq_send で書き込んだデータを読み取ることを前提とします。

アルゴリズムまたは振る舞い

  • 受信側プロセスは、本構造体を使用して共有メモリからタプルを順次取り出します。
  • TupleQueueReaderNext() を繰り返し呼び出すことで、キュー内のタプルを消費します。
  • 送信側が終了した場合は、キューが空であることを判定可能です。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/executor/execParallel.c
  • src/backend/executor/nodeGather.c
  • src/backend/executor/nodeGatherMerge.c
  • 呼び出し先:
  • ソースコード上での直接的な呼び出し先は、本構造体が不透明であるため、提供された情報からは特定できません。

関連データ構造

  • shm_mq: 共有メモリベースのメッセージキュー。
  • MinimalTuple: キューを通じて転送されるタプルの形式。

失敗モードと境界条件

  • キューが空の場合、TupleQueueReaderNext() の nowait フラグ設定により非ブロッキング読み取りが可能ですが、それ以外の失敗モードについては提供された情報に記載がありません。

保守担当者向けメモ

  • 本構造体は shm_mq のラッパーとして機能します。
  • TupleQueueReaderNext の done 引数を確認することで、キューの終了状態を適切にハンドリングしてください。
