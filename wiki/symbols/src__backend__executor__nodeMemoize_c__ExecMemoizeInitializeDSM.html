ExecMemoizeInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMemoize.c:1210

目的

並列クエリ実行時、各ワーカー間で統計情報を共有するための共有メモリ領域（DSM: Dynamic Shared Memory）を初期化する。

シグネチャまたは形状

void ExecMemoizeInitializeDSM(MemoizeState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: MemoizeState 構造体へのポインタ。
  • pcxt: ParallelContext 構造体へのポインタ。
  • **出力**: なし。node->shared_info に割り当てられた共有メモリ領域が格納される。

前提条件と不変条件

  • node->ss.ps.instrument が有効であり、かつ pcxt->nworkers > 0 である場合にのみ実行される。
  • 共有メモリ領域は shm_toc_allocate を介して割り当てられ、ゼロクリアされる。

アルゴリズムまたは振る舞い

1. インストゥルメンテーションが不要またはワーカーが存在しない場合は直ちに終了する。 2. SharedMemoizeInfo 構造体とワーカー数分の MemoizeInstrumentation 配列を格納するために必要なサイズを計算する。 3. shm_toc_allocate を使用して共有メモリを確保する。 4. 確保した領域を memset でゼロクリアする。 5. num_workers を設定し、shm_toc_insert を用いてプランノードIDをキーとして共有メモリテーブルに登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:533 (確認済み)
  • **呼び出し先**: shm_toc_allocate, shm_toc_insert, memset (標準ライブラリ)

関連データ構造

  • MemoizeState: ノードの実行状態を保持。
  • SharedMemoizeInfo: 並列ワーカー間で共有される統計情報構造体。
  • MemoizeInstrumentation: 各ワーカーの統計情報。
  • ParallelContext: 並列実行のコンテキスト。

失敗モードと境界条件

  • pcxt->nworkers == 0 の場合、共有メモリの初期化はスキップされる。
  • instrument が無効な場合、共有メモリの初期化はスキップされる。

保守担当者向けメモ

  • 本関数は並列クエリのセットアップフェーズで呼び出される。
  • 共有メモリのレイアウト変更を行う場合は、SharedMemoizeInfo 構造体の定義と本関数のサイズ計算ロジックを同期させる必要がある。
