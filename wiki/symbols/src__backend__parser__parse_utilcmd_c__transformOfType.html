transformOfType

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_utilcmd.c:1638

目的

CREATE TABLE ... OF type_name 構文において、指定された複合型（row type）の属性定義を抽出し、テーブルの列定義として CreateStmtContext に追加する。

シグネチャまたは形状

static void transformOfType(CreateStmtContext *cxt, TypeName *ofTypename)

入力と出力

  • **入力**:
  • cxt: 変換状態を保持する CreateStmtContext 構造体へのポインタ。
  • ofTypename: 継承元となる型名を表す TypeName 構造体。
  • **出力**: なし（副作用として cxt->columns に ColumnDef ノードが追加される）。

前提条件と不変条件

  • ofTypename は NULL であってはならない（Assert により検証）。
  • 指定された型は複合型（row type）である必要がある（check_of_type による検証）。

アルゴリズムまたは振る舞い

1. typenameType を使用して型名から pg_type カタログのタプルを取得する。 2. check_of_type で対象が複合型であることを確認し、型 OID を取得して ofTypename->typeOid にキャッシュする。 3. lookup_rowtype_tupdesc で型の属性記述子（TupleDesc）を取得する。 4. 属性記述子を走査し、削除済み（attisdropped）でない各属性について以下の処理を行う：

  • makeColumnDef を用いて列定義ノードを生成する。
  • is_from_type フラグを true に設定する。
  • cxt->columns リストに生成した列定義を追加する。

    • 5. 最後に TupleDesc およびシステムキャッシュのタプルを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: transformCreateStmt（src/backend/parser/parse_utilcmd.c 内）。
  • **呼び出し先**: typenameType, check_of_type, lookup_rowtype_tupdesc, TupleDescAttr, makeColumnDef, lappend, ReleaseTupleDesc, ReleaseSysCache。

関連データ構造

  • CreateStmtContext: 変換中のテーブル定義状態を保持する構造体。
  • TypeName: 構文解析された型名情報。
  • ColumnDef: 生成される列定義ノード。
  • TupleDesc: 複合型の属性定義情報。

失敗モードと境界条件

  • 指定された型名が存在しない、または複合型ではない場合、typenameType や check_of_type 内でエラーが発生し処理が中断される。
  • 削除済みの属性（attisdropped）はスキップされる。

保守担当者向けメモ

  • 本関数は CREATE TABLE ... OF 構文専用の処理である。
  • is_from_type フラグは、後の処理でこの列が型定義由来であることを識別するために使用される。
  • TupleDesc の解放漏れを防ぐため、ReleaseTupleDesc が適切に呼び出されていることを確認すること。
