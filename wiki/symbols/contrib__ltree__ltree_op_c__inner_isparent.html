inner_isparent

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_op.c:212

目的

2つの ltree 型データ（子パス c と親パス p）を比較し、p が c の先頭部分と一致する階層構造（親）であるかを判定する。

シグネチャまたは形状

bool inner_isparent(const ltree *c, const ltree *p)

入力と出力

  • **入力**:
  • c: 比較対象となる子パス (ltree *)
  • p: 比較対象となる親パス候補 (ltree *)
  • **出力**:
  • true: p が c の先頭部分と一致する場合
  • false: 一致しない場合、または p の階層数が c より多い場合

前提条件と不変条件

  • c および p は有効な ltree 構造体であること。
  • ltree 構造体内の numlevel フィールドが正しく設定されていること。

アルゴリズムまたは振る舞い

1. 親パスの階層数 (pn) が子パスの階層数より多い場合、直ちに false を返す。 2. 親パスの各レベルを先頭から順に走査する。 3. 各レベルにおいて、ラベルの長さ (len) が一致し、かつ memcmp によるラベル名が一致することを確認する。 4. 全てのレベルで一致が確認できれば true を返す。不一致があればその時点で false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/ltree_gist.c: GiST インデックスの比較処理で使用。
  • contrib/ltree/ltree_op.c: ltree_isparent 等のラッパー関数から呼び出される。
  • **呼び出し先**:
  • LTREE_FIRST (マクロ): ltree の最初のレベルを取得。
  • LEVEL_NEXT (マクロ): 次のレベルへポインタを進める。
  • memcmp: ラベル名のバイト比較。

関連データ構造

  • ltree: 階層化されたラベルパスを保持する構造体。
  • ltree_level: ltree 内の個々のラベル情報を保持する構造体。

失敗モードと境界条件

  • pn > c->numlevel: 親の階層が子より深い場合は常に false となる。
  • ラベルの長さや内容が異なる場合は false となる。

保守担当者向けメモ

  • 本関数はメモリ確保を行わないため、比較的高速に動作する。
  • ltree 構造の内部レイアウト（ltree_level の並び）に強く依存しているため、データ構造の変更時には注意が必要。
