Cash

定義

  • typedef int64 Cash;
  • 定義箇所: src/include/utils/cash.h:17

目的

PostgreSQL における money 型の内部表現を提供します。64 ビット整数を使用することで、通貨データの演算性能と精度の両立を図ります。

シグネチャまたは形状

int64 のエイリアスです。

入力と出力

  • 入力: SQL レイヤーから渡される Datum 形式のデータ。
  • 出力: 演算結果としての int64 値、または Datum への変換後の値。

前提条件と不変条件

  • int64 のエイリアスであるため、アーキテクチャごとの int64 の扱いに依存します。
  • 演算時にはオーバーフローチェックが必須です（src/backend/utils/adt/cash.c 内の関数が pg_add_s64_overflow 等を用いて担保）。

アルゴリズムまたは振る舞い

本シンボル自体は型定義ですが、PostgreSQL の FMGR（Function Manager）を介して Datum と相互変換されます。PG_GETARG_CASH マクロで Datum から展開され、演算後に PG_RETURN_CASH マクロで Datum に戻されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）: contrib/btree_gist/btree_cash.c において、比較演算（>、>=、==、<=、<）や距離計算、および PG_GETARG_CASH を用いた引数取得に使用されています。
  • 呼び出し先: 本シンボルは型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • int64: 基底となるデータ型。
  • Datum: PostgreSQL の汎用データ表現。

失敗モードと境界条件

  • 64 ビット整数の範囲を超える演算はオーバーフローを引き起こします。
  • 変換時や演算時の境界条件については、ペアファイルである src/backend/utils/adt/cash.c の実装に依存します。

保守担当者向けメモ

  • 本型は money 型の内部表現です。変更を加える際は、src/backend/utils/adt/cash.c にある演算ロジックおよびオーバーフローチェックへの影響を必ず確認してください。
