map_partition_varattnos

定義

  • src/backend/catalog/partition.c:221

目的

パーティション階層間において、列の物理的な順序や属性番号（attnum）が異なる場合に、式（expr）内の属性参照をターゲットのパーティションに合わせて変換します。

シグネチャまたは形状

List *map_partition_varattnos(List *expr, int fromrel_varno, Relation to_rel, Relation from_rel)

入力と出力

  • **入力**:
  • expr: 変換対象の式リスト。
  • fromrel_varno: 式内の Var ノードが参照するリレーション番号。
  • to_rel: 変換先のパーティションリレーション。
  • from_rel: 変換元のリレーション。
  • **出力**:
  • 変換後の式リスト。expr が NIL の場合はそのまま返されます。

前提条件と不変条件

  • to_rel および from_rel は有効な Relation オブジェクトである必要があります。
  • to_rel の行型（reltype）情報が変換処理に使用されます。

アルゴリズムまたは振る舞い

1. expr が空でない場合、build_attrmap_by_name を使用して to_rel と from_rel の属性マッピング（AttrMap）を生成します。 2. map_variable_attnos を呼び出し、式ツリー内の Var ノードを新しい属性番号に書き換えます。 3. この際、to_rel の行型情報を渡すことで、found_whole_row（行全体参照の有無）の判定結果を無視する安全な変換を行います。 4. 変換された式リストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c
  • src/backend/commands/trigger.c
  • src/backend/partitioning/partbounds.c
  • src/backend/utils/cache/partcache.c
  • **呼び出し先**:
  • build_attrmap_by_name
  • map_variable_attnos
  • RelationGetDescr
  • RelationGetForm

関連データ構造

  • AttrMap: 属性番号の対応関係を保持する構造体。
  • Relation: リレーションのメタデータ。
  • List: 式ツリーを構成するノードリスト。

失敗モードと境界条件

  • expr が NIL の場合は変換を行わず、そのまま返します。
  • 属性名が一致しない場合やマッピングが不可能な場合の挙動は、内部で呼び出される build_attrmap_by_name および map_variable_attnos の仕様に依存します。

保守担当者向けメモ

  • 本関数は、パーティションテーブルの制約やトリガー定義を継承・適用する際に、物理的な列順序の差異を吸収するために不可欠です。
  • 変換処理において found_whole_row を無視しているため、行全体を指す Var ノードの扱いには注意が必要です。
