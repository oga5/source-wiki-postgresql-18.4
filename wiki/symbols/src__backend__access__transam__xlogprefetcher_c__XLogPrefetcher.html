XLogPrefetcher

定義

  • **種別**: struct
  • **定義箇所**: src/backend/access/transam/xlogprefetcher.c:124

目的

リカバリ処理中のI/O待ち時間を削減するため、WALを先行デコードし、将来必要となるデータブロックをバッファプールへ事前にロード（先読み）するための状態管理構造体です。

シグネチャまたは形状

struct XLogPrefetcher
{
    XLogReaderState *reader;
    DecodedXLogRecord *record;
    int         next_block_id;
    XLogRecPtr  next_stats_shm_lsn;
    HTAB       *filter_table;
    dlist_head  filter_queue;
    RelFileLocator recent_rlocator[XLOGPREFETCHER_SEQ_WINDOW_SIZE];
    BlockNumber recent_block[XLOGPREFETCHER_SEQ_WINDOW_SIZE];
    int         recent_idx;
    XLogRecPtr  no_readahead_until;
    LsnReadQueue *streaming_read;
    XLogRecPtr  begin_ptr;
    int         reconfigure_count;
};

入力と出力

  • **入力**: XLogPrefetcherAllocate により初期化され、XLogReaderState と紐付けられます。
  • **出力**: XLogPrefetcherNextBlock 等の処理を通じて、PrefetchSharedBuffer を介したバッファプールへのI/O発行が行われます。

前提条件と不変条件

  • maintenance_io_concurrency が0より大きい場合にのみ機能します。
  • メインフォークのブロックのみが先読み対象となります。
  • filter_table および filter_queue は、存在しないファイルや切り詰められた範囲への不要なI/Oを抑制するために使用されます。

アルゴリズムまたは振る舞い

1. **WAL先行デコード**: XLogPrefetcherReadRecord をエントリポイントとして、XLogReaderState を用いてWALを先行して解析します。 2. **I/O発行判断**: 解析したレコードからブロック参照を抽出し、LsnReadQueue を介してI/Oを発行します。 3. **重複抑制**: recent_rlocator および recent_block を使用して、直近の先読みリクエストとの重複を回避します。 4. **フィルタリング**: XLogPrefetcherAddFilter で登録された範囲に基づき、リプレイが追いつく前の不正なI/Oを防止します。 5. **統計管理**: next_stats_shm_lsn に到達するごとに、共有メモリ上の統計情報を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: XLogPrefetcherAllocate（生成）、XLogPrefetcherAddFilter、XLogPrefetcherCompleteFilters、XLogPrefetcherFree 等のモジュール内関数から参照されます。
  • **呼び出し先**: PrefetchSharedBuffer（I/O発行）、XLogReaderState（WAL読み込み）。

関連データ構造

  • XLogReaderState: WALデコードの基盤。
  • LsnReadQueue: 先読み中のI/Oリクエストを管理する循環キュー。
  • XLogPrefetcherFilter: 先読み禁止範囲を管理する構造体。
  • RelFileLocator: 対象ファイルの識別子。

失敗モードと境界条件

  • **フィルタリング**: リプレイが追いつく前に存在しないファイルへアクセスしようとした場合、フィルタ機構によりI/Oが抑制されます。
  • **再構成**: GUC変数の変更に伴い reconfigure_count が更新され、先読みキューの再構成が行われます。

保守担当者向けメモ

  • #define XLOGPREFETCHER_DEBUG_LEVEL LOG を有効にすると、先読みの抑制理由がログに出力されます。
  • 統計情報は pg_stat_recovery_prefetch ビューを通じて確認可能です。
  • 現在の設計ではメインフォーク以外のブロックは先読み対象外です。
