MultiExecBitmapOr

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapOr.c:110

目的

複数のサブプランから取得した TIDBitmap を論理和（OR）で統合し、単一のビットマップとして返す。

シグネチャまたは形状

Node *MultiExecBitmapOr(BitmapOrState *node)

入力と出力

  • **入力**: BitmapOrState *node（実行状態を保持するノード構造体）
  • **出力**: Node *（統合された TIDBitmap へのポインタ）

前提条件と不変条件

  • node->bitmapplans に有効なサブプランが格納されていること。
  • 少なくとも1つのサブプランが存在すること（0入力の場合はエラーとなる）。
  • work_mem 設定に基づきメモリが割り当てられる。

アルゴリズムまたは振る舞い

1. node->bitmapplans に含まれる全サブプランを順次走査する。 2. **最適化パス**: 子ノードが BitmapIndexScanState の場合、結果用ビットマップを直接子ノードへ渡し、子ノード側で直接書き込ませることでコピーを回避する。 3. **標準パス**: それ以外の子ノードの場合、MultiExecProcNode で取得したビットマップを tbm_union を用いて既存の結果にマージし、取得したビットマップは tbm_free で解放する。 4. 実行中、node->ps.instrument が有効であれば計測を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:537
  • **呼び出し先**:
  • MultiExecProcNode（各サブプランの実行）
  • tbm_create, tbm_union, tbm_free（ビットマップ操作）
  • InstrStartNode, InstrStopNode（計測）

関連データ構造

  • BitmapOrState: 実行状態を管理する構造体。
  • TIDBitmap: タプルIDの集合を保持するビットマップ構造体。
  • BitmapIndexScanState: 最適化対象となる子ノードの型。

失敗モードと境界条件

  • **入力なし**: サブプランが空の場合、elog(ERROR) を発生させる。
  • **不正な結果**: 子ノードから TIDBitmap 以外の型が返された場合、または最適化パスで期待したビットマップと異なるポインタが返された場合に elog(ERROR) を発生させる。

保守担当者向けメモ

  • BitmapIndexScan に対する最適化パスはパフォーマンス上重要であるため、変更時はビットマップの共有状態が正しく維持されるか注意すること。
  • work_mem を使用したメモリ制限が適切かどうかの検討が必要（ソース内の XXX コメント参照）。
