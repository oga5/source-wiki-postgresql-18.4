pg_control_checkpoint

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/pg_controldata.c:69

目的

データベースの制御ファイル（global/pg_control）から最新のチェックポイント情報を読み取り、SQLの行型として出力する。

シグネチャまたは形状

Datum pg_control_checkpoint(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数インターフェース経由で呼び出される）。
  • **出力**: Datum（チェックポイント情報の各フィールドを格納した HeapTuple）。

前提条件と不変条件

  • 呼び出し元は、戻り値の型が複合型（TYPEFUNC_COMPOSITE）であることを期待する。
  • 制御ファイルへのアクセスには ControlFileLock（共有ロック）が必須である。
  • 制御ファイルのCRCチェックが成功しなければならない。

アルゴリズムまたは振る舞い

1. get_call_result_type で戻り値の型を確認する。 2. LWLockAcquire で ControlFileLock を取得し、get_controlfile で制御ファイルの内容を読み込む。 3. CRCチェックを行い、不一致の場合はエラーを投げる。 4. XLByteToSeg および XLogFileName を使用し、チェックポイントのREDO開始位置からWALファイル名を算出する。 5. ControlFileData 構造体からチェックポイント関連の各フィールド（LSN、タイムラインID、トランザクションID、OID、タイムスタンプ等）を抽出し、Datum 配列に格納する。 6. heap_form_tuple を用いて行データを作成し、PG_RETURN_DATUM で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（直接的なC言語レベルの呼び出し元は提供されていない）。
  • **呼び出し先**: get_call_result_type, LWLockAcquire, get_controlfile, LWLockRelease, XLByteToSeg, XLogFileName, LSNGetDatum, CStringGetTextDatum, Int32GetDatum, BoolGetDatum, ObjectIdGetDatum, TransactionIdGetDatum, TimestampTzGetDatum, heap_form_tuple。

関連データ構造

  • ControlFileData: 制御ファイルの内容を保持する構造体。
  • CheckPoint: ControlFileData 内でチェックポイント情報を保持するサブ構造体。
  • HeapTuple: SQL関数として返す行データ。

失敗モードと境界条件

  • 戻り値の型が複合型でない場合、elog(ERROR) を発生させる。
  • 制御ファイルのCRCチェックが失敗した場合、ereport(ERROR) を発生させる。

保守担当者向けメモ

  • 本関数はSQL層へのインターフェースであり、ビジネスロジックの多くは get_controlfile（src/common/controldata_utils.c）に依存している。
  • 制御ファイルの構造（ControlFileData）に変更があった場合、values 配列のインデックスとフィールドの対応関係を更新する必要がある。
