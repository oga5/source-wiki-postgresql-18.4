pull_varnos_of_level

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/var.c:139

目的

指定されたクエリレベル（levelsup）に属するすべての Var ノードおよび PlaceHolderVar ノードの varno を収集し、ビットマップセット（Relids）として返します。

シグネチャまたは形状

Relids pull_varnos_of_level(PlannerInfo *root, Node *node, int levelsup)

入力と出力

  • **入力**:
  • root: PlannerInfo 構造体へのポインタ（オプティマイザのコンテキスト）。
  • node: 走査対象のクエリツリーまたは式ツリー。
  • levelsup: 抽出対象とするサブクエリの深さ。
  • **出力**:
  • Relids: 抽出された varno の集合を表すビットマップセット。

前提条件と不変条件

  • node は Query 構造体または任意の式ツリーである必要があります。
  • levelsup は、対象とする変数が現在のクエリから見て何階層上にあるかを指定します。

アルゴリズムまたは振る舞い

1. pull_varnos_context 構造体を初期化し、sublevels_up に指定された levelsup を設定します。 2. query_or_expression_tree_walker を呼び出し、ツリー全体を再帰的に走査します。 3. 走査中、pull_varnos_walker 関数を使用して、現在のレベルと一致する Var または PlaceHolderVar を特定し、その varno を context.varnos に追加します。 4. 走査完了後、収集された Relids を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/subselect.c
  • src/backend/optimizer/prep/prepjointree.c
  • src/backend/optimizer/util/var.c（内部再帰および関連関数）
  • **呼び出し先**:
  • query_or_expression_tree_walker
  • pull_varnos_walker（走査用コールバック）

関連データ構造

  • Relids: Bitmapset への型定義。
  • pull_varnos_context: 走査中に状態（varnos セット、root、sublevels_up）を保持するための内部構造体。
  • PlannerInfo: クエリ最適化の情報を保持する構造体。

失敗モードと境界条件

  • node が NULL の場合、空のビットマップセット（NULL）を返します。
  • 指定したレベルに該当する変数が存在しない場合、空のビットマップセットを返します。

保守担当者向けメモ

  • 本関数は query_or_expression_tree_walker を使用しているため、サブクエリの境界を跨ぐ際の sublevels_up の管理が重要です。
  • PlaceHolderVar は Var と同様に扱われるため、結合のNULL化情報を含む式を扱う際は注意してください。
