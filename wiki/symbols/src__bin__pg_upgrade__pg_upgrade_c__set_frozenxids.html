set_frozenxids

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.c:874

目的

アップグレード後の新クラスタにおいて、システムカタログ（pg_database, pg_class）のトランザクションID（XID）およびマルチトランザクションID（MXID）の凍結カウンタを、旧クラスタのチェックポイント値と同期させる。

シグネチャまたは形状

static void set_frozenxids(bool minmxid_only)

入力と出力

  • **入力**:
  • minmxid_only (bool): true の場合、MXIDカウンタのみを更新する。false の場合、XIDとMXIDの両方を更新する。
  • **出力**: なし（副作用として新クラスタのシステムカタログを更新）。

前提条件と不変条件

  • old_cluster および new_cluster グローバル構造体が初期化済みであること。
  • 新クラスタが起動しており、connectToServer を介して接続可能であること。
  • old_cluster.controldata に旧クラスタのチェックポイント情報（chkpnt_nxtxid, chkpnt_nxtmulti）が保持されていること。

アルゴリズムまたは振る舞い

1. minmxid_only フラグに基づき、処理内容を決定する。 2. template1 データベースに接続し、pg_database の datminmxid を更新する（minmxid_only が false なら datfrozenxid も更新）。 3. 全データベースのリストを取得し、各データベースに対して以下の処理を繰り返す。

  • 接続不可のデータベース（datallowconn = false）の場合、一時的に ALLOW_CONNECTIONS = true に変更する。
  • 対象データベースに接続し、pg_class の relminmxid を更新する（minmxid_only が false なら relfrozenxid も更新）。更新対象は RELKIND_RELATION, RELKIND_MATVIEW, RELKIND_TOASTVALUE に限定される。
  • 接続を終了し、必要に応じて datallowconn を元の false に戻す。

    • 4. 最後に check_ok() を呼び出し、処理完了を通知する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/pg_upgrade.c 内のメインフロー（main 関数等から set_frozenxids(false) または set_frozenxids(true) として呼び出される）。
  • **呼び出し先**:
  • connectToServer, executeQueryOrDie, prep_status, check_ok, quote_identifier, PQclear, PQfnumber, PQntuples, PQgetvalue, PQfinish。

関連データ構造

  • old_cluster (グローバル変数): 旧クラスタの制御データ（controldata）を保持。
  • new_cluster (グローバル変数): 新クラスタへの接続情報を保持。

失敗モードと境界条件

  • データベース接続に失敗した場合、executeQueryOrDie により処理が中断される。
  • datallowconn が false のデータベースに対する ALTER DATABASE 操作が失敗した場合、接続状態の復元が正しく行われない可能性がある。
  • relkind が対象外のテーブル（インデックス等）は更新対象から除外される。

保守担当者向けメモ

  • 本関数は pg_upgrade の最終段階に近い処理であり、新クラスタのメタデータ整合性に直結する。
  • datallowconn を一時的に変更するロジックは、template0 等の特殊なデータベースを扱うために必須である。
  • CppAsString2 マクロを使用して pg_class の relkind 定数を文字列化している点に注意。
