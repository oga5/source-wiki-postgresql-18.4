PgFdwModifyState

定義

  • 種別: struct
  • 定義箇所: contrib/postgres_fdw/postgres_fdw.c:182

目的

外部テーブルに対する INSERT、UPDATE、DELETE 操作の実行状態を保持し、リモートサーバーとの通信およびデータ変換を管理する。

シグネチャまたは形状

typedef struct PgFdwModifyState {
    Relation rel;
    AttInMetadata *attinmeta;
    PGconn *conn;
    PgFdwConnState *conn_state;
    char *p_name;
    char *query;
    char *orig_query;
    List *target_attrs;
    int values_end;
    int batch_size;
    bool has_returning;
    List *retrieved_attrs;
    AttrNumber ctidAttno;
    int p_nums;
    FmgrInfo *p_flinfo;
    int num_slots;
    MemoryContext temp_cxt;
    struct PgFdwModifyState *aux_fmstate;
} PgFdwModifyState;

入力と出力

  • **入力**: ResultRelInfo の ri_FdwState フィールドを通じてエグゼキュータから参照される。
  • **出力**: リモートサーバーへの SQL 送信、および RETURNING 句がある場合はその結果の取得に使用される。

前提条件と不変条件

  • rel は対象となる外部テーブルの relcache エントリを指す必要がある。
  • conn はリモートサーバーへの有効な接続を保持している必要がある。
  • temp_cxt はタプルごとの一時的なデータ処理のために使用されるメモリコンテキストである。

アルゴリズムまたは振る舞い

  • リモートクエリの実行に必要な接続情報、準備済みステートメント名、およびクエリ文字列を保持する。
  • batch_size に基づき、バッチ処理の制御を行う。
  • RETURNING 句が存在する場合、その属性リストと結果取得のための情報を管理する。
  • 行移動（row movement）が発生するサブプランの結果リレーションである場合、aux_fmstate を介して挿入状態を保持する。

呼び出し元と呼び出し先

  • **呼び出し元**: postgresExecForeignDelete 等の DML 実行関数が resultRelInfo->ri_FdwState から本構造体を取得する。
  • **呼び出し先**: create_foreign_modify により生成され、prepare_foreign_modify, convert_prep_stmt_params, store_returning_result, finish_foreign_modify, deallocate_query 等の関数に引数として渡される。

関連データ構造

  • Relation: 外部テーブルのキャッシュ情報。
  • PGconn: リモートサーバーとの接続。
  • PgFdwConnState: 接続ごとの追加状態。
  • AttInMetadata: データ型変換メタデータ。

失敗モードと境界条件

  • 接続が切断された場合、conn を介した操作が失敗する。
  • p_name が設定されている場合、準備済みステートメントの割り当てと解放の整合性が求められる。

保守担当者向けメモ

  • 本構造体は postgres_fdw の DML 処理の核となる状態保持オブジェクトである。
  • aux_fmstate は行移動に関連する特殊なケースで使用されるため、更新処理のフローを追う際は注意が必要である。
