dsa_area

定義

  • 種別: struct
  • 定義箇所: src/include/utils/dsa.h:22

目的

動的共有メモリ（DSM）セグメントを基盤とし、プロセス間で共有可能なヒープメモリ領域を管理するための不透明な構造体です。

シグネチャまたは形状

typedef struct dsa_area dsa_area;

（不透明な構造体として定義されており、詳細は src/backend/utils/mmgr/dsa.c に隠蔽されています）

入力と出力

  • 入力: dsa_create 等で生成された領域のハンドル（dsa_handle）を介してアタッチされます。
  • 出力: dsa_allocate を通じて dsa_pointer（共有メモリ内相対ポインタ）を返します。

前提条件と不変条件

  • dsa_pointer は直接デリファレンスできません。必ず dsa_get_address を経由して現在のプロセスの仮想アドレス空間上のポインタに変換する必要があります。
  • 32bit 環境と 64bit 環境で dsa_pointer のサイズが異なる場合があるため、シリアライズ時には注意が必要です。

アルゴリズムまたは振る舞い

  • 複数の DSM セグメントを束ね、単一のヒープとして抽象化します。
  • 割り当てサイズが 8KB を超える場合とそれ以下の場合で、内部的な管理戦略（ロックの粒度など）が切り替わります。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/access/common/session.c
  • src/backend/access/common/tidstore.c
  • src/backend/executor/execParallel.c
  • src/backend/executor/nodeBitmapHeapscan.c
  • src/backend/lib/dshash.c
  • 呼び出し先:
  • 内部実装として src/backend/utils/mmgr/dsa.c を参照します。
  • 低レベルな管理に storage/dsm.h および port/atomics.h を利用します。

関連データ構造

  • dsa_pointer: 共有メモリ内の相対アドレス。
  • dsa_handle: 領域を識別し、他プロセスがアタッチするために使用する識別子。
  • freepage.h: 内部的なフリーページ管理ロジック。
  • relptr.h: 共有メモリ内ポインタ管理のユーティリティ。

失敗モードと境界条件

  • 共有メモリの枯渇時には割り当てが失敗します。
  • 割り当てサイズによる管理戦略の差異がパフォーマンスに影響を与えるため、チューニング時は dsa.c のコメントを参照してください。

保守担当者向けメモ

  • 本構造体は不透明型であるため、構造体のメンバに直接アクセスせず、dsa.h で提供される公開 API を使用してください。
  • 実装の詳細は src/backend/utils/mmgr/dsa.c にあります。
