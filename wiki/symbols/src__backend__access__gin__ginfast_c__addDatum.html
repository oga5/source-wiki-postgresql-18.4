addDatum

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginfast.c:684

目的

KeyArray 構造体に対して、新しい Datum とその GinNullCategory を追加します。必要に応じて内部配列の容量を動的に拡張します。

シグネチャまたは形状

static void addDatum(KeyArray *keys, Datum datum, GinNullCategory category)

入力と出力

  • **入力**:
  • keys: 操作対象の KeyArray 構造体へのポインタ。
  • datum: 追加する Datum 値。
  • category: 追加する GinNullCategory 値。
  • **出力**: なし（keys が指す構造体の内容が更新されます）。

前提条件と不変条件

  • keys は有効な KeyArray 構造体を指している必要があります。
  • keys->keys および keys->categories は repalloc_array で再割り当て可能なメモリ領域である必要があります。

アルゴリズムまたは振る舞い

1. 現在の要素数 (keys->nvalues) が最大容量 (keys->maxvalues) に達しているか確認します。 2. 達している場合、maxvalues を2倍に拡張し、repalloc_array を用いて keys->keys と keys->categories のメモリ領域を再確保します。 3. 新しい datum と category を配列の末尾（nvalues の位置）に格納します。 4. keys->nvalues をインクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginfast.c 内のコードから呼び出されています（行757など）。
  • **呼び出し先**: repalloc_array（メモリ再割り当て用マクロ）。

関連データ構造

  • KeyArray: Datum の配列、GinNullCategory の配列、および現在の要素数と最大容量を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: repalloc_array が失敗した場合、PostgreSQLの標準的なメモリ管理ルールに従いエラーが発生します。
  • keys が NULL の場合: セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は KeyArray のサイズを倍加させる戦略をとっているため、要素数が増大する際の再割り当てコストは償却計算量として抑えられています。
  • repalloc_array を使用しているため、呼び出し元は KeyArray の初期化時に適切なメモリコンテキストが設定されていることを保証する必要があります。
