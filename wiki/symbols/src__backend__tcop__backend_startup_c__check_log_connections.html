check_log_connections

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/backend_startup.c:1086

目的

GUCパラメータ log_connections の設定値が変更された際に、その妥当性を検証し、後続の適用処理（assign_log_connections）で使用するフラグを準備する。

シグネチャまたは形状

bool check_log_connections(char **newval, void **extra, GucSource source);

入力と出力

  • **入力**:
  • newval: 設定される新しい値（文字列のポインタ）。
  • extra: 検証済みフラグを格納するための出力用ポインタ。
  • source: GUCの設定ソース（使用されていない）。
  • **出力**:
  • 成功時は true を返し、extra に検証済みのフラグ値を格納する。
  • 失敗時は false を返し、必要に応じてエラー詳細を記録する。

前提条件と不変条件

  • newval はカンマ区切りの識別子リストとして解析可能である必要がある。
  • validate_log_connections_options が提供する検証ロジックに準拠すること。
  • 成功時に extra へ割り当てられるメモリは guc_malloc を使用すること。

アルゴリズムまたは振る舞い

1. newval のコピーを作成し、SplitIdentifierString を用いてカンマ区切りのリストに分割する。 2. 分割に失敗した場合はエラー詳細を記録し、メモリを解放して false を返す。 3. validate_log_connections_options を呼び出し、リスト内のオプションを検証して flags を生成する。 4. 一時的な文字列とリストのメモリを解放する。 5. 検証が成功した場合、guc_malloc で extra 用のメモリを確保し、flags を格納して true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/guc_tables.c: GUC設定のフックとして登録されている。
  • **呼び出し先**:
  • SplitIdentifierString: 文字列の分割。
  • validate_log_connections_options: オプションの妥当性検証。
  • guc_malloc: extra 領域の確保。

関連データ構造

  • GucSource: GUC設定のソース種別。
  • log_connections: 接続ログ出力を制御するビットフラグ（extra に格納される）。

失敗モードと境界条件

  • **構文エラー**: SplitIdentifierString が失敗した場合、GUC_check_errdetail を通じてエラーが報告される。
  • **検証エラー**: validate_log_connections_options が false を返した場合、関数は直ちに false を返す。
  • **メモリ不足**: guc_malloc が失敗した場合、関数は false を返す。

保守担当者向けメモ

  • 本関数は assign_log_connections と対で動作する。extra を介して渡されるフラグの型や構造を変更する場合は、両方の関数を更新する必要がある。
  • elemlist のメモリ管理は本関数内で完結しているため、検証失敗時も適切に解放される。
