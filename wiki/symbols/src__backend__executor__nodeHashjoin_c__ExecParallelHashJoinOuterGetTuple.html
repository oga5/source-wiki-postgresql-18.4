ExecParallelHashJoinOuterGetTuple

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHashjoin.c:1057

目的

並列ハッシュ結合において、外部（Outer）リレーションから結合対象のタプルを取得する。

シグネチャまたは形状

TupleTableSlot *ExecParallelHashJoinOuterGetTuple(PlanState *outerNode, HashJoinState *hjstate, uint32 *hashvalue)

入力と出力

  • **入力**:
  • outerNode: 外部リレーションの実行状態（PlanState）。
  • hjstate: ハッシュ結合の実行状態（HashJoinState）。
  • hashvalue: 計算されたハッシュ値を格納するポインタ。
  • **出力**:
  • 取得したタプルを含む TupleTableSlot。取得できない場合は NULL。

前提条件と不変条件

  • hjstate->hj_HashTable が初期化されていること。
  • 並列実行環境下で呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. **単一バッチの場合**: curbatch == 0 かつ nbatch == 1 の場合、outerNode から直接タプルをスキャンする。

  • ExecProcNode でタプルを取得し、ハッシュ値を計算する。
  • ハッシュ値が NULL の場合はスキップし、次のタプルを取得する。
  • 有効なタプルが見つかればそれを返す。

    • 2. **複数バッチの場合**: curbatch < nbatch の場合、共有メモリ上の sts_parallel_scan_next を使用してバッチファイルからタプルを読み込む。

  • 取得した MinimalTuple を hjstate->hj_OuterTupleSlot に格納して返す。

    • 3. **終了処理**: バッチ内のタプルが枯渇した場合、outer_eof を true に設定し NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHashjoin.c 内の並列ハッシュ結合処理フロー（行195, 428, 1058）。
  • **呼び出し先**:
  • ExecProcNode（単一バッチ時）
  • ExecEvalExprSwitchContext（ハッシュ値計算時）
  • sts_parallel_scan_next（複数バッチ時）
  • ExecForceStoreMinimalTuple / ExecClearTuple

関連データ構造

  • HashJoinState: 結合の全体状態を保持。
  • HashJoinTable: ハッシュテーブルおよびバッチ管理情報。
  • TupleTableSlot: タプルを保持するコンテナ。

失敗モードと境界条件

  • 外部リレーションが空の場合、NULL を返す。
  • バッチ処理中にタプルが取得できない場合、outer_eof が設定され NULL を返す。

保守担当者向けメモ

  • 本関数は並列実行専用です。シリアル実行版の ExecHashJoinOuterGetTuple と混同しないよう注意してください。
  • 並列実行時は Barrier 同期との整合性が重要であり、バッチの切り替えタイミングに依存します。
