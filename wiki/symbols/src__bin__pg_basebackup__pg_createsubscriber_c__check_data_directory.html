check_data_directory

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_createsubscriber.c:406

目的

指定されたディレクトリがPostgreSQLのデータベースクラスタデータディレクトリとして有効であるかを検証する。

シグネチャまたは形状

static void check_data_directory(const char *datadir)

入力と出力

  • **入力**: datadir (検証対象のディレクトリパスを示す文字列)
  • **出力**: なし（検証に失敗した場合は pg_fatal を呼び出し、プロセスを終了する）

前提条件と不変条件

  • datadir は有効なパス文字列であること。
  • 実行ユーザーは、指定されたディレクトリおよびその内部の PG_VERSION ファイルに対して読み取り権限を持つこと。

アルゴリズムまたは振る舞い

1. 指定された datadir に対して stat を実行し、ディレクトリの存在を確認する。存在しない場合やアクセスできない場合はエラーで終了する。 2. datadir 内に PG_VERSION ファイルが存在するかを確認する。 3. PG_VERSION ファイルが存在しない場合、指定されたディレクトリはデータベースクラスタではないと判断し、エラーで終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/pg_createsubscriber.c:2403 にて subscriber_dir を引数として呼び出されている。
  • **呼び出し先**: stat (システムコール), pg_log_info, pg_fatal (PostgreSQL共通ロギング関数)。

関連データ構造

  • struct stat: ファイル情報を保持するために使用。

失敗モードと境界条件

  • **ディレクトリ不在**: errno が ENOENT の場合、pg_fatal により「does not exist」エラーを出力。
  • **アクセス権限不足**: stat が失敗し errno が ENOENT 以外の場合、アクセスエラーとして終了。
  • **クラスタディレクトリ不適合**: PG_VERSION ファイルが欠落している場合、データベースクラスタではないと判定し終了。

保守担当者向けメモ

  • 本関数は、pg_createsubscriber が操作対象とするディレクトリが正しいクラスタであることを保証するための基本的なガードレールとして機能する。
  • MAXPGPATH を使用してパスを管理しているため、パス長がこれを超える環境では注意が必要である。
