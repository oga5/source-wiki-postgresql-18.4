xl_dbase_create_wal_log_rec

定義

  • **種別**: struct
  • **定義箇所**: src/include/commands/dbcommands_xlog.h:42

目的

データベース作成操作（CREATE DATABASE）において、WAL（Write-Ahead Logging）ログ方式を採用する際のレコード構造を定義します。

シグネチャまたは形状

typedef struct xl_dbase_create_wal_log_rec
{
    Oid    db_id;
    Oid    tablespace_id;
} xl_dbase_create_wal_log_rec;

入力と出力

  • **入力**: データベース作成時に、対象データベースの識別子（db_id）とテーブルスペースの識別子（tablespace_id）を保持します。
  • **出力**: WAL ログとして書き込まれ、リカバリ時に dbase_redo 関数によって読み取られます。

前提条件と不変条件

  • 本構造体は、データベース作成戦略として XLOG_DBASE_CREATE_WAL_LOG が選択された場合に使用されます。

アルゴリズムまたは振る舞い

1. **書き込み**: src/backend/commands/dbcommands.c において、データベース作成時に本構造体に値を詰め、XLogInsert を介して WAL に記録されます。 2. **読み込み**: XLogReader を通じて WAL から読み出されます。 3. **適用**: dbase_redo 関数がこの構造体を参照し、ログの内容に基づいてデータベースの作成操作を再現します。

呼び出し元と呼び出し先

  • **呼び出し元（書き込み）**: src/backend/commands/dbcommands.c
  • **呼び出し先（読み込み・処理）**:
  • src/backend/access/rmgrdesc/dbasedesc.c（ログのデバッグ出力用）
  • src/backend/commands/dbcommands.c（リカバリ時のリプレイ用）
  • src/backend/postmaster/walsummarizer.c（WAL 要約処理用）

関連データ構造

  • XLOG_DBASE_CREATE_WAL_LOG: 本構造体に関連付けられた WAL レコード型。

失敗モードと境界条件

  • 構造体内の Oid が無効な場合、リカバリ時にデータベースの特定やディレクトリ作成に失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は固定長です。可変長構造体である xl_dbase_drop_rec と混同しないよう注意してください。
  • データベース作成戦略には「ファイルコピー方式」と「WAL ログ方式」の2種類があり、本構造体は後者専用です。リカバリ処理を修正する際は、対象の戦略がどちらであるかを確認してください。
