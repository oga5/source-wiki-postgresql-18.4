AlterConstraintNamespaces

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_constraint.c:1054

目的

指定されたテーブルまたは型に関連付けられた制約の名前空間（connamespace）を一括更新します。

シグネチャまたは形状

void AlterConstraintNamespaces(Oid ownerId, Oid oldNspId, Oid newNspId, bool isType, ObjectAddresses *objsMoved)

入力と出力

  • ownerId: 制約の所有者（テーブルまたは型）の OID。
  • oldNspId: 更新対象の現在の名前空間 OID。
  • newNspId: 変更後の新しい名前空間 OID。
  • isType: ownerId が型（true）かテーブル（false）かを示すフラグ。
  • objsMoved: 処理済みオブジェクトを追跡するための ObjectAddresses 構造体（入出力）。

前提条件と不変条件

  • pg_constraint テーブルに対する RowExclusiveLock が保持されている必要があります。
  • oldNspId と newNspId が異なる場合のみ更新処理が実行されます。

アルゴリズムまたは振る舞い

1. ConstraintRelationId を開き、ConstraintRelidTypidNameIndexId インデックスを使用して、ownerId に関連する制約をスキャンします。 2. スキャンされた各制約について以下を実行します：

  • objsMoved に既に存在する場合はスキップします。
  • 制約の connamespace が oldNspId と一致する場合、そのタプルをコピーし、connamespace を newNspId に更新して CatalogTupleUpdate を呼び出します。
  • InvokeObjectPostAlterHook を呼び出し、変更を通知します。
  • add_exact_object_address を呼び出し、処理済みオブジェクトとして objsMoved に追加します。

    • 3. スキャン終了後、テーブルを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c
  • src/backend/commands/typecmds.c
  • **呼び出し先**:
  • table_open, table_close
  • systable_beginscan, systable_getnext, systable_endscan
  • CatalogTupleUpdate
  • InvokeObjectPostAlterHook
  • add_exact_object_address, object_address_present

関連データ構造

  • pg_constraint: 操作対象のシステムカタログ。
  • ObjectAddresses: 処理済みオブジェクトの追跡用構造体。

失敗モードと境界条件

  • oldNspId が newNspId と等しい場合、更新処理はスキップされます。
  • isType フラグに応じて、conrelid または contypid のいずれか一方が検索キーとして使用されます。

保守担当者向けメモ

  • 本関数は制約自体が名前空間への依存関係を持たないことを前提としており、changeDependencyFor() の呼び出しは行われません。
  • objsMoved を介して重複処理を回避しているため、呼び出し側でこの構造体を適切に初期化・管理する必要があります。
