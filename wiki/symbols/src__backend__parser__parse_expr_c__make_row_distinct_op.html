make_row_distinct_op

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_expr.c:3039

目的

2つの行式（RowExpr）を比較し、それらが「異なる」かどうかを判定する論理式ツリーを構築します。

シグネチャまたは形状

Node *make_row_distinct_op(ParseState *pstate, List *opname, RowExpr *lrow, RowExpr *rrow, int location)

入力と出力

  • **入力**:
  • pstate: パーサーの状態管理構造体。
  • opname: 比較に使用する演算子名（例: IS DISTINCT FROM）。
  • lrow, rrow: 比較対象の行式。
  • location: エラー報告用の構文位置。
  • **出力**: 構築された Node（BoolExpr または Const）。

前提条件と不変条件

  • lrow と rrow の要素数が一致している必要があります。

アルゴリズムまたは振る舞い

1. lrow と rrow の要素数を比較し、不一致であればエラーを報告します。 2. 各要素ペアに対して make_distinct_op を呼び出し、個別の比較式を生成します。 3. 生成された比較式を OR 演算子（BoolExpr）で順次結合します。 4. 行が空（要素数0）の場合は、定数 FALSE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_expr.c:1054（transformExprRecurse 内の DISTINCT 演算子処理箇所）から呼び出されます。
  • **呼び出し先**: make_distinct_op（各要素の比較用）、makeBoolExpr（結果の結合用）、makeBoolConst（空行用）。

関連データ構造

  • RowExpr: 比較対象となる行データ構造。
  • ParseState: 構文解析の状態保持。
  • BoolExpr: 構築される論理式ノード。

失敗モードと境界条件

  • **要素数不一致**: lrow と rrow の要素数が異なると ERRCODE_SYNTAX_ERROR を発生させます。
  • **空の行**: 要素数が0の場合、比較結果として FALSE を返します。

保守担当者向けメモ

  • 本関数は IS DISTINCT FROM 演算子を RowExpr に対して適用する際に使用されます。
  • make_distinct_op が返す個別の比較結果を OR で繋ぐことで、「いずれかの要素が異なれば行全体として異なる」という論理を実装しています。
