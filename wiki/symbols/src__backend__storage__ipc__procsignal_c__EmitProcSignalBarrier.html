EmitProcSignalBarrier

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/procsignal.c:356

目的

全バックエンドプロセスに対してバリアイベントを通知し、同期を促すためのグローバルな同期メカニズムを提供します。

シグネチャまたは形状

uint64 EmitProcSignalBarrier(ProcSignalBarrierType type)

入力と出力

  • **入力**: type (ProcSignalBarrierType) - 発生させるバリアの種類。
  • **出力**: uint64 - 呼び出し時点でのバリア生成番号（generation counter）。

前提条件と不変条件

  • 呼び出し側は、バリアに関連する状態変更が全プロセスから可視であることを保証する責任があります。
  • pg_atomic_fetch_or_u32 を使用しているため、呼び出し前後の操作と順序が厳密に保護されます。

アルゴリズムまたは振る舞い

1. 全 ProcSignalSlot を走査し、指定された type に対応するビットを pss_barrierCheckMask にセットします。 2. 共有メモリ上の psh_barrierGeneration をインクリメントし、現在の生成番号を取得します。 3. 全プロセスに対して SIGUSR1 シグナルを送信します。

  • 各スロットの pss_mutex を取得し、PIDが有効な場合に pss_signalFlags[PROCSIG_BARRIER] を true に設定してシグナルを送信します。

    • 4. 最終的に生成番号を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/dbcommands.c (複数箇所)
  • src/backend/commands/tablespace.c (複数箇所)
  • **呼び出し先**:
  • pg_atomic_fetch_or_u32 (アトミック操作)
  • pg_atomic_add_fetch_u64 (アトミック操作)
  • SpinLockAcquire / SpinLockRelease
  • kill (シグナル送信)

関連データ構造

  • ProcSignalSlot: 各プロセスのシグナル状態を保持する共有メモリ構造体。
  • ProcSignalHeader: 全スロットを管理するヘッダー。

失敗モードと境界条件

  • プロセスが終了済みの場合、そのスロットのPIDは0であり、シグナル送信はスキップされます。
  • 頻繁な呼び出しは全プロセスを中断させるため、パフォーマンス上のオーバーヘッドが大きくなります。

保守担当者向けメモ

  • 本関数は全プロセスを対象とするため、高頻度な実行は避けてください。
  • SIGUSR1 の多重化により、シグナルが連続して届いた場合に古い通知が上書きされる可能性がある点に留意してください。
