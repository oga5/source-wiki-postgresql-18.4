brinGetTupleForHeapBlock

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_revmap.c:193

目的

指定されたヒープブロック番号に対応するBRINインデックス内の要約タプル（BrinTuple）を、リバースレンジマップ（revmap）を使用して検索・取得する。

シグネチャまたは形状

BrinTuple *brinGetTupleForHeapBlock(BrinRevmap *revmap, BlockNumber heapBlk,
                                    Buffer *buf, OffsetNumber *off, Size *size, int mode)

入力と出力

  • **入力**:
  • revmap: アクセス対象のBrinRevmap構造体。
  • heapBlk: 検索対象のヒープブロック番号。
  • buf: 検索結果のタプルを含むバッファ（入出力用）。
  • off: タプルのオフセット番号（出力用）。
  • size: タプルのサイズ（出力用、不要な場合はNULL可）。
  • mode: バッファロックのモード（BUFFER_LOCK_SHARE等）。
  • **出力**:
  • 成功時: 対象のBrinTupleへのポインタ。
  • 失敗時: NULL（該当する範囲が要約されていない、またはタプルが見つからない場合）。

前提条件と不変条件

  • heapBlkは、内部でその範囲の先頭ページ番号に正規化される。
  • revmapページが未作成の場合、検索は即座に終了しNULLを返す。
  • 取得したタプルのbt_blknoが入力のheapBlkと一致することが、有効なタプルであるための条件。

アルゴリズムまたは振る舞い

1. heapBlkを範囲の先頭ブロック番号に正規化する。 2. revmap_get_blknoを用いて、対応するrevmapページ番号を取得する。 3. ループ内で以下を繰り返す:

  • revmapページを読み込み、HEAPBLK_TO_REVMAP_INDEXで該当するTID（ItemPointer）を特定する。
  • TIDが指すインデックスページを読み込み、BRIN_IS_REGULAR_PAGEでページ種別を確認する。
  • ページ内のオフセット*offにあるタプルを取得し、bt_blknoが一致するか検証する。
  • 一致すればそのタプルを返し、一致しなければ並行更新を想定してループを継続する。

    • 4. 整合性チェックとして、ループ内で取得したTIDが前回と同一であればインデックス破損としてエラーを投げる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin.c 内の複数箇所（brin.c:402, 422, 752, 1860, 1938）。
  • **呼び出し先**: revmap_get_blkno, ReadBuffer, LockBuffer, PageGetContents, PageGetItemId, PageGetItem 等。

関連データ構造

  • BrinRevmap: revmapの状態管理構造体。
  • BrinTuple: インデックス内の要約データ構造。
  • RevmapContents: revmapページ内のTID配列。

失敗モードと境界条件

  • **インデックス破損**: 連続するループで同一のTIDが返された場合、ERRCODE_INDEX_CORRUPTEDでエラー終了する。
  • **未要約**: 指定範囲がまだ要約されていない場合、NULLを返す。
  • **並行更新**: ページ取得後にタプルが見つからない場合、並行してdesummarize（要約解除）が発生したとみなし、NULLを返す。

保守担当者向けメモ

  • 本関数は、revmapのキャッシュ（rm_currBuf）を利用して効率化を図っている。
  • ページロックの取得・解放が頻繁に行われるため、ロック順序やデッドロックの可能性に注意すること。
  • mode引数は、呼び出し元がタプルに対してどのような操作（読み取りのみか更新か）を意図しているかに依存する。
