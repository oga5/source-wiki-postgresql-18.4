ExecSeqScanEPQ

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSeqscan.c:192

目的

EvalPlanQual (EPQ) 実行モードにおいて、シーケンシャルスキャンを実行する。同時実行制御下でのタプル再評価を目的とする。

シグネチャまたは形状

TupleTableSlot *ExecSeqScanEPQ(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（シーケンシャルスキャンの実行状態を保持するノード）
  • **出力**: TupleTableSlot *（スキャンによって取得されたタプル、またはスキャン終了時の NULL）

前提条件と不変条件

  • pstate は SeqScanState 型にキャスト可能であること。
  • SeqScanState 内の ss フィールドが適切に初期化されていること。

アルゴリズムまたは振る舞い

1. pstate を SeqScanState にキャストする。 2. ExecScan 関数を呼び出し、スキャンを実行する。

  • アクセスメソッドとして SeqNext を使用する。
  • 再評価メソッドとして SeqRecheck を使用する。

    • 3. ExecScan が返した TupleTableSlot をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecProcNode（scanstate->ss.ps.ExecProcNode に本関数が登録されることで呼び出される）。
  • **呼び出し先**:
  • ExecScan（共通ユーティリティ）
  • SeqNext（AMからのタプル取得）
  • SeqRecheck（タプルの再検証）

関連データ構造

  • SeqScanState: シーケンシャルスキャンの実行状態を管理する構造体。
  • PlanState: 実行計画ノードの共通状態。
  • TupleTableSlot: タプルを保持するためのスロット。

失敗モードと境界条件

  • スキャン対象のテーブルが空の場合、SeqNext を通じて NULL が返される。
  • EPQ 実行中にタプルがロックされている場合や、可視性判定に失敗した場合の挙動は SeqNext および SeqRecheck の実装に依存する。

保守担当者向けメモ

  • 本関数は通常のシーケンシャルスキャンパスとは異なり、EPQ 専用の実行パスである。
  • ExecProcNode に動的に割り当てられるため、変更時は ExecInitSeqScan での登録ロジックと整合性を保つこと。
