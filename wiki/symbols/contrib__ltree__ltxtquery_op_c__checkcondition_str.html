checkcondition_str

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltxtquery_op.c:55

目的

ltree 型のデータに対して、ltxtquery 内の特定の条件（ITEM）が一致するかを判定する照合関数です。

シグネチャまたは形状

static bool checkcondition_str(void *checkval, ITEM *val);

入力と出力

  • **入力**:
  • checkval: CHKVAL 構造体へのポインタ（評価対象の ltree ノードとクエリ文字列を保持）。
  • val: 評価対象のクエリ条件を表す ITEM 構造体。
  • **出力**:
  • bool: 条件に一致するラベルが ltree 内に存在すれば true、そうでなければ false を返します。

前提条件と不変条件

  • checkval は CHKVAL 型としてキャスト可能である必要があります。
  • val に含まれる distance は、CHKVAL 内の operand 文字列の有効なオフセットを指している必要があります。

アルゴリズムまたは振る舞い

1. CHKVAL から評価対象の ltree ノードの全レベルを取得します。 2. クエリ条件 val のフラグ（LVAR_ANYEND による前方一致、LVAR_INCASE による大文字小文字無視）を抽出します。 3. ltree の各レベルをループで走査します。

  • LVAR_SUBLEXEME フラグが設定されている場合、compare_subnode を使用して部分文字列の一致を確認します。
  • それ以外の場合、ltree_label_match を使用してラベルの一致を確認します。

    • 4. いずれかのレベルで一致が見つかった時点で true を返し、全レベルを走査しても一致しなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/ltree/ltxtquery_op.c 内の ltree_execute からコールバックとして呼び出されます。
  • **呼び出し先**:
  • compare_subnode
  • ltree_label_match
  • 注: src/backend/utils/adt/tsvector_op.c に同名のシンボルが存在しますが、これらは別個のコンテキストであり、本関数との直接的な呼び出し関係はソース上確認できません。

関連データ構造

  • CHKVAL: 照合に必要な ltree データとクエリのオペランド文字列を保持するコンテキスト。
  • ITEM: ltxtquery 内の個々の条件要素。
  • ltree_level: ltree の各階層構造。

失敗モードと境界条件

  • ltree が空の場合、ループは実行されず false を返します。
  • val->length が不正な場合、ltree_label_match 内部で予期しない挙動が発生する可能性があります。

保守担当者向けメモ

  • 本関数は ltree_execute のコールバックとして設計されています。ロジックを変更する際は、LVAR_ 系フラグの解釈が ltree_label_match と整合していることを確認してください。
