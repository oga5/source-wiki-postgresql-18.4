pgwin32_dispatch_queued_signals

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/signal.c:119

目的

Windows環境において、キューイングされたシグナルをメインスレッドで安全に処理し、対応するシグナルハンドラを実行する。

シグネチャまたは形状

void pgwin32_dispatch_queued_signals(void)

入力と出力

  • 入力: なし（グローバルなシグナルキュー pg_signal_queue および pg_signal_array を参照）
  • 出力: なし（副作用としてシグナルハンドラの実行およびキューの更新を行う）

前提条件と不変条件

  • pgwin32_signal_event が初期化されていること。
  • pg_signal_crit_sec クリティカルセクションによる排他制御下で実行される。

アルゴリズムまたは振る舞い

1. pg_signal_crit_sec を取得し、UNBLOCKED_SIGNAL_QUEUE() を用いてブロックされていないシグナルを特定する。 2. キュー内の各シグナルに対し、以下の処理を繰り返す：

  • シグナルハンドラを取得し、SIG_DFL の場合はデフォルトハンドラを適用する。
  • キューから当該シグナルをクリアする。
  • ハンドラが有効（SIG_ERR, SIG_IGN, SIG_DFL 以外）な場合：
  • クリティカルセクションを一時的に解放する。
  • sigprocmask でシグナルマスクを適用し、ハンドラを実行する。
  • 再度 sigprocmask でマスクを復元し、クリティカルセクションを再取得する。
  • ハンドラ実行による状態変化を考慮し、ループを再開する。

    • 3. 全ての処理終了後、pgwin32_signal_event をリセットし、クリティカルセクションを解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/port/win32/signal.c (複数箇所)
  • src/backend/port/win32/socket.c (複数箇所)
  • src/backend/port/win32_sema.c
  • src/backend/storage/ipc/waiteventset.c (複数箇所)
  • 呼び出し先:
  • EnterCriticalSection / LeaveCriticalSection
  • sigprocmask
  • ResetEvent
  • 登録されたシグナルハンドラ関数

関連データ構造

  • pg_signal_queue: 受信済みシグナルのビットマスク。
  • pg_signal_array: sigaction 構造体の配列。
  • pgwin32_signal_event: シグナル到着通知用イベントオブジェクト。
  • pg_signal_crit_sec: 同期用クリティカルセクション。

失敗モードと境界条件

  • ハンドラ実行中にシグナルマスクやキューが変更された場合、ループを再開して整合性を保つ。
  • pgwin32_signal_event が NULL の場合、Assert により停止する。

保守担当者向けメモ

  • 本関数はメインスレッドの待機ポイント（CHECK_FOR_INTERRUPTS 等）から呼び出されることを想定している。
  • ハンドラ実行中のクリティカルセクション解放は、デッドロック防止のために必須である。
