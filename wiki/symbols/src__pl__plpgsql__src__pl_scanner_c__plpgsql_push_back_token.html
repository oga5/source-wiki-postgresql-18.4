plpgsql_push_back_token

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_scanner.c:400

目的

構文解析器が先読みしたトークンをスキャナのプッシュバックスタックに戻し、次回の字句解析で再利用可能にする。

シグネチャまたは形状

void plpgsql_push_back_token(int token, YYSTYPE *yylvalp, YYLTYPE *yyllocp, yyscan_t yyscanner)

入力と出力

  • **入力**:
  • token: 戻すトークンの識別子。
  • yylvalp: トークンに関連付けられた値（YYSTYPE）。
  • yyllocp: トークンの位置情報（YYLTYPE）。
  • yyscanner: スキャナの状態を保持するコンテキスト。
  • **出力**: なし。

前提条件と不変条件

  • yyscanner が初期化済みであること。
  • プッシュバックされるトークン数は、スキャナが許容する最大数（MAX_PUSHBACKS = 4）を超えてはならない。

アルゴリズムまたは振る舞い

1. ローカル変数 TokenAuxData auxdata を宣言する。 2. 引数として渡された yylvalp および yyllocp の値を auxdata にコピーする。 3. 現在のスキャナ状態（yyextra）からトークン長 plpgsql_yyleng を取得し、auxdata.leng に格納する。 4. 内部関数 push_back_token を呼び出し、トークンと構築した auxdata をスタックへプッシュする。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内で直接的な呼び出し元は明示されていませんが、plpgsql.h で外部公開されています。
  • **呼び出し先**: push_back_token（同ファイル内の内部関数）。

関連データ構造

  • TokenAuxData: トークンの値、位置情報、長さを保持する構造体。
  • YYSTYPE, YYLTYPE: Bison が生成する標準的なトークン値および位置情報の型。
  • yyscan_t: スキャナの内部状態を保持する不透明ポインタ。

失敗モードと境界条件

  • スタックが満杯の状態で呼び出された場合、内部の push_back_token の挙動に依存する（通常はエラーまたはアサーション失敗）。

保守担当者向けメモ

  • 本関数は push_back_token のラッパーであり、呼び出し元から提供されたポインタ経由のデータを TokenAuxData 形式に変換する役割を担います。
  • トークン長（plpgsql_yyleng）の取得にはグローバルな yyextra を参照しているため、マルチスレッド環境や複数のスキャナインスタンスを扱う際は、コンテキストの整合性に注意してください。
