func_match_argtypes

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_func.c:922

目的

提供された関数候補リスト（raw_candidates）の中から、入力引数の型（input_typeids）に対して暗黙的な型変換（COERCION_IMPLICIT）が適用可能な候補のみを抽出・選別します。

シグネチャまたは形状

int func_match_argtypes(int nargs, Oid *input_typeids, FuncCandidateList raw_candidates, FuncCandidateList *candidates);

入力と出力

  • **入力**:
  • nargs: 引数の数。
  • input_typeids: 入力引数の型OID配列。
  • raw_candidates: 検索された関数候補のリスト。
  • **出力**:
  • *candidates: 適合した候補のみで構成されたリスト（ポインタ経由で返却）。
  • **戻り値**: 適合した候補の数（int）。

前提条件と不変条件

  • raw_candidates が NULL の場合、戻り値は 0 となり、*candidates は NULL に設定されます。
  • can_coerce_type が COERCION_IMPLICIT モードで true を返す候補のみが結果リストに含まれます。

アルゴリズムまたは振る舞い

1. *candidates を NULL で初期化します。 2. raw_candidates リストを走査します。 3. 各候補に対し can_coerce_type を呼び出し、入力型から候補の引数型へ暗黙的に変換可能か判定します。 4. 変換可能な場合、その候補を *candidates リストの先頭に追加し、カウントをインクリメントします。 5. 最終的に適合した候補の総数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_func.c (関数解決プロセス内)
  • src/backend/parser/parse_oper.c (演算子解決プロセス内)
  • **呼び出し先**:
  • can_coerce_type (型変換の可否判定)

関連データ構造

  • FuncCandidateList: 関数候補を保持する連結リスト構造体。

失敗モードと境界条件

  • 引数の数が一致しない、または型変換が不可能な場合は候補から除外されます。
  • 適合する候補が一つも存在しない場合、戻り値は 0 となります。

保守担当者向けメモ

  • 本関数は型解決のフィルタリング層を担っています。候補の順序は raw_candidates の入力順序に依存し、結果リストは逆順で構築されます。
