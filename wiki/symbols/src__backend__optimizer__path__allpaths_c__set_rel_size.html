set_rel_size

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/allpaths.c:359

目的

クエリ内の各リレーション（RelOptInfo）に対して、サイズ見積もり（行数など）の計算および初期パス生成を行う。

シグネチャまたは形状

static void set_rel_size(PlannerInfo *root, RelOptInfo *rel, Index rti, RangeTblEntry *rte)

入力と出力

  • root: プランナのグローバル情報。
  • rel: サイズ見積もり対象のリレーション情報。
  • rti: リレーションの範囲テーブルインデックス。
  • rte: リレーションの範囲テーブルエントリ。
  • 出力: なし（rel 構造体のフィールドを更新、またはパスを割り当てる）。

前提条件と不変条件

  • 非ダミーリレーション（!IS_DUMMY_REL(rel)）の場合、実行終了時に rel->rows > 0 であること。

アルゴリズムまたは振る舞い

1. **制約排除**: RELOPT_BASEREL かつ制約排除により不要と判断された場合、set_dummy_rel_pathlist を呼び出し終了。 2. **継承テーブル**: rte->inh が真の場合、set_append_rel_size を呼び出し。 3. **リレーション種別ごとの分岐**:

  • RTE_RELATION: relkind に応じて外部テーブル、パーティションテーブル（ダミー化）、サンプリング、または通常のテーブル（set_plain_rel_size）を処理。
  • RTE_SUBQUERY, RTE_CTE, RTE_NAMEDTUPLESTORE, RTE_RESULT: それぞれ専用のパス生成関数を呼び出し、即座にパスを構築。
  • RTE_FUNCTION, RTE_TABLEFUNC, RTE_VALUES: それぞれ専用のサイズ見積もり関数を呼び出し。

    • 4. **検証**: 最後に rel->rows が正の値であることをアサーションで確認。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/allpaths.c 内の set_base_rel_sizes および set_append_rel_pathlist から呼び出される。
  • **呼び出し先**: set_dummy_rel_pathlist, set_append_rel_size, set_foreign_size, set_tablesample_rel_size, set_plain_rel_size, set_subquery_pathlist, set_function_size_estimates, set_tablefunc_size_estimates, set_values_size_estimates, set_worktable_pathlist, set_cte_pathlist, set_namedtuplestore_pathlist, set_result_pathlist 等。

関連データ構造

  • PlannerInfo: プランナのコンテキスト。
  • RelOptInfo: 最適化対象のリレーション情報。
  • RangeTblEntry: 範囲テーブルエントリ。

失敗モードと境界条件

  • 未知の rtekind が渡された場合、elog(ERROR) により処理を中断する。
  • パーティションテーブルに対して ONLY 指定がある場合、ダミーリレーションとして処理される。

保守担当者向けメモ

  • 本関数はリレーションの種類に応じて処理を振り分けるディスパッチャの役割を果たす。
  • サブクエリやCTEなど、パス生成を即座に行う必要がある型と、そうでない型が混在しているため、修正時は各 rtekind の処理フローに注意すること。
