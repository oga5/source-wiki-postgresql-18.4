recordSharedDependencyOn

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_shdepend.c:124

目的

2つのオブジェクト間における共有依存関係（Shared Dependency）を pg_shdepend カタログに記録します。

シグネチャまたは形状

void recordSharedDependencyOn(ObjectAddress *depender, ObjectAddress *referenced, SharedDependencyType deptype)

入力と出力

  • **入力**:
  • depender: 依存元オブジェクトのアドレス。
  • referenced: 依存先（参照先）オブジェクトのアドレス。
  • deptype: 依存関係の種別（SharedDependencyType）。
  • **出力**: なし（副作用としてカタログへレコードを挿入）。

前提条件と不変条件

  • depender および referenced の objectSubId は 0 である必要があります（共有依存関係はサブオブジェクトをサポートしません）。
  • ブートストラップ処理モード（IsBootstrapProcessingMode）中は何も行われません。

アルゴリズムまたは振る舞い

1. depender と referenced の objectSubId が 0 であることを検証します。 2. ブートストラップモードであれば即座に終了します。 3. pg_shdepend カタログを RowExclusiveLock モードで開きます。 4. 参照先オブジェクトが IsPinnedObject により固定されている場合、記録をスキップします。 5. 固定されていない場合、shdepAddDependency を呼び出してカタログに依存関係を挿入します。 6. 最後に pg_shdepend カタログを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_shdepend.c 内のラッパー関数
  • src/backend/commands/policy.c
  • **呼び出し先**:
  • table_open / table_close
  • IsPinnedObject
  • shdepAddDependency

関連データ構造

  • ObjectAddress: オブジェクトを識別するための構造体（classId, objectId, objectSubId を保持）。
  • SharedDependencyType: 依存関係の性質を定義する列挙型。
  • pg_shdepend: 共有依存関係を格納するシステムカタログ。

失敗モードと境界条件

  • 参照先オブジェクトが固定（Pinned）されている場合、依存関係は記録されません。
  • ブートストラップ処理中はカタログが存在しない可能性があるため、処理を回避します。

保守担当者向けメモ

  • 本関数は共有オブジェクト間の依存関係管理の入り口です。データベース固有の依存関係（pg_depend）とは管理対象が異なる点に注意してください。
  • objectSubId が 0 であるという制約は、共有依存関係の設計上の不変条件です。
