cidr_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network.c:257

目的

バイナリ形式のデータストリームから cidr 型の値を読み込み、内部表現に変換します。

シグネチャまたは形状

Datum cidr_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して StringInfo 形式のバッファを受け取ります。
  • **出力**: cidr 型の内部表現である Datum を返します。

前提条件と不変条件

  • 入力バッファは network_recv が処理可能な形式である必要があります。
  • cidr 型の不変条件として、マスク長より右側のビットは常に 0 である必要があります。この検証は内部で呼び出される network_recv 内の addressOK によって保証されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から StringInfo バッファを取得します。 2. network_recv 関数を呼び出し、第2引数に true を渡すことで cidr 型としての処理を指示します。 3. network_recv が返した結果を PG_RETURN_INET_P マクロでラップし、Datum として返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: network_recv
  • **呼び出し元**: ソースコード内に直接的な呼び出し元の記述はありません。

関連データ構造

  • StringInfo: バイナリデータストリームを保持する構造体。
  • inet: cidr 型の内部表現として使用される構造体。

失敗モードと境界条件

  • 入力バッファが不正な形式である場合、network_recv 内でエラーが発生します。
  • cidr 型の不変条件（マスク長以降のビットが 0 であること）に違反するデータが入力された場合、network_recv 内の検証処理によりエラーとなります。

保守担当者向けメモ

  • 本関数は cidr 型専用のラッパーであり、実質的なバイナリ解析ロジックは network_recv に集約されています。
  • cidr と inet の違いを制御するフラグとして network_recv に true を渡している点に注意してください。
