PGTYPESdate_fmt_asc

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/datetime.c:167

目的

指定されたフォーマット文字列に従い、date 型の値を文字列に変換して出力バッファに格納します。

シグネチャまたは形状

int PGTYPESdate_fmt_asc(date dDate, const char *fmtstring, char *outbuf);

入力と出力

  • **入力**:
  • dDate: 変換対象の日付（2000年1月1日を基準としたオフセット値）。
  • fmtstring: フォーマットパターンを含む文字列。
  • outbuf: 結果を格納するバッファ（呼び出し元で確保済みであること）。
  • **出力**:
  • 成功時は 0 を返します。
  • メモリ確保に失敗した場合は -1 を返します。

前提条件と不変条件

  • outbuf は fmtstring の内容を保持するのに十分なサイズが必要です。
  • 日付計算は 2000年1月1日を基準として行われます。
  • フォーマットパターン（ddd, dd, mmm, mm, yyyy, yy）は、長さの長い順にマッチングされます。

アルゴリズムまたは振る舞い

1. outbuf に fmtstring をコピーします。 2. dDate を j2date を用いて年・月・日に分解し、PGTYPESdate_dayofweek で曜日を取得します。 3. 定義済みの mapping テーブルに基づき、outbuf 内のパターンを検索します。 4. マッチしたパターンを、対応する日付要素（曜日文字列、月名、数値など）で置換します。

  • 数値置換時は pgtypes_alloc で一時領域を確保し、snprintf でフォーマットして outbuf にコピーします。
  • 文字列置換時は直接コピーします。

    • 5. 処理終了後、確保した一時領域を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c:582
  • src/interfaces/ecpg/test/expected/pgtypeslib-dt_test.c:147
  • **呼び出し先**:
  • j2date
  • date2j
  • PGTYPESdate_dayofweek
  • pgtypes_alloc
  • free
  • snprintf

関連データ構造

  • date: ECPGの日付型（内部的には整数）。
  • mapping[]: フォーマットパターンとコンポーネントIDを対応付ける静的構造体配列。
  • union un_fmt_comb: 置換値を保持する共用体。

失敗モードと境界条件

  • pgtypes_alloc がメモリ確保に失敗した場合、関数は -1 を返して終了します。
  • フォーマット文字列内に未知のパターンが含まれる場合、その部分は置換されずに残ります。

保守担当者向けメモ

  • mapping テーブルの順序は、パターンマッチングの優先順位に影響するため変更時に注意が必要です。
  • pgtypes_alloc で確保したメモリは、各置換処理の終了時に必ず free される設計になっています。
