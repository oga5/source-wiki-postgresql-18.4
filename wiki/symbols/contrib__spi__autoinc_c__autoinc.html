autoinc

定義

  • 種別: function
  • 定義箇所: contrib/spi/autoinc.c:21

目的

テーブルへの INSERT または UPDATE 時に、指定された INT4 型カラムが未設定（NULL または 0）である場合、シーケンスから取得した値で自動的に埋めるトリガー関数。

シグネチャまたは形状

Datum autoinc(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（トリガー実行コンテキスト TriggerData を含む）。トリガー引数として「カラム名」と「シーケンス名」のペアを偶数個受け取る。
  • **出力**: Datum（更新された HeapTuple を返す）。

前提条件と不変条件

  • BEFORE INSERT または BEFORE UPDATE トリガーとして呼び出される必要がある。
  • 対象カラムは INT4 型でなければならない。
  • トリガー引数は、カラム名とシーケンス名のペアとして偶数個指定する必要がある。

アルゴリズムまたは振る舞い

1. **検証**: トリガーが行単位かつ BEFORE イベントで呼び出されているかを確認する。 2. **解析**: トリガー引数を走査し、各カラムの現在の値を確認する。 3. **値の判定**: カラム値が NULL または 0 の場合のみ、シーケンスから値を取得する。 4. **シーケンス取得**: nextval() を呼び出し、戻り値を int32 にキャストする。取得値が 0 の場合は再試行する。 5. **タプル更新**: 変更が必要なカラムがある場合、heap_modify_tuple_by_cols を使用して新しいタプルを構築し、それを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のトリガーマネージャ（直接の呼び出し元情報はソースコード上には明示されていない）。
  • **呼び出し先**:
  • SPI_getrelname, SPI_fnumber, SPI_gettypeid, SPI_getbinval (SPI API)
  • nextval (シーケンス関数)
  • heap_modify_tuple_by_cols (タプル操作)
  • palloc, pfree (メモリ管理)

関連データ構造

  • TriggerData: トリガーの実行コンテキスト情報。
  • Trigger: トリガー定義情報（引数 tgargs を保持）。
  • TupleDesc: タプルの記述子。

失敗モードと境界条件

  • **エラー**:
  • トリガーが BEFORE 以外、または行単位以外で呼び出された場合。
  • トリガー引数が偶数でない場合。
  • 指定したカラムが存在しない場合。
  • 指定したカラムが INT4 型でない場合。
  • **境界条件**: nextval() が 0 を返した場合は再試行を行う。

保守担当者向けメモ

  • 本関数は INT4 型専用である。他の型への拡張には型チェックロジックの変更が必要。
  • nextval の戻り値は int64 であるため、int32 へのキャストを行っている点に注意すること。
