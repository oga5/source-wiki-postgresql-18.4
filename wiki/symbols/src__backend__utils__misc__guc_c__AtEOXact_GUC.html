AtEOXact_GUC

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/guc.c:2261

目的

トランザクションまたはサブトランザクションの終了（コミットまたはアボート）時に、GUC（Grand Unified Configuration）パラメータの状態を復元し、スタックを整理する。

シグネチャまたは形状

void AtEOXact_GUC(bool isCommit, int nestLevel)

入力と出力

  • isCommit (bool): true の場合はコミット処理、false の場合はアボート処理として振る舞う。
  • nestLevel (int): 処理対象となるトランザクションの入れ子レベル。
  • 戻り値: なし。

前提条件と不変条件

  • nestLevel > 0 であること。
  • アボート時を除き、nestLevel <= GUCNestLevel であること。
  • guc_stack_list に保持された各パラメータのスタックエントリを処理する。

アルゴリズムまたは振る舞い

1. guc_stack_list を走査し、指定された nestLevel 以上のスタックエントリを持つパラメータを特定する。 2. isCommit が false（アボート）の場合、常に以前の値（prior）へ復元する。 3. isCommit が true（コミット）の場合、スタックの state（GUC_SAVE, GUC_SET, GUC_LOCAL, GUC_SET_LOCAL）に応じて以下の処理を行う：

  • GUC_SAVE: 以前の値へ復元。
  • GUC_SET_LOCAL: マスクされた値を復元。
  • GUC_SET: 現在の値を維持し、スタックエントリを破棄。
  • GUC_LOCAL: 以前の値へ復元。

    • 4. 復元が必要な場合、config_bool や config_int 等の型に応じて assign_hook を呼び出し、変数を更新する。 5. スタックの統合が必要な場合、上位レベルのスタックエントリへ状態をマージする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c (トランザクション管理)
  • src/backend/catalog/index.c (インデックス操作)
  • src/backend/access/brin/brin.c
  • contrib/ 配下の各モジュール (amcheck, dblink, postgres_fdw)
  • **呼び出し先**:
  • discard_stack_value (スタックエントリのメモリ解放)
  • 各パラメータの assign_hook (値変更時のコールバック)
  • set_extra_field (追加情報の更新)

関連データ構造

  • config_generic: GUC変数の基底構造体。
  • GucStack: トランザクションごとの設定変更履歴を保持するスタック構造体。
  • guc_stack_list: アクティブなGUCスタックのリスト。

失敗モードと境界条件

  • nestLevel が不正な場合、Assert により停止する可能性がある。
  • assign_hook がエラーを発生させた場合、トランザクション全体のアボート処理へ伝播する。
  • スタックの統合処理において、prev が存在しない、またはレベルが不連続な場合の特殊な遷移ロジックが存在する。

保守担当者向けメモ

  • 本関数はトランザクションの整合性を維持する極めて重要な箇所である。スタックのマージロジック（switch (stack->state) 内）を変更する際は、README の記述と照らし合わせること。
  • assign_hook の呼び出し順序やメモリ管理（discard_stack_value）に漏れがあると、メモリリークや設定値の不整合を招くため注意が必要。
