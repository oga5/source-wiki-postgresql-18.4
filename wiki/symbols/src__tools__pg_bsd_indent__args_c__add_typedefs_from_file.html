add_typedefs_from_file

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/args.c:334

目的

指定されたファイルから型名リストを読み込み、プログラムの型定義として登録する。

シグネチャまたは形状

void add_typedefs_from_file(const char *str)

入力と出力

  • **入力**: str (読み込むファイルパスを示す文字列)
  • **出力**: なし（副作用として内部的な型定義リストを更新）

前提条件と不変条件

  • 指定されたファイルは読み取り可能である必要がある。
  • ファイル内の各行は、型名として登録される文字列であると想定される。

アルゴリズムまたは振る舞い

1. fopen を使用して指定されたファイルを読み取りモードで開く。 2. ファイルが開けない場合は、エラーメッセージを stderr に出力し、exit(1) でプログラムを終了する。 3. ファイルから fgets を用いて1行ずつ読み込む。 4. 読み込んだ行の末尾にある空白、タブ、改行コードを strcspn で検出し、ヌル文字で置換して除去する。 5. 加工した文字列を add_typename に渡し、型名として登録する。 6. 全行の処理終了後、fclose でファイルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/tools/pg_bsd_indent/args.c:302 (引数解析処理内)
  • **呼び出し先**: add_typename (型名登録用関数)

関連データ構造

  • FILE: ファイル操作に使用。
  • BUFSIZ: 行読み込み用バッファのサイズ定義。

失敗モードと境界条件

  • ファイルが存在しない、または読み取り権限がない場合、exit(1) によりプロセスが異常終了する。
  • ファイルの各行が BUFSIZ を超える場合、fgets の挙動に依存する（行が分割される可能性がある）。

保守担当者向けメモ

  • 本関数は add_typename を呼び出すことで型定義を拡張する。字句解析器（lexi.c）が識別子を型名として認識するために不可欠な処理である。
