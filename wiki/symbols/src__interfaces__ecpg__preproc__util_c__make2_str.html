make2_str

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/util.c:242

目的

2つの文字列を連結し、一時メモリ領域に新しい文字列として生成します。

シグネチャまたは形状

char *make2_str(const char *str1, const char *str2)

入力と出力

  • **入力**:
  • str1: 連結対象の最初の文字列（const char *）
  • str2: 連結対象の2番目の文字列（const char *）
  • **出力**:
  • 連結された結果を保持する新しい文字列へのポインタ（char *）。

前提条件と不変条件

  • str1 および str2 はヌル終端文字列である必要があります。
  • 戻り値のメモリは loc_alloc を介して確保されるため、呼び出し側で free を行う必要はありません。解析サイクル終了時に reclaim_local_storage() によって一括解放されます。

アルゴリズムまたは振る舞い

1. strlen(str1) + strlen(str2) + 1 のサイズで loc_alloc を呼び出し、メモリを確保します。 2. strcpy を使用して str1 を確保したメモリ領域にコピーします。 3. strcat を使用して str2 をその末尾に連結します。 4. 連結後の文字列ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し先**: loc_alloc（一時メモリ確保用）
  • **呼び出し元**: 提供されたソース excerpt 内に具体的な呼び出し元の記述はありません。src/interfaces/ecpg/preproc/preproc_extern.h に外部宣言が存在します。

関連データ構造

  • loc_chunk: loc_alloc が使用する一時メモリ管理用のチャンク構造体。

失敗モードと境界条件

  • loc_alloc がメモリ確保に失敗した場合、loc_alloc 内部の挙動（通常はエラー終了）に従います。
  • 入力文字列が NULL の場合、strlen でセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は loc_alloc を使用するため、戻り値を長期間保持することはできません。解析の節目で reclaim_local_storage() が呼び出されることを前提とした設計です。
