PLyExecutionContext

定義

  • 種別: struct
  • 定義箇所: src/pl/plpython/plpy_main.h:18

目的

PL/Python におけるネストされた関数呼び出しの実行状態を管理し、スタック構造として保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct PLyExecutionContext
{
	PLyProcedure *curr_proc;	/* 現在実行中のプロシージャ */
	MemoryContext scratch_ctx;	/* 型I/O等に使用するメモリコンテキスト */
	struct PLyExecutionContext *next;	/* 前のスタックレベルへのポインタ */
} PLyExecutionContext;

入力と出力

  • 入力: PostgreSQL のハンドラから Python コードが呼び出される際、スタックにプッシュされます。
  • 出力: PLy_current_execution_context() を通じて、現在の実行スタックのトップが参照されます。

前提条件と不変条件

  • scratch_ctx は、実行コンテキストのライフサイクルと同期して管理される必要があります。
  • 再帰的な Python 呼び出しが発生するたびに、新しいコンテキストがスタックにプッシュされます。

アルゴリズムまたは振る舞い

  • PostgreSQL の関数呼び出しから Python インタープリタへの橋渡しを行う際、実行状態をスタックに保持します。
  • Python コード実行中、この構造体を通じて現在のプロシージャ情報や一時的なメモリ領域（scratch_ctx）にアクセスします。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpython/plpy_cursorobject.c、src/pl/plpython/plpy_elog.c、src/pl/plpython/plpy_exec.c、src/pl/plpython/plpy_main.c 内の各関数が PLy_current_execution_context() を介して本構造体を参照しています。
  • 呼び出し先: src/pl/plpython/plpy_main.c の PLy_push_execution_context によりスタック操作が行われます。

関連データ構造

  • PLyProcedure: curr_proc フィールドで参照される、実行中のプロシージャ情報。
  • MemoryContext: scratch_ctx フィールドで参照される、一時的なメモリ割り当て領域。

失敗モードと境界条件

  • メモリ管理: scratch_ctx のライフサイクル管理が不適切である場合、メモリリークが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はスタックとして機能するため、プッシュおよびポップ操作の整合性が重要です。
  • scratch_ctx は型変換などの一時的な処理に利用されるため、コンテキストの終了時に適切に解放されることを確認してください。
