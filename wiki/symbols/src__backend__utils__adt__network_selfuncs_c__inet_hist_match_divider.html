inet_hist_match_divider

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/network_selfuncs.c:956

目的

ネットワークアドレス（inet型）のヒストグラム境界値とクエリ値の間で、演算子に基づく「一致度（非共通ビット数）」を計算し、選択率推定の補正値を算出します。

シグネチャまたは形状

static int inet_hist_match_divider(inet *boundary, inet *query, int opr_codenum)

入力と出力

  • **入力**:
  • boundary: ヒストグラムの境界値（inet *）。
  • query: 検索対象のクエリ値（inet *）。
  • opr_codenum: 演算子の包含関係を示す数値（inet_opr_codenumにより変換済み）。
  • **出力**:
  • int: 境界値とクエリ値の間の非共通ビット数。一致する場合は 0、不適合な場合は -1 を返します。

前提条件と不変条件

  • boundary と query の IP ファミリ（IPv4/IPv6）が一致している必要があります。
  • inet_masklen_inclusion_cmp が 0 を返す（包含関係が成立する）場合にのみ、有効な計算が行われます。

アルゴリズムまたは振る舞い

1. **検証**: IP ファミリが一致し、かつ inet_masklen_inclusion_cmp が包含関係を認めるか確認します。不適合なら -1 を返します。 2. **決定ビットの特定**: opr_codenum に応じて、包含関係の基準となるマスク長（decisive_bits）を決定します。

  • opr_codenum < 0: boundary のマスク長を使用。
  • opr_codenum > 0: query のマスク長を使用。
  • opr_codenum == 0: 両者の最小マスク長を使用。

    • 3. **ビット差分計算**: min_bits（両者のマスク長の最小値）を上限として、bitncommon 関数により共通するビット数を取得します。 4. **結果算出**: decisive_bits から共通ビット数を減算した値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/network_selfuncs.c 内のヒストグラム計算ロジック（663行目、664行目付近）。
  • **呼び出し先**:
  • ip_family, ip_bits, ip_addr（マクロ）
  • inet_masklen_inclusion_cmp
  • Min（マクロ）
  • bitncommon

関連データ構造

  • inet: PostgreSQL のネットワークアドレス型。

失敗モードと境界条件

  • **不適合**: IP ファミリが異なる、または包含関係が成立しない場合、-1 を返します。
  • **ビット長**: min_bits が 0 の場合、decisive_bits をそのまま返します。

保守担当者向けメモ

  • 本関数はヒューリスティックな確率補正用です。包含関係の判定ロジックは inet_masklen_inclusion_cmp に依存しているため、包含判定の仕様変更時は両方の整合性を確認してください。
