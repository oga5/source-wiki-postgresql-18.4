PortalHashTableDeleteAll

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/portalmem.c:606

目的

PortalHashTable に登録されているすべてのポータルを走査し、アクティブなものを除いて破棄（PortalDrop）します。

シグネチャまたは形状

void PortalHashTableDeleteAll(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてハッシュテーブル内のポータルが削除される）

前提条件と不変条件

  • PortalHashTable が初期化されていない場合、関数は何もせず即座に復帰します。
  • 実行中のコマンドに関連付けられたポータル（status == PORTAL_ACTIVE）は削除対象から除外されます。

アルゴリズムまたは振る舞い

1. PortalHashTable が存在しない場合は終了します。 2. hash_seq_init を用いてハッシュテーブルの走査を開始します。 3. 各エントリを順次取得し、ポータルのステータスを確認します。 4. ステータスが PORTAL_ACTIVE でない場合、PortalDrop(portal, false) を呼び出してポータルを破棄します。 5. PortalDrop 実行後、ハッシュテーブルの構造変更に対応するため、hash_seq_term で走査を終了し、再度 hash_seq_init で走査を再開します。 6. すべてのエントリを処理するまで繰り返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/commands/discard.c:69
  • src/backend/commands/portalcmds.c:231
  • 呼び出し先:
  • hash_seq_init
  • hash_seq_search

関連データ構造

  • PortalHashTable: 全ポータルを管理するハッシュテーブル。
  • PortalHashEnt: ハッシュテーブル内のエントリ構造体。
  • Portal: 削除対象となるポータルオブジェクト。

失敗モードと境界条件

  • PortalHashTable が NULL の場合、安全に処理を終了します。
  • PORTAL_ACTIVE 状態のポータルは、削除処理からスキップされるため、実行中のクエリが意図せず中断されることはありません。

保守担当者向けメモ

  • PortalDrop を呼び出すとハッシュテーブルの構造が変化する可能性があるため、走査を再開するアルゴリズムを採用しています。このため、大量のポータルが存在する場合、計算コストが増大する点に留意してください。
