ExecutorStart

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execMain.c:121

目的

PostgreSQL のクエリ実行エンジンにおける初期化フェーズの最上位インターフェースです。クエリ実行に必要な環境の準備と、実行開始時の統計情報の報告を行います。

シグネチャまたは形状

void ExecutorStart(QueryDesc *queryDesc, int eflags)

入力と出力

  • **入力**:
  • queryDesc: 実行対象のクエリ情報（プランツリーやスナップショット等）を保持する構造体。
  • eflags: 実行オプションを指定するビットフラグ。
  • **出力**: なし（void）。副作用として queryDesc 内の実行状態が初期化されます。

前提条件と不変条件

  • queryDesc は有効なポインタである必要があります。
  • pgstat_report_query_id により、クエリ ID が統計情報コレクタに報告されます（既に報告済みの場合は無視されます）。

アルゴリズムまたは振る舞い

1. pgstat_report_query_id を呼び出し、プラン内の queryId を統計情報として報告します。 2. ExecutorStart_hook が設定されている場合、フック関数を呼び出して制御を委譲します。 3. フックが設定されていない場合、標準の standard_ExecutorStart を呼び出し、プランツリーの初期化やリソース確保を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyto.c
  • src/backend/commands/createas.c
  • src/backend/commands/explain.c
  • src/backend/commands/extension.c
  • src/backend/commands/matview.c
  • contrib/auto_explain/auto_explain.c
  • contrib/pg_stat_statements/pg_stat_statements.c
  • **呼び出し先**:
  • pgstat_report_query_id
  • ExecutorStart_hook (存在する場合)
  • standard_ExecutorStart

関連データ構造

  • QueryDesc: 実行対象のプランやステートメントを保持する主要構造体。
  • EState: 実行中のクエリの状態を管理する構造体（standard_ExecutorStart 内で構築される）。

失敗モードと境界条件

  • queryDesc が不正な場合、セグメンテーション違反や予期せぬエラーが発生する可能性があります。
  • フック関数が設定されている場合、その実装に依存した失敗モードが存在します。

保守担当者向けメモ

  • 本関数は実行エンジンの入り口であり、拡張機能（auto_explain 等）が介入するための重要なフックポイントです。
  • デバッグ時には、ExecutorStart_hook が外部モジュールによって上書きされていないか確認してください。
  • 実際のプラン初期化処理の詳細は standard_ExecutorStart または InitPlan を参照してください。
