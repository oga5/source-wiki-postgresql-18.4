entryFindChildPtr

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginentrypage.c:404

目的

GINエントリーツリーの内部ノードにおいて、指定されたブロック番号（blkno）を指すダウンリンクを持つインデックスタプルのオフセット番号を特定します。

シグネチャまたは形状

static OffsetNumber entryFindChildPtr(GinBtree btree, Page page, BlockNumber blkno, OffsetNumber storedOff)

入力と出力

  • **入力**:
  • btree: GIN B-tree構造体。
  • page: 検索対象のページ。
  • blkno: 検索対象のブロック番号。
  • storedOff: 以前に記録されたオフセット番号（ヒントとして使用）。
  • **出力**:
  • 指定された blkno を持つタプルの OffsetNumber。見つからない場合は InvalidOffsetNumber を返します。

前提条件と不変条件

  • 対象の page はリーフページであってはならず（!GinPageIsLeaf(page)）、かつデータページであってもなりません（!GinPageIsData(page)）。

アルゴリズムまたは振る舞い

1. **ヒントの検証**: storedOff が有効な範囲内であれば、まずその位置のタプルを確認し、ダウンリンクが blkno と一致するかチェックします。 2. **右方向の探索**: ヒント位置で一致しない場合、ページ分割等により対象が右側に移動した可能性を考慮し、storedOff + 1 から maxoff まで順次探索します。 3. **全探索**: 上記で見つからない場合、ページ内の先頭（FirstOffsetNumber）から maxoff まで全探索を行います。 4. **戻り値**: 一致するダウンリンクを持つタプルが見つかればそのオフセットを返し、最後まで見つからなければ InvalidOffsetNumber を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginentrypage.c 内の btree->findChildPtr への関数ポインタ代入（761行目）を通じて利用されます。
  • **呼び出し先**:
  • PageGetMaxOffsetNumber, PageGetItem, PageGetItemId (PostgreSQLのページ操作マクロ)
  • GinPageIsLeaf, GinPageIsData, GinGetDownlink (GIN固有のアクセサ)

関連データ構造

  • GinBtree: GINインデックスのB-tree操作を管理する構造体。
  • IndexTuple: GINエントリーツリーに格納されるインデックスタプル。

失敗モードと境界条件

  • ページ内に該当する blkno が存在しない場合、InvalidOffsetNumber を返します。
  • ページが空の場合、maxoff が FirstOffsetNumber 未満となり、ループをスキップして即座に InvalidOffsetNumber を返します。

保守担当者向けメモ

  • 本関数はエントリーツリーの内部ノード専用です。データページやリーフページに対して呼び出さないよう注意してください。
  • 削除操作が行われないエントリーツリーの特性を前提としており、ヒント情報（storedOff）を活用することで効率的な探索を行っています。
