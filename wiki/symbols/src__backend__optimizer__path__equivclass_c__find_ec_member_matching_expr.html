find_ec_member_matching_expr

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/equivclass.c:915

目的

指定された等価クラス（EquivalenceClass）内に、特定の式（Expr）と一致するメンバーが存在するかを検索します。

シグネチャまたは形状

EquivalenceMember *find_ec_member_matching_expr(EquivalenceClass *ec, Expr *expr, Relids relids)

入力と出力

  • **入力**:
  • ec: 検索対象の等価クラス。
  • expr: 検索対象の式。
  • relids: 検索対象のリレーションIDセット。
  • **出力**:
  • 一致する EquivalenceMember 構造体へのポインタ。見つからない場合は NULL。

前提条件と不変条件

  • 入力された expr および等価クラス内の各メンバー式において、RelabelType ノードはバイナリ互換として無視されます。
  • 定数（em_is_const が真）であるメンバーは検索対象から除外されます。

アルゴリズムまたは振る舞い

1. 入力 expr から RelabelType を再帰的に剥がし、比較用のベース式を特定します。 2. setup_eclass_member_iterator を用いて、等価クラス内のメンバーを反復処理します。 3. 各メンバーに対し以下の判定を行います：

  • 定数メンバーはスキップします。
  • 子メンバー（em_is_child が真）の場合、そのメンバーのリレーションIDが relids のサブセットでない限りスキップします。

    • 4. メンバーの式からも RelabelType を剥がし、equal() 関数を用いて入力式と一致するかを比較します。 5. 一致した最初のメンバーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/equivclass.c
  • src/backend/optimizer/plan/createplan.c
  • **呼び出し先**:
  • setup_eclass_member_iterator
  • eclass_member_iterator_next
  • equal (ノード比較用)

関連データ構造

  • EquivalenceClass: 等価な式の集合。
  • EquivalenceMember: 等価クラス内の個別の式。
  • RelabelType: 型変換ノード（比較時に無視される）。

失敗モードと境界条件

  • 該当するメンバーが存在しない場合は NULL を返します。
  • relids が空の場合、子メンバーは一致条件を満たさないため除外されます。

保守担当者向けメモ

  • 本関数は RelabelType を透過的に扱うため、式ツリーの構造が異なる場合でも意味的に等価であればマッチします。
  • em_is_child の判定ロジックは、継承テーブル等の子リレーションを扱う際に重要です。
