HASHBUCKET

定義

  • 種別: typedef
  • 定義箇所: src/backend/utils/hash/dynahash.c:131

目的

ハッシュテーブル内の各バケット（連結リスト）の先頭要素、またはリスト内の各要素を指すポインタ型として機能します。

シグネチャまたは形状

typedef HASHELEMENT *HASHBUCKET;

入力と出力

  • **入力**: HASHELEMENT 構造体へのポインタ。
  • **出力**: HASHBUCKET 型の変数（特定のハッシュバケットの先頭、またはリスト内の要素を指す）。

前提条件と不変条件

  • HASHELEMENT 構造体へのポインタであることが前提です。
  • ハッシュテーブルのバケット配列（HASHSEGMENT）は、この HASHBUCKET 型の配列として管理されます。

アルゴリズムまたは振る舞い

  • ハッシュテーブルにおけるチェイン法（連結リスト）の各ノードを指すために使用されます。
  • hash_search 等の関数内で、特定のバケットを走査する際のイテレータとして利用されます。
  • HASHSEGMENT は HASHBUCKET の配列として定義されており、ハッシュテーブルのディレクトリ構造を構成します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • get_hash_entry (内部関数)
  • hash_search 関連の内部ロジック
  • expand_table 関連の内部ロジック
  • **呼び出し先**:
  • HASHELEMENT 構造体（この型が指し示す実体）

関連データ構造

  • HASHELEMENT: HASHBUCKET が指し示す実際のデータ要素。
  • HASHSEGMENT: HASHBUCKET の配列として定義されるセグメント構造。
  • HTAB: ハッシュテーブルの制御構造体。

失敗モードと境界条件

  • NULL ポインタ: バケットが空の場合、HASHBUCKET は NULL を保持します。
  • メモリ配置: HASHSEGMENT の割り当て時に MAXALIGN を用いて HASHBUCKET のサイズが調整されます。

保守担当者向けメモ

  • HASHBUCKET は単なる HASHELEMENT* のエイリアスです。コード内で HASHBUCKET * と記述されている場合は、バケットの先頭ポインタそのものを指す二重ポインタ（HASHELEMENT **）として機能している点に注意してください。
