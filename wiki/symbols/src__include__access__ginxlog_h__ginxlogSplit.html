ginxlogSplit

定義

  • 種別: struct
  • 定義箇所: src/include/access/ginxlog.h:111

目的

GINインデックスのページ分割操作をWAL（Write-Ahead Log）に記録し、リカバリ時にツリー構造を再構築するための状態を保持します。

シグネチャまたは形状

typedef struct ginxlogSplit
{
    RelFileLocator locator;
    BlockNumber rrlink;         /* 右リンク、またはルート分割時のルートブロック番号 */
    BlockNumber leftChildBlkno; /* 非リーフ分割時に有効 */
    BlockNumber rightChildBlkno;
    uint16      flags;          /* GIN_SPLIT_* フラグ */
} ginxlogSplit;

入力と出力

  • **入力**: GINインデックス更新処理において、ページ分割が発生した際に本構造体にメタデータを格納し、XLogRegisterData を通じてWALレコードとして書き出します。
  • **出力**: リカバリ処理（gin_redo）において、XLogRecGetData を介してWALレコードから本構造体を読み取り、インデックスページの更新に利用します。

前提条件と不変条件

  • 本構造体はWALレコードのペイロードとして使用されるため、フィールド順序とサイズはバイナリ互換性を維持するために厳密に管理されています。
  • 構造体定義へのフィールド追加時は、アライメントによるパディングがWALのバイナリ互換性に影響しないよう注意が必要です。

アルゴリズムまたは振る舞い

1. **記録**: GINインデックスの分割時、分割後のページ状態（右リンクや子ブロック番号など）を本構造体に設定し、WALに永続化します。 2. **再生**: リカバリ時、gin_redo がWALレコードを読み取り、本構造体の情報に基づいて対象ページの分割状態を再現します。flags フィールド（例: GIN_SPLIT_ROOT）により、ルート分割か否か等の詳細な処理分岐が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginbtree.c（WALへのデータ登録）
  • **呼び出し先**: src/backend/access/gin/ginxlog.c（REDO処理でのデータ読み取り）、src/backend/access/rmgrdesc/gindesc.c（WAL内容のデコード・表示）

関連データ構造

  • RelFileLocator: 対象インデックスの識別子。
  • BlockNumber: GINインデックス内のブロック位置。
  • ginxlogInsert: GINの挿入操作を記述する関連構造体。

失敗モードと境界条件

  • WALレコードの読み取り時に、レコードサイズが sizeof(ginxlogSplit) と一致しない場合、データ破損や不正なポインタ参照が発生する可能性があります。
  • flags に定義外の値が含まれる場合、REDO処理が予期しない動作をする可能性があります。

保守担当者向けメモ

  • 本構造体は memcpy で直接読み書きされるため、構造体メンバの変更はWALのバイナリ互換性を破壊します。
  • pg_waldump 等で内容を確認する際は、src/backend/access/rmgrdesc/gindesc.c のデコードロジックが本構造体の flags を正しく解釈しているか確認してください。
