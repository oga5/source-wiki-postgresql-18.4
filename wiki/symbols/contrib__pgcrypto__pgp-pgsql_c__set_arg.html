set_arg

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pgsql.c:167

目的

PGP暗号化/復号化処理の設定オプション（キーと値のペア）を解析し、PGP_Context構造体またはテスト用のdebug_expect構造体に反映させる。

シグネチャまたは形状

static int set_arg(PGP_Context *ctx, char *key, char *val, struct debug_expect *ex);

入力と出力

  • **入力**:
  • ctx: 設定を適用するPGP_Contextへのポインタ。
  • key: 設定項目名（文字列）。
  • val: 設定値（文字列）。
  • ex: テスト用の期待値保持構造体（NULL可）。
  • **出力**:
  • 成功時は 0 を返す。
  • 不明なキーが指定された場合は PXE_ARGUMENT_ERROR を返す。
  • 各設定関数（pgp_set_*系）がエラーを返した場合は、そのエラーコードを返す。

前提条件と不変条件

  • ctx は有効なPGP_Contextを指している必要がある。
  • key および val はヌル終端文字列であること。

アルゴリズムまたは振る舞い

1. key 文字列を比較し、対応する pgp_set_* 関数を呼び出して ctx に値を設定する。 2. ex が NULL でない場合、debug や expect-* で始まるキーに対しては、ex 構造体の各フィールドを更新する。 3. 該当するキーが見つからない場合は PXE_ARGUMENT_ERROR を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-pgsql.c 内の parse_args 関数から呼び出される。
  • **呼び出し先**:
  • pgp_set_* 系関数（pgp_set_cipher_algo, pgp_set_s2k_mode 等）。
  • pgp_get_* 系関数（pgp_get_cipher_code, pgp_get_digest_code）。
  • atoi（標準ライブラリ）。

関連データ構造

  • PGP_Context: PGP処理のコンテキスト。
  • debug_expect: テストおよびデバッグ用の期待値保持構造体。

失敗モードと境界条件

  • 未知の key が渡された場合、PXE_ARGUMENT_ERROR が返される。
  • val が数値として解釈できない場合、atoi の挙動に依存する（通常は 0 が返される）。
  • ex が NULL の状態で debug や expect-* 系のキーを渡すと、その分岐は無視される。

保守担当者向けメモ

  • 新しい設定オプションを追加する場合は、本関数の if-else ブロックに新しい条件を追加し、対応する pgp_set_* 関数を呼び出すこと。
  • expect-* 系のオプションはユーザー向けドキュメントには記載されないテスト用機能であるため、変更時はテストスイートへの影響を確認すること。
