ExprState

定義

  • 種別: struct
  • 定義箇所: src/include/jit/jit.h:72

目的

PostgreSQL の実行エンジンにおいて、評価対象となる式（Expression）の実行時状態を保持する構造体です。JIT コンパイルの文脈では、式を機械語へ変換する際の入力対象として機能します。

シグネチャまたは形状

ソースコード上では前方宣言（struct ExprState）として参照されています。具体的なメンバ構成は本ヘッダーファイル内では定義されていません。

入力と出力

  • **入力**: JitProviderCompileExprCB 型の関数ポインタにおいて、コンパイル対象の式状態として引数 state を受け取ります。
  • **出力**: なし（本構造体自体はデータ保持用）。

前提条件と不変条件

  • JIT コンパイル処理において、有効な ExprState ポインタが提供される必要があります。
  • 実行エンジンによって適切に初期化された状態である必要があります。

アルゴリズムまたは振る舞い

JIT プロバイダ（LLVM 等）は、この構造体が保持する情報を解析し、対応する式を最適化された機械語コードへ変換します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_subs.c (hstore_subscript_fetch, hstore_subscript_assign)
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/access/heap/heapam_handler.c
  • src/backend/catalog/index.c
  • src/backend/commands/analyze.c
  • src/backend/commands/copyfrom.c
  • src/backend/commands/copyfromparse.c
  • **呼び出し先**:
  • 提供されたソース excerpt からは、本構造体が直接呼び出す関数や、本構造体を操作する具体的な関数は特定できません。

関連データ構造

  • JitProviderCompileExprCB: ExprState を引数に取るコールバック関数型。

失敗モードと境界条件

  • NULL ポインタが渡された場合の挙動については、各プロバイダの実装に依存します。
  • 実行エンジン側で式が正しく構築されていない場合、コンパイル処理が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体の詳細な定義は jit.h ではなく、実行エンジン側のヘッダー（nodes/execnodes.h 等）で定義されている可能性があります。JIT 側の変更を行う際は、実行エンジン側の定義との整合性に注意してください。
