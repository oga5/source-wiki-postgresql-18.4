DictThesaurus

定義

  • **種別**: struct
  • **定義箇所**: src/backend/tsearch/dict_thesaurus.c:52

目的

シソーラス（類語辞典）辞書のメイン構造体です。テキスト検索において、特定のフレーズを別のフレーズに置換するためのルール定義、置換結果、および正規化用のサブ辞書への参照を保持します。

シグネチャまたは形状

typedef struct {
    Oid                     subdictOid;
    TSDictionaryCacheEntry *subdict;
    TheLexeme              *wrds;
    int                     nwrds;
    int                     ntwrds;
    TheSubstitute          *subst;
    int                     nsubst;
} DictThesaurus;

入力と出力

  • **入力**: 辞書初期化時に設定ファイル（.ths）から読み込まれたデータ。
  • **出力**: thesaurus_lexize 関数において、入力トークン列とマッチした際に置換後のトークン列を生成します。

前提条件と不変条件

  • subdict は、単語の正規化のために利用される外部辞書へのキャッシュエントリです。
  • wrds は正確な一致検索を行うための配列であり、nwrds（現在の単語数）と ntwrds（確保された配列長）によって管理されます。
  • subst は置換結果を格納する配列であり、nsubst で要素数が管理されます。

アルゴリズムまたは振る舞い

  • **初期化**: thesaurus_init によりメモリが割り当てられ、設定ファイルの内容がコンパイルされます。
  • **正規化**: コンパイル過程で subdict を使用して単語の正規化が行われます。
  • **マッチング**: thesaurus_lexize が呼び出されると、現在の単語列が wrds 配列内のルールと照合されます。
  • **置換**: マッチしたフレーズは subst に格納された置換後のトークン列に置き換えられます。

呼び出し元と呼び出し先

  • **呼び出し元**: thesaurus_init、thesaurus_lexize、compileTheLexeme、compileTheSubstitute、findTheLexeme、checkMatch 等の dict_thesaurus.c 内の関数群。
  • **呼び出し先**: palloc0（メモリ確保）、および subdict を介した辞書キャッシュ機能。

関連データ構造

  • TheLexeme: 検索対象の語彙情報を保持する構造体。
  • TheSubstitute: 置換後のフレーズ情報を保持する構造体。
  • LexemeInfo: マッチング処理中に使用される補助構造体。
  • TSDictionaryCacheEntry: サブ辞書への参照。

失敗モードと境界条件

  • 設定ファイルの読み込み失敗や、メモリ不足による palloc0 の失敗が発生する可能性があります。
  • nwrds が ntwrds を超える場合、配列の再割り当てが必要です。
  • 置換対象のフレーズがサブ辞書で正規化できない場合、マッチングに影響が出る可能性があります。

保守担当者向けメモ

  • 本構造体はフレーズ単位の置換を扱うため、単一の単語だけでなく、複数の単語にまたがる状態管理（DictSubState）と密接に関連しています。
  • 辞書のコンパイル時にサブ辞書を呼び出すため、初期化コストが高い点に留意してください。
  • 設定ファイル内の ? はストップワードを表す特殊なマーカーとして扱われます。
