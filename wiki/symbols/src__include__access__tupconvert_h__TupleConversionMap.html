TupleConversionMap

定義

  • 種別: struct
  • 定義箇所: src/include/access/tupconvert.h:24

目的

論理的に等価だが物理構造（列順序や削除済み列の有無）が異なる行型の間で、タプルを変換するためのメタデータおよびワークスペースを保持する。

シグネチャまたは形状

typedef struct TupleConversionMap
{
    TupleDesc   indesc;         /* 変換元行型の記述子 */
    TupleDesc   outdesc;        /* 変換先行型の記述子 */
    AttrMap    *attrMap;        /* 入力フィールドのインデックス（削除列は0） */
    Datum      *invalues;       /* 変換元分解用ワークスペース */
    bool       *inisnull;
    Datum      *outvalues;      /* 変換先構築用ワークスペース */
    bool       *outisnull;
} TupleConversionMap;

入力と出力

  • **入力**: 変換元および変換先の TupleDesc。
  • **出力**: 変換後の HeapTuple または TupleTableSlot。

前提条件と不変条件

  • 変換元と変換先の TupleDesc は、マップの生存期間中破棄されないこと。
  • 物理的に同一な構造の場合、本構造体は生成されず NULL が返される。

アルゴリズムまたは振る舞い

1. attrMap を参照し、入力データの各列を適切な出力位置へマッピングする。 2. 変換先で削除された列には NULL を割り当てる。 3. invalues/inisnull および outvalues/outisnull をワークスペースとして使用し、タプルの分解と再構築を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/tupconvert.c 内の変換生成関数（convert_tuples_by_position 等）。
  • **呼び出し先**: execute_attr_map_tuple および execute_attr_map_slot が本構造体を利用して変換を実行する。

関連データ構造

  • TupleDesc: 行型の記述子。
  • AttrMap: 列の対応関係を定義する構造体。
  • HeapTuple, TupleTableSlot: 変換対象となるデータ形式。

失敗モードと境界条件

  • 変換が不要な場合に本構造体を生成しようとすると冗長なメモリ消費が発生する。
  • 変換マップの生存期間を超えて TupleDesc を参照すると、ダングリングポインタが発生する可能性がある。

保守担当者向けメモ

  • 本構造体は palloc で割り当てられるため、不要になった際は free_conversion_map() で明示的に解放する必要がある。
  • 変換処理は継承やテーブル再構築で多用されるため、変更時はパフォーマンスへの影響を考慮すること。
