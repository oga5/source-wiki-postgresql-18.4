ExecBitmapIndexScanRetrieveInstrumentation

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapIndexscan.c:423

目的

並列クエリ実行時、共有メモリ上のインデックススキャン統計情報（SharedIndexScanInstrumentation）をバックエンドローカルなメモリへコピーし、集約可能な状態にする。

シグネチャまたは形状

void ExecBitmapIndexScanRetrieveInstrumentation(BitmapIndexScanState *node)

入力と出力

  • **入力**: BitmapIndexScanState *node（スキャン状態を保持するノード構造体）
  • **出力**: なし（副作用として node->biss_SharedInfo が更新される）

前提条件と不変条件

  • node->biss_SharedInfo が NULL の場合、本関数は何もせず終了する。
  • 呼び出し時点で node->biss_SharedInfo には有効な共有メモリ上の統計情報が格納されている必要がある。

アルゴリズムまたは振る舞い

1. node->biss_SharedInfo が NULL であれば即座にリターンする。 2. SharedIndexScanInstrumentation 構造体のサイズを計算する（ヘッダーサイズ + ワーカー数分の IndexScanInstrumentation 配列サイズ）。 3. 計算したサイズ分を palloc でバックエンドローカルメモリに確保する。 4. memcpy を使用して、共有メモリ上の統計情報を確保したローカルメモリへコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:1084 から呼び出されることが確認されている。
  • **呼び出し先**: 本関数内から直接呼び出される関数はない。

関連データ構造

  • BitmapIndexScanState: スキャンノードの状態を管理する構造体。
  • SharedIndexScanInstrumentation: 並列ワーカー間で共有されるインデックススキャンの統計情報構造体。

失敗モードと境界条件

  • SharedInfo が NULL の場合は安全に無視される。
  • メモリ確保（palloc）が失敗した場合は、PostgreSQLの標準的なエラーハンドリングにより例外が発生する。

保守担当者向けメモ

  • 本関数は並列クエリの統計収集プロセスの一部である。共有メモリからローカルメモリへのコピーを行うため、並列ワーカーの数が増加してもメモリ消費量は線形に増加する点に注意すること。
