hash_uint32_extended

定義

  • 種別: function
  • 定義箇所: src/include/common/hashfn.h:48

目的

32ビット整数に対して、シード値を用いたハッシュ計算を行い、その結果を Datum 型として返します。ハッシュ衝突攻撃を回避するために実行ごとに異なるシード値を指定する用途に適しています。

シグネチャまたは形状

static inline Datum hash_uint32_extended(uint32 k, uint64 seed)

入力と出力

  • **入力**:
  • k (uint32): ハッシュ化対象の32ビット整数。
  • seed (uint64): ハッシュ計算に使用するシード値。
  • **出力**:
  • Datum: 計算されたハッシュ値を UInt64GetDatum で変換した結果。

前提条件と不変条件

  • 本関数はインライン関数として定義されており、呼び出し側のオーバーヘッドを最小化します。
  • 返される32ビットのハッシュ値は、ハッシュテーブルの性能を維持するため、均等にランダムに分布していることが期待されます。

アルゴリズムまたは振る舞い

1. 内部関数 hash_bytes_uint32_extended(k, seed) を呼び出し、指定されたシード値に基づいたハッシュ値を計算します。 2. 計算結果を UInt64GetDatum マクロで Datum 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し先**: hash_bytes_uint32_extended
  • **呼び出し元**:
  • src/backend/access/brin/brin_bloom.c
  • src/backend/access/hash/hashfunc.c (複数のデータ型に対するハッシュ関数として利用)
  • src/backend/utils/adt/acl.c
  • src/backend/utils/adt/bool.c
  • src/backend/utils/adt/date.c
  • src/backend/utils/adt/multirangetypes.c

関連データ構造

  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • 特筆すべき失敗モードは定義されていません。
  • シード値に 0 を指定した場合の挙動は、呼び出し元の実装（例: acl.c）に依存します。

  • 頻繁に呼び出されるループ内での使用に適した設計となっています。
