MemoizeKey

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeMemoize.c:105

目的

パラメータ化されたプランノードの実行結果をキャッシュする際、ハッシュテーブルのキーとして機能し、同時にLRU（Least Recently Used）リストのノードとしてメモリ管理を補助する。

シグネチャまたは形状

typedef struct MemoizeKey
{
    MinimalTuple params;
    dlist_node   lru_node;
} MemoizeKey;

入力と出力

  • **入力**: キャッシュ検索時に使用されるパラメータ（MinimalTuple）。
  • **出力**: ハッシュテーブルのキーとして、MemoizeHash_hash および MemoizeHash_equal を通じてキャッシュヒット判定に使用される。

前提条件と不変条件

  • lru_node は、キャッシュエントリのメモリ管理（LRUリスト）において、前後のキーへのポインタを保持するために使用される。
  • ハッシュテーブルのキーとして使用されるため、MemoizeHash_hash および MemoizeHash_equal と整合性が取れている必要がある。

アルゴリズムまたは振る舞い

  • キャッシュ検索時、現在のパラメータと一致する MemoizeKey をハッシュテーブルから探索する。
  • メモリ制限に達した際、lru_node を用いてLRUリストを操作し、古いエントリを破棄する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeMemoize.c 内のハッシュテーブル操作関数（MemoizeHash_hash, MemoizeHash_equal など）。
  • **呼び出し先**: lib/ilist.h（dlist_node の管理用）。

関連データ構造

  • MemoizeEntry: キャッシュのハッシュテーブルに格納される実体。
  • dlist_node: LRUリスト管理のための双方向リスト構造。
  • MinimalTuple: パラメータ値を保持するタプル形式。

失敗モードと境界条件

  • メモリ不足時: cache_reduce_memory が呼び出され、LRUリストに基づいてエントリが削除される。
  • ハッシュ衝突: MemoizeHash_equal が呼び出され、キーの等価性が検証される。

保守担当者向けメモ

  • simplehash.h のリサイズ発生時、MemoizeKey 内の lru_node が正しく維持されるよう注意すること。
  • MemoizeKey はハッシュテーブルのキーとしてポインタ経由で参照されることが多いため、メモリレイアウトの変更には慎重を期すこと。
