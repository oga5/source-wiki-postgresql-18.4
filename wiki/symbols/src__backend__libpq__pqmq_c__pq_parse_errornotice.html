pq_parse_errornotice

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqmq.c:215

目的

シリアライズされたエラーまたは通知メッセージ（StringInfo）を解析し、ErrorData 構造体へ再構築します。主に並列クエリや論理レプリケーションにおいて、ワーカープロセスから送信されたエラー情報をリーダープロセス側で再現するために使用されます。

シグネチャまたは形状

void pq_parse_errornotice(StringInfo msg, ErrorData *edata)

入力と出力

  • **入力**:
  • msg: 解析対象のメッセージデータを含む StringInfo。
  • edata: 解析結果を格納する ErrorData 構造体へのポインタ。
  • **出力**: なし（edata が更新されます）。

前提条件と不変条件

  • edata は関数内で MemSet により 0 クリアされ、elevel はデフォルトで ERROR に設定されます。
  • edata->assoc_context は呼び出し時の CurrentMemoryContext に設定されます。
  • 入力 msg は、libpq プロトコルのエラー通知形式（フィールドコードと文字列値のペア）に従っている必要があります。

アルゴリズムまたは振る舞い

1. edata を初期化します。 2. msg からフィールドコードを 1 バイトずつ読み取ります。 3. コードが \0 の場合は終了します。 4. それ以外の場合、対応する文字列値を読み取り、コードに応じて edata の各フィールド（sqlerrcode, message, detail, hint, cursorpos 等）に値をコピーします。 5. PG_DIAG_SEVERITY_NONLOCALIZED フィールドに基づき、edata->elevel を適切なログレベル（DEBUG1, LOG, INFO, NOTICE, WARNING, ERROR, FATAL, PANIC）にマッピングします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c
  • src/backend/replication/logical/applyparallelworker.c
  • **呼び出し先**:
  • pq_getmsgbyte, pq_getmsgrawstring, pq_getmsgend（プロトコル解析用）
  • pstrdup, pg_strtoint32, elog（メモリ確保および変換・エラー報告用）

関連データ構造

  • StringInfo: メッセージバッファ。
  • ErrorData: エラー情報を保持する構造体。

失敗モードと境界条件

  • 不明なフィールドコードや、不正な SQLSTATE 文字列（5文字以外）、認識できない重大度レベルが検出された場合、elog(ERROR, ...) を呼び出し処理を中断します。

保守担当者向けメモ

  • 本関数はプロトコル仕様に強く依存しています。プロトコル側で新しいエラーフィールドが追加された場合は、本関数の switch 文にも対応するケースを追加する必要があります。
  • PG_DIAG_SEVERITY はローカライズされている可能性があるため無視し、PG_DIAG_SEVERITY_NONLOCALIZED を使用してレベルを判定する設計となっています。
