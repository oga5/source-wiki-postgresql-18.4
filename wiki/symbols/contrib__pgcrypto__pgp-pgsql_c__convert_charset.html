convert_charset

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pgsql.c:67

目的

PostgreSQLのテキストデータに対して、指定された文字コードセット間での変換を行うラッパー関数です。

シグネチャまたは形状

text *convert_charset(text *src, int cset_from, int cset_to)

入力と出力

  • **入力**:
  • src: 変換対象のPostgreSQL text 型データ。
  • cset_from: 変換元のエンコーディングID。
  • cset_to: 変換先のエンコーディングID。
  • **出力**: 変換後の新しい text 型データ。変換が不要な場合は元の src がそのまま返されます。

前提条件と不変条件

  • src は有効なPostgreSQLの text 型データである必要があります。
  • 変換処理にはPostgreSQLの内部エンコーディング変換機能を利用します。

アルゴリズムまたは振る舞い

1. VARSIZE_ANY_EXHDR を用いて入力テキストの長さを取得します。 2. pg_do_encoding_conversion を呼び出し、指定されたエンコーディング間でデータを変換します。 3. 変換後のデータポインタが元のデータポインタと一致する場合（変換不要と判断された場合）、src をそのまま返します。 4. 変換が行われた場合、cstring_to_text を使用して変換後のバイト列を新しい text 型に変換し、一時的な変換バッファを pfree で解放してから結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-pgsql.c 内の convert_from_utf8 および convert_to_utf8 関数から呼び出されます。
  • **呼び出し先**:
  • pg_do_encoding_conversion (PostgreSQLコア関数)
  • cstring_to_text (PostgreSQLコア関数)
  • pfree (PostgreSQLメモリ管理関数)

関連データ構造

  • text: PostgreSQLの可変長文字列型。

失敗モードと境界条件

  • pg_do_encoding_conversion が失敗した場合の挙動は、PostgreSQLコアのエンコーディング変換仕様に依存します。
  • メモリ不足時には pfree や cstring_to_text 内でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は pgcrypto モジュールにおける文字コード変換の共通ユーティリティです。
  • 変換後のバッファは pfree で解放されますが、変換不要と判断された場合に元の src を返すため、呼び出し側で結果を二重に解放しないよう注意してください。
