system_rows_initsamplescan

定義

  • 種別: function
  • 定義箇所: contrib/tsm_system_rows/tsm_system_rows.c:163

目的

SYSTEM_ROWS テーブルサンプルメソッドにおけるスキャン状態の初期化を行います。

シグネチャまたは形状

static void system_rows_initsamplescan(SampleScanState *node, int eflags)

入力と出力

  • **入力**:
  • node: SampleScanState 構造体へのポインタ。
  • eflags: スキャン実行時のフラグ（本関数内では直接使用されません）。
  • **出力**: なし（node->tsm_state にメモリを割り当てて格納します）。

前提条件と不変条件

  • node は有効な SampleScanState 構造体を指している必要があります。
  • 割り当てられた SystemRowsSamplerData の step メンバは、初期化直後に 0 となることが保証されます。

アルゴリズムまたは振る舞い

1. palloc0 を使用して SystemRowsSamplerData 構造体分のメモリをゼロクリアして確保します。 2. 確保したメモリ領域を node->tsm_state に代入します。

呼び出し元と呼び出し先

  • **呼び出し元**: TsmRoutine 構造体の InitSampleScan フィールドを通じて、PostgreSQL の実行エンジンから呼び出されます。
  • **呼び出し先**: palloc0（メモリ確保用ユーティリティ）。

関連データ構造

  • SampleScanState: スキャン実行時の状態を保持する構造体。
  • SystemRowsSamplerData: 本関数によって確保される、SYSTEM_ROWS メソッド固有のスキャン状態管理構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。

保守担当者向けメモ

  • 本関数は tsm_system_rows_handler 内で TsmRoutine に登録されます。
  • step が 0 で初期化される点は、後続の system_rows_nextsampleblock 等で状態が正しく設定されるための前提となります。
