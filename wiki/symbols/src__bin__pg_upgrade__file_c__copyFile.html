copyFile

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/file.c:81

目的

pg_upgrade において、データディレクトリ間でファイルを物理的にコピーする汎用的な転送処理を提供します。

シグネチャまたは形状

void copyFile(const char *src, const char *dst, const char *schemaName, const char *relName)

入力と出力

  • **入力**:
  • src: コピー元ファイルのパス。
  • dst: コピー先ファイルのパス。
  • schemaName: エラーログ出力用のスキーマ名。
  • relName: エラーログ出力用のリレーション名。
  • **出力**: なし（成功時はファイルを複製し、失敗時は pg_fatal によりプロセスを終了します）。

前提条件と不変条件

  • src ファイルは読み取り可能である必要があります。
  • dst ファイルは新規作成されるか、存在しない状態である必要があります（O_EXCL フラグを使用）。
  • Windows 環境以外では、pg_file_create_mode に基づくパーミッションでファイルが作成されます。

アルゴリズムまたは振る舞い

  • **Windows 以外**:

    • 1. src を読み取り専用、dst を書き込み専用（新規作成・排他）で開きます。 2. 50 * BLCKSZ バイトのバッファを確保し、チャンク単位でデータを読み書きします。 3. 読み込みまたは書き込みに失敗した場合、pg_fatal を呼び出してエラーメッセージ（システムエラー含む）を出力し終了します。 4. 完了後、バッファを解放し、両方のファイル記述子を閉じます。

  • **Windows**:

    • 1. CopyFile API を使用してファイルを複製します。 2. 失敗した場合は GetLastError を通じてエラーをマッピングし、pg_fatal で終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/relfilenumber.c:630 からの呼び出しが確認されています。
  • **呼び出し先**: OS のファイル操作関数（open, read, write, close）または Windows API (CopyFile) を直接呼び出します。

関連データ構造

  • BLCKSZ: バッファサイズ計算に使用されるブロックサイズ定数。
  • pg_file_create_mode: ファイル作成時のパーミッション定義。

失敗モードと境界条件

  • ファイルのオープン、読み込み、書き込みのいずれかが失敗した場合、pg_fatal により即座に処理が中断されます。
  • 書き込み時に errno が設定されない場合、ディスク容量不足（ENOSPC）とみなしてエラーを報告します。
  • Windows 環境において CopyFile が失敗した場合、システムエラーコードに基づいた詳細なエラーメッセージが出力されます。

保守担当者向けメモ

  • 本関数はプラットフォーム依存の分岐（#ifndef WIN32）を含んでおり、OS 固有の挙動を抽象化しています。
  • 大容量ファイルのコピー効率を考慮し、固定サイズのバッファリングを行っています。
