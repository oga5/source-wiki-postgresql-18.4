PGTYPESnumeric_from_asc

定義

  • src/interfaces/ecpg/pgtypeslib/numeric.c:320

目的

ASCII 文字列を PostgreSQL の numeric 構造体に変換し、動的に割り当てられたメモリを返します。

シグネチャまたは形状

numeric *PGTYPESnumeric_from_asc(char *str, char **endptr)

入力と出力

  • **入力**:
  • str: 変換対象の文字列。
  • endptr: 解析終了位置を受け取るポインタ（NULL 指定可能）。
  • **出力**:
  • 成功時: 変換された numeric 構造体へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • 呼び出し元は、返された numeric 構造体に対して PGTYPESnumeric_free を呼び出し、メモリを解放する責任があります。
  • str は有効な数値形式である必要があります。

アルゴリズムまたは振る舞い

1. pgtypes_alloc を使用して numeric 構造体用のメモリを確保します。 2. endptr が NULL の場合は内部ポインタ realptr を使用し、解析位置を追跡します。 3. set_var_from_str を呼び出し、文字列を解析して numeric 構造体の内部表現（weight, digits 等）を構築します。 4. 解析が失敗した場合は、確保したメモリを PGTYPESnumeric_free で解放し、NULL を返します。 5. 成功した場合は、構築された numeric 構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c

  • **呼び出し先**:

  • PGTYPESnumeric_free

関連データ構造

  • numeric: 数値の内部表現を保持する構造体。

失敗モードと境界条件

  • メモリ割り当て（pgtypes_alloc）が失敗した場合、NULL を返します。
  • set_var_from_str による解析が失敗した場合、メモリを解放した上で NULL を返します。

保守担当者向けメモ

  • 本関数は numeric 構造体のメモリ確保を伴うため、呼び出し側でのメモリ管理漏れに注意してください。
  • 内部で使用している set_var_from_str は、文字列の解析と numeric 構造体への値の設定を担うコア関数です。
