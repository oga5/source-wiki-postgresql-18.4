calc_hist_selectivity_contained

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_selfuncs.c:1017

目的

範囲型に対する「含まれる（contained by）」演算子（@> の逆）の選択率を、境界ヒストグラムと長さヒストグラムを用いて推定する。

シグネチャまたは形状

static double calc_hist_selectivity_contained(
    TypeCacheEntry *typcache,
    const RangeBound *lower, RangeBound *upper,
    const RangeBound *hist_lower, int hist_nvalues,
    Datum *length_hist_values, int length_hist_nvalues
)

入力と出力

  • **入力**:
  • typcache: 型固有のキャッシュ情報。
  • lower, upper: クエリ範囲の境界。
  • hist_lower: 範囲の下限値ヒストグラムの配列。
  • hist_nvalues: ヒストグラムの要素数。
  • length_hist_values: 範囲の長さヒストグラムの配列。
  • length_hist_nvalues: 長さヒストグラムの要素数。
  • **出力**:
  • double: 推定された選択率（0.0 から 1.0 の範囲）。

前提条件と不変条件

  • 範囲の「長さ」と「下限値」の分布は独立していると仮定する。
  • hist_lower は昇順にソートされている必要がある。
  • upper_index が負の場合、クエリ範囲はヒストグラムの範囲外とみなされ 0.0 を返す。

アルゴリズムまたは振る舞い

1. **初期探索**: クエリ範囲の upper を含むヒストグラム上のビン（upper_index）を二分探索で特定する。 2. **境界補正**: ヒストグラムの範囲外にある場合は 0.0 を返す。 3. **ループ処理**: upper_index からヒストグラムの先頭に向かって逆順に走査する。

  • 各ビンにおいて、クエリ範囲の upper までの距離を計算する。
  • calc_length_hist_frac を呼び出し、その距離内に収まる長さを持つ範囲の割合を算出する。
  • ビン幅と長さの割合を乗算し、累積する。
  • クエリの lower を含むビンに到達した時点で計算を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/rangetypes_selfuncs.c 内の関数（calc_hist_selectivity 等から参照）。
  • src/backend/utils/adt/multirangetypes_selfuncs.c 内の関数。
  • **呼び出し先**:
  • rbound_bsearch, get_position, get_distance, calc_length_hist_frac, range_cmp_bounds。

関連データ構造

  • RangeBound: 範囲の境界（値、包含フラグ、下限/上限フラグ）。
  • TypeCacheEntry: 型固有の比較関数やサブ差分関数を保持するキャッシュ。

失敗モードと境界条件

  • upper_index < 0: クエリ範囲がヒストグラムの最小値より小さい場合、選択率は 0.0 となる。
  • ヒストグラムの範囲を超える場合: Min(upper_index, hist_nvalues - 2) により、最後のビンとして扱われる。
  • bin_width が負になる場合: 0.0 にクランプされる。

保守担当者向けメモ

  • 本関数は「台形面積」の計算ロジックを用いて累積分布関数を近似している。
  • get_position および get_distance は型キャッシュ内の subdiff 関数に依存しているため、カスタム範囲型を追加する際は subdiff の実装が不可欠である。
