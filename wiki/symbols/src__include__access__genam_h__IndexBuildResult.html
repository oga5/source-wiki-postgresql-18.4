IndexBuildResult

定義

  • 種別: struct
  • 定義箇所: src/include/access/genam.h:53

目的

インデックス構築処理（build）の結果として、処理されたタプル数およびインデックスに挿入されたタプル数を報告するために使用される構造体です。

シグネチャまたは形状

typedef struct IndexBuildResult
{
    double      heap_tuples;    /* 親テーブルで確認されたタプル数 */
    double      index_tuples;   /* インデックスに挿入されたタプル数 */
} IndexBuildResult;

入力と出力

  • **出力**: インデックス構築関数（例: blbuild, ginbuild 等）が構築完了時に呼び出し元へ返す統計情報として利用されます。

前提条件と不変条件

  • heap_tuples および index_tuples は、構築プロセス全体を通じてカウントされた累積値です。
  • 構築処理の成功時に、呼び出し元によってメモリが割り当てられ、値が設定されます。

アルゴリズムまたは振る舞い

インデックス構築の最終段階で、各インデックスアクセス手法（AM）の実装がこの構造体をインスタンス化し、スキャンした親テーブルのタプル数と、実際にインデックスへ書き込んだタプル数を格納して返します。

呼び出し元と呼び出し先

  • **呼び出し元（確認された参照先）**:
  • contrib/bloom/blinsert.c (blbuild)
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/gist/gistbuild.c
  • **呼び出し先**: 構造体であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • Relation: インデックスおよび親テーブルの情報を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: palloc 等によるメモリ確保に失敗した場合、構築処理自体が中断されます。
  • 統計値の不整合: 構築ロジック内でカウントが正しく行われない場合、heap_tuples と index_tuples の値が不正確になる可能性があります。

保守担当者向けメモ

  • 本構造体は、インデックス構築の統計情報を標準化するためのものです。新しいインデックス AM を追加する際は、構築関数で本構造体を使用して結果を返す必要があります。
