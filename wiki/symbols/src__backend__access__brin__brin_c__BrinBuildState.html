BrinBuildState

定義

  • 種別: struct
  • 定義箇所: src/backend/access/brin/brin.c:155

目的

BRIN（Block Range Index）インデックスの構築プロセスにおいて、現在の構築状態、範囲要約情報、およびメモリ管理を保持するために使用されます。

シグネチャまたは形状

typedef struct BrinBuildState {
    Relation        bs_irel;
    double          bs_numtuples;
    double          bs_reltuples;
    Buffer          bs_currentInsertBuf;
    BlockNumber     bs_pagesPerRange;
    BlockNumber     bs_currRangeStart;
    BlockNumber     bs_maxRangeStart;
    BrinRevmap     *bs_rmAccess;
    BrinDesc       *bs_bdesc;
    BrinMemTuple   *bs_dtuple;
    BrinTuple      *bs_emptyTuple;
    Size            bs_emptyTupleLen;
    MemoryContext   bs_context;
    BrinLeader     *bs_leader;
    int             bs_worker_id;
    Tuplesortstate *bs_sortstate;
} BrinBuildState;

入力と出力

  • **入力**: initialize_brin_buildstate 関数により、対象のインデックスリレーション（Relation）等から初期化されます。
  • **出力**: 構築完了後、terminate_brin_buildstate を通じてリソースが解放されます。

前提条件と不変条件

  • インデックス構築プロセス（シリアルまたは並列）の期間中のみ存在します。
  • bs_leader は並列構築時のリーダープロセスにのみ存在します。
  • bs_sortstate は並列構築時にワーカーおよびリーダーによって使用されます。

アルゴリズムまたは振る舞い

  • インデックス構築中、テーブルをスキャンして範囲ごとの要約を生成します。
  • bs_currRangeStart 等を用いて現在のページ範囲を追跡し、bs_dtuple に要約値を蓄積します。
  • 必要に応じて form_and_insert_tuple や form_and_spill_tuple を呼び出し、インデックスページへの書き込みを行います。
  • 並列構築時には _brin_parallel_merge を介してワーカーの結果を統合します。

呼び出し元と呼び出し先

  • **呼び出し元**: brinbuild（インデックス構築の開始点）
  • **呼び出し先**:
  • initialize_brin_buildstate
  • terminate_brin_buildstate
  • form_and_insert_tuple
  • form_and_spill_tuple
  • brin_fill_empty_ranges
  • _brin_begin_parallel / _brin_end_parallel
  • _brin_parallel_heapscan / _brin_parallel_merge

関連データ構造

  • BrinDesc: インデックスの記述子。
  • BrinRevmap: ページ範囲とインデックス位置の対応表。
  • BrinMemTuple: メモリ上のタプル要約。
  • BrinLeader: 並列構築時のリーダー管理構造体。
  • Tuplesortstate: 並列構築時のソート用状態。

失敗モードと境界条件

  • 範囲が未要約の場合、brin_fill_empty_ranges によって空のタプルが挿入される可能性があります。
  • 並列構築時、リーダーとワーカー間での bs_sortstate の共有状態に依存します。

保守担当者向けメモ

  • 本構造体はインデックス構築のライフサイクル全体を管理するため、構築ロジックを変更する際は、bs_context によるメモリ管理と、並列処理時の bs_leader の生存期間に注意してください。
  • bs_emptyTuple は、範囲がデータを含まない場合にデフォルト値として使用されます。
