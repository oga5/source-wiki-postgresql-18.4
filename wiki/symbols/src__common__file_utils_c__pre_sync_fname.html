pre_sync_fname

定義

  • 種別: function
  • 定義箇所: src/common/file_utils.c:358

目的

指定されたファイルに対して、OSのキャッシュからデータをフラッシュするためのヒント（sync_file_range または posix_fadvise）を送信します。

シグネチャまたは形状

int pre_sync_fname(const char *fname, bool isdir)

入力と出力

  • **入力**:
  • fname: 対象ファイルのパス。
  • isdir: ディレクトリであるかを示すフラグ。
  • **出力**:
  • 成功時は 0 を返します。
  • ファイルオープン失敗時、EACCES またはディレクトリに対して EISDIR が発生した場合は 0 を返します。その他のエラー時は -1 を返します。

前提条件と不変条件

  • PG_FLUSH_DATA_WORKS マクロが定義されている環境でのみ有効な処理を実行します。
  • 本関数は「ヒント」として動作するため、内部でのフラッシュ処理失敗はエラーとして扱われません。

アルゴリズムまたは振る舞い

1. fname を読み取り専用（O_RDONLY）で開きます。 2. 開いたファイル記述子に対し、環境に応じて以下のいずれかを使用してデータフラッシュを試みます。

  • HAVE_SYNC_FILE_RANGE が定義されている場合: sync_file_range を使用。
  • USE_POSIX_FADVISE かつ POSIX_FADV_DONTNEED が定義されている場合: posix_fadvise を使用。

    • 3. 実行結果に関わらず、ファイル記述子を閉じて終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/file_utils.c 内の walkdir から複数回呼び出されています。
  • src/backend/storage/file/fd.c 内の walkdir から呼び出されています。
  • src/bin/pg_upgrade/relfilenumber.c から呼び出されています。
  • **呼び出し先**:
  • 標準ライブラリ関数: open, close, sync_file_range (または posix_fadvise)。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • ファイルが存在しない、またはアクセス権限がない場合、エラーログを出力して -1 を返すか、特定の条件下（EACCES 等）では無視して 0 を返します。
  • PG_FLUSH_DATA_WORKS が定義されていない環境では、何も行わず 0 を返します。

保守担当者向けメモ

  • 本関数はバックエンドの pg_flush_data() と同等の役割をフロントエンド向けに提供するものです。
  • sync_file_range 等の呼び出しはヒントであるため、エラーが発生しても処理を中断させない設計となっています。
