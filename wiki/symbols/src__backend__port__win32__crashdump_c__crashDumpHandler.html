crashDumpHandler

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/crashdump.c:89

目的

Windows環境において、プロセスで未処理の例外が発生した際に、メモリ状態をミニダンプファイル（.mdmp）として保存し、事後解析を可能にする。

シグネチャまたは形状

static LONG WINAPI crashDumpHandler(struct _EXCEPTION_POINTERS *pExceptionInfo);

入力と出力

  • **入力**: pExceptionInfo (例外発生時のコンテキスト情報を含む _EXCEPTION_POINTERS 構造体へのポインタ)
  • **出力**: LONG 型のステータスコード（常に EXCEPTION_CONTINUE_SEARCH を返す）

前提条件と不変条件

  • データディレクトリ直下に crashdumps という名前のディレクトリが存在すること。存在しない場合、ダンプ生成処理はスキップされる。
  • dbghelp.dll がシステムに存在し、ロード可能であること。
  • クラッシュコンテキストで実行されるため、PostgreSQL内部のメモリ割り当てやログ出力関数は使用せず、write_stderr のみを使用すること。

アルゴリズムまたは振る舞い

1. crashdumps ディレクトリの存在を確認する。 2. dbghelp.dll を動的にロードし、MiniDumpWriteDump 関数のアドレスを取得する。 3. MiniDumpWriteDump のバージョンに応じてダンプタイプ（MiniDumpNormal 等）を決定する。 4. プロセスIDとシステム時刻に基づいた一意なファイル名を生成する。 5. CreateFile でダンプファイルを作成し、MiniDumpWriteDump を呼び出してメモリ情報を書き出す。 6. 書き込み結果を write_stderr で報告する。 7. 最後に EXCEPTION_CONTINUE_SEARCH を返し、OSのデフォルトの例外処理へ制御を戻す。

呼び出し元と呼び出し先

  • **呼び出し元**: OSの例外フィルタ機構（SetUnhandledExceptionFilter を介して登録）。
  • **呼び出し先**:
  • LoadLibrary, GetProcAddress (Windows API)
  • MiniDumpWriteDump (dbghelp.dll 内の関数)
  • write_stderr (PostgreSQL内部関数)

関連データ構造

  • _EXCEPTION_POINTERS: 例外発生時のスレッドおよび例外情報。
  • _MINIDUMP_EXCEPTION_INFORMATION: MiniDumpWriteDump に渡す例外情報構造体。
  • MINIDUMPWRITEDUMP: MiniDumpWriteDump 関数への関数ポインタ型。

失敗モードと境界条件

  • crashdumps ディレクトリが存在しない場合、何も行わずに終了する。
  • dbghelp.dll のロード失敗や、必要な関数の取得失敗時にはエラーメッセージを出力して終了する。
  • ダンプファイルの作成に失敗した場合は、エラーコードと共にメッセージを出力する。
  • スタックオーバーフローや深刻なメモリ不足時には、本関数自体が正しく動作しない可能性がある。

保守担当者向けメモ

  • 本関数はクラッシュ時に実行されるため、副作用を最小限に抑える必要がある。
  • 生成された .mdmp ファイルの解析には、対応するビルドのデバッグシンボル（PDB）が必須である。
