pglz_compress

定義

  • 種別: function
  • 定義箇所: src/common/pg_lzcompress.c:508

目的

LZ77 アルゴリズムに基づき、入力データを圧縮して出力バッファに格納する。

シグネチャまたは形状

int32 pglz_compress(const char *source, int32 slen, char *dest, const PGLZ_Strategy *strategy);

入力と出力

  • **入力**:
  • source: 圧縮対象のデータポインタ。
  • slen: 入力データのバイト長。
  • dest: 圧縮結果を格納するバッファ。
  • strategy: 圧縮の挙動を制御する PGLZ_Strategy 構造体（NULL の場合はデフォルト戦略を使用）。
  • **出力**:
  • 成功時: 圧縮後のデータ長（int32）。
  • 失敗時: -1。

前提条件と不変条件

  • strategy が指定する match_size_good は 17 以上 PGLZ_MAX_MATCH 以下に調整される。
  • slen が strategy->min_input_size 未満、または strategy->max_input_size を超える場合は圧縮を行わず -1 を返す。
  • 出力バッファのオーバーランを防ぐため、圧縮後のサイズが result_max（戦略に基づく許容最大サイズ）を超える場合は -1 を返す。

アルゴリズムまたは振る舞い

1. **初期化**: 戦略に基づきハッシュテーブルサイズを決定し、履歴テーブルを初期化する。 2. **圧縮ループ**: 入力データを走査し、pglz_find_match を用いて履歴内から一致パターンを検索する。

  • 一致が見つかった場合: タグを出力し、一致長分だけ履歴を更新してポインタを進める。
  • 一致が見つからない場合: リテラルバイトを出力し、履歴を更新してポインタを進める。

    • 3. **終了判定**: 圧縮率が戦略の閾値を下回る場合、または出力が許容サイズを超えた時点で処理を中断し -1 を返す。 4. **完了**: 最終的な制御バイトを書き込み、圧縮後のサイズを返す。

関連データ構造

  • PGLZ_Strategy: 圧縮戦略を定義する構造体。
  • PGLZ_HistEntry: 履歴検索用ハッシュテーブルの要素。

失敗モードと境界条件

  • 入力サイズが戦略の許容範囲外である場合。
  • 圧縮後のサイズが result_max を超える場合。
  • 圧縮効率が低く、first_success_by バイト以内に一致が見つからない場合。

保守担当者向けメモ

  • pglz_hist_add マクロは引数を複数回評価するため、副作用のある式を渡さないこと。
  • 伸長時の安全なコピー処理（memcpy の重なり対応）と整合性を保つ必要がある。
  • 圧縮ループ内での bp - bstart によるサイズチェックは、出力バッファのオーバーランを防止する重要な不変条件である。
