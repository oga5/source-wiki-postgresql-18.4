find_matching_idle_slot

定義

  • 種別: function
  • 定義箇所: src/fe_utils/parallel_slot.c:134

目的

指定されたデータベース名に一致する、現在使用されていない（アイドル状態の）接続済みスロットを ParallelSlotArray 内から検索します。

シグネチャまたは形状

static int find_matching_idle_slot(const ParallelSlotArray *sa, const char *dbname);

入力と出力

  • **入力**:
  • sa: 検索対象の ParallelSlotArray 構造体へのポインタ。
  • dbname: 検索対象のデータベース名（文字列）。NULL の場合は、接続済みの任意のアイドルスロットを対象とします。
  • **出力**:
  • 一致するスロットのインデックス（0 以上）。
  • 一致するものがない場合は -1。

前提条件と不変条件

  • sa は有効な ParallelSlotArray を指している必要があります。
  • 検索対象のスロットは inUse が false かつ connection が NULL でない状態である必要があります。

アルゴリズムまたは振る舞い

1. sa->numslots 分だけスロット配列を反復処理します。 2. 各スロットにおいて以下を順次確認します。

  • inUse が true の場合はスキップします。
  • connection が NULL の場合はスキップします。
  • dbname が NULL であるか、または PQdb で取得した接続先データベース名が dbname と一致する場合、そのインデックスを返します。

    • 3. ループ終了まで一致が見つからない場合、-1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/parallel_slot.c:381（ParallelSlotsGetIdle 内での利用が確認されています）。
  • **呼び出し先**: PQdb（libpq関数）、strcmp（標準Cライブラリ）。

関連データ構造

  • ParallelSlotArray: スロットの配列とメタデータを保持する構造体。
  • ParallelSlot: 個別の接続状態を保持する構造体。

失敗モードと境界条件

  • sa が空、またはすべてのスロットが使用中、あるいは接続が確立されていない場合は -1 を返します。
  • dbname が NULL の場合、接続済みであれば最初のアイドルスロットを返します。

保守担当者向けメモ

  • 本関数は ParallelSlotsGetIdle の内部ロジックの一部であり、接続の再利用性を高めるために使用されます。
  • PQdb の結果が NULL を返す可能性については、本関数のソースコード上では明示的なチェックが行われていないため、呼び出し側の前提条件に依存します。
