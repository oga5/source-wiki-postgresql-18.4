CommandDest

定義

  • 種別: enum
  • 定義箇所: src/include/tcop/dest.h:85

目的

クエリ実行結果（タプル）の出力先を識別するための列挙型です。実行エンジンがタプルの送信先を抽象化し、適切な処理ルーチンを選択するために使用されます。

シグネチャまたは形状

typedef enum {
    DestNone, DestDebug, DestRemote, DestRemoteExecute, DestRemoteSimple,
    DestSPI, DestTuplestore, DestIntoRel, DestCopyOut, DestSQLFunction,
    DestTransientRel, DestTupleQueue, DestExplainSerialize
} CommandDest;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: CreateDestReceiver 等の関数において、出力先に応じた DestReceiver インスタンスを決定するための識別子として機能します。

前提条件と不変条件

  • CommandDest の値は、クエリ実行のライフサイクル全体を通じて、出力先の性質を決定する不変の識別子として扱われます。

アルゴリズムまたは振る舞い

  • CreateDestReceiver(CommandDest dest) 関数に渡されることで、対応する DestReceiver（タプル処理用関数ポインタ群）を生成するトリガーとなります。
  • BeginCommand や EndCommand などの制御関数において、プロトコルレベルの同期処理（ReadyForQuery 等）を切り替えるための条件分岐に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/tcop/dest.c: BeginCommand, CreateDestReceiver, EndCommand, NullCommand, ReadyForQuery
  • src/backend/access/common/printtup.c: printtup_create_DR
  • src/backend/tcop/postgres.c: whereToSendOutput 変数および各実行関数
  • 呼び出し先:
  • 本シンボル自体は列挙型であり、関数ではないため呼び出し先はありません。

関連データ構造

  • DestReceiver: CommandDest に基づいて生成される、タプル処理の具体的な振る舞いを定義する構造体。
  • CommandTag: クエリの識別子。BeginCommand 等で CommandDest と共に使用されます。

失敗モードと境界条件

  • 未定義の CommandDest 値が CreateDestReceiver に渡された場合、適切なレシーバーが生成されず、実行エンジンが予期せぬ動作をする可能性があります。

保守担当者向けメモ

  • 新しい出力先を追加する場合は、本列挙型に値を追加し、src/backend/tcop/dest.c 内の CreateDestReceiver 関数で対応するレシーバー生成ロジックを実装する必要があります。
