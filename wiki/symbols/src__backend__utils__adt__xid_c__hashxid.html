hashxid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xid.c:101

目的

TransactionId 型の値をハッシュ化し、ハッシュインデックスやハッシュ結合で使用可能な uint32 型のハッシュ値を生成します。

シグネチャまたは形状

Datum hashxid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される TransactionId 型の値。
  • **出力**: Datum 型として返される uint32 のハッシュ値。

前提条件と不変条件

  • PG_GETARG_TRANSACTIONID(0) マクロを使用して、第1引数から TransactionId を正しく取得できる必要があります。
  • 内部的に hash_uint32 を利用するため、入力値のビットパターンに基づいた決定論的なハッシュ値が生成されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_TRANSACTIONID(0) を呼び出し、入力引数から TransactionId を抽出します。 2. 抽出した値を hash_uint32 関数に渡し、ハッシュ計算を実行します。 3. 計算されたハッシュ値を Datum 型に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: 特定されていません。
  • **呼び出し先**: hash_uint32（common/hashfn.h で定義）。

関連データ構造

  • TransactionId: PostgreSQL のトランザクション識別子。
  • Datum: PostgreSQL の汎用データ表現型。

失敗モードと境界条件

  • TransactionId は循環する可能性がある型ですが、本関数は単なるビット値としてのハッシュ計算を行うため、値の論理的な前後関係や循環の考慮は行いません。

保守担当者向けメモ

  • 本関数は TransactionId のハッシュ化に特化しています。64bit の FullTransactionId を扱う場合は、xid8 関連の関数を使用してください。
  • hashxidextended と比較して、本関数は単一のハッシュ値のみを生成します。
