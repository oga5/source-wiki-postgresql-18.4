logicalrep_relmap_init

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/relation.c:104

目的

論理レプリケーションにおけるリレーションマッピング管理用のメモリコンテキストとハッシュテーブルを初期化し、リレーションキャッシュの無効化コールバックを登録する。

シグネチャまたは形状

static void logicalrep_relmap_init(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 LogicalRepRelMapContext および LogicalRepRelMap を初期化）

前提条件と不変条件

  • CacheMemoryContext が存在していること。
  • 本関数は、論理レプリケーションのキャッシュアクセスが発生する前に呼び出される必要がある。

アルゴリズムまたは振る舞い

1. LogicalRepRelMapContext が未作成の場合、CacheMemoryContext を親として AllocSetContextCreate を実行し、メモリコンテキストを作成する。 2. LogicalRepRelId をキー、LogicalRepRelMapEntry を要素とするハッシュテーブル LogicalRepRelMap を作成する。 3. CacheRegisterRelcacheCallback を呼び出し、logicalrep_relmap_invalidate_cb をリレーションキャッシュ無効化イベントの監視対象として登録する。

呼び出し元と呼び出し先

  • 呼び出し元: ソースコード内の参照情報に基づくと、logicalrep_relmap_init() 自体および同ファイル内の他の関数から呼び出されている可能性がある（詳細は提供された断片からは特定不可）。
  • 呼び出し先: AllocSetContextCreate, hash_create, CacheRegisterRelcacheCallback。

関連データ構造

  • LogicalRepRelMap: リレーションマッピングを保持するハッシュテーブル。
  • LogicalRepRelMapEntry: ハッシュテーブルの各エントリ。
  • LogicalRepRelId: ハッシュテーブルのキー型。

失敗モードと境界条件

  • メモリ不足時には AllocSetContextCreate または hash_create がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は論理レプリケーションの初期化プロセスにおいて重要な役割を果たす。
  • logicalrep_relmap_invalidate_cb が正しく登録されることで、DDL 変更時のキャッシュ整合性が維持される。
