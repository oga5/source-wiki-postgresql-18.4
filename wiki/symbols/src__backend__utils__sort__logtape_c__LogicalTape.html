LogicalTape

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/sort/logtape.c:137

目的

単一の物理ファイル（LogicalTapeSet）内で、独立した論理的なデータストリームをエミュレートするための状態管理構造体です。外部ソートにおける一時的なデータ保持や、マージパスでの読み取りを制御します。

シグネチャまたは形状

struct LogicalTape {
    LogicalTapeSet *tapeSet;
    bool writing;
    bool frozen;
    bool dirty;
    int64 firstBlockNumber;
    int64 curBlockNumber;
    int64 nextBlockNumber;
    int64 offsetBlockNumber;
    char *buffer;
    int buffer_size;
    int max_size;
    int pos;
    int nbytes;
    int64 *prealloc;
    int nprealloc;
    int prealloc_size;
};

入力と出力

  • **入力**: LogicalTapeSet へのポインタを保持し、書き込み時には LogicalTapeWrite を介してデータを受け取ります。
  • **出力**: LogicalTapeRead を介してバッファ内のデータを提供します。frozen 状態ではランダムアクセスが可能です。

前提条件と不変条件

  • writing が真の間は書き込みフェーズであり、curBlockNumber が有効です。
  • frozen が真の場合、読み込み時にブロックは解放されず、ランダムアクセスが許可されます。
  • prealloc 配列内のブロック番号は降順でソートされており、末尾から消費されます。

アルゴリズムまたは振る舞い

1. **書き込み**: LogicalTapeWrite は ltsGetBlock でブロックを取得し、満杯になると物理ファイルへフラッシュしてチェーンを繋ぎます。 2. **読み込み**: LogicalTapeRead は ltsReadFillBuffer を使用してデータをバッファへ読み込みます。通常は読み込み後にブロックを解放しますが、frozen 状態では解放をスキップします。 3. **凍結**: LogicalTapeFreeze を呼び出すと、以降の読み込みでブロックの再利用が停止し、LogicalTapeSeek 等によるランダムアクセスが可能になります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeAgg.c 等でパーティション管理用に使用されています。
  • **呼び出し先**: ltsCreateTape（生成）、ltsGetBlock（ブロック取得）、ltsGetPreallocBlock（事前割り当てブロック取得）、ltsInitReadBuffer（バッファ初期化）などが内部的に参照しています。

関連データ構造

  • LogicalTapeSet: このテープが所属する管理セット。
  • TapeBlockTrailer: 各ブロックの末尾に配置されるメタデータ。

失敗モードと境界条件

  • buffer は個別に palloc されるため、メモリ不足時にエラーが発生する可能性があります。
  • frozen 状態での読み込みと、非凍結状態での読み込みでは、バッファリングの挙動（ブロックの曖昧さ）が異なります。

保守担当者向けメモ

  • LogicalTape は tuplesort.c の外部ソート処理において、ディスク使用量を最小化するための重要な抽象化です。
  • ブロックの再利用ロジックは「読み込み済みのデータは不要」という前提に基づいているため、frozen 状態への移行タイミングには注意してください。
  • 並列ソート時には offsetBlockNumber を用いて、ワーカーが作成した BufFile とのオフセット計算が行われます。
