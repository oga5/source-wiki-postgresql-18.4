BTLeader

定義

  • 種別: struct
  • 定義箇所: src/backend/access/nbtree/nbtsort.c:167

目的

並列インデックス構築において、リーダープロセスが共有状態を管理し、ワーカープロセスとの同期や統計情報の集約を行うための制御構造体です。

シグネチャまたは形状

typedef struct BTLeader {
    ParallelContext *pcxt;
    int             nparticipanttuplesorts;
    BTShared        *btshared;
    Sharedsort      *sharedsort;
    Sharedsort      *sharedsort2;
    Snapshot        snapshot;
    WalUsage        *walusage;
    BufferUsage     *bufferusage;
} BTLeader;

入力と出力

  • **入力**: 並列構築開始時に palloc0 で割り当てられ、ParallelContext や共有状態へのポインタが格納されます。
  • **出力**: 並列構築の終了処理（_bt_end_parallel）において、統計情報の集約やリソースの解放に使用されます。

前提条件と不変条件

  • nparticipanttuplesorts は、正常に起動したワーカー数に、リーダー自身がワーカーとして参加する場合は1を加えた値となります。
  • btshared、sharedsort 等のポインタは、TOC（Table of Contents）ルックアップを回避するためのキャッシュとして機能します。

アルゴリズムまたは振る舞い

  • リーダープロセスが共有メモリ上の状態へ直接アクセスするための便宜的なポインタを提供します。
  • 一意インデックス（Unique Index）構築時には sharedsort2 が使用されます。
  • 並列構築の終了時、リーダーは本構造体を通じてワーカーの終了を待ち合わせ、統計情報を集約します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtsort.c 内の並列構築ロジック（_bt_parallel_build_main 等の準備段階）。
  • **呼び出し先**: _bt_end_parallel(BTLeader *btleader) が本構造体を引数として受け取り、並列構築の終了処理を実行します。

関連データ構造

  • ParallelContext: 並列処理の基盤。
  • BTShared: インデックス構築全体の共有状態。
  • Sharedsort: tuplesort が管理するプロセス間共有のソート状態。
  • BTBuildState: インデックス構築の全体状態を保持する構造体。

失敗モードと境界条件

  • DISABLE_LEADER_PARTICIPATION が設定されている場合、リーダーはワーカーとして参加しません。
  • 共有メモリの割り当てや並列コンテキストの初期化に失敗した場合、本構造体の生成段階でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は、並列構築時のリーダープロセス専用のキャッシュです。TOCルックアップを避ける設計であるため、共有状態へのアクセス頻度が高いフィールドを追加する際は、ここへの追加を検討してください。
  • walusage および bufferusage は、並列構築全体の統計情報を集約するために使用されます。
