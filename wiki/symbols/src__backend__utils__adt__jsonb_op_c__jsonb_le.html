jsonb_le

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_op.c:193

目的

2つの jsonb 値を比較し、左側の値が右側の値以下であるか（<=）を判定する。

シグネチャまたは形状

Datum jsonb_le(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの jsonb 型データを受け取る。
  • **出力**: bool 型の判定結果を Datum として返す。

前提条件と不変条件

  • 入力された jsonb ポインタは有効であること。
  • PG_GETARG_JSONB_P マクロにより、入力値の取得と必要に応じたTOAST展開が行われる。

アルゴリズムまたは振る舞い

1. PG_GETARG_JSONB_P を使用して、第1引数 jba と第2引数 jbb を取得する。 2. compareJsonbContainers 関数を呼び出し、両者のルートコンテナを比較する。 3. 比較結果が 0 以下（<= 0）であれば true、そうでなければ false を判定結果とする。 4. PG_FREE_IF_COPY を使用して、必要に応じてコピーされたメモリを解放する。 5. PG_RETURN_BOOL を介して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（fmgr）を介してSQL演算子から呼び出される。
  • **呼び出し先**:
  • compareJsonbContainers (比較ロジックの実行)
  • PG_GETARG_JSONB_P, PG_FREE_IF_COPY, PG_RETURN_BOOL (PostgreSQL関数インターフェース)

関連データ構造

  • Jsonb: jsonb 型の内部表現。
  • JsonbContainer: Jsonb 構造体内のルート要素。

失敗モードと境界条件

  • 入力が NULL の場合、PostgreSQLの関数呼び出し規約に従い、呼び出し元で適切に処理される必要がある（通常は NULL が返る）。
  • 比較対象の jsonb 構造が複雑な場合、compareJsonbContainers の再帰的な比較処理によりスタックを消費する可能性がある。

保守担当者向けメモ

  • 本関数はB-Treeインデックスの比較演算子として使用される。
  • 比較ロジックの詳細は compareJsonbContainers に依存しているため、比較順序の仕様変更時は同関数を確認すること。
