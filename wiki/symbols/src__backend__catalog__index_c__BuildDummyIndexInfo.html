BuildDummyIndexInfo

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/index.c:2487

目的

指定されたインデックスリレーションから、述語（predicate）を含まない最小限の IndexInfo 構造体を構築します。

シグネチャまたは形状

IndexInfo *BuildDummyIndexInfo(Relation index)

入力と出力

  • **入力**: Relation index（対象となるインデックスのリレーション記述子）
  • **出力**: IndexInfo *（構築されたインデックス情報構造体）

前提条件と不変条件

  • index->rd_index（pg_index カタログ情報）が有効であること。
  • インデックスのキー数（indnatts）が 1 以上 INDEX_MAX_KEYS 以下であること。これに違反する場合、エラーが発生します。

アルゴリズムまたは振る舞い

1. index->rd_index から属性数を確認し、範囲外であればエラーをスローします。 2. makeIndexInfo を呼び出し、以下の設定で IndexInfo を生成します。

  • 述語（predicate）は NIL（なし）として扱います。
  • インデックス式は RelationGetDummyIndexExpressions(index) を使用します。
  • 一意性、NULL値の扱い、準備状態などのフラグを pg_index からコピーします。

    • 3. ii_IndexAttrNumbers 配列に、pg_index の indkey から属性番号をコピーします。 4. 除外制約（exclusion constraint）の情報は無視されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/heap.c:3578
  • **呼び出し先**: makeIndexInfo, RelationGetDummyIndexExpressions, RelationGetRelid, elog

関連データ構造

  • IndexInfo: インデックスのメタデータを保持する構造体。
  • Relation: リレーション記述子。
  • Form_pg_index: pg_index システムカタログの行構造。

失敗モードと境界条件

  • indnatts が 1 未満または INDEX_MAX_KEYS を超える場合、elog(ERROR, ...) によりトランザクションが中断されます。

保守担当者向けメモ

  • 本関数は「ダミー」の情報を生成するため、複雑な述語や除外制約が必要なコンテキストでは使用できません。
  • RelationGetDummyIndexExpressions が返す式の内容が、呼び出し側の期待する用途と合致しているか確認してください。
