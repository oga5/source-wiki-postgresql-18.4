Gzip_open

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_gzip.c:371

目的

GZIP 圧縮を適用したファイルまたはファイル記述子を開き、CompressFileHandle 構造体にその状態を保持させる。

シグネチャまたは形状

static bool Gzip_open(const char *path, int fd, const char *mode, CompressFileHandle *CFH)

入力と出力

  • **入力**:
  • path: ファイルパス（fd < 0 の場合に使用）。
  • fd: ファイル記述子（>= 0 の場合に使用）。
  • mode: gzopen に渡すモード文字列。
  • CFH: 圧縮状態を管理する CompressFileHandle 構造体。
  • **出力**:
  • 成功時は true を返し、CFH->private_data に gzFile ハンドルを格納する。
  • 失敗時は false を返す。

前提条件と不変条件

  • CFH->compression_spec.level が Z_DEFAULT_COMPRESSION 以外の場合、指定された圧縮レベルが mode 文字列に付与される。
  • fd >= 0 の場合、dup(fd) が呼び出され、複製された記述子が gzdopen に渡される。

アルゴリズムまたは振る舞い

1. CFH->compression_spec.level を確認し、デフォルト値でなければ mode 文字列に圧縮レベルを連結する。 2. fd が有効（>= 0）であれば gzdopen を使用し、そうでなければ gzopen を使用して gzFile を開く。 3. gzFile が NULL であれば false を返す。 4. 成功した場合、gzFile ハンドルを CFH->private_data に保存し、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_gzip.c 内で CFH->open_func に割り当てられている。
  • **呼び出し先**: zlib ライブラリの gzopen, gzdopen, dup。

関連データ構造

  • CompressFileHandle: 圧縮処理の抽象化インターフェース。
  • gzFile: zlib が提供する圧縮ファイルハンドル。

失敗モードと境界条件

  • gzopen または gzdopen が NULL を返した場合、関数は false を返す。
  • fd が無効な場合や、ファイルパスが不正な場合、zlib 側のエラーにより gzfp が NULL となる。

保守担当者向けメモ

  • 本関数は zlib に依存しており、HAVE_LIBZ が定義されている環境でのみ有効に機能する。
  • fd を渡す際は dup されるため、呼び出し元で元の fd を閉じる必要があるかどうかのライフサイクル管理に注意すること。
