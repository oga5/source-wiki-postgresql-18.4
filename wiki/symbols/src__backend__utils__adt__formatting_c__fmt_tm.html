fmt_tm

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/formatting.c:495

目的

日時情報の書式化および解析処理において、標準的な tm 構造体と類似した形式で日時要素を保持するために使用されます。

シグネチャまたは形状

struct fmt_tm
{
	int			tm_sec;
	int			tm_min;
	int64		tm_hour;
	int			tm_mday;
	int			tm_mon;
	int			tm_year;
	int			tm_wday;
	int			tm_yday;
	long int	tm_gmtoff;
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。主に TO_CHAR や TO_TIMESTAMP 等の変換処理において、日時フィールドの受け渡しに使用されます。

前提条件と不変条件

  • 標準的な tm 構造体と構造が似ていますが、ビット単位のコピーは推奨されません（pg_tm からのコピーには専用の処理が必要です）。
  • 構造体の初期化には、pg_tm と併せてゼロクリアを行うための専用の処理が用意されています。

アルゴリズムまたは振る舞い

  • 日時変換処理において、入力文字列から解析された値や、内部データ型から変換された値を一時的に格納するコンテナとして機能します。
  • formatting.c 内の各変換関数（DCH_to_char 等）において、この構造体を経由して日時フィールドが操作されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/formatting.c 内の複数の箇所で、ローカル変数として宣言・利用されています（例: 510行目、2511行目、4007行目、4043行目、4082行目）。
  • **呼び出し先**: 構造体であるため、直接的な呼び出し先はありません。ただし、この構造体へのポインタが変換ロジックの引数として渡されます。

関連データ構造

  • pg_tm: 本構造体と対になる標準的な日時構造体。
  • TmFromChar: 文字列からの解析結果を保持する構造体。
  • TmToChar: 書式化処理に関連する構造体。

失敗モードと境界条件

  • 構造体自体の定義に起因する失敗モードはありません。
  • 変換処理において、tm_year や tm_mon 等の各フィールドに不正な値が設定された場合、後続の書式化処理で予期しない結果を招く可能性があります。

保守担当者向けメモ

  • 本構造体は pg_tm との互換性を意識して設計されていますが、フィールドの型（int64 等）が異なるため、コピーや初期化の際は必ず専用のロジックを介してください。
  • 新しい日時フィールドを追加する場合は、formatting.c 内の変換ロジック全体への影響を確認してください。
