gbt_float8_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_float8.c:137

目的

GiSTインデックスの検索時、インデックス内の float8 型の範囲（float8KEY）がクエリ条件を満たすかどうかを判定します。

シグネチャまたは形状

Datum gbt_float8_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • entry (GISTENTRY*): インデックス内のキー情報。
  • query (float8): 検索対象の値。
  • strategy (StrategyNumber): 比較演算子（<, <=, =, >=, > 等）を示す戦略番号。
  • recheck (bool*): 判定結果の再確認が必要かを示すフラグ。
  • **出力**:
  • Datum (bool): 条件に合致すれば true、そうでなければ false を返します。
  • *recheck: 常に false が設定されます。

前提条件と不変条件

  • *recheck は常に false に設定されます。これは、本関数による判定が完全であり、インデックス検索後にヒープでの再確認が不要であることを示します。
  • 入力された entry->key は float8KEY 構造体としてキャスト可能である必要があります。

アルゴリズムまたは振る舞い

1. *recheck を false に初期化します。 2. entry->key を float8KEY 構造体にキャストし、その lower および upper メンバを GBT_NUMKEY_R 構造体に格納します。 3. gbt_num_consistent 関数を呼び出し、float8KEY の範囲とクエリ値、戦略番号を比較します。 4. gbt_num_consistent の結果を PG_RETURN_BOOL で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: GiSTインデックスの検索エンジン（直接的な呼び出し元はソースコード上では明示されていませんが、PostgreSQLのGiSTインターフェースから呼び出されます）。
  • **呼び出し先**: gbt_num_consistent（contrib/btree_gist/btree_utils_num.c で定義）。

関連データ構造

  • float8KEY: インデックス内で float8 型の範囲（下限と上限）を保持する構造体。
  • GBT_NUMKEY_R: 数値比較のために lower と upper を保持する汎用的な範囲構造体。
  • tinfo: gbtree_ninfo 構造体。float8 型の比較ロジックを定義する設定オブジェクト。

失敗モードと境界条件

  • float8 型のオーバーフローや不正な値については、依存先の gbt_num_consistent および utils/float.h のロジックに準拠します。
  • recheck が false であるため、本関数が true を返した場合は、インデックス内のデータが確実にクエリ条件を満たしていると見なされます。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で提供される汎用的な数値比較ロジックのラッパーです。ロジックの変更が必要な場合は、まず btree_utils_num 側の実装を確認してください。
  • PG_FUNCTION_INFO_V1(gbt_float8_consistent) により、PostgreSQLの関数マネージャに登録されています。
