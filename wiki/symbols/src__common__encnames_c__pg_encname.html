pg_encname

定義

  • 種別: struct
  • 定義箇所: src/common/encnames.c:33

目的

文字エンコーディングの名称（文字列）と、PostgreSQL 内部で使用されるエンコーディング ID (pg_enc) を紐付けるためのデータ構造です。

シグネチャまたは形状

typedef struct pg_encname
{
	const char *name;
	pg_enc		encoding;
} pg_encname;

入力と出力

  • **入力**: エンコーディング名を表す文字列 (name) および内部エンコーディング ID (encoding)。
  • **出力**: なし（本構造体はデータ保持用）。

前提条件と不変条件

  • name は、正規化された（小文字かつ英数字のみの）エンコーディング名である必要があります。
  • この構造体を利用するテーブル（pg_encname_tbl）は、二分探索を可能にするため、name フィールドに基づいてアルファベット順にソートされている必要があります。

アルゴリズムまたは振る舞い

本構造体は、pg_char_to_encoding 関数において、文字列から内部 ID を検索するための検索キーおよび値のペアとして利用されます。検索時には、入力文字列を clean_encoding_name で正規化した後、この構造体の配列に対して二分探索が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/encnames.c 内の pg_encname_tbl 定義および検索ロジック。
  • **呼び出し先**: 情報なし。

関連データ構造

  • pg_enc: mb/pg_wchar.h で定義されるエンコーディング ID 型。
  • pg_encname_tbl: 本構造体の配列であり、エンコーディング名と ID のマッピングを保持する静的テーブル。

失敗モードと境界条件

  • 検索対象のエンコーディング名がテーブル内に存在しない場合、検索関数は無効な ID を返します。
  • 構造体内の name が NULL または不正な文字列である場合、検索ロジックが正しく動作しない可能性があります。

保守担当者向けメモ

  • 本構造体は src/common/encnames.c におけるエンコーディング管理の基盤です。新しいエンコーディングを追加する際は、pg_encname_tbl に本構造体のインスタンスを追加し、テーブルのソート順序を維持してください。
