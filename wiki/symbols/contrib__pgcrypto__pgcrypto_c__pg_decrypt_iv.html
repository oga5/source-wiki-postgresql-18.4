pg_decrypt_iv

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgcrypto.c:402

目的

SQL インターフェースを介して、指定されたアルゴリズム、鍵、および初期化ベクトル（IV）を用いてデータを復号する。

シグネチャまたは形状

Datum pg_decrypt_iv(PG_FUNCTION_ARGS)

  • 引数: SQL 関数 decrypt_iv(bytea, bytea, bytea, text) に対応する 4 つの引数。
  • 戻り値: 復号されたデータを含む bytea 型。

入力と出力

  • 入力:
  • data (bytea): 復号対象の暗号化データ。
  • key (bytea): 復号に使用する鍵。
  • iv (bytea): 復号に使用する初期化ベクトル。
  • type (text): 使用する暗号アルゴリズム名。
  • 出力: 復号後のバイナリデータ (bytea)。

前提条件と不変条件

  • find_provider により、指定された type に対応する PX_Combo プロバイダーが正常に取得できること。
  • px_combo_init および px_combo_decrypt が成功すること。
  • PostgreSQL のメモリ管理ルールに従い、palloc で確保されたメモリが適切に返却されること。

アルゴリズムまたは振る舞い

1. find_provider を呼び出し、指定されたアルゴリズム用の PX_Combo コンテキストを取得する。 2. 入力引数からデータ、鍵、IV のサイズを取得する。 3. px_combo_decrypt_len で復号後の最大サイズを算出し、結果格納用のメモリを確保する。 4. px_combo_init で鍵と IV を設定し、px_combo_decrypt で復号を実行する。 5. px_combo_free でコンテキストを解放する。 6. 処理中にエラーが発生した場合は ereport で例外を投げる。 7. PG_FREE_IF_COPY を使用して、入力引数のメモリを適切に解放する。

呼び出し元と呼び出し先

  • 呼び出し元: SQL インターフェース（decrypt_iv 関数）。
  • 呼び出し先: find_provider, px_combo_decrypt_len, px_combo_init, px_combo_decrypt, px_combo_free, px_strerror。

関連データ構造

  • PX_Combo: 暗号化/復号処理を抽象化したコンテキスト構造体。
  • bytea, text: PostgreSQL のデータ型。

失敗モードと境界条件

  • アルゴリズム名が不正な場合、find_provider がエラーを発生させる。
  • 鍵や IV のサイズがアルゴリズムの要件を満たさない場合、px_combo_init がエラーを返す。
  • 復号処理自体が失敗した場合、px_strerror を含む ereport により ERRCODE_EXTERNAL_ROUTINE_INVOCATION_EXCEPTION が発生する。

保守担当者向けメモ

  • 本関数は pgcrypto モジュールの SQL ブリッジ層である。暗号化アルゴリズムの具体的な実装については px レイヤー（px.c 等）を参照すること。
  • メモリリークを防ぐため、palloc したメモリと PG_FREE_IF_COPY の対応関係を維持すること。
