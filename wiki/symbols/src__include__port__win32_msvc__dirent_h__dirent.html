dirent

定義

  • 種別: struct
  • 定義箇所: src/include/port/win32_msvc/dirent.h:9

目的

Windows環境において、POSIX準拠のディレクトリ走査を行う際に、ディレクトリ内の各エントリ情報を保持するために使用されます。

シグネチャまたは形状

struct dirent
{
	long		d_ino;
	unsigned short d_reclen;
	unsigned char d_type;
	unsigned short d_namlen;
	char		d_name[MAX_PATH];
};

入力と出力

本構造体は readdir 関数によって値が設定され、呼び出し元へディレクトリ内のファイル情報を提供します。

前提条件と不変条件

  • d_ino はWindows環境ではサポートされていないため、常に 0 が返されます。
  • d_name は MAX_PATH のサイズ制限を受けます。

アルゴリズムまたは振る舞い

src/port/dirent.c において、Windowsの WIN32_FIND_DATA 構造体から取得した情報が本構造体の各フィールドにマッピングされます。d_type の判定には、Windows固有のファイル属性（FILE_ATTRIBUTE_REPARSE_POINT 等）が利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下のソースファイルで struct dirent 型のポインタが使用されていることが確認されています。
  • src/backend/access/heap/rewriteheap.c
  • src/backend/access/transam/slru.c
  • src/backend/access/transam/twophase.c
  • src/backend/access/transam/xlog.c
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • DIR: ディレクトリ走査の状態を保持する不透明な構造体。
  • WIN32_FIND_DATA: Windowsネイティブのディレクトリ走査用構造体（dirent.c でのマッピング元）。

失敗モードと境界条件

  • d_name に格納されるファイル名が MAX_PATH を超える場合、切り詰めやバッファオーバーフローのリスクがあるため、Windowsのパス制限に依存します。

保守担当者向けメモ

  • 本構造体はPOSIX標準の完全な模倣ではなく、PostgreSQLのビルドに必要な最小限の機能を提供しています。
  • Windows環境特有のファイルシステム挙動（シンボリックリンクやディレクトリの判別）は d_type の実装に依存しているため、変更時は src/port/dirent.c のマッピングロジックを併せて確認してください。
