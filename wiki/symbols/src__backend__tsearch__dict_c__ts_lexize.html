ts_lexize

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/tsearch/dict.c:26

目的

指定された辞書（OID）を用いて入力テキストを語幹抽出（lexize）し、その結果を SQL で利用可能な text 型の配列として返却します。

シグネチャまたは形状

Datum ts_lexize(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): 辞書を識別する OID。
  • PG_GETARG_TEXT_PP(1): 処理対象の入力テキスト。
  • **出力**:
  • Datum: 語幹抽出結果を含む text 型の配列（ArrayType）。結果がない場合は NULL を返却。

前提条件と不変条件

  • lookup_ts_dictionary_cache により、指定された OID に対応する辞書キャッシュエントリが取得可能であること。
  • 辞書処理中に確保されたメモリ（TSLexeme 配列、各語彙文字列、一時的な Datum 配列）は、関数終了前にすべて pfree で解放されること。

アルゴリズムまたは振る舞い

1. **辞書検索**: lookup_ts_dictionary_cache を呼び出し、辞書定義を取得します。 2. **辞書実行**: キャッシュ内の lexize 関数を呼び出します。DictSubState を使用して、必要に応じて複数回の反復処理（継続処理）を行います。 3. **変換**: 辞書から返された TSLexeme 構造体の配列を走査し、各語彙を CStringGetTextDatum を用いて text 型に変換します。 4. **配列構築**: construct_array_builtin を使用して、変換した text 型の配列を構築します。 5. **メモリ解放**: 構築に使用した一時的な Datum 配列、各語彙文字列、および TSLexeme 配列を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されます（直接的な C 言語レベルの呼び出し元情報は提供されていません）。
  • **呼び出し先**:
  • lookup_ts_dictionary_cache
  • dict->lexize（キャッシュされた辞書固有の関数）
  • construct_array_builtin
  • pfree / palloc

関連データ構造

  • TSDictionaryCacheEntry: 辞書キャッシュ構造体。
  • TSLexeme: 辞書から返される語彙の内部表現。
  • DictSubState: 辞書処理の継続状態を管理する構造体。

失敗モードと境界条件

  • 辞書が結果を返さない場合、NULL を返却します。
  • 辞書が複数の語彙を返す場合、DictSubState を介してすべてを走査し、配列に格納します。

保守担当者向けメモ

  • 辞書から返された TSLexeme 配列の各要素と配列自体の解放順序が重要です。メモリリークを防ぐため、pfree の呼び出し順序を変更する際は注意してください。
  • DictSubState を用いた反復処理は、シノニム展開などの複数語彙を返す辞書の実装をサポートしています。
