PQcancelReset

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-cancel.c:336

目的

キャンセル用接続オブジェクト（PGcancelConn）の状態をリセットし、再利用可能な状態に戻します。

シグネチャまたは形状

void PQcancelReset(PGcancelConn *cancelConn);

入力と出力

  • **入力**: cancelConn - リセット対象の PGcancelConn 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • cancelConn は有効な PGcancelConn オブジェクトを指している必要があります。
  • 内部の PGconn 構造体に対して pqClosePGconn が実行され、ソケット等のリソースが解放されます。

アルゴリズムまたは振る舞い

1. pqClosePGconn を呼び出し、内部の接続リソースをクローズします。 2. 接続状態を CONNECTION_ALLOCATED に設定します。 3. ホストおよびアドレスの選択インデックス（whichhost, whichaddr）を 0 にリセットします。 4. 次のホストやアドレスへの試行フラグ（try_next_host, try_next_addr）を false に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/libpq_pipeline/libpq_pipeline.c にて、キャンセル接続の再利用前に呼び出されています。
  • **呼び出し先**:
  • pqClosePGconn（内部関数）

関連データ構造

  • PGcancelConn: キャンセル要求専用の接続情報を保持する構造体。

失敗モードと境界条件

  • cancelConn が NULL の場合、未定義の動作（セグメンテーション違反等）を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は接続状態を初期化するのみであり、再接続処理は含まれていません。再利用時には必要に応じて接続確立処理を別途呼び出す必要があります。
