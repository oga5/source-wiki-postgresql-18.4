gbt_textgt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_text.c:27

目的

PostgreSQLのGiSTインデックスにおいて、text型データに対する「より大きい（greater than）」比較を判定するサポート関数です。

シグネチャまたは形状

bool gbt_textgt(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象のtext型データへのポインタ。
  • collation: 比較に使用する照合順序（Collation）のOID。
  • flinfo: 関数マネージャ情報（本関数内では直接使用されません）。
  • **出力**:
  • bool: a > b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は有効なtext型のデータポインタである必要があります。
  • 内部で呼び出される text_gt は、指定された collation に基づいて比較を行います。

アルゴリズムまたは振る舞い

1. PointerGetDatum を使用して、入力された void * ポインタを Datum 型に変換します。 2. DirectFunctionCall2Coll を介して、PostgreSQLの標準関数 text_gt を呼び出します。 3. text_gt の実行結果（Datum）を DatumGetBool で論理値に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し先**: text_gt（PostgreSQL標準のテキスト比較関数）。
  • **呼び出し元**: contrib/btree_gist/btree_text.c 内の tinfo 構造体（gbtree_vinfo）にて、比較演算のコールバックとして登録されています。
  • その他、直接的な呼び出し関係の詳細はソースコードの静的解析範囲では限定的です。

関連データ構造

  • gbtree_vinfo: 本関数を含む比較関数群を保持し、GiSTインデックスの汎用ルーチンへ提供するための構造体。

失敗モードと境界条件

  • text_gt が内部でエラー（照合順序の不整合やメモリ不足など）を発生させた場合、その例外が伝播します。
  • a または b が NULL の場合の挙動は、text_gt の実装に依存します。

保守担当者向けメモ

  • 本関数は text 型専用です。bpchar 型に対しては gbt_bpchargt が別途定義されています。
  • DirectFunctionCall2Coll を使用しているため、標準の比較ロジックを直接利用しており、比較ルールに変更がある場合は text_gt 側の仕様に従います。
