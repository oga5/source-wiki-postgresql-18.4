icu_test_full

定義

  • 種別: function
  • 定義箇所: src/common/unicode/case_test.c:104

目的

PostgreSQL の Unicode 大文字・小文字変換関数（unicode_strlower, unicode_strupper, unicode_strtitle, unicode_strfold）の出力結果を、ICU ライブラリの出力と比較し、整合性を検証する。

シグネチャまたは形状

static void icu_test_full(char *str)

入力と出力

  • **入力**: str (検証対象の UTF-8 文字列)
  • **出力**: なし（検証失敗時は printf でエラーを出力し exit(1) でプロセスを終了する）

前提条件と不変条件

  • BUFSZ サイズのバッファが変換結果を保持するのに十分であること。
  • ICU の casemap オブジェクトが初期化済みであること。
  • タイトルケース変換には initcap_wbnext 関数と WordBoundaryState 構造体を使用し、POSIX セマンティクスは無効（posix = false）とする。

アルゴリズムまたは振る舞い

1. WordBoundaryState を初期化し、入力文字列のメタデータを設定する。 2. PostgreSQL の各変換関数を呼び出し、ローカルバッファ（lower, title, upper, fold）に結果を格納する。 3. ICU の ucasemap_utf8To... 系関数を呼び出し、比較用バッファ（icu_lower, icu_title, icu_upper, icu_fold）に結果を格納する。 4. strcmp を用いて、PostgreSQL の変換結果と ICU の変換結果を各ケースごとに比較する。 5. 一致しない場合、該当する文字列と期待値・実測値を表示して異常終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/unicode/case_test.c 内の各テストケース（main 関数周辺で、固定文字列や動的文字列を引数として呼び出される）。
  • **呼び出し先**:
  • PostgreSQL 側: unicode_strlower, unicode_strtitle, unicode_strupper, unicode_strfold, initcap_wbnext
  • ICU 側: ucasemap_utf8ToLower, ucasemap_utf8ToTitle, ucasemap_utf8ToUpper, ucasemap_utf8FoldCase

関連データ構造

  • WordBoundaryState: タイトルケース変換時の単語境界状態を保持。
  • UErrorCode: ICU のエラー状態管理用。

失敗モードと境界条件

  • **不一致**: PostgreSQL と ICU の変換結果が 1 バイトでも異なる場合、exit(1) で終了する。
  • **バッファオーバーフロー**: 入力文字列が BUFSZ を超える場合、変換関数の挙動はバッファサイズに依存する（本関数内では BUFSZ を固定値として使用）。

保守担当者向けメモ

  • 本関数は USE_ICU が定義されている環境でのみ ICU との比較が有効に機能する。
  • Unicode バージョンの差異により、特定のコードポイントで ICU と結果が乖離する場合があるため、テスト失敗時はバージョンを確認すること。
