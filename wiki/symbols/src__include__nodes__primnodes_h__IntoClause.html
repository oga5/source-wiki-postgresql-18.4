IntoClause

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/primnodes.h:159

目的

SELECT INTO 文やマテリアライズドビューの作成において、結果を格納する対象テーブルの仕様を保持するために使用されます。

シグネチャまたは形状

typedef struct IntoClause
{
	NodeTag		type;
	RangeVar   *rel;
	List	   *colNames;
	char	   *accessMethod;
	List	   *options;
	OnCommitAction onCommit;
	char	   *tableSpaceName;
	struct Query *viewQuery;
	bool		skipData;
} IntoClause;

入力と出力

  • **入力**: クエリ解析時に gram.y 等から生成される各フィールドの値。
  • **出力**: CreateIntoRelDestReceiver や GetIntoRelEFlags などの関数へ渡され、テーブル作成やデータ挿入の制御に使用されます。

前提条件と不変条件

  • viewQuery フィールドは pg_node_attr(query_jumble_ignore) が指定されており、クエリのハッシュ化（Jumbling）の対象外です。
  • colNames が NIL の場合、列名は自動的に割り当てられます。
  • tableSpaceName は NULL の場合、デフォルトのテーブルスペースが使用されます。

アルゴリズムまたは振る舞い

本構造体はデータ構造の定義のみを行い、実行時のロジックは持ちません。主に CREATE TABLE AS (CTAS) やマテリアライズドビューの作成プロセスにおいて、ターゲットとなるリレーションの属性（アクセス手法、コミット時の動作、データ保持の有無など）を伝達する役割を担います。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/commands/createas.c: create_ctas_internal, create_ctas_nodata, GetIntoRelEFlags, CreateIntoRelDestReceiver 等で参照。
  • contrib/pg_overexplain/pg_overexplain.c: 統計情報や説明の出力処理で参照。
  • **呼び出し先**: 構造体であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • RangeVar: ターゲットリレーションの名前指定に使用。
  • OnCommitAction: コミット時の動作（ON COMMIT 句）を定義。
  • Query: マテリアライズドビューの元となる SELECT クエリを保持。

失敗モードと境界条件

  • skipData が true の場合、データは挿入されず、テーブル定義のみが作成されます（WITH NO DATA）。
  • accessMethod や tableSpaceName が無効な値である場合、後続のコマンド実行フェーズでエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は primnodes.h に定義されていますが、主に commands 系の処理で利用されます。
  • viewQuery フィールドの変更や追加を行う際は、query_jumble_ignore 属性の意図を考慮してください。
