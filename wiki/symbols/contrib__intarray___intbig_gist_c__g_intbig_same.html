g_intbig_same

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_intbig_gist.c:112

目的

2つの GISTTYPE インデックスエントリが同一であるかどうかを判定し、GiST インデックスの重複チェックをサポートします。

シグネチャまたは形状

Datum g_intbig_same(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): 比較対象の GISTTYPE *a
  • PG_GETARG_POINTER(1): 比較対象の GISTTYPE *b
  • PG_GETARG_POINTER(2): 結果を格納する bool *result
  • **出力**:
  • PG_RETURN_POINTER(result): 比較結果（true または false）を指すポインタを返します。

前提条件と不変条件

  • a および b は有効な GISTTYPE 構造体へのポインタである必要があります。
  • siglen は GET_SIGLEN() マクロにより取得される現在のシグネチャ長に基づきます。

アルゴリズムまたは振る舞い

1. 両方のエントリが ISALLTRUE（全ビットがセットされた状態）である場合、result を true に設定します。 2. 片方のみが ISALLTRUE である場合、result を false に設定します。 3. いずれも ISALLTRUE でない場合、GETSIGN マクロで取得したビットベクトルを siglen 分だけバイト単位で比較します。 4. 全てのバイトが一致すれば result は true、一つでも不一致があれば false となります。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス管理機構から呼び出されます（直接的な呼び出し元関数の詳細はソースコード内に明示されていません）。
  • **呼び出し先**: GET_SIGLEN(), ISALLTRUE(), GETSIGN(), LOOPBYTE() などのマクロを使用します。

関連データ構造

  • GISTTYPE: intbig 型の GiST インデックスエントリを保持する構造体。
  • BITVECP: ビットベクトルを操作するためのポインタ型。

失敗モードと境界条件

  • siglen が正しく設定されていない場合、メモリ境界外へのアクセスが発生する可能性があります。
  • 入力ポインタが NULL の場合の挙動は定義されていません。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_INFO_V1(g_intbig_same) によりエクスポートされています。
  • 比較ロジックはビットベクトルの全バイト走査に依存しているため、siglen が大きい場合はパフォーマンスに影響を与える可能性があります。
