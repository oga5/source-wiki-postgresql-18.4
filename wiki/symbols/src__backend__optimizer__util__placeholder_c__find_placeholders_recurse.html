find_placeholders_recurse

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/placeholder.c:208

目的

クエリのジョイントリー（jtnode）を再帰的に走査し、含まれるすべての式（quals）に対して find_placeholders_in_expr を呼び出すことで、クエリ内の PlaceHolderVar を特定し、対応する PlaceHolderInfo を作成・登録します。

シグネチャまたは形状

static void find_placeholders_recurse(PlannerInfo *root, Node *jtnode)

入力と出力

  • root: クエリ全体の最適化情報を保持する PlannerInfo 構造体へのポインタ。
  • jtnode: 走査対象のジョイントリーノード（RangeTblRef, FromExpr, または JoinExpr）。
  • 戻り値: なし（副作用として root 内のプレースホルダー情報を更新）。

前提条件と不変条件

  • jtnode が NULL の場合は何も行いません。
  • RangeTblRef ノードには評価すべき式（quals）が含まれないため、走査はスキップされます。

アルゴリズムまたは振る舞い

1. **ノード種別の判定**:

  • FromExpr: fromlist 内の各子ノードに対して再帰的に本関数を呼び出し、その後 quals に対して find_placeholders_in_expr を実行します。
  • JoinExpr: 左引数（larg）と右引数（rarg）に対して再帰的に本関数を呼び出し、その後 quals に対して find_placeholders_in_expr を実行します。
  • RangeTblRef: 何も行いません。

    • 2. **エラーハンドリング**: 上記以外のノード種別が渡された場合、elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • find_placeholders_in_jointree（src/backend/optimizer/util/placeholder.c:198）から、クエリのルートジョイントリーを引数として呼び出されます。
  • 本関数自身による再帰呼び出し（FromExpr の子要素、および JoinExpr の左右引数に対して）。
  • **呼び出し先**:
  • find_placeholders_in_expr: 各ノードに含まれる quals を処理するために呼び出されます。

関連データ構造

  • PlannerInfo: クエリ全体の最適化状態を保持。
  • Node / FromExpr / JoinExpr / RangeTblRef: PostgreSQLのクエリツリーを構成するノード構造体。

失敗モードと境界条件

  • 未知のノード種別が渡された場合、elog(ERROR) により処理が中断されます。
  • jtnode が NULL の場合は安全に終了します。

保守担当者向けメモ

  • 本関数はジョイントリーの構造を辿るための基盤です。新しいジョイントリーノードの種類が追加された場合は、本関数の条件分岐に追加が必要です。
  • find_placeholders_in_expr が quals を処理する際、式ツリー内の PlaceHolderVar を探索するため、式ツリーの複雑さに応じた再帰が発生します。
