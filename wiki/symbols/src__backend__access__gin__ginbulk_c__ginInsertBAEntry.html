ginInsertBAEntry

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginbulk.c:147

目的

GINインデックス構築時、指定されたキーとTID（タプル識別子）のペアを BuildAccumulator 内の赤黒木（RBT）に挿入または集約します。

シグネチャまたは形状

static void ginInsertBAEntry(BuildAccumulator *accum, ItemPointer heapptr, OffsetNumber attnum, Datum key, GinNullCategory category)

入力と出力

  • **入力**:
  • accum: 構築状態を保持する BuildAccumulator 構造体。
  • heapptr: 挿入対象の ItemPointer。
  • attnum: 属性番号。
  • key: 挿入対象の Datum キー。
  • category: キーのカテゴリ（GinNullCategory）。
  • **出力**: なし（副作用として accum->tree を更新）。

前提条件と不変条件

  • accum->tree は初期化済みであること。
  • category が GIN_CAT_NORM_KEY の場合、キーの永続的なコピーが作成されます。

アルゴリズムまたは振る舞い

1. 一時的な GinEntryAccumulator を作成し、検索キーとして設定します。 2. rbt_insert を呼び出し、赤黒木から該当ノードを取得または新規作成します。 3. 新規作成されたノード（isNew == true）の場合:

  • キーが正規キーであれば getDatumCopy でコピーを保持します。
  • 初期TIDリスト用メモリを確保し、heapptr を格納します。
  • accum->allocatedMemory を更新します。

    • 4. 既存ノードの場合:

  • ginCombineData が既に呼び出されており、TIDの追加等の処理は完了しているものとして扱います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginbulk.c 内の ginInsertBAEntries から呼び出されます。
  • **呼び出し先**:
  • rbt_insert（赤黒木への挿入）
  • getDatumCopy（キーのコピー）
  • palloc（メモリ確保）
  • ginCombineData（既存ノードの場合の処理）

関連データ構造

  • BuildAccumulator: 構築全体の管理構造体。
  • GinEntryAccumulator: 赤黒木の各ノード。キーとTIDリストを保持。
  • ItemPointerData: TIDを格納する構造体。

失敗モードと境界条件

  • メモリ不足時: palloc が失敗した場合、PostgreSQLの標準的なメモリ管理によりエラーが発生します。
  • maintenance_work_mem 制限: accum->allocatedMemory が増大し、設定値を超過する可能性がある場合、呼び出し元で適切に制御される必要があります。

保守担当者向けメモ

  • 本関数は ginInsertBAEntries からの呼び出しを前提としています。
  • ノードの新規作成時に palloc を使用していますが、メモリ断片化を避けるため、将来的には ginAllocEntryAccumulator 等の専用アロケータとの整合性に注意してください。
