pgp_cfb_decrypt

定義

  • contrib/pgcrypto/pgp-cfb.c:259

目的

PGP暗号化サブシステムにおいて、CFB（Cipher Feedback）モードを用いたデータの復号処理を実行する。

シグネチャまたは形状

int pgp_cfb_decrypt(PGP_CFB *ctx, const uint8 *data, int len, uint8 *dst);

入力と出力

  • ctx: 暗号化コンテキスト（PGP_CFB 構造体）。
  • data: 復号対象の入力データバッファ。
  • len: 入力データの長さ。
  • dst: 復号結果を格納する出力バッファ。
  • 戻り値: 処理結果を示す整数値（cfb_process の戻り値に依存）。

前提条件と不変条件

  • ctx は pgp_cfb_create によって初期化済みであること。
  • ctx->resync フラグの状態に基づき、適切な復号ロジックが選択される。

アルゴリズムまたは振る舞い

1. ctx->resync の値を確認し、mix_decrypt_resync または mix_decrypt_normal のいずれかを復号用関数として選択する。 2. 選択した関数を mix 引数として cfb_process を呼び出し、実際の復号処理を実行する。 3. cfb_process は内部でブロック暗号によるフィードバックレジスタの更新と、入力データとの XOR 演算を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-decrypt.c 内の各箇所から呼び出される。
  • **呼び出し先**: cfb_process を呼び出す。また、ctx->resync の値に応じて mix_decrypt_resync または mix_decrypt_normal を間接的に使用する。

関連データ構造

  • PGP_CFB: 暗号器、現在のブロック位置、再同期フラグ、内部バッファを保持するコンテキスト構造体。

失敗モードと境界条件

  • cfb_process の仕様に依存する（ソースコード上では明示的なエラーハンドリングの詳細は未定義）。

保守担当者向けメモ

  • resync フラグが有効な場合、通常の CFB とは異なるレジスタ更新が行われるため、デバッグ時には ctx->resync の値に注意すること。
  • mix_decrypt_resync は古い PGP 仕様に対応するための特殊なロジックを含んでいる。
