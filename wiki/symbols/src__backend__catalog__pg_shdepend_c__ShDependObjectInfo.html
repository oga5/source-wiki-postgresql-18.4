ShDependObjectInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/catalog/pg_shdepend.c:74

目的

共有依存関係（Shared Dependency）の対象となるオブジェクトの情報を保持し、メモリ上での管理やソートを容易にするためのデータ構造です。

シグネチャまたは形状

typedef struct
{
    ObjectAddress object;
    char        deptype;
    SharedDependencyObjectType objtype;
} ShDependObjectInfo;

入力と出力

本構造体はデータ保持用であり、関数としての入出力はありません。メモリ確保時には palloc を用いて ShDependObjectInfo 型の配列として確保されます。

前提条件と不変条件

  • object フィールドは ObjectAddress 型であり、対象オブジェクトを一意に特定します。
  • deptype は依存関係の性質を、objtype は共有オブジェクトの種別をそれぞれ示します。

アルゴリズムまたは振る舞い

  • pg_shdepend.c 内で、共有依存関係のリストをソートする際の比較対象として利用されます（shared_dependency_comparator 関数で使用）。
  • 共有オブジェクトに関連する一括処理（DROP OWNED 等）において、対象オブジェクトのリストをメモリ上に構築するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/pg_shdepend.c 内の複数の関数（shared_dependency_comparator、およびオブジェクトリストを構築する処理）から参照されています。
  • 呼び出し先: 構造体であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • ObjectAddress: オブジェクトを識別するための基本構造体。
  • SharedDependencyObjectType: 共有依存関係の対象種別を定義する列挙型。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • 構造体内の各フィールドに不正な値が設定された場合、依存関係の検索や削除処理において予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は pg_shdepend.c 内での一時的なデータ管理に特化しています。
  • 比較関数 shared_dependency_comparator を変更する際は、本構造体のフィールド定義との整合性に注意してください。
