DIR

定義

  • 種別: struct
  • 定義箇所: src/port/dirent.c:25

目的

Windows環境において、POSIX準拠のディレクトリ走査を行うための状態保持構造体です。WindowsのネイティブAPI（FindFirstFile/FindNextFile）による検索ハンドルと、現在のディレクトリ情報を管理します。

シグネチャまたは形状

struct DIR
{
    char       *dirname;
    struct dirent ret;
    HANDLE      handle;
};

入力と出力

  • **入力**: opendir() 呼び出し時に初期化され、ディレクトリパスが dirname に保持されます。
  • **出力**: readdir() 呼び出し時に、走査中の現在のファイルエントリ情報が ret メンバを介して呼び出し元へ返されます。

前提条件と不変条件

  • 本構造体はWindows環境専用です。
  • handle は FindFirstFile によって取得された有効なWindows検索ハンドルである必要があります。
  • ret メンバ内の d_ino および d_reclen は、Windows APIの制約により常に 0 に固定されます。

アルゴリズムまたは振る舞い

1. **初期化**: opendir() がパスに * を付与して検索文字列を作成し、FindFirstFile を実行して handle を取得します。 2. **走査**: readdir() が handle を使用して FindNextFile を呼び出し、結果を ret に格納します。 3. **終了**: closedir() が FindClose を呼び出して handle を解放し、メモリをクリーンアップします。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下のバックエンド処理から参照されています。
  • src/backend/access/heap/rewriteheap.c
  • src/backend/access/transam/slru.c
  • src/backend/access/transam/twophase.c
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/backup/basebackup.c
  • **呼び出し先**: Windows API (FindFirstFile, FindNextFile, FindClose) を内部的に利用します。

関連データ構造

  • struct dirent: ret メンバの型であり、ディレクトリ内のファイル情報を保持します。
  • WIN32_FIND_DATA: readdir 内部でファイル属性の変換元として利用されます。

失敗モードと境界条件

  • opendir 失敗時: handle が INVALID_HANDLE_VALUE となり、ディレクトリのオープンに失敗します。
  • リパースポイント: Windows特有のジャンクションやシンボリックリンクが含まれる場合、dwFileAttributes を評価して適切なPOSIXファイルタイプ（DT_LNK 等）へマッピングされます。

保守担当者向けメモ

  • 本構造体は src/include/port/win32_msvc/dirent.h にも定義が存在します。修正時は両方の整合性に注意してください。
  • d_ino 等のPOSIXフィールドが欠落している点は、Windows APIの仕様に基づく意図的な制限です。
