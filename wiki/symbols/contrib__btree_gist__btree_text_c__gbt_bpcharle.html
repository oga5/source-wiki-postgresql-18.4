gbt_bpcharle

定義

  • **種別**: 関数
  • **定義箇所**: contrib/btree_gist/btree_text.c:124

目的

bpchar（空白埋め固定長文字）型に対する「以下（less than or equal to）」の比較を、GiSTインデックスのサポート関数として提供します。

シグネチャまたは形状

static bool gbt_bpcharle(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の bpchar 型データへのポインタ。
  • collation: 照合順序（Collation）のOID。
  • flinfo: 関数マネージャ情報。
  • **出力**: a <= b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は有効な bpchar 型データである必要があります。
  • 内部的に bpcharle 関数を呼び出すため、PostgreSQLの標準的な bpchar 比較ルールに従います。

アルゴリズムまたは振る舞い

1. PointerGetDatum を使用して、入力ポインタを Datum 型に変換します。 2. DirectFunctionCall2Coll を介して、PostgreSQLの標準関数 bpcharle を呼び出します。 3. 比較結果を DatumGetBool で論理値に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し先**: bpcharle（PostgreSQL標準の比較関数）。
  • **呼び出し元**: ソースコード内の静的配列（bptinfo 構造体など）で参照されています。具体的な実行時の呼び出し元は、GiSTインデックスの検索処理（consistent 関数など）から間接的に呼び出されます。

関連データ構造

  • gbtree_vinfo: bpchar 型の比較関数を保持する構造体で使用されます。

失敗モードと境界条件

  • bpcharle 関数に依存するため、照合順序が不正な場合やメモリ不足等の低レベルなエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は bpchar 型専用の比較ロジックをカプセル化しています。
  • text 型の同等関数（gbt_textle）と設計を共有していますが、比較対象の型が異なるため個別に定義されています。
