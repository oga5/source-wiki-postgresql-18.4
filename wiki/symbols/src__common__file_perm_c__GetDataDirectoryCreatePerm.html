GetDataDirectoryCreatePerm

定義

  • 種別: function
  • 定義箇所: src/common/file_perm.c:65

目的

指定されたデータディレクトリの現在のアクセス権限（stat）を取得し、その設定をシステム全体のファイル作成モード（pg_file_create_mode 等）に反映させる。

シグネチャまたは形状

bool GetDataDirectoryCreatePerm(const char *dataDir)

入力と出力

  • **入力**: dataDir (データディレクトリのパスを示す文字列)
  • **出力**: 成功した場合は true、ディレクトリのアクセスに失敗した場合は false を返す。

前提条件と不変条件

  • 本関数はフロントエンドツール（initdb 等）での利用を想定している。
  • Windows環境では、stat の取得のみを行い、権限の自動設定はスキップされる。

アルゴリズムまたは振る舞い

1. stat(dataDir, &statBuf) を呼び出し、指定ディレクトリの情報を取得する。 2. stat が失敗した場合は false を返して終了する。 3. WindowsおよびCygwin以外の環境では、取得した statBuf.st_mode を引数として SetDataDirectoryCreatePerm を呼び出し、グローバルな作成モード変数を更新する。 4. 成功時に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_ctl/pg_ctl.c
  • src/bin/pg_resetwal/pg_resetwal.c
  • src/bin/pg_rewind/pg_rewind.c
  • src/bin/pg_upgrade/pg_upgrade.c
  • **呼び出し先**:
  • stat (システムコール)
  • SetDataDirectoryCreatePerm (同一ファイル内で定義)

関連データ構造

  • struct stat: ファイルシステム上の属性を保持する構造体。

失敗モードと境界条件

  • 指定されたパスが存在しない、またはアクセス権限がないために stat が失敗した場合、false を返す。
  • Windows環境では、権限の自動設定（SetDataDirectoryCreatePerm）が実行されないため、呼び出し側は環境による挙動の違いを考慮する必要がある。

保守担当者向けメモ

  • 本関数は #ifdef FRONTEND で囲まれているため、サーバー本体のコードからは参照できない。
  • ファイル作成時の権限一貫性を保つため、データディレクトリを扱うツールでは初期化時に必ず本関数を呼び出すことが推奨される。
