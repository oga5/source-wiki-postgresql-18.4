ExecInitAgg

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAgg.c:3278

目的

集約演算（Agg ノード）の実行状態を初期化し、集約処理に必要なメモリコンテキスト、式評価コンテキスト、および子ノードの準備を行う。

シグネチャまたは形状

AggState *ExecInitAgg(Agg *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の Agg プランノード。
  • estate: クエリ実行状態を保持する EState。
  • eflags: 実行フラグ（EXEC_FLAG_BACKWARD や EXEC_FLAG_MARK はサポート外）。
  • **出力**:
  • 初期化済みの AggState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK が含まれていないこと（Assert で検証）。
  • ハッシュ集約（AGG_HASHED）の場合、子ノードの REWIND フラグは無効化される。
  • numPhases が 2 を超える場合、タプルソート用のスロット（sort_slot）が確保される。

アルゴリズムまたは振る舞い

1. AggState 構造体を割り当て、ExecAgg を実行関数として設定する。 2. 集約戦略（ソート/ハッシュ/混合）に基づき、フェーズ数とハッシュ数を決定する。 3. ExecAssignExprContext を使用して、入力タプル処理用、出力用、および各グループ化セット用の ExprContext を生成する。 4. ハッシュ集約が必要な場合、hash_create_memory を呼び出してハッシュテーブルを準備する。 5. ExecInitNode を呼び出し、子プランノードを再帰的に初期化する。 6. 入力タプルのスキャン用スロットを作成し、必要に応じてソート用スロットを初期化する。 7. 最終的な結果タプルスロットと射影（Projection）情報を初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:341（ExecInitNode 経由での呼び出しが確認されている）。
  • **呼び出し先**:
  • makeNode
  • ExecAssignExprContext
  • hash_create_memory
  • ExecInitNode
  • ExecGetResultSlotOps
  • ExecCreateScanSlotFromOuterPlan
  • ExecInitExtraTupleSlot
  • ExecInitResultTupleSlotTL
  • ExecAssignProjectionInfo

関連データ構造

  • AggState: 集約ノードの実行状態を保持するメイン構造体。
  • Agg: プランノード。
  • EState: クエリ実行状態。
  • ExprContext: 式評価のためのメモリコンテキスト。

失敗モードと境界条件

  • eflags に不正なフラグが渡された場合、Assert により停止する。
  • メモリ不足時は palloc 関連関数によりエラーが発生する可能性がある。

保守担当者向けメモ

  • aggcontexts はグループ境界（ハッシュ集約以外）や ExecReScanAgg 時にリセットされるため、メモリ管理の整合性に注意すること。
  • numPhases > 2 の場合の outeropsfixed の調整は、ソート処理と入力スロットの型不一致を回避するための重要な最適化である。
