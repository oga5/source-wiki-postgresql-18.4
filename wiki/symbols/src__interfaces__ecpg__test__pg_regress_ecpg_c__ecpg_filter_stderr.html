ecpg_filter_stderr

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/pg_regress_ecpg.c:92

目的

ECPGの回帰テスト実行時に生成されるエラーログ（stderr）から、環境依存の可変情報（ホスト名やパスなど）を除去し、テスト結果の比較可能性を確保する。

シグネチャまたは形状

static void ecpg_filter_stderr(const char *resultfile, const char *tmpfile)

入力と出力

  • **入力**:
  • resultfile: フィルタリング対象のファイルパス。
  • tmpfile: 処理中に使用する一時ファイルパス。
  • **出力**: なし（副作用として resultfile をインプレースで書き換える）。

前提条件と不変条件

  • resultfile は読み取り可能、tmpfile は書き込み可能であること。
  • 処理終了後、tmpfile は resultfile にリネームされ、元のファイルは上書きされる。
  • ファイルオープンやリネームに失敗した場合は exit(2) によりプロセスが終了する。

アルゴリズムまたは振る舞い

1. resultfile を読み込みモード、tmpfile を書き込みモードで開く。 2. pg_get_line_buf を使用して、各行を StringInfoData バッファに読み込む。 3. 各行に "connection to server " という文字列が含まれているか検索する。 4. 該当する場合、その後の "failed: " という文字列を探し、接続先ホスト名などの可変部分を削除してメッセージを短縮する。 5. フィルタリング後の内容を tmpfile に書き出す。 6. 最後に tmpfile を resultfile にリネームして置換する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/pg_regress_ecpg.c 内の ecpg_postprocess_result（推測される文脈上の呼び出し元）。
  • **呼び出し先**:
  • fopen, fclose, rename (標準ライブラリ)
  • initStringInfo, pg_get_line_buf, pfree (PostgreSQL 内部ユーティリティ)

関連データ構造

  • StringInfoData: 行単位の文字列処理に使用される動的文字列バッファ。

失敗モードと境界条件

  • ファイルが開けない場合、標準エラー出力にメッセージを出力して終了コード 2 で異常終了する。
  • リネームに失敗した場合も同様に終了コード 2 で異常終了する。
  • 検索対象の文字列パターン（"connection to server " や "failed: "）が見つからない行は、変更されずにそのまま出力される。

保守担当者向けメモ

  • 本関数はファイルをインプレースで書き換えるため、処理の中断が発生すると resultfile が破損する可能性がある。
  • memmove を使用して文字列を短縮しているが、linebuf.len の更新は行われていないため、後続の処理で linebuf を再利用する際は注意が必要である。
