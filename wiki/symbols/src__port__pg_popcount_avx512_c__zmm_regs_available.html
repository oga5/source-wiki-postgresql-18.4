zmm_regs_available

定義

  • 種別: function
  • 定義箇所: src/port/pg_popcount_avx512.c:63

目的

OSおよびCPUがAVX-512命令セットで使用されるZMMレジスタの状態保存（XSAVE）をサポートしているかを確認する。

シグネチャまたは形状

static inline bool zmm_regs_available(void)

入力と出力

  • 入力: なし
  • 出力: bool 型。ZMMレジスタの状態がOSによって管理・保存可能であれば true、そうでなければ false を返す。

前提条件と不変条件

  • HAVE_XSAVE_INTRINSICS マクロが定義されている環境でのみ、実際のハードウェアチェックが行われる。
  • 未定義の場合は常に false を返す。

アルゴリズムまたは振る舞い

1. HAVE_XSAVE_INTRINSICS が定義されている場合、_xgetbv(0) を呼び出して拡張制御レジスタ（XCR0）の値を取得する。 2. 取得した値に対して 0xe6 とのビット論理積（AND）を行い、結果が 0xe6 と一致するかを判定する。 3. 一致すればZMMレジスタの使用が可能と判断し true を返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/port/pg_crc32c_sse42_choose.c (行47, 86)
  • src/port/pg_popcount_avx512.c (行64, 101)
  • 呼び出し先:
  • _xgetbv (組み込み関数)

関連データ構造

  • なし

失敗モードと境界条件

  • HAVE_XSAVE_INTRINSICS が定義されていない環境では、ハードウェアの能力に関わらず常に false を返す。
  • _xgetbv がサポートされていない環境や、OSがXCR0へのアクセスを許可していない環境では、実行時エラーや不正な値の取得が発生する可能性がある。

保守担当者向けメモ

  • 本関数は pg_popcount_avx512.c 内の実行時判定ロジックの一部であり、AVX-512命令の実行前に安全性を確保するために使用される。
  • 0xe6 というマジックナンバーは、AVX-512に必要なレジスタ状態（XCR0のビット0, 1, 2, 5, 6, 7）を検証するためのビットマスクである。
