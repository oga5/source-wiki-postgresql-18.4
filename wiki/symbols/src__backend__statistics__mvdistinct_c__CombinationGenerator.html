CombinationGenerator

定義

  • 種別: struct
  • 定義箇所: src/backend/statistics/mvdistinct.c:61

目的

多変量NDistinct（列の組み合わせごとの一意な値の数）を推定する際、指定された列数から可能なすべての組み合わせを効率的に生成・管理するための内部状態保持構造体です。

シグネチャまたは形状

typedef struct CombinationGenerator
{
	int			k;				/* 組み合わせのサイズ */
	int			n;				/* 要素の総数 */
	int			current;		/* 次に返す組み合わせのインデックス */
	int			ncombinations;	/* 組み合わせの総数（配列サイズ） */
	int		   *combinations;	/* 事前に構築された組み合わせの配列 */
} CombinationGenerator;

入力と出力

  • 入力: generator_init 関数により、総要素数 n と組み合わせサイズ k を指定して初期化されます。
  • 出力: generator_next 関数を通じて、combinations 配列から順次、特定の組み合わせデータが返されます。

前提条件と不変条件

  • ncombinations は、n 個から k 個を選ぶ組み合わせの総数と一致する必要があります。
  • combinations 配列は、generate_combinations 関数によって事前に計算・格納されている必要があります。
  • current は常に 0 以上 ncombinations 未満の範囲内である必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: generator_init がメモリを確保し、n と k を設定します。 2. **生成**: generate_combinations（および再帰的な generate_combinations_recurse）が、可能なすべての組み合わせを計算し、フラットな配列として combinations に格納します。 3. **反復**: generator_next が呼び出されるたびに、current インデックスを更新しながら、次の組み合わせを指すポインタを返します。 4. **解放**: generator_free により、構造体および内部の combinations 配列が解放されます。

呼び出し元と呼び出し先

  • 呼び出し元: statext_ndistinct_build が統計オブジェクト構築の過程で本構造体を利用します。
  • 呼び出し先: palloc（メモリ確保）、generator_init、generator_next、generator_free、generate_combinations、generate_combinations_recurse が本構造体を操作します。

関連データ構造

  • pg_ndistinct: 本構造体を用いて計算された結果を保持するバイナリ形式の構造体。

失敗モードと境界条件

  • **組み合わせ爆発**: 列数が増加すると組み合わせ数が指数関数的に増大するため、統計オブジェクト作成時の列数には制限があります。
  • **メモリ不足**: palloc によるメモリ確保が失敗した場合、PostgreSQLの標準的なエラーハンドリングにより中断されます。

保守担当者向けメモ

  • 本構造体は mvdistinct.c 内でのみ使用される内部ユーティリティです。
  • combinations 配列はフラット化されているため、アクセス時は k をオフセットとして計算する必要があります。
  • 再帰的な組み合わせ生成ロジックを変更する場合は、generate_combinations_recurse のスタック消費量に注意してください。
