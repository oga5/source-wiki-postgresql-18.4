tsquery_numnode

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_op.c:22

目的

tsquery 型のデータ内に含まれるノード（演算子およびオペランド）の総数を取得します。

シグネチャまたは形状

Datum tsquery_numnode(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される tsquery 型の引数。
  • **出力**: int32 型のノード数。

前提条件と不変条件

  • 入力された tsquery オブジェクトは有効である必要があります。
  • query->size フィールドがノード数を保持しているという構造上の前提に基づいています。

アルゴリズムまたは振る舞い

1. PG_GETARG_TSQUERY(0) を使用して、入力引数から TSQuery 構造体を取得します。 2. 構造体内の size メンバを参照し、ノード数を取得します。 3. PG_FREE_IF_COPY を呼び出し、必要に応じて入力引数のメモリを解放します。 4. PG_RETURN_INT32 を使用して、取得したノード数を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定できません。
  • **呼び出し先**:
  • PG_GETARG_TSQUERY (マクロ)
  • PG_FREE_IF_COPY (マクロ)
  • PG_RETURN_INT32 (マクロ)

関連データ構造

  • TSQuery: 全文検索クエリを保持するデータ型。size メンバにノード数を含みます。

失敗モードと境界条件

  • 入力引数が NULL の場合、PG_GETARG_TSQUERY の挙動に依存します。
  • tsquery 構造体が不正なメモリレイアウトである場合、query->size の参照時に予期せぬ動作が発生する可能性があります。

保守担当者向けメモ

  • 本関数は tsquery の内部構造（size フィールド）に直接依存しています。構造体の定義が変更された場合は、本関数の実装も確認が必要です。
