int64KEY

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_int8.c:11

目的

PostgreSQLのGiSTインデックスにおいて、int8（64ビット整数）型の範囲を表現するためのデータ構造です。

シグネチャまたは形状

typedef struct int64key
{
    int64 lower;
    int64 upper;
} int64KEY;

入力と出力

  • 入力: 64ビット整数のペア（下限値および上限値）。
  • 出力: GiSTインデックスのノードに格納される圧縮されたキー形式。

前提条件と不変条件

  • リーフノードにおいては、lower == upper となる不変条件を持ちます。
  • lower は範囲の開始値、upper は範囲の終了値を表します。

アルゴリズムまたは振る舞い

  • GiSTインデックス構築時、個々の int8 値は本構造体に変換されます。
  • 検索時には、クエリ条件と本構造体の lower/upper を比較することで、範囲の包含関係を判定します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/btree_gist/btree_int8.c 内の各サポート関数（gbt_int8_compress, gbt_int8_consistent, gbt_int8_fetch 等）から参照されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • int8: 本構造体が表現する対象のデータ型。
  • GISTENTRY: GiSTインデックスの各エントリを保持する構造体。本構造体へのポインタが Datum として格納されます。

失敗モードと境界条件

  • 整数演算時のオーバーフローについては、int8_dist 等の関連関数が適切に処理を行う設計となっています。

保守担当者向けメモ

  • 本構造体は int8 型専用のラッパーです。新しい数値型を追加する際は、本ファイルをテンプレートとして参照してください。
  • 複雑なロジックは btree_utils_num.c に委譲されているため、共通ロジックの変更時には影響範囲を確認してください。
