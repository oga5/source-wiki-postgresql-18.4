placeOne

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistsplit.c:199

目的

GiSTインデックスのページ分割において、特定のインデックス・タプルを左ページまたは右ページのいずれに配置すべきかを、指定された列以降のペナルティに基づいて決定します。

シグネチャまたは形状

static void placeOne(Relation r, GISTSTATE *giststate, GistSplitVector *v, IndexTuple itup, OffsetNumber off, int attno)

入力と出力

  • **入力**:
  • r: 対象のリレーション。
  • giststate: GiSTインデックスの操作状態。
  • v: 分割結果を保持するベクトル構造体。
  • itup: 配置を判定するインデックス・タプル。
  • off: タプルのオフセット番号。
  • attno: 判定を開始する列インデックス。
  • **出力**: なし（副作用として v 内の spl_left または spl_right にオフセットが追加されます）。

前提条件と不変条件

  • giststate は初期化済みであること。
  • attno は有効な列インデックスであること。
  • v が指す GistSplitVector は、タプルを追加可能な状態であること。

アルゴリズムまたは振る舞い

1. gistDeCompressAtt を使用し、対象タプル itup を全列の属性値に展開します。 2. attno から開始し、インデックスの最終列までループします。 3. 各列において、現在の左ページおよび右ページのユニオンキーに対するペナルティを gistpenalty で計算します。 4. 左と右のペナルティが異なる場合、ペナルティが小さい方のページを選択してループを終了します。 5. ペナルティが等しい場合は、次の列へ進んで判定を継続します。 6. 最終的に決定したページ（spl_left または spl_right）にオフセット off を追加し、対応するカウンタをインクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistsplit.c 内の再帰的な分割処理から呼び出されます。
  • **呼び出し先**:
  • gistDeCompressAtt: タプルの属性展開。
  • gistentryinit: 比較用エントリーの初期化。
  • gistpenalty: ページ配置コストの計算。

関連データ構造

  • GISTSTATE: GiSTインデックスのメタデータおよび状態管理。
  • GistSplitVector: 分割後の左右ページへのタプル割り当てを保持する構造体。
  • IndexTuple: インデックス内の個々のタプル。

失敗モードと境界条件

  • attno がインデックスの列数を超えた場合、ループは実行されず、デフォルトで左ページに配置されます。
  • すべての列でペナルティが等しい場合、最終的に左ページへ配置されます。

保守担当者向けメモ

  • 本関数は、マルチカラムGiSTインデックスにおいて、最初の列で分割を決定できない（ペナルティが等しい）場合に、後続の列を評価して最適化を行う重要な役割を担います。
  • gistpenalty の結果に依存するため、インデックスの品質は各列のペナルティ関数の精度に左右されます。
