buildSubPlanHash

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSubplan.c:476

目的

ANY サブリンクの評価を高速化するため、サブクエリの実行結果をメモリ上のハッシュテーブルにロードする。

シグネチャまたは形状

static void buildSubPlanHash(SubPlanState *node, ExprContext *econtext)

入力と出力

  • **入力**:
  • node: SubPlanState 構造体。ハッシュテーブルの格納先および実行に必要なプラン情報を含む。
  • econtext: 式評価コンテキスト。サブクエリ実行時のメモリ管理に使用。
  • **出力**: なし（node 内の hashtable および hashnulls が更新される）。

前提条件と不変条件

  • subplan->subLinkType が ANY_SUBLINK であること。
  • node->hashtablecxt および node->hashtempcxt が初期化済みであること。
  • ExecProject を介した入力スロットを使用するため、TTSOpsVirtual を使用する。

アルゴリズムまたは振る舞い

1. **初期化**: hashtablecxt をリセットし、既存のハッシュテーブルをリセットまたは新規作成する。 2. **ハッシュテーブルの構築**:

  • subplan->unknownEqFalse が偽の場合、NULLを含む行を格納するための hashnulls テーブルを別途作成する。

    • 3. **サブクエリの実行**:

  • ExecReScan でサブクエリを再開する。
  • ExecProcNode でサブクエリの全行をスキャンする。
  • 各行を ExecProject で投影し、NULLの有無に応じて hashtable または hashnulls に格納する。

    • 4. **クリーンアップ**: 各行の処理後に innerecontext と hashtempcxt をリセットし、メモリリークを防ぐ。最後に projRight のスロットをクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecHashSubPlan（src/backend/executor/nodeSubplan.c 内）。
  • **呼び出し先**: ExecReScan, ExecProcNode, ExecProject, BuildTupleHashTable, LookupTupleHashEntry, ResetTupleHashTable, ResetExprContext, MemoryContextReset, MemoryContextSwitchTo。

関連データ構造

  • SubPlanState: サブプランの実行状態を保持する主要構造体。
  • TupleHashTable: サブクエリ結果を保持するハッシュテーブル。
  • ExprContext: 式評価およびメモリコンテキスト管理に使用。

失敗モードと境界条件

  • nbuckets が 1 未満の場合は 1 に補正される。
  • subplan->unknownEqFalse が真の場合、hashnulls は NULL に設定され、NULLを含む行は無視される。
  • 重複行はハッシュテーブル内で一意にまとめられる。

保守担当者向けメモ

  • hashtempcxt はルックアップごとにリセットされるため、このコンテキストに永続的なデータを置かないこと。
  • ExecClearTuple は、サブクエリのコンテキストリセット時に発生しうる二重解放を防ぐために重要である。
