pg_atomic_fetch_add_u32_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/generic.h:172

目的

ハードウェアによる直接的なアトミック加算命令が利用できない環境において、compare_exchange 操作を組み合わせてアトミックな加算を実現するフォールバック実装を提供します。

シグネチャまたは形状

static inline uint32 pg_atomic_fetch_add_u32_impl(volatile pg_atomic_uint32 *ptr, int32 add_)

入力と出力

  • **入力**:
  • ptr: 操作対象となる pg_atomic_uint32 型のポインタ。
  • add_: 加算する値（int32）。
  • **出力**:
  • 加算前の元の値を uint32 として返します。

前提条件と不変条件

  • 本関数は atomics.h を経由して呼び出されることを前提としています。
  • ptr は volatile 修飾されており、コンパイラによる最適化で読み込みが省略されないことが保証されています。

アルゴリズムまたは振る舞い

1. ptr->value から現在の値を読み込み、変数 old に格納します（この読み込み自体はアトミックである必要はありません）。 2. pg_atomic_compare_exchange_u32_impl を使用して、ptr の値が old と一致するかを確認します。 3. 一致すれば old + add_ を書き込み、ループを終了します。 4. 一致しなければ、pg_atomic_compare_exchange_u32_impl が ptr の最新の値を old に更新するため、成功するまでステップ2〜3の「CASループ」を繰り返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/port/atomics.h:369
  • src/include/port/atomics/generic.h:188 (pg_atomic_fetch_sub_u32_impl の実装内)
  • src/include/port/atomics/generic.h:223
  • src/include/port/atomics/generic.h:241
  • **呼び出し先**:
  • pg_atomic_compare_exchange_u32_impl

関連データ構造

  • pg_atomic_uint32: アトミック操作対象の 32 ビット整数型。

失敗モードと境界条件

  • 高負荷な競合環境下では、CASループの再試行回数が増加し、パフォーマンスが低下する可能性があります。
  • 整数オーバーフロー時の挙動は、標準的な C 言語の符号付き整数加算のルールに従います。

保守担当者向けメモ

  • 本関数はフォールバック層であるため、アーキテクチャ固有の最適化命令（arch-*.h）が存在する場合はそちらが優先されます。
  • CASループの性質上、競合が激しい環境ではアーキテクチャ固有の命令への置き換えを検討してください。
