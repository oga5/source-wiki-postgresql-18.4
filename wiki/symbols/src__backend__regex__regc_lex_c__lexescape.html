lexescape

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_lex.c:600

目的

ARE（Advanced Regular Expressions）におけるバックスラッシュエスケープシーケンスを解析し、対応するトークンを生成する。

シグネチャまたは形状

static int lexescape(struct vars *v)

入力と出力

  • **入力**: struct vars *v（解析状態を保持する構造体）。v->now がエスケープ文字の直後の文字を指している必要がある。
  • **出力**: int 型のトークンタイプ。成功時は RETV マクロを通じてトークンをセットし、失敗時は FAILW マクロでエラーを報告する。

前提条件と不変条件

  • v->cflags に REG_ADVF フラグが設定されていること。
  • 入力文字列が終端（EOS）に達していないこと（!ATEOS()）。

アルゴリズムまたは振る舞い

1. v->now が指す文字を読み取り、英数字（ASCII）以外であればそのまま PLAIN トークンとして返す。 2. 英数字の場合、switch 文でエスケープの種類を判定する。

  • a, b, e, f, n, r, t, v: 各種制御文字や特殊文字を PLAIN トークンとして返す。
  • A, Z: 文字列の先頭（SBEGIN）または末尾（SEND）を返す。
  • d, D, s, S, w, W: 文字クラス（CCLASSS / CCLASSC）を返す。
  • u, U, x: lexdigits を呼び出し、指定された桁数の16進数を解析して PLAIN トークンとして返す。
  • y, Y: 単語境界（WBDRY / NWBDRY）を返す。
  • 1-9: 後方参照（BACKREF）を試みる。失敗した場合は8進数として解釈する。
  • 0: 8進数として解釈する。

    • 3. 未定義の英数字エスケープシーケンスに対しては REG_EESCAPE エラーを発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regc_lex.c 内の字句解析ループ（next 関数など）。
  • **呼び出し先**: lexdigits（数値解析）、chrnamed（名前付き文字の解決）。

関連データ構造

  • struct vars: 解析状態、現在の位置、フラグ、サブエクスプレッション数などを保持する。

失敗モードと境界条件

  • lexdigits が不正な値を返した場合や、範囲外の文字コードが指定された場合は REG_EESCAPE で失敗する。
  • 終端に達した後の不正なエスケープ指定はエラーとなる。
  • 後方参照の解析において、数値がサブエクスプレッション数を超過する場合の挙動は、ヒューリスティックな判定に基づき8進数解釈へフォールバックする。

保守担当者向けメモ

  • 本関数は regcomp.c に展開されるため、v ポインタを通じて暗黙的に共有される状態に依存している。
  • REG_UUNPORT や REG_ULOCALE などの NOTE マクロは、移植性やロケール依存性に関する警告を記録するために使用されている。
  • 新しいエスケープシーケンスを追加する場合は、switch 文の default 節でエラーを返す仕様を維持すること。
