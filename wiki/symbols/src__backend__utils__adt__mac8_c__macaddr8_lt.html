macaddr8_lt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mac8.c:337

目的

2つの macaddr8 型の値が「より小さい（less than）」という関係にあるかを判定し、SQLの比較演算子 < をサポートします。

シグネチャまたは形状

Datum macaddr8_lt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの macaddr8 型へのポインタを受け取ります。
  • **出力**: Datum 型として、第1引数が第2引数より小さい場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力される macaddr8 ポインタは有効なメモリ領域を指している必要があります。
  • 内部比較ロジックは macaddr8_cmp_internal に依存しており、8バイトのデータ構造が正しく初期化されていることが前提です。

アルゴリズムまたは振る舞い

1. PG_GETARG_MACADDR8_P マクロを使用して、引数から2つの macaddr8 構造体へのポインタを取得します。 2. macaddr8_cmp_internal 関数を呼び出し、2つの値を比較します。 3. 比較結果が 0 未満（第1引数が小さい）であれば true を、そうでなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_macaddr8.c:55（直接的な呼び出しの証拠あり）
  • **呼び出し先**:
  • macaddr8_cmp_internal（直接的な呼び出しの証拠あり）

関連データ構造

  • macaddr8: 8バイト（EUI-64）のMACアドレスを保持する構造体。

失敗モードと境界条件

  • 入力引数が NULL の場合、PG_GETARG_MACADDR8_P の仕様に従い、呼び出し元で適切に処理される必要があります。
  • 比較ロジックは macaddr8_cmp_internal の実装に依存するため、構造体のメモリレイアウトやバイト順序が不正な場合、期待しない比較結果となる可能性があります。

保守担当者向けメモ

  • 本関数は macaddr8_cmp_internal のラッパーです。比較ロジックに変更を加える場合は、macaddr8_cmp_internal を参照してください。
  • B-treeインデックス等の比較演算子として利用されるため、一貫した順序付けが維持される必要があります。
