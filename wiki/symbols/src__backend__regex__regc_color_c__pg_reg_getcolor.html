pg_reg_getcolor

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_color.c:119

目的

MAX_SIMPLE_CHR を超える文字コード（chr）に対し、カラーマップから対応する「色（color）」を検索して返します。

シグネチャまたは形状

color pg_reg_getcolor(struct colormap *cm, chr c);

入力と出力

  • **入力**:
  • cm: カラーマップ構造体へのポインタ。
  • c: 検索対象の文字コード（chr 型）。
  • **出力**:
  • 該当する文字コードに割り当てられた color 値。

前提条件と不変条件

  • c > MAX_SIMPLE_CHR であること（locolormap に収まらない文字のみを対象とする）。
  • cm->cmranges は昇順にソートされていること。

アルゴリズムまたは振る舞い

1. **行の特定**: cm->cmranges に対して二分探索を行い、文字 c が含まれる範囲に対応する rownum を特定します。一致しない場合は rownum = 0 とします。 2. **列の特定**:

  • cm->hiarraycols > 1 の場合、cclass_column_index を呼び出して列番号を算出し、hicolormap 配列から色を取得します。
  • それ以外（hiarraycols <= 1）の場合は、rownum を直接インデックスとして hicolormap から色を取得します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/regex/regguts.h 内の GETCOLOR() マクロ（c > MAX_SIMPLE_CHR の場合に呼び出される）。
  • **呼び出し先**:
  • cclass_column_index（ロケール依存の列インデックス算出）。

関連データ構造

  • colormap: カラーマップ全体を管理する構造体。
  • colormaprange: 文字範囲と行番号の対応を保持する構造体。
  • color: 色を表す整数型。

失敗モードと境界条件

  • c がどの範囲にも該当しない場合、デフォルトの rownum = 0 が使用されます。
  • cm->hiarraycols が 1 以下の場合、列計算をスキップする最適化パスが実行されます。

保守担当者向けメモ

  • 本関数は GETCOLOR() マクロの「低速パス（slow case）」として実装されています。
  • cclass_column_index はロケールに依存するため、実行環境のロケール設定によって結果が変化する可能性があります。
