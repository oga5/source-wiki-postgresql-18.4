getladfa

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexec.c:401

目的

LACON（Look-Ahead Constraint）サブ式ノードに対応するDFAを生成、またはキャッシュから取得します。

シグネチャまたは形状

static struct dfa *getladfa(struct vars *v, int n);

入力と出力

  • **入力**:
  • v: マッチング状態を保持する vars 構造体へのポインタ。
  • n: 取得対象のLACONインデックス。
  • **出力**:
  • 指定されたインデックスに対応する struct dfa へのポインタ。

前提条件と不変条件

  • n は 0 より大きく、かつ v->g->nlacons 未満であること。
  • v->g->lacons が NULL でないこと。
  • LACONはバックリファレンスを含まない前提で処理されます。

アルゴリズムまたは振る舞い

1. v->ladfas[n] が既に存在するか確認します。 2. キャッシュが存在しない場合、newdfa を呼び出して新しいDFAを構築し、v->ladfas[n] に格納します。 3. 構築時には DOMALLOC フラグが使用されます。 4. 最終的にキャッシュされた、または新規作成された v->ladfas[n] を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/rege_dfa.c:947
  • **呼び出し先**:
  • newdfa (内部で呼び出されるDFA構築関数)

関連データ構造

  • struct vars: マッチングのコンテキスト。
  • struct dfa: 遅延評価されるDFAの状態管理構造体。
  • struct subre: 正規表現のサブ式ノード（LACONの定義元）。

失敗モードと境界条件

  • n が範囲外の場合、assert により異常終了します。
  • メモリ不足時には newdfa 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はLACON専用のDFAキャッシュ管理を行います。LACONはバックリファレンスを含まないため、単純なDFA構築で完結します。
