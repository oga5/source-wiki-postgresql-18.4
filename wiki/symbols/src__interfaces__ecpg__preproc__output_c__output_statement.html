output_statement

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/output.c:133

目的

ECPGプリプロセッサにおいて、SQLステートメントをランタイムライブラリ関数 ECPGdo を呼び出すCコードに変換し、出力ストリームへ書き出します。

シグネチャまたは形状

void output_statement(const char *stmt, int whenever_mode, enum ECPG_statement_type st)

入力と出力

  • **入力**:
  • stmt: 出力対象のSQL文または変数名。
  • whenever_mode: エラーハンドリングの動作を指定するフラグ。
  • st: ステートメントの種類（ECPG_statement_type 列挙型）。
  • **出力**: なし（副作用として base_yyout ストリームへCコードを出力）。

前提条件と不変条件

  • base_yyout が書き込み可能なファイルストリームとして初期化されていること。
  • argsinsert および argsresult が、呼び出し前に適切に設定されていること。
  • compat, force_indicator, connection, questionmarks などのグローバル変数が、現在のプリプロセスコンテキストで有効であること。

アルゴリズムまたは振る舞い

1. ECPGdo 関数の開始部分を base_yyout に出力します。 2. st が ECPGst_prepnormal かつ auto_prepare が偽の場合、st を ECPGst_normal に補正します。 3. ステートメントの種類に応じて、SQL文をそのまま出力するか、output_escaped_str を介してエスケープ処理した文字列を出力します。 4. dump_variables を呼び出し、入力引数（argsinsert）と出力引数（argsresult）をCコードとして展開します。 5. ECPGdo の呼び出しを完了させます。 6. whenever_action を呼び出し、指定された whenever_mode に基づくエラー処理コードを挿入します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内で直接的な呼び出し元は特定されていませんが、src/interfaces/ecpg/preproc/preproc_extern.h で宣言されており、パーサー等の主要ロジックから呼び出されることが想定されます。
  • **呼び出し先**:
  • output_escaped_str
  • dump_variables
  • whenever_action

関連データ構造

  • ECPG_statement_type: ステートメントの種類を定義する列挙型。
  • base_yyout: 出力先のファイルストリーム。
  • argsinsert, argsresult: 変数リストを保持するグローバル変数。

失敗モードと境界条件

  • stmt が空文字列や不正な形式の場合、生成されるCコードがコンパイルエラーを引き起こす可能性があります。
  • whenever_mode の指定が不適切な場合、生成されたコード内のエラーハンドリングブロックが意図しないスコープで動作する可能性があります。

保守担当者向けメモ

  • whenever_action(whenever_mode | 2) のビットフラグ 2 は、処理のスコープ終了を制御するために使用されています。
  • dump_variables 呼び出し後に argsinsert と argsresult を NULL にリセットしており、呼び出しごとの状態クリアを保証しています。
