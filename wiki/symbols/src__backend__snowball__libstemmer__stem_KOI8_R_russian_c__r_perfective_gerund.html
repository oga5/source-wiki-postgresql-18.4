r_perfective_gerund

目的

ロシア語のステミング処理において、単語末尾の完了体動詞分詞（perfective gerund）を特定し、該当する接尾辞を削除する。

シグネチャまたは形状

static int r_perfective_gerund(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（ステミング環境構造体）。現在の単語データおよびカーソル位置（z->c）を含む。
  • **出力**: int 型。処理が成功した場合は 1、マッチしなかった場合や処理が不要な場合は 0、エラー発生時は負の値を返す。

前提条件と不変条件

  • z->c が有効な単語範囲内にあること。
  • find_among_b を使用するため、単語は末尾から前方に向かって走査される。

アルゴリズムまたは振る舞い

1. **事前チェック**: 現在のカーソル位置の文字が、完了体分詞の接尾辞として妥当な範囲内にあるかビット演算で高速判定する。 2. **パターンマッチング**: a_0 配列を用いて、単語末尾から接尾辞を検索する。 3. **条件分岐**:

  • case 1: 特定の接尾辞（0xC1 または 0xD1 で終わるもの）を判定し、条件を満たせば slice_del で削除する。
  • case 2: その他の該当接尾辞を slice_del で削除する。

    • 4. **終了**: 処理が完了すれば 1 を返し、マッチしなければ 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/snowball/libstemmer/stem_KOI8_R_russian.c 内のメイン処理フローから呼び出される。
  • src/backend/snowball/libstemmer/stem_UTF_8_russian.c からも参照されている。
  • **呼び出し先**:
  • find_among_b（接尾辞検索）
  • slice_del（文字列削除）

関連データ構造

  • struct SN_env: ステミングの実行状態を保持する構造体。
  • a_0: 完了体分詞の接尾辞パターンを格納した among 構造体の配列。

失敗モードと境界条件

  • z->c <= z->lb: カーソルが左境界を超えている場合は 0 を返す。
  • slice_del が負の値を返した場合、そのエラーコードをそのまま伝播させる。
  • マッチする接尾辞がない場合は 0 を返す。
