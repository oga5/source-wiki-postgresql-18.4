spgxlogSplitTuple

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgxlog.h:141

目的

SP-GiST インデックスにおけるタプル分割操作の WAL（Write-Ahead Log）レコード構造を定義します。インデックスの再構築やリカバリ時に、分割されたプレフィックスおよびポストフィックス・タプルの配置情報を保持するために使用されます。

シグネチャまたは形状

typedef struct spgxlogSplitTuple
{
    OffsetNumber offnumPrefix;
    OffsetNumber offnumPostfix;
    bool         newPage;
    bool         postfixBlkSame;
    /* 可変長データ（プレフィックスおよびポストフィックスのインナータプル）が続く */
} spgxlogSplitTuple;

入力と出力

  • **入力**: インデックス操作時に本構造体に値を設定し、WAL へ書き出します。
  • **出力**: リカバリ時、spg_redo 関数が本構造体を読み取り、ページ上のアイテム配置を復元します。

前提条件と不変条件

  • 構造体直後に続くプレフィックスおよびポストフィックスのインナータプルは、アライメントされていない（unaligned）状態で格納されます。
  • 構造体サイズは sizeof(spgxlogSplitTuple) で取得可能ですが、可変長データを含む場合はポインタ演算によるアクセスが必要です。

アルゴリズムまたは振る舞い

1. **記録**: spgdoinsert.c 等で分割が発生した際、この構造体にオフセット情報とフラグを格納し、WAL に書き込みます。 2. **再生**: リカバリ時、spgxlog.c 内の処理でこの構造体をキャストして読み込みます。newPage フラグが真であればページ初期化を行い、postfixBlkSame に基づいてタプルの配置先を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgdoinsert.c: 分割操作時に本構造体を初期化。
  • **呼び出し先**:
  • src/backend/access/spgist/spgxlog.c: spg_redo 等のリカバリ処理で本構造体のデータを参照。
  • src/backend/access/rmgrdesc/spgdesc.c: WAL ログのデコード（説明出力）に使用。

関連データ構造

  • OffsetNumber: storage/off.h で定義されるページ内オフセット。
  • spgxlogState: SP-GiST のリカバリ状態を管理する関連構造体。

失敗モードと境界条件

  • **アライメント**: 可変長データがアライメントされていないため、直接構造体メンバとしてアクセスせず、memcpy 等を用いて安全にコピーする必要があります。
  • **データ長**: 構造体直後に続くタプルデータのサイズが正しく計算されていない場合、メモリ破壊や不正な読み取りが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER 的なパターンで利用されています。構造体サイズを計算する際は、固定部分と可変部分を明確に区別してください。
  • postfixBlkSame フラグは、物理的なページ配置の最適化に関わるため、リカバリロジック変更時にはこのフラグの解釈を慎重に確認してください。
