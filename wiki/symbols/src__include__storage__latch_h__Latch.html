Latch

定義

  • 種別: struct
  • 定義箇所: src/include/storage/latch.h:113

目的

プロセス間通信および同期のための軽量なプリミティブ。プロセスが特定のイベントを待機してスリープし、イベント発生時に効率的に起床するための状態を保持する。

シグネチャまたは形状

typedef struct Latch
{
	sig_atomic_t is_set;
	sig_atomic_t maybe_sleeping;
	bool		is_shared;
	int			owner_pid;
#ifdef WIN32
	HANDLE		event;
#endif
} Latch;

入力と出力

本構造体はデータ保持用であり、直接的な入力・出力関数を持たない。操作には InitLatch、SetLatch、ResetLatch、WaitLatch 等の関連関数を使用する。

前提条件と不変条件

  • is_set および maybe_sleeping は sig_atomic_t 型であり、シグナルハンドラからの安全なアクセスを保証する。
  • ResetLatch は、待機ループ内の「作業の確認」直前に呼び出す必要がある（レースコンディション回避のため）。

アルゴリズムまたは振る舞い

  • **状態管理**: プロセスの待機状態（maybe_sleeping）とラッチのセット状態（is_set）を管理する。
  • **プラットフォーム抽象化**: Unix系OSではシグナルベースの通知、Windowsでは HANDLE（イベントオブジェクト）を用いた通知を行う。
  • **通知**: SetLatch が呼び出されると、待機中のプロセスが起床する。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/postmaster/postmaster.c
  • src/backend/postmaster/startup.c
  • src/backend/storage/aio/method_worker.c
  • **呼び出し先**:
  • src/backend/storage/ipc/latch.c（InitLatch, InitSharedLatch, OwnLatch, DisownLatch, WaitLatch, WaitLatchOrSocket 等の実装）

関連データ構造

  • WaitEventSet: 複数のイベントを同時に待機するための上位インターフェース。
  • PGPROC.procLatch: プロセス固有のラッチ。

失敗モードと境界条件

  • ResetLatch の呼び出し順序を誤ると、確認とリセットの間に発生した通知を見逃す可能性がある。
  • シグナルハンドラ内では SetLatch 以外の複雑な処理を行うべきではない。

保守担当者向けメモ

  • 本構造体は不透明な型として扱うことが推奨される。
  • 補助プロセスへの通知には、独自ラッチの作成よりも PGPROC.procLatch の利用が推奨される。
