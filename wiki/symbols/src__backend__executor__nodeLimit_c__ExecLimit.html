ExecLimit

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeLimit.c:39

目的

SQL の LIMIT および OFFSET 句に基づき、下位ノードから取得したタプルのストリームをフィルタリングして返却する実行ノードのメイン処理関数です。

シグネチャまたは形状

TupleTableSlot *ExecLimit(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（LimitState 構造体を含むプラン実行状態）
  • **出力**: フィルタリングを通過した TupleTableSlot、またはストリーム終了・範囲外の場合は NULL。

前提条件と不変条件

  • pstate は LimitState 型にキャスト可能であること。
  • CHECK_FOR_INTERRUPTS() が呼び出され、割り込みがチェックされる。
  • LIMIT_INITIAL 状態では、パラメータ評価のために recompute_limits が実行される必要がある。

アルゴリズムまたは振る舞い

状態マシンとして動作し、node->lstate に基づいて以下の処理を行う： 1. **LIMIT_INITIAL**: recompute_limits を呼び出し、LIMIT_RESCAN へ遷移。 2. **LIMIT_RESCAN**: 順方向スキャンのみ許可。offset 分のタプルをスキップし、LIMIT_INWINDOW へ遷移。WITH TIES 指定時は境界タプルを last_slot に保持。 3. **LIMIT_INWINDOW**:

  • 順方向: count に達するまでタプルを返却。終了時に LIMIT_WINDOWEND または LIMIT_WINDOWEND_TIES へ遷移。
  • 逆方向: offset を下回らない範囲でタプルを返却。

    • 4. **LIMIT_EMPTY**: 常に NULL を返却。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecProcNode を介して実行エンジンから呼び出される（limitstate->ps.ExecProcNode = ExecLimit; による登録）。
  • **呼び出し先**:
  • recompute_limits（初期化時）
  • ExecProcNode(outerPlan)（下位ノードからのタプル取得）
  • ExecCopySlot（WITH TIES 時の境界タプル保存）

関連データ構造

  • LimitState: ノードの状態（lstate）、現在の位置（position）、offset/count 値、境界タプル（last_slot）を保持。
  • PlanState: 実行中のプランツリーの状態。
  • TupleTableSlot: タプルを保持するスロット。

失敗モードと境界条件

  • **逆方向スキャン**: LIMIT_INWINDOW 状態で position が offset + 1 以下の場合、LIMIT_WINDOWSTART へ遷移し NULL を返す。
  • **サブプラン終了**: ExecProcNode が NULL を返した場合、LIMIT_EMPTY または LIMIT_SUBPLANEOF へ遷移。
  • **エラー**: 逆方向スキャン時にサブプランからタプルが取得できない場合、elog(ERROR) を発生させる。

保守担当者向けメモ

  • WITH TIES 処理時は last_slot を使用するため、メモリ管理に注意が必要。
  • 並列実行時のリソース解放については、再スキャンの可能性を考慮した設計上の課題が残されている。
