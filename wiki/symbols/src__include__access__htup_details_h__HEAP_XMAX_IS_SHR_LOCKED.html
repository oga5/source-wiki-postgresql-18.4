HEAP_XMAX_IS_SHR_LOCKED

定義

  • 種別: function
  • 定義箇所: src/include/access/htup_details.h:266

目的

タプルの infomask を検査し、当該タプルが共有ロック（Shared Lock）状態にあるかどうかを判定します。

シグネチャまたは形状

static inline bool HEAP_XMAX_IS_SHR_LOCKED(int16 infomask)

入力と出力

  • **入力**: infomask (タプルの t_infomask フィールド)
  • **出力**: bool 型。共有ロックが設定されている場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力される infomask は、ヒープタプルのヘッダーから取得された有効なビットマスクである必要があります。

アルゴリズムまたは振る舞い

1. infomask に対して HEAP_LOCK_MASK を適用し、ロック関連のビットを抽出します。 2. 抽出した値が HEAP_XMAX_SHR_LOCK と一致するかを比較し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgrowlocks/pgrowlocks.c:235
  • src/backend/access/heap/heapam.c:4719, 4724, 5482, 5888
  • **呼び出し先**:
  • 本関数はインライン関数であり、特定の関数を呼び出すことはありません。

関連データ構造

  • HeapTupleHeaderData: タプルヘッダー構造体。t_infomask フィールドを保持します。

失敗モードと境界条件

  • infomask が正しく初期化されていない場合、誤った判定結果を返す可能性があります。

保守担当者向けメモ

  • 本関数は HEAP_LOCK_MASK および HEAP_XMAX_SHR_LOCK の定義に依存しています。これらの定数が変更された場合、本関数のロジックも影響を受ける可能性があります。
