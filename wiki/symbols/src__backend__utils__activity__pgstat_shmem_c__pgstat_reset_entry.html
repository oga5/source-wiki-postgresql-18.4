pgstat_reset_entry

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_shmem.c:1100

目的

指定された統計エントリの統計情報をリセットし、初期状態に戻します。

シグネチャまたは形状

void pgstat_reset_entry(PgStat_Kind kind, Oid dboid, uint64 objid, TimestampTz ts)

入力と出力

  • kind (PgStat_Kind): 統計情報の種類。
  • dboid (Oid): データベースOID。
  • objid (uint64): オブジェクトID。
  • ts (TimestampTz): リセット時刻。
  • 戻り値: なし。

前提条件と不変条件

  • pgstat_get_kind_info(kind)->fixed_amount が偽であること（固定サイズの統計エントリではないこと）をアサートします。

アルゴリズムまたは振る舞い

1. pgstat_get_entry_ref を呼び出し、対象の統計エントリへの参照を取得します。 2. エントリが存在しない、または dropped フラグが立っている場合は何もせず終了します。 3. pgstat_lock_entry を使用して対象エントリを排他ロックします。 4. shared_stat_reset_contents を呼び出し、共有メモリ上の統計情報をリセットします。 5. pgstat_unlock_entry を使用してロックを解放します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/activity/pgstat.c:862
  • src/backend/utils/activity/pgstat_subscription.c:70
  • 呼び出し先:
  • pgstat_get_entry_ref
  • pgstat_lock_entry
  • shared_stat_reset_contents
  • pgstat_unlock_entry

関連データ構造

  • PgStat_EntryRef: 統計エントリへの参照を保持する構造体。
  • PgStat_Kind: 統計情報の種類を識別する列挙型。

失敗モードと境界条件

  • エントリが既に削除済み（dropped）である場合、リセット処理はスキップされます。
  • pgstat_get_entry_ref がエントリを取得できない場合、処理は安全に終了します。

保守担当者向けメモ

  • 本関数は共有メモリ上の統計情報を直接操作するため、必ず pgstat_lock_entry による保護下で実行されます。
  • 統計情報の削除とリセットが競合する場合、dropped フラグのチェックにより整合性が保たれます。
