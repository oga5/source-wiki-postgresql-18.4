close_tsvector_parser

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector_parser.c:89

目的

TSVectorParseState オブジェクトに関連付けられたメモリリソースを解放し、パーサーの状態を破棄します。

シグネチャまたは形状

void close_tsvector_parser(TSVectorParseState state)

入力と出力

  • **入力**: state (TSVectorParseState) - 解放対象のパーサー状態構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • state は init_tsvector_parser によって正常に初期化された有効なポインタである必要があります。
  • 本関数呼び出し後、state が指すメモリ領域および内部の word バッファへのアクセスは無効となります。

アルゴリズムまたは振る舞い

1. state->word が保持する単語バッファを pfree で解放します。 2. state 構造体自体を pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery.c:870
  • src/backend/utils/adt/tsvector.c:260
  • **呼び出し先**:
  • pfree (PostgreSQL メモリ管理関数)

関連データ構造

  • TSVectorParseState: 解析中の状態を保持する構造体。本関数はこの構造体のライフサイクルを終了させます。

失敗モードと境界条件

  • state が NULL の場合、pfree の仕様に依存してエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は init_tsvector_parser と対になるデストラクタです。パーサーの状態構造体に変更を加えた場合は、本関数内の解放順序や対象が適切か確認してください。
