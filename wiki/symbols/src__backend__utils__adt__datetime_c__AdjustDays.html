AdjustDays

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/datetime.c:642

目的

指定された値とスケールを乗算し、その結果を pg_itm_in 構造体の tm_mday に安全に加算する。

シグネチャまたは形状

static bool AdjustDays(int64 val, int scale, struct pg_itm_in *itm_in)

入力と出力

  • **入力**:
  • val: 加算する基準値（int64）。
  • scale: val に乗算する倍率（int）。
  • itm_in: 日付情報を保持する pg_itm_in 構造体へのポインタ。
  • **出力**:
  • 成功時は true、オーバーフロー等のエラー発生時は false を返す。

前提条件と不変条件

  • val が INT_MIN から INT_MAX の範囲外である場合、処理は行われず false を返す。
  • itm_in は有効なメモリ領域を指している必要がある。

アルゴリズムまたは振る舞い

1. val が int32 の範囲に収まるかを確認する。 2. val * scale の計算において、int32 のオーバーフローが発生しないかを確認する。 3. itm_in->tm_mday + (val * scale) の計算において、int32 のオーバーフローが発生しないかを確認する。 4. 両方の計算が安全であれば、itm_in->tm_mday を更新し true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/datetime.c 内の各箇所（3727, 3734, 4006, 4011, 4021, 4070 行目付近）。
  • **呼び出し先**: pg_mul_s32_overflow および pg_add_s32_overflow（common/int.h で定義されたオーバーフローチェック関数）。

関連データ構造

  • struct pg_itm_in: 日付・時刻の構成要素を保持する構造体。

失敗モードと境界条件

  • val が int64 の範囲を超えている場合、false を返す。
  • 乗算または加算の結果が int32 の範囲を超える場合、false を返す。

保守担当者向けメモ

  • 本関数は int32 へのキャストを伴うため、入力値の範囲チェックが重要である。
  • 呼び出し元で val に 1 や 7 などの定数が渡されているケースが多く、日付計算の調整用として利用されている。
