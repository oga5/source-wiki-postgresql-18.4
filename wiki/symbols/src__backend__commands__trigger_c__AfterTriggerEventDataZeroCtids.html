AfterTriggerEventDataZeroCtids

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/trigger.c:3738

目的

AFTER トリガーイベント発生時に、CTID（行識別子）を保持しない形式のイベントデータを格納するための構造体です。

シグネチャまたは形状

typedef struct AfterTriggerEventDataZeroCtids
{
	TriggerFlags ate_flags;
} AfterTriggerEventDataZeroCtids;

入力と出力

  • 入力: TriggerFlags 型の ate_flags（ステータスビットおよび共有データへのオフセットを含む）。
  • 出力: なし（構造体定義）。

前提条件と不変条件

  • ate_flags は、トリガー実行エンジンがイベントを処理する際に必要な状態情報および共有データへの参照を保持している必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、CTID を必要としない特定のトリガーイベントにおいて、イベントデータを最小限のメモリフットプリントで保持するために使用されます。
  • src/backend/commands/trigger.c 内で、イベントデータのサイズ計算（sizeof(AfterTriggerEventDataZeroCtids)）の対象として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数関係は、提供されたソース情報からは特定できません。

関連データ構造

  • TriggerFlags: イベントのステータスや共有データへのオフセットを定義する型。
  • AfterTriggerEvent: 本構造体を含む上位のイベント管理構造体。

失敗モードと境界条件

  • 提供されたソース情報には、本構造体固有の失敗モードや境界条件に関する記述はありません。

保守担当者向けメモ

  • 本構造体は、AfterTriggerEventDataNoOids や AfterTriggerEventDataOneCtid と同様のイベントデータバリエーションの一つです。メモリ効率化のためにデータ構造が細分化されているため、変更時は他のバリエーションとの整合性に注意してください。
