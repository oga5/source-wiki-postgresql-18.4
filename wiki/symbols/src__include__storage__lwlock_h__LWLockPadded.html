LWLockPadded

定義

  • 種別: typedef (共用体)
  • 定義箇所: src/include/storage/lwlock.h:68

目的

LWLock 構造体を LWLOCK_PADDED_SIZE にパディングし、キャッシュライン境界に配置することで、マルチプロセッサ環境における「偽の共有（False Sharing）」を防止し、ロック競合時のパフォーマンスを最適化する。

シグネチャまたは形状

typedef union LWLockPadded
{
    LWLock      lock;
    char        pad[LWLOCK_PADDED_SIZE];
} LWLockPadded;

入力と出力

  • **入力**: なし（データ構造定義）。
  • **出力**: なし。

前提条件と不変条件

  • LWLOCK_PADDED_SIZE は、ターゲットアーキテクチャのキャッシュラインサイズ以上である必要がある。
  • 共用体として定義されているため、lock メンバと pad メンバはメモリ領域を共有する。

アルゴリズムまたは振る舞い

  • 本構造体は、LWLock を単体で扱うのではなく、配列として確保する際に使用される。
  • LWLock のサイズがキャッシュラインサイズより小さい場合でも、pad によって強制的にサイズを拡張することで、配列内の各ロックが異なるキャッシュラインに配置されることを保証する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/slru.c: buffer_locks および bank_locks のメモリ確保およびポインタキャストに使用。
  • src/backend/postmaster/launch_backend.c: MainLWLockArray の宣言に使用。
  • src/backend/storage/lmgr/lwlock.c: MainLWLockArray の初期化およびメモリ確保に使用。
  • **呼び出し先**:
  • LWLock 構造体（メンバとして保持）。

関連データ構造

  • LWLock: 本構造体がラップする軽量ロック本体。
  • MainLWLockArray: システム全体で共有される LWLockPadded 型の配列。

失敗モードと境界条件

  • **メモリ不足**: slru.c や lwlock.c でのメモリ確保時に、LWLockPadded のサイズ（sizeof(LWLockPadded)）を考慮した計算が必要。計算を誤るとバッファオーバーフローやメモリ破壊を引き起こす可能性がある。
  • **アライメント**: コンパイラやアーキテクチャの制約により、期待されるキャッシュライン境界に配置されない場合、パフォーマンス低下を招く可能性がある。

保守担当者向けメモ

  • LWLock 構造体のメンバを変更した際、LWLOCK_PADDED_SIZE を超えないか確認すること。
  • LWLOCK_PADDED_SIZE の値は、プラットフォームごとのキャッシュラインサイズに合わせて適切に定義されている必要がある。
