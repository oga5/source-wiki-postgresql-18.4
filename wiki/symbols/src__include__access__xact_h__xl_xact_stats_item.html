xl_xact_stats_item

定義

  • 種別: struct
  • 定義箇所: src/include/access/xact.h:282

目的

トランザクションに関連する統計情報の項目をWAL（Write-Ahead Logging）に記録するためのデータ構造です。

シグネチャまたは形状

typedef struct xl_xact_stats_item
{
    int         kind;
    Oid         dboid;
    uint32      objid_lo;
    uint32      objid_hi;
} xl_xact_stats_item;

入力と出力

  • **入力**: トランザクション処理中に発生した統計情報の種別、データベースOID、およびオブジェクトID（objid）。
  • **出力**: WALレコードの一部としてシリアライズされたバイナリデータ。

前提条件と不変条件

  • xl_xact_xinfo 構造体の一部として扱われるため、すべてのフィールドは sizeof(int) の倍数である必要があります。
  • objid は PgStat_HashKey.objid の値を保持しており、構造体の整合性を保つために uint32 2つ（objid_lo, objid_hi）に分割して格納されます。

アルゴリズムまたは振る舞い

トランザクションのコミットまたはアボート時に、関連する統計情報をWALに書き込む際、この構造体の配列が使用されます。解析時には、WALレコードのバッファポインタを xl_xact_stats_item 型にキャストし、MAXALIGN を用いて適切にオフセットを進めることで、可変長の統計情報リストを順次読み取ります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/rmgrdesc/xactdesc.c（WALレコードの解析および記述）、src/backend/access/transam/twophase.c（2フェーズコミット時の統計情報処理）。
  • **呼び出し先**: 直接的な関数呼び出しはありません。データ構造として利用されます。

関連データ構造

  • xl_xact_stats_items: 本構造体のリストを管理するためのラッパー構造体。
  • PgStat_HashKey: 統計情報のキーとして参照される構造体。

失敗モードと境界条件

  • WALレコードの解析時に、ncommitstats や nabortstats の数と実際のバッファサイズが一致しない場合、メモリ破壊や不正なポインタ参照が発生する可能性があります。必ず MAXALIGN を使用したポインタ演算が必要です。

保守担当者向けメモ

  • 本構造体はWALレコードのバイナリレイアウトに直接依存しています。フィールドの追加や変更を行う際は、xactdesc.c 内の解析ロジックおよび twophase.c でのシリアライズ処理との整合性を確認してください。
