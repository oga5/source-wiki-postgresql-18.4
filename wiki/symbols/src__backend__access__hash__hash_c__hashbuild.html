hashbuild

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash.c:121

目的

ハッシュインデックスの初期構築を行うメインルーチンです。テーブルをスキャンし、インデックス構造を初期化してタプルを挿入します。

シグネチャまたは形状

IndexBuildResult *hashbuild(Relation heap, Relation index, IndexInfo *indexInfo)

入力と出力

  • **入力**:
  • heap: 対象のテーブルリレーション。
  • index: 新規作成するハッシュインデックスリレーション。
  • indexInfo: インデックスの構成情報。
  • **出力**:
  • IndexBuildResult*: 構築されたインデックスの統計情報（処理されたヒープタプル数およびインデックスタプル数）を保持する構造体。

前提条件と不変条件

  • インデックスリレーションは空である必要があります（空でない場合はエラーが発生します）。
  • 本関数は、インデックスリレーションに対して一度だけ呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. **初期化**: _hash_init を呼び出し、メタページと初期バケットを作成します。 2. **ソート判定**: インデックスサイズが maintenance_work_mem または利用可能なバッファ数を超える場合、メモリ不足によるスラッシングを防ぐためにソート処理（_h_spoolinit）を有効化します。 3. **ヒープスキャン**: table_index_build_scan を実行し、hashbuildCallback を介してタプルを処理します。 4. **挿入**: ソートが有効な場合は _h_indexbuild を呼び出してソート済みタプルを挿入し、その後スプールを破棄します。 5. **統計更新**: 処理したタプル数を pgstat に報告し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のインデックス AM インターフェース（ambuild コールバック経由）。
  • **呼び出し先**:
  • _hash_init: インデックスの初期化。
  • table_index_build_scan: ヒープのスキャン。
  • _h_spoolinit / _h_indexbuild / _h_spooldestroy: ソートが必要な場合のインデックス構築処理。
  • hashbuildCallback: スキャン中の各タプルに対するコールバック。

関連データ構造

  • HashBuildState: 構築中の状態（スプール状況、タプル数、ヒープリレーション）を保持。
  • IndexBuildResult: 構築結果の統計情報。

失敗モードと境界条件

  • インデックスが既にデータを含んでいる場合、elog(ERROR) により処理が中断されます。
  • メモリ制限（maintenance_work_mem）やバッファ数に応じて、構築戦略（直接挿入かソート挿入か）が自動的に切り替わります。

保守担当者向けメモ

  • 本関数は hashhandler を通じて登録されるインデックス AM の主要なエントリポイントです。
  • ソート処理の閾値判定ロジックは、バケットサイズやページ構成の変更時に調整が必要となる可能性があります。
