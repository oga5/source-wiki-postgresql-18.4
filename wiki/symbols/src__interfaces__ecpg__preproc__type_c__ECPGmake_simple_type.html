ECPGmake_simple_type

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/type.c:71

目的

ECPGプリプロセッサにおいて、C言語の基本データ型を表す ECPGtype 構造体を動的に生成・初期化する。

シグネチャまたは形状

struct ECPGtype *ECPGmake_simple_type(enum ECPGttype type, const char *size, int counter);

入力と出力

  • **入力**:
  • type: ECPGttype 列挙型によるデータ型指定。
  • size: 型のサイズを表す文字列。
  • counter: varchar や bytea 型で使用されるカウンタ値。
  • **出力**:
  • 初期化済みの struct ECPGtype へのポインタ。

前提条件と不変条件

  • mm_alloc を使用してメモリを確保するため、呼び出し側は適切なタイミングで ECPGfree_type を用いてメモリを解放する必要がある。
  • size 引数は mm_strdup によって内部的に複製される。

アルゴリズムまたは振る舞い

1. mm_alloc を用いて ECPGtype 構造体分のメモリを確保する。 2. 引数 type および counter を構造体のフィールドに代入する。 3. size を mm_strdup で複製して格納する。 4. type_name、u.element、struct_sizeof を NULL で初期化する。 5. 構築した構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/preproc/type.c 内の型構築処理、および src/interfaces/ecpg/preproc/variable.c 内の変数定義処理から呼び出される。
  • **呼び出し先**: mm_alloc および mm_strdup（メモリ管理用ラッパー）。

関連データ構造

  • ECPGtype: 型情報を保持する主要な構造体。
  • ECPGttype: 型の種類を定義する列挙型。

失敗モードと境界条件

  • メモリ不足時には mm_alloc 経由でシステムが異常終了する可能性がある。
  • size が NULL の場合、mm_strdup の挙動に依存する（通常はセグメンテーション違反の可能性がある）。

保守担当者向けメモ

  • 本関数は ECPGtype の基本型生成に特化している。構造体や配列など複雑な型については、ECPGmake_struct_type や ECPGmake_array_type が別途用意されている。
  • counter フィールドは特定の型（varchar, bytea）でのみ意味を持つため、他の型で使用する際は注意が必要。
