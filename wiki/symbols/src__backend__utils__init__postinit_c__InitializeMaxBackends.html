InitializeMaxBackends

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/postinit.c:554

目的

PostgreSQLサーバー全体で許可されるバックエンドプロセスの最大数（MaxBackends）を計算し、設定値が許容範囲内であることを検証する。

シグネチャまたは形状

void InitializeMaxBackends(void)

入力と出力

  • **入力**: グローバル変数 MaxConnections, autovacuum_worker_slots, max_worker_processes, max_wal_senders, NUM_SPECIAL_WORKER_PROCS を参照。
  • **出力**: なし（副作用としてグローバル変数 MaxBackends を設定）。

前提条件と不変条件

  • MaxBackends が初期化前（0）であることを前提とする（Assert(MaxBackends == 0)）。
  • 補助プロセス（auxiliary processes）は計算に含まれない。

アルゴリズムまたは振る舞い

1. 以下の各設定値の合計を計算し、MaxBackends に代入する。

  • MaxConnections
  • autovacuum_worker_slots
  • max_worker_processes
  • max_wal_senders
  • NUM_SPECIAL_WORKER_PROCS

    • 2. 合計値が MAX_BACKENDS 定数を超えている場合、ERROR レベルでログを出力し、プロセスを停止させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/bootstrap/bootstrap.c
  • src/backend/postmaster/postmaster.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • ereport（エラー発生時）

関連データ構造

  • MAX_BACKENDS（上限定数）
  • NUM_SPECIAL_WORKER_PROCS（特殊ワーカプロセス数）

失敗モードと境界条件

  • 合計値が MAX_BACKENDS を超過した場合、ERRCODE_INVALID_PARAMETER_VALUE を伴うエラーが発生する。

保守担当者向けメモ

  • 本関数はバックエンド起動時の初期段階で一度だけ実行される必要がある。
  • MAX_BACKENDS の制限は src/include/storage/procnumber.h で定義されており、システム全体のプロセス管理能力に直結するため、変更時は注意が必要。
