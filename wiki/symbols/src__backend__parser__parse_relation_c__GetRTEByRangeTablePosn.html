GetRTEByRangeTablePosn

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_relation.c:544

目的

指定されたクエリレベルおよび範囲テーブル位置（varno）に対応する RangeTblEntry (RTE) を取得する。

シグネチャまたは形状

RangeTblEntry *GetRTEByRangeTablePosn(ParseState *pstate, int varno, int sublevels_up)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態（ParseState）。
  • varno: 取得対象の範囲テーブルインデックス（1始まり）。
  • sublevels_up: 探索対象のクエリレベル（0は現在のクエリ、1以上は親クエリ）。
  • **出力**:
  • 指定された位置に対応する RangeTblEntry 構造体へのポインタ。

前提条件と不変条件

  • sublevels_up が 0 より大きい場合、pstate->parentParseState が存在しなければならない（Assert により検証）。
  • varno は 1 以上かつ、対象 pstate の範囲テーブル長以下でなければならない（Assert により検証）。

アルゴリズムまたは振る舞い

1. sublevels_up の回数分、pstate を parentParseState に遡る。 2. 最終的な pstate の範囲テーブル（p_rtable）から、rt_fetch を用いて指定された varno の RangeTblEntry を取得して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/analyze.c
  • src/backend/parser/parse_expr.c
  • src/backend/parser/parse_target.c
  • **呼び出し先**:
  • rt_fetch（マクロまたはインライン関数と推測される）

関連データ構造

  • RangeTblEntry: 範囲テーブルの各エントリを保持する構造体。
  • ParseState: クエリ解析中の状態を保持する構造体。

失敗モードと境界条件

  • varno が範囲外の場合、または sublevels_up が深すぎて親 ParseState が存在しない場合、Assert によりプログラムが停止する（デバッグビルド時）。

保守担当者向けメモ

  • 本関数はクエリのネスト構造を跨いだ参照解決の基盤となる。varno は SQL の Var ノードが保持するインデックスと直接対応している。
