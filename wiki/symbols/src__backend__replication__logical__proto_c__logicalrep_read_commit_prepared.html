logicalrep_read_commit_prepared

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/proto.c:266

目的

論理レプリケーションのプロトコルにおいて、準備済みトランザクションのコミット（COMMIT PREPARED）メッセージをデシリアライズし、構造体へ格納する。

シグネチャまたは形状

void logicalrep_read_commit_prepared(StringInfo in, LogicalRepCommitPreparedTxnData *prepare_data)

入力と出力

  • **入力**:
  • in: 受信したメッセージデータを含む StringInfo バッファ。
  • prepare_data: 読み取ったデータを格納するための LogicalRepCommitPreparedTxnData 構造体へのポインタ。
  • **出力**: なし（prepare_data を介して値を書き戻す）。

前提条件と不変条件

  • in バッファには、プロトコル仕様に従った COMMIT PREPARED メッセージのバイナリデータが格納されている必要がある。
  • prepare_data->gid は、コピー先のバッファサイズが十分であることを前提としている。

アルゴリズムまたは振る舞い

1. メッセージのフラグを読み取り、0 以外であればエラーを発生させる。 2. commit_lsn を読み取り、InvalidXLogRecPtr の場合はエラーを発生させる。 3. end_lsn を読み取り、InvalidXLogRecPtr の場合はエラーを発生させる。 4. commit_time および xid を順次読み取る。 5. 最後に gid（グローバル・トランザクション識別子）を読み取り、prepare_data->gid にコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/worker.c:1178
  • **呼び出し先**: pq_getmsgbyte, pq_getmsgint64, pq_getmsgint, pq_getmsgstring, elog

関連データ構造

  • LogicalRepCommitPreparedTxnData: 準備済みトランザクションのコミット情報を保持する構造体。
  • StringInfo: PostgreSQL のネットワーク通信用バッファ構造体。

失敗モードと境界条件

  • **フラグ不正**: フラグが 0 以外の場合、elog(ERROR) により処理が中断される。
  • **LSN 不正**: commit_lsn または end_lsn が InvalidXLogRecPtr の場合、elog(ERROR) により処理が中断される。
  • **バッファ不足**: StringInfo の読み取り位置がメッセージの末尾を超えた場合、pq_getmsg* 関数群が内部的にエラーを発生させる。

保守担当者向けメモ

  • 本関数は logicalproto.h で定義されたプロトコル形式と密接に結合している。メッセージフォーマットを変更する場合は、対応する logicalrep_write_commit_prepared との整合性を必ず確認すること。
  • gid のコピーには strlcpy を使用しており、prepare_data->gid のサイズ制限に依存している点に注意。
