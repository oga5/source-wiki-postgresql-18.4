preprocess_targetlist

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/preptlist.c:63

目的

プランナがクエリを実行可能な形式にするため、ターゲットリスト（targetList）の補完、正規化、および実行に必要なJunk列（resjunk）の追加を行う。

シグネチャまたは形状

void preprocess_targetlist(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root（クエリ情報およびプランナの状態を保持する構造体）
  • **出力**: なし（root->processed_tlist 等の内部状態を直接更新する）

前提条件と不変条件

  • root->parse が有効な Query 構造体であること。
  • 対象リレーションが存在する場合、呼び出し元で既に AccessShareLock が取得されていること。
  • result_relation が指定されている場合、それはサブクエリではなく物理的なリレーションであること。

アルゴリズムまたは振る舞い

1. **リレーションの準備**: result_relation があれば table_open で開く。 2. **コマンド別の処理**:

  • CMD_INSERT: expand_insert_targetlist を呼び出し、テーブル定義順に列を整列し、欠落列をNULLで補完する。
  • CMD_UPDATE: extract_update_targetlist_colnos を呼び出し、更新対象列番号を抽出・正規化する。

    • 3. **Junk列の追加**:

  • UPDATE/DELETE/MERGE の場合、継承テーブルでなければ add_row_identity_columns を呼び出し、行識別用列を追加する。
  • MERGE の場合、各アクション（INSERT/UPDATE）のターゲットリストを個別に処理し、条件式（qual）や結合条件で使用される変数をターゲットリストに追加する。

    • 4. **行ロックの処理**: root->rowMarks を走査し、FOR UPDATE/SHARE 等に必要なJunk列を追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planner.c:1809
  • **呼び出し先**:
  • table_open
  • expand_insert_targetlist
  • extract_update_targetlist_colnos
  • add_row_identity_columns
  • pull_var_clause
  • makeTargetEntry
  • lappend 等のリスト操作関数

関連データ構造

  • PlannerInfo: プランナのメイン状態保持構造体。
  • Query: パーサから渡されるクエリツリー。
  • TargetEntry: ターゲットリストの各要素。
  • PlanRowMark: 行ロックやEvalPlanQual再チェックに必要な情報。
  • MergeAction: MERGE 文の各アクション定義。

失敗モードと境界条件

  • target_rte->rtekind が RTE_RELATION でない場合、elog(ERROR) を発生させる。
  • MERGE 文において、ターゲットリレーション以外の変数が参照されている場合、それらをターゲットリストに追加する。
  • 継承テーブル（target_rte->inh が真）の場合、一部のJunk列追加処理をスキップする。

保守担当者向けメモ

  • 本関数は物理アクセスに近い処理を統括する。resjunk = true とされた列は最終出力には含まれないが、プランナ内部での行特定に必須である。
  • MERGE 文のターゲットリスト構築は他のコマンドより複雑であり、条件式に含まれる変数も適切に抽出する必要がある。
  • rowMarks の処理において、将来的に行識別用列の管理と統合される可能性があるが、現時点では分離されている。
