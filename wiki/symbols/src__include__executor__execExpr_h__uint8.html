uint8

定義

  • 種別: typedef（符号なし8ビット整数型）
  • 定義箇所: src/include/executor/execExpr.h:371（※注：ソースコード上では uint8 型として宣言されたメンバ変数として出現）

目的

メモリ効率を重視したフラグ管理や、特定のビット状態を保持するためのデータ型です。

シグネチャまたは形状

uint8 nullflag;

入力と出力

  • 入力: 8ビット幅の整数値。
  • 出力: 0から255までの範囲の整数値。

前提条件と不変条件

  • execExpr.h 内の returningexpr 構造体においては、OLD/NEW 行が NULL かどうかを判定するためのフラグとして使用されます。

アルゴリズムまたは振る舞い

  • 実行エンジンにおいて、特定の条件（行の NULL 判定など）を保持し、後続のジャンプ処理（jumpdone）のトリガーとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元/呼び出し先の直接的な関係は、本ソース抜粋からは特定できません。
  • ツールによるヒューリスティックな検索結果として、contrib/amcheck や contrib/ltree 等の外部モジュールで同様の型名が頻繁に使用されていますが、これらが本定義と直接的に依存関係にあるかは不明です。

関連データ構造

  • returningexpr 構造体: nullflag を保持する親構造体。
  • ExprEvalStep: 式評価の各ステップを定義する構造体。

失敗モードと境界条件

  • 8ビット（0-255）の範囲を超える値を代入した場合、オーバーフローが発生します。

保守担当者向けメモ

  • 本シンボルは PostgreSQL 全体で標準的に使用される uint8 型を指します。
  • execExpr.h 内で使用される際は、構造体のパディングやメモリレイアウトに影響を与える可能性があるため、メンバの追加・変更時にはサイズに注意してください。
