verify_heap_slot_handler

定義

  • 種別: function
  • 定義箇所: src/bin/pg_amcheck/pg_amcheck.c:1038

目的

並列実行されたヒープ検証クエリの結果（PGresult）を処理し、破損情報の出力および後続処理の可否を判定するコールバック関数。

シグネチャまたは形状

static bool verify_heap_slot_handler(PGresult *res, PGconn *conn, void *context)

入力と出力

  • **入力**:
  • res: 実行された検証クエリの PGresult。
  • conn: データベース接続。
  • context: RelationInfo 構造体へのポインタ。
  • **出力**:
  • bool: should_processing_continue の戻り値に基づき、処理を継続すべきか否かを返す。

前提条件と不変条件

  • context は RelationInfo 型としてキャスト可能であること。
  • res が PGRES_TUPLES_OK の場合、クエリ結果には破損情報が含まれる可能性がある。
  • 処理終了時に RelationInfo 内の動的メモリ（sql, nspname, relname）が解放される。

アルゴリズムまたは振る舞い

1. PGresult のステータスを確認する。 2. PGRES_TUPLES_OK の場合:

  • タプルが存在すれば all_checks_pass を false に設定する。
  • 各タプルからブロック番号、オフセット、属性番号を抽出し、破損メッセージを標準出力にフォーマット表示する。

    • 3. それ以外の場合:

  • PQerrorMessage を取得し、インデントを付与してエラーメッセージを表示する。all_checks_pass を false に設定する。
  • opts.verbose が有効な場合、実行したクエリ文字列を表示する。

    • 4. RelationInfo のメンバ変数を解放する。 5. should_processing_continue(res) を呼び出し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_amcheck/pg_amcheck.c 内の ParallelSlotSetHandler に引数として渡される。
  • **呼び出し先**: PQntuples, PQgetisnull, PQgetvalue, PQresultStatus, PQerrorMessage, indent_lines, should_processing_continue。

関連データ構造

  • RelationInfo: チェック対象リレーションのメタデータおよびクエリ情報を保持。
  • PGresult, PGconn: libpq のデータ構造。

失敗モードと境界条件

  • PQresultStatus が PGRES_TUPLES_OK でない場合、エラーメッセージの解析と表示を行う。
  • メッセージ文字列が NULL の場合、「NO MESSAGE」として処理する。
  • 破損箇所（ブロック、オフセット、属性）の有無に応じて、出力メッセージの粒度を動的に変更する。

保守担当者向けメモ

  • 本関数は ParallelSlotSetHandler を介して非同期に呼び出されるため、グローバルな状態（all_checks_pass）の更新には注意が必要。
  • RelationInfo のメンバ変数を解放しているため、本関数が呼び出された後の RelationInfo オブジェクトの再利用は不可。
