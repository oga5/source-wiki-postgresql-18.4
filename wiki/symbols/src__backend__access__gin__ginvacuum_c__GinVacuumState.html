GinVacuumState

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gin/ginvacuum.c:27 および access/gin_private.h:243

目的

GINインデックスのバキューム処理（VACUUMおよびVACUUM ANALYZE）実行中に、処理対象のインデックス情報、コールバック関数、メモリコンテキスト、およびバキュームの進捗状態を保持するために使用されます。

シグネチャまたは形状

struct GinVacuumState
{
    Relation    index;
    IndexBulkDeleteResult *result;
    IndexBulkDeleteCallback callback;
    void       *callback_state;
    GinState    ginstate;
    BufferAccessStrategy strategy;
    MemoryContext tmpCxt;
};

入力と出力

本構造体は関数引数として渡され、バキューム処理の各フェーズで参照・更新されます。

  • **入力**: バキューム開始時に index（対象リレーション）、callback（TID削除判定用関数）、callback_state 等が初期化されます。
  • **出力**: 処理結果は result フィールドに格納され、バキューム終了時に呼び出し元へ返されます。

前提条件と不変条件

  • index は有効なGINインデックスのリレーションである必要があります。
  • tmpCxt はバキューム処理中に生成される一時的なメモリ割り当てを管理するために使用され、処理終了時に適切にリセットまたは解放される必要があります。

アルゴリズムまたは振る舞い

バキューム処理の各ステップにおいて、以下の情報を集約・提供します。

  • **TID削除**: callback を使用して、特定のTIDを削除すべきかを判定します。
  • **メモリ管理**: tmpCxt を通じて、バキューム中のメモリ消費を制御します。
  • **状態保持**: ginstate を介して、GINインデックス固有の操作に必要な情報を維持します。

呼び出し元と呼び出し先

  • **呼び出し元**: ginbulkdelete および ginvacuumcleanup が本構造体をインスタンス化し、バキューム処理の起点となります。
  • **呼び出し先**: 以下の関数が GinVacuumState *gvs を引数として受け取り、処理を実行します。
  • ginVacuumItemPointers
  • ginDeletePage
  • ginScanToDelete
  • ginVacuumPostingTreeLeaves
  • ginVacuumPostingTree
  • ginVacuumEntryPage
  • ginVacuumPostingTreeLeaf (gindatapage.c)

関連データ構造

  • Relation: バキューム対象のインデックス。
  • IndexBulkDeleteResult: バキュームの統計結果。
  • GinState: GINインデックスの操作状態。
  • BufferAccessStrategy: バッファアクセスの最適化戦略。
  • MemoryContext: メモリ管理コンテキスト。

失敗モードと境界条件

  • callback がNULLの場合、TIDの削除判定が正しく行われない可能性があります。
  • tmpCxt が適切に管理されない場合、バキューム処理中のメモリリークが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は ginvacuum.c と gin_private.h の両方で定義されています。変更時は両方の整合性に注意してください。
  • ginScanToDelete 等の複雑なロック処理を伴う関数に渡されるため、gvs を介した状態変更が並行処理に与える影響を考慮する必要があります。
