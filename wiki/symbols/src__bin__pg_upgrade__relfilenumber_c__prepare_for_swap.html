prepare_for_swap

定義

  • **種別**: function
  • **定義箇所**: src/bin/pg_upgrade/relfilenumber.c:236

目的

pg_upgrade の --swap モードにおいて、旧クラスタのデータベースディレクトリを新クラスタへ移動させるための準備を行う。具体的には、ディレクトリ構造の入れ替えに必要なパスの算出と、既存ディレクトリの退避および配置を行う。

シグネチャまたは形状

static bool prepare_for_swap(const char *old_tablespace, Oid db_oid,
                             char *old_catalog_dir, char *new_db_dir, char *moved_db_dir)

入力と出力

  • **入力**:
  • old_tablespace: 旧クラスタのテーブルスペースパス。
  • db_oid: 対象データベースの OID。
  • **出力**:
  • old_catalog_dir (引数経由): 退避された旧カタログファイルの格納先パス。
  • new_db_dir (引数経由): 新クラスタのデータベースディレクトリパス。
  • moved_db_dir (引数経由): 退避された新クラスタのデータベースディレクトリパス。
  • **戻り値**: 成功時に true、対象ディレクトリが存在しない場合に false。

前提条件と不変条件

  • old_cluster および new_cluster グローバル変数が初期化されていること。
  • old_db_dir がファイルシステム上に存在すること（存在しない場合は false を返す）。
  • pg_dir_create_mode が適切なパーミッションで設定されていること。

アルゴリズムまたは振る舞い

1. **パスの算出**: テーブルスペースの構成に基づき、旧クラスタおよび新クラスタのデータベースディレクトリパスを生成する。 2. **退避先ディレクトリの作成**: moved_for_upgrade ディレクトリおよびカタログ退避用ディレクトリを pg_mkdir_p で作成する。 3. **ディレクトリの入れ替え**:

  • 新クラスタの既存ディレクトリを moved_db_dir へ rename で退避する。
  • 旧クラスタのディレクトリを new_db_dir へ rename で移動する。

    • 4. 失敗時は pg_fatal を呼び出し、プロセスを終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/relfilenumber.c 内の do_swap 関数（行 473, 480, 487 付近）。
  • **呼び出し先**: stat, pg_mkdir_p, rename, pg_fatal。

関連データ構造

  • old_cluster, new_cluster (グローバル変数): クラスタの構成情報。
  • Oid: データベース識別子。

失敗モードと境界条件

  • stat で ENOENT 以外のエラーが発生した場合、pg_fatal で異常終了する。
  • pg_mkdir_p や rename が失敗した場合、pg_fatal で異常終了する。
  • old_tablespace が old_cluster.pgdata と一致しない場合、テーブルスペースのサフィックス処理に制限がある（インプレーステーブルスペースのサポートに関する既知の制約）。

保守担当者向けメモ

  • 本関数は swap モード専用のロジックであり、他の転送モードとは独立したパスを通る。
  • 退避ディレクトリは old_cluster 内に作成されるため、delete_old_cluster スクリプトによって後続処理される設計となっている。
  • インプレーステーブルスペースのサポートを追加する場合、new_tablespace のパス解決ロジックを修正する必要がある。
