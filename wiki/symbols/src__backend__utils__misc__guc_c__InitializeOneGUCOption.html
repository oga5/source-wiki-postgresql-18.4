InitializeOneGUCOption

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/misc/guc.c:1643

目的

個別のGUC（Grand Unified Configuration）パラメータをデフォルト値で初期化し、関連するチェックフックおよび代入フックを実行してシステム状態を整合させる。

シグネチャまたは形状

static void InitializeOneGUCOption(struct config_generic *gconf)

入力と出力

  • **入力**: gconf (初期化対象の config_generic 構造体へのポインタ)
  • **出力**: なし（副作用として gconf が指す構造体および対応する変数値を更新）

前提条件と不変条件

  • gconf は有効な config_generic 構造体を指していること。
  • PGC_INT および PGC_REAL 型の場合、boot_val が min から max の範囲内にあること（Assert による検証）。
  • 初期化に失敗した場合、elog(FATAL, ...) によりプロセスが終了する。

アルゴリズムまたは振る舞い

1. **メタデータのリセット**: status, source, scontext 等の管理用フィールドをデフォルト値（PGC_S_DEFAULT, PGC_INTERNAL 等）に初期化する。 2. **型別の処理**: vartype に応じて以下の処理を行う：

  • boot_val を取得し、型に応じた check_hook を呼び出す。
  • assign_hook が定義されている場合、それを実行する。
  • variable（実際の変数）および reset_val に boot_val を代入する。
  • extra 情報を reset_extra に保存する。

    • 3. **文字列型の特殊処理**: PGC_STRING 型の場合、boot_val が存在すれば guc_strdup を使用してメモリを確保する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/guc.c 内の複数箇所（GUCテーブル構築時や個別のパラメータリセット時など）。
  • **呼び出し先**:
  • call_bool_check_hook, call_int_check_hook, call_real_check_hook, call_string_check_hook, call_enum_check_hook
  • assign_hook（各パラメータで定義されている場合）
  • guc_strdup

関連データ構造

  • config_generic: 全GUCパラメータの基底構造体。
  • config_bool, config_int, config_real, config_string, config_enum: 各型に対応する派生構造体。

失敗モードと境界条件

  • **初期化失敗**: check_hook が偽を返した場合、FATAL エラーが発生し、サーバーの起動または設定適用が停止する。
  • **メモリ不足**: guc_strdup 内部でメモリ確保に失敗した場合、FATAL エラーとなる。

保守担当者向けメモ

  • 本関数は、パラメータの「リセット」操作としても利用される。
  • check_hook や assign_hook の実行順序は、初期化時において一貫性が保たれるよう設計されている。
  • 新しいGUC型を追加する場合は、本関数の switch 文に新しいケースを追加する必要がある。
