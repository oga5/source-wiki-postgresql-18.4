int16KEY

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_int2.c:11

目的

PostgreSQL の GiST インデックスにおいて、int2（2バイト整数）型の値の範囲（下限値および上限値）を保持し、インデックスのキーとして管理するために使用されます。

シグネチャまたは形状

typedef struct int16key
{
	int16		lower;
	int16		upper;
} int16KEY;

入力と出力

  • 本構造体はデータ構造であり、関数ではありません。
  • GiST インデックスの操作において、int2 型の値をこの構造体に格納（圧縮）したり、インデックスから読み出して範囲判定に使用したりします。

前提条件と不変条件

  • lower は範囲の下限、upper は範囲の上限を表します。
  • 通常、lower <= upper であることが期待されます。

アルゴリズムまたは振る舞い

  • GiST インデックスの構築および検索時に、int2 型の値をこの構造体へ変換（gbt_int2_compress 等）し、インデックスツリー内のノードとして保持します。
  • 検索時には、この構造体の lower および upper を参照して、クエリ条件との一致判定（gbt_int2_consistent）が行われます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • contrib/btree_gist/btree_int2.c 内の各サポート関数（gbt_int2_consistent, gbt_int2_compress 等）において、インデックスキーの型キャストやメモリ確保に使用されています。
  • 呼び出し先:
  • 構造体であるため、直接の呼び出し先はありません。

関連データ構造

  • gbtree_ninfo: int2 型の比較ロジックを保持する静的構造体（tinfo）。
  • GISTENTRY: PostgreSQL の GiST インデックスエントリ構造体。

失敗モードと境界条件

  • int16 型の範囲外の値が入力された場合の挙動は、PostgreSQL の整数演算ルールおよび btree_utils_num の実装に依存します。
  • メモリ割り当て時に palloc が失敗した場合は、PostgreSQL の標準的なエラー処理が実行されます。

保守担当者向けメモ

  • 本構造体は btree_int4.c や btree_int8.c における int32KEY 等と同様のパターンで実装されています。
  • 比較ロジックを変更する場合は、本構造体そのものではなく、btree_utils_num.c で定義されているユーティリティ関数や、btree_int2.c 内の比較関数（gbt_int2key_cmp 等）を確認してください。
