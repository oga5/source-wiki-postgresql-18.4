gai_strerror

定義

  • 種別: function
  • 定義箇所: src/port/win32gai_strerror.c:21

目的

Windows環境において、getaddrinfo 等で発生する EAI_* エラーコードを人間が読み取り可能な文字列に変換します。標準のWindows APIである gai_strerrorA が持つ非スレッドセーフ性の問題を回避し、スレッドセーフなエラーメッセージ取得機能を提供します。

シグネチャまたは形状

const char *gai_strerror(int errcode);

入力と出力

  • **入力**: errcode (int型) - getaddrinfo 等から返されるネットワーク関連のエラーコード。
  • **出力**: const char * - エラーコードに対応する静的文字列リテラルへのポインタ。

前提条件と不変条件

  • 返される文字列は静的メモリ領域に配置されているため、呼び出し側でメモリ解放（free）を行う必要はありません。
  • 本関数はスレッドセーフです。

アルゴリズムまたは振る舞い

1. 引数 errcode を switch 文で評価します。 2. EAI_AGAIN, EAI_BADFLAGS, EAI_FAIL, EAI_FAMILY, EAI_MEMORY, EAI_NONAME, EAI_SERVICE, EAI_SOCKTYPE の各ケースに対し、対応するエラーメッセージ文字列を返します。 3. いずれのケースにも一致しない場合、デフォルトとして "Unknown server error" を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth.c
  • src/backend/libpq/hba.c
  • src/backend/libpq/pqcomm.c
  • src/backend/tcop/backend_startup.c
  • **呼び出し先**:
  • 本関数は標準ライブラリや他の関数を呼び出さず、静的文字列を返すのみです。

関連データ構造

  • <sys/socket.h>: EAI_* 定数の定義元。

失敗モードと境界条件

  • 定義されていないエラーコードが入力された場合、エラーとして "Unknown server error" を返します。
  • 新しい EAI_* エラーコードがシステムに追加された場合、本関数の switch 文を更新する必要があります。

保守担当者向けメモ

  • Windowsの gai_strerrorA が内部で静的バッファを共有し、マルチスレッド環境で競合するリスクを避けるための代替実装です。
  • 移植性を維持するため、プラットフォーム固有の差異を吸収する src/port ディレクトリの責務に従っています。
