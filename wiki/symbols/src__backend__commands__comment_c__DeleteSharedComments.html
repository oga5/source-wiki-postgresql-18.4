DeleteSharedComments

定義

  • 種別: function
  • 定義箇所: src/backend/commands/comment.c:373

目的

共有データベースオブジェクト（データベース、ロール、テーブルスペース等）に関連付けられたコメントをシステムカタログ pg_shdescription から削除します。

シグネチャまたは形状

void DeleteSharedComments(Oid oid, Oid classoid)

入力と出力

  • **入力**:
  • oid: 削除対象オブジェクトのOID。
  • classoid: 削除対象オブジェクトが属するカタログのOID。
  • **出力**: なし（副作用としてカタログからレコードを削除）。

前提条件と不変条件

  • 呼び出し元は、対象オブジェクトに対する適切なロック（RowExclusiveLock 等）を保持している必要があります。
  • 本関数は pg_shdescription に対して RowExclusiveLock を取得し、操作終了後に解放します。

アルゴリズムまたは振る舞い

1. pg_shdescription カタログに対して、指定された oid および classoid に一致するレコードを検索するためのスキャンキー（skey）を初期化します。 2. table_open を用いて pg_shdescription を RowExclusiveLock モードで開きます。 3. systable_beginscan を使用し、インデックス SharedDescriptionObjIndexId を介して該当する全レコードをスキャンします。 4. systable_getnext で取得した各タプルに対し、CatalogTupleDelete を実行して削除します。 5. スキャンを終了し、カタログテーブルを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/dbcommands.c (データベース削除時)
  • src/backend/commands/tablespace.c (テーブルスペース削除時)
  • src/backend/commands/user.c (ロール削除時)
  • **呼び出し先**:
  • table_open / table_close
  • systable_beginscan / systable_getnext / systable_endscan
  • CatalogTupleDelete

関連データ構造

  • pg_shdescription: 共有オブジェクトのコメントを保持するシステムカタログ。
  • ScanKeyData: インデックススキャン条件を定義する構造体。
  • SysScanDesc: システムカタログスキャンの状態を保持する記述子。

失敗モードと境界条件

  • 対象オブジェクトにコメントが存在しない場合、スキャンは何もヒットせず、エラーにはならず正常終了します。
  • カタログへのアクセス権限やロック取得に失敗した場合、PostgreSQLの標準的なエラーメカニズムによりトランザクションがアボートされる可能性があります。

保守担当者向けメモ

  • 本関数は共有オブジェクト専用です。通常オブジェクトのコメント削除には DeleteComments を使用してください。
  • pg_shdescription はクラスタ全体で共有されるため、操作時には適切なロック管理が不可欠です。
