ExecBitmapIndexScanInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapIndexscan.c:376

目的

並列クエリ実行時、BitmapIndexScan ノードの統計情報を保持するための動的共有メモリ（DSM）領域を割り当て、初期化する。

シグネチャまたは形状

void ExecBitmapIndexScanInitializeDSM(BitmapIndexScanState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: 初期化対象の BitmapIndexScanState 構造体へのポインタ。
  • pcxt: 並列クエリのコンテキスト情報を含む ParallelContext 構造体へのポインタ。
  • **出力**: なし（副作用として node->biss_SharedInfo が設定され、共有メモリが初期化される）。

前提条件と不変条件

  • node->ss.ps.instrument が有効であり、かつ pcxt->nworkers > 0 である場合にのみ実行される。
  • 割り当てられた共有メモリ領域は、ゼロクリアされることが保証される。

アルゴリズムまたは振る舞い

1. インストゥルメンテーションが不要、またはワーカーが存在しない場合は即座にリターンする。 2. SharedIndexScanInstrumentation 構造体と、ワーカー数分の IndexScanInstrumentation 配列を格納するために必要なサイズを計算する。 3. shm_toc_allocate を使用して共有メモリから領域を確保する。 4. shm_toc_insert を使用して、プランノードIDをキーとして確保した領域を登録する。 5. 確保した領域を memset でゼロクリアする。 6. 共有構造体の num_workers フィールドにワーカー数を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:488 からの呼び出しが確認されている。
  • **呼び出し先**: shm_toc_allocate および shm_toc_insert（共有メモリ管理関数）を呼び出す。

関連データ構造

  • BitmapIndexScanState: スキャンノードの実行状態を保持する構造体。
  • SharedIndexScanInstrumentation: 並列ワーカー間で共有される統計情報構造体。
  • ParallelContext: 並列実行のコンテキスト。

失敗モードと境界条件

  • pcxt->nworkers が 0 の場合、処理はスキップされる。
  • 共有メモリの割り当てに失敗した場合、shm_toc_allocate がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は ExecProcNode インターフェースの一部ではないため、直接呼び出されることはない。
  • 並列クエリにおける統計情報の集約において重要な役割を果たす。
