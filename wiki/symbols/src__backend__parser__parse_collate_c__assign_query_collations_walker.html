assign_query_collations_walker

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_collate.c:125

目的

クエリの構文木を走査し、各ノードに対して照合順序（Collation）の割り当ておよび解決を行うためのウォーカー関数です。

シグネチャまたは形状

static bool assign_query_collations_walker(Node *node, ParseState *pstate);

入力と出力

  • **入力**:
  • node: 走査対象の構文木ノード。
  • pstate: 解析状態を保持する ParseState 構造体。
  • **出力**:
  • bool: 常に false を返し、走査を継続します。

前提条件と不変条件

  • node が NULL の場合は何もしません。
  • SetOperationStmt ノードは既に処理済みとみなされ、再帰的に走査されません。

アルゴリズムまたは振る舞い

1. ノードが NULL であれば直ちに終了します。 2. ノードが SetOperationStmt 型であれば、処理をスキップして終了します。 3. ノードが List 型の場合、assign_list_collations を呼び出してリスト内の各要素を処理します。 4. それ以外のノードの場合、assign_expr_collations を呼び出して個別の式に対する照合順序の割り当てを行います。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の参照情報に基づくと、assign_query_collations 等から呼び出されることが示唆されていますが、明示的な呼び出し関係の全容は提供されたソース excerpt には含まれていません。
  • **呼び出し先**:
  • assign_list_collations
  • assign_expr_collations

関連データ構造

  • Node: 構文木の基本単位。
  • ParseState: 解析状態を管理する構造体。
  • SetOperationStmt: 集合演算ノード。

失敗モードと境界条件

  • node が NULL の場合、安全に無視されます。
  • 照合順序の競合が発生した場合、呼び出し先の assign_expr_collations 等の内部でエラーが報告される可能性があります。

保守担当者向けメモ

  • 本関数は query_tree_walker 等の走査メカニズムから呼び出されることを想定しています。
  • SetOperationStmt を除外している理由は、transformSetOperationStmt において既に照合順序の解決が完了しているためです。
