JsonTokenType

定義

  • 種別: enum
  • 定義箇所: src/include/common/jsonapi.h:17

目的

JSON 解析プロセスにおいて、入力ストリームから切り出された各トークン（要素）の種類を識別するために使用されます。

シグネチャまたは形状

typedef enum JsonTokenType
{
	JSON_TOKEN_INVALID,
	JSON_TOKEN_STRING,
	JSON_TOKEN_NUMBER,
	JSON_TOKEN_OBJECT_START,
	JSON_TOKEN_OBJECT_END,
	JSON_TOKEN_ARRAY_START,
	JSON_TOKEN_ARRAY_END,
	JSON_TOKEN_COMMA,
	JSON_TOKEN_COLON,
	JSON_TOKEN_TRUE,
	JSON_TOKEN_FALSE,
	JSON_TOKEN_NULL,
	JSON_TOKEN_END,
} JsonTokenType;

入力と出力

  • 入力: JSON パーサーが字句解析を行う過程で、現在の入力文字から特定されたトークン種別が本列挙型として返されます。
  • 出力: 解析コールバック関数（JsonSemAction 等）の引数として渡され、後続のセマンティック解析の判断基準となります。

前提条件と不変条件

  • JSON_TOKEN_INVALID は解析不能なトークンを示します。
  • JSON_TOKEN_END は入力ストリームの終端に到達したことを示します。
  • 各列挙値は、JSON 仕様（RFC 8259）で定義された構造要素およびリテラルと一対一で対応しています。

アルゴリズムまたは振る舞い

パーサーは入力データを走査し、文字の並びに基づいて本列挙型のいずれかを割り当てます。この型情報は、JsonSemAction で定義されたコールバック関数に渡され、オブジェクトの開始/終了やスカラ値の処理といったイベント駆動型の解析を制御します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/utils/adt/jsonb.c: jsonb_in_scalar 関数が引数として使用。
  • src/backend/utils/adt/jsonfuncs.c: okeys_scalar, get_scalar, alen_scalar, each_scalar, elements_scalar 等の関数が引数として使用。
  • 呼び出し先: 本シンボルは列挙型であるため、関数としての呼び出し先はありません。

関連データ構造

  • JsonLexContext: 解析状態を保持する構造体。現在のトークン種別を管理します。
  • JsonSemAction: 本型を使用して解析イベントを振り分けるコールバックテーブル。

失敗モードと境界条件

  • JSON_TOKEN_INVALID が返された場合、パーサーは不正な JSON 構文を検出したことを意味します。
  • 予期しないトークン種別がコールバックに渡された場合、セマンティック解析側でエラーハンドリングを行う必要があります。

保守担当者向けメモ

  • 新しい JSON 構文要素を追加する場合は、本列挙型に値を追加し、パーサーの実装（jsonapi.c）および各コールバック関数でのスイッチ文を更新してください。
