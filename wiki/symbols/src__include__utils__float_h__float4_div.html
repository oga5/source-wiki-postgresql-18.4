float4_div

定義

  • 種別: function
  • 定義箇所: src/include/utils/float.h:221

目的

float4 型の除算を実行し、PostgreSQLの要件に基づいたエラーチェック（ゼロ除算、オーバーフロー、アンダーフロー）を行う。

シグネチャまたは形状

static inline float4 float4_div(const float4 val1, const float4 val2)

入力と出力

  • **入力**: val1 (被除数), val2 (除数)
  • **出力**: val1 / val2 の計算結果。エラー発生時は例外をスローし、呼び出し元へは戻らない。

前提条件と不変条件

  • math.h の isinf() および isnan() が利用可能であること。
  • 演算結果が異常な場合、PostgreSQLの内部エラー報告関数を介して処理を中断する。

アルゴリズムまたは振る舞い

1. **ゼロ除算チェック**: val2 が 0.0f かつ val1 が NaN でない場合、float_zero_divide_error() を呼び出す。 2. **除算実行**: result = val1 / val2 を計算。 3. **オーバーフローチェック**: 結果が無限大かつ val1 が無限大でない場合、float_overflow_error() を呼び出す。 4. **アンダーフローチェック**: 結果が 0.0f かつ val1 が 0.0f でなく、かつ val2 が無限大でない場合、float_underflow_error() を呼び出す。 5. **戻り値**: 計算された result を返す。

関連データ構造

  • float4 (PostgreSQLの単精度浮動小数点型)

失敗モードと境界条件

  • val2 が 0.0f の場合: ゼロ除算エラー。
  • 演算結果が浮動小数点の表現範囲を超えた場合: オーバーフローエラー。
  • 演算結果が非ゼロからゼロへアンダーフローした場合: アンダーフローエラー。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、パフォーマンスを重視している。
  • エラー報告関数（float_zero_divide_error 等）は pg_noinline 属性を持つため、エラー発生時のコードパスはインライン展開されない設計となっている。
