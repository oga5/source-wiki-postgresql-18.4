astreamer_tar_parser_finalize

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_tar.c:353

目的

tarストリーム解析処理の終了処理を行い、アーカイブの整合性を確認した上で、後続のストリーマーへトレーラー情報を転送し、終了処理を委譲する。

シグネチャまたは形状

static void astreamer_tar_parser_finalize(astreamer *streamer)

入力と出力

  • **入力**: astreamer *streamer（astreamer_tar_parser 構造体としてキャストして使用）
  • **出力**: なし（void）

前提条件と不変条件

  • streamer は astreamer_tar_parser 型のインスタンスであること。
  • ストリームの終了時、解析状態（next_context）が ASTREAMER_ARCHIVE_TRAILER であるか、あるいはメンバーヘッダーの待機状態かつバッファが空である必要がある。

アルゴリズムまたは振る舞い

1. **整合性チェック**: 現在の解析状態が ASTREAMER_ARCHIVE_TRAILER 以外であり、かつ ASTREAMER_MEMBER_HEADER 状態であってもバッファにデータが残っている場合、pg_fatal を呼び出しエラー終了する。 2. **トレーラー転送**: astreamer_content を呼び出し、現在のバッファ内容を ASTREAMER_ARCHIVE_TRAILER として後続のストリーマー（streamer->bbs_next）へ渡す。 3. **後続の終了**: astreamer_finalize を呼び出し、後続のストリーマーの終了処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer_tar_parser 構造体の関数ポインタテーブル（.finalize）を介して呼び出される。
  • **呼び出し先**:
  • astreamer_content（後続ストリーマーへのデータ転送）
  • astreamer_finalize（後続ストリーマーの終了処理）
  • pg_fatal（エラー発生時）

関連データ構造

  • astreamer_tar_parser: パーサーの状態を保持する構造体。
  • astreamer: ストリーマーの基底構造体。

失敗モードと境界条件

保守担当者向けメモ

  • 本関数は astreamer インターフェースの finalize コールバックとして登録されている。
  • アーカイブのトレーラーが欠落している場合でも、本関数を通じて強制的に終了処理が実行される設計となっている。
