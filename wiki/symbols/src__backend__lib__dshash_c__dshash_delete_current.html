dshash_delete_current

定義

  • 種別: function
  • 定義箇所: src/backend/lib/dshash.c:756

目的

ハッシュテーブルのシーケンシャルスキャン中に、現在走査しているエントリを削除します。

シグネチャまたは形状

void dshash_delete_current(dshash_seq_status *status)

入力と出力

  • **入力**: status (dshash_seq_status *) - 現在のスキャン状態を保持する構造体。
  • **出力**: なし。

前提条件と不変条件

  • status->exclusive が真であること（排他モードでのスキャンが必要）。
  • ハッシュテーブルの制御ブロックの magic 値が DSHASH_MAGIC と一致すること。
  • 対象エントリが属するパーティションの LWLock を、呼び出し元が排他モード（LW_EXCLUSIVE）で保持していること。

アルゴリズムまたは振る舞い

1. status から現在のハッシュテーブル (hash_table) と現在指しているエントリ (curitem) を取得します。 2. エントリのハッシュ値から、対応するパーティションを特定します。 3. 前提条件（ロックの保持状況等）を Assert で検証します。 4. 内部関数 delete_item を呼び出し、共有メモリ上のハッシュテーブルから該当エントリを削除します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/lib/dshash.c:634
  • src/backend/utils/activity/pgstat_shmem.c:876
  • **呼び出し先**:
  • delete_item（内部関数）

関連データ構造

  • dshash_seq_status: スキャン状態を管理する構造体。
  • dshash_table: ハッシュテーブルのローカル状態。
  • dshash_table_item: 削除対象となるハッシュテーブル内のエントリ。

失敗モードと境界条件

  • 前提条件の Assert に違反した場合、デバッグビルドではアボートします。
  • スキャン中にロックを保持していない状態で呼び出した場合、競合状態やデータ破損のリスクがあります。

保守担当者向けメモ

  • 本関数はシーケンシャルスキャン中に使用することを前提としています。スキャン外での削除には dshash_delete_entry を使用してください。
  • 削除操作はパーティション単位のロックに依存するため、呼び出し前に適切なロックが確保されていることを確認してください。
