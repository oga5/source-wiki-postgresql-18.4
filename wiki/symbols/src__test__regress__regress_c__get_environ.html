get_environ

定義

  • 種別: function
  • 定義箇所: src/test/regress/regress.c:433

目的

現在のプロセス環境変数リスト（environ）を取得し、PostgreSQL の text 型配列として SQL レベルへ返却するテスト用関数です。

シグネチャまたは形状

Datum get_environ(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL 経由の呼び出し引数）。
  • **出力**: Datum 型（text[] 型の配列へのポインタ）。

前提条件と不変条件

  • environ グローバル変数にアクセス可能であること（非 Windows 環境、または MSVC 環境）。
  • palloc によるメモリ割り当てが成功すること。

アルゴリズムまたは振る舞い

1. グローバル変数 environ を走査し、環境変数の総数 nvals を算出する。 2. nvals 分の Datum 配列をメモリ上に確保する。 3. 各環境変数の文字列を CStringGetTextDatum を用いて text 型の Datum に変換し、配列に格納する。 4. construct_array_builtin を使用して、TEXTOID 型の PostgreSQL 配列を構築する。 5. PG_RETURN_POINTER を介して構築した配列を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1(get_environ) によりエクスポート）。
  • **呼び出し先**: PostgreSQL 内部 API（palloc, CStringGetTextDatum, construct_array_builtin）。

関連データ構造

  • ArrayType: PostgreSQL の配列構造体。
  • TEXTOID: text 型を識別する OID。

失敗モードと境界条件

  • 環境変数が存在しない場合、空の配列を返却する。
  • メモリ不足時には palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は回帰テスト用であり、本番環境での利用は想定されていません。
  • Windows 環境（MSVC 以外）での environ の可視性については、コンパイル環境の制約に依存します。
