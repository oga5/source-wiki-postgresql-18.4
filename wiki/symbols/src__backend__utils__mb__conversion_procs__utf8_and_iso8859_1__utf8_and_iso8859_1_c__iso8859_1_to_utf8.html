iso8859_1_to_utf8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_iso8859_1/utf8_and_iso8859_1.c:40

目的

ISO-8859-1（Latin-1）エンコーディングの文字列を、UTF-8エンコーディングへ変換します。

シグネチャまたは形状

Datum iso8859_1_to_utf8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を受け取ります。
  • src (第3引数): 変換元文字列のポインタ。
  • dest (第4引数): 変換先バッファのポインタ。
  • len (第5引数): 変換対象のバイト数。
  • noError (第6引数): エラー発生時の挙動を制御するフラグ。
  • **出力**: 変換されたバイト数を Datum 型（int32）として返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_LATIN1, PG_UTF8) により、ソースがLatin-1、ターゲットがUTF-8であることを検証します。
  • 出力バッファ dest は、変換後のUTF-8文字列を格納するために十分な領域を確保している必要があります。
  • 変換完了後、dest の末尾にはヌル終端文字が書き込まれます。

アルゴリズムまたは振る舞い

1. 入力バッファの長さ len が0になるまで、1バイトずつ処理を繰り返します。 2. IS_HIGHBIT_SET マクロを用いて、文字がASCII範囲（0x00-0x7F）か否かを判定します。

  • ASCII範囲の場合: そのまま dest にコピーします。
  • それ以外の場合: 2バイトのUTF-8シーケンス（上位ビットがセットされた形式）に変換して dest に書き込みます。

    • 3. 変換処理の終了後、実際に処理したバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（PG_FUNCTION_INFO_V1 マクロによる登録経由）。
  • **呼び出し先**:
  • CHECK_ENCODING_CONVERSION_ARGS（マクロ）
  • report_invalid_encoding（エラー発生時）
  • PG_GETARG_* / PG_RETURN_*（PostgreSQL関数インターフェース）

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQLの関数引数構造体。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • c == 0（ヌル文字）が入力された場合: noError が false であれば report_invalid_encoding を呼び出し、トランザクションを中断します。true の場合はループを終了します。
  • 入力バッファの長さが不足している場合や、dest バッファが変換後のサイズを保持できない場合の挙動については、本ソースコード内では明示的なチェックは行われていません。

保守担当者向けメモ

  • 本関数はPostgreSQLの動的ロード可能なモジュールとして実装されています。
  • 変換ロジックはLatin-1の1バイトを最大2バイトのUTF-8へ変換する仕様に基づいています。
  • 呼び出し元に関する詳細なスタックトレース情報は、本ソースコード単体からは提供されていません。
