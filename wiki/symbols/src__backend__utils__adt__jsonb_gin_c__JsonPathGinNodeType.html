JsonPathGinNodeType

定義

  • 種別: enum
  • 定義箇所: src/backend/utils/adt/jsonb_gin.c:87

目的

jsonpathクエリをGINインデックスで処理可能な論理ツリー構造（JsonPathGinNode）に変換する際、各ノードの型を識別するために使用されます。

シグネチャまたは形状

typedef enum JsonPathGinNodeType
{
	JSP_GIN_OR,
	JSP_GIN_AND,
	JSP_GIN_ENTRY,
} JsonPathGinNodeType;

入力と出力

  • 本シンボル自体は列挙型であり、入力や出力は持ちません。
  • この列挙値は、make_jsp_expr_node などのノード生成関数において、構築する論理ノードの種類を指定する引数として入力されます。

前提条件と不変条件

  • JSP_GIN_OR および JSP_GIN_AND は論理演算ノードとして使用されます。
  • JSP_GIN_ENTRY はインデックス検索の最小単位であるエントリノードとして使用されます。

アルゴリズムまたは振る舞い

  • jsonpathクエリの解析過程において、クエリの論理構造を表現するツリーの各ノードの種別を定義します。
  • GINインデックスの整合性判定関数（execute_jsp_gin_node等）において、ノードの処理分岐を決定するために参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • make_jsp_expr_node
  • make_jsp_expr_node_args
  • make_jsp_expr_node_binary
  • **呼び出し先**:
  • 本シンボルは列挙型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • JsonPathGinNode: この列挙型をメンバとして保持し、論理ツリーを構成します。

失敗モードと境界条件

  • 定義外の列挙値が渡された場合の挙動については、ソースコードの文脈からは明示されていません。

保守担当者向けメモ

  • 新たな論理演算子やノードタイプをサポートする必要が生じた場合、この列挙型に値を追加し、関連するツリー構築関数および整合性判定ロジックを更新してください。
