QualItem

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/createplan.c:5422

目的

クエリの制約（quals）を評価する際のコストやセキュリティレベルを保持し、最適化プロセスにおける制約の順序付けや管理を支援するために使用されます。

シグネチャまたは形状

typedef struct {
    Node       *clause;         /* 評価対象の制約式 */
    Cost        cost;           /* 評価コスト */
    Index       security_level; /* セキュリティレベル */
} QualItem;

入力と出力

本構造体はデータ保持用であり、関数ではありません。palloc を用いて QualItem の配列としてメモリ確保され、制約の並べ替え処理等で利用されます。

前提条件と不変条件

  • clause は有効な Node ツリーを指している必要があります。
  • cost および security_level は、プランナが制約を評価する順序を決定するための指標として使用されます。

アルゴリズムまたは振る舞い

制約の並べ替え処理において、QualItem の配列が生成されます。その後、各 QualItem の cost や security_level を比較基準として、制約の評価順序を最適化するロジック内で参照・交換されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/createplan.c 内の制約順序付け処理（order_qual_clauses 等の文脈）。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接的な関数呼び出しは行いません。

関連データ構造

  • Node: 制約式を表現するために使用されます。
  • Cost: optimizer/cost.h で定義されるコスト計算用の型。

失敗モードと境界条件

  • メモリ不足時: palloc による配列確保時にエラーが発生する可能性があります。
  • 無効な clause: clause が NULL または不正なノードである場合、後続の評価処理でクラッシュする可能性があります。

保守担当者向けメモ

  • 本構造体は createplan.c 内の局所的な最適化ロジックで使用されています。
  • 制約の評価順序を変更するロジックを修正する際は、QualItem の各フィールドが正しく更新・ソートされているかを確認してください。
