unknown_attribute

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_expr.c:390

目的

SQLの式解析において、指定された属性（列名）が見つからない場合に、適切なエラーメッセージを生成して処理を中断する。

シグネチャまたは形状

static void unknown_attribute(ParseState *pstate, Node *relref, const char *attname, int location)

入力と出力

  • pstate: 解析状態を保持する ParseState 構造体へのポインタ。
  • relref: 属性の参照元となるノード。
  • attname: 見つからなかった属性名。
  • location: エラー位置を示す整数値。
  • 戻り値: なし（常に ereport(ERROR, ...) を呼び出し、制御を戻さない）。

前提条件と不変条件

  • 本関数は、呼び出し元が既に属性の解決を試みたが失敗した状況で呼び出されることを前提とする。
  • pstate は有効なポインタである必要がある。

アルゴリズムまたは振る舞い

1. relref が Var ノードであり、かつ varattno が InvalidAttrNumber である場合：

  • GetRTEByRangeTablePosn を使用して、該当する範囲テーブルエントリ（RTE）を取得する。
  • ERRCODE_UNDEFINED_COLUMN を発生させ、RTEのエイリアス名を用いてエラーを報告する。

    • 2. それ以外の場合：

  • exprType(relref) で式の型（relTypeId）を取得する。
  • 型が複合型（ISCOMPLEX）であれば、型名と共に列が見つからない旨を報告する。
  • 型が RECORDOID であれば、レコード型内で列を特定できない旨を報告する。
  • 上記以外の場合は、複合型ではない型に対してドット記法が適用されたとして ERRCODE_WRONG_OBJECT_TYPE を報告する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/parser/parse_expr.c 内の式変換ロジック（transformIndirection 等の文脈で利用）。
  • 呼び出し先: GetRTEByRangeTablePosn, exprType, format_type_be, ereport, parser_errposition。

関連データ構造

  • ParseState: 解析のコンテキスト。
  • Node, Var: 式ツリーのノード。
  • RangeTblEntry: 範囲テーブルエントリ。

失敗モードと境界条件

  • 本関数自体がエラーを投げるための関数であるため、正常終了は存在しない。
  • relref が Var ノードでない場合や、型情報が不明な場合でも、型に応じた適切なエラーメッセージを選択するよう設計されている。

保守担当者向けメモ

  • 本関数は static であり、parse_expr.c 内でのみ使用される。
  • エラーメッセージの追加や変更を行う際は、ERRCODE_UNDEFINED_COLUMN と ERRCODE_WRONG_OBJECT_TYPE の使い分けを維持すること。
