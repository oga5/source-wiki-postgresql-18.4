varchar_4

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c:121

目的

ECPG（Embedded SQL for C）プリプロセッサが生成するコードにおいて、SQLのVARCHAR型をC言語の構造体として表現するためのデータ型定義です。

シグネチャまたは形状

struct varchar_4 {
    int len;
    char arr[50];
};

入力と出力

  • **入力**: SQLクエリの結果セットから取得される文字列データ。
  • **出力**: len メンバに文字列長、arr メンバに実際の文字列データが格納されます。

前提条件と不変条件

  • arr メンバのサイズは50バイトに固定されています。
  • ECPGのランタイムライブラリは、この構造体のメモリレイアウトが len（整数）と arr（文字配列）の順序であることを前提としています。

アルゴリズムまたは振る舞い

本構造体は、onlyname という名前で2要素の配列としてインスタンス化されています。ECPGのSQL実行関数（ECPGdo等）において、ホスト変数としてこの構造体またはその配列が渡される際、ライブラリはsizeof(struct varchar_4)を用いてメモリサイズを特定し、データベースとの間でデータを転送します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c:262 において、ECPGdo 関数の引数として参照されています。
  • **呼び出し先**: 情報なし。

関連データ構造

  • onlyname: struct varchar_4 型の2要素配列。
  • ECPGdo: 本構造体をホスト変数として扱うECPGライブラリ関数。

失敗モードと境界条件

  • arr のサイズ（50バイト）を超える文字列がデータベースからフェッチされた場合、バッファオーバーフローやデータ切り捨てが発生する可能性があります。
  • len メンバが正しく設定されていない場合、SQL実行時に不正なメモリ参照やデータ不整合を引き起こす可能性があります。

保守担当者向けメモ

  • 構造体のメンバ構成を変更する場合は、ECPGプリプロセッサ側の型変換ロジックとの整合性を確認する必要があります。
