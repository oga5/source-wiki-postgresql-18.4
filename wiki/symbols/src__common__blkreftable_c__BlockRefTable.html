BlockRefTable

定義

  • 種別: struct
  • 定義箇所: src/common/blkreftable.c:144

目的

特定のLSN範囲内で変更されたブロックを追跡するための、メモリ上のメインデータ構造。WALレコードの解析結果を保持し、ディスクへのシリアライズおよびデシリアライズを仲介する。

シグネチャまたは形状

struct BlockRefTable
{
	blockreftable_hash *hash;
#ifndef FRONTEND
	MemoryContext mcxt;
#endif
};

入力と出力

  • **入力**: CreateEmptyBlockRefTable により初期化され、BlockRefTableMarkBlockModified を通じて変更済みブロック情報が入力される。
  • **出力**: WriteBlockRefTable を介して、ソートされたバイナリ形式で外部ストレージへ出力される。

前提条件と不変条件

  • lib/simplehash.h を使用したハッシュテーブルとして管理される。
  • シリアライズ時には、tablespace -> database -> relfilenumber -> forknumber の順序でソートされる必要がある。
  • 変更状態の管理において、変更密度に応じて「オフセット配列」から「ビットマップ」へ動的に切り替わるが、一度ビットマップへ変換されるとオフセット配列には戻らない。

アルゴリズムまたは振る舞い

  • BlockRefTableMarkBlockModified が呼び出されると、対応する BlockRefTableEntry を取得し、chunk を更新する。
  • 64KB（BLOCKS_PER_CHUNK）単位で管理され、MAX_ENTRIES_PER_CHUNK に達するとビットマップ形式へ最適化される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_incremental.c
  • src/backend/postmaster/walsummarizer.c (複数の関数で参照)
  • src/common/blkreftable.c (内部関数群)
  • **呼び出し先**:
  • lib/simplehash.h (ハッシュテーブル操作)
  • palloc (メモリ確保)

関連データ構造

  • BlockRefTableEntry: 特定のリレーションフォークの変更状態を保持。
  • BlockRefTableReader / BlockRefTableWriter: ディスクI/Oの抽象化。
  • BlockRefTableKey: ハッシュテーブルのキー。

失敗モードと境界条件

  • メモリ不足時: palloc によるメモリ確保失敗の可能性がある。
  • 整合性チェック: WriteBlockRefTable 実行時、ソート順序が不正な場合は期待通りのシリアライズが行われない。
  • 読み込み時: BlockRefTableReader による復元時にCRCチェックが行われ、不一致の場合はエラーとなる。

保守担当者向けメモ

  • FRONTEND 定義の有無により MemoryContext の保持が条件付きとなるため、ポータビリティに注意すること。
  • I/Oはコールバック関数経由で抽象化されているため、ファイル以外のストリームへの適用が可能である。
