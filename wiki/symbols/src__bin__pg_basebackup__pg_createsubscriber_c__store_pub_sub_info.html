store_pub_sub_info

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_createsubscriber.c:462

目的

データベースごとのパブリケーションおよびサブスクリプション情報を管理する LogicalRepInfo 構造体の配列を初期化し、接続情報や関連名を割り当てる。

シグネチャまたは形状

static struct LogicalRepInfo *store_pub_sub_info(
    const struct CreateSubscriberOptions *opt,
    const char *pub_base_conninfo,
    const char *sub_base_conninfo
)

入力と出力

  • **入力**:
  • opt: コマンドライン引数や設定を保持する構造体。
  • pub_base_conninfo: パブリッシャーへの接続文字列のベース。
  • sub_base_conninfo: サブスクライバーへの接続文字列のベース。
  • **出力**: 初期化された LogicalRepInfo 構造体の配列へのポインタ。

前提条件と不変条件

  • グローバル変数 num_dbs に基づいてメモリが確保される。
  • opt->database_names リスト内の各データベースに対して、パブリッシャーおよびサブスクライバーの接続情報が concat_conninfo_dbname を介して生成される。

アルゴリズムまたは振る舞い

1. num_dbs 分の LogicalRepInfo 配列をメモリ確保する。 2. opt 内のデータベース名リストを反復処理する。 3. 各データベースに対し、パブリッシャーおよびサブスクライバーの接続文字列を構築し、構造体に格納する。 4. パブリケーション名、サブスクリプション名、レプリケーションスロット名が指定されている場合、リストから順次割り当てる。 5. made_replslot および made_publication フラグを false に初期化する。 6. 処理内容をデバッグログに出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/pg_createsubscriber.c:2412 にて dbinfos.dbinfo の初期化に使用。
  • **呼び出し先**: concat_conninfo_dbname（接続文字列の結合）、pg_malloc_array（メモリ確保）、pg_log_debug（ロギング）。

関連データ構造

  • LogicalRepInfo: データベースごとのパブリケーション/サブスクリプション情報を保持。
  • CreateSubscriberOptions: 実行時の設定オプションを保持。
  • SimpleStringListCell: データベース名やパブリケーション名等のリスト管理に使用。

失敗モードと境界条件

  • num_dbs が 0 の場合、空の配列が返される可能性がある。
  • メモリ確保失敗時は pg_malloc_array の内部挙動により終了する。
  • パブリケーション名やスロット名のリストがデータベース数より少ない場合、リストの末尾に達した後の挙動はリストの構造に依存する（ソース上は next ポインタを辿るため、リスト長が不足している場合は未定義の動作となる可能性がある）。

保守担当者向けメモ

  • 本関数は LogicalRepInfo の初期化のみを担当し、実際のレプリケーションオブジェクト作成は後続の処理で行われる。
  • two_phase 設定は dbinfos グローバル変数から参照されているため、変更時は注意が必要。
