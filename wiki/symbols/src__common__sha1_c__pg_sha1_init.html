pg_sha1_init

定義

  • src/common/sha1.c:315

目的

SHA-1 ハッシュ計算のためのコンテキスト構造体（pg_sha1_ctx）を初期化し、アルゴリズムの初期状態を設定する。

シグネチャまたは形状

void pg_sha1_init(pg_sha1_ctx *ctx)

入力と出力

  • **入力**: pg_sha1_ctx *ctx（初期化対象となるコンテキスト構造体へのポインタ）。
  • **出力**: なし（引数として渡された ctx の内部状態が書き換えられる）。

前提条件と不変条件

  • ctx は有効なメモリ領域を指している必要がある。
  • 初期化後、コンテキスト内のハッシュ状態（H）は RFC 3174 で規定された SHA-1 の初期定数に設定される。

アルゴリズムまたは振る舞い

1. memset を使用して、ctx が指すメモリ領域全体を 0 でクリアする。 2. SHA-1 アルゴリズムの初期ハッシュ値（H0〜H4）を以下の定数で設定する：

  • H(0) = 0x67452301
  • H(1) = 0xefcdab89
  • H(2) = 0x98badcfe
  • H(3) = 0x10325476
  • H(4) = 0xc3d2e1f0

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/cryptohash.c:111 からの呼び出しが確認されている。
  • **呼び出し先**: 標準ライブラリの memset を使用。

関連データ構造

  • pg_sha1_ctx: sha1_int.h で定義される SHA-1 計算用コンテキスト。

失敗モードと境界条件

  • ctx が NULL ポインタである場合、memset によるセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は RFC 3174 に準拠したフォールバック実装の一部である。
  • H() マクロの定義については sha1_int.h を参照すること。
  • 現代的なハードウェアアクセラレーションは含まれていないため、パフォーマンスが重要な箇所では注意が必要である。
