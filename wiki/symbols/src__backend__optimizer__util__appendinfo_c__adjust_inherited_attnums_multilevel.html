adjust_inherited_attnums_multilevel

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/appendinfo.c:685

目的

多階層の継承関係（パーティションの入れ子など）において、最上位の親リレーションで定義された属性番号リストを、指定された子リレーションの属性番号へ再帰的に変換する。

シグネチャまたは形状

List *adjust_inherited_attnums_multilevel(PlannerInfo *root, List *attnums,
                                          Index child_relid, Index top_parent_relid)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • attnums: 変換対象の属性番号リスト（List）。
  • child_relid: 変換先の対象となる子リレーションのインデックス。
  • top_parent_relid: 変換の起点となる最上位親リレーションのインデックス。
  • **出力**:
  • 変換後の属性番号リスト（List）。

前提条件と不変条件

  • root->append_rel_array に child_relid が存在すること。存在しない場合はエラーとなる。
  • 変換は top_parent_relid に到達するまで再帰的に行われる。

アルゴリズムまたは振る舞い

1. root->append_rel_array から child_relid に対応する AppendRelInfo を取得する。 2. 現在の appinfo->parent_relid が top_parent_relid と一致しない場合、自身を再帰的に呼び出し、親の親へと遡って変換を行う。 3. 再帰呼び出しから戻った後、あるいは直接の親である場合に、adjust_inherited_attnums を呼び出して現在の appinfo に基づく属性変換を実行し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planner.c
  • src/backend/optimizer/util/appendinfo.c
  • **呼び出し先**:
  • adjust_inherited_attnums (直接の変換処理)
  • adjust_inherited_attnums_multilevel (再帰呼び出し)

関連データ構造

  • PlannerInfo: プランナの全体状態を保持。
  • AppendRelInfo: 親子リレーション間の変換メタデータ。

失敗モードと境界条件

  • child_relid が append_rel_array に存在しない場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は、継承階層が深い場合に属性番号を正しく伝播させるために不可欠である。
  • 変換ロジックは adjust_inherited_attnums に依存しているため、属性マッピングの仕様変更時は両方の整合性を確認すること。
