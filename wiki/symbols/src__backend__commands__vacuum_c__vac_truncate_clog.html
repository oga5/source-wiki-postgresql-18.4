vac_truncate_clog

定義

  • 種別: function
  • 定義箇所: src/backend/commands/vacuum.c:1842

目的

トランザクションログ（CLOG）、マルチトランザクションID、およびコミットタイムスタンプのストレージを、指定された凍結トランザクションID（frozenXID）および最小マルチトランザクションID（minMulti）まで切り詰める。

シグネチャまたは形状

void vac_truncate_clog(TransactionId frozenXID,
                       MultiXactId minMulti,
                       TransactionId lastSaneFrozenXid,
                       MultiXactId lastSaneMinMulti)

入力と出力

  • **入力**:
  • frozenXID: 切り詰め対象の凍結トランザクションID。
  • minMulti: 切り詰め対象の最小マルチトランザクションID。
  • lastSaneFrozenXid, lastSaneMinMulti: 整合性チェック用の基準値。
  • **出力**: なし（副作用としてストレージの切り詰めと統計情報の更新を行う）。

前提条件と不変条件

  • WrapLimitsVacuumLock 排他ロックを保持して実行する必要がある。
  • pg_database 内の各データベースの datfrozenxid および datminmxid を走査し、システム全体の最小値を算出する。
  • 算出された値が lastSane 値を超えている場合、またはトランザクションのラップアラウンドが既に発生していると判断された場合は、切り詰めを行わずに終了する。

アルゴリズムまたは振る舞い

1. WrapLimitsVacuumLock を取得する。 2. pg_database をスキャンし、全データベースの中で最も古い datfrozenxid と datminmxid を特定する。 3. 無効なデータベースや、将来の日付を持つ不正なデータ（bogus）を検出した場合は処理を中断する。 4. AsyncNotifyFreezeXids を呼び出し、非同期通知キュー内の古いXIDを凍結する。 5. AdvanceOldestCommitTsXid を呼び出し、コミットタイムスタンプの最古値を更新する。 6. TruncateCLOG, TruncateCommitTs, TruncateMultiXact を順次呼び出し、ストレージを切り詰める。 7. SetTransactionIdLimit および SetMultiXactIdLimit で新しい制限値を設定する。 8. ロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/vacuum.c 内の vacuum() 等から呼び出される。
  • **呼び出し先**: table_open, table_beginscan_catalog, heap_getnext, AsyncNotifyFreezeXids, AdvanceOldestCommitTsXid, TruncateCLOG, TruncateCommitTs, TruncateMultiXact, SetTransactionIdLimit, SetMultiXactIdLimit 等。

関連データ構造

  • pg_database（システムカタログ）
  • TransactionId, MultiXactId
  • WrapLimitsVacuumLock（LWLock）

失敗モードと境界条件

  • **ラップアラウンド**: frozenAlreadyWrapped が真の場合、警告を出して切り詰めを中止する。
  • **不正なデータ**: bogus フラグが立つ場合（pg_upgrade のバグ等に起因する将来のXID検出時）、切り詰めを中止する。
  • **同時実行**: pg_database の更新はアトミックであると仮定している。

保守担当者向けメモ

  • 本関数はクラスタ全体で1つのバックエンドのみが実行するように設計されている。
  • pg_database のスキャン中に CREATE DATABASE 等による競合が発生しても、最小値の算出には影響しないよう設計されている。
  • TruncateCLOG 等の呼び出し順序は、コミットタイムスタンプの参照エラーを防ぐために重要である。
