FetchPortalTargetList

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/pquery.c:326

目的

指定されたポータル（Portal）が返すタプルのターゲットリスト（List *）を取得します。

シグネチャまたは形状

List *FetchPortalTargetList(Portal portal)

入力と出力

  • **入力**: portal (実行状態を保持する Portal 構造体へのポインタ)
  • **出力**: ターゲットリストを表す List 構造体へのポインタ。タプルを返さない場合は NIL を返します。

前提条件と不変条件

  • portal は有効なポータルオブジェクトを指している必要があります。
  • ポータルの実行戦略が PORTAL_MULTI_QUERY である場合、タプルを返さないものとみなされ、直ちに NIL が返されます。

アルゴリズムまたは振る舞い

1. ポータルの strategy フィールドを確認し、PORTAL_MULTI_QUERY であれば NIL を返して終了します。 2. それ以外の場合、PortalGetPrimaryStmt(portal) を呼び出してポータル内のプライマリステートメントを取得します。 3. 取得したステートメントを Node * にキャストし、FetchStatementTargetList を呼び出してその結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/printtup.c:139
  • src/backend/tcop/postgres.c:2776
  • src/backend/tcop/pquery.c:397
  • **呼び出し先**:
  • FetchStatementTargetList (ソースコード内の直接的な呼び出し)
  • PortalGetPrimaryStmt (マクロまたはインライン関数と推測される)

関連データ構造

  • Portal: ポータルの実行状態を保持する構造体。
  • List: PostgreSQL のリスト構造体。
  • PlannedStmt: PortalGetPrimaryStmt が返すクエリ計画。

失敗モードと境界条件

  • portal が PORTAL_MULTI_QUERY 戦略を持つ場合、ターゲットリストの取得は行われず NIL が返されます。
  • FetchStatementTargetList が内部で失敗した場合の挙動は、当該関数の実装に依存します。

保守担当者向けメモ

  • 本関数は、ポータルが保持するクエリの出力形式を特定するためのユーティリティです。
  • FetchStatementTargetList への依存が強いため、ターゲットリストの抽出ロジックを変更する場合は FetchStatementTargetList の仕様も併せて確認してください。
