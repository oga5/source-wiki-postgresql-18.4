PLyUnicode_AsString

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_util.c:80

目的

PythonのUnicodeオブジェクトを、PostgreSQLのサーバーエンコーディングに準拠したC文字列（palloc済み）へ変換します。

シグネチャまたは形状

char *PLyUnicode_AsString(PyObject *unicode)

入力と出力

  • **入力**: PyObject *unicode（変換対象のPython Unicodeオブジェクト）
  • **出力**: char *（PostgreSQLのメモリコンテキストで割り当てられた、サーバーエンコーディングのヌル終端文字列）

前提条件と不変条件

  • 入力 unicode は有効なPythonオブジェクトである必要があります。
  • 戻り値の文字列は palloc を使用してメモリ確保されているため、呼び出し元で不要になった際に pfree で解放する必要があります。

アルゴリズムまたは振る舞い

1. PLyUnicode_Bytes(unicode) を呼び出し、Python Unicodeオブジェクトをサーバーエンコーディングのバイト列（PyObject）に変換します。 2. PyBytes_AsString を使用して、変換されたバイト列からC文字列ポインタを取得します。 3. pstrdup を使用して、その文字列をPostgreSQLのメモリコンテキストへ複製します。 4. Py_XDECREF を呼び出し、手順1で生成した一時的なバイト列オブジェクトの参照カウントを減らします。 5. 複製した文字列ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_cursorobject.c
  • src/pl/plpython/plpy_elog.c
  • src/pl/plpython/plpy_exec.c
  • src/pl/plpython/plpy_plpymodule.c
  • **呼び出し先**:
  • PLyUnicode_Bytes
  • PyBytes_AsString
  • pstrdup
  • Py_XDECREF

関連データ構造

  • PyObject: Python側のオブジェクト表現。
  • char *: PostgreSQL側で扱われるC文字列。

失敗モードと境界条件

  • PLyUnicode_Bytes が内部でエラーを発生させた場合、Pythonの例外機構またはPostgreSQLの elog を介したエラー報告が行われます。
  • 入力 unicode が NULL の場合、PLyUnicode_Bytes の実装依存の挙動となります。

保守担当者向けメモ

  • 本関数は pstrdup を使用しているため、呼び出し元でのメモリリークを防ぐために pfree の呼び出しが必須です。
  • Pythonの参照カウント管理とPostgreSQLのメモリ管理の橋渡しを行うため、変更時は Py_XDECREF の整合性に注意してください。
