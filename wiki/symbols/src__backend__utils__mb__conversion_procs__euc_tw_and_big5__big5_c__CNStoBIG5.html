CNStoBIG5

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_tw_and_big5/big5.c:344

目的

CNS 11643（Mule内部コード）からBIG5エンコーディングへの文字コード変換を行う。

シグネチャまたは形状

unsigned short CNStoBIG5(unsigned short cns, unsigned char lc)

入力と出力

  • **入力**:
  • cns: 変換対象のCNS 11643コード。
  • lc: CNSのプレーンを指定する識別子（LC_CNS11643_1〜4）。
  • **出力**:
  • 変換後のBIG5コード（unsigned short）。変換不能な場合は 0 を返す。

前提条件と不変条件

  • 入力された cns は関数冒頭で 0x7f7f とのビット論理積がとられ、上位ビットがマスクされる。
  • 変換ロジックは lc で指定されたプレーンに基づき、範囲ベースの計算または静的テーブル検索を行う。

アルゴリズムまたは振る舞い

1. cns の上位・下位バイトをそれぞれ7ビットに制限する。 2. lc に基づき以下のいずれかの処理を実行する:

  • LC_CNS11643_1: cnsPlane1ToBig5Level1 テーブルを用いて BinarySearchRange で変換。
  • LC_CNS11643_2: cnsPlane2ToBig5Level2 テーブルを用いて BinarySearchRange で変換。
  • LC_CNS11643_3: b2c3 テーブルを線形探索し、一致するCNSコードに対応するBIG5コードを返す。
  • LC_CNS11643_4: b1c4 テーブルを線形探索し、一致するCNSコードに対応するBIG5コードを返す。

    • 3. 該当する変換が見つからない場合、初期値 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mb/conversion_procs/euc_tw_and_big5/euc_tw_and_big5.c 内の変換処理関数。
  • **呼び出し先**:
  • BinarySearchRange（範囲変換用）

関連データ構造

  • cnsPlane1ToBig5Level1, cnsPlane2ToBig5Level2: 範囲変換用テーブル。
  • b2c3, b1c4: 例外マッピング用の静的配列。

失敗モードと境界条件

  • 指定された lc が定義されたプレーン（1〜4）以外の場合、変換を行わず 0 を返す。
  • 探索対象のテーブル内に一致するコードが存在しない場合、0 を返す。

保守担当者向けメモ

  • 本関数はフロントエンドとバックエンドの両方から利用されるため、PostgreSQLのコア機能への依存を避ける必要がある。
  • 新しいマッピングの追加は、b1c4 や b2c3 のような静的配列の更新、または BinarySearchRange 用の範囲テーブルの調整によって行う。
