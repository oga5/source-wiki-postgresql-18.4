heap_prune_record_dead_or_unused

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/pruneheap.c:1279

目的

ヒープページ内の不要なタプル（デッドタプル）に対し、PruneState の設定に基づいて、即座に未使用（LP_UNUSED）としてマークするか、単にデッドとして記録するかを決定し、適切な記録関数へ委譲する。

シグネチャまたは形状

static void heap_prune_record_dead_or_unused(PruneState *prstate, OffsetNumber offnum, bool was_normal)

入力と出力

  • prstate: 剪定処理の状態を保持する PruneState 構造体へのポインタ。
  • offnum: 処理対象のタプルのオフセット番号。
  • was_normal: タプルが通常のタプルであるかを示すフラグ。
  • 出力: なし（副作用として prstate 内の記録配列を更新）。

前提条件と不変条件

  • prstate は有効な PruneState 構造体を指している必要がある。
  • offnum はページ内の有効なオフセット番号であること。

アルゴリズムまたは振る舞い

1. prstate->mark_unused_now が真の場合、heap_prune_record_unused を呼び出し、対象タプルのラインポインタを即座に LP_UNUSED に設定する。 2. prstate->mark_unused_now が偽の場合、heap_prune_record_dead を呼び出し、デッドタプルとして記録する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/pruneheap.c 内の剪定ロジック（heap_prune_chain 等から参照されている）。
  • 呼び出し先: heap_prune_record_unused または heap_prune_record_dead。

関連データ構造

  • PruneState: 剪定処理の制御フラグ（mark_unused_now）を保持する。

失敗モードと境界条件

  • 特筆すべき失敗モードは定義されていない。入力値が不正な場合、呼び出し先の関数でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は mark_unused_now フラグによる条件分岐のラッパーである。将来的に剪定の記録方法に変更を加える場合は、この分岐ロジックを維持すること。
