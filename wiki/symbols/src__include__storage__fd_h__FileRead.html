FileRead

定義

  • 種別: static inline function
  • 定義箇所: src/include/storage/fd.h:209

目的

仮想ファイル記述子（VFD）を介して、指定されたオフセットからデータを読み取るためのラッパー関数です。

シグネチャまたは形状

ssize_t FileRead(File file, void *buffer, size_t amount, off_t offset, uint32 wait_event_info)

入力と出力

  • **入力**:
  • file: 読み取り対象の File 型ハンドル。
  • buffer: 読み取ったデータを格納するメモリ領域。
  • amount: 読み取るバイト数。
  • offset: ファイル内の読み取り開始位置。
  • wait_event_info: 待機イベント情報。
  • **出力**:
  • 読み取ったバイト数（ssize_t）。エラー時は負の値を返す。

前提条件と不変条件

  • VFD管理システムにおいて、ファイルポインタの概念は存在しないため、必ず offset を指定する必要があります。
  • 呼び出し元は、PostgreSQLのVFD管理ルールに従い、標準の read(2) 等を直接使用せず、本関数を経由してアクセスする必要があります。

アルゴリズムまたは振る舞い

1. 受け取った buffer と amount を用いて struct iovec を初期化します。 2. 内部関数である FileReadV を呼び出し、単一のベクトル（iovcnt=1）として読み取り処理を委譲します。

呼び出し元と呼び出し先

  • **呼び出し先**: FileReadV
  • **呼び出し元**:
  • src/backend/backup/walsummary.c
  • src/backend/replication/logical/reorderbuffer.c
  • src/backend/storage/file/buffile.c

関連データ構造

  • File: VFDを表す整数型。
  • iovec: 読み取りバッファの構造体（port/pg_iovec.h で定義）。

失敗モードと境界条件

  • VFDが正しくオープンされていない場合や、指定したオフセットがファイルサイズを超過している場合などの挙動は、内部の FileReadV の実装およびOSの pread(2) 等の挙動に依存します。

保守担当者向けメモ

  • 本関数は FileReadV の単一バッファ用ラッパーです。ロジックの変更は FileReadV 側で行うことが推奨されます。
  • File 型はOSのファイル記述子と直接互換性がないため、直接的なOSレベルの操作と混同しないよう注意してください。
