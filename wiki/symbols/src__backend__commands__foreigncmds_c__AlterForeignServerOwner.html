AlterForeignServerOwner

定義

  • 種別: function
  • 定義箇所: src/backend/commands/foreigncmds.c:425

目的

指定された外部サーバーの所有者を変更する。

シグネチャまたは形状

ObjectAddress AlterForeignServerOwner(const char *name, Oid newOwnerId)

入力と出力

  • **入力**:
  • name: 変更対象の外部サーバー名。
  • newOwnerId: 新しい所有者のロールOID。
  • **出力**:
  • 変更された外部サーバーの ObjectAddress を返す。

前提条件と不変条件

  • pg_foreign_server システムカタログに対する RowExclusiveLock を保持して実行される。
  • 指定された名前のサーバーが存在しない場合、エラーが発生する。

アルゴリズムまたは振る舞い

1. pg_foreign_server テーブルを RowExclusiveLock でオープンする。 2. FOREIGNSERVERNAME システムキャッシュを使用して、対象サーバーのタプルを検索・コピーする。 3. サーバーが存在しない場合は ERRCODE_UNDEFINED_OBJECT でエラーを報告する。 4. AlterForeignServerOwner_internal を呼び出し、所有者変更の内部処理（権限チェックやカタログ更新）を実行する。 5. 変更対象の ObjectAddress を構築する。 6. コピーしたタプルを解放し、テーブルをクローズして終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/alter.c:859 (SQLコマンド実行時)
  • **呼び出し先**:
  • table_open / table_close
  • SearchSysCacheCopy1 / heap_freetuple
  • AlterForeignServerOwner_internal
  • ObjectAddressSet

関連データ構造

  • ObjectAddress: 操作対象オブジェクトの識別子。
  • Form_pg_foreign_server: 外部サーバーカタログのタプル構造体。
  • HeapTuple: システムカタログの行データ。

失敗モードと境界条件

  • 指定した名前の外部サーバーが見つからない場合、ereport(ERROR, ...) によりトランザクションが中断される。

保守担当者向けメモ

  • 本関数は所有者変更のラッパーであり、実際のカタログ更新ロジックは AlterForeignServerOwner_internal に委譲されている。
  • 外部サーバーの定義変更に伴う依存関係や権限の整合性は、内部関数側で管理される。
