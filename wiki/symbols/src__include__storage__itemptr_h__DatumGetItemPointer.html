DatumGetItemPointer

定義

  • 種別: function
  • 定義箇所: src/include/storage/itemptr.h:230

目的

PostgreSQL の内部データ型である Datum を、物理的なタプル位置を指す ItemPointer（ItemPointerData へのポインタ）型へキャストして取得します。

シグネチャまたは形状

static inline ItemPointer DatumGetItemPointer(Datum X)

入力と出力

  • **入力**: Datum X（ItemPointer を保持する Datum 型の値）
  • **出力**: ItemPointer（ItemPointerData 構造体へのポインタ）

前提条件と不変条件

  • 入力 Datum X は、有効な ItemPointer を指すポインタとして格納されている必要があります。
  • 本関数は型変換を行うのみであり、ポインタの有効性や指し示すメモリ領域の整合性については検証しません。

アルゴリズムまたは振る舞い

1. DatumGetPointer(X) を呼び出し、Datum から汎用ポインタ型へ変換します。 2. 変換されたポインタを ItemPointer 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/storage/itemptr.h:242 に定義されたマクロ PG_GETARG_ITEMPOINTER(n) から呼び出されます。
  • **呼び出し先**:
  • DatumGetPointer（Datum をポインタへ変換する基底関数）。

関連データ構造

  • ItemPointerData: 物理的なタプル位置を保持する 6 バイトの構造体。
  • ItemPointer: ItemPointerData へのポインタ型。
  • Datum: PostgreSQL における汎用データ表現型。

失敗モードと境界条件

  • 本関数自体は単純なキャストであるため、関数内部での失敗モードはありません。
  • 無効な Datum 値を渡した場合、戻り値の ItemPointer は不正なメモリ領域を指すことになります。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、パフォーマンスを重視するパスで使用されます。
  • ItemPointerData のメモリレイアウト（6 バイト）に依存しているため、構造体定義に変更を加える際は本関数が正しく動作するか確認してください。
