PortalCleanup

定義

  • 種別: function
  • 定義箇所: src/backend/commands/portalcmds.c:272

目的

ポータル破棄時に実行されるクリーンアップフックであり、ポータルに関連付けられたエグゼキュータを安全に停止し、リソースを解放する。

シグネチャまたは形状

void PortalCleanup(Portal portal)

入力と出力

  • **入力**: Portal portal（クリーンアップ対象のポータル構造体へのポインタ）
  • **出力**: なし（void）

前提条件と不変条件

  • portal は有効であること（PortalIsValid(portal)）。
  • portal->cleanup は PortalCleanup 関数自身を指していること。

アルゴリズムまたは振る舞い

1. ポータルが保持する queryDesc を取得する。 2. queryDesc が存在する場合、二重解放を防ぐためにポータルの queryDesc を NULL にリセットする。 3. ポータルの状態が PORTAL_FAILED でない場合のみ、以下の順序で実行する：

  • ポータルのリソースオーナー（resowner）を現在のリソースオーナーとして設定する。
  • ExecutorFinish(queryDesc) を呼び出し、実行を完了させる。
  • ExecutorEnd(queryDesc) を呼び出し、エグゼキュータを終了させる。
  • FreeQueryDesc(queryDesc) を呼び出し、クエリ記述子を解放する。
  • 元のリソースオーナーを復元する。
  • エラーによるアボート時は、エグゼキュータの停止処理をスキップする（他のメカニズムがリソース解放を担うため）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/portalmem.c（ポータルの初期化時にフックとして設定）
  • src/backend/commands/portalcmds.c（ポータル操作の文脈で呼び出される）
  • src/backend/tcop/pquery.c（エグゼキュータ停止の指示として利用）
  • **呼び出し先**:
  • ExecutorFinish
  • ExecutorEnd
  • FreeQueryDesc

関連データ構造

  • Portal: ポータルの状態を保持する構造体。
  • QueryDesc: クエリ実行に必要な情報を保持する構造体。
  • ResourceOwner: リソースの所有権を管理する構造体。

失敗モードと境界条件

  • portal->status == PORTAL_FAILED の場合、エグゼキュータの停止処理はスキップされる。
  • エラー発生時のアボート処理中には、ExecutorEnd 自体が失敗する可能性があるため、実行を避ける設計となっている。

保守担当者向けメモ

  • 本関数はエラー発生時にも呼び出されるため、二重エラーを防ぐ堅牢な実装が求められる。
  • portal->queryDesc を NULL にリセットする操作は、再帰的な呼び出しやエラー発生時の二重解放を防止するための重要なガードである。
