LexemeHashKey

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/ts_typanalyze.c:26

目的

tsvector 型の統計情報収集（ANALYZE）において、ハッシュテーブル内で個々の語彙（lexeme）を識別するためのキーとして使用されます。

シグネチャまたは形状

typedef struct {
    char   *lexeme; /* 語彙（NULL終端ではない） */
    int     length; /* バイト単位の長さ */
} LexemeHashKey;

入力と出力

  • 本構造体は、ハッシュテーブルのキーとして入力され、ハッシュ計算や比較関数の引数として利用されます。

前提条件と不変条件

  • lexeme フィールドは NULL 終端されていません。
  • length フィールドは lexeme が指すメモリ領域の正確なバイト長を保持している必要があります。

アルゴリズムまたは振る舞い

  • ts_typanalyze における Lossy Counting アルゴリズムの実行中、語彙の出現を追跡するためのハッシュテーブルのキーとして機能します。
  • ハッシュテーブルの構築時、hash_ctl.keysize に sizeof(LexemeHashKey) が指定されます。

呼び出し元と呼び出し先

  • 呼び出し元: compute_tsvector_stats 等、ts_typanalyze.c 内の統計計算ロジック。
  • 呼び出し先: lexeme_hash（ハッシュ値計算）、lexeme_match（キー比較）、lexeme_compare（ソート用比較）などの関数が、この構造体を引数として受け取ります。

関連データ構造

  • TrackItem: ハッシュテーブルのエントリとして、LexemeHashKey と共に頻度や誤差情報を保持します。

失敗モードと境界条件

  • lexeme が指すメモリ領域が適切に確保されていない場合、または length が実際のデータ長と一致しない場合、ハッシュテーブルの検索や比較処理で未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は palloc されたメモリを指すことが多いため、メモリ管理（特に ANALYZE 終了時の解放）には注意が必要です。
  • lexeme が NULL 終端されていない点に留意し、文字列操作を行う際は必ず length を使用してください。
