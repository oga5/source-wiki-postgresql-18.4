setop_load_group

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSetOp.c:330

目的

ソート済み入力から同一グループに属するタプルを読み込み、その出現回数をカウントする。

シグネチャまたは形状

static void setop_load_group(SetOpStatePerInput *input, PlanState *inputPlan, SetOpState *setopstate)

入力と出力

  • **入力**:
  • input: 現在の入力状態を保持する SetOpStatePerInput 構造体。
  • inputPlan: 子プランノードの実行状態。
  • setopstate: SetOp ノード全体の実行状態。
  • **出力**: なし（副作用として input 構造体のフィールドを更新する）。

前提条件と不変条件

  • 入力プランはソート済みであること。
  • setop_compare_slots による比較結果が 0 以下であること（ソート順序の不変条件）。

アルゴリズムまたは振る舞い

1. input->needGroup を false に設定する。 2. input->nextTupleSlot が空の場合、グループは空とみなし input->numTuples を 0 に設定して終了する。 3. input->nextTupleSlot の内容を input->firstTupleSlot にコピーし、カウントを 1 に初期化する。 4. ループ内で ExecProcNode を呼び出し、後続のタプルを取得する。 5. setop_compare_slots を使用して、取得したタプルが現在のグループ（firstTupleSlot）と一致するか判定する。

  • 一致する場合: input->numTuples をインクリメントし、ループを継続する。
  • 不一致の場合: ループを終了し、次のグループの先頭タプルを input->nextTupleSlot に保持したまま戻る。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeSetOp.c 内の ExecSetOp（または関連する実行ロジック）から、左入力および右入力のグループ読み込み時に呼び出される。
  • **呼び出し先**:
  • ExecProcNode (子ノードからのタプル取得)
  • setop_compare_slots (グループ判定)
  • ExecStoreMinimalTuple / ExecCopySlotMinimalTuple (タプル保持)

関連データ構造

  • SetOpStatePerInput: 入力状態を管理する構造体。
  • SetOpState: SetOp ノードの実行状態を保持する構造体。
  • PlanState: 子プランの実行状態。

失敗モードと境界条件

  • 入力が空の場合、numTuples は 0 となり、正常に処理される。
  • 入力がソートされていない場合、Assert によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は SETOP_SORTED 戦略において、マージ処理の基盤となるグループ化ロジックを担う。
  • input->nextTupleSlot には常に「現在のグループに含まれない次のタプル」または「空」が保持されるため、呼び出し側はこの状態を維持して処理を進める必要がある。
