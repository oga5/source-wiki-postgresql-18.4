tsquery_and

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_op.c:53

目的

2つの tsquery 型の値を論理積（AND）演算子で結合し、新しい tsquery を生成する。

シグネチャまたは形状

Datum tsquery_and(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの TSQuery 型の引数を受け取る。
  • **出力**: Datum 型として、結合された結果の TSQuery を返す。

前提条件と不変条件

  • 入力された TSQuery は PG_GETARG_TSQUERY_COPY によりコピーとして取得される。
  • 引数のいずれかが空（size == 0）の場合、もう一方の引数がそのまま返される。
  • 処理終了時には、一時的に生成された QTNode ツリーおよび入力引数のコピーを適切に解放する必要がある。

アルゴリズムまたは振る舞い

1. 引数 a および b のサイズを確認し、一方が空であれば他方をそのまま返す。 2. join_tsqueries を呼び出し、OP_AND を用いて2つの tsquery を QTNode ツリー構造に結合する。 3. QTN2QT を使用して、構築された QTNode ツリーを TSQuery 形式（バイト列）に変換する。 4. QTNFree でツリー構造を解放し、PG_FREE_IF_COPY で入力引数のコピーを解放する。 5. 変換された TSQuery を PG_RETURN_TSQUERY で返す。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • join_tsqueries: ツリー構造の結合に使用。
  • QTN2QT: ツリーから TSQuery への変換に使用。
  • QTNFree: メモリ解放に使用。
  • **呼び出し元**: ソースコード内での直接的な呼び出し元は明示されていない（PostgreSQL の演算子ディスパッチャから呼び出される想定）。

関連データ構造

  • TSQuery: 全文検索クエリの内部表現。
  • QTNode: 演算処理のための中間的なツリー構造。

失敗モードと境界条件

  • 引数が空（size == 0）の場合、演算を行わずに早期リターンする。
  • メモリ管理において、QTNFree や PG_FREE_IF_COPY を適切に呼び出さない場合、メモリリークが発生する可能性がある。

保守担当者向けメモ

  • 本関数は tsquery の論理演算の基本形である。他の論理演算（tsquery_or 等）も同様のパターンで実装されているため、修正時は一貫性に注意すること。
  • QTN_NEEDFREE フラグを持つノードの解放漏れがないよう、QTNFree の呼び出し位置を維持すること。
