RelationGetSmgr

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/rel.h:577

目的

リレーション記述子（Relation）に関連付けられたストレージマネージャ（SMgrRelation）を取得します。必要に応じてストレージマネージャの初期化（オープン）を遅延実行します。

シグネチャまたは形状

static inline SMgrRelation RelationGetSmgr(Relation rel)

入力と出力

  • **入力**: rel (Relation) - 対象となるリレーション記述子へのポインタ。
  • **出力**: SMgrRelation - リレーションに対応するストレージマネージャへのポインタ。

前提条件と不変条件

  • rel は有効な Relation 構造体を指している必要があります。
  • rel->rd_smgr が NULL の場合、本関数は smgropen を呼び出してストレージマネージャを初期化します。
  • 呼び出し後、rd_smgr は常に有効な SMgrRelation を保持します。

アルゴリズムまたは振る舞い

1. rel->rd_smgr が NULL であるかを確認します（unlikely マクロを使用）。 2. NULL の場合:

  • smgropen(rel->rd_locator, rel->rd_backend) を呼び出し、ストレージマネージャをオープンして rd_smgr に代入します。
  • smgrpin(rel->rd_smgr) を呼び出し、ストレージマネージャをピン留めします。

    • 3. rel->rd_smgr を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_nbtree.c
  • contrib/pg_prewarm/autoprewarm.c
  • contrib/pg_prewarm/pg_prewarm.c
  • contrib/pg_visibility/pg_visibility.c
  • src/backend/access/gist/gistbuild.c
  • src/backend/access/hash/hashpage.c
  • src/backend/access/heap/heapam_handler.c
  • **呼び出し先**:
  • smgropen
  • smgrpin

関連データ構造

  • RelationData (Relation): リレーションのメタデータを保持する構造体。
  • SMgrRelation: 物理ストレージ管理のためのハンドル。

失敗モードと境界条件

  • rel が NULL の場合、セグメンテーション違反が発生します。
  • smgropen が失敗した場合（メモリ不足やファイルオープンエラー等）、その挙動は smgropen の実装に依存します。

保守担当者向けメモ

  • 本関数はインライン関数であり、頻繁に呼び出されるためオーバーヘッドを最小限に抑える設計となっています。
  • rd_smgr のクローズ処理は RelationCloseSmgr が担当します。キャッシュの整合性を維持するため、直接 rd_smgr を操作しないでください。
