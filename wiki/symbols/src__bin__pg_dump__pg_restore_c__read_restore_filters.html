read_restore_filters

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_restore.c:618

目的

外部ファイルから復元対象のフィルタリングルール（インクルード/エクスクルーシブ）を読み込み、RestoreOptions 構造体に適用する。

シグネチャまたは形状

static void read_restore_filters(const char *filename, RestoreOptions *opts)

入力と出力

  • **入力**:
  • filename: フィルタ定義が記述されたファイルパス。
  • opts: 復元オプションを保持する RestoreOptions 構造体へのポインタ。
  • **出力**: なし（副作用として opts の各メンバを更新する）。

前提条件と不変条件

  • filter_init により FilterStateData が初期化されていること。
  • フィルタファイル内の各項目は filter_read_item で解析可能であること。
  • 許可されていないオブジェクトタイプに対してフィルタが指定された場合、exit_nicely(1) を呼び出しプログラムを終了する。

アルゴリズムまたは振る舞い

1. filter_init を呼び出し、指定されたファイルからフィルタ状態を初期化する。 2. filter_read_item をループで呼び出し、各フィルタ項目を取得する。 3. FILTER_COMMAND_TYPE_INCLUDE の場合:

  • 関数、インデックス、テーブル、トリガーについては opts->selTypes を有効にし、対応する opts メンバに名前を追加する。
  • スキーマについては opts->schemaNames に追加する。
  • テーブルデータやデータベース等の指定は許可されておらず、エラーを出力して終了する。

    • 4. FILTER_COMMAND_TYPE_EXCLUDE の場合:

  • スキーマのみ opts->schemaExcludeNames への追加を許可する。
  • それ以外のオブジェクトタイプはエラーを出力して終了する。

    • 5. 処理終了後、filter_free でリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_restore.c 内のメイン処理（main 関数等から optarg を引数として呼び出される）。
  • **呼び出し先**: filter_init, filter_read_item, filter_object_type_name, pg_log_filter_error, simple_string_list_append, exit_nicely, free。

関連データ構造

  • RestoreOptions: 復元設定を保持する構造体。
  • FilterStateData: フィルタ読み込みの状態管理用構造体。
  • FilterCommandType, FilterObjectType: フィルタの種別を定義する列挙型。

失敗モードと境界条件

  • 指定されたファイルが読み込めない場合や、フィルタ形式が不正な場合は filter_read_item の挙動に依存する。
  • 許可されていないオブジェクトタイプに対するフィルタ指定は、pg_log_filter_error を経由して exit_nicely(1) により強制終了される。

保守担当者向けメモ

  • 本関数は pg_restore のコマンドライン引数で指定されたフィルタファイルを処理する。
  • 新たなフィルタ対象オブジェクトを追加する場合は、switch 文の各ケースを更新する必要がある。
