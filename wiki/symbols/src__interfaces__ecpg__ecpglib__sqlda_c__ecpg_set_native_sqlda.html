ecpg_set_native_sqlda

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/sqlda.c:443

目的

構築済みのネイティブモード SQLDA 構造体に対し、PGresult から取得した特定の行データを格納し、各フィールドのデータポインタおよびメタデータを設定します。

シグネチャまたは形状

void ecpg_set_native_sqlda(int lineno, struct sqlda_struct **_sqlda, const PGresult *res, int row, enum COMPAT_MODE compat)

入力と出力

  • **入力**:
  • lineno: エラーログ出力用行番号。
  • _sqlda: 設定対象の SQLDA 構造体へのポインタ。
  • res: データソースとなる PGresult オブジェクト。
  • row: 抽出対象の行インデックス。
  • compat: 互換モード指定。
  • **出力**: なし（_sqlda が指す構造体内部の sqldata および sqlind が更新されます）。

前提条件と不変条件

  • row が 0 未満の場合は何も行われません。
  • SQLDA 構造体は事前に ecpg_build_native_sqlda 等でメモリ確保および初期化されている必要があります。
  • SQLDA 内のメタデータとデータ領域は単一の連続したメモリブロックとして管理されており、ecpg_sqlda_align_add_size によってアライメントが保証されます。

アルゴリズムまたは振る舞い

1. sqlda_native_empty_size を用いて、データ格納開始位置のオフセットを算出します。 2. sqlda->sqld（列数）分だけループし、各列の sqltype に応じて以下の処理を行います。

  • ecpg_sqlda_align_add_size を呼び出し、型に応じたアライメント調整とオフセット更新を行います。
  • sqldata ポインタを計算されたオフセット位置へ設定します。
  • numeric 型の場合、PGTYPESnumeric_from_asc を使用して変換し、必要に応じてバッファを再配置します。
  • その他の型は ecpg_get_data を介してデータを転送します。

    • 3. PQgetisnull で NULL 判定を行い、sqlind に value_is_null または value_is_not_null を設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/execute.c 内の ecpg_set_native_sqlda 呼び出しが確認されています。
  • **呼び出し先**: sqlda_native_empty_size, ecpg_sqlda_align_add_size, PQgetisnull, PQgetvalue, PGTYPESnumeric_from_asc, ECPGset_noind_null, PGTYPESnumeric_free, ecpg_get_data, ecpg_log。

関連データ構造

  • sqlda_struct: ECPG ネイティブモードの SQLDA 構造体。
  • PGresult: libpq のクエリ結果構造体。
  • COMPAT_MODE: 互換モード定義。

失敗モードと境界条件

  • row が範囲外の場合、PQgetisnull 等で libpq 側のエラーが発生する可能性があります。
  • numeric 型の変換に失敗した場合、ECPGset_noind_null が呼び出され、NULL として扱われます。
  • メモリ配置は ecpg_sqlda_align_add_size に依存しており、アライメント計算が不適切な場合、不正なメモリアクセスが発生するリスクがあります。

保守担当者向けメモ

  • 本関数はネイティブモード専用です。Informix 互換モードの修正時は ecpg_set_compat_sqlda を確認してください。
  • SQLDA はメタデータとデータが同一ブロックにあるため、メモリ管理の際は sqlda ポインタの整合性に注意してください。
