make_restrictinfo

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/restrictinfo.c:51

目的

クエリプランナにおいて、式（Expr）を最適化に必要なメタデータを含む RestrictInfo ノードへ変換する主要なエントリポイントです。

シグネチャまたは形状

RestrictInfo *make_restrictinfo(PlannerInfo *root,
                                Expr *clause,
                                bool is_pushed_down,
                                bool has_clone,
                                bool is_clone,
                                bool pseudoconstant,
                                Index security_level,
                                Relids required_relids,
                                Relids incompatible_relids,
                                Relids outer_relids)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • clause: 変換対象の式。
  • is_pushed_down, has_clone, is_clone, pseudoconstant, security_level: 条件の特性を示すフラグおよび属性。
  • required_relids, incompatible_relids, outer_relids: 条件の評価に必要なリレーションや制約情報。
  • **出力**:
  • 生成された RestrictInfo 構造体へのポインタ。

前提条件と不変条件

  • 入力 clause が AND 句であることは許容されません（AND/OR 平坦化処理が完了していることが前提）。
  • OR 句が渡された場合、再帰的に処理されます。

アルゴリズムまたは振る舞い

1. **OR 句の判定**: is_orclause(clause) が真の場合、make_sub_restrictinfos を呼び出し、OR 構造の各サブ句に対して再帰的に RestrictInfo を付与して返します。 2. **AND 句の検証**: is_andclause(clause) が真でないことを Assert で確認します。 3. **ノード生成**: 上記以外の場合、make_plain_restrictinfo を呼び出し、指定された属性を保持する RestrictInfo を生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/optimizer/path/equivclass.c
  • src/backend/optimizer/plan/initsplan.c
  • src/backend/optimizer/util/inherit.c
  • src/backend/optimizer/util/joininfo.c
  • **呼び出し先**:
  • make_sub_restrictinfos (OR 句の場合)
  • make_plain_restrictinfo (通常の場合)

関連データ構造

  • RestrictInfo: 本関数が生成する主要な構造体。
  • PlannerInfo: プランナのコンテキスト情報。
  • Expr: 変換対象となる式ノード。

失敗モードと境界条件

  • AND 句が渡された場合、Assert によりデバッグビルド環境では停止します。
  • OR 句が渡された場合、内部で make_sub_restrictinfos が再帰的に処理を行うため、スタックオーバーフローの可能性は理論上存在しますが、通常のクエリ構造では問題になりません。

保守担当者向けメモ

  • 本関数は RestrictInfo 生成のゲートウェイです。属性の追加や初期化ロジックの変更は、主に make_plain_restrictinfo で行う必要があります。
  • rinfo_serial の付与や遅延評価されるフィールド（mergeopfamilies 等）の扱いは make_plain_restrictinfo に依存しています。
