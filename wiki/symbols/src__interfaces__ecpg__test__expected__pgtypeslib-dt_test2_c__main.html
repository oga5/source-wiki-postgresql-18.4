目的

PostgreSQL の ECPG ライブラリにおける日付・時刻型（pgtypes）の変換関数群の正確性を検証する回帰テストを実行します。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム内部で定義された dates[], times[], intervals[] 配列を使用）。
  • 出力: 標準出力（stdout）へテスト結果を出力し、終了コード 0 を返します。

前提条件と不変条件

  • ECPGdebug が有効化されており、デバッグ情報が stderr に出力されます。
  • PGTYPESchar_free を用いて、変換関数が動的に割り当てた文字列メモリを適切に解放する必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug(1, stderr) を呼び出し、デバッグ出力を有効化します。 2. **基本変換テスト**: タイムスタンプの文字列変換および日付への変換を行い、結果を標準出力に書き出します。 3. **日付パースループ**: dates[] 配列を走査し、PGTYPESdate_from_asc でパースを試みます。パース失敗時は INT_MIN を検出し、エラーフラグを立てます。 4. **時刻結合テスト**: 日付パースが成功した場合、times[] 配列の各要素と結合し、タイムスタンプとしてのパースおよび変換を検証します。 5. **期間演算テスト**: intervals[] 配列を走査し、PGTYPESinterval_from_asc で生成した期間をタイムスタンプに加算します。また、PGTYPESinterval_copy によるメモリコピーの正当性を検証します。 6. **終了**: すべてのテスト終了後、0 を返して終了します。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）。
  • 呼び出し先: ECPGdebug, PGTYPEStimestamp_from_asc, PGTYPEStimestamp_to_asc, PGTYPESchar_free, PGTYPESdate_from_timestamp, PGTYPESdate_new, PGTYPESdate_to_asc, PGTYPESdate_free, PGTYPESdate_from_asc, PGTYPESinterval_from_asc, PGTYPEStimestamp_add_interval, PGTYPESinterval_to_asc, PGTYPESinterval_new, PGTYPESinterval_copy, PGTYPESinterval_free。

関連データ構造

  • date, timestamp, interval: ECPG の日付・時刻型。
  • dates[], times[], intervals[]: テスト用の入力文字列配列。

失敗モードと境界条件

  • PGTYPESdate_from_asc が INT_MIN を返す場合、パースエラーとして処理されます。
  • PGTYPEStimestamp_add_interval が負の値を返す場合、加算処理をスキップします。
  • intervals[] のパースで i1 が NULL の場合、エラーメッセージを出力して次のループへ進みます。

保守担当者向けメモ

  • 本関数はプリプロセッサ処理後のコードであり、#line 指令により元の .pgc ファイルと対応しています。
  • times 変数は libc の関数と衝突しないよう static 宣言されています。
  • dates[] 配列の末尾にある長い文字列は、バッファ境界や最大フィールド数のテストを意図しています。
