GistPageSetDeleted

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/gist.h:206

目的

GiSTインデックスページを削除済みとしてマークし、削除を実行したトランザクションIDをページメタデータに記録します。

シグネチャまたは形状

static inline void GistPageSetDeleted(Page page, FullTransactionId deletexid)

入力と出力

  • **入力**:
  • page: 操作対象のGiSTインデックスページ（Page型）。
  • deletexid: ページ削除を実行したトランザクションのID（FullTransactionId型）。
  • **出力**: なし（副作用としてページ内容を更新）。

前提条件と不変条件

  • page は空（PageIsEmpty(page) が真）である必要があります。

アルゴリズムまたは振る舞い

1. PageIsEmpty(page) を検証し、ページが空であることを確認します。 2. ページ内の GISTPageOpaqueData に F_DELETED フラグをセットします。 3. ページヘッダーの pd_lower を、ページヘッダーサイズと GISTDeletedPageContents の合計サイズに更新します。 4. ページコンテンツ領域に deleteXid を書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistvacuum.c: バキューム処理中のページ削除時に呼び出されます。
  • src/backend/access/gist/gistxlog.c: WALリプレイ時に削除状態を復元するために呼び出されます。
  • **呼び出し先**:
  • PageIsEmpty (マクロ)
  • GistPageGetOpaque (マクロ)
  • PageGetContents (マクロ)

関連データ構造

  • GISTPageOpaqueData: ページ末尾のメタデータ構造。
  • GISTDeletedPageContents: 削除されたページに格納される削除トランザクションIDを保持する構造体。
  • FullTransactionId: 64ビットのトランザクションID。

失敗モードと境界条件

  • ページが空でない状態で呼び出された場合、Assert により異常終了します。

保守担当者向けメモ

  • 本関数はインライン関数であり、パフォーマンスを考慮してヘッダーファイルで定義されています。
  • ページレイアウトの変更（特に GISTDeletedPageContents の配置）を行う際は、GistPageGetDeleteXid との整合性に注意してください。
