pgstat_get_entry_ref_locked

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_shmem.c:719

目的

統計情報エントリへの参照を取得し、その内容を保護するために共有メモリ上のロックを即座に獲得する。

シグネチャまたは形状

PgStat_EntryRef *pgstat_get_entry_ref_locked(PgStat_Kind kind, Oid dboid, uint64 objid, bool nowait)

入力と出力

  • **入力**:
  • kind: 統計情報の種類（PgStat_Kind）。
  • dboid: データベースOID。
  • objid: オブジェクトID。
  • nowait: true の場合、ロックが即座に取得できなければ待機せずに失敗する。
  • **出力**:
  • 成功時: 取得した統計エントリへの参照（PgStat_EntryRef *）。
  • 失敗時: NULL（nowait が true でロックが取得できなかった場合）。

前提条件と不変条件

  • 呼び出し元は、統計システムが初期化されている環境で実行する必要がある。
  • 戻り値の PgStat_EntryRef は、呼び出し元が適切に解放または管理を行う必要がある。

アルゴリズムまたは振る舞い

1. pgstat_get_entry_ref を呼び出し、指定された統計エントリへの参照を取得する（存在しない場合は作成される）。 2. pgstat_lock_entry を呼び出し、取得したエントリに対してロックを試みる。 3. ロックに失敗し、かつ nowait が指定されている場合は NULL を返す。 4. ロックに成功した場合は、エントリへのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/pgstat_backend.c
  • src/backend/utils/activity/pgstat_database.c
  • src/backend/utils/activity/pgstat_relation.c
  • src/backend/utils/activity/pgstat_replslot.c
  • **呼び出し先**:
  • pgstat_get_entry_ref
  • pgstat_lock_entry

関連データ構造

  • PgStat_EntryRef: 共有統計エントリへのローカルな参照構造体。
  • PgStat_Kind: 統計情報のカテゴリを定義する列挙型。

失敗モードと境界条件

  • pgstat_lock_entry がロック獲得に失敗した場合、本関数は NULL を返す。
  • pgstat_get_entry_ref が内部的にエラーを発生させた場合、その例外が伝播する可能性がある。

保守担当者向けメモ

  • 本関数は、統計情報の読み取りや更新を行う前にロックを確実に行うためのラッパーである。
  • ロックの競合を避けるため、nowait フラグの利用が推奨されるケースが多い。
