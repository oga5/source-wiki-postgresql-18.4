xid8_smaller

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xid.c:327

目的

2つの64bitトランザクション識別子（FullTransactionId）を比較し、小さい方の値を返します。

シグネチャまたは形状

Datum xid8_smaller(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの FullTransactionId を受け取ります。
  • **出力**: 小さい方の FullTransactionId を Datum 型として返します。

前提条件と不変条件

  • 入力される2つの値は FullTransactionId 型である必要があります。
  • 比較には FullTransactionIdPrecedes 関数が使用されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_FULLTRANSACTIONID マクロを使用して、第1引数と第2引数をそれぞれ fxid1 と fxid2 として取得します。 2. FullTransactionIdPrecedes(fxid1, fxid2) を呼び出し、fxid1 が fxid2 より先行（小さい）しているか判定します。 3. 先行している場合は fxid1 を、そうでない場合は fxid2 を PG_RETURN_FULLTRANSACTIONID マクロで返します。

呼び出し元と呼び出し先

  • **呼び出し先**: FullTransactionIdPrecedes
  • **呼び出し元**: ソースコード内での直接的な呼び出し元は特定されていません。

関連データ構造

  • FullTransactionId: 64bitトランザクション識別子。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。FullTransactionId の比較ロジックに依存します。

保守担当者向けメモ

  • 本関数はSQLレベルの xid8 型に対する比較演算子として利用されます。
  • 比較ロジックを変更する場合は、FullTransactionIdPrecedes の定義（src/include/access/transam.h 等）を確認してください。
