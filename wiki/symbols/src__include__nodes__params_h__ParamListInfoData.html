ParamListInfoData

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/params.h:110

目的

クエリ実行時に使用される外部パラメータ（PARAM_EXTERN）の集合を保持・管理するためのデータ構造です。静的な配列保持と、フック関数による動的な値取得の両方をサポートします。

シグネチャまたは形状

typedef struct ParamListInfoData
{
    ParamFetchHook paramFetch;
    void       *paramFetchArg;
    ParamCompileHook paramCompile;
    void       *paramCompileArg;
    ParserSetupHook parserSetup;
    void       *parserSetupArg;
    char       *paramValuesStr;
    int         numParams;
    ParamExternData params[FLEXIBLE_ARRAY_MEMBER];
} ParamListInfoData;

入力と出力

  • **入力**: 構造体内のフック関数（paramFetch, paramCompile, parserSetup）および params 配列を介してパラメータ情報を提供します。
  • **出力**: エグゼキュータがクエリ実行時に参照するパラメータ値および型情報を提供します。

前提条件と不変条件

  • params 配列は FLEXIBLE_ARRAY_MEMBER を使用しており、構造体の末尾に配置されます。
  • paramFetch フックが提供されている場合、params 配列の長さはゼロでも許容されます。それ以外の場合、params 配列は numParams 分の長さを確保する必要があります。

アルゴリズムまたは振る舞い

  • **静的アクセス**: params 配列を直接参照して ParamExternData を取得します。
  • **動的アクセス**: paramFetch フックが設定されている場合、エグゼキュータは配列の代わりにこの関数を呼び出し、値を遅延評価または外部から取得します。
  • **コンパイル最適化**: paramCompile フックにより、式評価のコンパイル時に特定のコールバックを埋め込むことが可能です。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/params.c（makeParamList 等による生成）
  • src/include/optimizer/optimizer.h（最適化プロセスでの参照）
  • **呼び出し先**:
  • 提供されたソース情報には、具体的な呼び出し先関数は記載されていません。

関連データ構造

  • ParamExternData: 個々のパラメータの値を保持する構造体。
  • ParamFetchHook, ParamCompileHook, ParserSetupHook: パラメータ処理を拡張するためのコールバック関数型。

失敗モードと境界条件

  • paramFetch が指定されていない場合、params 配列のメモリ確保が不十分だとバッファオーバーランのリスクがあります。
  • numParams と実際の params 配列のサイズが一致していない場合、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 「リスト」という名称ですが、実際には柔軟な配列を持つ構造体です。
  • PARAM_EXEC（実行時内部計算用）ではなく、主にユーザー入力やバインド変数などの PARAM_EXTERN 用途で使用されます。
