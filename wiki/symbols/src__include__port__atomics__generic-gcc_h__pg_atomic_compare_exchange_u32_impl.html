pg_atomic_compare_exchange_u32_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/generic-gcc.h:160

目的

GCC組み込み関数を利用し、32ビット符号なし整数に対するアトミックな比較・交換（CAS: Compare-and-Swap）操作を実行する。

シグネチャまたは形状

static inline bool pg_atomic_compare_exchange_u32_impl(volatile pg_atomic_uint32 *ptr, uint32 *expected, uint32 newval)

入力と出力

  • ptr: 操作対象となる pg_atomic_uint32 構造体へのポインタ（volatile修飾）。
  • expected: 期待値が格納された変数へのポインタ。操作成功時は更新されず、失敗時は現在の値で上書きされる。
  • newval: 比較が成功した場合に書き込まれる新しい値。
  • 戻り値: 比較が成功し、値が更新された場合は true、失敗した場合は false を返す。

前提条件と不変条件

  • ptr はアトミック操作に適したメモリ配置である必要がある。
  • atomics.h を経由してインクルードされる環境（INSIDE_ATOMICS_H が定義されている状態）で使用すること。

アルゴリズムまたは振る舞い

GCCの組み込み関数 __atomic_compare_exchange_n を呼び出す。メモリ順序として __ATOMIC_SEQ_CST（逐次一貫性）を指定し、成功時および失敗時の両方で厳密な順序付けを保証する。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/port/atomics.h および src/include/port/atomics/generic.h 内の各種アトミック操作関数から呼び出される。
  • 呼び出し先: コンパイラ組み込み関数 __atomic_compare_exchange_n を直接呼び出す。

関連データ構造

  • pg_atomic_uint32: 32ビットのアトミック操作対象値を保持する構造体。

失敗モードと境界条件

  • expected が指す値と ptr の現在の値が一致しない場合、操作は失敗し、expected に現在の値が読み込まれる。
  • 戻り値の bool を確認することで、呼び出し側は操作の成否を判定できる。
