pgstat_get_transactional_drops

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_xact.c:271

目的

現在のトランザクションスタックに蓄積された統計情報の変更（作成または削除）を抽出し、コミットまたはアボート処理のためにリスト化して返します。

シグネチャまたは形状

int pgstat_get_transactional_drops(bool isCommit, xl_xact_stats_item **items);

入力と出力

  • **入力**:
  • isCommit (bool): true の場合はコミット時の処理（削除予定のアイテムを抽出）、false の場合はアボート時の処理（作成予定のアイテムを抽出）として動作します。
  • items (xl_xact_stats_item **): 抽出された統計アイテムの配列を格納するためのポインタ。
  • **出力**:
  • 戻り値 (int): 抽出された統計アイテムの個数。

前提条件と不変条件

  • pgStatXactStack が NULL の場合、処理は即座に終了し 0 を返します。
  • isCommit が true の場合、現在のスタックレベルはトップレベル（nest_level == 1 かつ prev == NULL）である必要があります。

アルゴリズムまたは振る舞い

1. 現在のトランザクション状態 (pgStatXactStack) にある pending_drops リストを走査します。 2. isCommit が true の場合、pending->is_create が false である（削除予定の）アイテムを抽出します。 3. isCommit が false の場合、pending->is_create が true である（作成予定の）アイテムを抽出します。 4. 抽出したアイテムを palloc で確保したメモリ領域にコピーし、その個数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/twophase.c (2PC関連処理)
  • src/backend/access/transam/xact.c (トランザクション終了処理)
  • **呼び出し先**:
  • palloc (メモリ確保)
  • dclist_count / dclist_foreach (リスト操作)

関連データ構造

  • PgStat_SubXactStatus: トランザクションの統計状態を保持するスタック構造体。
  • PgStat_PendingDroppedStatsItem: トランザクション内で保留中の統計変更を保持する構造体。
  • xl_xact_stats_item: WALに記録される統計アイテムの形式。

失敗モードと境界条件

  • pgStatXactStack が空の場合、安全に 0 を返します。
  • isCommit が true でありながらサブトランザクションレベルで呼び出された場合、アサーション違反が発生します。

保守担当者向けメモ

  • 本関数は、統計情報の「作成」と「削除」がトランザクションの成否と逆の挙動をとる（コミット時は削除を確定させ、アボート時は作成を無効化する）という統計システムの設計原則に基づいています。
  • items 引数に割り当てられたメモリは、呼び出し元で適切に解放される必要があります。
