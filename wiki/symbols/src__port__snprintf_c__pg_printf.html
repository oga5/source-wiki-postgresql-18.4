pg_printf

定義

  • 種別: function
  • 定義箇所: src/port/snprintf.c:281

目的

PostgreSQL 独自の printf ファミリー実装の一つとして、標準出力 (stdout) へのフォーマット済み出力機能を提供します。

シグネチャまたは形状

int pg_printf(const char *fmt, ...);

入力と出力

  • **入力**:
  • fmt: フォーマット文字列。
  • ...: 可変長引数。
  • **出力**:
  • 戻り値: 出力されたバイト数（エラー発生時は負の値）。

前提条件と不変条件

  • stdout が有効なストリームとして存在している必要があります。
  • C99/POSIX 準拠のフォーマット指定子（%n$ による引数順序制御や %m による strerror(errno) 展開など）をサポートします。

アルゴリズムまたは振る舞い

1. va_start を使用して可変長引数リストを初期化します。 2. pg_vfprintf を呼び出し、stdout に対してフォーマット処理と書き込みを実行します。 3. va_end で引数リストを終了します。 4. pg_vfprintf から返された書き込みバイト数を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/port.h にて printf マクロとして定義されており、PostgreSQL コードベース全体から printf を通じて呼び出される可能性があります。
  • src/pl/plperl/plperl_system.h にて同様に printf マクロとして再定義されています。
  • **呼び出し先**:
  • pg_vfprintf: 実際のフォーマット処理とストリーム出力を担当する内部関数。

関連データ構造

  • PrintfTarget: 出力先と状態を管理する内部構造体（pg_vfprintf 経由で使用）。

失敗モードと境界条件

  • バッファオーバーランや出力エラーが発生した場合、pg_vfprintf の仕様に従い、書き込まれるべきだったバイト数やエラーを示す値が返されます。

保守担当者向けメモ

  • 本関数は標準ライブラリの printf を置き換える目的で #define されているため、このファイル内で標準関数を直接呼び出す際は #undef が必要です。
  • %n$ 指定子を使用する場合、最大 31 引数 (PG_NL_ARGMAX) までの制約があります。
