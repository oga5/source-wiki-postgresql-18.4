system_rows_nextsampletuple

定義

  • 種別: function
  • 定義箇所: contrib/tsm_system_rows/tsm_system_rows.c:272

目的

TABLESAMPLE SYSTEM_ROWS メソッドにおいて、現在のブロック内から次に読み込むべきタプルのオフセット番号を決定し、サンプリング状態を更新する。

シグネチャまたは形状

static OffsetNumber system_rows_nextsampletuple(SampleScanState *node,
                                                BlockNumber blockno,
                                                OffsetNumber maxoffset)

入力と出力

  • **入力**:
  • node: SampleScanState 構造体へのポインタ。サンプリングの進捗状況（donetuples）および状態データ（tsm_state）を保持。
  • blockno: 現在走査中のブロック番号。
  • maxoffset: 現在のページにおける最大オフセット番号。
  • **出力**:
  • 次に読み込むべき OffsetNumber。これ以上読み込むタプルがない場合、または指定された行数に達した場合は InvalidOffsetNumber を返す。

前提条件と不変条件

  • node->tsm_state は SystemRowsSamplerData 型としてキャスト可能であること。
  • node->donetuples が sampler->ntuples に達していない間のみ、有効なオフセットが返される。

アルゴリズムまたは振る舞い

1. **終了判定**: node->donetuples が sampler->ntuples 以上であれば、直ちに InvalidOffsetNumber を返す。 2. **オフセットの更新**:

  • 現在のオフセット（sampler->lt）が InvalidOffsetNumber の場合、ページ内の最初のタプル（FirstOffsetNumber）から開始する。
  • それ以外の場合は、現在のオフセットをインクリメントする。

    • 3. **境界チェック**: 更新後のオフセットが maxoffset を超えた場合、ページ内の走査終了とみなし InvalidOffsetNumber を返す。 4. **状態保存**: 更新したオフセットを sampler->lt に保存し、呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: tsm_system_rows_handler 内で TsmRoutine 構造体の NextSampleTuple フィールドに登録され、PostgreSQL の実行エンジンから呼び出される。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • SampleScanState: スキャン全体の実行状態を保持。
  • SystemRowsSamplerData: SYSTEM_ROWS メソッド固有のサンプリング状態（現在のタプルオフセット lt や目標行数 ntuples など）を保持。

失敗モードと境界条件

  • ページ内にタプルが存在しない場合や、maxoffset が FirstOffsetNumber 未満の場合、適切に InvalidOffsetNumber が返される。
  • 指定された行数（ntuples）に達した時点で、それ以降の呼び出しは常に InvalidOffsetNumber を返す。

保守担当者向けメモ

  • 本関数はページ内のタプルを順次走査する役割を担う。ブロック間の遷移は system_rows_nextsampleblock が担当する。
  • sampler->lt はページ内での現在の位置を保持する重要な状態変数であるため、変更時には整合性に注意すること。
