multirange_get_bounds

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/multirangetypes.c:745

目的

MultirangeType 構造体から、指定されたインデックス i に対応するレンジの境界値（下限および上限）を抽出します。

シグネチャまたは形状

void multirange_get_bounds(TypeCacheEntry *rangetyp,
                           const MultirangeType *multirange,
                           uint32 i,
                           RangeBound *lower,
                           RangeBound *upper)

入力と出力

  • **入力**:
  • rangetyp: レンジ要素の型情報を含むキャッシュエントリ。
  • multirange: 対象のマルチレンジ構造体。
  • i: 抽出対象のレンジインデックス。
  • **出力**:
  • lower: 抽出された下限値が格納される RangeBound 構造体。
  • upper: 抽出された上限値が格納される RangeBound 構造体。

前提条件と不変条件

  • i は multirange->rangeCount 未満である必要があります（Assert で検証）。
  • マルチレンジ内に空のレンジは含まれません（flags & RANGE_EMPTY が 0 であることを Assert で検証）。
  • rangetyp は有効な型キャッシュエントリである必要があります。

アルゴリズムまたは振る舞い

1. multirange_get_bounds_offset を使用して、指定インデックスの境界値が格納されているオフセットを特定します。 2. MultirangeGetFlagsPtr から該当レンジのフラグを取得し、境界値の存在有無や無限大フラグを確認します。 3. MultirangeGetBoundariesPtr を基点に、型ごとのアライメント（typalign）を考慮して境界値のポインタを算出します。 4. RANGE_HAS_LBOUND および RANGE_HAS_UBOUND フラグに基づき、fetch_att を用いて Datum 形式の境界値を抽出します。 5. 抽出した値とフラグを lower および upper 構造体に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/multirangetypes.c 内の複数の箇所で、マルチレンジの先頭や末尾のレンジを取得する際や、特定のインデックスのレンジを走査する際に呼び出されています。
  • **呼び出し先**:
  • multirange_get_bounds_offset
  • MultirangeGetFlagsPtr
  • MultirangeGetBoundariesPtr
  • fetch_att
  • att_addlength_pointer
  • att_align_pointer

関連データ構造

  • MultirangeType: マルチレンジのメモリ上の表現。
  • RangeBound: レンジの境界（値、無限大フラグ、包含フラグ）を保持する構造体。
  • TypeCacheEntry: レンジ要素の型属性（長さ、アライメント等）を保持。

失敗モードと境界条件

  • i が範囲外の場合、Assert によりデバッグビルドで停止します。
  • 空のレンジ（RANGE_EMPTY）が含まれている場合、Assert により停止します。

保守担当者向けメモ

  • 本関数は、マルチレンジの内部データレイアウト（可変長データ）に強く依存しています。レイアウト変更時は MultirangeGetBoundariesPtr 等のマクロと整合性を確認してください。
  • att_align_pointer 等のポインタ演算は、要素型の typalign に依存するため、rangetyp が正しく渡されていることが必須です。
