brin_doinsert

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/brin/brin_pageops.c:341

目的

BRINインデックスの物理ページへ新しいタプルを挿入し、対応するRevmap（ヒープブロックとインデックスの対応表）を更新する。

シグネチャまたは形状

OffsetNumber brin_doinsert(Relation idxrel, BlockNumber pagesPerRange,
                           BrinRevmap *revmap, Buffer *buffer, BlockNumber heapBlk,
                           BrinTuple *tup, Size itemsz)

入力と出力

  • **入力**:
  • idxrel: インデックスのリレーション。
  • pagesPerRange: 範囲あたりのブロック数。
  • revmap: Revmap構造体。
  • buffer: 挿入先バッファ（InvalidBufferも可）。
  • heapBlk: 挿入対象のヒープブロック番号。
  • tup: 挿入するBRINタプル。
  • itemsz: タプルのサイズ。
  • **出力**:
  • 挿入されたタプルの OffsetNumber。

前提条件と不変条件

  • itemsz は MAXALIGN でアライメントされている必要がある。
  • itemsz が BrinMaxItemSize を超える場合はエラーとなる。
  • ページ操作はクリティカルセクション内で実行され、WAL記録が必須となる。

アルゴリズムまたは振る舞い

1. **Revmap拡張**: brinRevmapExtend を呼び出し、対象ヒープブロックをカバーするまでRevmapを拡張する。 2. **バッファ確保**: 指定された buffer が不適切（空き不足）な場合、brin_getinsertbuffer を用いて適切なページを確保する。 3. **ロック取得**: Revmapページを更新用にロックする。 4. **挿入実行**:

  • 新規ページの場合は brin_page_init で初期化。
  • PageAddItem でタプルをページに書き込む。
  • brinSetHeapBlockItemptr でRevmapを更新。

    • 5. **WAL記録**: RelationNeedsWAL が真の場合、XLogInsert を用いて挿入操作を記録する。 6. **後処理**: バッファおよびRevmapのロックを解放し、必要に応じてFSM（空き領域マップ）を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin.c 内の複数箇所（インデックス構築および更新処理）。
  • **呼び出し先**:
  • brinRevmapExtend, brin_getinsertbuffer, brinLockRevmapPageForUpdate, brin_page_init, brinSetHeapBlockItemptr, br_page_get_freespace, PageAddItem, XLogInsert, RecordPageWithFreeSpace, FreeSpaceMapVacuumRange 等。

関連データ構造

  • Relation, BrinRevmap, Buffer, BrinTuple, Page, OffsetNumber。

失敗モードと境界条件

  • itemsz > BrinMaxItemSize: プログラム制限超過エラー。
  • PageAddItem 失敗: ページへの追加失敗時にエラーログを出力。
  • バッファ確保失敗: brin_getinsertbuffer が有効なバッファを返すまでループする。

保守担当者向けメモ

  • ページ操作はクリティカルセクション内で行われるため、エラー発生時の整合性に注意すること。
  • ページ拡張（extended）が発生した場合、FSMの更新と真空処理（FreeSpaceMapVacuumRange）が必須となる。
