RegisterTimeout

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/timeout.c:504

目的

指定されたタイムアウトIDに対して、タイムアウト発生時に実行されるコールバック関数（ハンドラ）を登録する。

シグネチャまたは形状

TimeoutId RegisterTimeout(TimeoutId id, timeout_handler_proc handler)

入力と出力

  • **入力**:
  • id: 登録対象の TimeoutId。USER_TIMEOUT 以上の値を指定した場合、空いているIDを自動的に割り当てる。
  • handler: タイムアウト発生時に呼び出される timeout_handler_proc 型の関数ポインタ。
  • **出力**:
  • 登録に使用された TimeoutId を返す。

前提条件と不変条件

  • all_timeouts_initialized が真であること（初期化済みであること）。
  • 指定された id（または自動割り当てされた id）に対応する all_timeouts エントリの timeout_handler が NULL であること。

アルゴリズムまたは振る舞い

1. all_timeouts_initialized を確認し、未初期化であればアサーションエラーとする。 2. id が USER_TIMEOUT 以上の場合、all_timeouts 配列を走査し、timeout_handler が NULL である最初の空きスロットを探す。

  • 空きが見つからない場合は FATAL エラーで終了する。

    • 3. 指定された（または決定された）id の all_timeouts エントリに handler を格納する。 4. 登録した id を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c
  • src/backend/postmaster/startup.c
  • src/backend/tcop/backend_startup.c
  • src/backend/utils/init/postinit.c
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わないが、ereport マクロを使用してエラー報告を行う。

関連データ構造

  • TimeoutId: タイムアウトの種類を識別する列挙型。
  • all_timeouts: タイムアウト設定を保持する静的配列。

失敗モードと境界条件

  • USER_TIMEOUT 以上のIDを要求したが、利用可能な空きスロットが存在しない場合、ERRCODE_CONFIGURATION_LIMIT_EXCEEDED を伴う FATAL エラーが発生する。
  • 既にハンドラが登録されているIDに対して登録を試みた場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数はシグナルハンドラの無効化を行わない。
  • ユーザー定義のタイムアウトIDを動的に割り当てる機能を持つため、MAX_TIMEOUTS の上限に注意すること。
