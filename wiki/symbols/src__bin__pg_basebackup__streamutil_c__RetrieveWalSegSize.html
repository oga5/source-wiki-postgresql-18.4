RetrieveWalSegSize

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/streamutil.c:275

目的

接続先のPostgreSQLサーバーからWALセグメントサイズを取得し、グローバル変数 WalSegSz を設定する。

シグネチャまたは形状

bool RetrieveWalSegSize(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn（確立済みのデータベース接続）
  • **出力**: 成功時に true、失敗時に false を返す。
  • **副作用**: 成功時、グローバル変数 WalSegSz にサーバーから取得したバイト単位のWALセグメントサイズが格納される。

前提条件と不変条件

  • conn は NULL であってはならない（Assertによる検証）。
  • 取得された WalSegSz は、1MBから1GBの範囲内の2のべき乗である必要がある。

アルゴリズムまたは振る舞い

1. サーバーバージョンが MINIMUM_VERSION_FOR_SHOW_CMD 未満の場合、デフォルト値 DEFAULT_XLOG_SEG_SIZE を WalSegSz に設定して終了する。 2. SHOW wal_segment_size コマンドを実行し、結果を取得する。 3. 取得した文字列（例: "16MB"）を数値と単位にパースする。 4. 単位（MB, GB）に基づき数値をバイト単位に変換し、WalSegSz に代入する。 5. IsValidWalSegSize マクロを用いて、設定された値が妥当な範囲内か検証する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_basebackup/pg_receivewal.c
  • **呼び出し先**:
  • PQexec, PQresultStatus, PQntuples, PQnfields, PQgetvalue, PQclear, PQerrorMessage, PQserverVersion (libpq)
  • pg_log_error, pg_log_error_detail (共通ログ出力)
  • sscanf, strcmp (標準Cライブラリ)
  • IsValidWalSegSize (マクロ)

関連データ構造

  • PGconn: libpqの接続ハンドル。
  • WalSegSz: 本関数が値を設定するグローバル変数。

失敗モードと境界条件

  • 接続が確立されていない場合、アサーションエラーが発生する。
  • サーバーが SHOW wal_segment_size をサポートしていない古いバージョンの場合、デフォルト値が使用される。
  • クエリ実行失敗、パース失敗、または取得した値が妥当な範囲（1MB〜1GBの2のべき乗）外である場合、エラーログを出力して false を返す。

保守担当者向けメモ

  • 本関数はフロントエンド用ユーティリティであり、バックエンドの内部関数には依存しない。
  • 新しいWALセグメントサイズの制約が導入された場合は、IsValidWalSegSize の定義と併せて確認すること。
