PLy_get_error_data

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_elog.c:429

目的

Python 例外オブジェクトから、PostgreSQL のエラー報告に必要なメタデータ（SQLSTATE コードおよびエラー詳細情報）を抽出する。

シグネチャまたは形状

static void PLy_get_error_data(PyObject *exc, int *sqlerrcode, char **detail, char **hint,
                               char **schema_name, char **table_name, char **column_name,
                               char **datatype_name, char **constraint_name)

入力と出力

  • **入力**:
  • exc: 情報を抽出する対象の Python 例外オブジェクト。
  • 各 char ** 引数: 抽出した文字列を格納するためのポインタ。
  • **出力**:
  • sqlerrcode: 抽出された SQLSTATE コードが格納される整数ポインタ。
  • 各 char ** 引数: 成功時に対応する属性値（文字列）が格納される。

前提条件と不変条件

  • exc は有効な Python オブジェクトである必要がある。
  • 抽出される各属性は、Python 例外オブジェクトが保持する属性名（"detail", "hint" 等）に対応している必要がある。

アルゴリズムまたは振る舞い

1. PLy_get_sqlerrcode を呼び出し、exc から SQLSTATE コードを取得して sqlerrcode に格納する。 2. get_string_attr を順次呼び出し、以下の属性を exc から取得して各引数に格納する。

  • detail, hint, schema_name, table_name, column_name, datatype_name, constraint_name

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_elog.c 内の関数（行86など）から呼び出されている。
  • **呼び出し先**:
  • PLy_get_sqlerrcode
  • get_string_attr

関連データ構造

  • PyObject (Python C API)

失敗モードと境界条件

  • exc が指定された属性を持たない場合、get_string_attr の内部実装に依存する（通常は NULL が返される）。

保守担当者向けメモ

  • 本関数は plpy_elog.c 内のプライベートなヘルパー関数である。
  • 属性の追加や変更が必要な場合は、get_string_attr の呼び出しリストを更新すること。
