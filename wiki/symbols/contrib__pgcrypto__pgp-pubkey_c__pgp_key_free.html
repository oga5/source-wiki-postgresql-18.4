pgp_key_free

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pubkey.c:47

目的

PGP_PubKey 構造体が保持する多倍長整数（MPI）メモリを解放し、構造体自体をゼロクリアした上でメモリを解放する。

シグネチャまたは形状

void pgp_key_free(PGP_PubKey *pk)

入力と出力

  • **入力**: PGP_PubKey *pk（解放対象の鍵構造体へのポインタ）
  • **出力**: なし

前提条件と不変条件

  • pk が NULL の場合、関数は何もせず直ちに復帰する。
  • pk->algo に基づき、対応するアルゴリズム（ElGamal, RSA, DSA）の MPI メンバが適切に初期化されている必要がある。

アルゴリズムまたは振る舞い

1. pk が NULL でないことを確認する。 2. pk->algo の値に応じて switch 文を実行し、アルゴリズム固有の MPI メンバに対して pgp_mpi_free を呼び出す。

  • **ElGamal**: p, g, y, x を解放。
  • **RSA**: n, e, d, p, q, u を解放。
  • **DSA**: p, q, g, y, x を解放。

    • 3. px_memset を使用して pk 構造体全体をゼロで上書きする。 4. pfree(pk) を呼び出し、構造体メモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-info.c:66
  • contrib/pgcrypto/pgp-pubkey.c:241, 450, 539, 553
  • contrib/pgcrypto/pgp.c:216
  • **呼び出し先**:
  • pgp_mpi_free（各 MPI メンバの解放）
  • px_memset（メモリのゼロクリア）
  • pfree（構造体メモリの解放）

関連データ構造

  • PGP_PubKey: 鍵情報を保持する主要な構造体。

失敗モードと境界条件

  • pk が指すメモリが既に解放済みである場合、二重解放によるクラッシュが発生する可能性がある。
  • pk->algo が未知の値である場合、MPI の解放処理がスキップされ、メモリリークが発生する可能性がある。

保守担当者向けメモ

  • 本関数はアルゴリズムごとに解放すべき MPI メンバが厳密に定義されている。新しいアルゴリズムを PGP_PubKey に追加する際は、必ず本関数の switch 文に解放処理を追加すること。
