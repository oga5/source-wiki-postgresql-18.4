MergeAppend

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/plannodes.h:369

目的

複数の子プラン（mergeplans）から出力されるソート済みのタプルを、指定されたソートキーに基づいてマージし、単一のソート済みストリームとして出力するための実行計画ノードです。

シグネチャまたは形状

typedef struct MergeAppend {
    Plan        plan;
    Bitmapset  *apprelids;
    List       *mergeplans;
    int         numCols;
    AttrNumber *sortColIdx;
    Oid        *sortOperators;
    Oid        *collations;
    bool       *nullsFirst;
    int         part_prune_index;
} MergeAppend;

入力と出力

  • **入力**: mergeplans リストに含まれる複数の子プラン。各子プランは、sortColIdx 等で指定された条件でソート済みである必要があります。
  • **出力**: 子プランからの入力をマージした、単一のソート済みタプルストリーム。

前提条件と不変条件

  • plan メンバが構造体の先頭に配置されており、Plan 構造体へのキャストが可能です。
  • sortColIdx, sortOperators, collations, nullsFirst はすべて numCols の長さを持つ配列です。
  • part_prune_index は、実行時パーティションプルーニングが不要な場合 -1 となります。

アルゴリズムまたは振る舞い

1. 子ノードから提供される複数のソート済み入力を受け取ります。 2. sortColIdx および関連するソート演算子（sortOperators）を使用して、各入力ストリームの先頭要素を比較します。 3. 最小（または最大）の要素を順次選択して出力し、対応する子ノードから次のタプルを取得します。 4. 実行時パーティションプルーニングが有効な場合、part_prune_index を使用して不要な子プランを動的に除外します。

呼び出し元と呼び出し先

  • **呼び出し元**: プランナ（src/backend/optimizer/）がこのノードを生成します。
  • **呼び出し先**: エグゼキュータ（src/backend/executor/）がこのノードの実行状態を管理し、子ノードの実行関数を呼び出します。
  • **参照元**: src/backend/commands/explain.c が実行計画の表示（EXPLAIN）のために本構造体を参照しています。

関連データ構造

  • Plan: 基底構造体。
  • Bitmapset: apprelids の管理に使用。
  • PlannedStmt: クエリ実行計画のルート。part_prune_index はこれに含まれる情報と関連付けられます。

失敗モードと境界条件

  • numCols が 0 の場合、ソート順序が定義されず、マージの振る舞いは未定義となります。
  • part_prune_index が不適切なインデックスを指している場合、実行時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は Plan を継承しているため、Plan 構造体の変更が影響します。
  • sortColIdx 等の配列メンバは numCols に依存しているため、これらを操作する際は必ず整合性を確認してください。
  • EXPLAIN 出力の実装（explain.c）で IsA(plan, MergeAppend) による型チェックが行われているため、ノードの追加・変更時には当該箇所への追従が必要です。
