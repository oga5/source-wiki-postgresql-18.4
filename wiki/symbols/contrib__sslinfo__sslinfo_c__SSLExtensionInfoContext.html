SSLExtensionInfoContext

定義

  • 種別: struct
  • 定義箇所: contrib/sslinfo/sslinfo.c:33

目的

SQL関数 ssl_extension_info が、セットを返す関数（SRF: Set-Returning Function）として複数回の呼び出し間で状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct
{
	TupleDesc	tupdesc;
} SSLExtensionInfoContext;

入力と出力

  • **入力**: 構造体内部の tupdesc（タプル記述子）は、SRFが返す行の構造を定義するために使用されます。
  • **出力**: なし（コンテキスト保持用構造体）。

前提条件と不変条件

  • ssl_extension_info 関数の初回呼び出し時に palloc を用いてメモリが割り当てられる必要があります。
  • tupdesc は、SRFが結果を生成する際に適切なタプル形式を維持するために使用されます。

アルゴリズムまたは振る舞い

1. ssl_extension_info 関数が SRF_IS_FIRSTCALL() を確認した際、この構造体をメモリ上に確保します。 2. 複数回の呼び出しを通じて、この構造体は tupdesc を保持し、各行の生成時に参照されます。 3. SRF_RETURN_NEXT() を用いた行生成プロセスにおいて、状態維持の基盤として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sslinfo/sslinfo.c 内の ssl_extension_info 関数。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • TupleDesc: PostgreSQLのタプル記述子。本構造体内で保持され、SRFの出力形式を定義します。

失敗モードと境界条件

  • メモリ不足時: palloc によるメモリ確保が失敗した場合、PostgreSQLの標準的なエラーハンドリングによりトランザクションが中断されます。

保守担当者向けメモ

  • 本構造体は ssl_extension_info の実行状態を管理する目的でのみ使用されています。
  • SRFの実装において、メモリコンテキストの切り替え（MemoryContextSwitchTo）が必要な場合があるため、本構造体を利用する関数内のメモリ管理には注意してください。
