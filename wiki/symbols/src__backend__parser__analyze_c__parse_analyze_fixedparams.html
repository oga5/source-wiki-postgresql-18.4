parse_analyze_fixedparams

定義

  • 種別: function
  • 定義箇所: src/backend/parser/analyze.c:104

目的

生の解析ツリー（RawStmt）を、オプティマイザーやエグゼキューターが処理可能なクエリツリー（Query）へ変換する。特に、パラメーターの型が事前に確定している場合に使用されるエントリーポイントである。

シグネチャまたは形状

Query *parse_analyze_fixedparams(RawStmt *parseTree, const char *sourceText,
                                 const Oid *paramTypes, int numParams,
                                 QueryEnvironment *queryEnv)

入力と出力

  • **入力**:
  • parseTree: 解析対象の RawStmt。
  • sourceText: SQL 文のソース文字列（必須）。
  • paramTypes: パラメーターの型（OID）の配列。
  • numParams: パラメーター数。
  • queryEnv: クエリ実行環境。
  • **出力**: 変換済みの Query 構造体へのポインタ。

前提条件と不変条件

  • sourceText は NULL であってはならない。
  • numParams > 0 の場合、setup_parse_fixed_parameters を介してパラメーター型が ParseState に設定される。

アルゴリズムまたは振る舞い

1. make_parsestate を呼び出し、解析状態（ParseState）を初期化する。 2. sourceText を pstate に格納する。 3. パラメーターが存在する場合、setup_parse_fixed_parameters で型情報を登録する。 4. transformTopLevelStmt を呼び出し、解析ツリーをクエリツリーへ変換する。 5. クエリ ID の生成が有効な場合、JumbleQuery を実行する。 6. post_parse_analyze_hook が定義されていれば実行する。 7. free_parsestate で ParseState を解放し、クエリ ID を統計情報として報告した後、Query を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/view.c:375
  • src/backend/tcop/postgres.c:683
  • **呼び出し先**:
  • make_parsestate
  • setup_parse_fixed_parameters
  • transformTopLevelStmt
  • JumbleQuery
  • free_parsestate
  • pgstat_report_query_id

関連データ構造

  • RawStmt: 入力となる解析ツリー。
  • Query: 出力となるクエリツリー。
  • ParseState: 解析中に保持される状態管理構造体。
  • QueryEnvironment: クエリ実行環境。

失敗モードと境界条件

  • sourceText が NULL の場合、Assert により停止する。
  • transformTopLevelStmt 内部でセマンティックエラー（型不一致や権限不足など）が発生した場合、例外がスローされる。

保守担当者向けメモ

  • 本関数は parse_analyze_varparams と対をなす。パラメーターの型解決戦略が異なるため、修正時は両者の差異に留意すること。
  • post_parse_analyze_hook を介した外部プラグインの介入が可能である点に注意。
