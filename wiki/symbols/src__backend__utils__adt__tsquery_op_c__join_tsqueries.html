join_tsqueries

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_op.c:32

目的

2つの TSQuery を指定された演算子で結合し、操作用の QTNode ツリー構造を構築する内部ヘルパー関数です。

シグネチャまたは形状

static QTNode *join_tsqueries(TSQuery a, TSQuery b, int8 operator, uint16 distance)

入力と出力

  • **入力**:
  • a, b: 結合対象の TSQuery 型データ。
  • operator: 適用する演算子（OP_AND, OP_OR, OP_PHRASE 等）。
  • distance: OP_PHRASE 演算子使用時の単語間距離。
  • **出力**:
  • 結合されたノードを根とする QTNode 構造体へのポインタ。

前提条件と不変条件

  • 戻り値の QTNode には QTN_NEEDFREE フラグが設定され、メモリ解放が必要であることを示します。
  • a と b は QT2QTN を介してツリー構造へ変換されます。

アルゴリズムまたは振る舞い

1. palloc0 を使用して QTNode を確保し、QTN_NEEDFREE フラグを付与します。 2. 演算子情報を保持する QueryItem を作成し、res->valnode に格納します。OP_PHRASE の場合は distance を設定します。 3. res->child 配列を確保し、b を左の子（child[0]）、a を右の子（child[1]）として QT2QTN で変換したツリーを割り当てます。 4. 構築した QTNode ツリーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery_op.c 内の論理演算子実装関数（OP_AND, OP_OR, OP_PHRASE を用いる箇所）。
  • **呼び出し先**:
  • QT2QTN: TSQuery を QTNode ツリーへ変換するために呼び出されます。
  • palloc0: メモリ確保のために呼び出されます。

関連データ構造

  • QTNode: tsquery 操作用の中間ツリー構造。
  • TSQuery: シリアライズされた全文検索クエリ形式。
  • QueryItem: tsquery 内の個々の演算子や単語を表現する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • a または b が無効な TSQuery 構造である場合、QT2QTN 内部で予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は QTN_NEEDFREE を設定するため、呼び出し側で適切に QTNFree を呼び出さないとメモリリークが発生します。
  • child[0] に b、child[1] に a が割り当てられる順序は、演算子の結合規則に依存するため変更時は注意してください。
