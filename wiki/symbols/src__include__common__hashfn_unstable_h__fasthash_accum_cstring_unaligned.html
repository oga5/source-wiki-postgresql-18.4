fasthash_accum_cstring_unaligned

定義

  • 種別: function
  • 定義箇所: src/include/common/hashfn_unstable.h:223

目的

NUL終端文字列を効率的に処理し、ハッシュ状態へ段階的に蓄積します。アライメントを考慮しない環境やケースにおいて、文字列をチャンク単位で読み込み、ハッシュ計算へ渡す役割を担います。

シグネチャまたは形状

static inline size_t fasthash_accum_cstring_unaligned(fasthash_state *hs, const char *str)

入力と出力

  • **入力**:
  • hs: ハッシュ計算の状態を保持する fasthash_state 構造体へのポインタ。
  • str: ハッシュ化対象のNUL終端文字列。
  • **出力**:
  • 処理した文字列の総バイト数（size_t 型）を返します。

前提条件と不変条件

  • str は有効なNUL終端文字列である必要があります。
  • 本関数は「不安定」なハッシュアルゴリズムの一部であり、プラットフォームやバージョン間で結果が一致する保証はありません。永続化データには使用しないでください。

アルゴリズムまたは振る舞い

1. 文字列の開始位置を記録します。 2. 文字列の終端（\0）に達するまでループを実行します。 3. 内部ループにて、FH_SIZEOF_ACCUM バイトまたはNUL文字に達するまでのチャンク長を計算します。 4. fasthash_accum を呼び出し、現在のチャンクをハッシュ状態に混合します。 5. 文字列ポインタをチャンク分進めます。 6. 最後に、処理した総バイト数（開始位置からの差分）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/common/hashfn_unstable.h 内の複数の箇所から呼び出されています（行279, 299, 314）。
  • **呼び出し先**: fasthash_accum を呼び出します。

関連データ構造

  • fasthash_state: ハッシュ計算の進行状況を保持する構造体。

失敗モードと境界条件

  • 空文字列（""）が渡された場合、ループは実行されず、戻り値として 0 を返します。
  • 文字列が FH_SIZEOF_ACCUM より短い場合、その長さ分だけを一度に処理します。

保守担当者向けメモ

  • 本関数はアライメントを考慮しない実装です。アライメントが保証されている場合は fasthash_accum_cstring_aligned の使用を検討してください。
  • 入力文字列の長さは、最終的なハッシュ計算の「tweak」として利用されるため、正確なバイト数を計算して返すことが重要です。
