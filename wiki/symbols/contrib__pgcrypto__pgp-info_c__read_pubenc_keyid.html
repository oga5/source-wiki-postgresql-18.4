read_pubenc_keyid

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-info.c:70

目的

PGP 公開鍵暗号化セッション鍵（Public-Key Encrypted Session Key）パケットから、暗号化に使用された鍵の識別子（Key ID）を抽出します。

シグネチャまたは形状

static int read_pubenc_keyid(PullFilter *pkt, uint8 *keyid_buf);

入力と出力

  • pkt: 読み込み対象の PGP パケットストリーム（PullFilter）。
  • keyid_buf: 抽出された 8 バイトの Key ID を格納するバッファ。
  • 戻り値: 成功時は pgp_skip_packet の結果（通常 0）、バージョン不一致時は -1、読み込み失敗時はエラーコードを返します。

前提条件と不変条件

  • keyid_buf は少なくとも 8 バイトの領域を確保している必要があります。
  • パケットの先頭バイトがバージョン番号として読み込まれます。

アルゴリズムまたは振る舞い

1. パケットから 1 バイトのバージョン番号を読み取ります。 2. バージョンが 3 でない場合、処理を中断し -1 を返します。 3. パケットから 8 バイトの Key ID を keyid_buf に読み込みます。 4. 読み込み成功後、pgp_skip_packet を呼び出してパケットの残りをスキップし、その結果を返します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/pgp-info.c 内のパケット解析ループ（163行目付近）。
  • 呼び出し先: GETBYTE マクロ、pullf_read_fixed、pgp_skip_packet。

関連データ構造

  • PullFilter: PGP パケットストリームを抽象化する構造体。

失敗モードと境界条件

  • バージョン番号が 3 以外の場合、エラーとして -1 を返します。
  • pullf_read_fixed が失敗した場合、そのエラーコードをそのまま伝播させます。

保守担当者向けメモ

  • 本関数は PGP バージョン 3 のパケット形式に特化しています。
  • pgp_skip_packet を呼び出すことで、パケットの末尾までストリームを正しく進める設計になっています。
