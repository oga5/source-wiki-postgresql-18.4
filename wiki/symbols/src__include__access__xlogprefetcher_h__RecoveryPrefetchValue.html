RecoveryPrefetchValue

定義

  • 種別: enum
  • 定義箇所: src/include/access/xlogprefetcher.h:24

目的

PostgreSQLのリカバリ処理（WALリプレイ）における、データブロックの先読み（Prefetching）機能の有効化状態を制御するために使用されます。

シグネチャまたは形状

typedef enum
{
    RECOVERY_PREFETCH_OFF,
    RECOVERY_PREFETCH_ON,
    RECOVERY_PREFETCH_TRY,
} RecoveryPrefetchValue;

入力と出力

  • 本シンボル自体は設定値の列挙型であり、関数ではありません。
  • GUCパラメータ（recovery_prefetch）の値として利用され、先読みロジックの挙動を決定します。

前提条件と不変条件

  • RECOVERY_PREFETCH_OFF が選択されている場合、先読み処理は実行されません。
  • 先読み機能は、主に PrefetchBuffer() がサポートされている環境（Linux等）で効果を発揮します。

アルゴリズムまたは振る舞い

  • RECOVERY_PREFETCH_OFF: 先読みを無効化します。
  • RECOVERY_PREFETCH_ON: 先読みを強制的に有効化します。
  • RECOVERY_PREFETCH_TRY: 環境や条件に応じて先読みを試行します。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的なコードパスは、提供されたソース情報からは特定できません。

関連データ構造

  • XLogPrefetcher: この列挙型の設定値に基づいて先読み状態を管理する構造体。

失敗モードと境界条件

  • 提供されたソース情報には、本シンボルに関連する失敗モードや境界条件の記述はありません。

保守担当者向けメモ

  • リカバリのパフォーマンスチューニングを行う際は、本設定と maintenance_io_concurrency の関連性に注意してください。
  • XLogPrefetcherReadRecord() などの関数内で、この列挙値がどのように参照されているかを確認してください。
