rbt_find_less

定義

  • 種別: function
  • 定義箇所: src/backend/lib/rbtree.c:202

目的

赤黒木（RBTree）内において、指定されたキーより小さい値を持つノード、または指定されたキーと等しいノードを検索します。

シグネチャまたは形状

RBTNode *rbt_find_less(RBTree *rbt, const RBTNode *data, bool equal_match)

入力と出力

  • **入力**:
  • rbt: 検索対象の赤黒木構造体へのポインタ。
  • data: 検索キーとして使用するノードデータ。
  • equal_match: true の場合、キーが一致するノードを優先的に返します。
  • **出力**:
  • 条件に合致する RBTNode へのポインタ。該当するノードがない場合は NULL を返します。

前提条件と不変条件

  • rbt は初期化済みであること。
  • rbt->comparator コールバック関数が正しく設定されていること。
  • 比較ロジックは、data と木内のノードを比較し、負、ゼロ、正の値を返す必要がある。

アルゴリズムまたは振る舞い

1. 木のルートから走査を開始します。 2. rbt->comparator を用いて、現在のノードと data を比較します。 3. equal_match が true で比較結果が 0（一致）の場合、そのノードを即座に返します。 4. 比較結果が正（data > node）の場合、現在のノードを「候補（lesser）」として保持し、より大きい値を探すために右の子へ移動します。 5. 比較結果が負の場合、左の子へ移動します。 6. RBTNIL（番兵）に到達した時点で、最後に保持していた lesser を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_rbtree/test_rbtree.c にて、テストおよび検証目的で複数回呼び出されています。
  • **呼び出し先**: rbt->comparator（RBTree 構造体に登録された比較関数）。

関連データ構造

  • RBTree: 木全体を管理する制御構造体。
  • RBTNode: 木の各ノードのヘッダー。
  • RBTNIL: 葉ノードを表す静的な番兵。

失敗モードと境界条件

  • 木が空の場合、NULL を返します。
  • 指定されたキーより小さい値が存在しない場合、NULL を返します。
  • equal_match が false の場合、キーが完全に一致するノードがあっても、それより小さいノードを探索し続けます。

保守担当者向けメモ

  • 本関数は侵入型設計に基づいています。data 引数は RBTNode を含む構造体としてキャストして利用されることを想定しています。
  • RBTNIL を使用しているため、NULLポインタチェックの代わりに node != RBTNIL を使用する実装となっています。
