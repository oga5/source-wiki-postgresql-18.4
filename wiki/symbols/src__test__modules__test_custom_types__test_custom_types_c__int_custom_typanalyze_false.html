int_custom_typanalyze_false

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_custom_types/test_custom_types.c:72

目的

PostgreSQLのカスタム型に対するANALYZE処理において、統計情報収集をサポートしない（拒否する）挙動をシミュレートするために使用されます。

シグネチャまたは形状

Datum int_custom_typanalyze_false(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（標準的なPostgreSQL関数呼び出し引数）
  • 出力: Datum型としてfalse（bool値）を返します。

前提条件と不変条件

  • 本関数はPostgreSQLの関数マネージャインターフェース（fmgr.h）に従って実装されています。
  • 常にfalseを返すことが不変条件です。

アルゴリズムまたは振る舞い

1. PG_RETURN_BOOL(false)を実行し、呼び出し元に対して統計情報の収集が不可能であることを通知します。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLの統計情報収集サブシステム（ANALYZE実行時）。直接的な呼び出し箇所はソースコード内に明示されていませんが、PG_FUNCTION_INFO_V1(int_custom_typanalyze_false)により関数として登録されています。
  • 呼び出し先: なし。

関連データ構造

  • PG_FUNCTION_ARGS: 関数呼び出しの引数コンテキスト。

失敗モードと境界条件

  • 本関数自体は常にfalseを返すため、関数内部での失敗モードはありません。この関数がfalseを返すこと自体が、統計情報収集プロセスにおける「意図的な失敗（サポート外）」として機能します。

保守担当者向けメモ

  • 本関数はテスト専用です。実運用環境の型定義には使用しないでください。
  • ANALYZE処理がこの関数を呼び出した際、統計情報が収集されない挙動を検証するために使用されます。
