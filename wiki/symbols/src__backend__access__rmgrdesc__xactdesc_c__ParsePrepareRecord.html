ParsePrepareRecord

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/xactdesc.c:238

目的

トランザクションのプリペア（Prepare）に関連するWALレコードのバイナリデータを解析し、人間が読み取り可能な形式へ変換するための構造体 xl_xact_parsed_prepare に情報を展開する。

シグネチャまたは形状

void ParsePrepareRecord(uint8 info, xl_xact_prepare *xlrec, xl_xact_parsed_prepare *parsed)

入力と出力

  • **入力**:
  • info: WALレコードの操作種別を示すビットフラグ。
  • xlrec: 解析対象となる xl_xact_prepare 構造体へのポインタ。
  • **出力**:
  • parsed: 解析結果が格納される xl_xact_parsed_prepare 構造体。

前提条件と不変条件

  • xlrec は有効なメモリ領域を指していること。
  • parsed は memset によりゼロクリアされた状態で初期化される。
  • WALレコード内の可変長データ（GID、サブトランザクションID、リレーションロケータ、統計情報、無効化メッセージ）は、xlrec の直後に配置されている必要がある。

アルゴリズムまたは振る舞い

1. xlrec の直後のメモリ位置を bufptr として取得する。 2. parsed 構造体をゼロクリアする。 3. xlrec から固定長のフィールド（タイムスタンプ、XID、各種カウント数など）を parsed にコピーする。 4. bufptr を使用して、可変長データ（GID文字列、サブトランザクションID配列、リレーションロケータ配列、統計情報配列、無効化メッセージ配列）を順次読み取り、parsed 内の各ポインタへ割り当てる。 5. 各データ読み取り後、MAXALIGN を用いて bufptr を次のデータ境界まで進める。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/rmgrdesc/xactdesc.c:401
  • src/backend/replication/logical/decode.c:318
  • **呼び出し先**:
  • 直接的な関数呼び出しは含まれないが、MAXALIGN マクロおよび memset, strncpy 標準ライブラリ関数を利用する。

関連データ構造

  • xl_xact_prepare: WALに書き込まれるプリペアレコードの物理構造。
  • xl_xact_parsed_prepare: 解析後の情報を保持する構造体。
  • RelFileLocator, TransactionId, SharedInvalidationMessage: WALレコード内に含まれる各要素の型。

失敗モードと境界条件

  • xlrec が指すメモリ領域が不十分な場合、バッファオーバーランの可能性がある（本関数内では境界チェックを行わないため、呼び出し元でレコードの整合性を保証する必要がある）。
  • MAXALIGN によるアライメント計算が、プラットフォーム固有のメモリ配置と一致しない場合、データ読み取りが不正になる可能性がある。

保守担当者向けメモ

  • 本関数はバックエンドとフロントエンド（pg_waldump）で共有されるため、構造体の定義やパースロジックを変更する際は、両方の互換性に注意すること。
  • 可変長データの順序は xl_xact_prepare の構造と密接に結びついているため、WALフォーマットの変更時には必ず本関数のポインタ加算ロジックを更新すること。
