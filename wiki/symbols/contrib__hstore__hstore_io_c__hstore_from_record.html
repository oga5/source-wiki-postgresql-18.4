hstore_from_record

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_io.c:834

目的

PostgreSQL のレコード型（record）を hstore 型に変換する。レコードの各フィールド名をキー、値を文字列化したものを値としてマッピングする。

シグネチャまたは形状

Datum hstore_from_record(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取るレコード型データ。
  • **出力**: Datum 型として返される HStore 構造体へのポインタ。

前提条件と不変条件

  • 入力が NULL の場合、型情報のみを使用して空の hstore を生成する。
  • レコード型が不明な場合、lookup_rowtype_tupdesc_domain がエラーを発生させる。
  • fn_extra を使用して、レコードの型情報と列ごとの I/O 関数情報をキャッシュし、呼び出し間で再利用する。

アルゴリズムまたは振る舞い

1. **型情報の取得**: 入力レコードから TupleDesc を取得する。 2. **キャッシュ管理**: fcinfo->flinfo->fn_extra を確認し、レコードの型が変更されていない場合は既存の RecordIOData を再利用する。 3. **タプルの分解**: heap_deform_tuple を使用して、レコードを各フィールドの Datum と NULL フラグ配列に分解する。 4. **変換処理**:

  • 各列をループし、列名（attname）を hstore のキーとして抽出する。
  • 値が NULL の場合は isnull = true として処理する。
  • 値が存在する場合は、対応する型の出力関数（OutputFunctionCall）を呼び出し、文字列に変換する。

    • 5. **構築**: hstoreUniquePairs で重複キーを排除し、hstorePairs で最終的な HStore 構造体を生成する。 6. **後処理**: ReleaseTupleDesc を呼び出し、メモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に直接的な呼び出し元は明示されていない。
  • **呼び出し先**:
  • lookup_rowtype_tupdesc_domain, heap_deform_tuple, OutputFunctionCall, hstoreUniquePairs, hstorePairs 等の PostgreSQL 内部関数。

関連データ構造

  • HStore: 生成されるデータ構造。
  • Pairs: 変換中の中間データ保持用配列。
  • RecordIOData: レコードの型情報および列ごとの I/O 情報を保持するキャッシュ構造体。

失敗モードと境界条件

  • **Dropped Columns**: attisdropped が真である列は無視される。
  • **型不一致**: キャッシュされた型情報と現在のレコード型が異なる場合、キャッシュが再構築される。
  • **メモリ制限**: palloc によるメモリ確保が失敗した場合は PostgreSQL の標準的なエラー処理に従う。

保守担当者向けメモ

  • 本関数は record_out の実装を参考にしている。
  • hstoreCheckKeyLen および hstoreCheckValLen による長さ制限チェックが適用される。
  • fn_extra のメモリコンテキストは fcinfo->flinfo->fn_mcxt を使用しており、呼び出し間で安全に保持される。
