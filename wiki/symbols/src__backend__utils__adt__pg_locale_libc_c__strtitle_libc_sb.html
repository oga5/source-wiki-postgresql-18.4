strtitle_libc_sb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_libc.c:237

目的

シングルバイトエンコーディング環境において、指定されたロケールに基づき文字列をタイトルケース（単語の先頭を大文字、それ以外を小文字）に変換する。

シグネチャまたは形状

static size_t strtitle_libc_sb(char *dest, size_t destsize, const char *src, ssize_t srclen, pg_locale_t locale)

入力と出力

  • **入力**:
  • dest: 変換結果を格納するバッファ。
  • destsize: dest バッファのサイズ。
  • src: 変換対象の文字列。
  • srclen: src の長さ（負の場合は strlen(src) が使用される）。
  • locale: ロケール情報を含む pg_locale_t 構造体。
  • **出力**: 変換後の文字列長（srclen を返す）。

前提条件と不変条件

  • srclen + 1 <= destsize を満たす場合のみ変換が実行される。
  • locale が指す構造体は有効である必要がある。

アルゴリズムまたは振る舞い

1. srclen が負の場合、strlen(src) で長さを確定する。 2. バッファサイズが十分な場合、src を dest にコピーし、NUL終端する。 3. 文字列を走査し、以下のロジックで変換を行う：

  • locale->is_default が真の場合: pg_toupper / pg_tolower を使用（ASCIIベースの伝統的なPostgres挙動）。
  • それ以外の場合: toupper_l / tolower_l を使用（指定ロケールの定義に従う）。
  • 直前の文字が英数字（isalnum_l）であれば小文字へ、そうでなければ大文字へ変換する。

    • 4. 変換後の文字列長を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/pg_locale_libc.c 内の strtitle_libc 関数から呼び出される。
  • **呼び出し先**: strlen, memcpy, pg_toupper, pg_tolower, toupper_l, tolower_l, isalnum_l。

関連データ構造

  • pg_locale_t: ロケール情報を保持する構造体。locale->info.lt（locale_t 型）を介してOSのロケールハンドルにアクセスする。

失敗モードと境界条件

  • srclen + 1 > destsize の場合、変換処理はスキップされ、何も書き込まれない。
  • srclen が負の値で、かつ src がNUL終端されていない場合は未定義動作となる可能性がある。

保守担当者向けメモ

  • 本関数はシングルバイト専用である。マルチバイト対応が必要な場合は strtitle_libc_mb を参照すること。
  • toupper_l / tolower_l の挙動はOSのロケール実装に依存する。
