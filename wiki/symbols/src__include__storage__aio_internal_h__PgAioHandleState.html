PgAioHandleState

定義

  • 種別: enum
  • 定義箇所: src/include/storage/aio_internal.h:43

目的

PostgreSQL の非同期 I/O (AIO) サブシステムにおいて、個別の I/O 操作（PgAioHandle）のライフサイクルと状態遷移を管理するために使用されます。

シグネチャまたは形状

typedef enum PgAioHandleState {
    PGAIO_HS_IDLE,
    PGAIO_HS_HANDED_OUT,
    PGAIO_HS_DEFINED,
    PGAIO_HS_STAGED,
    PGAIO_HS_SUBMITTED,
    PGAIO_HS_COMPLETED_IO,
    PGAIO_HS_COMPLETED_SHARED,
    PGAIO_HS_COMPLETED_LOCAL
} PgAioHandleState;

入力と出力

  • 入力: pgaio_io_update_state() 関数等を通じて、現在の状態から次の状態へ遷移させる際に使用されます。
  • 出力: pgaio_io_state_get_name() を通じて、デバッグ用の文字列として状態名が出力されます。

前提条件と不変条件

  • 状態遷移は必ず pgaio_io_update_state() を経由して行われる必要があります。
  • COMPLETED_LOCAL 状態に達したハンドルは、その後 IDLE 状態へ戻り、再利用可能となります。

アルゴリズムまたは振る舞い

1. **IDLE**: 未使用状態。 2. **HANDED_OUT**: pgaio_io_acquire() により取得された直後の状態。 3. **DEFINED**: pgaio_io_start_*() 呼び出し後。I/O 実行に必要な情報が揃った状態。 4. **STAGED**: stage() コールバック完了後。実行待ち状態。 5. **SUBMITTED**: I/O メソッドへ実行依頼済み。 6. **COMPLETED_IO**: I/O 完了後、結果処理前。 7. **COMPLETED_SHARED**: 共有完了処理済み。発行元バックエンドが完了を待機している可能性がある状態。 8. **COMPLETED_LOCAL**: ローカル完了コールバック実行済み。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/aio/aio.c 内の各関数（pgaio_io_update_state, pgaio_io_was_recycled 等）および src/backend/storage/aio/aio_funcs.c がこの列挙型を参照しています。
  • 呼び出し先: 状態そのものはデータ型であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • PgAioHandle: この状態を保持する主要な構造体。

失敗モードと境界条件

  • 状態遷移の順序が不正な場合（例: IDLE から直接 SUBMITTED への遷移など）の挙動については、本ソース excerpt には明記されていません。

保守担当者向けメモ

  • 状態遷移のロジックを変更する際は、src/backend/storage/aio/aio.c 内の pgaio_io_update_state および関連する switch 文をすべて更新する必要があります。
  • デバッグ時には pgaio_io_state_get_name を利用して、現在の状態をトレースしてください。
