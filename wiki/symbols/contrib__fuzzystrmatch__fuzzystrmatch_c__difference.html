difference

定義

  • 種別: function
  • 定義箇所: contrib/fuzzystrmatch/fuzzystrmatch.c:777

目的

2つの入力文字列のSoundexコードを計算し、それらが一致する文字数を比較することで、発音の類似度を整数値として算出する。

シグネチャまたは形状

Datum difference(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る2つの text 型引数。
  • **出力**: int32 型の整数。一致したSoundexコードの文字数（0から SOUNDEX_LEN の範囲）。

前提条件と不変条件

  • 入力引数はPostgreSQLの text 型であること。
  • 内部で呼び出される _soundex 関数が、入力文字列を SOUNDEX_LEN 長のコードに変換できること。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を使用して2つの入力文字列を取得し、text_to_cstring でC文字列に変換する。 2. 内部関数 _soundex を呼び出し、それぞれの文字列に対応するSoundexコードを sndx1 および sndx2 バッファに格納する。 3. sndx1 と sndx2 の各インデックスを比較し、一致する文字数をカウントする。 4. カウントした結果を PG_RETURN_INT32 で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのSQLインターフェース（PG_FUNCTION_INFO_V1(difference) により登録）。
  • **呼び出し先**: _soundex（内部静的関数）、text_to_cstring、PG_GETARG_TEXT_PP、PG_RETURN_INT32。
  • **注記**: ソースコード外からの具体的な呼び出し元情報は提供されていません。

関連データ構造

  • SOUNDEX_LEN: Soundexコードの長さを定義する定数。

失敗モードと境界条件

  • 入力文字列が空またはSoundexアルゴリズムで変換できない場合、_soundex の挙動に依存する。
  • SOUNDEX_LEN を超える比較は行われない。

保守担当者向けメモ

  • 本関数は英語の音素に基づくSoundexアルゴリズムに依存しているため、非ラテン文字（日本語など）に対しては適切な類似度を返さない可能性がある。
  • _soundex 関数の実装変更が本関数の出力に直接影響を与える。
