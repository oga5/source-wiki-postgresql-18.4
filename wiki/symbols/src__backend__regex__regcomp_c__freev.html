freev

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regcomp.c:592

目的

正規表現コンパイル中に使用された vars 構造体内の各サブ構造体（NFA、サブ式ツリー、文字ベクトル等）を解放し、リソースをクリーンアップする。

シグネチャまたは形状

static int freev(struct vars *v, int err);

入力と出力

  • **入力**:
  • v: コンパイル状態を保持する vars 構造体へのポインタ。
  • err: 発生したエラーコード（0の場合はエラーなし）。
  • **出力**:
  • int: v->err に格納されているエラーコードを返す。

前提条件と不変条件

  • v は有効な vars 構造体を指している必要がある。
  • v->subs が v->sub10 と等しい場合、subs は静的領域にあるため解放されない。
  • ERR(err) マクロにより、err が 0 でない場合は v->err が更新される。

アルゴリズムまたは振る舞い

1. v->re が存在すれば rfree() で解放する。 2. v->subs が v->sub10 でなければ FREE() で解放する。 3. v->nfa が存在すれば freenfa() で解放する。 4. v->tree が存在すれば freesubre() で解放する。 5. v->treechain が存在すれば cleanst() で解放する。 6. v->cv および v->cv2 が存在すれば freecvec() で解放する。 7. v->lacons が存在すれば freelacons() で解放する。 8. ERR(err) を呼び出し、最後に v->err を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regcomp.c 内の各所（エラー発生時の終了処理や、コンパイル完了時の正常終了処理として呼び出される）。
  • **呼び出し先**:
  • rfree, FREE, freenfa, freesubre, cleanst, freecvec, freelacons, ERR

関連データ構造

  • struct vars: コンパイル中の状態を保持する主要なコンテキスト構造体。

失敗モードと境界条件

  • v が NULL の場合の挙動は定義されていない（ソースコード上はポインタ参照が発生する）。
  • 各ポインタが NULL の場合は、該当する解放処理をスキップする安全な設計となっている。

保守担当者向けメモ

  • v->re は、特定の条件下（regcomp.c:526 付近）で NULL に設定されることで、freev による解放を回避するケースがある。
  • v->subs の解放判定には v->sub10 との比較が使用されており、メモリ管理の最適化が行われている。
