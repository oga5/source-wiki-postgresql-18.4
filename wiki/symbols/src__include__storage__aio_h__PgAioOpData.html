PgAioOpData

定義

  • **種別**: union
  • **定義箇所**: src/include/storage/aio.h:134

目的

非同期 I/O 操作（読み込みまたは書き込み）に必要なパラメータを保持するためのデータ構造です。

シグネチャまたは形状

typedef union {
    struct {
        int      fd;
        uint16   iov_length;
        uint64   offset;
    } read;
    struct {
        int      fd;
        uint16   iov_length;
        uint64   offset;
    } write;
} PgAioOpData;

入力と出力

  • **入力**: I/O 操作開始時に、ファイル記述子 (fd)、I/O ベクタ長 (iov_length)、およびファイルオフセット (offset) を設定します。
  • **出力**: pgaio_io_get_op_data() を介して、特定の PgAioHandle に関連付けられた操作データへのポインタが取得されます。

前提条件と不変条件

  • read 構造体と write 構造体は同一のメンバ構成を持ち、共用体としてメモリを共有します。
  • 非同期 I/O サブシステムにおいて、操作の実行前に有効な値が格納されている必要があります。

アルゴリズムまたは振る舞い

  • pgaio_io_get_op_data 関数を通じて、ハンドルに関連付けられた操作データにアクセスします。
  • 読み込み操作と書き込み操作で共通のデータレイアウトを使用することで、I/O サブシステム内での統一的なハンドリングを可能にしています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/smgr/smgr.c: pgaio_io_get_op_data(ioh) を呼び出し、操作データを取得しています。
  • src/backend/storage/aio/aio_io.c: ポインタとして参照されています。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • PgAioHandle: I/O 操作のコンテキストを保持する構造体であり、本構造体と密接に関連します。
  • PgAioOp: 操作の種類を定義する列挙型。

失敗モードと境界条件

  • fd が無効な場合、または offset が対象ファイルの範囲外である場合の挙動は、下位の I/O 実装（io_uring 等）に依存します。

保守担当者向けメモ

  • 本構造体は公開インターフェースの一部として定義されています。
  • read と write の構造体定義は現在同一ですが、将来的に操作種別ごとの拡張が必要になった場合に備えて共用体として設計されています。
