xl_btree_newroot

定義

  • 種別: struct
  • 定義箇所: src/include/access/nbtxlog.h:344

目的

B-treeインデックスにおいて、新しいルートページが作成された際の操作をWAL（Write-Ahead Log）に記録するための構造体です。

シグネチャまたは形状

typedef struct xl_btree_newroot
{
    BlockNumber rootblk;    /* 新しいルートの場所（ブロック0と冗長） */
    uint32      level;      /* ツリーのレベル */
} xl_btree_newroot;

入力と出力

  • **入力**: B-treeのルート分割操作時に、新しいルートのブロック番号とツリーのレベル情報が本構造体に格納されます。
  • **出力**: XLogInsert を介してWALレコードの一部として書き出されます。

前提条件と不変条件

  • rootblk は、インデックス内の新しいルートページの物理的な位置を示します。
  • level は、作成されたルートページのツリー内での階層レベルを保持します。

アルゴリズムまたは振る舞い

1. **記録**: nbtree のルート分割処理（_bt_insert 等）において、新しいルートページが割り当てられた際に本構造体が初期化されます。 2. **再生**: クラッシュリカバリ時、btree_redo がWALレコードを読み込み、この構造体から得られる情報を用いてインデックスのルート情報を再構築します。

呼び出し元と呼び出し先

  • **呼び出し元（記録側）**:
  • src/backend/access/nbtree/nbtinsert.c
  • src/backend/access/nbtree/nbtpage.c
  • **呼び出し先（再生側）**:
  • src/backend/access/nbtree/nbtxlog.c（btree_redo 内で XLogRecGetData を介して参照）
  • src/backend/access/rmgrdesc/nbtdesc.c（ログ解析時のデバッグ出力用）

関連データ構造

  • SizeOfBtreeNewroot: 本構造体のサイズを計算するためのマクロ（offsetof を使用）。

失敗モードと境界条件

  • 構造体内の rootblk とメタページのブロック0が不整合を起こす場合、インデックスの整合性が損なわれる可能性があります。REDO処理時には、LSNチェックによる冪等性の確保が求められます。

保守担当者向けメモ

  • 本構造体は、B-treeの構造変更（ルートの昇格）という重要なイベントを記録するものです。変更を加える際は、nbtxlog.c のREDOロジックと整合性が取れていることを確認してください。
