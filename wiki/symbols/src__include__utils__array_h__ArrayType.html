ArrayType

定義

  • 種別: struct
  • 定義箇所: src/include/utils/array.h:92

目的

PostgreSQLにおける標準的な「フラット」な可変長配列のメモリレイアウトを規定し、次元数、データオフセット、要素型、NULLビットマップ情報を含むバイナリ形式を定義する。

シグネチャまたは形状

typedef struct ArrayType
{
	int32		vl_len_;		/* varlena header */
	int			ndim;			/* 次元数 */
	int32		dataoffset;		/* データへのオフセット、ビットマップなしの場合は0 */
	Oid			elemtype;		/* 要素型のOID */
} ArrayType;

入力と出力

  • 入力: Datum として受け渡される配列データ。PG_GETARG_ARRAYTYPE_P マクロ等により detoast された状態で取得される。
  • 出力: 配列の各フィールド（次元数、データオフセット、要素型）への直接アクセスが可能。

前提条件と不変条件

  • 配列データは MAXALIGN 境界で開始される。
  • dataoffset が 0 の場合、NULLビットマップは存在しない。
  • dataoffset が 0 でない場合、NULLビットマップはタプルと同様の形式（LSBファースト）で格納される。
  • 配列要素自体は detoast された状態で格納される必要がある。

アルゴリズムまたは振る舞い

  • varlena ヘッダーを含むため、直接的なメモリ操作には注意が必要。
  • ARR_DATA_PTR(a) マクロを使用して、配列のデータ領域へのポインタを取得する。
  • 展開済み配列（ExpandedArrayHeader）とは異なり、フラットなバイナリ形式として保持される。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の例）:
  • contrib/cube/cube.c: PG_GETARG_ARRAYTYPE_P を使用して引数を取得。
  • contrib/dblink/dblink.c: get_text_array_contents 等で配列内容を解析。
  • contrib/hstore/hstore.h: hstoreArrayToPairs の引数として使用。
  • 呼び出し先: 構造体定義のため、特定の関数を呼び出すものではない。

関連データ構造

  • ExpandedArrayHeader: メモリ上で操作効率を高めた展開済み配列。
  • AnyArrayType: フラット配列と展開済み配列を透過的に扱うための共用体。
  • ArrayBuildState: 配列構築時に使用される作業用構造体。

失敗モードと境界条件

  • dataoffset の設定値が不適切な場合、NULLビットマップの読み取りでメモリ破壊が発生する可能性がある。
  • 要素型（elemtype）が正しく設定されていない場合、後続のデータアクセスで型不整合が発生する。

保守担当者向けメモ

  • vl_len_ フィールドには直接触れないこと。
  • 配列要素の個別の TOAST は避けるべきである。
  • アライメント要件（MAXALIGN）を遵守すること。
