ExecNestLoop

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeNestloop.c:59

目的

ネステッドループ結合（Nested Loop Join）のメインループを実行し、外部プランと内部プランからタプルを取得して結合条件およびフィルタ条件に基づいた結果タプルを生成する。

シグネチャまたは形状

TupleTableSlot *ExecNestLoop(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（NestLoopStateとしてキャストされる実行ノードの状態）
  • **出力**: 結合条件を満たす TupleTableSlot*。結合が完了した場合は NULL を返す。

前提条件と不変条件

  • pstate は NestLoopState 型としてキャスト可能であること。
  • ループ内で ResetExprContext(econtext) が呼び出され、タプルごとのメモリコンテキストがリセットされる。
  • 外部タプルが更新されるたびに、内部プランに対して ExecReScan が実行される。

アルゴリズムまたは振る舞い

1. **外部ループ制御**: nl_NeedNewOuter が真の場合、外部プランからタプルを取得する。取得できない場合は結合終了（NULL を返す）。 2. **パラメータ更新**: 外部タプルの値を内部プランのパラメータ（PARAM_EXEC）に設定し、内部プランの chgParam を更新する。 3. **内部スキャン**: ExecReScan で内部プランを再スキャンし、ExecProcNode で内部タプルを取得する。 4. **結合判定**:

  • 内部タプルが取得できた場合、joinqual を評価する。一致すれば MatchedOuter を真にし、otherqual を評価して投影（ExecProject）を行う。
  • 内部タプルが取得できない場合、外部結合（LEFT JOIN / ANTI JOIN）であれば NULL 補完を行い、otherqual を評価して投影を試みる。

    • 5. **ループ継続**: 条件不一致や ANTI JOIN のマッチング時は、次のタプル取得へ進む。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c（ExecProcNode ディスパッチャ経由）
  • **呼び出し先**:
  • ExecProcNode（外部・内部プランの取得）
  • ExecReScan（内部プランの再スキャン）
  • ExecQual（結合条件およびフィルタ条件の評価）
  • ExecProject（結果タプルの投影）

関連データ構造

  • NestLoopState: 実行時の状態（nl_NeedNewOuter, nl_MatchedOuter 等）を保持。
  • NestLoop: プランノードの定義。
  • ExprContext: 式評価用のコンテキスト。

失敗モードと境界条件

  • 外部プランからタプルが取得できない場合、即座に結合終了とみなす。
  • LEFT JOIN / ANTI JOIN において内部タプルが一致しない場合、NULL 補完による判定処理が行われる。
  • single_match が設定されている場合、最初のマッチング後に外部タプルの再取得へ移行する。

保守担当者向けメモ

  • 内部プランは外部タプルごとに ExecReScan されるため、インデックススキャン等の効率的なプランが選択されていることが重要。
  • ENL1_printf マクロによるデバッグ出力が各所に配置されている。
  • CHECK_FOR_INTERRUPTS() がループの先頭で呼び出されており、クエリの中断要求に対応している。
