create_target

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/file_ops.c:158

目的

file_entry_t で指定されたファイルシステムオブジェクト（ディレクトリまたはシンボリックリンク）をターゲットディレクトリ内に作成します。

シグネチャまたは形状

void create_target(file_entry_t *entry)

入力と出力

  • **入力**: entry (file_entry_t *) - 作成対象のファイルメタデータ。
  • **出力**: なし（成功時）。失敗時は pg_fatal によりプロセスを終了します。

前提条件と不変条件

  • entry->action が FILE_ACTION_CREATE であること。
  • entry->target_exists が false であること（ターゲットが未存在であること）。

アルゴリズムまたは振る舞い

1. entry->source_type に基づき、以下のいずれかの処理を実行します。

  • FILE_TYPE_DIRECTORY: create_target_dir を呼び出し、ディレクトリを作成します。
  • FILE_TYPE_SYMLINK: create_target_symlink を呼び出し、シンボリックリンクを作成します。

    • 2. FILE_TYPE_REGULAR が指定された場合は、通常のファイル作成には open_target_file を使用すべきであるため、エラーとして終了します。 3. FILE_TYPE_UNDEFINED の場合は、型が不明なためエラーとして終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/pg_rewind.c:619
  • **呼び出し先**:
  • create_target_dir
  • create_target_symlink
  • pg_fatal

関連データ構造

  • file_entry_t: 同期対象のファイル情報（パス、アクション、型、リンク先情報など）を保持する構造体。

失敗モードと境界条件

  • FILE_TYPE_REGULAR を渡すと pg_fatal が発生します。
  • FILE_TYPE_UNDEFINED を渡すと pg_fatal が発生します。
  • create_target_dir または create_target_symlink 内部で OS エラーが発生した場合、それらの関数が pg_fatal を呼び出し終了します。

保守担当者向けメモ

  • 本関数はディレクトリとシンボリックリンクの作成に特化しています。通常のファイル作成は open_target_file が担うため、混同しないよう注意してください。
  • dry_run モードのチェックは、呼び出し先の create_target_dir や create_target_symlink 内で個別に処理されます。
