PgBenchFunction

定義

  • 種別: enum
  • 定義箇所: src/bin/pgbench/pgbench.h:65-104

目的

pgbench の式評価エンジンにおいて、サポートされる演算子や組み込み関数（ADD, RANDOM, HASH_FNV1A 等）を識別するための列挙型です。

シグネチャまたは形状

typedef enum PgBenchFunction {
    /* ... 各種関数・演算子の列挙 ... */
} PgBenchFunction;

入力と出力

  • **入力**: 式評価時に PgBenchExpr 構造体内の function フィールドとして保持されます。
  • **出力**: 評価エンジン（pgbench.c）がこの値に基づいて、対応する計算ロジックを選択します。

前提条件と不変条件

  • 新しい関数や演算子を追加する場合、この列挙型にシンボルを追加し、pgbench.c 側の評価ロジックを更新する必要があります。
  • PgBenchExpr 構造体の一部として使用される際は、args（引数リスト）と組み合わせて関数呼び出しを表現します。

アルゴリズムまたは振る舞い

pgbench.c 内の評価関数（eval_expr 等）において、この列挙値が switch 文や条件分岐のキーとして使用され、具体的な演算処理が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/bin/pgbench/pgbench.c: isLazyFunc 関数、および式評価を行う各関数で引数として使用。
  • src/bin/pgbench/pgbench.h: PgBenchExpr 構造体内のメンバとして定義。
  • **呼び出し先**:
  • 本シンボル自体は列挙型であり、直接的な呼び出し先はありません。

関連データ構造

  • PgBenchExpr: 式のノードを定義する構造体。この中で PgBenchFunction が使用されます。
  • PgBenchExprLink: 関数呼び出しの引数リストを保持する構造体。

失敗モードと境界条件

  • 定義されていない PgBenchFunction 値が評価エンジンに渡された場合、未定義の動作や評価エラーが発生する可能性があります。

保守担当者向けメモ

  • PgBenchFunction は pgbench の拡張性の中心です。新しい組み込み関数を追加する際は、必ずこの列挙型を更新してください。
  • pgbench.c 内の isLazyFunc など、特定の関数が「遅延評価」を行うかどうかの判定にも使用されているため、変更時は影響範囲に注意してください。
