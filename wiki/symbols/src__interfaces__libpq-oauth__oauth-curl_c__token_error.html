token_error

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq-oauth/oauth-curl.c:169

目的

OAuth 2.0 トークンリクエスト時に認可サーバーから返されるエラー情報を保持するデータ構造です。

シグネチャまたは形状

struct token_error
{
    char *error;
    char *error_description;
};

入力と出力

  • 入力: 認可サーバーからの JSON レスポンスをパースした結果を格納します。
  • 出力: record_token_error を通じて async_ctx に渡され、最終的に PGconn のエラーバッファへ伝播されます。

前提条件と不変条件

  • error および error_description は動的に割り当てられた文字列を指すことが想定されており、使用後は free_token_error による解放が必要です。

アルゴリズムまたは振る舞い

  • parse_token_error 関数により、JSON レスポンスからエラーコードと詳細説明を抽出するために使用されます。
  • record_token_error 関数により、取得したエラー情報を認証コンテキスト（async_ctx）に記録します。

呼び出し元と呼び出し先

  • 呼び出し元（参照）:
  • parse_token_error (引数として利用)
  • record_token_error (引数として利用)
  • free_token_error (引数として利用)
  • 呼び出し先:
  • 情報なし（本構造体はデータ保持用であり、関数呼び出しを含みません）。

関連データ構造

  • async_ctx: 認証フローのコンテキスト。エラー発生時に本構造体を保持します。
  • free_token_error: 本構造体のメンバを解放するための関数。

失敗モードと境界条件

  • JSON パース時にエラーフィールドが存在しない場合、メンバは NULL となる可能性があります。
  • メモリ割り当て失敗時は、error や error_description が適切に初期化されないリスクがあります。

保守担当者向けメモ

  • 本構造体は libpq のエラー報告メカニズムと密接に関連しています。エラーメッセージを更新する際は、PGconn への伝播経路である record_token_error の実装を確認してください。
