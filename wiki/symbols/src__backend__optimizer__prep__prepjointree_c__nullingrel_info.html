nullingrel_info

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/prep/prepjointree.c:49

目的

クエリのプランニング前処理において、各リーフRTE（Range Table Entry）に対して、そのRTEをNULL化する可能性のある外部結合（Outer Join）の集合を保持するために使用されます。

シグネチャまたは形状

typedef struct nullingrel_info
{
    Relids     *nullingrels;
    int         rtlength;
} nullingrel_info;

入力と出力

  • **入力**: nullingrels は Relids 型の配列であり、各インデックス rti はRTEのインデックスに対応します。
  • **出力**: 保持された Relids の集合は、後続の最適化処理（特に PlaceHolderVar の挿入判断など）において、NULL伝播の制御に使用されます。

前提条件と不変条件

  • rtlength はレンジテーブルの長さを表し、主にアサーションチェックに使用されます。
  • nullingrels[rti] には、該当RTEをNULL化し得る外部結合の relid セットが格納されます。

アルゴリズムまたは振る舞い

本構造体は、get_nullingrels 関数によって生成・初期化されます。ジョインツリーの走査過程で、外部結合の制約条件に基づき、どのRTEがどの結合によってNULL化されるかを追跡するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/prep/prepjointree.c 内の複数の箇所（rcon->nullinfo を介した参照）から利用されています。
  • **呼び出し先**: get_nullingrels 関数によってインスタンス化され、palloc_object(nullingrel_info) を通じてメモリ確保が行われます。

関連データ構造

  • Relids: 結合リレーションの識別子集合。
  • pullup_replace_vars_context: 本構造体（nullinfo）を保持し、変数置換処理中に参照されます。

失敗モードと境界条件

  • rtlength と nullingrels 配列のインデックス範囲が一致しない場合、アサーションエラーが発生する可能性があります。
  • メモリ不足時には palloc_object が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は外結合のNULL伝播を管理する重要なコンポーネントです。pullup_replace_vars における PlaceHolderVar の挿入判断ロジックと密接に関連しているため、変更時は結合の強度削減処理への影響を考慮してください。
