pg_conv_map

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_iso8859/utf8_and_iso8859.c:66

目的

PostgreSQLのマルチバイト文字変換において、特定のエンコーディングIDと、UTF-8との相互変換に使用するラジックツリー（マッピングテーブル）を紐付けるためのデータ構造です。

シグネチャまたは形状

typedef struct {
    pg_enc                  encoding;
    const pg_mb_radix_tree *map1; /* ISO 8859からUTF-8への変換用 */
    const pg_mb_radix_tree *map2; /* UTF-8からISO 8859への変換用 */
} pg_conv_map;

入力と出力

本構造体自体はデータ保持用であり、関数ではありません。各フィールドは以下の通りです。

  • encoding: pg_enc 型のエンコーディング識別子。
  • map1: 変換元からUTF-8への変換テーブルへのポインタ。
  • map2: UTF-8から変換元への変換テーブルへのポインタ。

前提条件と不変条件

  • map1 および map2 は、pg_mb_radix_tree 型の静的な変換テーブルを指す必要があります。
  • encoding は、PostgreSQLのエンコーディングシステムで定義された有効な値である必要があります。

アルゴリズムまたは振る舞い

本構造体は、maps[] 配列の要素として定義されます。変換関数（iso8859_to_utf8 等）が呼び出された際、引数として渡されたエンコーディングIDと一致する pg_conv_map エントリが検索され、そのエントリ内の map1 または map2 が変換エンジン（LocalToUtf / UtfToLocal）へ渡されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/mb/conversion_procs/utf8_and_iso8859/utf8_and_iso8859.c 内の maps[] 配列定義にて参照されています。
  • 呼び出し先: 本構造体自体はデータ構造であり、直接関数を呼び出すことはありません。

関連データ構造

  • pg_mb_radix_tree: 実際の文字変換ロジックを保持するラジックツリー構造体。
  • maps[]: pg_conv_map 型の静的配列。サポートされる全エンコーディングの定義を保持します。

失敗モードと境界条件

  • 構造体自体に失敗モードはありませんが、検索時に一致する encoding が見つからない場合、変換処理は失敗します。

保守担当者向けメモ

  • 新しいISO 8859規格を追加する場合は、この構造体を用いて maps[] 配列にエントリを追加してください。
  • 実際の変換処理は wchar.c の汎用関数に依存しているため、本構造体のフィールド定義を変更する場合は、それらの関数との整合性に注意してください。
