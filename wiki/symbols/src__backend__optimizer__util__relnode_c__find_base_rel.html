find_base_rel

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/relnode.c:413

目的

プランナの管理下にあるベースリレーション（RelOptInfo）を、指定されたリレーションID（relid）に基づいて検索・取得する。

シグネチャまたは形状

RelOptInfo *find_base_rel(PlannerInfo *root, int relid)

入力と出力

  • **入力**:
  • root: 現在のクエリプランニング情報を保持する PlannerInfo 構造体へのポインタ。
  • relid: 検索対象となるリレーションの識別子（RangeTblEntry インデックス）。
  • **出力**:
  • 成功時: 指定された relid に対応する RelOptInfo 構造体へのポインタ。
  • 失敗時: 該当するエントリが存在しない場合、elog(ERROR) によりエラーを発生させ、処理を中断する。

前提条件と不変条件

  • root->simple_rel_array が初期化済みであること。
  • relid は root->simple_rel_array_size の範囲内である必要がある。

アルゴリズムまたは振る舞い

1. relid を符号なし整数として扱い、root->simple_rel_array_size と比較して配列外アクセスを防止する。 2. root->simple_rel_array 配列の relid 番目の要素を確認する。 3. 要素が非 NULL であれば、その RelOptInfo ポインタを返す。 4. 配列範囲外であるか、該当するエントリが NULL の場合は elog(ERROR) を呼び出し、処理を停止する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/optimizer/path/allpaths.c
  • src/backend/optimizer/path/clausesel.c
  • src/backend/optimizer/path/costsize.c
  • src/backend/optimizer/path/joinrels.c
  • src/backend/optimizer/plan/analyzejoins.c
  • **呼び出し先**:
  • elog (PostgreSQL のエラー報告マクロ)

関連データ構造

  • PlannerInfo: プランナのグローバル状態を保持する構造体。simple_rel_array を介してリレーションノードを管理する。
  • RelOptInfo: プランナが扱うリレーションの基本構造体。

失敗モードと境界条件

  • relid が simple_rel_array_size を超える場合、または対応する配列要素が NULL の場合、elog(ERROR) が発生し、クエリのプランニングプロセスが異常終了する。

保守担当者向けメモ

  • 本関数はエラー時に NULL を返さずエラーを投げるため、呼び出し側で存在確認が必要な場合は find_base_rel_noerr の使用を検討すること。
  • relid の比較時に符号なしキャストを行っているため、負の relid が渡された場合も安全にエラー処理へ遷移する。
