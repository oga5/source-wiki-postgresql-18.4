ObjectAddressExtra

定義

  • 種別: struct
  • 定義箇所: src/backend/catalog/dependency.c:94

目的

削除処理中の各オブジェクトに対して、その削除を強制した原因となったオブジェクト（依存元）の情報と、処理状態を示すフラグを保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	int			flags;			/* ビットマスク */
	ObjectAddress dependee;		/* 削除を強制したオブジェクト */
} ObjectAddressExtra;

入力と出力

本構造体は単体で関数入出力されるものではなく、ObjectAddresses 構造体内の配列（extras）の要素として管理されます。

前提条件と不変条件

  • ObjectAddresses 構造体において、extras フィールドが NULL でない場合、この構造体の配列としてメモリ確保されます。
  • dependee フィールドには、カスケード削除の起点となったオブジェクトのアドレスが格納されます。

アルゴリズムまたは振る舞い

  • 依存関係の探索（findDependentObjects）および削除処理の過程で、各オブジェクトのメタデータとして参照されます。
  • flags ビットマスクにより、削除対象オブジェクトに対する特定の処理状態や制約を追跡します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/dependency.c 内の findDependentObjects、deleteObjectsInList、reportDependentObjects 等の関数が、ObjectAddresses を介して本構造体の配列を参照します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ObjectAddresses: 削除対象オブジェクトのリストを保持する親構造体。extras フィールドを通じて本構造体の配列を管理します。
  • ObjectAddress: PostgreSQLのカタログオブジェクトを一意に識別するための構造体。

失敗モードと境界条件

  • extras 配列のメモリ確保に失敗した場合、またはインデックス範囲外へのアクセスが発生した場合に不整合が生じます。
  • 循環参照や複雑な依存関係において、dependee の設定が正しく行われない場合、カスケード削除の順序制御に影響を与える可能性があります。

保守担当者向けメモ

  • 新しいフラグビットを追加する場合は、dependency.c 内の定義と整合性を取る必要があります。
  • ObjectAddresses のリスト操作を行う際は、必ず extras 配列のサイズと同期させてください。
