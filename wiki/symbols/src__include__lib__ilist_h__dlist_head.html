dlist_head

定義

  • 種別: struct
  • 定義箇所: src/include/lib/ilist.h:151

目的

PostgreSQLの統合型二重連結リストにおけるリストの起点（ヘッド）を保持する構造体。リストの先頭および末尾へのポインタを管理し、双方向の走査を可能にする。

シグネチャまたは形状

typedef struct dlist_head
{
    dlist_node  head;
} dlist_head;

入力と出力

  • **入力**: リスト操作関数（dlist_delete 等）に対して、リストの起点として本構造体へのポインタが渡される。
  • **出力**: リスト内のノードを辿ることで、リストに含まれる要素（dlist_node を埋め込んだ構造体）にアクセス可能となる。

前提条件と不変条件

  • head.next はリストの最初の要素、または空の場合は &head（循環リストの場合）か NULL を指す。
  • head.prev はリストの最後の要素、または空の場合は &head（循環リストの場合）か NULL を指す。
  • リストに含まれている間、ノードを含む構造体を free() してはならない。

アルゴリズムまたは振る舞い

  • リストの起点として機能し、dlist_node 型のメンバを通じてリスト内の要素を双方向にリンクする。
  • 循環リストとして運用する場合、空リスト時は head 自身を指すことで分岐を減らし、高速な操作を実現する。

呼び出し元と呼び出し先

  • **呼び出し元（参照例）**:
  • src/backend/access/gin/gindatapage.c
  • src/backend/access/transam/parallel.c
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/bgworker.c
  • src/backend/postmaster/pmchild.c
  • **呼び出し先**:
  • src/backend/lib/ilist.c 内の dlist_member_check や dlist_check が本構造体を引数として受け取り、整合性検証を行う。

関連データ構造

  • dlist_node: リストの各要素に埋め込まれるリンク構造体。
  • dlist_iter / dlist_mutable_iter: リスト走査のためのイテレータ構造体。

失敗モードと境界条件

  • **空リスト**: NULL を指す場合と、自身を指す（循環）場合の2通りの表現がある。
  • **整合性**: ILIST_DEBUG を定義することで、リスト操作時の整合性チェックが可能。

保守担当者向けメモ

  • リストの操作はメモリ割り当てを伴わず、ポインタの書き換えのみで完結する。
  • 循環リストの方が分岐が少なく高速であるため、可能な限り循環リストとしての運用が推奨される。
