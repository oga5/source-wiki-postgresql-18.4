push_to_sink

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup.c:1952

目的

バックアップデータを bbsink バッファに蓄積し、バッファが満杯になった際にアーカイブ処理（bbsink_archive_contents）とチェックサム更新をトリガーするバッファリング関数。

シグネチャまたは形状

static void push_to_sink(bbsink *sink, pg_checksum_context *checksum_ctx, size_t *bytes_done, void *data, size_t length)

入力と出力

  • sink: データ送信先となる bbsink 構造体。
  • checksum_ctx: データの整合性検証に使用するチェックサムコンテキスト。
  • bytes_done: 現在のバッファ内の書き込み済みバイト数（入出力両用）。
  • data: 送信対象のデータポインタ。
  • length: 送信するデータの長さ。
  • 出力: なし（副作用として sink のバッファ更新およびチェックサム更新を行う）。

前提条件と不変条件

  • sink->bbs_buffer が割り当て済みであること。
  • *bytes_done は常に sink->bbs_buffer_length 未満であること。

アルゴリズムまたは振る舞い

1. 入力データが現在のバッファの空き容量に収まる場合、memcpy でバッファにコピーし、bytes_done を更新して終了する。 2. 入力データがバッファ容量を超える場合、以下のループを繰り返す：

  • バッファの残り分を埋めて満杯にする。
  • bbsink_archive_contents を呼び出し、バッファの内容を送信先にフラッシュする。
  • pg_checksum_update を呼び出し、フラッシュしたバッファ内容でチェックサムを更新する。
  • bytes_done を 0 にリセットし、残りのデータを処理する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/backup/basebackup.c 内の複数の箇所（主にファイル送信処理中）。
  • 呼び出し先: bbsink_archive_contents（bbsink インターフェース経由）、pg_checksum_update。

関連データ構造

  • bbsink: バックアップデータの出力先を抽象化した構造体。
  • pg_checksum_context: ページチェックサム計算用の状態保持構造体。

失敗モードと境界条件

  • pg_checksum_update が負の値を返した場合、elog(ERROR) によりバックアップ処理全体が中断される。
  • 入力 length が 0 の場合、ループは実行されず即座に終了する。

保守担当者向けメモ

  • 本関数は、bbsink のバッファ管理とチェックサム計算を同期させるための重要なユーティリティです。
  • memcpy を使用しているため、データポインタの型変換（char *）に注意してください。
  • バッファのフラッシュタイミングは length < sink->bbs_buffer_length - *bytes_done という条件式により、バッファが完全に満たされた瞬間に発生するように制御されています。
