testcustomrmgrs_identify

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_custom_rmgrs/test_custom_rmgrs.c:105

目的

WALレコードの info ビットフィールドを解析し、そのレコード種別を文字列として特定する。

シグネチャまたは形状

const char *testcustomrmgrs_identify(uint8 info)

入力と出力

  • **入力**: uint8 info - WALレコードの xl_info フィールド。
  • **出力**: const char * - レコード種別を表す文字列（"TEST_CUSTOM_RMGRS_MESSAGE"）、または一致しない場合は NULL。

前提条件と不変条件

  • 入力された info は、XLR_INFO_MASK を用いてマスクされた後に XLOG_TEST_CUSTOM_RMGRS_MESSAGE と比較される。

アルゴリズムまたは振る舞い

1. info から XLR_INFO_MASK を除外した値が XLOG_TEST_CUSTOM_RMGRS_MESSAGE と一致するか確認する。 2. 一致する場合、"TEST_CUSTOM_RMGRS_MESSAGE" という文字列リテラルを返す。 3. 一致しない場合、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: WALサブシステム（RmgrData 構造体の .rm_identify フィールドを通じて間接的に呼び出される）。
  • **呼び出し先**: 情報なし。

関連データ構造

  • RmgrData: カスタムRmgrのコールバックを保持する構造体。本関数はこの構造体のメンバとして登録される。

失敗モードと境界条件

  • info が定義された XLOG_TEST_CUSTOM_RMGRS_MESSAGE 以外の値である場合、NULL を返す。

保守担当者向けメモ

  • 本関数は testcustomrmgrs_rmgr 構造体の初期化時に rm_identify コールバックとして割り当てられている。
  • 将来的に新しいレコード種別を追加する場合は、条件分岐を拡張する必要がある。
