AdjustFractMicroseconds

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/datetime.c:546

目的

浮動小数点数で表現された時間単位（frac）を、指定されたスケール（scale）に基づいてマイクロ秒単位に変換し、pg_itm_in 構造体のマイクロ秒フィールド（tm_usec）に加算する。

シグネチャまたは形状

static bool AdjustFractMicroseconds(double frac, int64 scale, struct pg_itm_in *itm_in)

入力と出力

  • **入力**:
  • frac: 変換対象の浮動小数点数（絶対値が1未満であることを想定）。
  • scale: frac をマイクロ秒に換算するための乗数。
  • itm_in: 更新対象の pg_itm_in 構造体へのポインタ。
  • **出力**:
  • 成功時は true、加算時にオーバーフローが発生した場合は false を返す。

前提条件と不変条件

  • frac の絶対値は 1 未満であると想定されている。
  • itm_in->tm_usec の値は、加算処理中に int64 の範囲を超える可能性がある。

アルゴリズムまたは振る舞い

1. frac が 0 の場合は、何もせず true を返す。 2. frac に scale を乗算し、整数部をマイクロ秒（usec）として抽出する。 3. 残りの小数部に対して四捨五入を行い、usec を調整する。 4. pg_add_s64_overflow を使用して、itm_in->tm_usec に usec を加算する。 5. 加算結果がオーバーフローした場合は false を返し、成功した場合は true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/datetime.c 内の複数の箇所から呼び出されている（例: 601, 635, 3728, 4012, 4022, 4071, 4109 行目）。
  • **呼び出し先**: pg_add_s64_overflow（オーバーフローチェック用マクロ/関数）。

関連データ構造

  • struct pg_itm_in: 日時計算に使用される内部構造体。tm_usec フィールドを保持する。

失敗モードと境界条件

  • **オーバーフロー**: itm_in->tm_usec への加算結果が int64 の範囲を超えた場合、false を返す。
  • **四捨五入**: 小数部が 0.5 を超える場合は切り上げ、-0.5 未満の場合は切り下げを行う。

保守担当者向けメモ

  • 本関数は datetime.c 内で頻繁に利用されるユーティリティである。
  • frac が 1 未満であることを前提とした最適化が行われているため、この前提を崩すような変更を行う場合は注意が必要である。
