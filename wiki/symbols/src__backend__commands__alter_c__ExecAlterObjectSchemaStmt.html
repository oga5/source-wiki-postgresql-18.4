ExecAlterObjectSchemaStmt

定義

  • 種別: function
  • 定義箇所: src/backend/commands/alter.c:533

目的

SQLの ALTER ... SET SCHEMA ステートメントを実行し、指定されたデータベースオブジェクトを新しいスキーマへ移動させるためのディスパッチャ関数です。

シグネチャまたは形状

ObjectAddress ExecAlterObjectSchemaStmt(AlterObjectSchemaStmt *stmt, ObjectAddress *oldSchemaAddr);

入力と出力

  • **入力**:
  • stmt: 実行対象の AlterObjectSchemaStmt 構造体（オブジェクト型、オブジェクト名、移動先のスキーマ名を含む）。
  • oldSchemaAddr: 移動前のスキーマ情報を格納するためのポインタ（NULL指定可能）。
  • **出力**:
  • 移動したオブジェクトの ObjectAddress を返します。

前提条件と不変条件

  • stmt->objectType がサポートされているオブジェクト型である必要があります。
  • 汎用的なコードパス（OBJECT_FUNCTION 等）では、対象オブジェクトに対して AccessExclusiveLock を取得します。

アルゴリズムまたは振る舞い

1. **ディスパッチ**: stmt->objectType に基づき、以下のいずれかの処理へ分岐します。

  • **専用処理**: OBJECT_EXTENSION は AlterExtensionNamespace、テーブル系オブジェクトは AlterTableNamespace、型系オブジェクトは AlterTypeNamespace を呼び出します。
  • **汎用処理**: 関数、演算子、統計情報などのオブジェクトに対しては、get_object_address で対象を特定し、AlterObjectNamespace_internal を呼び出してカタログ上のスキーマ情報を更新します。

    • 2. **スキーマ更新**: 汎用処理では LookupCreationNamespace で移動先スキーマのOIDを取得し、カタログを更新します。 3. **戻り値設定**: oldSchemaAddr が指定されている場合、移動前のスキーマOIDを NamespaceRelationId と共に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c（パーサーからのステートメント実行時）
  • **呼び出し先**:
  • AlterExtensionNamespace, AlterTableNamespace, AlterTypeNamespace
  • get_object_address, table_open, table_close
  • LookupCreationNamespace, AlterObjectNamespace_internal

関連データ構造

  • AlterObjectSchemaStmt: パーサーが生成するステートメントノード。
  • ObjectAddress: オブジェクトの識別情報（クラスIDとオブジェクトID）。

失敗モードと境界条件

  • stmt->objectType が未対応の型である場合、elog(ERROR) を発生させます。
  • table_open や get_object_address などの内部処理で、対象オブジェクトが存在しない場合や権限がない場合にエラーが発生します。

保守担当者向けメモ

  • 本関数は「単純なカタログ更新」を行うオブジェクトの集約点です。物理的なストレージ変更を伴うオブジェクト（テーブル等）は専用の関数へ委譲されます。
  • 新しいオブジェクト型を追加する際は、switch 文の汎用パスに追加可能か検討してください。
