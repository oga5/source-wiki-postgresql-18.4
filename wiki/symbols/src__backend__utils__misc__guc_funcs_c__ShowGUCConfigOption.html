ShowGUCConfigOption

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/guc_funcs.c:427

目的

指定されたGUC（Grand Unified Configuration）パラメータの現在の値を、単一カラムのテキスト形式としてクライアントへ出力する。

シグネチャまたは形状

static void ShowGUCConfigOption(const char *name, DestReceiver *dest)

入力と出力

  • **入力**:
  • name: 取得対象のGUCパラメータ名（文字列）。
  • dest: 出力先を制御する DestReceiver 構造体。
  • **出力**: なし（dest を介してタプルを送信する）。

前提条件と不変条件

  • GetConfigOptionByName を使用してパラメータ値を取得するため、指定された name が存在しない場合はエラーが発生する可能性がある。
  • 出力されるタプルは TEXT 型の単一カラムとして構成される。

アルゴリズムまたは振る舞い

1. GetConfigOptionByName を呼び出し、パラメータの正規化された名前と現在の値を文字列として取得する。 2. CreateTemplateTupleDesc を用いて、1カラム（TEXT型）のタプル記述子を作成する。 3. begin_tup_output_tupdesc を使用して、指定された dest に対する出力準備を行う。 4. do_text_output_oneline を呼び出し、取得した値をクライアントへ送信する。 5. end_tup_output を呼び出し、出力処理を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/guc_funcs.c 内の GetPGVariable 関数から呼び出される。
  • **呼び出し先**:
  • GetConfigOptionByName
  • CreateTemplateTupleDesc
  • TupleDescInitBuiltinEntry
  • begin_tup_output_tupdesc
  • do_text_output_oneline
  • end_tup_output

関連データ構造

  • DestReceiver: 出力先（クライアントや内部バッファなど）を抽象化した構造体。
  • TupOutputState: タプル出力処理の状態を保持する構造体。
  • TupleDesc: タプルの構造を定義する記述子。

失敗モードと境界条件

  • name に該当するGUCパラメータが存在しない場合、GetConfigOptionByName の内部ロジックによりエラーがスローされる。
  • メモリ不足等のシステムエラーが発生した場合、各出力関数がエラーを報告する。

保守担当者向けメモ

  • 本関数は SHOW <parameter> コマンドのバックエンド処理の一部である。
  • 出力形式（TEXT型）は固定されているため、将来的に他の型や複数カラムの出力をサポートする場合は、タプル記述子の初期化ロジックを修正する必要がある。
