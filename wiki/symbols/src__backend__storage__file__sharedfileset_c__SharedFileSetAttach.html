SharedFileSetAttach

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/sharedfileset.c:55

目的

既存の SharedFileSet にバックエンドが参加し、参照カウンタをインクリメントするとともに、DSM（動的共有メモリ）デタッチ時のクリーンアップ処理を登録します。

シグネチャまたは形状

void SharedFileSetAttach(SharedFileSet *fileset, dsm_segment *seg)

入力と出力

  • **入力**:
  • fileset: 操作対象の SharedFileSet 構造体へのポインタ。
  • seg: 関連付けられた dsm_segment へのポインタ。
  • **出力**: なし（成功時）。失敗時はエラーを報告します。

前提条件と不変条件

  • fileset->mutex を用いて参照カウンタの整合性が保護されています。
  • 参照カウンタ（refcnt）が 0 の場合、そのセットは既に破棄済みとみなされます。

アルゴリズムまたは振る舞い

1. fileset->mutex を取得し、refcnt が 0 でないことを確認します。 2. refcnt が 0 の場合、エラー（ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE）を発生させます。 3. refcnt が 0 でない場合、refcnt をインクリメントします。 4. スピンロックを解放します。 5. on_dsm_detach を呼び出し、DSM デタッチ時に SharedFileSetOnDetach が実行されるようコールバックを登録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeHashjoin.c:1761
  • src/backend/utils/sort/tuplesort.c:2964
  • **呼び出し先**:
  • SpinLockAcquire / SpinLockRelease
  • ereport
  • on_dsm_detach
  • SharedFileSetOnDetach (コールバックとして登録)

関連データ構造

  • SharedFileSet: 共有ファイルセットを管理する構造体。
  • dsm_segment: 動的共有メモリセグメント。

失敗モードと境界条件

  • refcnt が 0 の状態で呼び出された場合、既に破棄されたセットへのアタッチとみなされ、エラーが発生します。

保守担当者向けメモ

  • 本関数はエラー発生時のクリーンアップパスでも実行される可能性があるため、コールバック登録後の処理でエラーを発生させないよう注意してください。
  • 参照カウンタの操作は必ずスピンロックで保護されている必要があります。
