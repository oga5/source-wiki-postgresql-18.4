gbt_macadlt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_macaddr.c:53

目的

macaddr 型の2つの値に対して「より小さい（less than）」という比較を行い、その結果を真偽値で返すためのコールバック関数です。

シグネチャまたは形状

bool gbt_macadlt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象となる macaddr 型のデータへのポインタ。
  • flinfo: 関数管理情報（本関数内では直接使用されません）。
  • **出力**:
  • a が b より小さい場合に true、それ以外の場合に false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な macaddr 型のデータとして解釈可能である必要があります。
  • PostgreSQLの内部関数 macaddr_lt が正常に動作する環境である必要があります。

アルゴリズムまたは振る舞い

1. PointerGetDatum を使用して、入力されたポインタを Datum 型に変換します。 2. PostgreSQLの標準関数 macaddr_lt を DirectFunctionCall2 を介して呼び出します。 3. macaddr_lt の戻り値を DatumGetBool で真偽値に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_macaddr.c 内の静的構造体 tinfo（gbtree_ninfo 型）から参照されています。
  • **呼び出し先**: macaddr_lt（PostgreSQLのコア関数）。

関連データ構造

  • tinfo: 本関数を比較ロジックとして保持する gbtree_ninfo 構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、PointerGetDatum および macaddr_lt の挙動に依存します。
  • macaddr_lt がエラーをスローした場合、本関数もその例外を伝播します。

保守担当者向けメモ

  • 本関数は btree_utils_num が提供する汎用的なGiSTインデックス処理ロジックからコールバックされることを前提としています。
  • macaddr 型の比較ロジックを変更する必要がある場合は、src/backend/utils/adt/mac.c の macaddr_lt を確認してください。
