reduce_outer_joins_pass2_state

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/prep/prepjointree.c:93

目的

外結合（Outer Join）の削減処理における第2パス（pass2）の実行状態を保持するための構造体です。主に、内結合へ変換されたリレーションの追跡と、部分的に削減された完全外結合（FULL JOIN）の情報を管理します。

シグネチャまたは形状

typedef struct reduce_outer_joins_pass2_state
{
	Relids		inner_reduced;	/* OJ relids reduced to plain inner joins */
	List	   *partial_reduced;	/* List of partially reduced FULL joins */
} reduce_outer_joins_pass2_state;

入力と出力

本構造体は関数間で状態を共有するために使用されます。

  • inner_reduced: 内結合へ変換された外結合の Relids セット。
  • partial_reduced: 部分的に削減された完全外結合のリスト。

前提条件と不変条件

  • inner_reduced は Relids 型であり、結合の削減過程で更新されます。
  • partial_reduced は List 型であり、report_reduced_full_join 等の関数を通じて要素が追加されます。

アルゴリズムまたは振る舞い

本構造体は reduce_outer_joins の処理フローにおいて、結合の型変換結果を蓄積するために使用されます。第2パスでは、第1パスで特定された情報を基に、ジョインツリー内の結合ノードの属性を更新する際のコンテキストとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/prep/prepjointree.c 内の reduce_outer_joins 関連処理にてインスタンス化されます（例: 3124 行目）。
  • **呼び出し先**: report_reduced_full_join 関数（158 行目、3546 行目）が本構造体を引数として受け取り、状態の更新を行います。

関連データ構造

  • Relids: 結合対象のリレーション識別子セット。
  • List: PostgreSQL の汎用リスト構造。

失敗モードと境界条件

  • 構造体自体に起因する失敗モードは定義されていません。
  • 結合の削減ロジックが複雑であるため、本構造体に格納される Relids や List が不整合な状態になると、プランナが不正なジョインツリーを生成する可能性があります。

保守担当者向けメモ

  • 本構造体は reduce_outer_joins の第2パス専用です。第1パス用の reduce_outer_joins_pass1_state と混同しないよう注意してください。
  • report_reduced_full_join 関数が本構造体をどのように更新しているかを追跡することで、完全外結合の削減ロジックを理解できます。
