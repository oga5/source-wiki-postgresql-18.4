gtsquery_consistent_oldsig

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_gist.c:272

目的

古いバージョンの PostgreSQL との互換性を維持するためのラッパー関数です。

シグネチャまたは形状

Datum gtsquery_consistent_oldsig(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS (関数マネージャ経由の引数)
  • 出力: Datum (内部的には gtsquery_consistent の戻り値である bool を返却)

前提条件と不変条件

  • gtsquery_consistent が利用可能であること。
  • 本関数は互換性維持のみを目的としており、新規開発での使用は推奨されません。

アルゴリズムまたは振る舞い

gtsquery_consistent 関数を呼び出し、その結果をそのまま返却します。

呼び出し元と呼び出し先

  • 呼び出し先: gtsquery_consistent
  • 呼び出し元: 提供されたソース情報には具体的な呼び出し元の記載はありません。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約で使用される引数構造体。

失敗モードと境界条件

  • 内部で呼び出される gtsquery_consistent の失敗モードに依存します。

保守担当者向けメモ

  • 本関数は互換性維持のためのエイリアスです。ロジックの変更が必要な場合は gtsquery_consistent を修正してください。
