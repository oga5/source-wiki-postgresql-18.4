MemoryContextAlloc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/mcxt.c:1190

目的

指定されたメモリコンテキスト内で、指定されたサイズのメモリ領域を割り当てます。

シグネチャまたは形状

void *MemoryContextAlloc(MemoryContext context, Size size)

入力と出力

  • **入力**:
  • context: 割り当て対象の MemoryContext 構造体へのポインタ。
  • size: 割り当てるメモリサイズ（バイト単位）。
  • **出力**:
  • 割り当てられたメモリ領域へのポインタ。

前提条件と不変条件

  • context は MemoryContextIsValid を満たす有効なポインタであること。
  • 呼び出し時にクリティカルセクション内にいないこと（AssertNotInCriticalSection による検証）。

アルゴリズムまたは振る舞い

1. コンテキストの有効性とクリティカルセクションの状態を検証します。 2. コンテキストの isReset フラグを false に設定します。 3. コンテキストのメソッドテーブルにある alloc 関数を呼び出し、実際のメモリ確保を委譲します。 4. Valgrind 用のメモリプール割り当て通知（VALGRIND_MEMPOOL_ALLOC）を実行します。 5. 確保したメモリ領域へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_nbtree.c、contrib/bloom/blutils.c、contrib/btree_gist/btree_enum.c、contrib/dblink/dblink.c、contrib/hstore/hstore_io.c、contrib/pg_trgm/trgm_gist.c、contrib/pg_trgm/trgm_regexp.c など多数。
  • **呼び出し先**: context->methods->alloc（各メモリコンテキスト型固有の実装）。

関連データ構造

  • MemoryContext: メモリ管理のコンテキストを定義する構造体。
  • MemoryContextMethods: 各コンテキスト型が実装するメソッドテーブル。

失敗モードと境界条件

  • メモリ割り当て失敗時の処理は、呼び出される context->methods->alloc 実装側に委ねられています。本関数自体は割り当て後のチェックを行いません。

保守担当者向けメモ

  • 本関数はコンパイラの「末尾呼び出し最適化（sibling call optimization）」を有効にするため、割り当て後の処理を最小限に抑えています。本関数の後にコードを追加する場合は、alloc メソッド側での実装を検討してください。
