PLy_spi_subtransaction_commit

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_spi.c:572

目的

SPI 操作のために開始された内部サブトランザクションをコミットし、呼び出し元のトランザクションコンテキストおよびリソース所有者へ安全に復帰させる。

シグネチャまたは形状

void PLy_spi_subtransaction_commit(MemoryContext oldcontext, ResourceOwner oldowner)

入力と出力

  • **入力**:
  • oldcontext: サブトランザクション開始前のメモリコンテキスト。
  • oldowner: サブトランザクション開始前のリソース所有者。
  • **出力**: なし（void）。

前提条件と不変条件

  • 本関数を呼び出す前に、対応する PLy_spi_subtransaction_begin が呼び出され、サブトランザクションがアクティブである必要がある。
  • 呼び出し後、現在のメモリコンテキストは oldcontext に、リソース所有者は oldowner に復元される。

アルゴリズムまたは振る舞い

1. ReleaseCurrentSubTransaction() を呼び出し、現在のサブトランザクションをコミットして終了する。 2. MemoryContextSwitchTo(oldcontext) を実行し、メモリコンテキストをサブトランザクション開始前の状態に戻す。 3. CurrentResourceOwner を oldowner に更新し、リソース所有権を復元する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_spi.c 内の各 SPI 実行関数。
  • src/pl/plpython/plpy_cursorobject.c 内のカーソル操作関数。
  • **呼び出し先**:
  • ReleaseCurrentSubTransaction() (PostgreSQL バックエンド関数)
  • MemoryContextSwitchTo() (PostgreSQL バックエンド関数)

関連データ構造

  • MemoryContext: PostgreSQL のメモリ管理単位。
  • ResourceOwner: リソース（ファイル記述子、ロック等）の所有権を追跡する構造体。

失敗モードと境界条件

  • サブトランザクションがアクティブでない状態で呼び出した場合の挙動は、PostgreSQL バックエンドの ReleaseCurrentSubTransaction の仕様に依存する。

保守担当者向けメモ

  • 本関数は PG_TRY ブロックの正常終了パスで使用されることを想定している。エラー発生時には PLy_spi_subtransaction_abort を使用すること。
  • メモリコンテキストとリソース所有者の復元順序は、PostgreSQL のトランザクション管理の不変条件に従っている。
