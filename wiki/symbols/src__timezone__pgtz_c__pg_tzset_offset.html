pg_tzset_offset

定義

  • 種別: function
  • 定義箇所: src/timezone/pgtz.c:319

目的

指定された秒単位のオフセット値（GMTからの差分）に基づき、POSIX形式のタイムゾーン文字列を生成し、対応する pg_tz 構造体を取得する。

シグネチャまたは形状

pg_tz *pg_tzset_offset(long gmtoffset)

入力と出力

  • **入力**: gmtoffset (long) - GMTからのオフセット秒数。
  • **出力**: pg_tz * - 成功時は生成されたタイムゾーン定義へのポインタ、失敗時は NULL。

前提条件と不変条件

  • gmtoffset は秒単位で指定される。
  • 生成されるタイムゾーン名は <-HH:MM:SS>+HH:MM:SS または <+HH:MM:SS>-HH:MM:SS の形式に従う。
  • 内部で pg_tzset を呼び出すため、キャッシュの制約を受ける。

アルゴリズムまたは振る舞い

1. gmtoffset の絶対値を計算し、時・分・秒に分解する。 2. snprintf を使用して、HH:MM:SS 形式の文字列（offsetstr）を構築する。 3. gmtoffset の正負に応じて、POSIX 形式のタイムゾーン名（tzname）を組み立てる。

  • 正のオフセットの場合: <-offset>+offset
  • 負のオフセットの場合: <+offset>-offset

    • 4. 構築した tzname を引数として pg_tzset を呼び出し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/variable.c
  • src/backend/utils/adt/datetime.c
  • **呼び出し先**:
  • pg_tzset (同ファイル内)

関連データ構造

  • pg_tz: タイムゾーン情報を保持する構造体。

失敗モードと境界条件

  • pg_tzset が失敗した場合（メモリ不足や無効な形式など）、本関数も NULL を返す。
  • gmtoffset が極端に大きい場合、tzname バッファ（128バイト）のオーバーフローは snprintf により防止されるが、意図しない文字列生成となる可能性がある。

保守担当者向けメモ

  • 本関数は POSIX 形式のタイムゾーン指定を動的に生成する。生成された名前は pg_tzset のキャッシュ機構によって管理されるため、頻繁な呼び出しはキャッシュの肥大化を招く可能性がある。
