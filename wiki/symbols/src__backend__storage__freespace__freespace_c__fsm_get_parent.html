fsm_get_parent

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/freespace.c:526

目的

FSM（Free Space Map）の木構造において、指定された子ノードの論理アドレスから、その親ノードの論理アドレスおよび親ページ内でのスロット位置を算出する。

シグネチャまたは形状

static FSMAddress fsm_get_parent(FSMAddress child, uint16 *slot)

入力と出力

  • **入力**:
  • child: 親を特定するための FSMAddress 構造体。
  • slot: 親ページ内でのインデックス（スロット番号）を格納するための出力用ポインタ。
  • **出力**:
  • 戻り値: 親ノードの FSMAddress 構造体。

前提条件と不変条件

  • child.level が FSM_ROOT_LEVEL 未満であること（ルートノードには親が存在しないため、アサーションで検証される）。

アルゴリズムまたは振る舞い

1. child.level に 1 を加算して親のレベルを決定する。 2. child.logpageno を SlotsPerFSMPage で除算し、親の論理ページ番号を算出する。 3. child.logpageno を SlotsPerFSMPage で剰余演算し、親ページ内でのスロット位置を slot に書き込む。 4. 計算した親のレベルとページ番号を保持する FSMAddress を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/freespace/freespace.c 内の関数（行 788, 869, 870 付近で確認）。
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わない。算術演算のみを実行する。

関連データ構造

  • FSMAddress: FSM の木構造上のレベルと論理ページ番号を保持する構造体。

失敗モードと境界条件

  • child.level が FSM_ROOT_LEVEL の場合、アサーション失敗によりプロセスが停止する。

保守担当者向けメモ

  • 本関数は FSM の木構造を上方向に辿るための基本的なナビゲーションロジックである。
  • SlotsPerFSMPage は fsm_internals.h で定義された定数であり、この値の変更は本関数の計算結果に直接影響する。
