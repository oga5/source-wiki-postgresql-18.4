WalRcvFetchTimeLineHistoryFiles

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walreceiver.c:724

目的

プライマリサーバーから指定された範囲のタイムライン履歴ファイルを取得し、ローカルの pg_wal ディレクトリへ保存する。

シグネチャまたは形状

static void WalRcvFetchTimeLineHistoryFiles(TimeLineID first, TimeLineID last)

入力と出力

  • **入力**:
  • first: 取得を開始するタイムラインID。
  • last: 取得を終了するタイムラインID。
  • **出力**: なし（副作用としてファイルシステムへの書き込みを行う）。

前提条件と不変条件

  • wrconn（接続ハンドル）が有効であること。
  • タイムラインID 1 には履歴ファイルが存在しないため、スキップされる。
  • 取得対象のタイムライン履歴ファイルがローカルに存在しない場合のみ取得を行う。

アルゴリズムまたは振る舞い

1. first から last までの各タイムラインIDについてループを実行する。 2. existsTimeLineHistory を用いてローカルにファイルが存在するか確認し、存在しない場合のみ以下の処理を行う。 3. walrcv_readtimelinehistoryfile を呼び出し、プライマリからファイル名、内容、サイズを取得する。 4. TLHistoryFileName で計算した期待されるファイル名と、プライマリが報告したファイル名を比較し、不一致があれば ERRCODE_PROTOCOL_VIOLATION でエラー終了する。 5. writeTimeLineHistoryFile を呼び出し、取得した内容をローカルに書き込む。 6. XLogArchiveMode の設定に基づき、XLogArchiveForceDone または XLogArchiveNotify を呼び出し、アーカイブの準備状態を更新する。 7. 取得に使用したメモリ領域（fname, content）を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/walreceiver.c 内のストリーミング開始処理（349行目、601行目）。
  • **呼び出し先**:
  • existsTimeLineHistory
  • walrcv_readtimelinehistoryfile
  • TLHistoryFileName
  • writeTimeLineHistoryFile
  • XLogArchiveForceDone / XLogArchiveNotify
  • pfree

関連データ構造

  • TimeLineID: タイムラインを識別する型。
  • wrconn: プライマリサーバーとの接続を保持するグローバル変数（walreceiver.c 内）。

失敗モードと境界条件

  • **プロトコル違反**: プライマリから報告されたファイル名が、ローカルで計算した期待値と異なる場合にエラーとなる。
  • **ファイル存在時**: すでにローカルに履歴ファイルが存在する場合、そのタイムラインの取得はスキップされる。
  • **タイムライン1**: タイムライン1は履歴ファイルを持たないため、常にスキップされる。

保守担当者向けメモ

  • 本関数は libpqwalreceiver を介してプライマリと通信するため、ネットワークエラーやプライマリ側の状態異常により walrcv_readtimelinehistoryfile が失敗する可能性がある。
  • アーカイブモードの切り替え（ARCHIVE_MODE_ALWAYS）に応じてアーカイブ通知の挙動が変わるため、変更時は注意が必要である。
