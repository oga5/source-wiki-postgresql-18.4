GetReplicationApplyDelay

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walreceiverfuncs.c:364

目的

レプリケーションにおけるWALの適用遅延時間（ミリ秒単位）を算出する。

シグネチャまたは形状

int GetReplicationApplyDelay(void)

入力と出力

  • **入力**: なし（共有メモリ上の WalRcv 構造体および現在のリカバリ状態を参照）。
  • **出力**:
  • 0: 受信済みWALと適用済みWALが一致している場合。
  • -1: 適用開始時刻が未設定の場合。
  • int: 適用遅延時間（ミリ秒）。

前提条件と不変条件

  • 共有メモリ上の WalRcv 構造体へのアクセスには、walrcv->mutex によるスピンロックが使用される。
  • GetXLogReplayRecPtr および GetCurrentChunkReplayStartTime が有効な値を返す環境で呼び出される必要がある。

アルゴリズムまたは振る舞い

1. WalRcv->mutex を取得し、共有メモリから最新の受信済みWAL位置（flushedUpto）を読み取ってロックを解放する。 2. GetXLogReplayRecPtr(NULL) を呼び出し、現在の適用済みWAL位置を取得する。 3. 受信位置と適用位置が一致していれば 0 を返す。 4. GetCurrentChunkReplayStartTime() を呼び出し、現在のチャンクの適用開始時刻を取得する。 5. 適用開始時刻が 0 であれば -1 を返す。 6. 現在時刻と適用開始時刻の差分を TimestampDifferenceMilliseconds で計算し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walreceiver.c:1280
  • **呼び出し先**:
  • SpinLockAcquire / SpinLockRelease
  • GetXLogReplayRecPtr
  • GetCurrentChunkReplayStartTime
  • GetCurrentTimestamp
  • TimestampDifferenceMilliseconds

関連データ構造

  • WalRcvData (共有メモリ上の構造体)
  • XLogRecPtr (WAL位置)

失敗モードと境界条件

  • 適用開始時刻が取得できない場合（0 が返る場合）、関数は -1 を返す。
  • 受信位置と適用位置が一致している場合、遅延は 0 とみなされる。

保守担当者向けメモ

  • 本関数は walreceiver.c から呼び出されることを想定している。
  • 共有メモリの競合を避けるため、flushedUpto の読み取り時には必ずスピンロックを保持すること。
