PGRUsage

定義

  • 種別: struct
  • 定義箇所: src/include/utils/pg_rusage.h:22

目的

PostgreSQL バックエンドにおけるリソース使用状況（壁時計時間およびCPU時間等）を計測・記録するためのデータ構造です。

シグネチャまたは形状

typedef struct PGRUsage
{
	struct timeval tv;
	struct rusage ru;
} PGRUsage;

入力と出力

本構造体自体はデータ保持用であり、直接的な入力・出力関数を持ちません。計測には以下の関連関数を使用します。

  • pg_rusage_init(PGRUsage *ru0): 現在のシステム状態を構造体に保存します。
  • pg_rusage_show(const PGRUsage *ru0): 構造体に記録された開始時点と現在の状態の差分を計算し、フォーマットされた文字列を返します。

前提条件と不変条件

  • getrusage および gettimeofday の結果に依存します。
  • pg_rusage_show が返す文字列は静的バッファを使用するため、スレッドセーフではありません。

アルゴリズムまたは振る舞い

1. pg_rusage_init により、計測開始時の timeval（壁時計）と rusage（CPU時間等）のスナップショットを保持します。 2. 任意の処理実行後、pg_rusage_show を呼び出すことで、現在のシステム状態との差分を算出し、パフォーマンス計測結果として出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: vacuumlazy.c, xlogrecovery.c, index.c, analyze.c, cluster.c, indexcmds.c, tuplesort.c 等の各バックエンド処理。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • struct timeval (sys/time.h): 壁時計時間の保持に使用。
  • struct rusage (sys/resource.h): CPU使用時間等のリソース統計の保持に使用。

失敗モードと境界条件

  • OS の getrusage 実装に計測精度が依存します。
  • 静的バッファを使用しているため、並列実行環境ではデータ競合が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は PostgreSQL のシングルスレッドモデルを前提として設計されています。将来的な並列化の際には、pg_rusage_show の静的バッファ利用を再検討する必要があります。
