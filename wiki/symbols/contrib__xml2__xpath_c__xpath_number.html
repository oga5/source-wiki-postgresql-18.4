xpath_number

定義

  • 種別: function
  • 定義箇所: contrib/xml2/xpath.c:305

目的

XMLドキュメントに対してXPathクエリを実行し、その結果を数値（float4）として抽出するPostgreSQLのSQL呼び出し可能関数です。

シグネチャまたは形状

Datum xpath_number(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_PP(0): 対象となるXMLドキュメント（text型）。
  • PG_GETARG_TEXT_PP(1): 評価するXPath式（text型）。
  • **出力**:
  • float4型（数値として評価可能な場合）。
  • 結果が空、または数値に変換できない（NaN）場合はNULLを返す。

前提条件と不変条件

  • libxml2のXPathコンテキストが正しく初期化されている必要があります。
  • pgxml_xpathによる評価後、cleanup_workspaceを呼び出してメモリを解放する必要があります。

アルゴリズムまたは振る舞い

1. pgxml_texttoxmlcharを用いて入力XPath式をxmlChar形式に変換します。 2. pgxml_xpathを呼び出し、XMLドキュメントに対してXPath評価を実行し、結果オブジェクトを取得します。 3. xmlXPathCastToNumberを用いて、取得したXPath結果を数値にキャストします。 4. cleanup_workspaceを呼び出し、評価に使用したワークスペースを解放します。 5. 結果がNaN（非数）であるか、または評価結果がNULLの場合はSQLのNULLを返します。 6. それ以外の場合は、計算されたfloat4値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（PG_FUNCTION_INFO_V1により登録）。
  • **呼び出し先**:
  • pgxml_texttoxmlchar
  • pgxml_xpath
  • pfree
  • xmlXPathCastToNumber
  • cleanup_workspace
  • xmlXPathIsNaN

関連データ構造

  • xpath_workspace: XPath評価中のドキュメントやコンテキストを保持する管理構造体。

失敗モードと境界条件

  • XPath式が不正な場合やXMLのパースに失敗した場合、pgxml_xpath内部でPostgreSQLのelog(ERROR)が発生し、トランザクションが中断される可能性があります。
  • 評価結果が数値に変換できない場合、xmlXPathCastToNumberはNaNを返し、本関数はNULLを返します。

保守担当者向けメモ

  • 本関数はlibxml2のメモリを直接扱うため、cleanup_workspaceの呼び出し漏れはメモリリークに直結します。
  • PG_TRY/CATCHブロックの詳細はpgxml_xpath内部でカプセル化されていますが、エラー発生時のメモリ解放フローには注意してください。
