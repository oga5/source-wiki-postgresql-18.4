SearchSysCacheCopy

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/syscache.c:378

目的

システムカタログキャッシュからタプルを検索し、そのコピーを呼び出し側に返します。キャッシュ内のタプルを直接変更することなく、安全に編集可能なタプルを取得するために使用されます。

シグネチャまたは形状

HeapTuple SearchSysCacheCopy(int cacheId, Datum key1, Datum key2, Datum key3, Datum key4)

入力と出力

  • **入力**:
  • cacheId: 検索対象のキャッシュID。
  • key1 ～ key4: 検索キー（Datum型）。使用しないキーには 0 を指定します。
  • **出力**:
  • HeapTuple: 検索に成功した場合はキャッシュされたタプルのコピーを返します。検索に失敗した場合は NULL（または無効なタプル）を返します。

前提条件と不変条件

  • 呼び出し側は、返された HeapTuple を不要になった時点で heap_freetuple() を使用して解放する責任があります。
  • キャッシュ内のタプル自体は変更してはなりません。変更が必要な場合は必ず本関数を使用してコピーを取得してください。

アルゴリズムまたは振る舞い

1. SearchSysCache を呼び出し、指定されたキーでキャッシュを検索します。 2. 検索結果が有効なタプルでない場合、そのまま返します。 3. 有効なタプルが見つかった場合、heap_copytuple を使用してメモリ上にタプルのコピーを作成します。 4. ReleaseSysCache を呼び出し、キャッシュから取得した元のタプルの参照カウントを減らします。 5. 作成したコピーを呼び出し側に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/utils/syscache.h で定義されたマクロ経由で、システムカタログを参照する各バックエンド機能から呼び出されます。
  • **呼び出し先**:
  • SearchSysCache: キャッシュ検索の実行。
  • heap_copytuple: タプルのメモリコピー作成。
  • ReleaseSysCache: キャッシュエントリの参照カウント解放。

関連データ構造

  • HeapTuple: PostgreSQLのタプル表現。
  • CatCache: syscache.c 内で管理されるカタログキャッシュ構造体。

失敗モードと境界条件

  • 検索キーに一致するタプルが存在しない場合、HeapTupleIsValid が偽となるタプルが返されます。
  • メモリ不足時には heap_copytuple 内でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は、キャッシュされたタプルを呼び出し側で変更したい場合（インプレース更新ではないケース）の標準的な手段です。
  • src/include/utils/syscache.h にて、キー数に応じたラッパーマクロが提供されています。
