DictSnowball

定義

  • **種別**: struct
  • **定義箇所**: src/backend/snowball/dict_snowball.c:167

目的

PostgreSQLの全文検索（Tsearch）において、Snowballアルゴリズムを用いたステミング（語幹抽出）辞書のインスタンス状態を保持するための構造体です。

シグネチャまたは形状

typedef struct DictSnowball
{
    struct SN_env *z;
    StopList    stoplist;
    bool        needrecode;
    int         (*stem) (struct SN_env *z);
    MemoryContext dictCtx;
} DictSnowball;

入力と出力

本構造体自体はデータ保持用であり、関数ではありません。各フィールドは以下の役割を持ちます。

  • z: Snowballライブラリの実行環境（SN_env）。
  • stoplist: ストップワードリスト。
  • needrecode: ステミング前後のエンコーディング変換の要否。
  • stem: 実行するステミングアルゴリズムへの関数ポインタ。
  • dictCtx: ステミング実行時に切り替えるためのプライベートメモリコンテキスト。

前提条件と不変条件

  • Snowballライブラリが内部でメモリを保持するため、ステミング処理は必ず dictCtx に切り替えた状態で実行する必要があります。
  • 初期化関数は長寿命なコンテキストで実行されるため、dictCtx には初期化時のコンテキストが保持されます。

アルゴリズムまたは振る舞い

本構造体は、dsnowball_init で構築され、dsnowball_lexize における語幹抽出処理の実行環境として利用されます。ステミング実行時には、MemoryContextSwitchTo を用いて dictCtx へ切り替え、Snowball環境のセットアップとアルゴリズムの適用が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/snowball/dict_snowball.c 内の locate_stem_module
  • dsnowball_init
  • dsnowball_lexize
  • **呼び出し先**:
  • 構造体内部の関数ポインタ stem が、各言語固有のステミング関数を指します。

関連データ構造

  • SN_env: Snowballライブラリの環境構造体。
  • StopList: ストップワードを管理する構造体。
  • MemoryContext: PostgreSQLのメモリ管理コンテキスト。

失敗モードと境界条件

  • 1000バイトを超える入力文字列に対しては、ステマーの誤動作を防ぐためステミング処理をスキップする設計となっています。

保守担当者向けメモ

  • サーバーエンコーディングとステマーの対応関係が複雑なため、needrecode フラグによる動的な変換処理が不可欠です。
  • メモリリークを防ぐため、Snowballライブラリのメモリ管理とPostgreSQLのメモリコンテキストの整合性に注意してください。
