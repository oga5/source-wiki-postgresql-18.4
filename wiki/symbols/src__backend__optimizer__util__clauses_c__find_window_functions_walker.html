find_window_functions_walker

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/clauses.c:240

目的

式ツリーを走査し、含まれるすべての WindowFunc ノードを検索して、それらを WindowFuncLists 構造体に分類・格納する。

シグネチャまたは形状

static bool find_window_functions_walker(Node *node, WindowFuncLists *lists);

入力と出力

  • **入力**:
  • node: 走査対象の式ツリーノード。
  • lists: WindowFunc ノードを蓄積するための WindowFuncLists 構造体へのポインタ。
  • **出力**:
  • 走査を継続すべき場合は false、中断する場合は true を返す（本関数は常に false を返す）。

前提条件と不変条件

  • パーサーにより、WindowFunc の引数やフィルタ句内に別のウィンドウ関数が含まれていないことが保証されているものと仮定する。
  • lists->maxWinRef は、WindowFunc の winref フィールドの有効範囲を定義する。

アルゴリズムまたは振る舞い

1. ノードが NULL の場合は false を返す。 2. ノードが WindowFunc 型の場合:

  • winref が lists->maxWinRef を超えていないか検証し、超えている場合はエラーを投げる。
  • winref に対応する lists->windowFuncs のリストにノードを追加し、numWindowFuncs をインクリメントする。
  • 子ノード（引数やフィルタ）への再帰は行わず false を返す。

    • 3. ノードが SubLink 型でないことを確認（Assert）する。 4. expression_tree_walker を呼び出し、ツリーの残りの部分を再帰的に走査する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/clauses.c 内の find_window_functions から呼び出される。
  • **呼び出し先**: expression_tree_walker（nodes/nodeFuncs.h）を再帰的に呼び出す。

関連データ構造

  • WindowFunc: 走査対象のウィンドウ関数ノード。
  • WindowFuncLists: ウィンドウ関数を winref ごとに分類して保持する構造体。

失敗モードと境界条件

  • WindowFunc の winref が lists->maxWinRef を超えている場合、elog(ERROR) により処理を中断する。
  • SubLink ノードが検出された場合、アサーション違反が発生する。

保守担当者向けメモ

  • 本関数は expression_tree_walker を介した再帰的な走査を行う。
  • ウィンドウ関数の引数やフィルタ句を走査しない設計は、パーサーの不変条件に依存している。もし将来的にこの前提が崩れる場合は、再帰処理の追加が必要となる可能性がある。
