visibilitymap_count

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/visibilitymap.c:390

目的

リレーションの可視性マップ全体を走査し、設定されている「全可視（All-Visible）」ビットおよび「全凍結（All-Frozen）」ビットの総数をカウントします。

シグネチャまたは形状

void visibilitymap_count(Relation rel, BlockNumber *all_visible, BlockNumber *all_frozen)

入力と出力

  • **入力**:
  • rel: 対象のリレーション。
  • all_visible: カウント結果を格納するポインタ（必須）。
  • all_frozen: カウント結果を格納するポインタ（NULL許容）。
  • **出力**:
  • 引数 all_visible および all_frozen を通じて、それぞれカウントされたブロック数が返されます。

前提条件と不変条件

  • all_visible は必ず指定されている必要があります（Assert による検証あり）。
  • マップの最終ページに含まれる余剰バイトはゼロ埋めされていると仮定し、カウント対象から除外しません。

アルゴリズムまたは振る舞い

1. vm_readbuf を使用して、マップの先頭から順にページを読み込みます。 2. 読み込みに失敗（BufferIsValid が偽）するまでループを継続します。 3. 各ページに対してロックを取得せず、PageGetContents で内容を取得します。 4. pg_popcount_masked を用いて、VISIBLE_MASK8 および FROZEN_MASK8 に基づくビット数を加算します。 5. 最後に ReleaseBuffer でバッファを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/vacuumlazy.c
  • src/backend/catalog/index.c
  • src/backend/commands/analyze.c
  • **呼び出し先**:
  • vm_readbuf
  • BufferIsValid
  • BufferGetPage
  • PageGetContents
  • pg_popcount_masked
  • ReleaseBuffer

関連データ構造

  • Relation: 操作対象のリレーション情報。
  • Buffer: マップページを保持するバッファ。
  • MAPSIZE: 可視性マップの1ページあたりの有効データサイズ。

失敗モードと境界条件

  • マップが存在しない、または読み取り不能な場合はループが終了し、それまでにカウントされた値が返されます。
  • 並行してビットのセットやクリアが行われている場合、取得される値は近似値となります（ロックを取得しない設計のため）。

保守担当者向けメモ

  • 本関数はロックを取得せずに読み取りを行うため、結果は即座に古くなる可能性があります。正確な値ではなく、統計的な近似値が必要な処理での使用を想定しています。
