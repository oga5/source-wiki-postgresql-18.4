shell_finish_command

定義

  • 種別: function
  • 定義箇所: contrib/basebackup_to_shell/basebackup_to_shell.c:222

目的

バックアップストリームのアーカイブまたはマニフェスト転送に使用していた外部シェルコマンドのプロセスを終了し、パイプをクローズしてリソースを解放する。

シグネチャまたは形状

static void shell_finish_command(bbsink_shell *sink)

入力と出力

  • **入力**: bbsink_shell *sink - 実行中のコマンド情報とパイプを保持する構造体へのポインタ。
  • **出力**: なし（戻り値なし）。外部コマンドの終了ステータスが非ゼロの場合、エラーを報告してバックアップ処理を中断する。

前提条件と不変条件

  • sink->current_command が NULL でないこと。
  • sink->pipe が NULL でないこと。

アルゴリズムまたは振る舞い

1. ClosePipeStream を呼び出し、外部コマンドのパイプを閉じる。 2. パイプのクローズに失敗した場合（pclose_rc == -1）、ファイルアクセスエラーとして報告する。 3. 外部コマンドが非ゼロの終了コードを返した場合、ERRCODE_EXTERNAL_ROUTINE_EXCEPTION を発生させ、コマンド名と終了ステータスの詳細をエラーメッセージに含めて報告する。 4. 正常終了後、sink->pipe を NULL にリセットし、sink->current_command をメモリ解放した上で NULL にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • bbsink_shell_end_archive
  • bbsink_shell_end_manifest
  • （ソース内の参照箇所: contrib/basebackup_to_shell/basebackup_to_shell.c の 297行目、338行目、374行目）
  • **呼び出し先**:
  • ClosePipeStream
  • ereport
  • pfree
  • wait_result_to_str

関連データ構造

  • bbsink_shell: 外部コマンド実行状態（パイプ、現在のコマンド文字列）を保持する構造体。

失敗モードと境界条件

  • **パイプクローズ失敗**: ClosePipeStream が -1 を返した場合、システムエラーとして処理される。
  • **コマンド実行失敗**: 外部プロセスが非ゼロの終了コードを返した場合、バックアップ処理全体がエラーとして扱われる。

保守担当者向けメモ

  • 本関数は bbsink インターフェースのライフサイクル管理の一部であり、必ず shell_run_command と対で使用される。
  • 外部コマンドの終了コードは wait_result_to_str を通じて人間が読みやすい形式に変換されるため、デバッグ時にはこの出力が重要となる。
