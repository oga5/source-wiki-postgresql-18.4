ForeignRecheck

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeForeignscan.c:77

目的

EvalPlanQual（同時実行制御）処理において、外部テーブルから取得したタプルが現在も条件を満たしているかを再検証する。

シグネチャまたは形状

static bool ForeignRecheck(ForeignScanState *node, TupleTableSlot *slot)

入力と出力

  • **入力**:
  • node: 外部スキャン実行状態を保持する ForeignScanState 構造体。
  • slot: 再検証対象のタプルを保持する TupleTableSlot。
  • **出力**:
  • bool: タプルが条件を通過した場合は true、それ以外は false。

前提条件と不変条件

  • node->fdwroutine が有効な FdwRoutine を指していること。
  • node->ss.ps.ps_ExprContext が初期化されていること。

アルゴリズムまたは振る舞い

1. node から式コンテキスト（econtext）を取得し、econtext->ecxt_scantuple に検証対象の slot を設定する。 2. ResetExprContext(econtext) を呼び出し、式評価の準備を行う。 3. fdwroutine->RecheckForeignScan が定義されている場合、それを呼び出して外部ソース側での再検証を行う。この関数が false を返した場合は、直ちに false を返す。 4. 最後に ExecQual を呼び出し、node->fdw_recheck_quals に格納されたローカル条件式を評価して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeForeignscan.c 内で ExecScanRecheckMtd として登録されている。
  • **呼び出し先**:
  • fdwroutine->RecheckForeignScan (FDW固有の再検証ロジック)
  • ExecQual (ローカル条件式の評価)

関連データ構造

  • ForeignScanState: 外部スキャンの実行状態を管理する構造体。
  • FdwRoutine: 外部データソースごとの操作関数ポインタを保持する構造体。
  • TupleTableSlot: タプルデータを保持するコンテナ。

失敗モードと境界条件

  • 外部結合（Outer Join）がプッシュダウンされている場合、再検証によってタプルの内容が変更される可能性がある。
  • fdw_recheck_quals が空の場合、ExecQual は常に true を返す。

保守担当者向けメモ

  • 本関数は EvalPlanQual（EPQ）実行時のコールバックとして機能する。
  • 外部結合のプッシュダウン時は、RecheckForeignScan 内でスロットの内容が書き換わる可能性があるため注意が必要である。
