dlerror

定義

  • 種別: function
  • 定義箇所: src/port/win32dlopen.c:39

目的

Windows環境において、直近の動的リンク操作（dlopenやdlsym等）で発生したエラーメッセージを取得するために使用されます。

シグネチャまたは形状

char *dlerror(void);

入力と出力

  • **入力**: なし。
  • **出力**: エラーが発生している場合はエラー内容を示す文字列へのポインタを返し、エラーがない場合は NULL を返します。

前提条件と不変条件

  • 内部で保持される静的バッファ last_dyn_error を参照します。
  • 本関数はスレッドセーフではありません。

アルゴリズムまたは振る舞い

1. 静的変数 last_dyn_error の先頭バイトを確認します。 2. last_dyn_error[0] が非ゼロであれば、そのバッファの内容（エラーメッセージ）を返します。 3. last_dyn_error[0] がゼロであれば、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/fmgr/dfmgr.c
  • src/interfaces/libpq/fe-auth-oauth.c
  • **呼び出し先**:
  • 直接的な関数呼び出しはありませんが、内部的に静的変数 last_dyn_error を読み取ります。

関連データ構造

  • last_dyn_error: src/port/win32dlopen.c 内で定義されている静的バッファ。エラーメッセージを保持します。

失敗モードと境界条件

  • エラーが発生していない状態で呼び出した場合、NULL を返します。
  • 複数のスレッドから同時に呼び出された場合、最後に発生したエラーが上書きされる可能性があるため、スレッド間での整合性は保証されません。

保守担当者向けメモ

  • 本関数はWindows環境専用のラッパーです。POSIX環境との互換性を維持するため、src/include/port/win32_port.h で宣言されています。
  • エラーメッセージの更新は、内部ヘルパー関数 set_dl_error によって行われます。
