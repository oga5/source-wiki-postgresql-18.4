GetSharedMemName

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32_shmem.c:64

目的

Windows環境において、PostgreSQLのデータディレクトリ（DataDir）に基づいた一意な共有メモリ名を生成する。

シグネチャまたは形状

static char *GetSharedMemName(void)

入力と出力

  • **入力**: なし（グローバル変数 DataDir を参照）
  • **出力**: 生成された共有メモリ名を含む char* 型の文字列（呼び出し元で解放が必要）

前提条件と不変条件

  • DataDir が有効なパスとして設定されていること。
  • メモリ割り当て（malloc）が成功すること。

アルゴリズムまたは振る舞い

1. GetFullPathName を使用して DataDir のフルパス長を取得する。 2. Global\PostgreSQL: というプレフィックスを含めた十分なサイズのメモリを確保する。 3. GetFullPathName を再度呼び出し、確保した領域にフルパスを書き込む。 4. 文字列内のバックスラッシュ（\）をすべてフォワードスラッシュ（/）に置換する。 5. プレフィックスとして付与した Global\ 部分は、権限エラーを回避するため、ループ処理により実質的に無効化（上書き）される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/port/win32_shmem.c 内の PGSharedMemoryCreate および PGSharedMemoryReAttach
  • src/backend/storage/ipc/dsm_impl.c（ツールによる参照の可能性あり）
  • **呼び出し先**:
  • GetFullPathName (Windows API)
  • malloc
  • strcpy
  • elog

関連データ構造

  • DataDir: PostgreSQLのデータディレクトリパスを保持するグローバル変数。

失敗モードと境界条件

  • GetFullPathName が失敗した場合、またはメモリ確保に失敗した場合は FATAL エラーとしてプロセスを終了する。
  • パス名に \ が含まれる場合、すべて / に変換される。

保守担当者向けメモ

  • 現在、Global\ 名前空間を使用すると権限エラーが発生するため、デフォルトの名前空間を使用するように意図的にコードが修正されている。この挙動はソース内のコメントに明記されている。
