pgaio_io_start_readv

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_io.c:77

目的

非同期I/O（AIO）サブシステムにおいて、読み込み操作（readv）を開始するための標準的なエントリーポイントを提供します。

シグネチャまたは形状

void pgaio_io_start_readv(PgAioHandle *ioh, int fd, int iovcnt, uint64 offset)

入力と出力

  • **入力**:
  • ioh: 操作対象の PgAioHandle 構造体へのポインタ。
  • fd: 読み込み対象のファイル記述子。
  • iovcnt: iovec の要素数。
  • offset: 読み込みを開始するファイル内のオフセット。
  • **出力**: なし（void）。

前提条件と不変条件

  • pgaio_io_before_start(ioh) が内部で呼び出され、割り込み禁止状態の確認などのガード条件が強制されます。
  • 呼び出し時に PgAioHandle が適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. pgaio_io_before_start を呼び出し、I/O開始前の状態チェックを実行します。 2. 引数として渡された fd、offset、iovcnt を ioh->op_data.read 構造体に格納します。 3. pgaio_io_stage を呼び出し、操作種別 PGAIO_OP_READV を指定してI/Oサブシステムへ処理を登録します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/file/fd.c:2241 からの呼び出しが確認されています。
  • **呼び出し先**:
  • pgaio_io_before_start
  • pgaio_io_stage

関連データ構造

  • PgAioHandle: I/O操作の状態とパラメータを保持する主要な構造体。

失敗モードと境界条件

  • pgaio_io_before_start 内の不変条件（Assert(!INTERRUPTS_CAN_BE_PROCESSED())）に違反した場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数はI/Oの「準備」と「実行」の境界を定義しています。
  • pgaio_io_before_start 内の割り込みチェックは、I/O操作中にファイル記述子が閉じられることを防ぐための重要な安全策です。
