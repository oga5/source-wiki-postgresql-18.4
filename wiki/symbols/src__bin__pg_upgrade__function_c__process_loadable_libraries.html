process_loadable_libraries

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/function.c:59

目的

旧クラスタから取得した共有ライブラリ情報のクエリ結果（PGresult）を、アップグレード処理中の状態管理構造体（loadable_libraries_state）に格納するコールバック関数です。

シグネチャまたは形状

static void process_loadable_libraries(DbInfo *dbinfo, PGresult *res, void *arg)

入力と出力

  • dbinfo: 現在処理中のデータベース情報へのポインタ。
  • res: データベースから取得されたライブラリ情報を含む PGresult。
  • arg: loadable_libraries_state 構造体へのポインタ（void * として渡される）。
  • 出力: なし（副作用として state 構造体の更新を行う）。

前提条件と不変条件

  • arg は loadable_libraries_state 型としてキャスト可能であること。
  • process_loadable_libraries が UpgradeTaskProcessCB 型と一致していることを AssertVariableIsOfType で検証している。

アルゴリズムまたは振る舞い

1. arg を loadable_libraries_state 型にキャストする。 2. dbinfo と old_cluster.dbarr.dbs の差分からインデックスを算出し、state->ress 配列の該当位置に res を格納する。 3. res に含まれるタプル数（PQntuples(res)）を state->totaltups に加算する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_upgrade/function.c 内の get_loadable_libraries 関数において、upgrade_task_add_step の引数として渡される。
  • 呼び出し先: libpq の PQntuples を呼び出す。

関連データ構造

  • DbInfo: データベース情報を保持する構造体。
  • PGresult: libpq のクエリ結果構造体。
  • loadable_libraries_state: ライブラリ収集プロセス中の状態を保持する構造体。

失敗モードと境界条件

  • dbinfo が old_cluster.dbarr.dbs の範囲外である場合、配列アクセスによるメモリ破壊の可能性がある。
  • res が NULL の場合、PQntuples の呼び出しでクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数は upgrade_task のステップとして実行されるコールバックです。
  • state->ress 配列のインデックス計算は old_cluster の構造に依存しているため、クラスタ情報の管理方法に変更があった場合は注意が必要です。
