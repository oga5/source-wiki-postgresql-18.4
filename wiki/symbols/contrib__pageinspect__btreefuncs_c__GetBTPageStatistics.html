GetBTPageStatistics

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/btreefuncs.c:107

目的

B-treeインデックスの特定のページを解析し、その統計情報（ライブ/デッドタプル数、空き容量、ページタイプ等）を BTPageStat 構造体に格納する。

シグネチャまたは形状

static void GetBTPageStatistics(BlockNumber blkno, Buffer buffer, BTPageStat *stat)

入力と出力

  • **入力**:
  • blkno: 対象のブロック番号。
  • buffer: 解析対象のページを含むバッファ。
  • stat: 統計情報を書き込むための BTPageStat 構造体へのポインタ。
  • **出力**: なし（stat 引数を通じて結果を返す）。

前提条件と不変条件

  • buffer は有効なB-treeインデックスページを指している必要がある。
  • stat は呼び出し元によってメモリ確保済みであること。

アルゴリズムまたは振る舞い

1. ページヘッダーおよび不透明領域（opaque）から基本情報を抽出する。 2. ページタイプ（削除済み、無視、リーフ、ルート、内部）を判定し、stat->type に設定する。 3. ページ内の各アイテムを走査し、以下の処理を行う:

  • IndexTupleSize を用いてアイテムサイズを累積。
  • ItemIdIsDead を用いてライブタプルとデッドタプルをカウント。

    • 4. PageGetFreeSpace で空き容量を取得する。 5. 最後に平均アイテムサイズを算出し、stat に格納する。 6. 削除済みページの場合は、FullTransactionId（safexid）の情報をデバッグログに出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pageinspect/btreefuncs.c 内の関数（bt_page_stats_internal 等の統計情報取得関数から呼び出される）。
  • **呼び出し先**:
  • PostgreSQLの内部マクロおよび関数（BufferGetPage, PageGetMaxOffsetNumber, BTPageGetOpaque, PageGetItemId, PageGetItem, IndexTupleSize, PageGetFreeSpace 等）。

関連データ構造

  • BTPageStat: ページ統計情報を保持する構造体。
  • BTPageOpaqueData: B-treeページ固有のメタデータ構造。
  • IndexTuple: インデックス内のタプルデータ。

失敗モードと境界条件

  • ページが破損している場合、PageGetMaxOffsetNumber や PageGetItem の呼び出しで予期せぬ動作が発生する可能性がある。
  • ライブタプルとデッドタプルの合計が0の場合、ゼロ除算を避けるため avg_item_size は0に設定される。

保守担当者向けメモ

  • 本関数はスーパーユーザー権限での実行を前提としたモジュールの一部である。
  • 削除済みページの safexid 処理において、pg_upgrade 互換性のための古い表現（btpo_level フィールドの再利用）を考慮したロジックが含まれている。
