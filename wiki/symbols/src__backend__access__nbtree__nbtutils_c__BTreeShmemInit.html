BTreeShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtutils.c:3653

目的

B-tree インデックスのバキューム状態を管理するための共有メモリ領域を確保および初期化する。

シグネチャまたは形状

void BTreeShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上の btvacinfo が初期化される）

前提条件と不変条件

  • ShmemInitStruct を使用して共有メモリを割り当てるため、PostgreSQL の共有メモリ初期化プロセスの一部として呼び出される必要がある。
  • IsUnderPostmaster の状態に基づき、Postmaster プロセス（初期化側）かバックエンドプロセス（参照側）かで動作が分岐する。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を呼び出し、"BTree Vacuum State" という名前で BTreeShmemSize() 分の共有メモリを確保する。 2. IsUnderPostmaster が偽（Postmaster 起動時）の場合:

  • 共有メモリが未初期化であることを確認する。
  • btvacinfo->cycle_ctr を現在時刻（time(NULL)）でシード初期化する。
  • num_vacuums を 0 に、max_vacuums を MaxBackends に設定する。

    • 3. IsUnderPostmaster が真（バックエンド起動時）の場合:

  • 共有メモリが既に存在していることを確認する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:341
  • 呼び出し先: ShmemInitStruct, BTreeShmemSize, time

関連データ構造

  • BTVacInfo: 共有メモリ上に配置されるバキューム状態管理構造体。
  • BTCycleId: cycle_ctr の型。

失敗モードと境界条件

  • IsUnderPostmaster が偽の状態で共有メモリが既に存在していた場合、Assert(!found) により停止する。
  • IsUnderPostmaster が真の状態で共有メモリが見つからなかった場合、Assert(found) により停止する。

保守担当者向けメモ

  • 本関数はシステム起動時の共有メモリ初期化シーケンスに組み込まれている。
  • cycle_ctr の初期値に time(NULL) を使用しているのは、起動ごとに異なる値をシードとして設定するためである。
