QueryItem

定義

  • 種別: typedef (共用体)
  • 定義箇所: src/include/tsearch/ts_type.h:208

目的

PostgreSQL の全文検索機能における tsquery 構造体内部の各ノードを表現するための共用体です。クエリツリーを構成する演算子またはオペランドのいずれかを保持します。

シグネチャまたは形状

typedef union
{
    QueryItemType type;
    QueryOperator qoperator;
    QueryOperand qoperand;
} QueryItem;

入力と出力

  • **入力**: tsquery 構造体内のバイナリデータとして配置されます。
  • **出力**: type メンバを参照することで、ノードが演算子（QueryOperator）かオペランド（QueryOperand）かを判別可能です。

前提条件と不変条件

  • tsquery は 4 バイトアライメントを前提としています。
  • QueryItem は tsquery 内で連続した配列として保持されることが一般的です。

アルゴリズムまたは振る舞い

  • type フィールドにより、共用体内のどのメンバが有効であるかを識別します。
  • クエリの走査や評価を行う際、この型を介してノードの種類に応じた処理（演算子の評価やオペランドの照合）へ分岐します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/tsearch/ts_parse.c
  • src/backend/tsearch/ts_selfuncs.c
  • src/backend/utils/adt/tsginidx.c
  • src/backend/utils/adt/tsquery.c
  • **呼び出し先**:
  • QueryItemType（型判別用）
  • QueryOperator（演算子データ）
  • QueryOperand（オペランドデータ）

関連データ構造

  • TSQueryData / TSQuery: QueryItem の配列を保持する上位構造体。
  • QueryItemType: ノードの種類を定義する列挙型。
  • QueryOperator: 演算子ノードの構造体。
  • QueryOperand: オペランドノードの構造体。

失敗モードと境界条件

  • 不正な type 値が設定されている場合、tsquery の処理関数（例: tsquery.c 内の findoprnd 等）でエラーが発生します。
  • tsquery のメモリレイアウトを直接操作する場合、アライメントや varlena ヘッダーの整合性に注意が必要です。

保守担当者向けメモ

  • QueryItem は tsquery のバイナリ表現の核となるため、構造の変更は tsquery を扱うすべてのインデックスおよび検索ロジックに影響を与えます。
  • 構造体のサイズ計算には、必ず ts_type.h で提供される専用マクロを使用してください。
