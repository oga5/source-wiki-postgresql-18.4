calc_hist_selectivity_contains

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/multirangetypes_selfuncs.c:1251

目的

ヒストグラム統計情報を用いて、指定されたクエリ範囲が対象の範囲（またはマルチレンジ）を「含む（@>）」演算子に対する選択率を推定します。

シグネチャまたは形状

double calc_hist_selectivity_contains(TypeCacheEntry *typcache,
                                      const RangeBound *lower, const RangeBound *upper,
                                      const RangeBound *hist_lower, int hist_nvalues,
                                      Datum *length_hist_values, int length_hist_nvalues);

入力と出力

  • **入力**:
  • typcache: 型情報のキャッシュ。
  • lower, upper: クエリ範囲の下限および上限。
  • hist_lower: 境界値ヒストグラムの配列。
  • hist_nvalues: ヒストグラムの要素数。
  • length_hist_values, length_hist_nvalues: レンジ長ヒストグラムとその要素数。
  • **出力**:
  • double: 推定された選択率（0.0 から 1.0 の範囲）。

前提条件と不変条件

  • hist_lower は昇順にソートされている必要があります。
  • typcache は対象の型に対して有効である必要があります。
  • 統計情報の欠如や範囲外の入力に対しては、安全なデフォルト値（0.0など）を返す設計となっています。

アルゴリズムまたは振る舞い

1. **ビン特定**: rbound_bsearch を使用し、クエリ下限値が含まれるヒストグラムのビンを特定します。 2. **境界外処理**: クエリ下限値がヒストグラムの範囲外である場合は 0.0 を返します。 3. **補間計算**: 特定したビン内でのクエリ下限値の位置を get_position で算出し、重み付けを行います。 4. **累積計算**: クエリ下限値より小さいヒストグラムの各ビンについて、get_distance でクエリ上限値までの距離を求めます。 5. **積分推定**: calc_length_hist_frac を用いて、各ビンにおけるレンジ長分布に基づいた確率を算出し、累積加算して最終的な選択率を算出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/multirangetypes_selfuncs.c 内の統計計算ロジック。
  • src/backend/utils/adt/rangetypes_selfuncs.c 内の統計計算ロジック。
  • **呼び出し先**:
  • rbound_bsearch: ヒストグラムの二分探索。
  • get_position: ビン内での相対位置計算。
  • get_distance: 範囲境界間の距離計算。
  • calc_length_hist_frac: レンジ長ヒストグラムに基づく累積分布計算。

関連データ構造

  • RangeBound: 範囲の境界値を保持する構造体。
  • TypeCacheEntry: 型の比較関数やサブタイプ情報を含むキャッシュ構造体。
  • pg_statistic: ヒストグラムデータが格納されるシステムカタログ。

失敗モードと境界条件

  • lower_index < 0: クエリ下限値がヒストグラムの最小値より小さい場合、選択率は 0.0 となります。
  • lower_index >= hist_nvalues - 1: クエリ下限値がヒストグラムの最大値を超える場合、最後のビンを基準に計算されます。

保守担当者向けメモ

  • 本関数はレンジ型とマルチレンジ型の両方で共有されています。
  • 統計的な仮定として「レンジの長さ」と「下限値」の独立性を前提としています。
  • 精度向上のためには、ANALYZE によって生成されるヒストグラムの品質が重要です。
