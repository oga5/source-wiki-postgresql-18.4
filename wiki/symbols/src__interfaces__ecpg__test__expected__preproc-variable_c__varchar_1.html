varchar_1

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-variable.c:80

目的

ECPG（Embedded SQL for C）において、可変長文字列（VARCHAR）を表現するためのデータ構造です。

シグネチャまたは形状

struct varchar_1 {
    int len;
    char arr[BUFFERSIZ];
};

入力と出力

  • 入力: SQL文の実行結果としてデータベースから取得される文字列データ。
  • 出力: C言語のホスト変数として、文字列の長さ（len）と内容（arr）を保持。

前提条件と不変条件

  • BUFFERSIZ で定義された固定長のバッファサイズを保持します。
  • ECPGプリプロセッサによって生成されるコードにおいて、SQLの VARCHAR 型とマッピングされます。

アルゴリズムまたは振る舞い

ECPGのランタイムライブラリ（ecpglib）が、データベースから取得した文字列をこの構造体の arr に格納し、その長さを len に設定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/test/expected/preproc-cursor.c 内の複数の箇所（行692, 704, 719, 734, 752, 767, 777, 795, 810）で、ECPGt_varchar 型として参照されています。
  • 呼び出し先: 直接的な関数呼び出しはありませんが、ECPGライブラリのデータバインド処理において利用されます。

関連データ構造

  • varchar_2, varchar_3, varchar_4, varchar_5: 同様の目的で定義された他の可変長文字列構造体。

失敗モードと境界条件

  • BUFFERSIZ を超える文字列がデータベースから返された場合、バッファオーバーフローのリスクがあります（ECPGライブラリ側の実装に依存）。
