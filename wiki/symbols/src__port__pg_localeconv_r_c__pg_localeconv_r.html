pg_localeconv_r

定義

  • src/port/pg_localeconv_r.c:230

目的

標準 C ライブラリの localeconv() が持つスレッドセーフではないという制約を回避し、指定されたロケールカテゴリ（LC_MONETARY および LC_NUMERIC）に基づいた struct lconv を安全に取得する。

シグネチャまたは形状

int pg_localeconv_r(const char *lc_monetary, const char *lc_numeric, struct lconv *output);

入力と出力

  • **入力**:
  • lc_monetary: LC_MONETARY カテゴリに使用するロケール名。
  • lc_numeric: LC_NUMERIC カテゴリに使用するロケール名。
  • output: 結果を格納する struct lconv へのポインタ。
  • **出力**:
  • 成功時は 0 を返す。
  • 失敗時は -1 を返す。

前提条件と不変条件

  • output が指すメモリ領域は呼び出し元によって確保されている必要がある。
  • pg_localeconv_r() が確保した struct lconv 内の文字列メンバは動的メモリであるため、呼び出し元は後続の処理で pg_localeconv_free() を呼び出して解放しなければならない。

アルゴリズムまたは振る舞い

プラットフォームに応じて以下の手法でスレッドセーフ性を確保する。

  • **Windows**: _configthreadlocale(_ENABLE_PER_THREAD_LOCALE) を使用してスレッドローカルモードに切り替え、一時的にロケールを変更して情報を取得後、元の状態に復元する。
  • **POSIX (現代的環境)**: newlocale() で作成した locale_t オブジェクトを使用し、localeconv_l() または pg_localeconv_from_langinfo() を経由して情報を抽出する。
  • **POSIX (レガシー環境)**: pthread_mutex_t によるグローバルロックを使用して localeconv() を保護する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/pg_locale.c からの呼び出しが確認されている。
  • **呼び出し先**: pg_localeconv_copy_members()、localeconv()、localeconv_l()、newlocale()、freelocale()、uselocale() 等を環境に応じて呼び出す。

関連データ構造

  • struct lconv: ロケール情報を保持する標準構造体。
  • lconv_member_info: 構造体メンバのコピー管理用メタデータ。

失敗モードと境界条件

  • ロケールの生成（newlocale）に失敗した場合、-1 を返す。
  • メモリ確保（strdup 等）に失敗した場合、処理を中断しエラーを返す。
  • POSIX 環境で LC_CTYPE のエンコーディング不一致を避けるため、全カテゴリを対象としたロケール適用を行う。

保守担当者向けメモ

  • 本関数が返す文字列メンバは動的確保されるため、メモリリークを防ぐために pg_localeconv_free() の呼び出しが必須である。
  • localeconv_l() が利用できない環境ではグローバルロックが使用されるため、パフォーマンス上のボトルネックとなる可能性がある。
