crc32_bytea

定義

  • 種別: function
  • 定義箇所: src/backend/utils/hash/pg_crc.c:106

目的

SQLインターフェースを通じて、bytea 型のデータに対する従来のCRC-32チェックサムを計算します。

シグネチャまたは形状

Datum crc32_bytea(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される bytea 型のデータ。
  • **出力**: int64 型として返される計算済みのCRC-32値。

前提条件と不変条件

  • 入力データは bytea 型として正当である必要があります。
  • 従来のCRC-32アルゴリズムを使用するため、新規開発用途ではなく、既存のディスク構造との互換性維持を目的として使用されます。

アルゴリズムまたは振る舞い

1. INIT_TRADITIONAL_CRC32 マクロにより、CRCアキュムレータを 0xFFFFFFFF で初期化します。 2. COMP_TRADITIONAL_CRC32 マクロを用い、入力データのバイト列に対して pg_crc32_table を参照しながらビット演算（XORとシフト）を行い、CRC値を更新します。 3. FIN_TRADITIONAL_CRC32 マクロにより、最終的なビット反転処理を行います。 4. PG_RETURN_INT64 を介して結果を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（SQL関数として呼び出される）。
  • **呼び出し先**: INIT_TRADITIONAL_CRC32, COMP_TRADITIONAL_CRC32, FIN_TRADITIONAL_CRC32 などのマクロ（src/include/utils/pg_crc.h で定義）。
  • **注記**: 具体的な呼び出し元のコードパスは、提供されたソース情報からは特定できません。

関連データ構造

  • pg_crc32_table: CRC-32計算に使用される256要素のルックアップテーブル。
  • bytea: PostgreSQLの可変長バイナリデータ型。

失敗モードと境界条件

  • 入力データが空の場合の挙動は、使用されるマクロの初期化および終了処理の仕様に依存します。

保守担当者向けメモ

  • 本関数はレガシーなCRC-32アルゴリズムを実装しています。新しい機能や高いエラー検出能力が求められる用途では、crc32c_bytea（CRC-32C/Castagnoli）の使用を推奨してください。
