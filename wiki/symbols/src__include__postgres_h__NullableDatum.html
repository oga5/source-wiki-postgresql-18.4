NullableDatum

定義

  • 種別: struct
  • 定義箇所: src/include/postgres.h:77

目的

PostgreSQL の汎用データ型である Datum と、その値が NULL であるかを示すフラグをペアで保持し、空間効率と局所性を向上させるために使用されます。

シグネチャまたは形状

typedef struct NullableDatum
{
    Datum       value;   /* 値 */
    bool        isnull;  /* NULLフラグ */
} NullableDatum;

入力と出力

  • 入力: Datum 型の値および bool 型の NULL 状態。
  • 出力: 構造体としてパッケージ化されたデータ。

前提条件と不変条件

  • value フィールドは Datum 型であり、sizeof(void *) と同じサイズであると仮定されます。
  • isnull が true の場合、value の内容は不定または無視されるべきです。

アルゴリズムまたは振る舞い

  • 実行エンジン内でのデータ受け渡しにおいて、値と NULL 状態を分離せずに単一のメモリブロックとして扱うことで、キャッシュ効率を高めます。
  • 構造体内のパディング領域は、将来的にフラグ等として利用可能な余地があります。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/catalog/index.c: AppendAttributeTuples 関数等でインデックス属性の処理に使用。
  • src/backend/executor/execExpr.c: 式評価時の引数や結果の保持に使用。
  • src/backend/executor/execExprInterp.c: fcinfo->args を介した関数呼び出し引数の保持に使用。
  • 呼び出し先: 構造体定義であるため、特定の関数を呼び出すことはありません。

関連データ構造

  • Datum: NullableDatum の value フィールドとして格納される汎用データ型。

失敗モードと境界条件

  • メモリ割り当て: palloc 等で配列として確保する場合、サイズ計算に sizeof(NullableDatum) を使用する必要があります。
  • アライメント: 構造体として定義されているため、プラットフォームごとのアライメント規則に従います。

保守担当者向けメモ

  • 本構造体はバックエンドの実行エンジン全体で頻繁に使用されるため、フィールドの追加や変更はメモリレイアウトに大きな影響を与えます。
  • isnull フラグの後にパディングが存在するため、構造体のサイズを増やすことなくフラグを追加できる可能性があります。
