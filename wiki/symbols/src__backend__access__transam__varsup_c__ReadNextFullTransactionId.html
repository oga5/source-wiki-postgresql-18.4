ReadNextFullTransactionId

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/varsup.c:287

目的

共有メモリ上の TransamVariables から、次に割り当てられるべき FullTransactionId を読み取ります。

シグネチャまたは形状

FullTransactionId ReadNextFullTransactionId(void)

入力と出力

  • 入力: なし
  • 出力: 現在の nextXid を保持する FullTransactionId 型の値。

前提条件と不変条件

  • 共有メモリ上の TransamVariables が初期化されている必要があります。
  • 呼び出し元は、XidGenLock を保持する必要はありません（関数内部で取得・解放を行うため）。

アルゴリズムまたは振る舞い

1. XidGenLock を共有モード（LW_SHARED）で取得します。 2. 共有メモリ構造体 TransamVariables 内の nextXid をローカル変数にコピーします。 3. XidGenLock を解放します。 4. コピーした FullTransactionId を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistvacuum.c
  • src/backend/access/nbtree/nbtpage.c
  • src/backend/access/transam/twophase.c
  • src/backend/replication/walreceiver.c
  • src/backend/replication/walsender.c
  • src/backend/storage/ipc/standby.c
  • src/backend/utils/adt/xid8funcs.c
  • **呼び出し先**:
  • LWLockAcquire
  • LWLockRelease

関連データ構造

  • FullTransactionId: 64bitのトランザクションIDを保持する型。
  • TransamVariables: ID生成状態を保持する共有メモリ構造体。
  • XidGenLock: トランザクションID生成を保護する軽量ロック。

失敗モードと境界条件

  • ロック取得時に他のプロセスとの競合が発生する可能性がありますが、LW_SHARED モードのため、読み取り操作自体は並列実行可能です。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、ロックの保持時間は最小限に抑えられています。
  • 64bitの FullTransactionId を扱うため、32bit XIDとの変換が必要な箇所では注意が必要です。
