SignalHandlerForShutdownRequest

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/interrupt.c:103

目的

PostgreSQLのバックグラウンドプロセスにおいて、シャットダウン要求シグナル（SIGTERM, SIGINT, SIGUSR2等）を受信した際に、プロセスを安全に終了させるための準備を行う。

シグネチャまたは形状

void SignalHandlerForShutdownRequest(SIGNAL_ARGS)

※ SIGNAL_ARGS はシグナルハンドラの標準的な引数定義に展開されるマクロ。

入力と出力

  • 入力: シグナルハンドラとしてOSから呼び出される（引数は SIGNAL_ARGS）。
  • 出力: なし（void）。

前提条件と不変条件

  • シグナルハンドラ内での非同期安全性を維持するため、本関数内ではフラグ設定とラッチ通知のみを行う。
  • MyLatch が現在のプロセスに対して適切に初期化されている必要がある。

アルゴリズムまたは振る舞い

1. グローバル変数 ShutdownRequestPending を true に設定し、シャットダウン要求があったことを記録する。 2. SetLatch(MyLatch) を呼び出し、待機状態にあるメインループを即座に起床させる。これにより、メインループは次回の反復で ProcessMainLoopInterrupts() を通じてシャットダウン処理を実行する。

呼び出し元と呼び出し先

  • 呼び出し元: OSのシグナル配送機構。各バックグラウンドプロセス（autovacuum, bgwriter, checkpointer, pgarch, walsummarizer, walwriter, applyparallelworker 等）にて pqsignal を介して登録される。
  • 呼び出し先: SetLatch（src/backend/storage/ipc/latch.c）。

関連データ構造

  • ShutdownRequestPending: シャットダウン要求の有無を示す sig_atomic_t 型のフラグ。
  • MyLatch: 現在のプロセスの待機状態を管理するラッチ構造体。

失敗モードと境界条件

  • 本関数自体は失敗しない設計となっている。
  • シグナルが連続して送られた場合、ShutdownRequestPending は重複して true に設定されるが、動作上の問題はない。

保守担当者向けメモ

  • 本関数はシグナルハンドラであるため、複雑な処理やメモリ確保、ロック取得は行わないこと。
  • 実際のシャットダウン処理（proc_exit 等）は、メインループ内の ProcessMainLoopInterrupts() がフラグを検知した後に実行される。
