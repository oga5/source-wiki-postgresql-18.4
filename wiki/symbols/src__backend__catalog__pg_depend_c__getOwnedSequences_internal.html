getOwnedSequences_internal

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_depend.c:949

目的

指定されたリレーション（テーブル）またはその特定の列に依存しているシーケンスの OID リストを取得します。

シグネチャまたは形状

List *getOwnedSequences_internal(Oid relid, AttrNumber attnum, char deptype)

入力と出力

  • **入力**:
  • relid: 依存先リレーションの OID。
  • attnum: 依存先列番号（0 の場合はテーブル全体を対象とする）。
  • deptype: フィルタリングする依存関係の型（DEPENDENCY_AUTO や DEPENDENCY_INTERNAL 等）。0 の場合は型による制限を行わない。
  • **出力**:
  • 該当するシーケンスの OID を格納した List*。

前提条件と不変条件

  • pg_depend システムカタログに対する AccessShareLock を保持して実行されます。
  • 依存関係の検索には DependReferenceIndexId インデックスを使用します。

アルゴリズムまたは振る舞い

1. pg_depend を AccessShareLock で開きます。 2. refclassid（RelationRelationId）および refobjid（relid）をキーとしてスキャンを開始します。attnum が指定されている場合は refobjsubid もキーに加えます。 3. スキャン結果を順次走査し、以下の条件を満たすレコードを抽出します：

  • classid が RelationRelationId であること。
  • objsubid が 0 であること（シーケンス自身を指す）。
  • refobjsubid が 0 ではないこと（列への依存）。
  • 依存タイプが DEPENDENCY_AUTO または DEPENDENCY_INTERNAL であること。
  • 対象オブジェクトが RELKIND_SEQUENCE であること。
  • deptype 引数が指定されている場合、その型と一致すること。

    • 4. 条件に合致したシーケンスの OID をリストに追加し、最後にスキャンを終了してリレーションを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_depend.c 内の関数から呼び出されています。
  • **呼び出し先**:
  • table_open, systable_beginscan, systable_getnext, systable_endscan, table_close 等のカタログ操作関数。
  • get_rel_relkind（リレーション種別の判定）。

関連データ構造

  • pg_depend: 依存関係を保持するシステムカタログ。
  • List: OID を格納する PostgreSQL のリスト構造。

失敗モードと境界条件

  • attnum が 0 の場合、テーブル全体に紐付くシーケンスを検索します。
  • 該当するシーケンスが存在しない場合、空のリスト（NIL）を返します。

保守担当者向けメモ

  • 本関数は pg_depend の内部的な検索ロジックをカプセル化しています。
  • インデックス DependReferenceIndexId を利用しているため、pg_depend のスキーマ変更時にはインデックス定義との整合性に注意してください。
