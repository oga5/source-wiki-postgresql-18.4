make_rels_by_clause_joins

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/joinrels.c:279

目的

指定されたリレーション（old_rel）と、候補リスト（other_rels）内の各リレーションを比較し、結合条件または結合順序の制約が存在する場合に、それらの結合リレーションを構築する。

シグネチャまたは形状

static void make_rels_by_clause_joins(PlannerInfo *root, RelOptInfo *old_rel, List *other_rels, int first_rel_idx)

入力と出力

  • **入力**:
  • root: クエリ全体のプランナ情報。
  • old_rel: 結合の起点となるリレーション。
  • other_rels: 結合候補となるリレーションのリスト。
  • first_rel_idx: other_rels リスト内で走査を開始するインデックス。
  • **出力**: なし（副作用として結合リレーションの構築を行う）。

前提条件と不変条件

  • old_rel と other_rels 内の各リレーションは、互いに重複するリレーションID（relids）を含んでいないこと。
  • make_join_rel が呼び出されるためには、have_relevant_joinclause または have_join_order_restriction のいずれかが真である必要がある。

アルゴリズムまたは振る舞い

1. first_rel_idx から開始し、other_rels リストを順次走査する。 2. 各 other_rel に対して以下の条件を判定する:

  • old_rel と other_rel の relids が重複していないこと（bms_overlap が偽）。
  • 結合条件が存在する（have_relevant_joinclause）か、または結合順序の制約が存在する（have_join_order_restriction）。

    • 3. 上記条件を満たす場合、make_join_rel を呼び出して結合リレーションを構築する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/joinrels.c 内の結合探索ロジック（join_search_one_level 等から呼び出される）。
  • **呼び出し先**:
  • bms_overlap
  • have_relevant_joinclause
  • have_join_order_restriction
  • make_join_rel

関連データ構造

  • PlannerInfo: クエリ全体の最適化状態を保持。
  • RelOptInfo: 結合対象となるリレーションのメタデータ。

失敗モードと境界条件

  • other_rels が空、または first_rel_idx がリストの範囲外である場合、ループは実行されず何も行われない。
  • bms_overlap によりリレーションの重複が検出された場合、そのペアはスキップされる。

保守担当者向けメモ

  • 本関数は、結合条件（JOIN 句など）に基づいた結合探索の主要なループを構成する。
  • 結合条件がない場合の探索については、別途 make_rels_by_clauseless_joins が担当する。
