write_eventlog

定義

  • 種別: function
  • 定義箇所: src/bin/pg_ctl/pg_ctl.c:166

目的

Windows イベントログサービスに対して、指定されたメッセージを記録します。

シグネチャまたは形状

static void write_eventlog(int level, const char *line);

入力と出力

  • level: イベントの重大度（EVENTLOG_INFORMATION_TYPE や EVENTLOG_ERROR_TYPE など）。
  • line: 記録するメッセージ文字列。
  • 出力: なし（戻り値なし）。

前提条件と不変条件

  • Windows 環境での実行を前提としています。
  • evtHandle は静的変数として保持され、初回呼び出し時に RegisterEventSource を使用して初期化されます。

アルゴリズムまたは振る舞い

1. silent_mode が有効かつ level が EVENTLOG_INFORMATION_TYPE である場合、何も記録せずに終了します。 2. evtHandle が未初期化（INVALID_HANDLE_VALUE）の場合、RegisterEventSource を呼び出し、イベントソースを登録します。登録に失敗した場合は処理を中断します。 3. ReportEvent を呼び出し、指定された level とメッセージ line をイベントログに書き込みます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_ctl/pg_ctl.c 内の各所（サーバー起動待機時のメッセージ出力やエラー報告など）。
  • 呼び出し先: Windows API の RegisterEventSource および ReportEvent。
  • 注記: src/backend/utils/error/elog.c にも同名の関数が存在しますが、シグネチャが異なり、本シンボルとは別の実装です。

関連データ構造

  • evtHandle: イベントログソースへのハンドルを保持する静的変数。

失敗モードと境界条件

  • RegisterEventSource が失敗した場合、evtHandle は INVALID_HANDLE_VALUE に維持され、以降の呼び出しでもログ出力は行われません。
  • silent_mode 設定により、情報レベルのログ出力が抑制される場合があります。

保守担当者向けメモ

  • 本関数は pg_ctl の Windows 版におけるログ出力の抽象化レイヤーです。
  • event_source グローバル変数が設定されていない場合は DEFAULT_EVENT_SOURCE が使用されます。
