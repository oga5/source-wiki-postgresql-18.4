LWLockInitialize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lwlock.c:718

目的

共有メモリ上に確保された LWLock 構造体を初期化し、ロックとして利用可能な状態にする。

シグネチャまたは形状

void LWLockInitialize(LWLock *lock, int tranche_id)

入力と出力

  • **入力**:
  • lock: 初期化対象となる LWLock 構造体へのポインタ。
  • tranche_id: このロックが属するトランチ（Tranche）の識別子。
  • **出力**: なし。

前提条件と不変条件

  • lock ポインタは有効なメモリ領域を指している必要がある。
  • 本関数は、ロックが他のプロセスから使用される前に呼び出されなければならない。

アルゴリズムまたは振る舞い

1. lock->state を LW_FLAG_RELEASE_OK でアトミックに初期化する。 2. LOCK_DEBUG が定義されている場合、lock->nwaiters を 0 に初期化する。 3. lock->tranche に tranche_id を設定する。 4. proclist_init を呼び出し、ロック待ちプロセスリストを初期化する。

関連データ構造

  • LWLock: ロックの本体。
  • proclist: 待機プロセスを管理するリスト構造。

失敗モードと境界条件

  • 特筆すべき失敗モードはない。引数が無効なポインタである場合、未定義動作を引き起こす。

保守担当者向けメモ

  • 本関数はロックの初期化のみを行う。ロックの取得や解放を行う場合は、LWLockAcquire や LWLockRelease を使用すること。
