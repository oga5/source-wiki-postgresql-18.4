HeapCheckContext

定義

  • **種別**: struct
  • **定義箇所**: contrib/amcheck/verify_heapam.c:86

目的

PostgreSQL のヒープリレーション検証プロセスにおいて、検証対象のリレーション情報、トランザクション状態のキャッシュ、および現在走査中のページ・タプル・属性のコンテキストを保持するために使用されます。

シグネチャまたは形状

typedef struct HeapCheckContext {
    /* トランザクション/マルチトランザクションIDのキャッシュ */
    FullTransactionId next_fxid;
    TransactionId next_xid;
    TransactionId oldest_xid;
    FullTransactionId oldest_fxid;
    TransactionId safe_xmin;
    MultiXactId next_mxact;
    MultiXactId oldest_mxact;
    TransactionId cached_xid;
    XidCommitStatus cached_status;

    /* ヒープリレーション情報 */
    Relation rel;
    TransactionId relfrozenxid;
    FullTransactionId relfrozenfxid;
    TransactionId relminmxid;
    Relation toast_rel;
    Relation *toast_indexes;
    Relation valid_toast_index;
    int num_toast_indexes;

    /* 走査状態（ページ、タプル、属性） */
    BlockNumber blkno;
    BufferAccessStrategy bstrategy;
    Buffer buffer;
    Page page;
    OffsetNumber offnum;
    ItemId itemid;
    uint16 lp_len;
    uint16 lp_off;
    HeapTupleHeader tuphdr;
    int natts;
    uint32 offset;
    AttrNumber attnum;

    /* 検証結果と制御フラグ */
    bool tuple_could_be_pruned;
    List *toasted_attributes;
    bool is_corrupt;
    TupleDesc tupdesc;
    Tuplestorestate *tupstore;
} HeapCheckContext;

入力と出力

  • **入力**: verify_heapam 関数によって初期化され、検証対象の Relation やトランザクション状態が格納されます。
  • **出力**: 検証中に検出された破損情報は tupstore に蓄積され、最終的に SQL 呼び出し元へ返却されます。

前提条件と不変条件

  • 検証は読み取り専用で行われますが、破損したデータにアクセスするため、サーバークラッシュのリスクを伴います。
  • safe_xmin は、検証実行中に all-visible になり得ないトランザクションIDの境界を定義します。

アルゴリズムまたは振る舞い

1. **初期化**: verify_heapam がリレーションと TOAST テーブルを開き、コンテキストを構築します。 2. **走査**: read_stream API を用いてページを順次読み込み、blkno を更新します。 3. **検証**: ページ内の各タプルに対し、ヘッダー、可視性、属性を順次チェックします。 4. **TOAST 検証**: ページ内の全タプル処理後、toasted_attributes リストに蓄積されたデータを検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: verify_heapam が本構造体をインスタンス化し、検証の各フェーズで参照します。
  • **呼び出し先**: check_tuple, check_tuple_header, check_tuple_visibility, check_tuple_attribute, check_toasted_attribute, report_corruption 等の各検証関数が、コンテキストを引数として受け取り、状態を更新・参照します。

関連データ構造

  • XidCommitStatus: トランザクション状態のキャッシュに使用。
  • ToastedAttribute: TOAST 属性の検証用リスト要素。
  • Tuplestorestate: 破損報告の蓄積に使用。

失敗モードと境界条件

  • is_corrupt フラグが破損検出時にセットされます。
  • 大量の破損が検出された場合、tuplestore によるメモリ消費が増大する可能性があります。

保守担当者向けメモ

  • 本構造体は検証の全ライフサイクルを管理するため、フィールドの追加や変更は check_tuple 系の関数群に影響を与えます。
  • toasted_attributes リストはページ単位で蓄積・検証されるため、メモリリークがないよう適切に管理してください。
