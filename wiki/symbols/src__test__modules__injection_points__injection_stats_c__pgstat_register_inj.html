pgstat_register_inj

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_stats.c:111

目的

PostgreSQLの統計サブシステム（pgstat）に対し、インジェクションポイント専用の統計種別（PGSTAT_KIND_INJECTION）を登録し、統計機能の利用準備を完了させる。

シグネチャまたは形状

void pgstat_register_inj(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 inj_stats_loaded を更新）

前提条件と不変条件

  • pgstat サブシステムが初期化されている環境で呼び出される必要がある。
  • injection_stats 構造体が適切に定義されていること。

アルゴリズムまたは振る舞い

1. pgstat_register_kind を呼び出し、PGSTAT_KIND_INJECTION と injection_stats（統計定義情報）を pgstat サブシステムに登録する。 2. 統計のロード状態を示すグローバル変数 inj_stats_loaded を true に設定する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/test/modules/injection_points/injection_points.c:574
  • 呼び出し先:
  • pgstat_register_kind (PostgreSQL統計サブシステム)

関連データ構造

  • PGSTAT_KIND_INJECTION: 登録される統計種別の識別子。
  • injection_stats: 統計のコールバックやサイズ情報を保持する定義構造体。

失敗モードと境界条件

  • pgstat_register_kind が内部でエラーを発生させた場合、本関数も正常に完了しない可能性がある。

保守担当者向けメモ

  • 本関数はモジュール初期化時に一度だけ実行されることを想定している。
  • PGSTAT_KIND_INJECTION は 25 とハードコードされているため、他の統計種別との競合に注意すること。
