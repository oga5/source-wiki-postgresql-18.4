toast_close_indexes

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/toast_internals.c:620

目的

TOASTテーブルに関連付けられたインデックスリレーションの配列をクローズし、メモリを解放します。

シグネチャまたは形状

void toast_close_indexes(Relation *toastidxs, int num_indexes, LOCKMODE lock)

入力と出力

  • toastidxs: クローズ対象のインデックスリレーションの配列。
  • num_indexes: 配列内のインデックス数。
  • lock: 各インデックスをクローズする際に使用するロックモード。
  • 戻り値: なし。

前提条件と不変条件

  • toastidxs は palloc 等で確保された有効な配列である必要があります。
  • 関数実行後、toastidxs が指すメモリ領域は pfree により解放されるため、呼び出し元で再利用してはなりません。

アルゴリズムまたは振る舞い

1. num_indexes で指定された回数分ループし、index_close を呼び出して各インデックスリレーションをクローズします。 2. 最後に pfree(toastidxs) を実行し、インデックスリレーションのポインタ配列自体を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/toast_internals.c 内の各関数（toast_save_datum 等の処理終了時）。
  • src/backend/access/heap/heaptoast.c
  • contrib/amcheck/verify_heapam.c
  • **呼び出し先**:
  • index_close (PostgreSQL内部API)
  • pfree (PostgreSQLメモリ管理API)

関連データ構造

  • Relation: PostgreSQLのリレーション記述子。

失敗モードと境界条件

  • toastidxs が NULL の場合、または num_indexes が不適切な値である場合、メモリ破壊やセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は toast_open_indexes と対になるクリーンアップ関数です。インデックスのオープンとクローズの整合性を保つために使用してください。
