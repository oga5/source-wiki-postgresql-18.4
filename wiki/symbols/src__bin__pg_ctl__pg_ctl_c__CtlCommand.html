CtlCommand

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_ctl/pg_ctl.c:52

目的

pg_ctl ユーティリティが実行可能な操作の種類を定義し、コマンドライン引数から決定された動作モードを保持するために使用されます。

シグネチャまたは形状

typedef enum {
    NO_COMMAND = 0,
    INIT_COMMAND,
    START_COMMAND,
    STOP_COMMAND,
    RESTART_COMMAND,
    RELOAD_COMMAND,
    STATUS_COMMAND,
    PROMOTE_COMMAND,
    LOGROTATE_COMMAND,
    KILL_COMMAND,
    REGISTER_COMMAND,
    UNREGISTER_COMMAND,
    RUN_AS_SERVICE_COMMAND,
} CtlCommand;

入力と出力

  • 入力: コマンドライン引数に基づき、main() 関数内で適切な列挙値が選択されます。
  • 出力: 選択された列挙値は、後続の制御フローにおける switch 文の分岐条件として機能します。

前提条件と不変条件

  • NO_COMMAND は初期状態（デフォルト値）として使用されます。
  • 実行時には、いずれか一つのコマンドが明示的に選択される必要があります。

アルゴリズムまたは振る舞い

pg_ctl のメインループにおいて、この列挙値に基づいて do_start(), do_stop() 等の各操作関数へ処理が振り分けられます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_ctl/pg_ctl.c 内の main() 関数および静的変数 ctl_command の初期化箇所（src/bin/pg_ctl/pg_ctl.c:81）。
  • 呼び出し先: 直接的な関数呼び出し関係はありませんが、この値に基づいて各操作関数（do_start 等）が実行されます。

関連データ構造

  • ShutdownMode: 停止操作時の挙動を制御する関連列挙型。
  • WaitPMResult: サーバー起動待機時の結果を保持する関連列挙型。

失敗モードと境界条件

  • 未定義のコマンドや NO_COMMAND のまま処理が進んだ場合、意図しない動作やエラーハンドリングが必要となる可能性があります。

保守担当者向けメモ

  • 新しい操作モードを追加する場合は、この列挙型に値を追加し、main() 内の引数解析ロジックおよび switch 文を更新してください。
