check_password

定義

  • 種別: function
  • 定義箇所: contrib/passwordcheck/passwordcheck.c:56

目的

PostgreSQLのロール作成やパスワード変更時において、パスワードの強度を検証し、不適切なパスワードを拒否する。

シグネチャまたは形状

static void check_password(const char *username, const char *shadow_pass, PasswordType password_type, Datum validuntil_time, bool validuntil_null)

入力と出力

  • **入力**:
  • username: ユーザー名。
  • shadow_pass: 検証対象のパスワード文字列。
  • password_type: パスワードの形式（PASSWORD_TYPE_PLAINTEXT か否か）。
  • validuntil_time, validuntil_null: 有効期限情報（本関数内では未使用）。
  • **出力**: なし（成功時は正常終了）。
  • **副作用**: 検証失敗時に ereport(ERROR, ...) を発行し、トランザクションを中断させる。

前提条件と不変条件

  • prev_check_password_hook が設定されている場合、本関数の冒頭で必ず呼び出される（フックの連鎖）。
  • PASSWORD_TYPE_PLAINTEXT 以外の場合は、ユーザー名とパスワードの一致チェックのみを行う。

アルゴリズムまたは振る舞い

1. **フック連鎖**: 以前に登録された prev_check_password_hook があれば実行する。 2. **暗号化パスワードの検証**: password_type が平文でない場合、plain_crypt_verify を使用してユーザー名とパスワードが一致していないかを確認する。 3. **平文パスワードの検証**:

  • **最小長**: min_password_length を下回っていないか確認。
  • **ユーザー名包含**: パスワード内にユーザー名が含まれていないか確認。
  • **文字種**: 英字と非英字（非ASCII含む）の両方が含まれているか確認。
  • **辞書チェック**: USE_CRACKLIB が定義されている場合、FascistCheck を呼び出し辞書攻撃耐性を検証する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/passwordcheck/passwordcheck.c 内の _PG_init にて check_password_hook に代入されることで、PostgreSQLコアのパスワード設定処理から間接的に呼び出される。
  • **呼び出し先**: prev_check_password_hook（存在する場合）、plain_crypt_verify、FascistCheck（USE_CRACKLIB 有効時）。

関連データ構造

  • PasswordType: パスワードの形式を定義する列挙型。
  • min_password_length: 最小長を保持するGUC変数。

失敗モードと境界条件

  • **エラー**: 条件を満たさない場合、すべて ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告する。
  • **境界条件**:
  • isalpha はマルチバイト文字に対応していないため、非ASCII文字はすべて「非英字」として扱われる。
  • cracklib が無効な環境では、辞書チェックは実行されない。

保守担当者向けメモ

  • 本関数は check_password_hook に登録される前提で設計されている。
  • 暗号化されたパスワードに対しては、平文と比較する以外の高度な検証は不可能であるため、運用上は可能な限り平文での検証が推奨される。
