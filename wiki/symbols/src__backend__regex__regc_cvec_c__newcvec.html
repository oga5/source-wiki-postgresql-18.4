newcvec

定義

  • 種別: static function
  • 定義箇所: src/backend/regex/regc_cvec.c:47

目的

正規表現のブラケット式（[...]）解析時に使用する文字セット構造体 cvec を、指定された容量で新規確保および初期化します。

シグネチャまたは形状

static struct cvec *newcvec(int nchrs, int nranges)

入力と出力

  • **入力**:
  • nchrs: 保持する個別の文字数。
  • nranges: 保持する文字範囲の数。
  • **出力**:
  • 成功時: 初期化済みの struct cvec へのポインタ。
  • 失敗時: NULL（メモリ確保失敗時）。

前提条件と不変条件

  • cvec 構造体は、ヘッダー部分と文字配列、範囲配列を連続したメモリ領域として確保します。
  • 戻り値の cvec は clearcvec を経由して初期化されるため、カウンタがゼロにリセットされた状態で返されます。

アルゴリズムまたは振る舞い

1. nchrs と nranges * 2 の合計から必要なメモリサイズを算出します。 2. MALLOC を使用して struct cvec とデータ領域を合わせたメモリを確保します。 3. 確保したメモリ領域内に chrs 配列と ranges 配列のポインタを配置します。 4. chrspace および rangespace に指定された容量を設定します。 5. clearcvec を呼び出し、構造体のカウンタを初期化して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_cvec.c (内部の getcvec)
  • src/backend/regex/regcomp.c
  • **呼び出し先**:
  • MALLOC (マクロ)
  • clearcvec

関連データ構造

  • struct cvec: 正規表現コンパイル時に文字集合を保持する構造体。regguts.h で定義されています。

失敗モードと境界条件

  • MALLOC がメモリ不足により NULL を返した場合、関数は NULL を返します。

保守担当者向けメモ

  • 本関数で確保したメモリは、vars 構造体を通じて管理されるライフサイクルに従う必要があります。呼び出し元で直接 free することは避けてください。
  • regcomp.c にインクルードされるため、静的関数として定義されています。
