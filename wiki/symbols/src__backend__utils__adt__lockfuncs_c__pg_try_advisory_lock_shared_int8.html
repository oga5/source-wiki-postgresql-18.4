pg_try_advisory_lock_shared_int8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/lockfuncs.c:732

目的

ユーザーが指定した64ビット整数キーに対して、共有モード（ShareLock）のアドバイザリーロックを非ブロック的に取得することを試みます。

シグネチャまたは形状

Datum pg_try_advisory_lock_shared_int8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して int64 型のキーを受け取ります。
  • **出力**: ロックの取得に成功した場合は true、既に他のプロセスによって競合するロックが保持されており取得できなかった場合は false を返します。

前提条件と不変条件

  • ロックの対象となるキーは、現在のデータベースIDと組み合わされて LOCKTAG を構成します。そのため、データベースを跨いだロックは行えません。
  • 本関数は非ブロック的（try）な操作であり、ロックが即座に取得できない場合でも待機せず、直ちに結果を返します。

アルゴリズムまたは振る舞い

1. SET_LOCKTAG_INT64 マクロを使用して、入力された int64 キーを LOCKTAG 構造体に変換します。 2. LockAcquire 関数を呼び出し、以下の引数を渡します：

  • tag: 生成したロックタグ。
  • ShareLock: 共有ロックモードを指定。
  • true: dontWait フラグを有効にし、ロックが即座に取得できない場合に待機しないようにします。
  • true: reportPrio フラグを有効にします。

    • 3. LockAcquire の戻り値 res を確認し、LOCKACQUIRE_NOT_AVAIL（ロック取得失敗）ではない場合に true を、それ以外の場合に false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースから呼び出されます（直接的な呼び出し元の詳細はソースコード上では明示されていません）。
  • **呼び出し先**:
  • LockAcquire（src/backend/storage/lmgr/lock.c）

関連データ構造

  • LOCKTAG: ロック対象を一意に識別するための構造体。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • ロックの取得に失敗した場合、エラーを発生させるのではなく false を返して正常に終了します。
  • データベースIDが異なる環境では、同じキー値であっても異なるロックとして扱われます。

保守担当者向けメモ

  • 本関数は pg_try_advisory_lock_int8 等と同様のパターンで実装されています。修正を行う際は、他のアドバイザリーロック関数との一貫性に注意してください。
  • ロックマネージャの内部状態を直接操作するため、LockAcquire の仕様変更時には影響を受ける可能性があります。
