TupleHashTableMatch

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execGrouping.c:519

目的

ハッシュテーブル内のタプルと、検索対象のタプルが等価であるかを判定します。

シグネチャまたは形状

static int TupleHashTableMatch(struct tuplehash_hash *tb, const MinimalTuple tuple1, const MinimalTuple tuple2)

入力と出力

  • **入力**:
  • tb: ハッシュテーブルの構造体ポインタ。
  • tuple1: 比較対象のタプル（ハッシュテーブル内のエントリ）。
  • tuple2: 比較対象のタプル（LookupTupleHashEntry から渡されるダミーエントリ）。
  • **出力**:
  • int: 比較結果。等価であれば 0 を返し、それ以外は非ゼロを返します。

前提条件と不変条件

  • tuple1 は必ず有効なタプルである必要があります。
  • tuple2 は LookupTupleHashEntry が使用するダミーエントリであることを前提としています。
  • tb->private_data を通じて TupleHashTable 構造体へアクセス可能である必要があります。

アルゴリズムまたは振る舞い

1. tb->private_data から TupleHashTable を取得し、式評価コンテキスト（econtext）を特定します。 2. ExecStoreMinimalTuple を使用して、tuple1 を hashtable->tableslot に格納します。 3. tuple2 が NULL であることを確認します。 4. econtext の ecxt_innertuple に inputslot を、ecxt_outertuple に tableslot を設定します。 5. ExecQualAndReset を呼び出し、hashtable->cur_eq_func を用いて等価比較を実行します。 6. 比較結果を反転させて返します（ExecQualAndReset が真を返す場合、等価とみなして 0 を返します）。

呼び出し元と呼び出し先

  • **呼び出し元**: simplehash.h のマクロ SH_EQUAL を通じて、ハッシュテーブルの検索・挿入処理から間接的に呼び出されます。
  • **呼び出し先**: ExecStoreMinimalTuple, ExecQualAndReset。

関連データ構造

  • TupleHashTable: グループ化のためのハッシュテーブルデータ構造。
  • MinimalTuple: 比較対象のタプル形式。
  • TupleTableSlot: タプルを保持するためのスロット。

失敗モードと境界条件

  • tuple1 が NULL の場合、アサーションエラーが発生します。
  • tuple2 が NULL でない場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は simplehash.h によって生成されるハッシュテーブルの等価比較関数として機能します。
  • ExecQualAndReset を呼び出すため、呼び出し側でメモリコンテキストの適切な管理が行われている必要があります。
