standby_desc_running_xacts

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/standbydesc.c:19

目的

WALレコードに含まれる xl_running_xacts 構造体の内容を人間が読める形式に変換し、指定された StringInfo バッファへ出力する。

シグネチャまたは形状

static void standby_desc_running_xacts(StringInfo buf, xl_running_xacts *xlrec)

入力と出力

  • **入力**:
  • buf: 出力先となる StringInfo 構造体へのポインタ。
  • xlrec: 解析対象の xl_running_xacts 構造体へのポインタ。
  • **出力**: なし（buf の内容が更新される）。

前提条件と不変条件

  • xlrec は有効な xl_running_xacts 構造体を指している必要がある。
  • buf は初期化済みの StringInfo であること。

アルゴリズムまたは振る舞い

1. nextXid、latestCompletedXid、oldestRunningXid をバッファに書き込む。 2. xlrec->xcnt が0より大きい場合、実行中のトランザクションIDリストを順次バッファに追加する。 3. xlrec->subxid_overflow が真であれば、サブトランザクションIDのオーバーフローを示す文字列を追加する。 4. xlrec->subxcnt が0より大きい場合、サブトランザクションIDリストをバッファに追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/rmgrdesc/standbydesc.c 内のメインディスパッチャ関数（standby_desc）から呼び出される。
  • **呼び出し先**: appendStringInfo および appendStringInfoString（PostgreSQLの文字列操作ユーティリティ）。

関連データ構造

  • xl_running_xacts: スタンバイサーバーにおける実行中トランザクションの状態を保持するWALレコード構造体。
  • StringInfo: PostgreSQLの可変長文字列バッファ。

失敗モードと境界条件

  • xcnt や subxcnt が0の場合、それぞれのリスト出力はスキップされる。
  • subxid_overflow が偽の場合、オーバーフローに関する記述は出力されない。

保守担当者向けメモ

  • 本関数はWAL解析ツール（pg_waldump等）専用であり、データベースの実行時ロジックには影響しない。
  • xl_running_xacts 構造体の定義が変更された場合は、本関数の出力ロジックも追従させる必要がある。
