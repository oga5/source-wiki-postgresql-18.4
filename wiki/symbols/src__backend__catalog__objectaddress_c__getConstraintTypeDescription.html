getConstraintTypeDescription

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaddress.c:4749

目的

指定された制約（Constraint）のOIDに基づき、その制約が「テーブル制約」か「ドメイン制約」かを判別し、人間が読める形式の説明文を StringInfo バッファに追加します。

シグネチャまたは形状

static void getConstraintTypeDescription(StringInfo buffer, Oid constroid, bool missing_ok)

入力と出力

  • **入力**:
  • buffer: 説明文を格納する StringInfo へのポインタ。
  • constroid: 対象となる制約のOID。
  • missing_ok: true の場合、制約が見つからなくてもエラーにせず "constraint" という文字列をバッファに追加します。
  • **出力**: なし（buffer を更新して終了）。

前提条件と不変条件

  • pg_constraint システムカタログへのアクセス権が必要です。
  • AccessShareLock を使用して pg_constraint テーブルを読み取ります。

アルゴリズムまたは振る舞い

1. pg_constraint テーブルを AccessShareLock で開きます。 2. 指定された constroid をキーとしてカタログからタプルを取得します。 3. タプルが存在しない場合:

  • missing_ok が false ならエラーを発生させます。
  • missing_ok が true なら "constraint" をバッファに追加して終了します。

    • 4. タプルが存在する場合、pg_constraint の構造体から conrelid（テーブル制約）または contypid（ドメイン制約）の有効性を確認します。 5. 判別結果に応じて "table constraint" または "domain constraint" をバッファに追加します。 6. どちらでもない場合はエラーを発生させます。 7. 最後に pg_constraint テーブルを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c 内の getObjectDescription 等から参照されています。
  • **呼び出し先**:
  • table_open, table_close: カタログアクセスのための標準関数。
  • get_catalog_object_by_oid: カタログからの行取得。
  • elog: エラー報告。
  • appendStringInfoString: バッファへの文字列追加。

関連データ構造

  • StringInfo: 文字列構築用バッファ。
  • Form_pg_constraint: pg_constraint カタログの行構造体。
  • ConstraintRelationId: pg_constraint テーブルのOID。

失敗モードと境界条件

  • 指定された constroid がカタログ内に存在しない場合、missing_ok が false であれば elog(ERROR, ...) が発生します。
  • conrelid も contypid も有効でない（OIDが0である）場合、整合性エラーとして elog(ERROR, ...) が発生します。

保守担当者向けメモ

  • 新しい制約タイプが追加された場合は、この関数の条件分岐を更新する必要があります。
  • table_open / table_close で AccessShareLock を使用しているため、カタログの読み取り整合性は保証されますが、呼び出し側は適切なトランザクションコンテキストで実行してください。
