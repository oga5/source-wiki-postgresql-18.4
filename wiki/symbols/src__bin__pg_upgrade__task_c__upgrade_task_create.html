upgrade_task_create

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/task.c:116

目的

データベースに対する一連のクエリ実行タスクを管理するための UpgradeTask 構造体を初期化し、必須のセキュリティ設定ステップを追加する。

シグネチャまたは形状

UpgradeTask *upgrade_task_create(void)

入力と出力

  • **入力**: なし
  • **出力**: 初期化済みの UpgradeTask 構造体へのポインタ。

前提条件と不変条件

  • pg_malloc0 によりメモリがゼロクリアされた状態で割り当てられる。
  • 生成されたタスクには、常に ALWAYS_SECURE_SEARCH_PATH_SQL を実行するステップが最初のタスクとして自動的に登録される。

アルゴリズムまたは振る舞い

1. pg_malloc0 を使用して UpgradeTask 構造体分のメモリを確保する。 2. createPQExpBuffer() を呼び出し、クエリ格納用のバッファを初期化して task->queries に割り当てる。 3. upgrade_task_add_step() を呼び出し、セキュリティ確保のための ALWAYS_SECURE_SEARCH_PATH_SQL をタスクの最初のステップとして追加する。 4. 初期化済みのタスク構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/check.c (複数箇所)
  • src/bin/pg_upgrade/function.c:84
  • src/bin/pg_upgrade/info.c:282
  • **呼び出し先**:
  • pg_malloc0
  • createPQExpBuffer
  • upgrade_task_add_step

関連データ構造

  • UpgradeTask: タスクの実行状態やクエリステップを保持するメイン構造体。

失敗モードと境界条件

  • メモリ不足時には pg_malloc0 が内部でエラーを発生させ、プロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数で追加される ALWAYS_SECURE_SEARCH_PATH_SQL は、このタスクを使用して実行されるすべてのクエリの前に必ず実行される。
  • 戻り値の UpgradeTask は、使用後に upgrade_task_free() で適切に解放する必要がある。
