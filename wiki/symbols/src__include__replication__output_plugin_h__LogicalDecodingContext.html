LogicalDecodingContext

定義

  • 種別: struct
  • 定義箇所: src/include/replication/output_plugin.h:102

目的

論理デコーディング処理における現在の状態を保持し、コアエンジンと出力プラグイン間でのデータ受け渡しや制御を行うためのコンテキスト構造体です。

シグネチャまたは形状

構造体の詳細なメンバー定義は提供されたソース抜粋には含まれていませんが、論理デコーディングの各コールバック関数（startup_cb, change_cb, commit_cb 等）の第一引数として渡されるポインタ型として利用されます。

入力と出力

  • **入力**: 論理デコーディングの進行に伴い、コアエンジンから各コールバック関数へ引数として渡されます。
  • **出力**: プラグイン側で OutputPluginPrepareWrite や OutputPluginWrite を呼び出す際の基点として使用され、変換後のデータをクライアントへ送信するために利用されます。

前提条件と不変条件

  • 論理デコーディングのセッション期間中、有効なインスタンスが存在する必要があります。
  • プラグインの各コールバック関数は、このコンテキストを介して状態を管理し、データ出力を行います。

アルゴリズムまたは振る舞い

コアエンジンは論理デコーディングの各フェーズ（トランザクションの開始・終了、データ変更の検出など）において、登録されたプラグインのコールバックを呼び出します。その際、本構造体を引数として渡すことで、プラグインは現在のデコーディング状態にアクセスし、必要に応じて補助関数を通じてデータを出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの論理デコーディングコアエンジン。
  • **呼び出し先**: contrib/test_decoding/test_decoding.c 内の各コールバック関数（pg_decode_startup, pg_decode_change, pg_decode_commit_txn 等）が、本構造体を引数として受け取ります。

関連データ構造

  • OutputPluginCallbacks: プラグインが実装すべきコールバック関数の集合。
  • OutputPluginOptions: プラグインの動作設定。
  • ReorderBufferTXN / ReorderBufferChange: トランザクションや変更内容を保持する構造体（replication/reorderbuffer.h 経由）。

失敗モードと境界条件

提供されたソース抜粋には、本構造体に関連する具体的な失敗モードや境界条件の記述はありません。

保守担当者向けメモ

  • 本構造体は論理デコーディングのインターフェースの核となるため、構造体のメンバー変更はすべての出力プラグインの実装に影響を与える可能性があります。
  • 巨大なトランザクションのストリーミング処理など、新しい機能追加に伴いコンテキストが保持すべき情報が増加する傾向にあります。最新の定義については src/include/replication/output_plugin.h を参照してください。
