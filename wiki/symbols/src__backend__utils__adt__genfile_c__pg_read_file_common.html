pg_read_file_common

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/genfile.c:239

目的

SQL関数から呼び出されるファイル読み込み処理の共通ラッパーであり、指定されたパスの検証とファイル内容のテキスト形式での取得を行います。

シグネチャまたは形状

static text *pg_read_file_common(text *filename_t, int64 seek_offset, int64 bytes_to_read, bool read_to_eof, bool missing_ok)

入力と出力

  • **入力**:
  • filename_t: 読み込むファイルパス（text型）。
  • seek_offset: 読み込み開始位置（int64）。
  • bytes_to_read: 読み込むバイト数（int64）。
  • read_to_eof: ファイル末尾まで読み込むかを示すフラグ（bool）。
  • missing_ok: ファイルが存在しない場合にエラーとせずNULLを返すかを示すフラグ（bool）。
  • **出力**:
  • 読み込んだファイル内容を格納した text 型のポインタ。

前提条件と不変条件

  • read_to_eof が true の場合、bytes_to_read は -1 でなければならない（Assertによる検証）。
  • read_to_eof が false の場合、bytes_to_read は負の値であってはならない。

アルゴリズムまたは振る舞い

1. 引数の妥当性を検証し、bytes_to_read が負の値であればエラーを報告します。 2. convert_and_check_filename を呼び出し、パスの正規化と pg_read_server_files ロールに基づく権限チェックを行います。 3. 検証済みのパスとオフセット、読み込みサイズを read_text_file に渡し、ファイルシステムからデータを読み込みます。 4. 読み込んだ結果を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/genfile.c 内の各SQLラッパー関数（pg_read_file_off_len 等）。
  • **呼び出し先**:
  • convert_and_check_filename（パス検証）
  • read_text_file（ファイル読み込み）

関連データ構造

  • text: PostgreSQLの可変長文字列型。

失敗モードと境界条件

  • bytes_to_read が負の値の場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーとなります。
  • convert_and_check_filename による権限チェックまたはパス検証に失敗した場合、エラーが発生します。
  • ファイルが存在しない場合、missing_ok が false であればエラーとなります。

保守担当者向けメモ

  • 本関数はテキスト形式の読み込み専用です。バイナリ形式の読み込みには pg_read_binary_file_common を使用してください。
  • 権限チェックのバイパスを防ぐため、convert_and_check_filename の呼び出し順序や引数には変更を加えないでください。
