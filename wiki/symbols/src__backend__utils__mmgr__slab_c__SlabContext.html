SlabContext

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/mmgr/slab.c:103

目的

PostgreSQL におけるメモリコンテキストの一種であり、固定サイズのオブジェクトを効率的に割り当て・解放するための「SLAB アロケータ」のメイン管理構造体です。

シグネチャまたは形状

typedef struct SlabContext
{
    MemoryContextData header;
    uint32      chunkSize;
    uint32      fullChunkSize;
    uint32      blockSize;
    int32       chunksPerBlock;
    int32       curBlocklistIndex;
#ifdef MEMORY_CONTEXT_CHECKING
    bool       *isChunkFree;
#endif
    int32       blocklist_shift;
    dclist_head emptyblocks;
    dlist_head  blocklist[SLAB_BLOCKLIST_COUNT];
} SlabContext;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入出力はありません。ただし、SlabContextCreate によって初期化され、SlabAlloc や SlabFree などの関数において、メモリ管理のパラメータおよび状態保持のコンテキストとして参照されます。

前提条件と不変条件

  • chunkSize は、コンテキスト作成時に固定されます。
  • SlabRealloc はサポートされていません。
  • SlabAlloc で要求されるサイズは、作成時に指定された chunkSize と一致する必要があります。
  • blocklist は、空きチャンク数に応じてブロックを分類して保持します。

アルゴリズムまたは振る舞い

  • **ブロック管理**: メモリを blockSize 単位で確保し、内部を chunkSize ごとに分割して管理します。
  • **効率化**: 空きチャンク数に基づいてブロックを blocklist に分類し、最も埋まっているブロックを優先的に使用することで断片化を抑制します。
  • **再利用**: 完全に空になったブロックは emptyblocks リストへ移動し、新規メモリ確保（malloc）を抑制します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/reorderbuffer.c が SlabContext を利用していることが確認されています。また、slab.c 内の各関数（SlabAlloc, SlabFree, SlabContextCreate 等）がこの構造体を引数として受け取ります。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • MemoryContextData: 基底となるメモリコンテキスト構造体。
  • SlabBlock: ブロック単位の管理構造体。
  • dlist_head / dclist_head: lib/ilist.h で定義される双方向リスト。

失敗モードと境界条件

  • MEMORY_CONTEXT_CHECKING が有効な場合、SlabCheck 関数により二重解放や境界外書き込みが検出されます。
  • SlabAlloc で不正なサイズが要求された場合、SlabAllocInvalidSize が関連する処理として存在します。

保守担当者向けメモ

  • SlabContext のサイズ計算には Slab_CONTEXT_HDRSZ マクロが使用されます。
  • blocklist_shift は、空きチャンク数から blocklist のインデックスを高速に算出するために使用されます。
  • SlabAlloc は頻繁に呼び出されるため、パフォーマンス維持のためにインライン化や最適化が考慮されています。
