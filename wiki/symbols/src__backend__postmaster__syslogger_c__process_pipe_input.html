process_pipe_input

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/syslogger.c:878

目的

パイプ経由で受信したログデータのチャンクを解析し、PID ごとにメッセージを再構成して適切なログファイルへ書き出す。

シグネチャまたは形状

static void process_pipe_input(char *logbuffer, int *bytes_in_logbuffer)

入力と出力

  • logbuffer: 受信した未処理のログデータが格納されたバッファ。
  • bytes_in_logbuffer: バッファ内の有効なデータ長（入力時）。処理後、残存するデータ長に更新される（出力時）。

前提条件と不変条件

  • パイプ上のデータは PipeProtoHeader 構造体で定義された独自プロトコルに従う。
  • ログメッセージは複数のチャンクに分割される可能性があり、PIPE_PROTO_IS_LAST フラグによって最終チャンクが識別される。
  • プロトコル外のデータ（非プロトコルデータ）は LOG_DESTINATION_STDERR として扱われる。

アルゴリズムまたは振る舞い

1. **ヘッダー解析**: バッファ先頭から PipeProtoHeader を読み取り、有効性を検証する。 2. **メッセージ再構成**:

  • チャンクが最終でない場合、PID に対応する save_buffer にデータを蓄積する。
  • 最終チャンクの場合、蓄積されたデータと結合し、write_syslogger_file を呼び出して書き出す。
  • 蓄積バッファが存在しない単一チャンクのメッセージは、そのまま書き出す。

    • 3. **非プロトコルデータの処理**: プロトコルヘッダーが見つからない場合、次のヘッダー開始位置までを stderr として書き出す。 4. **バッファ整理**: 処理しきれなかった残りのデータをバッファの先頭へ移動し、bytes_in_logbuffer を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: SysLoggerMain（src/backend/postmaster/syslogger.c 内のメインループ）。
  • **呼び出し先**: write_syslogger_file, appendBinaryStringInfo, initStringInfo, palloc, pfree。

関連データ構造

  • PipeProtoHeader: パイプ通信用のプロトコルヘッダー。
  • save_buffer: PID ごとに分割されたログメッセージを一時保持する構造体。
  • buffer_lists: save_buffer を管理するハッシュリスト。

失敗モードと境界条件

  • **ヘッダー不整合**: プロトコルヘッダーが破損している場合、非プロトコルデータとして処理を試みる。
  • **バッファ不足**: チャンク全体が受信できていない場合、ループを中断し、残りのデータを次回の呼び出しまで保持する。
  • **メモリ不足**: palloc によるメモリ確保失敗時はプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は SysLoggerMain におけるメインのデータ処理ロジックである。
  • 非プロトコルデータが混入した場合の挙動は、ログの整合性を維持するために read() の境界を尊重する設計となっている。
