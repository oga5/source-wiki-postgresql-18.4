CopyMultiInsertBufferFlush

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfrom.c:445

目的

CopyMultiInsertBuffer に蓄積されたタプルをターゲットテーブル（または外部テーブル）へ一括挿入し、関連するトリガー実行やインデックス更新、進捗状況の更新を行う。

シグネチャまたは形状

static inline void CopyMultiInsertBufferFlush(CopyMultiInsertInfo *miinfo, CopyMultiInsertBuffer *buffer, int64 *processed)

入力と出力

  • miinfo: バルク挿入の全体状態を保持する構造体。
  • buffer: フラッシュ対象のバッファ。
  • processed: 処理済み行数をカウントするポインタ（出力）。

前提条件と不変条件

  • buffer->nused が 0 より大きい場合に呼び出されることを想定している。
  • 外部テーブルの場合、buffer->bistate は NULL である必要がある。
  • ローカルテーブルの場合、buffer->bistate は初期化済みである必要がある。

アルゴリズムまたは振る舞い

1. **外部テーブル（FDW）の場合**:

  • ExecForeignBatchInsert を使用してバッチ挿入を実行する。
  • 挿入された各行に対して ExecARInsertTriggers を呼び出し、AFTER ROW INSERT トリガーを実行する。

    • 2. **ローカルテーブルの場合**:

  • table_multi_insert を呼び出し、蓄積されたタプルを一括挿入する。
  • インデックスが存在する場合、ExecInsertIndexTuples で各行のインデックスを更新し、AFTER ROW INSERT トリガーを実行する。
  • インデックスがない場合でも、トリガー定義があれば AFTER ROW INSERT トリガーを実行する。

    • 3. **共通処理**:

  • 処理した行数を processed に加算し、pgstat_progress_update_param で進捗を更新する。
  • ExecClearTuple を使用してバッファ内のスロットを解放し、buffer->nused を 0 にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfrom.c 内の COPY 実行ループおよび関連処理。
  • **呼び出し先**: ExecForeignBatchInsert (FDW), table_multi_insert, ExecInsertIndexTuples, ExecARInsertTriggers, ExecClearTuple, pgstat_progress_update_param 等。

関連データ構造

  • CopyMultiInsertInfo: バルク挿入のコンテキスト。
  • CopyMultiInsertBuffer: 挿入待ちタプルを保持するバッファ。
  • ResultRelInfo: 挿入先リレーション情報。
  • TupleTableSlot: タプルデータ保持用。

失敗モードと境界条件

  • table_multi_insert や FDW の挿入処理でエラーが発生した場合、現在の行番号やリレーション名に基づいたエラーコンテキストが適用される。
  • 外部テーブル挿入時は cstate->relname_only を一時的に有効化し、エラー時の詳細情報を抑制する。

保守担当者向けメモ

  • table_multi_insert はメモリリークの可能性があるため、呼び出し前後で MemoryContextSwitchTo を使用して一時的なメモリコンテキストへ切り替えている点に注意すること。
  • AFTER ROW トリガー実行時には、tts_tableOid を適切に再設定する必要がある。
