expanded_record_set_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/expandedrecord.c:439

目的

展開されたレコード（ExpandedRecordHeader）の物理的なタプル表現を、指定された新しいタプルで置き換える。

シグネチャまたは形状

void expanded_record_set_tuple(ExpandedRecordHeader *erh, HeapTuple tuple, bool copy, bool expand_external)

入力と出力

  • erh: 更新対象の展開レコードヘッダー。
  • tuple: 新しいタプルデータ。NULL の場合、レコードは空の状態にリセットされる。
  • copy: true の場合、tuple を erh のメモリコンテキストにコピーする。
  • expand_external: true かつ tuple が外部ストレージ（TOAST）を含む場合、インライン化を行う。

前提条件と不変条件

  • erh はダミーヘッダー（ER_FLAG_IS_DUMMY）であってはならない。
  • ドメイン型である場合、check_domain_for_new_tuple による制約チェックが先行して実行される。
  • 処理完了後、erh->flat_size は 0 にリセットされ、フラット化されたサイズ情報は無効化される。

アルゴリズムまたは振る舞い

1. **制約検証**: ドメイン型であれば、新しいタプルが制約を満たすか検証する。 2. **TOAST処理**: expand_external が有効な場合、toast_flatten_tuple を用いて外部データをインライン化する。 3. **メモリ管理**: copy が指定されていれば、erh->hdr.eoh_context 上にタプルのコピーを作成する。 4. **状態更新**: erh のタプルポインタおよびフラグを更新する。 5. **クリーンアップ**: 古いフィールド値（dvalues）および古いタプルが保持していたメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/pl_exec.c 内の複数の箇所（トリガー処理やレコード代入時）および src/backend/utils/adt/expandedrecord.c 内の初期化処理。
  • **呼び出し先**: check_domain_for_new_tuple, toast_flatten_tuple, heap_copytuple, heap_freetuple, pfree 等。

関連データ構造

  • ExpandedRecordHeader: 展開レコードのメイン構造体。
  • HeapTuple: PostgreSQL の物理的なタプル表現。

失敗モードと境界条件

  • erh がダミーヘッダーである場合、アサーションエラーが発生する。
  • copy が true の場合、メモリ不足により heap_copytuple が失敗する可能性がある。

保守担当者向けメモ

  • 本関数は、古いフィールド値のメモリ解放を行う際、それらが oldfstartptr から oldfendptr の範囲外にある場合にのみ pfree を実行する。これは、タプル内のデータ領域を指している値を誤って解放しないための安全策である。
