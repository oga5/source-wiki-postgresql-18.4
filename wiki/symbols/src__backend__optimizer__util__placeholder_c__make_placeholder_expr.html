make_placeholder_expr

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/placeholder.c:55

目的

クエリ最適化プロセスにおいて、外部結合によるNULL生成の影響を表現するための PlaceHolderVar (PHV) ノードを新規作成する。

シグネチャまたは形状

PlaceHolderVar *make_placeholder_expr(PlannerInfo *root, Expr *expr, Relids phrels)

入力と出力

  • **入力**:
  • root: 現在のクエリの PlannerInfo 構造体。
  • expr: PHVがカプセル化する対象の式。
  • phrels: この式が依存するリレーションの集合（Relids）。
  • **出力**:
  • 初期化済みの PlaceHolderVar 構造体へのポインタ。

前提条件と不変条件

  • root->glob->lastPHId が有効であり、新しい一意な phid を割り当てるためにインクリメントされる。
  • 生成されたPHVの phnullingrels は初期状態で NULL、phlevelsup は 0 に設定される。

アルゴリズムまたは振る舞い

1. makeNode(PlaceHolderVar) を呼び出し、メモリを確保する。 2. 引数 expr と phrels をノードの各フィールドに代入する。 3. root->glob->lastPHId をインクリメントし、その値を phid に割り当てることで、クエリ全体で一意な識別子を付与する。 4. phnullingrels を NULL に、phlevelsup を 0 に初期化する。 5. 完成した PlaceHolderVar を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/prep/prepjointree.c
  • src/backend/optimizer/util/var.c
  • **呼び出し先**:
  • makeNode (PostgreSQLのノード生成マクロ)

関連データ構造

  • PlaceHolderVar: 本関数が生成する主要な構造体。
  • PlannerInfo: クエリ全体の最適化状態を保持する構造体。
  • Relids: リレーションの集合を表すビットマップ。

失敗モードと境界条件

  • メモリ不足時には makeNode 経由でエラーが発生する可能性がある。
  • root が NULL の場合、root->glob へのアクセスによりクラッシュする。

保守担当者向けメモ

  • phnullingrels および phlevelsup は本関数ではデフォルト値に設定されるが、呼び出し側で後から変更されることが前提となっている。
  • 本関数は単なるノード生成器であり、PlaceHolderInfo の管理や評価レベルの計算は行わない。それらの処理は find_placeholder_info 等の別関数が担う。
