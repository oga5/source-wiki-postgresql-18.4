core_YYSTYPE

定義

  • 種別: typedef union
  • 定義箇所: src/include/parser/scanner.h:29

目的

字句解析器（スキャナ）がトークンを認識した際、そのトークンに関連付けられた値（意味値）を保持するための共用体です。

シグネチャまたは形状

typedef union core_YYSTYPE
{
    int         ival;        /* 整数リテラル用 */
    char       *str;         /* 識別子および非整数リテラル用 */
    const char *keyword;     /* キーワードの正規表記用 */
} core_YYSTYPE;

入力と出力

  • **入力**: core_yylex() 関数呼び出し時に、トークンの種類に応じて適切なメンバ（ival, str, keyword）に値が格納されます。
  • **出力**: パーサー（Bison）がトークンを処理する際、この共用体を通じてトークンの値を取得します。

前提条件と不変条件

  • 共用体であるため、一度に保持できる値はメンバのうち一つのみです。
  • str メンバを使用する場合、その文字列はスキャナのライフサイクル内で適切に管理されている必要があります。

アルゴリズムまたは振る舞い

スキャナが入力文字列を解析し、トークンを特定するたびに、そのトークンに応じた値を core_YYSTYPE 型の変数に書き込みます。パーサーはこの値を参照して構文解析を進めます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • core_yylex(): トークン値を書き込むために使用。
  • contrib/pg_stat_statements/pg_stat_statements.c: yylval として使用。
  • src/backend/parser/gramparse.h: lookahead_yylval として使用。
  • src/pl/plpgsql/src/pl_scanner.c: YYSTYPE とのインターフェースとして使用。
  • **呼び出し先**: 情報なし。

関連データ構造

  • core_yylex(): この型を引数として受け取るメインの字句解析関数。
  • YYLTYPE: トークンの位置情報を保持する構造体。

失敗モードと境界条件

  • 共用体のメンバを誤った型で読み取った場合、未定義の動作を引き起こします。
  • str メンバへのポインタが指すメモリ領域が、スキャナの終了後に解放される可能性があるため、保持期間には注意が必要です。

保守担当者向けメモ

  • 本構造体はコアスキャナのAPIの一部です。拡張が必要な場合は、core_yy_extra_type の拡張性との整合性を考慮してください。
