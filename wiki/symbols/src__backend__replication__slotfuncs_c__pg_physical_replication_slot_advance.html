pg_physical_replication_slot_advance

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slotfuncs.c:464

目的

現在の物理レプリケーションスロットの restart_lsn を指定された位置まで強制的に進める。

シグネチャまたは形状

static XLogRecPtr pg_physical_replication_slot_advance(XLogRecPtr moveto)

入力と出力

  • **入力**: moveto (進めたい先の XLogRecPtr)
  • **出力**: 実際に設定された restart_lsn を表す XLogRecPtr

前提条件と不変条件

  • moveto は InvalidXLogRecPtr であってはならない（Assert により検証）。
  • MyReplicationSlot が有効である必要がある。

アルゴリズムまたは振る舞い

1. 現在の restart_lsn（startlsn）を取得する。 2. startlsn が moveto より小さい場合のみ以下の処理を行う:

  • MyReplicationSlot->mutex を取得し、restart_lsn を moveto に更新する。
  • ReplicationSlotMarkDirty() を呼び出し、次回のチェックポイントでディスクへ書き出されるようマークする。
  • PhysicalWakeupLogicalWalSnd() を呼び出し、論理フェイルオーバースロットを保持する WAL 送信プロセスを起床させる。

    • 3. 更新後の restart_lsn（または更新不要時の元の値）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/slotfuncs.c:576 (関数名不明の箇所から呼び出されている)
  • **呼び出し先**:
  • SpinLockAcquire / SpinLockRelease
  • ReplicationSlotMarkDirty
  • PhysicalWakeupLogicalWalSnd

関連データ構造

  • MyReplicationSlot: 現在のバックエンドが使用しているレプリケーションスロット。
  • XLogRecPtr: WAL 位置を示すデータ型。

失敗モードと境界条件

  • moveto が現在の restart_lsn 以下の場合、スロットの更新は行われず、現在の restart_lsn がそのまま返される。
  • クラッシュが発生した場合、この関数による restart_lsn の進捗は失われる可能性がある（永続化は次回のチェックポイントに依存するため）。

保守担当者向けメモ

  • 本関数は物理スロット専用である。
  • スロットの更新は共有メモリ上で行われるため、スピンロックによる保護が必須である。
