pgstat_get_local_beentry_by_proc_number

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/backend_status.c:1247

目的

指定されたプロセス番号（ProcNumber）に対応するバックエンドのステータス情報を、ローカルメモリ上のスナップショットから検索して取得する。

シグネチャまたは形状

LocalPgBackendStatus *pgstat_get_local_beentry_by_proc_number(ProcNumber procNumber)

入力と出力

  • **入力**: procNumber (検索対象のプロセス番号)
  • **出力**: 見つかった場合は LocalPgBackendStatus 構造体へのポインタ、見つからない場合は NULL を返す。

前提条件と不変条件

  • 本関数を呼び出す前に、共有メモリ上の最新状態をローカルメモリに同期させるため pgstat_read_current_status() が実行される。
  • ローカルのバックエンドステータス配列（localBackendStatusTable）は、proc_number の昇順でソートされている必要がある。

アルゴリズムまたは振る舞い

1. pgstat_read_current_status() を呼び出し、共有メモリからローカルメモリへ最新のステータス情報をコピーする。 2. 検索キーとして LocalPgBackendStatus 構造体を作成し、proc_number に引数の値を設定する。 3. bsearch() を使用し、ソート済みの localBackendStatusTable に対して二分探索を行う。 4. 比較関数 cmp_lbestatus を用いて一致するエントリを特定し、そのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/backend_status.c (内部参照)
  • src/backend/utils/adt/pgstatfuncs.c:753
  • **呼び出し先**:
  • pgstat_read_current_status()
  • bsearch() (標準ライブラリ)
  • cmp_lbestatus (比較関数)

関連データ構造

  • LocalPgBackendStatus: ローカルメモリ上に保持されるバックエンドのステータス構造体。
  • localBackendStatusTable: バックエンドステータスのローカルコピーを保持する配列。
  • ProcNumber: プロセスを一意に識別するための型。

失敗モードと境界条件

  • 指定された procNumber に該当するエントリが存在しない場合、bsearch() は NULL を返し、本関数もそのまま NULL を返す。
  • localBackendStatusTable が空の場合、bsearch() は NULL を返す。

保守担当者向けメモ

  • 本関数は localBackendStatusTable が proc_number 順にソートされているという前提に強く依存している。配列の順序を維持するロジックに変更を加える際は注意が必要である。
