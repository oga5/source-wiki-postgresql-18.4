dataLeafPageGetUncompressed

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/gindatapage.c:210

目的

GINポスティングツリーのリーフページから、PostgreSQL 9.4以前の旧形式で格納されている非圧縮状態のTID（Tuple Identifier）リストを取得します。

シグネチャまたは形状

static ItemPointer dataLeafPageGetUncompressed(Page page, int *nitems)

入力と出力

  • **入力**:
  • page: 対象となるGINリーフページへのポインタ。
  • nitems: 取得したTIDの個数を格納するための出力用整数ポインタ。
  • **出力**:
  • ItemPointer: ページ内に格納されているTID配列の先頭アドレス。

前提条件と不変条件

  • page は圧縮されていない状態である必要があります（Assert(!GinPageIsCompressed(page)) により検証）。

アルゴリズムまたは振る舞い

1. ページが圧縮されていないことを確認します。 2. GinDataPageGetData(page) を使用して、ページ内のデータ領域からTID配列の先頭位置を特定します。 3. ページヘッダー（GinPageGetOpaque(page)）から maxoff を読み取り、それを *nitems に代入します。 4. TID配列の先頭アドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gindatapage.c 内の複数箇所（行169, 196, 1413）から呼び出されています。
  • **呼び出し先**:
  • GinPageIsCompressed (マクロ)
  • GinDataPageGetData (マクロ)
  • GinPageGetOpaque (マクロ)

関連データ構造

  • ItemPointer: TID（Tuple Identifier）の型。
  • Page: PostgreSQLの標準的なページ構造体。

失敗モードと境界条件

  • 圧縮済みのページに対して呼び出した場合、アサーションエラーが発生します。
  • ページが旧形式（9.4以前）のレイアウトであることを前提としているため、現在の圧縮形式のページに対して使用すると誤った値を返します。

保守担当者向けメモ

  • 本関数はレガシーなページ形式を扱うためのものです。新規開発や機能拡張において、現在の圧縮形式のページを扱う場合は GinDataLeafPageGetItems 等の適切な関数を使用してください。
