checkCond

定義

  • 種別: function
  • 定義箇所: contrib/ltree/lquery_op.c:198

目的

lquery（検索パターン）が ltree（階層パス）に適合するかを、再帰的に走査して判定する。

シグネチャまたは形状

static bool checkCond(lquery_level *curq, int qlen, ltree_level *curt, int tlen)

入力と出力

  • **入力**:
  • curq: 現在評価中の lquery レベル。
  • qlen: 残りのクエリレベル数。
  • curt: 現在照合対象の ltree レベル。
  • tlen: 残りのツリーレベル数。
  • **出力**:
  • bool: パターンが適合した場合は true、それ以外は false。

前提条件と不変条件

  • 再帰呼び出しを行うため、スタックオーバーフローを防ぐために check_stack_depth() が実行される。
  • 長時間実行されるクエリを中断可能にするため、ループ内で CHECK_FOR_INTERRUPTS() が実行される。

アルゴリズムまたは振る舞い

1. qlen が 0 になるまで、クエリの各レベルを順次処理する。 2. LQL_COUNT フラグに基づき、現在のクエリ項目の繰り返し回数（low から high）を決定する。 3. high を残りのツリー長 tlen で制限し、high < low の場合は不適合として終了する。 4. matchcnt を用いて、現在のクエリ項目とツリーの各レベルを checkLevel で照合する。 5. 繰り返し回数が low 以上に達した時点で、残りのパターンに対して再帰的に checkCond を呼び出し、適合を確認する。 6. 全てのクエリ項目を処理した後、ツリー側に残りのレベルが存在しない（tlen == 0）場合にのみ成功と判定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/lquery_op.c 内の演算子実装関数（ltq_regex 等）。
  • checkCond 自身（再帰呼び出し）。
  • **呼び出し先**:
  • checkLevel: 個別のレベル比較を実行。
  • checkCond: 再帰呼び出し。

関連データ構造

  • lquery_level: クエリの各階層構造。
  • ltree_level: ツリーの各階層構造。

失敗モードと境界条件

  • high < low: 必要な繰り返し回数がツリーの残りの長さで満たせない場合に失敗する。
  • tlen != 0: クエリの走査終了時にツリー側に余剰レベルがある場合は失敗する。
  • スタックオーバーフロー: 非常に深い階層構造に対して再帰が深すぎる場合に発生する可能性がある。

保守担当者向けメモ

  • 本関数は lquery の繰り返し指定（* 等）を処理するコアロジックである。
  • LQL_COUNT フラグの有無による low/high の解釈は後方互換性のためのハックである点に注意すること。
  • パフォーマンス向上のため、checkLevel の呼び出し前に可能な限り早期リターンを行う設計となっている。
