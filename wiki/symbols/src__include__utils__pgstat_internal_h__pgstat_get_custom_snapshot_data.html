pgstat_get_custom_snapshot_data

定義

  • 種別: function
  • 定義箇所: src/include/utils/pgstat_internal.h:932

目的

カスタム統計種別に対応するバックエンドローカルなスナップショットデータへのポインタを取得します。

シグネチャまたは形状

static inline void *pgstat_get_custom_snapshot_data(PgStat_Kind kind)

入力と出力

  • **入力**: PgStat_Kind kind（取得対象の統計種別）
  • **出力**: 指定された種別に対応するローカルスナップショットデータへの void * ポインタ。

前提条件と不変条件

  • kind は PGSTAT_KIND_CUSTOM_MIN 以上であること（pgstat_is_kind_custom(kind) が真であること）。
  • 指定された統計種別が fixed_amount（固定個数の統計）として定義されていること。

アルゴリズムまたは振る舞い

1. kind から PGSTAT_KIND_CUSTOM_MIN を減算し、カスタム統計用配列のインデックス idx を算出します。 2. 内部状態の整合性を確認するアサーションを実行します。 3. pgStatLocal.snapshot.custom_data 配列から、インデックス idx に対応するスナップショットデータを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/injection_points/injection_stats_fixed.c 内の関数（PGSTAT_KIND_INJECTION_FIXED を指定して呼び出し）。
  • **呼び出し先**:
  • pgstat_is_kind_custom（マクロまたは関数）
  • pgstat_get_kind_info（関数）

関連データ構造

  • PgStat_Kind: 統計種別を識別する列挙型。
  • PgStatLocal: バックエンドローカルな統計管理構造体。
  • PgStat_KindInfo: 統計種別のメタデータ（fixed_amount フラグを含む）。

失敗モードと境界条件

  • kind がカスタム統計の範囲外である場合、または fixed_amount ではない統計種別に対して呼び出された場合、アサーション失敗によりプロセスが異常終了します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結しています。
  • pgStatLocal 構造体への直接アクセスを行うため、呼び出し側は統計システムが初期化済みであることを前提としています。
