get_position

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_selfuncs.c:682

目的

範囲型の統計情報（ヒストグラム）における、特定の境界値の相対的な位置（0.0 から 1.0 の範囲）を算出します。

シグネチャまたは形状

static float8 get_position(TypeCacheEntry *typcache, const RangeBound *value, const RangeBound *hist1, const RangeBound *hist2)

入力と出力

  • **入力**:
  • typcache: 型固有のキャッシュ情報（rng_subdiff_finfo を含む）。
  • value: 位置を特定したい対象の境界値。
  • hist1, hist2: ヒストグラムのビンを構成する下限および上限の境界値。
  • **出力**:
  • float8: hist1 から hist2 までの範囲内における value の相対位置（0.0〜1.0）。

前提条件と不変条件

  • hist1 と hist2 が有限の場合、value も有限であることが期待されます。
  • typcache->rng_subdiff_finfo が有効である必要があります。
  • 戻り値は常に [0.0, 1.0] の範囲にクランプされます。

アルゴリズムまたは振る舞い

1. **有限境界の場合**:

  • value が無限であれば 0.5 を返します。
  • subdiff 関数が利用できない場合、または計算結果が不正（NaNやゼロ以下）な場合は 0.5 を返します。
  • subdiff 関数を用いて (value - hist1) / (hist2 - hist1) を計算し、相対位置を求めます。

    • 2. **無限境界を含む場合**:

  • hist1 が -inf、hist2 が有限の場合、value が -inf なら 0.0、それ以外は 1.0 を返します。
  • hist1 が有限、hist2 が +inf の場合、value が +inf なら 1.0、それ以外は 0.0 を返します。
  • 両方が無限の場合は 0.5 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/rangetypes_selfuncs.c 内の複数箇所（選択率計算ロジック）。
  • src/backend/utils/adt/multirangetypes_selfuncs.c 内の複数箇所。
  • **呼び出し先**:
  • typcache->rng_subdiff_finfo（型固有の差分計算関数）。

関連データ構造

  • RangeBound: 範囲の境界値を保持する構造体。
  • TypeCacheEntry: 型の比較関数や差分関数を保持するキャッシュ構造体。

失敗モードと境界条件

  • subdiff 関数が NaN を返す、または計算結果がゼロ以下の場合、フォールバックとして 0.5 を返します。
  • 境界値が無限である場合、比較関数が壊れている可能性を考慮し、安全なデフォルト値（0.5）を返す設計になっています。

保守担当者向けメモ

  • 本関数は subdiff 関数に強く依存しています。カスタム範囲型を定義する際は、subdiff 関数が適切に実装されていることを確認してください。
  • 幾何学的な線形補間を行っているため、データ分布が極端に偏っている場合、推定精度に影響を与える可能性があります。
