numeric

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-outofscope.c:45

目的

PostgreSQLの数値型（NUMERIC）データをC言語の構造体として表現し、ECPGプリプロセッサによるSQL操作時のデータ保持およびバインドを行う。

シグネチャまたは形状

typedef struct
{
    int           ndigits;      /* digits[]内の桁数 */
    int           weight;       /* 先頭桁の重み */
    int           rscale;       /* 結果のスケール */
    int           dscale;       /* 表示スケール */
    int           sign;         /* NUMERIC_POS, NUMERIC_NEG, または NUMERIC_NAN */
    NumericDigit *buf;          /* digits[]用に確保された領域の先頭 */
    NumericDigit *digits;       /* 10進数桁データ */
} numeric;

入力と出力

  • **入力**: データベースからフェッチされた数値データ、またはアプリケーション側で初期化された数値データ。
  • **出力**: SQL操作（FETCH等）を通じてデータベースへ書き込まれる、またはアプリケーション側で参照される数値データ。

前提条件と不変条件

  • ndigits は 0 を取り得る。
  • buf は digits 配列を保持するために動的に確保されたメモリ領域を指す必要がある。
  • sign は NUMERIC_POS、NUMERIC_NEG、NUMERIC_NAN のいずれかであること。

アルゴリズムまたは振る舞い

ECPGプリプロセッサによって生成されたコード内で、SQLの NUMERIC 型とC言語の構造体間のマッピングを仲介する。ECPGget_var 等のランタイム関数を通じて、データベースから取得した値をこの構造体に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPGプリプロセッサが生成したSQL操作コード（FETCH 等の実行時）。
  • **呼び出し先**: 不明（提供されたソース excerpt には本構造体を利用する具体的な関数呼び出しの記述なし）。

関連データ構造

  • NumericDigit: 数値の各桁を保持する型。
  • MYTYPE: 本ファイル内で定義される、データベースの行データを保持する構造体。

失敗モードと境界条件

  • buf が適切に初期化されていない場合、メモリ破壊や不正なアクセスが発生する可能性がある。
  • ndigits が負の値になる等の不正な状態は、数値演算や表示時にエラーを引き起こす。

保守担当者向けメモ

  • 本構造体は pgtypes.h に関連しており、ECPGの回帰テストにおいて数値型のデータバインドが正しく行われるかを検証するために使用されている。
  • 構造体の各フィールド（特に buf と digits）のメモリ管理は、ECPGランタイムライブラリの挙動に依存するため、変更時は ecpglib の仕様を確認すること。
