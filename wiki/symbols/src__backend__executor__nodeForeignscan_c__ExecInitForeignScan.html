ExecInitForeignScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeForeignscan.c:141

目的

外部テーブルスキャンを実行するための ForeignScanState ノードを生成および初期化し、対応する外部データラッパー（FDW）の初期化ルーチンを呼び出す。

シグネチャまたは形状

ForeignScanState *ExecInitForeignScan(ForeignScan *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 実行対象の ForeignScan プランノード。
  • estate: 実行状態を保持する EState。
  • eflags: 実行フラグ（EXEC_FLAG_BACKWARD や EXEC_FLAG_MARK はサポート外）。
  • **出力**:
  • 初期化済みの ForeignScanState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK を含めてはならない（Assert で検証）。
  • node->resultRelation が指定されている場合、EvalPlanQual（EPQ）実行中でない限り、対応する ResultRelInfo が estate に存在しなければならない。

アルゴリズムまたは振る舞い

1. ForeignScanState を作成し、実行エンジン用の基本情報を設定する。 2. ExecAssignExprContext で式評価コンテキストを割り当てる。 3. scanrelid に基づき、GetFdwRoutineForRelation または GetFdwRoutineByServerId を使用して FdwRoutine を取得する。 4. スキャン対象のタプル型（TupleDesc）を決定し、スロットを初期化する。 5. フィルタ条件（qual および fdw_recheck_quals）を初期化する。 6. 非同期スキャンの可否を判定する（EPQ実行中は無効化）。 7. outerPlan が存在する場合は ExecInitNode を再帰的に呼び出す。 8. 操作の種類に応じて、FDWの BeginForeignScan または BeginDirectModify を呼び出し、スキャンを開始する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c（ExecInitNode 経由）。
  • **呼び出し先**:
  • FdwRoutine の関数ポインタ（BeginForeignScan または BeginDirectModify）。
  • ExecInitNode（子プランノードがある場合）。
  • ExecInitQual（条件式の初期化）。
  • GetFdwRoutineForRelation / GetFdwRoutineByServerId（FDWルーチンの取得）。

関連データ構造

  • ForeignScanState: 本関数が構築する主要な状態構造体。
  • ForeignScan: プランノード。
  • FdwRoutine: 外部データソースとのインターフェースを定義する関数ポインタ群。
  • EState: クエリ実行全体のグローバル状態。

失敗モードと境界条件

  • resultRelation が指定されているにもかかわらず estate->es_result_relations が未初期化の場合、elog(ERROR) を発生させる。
  • eflags にサポート外のフラグが含まれる場合、Assert によりデバッグビルドで停止する。

保守担当者向けメモ

  • 本関数は「薄いラッパー」であり、実際のデータ取得ロジックは fdwroutine 経由で外部モジュールに委譲される。
  • EvalPlanQual（EPQ）処理中は、直接更新（DirectModify）がスキップされるため、このパスを通る際は注意が必要である。
