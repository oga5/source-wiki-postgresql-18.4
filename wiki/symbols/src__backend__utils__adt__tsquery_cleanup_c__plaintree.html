plaintree

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_cleanup.c:96

目的

NODE 構造体で表現された tsquery ツリーを、シリアライズ可能なフラットな QueryItem 配列形式へ変換します。

シグネチャまたは形状

QueryItem *plaintree(NODE *root, int *len)

入力と出力

  • **入力**:
  • root: 変換対象となる NODE ツリーのルートノード。
  • len: 変換後の QueryItem 配列の要素数を格納するための出力用ポインタ。
  • **出力**:
  • 変換された QueryItem 配列へのポインタ。メモリは palloc により確保されます。

前提条件と不変条件

  • root が NULL であるか、または root->valnode の型が QI_VAL（値）または QI_OPR（演算子）である必要があります。
  • len は有効な整数ポインタである必要があります。

アルゴリズムまたは振る舞い

1. 内部管理構造体 PLAINTREE を初期化し、初期バッファサイズを 16 要素分として設定します。 2. root が有効かつ適切なノード型を持つ場合、palloc でメモリを確保し、plainnode 関数を呼び出して再帰的にツリーを走査し、配列へ変換します。 3. root が無効な場合、配列ポインタを NULL に設定します。 4. 最終的な配列の要素数を *len に書き込み、配列の先頭ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery_cleanup.c:194 (clean_NOT 関数内)
  • src/backend/utils/adt/tsquery_cleanup.c:421 (cleanup_tsquery_stopwords 関数内)
  • **呼び出し先**:
  • plainnode (ツリーの再帰的走査と配列への書き込み)

関連データ構造

  • NODE: クエリツリーのノード構造体。
  • QueryItem: tsquery のフラットな表現形式。
  • PLAINTREE: 変換処理中の状態（現在の配列ポインタ、要素数、バッファサイズ）を保持する内部構造体。

失敗モードと境界条件

  • root が NULL の場合、len に 0 が代入され、NULL が返されます。
  • メモリ不足時には palloc がエラーを発生させます。
  • root->valnode が QI_VAL または QI_OPR 以外の場合、変換は行われず NULL が返されます。

保守担当者向けメモ

  • 本関数は plainnode と密接に連携しており、PLAINTREE 構造体を介して状態を共有しています。
  • palloc によるメモリ確保サイズは初期値 16 ですが、plainnode 内で必要に応じて拡張される設計となっています。
