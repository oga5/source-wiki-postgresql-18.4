PrintControlValues

定義

  • 種別: function
  • 定義箇所: src/bin/pg_resetwal/pg_resetwal.c:742

目的

グローバル変数 ControlFile に格納されているデータベースの制御情報を標準出力に表示する。

シグネチャまたは形状

static void PrintControlValues(bool guessed)

入力と出力

  • **入力**:
  • guessed (bool): true の場合、値が推測されたものである旨のヘッダーを表示する。
  • ControlFile (グローバル変数): 表示対象となる制御データ構造体。
  • **出力**: なし（標準出力への書き出しのみ）。

前提条件と不変条件

  • ControlFile が有効な値で初期化されている必要がある。
  • 本関数は pg_resetwal ユーティリティのコンテキストで実行されることを前提としている。

アルゴリズムまたは振る舞い

1. guessed 引数に基づき、ヘッダーメッセージ（"Guessed pg_control values:" または "Current pg_control values:"）を出力する。 2. ControlFile 構造体の各フィールド（バージョン番号、システム識別子、チェックポイント情報、ブロックサイズ、データ型設定など）を順次読み取り、フォーマットして出力する。 3. 一部のフィールド（floatFormat）は情報価値が低いため表示をスキップし、一部（Date/time type storage）は固定値として表示する。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/bin/pg_resetwal/pg_resetwal.c:431）から呼び出される。
  • **呼び出し先**: 標準ライブラリの printf 関数を使用。

関連データ構造

  • ControlFileData (グローバル変数 ControlFile の型): データベースの制御情報を保持する構造体。

失敗モードと境界条件

  • ControlFile の各フィールドが未初期化または破損している場合、誤った値が表示される可能性がある。
  • 本関数自体はエラーハンドリングを行わず、標準出力への書き込みが失敗した場合の挙動は呼び出し環境に依存する。

保守担当者向けメモ

  • ControlFileData 構造体の定義が変更された場合、本関数内の表示項目も同期して更新する必要がある。
  • printf のフォーマット文字列は国際化（_() マクロ）に対応しているため、変更時は注意すること。
