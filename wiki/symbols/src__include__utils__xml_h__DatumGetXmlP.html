DatumGetXmlP

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/xml.h:50

目的

PostgreSQLの Datum 形式で保持されているXMLデータを、xmltype ポインタとして安全に取り出すための変換関数です。

シグネチャまたは形状

static inline xmltype *DatumGetXmlP(Datum X)
  • 引数: Datum X (変換対象のDatum)
  • 戻り値: xmltype * (キャストされたXML型ポインタ)

入力と出力

  • 入力: Datum 型のデータ。
  • 出力: xmltype 構造体へのポインタ。

前提条件と不変条件

  • 入力された Datum X は、有効な xmltype を保持している必要があります。
  • PG_DETOAST_DATUM を使用しているため、入力データがTOAST（圧縮または外部格納）されている場合は自動的に展開されます。

アルゴリズムまたは振る舞い

1. 引数 X に対して PG_DETOAST_DATUM マクロを適用し、必要に応じてデータの展開（detoast）を行います。 2. 展開されたデータを xmltype * 型にキャストして返します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • src/backend/executor/execExprInterp.c
  • src/backend/utils/adt/xml.c
  • src/include/utils/xml.h (マクロ PG_GETARG_XML_P 経由)
  • 呼び出し先:
  • PG_DETOAST_DATUM (PostgreSQLのコアメモリ管理マクロ)

関連データ構造

  • xmltype: XMLデータを保持する varlena 型のエイリアス。

失敗モードと境界条件

  • PG_DETOAST_DATUM の仕様に従い、メモリ不足時にはエラーが発生する可能性があります。
  • 入力 Datum が xmltype として正しく構成されていない場合、後続の処理で不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、パフォーマンスを考慮してヘッダーファイル内に配置されています。
  • libxml2 が無効な環境下での動作については、xml.h の全体的な設計方針（libxml2 依存の抽象化）に従います。
