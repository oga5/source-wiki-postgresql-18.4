pg_last_wal_replay_lsn

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogfuncs.c:356

目的

最後にリプレイ（適用）されたWALのログシーケンス番号（LSN）を取得し、SQL層へ返却します。

シグネチャまたは形状

Datum pg_last_wal_replay_lsn(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数インターフェース経由の引数）
  • **出力**: Datum 型（LSN値、またはリプレイが一度も行われていない場合は NULL）

前提条件と不変条件

  • 内部的に GetXLogReplayRecPtr(NULL) を呼び出し、現在のリプレイ位置を取得します。
  • 戻り値が 0 の場合、リプレイされたWALが存在しないとみなされ、SQLの NULL が返されます。

アルゴリズムまたは振る舞い

1. GetXLogReplayRecPtr(NULL) を呼び出し、最後にリプレイされたWALのLSNを recptr に格納します。 2. recptr が 0 であるかを確認します。 3. 0 であれば PG_RETURN_NULL() を実行して終了します。 4. 0 でなければ PG_RETURN_LSN(recptr) を実行し、LSN値をSQL層へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（SQLクエリから直接呼び出されることを想定）。
  • **呼び出し先**: GetXLogReplayRecPtr（内部的なWALリカバリ状態取得関数）。

関連データ構造

  • XLogRecPtr: LSN（Log Sequence Number）を保持するデータ型。

失敗モードと境界条件

  • リカバリが一度も実行されていない、あるいはリプレイされたレコードが存在しない場合、0 が返され、結果として NULL が出力されます。

保守担当者向けメモ

  • 本関数は xlogrecovery.c が提供する低レベルAPIをラップするものです。リカバリ状態の判定ロジックに変更がある場合は、GetXLogReplayRecPtr の仕様を確認してください。
