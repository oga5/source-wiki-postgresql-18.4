metaphone

定義

  • 種別: function
  • 定義箇所: contrib/fuzzystrmatch/fuzzystrmatch.c:259

目的

入力された文字列を英語の音素（Metaphone）表現に変換し、SQLインターフェースを通じて提供する。

シグネチャまたは形状

Datum metaphone(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • 第1引数: 変換対象の文字列（text型）。
  • 第2引数: 出力される音素表現の最大長（int32型）。
  • **出力**:
  • 変換された音素表現を含む text 型のデータ。

前提条件と不変条件

  • 入力文字列の長さが MAX_METAPHONE_STRLEN を超えてはならない。
  • 要求される出力長（第2引数）が MAX_METAPHONE_STRLEN を超えてはならない。
  • 要求される出力長は 0 より大きくなければならない。
  • 英語の音素を前提としているため、非ラテン文字に対しては期待通りの結果が得られない可能性がある。

アルゴリズムまたは振る舞い

1. 入力文字列が空の場合は、空文字列を即座に返す。 2. 入力文字列および要求された出力長が制限値を超えていないか検証し、超えている場合はエラーを報告する。 3. 内部関数 _metaphone を呼び出し、音素変換を実行する。 4. 変換結果をPostgreSQLの text 型に変換して返却する。

呼び出し元と呼び出し先

  • **呼び出し先**: _metaphone（コアロジックの実装）。
  • **呼び出し元**: PostgreSQLのSQL実行エンジン（PG_FUNCTION_INFO_V1 マクロにより登録）。

関連データ構造

  • text 型（PostgreSQLの可変長文字列型）。

失敗モードと境界条件

  • **エラー**:
  • 入力文字列が MAX_METAPHONE_STRLEN を超える場合。
  • 要求された出力長が MAX_METAPHONE_STRLEN を超える場合。
  • 要求された出力長が 0 以下の場合。
  • **境界条件**: 入力文字列が空の場合、空文字列を返す。

保守担当者向けメモ

  • _metaphone 関数は伝統的なMetaphoneアルゴリズムを調整したものであり、USE_TRADITIONAL_METAPHONE マクロの定義により挙動を変更可能。
  • メモリ管理には palloc を使用しているため、PostgreSQLのメモリコンテキスト内で安全に動作する。
