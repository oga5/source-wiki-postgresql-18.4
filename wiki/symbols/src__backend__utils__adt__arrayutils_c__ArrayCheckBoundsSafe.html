ArrayCheckBoundsSafe

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arrayutils.c:126

目的

多次元配列の各次元における境界値（次元サイズと下限値の和）が、整数オーバーフローを起こさずに計算可能であるかを検証します。

シグネチャまたは形状

bool ArrayCheckBoundsSafe(int ndim, const int *dims, const int *lb, struct Node *escontext)

入力と出力

  • ndim: 配列の次元数。
  • dims: 各次元のサイズを格納した配列。
  • lb: 各次元の下限値を格納した配列。
  • escontext: エラー報告用コンテキスト。
  • 戻り値: 全ての次元でオーバーフローが発生しない場合は true を返し、発生した場合は false を返します（escontext が設定されている場合はエラーを報告します）。

前提条件と不変条件

  • dims および lb は少なくとも ndim 個の要素を持つ必要があります。
  • pg_add_s32_overflow を使用し、int32 の範囲内での加算が可能であることを不変条件として検証します。

アルゴリズムまたは振る舞い

1. 0 から ndim - 1 までループ処理を行います。 2. 各次元において、現在の次元サイズ dims[i] と下限値 lb[i] の和を計算します。 3. pg_add_s32_overflow を用いて、この加算が int32 の範囲を超えるかを確認します。 4. オーバーフローが検出された場合、ereturn マクロを介して ERRCODE_PROGRAM_LIMIT_EXCEEDED エラーを報告し、false を返します。 5. 全ての次元で問題がない場合、true を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/arrayutils.c:119 にて呼び出しを確認。
  • 呼び出し先: pg_add_s32_overflow（オーバーフローチェック用ユーティリティ）、ereturn（エラー報告用マクロ）。

関連データ構造

  • Node: エラーコンテキストとして使用される構造体。

失敗モードと境界条件

  • dims[i] + lb[i] が int32 の最大値を超える場合、エラーとして処理されます。
  • ndim が 0 の場合、ループは実行されず true を返します。

保守担当者向けメモ

  • 本関数は、配列の物理的なメモリ配置計算を行う前の安全策として機能します。
  • PG_USED_FOR_ASSERTS_ONLY が使用されている通り、計算結果の sum はアサーション目的以外では利用されません。
