dsa_attach_in_place

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/dsa.c:544

目的

既存のメモリ領域（place）に配置された動的共有メモリ領域（DSA）を、現在のプロセスのアドレス空間にアタッチする。

シグネチャまたは形状

dsa_area *dsa_attach_in_place(void *place, dsm_segment *segment)

入力と出力

  • **入力**:
  • place: DSA 制御構造体が配置されているメモリ上の開始アドレス。
  • segment: （オプション）この DSA を保持する DSM セグメント。指定された場合、セグメントのデタッチ時に DSA の解放処理が自動的に登録される。
  • **出力**:
  • アタッチされた領域を管理する dsa_area 構造体へのポインタ。

前提条件と不変条件

  • place は、以前に dsa_create_in_place 等で初期化された有効な DSA 制御構造体を指している必要がある。
  • segment が提供される場合、そのセグメントは DSA のライフサイクルと同期して管理される。

アルゴリズムまたは振る舞い

1. attach_internal を呼び出し、指定された place を基に dsa_area 構造体を初期化・取得する。 2. segment が NULL でない場合、on_dsm_detach を使用して、セグメントがデタッチされる際に dsa_on_dsm_detach_release_in_place 関数が実行されるようコールバックを登録する。 3. 取得した dsa_area ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/session.c
  • src/backend/executor/execParallel.c
  • src/backend/utils/activity/pgstat_shmem.c
  • **呼び出し先**:
  • attach_internal (内部関数)
  • on_dsm_detach (DSM セグメントのデタッチ時コールバック登録)

関連データ構造

  • dsa_area: DSA 領域を管理するバックエンドごとの状態構造体。
  • dsm_segment: 基盤となる動的共有メモリセグメント。

失敗モードと境界条件

  • place が無効なアドレスや初期化されていない領域を指している場合、attach_internal 内で予期せぬ動作を引き起こす可能性がある。
  • segment が NULL の場合、自動的な解放コールバックは登録されないため、呼び出し側で明示的な管理が必要となる。

保守担当者向けメモ

  • 本関数は dsa_attach と共通の内部処理（attach_internal）を利用している。
  • dsa_on_dsm_detach_release_in_place は、DSM セグメントのデタッチ時に DSA 領域のクリーンアップを行うためのフックである。
