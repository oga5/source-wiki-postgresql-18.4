gbt_intvgt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_interval.c:31

目的

interval 型の2つの値に対して「より大きい（greater than）」という比較を行い、その結果を真偽値で返します。

シグネチャまたは形状

static bool gbt_intvgt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象となる interval 型へのポインタ（void * として渡される）。
  • flinfo: 関数呼び出し情報（FmgrInfo *）。
  • **出力**:
  • bool: a > b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な interval 型のデータ構造を指している必要があります。
  • 内部で interval_gt を呼び出すため、PostgreSQL の interval 型比較規則に従います。

アルゴリズムまたは振る舞い

1. IntervalPGetDatum マクロを使用して、入力された void * ポインタを Datum 型に変換します。 2. DirectFunctionCall2 を介して、PostgreSQL のコア関数である interval_gt を呼び出します。 3. interval_gt の戻り値を DatumGetBool で真偽値に変換し、呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**: interval_gt（PostgreSQL コアの比較関数）。
  • **呼び出し元**: 同一ファイル内の gbt_intvgt 自体（再帰的な参照ではなく、関数ポインタとしての登録用）および、contrib/btree_gist/btree_interval.c:102 で定義されている tinfo 構造体（gbtree_ninfo）の初期化時に参照されています。

関連データ構造

  • interval: 比較対象となる PostgreSQL の標準データ型。
  • FmgrInfo: 関数マネージャ情報。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、IntervalPGetDatum の挙動により予期せぬ動作を引き起こす可能性があります。
  • interval_gt がエラーを投げる場合（例：無効なインターバルデータ）、本関数もその例外を伝播します。

保守担当者向けメモ

  • 本関数は btree_gist における interval 型の比較ロジックを抽象化するためのラッパーです。
  • ロジックの変更が必要な場合は、interval_gt の仕様を確認してください。
  • tinfo 構造体を通じて btree_utils_num.h のユーティリティ関数から利用されるため、シグネチャの変更には注意が必要です。
