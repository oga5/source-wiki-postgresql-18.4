makeArrayTypeName

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_type.c:839

目的

指定された基本型名に対応する配列型の名前を生成する。既存の型名と衝突しない一意な名前を決定する役割を持つ。

シグネチャまたは形状

char *makeArrayTypeName(const char *typeName, Oid typeNamespace)

入力と出力

  • **入力**:
  • typeName: 基本となる型名。
  • typeNamespace: 型が存在する名前空間（OID）。
  • **出力**:
  • 生成された配列型名の文字列（呼び出し元で pfree が必要）。

前提条件と不変条件

  • 配列型の名前は、慣習的に基本型名の先頭にアンダースコア _ を付与する。
  • 名前が長すぎる場合や既存の型名と衝突する場合は、基本型名を切り詰め、必要に応じてアンダースコアと連番（_1, _2 等）を付与して一意性を確保する。

アルゴリズムまたは振る舞い

1. makeObjectName("", typeName, NULL) を呼び出し、基本となる配列型名（_ + typeName）を生成する。 2. SearchSysCacheExists2 を使用し、指定された名前空間内に同名の型が存在するか確認する。 3. 衝突が発生した場合、pfree でメモリを解放し、連番（pass）をインクリメントして makeObjectName を再試行する。 4. 衝突しない名前が見つかるまでこのプロセスを繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c
  • src/backend/catalog/pg_type.c
  • src/backend/commands/typecmds.c
  • **呼び出し先**:
  • makeObjectName (PostgreSQL内部の汎用名前生成関数)
  • SearchSysCacheExists2 (システムカタログ検索)
  • pfree (メモリ解放)

関連データ構造

  • pg_type: 型情報を保持するシステムカタログ。
  • NAMEDATALEN: 型名の最大長を規定する定数。

失敗モードと境界条件

  • 指定された名前空間内で一意な名前が見つからない場合、ループが終了しない可能性がある（ただし、システム上の制限により通常は一意な名前が生成される）。
  • メモリ不足時には makeObjectName 等からエラーが送出される可能性がある。

保守担当者向けメモ

  • 本関数は ChooseRelationName と類似のロジックで一意性を保証している。
  • 配列型の命名規則は外部クライアントコードにも依存されているため、先頭のアンダースコアという慣習は変更しないこと。
