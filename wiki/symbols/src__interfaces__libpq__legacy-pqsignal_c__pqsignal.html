pqsignal

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/legacy-pqsignal.c:41

目的

PostgreSQL 9.2 以前の libpq とのバイナリ互換性を維持するためのレガシー・インターフェースです。古いクライアントアプリケーションがシグナルハンドラを設定できるようにします。

シグネチャまたは形状

pqsigfunc pqsignal(int signo, pqsigfunc func);

入力と出力

  • **入力**:
  • signo (int): 設定対象のシグナル番号。
  • func (pqsigfunc): 設定するシグナルハンドラ関数。
  • **出力**:
  • 設定変更前の古いシグナルハンドラを返します。エラー発生時は SIG_ERR を返します。

前提条件と不変条件

  • 本関数は現代の PostgreSQL コードベースでは使用が推奨されません。
  • 内部コードから本関数を呼び出すことは、src/include/port.h のマクロ定義により制限されています。

アルゴリズムまたは振る舞い

プラットフォームに応じて以下の処理を行います。 1. **非 Windows 環境**:

  • sigaction を使用してシグナルハンドラを設定します。
  • SIGALRM 以外のシグナルには SA_RESTART フラグを付与し、システムコールの自動再開を有効にします。
  • SIGCHLD の場合、SA_NOCLDSTOP フラグを付与します（定義されている場合）。

    • 2. **Windows 環境**:

  • 標準の signal() 関数を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**: 外部の古いクライアントアプリケーション。
  • **呼び出し先**:
  • 非 Windows: sigaction
  • Windows: signal
  • 注: ツールが検出した contrib/pg_prewarm 等からの呼び出しは、本レガシー関数ではなく、同名の別実装（src/port/pqsignal.c 等）を指している可能性が高いです。

関連データ構造

  • struct sigaction (非 Windows 環境で使用)
  • pqsigfunc (シグナルハンドラの型定義)

失敗モードと境界条件

  • sigaction システムコールが失敗した場合、SIG_ERR を返します。
  • SIGALRM に対しては SA_RESTART を付与しないという、当時の仕様を厳密に再現した挙動をとります。

保守担当者向けメモ

  • 本ファイルは歴史的互換性維持のみを目的としています。新規開発や保守においてこの関数を呼び出さないでください。
  • シグナル処理の標準実装については src/port/pqsignal.c を参照してください。
