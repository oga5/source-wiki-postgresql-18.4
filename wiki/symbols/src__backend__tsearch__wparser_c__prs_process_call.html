prs_process_call

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/wparser.c:215

目的

SRF（Set-Returning Function）の反復処理において、メモリ上に展開されたトークンリストから順次タプルを生成して返す。

シグネチャまたは形状

static Datum prs_process_call(FuncCallContext *funcctx)

入力と出力

  • **入力**: FuncCallContext *funcctx（SRFの実行状態を保持するコンテキスト）。
  • **出力**: Datum（生成されたタプル、または反復終了時に (Datum) 0）。

前提条件と不変条件

  • funcctx->user_fctx に PrsStorage 構造体へのポインタが格納されていること。
  • PrsStorage 内の list に解析済みのトークン群が保持されており、cur が現在のインデックスを指していること。

アルゴリズムまたは振る舞い

1. funcctx->user_fctx から PrsStorage を取得する。 2. 現在のインデックス st->cur がリストの長さ st->len 未満であるかを確認する。 3. 未満であれば、以下の処理を行う：

  • 現在のトークン型（type）と語彙（lexeme）を文字列配列に格納する。
  • BuildTupleFromCStrings を使用してタプルを構築する。
  • 語彙文字列のメモリを解放（pfree）する。
  • インデックス st->cur をインクリメントする。
  • 構築したタプルを Datum として返す。

    • 4. インデックスがリストの長さに達している場合は (Datum) 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/wparser.c:258 (ts_parse_byid 内)
  • src/backend/tsearch/wparser.c:282 (ts_parse_byname 内)
  • **呼び出し先**:
  • BuildTupleFromCStrings（PostgreSQL内部関数）
  • pfree（PostgreSQLメモリ管理関数）

関連データ構造

  • FuncCallContext: SRFの状態保持用構造体。
  • PrsStorage: トークンリスト（list）、現在の位置（cur）、リスト長（len）を保持する構造体。

失敗モードと境界条件

  • リストの末尾に到達した際は (Datum) 0 を返し、SRFの反復終了を通知する。
  • メモリ不足等のシステムエラー時は、PostgreSQLの標準的なエラーハンドリングによりアボートされる。

保守担当者向けメモ

  • 本関数は ts_parse 系関数の反復処理を担う。PrsStorage のメモリ管理と整合性を保つ必要がある。
  • pfree(values[1]) は、st->list 内の文字列が動的に割り当てられていることを前提としているため、リスト構築時のメモリ確保方針を変更する場合は注意が必要である。
