PGOutputTxnData

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/pgoutput/pgoutput.c:214

目的

論理レプリケーションにおけるトランザクション単位の送信状態を保持する構造体です。特に、トランザクション開始メッセージ（BEGIN）が既に送信済みかどうかを追跡するために使用されます。

シグネチャまたは形状

typedef struct PGOutputTxnData
{
	bool		sent_begin_txn; /* BEGINが送信済みかを示すフラグ */
} PGOutputTxnData;

入力と出力

  • **入力**: txn->output_plugin_private を介して、論理デコーディングのトランザクションコンテキストに紐付けられます。
  • **出力**: sent_begin_txn フラグの状態に基づき、後続の変更イベント送信時にBEGINメッセージを再送すべきかどうかの判定に使用されます。

前提条件と不変条件

  • PGOutputTxnData インスタンスは、MemoryContextAllocZero を使用してゼロ初期化された状態で割り当てられることが前提です。
  • sent_begin_txn は、トランザクションの開始時に false で初期化され、BEGINメッセージの送信後に true に更新されます。

アルゴリズムまたは振る舞い

1. **初期化**: トランザクション処理の開始時に、txn->output_plugin_private に PGOutputTxnData が割り当てられます。 2. **状態追跡**: pgoutput プラグインは、変更イベントを書き出す前に sent_begin_txn を参照します。 3. **条件付き送信**: sent_begin_txn が false の場合、BEGINメッセージを送信し、フラグを true に設定します。これにより、トランザクション内で重複したBEGINメッセージの送信を防ぎます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/pgoutput/pgoutput.c 内のトランザクション処理関数（pgoutput_begin_txn, pgoutput_commit_txn, pgoutput_begin_prepare_txn, pgoutput_prepare_txn 等）が、txn->output_plugin_private をキャストして本構造体にアクセスしています。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • PGOutputData: プラグイン全体のセッション状態を保持する構造体。
  • LogicalDecodingContext: 論理デコーディングの基盤となるコンテキスト。

失敗モードと境界条件

  • **メモリ不足**: MemoryContextAllocZero が失敗した場合、トランザクション処理が中断されます。
  • **キャストの不整合**: txn->output_plugin_private が期待される型（PGOutputTxnData）でない場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本構造体はトランザクションのライフサイクルに密接に関連しています。sent_begin_txn のフラグ管理は、ストリーミングトランザクション（in_streaming）の制御と併せて確認してください。
  • 現在のソースコード上では、txn->output_plugin_private へのキャストが複数の関数で行われているため、型安全性には注意が必要です。
