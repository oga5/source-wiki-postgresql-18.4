pltcl_proc_key

定義

  • 種別: struct
  • 定義箇所: src/pl/tcl/pltcl.c:194

目的

PL/Tcl において、コンパイル済み関数をキャッシュするためのハッシュテーブルの検索キーとして機能します。

シグネチャまたは形状

typedef struct pltcl_proc_key
{
    Oid proc_id;    /* 関数 OID */
    Oid is_trigger; /* トリガー関数かどうかのフラグ（パディング回避のため Oid 型） */
    Oid user_id;    /* 呼び出しユーザーの OID、または 0 */
} pltcl_proc_key;

入力と出力

本構造体はデータ構造であり、関数呼び出しの入力や出力には直接関与しません。ハッシュテーブルのキーとして使用されます。

前提条件と不変条件

  • 構造体内にパディングが発生しないよう、すべてのフィールドが Oid 型で定義されています。
  • ハッシュテーブルのキーとして使用される際、構造体全体がメモリ上で連続している必要があります。

アルゴリズムまたは振る舞い

  • proc_id、is_trigger、user_id の組み合わせにより、特定のユーザーが実行する特定の関数（またはトリガー）を一意に識別します。
  • ハッシュテーブルのサイズ設定（hash_ctl.keysize）において、本構造体のサイズが直接参照されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/tcl/pltcl.c 内の関数キャッシュ処理（compile_pltcl_function 等）において、ローカル変数としてインスタンス化され、ハッシュ検索に使用されます。
  • 呼び出し先: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • pltcl_proc_desc: 本キーを使用してハッシュテーブルから取得される、コンパイル済み関数の詳細情報を含む構造体。

失敗モードと境界条件

  • user_id が 0 の場合、ユーザー非依存のキャッシュエントリとして扱われます。
  • 構造体内のフィールドが正しく初期化されていない場合、ハッシュテーブルの検索ミスや誤ったキャッシュエントリの取得が発生する可能性があります。

保守担当者向けメモ

  • is_trigger は論理値（bool）として機能しますが、構造体のパディングを排除するために Oid 型で定義されています。将来的にフィールドを追加する場合は、アライメントに注意してください。
  • ハッシュテーブルのキーとして使用されるため、構造体のメモリレイアウトを変更する際は、関連するハッシュテーブルの初期化コード（hash_ctl.keysize の設定箇所）との整合性を確認してください。
