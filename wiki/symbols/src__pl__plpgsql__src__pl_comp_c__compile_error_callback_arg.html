compile_error_callback_arg

定義

  • 種別: struct
  • 定義箇所: src/pl/plpgsql/src/pl_comp.c:138

目的

PL/pgSQL のコンパイル中に発生したエラーに対して、エラーコンテキスト情報を保持し、エラーメッセージにソースコードの文脈を含めるためのデータ構造です。

シグネチャまたは形状

struct compile_error_callback_arg
{
	const char *proc_source;
	yyscan_t	yyscanner;
};

入力と出力

  • proc_source: コンパイル対象のソースコード文字列へのポインタ。
  • yyscanner: コンパイルに使用されているスキャナの状態を保持するハンドル。

前提条件と不変条件

  • plpgsql_compile または plpgsql_compile_inline の実行中に、エラーハンドラを設定する目的でのみ使用されます。
  • yyscanner は有効なスキャナ状態を指している必要があります。

アルゴリズムまたは振る舞い

コンパイル中にエラーが発生した際、plpgsql_compile_error_callback 関数がこの構造体を参照します。この構造体から取得した proc_source と yyscanner を用いて、エラーが発生した位置のソースコードを特定し、ユーザーに分かりやすいエラーメッセージを生成します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/pl/plpgsql/src/pl_comp.c:185
  • src/pl/plpgsql/src/pl_comp.c:744
  • 呼び出し先:
  • src/pl/plpgsql/src/pl_comp.c:887 (エラーコールバック関数内で引数としてキャストされ参照されます)

関連データ構造

  • yyscan_t: スキャナの状態管理に使用される型。
  • PLpgSQL_function: コンパイル結果として生成される構造体。

失敗モードと境界条件

  • proc_source が NULL の場合、エラーメッセージ生成時にソースコードの表示が失敗する可能性があります。
  • yyscanner が初期化されていない状態で参照された場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は plpgsql_compile_error_callback と密接に関連しています。エラー報告ロジックを変更する場合は、この構造体のフィールドが適切に初期化されているか確認してください。
