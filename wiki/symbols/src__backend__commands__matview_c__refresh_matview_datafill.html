refresh_matview_datafill

定義

  • 種別: function
  • 定義箇所: src/backend/commands/matview.c:404

目的

マテリアライズド・ビュー（MV）の更新または作成時に、定義クエリを実行し、その結果を DestReceiver を介して対象のテーブルへ流し込む。

シグネチャまたは形状

static uint64 refresh_matview_datafill(DestReceiver *dest, Query *query, const char *queryString, bool is_create)

入力と出力

  • **入力**:
  • dest: データを格納するための DestReceiver。
  • query: MVの定義クエリ（Query 構造体）。
  • queryString: クエリの文字列形式。
  • is_create: CREATE MATERIALIZED VIEW の実行中か否かを示すフラグ。
  • **出力**:
  • uint64: 処理されたタプル数。

前提条件と不変条件

  • query は copyObject によって複製され、書き換え（Rewrite）処理が行われる。
  • 書き換え後のクエリは単一の SELECT 文でなければならない。
  • CHECK_FOR_INTERRUPTS() が呼び出され、ユーザーによる中断要求を監視する。

アルゴリズムまたは振る舞い

1. **クエリの準備**: 入力クエリを複製し、AcquireRewriteLocks および QueryRewrite を実行して実行可能な状態にする。 2. **計画作成**: pg_plan_query を用いてクエリの実行計画を作成する。 3. **スナップショット管理**: PushCopiedSnapshot および UpdateActiveSnapshotCommandId を呼び出し、一貫性のあるスナップショットで実行する。 4. **実行**: CreateQueryDesc で実行記述子を作成し、ExecutorStart、ExecutorRun、ExecutorFinish、ExecutorEnd の順に実行エンジンを駆動する。 5. **後処理**: FreeQueryDesc でリソースを解放し、スナップショットを復元して処理されたタプル数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/matview.c 内の RefreshMatViewByOid から呼び出される。
  • **呼び出し先**: copyObject, AcquireRewriteLocks, QueryRewrite, pg_plan_query, PushCopiedSnapshot, CreateQueryDesc, ExecutorStart, ExecutorRun, ExecutorFinish, ExecutorEnd 等の実行エンジン関数。

関連データ構造

  • DestReceiver: クエリ結果の出力先を抽象化した構造体。
  • Query, PlannedStmt, QueryDesc: PostgreSQLのクエリ実行パイプラインで使用される主要構造体。

失敗モードと境界条件

  • 書き換え後のクエリが単一の SELECT 文でない場合、elog(ERROR) により処理が中断される。
  • 実行中に割り込みが発生した場合、CHECK_FOR_INTERRUPTS() によりエラーがスローされる可能性がある。

保守担当者向けメモ

  • 本関数は REFRESH MATERIALIZED VIEW のデータ生成フェーズを担う。
  • is_create フラグはエラーメッセージの文脈を切り替えるためにのみ使用される。
  • 実行計画の生成には CURSOR_OPT_PARALLEL_OK が指定されており、並列実行が許可されている。
