check_expect

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pgsql.c:146

目的

PGP暗号化コンテキスト（PGP_Context）の設定値が、期待されるデバッグ用設定（debug_expect）と一致しているかを検証します。

シグネチャまたは形状

static void check_expect(PGP_Context *ctx, struct debug_expect *ex);

入力と出力

  • ctx: 検証対象の PGP_Context 構造体へのポインタ。
  • ex: 期待値が格納された debug_expect 構造体へのポインタ。
  • 戻り値: なし（検証失敗時は EX_CHECK マクロを通じてエラーを発生させる）。

前提条件と不変条件

  • ctx および ex は有効なメモリ領域を指している必要があります。
  • EX_CHECK マクロが定義されており、期待値と実際の値が一致しない場合に例外を投げる環境であること。

アルゴリズムまたは振る舞い

1. cipher_algo, s2k_mode, s2k_count, s2k_digest_algo, use_sess_key, disable_mdc, compress_algo, unicode_mode の各フィールドについて、ctx の値と ex の期待値を比較します。 2. ctx->use_sess_key が真の場合のみ、追加で s2k_cipher_algo の検証を行います。 3. 一致しない場合、EX_CHECK マクロによりエラーが報告されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/pgp-pgsql.c 内の関数（例: 512 行目付近）。
  • 呼び出し先: EX_CHECK マクロ（実装詳細はソース excerpt に含まれていません）。

関連データ構造

  • PGP_Context: PGP処理のコンテキストを保持する構造体。
  • debug_expect: テスト目的で暗号化設定を検証するために使用される構造体。

失敗モードと境界条件

  • ctx または ex が NULL の場合、未定義の動作を引き起こす可能性があります。
  • EX_CHECK マクロが期待値との不一致を検知した場合、処理は中断されます。

保守担当者向けメモ

  • 本関数はテストおよびデバッグ専用です。通常の運用コードでは使用されません。
  • EX_CHECK マクロの定義およびエラーハンドリングの詳細は、pgp-pgsql.c の他の箇所または関連ヘッダーを確認してください。
