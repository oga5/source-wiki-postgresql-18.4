config_string

定義

  • 種別: struct
  • 定義箇所: src/include/utils/guc_tables.h:272

目的

PostgreSQLのGUC（Grand Unified Configuration）システムにおいて、文字列型の設定変数を定義および管理するための構造体です。

シグネチャまたは形状

struct config_string
{
    struct config_generic gen;
    char      **variable;
    const char *boot_val;
    GucStringCheckHook check_hook;
    GucStringAssignHook assign_hook;
    GucShowHook show_hook;
    char       *reset_val;
    void       *reset_extra;
};

入力と出力

  • **入力**: variable（設定値を保持する変数へのポインタ）、boot_val（デフォルト値）、および各種フック関数。
  • **出力**: 実行時に reset_val および reset_extra が初期化され、設定値の変更履歴や検証状態が管理されます。

前提条件と不変条件

  • gen フィールド（config_generic）を構造体の先頭に保持する必要があります。
  • boot_val などの定数フィールドは、初期化時に正しく設定されている必要があります。

アルゴリズムまたは振る舞い

  • GUCシステムによる設定値の変更時、check_hook で検証を行い、assign_hook で値を適用します。
  • reset_val および reset_extra は実行時に初期化され、設定の再読み込みやリセット処理に使用されます。
  • extra ポインタ（config_generic 経由）と reset_extra を用いて、検証後の複雑な設定値を保持・伝播します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/guc.c 内の関数群（call_string_check_hook, string_field_used, set_string_field など）が本構造体を操作します。
  • **呼び出し先**: check_hook, assign_hook, show_hook に登録された関数が呼び出されます。

関連データ構造

  • config_generic: 全てのGUC変数の基底構造体。
  • GucStringCheckHook, GucStringAssignHook, GucShowHook: 設定値の検証、割り当て、表示を行うための関数ポインタ型。

失敗モードと境界条件

  • variable が指す先が適切に確保されていない場合、メモリ破壊の可能性があります。
  • check_hook が失敗を返した場合、設定値の変更は拒否されます。

保守担当者向けメモ

  • config_generic の extra フィールドと reset_extra の関係性に注意してください。check_hook で生成された値が assign_hook に渡される際のブリッジとして機能します。
  • src/backend/utils/misc/guc.c における sizeof(struct config_string) + sizeof(char *) のような動的確保パターンがあるため、構造体のメモリレイアウト変更時は注意が必要です。
