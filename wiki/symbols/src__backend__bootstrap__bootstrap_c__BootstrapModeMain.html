BootstrapModeMain

定義

  • 種別: function
  • 定義箇所: src/backend/bootstrap/bootstrap.c:197

目的

PostgreSQLのブートストラップモード（initdb）におけるメインエントリポイントであり、システムカタログの初期構築環境をセットアップする。

シグネチャまたは形状

void BootstrapModeMain(int argc, char *argv[], bool check_only)

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。
  • check_only: true の場合、チェックモードとして動作し、カタログ構築を行わずに終了する。
  • **出力**: なし（成功時はプロセス終了、失敗時はエラー報告または異常終了）。

前提条件と不変条件

  • IsUnderPostmaster が false であること（スタンドアロンプロセスであること）。
  • 実行時に --boot または --check 引数が必須。
  • データディレクトリの初期化およびロックファイルの作成が完了していること。

アルゴリズムまたは振る舞い

1. **環境初期化**: InitStandaloneProcess および InitializeGUCOptions を実行。 2. **引数解析**: getopt を使用し、GUC設定（shared_buffers, fsync 等）やブートストラップ固有のオプションを処理。 3. **ディレクトリ設定**: checkDataDir および ChangeToDataDir で作業ディレクトリを確定。 4. **システム状態の準備**:

  • BootstrapProcessing モードへ移行。
  • IgnoreSystemIndexes = true を設定し、インデックスなしでのカタログ操作を許可。
  • 共有メモリ、セマフォ、およびバックエンド用スロットを初期化。

    • 5. **分岐**: check_only が true の場合、CheckerModeMain を呼び出して終了。 6. **ブートストラップ開始**:

  • InitProcess, BaseInit, bootstrap_signals を実行。
  • BootStrapXLOG でトランザクションログを初期化。
  • InitPostgres でバックエンド環境を構築。

    • 7. **終了処理**: 属性定義用のループ（MAXATTR）へ移行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/main/main.c（BootstrapModeMain(argc, argv, true) または false）
  • **呼び出し先**:
  • InitStandaloneProcess, InitializeGUCOptions, SetConfigOption, ParseLongOption, SelectConfigFiles, checkDataDir, ChangeToDataDir, CreateDataDirLockFile, SetProcessingMode, InitializeMaxBackends, InitPostmasterChildSlots, InitializeFastPathLocks, CreateSharedMemoryAndSemaphores, set_max_safe_fds, CheckerModeMain (条件付き), InitProcess, BaseInit, bootstrap_signals, BootStrapXLOG, pg_link_canary_is_frontend, InitPostgres

関連データ構造

  • MAXATTR: 属性定義ループの制限値。
  • OutputFileName: 出力ファイルパスを保持するグローバル変数。

失敗モードと境界条件

  • 引数に --boot または --check が含まれない場合、Assert により停止。
  • 不正なコマンドライン引数や、--c オプション等で値が欠落している場合はエラー報告を行い終了。
  • check_only モードでは、リソース不足（max_connections 過多など）を検出して終了する可能性がある。
  • pg_link_canary_is_frontend が真を返す場合、リンクエラーとして elog(ERROR) を発生させる。

保守担当者向けメモ

  • 本関数は「鶏と卵」の問題を解決するため、システムカタログが未構築の状態で動作する。
  • check_only モードの終了時に abort() が呼ばれる点に注意。
