test_lfind8_le_internal

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_lfind/test_lfind.c:74

目的

pg_lfind8_le 関数の正確性を検証するための内部ヘルパー関数。指定されたキー値に基づき、ベクトル演算（SIMD）が適用される範囲と、末尾の端数処理（テール処理）が発生する範囲の両方で、期待通りの探索結果が得られるかを確認する。

シグネチャまたは形状

static void test_lfind8_le_internal(uint8 key)

入力と出力

  • **入力**: key (検証対象の8ビット値)
  • **出力**: なし（検証失敗時に elog(ERROR) を発行してトランザクションをアボートする）

前提条件と不変条件

  • Vector8 型のサイズに基づいたバッファ長（LEN_WITH_TAIL）が定義されていること。
  • pg_lfind8_le が検索対象のバッファに対して正しく動作する環境であること。

アルゴリズムまたは振る舞い

1. **テール処理の検証**:

  • バッファを 0xFF で初期化し、末尾に key を配置する。
  • pg_lfind8_le を使用し、key-1（存在しないはず）、key（存在するはず）、key+1（存在するはず）の各値で検索を実行する。
  • 期待値と異なる結果が得られた場合、エラーを報告する。

    • 2. **ベクトル演算の検証**:

  • バッファを 0xFF で初期化し、ベクトル境界内（len_no_tail - 1）に key を配置する。
  • 同様に key-1、key、key+1 で検索を実行し、結果を検証する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_lfind/test_lfind.c 内のテスト関数（test_lfind8_le 等から、異なる key 値を指定して複数回呼び出される）。
  • **呼び出し先**: pg_lfind8_le（検証対象）、elog（エラー報告用）。

関連データ構造

  • Vector8: 探索アルゴリズムで使用されるベクトル型。
  • charbuf: テスト用の uint8 配列。

失敗モードと境界条件

  • key が 0x00 の場合、key-1 のチェックはスキップされる。
  • key が 0xFF または 0xFE の場合、key+1 のチェックはスキップされる。
  • 期待される値が見つからない、または存在しない値が見つかった場合に elog(ERROR) によりテストが中断される。

保守担当者向けメモ

  • 本関数は test_lfind8_le からの引数として、0, 1, 0x7F, 0x80, 0x81, 0xFD, 0xFE, 0xFF が渡されることを想定している。
  • テストケースを追加する際は、境界値付近の挙動が網羅されているか確認すること。
