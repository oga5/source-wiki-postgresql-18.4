intarray_push_elem

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_op.c:325

目的

PostgreSQLの整数配列（intarray）に対して、指定された整数要素を末尾に追加した新しい配列を生成するSQL呼び出し可能関数です。

シグネチャまたは形状

Datum intarray_push_elem(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_ARRAYTYPE_P(0): 操作対象の ArrayType（整数配列）。
  • PG_GETARG_INT32(1): 配列に追加する int32 型の整数値。
  • **出力**:
  • Datum: 要素が追加された新しい ArrayType へのポインタ。

前提条件と不変条件

  • 入力配列は intarray モジュールが扱う有効な整数配列である必要があります。
  • PG_FUNCTION_ARGS を介した標準的なPostgreSQL関数呼び出し規約に従います。

アルゴリズムまたは振る舞い

1. PG_GETARG_ARRAYTYPE_P を使用して入力配列を取得します。 2. intarray_add_elem を呼び出し、配列に要素を追加する内部処理を実行します。 3. PG_FREE_IF_COPY を使用して、入力配列がコピーである場合にメモリを解放します。 4. PG_RETURN_POINTER を介して、結果として生成された新しい配列を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: intarray_add_elem（要素追加の内部ロジック）。

関連データ構造

  • ArrayType: PostgreSQLの配列データ型。
  • Datum: PostgreSQLの汎用データ表現。

失敗モードと境界条件

  • 入力配列がNULLの場合や、メモリ不足時にはPostgreSQLの標準的なエラーハンドリング（elog 等）が呼び出される可能性があります。
  • 内部関数 intarray_add_elem の実装に依存します。

保守担当者向けメモ

  • 本関数はラッパーであり、実際の配列操作ロジックは intarray_add_elem に委譲されています。
  • メモリ管理のため、PG_GETARG_ARRAYTYPE_P で取得したポインタに対して適切に PG_FREE_IF_COPY を呼び出している点に注意してください。
