pg_wc_isgraph

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_pg_locale.c:442

目的

与えられたワイド文字（pg_wchar）が、現在のロケール設定において「表示可能な文字（graphical character）」であるかを判定します。

シグネチャまたは形状

static int pg_wc_isgraph(pg_wchar c)

入力と出力

  • **入力**: pg_wchar c（判定対象の文字コード）
  • **出力**: int（真であれば非ゼロ、偽であれば 0 を返す）

前提条件と不変条件

  • pg_regex_strategy が適切に設定されている必要があります。
  • pg_wchar は符号なし型として扱われます。
  • pg_regex_locale が有効なロケール情報を保持している必要があります（PG_REGEX_STRATEGY_LIBC_* 使用時）。

アルゴリズムまたは振る舞い

現在の pg_regex_strategy に基づき、以下のいずれかの戦略で判定を行います。 1. **PG_REGEX_STRATEGY_C**: pg_char_properties 配列を参照し、ASCII範囲内（127以下）の PG_ISGRAPH プロパティを確認します。 2. **PG_REGEX_STRATEGY_BUILTIN**: pg_u_isgraph(c) を呼び出し、組み込みのUnicodeテーブルで判定します。 3. **PG_REGEX_STRATEGY_LIBC_WIDE**: wchar_t のサイズに応じて iswgraph_l を使用します。 4. **PG_REGEX_STRATEGY_LIBC_1BYTE**: isgraph_l を使用し、文字が UCHAR_MAX 以下であることを確認した上で判定します。 5. **PG_REGEX_STRATEGY_ICU**: u_isgraph(c) を呼び出します（USE_ICU 定義時）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_locale.c:657 (pg_ctype_get_cache への引数として使用)
  • src/backend/regex/regc_locale.c:704
  • **呼び出し先**:
  • pg_u_isgraph
  • iswgraph_l
  • isgraph_l
  • u_isgraph

関連データ構造

  • pg_wchar: 判定対象の文字型。
  • PG_Locale_Strategy: 判定戦略を決定する列挙型。
  • pg_char_properties: Cロケール用プロパティテーブル。

失敗モードと境界条件

  • PG_REGEX_STRATEGY_LIBC_1BYTE 戦略において、文字が UCHAR_MAX を超える場合は偽（0）を返します。
  • PG_REGEX_STRATEGY_LIBC_WIDE 戦略において、wchar_t が4バイト未満かつ文字が 0xFFFF を超える場合、後続の LIBC_1BYTE 戦略へフォールスルーします。
  • どの戦略にも該当しない場合、安全のため 0 を返します。

保守担当者向けメモ

  • 本関数は regcomp.c からインクルードされることを前提としています。
  • ICU 関連の呼び出しはコンパイル時の USE_ICU マクロ定義に依存します。
