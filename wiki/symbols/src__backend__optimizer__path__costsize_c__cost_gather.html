cost_gather

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/costsize.c:445

目的

GatherPath ノードのコスト（startup_cost, total_cost）および行数（rows）を算出し、当該パス構造体に設定する。

シグネチャまたは形状

void cost_gather(GatherPath *path, PlannerInfo *root, RelOptInfo *rel, ParamPathInfo *param_info, double *rows)

入力と出力

  • **入力**:
  • path: コストを計算・設定する対象の GatherPath 構造体。
  • root: プランナのグローバル情報。
  • rel: 対象リレーション情報。
  • param_info: パラメータ化されたパス情報（存在する場合）。
  • rows: 行数の推定値（指定がある場合）。
  • **出力**:
  • なし（引数の path 構造体のフィールドを直接更新する）。

前提条件と不変条件

  • path->subpath が初期化済みであること。
  • parallel_setup_cost および parallel_tuple_cost がグローバルなコストパラメータとして利用可能であること。

アルゴリズムまたは振る舞い

1. **行数の決定**: 引数 rows があればそれを優先し、次に param_info->ppi_rows、最後に rel->rows を使用して path->path.rows を設定する。 2. **コスト計算**:

  • startup_cost をサブパスの startup_cost で初期化。
  • run_cost をサブパスの total_cost から startup_cost を引いた値で初期化。
  • 並列処理のセットアップコスト（parallel_setup_cost）を startup_cost に加算。
  • 並列タプル処理コスト（parallel_tuple_cost × 行数）を run_cost に加算。

    • 3. **結果の格納**: 計算したコストを path->path.startup_cost および path->path.total_cost に書き込む。また、サブパスの disabled_nodes を継承する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/pathnode.c:2208
  • **呼び出し先**:
  • 直接的な関数呼び出しは含まれないが、path->subpath の各フィールドを参照する。

関連データ構造

  • GatherPath: 本関数がコストを計算する対象のパス構造体。
  • PlannerInfo: プランナのコンテキスト情報。
  • RelOptInfo: リレーションの統計情報。

失敗モードと境界条件

  • rows が NULL の場合、param_info の有無に応じて行数推定元を切り替える。
  • subpath が無効な値を持つ場合、計算結果もそれに依存する。

保守担当者向けメモ

  • 本関数は GatherPath のコストモデルを定義しており、並列クエリのオーバーヘッドを考慮している。
  • src/backend/optimizer/plan/planner.c:511 に本関数の利用を検討する旨のコメントがあるが、現時点では pathnode.c からの呼び出しが主である。
