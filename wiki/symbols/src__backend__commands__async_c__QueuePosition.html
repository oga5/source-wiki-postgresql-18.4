QueuePosition

定義

  • **種別**: struct
  • **定義箇所**: src/backend/commands/async.c:194

目的

非同期通知システムにおいて、SLRU（Simple Least Recently Used）バッファ上の特定のデータ位置を指し示すためのポインタ構造体です。

シグネチャまたは形状

typedef struct QueuePosition
{
    int64       page;           /* SLRU page number */
    int         offset;         /* byte offset within page */
} QueuePosition;

入力と出力

  • **入力**: 構造体の各フィールド（page および offset）は、通知キュー内の絶対的な位置を特定するために使用されます。
  • **出力**: 該当しません。

前提条件と不変条件

  • page は SLRU ページ番号を表します。
  • offset は指定されたページ内でのバイトオフセットを表します。
  • キューの操作において、tail 位置は常に各バックエンドが読み取った位置以下である必要があります。

アルゴリズムまたは振る舞い

  • 通知キュー内の位置比較（論理的な大小比較）に使用されます。
  • asyncQueueAdvance 関数により、指定されたエントリ長分だけ位置を進めるために利用されます。
  • asyncQueueProcessPageEntries 関数において、処理範囲の開始点および終了点（current, stop）として使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/async.c 内のキュー管理関数群（asyncQueueAdvance, asyncQueueProcessPageEntries など）。
  • **呼び出し先**: 直接的な関数呼び出し関係はありません（データ構造体のため）。

関連データ構造

  • AsyncQueueControl: 通知キューのヘッドおよびテール位置を保持するために本構造体を使用します。
  • QueueBackendStatus: 各バックエンドの読み取り位置を保持するために本構造体を使用します。

失敗モードと境界条件

  • offset がページ境界を超えた場合、SLRU のページングロジックと整合させる必要があります。
  • 読み取り位置（tail）が遅延している場合、ディスク領域の消費が増大する可能性があります。

保守担当者向けメモ

  • 本構造体は async.c 内のキュー操作の基盤です。位置の比較や更新ロジックを変更する際は、AsyncQueueControl のロック順序（NotifyQueueTailLock -> NotifyQueueLock）を遵守してください。
