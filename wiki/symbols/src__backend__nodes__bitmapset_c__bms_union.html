bms_union

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/bitmapset.c:250

目的

2つの Bitmapset の和集合を計算し、新しい Bitmapset として返します。

シグネチャまたは形状

Bitmapset *bms_union(const Bitmapset *a, const Bitmapset *b)

入力と出力

  • **入力**:
  • a: 集合を表す Bitmapset へのポインタ（NULL 許容）。
  • b: 集合を表す Bitmapset へのポインタ（NULL 許容）。
  • **出力**:
  • a と b の和集合を含む、新しく割り当てられた Bitmapset へのポインタ。入力が両方とも NULL の場合は NULL を返します。

前提条件と不変条件

  • 入力 a および b は、bms_is_valid_set によって検証可能である必要があります。
  • 戻り値は呼び出し元が所有権を持ち、不要になった時点で解放する必要があります。

アルゴリズムまたは振る舞い

1. 入力 a または b が NULL の場合、他方のコピーを返します。 2. 両方の入力が有効な場合、ワード数（nwords）が大きい方の集合をコピーして result とします。 3. 小さい方の集合の各ワードを、do-while ループを用いて result の対応するワードに対してビット単位の論理和（|=）で結合します。 4. 結合後の result を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • contrib/sepgsql/dml.c
  • src/backend/executor/execIndexing.c
  • src/backend/executor/execMain.c
  • **呼び出し先**:
  • bms_is_valid_set (アサーション内)
  • bms_copy

関連データ構造

  • Bitmapset: 整数集合を保持する構造体。nwords（ワード数）と words（ビット配列）で構成されます。

失敗モードと境界条件

  • **NULL入力**: 片方が NULL の場合は他方のコピーを返し、両方が NULL の場合は NULL を返します。
  • **メモリ不足**: bms_copy 内部でメモリ割り当てに失敗した場合、システム標準のメモリ不足エラーが発生します。

保守担当者向けメモ

  • 本関数は入力集合を破壊せず、常に新しいメモリ領域を割り当てて結果を返します。
  • do-while ループは、入力集合のワード数に基づき、効率的にビット演算を行うよう最適化されています。
