get_command_type

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_ddl_deparse/test_ddl_deparse.c:29

目的

CollectedCommand 構造体の type フィールドを、人間が読み取り可能な文字列形式に変換して返します。主に DDL デパース機能のテストにおいて、コマンド種別を SQL レベルで検証するために使用されます。

シグネチャまたは形状

Datum get_command_type(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_POINTER(0) を介して渡される CollectedCommand * 型のポインタ。
  • **出力**: コマンド種別を表す text 型の PostgreSQL Datum。

前提条件と不変条件

  • 入力ポインタ cmd は有効な CollectedCommand 構造体を指している必要があります。
  • PostgreSQL の関数呼び出し規約（PG_FUNCTION_ARGS）に従う必要があります。

アルゴリズムまたは振る舞い

1. 引数から CollectedCommand 構造体を取得します。 2. cmd->type の列挙値に基づき、以下のいずれかの文字列を選択します。

  • SCT_Simple -> "simple"
  • SCT_AlterTable -> "alter table"
  • SCT_Grant -> "grant"
  • SCT_AlterOpFamily -> "alter operator family"
  • SCT_AlterDefaultPrivileges -> "alter default privileges"
  • SCT_CreateOpClass -> "create operator class"
  • SCT_AlterTSConfig -> "alter text search configuration"
  • その他 -> "unknown command type"

    • 3. 選択した文字列を cstring_to_text で変換し、PG_RETURN_TEXT_P を通じて返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の SQL インターフェースから呼び出されることを想定しています。ソースコード内では PG_FUNCTION_INFO_V1(get_command_type) による登録が確認できます。
  • **呼び出し先**: cstring_to_text などの PostgreSQL 組み込みユーティリティ関数を呼び出します。

関連データ構造

  • CollectedCommand: tcop/deparse_utility.h で定義される、DDL 操作情報を保持する構造体。

失敗モードと境界条件

  • cmd->type が定義された列挙値のいずれにも該当しない場合、"unknown command type" が返されます。
  • 入力ポインタが NULL の場合、PostgreSQL の標準的なポインタ参照エラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はテスト専用モジュールの一部です。
  • CollectedCommand の type 列挙型に新しいコマンド種別が追加された場合は、本関数の switch 文にもケースを追加する必要があります。
