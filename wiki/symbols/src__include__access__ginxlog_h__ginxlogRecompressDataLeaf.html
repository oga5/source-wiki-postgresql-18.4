ginxlogRecompressDataLeaf

定義

  • 種別: struct
  • 定義箇所: src/include/access/ginxlog.h:65

目的

GINインデックスのデータリーフページに対する再圧縮操作を記述し、WAL（Write-Ahead Log）リカバリ時にページ状態を再構築するためのメタデータを提供します。

シグネチャまたは形状

typedef struct
{
    uint16      nactions;
    /* 可変長の 'actions' が後続する */
} ginxlogRecompressDataLeaf;

入力と出力

  • **入力**: WALレコードの一部として書き込まれる構造体。nactions は実行されるアクションの数を示します。
  • **出力**: なし（構造体自体はデータ保持用）。

前提条件と不変条件

  • 構造体の直後に、nactions 分の可変長データがメモリ上に連続して配置されている必要があります。
  • WAL再生時には、この構造体のサイズ（sizeof(ginxlogRecompressDataLeaf)）をオフセットとして、後続のデータ領域を読み取る必要があります。

アルゴリズムまたは振る舞い

1. **記録**: GINデータページ更新時、src/backend/access/gin/gindatapage.c にて本構造体が初期化され、WALバッファに書き込まれます。 2. **再生**: src/backend/access/gin/ginxlog.c の ginRedoRecompress 関数が本構造体を受け取り、後続の可変長データ領域を解析して対象ページに適用します。 3. **可視化**: src/backend/access/rmgrdesc/gindesc.c の desc_recompress_leaf 関数が、WALダンプ出力のために本構造体から情報を抽出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gindatapage.c: WALレコード構築時に使用。
  • **呼び出し先**:
  • src/backend/access/gin/ginxlog.c: ginRedoRecompress 関数が本構造体を引数として受け取り、REDO処理を実行。
  • src/backend/access/rmgrdesc/gindesc.c: desc_recompress_leaf 関数がWAL内容の記述に使用。

関連データ構造

  • ginxlogInsert: GINの挿入操作用構造体（本構造体と並び、WALレコードの型として定義）。
  • StringInfo: gindesc.c でのWAL内容出力時に使用。

失敗モードと境界条件

  • **アライメント**: 構造体定義にフィールドを追加する場合、パディングによるバイナリ互換性の破壊に注意が必要です。
  • **データ長**: nactions の値と、実際に後続するデータサイズが一致しない場合、WAL読み取り時にメモリ破壊や不正なページ更新が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は memcpy で直接読み書きされるため、フィールドの順序やサイズ変更はWALのバイナリ互換性に直接影響します。
  • ginxlogInsert 等と同様に、可変長データが後続することを前提とした設計であるため、構造体定義の変更時は ginxlog.c および gindesc.c のポインタ演算箇所を必ず確認してください。
