InitPostmasterDeathWatchHandle

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/postmaster.c:4559

目的

Postmaster プロセスの生存を子プロセスが監視するためのメカニズムを初期化する。Postmaster が異常終了した際に、子プロセスがそれを検知して自律的に終了できるようにする。

シグネチャまたは形状

static void InitPostmasterDeathWatchHandle(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 postmaster_alive_fds または PostmasterHandle を設定）

前提条件と不変条件

  • 非 Windows 環境では、呼び出し時に MyProcPid == PostmasterPid であること。
  • 呼び出し後、子プロセスは書き込み側のファイル記述子（postmaster_alive_fds[POSTMASTER_FD_OWN]）を速やかに閉じる必要がある。

アルゴリズムまたは振る舞い

  • **非 Windows 環境**:

    • 1. pipe() を作成し、読み込み側と書き込み側のファイル記述子を postmaster_alive_fds に格納する。 2. ReserveExternalFD() を2回呼び出し、ファイル記述子の消費をシステムに通知する。 3. fcntl() を使用して、読み込み側の記述子を O_NONBLOCK モードに設定する。

  • **Windows 環境**:

    • 1. DuplicateHandle() を使用して、現在のプロセスハンドルを複製し、PostmasterHandle に格納する。

呼び出し元と呼び出し先

  • 呼び出し元: PostmasterMain (src/backend/postmaster/postmaster.c:1016)
  • 呼び出し先:
  • pipe, fcntl, DuplicateHandle, GetCurrentProcess, GetLastError, ReserveExternalFD, ereport (システムコールおよび内部ユーティリティ)

関連データ構造

  • postmaster_alive_fds (非 Windows 用のパイプ記述子配列)
  • PostmasterHandle (Windows 用のプロセスハンドル)

失敗モードと境界条件

  • パイプ作成失敗、ファイル記述子の設定失敗、またはハンドル複製失敗時に FATAL エラーを発生させ、Postmaster プロセスを終了させる。

保守担当者向けメモ

  • 子プロセスが読み込み側のパイプで read() == 0 を受け取った場合、Postmaster が終了したと判断できる。
  • Windows 環境では、プロセスハンドルがシグナル状態になることで Postmaster の終了を検知する。
