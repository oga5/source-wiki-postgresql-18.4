initPQExpBuffer

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/pqexpbuffer.c:89

目的

PQExpBuffer 構造体を初期化し、動的文字列バッファとして利用可能な状態にする。

シグネチャまたは形状

void initPQExpBuffer(PQExpBuffer str)

入力と出力

  • **入力**: str (初期化対象となる PQExpBuffer 構造体へのポインタ)
  • **出力**: なし（引数 str が指す構造体のメンバを直接更新する）

前提条件と不変条件

  • str は有効なメモリ領域を指している必要がある。
  • 成功時、str->data は INITIAL_EXPBUFFER_SIZE 分のメモリを保持し、str->len は 0、str->maxlen は初期サイズとなる。
  • メモリ確保失敗時、str->data は静的な oom_buffer_ptr を指し、str->len および str->maxlen は 0 に設定される。

アルゴリズムまたは振る舞い

1. malloc を使用して INITIAL_EXPBUFFER_SIZE 分のメモリ確保を試みる。 2. 確保に成功した場合:

  • str->maxlen に INITIAL_EXPBUFFER_SIZE を設定。
  • str->len を 0 に設定。
  • str->data[0] にヌル終端文字を書き込む。

    • 3. 確保に失敗した場合:

  • str->data を oom_buffer_ptr に設定（メモリ不足状態のマーク）。
  • str->maxlen と str->len を 0 に設定。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/libpqwalreceiver/libpqwalreceiver.c
  • src/bin/initdb/initdb.c
  • src/bin/pg_amcheck/pg_amcheck.c
  • **呼び出し先**: malloc（標準ライブラリ）

関連データ構造

  • PQExpBufferData: 本関数が初期化対象とする構造体。
  • oom_buffer_ptr: メモリ確保失敗時に参照される静的バッファ。

失敗モードと境界条件

  • malloc が NULL を返した場合、関数はエラーを返さず、内部的に「broken」状態（oom_buffer_ptr 参照）へ遷移する。この状態では以降のバッファ操作は無効化される。

保守担当者向けメモ

  • 本関数はメモリを確保するため、使用後は必ず destroyPQExpBuffer を呼び出してメモリを解放すること。
  • 「broken」状態のバッファは、resetPQExpBuffer を呼び出すことで再利用可能な状態へ復帰を試みることが可能。
