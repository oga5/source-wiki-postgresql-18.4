pgstat_prepare_io_time

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_io.c:90

目的

I/O操作の実行時間を計測するための開始時刻（タイムスタンプ）を準備します。

シグネチャまたは形状

instr_time pgstat_prepare_io_time(bool track_io_guc)

入力と出力

  • **入力**: track_io_guc (bool) - I/Oタイミング計測を有効にするかを示すフラグ。
  • **出力**: instr_time - 計測開始時刻。計測が無効な場合はゼロ値が返されます。

前提条件と不変条件

  • track_io_guc が true の場合、現在時刻が取得されます。
  • track_io_guc が false の場合、戻り値はゼロ値となります。これは後続の pgstat_count_io_op_time() において、タイミング計測を無視すべきであるというシグナルとして機能します。

アルゴリズムまたは振る舞い

1. 引数 track_io_guc を評価します。 2. true の場合、INSTR_TIME_SET_CURRENT マクロを使用して現在のシステム時刻を io_start に格納します。 3. false の場合、INSTR_TIME_SET_ZERO マクロを使用して io_start をゼロに初期化します。 4. 決定した io_start を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c (WAL関連)
  • src/backend/access/transam/xlogreader.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/replication/walreceiver.c
  • src/backend/storage/buffer/bufmgr.c (バッファ管理関連)
  • **呼び出し先**:
  • executor/instrument.h で定義された INSTR_TIME_SET_CURRENT および INSTR_TIME_SET_ZERO マクロ。

関連データ構造

  • instr_time: 計測時間を保持する型。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。計測が無効な場合にゼロ値を返すことで、後続処理での誤った計算を防ぐ設計となっています。

保守担当者向けメモ

  • 本関数はI/O統計の計測開始点です。計測対象のI/O操作を追加する際は、必ず本関数で開始時刻を取得し、終了後に pgstat_count_io_op_time() へその値を渡す必要があります。
