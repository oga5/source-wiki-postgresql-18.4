create_collation_from_locale

定義

  • 種別: function
  • 定義箇所: src/backend/commands/collationcmds.c:695

目的

OSから取得したロケール名を基に、PostgreSQLの照合順序（Collation）オブジェクトをシステムカタログ pg_collation に登録する。

シグネチャまたは形状

static int create_collation_from_locale(const char *locale, int nspid, int *nvalidp, int *ncreatedp);

入力と出力

  • **入力**:
  • locale: ロケール名（文字列）。
  • nspid: 登録先名前空間のOID。
  • nvalidp: 有効なロケールとしてカウントされた総数へのポインタ。
  • ncreatedp: 新規作成された照合順序の数へのポインタ。
  • **出力**:
  • 成功時: 登録された照合順序に対応するエンコーディングID（整数）。
  • 失敗時: -1。

前提条件と不変条件

  • ロケール名はASCII文字のみで構成されている必要がある。
  • ロケールに対応するエンコーディングがPostgreSQLで認識可能かつバックエンドで使用可能であること。
  • PG_SQL_ASCII エンコーディングは登録対象外。

アルゴリズムまたは振る舞い

1. **検証**: pg_is_ascii でロケール名をチェックし、非ASCII文字を含む場合はスキップする。 2. **エンコーディング確認**: pg_get_encoding_from_locale でロケールに対応するエンコーディングを特定し、有効性を検証する。 3. **カウント**: 有効なロケールとして nvalidp をインクリメントする。 4. **カタログ登録**: CollationCreate を呼び出し、COLLPROVIDER_LIBC プロバイダとしてカタログへ挿入する。既に存在する場合は何もしない。 5. **状態更新**: 新規作成された場合は ncreatedp をインクリメントし、CommandCounterIncrement を実行してトランザクション内の可視性を確保する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/collationcmds.c 内の pg_import_system_collations 関数から複数回呼び出される。
  • **呼び出し先**: pg_is_ascii, pg_get_encoding_from_locale, CollationCreate, get_collation_actual_version, CommandCounterIncrement。

関連データ構造

  • pg_collation: 照合順序情報を保持するシステムカタログ。

失敗モードと境界条件

  • 非ASCIIロケール名、認識不能なエンコーディング、またはクライアント専用エンコーディングの場合は -1 を返し、登録をスキップする。
  • 既に存在するロケール名に対しては、エラーを出さずに処理を継続する。

保守担当者向けメモ

  • 本関数は pg_import_system_collations による一括インポート処理の一部として機能する。
  • COLLPROVIDER_LIBC を前提としており、他のプロバイダ（ICU等）には対応していない。
  • CommandCounterIncrement は、同一トランザクション内での重複登録を正しく処理するために必須である。
