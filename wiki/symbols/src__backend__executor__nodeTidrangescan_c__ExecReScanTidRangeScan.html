ExecReScanTidRangeScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTidrangescan.c:319

目的

TID範囲スキャンノードの状態をリセットし、次回のスキャン実行時に範囲の再計算とスキャンの再開が行われるように準備する。

シグネチャまたは形状

void ExecReScanTidRangeScan(TidRangeScanState *node)

入力と出力

  • **入力**: TidRangeScanState *node - 再スキャン対象となるTID範囲スキャンノードの状態構造体。
  • **出力**: なし（副作用として node の状態を更新）。

前提条件と不変条件

  • node は有効な TidRangeScanState を指している必要がある。
  • 本関数呼び出し後、TidRangeNext が呼び出されるまでテーブルアクセス層の再スキャン（table_rescan_tidrange）は保留される。

アルゴリズムまたは振る舞い

1. node->trss_inScan を false に設定し、スキャンが進行中ではない状態にする。 2. TID範囲リストが未計算であることを示す状態にする。 3. ExecScanReScan(&node->ss) を呼び出し、共通のスキャンノード再スキャン処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:202 (直接の呼び出しを確認)
  • **呼び出し先**: ExecScanReScan (共通基盤関数)

関連データ構造

  • TidRangeScanState: TID範囲スキャンの実行状態を保持する構造体。
  • ScanState: TidRangeScanState が継承する基底構造体。

失敗モードと境界条件

  • node が NULL の場合、セグメンテーション違反が発生する可能性がある（呼び出し元でのガードは本ソース内では確認できない）。

保守担当者向けメモ

  • 本関数はスキャン状態のフラグをリセットするのみであり、実際のテーブルアクセス層の再スキャンは TidRangeNext 内で遅延実行される設計となっている。この実行順序を変更しないこと。
