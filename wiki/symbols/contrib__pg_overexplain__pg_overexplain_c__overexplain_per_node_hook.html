overexplain_per_node_hook

定義

  • 種別: function
  • 定義箇所: contrib/pg_overexplain/pg_overexplain.c:133

目的

PostgreSQL の EXPLAIN コマンド実行時に、プランノードごとの詳細な内部情報を追加出力するためのフック関数です。

シグネチャまたは形状

static void overexplain_per_node_hook(
    PlanState *planstate, 
    List *ancestors, 
    const char *relationship, 
    const char *plan_name, 
    ExplainState *es
)

入力と出力

  • **入力**:
  • planstate: 現在処理中のプランノードの状態。
  • ancestors, relationship, plan_name: EXPLAIN サブシステムから渡される標準的なコンテキスト情報。
  • es: EXPLAIN の出力状態を管理する構造体。
  • **出力**: なし（ExplainState を介して出力ストリームに情報を書き込む）。

前提条件と不変条件

  • ExplainState に es_extension_id を用いて overexplain_options がアタッチされている必要があります。
  • prev_explain_per_node_hook が設定されている場合、本関数は必ずそれを先に呼び出し、標準的な EXPLAIN 出力を維持します。

アルゴリズムまたは振る舞い

1. **標準フックの実行**: prev_explain_per_node_hook が存在すれば呼び出します。 2. **オプションの取得**: GetExplainExtensionState を使用して、現在の ExplainState から overexplain_options を取得します。 3. **デバッグ情報の出力**: options->debug が有効な場合、以下の情報を出力します。

  • disabled_nodes (整数)
  • parallel_safe (真偽値)
  • plan_node_id (整数)
  • extParam, allParam (ビットマップセット、TEXT形式以外または空でない場合)

    • 4. **レンジテーブル情報の出力**: options->range_table が有効な場合、プランノードのタグ（nodeTag）に応じて、関連するリレーションID（RTI）やビットマップセットを ExplainProperty 系関数で出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の EXPLAIN サブシステム（explain_per_node_hook フック経由）。
  • **呼び出し先**:
  • prev_explain_per_node_hook（存在する場合）
  • GetExplainExtensionState
  • ExplainPropertyInteger, ExplainPropertyBool
  • overexplain_bitmapset

関連データ構造

  • PlanState, Plan: プランノードの実行時および静的情報。
  • ExplainState: EXPLAIN コマンドの出力状態。
  • overexplain_options: 本拡張モジュールが保持するオプション設定。

失敗モードと境界条件

  • GetExplainExtensionState が NULL を返した場合、何も出力せずに即座に終了します。
  • nodeTag が default ケース（対象外のノード）に該当する場合、レンジテーブル情報は出力されません。
  • TEXT 形式の出力時、extParam や allParam が空の場合は冗長さを避けるため出力が抑制されます。

保守担当者向けメモ

  • 本関数は EXPLAIN のフックチェーンの一部として動作します。新しいプランノードタイプを追加する場合は、switch 文の case 節を更新してください。
  • ExplainProperty 系関数を使用することで、JSON や YAML などの出力フォーマットへの自動適応が保証されます。
