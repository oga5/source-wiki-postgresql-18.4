sqlca_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/thread-prep.c:47

目的

ECPG（Embedded SQL for C）において、SQL文の実行結果やエラー情報を保持・伝達するための通信領域（SQL Communication Area）を定義する構造体です。

シグネチャまたは形状

struct sqlca_t {
    char        sqlcaid[8];
    long        sqlabc;
    long        sqlcode;
    struct {
        int     sqlerrml;
        char    sqlerrmc[SQLERRMC_LEN];
    }           sqlerrm;
    char        sqlerrp[8];
    long        sqlerrd[6];
    char        sqlwarn[8];
    char        sqlstate[5];
};

入力と出力

  • **入力**: SQL実行エンジンが、実行結果（成功・失敗コード、警告、処理行数など）を本構造体の各フィールドに書き込みます。
  • **出力**: アプリケーション側が、エラーハンドリングや実行後の状態確認のために各フィールドを参照します。

前提条件と不変条件

  • sqlerrd[2] は、INSERT、UPDATE、DELETE 文の実行後に処理された行数を保持します。
  • sqlwarn[0] は、他の警告フラグが一つでもセットされた場合に 'W' に設定されます。
  • sqlstate[5] は、SQL標準に準拠した5文字のエラーコードを保持します。

アルゴリズムまたは振る舞い

ECPGライブラリの各関数（ECPGdo 等）が実行される際、その結果がこの構造体に反映されます。アプリケーションはこれを確認することで、後続の処理を継続するか、エラー処理へ分岐するかを判断します。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPGget_sqlca() を介して、src/interfaces/ecpg/ecpglib/connect.c、data.c、descriptor.c および src/interfaces/ecpg/compatlib/informix.c 等のECPGライブラリ内部関数から参照・更新されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • sqlda_t: SQL記述子領域。動的SQLにおいてパラメータや結果セットの情報を管理するために併用されます。

失敗モードと境界条件

  • sqlerrmc のサイズは SQLERRMC_LEN に制限されており、エラーメッセージがこれを超える場合は切り詰められる可能性があります。
  • sqlerrd 配列の各要素は特定の用途（OID、処理行数など）に固定されており、用途外のアクセスは未定義の振る舞いを引き起こします。

保守担当者向けメモ

  • 本構造体はECPGの標準的なインターフェースであるため、変更時は互換性に十分注意してください。
  • thread-prep.c はテスト用の期待値ファイルであるため、本構造体の定義自体は sqlca.h 等の共通ヘッダーから提供されるべきものです。本ファイル内の定義は、プリプロセッサの出力結果として生成されたものです。
