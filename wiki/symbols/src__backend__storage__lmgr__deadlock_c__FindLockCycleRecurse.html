FindLockCycleRecurse

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/deadlock.c:456

目的

ロック待ちグラフを再帰的に探索し、プロセス間の循環参照（デッドロック）を検出します。

シグネチャまたは形状

static bool FindLockCycleRecurse(PGPROC *checkProc, int depth, EDGE *softEdges, int *nSoftEdges)

入力と出力

  • **入力**:
  • checkProc: 探索対象のプロセス（PGPROC）。
  • depth: 現在の再帰深さ。
  • softEdges: 検出されたソフトエッジを格納する配列（出力引数）。
  • nSoftEdges: softEdges の現在の要素数（出力引数）。
  • **出力**:
  • bool: 循環参照が検出された場合は true、それ以外は false。

前提条件と不変条件

  • visitedProcs 配列および nVisitedProcs 変数が、探索中のプロセス追跡のために外部で管理されている必要があります。
  • checkProc がロックグループのメンバーである場合、内部でリーダープロセスに置き換えられます。
  • depth は MaxBackends を超えないことが保証されます。

アルゴリズムまたは振る舞い

1. **リーダー解決**: checkProc がロックグループに属する場合、リーダープロセスを対象とします。 2. **訪問済みチェック**: visitedProcs を走査し、既に訪問済みか確認します。

  • 最初の要素（開始点）と一致した場合、デッドロックと判定し true を返します。
  • それ以外の要素と一致した場合は、開始点を含まないサイクルであるため false を返します。

    • 3. **状態更新**: 未訪問であれば visitedProcs に追加します。 4. **再帰探索**:

  • プロセスが待機中であれば、FindLockCycleRecurseMember を呼び出し、ブロックしているプロセスを探索します。
  • プロセスが待機中でない場合でも、ロックグループのメンバーが待機中であれば、同様に FindLockCycleRecurseMember を通じて探索を継続します。

    • 5. 探索の結果、いずれかの経路でサイクルが発見されれば true を伝播します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/deadlock.c 内の複数箇所（FindLockCycle からの初期呼び出し、および再帰的な探索過程）。
  • **呼び出し先**:
  • FindLockCycleRecurseMember

関連データ構造

  • PGPROC: プロセス状態を保持する構造体。
  • EDGE: ロック待ちグラフ上のエッジ情報。
  • visitedProcs: 探索済みプロセスを記録するグローバル配列（ファイルスコープ）。

失敗モードと境界条件

  • nVisitedProcs が MaxBackends を超える場合、アサーションエラーが発生します。
  • 探索深さが MaxBackends を超える場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は DeadLockCheck の一部として、ロック待ちグラフの深さ優先探索を行います。
  • ロックグループの概念を考慮しており、メンバー個別の待機状態もサイクル検出の対象となります。
  • 探索結果として deadlockDetails に詳細情報を記録する責務を一部担っています。
