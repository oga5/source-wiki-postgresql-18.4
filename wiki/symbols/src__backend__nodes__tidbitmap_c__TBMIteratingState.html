TBMIteratingState

定義

  • 種別: enum
  • 定義箇所: src/backend/nodes/tidbitmap.c:130

目的

TIDBitmap における反復処理（イテレーション）の現在の状態を管理し、ビットマップが読み取り専用のページおよびチャンク配列へ変換済みかどうかを追跡するために使用されます。

シグネチャまたは形状

typedef enum
{
    TBM_NOT_ITERATING,      /* 反復処理未開始 */
    TBM_ITERATING_PRIVATE,  /* ローカルのページ/チャンク配列へ変換済み */
    TBM_ITERATING_SHARED,   /* 共有メモリ上のページ/チャンク配列へ変換済み */
} TBMIteratingState;

入力と出力

  • 本シンボル自体は状態値の列挙型です。
  • TIDBitmap 構造体内の iterating メンバとして保持されます。

前提条件と不変条件

  • TBM_NOT_ITERATING の状態では、ビットマップはまだ反復処理用にソート・変換されていません。
  • TBM_ITERATING_PRIVATE または TBM_ITERATING_SHARED に遷移した後は、ビットマップは読み取り専用となり、構造の変更は制限されます。

アルゴリズムまたは振る舞い

  • tbm_begin_iterate が呼び出されると、ビットマップの状態に応じて適切な反復モードへ遷移します。
  • 反復処理の終了後、この状態フラグにより、後続の反復処理やメモリ解放処理がどのデータ構造（プライベートまたは共有）を参照すべきかが決定されます。

呼び出し元と呼び出し先

  • 呼び出し元: TIDBitmap 構造体の iterating メンバ（src/backend/nodes/tidbitmap.c:150）。
  • 呼び出し先: 具体的な関数呼び出しの詳細はソース抜粋からは不明です。

関連データ構造

  • TIDBitmap: 本状態を保持するメイン構造体。
  • TBMPrivateIterator: TBM_ITERATING_PRIVATE 時に使用される反復子。

失敗モードと境界条件

  • 状態遷移の整合性が保たれていない場合（例：未変換の状態で共有反復子を使用しようとする等）、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本列挙型は TIDBitmap のライフサイクル管理において重要です。新しい反復モードを追加する場合は、この列挙型を拡張し、tbm_begin_iterate 内の遷移ロジックを更新してください。
