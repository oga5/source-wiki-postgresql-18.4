InitializeSystemUser

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/miscinit.c:924

目的

認証識別子（authn_id）と認証手法（auth_method）を結合し、システム全体の識別子として SystemUser グローバル変数に永続的に格納する。

シグネチャまたは形状

void InitializeSystemUser(const char *authn_id, const char *auth_method)

入力と出力

  • **入力**:
  • authn_id: 認証されたユーザー識別子（NULL不可）。
  • auth_method: 認証に使用された手法を示す文字列。
  • **出力**: なし（副作用として SystemUser グローバル変数を更新）。

前提条件と不変条件

  • SystemUser がまだ初期化されていないこと（Assert(SystemUser == NULL)）。
  • authn_id が NULL でないこと。
  • TopMemoryContext が有効であること。

アルゴリズムまたは振る舞い

1. auth_method と authn_id を auth_method:authn_id の形式で文字列結合する。 2. TopMemoryContext を使用して、結合された文字列を永続的なメモリ領域にコピーする。 3. SystemUser グローバル変数にそのポインタを代入する。 4. 一時的に作成した文字列バッファを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c:1554
  • src/backend/utils/init/postinit.c:904
  • **呼び出し先**:
  • psprintf
  • MemoryContextStrdup
  • pfree

関連データ構造

  • SystemUser: 本関数によって設定されるグローバル変数。

失敗モードと境界条件

  • SystemUser が既に設定されている状態で呼び出された場合、アサーション違反によりプロセスが停止する。
  • authn_id が NULL の場合、アサーション違反によりプロセスが停止する。

保守担当者向けメモ

  • 本関数はプロセスライフサイクル内で一度だけ実行されることを想定している。
  • SystemUser は TopMemoryContext に割り当てられるため、プロセス終了まで生存する。
