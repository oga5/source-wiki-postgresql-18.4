sepgsql_audit_log

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/selinux.c:677

目的

SELinux のアクセス制御判定結果を PostgreSQL のログに出力する。

シグネチャまたは形状

void sepgsql_audit_log(bool denied,
                       bool enforcing,
                       const char *scontext,
                       const char *tcontext,
                       uint16 tclass,
                       uint32 audited,
                       const char *audit_name)

入力と出力

  • **入力**:
  • denied: 判定が拒否されたか否か。
  • enforcing: SELinux が強制モードであるか否か。
  • scontext, tcontext: 主体および対象のセキュリティコンテキスト文字列。
  • tclass: オブジェクトクラスの識別子。
  • audited: 監査対象の権限ビットマスク。
  • audit_name: (オプション) 監査対象オブジェクトの名称。
  • **出力**: なし（ereport(LOG, ...) を通じてログを出力）。

前提条件と不変条件

  • tclass は SEPG_CLASS_MAX 未満である必要がある（Assert による検証）。
  • selinux_catalog が初期化されており、指定された tclass に対応するクラス名および権限名が定義されていること。

アルゴリズムまたは振る舞い

1. tclass をキーとして selinux_catalog からオブジェクトクラス名を取得する。 2. StringInfo バッファを初期化し、denied の値に基づいて "denied" または "allowed" を記録する。 3. audited ビットマスクを走査し、対応する権限名を selinux_catalog から取得してバッファに追加する。 4. 主体・対象のコンテキスト、クラス名、および（存在すれば）オブジェクト名をバッファに追加する。 5. enforcing フラグに基づき、permissive=0 または permissive=1 を付与する。 6. 構築した文字列を ereport(LOG, ...) で出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/uavc.c (判定結果のログ出力時)
  • **呼び出し先**:
  • initStringInfo, appendStringInfo, appendStringInfoString (PostgreSQL 文字列操作関数)
  • ereport (PostgreSQL ログ出力関数)

関連データ構造

  • selinux_catalog: オブジェクトクラスと権限名のマッピングを保持するグローバル構造体。
  • StringInfoData: ログメッセージ構築用の一時バッファ。

失敗モードと境界条件

  • tclass が範囲外の場合、Assert によりデバッグビルドでは異常終了する。
  • selinux_catalog に定義されていない権限ビットが audited に含まれる場合、その権限名はログに出力されない。

保守担当者向けメモ

  • 本関数は lib/stringinfo.h を利用して動的にログメッセージを構築するため、メモリ管理は PostgreSQL のメモリコンテキストに依存する。
  • 監査ログのフォーマットを変更する場合は、selinux_catalog の構造と整合性を取る必要がある。
