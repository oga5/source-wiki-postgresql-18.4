l_gep

定義

  • 種別: function
  • 定義箇所: src/include/jit/llvmjit_emit.h:113

目的

LLVM IR の GetElementPtr (GEP) 命令を生成するためのラッパー関数です。LLVM C API の LLVMBuildGEP2 を呼び出し、構造体や配列の要素アドレス計算を簡略化します。

シグネチャまたは形状

LLVMValueRef l_gep(LLVMBuilderRef b, LLVMTypeRef t, LLVMValueRef v, LLVMValueRef *indices, int32 nindices, const char *name)

入力と出力

  • **入力**:
  • b: 命令を挿入する LLVMBuilderRef。
  • t: ポインタが指す要素の LLVMTypeRef。
  • v: ベースとなるポインタ値 (LLVMValueRef)。
  • indices: インデックス値の配列 (LLVMValueRef *)。
  • nindices: インデックスの数 (int32)。
  • name: 生成される命令の名前 (const char *)。
  • **出力**:
  • 計算されたアドレスを表す LLVMValueRef。

前提条件と不変条件

  • 本関数は USE_LLVM が定義されている環境でのみ有効です。
  • LLVMBuilderRef が有効な状態である必要があります。

アルゴリズムまたは振る舞い

内部で LLVMBuildGEP2 を呼び出し、指定されたビルダに対して GEP 命令を生成します。インライン関数として定義されており、呼び出し側のコンテキストで直接展開されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit_deform.c
  • src/backend/jit/llvm/llvmjit_expr.c
  • **呼び出し先**:
  • LLVMBuildGEP2 (LLVM C API)

関連データ構造

  • LLVMBuilderRef: 命令生成のコンテキスト。
  • LLVMValueRef: LLVM IR 上の値。
  • LLVMTypeRef: LLVM IR 上の型。

失敗モードと境界条件

  • LLVM C API の仕様に準拠します。無効なインデックスや型が渡された場合、LLVM 側のエラーハンドリング（通常はアボート）に従います。

保守担当者向けメモ

  • 本関数は LLVMBuildGEP2 の単純なラッパーです。LLVM のバージョンアップに伴い API が変更された場合、ここを修正することで一括対応が可能です。
  • 構造体メンバーへのアクセスには、より特化した l_struct_gep の利用も検討してください。
