check_valid_int2vector

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/int.c:144

目的

int2vector 型のデータ構造が、期待される内部レイアウト（次元数、オフセット、要素型）を満たしているかを検証します。

シグネチャまたは形状

static void check_valid_int2vector(const int2vector *int2Array)

入力と出力

  • **入力**: int2Array (const int2vector *) - 検証対象の int2vector 構造体へのポインタ。
  • **出力**: なし。検証に失敗した場合は ereport(ERROR, ...) を呼び出し、トランザクションを中断します。

前提条件と不変条件

  • ndim が 1 であること。
  • dataoffset が 0 であること（NULL値を含まないこと）。
  • elemtype が INT2OID であること。

アルゴリズムまたは振る舞い

1. 引数として受け取った int2Array の各フィールドを検査します。 2. ndim != 1、dataoffset != 0、または elemtype != INT2OID のいずれか一つでも真であれば、ERRCODE_DATATYPE_MISMATCH エラーを発生させます。 3. 条件をすべて満たす場合は、何も返さずに正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/int.c 内の関数（直接的な呼び出しを確認）。
  • **呼び出し先**:
  • ereport（PostgreSQLの標準エラー報告マクロ）。

関連データ構造

  • int2vector: int2 型の配列を保持する特殊な内部データ構造。

失敗モードと境界条件

  • **失敗モード**: 構造体のレイアウトが期待と異なる場合、ereport によりエラーがスローされます。
  • **境界条件**: 配列のインデックス下限値（lower bound）については検証を行いません。

保守担当者向けメモ

  • 本関数は、int2vector が通常の int2[] 配列とは異なる内部構造を持つことに起因する整合性チェックを目的としています。
  • array_send のような、レイアウトを厳密に問わない処理では本関数をスキップするケースがあります。
