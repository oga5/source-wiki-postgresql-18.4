parse_lquery

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_io.c:268

目的

文字列形式の lquery（検索パターン）を解析し、内部バイナリ構造体 lquery を構築する。

シグネチャまたは形状

static lquery *parse_lquery(const char *buf, struct Node *escontext);

入力と出力

  • **入力**:
  • buf: 解析対象の lquery 文字列。
  • escontext: エラー報告用のコンテキスト（Node 構造体）。
  • **出力**:
  • 成功時: 構築された lquery 構造体へのポインタ。
  • 失敗時: NULL（エラーは escontext を通じて報告される）。

前提条件と不変条件

  • buf は有効な文字列であること。
  • 解析中に LQUERY_MAX_LEVELS を超える階層が検出された場合、エラーを返す。
  • メモリ確保には palloc0 を使用し、PostgreSQL のメモリ管理下で動作する。

アルゴリズムまたは振る舞い

1. 文字列をスキャンし、階層数（.）と OR 条件数（|）を事前カウントする。 2. palloc0 を用いて必要なメモリを確保する。 3. ステートマシン（LQPRS_ 系状態）を用いて文字列を走査する。

  • LQPRS_WAITLEVEL: ラベル、!（NOT）、または *（ワイルドカード）を待機。
  • LQPRS_WAITDELIM: ラベル後の修飾子（@, *, %）や区切り文字（|, {, .）を処理。
  • LQPRS_WAITOPEN: ワイルドカード後の { を待機。
  • LQPRS_WAITFNUM / LQPRS_WAITSNUM: {} 内の数値範囲を解析。

    • 4. finish_nodeitem を呼び出し、各ノードの妥当性を検証する。 5. 最終的な lquery 構造体を完成させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/ltree_io.c:764 (lquery_in 関数内)
  • contrib/ltree/ltree_io.c:826
  • **呼び出し先**:
  • finish_nodeitem: 各ノードの検証と確定。
  • palloc0: メモリ確保。
  • pg_mblen_cstr: マルチバイト文字の長さ取得。
  • t_iseq: 文字比較。
  • ereturn: エラー報告。

関連データ構造

  • lquery: 戻り値となる構造体。
  • lquery_level: 階層ごとの情報を保持。
  • nodeitem: 解析中の一時的なラベル情報を保持。

失敗モードと境界条件

  • **構文エラー**: 不正な文字や順序が検出された場合、UNCHAR マクロによりエラーを返す。
  • **制限超過**: 階層数が LQUERY_MAX_LEVELS を超える場合、または数値範囲が LTREE_MAX_LEVELS を超える場合にエラーを返す。
  • **メモリ不足**: palloc0 が失敗した場合、PostgreSQL のエラー処理機構により中断される。

保守担当者向けメモ

  • lquery は *, |, !, {} などのメタ文字をサポートしており、解析ロジックはステートマシンに依存している。変更を加える際は LQPRS_ 状態遷移の整合性に注意すること。
  • エラー報告には escontext を使用しており、直接 elog を呼ぶのではなく、提供されたエラーコンテキスト経由で報告する設計となっている。
