texticregexeq_support

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like_support.c:138

目的

大文字小文字を区別しない正規表現演算子（~*）に対するプランナのサポート関数として機能し、インデックススキャンを可能にするための条件生成や選択率推定を仲介します。

シグネチャまたは形状

Datum texticregexeq_support(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して Node *rawreq（プランナからのサポート要求）を受け取ります。
  • **出力**: like_regex_support の実行結果である Datum を返します。

前提条件と不変条件

  • rawreq は SupportRequestSelectivity または SupportRequestIndexCondition のいずれかとして解釈可能なノードである必要があります。

アルゴリズムまたは振る舞い

1. 引数から rawreq を取得します。 2. Pattern_Type_Regex_IC（大文字小文字を区別しない正規表現）を指定して like_regex_support を呼び出します。 3. like_regex_support が返す処理結果（インデックス条件のリストまたは選択率の推定値）をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: プランナ（インデックスの利用可能性を判断する過程で呼び出されます）。
  • **呼び出し先**: like_regex_support。

関連データ構造

  • Pattern_Type: パターンマッチングの種類を定義する列挙型。ここでは Pattern_Type_Regex_IC が使用されます。

失敗モードと境界条件

  • 呼び出し元から不正な rawreq が渡された場合、like_regex_support 内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的なロジックは like_regex_support に集約されています。
  • 大文字小文字の区別に関するロジックを変更する場合は、Pattern_Type_Regex_IC を受け取る側の実装を確認してください。
