tbm_exhausted

定義

  • 種別: function
  • 定義箇所: src/include/nodes/tidbitmap.h:118

目的

指定された TBMIterator が走査を終了したかどうかを判定します。

シグネチャまたは形状

static inline bool tbm_exhausted(TBMIterator *iterator)

入力と出力

  • **入力**: TBMIterator *iterator（走査対象のイテレータ）
  • **出力**: bool 型。走査が終了している（またはイテレータが無効化されている）場合に true を返し、継続可能な場合に false を返します。

前提条件と不変条件

  • iterator は有効なポインタである必要があります。
  • tbm_end_iterate() が呼び出された後、内部のイテレータポインタは NULL に設定されるため、本関数は true を返します。

アルゴリズムまたは振る舞い

内部の共用体メンバ i.private_iterator の存在を確認します。プライベートイテレータと共有イテレータのどちらが使用されているかに関わらず、この判定結果はイテレータの有効性を示します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeBitmapHeapscan.c: ビットマップヒープスキャンのループ条件として使用。
  • src/backend/nodes/tidbitmap.c: Assert 文内での状態検証に使用。
  • **呼び出し先**:
  • 特になし（インライン関数として定義）。

関連データ構造

  • TBMIterator: 走査状態を保持する構造体。
  • TBMPrivateIterator / TBMSharedIterator: TBMIterator 内部で管理される実体。

失敗モードと境界条件

  • iterator が NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • tbm_end_iterate() 呼び出し後のイテレータに対して本関数を呼び出すことは、不変条件により安全に true を返します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結しています。
  • 内部の i.private_iterator を参照していますが、これは共有イテレータの場合でも同様に終了判定として機能する設計となっています。
