errdetail_relkind_not_supported

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_class.c:23

目的

リレーションの種類（relkind）が特定の操作をサポートしていない場合に、エラー詳細メッセージを生成して errdetail() に設定する。

シグネチャまたは形状

int errdetail_relkind_not_supported(char relkind)

入力と出力

  • **入力**: char relkind（pg_class.h で定義されるリレーション種別コード）。
  • **出力**: int（errdetail() の戻り値）。

前提条件と不変条件

  • 呼び出し元は、操作対象のリレーションがその操作をサポートしていないと判断した後に本関数を呼び出す。
  • relkind は pg_class.h で定義された有効な定数である必要がある。

アルゴリズムまたは振る舞い

1. 引数 relkind を switch 文で評価する。 2. 対応するリレーション種別（テーブル、インデックス、ビュー等）に応じたエラー詳細メッセージを errdetail() を用いて設定する。 3. 未知の relkind が渡された場合は、elog(ERROR, ...) を発行し、システムエラーとして処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_heapam.c
  • contrib/pageinspect/rawpage.c
  • contrib/pg_freespacemap/pg_freespacemap.c
  • contrib/pg_prewarm/pg_prewarm.c
  • contrib/pg_surgery/heap_surgery.c
  • contrib/pg_visibility/pg_visibility.c
  • contrib/pgstattuple/pgstatapprox.c
  • contrib/pgstattuple/pgstatindex.c
  • contrib/pgstattuple/pgstattuple.c
  • src/backend/access/sequence/sequence.c
  • src/backend/access/table/table.c
  • **呼び出し先**: errdetail()（エラー詳細設定用）、elog()（エラー報告用）。

関連データ構造

  • pg_class（システムカタログ）
  • RELKIND_xxx 定数（src/include/catalog/pg_class.h で定義）

失敗モードと境界条件

  • relkind が switch 文で定義されていない値の場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 新しい relkind がシステムに追加された場合、本関数の switch 文に該当するケースを追加する必要がある。
