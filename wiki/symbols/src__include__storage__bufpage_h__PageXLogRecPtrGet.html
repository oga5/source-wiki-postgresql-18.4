PageXLogRecPtrGet

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/storage/bufpage.h:104

目的

PageXLogRecPtr 構造体として保持されているログシーケンス番号（LSN）の断片を、64ビットの単一整数値（XLogRecPtr）へ変換して取得します。

シグネチャまたは形状

static inline XLogRecPtr PageXLogRecPtrGet(PageXLogRecPtr val)

入力と出力

  • **入力**: PageXLogRecPtr val（xlogid と xrecoff を含む構造体）
  • **出力**: uint64 型の XLogRecPtr（ログシーケンス番号）

前提条件と不変条件

  • 入力 val は有効な PageXLogRecPtr 構造体である必要があります。
  • 戻り値は xlogid を上位32ビット、xrecoff を下位32ビットとして結合した値となります。

アルゴリズムまたは振る舞い

1. 引数 val の xlogid を uint64 にキャストし、32ビット左シフトします。 2. val の xrecoff とのビット論理和（OR）をとることで、64ビットの完全なLSNを構築して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/access/gist.h: GistPageGetNSN マクロ内で使用。
  • src/include/storage/bufpage.h: PageHeaderData の pd_lsn を取得する箇所で使用。
  • **呼び出し先**:
  • 特筆すべき呼び出し先はありません（算術演算のみを実行）。

関連データ構造

  • PageXLogRecPtr: LSNを保持するための構造体。
  • XLogRecPtr: ログシーケンス番号を表す64ビット整数型。
  • PageHeaderData: ページヘッダー構造体（pd_lsn フィールドを保持）。

失敗モードと境界条件

  • 入力値のビット構成が XLogRecPtr の範囲を超えている場合の動作は、型定義に依存します。
  • 算術演算であるため、この関数自体がエラーを発生させることはありません。

保守担当者向けメモ

  • 本関数はインライン化されており、パフォーマンスが重視されるパスで使用されます。
  • LSNの表現形式（構造体と64ビット整数の変換）に変更がある場合は、本関数と PageXLogRecPtrSet 等の関連関数を同時に更新する必要があります。
