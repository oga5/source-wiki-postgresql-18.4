gen_partprune_steps_internal

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partprune.c:990

目的

クエリの条件式（clauses）を解析し、パーティションプルーニングを実行するための PartitionPruneStep リストを再帰的に生成します。

シグネチャまたは形状

static List *gen_partprune_steps_internal(GeneratePruningStepsContext *context, List *clauses)

入力と出力

  • **入力**:
  • context: プルーニングステップ生成の状態を保持する GeneratePruningStepsContext。
  • clauses: 解析対象となる RestrictInfo または Expr のリスト。
  • **出力**:
  • 生成された PartitionPruneStep のリスト。矛盾が検出された場合は NIL を返し、context->contradictory を true に設定します。

前提条件と不変条件

  • context->contradictory は、再帰呼び出しの前後で適切に管理される必要があります。
  • デフォルトパーティションが存在する場合、context->rel->partition_qual と clauses の矛盾判定が優先されます。

アルゴリズムまたは振る舞い

1. **矛盾チェック**: デフォルトパーティションが存在し、かつ clauses がパーティション制約と矛盾する場合、即座に終了します。 2. **定数評価**: Const ノードが false または null の場合、矛盾として処理します。 3. **論理式処理**:

  • OR 句: 各引数に対して再帰的にステップを生成し、PARTPRUNE_COMBINE_UNION を用いた結合ステップを作成します。すべての引数が矛盾する場合は全体を矛盾とみなします。
  • AND 句: 引数に対して再帰的にステップを生成し、結果を統合します。いずれかの引数が矛盾する場合は全体を矛盾とみなします。

    • 4. **ステップ生成**: 最終的に、生成されたステップのリストまたは結合ステップを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: gen_partprune_steps および同関数内での再帰呼び出し。
  • **呼び出し先**: gen_prune_step_combine、predicate_refuted_by、および自身（再帰）。

関連データ構造

  • GeneratePruningStepsContext: ステップ生成の状態管理。
  • PartitionPruneStep: 生成されるプルーニング単位。
  • BoolExpr: AND/OR 句の解析に使用。

失敗モードと境界条件

  • clauses が空、またはパーティションキーと一致しない場合、ステップは生成されません。
  • OR 句の引数に一致する条件がない場合、空の source_stepids を持つダミーの結合ステップが生成されます。
  • 矛盾が検出された場合、context->contradictory が true に設定され、処理が中断されます。

保守担当者向けメモ

  • 本関数は PartitionPruneStep の構築における中心的な再帰関数です。
  • gen_partprune_steps_internal は常に単一の結合ステップをリストの末尾に追加する性質があるため、呼び出し側ではリストの llast を参照してステップを取得する実装になっています。
