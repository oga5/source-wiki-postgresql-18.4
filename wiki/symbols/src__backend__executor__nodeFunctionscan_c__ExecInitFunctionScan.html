ExecInitFunctionScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeFunctionscan.c:278

目的

FunctionScan ノードの初期化を行い、実行に必要な状態構造体（FunctionScanState）の構築、各関数のメタデータ解析、およびタプル記述子（TupleDesc）の準備を行う。

シグネチャまたは形状

FunctionScanState *ExecInitFunctionScan(FunctionScan *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の FunctionScan プランノード。
  • estate: クエリ実行状態（EState）。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの FunctionScanState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_MARK が含まれていないこと（Assert で検証）。
  • FunctionScan ノードは子プランを持たないこと（outerPlan および innerPlan が NULL であること）。
  • WITH ORDINALITY 指定の有無や関数数に応じて、simple モード（最適化パス）が適切に設定されること。

アルゴリズムまたは振る舞い

1. FunctionScanState を割り当て、ExecFunctionScan を実行関数として登録する。 2. ordinality フラグと simple モード（関数が単一かつ序数なしの場合に有効）を決定する。 3. 式評価用のコンテキスト（ExprContext）を作成する。 4. 各関数について以下の処理をループで実行する:

  • ExecInitTableFunctionResult を呼び出し、関数実行状態を初期化する。
  • tstore（結果格納用）を NULL に初期化する。
  • 関数定義（RangeTblFunction）に基づき、戻り値の型を解析して TupleDesc を構築する。
  • simple モードでない場合、ExecInitExtraTupleSlot を使用して関数ごとのタプルスロットを確保する。

    • 5. simple モードの場合、単一関数の TupleDesc をコピーしてノード全体の scan_tupdesc を作成する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:255（ExecInitNode 経由での呼び出しが示唆される）。
  • **呼び出し先**:
  • makeNode, palloc, ExecAssignExprContext, ExecInitTableFunctionResult, BuildDescFromLists, BlessTupleDesc, get_expr_result_type, CreateTupleDescCopy, CreateTemplateTupleDesc, TupleDescInitEntry, TupleDescInitEntryCollation, ExecInitExtraTupleSlot, CreateTupleDescCopy。

関連データ構造

  • FunctionScanState: ノードの実行時状態を保持。
  • FunctionScanPerFuncState: 各関数ごとの実行状態（tstore, tupdesc, func_slot 等）を保持。
  • RangeTblFunction: プランノード内の関数定義情報。
  • TupleDesc: タプルの構造定義。

失敗モードと境界条件

  • 関数がサポートされていない戻り値型（TYPEFUNC_SCALAR 等以外）を返す場合、elog(ERROR, ...) を発生させる。
  • RECORD 型の結果に対して typmod が未割り当ての場合、BlessTupleDesc で補完を試みる。

保守担当者向けメモ

  • simple フラグは、関数が1つかつ ORDINALITY がない場合にオーバーヘッドを削減するための最適化パスである。
  • ordinal カウンタは、後方スキャン時に複数の関数結果サイズが異なる場合でも正しく位置を追跡するために維持される。
  • fs->tstore は初期化時には NULL であり、実際の関数実行時に割り当てられる。
