TSVectorData

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/ts_type.h:90

目的

PostgreSQL の全文検索機能における tsvector 型の内部バイナリ表現を定義する構造体です。語彙（lexeme）とそれに関連する位置情報の配列をメモリ上に保持します。

シグネチャまたは形状

typedef struct {
    int32       vl_len_;    /* varlena header */
    int32       size;       /* 語彙の数 */
    WordEntry   entries[FLEXIBLE_ARRAY_MEMBER];
    /* 語彙データは entries[] の直後に配置される */
} TSVectorData;

入力と出力

  • 入力: PostgreSQL の Datum として渡される tsvector データを、TSVector ポインタ経由でキャストしてアクセスします。
  • 出力: TSVectorGetDatum マクロを介して、構造体ポインタから Datum 型へ変換されます。

前提条件と不変条件

  • vl_len_ は varlena ヘッダーであり、直接操作してはなりません。
  • entries は可変長配列（FLEXIBLE_ARRAY_MEMBER）として定義されており、構造体のサイズ計算には CALCDATASIZE や COMPUTESIZE マクロを使用する必要があります。
  • メモリレイアウトは、ヘッダー、WordEntry 配列、語彙文字列、位置情報ベクトルの順に連続して配置されることが前提です。

アルゴリズムまたは振る舞い

  • tsvector のバイナリデータ構造を規定します。
  • ARRPTR() や STRPTR() などのマクロを使用して、構造体内の特定セクション（WordEntry 配列や文字列データ）へ直接ポインタ演算を行いアクセスします。

呼び出し元と呼び出し先

  • 呼び出し元: TSVectorGetDatum マクロ（src/include/tsearch/ts_type.h:130）が本構造体へのポインタを引数として受け取ります。
  • 呼び出し先: 本構造体自体はデータ保持用であり、具体的な操作は src/backend/tsearch/ 配下の関数群によって行われます。

関連データ構造

  • TSVector: TSVectorData * の型定義。
  • WordEntry: 各語彙のメタデータを管理する構造体。
  • WordEntryPos: 位置情報を保持する構造体。

失敗モードと境界条件

  • 可変長データ構造であるため、手動でサイズを計算する際の誤りはメモリ破壊や不正な読み取りを引き起こします。必ず提供されているマクロ（DATAHDRSIZE 等）を使用してください。

保守担当者向けメモ

  • 本構造体はディスク保存やネットワーク転送を考慮した厳密なバイナリレイアウトを持っています。フィールドの追加や変更を行う際は、アライメントや varlena ヘッダーの整合性に十分注意してください。
