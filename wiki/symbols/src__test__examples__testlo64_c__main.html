目的

PostgreSQL の 64 ビットオフセット API を使用したラージオブジェクト操作（インポート、読み取り、書き込み、切り詰め、エクスポート）を検証するサンプルプログラムのエントリポイントです。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**:
  • argc: 引数の数（期待値は 5）。
  • argv: 以下の順序で構成される引数配列。

    • 1. database_name: 接続先データベース名。 2. in_filename: インポート対象のローカルファイルパス。 3. out_filename: 最初のエクスポート先ファイルパス。 4. out_filename2: 切り詰め後のエクスポート先ファイルパス。

  • **出力**:
  • 正常終了時は 0 を返します。
  • 引数が不足している場合や接続エラー時は 1 を返し、標準エラー出力にエラーメッセージを表示します。

前提条件と不変条件

  • 実行には PostgreSQL データベースへの接続が必要です。
  • 4GB を超えるオフセット（4294967000U 等）を扱うため、64 ビット API をサポートする環境が必要です。
  • トランザクション内での操作が前提となっており、BEGIN で開始し END でコミットされます。

アルゴリズムまたは振る舞い

1. **接続**: PQsetdb でデータベースに接続し、PQstatus で接続を確認します。 2. **初期化**: セキュリティのため search_path を空に設定します。 3. **トランザクション開始**: BEGIN を発行します。 4. **ラージオブジェクト操作**:

  • lo_import: ファイルをラージオブジェクトとしてインポートします。
  • pickout: 4GB を超えるオフセットからデータを読み取ります。
  • overwrite: 特定範囲を上書きします。
  • lo_export: 変更後のオブジェクトをファイルへエクスポートします。
  • my_truncate: オブジェクトを指定サイズに切り詰めます。
  • lo_export: 切り詰め後のオブジェクトを別のファイルへエクスポートします。

    • 5. **終了**: END でコミットし、PQfinish で接続を閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（エントリポイントのため）。
  • **呼び出し先**:
  • PQsetdb, PQstatus, PQexec, PQclear, PQerrorMessage, PQfinish (libpq)
  • lo_import, lo_export (libpq-fs)
  • pickout, overwrite, my_truncate, exit_nicely (同一ファイル内の関数)
  • 注意: ツールが提示した contrib/amcheck 等の参照は、本シンボルとは無関係な「main」という単語の検索結果であり、本関数の呼び出し元ではありません。

関連データ構造

  • PGconn: データベース接続ハンドル。
  • PGresult: クエリ実行結果。
  • Oid: ラージオブジェクトの識別子。

失敗モードと境界条件

  • 引数が 5 つ以外の場合、使用法を表示して終了します。
  • データベース接続失敗時、エラーメッセージを表示して exit_nicely を呼び出します。
  • lo_import や lo_export が失敗した場合、エラーメッセージを表示します。
  • 4GB を超えるオフセット指定が環境の制限を超える場合、予期せぬ動作が発生する可能性があります。

保守担当者向けメモ

  • 本関数はサンプルコードであり、エラーハンドリングは exit_nicely に依存しています。
  • importFile や exportFile といった自作関数がコメントアウトされており、標準の lo_import / lo_export が使用されている点に注意してください。
