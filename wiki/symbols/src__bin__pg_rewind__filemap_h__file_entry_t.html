file_entry_t

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_rewind/filemap.h:49

目的

pg_rewind において、同期対象のファイルパスごとにソースおよびターゲットディレクトリ上の状態を保持し、最終的な同期アクションを決定・管理するためのデータ構造です。

シグネチャまたは形状

typedef struct file_entry_t
{
    uint32      status;             /* ハッシュテーブル用ステータス */
    const char *path;               /* ファイルパス */
    bool        isrelfile;          /* リレーションデータファイルか否か */
    bool        target_exists;      /* ターゲット側の存在有無 */
    file_type_t target_type;        /* ターゲット側のファイル種別 */
    size_t      target_size;        /* ターゲット側のファイルサイズ */
    char       *target_link_target; /* シンボリックリンク先 */
    datapagemap_t target_pages_to_overwrite; /* ターゲット側で上書きが必要なページマップ */
    bool        source_exists;      /* ソース側の存在有無 */
    file_type_t source_type;        /* ソース側のファイル種別 */
    size_t      source_size;        /* ソース側のファイルサイズ */
    char       *source_link_target; /* シンボリックリンク先 */
    file_action_t action;           /* 実行すべき同期アクション */
} file_entry_t;

入力と出力

本構造体は、process_source_file および process_target_file によってファイル属性が入力され、decide_file_actions によって action フィールドが設定されます。

前提条件と不変条件

  • isrelfile が真の場合、target_pages_to_overwrite を介してブロックレベルの差分管理が行われます。
  • action は file_action_t で定義された列挙値のいずれかであり、同期プロセスにおいて安全な順序で処理される必要があります。

アルゴリズムまたは振る舞い

1. **収集**: ファイルシステム走査時に、パスをキーとしてハッシュテーブルに格納されます。 2. **差分記録**: process_target_wal_block_change を通じて、ターゲット側で変更されたページが target_pages_to_overwrite に記録されます。 3. **決定**: 全ファイルの状態比較後、decide_file_actions が各エントリの action を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/filemap.c 内のハッシュテーブル操作関数（insert_filehash_entry, lookup_filehash_entry 等）が本構造体を生成・参照します。また、src/bin/pg_rewind/file_ops.c の remove_target および create_target が引数として受け取ります。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • file_action_t: 実行する操作の定義。
  • file_type_t: ファイル種別の定義。
  • datapagemap_t: リレーションファイルのページ変更管理に使用。

失敗モードと境界条件

  • target_link_target および source_link_target は、ファイル種別がシンボリックリンクである場合にのみ有効な値が割り当てられます。
  • target_size は通常ファイルの場合にのみ意味を持ちます。

保守担当者向けメモ

  • filemap.c で simplehash.h を用いたハッシュテーブルの要素（SH_ELEMENT_TYPE）として定義されているため、構造体の変更時はハッシュテーブルの整合性に注意してください。
  • リレーションファイルの同期において、target_pages_to_overwrite の内容は同期の効率を左右する重要な要素です。
