macaddr_sortsupport_state

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/mac.c:37

目的

macaddr 型のソート処理において、省略キー（abbreviated key）を用いた最適化の有効性を判定し、統計情報を保持するための状態管理構造体です。

シグネチャまたは形状

typedef struct {
    int64               input_count;    /* 処理済みの非NULL値の数 */
    bool                estimating;     /* カーディナリティ推定中か否か */
    hyperLogLogState    abbr_card;      /* カーディナリティ推定用状態 */
} macaddr_sortsupport_state;

入力と出力

  • 本構造体自体はデータ保持用であり、直接的な入出力関数はありません。
  • macaddr_sortsupport 関数内で palloc を介してメモリ確保され、ソート処理のコンテキスト（SortSupport）に格納されます。

前提条件と不変条件

  • input_count は、ソート対象のデータが処理されるたびにインクリメントされます。
  • abbr_card は、HyperLogLog アルゴリズムを用いて、省略キーのカーディナリティ（一意な値の数）を推定するために使用されます。

アルゴリズムまたは振る舞い

  • ソート最適化の過程で、省略キーへの変換が有効かどうかを判断するために使用されます。
  • macaddr_abbrev_convert 関数内で、この構造体を通じて現在のデータ分布の統計が更新されます。
  • macaddr_abbrev_abort 関数が、この構造体の情報を参照して、省略キーによる比較を継続すべきか、あるいは通常の比較ロジックへフォールバックすべきかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/mac.c 内の macaddr_sortsupport（初期化）、macaddr_abbrev_convert（更新）、macaddr_abbrev_abort（判定）から参照されます。
  • **呼び出し先**: HyperLogLog 関連のライブラリ関数が abbr_card フィールドを操作します。

関連データ構造

  • SortSupport: ソート最適化フレームワークのメイン構造体。本構造体は ssup_extra フィールドに格納されます。
  • hyperLogLogState: カーディナリティ推定のための外部データ構造。

失敗モードと境界条件

  • メモリ不足: palloc によるメモリ確保時に失敗する可能性があります。
  • 推定の精度: HyperLogLog の特性上、カーディナリティの推定値には統計的な誤差が含まれます。

保守担当者向けメモ

  • 本構造体は macaddr 型のソートパフォーマンスを最適化するための補助的なものです。
  • macaddr_abbrev_convert 内で DatumBigEndianToNative が使用されているため、プラットフォームのエンディアンに関わらず、省略キーとしての整数比較が論理的な順序を維持するように設計されています。
