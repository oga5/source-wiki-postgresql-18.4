xsave_available

定義

  • 種別: function
  • 定義箇所: src/port/pg_popcount_avx512.c:39

目的

実行環境のCPUが XSAVE 命令セット（OSによる状態保存機能）をサポートしているかを判定する。

シグネチャまたは形状

static inline bool xsave_available(void)

入力と出力

  • 入力: なし
  • 出力: bool 型。true の場合、OSが XSAVE をサポートしていることを示す。

前提条件と不変条件

  • HAVE__GET_CPUID または HAVE__CPUID マクロが定義されている環境であること。
  • CPUID命令が実行可能であること。

アルゴリズムまたは振る舞い

1. CPUID 命令（EAX=1）を呼び出し、プロセッサ情報を取得する。 2. 取得した結果の3番目のレジスタ（ECX）の第27ビット（osxsave フラグ）を確認する。 3. 当該ビットがセットされていれば true を返し、そうでなければ false を返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/port/pg_popcount_avx512.c 内の各AVX-512関連関数（pg_popcount_avx512_available 等）から、機能判定の一部として呼び出される。
  • 呼び出し先:
  • __get_cpuid または __cpuid（コンパイル環境に依存）。

関連データ構造

  • unsigned int exx[4]: CPUIDの結果を格納する一時的な配列。

失敗モードと境界条件

  • CPUID 命令が利用できない環境では、コンパイル時に #error が発生し、ビルドが停止する。

保守担当者向けメモ

  • 本関数はAVX-512命令セットを利用する前のガード条件として機能する。
  • osxsave ビットの確認は、AVX命令を使用する際にOSがYMM/ZMMレジスタの状態を正しく保存・復元できるかを保証するために不可欠である。
