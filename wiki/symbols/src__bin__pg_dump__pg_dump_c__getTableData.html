getTableData

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dump.c:2965

目的

提供されたテーブル情報の配列から、ダンプ対象としてマークされているテーブルのデータ抽出処理を順次実行する。

シグネチャまたは形状

static void getTableData(DumpOptions *dopt, TableInfo *tblinfo, int numTables, char relkind)

入力と出力

  • **入力**:
  • dopt: ダンプ設定を保持する DumpOptions 構造体へのポインタ。
  • tblinfo: ダンプ対象テーブル情報の配列。
  • numTables: tblinfo 配列内の要素数。
  • relkind: 処理対象を絞り込むためのリレーション種別（0の場合は全種別を対象とする）。
  • **出力**: なし（戻り値 void）。

前提条件と不変条件

  • tblinfo は numTables 分の有効な要素を持つ配列であること。
  • 各 TableInfo 要素内の dobj.dump フラグが適切に設定されていること。

アルゴリズムまたは振る舞い

1. numTables 回のループ処理を行う。 2. 各テーブルについて以下の条件を判定する:

  • dobj.dump に DUMP_COMPONENT_DATA フラグが含まれていること。
  • relkind が 0 であるか、またはテーブルの relkind が指定された relkind と一致すること。

    • 3. 上記条件を満たす場合、makeTableDataInfo(dopt, &(tblinfo[i])) を呼び出し、データダンプのための情報を構築する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c:1104 (全テーブルのデータ処理)
  • src/bin/pg_dump/pg_dump.c:1111 (シーケンスのデータ処理)
  • **呼び出し先**:
  • makeTableDataInfo

関連データ構造

  • DumpOptions: ダンプの動作設定を保持。
  • TableInfo: テーブルのメタデータおよびダンプ状態を保持。

失敗モードと境界条件

  • numTables が 0 の場合、ループは実行されず何も行われない。
  • relkind に一致するテーブルが存在しない場合、何も処理されない。
  • makeTableDataInfo 内部でのエラー発生時の挙動については本関数の範囲外。

保守担当者向けメモ

  • 本関数はデータダンプのトリガーであり、実際のデータ抽出ロジックは makeTableDataInfo に委譲されている。
  • relkind によるフィルタリングは、通常のテーブルとシーケンスなどの特殊なリレーションを分けて処理するために使用されている。
