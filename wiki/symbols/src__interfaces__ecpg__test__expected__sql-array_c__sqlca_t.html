sqlca_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-array.c:43

目的

ECPG（Embedded SQL for C）において、SQL文の実行結果やエラー情報を保持・伝達するための通信領域（SQL Communication Area）を定義する。

シグネチャまたは形状

struct sqlca_t {
    char        sqlcaid[8];
    long        sqlabc;
    long        sqlcode;
    struct {
        int     sqlerrml;
        char    sqlerrmc[SQLERRMC_LEN];
    }           sqlerrm;
    char        sqlerrp[8];
    long        sqlerrd[6];
    char        sqlwarn[8];
    char        sqlstate[5];
};

入力と出力

  • **入力**: SQL実行エンジンが各操作（INSERT, UPDATE, DELETE, SELECT 等）の完了後に、実行結果や警告、エラー状態を本構造体の各フィールドに書き込む。
  • **出力**: アプリケーション側が本構造体のフィールド（sqlcode, sqlstate 等）を読み取ることで、SQL実行の成否や処理件数を確認する。

前提条件と不変条件

  • sqlerrd[2] は、INSERT, UPDATE, DELETE 文の実行後に処理された行数を保持する。
  • sqlwarn[0] は、他の警告フラグがセットされた場合に 'W' に設定される。
  • sqlwarn[1] は、ホスト変数への格納時に文字列の切り詰めが発生した場合に 'W' に設定される。

アルゴリズムまたは振る舞い

SQL実行時にECPGライブラリが内部的に更新する。アプリケーションは各SQL文の実行直後に本構造体の内容を検査することで、エラーハンドリングを行う。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPGget_sqlca() を介して、src/interfaces/ecpg/ecpglib/connect.c, src/interfaces/ecpg/ecpglib/data.c, src/interfaces/ecpg/ecpglib/descriptor.c 等のECPGライブラリ関数から参照される。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはない。

関連データ構造

  • sqlda_t（動的SQL用記述子）
  • sqlvar_t（sqlda_t 内の各列定義）

失敗モードと境界条件

  • sqlerrmc は SQLERRMC_LEN で定義された長さを超えるエラーメッセージを格納できない。
  • sqlerrd 配列の各要素は、特定のSQL操作（OID取得や処理行数など）に対応しており、操作の種類によっては値が更新されない（空のままとなる）。

保守担当者向けメモ

  • 本構造体はECPGの標準的な通信領域定義である。変更を行う際は、ecpglib および関連する互換ライブラリ（compatlib）との整合性に注意すること。
  • sql-array.c はテスト用の期待値ファイルであり、本構造体の定義自体は sqlca.h 等のヘッダーファイルから供給されるのが一般的である。
