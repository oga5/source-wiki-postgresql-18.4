brin_bloom_union

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_bloom.c:664

目的

2つのブルームフィルタをビット単位のOR演算で結合し、BRINインデックスの要約情報を統合する。

シグネチャまたは形状

Datum brin_bloom_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して BrinValues 構造体（col_a, col_b）を受け取る。これらは結合対象のブルームフィルタを保持している。
  • **出力**: PG_RETURN_VOID()。結果は col_a に格納される。

前提条件と不変条件

  • col_a と col_b の属性番号（bv_attno）が一致していること。
  • 両方の bv_allnulls が false であること。
  • 結合される2つのブルームフィルタのビット数（nbits）およびハッシュ関数数（nhashes）が一致していること。
  • nbits が 0 より大きく、かつ 8 の倍数であること。

アルゴリズムまたは振る舞い

1. PG_DETOAST_DATUM を使用して、入力された BrinValues から BloomFilter 構造体を取り出す。 2. フィルタのパラメータ（nbits, nhashes）が一致していることをアサーションで確認する。 3. filter_a のビットマップデータに対して、filter_b のビットマップデータをビット単位のOR演算でマージする。 4. pg_popcount を用いて、更新後のフィルタ内のセットされたビット数を再計算する。 5. デトースト（解凍）されたフィルタが一時的なメモリ領域にある場合、col_a の値を更新し、不要になったメモリを pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は明示されていませんが、BRINアクセスメソッドのインデックス要約処理の一部として呼び出されます。
  • **呼び出し先**: PG_GETARG_POINTER, PG_DETOAST_DATUM, pg_popcount, pfree, DatumGetPointer, PointerGetDatum, PG_RETURN_VOID。

関連データ構造

  • BrinValues: BRINインデックスの要約値を保持する構造体。
  • BloomFilter: ブルームフィルタのビットマップとメタデータを保持する構造体。

失敗モードと境界条件

  • 入力されたフィルタのパラメータが一致しない場合、アサーションにより停止する。
  • nbits が 8 の倍数でない場合、アサーションにより停止する。

保守担当者向けメモ

  • 本関数は col_a を更新して結果を返すため、col_a が保持するメモリ管理（デトースト後の解放）に注意すること。
  • pg_popcount によるビット数更新は、フィルタの整合性を保つために必須である。
