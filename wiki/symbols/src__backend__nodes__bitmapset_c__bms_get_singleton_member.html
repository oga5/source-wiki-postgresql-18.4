bms_get_singleton_member

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/bitmapset.c:714

目的

Bitmapset が単一の要素のみを含むかどうかを判定し、その要素を取得します。

シグネチャまたは形状

bool bms_get_singleton_member(const Bitmapset *a, int *member);

入力と出力

  • **入力**:
  • a: 検査対象の Bitmapset へのポインタ。
  • member: 単一要素が格納される整数へのポインタ。
  • **出力**:
  • 戻り値: 集合が単一の要素のみを持つ場合は true、空集合または複数の要素を持つ場合は false。
  • member: 戻り値が true の場合、集合内の唯一の要素が書き込まれます。

前提条件と不変条件

  • a が NULL の場合、空集合として扱われ false を返します。
  • a が非 NULL の場合、bms_is_valid_set(a) が真であることを前提とします。
  • 非 NULL の集合は、少なくとも1つの要素を持つことが期待されます。

アルゴリズムまたは振る舞い

1. a が NULL ならば false を返します。 2. 集合内の全ワードを走査します。 3. ビットがセットされているワードが見つかった場合：

  • すでに別の要素が見つかっているか、そのワード内に複数のビットが立っている（HAS_MULTIPLE_ONES）場合は、単一要素ではないと判断し false を返します。
  • そうでなければ、その位置を計算して result に保持します。

    • 4. 全ワードの走査終了後、result が有効（0以上）であれば *member に値を格納し true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/allpaths.c
  • src/backend/optimizer/path/clausesel.c
  • src/backend/optimizer/path/equivclass.c
  • src/backend/optimizer/plan/analyzejoins.c
  • src/backend/optimizer/plan/initsplan.c
  • src/backend/optimizer/util/placeholder.c
  • **呼び出し先**:
  • bms_is_valid_set (マクロまたは関数)
  • HAS_MULTIPLE_ONES (マクロ)
  • bmw_rightmost_one_pos (マクロまたは関数)

関連データ構造

  • Bitmapset: 集合を表現する構造体。nwords と words 配列を保持。

失敗モードと境界条件

  • a が NULL の場合、安全に false を返します。
  • 集合が空（NULL ではないが要素がない状態）の場合、Assert(result >= 0) によりアサーションエラーが発生する可能性があります。
  • 複数の要素を持つ集合に対しては、即座に false を返します。

保守担当者向けメモ

  • 本関数は、クエリプランナーにおいて特定の関係（relid）が単一であるかを判定する際によく利用されます。
  • HAS_MULTIPLE_ONES および bmw_rightmost_one_pos は port/pg_bitutils.h に依存する低レベルなビット操作です。
