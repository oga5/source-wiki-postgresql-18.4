buf_finalize

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xid8funcs.c:247

目的

StringInfo バッファに構築されたスナップショットデータを pg_snapshot 構造体として確定させ、メモリ管理を完了させる。

シグネチャまたは形状

pg_snapshot *buf_finalize(StringInfo buf)

入力と出力

  • **入力**: StringInfo buf（構築済みのスナップショットデータを含むバッファ）
  • **出力**: pg_snapshot *（バッファから変換されたスナップショット構造体へのポインタ）

前提条件と不変条件

  • 入力された buf は pg_snapshot 構造体として解釈可能なデータを含んでいる必要がある。
  • 本関数呼び出し後、元の buf は解放されるため、呼び出し元は buf を参照してはならない。

アルゴリズムまたは振る舞い

1. buf->data を pg_snapshot * 型にキャストして取得する。 2. SET_VARSIZE マクロを用いて、バッファの長さ (buf->len) をスナップショット構造体のサイズとして設定する。 3. buf->data を NULL に設定し、バッファの所有権を無効化する。 4. pfree(buf) を呼び出し、バッファヘッダを解放する。 5. 確定した pg_snapshot ポインタを返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/xid8funcs.c 内の parse_snapshot 等（直接の呼び出し箇所として src/backend/utils/adt/xid8funcs.c:318 が確認されている）。
  • **呼び出し先**: pfree（PostgreSQL メモリ管理関数）。

関連データ構造

  • pg_snapshot: 64ビットトランザクションIDを保持するスナップショット構造体。
  • StringInfo: PostgreSQL の可変長文字列バッファ構造体。

失敗モードと境界条件

  • buf が NULL の場合、あるいは buf->data が不正なメモリ領域を指している場合はセグメンテーションフォールトが発生する可能性がある。

保守担当者向けメモ

  • 本関数は buf の所有権を奪い、内部のデータを pg_snapshot として再利用する。メモリリークを防ぐため、本関数を呼び出した後は buf 自体へのアクセスを行わないこと。
