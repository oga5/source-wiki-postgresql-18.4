ExecBitmapHeapInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapHeapscan.c:477

目的

並列ビットマップヒープスキャン実行時の共有メモリ（DSM）領域を初期化し、ワーカープロセス間で共有される状態構造体および統計情報を設定する。

シグネチャまたは形状

void ExecBitmapHeapInitializeDSM(BitmapHeapScanState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: 初期化対象の BitmapHeapScanState 構造体。
  • pcxt: 並列実行コンテキスト（ParallelContext）。
  • **出力**: なし（副作用として node のメンバおよび共有メモリ領域を更新）。

前提条件と不変条件

  • node->ss.ps.state->es_query_dsa が存在すること。
  • pcxt->toc が有効な共有メモリテーブルであること。
  • 並列実行時、リーダープロセスによって呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. **DSAチェック**: es_query_dsa が NULL の場合は何もせず終了する。 2. **メモリ確保**: ParallelBitmapHeapState および（計測が有効な場合）SharedBitmapHeapInstrumentation 用の領域を shm_toc_allocate で確保する。 3. **状態初期化**:

  • ParallelBitmapHeapState の tbmiterator を 0 に初期化。
  • SpinLock を初期化し、状態を BM_INITIAL に設定。
  • ConditionVariable を初期化。

    • 4. **統計情報初期化**: 計測が有効な場合、SharedBitmapHeapInstrumentation をゼロクリアし、ワーカー数を設定。 5. **登録**: shm_toc_insert を使用して、プランノード ID をキーに共有メモリへ状態を登録し、node にポインタを保持させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:507（ExecParallelInitializeDSM 経由での呼び出しが確認されている）。
  • **呼び出し先**: SpinLockInit, ConditionVariableInit, shm_toc_allocate, shm_toc_insert, add_size, mul_size, memset。

関連データ構造

  • BitmapHeapScanState: スキャンノードの実行状態。
  • ParallelBitmapHeapState: 並列スキャン用の共有状態構造体。
  • SharedBitmapHeapInstrumentation: 並列実行時の統計情報共有用構造体。
  • ParallelContext: 並列処理のコンテキスト情報。

失敗モードと境界条件

  • dsa_area が NULL の場合、初期化はスキップされる。
  • pcxt->nworkers が 0 の場合、統計情報領域の確保は行われない。

保守担当者向けメモ

  • 本関数は並列実行のセットアップフェーズで実行される。共有メモリのレイアウト変更を行う際は、shm_toc_allocate のサイズ計算と ptr のオフセット計算の整合性に注意すること。
  • BM_INITIAL 状態から開始されるため、後続のワーカープロセスが BitmapShouldInitializeSharedState を通じて正しく同期できることを前提としている。
