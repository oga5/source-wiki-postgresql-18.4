pgstat_checkpointer_snapshot_cb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_checkpointer.c:111

目的

チェックポイント統計のスナップショット取得時に、統計リセット以降の差分を正しく計算するために、共有メモリ上の累積値からリセットオフセットを減算する。

シグネチャまたは形状

void pgstat_checkpointer_snapshot_cb(void)

入力と出力

  • **入力**: 共有メモリ上の pgStatLocal.shmem->checkpointer（統計値およびリセットオフセット）。
  • **出力**: pgStatLocal.snapshot.checkpointer に格納された、リセットオフセット補正済みの統計データ。

前提条件と不変条件

  • 呼び出し前に pgStatLocal が初期化されていること。
  • 統計の更新には changecount によるアトミックな同期が適用されていること。

アルゴリズムまたは振る舞い

1. pgstat_copy_changecounted_stats を使用し、共有メモリからローカルスナップショットへ統計値をコピーする。 2. stats_shmem->lock を LW_SHARED モードで取得し、共有メモリ上の reset_offset をローカル変数へコピーする。 3. マクロ CHECKPOINTER_COMP を用いて、以下の各フィールドから対応する reset_offset の値を減算する。

  • num_timed, num_requested, num_performed
  • restartpoints_timed, restartpoints_requested, restartpoints_performed
  • write_time, sync_time, buffers_written, slru_written

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c:429（スナップショット取得処理の一部としてコールバック登録されている）。
  • **呼び出し先**:
  • pgstat_copy_changecounted_stats
  • LWLockAcquire / LWLockRelease
  • memcpy

関連データ構造

  • PgStatShared_Checkpointer: 共有メモリ上のチェックポイント統計構造体。
  • PgStat_CheckpointerStats: 統計値の各フィールドを保持する構造体。

失敗モードと境界条件

  • LWLock の取得に失敗した場合、プロセスは待機またはエラーとなる可能性がある。
  • 統計がリセットされていない場合、reset_offset はゼロであり、補正は行われない。

保守担当者向けメモ

  • 本関数は統計の「保存」と「表示」を分離する設計の一部であり、統計リセット機能（pg_stat_reset 等）と密接に関連している。
  • フィールドを追加した際は、必ず CHECKPOINTER_COMP マクロのリストにも追加すること。
