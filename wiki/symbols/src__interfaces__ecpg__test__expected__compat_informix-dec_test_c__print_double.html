print_double

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/compat_informix-dec_test.c:36

目的

プラットフォーム間で浮動小数点数の指数表記（特に指数部の桁数）を統一し、テスト出力の一貫性を保証する。

シグネチャまたは形状

static void print_double(double x)

入力と出力

  • 入力: double x （出力対象の浮動小数点数）
  • 出力: なし（標準出力 stdout へ直接書き出し）

前提条件と不変条件

  • WIN32 環境下では、sprintf による指数表記が 3 桁になる場合があることを前提としている。
  • 非 WIN32 環境では、標準の %g 書式指定子による出力を行う。

アルゴリズムまたは振る舞い

1. WIN32 定義時:

  • sprintf を用いて double 値を文字列に変換する。
  • 文字列の末尾付近を走査し、指数部が 3 桁（例: e+0xx）であるかを確認する。
  • 3 桁の指数部を 2 桁（例: e+xx）に詰め、文字列を修正する。
  • 修正後の文字列を printf で出力する。

    • 2. それ以外:

  • printf("%g", x) を実行し、標準のフォーマットで出力する。

呼び出し元と呼び出し先

  • 呼び出し元:

  • 呼び出し先:

失敗モードと境界条件

  • vallen が 6 未満の場合、指数部の修正ロジックはスキップされる。
  • sprintf のバッファサイズ（128 バイト）を超えるような極端な数値表現は考慮されていない。

保守担当者向けメモ

  • 本関数は Windows とそれ以外のプラットフォーム間でのテスト結果の差異（回帰テストの不一致）を吸収するためのハックである。
  • printf_hack.h を介して複数のテストファイルから参照されているため、修正時は影響範囲に注意すること。
