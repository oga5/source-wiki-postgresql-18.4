injection_init_shmem

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_points.c:180

目的

インジェクションポイント機能で使用する共有メモリ領域（InjectionPointSharedState）を初期化または取得します。

シグネチャまたは形状

static void injection_init_shmem(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 inj_state を初期化）

前提条件と不変条件

  • inj_state が既に初期化されている場合、関数は何もせず即座に復帰します。
  • 共有メモリの初期化には injection_point_init_state コールバックが使用されます。

アルゴリズムまたは振る舞い

1. グローバル変数 inj_state が NULL でないか確認し、初期化済みであれば終了します。 2. GetNamedDSMSegment を呼び出し、"injection_points" という名前の動的共有メモリ（DSM）セグメントを取得または作成します。 3. セグメントのサイズとして InjectionPointSharedState 構造体のサイズを指定します。 4. セグメントの初期化が必要な場合、injection_point_init_state を実行します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/test/modules/injection_points/injection_points.c 内の各インジェクションポイント関連関数（injection_points_attach 等の実行時）から、共有メモリへのアクセス前に呼び出されます。
  • 呼び出し先:
  • GetNamedDSMSegment
  • injection_point_init_state (初期化時のみ)

関連データ構造

  • InjectionPointSharedState: 共有メモリ上で管理されるインジェクションポイントの状態構造体。

失敗モードと境界条件

  • GetNamedDSMSegment が失敗した場合、PostgreSQLの標準的なエラーハンドリングによりプロセスが停止する可能性があります。

保守担当者向けメモ

  • 本関数は、インジェクションポイント機能が共有メモリを必要とする際に、遅延初期化を行うための重要なゲートキーパーです。
  • inj_state はモジュール全体で共有されるため、マルチプロセス環境下での競合を避ける設計になっています。
