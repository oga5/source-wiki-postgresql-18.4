GetTokenUser

定義

  • 種別: function
  • 定義箇所: src/common/exec.c:660

目的

指定されたアクセストークンからユーザー情報を取得し、呼び出し元が利用可能なメモリ領域に格納する。

シグネチャまたは形状

static BOOL GetTokenUser(HANDLE hToken, PTOKEN_USER *ppTokenUser)

入力と出力

  • **入力**:
  • hToken: 情報を取得する対象のアクセストークンハンドル。
  • ppTokenUser: 取得した TOKEN_USER 構造体を格納するためのポインタのポインタ。
  • **出力**:
  • 成功時は TRUE を返し、*ppTokenUser に割り当てられたメモリへのポインタを格納する。
  • 失敗時は FALSE を返し、*ppTokenUser は NULL に設定される。

前提条件と不変条件

  • *ppTokenUser は呼び出し前に NULL に初期化される。
  • 成功時に割り当てられたメモリは、呼び出し元が LocalFree() を使用して解放する責任を負う。

アルゴリズムまたは振る舞い

1. GetTokenInformation を呼び出し、必要なバッファサイズ dwLength を取得する。 2. ERROR_INSUFFICIENT_BUFFER が発生した場合、LocalAlloc を用いて必要なサイズのメモリを確保する。メモリ確保に失敗した場合はエラーログを出力し FALSE を返す。 3. 確保したバッファに対して再度 GetTokenInformation を呼び出し、ユーザー情報を取得する。 4. 2回目の取得に失敗した場合は、確保したメモリを LocalFree で解放し、エラーログを出力して FALSE を返す。 5. 正常終了時は TRUE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/exec.c 内のコード（行576など）から参照されている。
  • **呼び出し先**: Windows API の GetTokenInformation、GetLastError、LocalAlloc、LocalFree、および内部の log_error マクロ。

関連データ構造

  • TOKEN_USER: Windows API で定義される、トークンに関連付けられたユーザー情報を保持する構造体。
  • HANDLE: Windows のトークンハンドル。

失敗モードと境界条件

  • LocalAlloc によるメモリ割り当て失敗時: ERRCODE_OUT_OF_MEMORY を記録。
  • GetTokenInformation の呼び出し失敗時: ERRCODE_SYSTEM_ERROR を記録し、エラーコードをログに出力。
  • 呼び出し元がメモリ解放を怠った場合、メモリリークが発生する。

保守担当者向けメモ

  • 本関数は Windows 環境専用の処理である。
  • log_error マクロを使用しているため、フロントエンドとバックエンドの両方でエラー報告の一貫性が保たれる。
