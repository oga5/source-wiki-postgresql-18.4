btree_xlog_dedup

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtxlog.c:463

目的

B-tree インデックスページに対する重複排除（deduplication）操作の WAL リプレイを実行し、ページの状態を再構築します。

シグネチャまたは形状

static void btree_xlog_dedup(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象の WAL レコード）。
  • **出力**: なし（副作用としてバッファページを更新）。

前提条件と不変条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返す場合のみ、ページ更新処理が実行されます。
  • 処理終了後、ページ内の BTP_HAS_GARBAGE フラグが適切にクリアされます。
  • 処理されたインターバル数は、WAL レコード内の nintervals と一致する必要があります。

アルゴリズムまたは振る舞い

1. XLogReadBufferForRedo を呼び出し、対象ページを読み込みます。 2. 一時的なページ（newpage）を作成し、必要に応じてハイキー（High Key）をコピーします。 3. WAL レコードから取得した BTDedupInterval 情報に基づき、既存のタプルを走査します。 4. _bt_dedup_start_pending および _bt_dedup_save_htid を使用して、重複排除後のタプルを newpage 上に再構築します。 5. 最後に _bt_dedup_finish_pending を呼び出し、残りのタプルを確定させます。 6. 元のページに BTP_HAS_GARBAGE フラグがある場合、これをクリアします。 7. PageRestoreTempPage で一時ページの内容を元のページに反映し、LSN を更新してバッファをダーティとしてマークします。

呼び出し元と呼び出し先

  • **呼び出し元**: btree_redo（src/backend/access/nbtree/nbtxlog.c 内のディスパッチャ）。
  • **呼び出し先**:
  • XLogRecGetData, XLogRecGetBlockData
  • PageGetTempPageCopySpecial, PageRestoreTempPage
  • _bt_dedup_start_pending, _bt_dedup_save_htid, _bt_dedup_finish_pending
  • PageSetLSN, MarkBufferDirty, UnlockReleaseBuffer

関連データ構造

  • xl_btree_dedup: WAL レコードの構造体。
  • BTDedupState: 重複排除処理中の状態管理用構造体。
  • BTDedupInterval: 重複排除の範囲を示す構造体。
  • BTPageOpaque: B-tree ページヘッダーの不透明部分。

失敗モードと境界条件

  • PageAddItem が失敗した場合、ERROR を発生させます。
  • _bt_dedup_save_htid が失敗した場合、ERROR を発生させます。
  • 処理後のインターバル数や内容が WAL レコードと一致しない場合、Assert により停止します。

保守担当者向けメモ

  • 本関数は物理的なページ再構築を行うため、nbtdedup.c で定義される論理的な重複排除ロジックと密接に関連しています。
  • リカバリ時は並行更新が発生しないため、ロックはページ単位で簡略化されています。
