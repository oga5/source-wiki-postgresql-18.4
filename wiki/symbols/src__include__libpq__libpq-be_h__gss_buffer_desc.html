gss_buffer_desc

定義

  • 種別: struct
  • 定義箇所: src/include/libpq/libpq-be.h:44

目的

GSSAPI（Generic Security Services Application Program Interface）認証および通信において、トークンデータとその長さを保持するための汎用的なバッファ構造体です。

シグネチャまたは形状

typedef struct
{
    void       *value;
    int         length;
} gss_buffer_desc;

入力と出力

  • value: データ本体を指すポインタ。
  • length: value が指すデータのバイト長。

前提条件と不変条件

  • GSSAPI ライブラリの仕様に準拠したデータ構造であり、認証トークンの受け渡しに使用されます。
  • value が指すメモリ領域は、呼び出し元のコンテキストにおいて適切に管理されている必要があります。

アルゴリズムまたは振る舞い

GSSAPI の関数呼び出しにおいて、入力トークン（input）や出力トークン（outbuf）を保持するコンテナとして機能します。認証プロセス中に、トークンの送受信や暗号化・復号化の引数として参照されます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • src/backend/libpq/auth.c
  • src/backend/libpq/be-gssapi-common.c
  • src/backend/libpq/be-secure-gssapi.c
  • src/interfaces/libpq/fe-auth.c
  • src/interfaces/libpq/fe-gssapi-common.c
  • **呼び出し先**:
  • GSSAPI 関連の外部ライブラリ関数（具体的な呼び出し先はソースコードの文脈からは直接特定されていません）。

関連データ構造

  • pg_gssinfo: GSSAPI 認証および暗号化の状態を保持する構造体。
  • Port: クライアント接続情報を保持する構造体（libpq-be.h 内で定義）。

失敗モードと境界条件

  • value が NULL の場合や、length が実際のデータサイズと一致しない場合、GSSAPI ライブラリ側でエラーが発生する可能性があります。
  • メモリ割り当ての失敗や、トークン処理時のバッファオーバーフローについては、各実装側のロジックに依存します。

保守担当者向けメモ

  • 本構造体は GSSAPI の標準的なデータ構造をラップしたものです。PostgreSQL 内部だけでなく、クライアント側の libpq でも同様の定義が使用されています。
  • 構造体の変更は GSSAPI 関連の全モジュールに影響するため、慎重に行う必要があります。
