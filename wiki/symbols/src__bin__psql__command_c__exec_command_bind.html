exec_command_bind

定義

  • 種別: function
  • 定義箇所: src/bin/psql/command.c:519

目的

psql の拡張クエリ実行用パラメータを設定します。ユーザーが指定したバインドパラメータを収集し、グローバル設定に格納します。

シグネチャまたは形状

static backslashResult exec_command_bind(PsqlScanState scan_state, bool active_branch);

入力と出力

  • **入力**:
  • scan_state: コマンド引数を解析するためのスキャン状態。
  • active_branch: 現在の条件分岐（\if）がアクティブかどうかを示すフラグ。
  • **出力**:
  • 戻り値: backslashResult 型。常に PSQL_CMD_SKIP_LINE を返します。

前提条件と不変条件

  • active_branch が true の場合、既存の拡張クエリ状態（pset.bind_params 等）をクリアするために clean_extended_state() が呼び出されます。
  • active_branch が false の場合、引数の解析は行われず、ignore_slash_options によって残りの引数が読み飛ばされます。

アルゴリズムまたは振る舞い

1. active_branch が真であれば、psql_scan_slash_option を用いて引数をループで取得します。 2. 取得した各引数を pset.bind_params 配列に格納します。必要に応じて pg_realloc_array で配列サイズを動的に拡張します。 3. 最終的なパラメータ数を pset.bind_nparams に設定します。 4. 実行モードを PSQL_SEND_EXTENDED_QUERY_PARAMS に変更します。 5. active_branch が偽であれば、ignore_slash_options を呼び出して引数を消費します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c 内の exec_command 関数から呼び出されます。
  • **呼び出し先**:
  • clean_extended_state()
  • psql_scan_slash_option()
  • pg_realloc_array()
  • ignore_slash_options()

関連データ構造

  • PsqlScanState: コマンド解析の状態管理。
  • pset: psql のグローバル設定構造体（bind_params, bind_nparams, send_mode を保持）。
  • backslashResult: コマンド実行結果の列挙型。

失敗モードと境界条件

  • メモリ割り当て失敗時: pg_realloc_array が失敗した場合の挙動は、当該関数の実装に依存します。
  • 引数なし: nparams が 0 の場合、pset.bind_nparams は 0 となり、パラメータなしの拡張クエリとして処理されます。

保守担当者向けメモ

  • 本関数は \bind コマンドの実装です。
  • pset.bind_params のメモリ管理は pset 構造体を通じて行われるため、他のコマンドとの競合やメモリリークに注意してください。
