ReplicationOriginExitCleanup

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/origin.c:1077

目的

プロセス終了時に、現在セッションが保持しているレプリケーション起点の獲得状態を解放し、待機中のプロセスへ通知を行う。

シグネチャまたは形状

static void ReplicationOriginExitCleanup(int code, Datum arg)

入力と出力

  • code: on_shmem_exit から渡される終了コード。
  • arg: on_shmem_exit から渡される補助引数（本関数では未使用）。
  • 出力: なし。

前提条件と不変条件

  • session_replication_state が有効な場合、その起点は現在のプロセス（MyProcPid）によって獲得されている必要がある。
  • ReplicationOriginLock を排他モードで取得して共有メモリ上の状態を更新する。

アルゴリズムまたは振る舞い

1. session_replication_state が NULL の場合は即座に終了する。 2. ReplicationOriginLock を排他モードで取得する。 3. 現在のセッションの acquired_by が自身のプロセスID（MyProcPid）と一致する場合：

  • session_replication_state->origin_cv をローカル変数に退避する。
  • acquired_by を 0 にリセットし、session_replication_state を NULL にクリアする。

    • 4. ReplicationOriginLock を解放する。 5. 退避した ConditionVariable が存在する場合、ConditionVariableBroadcast を呼び出し、当該起点の解放を待機しているプロセスを起床させる。

呼び出し元と呼び出し先

  • 呼び出し元: on_shmem_exit(ReplicationOriginExitCleanup, 0) を介してプロセス終了時に呼び出される。
  • 呼び出し先: LWLockAcquire, LWLockRelease, ConditionVariableBroadcast。

関連データ構造

  • ReplicationState: セッションが保持するレプリケーション起点の状態構造体。
  • ReplicationOriginLock: 起点テーブル全体を保護するロック。

失敗モードと境界条件

  • session_replication_state が NULL の場合、何もしない（安全に終了する）。
  • 自身のプロセスが起点を獲得していない場合、ロック取得後に状態変更は行われない。

保守担当者向けメモ

  • 本関数は on_shmem_exit に登録されるコールバックであるため、プロセスが異常終了または正常終了する際に確実に実行される必要がある。
  • ConditionVariableBroadcast はロックを解放した後に呼び出されるため、デッドロックの懸念はない。
