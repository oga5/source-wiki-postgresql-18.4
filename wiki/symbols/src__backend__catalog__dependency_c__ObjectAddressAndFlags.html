ObjectAddressAndFlags

定義

  • 種別: struct
  • 定義箇所: src/backend/catalog/dependency.c:132

目的

削除対象のデータベースオブジェクトのアドレス情報と、そのオブジェクトを再帰的に処理する際に適用されるフラグを保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	ObjectAddress obj;			/* 削除対象オブジェクト --- 構造体の先頭である必要がある */
	int			subflags;		/* 再帰処理時に引き継ぐフラグ */
} ObjectAddressAndFlags;

入力と出力

  • 本構造体は、依存関係の探索および削除処理において、オブジェクトのリスト（配列）として管理されます。
  • obj フィールドには対象オブジェクトの識別情報が格納され、subflags には削除時の挙動を制御するビットフラグが格納されます。

前提条件と不変条件

  • obj フィールドは構造体の先頭に配置されている必要があります。これは、qsort 等で ObjectAddress として直接比較を行う際に利用されるためです。

アルゴリズムまたは振る舞い

  • 依存関係の探索（findDependentObjects 等）において、削除対象のリストを構築する際に使用されます。
  • 削除処理の実行フェーズにおいて、リスト内の各要素に対して subflags を参照しながら再帰的な削除制御が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/dependency.c 内の findDependentObjects や削除リストを管理する関数群から、配列として動的に割り当てられ利用されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ObjectAddress: オブジェクトを識別するための基本構造体。
  • ObjectAddresses: ObjectAddressAndFlags の配列を管理する上位の構造体。

失敗モードと境界条件

  • メモリ割り当て: palloc を使用してリストを確保するため、メモリ不足時にはエラーが発生します。
  • 構造体の配置: obj が先頭にない場合、ポインタキャストによるソート処理などで未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は qsort によるソート対象となるため、フィールドの順序変更には注意が必要です。
  • subflags に追加の制御フラグを導入する場合は、既存のフラグと競合しないよう注意してください。
