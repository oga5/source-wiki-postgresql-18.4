IsInjectionPointAttached

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/injection_point.c:578

目的

指定された名前のインジェクションポイントが現在アタッチ（登録）されているかどうかを確認します。

シグネチャまたは形状

bool IsInjectionPointAttached(const char *name)

入力と出力

  • **入力**: name (インジェクションポイントの識別名)
  • **出力**: アタッチされている場合は true、それ以外の場合は false を返します。

前提条件と不変条件

  • 本機能は --enable-injection-points を指定してビルドされた環境でのみ動作します。
  • ビルド時に無効化されている場合、本関数を呼び出すとエラーが発生します。

アルゴリズムまたは振る舞い

1. USE_INJECTION_POINTS が定義されている場合、InjectionPointCacheRefresh(name) を呼び出します。 2. キャッシュの更新結果が NULL でなければ true を、NULL であれば false を返します。 3. USE_INJECTION_POINTS が定義されていない場合、elog(ERROR) を発行して処理を停止します。

呼び出し元と呼び出し先

  • **呼び出し元**: 直接の呼び出し元はソースコード上では明示されていませんが、src/include/utils/injection_point.h にて IS_INJECTION_POINT_ATTACHED(name) マクロとして公開されています。
  • **呼び出し先**: InjectionPointCacheRefresh を呼び出します。

関連データ構造

  • InjectionPointCacheEntry (キャッシュ管理に使用)

失敗モードと境界条件

  • ビルド時にインジェクションポイント機能が無効な場合、実行時に ERROR レベルのログが発生し、プロセスが中断されます。
  • name が無効な場合や、共有メモリ上に存在しない場合は false を返します。

保守担当者向けメモ

  • 本関数はデバッグおよびテスト目的のインターフェースです。本番環境での使用は想定されていません。
  • IS_INJECTION_POINT_ATTACHED マクロ経由での利用が推奨されます。
