ViewOptCheckOption

定義

  • 種別: enum
  • 定義箇所: src/include/utils/rel.h:413

目的

ビューの更新可能性を検証する際の「CHECK OPTION」制約の適用範囲を識別するために使用されます。

シグネチャまたは形状

typedef enum ViewOptCheckOption
{
    VIEW_OPTION_CHECK_OPTION_NOT_SET,
    VIEW_OPTION_CHECK_OPTION_LOCAL,
    VIEW_OPTION_CHECK_OPTION_CASCADED,
} ViewOptCheckOption;

入力と出力

  • 本シンボルは列挙型であり、関数への入力や構造体のフィールド値として利用されます。
  • 出力は、ビュー定義における WITH CHECK OPTION の設定状態（未設定、LOCAL、CASCADED）を表す定数です。

前提条件と不変条件

  • VIEW_OPTION_CHECK_OPTION_NOT_SET がデフォルト値として想定されます。
  • ビューのメタデータ管理において、この列挙値は check_option フィールドとして保持されます。

アルゴリズムまたは振る舞い

  • ビューに対する更新操作時に、この値に基づいて制約チェックの伝播範囲（現在のビューのみか、基底ビューまで含めるか）が決定されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/common/reloptions.c（値の参照を確認）
  • 呼び出し先: 提供されたソース情報からは特定の関数呼び出し関係は特定できません。

関連データ構造

  • ViewOptions: ビューのオプション情報を保持する構造体内で、check_option フィールドとして本型が使用されています。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 本シンボルはビューの更新制約に関連する重要なフラグです。reloptions.c 等でビューのオプションを解析する際は、この列挙値の整合性に注意してください。
