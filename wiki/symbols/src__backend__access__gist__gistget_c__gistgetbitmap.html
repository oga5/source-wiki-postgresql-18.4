gistgetbitmap

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistget.c:744

目的

GiSTインデックススキャンにおいて、検索条件に合致するすべてのヒープタプルのTID（Tuple Identifier）をビットマップ（TIDBitmap）へ格納し、その総数を返却する。

シグネチャまたは形状

int64 gistgetbitmap(IndexScanDesc scan, TIDBitmap *tbm)

入力と出力

  • **入力**:
  • scan: インデックススキャンの状態を保持する IndexScanDesc 構造体。
  • tbm: 検索結果のTIDを格納するための TIDBitmap 構造体。
  • **出力**:
  • int64: ビットマップに追加されたTIDの総数。

前提条件と不変条件

  • scan->opaque が GISTScanOpaque 型としてキャスト可能であること。
  • スキャン対象のインデックスが有効であること。
  • so->qual_ok が偽の場合、検索は即座に終了し 0 を返す。

アルゴリズムまたは振る舞い

1. **初期化**: スキャン統計情報を更新し、so->pageDataCxt をリセットしてメモリを解放する。 2. **ルート走査**: GIST_ROOT_BLKNO を起点として gistScanPage を呼び出し、最初のページを処理する。 3. **反復処理**: getNextGISTSearchItem を用いて、キュー（pairingheap）から次に走査すべきアイテムを順次取得する。 4. **ページ走査**: 取得した各アイテムに対して gistScanPage を呼び出し、条件に合致するTIDを tbm に追加する。 5. **終了**: キューが空になるまで繰り返し、最終的なTIDの総数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gist.c 内の amroutine->amgetbitmap に登録されており、インデックスアクセスメソッドのインターフェース経由で呼び出される。
  • **呼び出し先**:
  • pgstat_count_index_scan: 統計情報の更新。
  • MemoryContextReset: メモリコンテキストのリセット。
  • gistScanPage: インデックスページの走査とTIDの抽出。
  • getNextGISTSearchItem: 次の走査対象アイテムの取得。
  • CHECK_FOR_INTERRUPTS: 割り込みチェック。
  • pfree: メモリ解放。

関連データ構造

  • IndexScanDesc: スキャン全体の制御構造体。
  • GISTScanOpaque: GiST固有のスキャン状態（キューやメモリコンテキストを含む）。
  • TIDBitmap: 検索結果を保持するビットマップ。
  • GISTSearchItem: 走査対象のページや距離情報を保持する構造体。

失敗モードと境界条件

  • so->qual_ok が偽の場合、検索を行わずに 0 を返す。
  • 検索対象が存在しない場合、0 を返す。
  • 割り込みが発生した場合、CHECK_FOR_INTERRUPTS により処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数はビットマップスキャン専用のエントリポイントである。
  • gistScanPage 内でリーフページを処理する際、TIDは直接 tbm に格納されるため、本関数側での追加処理は不要である。
  • メモリ管理は so->pageDataCxt に依存しているため、コンテキストのライフサイクルに注意すること。
