check_timezone_abbreviations

定義

  • 種別: function
  • 定義箇所: src/backend/commands/variable.c:486

目的

GUCパラメータ timezone_abbreviations の設定値を検証し、指定されたタイムゾーン省略形ファイルを読み込んで適用可能な形式に変換します。

シグネチャまたは形状

bool check_timezone_abbreviations(char **newval, void **extra, GucSource source)

入力と出力

  • **入力**:
  • newval: 設定されるタイムゾーン省略形ファイル名（文字列へのポインタ）。
  • extra: 検証済みデータを格納するための出力用ポインタ。
  • source: 設定の発生源（GucSource）。
  • **出力**:
  • 成功時は true を返し、*extra に guc_malloc で確保された TimeZoneAbbrevTable を格納します。
  • 失敗時は false を返します。

前提条件と不変条件

  • *newval が NULL の場合、source は PGC_S_DEFAULT である必要があります。
  • *extra に割り当てられたメモリは、GUCサブシステムによって管理されます。

アルゴリズムまたは振る舞い

1. *newval が NULL の場合、何もせず true を返します（初期化時の最適化）。 2. *newval が指定されている場合、load_tzoffsets(*newval) を呼び出してファイルを解析します。 3. 解析結果が NULL の場合、GUC_check_errmsg を通じてエラーを報告し、false を返します。 4. 解析に成功した場合、結果を *extra に格納し、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: GUCサブシステム（src/backend/utils/misc/guc_tables.c にてフックとして登録）。
  • **呼び出し先**: load_tzoffsets（utils/tzparser.h 経由）。

関連データ構造

  • TimeZoneAbbrevTable: load_tzoffsets によって生成される、タイムゾーン省略形を保持する構造体。

失敗モードと境界条件

  • load_tzoffsets が失敗した場合（ファイルが存在しない、構文エラーなど）、false を返します。
  • InitializeGUCOptions() 実行時など、PGSHAREDIR が特定できない環境での読み込みを回避する設計となっています。

保守担当者向けメモ

  • 本関数は assign_timezone_abbreviations と対で使用されます。
  • *extra に割り当てられたメモリは guc_malloc を使用しているため、手動での pfree は不要です。
