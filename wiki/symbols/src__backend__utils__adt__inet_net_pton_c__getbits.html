getbits

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/inet_net_pton.c:347

目的

CIDR 表記のプレフィックス長（ビット数）を表す文字列を解析し、整数値に変換します。

シグネチャまたは形状

static int getbits(const char *src, int *bitsp);

入力と出力

  • **入力**:
  • src: 解析対象の文字列（例: "24"）。
  • bitsp: 変換結果を格納する整数へのポインタ。
  • **出力**:
  • 成功時: 1 を返し、*bitsp に解析された値を格納。
  • 失敗時: 0 を返す。

前提条件と不変条件

  • src は NULL 終端文字列であること。
  • bitsp は有効なメモリ領域を指していること。
  • 解析される数値は 0 から 128 の範囲内であること。

アルゴリズムまたは振る舞い

1. src を走査し、各文字が数字（'0'-'9'）であるかを確認します。 2. 先頭が '0' で始まる複数桁の数値（例: "01"）は不正として扱います。 3. 文字列から数値を計算し、val に蓄積します。 4. val が 128 を超えた場合、または文字列が空の場合、あるいは非数字が含まれる場合は 0 を返します。 5. 正常に解析が完了した場合、*bitsp に値を代入し 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/inet_net_pton.c 内の inet_cidr_pton_ipv4 および inet_cidr_pton_ipv6 から呼び出されます。
  • **呼び出し先**:
  • 標準ライブラリ関数 strchr を使用して数字の判定を行います。

関連データ構造

  • なし。

失敗モードと境界条件

  • **失敗モード**:
  • 文字列が空の場合。
  • 非数字が含まれる場合。
  • 先頭に不要な '0' がある場合（例: "01"）。
  • 数値が 128 を超える場合。
  • **境界条件**:
  • 0 は有効な値として扱われます。
  • 128 は有効な値として扱われます。

保守担当者向けメモ

  • 本関数は CIDR 表記のパース専用です。IPv6 のプレフィックス長（最大 128）を考慮した設計になっています。
  • strchr を使用した数字判定は、ロケールに依存せず ASCII 文字列を対象としています。
