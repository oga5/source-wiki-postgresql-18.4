PrintfArgType

定義

  • 種別: enum
  • 定義箇所: src/port/snprintf.c:142

目的

printf ファミリーのフォーマット文字列解析において、va_list から取得される引数の型を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
	ATYPE_NONE = 0,
	ATYPE_INT,
	ATYPE_LONG,
	ATYPE_LONGLONG,
	ATYPE_DOUBLE,
	ATYPE_CHARPTR
} PrintfArgType;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし

前提条件と不変条件

  • ATYPE_NONE は 0 に初期化され、未設定または無効な引数型を表します。
  • %n$ 形式の引数順序制御を行う際、各引数の型を保持するために使用されます。

アルゴリズムまたは振る舞い

find_arguments() 関数内でフォーマット文字列を走査する際、各変換指定子に対応する引数の型を判定し、この列挙値として記録します。その後、記録された型に基づいて va_list から適切なサイズの値を抽出します。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/snprintf.c 内の find_arguments() およびフォーマット解析ロジック。
  • 呼び出し先: なし（型定義のため）。

関連データ構造

  • PrintfArgValue: PrintfArgType で指定された型に対応する実際の値を保持する共用体（union）。

失敗モードと境界条件

  • %n$ 指定子で指定された引数インデックスが PG_NL_ARGMAX (31) を超える場合、または型が正しく解決できない場合に、ATYPE_NONE が使用される可能性があります。

保守担当者向けメモ

  • 新しいフォーマット指定子（例: long double や intmax_t 等）をサポートする必要がある場合は、この列挙型に新しいメンバーを追加し、対応する PrintfArgValue 共用体および find_arguments() の解析ロジックを更新してください。
