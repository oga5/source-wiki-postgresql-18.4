OperatorShellMake

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_operator.c:192

目的

システムカタログ pg_operator において、未定義の演算子を「シェル（プレースホルダー）」として先行作成し、OID を確保する。これにより、相互参照（交換子や否定子など）を持つ演算子の定義を可能にする。

シグネチャまたは形状

static Oid OperatorShellMake(const char *operatorName, Oid operatorNamespace, Oid leftTypeId, Oid rightTypeId)

入力と出力

  • **入力**:
  • operatorName: 演算子名。
  • operatorNamespace: 演算子が属する名前空間の OID。
  • leftTypeId: 左オペランドの型 OID（単項演算子の場合は 0）。
  • rightTypeId: 右オペランドの型 OID。
  • **出力**:
  • 作成されたシェル演算子の OID を返す。

前提条件と不変条件

  • operatorName は validOperatorName による妥当性検証を通過する必要がある。
  • pg_operator テーブルへの書き込み権限が必要。
  • 作成されるシェルは、oprcode（実装関数）が InvalidOid であることで識別される。

アルゴリズムまたは振る舞い

1. validOperatorName で名前の妥当性を確認し、不正な場合はエラーを報告する。 2. pg_operator テーブルを RowExclusiveLock で開く。 3. GetNewOidWithIndex を使用して、新しい演算子用の OID を取得する。 4. 演算子の属性（名前、名前空間、所有者、型、演算子種別など）を初期化する。oprcode、oprcom、oprnegate 等の実行時属性はすべて InvalidOid に設定する。 5. heap_form_tuple でタプルを構築し、CatalogTupleInsert でカタログに挿入する。 6. makeOperatorDependencies を呼び出し、依存関係を記録する。 7. InvokeObjectPostCreateHook を実行し、CommandCounterIncrement で変更を可視化する。 8. テーブルを閉じて、取得した OID を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: get_other_operator（src/backend/catalog/pg_operator.c 内）。
  • **呼び出し先**: validOperatorName, table_open, GetNewOidWithIndex, heap_form_tuple, CatalogTupleInsert, makeOperatorDependencies, heap_freetuple, InvokeObjectPostCreateHook, CommandCounterIncrement, table_close。

関連データ構造

  • pg_operator: 演算子情報を保持するシステムカタログ。
  • Natts_pg_operator: カタログの属性数。
  • OperatorOidIndexId: OID 検索用のインデックス。

失敗モードと境界条件

  • validOperatorName が偽を返す場合、ERRCODE_INVALID_NAME でエラー終了する。
  • カタログへの挿入や OID 取得に失敗した場合、PostgreSQL の標準的なエラー処理によりトランザクションがアボートされる可能性がある。

保守担当者向けメモ

  • 本関数は「シェル」を作成するのみであり、実際の演算子ロジック（oprcode）は含まれない。実体化には別途 OperatorUpd 等による更新が必要である。
  • 依存関係の登録において makeOperatorDependencies を使用しているため、型定義の変更や削除時に適切に追跡される。
