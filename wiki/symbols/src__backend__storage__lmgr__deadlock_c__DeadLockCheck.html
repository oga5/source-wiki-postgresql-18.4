DeadLockCheck

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/deadlock.c:219

目的

指定されたプロセス（PGPROC）に関連するデッドロックの検出および、ロック待ち行列の再構成によるデッドロックの解消を試みる。

シグネチャまたは形状

DeadLockState DeadLockCheck(PGPROC *proc)

入力と出力

  • **入力**: proc (デッドロックチェックの起点となるプロセス)
  • **出力**: DeadLockState 型の列挙値
  • DS_HARD_DEADLOCK: 解消不可能なデッドロックを検出。
  • DS_SOFT_DEADLOCK: ロック待ち行列の順序変更によりデッドロックを解消。
  • DS_BLOCKED_BY_AUTOVACUUM: オートバキュームによるブロックを検出。
  • DS_NO_DEADLOCK: デッドロックなし。

前提条件と不変条件

  • 呼び出し時に全ロックパーティションのロックを保持している必要がある。
  • 内部で使用する作業用メモリ（possibleConstraints 等）は初期化済みであること。

アルゴリズムまたは振る舞い

1. **初期化**: 制約および待ち順序のカウンタをリセット。 2. **再帰的探索**: DeadLockCheckRecurse を呼び出し、循環参照を探索。 3. **ハードデッドロック判定**: 循環が解消できない場合、FindLockCycle で詳細を記録し FATAL エラーを発生させる。 4. **待ち行列の再構成**: 解決可能な場合、waitOrders に基づいて対象ロックの待ち行列（waitProcs）を再構築し、ProcLockWakeup を呼び出して待機中のプロセスを再評価させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/proc.c:1824 (DeadLockCheck(MyProc))
  • src/backend/storage/ipc/standby.c:900 (間接的な言及)
  • **呼び出し先**:
  • DeadLockCheckRecurse
  • FindLockCycle
  • dclist_init, dclist_push_tail
  • ProcLockWakeup
  • GetLocksMethodTable

関連データ構造

  • PGPROC: ロック待ち状態のプロセス情報。
  • LOCK: ロック対象と待ち行列（waitProcs）を保持。
  • WAIT_ORDER: 待ち行列の再構成に必要な情報。

失敗モードと境界条件

  • 循環参照が解消できない場合、elog(FATAL) によりプロセスが終了する。
  • 待ち行列の再構成中に nProcs と dclist_count が一致しない場合、Assert により停止する。

保守担当者向けメモ

  • 本関数はシグナルハンドラから呼び出される可能性があるため、メモリ割り当て等の重い処理は避けること。
  • waitOrders の適用は、デッドロック解消のための「ソフトエッジ」の解決を意味する。
