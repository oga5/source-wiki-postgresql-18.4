CopyLoadRawBuf

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfromparse.c:589

目的

COPY FROM コマンドにおいて、データソース（ファイルまたはクライアント）から未処理の生データを読み込み、raw_buf に格納する。

シグネチャまたは形状

static void CopyLoadRawBuf(CopyFromState cstate)

入力と出力

  • **入力**: CopyFromState cstate（COPY 操作の現在の状態を保持する構造体）。
  • **出力**: なし（副作用として cstate 内のバッファ状態および統計情報を更新）。

前提条件と不変条件

  • raw_buf と input_buf が同一の場合、エンコーディング変換が不要であること（!cstate->need_transcoding）がアサートされる。
  • 同一バッファの場合、raw_buf_index と input_buf_index が一致している必要がある。

アルゴリズムまたは振る舞い

1. **バッファ整理**: raw_buf 内の未処理データ（raw_buf_index 以降）をバッファの先頭へ memmove で移動し、インデックスを 0 にリセットする。 2. **同期**: raw_buf と input_buf が同一であれば、input_buf のインデックスと長さも同様に調整する。 3. **データ読み込み**: CopyGetData を呼び出し、バッファの空き領域に新しいデータを読み込む。 4. **終端処理**: 読み込んだデータの直後にヌル終端文字（\0）を配置し、raw_buf_len を更新する。 5. **統計更新**: bytes_processed を加算し、pgstat_progress_update_param を通じて進捗状況を更新する。 6. **EOF 判定**: 読み込みバイト数が 0 の場合、raw_reached_eof フラグを true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の参照から、src/backend/commands/copyfromparse.c 内の他の解析関数から呼び出されていることが確認できる。
  • **呼び出し先**: CopyGetData を呼び出す。

関連データ構造

  • CopyFromState: COPY FROM の実行状態を管理する構造体（commands/copyfrom_internal.h で定義）。

失敗モードと境界条件

  • CopyGetData が 0 バイトを返した場合、EOF に到達したものとして処理される。
  • バッファが満杯の場合、読み込みは行われない。

保守担当者向けメモ

  • 本関数は raw_buf のメモリ管理とデータソースからの供給を抽象化している。
  • raw_buf と input_buf が共有されているケースと分離されているケースの両方を考慮した実装となっているため、バッファ操作時は両者の整合性に注意すること。
