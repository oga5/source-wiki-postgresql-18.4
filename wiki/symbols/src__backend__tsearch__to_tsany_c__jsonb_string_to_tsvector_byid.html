jsonb_string_to_tsvector_byid

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/to_tsany.c:300

目的

指定された全文検索設定（Config OID）に基づき、JSONB オブジェクト内の文字列要素を抽出して tsvector 型へ変換します。

シグネチャまたは形状

Datum jsonb_string_to_tsvector_byid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): 全文検索設定の OID。
  • PG_GETARG_JSONB_P(1): 変換対象の Jsonb ポインタ。
  • **出力**:
  • PG_RETURN_TSVECTOR(result): 変換された TSVector 構造体。

前提条件と不変条件

  • 入力された Jsonb オブジェクトは、関数終了時に PG_FREE_IF_COPY によって適切にメモリ解放されます。
  • 内部処理において jtiString モードが指定されます。

アルゴリズムまたは振る舞い

1. jsonb_to_tsvector_worker を呼び出し、第1引数の OID と第2引数の JSONB データ、および文字列抽出モード（jtiString）を渡して処理を実行します。 2. ワーカ関数から返された TSVector 結果を保持します。 3. 入力された JSONB がコピーである場合に限り、メモリを解放します。 4. 構築された TSVector を Datum として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されます（直接的な C 言語レベルの呼び出し元情報は提供されていません）。
  • **呼び出し先**: jsonb_to_tsvector_worker を呼び出します。

関連データ構造

  • Jsonb: 入力データ構造。
  • TSVector: 出力データ構造。
  • jtiString: JSONB から文字列のみを抽出するための列挙型定数。

失敗モードと境界条件

  • JSONB の解析や全文検索設定の取得に失敗した場合、PostgreSQL のエラーハンドリングメカニズムにより例外がスローされます。

保守担当者向けメモ

  • 本関数は jsonb_to_tsvector_worker のラッパーです。JSONB の構造走査ロジックに変更を加える場合は、ワーカ関数側を確認してください。
  • jtiString モードを使用しているため、JSONB 内の数値やブール値などは無視され、文字列値のみが全文検索の対象となります。
