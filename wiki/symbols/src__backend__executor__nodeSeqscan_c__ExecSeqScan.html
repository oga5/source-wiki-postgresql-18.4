ExecSeqScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSeqscan.c:109

目的

リレーショナルテーブルのシーケンシャルスキャン（全件走査）を実行する。本関数は、フィルタリング（Qual）やプロジェクションを伴わない、最も単純なスキャン実行パスを担当する。

シグネチャまたは形状

static TupleTableSlot *ExecSeqScan(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（実行中のスキャンノードの状態）
  • **出力**: TupleTableSlot *（スキャンによって取得されたタプル、またはスキャン終了時は空のスロット）

前提条件と不変条件

  • pstate->state->es_epq_active が NULL であること（EvalPlanQualがアクティブでないこと）。
  • pstate->qual が NULL であること（フィルタ条件が存在しないこと）。
  • pstate->ps_ProjInfo が NULL であること（プロジェクションが存在しないこと）。

アルゴリズムまたは振る舞い

1. pstate を SeqScanState にキャストする。 2. 前提条件を満たしていることを Assert で検証する。 3. ExecScanExtended を呼び出し、以下のメソッドを渡してスキャンを実行する：

  • SeqNext: テーブルアクセスメソッドから次のタプルを取得する。
  • SeqRecheck: タプルの再評価を行う。
  • その他（Qual, ProjInfo, ExprContext）には NULL を渡す。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecProcNode（実行エンジンの中核）から呼び出される。
  • **呼び出し先**:
  • SeqNext（タプル取得用）
  • SeqRecheck（再評価用）
  • ExecScanExtended（スキャン実行の共通基盤）

関連データ構造

  • SeqScanState: スキャンノードの実行状態を保持する構造体。
  • PlanState: 実行計画ノードの共通状態。
  • TupleTableSlot: タプルを格納する一時的なメモリ領域。

失敗モードと境界条件

  • テーブルの末尾に到達した場合、SeqNext を介して空のスロットが返される。
  • 前提条件（QualやProjInfoの不在）が満たされない場合、Assert によりデバッグビルド環境で停止する。

保守担当者向けメモ

  • 本関数は最適化された実行パスの一つである。フィルタリングやプロジェクションが必要な場合は、ExecSeqScanWithQual、ExecSeqScanWithProject、ExecSeqScanWithQualProject がそれぞれ使用される。
  • 実行パスの選択は ExecInitSeqScan で行われるため、ロジック変更時は初期化処理との整合性に注意すること。
