ShutdownWalRecovery

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogrecovery.c:1626

目的

WALリカバリプロセスの終了処理を行い、リカバリに関連するリソースの解放および一時ファイルのクリーンアップを実行します。

シグネチャまたは形状

void ShutdownWalRecovery(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてリソース解放およびファイル削除を実行）

前提条件と不変条件

  • 本関数はリカバリ終了時に呼び出されることを想定しています。
  • xlogreader および xlogprefetcher が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. **統計情報の更新**: XLogPrefetcherComputeStats を呼び出し、xlogprefetcher の最終統計を計算します。 2. **リーダーの解放**: readFile が開いている場合はクローズし、XLogReaderFree および XLogPrefetcherFree を呼び出してメモリを解放します。 3. **一時ファイルの削除**: ArchiveRecoveryRequested が真の場合、リカバリ中に作成された可能性のある RECOVERYXLOG および RECOVERYHISTORY ファイルを unlink で削除します。 4. **ラッチの解放**: ArchiveRecoveryRequested が真の場合、DisownLatch を呼び出して XLogRecoveryCtl->recoveryWakeupLatch の所有権を放棄します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlog.c:6147
  • 呼び出し先:
  • XLogPrefetcherComputeStats
  • XLogReaderFree
  • XLogPrefetcherFree
  • unlink
  • DisownLatch

関連データ構造

  • XLogRecoveryCtl: リカバリ状態を保持する共有メモリ構造体（ラッチの所有権管理に使用）。
  • xlogreader, xlogprefetcher: リカバリ中に使用されるグローバルな読み込み・先読み状態管理オブジェクト。

失敗モードと境界条件

  • unlink 失敗時: エラーは無視され、処理は継続されます。
  • readFile が既に閉じられている場合: readFile が -1 であればクローズ処理はスキップされます。

保守担当者向けメモ

  • 本関数はリカバリ終了時のクリーンアップを網羅する役割を持ちます。新しいリカバリ関連リソースを追加した場合は、本関数での解放処理を検討してください。
  • ArchiveRecoveryRequested フラグに基づいた条件分岐が含まれているため、スタンバイモードやアーカイブモードの切り替えロジックに変更を加える際は注意が必要です。
