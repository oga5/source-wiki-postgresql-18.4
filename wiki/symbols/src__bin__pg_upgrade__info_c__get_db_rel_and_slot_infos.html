get_db_rel_and_slot_infos

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/info.c:279

目的

PostgreSQL クラスタ（新旧）からデータベース、リレーション、および論理レプリケーションスロットのメタデータを収集し、ClusterInfo 構造体に格納する高レベルなエントリーポイントです。

シグネチャまたは形状

void get_db_rel_and_slot_infos(ClusterInfo *cluster)

入力と出力

  • **入力**: ClusterInfo *cluster（情報を収集対象とするクラスタのポインタ）
  • **出力**: なし（副作用として cluster 引数が指す構造体の内部データが更新されます）

前提条件と不変条件

  • cluster->dbarr.dbs が既に存在する場合、関数内で free_db_and_rel_infos が呼び出され、メモリが解放されます。
  • 論理レプリケーションスロットの収集は、対象が old_cluster であり、かつメジャーバージョンが 17 以降（> 1600）である場合にのみ実行されます。

アルゴリズムまたは振る舞い

1. 既存のデータベースおよびリレーション情報を解放します。 2. get_template0_info および get_db_infos を呼び出し、基本情報を取得します。 3. upgrade_task_create でタスクを生成し、リレーション情報取得クエリ（get_rel_infos_query）をタスクに追加します。 4. 条件を満たす場合、論理スロット情報取得クエリ（get_old_cluster_logical_slot_infos_query）をタスクに追加します。 5. upgrade_task_run を実行してクエリを処理し、タスクを解放します。 6. 最後に、ログ設定が有効であれば print_db_infos を使用して収集した情報を出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/check.c (old_cluster および new_cluster に対して呼び出し)
  • src/bin/pg_upgrade/pg_upgrade.c (new_cluster に対して呼び出し)
  • **呼び出し先**:
  • upgrade_task_create, upgrade_task_add_step, upgrade_task_run, upgrade_task_free
  • get_template0_info, get_db_infos, get_rel_infos_query, get_old_cluster_logical_slot_infos_query
  • process_rel_infos, process_old_cluster_logical_slot_infos (コールバックとして登録)
  • free_db_and_rel_infos, print_db_infos, pg_free, pg_log

関連データ構造

  • ClusterInfo: クラスタのメタデータを保持する主要構造体。
  • UpgradeTask: クエリ実行と結果処理を管理するタスク構造体。

失敗モードと境界条件

  • 論理スロットの移行は PG17 未満のクラスタではサポートされず、スキップされます。
  • old_cluster 以外のクラスタに対しては、論理スロットの収集ロジックは実行されません。

保守担当者向けメモ

  • 本関数は pg_upgrade におけるデータ収集の起点です。クエリの追加や処理ロジックの変更は、UpgradeTask のステップとして実装する必要があります。
  • 論理スロットの移行に関する条件分岐は、データ損失を防ぐための重要な制約（PG17 以降の仕様）に基づいています。
