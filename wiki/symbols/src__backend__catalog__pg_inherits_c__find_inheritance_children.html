find_inheritance_children

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_inherits.c:57

目的

指定された親テーブル（parentrelId）の直系の子テーブルのリストを取得します。

シグネチャまたは形状

List *find_inheritance_children(Oid parentrelId, LOCKMODE lockmode)

入力と出力

  • **入力**:
  • parentrelId (Oid): 検索対象の親テーブルのOID。
  • lockmode (LOCKMODE): 対象リレーションに対して取得するロックモード。
  • **出力**:
  • List *: 子テーブルのOIDを格納したリスト。

前提条件と不変条件

  • 呼び出し元は、対象のリレーションに対して適切なロックを保持している必要があります。NoLock を指定する場合は、並行的な DROP 操作によるレースコンディションに注意してください。

アルゴリズムまたは振る舞い

1. find_inheritance_children_extended を呼び出し、デフォルトの引数（missing_ok=true）を渡して処理を委譲します。 2. 内部的には pg_inherits システムカタログを InheritsParentIndexId でスキャンし、該当する子テーブルを収集します。 3. パーティションのデタッチ処理中（inhdetachpending）のテーブルについては、現在のスナップショットに基づいて可視性を判断し、必要に応じて除外します。

呼び出し元と呼び出し先

  • **呼び出し先**: find_inheritance_children_extended
  • **呼び出し元**:
  • src/backend/catalog/pg_inherits.c (内部参照)
  • src/backend/commands/tablecmds.c (複数のDDLコマンド処理箇所)

関連データ構造

  • List (PostgreSQLのリスト構造)
  • pg_inherits (システムカタログ)

失敗モードと境界条件

  • 指定された parentrelId が存在しない場合や、継承関係を持たない場合の挙動は、find_inheritance_children_extended の実装に依存します。
  • ロックモードの不適切な指定は、並行処理における整合性エラーを引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は find_inheritance_children_extended のラッパーです。詳細なロジックやデタッチ処理の挙動については、find_inheritance_children_extended を参照してください。
  • 継承関係の探索は、クエリプランニング時の継承展開やDDL操作の整合性チェックにおいて重要な役割を果たします。
