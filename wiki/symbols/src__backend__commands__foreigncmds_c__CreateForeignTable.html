CreateForeignTable

定義

  • 種別: function
  • 定義箇所: src/backend/commands/foreigncmds.c:1419

目的

外部テーブル定義をシステムカタログ pg_foreign_table に登録し、外部サーバーとの依存関係を確立する。

シグネチャまたは形状

void CreateForeignTable(CreateForeignTableStmt *stmt, Oid relid)

入力と出力

  • **入力**:
  • stmt: 外部テーブル作成のための構文解析ツリー（サーバー名、オプション等を含む）。
  • relid: 作成対象テーブルのオブジェクトID。
  • **出力**: なし（副作用としてシステムカタログを更新）。

前提条件と不変条件

  • CommandCounterIncrement() を呼び出し、先行する処理（制約追加等）による pg_attribute の変更を可視化している。
  • 実行ユーザーは、指定された外部サーバーに対して USAGE 権限を保持している必要がある。

アルゴリズムまたは振る舞い

1. pg_foreign_table リレーションを RowExclusiveLock でオープンする。 2. GetForeignServerByName によりサーバー情報を取得し、権限チェックを行う。 3. GetForeignDataWrapper を介して、オプション検証に必要なFDW情報を取得する。 4. transformGenericOptions を呼び出し、ユーザー指定のオプションを検証・変換する。 5. heap_form_tuple を使用して pg_foreign_table 用のタプルを構築し、CatalogTupleInsert で挿入する。 6. recordDependencyOn を使用し、作成したテーブルと外部サーバー間の DEPENDENCY_NORMAL な依存関係を記録する。 7. リレーションをクローズする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c:1210（SQL解析後の実行フロー）
  • **呼び出し先**:
  • table_open / table_close
  • GetForeignServerByName
  • object_aclcheck / aclcheck_error
  • GetForeignDataWrapper
  • transformGenericOptions
  • heap_form_tuple / CatalogTupleInsert / heap_freetuple
  • recordDependencyOn

関連データ構造

  • CreateForeignTableStmt: SQLステートメントの解析結果。
  • pg_foreign_table: 外部テーブル情報を保持するシステムカタログ。
  • ForeignServer, ForeignDataWrapper: 外部サーバーおよびFDWのメタデータ構造。

失敗モードと境界条件

  • 指定された外部サーバーが存在しない場合、エラーが発生する。
  • 外部サーバーに対する USAGE 権限がない場合、aclcheck_error によりエラーが発生する。
  • transformGenericOptions によるオプション検証で不正な値が検出された場合、エラーが発生する。

保守担当者向けメモ

  • 本関数はDDL定義のみを扱い、実行時のデータアクセスは担当しない。
  • オプションのバリデーションロジックは transformGenericOptions に集約されているため、仕様変更時は同関数を確認すること。
