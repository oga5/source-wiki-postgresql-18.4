pgwin32_setenv

定義

  • 種別: function
  • 定義箇所: src/port/win32env.c:120

目的

Windows 環境において、POSIX 互換の setenv セマンティクスを提供し、プロセス内の環境変数を設定します。

シグネチャまたは形状

int pgwin32_setenv(const char *name, const char *value, int overwrite);

入力と出力

  • **入力**:
  • name: 設定する環境変数名。
  • value: 設定する値。
  • overwrite: 0 以外の場合、既存の値を上書きする。0 の場合は既存の値を保持する。
  • **出力**: 成功時に 0、失敗時に -1 を返す。

前提条件と不変条件

  • name が NULL、空文字列、または '=' を含む場合はエラー（errno = EINVAL）となります。
  • value が NULL の場合はエラー（errno = EINVAL）となります。

アルゴリズムまたは振る舞い

1. 引数の妥当性を検証し、不正な場合は EINVAL を設定して終了します。 2. overwrite が 0 で、かつ環境変数が既に存在する場合は、何もせず 0 を返します。 3. name=value 形式の文字列を生成するために必要なメモリを確保します。 4. pgwin32_putenv を呼び出し、OS レベルおよびロード済みの全 CRT 間で環境変数を同期させます。 5. 確保したメモリを解放し、pgwin32_putenv の結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/win32_port.h にて setenv マクロとして定義されており、POSIX 互換性を求めるコードから呼び出されます。
  • **呼び出し先**: pgwin32_putenv（内部で環境変数の同期を実行）。

関連データ構造

  • なし。

失敗モードと境界条件

  • malloc が失敗した場合、-1 を返します。
  • 引数に不正な文字（'='）が含まれる場合や、name/value が NULL の場合は -1 を返し、errno に EINVAL が設定されます。

保守担当者向けメモ

  • 本関数は引数の文字列を永続化しないため、呼び出し元でメモリ管理を行う必要があります。
  • pgwin32_putenv が依存する modulenames 配列に新しい CRT が追加された場合、同期対象が自動的に拡張されます。
