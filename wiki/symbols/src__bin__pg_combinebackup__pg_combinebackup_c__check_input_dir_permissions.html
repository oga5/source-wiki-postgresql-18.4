check_input_dir_permissions

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/pg_combinebackup.c:679

目的

指定されたディレクトリのファイル属性（パーミッション）を取得し、その情報を基にデータディレクトリ作成時のパーミッション設定を更新する。

シグネチャまたは形状

void check_input_dir_permissions(char *dir)

入力と出力

  • **入力**: dir (対象ディレクトリのパスを示す文字列)
  • **出力**: なし（副作用としてグローバルなパーミッション設定を更新）

前提条件と不変条件

  • dir で指定されたパスが存在し、プロセスが stat を実行する権限を持っていること。

アルゴリズムまたは振る舞い

1. stat(dir, &st) を呼び出し、指定ディレクトリのファイル情報を取得する。 2. stat が失敗した場合は pg_fatal を呼び出し、プログラムを異常終了させる。 3. 取得した st.st_mode を引数として SetDataDirectoryCreatePerm を呼び出し、データディレクトリ作成時のパーミッションを同期する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_combinebackup/pg_combinebackup.c:315 (メイン処理フロー内)
  • **呼び出し先**:
  • stat (システムコール)
  • pg_fatal (エラーハンドリング)
  • SetDataDirectoryCreatePerm (パーミッション設定更新)

関連データ構造

  • struct stat (ファイル属性保持用)

失敗モードと境界条件

  • stat が失敗した場合（ディレクトリが存在しない、またはアクセス権限がない場合）、pg_fatal によりエラーメッセージを出力して即座に終了する。

保守担当者向けメモ

  • 本関数は、入力バックアップディレクトリのパーミッションを後続の出力ディレクトリ作成時に継承させるために使用される。
  • SetDataDirectoryCreatePerm がグローバルな状態を変更するため、呼び出し順序に注意すること。
