mask_page_lsn_and_checksum

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/bufmask.c:30

目的

WAL（Write-Ahead Logging）の整合性チェックにおいて、ページ比較時にノイズとなるページヘッダー内の LSN（Log Sequence Number）とチェックサムを、共通のマーカー値で上書きして無効化する。

シグネチャまたは形状

void mask_page_lsn_and_checksum(Page page)

入力と出力

  • **入力**: Page page - マスク対象となるページへのポインタ。
  • **出力**: なし（引数として渡されたページの内容をインプレースで書き換える）。

前提条件と不変条件

  • 引数 page は有効な PageHeader 構造体としてキャスト可能であること。
  • 本関数はテスト目的の正規化ツールであり、実稼働時のページ更新には使用しないこと。

アルゴリズムまたは振る舞い

1. page を PageHeader 型にキャストする。 2. PageXLogRecPtrSet マクロを使用して、pd_lsn を MASK_MARKER（通常 0）で上書きする。 3. pd_checksum フィールドを MASK_MARKER で上書きする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_xlog.c
  • src/backend/access/gin/ginxlog.c
  • src/backend/access/gist/gistxlog.c
  • src/backend/access/hash/hash_xlog.c
  • src/backend/access/heap/heapam_xlog.c
  • src/backend/access/nbtree/nbtxlog.c
  • src/backend/access/spgist/spgxlog.c
  • src/backend/access/transam/generic_xlog.c
  • src/backend/commands/sequence.c
  • **呼び出し先**:
  • PageXLogRecPtrSet（マクロ）

関連データ構造

  • PageHeader: ページヘッダーの構造体。
  • MASK_MARKER: マスクに使用される定数（通常 0）。

失敗モードと境界条件

  • page が NULL または不正なメモリ領域を指している場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は memcmp 等によるページ比較の前に呼び出すことで、WAL 適用前後で変化するメタデータによる誤検知を防ぐ。
  • 変更が必要な場合は、src/include/access/bufmask.h で定義されている MASK_MARKER の定義と整合性を取ること。
