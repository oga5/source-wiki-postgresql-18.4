lazy_scan_new_or_empty

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/vacuumlazy.c:1808

目的

VACUUM処理中に遭遇した「新規ページ（All-zeroes）」または「空ページ」を検出し、FSM（空き領域マップ）への登録や可視性マップの更新を行う。

シグネチャまたは形状

static bool lazy_scan_new_or_empty(LVRelState *vacrel, Buffer buf, BlockNumber blkno,
                                   Page page, bool sharelock, Buffer vmbuffer)

入力と出力

  • **入力**:
  • vacrel: VACUUM実行状態を保持する LVRelState 構造体。
  • buf: 対象ページのバッファ。
  • blkno: 対象ページのブロック番号。
  • page: 対象ページのポインタ。
  • sharelock: 現在のロックが共有ロックであるかを示すフラグ。
  • vmbuffer: 可視性マップのバッファ。
  • **出力**:
  • bool: ページが新規または空であり、処理が完了した場合は true。それ以外の場合は false。

前提条件と不変条件

  • 呼び出し時点で、対象ページに対するバッファピンとロックが保持されていること。
  • 処理が完了（true を返す）した場合、対象ページのバッファはアンロックおよび解放される。

アルゴリズムまたは振る舞い

1. **新規ページ（PageIsNew）の処理**:

  • ページが完全にゼロ埋めされている場合、バッファを解放する。
  • FSMに空き領域が記録されていない場合、ページ全体を空き領域としてFSMに登録する。

    • 2. **空ページ（PageIsEmpty）の処理**:

  • sharelock が真の場合、排他ロックに昇格させる。昇格後も空であることを再確認する。
  • ページが可視性マップで all-visible でない場合、クリティカルセクションを開始し、以下を実行する：
  • 必要に応じてWALログを出力（log_newpage_buffer）。
  • ページを all-visible および all-frozen に設定し、可視性マップを更新する。
  • 統計情報を更新し、FSMに空き領域を登録してバッファを解放する。

    • 3. **その他**:

  • どちらにも該当しない場合、ロックとピンを保持したまま false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: lazy_scan_heap（src/backend/access/heap/vacuumlazy.c 内のメインスキャンループ）。
  • **呼び出し先**: UnlockReleaseBuffer, GetRecordedFreeSpace, RecordPageWithFreeSpace, LockBuffer, MarkBufferDirty, RelationNeedsWAL, PageGetLSN, log_newpage_buffer, PageSetAllVisible, visibilitymap_set, PageGetHeapFreeSpace。

関連データ構造

  • LVRelState: VACUUMの実行状態を管理。
  • Buffer: バッファマネージャの管理単位。
  • Page: ヒープページのメモリ表現。

失敗モードと境界条件

  • **ロック昇格**: 空ページ処理中に共有ロックから排他ロックへ昇格する際、ページ状態が変化した場合は処理を中断し false を返す。
  • **WALログ**: ページが初期化されていない状態でリカバリが発生しないよう、必要に応じてWALログを出力する。

保守担当者向けメモ

  • 本関数は lazy_scan_heap からの呼び出し専用である。
  • 空ページに対する可視性マップの更新は、将来のVACUUM負荷を軽減するための「意欲的（Eager）」な最適化の一環である。
