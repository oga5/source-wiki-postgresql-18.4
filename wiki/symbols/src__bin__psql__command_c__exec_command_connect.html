exec_command_connect

定義

  • 種別: function
  • 定義箇所: src/bin/psql/command.c:637

目的

psql の \connect コマンドを実行し、データベース接続の確立または切り替えを行う。

シグネチャまたは形状

static backslashResult exec_command_connect(PsqlScanState scan_state, bool active_branch)

入力と出力

  • **入力**:
  • scan_state: コマンド引数を読み取るためのスキャン状態。
  • active_branch: 現在の \if ブロックがアクティブかどうかを示すフラグ。
  • **出力**:
  • backslashResult: 処理結果（成功時は PSQL_CMD_SKIP_LINE、失敗時は PSQL_CMD_ERROR）。

前提条件と不変条件

  • active_branch が false の場合、コマンド引数の解析は行われず、ignore_slash_options により引数がスキップされる。
  • active_branch が true の場合、最大4つの引数（reuse-previous オプションを含む）を読み取る。

アルゴリズムまたは振る舞い

1. active_branch が false の場合、ignore_slash_options を呼び出して終了する。 2. active_branch が true の場合:

  • 最初の引数が -reuse-previous= で始まるか確認し、該当する場合は ParseVariableBool で値を解析する。
  • 続いて最大3つの引数を read_connect_arg で取得する。
  • do_connect を呼び出し、接続処理を実行する。
  • 取得した引数メモリを解放する。

    • 3. 接続処理の成否に応じて PSQL_CMD_SKIP_LINE または PSQL_CMD_ERROR を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c 内の exec_command 関数（推測）。
  • **呼び出し先**:
  • read_connect_arg
  • ParseVariableBool
  • do_connect
  • ignore_slash_options
  • free

関連データ構造

  • PsqlScanState: コマンド引数の解析に使用。
  • backslashResult: 戻り値の型。
  • trivalue: reuse_previous の状態管理に使用。

失敗モードと境界条件

  • ParseVariableBool が失敗した場合、接続処理は行われずエラーとなる。
  • do_connect が失敗した場合、戻り値は PSQL_CMD_ERROR となる。
  • 引数が不足している場合や不正な形式の場合、do_connect の内部ロジックに依存する。

保守担当者向けメモ

  • 本関数は \connect コマンドのディスパッチャである。
  • reuse-previous オプションの解析ロジックは、引数の順序と read_connect_arg の呼び出し回数に依存しているため、引数仕様を変更する際は注意が必要である。
