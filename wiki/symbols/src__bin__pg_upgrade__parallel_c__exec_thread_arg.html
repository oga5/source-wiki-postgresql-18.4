exec_thread_arg

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_upgrade/parallel.c:29

目的

Windows 環境において、外部コマンドを並列実行するスレッドへ引数を渡すためのデータコンテナです。

シグネチャまたは形状

typedef struct
{
    char *log_file;
    char *opt_log_file;
    char *cmd;
} exec_thread_arg;

入力と出力

  • **入力**: 構造体の各フィールドは、実行対象のコマンド文字列（cmd）およびログ出力先パス（log_file, opt_log_file）を保持します。
  • **出力**: 本構造体自体はデータ保持用であり、win32_exec_prog 関数にポインタとして渡されることで、スレッド内でのコマンド実行に必要なコンテキストを提供します。

前提条件と不変条件

  • pg_upgrade の並列実行モードにおいて、Windows 環境でのみ使用されます。
  • 実行時に各フィールドが適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

  • parallel.c 内でジョブが投入される際、pg_malloc0 を用いてメモリが確保され、各フィールドに値が格納されます。
  • Windows のスレッド生成時に、この構造体へのポインタがスレッド関数へ渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/parallel.c 内の並列ジョブ管理ロジック（parallel_exec_prog 等）が、ジョブ投入時に本構造体をインスタンス化します。
  • **呼び出し先**: win32_exec_prog(exec_thread_arg *args) 関数が、引数として本構造体を受け取り、コマンドを実行します。

関連データ構造

  • exec_thread_args: 本構造体へのポインタを保持する配列（exec_thread_arg **）。現在実行中のジョブを管理するために使用されます。

失敗モードと境界条件

  • メモリ確保失敗: pg_malloc0 が失敗した場合、プロセスは停止します。
  • Windows のスレッド数制限: user_opts.jobs が MAXIMUM_WAIT_OBJECTS（64）を超えないよう設計されています。

保守担当者向けメモ

  • 本構造体は Windows 特有の実装（#ifdef WIN32）に依存しています。
  • 構造体のフィールドを変更する場合は、win32_exec_prog 内での参照箇所も併せて更新してください。
