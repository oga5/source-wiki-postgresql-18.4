ParallelSlot

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/parallel.h:52

目的

並列処理における個別のワーカープロセスまたはスレッドの状態を管理するための構造体です。

シグネチャまたは形状

不透明ポインタ（opaque struct）として定義されており、具体的なメンバ変数は parallel.h 上では公開されていません。

入力と出力

本構造体は ParallelState に保持される配列の一部として管理されます。関数 GetMyPSlot 等を通じて、現在の実行コンテキストに対応するスロットが取得されます。

前提条件と不変条件

  • parallel.c 内でのみ内部構造が参照可能です。
  • ワーカープロセスは、リーダーがカタログを読み込んだ後にフォークされるため、カタログ情報へのアクセス権を継承します。

アルゴリズムまたは振る舞い

  • pg_dump および pg_restore における並列ジョブの実行状態を保持します。
  • ワーカープロセスやスレッドが自身の実行状況を追跡し、リーダープロセスとの同期を行うために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/parallel.c 内の各関数（GetMyPSlot, RunWorker 等）および src/bin/pg_amcheck/pg_amcheck.c の run_command 等から参照されています。
  • **呼び出し先**: 構造体内部の具体的な関数ポインタやメソッドについては、提供されたソース範囲内では詳細不明です。

関連データ構造

  • ParallelState: 本構造体を保持する上位の管理構造体。
  • ArchiveHandle: ワーカーが操作するアーカイブのハンドル。

失敗モードと境界条件

  • Windows 環境では MAXIMUM_WAIT_OBJECTS の制限により、並列数は最大 64 に制限されます。これに伴い、ParallelSlot の配列サイズもこの制限の影響を受けます。

保守担当者向けメモ

  • 本構造体は不透明ポインタとして定義されているため、内部メンバへの直接アクセスは parallel.c 内に限定してください。
  • pg_amcheck からの参照があるため、変更時は pg_dump 以外のツールへの影響も考慮する必要があります。
