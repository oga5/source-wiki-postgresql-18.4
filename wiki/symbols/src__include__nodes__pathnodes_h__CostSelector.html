CostSelector

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/pathnodes.h:36

目的

クエリプランナがパス（Path）のコストを比較する際、評価対象とするコストの種類（開始コストまたは総コスト）を指定するために使用されます。

シグネチャまたは形状

typedef enum CostSelector
{
    STARTUP_COST, 
    TOTAL_COST
} CostSelector;

入力と出力

  • 入力: compare_path_costs 等の関数において、比較基準として渡されます。
  • 出力: なし（列挙型定数）。

前提条件と不変条件

  • STARTUP_COST または TOTAL_COST のいずれかを選択する必要があります。

アルゴリズムまたは振る舞い

  • STARTUP_COST: 最初のタプルを取得するまでのコストを基準に比較を行います。
  • TOTAL_COST: すべてのタプルを処理し終えるまでの総コストを基準に比較を行います。

呼び出し元と呼び出し先

  • 呼び出し元（確認された参照）:
  • src/backend/optimizer/path/pathkeys.c
  • src/backend/optimizer/util/pathnode.c
  • src/include/optimizer/pathnode.h
  • src/include/optimizer/paths.h
  • 呼び出し先: なし（列挙型定義のため）。

関連データ構造

  • Path: プランナが生成する実行計画の候補。compare_path_costs を通じて本列挙型による比較対象となります。

失敗モードと境界条件

  • 定義外の値をキャストして渡した場合の挙動は未定義です。

保守担当者向けメモ

  • 本列挙型はプランナのコスト評価ロジックの根幹に関わるため、新しいコスト指標を追加する場合は、関連する比較関数（compare_path_costs 等）の更新が必須です。
