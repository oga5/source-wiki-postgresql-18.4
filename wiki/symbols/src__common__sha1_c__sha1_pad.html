sha1_pad

定義

  • 種別: function
  • 定義箇所: src/common/sha1.c:232

目的

SHA-1 ハッシュ計算の最終段階において、メッセージデータにパディングを適用し、メッセージ長を付与してブロックを完成させる。

シグネチャまたは形状

static void sha1_pad(pg_sha1_ctx *ctx)

入力と出力

  • **入力**: pg_sha1_ctx *ctx（SHA-1 コンテキスト構造体）。内部のバッファ状態およびメッセージ長情報を使用する。
  • **出力**: なし（副作用として ctx 内のバッファおよび内部状態を更新する）。

前提条件と不変条件

  • ctx は初期化済みであること。
  • PUTPAD マクロおよび sha1_step 関数が利用可能であること。
  • COUNT は現在のバッファ内のデータ長を保持していること。

アルゴリズムまたは振る舞い

1. 0x80（ビット 1 とそれに続く 0）をパディングの開始として追加する。 2. 現在のバッファ残量を確認し、メッセージ長（8バイト）を格納するスペースが不足している場合は、ゼロ埋めして sha1_step を実行し、ブロックをフラッシュする。 3. 残りの領域をゼロで埋め、最後にメッセージ長をビッグエンディアン形式でバッファの末尾に書き込む。 4. エンディアン環境（WORDS_BIGENDIAN）に応じて、メッセージ長のバイト順序を適切に配置する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/sha1.c 内の pg_sha1_final から呼び出される。
  • **呼び出し先**: sha1_step（ブロック処理用）、および PUTPAD マクロ（内部ヘルパー）。

関連データ構造

  • pg_sha1_ctx: SHA-1 計算の状態を保持する構造体（sha1_int.h で定義）。

失敗モードと境界条件

  • バッファが 64 バイト境界に近い場合、パディング処理中に sha1_step が追加で呼び出され、ブロックが完了する。
  • メッセージ長の書き込み時に、プラットフォームのエンディアン設定が正しく反映されない場合、ハッシュ値が不整合となる。

保守担当者向けメモ

  • 本関数は RFC 3174 に準拠している。
  • PUTPAD マクロは sha1_step を内部で呼び出す可能性があるため、呼び出し時のコンテキスト状態に注意すること。
  • エンディアン変換ロジックは WORDS_BIGENDIAN マクロに依存しているため、移植性を考慮する際はここを確認すること。
