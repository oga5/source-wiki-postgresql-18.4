mbuf_append

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/mbuf.c:93

目的

MBuf 構造体で管理されるメモリバッファの末尾に、指定されたバイト列を追加します。

シグネチャまたは形状

int mbuf_append(MBuf *dst, const uint8 *buf, int len);

入力と出力

  • **入力**:
  • dst: 追加先となる MBuf 構造体へのポインタ。
  • buf: 追加するデータを含むバイト配列。
  • len: 追加するデータの長さ（バイト単位）。
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時はエラーコード（PXE_BUG）を返します。

前提条件と不変条件

  • dst は有効な MBuf 構造体を指している必要があります。
  • dst->no_write が真の場合、書き込みは拒否されます。

アルゴリズムまたは振る舞い

1. dst->no_write フラグを確認し、真であれば px_debug でログを出力して PXE_BUG を返します。 2. prepare_room(dst, len) を呼び出し、バッファに十分な空き容量を確保します（必要に応じてメモリの再割り当てが行われます）。 3. memcpy を使用して、dst->data_end が指す位置に buf の内容をコピーします。 4. dst->data_end ポインタをコピーしたバイト数分進めます。 5. 正常終了として 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/mbuf.c (内部)
  • contrib/pgcrypto/pgp-decrypt.c
  • contrib/pgcrypto/pgp-pgsql.c
  • **呼び出し先**:
  • prepare_room (同ファイル内)
  • memcpy (標準ライブラリ)
  • px_debug (共通ユーティリティ)

関連データ構造

  • MBuf: データを保持する可変長バッファ構造体。

失敗モードと境界条件

  • dst->no_write が設定されている場合、書き込みは行われず PXE_BUG が返されます。
  • prepare_room 内でメモリ確保に失敗した場合の挙動は、当該関数の実装に依存します。

保守担当者向けメモ

  • 本関数は palloc を使用する環境下で動作するため、メモリ不足時は PostgreSQL のエラーハンドリング機構によりアボートする可能性があります。
  • dst->data_end の更新は prepare_room によるバッファ拡張後に行われるため、メモリ境界外への書き込みは防止されています。
