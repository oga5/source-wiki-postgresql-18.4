prepare_vacuum_command

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/vacuumdb.c:1141

目的

ユーザーが指定した vacuumingOptions に基づき、接続先サーバーのバージョンを考慮して、実行対象テーブルに対する適切な VACUUM または ANALYZE SQL コマンドを構築します。

シグネチャまたは形状

static void prepare_vacuum_command(PQExpBuffer sql, int serverVersion, vacuumingOptions *vacopts, const char *table);

入力と出力

  • **入力**:
  • sql: コマンド文字列を格納する PQExpBuffer（出力先）。
  • serverVersion: サーバーのバージョン番号（整数）。
  • vacopts: 実行オプションを保持する vacuumingOptions 構造体。
  • table: 対象テーブル名。
  • **出力**: なし（sql バッファが更新されます）。

前提条件と不変条件

  • sql は事前に初期化されている必要があります。
  • serverVersion に応じて、サポートされていないオプションが指定された場合に Assert が発生する可能性があります。
  • vacopts->no_index_cleanup と vacopts->force_index_cleanup は同時に真にはなりません。

アルゴリズムまたは振る舞い

1. sql バッファをリセットします。 2. vacopts->analyze_only が真であれば ANALYZE を、偽であれば VACUUM をコマンドの先頭に配置します。 3. サーバーバージョンが各オプションの導入バージョンを満たしているか確認し、括弧付き構文（(...)）または従来の構文でオプションを順次追加します。 4. 最後にターゲットとなる table 名をコマンドの末尾に付与し、セミコロンで閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/scripts/vacuumdb.c 内の vacuum_one_database 関数から呼び出されます。
  • **呼び出し先**: resetPQExpBuffer, appendPQExpBufferStr, appendPQExpBuffer, appendPQExpBufferChar などの libpq 関連関数を呼び出します。

関連データ構造

  • PQExpBuffer: 文字列構築用のバッファ構造体。
  • vacuumingOptions: ユーザー指定のオプションを保持する構造体。

失敗モードと境界条件

  • サーバーバージョンが古い場合、新しいオプション（例: BUFFER_USAGE_LIMIT や SKIP_LOCKED）を指定すると Assert によりプログラムが停止します。
  • ターゲットテーブル名が正しくエスケープされていない場合、生成された SQL が構文エラーを引き起こす可能性があります。

保守担当者向けメモ

  • 本関数はサーバーのバージョンアップに伴う新機能追加のたびに、対応する Assert と条件分岐の更新が必要です。
  • serverVersion の判定ロジックは、PostgreSQL のバージョン体系（例: 160000）に依存しています。
