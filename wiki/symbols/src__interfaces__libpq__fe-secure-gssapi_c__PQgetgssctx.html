PQgetgssctx

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure-gssapi.c:758

目的

PGconn オブジェクトから、現在保持されている GSSAPI セキュリティコンテキスト（gctx）を取得します。

シグネチャまたは形状

void *PQgetgssctx(PGconn *conn);

入力と出力

  • **入力**: PGconn *conn （PostgreSQL 接続オブジェクトへのポインタ）
  • **出力**: void * （GSSAPI コンテキストへのポインタ。コンテキストが存在しない場合や conn が NULL の場合は NULL）

前提条件と不変条件

  • conn が NULL の場合、関数は安全に NULL を返します。
  • 戻り値は PGconn 構造体内部の gctx メンバを直接参照します。

アルゴリズムまたは振る舞い

1. 引数 conn が NULL かどうかを判定し、NULL であれば直ちに NULL を返します。 2. conn が有効な場合、その構造体内に保持されている gctx メンバの値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq/fe-secure.c:483
  • **呼び出し先**:
  • 本関数は他の関数を呼び出しません。

関連データ構造

  • PGconn: libpq の接続状態を保持する構造体（src/interfaces/libpq/libpq-int.h で定義）。本関数はこの構造体内の gctx フィールドにアクセスします。

失敗モードと境界条件

  • conn が NULL の場合、エラーにはならず NULL を返します。

保守担当者向けメモ

  • 本関数は公開 API (libpq-fe.h) として定義されており、外部からの GSSAPI コンテキストへのアクセスを提供します。
  • コンテキストのライフサイクル管理は fe-secure-gssapi.c 内の他の関数（pqsecure_open_gss 等）に依存しており、本関数は単なるゲッターとして機能します。
