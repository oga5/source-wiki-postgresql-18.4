PartitionDirectoryEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/partitioning/partdesc.c:42

目的

PartitionDirectory 内で管理される、個別のリレーションに対するパーティション記述子とその関連情報を保持するためのエントリ構造体です。

シグネチャまたは形状

typedef struct PartitionDirectoryEntry
{
	Oid			reloid;    /* リレーションのOID */
	Relation	rel;       /* リレーションのキャッシュエントリ */
	PartitionDesc pd;      /* パーティション記述子 */
} PartitionDirectoryEntry;

入力と出力

本構造体はデータ保持を目的としており、関数としての入力・出力はありません。

前提条件と不変条件

  • PartitionDirectory 内のハッシュテーブルエントリとして使用されます。
  • reloid は対象リレーションを一意に識別するキーとして機能します。

アルゴリズムまたは振る舞い

本構造体自体はデータコンテナであり、特定のアルゴリズムは持ちません。PartitionDirectory のハッシュテーブル構築時（sizeof(PartitionDirectoryEntry)）や、ディレクトリ内の検索・参照時に利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/partitioning/partdesc.c 内のハッシュテーブル初期化処理（ctl.entrysize の設定）。
  • PartitionDirectoryLookup 等のディレクトリ操作関数内で、エントリの取得や参照に使用されます。
  • **呼び出し先**:
  • 構造体メンバとして Relation および PartitionDesc を保持しており、これらを通じてリレーションやパーティション情報にアクセスします。

関連データ構造

  • PartitionDirectoryData: 本構造体を要素として管理するハッシュテーブルを含むコンテナ。
  • PartitionDesc: パーティションの階層構造や境界情報を保持する構造体。
  • Relation: PostgreSQLのリレーションキャッシュエントリ。

失敗モードと境界条件

  • 構造体自体に失敗モードはありませんが、保持する rel や pd が無効な場合、それらを参照する処理でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は PartitionDirectory の内部実装詳細です。PartitionDirectory を介さずに直接操作することは避けてください。
  • メモリ管理は PartitionDirectory のライフサイクルに依存します。
