create_s

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/utilities.c:5

目的

Snowball ステミングアルゴリズムで使用する、文字列バッファ（symbol 型）の動的メモリ確保と初期化を行います。

シグネチャまたは形状

extern symbol * create_s(void);

入力と出力

  • 入力: なし
  • 出力: 確保された symbol 構造体へのポインタ。メモリ確保に失敗した場合は NULL を返します。

前提条件と不変条件

  • HEAD および CREATE_SIZE 定数が定義されている必要があります。
  • 戻り値のポインタは、CAPACITY が CREATE_SIZE に設定され、現在のサイズが 0 に初期化された状態を保証します。

アルゴリズムまたは振る舞い

1. HEAD（ヘッダーサイズ）と CREATE_SIZE に基づくバッファサイズを計算し、malloc でメモリを確保します。 2. 確保したメモリが NULL であれば即座に NULL を返します。 3. 確保したメモリ領域の先頭から HEAD 分オフセットした位置を symbol 型のポインタとしてキャストします。 4. CAPACITY マクロを使用してバッファ容量を CREATE_SIZE に設定します。 5. SET_SIZE マクロを使用して、現在の文字列長を 0 に初期化します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/snowball/libstemmer/api.c (複数箇所)
  • src/backend/snowball/libstemmer/utilities.c (内部)
  • 呼び出し先:
  • malloc (標準ライブラリ)

関連データ構造

  • symbol: ステミング対象の文字列を保持する型。
  • SN_env: ステミング環境を保持する構造体（本関数で生成されたバッファがこの構造体に格納されます）。

失敗モードと境界条件

  • malloc がメモリ不足により NULL を返した場合、関数は安全に NULL を返します。

保守担当者向けメモ

  • 本関数で確保されたメモリは、対応する lose_s 関数を使用して解放する必要があります。
  • symbol 型の定義や HEAD のサイズについては header.h を参照してください。
