getSide

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgkdtreeproc.c:40

目的

k-d 木における空間分割のため、指定された座標値と Point 構造体の特定の座標（x または y）を比較し、その相対的な位置関係を判定する。

シグネチャまたは形状

static int getSide(double coord, bool isX, Point *tst);

入力と出力

  • **入力**:
  • coord: 比較対象となる基準座標値。
  • isX: true の場合は tst->x を、false の場合は tst->y を比較対象として選択するフラグ。
  • tst: 比較対象の座標値を持つ Point 構造体へのポインタ。
  • **出力**:
  • 0: coord と tst の指定座標が等しい場合。
  • 1: coord が tst の指定座標より大きい場合。
  • -1: coord が tst の指定座標より小さい場合。

前提条件と不変条件

  • tst は有効な Point 構造体を指している必要がある。
  • 浮動小数点数としての比較が行われるため、IEEE 754 準拠の比較規則に従う。

アルゴリズムまたは振る舞い

1. isX フラグに基づき、tst から比較対象の座標値（tstcoord）を抽出する。 2. coord と tstcoord を比較する。 3. 比較結果に応じて、0（一致）、1（coord が大きい）、-1（coord が小さい）のいずれかを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgkdtreeproc.c 内の挿入処理ロジック（spg_kd_choose 等の文脈）から呼び出される。
  • **呼び出し先**:
  • 本関数は直接的な外部関数呼び出しを行わない。

関連データ構造

  • Point: utils/geo_decls.h で定義される、x と y 座標を持つ構造体。

失敗モードと境界条件

  • tst が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • NaN（非数）が含まれる座標値が渡された場合の挙動は、標準的な浮動小数点比較規則に依存する。

保守担当者向けメモ

  • 本関数は k-d 木のノード振り分けにおける基本的な比較器である。
  • 戻り値の 1 または -1 は、呼び出し側でノードの左右（あるいは上下）の分岐先を決定するために利用される。
