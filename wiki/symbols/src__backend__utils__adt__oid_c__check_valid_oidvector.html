check_valid_oidvector

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oid.c:117

目的

oidvector 型のデータ構造が、呼び出し側が期待するレイアウト（1次元、NULLなし）を満たしているかを検証します。

シグネチャまたは形状

void check_valid_oidvector(const oidvector *oidArray)

入力と出力

  • **入力**: oidvector *oidArray （検証対象の oidvector 構造体へのポインタ）
  • **出力**: なし（戻り値は void）
  • **副作用**: 検証に失敗した場合、ereport(ERROR, ...) を呼び出し、トランザクションを中断させます。

前提条件と不変条件

  • oidvector は ndim == 1 であること。
  • dataoffset == 0 であること（NULL値を含まないこと）。
  • elemtype が OIDOID であること。
  • インデックスの下限値については制約を課しません。

アルゴリズムまたは振る舞い

1. 引数として受け取った oidArray の各フィールドをチェックします。 2. ndim が 1 ではない、dataoffset が 0 ではない、または elemtype が OIDOID ではない場合、ERRCODE_DATATYPE_MISMATCH エラーを発生させます。 3. 条件をすべて満たす場合は正常に終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/hash/hashfunc.c
  • src/backend/access/nbtree/nbtcompare.c
  • src/backend/utils/adt/format_type.c
  • src/backend/utils/adt/oid.c
  • **呼び出し先**:
  • ereport （エラー発生時）

関連データ構造

  • oidvector: Oid の固定長配列を表現する構造体。

失敗モードと境界条件

  • ndim が 1 以外の場合、または dataoffset が 0 以外（NULLを含む）の場合、データ型不一致としてエラーとなります。
  • elemtype が OIDOID ではない場合もエラーとなります。

保守担当者向けメモ

  • 本関数は、oidvector が汎用的な配列型からキャストされて渡される可能性があるため、厳格な検証を行うために使用されます。
  • array_send など、配列の内部レイアウトを直接扱わない関数では本チェックを省略することがあります。
