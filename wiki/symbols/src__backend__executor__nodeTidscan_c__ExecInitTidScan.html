ExecInitTidScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTidscan.c:498

目的

TidScan ノードの実行状態（TidScanState）を初期化し、クエリ実行に必要なリレーションのオープンや式コンテキストの構築を行う。

シグネチャまたは形状

TidScanState *ExecInitTidScan(TidScan *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の TidScan プランノード。
  • estate: クエリ実行全体の実行状態。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの TidScanState 構造体へのポインタ。

前提条件と不変条件

  • node は有効な TidScan プランノードであること。
  • estate は初期化済みであること。
  • 戻り値の TidScanState 内の tss_TidList は NULL、tss_TidPtr は -1 に初期化され、TID リストが未計算であることを示す。

アルゴリズムまたは振る舞い

1. TidScanState 構造体を割り当て、ExecTidScan を実行関数として登録する。 2. ExecAssignExprContext を呼び出し、式評価用のコンテキストを作成する。 3. ExecOpenScanRelation を使用して対象リレーションをオープンし、スキャン記述子を初期化する。 4. ExecInitScanTupleSlot を呼び出し、タプルスロットを準備する。 5. ExecInitResultTypeTL および ExecAssignScanProjectionInfo により、結果型と射影情報を設定する。 6. ExecInitQual でクオリファイア（qual）を初期化する。 7. TidExprListCreate を呼び出し、TID 抽出用の式リストを構築する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内の ExecInitNode 等から呼び出される（直接の呼び出し元は execProcnode.c:240）。
  • **呼び出し先**: makeNode, ExecAssignExprContext, ExecOpenScanRelation, ExecInitScanTupleSlot, RelationGetDescr, table_slot_callbacks, ExecInitResultTypeTL, ExecAssignScanProjectionInfo, ExecInitQual, TidExprListCreate。

関連データ構造

  • TidScan: プランノード。
  • TidScanState: 実行状態を保持するノード。
  • EState: クエリ実行状態。
  • Relation: スキャン対象のリレーション。

失敗モードと境界条件

  • リレーションのオープン失敗やメモリ不足時には、PostgreSQL の標準的なエラーハンドリングにより例外が発生する。
  • node->scan.scanrelid が無効な場合、ExecOpenScanRelation が失敗する可能性がある。

保守担当者向けメモ

  • 本関数は TidScan ノードのセットアップのみを行い、実際の TID 抽出やスキャンは TidNext（初回実行時）に遅延評価される。
  • tss_TidPtr を -1 に初期化することで、初回実行時に TidListEval がトリガーされる仕組みになっている。
