binaryheap_first

定義

  • 種別: function
  • 定義箇所: src/common/binaryheap.c:176

目的

バイナリヒープのルート要素（ヒープの先頭）を取得します。

シグネチャまたは形状

bh_node_type binaryheap_first(binaryheap *heap)

入力と出力

  • **入力**: heap (バイナリヒープ構造体へのポインタ)
  • **出力**: ヒープのルートに位置する bh_node_type 型の要素を返します。

前提条件と不変条件

  • ヒープが空であってはなりません（!binaryheap_empty(heap)）。
  • ヒープは有効なヒーププロパティを保持している必要があります（heap->bh_has_heap_property）。
  • 上記条件は Assert により実行時に検証されます。

アルゴリズムまたは振る舞い

ヒープの内部配列 bh_nodes のインデックス 0 に格納されている要素を直接参照して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeGatherMerge.c
  • src/backend/executor/nodeMergeAppend.c
  • src/backend/postmaster/pgarch.c
  • src/backend/replication/logical/reorderbuffer.c
  • src/backend/storage/buffer/bufmgr.c
  • **呼び出し先**:
  • 本関数は内部で binaryheap_empty マクロ（または関数）を呼び出しますが、それ以外の関数呼び出しは行いません。

関連データ構造

  • binaryheap: ヒープを管理する構造体。
  • bh_node_type: ヒープに格納される要素の型（Datum または void *）。

失敗モードと境界条件

  • ヒープが空の状態で呼び出した場合、アサーション違反によりプログラムが停止する可能性があります。

保守担当者向けメモ

  • 本関数はヒープの順序を維持するための再構築（sift_down 等）は行いません。単なる参照用です。
  • 戻り値の型 bh_node_type は、コンパイル環境（バックエンドかフロントエンドか）によって定義が異なるため注意してください。
