freecolor

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_color.c:257

目的

カラーマップ内の指定された色（color）を解放し、再利用可能な状態にする。

シグネチャまたは形状

static void freecolor(struct colormap *cm, color co)

入力と出力

  • cm: 操作対象のカラーマップ構造体へのポインタ。
  • co: 解放する色（color 型）。
  • 出力: なし（副作用としてカラーマップの状態を更新）。

前提条件と不変条件

  • co >= 0 であること。
  • co が WHITE（デフォルト色）ではないこと。
  • 解放対象の色 co に関連付けられた NFA エッジ（arcs）が存在しないこと。
  • co がサブカラー（sub）を持たないこと。
  • co に文字（nschrs, nuchrs）が割り当てられていないこと。

アルゴリズムまたは振る舞い

1. 指定された色 co の記述子（colordesc）を取得し、フラグを FREECOL に設定する。 2. co がカラーマップの最大色番号（cm->max）と一致する場合:

  • cm->max をデクリメントし、未使用の色をスキップする。
  • フリーリストを走査し、cm->max を超える色番号をリストから除外する。

    • 3. co が最大色番号でない場合:

  • co を現在のフリーリストの先頭に追加し、cm->free を更新する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/regex/regc_color.c:958
  • 呼び出し先:
  • 直接的な関数呼び出しは含まれていないが、cm->cd（カラーマップ記述子配列）および cm->free（フリーリスト）を操作する。

関連データ構造

  • colormap: カラーマップ全体を管理する構造体。
  • colordesc: 各色の状態（エッジ、サブカラー、フラグ等）を保持する構造体。

失敗モードと境界条件

  • WHITE 色を解放しようとした場合は即座にリターンする。
  • 前提条件（arcs == NULL 等）が満たされない場合、assert により異常終了する。

保守担当者向けメモ

  • 本関数は、不要になった色をフリーリストに繋ぎ直すことでメモリの再利用を管理する。
  • cm->max を縮小する際のフリーリスト走査は、リストの整合性を維持するために重要である。
