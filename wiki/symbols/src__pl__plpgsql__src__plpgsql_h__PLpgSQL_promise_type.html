PLpgSQL_promise_type

定義

  • 種別: enum
  • 定義箇所: src/pl/plpgsql/src/plpgsql.h:74

目的

PL/pgSQL において、トリガー実行時に提供される特殊な変数（TG_* 変数）の遅延評価または要求状態を識別するために使用されます。

シグネチャまたは形状

typedef enum PLpgSQL_promise_type
{
    PLPGSQL_PROMISE_NONE = 0,
    PLPGSQL_PROMISE_TG_NAME,
    PLPGSQL_PROMISE_TG_WHEN,
    PLPGSQL_PROMISE_TG_LEVEL,
    PLPGSQL_PROMISE_TG_OP,
    PLPGSQL_PROMISE_TG_RELID,
    PLPGSQL_PROMISE_TG_TABLE_NAME,
    PLPGSQL_PROMISE_TG_TABLE_SCHEMA,
    PLPGSQL_PROMISE_TG_NARGS,
    PLPGSQL_PROMISE_TG_ARGV,
    PLPGSQL_PROMISE_TG_EVENT,
    PLPGSQL_PROMISE_TG_TAG,
} PLpgSQL_promise_type;

入力と出力

  • 入力: なし（列挙型定義）。
  • 出力: トリガー関連のメタデータ（名前、タイミング、操作種別、引数など）を特定するための識別子。

前提条件と不変条件

  • PLPGSQL_PROMISE_NONE は、プロミスが設定されていないか、既に充足済みであることを示します。
  • 各列挙値は、トリガー実行コンテキストにおいて特定のシステム情報に対応します。

アルゴリズムまたは振る舞い

PL/pgSQL の実行エンジンが、トリガー関数内で参照される TG_ 変数を解決する際、この列挙型を使用して必要な情報の種類を特定します。これにより、実行時に必要な情報のみを動的に取得・提供する仕組みを支えています。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpgsql/src/plpgsql.h:365 にて、構造体メンバー promise の型として参照されています。
  • 呼び出し先: 提供されたソース情報からは、具体的な関数呼び出し関係は特定できません。

関連データ構造

  • PLpgSQL_expr: 式の評価に関連する構造体であり、この列挙型を保持するフィールドを持つ可能性があります。

失敗モードと境界条件

  • PLPGSQL_PROMISE_NONE 以外の値が設定されている場合、対応するトリガーコンテキストが有効でないと、実行時に値の取得に失敗する可能性があります。

保守担当者向けメモ

  • 本列挙型はトリガー固有の変数を扱うため、トリガー関連のロジック（pl_exec.c 等）を修正する際は、この列挙値の追加や変更が影響範囲に含まれるか確認してください。
