find_wordentry

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsrank.c:86

目的

TSVector 内から、指定された TSQuery のオペランド（QueryOperand）に一致する単語エントリを検索し、その開始位置と一致数を返します。

シグネチャまたは形状

WordEntry *find_wordentry(TSVector t, TSQuery q, QueryOperand *item, int32 *nitem)

入力と出力

  • **入力**:
  • t: 検索対象の TSVector。
  • q: 検索条件を含む TSQuery。
  • item: 検索対象の QueryOperand。
  • nitem: 一致したエントリ数を格納するポインタ（出力用）。
  • **出力**:
  • 一致した最初の WordEntry へのポインタ。一致しない場合は NULL を返します。

前提条件と不変条件

  • TSVector はソート済みである必要があります。
  • ループ不変条件: StopLow <= item < StopHigh。

アルゴリズムまたは振る舞い

1. **二分探索**: TSVector 内の WordEntry 配列に対して、WordECompareQueryItem を用いて指定された item を二分探索します。 2. **プレフィックス検索**: item->prefix が真の場合、二分探索で見つかった位置から後続の要素を順次走査し、プレフィックスとして一致するすべてのエントリ数をカウントします。 3. **戻り値**: 一致したエントリが存在する場合、その最初の要素へのポインタを返し、nitem にカウント数を格納します。存在しない場合は NULL を返します。

関連データ構造

  • TSVector: 全文検索用の文書表現。
  • TSQuery: 全文検索用のクエリ表現。
  • WordEntry: TSVector 内の各単語のメタデータ。
  • QueryOperand: TSQuery 内の単語ノード。

失敗モードと境界条件

  • 一致する単語がない場合、nitem は 0 となり、戻り値は NULL となります。
  • プレフィックス検索時に TSVector の末尾に達した場合、走査を終了します。

保守担当者向けメモ

  • WordECompareQueryItem の第5引数（prefix フラグ）の切り替えにより、完全一致とプレフィックス一致を制御しています。
  • プレフィックス検索は線形走査であるため、同一プレフィックスを持つ単語が極端に多い場合、パフォーマンスに影響を与える可能性があります。
