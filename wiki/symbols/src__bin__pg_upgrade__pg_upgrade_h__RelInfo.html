RelInfo

定義

  • **種別**: struct
  • **定義箇所**: src/bin/pg_upgrade/pg_upgrade.h:137

目的

PostgreSQL のアップグレード処理において、個別のリレーション（テーブル、インデックス、TOASTテーブル等）のメタデータを保持し、旧クラスタと新クラスタ間でのマッピングや検証を行うために使用されます。

シグネチャまたは形状

typedef struct {
    char       *nspname;        /* 名前空間名 */
    char       *relname;        /* リレーション名 */
    Oid         reloid;         /* リレーションOID */
    RelFileNumber relfilenumber; /* リレーションファイル番号 */
    Oid         indtable;       /* インデックスの場合、対象テーブルのOID (それ以外は0) */
    Oid         toastheap;      /* TOASTテーブルの場合、ベーステーブルのOID (それ以外は0) */
    char       *tablespace;     /* テーブルスペースパス ("" はクラスタデフォルト) */
    bool        nsp_alloc;      /* nspnameのメモリ解放要否 */
    bool        tblsp_alloc;    /* tablespaceのメモリ解放要否 */
} RelInfo;

入力と出力

本構造体はデータ保持用であり、関数ではありません。info.c 内の関数によって、システムカタログから取得された情報が各フィールドに格納されます。

前提条件と不変条件

  • nspname および relname は、クライアント環境での NAMEDATALEN の差異を考慮し、固定長配列ではなく動的確保された文字列として保持されます。
  • RelInfo の配列は、OID順にソートされていることが期待されます（info.c の処理ロジックに基づく）。

アルゴリズムまたは振る舞い

  • pg_upgrade の情報収集フェーズにおいて、各データベース内のリレーション情報を走査し、RelInfo の配列として構築されます。
  • nsp_alloc および tblsp_alloc フラグにより、動的に確保されたメモリのライフサイクル管理を制御します。

呼び出し元と呼び出し先

  • **主な呼び出し元**: src/bin/pg_upgrade/info.c 内の関数群（リレーション情報の収集、比較、不一致レポート生成など）。
  • **呼び出し先**: 本構造体自体はデータ構造であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • RelInfoArr: RelInfo の配列を管理する構造体。
  • DbInfo: RelInfoArr を保持し、データベース単位のリレーション情報を管理します。

失敗モードと境界条件

  • メモリ確保失敗: pg_malloc を使用して構築されるため、システムメモリ不足時に失敗します。
  • 識別子の不一致: nspname や relname が適切に取得できない場合、アップグレードの整合性チェックに影響を及ぼします。

保守担当者向けメモ

  • NAMEDATALEN に依存しない設計となっているため、異なるメジャーバージョン間でのアップグレード時にも安全に動作するよう配慮されています。
  • RelInfo の配列を操作する際は、OID順序が維持されているか確認してください。
