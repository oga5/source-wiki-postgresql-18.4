brin_initialize_empty_new_buffer

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_pageops.c:883

目的

BRINインデックスの新規ページを初期化し、バッファ管理および空き領域マップ（FSM）への登録を行う。

シグネチャまたは形状

static void brin_initialize_empty_new_buffer(Relation idxrel, Buffer buffer)

入力と出力

  • idxrel: 対象のBRINインデックスリレーション。
  • buffer: 初期化対象のバッファ。
  • 戻り値: なし。

前提条件と不変条件

  • buffer は既に呼び出し元で確保されている必要がある。
  • ページ操作はクリティカルセクション内で実行される。

アルゴリズムまたは振る舞い

1. **初期化**: brin_page_init を呼び出し、ページタイプを BRIN_PAGETYPE_REGULAR として初期化する。 2. **バッファ更新**: MarkBufferDirty を呼び出し、バッファをダーティ状態にする。 3. **WAL記録**: リレーションがWALを必要とする場合、log_newpage_buffer を使用してページ全体をWALに記録する。 4. **FSM更新**: RecordPageWithFreeSpace を呼び出し、初期化されたページの空き領域情報をFSMに登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin_pageops.c 内の複数の箇所（brin_getinsertbuffer 等のページ確保処理に関連する関数群）から呼び出される。
  • **呼び出し先**: brin_page_init, MarkBufferDirty, RelationNeedsWAL, log_newpage_buffer, RecordPageWithFreeSpace, br_page_get_freespace。

関連データ構造

  • Relation: インデックスリレーション情報。
  • Buffer: バッファマネージャのバッファ記述子。
  • Page: ページレイアウト構造。

失敗モードと境界条件

  • WAL記録が必要な環境でログ書き込みに失敗した場合、クリティカルセクションによりシステムがパニックする可能性がある。
  • FSMへの登録はWAL記録されないが、これはクラッシュ後に VACUUM がインデックスをスキャンしてFSMを再構築することで整合性が保たれる設計となっている。

保守担当者向けメモ

  • 本関数はページを「空」として初期化する低レベルな操作である。
  • FSMの更新はWALログに含まれないため、クラッシュリカバリ後のFSM状態は VACUUM による修正に依存している点に注意すること。
