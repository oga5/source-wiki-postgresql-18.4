TransactionIdSetStatusBit

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/clog.c:660

目的

指定されたトランザクションID（XID）のコミット状態を CLOG（Commit Log）ページ上のビットとして永続化し、必要に応じてグループ LSN を更新する。

シグネチャまたは形状

static void TransactionIdSetStatusBit(TransactionId xid, XidStatus status, XLogRecPtr lsn, int slotno)

入力と出力

  • xid: 状態を更新する対象のトランザクションID。
  • status: 設定する状態（TRANSACTION_STATUS_IN_PROGRESS, COMMITTED, ABORTED, SUB_COMMITTED）。
  • lsn: この状態変化に関連するログシーケンス番号。
  • slotno: 対象ページが格納されている SLRU バッファスロット番号。
  • 出力: なし（副作用として共有メモリ上の CLOG ページを更新）。

前提条件と不変条件

  • slotno に対応する SLRU ページが xid のページと一致していること。
  • 呼び出し元が、対象の SLRU ページに対する排他ロック（LW_EXCLUSIVE）を保持していること。
  • リカバリ中を除き、状態遷移は 0 またはサブコミット状態からターゲット状態へ向かうこと。

アルゴリズムまたは振る舞い

1. TransactionIdToByte および TransactionIdToBIndex を用いて、XID に対応するバイト位置とビットシフト量を算出する。 2. 現在のビット値を読み取り、リカバリ中の特殊ケース（サブコミットからコミットへの遷移）を判定して早期リターンする。 3. ビットマスク操作により、対象の 2 ビットを新しい status で上書きする。 4. lsn が有効な場合、GetLSNIndex で特定したスロットのグループ LSN を、提供された lsn と比較し、大きい値で更新する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/clog.c 内の TransactionIdSetTreeStatus および関連する内部関数。
  • 呼び出し先: TransactionIdToByte, TransactionIdToBIndex, SimpleLruGetBankLock, GetLSNIndex, XLogRecPtrIsInvalid。

関連データ構造

  • XactCtl: CLOG の共有メモリ状態を管理する SlruCtlData 構造体。
  • CLOG_BITS_PER_XACT: 1 トランザクションあたりのビット数（2ビット）。

失敗モードと境界条件

  • 排他ロックを保持していない場合、Assert により異常終了する。
  • リカバリ中に無効な状態遷移が発生した場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数は CLOG ページへの直接的なビット操作を行う低レイヤー関数である。
  • グループ LSN の更新は、非同期コミットの整合性を保つために重要である。
  • リカバリ中の動作については、InRecovery フラグを用いた条件分岐が含まれており、変更時にはクラッシュリカバリへの影響を考慮すること。
