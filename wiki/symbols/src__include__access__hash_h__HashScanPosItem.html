HashScanPosItem

定義

  • 種別: struct
  • 定義箇所: src/include/access/hash.h:103

目的

ハッシュインデックスのスキャン中に、一致した各項目（タプル）の情報を一時的に保持するために使用されます。

シグネチャまたは形状

typedef struct HashScanPosItem
{
    ItemPointerData heapTid;    /* ヒープ内の参照先タプルのTID */
    OffsetNumber indexOffset;   /* ページ内でのインデックス項目の位置 */
} HashScanPosItem;

入力と出力

本構造体はデータ保持用であり、関数としての入出力は持ちません。スキャン処理において、インデックスページから読み取った情報を格納するコンテナとして機能します。

前提条件と不変条件

  • HashScanPosData 構造体内の items 配列の一部として使用されます。
  • indexOffset は、有効なインデックスページ内のオフセット番号である必要があります。

アルゴリズムまたは振る舞い

ハッシュインデックスのスキャン処理において、インデックスページからタプル情報を抽出する際、各一致項目に対して本構造体のインスタンスが生成・参照されます。具体的には、hashsearch.c や hashutil.c 内で、スキャン状態（HashScanOpaque）が保持する現在の位置情報（currPos）の配列要素としてアクセスされます。

呼び出し元と呼び出し先

  • **参照元**:
  • src/backend/access/hash/hash.c
  • src/backend/access/hash/hashsearch.c
  • src/backend/access/hash/hashutil.c
  • **呼び出し先**: 本構造体はデータ構造であり、関数呼び出しは行いません。

関連データ構造

  • HashScanPosData: 本構造体の配列を保持する親構造体。
  • HashScanOpaqueData: スキャン状態全体を管理する構造体。
  • ItemPointerData: ヒープタプルを指すための標準的なTID構造体。

失敗モードと境界条件

  • HashScanPosData の items 配列のサイズ（MaxIndexTuplesPerPage）を超えるインデックス項目を処理しようとした場合、メモリ破壊や境界外アクセスのリスクがあります。

保守担当者向けメモ

  • 本構造体は HashScanPosData の末尾に配置される配列の要素として定義されています。構造を変更する場合は、スキャン状態のメモリレイアウト全体に影響するため注意してください。
