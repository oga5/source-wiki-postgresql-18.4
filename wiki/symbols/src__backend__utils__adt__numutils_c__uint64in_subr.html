uint64in_subr

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/numutils.c:984

目的

文字列を uint64 型の整数へ変換し、PostgreSQL の型システムに適合したエラーチェックと構文検証を行います。

シグネチャまたは形状

uint64 uint64in_subr(const char *s, char **endloc, const char *typname, Node *escontext)

入力と出力

  • **入力**:
  • s: 解析対象の文字列。
  • endloc: 解析終了位置を格納するポインタ（NULL の場合、文字列末尾まで空白以外の文字がないことを検証する）。
  • typname: エラーメッセージに使用する型名。
  • escontext: エラー報告用コンテキスト。
  • **出力**:
  • 変換された uint64 値。エラー時は escontext を介して報告され、戻り値は 0 となります。

前提条件と不変条件

  • strtou64 を内部的に利用しており、標準的な整数リテラル（10進数、16進数、8進数等）の構文を前提とします。
  • escontext が提供されている場合、エラー発生時に呼び出し元へ制御が戻ります。

アルゴリズムまたは振る舞い

1. strtou64 を呼び出し、文字列を数値へ変換します。 2. errno および endptr を確認し、構文エラー（endptr == s）または範囲外エラー（ERANGE）を判定します。 3. エラーが検出された場合、ereturn マクロを用いて ERRCODE_INVALID_TEXT_REPRESENTATION または ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE を報告します。 4. endloc が指定されている場合は、解析終了位置を格納します。 5. endloc が NULL の場合、数値の後に空白以外の文字が残っていないかを確認し、残っている場合はエラーとします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/xid.c:200
  • **呼び出し先**:
  • strtou64（標準ライブラリ相当の内部関数）
  • ereturn（エラー報告マクロ）
  • isspace（文字判定）

関連データ構造

  • Node *escontext: エラー処理の伝播に使用される構造体。

失敗モードと境界条件

  • **構文エラー**: 文字列が数値として解釈できない場合（endptr == s）、ERRCODE_INVALID_TEXT_REPRESENTATION を返します。
  • **範囲外エラー**: 数値が uint64 の範囲を超える場合、ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE を返します。
  • **末尾の不正文字**: endloc が NULL の際、数値の後に空白以外の文字が存在するとエラーとなります。

保守担当者向けメモ

  • 本関数は strtou64 のラッパーであり、PostgreSQL のエラーハンドリング規約（escontext）を適用する役割を担います。
  • endloc が NULL の場合の厳密な末尾チェックは、入力文字列の完全性を保証するために重要です。
