PGSemaphoreCreate

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32_sema.c:78

目的

Windows環境において、PostgreSQLの同期処理に使用する匿名セマフォを作成し、そのハンドルを返します。

シグネチャまたは形状

PGSemaphore PGSemaphoreCreate(void);

入力と出力

  • **入力**: なし
  • **出力**: 作成されたセマフォを指す PGSemaphore 型のハンドルを返します。

前提条件と不変条件

  • IsUnderPostmaster が false であること（ポストマスター配下では実行不可）。
  • 現在のセマフォ数が上限（maxSems）に達していないこと。

アルゴリズムまたは振る舞い

1. Assert によりポストマスター配下でないことを検証します。 2. セマフォ数が上限を超えていないか確認し、超過時は PANIC を発生させます。 3. SECURITY_ATTRIBUTES を設定し、ハンドルを継承可能（bInheritHandle = TRUE）にします。 4. CreateSemaphore を呼び出し、初期値1、最大値32767の匿名セマフォを作成します。 5. 作成成功時、ハンドルを静的配列 mySemSet に格納し、インデックスをインクリメントします。 6. 作成失敗時は GetLastError を含むエラーメッセージと共に PANIC を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/lmgr/proc.c 等から呼び出されます。
  • **呼び出し先**: Win32 APIの CreateSemaphore および ZeroMemory を使用します。
  • **注記**: posix_sema.c や sysv_sema.c にも同名の関数が存在しますが、これらはプラットフォーム別の実装であり、本関数とは別物です。

関連データ構造

  • mySemSet: プロセス内で確保されたセマフォハンドルを保持する静的配列。
  • PGSemaphore: セマフォハンドルを抽象化した型。

失敗モードと境界条件

  • **セマフォ上限超過**: numSems >= maxSems の場合、PANIC で異常終了します。
  • **OSリソース不足**: CreateSemaphore が失敗した場合、GetLastError を報告して PANIC で異常終了します。

保守担当者向けメモ

  • 本関数はWindows専用の実装です。共有メモリを介さない匿名セマフォを使用しているため、プロセス終了時に ReleaseSemaphores を通じて適切にクリーンアップされる必要があります。
  • bInheritHandle が TRUE に設定されているため、子プロセスへのハンドル継承が有効です。
