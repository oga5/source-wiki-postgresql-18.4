gen_reindex_command

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/reindexdb.c:498

目的

PostgreSQLの REINDEX SQLコマンド文字列を生成し、指定されたバッファに格納する。

シグネチャまたは形状

static void gen_reindex_command(
    PGconn *conn, 
    ReindexType type, 
    const char *name,
    bool echo, 
    bool verbose, 
    bool concurrently,
    const char *tablespace, 
    PQExpBufferData *sql
)

入力と出力

  • **入力**:
  • conn: データベース接続ハンドル。
  • type: 再構築対象の種類（ReindexType 列挙型）。
  • name: 対象オブジェクト名。
  • echo, verbose, concurrently: SQL生成オプション。
  • tablespace: 指定されたテーブルスペース名（NULL可）。
  • **出力**:
  • sql: 生成されたSQL文が追記される PQExpBufferData。

前提条件と不変条件

  • name 引数はNULLであってはならない（Assert(name) による検証）。
  • sql バッファは呼び出し前に初期化されている必要がある。

アルゴリズムまたは振る舞い

1. REINDEX キーワードをバッファに追加する。 2. verbose が真の場合、VERBOSE オプションを追加する。 3. tablespace が指定されている場合、TABLESPACE 句を追加する。 4. type に応じて対象キーワード（DATABASE, INDEX, SCHEMA, SYSTEM, TABLE）を追加する。 5. concurrently が真の場合、CONCURRENTLY キーワードを追加する。 6. type に応じて対象オブジェクト名を適切にフォーマット（fmtIdEnc または appendQualifiedRelation）して追加する。 7. 最後にセミコロン ; を付与する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/scripts/reindexdb.c 内の並列実行処理（run_reindex_command 関連のロジック）から呼び出される。
  • **呼び出し先**: appendPQExpBuffer 系の文字列操作関数、fmtIdEnc、appendQualifiedRelation、PQclientEncoding。

関連データ構造

  • ReindexType: 再構築対象を定義する列挙型。
  • PQExpBufferData: SQL文構築用の文字列バッファ。
  • PGconn: PostgreSQL接続オブジェクト。

失敗モードと境界条件

  • tablespace 指定時のエンコーディング変換失敗や、不正なオブジェクト名が渡された場合の挙動は、依存する fe_utils 側の実装に依存する。
  • サーバーバージョンが古い場合に CONCURRENTLY 等のオプションを付与した場合の整合性は、本関数ではなく呼び出し側のロジックで制御される。

保守担当者向けメモ

  • 新しい REINDEX オプションを追加する場合は、本関数のSQL生成ロジックと、呼び出し側の引数解析ロジックの両方を更新する必要がある。
  • CONCURRENTLY の配置はPostgreSQLのバージョンによって文法が異なるため、変更時は公式ドキュメントの文法定義を確認すること。
