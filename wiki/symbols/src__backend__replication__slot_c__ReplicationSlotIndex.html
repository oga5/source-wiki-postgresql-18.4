ReplicationSlotIndex

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slot.c:541

目的

共有メモリ上のレプリケーションスロット配列内における、指定されたスロットのインデックス（0から始まるオフセット）を算出します。

シグネチャまたは形状

int ReplicationSlotIndex(ReplicationSlot *slot)

入力と出力

  • **入力**: ReplicationSlot *slot - インデックスを特定したいスロットへのポインタ。
  • **出力**: int - ReplicationSlotCtl->replication_slots 配列内での当該スロットのインデックス。

前提条件と不変条件

  • slot は ReplicationSlotCtl->replication_slots 配列の範囲内（先頭から max_replication_slots 個の要素の範囲）に存在しなければなりません。この条件は Assert によって検証されます。

アルゴリズムまたは振る舞い

1. slot ポインタが有効な共有メモリ領域内にあることを Assert で確認します。 2. ポインタ演算（slot - ReplicationSlotCtl->replication_slots）を行い、配列の先頭からの相対位置を整数として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/pgstat_replslot.c (複数箇所)
  • src/backend/replication/slot.c:542 (自己参照の可能性あり)
  • **呼び出し先**:
  • 特筆すべき関数呼び出しはありません（ポインタ演算のみ）。

関連データ構造

  • ReplicationSlot: スロットの情報を保持する構造体。
  • ReplicationSlotCtl: スロットの共有メモリ状態を保持するグローバル制御構造体。

失敗モードと境界条件

  • slot が ReplicationSlotCtl->replication_slots 配列の範囲外である場合、アサーション失敗（デバッグビルド時）または不正なインデックス値の算出が発生します。

保守担当者向けメモ

  • 本関数は単なるポインタ演算であり、ロックの取得は行いません。
  • pgstat_replslot.c における統計情報の管理において、スロットを識別するためのキーとして頻繁に使用されています。
