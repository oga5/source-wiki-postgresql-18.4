next

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_lex.c:199

目的

正規表現の字句解析におけるメイン関数であり、入力文字列を走査して次のトークンを特定し、解析状態を更新します。

シグネチャまたは形状

int next(struct vars *v)

入力と出力

  • **入力**: struct vars *v（解析状態および入力文字列ポインタ v->now を保持する構造体）
  • **出力**: int（成功時に 1、失敗時に 0 を返す）
  • **副作用**:
  • v->nexttype および v->nextvalue に特定されたトークン情報を格納。
  • v->now ポインタを進め、入力文字列を消費。
  • v->lasttype を更新。

前提条件と不変条件

  • v は有効な vars 構造体を指していること。
  • v->lexcon により現在の解析コンテキストが正しく管理されていること。
  • エラー発生時（ISERR() が真）は、即座に 0 を返し、nexttype を EOS に設定する。

アルゴリズムまたは振る舞い

1. **初期チェック**: エラー状態を確認し、必要に応じて REG_BOSONLY フラグに基づく SBEGIN トークンを生成する。 2. **空白スキップ**: REG_EXPANDED フラグが有効かつ特定のコンテキスト（L_ERE, L_BRE 等）であれば skip(v) を呼び出す。 3. **終端処理**: 入力文字列の終端（EOS）に達した場合、現在のコンテキストに応じてトークン（EOS）を返すか、未閉じの括弧やブラケットに対するエラー（REG_EBRACE, REG_EBRACK）を発生させる。 4. **文字取得とコンテキスト分岐**:

  • L_BRE: brenext(v, c) に処理を委譲。
  • L_Q: リテラルとして PLAIN トークンを生成。
  • L_BBND/L_EBND: 数字や ,, } を解析し、境界指定トークンを生成。
  • L_ERE: 後続のコード（ソース抜粋外）で解析を継続。

呼び出し元と呼び出し先

  • **呼び出し元**: regcomp.c（字句解析器をループ内で呼び出し、構文木を構築する）。
  • **呼び出し先**:
  • skip(v): 空白スキップ用。
  • brenext(v, c): BRE（基本正規表現）の解析用。
  • RETV, RET, FAILW: トークン返却およびエラー処理用マクロ。
  • **注意**: ツールによる contrib/amcheck/verify_gin.c 等の参照は、本シンボル next とは無関係な構造体メンバ名との衝突であり、本関数への呼び出しではありません。

関連データ構造

  • struct vars: 解析状態、フラグ、現在のトークンタイプ、入力ポインタを保持する主要構造体。
  • lexcon: 現在の解析コンテキスト（L_ERE, L_BRE, L_BRACK 等）。

失敗モードと境界条件

  • **エラー発生時**: ISERR() が真の場合、即座に 0 を返す。
  • **構文エラー**: 括弧やブラケットが閉じられていない場合、FAILW マクロにより REG_EBRACE や REG_EBRACK を発生させる。
  • **EOS**: コンテキストに応じた適切な終了処理が行われない場合、assert(NOTREACHED) に到達する可能性がある。

保守担当者向けメモ

  • 本関数は regcomp.c に展開されるため、v を介した暗黙的な状態共有に依存しています。
  • L_BRACK や L_EBND などのコンテキスト遷移はバグが発生しやすいため、変更時は lexcon の状態遷移図を確認してください。
  • ARE（Advanced Regular Expressions）の拡張構文を追う場合は、CHR('(') のケースから解析フローを確認することを推奨します。
