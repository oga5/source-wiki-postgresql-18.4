SQLFunctionParseInfo

定義

  • 種別: struct
  • 定義箇所: src/include/executor/functions.h:25

目的

SQL言語で記述された関数の本体を解析する際、関数の引数名、型、および照合順序（collation）情報を保持するために使用されます。

シグネチャまたは形状

typedef struct SQLFunctionParseInfo
{
    char       *fname;          /* 関数名 */
    int         nargs;          /* 入力引数の数 */
    Oid        *argtypes;       /* 入力引数の解決済み型 */
    char      **argnames;       /* 入力引数名（名前がない場合はNULL） */
    Oid         collation;      /* 関数の入力照合順序（既知の場合） */
} SQLFunctionParseInfo;

入力と出力

  • 本構造体自体はデータ保持用であり、直接的な関数入出力は持ちません。
  • prepare_sql_fn_parse_info 関数によって pg_proc タプルから情報が抽出され、本構造体に格納されます。

前提条件と不変条件

  • argnames[i] は、引数に名前が付けられていない場合に NULL となる可能性があります。
  • 実行時ではなく、解析フェーズでのみ使用される情報です。

アルゴリズムまたは振る舞い

1. prepare_sql_fn_parse_info が呼び出されると、本構造体がメモリ上に確保され、関数のメタデータで初期化されます。 2. 解析プロセスにおいて、関数内のパラメータ参照を解決するためのコンテキストとして利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/functioncmds.c および src/backend/executor/functions.c 内の解析準備ロジック。
  • 呼び出し先: 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • SQLFunctionParseInfoPtr: 本構造体へのポインタ型。
  • pg_proc: 本構造体の情報を生成するためのソースとなるシステムカタログ。

失敗モードと境界条件

  • メモリ割り当て失敗（palloc0 等）が発生した場合、プロセスは終了します。
  • 引数名が未定義の場合、argnames 配列内の該当要素が NULL であることを適切に処理する必要があります。

保守担当者向けメモ

  • 本構造体は実行時のキャッシュ構造体とは分離されています。変更を加える際は、解析フェーズのロジックにのみ影響を与えることを確認してください。
  • PL/pgSQL等の手続き型言語の関数とは処理経路が異なるため、SQL言語関数専用の解析ロジックであることを留意してください。
