add_size

定義

  • 種別: function
  • 定義箇所: src/common/fe_memutils.c:195

目的

2つのサイズ値の加算を行い、整数オーバーフローおよびPostgreSQLのメモリ管理における安全な上限値（SIZE_MAX / 2）を超過していないかを検証します。

シグネチャまたは形状

Size add_size(Size s1, Size s2)

入力と出力

  • **入力**: s1 (Size), s2 (Size) - 加算対象の2つのサイズ。
  • **出力**: 加算結果（Size 型）。成功時は s1 + s2 を返します。

前提条件と不変条件

  • 加算結果が SIZE_MAX を超えるオーバーフローが発生してはなりません。
  • 加算結果は SIZE_MAX / 2 以下でなければなりません。この制約はバックエンドとの互換性を維持するための設計上の制限です。

アルゴリズムまたは振る舞い

1. pg_add_size_overflow(s1, s2, &result) を呼び出し、加算による整数オーバーフローをチェックします。 2. 加算結果 result が SIZE_MAX / 2 を超えているかを確認します。 3. 上記のいずれかの条件に該当する場合、add_size_error(s1, s2) を呼び出し、エラー処理（通常はプロセスの終了）を行います。 4. 条件を満たす場合、計算された result を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_stat_statements/pg_stat_statements.c
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/index/indexam.c
  • src/backend/access/nbtree/nbtree.c
  • **呼び出し先**:
  • pg_add_size_overflow (マクロまたはインライン関数)
  • add_size_error

関連データ構造

  • Size: 通常 size_t に相当する型。

失敗モードと境界条件

  • **オーバーフロー**: s1 + s2 が SIZE_MAX を超える場合、add_size_error が呼び出されます。
  • **上限超過**: s1 + s2 > SIZE_MAX / 2 の場合、物理メモリの有無に関わらずエラーとして扱われます。

保守担当者向けメモ

  • 本関数はフロントエンド環境（FRONTEND マクロ定義時）でのメモリ確保の安全性を担保するために使用されます。
  • add_size_error は呼び出し側でプロセスを終了させるため、本関数が戻り値を返す場合は常に安全なサイズであることが保証されます。
