SubXactCallbackItem

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/xact.c:320

目的

サブトランザクションのイベント（開始、コミット、アボートなど）発生時に実行されるコールバック関数を管理するためのリスト要素です。

シグネチャまたは形状

typedef struct SubXactCallbackItem
{
    struct SubXactCallbackItem *next;
    SubXactCallback callback;
    void       *arg;
} SubXactCallbackItem;

入力と出力

  • next: リスト内の次の要素へのポインタ。
  • callback: 登録された SubXactCallback 型の関数ポインタ。
  • arg: コールバック関数に渡される任意のデータへのポインタ。

前提条件と不変条件

  • リストの管理は TopMemoryContext 上で行われます。
  • リストの先頭は静的変数 SubXact_callbacks によって保持されます。

アルゴリズムまたは振る舞い

  • サブトランザクション関連のイベントが発生した際、この構造体で構成されるリストを走査し、登録されたすべてのコールバック関数を順次実行します。
  • 新しいコールバックの登録時には、MemoryContextAlloc を用いてメモリを確保し、リストの先頭に追加されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c 内のコールバック登録および削除関数（RegisterSubXactCallback 等）。
  • 呼び出し先: 登録された SubXactCallback 型の関数。
  • 注: 具体的な関数呼び出しのフローについては、ソース抜粋からは詳細な依存関係が特定できません。

関連データ構造

  • SubXactCallback: コールバック関数の型定義。
  • XactCallbackItem: トランザクション全体のコールバックを管理する類似の構造体。

失敗モードと境界条件

  • メモリ不足: MemoryContextAlloc が失敗した場合、システムはエラーを発生させます。
  • リストの空状態: SubXact_callbacks が NULL の場合、コールバックは実行されません。

保守担当者向けメモ

  • 本構造体はサブトランザクションのライフサイクルに密接に関連しています。コールバック関数の追加・削除を行う際は、リストの走査順序やメモリリークの可能性に注意してください。
