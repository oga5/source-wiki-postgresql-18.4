get_join_index_paths

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/indxpath.c:605

目的

特定の結合条件セット（relids）に対して、インデックスを利用可能なパスを生成する。パラメータ化されたインデックススキャンを構築するために、制約条件と結合条件を組み合わせてインデックス列に適合させる。

シグネチャまたは形状

static void get_join_index_paths(PlannerInfo *root, RelOptInfo *rel,
                                 IndexOptInfo *index,
                                 IndexClauseSet *rclauseset,
                                 IndexClauseSet *jclauseset,
                                 IndexClauseSet *eclauseset,
                                 List **bitindexpaths,
                                 Relids relids,
                                 List **considered_relids)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel: 対象リレーション情報。
  • index: 使用するインデックス情報。
  • rclauseset, jclauseset, eclauseset: それぞれ制約条件、結合条件、等価クラス条件のセット。
  • relids: 現在考慮中の結合対象リレーションの集合。
  • **出力**:
  • bitindexpaths: 生成されたパスが追加されるリスト。
  • considered_relids: 処理済みリレーション集合のリスト（更新される）。

前提条件と不変条件

  • relids が既に considered_relids に含まれている場合、関数は即座に終了する。
  • 収集された clauseset が空でないこと（Assert(clauseset.nonempty)）。

アルゴリズムまたは振る舞い

1. considered_relids を確認し、重複処理を回避する。 2. インデックスの各列に対し、以下の条件を収集する:

  • jclauseset から relids のサブセットである結合条件。
  • eclauseset から relids のサブセットである等価クラス条件（列ごとに最大1つ）。
  • rclauseset からの制約条件。

    • 3. 収集した条件を用いて get_index_paths を呼び出し、パスを生成する。 4. relids を considered_relids に追加し、今後の重複実行を防ぐ。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/indxpath.c 内の consider_index_join_clauses 等から呼び出される。
  • **呼び出し先**: get_index_paths を呼び出す。

関連データ構造

  • IndexClauseSet: インデックス列ごとの条件リストを保持する構造体。
  • IndexClause: インデックスに適合する個別の条件。
  • RelOptInfo, IndexOptInfo: プランナが使用するリレーションおよびインデックスのメタデータ。

失敗モードと境界条件

  • relids に適合する条件が一つも存在しない場合、Assert により停止する。
  • 既に処理済みの relids が渡された場合は何も行わない。

保守担当者向けメモ

  • 本関数はパラメータ化されたインデックススキャンの生成において重要な役割を果たす。
  • eclauseset の処理において、冗長な条件を避けるために break を使用している点に注意すること。
  • considered_relids による重複チェックは、プランナの計算量を削減するための重要な最適化である。
