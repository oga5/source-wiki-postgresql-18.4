SupportRequestRows

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/supportnodes.h:158

目的

関数呼び出しによって返されると予想される行数（カーディナリティ）を、プランナーが取得するためのリクエスト構造体です。

シグネチャまたは形状

typedef struct SupportRequestRows
{
	NodeTag		type;
	struct PlannerInfo *root;
	Oid			funcid;
	Node	   *node;
	double		rows;
} SupportRequestRows;

入力と出力

  • **入力**:
  • root: プランナーのインフラストラクチャ情報（NULL の可能性あり）。
  • funcid: 問い合わせ対象の関数の OID。
  • node: 関数を呼び出している構文解析ノード。
  • **出力**:
  • rows: 予想される行数（double 型）。

前提条件と不変条件

  • SupportRequest プロトコルに従い、サポート関数が処理できない場合は NULL を返します。
  • 本構造体は一時的な利用を想定しており、サポート関数内で永続的なメモリ確保を行う必要はありません。

アルゴリズムまたは振る舞い

1. プランナーが特定の関数呼び出しの行数見積もりを必要とする際、本構造体を生成します。 2. 対象関数に紐付けられたサポート関数が呼び出され、見積もり可能な場合は rows フィールドに値を格納します。 3. 呼び出し元は、返された値を使用してプランの最適化やコスト見積もりを調整します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • src/backend/optimizer/util/plancat.c
  • **呼び出し先（型チェック等での参照）**:
  • src/backend/utils/adt/arrayfuncs.c
  • src/backend/utils/adt/int.c
  • src/backend/utils/adt/int8.c
  • src/backend/utils/adt/numeric.c
  • src/backend/utils/adt/timestamp.c

関連データ構造

  • PlannerInfo: プランナーのコンテキスト情報。
  • Node: 構文解析ノードの基底型。

失敗モードと境界条件

  • サポート関数がこのリクエストを処理できない場合、エラーではなく NULL を返すことで、呼び出し元に処理不能であることを伝えます。

保守担当者向けメモ

  • 本構造体はプランナーだけでなく、実行エンジンからも利用される可能性があるため、プランナー専用のロジックと限定しないよう注意してください。
