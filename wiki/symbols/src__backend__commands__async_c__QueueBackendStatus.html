QueueBackendStatus

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/async.c:243

目的

非同期通知（NOTIFY）システムにおいて、各バックエンドプロセスの状態と通知キューの読み取り進捗を追跡するために使用される構造体です。

シグネチャまたは形状

typedef struct QueueBackendStatus
{
    int32       pid;            /* プロセスIDまたはInvalidPid */
    Oid         dboid;          /* バックエンドのデータベースOIDまたはInvalidOid */
    ProcNumber  nextListener;   /* 次のリスナーのIDまたはINVALID_PROC_NUMBER */
    QueuePosition pos;          /* バックエンドが読み取り済みのキュー位置 */
} QueueBackendStatus;

入力と出力

  • 本構造体は共有メモリ上に配置され、AsyncQueueControl 構造体内の配列として管理されます。
  • 入力: バックエンドのプロセス情報および通知キューの読み取り位置。
  • 出力: なし（状態保持のためのデータ構造）。

前提条件と不変条件

  • AsyncQueueControl 内の backend 配列として、MaxBackends 分の領域が確保されている必要があります。
  • 読み取り位置 pos は、通知キューのクリーンアップ（asyncQueueAdvanceTail）において、どのSLRUページまでが不要かを判断するための基準として使用されます。

アルゴリズムまたは振る舞い

  • 各バックエンドは自身の QueueBackendStatus を更新することで、通知キューの読み取り状況をシステム全体に公開します。
  • システムは全バックエンドの pos を参照し、最も古い読み取り位置を特定することで、キューのテール（末尾）を進め、不要になったSLRUページを解放します。

呼び出し元と呼び出し先

  • 呼び出し元: AsyncShmemInit 等で共有メモリ領域の初期化時に使用されます。
  • 呼び出し先: 直接的な関数呼び出し関係はありませんが、asyncQueueAdvanceTail 等の関数が本構造体の pos フィールドを参照します。

関連データ構造

  • AsyncQueueControl: 本構造体を配列として保持する親構造体。
  • QueuePosition: pos フィールドの型。
  • AsyncQueueEntry: キューに格納される実際の通知データ。

失敗モードと境界条件

  • MaxBackends を超えるバックエンドが同時に存在することは想定されていません。
  • pos が適切に更新されない場合、読み取りが遅いバックエンドが存在すると見なされ、キューのテールが進まないため、SLRU領域のディスク消費量が増大し続けます。

保守担当者向けメモ

  • 本構造体は共有メモリ上に存在するため、変更を加える際は NotifyQueueLock 等の適切なロック取得が必要です。
  • pos フィールドの更新タイミングは、通知の読み取り処理と密接に関連しているため、変更時は asyncQueueReadAllNotifications への影響を考慮してください。
