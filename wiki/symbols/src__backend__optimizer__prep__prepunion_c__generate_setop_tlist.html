generate_setop_tlist

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepunion.c:1356

目的

集合演算（UNION, INTERSECT, EXCEPT）の出力ターゲットリスト（tlist）を生成します。入力列の型や照合順序（Collation）を集合演算の要求仕様に適合させ、必要に応じて型変換や照合順序の再ラベル付けを行います。

シグネチャまたは形状

List *generate_setop_tlist(List *colTypes, List *colCollations,
                           Index varno,
                           bool hack_constants,
                           List *input_tlist,
                           List *refnames_tlist,
                           bool *trivial_tlist)

入力と出力

  • **入力**:
  • colTypes, colCollations: 集合演算結果に要求される型と照合順序のリスト。
  • varno: 生成される Var ノードで使用するリレーション番号。
  • hack_constants: 定数をサブクエリ出力としてではなく定数として扱うかのフラグ。
  • input_tlist: 入力サブクエリのターゲットリスト。
  • refnames_tlist: 参照名（列名）を取得するためのターゲットリスト。
  • **出力**:
  • 構築された TargetEntry のリスト。
  • trivial_tlist: 型変換や再ラベル付けが不要な場合に true が設定される出力引数。

前提条件と不変条件

  • input_tlist と refnames_tlist の各要素数は colTypes および colCollations と一致している必要があります。
  • 各ターゲットリスト要素の resno は 1 から始まる連番である必要があります。
  • 生成される全ての TargetEntry に対して、ressortgroupref は自身の resno と等しく設定されます。

アルゴリズムまたは振る舞い

1. trivial_tlist を true に初期化します。 2. 入力リストを走査し、各列に対して以下を実行します。

  • hack_constants が有効かつ入力が定数の場合、その定数をそのまま式として使用します。それ以外の場合は makeVar を用いて入力列を参照する Var ノードを生成します。
  • 式の型が colTypes と異なる場合、coerce_to_common_type を適用し、trivial_tlist を false に設定します。
  • 式の照合順序が colCollations と異なる場合、applyRelabelType を適用し、trivial_tlist を false に設定します。
  • 最終的な式を用いて TargetEntry を作成し、リストに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/prep/prepunion.c 内の plan_set_operations 等から呼び出されます。
  • **呼び出し先**: makeVar, coerce_to_common_type, applyRelabelType, makeTargetEntry 等のノード構築・変換関数。

関連データ構造

  • TargetEntry: ターゲットリストの各要素を構成する構造体。
  • Var: 集合演算の入力を参照するために生成されるノード。
  • RelabelType: 照合順序の変更時に使用されるノード。

失敗モードと境界条件

  • 入力リストの長さが不一致の場合、forfour マクロの挙動により予期せぬ動作やアサーション失敗が発生する可能性があります。
  • hack_constants を使用する際は、上位レベルのノードで不要な定数が混入しないよう注意が必要です。

保守担当者向けメモ

  • coerce_to_common_type の適用は、assign_expr_collations が実行されないという副作用があるため、将来的にパーサー側で適切なターゲットリストを生成するように改善することが望ましいです。
  • varno 0 を指定することで、集合演算の出力列を指す特殊な Var ノードを生成する慣習があります。
