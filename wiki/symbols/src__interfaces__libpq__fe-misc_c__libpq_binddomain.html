libpq_binddomain

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-misc.c:1303

目的

libpq のメッセージカタログを bindtextdomain を用いてプロセスごとに一度だけ初期化し、国際化対応を有効にする。

シグネチャまたは形状

static void libpq_binddomain(void)

入力と出力

  • 入力: なし（環境変数 PGLOCALEDIR または定数 LOCALEDIR を参照）
  • 出力: なし（副作用として bindtextdomain を実行）

前提条件と不変条件

  • プロセス内で一度だけ実行されることが保証される（already_bound フラグと binddomain_mutex による排他制御）。
  • bindtextdomain 呼び出し前後で errno（Windows の場合は GetLastError()）が保存・復元される。

アルゴリズムまたは振る舞い

1. 静的フラグ already_bound を確認し、未実行の場合のみ処理を進める。 2. 現在の errno を保存する。 3. pthread_mutex_lock を用いてスレッドセーフに再確認を行う。 4. 環境変数 PGLOCALEDIR が設定されていればその値を、なければ LOCALEDIR をロケールディレクトリとして使用する。 5. bindtextdomain(PG_TEXTDOMAIN("libpq"), ldir) を呼び出し、already_bound を true に設定する。 6. pthread_mutex_unlock を実行し、保存しておいた errno を復元する。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-misc.c 内の翻訳が必要な関数（libpq_gettext 等の内部から呼び出されることが示唆される）。
  • 呼び出し先: bindtextdomain（外部ライブラリ関数）、pthread_mutex_lock/unlock、getenv。

関連データ構造

  • already_bound: 実行状態を保持する static volatile bool。
  • binddomain_mutex: 排他制御用の pthread_mutex_t。

失敗モードと境界条件

  • bindtextdomain が失敗した場合でも、already_bound は true に設定される。
  • Windows 環境では GetLastError() を使用し、それ以外では errno を使用してエラーコードの保存・復元を行う。

保守担当者向けメモ

  • Windows における gettext 実装の競合回避を主目的としているが、プラットフォームを問わず一貫した挙動をとるよう設計されている。
  • 呼び出し元に関する詳細なスタックトレースは提供されていないが、fe-misc.c 内の翻訳処理の起点として機能している。
