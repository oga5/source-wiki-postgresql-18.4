pgstat_report_wait_end

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/wait_event.h:84

目的

現在実行中のバックエンドプロセスが待機状態を終了したことを記録し、統計情報収集サブシステムに対して待機イベントの解除を通知します。

シグネチャまたは形状

static inline void pgstat_report_wait_end(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として my_wait_event_info が指すメモリ領域を 0 に更新）

前提条件と不変条件

  • my_wait_event_info は、プロセスが待機状態にある間、有効なメモリ領域（ローカル変数または共有メモリ上の MyProc 構造体内の領域）を指している必要があります。
  • 待機イベント情報の更新はアトミックに行われることが保証されています。

アルゴリズムまたは振る舞い

1. my_wait_event_info が指すメモリ領域を volatile uint32 としてキャストします。 2. その値を 0 に設定することで、待機状態の終了を明示します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/heap/rewriteheap.c
  • src/backend/access/transam/clog.c
  • src/backend/access/transam/slru.c
  • 呼び出し先: なし（インライン関数であり、直接的な関数呼び出しは行いません）

関連データ構造

  • my_wait_event_info: 現在のプロセスの待機イベント情報を保持するポインタ。

失敗モードと境界条件

  • my_wait_event_info が初期化されていない、あるいは不正なアドレスを指している場合、セグメンテーション違反が発生する可能性があります。
  • 待機開始の報告（pgstat_report_wait_start）が行われていない状態で呼び出した場合、単に 0 を書き込む操作となります。

保守担当者向けメモ

  • 本関数は非常に頻繁に呼び出されるため、オーバーヘッドを最小限に抑える設計となっています。設定値のチェックは行わず、常に書き込みを実行します。
  • pgstat_report_wait_start() と対になる関数です。待機イベントの追跡ロジックを変更する際は、必ず両方の関数を確認してください。
