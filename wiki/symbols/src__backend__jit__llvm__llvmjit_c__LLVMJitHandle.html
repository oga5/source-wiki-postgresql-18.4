LLVMJitHandle

定義

  • 種別: struct
  • 定義箇所: src/backend/jit/llvm/llvmjit.c:48

目的

LLVM ORC JIT の実行環境と、それに関連付けられたリソーストラッカーを保持し、PostgreSQL の JIT コンパイルライフサイクルを管理するためのハンドル構造体です。

シグネチャまたは形状

typedef struct LLVMJitHandle
{
	LLVMOrcLLJITRef lljit;
	LLVMOrcResourceTrackerRef resource_tracker;
} LLVMJitHandle;

入力と出力

  • **入力**: MemoryContextAlloc を介してメモリが割り当てられます。
  • **出力**: 構造体内部の lljit および resource_tracker フィールドを通じて、LLVM の JIT インスタンスおよびリソース管理機能へのアクセスを提供します。

前提条件と不変条件

  • lljit は LLVMOrcLLJITRef 型であり、有効な ORC JIT インスタンスを指している必要があります。
  • resource_tracker は、当該 JIT インスタンスに関連付けられたリソースの追跡に使用されます。

アルゴリズムまたは振る舞い

本構造体は、JIT コンパイルされたコードの実行環境をカプセル化します。llvm_compile_module 等の処理において、このハンドルを介してモジュールの登録や実行エンジンの制御が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/jit/llvm/llvmjit.c 内の複数の箇所で参照されています。
  • llvm_compile_module 関連のフロー（行 281, 384）
  • コンテキスト初期化および管理処理（行 711, 762, 763）
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • LLVMOrcLLJITRef: LLVM ORC JIT インスタンスへの参照。
  • LLVMOrcResourceTrackerRef: JIT されたリソースの追跡用ハンドル。
  • LLVMJitContext: 本構造体を利用して JIT 状態を管理する上位のコンテキスト構造体。

失敗モードと境界条件

  • MemoryContextAlloc によるメモリ割り当て失敗時、本構造体の生成自体が失敗します。
  • lljit や resource_tracker が無効な場合、後続の LLVM C API 呼び出しでエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は TopMemoryContext 上に割り当てられるケース（行 763）があるため、ライフサイクル管理には注意が必要です。
  • LLVM C API の変更に伴い、lljit や resource_tracker の型定義が変更される可能性があるため、llvm-c ヘッダーとの整合性に留意してください。
