ExecIndexOnlyRestrPos

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexonlyscan.c:491

目的

インデックスのみスキャン（Index-Only Scan）において、以前にマークされたスキャン位置へカーソルを復元する。

シグネチャまたは形状

void ExecIndexOnlyRestrPos(IndexOnlyScanState *node)

入力と出力

  • 入力: IndexOnlyScanState *node（インデックスのみスキャンの実行状態を保持する構造体）
  • 出力: なし（副作用としてスキャン位置を更新）

前提条件と不変条件

  • node は初期化済みであること。
  • EvalPlanQual（EPQ）実行中の場合、対象リレーションがサブプランや行マークによって処理済みである必要がある。

アルゴリズムまたは振る舞い

1. **EPQチェック**: estate->es_epq_active が存在する場合、現在のスキャン対象リレーションがEPQの再チェック対象かを確認する。 2. **EPQ検証**: 対象リレーションがEPQで処理中であるにもかかわらず relsubs_done が完了していない場合、エラーを発生させる。 3. **位置復元**: 上記のEPQ制約に抵触しない場合、node->ioss_ScanDesc を使用して index_restrpos を呼び出し、インデックススキャンの位置を復元する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execAmi.c:385（直接的な呼び出しを確認）
  • 呼び出し先: index_restrpos（インデックスアクセスメソッドのAPI）

関連データ構造

  • IndexOnlyScanState: スキャンノードの実行状態を管理。
  • EPQState: EvalPlanQual実行時の状態管理。
  • Scan: プランノードの基底構造。

失敗モードと境界条件

  • **EPQエラー**: EPQ再チェック中に本関数が呼び出された際、relsubs_done が偽であれば ERROR を投げる。これはインデックスのみスキャンがEPQの再チェックをサポートしていないことに起因する。

保守担当者向けメモ

  • 本関数は ExecIndexMarkPos と対になる操作を行う。
  • インデックスのみスキャンはEvalPlanQual（行レベルロックの再チェック）をサポートしていないため、EPQ環境下での呼び出しには厳格なガードが存在する。
