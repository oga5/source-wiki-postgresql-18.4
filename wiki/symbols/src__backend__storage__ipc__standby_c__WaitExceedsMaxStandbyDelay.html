WaitExceedsMaxStandbyDelay

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/standby.c:233

目的

ホットスタンバイ環境において、リカバリプロセスが競合解決を待機する際、設定された最大待機時間を超過したかどうかを判定し、超過していない場合は指数バックオフを用いた待機を行う。

シグネチャまたは形状

bool WaitExceedsMaxStandbyDelay(uint32 wait_event_info)

入力と出力

  • **入力**: wait_event_info (待機イベントの種類を示す識別子)
  • **出力**: bool 型。最大待機時間を超過した場合は true、それ以外の場合は false を返す。

前提条件と不変条件

  • GetStandbyLimitTime() が有効な期限を返していること。
  • standbyWait_us 変数がモジュール内で適切に初期化・維持されていること。
  • CHECK_FOR_INTERRUPTS() により、呼び出し元で割り込みが処理されること。

アルゴリズムまたは振る舞い

1. CHECK_FOR_INTERRUPTS() を実行し、割り込みを確認する。 2. GetStandbyLimitTime() を呼び出し、現在の時刻が期限を超過しているか判定する。超過していれば true を返す。 3. 超過していない場合、pgstat_report_wait_start と pgstat_report_wait_end で囲まれた範囲で pg_usleep(standbyWait_us) を実行し、待機する。 4. 次回の待機時間を増やすため、standbyWait_us を2倍にする（最大上限は 1,000,000 マイクロ秒）。 5. false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/standby.c 内の競合解決処理（ResolveRecoveryConflict... 系関数など）から呼び出されることが示唆されている。
  • **呼び出し先**: CHECK_FOR_INTERRUPTS(), GetStandbyLimitTime(), GetCurrentTimestamp(), pgstat_report_wait_start(), pg_usleep(), pgstat_report_wait_end()。

関連データ構造

  • standbyWait_us (モジュール内の静的変数として待機時間を保持)

失敗モードと境界条件

  • **待機時間の上限**: standbyWait_us は最大 1 秒（1,000,000 マイクロ秒）に制限される。これは一部のプラットフォームで pg_usleep が割り込み不可能であることに起因する。
  • **期限の未設定**: GetStandbyLimitTime() が 0 を返す場合、期限超過判定は行われない。

保守担当者向けメモ

  • 本関数はビジーウェイトを回避するための指数バックオフを実装している。
  • 呼び出し元で wait_event_info を適切に指定することで、統計情報（pg_stat_activity 等）に正確な待機理由が反映される。
