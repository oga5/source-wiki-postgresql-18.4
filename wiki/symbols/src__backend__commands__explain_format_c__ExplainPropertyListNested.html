ExplainPropertyListNested

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_format.c:107

目的

文字列リストを、現在の EXPLAIN 出力フォーマット（TEXT, XML, JSON, YAML）に応じた形式で出力バッファに書き込む。

シグネチャまたは形状

void ExplainPropertyListNested(const char *qlabel, List *data, ExplainState *es)

入力と出力

  • qlabel: プロパティのラベル名（TEXT/XML形式で使用）。
  • data: 出力対象の文字列リスト（List）。
  • es: 出力状態を管理する ExplainState 構造体。
  • 戻り値: なし（es->str に結果が追記される）。

前提条件と不変条件

  • es が有効な ExplainState を指していること。
  • data 内の各要素は const char * としてキャスト可能であること。

アルゴリズムまたは振る舞い

es->format に基づき以下の処理を行う： 1. **TEXT / XML**: ExplainPropertyList を呼び出し、共通のリスト出力処理に委譲する。 2. **JSON**:

  • ExplainJSONLineEnding で行末処理を行う。
  • インデントを挿入し、[ で開始する。
  • リスト内の各要素を escape_json でエスケープし、カンマ区切りで出力する。
  • ] で閉じる。

    • 3. **YAML**:

  • ExplainYAMLLineStarting を呼び出す。
  • - [ で開始し、各要素を escape_yaml でエスケープして出力する。
  • ] で閉じる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/explain.c:2733 で確認されている。
  • 呼び出し先: ExplainPropertyList, ExplainJSONLineEnding, escape_json, ExplainYAMLLineStarting, escape_yaml, appendStringInfo 系関数。

関連データ構造

  • ExplainState: 出力バッファやフォーマット状態を保持する。
  • List: 出力対象の文字列を格納する PostgreSQL のリスト構造。

失敗モードと境界条件

  • リストが空の場合、JSON/YAML では [] が出力される。
  • data 内の要素が NULL である場合の挙動については、本ソース内では明示的なチェックは行われていない。

保守担当者向けメモ

  • 新しいフォーマットを追加する場合は、switch 文にケースを追加する必要がある。
  • JSON/YAML 形式でのカンマ管理は first フラグによって制御されている。
