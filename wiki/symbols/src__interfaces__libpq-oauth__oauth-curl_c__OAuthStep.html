OAuthStep

定義

  • 種別: enum
  • 定義箇所: src/interfaces/libpq-oauth/oauth-curl.c:214

目的

PostgreSQLの libpq における OAuth 2.0 Device Authorization Grant フローの現在地を管理するステートマシン。

シグネチャまたは形状

enum OAuthStep
{
	OAUTH_STEP_INIT = 0,
	OAUTH_STEP_DISCOVERY,
	OAUTH_STEP_DEVICE_AUTHORIZATION,
	OAUTH_STEP_TOKEN_REQUEST,
	OAUTH_STEP_WAIT_INTERVAL,
};

入力と出力

  • 入力: 状態遷移のトリガーとなる非同期イベント（libcurl の通信結果等）。
  • 出力: 現在の認証ステップを示す列挙値。

前提条件と不変条件

  • OAUTH_STEP_INIT から開始し、定義された順序（Discovery → Device Authorization → Token Request）で遷移する。
  • TOKEN_REQUEST 失敗時は OAUTH_STEP_WAIT_INTERVAL へ遷移し、待機後に再試行を行う。

アルゴリズムまたは振る舞い

認証フローの各段階を識別し、async_ctx 内で現在の処理フェーズを保持するために使用される。pg_fe_run_oauth_flow がこの状態に基づき、適切なネットワークリクエストを駆動する。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq-oauth/oauth-curl.c:230 にて、認証フローの現在地を保持する変数として参照されている。
  • 呼び出し先: 情報なし。

関連データ構造

  • async_ctx: 認証フローの全状態を保持するコンテキスト構造体。本列挙型はこの構造体の一部として利用される。

失敗モードと境界条件

  • 状態遷移の順序が不正な場合、認証フローが適切に進行しない可能性がある。
  • TOKEN_REQUEST 失敗時の WAIT_INTERVAL への遷移は、サーバー指定の待機時間に基づき制御される。

保守担当者向けメモ

  • 本列挙型は libpq-oauth のステートマシンの中核であるため、新しい OAuth ステップを追加する際は、drive_request 内の遷移ロジックも併せて更新する必要がある。
