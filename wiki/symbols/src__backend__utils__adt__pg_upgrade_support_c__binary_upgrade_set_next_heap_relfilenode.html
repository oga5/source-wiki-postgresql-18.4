binary_upgrade_set_next_heap_relfilenode

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/pg_upgrade_support.c:107

目的

pg_upgrade 実行時に、次に作成されるヒープリレーションの物理ファイル番号（RelFileNumber）を強制的に指定するために使用されます。

シグネチャまたは形状

Datum binary_upgrade_set_next_heap_relfilenode(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介して、設定対象となる RelFileNumber（OID として渡される）を受け取ります。
  • **出力**: PG_RETURN_VOID() により、値を返さずに終了します。

前提条件と不変条件

  • **ガード条件**: CHECK_IS_BINARY_UPGRADE マクロにより、サーバーが binary_upgrade モードで起動されている必要があります。それ以外の場合はエラーが発生します。

アルゴリズムまたは振る舞い

1. 引数から RelFileNumber を取得します。 2. CHECK_IS_BINARY_UPGRADE を実行し、アップグレードモードであることを検証します。 3. グローバル変数 binary_upgrade_next_heap_pg_class_relfilenumber に、引数で受け取った値を代入します。 4. 処理を完了し、void を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c 内の SQL 生成ロジックから呼び出されます。
  • **呼び出し先**: 内部的なグローバル変数への代入のみであり、他の関数呼び出しは行いません。

関連データ構造

  • binary_upgrade_next_heap_pg_class_relfilenumber: この関数によって値が設定されるグローバル変数（src/include/catalog/binary_upgrade.h で宣言）。

失敗モードと境界条件

  • binary_upgrade モード以外で呼び出された場合、CHECK_IS_BINARY_UPGRADE により処理が中断されます。

保守担当者向けメモ

  • 本関数はシステムカタログの物理的な整合性を直接操作する強力なツールです。誤った値を設定するとデータベースの破損を招くため、pg_upgrade 以外のコンテキストでの利用は厳禁です。
