lazy_scan_heap

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/vacuumlazy.c:1199

目的

VACUUM処理におけるヒープスキャンのメインループを実行し、不要なタプルの特定、タプルの凍結、およびインデックスクリーンアップのトリガーを行う。

シグネチャまたは形状

static void lazy_scan_heap(LVRelState *vacrel)

入力と出力

  • **入力**: LVRelState *vacrel（VACUUM実行中の状態を保持する構造体）
  • **出力**: なし（副作用としてヒープのメンテナンスと統計情報の更新を行う）

前提条件と不変条件

  • vacrel は初期化済みであること。
  • read_stream を使用してヒープを走査する。
  • TidStore の容量制限（max_bytes）に達した場合、インデックスのクリーンアップを優先する。

アルゴリズムまたは振る舞い

1. **初期化**: 進捗状況の報告を開始し、ヒープ走査用の read_stream を設定する。 2. **ループ処理**:

  • vacuum_delay_point を呼び出し、必要に応じて待機する。
  • 定期的に lazy_check_wraparound_failsafe を呼び出し、トランザクションID周回防止の安全装置を確認する。
  • TidStore が溢れそうな場合、lazy_vacuum を呼び出してインデックスとヒープのクリーンアップを実行し、FSM（空き領域マップ）を更新する。
  • read_stream_next_buffer で次のページを取得する。
  • visibilitymap_pin で可視性マップをピン留めする。
  • ConditionalLockBufferForCleanup を試み、失敗した場合は共有ロックを取得する。
  • lazy_scan_new_or_empty で新規または空のページを処理する。
  • lazy_scan_noprune でクリーンアップロックなしの処理を試み、失敗した場合はクリーンアップロックを取得して lazy_scan_prune へ移行する。

呼び出し元と呼び出し先

  • **呼び出し元**: heap_vacuum_rel（src/backend/access/heap/vacuumlazy.c:839）
  • **呼び出し先**: lazy_check_wraparound_failsafe, lazy_vacuum, FreeSpaceMapVacuumRange, read_stream_next_buffer, visibilitymap_pin, lazy_scan_new_or_empty, lazy_scan_noprune, lazy_scan_prune 等。

関連データ構造

  • LVRelState: VACUUM全体の状態管理。
  • TidStore: デッドタプルのTIDを保持。
  • ReadStream: ヒープ走査のストリーム制御。
  • Buffer: ヒープページへのアクセス用。

失敗モードと境界条件

  • **TidStoreの溢れ**: メモリ制限に達すると、スキャンを中断してインデックス掃除を優先する。
  • **ロック競合**: クリーンアップロックが取得できない場合、lazy_scan_noprune による限定的な処理を行うか、ロック取得まで待機する。
  • **トランザクションID周回**: lazy_check_wraparound_failsafe により、周回リスクが高い場合は強制的に処理を切り替える。

保守担当者向けメモ

  • 本関数はVACUUMの「フェーズI」を担う。
  • lazy_scan_prune への移行ロジックは、aggressive フラグやロックの可否に依存するため注意すること。
  • read_stream の使用により、ページ読み込みの効率化が図られている。
