gbt_int4eq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int4.c:37

目的

2つの int4（4バイト整数）値が等しいかどうかを判定し、GiST インデックスの比較処理をサポートします。

シグネチャまたは形状

static bool gbt_int4eq(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の int4 値へのポインタ。
  • b: 比較対象の int4 値へのポインタ。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • bool: a と b が等しい場合は true、それ以外は false を返します。

前提条件と不変条件

  • a および b は、有効な int32 型の値へのポインタである必要があります。

アルゴリズムまたは振る舞い

1. ポインタ a および b を const int32 * 型にキャストします。 2. それぞれが指す値をデリファレンスし、比較演算子 == を用いて等価性を評価します。 3. 評価結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_int4.c 内の tinfo 構造体定義（84行目）において、比較関数テーブルの一部として参照されています。
  • **呼び出し先**: 外部関数への呼び出しはありません。

関連データ構造

  • int32: PostgreSQL の 4バイト整数型。

失敗モードと境界条件

  • ポインタが NULL の場合、デリファレンスによりセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で提供される数値型 GiST ユーティリティから呼び出されることを想定した、型固有の比較ロジックです。
  • flinfo 引数は現在の実装では使用されていませんが、GiST のサポート関数インターフェースとの互換性のために保持されています。
