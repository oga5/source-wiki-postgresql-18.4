build_hash_table

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeRecursiveunion.c:31

目的

UNION（ALLなし）クエリにおいて、再帰的なデータセット内の重複タプルを排除するためのハッシュテーブルを初期化する。

シグネチャまたは形状

static void build_hash_table(RecursiveUnionState *rustate)

入力と出力

  • **入力**: RecursiveUnionState *rustate（再帰実行ノードの状態構造体）。
  • **出力**: なし（副作用として rustate->hashtable に初期化されたハッシュテーブルを格納する）。

前提条件と不変条件

  • node->numCols > 0 および node->numGroups > 0 であること（アサーションにより検証）。
  • rustate が有効な RecursiveUnion プランノードおよび実行状態を保持していること。

アルゴリズムまたは振る舞い

1. outerPlanState(rustate) から結果のタプル記述子（TupleDesc）を取得する。 2. BuildTupleHashTable を呼び出し、以下の情報を渡してハッシュテーブルを構築する。

  • 実行状態、タプル記述子、スロット操作関数。
  • 重複排除対象の列数（numCols）および列インデックス（dupColIdx）。
  • 等価比較関数（eqfuncoids）およびハッシュ関数（hashfunctions）。
  • 照合順序（dupCollations）およびグループ数（numGroups）。
  • メモリコンテキスト（クエリコンテキスト、テーブルコンテキスト、一時コンテキスト）。

    • 3. 構築されたハッシュテーブルを rustate->hashtable に割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeRecursiveunion.c 内の実行ロジックから呼び出される（ソースコード上の直接的な呼び出し元情報は提供されていない）。
  • **呼び出し先**: BuildTupleHashTable（ハッシュテーブル構築）、ExecGetResultType、ExecGetCommonChildSlotOps。

関連データ構造

  • RecursiveUnionState: 再帰実行ノードの状態を保持する構造体。
  • RecursiveUnion: プランノードの定義。
  • TupleHashTable: 構築されるハッシュテーブルの実体。

失敗モードと境界条件

  • numCols または numGroups が 0 以下の場合、アサーション違反により停止する。
  • メモリ不足時には BuildTupleHashTable 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は UNION 実行時の重複排除に特化している。UNION ALL の場合は呼び出されない。
  • メモリ管理は rustate->tableContext および rustate->tempContext に依存しているため、再スキャン時のメモリ解放挙動に注意すること。
