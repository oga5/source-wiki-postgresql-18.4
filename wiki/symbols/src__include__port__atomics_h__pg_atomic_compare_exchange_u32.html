pg_atomic_compare_exchange_u32

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/port/atomics.h:348

目的

32ビット符号なし整数のアトミックな比較交換（CAS: Compare-And-Swap）操作を実行します。ロックレスアルゴリズムにおいて、メモリ値が期待値と一致する場合のみ新しい値へ更新するために使用されます。

シグネチャまたは形状

bool pg_atomic_compare_exchange_u32(volatile pg_atomic_uint32 *ptr, uint32 *expected, uint32 newval);

入力と出力

  • ptr: 操作対象となるアトミック変数へのポインタ（入力）。
  • expected: 期待値が格納されたアドレス（入出力）。操作成功時は元の値が保持され、失敗時は現在の値で更新されます。
  • newval: 書き込みたい新しい値（入力）。
  • 戻り値: 比較が成功し、値が更新された場合に true を返し、失敗した場合は false を返します。

前提条件と不変条件

  • ptr および expected が指すアドレスは、4バイト境界にアライメントされている必要があります（AssertPointerAlignment により検証）。

アルゴリズムまたは振る舞い

1. ptr および expected のアライメントを検証します。 2. プラットフォーム固有の実装である pg_atomic_compare_exchange_u32_impl を呼び出し、実際の原子操作を実行します。 3. 戻り値として、操作の成否を呼び出し元に返します。

呼び出し元と呼び出し先

  • 呼び出し先: pg_atomic_compare_exchange_u32_impl
  • 呼び出し元（直接的な証拠）:
  • src/backend/access/transam/clog.c
  • src/backend/storage/buffer/bufmgr.c
  • src/backend/storage/buffer/freelist.c
  • src/backend/storage/ipc/procarray.c
  • src/backend/storage/lmgr/lwlock.c

関連データ構造

  • pg_atomic_uint32: アトミック操作対象の32ビット整数型。

失敗モードと境界条件

  • アライメントが4バイトでない場合、アサーションエラーが発生します。
  • 比較対象の値が expected と一致しない場合、操作は失敗し false を返します。この際、expected には現在の値が書き戻されます。

保守担当者向けメモ

  • 本関数は抽象化レイヤーであり、実際の挙動はコンパイル時に選択される arch-*.h または generic-*.h 内の _impl 関数に依存します。
  • 64ビット版のアトミック操作が必要な場合は、別途対応する関数を使用してください。
