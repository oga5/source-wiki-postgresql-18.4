heap_prune_chain

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/pruneheap.c:998

目的

HOT (Heap Only Tuple) チェーンを走査し、チェーン内のどのタプルを削除またはリダイレクトすべきかを決定する。

シグネチャまたは形状

static void heap_prune_chain(Page page, BlockNumber blockno, OffsetNumber maxoff, OffsetNumber rootoffnum, PruneState *prstate)

入力と出力

  • **入力**:
  • page: 対象のヒープページ。
  • blockno: ページ番号。
  • maxoff: ページ内の最大オフセット。
  • rootoffnum: チェーンの起点となるオフセット。
  • prstate: 剪定処理の状態を保持する PruneState 構造体。
  • **出力**: なし（prstate を通じて削除・リダイレクト情報を記録する）。

前提条件と不変条件

  • rootoffnum が指すアイテムは使用中であること。
  • prstate->processed 配列により、既に処理済みのアイテムはスキップされる。
  • チェーン内のタプルは、priorXmax（直前のタプルの更新XID）と自身の Xmin が一致している必要がある。

アルゴリズムまたは振る舞い

1. **チェーン走査**: 起点から t_ctid を辿り、HOT チェーンを順次走査する。 2. **可視性判定**: htsv_get_valid_status を使用し、各タプルの状態（DEAD, LIVE 等）を判定する。 3. **DEAD タプルの追跡**: 最後に確認された DEAD タプルの位置を ndeadchain に記録する。 4. **分岐処理**:

  • **DEAD タプルなし**: チェーン全体が有効な場合、変更なしとして記録。
  • **チェーン全体が DEAD**: ルートを LP_DEAD に設定し、チェーン全体を削除対象として記録。
  • **一部が DEAD**: DEAD タプル以降のチェーンを切り離し、リダイレクト先を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/pruneheap.c 内の heap_page_prune_and_freeze から呼び出される。
  • **呼び出し先**: heap_prune_record_dead_or_unused, heap_prune_record_unchanged_lp_redirect, heap_prune_record_unchanged_lp_normal 等の記録用関数。

関連データ構造

  • PruneState: 剪定処理の進捗と結果を保持。
  • HeapTupleHeader: タプルのヘッダー情報（t_ctid, Xmin 等）。
  • ItemId: ページ内のラインポインタ。

失敗モードと境界条件

  • offnum が FirstOffsetNumber 未満、または maxoff を超える場合は走査を停止する。
  • 予期しない HeapTupleSatisfiesVacuum の結果が返された場合、elog(ERROR) を発生させる。
  • リダイレクト先が不正な場合、heap_prune_record_dead_or_unused を呼び出し、整合性を維持する。

保守担当者向けメモ

  • 本関数は HOT チェーンの論理的な剪定決定を行う。物理的なページ変更は呼び出し元の heap_page_prune_and_freeze が行う heap_page_prune_execute に委ねられる。
  • ndeadchain の管理は、VACUUM が誤ったタプルを削除しないようにするための重要な不変条件である。
