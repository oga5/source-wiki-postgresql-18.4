ModifyWaitEvent

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/waiteventset.c:655

目的

WaitEventSet 内に既に登録されているイベントの監視条件（イベントマスクや関連付けられたラッチ）を変更する。

シグネチャまたは形状

void ModifyWaitEvent(WaitEventSet *set, int pos, uint32 events, Latch *latch)

入力と出力

  • set: 変更対象の WaitEventSet 構造体へのポインタ。
  • pos: 変更するイベントのインデックス（0 から set->nevents - 1 の範囲）。
  • events: 新しいイベントマスク（WL_* フラグの組み合わせ）。
  • latch: イベントに関連付ける Latch 構造体（WL_LATCH_SET 指定時のみ使用）。
  • 戻り値: なし（void）。

前提条件と不変条件

  • pos は set->nevents 未満であること。
  • WL_LATCH_SET を指定する場合、latch は呼び出し元プロセスが所有している必要がある。
  • ポストマスター死滅イベント（WL_POSTMASTER_DEATH）を削除することはできない。

アルゴリズムまたは振る舞い

1. **早期リターン**: イベントマスクとラッチに変更がない場合は即座に終了する。 2. **特殊処理**:

  • WL_POSTMASTER_DEATH イベントの場合、WL_EXIT_ON_PM_DEATH フラグの更新のみを行い終了する。
  • WL_LATCH_SET イベントの場合、set->latch を更新し、プラットフォーム固有の処理（Unix系ではパイプが共通のため即時終了、Windowsではハンドル更新）を行う。

    • 3. **OS 固有の更新**:

  • 上記以外の場合、プラットフォームに応じて WaitEventAdjustEpoll、WaitEventAdjustKqueue、WaitEventAdjustPoll、または WaitEventAdjustWin32 を呼び出し、カーネル側の監視状態を同期させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-secure.c
  • src/backend/libpq/pqcomm.c
  • src/backend/replication/walsender.c
  • src/backend/storage/ipc/latch.c
  • src/backend/utils/init/miscinit.c
  • **呼び出し先**:
  • WaitEventAdjustEpoll / WaitEventAdjustKqueue / WaitEventAdjustPoll / WaitEventAdjustWin32（プラットフォーム依存）
  • elog（エラー発生時）

関連データ構造

  • WaitEventSet: 待機イベント群を管理する構造体。
  • WaitEvent: 個別のイベント情報を保持する構造体。
  • Latch: プロセス間通信や起床に使用される同期プリミティブ。

失敗モードと境界条件

  • pos が範囲外の場合: Assert により停止。
  • 他プロセス所有のラッチを指定した場合: ERROR をスロー。
  • 既存のラッチイベントに対して異なるイベントマスクを指定した場合: ERROR をスロー。
  • ポストマスター死滅イベントを削除しようとした場合: ERROR をスロー。

保守担当者向けメモ

  • 本関数はソケットの読み取り/書き込み切り替えなどで頻繁に呼び出されるため、変更がない場合の早期リターンは重要な最適化となっている。
  • WAIT_USE_KQUEUE 等のコンパイル時フラグにより、OS 固有の調整ロジックが切り替わる点に注意すること。
