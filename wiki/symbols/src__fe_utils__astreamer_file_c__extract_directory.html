extract_directory

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_file.c:333

目的

アーカイブ展開処理において、指定されたパスにディレクトリを作成し、適切な権限を適用します。

シグネチャまたは形状

static void extract_directory(const char *filename, mode_t mode)

入力と出力

  • filename (const char *): 作成対象のディレクトリパス。
  • mode (mode_t): ディレクトリに適用するファイルパーミッション。
  • 戻り値: なし（成功時）。失敗時は pg_fatal によりプロセスを終了します。

前提条件と不変条件

  • pg_dir_create_mode を使用してディレクトリを作成します。
  • 非Windows環境では、作成後に chmod を用いて指定された mode を適用します。

アルゴリズムまたは振る舞い

1. mkdir を呼び出し、pg_dir_create_mode でディレクトリを作成します。 2. 作成に失敗した場合、エラーが EEXIST（既に存在）であり、かつ should_allow_existing_directory(filename) が真を返す場合は処理を継続します。それ以外の場合は pg_fatal で異常終了します。 3. 非Windows環境の場合、chmod を呼び出し、引数で指定された mode をディレクトリに適用します。失敗した場合は pg_fatal で異常終了します。

呼び出し元と呼び出し先

  • 呼び出し元: src/fe_utils/astreamer_file.c 内の astreamer_extractor_content（推測される文脈）。
  • 呼び出し先: mkdir, should_allow_existing_directory, pg_fatal, chmod（非Windows環境のみ）。

関連データ構造

  • mode_t: POSIX標準のファイルモード型。

失敗モードと境界条件

  • ディレクトリ作成失敗時（権限不足やパス不正など）は pg_fatal により終了します。
  • 既存ディレクトリが存在する場合、should_allow_existing_directory の判定結果に依存して動作が分岐します。

保守担当者向けメモ

  • 本関数は astreamer_extractor の一部として動作します。
  • Windows環境では chmod がスキップされるため、パーミッション設定の挙動がプラットフォーム間で異なる点に注意してください。
