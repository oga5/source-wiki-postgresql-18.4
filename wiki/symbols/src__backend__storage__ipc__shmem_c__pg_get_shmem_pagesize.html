pg_get_shmem_pagesize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shmem.c:718

目的

現在のシステムにおける共有メモリのページサイズ（またはHuge Pageが有効な場合はそのサイズ）を取得します。

シグネチャまたは形状

Size pg_get_shmem_pagesize(void);

入力と出力

  • 入力: なし
  • 出力: Size 型のページサイズ（バイト単位）

前提条件と不変条件

  • IsUnderPostmaster が真であること（Postmaster配下で実行されていること）。
  • huge_pages_status が HUGE_PAGES_UNKNOWN ではないこと。

アルゴリズムまたは振る舞い

1. OSの標準ページサイズを取得します。

  • Windows環境では GetSystemInfo を使用します。
  • それ以外の環境では sysconf(_SC_PAGESIZE) を使用します。

    • 2. huge_pages_status が HUGE_PAGES_ON である場合、GetHugePageSize を呼び出し、取得したページサイズをHuge Pageサイズで上書きします。 3. 最終的なページサイズを返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/pg_buffercache/pg_buffercache_pages.c:347
  • src/backend/storage/ipc/shmem.c:581
  • 呼び出し先:
  • GetSystemInfo (Windows環境)
  • sysconf (非Windows環境)
  • GetHugePageSize (Huge Page有効時)

関連データ構造

  • Size (PostgreSQLのメモリサイズ型)
  • huge_pages_status (グローバル変数)

失敗モードと境界条件

  • huge_pages_status が初期化されていない状態で呼び出された場合、Assert により停止します。
  • IsUnderPostmaster が偽の状態で呼び出された場合、Assert により停止します。

保守担当者向けメモ

  • 本関数は src/include/storage/shmem.h で公開されています。
  • Huge Pageのサイズ取得ロジックは GetHugePageSize に依存しているため、Huge Page関連の挙動を変更する場合はそちらも確認してください。
