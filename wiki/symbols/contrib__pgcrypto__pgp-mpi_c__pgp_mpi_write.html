pgp_mpi_write

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-mpi.c:104

目的

OpenPGP 形式の多倍長整数（MPI）を、指定された出力ストリーム（PushFilter）へシリアライズして書き出します。

シグネチャまたは形状

int pgp_mpi_write(PushFilter *dst, PGP_MPI *n)

入力と出力

  • **入力**:
  • dst: 書き込み先の PushFilter 構造体へのポインタ。
  • n: 書き出す対象の PGP_MPI 構造体へのポインタ。
  • **出力**:
  • int: 書き込み成功時は 0 以上の値、失敗時は負の値を返します。

前提条件と不変条件

  • n が指す PGP_MPI 構造体は有効であり、bits（ビット長）と bytes（バイト長）、および data（データ本体）が正しく初期化されている必要があります。
  • dst は書き込み可能な状態である必要があります。

アルゴリズムまたは振る舞い

1. n->bits をビッグエンディアン形式の 2 バイトデータとして buf に格納します。 2. pushf_write を呼び出し、ヘッダー（2 バイト）を dst に書き出します。 3. ヘッダーの書き込みが成功した場合（res >= 0）、続けて n->data から n->bytes 分のデータを dst に書き出します。 4. 最終的な書き込み結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-pubenc.c（公開鍵暗号処理において、暗号化された MPI データの出力に使用）
  • **呼び出し先**:
  • pushf_write（contrib/pgcrypto/px.h で定義されるストリーム書き込み関数）

関連データ構造

  • PGP_MPI: MPI データを保持する構造体（ビット数、バイト数、データ本体へのポインタ）。
  • PushFilter: 出力ストリームを抽象化する構造体。

失敗モードと境界条件

  • pushf_write がエラーを返した場合、そのエラーコードがそのまま本関数の戻り値となります。
  • n->data が NULL である場合や、n->bytes が実際のデータ長と一致しない場合、未定義の動作やメモリ読み取りエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は MPI のデータ構造をそのままストリームへ流し込む役割を担います。数学的な演算は行いません。
  • 書き込み順序は OpenPGP 仕様に従い、ヘッダー（ビット長）の後にデータ本体が続きます。
