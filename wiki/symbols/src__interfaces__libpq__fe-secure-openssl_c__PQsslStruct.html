PQsslStruct

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure-openssl.c:1633

目的

PGconn オブジェクトに関連付けられた OpenSSL 固有の SSL 構造体へのポインタを取得する。

シグネチャまたは形状

void *PQsslStruct(PGconn *conn, const char *struct_name);

入力と出力

  • **入力**:
  • conn: 接続状態を保持する PGconn 構造体へのポインタ。
  • struct_name: 取得対象の構造体名を示す文字列。現在は "OpenSSL" のみがサポートされている。
  • **出力**:
  • struct_name が "OpenSSL" であり、かつ conn が有効な場合、conn->ssl（OpenSSL の SSL オブジェクト）を返す。
  • それ以外（conn が NULL、または未知の struct_name）の場合は NULL を返す。

前提条件と不変条件

  • conn が NULL の場合、関数は安全に NULL を返す。
  • struct_name に "OpenSSL" 以外が指定された場合、関数は NULL を返す。

アルゴリズムまたは振る舞い

1. conn が NULL かどうかを判定し、NULL であれば即座に NULL を返す。 2. struct_name が文字列 "OpenSSL" と一致するかを strcmp で比較する。 3. 一致する場合、conn->ssl を返す。 4. 一致しない場合、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq/fe-secure.c:434
  • src/interfaces/libpq/fe-secure-openssl.c:1634（再帰的または内部的な参照の可能性あり）
  • **呼び出し先**:
  • strcmp (標準ライブラリ)

関連データ構造

  • PGconn: libpq の接続状態を管理する構造体。conn->ssl を通じて OpenSSL の SSL オブジェクトを保持する。

失敗モードと境界条件

  • conn が NULL の場合、セグメンテーションフォールトを回避し NULL を返す。
  • struct_name が NULL の場合、strcmp の動作により未定義の挙動となる可能性がある（ソースコード上はガードされていない）。

保守担当者向けメモ

  • 本関数は OpenSSL 固有の内部構造体へのアクセスを提供するインターフェースである。将来的に他の SSL ライブラリ（GSSAPI 等）をサポートする場合、struct_name の判定ロジックを拡張する必要がある。
