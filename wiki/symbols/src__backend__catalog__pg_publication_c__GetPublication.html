GetPublication

定義

  • **種別**: function
  • **定義箇所**: src/backend/catalog/pg_publication.c:1069

目的

システムカタログ pg_publication から指定された OID に対応するパブリケーション情報を取得し、メモリ上の Publication 構造体に格納して返します。

シグネチャまたは形状

Publication *GetPublication(Oid pubid)

入力と出力

  • **入力**: pubid (Oid) - 取得対象のパブリケーションの OID。
  • **出力**: Publication * - 取得したパブリケーション情報を保持するメモリ割り当て済みの構造体。

前提条件と不変条件

  • PUBLICATIONOID システムキャッシュが利用可能であること。
  • 指定された pubid がシステムカタログに存在すること（存在しない場合はエラーをスロー）。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用して PUBLICATIONOID キャッシュから対象のタプルを検索します。 2. タプルが見つからない場合、elog(ERROR) を発生させます。 3. GETSTRUCT を介してタプルから Form_pg_publication 形式のデータを抽出します。 4. palloc で Publication 構造体用のメモリを確保し、以下のフィールドをコピーします。

  • oid, name (文字列コピー), alltables
  • pubactions (insert, update, delete, truncate)
  • pubviaroot, pubgencols_type

    • 5. ReleaseSysCache を呼び出してキャッシュ参照を解放します。 6. 構築した構造体へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_publication.c 内の複数箇所（行 439, 677, 1108, 1249）
  • src/backend/commands/publicationcmds.c:395
  • **呼び出し先**:
  • SearchSysCache1, HeapTupleIsValid, GETSTRUCT, palloc, pstrdup, NameStr, ReleaseSysCache, elog

関連データ構造

  • Publication: パブリケーションの属性を保持する構造体。
  • Form_pg_publication: pg_publication システムカタログのタプル構造。

失敗モードと境界条件

  • 指定された pubid が存在しない場合、elog(ERROR) により処理が中断されます。
  • メモリ不足時には palloc がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数はシステムキャッシュを利用するため、カタログ更新時にはキャッシュの無効化が適切に行われていることを確認してください。
  • Publication 構造体のフィールド追加・変更時は、本関数内のコピー処理も併せて更新する必要があります。
