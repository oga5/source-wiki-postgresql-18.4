make_one_rel

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/allpaths.c:170

目的

クエリ全体の結合ツリーに基づき、最適な実行パス（RelOptInfo）を生成するオプティマイザのメインエントリポイントです。

シグネチャまたは形状

RelOptInfo *make_one_rel(PlannerInfo *root, List *joinlist)

入力と出力

  • **入力**:
  • root: プランナのグローバル状態を保持する PlannerInfo 構造体。
  • joinlist: 結合対象となるリレーションのリスト。
  • **出力**:
  • クエリ全体を網羅する結合済みの RelOptInfo 構造体。

前提条件と不変条件

  • root->simple_rel_array が初期化されていること。
  • 戻り値の rel->relids は、クエリ内のすべてのベースリレーションおよび外部結合リレーションと一致しなければならない（Assert による検証）。

アルゴリズムまたは振る舞い

1. **初期化**: set_base_rel_consider_startup を呼び出し、各ベースリレーションのスタートアップコスト考慮フラグを設定。 2. **サイズ見積もり**: set_base_rel_sizes により、各ベースリレーションのサイズ見積もりと並列実行可否を計算。 3. **統計集計**: root->simple_rel_array を走査し、ダミーリレーションを除外した上で、クエリ全体で使用される total_table_pages を算出。 4. **パス生成（ベース）**: set_base_rel_pathlists を呼び出し、各ベースリレーションに対するアクセスパスを生成。 5. **パス生成（結合）**: make_rel_from_joinlist を呼び出し、結合ツリー全体のアクセスパスを生成。 6. **検証と返却**: 最終的な RelOptInfo がクエリ全体をカバーしていることを確認し、返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planmain.c（プランナのメイン処理）から呼び出される。
  • **呼び出し先**:
  • set_base_rel_consider_startup
  • set_base_rel_sizes
  • set_base_rel_pathlists
  • make_rel_from_joinlist

関連データ構造

  • PlannerInfo: プランナの全体状態。
  • RelOptInfo: リレーションのパスやコスト情報を保持する構造体。

失敗モードと境界条件

  • brel == NULL の場合、そのスロットはスキップされる。
  • IS_DUMMY_REL(brel) が真の場合、そのリレーションは total_table_pages の計算から除外される。
  • 自己結合（Self-join）が発生している場合、同一テーブルが複数回カウントされる可能性がある。

保守担当者向けメモ

  • 本関数はクエリ最適化のパス生成フェーズにおける最上位の制御関数です。
  • 外部プラグインがパス生成に介入する場合、本関数内で呼び出される各サブ関数（set_rel_pathlist 等）にフックが仕込まれている点に注意してください。
