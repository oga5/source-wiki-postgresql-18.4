pg_buffercache_evict_relation

定義

  • 種別: function
  • 定義箇所: contrib/pg_buffercache/pg_buffercache_pages.c:710

目的

指定されたリレーションに関連付けられた共有バッファキャッシュ内のバッファを強制的に解放（エビクション）し、その結果（解放数、フラッシュ数、スキップ数）を返します。

シグネチャまたは形状

Datum pg_buffercache_evict_relation(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介してリレーションの OID を受け取ります。
  • **出力**: Datum 型として、解放されたバッファ数、フラッシュされたバッファ数、スキップされたバッファ数を含む複合型（行）を返します。

前提条件と不変条件

  • 呼び出し元はスーパーユーザーである必要があります（pg_buffercache_superuser_check による検証）。
  • 対象のリレーションは共有バッファを使用している必要があります。ローカルバッファを使用するリレーションが指定された場合、エラーが発生します。
  • リレーションのオープンには AccessShareLock が使用されます。

アルゴリズムまたは振る舞い

1. 戻り値の型が複合型であることを確認します。 2. スーパーユーザー権限を確認します。 3. 指定された OID からリレーションをオープンします。 4. RelationUsesLocalBuffers を使用して、対象が共有バッファであることを検証します。 5. EvictRelUnpinnedBuffers を呼び出し、リレーションに関連するピン留めされていないバッファを解放します。 6. リレーションをクローズします。 7. 解放結果をタプルに格納し、Datum として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • pg_buffercache_superuser_check: 権限確認。
  • relation_open / relation_close: リレーションの管理。
  • EvictRelUnpinnedBuffers: バッファ解放のコアロジック。
  • heap_form_tuple: 戻り値の構築。

関連データ構造

  • TupleDesc: 戻り値の行定義。
  • HeapTuple: 戻り値のタプル。
  • Relation: 操作対象のリレーション情報。

失敗モードと境界条件

  • **権限エラー**: スーパーユーザーでない場合、pg_buffercache_superuser_check によりエラーが発生します。
  • **型エラー**: 戻り値の型が複合型でない場合、elog(ERROR) が発生します。
  • **リレーション種別エラー**: ローカルバッファを使用するリレーションを指定した場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーが発生します。

保守担当者向けメモ

  • 本関数は共有バッファのみを対象としています。
  • EvictRelUnpinnedBuffers の実行結果をそのまま SQL レイヤーへ渡す設計となっています。
