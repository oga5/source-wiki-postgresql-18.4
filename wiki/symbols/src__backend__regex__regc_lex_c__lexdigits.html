lexdigits

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_lex.c:779

目的

正規表現内のエスケープシーケンス等に含まれる数値表現を読み取り、対応する文字コード（chr）値を算出する。

シグネチャまたは形状

chr lexdigits(struct vars *v, int base, int minlen, int maxlen)

入力と出力

  • **入力**:
  • v: 解析状態を保持する vars 構造体へのポインタ。v->now から入力文字列を消費する。
  • base: 数値の基数（8, 10, 16など）。
  • minlen: 必要な最小桁数。
  • maxlen: 許容される最大桁数。
  • **出力**:
  • 読み取った数値に対応する chr 型の値。
  • **エラー**:
  • 読み取った桁数が minlen 未満の場合、ERR(REG_EESCAPE) を介してエラーを通知する。

前提条件と不変条件

  • v->now は解析対象の文字列を指しており、呼び出しごとに前進する。
  • base は 16 以下を想定している（a-f / A-F の判定ロジックに基づく）。
  • 内部計算には uchr 型を使用し、オーバーフローを回避する。

アルゴリズムまたは振る舞い

1. n を 0 に初期化する。 2. maxlen に達するか、文字列終端（ATEOS()）に達するまでループする。 3. 現在の文字を読み取り、base に応じた数値 d に変換する。

  • 0-9 は DIGITVAL(c) で変換。
  • a-f / A-F は 10-15 に変換。
  • 該当しない文字や base 以上の値が出現した場合はループを終了する。

    • 4. n = n * base + d の計算を行い、結果を蓄積する。 5. 読み取った桁数が minlen 未満であればエラーを発生させる。 6. 最終的な n を chr 型にキャストして返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regc_lex.c 内の各所（エスケープシーケンス解析処理）。
  • **呼び出し先**:
  • ATEOS()（マクロ）
  • DIGITVAL()（マクロ）
  • ERR()（マクロ）

関連データ構造

  • struct vars: 解析状態を保持する構造体。
  • chr: 文字コードを表す型。
  • uchr: chr の符号なし型。

失敗モードと境界条件

  • **桁数不足**: minlen に満たない場合は REG_EESCAPE エラーとなる。
  • **無効な文字**: base に適合しない文字が出現した時点で読み取りを停止し、その文字は v->now-- によって消費されなかったものとして扱われる。
  • **オーバーフロー**: uchr を使用することで、計算過程での意図しないオーバーフローを防止している。

保守担当者向けメモ

  • 本関数は regc_lex.c 内で静的に定義されており、regcomp.c からも参照される。
  • v->now-- によるポインタの巻き戻しは、トークン解析の整合性を保つために重要である。
  • base が 16 を超えるような拡張を行う場合は、switch 文のケース追加が必要となる。
