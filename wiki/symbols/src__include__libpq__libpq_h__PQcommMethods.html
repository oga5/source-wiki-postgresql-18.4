PQcommMethods

定義

  • 種別: struct
  • 定義箇所: src/include/libpq/libpq.h:33

目的

PostgreSQL バックエンドにおけるフロントエンド・バックエンド間の通信操作を抽象化し、統一的なインターフェースを提供します。具体的な通信実装（TCP/IP、Unixドメインソケット、メッセージキュー等）の詳細を隠蔽するために使用されます。

シグネチャまたは形状

typedef struct
{
	void		(*comm_reset) (void);
	int			(*flush) (void);
	int			(*flush_if_writable) (void);
	bool		(*is_send_pending) (void);
	int			(*putmessage) (char msgtype, const char *s, size_t len);
	void		(*putmessage_noblock) (char msgtype, const char *s, size_t len);
} PQcommMethods;

入力と出力

  • comm_reset: 引数なし、戻り値なし。通信状態をリセットします。
  • flush / flush_if_writable: 引数なし、戻り値は int（成功時は0、失敗時はEOF等）。バッファのフラッシュを行います。
  • is_send_pending: 引数なし、戻り値は bool。送信保留中のデータがあるかを示します。
  • putmessage / putmessage_noblock: メッセージ型（char）、データ本体（const char *）、長さ（size_t）を入力とし、putmessage は結果を int で返します。

前提条件と不変条件

  • PqCommMethods グローバル変数を通じて、現在有効な実装が参照される必要があります。
  • 通信メソッドの呼び出しは、接続が確立された状態で行われることが想定されています。

アルゴリズムまたは振る舞い

通信の抽象化レイヤーとして機能します。上位モジュールは具体的な実装を意識することなく、本構造体の関数ポインタを介してメッセージの送受信やバッファのフラッシュを行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/libpq/pqcomm.c および src/backend/libpq/pqmq.c にて、それぞれ PqCommSocketMethods および PqCommMqMethods として実装が定義されています。
  • 呼び出し先: PqCommMethods グローバル変数を介して、実行時に選択された通信メソッドの実装関数が呼び出されます。

関連データ構造

  • PqCommMethods: 現在有効な通信メソッドの実装を指すグローバル変数。
  • StringInfo: メッセージの読み込みやバッファ管理に使用されるデータ構造。

失敗モードと境界条件

  • flush 系の関数は、ソケットエラーや書き込み失敗時にエラーコードを返します。
  • putmessage は、メッセージサイズが制限を超えた場合や通信エラー発生時に失敗する可能性があります。

保守担当者向けメモ

  • 本構造体はバックエンド側の通信定義です。クライアント側の libpq-fe.h と混同しないよう注意してください。
  • 新しい通信プロトコルや実装を追加する場合は、本構造体の全関数ポインタを実装したインスタンスを作成し、PqCommMethods を切り替える必要があります。
