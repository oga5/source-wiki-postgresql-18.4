pgp_s2k_process

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-s2k.c:278

目的

OpenPGP仕様（RFC 4880）に基づき、パスフレーズから暗号化アルゴリズム用のバイナリ鍵を導出（S2K: String-to-Key）します。

シグネチャまたは形状

int pgp_s2k_process(PGP_S2K *s2k, int cipher, const uint8 *key, int key_len);

入力と出力

  • **入力**:
  • s2k: S2K設定情報（モード、ダイジェストアルゴリズム等）を保持する構造体。
  • cipher: 使用する暗号アルゴリズムの識別子。
  • key: パスフレーズ文字列。
  • key_len: パスフレーズの長さ。
  • **出力**: 成功時は 0、失敗時は負の PXE_* エラーコードを返します。

前提条件と不変条件

  • s2k->digest_algo が有効なハッシュアルゴリズムを指している必要があります。
  • cipher に対応する鍵長が pgp_get_cipher_key_size で正しく取得できる必要があります。

アルゴリズムまたは振る舞い

1. pgp_get_cipher_key_size を呼び出し、指定された cipher に必要な鍵長を s2k->key_len に設定します。 2. pgp_load_digest を使用して、指定されたダイジェストアルゴリズムのコンテキスト（PX_MD）を初期化します。 3. s2k->mode に応じて以下のいずれかの鍵導出関数を呼び出します。

  • 0: calc_s2k_simple
  • 1: calc_s2k_salted
  • 3: calc_s2k_iter_salted

    • 4. 処理終了後、px_md_free を呼び出してダイジェストコンテキストを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-decrypt.c
  • contrib/pgcrypto/pgp-encrypt.c
  • contrib/pgcrypto/pgp-pubkey.c
  • **呼び出し先**:
  • pgp_get_cipher_key_size
  • pgp_load_digest
  • calc_s2k_simple
  • calc_s2k_salted
  • calc_s2k_iter_salted
  • px_md_free

関連データ構造

  • PGP_S2K: S2Kのモードやパラメータを保持する構造体。
  • PX_MD: ハッシュ計算用インターフェース。

失敗モードと境界条件

  • pgp_get_cipher_key_size が無効な値を返した場合、PXE_PGP_UNSUPPORTED_CIPHER を返します。
  • pgp_load_digest が失敗した場合、そのエラーコードを返します。
  • s2k->mode が定義された値（0, 1, 3）以外の場合、PXE_PGP_BAD_S2K_MODE を返します。

保守担当者向けメモ

  • 本関数はS2K処理のディスパッチャーとして機能します。新しいS2Kモードを追加する場合は、switch 文にケースを追加してください。
  • メモリリークを防ぐため、px_md_free は必ず呼び出されるパスを通るように設計されています。
