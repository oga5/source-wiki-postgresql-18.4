DecodeDate

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/dt_common.c:1307

目的

日付文字列を解析し、struct tm 構造体へ分解して格納する。

シグネチャまたは形状

int DecodeDate(char *str, int fmask, int *tmask, struct tm *tm, bool EuroDates)

入力と出力

  • **入力**:
  • str: 解析対象の日付文字列。
  • fmask: 既に設定済みのフィールドマスク。
  • EuroDates: 欧州形式（DD/MM/YYYY）を優先するかどうかのフラグ。
  • **出力**:
  • tmask: 解析により設定されたフィールドマスク。
  • tm: 解析結果が格納される struct tm 構造体。
  • 戻り値: 成功時は 0、失敗時は -1。

前提条件と不変条件

  • 入力文字列は MAXDATEFIELDS（ソースコード上の定義値）以下のフィールド数である必要がある。
  • AD/BC 表記において「0年」は存在せず、1 BC は 0 年として扱われる。
  • 2桁の年表記は、70未満であれば 2000 年代、それ以外は 1900 年代として解釈される。

アルゴリズムまたは振る舞い

1. **トークン化**: 入力文字列を非英数字を区切り文字として分割し、各フィールドを抽出する。 2. **テキスト解析**: まず英字を含むフィールドを DecodeSpecial で解析し、月名や AD/BC 指定を tm 構造体に反映させる。 3. **数値解析**: 残りの数値フィールドを DecodeNumber で解析し、年・月・日の各要素を特定する。 4. **検証**: 解析されたフィールドの組み合わせが日付として妥当か（DTK_DATE_M マスクとの照合）を確認する。 5. **補正**: AD/BC フラグに基づき、年を正負の数値に変換する。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の src/interfaces/ecpg/pgtypeslib/dt_common.c（1907行目、2095行目）から呼び出されている。
  • **呼び出し先**: DecodeSpecial および DecodeNumber を呼び出す。
  • 注: src/backend/utils/adt/datetime.c にも同名の関数が存在するが、本シンボルとの直接的な呼び出し関係はソースコード上からは確認できない。

関連データ構造

  • struct tm: 日付・時刻情報を保持する標準構造体。
  • MAXDATEFIELDS: 解析可能な最大フィールド数。

失敗モードと境界条件

  • フィールド数が多すぎる場合、または解析不能な形式の場合は -1 を返す。
  • 既に設定済みのフィールドと重複する情報が入力された場合は -1 を返す。
  • 0年（AD/BC表記の境界）の扱いに起因する不正な年指定はエラーとなる。

保守担当者向けメモ

  • 本関数は EuroDates フラグに依存して数値の解釈順序を切り替えるため、ロジック変更時は DecodeNumber との整合性に注意すること。
  • fmask によるフィールド重複チェックは厳密に行われており、解析の順序やマスクの定義変更は副作用を伴う可能性がある。
