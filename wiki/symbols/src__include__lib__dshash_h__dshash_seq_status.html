dshash_seq_status

定義

  • 種別: struct
  • 定義箇所: src/include/lib/dshash.h:72

目的

dshash（動的共有ハッシュテーブル）に対するシーケンシャルスキャン実行時の状態を保持するための構造体です。

シグネチャまたは形状

typedef struct dshash_seq_status
{
	dshash_table *hash_table;
	int			curbucket;
	int			nbuckets;
	dshash_table_item *curitem;
	dsa_pointer pnextitem;
	int			curpartition;
	bool		exclusive;
} dshash_seq_status;

入力と出力

本構造体は、スキャン操作の各ステップで更新される内部状態を保持します。

  • hash_table: 対象となるハッシュテーブルへのポインタ。
  • curbucket, curpartition: 現在走査中のバケットおよびパーティション番号。
  • curitem: 現在参照しているアイテムへのポインタ。
  • pnextitem: 次のアイテムを指す dsa_pointer。
  • exclusive: ロックモード（排他制御の有無）。

前提条件と不変条件

  • 内部メンバへの直接アクセスは推奨されません。提供されている API（dshash_seq_init 等）を通じて操作する必要があります。
  • スキャン中は、対象となるハッシュテーブルのパーティションロックが適切に管理される必要があります。

アルゴリズムまたは振る舞い

ハッシュテーブル全体をバケット単位およびパーティション単位で順次走査します。dshash_seq_init で初期化され、dshash_seq_next で次のアイテムへ進み、dshash_seq_term で終了処理を行います。スキャン中に dshash_delete_current を呼び出すことで、現在位置のアイテムを安全に削除可能です。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/lib/dshash.c（スキャン操作の実装）
  • src/backend/utils/activity/pgstat.c
  • src/backend/utils/activity/pgstat_shmem.c
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、関数呼び出しは行いません。操作関数（dshash_seq_init 等）が本構造体を引数として受け取り、操作を行います。

関連データ構造

  • dshash_table: スキャン対象のハッシュテーブル本体。
  • dshash_table_item: ハッシュテーブル内の各要素。
  • dsa_pointer: 共有メモリ上のアイテムを指すポインタ。

失敗モードと境界条件

  • スキャン中にハッシュテーブルの構造が変更（リサイズ等）された場合、スキャンの整合性が損なわれる可能性があります。
  • dshash_seq_init を呼び出す前に構造体を初期化していない場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本構造体は dshash.c の内部実装と密接に関連しています。構造体のメンバを変更する場合は、スキャンロジック全体への影響を考慮してください。
  • pgstat_shmem.c 等で頻繁に使用されているため、変更時はこれらのモジュールへの影響も確認してください。
