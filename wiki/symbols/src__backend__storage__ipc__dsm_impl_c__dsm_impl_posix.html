dsm_impl_posix

定義

  • **種別**: function
  • **定義箇所**: src/backend/storage/ipc/dsm_impl.c:211

目的

POSIX共有メモリAPI（shm_open, mmap等）を使用して、動的共有メモリ（DSM）セグメントの作成、アタッチ、デタッチ、および破棄を行う低レベル実装を提供します。

シグネチャまたは形状

static bool dsm_impl_posix(dsm_op op, dsm_handle handle, Size request_size,
                           void **impl_private, void **mapped_address, Size *mapped_size,
                           int elevel);

入力と出力

  • **入力**:
  • op: 実行する操作（DSM_OP_CREATE, DSM_OP_ATTACH, DSM_OP_DETACH, DSM_OP_DESTROY）。
  • handle: セグメントを一意に識別するハンドル。
  • request_size: セグメントの要求サイズ。
  • elevel: エラー発生時のログレベル。
  • **入出力**:
  • impl_private: 実装固有のデータ（本関数では未使用）。
  • mapped_address: マッピングされたメモリの開始アドレス。
  • mapped_size: マッピングされたメモリのサイズ。
  • **出力**: 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • セグメント名は /PostgreSQL.<handle> という形式で生成されます。
  • ReserveExternalFD() および ReleaseExternalFD() を使用して、ファイル記述子の枯渇を防ぐためのリソース管理が行われます。

アルゴリズムまたは振る舞い

1. **終了処理 (DETACH / DESTROY)**:

  • munmap を実行し、メモリマッピングを解除します。
  • DESTROY の場合、shm_unlink を呼び出して共有メモリセグメントを削除します。

    • 2. **作成・アタッチ処理**:

  • shm_open で共有メモリオブジェクトを開きます。
  • ATTACH の場合、fstat で既存セグメントのサイズを取得します。
  • CREATE の場合、dsm_impl_posix_resize を呼び出してセグメントサイズを確保します。
  • mmap を使用してプロセスのアドレス空間にマッピングします。
  • 成功後、ファイル記述子を閉じ、リソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/dsm_impl.c 内のディスパッチャ（dsm_impl_op）から呼び出されます。
  • **呼び出し先**: shm_open, shm_unlink, mmap, munmap, fstat, dsm_impl_posix_resize, ReserveExternalFD, ReleaseExternalFD 等のシステムコールおよび補助関数。

関連データ構造

  • dsm_op: 操作種別を定義する列挙型。
  • dsm_handle: セグメント識別子。

失敗モードと境界条件

  • shm_open が失敗した場合（例: 権限不足、リソース不足）、エラーを報告し false を返します。
  • mmap が MAP_FAILED を返した場合、確保したファイル記述子を閉じ、必要に応じて shm_unlink を実行してクリーンアップを行います。
  • munmap や shm_unlink が失敗した場合、ereport を通じてエラーを通知します。

保守担当者向けメモ

  • 本関数はPOSIX環境専用です。WindowsやSystem Vなど他の実装方式と混同しないよう注意してください。
  • mmap には MAP_HASSEMAPHORE および MAP_NOSYNC フラグが付与されており、PostgreSQLの共有メモリ要件に最適化されています。
