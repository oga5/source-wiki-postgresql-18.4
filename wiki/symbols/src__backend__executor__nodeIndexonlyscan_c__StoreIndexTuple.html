StoreIndexTuple

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexonlyscan.c:268

目的

インデックスの内部形式（IndexTuple）からデータを抽出し、実行エンジンが利用可能な TupleTableSlot へ変換して格納する。

シグネチャまたは形状

static void StoreIndexTuple(IndexOnlyScanState *node, TupleTableSlot *slot, IndexTuple itup, TupleDesc itupdesc)

入力と出力

  • **入力**:
  • node: 現在の IndexOnlyScanState。
  • slot: データを格納する TupleTableSlot。
  • itup: 変換元の IndexTuple。
  • itupdesc: インデックスのタプル記述子。
  • **出力**: なし（slot の内容を更新する副作用を持つ）。

前提条件と不変条件

  • slot のタプル記述子と itupdesc は、列数（natts）が一致している必要がある。
  • itupdesc は、スロットの記述子ではなく、アクセスメソッド（AM）から提供されたものを使用しなければならない（btreeの name_ops など、型定義が異なる場合があるため）。

アルゴリズムまたは振る舞い

1. ExecClearTuple(slot) を呼び出し、スロットを初期化する。 2. index_deform_tuple を使用して、itup から値を抽出し、スロットの tts_values および tts_isnull 配列に展開する。 3. node->ioss_NameCStringAttNums が設定されている場合（name 型の列が存在する場合）、該当する列に対して以下の処理を行う：

  • NULL 値であればスキップする。
  • MemoryContextAlloc を使用して NAMEDATALEN 分のメモリを確保する。
  • namestrcpy を使用して、cstring 形式のデータを確保したメモリにコピーし、ゼロパディングを行う。
  • スロットの値を更新する。

    • 4. ExecStoreVirtualTuple(slot) を呼び出し、スロットを仮想タプルとしてマークする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeIndexonlyscan.c 内の IndexOnlyNext 関数から呼び出される。
  • **呼び出し先**: ExecClearTuple, index_deform_tuple, MemoryContextAlloc, namestrcpy, NameGetDatum, ExecStoreVirtualTuple。

関連データ構造

  • IndexOnlyScanState: スキャンノードの状態管理構造体。
  • TupleTableSlot: 実行エンジンがタプルを保持するための汎用スロット。
  • IndexTuple: インデックス内の物理的なタプル表現。

失敗モードと境界条件

  • slot->tts_tupleDescriptor->natts と itupdesc->natts が不一致の場合、Assert により停止する。
  • name 型の変換において、メモリ不足が発生した場合は MemoryContextAlloc がエラーを投げる。

保守担当者向けメモ

  • name 型の列に対する NAMEDATALEN の割り当ては、システムカタログや特定のユーザーテーブルでインデックスが使用される場合にのみ発生する。
  • ioss_NameCStringAttNums のチェックは unlikely マクロで囲まれており、一般的なケースでは最適化されている。
