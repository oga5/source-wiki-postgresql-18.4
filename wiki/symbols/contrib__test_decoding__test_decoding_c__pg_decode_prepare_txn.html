pg_decode_prepare_txn

定義

  • 種別: function
  • 定義箇所: contrib/test_decoding/test_decoding.c:370

目的

論理デコーディングにおける「2フェーズコミット」の準備（PREPARE TRANSACTION）イベントをテキスト形式で出力するコールバック関数です。

シグネチャまたは形状

static void pg_decode_prepare_txn(LogicalDecodingContext *ctx, ReorderBufferTXN *txn, XLogRecPtr prepare_lsn)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • txn: 準備対象のトランザクション情報。
  • prepare_lsn: 準備イベントのWAL位置（本関数内では未使用）。
  • **出力**: なし（副作用として ctx->out に文字列を書き出す）。

前提条件と不変条件

  • ctx->output_plugin_private に TestDecodingData が格納されている必要があります。
  • txn->output_plugin_private に TestDecodingTxnData が格納されている必要があります。

アルゴリズムまたは振る舞い

1. **空トランザクションの判定**: data->skip_empty_xacts が有効かつトランザクションに変更が含まれていない場合、何も出力せずに終了します。 2. **出力準備**: OutputPluginPrepareWrite(ctx, true) を呼び出し、出力バッファを初期化します。 3. **文字列構築**:

  • PREPARE TRANSACTION <gid> を書き込みます。
  • data->include_xids が有効な場合、txid を追記します。
  • data->include_timestamp が有効な場合、準備時刻を追記します。

    • 4. **出力実行**: OutputPluginWrite(ctx, true) を呼び出し、構築した文字列を送信します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの論理デコーディングコアエンジン（prepare_cb コールバック経由）。
  • **呼び出し先**:
  • OutputPluginPrepareWrite
  • appendStringInfo
  • quote_literal_cstr
  • timestamptz_to_str
  • OutputPluginWrite

関連データ構造

  • LogicalDecodingContext: デコーディング状態を保持するコア構造体。
  • ReorderBufferTXN: トランザクションのメタデータ（GID、XID、時刻等）を保持。
  • TestDecodingData: プラグインの設定（オプション値）。
  • TestDecodingTxnData: トランザクションごとの状態（変更の有無）。

失敗モードと境界条件

  • skip_empty_xacts オプションが設定されている場合、変更を含まないトランザクションは無視されます。
  • txn->gid が空の場合、SQL構文として不正な出力になる可能性があります。

保守担当者向けメモ

  • 本関数は _PG_output_plugin_init 内で cb->prepare_cb に登録されます。
  • prepare_lsn 引数は現在の実装では使用されていませんが、将来的な拡張のためにシグネチャに含まれています。
