to_regoper

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/regproc.c:526

目的

演算子名（文字列）を対応するシステムカタログ上の演算子 OID に変換します。エラー発生時に例外をスローせず、NULL を返す安全な変換インターフェースを提供します。

シグネチャまたは形状

Datum to_regoper(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_TEXT_PP(0) を介して演算子名を表す text 型の値を受け取ります。
  • **出力**: 成功した場合は演算子の OID を Datum として返し、解決に失敗した場合は NULL を返します。

前提条件と不変条件

  • 入力文字列は text_to_cstring により C 文字列へ変換されます。
  • 内部的に ErrorSaveContext を使用しており、エラー発生時に呼び出し元へ制御を戻すための安全な状態を保持します。

アルゴリズムまたは振る舞い

1. 入力された text 型の演算子名を C 文字列 opr_name に変換します。 2. ErrorSaveContext を初期化します。 3. DirectInputFunctionCallSafe を呼び出し、regoperin 関数を使用して opr_name から OID への解決を試みます。 4. 解決が成功すればその OID を返し、失敗した場合は PG_RETURN_NULL() を実行して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上の直接的な呼び出し元は確認されていません。
  • **呼び出し先**:
  • text_to_cstring
  • regoperin (演算子名から OID への変換を行う入力関数)
  • DirectInputFunctionCallSafe

関連データ構造

  • ErrorSaveContext: エラー情報を保持し、例外を抑制するための構造体。
  • Datum: PostgreSQL の汎用データ表現。

失敗モードと境界条件

  • 指定された演算子名がシステムカタログ内に存在しない場合、regoperin がエラーを検出し、to_regoper は NULL を返します。
  • 入力文字列が不正な形式である場合も同様に NULL が返されます。

保守担当者向けメモ

  • 本関数は SQL レベルから呼び出されることを想定したラッパーです。
  • 演算子の解決ロジック自体は regoperin に依存しているため、名前解決の挙動を変更する場合は regoperin およびその内部で呼ばれる parseNameAndArgTypes 等を確認してください。
