PGSemaphoreTryLock

定義

  • 種別: function
  • 定義箇所: src/backend/port/posix_sema.c:363

目的

POSIX セマフォに対して非ブロッキングなロック試行を行い、成功したかどうかを呼び出し元に通知する。

シグネチャまたは形状

bool PGSemaphoreTryLock(PGSemaphore sema)

入力と出力

  • **入力**: sema (PGSemaphore型) - 操作対象のセマフォへのポインタ。
  • **出力**: bool - ロックの獲得に成功した場合は true、失敗した場合は false を返す。

前提条件と不変条件

  • sema は有効な PGSemaphore オブジェクトを指している必要がある。
  • 内部的に PG_SEM_REF(sema) マクロを使用して sem_t 構造体へアクセスする。

アルゴリズムまたは振る舞い

1. sem_trywait を呼び出し、セマフォのロックを試みる。 2. シグナル割り込み (errno == EINTR) が発生した場合は、ループ内で再試行を行う。 3. sem_trywait が成功した場合、true を返す。 4. EAGAIN または EDEADLK エラーが発生した場合は、ロック獲得失敗とみなし false を返す。 5. その他のエラーが発生した場合は、elog(FATAL, ...) を呼び出しプロセスを終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/port/win32_sema.c (直接的な呼び出しを確認)
  • その他、src/include/storage/pg_sema.h で宣言されており、PostgreSQL のストレージサブシステム全体から利用される可能性がある。
  • **呼び出し先**:
  • sem_trywait (POSIX API)
  • elog (エラー報告用)

関連データ構造

  • PGSemaphore: sem_t * を保持する抽象型。
  • PG_SEM_REF(x): PGSemaphore から内部の sem_t を抽出するマクロ。

失敗モードと境界条件

  • **EAGAIN**: セマフォが現在ロックされており、即座に獲得できない状態。false を返す。
  • **EDEADLK**: デッドロックが検出された場合。false を返す。
  • **EINTR**: システムコール中にシグナルを受信した場合。ループにより再試行される。
  • **その他のエラー**: sem_trywait が予期せぬエラーを返した場合、致命的エラーとしてプロセスを停止させる。

保守担当者向けメモ

  • 本関数は POSIX セマフォ実装に依存している。System V セマフォや Windows セマフォの実装とは別個のファイル (sysv_sema.c, win32_sema.c) で同名の関数が定義されているため、プラットフォームに応じたビルド構成に注意すること。
