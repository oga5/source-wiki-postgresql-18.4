pgstat_update_heap_dead_tuples

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_relation.c:452

目的

リレーションの統計情報において、デッドタプル（削除済みタプル）の増減数を更新します。

シグネチャまたは形状

void pgstat_update_heap_dead_tuples(Relation rel, int delta)

入力と出力

  • rel: 更新対象のリレーション（Relation 構造体）。
  • delta: 変更量。負の値が渡されると、内部的には delta_dead_tuples から減算（実質的な加算）が行われます。
  • 出力: なし。

前提条件と不変条件

  • rel->pgstat_info が有効である必要があります。
  • pgstat_should_count_relation(rel) が真を返す場合のみ、統計情報の更新が実行されます。

アルゴリズムまたは振る舞い

1. pgstat_should_count_relation(rel) を呼び出し、対象リレーションが統計追跡の対象かを確認します。 2. 対象であれば、rel->pgstat_info から PgStat_TableStatus 構造体を取得します。 3. pgstat_info->counts.delta_dead_tuples に対して、-delta を加算（delta_dead_tuples -= delta）します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/pruneheap.c:281
  • 呼び出し先: pgstat_should_count_relation（インライン化または内部参照の可能性あり）

関連データ構造

  • Relation: リレーションキャッシュ（RelCache）のデータ構造。
  • PgStat_TableStatus: リレーションの統計情報を保持する構造体。

失敗モードと境界条件

  • rel->pgstat_info が NULL の場合、関数は何も行わず安全に終了します（pgstat_should_count_relation の判定による）。

保守担当者向けメモ

  • 本関数は delta を減算する実装となっているため、呼び出し側で渡す値の符号に注意してください。
  • 統計情報は即座に共有メモリへ反映されるわけではなく、ローカルの PgStat_TableStatus が更新されるのみです。共有メモリへのフラッシュは、後続のトランザクション終了処理やコールバックによって行われます。
