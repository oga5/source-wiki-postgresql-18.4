AioShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_init.c:148

目的

PostgreSQL の非同期 I/O (AIO) サブシステムに必要な共有メモリ領域の確保および初期化を行います。

シグネチャまたは形状

void AioShmemInit(void)

入力と出力

  • 入力: なし（グローバル変数 io_max_concurrency および io_max_combine_limit を参照）
  • 出力: なし（副作用として共有メモリ上に pgaio_ctl 構造体および関連する配列を割り当て、初期化する）

前提条件と不変条件

  • PostgreSQL の共有メモリ初期化シーケンスの一部として呼び出される必要があります。
  • ShmemInitStruct を使用して共有メモリを割り当てるため、本関数は共有メモリのレイアウトが確定した後に実行される必要があります。

アルゴリズムまたは振る舞い

1. AioCtl 構造体を共有メモリから確保します。既に確保済み（found が真）の場合は、初期化をスキップして out ラベルへジャンプします。 2. AioBackend、AioHandle、AioHandleIOV、AioHandleData の各領域を共有メモリから確保し、pgaio_ctl にポインタを格納します。 3. 全バックエンドプロセス数（AioProcs()）分ループし、各バックエンドの PgAioBackend 構造体を初期化します。

  • idle_ios および in_flight_ios リストを初期化します。
  • 各バックエンドに割り当てられた io_max_concurrency 分の PgAioHandle を初期化し、idle_ios リストに追加します。

    • 4. 最後に、pgaio_method_ops->shmem_init が定義されている場合、それを呼び出して I/O メソッド固有のリソースを初期化します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:347
  • 呼び出し先: ShmemInitStruct、dclist_init、dclist_push_tail、ConditionVariableInit、pgaio_method_ops->shmem_init

関連データ構造

  • PgAioCtl: AIO サブシステムの制御構造体。
  • PgAioBackend: バックエンドごとの状態管理構造体。
  • PgAioHandle: 個別の I/O 操作を管理するハンドル。

失敗モードと境界条件

  • ShmemInitStruct が失敗した場合、PostgreSQL の共有メモリ割り当てルールに従い、プロセスは異常終了する可能性があります。
  • io_max_concurrency の値が極端に大きい場合、共有メモリの枯渇を招く可能性があります。

保守担当者向けメモ

  • 本関数は共有メモリの静的なレイアウトを決定します。構造体の追加やサイズ変更を行う際は、対応する ...ShmemSize 関数を更新する必要があります。
  • pgaio_method_ops->shmem_init の呼び出しは、共有メモリの初期化が初回（!found）であるかどうかに依存するため、メソッド側の実装ではこのフラグを適切に扱う必要があります。
