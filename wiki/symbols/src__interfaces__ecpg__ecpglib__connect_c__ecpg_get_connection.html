ecpg_get_connection

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/connect.c:75

目的

指定された名前、または現在のスレッドに関連付けられたデータベース接続（connection 構造体）を取得します。

シグネチャまたは形状

struct connection *ecpg_get_connection(const char *connection_name);

入力と出力

  • **入力**: connection_name（接続名を示す文字列、または NULL / "CURRENT"）。
  • **出力**: 該当する connection 構造体へのポインタ。見つからない場合は NULL を返します。

前提条件と不変条件

  • connection_name が NULL または "CURRENT" の場合、スレッドローカルな接続状態（TSD）が初期化されている必要があります。
  • 名前指定がある場合、接続リストの走査には connections_mutex による排他制御が必要です。

アルゴリズムまたは振る舞い

1. **名前が NULL または "CURRENT" の場合**:

  • ecpg_pthreads_init() を呼び出し、スレッドローカルストレージ（TSD）のキーを初期化します。
  • pthread_getspecific を使用して、現在のスレッドに関連付けられた接続を取得します。
  • TSD に接続が存在しない場合、グローバル変数 actual_connection をフォールバックとして返します。

    • 2. **名前が指定されている場合**:

  • connections_mutex をロックします。
  • ecpg_get_connection_nr を呼び出し、リストから指定された名前の接続を検索します。
  • 検索完了後、connections_mutex をアンロックして結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPGconnect, ECPGdisconnect, ECPGsetcommit, ECPGsetconn, ECPGget_PGconn 等の connect.c 内の関数、および descriptor.c, execute.c, misc.c 内の各関数から参照されています。
  • **呼び出し先**:
  • ecpg_pthreads_init
  • pthread_getspecific
  • pthread_mutex_lock / pthread_mutex_unlock
  • ecpg_get_connection_nr

関連データ構造

  • struct connection: 接続情報を保持する構造体。
  • connections_mutex: 接続リストを保護するグローバルミューテックス。
  • actual_connection_key: スレッドローカルな接続を識別するための pthread_key_t。
  • actual_connection: グローバルなデフォルト接続を指すポインタ。

失敗モードと境界条件

  • 指定された名前の接続が存在しない場合、ecpg_get_connection_nr が NULL を返し、本関数も NULL を返します。
  • スレッドローカルな接続もグローバルな接続も設定されていない場合、NULL を返します。

保守担当者向けメモ

  • 本関数はマルチスレッド環境での接続取得を抽象化しています。グローバル接続へのフォールバックは、ユーザー側で適切な排他制御が行われていることを前提としています。
