目的

Unicode文字変換処理において、実行する変換の種類（大文字、小文字、タイトルケース、ケースフォールディング）を識別するために使用されます。

typedef enum
{
	CaseLower = 0,
	CaseTitle = 1,
	CaseUpper = 2,
	CaseFold = 3,
	NCaseKind
} CaseKind;

入力と出力

  • 入力: 文字変換関数（例: casemap）の引数として渡される列挙値。
  • 出力: 該当する変換ルールを特定するための識別子。

前提条件と不変条件

  • NCaseKind は列挙型の要素数を示すために定義されており、変換処理のループや配列の境界チェックに使用されます。
  • 値は 0 から 3 の範囲で定義されています。

アルゴリズムまたは振る舞い

  • CaseLower: 小文字変換を指定します。
  • CaseTitle: タイトルケース変換を指定します。
  • CaseUpper: 大文字変換を指定します。
  • CaseFold: ケースフォールディング（正規化）を指定します。
  • 変換関数は、この値に基づいて適切な変換テーブル（case_map_lower 等）を選択します。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/unicode_case.c 内の casemap 関数および関連する文字列処理関数がこの型を引数として受け取ります。
  • 呼び出し先: 本シンボル自体は列挙型であり、関数を呼び出すことはありません。

関連データ構造

  • case_map_lower, case_map_upper 等の変換テーブル: CaseKind の値に応じて参照先が切り替わります。
