get_hex

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/encode.c:196

目的

1文字の16進数文字（ASCII）を対応する数値（0-15）に変換し、その結果を書き込みます。

シグネチャまたは形状

static inline bool get_hex(const char *cp, char *out)

入力と出力

  • **入力**:
  • cp: 変換対象の文字を指すポインタ。
  • out: 変換結果（数値）を格納するメモリ領域へのポインタ。
  • **出力**:
  • 戻り値: 変換が成功した場合は true、失敗した場合は false を返します。
  • out: 変換成功時は 0-15 の値が、失敗時は -1 が格納されます。

前提条件と不変条件

  • cp は有効なメモリ領域を指している必要があります。
  • hexlookup 配列が定義されている環境で動作します。

アルゴリズムまたは振る舞い

1. 入力文字 *cp を unsigned char にキャストします。 2. 文字コードが 127 未満の場合、静的配列 hexlookup を参照して対応する数値を取得します。 3. 取得した値を out に書き込みます。 4. res が 0 以上であれば true を、それ以外（無効な文字など）であれば false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/encode.c 内の hex_decode 関連処理から呼び出されています。
  • 外部プロジェクト（ecpglib, libpq）にも同名の関数が存在しますが、これらは本ソースファイル内の get_hex とは別個の実装である可能性があります。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありませんが、グローバルな hexlookup 配列を参照します。

関連データ構造

  • hexlookup: 16進数文字から数値への変換テーブル（本ファイル内で定義）。

失敗モードと境界条件

  • 入力文字が 16進数として無効な場合（hexlookup で -1 が返される場合）、false を返し、out には -1 が書き込まれます。
  • 文字コードが 127 以上の場合、変換は行われず false が返されます。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスを重視した設計となっています。
  • 外部プロジェクト（ecpg, libpq）で同名の関数が使用されていますが、それらとの互換性や重複定義の可能性については、ビルド環境の構成を確認してください。
