tokenize_include_file

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/hba.c:439

目的

認証設定ファイル（pg_hba.conf 等）内で指定されたインクルードファイルを読み込み、トークン化してリストに追加する。

シグネチャまたは形状

static void tokenize_include_file(const char *outer_filename,
                                  const char *inc_filename,
                                  List **tok_lines,
                                  int elevel,
                                  int depth,
                                  bool missing_ok,
                                  char **err_msg)

入力と出力

  • outer_filename: インクルード元ファイルのパス。
  • inc_filename: インクルード対象のファイル名またはパス。
  • tok_lines: トークン化された行を格納するリストへのポインタ。
  • elevel: エラー報告レベル。
  • depth: インクルードの再帰深度。
  • missing_ok: ファイルが存在しない場合にエラーとせずスキップするかどうか。
  • err_msg: エラー発生時にメッセージを格納するポインタ。

前提条件と不変条件

  • AbsoluteConfigLocation により、インクルードファイルの絶対パスが解決される。
  • open_auth_file を介してファイルハンドルを取得する。
  • 処理終了時には free_auth_file を呼び出し、リソースを解放する。

アルゴリズムまたは振る舞い

1. AbsoluteConfigLocation でインクルードファイルのフルパスを生成する。 2. open_auth_file でファイルを開く。 3. ファイルが開けない場合:

  • missing_ok が真かつ errno が ENOENT ならば、警告を出して終了する。
  • それ以外はエラーメッセージをセットして終了する。

    • 4. ファイルが開けた場合、tokenize_auth_file を呼び出して内容をトークン化し、tok_lines に追加する。 5. free_auth_file でファイルを閉じ、パス文字列を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/hba.c 内の複数箇所（行815, 846, 876など）。
  • **呼び出し先**: AbsoluteConfigLocation, open_auth_file, tokenize_auth_file, free_auth_file, ereport, pfree。

関連データ構造

  • List: トークン化された行を保持するリスト構造。

失敗モードと境界条件

  • ファイルが存在しない場合、missing_ok フラグに応じて挙動が分岐する。
  • 再帰的なインクルードが発生した場合、depth パラメータにより制御される（open_auth_file 側の実装に依存）。

保守担当者向けメモ

  • 本関数は設定ファイルの再帰的な読み込みをサポートしている。
  • err_msg は呼び出し元で適切に管理される必要がある。
