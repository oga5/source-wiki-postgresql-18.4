r_consonant_pair

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_danish.c:191

目的

単語の末尾付近にある特定の子音ペアを検出し、その一部を削除することで語幹を正規化する。

シグネチャまたは形状

static int r_consonant_pair(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境構造体。現在のカーソル位置 c や文字列バッファ p を含む）
  • 出力: int（成功時に 1、失敗または条件不適合時に 0、エラー時に負の値を返す）

前提条件と不変条件

  • z->I[1] に格納された境界値（R1領域など）よりカーソル位置 c が後ろにあること。
  • z->p が有効なUTF-8文字列バッファであること。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置を保存し、z->I[1] を一時的な境界として設定する。 2. find_among_b を使用し、末尾から前方へ向かって a_1 テーブルに定義された子音ペアを検索する。この際、対象文字が 'd' (100) または 't' (116) であることを確認する。 3. 条件に合致した場合、skip_b_utf8 で直前の文字をスキップし、slice_del を呼び出して該当箇所を削除する。 4. 処理終了後、一時的に変更した境界設定を復元する。

呼び出し元と呼び出し先

  • 呼び出し元: danish_UTF_8_stem（および同等のノルウェー語・スウェーデン語ステマー）から、ステミング処理の順次適用プロセスの一部として呼び出される。
  • 呼び出し先: find_among_b（検索）、skip_b_utf8（文字スキップ）、slice_del（文字列削除）。

関連データ構造

  • SN_env: ステミングの状態を保持する構造体。
  • a_1: 削除対象となる子音ペアを定義した among テーブル。

失敗モードと境界条件

  • カーソル位置が境界 z->I[1] より前にある場合は 0 を返す。
  • find_among_b が一致を見つけられない場合は 0 を返す。
  • slice_del が内部エラー（負の値）を返した場合は、その値をそのまま伝播させる。

保守担当者向けメモ

  • 本関数は自動生成コードの一部である。ロジックの変更が必要な場合は、直接編集せず元のSnowballソース（.sbl）を修正すること。
  • デバッグ時は z->c（カーソル位置）と z->lb（境界）の推移を追跡すること。
