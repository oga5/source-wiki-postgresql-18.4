resize_intArrayType

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_tool.c:251

目的

既存の PostgreSQL ArrayType（整数配列）のサイズを動的に変更し、新しい要素数に合わせてメモリを再確保または調整します。

シグネチャまたは形状

ArrayType *resize_intArrayType(ArrayType *a, int num)

入力と出力

  • **入力**:
  • a: 変更対象の ArrayType ポインタ。
  • num: 変更後の要素数。
  • **出力**:
  • サイズ変更後の ArrayType ポインタ。num <= 0 の場合は空の配列を返します。

前提条件と不変条件

  • 入力 a は有効な ArrayType である必要があります。
  • num <= 0 の場合、Assert(num == 0) が検証されます。
  • 戻り値の配列は、内部的に SET_VARSIZE によって新しいサイズが設定され、次元情報が更新されます。

アルゴリズムまたは振る舞い

1. **空配列の処理**: num <= 0 の場合、construct_empty_array(INT4OID) を呼び出し、空の配列を返します。 2. **サイズ一致の確認**: 現在の要素数と num が一致する場合は、何もせず a をそのまま返します。 3. **メモリ再確保**: ARR_DATA_OFFSET(a) に sizeof(int) * num を加算して必要なバイト数を計算し、repalloc を実行します。 4. **メタデータ更新**: SET_VARSIZE で全体のサイズを更新します。 5. **次元の調整**: 配列の次元数（ARR_NDIM）分ループし、各次元のサイズを num に設定します（2次元目以降は 1 に設定）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_gist.c
  • contrib/intarray/_int_op.c
  • contrib/intarray/_int_tool.c 内の各関数
  • **呼び出し先**:
  • construct_empty_array
  • repalloc
  • SET_VARSIZE, ARR_DATA_OFFSET, ARR_NDIM, ARR_DIMS などの PostgreSQL 配列操作マクロ

関連データ構造

  • ArrayType: PostgreSQL の可変長配列構造体。

失敗モードと境界条件

  • num <= 0: 0次元の空配列を返します。
  • num == ARRNELEMS(a): 処理を行わず入力をそのまま返します。
  • メモリ不足時: repalloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数は palloc されたメモリを操作するため、呼び出し側でメモリリークが発生しないよう注意してください。
  • 配列の次元操作において、1次元目以外を 1 に強制設定するロジックが含まれています。多次元配列を扱う際は、この挙動が意図通りか確認してください。
