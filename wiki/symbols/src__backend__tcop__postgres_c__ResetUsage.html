ResetUsage

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/postgres.c:5055

目的

リソース使用状況の統計計測を開始するための基準点（ベースライン）を記録します。

シグネチャまたは形状

void ResetUsage(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として静的変数 Save_r および Save_t を更新）

前提条件と不変条件

  • getrusage および gettimeofday システムコールが利用可能であること。
  • 記録された値は、後続の統計計算（経過時間やリソース消費量の算出）の起点として使用されることを前提としています。

アルゴリズムまたは振る舞い

1. getrusage(RUSAGE_SELF, &Save_r) を呼び出し、現在のプロセス自身のCPU時間やページフォールト等のリソース使用状況を Save_r に格納します。 2. gettimeofday(&Save_t, NULL) を呼び出し、現在のシステム時刻を Save_t に格納します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/nbtree/nbtsort.c (インデックス構築時の統計計測)
  • src/backend/executor/spi.c (SPI実行時の統計計測)
  • src/backend/tcop/postgres.c (クエリ実行サイクルの統計計測)
  • 呼び出し先:
  • getrusage (システムコール)
  • gettimeofday (システムコール)

関連データ構造

  • Save_r: struct rusage 型（リソース使用統計を保持）
  • Save_t: struct timeval 型（時刻情報を保持）

失敗モードと境界条件

  • システムコール getrusage または gettimeofday が失敗した場合の戻り値チェックは行われていません。

保守担当者向けメモ

  • 本関数は、統計情報の差分を取るための「スナップショット取得」関数です。
  • 統計結果を表示または利用する際は、本関数で記録した値と、処理終了後の値を比較するペアとなる関数（例: ShowUsage 等）と組み合わせて使用されます。
