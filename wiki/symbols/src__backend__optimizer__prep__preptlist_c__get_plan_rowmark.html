get_plan_rowmark

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/preptlist.c:525

目的

指定されたレンジテーブルインデックス（rtindex）に対応する PlanRowMark 構造体を、リスト内から検索して取得します。

シグネチャまたは形状

PlanRowMark *get_plan_rowmark(List *rowmarks, Index rtindex)

入力と出力

  • **入力**:
  • rowmarks: 検索対象となる PlanRowMark 構造体のリスト。
  • rtindex: 検索キーとなるレンジテーブルインデックス。
  • **出力**:
  • 見つかった場合は対応する PlanRowMark へのポインタ。
  • 見つからない場合は NULL。

前提条件と不変条件

  • rowmarks リスト内の各要素は PlanRowMark 構造体へのポインタである必要があります。
  • rtindex はレンジテーブル内の有効なインデックスであると想定されます。

アルゴリズムまたは振る舞い

1. rowmarks リストを先頭から順に走査します。 2. 各要素（PlanRowMark）の rti フィールドと、引数 rtindex を比較します。 3. 一致する要素が見つかった場合、そのポインタを即座に返します。 4. リストの末尾まで走査しても一致するものがない場合、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/deparse.c
  • src/backend/optimizer/path/indxpath.c
  • src/backend/optimizer/prep/preptlist.c
  • src/backend/optimizer/util/inherit.c
  • **呼び出し先**:
  • 本関数はリスト走査のための標準的なマクロ（foreach, lfirst）を使用しており、特定の関数呼び出しは行いません。

関連データ構造

  • PlanRowMark: src/include/nodes/plannodes.h で定義される、行ロックや更新の追跡に使用される構造体。

失敗モードと境界条件

  • rowmarks が空リスト（NIL）の場合、ループは実行されず NULL を返します。
  • 指定した rtindex がリスト内に存在しない場合、NULL を返します。

保守担当者向けメモ

  • 本関数は単純な線形探索を行うため、rowmarks リストが非常に長い場合にはパフォーマンスに影響を与える可能性がありますが、通常このリストはクエリ内のリレーション数に比例するため、実用上の問題は限定的です。
