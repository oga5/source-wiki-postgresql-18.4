pq_set_parallel_leader

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqmq.c:77

目的

共有メモリメッセージキュー通信において、並列クエリのリーダープロセスを識別するためのプロセスIDとプロセス番号をグローバル変数に設定します。

シグネチャまたは形状

void pq_set_parallel_leader(pid_t pid, ProcNumber procNumber)

入力と出力

  • pid (pid_t): リーダープロセスのプロセスID。
  • procNumber (ProcNumber): リーダープロセスのプロセス番号。
  • 戻り値: なし。

前提条件と不変条件

  • PqCommMethods が PqCommMqMethods に設定されていること（Assert により検証）。

アルゴリズムまたは振る舞い

1. 現在の通信メソッドが共有メモリキューモードであることを確認します。 2. 引数として渡された pid を pq_mq_parallel_leader_pid に格納します。 3. 引数として渡された procNumber を pq_mq_parallel_leader_proc_number に格納します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/parallel.c:1383
  • src/backend/replication/logical/applyparallelworker.c:952
  • 呼び出し先:
  • なし（本関数は変数の代入のみを行います）。

関連データ構造

  • pq_mq_parallel_leader_pid (静的変数)
  • pq_mq_parallel_leader_proc_number (静的変数)
  • PqCommMqMethods (通信メソッドテーブル)

失敗モードと境界条件

  • PqCommMethods が PqCommMqMethods 以外の場合、アサーション違反によりプロセスが停止します。

保守担当者向けメモ

  • 本関数は、ワーカープロセスがリーダープロセスに対してシグナルを送るための宛先情報を保持するために使用されます。
  • pq_mq_parallel_leader_pid 等の変数は、mq_putmessage 内でリーダーへの通知（SendProcSignal）を行う際に参照されます。
