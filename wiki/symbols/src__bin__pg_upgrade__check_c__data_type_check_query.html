data_type_check_query

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/check.c:332

目的

特定のデータ型がデータベース内で使用されているかを検証するための SQL クエリ文字列を生成する。このクエリは、指定された型だけでなく、それらに依存するドメイン、配列、複合型、範囲型を再帰的に探索する。

シグネチャまたは形状

char *data_type_check_query(int checknum)

入力と出力

  • **入力**: checknum (グローバル配列 data_types_usage_checks のインデックス)
  • **出力**: 実行可能な SQL クエリ文字列（psprintf により動的に割り当てられる）

前提条件と不変条件

  • checknum は data_types_usage_checks 配列の有効な範囲内であること。
  • data_types_usage_checks[checknum].base_query が有効な SQL フラグメントを保持していること。

アルゴリズムまたは振る舞い

1. data_types_usage_checks[checknum] からベースとなるクエリを取得する。 2. WITH RECURSIVE 句を用いて、以下の型依存関係を再帰的に追跡する:

  • ドメイン (typtype = 'd')
  • 配列 (typtype = 'b')
  • 複合型 (typtype = 'c')
  • 範囲型 (typtype = 'r')

    • 3. 抽出された型を使用しているテーブルカラムを検索する。 4. システムカタログ（pg_catalog, information_schema）や一時テーブル（pg_temp_）を除外するフィルタリングを行う。 5. 最終的に、該当する型を使用しているテーブル名、スキーマ名、カラム名を返すクエリを構築する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/check.c 内のループ処理（queries[i] = data_type_check_query(i);）から呼び出される。
  • **呼び出し先**: psprintf（文字列フォーマット用）を呼び出す。

関連データ構造

  • DataTypesUsageChecks: チェック対象の型とベースクエリを保持する構造体。

失敗モードと境界条件

  • psprintf によるメモリ割り当て失敗時にプロセスが終了する可能性がある。
  • 依存関係の探索において、循環参照や極端に深い型定義が存在する場合、クエリの実行コストが増大する可能性がある。

保守担当者向けメモ

  • 本関数が生成するクエリは、PostgreSQL のシステムカタログ構造に強く依存している。カタログスキーマに変更があった場合は、再帰クエリの結合条件を更新する必要がある。
  • RELKIND_RELATION, RELKIND_MATVIEW, RELKIND_INDEX を使用して対象オブジェクトを限定しているため、新しいオブジェクト種別をチェック対象に含める場合はここを修正すること。
