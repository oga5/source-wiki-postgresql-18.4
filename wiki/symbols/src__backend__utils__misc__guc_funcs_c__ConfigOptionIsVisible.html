ConfigOptionIsVisible

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/guc_funcs.c:580

目的

現在のユーザーが、特定のGUC（Grand Unified Configuration）パラメータを参照する権限を持っているかどうかを判定します。

シグネチャまたは形状

bool ConfigOptionIsVisible(struct config_generic *conf)

入力と出力

  • **入力**: conf (判定対象となるGUCパラメータの構造体へのポインタ)
  • **出力**: bool 型。参照可能であれば true、そうでなければ false を返します。

前提条件と不変条件

  • conf は有効な config_generic 構造体を指している必要があります。
  • 判定には現在のセッションユーザーの権限情報が使用されます。

アルゴリズムまたは振る舞い

1. 指定されたパラメータ conf に GUC_SUPERUSER_ONLY フラグが設定されているか確認します。 2. フラグが設定されている場合、現在のユーザーが ROLE_PG_READ_ALL_SETTINGS ロールを保持しているか（またはスーパーユーザー権限があるか）を has_privs_of_role で検証します。 3. 権限がない場合は false を返し、それ以外の場合は true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/guc.c (複数箇所)
  • src/backend/utils/misc/guc_funcs.c (複数箇所)
  • **呼び出し先**:
  • has_privs_of_role (権限チェック用)
  • GetUserId (現在のユーザーID取得用)

関連データ構造

  • struct config_generic: GUCパラメータの共通属性を保持する構造体。flags フィールドを通じてアクセス制御情報を保持します。

失敗モードと境界条件

  • conf が NULL の場合の挙動については、本ソースコード内では明示的なガードは存在しません。

保守担当者向けメモ

  • 本関数はセキュリティ上のアクセス制御を担うため、新しいGUCフラグを追加する際は、本関数での判定ロジックに影響がないか確認してください。
  • has_privs_of_role の仕様変更が本関数の判定結果に直接影響します。
