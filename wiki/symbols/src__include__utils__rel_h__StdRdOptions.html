StdRdOptions

定義

  • **種別**: struct
  • **定義箇所**: src/include/utils/rel.h:340

目的

ヒープリレーションに関連する構成オプション（fillfactor、autovacuum 設定、並列ワーカー数など）を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct StdRdOptions
{
    int32       vl_len_;
    int         fillfactor;
    int         toast_tuple_target;
    AutoVacOpts autovacuum;
    bool        user_catalog_table;
    int         parallel_workers;
    StdRdOptIndexCleanup vacuum_index_cleanup;
    bool        vacuum_truncate;
    bool        vacuum_truncate_set;
    double      vacuum_max_eager_freeze_failure_rate;
} StdRdOptions;

入力と出力

  • **入力**: リレーションのオプション設定値（システムカタログ等から読み込まれる）。
  • **出力**: RelationData 構造体の rd_options フィールドを通じて保持され、各アクセスマクロ（例: RelationGetFillFactor）により個別の設定値が抽出される。

前提条件と不変条件

  • vl_len_ は varlena ヘッダーであり、直接操作してはならない。
  • rd_options として格納される際は bytea 形式として扱われるため、キャストして使用する必要がある。

アルゴリズムまたは振る舞い

  • リレーションの物理的な格納特性や、VACUUM 処理時の挙動を制御するパラメータを保持する。
  • vacuum_max_eager_freeze_failure_rate は、VACUUM が凍結をスキップできるページの割合を定義する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/reloptions.c において、オプションのパースおよびオフセット計算に使用されている。
  • **呼び出し先**: 構造体内部で AutoVacOpts および StdRdOptIndexCleanup を保持している。

関連データ構造

  • RelationData: 本構造体を保持する親構造体。
  • AutoVacOpts: 自動バキューム設定を格納する構造体。
  • StdRdOptIndexCleanup: インデックスのバキューム処理を制御する列挙型。

失敗モードと境界条件

  • fillfactor は 0 から 100 の範囲である必要がある。
  • vacuum_max_eager_freeze_failure_rate は、無効な場合は 0 または -1 が指定される。

保守担当者向けメモ

  • RelationData の rd_options フィールドに直接アクセスせず、提供されているアクセスマクロを介して値を取得すること。
  • 新しいオプションを追加する場合は、reloptions.c のパーサー定義を更新する必要がある。
