gin_extract_tsvector

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsginidx.c:63

目的

GINインデックス作成時、tsvector型データからインデックス化対象となる語彙（lexeme）のリストを抽出する。

シグネチャまたは形状

Datum gin_extract_tsvector(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TSVECTOR(0): インデックス対象のtsvectorデータ。
  • PG_GETARG_POINTER(1): インデックス化される語彙の総数を格納するためのint32ポインタ。
  • **出力**:
  • 戻り値: 抽出された語彙（text型）のDatum配列へのポインタ。
  • 副作用: 第2引数（nentries）に抽出された語彙数が書き込まれる。

前提条件と不変条件

  • tsvectorが有効なデータ構造であること。
  • nentriesポインタが有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. tsvectorからサイズを取得し、nentriesに設定する。 2. tsvectorのサイズが0より大きい場合：

  • 語彙数分のDatum配列をpallocで確保する。
  • tsvector内の各WordEntryを走査し、対応する文字列（lexeme）をcstring_to_text_with_lenでtext型に変換する。
  • 変換したtextをDatumとして配列に格納する。

    • 3. 入力されたtsvectorがコピーである場合、PG_FREE_IF_COPYで解放する。 4. 抽出されたDatum配列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • GINインデックスの構築プロセス（PostgreSQLのインデックスアクセスメソッド経由）。
  • src/backend/utils/adt/tsginidx.c:308（gin_extract_tsvector_oldsig等の互換性スタブから呼び出されるケースあり）。
  • **呼び出し先**:
  • palloc: メモリ確保。
  • cstring_to_text_with_len: 文字列の型変換。
  • PG_GETARG_*, PG_RETURN_POINTER, PG_FREE_IF_COPY: PostgreSQL関数マネージャインターフェース。

関連データ構造

  • TSVector: 全文検索用データ型。
  • WordEntry: tsvector内の各語彙の位置と長さを保持する構造体。
  • Datum: PostgreSQLの汎用データ表現。

失敗モードと境界条件

  • vector->sizeが0の場合、entriesはNULLのまま返される。
  • メモリ不足時にはpallocがエラーを発生させる。

保守担当者向けメモ

  • 本関数はGINインデックスのキー抽出インターフェースとして機能する。
  • 過去のシグネチャ変更により、現在では3引数（PG_FUNCTION_ARGS経由で取得）を前提としている。
  • tsvectorの内部構造（ARRPTR, STRPTR）に強く依存しているため、tsvectorのバイナリ形式に変更があった場合は本関数の修正が必要。
