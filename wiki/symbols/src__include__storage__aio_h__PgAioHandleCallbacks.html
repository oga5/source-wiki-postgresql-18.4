PgAioHandleCallbacks

定義

  • 種別: struct
  • 定義箇所: src/include/storage/aio.h:213

目的

非同期 I/O (AIO) 操作のライフサイクルにおける各段階（準備、完了、報告）で実行されるコールバック関数群を保持する構造体です。

シグネチャまたは形状

struct PgAioHandleCallbacks
{
    PgAioHandleCallbackStage stage;
    PgAioHandleCallbackComplete complete_shared;
    PgAioHandleCallbackComplete complete_local;
    PgAioHandleCallbackReport report;
};

入力と出力

  • **入力**: 各フィールドは関数ポインタ（またはそれに関連する型）であり、AIO サブシステムによって呼び出されます。
  • **出力**: complete_shared および complete_local は、戻り値を通じてエラー発生を通知可能です。

前提条件と不変条件

  • complete_shared はクリティカルセクション内で実行されます。
  • complete_shared は共有メモリ内のリソースのみを操作可能です。
  • 複数のコールバックが登録されている場合、最後に登録されたものが最初に実行されます。
  • complete_local は、すべての complete_shared が完了した後に、I/O を発行したバックエンド内でのみ実行されます。

アルゴリズムまたは振る舞い

1. **準備 (stage)**: I/O 実行前にリソース（バッファピンの所有権移動など）を準備します。 2. **共有完了 (complete_shared)**: I/O 完了時に共有メモリ上の状態を更新します。I/O を発行したバックエンドとは異なるプロセスで実行される可能性があります。 3. **ローカル完了 (complete_local)**: I/O を発行したバックエンド固有の状態（一時バッファの BufferDesc など）を更新します。 4. **報告 (report)**: I/O 操作の結果を報告し、必要に応じてエラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: AIO サブシステム（src/backend/storage/aio/aio.c 等）が、I/O の各フェーズでこれらの関数ポインタを呼び出します。
  • **呼び出し先**: 以下の実装が確認されています。
  • aio_shared_buffer_readv_cb / aio_local_buffer_readv_cb (src/backend/storage/buffer/bufmgr.c)
  • aio_md_readv_cb (src/backend/storage/smgr/md.c)
  • aio_invalid_cb (src/backend/storage/aio/aio_callback.c)

関連データ構造

  • PgAioHandle: このコールバック群を保持し、I/O コンテキストを管理する構造体。
  • PgAioHandleCallbackStage, PgAioHandleCallbackComplete, PgAioHandleCallbackReport: 各フェーズの関数型定義。

失敗モードと境界条件

  • complete_shared はクリティカルセクション内で実行されるため、エラー処理は呼び出し元（I/O 発行側）の責任となります。
  • complete_shared 内では、共有メモリ外のリソース（ローカルメモリ等）へのアクセスは禁止されています。

保守担当者向けメモ

  • EXEC_BACKEND 環境下での関数ポインタの安定性を確保するため、直接関数ポインタを扱う代わりに PgAioHandleCallbackID を介した識別が必要になる場合があります。
  • コールバックの実行順序（後入れ先出し）は、下位層の処理が上位層の処理に依存できるように設計されています。
