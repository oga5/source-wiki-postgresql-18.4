substitute_grouped_columns

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_agg.c:1359

目的

クエリ内の式に含まれるグループ化された列（grouped columns）を、RTE_GROUP を参照する Var ノードへ置換する。

シグネチャまたは形状

Node *substitute_grouped_columns(Node *node, ParseState *pstate, Query *qry,
                                 List *groupClauses, List *groupClauseCommonVars,
                                 List *gset_common,
                                 bool have_non_var_grouping,
                                 List **func_grouped_rels)

入力と出力

  • **入力**:
  • node: 変換対象の式ツリー。
  • pstate, qry: パーサーの状態およびクエリ構造。
  • groupClauses, groupClauseCommonVars, gset_common: グループ化の定義および共通変数リスト。
  • have_non_var_grouping: 変数以外のグループ化が存在するかを示すフラグ。
  • func_grouped_rels: 関数に関連付けられたグループ化リレーションのリスト。
  • **出力**:
  • 変換後の式ツリー（Node *）。

前提条件と不変条件

  • 呼び出し元で結合（Join）の Vars はフラット化されていると仮定する（context.hasJoinRTEs = false）。
  • sublevels_up は 0 に初期化される。

アルゴリズムまたは振る舞い

1. substitute_grouped_columns_context 構造体を初期化し、引数で受け取った情報を格納する。 2. substitute_grouped_columns_mutator を呼び出し、式ツリーを再帰的に走査して置換を実行する。 3. expression_tree_mutator を利用してツリーを書き換える。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_agg.c 内の複数箇所（parseCheckAggregates 等の文脈で呼び出される）。
  • **呼び出し先**:
  • substitute_grouped_columns_mutator（直接の再帰的処理）。

関連データ構造

  • substitute_grouped_columns_context: 変換処理中に保持される状態管理用構造体。
  • Query, ParseState, Node: PostgreSQL のパーサーが使用する標準的なノード構造。

失敗モードと境界条件

  • 複雑な再帰処理を含むため、sublevels_up カウンタの管理が不適切である場合、誤ったスコープの変数を参照する可能性がある。
  • 呼び出し元で結合がフラット化されていない場合、hasJoinRTEs の仮定により予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は substitute_grouped_columns_mutator を呼び出すためのラッパーである。
  • sublevels_up の管理が正しく行われているか、デバッグ時に注意すること。
  • GROUPING SETS の展開ロジックと密接に関連しているため、変更時は関連する expand_grouping_sets 等の挙動も考慮すること。
