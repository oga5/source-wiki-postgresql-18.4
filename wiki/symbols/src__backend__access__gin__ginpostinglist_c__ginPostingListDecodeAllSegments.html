ginPostingListDecodeAllSegments

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginpostinglist.c:296

目的

圧縮された複数の GinPostingList セグメントを走査し、格納されているすべての ItemPointer（TID）を復元して配列として返します。

シグネチャまたは形状

ItemPointer ginPostingListDecodeAllSegments(GinPostingList *segment, int len, int *ndecoded_out)

入力と出力

  • **入力**:
  • segment: 復元対象の GinPostingList 構造体へのポインタ。
  • len: 処理対象のバイト長。
  • ndecoded_out: 復元されたTIDの総数を格納する出力用ポインタ（NULL可）。
  • **出力**:
  • 復元された ItemPointer の配列（palloc によりメモリ確保）。

前提条件と不変条件

  • 入力されるTIDはソート済みである必要があります。
  • 復元処理中、TIDの順序が維持されることを Assert で検証しています。
  • 出力配列は必要に応じて repalloc により動的に拡張されます。

アルゴリズムまたは振る舞い

1. segment->nbytes * 2 + 1 を初期サイズとしてメモリを確保します。 2. segment から endseg に達するまで、以下の処理を繰り返します。

  • セグメント内の最初のTID（first）を結果配列に追加します。
  • itemptr_to_uint64 でTIDを整数値に変換し、後続の差分（Delta）を decode_varbyte で順次加算してTIDを復元します。
  • 復元したTIDを uint64_to_itemptr で ItemPointer 形式に戻し、結果配列に格納します。
  • GinNextPostingListSegment を用いて次のセグメントへ移動します。

    • 3. 最後に復元された総数を ndecoded_out に書き込み、配列を返します。

関連データ構造

  • GinPostingList: 圧縮されたTIDリストを保持する構造体。
  • ItemPointerData: 復元されるTIDのデータ型。

失敗モードと境界条件

  • メモリ不足時には palloc / repalloc がエラーを発生させます。
  • len が不正な場合、セグメントの境界を越えてメモリを読み取る可能性があります。
  • ndecoded が nallocated を超えるたびに配列の再確保が行われます。

保守担当者向けメモ

  • 本関数は差分エンコーディングの復元を行うため、decode_varbyte の仕様変更や GinPostingList の構造変更時には本関数のロジックも追従が必要です。
  • ndecoded_out が NULL の場合、復元数は呼び出し元で取得できません。
