find_appinfos_by_relids

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/appendinfo.c:756

目的

指定されたリレーションIDセット（Relids）に対応する AppendRelInfo 構造体の配列を、プランナの管理情報から抽出します。

シグネチャまたは形状

AppendRelInfo **find_appinfos_by_relids(PlannerInfo *root, Relids relids, int *nappinfos)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報を保持する PlannerInfo 構造体。
  • relids: 検索対象のリレーションIDを示す Relids ビットマップセット。
  • nappinfos: 抽出された AppendRelInfo の個数を格納するための出力用ポインタ。
  • **出力**:
  • AppendRelInfo **: 該当する AppendRelInfo 構造体へのポインタ配列。呼び出し元でメモリ解放が必要。

前提条件と不変条件

  • relids に含まれる各リレーションIDは、root->append_rel_array に存在するか、あるいは find_base_rel_ignore_join で特定可能なベースリレーションである必要があります。
  • 該当する AppendRelInfo が見つからない場合、関数は elog(ERROR) を発生させます。

アルゴリズムまたは振る舞い

1. relids のメンバー数に基づき、結果を格納する配列を palloc で確保します。 2. bms_next_member を使用して relids 内の各リレーションIDを走査します。 3. 各IDについて root->append_rel_array を参照し、AppendRelInfo を取得します。 4. AppendRelInfo が存在しない場合、find_base_rel_ignore_join を呼び出し、それが外部結合（OJ）インデックス等で無視すべきものか確認します。無視できない場合はエラーを報告します。 5. 取得した AppendRelInfo を配列に格納し、最後に nappinfos に件数を設定して配列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/joinrels.c
  • src/backend/optimizer/plan/planner.c
  • src/backend/optimizer/util/appendinfo.c
  • src/backend/partitioning/partprune.c
  • **呼び出し先**:
  • bms_num_members, bms_next_member (ビットマップ操作)
  • find_base_rel_ignore_join (ベースリレーションの検証)
  • palloc (メモリ確保)
  • elog (エラー報告)

関連データ構造

  • AppendRelInfo: 親子関係の変換情報を保持する構造体。
  • PlannerInfo: プランナのコンテキスト情報。
  • Relids: リレーションIDの集合を表すビットマップセット。

失敗モードと境界条件

  • relids が空の場合、空の配列が返され nappinfos は 0 となります。
  • append_rel_array にエントリが存在しないリレーションIDが含まれている場合、それがベースリレーションであれば ERROR が発生します。

保守担当者向けメモ

  • 本関数は AppendRelInfo の配列を動的に確保するため、呼び出し元でのメモリ管理（pfree）が適切に行われているか確認してください。
  • append_rel_array のインデックスとリレーションIDの整合性に依存しているため、プランナの初期化フェーズとの順序関係に注意が必要です。
