InitProcessPhase2

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/proc.c:579

目的

現在のバックエンドプロセス（MyProc）を共有メモリ上の ProcArray に登録し、プロセス終了時に自動的に配列から削除されるよう設定する。

シグネチャまたは形状

void InitProcessPhase2(void)

入力と出力

  • 入力: なし（グローバル変数 MyProc を参照）
  • 出力: なし

前提条件と不変条件

  • MyProc が NULL であってはならない（Assert により検証済み）。

アルゴリズムまたは振る舞い

1. ProcArrayAdd(MyProc) を呼び出し、現在のプロセスを共有メモリの ProcArray に追加して可視化する。 2. on_shmem_exit(RemoveProcFromArray, 0) を呼び出し、バックエンド終了時に RemoveProcFromArray が実行されるよう登録する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/init/postinit.c:730
  • src/backend/storage/lmgr/proc.c:573（コメント内での言及）
  • 呼び出し先:
  • ProcArrayAdd
  • on_shmem_exit
  • RemoveProcFromArray（on_shmem_exit 経由の登録先）

関連データ構造

  • PGPROC: MyProc が指すプロセス管理構造体。
  • ProcArray: プロセス状態を保持する共有メモリ上の配列。

失敗モードと境界条件

  • MyProc が初期化されていない場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数は InitProcess の後続処理として実行される。
  • on_shmem_exit を使用しているため、バックエンドの正常終了および異常終了時のクリーンアップパスで RemoveProcFromArray が確実に呼び出される設計となっている。
