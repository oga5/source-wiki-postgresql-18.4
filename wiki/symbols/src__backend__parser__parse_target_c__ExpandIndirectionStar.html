ExpandIndirectionStar

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_target.c:1347

目的

A_Indirection ノードに含まれるワイルドカード（*）を解決し、対象となる行型（rowtype）の全フィールドを展開してリスト化する。

シグネチャまたは形状

List *ExpandIndirectionStar(ParseState *pstate, A_Indirection *ind, bool make_target_entry, ParseExprKind exprKind)

入力と出力

  • **入力**:
  • pstate: パーサーの状態管理構造体。
  • ind: 展開対象の A_Indirection ノード。
  • make_target_entry: TargetEntry ノードとして構築するかどうかを指定するフラグ。
  • exprKind: 式のコンテキストを示す列挙型。
  • **出力**:
  • 展開されたフィールドのリスト（List *）。

前提条件と不変条件

  • 入力された ind は A_Indirection 型であり、末尾に * を含むことが想定される。
  • copyObject(ind) により入力ノードの複製が作成され、元のノードは変更されない。

アルゴリズムまたは振る舞い

1. 入力された ind を複製する。 2. 複製したノードから末尾の * を取り除き、行型オブジェクトへの参照を作成する。 3. transformExpr を呼び出し、その参照を意味解析済みの式（Node *expr）に変換する。 4. ExpandRowReference を呼び出し、変換された式を個別のフィールドに展開して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_target.c 内の解析ロジックから参照されている（行番号: 167, 256）。
  • **呼び出し先**:
  • copyObject
  • list_truncate
  • list_length
  • transformExpr
  • ExpandRowReference

関連データ構造

  • A_Indirection: 添字やフィールド参照を含む構文解析ノード。
  • ParseState: パーサーの解析状態を保持する構造体。
  • TargetEntry: ターゲットリストの各要素を表すノード。

失敗モードと境界条件

  • ind が * を含まない場合や、transformExpr が解決できない式を返した場合の挙動は、依存する transformExpr および ExpandRowReference の実装に依存する。

保守担当者向けメモ

  • 本関数は SELECT * や table.* のような構文を、複合型（rowtype）の展開を通じて処理する。
  • 修正を行う際は、ExpandRowReference が期待するノード形式と整合しているか確認すること。
