ReorderBufferAllocTXN

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/reorderbuffer.c:433

目的

論理デコーディングにおいて、トランザクションの状態を管理するための ReorderBufferTXN 構造体をメモリ上に割り当て、初期化する。

シグネチャまたは形状

static ReorderBufferTXN *ReorderBufferAllocTXN(ReorderBuffer *rb)

入力と出力

  • **入力**: rb (ReorderBuffer*) - トランザクションが所属するバッファ管理構造体。
  • **出力**: 初期化済みの ReorderBufferTXN* へのポインタ。

前提条件と不変条件

  • rb->txn_context が有効なメモリコンテキストとして初期化されていること。
  • 割り当てられた ReorderBufferTXN は、changes、tuplecids、subtxns の各リストが空の状態（dlist_init 済み）であること。
  • command_id が InvalidCommandId で初期化されていること。

アルゴリズムまたは振る舞い

1. rb->txn_context から ReorderBufferTXN サイズ分のメモリを確保する。 2. 確保したメモリ領域を memset で 0 クリアする。 3. トランザクション内の変更リスト、タプル CID リスト、サブトランザクションリストを dlist_init で初期化する。 4. command_id を InvalidCommandId に設定し、output_plugin_private を NULL に設定する。 5. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/reorderbuffer.c:705 (トランザクションエントリ作成時)
  • **呼び出し先**:
  • MemoryContextAlloc (メモリ確保)
  • memset (メモリ初期化)
  • dlist_init (リスト初期化)

関連データ構造

  • ReorderBuffer: デコーディングセッション全体の状態を保持。
  • ReorderBufferTXN: トランザクションの変更内容やサブトランザクションを保持するメイン構造体。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAlloc がエラーを発生させ、処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数は ReorderBufferTXN のメモリレイアウトと初期状態を規定する。構造体定義に変更を加えた場合は、本関数内の初期化ロジックも適切に更新すること。
  • command_id の初期値は 0 ではないため、memset による 0 クリア後に明示的な代入が必要である点に注意。
