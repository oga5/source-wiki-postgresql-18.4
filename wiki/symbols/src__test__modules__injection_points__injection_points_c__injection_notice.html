injection_notice

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_points.c:262

目的

インジェクションポイントが実行された際に、PostgreSQLのログへ NOTICE レベルのメッセージを出力し、実行統計を記録するコールバック関数です。

シグネチャまたは形状

void injection_notice(const char *name, const void *private_data, void *arg)

入力と出力

  • **入力**:
  • name: インジェクションポイントの識別名。
  • private_data: InjectionPointCondition 構造体へのポインタ。実行条件の判定に使用。
  • arg: ログ出力時に付加する追加情報（文字列）。
  • **出力**: なし（void）。副作用としてログ出力および統計更新を行う。

前提条件と不変条件

  • private_data は InjectionPointCondition 型としてキャスト可能である必要があります。
  • injection_point_allowed による条件判定が真（許可）である場合にのみ、後続の処理が実行されます。

アルゴリズムまたは振る舞い

1. injection_point_allowed(condition) を呼び出し、実行が許可されているか確認します。許可されていない場合は即座に終了します。 2. pgstat_report_inj(name) を呼び出し、当該インジェクションポイントの実行統計を記録します。 3. arg が提供されている場合、elog(NOTICE, ...) を使用して「name ( arg )」という形式で通知を出力します。 4. arg が提供されていない場合、elog(NOTICE, ...) を使用して「name」という形式で通知を出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/injection_points/injection_points.c 内で、インジェクションポイントのコールバックとして登録される際に参照されます（行361）。
  • **呼び出し先**:
  • injection_point_allowed: 実行可否の判定。
  • pgstat_report_inj: 統計情報の記録。
  • elog: ログ出力。

関連データ構造

  • InjectionPointCondition: インジェクションポイントの実行条件を保持する構造体。

失敗モードと境界条件

  • arg が NULL の場合、通知メッセージにはポイント名のみが含まれます。
  • injection_point_allowed が偽を返した場合、統計記録やログ出力は行われません。

保守担当者向けメモ

  • 本関数はテストモジュール専用です。本番環境のコードパスには影響を与えません。
  • elog を使用しているため、クライアント側で NOTICE メッセージを受け取ることが可能です。テストスクリプト等でこの出力を検証に使用できます。
