insert_filehash_entry

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/filemap.c:203

目的

指定されたファイルパスに対応する file_entry_t 構造体をハッシュテーブル filehash から取得、または新規作成して返します。

シグネチャまたは形状

static file_entry_t *insert_filehash_entry(const char *path)

入力と出力

  • **入力**: path (ファイルパスを示す文字列)
  • **出力**: 該当する file_entry_t 構造体へのポインタ。

前提条件と不変条件

  • filehash が初期化済みであること。
  • 新規作成されたエントリは、各フィールドがデフォルト値（false, FILE_TYPE_UNDEFINED, 0, NULL, FILE_ACTION_UNDECIDED）で初期化されます。

アルゴリズムまたは振る舞い

1. filehash_insert を呼び出し、ハッシュテーブル内に path が存在するか確認します。 2. 既に存在する場合（found が true）、そのエントリをそのまま返します。 3. 存在しない場合、新規エントリを確保し、以下の初期化を行います：

  • path を複製して格納。
  • isRelDataFile(path) を用いてリレーションデータファイルかどうかを判定。
  • 各種メタデータ（target_exists, source_exists 等）を初期状態に設定。
  • アクションを FILE_ACTION_UNDECIDED に設定。

    • 4. 最終的にエントリへのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_rewind/filemap.c:301 (process_source_file 内)
  • src/bin/pg_rewind/filemap.c:334 (process_target_file 内)
  • **呼び出し先**:
  • filehash_insert (ハッシュテーブル操作)
  • pg_strdup (メモリ確保)
  • isRelDataFile (ファイル種別判定)

関連データ構造

  • file_entry_t: ファイルのメタデータおよび同期アクションを保持する構造体。
  • filehash: ファイルパスをキーとするハッシュテーブル。

失敗モードと境界条件

  • メモリ不足時には pg_strdup 等の内部関数によりプロセスが終了する可能性があります。
  • path が NULL の場合の挙動は未定義です。

保守担当者向けメモ

  • 本関数は filemap.c 内のファイル処理の起点となります。エントリの初期化ロジックを変更する場合は、file_entry_t の構造体定義と整合性を取る必要があります。
