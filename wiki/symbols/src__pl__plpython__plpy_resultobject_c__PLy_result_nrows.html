PLy_result_nrows

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_resultobject.c:217

目的

PLyResultObject が保持するクエリ結果の行数を表すPythonオブジェクトを返します。

シグネチャまたは形状

static PyObject *PLy_result_nrows(PyObject *self, PyObject *args)

入力と出力

  • **入力**:
  • self: PLyResultObject へのポインタ（PyObject としてキャストされたもの）。
  • args: Pythonの引数タプル（本関数では使用されません）。
  • **出力**:
  • ob->nrows: 行数を保持するPythonオブジェクト。参照カウントがインクリメントされた状態で返されます。

前提条件と不変条件

  • self は PLyResultObject 型のインスタンスである必要があります。
  • PLyResultObject 構造体内の nrows フィールドは、有効なPythonオブジェクトを指している必要があります。

アルゴリズムまたは振る舞い

1. 引数 self を PLyResultObject * にキャストします。 2. 構造体メンバ ob->nrows の参照カウントを Py_INCREF で増やします。 3. インクリメントされた ob->nrows を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_resultobject.c 内のメソッド定義テーブル（PLyResult_spec に関連する定義）から参照されています。
  • **呼び出し先**:
  • Python C API: Py_INCREF

関連データ構造

  • PLyResultObject: クエリ結果を保持するC構造体。nrows メンバを保持します。

失敗モードと境界条件

  • self が PLyResultObject ではない場合、キャストにより未定義の動作を引き起こす可能性があります。
  • ob->nrows が NULL の場合、Py_INCREF でセグメンテーション違反が発生します。

保守担当者向けメモ

  • 本関数は METH_VARARGS 形式のメソッドとして登録されています。
  • 戻り値の参照カウント管理はPythonの規約に従い、呼び出し元が所有権を持つように設計されています。
