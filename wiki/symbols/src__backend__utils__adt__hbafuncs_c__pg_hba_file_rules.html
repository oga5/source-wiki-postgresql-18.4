pg_hba_file_rules

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/hbafuncs.c:448

目的

pg_hba.conf の内容を SQL クエリから参照可能にするための Set-Returning Function (SRF) です。

シグネチャまたは形状

Datum pg_hba_file_rules(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (SQL 関数呼び出し規約に基づく引数)
  • **出力**: PG_RETURN_NULL() (関数自体は Tuplestore を介して行セットを返すため、戻り値としては NULL を返す)

前提条件と不変条件

  • InitMaterializedSRF を使用して、結果を保持するための Tuplestore を初期化する必要があります。
  • クエリ実行中に pg_hba.conf が変更されても一貫性を保つため、Materialize モードで動作します。

アルゴリズムまたは振る舞い

1. InitMaterializedSRF を呼び出し、結果格納用の Tuplestore を作成します。 2. fcinfo->resultinfo から ReturnSetInfo を取得します。 3. fill_hba_view を呼び出し、設定ファイルの内容をパースして Tuplestore に書き込みます。 4. PG_RETURN_NULL() を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL ビュー pg_hba_file_rules から呼び出されます。
  • **呼び出し先**:
  • InitMaterializedSRF (PostgreSQL 内部関数)
  • fill_hba_view (同ファイル内のヘルパー関数)

関連データ構造

  • ReturnSetInfo: SRF の結果セット情報を保持する構造体。
  • Tuplestore: 構築された行セットを保持するメモリ/ディスク上の構造体。

失敗モードと境界条件

  • 設定ファイルの読み込みやパースに失敗した場合の挙動は、呼び出し先の fill_hba_view の実装に依存します。
  • Tuplestore の構築失敗時は、PostgreSQL のエラーハンドリング機構により例外がスローされます。

保守担当者向けメモ

  • 本関数は読み取り専用のビューをサポートするものであり、設定ファイルの書き込みや再読み込みロジックは含みません。
  • Tuplestore を使用することで、クエリ実行中のファイル変更に対する一貫性を保証しています。
