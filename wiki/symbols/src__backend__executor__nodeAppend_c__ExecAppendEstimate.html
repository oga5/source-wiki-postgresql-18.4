ExecAppendEstimate

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAppend.c:498

目的

並列クエリ実行時において、Append ノードの状態を共有メモリ（DSM）に保持するために必要なメモリサイズを見積もり、共有メモリの割り当て計画に登録する。

シグネチャまたは形状

void ExecAppendEstimate(AppendState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: AppendState 構造体へのポインタ。子プランの数（as_nplans）の情報源となる。
  • pcxt: ParallelContext 構造体へのポインタ。共有メモリの推定器（estimator）を保持する。
  • **出力**: なし（副作用として pcxt->estimator が更新される）。

前提条件と不変条件

  • node が指す AppendState は初期化済みであること。
  • node->as_nplans には、この Append ノードが管理するサブプランの数が正しく格納されていること。

アルゴリズムまたは振る舞い

1. ParallelAppendState 構造体のサイズを計算する。具体的には、pa_finished メンバまでのオフセットに、サブプラン数（as_nplans）分の bool 型配列サイズを加算する。 2. 計算したサイズを node->pstate_len に格納する。 3. shm_toc_estimate_chunk を呼び出し、計算したサイズ分を共有メモリのチャンクとして見積もりに加える。 4. shm_toc_estimate_keys を呼び出し、共有メモリのキーとして 1 つ分を見積もりに加える。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:270（ExecParallelEstimate からの呼び出しが確認されている）。
  • **呼び出し先**: add_size, shm_toc_estimate_chunk, shm_toc_estimate_keys（いずれも共有メモリ管理用のユーティリティ関数）。

関連データ構造

  • AppendState: Append ノードの実行時状態を保持する構造体。
  • ParallelAppendState: 並列実行時にワーカー間で共有される状態。
  • ParallelContext: 並列実行のコンテキスト。

失敗モードと境界条件

  • node->as_nplans が 0 の場合でも、ParallelAppendState のベースサイズ分のメモリが確保される。
  • メモリ計算時にオーバーフローが発生した場合、add_size が適切に処理を行う（PostgreSQL の標準的なメモリ計算ルールに従う）。

保守担当者向けメモ

  • 本関数は並列実行の準備フェーズで実行される。Append ノードのサブプラン数に変更があった場合は、本関数でのサイズ計算ロジックも整合性を保つ必要がある。
