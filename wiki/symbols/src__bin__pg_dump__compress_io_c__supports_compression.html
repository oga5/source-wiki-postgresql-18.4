supports_compression

定義

  • **種別**: 関数
  • **定義箇所**: src/bin/pg_dump/compress_io.c:86

目的

指定された圧縮アルゴリズムが現在のビルドでサポートされているかを検証し、サポートされていない場合はエラーメッセージを返します。

シグネチャまたは形状

char *supports_compression(const pg_compress_specification compression_spec)

入力と出力

  • **入力**: pg_compress_specification 型の構造体（検証対象のアルゴリズム情報を含む）。
  • **出力**:
  • サポートされている場合: NULL
  • サポートされていない場合: エラー内容を示す動的に割り当てられた文字列（psprintf による）。

前提条件と不変条件

  • PG_COMPRESSION_NONE は常にサポートされているとみなされます。
  • その他のアルゴリズム（GZIP, LZ4, ZSTD）は、コンパイル時のフラグ（HAVE_LIBZ, USE_LZ4, USE_ZSTD）が定義されている場合にのみサポート対象となります。

アルゴリズムまたは振る舞い

1. compression_spec からアルゴリズム種別を取得します。 2. コンパイル時に有効化されたライブラリに基づき、該当アルゴリズムがサポートされているかフラグ（supported）を立てます。 3. supported が false の場合、get_compress_algorithm_name を用いてアルゴリズム名を取得し、エラーメッセージを生成して返します。 4. サポートされている場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_archiver.c (2箇所)
  • src/bin/pg_dump/pg_dump.c
  • **呼び出し先**:
  • get_compress_algorithm_name (アルゴリズム名取得用)
  • psprintf (エラーメッセージ生成用)

関連データ構造

  • pg_compress_specification: 圧縮仕様を保持する構造体。
  • pg_compress_algorithm: アルゴリズム種別を定義する列挙型。

失敗モードと境界条件

  • 未知のアルゴリズムが指定された場合、サポート対象外としてエラーメッセージが返されます。
  • メモリ不足により psprintf が失敗した場合の挙動は、当該関数の実装に依存します。

保守担当者向けメモ

  • 新しい圧縮アルゴリズムを追加する際は、本関数の条件分岐に #ifdef ブロックを追加してください。
  • 戻り値が NULL でない場合、呼び出し元でその文字列を解放する必要があります。
