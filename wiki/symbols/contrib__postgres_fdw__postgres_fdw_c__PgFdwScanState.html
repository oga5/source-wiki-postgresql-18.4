PgFdwScanState

定義

  • 種別: struct
  • 定義箇所: contrib/postgres_fdw/postgres_fdw.c:139

目的

外部テーブルまたは外部結合のスキャン実行中における状態を保持するための構造体です。リモートサーバーへの接続情報、実行中のクエリ、フェッチしたタプルのバッファ、およびメモリ管理コンテキストを管理します。

シグネチャまたは形状

typedef struct PgFdwScanState {
    Relation    rel;
    TupleDesc   tupdesc;
    AttInMetadata *attinmeta;
    char       *query;
    List       *retrieved_attrs;
    PGconn     *conn;
    PgFdwConnState *conn_state;
    unsigned int cursor_number;
    bool        cursor_exists;
    int         numParams;
    FmgrInfo   *param_flinfo;
    List       *param_exprs;
    const char **param_values;
    HeapTuple  *tuples;
    int         num_tuples;
    int         next_tuple;
    int         fetch_ct_2;
    bool        eof_reached;
    bool        async_capable;
    MemoryContext batch_cxt;
    MemoryContext temp_cxt;
    int         fetch_size;
} PgFdwScanState;

入力と出力

  • **入力**: ForeignScan ノードの fdw_state フィールドを介してエグゼキュータから参照されます。
  • **出力**: リモートからフェッチされたタプルが tuples 配列に格納され、next_tuple を通じて順次返されます。

前提条件と不変条件

  • rel は外部テーブルスキャンの場合は有効なリレーションキャッシュエントリを指しますが、外部結合スキャンの場合は NULL となります。
  • batch_cxt は現在のタプルバッチを保持し、temp_cxt はタプルごとの一時的なデータ処理に使用されます。

アルゴリズムまたは振る舞い

1. **初期化**: palloc0 によりメモリが確保され、リモートクエリの実行準備が行われます。 2. **実行**: conn を使用してリモートサーバーと通信し、query を実行します。 3. **フェッチ**: fetch_size に基づいてリモートからタプルを取得し、tuples 配列に格納します。 4. **反復**: next_tuple をインクリメントしながら、num_tuples に達するまでタプルを返します。eof_reached が true になるまでこのプロセスを繰り返します。 5. **非同期処理**: async_capable が設定されている場合、非同期実行ロジックが適用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: postgres_fdw.c 内の postgresBeginForeignScan（初期化）、postgresIterateForeignScan（反復）、postgresReScanForeignScan、postgresEndForeignScan 等から node->fdw_state として参照されます。
  • **呼び出し先**: 構造体内部で PGconn（libpq）やメモリコンテキスト管理関数を直接的または間接的に利用します。

関連データ構造

  • PgFdwConnState: 接続ごとの追加状態を保持します。
  • Relation, TupleDesc, HeapTuple: PostgreSQL のコアデータ構造。
  • MemoryContext: メモリ管理に使用されます。

失敗モードと境界条件

  • リモート接続が切断された場合、conn を介した操作が失敗します。
  • cursor_exists が正しく管理されていない場合、カーソルの二重作成や未作成のカーソルへのアクセスが発生する可能性があります。
  • fetch_ct_2 はフェッチ回数の最適化に使用され、境界条件（初回フェッチ、EOF 到達時）の制御に影響します。

保守担当者向けメモ

  • 本構造体は postgres_fdw.c 内で広範囲に利用されています。フィールドを追加・変更する場合は、postgresBeginForeignScan での初期化処理と postgresEndForeignScan でのクリーンアップ処理（メモリ解放等）が適切に行われているか確認してください。
