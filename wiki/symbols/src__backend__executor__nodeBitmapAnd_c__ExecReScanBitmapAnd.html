ExecReScanBitmapAnd

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapAnd.c:200

目的

BitmapAnd ノードの再スキャン処理を実行し、保持しているすべてのサブプランに対してパラメータ変更の伝播および再スキャン指示を行う。

シグネチャまたは形状

void ExecReScanBitmapAnd(BitmapAndState *node)

入力と出力

  • **入力**: BitmapAndState *node （再スキャン対象の BitmapAnd 実行状態構造体）
  • **出力**: なし（副作用としてサブプランの状態を更新）

前提条件と不変条件

  • node->bitmapplans に格納された各サブプランは、ExecInitBitmapAnd によって初期化済みであること。
  • node->ps.chgParam が設定されている場合、その変更内容が各サブプランへ適切に伝播される必要がある。

アルゴリズムまたは振る舞い

1. node->nplans 個のサブプランを順次走査する。 2. 親ノード（BitmapAnd）に chgParam が存在する場合、UpdateChangedParamSet を呼び出し、各サブプランの chgParam に変更を反映させる。 3. サブプランの chgParam が NULL である（＝パラメータ変更による再スキャンが不要な）場合、ExecReScan を呼び出して当該サブプランを明示的に再スキャンする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c 内の ExecReScan 経由で呼び出されることが示唆されている。
  • **呼び出し先**:
  • UpdateChangedParamSet
  • ExecReScan （各サブプランに対して）

関連データ構造

  • BitmapAndState: BitmapAnd ノードの実行状態を保持する構造体。
  • PlanState: 各サブプランの実行状態を保持する構造体。

失敗モードと境界条件

  • node->bitmapplans が空の場合、ループは実行されず何も行わない。
  • node->ps.chgParam が NULL の場合、サブプランへのパラメータ伝播はスキップされる。

保守担当者向けメモ

  • 本関数は ExecProcNode 呼び出し規約をサポートしていないため、実行時には MultiExecProcNode を介する必要がある点に注意すること。
  • ExecReScan がサブプランの再スキャンを自動的に処理しないため、本関数内で明示的に UpdateChangedParamSet を呼び出し、変更を伝播させる設計となっている。
