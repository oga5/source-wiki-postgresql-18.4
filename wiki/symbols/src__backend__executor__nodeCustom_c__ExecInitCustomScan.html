ExecInitCustomScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeCustom.c:25

目的

カスタムスキャン実行ノード（CustomScanState）を初期化し、拡張機能が提供するカスタムロジックを標準の実行エンジンに統合する。

シグネチャまたは形状

CustomScanState *ExecInitCustomScan(CustomScan *cscan, EState *estate, int eflags)

入力と出力

  • **入力**:
  • cscan: 初期化対象の CustomScan プランノード。
  • estate: 実行状態（EState）。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの CustomScanState 構造体へのポインタ。

前提条件と不変条件

  • cscan->methods->CreateCustomScanState が有効な CustomScanState を返す必要がある。
  • 拡張機能側で CustomScanState を拡張している場合、その構造体の先頭フィールドとして CustomScanState が配置されている必要がある。

アルゴリズムまたは振る舞い

1. **状態生成**: 拡張機能の CreateCustomScanState を呼び出し、CustomScanState を割り当てる。 2. **基本設定**: ScanState の標準フィールド（プラン、実行状態、ExecProcNode エントリポイント）を初期化する。 3. **環境構築**: 式評価コンテキストの作成、および必要に応じてスキャン対象リレーションをオープンする。 4. **スロット初期化**: CustomScanState で指定された slotOps、またはデフォルトの TTSOpsVirtual を使用してタプルスロットを初期化する。 5. **型定義**: custom_scan_tlist の有無に基づき、スキャンタプルの型（TupleDesc）を決定し、スロットに適用する。 6. **射影と制約**: スキャン射影情報の割り当てと、プランノードの qual（制約）を初期化する。 7. **拡張初期化**: 最後に拡張機能側の BeginCustomScan コールバックを呼び出し、カスタムロジックによる最終初期化を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内の ExecInitCustomScan 呼び出し箇所。
  • **呼び出し先**:
  • 拡張機能が提供する CreateCustomScanState（cscan->methods 経由）。
  • 拡張機能が提供する BeginCustomScan（css->methods 経由）。
  • 標準ライブラリ関数: ExecAssignExprContext, ExecOpenScanRelation, ExecInitScanTupleSlot, ExecTypeFromTL, ExecInitResultTupleSlotTL, ExecAssignScanProjectionInfoWithVarno, ExecInitQual。

関連データ構造

  • CustomScan: プランノード。
  • CustomScanState: 実行時の状態保持構造体。
  • CustomExecMethods: 拡張機能が実装するコールバック関数テーブル。
  • TupleTableSlotOps: タプルスロットの操作関数群。

失敗モードと境界条件

  • scanrelid > 0 の場合、ExecOpenScanRelation が失敗するとエラーが発生する。
  • cscan->methods や css->methods が適切に設定されていない場合、NULLポインタ参照が発生する可能性がある。

保守担当者向けメモ

  • 本関数は「薄いラッパー」であり、実際の初期化ロジックの大部分は拡張機能側の CreateCustomScanState および BeginCustomScan に依存している。
  • 拡張機能が独自のデータを保持する場合、CreateCustomScanState で適切なメモリサイズを確保する必要がある。
