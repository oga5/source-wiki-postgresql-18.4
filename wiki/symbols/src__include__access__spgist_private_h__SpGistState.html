SpGistState

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgist_private.h:144

目的

SP-GiSTインデックスの操作（挿入・検索）において、インデックスの構成情報や演算子クラスの定義、および実行時の作業領域を保持する共通状態構造体です。

シグネチャまたは形状

typedef struct SpGistState
{
    Relation    index;
    spgConfigOut config;
    SpGistTypeDesc attType;
    SpGistTypeDesc attLeafType;
    SpGistTypeDesc attPrefixType;
    SpGistTypeDesc attLabelType;
    TupleDesc   leafTupDesc;
    char       *deadTupleStorage;
    TransactionId redirectXid;
    bool        isBuild;
} SpGistState;

入力と出力

  • **入力**: initSpGistState 関数により、対象となる Relation からインデックスのメタデータや演算子クラスの構成情報が読み込まれ、本構造体に格納されます。
  • **出力**: インデックス操作関数（spgdoinsert 等）が、本構造体を参照してタプルの構築やページ操作を行います。

前提条件と不変条件

  • leafTupDesc は通常インデックスの tupdesc を指しますが、常に一致するとは限りません。
  • deadTupleStorage は spgFormDeadTuple 実行時の作業用メモリとして使用されます。
  • インデックス構築中（isBuild が true）か否かで、一部の処理ロジックが分岐します。

アルゴリズムまたは振る舞い

  • インデックス操作のライフサイクル全体を通じて、演算子クラスの構成情報（config）や各データ型（attType 等）へのアクセスポイントを提供します。
  • redirectXid は、インデックス再編成時に REDIRECT タプルを生成する際のトランザクションIDとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: spgdoinsert、addNode、addLeafTuple、checkSplitConditions、moveLeafs、doPickSplit、spgMatchNodeAction、spgAddNodeAction、spgSplitNodeAction、spginsert 等の src/backend/access/spgist/ 配下の関数群。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありませんが、各操作関数が本構造体のメンバを参照して処理を決定します。

関連データ構造

  • Relation: 操作対象のインデックスリレーション。
  • spgConfigOut: 演算子クラスの構成情報。
  • SpGistTypeDesc: インデックス化される値や、リーフ/内部タプルの型記述子。
  • TupleDesc: リーフタプルの記述子。

失敗モードと境界条件

  • 演算子クラスの構成情報が正しくロードされていない場合、インデックス操作が正しく行われません。
  • メモリ不足時には deadTupleStorage の確保や操作に失敗する可能性があります。

保守担当者向けメモ

  • 本構造体はSP-GiSTの全操作の基盤となるため、メンバの追加や変更は spgdoinsert.c や spgutils.c 等の広範な実装に影響を与えます。
  • REDIRECT タプル生成時の redirectXid の扱いは、同時実行制御において重要です。
