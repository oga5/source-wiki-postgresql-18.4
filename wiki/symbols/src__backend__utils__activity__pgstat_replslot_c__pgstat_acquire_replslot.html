pgstat_acquire_replslot

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_replslot.c:145

目的

指定されたレプリケーションスロットに対応する統計エントリへの参照を取得し、統計情報の操作を可能にする。

シグネチャまたは形状

void pgstat_acquire_replslot(ReplicationSlot *slot)

入力と出力

  • **入力**: ReplicationSlot *slot - 統計対象となるレプリケーションスロット構造体へのポインタ。
  • **出力**: なし（副作用として統計サブシステム内の参照を確保する）。

前提条件と不変条件

  • 呼び出し元は、対象の ReplicationSlot が有効であることを保証する必要がある。
  • 内部で pgstat_get_entry_ref を呼び出しており、統計サブシステムの共有メモリ状態に依存する。

アルゴリズムまたは振る舞い

1. ReplicationSlotIndex(slot) を用いて、対象スロットのインデックスを取得する。 2. pgstat_get_entry_ref を呼び出し、PGSTAT_KIND_REPLSLOT（レプリケーションスロット種別）として、該当インデックスの統計エントリ参照を確保する。 3. 第4引数に true を指定することで、エントリが存在しない場合に作成を試みる動作となる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/slot.c:690, 695 からの呼び出しが確認されている。
  • **呼び出し先**: pgstat_get_entry_ref（統計サブシステムの内部関数）。

関連データ構造

  • ReplicationSlot: レプリケーションスロットの制御構造体。
  • PGSTAT_KIND_REPLSLOT: 統計種別を識別するための定数。

失敗モードと境界条件

  • slot が NULL の場合の動作は定義されていない。
  • 統計サブシステムの共有メモリが枯渇している場合、pgstat_get_entry_ref 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、スロットの統計情報に対する操作を開始する前の保護的な役割を果たす。
  • 物理レプリケーションスロットではなく、論理レプリケーションスロットの統計管理を前提としている。
