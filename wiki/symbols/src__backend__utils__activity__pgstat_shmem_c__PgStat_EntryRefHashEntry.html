PgStat_EntryRefHashEntry

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/activity/pgstat_shmem.c:24

目的

各バックエンドが保持する共有統計情報への参照（PgStat_EntryRef）を管理するための、ローカルキャッシュ用ハッシュテーブルの要素構造体です。

シグネチャまたは形状

typedef struct PgStat_EntryRefHashEntry
{
	PgStat_HashKey key;			/* ハッシュキー */
	char		status;			/* simplehash用ステータス */
	PgStat_EntryRef *entry_ref;	/* 統計エントリ参照へのポインタ */
} PgStat_EntryRefHashEntry;

入力と出力

本構造体はデータ保持を目的としており、関数としての入力・出力はありません。ハッシュテーブルの要素として、キー（key）に基づいた統計エントリ参照（entry_ref）の検索・格納に使用されます。

前提条件と不変条件

  • lib/simplehash.h を用いたハッシュテーブルの要素として定義されています。
  • status フィールドは simplehash の内部管理用として使用されます。

アルゴリズムまたは振る舞い

  • バックエンドが統計情報にアクセスする際、この構造体を介してローカルキャッシュ（pgStatEntryRefHash）を検索します。
  • キャッシュヒット時は、保持している entry_ref を通じて共有メモリ上の統計情報へアクセスします。

呼び出し元と呼び出し先

  • **呼び出し元**: pgstat_shmem.c 内の統計情報検索・管理関数（pgstat_get_entry_ref 等）から参照されます。
  • **呼び出し先**: 構造体定義のため、特定の関数を呼び出すことはありません。
  • **注記**: 具体的な呼び出し関係の詳細は、ソースコード内の各関数（match_db 等）におけるポインタ操作として現れますが、本構造体自体はデータコンテナです。

関連データ構造

  • PgStat_HashKey: ハッシュテーブルの検索キー。
  • PgStat_EntryRef: 共有メモリ上の統計エントリへの参照を保持する構造体。
  • pgStatEntryRefHash: 本構造体を要素として格納するローカルキャッシュ用ハッシュテーブル。

失敗モードと境界条件

  • ハッシュテーブルのメモリ枯渇や、キーの衝突によるパフォーマンス低下が考えられますが、これらは simplehash の実装に依存します。

保守担当者向けメモ

  • SH_ELEMENT_TYPE として simplehash.h に登録されているため、ハッシュテーブルの定義を変更する際は注意してください。
  • entry_ref が指す先のライフサイクルは、共有メモリのGCメカニズムと密接に関連しています。参照の無効化や解放時には、このキャッシュエントリの整合性も考慮する必要があります。
