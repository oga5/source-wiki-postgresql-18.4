PG_Lock_Status

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/lockfuncs.c:57

目的

SQL関数 pg_lock_status がロックマネージャから取得したロック状態を保持し、SRF（Set Returning Function）の呼び出し間で状態を維持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    LockData          *lockData;      /* lmgrからの状態データ */
    int                currIdx;       /* 現在のPROCLOCKインデックス */
    PredicateLockData *predLockData;  /* 述語ロックの状態データ */
    int                predLockIdx;   /* 現在の述語ロックインデックス */
} PG_Lock_Status;

入力と出力

  • 入力: pg_lock_status 関数内で palloc を用いてメモリ確保され、funcctx->user_fctx に格納されます。
  • 出力: なし（構造体自体は内部状態保持用）。

前提条件と不変条件

  • pg_lock_status の実行中にのみ有効です。
  • ロックマネージャから取得したスナップショット（lockData および predLockData）を保持するため、呼び出し元はこれらが破壊的に読み進められることを前提とします。

アルゴリズムまたは振る舞い

1. pg_lock_status の初回呼び出し時に palloc でインスタンス化されます。 2. SRF_PERCALL_SETUP を通じて、保持しているロックと待機中のロックを順次タプルに変換するためのインデックス（currIdx, predLockIdx）を管理します。 3. ロックマネージャから取得した全ロック状態を反復処理し、終了するまで状態を保持し続けます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/lockfuncs.c 内の pg_lock_status 関数。
  • 呼び出し先: 構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • LockData: ロックマネージャから取得したロック情報のスナップショット。
  • PredicateLockData: 述語ロック（SIReadLock）の状態データ。

失敗モードと境界条件

  • メモリ不足: palloc 失敗時にエラーが発生します。
  • 状態の不整合: ロックマネージャの状態が反復中に変更された場合、インデックスが範囲外になる可能性がありますが、これは pg_lock_status 側のロジックで制御されます。

保守担当者向けメモ

  • pg_lock_status はロックマネージャのデータを破壊的に読み進めるため、本構造体のインデックス管理が正しく行われているか確認してください。
  • 述語ロックは通常のロックと管理体系が異なるため、本構造体内で個別にインデックスを保持しています。
