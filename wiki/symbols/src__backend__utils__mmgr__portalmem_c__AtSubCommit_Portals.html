AtSubCommit_Portals

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/portalmem.c:942

目的

サブトランザクションのコミット時に、そのサブトランザクションに関連付けられたポータルの所有権を親トランザクションへ引き継ぎ、管理情報を更新する。

シグネチャまたは形状

void AtSubCommit_Portals(SubTransactionId mySubid,
                         SubTransactionId parentSubid,
                         int parentLevel,
                         ResourceOwner parentXactOwner)

入力と出力

  • mySubid: コミットされるサブトランザクションのID。
  • parentSubid: 親トランザクション（または親サブトランザクション）のID。
  • parentLevel: 親のトランザクションレベル。
  • parentXactOwner: 親の ResourceOwner。
  • 出力なし（void）。

前提条件と不変条件

  • PortalHashTable が初期化されていること。
  • mySubid に関連付けられたポータルが存在する場合、そのポータルは PortalHashTable 内で管理されていること。

アルゴリズムまたは振る舞い

1. PortalHashTable を走査し、全ポータルを順次確認する。 2. ポータルの createSubid が mySubid と一致する場合：

  • createSubid を parentSubid に更新する。
  • createLevel を parentLevel に更新する。
  • ポータルが resowner を保持している場合、ResourceOwnerNewParent を呼び出し、リソースの所有権を parentXactOwner に移管する。

    • 3. ポータルの activeSubid が mySubid と一致する場合：

  • activeSubid を parentSubid に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c:5147（サブトランザクションコミット処理の一部）。
  • **呼び出し先**: hash_seq_init, hash_seq_search（ハッシュテーブル走査用）、ResourceOwnerNewParent（リソース所有権の移管）。

関連データ構造

  • PortalHashTable: 全ポータルを管理するハッシュテーブル。
  • PortalHashEnt: ハッシュテーブルの各エントリ。
  • Portal: ポータル構造体（createSubid, createLevel, resowner, activeSubid を保持）。

失敗モードと境界条件

  • ハッシュテーブルが空の場合、何も処理せず正常終了する。
  • resowner が NULL のポータルについては、所有権の移管処理はスキップされる。

保守担当者向けメモ

  • 本関数はサブトランザクションのコミット時にのみ実行される。アボート時の処理については AtSubAbort_Portals を参照すること。
  • ResourceOwnerNewParent の呼び出しにより、リソースの生存期間が親トランザクションの終了まで延長される点に注意。
