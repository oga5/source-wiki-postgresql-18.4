inner_int_overlap

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_tool.c:49

目的

2つの整数配列間に共通の要素が存在するか（重なりがあるか）を判定する。

シグネチャまたは形状

bool inner_int_overlap(ArrayType *a, ArrayType *b)

入力と出力

  • **入力**:
  • a: ArrayType* 型のソート済み整数配列。
  • b: ArrayType* 型のソート済み整数配列。
  • **出力**:
  • bool: 両方の配列に共通する要素が少なくとも1つ存在する場合は true、それ以外は false を返す。

前提条件と不変条件

  • 入力される ArrayType 配列 a および b は、あらかじめソート済みである必要がある。ソートされていない配列を渡した場合の動作は保証されない。
  • 呼び出し側で CHECKARRVALID マクロ等を用いて、入力が有効な整数配列であることを確認済みであること。

アルゴリズムまたは振る舞い

1. ARRNELEMS マクロで各配列の要素数を取得し、ARRPTR マクロでデータポインタを取得する。 2. 2つのポインタを同時に走査し、以下のロジックで比較を行う（線形時間 $O(N+M)$）:

  • da[i] < db[j] の場合: a のポインタをインクリメント。
  • da[i] == db[j] の場合: 共通要素を発見したとみなし、直ちに true を返す。
  • da[i] > db[j] の場合: b のポインタをインクリメント。

    • 3. いずれかの配列の末尾に到達し、共通要素が見つからなかった場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_gist.c:77
  • contrib/intarray/_int_op.c:116
  • **呼び出し先**:
  • 直接的な関数呼び出しはなし（マクロ ARRNELEMS, ARRPTR を使用）。

関連データ構造

  • ArrayType: PostgreSQL の標準的な配列構造体。

失敗モードと境界条件

  • 空配列が入力された場合、ループ条件 i < na && j < nb が即座に偽となり、正しく false を返す。
  • 入力配列がソートされていない場合、アルゴリズムは共通要素を見逃す可能性がある。

保守担当者向けメモ

  • 本関数は線形時間で動作するよう設計されている。計算量を維持するため、ロジック変更時は走査の効率に注意すること。
  • _int_op.c や _int_gist.c から頻繁に呼び出されるため、パフォーマンスが重要なパスである。
