WalRcvState

定義

  • 種別: enum
  • 定義箇所: src/include/replication/walreceiver.h:45

目的

WALレシーバープロセスのライフサイクルにおける現在の実行状態を管理・追跡するために使用されます。

シグネチャまたは形状

typedef enum {
    WALRCV_STOPPED,
    WALRCV_STARTING,
    WALRCV_STREAMING,
    WALRCV_WAITING,
    WALRCV_RESTARTING,
    WALRCV_STOPPING,
} WalRcvState;

入力と出力

  • 入力: プロセスの制御ロジック（スタートアッププロセスやシグナルハンドラ）による状態遷移の要求。
  • 出力: 現在の状態値。WalRcvGetStateString を通じてデバッグやログ出力用の文字列として取得可能。

前提条件と不変条件

  • WalRcvData 構造体の一部として共有メモリ上に保持されます。
  • 共有メモリ上のデータであるため、アクセスには適切な同期（スピンロックやアトミック操作）が必要です。

アルゴリズムまたは振る舞い

プロセスの状態遷移を以下の通り定義します。

  • WALRCV_STOPPED: プロセスが停止しており、再起動不可。
  • WALRCV_STARTING: プロセス起動直後、初期化中。
  • WALRCV_STREAMING: プライマリからWALを受信中。
  • WALRCV_WAITING: ストリーミングを停止し、次の指示を待機中。
  • WALRCV_RESTARTING: ストリーミングの再開要求中。
  • WALRCV_STOPPING: 停止要求を受け取ったが、プロセスはまだ実行中。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/walreceiver.c および src/backend/replication/walreceiverfuncs.c 内のプロセス制御ロジック。
  • 呼び出し先: WalRcvGetStateString が本型を引数として受け取ります。

関連データ構造

  • WalRcvData: 本列挙型をメンバ walRcvState として保持する共有メモリ上の管理構造体。

失敗モードと境界条件

  • 共有メモリ上の値として保持されるため、プロセス間での競合が発生する可能性があります。
  • 状態遷移の順序が不正な場合（例: STOPPED から直接 STREAMING への遷移など）、レプリケーションプロセスの異常終了やデッドロックを招く恐れがあります。

保守担当者向けメモ

  • 本列挙型は WalRcvData 構造体と密接に関連しています。状態遷移ロジックを変更する場合は、src/backend/replication/walreceiver.c のメインループおよびスタートアッププロセス側の制御ロジックを併せて確認してください。
