mic_to_win1251

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/cyrillic_and_mic/cyrillic_and_mic.c:389

目的

PostgreSQL 内部のマルチバイト表現（MULE_INTERNAL）から WIN1251 エンコーディングへ文字列を変換します。

シグネチャまたは形状

Datum mic_to_win1251(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を取得
  • src: 変換元の文字列（CSTRING）
  • dest: 変換後の文字列を格納するバッファ（CSTRING）
  • len: 変換対象の長さ（INT32）
  • noError: エラーハンドリングフラグ（BOOL）
  • **出力**: 変換されたバイト数を PG_RETURN_INT32 で返却

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_MULE_INTERNAL, PG_WIN1251) により、ソースエンコーディングが PG_MULE_INTERNAL、宛先エンコーディングが PG_WIN1251 であることが検証されます。

アルゴリズムまたは振る舞い

1. mic2latin_with_table 関数を呼び出し、変換を実行します。 2. 変換には koi2win1251 マッピングテーブルを使用します。 3. 変換後のバイト数を取得し、呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: mic2latin_with_table（変換処理の実行）。

関連データ構造

  • koi2win1251: 変換に使用される static const unsigned char 型のマッピングテーブル。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、PostgreSQL のエラーハンドリング機構により処理が中断されます。
  • noError フラグの設定に応じて、変換エラー時の挙動が制御されます。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_ARGS を引数に取るため、PostgreSQL の関数呼び出し規約に従って利用してください。
  • 変換ロジックの変更が必要な場合は、マッピングテーブル koi2win1251 の整合性を確認してください。
