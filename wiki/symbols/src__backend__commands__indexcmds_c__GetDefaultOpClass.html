GetDefaultOpClass

定義

  • 種別: function
  • 定義箇所: src/backend/commands/indexcmds.c:2346

目的

指定されたデータ型とアクセスメソッドに対して、システムカタログ上で「デフォルト」としてマークされている演算子クラス（Operator Class）の OID を取得する。

シグネチャまたは形状

Oid GetDefaultOpClass(Oid type_id, Oid am_id)

入力と出力

  • **入力**:
  • type_id: 対象となるデータ型の OID。ドメイン型の場合はその基底型が使用される。
  • am_id: 対象となるインデックスアクセスメソッドの OID。
  • **出力**:
  • 適合するデフォルト演算子クラスの OID。該当なしの場合は InvalidOid を返す。

前提条件と不変条件

  • pg_opclass システムカタログが読み取り可能であること。
  • 入力された type_id がドメイン型である場合、内部で getBaseType により基底型へ解決される。

アルゴリズムまたは振る舞い

1. pg_opclass を am_id でスキャンし、opcdefault が真であるエントリを抽出する。 2. 以下の優先順位で適合する演算子クラスを選択する:

  • **完全一致**: opcintype が type_id と一致するもの。
  • **バイナリ互換**: type_id から opcintype へバイナリ変換可能なもの。
  • 優先型（Preferred Type）への変換が可能な場合はそれを優先する。

    • 3. 複数の完全一致が存在する場合はエラー（ERRCODE_DUPLICATE_OBJECT）を発生させる。 4. 最終的に単一の候補が特定できた場合のみその OID を返し、それ以外は InvalidOid を返す。

関連データ構造

  • pg_opclass: 演算子クラス情報を保持するシステムカタログ。
  • Relation, SysScanDesc, HeapTuple: システムカタログスキャンに使用される構造体。

失敗モードと境界条件

  • pg_opclass 内に同一型・同一アクセスメソッドに対して複数のデフォルト演算子クラスが定義されている場合、ereport によりエラー終了する。
  • 適合する演算子クラスが見つからない場合、InvalidOid を返す。

保守担当者向けメモ

  • varchar 型のように、複数の型に対してバイナリ互換性を持つケースでの曖昧さを解消するため、IsPreferredType を用いたタイブレーカー（優先順位付け）が実装されている。このロジックを変更する際は、既存の型変換ルールへの影響に注意すること。
