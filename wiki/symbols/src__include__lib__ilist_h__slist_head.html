slist_head

定義

  • 種別: struct
  • 定義箇所: src/include/lib/ilist.h:236

目的

PostgreSQLの統合型単一連結リスト（slist）の起点となる構造体です。リストの先頭ノードを保持し、リスト全体の管理ポイントとして機能します。

シグネチャまたは形状

typedef struct slist_head
{
	slist_node	head;
} slist_head;

入力と出力

  • **入力**: リスト操作関数（slist_init, slist_is_empty, slist_delete 等）の引数として、この構造体へのポインタが渡されます。
  • **出力**: リストの状態（空か否か）や、リスト内のノード操作結果がこの構造体を通じて反映されます。

前提条件と不変条件

  • slist_head は、リストが空であっても初期化されている必要があります。
  • リストに含まれるノードは、ユーザー定義構造体に埋め込まれた slist_node である必要があります。
  • リストが保持しているノードがメモリから解放される場合、そのノードは事前にリストから削除されていなければなりません。

アルゴリズムまたは振る舞い

  • slist_head は内部に slist_node を保持しており、これがリストの先頭を指します。
  • メモリ割り当てを行わず、既存の構造体内のリンクを直接操作することでリストを構成します。
  • slist_init を呼び出すことで、リストを空の状態に初期化します。

呼び出し元と呼び出し先

  • **呼び出し元（参照例）**:
  • src/backend/commands/event_trigger.c: SQLDropList として使用。
  • src/backend/utils/misc/guc.c: guc_stack_list, guc_report_list として使用。
  • src/include/executor/spi_priv.h: tuptables として使用。
  • src/backend/storage/ipc/dsm.c: on_detach コールバックリストとして使用。
  • **呼び出し先**:
  • slist_init, slist_is_empty, slist_delete, slist_check などの ilist.h で定義された操作関数が、この構造体を操作対象として受け取ります。

関連データ構造

  • slist_node: リストの各要素に埋め込まれるリンク構造体。
  • slist_iter: リストを走査するためのイテレータ構造体。

失敗モードと境界条件

  • **未初期化状態**: 初期化されていない slist_head を操作した場合の挙動は未定義です。
  • **空リスト**: slist_is_empty を使用してリストが空であることを確認可能です。
  • **整合性チェック**: ILIST_DEBUG が定義されている場合、slist_check を通じてリストの構造的整合性を検証できます。

保守担当者向けメモ

  • 本構造体はメモリを動的に確保しません。リストの寿命管理は呼び出し側の責任で行ってください。
  • slist は単一連結リストであるため、双方向リスト（dlist）と比較してメモリ消費は抑えられますが、逆方向の走査や任意のノードの定数時間削除には制限があります。
