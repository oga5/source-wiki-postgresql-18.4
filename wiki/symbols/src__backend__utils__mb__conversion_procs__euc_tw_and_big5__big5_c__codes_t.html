codes_t

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_tw_and_big5/big5.c:18

目的

BIG5コードとCNS 11643コード間の相互変換において、変換範囲の開始点とそれに対応するピア（対向）コードを保持するために使用されるデータ構造です。

シグネチャまたは形状

typedef struct
{
	unsigned short code;
	unsigned short peer;
} codes_t;

入力と出力

  • code: 変換元の文字コード（unsigned short）。
  • peer: 変換先の対応する文字コード（unsigned short）。

前提条件と不変条件

  • 本構造体は、主に範囲ベースの変換テーブル（big5Level1ToCnsPlane1 等）の要素として定義されます。
  • BinarySearchRange 関数において、変換対象のコードを検索するためのキーとして code が使用されます。

アルゴリズムまたは振る舞い

  • BinarySearchRange 関数がこの構造体の配列を受け取り、二分探索アルゴリズムを用いて指定されたコードに対応する peer を導出します。

呼び出し元と呼び出し先

  • **呼び出し元**: BinarySearchRange 関数がこの構造体の配列を引数として受け取ります。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数を呼び出すことはありません。
  • **参照元**: 以下の静的配列で定義されています。
  • big5Level1ToCnsPlane1
  • cnsPlane1ToBig5Level1
  • big5Level2ToCnsPlane2
  • cnsPlane2ToBig5Level2

関連データ構造

  • BinarySearchRange: 本構造体の配列を検索する関数。

失敗モードと境界条件

  • 構造体自体に失敗モードはありませんが、検索対象の code が配列内の範囲外である場合、BinarySearchRange は適切な変換結果を返せません。

保守担当者向けメモ

  • 新しい文字コード変換範囲を追加する場合は、本構造体を用いた静的配列を定義し、BinarySearchRange を介して検索ロジックに組み込んでください。
