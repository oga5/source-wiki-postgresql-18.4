pgstat_archiver_snapshot_cb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_archiver.c:88

目的

アーカイバ統計のスナップショット取得時に、共有メモリ上の累積値からリセットオフセットを減算し、統計リセット後の正しい差分値を算出する。

シグネチャまたは形状

void pgstat_archiver_snapshot_cb(void)

入力と出力

  • **入力**: 共有メモリ上の pgStatLocal.shmem->archiver（統計データおよびリセットオフセット）。
  • **出力**: pgStatLocal.snapshot.archiver を更新し、リセット後の差分値を格納する。

前提条件と不変条件

  • pgStatLocal が初期化されていること。
  • 統計データ取得の整合性を保つため、pgstat_copy_changecounted_stats によるロックフリーなコピーが行われること。
  • リセットオフセットの読み取りには stats_shmem->lock（共有ロック）が必要であること。

アルゴリズムまたは振る舞い

1. pgstat_copy_changecounted_stats を使用して、共有メモリから現在の統計スナップショットへデータをコピーする。 2. stats_shmem->lock を取得し、reset_offset の値をローカル変数 reset にコピーする。 3. archived_count および failed_count について、スナップショット値から reset 値を減算する。 4. 減算の結果、カウンタがリセット値と一致する場合（つまり差分がゼロの場合）、対応する最終実行時刻および WAL 名をクリア（0 に設定）する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c 内の統計スナップショット生成処理からコールバックとして参照される。
  • **呼び出し先**:
  • pgstat_copy_changecounted_stats
  • LWLockAcquire / LWLockRelease
  • memcpy

関連データ構造

  • PgStatShared_Archiver: 共有メモリ上のアーカイバ統計構造体。
  • PgStat_ArchiverStats: アーカイバ統計のデータ構造。

失敗モードと境界条件

  • 共有メモリ上の changecount が更新中の場合、pgstat_copy_changecounted_stats は再試行を行う。
  • リセットオフセットが現在の統計値より大きい場合、カウンタはアンダーフローする可能性がある（統計システム全体の設計として、リセット操作が適切に行われていることが前提）。

保守担当者向けメモ

  • 本関数は統計の「リセット」を累積値のクリアではなく、オフセットの減算によって実現するデルタ計算方式を採用している。
  • 統計項目（archived_count 等）を追加・変更する場合は、本関数内の補正ロジックにも同様の減算処理を追加する必要がある。
