HeapTupleTableSlot

定義

  • 種別: struct
  • 定義箇所: src/include/executor/tuptable.h:257

目的

物理的なヒープタプルを保持し、実行エンジン内で効率的にアクセスするためのタプルテーブルスロットの具象実装です。

シグネチャまたは形状

typedef struct HeapTupleTableSlot
{
	pg_node_attr(abstract)

	TupleTableSlot base;
	HeapTuple	tuple;			/* 物理タプル */
	uint32		off;			/* slot_deform_heap_tuple 用の保存状態 */
	HeapTupleData tupdata;		/* タプル格納用のオプションのワークスペース */
} HeapTupleTableSlot;

入力と出力

  • **入力**: TupleTableSlot を継承しており、初期化時に TupleDesc を受け取ります。tuple フィールドに物理的な HeapTuple を格納します。
  • **出力**: slot_getattr 等のインターフェースを通じて、格納されたタプルの属性値を提供します。

前提条件と不変条件

  • base フィールドが構造体の先頭に配置されており、TupleTableSlot* としてキャスト可能です。
  • tuple フィールドは、スロットが有効な間、適切なメモリ領域またはバッファを指している必要があります。

アルゴリズムまたは振る舞い

  • **タプル保持**: 物理的な HeapTuple へのポインタを保持します。
  • **遅延評価**: off フィールドを使用して、タプルから属性を抽出する際のデフォーム（deform）処理の状態を保持し、必要になるまで属性値の抽出を遅延させます。
  • **ワークスペース**: tupdata を使用して、必要に応じてタプルデータをスロット内に保持する領域を提供します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execTuples.c 内の各種スロット操作関数（ExecStoreHeapTuple 等）が、この構造体を TupleTableSlot として操作します。
  • **呼び出し先**: slot_deform_heap_tuple が off フィールドを参照して属性抽出を行います。

関連データ構造

  • TupleTableSlot: 基底構造体。
  • HeapTuple: 保持対象の物理タプル形式。
  • TupleTableSlotOps: このスロットの挙動を定義する関数ポインタテーブル（src/backend/executor/execTuples.c で実装）。

失敗モードと境界条件

  • tuple が NULL の状態で属性アクセスを行うと、未定義の動作を引き起こす可能性があります。
  • スロットがクリアされる前に tuple が指すメモリが解放された場合、ダングリングポインタとなります。

保守担当者向けメモ

  • pg_node_attr(abstract) が付与されており、直接的なインスタンス化よりも TupleTableSlot インターフェースを通じた操作が前提です。
  • off フィールドの更新ロジックは slot_deform_heap_tuple の実装に強く依存しているため、変更時は注意が必要です。
