ExecInitIncrementalSort

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIncrementalSort.c:975

目的

インクリメンタルソートを実行するための IncrementalSortState 構造体を初期化し、実行環境をセットアップする。

シグネチャまたは形状

IncrementalSortState *ExecInitIncrementalSort(IncrementalSort *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の IncrementalSort プランノード。
  • estate: 実行状態（EState）。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの IncrementalSortState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK が含まれていないこと（アサーションで検証）。
  • インクリメンタルソートは単一のソートバッチのみを保持する設計であるため、後方スキャンやマーク/リストア操作はサポートされない。

アルゴリズムまたは振る舞い

1. IncrementalSortState 構造体を割り当て、プランノードと実行状態を紐付ける。 2. 実行関数として ExecIncrementalSort を登録する。 3. 内部状態（execution_status、fullsort_state、prefixsort_state 等）を初期化する。 4. instrument が有効な場合、fullsortGroupInfo および prefixsortGroupInfo の統計カウンタをゼロクリアする。 5. 子ノードを ExecInitNode で初期化する。 6. MinimalTuple 形式のタプルスロット（スキャン用、戻り値用、ピボット用、転送用）を作成・設定する。 7. プロジェクション情報は使用しないため、ps_ProjInfo を NULL に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:326（ExecInitNode 経由）。
  • **呼び出し先**:
  • makeNode
  • ExecInitNode（子ノードの初期化）
  • ExecCreateScanSlotFromOuterPlan
  • ExecInitResultTupleSlotTL
  • MakeSingleTupleTableSlot

関連データ構造

  • IncrementalSortState: 本ノードの実行状態を保持する主要構造体。
  • IncrementalSort: プランノード。
  • IncrementalSortGroupInfo: 実行統計情報。

失敗モードと境界条件

  • EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK が指定された場合、アサーション違反により停止する。

保守担当者向けメモ

  • 本関数は ExprContext を初期化しない（ExecQual や ExecProject を呼び出さないため）。
  • REWIND フラグは子ノードに渡されるが、インクリメンタルソート自体はこれを活用しない。
