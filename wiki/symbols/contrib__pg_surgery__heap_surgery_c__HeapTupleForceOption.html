HeapTupleForceOption

定義

  • 種別: enum
  • 定義箇所: contrib/pg_surgery/heap_surgery.c:32

目的

contrib/pg_surgery モジュールにおいて、破損したヒープタプルに対して実行する強制的な状態変更操作の種類を識別するために使用されます。

シグネチャまたは形状

typedef enum HeapTupleForceOption
{
    HEAP_FORCE_KILL,
    HEAP_FORCE_FREEZE,
} HeapTupleForceOption;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: HEAP_FORCE_KILL（タプルの強制削除用）または HEAP_FORCE_FREEZE（タプルの強制フリーズ用）のいずれかの値。

前提条件と不変条件

  • 本列挙型は heap_force_common 関数等の引数として渡され、実行する処理の分岐条件として機能します。

アルゴリズムまたは振る舞い

  • HEAP_FORCE_KILL: heap_force_common 内で ItemIdSetDead を呼び出し、タプルを無効化する処理を選択します。
  • HEAP_FORCE_FREEZE: heap_force_common 内でタプルヘッダーを直接書き換え、FrozenTransactionId を設定する処理を選択します。

呼び出し元と呼び出し先

  • 呼び出し元: heap_force_common 関数が引数として受け取ります。
  • 呼び出し先: 本シンボル自体は列挙型であるため、直接の呼び出し先はありません。

関連データ構造

  • heap_force_common: 本列挙型を使用して処理を分岐させる共通関数。

失敗モードと境界条件

  • 定義された値以外の値を渡した場合の挙動は、heap_force_common 内のスイッチ文の実装に依存します（ソース抜粋からは詳細不明）。

保守担当者向けメモ

  • 本列挙型に新しい操作を追加する場合は、heap_force_common 内の処理ロジックも併せて更新する必要があります。
