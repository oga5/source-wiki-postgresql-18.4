float4KEY

定義

  • **種別**: struct
  • **定義箇所**: contrib/btree_gist/btree_float4.c:11

目的

PostgreSQLのGiSTインデックスにおいて、float4（単精度浮動小数点数）型の範囲（lower, upper）を保持し、インデックス内での検索や比較を可能にするためのデータ構造です。

シグネチャまたは形状

typedef struct float4key
{
    float4 lower;
    float4 upper;
} float4KEY;

入力と出力

本構造体は、GiSTインデックスのキーとして使用されます。

  • **入力**: インデックス構築時や検索時に、float4 型のデータから変換・格納されます。
  • **出力**: インデックス走査時、lower および upper フィールドが範囲判定や距離計算の基準として参照されます。

前提条件と不変条件

  • リーフノードにおいては、lower == upper となる不変条件を持ちます。

アルゴリズムまたは振る舞い

  • btree_gist モジュールのフレームワーク内で、float4 型の値をGiSTインデックスに適した形式へ圧縮・変換する際に利用されます。
  • gbt_float4_consistent などのサポート関数を通じて、クエリ条件とインデックスノードの範囲を比較する際のデータコンテナとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_float4.c 内の各サポート関数（gbt_float4_compress, gbt_float4_consistent, gbt_float4_dist 等）が、Datum からのキャストやメモリ確保を通じて本構造体を参照・操作します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • gbtree_ninfo（tinfo 変数）: float4KEY を扱うための比較関数や型情報を集約した設定テーブル。
  • GISTENTRY: GiSTインデックスの各エントリを保持する構造体。

失敗モードと境界条件

  • 範囲の整合性: lower > upper となるような不正な状態が生成された場合、インデックス検索結果が正しく得られない可能性があります。

保守担当者向けメモ

  • 本構造体は btree_utils_num.c で定義された汎用的な数値処理ロジックと組み合わせて使用されます。ロジックの変更が必要な場合は、まず btree_utils_num.c の仕様を確認してください。
  • float8KEY と同様の設計思想に基づいています。
