ExprContext_CB

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/execnodes.h:232

目的

ExprContext に関連付けられたコールバック関数を管理するためのリスト構造体です。クエリ実行コンテキストの終了時やリセット時に実行すべきクリーンアップ処理を登録するために使用されます。

シグネチャまたは形状

typedef struct ExprContext_CB
{
    struct ExprContext_CB *next;
    ExprContextCallbackFunction function;
    Datum        arg;
} ExprContext_CB;

入力と出力

  • next: リスト内の次のコールバック要素へのポインタ。
  • function: 実行されるコールバック関数（ExprContextCallbackFunction 型）。
  • arg: コールバック関数に渡される引数（Datum 型）。

前提条件と不変条件

  • ExprContext 構造体の ecxt_callbacks フィールドを起点とする単方向連結リストの一部として機能します。
  • リストの終端は NULL で示されます。

アルゴリズムまたは振る舞い

  • ExprContext が破棄またはリセットされる際、登録されたコールバックリストを順次辿り、各 function を arg を引数として呼び出します。
  • 新しいコールバックは、リストの先頭に追加されるか、既存のリストを走査して管理されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execUtils.c 内の関数群が、この構造体のメモリ確保およびリストへの追加・操作を行っています。
  • 呼び出し先: 登録された function ポインタが指す関数が実行時に呼び出されます。

関連データ構造

  • ExprContext: 本構造体をリストとして保持する親構造体。
  • ExprContextCallbackFunction: function フィールドが指す関数ポインタの型定義。

失敗モードと境界条件

  • メモリ不足時: execUtils.c における構造体の確保に失敗する可能性があります。
  • リスト操作: next ポインタの更新ミスによるリストの断絶や循環参照には注意が必要です。

保守担当者向けメモ

  • 本構造体は実行時専用のデータ構造であり、永続化や copyObject 等のノードサポート関数によるコピーは想定されていません。
  • コールバック関数内でのメモリ管理やリソース解放の順序については、execUtils.c の実装を参照してください。
