ECPGdump_a_struct

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/ecpg/preproc/type.c:557

目的

C言語の構造体型をECPGライブラリが解釈可能な形式で出力（ダンプ）する。構造体の各メンバーを再帰的に処理し、対応するインジケータ変数がある場合はそれも併せて出力する。

シグネチャまたは形状

static void ECPGdump_a_struct(
    FILE *o, 
    const char *name, 
    const char *ind_name, 
    char *arrsize, 
    struct ECPGtype *type, 
    struct ECPGtype *ind_type, 
    const char *prefix, 
    const char *ind_prefix
)

入力と出力

  • **入力**:
  • o: 出力先ファイルストリーム。
  • name, ind_name: 構造体およびインジケータ構造体の名前。
  • arrsize: 配列サイズを示す文字列。
  • type, ind_type: 構造体およびインジケータの型定義構造体。
  • prefix, ind_prefix: メンバー名生成時に付与する接頭辞。
  • **出力**: なし（副作用として o にコードを出力）。

前提条件と不変条件

  • arrsize は数値として評価可能であること。
  • type は構造体型（u.members を持つ）であること。
  • メモリ管理は mm_alloc を使用しており、関数終了時に pbuf および ind_pbuf は解放される。

アルゴリズムまたは振る舞い

1. arrsize に基づき、構造体メンバーへのアクセス演算子（. または ->）を決定し、prefix を更新する。 2. インジケータ変数が指定されている場合、同様にインジケータ構造体用の接頭辞を生成する。 3. type->u.members を走査し、各メンバーに対して ECPGdump_a_type を再帰的に呼び出す。 4. 走査中、インジケータ構造体のメンバー (ind_p) も同時に進め、構造体とインジケータのメンバー数が一致するかを検証する。 5. メンバー数の不一致が検出された場合、mmerror を通じて警告を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/preproc/type.c 内の複数箇所（ECPGdump_a_type 等からの再帰呼び出し）。
  • **呼び出し先**: ECPGdump_a_type（再帰的）、mm_alloc、mmerror、free。

関連データ構造

  • ECPGtype: 型情報を保持する構造体。
  • ECPGstruct_member: 構造体のメンバーリストを保持する構造体。
  • ecpg_no_indicator, struct_no_indicator: インジケータが存在しない場合のデフォルト値。

失敗モードと境界条件

  • **メンバー数不一致**: 構造体のメンバー数とインジケータ構造体のメンバー数が一致しない場合、mmerror で警告が発生する。
  • **メモリ不足**: mm_alloc が失敗した場合、プログラムは終了する（mm_alloc の仕様に依存）。

保守担当者向けメモ

  • ECPGdump_a_type を再帰的に呼び出すため、構造体のネストが深い場合のスタック消費に注意すること。
  • arrsize の評価に atoi を使用しているため、不正な文字列が渡されないよう呼び出し元で保証する必要がある。
