parse_ltree

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_io.c:36

目的

文字列形式の階層パス（ltree）を解析し、PostgreSQL 内部で利用可能なバイナリ構造体へ変換する。

シグネチャまたは形状

ltree *parse_ltree(const char *buf, struct Node *escontext)

入力と出力

  • **入力**:
  • buf: 解析対象の文字列（ltree 形式）。
  • escontext: エラー報告用のコンテキスト（NULL 可）。
  • **出力**:
  • 成功時: 構築された ltree 構造体へのポインタ。
  • 失敗時: NULL（escontext を介してエラーが報告される）。

前提条件と不変条件

  • buf は有効な文字列であること。
  • 階層レベル数が LTREE_MAX_LEVELS を超えないこと。
  • 解析中にメモリ不足や構文エラーが発生した場合、palloc されたメモリは適切に処理されるか、エラーとして終了する。

アルゴリズムまたは振る舞い

1. 文字列をスキャンし、ドット（.）の数を数えてレベル数を特定する。 2. レベル数が上限を超えていないか検証する。 3. nodeitem 配列を確保し、ステートマシン（LTPRS_WAITNAME / LTPRS_WAITDELIM）を用いて文字列を走査する。 4. 各ラベルの妥当性を finish_nodeitem で確認し、一時的なリストに格納する。 5. 最終的な ltree 構造体に必要なメモリサイズを計算し、palloc0 で確保する。 6. 一時リストから構造体へデータをコピーし、SET_VARSIZE でサイズを設定して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/ltree_io.c 内の ltree_in 関数（SQL 入力用）。
  • その他、内部的な文字列変換処理。
  • **呼び出し先**:
  • finish_nodeitem: 各ラベルの検証と確定。
  • palloc / pfree: メモリ管理。
  • pg_mblen_cstr: マルチバイト文字の長さ計算。

関連データ構造

  • ltree: 変換後の内部構造体。
  • nodeitem: 解析中に各ラベルの開始位置と長さを保持する一時構造体。

失敗モードと境界条件

  • **レベル数超過**: LTREE_MAX_LEVELS を超える階層は ERRCODE_PROGRAM_LIMIT_EXCEEDED を発生させる。
  • **構文エラー**: ラベルの形式不正や予期せぬ終端は ERRCODE_SYNTAX_ERROR を発生させる。
  • **メモリ不足**: palloc 失敗時は PostgreSQL の標準的なエラー処理に従う。

保守担当者向けメモ

  • 本関数は palloc を使用しているため、呼び出し元でのメモリ管理に注意すること。
  • ステートマシンは LTPRS_ 系マクロに依存しており、構文拡張時はこの状態遷移を維持する必要がある。
