DecodeUpdate

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/decode.c:952

目的

WAL（Write-Ahead Log）の xl_heap_update レコードを解析し、論理デコーディング用の変更情報として再構築して ReorderBuffer にキューイングする。

シグネチャまたは形状

static void DecodeUpdate(LogicalDecodingContext *ctx, XLogRecordBuffer *buf)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • buf: 解析対象の WAL レコードおよび LSN 情報を含むバッファ。
  • **出力**: なし（副作用として ReorderBuffer への変更追加を行う）。

前提条件と不変条件

  • buf->record は xl_heap_update 型のデータ構造を保持している必要がある。
  • ctx->slot が有効であり、データベースの OID が一致している必要がある。

アルゴリズムまたは振る舞い

1. **フィルタリング**: 対象レコードのデータベース OID が ctx->slot と一致するか確認し、不一致なら終了する。また、FilterByOrigin によりレプリケーションオリジンに基づく除外判定を行う。 2. **変更の割り当て**: ReorderBufferAllocChange を呼び出し、REORDER_BUFFER_CHANGE_UPDATE 型の変更オブジェクトを作成する。 3. **タプルの抽出**:

  • XLH_UPDATE_CONTAINS_NEW_TUPLE フラグがある場合、XLogRecGetBlockData から新しいタプルを抽出し、DecodeXLogTuple でデコードして change->data.tp.newtuple に格納する。
  • XLH_UPDATE_CONTAINS_OLD フラグがある場合、レコードデータから古いタプルを抽出し、同様にデコードして change->data.tp.oldtuple に格納する。

    • 4. **キューイング**: ReorderBufferQueueChange を使用し、トランザクション ID と LSN を紐付けて変更をバッファへ登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/decode.c 内のレコードディスパッチャ（LogicalDecodingProcessRecord からの派生パス）。
  • **呼び出し先**: FilterByOrigin, ReorderBufferAllocChange, XLogRecGetBlockData, DecodeXLogTuple, ReorderBufferAllocTupleBuf, ReorderBufferQueueChange。

関連データ構造

  • LogicalDecodingContext: デコーディングの全体状態。
  • XLogRecordBuffer: WAL レコードのラッパー。
  • ReorderBufferChange: 蓄積される論理変更オブジェクト。
  • xl_heap_update: ヒープ更新レコードの物理構造。

失敗モードと境界条件

  • データベース OID が一致しない場合、処理は早期終了する。
  • レプリケーションオリジンによるフィルタリングに該当する場合、処理は早期終了する。
  • タプルデータがレコード内に存在しない場合、対応する newtuple や oldtuple のフィールドは設定されない。

保守担当者向けメモ

  • XLH_UPDATE_CONTAINS_OLD のデータ抽出時に、SizeOfHeapUpdate を使用してオフセットを計算している点に注意。
  • change->data.tp.clear_toast_afterwards は常に true に設定される。
  • 本関数は物理的な WAL レコードを論理的な変更に変換する役割に特化している。
