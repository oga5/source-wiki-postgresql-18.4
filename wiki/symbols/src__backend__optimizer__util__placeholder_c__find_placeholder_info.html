find_placeholder_info

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/placeholder.c:84

目的

クエリ内の PlaceHolderVar (PHV) に対応する PlaceHolderInfo 構造体を取得、または新規作成して管理リストに登録する。

シグネチャまたは形状

PlaceHolderInfo *find_placeholder_info(PlannerInfo *root, PlaceHolderVar *phv)

入力と出力

  • **入力**:
  • root: クエリの最適化状態を保持する PlannerInfo 構造体。
  • phv: 情報を取得・作成対象となる PlaceHolderVar ノード。
  • **出力**:
  • 対象の PlaceHolderInfo へのポインタ。

前提条件と不変条件

  • phv->phlevelsup == 0 であること（サブクエリを跨ぐPHVは非対応）。
  • root->placeholdersFrozen が真の場合、新規作成は許可されずエラーとなる。
  • phinfo->ph_var->phnullingrels は常に NULL として初期化される。
  • ph_eval_at が空の場合は、構文上の位置（phrels）で評価されるよう強制される。

アルゴリズムまたは振る舞い

1. root->placeholder_array を参照し、既存の PlaceHolderInfo があればそれを返す。 2. 存在しない場合、makeNode(PlaceHolderInfo) で新規作成する。 3. pull_varnos を使用して式の依存関係を特定し、ph_lateral（LATERAL依存）と ph_eval_at（評価可能リレーション）を計算する。 4. ph_width をデータ型情報から推定する。 5. root->placeholder_list および root->placeholder_array に登録する（必要に応じて配列を拡張）。 6. find_placeholders_in_expr を再帰的に呼び出し、式内の下位PHVも登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/costsize.c
  • src/backend/optimizer/path/joinpath.c
  • src/backend/optimizer/plan/createplan.c
  • src/backend/optimizer/plan/initsplan.c

  • src/backend/optimizer/util/relnode.c
  • **呼び出し先**:
  • pull_varnos
  • get_typavgwidth

  • lappend
  • repalloc0_array / palloc0_array

関連データ構造

  • PlaceHolderVar: クエリツリー上のPHV表現。
  • PlaceHolderInfo: 最適化プロセスで利用されるPHVのメタデータ。
  • PlannerInfo: クエリ全体の最適化状態を管理する構造体。

失敗モードと境界条件

  • root->placeholdersFrozen が真の状態で新規PHVを作成しようとした場合、elog(ERROR) により処理が中断される。
  • phid が placeholder_array_size を超える場合、配列の動的拡張が行われる。

保守担当者向けメモ

  • placeholder_array と placeholder_list の両方でポインタを保持しているため、構造体全体を expression_tree_mutator 等で直接操作することは避けること。
  • ph_eval_at と ph_needed のビットマップを確認することで、評価レベルの妥当性をデバッグ可能。
