index_endscan

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/indexam.c:381

目的

インデックススキャンを終了し、関連するリソースを解放する。

シグネチャまたは形状

void index_endscan(IndexScanDesc scan)

入力と出力

  • **入力**: scan (IndexScanDesc) - 終了対象のスキャン記述子。
  • **出力**: なし。

前提条件と不変条件

  • SCAN_CHECKS マクロにより、scan が有効な状態であることが検証される。
  • 対象のインデックスアクセス手法（AM）が amendscan 関数を実装している必要がある。

アルゴリズムまたは振る舞い

1. **テーブルアクセスリソースの解放**: scan->xs_heapfetch が存在する場合、table_index_fetch_end を呼び出してテーブルアクセスに関連するバッファピン等を解放し、ポインタを NULL に設定する。 2. **AM 固有の終了処理**: インデックスリレーションの amendscan メソッドを呼び出し、AM 固有のスキャン終了処理を実行する。 3. **参照カウントの更新**: index_beginscan で取得したインデックスリレーションの参照カウントを RelationDecrementReferenceCount で減らす。 4. **スナップショットの解放**: scan->xs_temp_snap が設定されている場合、UnregisterSnapshot を呼び出してスナップショットを解放する。 5. **構造体の解放**: IndexScanEnd(scan) を呼び出し、スキャン記述子自体を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c
  • src/backend/access/index/genam.c
  • src/backend/executor/execIndexing.c
  • **呼び出し先**:
  • table_index_fetch_end
  • scan->indexRelation->rd_indam->amendscan
  • RelationDecrementReferenceCount
  • UnregisterSnapshot
  • IndexScanEnd

関連データ構造

  • IndexScanDesc: スキャン状態を保持する記述子。
  • Relation: インデックスリレーション。

失敗モードと境界条件

  • CHECK_SCAN_PROCEDURE(amendscan) により、AM が amendscan を実装していない場合はエラーが発生する。
  • 無効な IndexScanDesc を渡した場合、SCAN_CHECKS によりエラーが発生する。

保守担当者向けメモ

  • 本関数はスキャン終了時のクリーンアップを一元管理する。AM 固有の終了処理（amendscan）の後に、共通の参照カウント管理やメモリ解放が行われる点に注意すること。
