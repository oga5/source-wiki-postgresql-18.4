dsa_get_handle

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/dsa.c:497

目的

指定された dsa_area に関連付けられた共有メモリ領域のハンドル（dsa_handle）を取得します。

シグネチャまたは形状

dsa_handle dsa_get_handle(dsa_area *area)

入力と出力

  • **入力**: area - 操作対象の dsa_area 構造体へのポインタ。
  • **出力**: dsa_handle 型の領域ハンドルを返します。

前提条件と不変条件

  • area->control->handle が DSA_HANDLE_INVALID ではないこと（Assert により検証）。
  • 対象の dsa_area は、dsa_create 等で適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. area が保持する制御構造体（area->control）にアクセスします。 2. 制御構造体内に格納されている handle 値を読み取ります。 3. 読み取ったハンドル値を呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/vacuumparallel.c
  • src/backend/replication/logical/launcher.c
  • src/backend/storage/ipc/dsm_registry.c
  • **呼び出し先**:
  • 本関数は内部で他の関数を呼び出しません。

関連データ構造

  • dsa_area: DSA 領域の状態を管理する構造体。
  • dsa_area_control: 共有メモリの先頭に配置される制御構造体。
  • dsa_handle: 共有メモリ領域を一意に識別するためのハンドル型。

失敗モードと境界条件

  • area が NULL の場合、または area->control が不正な場合、セグメンテーション違反が発生します。
  • area->control->handle が DSA_HANDLE_INVALID である場合、Assert が有効なビルドではアボートします。

保守担当者向けメモ

  • 本関数は、領域が dsa_create で作成されたことを前提としています。
  • 取得したハンドルは、他のプロセスが dsa_attach を使用して同じ領域に接続するために使用されます。
