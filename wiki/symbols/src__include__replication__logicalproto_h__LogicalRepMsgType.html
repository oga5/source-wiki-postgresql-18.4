LogicalRepMsgType

定義

  • 種別: enum
  • 定義箇所: src/include/replication/logicalproto.h:57

目的

論理レプリケーションのワイヤプロトコルにおいて、送信される各メッセージの種別を識別するための列挙型です。

シグネチャまたは形状

typedef enum LogicalRepMsgType
{
    LOGICAL_REP_MSG_BEGIN = 'B',
    LOGICAL_REP_MSG_COMMIT = 'C',
    /* ... 他のメッセージ型 ... */
    LOGICAL_REP_MSG_STREAM_PREPARE = 'p',
} LogicalRepMsgType;

各列挙子は、ワイヤプロトコル上で1バイトの文字コードとして表現されます。

入力と出力

  • **入力**: ネットワーク経由で受信したバイトストリームから読み取られた1バイトの値。
  • **出力**: プロトコル処理関数（logicalrep_read_* 等）が次に処理すべきメッセージを特定するための識別子。

前提条件と不変条件

  • 各値は、プロトコル定義に従い、常に1バイトの文字としてシリアライズされる必要があります。
  • 列挙子の値は、パブリッシャーとサブスクライバー間で厳密に一致している必要があります。

アルゴリズムまたは振る舞い

論理レプリケーションのストリームにおいて、メッセージの先頭バイトとして読み取られ、受信側（worker.c）での分岐処理に使用されます。デバッグ時には、この文字コードを追跡することで通信内容を直接把握可能です。

呼び出し元と呼び出し先

  • **呼び出し元（参照）**:
  • src/backend/replication/logical/worker.c: 受信メッセージの識別に使用。
  • src/backend/replication/logical/proto.c: メッセージ書き込み時の型指定や、型名取得に使用。
  • **呼び出し先**:
  • 呼び出し先に関する情報は、提供されたソース抜粋からは特定できません。

関連データ構造

  • StringInfo: メッセージの読み書きに使用されるバッファ構造体。
  • LogicalRepBeginData, LogicalRepCommitData 等: 各メッセージ型に対応するデータ構造。

失敗モードと境界条件

  • 未定義の文字コードが受信された場合、プロトコル解析エラーが発生する可能性があります。
  • 呼び出し元（worker.c）では、無効な値に対して 0 を割り当てるなどの処理が行われる場合があります。

保守担当者向けメモ

  • 新しいメッセージ型を追加する際は、本列挙型への追加に加え、プロトコルバージョン（LOGICALREP_PROTO_VERSION_NUM）の更新を検討してください。
  • 各値はワイヤプロトコルと直結しているため、既存の値の変更は後方互換性を損なう可能性があります。
