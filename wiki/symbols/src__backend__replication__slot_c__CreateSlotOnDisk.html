CreateSlotOnDisk

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slot.c:2257

目的

レプリケーションスロットの状態をディスク上の永続的なディレクトリ構造として作成・保存します。

シグネチャまたは形状

static void CreateSlotOnDisk(ReplicationSlot *slot)

入力と出力

  • **入力**: ReplicationSlot *slot（永続化対象のスロット構造体）
  • **出力**: なし（成功時はディスクへ書き出し、失敗時はエラーをスロー）

前提条件と不変条件

  • 対象のスロットは、この関数が呼び出される時点では他のプロセスから参照されていない必要があります（io_in_progress_lock を取得しないため）。
  • PG_REPLSLOT_DIR 配下にスロット名に対応するディレクトリを作成します。

アルゴリズムまたは振る舞い

1. **一時ディレクトリの準備**: slot->data.name に基づく一時ディレクトリ（.tmp 接尾辞）を作成します。既存の同名一時ディレクトリが存在する場合は削除を試みます。 2. **ディレクトリ作成と同期**: MakePGDirectory で一時ディレクトリを作成し、fsync_fname でディスクへフラッシュします。 3. **状態の書き出し**: slot->dirty を true に設定し、SaveSlotToPath を呼び出してスロットの状態ファイルを一時ディレクトリ内に保存します。 4. **アトミックな配置**: rename を使用して一時ディレクトリを正式なパスへ移動します。 5. **永続化の保証**: クリティカルセクションを開始し、作成したディレクトリと親ディレクトリ（PG_REPLSLOT_DIR）に対して fsync_fname を実行し、クラッシュ後の整合性を確保します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/slot.c 内の ReplicationSlotCreate から呼び出されます。
  • **呼び出し先**: rmtree, MakePGDirectory, fsync_fname, SaveSlotToPath, rename。

関連データ構造

  • ReplicationSlot: スロットのメモリ内表現。
  • PG_REPLSLOT_DIR: スロットデータが格納されるベースディレクトリ。

失敗モードと境界条件

  • 一時ディレクトリの作成やリネームに失敗した場合、ereport(ERROR, ...) により処理が中断されます。
  • fsync 失敗時など、永続化が保証できない状況ではクリティカルセクション内で強制的な再起動（パニックに近い挙動）を誘発する設計となっています。

保守担当者向けメモ

  • 本関数は io_in_progress_lock を取得しません。これは SaveSlotToPath が内部でロックを取得するためであり、デッドロックを回避するための設計です。
  • ディレクトリの作成とリネームによるアトミックな更新は、クラッシュセーフティを維持するための重要なパターンです。
