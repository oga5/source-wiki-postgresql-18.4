y_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgkdtreeproc.c:95

目的

SortedPoint 構造体の配列を y 座標に基づいて昇順にソートするための比較関数です。

シグネチャまたは形状

int y_cmp(const void *a, const void *b)

入力と出力

  • **入力**: 比較対象となる2つの SortedPoint 構造体へのポインタ (a, b)。
  • **出力**:
  • a の y 座標が b より小さい場合は -1
  • 等しい場合は 0
  • a の y 座標が b より大きい場合は 1

前提条件と不変条件

  • 入力ポインタ a および b は SortedPoint 型にキャスト可能であること。
  • SortedPoint 構造体内の p（Point 型）が有効なポインタであり、y メンバにアクセス可能であること。

アルゴリズムまたは振る舞い

1. 引数 a と b を SortedPoint* にキャストします。 2. それぞれの p->y 値を比較します。 3. y 座標の大小関係に基づき、標準的な比較結果（-1, 0, 1）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgkdtreeproc.c: spg_kd_picksplit 関数内で、k-d 木の分割軸を選択する際に qsort の比較関数として渡されます。
  • src/backend/access/spgist/spgquadtreeproc.c: qsort の比較関数として使用されています。
  • **呼び出し先**: なし（標準的な比較演算のみを実行）。

関連データ構造

  • SortedPoint: spgkdtreeproc.c 内で定義される、ポイントデータとインデックス情報を保持する構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、未定義の動作（セグメンテーション違反）を引き起こします。
  • Point 構造体内の y 値が NaN である場合の挙動は、標準的な浮動小数点比較ルールに従います。

保守担当者向けメモ

  • 本関数は qsort 互換のシグネチャを持っていますが、spgquadtreeproc.c 内では引数構成が異なる可能性のある同名の関数が参照されているため、リンク時の競合や意図しない呼び出しに注意してください。
