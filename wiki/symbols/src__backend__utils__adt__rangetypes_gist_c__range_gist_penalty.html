range_gist_penalty

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_gist.c:361

目的

GiSTインデックスにおいて、新しい範囲（newentry）を既存の範囲（origentry）に追加する際に発生するインデックスの拡大コスト（ペナルティ）を算出します。

シグネチャまたは形状

Datum range_gist_penalty(PG_FUNCTION_ARGS)
  • 引数: PG_FUNCTION_ARGS (内部的に GISTENTRY *origentry, GISTENTRY *newentry, float *penalty を取得)
  • 戻り値: Datum (実際には penalty ポインタ経由で float 値を書き戻す)

入力と出力

  • 入力:
  • origentry: 既存のインデックスエントリ。
  • newentry: 追加対象の新しい範囲エントリ。
  • 出力:
  • penalty: 範囲の拡大度合いを示す float 値。値が小さいほど、そのノードへの挿入が好ましいことを意味します。

前提条件と不変条件

  • origentry と newentry の範囲型（OID）が一致している必要があります。一致しない場合はエラーとなります。
  • penalty 値は、同じクラスの範囲内でのみ比較可能です。

アルゴリズムまたは振る舞い

範囲のクラス（空、無限、通常）に基づいてペナルティを計算します。 1. **空範囲の挿入**: 既存範囲の状態に応じて 0.0 から 4 * CONTAIN_EMPTY_PENALTY の範囲でペナルティを割り当てます。 2. **無限範囲の挿入**: (-inf, +inf) の挿入に対し、既存範囲の広さに応じてペナルティを算出します。 3. **片側無限範囲の挿入**: (-inf, x) や (x, +inf) の挿入時、既存範囲の境界を拡張する必要がある場合は subtype_diff 関数（定義されている場合）を用いて差分を計算し、利用できない場合はデフォルトのペナルティを適用します。拡張が不要な場合は 0.0 を返します。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLのGiSTインデックス構築・挿入プロセス（詳細はソースコード外のGiSTコアエンジンに依存）。
  • 呼び出し先: range_get_typcache, range_deserialize, range_cmp_bounds, call_subtype_diff 等の範囲型操作関数。

関連データ構造

  • GISTENTRY: GiSTインデックスのノードエントリ。
  • RangeType: 範囲型の内部表現。
  • TypeCacheEntry: 型のキャッシュ情報（subtype_diff 関数ポインタ等を含む）。

失敗モードと境界条件

  • **型不一致**: orig と new のOIDが異なる場合、elog(ERROR) で停止します。
  • **拡張不可**: 下限または上限が無限でない範囲に対して無限の境界を挿入しようとする場合、get_float4_infinity() を返して非常に大きなペナルティを課します。

保守担当者向けメモ

  • 本関数は subtype_diff の有無によってペナルティの精度が変化します。
  • penalty の計算ロジックはヒューリスティックであり、インデックスの重なりを最小化するために調整されています。変更時はインデックスの検索性能への影響を考慮してください。
