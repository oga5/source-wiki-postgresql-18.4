auto_mem

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/ecpglib/memory.c:65

目的

ECPGライブラリにおいて、動的に確保されたメモリ領域を追跡し、一括解放を可能にするための連結リストノードとして機能します。

シグネチャまたは形状

struct auto_mem
{
    void           *pointer;
    struct auto_mem *next;
};

入力と出力

  • pointer: 追跡対象となる確保済みメモリ領域へのポインタ。
  • next: リスト内の次の auto_mem 構造体へのポインタ。

前提条件と不変条件

  • 各スレッドは、pthread_getspecific を介して独立した auto_mem リストを保持します。
  • リストの各ノードは、ecpg_alloc を使用して確保されることが想定されています。

アルゴリズムまたは振る舞い

  • ecpg_auto_alloc 呼び出し時に新しいノードが作成され、リストの先頭に追加されます。
  • ECPGfree_auto_mem 呼び出し時にリストを走査し、各ノードが保持する pointer を free した後に、ノード自体を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: ecpg_auto_alloc（ノード作成）、ECPGfree_auto_mem（リスト走査）、ecpg_clear_auto_mem（リスト走査）、および get_auto_allocs / set_auto_allocs（リストの取得・設定）。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • auto_mem_key: スレッドローカルなリストの起点（ヘッド）を特定するためのキー。

失敗モードと境界条件

  • メモリ不足時: ecpg_alloc が失敗した場合、リストへの追加処理も失敗します。
  • 空リスト: get_auto_allocs が NULL を返す場合、リスト操作関数は適切に処理を終了します。

保守担当者向けメモ

  • 本構造体は ecpglib のメモリ管理の根幹です。リストの操作（追加・削除）を行う際は、スレッドセーフ性が維持されていることを確認してください。
  • ecpg_clear_auto_mem を使用する場合、リスト構造のみが解放され、pointer が指す実データは解放されない点に注意してください。
