time_dist

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_time.c:140

目的

2つの time 型値の間の絶対的な時間差（間隔）を計算し、interval 型として返します。

シグネチャまたは形状

Datum time_dist(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る2つの time 型の Datum。
  • **出力**: 2つの入力値の差の絶対値を示す interval 型の Datum。

前提条件と不変条件

  • 入力される2つの Datum は、PostgreSQLの time 型として有効である必要があります。
  • time_mi_time 関数が正常に動作し、差分を計算できることが前提です。

アルゴリズムまたは振る舞い

1. DirectFunctionCall2 を使用して time_mi_time を呼び出し、第1引数と第2引数の差分（diff）を計算します。 2. 計算された差分に対して DatumGetIntervalP を適用し、interval 構造体へ変換します。 3. abs_interval を適用して差分の絶対値を取得します。 4. PG_RETURN_INTERVAL_P を使用して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上では PG_FUNCTION_INFO_V1(time_dist) によるマクロ定義が確認できますが、具体的な実行時の呼び出し元はソースコードからは特定できません。
  • **呼び出し先**: time_mi_time（差分計算）、DatumGetIntervalP（型変換）、abs_interval（絶対値計算）。

関連データ構造

  • Datum: PostgreSQLの汎用データ型表現。
  • interval: 時間の長さを表すPostgreSQLの内部型。

失敗モードと境界条件

  • 入力値が time 型として不正な場合、time_mi_time の内部でエラーが発生する可能性があります。
  • 差分計算の結果が interval 型の許容範囲を超える場合、オーバーフロー等のエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は time 型専用の距離計算ロジックです。timetz 型の距離計算については、別途タイムゾーンの正規化を考慮した処理が必要となるため注意してください。
