pg_reg_getnumcolors

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexport.c:173

目的

コンパイル済みの正規表現オブジェクトが内部的に保持するカラーマップ（文字クラスの集合）の総数を取得する。

シグネチャまたは形状

int pg_reg_getnumcolors(const regex_t *regex)

入力と出力

  • **入力**: regex_t *regex（初期化済みの正規表現オブジェクトへのポインタ）
  • **出力**: int（カラーマップに含まれるカラーの総数）

前提条件と不変条件

  • regex は NULL であってはならない。
  • regex->re_magic が REMAGIC と一致している必要がある（アサーションにより検証）。

アルゴリズムまたは振る舞い

1. 引数 regex の re_guts メンバを struct guts * 型にキャストし、内部の colormap 構造体（cmap）にアクセスする。 2. cmap->max（最大カラーID）に 1 を加算した値を返す。これはカラーIDが 0 から始まるため、総数を表す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_trgm/trgm_regexp.c:767
  • **呼び出し先**: なし（直接的な関数呼び出しは行わず、構造体メンバへのアクセスのみを行う）

関連データ構造

  • regex_t: 正規表現オブジェクトの公開構造体。
  • struct guts: 正規表現エンジンの内部状態を保持する構造体。
  • struct colormap: 正規表現内で使用される文字クラス（カラー）を管理する構造体。

失敗モードと境界条件

  • regex が NULL または不正なマジックナンバーを持つ場合、assert によりプログラムが異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は colormap の内部構造に直接依存している。regguts.h で定義される colormap 構造体の変更時には注意が必要である。
