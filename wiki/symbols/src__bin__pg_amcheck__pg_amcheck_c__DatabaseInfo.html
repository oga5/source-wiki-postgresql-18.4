DatabaseInfo

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_amcheck/pg_amcheck.c:150

目的

検証対象となる個々のデータベースに関するメタデータを保持し、pg_amcheck における並列処理やクエリ構築のコンテキストとして利用する。

シグネチャまたは形状

typedef struct DatabaseInfo
{
	char	   *datname;
	char	   *amcheck_schema; /* エスケープおよびクォート済みのリテラル */
	bool		is_checkunique;
} DatabaseInfo;

入力と出力

  • 本構造体は、主に pg_malloc0 を介して動的にメモリ確保され、データベース接続や検証コマンド構築の過程で初期化・参照される。

前提条件と不変条件

  • amcheck_schema フィールドは、SQL クエリに安全に埋め込めるよう、エスケープおよびクォート処理が施されている必要がある。

アルゴリズムまたは振る舞い

  • RelationInfo 構造体と組み合わせて使用され、複数のリレーション間で共有されるデータベースレベルの情報を保持する。
  • データベースリストのコンパイル時や、検証コマンドの準備時に、対象データベースの属性を特定するために参照される。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_amcheck/pg_amcheck.c 内のデータベースリスト構築処理（compile_database_list 等）およびリレーション処理ループ。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行わない。

関連データ構造

  • RelationInfo: チェック対象のリレーション情報を保持する構造体。DatabaseInfo へのポインタを保持し、所属データベース情報を共有する。

失敗モードと境界条件

  • メモリ不足時に pg_malloc0 が失敗した場合、プログラムは終了する。
  • datname や amcheck_schema が適切に設定されていない場合、後続の SQL クエリ構築時に不正なクエリが生成される可能性がある。

保守担当者向けメモ

  • 本構造体は pg_amcheck の並列実行制御において、各スロットがどのデータベースを対象としているかを追跡するために不可欠である。
  • amcheck_schema のエスケープ処理ロジックに変更を加える際は、本構造体のフィールド定義と整合性が取れているか確認すること。
