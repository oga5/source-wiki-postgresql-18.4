AddWaitEventToSet

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/waiteventset.c:569

目的

WaitEventSet に監視対象のイベント（ソケット、ラッチ、ポストマスターの死滅など）を追加し、OS 固有の待機メカニズム（epoll, kqueue, poll 等）へ登録するための準備を行う。

シグネチャまたは形状

int AddWaitEventToSet(WaitEventSet *set, uint32 events, pgsocket fd, Latch *latch, void *user_data)

入力と出力

  • **入力**:
  • set: 監視対象を保持する WaitEventSet 構造体。
  • events: 監視するイベントの種類（WL_SOCKET_READABLE, WL_LATCH_SET 等）。
  • fd: 監視対象のソケット記述子。
  • latch: 監視対象のラッチ（NULL 可）。
  • user_data: イベント発生時に呼び出し元が識別するための任意のポインタ。
  • **出力**:
  • 戻り値: 追加されたイベントのインデックス（int 型）。

前提条件と不変条件

  • set->nevents が set->nevents_space 未満であること（領域不足時は Assert で停止）。
  • ラッチを指定する場合、そのラッチの所有者（owner_pid）が現在のプロセスと一致していること。
  • 1つの WaitEventSet に登録できるラッチは1つまで。
  • ソケット関連のイベントを監視する場合、fd が PGINVALID_SOCKET であってはならない。

アルゴリズムまたは振る舞い

1. **検証**: 入力されたイベントとリソース（ラッチ、ソケット）の整合性をチェックし、不正な組み合わせであれば elog(ERROR) を発行する。 2. **登録**: WaitEventSet 内のイベント配列に新しい WaitEvent を追加し、位置情報を記録する。 3. **特殊処理**:

  • WL_LATCH_SET の場合、内部のパイプやシグナル通知用 FD を設定する。
  • WL_POSTMASTER_DEATH の場合、ポストマスター監視用の FD を設定する。

    • 4. **OS 固有の初期化**: コンパイル時に選択されたバックエンド（epoll, kqueue, poll, win32）に応じて、対応する WaitEventAdjust* 関数を呼び出し、カーネルレベルでの監視登録を行う。 5. **返却**: 追加されたイベントのインデックスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/postgres_fdw.c, src/backend/executor/execAsync.c, src/backend/executor/nodeAppend.c, src/backend/libpq/pqcomm.c, src/backend/postmaster/postmaster.c, src/backend/postmaster/syslogger.c, src/backend/storage/ipc/latch.c など。
  • **呼び出し先**: WaitEventAdjustEpoll, WaitEventAdjustKqueue, WaitEventAdjustPoll, WaitEventAdjustWin32（いずれもプラットフォーム依存）。

関連データ構造

  • WaitEventSet: 待機イベント群を管理するメイン構造体。
  • WaitEvent: 個々のイベントの属性（FD、イベント種別、ユーザーデータ）を保持する構造体。
  • Latch: プロセス間通信や起床通知に使用される構造体。

失敗モードと境界条件

  • **領域不足**: set->nevents が set->nevents_space を超える場合、Assert により異常終了する。
  • **所有権違反**: 他プロセスが所有するラッチを登録しようとした場合、エラーとなる。
  • **重複登録**: 既にラッチが登録されているセットに別のラッチを追加しようとした場合、エラーとなる。
  • **無効な引数**: ソケットなしでソケットイベントを監視しようとした場合、エラーとなる。

保守担当者向けメモ

  • 本関数はプラットフォームごとの差異を吸収する重要な抽象化ポイントです。新しい OS プリミティブを追加する場合は、WaitEventAdjust* 関数の実装と、本関数内の条件分岐を適切に更新してください。
  • WIN32 環境では event->reset フラグの初期化など、特有の処理が含まれています。変更時は Windows 版の挙動に注意してください。
