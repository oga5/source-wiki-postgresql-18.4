SMgrSortArray

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/buffer/bufmgr.c:137

目的

リレーションのソート処理において、RelFileLocator と SMgrRelation をペアで保持し、ソート可能な単位として管理するために使用されます。

シグネチャまたは形状

typedef struct SMgrSortArray
{
	RelFileLocator rlocator;	/* 最初のメンバである必要がある */
	SMgrRelation srel;
} SMgrSortArray;

入力と出力

  • 入力: 本構造体の配列が palloc によりメモリ上に確保され、ソート対象として利用されます。
  • 出力: ソートされた SMgrSortArray 配列が、後続の処理（リレーションの順次処理など）に渡されます。

前提条件と不変条件

  • rlocator メンバが構造体の先頭に配置されている必要があります。これは、ソート関数（rlocator_comparator）が RelFileLocator を基準に比較を行う際に、構造体の先頭アドレスをそのままキャストして利用するためです。

アルゴリズムまたは振る舞い

1. palloc を使用して nrels 分の SMgrSortArray 配列を確保します。 2. qsort を呼び出し、rlocator_comparator を比較関数として配列をソートします。 3. ソートされた配列を順次走査し、各要素の srel を利用してストレージ操作を行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/buffer/bufmgr.c 内の関数（nrels 個のリレーションをソートする必要がある箇所）。
  • 呼び出し先: qsort（標準ライブラリ関数）および rlocator_comparator（比較関数）。

関連データ構造

  • RelFileLocator: リレーションを識別するための識別子。
  • SMgrRelation: ストレージマネージャレベルのリレーション記述子。

失敗モードと境界条件

  • メモリ不足: palloc が失敗した場合、PostgreSQL のエラーハンドリングによりアボートします。
  • nrels が 0 の場合: palloc および qsort の呼び出しは適切に処理される必要がありますが、具体的な挙動は呼び出し側の実装に依存します。

保守担当者向けメモ

  • 本構造体は rlocator が先頭にあるというメモリレイアウトに強く依存しています。メンバの順序を変更する場合は、rlocator_comparator の実装と整合性が取れているか確認してください。
