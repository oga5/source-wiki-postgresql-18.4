PgStat_StatInjEntry

定義

  • 種別: struct
  • 定義箇所: src/test/modules/injection_points/injection_stats.c:26

目的

PostgreSQLのインジェクションポイント（テスト用フック）が実行された回数を追跡・保持するための統計データ構造です。

シグネチャまたは形状

typedef struct PgStat_StatInjEntry
{
    PgStat_Counter numcalls;    /* インジェクションポイントの実行回数 */
} PgStat_StatInjEntry;

入力と出力

  • **入力**: インジェクションポイントの実行時に pgstat_report_inj() を通じて numcalls が加算されます。
  • **出力**: pgstat_fetch_stat_injentry() 等の関数を介して、現在の実行回数が読み取られます。

前提条件と不変条件

  • numcalls は PgStat_Counter 型であり、統計サブシステムによって管理されます。
  • 共有メモリ上の統計データとローカルの pending 領域の間で同期が行われることを前提としています。

アルゴリズムまたは振る舞い

  • 本構造体は、pgstat サブシステムにおけるカスタム統計種別 PGSTAT_KIND_INJECTION のエントリとして使用されます。
  • インジェクションポイント実行時にローカルの pending 領域が更新され、定期的なフラッシュ処理（injection_stats_flush_cb）によって共有メモリ上の統計データに反映されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • injection_stats.c 内の統計管理関数群（pgstat_fetch_stat_injentry 等）が本構造体を型キャストして参照します。
  • pgstat_report_inj() が実行回数の更新時に使用します。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • PgStatShared_InjectionPoint: 共有メモリ上で統計を管理するための対応する共有構造体。

失敗モードと境界条件

  • 統計情報の取得時に該当するインジェクションポイントが存在しない場合、pgstat_fetch_entry は NULL を返します。
  • 共有メモリの制限や統計サブシステムの初期化状態に依存します。

保守担当者向けメモ

  • 本構造体は PGSTAT_KIND_INJECTION 統計種別のテンプレートとして機能します。
  • 統計項目を追加する場合は、pgstat サブシステムの仕様に従い、injection_stats_flush_cb 等のコールバック関数との整合性を維持してください。
