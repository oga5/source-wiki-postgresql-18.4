vbits

定義

  • 種別: struct
  • 定義箇所: contrib/pg_visibility/pg_visibility.c:33

目的

リレーションの可視性マップ（VM）情報をメモリ上に一時的に保持するためのデータ構造です。

シグネチャまたは形状

typedef struct vbits
{
    BlockNumber next;
    BlockNumber count;
    uint8       bits[FLEXIBLE_ARRAY_MEMBER];
} vbits;

入力と出力

  • next: 次に処理すべきブロック番号を保持します。
  • count: 保持しているビット情報の総ブロック数です。
  • bits: 可視性情報を格納する可変長配列です。

前提条件と不変条件

  • bits 配列は FLEXIBLE_ARRAY_MEMBER を使用しており、構造体生成時に必要なブロック数分だけメモリを確保して使用する必要があります。

アルゴリズムまたは振る舞い

  • collect_visibility_data 関数において、リレーション全体のVM状態を収集する際に使用されます。
  • palloc0(offsetof(vbits, bits) + nblocks) のように、構造体のオフセットとブロック数を加算してメモリ確保が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: collect_visibility_data 関数内でインスタンスが生成・操作されます。
  • 呼び出し先: FuncCallContext の user_fctx を通じて、SRF（Set-Returning Function）の各呼び出し間で状態が引き継がれます。

関連データ構造

  • BlockNumber: PostgreSQLのブロック番号型。
  • FuncCallContext: SRFの実行状態を管理する構造体。

失敗モードと境界条件

  • メモリ確保時、nblocks が極端に大きい場合に palloc0 が失敗する可能性があります。
  • 構造体内の next がリレーションの総ブロック数を超えないよう、呼び出し側で制御する必要があります。

保守担当者向けメモ

  • 本構造体は可変長配列を含むため、sizeof(vbits) ではなく offsetof を使用したメモリ確保が必須です。
  • pg_visibility 拡張機能のデータ収集フェーズにおける中心的なコンテナであるため、変更時は collect_visibility_data への影響を考慮してください。
