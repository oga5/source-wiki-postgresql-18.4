AllocateSnapshotBuilder

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/snapbuild.c:184

目的

論理デコーディングにおけるスナップショット構築プロセスを管理するための SnapBuild 構造体を初期化し、専用のメモリコンテキストを割り当てる。

シグネチャまたは形状

SnapBuild *AllocateSnapshotBuilder(ReorderBuffer *reorder,
                                   TransactionId xmin_horizon,
                                   XLogRecPtr start_lsn,
                                   bool need_full_snapshot,
                                   bool in_slot_creation,
                                   XLogRecPtr two_phase_at)

入力と出力

  • **入力**:
  • reorder: 関連付けられる ReorderBuffer。
  • xmin_horizon: スナップショット構築の基準となる最小トランザクションID。
  • start_lsn: デコーディングを開始するWALのLSN。
  • need_full_snapshot: 完全なスナップショットが必要かどうかのフラグ。
  • in_slot_creation: レプリケーションスロット作成中かどうかのフラグ。
  • two_phase_at: 2フェーズコミットの追跡を開始するLSN。
  • **出力**: 初期化された SnapBuild 構造体へのポインタ。

前提条件と不変条件

  • AllocSetContextCreate により、SnapBuild 構造体は専用のメモリコンテキスト内に割り当てられる。
  • 初期状態は SNAPBUILD_START に設定される。
  • コミット済みトランザクションを保持する committed.xip 配列は、初期容量 128 で確保される。

アルゴリズムまたは振る舞い

1. snapshot builder context という名称のメモリコンテキストを作成し、現在のメモリコンテキストを切り替える。 2. palloc0 を使用して SnapBuild 構造体をゼロ初期化し、割り当てる。 3. 引数で渡された各パラメータ（reorder, xmin_horizon, start_decoding_at 等）を構造体のメンバに格納する。 4. committed トランザクションリストの初期化を行い、includes_all_transactions を true に設定する。 5. 元のメモリコンテキストへ戻し、構築された builder を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/logical.c:217 からの呼び出しが確認されている。
  • **呼び出し先**: AllocSetContextCreate, MemoryContextSwitchTo, palloc0 等のメモリ管理関数を呼び出す。

関連データ構造

  • SnapBuild: スナップショット構築のメイン状態構造体。
  • ReorderBuffer: 変更のバッファリングとトランザクション管理を行う構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 または AllocSetContextCreate がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は SnapBuild のライフサイクルの起点となる。
  • committed.xcnt_space は現在 128 に固定されているが、必要に応じて調整可能である。
  • 構造体のメンバの多くは palloc0 によるゼロ初期化に依存しているため、構造体定義の変更時には注意が必要である。
