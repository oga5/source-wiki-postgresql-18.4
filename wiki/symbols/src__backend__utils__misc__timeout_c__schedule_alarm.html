schedule_alarm

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/timeout.c:209

目的

有効なタイムアウトイベントのリストから最も近い期限を特定し、setitimer を用いて SIGALRM をスケジュールする。

シグネチャまたは形状

static void schedule_alarm(TimestampTz now)

入力と出力

  • **入力**: now (現在の時刻を示す TimestampTz)
  • **出力**: なし（副作用として setitimer の設定およびグローバル状態の更新を行う）

前提条件と不変条件

  • active_timeouts 配列が期限順にソートされていること。
  • num_active_timeouts > 0 の場合にのみ実行される。
  • setitimer 呼び出し前に enable_alarm() を呼び出し、シグナルハンドラを有効化する必要がある。

アルゴリズムまたは振る舞い

1. **期限の特定**: active_timeouts[0] から最も近い期限を取得する。 2. **異常検知と補正**:

  • 現在時刻が期限を過ぎている場合、signal_pending を解除し、即時割り込みを強制する。
  • signal_pending が真で、かつ期限から10ms以上経過している場合は、タイマー要求の喪失とみなして状態をリセットする。

    • 3. **タイマー設定**:

  • TimestampDifference で残り時間を計算し、setitimer を呼び出す。
  • 既に適切なタイマーが設定されている場合（signal_pending が真かつ期限が妥当）、重複した setitimer 呼び出しを抑制する。

    • 4. **状態更新**: signal_due_at を更新し、signal_pending を真に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: enable_timeout_after、enable_timeout_every、reschedule_timeouts、remove_timeout_index 等、タイムアウト状態を変更する関数群から呼び出される。
  • **呼び出し先**: enable_alarm、setitimer、TimestampDifference、MemSet、elog。

関連データ構造

  • active_timeouts: 有効なタイムアウトイベントを保持する配列。
  • signal_pending: 割り込みが予約済みかを示すフラグ。
  • signal_due_at: 次の割り込み予定時刻。
  • itimerval: setitimer に渡す時間構造体。

失敗モードと境界条件

  • setitimer が失敗した場合、elog(FATAL) によりプロセスを終了させる。
  • 期限が過去である場合、即時割り込み（secs=0, usecs=1）をスケジュールする。
  • 割り込みハンドラと本関数の間で競合が発生する可能性があるが、signal_pending フラグと enable_alarm の順序制御により安全性が担保されている。

保守担当者向けメモ

  • setitimer はコストが高いため、不要な呼び出しを避けるロジックが含まれている。
  • シグナルハンドラとの競合を避けるため、signal_pending の更新順序（setitimer の前）は厳守すること。
  • 10ms の閾値は現代的なシステムのタイムアウト粒度を考慮した経験的な値である。
