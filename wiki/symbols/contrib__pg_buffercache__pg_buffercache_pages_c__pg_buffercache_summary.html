pg_buffercache_summary

定義

  • 種別: function
  • 定義箇所: contrib/pg_buffercache/pg_buffercache_pages.c:552

目的

PostgreSQL の共有バッファキャッシュ全体の統計情報（使用状況、ダーティ率、参照カウント、平均使用回数）を単一行のタプルとして集計・返却する。

シグネチャまたは形状

Datum pg_buffercache_summary(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL インターフェース経由の呼び出し）。
  • **出力**: Datum（バッファキャッシュの統計情報を含む HeapTuple）。

前提条件と不変条件

  • 呼び出し元の戻り値型が複合型（TYPEFUNC_COMPOSITE）である必要がある。
  • システム全体の一貫したスナップショットは保証されない。バッファ状態は実行中も変化し得る。

アルゴリズムまたは振る舞い

1. NBuffers 分の全バッファヘッダーを順次スキャンする。 2. 各バッファについて pg_atomic_read_u32 を用いて状態を読み取る（ロックは行わない）。 3. BM_VALID フラグに基づき、使用中/未使用を分類する。 4. 使用中のバッファについては、BM_DIRTY フラグの確認および usagecount の合計値を算出する。 5. 参照カウント（refcount）が 0 より大きいバッファを「ピン留め（pinned）」としてカウントする。 6. 全スキャン終了後、集計値をタプルに格納して返却する。使用中バッファが 0 の場合、平均使用回数は NULL となる。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(pg_buffercache_summary) マクロによる登録を経て、SQL インターフェースから呼び出される。
  • **呼び出し先**:
  • get_call_result_type
  • GetBufferDescriptor
  • pg_atomic_read_u32
  • CHECK_FOR_INTERRUPTS
  • heap_form_tuple

関連データ構造

  • BufferDesc: バッファヘッダー構造体。
  • NUM_BUFFERCACHE_SUMMARY_ELEM: 集計結果の要素数。

失敗モードと境界条件

  • 戻り値型が複合型でない場合、elog(ERROR) を発生させる。
  • NBuffers が大きい環境では、スキャン処理によりメモリ消費や CPU 負荷が増大する可能性がある。

保守担当者向けメモ

  • 本関数はパフォーマンスを優先し、バッファヘッダーのロックを行わない。そのため、取得される統計値は調査時点の近似値である。
  • usagecount の平均値計算時にゼロ除算が発生しないよう、buffers_used のチェックを行っている。
