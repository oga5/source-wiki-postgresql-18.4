MDCBufData

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/pgp-decrypt.c:433

目的

OpenPGP の SYMENCRYPTED_DATA_MDC パケット処理において、データ本体と末尾の 22 バイトの MDC（Modification Detection Code）を分離・検証するためのバッファリング構造体です。

シグネチャまたは形状

struct MDCBufData
{
    PGP_Context *ctx;       /* PGP 復号コンテキスト */
    int         eof;        /* EOF 到達フラグ */
    int         buflen;     /* 現在のバッファ長 */
    int         avail;      /* バッファ内の利用可能データ量 */
    uint8      *pos;        /* 現在の読み取り位置 */
    int         mdc_avail;  /* MDC バッファ内の利用可能データ量 */
    uint8       mdc_buf[22];/* MDC 格納用バッファ */
    uint8       buf[MDCBUF_LEN]; /* データ本体用バッファ */
};

入力と出力

  • 入力: PullFilter から読み込まれる暗号化データストリーム。
  • 出力: 復号および MDC 検証済みのデータ。

前提条件と不変条件

  • mdc_buf は常に 22 バイトの固定長であり、パケット末尾の MDC を保持するために使用されます。
  • buf のサイズは MDCBUF_LEN で定義されます。

アルゴリズムまたは振る舞い

1. **先読み**: 復号ストリームからデータを読み込み、MDC パケット（末尾 22 バイト）が混入しないよう buf と mdc_buf に振り分けます。 2. **バッファ管理**: mdcbuf_refill を通じて PullFilter からデータを補充し、pos と avail を更新してストリームの整合性を維持します。 3. **検証**: mdcbuf_finish により、蓄積されたデータと MDC の整合性を検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: mdcbuf_finish, mdcbuf_load_data, mdcbuf_load_mdc, mdcbuf_refill 等の mdcbuf_* 系関数。
  • **呼び出し先**: PullFilter（データ供給元として使用）。

関連データ構造

  • PGP_Context: 復号状態を保持するコンテキスト。
  • PullFilter: データを逐次処理するためのフィルタ構造体。

失敗モードと境界条件

保守担当者向けメモ

  • 本構造体は、MDC パケットの先読みと検証を同時に行うための特殊なフィルタの一部です。
  • mdc_buf の 22 バイトというサイズは RFC 4880 に準拠した固定値です。変更時は仕様を確認してください。
