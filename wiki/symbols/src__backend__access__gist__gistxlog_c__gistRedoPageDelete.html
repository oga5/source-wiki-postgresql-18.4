gistRedoPageDelete

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistxlog.c:341

目的

GiSTインデックスのページ削除操作をWALリプレイ時に再実行し、対象リーフページの削除フラグ設定と親ページからのダウンリンク削除を同期的に適用する。

シグネチャまたは形状

static void gistRedoPageDelete(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）。レコード内には gistxlogPageDelete 構造体が含まれ、削除対象のトランザクションIDおよび親ページ内のダウンリンクオフセットが格納されている。
  • **出力**: なし（副作用としてバッファの状態を更新）。

前提条件と不変条件

  • XLogReadBufferForRedo を使用して、リーフページ（ブロック0）および親ページ（ブロック1）を読み込む必要がある。
  • ページ更新後は MarkBufferDirty を呼び出し、変更を永続化対象とする必要がある。
  • 処理終了時に、読み込んだバッファが有効であれば UnlockReleaseBuffer を呼び出して解放する必要がある。

アルゴリズムまたは振る舞い

1. WALレコードから gistxlogPageDelete データを取得する。 2. **リーフページの処理**: XLogReadBufferForRedo でブロック0を読み込み、BLK_NEEDS_REDO の場合に GistPageSetDeleted を用いて削除済みフラグを立て、LSNを更新してダーティにする。 3. **親ページの処理**: XLogReadBufferForRedo でブロック1を読み込み、BLK_NEEDS_REDO の場合に PageIndexTupleDelete を用いて該当するダウンリンクを削除し、LSNを更新してダーティにする。 4. **クリーンアップ**: 読み込んだバッファ（親、リーフ）が有効な場合、それぞれロックを解除し解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: gist_redo（src/backend/access/gist/gistxlog.c 内のディスパッチャ）。
  • **呼び出し先**: XLogReadBufferForRedo, GistPageSetDeleted, PageSetLSN, MarkBufferDirty, PageIndexTupleDelete, BufferIsValid, UnlockReleaseBuffer。

関連データ構造

  • XLogReaderState: WALレコードの読み取り状態を保持。
  • gistxlogPageDelete: 削除操作に必要なメタデータ（削除XID、ダウンリンクオフセット）を保持する構造体。
  • Buffer: PostgreSQLのバッファ管理用ハンドル。

失敗モードと境界条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返さない場合、そのページに対する更新はスキップされる。
  • バッファが有効でない場合、UnlockReleaseBuffer は呼び出されない。

保守担当者向けメモ

  • 本関数はGiSTのページ削除の不可分性を保証するために、リーフと親の両方の更新を単一のWALレコード内で処理する。
  • ページ削除後のスタンバイ競合解決については、本関数を呼び出す上位の gist_redo ロジックや、関連する gistRedoPageReuse 等の処理との整合性に注意すること。
