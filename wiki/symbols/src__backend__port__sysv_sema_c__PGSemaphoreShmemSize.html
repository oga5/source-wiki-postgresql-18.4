PGSemaphoreShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/port/sysv_sema.c:306

目的

セマフォ管理のために共有メモリ領域へ割り当てるべきバイト数を算出します。

シグネチャまたは形状

Size PGSemaphoreShmemSize(int maxSemas)

入力と出力

  • **入力**: maxSemas (int) - 確保するセマフォの最大数。
  • **出力**: Size - 必要な共有メモリの合計バイト数。

前提条件と不変条件

  • mul_size を使用して計算されるため、オーバーフローが発生した場合は PostgreSQL の標準的なメモリ割り当てエラー処理に従います。

アルゴリズムまたは振る舞い

  • 指定されたセマフォ数 maxSemas に PGSemaphoreData 構造体のサイズを乗算し、必要なメモリサイズを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/ipci.c: セマフォサブシステムの初期化時に呼び出されます。
  • src/backend/port/posix_sema.c: POSIX 版セマフォの実装内でも参照されています。
  • src/backend/port/sysv_sema.c: 同ファイル内の初期化処理で参照されています。
  • src/backend/port/win32_sema.c: Win32 版セマフォの実装内でも参照されています。
  • **呼び出し先**:
  • mul_size() (PostgreSQL ユーティリティ関数)

関連データ構造

  • PGSemaphoreData: 各セマフォの状態を保持する構造体。

失敗モードと境界条件

  • maxSemas が負の値や極端に大きい値である場合、mul_size の内部ロジックにより安全に処理されます。

保守担当者向けメモ

  • 本関数はプラットフォーム非依存のインターフェースとして src/include/storage/pg_sema.h で宣言されています。実装は各ポート（SysV, POSIX, Win32）で共通のロジックを使用していますが、将来的にセマフォのデータ構造がプラットフォームごとに大きく異なる場合は、実装の分離が必要になる可能性があります。
