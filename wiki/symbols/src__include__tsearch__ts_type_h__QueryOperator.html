QueryOperator

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/ts_type.h:194

目的

tsquery 構造体における演算子ノードを表現し、クエリツリー内の論理演算やフレーズ検索の制御情報を保持する。

シグネチャまたは形状

typedef struct {
    QueryItemType type;    /* ノード種別 */
    int8          oper;    /* 演算子識別子 */
    int16         distance;/* OP_PHRASE 用の引数間距離 */
    uint32        left;    /* 左オペランドへのオフセット */
} QueryOperator;

入力と出力

  • **入力**: tsquery の構築時または評価時に、演算子ノードとして初期化される。
  • **出力**: QO_PRIORITY マクロを通じて、演算子の優先順位を算出するために参照される。

前提条件と不変条件

  • tsquery は 4 バイトアライメントを前提とする。
  • left フィールドは、現在のアイテム位置からの相対オフセットとして機能する。
  • 右オペランドは常に item + 1 に配置される。

アルゴリズムまたは振る舞い

  • クエリツリーのトラバーサルにおいて、left フィールドを用いて左側のサブツリーを特定する。
  • OP_PHRASE 演算子の場合、distance フィールドを使用して語彙間の許容距離を制御する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsquery.c および src/backend/utils/adt/tsquery_util.c において、クエリの解析、構築、およびメモリコピー操作で利用される。
  • **呼び出し先**: QO_PRIORITY マクロが OP_PRIORITY を呼び出す。

関連データ構造

  • QueryItem: tsquery 内のノードを表現する共用体。QueryOperator をメンバとして含む。
  • TSQueryData: tsquery の全体構造。
  • QueryItemType: ノードの型定義。

失敗モードと境界条件

  • left オフセットの計算誤りは、クエリツリーの不正な走査を引き起こす。
  • 構造体サイズが固定されているため、memcpy 等での操作時には sizeof(QueryOperator) を使用する必要がある。

保守担当者向けメモ

  • 本構造体は tsquery のバイナリレイアウトの一部であるため、メンバの追加や変更はディスク上のデータ互換性に影響を与える可能性がある。
  • FLEXIBLE_ARRAY_MEMBER を含む構造体と混在するため、メモリ確保時には CALCDATASIZE 等の専用マクロを使用すること。
