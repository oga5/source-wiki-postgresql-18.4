process_startup_packet_die

定義

  • 種別: static function
  • 定義箇所: src/backend/tcop/backend_startup.c:965

目的

バックエンドの初期化フェーズ（スタートアップパケット受信待ち状態）において、SIGTERM シグナルを受信した際にプロセスを即座に終了させるためのシグナルハンドラです。

シグネチャまたは形状

static void process_startup_packet_die(SIGNAL_ARGS);

入力と出力

  • 入力: SIGNAL_ARGS（シグナルハンドラに渡される標準的な引数）
  • 出力: なし（プロセスが終了するため戻り値はありません）

前提条件と不変条件

  • 本関数は、バックエンドの初期化中（BackendInitialize 内など）に呼び出されることを想定しています。
  • 共有メモリへのアクセスが許可されていない初期化段階で使用されるため、proc_exit ではなく _exit(1) を使用して安全に終了します。

アルゴリズムまたは振る舞い

1. シグナルハンドラとして呼び出されます。 2. _exit(1) を実行し、クリーンアップ処理を介さずにプロセスを即座に終了させます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/tcop/backend_startup.c: pqsignal(SIGTERM, process_startup_packet_die); により、SIGTERM シグナルのハンドラとして登録されています。
  • 呼び出し先:
  • _exit(1)（システムコール）

関連データ構造

  • なし

失敗モードと境界条件

  • 本関数自体が失敗することはありません。_exit(1) を呼び出すことで、呼び出し元のプロセスを強制終了させます。

保守担当者向けメモ

  • 本関数は、バックエンドがまだ完全に初期化されていない（共有メモリが利用できない）段階での終了処理を担います。
  • 終了時に proc_exit を使用すると共有メモリのクリーンアップを試みてデッドロックや不正なメモリ操作を引き起こす可能性があるため、必ず _exit(1) を使用してください。
