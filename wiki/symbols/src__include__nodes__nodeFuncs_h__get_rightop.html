get_rightop

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/nodes/nodeFuncs.h:96

目的

OpExpr ノード（演算子式）から、その右側のオペランド（引数）を取得します。

シグネチャまたは形状

Node *get_rightop(const void *clause)

入力と出力

  • **入力**: clause (const void *) - OpExpr 構造体へのポインタ。
  • **出力**: Node * - 演算子の右オペランド。引数が2つ未満の場合は NULL を返します。

前提条件と不変条件

  • 入力 clause は OpExpr 構造体としてキャスト可能である必要があります。
  • OpExpr の args リストが保持する要素のうち、2番目の要素を右オペランドとして返します。

アルゴリズムまたは振る舞い

1. 引数 clause を OpExpr * にキャストします。 2. expr->args リストの長さを確認します。 3. リストの長さが2以上であれば、lsecond() マクロを使用してリストの2番目の要素（右オペランド）を返します。 4. リストの長さが2未満であれば、NULL を返します。

関連データ構造

  • OpExpr: 演算子式を表すノード構造体。args フィールドにオペランドのリストを保持します。
  • Node: PostgreSQLのノードツリーを構成する基本構造体。

失敗モードと境界条件

  • expr->args が空、または要素数が1つのみの場合、右オペランドが存在しないと判断され NULL を返します。
  • 入力ポインタが NULL の場合の挙動は定義されていません（セグメンテーション違反の可能性があります）。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、頻繁に呼び出されるためパフォーマンスが重視されています。
  • OpExpr 以外のノード型が渡された場合の動作は保証されていません。呼び出し側で is_opclause 等を用いて事前に型を確認することが推奨されます。
