donothingCleanup

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/dest.c:60

目的

DestReceiver オブジェクトの終了処理（rShutdown）および破棄処理（rDestroy）において、何も行わないための共通ハンドラとして機能します。

シグネチャまたは形状

static void donothingCleanup(DestReceiver *self);

入力と出力

  • **入力**: DestReceiver *self - 処理対象のレシーバー構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • 本関数は状態を持たない静的なレシーバー（DestNone 等）に対して使用されます。
  • 本関数が呼び出された際、メモリの解放やリソースのクリーンアップは一切行われません。

アルゴリズムまたは振る舞い

  • 関数本体は空であり、呼び出されても何も実行せずに即座に制御を戻します。
  • DestReceiver 構造体の rShutdown および rDestroy メソッドのポインタとして割り当てられ、クリーンアップが不要な出力先において一貫したインターフェースを提供します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/dest.c 内の各レシーバー定義（donothingReceive、debugtup、printsimple、spi_printtup 等）において、rShutdown および rDestroy メソッドとして参照されています。
  • **呼び出し先**: なし。

関連データ構造

  • DestReceiver: 出力先を管理する構造体。本関数はこの構造体のメソッドとして登録されます。

失敗モードと境界条件

  • 本関数自体は副作用を持たないため、失敗モードは存在しません。
  • 静的レシーバーに対してのみ使用されるべきであり、動的に割り当てられたレシーバーに対して本関数を rDestroy として割り当てた場合、メモリリークが発生します。

保守担当者向けメモ

  • 本関数は「何もしない」ことが仕様です。新しい出力先を追加する際、終了処理が不要な場合はこの関数を再利用してください。
  • 呼び出し元情報については、ソースコード内の静的初期化子からの参照が確認されています。
