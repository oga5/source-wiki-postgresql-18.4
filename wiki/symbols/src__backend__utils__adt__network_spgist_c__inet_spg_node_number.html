inet_spg_node_number

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/adt/network_spgist.c:349

目的

SP-GiSTインデックスにおいて、ネットワーク型（inet）の値が4ノード構成のツリー構造のどのノードに属するかをビット演算を用いて特定します。

シグネチャまたは形状

static int inet_spg_node_number(const inet *val, int commonbits)

入力と出力

  • **入力**:
  • val: 評価対象の inet 型データ。
  • commonbits: 既存のプレフィックスと共通するビット数。
  • **出力**:
  • int: 0から3の範囲のノード番号。

前提条件と不変条件

  • val は有効な inet 構造体である必要があります。
  • 4ノード構成のインデックス構造において、ノード0/1はプレフィックス長の一致、ノード2/3はプレフィックス長の長短に基づいた分岐を前提としています。

アルゴリズムまたは振る舞い

1. 初期値 nodeN を 0 に設定します。 2. commonbits が val の最大ビット数未満であり、かつ commonbits 番目のビットが立っている場合、nodeN に 1 を加算（ビット0をセット）します。 3. commonbits が val の実際のビット数（ip_bits(val)）未満である場合、nodeN に 2 を加算（ビット1をセット）します。 4. 最終的な nodeN を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network_spgist.c 内の以下の箇所から呼び出されています。
  • 141行目: inet_spg_choose 内
  • 155行目: inet_spg_choose 内
  • 227行目: inet_spg_picksplit 内
  • **呼び出し先**:
  • ip_maxbits(val)
  • ip_addr(val)
  • ip_bits(val)

    • （これらは utils/inet.h で定義されたマクロまたは関数です）

関連データ構造

  • inet: PostgreSQLのネットワーク型を保持する構造体。

失敗モードと境界条件

  • commonbits が val のビット長を超過する場合の挙動については、ソースコード上のガード条件（commonbits < ip_maxbits 等）により制御されています。

保守担当者向けメモ

  • 本関数は4ノード構成のインデックス専用です。インデックスの分割戦略が変更された場合、このビット判定ロジックも修正が必要です。
  • ビット演算はネットワークバイトオーダーを考慮した ip_addr マクロに依存しているため、ネットワーク型の内部表現に変更があった場合は注意してください。
