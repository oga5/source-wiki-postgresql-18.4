scram_exchange

定義

  • **種別**: function
  • **定義箇所**: src/backend/libpq/auth-scram.c:351

目的

SASL SCRAM-SHA-256認証におけるクライアントとのメッセージ交換を管理し、認証プロトコルの各フェーズ（初期化、チャレンジ送信、検証、完了）を制御するメインループ関数です。

シグネチャまたは形状

int scram_exchange(void *opaq, const char *input, int inputlen,
                   char **output, int *outputlen, const char **logdetail);

入力と出力

  • **入力**:
  • opaq: scram_state 構造体へのポインタ。
  • input: クライアントからのメッセージ文字列。
  • inputlen: 入力メッセージの長さ。
  • **出力**:
  • output: サーバーからクライアントへ送信する応答メッセージ。
  • outputlen: 応答メッセージの長さ。
  • logdetail: 認証失敗時の詳細なログメッセージ（必要時）。
  • **戻り値**: PG_SASL_EXCHANGE_CONTINUE（継続）、PG_SASL_EXCHANGE_SUCCESS（成功）、または PG_SASL_EXCHANGE_FAILURE（失敗）。

前提条件と不変条件

  • opaq は有効な scram_state 構造体であること。
  • input が NULL の場合、認証の初期段階（SCRAM_AUTH_INIT）であること。
  • 認証成功時、MyProcPort に ClientKey および ServerKey が保存されること。

アルゴリズムまたは振る舞い

1. **初期化**: input が NULL の場合、空のチャレンジを返して継続。 2. **メッセージ解析**: inputlen が入力文字列の長さと一致するか検証し、プロトコル違反をチェック。 3. **状態遷移**:

  • SCRAM_AUTH_INIT: クライアントの最初のメッセージを解析し、サーバーのnonceとsaltを含むメッセージを生成して SCRAM_AUTH_SALT_SENT へ移行。
  • SCRAM_AUTH_SALT_SENT: クライアントの最終メッセージを解析し、nonceとProofを検証。

    • 4. **検証**: verify_client_proof を実行。ユーザーが存在しない場合でもモック認証として計算を行い、タイミング攻撃を防止する。 5. **完了**: 成功時に SCRAM_AUTH_FINISHED 状態へ移行し、セッション鍵を MyProcPort に格納。

呼び出し元と呼び出し先

  • **呼び出し元**: 認証サブシステム（src/backend/libpq/auth.c 等の認証フロー制御部）。
  • **呼び出し先**: read_client_first_message, build_server_first_message, read_client_final_message, verify_final_nonce, verify_client_proof, build_server_final_message。

関連データ構造

  • scram_state: 認証セッションの状態を保持する構造体。
  • MyProcPort: 認証成功後にセッション鍵が格納されるプロセスポート構造体。

失敗モードと境界条件

  • **プロトコル違反**: メッセージが空、または長さが不一致の場合に ERRCODE_PROTOCOL_VIOLATION でエラー終了。
  • **検証失敗**: nonceの不一致や、verify_client_proof による証明の不一致が発生した場合、PG_SASL_EXCHANGE_FAILURE を返す。
  • **モック認証**: ユーザーが存在しない場合、doomed フラグにより意図的に失敗させる。

保守担当者向けメモ

  • 本関数は scram_state の状態遷移を厳密に管理しています。新しい状態を追加する際は switch 文の網羅性に注意してください。
  • 認証失敗時のログ出力は、ユーザーの存在を特定させないよう設計されています。エラーメッセージの変更には細心の注意を払ってください。
