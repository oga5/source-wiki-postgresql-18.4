LogicalDecodingProcessRecord

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/decode.c:87

目的

WAL（Write-Ahead Log）レコードを解析し、論理デコーディングパイプラインにおける適切なリソースマネージャ（Rmgr）のデコード関数へディスパッチする。

シグネチャまたは形状

void LogicalDecodingProcessRecord(LogicalDecodingContext *ctx, XLogReaderState *record)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト（LogicalDecodingContext）。
  • record: 読み込まれた WAL レコードの状態（XLogReaderState）。
  • **出力**: なし（副作用として reorderbuffer への変更蓄積やスナップショット更新を行う）。

前提条件と不変条件

  • ctx および record は有効なポインタであること。
  • record は xlogreader によって既に読み込まれていること。

アルゴリズムまたは振る舞い

1. **バッファ初期化**: XLogRecordBuffer を作成し、レコードの LSN 範囲（origptr, endptr）とレコード本体を格納する。 2. **サブトランザクション登録**: トップレベルのトランザクション ID が有効な場合、ReorderBufferAssignChild を呼び出し、現在のレコードをサブトランザクションとしてトップレベルトランザクションに紐付ける。 3. **デコード処理のディスパッチ**:

  • GetRmgr を使用してレコードタイプに対応するリソースマネージャ（RmgrData）を取得する。
  • rm_decode 関数が定義されている場合、それを実行してレコードを解析する。
  • rm_decode が定義されていない場合、ReorderBufferProcessXid を呼び出し、トランザクション ID の処理のみを行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/logical.c（複数箇所）
  • **呼び出し先**:
  • ReorderBufferAssignChild
  • GetRmgr
  • rmgr.rm_decode（動的ディスパッチ）
  • ReorderBufferProcessXid

関連データ構造

  • LogicalDecodingContext: 論理デコーディングの全体状態を保持。
  • XLogReaderState: WAL レコードの読み込み状態。
  • XLogRecordBuffer: デコード処理に渡されるレコードのラッパー構造体。
  • RmgrData: リソースマネージャの定義とデコード関数ポインタ。

失敗モードと境界条件

  • rm_decode が定義されていない場合、トランザクション ID の処理のみが行われ、レコードの内容自体はスキップされる。
  • 無効なトランザクション ID の場合、サブトランザクションの割り当て処理はスキップされる。

保守担当者向けメモ

  • 本関数は論理デコーディングの主要なエントリポイントである。新しいレコードタイプを追加する際は、対応するリソースマネージャの rm_decode 関数が適切に実装されているか確認すること。
  • XLogRecGetTopXid や XLogRecGetXid を介してトランザクション情報を取得しているため、WAL レコードの構造変更時にはこれらのアクセサの整合性に注意が必要である。
