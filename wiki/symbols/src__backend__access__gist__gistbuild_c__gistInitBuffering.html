gistInitBuffering

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistbuild.c:625

目的

GiSTインデックス構築において、メモリ（キャッシュ）容量を超過するデータセットを効率的に処理するため、バッファリング構築モードへ移行するための初期化を行う。

シグネチャまたは形状

static void gistInitBuffering(GISTBuildState *buildstate)

入力と出力

  • **入力**: buildstate (GISTBuildState *) - インデックス構築の現在の状態を保持する構造体。
  • **出力**: なし（副作用として buildstate 内のバッファ管理構造体および構築モードを更新する）。

前提条件と不変条件

  • buildstate が有効なインデックス関係および統計情報を含んでいること。
  • effective_cache_size および maintenance_work_mem が設定されていること。

アルゴリズムまたは振る舞い

1. **統計計算**: 既存の挿入済みタプルから平均サイズを算出し、インデックスページあたりの平均および最大タプル数を推定する。 2. **パラメータ決定**: levelStep（サブツリーの深さ）を、サブツリーサイズが effective_cache_size の1/4以下、かつメモリ消費が maintenance_work_mem を超えない範囲で最大化する。 3. **モード移行**:

  • levelStep が1未満の場合はバッファリングを断念し、GIST_BUFFERING_DISABLED に設定する。
  • 条件を満たす場合、calculatePagesPerBuffer で pagesPerBuffer を算出し、gistInitBuildBuffers でバッファを初期化する。
  • gistInitParentMap を呼び出し、構築モードを GIST_BUFFERING_ACTIVE に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistbuild.c 内のインデックス構築プロセス（gistBuildCallback 等から間接的に呼び出される）。
  • **呼び出し先**:
  • calculatePagesPerBuffer
  • gistInitBuildBuffers
  • gistInitParentMap
  • gistGetMaxLevel

関連データ構造

  • GISTBuildState: 構築状態を管理するメイン構造体。
  • GISTBuildBuffers: バッファリング構築で使用されるバッファ管理構造体。

失敗モードと境界条件

  • levelStep <= 0: キャッシュやメモリ制限によりバッファリング構築が不可能な場合、GIST_BUFFERING_DISABLED にフォールバックし、デバッグログを出力して終了する。

保守担当者向けメモ

  • 本関数は、Argeらの論文に基づくアルゴリズムを採用している。levelStep の計算において、effective_cache_size に対する安全係数（4）が適用されている点に注意。
  • itupMinSize の計算において、可変長属性の最小サイズに関する XXX コメントがあるため、将来的な最適化の余地がある。
