FDWCollateState

定義

  • 種別: enum
  • 定義箇所: contrib/postgres_fdw/deparse.c:78

目的

リモートサーバーへ送信する式における、照合順序（Collation）の安全性を分類するために使用されます。ローカルの式がリモートで実行可能か判断する際、照合順序の整合性を追跡する目的で定義されています。

シグネチャまたは形状

typedef enum {
    FDW_COLLATE_NONE,
    FDW_COLLATE_SAFE,
    FDW_COLLATE_UNSAFE,
} FDWCollateState;

入力と出力

本シンボルは列挙型であり、関数ではありません。状態遷移の判定結果として、deparse_expr_cxt 構造体などの内部で保持されます。

前提条件と不変条件

  • FDW_COLLATE_NONE: 式が照合順序を持たない型であるか、デフォルトの照合順序が外部テーブルの列（foreign Var）に由来しない場合に適用されます。
  • FDW_COLLATE_SAFE: 照合順序が外部テーブルの列に由来しており、リモート実行において安全とみなされる場合に適用されます。
  • FDW_COLLATE_UNSAFE: 照合順序がデフォルトではなく、かつ外部テーブルの列以外に由来する場合に適用されます。

アルゴリズムまたは振る舞い

式ツリーの走査（foreign_expr_walker 等）において、各ノードの照合順序を評価する際のステータスとして利用されます。この状態に基づき、リモートサーバーでの実行可否が決定されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/postgres_fdw/deparse.c 内の式走査ロジック。
  • 呼び出し先: 本シンボルは列挙型であり、関数ではないため呼び出し先はありません。

関連データ構造

  • deparse_expr_cxt: SQL 構築中の状態を保持する構造体であり、この中で FDWCollateState 型のメンバが使用されています。

失敗モードと境界条件

  • 照合順序が不明または不整合な式を FDW_COLLATE_UNSAFE と判定することで、リモート実行を回避し、ローカルでの処理へフォールバックさせる境界条件として機能します。

保守担当者向けメモ

  • 本列挙型は、リモートサーバーとローカルサーバー間での照合順序の不一致を防ぐための重要な判定基準です。新しい式ノードをサポートする際は、そのノードが照合順序に与える影響を考慮し、適切にこの状態を更新する必要があります。
