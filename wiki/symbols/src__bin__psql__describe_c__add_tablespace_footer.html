add_tablespace_footer

定義

  • 種別: function
  • 定義箇所: src/bin/psql/describe.c:3646

目的

データベースオブジェクト（テーブル、インデックス等）の表示出力に対し、関連付けられたテーブルスペース情報をフッターとして追加する。

シグネチャまたは形状

static void add_tablespace_footer(printTableContent *const cont, char relkind, Oid tablespace, const bool newline)

入力と出力

  • cont: 出力対象の printTableContent 構造体へのポインタ。
  • relkind: オブジェクトの種類を示す文字コード。
  • tablespace: テーブルスペースの OID。
  • newline: true の場合、新しいフッター行として追加。false の場合、既存のフッター行の末尾に追記する。
  • 出力: なし（副作用として cont を更新）。

前提条件と不変条件

  • relkind が RELKIND_RELATION, RELKIND_MATVIEW, RELKIND_INDEX, RELKIND_PARTITIONED_TABLE, RELKIND_PARTITIONED_INDEX, RELKIND_TOASTVALUE のいずれかである場合にのみ処理を実行する。
  • tablespace が 0（デフォルトテーブルスペース）の場合は何も行わない。

アルゴリズムまたは振る舞い

1. 対象の relkind であるかを確認する。 2. tablespace が 0 でない場合、pg_catalog.pg_tablespace から spcname を取得するクエリを実行する。 3. クエリ結果が存在する場合：

  • newline が true の場合、Tablespace: "..." という形式で新しいフッターを追加する。
  • newline が false の場合、既存のフッター文字列を取得し、, tablespace "..." という形式で末尾に追記してフッターを更新する。

    • 4. 使用したバッファと結果セットを解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/describe.c 内の各所（describeOneTableDetails 等のテーブル詳細表示ロジック）。
  • 呼び出し先: PSQLexec（クエリ実行）、printTableAddFooter / printTableSetFooter（出力整形）、initPQExpBuffer / printfPQExpBuffer / appendPQExpBuffer / termPQExpBuffer（文字列バッファ操作）。

関連データ構造

  • printTableContent: 出力テーブルの構造を保持する構造体。
  • PQExpBufferData: SQL クエリおよびフッター文字列構築用のバッファ。
  • PGresult: クエリ実行結果を保持する構造体。

失敗モードと境界条件

  • PSQLexec が失敗（result が NULL）した場合、処理を中断して終了する。
  • tablespace が 0 の場合、何も出力せず終了する。
  • クエリ結果が空の場合、フッターの追加は行われない。

保守担当者向けメモ

  • 本関数は psql の \d コマンド系で共通利用される。
  • relkind の判定条件を変更する場合は、サポート対象のオブジェクト種別が増減する点に注意すること。
  • newline が false の場合、既存のフッター内容を cont->footer->data から読み取って再構築するため、フッターの順序や形式に依存している。
