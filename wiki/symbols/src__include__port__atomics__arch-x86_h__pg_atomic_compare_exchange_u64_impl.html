pg_atomic_compare_exchange_u64_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/arch-x86.h:198

目的

x86アーキテクチャにおいて、64ビット整数値のアトミックな比較および交換（Compare-and-Swap）を実行する。

シグネチャまたは形状

static inline bool pg_atomic_compare_exchange_u64_impl(volatile pg_atomic_uint64 *ptr, uint64 *expected, uint64 newval)

入力と出力

  • ptr: 操作対象となる pg_atomic_uint64 型のメモリ領域へのポインタ。
  • expected: 比較対象の値。成功時は元の値が保持され、失敗時は現在の値で更新される。
  • newval: 交換後の新しい値。
  • 戻り値: 比較が一致し、交換が成功した場合は true、失敗した場合は false を返す。

前提条件と不変条件

  • expected ポインタは8バイト境界にアライメントされている必要がある（AssertPointerAlignmentによる検証）。
  • ptr は volatile 修飾されており、コンパイラによる最適化でメモリ読み書きが省略されないことが保証される。

アルゴリズムまたは振る舞い

1. expected が指す値と ptr->value を比較する。 2. インラインアセンブラの lock cmpxchgq 命令を使用し、アトミックに比較と交換を行う。 3. cmpxchgq が設定するゼロフラグ（ZF）を setz 命令で読み取り、成功/失敗の結果を ret に格納する。 4. expected の値は、アセンブラの出力制約により、操作前の ptr->value で更新される。 5. 最後に ret を bool 型にキャストして返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/port/atomics/generic.h 等の汎用アトミックインターフェースから呼び出される。
  • 呼び出し先: ハードウェア命令（lock cmpxchgq）を直接実行するため、関数呼び出し先は存在しない。

関連データ構造

  • pg_atomic_uint64: 64ビットアトミック操作用の構造体。

失敗モードと境界条件

  • expected が指す値が ptr->value と一致しない場合、交換は行われず、expected に現在の ptr->value が書き込まれ、戻り値は false となる。
  • 32ビット環境では本関数は提供されない（__x86_64__ 環境専用）。

保守担当者向けメモ

  • 本実装はx86のTSO（Total Store Order）モデルに依存している。
  • memory クローバーにより、コンパイラに対してメモリバリアとしての振る舞いを明示している。
  • 64ビットアトミック操作は、32ビット環境ではインラインアセンブラ実装が複雑になるため、本ファイルでは64ビット環境でのみ有効化される設計となっている。
