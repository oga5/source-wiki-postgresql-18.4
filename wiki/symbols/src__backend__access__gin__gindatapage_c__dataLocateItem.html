dataLocateItem

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/gindatapage.c:251

目的

GINポスティングツリーの内部ノードにおいて、指定されたアイテムポインタを保持する子ページ（BlockNumber）を二分探索により特定します。

シグネチャまたは形状

BlockNumber dataLocateItem(GinBtree btree, GinBtreeStack *stack)

入力と出力

  • **入力**:
  • btree: GIN B-treeの制御構造体。検索対象の itemptr を保持。
  • stack: 現在のページバッファと、探索結果のオフセットを格納するスタック構造体。
  • **出力**:
  • 探索対象のアイテムポインタを含む子ページの BlockNumber を返します。

前提条件と不変条件

  • stack->buffer が指すページは、リーフページであってはなりません（!GinPageIsLeaf(page)）。
  • ページはGINデータページである必要があります（GinPageIsData(page)）。
  • ページ内のオフセット範囲は FirstOffsetNumber 以上 maxoff 以下である必要があります。

アルゴリズムまたは振る舞い

1. **フルスキャン判定**: btree->fullScan が真の場合、ページ内の最初のオフセットをスタックに設定し、getLeftMostChild を呼び出して左端の子ページを返します。 2. **二分探索**: ページ内のアイテムに対して ginCompareItemPointers を用いて比較を行います。

  • 一致する場合、そのオフセットをスタックに記録し、対応する子ページのブロック番号を返します。
  • 比較結果に基づき、探索範囲（low, high）を狭めていきます。

    • 3. **終了処理**: 探索終了後、最終的なオフセットをスタックに格納し、その位置にある PostingItem から子ページのブロック番号を取得して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gindatapage.c 内で btree->findChildPage に関数ポインタとして代入されています。
  • **呼び出し先**:
  • GinPageGetOpaque: ページヘッダー情報の取得。
  • GinDataPageGetPostingItem: ページ内のアイテム取得。
  • ginCompareItemPointers: アイテムポインタの比較。
  • PostingItemGetBlockNumber: アイテムからブロック番号の抽出。
  • btree->getLeftMostChild: フルスキャン時の左端子ページ取得。

関連データ構造

  • GinBtree: GINインデックスのB-tree操作状態を保持。
  • GinBtreeStack: 探索経路上のページバッファとオフセットを管理。
  • PostingItem: ポスティングツリーのノードに格納されるキーとブロック番号のペア。

失敗モードと境界条件

  • high < low となるような不正なページ状態ではアサーションエラーが発生します。
  • ページ内の最大オフセット（maxoff）に達した場合、右側の無限大として処理されます。

保守担当者向けメモ

  • 本関数は ginbtree.c が提供する汎用B-treeインターフェースの一部として機能します。
  • dataIsMoveRight によるページ移動判定が前提となっているため、ページ分割中の並行アクセスには注意が必要です。
