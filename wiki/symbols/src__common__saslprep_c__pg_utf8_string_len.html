pg_utf8_string_len

定義

  • 種別: function
  • 定義箇所: src/common/saslprep.c:1002

目的

UTF-8エンコードされた文字列に含まれる文字数（コードポイント数）をカウントし、同時に文字列が有効なUTF-8形式であるかを検証します。

シグネチャまたは形状

static int pg_utf8_string_len(const char *source)

入力と出力

  • **入力**: source (ヌル終端されたUTF-8文字列へのポインタ)
  • **出力**:
  • 成功時: 文字列内の文字数（int）
  • 失敗時: -1（不正なUTF-8シーケンスが検出された場合）

前提条件と不変条件

  • 入力 source は有効なヌル終端文字列である必要があります。
  • 内部で strlen(source) を使用するため、文字列全体がメモリ上に存在する必要があります。

アルゴリズムまたは振る舞い

1. strlen で文字列のバイト長を取得します。 2. pg_utf_mblen を使用して、現在のポインタ位置にあるマルチバイト文字のバイト長 l を特定します。 3. 残りのバイト数 len が l より小さい場合、または pg_utf8_islegal がそのシーケンスを不正と判断した場合は -1 を返します。 4. ポインタを l 進め、len を減算し、文字数カウンタをインクリメントします。 5. 文字列の終端に達するまで上記を繰り返し、最終的な文字数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/saslprep.c 内の pg_saslprep 関数から呼び出されます。
  • **呼び出し先**:
  • strlen (標準ライブラリ)
  • pg_utf_mblen (マルチバイト文字長取得)
  • pg_utf8_islegal (UTF-8妥当性検証)

関連データ構造

  • 特になし（標準的なC文字列および pg_wchar 関連のユーティリティを使用）。

失敗モードと境界条件

  • 入力文字列に不正なUTF-8バイトシーケンスが含まれている場合、-1 を返します。

保守担当者向けメモ

  • 本関数は static であり、src/common/saslprep.c 内でのみ使用されます。
  • pg_utf_mblen および pg_utf8_islegal の実装に依存しているため、これらの関数がサポートするエンコーディング仕様の変更には注意が必要です。
