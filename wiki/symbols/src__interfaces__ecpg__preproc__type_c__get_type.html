get_type

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/type.c:109

目的

ECPGプリプロセッサにおいて、内部的な型列挙値 (enum ECPGttype) を、対応するC言語の文字列リテラル（型名）に変換します。

シグネチャまたは形状

static const char *get_type(enum ECPGttype type);

入力と出力

  • **入力**: enum ECPGttype 型の列挙値。
  • **出力**: 対応する型名を表す const char * 型の文字列リテラル。

前提条件と不変条件

  • 入力された type が ECPGttype で定義された有効な値である必要があります。
  • 戻り値は静的な文字列リテラルを指すため、呼び出し側で解放する必要はありません。

アルゴリズムまたは振る舞い

1. switch 文を用いて、入力された type を評価します。 2. 各 ECPGt_* 列挙値に対し、対応する文字列（例: ECPGt_int に対して "ECPGt_int"）を返します。 3. 定義されていない型が渡された場合、mmerror を呼び出し、PARSE_ERROR としてエラー処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/preproc/type.c 内の ECPGdump_a_simple 関数（546行目および548行目）から呼び出され、型情報をCソースコードとして出力するために使用されています。
  • **呼び出し先**: mmerror（エラー発生時）。

関連データ構造

  • enum ECPGttype: 型の識別に使用される列挙型。

失敗モードと境界条件

  • switch 文の default ケースに到達した場合、mmerror によって処理が中断されます。これは未知の型コードが渡された場合に発生します。

保守担当者向けメモ

  • 新しいデータ型を ECPGttype に追加した場合は、必ずこの関数の switch 文にもケースを追加してください。
  • 戻り値の文字列は、生成されるCコードの型記述子として直接使用されるため、文字列の綴りを変更する際はECPGライブラリ側の期待値と整合しているか確認が必要です。
