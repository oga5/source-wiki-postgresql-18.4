single_color_transition

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_nfa.c:1524

目的

2つのNFA状態（s1 と s2）の間が、単一の PLAIN 型の遷移（アーク）のみで接続されているかを判定し、条件を満たす場合にその遷移元状態を返します。

シグネチャまたは形状

static struct state *single_color_transition(struct state *s1, struct state *s2)

入力と出力

  • **入力**:
  • s1: 遷移元のNFA状態。
  • s2: 遷移先のNFA状態。
  • **出力**:
  • 条件を満たす場合: s1（または EMPTY 遷移をスキップした後の実質的な遷移元状態）。
  • 条件を満たさない場合: NULL。

前提条件と不変条件

  • s1 および s2 は有効なNFA状態構造体である必要があります。
  • 遷移の判定において、先頭または末尾に存在する EMPTY 型の遷移は無視されます。

アルゴリズムまたは振る舞い

1. **EMPTY遷移のスキップ**: s1 から出る唯一の遷移が EMPTY であれば、その遷移先を新たな s1 とします。同様に、s2 に入る唯一の遷移が EMPTY であれば、その遷移元を新たな s2 とします。 2. **自己ループの除外**: 処理後の s1 と s2 が同一であれば NULL を返します。 3. **遷移の検証**:

  • s1 から出る遷移が存在しない場合は NULL を返します。
  • s1 から出るすべての遷移を走査し、それらすべてが PLAIN 型であり、かつ遷移先が s2 であることを確認します。

    • 4. **結果の返却**: すべての条件を満たした場合、s1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regcomp.c:2044
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません（構造体メンバへのアクセスのみ）。

関連データ構造

  • struct state: NFAの状態を表現する構造体。
  • struct arc: 状態間の遷移を表現する構造体。type（EMPTY, PLAIN 等）および to（遷移先）を保持します。

失敗モードと境界条件

  • s1 から出る遷移が PLAIN 以外（制約など）である場合、または s2 以外へ向かう遷移が存在する場合は NULL を返します。
  • s1 から出る遷移が一つも存在しない場合は NULL を返します。
  • 処理の結果、s1 と s2 が同一状態になる場合は NULL を返します。

保守担当者向けメモ

  • 本関数は、NFAのグラフ構造が単純な「単一色」の遷移で構成されているかを検証するユーティリティです。
  • EMPTY 遷移のスキップ処理は、グラフの正規化状態に依存するため、最適化パイプラインのどの段階で呼び出されるかを考慮してください。
