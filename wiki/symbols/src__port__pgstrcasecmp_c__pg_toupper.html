pg_toupper

定義

  • 種別: function
  • 定義箇所: src/port/pgstrcasecmp.c:104

目的

ロケールを考慮しつつ、プラットフォーム間で一貫した大文字変換を行う。

シグネチャまたは形状

unsigned char pg_toupper(unsigned char ch);

入力と出力

  • **入力**: unsigned char ch （変換対象の文字）
  • **出力**: unsigned char （変換後の文字。変換不要な場合は入力値をそのまま返す）

前提条件と不変条件

  • 7ビットASCII範囲（'a'-'z'）の文字は、ロケール設定に関わらず常に大文字へ変換される。
  • ASCII範囲外の文字については、現在の環境のロケール設定に依存する。

アルゴリズムまたは振る舞い

1. 入力文字 ch が 'a' から 'z' の範囲内である場合、固定のオフセット計算により大文字へ変換する。 2. それ以外の場合、IS_HIGHBIT_SET(ch) が真かつ islower(ch) が真であれば、標準ライブラリの toupper(ch) を呼び出して変換する。 3. 上記いずれにも該当しない場合、入力値をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogfuncs.c
  • src/backend/utils/adt/cash.c
  • src/backend/utils/adt/datetime.c
  • src/backend/utils/adt/pg_locale_libc.c
  • src/bin/psql/tab-complete.in.c
  • src/timezone/pgtz.c
  • **呼び出し先**:
  • islower (標準Cライブラリ)
  • toupper (標準Cライブラリ)
  • IS_HIGHBIT_SET (マクロ)

関連データ構造

  • なし

失敗モードと境界条件

  • マルチバイト文字セットに対しては不完全な実装であるため、マルチバイト文字を渡した場合の挙動は保証されない。

保守担当者向けメモ

  • 本関数はトルコ語ロケール等の特殊なケースでの不整合を避けるため、ASCII範囲外の文字に対してのみロケール関数を呼び出す設計となっている。
  • ロケールに左右されない厳密なASCII変換が必要な場合は、pg_ascii_toupper の使用を検討すること。
