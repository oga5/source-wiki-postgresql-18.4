hash_xlog_split_cleanup

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash_xlog.c:938

目的

ハッシュインデックスの分割処理完了後、対象バケットページに残存する「分割クリーンアップが必要」というフラグを解除し、インデックスの整合性を復元する。

シグネチャまたは形状

static void hash_xlog_split_cleanup(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record - リプレイ対象のWALレコード情報。
  • **出力**: なし。副作用としてバッファの更新と解放を行う。

前提条件と不変条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返す場合のみ、ページ内容の更新が行われる。
  • 処理対象のバッファは、関数終了時に必ず解放される。

アルゴリズムまたは振る舞い

1. XLogReadBufferForRedo を使用して、WALレコードに関連付けられたバッファを取得する。 2. 取得したバッファがリプレイを必要とする場合:

  • ページ内の HashPageOpaque 構造体を取得する。
  • hasho_flag から LH_BUCKET_NEEDS_SPLIT_CLEANUP フラグをビット演算で除去する。
  • ページのLSNをWALレコードの終了位置（EndRecPtr）に更新する。
  • MarkBufferDirty を呼び出し、変更をバッファに反映させる。

    • 3. バッファが有効な場合、UnlockReleaseBuffer を呼び出してロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: hash_redo 関数（src/backend/access/hash/hash_xlog.c:1104）から、特定のWALレコード種別に対するリプレイ処理として呼び出される。
  • **呼び出し先**:
  • XLogReadBufferForRedo
  • BufferGetPage
  • HashPageGetOpaque
  • PageSetLSN
  • MarkBufferDirty
  • BufferIsValid
  • UnlockReleaseBuffer

関連データ構造

  • XLogReaderState: WALレコードの読み取り状態を保持する構造体。
  • HashPageOpaque: ハッシュインデックスページのメタデータ領域。hasho_flag を通じてページの状態を管理する。

失敗モードと境界条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO 以外を返した場合、ページ更新はスキップされるが、バッファが有効であればロック解放は行われる。

保守担当者向けメモ

  • 本関数は冪等である。既にフラグが解除されているページに対して適用されても、論理的な不整合は発生しない。
  • LH_BUCKET_NEEDS_SPLIT_CLEANUP フラグの操作は、ハッシュインデックスの分割プロセスにおける原子性を保証するために重要である。
