PLy_plan_init_type

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_planobject.c:55

目的

Pythonの型システムに対して PLyPlan 型を登録し、PL/Python内で実行計画オブジェクトを扱えるように初期化する。

シグネチャまたは形状

void PLy_plan_init_type(void)

入力と出力

  • 入力: なし
  • 出力: なし（成功時）

前提条件と不変条件

  • 本関数はモジュールロード時に呼び出されることを想定している。
  • PLyPlan_spec が有効な PyType_Spec 構造体として定義されている必要がある。

アルゴリズムまたは振る舞い

1. PyType_FromSpec を使用して、PLyPlan_spec からPythonの型オブジェクトを生成し、PLy_PlanType に格納する。 2. 型オブジェクトの生成に失敗した場合（PLy_PlanType が NULL の場合）、elog(ERROR, ...) を呼び出し、PostgreSQLの例外メカニズムを通じて処理を中断する。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpython/plpy_plpymodule.c 内のモジュール初期化処理から呼び出される。
  • 呼び出し先: Python C API の PyType_FromSpec および PostgreSQL の elog。

関連データ構造

  • PLy_PlanType: 初期化されたPython型オブジェクトを保持するグローバル変数。
  • PLyPlan_spec: 型の定義情報を含む PyType_Spec 構造体。

失敗モードと境界条件

  • PyType_FromSpec が失敗した場合、PostgreSQLの ERROR レベルでログ出力され、トランザクションがアボートされる。

保守担当者向けメモ

  • 本関数はPython 3の型システムへの統合を担う。Pythonのバージョンアップに伴う PyType_Spec の仕様変更に注意すること。
  • PLy_PlanType はモジュール全体で共有されるため、スレッドセーフな初期化が保証されている環境（PostgreSQLのモジュールロードプロセス）で実行される必要がある。
