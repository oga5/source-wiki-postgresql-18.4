build_physical_tlist

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/plancat.c:1799

目的

指定されたリレーション（RelOptInfo）の物理的な構造を反映したターゲットリスト（TargetEntry のリスト）を生成します。

シグネチャまたは形状

List *build_physical_tlist(PlannerInfo *root, RelOptInfo *rel)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報構造体。
  • rel: ターゲットとなるリレーション情報。
  • **出力**:
  • リレーションの全属性を順序通りに保持する List *（ターゲットリスト）。生成できない場合は NIL を返します。

前提条件と不変条件

  • rel->relid から取得される RangeTblEntry が有効であること。
  • RTE_RELATION の場合、呼び出し元で既に適切なロックが取得されていること（table_open に NoLock を渡すため）。

アルゴリズムまたは振る舞い

rtekind に基づき、以下の通りターゲットリストを構築します。 1. **RTE_RELATION**: リレーションの全属性を走査し、makeVar で各列の Var ノードを作成して TargetEntry に格納します。削除済み列（attisdropped）や欠損値を持つ列が存在する場合、構築を中断し NIL を返します。 2. **RTE_SUBQUERY**: サブクエリのターゲットリストを走査し、makeVarFromTargetEntry を用いて各列を反映した TargetEntry を作成します。 3. **RTE_FUNCTION / RTE_TABLEFUNC / RTE_VALUES / RTE_CTE / RTE_NAMEDTUPLESTORE / RTE_RESULT**: expandRTE を使用して列リストを取得します。出力に Var 以外のノード（削除済み列など）が含まれる場合は NIL を返します。 4. **その他**: elog(ERROR) によりエラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/createplan.c:658
  • **呼び出し先**: table_open, RelationGetNumberOfAttributes, TupleDescAttr, makeVar, lappend, makeTargetEntry, table_close, makeVarFromTargetEntry, expandRTE, elog

関連データ構造

  • PlannerInfo: プランナのコンテキスト情報。
  • RelOptInfo: 最適化対象のリレーション情報。
  • RangeTblEntry: レンジテーブルエントリ。
  • TargetEntry: ターゲットリストの各要素。
  • Var: 変数ノード。

失敗モードと境界条件

  • 削除済み列や欠損値を持つ列が含まれる場合、関数は NIL を返します。
  • サポートされていない rtekind が渡された場合、elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数は varno を現在のクエリの varno に合わせるために makeVar 等を利用しています。
  • RTE_RELATION 処理時のロック管理は呼び出し元に依存しているため、変更時はロックの整合性に注意してください。
