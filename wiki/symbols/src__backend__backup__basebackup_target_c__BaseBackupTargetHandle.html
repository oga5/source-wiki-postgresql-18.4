BaseBackupTargetHandle

定義

  • 種別: struct
  • 定義箇所: src/backend/backup/basebackup_target.c:28

目的

バックアップの出力先（ターゲット）を抽象化し、検証済みのターゲット情報と、それに関連する詳細な引数を保持するためのハンドル構造体です。

シグネチャまたは形状

struct BaseBackupTargetHandle
{
	BaseBackupTargetType *type;
	void	   *detail_arg;
};

入力と出力

  • **入力**: BaseBackupGetTargetHandle() によって生成される際、ターゲット名と詳細情報が検証され、本構造体に格納されます。
  • **出力**: BaseBackupGetSink() の引数として渡され、ターゲット固有のバックアップシンク（bbsink）パイプラインを構築するために使用されます。

前提条件と不変条件

  • type は BaseBackupTargetType を指し、ターゲットの検証関数やシンク生成関数を定義します。
  • detail_arg は、ターゲット固有の検証済み詳細情報（check_detail の結果など）を保持します。
  • 本構造体は TopMemoryContext 上でメモリ確保されることが想定されています。

アルゴリズムまたは振る舞い

1. BaseBackupGetTargetHandle() により、指定されたターゲット名に対応する BaseBackupTargetType が検索されます。 2. ターゲット固有の検証関数が実行され、成功した場合に本構造体がインスタンス化されます。 3. インスタンス化されたハンドルは、後続の BaseBackupGetSink() 呼び出しにおいて、データ転送パイプラインを生成するためのコンテキストとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c: バックアップ処理の開始時にターゲットを保持するために使用。
  • src/backend/backup/basebackup_target.c: BaseBackupGetTargetHandle で生成され、BaseBackupGetSink で参照される。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありませんが、保持する type メンバを通じてターゲット固有の関数（get_sink 等）が間接的に呼び出されます。

関連データ構造

  • BaseBackupTargetType: ターゲットの定義（名前、検証関数、シンク生成関数）を保持する構造体。
  • bbsink: バックアップデータの転送パイプラインを構成するインターフェース。

失敗モードと境界条件

  • BaseBackupGetTargetHandle() による検索失敗や検証エラーが発生した場合、有効なハンドルは返されません。
  • メモリ不足時には palloc によりエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は src/include/backup/basebackup_target.h にも定義が存在します。変更時は両方の定義が整合していることを確認してください。
  • 拡張モジュールが新しいターゲットを追加する際、detail_arg に格納するデータ構造のライフサイクルは TopMemoryContext に依存するため、メモリ管理に注意してください。
