oauth_json_scalar

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth-oauth.c:291

目的

JSON解析中にスカラー値（文字列）を抽出し、コンテキスト内のターゲットフィールドに格納する。

シグネチャまたは形状

JsonParseErrorType oauth_json_scalar(void *state, char *token, JsonTokenType type)

入力と出力

  • **入力**:
  • state: json_ctx 構造体へのポインタ。
  • token: 解析されたスカラー値の文字列。
  • type: トークンの型（JsonTokenType）。
  • **出力**:
  • JsonParseErrorType: 成功時は JSON_SUCCESS、エラー時は JSON_SEM_ACTION_FAILED または JSON_OUT_OF_MEMORY を返す。

前提条件と不変条件

  • state は json_ctx 型としてキャスト可能であること。
  • 解析対象のJSONは、トップレベルがオブジェクトである必要がある。
  • ctx->target_field が設定されている場合、ネストレベルは1であること。

アルゴリズムまたは振る舞い

1. **トップレベル検証**: ctx->nested が0の場合、トップレベルがオブジェクトでないと判断しエラーを返す。 2. **ターゲットフィールド処理**:

  • ctx->target_field が設定されている場合、ネストレベルが1であることを確認する（それ以外は内部エラー）。
  • 既に値が設定されている場合は重複エラーを返す。
  • トークン型が JSON_TOKEN_STRING 以外の場合はエラーを返す。
  • token を複製して *ctx->target_field に格納し、ctx->target_field および ctx->target_field_name をクリアする。

    • 3. **無視**: ターゲットフィールドが設定されていない場合は、値を無視して JSON_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq-oauth/oauth-curl.c:668
  • src/interfaces/libpq/fe-auth-oauth.c:292
  • **呼び出し先**:
  • oauth_json_set_error / oauth_json_set_error_internal (エラー報告)
  • strdup (メモリ確保)
  • Assert (デバッグ時の整合性チェック)

関連データ構造

  • json_ctx: 解析状態を保持する構造体。

失敗モードと境界条件

  • **JSON構造エラー**: トップレベルがオブジェクトでない場合。
  • **ネストエラー**: ターゲットフィールド探索中に予期せぬネストが発生した場合。
  • **重複エラー**: 同一フィールドが複数回出現した場合。
  • **型エラー**: 期待される文字列型以外のトークンが渡された場合。
  • **メモリ不足**: strdup によるメモリ確保失敗時。

保守担当者向けメモ

  • 本関数は json_ctx を介して状態を管理するため、解析器のコールバックとして登録される際は json_ctx の整合性が保たれていることを確認すること。
  • ctx->target_field は一度値が格納されると NULL にリセットされるため、単一のフィールド抽出を前提としている。
