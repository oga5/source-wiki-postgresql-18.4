create_target_dir

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/file_ops.c:237

目的

ターゲットデータディレクトリ内に新しいディレクトリを作成します。

シグネチャまたは形状

static void create_target_dir(const char *path)

入力と出力

  • **入力**: path (作成するディレクトリの相対パス)
  • **出力**: なし（成功時はディレクトリを作成、失敗時は pg_fatal によりプロセスを終了）

前提条件と不変条件

  • path は path_is_safe_for_extraction によってターゲットディレクトリ外へのアクセスを試みていないことが検証される必要があります。
  • datadir_target がグローバル変数として定義されている必要があります。

アルゴリズムまたは振る舞い

1. path_is_safe_for_extraction(path) を呼び出し、パスが安全でない場合はエラーで終了します。 2. dry_run フラグが有効な場合は、実際のファイルシステム操作を行わず直ちに復帰します。 3. datadir_target と path を結合し、完全なパスを dstpath に生成します。 4. mkdir を pg_dir_create_mode で実行し、ディレクトリを作成します。失敗した場合はエラーメッセージを出力して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/file_ops.c 内の create_target 関数から呼び出されます。
  • **呼び出し先**: path_is_safe_for_extraction (セキュリティチェック), pg_fatal (エラー報告), mkdir (OSシステムコール)。

関連データ構造

  • datadir_target: ターゲットデータディレクトリのパスを保持するグローバル変数。
  • pg_dir_create_mode: ディレクトリ作成時のパーミッション設定。

失敗モードと境界条件

  • **パスの安全性**: path_is_safe_for_extraction が偽を返す場合、pg_fatal により異常終了します。
  • **ディレクトリ作成失敗**: mkdir が失敗した場合（権限不足や親ディレクトリの欠如など）、%m を含むエラーメッセージとともに pg_fatal で異常終了します。

保守担当者向けメモ

  • 本関数は dry_run モードをサポートしています。新しいロジックを追加する際は、必ず dry_run チェックを維持してください。
  • path は相対パスとして扱われる前提です。絶対パスを渡す場合の挙動については datadir_target との結合ロジックに注意してください。
