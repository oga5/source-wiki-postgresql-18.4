JsonPathExecContext

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/adt/jsonpath_exec.c:96

目的

SQL/JSON パス言語の実行エンジンにおいて、クエリ実行時の状態（変数、ルート/カレント要素、モード設定など）を保持・管理するために使用されます。

シグネチャまたは形状

typedef struct JsonPathExecContext
{
    void       *vars;
    JsonPathGetVarCallback getVar;
    JsonbValue *root;
    JsonbValue *current;
    JsonBaseObjectInfo baseObject;
    int         lastGeneratedObjectId;
    int         innermostArraySize;
    bool        laxMode;
    bool        ignoreStructuralErrors;
    bool        throwErrors;
    bool        useTz;
} JsonPathExecContext;

入力と出力

  • **入力**: 実行時に vars（変数）、root（ルート要素）、current（カレント要素）などが設定されます。
  • **出力**: 本構造体自体は関数の戻り値ではなく、実行状態を保持するコンテキストとして各実行関数に渡されます。

前提条件と不変条件

  • laxMode が true の場合、配列の自動アンラップやエラーの抑制が行われます。
  • throwErrors が false の場合、抑制可能なエラーは発生せず、実行が継続されます。
  • JsonPathExecContext は実行のグローバルな状態として機能します。

アルゴリズムまたは振る舞い

  • executeItem などの実行関数群に引数として渡され、再帰的なパス探索の過程で現在の評価対象（current）や変数の解決に使用されます。
  • .keyvalue() 評価時には baseObject や lastGeneratedObjectId を用いてオブジェクトの識別と管理を行います。
  • lax モードと strict モードの切り替えにより、構造的エラーの扱いが変化します。

呼び出し元と呼び出し先

  • **呼び出し元**: executeJsonPath() 等の実行エントリポイントから初期化・参照されます。
  • **呼び出し先**: 以下の関数群が本コンテキストを引数として受け取り、状態を参照・更新します。
  • executeItem, executeItemOptUnwrapTarget, executeItemUnwrapTargetArray
  • executeNextItem, executeItemOptUnwrapResult, executeItemOptUnwrapResultNoThrow
  • executeBoolItem, executeNestedBoolItem, executeAnyItem, executePredicate

関連データ構造

  • JsonbValue: JSON の各要素を表現するデータ構造。
  • JsonBaseObjectInfo: .keyvalue() 評価用のベースオブジェクト情報。
  • JsonPathGetVarCallback: 変数抽出用のコールバック関数。

失敗モードと境界条件

  • ignoreStructuralErrors が true の場合、必須アイテムの欠如や予期しない型などの構造的エラーは無視されます。
  • throwErrors が false の場合、抑制可能なエラーは発生しません。
  • 深いパス構造に対しては、スタックオーバーフローを防ぐためのチェックが実行関数側で必要となります。

保守担当者向けメモ

  • 本構造体は jsonpath_exec.c 内の再帰的な実行ロジック全体で共有されるため、変更時は各 execute 系関数への影響を考慮してください。
  • lax モードと strict モードの挙動の違いが複雑であるため、フラグの取り扱いには注意が必要です。
