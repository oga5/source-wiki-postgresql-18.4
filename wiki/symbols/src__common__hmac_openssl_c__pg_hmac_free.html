pg_hmac_free

定義

  • 種別: function
  • 定義箇所: src/common/hmac_openssl.c:313

目的

HMAC 計算に使用したコンテキストリソースを解放し、機密情報をメモリから消去します。

シグネチャまたは形状

void pg_hmac_free(pg_hmac_ctx *ctx)

入力と出力

  • **入力**: pg_hmac_ctx *ctx（解放対象の HMAC コンテキストへのポインタ）
  • **出力**: なし（void）

前提条件と不変条件

  • ctx が NULL の場合、関数は何もせず直ちに復帰します。
  • バックエンド環境で USE_RESOWNER_FOR_HMAC が有効な場合、コンテキストに関連付けられた ResourceOwner からの登録解除が必須です。

アルゴリズムまたは振る舞い

1. ctx が NULL であれば終了します。 2. OpenSSL の HMAC_CTX_free を呼び出し、内部の OpenSSL コンテキストを破棄します。 3. USE_RESOWNER_FOR_HMAC が定義されている場合、ResourceOwnerForgetHMAC を呼び出してリソース所有者管理から除外します。 4. explicit_bzero を使用して pg_hmac_ctx 構造体全体をゼロクリアし、機密情報を消去します。 5. FREE マクロを使用してコンテキストメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth-scram.c
  • src/common/hmac.c
  • src/common/scram-common.c
  • **呼び出し先**:
  • HMAC_CTX_free (OpenSSL)
  • ResourceOwnerForgetHMAC (バックエンド環境のみ)
  • explicit_bzero
  • FREE

関連データ構造

  • pg_hmac_ctx: HMAC 計算の状態を保持する構造体。

失敗モードと境界条件

  • ctx が NULL の場合は安全に処理されます。
  • explicit_bzero によるメモリ消去は、解放前のメモリ領域に対して行われます。

保守担当者向けメモ

  • 本関数は OpenSSL 実装専用です。他のバックエンド実装（もしあれば）との整合性に注意してください。
  • explicit_bzero は機密情報（鍵など）の残留を防ぐための重要なセキュリティ対策であるため、削除しないでください。
