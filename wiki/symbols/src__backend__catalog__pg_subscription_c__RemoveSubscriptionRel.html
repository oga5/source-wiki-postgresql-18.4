RemoveSubscriptionRel

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_subscription.c:436

目的

システムカタログ pg_subscription_rel から、指定されたサブスクリプション（subid）またはリレーション（relid）に関連するマッピング情報を削除します。

シグネチャまたは形状

void RemoveSubscriptionRel(Oid subid, Oid relid)

入力と出力

  • **入力**:
  • subid (Oid): 対象のサブスクリプションID。InvalidOid を指定可能。
  • relid (Oid): 対象のリレーションID。InvalidOid を指定可能。
  • **出力**: なし（void）。

前提条件と不変条件

  • pg_subscription_rel リレーションに対する RowExclusiveLock が内部で取得されます。
  • subid と relid の少なくとも一方が有効な OID である必要があります。
  • サブスクリプション全体を削除する場合（subid が有効）を除き、同期中のテーブル（srsubstate != SUBREL_STATE_READY）のマッピング削除は許可されず、エラーが発生します。

アルゴリズムまたは振る舞い

1. pg_subscription_rel を RowExclusiveLock モードで開きます。 2. 指定された subid および relid に基づいてスキャンキーを構築します。 3. table_beginscan_catalog を使用して該当するタプルを検索します。 4. 検索された各タプルについて以下を実行します：

  • subid が無効かつテーブル同期が進行中（srsubstate が SUBREL_STATE_READY 以外）である場合、エラーを報告します。
  • CatalogTupleDelete を呼び出し、カタログからタプルを削除します。

    • 5. スキャンを終了し、リレーションを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c:1911
  • src/backend/commands/subscriptioncmds.c:969
  • src/backend/commands/subscriptioncmds.c:1804
  • **呼び出し先**:
  • table_open, table_beginscan_catalog, heap_getnext, table_endscan, table_close (アクセスメソッド)
  • ScanKeyInit, GETSTRUCT, CatalogTupleDelete (カタログ操作)
  • ereport, get_subscription_name, get_rel_name (ユーティリティ)

関連データ構造

  • pg_subscription_rel: 操作対象のシステムカタログ。
  • Form_pg_subscription_rel: カタログタプルの構造体。

失敗モードと境界条件

  • **エラー**: subid が指定されていない状態で、同期が完了していない（srsubstate != SUBREL_STATE_READY）テーブルのマッピングを削除しようとすると、ERRCODE_INVALID_PARAMETER_VALUE でエラーが発生します。
  • **境界条件**: subid または relid に InvalidOid を渡すことで、特定のサブスクリプション全体、あるいは特定のテーブルに関連する全サブスクリプションのマッピングを削除可能です。

保守担当者向けメモ

  • 本関数は pg_subscription_rel の整合性を維持する役割を担います。特にテーブル同期状態のチェックは、孤立したテーブル同期スロットやオリジンが残存するのを防ぐために重要です。
  • 変更を加える際は、pg_subscription_rel.h で定義されている srsubstate の状態遷移ルールとの整合性に注意してください。
