ginRedoCreatePTree

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginxlog.c:43

目的

GINインデックスのポスティングツリー作成操作をWALリプレイ時に再実行し、新しいインデックスページを初期化してデータを復元します。

シグネチャまたは形状

static void ginRedoCreatePTree(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）
  • **出力**: なし（副作用としてバッファへの書き込みと解放を行う）

前提条件と不変条件

  • XLogInitBufferForRedo により、対象ページが適切に初期化されていること。
  • WALレコードのデータ部には ginxlogCreatePostingTree 構造体と、それに続くポスティングリストデータが含まれていること。

アルゴリズムまたは振る舞い

1. XLogInitBufferForRedo を呼び出し、対象ページを初期化します。 2. GinInitBuffer を使用して、ページを GIN_DATA | GIN_LEAF | GIN_COMPRESSED 属性で初期化します。 3. WALレコードから ginxlogCreatePostingTree 情報を取得し、続くポスティングリストデータをページ内の適切な位置（GinDataLeafPageGetPostingList）へコピーします。 4. GinDataPageSetDataSize でデータサイズを設定します。 5. PageSetLSN でページのLSNを更新します。 6. MarkBufferDirty でバッファをダーティにし、UnlockReleaseBuffer でロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginxlog.c:741（gin_redo 関数内でのディスパッチ）
  • **呼び出し先**:
  • XLogInitBufferForRedo
  • BufferGetPage
  • GinInitBuffer
  • XLogRecGetData
  • GinDataLeafPageGetPostingList
  • GinDataPageSetDataSize
  • PageSetLSN
  • MarkBufferDirty
  • UnlockReleaseBuffer

関連データ構造

  • XLogReaderState: WALレコード読み取り状態。
  • ginxlogCreatePostingTree: ポスティングツリー作成時のWALレコードデータ構造。
  • Buffer, Page: GINインデックスのバッファおよびページ管理構造。

失敗モードと境界条件

  • WALレコードのデータサイズが不正な場合、メモリコピー時に境界外アクセスが発生する可能性があります。
  • バッファの初期化や取得に失敗した場合、基盤となる XLogInitBufferForRedo 等がエラーを発生させます。

保守担当者向けメモ

  • 本関数はGINインデックスの新規ポスティングツリー作成を再現する重要な処理です。
  • ページ構造の変更や GinInitBuffer のフラグ定義に変更があった場合、本関数の初期化ロジックも追従する必要があります。
