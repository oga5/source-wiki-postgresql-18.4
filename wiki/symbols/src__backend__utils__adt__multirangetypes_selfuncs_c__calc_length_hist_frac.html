calc_length_hist_frac

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/multirangetypes_selfuncs.c:965

目的

レンジ長ヒストグラムを用いて、指定された範囲 [length1, length2] 内に収まるデータの割合を積分計算により推定します。

シグネチャまたは形状

static double calc_length_hist_frac(Datum *length_hist_values, int length_hist_nvalues,
                                    double length1, double length2, bool equal)

入力と出力

  • **入力**:
  • length_hist_values: レンジ長ヒストグラムの境界値配列。
  • length_hist_nvalues: ヒストグラムの要素数。
  • length1, length2: 積分対象の範囲（下限と上限）。
  • equal: 境界値を含むかどうかのフラグ。
  • **出力**:
  • double: 指定範囲内における累積分布の割合（0.0〜1.0）。

前提条件と不変条件

  • length2 >= length1 であること。
  • ヒストグラムの各ビンは台形として近似され、累積分布関数 $P(x)$ を用いて計算されます。

アルゴリズムまたは振る舞い

1. **初期化**: length_hist_bsearch を用いて length1 が含まれるビンを特定します。 2. **積分計算**: ヒストグラムのビンを順次走査し、各台形の面積（0.5 * (PB + PA) * (B - A)）を累積します。 3. **補間**: 範囲の開始点および終了点がビン境界と一致しない場合、線形補間（get_len_position）を用いて累積確率を算出します。 4. **正規化**: 累積した面積を範囲の幅（length2 - length1）で除算し、平均的な割合を求めます。 5. **例外処理**: length2 が無限大かつ equal が真の場合は 1.0 を返します。また、area と length2 が共に無限大の場合は 0.5 を返して NaN を回避します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/multirangetypes_selfuncs.c 内の複数箇所（統計計算ロジック）。
  • src/backend/utils/adt/rangetypes_selfuncs.c 内の複数箇所。
  • **呼び出し先**:
  • length_hist_bsearch: ヒストグラム内の位置特定。
  • get_len_position: ビン内での相対位置の線形補間。
  • DatumGetFloat8: Datum 型から double への変換。

関連データ構造

  • Datum: PostgreSQL の汎用データ表現。
  • pg_statistic: ヒストグラムデータが格納されるシステムカタログ。

失敗モードと境界条件

  • length2 < 0.0: 0.0 を返します。
  • length1 == length2: ゼロ除算を避けるため、その点における累積確率を直接返します。
  • PA == PB == 0: 台形面積計算時の NaN を防ぐため、面積加算をスキップします。

保守担当者向けメモ

  • 本関数はレンジ長と下限値が独立しているという統計的仮定に基づいています。
  • 積分計算の精度はヒストグラムのビン数に依存します。推定値が不自然な場合は ANALYZE による統計情報の更新を確認してください。
