logicalmsg_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/logicalmsgdesc.c:18

目的

論理メッセージ（Logical Message）に関連するWALレコードの内容を解析し、人間が読める形式の文字列として StringInfo バッファに書き出す。主に pg_waldump 等のログ解析ツールで使用される。

シグネチャまたは形状

void logicalmsg_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 出力先となる StringInfo 構造体。
  • record: 解析対象のWALレコードを保持する XLogReaderState 構造体。
  • **出力**: なし（buf に文字列が追記される）。

前提条件と不変条件

  • record は有効なWALレコードを指していること。
  • xl_logical_message 構造体の prefix フィールドの末尾にはヌル文字（\0）が存在すること（Assert で検証済み）。

アルゴリズムまたは振る舞い

1. XLogRecGetData を使用してレコードの生データポインタを取得する。 2. XLogRecGetInfo でレコードの情報を取得し、XLOG_LOGICAL_MESSAGE 型であるかを確認する。 3. レコードデータを xl_logical_message 構造体にキャストし、プレフィックスとペイロードの開始位置を特定する。 4. トランザクション属性（transactional / non-transactional）、プレフィックス文字列、およびペイロードのサイズを buf に書き出す。 5. ペイロードの各バイトを16進数形式（%02X）に変換し、スペース区切りで buf に追記する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h 内で PG_RMGR マクロを通じて登録されており、WALリソースマネージャのディスパッチ機構から呼び出される。
  • **呼び出し先**: XLogRecGetData, XLogRecGetInfo, appendStringInfo（PostgreSQL内部API）。

関連データ構造

  • xl_logical_message: 論理メッセージのWALレコード構造体。
  • StringInfo: 文字列バッファ管理用構造体。
  • XLogReaderState: WALレコード読み取り状態管理用構造体。

失敗モードと境界条件

  • XLOG_LOGICAL_MESSAGE 以外のレコードタイプが渡された場合、関数は何も行わずに終了する。
  • ペイロードが空の場合、ループ処理はスキップされる。

保守担当者向けメモ

  • ペイロードはバイナリデータである可能性があるため、16進数形式での出力が必須である。
  • xlrec->prefix_size を用いて可変長データを安全に分離しているため、構造体定義の変更時にはこのオフセット計算の整合性に注意すること。
