blbeginscan

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blscan.c:24

目的

Bloom インデックススキャンの初期化を行い、スキャン状態を保持するためのメモリ領域を確保・設定します。

シグネチャまたは形状

IndexScanDesc blbeginscan(Relation r, int nkeys, int norderbys)

入力と出力

  • **入力**:
  • r: スキャン対象のインデックスリレーション。
  • nkeys: スキャンキーの数。
  • norderbys: オーダーバイ句の数。
  • **出力**:
  • 初期化済みの IndexScanDesc 構造体。

前提条件と不変条件

  • RelationGetIndexScan により、有効な IndexScanDesc が生成されている必要があります。
  • palloc によるメモリ確保が成功する必要があります。

アルゴリズムまたは振る舞い

1. RelationGetIndexScan を呼び出し、標準的なインデックススキャン記述子を生成します。 2. BloomScanOpaqueData 構造体分のメモリを palloc で確保し、BloomScanOpaque 型としてキャストします。 3. initBloomState を呼び出し、インデックスリレーションに基づいて so->state を初期化します。 4. 検索シグネチャ so->sign を NULL に初期化します（実際のシグネチャ計算は後のスキャン実行時に行われます）。 5. 確保した so を scan->opaque に格納し、スキャン記述子を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blutils.c: インデックスアクセスメソッドのルーチン定義 (ambeginscan) で参照されています。
  • **呼び出し先**:
  • RelationGetIndexScan (PostgreSQL API)
  • palloc (PostgreSQL API)
  • initBloomState (contrib/bloom/blutils.c)

関連データ構造

  • IndexScanDesc: PostgreSQL の標準的なインデックススキャン記述子。
  • BloomScanOpaqueData: Bloom インデックス固有のスキャン状態（BloomState および検索シグネチャ sign を保持）。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • initBloomState 内部での初期化失敗は、本関数の実行を中断させます。

保守担当者向けメモ

  • 本関数はスキャン開始時のメモリ割り当てのみを担当します。実際の検索条件に基づくシグネチャの生成は blgetbitmap 内で行われるため、本関数では so->sign を NULL に初期化する点に注意してください。
