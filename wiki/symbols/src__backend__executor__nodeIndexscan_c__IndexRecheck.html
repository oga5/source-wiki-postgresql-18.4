IndexRecheck

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexscan.c:387

目的

EvalPlanQual（EPQ）実行時において、インデックススキャンノードが保持するタプルがインデックス条件（indexqualorig）を満たしているかを再検証するために使用されます。

シグネチャまたは形状

static bool IndexRecheck(IndexScanState *node, TupleTableSlot *slot)

入力と出力

  • **入力**:
  • node: インデックススキャン実行状態を保持する IndexScanState 構造体。
  • slot: 再検証対象のタプルを格納した TupleTableSlot。
  • **出力**:
  • bool: タプルがインデックス条件を満たす場合は true、満たさない場合は false を返します。

前提条件と不変条件

  • node->ss.ps.ps_ExprContext が初期化されている必要があります。
  • slot には有効なタプルデータが格納されている必要があります。

アルゴリズムまたは振る舞い

1. node から式評価コンテキスト（ExprContext）を取得します。 2. コンテキストの ecxt_scantuple に検証対象の slot を設定します。 3. ExecQualAndReset を呼び出し、node->indexqualorig（インデックス条件式）を評価します。 4. 評価結果をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeIndexscan.c 内の ExecScanRecheckMtd 型の関数ポインタとして参照されています（行534, 538）。
  • **呼び出し先**:
  • ExecQualAndReset: 式の評価とコンテキストのリセットを実行します。

関連データ構造

  • IndexScanState: インデックススキャンの実行状態を管理する構造体。
  • TupleTableSlot: タプルを保持する汎用的なコンテナ。
  • ExprContext: 式評価のための実行時コンテキスト。

失敗モードと境界条件

  • node または slot が NULL の場合、セグメンテーション違反が発生する可能性があります。
  • indexqualorig が空（NULL）の場合の挙動は、ExecQualAndReset の実装に依存します。

保守担当者向けメモ

  • 本関数は EvalPlanQual 処理の一部として呼び出されるため、インデックススキャン特有のフィルタリングロジックを維持する必要があります。
  • ExecScanRecheckMtd インターフェースを介して利用されるため、シグネチャ変更時は関連する関数ポインタの定義も確認してください。
