PublicationDesc

定義

  • 種別: struct
  • 定義箇所: src/include/catalog/pg_publication.h:86

目的

論理レプリケーションにおいて、特定のテーブルに対するパブリケーションの有効性や制約（行フィルタ、列リスト、生成列）が、更新（UPDATE）や削除（DELETE）操作に対して適切であるかを判定するためのメタデータ構造体です。

シグネチャまたは形状

typedef struct PublicationDesc
{
	PublicationActions pubactions;
	bool		rf_valid_for_update;
	bool		rf_valid_for_delete;
	bool		cols_valid_for_update;
	bool		cols_valid_for_delete;
	bool		gencols_valid_for_update;
	bool		gencols_valid_for_delete;
} PublicationDesc;

入力と出力

  • **入力**: RelationBuildPublicationDesc 関数により、リレーション情報から構築されます。
  • **出力**: レプリケーション実行時の各操作（UPDATE/DELETE）において、対象の列や行フィルタがレプリカアイデンティティと整合しているかを示す真偽値フラグを提供します。

前提条件と不変条件

  • pubactions に UPDATE または DELETE が含まれない場合、各検証フラグ（rf_valid_*, cols_valid_*, gencols_valid_*）は論理的に真として扱われます。
  • 各フラグは、対象となる列がレプリカアイデンティティの一部であるか、あるいはパブリケーションの定義と整合している場合に真となります。

アルゴリズムまたは振る舞い

本構造体は、リレーションキャッシュ（relcache）内に保持され、レプリケーションの実行時に以下の検証結果を保持します。 1. **行フィルタ検証**: rf_valid_for_update/delete は、行フィルタで使用される列がレプリカアイデンティティに含まれているかを判定します。 2. **列リスト検証**: cols_valid_for_update/delete は、パブリケーションで指定された列がレプリカアイデンティティと整合しているかを判定します。 3. **生成列検証**: gencols_valid_for_update/delete は、レプリカアイデンティティに含まれる生成列が適切に公開されているかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/relcache.c: RelationBuildPublicationDesc 関数内で初期化および構築されます。
  • src/backend/executor/execReplication.c: レプリケーション実行時に参照されます。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • PublicationActions: パブリケーションが許可するDML操作（INSERT, UPDATE, DELETE, TRUNCATE）のフラグセット。
  • Relation: rd_pubdesc フィールドを通じて本構造体を保持します。

失敗モードと境界条件

  • RelationBuildPublicationDesc が呼び出されない場合、rd_pubdesc は NULL となる可能性があります。
  • パブリケーションの定義が変更された場合、キャッシュの無効化と再構築が必要です。

保守担当者向けメモ

  • 本構造体は relcache にキャッシュされるため、メモリレイアウトの変更は Relation 構造体との整合性に注意してください。
  • 各フラグの判定ロジックは src/backend/utils/cache/relcache.c 内の RelationBuildPublicationDesc 実装に依存しています。
