AlterTableGetRelOptionsLockLevel

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/common/reloptions.c:2143

目的

ALTER TABLE 等で指定されたストレージパラメータ（reloptions）の変更を適用する際に、リレーションに対して取得すべき最小限の LOCKMODE を算出する。

シグネチャまたは形状

LOCKMODE AlterTableGetRelOptionsLockLevel(List *defList)

入力と出力

  • **入力**: defList (変更対象のパラメータを含む DefElem リスト)
  • **出力**: LOCKMODE (算出されたロックレベル。パラメータが指定されない場合は AccessExclusiveLock を返す)

前提条件と不変条件

  • relOpts 配列が初期化されている必要がある（未初期化の場合は内部で initialize_reloptions() を呼び出す）。
  • 入力リストが空 (NIL) の場合、デフォルトとして AccessExclusiveLock を返す。

アルゴリズムまたは振る舞い

1. defList が空であれば AccessExclusiveLock を即座に返す。 2. need_initialization フラグを確認し、必要に応じて initialize_reloptions() を実行する。 3. defList 内の各 DefElem を走査し、定義済みの全オプション (relOpts) と名前を照合する。 4. 一致するオプションが見つかった場合、そのオプションが保持する lockmode を確認し、現在保持している lockmode よりも高い（強い）レベルであれば更新する。 5. 全てのパラメータを走査した後、最終的な lockmode を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/tablecmds.c:4829 (cmd_lockmode の決定に使用)
  • **呼び出し先**: initialize_reloptions (必要時のみ)

関連データ構造

  • DefElem: SQL の WITH 句等で指定されたパラメータ定義を保持する構造体。
  • relopt_gen: オプションのメタデータ（名前、ロックレベル等）を保持する基底構造体。

失敗モードと境界条件

  • 指定されたパラメータ名が relOpts に存在しない場合、そのパラメータはロックレベルの算出に影響を与えない（無視される）。
  • 複数のパラメータが指定された場合、それらの中で最も強いロックレベルが選択される。

保守担当者向けメモ

  • 新しいストレージパラメータを追加する際は、relOpts 配列に適切な lockmode を設定すること。これにより、本関数が自動的に正しいロックレベルを算出する。
