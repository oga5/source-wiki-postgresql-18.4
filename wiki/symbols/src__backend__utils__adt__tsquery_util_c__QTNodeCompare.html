QTNodeCompare

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_util.c:96

目的

2つの QTNode ツリー構造を比較し、その順序関係（等価、大小）を判定します。主に tsquery の正規化や等価性判定の基盤として使用されます。

シグネチャまたは形状

int QTNodeCompare(QTNode *an, QTNode *bn)

入力と出力

  • **入力**: 比較対象となる2つの QTNode 構造体へのポインタ an および bn。
  • **出力**:
  • 0: 両者が等価である場合。
  • 正の整数: an が bn より大きい場合。
  • 負の整数: an が bn より小さい場合。

前提条件と不変条件

  • 再帰的な関数であるため、スタックオーバーフローを防ぐために check_stack_depth() が呼び出されます。
  • valnode->type が QI_OPR または QI_VAL 以外である場合、エラーが発生します。

アルゴリズムまたは振る舞い

1. **型比較**: valnode->type を比較し、異なる場合は型値に基づいて順序を決定します。 2. **演算子ノード (QI_OPR) の場合**:

  • 演算子種別 (oper) を比較します。
  • 子ノードの数 (nchild) を比較します。
  • 子ノードを再帰的に比較し、最初に見つかった非ゼロの差異を返します。
  • OP_PHRASE の場合は、距離 (distance) も比較対象となります。

    • 3. **オペランドノード (QI_VAL) の場合**:

  • CRC 値 (valcrc) を比較します。
  • CRC が一致する場合、tsCompareString を用いて文字列の比較を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery_op.c
  • src/backend/utils/adt/tsquery_rewrite.c
  • src/backend/utils/adt/tsquery_util.c (内部再帰および QTNEq 等)
  • **呼び出し先**:
  • check_stack_depth()
  • tsCompareString()
  • QTNodeCompare() (再帰呼び出し)

関連データ構造

  • QTNode: tsquery を表現するツリー構造体。
  • QueryOperator: 演算子ノードのデータ構造。
  • QueryOperand: オペランドノードのデータ構造。

失敗モードと境界条件

  • **スタックオーバーフロー**: 極端に深いツリー構造を比較した場合に発生する可能性があります。
  • **エラー終了**: valnode->type が未知の値（QI_OPR および QI_VAL 以外）である場合、elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数は QTNSort によるソート済みツリーを前提として利用されることが一般的です。
  • OP_PHRASE 以外の演算子では距離の比較が行われないため、必要に応じて拡張を検討してください。
