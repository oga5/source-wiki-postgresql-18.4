check_single_dir

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/exec.c:311

目的

指定されたデータディレクトリ内の特定のサブディレクトリが存在し、かつそれがディレクトリであることを検証します。

シグネチャまたは形状

static void check_single_dir(const char *pg_data, const char *subdir)

入力と出力

  • **入力**:
  • pg_data: PostgreSQL データディレクトリのパス。
  • subdir: 検証対象のサブディレクトリ名（空文字列の場合は pg_data 自体を指す）。
  • **出力**: なし（検証失敗時は report_status を通じて致命的エラーとして終了）。

前提条件と不変条件

  • pg_data および subdir は有効な文字列ポインタであること。
  • Windows 環境において、末尾にスラッシュが含まれるパスに対して stat() が正しく動作しない制約を考慮したパス構築が行われること。

アルゴリズムまたは振る舞い

1. pg_data と subdir を結合し、完全なパスを subDirName に生成します。この際、subdir が空でない場合は間に / を挿入します。 2. stat() を呼び出し、パスの存在を確認します。 3. stat() が失敗した場合、または対象がディレクトリではない場合（S_ISDIR が偽）、report_status(PG_FATAL, ...) を呼び出して処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/exec.c 内の check_data_dir（推測される文脈）から、base, global, pg_multixact, pg_subtrans, PG_TBLSPC_DIR, pg_twophase, pg_xlog, pg_wal, pg_clog などの各ディレクトリ検証のために呼び出されます。
  • **呼び出し先**:
  • snprintf (標準ライブラリ)
  • stat (システムコール)
  • report_status (ユーティリティ関数)

関連データ構造

  • struct stat: ファイル属性情報を保持するために使用。

失敗モードと境界条件

  • **パスの存在確認失敗**: stat() が 0 以外を返した場合、%m を用いてシステムエラーメッセージと共に致命的エラーを報告します。
  • **ディレクトリ判定失敗**: 対象パスが存在しても、それがディレクトリでない場合は致命的エラーを報告します。
  • **Windows 特有の挙動**: subdir が空の場合、末尾にスラッシュを付加しないパス構築を行い、stat() の制限を回避しています。

保守担当者向けメモ

  • 本関数は pg_upgrade におけるディレクトリ構造の整合性チェックの基盤です。新しい PostgreSQL バージョンでディレクトリ構成が変更された場合、呼び出し側のリストを更新する必要があります。
