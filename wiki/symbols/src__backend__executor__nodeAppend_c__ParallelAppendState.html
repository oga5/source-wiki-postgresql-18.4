ParallelAppendState

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeAppend.c:69

目的

並列クエリ実行時において、複数のワーカー間で Append ノードのサブプラン実行状況を共有・管理するための状態保持構造体です。

シグネチャまたは形状

struct ParallelAppendState
{
    LWLock      pa_lock;
    int         pa_next_plan;
    bool        pa_finished[FLEXIBLE_ARRAY_MEMBER];
};

入力と出力

  • **入力**: 共有メモリ（DSM）上に配置され、各ワーカープロセスがアクセスします。
  • **出力**: pa_next_plan の更新によるサブプランの動的割り当て、および pa_finished 配列による各サブプランの完了状態の共有。

前提条件と不変条件

  • pa_lock は、pa_next_plan を選択・更新する際の排他制御に使用されます。
  • pa_finished[i] の不変条件:
  • 非部分（non-partial）プランの場合: ワーカーがプランを選択した時点で true に設定されます。
  • 部分（partial）プランの場合: ワーカーがプランを完了するまで false を維持します。

アルゴリズムまたは振る舞い

1. **サブプラン選択**: ワーカーは pa_lock を取得し、pa_next_plan を参照して次に実行すべきサブプランを決定します。 2. **状態更新**: 選択したプランの特性に応じて pa_finished を更新し、他のワーカーと実行状況を同期します。 3. **無効化**: パーティションプルーニング等で無効と判断されたサブプランについては、事前に pa_finished を true に設定することで、ワーカーによる選択を回避します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeAppend.c 内の並列実行ロジック（ExecAppend 等）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数呼び出しは含みません。
  • **補足**: src/include/nodes/execnodes.h において AppendState 構造体の as_pstate フィールドとして保持されています。

関連データ構造

  • AppendState: Append ノードの実行時状態を保持する構造体。
  • LWLock: 並列処理における排他制御用ロック。

失敗モードと境界条件

  • **競合**: 複数のワーカーが同時にサブプランを選択しようとする場合、pa_lock によるシリアライズが必須です。
  • **メモリ**: pa_finished は FLEXIBLE_ARRAY_MEMBER を使用しているため、サブプラン数に応じた適切なメモリ確保が必要です。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されるため、ポインタを含めることはできません。
  • pa_finished の更新タイミングは、プランが「部分」か「非部分」かによって厳密に制御する必要があるため、変更時は nodeAppend.c 内の選択ロジックとの整合性に注意してください。
