pgp_cfb_free

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-cfb.c:82

目的

PGP_CFB 暗号化コンテキストを破棄し、関連するリソースを解放する。

シグネチャまたは形状

void pgp_cfb_free(PGP_CFB *ctx)

入力と出力

  • 入力: PGP_CFB *ctx - 解放対象の暗号化コンテキストへのポインタ。
  • 出力: なし。

前提条件と不変条件

  • ctx は pgp_cfb_create 等で適切に初期化されている必要がある。
  • ctx が NULL の場合、pfree の仕様に依存する（PostgreSQL のメモリ管理規則に従う）。

アルゴリズムまたは振る舞い

1. ctx->ciph に保持されている暗号器を px_cipher_free を呼び出して解放する。 2. px_memset を使用して ctx 構造体全体を 0 で埋め、機密情報をメモリ上から消去する。 3. pfree(ctx) を呼び出し、コンテキスト構造体自体が使用していたメモリを解放する。

呼び出し元と呼び出し先

  • 呼び出し先: px_cipher_free, px_memset, pfree
  • 呼び出し元:
  • contrib/pgcrypto/pgp-decrypt.c (複数箇所)
  • contrib/pgcrypto/pgp-encrypt.c (複数箇所)
  • contrib/pgcrypto/pgp-pubkey.c

関連データ構造

  • PGP_CFB: 暗号化コンテキストを保持する構造体。本関数によりライフサイクルが終了する。

失敗モードと境界条件

  • ctx が NULL の場合、px_cipher_free や px_memset へのポインタ渡しでセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は機密情報を含むメモリをゼロクリアしてから解放する設計となっている。メモリ管理の変更を行う際は、このセキュリティ上の不変条件を維持すること。
