FreezeMultiXactId

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam.c:6712

目的

VACUUM 処理中に、ヒープタプルの xmax に設定された MultiXactId を凍結（freeze）または整理し、古いトランザクション情報を安全に除去または更新する。

シグネチャまたは形状

TransactionId FreezeMultiXactId(MultiXactId multi, uint16 t_infomask,
                                const struct VacuumCutoffs *cutoffs, uint16 *flags,
                                HeapPageFreeze *pagefrz)

入力と出力

  • **入力**:
  • multi: 処理対象の MultiXactId。
  • t_infomask: タプルの t_infomask。
  • cutoffs: relminmxid, OldestMxact, relfrozenxid, OldestXmin, FreezeLimit, MultiXactCutoff を含む構造体。
  • flags: 処理結果のフラグ（FRM_INVALIDATE_XMAX, FRM_RETURN_IS_XID, FRM_NOOP）。
  • pagefrz: ページ凍結状態を管理する構造体。
  • **出力**:
  • 戻り値: 新しい xmax となる TransactionId（または InvalidTransactionId）。
  • flags および pagefrz 経由で副作用を返却。

前提条件と不変条件

  • t_infomask に HEAP_XMAX_IS_MULTI が設定されている必要がある。
  • multi が cutoffs->relminmxid より古い場合はエラーとなる。
  • multi が cutoffs->OldestMxact より古い場合、実行中のメンバーが存在しないことを確認した上で、ロックのみであれば無効化し、更新が含まれる場合は更新者 XID を保持する。

アルゴリズムまたは振る舞い

1. **初期チェック**: MultiXactId が無効、またはアップグレード済みロックの場合は FRM_INVALIDATE_XMAX を設定して終了。 2. **古い MultiXact の処理**: OldestMxact より古い場合、実行中かを確認し、ロックのみなら無効化。更新が含まれる場合は、更新者 XID が relfrozenxid や OldestXmin と比較して妥当か検証し、必要に応じて XID を返す。 3. **メンバー走査**: GetMultiXactIdMembers で全メンバーを取得。 4. **凍結判定**: 各メンバーの XID が FreezeLimit より古い場合、または MultiXactId 自体が MultiXactCutoff より古い場合は need_replace を真とする。 5. **NOOP 処理**: need_replace が偽の場合、FRM_NOOP を設定し、pagefrz の追跡情報を更新して multi をそのまま返す。 6. **再構成**: need_replace が真の場合、メンバーを精査して保持すべき XID を特定する（実装はソース抜粋の後半へ続く）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam.c 内の VACUUM 関連処理（heapam.c:7139 等）。
  • **呼び出し先**: MultiXactIdIsValid, MultiXactIdPrecedes, MultiXactIdIsRunning, MultiXactIdGetUpdateXid, TransactionIdPrecedes, TransactionIdDidCommit, GetMultiXactIdMembers, pfree, palloc。

関連データ構造

  • MultiXactId, TransactionId
  • VacuumCutoffs
  • HeapPageFreeze
  • MultiXactMember

失敗モードと境界条件

  • relminmxid や relfrozenxid より古い値が発見された場合、ERRCODE_DATA_CORRUPTED でエラー終了する。
  • 実行中の古い MultiXactId が発見された場合、エラー終了する。
  • メンバー数が 0 の場合、FRM_INVALIDATE_XMAX を設定して終了する。

保守担当者向けメモ

  • 本関数は VACUUM によるヒープ凍結の核心部分であり、MultiXactId のライフサイクル管理と密接に関係している。
  • FRM_NOOP 以外のケースでは、xmax を InvalidTransactionId にするか、単一の XID に置き換えることで、確実に凍結状態へ移行させる設計となっている。
