gistBuffersGetFreeBlock

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistbuildbuffers.c:467

目的

GiST インデックスのバッファリングビルドにおいて、一時ファイル内で再利用可能なブロック番号を取得、または新規ブロック番号を割り当てる。

シグネチャまたは形状

static long gistBuffersGetFreeBlock(GISTBuildBuffers *gfbb)

入力と出力

  • **入力**: GISTBuildBuffers *gfbb（バッファリングビルドの状態を保持する構造体）
  • **出力**: long（割り当てられたブロック番号）

前提条件と不変条件

  • gfbb は初期化済みであること。
  • gfbb->nFreeBlocks が 0 以上であること。

アルゴリズムまたは振る舞い

1. gfbb->nFreeBlocks が 0 より大きい場合、freeBlocks 配列の末尾からブロック番号を取り出し、nFreeBlocks をデクリメントして返す（再利用）。 2. gfbb->nFreeBlocks が 0 の場合、現在のファイル末尾を示す gfbb->nFileBlocks を返し、その後 nFileBlocks をインクリメントする（新規割り当てによるファイル拡張）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistbuildbuffers.c:254
  • src/backend/access/gist/gistbuildbuffers.c:370
  • **呼び出し先**:
  • 外部関数への呼び出しは含まれない。

関連データ構造

  • GISTBuildBuffers: バッファリングビルドの全体状態を管理する構造体。freeBlocks 配列、nFreeBlocks カウンタ、nFileBlocks カウンタを保持する。

失敗モードと境界条件

  • nFileBlocks が増加し続けることで、一時ファイルが物理的に拡張される。
  • 呼び出し元でエラーハンドリングは行われていない（gistbuildbuffers.c の設計方針に従い、エラー時は上位層の elog に依存）。

保守担当者向けメモ

  • 本関数は gistUnloadNodeBuffer 等で解放されたブロックを優先的に再利用する仕組みである。
  • freeBlocks 配列の容量制限については、本関数内ではチェックされていないため、構造体定義側の管理に依存する。
