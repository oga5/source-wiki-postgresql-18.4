slist_mutable_iter

定義

  • 種別: struct
  • 定義箇所: src/include/lib/ilist.h:272

目的

単一連結リスト（slist）を走査する際、走査中のノードを安全に削除できるようにするためのイテレータ構造体です。

シグネチャまたは形状

typedef struct slist_mutable_iter
{
	slist_node *cur;  /* 現在の要素 */
	slist_node *next; /* 次に走査するノード */
	slist_node *prev; /* 削除操作のための直前のノード */
} slist_mutable_iter;

入力と出力

  • 本構造体は、リスト走査のコンテキストを保持するために使用されます。
  • slist_delete_current(slist_mutable_iter *iter) 関数において、現在のノードをリストから削除するための入力として利用されます。

前提条件と不変条件

  • cur は現在処理中のノードを指します。
  • next は、cur が削除された場合でも走査を継続できるよう、あらかじめ保持された次のノードを指します。
  • prev は、単一連結リストにおいてノード削除時にリンクを繋ぎ変えるために必要な、直前のノードを指します。

アルゴリズムまたは振る舞い

  • リストの走査中に slist_delete_current を呼び出すことで、リストの整合性を保ったまま現在のノードを安全に削除できます。
  • イテレータが「次のノード」を内部的に保持しているため、現在のノードを削除しても走査ループを中断せずに継続可能です。

呼び出し元と呼び出し先

  • **呼び出し元（確認された参照先）**:
  • src/backend/executor/spi.c
  • src/backend/storage/ipc/dsm.c
  • src/backend/utils/misc/guc.c
  • **呼び出し先**:
  • slist_delete_current（src/include/lib/ilist.h:1084）が本構造体を操作します。

関連データ構造

  • slist_node: リストの各要素を構成するリンク構造体。
  • slist_head: 単一連結リストの起点。

失敗モードと境界条件

  • リストが空の場合や、イテレータがリストの末尾に達している場合の挙動については、使用する走査マクロ（slist_foreach_modify 等）の仕様に依存します。
  • 削除対象のノードがリストの先頭である場合、prev の扱いに注意が必要です（slist_delete_current の実装に依存）。

保守担当者向けメモ

  • 本構造体は slist_foreach_modify マクロ内で使用されることを想定しています。
  • 構造体内の各ポインタは、リストの走査と削除の整合性を維持するために不可欠です。直接的な手動操作は避け、提供されているマクロおよび関数経由で利用してください。
