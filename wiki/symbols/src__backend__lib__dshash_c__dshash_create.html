dshash_create

定義

  • 種別: function
  • 定義箇所: src/backend/lib/dshash.c:205

目的

動的共有メモリ（DSA）領域上に、プロセス間で共有可能な新しいオープンハッシュテーブルを作成し、初期化する。

シグネチャまたは形状

dshash_table *dshash_create(dsa_area *area, const dshash_parameters *params, void *arg)

入力と出力

  • **入力**:
  • area: ハッシュテーブルを配置する DSA 領域へのポインタ。
  • params: ハッシュテーブルの動作を定義するパラメータ構造体。
  • arg: ハッシュテーブルに関連付ける任意のユーザーデータ。
  • **出力**:
  • 成功時: 初期化されたバックエンドローカルな dshash_table 構造体へのポインタ。
  • 失敗時: メモリ不足等のエラーが発生した場合は ereport(ERROR, ...) により中断される。

前提条件と不変条件

  • params->tranche_id が有効な LWLock トランシェ ID であること。
  • area が有効な DSA 領域であること。
  • 内部的に DSHASH_NUM_PARTITIONS 個のパーティションと、それと同数の初期バケットが確保される。

アルゴリズムまたは振る舞い

1. バックエンドローカルな dshash_table 構造体を palloc で確保する。 2. 共有メモリ上に dshash_table_control 構造体を dsa_allocate で確保する。 3. 制御ブロックにマジックナンバー、ハンドル、トランシェ ID 等のメタデータを設定する。 4. DSHASH_NUM_PARTITIONS 個のパーティションに対して LWLockInitialize を実行し、カウンタを 0 に初期化する。 5. 初期バケット配列を dsa_allocate_extended で確保する。確保に失敗した場合は、制御ブロックを解放しエラーを報告する。 6. ローカルの dshash_table に共有メモリ上のバケットアドレスとサイズ情報を反映し、構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/launcher.c
  • src/backend/storage/ipc/dsm_registry.c
  • src/backend/utils/activity/pgstat_shmem.c
  • src/backend/utils/cache/typcache.c
  • **呼び出し先**:
  • palloc
  • dsa_allocate / dsa_allocate_extended
  • dsa_get_address
  • LWLockInitialize
  • dsa_free
  • ereport

関連データ構造

  • dshash_table: バックエンドローカルな状態保持用。
  • dshash_table_control: 共有メモリ上の制御ブロック。
  • dshash_partition: ロックとエントリ数を管理するパーティション構造体。
  • dsa_area: メモリ割り当て基盤。

失敗モードと境界条件

  • **メモリ不足**: dsa_allocate_extended が失敗した場合、確保済みの制御ブロックを解放した上でエラーを投げる。
  • **初期サイズ**: 初期バケット数は DSHASH_NUM_PARTITIONS に固定されている。

保守担当者向けメモ

  • 本関数はハッシュテーブルの初期化のみを担当する。リサイズ処理は別途 resize 関数が担う。
  • 共有メモリ上のデータ構造へのアクセスには dsa_pointer を介したアドレス解決が必須である。
