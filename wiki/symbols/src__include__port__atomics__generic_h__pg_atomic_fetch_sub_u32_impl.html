pg_atomic_fetch_sub_u32_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/generic.h:185

目的

32ビット符号なし整数アトミック変数から指定された値を減算し、減算前の値を返す。

シグネチャまたは形状

static inline uint32 pg_atomic_fetch_sub_u32_impl(volatile pg_atomic_uint32 *ptr, int32 sub_)

入力と出力

  • **入力**:
  • ptr: 操作対象となる pg_atomic_uint32 型へのポインタ。
  • sub_: 減算する値（int32）。
  • **出力**:
  • 減算操作が行われる直前の、アトミック変数の値（uint32）。

前提条件と不変条件

  • ptr は volatile 修飾されており、アトミック操作が保証される必要がある。
  • 本関数は atomics.h を経由してインクルードされる環境（INSIDE_ATOMICS_H が定義された環境）で使用されることを前提とする。

アルゴリズムまたは振る舞い

  • 内部的に pg_atomic_fetch_add_u32_impl を呼び出し、減算値 sub_ の符号を反転させた値を加算することで減算をエミュレートする。
  • pg_atomic_fetch_add_u32_impl が提供するアトミックな加算ロジック（CASループ等）に依存する。

呼び出し元と呼び出し先

  • **呼び出し先**: pg_atomic_fetch_add_u32_impl
  • **呼び出し元**:
  • src/include/port/atomics.h:385
  • src/include/port/atomics/generic.h:232
  • src/include/port/atomics/generic-gcc.h:216（定義の重複または再定義の可能性あり）

関連データ構造

  • pg_atomic_uint32: アトミック操作対象となる32ビット整数型。

失敗モードと境界条件

  • ハードウェアが直接的なアトミック減算をサポートしていない場合、pg_atomic_fetch_add_u32_impl が行うCASループの競合状況により、処理がループする可能性がある。
  • 減算によるアンダーフローは符号なし整数のラップアラウンド規則に従う。

保守担当者向けメモ

  • 本関数は pg_atomic_fetch_add_u32_impl のラッパーであるため、加算側の実装が最適化された場合、本関数も自動的にその恩恵を受ける。
  • generic-gcc.h 等で個別に定義されている場合があるため、プラットフォーム固有の実装との競合に注意すること。
