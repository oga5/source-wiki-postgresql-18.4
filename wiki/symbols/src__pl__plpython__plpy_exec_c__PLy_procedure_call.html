PLy_procedure_call

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_exec.c:1059

目的

PL/Python プロシージャのコードを Python インタプリタ上で評価し、実行結果を PostgreSQL に返却するラッパー関数です。実行中のエラーを捕捉し、PostgreSQL のエラーとして伝播させる役割を担います。

シグネチャまたは形状

PyObject *PLy_procedure_call(PLyProcedure *proc, const char *kargs, PyObject *vargs)

入力と出力

  • **入力**:
  • proc: 実行対象のプロシージャ情報（PLyProcedure 構造体）。
  • kargs: Python グローバル辞書に引数を格納する際のキー名。
  • vargs: kargs に対応する Python オブジェクト（引数）。
  • **出力**:
  • 成功時: Python コードの実行結果である PyObject*。
  • 失敗時: PLy_elog(ERROR, ...) を呼び出し、PostgreSQL のエラーメカニズムへ制御を移すため、呼び出し元には戻りません。

前提条件と不変条件

  • proc->globals には有効な Python 辞書が保持されている必要があります。
  • 実行開始時のサブトランザクションレベルを記録し、終了時に PLy_abort_open_subtransactions を通じて整合性を維持します。
  • 実行中、explicit_subtransactions のリスト長が開始時より短くなることはありません（ネストのみ許可）。

アルゴリズムまたは振る舞い

1. PyDict_SetItemString を使用し、指定されたキー kargs で引数 vargs を proc->globals に登録します。 2. PG_TRY ブロック内で PyEval_EvalCode を呼び出し、Python コードを実行します。 3. PG_FINALLY ブロックにて、実行中に開かれたサブトランザクションを PLy_abort_open_subtransactions でクリーンアップします。 4. 実行結果 rv が NULL（Python 側で例外発生）の場合、PLy_elog を呼び出し PostgreSQL のエラーとして報告します。 5. 正常終了時は rv を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_exec.c:102
  • src/pl/plpython/plpy_exec.c:367
  • **呼び出し先**:
  • PyDict_SetItemString (Python C API)
  • PyEval_EvalCode (Python C API)
  • PLy_abort_open_subtransactions
  • PLy_elog

関連データ構造

  • PLyProcedure: 実行するコードやグローバル辞書を保持する構造体。
  • PyObject: Python のオブジェクト型。

失敗モードと境界条件

  • Python コード内で例外が発生した場合、rv が NULL となり、PLy_elog を介して PostgreSQL の ERROR が発生します。
  • サブトランザクションの不整合が発生した場合、Assert によりデバッグビルド時に停止します。

保守担当者向けメモ

  • 本関数は PG_TRY ブロックを使用しているため、内部でのメモリ割り当てや Python オブジェクトの参照カウント管理には細心の注意が必要です。
  • PLy_abort_open_subtransactions は、Python 実行中に開始されたサブトランザクションが適切に閉じられていない場合に備えた安全装置です。
