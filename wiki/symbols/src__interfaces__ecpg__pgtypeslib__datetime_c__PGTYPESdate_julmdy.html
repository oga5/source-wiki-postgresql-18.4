PGTYPESdate_julmdy

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/datetime.c:114

目的

内部的な date 型（2000年1月1日を基準としたオフセット値）を、月・日・年の整数配列形式に変換します。

シグネチャまたは形状

void PGTYPESdate_julmdy(date jd, int *mdy)

入力と出力

  • **入力**:
  • jd (date): 変換対象の日付値。
  • mdy (int *): 結果を格納するための3要素以上の整数配列へのポインタ。
  • **出力**:
  • なし（mdy 配列を直接更新）。mdy[0] に月、mdy[1] に日、mdy[2] に年が格納されます。

前提条件と不変条件

  • mdy は少なくとも3つの int を保持できるメモリ領域を指している必要があります。
  • 内部計算において、2000年1月1日を基準としたユリウス日への変換が行われます。

アルゴリズムまたは振る舞い

1. 入力された jd に date2j(2000, 1, 1) を加算し、絶対的なユリウス日を算出します。 2. j2date 関数を呼び出し、算出したユリウス日を年 (y)、月 (m)、日 (d) に分解します。 3. 分解された値を mdy 配列の各インデックス（0:月, 1:日, 2:年）に代入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c:545
  • src/interfaces/ecpg/test/expected/pgtypeslib-dt_test.c:139
  • **呼び出し先**:
  • j2date（日付分解用内部関数）
  • date2j（基準日からのオフセット計算用内部関数）

関連データ構造

  • date: ECPG内で使用される日付型（内部的には整数値）。

失敗モードと境界条件

  • mdy ポインタが NULL の場合、セグメンテーション違反が発生する可能性があります。
  • 入力 jd が極端な値である場合、j2date の処理範囲に依存します。

保守担当者向けメモ

  • 本関数は src/interfaces/ecpg/include/pgtypes_date.h で宣言されています。
  • 戻り値がないため、エラー発生時の通知機能はありません。呼び出し側で入力値の妥当性を確認してください。
