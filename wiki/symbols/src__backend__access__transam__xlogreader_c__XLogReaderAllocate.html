XLogReaderAllocate

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogreader.c:106

目的

WAL（Write-Ahead Log）レコードを読み取り、解析するための状態管理構造体 XLogReaderState を初期化および割り当てます。

シグネチャまたは形状

XLogReaderState *XLogReaderAllocate(int wal_segment_size, const char *waldir, XLogReaderRoutine *routine, void *private_data);

入力と出力

  • **入力**:
  • wal_segment_size: WALセグメントのサイズ。
  • waldir: WALファイルが格納されているディレクトリパス（NULL可）。
  • routine: ページ読み込み等のコールバック関数を定義した構造体。
  • private_data: 呼び出し元が保持する任意のデータへのポインタ。
  • **出力**:
  • 成功時: 初期化済みの XLogReaderState 構造体へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • palloc_extended を使用するため、適切なメモリコンテキストがアクティブである必要があります。
  • routine で指定されるコールバック関数は、呼び出し元によって適切に実装されている必要があります。
  • readBuf は XLOG_BLCKSZ でアライメントされた状態で確保されます。

アルゴリズムまたは振る舞い

1. XLogReaderState 構造体メモリをゼロ初期化して確保します。 2. routine を状態構造体にコピーします。 3. XLOG_BLCKSZ サイズの readBuf を確保し、アライメントを保証します。 4. WALOpenSegmentInit を呼び出し、セグメント情報を初期化します。 5. エラーメッセージ格納用の errormsg_buf を確保します。 6. allocate_recordbuf を呼び出し、レコード読み取り用のバッファを初期化します。 7. 確保した XLogReaderState を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_walinspect/pg_walinspect.c
  • src/backend/access/transam/twophase.c
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/postmaster/walsummarizer.c
  • src/backend/replication/logical/logical.c
  • src/backend/replication/walsender.c
  • src/bin/pg_rewind/parsexlog.c
  • src/bin/pg_waldump/pg_waldump.c
  • **呼び出し先**:
  • palloc_extended
  • WALOpenSegmentInit
  • allocate_recordbuf
  • pfree (エラー発生時のみ)

関連データ構造

  • XLogReaderState: リーダーの全状態を保持するメイン構造体。
  • XLogReaderRoutine: 外部から提供されるI/Oコールバック定義。

失敗モードと境界条件

  • メモリ確保（palloc_extended）が失敗した場合、関数は NULL を返します。

保守担当者向けメモ

  • 本関数は FRONTEND マクロの有無により挙動が変化します。フロントエンドツールで使用する場合、ereport が利用できないため、エラーハンドリングは errormsg_buf を介して行われる点に注意してください。
  • readBuf の確保には MCXT_ALLOC_NO_OOM が指定されており、メモリ不足時に即座に NULL を返す設計となっています。
