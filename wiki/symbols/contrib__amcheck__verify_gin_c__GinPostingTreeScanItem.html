GinPostingTreeScanItem

定義

  • 種別: struct
  • 定義箇所: contrib/amcheck/verify_gin.c:48

目的

GINインデックスのポスティングツリーを深さ優先探索（DFS）する際、走査中のスタック状態を保持するために使用されます。

シグネチャまたは形状

typedef struct GinPostingTreeScanItem
{
	int			depth;
	ItemPointerData parentkey;
	BlockNumber parentblk;
	BlockNumber blkno;
	struct GinPostingTreeScanItem *next;
} GinPostingTreeScanItem;

入力と出力

本構造体はデータ保持用であり、関数ではないため入出力はありません。各フィールドは以下の通りです。

  • depth: 現在のツリー深さ。
  • parentkey: 親ページにおけるキー情報。
  • parentblk: 親ページのブロック番号。
  • blkno: 現在走査中のブロック番号。
  • next: スタック内の次の要素へのポインタ。

前提条件と不変条件

  • next フィールドにより、単方向連結リストとしてスタックを構成します。
  • palloc0 等でメモリ確保され、走査の進行に伴い動的に生成・解放されます。

アルゴリズムまたは振る舞い

ポスティングツリーの再帰的な検証において、現在位置と親のコンテキストをスタックとして管理します。これにより、ツリー構造を辿る際のバックトラックや親キーとの整合性チェックを可能にします。

呼び出し元と呼び出し先

  • **呼び出し元**: gin_check_posting_tree_parent_keys_consistency 内で、ツリー走査のスタック操作（プッシュ処理）のためにインスタンス化されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • GinScanItem: GINインデックス全体を走査する際のスタック構造体。
  • ItemPointerData: GINインデックスのタプル位置を指すデータ構造。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • 連結リストの終端（next == NULL）は、スタックの底（ルートレベル）を示します。

保守担当者向けメモ

  • 本構造体は contrib/amcheck/verify_gin.c 内でのみ使用されます。
  • 走査ロジックの変更に伴い、スタックに保持すべき情報（parentkey 等）が増減する可能性があるため、変更時は gin_check_posting_tree_parent_keys_consistency のロジックと整合性を確認してください。
