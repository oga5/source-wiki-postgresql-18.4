icu_test

定義

  • 種別: function
  • 定義箇所: src/common/unicode/category_test.c:55

目的

PostgreSQL 内部の Unicode カテゴリ、バイナリプロパティ、および文字クラスの判定ロジックが、ICU ライブラリの定義と整合しているかを全コードポイント（0x000000 - 0x10FFFF）走査により検証する。

シグネチャまたは形状

static void icu_test(void)

入力と出力

  • **入力**: 引数なし。グローバル変数 pg_unicode_version および icu_unicode_version を参照。
  • **出力**: 戻り値なし。不一致が検出された場合は標準エラー出力に詳細を表示し、exit(1) でプロセスを終了する。成功時は検証したコードポイント数を標準出力に表示する。

前提条件と不変条件

  • USE_ICU が定義された環境で実行されること。
  • 全コードポイントの走査において、PostgreSQL と ICU の Unicode バージョン差異による「未割り当て（UNASSIGNED）」状態の不一致は、バージョンが古い側のスキップロジックによって許容される。

アルゴリズムまたは振る舞い

1. 0x000000 から 0x10FFFF までの全コードポイントをループ処理する。 2. 各コードポイントに対し、PostgreSQL の pg_u_* 関数群と ICU の u_* 関数群を呼び出し、結果を取得する。 3. バージョン不一致による「未割り当て」の差異を検知した場合、該当コードポイントをスキップする。 4. カテゴリ、バイナリプロパティ（Alphabetic, Lowercase 等）、および文字クラス（isalpha, isdigit 等）の判定結果を比較する。 5. 比較結果が一致しない場合、該当コードポイントと期待値・実測値を出力して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/unicode/category_test.c 内の main 関数から呼び出される。
  • **呼び出し先**:
  • PostgreSQL 側: unicode_category, pg_u_prop_* 関数群, pg_u_is* 関数群。
  • ICU 側: u_charType, u_hasBinaryProperty, u_isUAlphabetic, u_isULowercase, u_isUUppercase, u_ispunct, u_isdigit, u_isUWhiteSpace, u_isblank, u_isUAlphabetic 等。

関連データ構造

  • pg_wchar: コードポイントを保持する型。
  • PG_U_UNASSIGNED: 未割り当て状態を示す定数。

失敗モードと境界条件

  • **不一致**: カテゴリ、プロパティ、または文字クラスのいずれかで PostgreSQL と ICU の判定が異なると exit(1) で失敗する。
  • **バージョン差異**: バージョンが異なる場合に発生する「未割り当て」の不一致は、pg_unicode_version < icu_unicode_version などの条件に基づきスキップされる。

保守担当者向けメモ

  • テスト失敗時は、PostgreSQL 側のテーブル生成スクリプトまたはプロパティ判定ロジックが最新の Unicode 定義と乖離している可能性が高い。
  • icu_isgraph や icu_isprint 等の判定には u_hasBinaryProperty を使用しており、ICU の仕様変更に追従する必要がある。
