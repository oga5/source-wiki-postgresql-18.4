PGTYPESdecimal_new

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/numeric.c:58

目的

ECPGクライアントサイドにおいて、新しい decimal 構造体を動的に割り当て、ゼロ初期化して提供する。

シグネチャまたは形状

decimal *PGTYPESdecimal_new(void)

入力と出力

  • 入力: なし
  • 出力: 成功時は確保された decimal 構造体へのポインタ、失敗時は NULL を返す。

前提条件と不変条件

  • 戻り値が NULL でない場合、構造体内の全メンバは memset により 0 で初期化されている。
  • 割り当てられたメモリは、呼び出し元で PGTYPESdecimal_free を使用して解放する必要がある。

アルゴリズムまたは振る舞い

1. pgtypes_alloc を呼び出し、sizeof(decimal) 分のメモリを確保する。 2. 確保に失敗した場合は NULL を返して終了する。 3. 確保に成功した場合、memset を用いてメモリ領域を 0 で埋める。 4. 初期化済みのポインタを呼び出し元へ返す。

呼び出し元と呼び出し先

  • 呼び出し先: pgtypes_alloc（メモリ確保）、memset（メモリ初期化）。
  • 呼び出し元:
  • src/interfaces/ecpg/test/expected/compat_informix-dec_test.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test2.c
  • その他、pgtypes_numeric.h を介したアプリケーションコード。

関連データ構造

  • decimal: 数値を保持する構造体。

失敗モードと境界条件

  • メモリ不足により pgtypes_alloc が失敗した場合、関数は NULL を返す。

保守担当者向けメモ

  • 本関数は decimal 型のライフサイクル管理の起点となる。メモリリークを防ぐため、本関数で確保した領域は必ず対応する解放関数で処理すること。
