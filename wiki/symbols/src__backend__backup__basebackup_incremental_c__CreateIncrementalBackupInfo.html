CreateIncrementalBackupInfo

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_incremental.c:151

目的

増分バックアップ処理に必要なメタデータ構造体 IncrementalBackupInfo を初期化し、バックアップマニフェストを解析するためのパーサーコンテキストを設定します。

シグネチャまたは形状

IncrementalBackupInfo *CreateIncrementalBackupInfo(MemoryContext mcxt)

入力と出力

  • **入力**: mcxt (メモリコンテキスト)
  • **出力**: 初期化済みの IncrementalBackupInfo 構造体へのポインタ。

前提条件と不変条件

  • 指定された mcxt 上でメモリ確保が行われます。
  • 内部で backup_file_create を呼び出し、10,000ファイル分の初期容量を確保します。
  • JsonManifestParseContext を初期化し、マニフェスト解析用のコールバック関数（バージョン、システム識別子、ファイル、WAL範囲、エラー報告）を登録します。

アルゴリズムまたは振る舞い

1. 指定された mcxt にメモリコンテキストを切り替えます。 2. IncrementalBackupInfo 構造体をゼロクリアして確保し、バッファを初期化します。 3. マニフェスト内のファイル情報を保持するためのハッシュテーブルを backup_file_create で作成します。 4. JsonManifestParseContext を作成し、マニフェスト解析用の各コールバック関数を割り当てます。 5. json_parse_manifest_incremental_init を呼び出し、増分バックアップ用の解析状態を生成して構造体に格納します。 6. 元のメモリコンテキストに戻し、作成した構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walsender.c:687
  • **呼び出し先**:
  • palloc0, MemoryContextSwitchTo (PostgreSQLメモリ管理)
  • initStringInfo
  • backup_file_create
  • json_parse_manifest_incremental_init
  • 各種コールバック関数 (manifest_process_version, manifest_process_system_identifier, manifest_process_file, manifest_process_wal_range, manifest_report_error)

関連データ構造

  • IncrementalBackupInfo: 増分バックアップのメタデータを保持する主要構造体。
  • JsonManifestParseContext: JSONマニフェスト解析のためのコンテキスト。

失敗モードと境界条件

  • メモリ不足時には palloc0 等によりエラーが発生します。
  • マニフェスト解析の初期化に失敗した場合、json_parse_manifest_incremental_init がエラーを投げる可能性があります。

保守担当者向けメモ

  • 本関数は増分バックアップの準備フェーズの起点です。
  • backup_file_create に渡している初期容量 10000 は、標準的なインストール環境を想定したヒューリスティックな値です。大規模環境でのパフォーマンス特性については注意が必要です。
