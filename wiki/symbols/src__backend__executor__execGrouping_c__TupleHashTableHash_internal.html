TupleHashTableHash_internal

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execGrouping.c:420

目的

TupleHashTable におけるタプルのハッシュ値を計算します。入力タプルまたはテーブル内の既存タプルに基づき、式評価を通じてハッシュキーを生成し、最終的に murmurhash32 を適用してハッシュ値を算出します。

シグネチャまたは形状

static uint32 TupleHashTableHash_internal(struct tuplehash_hash *tb, const MinimalTuple tuple);

入力と出力

  • **入力**:
  • tb: simplehash.h で定義されるハッシュテーブル構造体へのポインタ。private_data 経由で TupleHashTable を参照します。
  • tuple: ハッシュ化対象の MinimalTuple。NULL の場合は入力スロットのタプルが使用されます。
  • **出力**:
  • uint32: 計算されたハッシュ値。

前提条件と不変条件

  • tb->private_data は有効な TupleHashTable 構造体を指している必要があります。
  • hashtable->exprcontext が適切に初期化されており、式評価に使用可能である必要があります。

アルゴリズムまたは振る舞い

1. **タプルの特定**:

  • tuple が NULL の場合: hashtable->inputslot を評価対象として設定し、hashtable->in_hash_expr を評価します。
  • tuple が存在する場合: hashtable->tableslot にタプルを格納し、hashtable->tab_hash_expr を評価します。

    • 2. **ハッシュ計算**:

  • ExecEvalExpr を用いて式から Datum を取得し、DatumGetUInt32 で uint32 に変換します。

    • 3. **ハッシュ摂動**:

  • 得られたハッシュキーに対し murmurhash32 を適用し、ハッシュ値の分散を向上させて返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execGrouping.c 内の SH_HASH_KEY マクロ（simplehash.h 経由）。
  • src/backend/executor/execGrouping.c 内のハッシュテーブル操作処理（LookupTupleHashEntry 等の内部ロジック）。
  • **呼び出し先**:
  • ExecEvalExpr: 式の評価に使用。
  • ExecStoreMinimalTuple: 既存タプルのスロット格納に使用。
  • murmurhash32: ハッシュ値の最終計算に使用。

関連データ構造

  • TupleHashTable: 本関数が操作するハッシュテーブルの管理構造体。
  • MinimalTuple: ハッシュ化対象のタプル形式。
  • tuplehash_hash: simplehash.h が生成するハッシュテーブルの制御構造体。

失敗モードと境界条件

  • tuple が NULL かつ hashtable->inputslot が未設定の場合、式評価時に予期せぬ動作を引き起こす可能性があります。
  • simplehash.h の仕様上、テーブル内の既存タプルに対するハッシュ計算は、ハッシュ値がエントリ内に保持されているため、通常は実行されません。

保守担当者向けメモ

  • 本関数は simplehash.h の SH_HASH_KEY マクロとして登録されています。
  • murmurhash32 による摂動は、ハッシュの衝突を避けるために重要です。変更時はハッシュの分布特性に注意してください。
