do_lo_import

定義

  • 種別: function
  • 定義箇所: src/bin/psql/large_obj.c:175

目的

指定されたファイルをデータベース内のラージオブジェクトとしてインポートし、必要に応じてコメントを付与します。

シグネチャまたは形状

bool do_lo_import(const char *filename_arg, const char *comment_arg)

入力と出力

  • **入力**:
  • filename_arg: インポート対象のファイルパス。
  • comment_arg: ラージオブジェクトに付与するコメント（NULL の場合はコメントなし）。
  • **出力**:
  • 成功時に true、失敗時に false を返します。
  • 成功時、生成された OID が pset.vars の LASTOID 変数に格納されます。

前提条件と不変条件

  • pset.db が有効なデータベース接続を保持していること。
  • トランザクション管理のため、内部で start_lo_xact を呼び出し、操作終了時に finish_lo_xact または fail_lo_xact を呼び出す必要があります。

アルゴリズムまたは振る舞い

1. start_lo_xact を呼び出し、トランザクション状態を準備します。 2. SetCancelConn で割り込みを許可し、libpq の lo_import を実行してファイルをインポートします。 3. loid が InvalidOid の場合、エラーログを出力し fail_lo_xact を実行して終了します。 4. comment_arg が指定されている場合、COMMENT ON LARGE OBJECT SQL を構築し、PSQLexec を介して実行します。 5. finish_lo_xact でトランザクションを確定します。 6. print_lo_result で結果を表示し、LASTOID 変数を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c:2407
  • **呼び出し先**:
  • start_lo_xact, finish_lo_xact, fail_lo_xact
  • SetCancelConn, ResetCancelConn
  • lo_import (libpq)
  • PSQLexec, PQerrorMessage, PQclear, PQescapeStringConn
  • pg_log_info, pg_malloc_extended, SetVariable

関連データ構造

  • pset: 接続情報や変数管理を行うグローバル構造体。
  • PGresult: SQL 実行結果を保持する libpq の構造体。

失敗モードと境界条件

  • ファイルの読み込み失敗やデータベースエラー発生時は fail_lo_xact を経由してロールバックが行われます。
  • pg_malloc_extended によるメモリ確保失敗時も同様に失敗として処理されます。

保守担当者向けメモ

  • 本関数は LASTOID 変数を更新するため、スクリプト実行時の後続処理に影響を与えます。
  • comment_arg のエスケープには PQescapeStringConn を使用しており、SQL インジェクション対策が施されています。
