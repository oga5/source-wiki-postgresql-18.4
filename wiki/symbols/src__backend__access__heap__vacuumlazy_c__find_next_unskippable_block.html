find_next_unskippable_block

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/vacuumlazy.c:1676

目的

VACUUM処理において、可視性マップ（VM）を参照し、次にスキャンが必要な（スキップ不可能な）ヒープブロックを特定する。

シグネチャまたは形状

static void find_next_unskippable_block(LVRelState *vacrel, bool *skipsallvis)

入力と出力

  • **入力**:
  • vacrel: VACUUM実行状態を保持する LVRelState 構造体。
  • **出力**:
  • skipsallvis: 呼び出し元がスキップ可能な全可視ページをスキップしたかどうかを示すフラグ。
  • vacrel 内部の next_unskippable_block 等のフィールドが更新される。

前提条件と不変条件

  • vacrel->rel は有効なリレーションであること。
  • next_unskippable_vmbuffer は、VMの読み込みに使用されるバッファとして維持される。

アルゴリズムまたは振る舞い

1. next_unskippable_block から順に、可視性マップのビット状態を確認する。 2. 各ブロックにおいて、以下の条件でスキップ可否を判定する：

  • **スキップ不可**: 全可視でないブロック、またはテーブルの最終ブロック（タプル有無確認のため）。
  • **スキップ可**: 全凍結（All-Frozen）されているブロック。
  • **条件付きスキップ**:
  • aggressive モードでない場合、全可視かつ全凍結でないブロックは、eager_scan_remaining_fails が残っていればスキップ可能（Eager Scan）。
  • skipwithvm が無効な場合はスキップ不可。

    • 3. 判定結果を vacrel に書き戻し、次のスキャン開始位置を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/vacuumlazy.c 内の lazy_scan_heap（行1607付近）。
  • **呼び出し先**: visibilitymap_get_status。

関連データ構造

  • LVRelState: VACUUMの進捗と状態を管理する主要構造体。

失敗モードと境界条件

  • **最終ブロック**: rel_pages - 1 は、タプルの有無を確認するため常にスキップ不可として扱われる。
  • **Eager Scan制限**: eager_scan_remaining_fails が枯渇すると、全可視ブロックのスキップが制限される。
  • **Aggressive Vacuum**: aggressive モード時は、全可視かつ全凍結でないブロックのスキップは許可されない。

保守担当者向けメモ

  • 本関数は lazy_scan_heap のループ内で繰り返し呼び出され、VMのビット状態に基づいてスキャン対象を決定する。
  • next_eager_scan_region_start を用いた領域ごとの失敗カウンタのリセットロジックが含まれているため、Eager Scanの挙動を変更する際は注意が必要。
