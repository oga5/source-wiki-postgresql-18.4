SERIALIZABLEXIDTAG

定義

  • 種別: struct
  • 定義箇所: src/include/storage/predicate_internals.h:222

目的

直列化可能分離レベル（SSI）において、特定のトランザクションID（XID）を識別するためのタグ構造体です。

シグネチャまたは形状

typedef struct SERIALIZABLEXIDTAG
{
	TransactionId xid;
} SERIALIZABLEXIDTAG;

入力と出力

  • 入力: TransactionId 型のトランザクション識別子。
  • 出力: なし（構造体として値を保持）。

前提条件と不変条件

  • 直列化可能トランザクションに関連付けられた有効な TransactionId を保持する必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、主にハッシュテーブルのキーとして使用されます。
  • src/backend/storage/lmgr/predicate.c 内の各関数において、トランザクションの状態検索や管理を行う際の識別子としてインスタンス化されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/lmgr/predicate.c 内の複数の箇所で、トランザクション管理処理のキーとして生成・参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • SERIALIZABLEXID: 本タグを使用して管理される、より広範なトランザクション情報構造体。

失敗モードと境界条件

  • TransactionId が無効な場合、または初期化されていない状態での参照は、SSIサブシステムにおけるトランザクション追跡の不整合を招く可能性があります。

保守担当者向けメモ

  • 本構造体は SERIALIZABLEXID のキーとして機能するため、定義を変更する際はハッシュテーブルのキーサイズやアライメントに注意してください。
  • predicate.c におけるハッシュテーブル操作（info.keysize = sizeof(SERIALIZABLEXIDTAG)）と整合性を保つ必要があります。
