UserMapping

定義

  • 種別: struct
  • 定義箇所: src/include/foreign/foreign.h:45

目的

PostgreSQLの外部データラッパー（FDW）において、ローカルのデータベースユーザーと外部サーバー上のユーザー認証情報を紐付けるためのメタデータを保持する構造体です。

シグネチャまたは形状

typedef struct UserMapping
{
	Oid			umid;			/* Oid of user mapping */
	Oid			userid;			/* local user Oid */
	Oid			serverid;		/* server Oid */
	List	   *options;		/* useoptions as DefElem list */
} UserMapping;

入力と出力

本構造体は、システムカタログから情報を取得する関数（foreign.cで実装）を通じてインスタンス化されます。

  • **入力**: システムキャッシュ内の pg_user_mapping カタログ情報。
  • **出力**: 外部サーバーへの接続や認証パラメータの決定に使用される構造体インスタンス。

前提条件と不変条件

  • options フィールドは DefElem のリストとして保持され、各FDWの実装側で解析される必要があります。
  • システムキャッシュからコピーして作成されるため、使用後はメモリ管理（pfree等）を考慮する必要があります。

アルゴリズムまたは振る舞い

1. 呼び出し元が GetForeign... 系の関数を呼び出す。 2. src/backend/foreign/foreign.c がシステムカタログから HeapTuple を取得。 3. GETSTRUCT および SysCacheGetAttr を用いて、カタログ上の属性値を本構造体の各フィールドにマッピングする。 4. 構築された構造体が呼び出し元に返却される。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • contrib/dblink/dblink.c: UseScramPassthrough 等で使用。
  • contrib/postgres_fdw/connection.c: make_new_connection, connect_pg_server, check_conn_params, UserMappingPasswordRequired, UseScramPassthrough, GetConnection, pgfdw_security_check 等で使用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、特定の関数を呼び出すことはありません。

関連データ構造

  • List: options フィールドの保持に使用（nodes/parsenodes.h に依存）。
  • Oid: ユーザーおよびサーバーの識別子。

失敗モードと境界条件

  • userid が InvalidOid の場合、MappingUserName マクロにより "public" として扱われる設計になっています。
  • システムカタログ上に該当するマッピングが存在しない場合、GetForeign...Extended 関数のフラグ設定に応じて、エラーを投げるかNULLを返すかの挙動が分岐します。

保守担当者向けメモ

  • 本構造体はシステムキャッシュのコピーであるため、永続的な変更を反映させる場合はカタログ更新関数を別途呼び出す必要があります。
  • options は DefElem のリストであるため、FDW実装側での解析時に型安全性を確保してください。
