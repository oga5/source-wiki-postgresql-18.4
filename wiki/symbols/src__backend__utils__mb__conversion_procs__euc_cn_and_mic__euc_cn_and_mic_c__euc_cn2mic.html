euc_cn2mic

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_cn_and_mic/euc_cn_and_mic.c:78

目的

EUC_CN形式の文字列を、PostgreSQL内部表現であるMULE_INTERNAL形式（LC_GB2312_80エスケープ付き）へ変換します。

シグネチャまたは形状

static int euc_cn2mic(const unsigned char *euc, unsigned char *p, int len, bool noError);

入力と出力

  • **入力**:
  • euc: 変換対象のEUC_CN文字列（unsigned char *）。
  • p: 変換結果を格納するバッファ。
  • len: 変換対象のバイト長。
  • noError: true の場合、不正なシーケンス時にエラーを投げず処理を中断する。
  • **出力**:
  • 戻り値: 処理したバイト数（int）。

前提条件と不変条件

  • p は変換結果を格納するのに十分な領域を確保している必要があります。
  • 変換後の文字列は \0 で終端されます。

アルゴリズムまたは振る舞い

1. 入力文字列をバイト単位で走査します。 2. 高ビットが立っている場合（マルチバイト文字）:

  • 後続バイトが存在し、かつ高ビットが立っていることを確認します。
  • LC_GB2312_80 を先頭に付与し、続く2バイトをコピーして3バイトのMULE_INTERNAL形式に変換します。

    • 3. 高ビットが立っていない場合（ASCII）:

  • そのままコピーします。ただし、0（ヌルバイト）は不正な文字として扱われます。

    • 4. noError が true の場合、不正なシーケンスやヌルバイトに遭遇した時点でループを終了します。 5. 処理した総バイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: euc_cn_to_mic（src/backend/utils/mb/conversion_procs/euc_cn_and_mic/euc_cn_and_mic.c:54）。
  • **呼び出し先**: report_invalid_encoding（エラー発生時）。

関連データ構造

  • LC_GB2312_80: MULE_INTERNAL表現におけるGB2312文字セットの識別子。

失敗モードと境界条件

  • **不正なマルチバイトシーケンス**: 2バイト目が存在しない、または2バイト目の高ビットが立っていない場合、noError が false なら report_invalid_encoding を呼び出します。
  • **ヌルバイト**: 入力中に 0 が含まれる場合、noError が false ならエラーとして報告されます。

保守担当者向けメモ

  • 本関数は静的関数であり、モジュール内の変換ロジックの中核を担います。
  • noError フラグの扱いは、変換処理の堅牢性に直結するため、変更時は呼び出し元の期待する挙動と整合性をとる必要があります。
