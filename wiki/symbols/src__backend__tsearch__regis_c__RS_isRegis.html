RS_isRegis

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/tsearch/regis.c:30

目的

入力文字列が、PostgreSQL の ISpell 辞書機能でサポートされる限定的な正規表現サブセット（[abc] や [^abc] 等）に適合するかを検証します。

シグネチャまたは形状

bool RS_isRegis(const char *str)

入力と出力

  • **入力**: str (検証対象のヌル終端文字列)
  • **出力**: 文字列がサポート対象の形式であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力文字列はマルチバイト文字を考慮して走査されます。
  • 内部状態は RS_IN_WAIT（待機状態）、RS_IN_ONEOF（[ 検出後）、RS_IN_ONEOF_IN（文字集合内）、RS_IN_NONEOF（[^ 検出後）のいずれかで遷移します。

アルゴリズムまたは振る舞い

1. 文字列を先頭から順に走査します。 2. RS_IN_WAIT 状態では、アルファベット文字を許容し、[ が現れると RS_IN_ONEOF へ遷移します。 3. RS_IN_ONEOF 状態では、^ であれば RS_IN_NONEOF へ、アルファベットであれば RS_IN_ONEOF_IN へ遷移します。 4. RS_IN_ONEOF_IN または RS_IN_NONEOF 状態では、アルファベットを許容し、] が現れると RS_IN_WAIT へ戻ります。 5. 許可されていない文字や構文が現れた場合は直ちに false を返します。 6. 文字列の終端に達した際、状態が RS_IN_WAIT であれば true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/spell.c (708行目) にて、辞書マスクの検証に使用されています。
  • **呼び出し先**: t_isalpha_cstr, t_iseq, pg_mblen_cstr などのユーティリティ関数を呼び出します。

関連データ構造

  • 本関数は RegisNode 構造体（src/include/tsearch/dicts/regis.h で定義）の構築前段階の検証として機能します。

失敗モードと境界条件

  • 構文エラー（例: [ で始まり ] で閉じない、[ の直後に ] が来る等）は false を返します。
  • 内部状態が定義外の値になった場合、elog(ERROR) により異常終了します。

保守担当者向けメモ

  • 本関数は汎用的な正規表現エンジンではありません。サポート対象の構文を拡張する場合は、状態遷移ロジックの修正が必要です。
  • マルチバイト文字の扱いは pg_mblen_cstr に依存しているため、ロケール設定の影響を受ける可能性があります。
