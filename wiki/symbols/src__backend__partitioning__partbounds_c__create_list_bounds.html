create_list_bounds

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partbounds.c:461

目的

PartitionBoundSpec のリストから、LISTパーティション用の実行時情報構造体 PartitionBoundInfo を生成します。

シグネチャまたは形状

static PartitionBoundInfo create_list_bounds(PartitionBoundSpec **boundspecs, int nparts, PartitionKey key, int **mapping)

入力と出力

  • **入力**:
  • boundspecs: パーティション境界定義の配列。
  • nparts: パーティション数。
  • key: パーティションキー情報。
  • mapping: 元のパーティションインデックスから正規化されたインデックスへのマッピング配列へのポインタ。
  • **出力**:
  • 構築された PartitionBoundInfo 構造体。

前提条件と不変条件

  • boundspecs 内のすべての要素の戦略が PARTITION_STRATEGY_LIST であること。
  • mapping 配列は呼び出し元で適切に初期化されていること。
  • null_index が複数存在する場合、エラーを発生させる。

アルゴリズムまたは振る舞い

1. PartitionBoundInfoData を割り当て、初期化します。 2. すべてのパーティションから非NULL値を収集し、all_values 配列に格納します。 3. qsort_arg を使用して、キーに基づいて all_values をソートします。 4. ソートされた値に基づき、boundinfo->datums および boundinfo->indexes を構築します。 5. mapping を更新し、各パーティションに正規化されたインデックスを割り当てます。 6. null_index および default_index が存在する場合、それらにも正規化インデックスを割り当てます。 7. パーティションがインターリーブ（交差）しているかを確認し、interleaved_parts ビットマップを構築します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/partitioning/partbounds.c 内の関数（partition_bounds_create 等から参照されていることが確認されています）。
  • **呼び出し先**: get_non_null_list_datum_count, qsort_arg, datumCopy, bms_add_member 等。

関連データ構造

  • PartitionBoundInfo: 生成される主要な構造体。
  • PartitionListValue: 値とパーティションインデックスを保持する一時構造体。
  • PartitionBoundSpec: 入力となる境界定義。

失敗モードと境界条件

  • spec->strategy が PARTITION_STRATEGY_LIST でない場合、elog(ERROR) を発生させます。
  • NULL値が複数のパーティションで定義されている場合、elog(ERROR) を発生させます。

保守担当者向けメモ

  • indexes 配列はソートされた Datum の順序に対応しており、この順序が崩れている場合に interleaved_parts が設定されます。
  • mapping 配列の管理は、パーティションの正規化において重要な役割を果たします。
