sort_snapshot

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xid8funcs.c:172

目的

pg_snapshot 構造体内のトランザクションID配列（xip）をソートし、重複を排除することで、スナップショットの正規化を行う。

シグネチャまたは形状

static void sort_snapshot(pg_snapshot *snap)

入力と出力

  • **入力**: pg_snapshot *snap - ソートおよび重複排除対象となるスナップショット構造体へのポインタ。
  • **出力**: なし（引数の snap を直接書き換える）。

前提条件と不変条件

  • snap->xip は FullTransactionId 型の配列であること。
  • snap->nxip は配列内の有効な要素数を示すこと。
  • cmp_fxid 関数が FullTransactionId の比較に利用可能であること。

アルゴリズムまたは振る舞い

1. snap->nxip が 1 より大きい場合のみ処理を実行する。 2. qsort を使用し、cmp_fxid を比較関数として snap->xip を昇順にソートする。 3. qunique を使用し、ソート済み配列から重複する FullTransactionId を排除する。 4. 排除後の新しい要素数を snap->nxip に再設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/xid8funcs.c 内の複数箇所（pg_current_snapshot 等の生成プロセス）。
  • **呼び出し先**:
  • qsort (標準ライブラリ)
  • cmp_fxid (比較関数)
  • qunique (重複排除関数)

関連データ構造

  • pg_snapshot: トランザクションIDの配列（xip）と要素数（nxip）を保持する構造体。
  • FullTransactionId: 64ビットのトランザクションID。

失敗モードと境界条件

  • nxip が 0 または 1 の場合、ソートや重複排除はスキップされ、副作用は発生しない。
  • メモリ不足等のシステムエラーが発生した場合の挙動は、標準ライブラリの qsort に依存する。

保守担当者向けメモ

  • 本関数は pg_snapshot の整合性を保つためのユーティリティである。
  • cmp_fxid の実装が変更された場合、本関数のソート順序および重複判定ロジックも影響を受けるため注意すること。
