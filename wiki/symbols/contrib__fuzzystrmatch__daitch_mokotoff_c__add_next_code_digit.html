add_next_code_digit

定義

  • 種別: function
  • 定義箇所: contrib/fuzzystrmatch/daitch_mokotoff.c:181

目的

Daitch-Mokotoff Soundex アルゴリズムの処理過程において、特定のノード（dm_node）に対して、次に続く音韻コードの桁（digit）を記録・更新します。

シグネチャまたは形状

static void add_next_code_digit(dm_node *node, int code_index, char code_digit)

入力と出力

  • **入力**:
  • node: 更新対象の dm_node 構造体へのポインタ。
  • code_index: next_code_index にビット単位で論理和をとるためのインデックス値（1 または 2）。
  • code_digit: 記録する音韻コードの文字。
  • **出力**: なし（副作用として node の内容を直接変更）。

前提条件と不変条件

  • node は有効な dm_node 構造体を指している必要があります。
  • next_code_digits 配列は最大で2つの異なるコードを保持可能です。

アルゴリズムまたは振る舞い

1. node->next_code_index に対して、引数 code_index をビット論理和（OR）で結合します。 2. node->next_code_digits[0] が空（null）であれば、そこに code_digit を格納します。 3. node->next_code_digits[0] が既に存在し、かつその値が code_digit と異なる場合、node->next_code_digits[1] に code_digit を格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/fuzzystrmatch/daitch_mokotoff.c:312 において、dirty_nodes 配列内のノードを更新する際に呼び出されています。
  • **呼び出し先**: 本関数内から他の関数を呼び出すことはありません。

関連データ構造

  • dm_node: 音韻コード生成のツリー構造を保持する構造体。本関数はこの構造体の next_code_index および next_code_digits フィールドを操作します。

失敗モードと境界条件

  • next_code_digits に既に2つの異なる値が格納されている場合、3つ目の値が渡されても本関数内では無視されます（上書きやエラー報告は行われません）。

保守担当者向けメモ

  • 本関数は dm_node の状態遷移を管理する低レベルなユーティリティです。音韻ルールの分岐が複雑化する際、next_code_digits の容量制限（2桁）がアルゴリズムの制約となっている点に注意してください。
