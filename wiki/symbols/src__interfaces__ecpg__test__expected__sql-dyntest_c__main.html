目的

ECPG（Embedded SQL for C）における動的SQLの機能検証を行う回帰テストプログラムのエントリポイントです。SQL記述子を使用してクエリ結果のメタデータを取得し、標準出力へ表示する一連の処理を実行します。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム実行時にデータベース ecpg1_regression へ接続）
  • 出力: 標準出力（取得した列のメタデータおよび型情報）、標準エラー（デバッグログ）

前提条件と不変条件

  • ECPGdebug(1, stderr) によりデバッグログが有効化されていること。
  • データベース接続およびテーブル dyntest の作成・データ挿入が成功していること。
  • MYDESC という名称のSQL記述子が正常に割り当てられていること。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGconnect でデータベースに接続し、テスト用テーブルの作成とデータの挿入を行います。 2. **準備**: ECPGprepare でクエリを準備し、DECLARE CURSOR でカーソルを定義します。 3. **フェッチとメタデータ取得**:

  • FETCH を実行し、結果を MYDESC 記述子に格納します。
  • ECPGget_desc_header で列数を取得します。
  • 各列に対して ECPGget_desc を呼び出し、列名、型、精度、スケール等のメタデータを抽出します。

    • 4. **表示**: 抽出した TYPE に基づき、switch 文で型（SQL3_BOOLEAN, SQL3_NUMERIC 等）を判定し、整形して標準出力へ出力します。 5. **終了**: カーソルを閉じ、記述子を解放して終了します。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先: ECPGdebug, ECPGallocate_desc, ECPGconnect, ECPGdo, ECPGprepare, ECPGget_desc_header, ECPGget_desc, error, printf

関連データ構造

  • sqlca: SQL通信領域（エラーチェックに使用）。
  • MYDESC: 動的SQL用の記述子。
  • SQL3_... 系定数: SQL標準の動的データ型コード。

失敗モードと境界条件

  • 各SQL操作後に sqlca.sqlcode < 0 をチェックし、エラー発生時には error() 関数を呼び出して終了します。
  • FETCH 実行時に sqlca.sqlcode が非ゼロ（データ終了など）になった場合、ループを抜けて処理を終了します。
