WalRcvDie

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walreceiver.c:780

目的

walreceiver プロセスの終了処理を実行し、共有メモリ状態のクリーンアップとプライマリサーバーとの接続切断を行う。

シグネチャまたは形状

static void WalRcvDie(int code, Datum arg)
  • code: 終了コード（on_shmem_exit の規約に従う）。
  • arg: TimeLineID へのポインタを保持する Datum 型の引数。

入力と出力

  • **入力**: arg を介して渡される TimeLineID。共有メモリ上の WalRcv 構造体。
  • **出力**: なし（副作用として共有メモリの更新とプロセス終了処理を行う）。

前提条件と不変条件

  • *startpointTLI_p が 0 でないこと。
  • 終了時に walrcv->pid が MyProcPid と一致していること。
  • 終了時に walrcv->walRcvState が WALRCV_STREAMING, RESTARTING, STARTING, WAITING, STOPPING のいずれかであること。

アルゴリズムまたは振る舞い

1. XLogWalRcvFlush を呼び出し、受信済みの全 WAL レコードをディスクへフラッシュする。 2. 共有メモリ上の WalRcv 構造体のスピンロックを取得し、状態を WALRCV_STOPPED に更新、pid を 0 にリセットする。 3. walRcvStoppedCV をブロードキャストし、待機中のプロセスに通知する。 4. wrconn が有効な場合、walrcv_disconnect を呼び出してプライマリとの接続を終了する。 5. WakeupRecovery を呼び出し、スタートアッププロセスへ終了を通知する。

呼び出し元と呼び出し先

  • **呼び出し元**: on_shmem_exit に登録された終了ハンドラとして呼び出される（src/backend/replication/walreceiver.c:246）。
  • **呼び出し先**:
  • XLogWalRcvFlush
  • SpinLockAcquire / SpinLockRelease
  • ConditionVariableBroadcast
  • walrcv_disconnect
  • WakeupRecovery

関連データ構造

  • WalRcvData (共有メモリ上の WalRcv)
  • TimeLineID

失敗モードと境界条件

  • wrconn が NULL の場合、接続切断処理はスキップされる。
  • 共有メモリのロック取得に失敗した場合、または前提条件の Assert に違反した場合はプロセスが異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は on_shmem_exit を通じて呼び出されるため、プロセスの終了シーケンスにおいて確実に実行されることが保証されている。
  • 終了処理の順序（フラッシュ → 状態更新 → 通知 → 切断）は、リカバリプロセスとの整合性を保つために重要である。
