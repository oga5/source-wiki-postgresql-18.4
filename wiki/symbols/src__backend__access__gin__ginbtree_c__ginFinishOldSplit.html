ginFinishOldSplit

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginbtree.c:778

目的

GINインデックスのページ操作中に遭遇した「不完全な分割（Incomplete Split）」状態を検知し、修復を完了させるために使用されます。

シグネチャまたは形状

static void ginFinishOldSplit(GinBtree btree, GinBtreeStack *stack, GinStatsData *buildStats, int access)

入力と出力

  • **入力**:
  • btree: 操作対象のGIN B-tree構造体。
  • stack: 不完全な分割状態にあるページを指すスタック要素。
  • buildStats: インデックス構築統計情報（任意）。
  • access: 現在のロックモード（GIN_SHARE または GIN_EXCLUSIVE）。
  • **出力**: なし（副作用としてページ状態を更新）。

前提条件と不変条件

  • 対象ページが GIN_INCOMPLETE_SPLIT フラグを保持していることが前提です。
  • GIN_SHARE モードで呼び出された場合、内部で排他ロックへの昇格が行われます。

アルゴリズムまたは振る舞い

1. インジェクションポイント gin-finish-incomplete-split を実行します。 2. DEBUG1 レベルでログを出力します。 3. access が GIN_SHARE の場合、一度ロックを解放し、排他ロック（GIN_EXCLUSIVE）を再取得します。 4. ロック再取得後、再度 GinPageIsIncompleteSplit を確認し、他プロセスによって既に修復済みであれば処理を終了します。 5. ginFinishSplit を呼び出し、親ページへのダウンリンク挿入等の分割完了処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginbtree.c 内の各所（ginFindLeafPage の降下中や、ページ操作の前後など、不完全な分割を検知した箇所から呼び出されます）。
  • **呼び出し先**: ginFinishSplit。

関連データ構造

  • GinBtree: GIN B-treeの操作状態を保持する構造体。
  • GinBtreeStack: ツリー降下時のパス情報とバッファを保持する構造体。

失敗モードと境界条件

  • 排他ロック取得後の再チェックにより、並行処理による競合を回避します。
  • ページが既に修復されている場合は、何もせず正常終了します。

保守担当者向けメモ

  • 本関数は、GINインデックスの分割処理がアトミックではないことに起因する整合性維持のための重要なルーチンです。
  • ginFinishSplit を呼び出す前に必ず排他ロックを確保する設計となっています。
