range_get_flags

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes.c:1986

目的

RangeType 構造体から範囲型のフラグバイトを抽出します。

シグネチャまたは形状

char range_get_flags(const RangeType *range)

入力と出力

  • **入力**: range (範囲型データのポインタ)
  • **出力**: 範囲型のフラグ情報を含む char 型の値。

前提条件と不変条件

  • range は有効な RangeType（varlena 構造体）を指している必要があります。
  • 内部的に VARSIZE(range) マクロを使用してデータサイズを計算するため、対象はデトースト（detoast）済みである必要があります。

アルゴリズムまたは振る舞い

1. VARSIZE(range) マクロを用いて、対象の RangeType 全体のサイズを取得します。 2. 範囲型データの末尾（range の先頭アドレス + サイズ - 1）に格納されているフラグバイトを直接参照し、その値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/rangetypes.c 内の複数箇所（範囲の比較や演算処理など）。
  • src/backend/utils/adt/rangetypes_gist.c:790（GiST インデックスの比較処理）。
  • **呼び出し先**:
  • VARSIZE マクロ（PostgreSQL の varlena 処理用マクロ）。

関連データ構造

  • RangeType: 範囲型を表現する可変長データ構造。

失敗モードと境界条件

  • range が NULL または不正なメモリ領域を指している場合、セグメンテーション違反が発生します。
  • 範囲型として正しく初期化されていないデータに対して呼び出した場合、無効なフラグ値が返されます。

保守担当者向けメモ

  • 本関数はフラグバイトを直接読み取る軽量なアクセサです。フラグのビット定義（空か否か、境界の包含関係など）については rangetypes.h を参照してください。
