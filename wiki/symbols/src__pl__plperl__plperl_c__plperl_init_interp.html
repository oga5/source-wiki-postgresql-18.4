plperl_init_interp

定義

  • **種別**: function
  • **定義箇所**: src/pl/plperl/plperl.c:706

目的

PostgreSQL のプロセス内で Perl インタプリタを新規に割り当て、基本初期化（ブートストラップ）を実行する。

シグネチャまたは形状

static PerlInterpreter *plperl_init_interp(void)

入力と出力

  • **入力**: なし（GUC 変数 plperl_on_init を内部的に参照）
  • **出力**: 初期化済みの PerlInterpreter * 構造体へのポインタ。失敗時は elog(ERROR) によりプロセスが中断される。

前提条件と不変条件

  • PERL_SYS_INIT3 が利用可能な環境では、初回呼び出し時に一度だけ実行される。
  • Windows 環境では、Perl の起動によるロケール破壊を防ぐため、呼び出し前後のロケール退避・復元が必須である。
  • 呼び出し後、PL_exit_flags に PERL_EXIT_DESTRUCT_END が設定され、END ブロックの実行タイミングが制御される。

アルゴリズムまたは振る舞い

1. **ロケール保護 (Windowsのみ)**: 現在のロケール設定を保存する。 2. **システム初期化**: PERL_SYS_INIT3 を呼び出し、SIGFPE ハンドラを PostgreSQL の標準設定に再設定する。 3. **メモリ割り当て**: perl_alloc() でインタプリタを確保し、perl_construct() で構築する。 4. **コンテキスト設定**: PERL_SET_CONTEXT で現在のスレッドコンテキストを更新する。 5. **オペコードマスク**: PLPERL_ENABLE_OPMASK_EARLY が定義されている場合、plperl_opmask を適用する。 6. **解析と実行**: perl_parse() で PLC_PERLBOOT および plperl_on_init を読み込み、perl_run() で初期化コードを実行する。 7. **ロケール復元 (Windowsのみ)**: 保存していたロケール設定を戻す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plperl/plperl.c:485 (plperl_held_interp の初期化)
  • src/pl/plperl/plperl.c:636 (select_perl_context 内でのインタプリタ生成)
  • **呼び出し先**: perl_alloc, perl_construct, perl_parse, perl_run, pqsignal, setlocale (プラットフォーム依存)

関連データ構造

  • PerlInterpreter: Perl の実行状態を保持する構造体。
  • plperl_opmask: セキュリティ制限のためのオペコードマスク配列。

失敗モードと境界条件

  • perl_alloc が失敗した場合、elog(ERROR) を送出する。
  • perl_parse または perl_run が 0 以外の値を返した場合、ERRSV からエラーメッセージを取得し、ereport(ERROR) を送出する。
  • マルチインタプリタ非対応の Perl ビルド環境では、複数インタプリタの生成時にエラーが発生する可能性がある。

保守担当者向けメモ

  • PERL_SYS_INIT3 呼び出し後の SIGFPE ハンドラ復元は、Perl 側の不適切な挙動に対する回避策であるため、将来の Perl バージョンアップ時にも注意が必要。
  • pp_require_orig を用いて require / dofile オペコードを保護・復元するロジックが含まれている。
