disassembledLeaf

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gin/gindatapage.c:48

目的

GINポスティングツリーのリーフページに格納された圧縮済みデータを、メモリ上で操作・再構築するために分解した中間表現を提供します。

シグネチャまたは形状

typedef struct {
    dlist_head  segments;       /* leafSegmentInfosのリスト */
    dlist_node *lastleft;       /* ページ分割時の左ページ末尾セグメント */
    int         lsize;          /* 左ページの合計サイズ */
    int         rsize;          /* 右ページの合計サイズ */
    bool        oldformat;      /* 9.4以前のディスクフォーマットか否か */
    void       *walinfo;        /* 再構築されたリーフページのWALデータ */
    int         walinfolen;     /* WALデータの長さ */
} disassembledLeaf;

入力と出力

  • **入力**: disassembleLeaf 関数により、物理的なリーフページから読み込まれた圧縮データが本構造体に展開されます。
  • **出力**: computeLeafRecompressWALData により、ページ変更を再現するためのWALデータが walinfo に格納されます。

前提条件と不変条件

  • segments フィールドは leafSegmentInfo 構造体のリストを保持します。
  • ページ分割が必要な場合、leafRepackItems によって lastleft、lsize、rsize が算出されます。

アルゴリズムまたは振る舞い

1. **展開**: disassembleLeaf がページ内の圧縮セグメントを解析し、dlist としてメモリ上に展開します。 2. **操作**: addItemsToLeaf 等により、TIDの挿入や削除がメモリ上のセグメントリストに対して行われます。 3. **再配置**: leafRepackItems がセグメントの再配置を行い、ページ容量に基づいた分割判定を実行します。 4. **WAL生成**: computeLeafRecompressWALData が、変更後のリーフページ状態をWALレコードとして書き出せる形式に変換します。

呼び出し元と呼び出し先

  • **呼び出し元**: gindatapage.c 内の disassembleLeaf、leafRepackItems、addItemsToLeaf、computeLeafRecompressWALData、dataPlaceToPageLeafRecompress、dataPlaceToPageLeafSplit などから参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • leafSegmentInfo: 各セグメントの変更状態を追跡する単位。
  • dlist_head / dlist_node: セグメントリストの管理に使用される双方向リスト。

失敗モードと境界条件

  • oldformat が true の場合、9.4以前の古いディスクフォーマットとして処理されます。
  • ページ容量を超えるデータ挿入が発生した場合、leafRepackItems によるページ分割処理が必須となります。

保守担当者向けメモ

  • 本構造体は圧縮されたポスティングリストを扱うための作業領域です。
  • ページ分割ロジックはインデックス構築時と通常の挿入時で挙動が異なるため、変更時は isBuild フラグの扱いに注意してください。
