walsummary_read_callback

定義

  • 種別: function
  • 定義箇所: src/bin/pg_walsummary/pg_walsummary.c:244

目的

BlockRefTableReader が WAL サマリファイルを読み込む際に使用するコールバック関数であり、ファイル記述子から指定された長さのデータを読み取ります。

シグネチャまたは形状

int walsummary_read_callback(void *callback_arg, void *data, int length)

入力と出力

  • **入力**:
  • callback_arg: ws_file_info 構造体へのポインタ。
  • data: 読み込み先バッファへのポインタ。
  • length: 読み込むバイト数。
  • **出力**:
  • 読み込んだバイト数を返します。

前提条件と不変条件

  • callback_arg は ws_file_info 型としてキャスト可能である必要があります。
  • ws_file_info 内のファイル記述子 fd は有効なオープン済みファイルである必要があります。

アルゴリズムまたは振る舞い

1. callback_arg を ws_file_info 構造体として取得します。 2. read() システムコールを使用して、ファイル記述子から length 分のデータを data バッファへ読み込みます。 3. read() が負の値を返した場合、pg_fatal を呼び出してエラーメッセージを表示し、プログラムを終了させます。 4. 読み込み成功時は、読み取ったバイト数を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: CreateBlockRefTableReader（src/bin/pg_walsummary/pg_walsummary.c:110）において、リーダーの初期化時に引数として渡されます。
  • **呼び出し先**: read()（システムコール）。

関連データ構造

  • ws_file_info: ファイル記述子とファイル名を保持する構造体。

失敗モードと境界条件

  • read() が失敗した場合、pg_fatal によりプロセスが終了します。

保守担当者向けメモ

  • 本関数は BlockRefTableReader のインターフェース仕様に従った単純なラッパーです。
  • エラー発生時の挙動は pg_fatal に依存しており、呼び出し元での個別のエラーハンドリングは行われません。
