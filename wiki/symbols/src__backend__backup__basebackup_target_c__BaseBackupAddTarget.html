BaseBackupAddTarget

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_target.c:60

目的

拡張モジュールが独自のバックアップターゲットを登録または更新するための公開APIを提供します。

シグネチャまたは形状

void BaseBackupAddTarget(char *name,
                         void *(*check_detail) (char *, char *),
                         bbsink *(*get_sink) (bbsink *, void *))

入力と出力

  • name: ターゲットを一意に識別する名称。
  • check_detail: ターゲットの詳細情報を検証する関数ポインタ。
  • get_sink: 最終的な bbsink パイプラインを生成する関数ポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • ターゲットリストは TopMemoryContext 上で管理されます。
  • 初回呼び出し時に initialize_target_list() が自動的に実行され、リストが初期化されます。

アルゴリズムまたは振る舞い

1. ターゲットリストが未初期化であれば initialize_target_list() を呼び出します。 2. 指定された name が既存のターゲットリストに存在するか検索します。

  • 存在する場合: 既存の check_detail および get_sink を新しい関数ポインタで上書きします。
  • 存在しない場合: TopMemoryContext に切り替えた上で、新しい BaseBackupTargetType 構造体を割り当て、name を複製し、リストの末尾に追加します。

    • 3. 最後に元のメモリコンテキストへ復帰します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/basebackup_to_shell/basebackup_to_shell.c 等の拡張モジュールから呼び出されることが確認されています。
  • 呼び出し先: initialize_target_list()（初期化時）、palloc/pstrdup（メモリ確保時）、MemoryContextSwitchTo（メモリ管理時）。

関連データ構造

  • BaseBackupTargetType: ターゲットの定義（名前、検証関数、シンク生成関数）を保持する構造体。
  • BaseBackupTargetTypeList: 登録されたターゲットを管理するリスト。

失敗モードと境界条件

  • 同一名称のターゲットが複数回登録された場合、既存の定義が上書きされます。
  • メモリ確保失敗時は palloc 等の標準的なPostgreSQLエラーハンドリングに従います。

保守担当者向けメモ

  • ターゲットの追加は TopMemoryContext を使用して永続化されるため、拡張モジュール側でメモリ管理を意識する必要はありません。
  • 既存ターゲットの上書きは可能ですが、意図しない挙動を防ぐため、同一名称での複数回登録は推奨されません。
