r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_german.c:241

目的

ドイツ語のステミング処理において、接尾辞削除の基準となる単語内の領域 R1 および R2 の境界位置を特定し、環境構造体 SN_env に記録する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z （ステミング実行環境）
  • **出力**: 成功時に 1、失敗時に 0 を返す。
  • **副作用**: z->I[0] に R1 の開始位置、z->I[1] に R2 の終了位置、z->I[2] に R1 の終了位置（兼 R2 の開始位置）を書き込む。

前提条件と不変条件

  • z が有効な SN_env 構造体を指していること。
  • z->p に処理対象の文字列が格納されていること。
  • g_v（母音グループ）が定義されていること。

アルゴリズムまたは振る舞い

1. **初期化**: z->I[1] と z->I[2] を文字列長 z->l で初期化する。 2. **R1の特定**:

  • 文字列の先頭から3文字スキップした位置を z->I[0] に記録する。
  • 母音（g_v）の直後の子音を探し、その位置を z->I[2] とする。
  • z->I[2] が z->I[0] より小さい場合は、z->I[2] を z->I[0] に更新する。

    • 3. **R2の特定**:

  • z->I[2] 以降で、再度母音の直後の子音を探し、その位置を z->I[1] に記録する。

呼び出し元と呼び出し先

  • **呼び出し元**: german_UTF_8_stem（直接の呼び出し元であることは文脈から推測されるが、本ソース excerpt には明示的な呼び出し関係の記述はない）。
  • **呼び出し先**: skip_utf8, out_grouping_U, in_grouping_U。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。I 配列（整数型作業領域）および c（カーソル位置）、l（文字列長）を使用する。

失敗モードと境界条件

  • skip_utf8、out_grouping_U、in_grouping_U が負の値を返した場合、処理を中断し 0 を返す。
  • 文字列が短い場合、領域境界は文字列の末尾に設定される。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードである。ロジック変更が必要な場合は、対応する .sbl ファイルを修正すること。
  • ステミングの精度は本関数が計算する R1, R2 の境界に依存するため、デバッグ時は z->I の値を確認すること。
