MemoryContextCallResetCallbacks

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/mcxt.c:590

目的

メモリコンテキストのリセットまたは削除時に、登録されたコールバック関数を順次実行し、コンテキストに関連付けられたリソースのクリーンアップを行う。

シグネチャまたは形状

static void MemoryContextCallResetCallbacks(MemoryContext context)

入力と出力

  • **入力**: context (MemoryContext型) - コールバックを実行する対象のメモリコンテキスト。
  • **出力**: なし（void）。

前提条件と不変条件

  • context が有効なメモリコンテキスト構造体であること。
  • コールバック実行中にエラーが発生した場合、当該コールバックはリストから既に削除されているため、再度の実行は行われない。

アルゴリズムまたは振る舞い

1. コンテキストの reset_cbs リストを走査する。 2. 各反復において、リストの先頭からコールバック cb を取り出し、リストから切り離す。 3. 切り離した後に cb->func(cb->arg) を呼び出す。 4. リストが空になるまでこれを繰り返す。

  • この順序により、コールバック内でエラーが発生してコンテキストがリセット/削除されても、二重実行を防止する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mmgr/mcxt.c 内の MemoryContextReset および MemoryContextDelete から呼び出される。
  • **呼び出し先**: 登録された各 MemoryContextCallback の func フィールドが指す関数。

関連データ構造

  • MemoryContext: メモリコンテキストを管理する構造体。reset_cbs フィールドを保持する。
  • MemoryContextCallback: コールバック関数ポインタ (func)、引数 (arg)、および次のコールバックへのポインタ (next) を含む構造体。

失敗モードと境界条件

  • reset_cbs が空の場合、ループは実行されず即座に終了する。
  • コールバック関数内でエラーが発生した場合、制御は呼び出し元へ戻るが、当該コールバックは既にリストから除去されている。

保守担当者向けメモ

  • 本関数は static であり、mcxt.c 内のコンテキスト破棄ロジックに密結合している。
  • コールバックの実行順序はリストの登録順（先入れ先出し）である。
