xml_encode_special_chars

定義

  • 種別: function
  • 定義箇所: contrib/xml2/xpath.c:87

目的

入力されたテキスト内のXML特殊文字（<, >, &, ", ' など）を、XMLエンティティ参照（&lt;, &gt; など）にエスケープ変換する。

シグネチャまたは形状

Datum xml_encode_special_chars(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_TEXT_PP(0) を介して取得される text 型の文字列。
  • **出力**: PG_RETURN_TEXT_P を介して返される、エスケープ処理済みの text 型文字列。

前提条件と不変条件

  • 入力テキストは pgxml_texttoxmlchar を通じて xmlChar 形式に変換可能であること。
  • 処理終了時には、内部で確保した xmlChar バッファおよび ts ポインタが適切に解放されること。

アルゴリズムまたは振る舞い

1. pgxml_texttoxmlchar を呼び出し、PostgreSQLの text 型を libxml2 が扱う xmlChar 型へ変換する。 2. xmlEncodeSpecialChars を呼び出し、特殊文字をエスケープした文字列を生成する。 3. pfree を使用して、ステップ1で確保した一時バッファを解放する。 4. cstring_to_text を使用して、エスケープ済み文字列をPostgreSQLの text 型へ変換する。 5. xmlFree を使用して、xmlEncodeSpecialChars が確保したメモリを解放する。 6. 変換後の text 型を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に直接的な呼び出し元関数の記述はありません。PG_FUNCTION_INFO_V1 マクロによる登録が確認されています。
  • **呼び出し先**:
  • pgxml_texttoxmlchar
  • xmlEncodeSpecialChars (libxml2)
  • pfree
  • cstring_to_text
  • xmlFree (libxml2)

関連データ構造

  • text (PostgreSQL型)
  • xmlChar (libxml2型)

失敗モードと境界条件

  • 入力文字列がNULLの場合、PG_GETARG_TEXT_PP の挙動に従いエラーとなる可能性がある。
  • libxml2 側のメモリ確保失敗時には、xmlEncodeSpecialChars がNULLを返す可能性がある（本関数内での明示的なNULLチェックはソースに含まれていない）。

保守担当者向けメモ

  • 本関数は libxml2 のメモリ管理とPostgreSQLのメモリ管理を混在させているため、pfree と xmlFree の使い分けに注意すること。
  • xmlEncodeSpecialChars の第1引数に NULL を渡しているため、デフォルトのエンコーディング処理が適用される。
