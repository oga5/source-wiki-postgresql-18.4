ExceptionLabelMap

定義

  • 種別: struct
  • 定義箇所: src/pl/plpgsql/src/pl_comp.c:59

目的

PL/pgSQL における例外処理のラベル名と、それに対応する PostgreSQL の SQL エラー状態コード（sqlerrstate）をマッピングするために使用されます。

シグネチャまたは形状

typedef struct
{
	const char *label;
	int			sqlerrstate;
} ExceptionLabelMap;

入力と出力

  • 入力: label（例外名を示す文字列）、sqlerrstate（エラーコードを示す整数）。
  • 出力: なし（データ構造定義）。

前提条件と不変条件

  • label は例外条件を識別する文字列である必要があります。
  • sqlerrstate は PostgreSQL のエラーコード体系に準拠した値である必要があります。

アルゴリズムまたは振る舞い

本構造体は、静的なマッピングテーブル（exception_label_map）の各要素を定義するために利用されます。このテーブルを通じて、ユーザーが指定した例外ラベルから対応する内部エラーコードへの変換が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpgsql/src/pl_comp.c 内の exception_label_map 配列定義にて使用されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • exception_label_map: ExceptionLabelMap 型の配列として定義されており、本構造体のインスタンスを保持します。

失敗モードと境界条件

  • 本構造体自体はデータ定義であるため、直接的な失敗モードはありません。ただし、マッピングテーブルに存在しないラベルを指定した場合の挙動は、この構造体を利用する側のロジックに依存します。

保守担当者向けメモ

  • 本構造体は pl_comp.c 内で静的に定義された例外マッピングを管理するためのものです。新しい例外ラベルを追加する場合は、exception_label_map 配列を更新してください。
