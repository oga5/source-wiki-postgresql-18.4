sqlca_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-dyntest.c:84

目的

ECPG（Embedded SQL for C）におけるSQL通信領域（SQL Communication Area）を定義する構造体です。SQL文の実行結果、エラー情報、警告、および処理された行数などのメタデータを保持します。

シグネチャまたは形状

struct sqlca_t {
    char    sqlcaid[8];
    long    sqlabc;
    long    sqlcode;
    struct {
        int     sqlerrml;
        char    sqlerrmc[SQLERRMC_LEN];
    } sqlerrm;
    char    sqlerrp[8];
    long    sqlerrd[6];
    char    sqlwarn[8];
    char    sqlstate[5];
};

入力と出力

本構造体は、SQL操作の結果を格納する出力先として機能します。

  • **入力**: SQL実行エンジンが各操作後に本構造体の各フィールドを更新します。
  • **出力**: アプリケーションは本構造体を参照することで、直前のSQL操作が成功したか、あるいはどのようなエラーや警告が発生したかを判定します。

前提条件と不変条件

  • sqlerrd[2] は、INSERT、UPDATE、DELETE 文の実行後に処理された行数を保持します。
  • sqlwarn[0] は、他の警告フラグ（sqlwarn[1]等）のいずれかが設定された場合に 'W' に設定されます。
  • sqlstate は、SQL標準に準拠した5文字のエラーコードを保持します。

アルゴリズムまたは振る舞い

ECPGライブラリは、SQL文の実行ごとにこの構造体を更新します。アプリケーションは、ECPGget_sqlca() を通じてこの構造体へのポインタを取得し、エラーチェックを行います。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/ecpglib/connect.c
  • src/interfaces/ecpg/ecpglib/data.c
  • src/interfaces/ecpg/ecpglib/descriptor.c
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • sqlda_t: SQL記述子領域（SQL Descriptor Area）。動的SQLのメタデータ管理に使用されます。
  • sqlvar_t: sqlda_t 内で使用される列属性定義構造体。

失敗モードと境界条件

  • sqlerrmc のサイズは SQLERRMC_LEN に制限されており、エラーメッセージがこれを超える場合は切り詰められます。
  • sqlerrd 配列の要素 0, 3, 4, 5 は現在使用されておらず、空の状態です。

保守担当者向けメモ

  • 本構造体はECPGの標準的なエラーハンドリング基盤です。変更を加える際は、ecpglib 全体のエラー伝播メカニズムへの影響を考慮してください。
  • プリプロセッサ生成コード（sql-dyntest.c 等）では、この構造体が暗黙的に使用されるため、定義の変更は広範囲に影響を及ぼす可能性があります。
