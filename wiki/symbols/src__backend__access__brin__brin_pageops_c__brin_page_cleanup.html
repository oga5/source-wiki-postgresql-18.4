brin_page_cleanup

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_pageops.c:623

目的

BRINインデックスのページ状態を検証し、未初期化ページの初期化および空き領域マップ（FSM）への空き容量の記録を行う。

シグネチャまたは形状

void brin_page_cleanup(Relation idxrel, Buffer buf)

入力と出力

  • idxrel: 対象のBRINインデックスリレーション。
  • buf: クリーンアップ対象のバッファ。
  • 出力: なし（副作用としてバッファの初期化またはFSMの更新を行う）。

前提条件と不変条件

  • buf は呼び出し元によってピン留めされている必要がある。
  • メタページおよびRevmapページに対しては、本関数は何も行わず即座に復帰する。

アルゴリズムまたは振る舞い

1. **未初期化ページの処理**:

  • PageIsNew でページが空か確認する。
  • 並行したリレーション拡張との競合を避けるため、一時的に ShareLock で拡張ロックを取得する。
  • 再度 PageIsNew を確認し、真であれば brin_initialize_empty_new_buffer を呼び出してページを初期化する。

    • 2. **特殊ページの除外**:

  • メタページまたはRevmapページの場合は処理を終了する。

    • 3. **FSMの更新**:

  • 通常のインデックスページであれば、br_page_get_freespace で空き領域を測定し、RecordPageWithFreeSpace を用いてFSMに記録する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/brin/brin.c:2191
  • 呼び出し先:
  • LockRelationForExtension / UnlockRelationForExtension
  • LockBuffer / PageIsNew
  • brin_initialize_empty_new_buffer
  • BRIN_IS_META_PAGE / BRIN_IS_REVMAP_PAGE
  • RecordPageWithFreeSpace
  • br_page_get_freespace

関連データ構造

  • Relation: インデックスリレーション情報。
  • Buffer: バッファ管理構造体。
  • Page: ページヘッダー情報。

失敗モードと境界条件

  • 並行してページが初期化された場合、二重初期化を防ぐための再チェックロジックが機能する。
  • メタページやRevmapページが渡された場合、安全に無視される。

保守担当者向けメモ

  • 本関数は、インデックスの物理的な整合性を保つための重要なユーティリティである。
  • FSMへの記録は、インデックスの再利用効率に直結するため、br_page_get_freespace の計算精度が重要となる。
