ResOwnerReleaseOSSLCipher

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/openssl.c:832

目的

PostgreSQL の ResourceOwner 機構において、トランザクション終了時やエラー発生時に OSSLCipher コンテキストを安全に解放するためのコールバック関数です。

シグネチャまたは形状

static void ResOwnerReleaseOSSLCipher(Datum res)

入力と出力

  • **入力**: res (Datum) - 解放対象の OSSLCipher 構造体へのポインタを保持する Datum 値。
  • **出力**: なし。

前提条件と不変条件

  • res は OSSLCipher 構造体への有効なポインタである必要があります。
  • 本関数は ResourceOwner の解放処理から呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. DatumGetPointer(res) を使用して、Datum 型から OSSLCipher 構造体へのポインタを復元します。 2. 復元したポインタを free_openssl_cipher 関数に渡し、OpenSSL のコンテキスト破棄およびメモリ解放を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: ResourceOwner のリソース解放処理（contrib/pgcrypto/openssl.c:275 にて ResourceReleaseCallback 構造体の .ReleaseResource フィールドに登録されている）。
  • **呼び出し先**: free_openssl_cipher（OSSLCipher の内部リソースを解放する関数）。

関連データ構造

  • OSSLCipher: 暗号化/復号用のコンテキストを保持する構造体。
  • ResourceOwner: PostgreSQL のリソース管理機構。

失敗モードと境界条件

  • res が NULL または不正なポインタである場合、free_openssl_cipher 内で未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は ResourceOwner による自動的なメモリ管理の一部です。新しい暗号化コンテキストを ResourceOwner に登録する際は、必ず本関数と対になる登録・解除ロジックが維持されていることを確認してください。
