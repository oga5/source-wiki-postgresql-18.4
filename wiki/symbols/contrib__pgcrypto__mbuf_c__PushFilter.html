PushFilter

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/mbuf.c:346

目的

暗号化や圧縮などのデータ変換処理を段階的に行うための、書き込み用フィルタチェーンを構成する構造体です。

シグネチャまたは形状

struct PushFilter
{
    PushFilter *next;           /* 次のフィルタへのポインタ */
    const PushFilterOps *op;    /* フィルタ操作の関数テーブル */
    int         block_size;     /* バッファリングするブロックサイズ */
    uint8      *buf;            /* 内部データバッファ */
    int         pos;            /* 現在のバッファ位置 */
    void       *priv;           /* フィルタ固有のプライベートデータ */
};

入力と出力

  • **入力**: pushf_write を通じて渡されるデータストリーム。
  • **出力**: フィルタチェーンの末尾、または op->push を経由して次のフィルタへ渡される変換済みデータ。

前提条件と不変条件

  • PostgreSQL のメモリコンテキスト内で動作します。
  • pushf_free_all を呼び出すことで、チェーン全体を適切に解放する必要があります。
  • op->push 関数ポインタは、フィルタ固有の処理を実装している必要があります。

アルゴリズムまたは振る舞い

1. pushf_write が呼び出されると、データはまず buf に蓄積されます。 2. pos が block_size に達すると、op->push 関数が実行され、変換処理が行われます。 3. 変換されたデータは、next ポインタが指す次のフィルタへ順次渡されます。 4. pushf_flush を呼び出すことで、バッファに残っているデータを強制的に処理し、チェーンの末尾まで流し込むことができます。

呼び出し元と呼び出し先

  • **呼び出し元**: pushf_create, pushf_write, pushf_flush, pushf_free, pushf_free_all, pushf_create_mbuf_writer などが本構造体を操作します。
  • **呼び出し先**: op メンバに含まれる関数ポインタ（PushFilterOps）を介して、具体的なフィルタ処理が実行されます。

関連データ構造

  • PushFilterOps: フィルタの振る舞いを定義する関数テーブル。
  • MBuf: データの保持に使用される可変長バッファ。

失敗モードと境界条件

  • メモリ不足時には PostgreSQL のメモリ管理関数によりエラーが発生します。
  • フィルタチェーンの構築や処理において、データ整合性が損なわれた場合は PXE_PGP_CORRUPT_DATA 等の例外が返される可能性があります。

保守担当者向けメモ

  • 本構造体はポリモーフィックな設計です。新しいフィルタを追加する場合は、PushFilterOps を適切に実装してください。
  • priv メンバには、各フィルタの実装に必要な状態（暗号化コンテキストなど）を保持します。メモリリークを防ぐため、priv の解放処理が op->free 等で正しく行われているか確認してください。
