GetIncrementalHeaderSize

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_incremental.c:880

目的

増分バックアップファイルにおけるヘッダーサイズを計算します。マジックナンバー、切り詰めブロック長、ブロック数、およびブロック番号リストを格納するために必要なバイト数を算出します。

シグネチャまたは形状

size_t GetIncrementalHeaderSize(unsigned num_blocks_required)

入力と出力

  • **入力**: num_blocks_required (unsigned) - ヘッダーに含めるブロックの数。
  • **出力**: size_t - 計算されたヘッダーサイズ（バイト単位）。

前提条件と不変条件

  • num_blocks_required は RELSEG_SIZE 以下である必要があります（Assert による検証あり）。
  • ブロックが1つ以上存在する場合、戻り値は BLCKSZ の倍数に切り上げられます。

アルゴリズムまたは振る舞い

1. 基本サイズとして、3つの uint32（マジックナンバー、切り詰めブロック長、ブロック数）と、指定された数の BlockNumber を加算します。 2. num_blocks_required > 0 かつ計算結果が BLCKSZ の倍数でない場合、BLCKSZ の倍数になるまで値を加算します。 3. ブロックが不要な場合（num_blocks_required == 0）、パディングは行わず小さいサイズを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_incremental.c:921
  • **呼び出し先**:
  • 情報なし（標準的な算術演算と Assert マクロのみを使用）。

関連データ構造

  • BlockNumber (型)
  • BLCKSZ (定数)
  • RELSEG_SIZE (定数)

失敗モードと境界条件

  • num_blocks_required が RELSEG_SIZE を超える場合、アサーション失敗によりプロセスが停止します。
  • num_blocks_required が 0 の場合、パディング処理はスキップされます。

保守担当者向けメモ

  • 本関数は増分バックアップのファイルフォーマットに強く依存しています。ヘッダー構造（マジックナンバー等の数や型）に変更を加える際は、この計算ロジックも更新する必要があります。
