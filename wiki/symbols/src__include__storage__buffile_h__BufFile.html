BufFile

定義

  • 種別: struct
  • 定義箇所: src/include/storage/buffile.h:33

目的

大規模な一時ファイルを効率的に扱うためのバッファリング層を提供する不透明な型。OS のファイルサイズ制限や単一ファイル記述子の制約を回避し、巨大なデータセットを透過的に管理する。

シグネチャまたは形状

typedef struct BufFile BufFile;

入力と出力

  • 入力: BufFileCreateTemp() 等により生成される。
  • 出力: BufFileClose() により関連するすべての仮想ファイル記述子が解放され、リソースがクリーンアップされる。

前提条件と不変条件

  • BufFile 構造体は palloc によってメモリ確保される。
  • BufFileClose を呼び出さない場合、ファイル記述子がリークする可能性がある。
  • stdio とは異なり、フォーマット機能（printf 等）は持たず、純粋なバイナリデータのバッファリングに特化している。

アルゴリズムまたは振る舞い

1. **バッファリング**: 内部バッファを操作し、バッファが溢れるか空になった場合にのみ fd.c を通じて物理的なファイル I/O を実行する。 2. **セグメント管理**: ファイルサイズが 1GB の上限に達すると、自動的に新しい物理ファイルを作成・オープンし、論理的なストリームを継続する。 3. **ライフサイクル**: トランザクション終了時の自動クリーンアップや、FileSet を介した並列クエリ間での共有をサポートする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistbuildbuffers.c (ReadTempFileBlock, WriteTempFileBlock)
  • src/backend/backup/basebackup.c
  • src/backend/executor/nodeAgg.c
  • src/backend/executor/nodeHash.c
  • **呼び出し先**:
  • src/backend/storage/file/buffile.c (実装本体)
  • src/include/storage/fd.h (仮想ファイル記述子管理)

関連データ構造

  • FileSet: 並列クエリなどで共有可能な一時ファイルを作成する際に使用。

失敗モードと境界条件

  • 物理ファイルサイズが 1GB を超える場合、自動的にセグメント分割が行われる。
  • BufFileClose の呼び出し漏れによるファイル記述子のリーク。

保守担当者向けメモ

  • 本構造体の内部定義は src/backend/storage/file/buffile.c に隠蔽されている。
  • 並列実行環境で一時ファイルを使用する場合は、必ず FileSet を介した API を使用すること。
