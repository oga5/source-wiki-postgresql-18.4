ginGetBAEntry

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginbulk.c:267

目的

BuildAccumulator 内の赤黒木（RBT）に蓄積されたキーとTIDリストを、構築完了後に順次取り出すためのイテレータです。

シグネチャまたは形状

ItemPointerData *ginGetBAEntry(BuildAccumulator *accum, OffsetNumber *attnum, Datum *key, GinNullCategory *category, uint32 *n);

入力と出力

  • **入力**:
  • accum: 構築状態を保持する BuildAccumulator 構造体へのポインタ。
  • attnum, key, category: 取得したエントリの属性番号、キー値、カテゴリを格納する出力用ポインタ。
  • n: 取得したTIDリストの要素数を格納する出力用ポインタ。
  • **出力**:
  • 成功時: ソート済みの ItemPointerData 配列へのポインタ。
  • 終了時: NULL。

前提条件と不変条件

  • ginBeginBAScan が事前に呼び出され、accum->tree_walk が初期化されていること。
  • 返されるリストは NULL ではなく、かつ entry->count > 0 であることがアサーションにより保証されている。

アルゴリズムまたは振る舞い

1. rbt_iterate を使用して、RBTから次の GinEntryAccumulator ノードを取得する。 2. ノードが取得できない場合は NULL を返し、走査を終了する。 3. 取得したノードから属性番号、キー、カテゴリ、TIDリスト、要素数を抽出する。 4. ノードの shouldSort フラグが真かつ要素数が1より大きい場合、qsort と qsortCompareItemPointers を用いてTIDリストをソートする。 5. ソート済み（またはソート不要な）TIDリストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginfast.c
  • src/backend/access/gin/gininsert.c
  • **呼び出し先**:
  • rbt_iterate (RBT走査用)
  • qsort (TIDリストのソート用)
  • qsortCompareItemPointers (比較関数)

関連データ構造

  • BuildAccumulator: 構築状態を管理するメイン構造体。
  • GinEntryAccumulator: RBTの各ノードであり、キーとTIDリストを保持する。
  • ItemPointerData: GINインデックスで管理されるタプル識別子。

失敗モードと境界条件

  • RBTの走査が完了した場合は NULL を返す。
  • shouldSort が偽の場合、TIDリストはソートされずにそのまま返される。

保守担当者向けメモ

  • 本関数は ginBeginBAScan と対で使用される。
  • qsort によるソートは、shouldSort フラグが立っている場合にのみ実行されるため、呼び出し側でこのフラグの状態を適切に管理する必要がある。
