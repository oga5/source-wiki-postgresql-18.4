pgstatindex_v1_5

定義

  • 種別: function
  • 定義箇所: contrib/pgstattuple/pgstatindex.c:166

目的

指定されたインデックス名からリレーションを開き、pgstatindex_impl を呼び出してインデックスの統計情報を取得・返却するSQLインターフェース関数です。

シグネチャまたは形状

Datum pgstatindex_v1_5(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して、対象インデックス名を表す text 型の引数を受け取ります。
  • **出力**: Datum 型。インデックスの統計情報を含むタプルを返します。

前提条件と不変条件

  • 対象のリレーションに対して AccessShareLock を取得します。
  • 呼び出し元は PostgreSQL の関数呼び出し規約に従う必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP(0) で引数からインデックス名を取得します。 2. makeRangeVarFromNameList および relation_openrv を使用して、対象インデックスを AccessShareLock モードで開きます。 3. pgstatindex_impl を呼び出し、開いたリレーションと関数呼び出し情報を渡して統計計算を実行します。 4. 計算結果を PG_RETURN_DATUM を通じて返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の SQL エンジン（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: pgstatindex_impl（統計計算の中核関数）。

関連データ構造

  • Relation: 開かれたインデックスリレーションを保持します。
  • RangeVar: インデックス名を解決するために使用されます。

失敗モードと境界条件

  • 指定された名前のリレーションが存在しない場合、relation_openrv によりエラーが発生します。
  • インデックス以外のリレーションが指定された場合、後続の pgstatindex_impl 内で適切な検証が行われます。

保守担当者向けメモ

  • 本関数は pgstattuple 拡張のバージョン 1.5 インターフェースです。
  • 権限チェックは関数内で行わず、SQLレベルでの REVOKE EXECUTE ON PUBLIC を前提としています。
