SharedInvalShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/sinvaladt.c:233

目的

共有キャッシュ無効化メッセージを管理するための共有メモリ領域（shmInvalBuffer）を初期化する。

シグネチャまたは形状

void SharedInvalShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（共有メモリ上の SISeg 構造体を初期化する副作用を持つ）

前提条件と不変条件

  • 共有メモリの割り当てが ShmemInitStruct を通じて行われる。
  • すでに共有メモリが初期化済み（found が真）の場合、関数は即座にリターンする。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を呼び出し、shmInvalBuffer を共有メモリ上に確保する。 2. 既に初期化されている場合は終了する。 3. メッセージカウンタ（minMsgNum, maxMsgNum）を 0 にリセットし、クリーンアップ閾値を設定する。 4. msgnumLock スピンロックを初期化する。 5. 全バックエンドの ProcState 配列を走査し、各スロットを非アクティブ状態（procPid = 0）に初期化する。 6. numProcs を 0 に設定し、pgprocnos ポインタを procState 配列の末尾に配置する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:320
  • 呼び出し先: ShmemInitStruct, SpinLockInit

関連データ構造

  • SISeg: 共有メモリ上のメイン構造体。
  • ProcState: 各バックエンドの進捗状態を保持する配列。

失敗モードと境界条件

  • 共有メモリの割り当てに失敗した場合、ShmemInitStruct の内部挙動によりプロセスが終了する可能性がある。
  • found フラグにより、二重初期化を防止する不変条件が維持される。

保守担当者向けメモ

  • 本関数はシステム起動時に一度だけ実行されることを想定している。
  • pgprocnos のポインタ設定は procState 配列の直後に配置する設計となっており、メモリレイアウトの変更時には注意が必要である。
