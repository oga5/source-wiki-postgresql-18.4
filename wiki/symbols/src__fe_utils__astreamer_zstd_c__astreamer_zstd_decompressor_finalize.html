astreamer_zstd_decompressor_finalize

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_zstd.c:338

目的

Zstandard伸長ストリームの終了処理を行い、バッファ内に残存するデータを後続のストリーマーへ転送した上で、パイプラインの終了を通知する。

シグネチャまたは形状

static void astreamer_zstd_decompressor_finalize(astreamer *streamer)

入力と出力

  • **入力**: astreamer *streamer（astreamer_zstd_frame構造体としてキャストして使用）
  • **出力**: なし（副作用として後続ストリーマーへのデータ転送および終了通知を実行）

前提条件と不変条件

  • 引数 streamer は astreamer_zstd_frame 型のインスタンスとして正しく初期化されていること。
  • mystreamer->base.bbs_next が有効な後続ストリーマーを指していること。

アルゴリズムまたは振る舞い

1. streamer を astreamer_zstd_frame 型にキャストする。 2. zstd_outBuf.pos を確認し、出力バッファに未処理のデータが残っている場合、astreamer_content を呼び出して後続のストリーマーへデータを転送する。 3. astreamer_finalize を呼び出し、後続のストリーマーに対してストリームの終了を通知する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/astreamer_zstd.c 内のストリーマー構造体定義（.finalize メンバへの代入）。
  • **呼び出し先**:
  • astreamer_content（後続ストリーマーへのデータ転送）
  • astreamer_finalize（後続ストリーマーの終了処理）

関連データ構造

  • astreamer_zstd_frame: 伸長コンテキストと出力バッファ（zstd_outBuf）を保持する構造体。
  • astreamer: ストリーマーの基底構造体。

失敗モードと境界条件

  • zstd_outBuf.pos が 0 の場合、データ転送はスキップされ、直ちに astreamer_finalize が実行される。
  • 後続ストリーマーが NULL である場合の挙動については、本ソースコード内では明示的なチェックは行われていない。

保守担当者向けメモ

  • 本関数は astreamer インフラストラクチャの終了フェーズを担う。パイプラインの整合性を保つため、必ず bbs_next の終了処理が最後に行われる必要がある。
