text_starts_with_support

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like_support.c:146

目的

starts_with 関数に対するプランナのサポート関数として機能します。クエリ最適化時に、インデックススキャンが利用可能な条件式への変換や、選択率の推定を支援します。

シグネチャまたは形状

Datum text_starts_with_support(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して Node *rawreq（プランナからのサポート要求ノード）を受け取ります。
  • **出力**: Datum 型として、like_regex_support が生成した結果（インデックス条件リストまたは選択率推定値）を返します。

前提条件と不変条件

  • rawreq は有効な Node 構造体である必要があります。
  • 内部的に Pattern_Type_Prefix を指定して呼び出すため、本関数は接頭辞マッチングの最適化に特化しています。

アルゴリズムまたは振る舞い

1. 引数から rawreq を取得します。 2. Pattern_Type_Prefix を指定して like_regex_support を呼び出します。 3. like_regex_support が返す結果をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: プランナ（インデックスの利用可能性判断時）。
  • **呼び出し先**: like_regex_support。

関連データ構造

  • Node: プランナの要求を保持する汎用ノード。
  • Pattern_Type: パターンマッチングの種類を定義する列挙型（ここでは Pattern_Type_Prefix を使用）。

失敗モードと境界条件

  • like_regex_support が内部で処理できないパターンや無効な入力を受け取った場合、その関数に準じた挙動（エラーまたはデフォルト値の返却）となります。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的なロジックは like_regex_support に集約されています。
  • starts_with の最適化ロジックを変更する場合は、like_regex_support または match_pattern_prefix の修正を検討してください。
