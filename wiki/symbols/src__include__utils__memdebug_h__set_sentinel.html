set_sentinel

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/memdebug.h:50

目的

メモリ領域の指定されたオフセット位置にセンチネル値（0x7E）を書き込み、バッファオーバーラン等のメモリ破壊を検出するための境界マーカーを設置します。

シグネチャまたは形状

static inline void set_sentinel(void *base, Size offset)

入力と出力

  • base: センチネルを設置するメモリ領域の開始アドレス。
  • offset: base からのオフセット値（バイト単位）。
  • 出力: なし（副作用としてメモリ内容を更新）。

前提条件と不変条件

  • base + offset が有効なメモリ領域を指していること。
  • Valgrind環境下では、対象の1バイトが一時的にアクセス可能としてマークされ、書き込み後にアクセス不可（NOACCESS）へ戻されます。

アルゴリズムまたは振る舞い

1. base と offset から対象アドレス ptr を算出します。 2. VALGRIND_MAKE_MEM_UNDEFINED を呼び出し、対象の1バイトをValgrindに対してアクセス可能にします。 3. ptr に 0x7E を書き込みます。 4. VALGRIND_MAKE_MEM_NOACCESS を呼び出し、対象の1バイトを再びアクセス不可に設定して保護します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/mmgr/aset.c
  • src/backend/utils/mmgr/bump.c
  • src/backend/utils/mmgr/generation.c
  • src/backend/utils/mmgr/mcxt.c
  • src/backend/utils/mmgr/slab.c
  • 呼び出し先:
  • VALGRIND_MAKE_MEM_UNDEFINED (Valgrindマクロ)
  • VALGRIND_MAKE_MEM_NOACCESS (Valgrindマクロ)

関連データ構造

  • Size (型)
  • MemoryChunk (各メモリコンテキスト実装で使用されるデータ構造)

失敗モードと境界条件

  • base が NULL または不正なアドレスである場合、セグメンテーション違反が発生します。
  • offset が確保されたメモリ領域の範囲外である場合、隣接するメモリ領域を破壊します。

保守担当者向けメモ

  • 本関数はデバッグビルド時（MEMORY_CONTEXT_CHECKING 等が有効な場合）に機能します。
  • Valgrind連携マクロは、Valgrindがインストールされていない環境では空の定義に置換される設計です。
  • センチネル値の検証には対となる sentinel_ok() 関数を使用してください。
