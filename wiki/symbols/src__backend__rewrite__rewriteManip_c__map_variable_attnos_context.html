map_variable_attnos_context

定義

  • 種別: struct
  • 定義箇所: src/backend/rewrite/rewriteManip.c:1562

目的

クエリツリー内の変数（Var）ノードを走査し、特定の範囲テーブルエントリ（RTE）に関連する属性番号（attno）をマッピングまたは置換するためのコンテキスト情報を保持します。

シグネチャまたは形状

typedef struct {
    int         target_varno;    /* 検索対象のRTEインデックス */
    int         sublevels_up;    /* 現在のネスト深さ */
    const AttrMap *attno_map;    /* ユーザー属性番号のマップ配列 */
    Oid         to_rowtype;      /* 行全体（whole-row）Varの変換先型 */
    bool       *found_whole_row; /* 出力フラグ */
} map_variable_attnos_context;

入力と出力

  • **入力**: 走査対象のクエリツリーおよび本構造体。
  • **出力**: found_whole_row を通じて、行全体を参照する変数が検出されたかどうかのフラグを返します。

前提条件と不変条件

  • sublevels_up は、ツリー走査のネストレベルに応じて適切に管理される必要があります。
  • attno_map は、属性番号の変換に必要な有効なマッピング情報を含んでいる必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、expression_tree_walker や expression_tree_mutator を用いたツリー走査において、現在の走査状態を保持するために使用されます。
  • target_varno と一致する Var ノードが見つかった場合、attno_map に基づいて属性番号の変換が行われます。
  • 行全体を参照する Var ノードが検出された場合、to_rowtype に基づいて型変換が行われ、found_whole_row が true に設定されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/rewrite/rewriteManip.c:1699 にてインスタンス化されることが確認されています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、具体的な関数呼び出し関係はソース抜粋からは特定できません。

関連データ構造

  • AttrMap: 属性番号の変換に使用されるマップ構造。
  • Var: クエリツリー内で操作対象となる変数ノード。

失敗モードと境界条件

  • attno_map が不適切な場合、属性番号の変換が正しく行われない可能性があります。
  • 走査中に sublevels_up の管理を誤ると、誤ったネストレベルの変数を操作するリスクがあります。
