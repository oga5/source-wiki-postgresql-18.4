BufFileOpenFileSet

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/buffile.c:290

目的

FileSet 内に存在する、分割された物理ファイル群を論理的に単一の BufFile として再構築して開く。

シグネチャまたは形状

BufFile *BufFileOpenFileSet(FileSet *fileset, const char *name, int mode, bool missing_ok)

入力と出力

  • **入力**:
  • fileset: 対象の FileSet 構造体へのポインタ。
  • name: 開くファイルのベース名。
  • mode: ファイルオープンモード（O_RDONLY 等）。
  • missing_ok: ファイルが存在しない場合にエラーとせず NULL を返すかどうかのフラグ。
  • **出力**:
  • 成功時: 構築された BufFile 構造体へのポインタ。
  • 失敗時: missing_ok が真であれば NULL、偽であればエラーを報告。

前提条件と不変条件

  • FileSet は初期化済みであること。
  • 物理ファイルは FileSetSegmentName に従った命名規則（ベース名 + セグメント番号）で存在していること。

アルゴリズムまたは振る舞い

1. capacity を初期値 16 として、ファイル記述子を格納する配列 files を割り当てる。 2. FileSetSegmentName を用いてセグメント名を生成し、FileSetOpen で順次ファイルを開く。 3. ファイルが開けなくなるまでセグメント番号をインクリメントし、必要に応じて files 配列を repalloc で拡張する。 4. ファイルが一つも開けなかった場合、missing_ok に応じて NULL を返すか、エラーを報告する。 5. 開いたファイル群を makeBufFileCommon に渡し、BufFile 構造体を初期化して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/worker.c
  • src/backend/utils/sort/logtape.c
  • src/backend/utils/sort/sharedtuplestore.c
  • **呼び出し先**:
  • palloc, repalloc, pfree
  • FileSetSegmentName
  • FileSetOpen
  • CHECK_FOR_INTERRUPTS
  • makeBufFileCommon
  • ereport

関連データ構造

  • BufFile: 戻り値として返される論理ファイル構造体。
  • FileSet: 共有ファイルセットを管理する構造体。
  • File: fd.c で管理される仮想ファイル記述子。

失敗モードと境界条件

  • 指定された名前のファイルが一つも存在しない場合、missing_ok が偽であれば ERROR を発生させる。
  • メモリ不足時には palloc/repalloc がエラーを発生させる。
  • FileSetOpen が失敗した時点でセグメントの探索を終了する。

保守担当者向けメモ

  • 本関数はセグメントを動的に探索するため、ファイル数が多い場合に repalloc が繰り返される可能性がある。
  • file->readOnly は mode が O_RDONLY かどうかで判定される。
