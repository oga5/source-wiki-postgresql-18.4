ReplicationSlotInvalidationCause

定義

  • 種別: enum
  • 定義箇所: src/include/replication/slot.h:51

目的

レプリケーションスロットが無効化された理由を識別するための列挙型です。

シグネチャまたは形状

typedef enum ReplicationSlotInvalidationCause
{
    RS_INVAL_NONE = 0,
    RS_INVAL_WAL_REMOVED = (1 << 0),
    RS_INVAL_HORIZON = (1 << 1),
    RS_INVAL_WAL_LEVEL = (1 << 2),
    RS_INVAL_IDLE_TIMEOUT = (1 << 3),
} ReplicationSlotInvalidationCause;

入力と出力

  • 本シンボルは列挙型であり、関数ではありません。
  • 状態遷移やログ出力において、無効化の要因を特定する値として使用されます。

前提条件と不変条件

  • RS_INVAL_NONE は無効化が発生していない状態を示します。
  • 各値はビットフラグとして定義されており、特定の無効化イベントに対応します。

アルゴリズムまたは振る舞い

  • RS_INVAL_WAL_REMOVED: 必要なWALファイルが削除された場合に発生します。
  • RS_INVAL_HORIZON: 必要な行（タプル）が削除された場合に発生します。
  • RS_INVAL_WAL_LEVEL: スロットに必要な wal_level が不足している場合に発生します。
  • RS_INVAL_IDLE_TIMEOUT: スロットがアイドル状態のままタイムアウトした場合に発生します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/slot.c 内の無効化処理ロジックや、src/backend/replication/logical/slotsync.c 等で参照されています。
  • 呼び出し先: 本シンボル自体はデータ型であり、直接の呼び出し先はありません。GetSlotInvalidationCauseName 関数などがこの型を引数として受け取り、名称への変換を行います。

関連データ構造

  • ReplicationSlot: レプリケーションスロットのメモリ内表現。スロットの状態管理において本列挙型が利用されます。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 新たな無効化要因を追加する場合は、ビットフラグの重複がないように注意してください。
  • GetSlotInvalidationCauseName 関数（src/backend/replication/slot.c）を更新し、新しい要因に対する文字列変換処理を追加する必要があります。
