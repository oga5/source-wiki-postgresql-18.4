gistRedoPageSplitRecord

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistxlog.c:246

目的

GiSTインデックスのページ分割操作をWALリプレイ時に再実行し、インデックスページの整合性を復元する。

シグネチャまたは形状

static void gistRedoPageSplitRecord(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）。
  • **出力**: なし（副作用としてバッファへの書き込みとロック解除を行う）。

前提条件と不変条件

  • 最初のページ（i=0）のロックは、処理全体を通じて保持され、最後に解除される。
  • 2番目以降のページは、書き込み完了後に順次ロックが解除される。
  • ルートページ分割時は isrootsplit フラグが真となる。

アルゴリズムまたは振る舞い

1. WALレコードから gistxlogPageSplit データを取得し、分割対象の全ページをループ処理する。 2. XLogInitBufferForRedo で各ページを初期化し、decodePageSplitRecord でタプルを復元して gistfillbuffer でページを再構築する。 3. ページの種類に応じて以下のメタデータを設定する:

  • **ルートページ**: rightlink を InvalidBlockNumber に設定し、NSNを更新、FollowRightフラグをクリアする。
  • **非ルートページ**: rightlink に次のページ番号（または元の右リンク）を設定し、NSNを更新。条件に応じてFollowRightフラグを制御する。

    • 4. ページにLSNを書き込み、MarkBufferDirty を呼び出す。 5. 最初のページ以外は UnlockReleaseBuffer でロックを解放する。 6. 必要に応じて gistRedoClearFollowRight を呼び出し、左側の子ページのFollowRightデータを修正する。 7. 最後に最初のページのロックを解除する。

呼び出し元と呼び出し先

  • **呼び出し元**: gist_redo（src/backend/access/gist/gistxlog.c 内のディスパッチャ）。
  • **呼び出し先**: XLogRecGetData, XLogRecGetBlockTag, XLogInitBufferForRedo, decodePageSplitRecord, GISTInitBuffer, gistfillbuffer, GistPageGetOpaque, GistPageSetNSN, GistClearFollowRight, GistMarkFollowRight, PageSetLSN, MarkBufferDirty, UnlockReleaseBuffer, XLogRecHasBlockRef, gistRedoClearFollowRight。

関連データ構造

  • gistxlogPageSplit: WALレコード内の分割情報構造体。
  • XLogReaderState: WAL読み取り状態。
  • Buffer, Page: バッファ管理およびページ構造。

失敗モードと境界条件

  • blkno == GIST_ROOT_BLKNO の場合、必ず i == 0 である必要がある（アサーションで検証）。
  • ページ分割の順序やFollowRightフラグの制御は、同時実行クエリが最初のページを経由せずに分割後のページへ到達しないよう設計されている。

保守担当者向けメモ

  • 本関数はGiSTのページ分割という複雑な操作をリプレイするため、rightlink や NSN の更新ロジックを変更する際は、インデックスの検索パスへの影響を慎重に評価すること。
  • gistRedoClearFollowRight による後処理が、分割された全ページに対して正しく整合性を保っているか確認が必要。
