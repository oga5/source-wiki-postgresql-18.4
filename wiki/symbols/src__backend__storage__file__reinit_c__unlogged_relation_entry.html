unlogged_relation_entry

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/file/reinit.c:32

目的

アンログド（unlogged）リレーションのクリーンアップ処理において、処理対象となるリレーションをハッシュテーブルで追跡するためのエントリ構造体です。

シグネチャまたは形状

typedef struct
{
	RelFileNumber relnumber;	/* hash key */
} unlogged_relation_entry;

入力と出力

  • 入力: RelFileNumber 型の relnumber を保持します。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • relnumber はハッシュテーブルのキーとして使用されます。

アルゴリズムまたは振る舞い

  • 本構造体は、ResetUnloggedRelations 内で作成されるハッシュテーブルの要素として機能します。
  • _init フォークを持つリレーションを特定し、それ以外のフォークを削除する際の識別子として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/file/reinit.c 内の ResetUnloggedRelations 関数が、ハッシュテーブルの初期化（ctl.entrysize）およびエントリの挿入・検索時に本構造体を使用します。
  • 呼び出し先: なし（データ構造体のため）。

関連データ構造

  • RelFileNumber: リレーションを識別するための型。
  • HTAB (ハッシュテーブル): 本構造体を要素として格納するために使用されます。

失敗モードと境界条件

  • 構造体自体に失敗モードはありませんが、ハッシュテーブルへの挿入時にメモリ不足が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は reinit.c 内のクリーンアップロジック専用です。将来的にリレーションの追跡情報が増える場合は、本構造体にメンバを追加することで対応可能です。
