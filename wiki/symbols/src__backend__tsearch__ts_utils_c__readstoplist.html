readstoplist

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_utils.c:68

目的

指定されたファイルからストップワード（除外単語）リストを読み込み、メモリ上にソート済み配列として格納します。

シグネチャまたは形状

void readstoplist(const char *fname, StopList *s, char *(*wordop) (const char *, size_t, Oid))

入力と出力

  • **入力**:
  • fname: 読み込むストップワードファイルのベース名。
  • s: 読み込み結果を格納する StopList 構造体へのポインタ。
  • wordop: 各単語に対して適用する正規化関数（オプション）。NULL の場合は適用されません。
  • **出力**:
  • なし（s が指す構造体の len および stop フィールドが更新されます）。

前提条件と不変条件

  • fname が有効な場合、get_tsearch_config_filename により share/tsearch_data 配下のファイルが特定されます。
  • 読み込まれたリストは、関数終了時に qsort によって辞書順にソートされます。
  • s->stop は palloc によって確保されたメモリ領域を指します。

アルゴリズムまたは振る舞い

1. get_tsearch_config_filename を使用してファイルパスを生成し、ファイルを開きます。 2. tsearch_readline を用いてファイルを一行ずつ読み込みます。 3. 各行の末尾の空白を除去し、空行をスキップします。 4. wordop が指定されている場合、読み込んだ単語に適用します。 5. 読み込んだ単語を動的配列に蓄積し、必要に応じて repalloc で領域を拡張します。 6. 最後に pg_qsort_strcmp を用いて配列をソートし、バイナリサーチ可能な状態にします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/snowball/dict_snowball.c
  • src/backend/tsearch/dict_ispell.c
  • src/backend/tsearch/dict_simple.c
  • **呼び出し先**:
  • get_tsearch_config_filename
  • tsearch_readline_begin / tsearch_readline / tsearch_readline_end
  • wordop (関数ポインタ)
  • qsort (比較関数: pg_qsort_strcmp)

関連データ構造

  • StopList: tsearch/ts_public.h で定義。ストップワードの配列と件数を保持します。

失敗モードと境界条件

  • ファイルが開けない場合、ereport(ERROR, ...) によりトランザクションが中断されます。
  • ファイルが空の場合、s->len は 0 となり、s->stop は NULL のままとなります。

保守担当者向けメモ

  • wordop コールバックは、辞書ごとの正規化ルール（大文字小文字の統一など）を適用するために使用されます。
  • パス生成時のバリデーションはセキュリティ要件（ディレクトリトラバーサル対策）に基づいているため、変更時は注意してください。
