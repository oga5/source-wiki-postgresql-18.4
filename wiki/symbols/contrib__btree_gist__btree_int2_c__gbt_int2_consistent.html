gbt_int2_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int2.c:135

目的

GiST インデックス検索において、int2 型のインデックスキーがクエリ条件と合致するかを判定します。

シグネチャまたは形状

Datum gbt_int2_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY 構造体（インデックスキーを含む）。
  • PG_GETARG_INT16(1): 検索対象の int16 値。
  • PG_GETARG_UINT16(2): 比較戦略を示す StrategyNumber。
  • PG_GETARG_POINTER(4): recheck フラグへのポインタ。
  • **出力**:
  • Datum (bool): 判定結果（true または false）。
  • *recheck: 常に false を設定（本関数による判定は完全であるため）。

前提条件と不変条件

  • entry->key は int16KEY 構造体として解釈可能であること。
  • recheck は常に false に設定されるため、呼び出し元は再チェックを必要としません。

アルゴリズムまたは振る舞い

1. entry->key を int16KEY として取得し、lower および upper 境界を GBT_NUMKEY_R 構造体に格納します。 2. recheck を false に設定します。 3. gbt_num_consistent 関数を呼び出し、実際の比較ロジックを委譲します。その際、tinfo（int2 型の比較ロジックを保持する静的構造体）を渡します。 4. gbt_num_consistent の戻り値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス検索エンジン（直接的な呼び出し元コードはソース内に明示されていません）。
  • **呼び出し先**: gbt_num_consistent（contrib/btree_gist/btree_utils_num.c 内）。

関連データ構造

  • int16KEY: int2 型の範囲（lower, upper）を保持する構造体。
  • GBT_NUMKEY_R: 汎用的な数値範囲を表現する構造体。
  • tinfo: int2 型の比較ロジックを定義する gbtree_ninfo 型の静的構造体。

失敗モードと境界条件

  • 本関数は recheck を false に固定しており、インデックス内の範囲判定が完全であることを前提としています。
  • 入力引数が不正な場合、PostgreSQL の PG_GETARG_* マクロによりエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.c のラッパーです。ロジックの変更が必要な場合は、まず btree_utils_num 側の共通ロジックを確認してください。
  • int2 型特有の比較ロジックについては、tinfo 構造体を通じて参照される関数群を確認してください。
