BootStrapSUBTRANS

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/subtrans.c:269

目的

データベースの初期化時（ブートストラップ）に、サブトランザクションログ（SUBTRANS）の最初のページを生成および初期化する。

シグネチャまたは形状

void BootStrapSUBTRANS(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリおよびディスク上の初期ページを生成）

前提条件と不変条件

  • SubTransCtl が初期化済みであること。
  • 本関数はデータベースの初期化プロセスにおいて一度だけ呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. SubTransCtl に関連付けられたバンク 0 の LWLock を排他モードで取得する。 2. ZeroSUBTRANSPage(0) を呼び出し、SUBTRANS の最初のページ（ページ番号 0）をゼロクリアしてスロットに読み込む。 3. SimpleLruWritePage を呼び出し、初期化したページをディスクへ書き出す。 4. ページがダーティ状態でないことを Assert で確認する。 5. 取得した LWLock を解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlog.c:5224
  • 呼び出し先: SimpleLruGetBankLock, LWLockAcquire, ZeroSUBTRANSPage, SimpleLruWritePage, LWLockRelease

関連データ構造

  • SubTransCtl: SUBTRANS の SLRU 管理を行う共有メモリ制御構造体。

失敗モードと境界条件

  • LWLock の取得失敗やディスク書き込みエラーが発生した場合、PostgreSQL のブートストラップ処理全体が停止する可能性がある。

保守担当者向けメモ

  • 本関数はクラッシュリカバリを目的としておらず、データベース作成時の初期状態構築のみを担う。
  • SubTransCtl の共有メモリ状態を直接操作するため、ロックの取得・解放順序には注意が必要である。
