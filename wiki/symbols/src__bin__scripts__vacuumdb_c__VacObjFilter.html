VacObjFilter

定義

  • 種別: enum
  • 定義箇所: src/bin/scripts/vacuumdb.c:55

目的

vacuumdb ユーティリティにおいて、ユーザーが指定したコマンドライン引数に基づき、処理対象となるデータベースオブジェクト（データベース全体、特定のテーブル、スキーマ等）のフィルタリング種別を識別するために使用されます。

シグネチャまたは形状

ビットフラグとして定義された列挙型です。

typedef enum
{
	OBJFILTER_NONE = 0,
	OBJFILTER_ALL_DBS = (1 << 0),
	OBJFILTER_DATABASE = (1 << 1),
	OBJFILTER_TABLE = (1 << 2),
	OBJFILTER_SCHEMA = (1 << 3),
	OBJFILTER_SCHEMA_EXCLUDE = (1 << 4),
} VacObjFilter;

入力と出力

  • **入力**: ユーザーが指定したコマンドラインオプション（-a, -d, -t, -n, -N 等）。
  • **出力**: 内部状態として保持される VacObjFilter 型の値。

前提条件と不変条件

  • OBJFILTER_NONE は初期状態を表します。
  • 各フラグはビット演算による組み合わせが可能な設計となっています。

アルゴリズムまたは振る舞い

  • ユーザーのオプション指定に応じて、対応するビットフラグが設定されます。
  • check_objfilter() 関数により、指定されたフラグの組み合わせに矛盾がないか（例：全データベース対象と特定テーブル対象の同時指定など）が検証されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/scripts/vacuumdb.c 内の静的変数 objfilter の初期化（line 65）およびオプション解析処理。
  • **呼び出し先**: 詳細は提供されていませんが、check_objfilter() 等の検証関数や、処理対象を決定するロジックで参照されます。

関連データ構造

  • vacuumingOptions: 実行オプションを保持する構造体。

失敗モードと境界条件

  • 矛盾するフィルタフラグが同時に設定された場合、check_objfilter() によってエラーとして扱われます。

保守担当者向けメモ

  • 新しいフィルタ種別を追加する場合は、ビットシフトによるフラグ定義を維持してください。
  • check_objfilter() 内のバリデーションロジックが、新しいフラグの組み合わせを正しく処理できるか確認してください。
