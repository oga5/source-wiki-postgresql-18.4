rbt_insert

定義

  • 種別: function
  • 定義箇所: src/backend/lib/rbtree.c:452

目的

赤黒木に新しいノードを挿入します。キーが既に存在する場合は、既存ノードに対して結合処理（combiner）を行います。

シグネチャまたは形状

RBTNode *rbt_insert(RBTree *rbt, const RBTNode *data, bool *isNew)

入力と出力

  • **入力**:
  • rbt: 操作対象の赤黒木構造体。
  • data: 挿入するデータを含むノード（RBTNode）。
  • isNew: 挿入が新規作成であれば true、既存キーの更新であれば false が設定される出力用ポインタ。
  • **出力**: 挿入または更新されたノードへのポインタを返します。

前提条件と不変条件

  • rbt は rbt_create で初期化済みであること。
  • rbt->comparator および rbt->combiner コールバックが適切に設定されていること。
  • 挿入されるノードは、呼び出し側が提供する構造体の先頭メンバとして埋め込まれている必要があります。

アルゴリズムまたは振る舞い

1. **検索**: rbt->comparator を使用して、挿入位置を特定します。 2. **重複処理**: キーが一致する場合、rbt->combiner を呼び出し、isNew を false に設定して既存ノードを返します。 3. **新規挿入**:

  • キーが存在しない場合、rbt->allocfunc を呼び出して新しいノードを確保します。
  • ノードを赤色（RBTRED）に初期化し、rbt_copy_data でデータをコピーします。
  • 親ノードの子ポインタを更新し、木に接続します。
  • rbt_insert_fixup を呼び出し、赤黒木の平衡条件を維持します。
  • isNew を true に設定して新しいノードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginbulk.c
  • src/test/modules/test_rbtree/test_rbtree.c
  • **呼び出し先**:
  • rbt->comparator (コールバック)
  • rbt->combiner (コールバック)
  • rbt->allocfunc (コールバック)
  • rbt_copy_data
  • rbt_insert_fixup

関連データ構造

  • RBTree: 木全体を管理する制御構造体。
  • RBTNode: 各ノードのヘッダー。
  • RBTNIL: 葉ノードを表す静的な番兵。

失敗モードと境界条件

  • rbt->allocfunc がメモリ不足等で失敗した場合の挙動は、当該コールバックの実装に依存します。
  • 木が空の場合、挿入ノードがルートとなります。
  • 比較関数が 0 を返す場合、常に既存ノードの更新として扱われます。

保守担当者向けメモ

  • 本関数は侵入型設計を採用しており、メモリ確保・解放は呼び出し側の責任です。
  • RBTNIL を使用して NULL チェックを回避しているため、ノードの比較時には必ず RBTNIL との比較を行ってください。
