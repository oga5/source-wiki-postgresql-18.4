DefineOpFamily

定義

  • 種別: function
  • 定義箇所: src/backend/commands/opclasscmds.c:771

目的

CREATE OPERATOR FAMILY SQLコマンドのメインエントリポイントとして、新しい演算子ファミリーをシステムカタログに登録する。

シグネチャまたは形状

ObjectAddress DefineOpFamily(CreateOpFamilyStmt *stmt)

入力と出力

  • **入力**: CreateOpFamilyStmt *stmt（SQLパースツリーから生成された演算子ファミリー定義情報）
  • **出力**: ObjectAddress（作成された演算子ファミリーのカタログ上の識別子）

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要。
  • 指定された名前空間（スキーマ）に対する CREATE 権限が必要。
  • 指定されたアクセスメソッド（AM）が存在すること。

アルゴリズムまたは振る舞い

1. QualifiedNameGetCreationNamespace を呼び出し、対象の演算子ファミリー名と名前空間のOIDを特定する。 2. object_aclcheck を使用し、現在のユーザーが対象名前空間にオブジェクトを作成する権限を持つか検証する。権限がない場合はエラーを投げる。 3. get_index_am_oid を呼び出し、指定されたアクセスメソッドのOIDを取得する。存在しない場合はエラーを投げる。 4. superuser() をチェックし、スーパーユーザーでない場合は ERRCODE_INSUFFICIENT_PRIVILEGE でエラーを投げる。 5. CreateOpFamily を呼び出し、pg_opfamily カタログへのエントリ挿入を実行してその結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（SQLコマンド実行のディスパッチャ）
  • **呼び出し先**:
  • QualifiedNameGetCreationNamespace
  • object_aclcheck
  • get_index_am_oid
  • superuser
  • CreateOpFamily

関連データ構造

  • CreateOpFamilyStmt: SQLの CREATE OPERATOR FAMILY 文を表現する構造体。
  • ObjectAddress: 作成されたオブジェクトを特定するための識別子。

失敗モードと境界条件

  • **権限エラー**: 名前空間への CREATE 権限不足、または非スーパーユーザーによる実行時にエラーが発生する。
  • **存在エラー**: 指定されたアクセスメソッドが存在しない場合にエラーが発生する。

保守担当者向けメモ

  • 本関数は CREATE OPERATOR FAMILY の検証と権限チェックを担当し、実際のカタログ操作は CreateOpFamily に委譲している。
  • アクセスメソッドに対する権限チェックの必要性については、ソースコード内に XXX コメントとして検討事項が残されている。
