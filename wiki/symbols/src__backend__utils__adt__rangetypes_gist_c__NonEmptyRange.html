NonEmptyRange

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/rangetypes_gist.c:94

目的

GiSTインデックスの分割処理（picksplit）において、範囲の境界をソートおよび評価するために使用される一時的なデータ構造です。

シグネチャまたは形状

typedef struct
{
	RangeBound	lower;
	RangeBound	upper;
} NonEmptyRange;

入力と出力

本構造体は、主に range_gist_picksplit 内で範囲データを一時的に保持・ソートするために使用されます。

  • **入力**: 既存の範囲データから抽出された RangeBound のペア。
  • **出力**: ソート済み配列（by_lower または by_upper）の要素として利用されます。

前提条件と不変条件

  • 空ではない範囲（Non-Empty Range）を表現することを前提としています。
  • lower および upper フィールドは RangeBound 型であり、範囲の開始点と終了点を保持します。

アルゴリズムまたは振る舞い

GiSTインデックスの二重ソート分割（double sorting split）アルゴリズムにおいて、以下の処理で利用されます。 1. インデックスエントリから NonEmptyRange の配列を生成し、メモリ確保（palloc）を行います。 2. qsort_arg を使用し、lower 境界および upper 境界に基づいてそれぞれソートを行います。 3. ソートされた結果に基づき、インデックスページの分割位置を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/rangetypes_gist.c 内の range_gist_picksplit 関数。
  • **呼び出し先**: qsort_arg（ソート用）、palloc（メモリ確保）、memcpy（データコピー）。

関連データ構造

  • RangeBound: 範囲の境界値を保持する構造体。
  • RangeType: 本ファイルが扱う対象である範囲型。

失敗モードと境界条件

  • メモリ不足時に palloc がエラーを発生させる可能性があります。
  • 範囲が「空」である場合、本構造体で正しく表現できるかは RangeBound の状態に依存します（本構造体は「空ではない」ことを前提とした処理で利用されます）。

保守担当者向けメモ

  • 本構造体は rangetypes_gist.c 内のローカルな最適化用構造体であり、外部モジュールからは参照されません。
  • qsort_arg の比較関数（1746行目以降）でキャストして使用されているため、構造体のメモリレイアウトを変更する場合は比較関数の実装も併せて確認してください。
