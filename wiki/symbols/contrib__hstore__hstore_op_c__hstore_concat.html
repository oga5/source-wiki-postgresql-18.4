hstore_concat

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_op.c:470

目的

2つの hstore 値を結合し、新しい hstore を生成する。キーが重複する場合は、第2引数（s2）の値が優先される。

シグネチャまたは形状

Datum hstore_concat(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの HStore 構造体ポインタを受け取る。
  • **出力**: 結合された結果を含む新しい HStore 構造体へのポインタを Datum として返す。

前提条件と不変条件

  • 入力される s1 および s2 は、内部的にキーの長さと内容でソートされている必要がある。
  • 戻り値の HStore は palloc によってメモリ確保される。

アルゴリズムまたは振る舞い

1. 入力 s1, s2 のサイズに基づき、出力用のメモリ領域を確保する。 2. いずれかの入力が空の場合、もう一方のコピーを返して終了する。 3. 両方の hstore をマージソートのように走査する。 4. キーの比較を行い、以下のルールで出力へコピーする:

  • s1 のキーが小さい場合: s1 の要素をコピー。
  • s2 のキーが小さい場合: s2 の要素をコピー。
  • キーが一致する場合: s2 の要素をコピーし、s1 のインデックスも進める（s2 の値を優先）。

    • 5. HS_FINALIZE マクロを用いて、最終的なサイズとエントリ数を調整し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_op.c 内で HSTORE_POLLUTE マクロにより登録されている。
  • contrib/hstore/hstore_subs.c:189 に言及あり。
  • **呼び出し先**:
  • palloc (メモリ確保)
  • memcpy (メモリコピー)
  • HS_COUNT, HS_SETCOUNT, HS_FIXSIZE, HS_COPYITEM, HS_FINALIZE などの hstore.h 定義マクロ。
  • HSTORE_KEY, HSTORE_KEYLEN, HSTORE_VALLEN, HSTORE_VALISNULL などのアクセサマクロ。

関連データ構造

  • HStore: hstore 型の内部表現。
  • HEntry: hstore 内の各エントリ（キーと値のメタデータ）を保持する構造体。

失敗モードと境界条件

  • 入力が空（s1count == 0 または s2count == 0）の場合、コピー処理により正常終了する。
  • メモリ不足時には palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数はマージソートのロジックに基づいているため、hstore のソート順序（キーの長さ優先、次に内容）に変更を加える場合は、比較ロジックの修正が必要となる。
  • HS_COPYITEM 等のマクロは、hstore の内部レイアウト（ヘッダー、エントリ配列、文字列領域）に強く依存しているため、構造変更時は注意が必要。
