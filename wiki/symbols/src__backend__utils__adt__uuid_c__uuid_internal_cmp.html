uuid_internal_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/uuid.c:203

目的

2つの pg_uuid_t 構造体に含まれる UUID バイナリデータを比較し、その順序関係を判定します。

シグネチャまたは形状

static int uuid_internal_cmp(const pg_uuid_t *arg1, const pg_uuid_t *arg2)

入力と出力

  • **入力**: 比較対象となる2つの pg_uuid_t 構造体へのポインタ (arg1, arg2)。
  • **出力**: 比較結果を示す整数。
  • 0: 両者が等しい場合。
  • 0 未満: arg1 が arg2 より小さい場合。
  • 0 より大きい: arg1 が arg2 より大きい場合。

前提条件と不変条件

  • arg1 および arg2 は有効な pg_uuid_t 構造体を指している必要があります。
  • UUID は 16 バイトの固定長データとして扱われます。

アルゴリズムまたは振る舞い

内部的に memcmp を呼び出し、pg_uuid_t 構造体内の data メンバ（16 バイト）をバイト単位で直接比較します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/uuid.c 内の比較関数（例: uuid_lt 等の内部実装）。
  • contrib/btree_gist/btree_uuid.c 内の各種比較演算および GIST インデックス操作関数。
  • **呼び出し先**:
  • 標準 C ライブラリの memcmp。

関連データ構造

  • pg_uuid_t: UUID を保持する 16 バイトの構造体（src/include/utils/uuid.h で定義）。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、未定義の動作を引き起こします。
  • 比較はバイナリレベルで行われるため、UUID の論理的なバージョンやタイムスタンプの順序ではなく、メモリ上のバイト列としての辞書順序に従います。

保守担当者向けメモ

  • 本関数は static であり、モジュール内および contrib/btree_gist からの利用が確認されています。
  • 比較ロジックを変更する場合は、インデックスの整合性に影響を与える可能性があるため注意してください。
