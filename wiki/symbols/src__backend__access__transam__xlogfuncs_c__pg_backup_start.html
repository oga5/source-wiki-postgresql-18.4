pg_backup_start

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogfuncs.c:56

目的

オンラインバックアップを開始するためのSQLインターフェースを提供します。

シグネチャまたは形状

Datum pg_backup_start(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_PP(0): バックアップラベル（text型）。
  • PG_GETARG_BOOL(1): fastフラグ（チェックポイントを即時実行するかどうか）。
  • **出力**:
  • PG_RETURN_LSN: バックアップ開始位置を示すLSN（Log Sequence Number）。

前提条件と不変条件

  • 同一セッション内で既にバックアップが実行中の場合、エラーが発生します。
  • backupcontext（メモリコンテキスト）は、pg_backup_stopが呼ばれるまでバックアップ状態を保持するために使用されます。

アルゴリズムまたは振る舞い

1. 引数からバックアップラベルを取得し、現在のセッション状態を確認します。 2. backupcontextが未作成であれば作成し、既存であればリセットします。 3. backup_stateおよびtablespace_mapをbackupcontext上に確保します。 4. register_persistent_abort_backup_handler()を呼び出し、異常終了時のクリーンアップを登録します。 5. do_pg_backup_start()を呼び出し、バックアップの物理的な開始処理を実行します。 6. 確定したバックアップ開始位置（startpoint）を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由でユーザーから呼び出されます。
  • **呼び出し先**:
  • do_pg_backup_start（src/backend/access/transam/xlog.c）: バックアップ開始の主要ロジック。
  • register_persistent_abort_backup_handler: 異常終了時のハンドラ登録。

関連データ構造

  • BackupState: バックアップの進行状況を保持する構造体。
  • backupcontext: バックアップ期間中に生存するメモリコンテキスト。
  • tablespace_map: テーブルスペース情報を保持するStringInfo。

失敗モードと境界条件

  • セッション内で既にバックアップが実行中の場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATEでエラーとなります。
  • backupcontextの管理により、バックアップ終了前にエラーが発生した場合、メモリリークが発生する可能性があります（次回pg_backup_start呼び出し時にリセットされます）。

保守担当者向けメモ

  • 本関数はdo_pg_backup_startのラッパーです。
  • backupcontextのライフサイクル管理はpg_backup_stopと密接に関連しているため、変更時は両方の関数を確認してください。
