find_computable_ec_member

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/equivclass.c:990

目的

指定された式リスト（exprs）に含まれる変数のみで構成され、かつ指定されたリレーション（relids）で計算可能な EquivalenceMember を等価クラス（ec）内から検索します。

シグネチャまたは形状

EquivalenceMember *find_computable_ec_member(PlannerInfo *root, EquivalenceClass *ec, List *exprs, Relids relids, bool require_parallel_safe)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • ec: 検索対象の等価クラス。
  • exprs: 計算の基礎となる式リスト。
  • relids: 計算に使用可能なリレーションの集合。
  • require_parallel_safe: 並列安全性を要求するかどうかのフラグ。
  • **出力**:
  • 条件を満たす EquivalenceMember へのポインタ。見つからない場合は NULL。

前提条件と不変条件

  • ec 内のメンバーが定数（em_is_const）である場合は無視されます。
  • 子メンバー（em_is_child）の場合、そのリレーションが relids のサブセットである必要があります。

アルゴリズムまたは振る舞い

1. exprs から含まれるすべての変数（Vars および準変数）を抽出します。 2. ec 内の各メンバーをイテレータで順次走査します。 3. 各メンバーの式に含まれる変数が、ステップ1で抽出した変数リストのサブセットであるかを確認します。 4. require_parallel_safe が true の場合、is_parallel_safe を用いて式の並列安全性を検証します（コストが高いため最後に実行）。 5. すべての条件を満たす最初のメンバーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/equivclass.c:1124
  • src/backend/optimizer/plan/createplan.c:6411
  • **呼び出し先**:
  • pull_var_clause（変数抽出）
  • setup_eclass_member_iterator / eclass_member_iterator_next（メンバー走査）
  • is_parallel_safe（並列安全性チェック）
  • bms_is_subset（リレーション包含判定）

関連データ構造

  • EquivalenceClass: 等価関係を保持する構造体。
  • EquivalenceMember: 等価クラス内の個別の式を保持する構造体。
  • PlannerInfo: プランナのコンテキスト情報。

失敗モードと境界条件

  • ec 内に計算可能なメンバーが存在しない場合、NULL を返します。
  • 定数メンバーは常にスキップされます。
  • 並列安全性が要求されているにもかかわらず、式が並列安全でない場合はそのメンバーをスキップします。

保守担当者向けメモ

  • pull_var_clause を使用して変数の包含関係を判定しているため、複雑な式が含まれる場合のパフォーマンスに注意してください。
  • em_is_child の判定ロジックは、継承テーブル等の特殊なリレーションを扱う際に重要です。
