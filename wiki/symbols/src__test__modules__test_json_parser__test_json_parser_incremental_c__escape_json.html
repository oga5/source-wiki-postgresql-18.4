escape_json

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_json_parser/test_json_parser_incremental.c:359

目的

入力文字列を JSON 形式の文字列リテラルとしてエスケープし、指定された StringInfo バッファに追加する。

シグネチャまたは形状

static void escape_json(StringInfo buf, const char *str)

入力と出力

  • buf: エスケープ後の文字列が書き込まれる StringInfo 型のバッファ（入出力）。
  • str: エスケープ対象のヌル終端文字列（入力）。
  • 戻り値: なし。

前提条件と不変条件

  • buf は初期化済みの StringInfo であること。
  • str は有効なヌル終端文字列であること。
  • 出力される文字列は、常に二重引用符 " で囲まれる。

アルゴリズムまたは振る舞い

1. バッファの先頭に " を追加する。 2. 入力文字列 str を一文字ずつ走査する。 3. 制御文字（\b, \f, \n, \r, \t）、二重引用符 "、およびバックスラッシュ \ を検出した場合、対応するエスケープシーケンス（例: \n は \\n）に変換してバッファに追加する。 4. 制御文字以外の文字で、ASCII 値が 32 未満（スペース未満）のものは \uXXXX 形式の Unicode エスケープに変換する。 5. それ以外の通常の文字はそのままバッファに追加する。 6. 最後にバッファの末尾に " を追加する。

呼び出し元と呼び出し先

  • 呼び出し先: appendStringInfoCharMacro および appendStringInfoString（lib/stringinfo.h）。
  • 呼び出し元: 提供されたソースファイル内での直接的な呼び出し元は特定されていません。なお、ツールによるヒューリスティックな検索では、src/backend/commands/explain_format.c や src/backend/libpq/auth-oauth.c 等の他ファイルからの参照が示唆されています。

関連データ構造

  • StringInfo: PostgreSQL の動的文字列バッファ構造体。

失敗モードと境界条件

  • 入力文字列が空の場合、"" という文字列がバッファに追加される。
  • 制御文字や特殊文字が含まれる場合、適切にエスケープされる。
  • メモリ不足が発生した場合、StringInfo の内部関数がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は static で定義されており、現在のファイル内でのみ使用されることを意図しています。
  • 他のモジュール（src/backend/ 配下など）でも同名の関数が使用されている可能性があるため、変更時は影響範囲に注意してください。
