SimpleStringList

定義

  • 種別: struct
  • 定義箇所: src/include/fe_utils/simple_list.h:40

目的

PostgreSQL のフロントエンドツールにおいて、文字列のリストを管理するための軽量な単方向連結リスト構造を提供します。

シグネチャまたは形状

typedef struct SimpleStringList
{
    SimpleStringListCell *head;
    SimpleStringListCell *tail;
} SimpleStringList;

入力と出力

  • **入力**: リスト操作関数（append 等）の引数として受け取られ、リストの先頭および末尾のポインタを保持します。
  • **出力**: リストの現在の状態（先頭および末尾のセル）を保持するコンテナとして機能します。

前提条件と不変条件

  • pg_malloc を使用してメモリを確保するため、使用後は必ず対応する破棄関数を呼び出し、メモリリークを防ぐ必要があります。
  • リストの各セルは SimpleStringListCell 型であり、文字列をインラインで保持します。

アルゴリズムまたは振る舞い

  • **追加**: append 関数により、リストの末尾に新しいセルが動的に割り当てられ、連結されます。文字列はセル内にコピーされます。
  • **検索**: member 関数による線形探索が行われます。文字列リストの場合、一致した要素の touched フラグが true に更新される副作用があります。
  • **破棄**: destroy 関数によりリストを走査し、各セルを pg_free で解放します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照）**:
  • src/bin/pg_basebackup/pg_createsubscriber.c: データベース名、パブリケーション名、サブスクリプション名、レプリケーションスロット名、オブジェクトタイプのリストとして使用。
  • src/bin/pg_dump/pg_backup.h: インデックス名、関数名、スキーマ名、トリガー名、テーブル名のリストとして使用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、操作には src/fe_utils/simple_list.c で定義された関数群（append, member, destroy 等）が呼び出されます。

関連データ構造

  • SimpleStringListCell: リストの各要素を構成するセル。
  • SimpleOidList / SimplePtrList: 同様の設計思想に基づく OID およびポインタ用リスト構造体。

失敗モードと境界条件

  • メモリ不足時には pg_malloc がエラーを発生させます。
  • 空のリストに対して検索や破棄を行った場合の挙動は、各操作関数の実装に依存します。

保守担当者向けメモ

  • バックエンドの List とは異なり、機能は最小限に限定されています。複雑な操作が必要な場合は別のデータ構造を検討してください。
  • 文字列リストの touched フラグは、全要素が処理されたかを確認する際に有用です。
