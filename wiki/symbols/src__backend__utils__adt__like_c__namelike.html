namelike

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like.c:241

目的

SQLの LIKE 演算子を name 型に対して適用し、パターンマッチングを実行します。

シグネチャまたは形状

Datum namelike(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_NAME(0): 対象となる name 型の文字列。
  • PG_GETARG_TEXT_PP(1): パターンとして使用される text 型の文字列。
  • **出力**:
  • PG_RETURN_BOOL(result): パターンに一致した場合は true、一致しない場合は false を返します。

前提条件と不変条件

  • 入力された name 型の文字列は NameStr マクロにより char * として取得されます。
  • パターン文字列は VARDATA_ANY を介してアクセスされ、TOAST展開が考慮されます。
  • 照合順序（Collation）は PG_GET_COLLATION() を通じて取得され、マッチングロジックに渡されます。

アルゴリズムまたは振る舞い

1. name 型の文字列から C 文字列ポインタと長さを取得します。 2. text 型のパターン文字列からデータポインタと長さを取得します。 3. GenericMatchText 関数を呼び出し、データベースのエンコーディングに応じた適切なマッチングアルゴリズムへ処理を委譲します。 4. GenericMatchText の戻り値が LIKE_TRUE であるかどうかを判定し、結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（演算子 LIKE の実行時）。
  • **呼び出し先**:
  • GenericMatchText: 実際のエンコーディング判定とマッチング関数のディスパッチを行います。

関連データ構造

  • Name: PostgreSQLの固定長文字列型（name）。
  • text: PostgreSQLの可変長文字列型。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • PG_GETARG_NAME および PG_GETARG_TEXT_PP は、引数が不正な場合にエラーを発生させます。
  • GenericMatchText 内でエンコーディングに応じたエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は name 型専用のラッパーです。同様のロジックが textlike や bytealike でも実装されています。
  • マッチングの核心部分は like_match.c に依存しており、エンコーディングごとの最適化が必要な場合は like_match.c の修正を検討してください。
