PGTYPESinterval_from_asc

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/interval.c:1002

目的

文字列形式の時間間隔（interval）を解析し、ECPG で使用される interval 構造体の内部表現に変換する。

シグネチャまたは形状

interval *PGTYPESinterval_from_asc(char *str, char **endptr)

入力と出力

  • **入力**:
  • str: 解析対象の文字列（ISO 8601 形式または PostgreSQL 独自形式）。
  • endptr: 解析終了位置を受け取るポインタ（NULL 指定可能）。
  • **出力**:
  • 成功時: 変換された interval 構造体へのポインタ。
  • 失敗時: NULL を返し、errno に PGTYPES_INTVL_BAD_INTERVAL を設定。

前提条件と不変条件

  • str の長さは MAXDATELEN 以下であること。
  • 内部で使用される tm 構造体は初期化時にすべて 0 に設定される。
  • 成功時には errno は 0 にリセットされる。

アルゴリズムまたは振る舞い

1. 入力文字列の長さを検証し、超過している場合はエラーを返す。 2. ParseDateTime を呼び出し、文字列をトークン化する。 3. DecodeInterval または DecodeISO8601Interval を使用して、トークンを struct tm（年・月・日・時・分・秒）および fsec_t（小数秒）に変換する。 4. pgtypes_alloc で interval 構造体用のメモリを確保する。 5. 解析結果が時間間隔（DTK_DELTA）であることを確認する。 6. tm2interval を呼び出し、tm と fsec_t の値を interval 構造体の内部形式に集約する。 7. 確保したメモリを返し、成功を示す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/data.c
  • 各種テストコード（compat_informix-test_informix2.c, pgtypeslib-dt_test.c 等）
  • **呼び出し先**:
  • ParseDateTime, DecodeInterval, DecodeISO8601Interval, tm2interval
  • pgtypes_alloc, free

関連データ構造

  • interval: 時間間隔を保持する構造体。
  • struct tm: 日時要素を保持する標準構造体。
  • fsec_t: 小数秒を保持する型。

失敗モードと境界条件

  • **入力長超過**: strlen(str) > MAXDATELEN の場合、PGTYPES_INTVL_BAD_INTERVAL を設定して NULL を返す。
  • **解析失敗**: ParseDateTime や Decode 系関数が失敗した場合、PGTYPES_INTVL_BAD_INTERVAL を設定して NULL を返す。
  • **メモリ確保失敗**: pgtypes_alloc が失敗した場合、NULL を返す。
  • **型不整合**: 解析結果が DTK_DELTA でない場合、メモリを解放し PGTYPES_INTVL_BAD_INTERVAL を設定して NULL を返す。

保守担当者向けメモ

  • 本関数は PostgreSQL バックエンドのロジックを移植したものであり、struct tm を使用している点に注意すること。
  • endptr が NULL の場合は、関数内部で一時的なポインタを使用して処理を完結させる。
