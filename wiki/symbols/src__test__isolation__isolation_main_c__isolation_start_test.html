isolation_start_test

定義

  • 種別: function
  • 定義箇所: src/test/isolation/isolation_main.c:28

目的

個別の分離テストケースを実行するために、isolationtester バイナリを外部プロセスとして起動する。

シグネチャまたは形状

static PID_TYPE isolation_start_test(
    const char *testname,
    _stringlist **resultfiles,
    _stringlist **expectfiles,
    _stringlist **tags
)

入力と出力

  • **入力**:
  • testname: 実行するテストケース名。
  • resultfiles, expectfiles, tags: テスト結果、期待値ファイル、およびタグ情報を管理するための文字列リストへのポインタ。
  • **出力**:
  • 戻り値: 起動したプロセスの PID_TYPE。

前提条件と不変条件

  • isolationtester バイナリのパスは、初回呼び出し時に find_other_exec を用いて解決される。
  • 入力ファイル（.spec）および期待値ファイル（.out）は、outputdir を優先的に探索し、存在しない場合は inputdir を探索する。
  • 環境変数 PGAPPNAME は、テスト実行中のみ isolation/<testname> に設定される。

アルゴリズムまたは振る舞い

1. **バイナリ検索**: looked_up_isolation_exec フラグを確認し、未実行であれば isolationtester の場所を特定する。 2. **パス解決**: テスト用入力ファイルおよび結果出力ファイルのパスを構築する。 3. **リスト登録**: resultfiles および expectfiles に構築したパスを追加する。 4. **コマンド構築**: StringInfo を使用して isolationtester 実行コマンドを生成する（launcher が設定されている場合は先頭に付与）。 5. **プロセス起動**: spawn_process を呼び出し、標準入力を .spec ファイルから、標準出力を .out ファイルへリダイレクトして実行する。 6. **環境制御**: 実行前後に PGAPPNAME を設定・解除する。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_regress フレームワーク（ソースコード上の明示的な呼び出し元関数名は提供されていません）。
  • **呼び出し先**: find_other_exec, file_exists, add_stringlist_item, initStringInfo, appendStringInfo, psprintf, setenv, spawn_process, unsetenv, pfree。

関連データ構造

  • StringInfoData: コマンド文字列の構築に使用。
  • _stringlist: ファイルパスやタグの管理に使用。

失敗モードと境界条件

  • isolationtester バイナリが見つからない場合、エラーメッセージを出力して終了コード 2 でプロセスを終了する。
  • spawn_process が失敗し INVALID_PID が返された場合、エラーメッセージを出力して終了コード 2 でプロセスを終了する。

保守担当者向けメモ

  • isolation_init ではなく本関数内でバイナリ検索を行うのは、pg_regress の初期化段階ではライブラリパスが不完全である可能性があるためである。
  • ファイル探索において outputdir を優先する設計は、ビルド外でのテスト実行をサポートするためのものである。
