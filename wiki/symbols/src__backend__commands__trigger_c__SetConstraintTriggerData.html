SetConstraintTriggerData

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/trigger.c:3608

目的

制約トリガーの識別情報と、そのトリガーが遅延実行対象であるかどうかの状態を保持するために使用されるデータ構造です。

シグネチャまたは形状

typedef struct SetConstraintTriggerData
{
	Oid			sct_tgoid;        /* トリガーのOID */
	bool		sct_tgisdeferred; /* 遅延実行されるか否か */
} SetConstraintTriggerData;

入力と出力

本構造体はデータ保持用であり、関数としての入出力は持ちません。メモリ割り当て時には、numalloc や numstates に基づくサイズ計算（sizeof(SetConstraintTriggerData)）が行われます。

前提条件と不変条件

  • sct_tgoid は pg_trigger カタログ内の有効なトリガーを指す必要があります。
  • sct_tgisdeferred は、当該トリガーが SET CONSTRAINTS コマンドによる遅延実行の対象であるかどうかを反映します。

アルゴリズムまたは振る舞い

本構造体は、トリガーの状態を配列として管理する際の各要素として機能します。実行エンジンがトリガーの有効性や遅延状態を評価する際、この構造体を通じて各トリガーの属性を参照します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/trigger.c 内のトリガー状態管理関数群（メモリ割り当てや状態更新を行う箇所）から参照されています。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。
  • **注記**: 具体的な関数レベルの呼び出し関係は、提供されたソース情報からは特定できません。

関連データ構造

  • SetConstraintTrigger: 本構造体へのポインタ型（typedef struct SetConstraintTriggerData *SetConstraintTrigger;）。
  • pg_trigger: トリガーのメタデータを格納するシステムカタログ。

失敗モードと境界条件

  • メモリ割り当て時、FLEXIBLE_ARRAY_MEMBER を含む配列として扱われる場合があり、サイズ計算の誤りはバッファオーバーフローを引き起こす可能性があります。
  • 参照する sct_tgoid が無効な場合、トリガー実行時に予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は src/backend/commands/trigger.c 内で定義されており、主に SET CONSTRAINTS 関連の処理で使用されます。
  • 構造体の変更は、トリガーの遅延実行ロジック全体に影響を及ぼすため注意が必要です。
