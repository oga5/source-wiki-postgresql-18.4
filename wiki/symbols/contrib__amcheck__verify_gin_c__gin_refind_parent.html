gin_refind_parent

定義

  • 種別: function
  • 定義箇所: contrib/amcheck/verify_gin.c:719

目的

GINインデックスの検証中に、ページ分割（Split）等によって親ページと子ページのリンク関係が一時的に不整合となった際、指定された親ブロックから子ブロックへのダウンリンクを持つインデックスエントリを再探索して取得します。

シグネチャまたは形状

static IndexTuple gin_refind_parent(Relation rel, BlockNumber parentblkno, BlockNumber childblkno, BufferAccessStrategy strategy)

入力と出力

  • **入力**:
  • rel: 対象のリレーション。
  • parentblkno: 探索対象の親ページブロック番号。
  • childblkno: 検索対象の子ページブロック番号。
  • strategy: バッファアクセスのための戦略。
  • **出力**:
  • IndexTuple: 見つかった場合は該当エントリのコピーを返し、見つからない場合や親がリーフページの場合は NULL を返します。

前提条件と不変条件

  • parentblkno で指定されたページは、GINインデックスの有効なページである必要があります。
  • 呼び出し元は、検証対象のインデックスに対して AccessShareLock を保持していることが想定されています。

アルゴリズムまたは振る舞い

1. ReadBufferExtended を使用して親ページを読み込み、GIN_SHARE ロックを取得します。 2. 親ページがリーフページ（GinPageIsLeaf）である場合は、探索を行わずにロックを解放して NULL を返します。 3. 親ページが内部ページである場合、ページ内の全オフセットを走査します。 4. 各オフセットのインデックスエントリ（IndexTuple）から GinGetDownlink を取得し、childblkno と一致するか確認します。 5. 一致するエントリが見つかった場合、CopyIndexTuple でそのコピーを作成し、ロックを解放して返します。 6. 走査終了まで見つからない場合は、ロックを解放して NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_gin.c 内の検証ロジック（例: stack->parenttup の再取得時）。
  • **呼び出し先**: ReadBufferExtended, LockBuffer, BufferGetPage, GinPageIsLeaf, PageGetMaxOffsetNumber, PageGetItemIdCareful, PageGetItem, GinGetDownlink, CopyIndexTuple, UnlockReleaseBuffer。

関連データ構造

  • Relation: インデックスのリレーション情報。
  • IndexTuple: GINインデックスの各エントリ構造。
  • Buffer: ページ読み込みに使用されるバッファ。

失敗モードと境界条件

  • 親ページがリーフページである場合、ダウンリンクを持たないため常に NULL を返します。
  • ページ内のエントリをすべて走査しても childblkno に一致するダウンリンクが見つからない場合、NULL を返します。
  • PageGetItemIdCareful が内部でエラーを発生させた場合、検証プロセス全体が中断される可能性があります。

保守担当者向けメモ

  • 本関数は並行処理によるインデックス更新を考慮した再探索用です。検証中にインデックスが変更される可能性があるため、戻り値が NULL の場合のハンドリングを呼び出し元で適切に行う必要があります。
