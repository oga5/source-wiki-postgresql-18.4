calc_inet_union_params

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_gist.c:344

目的

GiSTインデックスのノード統合処理において、複数の GistInetKey から共通のプレフィックス長および最小ネットマスク長を計算し、親ノードの代表値を決定する。

シグネチャまたは形状

static void calc_inet_union_params(GISTENTRY *ent, int m, int n, int *minfamily_p, int *maxfamily_p, int *minbits_p, int *commonbits_p)

入力と出力

  • **入力**:
  • ent: GISTENTRY 配列。
  • m, n: 処理対象となる配列の開始および終了インデックス。
  • **出力**:
  • minfamily_p, maxfamily_p: 統合対象キーのアドレスファミリー（IPv4/IPv6）の範囲。
  • minbits_p: 統合対象キーにおける最小ネットマスク長。
  • commonbits_p: 統合対象キー間で共有される共通プレフィックス長。

前提条件と不変条件

  • m <= n であること（少なくとも1つのキーが存在すること）。
  • 複数のアドレスファミリーが混在する場合、minbits および commonbits は強制的に 0 となる。

アルゴリズムまたは振る舞い

1. 最初のキー（ent[m]）を使用して、minfamily, maxfamily, minbits, commonbits を初期化する。 2. m + 1 から n までのキーを順次走査する。 3. 各キーのアドレスファミリーを比較し、ファミリーの最小値と最大値を更新する。 4. 各キーの minbits を比較し、最小値を保持する。 5. commonbits を比較し、bitncommon 関数を用いて、現在の共通プレフィックス長と各キーのアドレスを比較して値を更新する。 6. 全てのキーを走査後、minfamily != maxfamily であれば、minbits と commonbits を 0 にリセットする。 7. 計算結果をポインタ引数経由で返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network_gist.c:345 (自身)
  • src/backend/utils/adt/network_gist.c:518
  • src/backend/utils/adt/network_gist.c:695
  • **呼び出し先**:
  • DatumGetInetKeyP (マクロ)
  • gk_ip_family, gk_ip_minbits, gk_ip_commonbits, gk_ip_addr (マクロ)
  • bitncommon (関数)

関連データ構造

  • GistInetKey: GiSTインデックスで使用されるネットワークアドレスの内部表現。
  • GISTENTRY: GiSTインデックスの各エントリを保持する構造体。

失敗モードと境界条件

  • m > n の場合、Assert により停止する。
  • 異なるアドレスファミリー（IPv4とIPv6）が混在する場合、インデックスの共通パラメータは無効化（0）される。

保守担当者向けメモ

  • 本関数は inet_gist_union や inet_gist_picksplit 等のGiSTサポート関数から呼び出され、インデックスの階層構造を維持するための重要な計算ロジックを担う。
  • bitncommon の結果が commonbits に影響を与えるため、ビット単位の比較ロジックを変更する際は注意が必要。
