CreateSchemaStmtContext

定義

  • 種別: struct
  • 定義箇所: src/backend/parser/parse_utilcmd.c:99

目的

CREATE SCHEMA 文の解析および変換処理において、スキーマ内に含まれる各オブジェクト（テーブル、ビュー、インデックス等）の定義を一時的に保持し、管理するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    const char *schemaname;     /* スキーマ名 */
    List       *sequences;      /* CREATE SEQUENCE アイテム */
    List       *tables;         /* CREATE TABLE アイテム */
    List       *views;          /* CREATE VIEW アイテム */
    List       *indexes;        /* CREATE INDEX アイテム */
    List       *triggers;       /* CREATE TRIGGER アイテム */
    List       *grants;         /* GRANT アイテム */
} CreateSchemaStmtContext;

入力と出力

  • 本構造体は、CREATE SCHEMA 文の解析中に状態を保持するために使用されます。
  • 各フィールドは List 型であり、解析過程で生成された各コマンドノードが蓄積されます。

前提条件と不変条件

  • schemaname はスキーマ名を指す文字列へのポインタである必要があります。
  • 各リストフィールドは、解析処理の開始時に空（NIL）であるか、適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. CREATE SCHEMA 文の解析時、この構造体のインスタンスが生成されます。 2. スキーマ定義内の各要素（テーブル作成、権限付与など）が解析されるたび、対応するリストフィールドにノードが追加されます。 3. 最終的に、これらのリストに格納された各コマンドが順次実行されることで、スキーマの構築が完了します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/parser/parse_utilcmd.c 内の CREATE SCHEMA 文を処理する関数（例: transformCreateSchemaStmtElements 等の文脈）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • List: PostgreSQL の汎用リスト構造体。各フィールドの保持に使用。

失敗モードと境界条件

  • メモリ不足: 大規模なスキーマ定義でリストが肥大化した場合、メモリ割り当てに失敗する可能性があります。
  • 構文エラー: 解析対象の各要素が不正な場合、リストへの追加前にエラーが発生します。

保守担当者向けメモ

  • 本構造体は CREATE SCHEMA の変換処理に特化しています。スキーマ内のオブジェクト種別が増えた場合は、本構造体にフィールドを追加する必要があります。
  • src/backend/parser/parse_utilcmd.c:4113 でローカル変数としてインスタンス化されている箇所が参照例となります。
