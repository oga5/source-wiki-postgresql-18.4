sepgsql_subxact_callback

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/label.c:203

目的

サブトランザクションのロールバック時に、そのサブトランザクションに関連付けられた保留中のセキュリティラベル（pending_label）をリストから削除し、整合性を維持する。

シグネチャまたは形状

static void sepgsql_subxact_callback(SubXactEvent event, SubTransactionId mySubid, SubTransactionId parentSubid, void *arg)

入力と出力

  • event: 発生したサブトランザクションイベント（SubXactEvent）。
  • mySubid: 対象のサブトランザクションID。
  • parentSubid: 親トランザクションID（本関数内では未使用）。
  • arg: ユーザー定義データ（本関数内では未使用）。
  • 出力: なし（副作用として client_label_pending リストを更新）。

前提条件と不変条件

  • client_label_pending リストが初期化されていること。
  • 本関数は RegisterSubXactCallback を介してサブトランザクションイベントの通知を受ける前提で動作する。

アルゴリズムまたは振る舞い

1. event が SUBXACT_EVENT_ABORT_SUB であるかを確認する。 2. 該当する場合、client_label_pending リストを走査する。 3. リスト内の各 pending_label 要素の subid が mySubid と一致する場合、その要素をリストから削除する。 4. アボート以外のイベント（SUBXACT_EVENT_START_SUB や SUBXACT_EVENT_COMMIT_SUB 等）に対しては何も行わない。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQL のサブトランザクション管理機構（RegisterSubXactCallback により登録）。
  • 呼び出し先: foreach_delete_current（リスト操作用マクロ）。

関連データ構造

  • pending_label: 保留中のラベル情報を保持する構造体。
  • client_label_pending: 保留中のラベルを管理するグローバルなリスト。

失敗モードと境界条件

  • リストが空の場合、走査は即座に終了する。
  • 指定された mySubid を持つ要素が存在しない場合、リストに変更は加えられない。

保守担当者向けメモ

  • 本関数はサブトランザクションのアボート時のみを対象としている。コミット時の処理は sepgsql_xact_callback 等の別機構が担うため、混同しないこと。
  • client_label_pending リストのメモリ管理は呼び出し元および他のラベル管理関数と整合している必要がある。
