CommitTsShmemBuffers

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/commit_ts.c:505

目的

コミットタイムスタンプ機能で使用するSLRU（Simple Least Recently Used）バッファ数を決定する。

シグネチャまたは形状

int CommitTsShmemBuffers(void)

入力と出力

  • **入力**: なし（グローバル変数 commit_timestamp_buffers を参照）。
  • **出力**: 決定されたバッファ数（int 型）。

前提条件と不変条件

  • commit_timestamp_buffers が 0 の場合、共有バッファ数に基づく自動チューニングが行われる。
  • 返される値は、最小値 16 および最大値 SLRU_MAX_ALLOWED_BUFFERS の範囲内に制限される。

アルゴリズムまたは振る舞い

1. commit_timestamp_buffers が 0 の場合、SimpleLruAutotuneBuffers(512, 1024) を呼び出し、共有バッファ数に応じた値を算出する。 2. commit_timestamp_buffers が 0 以外の場合、その値を 16 以上かつ SLRU_MAX_ALLOWED_BUFFERS 以下にクランプ（制限）して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • CommitTsShmemSize (src/backend/access/transam/commit_ts.c:521)
  • CommitTsShmemInit (src/backend/access/transam/commit_ts.c:539, 556)
  • **呼び出し先**:
  • SimpleLruAutotuneBuffers (条件付き)
  • Min, Max (マクロ)

関連データ構造

  • CommitTsCtl (SLRU制御構造体)

失敗モードと境界条件

  • commit_timestamp_buffers に極端に小さい値や大きい値が設定された場合でも、Min/Max マクロにより安全な範囲（16 ～ SLRU_MAX_ALLOWED_BUFFERS）に収束される。

保守担当者向けメモ

  • 本関数は共有メモリのサイズ計算および初期化時に呼び出されるため、設定変更を反映させるには再起動が必要となる。
  • SimpleLruAutotuneBuffers の引数（512, 1024）は、PostgreSQLの共有バッファ設定に対するヒューリスティックなデフォルト値である。
