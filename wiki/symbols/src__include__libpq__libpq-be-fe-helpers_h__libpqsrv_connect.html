libpqsrv_connect

定義

  • 種別: function
  • 定義箇所: src/include/libpq/libpq-be-fe-helpers.h:65

目的

PostgreSQL バックエンドプロセス内から外部データベースへの接続を確立するためのラッパー関数です。バックエンドのファイル記述子（FD）管理ルールを遵守し、接続待機中に割り込み処理を可能にします。

シグネチャまたは形状

static inline PGconn *libpqsrv_connect(const char *conninfo, uint32 wait_event_info)

入力と出力

  • **入力**:
  • conninfo: 接続文字列（PQconnectdb と同様）。
  • wait_event_info: 待機イベントの識別子（utils/wait_event.h で定義）。
  • **出力**:
  • PGconn *: 確立された接続オブジェクト。接続失敗時も libpq の仕様に従いオブジェクトが返される場合がある。

前提条件と不変条件

  • バックエンドの FD 制限内で動作するため、事前に libpqsrv_connect_prepare が呼び出され、FD が確保されている必要がある。
  • 呼び出し側は、接続確立後のリソース解放に libpqsrv_disconnect を使用することが推奨される。

アルゴリズムまたは振る舞い

1. libpqsrv_connect_prepare() を呼び出し、バックエンドの FD 管理下でリソースを準備する。 2. PQconnectStart(conninfo) を呼び出し、非ブロッキング接続を開始する。 3. libpqsrv_connect_internal(conn, wait_event_info) を呼び出し、接続が完了するまで WaitLatchOrSocket を用いて待機する。この待機ループ内で CHECK_FOR_INTERRUPTS() が実行され、クエリキャンセル等のシグナルに応答する。 4. 確立された PGconn ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c (複数箇所)
  • **呼び出し先**:
  • libpqsrv_connect_prepare
  • PQconnectStart
  • libpqsrv_connect_internal

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • wait_event_info: 待機イベントを追跡するための識別子。

失敗モードと境界条件

  • FD 制限に達している場合、libpqsrv_connect_prepare 内でエラーが発生する可能性がある。
  • 接続文字列が不正な場合やサーバーが到達不能な場合、PQconnectStart が失敗状態の PGconn を返す。
  • 待機中に割り込み（クエリキャンセル等）が発生した場合、libpqsrv_connect_internal 内で処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーオンリーのライブラリとして提供される。
  • バックエンド本体のビルド時には libpq の使用が禁止されているため、拡張機能開発時のみ利用可能である。
  • 現在の接続処理は完全な非ブロッキングモードではないため、将来的な改善が推奨されている。
