MemoizeTuple

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeMemoize.c:94

目的

パラメータ化されたプランノードの実行結果として得られたタプルを、キャッシュエントリの一部として保持するために使用されます。

シグネチャまたは形状

typedef struct MemoizeTuple
{
    MinimalTuple mintuple;      /* キャッシュされたタプル */
    struct MemoizeTuple *next;  /* 同一パラメータ値を持つ次のタプル、または最後の場合はNULL */
} MemoizeTuple;

入力と出力

  • 入力: palloc を介してメモリが割り当てられ、mintuple にキャッシュ対象のタプルが格納されます。
  • 出力: キャッシュ検索時に entry->tuplehead を起点として、連結リストを辿ることでキャッシュされたタプル群が取得されます。

前提条件と不変条件

  • next ポインタは、同一のパラメータセットに対応するタプルの連結リストを形成します。
  • リストの終端は NULL で示されます。

アルゴリズムまたは振る舞い

  • キャッシュ検索時、MemoizeEntry が保持する tuplehead から順次 next を辿ることで、キャッシュされた全タプルにアクセスします。
  • 新規タプルの追加時には、palloc でメモリを確保し、リスト構造を更新します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/nodeMemoize.c 内のキャッシュ充填処理および検索処理。
  • 呼び出し先: palloc（メモリ確保）、および MinimalTuple 関連の操作関数。
  • 注記: 具体的な関数呼び出しの依存関係については、提供されたソース抜粋からは網羅的な特定ができません。

関連データ構造

  • MemoizeEntry: キャッシュのハッシュテーブルに格納される構造体であり、MemoizeTuple のリストの起点（tuplehead）を保持します。
  • MinimalTuple: MemoizeTuple が保持する実際のタプルデータ型。

失敗モードと境界条件

  • メモリ不足: palloc 失敗時にエラーが発生します。
  • リストの空状態: tuplehead が NULL の場合、キャッシュに該当するタプルが存在しないことを示します。

保守担当者向けメモ

  • CACHE_TUPLE_BYTES(t) マクロにより、MemoizeTuple 構造体自体のサイズとキャッシュされたタプルのサイズを計算する際に使用されます。
  • ExecEstimateCacheEntryOverheadBytes 等で、メモリ消費量の見積もりに利用されています。
