add_child_eq_member

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/equivclass.c:660

目的

継承テーブル等の子リレーションに関連する式を、既存の等価クラス（EquivalenceClass）に子メンバーとして追加する。

シグネチャまたは形状

static EquivalenceMember *add_child_eq_member(
    PlannerInfo *root, 
    EquivalenceClass *ec, 
    int ec_index,
    Expr *expr, 
    Relids relids, 
    JoinDomain *jdomain,
    EquivalenceMember *parent_em, 
    Oid datatype,
    Index child_relid
)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • ec: メンバーを追加する対象の等価クラス。
  • ec_index: 等価クラスのインデックス（負の場合は記録しない）。
  • expr, relids, jdomain, datatype: 新規メンバーの属性。
  • parent_em: 親となる等価メンバー（必須）。
  • child_relid: 子リレーションの識別子。
  • **出力**:
  • 新規作成された EquivalenceMember 構造体へのポインタ。

前提条件と不変条件

  • parent_em は NULL であってはならない。
  • ec->ec_childmembers 配列は root->simple_rel_array_size に基づいて管理される。

アルゴリズムまたは振る舞い

1. ec->ec_childmembers 配列が root->simple_rel_array_size を保持するのに十分なサイズか確認し、不足していれば palloc0_array または repalloc0_array で拡張する。 2. make_eq_member を呼び出し、指定された属性を持つ新しい EquivalenceMember を生成する。 3. 生成したメンバーを ec->ec_childmembers[child_relid] リストに追加する。 4. ec_index が有効な場合、対象子リレーションの eclass_indexes に当該インデックスを追加する。 5. 生成したメンバーを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/equivclass.c 内の複数箇所（行 2916, 3054, 3112 等）。
  • **呼び出し先**: make_eq_member, palloc0_array, repalloc0_array, lappend, bms_add_member。

関連データ構造

  • EquivalenceClass: 等価クラス本体。
  • EquivalenceMember: 等価クラス内のメンバー。
  • PlannerInfo: プランナの全体状態。
  • RelOptInfo: リレーション情報（eclass_indexes を保持）。

失敗モードと境界条件

  • メモリ不足時には palloc / repalloc がエラーを発生させる。
  • ec_index が負の場合、child_relid に対応する eclass_indexes への記録はスキップされる。

保守担当者向けメモ

  • ec_childmembers は relid をインデックスとする配列であり、継承関係にある子リレーションの等価性を管理するために使用される。
  • ec_members（親メンバー用）とは管理場所が異なる点に注意すること。
