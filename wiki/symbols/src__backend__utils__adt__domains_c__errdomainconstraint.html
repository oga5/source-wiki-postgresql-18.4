errdomainconstraint

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/domains.c:430

目的

ドメイン制約違反が発生した際に、エラーメッセージの文脈情報としてデータ型名と制約名を付与するために使用されます。

シグネチャまたは形状

int errdomainconstraint(Oid datatypeOid, const char *conname);

入力と出力

  • **入力**:
  • datatypeOid: 制約違反が発生したドメイン型のOID。
  • conname: 違反した制約の名前。
  • **出力**:
  • 常に 0 を返します（戻り値は利用されません）。

前提条件と不変条件

  • 本関数は、エラー報告の過程（ereport 内など）で呼び出されることを前提としています。
  • errdatatype(datatypeOid) を内部で呼び出し、型情報をエラーコンテキストに設定します。

アルゴリズムまたは振る舞い

1. errdatatype(datatypeOid) を呼び出し、エラー情報にデータ型の詳細を付与します。 2. err_generic_string(PG_DIAG_CONSTRAINT_NAME, conname) を呼び出し、エラー情報に制約名を付与します。 3. 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c: ドメインチェック失敗時のエラー報告。
  • src/backend/utils/adt/domains.c: ドメイン検証ロジック内。
  • **呼び出し先**:
  • errdatatype
  • err_generic_string

関連データ構造

  • Oid: PostgreSQLのシステムカタログにおけるオブジェクト識別子。

失敗モードと境界条件

  • 本関数自体はエラーを発生させませんが、引数として渡される conname が NULL の場合、err_generic_string の実装依存の挙動となります。

保守担当者向けメモ

  • 本関数はエラー報告用のヘルパー関数であり、直接的なロジックの制御には関与しません。
  • src/include/utils/builtins.h で外部公開されています。
