PLy_input_from_tuple

定義

  • **種別**: 関数
  • **定義箇所**: src/pl/plpython/plpy_typeio.c:133

目的

PostgreSQLの HeapTuple をPythonの辞書オブジェクト（PyObject）へ変換する。

シグネチャまたは形状

PyObject *PLy_input_from_tuple(PLyDatumToOb *arg, HeapTuple tuple, TupleDesc desc, bool include_generated)

入力と出力

  • **入力**:
  • arg: 変換情報を保持する PLyDatumToOb 構造体。
  • tuple: 変換対象の HeapTuple。
  • desc: タプルの構造を定義する TupleDesc。
  • include_generated: 生成列を含めるかどうかを指定するフラグ。
  • **出力**: 変換されたPython辞書オブジェクト（PyObject *）。

前提条件と不変条件

  • 変換処理は PLyExecutionContext から取得した scratch_context 上で実行される。
  • 呼び出し前後でメモリコンテキストの切り替えが行われ、処理終了時には元のコンテキストへ復帰する。

アルゴリズムまたは振る舞い

1. PLy_current_execution_context を介して現在の実行コンテキストを取得する。 2. 取得した scratch_context を MemoryContextReset で初期化する。 3. MemoryContextSwitchTo を使用して、一時的に scratch_context へ切り替える。 4. PLyDict_FromTuple を呼び出し、タプルから辞書への変換を実行する。 5. MemoryContextSwitchTo で元のコンテキストに戻し、生成された辞書を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_cursorobject.c
  • src/pl/plpython/plpy_exec.c
  • src/pl/plpython/plpy_spi.c
  • **呼び出し先**:
  • PLy_current_execution_context
  • PLy_get_scratch_context
  • MemoryContextReset
  • MemoryContextSwitchTo
  • PLyDict_FromTuple

関連データ構造

  • PLyDatumToOb: 変換ロジックと型情報を保持する構造体。
  • HeapTuple: PostgreSQLのタプル表現。
  • TupleDesc: タプルの属性記述子。
  • PLyExecutionContext: PL/Pythonの実行コンテキスト。

失敗モードと境界条件

  • メモリ不足時には MemoryContext 関連の関数がエラーを発生させる可能性がある。
  • PLyDict_FromTuple 内部で発生する変換エラーは、PostgreSQLの elog(ERROR) を通じて伝播する。

保守担当者向けメモ

  • 本関数は scratch_context を使用するため、呼び出し元でメモリコンテキストを破壊しないよう注意すること。
  • include_generated フラグの扱いは PLyDict_FromTuple に依存する。
