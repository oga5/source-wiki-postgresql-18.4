Form_pg_default_acl

定義

  • 種別: typedef
  • 定義箇所: src/include/catalog/pg_default_acl.h:50

目的

システムカタログ pg_default_acl の行データ構造体 FormData_pg_default_acl へのポインタ型を提供し、カタログ内のレコードを型安全に操作するために使用されます。

シグネチャまたは形状

typedef FormData_pg_default_acl *Form_pg_default_acl;

入力と出力

  • 入力: GETSTRUCT(tuple) 等で取得されたタプルデータ。
  • 出力: FormData_pg_default_acl 構造体へのポインタ。

前提条件と不変条件

  • GETSTRUCT マクロによって返されるメモリ領域が、FormData_pg_default_acl のレイアウトと一致している必要があります。
  • 参照先のタプルは、システムカタログ pg_default_acl の有効な行である必要があります。

アルゴリズムまたは振る舞い

この型は、システムカタログのタプルをキャストして、oid や defaclacl などのフィールドへ直接アクセスするために使用されます。実行時には、SearchSysCache 等で取得したタプルをこの型にキャストし、属性値の読み取りを行います。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • src/backend/catalog/aclchk.c: 権限チェック時にタプルから oid や構造体フィールドを取得するために使用。
  • src/backend/catalog/objectaddress.c: オブジェクトアドレスの解決や操作時にタプルをキャストするために使用。
  • 呼び出し先:
  • FormData_pg_default_acl 構造体（定義元）。

関連データ構造

  • FormData_pg_default_acl: カタログの物理的な行構造。
  • HeapTuple: カタログから取得されるタプル形式。

失敗モードと境界条件

  • キャスト先の構造体レイアウトと実際のタプルデータが不一致の場合、メモリ破壊や不正な値の読み取りが発生する可能性があります。
  • GETSTRUCT が NULL を返す場合や、無効なタプルをキャストした場合には未定義の動作となります。

保守担当者向けメモ

  • 本シンボルは pg_default_acl カタログのスキーマ変更（Catalog.pm による定義更新）に強く依存しています。構造体のフィールド順序を変更した場合は、この型を使用しているすべての箇所で整合性が保たれているか確認してください。
