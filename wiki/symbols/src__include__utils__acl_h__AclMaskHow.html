AclMaskHow

定義

  • 種別: enum
  • 定義箇所: src/include/utils/acl.h:174

目的

aclmask() 関数において、ACL（アクセス制御リスト）から権限ビットを抽出する際の探索戦略を指定するために使用されます。

シグネチャまたは形状

typedef enum
{
    ACLMASK_ALL,    /* normal case: compute all bits */
    ACLMASK_ANY,    /* return when result is known nonzero */
} AclMaskHow;

入力と出力

  • 本シンボルは、権限チェック関数（aclmask 等）の引数として入力されます。
  • ACLMASK_ALL を指定した場合、対象ロールに関連するすべての権限ビットを計算して返します。
  • ACLMASK_ANY を指定した場合、権限が一つでも確認された時点で計算を打ち切り、非ゼロの値を返します。

前提条件と不変条件

  • AclMaskHow は、aclmask() の呼び出し元で権限判定の最適化が必要な場合に選択されます。

アルゴリズムまたは振る舞い

  • ACLMASK_ALL: ACL全体を走査し、指定されたロールに対する累積的な権限ビットを算出します。
  • ACLMASK_ANY: 権限チェックの短絡評価（short-circuit evaluation）を行います。特定の権限の有無のみを判定したい場合に効率的です。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/aclchk.c 内の複数の権限チェック関数（pg_class_aclcheck 等）から引数として渡されます。
  • 呼び出し先: aclmask() 関数がこの列挙型を受け取り、内部的な探索ロジックを切り替えます。

関連データ構造

  • AclMode: 権限ビットを保持する型。AclMaskHow はこのビット操作の挙動を制御します。
  • Acl: AclItem の配列。aclmask が走査対象とするデータ構造です。

失敗モードと境界条件

  • 該当するロールがACL内に存在しない場合、ACLMASK_ALL は 0 を返し、ACLMASK_ANY も 0 を返します。

保守担当者向けメモ

  • aclmask() の実装において、新しい探索戦略が必要になった場合はこの列挙型を拡張してください。
  • src/backend/catalog/aclchk.c 内の多くの関数がこの引数を経由して権限チェックを行っているため、変更時は影響範囲に注意してください。
