PLy_cursor_plan

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_cursorobject.c:163

目的

プリペアドプラン（PLyPlanObject）から PostgreSQL のカーソルを作成し、それをラップする PLyCursorObject を生成する。

シグネチャまたは形状

PyObject *PLy_cursor_plan(PyObject *ob, PyObject *args)
  • ob: PLyPlanObject としてキャストされるプランオブジェクト。
  • args: カーソル実行時に使用する引数のシーケンス。

入力と出力

  • **入力**:
  • ob: 実行対象のプラン。
  • args: プランのパラメータに対応する Python シーケンス。
  • **出力**: 成功時は PLyCursorObject（PyObject*）、失敗時は NULL を返す。

前提条件と不変条件

  • args が提供される場合、それは Python のシーケンス型である必要がある。
  • args の要素数は、プランが要求する引数の数（plan->nargs）と一致しなければならない。
  • カーソル生成は SPI サブトランザクション内で実行される。

アルゴリズムまたは振る舞い

1. 引数の数を確認し、プランの期待値と一致しない場合は TypeError を送出する。 2. PLyCursorObject を割り当て、専用のメモリコンテキストを作成する。 3. SPI サブトランザクションを開始する。 4. 一時コンテキストを作成し、Python の引数を PostgreSQL の Datum 形式に変換する。 5. SPI_cursor_open を呼び出してポータルを作成し、カーソルをピン留めする。 6. カーソル名をメモリコンテキストに保存し、サブトランザクションをコミットする。 7. エラー発生時は PG_CATCH ブロックでサブトランザクションをアボートし、オブジェクトを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_cursorobject.c:89
  • src/pl/plpython/plpy_planobject.c:124
  • **呼び出し先**:
  • SPI_cursor_open
  • PLy_spi_subtransaction_begin / commit / abort
  • PLy_output_convert
  • AllocSetContextCreate

関連データ構造

  • PLyCursorObject: カーソルの状態（ポータル名、メモリコンテキスト、クローズ状態）を保持。
  • PLyPlanObject: 実行対象のプリペアドプラン。
  • PLyExecutionContext: 現在の実行コンテキスト。

失敗モードと境界条件

  • 引数の型や数が不適切な場合、TypeError を発生させ NULL を返す。
  • SPI_cursor_open が失敗した場合、elog(ERROR) により処理を中断する。
  • メモリ割り当て失敗時は NULL を返す。
  • サブトランザクション内でエラーが発生した場合、PG_CATCH によりクリーンアップが行われ NULL を返す。

保守担当者向けメモ

  • 本関数は Python の C API を使用しており、Py_DECREF 等の参照カウント管理が重要である。
  • PG_TRY ブロック内でのメモリ管理（tmpcontext の削除）は、サブトランザクションのコミット/アボートと連動している点に注意すること。
