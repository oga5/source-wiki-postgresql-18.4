ExecHashJoin

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHashjoin.c:683

目的

シリアル実行環境におけるハッシュ結合の実行ノードとして機能し、内側リレーションから構築したハッシュテーブルを用いて外側リレーションとの結合処理を行う。

シグネチャまたは形状

TupleTableSlot *ExecHashJoin(PlanState *pstate)

入力と出力

  • **入力**: pstate (実行中のプラン状態を保持する PlanState 構造体へのポインタ)
  • **出力**: 結合条件に合致したタプルを格納する TupleTableSlot、または結合終了時に NULL を返す。

前提条件と不変条件

  • ExecHashJoinImpl を呼び出す際、並列実行フラグを false に固定して実行する。
  • 実行状態は hj_JoinState ステートマシンによって管理される。

アルゴリズムまたは振る舞い

1. ExecHashJoinImpl(pstate, false) を呼び出し、シリアル実行用の結合ロジックを実行する。 2. コンパイラによるインライン化を前提としており、並列実行用の分岐が除去されたコードパスが生成される。 3. 内部的には HJ_BUILD_HASHTABLE から HJ_NEED_NEW_BATCH までのステートマシンを駆動し、メモリ不足時にはバッチ分割とディスク退避を伴うハイブリッド・ハッシュ結合を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHashjoin.c:740 にて hjstate->js.ps.ExecProcNode に関数ポインタとして代入されている。
  • **呼び出し先**: ExecHashJoinImpl (シリアル実行モードで呼び出し)。

関連データ構造

  • PlanState: 実行ノードの状態管理。
  • HashJoinState: ハッシュ結合固有の状態（hj_JoinState を含む）。
  • TupleTableSlot: タプルの受け渡しに使用。

失敗モードと境界条件

  • メモリ制限を超過した場合、バッチ分割およびディスクへの退避処理が発生する。
  • 結合対象のデータが空の場合、適切に NULL を返す。

保守担当者向けメモ

  • 本関数は ExecHashJoinImpl のシリアル実行用ラッパーである。ロジックの修正が必要な場合は、原則として ExecHashJoinImpl を編集すること。
  • ステートマシンの遷移（hj_JoinState）が複雑であるため、デバッグ時は現在の状態遷移を追跡することが推奨される。
