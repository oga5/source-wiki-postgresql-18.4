GetOldestUnsummarizedLSN

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/walsummarizer.c:508

目的

WAL要約プロセスが処理を開始すべき最も古いLSN（Log Sequence Number）を特定し、共有メモリ上の状態を初期化または更新する。

シグネチャまたは形状

XLogRecPtr GetOldestUnsummarizedLSN(TimeLineID *tli, bool *lsn_is_exact)

入力と出力

  • **入力**:
  • tli: タイムラインIDを格納するポインタ（NULL可）。
  • lsn_is_exact: LSNが正確かどうかを示すフラグを格納するポインタ（NULL可）。
  • **出力**:
  • 戻り値: 要約対象となる最も古い XLogRecPtr。summarize_wal が無効な場合は InvalidXLogRecPtr を返す。

前提条件と不変条件

  • GUCパラメータ summarize_wal が有効であること。
  • 共有メモリ上の WalSummarizerCtl が初期化されていない場合、この関数がディスク上の情報を走査して初期値を決定する。
  • WAL要約プロセス（AmWalSummarizerProcess()）が呼び出した場合、計算結果が共有メモリの権威ある値として保存される。

アルゴリズムまたは振る舞い

1. summarize_wal が無効なら即座に終了する。 2. WAL要約プロセス以外からの呼び出しで、共有メモリが既に初期化済みであれば、ロックを取得して値を読み取り即座に返す。 3. 初期化が必要な場合、またはWAL要約プロセス自身による呼び出しの場合：

  • GetLatestLSN と readTimeLineHistory を用いて、ディスク上に存在する最も古いWALセグメントを特定する。
  • GetWalSummaries を呼び出し、既存のサマリファイルを確認して、要約済み範囲を考慮したLSNに更新する。
  • 共有メモリ（WalSummarizerCtl）を排他ロックし、計算したLSN、TLI、正確性フラグを保存する。

    • 4. 最後に、要求されたポインタ引数に値を書き込み、ロックを解放して戻る。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c
  • src/backend/postmaster/walsummarizer.c（プロセス初期化およびメインループ内）
  • **呼び出し先**:
  • AmWalSummarizerProcess, GetLatestLSN, readTimeLineHistory, XLogGetOldestSegno, XLogSegNoOffsetToRecPtr, GetWalSummaries, LWLockAcquire, LWLockRelease, ereport

関連データ構造

  • WalSummarizerCtl: 共有メモリ上の制御構造体。
  • TimeLineHistoryEntry: タイムライン履歴情報。
  • WalSummaryFile: 既存のサマリファイル情報。

失敗モードと境界条件

  • **WAL不在**: タイムライン上にWALが見つからない場合、ereport(ERROR, ...) を発行する。
  • **共有メモリ未初期化**: WAL要約プロセス以外が呼び出した場合、共有メモリを初期化する役割を担う。
  • **サマリファイルとの整合性**: 既存のサマリファイルの終了LSNがディスク上の最古LSNより進んでいる場合、サマリファイルの終了LSNを優先する。

保守担当者向けメモ

  • 本関数はWAL要約プロセスの起動時およびエラー復旧時に、共有メモリの状態を再構築する重要な役割を持つ。
  • summarize_wal が動的に変更された場合、ディスク上のWAL状況と共有メモリの不整合が発生しやすいため、この関数の走査ロジックは堅牢である必要がある。
