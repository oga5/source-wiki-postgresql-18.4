pgstat_get_custom_shmem_data

定義

  • 種別: function
  • 定義箇所: src/include/utils/pgstat_internal.h:917

目的

統計システムの共有メモリ領域から、指定されたカスタム統計種別に対応するデータ領域へのポインタを取得します。

シグネチャまたは形状

static inline void *pgstat_get_custom_shmem_data(PgStat_Kind kind)

入力と出力

  • **入力**: kind (PgStat_Kind) - 取得対象の統計種別。
  • **出力**: void * - 共有メモリ上の該当統計データ領域へのポインタ。

前提条件と不変条件

  • kind は pgstat_is_kind_custom(kind) を満たすカスタム統計種別であること。
  • 指定された統計種別の PgStat_KindInfo において fixed_amount が真であること。
  • 上記条件は Assert により実行時に検証されます。

アルゴリズムまたは振る舞い

1. kind から PGSTAT_KIND_CUSTOM_MIN を減算し、カスタム統計用配列のインデックス idx を算出します。 2. pgStatLocal.shmem->custom_data[idx] に格納されている共有メモリ上のアドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/injection_points/injection_stats_fixed.c (複数の箇所で PGSTAT_KIND_INJECTION_FIXED を指定して呼び出し)
  • **呼び出し先**:
  • pgstat_is_kind_custom (マクロまたは関数)
  • pgstat_get_kind_info (関数)

関連データ構造

  • PgStat_Kind: 統計種別を識別する列挙型。
  • PgStat_ShmemControl: 統計システムの共有メモリルート構造体。custom_data 配列を保持します。
  • PgStat_KindInfo: 統計種別のメタデータ。fixed_amount フラグを保持します。

失敗モードと境界条件

  • kind がカスタム統計種別でない場合、または fixed_amount が設定されていない統計種別に対して呼び出した場合、Assert によりプロセスがアボートします。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結しています。
  • 共有メモリ上のデータに直接アクセスするため、呼び出し側で適切なロック（changecount プロトコル等）が考慮されている必要があります。
