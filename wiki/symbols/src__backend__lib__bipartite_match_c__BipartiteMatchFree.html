BipartiteMatchFree

定義

  • 種別: function
  • 定義箇所: src/backend/lib/bipartite_match.c:77

目的

BipartiteMatch 関数によって確保された BipartiteMatchState 構造体およびその内部の作業用メモリ領域を解放する。

シグネチャまたは形状

void BipartiteMatchFree(BipartiteMatchState *state)

入力と出力

  • **入力**: BipartiteMatchState *state - 解放対象となるマッチング状態構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • state は BipartiteMatch によって正常に初期化された有効なポインタであること。
  • 入力として渡された隣接リスト（adjacency）の所有権は呼び出し元にあるため、本関数はこれを解放しない。

アルゴリズムまたは振る舞い

以下の順序でメモリを解放する： 1. state->pair_uv を解放。 2. state->pair_vu を解放。 3. state->distance を解放。 4. state->queue を解放。 5. state 構造体自体を解放。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planner.c:3348
  • src/backend/lib/bipartite_match.c:78（内部的な再帰やテスト用途の可能性があるが、ソース上では定義直後の行として参照されている）
  • **呼び出し先**: pfree（PostgreSQL のメモリ解放関数）。

関連データ構造

  • BipartiteMatchState: マッチング計算に必要な状態を保持する構造体。

失敗モードと境界条件

  • state が NULL の場合、pfree の仕様に依存する（通常はセグメンテーション違反を引き起こす可能性がある）。

保守担当者向けメモ

  • 本関数は BipartiteMatch が内部的に確保したメモリのみを管理対象とする。隣接リストのメモリ管理は呼び出し元の責務である点に注意すること。
