find_lateral_references

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/initsplan.c:708

目的

クエリ内の LATERAL 参照をスキャンし、依存関係を RelOptInfo に記録することで、プラン作成時に必要な変数が各段階で利用可能になるよう準備します。

シグネチャまたは形状

void find_lateral_references(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root（クエリ全体のプランナ情報）
  • **出力**: なし（副作用として root 内の RelOptInfo 構造体が更新されます）

前提条件と不変条件

  • root->simple_rel_array が既に構築されている必要があります。
  • クエリに LATERAL 参照が含まれない場合、本関数は何も行わず即座に復帰します。

アルゴリズムまたは振る舞い

1. root->hasLateralRTEs を確認し、LATERAL 参照が存在しない場合は終了します。 2. root->simple_rel_array を走査し、各リレーション（RelOptInfo）を順次確認します。 3. RELOPT_BASEREL 以外のリレーション（other rels）はスキップします。

  • 継承展開やサブクエリのプルアップによって生成された other rels は無視し、親となる baserel を通じて依存関係を処理します。

    • 4. 対象の baserel に対して extract_lateral_references を呼び出し、具体的な参照情報を抽出・記録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planmain.c:189
  • **呼び出し先**:
  • extract_lateral_references (同ファイル内)

関連データ構造

  • PlannerInfo: クエリ全体のプランナ情報を保持。
  • RelOptInfo: 各リレーションの最適化情報を保持。

失敗モードと境界条件

  • simple_rel_array 内の空スロット（brel == NULL）は適切にスキップされます。
  • RELOPT_BASEREL 以外のリレーションは処理対象外となります。

保守担当者向けメモ

  • 本関数は LATERAL 参照の依存関係を特定する重要なフェーズです。継承関係やプルアップされたサブクエリの扱いについては、ソース内のコメントにある通り、親リレーションを基準に処理する設計となっています。
  • 外部結合の最適化ロジックなど、他の関数（src/backend/optimizer/plan/initsplan.c:854 等）でも本関数の処理結果が参照されるため、変更時は影響範囲に注意してください。
