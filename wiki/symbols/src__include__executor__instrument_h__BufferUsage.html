BufferUsage

定義

  • 種別: struct
  • 定義箇所: src/include/executor/instrument.h:24

目的

クエリ実行に伴うバッファI/O統計（ヒット数、読み書き回数、およびそれらに要した時間）を収集・管理するためのデータ構造です。

シグネチャまたは形状

typedef struct BufferUsage
{
    int64       shared_blks_hit;
    int64       shared_blks_read;
    int64       shared_blks_dirtied;
    int64       shared_blks_written;
    int64       local_blks_hit;
    int64       local_blks_read;
    int64       local_blks_dirtied;
    int64       local_blks_written;
    int64       temp_blks_read;
    int64       temp_blks_written;
    instr_time  shared_blk_read_time;
    instr_time  shared_blk_write_time;
    instr_time  local_blk_read_time;
    instr_time  local_blk_write_time;
    instr_time  temp_blk_read_time;
    instr_time  temp_blk_write_time;
} BufferUsage;

入力と出力

  • 本構造体自体はデータの保持用であり、関数ではないため入力・出力の概念は適用されません。
  • 統計値は累積的に加算されます。

前提条件と不変条件

  • 各フィールドはリセットされず、常に増加し続けるカウンタとして扱われます。
  • 特定の処理区間の統計を得るには、開始時と終了時の値の差分を計算する必要があります。

アルゴリズムまたは振る舞い

  • 共有バッファ、ローカルバッファ、および一時ファイル（temp）に対するI/O操作を追跡します。
  • instr_time 型を用いて、各I/O操作に費やされた時間を計測します。
  • BufferUsageAccumDiff 等の関数を通じて、期間ごとの増分を抽出する用途で利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照・操作）**:
  • contrib/pg_stat_statements/pg_stat_statements.c: 統計情報の収集および参照。
  • src/backend/access/brin/brin.c: パラレルクエリ実行時のワーカーごとの統計管理。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な呼び出し先はありません。

関連データ構造

  • Instrumentation: 本構造体を含む、プランノードごとの統計管理用構造体。
  • WalUsage: 同様にリソース使用量を追跡する構造体。
  • instr_time: 時間計測に使用される型（portability/instr_time.h で定義）。

失敗モードと境界条件

  • カウンタは int64 型であるため、極めて長時間の実行や膨大なI/Oが発生した場合のオーバーフローに注意が必要です。
  • 差分計算時に開始時の値が終了時の値より大きい場合、正しく処理されない可能性があります。

保守担当者向けメモ

  • 本構造体は EXPLAIN ANALYZE や pg_stat_statements の基盤となります。
  • フィールドを追加する際は、BufferUsageAccumDiff 等の集計ロジックにも変更が必要か確認してください。
