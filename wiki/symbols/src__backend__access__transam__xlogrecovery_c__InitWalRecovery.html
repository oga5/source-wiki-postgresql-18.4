InitWalRecovery

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogrecovery.c:518

目的

クラッシュリカバリ、アーカイブリカバリ、またはスタンバイモードを開始するための初期化処理を行います。制御ファイル（pg_control）や backup_label ファイルを解析し、リカバリの開始点、タイムライン、およびリカバリモードを決定します。

シグネチャまたは形状

void InitWalRecovery(ControlFileData *ControlFile, bool *wasShutdown_ptr,
                    bool *haveBackupLabel_ptr, bool *haveTblspcMap_ptr)

入力と出力

  • **入力**:
  • ControlFile: データベースの制御ファイル情報。
  • **出力**:
  • wasShutdown_ptr: データベースが正常終了していたかを示すフラグ。
  • haveBackupLabel_ptr: backup_label ファイルが存在したかを示すフラグ。
  • haveTblspcMap_ptr: tablespace_map ファイルが存在したかを示すフラグ。

前提条件と不変条件

  • readRecoverySignalFile() および validateRecoveryParameters() が呼び出され、リカバリ設定が検証済みであること。
  • wal_segment_size および wal_decode_buffer_size が適切に設定されていること。

アルゴリズムまたは振る舞い

1. **タイムライン決定**: pg_control の情報に基づき、リカバリ対象のタイムライン（recoveryTargetTLI）を決定します。 2. **環境設定**: アーカイブリカバリ要求がある場合、recoveryWakeupLatch を所有します。 3. **WALリーダー初期化**: XLogReaderAllocate を使用してWAL読み込みプロセッサを構築し、XLogPrefetcher を作成します。 4. **バックアップラベル処理**: read_backup_label を実行し、存在する場合は以下の処理を行います。

  • InArchiveRecovery を有効化し、スタンバイモードであれば EnableStandbyMode() を呼び出します。
  • ReadCheckpointRecord でチェックポイントレコードを読み込み、REDO開始位置を特定します。
  • 必要に応じて ReadRecord でREDO開始位置の妥当性を確認します。

    • 5. **テーブルスペースマップ**: read_tablespace_map が存在する場合、シンボリックリンクを再作成します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlog.c 内の StartupXLOG から呼び出されます。
  • **呼び出し先**:
  • readRecoverySignalFile, validateRecoveryParameters
  • OwnLatch, XLogReaderAllocate, XLogPrefetcherAllocate
  • read_backup_label, ReadCheckpointRecord, ReadRecord
  • read_tablespace_map, remove_tablespace_symlink, symlink

関連データ構造

  • ControlFileData: 制御ファイル情報。
  • XLogReaderState (xlogreader): WAL読み込み状態。
  • XLogPrefetcher (xlogprefetcher): WAL先読み管理。
  • XLogPageReadPrivate: WAL読み込み用のプライベートデータ。

失敗モードと境界条件

  • **メモリ不足**: XLogReaderAllocate が失敗した場合、ERROR で終了します。
  • **チェックポイント欠損**: backup_label が示すチェックポイントレコードが見つからない場合、FATAL で終了します。
  • **REDO位置不正**: チェックポイントが参照するREDO位置のWALレコードが読み込めない場合、FATAL で終了します。

保守担当者向けメモ

  • 本関数はスタートアッププロセスにおいて非常に重要な初期化フェーズです。backup_label の有無によってリカバリの挙動が大きく変わるため、修正時はPITRやスタンバイへの影響を十分に考慮してください。
  • xlogreader の設定（デコードバッファサイズ等）は、リカバリのパフォーマンスに直結します。
