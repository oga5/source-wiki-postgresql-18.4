join_clause_is_movable_into

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/restrictinfo.c:660

目的

結合条件（RestrictInfo）が、指定されたリレーション集合（currentrelids）を含むコンテキストへ移動可能かどうかを判定します。

シグネチャまたは形状

bool join_clause_is_movable_into(RestrictInfo *rinfo, Relids currentrelids, Relids current_and_outer)

入力と出力

  • **入力**:
  • rinfo: 評価対象の RestrictInfo 構造体。
  • currentrelids: 移動先となるターゲットリレーションの集合。
  • current_and_outer: 評価時に利用可能なリレーションの全集合。
  • **出力**:
  • bool: 移動可能な場合は true、そうでない場合は false を返します。

前提条件と不変条件

  • rinfo->clause_relids が current_and_outer のサブセットである必要があります。
  • rinfo が参照するリレーションと currentrelids が少なくとも一つ重なっている必要があります。

アルゴリズムまたは振る舞い

以下の条件をすべて満たす場合に true を返します。 1. 条件式が current_and_outer 内のリレーションのみで評価可能であること。 2. 条件式が currentrelids に含まれるリレーションを少なくとも一つ物理的に参照していること。 3. 条件式が外部結合（Outer Join）の制約に関与しておらず、結合の外部側（outer side）へ移動しようとしていないこと（currentrelids と rinfo->outer_relids が重複していないこと）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/costsize.c
  • src/backend/optimizer/path/equivclass.c
  • src/backend/optimizer/util/relnode.c
  • **呼び出し先**:
  • bms_is_subset (ビットマップ集合操作)
  • bms_overlap (ビットマップ集合操作)

関連データ構造

  • RestrictInfo: 結合条件やフィルタ条件のメタデータを保持する構造体。
  • Relids: リレーションIDの集合を表すビットマップ。

失敗モードと境界条件

  • rinfo->clause_relids が current_and_outer の範囲外である場合、即座に false を返します。
  • currentrelids と rinfo->clause_relids に共通のリレーションがない場合、移動不可とみなされます。
  • 外部結合の制約に抵触する場合（outer_relids との重複）は移動できません。

保守担当者向けメモ

  • 本関数はプランナの最適化フェーズにおいて、結合条件のプッシュダウン可能性を検証するために多用されます。
  • Relids のビットマップ操作に依存しているため、リレーションIDの管理に変更がある場合は注意が必要です。
