PgAioReturn

定義

  • 種別: struct
  • 定義箇所: src/include/storage/aio_types.h:130

目的

非同期I/O（AIO）操作完了後のステータス情報と、その操作対象となったリソース情報を集約して保持するために使用されます。

シグネチャまたは形状

typedef struct PgAioReturn
{
	PgAioResult result;
	PgAioTargetData target_data;
} PgAioReturn;

入力と出力

  • **入力**: I/O操作の完了時に、システムによって result（ステータス）および target_data（対象リレーションやブロック情報）が書き込まれます。
  • **出力**: 呼び出し元は本構造体を参照することで、I/Oの成否および対象となった物理リソースを特定します。

前提条件と不変条件

  • PgAioResult はメモリ効率のためサイズが固定されており、ビットフィールドの管理には注意が必要です。
  • 本構造体はI/O完了後のエラーハンドリングおよびリソース再特定のために使用されることを前提としています。

アルゴリズムまたは振る舞い

I/O操作が発行された後、完了通知とともに本構造体に結果が格納されます。呼び出し元は、pgaio_io_acquire 等の関数を通じて本構造体のアドレスを渡し、完了後にその内容を検査することで、I/Oが成功したか、あるいはどのようなエラーが発生したかを判断します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/storage/aio/aio.c: pgaio_io_acquire, pgaio_io_acquire_nb
  • src/backend/storage/buffer/bufmgr.c: バッファ管理におけるI/O操作の戻り値として利用。
  • src/include/storage/bufmgr.h: io_return メンバとして保持。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • PgAioResult: I/O結果のステータスを保持する構造体。
  • PgAioTargetData: I/O対象（リレーション、ブロック番号等）を特定するための共用体。

失敗モードと境界条件

  • I/O操作が失敗した場合、result メンバ内のビットフィールドにエラー情報が反映されます。
  • 構造体自体のメモリレイアウトやサイズについては、aio_types.h 内の静的アサーション（StaticAssertDecl）による制約を受ける可能性があります。

保守担当者向けメモ

  • 本構造体はI/Oサブシステムの基盤となるため、変更時は PgAioResult および PgAioTargetData への影響を考慮してください。
  • 呼び出し元で PgAioReturn を初期化または参照する際は、aio.c で定義されているAPIの仕様に従ってください。
