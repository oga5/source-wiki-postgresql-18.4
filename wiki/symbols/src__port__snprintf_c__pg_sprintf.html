pg_sprintf

定義

  • 種別: function
  • 定義箇所: src/port/snprintf.c:229

目的

PostgreSQL 独自の printf ファミリー実装の一部として、標準ライブラリの sprintf と同等の機能を提供します。プラットフォーム間の差異を吸収し、POSIX 準拠のフォーマット制御を可能にします。

シグネチャまたは形状

int pg_sprintf(char *str, const char *fmt, ...);

入力と出力

  • **入力**:
  • str: 書き込み先の文字バッファ。
  • fmt: フォーマット文字列。
  • ...: フォーマットに従う可変長引数。
  • **出力**:
  • 戻り値: 書き込まれた文字数（終端のヌル文字を含まない）。

前提条件と不変条件

  • fmt 文字列は有効なフォーマット指定子を含む必要があります。
  • str は、フォーマットされた結果を格納するのに十分な大きさのメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. va_start を使用して可変長引数リストを初期化します。 2. 内部関数 pg_vsprintf を呼び出し、実際のフォーマット処理とバッファへの書き込みを実行します。 3. va_end を使用して引数リストを終了します。 4. pg_vsprintf が返した書き込み文字数を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port.h にて sprintf マクロとして定義されており、PostgreSQL 全体で標準の sprintf の代わりに使用されます。
  • **呼び出し先**: pg_vsprintf を呼び出します。

関連データ構造

  • PrintfTarget: 出力先や状態を管理する構造体（pg_vsprintf 経由で使用）。

失敗モードと境界条件

  • バッファオーバーランが発生した場合の挙動は、C99 標準に従い「書き込まれるべきだったバイト数」を返します。
  • %n$ 指定子を使用する場合、PG_NL_ARGMAX (31) までの引数順序制御が可能です。

保守担当者向けメモ

  • 本関数は src/port/snprintf.c 内で定義されており、標準ライブラリの sprintf を #undef して使用しています。
  • 浮動小数点数の出力には fmtfloat が関与しており、プラットフォーム間の差異（特に Windows）を吸収する補正が含まれています。
