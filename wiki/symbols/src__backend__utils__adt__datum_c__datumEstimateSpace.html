datumEstimateSpace

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/datum.c:447

目的

Datum をシリアライズする際に必要なバイト数を算出します。

シグネチャまたは形状

Size datumEstimateSpace(Datum value, bool isnull, bool typByVal, int typLen)

入力と出力

  • **入力**:
  • value: 対象の Datum 値。
  • isnull: 値が NULL かどうかを示すフラグ。
  • typByVal: 型が値渡し（by-value）かどうか。
  • typLen: 型の長さ（バイト数）。
  • **出力**:
  • Size: シリアライズに必要な合計バイト数（sizeof(int) を含む）。

前提条件と不変条件

  • typByVal および typLen は、対象となるデータ型の正しいメタデータである必要があります。
  • isnull が true の場合、value は無視されます。

アルゴリズムまたは振る舞い

1. 初期値として sizeof(int) を確保します（NULL フラグ等のメタデータ用）。 2. isnull が false の場合、以下のいずれかのロジックでサイズを加算します。

  • typByVal が真の場合: sizeof(Datum) を加算します。
  • typLen が -1 かつ拡張オブジェクト（Expanded Object）の場合: EOH_get_flat_size を用いてフラット化後のサイズを加算します。
  • それ以外の場合: datumGetSize を呼び出し、型固有のサイズを加算します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtree.c
  • src/backend/executor/execParallel.c
  • src/backend/nodes/params.c
  • **呼び出し先**:
  • datumGetSize
  • EOH_get_flat_size（拡張オブジェクトの場合）

関連データ構造

  • Datum: PostgreSQL の汎用データ型表現。
  • ExpandedObject: VARATT_IS_EXTERNAL_EXPANDED で判定されるメモリ上の拡張データ構造。

失敗モードと境界条件

  • typByVal や typLen に誤った値が渡された場合、不正なメモリ参照や誤ったサイズ計算が発生する可能性があります。
  • 拡張オブジェクトのフラット化サイズ計算は、対象オブジェクトの内部状態に依存します。

保守担当者向けメモ

  • 本関数は、パラレルクエリ等のプロセス間通信におけるシリアライズ準備段階で使用されます。
  • 拡張オブジェクトのフラット化サイズを考慮しているため、TOAST されたデータや複雑なメモリ構造を持つデータ型を扱う際は、datumSerialize との整合性に注意してください。
