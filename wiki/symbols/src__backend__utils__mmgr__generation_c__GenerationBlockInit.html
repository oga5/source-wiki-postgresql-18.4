GenerationBlockInit

定義

  • 種別: static inline function
  • 定義箇所: src/backend/utils/mmgr/generation.c:608

目的

世代別メモリコンテキスト（GenerationContext）で使用されるメモリブロック（GenerationBlock）の初期化を行う。

シグネチャまたは形状

static inline void GenerationBlockInit(GenerationContext *context, GenerationBlock *block, Size blksize)

入力と出力

  • **入力**:
  • context: このブロックを所有する GenerationContext へのポインタ。
  • block: 初期化対象の GenerationBlock 構造体へのポインタ。
  • blksize: ブロック全体のサイズ（バイト単位）。
  • **出力**: なし（副作用として block の各メンバを初期化）。

前提条件と不変条件

  • block は、少なくとも Generation_BLOCKHDRSZ 以上のサイズを持つメモリ領域を指している必要がある。
  • blksize は、ヘッダーサイズよりも大きい値である必要がある。

アルゴリズムまたは振る舞い

1. block の context および blksize メンバを引数で設定する。 2. チャンク数（nchunks）および空きチャンク数（nfree）を 0 に初期化する。 3. freeptr をヘッダー直後のアドレスに設定し、endptr をブロックの末尾アドレスに設定する。 4. VALGRIND_MAKE_MEM_NOACCESS を呼び出し、ヘッダーを除くブロック内の全領域を Valgrind に対してアクセス不可としてマークする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/generation.c:230
  • src/backend/utils/mmgr/generation.c:493
  • **呼び出し先**:
  • VALGRIND_MAKE_MEM_NOACCESS（マクロ）

関連データ構造

  • GenerationContext: メモリコンテキストの管理構造体。
  • GenerationBlock: メモリ割り当ての単位となる構造体。

失敗モードと境界条件

  • blksize が Generation_BLOCKHDRSZ より小さい場合、freeptr が endptr を超える可能性があるが、本関数内での明示的なチェックは行われない。

保守担当者向けメモ

  • 本関数は static inline で定義されており、コンパイル時にインライン展開されることが期待されている。
  • Valgrind マクロを使用しているため、メモリの有効性チェックを厳密に行う環境では、この初期化がメモリ保護の起点となる。
