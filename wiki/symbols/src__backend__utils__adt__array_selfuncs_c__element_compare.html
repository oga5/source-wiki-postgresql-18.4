element_compare

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_selfuncs.c:1164

目的

qsort や二分探索などのソート・検索アルゴリズムにおいて、配列要素（Datum）同士を比較するための汎用的な比較関数を提供します。

シグネチャまたは形状

static int element_compare(const void *key1, const void *key2, void *arg)

入力と出力

  • **入力**:
  • key1, key2: 比較対象の Datum へのポインタ。
  • arg: TypeCacheEntry へのポインタ（比較関数の情報と照合用）。
  • **出力**:
  • int: key1 が key2 より小さい場合は負、等しい場合は 0、大きい場合は正の値を返します。

前提条件と不変条件

  • arg は TypeCacheEntry 型としてキャスト可能である必要があります。
  • typentry->cmp_proc_finfo に有効な比較関数情報が格納されている必要があります。

アルゴリズムまたは振る舞い

1. key1 および key2 から Datum 値を取り出します。 2. arg から取得した TypeCacheEntry 内の比較関数（cmp_proc_finfo）を特定します。 3. FunctionCall2Coll を使用して、指定された照合順序（typcollation）に基づき 2 つの Datum を比較します。 4. 比較結果を DatumGetInt32 で整数に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/array_selfuncs.c 内の複数の箇所（配列データのソートや重複排除、統計情報の検索処理など）から呼び出されています。
  • **呼び出し先**: FunctionCall2Coll（PostgreSQL の関数マネージャ経由で型固有の比較関数を実行）。

関連データ構造

  • Datum: PostgreSQL の汎用データ表現。
  • TypeCacheEntry: 型情報および比較関数へのアクセスを保持するキャッシュ構造体。

失敗モードと境界条件

  • arg が NULL である場合、または typentry 内の比較関数情報が初期化されていない場合、セグメンテーション違反が発生する可能性があります。
  • 比較対象の Datum が無効な値である場合、比較関数の振る舞いはその型定義に依存します。

保守担当者向けメモ

  • 本関数は qsort 等の標準ライブラリ関数に渡すためのラッパーとして設計されています。
  • src/backend/utils/adt/array_typanalyze.c にも同名の関数が存在しますが、引数構成が異なるため混同しないよう注意してください。
