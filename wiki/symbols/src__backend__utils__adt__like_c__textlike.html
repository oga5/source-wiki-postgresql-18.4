textlike

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like.c:283

目的

SQLの LIKE 演算子を実装し、文字列型（text）のパターンマッチングを実行する。

シグネチャまたは形状

Datum textlike(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_PP(0): 対象となる文字列（text 型）。
  • PG_GETARG_TEXT_PP(1): パターン文字列（text 型）。
  • **出力**:
  • PG_RETURN_BOOL(result): パターンに一致すれば true、そうでなければ false を返す。

前提条件と不変条件

  • 入力された text 型データは、VARDATA_ANY および VARSIZE_ANY_EXHDR マクロを用いて内部データポインタとヘッダを除いたサイズが正しく取得可能であること。
  • 現在のデータベースの照合順序（Collation）が PG_GET_COLLATION() を通じて取得可能であること。

アルゴリズムまたは振る舞い

1. 入力された text 型引数から、データ本体のポインタと長さを抽出する。 2. GenericMatchText 関数を呼び出し、抽出した文字列とパターン、および照合順序を渡してマッチングを実行する。 3. GenericMatchText の戻り値が LIKE_TRUE と等しいかどうかを判定し、その結果をブール値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（LIKE 演算子実行時）。
  • **呼び出し先**: GenericMatchText（src/backend/utils/adt/like.c 内で定義）。

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • 入力値が NULL の場合、PG_GETARG_TEXT_PP の仕様に従い、呼び出し元で適切に処理される必要がある（本関数内では直接的なNULLチェックは記述されていない）。
  • パターンマッチングの詳細は GenericMatchText に依存しており、エンコーディングや照合順序の不整合により予期せぬ挙動が発生する可能性がある。

保守担当者向けメモ

  • 本関数は LIKE のインターフェース層であり、実際の比較ロジックは GenericMatchText および like_match.c に委譲されている。
  • パフォーマンスやマッチングアルゴリズムの変更を行う場合は、like_match.c のインクルード構成および GenericMatchText のディスパッチロジックを確認すること。
