array_insert_slice

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arrayfuncs.c:5175

目的

既存の配列（origArray）の特定範囲（スライス）を、別の配列（srcArray）の内容で置換し、結果を destArray に書き込む。

シグネチャまたは形状

static void array_insert_slice(
    ArrayType *destArray,
    ArrayType *origArray,
    ArrayType *srcArray,
    int ndim,
    int *dim,
    int *lb,
    int *st,
    int *endp,
    int typlen,
    bool typbyval,
    char typalign
)

入力と出力

  • **入力**:
  • destArray: 書き込み先の配列（呼び出し元で十分なメモリが確保されている必要がある）。
  • origArray: 置換元のベースとなる配列。
  • srcArray: スライスとして挿入する内容を持つ配列。
  • ndim, dim, lb: 配列の次元数、各次元のサイズ、下限値。
  • st, endp: 置換対象となるスライスの開始および終了インデックス。
  • typlen, typbyval, typalign: 配列要素の型情報。
  • **出力**: なし（destArray を直接更新する）。

前提条件と不変条件

  • destArray は、置換後の全データを格納するのに十分なメモリ領域を保持していること。
  • origArray と srcArray は、指定された次元および型情報と整合していること。
  • NULLビットマップが存在する場合、destArray のビットマップも適切に更新される。

アルゴリズムまたは振る舞い

1. **初期コピー**: スライス開始位置より前のデータを origArray から destArray へコピーする。 2. **スライス置換ループ**:

  • 多次元配列のインデックス計算（mda_get_prod, mda_get_range, mda_get_offset_values）を用いて、スライス領域を走査する。
  • スライス外の領域は origArray からコピーし、スライス内の領域は srcArray からコピーする。
  • 各ステップで array_copy を使用して要素を転送し、NULLビットマップが存在する場合は array_bitmap_copy で対応するビットを更新する。

    • 3. **末尾コピー**: スライス終了位置以降の残りのデータを origArray から destArray へコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/arrayfuncs.c 内の array_set_slice 等から呼び出される。
  • **呼び出し先**: array_copy, array_bitmap_copy, array_seek, mda_get_prod, mda_get_range, mda_get_offset_values, mda_next_tuple 等の配列操作ユーティリティ。

関連データ構造

  • ArrayType: PostgreSQLの配列内部表現。
  • MAXDIM: サポートされる最大次元数（16）。

失敗モードと境界条件

  • destArray のメモリ不足: 呼び出し元で適切なサイズを確保していない場合、メモリ破壊が発生する可能性がある。
  • インデックス範囲外: st や endp が配列の境界を超えている場合、ArrayGetOffset や array_seek の挙動により予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は低レベルなメモリ操作を行うため、要素の型情報（typlen, typbyval, typalign）の不一致は致命的なエラーを招く。
  • destBitmap が NULL の場合、NULLビットマップの更新処理はスキップされる。
  • 多次元配列の走査ロジックは mda_ 系関数に依存しているため、次元数やインデックス計算の変更時は注意が必要。
