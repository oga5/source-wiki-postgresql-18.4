List

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/pg_list.h:53

目的

PostgreSQLにおける汎用的なリストデータ構造を提供します。クエリツリーやプランツリーなどのノードを保持するためのコンテナとして機能します。

シグネチャまたは形状

typedef struct List
{
    NodeTag     type;
    int         length;
    int         max_length;
    ListCell   *elements;
    ListCell    initial_elements[FLEXIBLE_ARRAY_MEMBER];
} List;

入力と出力

  • **入力**: NodeTag（リストの型）、要素数、および ListCell 配列。
  • **出力**: リスト操作関数（lappend 等）は、必要に応じて再配置された新しい List ポインタを返します。

前提条件と不変条件

  • 空リストは必ず NIL（NULLポインタ）で表現されます。length == 0 のリストという概念は存在しません。
  • リストが空でない限り、ヘッダと要素配列のメモリ位置は再配置されないことが保証されています。

アルゴリズムまたは振る舞い

  • 拡張可能な配列（expansible array）として実装されており、要素の追加時に必要に応じて elements 配列の再割り当てを行います。
  • initial_elements をヘッダと同時に割り当てることで、小規模なリストのメモリ効率を最適化しています。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • contrib/amcheck/verify_heapam.c
  • contrib/bloom/blvalidate.c
  • contrib/dblink/dblink.c
  • contrib/dict_int/dict_int.c
  • contrib/dict_xsyn/dict_xsyn.c
  • contrib/file_fdw/file_fdw.c
  • **呼び出し先**:
  • 呼び出し先に関する具体的な情報は提供されていません。

関連データ構造

  • ListCell: リスト内の各要素を保持する共用体。
  • NodeTag: リストの型（T_List, T_IntList, T_OidList, T_XidList）を識別するタグ。

失敗モードと境界条件

  • foreach ループ走査中に要素を削除する場合、通常の削除関数を使用するとイテレータの整合性が崩れます。必ず foreach_delete_current を使用する必要があります。

保守担当者向けメモ

  • 本構造体は src/backend/nodes/list.c に実装された関数群によって操作されます。
  • ノードを扱う際は、型安全性を確保するため lfirst ではなく lfirst_node(type, lc) の使用が推奨されます。
