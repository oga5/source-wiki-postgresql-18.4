SetCancelConn

定義

  • 種別: function
  • 定義箇所: src/fe_utils/cancel.c:76

目的

現在のデータベース接続（PGconn）に対応するキャンセル用オブジェクト（PGcancel）を更新し、割り込み発生時にバックエンドへキャンセル要求を送信できるようにします。

シグネチャまたは形状

void SetCancelConn(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn - キャンセル対象となるデータベース接続ポインタ。NULL を指定することで、既存のキャンセルオブジェクトを破棄し、新規設定を無効化できます。
  • **出力**: なし（void）。

前提条件と不変条件

  • Windows環境では、内部の cancelConn ポインタを保護するために cancelConnLock（CRITICAL_SECTION）による排他制御が行われます。
  • cancelConn ポインタの更新中、シグナルハンドラが古いポインタを参照しないよう、一時的に NULL に設定する安全策が講じられています。

アルゴリズムまたは振る舞い

1. **排他制御**: Windows環境の場合、cancelConnLock を取得します。 2. **既存オブジェクトの解放**: 現在保持している cancelConn を退避し、cancelConn を NULL に設定した上で、PQfreeCancel を呼び出してメモリを解放します。 3. **新規オブジェクトの生成**: PQgetCancel(conn) を呼び出し、新しい接続からキャンセル用オブジェクトを取得して cancelConn に格納します。 4. **排他制御の解除**: Windows環境の場合、cancelConnLock を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • pgbench (src/bin/pgbench/pgbench.c)
  • psql (src/bin/psql/common.c, src/bin/psql/large_obj.c)
  • parallel_slot (src/fe_utils/parallel_slot.c)
  • **呼び出し先**:
  • PQfreeCancel (libpq)
  • PQgetCancel (libpq)
  • EnterCriticalSection / LeaveCriticalSection (Windows環境のみ)

関連データ構造

  • PGconn: libpqの接続ハンドル。
  • PGcancel: libpqのキャンセル用オブジェクト。
  • cancelConn: src/fe_utils/cancel.c 内で定義された静的変数。

失敗モードと境界条件

  • conn が NULL の場合、PQgetCancel は NULL を返し、結果としてキャンセル機能が無効化されます。
  • PQgetCancel がメモリ不足等で失敗した場合、cancelConn は NULL となり、以降のキャンセル要求は送信されません。

保守担当者向けメモ

  • 本関数はシグナルハンドラと非同期に動作する可能性があるため、ポインタの入れ替え順序（NULL への一時設定）は必須の不変条件です。
  • Windows環境での排他制御は、handle_sigint 等のシグナルハンドラが cancelConn を参照する際にも同様のロックを必要とすることを前提としています。
