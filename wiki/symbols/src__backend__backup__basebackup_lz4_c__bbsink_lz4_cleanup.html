bbsink_lz4_cleanup

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_lz4.c:284

目的

LZ4圧縮シンクに関連付けられた圧縮コンテキストを解放し、リソースをクリーンアップする。

シグネチャまたは形状

static void bbsink_lz4_cleanup(bbsink *sink)

入力と出力

  • 入力: bbsink *sink （bbsink_lz4 構造体としてキャストされるシンクオブジェクト）
  • 出力: なし

前提条件と不変条件

  • 引数 sink は bbsink_lz4 型の構造体として安全にキャスト可能であること。
  • mysink->ctx が有効なポインタであるか、あるいは NULL であること。

アルゴリズムまたは振る舞い

1. 引数 sink を bbsink_lz4 * 型にキャストする。 2. mysink->ctx が存在（非NULL）する場合、LZ4F_freeCompressionContext を呼び出してLZ4圧縮コンテキストを解放する。 3. 解放後、mysink->ctx を NULL に設定し、二重解放を防止する。

呼び出し元と呼び出し先

  • 呼び出し元: bbsink_lz4_ops 構造体を通じて、シンクのライフサイクル管理プロセスから呼び出される。
  • 呼び出し先: LZ4F_freeCompressionContext （LZ4ライブラリ関数）。

関連データ構造

  • bbsink_lz4: 圧縮コンテキスト ctx を保持する構造体。
  • bbsink: 基底となるシンクインターフェース。

失敗モードと境界条件

  • mysink->ctx が既に NULL の場合、関数は何も行わず正常に終了する。

保守担当者向けメモ

  • 本関数は bbsink インターフェースの cleanup コールバックとして登録されている。
  • エラー発生時やバックアップ終了時のリソース解放を確実に行うために使用される。
