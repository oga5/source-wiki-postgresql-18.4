目的

ECPG（Embedded SQL for C）プリプロセッサによって生成された、構造体の配列をホスト変数として使用する際のデータ操作および型変換の正当性を検証する回帰テスト用関数。

シグネチャまたは形状

int main()

入力と出力

  • 入力: なし（プログラム引数なし）
  • 出力: 終了コード（正常終了時は 0）
  • 副作用: データベース ecpg1_regression への接続、テーブル作成、データ挿入、および標準出力へのテスト結果の書き出し。

前提条件と不変条件

  • 実行環境において ecpg1_regression データベースへの接続が可能であること。
  • 構造体のメンバ順序と ECPGdo に渡される引数の順序が、データベースの列定義と一致していること。

アルゴリズムまたは振る舞い

1. ECPGdebug(1, stderr) を呼び出し、デバッグログを有効化する。 2. ECPGconnect でデータベースに接続し、customers テーブルを作成する。 3. ECPGdo を用いてテーブルにデータを挿入する。 4. 構造体の配列（custs1, custs2, custs3 等）をホスト変数として SELECT 文を実行し、結果をバインドする。 5. 各フェッチ操作の直後に sqlca を参照し、エラーや警告をチェックする。 6. 取得したデータを printf で標準出力に書き出し、期待値との整合性を確認する。 7. 処理終了後、接続を解除する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先: ECPGconnect, ECPGdo, ECPGdisconnect, sqlprint, printf（ECPGライブラリおよび標準Cライブラリ関数）

関連データ構造

  • customer, customer2, customer3, customer4: テスト対象の構造体定義。
  • cust_ind: インジケータ変数を保持する構造体。
  • sqlca: SQL 通信領域（エラーハンドリング用）。

失敗モードと境界条件

  • sqlca.sqlcode < 0: SQL 操作でエラーが発生した場合、sqlprint() を呼び出してエラー情報を出力する。
  • sqlca.sqlwarn[0] == 'W': SQL 警告が発生した場合、sqlprint() を呼び出す。
  • ECPG_NOT_FOUND: データが見つからない場合、sqlprint() を呼び出す。

保守担当者向けメモ

  • 本ファイルはプリプロセッサ生成コードであるため、直接編集しないこと。
  • 構造体の定義変更や追加を行う場合は、対応する ECPGdo のバインド引数（ECPGt_...）の整合性を確認すること。
