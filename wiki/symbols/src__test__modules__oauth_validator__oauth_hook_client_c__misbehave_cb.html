misbehave_cb

定義

  • 種別: function
  • 定義箇所: src/test/modules/oauth_validator/oauth_hook_client.c:270

目的

OAuth 認証フックのテストにおいて、意図的に不正な挙動やエラー状態をシミュレートし、libpq の堅牢性を検証するためのコールバック関数です。

シグネチャまたは形状

static PostgresPollingStatusType misbehave_cb(PGconn *conn, PGoauthBearerRequest *req, pgsocket *altsock)

入力と出力

  • **入力**:
  • conn: PGconn 接続オブジェクト。
  • req: 認証要求データ構造。
  • altsock: 非同期処理時に使用するソケットへのポインタ。
  • **出力**:
  • PostgresPollingStatusType: libpq のポーリング状態（PGRES_POLLING_FAILED, PGRES_POLLING_OK, PGRES_POLLING_READING のいずれか）。

前提条件と不変条件

  • グローバル変数 misbehave_mode が有効なモード（fail-async, no-token, no-socket）に設定されている必要があります。
  • 不明なモードが指定された場合、標準エラー出力にメッセージを表示してプロセスを終了します。

アルゴリズムまたは振る舞い

misbehave_mode の値に基づき、以下のいずれかの異常系をシミュレートします。 1. fail-async: PGRES_POLLING_FAILED を返し、認証を失敗させます。 2. no-token: req->token を設定せずに PGRES_POLLING_OK を返し、トークン未設定の異常状態を誘発します。 3. no-socket: *altsock を設定せずに PGRES_POLLING_READING を返し、ソケット未設定の異常状態を誘発します。

呼び出し元と呼び出し先

  • **呼び出し元**: handle_auth_data 関数内において、req->async に本関数が代入されることで、認証プロセス中に libpq から間接的に呼び出されます。
  • **呼び出し先**: fprintf および exit（エラー時のみ）。

関連データ構造

  • PGconn: libpq の接続状態を保持する構造体。
  • PGoauthBearerRequest: OAuth 認証要求の詳細を保持する構造体。
  • PostgresPollingStatusType: 接続ポーリングのステータス列挙型。

失敗モードと境界条件

  • misbehave_mode が定義された文字列と一致しない場合、exit(1) によりプログラムが強制終了します。
  • 各モードは意図的に libpq の期待する契約（トークンの代入やソケットの準備）を破るように設計されています。

保守担当者向けメモ

  • 本関数はテスト専用です。実運用コードには含めないでください。
  • 新しい異常系テストケースを追加する場合は、misbehave_mode の条件分岐に新しいケースを追加してください。
