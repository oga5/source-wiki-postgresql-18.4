ShmemInitHash

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shmem.c:331

目的

共有メモリ上にハッシュテーブルを作成、または既存のハッシュテーブルにアタッチします。

シグネチャまたは形状

HTAB *ShmemInitHash(const char *name, long init_size, long max_size, HASHCTL *infoP, int hash_flags);

入力と出力

  • **入力**:
  • name: ハッシュテーブルを識別するための名前（ShmemIndex用）。
  • init_size: 初期サイズ。
  • max_size: 最大サイズ。
  • infoP: ハッシュキーやバケットサイズ等の設定情報。
  • hash_flags: ハッシュテーブルの動作フラグ。
  • **出力**:
  • 初期化またはアタッチされたハッシュテーブルへのポインタ (HTAB *)。

前提条件と不変条件

  • 共有メモリ上のハッシュテーブルはディレクトリサイズが固定である必要があります（動的な拡張は不可）。
  • infoP->alloc は ShmemAllocNoError に強制的に設定されます。
  • hash_flags には HASH_SHARED_MEM、HASH_ALLOC、HASH_DIRSIZE が強制的に付与されます。

アルゴリズムまたは振る舞い

1. hash_select_dirsize を用いて、max_size に基づく固定ディレクトリサイズを計算し infoP に設定します。 2. ShmemAllocNoError を割り当て関数として指定します。 3. ShmemInitStruct を呼び出し、指定された名前の領域を共有メモリから取得（または新規作成）します。 4. 既に領域が存在する場合（found が真）、hash_flags に HASH_ATTACH を追加します。 5. infoP->hctl に取得した共有メモリ上のヘッダーアドレスを格納します。 6. hash_create を呼び出し、ハッシュテーブルを構築して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/buf_table.c
  • src/backend/storage/ipc/shmem.c (自身の初期化用)
  • src/backend/storage/lmgr/lock.c
  • src/backend/storage/lmgr/predicate.c
  • src/backend/utils/activity/wait_event.c
  • contrib/pg_stat_statements/pg_stat_statements.c
  • **呼び出し先**:
  • hash_select_dirsize
  • ShmemAllocNoError
  • ShmemInitStruct
  • hash_get_shared_size
  • hash_create

関連データ構造

  • HTAB: ハッシュテーブルの制御構造体。
  • HASHCTL: ハッシュテーブル作成時の設定パラメータ。
  • HASHHDR: 共有メモリ上のハッシュテーブルヘッダー。
  • ShmemIndex: 共有メモリ上の全構造体を管理するハッシュテーブル。

失敗モードと境界条件

  • ShmemInitStruct が失敗した場合、内部でエラーが発生する可能性があります。
  • 共有メモリの容量が不足している場合、ShmemAllocNoError の挙動に従いエラーとなります。

保守担当者向けメモ

  • 共有メモリ上のハッシュテーブルは、他のバックエンドから発見可能である必要があるため、ディレクトリの再配置や拡張は行えません。
  • ShmemInitHash は ShmemInitStruct を呼び出すため、ShmemIndex が初期化済みである必要があります。
