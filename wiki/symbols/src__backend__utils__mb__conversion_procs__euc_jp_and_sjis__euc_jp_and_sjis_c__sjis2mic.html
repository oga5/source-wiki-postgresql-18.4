sjis2mic

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_jp_and_sjis/euc_jp_and_sjis.c:162

目的

Shift-JIS（SJIS）エンコーディングの文字列を、PostgreSQL内部表現であるMULE_INTERNAL（MIC）へ変換する。

シグネチャまたは形状

static int sjis2mic(const unsigned char *sjis, unsigned char *p, int len, bool noError)

入力と出力

  • **入力**:
  • sjis: 変換元SJIS文字列へのポインタ。
  • p: 変換先MIC文字列を格納するバッファ。
  • len: 変換対象のバイト長。
  • noError: 変換エラー時にエラー報告を抑制するかどうかを示すフラグ。
  • **出力**:
  • 戻り値: 変換処理したバイト数。
  • p: 変換後のMIC文字列（ヌル終端される）。

前提条件と不変条件

  • sjis は有効なSJISバイト列であること。
  • p は変換結果を格納するのに十分な領域を確保していること。
  • 変換処理は len が0になるまで継続される。

アルゴリズムまたは振る舞い

1. 入力文字列を1文字ずつ走査する。 2. ASCII文字（0x00-0x7F）はそのままコピーする。 3. JIS X 0201（1バイトカナ）は LC_JISX0201K を付与して変換する。 4. マルチバイト文字の場合:

  • ibmkanji テーブルを参照し、NEC選定IBM漢字やその他の拡張文字をマッピングする。
  • JIS X 0208、JIS X 0212、およびユーザー定義文字（UDC）の範囲に応じて、適切なMICコード（LC_JISX0208 または LC_JISX0212）を先頭に付与し、変換後のコードを書き込む。

    • 5. 変換終了後、バッファの末尾に \0 を書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の sjis_to_mic 関数（138行目付近）から呼び出される。
  • **呼び出し先**: report_invalid_encoding（エラー発生時）。

関連データ構造

  • ibmkanji: sjis.map で定義されたSJISとEUC/MIC間の変換テーブル。
  • LC_JISX0201K, LC_JISX0208, LC_JISX0212: MICの文字セット識別子。

失敗モードと境界条件

  • **不正なバイト列**: ISSJISHEAD や ISSJISTAIL の判定に適合しない場合、noError が false ならば report_invalid_encoding を呼び出し、true ならば処理を中断する。
  • **ヌル文字**: 入力中に予期せぬヌル文字が含まれる場合、エラーとして扱われる。
  • **バッファ不足**: len の範囲内で変換が完了しない場合、ループを抜けて処理を終了する。

保守担当者向けメモ

  • 本関数は内部的な変換ロジックであり、直接外部から呼び出されることはない。
  • IBM漢字の変換ロジックは ibmkanji テーブルの構造に依存しているため、マッピングの変更が必要な場合は sjis.map を確認すること。
