r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_armenian.c:435

目的

アルメニア語のステミング処理において、単語内の領域（R2）を特定し、ステミングの適用範囲となる境界位置を計算する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（ステミング環境構造体）。単語データおよび現在のカーソル位置 z->c を保持。
  • **出力**: int（常に 1 を返す）。副作用として z->I[0] および z->I[1] に計算された境界位置が格納される。

前提条件と不変条件

  • z が有効な SN_env 構造体を指していること。
  • g_v 配列がアルメニア語の母音グループとして定義されていること。
  • 処理終了後、z->c（カーソル位置）は呼び出し時の値に復元される。

アルゴリズムまたは振る舞い

1. 初期状態として、領域境界 z->I[0] および z->I[1] を単語の末尾 z->l に設定する。 2. g_v を用いて、母音と子音の交互の出現パターンを走査する。

  • out_grouping_U（子音）と in_grouping_U（母音）を交互に適用し、単語内の特定の構造を探索する。

    • 3. パターンに一致した場合、その位置を z->I[0] および z->I[1] に記録する。 4. 探索の成否に関わらず、カーソル位置 z->c を開始時の値に復元して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は本 excerpt には含まれていない。
  • **呼び出し先**: out_grouping_U および in_grouping_U（Snowball ランタイムのユーティリティ関数）。

関連データ構造

  • SN_env: ステミング環境を管理する構造体。I[] 配列に領域境界を保持する。
  • g_v: アルメニア語の母音を定義する配列。

失敗モードと境界条件

  • 単語が短く、母音・子音のパターンに一致しない場合は、z->I[0] と z->I[1] は初期値（単語末尾）のまま維持される。
  • out_grouping_U や in_grouping_U が失敗した場合、lab0 ラベルへジャンプし、カーソル位置を復元して正常終了する。
