dump_binary

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-bytea.c:29

目的

取得したバイナリデータのデバッグ出力用関数。バイナリの内容を 16 進数形式で標準出力に書き出し、整合性確認を支援する。

シグネチャまたは形状

void dump_binary(char *buf, int len, int ind)

入力と出力

  • **入力**:
  • buf: バイナリデータが格納されたバッファへのポインタ。
  • len: バッファ内のデータ長。
  • ind: インジケータ変数の値（NULL 判定や状態を示す）。
  • **出力**: なし（標準出力へフォーマットされた文字列を出力）。

前提条件と不変条件

  • len は buf が指すメモリ領域の有効なデータ長と一致している必要がある。
  • buf は少なくとも len バイト以上の領域を確保していること。

アルゴリズムまたは振る舞い

1. len（データ長）と ind（インジケータ値）を printf で出力する。 2. buf の各バイトをループで走査し、0xff とのビット論理積をとった値を 16 進数（%02x）で出力する。 3. 最後に改行を出力して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/sql-bytea.c 内の各テストケース（行 30, 161, 162, 219, 257, 258, 289, 290, 350, 351）。
  • **呼び出し先**: 標準ライブラリの printf。

関連データ構造

  • bytea_1 ～ bytea_4 構造体（len メンバと arr 配列を持つ）。本関数はこれらの構造体のメンバを引数として受け取る。

失敗モードと境界条件

  • len が 0 の場合、データ部分は出力されず、len=0, ind=... のみが表示される。
  • buf が NULL ポインタである場合、未定義動作となる可能性がある。

保守担当者向けメモ

  • 本関数は sql-bytea.c の回帰テスト専用である。
  • 出力形式は len=..., ind=..., data=0x... で固定されているため、テストの期待値（ゴールデンファイル）を変更する際は、この出力形式との整合性に注意すること。
