test_dsa_allocate

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_dsa/test_dsa.c:126

目的

指定された範囲のページ数でDSA（動的共有メモリ領域）の作成、メモリ割り当て、解放、デタッチを繰り返し実行し、セグメント生成ロジックおよび境界条件を検証する。

シグネチャまたは形状

Datum test_dsa_allocate(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_INT32(0): 開始ページ数 (start_num_pages)
  • PG_GETARG_INT32(1): 終了ページ数 (end_num_pages)
  • PG_GETARG_INT32(2): ステップ数 (step)
  • **出力**: なし (PG_RETURN_VOID())

前提条件と不変条件

  • start_num_pages が end_num_pages 以下であること。これに反する場合、elog(ERROR) が発生する。
  • LWLockNewTrancheId() によって確保されたロックトランチは、テスト実行プロセス終了まで解放されない（リークする）。

アルゴリズムまたは振る舞い

1. 引数の検証を行い、不正な場合はエラーを投げる。 2. LWLockNewTrancheId() で新しいロックトランチを確保し、LWLockRegisterTranche() で登録する。 3. start_num_pages から end_num_pages まで step ずつ値を増加させながらループを実行する。 4. ループ内での処理:

  • dsa_create() で新しいDSA領域を作成する。
  • dsa_allocate() で usable_pages * FPM_PAGE_SIZE 分のメモリを確保する。
  • dsa_free() で確保したメモリを解放する。
  • dsa_detach() でDSA領域を破棄する。

    • 5. ループ終了後、値を返さずに終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されることを想定（PG_FUNCTION_INFO_V1 マクロによる登録あり）。
  • **呼び出し先**:
  • LWLockNewTrancheId, LWLockRegisterTranche
  • dsa_create, dsa_allocate, dsa_free, dsa_detach
  • elog

関連データ構造

  • dsa_area: DSA領域を管理する構造体。
  • dsa_pointer: DSA領域内のメモリ位置を示すポインタ。
  • FPM_PAGE_SIZE: 割り当てサイズの計算に使用される定数。

失敗モードと境界条件

  • start_num_pages > end_num_pages の場合、エラー終了する。
  • DSAの作成や割り当てが失敗した場合、基盤となる dsa.c の実装に従いエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はテスト専用であり、ロックトランチのリークを許容している。
  • セグメント生成ロジックの検証を目的としているため、ループ内でのDSA領域の生成・破棄が繰り返される点に注意すること。
