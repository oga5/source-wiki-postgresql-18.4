random_relative_prime

定義

  • 種別: function
  • 定義箇所: contrib/tsm_system_rows/tsm_system_rows.c:321

目的

与えられた整数 n に対して、n と互いに素（最大公約数が 1）であるランダムな整数 r を生成します。これは、リレーション内のブロックを重複なく網羅的に走査するためのステップ幅を決定するために使用されます。

シグネチャまたは形状

static uint32 random_relative_prime(uint32 n, pg_prng_state *randstate)

入力と出力

  • **入力**:
  • n (uint32): 互いに素な数を選択するための上限値。
  • randstate (pg_prng_state *): 乱数生成器の状態。
  • **出力**:
  • uint32: 1 以上 n-1 以下の範囲で、n と互いに素な整数。

前提条件と不変条件

  • n <= 1 の場合は常に 1 を返します。
  • 戻り値 r は gcd(r, n) == 1 を満たします。

アルゴリズムまたは振る舞い

1. n <= 1 の場合は直ちに 1 を返します。 2. do-while ループ内で、sampler_random_fract を用いて 0 から n 未満の乱数 r を生成します。 3. r が 0 であるか、gcd(r, n) が 1 より大きい場合はループを継続します。 4. ループ内では CHECK_FOR_INTERRUPTS() を呼び出し、割り込み要求を処理します。 5. 条件を満たす r が得られた時点でその値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/tsm_system_rows/tsm_system_rows.c: system_rows_beginsamplescan 内で sampler->step の計算に使用。
  • contrib/tsm_system_time/tsm_system_time.c: 同様にサンプリングのステップ幅決定に使用。
  • **呼び出し先**:
  • sampler_random_fract: 乱数生成。
  • gcd: 最大公約数の計算。
  • CHECK_FOR_INTERRUPTS: 割り込みチェック。

関連データ構造

  • pg_prng_state: 乱数生成器の状態を保持する構造体。

失敗モードと境界条件

  • n が小さい場合（n <= 1）、無限ループを避けるために 1 を返す安全策が講じられています。
  • 乱数生成の結果として 0 が選択された場合、gcd の計算対象から除外されます。

保守担当者向けメモ

  • 互いに素な数が見つかる確率は約 61% であるため、ループ回数は通常 2〜3 回程度ですが、割り込みチェックが含まれているため、長時間実行されるクエリでも安全です。
