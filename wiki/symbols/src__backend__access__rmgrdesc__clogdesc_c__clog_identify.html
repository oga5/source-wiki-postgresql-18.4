clog_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/clogdesc.c:43

目的

WAL（Write-Ahead Log）レコードの info ビットから、CLOG（Commit Log）に関連する操作の種類を特定し、人間が読み取り可能な文字列として返す。

シグネチャまたは形状

const char *clog_identify(uint8 info);

入力と出力

  • **入力**: info (uint8) - WALレコードに含まれる情報ビット。
  • **出力**: const char * - 操作の種類を表す文字列（"ZEROPAGE" または "TRUNCATE"）。該当しない場合は NULL を返す。

前提条件と不変条件

  • info ビットには XLR_INFO_MASK が含まれている可能性があるため、関数内部でビットマスク処理を行い、純粋な操作タイプを抽出する。

アルゴリズムまたは振る舞い

1. info に対して ~XLR_INFO_MASK を適用し、補助フラグを除去する。 2. 抽出された値に基づき switch 文で判定を行う。

  • CLOG_ZEROPAGE の場合: "ZEROPAGE" を返す。
  • CLOG_TRUNCATE の場合: "TRUNCATE" を返す。

    • 3. いずれにも一致しない場合、初期値である NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/access/rmgrlist.h において、RM_CLOG_ID リソースマネージャの定義の一部として登録されている。
  • **呼び出し先**:
  • 提供されたソースコード内では、他の関数を呼び出していない。

関連データ構造

  • CLOG_ZEROPAGE, CLOG_TRUNCATE: access/clog.h で定義されたCLOG操作の定数。
  • XLR_INFO_MASK: WALレコードのフラグを分離するためのマスク定数。

失敗モードと境界条件

  • info が既知のCLOG操作（ZEROPAGE または TRUNCATE）のいずれにも該当しない場合、NULL を返す。

保守担当者向けメモ

  • 新しいCLOG操作が追加された場合は、本関数の switch 文に新しい case を追加する必要がある。
  • pg_waldump 等のツールがWALを解析する際に利用されるため、戻り値の文字列は一貫性を保つこと。
