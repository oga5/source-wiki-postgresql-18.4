HandleCatchupInterrupt

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/sinval.c:153

目的

共有キャッシュ無効化（SI）メッセージの処理が滞っているバックエンドに対し、強制的なキャッシュ更新（キャッチアップ）を予約するために使用されます。

シグネチャまたは形状

void HandleCatchupInterrupt(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルフラグとラッチを更新）

前提条件と不変条件

  • 本関数は**シグナルハンドラから呼び出されます**。そのため、実行可能な操作は極めて限定的である必要があります。

アルゴリズムまたは振る舞い

1. グローバル変数 catchupInterruptPending を true に設定し、キャッチアップが必要であることを記録します。 2. SetLatch(MyLatch) を呼び出し、メインループが待機状態にある場合に即座に復帰してイベントを処理できるようにします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/procsignal.c 内のシグナル処理ロジックから呼び出されることが確認されています。
  • 呼び出し先: SetLatch（src/backend/storage/ipc/latch.c）を呼び出します。

関連データ構造

  • catchupInterruptPending: キャッチアップの保留状態を保持するグローバルフラグ。
  • MyLatch: 現在のバックエンドプロセスに関連付けられたラッチ。

失敗モードと境界条件

  • シグナルハンドラ内で実行されるため、非同期安全でない関数や複雑なロック操作を行うことはできません。

保守担当者向けメモ

  • 本関数はシグナルハンドラから呼ばれるため、実際のキャッシュ同期処理は行いません。実際の処理は、メインループ側で catchupInterruptPending を確認した後に呼び出される ProcessCatchupInterrupt が担当します。
