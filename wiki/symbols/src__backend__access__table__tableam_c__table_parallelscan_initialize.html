table_parallelscan_initialize

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/tableam.c:145

目的

並列スキャン記述子（ParallelTableScanDesc）を初期化し、指定されたスナップショットを共有メモリ領域へシリアライズして、並列ワーカー間でスキャン状態を共有可能にする。

シグネチャまたは形状

void table_parallelscan_initialize(Relation rel, ParallelTableScanDesc pscan, Snapshot snapshot)

入力と出力

  • **入力**:
  • rel: 対象のテーブルリレーション。
  • pscan: 初期化対象の並列スキャン記述子。
  • snapshot: スキャンに使用するスナップショット。
  • **出力**: なし（pscan 構造体の内部状態を更新）。

前提条件と不変条件

  • rel->rd_tableam が有効であること。
  • pscan は、table_parallelscan_estimate で算出されたサイズに基づいて確保されたメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. リレーション固有のAM実装である rd_tableam->parallelscan_initialize を呼び出し、AM固有の初期化を実行してスナップショット格納用のオフセットを取得する。 2. 取得したオフセットを pscan->phs_snapshot_off に保存する。 3. snapshot が MVCC スナップショットである場合、SerializeSnapshot を用いて pscan 内の指定オフセット位置へスナップショットをシリアライズし、phs_snapshot_any を false に設定する。 4. snapshot が SnapshotAny である場合、シリアライズは行わず phs_snapshot_any を true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/nbtree/nbtsort.c
  • src/backend/executor/nodeSeqscan.c
  • **呼び出し先**:
  • rel->rd_tableam->parallelscan_initialize (AM固有の実装)
  • SerializeSnapshot (スナップショットのシリアライズ)

関連データ構造

  • Relation: テーブルのメタデータ構造。
  • ParallelTableScanDesc: 並列スキャンの共有状態を保持する構造体。
  • Snapshot: トランザクションの可視性を制御するスナップショット構造体。

失敗モードと境界条件

  • rel->rd_tableam が未設定の場合、セグメンテーション違反が発生する可能性がある。
  • pscan に割り当てられたメモリが不足している場合、シリアライズ時にメモリ破壊が発生する可能性がある。

保守担当者向けメモ

  • 本関数は、AM固有の初期化処理と汎用的なスナップショットのシリアライズ処理を橋渡しする役割を持つ。
  • SnapshotAny の場合はシリアライズをスキップする最適化が行われているため、新しいスナップショットタイプを追加する際は、この条件分岐が適切か確認すること。
