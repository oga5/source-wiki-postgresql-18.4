plpython_to_hstore

定義

  • 種別: function
  • 定義箇所: contrib/hstore_plpython/hstore_plpython.c:124

目的

Python の辞書（マッピング）オブジェクトを PostgreSQL の hstore 型に変換する。

シグネチャまたは形状

Datum plpython_to_hstore(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_POINTER(0) を介して取得される PyObject*（Python 辞書）。
  • **出力**: hstore 構造体へのポインタを Datum として返す。

前提条件と不変条件

  • 入力オブジェクトは Python のマッピング型である必要がある。
  • シーケンス型（リスト等）はマッピングとして扱われず、エラーとなる。
  • PG_TRY/PG_FINALLY ブロックにより、変換処理中のエラー発生時でも items オブジェクトの参照カウントが適切に解放される。

アルゴリズムまたは振る舞い

1. 入力がマッピング型であることを PySequence_Check および PyMapping_Check で検証する。 2. PyMapping_Items で辞書からキーと値のペアを取得する。 3. 各ペアを走査し、PLyObject_AsString を用いて文字列に変換する。

  • 値が Py_None の場合は hstore の NULL として扱う。
  • それ以外は文字列値として格納する。

    • 4. hstoreCheckKeyLen および hstoreCheckValLen で各要素の長さを検証する。 5. hstoreUniquePairs で重複キーを排除し、hstorePairs で最終的な hstore 構造体を生成する。 6. 処理終了後、Py_DECREF で items を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェースから呼び出される（PG_FUNCTION_INFO_V1 マクロによる登録）。
  • **呼び出し先**:
  • Python C API: PySequence_Check, PyMapping_Check, PyMapping_Size, PyMapping_Items, PyList_GetItem, PyTuple_GetItem, Py_DECREF
  • PostgreSQL/hstore API: palloc_array, hstoreCheckKeyLen, hstoreCheckValLen, hstoreUniquePairs, hstorePairs
  • 内部ユーティリティ: PLyObject_AsString

関連データ構造

  • PyObject: Python オブジェクトの汎用構造体。
  • Pairs: hstore 変換用の中間データ構造。
  • HStore: PostgreSQL の hstore 型の内部表現。

失敗モードと境界条件

  • 入力がマッピング型でない場合、ERRCODE_WRONG_OBJECT_TYPE でエラーを報告する。
  • キーや値の長さが hstore の制限を超える場合、hstoreCheckKeyLen / hstoreCheckValLen 内でエラーが発生する可能性がある。

保守担当者向けメモ

  • Python 3.10 未満の環境をサポートするため、マッピング判定に PySequence_Check を併用している。
  • PG_TRY ブロック内でのメモリ割り当てと Python オブジェクトの参照カウント管理が、メモリリーク防止の要となっている。
