printtup_create_DR

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/printtup.c:70

目的

クエリ結果のタプルをクライアントへ転送するための DestReceiver オブジェクトを生成・初期化します。

シグネチャまたは形状

DestReceiver *printtup_create_DR(CommandDest dest);

入力と出力

  • **入力**: dest (CommandDest) - 出力先を指定する列挙型。
  • **出力**: DestReceiver * - 初期化された DR_printtup 構造体へのポインタ。

前提条件と不変条件

  • palloc0 によりメモリがゼロクリアされた状態で割り当てられます。
  • self->pub.receiveSlot にはデフォルトの printtup 関数が設定されます。
  • self->sendDescrip は dest == DestRemote の場合にのみ真となります。

アルゴリズムまたは振る舞い

1. DR_printtup 構造体分のメモリを確保します。 2. DestReceiver インターフェースの関数ポインタ（receiveSlot, rStartup, rShutdown, rDestroy）を、printtup.c 内の対応する関数で初期化します。 3. mydest に引数の dest を格納します。 4. sendDescrip フラグを dest の値に基づいて設定します。 5. 内部状態（attrinfo, nattrs, myinfo, buf.data, tmpcontext）を初期値（NULL または 0）で初期化します。 6. 初期化済みの DestReceiver ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/dest.c:124 から呼び出されることが確認されています。
  • **呼び出し先**: palloc0 (メモリ確保)。

関連データ構造

  • DR_printtup: 本関数が生成するプライベート状態構造体。
  • DestReceiver: 本関数が返すインターフェース構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • dest が DestRemote 以外の場合、sendDescrip は偽となり、メタデータ送信の挙動が抑制されます。

保守担当者向けメモ

  • receiveSlot は後続の処理で変更される可能性があるため、固定的な参照は避けてください。
  • sendDescrip の判定ロジックは DestRemoteExecute を除外する仕様となっており、プロトコル上のメッセージ送信制御に直結しています。
