ExecIndexOnlyScanInitializeWorker

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexonlyscan.c:826

目的

パラレルクエリ実行時、各ワーカープロセスにおいてインデックスのみスキャン（Index-Only Scan）の状態を初期化し、共有メモリ（DSM）上の情報と同期させる。

シグネチャまたは形状

void ExecIndexOnlyScanInitializeWorker(IndexOnlyScanState *node, ParallelWorkerContext *pwcxt)

入力と出力

  • **入力**:
  • node: 初期化対象の IndexOnlyScanState 構造体。
  • pwcxt: 並列実行コンテキスト。共有メモリのテーブル（TOC）へのアクセスに使用。
  • **出力**: なし（副作用として node の内部状態を更新）。

前提条件と不変条件

  • node->ss.ps.plan が有効であること。
  • pwcxt->toc に、当該プランノードIDに対応する ParallelIndexScanDesc が格納されていること。

アルゴリズムまたは振る舞い

1. **初期化判定**: 計測（instrument）または並列実行（parallel_aware）が不要な場合は即座に終了する。 2. **DSM参照**: shm_toc_lookup を使用して、共有メモリ上のインデックススキャン記述子を取得する。 3. **統計情報の紐付け**: 計測が有効な場合、共有メモリ上の統計情報領域を node->ioss_SharedInfo にマッピングする。 4. **並列スキャンの開始**: parallel_aware が有効な場合、index_beginscan_parallel を呼び出してスキャン記述子を生成し、node->ioss_ScanDesc に設定する。 5. **スキャンの再開**: ランタイムキーが不要、または準備完了状態であれば、index_rescan を呼び出してインデックススキャンを初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:1351（直接的な呼び出しを確認）。
  • **呼び出し先**:
  • shm_toc_lookup
  • OffsetToPointer
  • index_beginscan_parallel
  • index_rescan

関連データ構造

  • IndexOnlyScanState: スキャンノードの実行状態を保持。
  • ParallelIndexScanDesc: 並列スキャン用の共有メモリ構造体。
  • SharedIndexScanInstrumentation: 並列実行時の統計情報共有用構造体。

失敗モードと境界条件

  • shm_toc_lookup が失敗した場合（false 指定のため）、エラーが発生する可能性がある。
  • parallel_aware が偽の場合、index_beginscan_parallel は呼び出されず、統計情報のセットアップのみが行われる。

保守担当者向けメモ

  • 本関数は、ワーカープロセスが並列実行に参加する際のDSMアタッチ処理を担う。
  • ioss_ScanDesc->xs_want_itup = true が設定されており、インデックスのみスキャン特有のタプル取得要件が強制されている点に注意。
