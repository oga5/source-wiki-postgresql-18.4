PGTYPEStimestamp_add_interval

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/timestamp.c:857

目的

timestamp 型の値に interval 型の期間を加算し、結果を新しい timestamp として出力する。

シグネチャまたは形状

int PGTYPEStimestamp_add_interval(timestamp * tin, interval * span, timestamp * tout)

入力と出力

  • tin (入力): 加算対象のタイムスタンプ（マイクロ秒単位の int64）。
  • span (入力): 加算する期間（月数および時間単位のオフセットを含む）。
  • tout (出力): 計算結果を格納するポインタ。
  • 戻り値: 成功時は 0、エラー時は -1 を返す。

前提条件と不変条件

  • tin が infinity などの有限でない値である場合、計算を行わず tout に tin をそのままコピーする。
  • span->month が 0 でない場合、内部的に struct tm への分解と再構築を伴う複雑な計算が行われる。

アルゴリズムまたは振る舞い

1. tin が有限値か確認し、有限でなければそのまま tout に代入して終了する。 2. span->month が 0 でない場合:

  • timestamp2tm を用いて tin を struct tm に分解する。
  • 月を加算し、年と月の繰り上げ・繰り下げを調整する。
  • 月末日の境界問題（例: 2月30日など）を day_tab を用いて補正する。
  • tm2timestamp を用いて struct tm を tin に再変換する。

    • 3. span->time（マイクロ秒単位のオフセット）を tin に直接加算する。 4. 最終的な値を tout に格納する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/interfaces/ecpg/pgtypeslib/timestamp.c (内部的な再帰的呼び出し)
  • src/interfaces/ecpg/test/expected/compat_informix-test_informix2.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-dt_test2.c
  • 呼び出し先:
  • timestamp2tm
  • tm2timestamp
  • isleap (マクロ)

関連データ構造

  • timestamp: int64 型（マイクロ秒単位）。
  • interval: 月数および時間オフセットを保持する構造体。
  • struct tm: C標準の日時構造体。

失敗モードと境界条件

  • timestamp2tm または tm2timestamp が失敗した場合、-1 を返す。
  • span->month の加算時に年を跨ぐ計算が発生する場合、day_tab を参照して月末日の補正を行う。

保守担当者向けメモ

  • 本関数は timestamp の精度（マイクロ秒）を維持する。
  • 月単位の加算は struct tm を経由するため、時間単位の加算よりも計算コストが高い。
  • day_tab を使用した月末補正は、うるう年を考慮した isleap マクロに依存している。
