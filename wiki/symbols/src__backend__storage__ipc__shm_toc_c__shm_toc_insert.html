shm_toc_insert

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shm_toc.c:170

目的

共有メモリセグメント内の目次（TOC）に、特定のキーとデータ構造の相対オフセットを登録する。

シグネチャまたは形状

void shm_toc_insert(shm_toc *toc, uint64 key, void *address)

入力と出力

  • toc: 操作対象の目次構造体へのポインタ。
  • key: 登録するデータ構造を一意に識別するための64ビットキー。
  • address: 登録対象のデータ構造が配置されている共有メモリ上の絶対アドレス。
  • 出力: なし（成功時）。失敗時はエラーを報告する。

前提条件と不変条件

  • address は toc 構造体よりも後のメモリ領域を指している必要がある。
  • 共有メモリセグメントの容量を超過しないこと。
  • toc_nentry が PG_UINT32_MAX を超えないこと。

アルゴリズムまたは振る舞い

1. address を toc の先頭からの相対オフセットに変換する。 2. toc->toc_mutex を取得し、排他制御を行う。 3. 現在の TOC エントリ数と割り当て済みバイト数に基づき、新しいエントリを追加する十分な空きがあるか確認する。不足している場合はエラーを発生させる。 4. toc_entry 配列の末尾に key と offset を書き込む。 5. pg_write_barrier() を実行し、エントリ内容の書き込みが完了したことを保証する。 6. toc_nentry をインクリメントし、エントリの有効化を完了させる。 7. toc_mutex を解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/brin/brin.c
  • src/backend/access/common/session.c
  • src/backend/access/gin/gininsert.c
  • 呼び出し先:
  • SpinLockAcquire / SpinLockRelease
  • pg_write_barrier
  • ereport

関連データ構造

  • shm_toc: 目次を保持する構造体。
  • shm_toc_entry: キーとオフセットを保持するエントリ構造体。

失敗モードと境界条件

  • メモリ不足: total_bytes を超えるエントリ追加を試みた場合、ERRCODE_OUT_OF_MEMORY でエラー終了する。
  • オーバーフロー: toc_bytes の計算結果がラップアラウンドする場合、エラー終了する。
  • エントリ数上限: nentry が PG_UINT32_MAX に達している場合、エラー終了する。

保守担当者向けメモ

  • 本関数はロックを取得して更新を行うが、shm_toc_lookup はロックフリーで動作するように設計されている。そのため、pg_write_barrier() の位置と順序は整合性維持のために極めて重要である。
  • 多数のキーを登録する用途には適していない。多数のポインタが必要な場合は、単一の構造体へのポインタを登録し、その内部で管理すること。
