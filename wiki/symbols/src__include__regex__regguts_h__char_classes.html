char_classes

定義

  • 種別: enum
  • 定義箇所: src/include/regex/regguts.h:146

目的

正規表現エンジンにおいて、文字クラス（[:alnum:], [:digit:] 等）を識別するための内部コードを定義します。

シグネチャまたは形状

enum char_classes {
    CC_ALNUM, CC_ALPHA, CC_ASCII, CC_BLANK, CC_CNTRL, CC_DIGIT, CC_GRAPH,
    CC_LOWER, CC_PRINT, CC_PUNCT, CC_SPACE, CC_UPPER, CC_XDIGIT, CC_WORD
};

入力と出力

  • 入力: 正規表現の構文解析時に、特定の文字クラス指定を識別するために使用されます。
  • 出力: lookupcclass 等の関数により、特定の文字クラスに対応する内部コードとして返されます。

前提条件と不変条件

  • regc_locale.c 内のテーブルエントリは、この列挙型の順序と一致している必要があります。

アルゴリズムまたは振る舞い

  • 正規表現コンパイル時、lookupcclass が文字列から対応する char_classes コードを特定します。
  • 特定されたコードに基づき、charclass や cclasscvec が文字クラスの集合（cvec）を構築し、NFA の状態遷移に反映させます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/regex/regc_locale.c: 文字クラスの判定およびコード変換に使用。
  • src/backend/regex/regcomp.c: 正規表現のコンパイル過程で、文字クラスの処理（charclass, charclasscomplement, lookupcclass, cclasscvec）に使用。
  • 呼び出し先: 本シンボル自体は列挙型定義であり、関数を呼び出すことはありません。

関連データ構造

  • struct cvec: 文字クラスに含まれる文字の集合を保持する構造体。
  • struct vars: コンパイル時の状態を保持する構造体。

失敗モードと境界条件

  • 該当する文字クラスが見つからない場合、lookupcclass は適切なエラーハンドリングまたはデフォルト値の返却を行う必要があります。

保守担当者向けメモ

  • 新しい文字クラスを追加する場合は、この列挙型を更新するだけでなく、regc_locale.c 内の対応するマッピングテーブルやロジックも更新する必要があります。
