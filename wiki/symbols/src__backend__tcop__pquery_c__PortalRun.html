PortalRun

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/pquery.c:684

目的

ポータル（Portal）に関連付けられたクエリを実行し、結果を DestReceiver を介して配送する。

シグネチャまたは形状

bool PortalRun(Portal portal, long count, bool isTopLevel,
               DestReceiver *dest, DestReceiver *altdest,
               QueryCompletion *qc)

入力と出力

  • **入力**:
  • portal: 実行対象のポータル構造体。
  • count: フェッチする行数。
  • isTopLevel: 最上位レベルの呼び出しかどうかを示すフラグ。
  • dest: 主な結果出力先。
  • altdest: 補助的な出力先（PORTAL_MULTI_QUERY で使用）。
  • qc: 実行完了情報を格納する構造体（ポインタ）。
  • **出力**:
  • bool: 実行が完了し、これ以上フェッチするデータがない場合に true を返す。

前提条件と不変条件

  • PortalIsValid(portal) が真であること。
  • 実行中、ActivePortal、CurrentResourceOwner、PortalContext などのグローバル変数が一時的にポータルの状態に切り替えられ、終了時に復元される。
  • PG_TRY/CATCH ブロックにより、エラー発生時にはポータル状態が PORTAL_FAILED に遷移し、リソースが適切に管理される。

アルゴリズムまたは振る舞い

1. **初期化**: TRACE_POSTGRESQL_QUERY_EXECUTE_START を発行し、qc を初期化。 2. **状態設定**: MarkPortalActive(portal) を呼び出し、ポータル実行用のメモリコンテキストとリソースオーナーをセットアップする。 3. **戦略分岐**: portal->strategy に基づき実行を分岐する。

  • PORTAL_ONE_SELECT 等: 必要に応じて FillPortalStore で結果を蓄積し、PortalRunSelect で指定行数をフェッチする。
  • PORTAL_MULTI_QUERY: PortalRunMulti を呼び出し、完了後に MarkPortalDone を実行する。

    • 4. **終了処理**: PG_CATCH ブロックでエラーを捕捉し、ポータルを失敗状態にしてから再スローする。正常終了時はコンテキストを復元し、TRACE_POSTGRESQL_QUERY_EXECUTE_DONE を発行して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/prepare.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • FillPortalStore
  • PortalRunSelect
  • PortalRunMulti
  • MarkPortalActive / MarkPortalDone / MarkPortalFailed
  • MemoryContextSwitchTo

関連データ構造

  • Portal: 実行状態を保持するコンテナ。
  • DestReceiver: 結果の配送先インターフェース。
  • QueryCompletion: クエリ実行後のステータス情報。

失敗モードと境界条件

  • 未知の portal->strategy が指定された場合、elog(ERROR) を発生させる。
  • 実行中にエラーが発生した場合、PG_CATCH によりポータルは PORTAL_FAILED 状態となり、呼び出し元へエラーが伝播する。
  • PORTAL_MULTI_QUERY 以外の戦略では、実行統計のログ出力（log_executor_stats）が行われる。

保守担当者向けメモ

  • VACUUM や CLUSTER のような内部でトランザクションを制御するコマンドを扱うため、TopTransactionResourceOwner 等のグローバル変数の退避・復元処理が複雑になっている。この箇所を変更する際は、メモリコンテキストの切り替え順序に注意すること。
