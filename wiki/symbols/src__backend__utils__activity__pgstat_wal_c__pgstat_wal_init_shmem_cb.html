pgstat_wal_init_shmem_cb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_wal.c:148

目的

WAL統計情報を保持する共有メモリ領域の初期化を行う。具体的には、統計構造体に含まれる排他制御用のロックを初期化する。

シグネチャまたは形状

void pgstat_wal_init_shmem_cb(void *stats)

入力と出力

  • **入力**: stats (共有メモリ上のWAL統計構造体 PgStatShared_Wal へのポインタ)
  • **出力**: なし（副作用として stats が指す構造体内のロックが初期化される）

前提条件と不変条件

  • stats 引数は PgStatShared_Wal 型としてキャスト可能であること。
  • 本関数は、統計サブシステムの共有メモリ初期化フェーズにおいて呼び出されること。

アルゴリズムまたは振る舞い

1. 引数 stats を PgStatShared_Wal * 型にキャストする。 2. LWLockInitialize を呼び出し、構造体内の lock メンバを LWTRANCHE_PGSTATS_DATA トランシェで初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c:479 （統計サブシステムの初期化プロセスからコールバックとして登録・呼び出し）
  • **呼び出し先**: LWLockInitialize

関連データ構造

  • PgStatShared_Wal: WAL統計を共有メモリ上で保持するための構造体。
  • LWLock: 統計データへのアクセスを同期するための軽量ロック。

失敗モードと境界条件

  • stats が NULL の場合、デリファレンスによりクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数は pgstat モジュールのコールバックインターフェースの一部である。
  • LWTRANCHE_PGSTATS_DATA トランシェを使用しているため、ロックの競合状況やデッドロックの解析時にはこのトランシェの挙動を考慮すること。
