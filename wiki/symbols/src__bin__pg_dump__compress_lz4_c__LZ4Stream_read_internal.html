LZ4Stream_read_internal

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_lz4.c:450

目的

LZ4 圧縮された入力ストリームからデータを読み込み、解凍して呼び出し元へ提供する。オーバーフローバッファを活用し、チャンク単位の解凍結果を適切に管理する。

シグネチャまたは形状

static int LZ4Stream_read_internal(LZ4State *state, void *ptr, int ptrsize, bool eol_flag)

入力と出力

  • **入力**:
  • state: LZ4 圧縮コンテキスト。
  • ptr: 解凍データを格納するバッファ。
  • ptrsize: 要求するデータサイズ。
  • eol_flag: 改行文字（\n）までを読み込むかを示すフラグ。
  • **出力**:
  • 成功時: 実際に読み込まれたバイト数。
  • 失敗時: -1。

前提条件と不変条件

  • LZ4Stream_init による遅延初期化が内部で行われる。
  • ptrsize が 0 以下の場合、処理を行わず 0 を返す。
  • state->buffer は ptrsize を保持できるサイズに動的に拡張される。

アルゴリズムまたは振る舞い

1. **初期化**: LZ4Stream_init を呼び出し、コンテキストを準備する。 2. **オーバーフロー確認**: LZ4Stream_read_overflow を呼び出し、既に解凍済みでバッファに残っているデータがあればそれを優先的に返す。 3. **解凍ループ**:

  • 入力ファイルから fread で圧縮データを読み込む。
  • LZ4F_decompress を使用してデータを解凍し、state->buffer に格納する。
  • eol_flag が有効な場合、\n を探索し、見つかった位置までを ptr にコピーする。
  • 解凍されたデータが要求サイズを超えた場合、残りを state->overflowbuf に蓄積する。

    • 4. **終了**: 読み込み完了または要求サイズに達した時点で、取得したバイト数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/compress_lz4.c 内のストリーム読み込み関数群（LZ4Stream_read 等）。
  • **呼び出し先**:
  • LZ4Stream_init, LZ4Stream_read_overflow, LZ4F_decompress, LZ4F_getErrorName, LZ4F_isError, pg_malloc, pg_realloc, pg_free, fread, memchr, memcpy, memmove, memset。

関連データ構造

  • LZ4State: 圧縮・解凍状態、バッファ、オーバーフロー領域を保持する構造体。

失敗モードと境界条件

  • LZ4Stream_init 失敗時: エラーログを出力し -1 を返す。
  • fread 失敗時: エラーログを出力し -1 を返す。
  • LZ4F_decompress 失敗時: LZ4 ライブラリのエラーコードを記録し -1 を返す。
  • ptrsize が 0 の場合: 0 を返す。

保守担当者向けメモ

  • 本関数は LZ4F API をラップしており、解凍後のデータが要求サイズと一致しない場合のバッファ管理（オーバーフローバッファ）が重要である。
  • eol_flag 使用時は、改行文字の有無によって戻り値の挙動が変わるため注意が必要。
