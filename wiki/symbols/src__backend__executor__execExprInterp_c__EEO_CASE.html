EEO_CASE

定義

  • 種別: macro (式評価ステップのラベル定義)
  • 定義箇所: src/backend/executor/execExprInterp.c:1741 (および関連箇所)

目的

PostgreSQL の式評価エンジンにおいて、ExprEvalOp で定義された各命令（ステップ）に対応する実行コードブロックの開始位置をラベル付けするために使用されます。

シグネチャまたは形状

EEO_CASE(name)
  • name: ExprEvalOp 列挙型に含まれる命令名（例: EEOP_SBSREF_FETCH）。

入力と出力

  • 入力: 実行コンテキスト (econtext)、式状態 (state)、現在の命令ステップ (op)。
  • 出力: なし（副作用として op->resvalue 等への書き込みや、EEO_NEXT() による次のステップへの遷移が行われる）。

前提条件と不変条件

  • EEO_USE_COMPUTED_GOTO の有効/無効に応じて、コンパイル時に case name: またはラベル CASE_name: に展開されます。
  • 呼び出し元は、ExecInterpExpr 内のディスパッチループ内である必要があります。

アルゴリズムまたは振る舞い

1. 指定された name に対応するコードブロックの入口を定義します。 2. 実行エンジンが opcode に基づいてジャンプしてきた際、対応する処理（例: EEOP_SBSREF_FETCH であれば subscriptfunc の実行）を開始します。 3. 処理完了後、EEO_NEXT() マクロを呼び出し、次の命令ステップへ制御を移します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execExprInterp.c 内の ExecInterpExpr 関数。
  • 呼び出し先: 各命令に対応する具体的な処理ロジック（例: op->d.sbsref.subscriptfunc）。

関連データ構造

  • ExprEvalStep: 式評価の各ステップを保持する構造体。
  • ExprEvalOp: 実行可能な命令の列挙型。

失敗モードと境界条件

  • 命令コードが不正な場合、ディスパッチが失敗し未定義の動作を引き起こす可能性があります。
  • subscriptfunc 等の関数ポインタが NULL の場合、実行時にセグメンテーションフォールトが発生します。

保守担当者向けメモ

  • 本マクロは式評価のホットパスを構成します。新しい命令を追加する際は、execExpr.c でのステップ生成と、本ファイルでの EEO_CASE による実装の両方が必要です。
  • デバッグ時は EEO_USE_COMPUTED_GOTO を無効化することで、標準的な switch 文として動作させることが可能です。
