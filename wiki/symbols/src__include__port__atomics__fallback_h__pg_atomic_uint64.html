pg_atomic_uint64

定義

  • 種別: struct
  • 定義箇所: src/include/port/atomics/fallback.h:26

目的

ハードウェアレベルでの64ビットアトミック操作をサポートしていないプラットフォームにおいて、ソフトウェアによる代替実装（シミュレーション）を提供するためのデータ構造です。

シグネチャまたは形状

typedef struct pg_atomic_uint64
{
	int			sema;
	volatile uint64 value;
} pg_atomic_uint64;

入力と出力

  • sema: 排他制御用のセマフォ識別子。
  • value: アトミック性を保証すべき64ビットのデータ値。

前提条件と不変条件

  • 本構造体は PG_HAVE_ATOMIC_U64_SUPPORT が定義されていない環境でのみ使用されます。
  • value フィールドへのアクセスは、必ずセマフォ（sema）を用いた排他制御を伴う必要があります。

アルゴリズムまたは振る舞い

ハードウェアによるネイティブなアトミック操作が不可能な環境において、セマフォを用いたロック機構により、64ビット変数の読み書きおよび更新操作の整合性を保証します。

呼び出し元と呼び出し先

  • **呼び出し元（参照例）**:
  • src/backend/access/transam/xlog.c: insertingAt, unloggedLSN, logInsertResult 等の変数として使用。
  • src/backend/access/transam/xlogprefetcher.c: reset_time, prefetch, hit 等の統計管理に使用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、操作には pg_atomic_compare_exchange_u64_impl 等の _impl 接尾辞を持つ関数群がセマフォを介して操作を行います。

関連データ構造

  • pg_atomic_uint32: 32ビット版のアトミック操作用構造体。
  • pg_atomic_flag: フラグ操作用のアトミック構造体。

失敗モードと境界条件

  • セマフォを使用するため、ハードウェアネイティブなアトミック操作と比較してオーバーヘッドが大きく、高頻度な更新が発生するパスではパフォーマンス低下を招く可能性があります。

保守担当者向けメモ

  • 本ファイルは単体でのインクルードを意図していません。必ず atomics.h を通じて利用してください。
  • 移植性の高いコードを維持するため、64ビットアトミックが提供されない古い環境での動作担保として機能します。
