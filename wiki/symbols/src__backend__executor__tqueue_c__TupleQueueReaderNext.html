TupleQueueReaderNext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/tqueue.c:175

目的

共有メモリキュー（shm_mq）からシリアライズされたタプルを読み出し、MinimalTuple 形式として取得する。

シグネチャまたは形状

MinimalTuple TupleQueueReaderNext(TupleQueueReader *reader, bool nowait, bool *done)

入力と出力

  • **入力**:
  • reader: 読み出し対象の TupleQueueReader 構造体へのポインタ。
  • nowait: true の場合、メッセージが即座に利用可能でない場合にブロックせず制御を戻す。
  • done: 読み出し完了状態を格納するフラグへのポインタ（NULL許容）。
  • **出力**:
  • 成功時: 取得した MinimalTuple へのポインタ。
  • 失敗またはキュー終了時: NULL。

前提条件と不変条件

  • 返される MinimalTuple は共有メモリ上の領域を直接指しており、次の読み出し呼び出しまでの間のみ有効である。
  • tuple->t_len は読み出したバイト数 nbytes と一致しなければならない。

アルゴリズムまたは振る舞い

1. done が指定されている場合、初期値として false を設定する。 2. shm_mq_receive を呼び出し、キューからメッセージを取得する。 3. キューが切断されている場合 (SHM_MQ_DETACHED)、done を true に設定し NULL を返す。 4. ノンブロッキングモード (nowait == true) でメッセージが未到着の場合 (SHM_MQ_WOULD_BLOCK)、NULL を返す。 5. 成功時、取得したデータを MinimalTuple にキャストして返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeGather.c
  • src/backend/executor/nodeGatherMerge.c
  • **呼び出し先**:
  • shm_mq_receive (共有メモリキューからのデータ取得)

関連データ構造

  • TupleQueueReader: 受信側の状態を保持する構造体。
  • MinimalTuple: 共有メモリ上で転送されるタプルの形式。

失敗モードと境界条件

  • **キューの切断**: SHM_MQ_DETACHED が返された場合、読み出しは終了とみなされる。
  • **ブロッキング**: nowait が false の場合、メッセージが到着するまで待機する。true の場合は SHM_MQ_WOULD_BLOCK を経由して即座に NULL を返す。

保守担当者向けメモ

  • 本関数は shm_mq のラッパーであり、タプルのデシリアライズは MinimalTuple 形式に依存している。
  • 読み取ったタプルは共有メモリ上のポインタを直接指すため、呼び出し側でのメモリ管理には注意が必要である。
