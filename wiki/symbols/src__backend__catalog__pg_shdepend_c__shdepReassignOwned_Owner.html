shdepReassignOwned_Owner

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_shdepend.c:1646

目的

REASSIGN OWNED コマンドの実行時、指定されたロールが所有するオブジェクトの所有権を新しいロールへ変更する。

シグネチャまたは形状

static void shdepReassignOwned_Owner(Form_pg_shdepend sdepForm, Oid newrole)

入力と出力

  • sdepForm: 変更対象の依存関係レコード（pg_shdepend の行データ）。
  • newrole: 所有権を移譲する先のロール OID。
  • 戻り値: なし。

前提条件と不変条件

  • sdepForm は有効な共有依存関係レコードを指している必要がある。
  • DefaultAclRelationId および UserMappingRelationId に該当するオブジェクトは、本関数では処理されず無視される。

アルゴリズムまたは振る舞い

1. sdepForm->classid を参照し、オブジェクトの型に応じた所有権変更関数を呼び出す。 2. RelationRelationId（テーブル等）の場合は、ATExecChangeOwner を呼び出し、インデックスやシーケンス等の依存オブジェクトを含めて再帰的に処理する。 3. TypeRelationId, NamespaceRelationId, ForeignServerRelationId 等の特定の型については、専用の Alter...Owner_oid 関数を呼び出す。 4. その他の一般的なオブジェクト型については、AlterObjectOwner_internal を使用して汎用的に処理する。 5. 未知の classid が渡された場合は elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/pg_shdepend.c 内の shdepReassignOwned（推論される文脈）。
  • 呼び出し先: AlterTypeOwner_oid, AlterSchemaOwner_oid, ATExecChangeOwner, AlterForeignServerOwner_oid, AlterForeignDataWrapperOwner_oid, AlterEventTriggerOwner_oid, AlterPublicationOwner_oid, AlterSubscriptionOwner_oid, AlterObjectOwner_internal。

関連データ構造

  • Form_pg_shdepend: pg_shdepend カタログの行構造体。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • DefaultAclRelationId および UserMappingRelationId は意図的に無視される（DROP OWNED で処理されるべき対象のため）。
  • サポートされていない classid が渡された場合、エラーで停止する。

保守担当者向けメモ

  • 本関数は REASSIGN OWNED のためのディスパッチャとして機能する。新しいオブジェクト型が追加された場合、適切な Alter...Owner 関数を switch 文に追加する必要がある。
  • RelationRelationId の処理において recursing = true を指定しているため、テーブルに関連する依存オブジェクトの所有権変更が自動的に行われる点に注意すること。
