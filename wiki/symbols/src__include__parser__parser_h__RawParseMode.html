RawParseMode

定義

  • 種別: enum
  • 定義箇所: src/include/parser/parser.h:37

目的

raw_parser() 関数が受け入れる SQL 文字列の形式や文脈を決定するために使用されます。

シグネチャまたは形状

typedef enum
{
	RAW_PARSE_DEFAULT = 0,
	RAW_PARSE_TYPE_NAME,
	RAW_PARSE_PLPGSQL_EXPR,
	RAW_PARSE_PLPGSQL_ASSIGN1,
	RAW_PARSE_PLPGSQL_ASSIGN2,
	RAW_PARSE_PLPGSQL_ASSIGN3,
} RawParseMode;

入力と出力

  • 本シンボルは列挙型であり、raw_parser 関数の引数として入力されます。
  • 出力はありません。

前提条件と不変条件

  • raw_parser に渡される文字列は、選択された RawParseMode に適合する形式である必要があります。

アルゴリズムまたは振る舞い

  • RAW_PARSE_DEFAULT: 標準的な SQL 文の解析に使用されます。
  • RAW_PARSE_TYPE_NAME: 型名の解析に使用されます。
  • RAW_PARSE_PLPGSQL_*: PL/pgSQL 特有の式や代入文の解析に使用されます。
  • src/backend/parser/parser.c 内の配列のインデックスとして利用される場合があります。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • src/backend/executor/spi.c
  • src/backend/parser/parser.c
  • src/include/executor/spi.h
  • src/include/executor/spi_priv.h
  • src/pl/plpgsql/src/plpgsql.h
  • **呼び出し先**:
  • 本シンボルは列挙型であり、関数ではないため呼び出し先はありません。

関連データ構造

  • raw_parser(const char *str, RawParseMode mode): この列挙型を制御パラメータとして受け取るメイン関数。

失敗モードと境界条件

  • 不適切な RawParseMode を指定した場合、raw_parser は構文解析に失敗するか、意図しない構文木を生成する可能性があります。

保守担当者向けメモ

  • 新しい解析モードを追加する場合は、src/backend/parser/parser.c 内の関連する配列やロジックも併せて更新する必要があります。
