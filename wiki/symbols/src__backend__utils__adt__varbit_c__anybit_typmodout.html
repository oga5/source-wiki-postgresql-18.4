anybit_typmodout

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varbit.c:126

目的

BIT および BIT VARYING 型の型修飾子（typmod）を、SQLの CREATE TABLE 等で表示可能な文字列形式に変換します。

シグネチャまたは形状

char *anybit_typmodout(int32 typmod)

入力と出力

  • **入力**: int32 typmod（ビット列の長さ。負の値は修飾子なしを意味する）。
  • **出力**: char *（変換後の文字列。呼び出し元で解放が必要）。

前提条件と不変条件

  • 内部で palloc(64) を使用してメモリを確保するため、PostgreSQLのメモリコンテキスト内で呼び出される必要があります。
  • typmod が負の場合、空文字列を返します。

アルゴリズムまたは振る舞い

1. 64バイトのメモリ領域を確保します。 2. typmod が 0 以上の場合、snprintf を用いて "(%d)" 形式の文字列を生成します。 3. typmod が負の場合、確保した領域の先頭にヌル終端文字を配置し、空文字列とします。 4. 生成した文字列ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/varbit.c 内の PG_RETURN_CSTRING を伴う箇所（行441, 786）。
  • **呼び出し先**:
  • palloc (PostgreSQLメモリ管理)
  • snprintf (標準Cライブラリ)

関連データ構造

  • int32: PostgreSQLの標準的な整数型。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • typmod が非常に大きな値であっても、snprintf のバッファサイズ（64バイト）を超えることはありませんが、表示上の制約となります。

保守担当者向けメモ

  • 本関数は BIT 型と BIT VARYING 型で共通の型修飾子出力ロジックとして利用されています。
  • 出力形式を変更する場合は、この関数を修正することで両方の型に反映されます。
