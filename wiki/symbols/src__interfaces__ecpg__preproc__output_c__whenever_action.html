whenever_action

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/output.c:63

目的

ECPGプリプロセッサが生成するCコードに対し、WHENEVER 句で定義されたエラー処理（sqlca の状態チェックとアクション）を挿入します。

シグネチャまたは形状

void whenever_action(int mode)

入力と出力

  • **入力**:
  • mode (int): 処理の制御フラグ。ビット1（mode & 1）は NOT FOUND 条件の評価を制御し、ビット2（mode & 2）はスコープ終了の閉じ括弧 } の出力を制御します。
  • **出力**: なし（base_yyout ストリームへの書き込みのみ）。

前提条件と不変条件

  • グローバル変数 when_nf、when_warn、when_error が適切に初期化されている必要があります。
  • base_yyout が書き込み可能なファイルストリームとしてオープンされている必要があります。

アルゴリズムまたは振る舞い

1. mode & 1 が真かつ when_nf.code が W_NOTHING でない場合、sqlca.sqlcode == ECPG_NOT_FOUND のチェックコードを出力します。 2. when_warn.code が W_NOTHING でない場合、sqlca.sqlwarn[0] == 'W' のチェックコードを出力します。 3. when_error.code が W_NOTHING でない場合、sqlca.sqlcode < 0 のチェックコードを出力します。 4. 各チェックコードの出力前には output_line_number() を呼び出し、ソースコードの行番号を同期させます。 5. mode & 2 が真の場合、ブロックを閉じるための } を出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/descriptor.c (複数箇所)
  • src/interfaces/ecpg/preproc/output.c (複数箇所)
  • **呼び出し先**:
  • output_line_number()
  • print_action()
  • fprintf (標準ライブラリ)
  • fputc (標準ライブラリ)

関連データ構造

  • when_nf, when_warn, when_error: WHENEVER 文の状態を保持するグローバル構造体。
  • base_yyout: 出力先のファイルストリーム。

失敗モードと境界条件

  • when_ 構造体の code が W_NOTHING の場合、対応する if ブロックは生成されません。
  • mode 引数のビットフラグが正しく渡されない場合、生成されるCコードの構文（特にブロックの閉じ括弧）が不整合を起こす可能性があります。

保守担当者向けメモ

  • mode & 2 による } の出力は、WHENEVER 処理のスコープ終了を制御するためのものです。
  • #line ディレクティブの挿入ロジックはデバッグの要であるため、変更時は生成されるCコードの行番号整合性を確認してください。
