InitWalSenderSlot

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walsender.c:2947

目的

現在のプロセスをWAL送信プロセス（walsender）として登録するため、共有メモリ上の空き WalSnd スロットを検索・確保し、初期化を行う。

シグネチャまたは形状

void InitWalSenderSlot(void)

入力と出力

  • **入力**: なし（グローバル変数 WalSndCtl および MyDatabaseId を参照）
  • **出力**: なし（副作用として共有メモリ上の WalSnd スロットを更新し、グローバル変数 MyWalSnd を設定する）

前提条件と不変条件

  • WalSndCtl が初期化済みであること。
  • MyWalSnd が NULL であること（未初期化状態）。
  • max_wal_senders 分の共有メモリ領域が確保されていること。

アルゴリズムまたは振る舞い

1. WalSndCtl->walsnds 配列を走査し、pid が 0 の空きスロットを探す。 2. 空きスロットを発見した場合、スピンロックを取得して以下の初期化を行う：

  • pid に MyProcPid を設定。
  • state を WALSNDSTATE_STARTUP に設定。
  • 各種LSNポインタ（sentPtr, write, flush, apply）を InvalidXLogRecPtr にリセット。
  • ラグ情報および同期レプリケーション関連のカウンタを初期化。
  • MyDatabaseId が InvalidOid であれば REPLICATION_KIND_PHYSICAL、そうでなければ REPLICATION_KIND_LOGICAL を kind に設定。

    • 3. スピンロックを解放し、MyWalSnd に確保したスロットのアドレスを格納する。 4. on_shmem_exit を登録し、プロセス終了時に WalSndKill が実行されるようにする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walsender.c 内の初期化プロセス（InitWalSender 等から参照される）。
  • **呼び出し先**: SpinLockAcquire, SpinLockRelease, on_shmem_exit, WalSndKill。

関連データ構造

  • WalSndCtl: 全WAL送信プロセスの状態を保持する共有メモリ上の制御構造体。
  • WalSnd: 個別のWAL送信プロセス用スロット構造体。
  • MyWalSnd: 現在のプロセスが使用するスロットへのポインタ。

失敗モードと境界条件

  • max_wal_senders を超える接続要求がある場合、空きスロットが見つからずループが終了する。この場合、Assert(MyWalSnd != NULL) によりアボートする。

保守担当者向けメモ

  • 本関数は fork() 後に呼び出されることを前提としている。
  • kind の判定は、物理レプリケーションと論理レプリケーションの区別を決定する重要な処理である。
  • プロセス終了時のクリーンアップ処理（WalSndKill）の登録を忘れないこと。
