RelfilenumberMapKey

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/relfilenumbermap.c:35

目的

(tablespace, relfilenumber) のペアをキーとして、対応する relid をハッシュテーブルから検索するための識別子として機能します。

シグネチャまたは形状

typedef struct
{
	Oid			reltablespace;
	RelFileNumber relfilenumber;
} RelfilenumberMapKey;

入力と出力

  • **入力**: reltablespace (Oid) および relfilenumber (RelFileNumber) の組み合わせ。
  • **出力**: ハッシュテーブル検索時のキーとして使用され、対応する RelfilenumberMapEntry を特定するために利用されます。

前提条件と不変条件

  • ハッシュテーブル（RelfilenumberMapHash）のキーとして使用される際、構造体全体が比較対象となります。
  • RelfilenumberMapEntry 構造体の先頭メンバとして配置される必要があります（src/backend/utils/cache/relfilenumbermap.c:43 参照）。

アルゴリズムまたは振る舞い

  • ハッシュテーブルのキーサイズ定義（sizeof(RelfilenumberMapKey)）に使用されます。
  • キャッシュ検索時に、指定されたテーブルスペースとファイル番号からハッシュ値を算出し、エントリの照合に用いられます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/relfilenumbermap.c 内のキャッシュ検索および管理ロジック。
  • **呼び出し先**: 構造体であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • RelfilenumberMapEntry: このキーを保持するキャッシュエントリ構造体。
  • RelfilenumberMapHash: このキーをインデックスとして使用するハッシュテーブル。

失敗モードと境界条件

  • 構造体内の各フィールドが適切に初期化されていない場合、ハッシュ検索が正しく機能しません。
  • 一時テーブルはバックエンド固有であり、このキーを用いた検索対象からは除外されます。

保守担当者向けメモ

  • 本構造体はハッシュテーブルのキーとして使用されるため、メンバの追加や順序変更を行う際は、ハッシュ計算ロジックおよび RelfilenumberMapEntry との整合性に注意してください。
