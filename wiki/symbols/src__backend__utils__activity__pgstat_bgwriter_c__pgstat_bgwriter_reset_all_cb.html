pgstat_bgwriter_reset_all_cb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_bgwriter.c:87

目的

バックグラウンドライター（bgwriter）の統計情報をリセットし、リセット時刻を更新します。

シグネチャまたは形状

void pgstat_bgwriter_reset_all_cb(TimestampTz ts)

入力と出力

  • **入力**: ts (TimestampTz) - 統計リセット時刻として設定するタイムスタンプ。
  • **出力**: なし。共有メモリ上の統計データが更新されます。

前提条件と不変条件

  • 共有メモリ上の pgStatLocal.shmem->bgwriter が初期化されている必要があります。
  • 統計更新プロトコルに従い、stats_shmem->lock を排他制御下で取得します。

アルゴリズムまたは振る舞い

1. 共有メモリ上の PgStatShared_BgWriter 構造体へのポインタを取得します。 2. LWLockAcquire を使用して、対象の統計領域を排他ロックします。 3. pgstat_copy_changecounted_stats を呼び出し、現在の統計値を reset_offset にコピーしてリセットの基準を更新します。 4. stat_reset_timestamp フィールドに引数 ts を代入します。 5. LWLockRelease を呼び出し、ロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c:412 にて、統計リセット用コールバックとして登録されています。
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease
  • pgstat_copy_changecounted_stats

関連データ構造

  • PgStatShared_BgWriter: バックグラウンドライターの共有統計データ構造。
  • pgStatLocal: 統計システムのローカル状態を保持するグローバル変数。

失敗モードと境界条件

  • ロック取得時に他のプロセスとの競合が発生する可能性がありますが、LWLock により排他制御されます。
  • 共有メモリが未初期化の場合、ポインタ参照によりクラッシュする可能性があります。

保守担当者向けメモ

  • 本関数は統計リセットプロトコルの一部です。PgStatShared_BgWriter 構造体の定義および changecount を用いたロックフリーに近い更新ロジックの変更時には、本関数の整合性を確認してください。
