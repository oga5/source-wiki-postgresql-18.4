XLogRecordBlockCompressHeader

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlogrecord.h:173

目的

WAL（Write-Ahead Logging）レコードにおいて、圧縮されたフルページイメージ（FPI）の「ホール（ゼロ埋め領域）」の長さを保持するためのメタデータ構造です。

シグネチャまたは形状

typedef struct XLogRecordBlockCompressHeader
{
    uint16      hole_length;    /* number of bytes in "hole" */
} XLogRecordBlockCompressHeader;

入力と出力

  • **入力**: 圧縮されたページイメージのホール長（uint16）。
  • **出力**: WALレコード内のデータ構造の一部として、圧縮データの復元に必要なメタデータを提供します。

前提条件と不変条件

  • 本構造体は、WALレコード内のブロックヘッダー（XLogRecordBlockHeader）およびイメージヘッダー（XLogRecordBlockImageHeader）に続く位置に配置されます。
  • hole_length は、圧縮されたページイメージから除外されたゼロ埋め領域のバイト数を示します。

アルゴリズムまたは振る舞い

  • WAL書き込み時、ページイメージからホールを除去して圧縮する際、この構造体を使用してホールの長さを記録します。
  • WAL読み取り時、XLogReader はこのヘッダーを読み取り、圧縮されたデータから元のページを再構築する際にホールを復元します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xloginsert.c において、WALレコード構築時に初期化されます。
  • **呼び出し先**: 提供されたソース情報からは、具体的な呼び出し先関数は特定できません。

関連データ構造

  • XLogRecordBlockHeader: レコードに関連付けられたブロックの基本メタデータ。
  • XLogRecordBlockImageHeader: フルページイメージが含まれる場合のヘッダー。

失敗モードと境界条件

  • hole_length は uint16 型であるため、ホール長が 65535 バイトを超える場合は表現できません。
  • 本構造体は整列（アライメント）されていないため、メモリ上で直接操作する際は適切なバッファへのコピーが必要です。

保守担当者向けメモ

  • 本構造体は XLogRecordBlockImageHeader の直後に続くことが想定されています。
  • 構造体のサイズは sizeof(XLogRecordBlockCompressHeader) で取得可能です。
