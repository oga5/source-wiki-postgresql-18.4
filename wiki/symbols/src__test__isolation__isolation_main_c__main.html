main

定義

  • 種別: function
  • 定義箇所: src/test/isolation/isolation_main.c:136

目的

PostgreSQL 分離テスト（Isolation Tests）の実行におけるエントリーポイントとして、pg_regress フレームワークを初期化し、テストスイートのライフサイクルを開始します。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • 入力:
  • argc: コマンドライン引数の数。
  • argv: コマンドライン引数の配列。
  • 出力:
  • int: regression_main の戻り値（テスト実行結果に基づく終了ステータス）。

前提条件と不変条件

  • pg_regress フレームワークが利用可能であること。
  • isolation_init および isolation_start_test がテスト実行に必要な初期化および実行ロジックを提供していること。

アルゴリズムまたは振る舞い

1. regression_main 関数を呼び出します。 2. その際、以下のコールバック関数を渡します。

  • isolation_init: テスト実行前の初期化処理。
  • isolation_start_test: 個別のテストケースを実行するためのプロセス起動処理。

    • 3. postfunc 引数には NULL を渡し、事後処理が不要であることを指定します。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラム実行時）。
  • 呼び出し先:
  • regression_main（pg_regress フレームワークのメイン関数）。
  • isolation_init（初期化用コールバック）。
  • isolation_start_test（テスト実行用コールバック）。
  • 注記: ツールによる find_callers の結果には contrib/amcheck 関連の参照が含まれていますが、これらは本シンボルとは無関係な同名の別シンボルへの言及であり、本関数の呼び出し元ではありません。

関連データ構造

  • pg_regress フレームワークが使用する各種構造体（内部的に利用）。

失敗モードと境界条件

  • regression_main の実行結果に依存します。引数の不正や環境設定の不備により、pg_regress 側でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は pg_regress のラッパーとして機能します。テスト実行ロジックの変更は、主に isolation_init または isolation_start_test 側で行ってください。
  • isolationtester バイナリの検索は isolation_start_test 内で遅延実行される設計となっているため、本関数内ではバイナリパスの解決は行われません。
