FlushPosition

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/worker.c:191

目的

論理レプリケーションにおいて、ローカルのトランザクション終了位置（LSN）と、それに対応するリモートの LSN を紐付けて管理するために使用されます。

シグネチャまたは形状

typedef struct FlushPosition
{
    dlist_node  node;        /* リスト管理用ノード */
    XLogRecPtr  local_end;   /* ローカル側の適用終了位置 */
    XLogRecPtr  remote_end;  /* リモート側の対応するLSN */
} FlushPosition;

入力と出力

  • 入力: palloc を介してメモリ確保され、各フィールドに XLogRecPtr 値が設定されます。
  • 出力: dlist（双方向リスト）の要素として保持され、適用済み位置の追跡に使用されます。

前提条件と不変条件

  • dlist_node を介してリスト構造に組み込まれることが前提です。
  • local_end および remote_end は、論理レプリケーションの適用進捗を示す有効な XLogRecPtr である必要があります。

アルゴリズムまたは振る舞い

  • dlist の要素としてリストに追加・走査されます。
  • worker.c 内のロジックにおいて、適用済み位置のリストから特定の要素を取得したり、末尾の要素を参照したりするために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/logical/worker.c 内の関数群（palloc によるインスタンス生成および dlist 操作関数）。
  • 呼び出し先: 直接的な関数呼び出し関係はソースから特定できませんが、dlist 操作用のマクロ（dlist_container, dlist_tail_element 等）を通じてリスト操作が行われます。

関連データ構造

  • dlist_node: リスト管理に使用される構造体。
  • XLogRecPtr: PostgreSQL のログシーケンス番号（LSN）を保持する型。

失敗モードと境界条件

  • メモリ不足時: palloc が失敗した場合、プロセスが終了する可能性があります。
  • リスト操作: 空のリストに対して dlist_tail_element 等を呼び出した場合の挙動は、dlist の実装に依存します。

保守担当者向けメモ

  • 本構造体は lsn_mapping リストの一部として管理されています。
  • 構造体の追加・削除を行う際は、必ず dlist の整合性が保たれるように注意してください。
