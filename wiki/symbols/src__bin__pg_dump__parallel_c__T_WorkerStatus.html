T_WorkerStatus

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_dump/parallel.c:77

目的

pg_dump および pg_restore における並列処理のワーカープロセスの現在の実行状態を管理・追跡するために使用されます。

シグネチャまたは形状

typedef enum
{
	WRKR_NOT_STARTED = 0,
	WRKR_IDLE,
	WRKR_WORKING,
	WRKR_TERMINATED,
} T_WorkerStatus;

入力と出力

  • 本シンボル自体は状態値の列挙型であり、入力や出力は持ちません。
  • ParallelSlot 構造体内の workerStatus メンバとして保持され、ワーカーのライフサイクル管理に使用されます。

前提条件と不変条件

  • ワーカーは WRKR_NOT_STARTED から開始し、ライフサイクルの進行に応じて各状態へ遷移します。
  • 状態遷移の具体的な順序や制約については、ソースコード内の状態管理ロジックに依存します。

アルゴリズムまたは振る舞い

  • WRKR_NOT_STARTED: ワーカープロセスがまだ生成されていない初期状態。
  • WRKR_IDLE: ワーカーが起動済みで、リーダーからのジョブ割り当てを待機している状態。
  • WRKR_WORKING: ワーカーが割り当てられたジョブ（TocEntry の処理）を実行中の状態。
  • WRKR_TERMINATED: ワーカープロセスが終了した状態。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数関係については、提供されたソース excerpt には記載されていません。
  • src/bin/pg_dump/parallel.c:97 にて、ParallelSlot 構造体のメンバとして参照されています。

関連データ構造

  • ParallelSlot: 個々のワーカーの通信パイプやプロセスIDを保持する構造体であり、本列挙型をメンバとして含みます。

失敗モードと境界条件

  • 列挙型であるため、定義外の値が代入された場合の挙動は未定義です。
  • 状態遷移が適切に行われない場合、リーダープロセスがワーカーの完了待ち合わせ（WaitForWorkers）でデッドロックやタイムアウトを引き起こす可能性があります。

保守担当者向けメモ

  • 本シンボルは parallel.c 内のワーカー管理ロジックの根幹を成します。状態遷移のロジックを変更する場合は、ParallelSlot を利用する DispatchJobForTocEntry や ListenToWorkers などの関数への影響を考慮してください。
