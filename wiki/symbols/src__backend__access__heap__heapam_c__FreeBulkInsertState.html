FreeBulkInsertState

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam.c:2029

目的

バルク挿入操作の終了後に、BulkInsertState 構造体が保持するリソースを解放し、クリーンアップを行う。

シグネチャまたは形状

void FreeBulkInsertState(BulkInsertState bistate)

入力と出力

  • **入力**: bistate (BulkInsertState) - 解放対象のバルク挿入状態構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • bistate は GetBulkInsertState 等によって適切に初期化されている必要がある。
  • bistate が NULL の場合の挙動は定義されていない（ソースコード上はポインタ参照が発生するため、非 NULL であることが前提）。

アルゴリズムまたは振る舞い

1. bistate->current_buf が InvalidBuffer でない場合、ReleaseBuffer を呼び出してバッファのピンを解放する。 2. bistate->strategy に関連付けられたアクセス戦略を FreeAccessStrategy で解放する。 3. pfree(bistate) を呼び出し、構造体自体が使用していたメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfrom.c
  • src/backend/commands/createas.c
  • src/backend/commands/matview.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • ReleaseBuffer
  • FreeAccessStrategy
  • pfree

関連データ構造

  • BulkInsertState: バルク挿入時のバッファ管理やアクセス戦略を保持する構造体。

失敗モードと境界条件

  • bistate が無効なポインタである場合、メモリ破壊やセグメンテーション違反が発生する可能性がある。
  • bistate->current_buf が既に解放されている場合、ReleaseBuffer 内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はバルク挿入の終了処理における標準的なクリーンアップ手順である。新しいバルク挿入処理を追加する際は、必ず本関数を呼び出してリソースリークを防ぐこと。
