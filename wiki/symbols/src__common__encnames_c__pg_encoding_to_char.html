pg_encoding_to_char

定義

  • 種別: function
  • 定義箇所: src/common/encnames.c:589

目的

内部的な文字エンコーディング ID (pg_enc) を、対応する公式のエンコーディング名（文字列）に変換します。

シグネチャまたは形状

const char *pg_encoding_to_char(int encoding);

入力と出力

  • **入力**: int encoding（エンコーディング ID）
  • **出力**: const char *（エンコーディング名を表すヌル終端文字列。無効な ID の場合は空文字列 "" を返す）

前提条件と不変条件

  • pg_enc2name_tbl が初期化済みであること。
  • PG_VALID_ENCODING(encoding) マクロにより、渡された ID が有効な範囲内であることが検証されます。
  • 内部で Assert(encoding == p->encoding) が実行され、テーブルのインデックスと格納された ID の整合性が保証されます。

アルゴリズムまたは振る舞い

1. PG_VALID_ENCODING マクロを使用して、入力された encoding が有効な範囲内か判定します。 2. 有効な場合、pg_enc2name_tbl 配列の指定されたインデックス（encoding）を参照し、対応する pg_enc2name 構造体を取得します。 3. 構造体内の name フィールド（エンコーディング名）を返します。 4. 無効な ID の場合は、空文字列 "" を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_collation.c
  • src/backend/catalog/pg_conversion.c
  • src/backend/commands/copyfrom.c
  • src/backend/commands/dbcommands.c
  • **呼び出し先**:
  • PG_VALID_ENCODING マクロ（mb/pg_wchar.h で定義）

関連データ構造

  • pg_enc2name_tbl: 内部 ID からエンコーディング名および関連情報を保持する静的テーブル。
  • pg_enc2name: pg_enc2name_tbl の各要素の型。

失敗モードと境界条件

  • encoding が有効範囲外（PG_VALID_ENCODING が偽）の場合、関数はエラーを発生させず、空文字列 "" を返します。

保守担当者向けメモ

  • 本関数は pg_enc2name_tbl のインデックスとして encoding を直接使用するため、テーブルのサイズと PG_VALID_ENCODING の定義が常に同期している必要があります。
  • 外部ライブラリ（ICU 等）用の名前が必要な場合は、別途 get_encoding_name_for_icu 等の専用関数を使用してください。
