EventTriggerCacheStateType

定義

  • 種別: enum
  • 定義箇所: src/backend/utils/cache/evtcache.c:33

目的

イベントトリガーキャッシュの現在の有効性状態を追跡し、キャッシュの再構築が必要かどうかを判断するために使用されます。

シグネチャまたは形状

typedef enum
{
	ETCS_NEEDS_REBUILD,
	ETCS_REBUILD_STARTED,
	ETCS_VALID,
} EventTriggerCacheStateType;

入力と出力

  • 本シンボル自体は状態値の列挙型であり、入力や出力は持ちません。
  • この型を持つ変数 EventTriggerCacheState がキャッシュの制御状態を保持します。

前提条件と不変条件

  • キャッシュの初期状態は ETCS_NEEDS_REBUILD です。
  • ETCS_VALID 状態にある場合のみ、キャッシュは検索可能な状態であるとみなされます。

アルゴリズムまたは振る舞い

  • ETCS_NEEDS_REBUILD: キャッシュが空または無効化されており、再構築が必要な状態。
  • ETCS_REBUILD_STARTED: 再構築処理が進行中であることを示し、再帰的な再構築を防ぐためのガードとして機能します。
  • ETCS_VALID: キャッシュが最新であり、検索に使用可能な状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/evtcache.c 内のキャッシュ管理ロジック（EventCacheLookup や InvalidateEventCacheCallback など）がこの状態を参照・更新します。
  • 呼び出し先: 本シンボルは列挙型であり、直接の呼び出し先はありません。

関連データ構造

  • EventTriggerCache: キャッシュの実体であるハッシュテーブル。
  • EventTriggerCacheState: この型を持つ静的変数。

失敗モードと境界条件

  • 再構築中に無効化通知が発生した場合、現在の再構築処理を完了させた後に再度 ETCS_NEEDS_REBUILD へ遷移させることで、無限ループや不整合を回避する設計となっています。

保守担当者向けメモ

  • キャッシュのライフサイクル管理において、状態遷移の整合性は src/backend/utils/cache/evtcache.c 内の関数群によって保証されています。状態を追加・変更する際は、再構築ロジックへの影響を考慮してください。
