buildoidvector

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oid.c:86

目的

Oid 型の配列から、PostgreSQL のシステムカタログ等で使用される oidvector 構造体を構築します。

シグネチャまたは形状

oidvector *buildoidvector(const Oid *oids, int n)

入力と出力

  • **入力**:
  • oids: Oid 型の配列へのポインタ。
  • n: 配列内の要素数。
  • **出力**:
  • 構築された oidvector 構造体へのポインタ。

前提条件と不変条件

  • n が 0 より大きく、かつ oids が NULL でない場合、指定された配列の内容がコピーされます。
  • oidvector は 1 次元配列として初期化されます。
  • 歴史的な理由により、インデックスの下限（lbound1）は 0 に設定されます。
  • NULL 値は含まれない前提で dataoffset は 0 に固定されます。

アルゴリズムまたは振る舞い

1. OidVectorSize(n) を用いて必要なメモリを確保し、ゼロクリアします。 2. n > 0 かつ oids が有効な場合、memcpy を使用して oids の内容を result->values にコピーします。 3. SET_VARSIZE を使用して構造体のサイズを設定します。 4. 配列ヘッダーのメタデータ（ndim=1, dataoffset=0, elemtype=OIDOID, dim1=n, lbound1=0）を初期化します。 5. 構築した oidvector を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c
  • src/backend/catalog/index.c
  • src/backend/commands/aggregatecmds.c
  • src/backend/commands/functioncmds.c
  • src/backend/commands/typecmds.c
  • **呼び出し先**:
  • palloc0 (メモリ確保)
  • memcpy (メモリコピー)
  • SET_VARSIZE (マクロ)

関連データ構造

  • oidvector: Oid の固定長配列を表現する構造体。
  • OIDOID: oidvector の要素型として指定される型定数。

失敗モードと境界条件

  • n が 0 の場合、要素のコピーは行われず、空の oidvector が作成されます。
  • メモリ不足時には palloc0 がエラーを発生させます。

保守担当者向けメモ

  • oidvector は一般的な配列型と異なり、インデックスの下限が 0 である点に注意してください。
  • 本関数は oidvector を作成する標準的な手段であり、カタログ操作やコマンド処理において頻繁に使用されます。
