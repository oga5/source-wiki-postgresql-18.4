ExplainCloseGroup

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_format.c:341

目的

EXPLAIN コマンドの出力において、階層構造（グループ）の終了処理を行う。フォーマットに応じて適切な閉じタグや区切り文字を挿入し、インデントレベルを調整する。

シグネチャまたは形状

void ExplainCloseGroup(const char *objtype, const char *labelname, bool labeled, ExplainState *es)

入力と出力

  • objtype: 閉じるグループの型名（XML出力時に使用）。
  • labelname: グループのラベル名（未使用）。
  • labeled: JSON出力において、オブジェクト（}）か配列（]）かを判定するフラグ。
  • es: ExplainState 構造体。現在のフォーマット状態、インデントレベル、出力バッファ、グループスタックを保持する。

前提条件と不変条件

  • es が有効な ExplainState を指していること。
  • ExplainOpenGroup によって開始された階層構造と対になるように呼び出されること。

アルゴリズムまたは振る舞い

es->format に基づき以下の処理を行う：

1. **TEXT**: 何も行わない。 2. **XML**: インデントを減らし、ExplainXMLTag を呼び出して終了タグを出力する。 3. **JSON**: インデントを減らし、改行とインデントを挿入後、labeled が真なら } を、偽なら ] を出力する。その後、es->grouping_stack から先頭要素を削除する。 4. **YAML**: インデントを減らし、es->grouping_stack から先頭要素を削除する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/explain.c（Planning, Query, Settings, Triggers, Options, Generation, Timing, JIT, Serialization 等の各セクション終了時）および contrib/pg_overexplain/pg_overexplain.c。
  • **呼び出し先**: ExplainXMLTag（XML形式時）、appendStringInfoChar / appendStringInfoSpaces（JSON形式時）、list_delete_first（JSON/YAML形式時）。

関連データ構造

  • ExplainState: 出力状態を管理する主要構造体。
  • grouping_stack: JSON/YAML出力において、現在のネスト状態を追跡するためのリスト。

失敗モードと境界条件

  • es->format が未定義または不正な値の場合の動作は定義されていない。
  • grouping_stack が空の状態で JSON/YAML 形式の終了を試みた場合、リスト操作関数がエラーを発生させる可能性がある。

保守担当者向けメモ

  • JSON/YAML 形式における grouping_stack の整合性は呼び出し側の責任である。
  • 新しい出力フォーマットを追加する場合は、本関数の switch 文に新しい case を追加する必要がある。
