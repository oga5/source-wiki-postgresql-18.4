PgStatShared_BgWriter

定義

  • 種別: struct
  • 定義箇所: src/include/utils/pgstat_internal.h:370

目的

バックグラウンドライター（BgWriter）の統計情報を共有メモリ上で保持・管理するための構造体です。

シグネチャまたは形状

typedef struct PgStatShared_BgWriter
{
    LWLock        lock;
    uint32        changecount;
    PgStat_BgWriterStats stats;
    PgStat_BgWriterStats reset_offset;
} PgStatShared_BgWriter;

入力と出力

  • **入力**: バックエンドプロセスによる統計情報の更新（PgStat_BgWriterStats 構造体への書き込み）。
  • **出力**: 統計情報の読み取りおよびリセット時のオフセット値の提供。

前提条件と不変条件

  • lock は reset_offset および stats 内の stat_reset_timestamp を保護します。
  • 統計情報の読み取りには changecount プロトコルを使用し、書き込みとの競合を検知して一貫性を保証する必要があります。

アルゴリズムまたは振る舞い

  • **統計更新**: バックエンドは共有メモリ上の本構造体にアクセスし、統計値を更新します。
  • **リセット処理**: reset_offset を使用して、統計リセット時点からの差分を計算するプロトコルが実装されています。
  • **同期**: 共有メモリ上のデータであるため、ポインタではなくオフセット等を用いてアクセスされます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/pgstat.c: 統計システムの初期化およびオフセット計算に使用。
  • src/backend/utils/activity/pgstat_bgwriter.c: BgWriter 統計の読み書きおよびリセット処理に使用。
  • **呼び出し先**:
  • 構造体内部で LWLock を使用し、共有メモリの排他制御を行います。

関連データ構造

  • PgStat_BgWriterStats: 実際の統計データ本体。
  • LWLock: 共有メモリ保護用の軽量ロック。
  • PgStat_ShmemControl: 本構造体を含む統計システム全体のルート構造体。

失敗モードと境界条件

  • 共有メモリ上のデータであるため、プロセス異常終了時に統計情報が不整合になる可能性がありますが、changecount プロトコルにより読み取り側での不整合検知が可能です。
  • ロックの取得失敗やデッドロックの可能性については、PostgreSQL の標準的な LWLock の制約に従います。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されるため、メンバの追加や変更を行う際は、メモリレイアウトとアライメントに注意してください。
  • リセットプロトコルの詳細については、src/backend/utils/activity/pgstat_bgwriter.c 内のコメントを参照してください。
