scram_ServerKey

定義

  • 種別: function
  • 定義箇所: src/common/scram-common.c:171

目的

SCRAM認証プロセスにおいて、SaltedPassword から ServerKey を導出する。

シグネチャまたは形状

int scram_ServerKey(const uint8 *salted_password,
                    pg_cryptohash_type hash_type, int key_length,
                    uint8 *result, const char **errstr);

入力と出力

  • **入力**:
  • salted_password: PBKDF2で生成されたソルト付きパスワード。
  • hash_type: 使用するハッシュアルゴリズム（例: PG_SHA256）。
  • key_length: 導出するキーの長さ。
  • **出力**:
  • result: 導出された ServerKey を格納するバッファ。
  • errstr: 失敗時にエラーメッセージが格納されるポインタ。
  • 戻り値: 成功時は 0、失敗時は -1。

前提条件と不変条件

  • salted_password は有効なメモリ領域を指していること。
  • hash_type に対応する暗号化コンテキストが生成可能であること。

アルゴリズムまたは振る舞い

1. pg_hmac_create を用いて指定されたハッシュ型のHMACコンテキストを生成する。 2. pg_hmac_init に salted_password を渡して初期化する。 3. pg_hmac_update を呼び出し、文字列 "Server Key" を入力としてHMAC計算を行う。 4. pg_hmac_final を呼び出し、結果を result バッファに書き込む。 5. 最後に pg_hmac_free でコンテキストを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth-scram.c: サーバー側の認証フロー内で呼び出される。
  • src/common/scram-common.c: 同一ファイル内の他関数から参照される可能性がある。
  • **呼び出し先**:
  • pg_hmac_create, pg_hmac_init, pg_hmac_update, pg_hmac_final, pg_hmac_free, pg_hmac_error: src/common/hmac.c で定義された暗号化プリミティブ。

関連データ構造

  • pg_hmac_ctx: HMAC計算用コンテキスト。
  • pg_cryptohash_type: ハッシュアルゴリズムの列挙型。

失敗モードと境界条件

  • pg_hmac_create が失敗した場合（メモリ不足等）、-1 を返し errstr にエラー内容を設定する。
  • HMAC計算の各ステップ（init, update, final）でエラーが発生した場合、コンテキストを解放し -1 を返す。

保守担当者向けメモ

  • 本関数はRFC 5802に準拠している。
  • 呼び出し元で errstr を適切に処理し、エラー発生時のログ出力等を行うこと。
