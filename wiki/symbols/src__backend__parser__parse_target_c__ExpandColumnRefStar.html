ExpandColumnRefStar

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_target.c:1122

目的

SQLクエリ内の *（ワイルドカード）または table.* 形式の列参照を、対象となるテーブルの全列リスト（TargetEntry または Var のリスト）に展開する。

シグネチャまたは形状

List *ExpandColumnRefStar(ParseState *pstate, ColumnRef *cref, bool make_target_entry)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態。
  • cref: 展開対象の ColumnRef ノード。
  • make_target_entry: 結果を TargetEntry リストとして構築するかどうか。
  • **出力**: 展開された列のリスト（List *）。

前提条件と不変条件

  • cref->fields が空でないこと。
  • make_target_entry が false の場合、cref が単一の * であることは許容されない（文法上、トップレベルの SELECT 以外では発生しないため）。

アルゴリズムまたは振る舞い

1. **単一 * の処理**: cref->fields の要素数が1の場合、ExpandAllTables を呼び出し、クエリ内の全テーブルの列を展開する。 2. **修飾付き * の処理**:

  • p_pre_columnref_hook が定義されていれば、まずフックを試行し、成功すれば ExpandRowReference で展開する。
  • numnames（修飾の深さ）に応じて名前空間から ParseNamespaceItem を検索する。
  • p_post_columnref_hook が定義されていれば、フックを試行し、成功すれば ExpandRowReference で展開する。
  • 検索に失敗した場合は、エラー（errorMissingRTE 等）を報告する。
  • 最終的に ExpandSingleTable を呼び出し、対象テーブルの列を展開する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_target.c 内のターゲットリスト変換処理（transformTargetList 等から間接的に呼び出される）。
  • **呼び出し先**:
  • ExpandAllTables, ExpandSingleTable, ExpandRowReference
  • refnameNamespaceItem
  • pstate->p_pre_columnref_hook, pstate->p_post_columnref_hook

関連データ構造

  • ColumnRef: 構文解析された列参照ノード。
  • ParseNamespaceItem: 名前空間内のテーブル参照情報。
  • ParseState: 解析中の状態保持構造体。

失敗モードと境界条件

  • **CRSERR_NO_RTE**: 指定されたテーブル名が見つからない場合にエラーを発生させる。
  • **CRSERR_WRONG_DB**: データベース間参照（4要素の修飾）が現在のデータベースと一致しない場合にエラーを発生させる。
  • **CRSERR_TOO_MANY**: 4要素を超える修飾名が指定された場合に構文エラーを発生させる。
  • **曖昧な参照**: post_columnref_hook がノードを返し、かつ nsitem も見つかる場合は曖昧な参照としてエラーを発生させる。

保守担当者向けメモ

  • transformColumnRef とのコード共有は、パーミッション管理（selectedCols ビットマップ）の不整合を招くため意図的に避けている。
  • post_columnref_hook に RangeTblEntry を渡す慣習があるため、フック側での型判定に注意が必要。
