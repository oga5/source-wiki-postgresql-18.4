RmgrStartup

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/rmgr.c:57

目的

システム起動時（リカバリ開始時など）に、登録されているすべてのリソースマネージャ（Resource Manager）に対して初期化処理を実行する。

シグネチャまたは形状

void RmgrStartup(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として各リソースマネージャの初期化関数を実行）

前提条件と不変条件

  • RmgrTable が適切に初期化されていること。
  • RM_MAX_ID までの範囲で、各リソースマネージャが RmgrTable に登録されていること。

アルゴリズムまたは振る舞い

1. 0 から RM_MAX_ID までの範囲でループを実行する。 2. RmgrIdExists(rmid) を使用して、現在のIDにリソースマネージャが存在するかを確認する。存在しない場合はスキップする。 3. RmgrTable[rmid].rm_startup が NULL でない場合、その関数ポインタを呼び出す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlogrecovery.c:1756
  • 呼び出し先: RmgrTable に登録された各リソースマネージャの rm_startup 関数

関連データ構造

  • RmgrTable: 全リソースマネージャの定義（RmgrData）を保持するグローバル配列。
  • RmgrData: リソースマネージャの各処理（rm_redo, rm_desc, rm_startup 等）を保持する構造体。

失敗モードと境界条件

  • rm_startup 関数が定義されていないリソースマネージャについては、単にスキップされるためエラーにはならない。
  • RmgrId は uint8 型であり、最大256個までの制限がある。

保守担当者向けメモ

  • 本関数はリカバリの開始時に呼び出されるため、ここで実行される初期化処理は、WALの適用（Redo）に必要な状態を準備するものである必要がある。
  • 新しいリソースマネージャを追加する場合は、rmgrlist.h を編集して RmgrTable に反映させる必要がある。
