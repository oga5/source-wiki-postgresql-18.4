gin_check_posting_tree_parent_keys_consistency

定義

  • **種別**: 関数
  • **定義箇所**: contrib/amcheck/verify_gin.c:133

目的

GINインデックスのポスティングツリー構造を再帰的に走査し、親ページと子ページのキー関係およびツリーの高さの整合性を検証します。

シグネチャまたは形状

void gin_check_posting_tree_parent_keys_consistency(Relation rel, BlockNumber posting_tree_root)

入力と出力

  • **入力**:
  • rel: 検証対象のインデックスリレーション。
  • posting_tree_root: 検証を開始するポスティングツリーのルートブロック番号。
  • **出力**: なし（検証失敗時はエラーをスロー）。

前提条件と不変条件

  • 検証は AccessShareLock 下で実行されるため、並行更新によるページ分割が発生する可能性があります。
  • ポスティングツリーの全枝は同じ高さである必要があります。
  • 内部ページの pd_lower 値は、maxoff（格納されているアイテム数）と整合している必要があります。
  • 内部ページのハイキー（High Key）は、親ページが保持するダウンリンクと一致する必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: 検証用のメモリコンテキストを作成し、ルートページをスタックに積みます。 2. **走査**: スタックが空になるまで深さ優先探索（DFS）を行います。 3. **ページ検証**:

  • **リーフページ**:
  • 初回遭遇時にツリーの高さを記録し、以降のリーフページで高さが一致するか確認します。
  • 保持するTID範囲が親ページのハイキーを超えていないか確認します。
  • **内部ページ**:
  • pd_lower の値から計算されるアイテム数と、maxoff が一致するか検証します。
  • ページ内のハイキーと、スタック上の親から渡されたダウンリンクを比較します。

    • 4. **終了**: メモリコンテキストを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_gin.c 内の gin_check_parent_keys_consistency から呼び出されます。
  • **呼び出し先**: ReadBufferExtended, LockBuffer, BufferGetPage, GinDataLeafPageGetItems, GinDataPageGetRightBound, GinDataPageGetPostingItem 等のGIN内部アクセサおよびバッファ管理関数。

関連データ構造

  • GinPostingTreeScanItem: 走査中のスタック状態（現在のブロック、親のキー、深さなど）を保持する構造体。
  • PostingItem: ポスティングツリー内部ページに格納されるアイテム。

失敗モードと境界条件

  • **ERRCODE_INDEX_CORRUPTED**:
  • ツリーの高さが枝によって異なる場合。
  • リーフページのTIDが親のハイキーを超える場合。
  • 内部ページの pd_lower が不正な場合。
  • 内部ページのハイキーが親のダウンリンクと一致しない場合。
  • **境界条件**:
  • 最右ページ（rightmost page）はハイキーを明示的に持たないため、比較対象から除外されます。

保守担当者向けメモ

  • 本関数は AccessShareLock で動作するため、検証中にページ分割が発生し、一時的な不整合が検知される可能性があります。
  • pd_lower のチェックは、PostgreSQL 9.4以前のバイナリアップグレードに起因する古いインデックス構造との互換性を考慮しています。
