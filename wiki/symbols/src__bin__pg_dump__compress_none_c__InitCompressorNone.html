InitCompressorNone

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_none.c:65

目的

pg_dump における非圧縮データストリーム処理のために、CompressorState 構造体へ非圧縮用の関数ポインタを割り当て、初期化を行う。

シグネチャまたは形状

void InitCompressorNone(CompressorState *cs, const pg_compress_specification compression_spec);

入力と出力

  • **入力**:
  • cs: 初期化対象となる CompressorState 構造体へのポインタ。
  • compression_spec: 圧縮設定を保持する pg_compress_specification 構造体。
  • **出力**: なし（cs が指す構造体のメンバを直接更新する）。

前提条件と不変条件

  • cs は有効なメモリ領域を指している必要がある。
  • 本関数は、圧縮を行わない（None）モードが選択された際に呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. cs->readData に ReadDataFromArchiveNone を割り当てる。 2. cs->writeData に WriteDataToArchiveNone を割り当てる。 3. cs->end に EndCompressorNone を割り当てる。 4. cs->compression_spec に引数 compression_spec を格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_io.c:133
  • **呼び出し先**:
  • ReadDataFromArchiveNone (関数ポインタとして代入)
  • WriteDataToArchiveNone (関数ポインタとして代入)
  • EndCompressorNone (関数ポインタとして代入)

関連データ構造

  • CompressorState: 圧縮サブシステムの関数ポインタと状態を保持する構造体。
  • pg_compress_specification: 圧縮設定を定義する構造体。

失敗モードと境界条件

  • cs が NULL の場合、セグメンテーション違反が発生する可能性がある（ソースコード上での明示的なチェックは存在しない）。

保守担当者向けメモ

  • 本関数は compress_io モジュールが期待するインターフェースを実装するラッパーの一部である。
  • 圧縮アルゴリズムの追加や変更を行う際は、本関数と同様の初期化パターンを他の compress_*.c ファイルでも参照すること。
