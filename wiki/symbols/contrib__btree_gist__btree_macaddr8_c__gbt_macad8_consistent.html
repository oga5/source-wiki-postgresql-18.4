gbt_macad8_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_macaddr8.c:121

目的

GiST インデックス検索において、インデックス内の mac8KEY（範囲キー）とクエリ条件が一致するかを判定します。

シグネチャまたは形状

Datum gbt_macad8_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY *entry（インデックスエントリ）
  • PG_GETARG_POINTER(1): macaddr8 *query（検索条件）
  • PG_GETARG_UINT16(2): StrategyNumber strategy（比較演算子）
  • PG_GETARG_POINTER(4): bool *recheck（再チェックフラグ）
  • **出力**:
  • Datum: bool 型（一致判定結果）
  • *recheck: false に設定（GiST 検索後の再チェックは不要）

前提条件と不変条件

  • entry->key は mac8KEY 構造体として解釈可能であること。
  • tinfo（静的変数）が macaddr8 型の比較ロジックを保持していること。

アルゴリズムまたは振る舞い

1. entry->key を mac8KEY として取得し、下限・上限を GBT_NUMKEY_R 構造体にマッピングします。 2. *recheck を false に設定します。 3. gbt_num_consistent 関数を呼び出し、tinfo に登録された比較関数を用いて、インデックスの範囲とクエリ条件の適合性を判定します。 4. 判定結果を PG_RETURN_BOOL で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: GiST インデックスの検索プロセスから呼び出されます（直接的な呼び出し元関数の明示的な記述はソース内にありません）。
  • **呼び出し先**:
  • gbt_num_consistent（btree_utils_num.c）
  • PG_GETARG_* マクロ群

関連データ構造

  • mac8KEY: インデックスノード内で macaddr8 の範囲を保持する構造体。
  • GBT_NUMKEY_R: 範囲比較のための汎用構造体。
  • tinfo: gbtree_ninfo 型の静的変数。比較関数や型情報を集約。

失敗モードと境界条件

  • 本関数は完全一致判定を行うため、GiST 検索後の再チェックは発生しません。
  • macaddr8 型の範囲外の入力や不正な strategy が渡された場合の挙動は、依存先の gbt_num_consistent に準拠します。

保守担当者向けメモ

  • ほとんどのロジックは btree_utils_num.c の汎用関数に委譲されています。複雑な挙動の調査時はそちらを参照してください。
  • macaddr8 は 8 バイトであるため、数値変換を伴う比較ロジックが tinfo を通じて適用されます。
