overexplain_range_table

定義

  • 種別: function
  • 定義箇所: contrib/pg_overexplain/pg_overexplain.c:382

目的

EXPLAIN コマンドの出力において、クエリのレンジテーブル（RangeTblEntry）に含まれる各エントリの詳細情報を追加出力する。

シグネチャまたは形状

static void overexplain_range_table(PlannedStmt *plannedstmt, ExplainState *es)

入力と出力

  • **入力**:
  • plannedstmt: 実行計画全体を保持する PlannedStmt 構造体。
  • es: EXPLAIN の出力状態を管理する ExplainState 構造体。
  • **出力**: なし（es を介して EXPLAIN 出力ストリームに情報を書き込む）。

前提条件と不変条件

  • plannedstmt->rtable が有効であること。
  • es が初期化済みであること。
  • EXPLAIN の出力フォーマット（TEXT, JSON, XML, YAML）に応じて、適切な ExplainProperty 系関数が呼び出される。

アルゴリズムまたは振る舞い

1. Range Table グループを開始する。 2. plannedstmt->rtable を走査し、各 RangeTblEntry (RTE) を取得する。 3. rtekind に基づき、RTE の種類（relation, subquery, join 等）を文字列化する。 4. フォーマットに応じて、RTI（レンジテーブルインデックス）、種類、継承フラグ、inFromCl フラグを出力する。 5. alias および eref 情報を出力する。 6. relid が存在する場合、verbose モードに応じて修飾名または識別名を出力する。 7. relkind（relation, index, view 等）および rellockmode が存在する場合はそれらを出力する。 8. perminfoindex が存在する場合は出力する。 9. security_barrier フラグを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: overexplain_range_table_handler (ソース内の直接的な呼び出し箇所より推測)。
  • **呼び出し先**:
  • ExplainOpenGroup, ExplainProperty... 系関数（commands/explain.h）
  • rt_fetch
  • overexplain_alias
  • get_rel_name, get_rel_namespace, get_namespace_name_or_temp
  • GetLockmodeName

関連データ構造

  • PlannedStmt: クエリの実行計画とレンジテーブルを保持。
  • RangeTblEntry: レンジテーブルの各要素。
  • ExplainState: EXPLAIN 出力の状態管理。

失敗モードと境界条件

  • rte == NULL: スキップされる。
  • relid == 0: リレーション関連情報の出力はスキップされる。
  • relkind が定義されていない場合: 出力されない。
  • rellockmode == 0: ロックモード情報は出力されない。
  • perminfoindex == 0: パーミッションインデックス情報は出力されない。

保守担当者向けメモ

  • add_rte_to_flat_rtable によって rte->tablesample や rte->subquery フィールドがクリアされるため、これらの情報は本関数では表示されない。
  • security_barrier はコアの EXPLAIN では表示されないため、本関数で明示的に出力している。
