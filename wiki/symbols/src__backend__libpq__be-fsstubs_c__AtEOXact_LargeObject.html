AtEOXact_LargeObject

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-fsstubs.c:606

目的

トランザクション終了時（コミットまたはアボート）に、開かれているラージオブジェクト（LO）に関連するリソースを解放し、クリーンアップを行う。

シグネチャまたは形状

void AtEOXact_LargeObject(bool isCommit)

入力と出力

  • isCommit (bool): トランザクションがコミットされる場合は true、アボートされる場合は false を指定する。
  • 戻り値: なし。

前提条件と不変条件

  • lo_cleanup_needed フラグが false の場合、本関数は即座に復帰する。
  • 本関数実行後、cookies 配列は NULL に、cookies_size は 0 にリセットされる。
  • fscxt（LO専用メモリコンテキスト）は本関数内で削除され、NULL にリセットされる。

アルゴリズムまたは振る舞い

1. lo_cleanup_needed を確認し、不要であれば終了する。 2. isCommit が true の場合、cookies 配列を走査し、有効な各LOディスクリプタに対して closeLOfd() を呼び出してクローズする。 3. cookies 配列と cookies_size をリセットする。 4. fscxt を MemoryContextDelete() で解放し、fscxt を NULL に設定する。 5. close_lo_relation(isCommit) を呼び出し、inv_api.c 側のクリーンアップを実行する。 6. lo_cleanup_needed を false に設定する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/xact.c（トランザクション終了処理の各フェーズから呼び出される）
  • 呼び出し先:
  • closeLOfd()
  • MemoryContextDelete()
  • close_lo_relation()

関連データ構造

  • cookies: 開いているLOディスクリプタを保持する動的配列。
  • fscxt: LO操作専用のメモリコンテキスト。
  • lo_cleanup_needed: クリーンアップの要否を示すフラグ。

失敗モードと境界条件

  • isCommit が false（アボート時）の場合、LOディスクリプタの明示的なクローズ処理はスキップされる（メモリコンテキストの解放によりリソースは回収されるため）。
  • fscxt が既に NULL の場合、メモリコンテキストの削除は行われない。

保守担当者向けメモ

  • 本関数はトランザクション境界でのリソース管理を担う。コミット時にクローズ処理を行うのは、リソースリークに関する警告を回避するためである。
  • pfree による個別のメモリ解放は行わず、コンテキストの一括削除に依存している。
