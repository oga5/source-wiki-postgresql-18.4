conv_18030_to_utf8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_gb18030/utf8_and_gb18030.c:130

目的

GB18030 エンコーディングの 4 バイト文字範囲を、対応する Unicode コードポイントへ変換し、さらに UTF-8 形式のワードへ変換する。

シグネチャまたは形状

uint32 conv_18030_to_utf8(uint32 code)

入力と出力

  • **入力**: code (GB18030 の 4 バイト文字を表現する uint32 値)
  • **出力**: 変換後の UTF-8 ワード（uint32）。マッピングが存在しない場合は 0 を返す。

前提条件と不変条件

  • 入力 code は GB18030 の 4 バイト文字範囲であると想定される。
  • 内部で使用される gb_linear 関数は、GB18030 のバイト構造を線形空間に変換するために使用される。

アルゴリズムまたは振る舞い

1. マクロ conv18030(minunicode, mincode, maxcode) を用いて、入力 code が定義済みの範囲内にあるか順次判定する。 2. 範囲内に該当する場合、以下の計算式で Unicode コードポイントを算出する: gb_linear(code) - gb_linear(mincode) + minunicode 3. 算出されたコードポイントを unicode_to_utf8word 関数に渡し、UTF-8 形式に変換して返す。 4. どの範囲にも該当しない場合は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は確認されていない。
  • **呼び出し先**:
  • gb_linear
  • unicode_to_utf8word

関連データ構造

  • なし（本関数は計算ロジックのみで構成される）。

失敗モードと境界条件

  • 入力が定義された範囲外である場合、関数は 0 を返す。
  • GB18030 の仕様（GB18030-2000）に基づいた特定の範囲外の入力に対するエラーハンドリングは、呼び出し元に依存する。

保守担当者向けメモ

  • 本関数はマッピングテーブル外の特殊な 4 バイト文字範囲を処理する。
  • 新しい範囲を追加する場合は、conv18030 マクロを使用して範囲とオフセットを定義すること。
  • 変換ロジックは GB18030 の線形空間計算に強く依存しているため、変更時は gb_linear の仕様と整合性を取ること。
