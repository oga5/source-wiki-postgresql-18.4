binary_oper_exact

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_oper.c:261

目的

指定された名前と引数型に完全に一致する二項演算子をシステムカタログから検索します。型変換を伴わない厳密な一致、または UNKNOWN 型を含む場合の推論による一致を試みます。

シグネチャまたは形状

Oid binary_oper_exact(List *opname, Oid arg1, Oid arg2)

入力と出力

  • **入力**:
  • opname: 演算子名（List 形式）。
  • arg1, arg2: 演算子の左辺および右辺の型を表す Oid。
  • **出力**:
  • 適合する演算子の Oid。見つからない場合は InvalidOid を返します。

前提条件と不変条件

  • arg1 または arg2 が UNKNOWNOID の場合、もう一方の型を代入して検索を試みます。
  • 検索対象の型がドメイン型である場合、その基底型（getBaseType）を用いた再検索が行われる可能性があります。

アルゴリズムまたは振る舞い

1. **型推論**: 片方の引数が UNKNOWNOID であれば、もう一方の型を適用して型を確定させます。 2. **直接検索**: OpernameGetOprid を呼び出し、指定された型で演算子を検索します。成功すればその Oid を返します。 3. **基底型による再試行**: UNKNOWN 型が含まれていた場合、引数の基底型を取得し、その基底型同士の組み合わせで再度 OpernameGetOprid を実行します。 4. **終了**: 適合する演算子が見つからない場合は InvalidOid を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_oper.c (内部の oper 関数などから呼び出し)
  • src/backend/parser/parse_func.c (ヒューリスティックな検索の一部として言及あり)
  • **呼び出し先**:
  • OpernameGetOprid
  • getBaseType

関連データ構造

  • Oid: PostgreSQLのシステムオブジェクト識別子。
  • List: 演算子名を保持するリスト構造。

失敗モードと境界条件

  • 演算子名がシステムカタログに存在しない場合、InvalidOid を返します。
  • arg1 と arg2 の両方が InvalidOid である場合や、型変換が不可能な組み合わせでは検索に失敗します。

保守担当者向けメモ

  • 本関数は厳密な一致を優先します。型キャストを伴う柔軟な演算子解決が必要な場合は、oper() 等の他の関数が適しています。
  • UNKNOWN 型の扱いは、SQL標準の型解決ルールに基づいています。変更時は parse_coerce.c 等の型変換ロジックとの整合性に注意してください。
