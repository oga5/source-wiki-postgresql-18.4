pairingheap_add

定義

  • 種別: function
  • 定義箇所: src/backend/lib/pairingheap.c:111

目的

ペアリングヒープに新しいノードを挿入します。

シグネチャまたは形状

void pairingheap_add(pairingheap *heap, pairingheap_node *node)

入力と出力

  • heap: 操作対象のペアリングヒープ構造体へのポインタ。
  • node: ヒープに追加するノード。
  • 出力: なし（副作用としてヒープ構造を更新）。

前提条件と不変条件

  • node は初期化済みである必要があります。
  • 挿入後、node->first_child は NULL にリセットされます。
  • 挿入後のヒープルートは、ヒープの比較関数に基づいた最小（または最大）値となります。

アルゴリズムまたは振る舞い

1. 追加する node の子ノードポインタを NULL に初期化します。 2. merge 関数を呼び出し、現在のヒープルート (heap->ph_root) と新しい node をマージして新しいルートを決定します。 3. 新しいルートの prev_or_parent および next_sibling を NULL に設定し、ヒープの整合性を維持します。 4. 償却計算量は O(1) です。

呼び出し元と呼び出し先

  • 呼び出し先: merge
  • 呼び出し元（直接的な証拠）:
  • src/backend/access/gist/gistget.c
  • src/backend/access/spgist/spgscan.c
  • src/backend/executor/nodeIndexscan.c
  • src/backend/replication/logical/reorderbuffer.c
  • src/backend/utils/time/snapmgr.c

関連データ構造

  • pairingheap: ヒープ全体を管理する構造体。
  • pairingheap_node: ヒープの各要素。親・子・兄弟へのポインタを持つ多重連結リスト構造。

失敗モードと境界条件

  • heap が NULL の場合、未定義の動作を引き起こします。
  • ヒープが空（ph_root が NULL）の場合、merge 関数が適切に処理し、新しいノードがルートとなります。

保守担当者向けメモ

  • 本関数はノードのメモリ管理を行いません。ノードのライフサイクルは呼び出し側で管理してください。
  • PAIRINGHEAP_DEBUG を定義すると、ヒープ構造の可視化が可能です。
