make_plain_restrictinfo

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/restrictinfo.c:102

目的

クエリプランナにおいて、式（Expr）を RestrictInfo 構造体にラップし、最適化に必要なメタデータ（結合可能性、参照リレーション、セキュリティレベル等）を初期化する。

シグネチャまたは形状

RestrictInfo *make_plain_restrictinfo(PlannerInfo *root, Expr *clause, Expr *orclause, bool is_pushed_down, bool has_clone, bool is_clone, bool pseudoconstant, Index security_level, Relids required_relids, Relids incompatible_relids, Relids outer_relids)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • clause: ラップ対象の式。
  • orclause: OR 句の場合のサブ句。
  • 各種フラグ（is_pushed_down, pseudoconstant 等）およびリレーション識別子（required_relids 等）。
  • **出力**: 初期化済みの RestrictInfo 構造体へのポインタ。

前提条件と不変条件

  • rinfo_serial は root->last_rinfo_serial をインクリメントして割り当てられる。
  • security_level > 0 の場合、leakproof プロパティが contain_leaked_vars により判定される。
  • 結合条件（バイナリ演算子）の場合、left_relids と right_relids が抽出され、それらが重ならない場合にのみ can_join が true となる。

アルゴリズムまたは振る舞い

1. makeNode(RestrictInfo) でメモリを確保し、引数で渡された値をフィールドにコピーする。 2. security_level に基づき leakproof を設定する。 3. is_opclause を判定し、バイナリ演算子の場合は左右の Relids を抽出して clause_relids を計算する。それ以外は式全体から pull_varnos で clause_relids を取得する。 4. root->outer_join_rels を除外して num_base_rels を算出する。 5. root->last_rinfo_serial を更新し、シリアル番号を付与する。 6. コスト見積もりや結合アルゴリズムに関連するキャッシュ用フィールド（mergeopfamilies, eval_cost 等）を「未設定」状態（デフォルト値）で初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/indxpath.c
  • src/backend/optimizer/util/restrictinfo.c 内の他の関数（make_restrictinfo 等）。
  • **呼び出し先**:
  • makeNode (マクロ)
  • contain_leaked_vars
  • is_opclause, get_leftop, get_rightop
  • pull_varnos
  • bms_union, bms_is_empty, bms_overlap, bms_difference, bms_num_members, bms_free

関連データ構造

  • RestrictInfo: 本関数が生成する主要な構造体。
  • PlannerInfo: プランナのコンテキスト情報。
  • Relids: リレーションの集合を表すビットマップ。

失敗モードと境界条件

  • required_relids が NULL の場合、clause_relids がデフォルトとして使用される。
  • バイナリ演算子でない式の場合、left_relids および right_relids は NULL に設定される。
  • can_join は、式がバイナリ演算子であり、かつ左右の引数が異なるリレーションを参照している場合にのみ true となる。

保守担当者向けメモ

  • 本関数で初期化されるキャッシュ用フィールド（mergeopfamilies 等）は、必要になるまで計算されない遅延評価の対象である。
  • rinfo_serial はプランナ実行中の一意な識別子として機能する。
  • pseudoconstant が true の場合、can_join は false である必要がある（アサーションで保護）。
