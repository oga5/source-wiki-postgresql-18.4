SearchSysCache

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/syscache.c:207

目的

システムカタログキャッシュから特定のタプルを検索し、取得します。

シグネチャまたは形状

HeapTuple SearchSysCache(int cacheId, Datum key1, Datum key2, Datum key3, Datum key4)

入力と出力

  • **入力**:
  • cacheId: 検索対象のキャッシュID。
  • key1 ～ key4: 検索に使用するキー値（Datum型）。未使用のキーには 0 を指定します。
  • **出力**:
  • 成功時: 見つかったタプルへのポインタ (HeapTuple)。
  • 失敗時: NULL。

前提条件と不変条件

  • cacheId は 0 以上かつ SysCacheSize 未満であること。
  • SysCache[cacheId] が有効なポインタであること。
  • 取得したタプルはキャッシュ内部のポインタであるため、呼び出し側で直接変更してはなりません。

アルゴリズムまたは振る舞い

1. Assert により、cacheId が有効範囲内であり、対応する SysCache エントリが初期化済みであることを検証します。 2. 指定された cacheId に対応する CatCache 構造体を取得します。 3. SearchCatCache を呼び出し、提供されたキーを使用してキャッシュ内を検索します。 4. 検索結果（タプルまたは NULL）をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/enum.c
  • src/backend/utils/adt/ruleutils.c
  • src/backend/utils/cache/syscache.c (内部のラッパー関数群)
  • **呼び出し先**:
  • SearchCatCache (src/backend/utils/cache/catcache.c)

関連データ構造

  • SysCache[]: システムキャッシュを保持する CatCache 構造体の配列。
  • HeapTuple: 検索結果として返されるタプル構造体。

失敗モードと境界条件

  • 指定されたキーに一致するタプルが存在しない場合、NULL を返します。
  • cacheId が無効な場合、Assert によりプロセスが停止します。

保守担当者向けメモ

  • 本関数が返すタプルはキャッシュ管理下にあるため、変更が必要な場合は SearchSysCacheCopy を使用してください。
  • 参照カウントの解放には ReleaseSysCache を使用する必要があります。
