ExecAggInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAgg.c:4806

目的

並列クエリ実行時、集約ノードの統計情報（AggregateInstrumentation）を共有メモリ上に割り当て、ワーカー間で共有するための初期化を行う。

シグネチャまたは形状

void ExecAggInitializeDSM(AggState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: 集約ノードの状態を保持する AggState 構造体。
  • pcxt: 並列クエリ実行のコンテキストを保持する ParallelContext。
  • **出力**: なし（副作用として node->shared_info が設定され、共有メモリが初期化される）。

前提条件と不変条件

  • node->ss.ps.instrument が有効であり、かつ pcxt->nworkers > 0 である場合にのみ処理が実行される。
  • 共有メモリ領域は shm_toc_allocate を介して割り当てられ、ゼロクリアされる。

アルゴリズムまたは振る舞い

1. インストルメンテーションが不要、またはワーカーが存在しない場合は即座にリターンする。 2. SharedAggInfo 構造体と、ワーカー数分の AggregateInstrumentation 配列を格納するために必要なサイズを計算する。 3. shm_toc_allocate を使用して共有メモリ領域を確保し、memset でゼロクリアする。 4. 確保した領域の num_workers にワーカー数を設定する。 5. shm_toc_insert を呼び出し、プランノードIDをキーとして共有メモリ領域を登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:529 （ExecAggInitializeDSM((AggState *) planstate, d->pcxt);）
  • **呼び出し先**:
  • shm_toc_allocate
  • shm_toc_insert
  • memset

関連データ構造

  • AggState: 集約ノードの実行状態を管理。
  • ParallelContext: 並列クエリの共有メモリ管理コンテキスト。
  • SharedAggInfo: 共有メモリ上に配置される集約統計情報のヘッダー構造体。
  • AggregateInstrumentation: 各ワーカーの集約実行統計を保持する構造体。

失敗モードと境界条件

  • pcxt->nworkers が 0 の場合、共有メモリの割り当ては行われない。
  • shm_toc_allocate がメモリ不足等で失敗した場合、PostgreSQLの標準的なエラーハンドリング（elog 等）により処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数は並列クエリの初期化フェーズで呼び出される。共有メモリのレイアウト変更を行う場合は、SharedAggInfo の構造と shm_toc への登録ロジックを整合させる必要がある。
