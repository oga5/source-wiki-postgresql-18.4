WalWriteMethodOps

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_basebackup/walmethods.h:41

目的

WAL（Write-Ahead Log）データの書き込みおよび管理を行うための抽象インターフェースを定義する関数ポインタのテーブルです。ディレクトリ出力やTARアーカイブ出力など、異なるストレージ形式を抽象化するために使用されます。

シグネチャまたは形状

typedef struct WalWriteMethodOps
{
    Walfile *(*open_for_write) (WalWriteMethod *wwmethod, const char *pathname, const char *temp_suffix, size_t pad_to_size);
    int     (*close) (Walfile *f, WalCloseMethod method);
    bool    (*existsfile) (WalWriteMethod *wwmethod, const char *pathname);
    ssize_t (*get_file_size) (WalWriteMethod *wwmethod, const char *pathname);
    char    *(*get_file_name) (WalWriteMethod *wwmethod, const char *pathname, const char *temp_suffix);
    ssize_t (*write) (Walfile *f, const void *buf, size_t count);
    int     (*sync) (Walfile *f);
    bool    (*finish) (WalWriteMethod *wwmethod);
    void    (*free) (WalWriteMethod *wwmethod);
} WalWriteMethodOps;

入力と出力

  • **open_for_write**: Walfile を返す。失敗時は NULL。
  • **close**: 成功時に 0、失敗時にそれ以外の値を返す。
  • **existsfile**: ファイル存在時に true を返す。
  • **get_file_size**: ファイルサイズを返す。失敗時は -1。
  • **get_file_name**: ファイル名文字列を返す（pg_malloc 済み）。
  • **write**: 書き込みバイト数を返す。失敗時は -1。
  • **sync**: 成功時に 0 を返す。
  • **finish**: 成功時に true を返す。
  • **free**: 戻り値なし。

前提条件と不変条件

  • open_for_write で temp_suffix が指定された場合、その名前でファイルを開き、close 時に自動的にリネームされる必要がある。
  • pad_to_size が指定された場合、メソッドがサポートしていれば指定サイズまでNULバイトでパディングされる。

アルゴリズムまたは振る舞い

各関数ポインタは、特定のストレージバックエンド（ディレクトリ、TAR等）に応じた実装を提供します。finish は共有リソースのクリーンアップ（TARヘッダーの書き込み等）を行い、free は構造体自体のメモリ解放を担当します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/walmethods.c 内の各関数が、WalWriteMethod 構造体経由でこのテーブルの関数を呼び出します。
  • **呼び出し先**: WalDirectoryMethodOps および WalTarMethodOps がこの構造体のインスタンスとして定義されています。

関連データ構造

  • WalWriteMethod: この構造体を保持する上位の抽象構造体。
  • Walfile: open_for_write が返すファイルハンドル。
  • WalCloseMethod: close 関数に渡される終了処理の指定。

失敗モードと境界条件

  • open_for_write はファイルオープン失敗時に NULL を返します。
  • get_file_size は失敗時に -1 を返します。
  • write はエラー時に -1 を返します。
  • finish はリソースのクリーンアップに失敗した場合、リソースを閉じた上で false を返します。

保守担当者向けメモ

  • 新しい書き込み方式を追加する場合は、この構造体の全関数ポインタを実装した静的インスタンスを walmethods.c に定義してください。
  • WalWriteMethodOps 自体は不変のテーブルとして扱うことが推奨されます。
