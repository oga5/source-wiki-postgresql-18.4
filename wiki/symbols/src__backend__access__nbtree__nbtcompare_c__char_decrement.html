char_decrement

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtcompare.c:559

目的

B-tree インデックスのスキップスキャンにおいて、char 型の値を 1 減算し、その結果を返す。

シグネチャまたは形状

static Datum char_decrement(Relation rel, Datum existing, bool *underflow)

入力と出力

  • **入力**:
  • rel: 対象のリレーション（使用されない）。
  • existing: 減算対象の char 型値（Datum 形式）。
  • underflow: アンダーフロー発生時に状態を書き込むためのポインタ。
  • **出力**:
  • 戻り値: 減算後の Datum 値。アンダーフロー時は 0。
  • *underflow: 減算結果がアンダーフローした場合は true、それ以外は false。

前提条件と不変条件

  • existing は char 型として解釈可能であること。
  • underflow ポインタは有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. existing を uint8 型の cexisting に変換する。 2. cexisting が 0 の場合:

  • *underflow を true に設定する。
  • 0 を返す（戻り値は未定義）。

    • 3. cexisting が 0 以外の場合:

  • *underflow を false に設定する。
  • cexisting - 1 を char 型の Datum に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtcompare.c 内の SkipSupport 構造体初期化処理（sksup->decrement = char_decrement;）。
  • **呼び出し先**: PostgreSQL の型変換マクロ（UInt8GetDatum, CharGetDatum）。

関連データ構造

  • SkipSupport: インデックススキップスキャン用の増減ロジックを保持する構造体。

失敗モードと境界条件

  • existing が 0 の場合、アンダーフローが発生し、*underflow が true となる。

保守担当者向けメモ

  • 本関数は char 型専用の減算ロジックである。他の型については、同ファイル内の int2_decrement 等の対応する関数を参照すること。
