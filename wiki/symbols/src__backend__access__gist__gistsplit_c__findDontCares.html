findDontCares

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistsplit.c:112

目的

GiSTインデックスのページ分割において、指定された列（attno）に関してペナルティなしで移動可能なタプル（don't-careタプル）を特定し、再分配の対象としてマークします。

シグネチャまたは形状

static int findDontCares(Relation r, GISTSTATE *giststate, GISTENTRY *valvec, GistSplitVector *spl, int attno);

入力と出力

  • **入力**:
  • r: 対象のリレーション。
  • giststate: GiSTインデックスの状態管理構造体。
  • valvec: タプルのエントリーベクトル。
  • spl: 分割ベクトル（左側・右側のタプル集合およびdon't-careフラグ配列を含む）。
  • attno: 評価対象の列インデックス。
  • **出力**:
  • int: 特定されたdon't-careタプルの総数。

前提条件と不変条件

  • attno で指定される列は、呼び出し元（gistSplitByKey）により非NULLであることが保証されています。
  • spl->spl_dontcare 配列は、呼び出し前に適切なサイズで初期化されている必要があります。

アルゴリズムまたは振る舞い

1. **左側タプルの評価**: 現在左側に配置されている各タプルに対し、右側のユニオンキーに追加した際のペナルティを gistpenalty で計算します。ペナルティが 0.0 の場合、そのタプルをdon't-careと見なし、spl->spl_dontcare を true に設定します。 2. **右側タプルの評価**: 同様に、現在右側に配置されている各タプルに対し、左側のユニオンキーに追加した際のペナルティを計算します。ペナルティが 0.0 の場合、同様にdon't-careとしてマークします。 3. **集計**: 特定されたdon't-careタプルの総数をカウントして返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistsplit.c 内の gistSplitByKey 関数から呼び出されます。
  • **呼び出し先**:
  • gistentryinit（マクロ）
  • gistpenalty（GiSTのペナルティ計算関数）

関連データ構造

  • GISTSTATE: GiSTインデックスの操作状態を保持。
  • GISTENTRY: インデックスエントリーの情報を保持。
  • GistSplitVector: 分割結果およびdon't-care状態を管理する構造体。

失敗モードと境界条件

  • valvec や spl が指すメモリ領域が不正な場合、セグメンテーション違反が発生する可能性があります。
  • gistpenalty が内部でエラーを発生させた場合、処理は中断されます。
  • 該当するタプルが一つも存在しない場合、0 を返します。

保守担当者向けメモ

  • 本関数は、ペナルティ関数が必ずしも最適ではないという前提に基づいています。
  • attno は0から始まる列インデックスであり、再帰的な分割処理の一部として呼び出されるため、呼び出し元の文脈（再帰深度）を考慮してください。
