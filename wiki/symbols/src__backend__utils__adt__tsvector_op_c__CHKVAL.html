CHKVAL

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/adt/tsvector_op.c:36

目的

全文検索クエリ（tsquery）を評価する際、対象となるデータ（tsvector 等）の構造体へのポインタと、評価に必要なオペランド情報を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct
{
	WordEntry  *arrb;     /* 語彙配列の開始位置 */
	WordEntry  *arre;     /* 語彙配列の終了位置 */
	char	   *values;   /* 語彙文字列データへのポインタ */
	char	   *operand;  /* クエリ評価用のオペランドデータ */
} CHKVAL;

入力と出力

本構造体は関数引数として渡される際、void * 型（checkval）としてキャストされて受け取られることが一般的です。各フィールドは、クエリ評価ロジックが対象データにアクセスするための参照先として機能します。

前提条件と不変条件

  • arrb および arre は、対象となる tsvector 内の WordEntry 配列を指している必要があります。
  • values は、tsvector 内の語彙文字列領域を指している必要があります。
  • operand は、評価対象のクエリに関連付けられたデータ領域を指します。

アルゴリズムまたは振る舞い

クエリ評価関数（TS_execute 等）において、この構造体は「評価対象のデータセット」をカプセル化します。評価ロジックは、この構造体を経由してバイナリサーチや文字列比較を行い、クエリ条件とのマッチングを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_bool.c
  • contrib/ltree/ltxtquery_op.c
  • src/backend/utils/adt/tsgistidx.c

    • （これらは CHKVAL をローカル変数として定義し、クエリ評価関数へ渡しています）

  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • WordEntry: tsvector 内の各語彙のメタデータ（位置、重み、長さ）。
  • TSVector: 全文検索用ベクトル型。

失敗モードと境界条件

  • checkval として渡されたポインタが NULL である場合、または構造体内の各ポインタが指すメモリ領域が不正な場合、セグメンテーションフォールトが発生します。
  • arrb と arre の整合性が取れていない場合、バイナリサーチ等の処理で範囲外アクセスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は tsvector だけでなく、intarray や ltree といった外部モジュールでも類似の目的で再定義・利用されています。変更を加える際は、これら関連モジュールへの影響を考慮してください。
  • operand フィールドの具体的な型や構造は、利用するコンテキスト（tsquery か ltxtquery か等）に依存するため、キャスト時の型安全性に注意してください。
