pg_logging_increase_verbosity

定義

  • 種別: function
  • 定義箇所: src/common/logging.c:184

目的

ログ出力の冗長性（verbosity）を一段階引き上げるためのヘルパー関数です。主にコマンドライン引数（--verbose 等）の処理時に使用されます。

シグネチャまたは形状

void pg_logging_increase_verbosity(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 __pg_log_level を更新）

前提条件と不変条件

  • __pg_log_level が PG_LOG_NOTSET + 1 より大きい場合にのみ、値のデクリメントが行われます。
  • ログレベルの列挙値は、値が小さいほど冗長性が高くなるように設計されています。

アルゴリズムまたは振る舞い

1. 現在のログレベル __pg_log_level が PG_LOG_NOTSET + 1 を超えているか確認します。 2. 条件を満たす場合、__pg_log_level を 1 減算します。これにより、より詳細なログが出力されるようになります。

呼び出し元と呼び出し先

  • 呼び出し元: 以下の各フロントエンドプログラムのメイン処理から呼び出されます。
  • pg_amcheck, pg_archivecleanup, pg_createsubscriber, pg_combinebackup, pg_dump, pg_dumpall, pg_restore, pg_rewind, pgbench
  • 呼び出し先: なし（直接的な関数呼び出しは含まれません）

関連データ構造

  • __pg_log_level: ログ出力の閾値を保持する内部グローバル変数。

失敗モードと境界条件

  • __pg_log_level が PG_LOG_NOTSET + 1 以下の場合、この関数を呼び出してもログレベルは変更されません。

保守担当者向けメモ

  • 本関数は src/include/common/logging.h で公開されています。
  • ログレベルの列挙値の定義順序に依存した実装であるため、列挙型の定義を変更する際は本関数のロジックとの整合性に注意してください。
