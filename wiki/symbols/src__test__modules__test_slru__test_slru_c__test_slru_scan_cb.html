test_slru_scan_cb

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_slru/test_slru.c:49

目的

SLRUディレクトリ内のセグメントファイルを走査する際のコールバック関数として機能し、対象ファイルを削除する処理を仲介します。

シグネチャまたは形状

static bool test_slru_scan_cb(SlruCtl ctl, char *filename, int64 segpage, void *data);

入力と出力

  • **入力**:
  • ctl: 対象となるSLRU制御構造体。
  • filename: 走査対象のファイル名。
  • segpage: セグメントに関連付けられたページ番号。
  • data: コールバックに渡される補助データ。
  • **出力**:
  • bool: 処理の継続可否（SlruScanDirCbDeleteAll の戻り値に依存）。

前提条件と不変条件

  • SlruScanDirectory から呼び出されることを想定しています。
  • ctl は有効な SlruCtlData を指している必要があります。

アルゴリズムまたは振る舞い

1. elog(NOTICE, ...) を使用して、関数が呼び出された旨をログに出力します。 2. SlruScanDirCbDeleteAll を呼び出し、指定されたセグメントファイルを削除する標準的な処理を実行してその結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: test_slru_delete_all 関数内において SlruScanDirectory の引数として参照されています。
  • **呼び出し先**: SlruScanDirCbDeleteAll（PostgreSQLのSLRUコアAPI）。

関連データ構造

  • SlruCtl: SLRUの制御状態を保持する構造体。
  • TestSlruCtl: 本モジュールで定義されているテスト用SLRU制御インスタンス。

失敗モードと境界条件

  • SlruScanDirCbDeleteAll が内部でファイル削除に失敗した場合、そのエラー状態が本関数を通じて伝播します。

保守担当者向けメモ

  • 本関数は SlruScanDirectory の走査ロジックにフックするためのラッパーです。削除以外の処理を追加したい場合は、この関数内で条件分岐を行うか、別のコールバック関数を定義してください。
