var_eq_const

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/selfuncs.c:302

目的

変数と定数の等価比較（var = const）における選択率（Selectivity）を推定する。

シグネチャまたは形状

double var_eq_const(VariableStatData *vardata, Oid oproid, Oid collation,
                    Datum constval, bool constisnull,
                    bool varonleft, bool negate)

入力と出力

  • **入力**:
  • vardata: 変数の統計情報（VariableStatData）。
  • oproid: 演算子の OID。
  • collation: 照合順序。
  • constval: 定数値。
  • constisnull: 定数が NULL かどうか。
  • varonleft: 変数が演算子の左側にあるか。
  • negate: 不等価比較（<>）として計算するか。
  • **出力**:
  • double: 推定された選択率（0.0〜1.0）。

前提条件と不変条件

  • constisnull が真の場合、演算子は厳密（strict）であると仮定され、常に 0.0 を返す。
  • 統計情報が存在しない場合、get_variable_numdistinct を用いて一様分布を仮定した推定を行う。
  • 最終的な戻り値は CLAMP_PROBABILITY マクロにより 0.0〜1.0 の範囲に収められる。

アルゴリズムまたは振る舞い

1. **NULL 処理**: 定数が NULL ならば選択率は 0.0。 2. **一意性チェック**: vardata->isunique が真であれば、1.0 / rel->tuples を返す。 3. **MCV（最頻値）利用**: pg_statistic に MCV 情報がある場合、演算子を適用して定数と一致する MCV を探す。

  • 一致した場合: その MCV の頻度を返す。
  • 一致しない場合: 1.0 - sum(MCV頻度) - nullfrac を残りの非 MCV 値の数で割った値を推定値とする。

    • 4. **統計情報欠落時**: get_variable_numdistinct を用いて、全値が等確率であると仮定して推定する。 5. **否定処理**: negate が真の場合、1.0 - 選択率 - nullfrac を計算する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/like_support.c
  • src/backend/utils/adt/selfuncs.c（eqsel 等の内部処理）
  • **呼び出し先**:
  • get_opcode, get_attstatsslot, free_attstatsslot, fmgr_info, FunctionCallInvoke, get_variable_numdistinct 等。

関連データ構造

  • VariableStatData: 変数の統計情報（statsTuple を含む）。
  • Form_pg_statistic: システムカタログ pg_statistic の行構造。
  • AttStatsSlot: 統計情報スロット。

失敗モードと境界条件

  • vardata->statsTuple が無効な場合、統計情報なしのデフォルト推定にフォールバックする。
  • otherdistinct が 1 以下の場合は除算を回避するロジックが含まれる。
  • MCV が存在しない場合、get_variable_numdistinct の結果に依存する。

保守担当者向けメモ

  • 本関数は eqsel の実体として機能する。
  • statistic_proc_security_check により、統計情報へのアクセス権限が検証される。
  • MCV との比較には FunctionCallInvoke を使用しており、演算子の実行コストが選択率計算に含まれる点に注意。
