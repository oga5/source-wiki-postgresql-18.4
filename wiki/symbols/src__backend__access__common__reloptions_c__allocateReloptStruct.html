allocateReloptStruct

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/reloptions.c:1721

目的

リレーションのストレージパラメータを格納するためのメモリ領域を動的に確保する。特に、可変長データ（文字列型オプション）を含む構造体のサイズを計算し、必要なメモリを割り当てる。

シグネチャまたは形状

void *allocateReloptStruct(Size base, relopt_value *options, int numoptions)

入力と出力

  • **入力**:
  • base: 構造体の固定部分のサイズ（sizeof(struct)）。
  • options: 解析済みのオプション値の配列（relopt_value）。
  • numoptions: options 配列内の要素数。
  • **出力**:
  • 確保されたメモリ領域へのポインタ（void *）。palloc0 によりゼロ初期化されている。

前提条件と不変条件

  • options 配列内の各要素は、relopt_value 構造体として正しく初期化されている必要がある。
  • 文字列型オプションの場合、fill_cb が定義されていればそのコールバックがサイズ計算に使用され、未定義の場合は GET_STRING_RELOPTION_LEN マクロが使用される。

アルゴリズムまたは振る舞い

1. base サイズを初期値として設定する。 2. options 配列を走査し、RELOPT_TYPE_STRING 型のオプションを特定する。 3. 文字列型オプションについて、fill_cb が存在する場合はその戻り値を、存在しない場合は GET_STRING_RELOPTION_LEN に 1 を加えた値を size に加算する。 4. 最終的な size を用いて palloc0 を呼び出し、メモリを確保して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/reloptions.c:1965
  • src/backend/access/common/reloptions.c:2002
  • **呼び出し先**:
  • palloc0 (メモリ確保)
  • optstr->fill_cb (文字列オプションのサイズ計算用コールバック)

関連データ構造

  • relopt_value: 解析されたオプション値を保持する構造体。
  • relopt_string: 文字列型オプションのメタデータ（fill_cb 等を含む）。

失敗モードと境界条件

  • numoptions が 0 の場合、base サイズのみのメモリが確保される。
  • palloc0 がメモリ不足に陥った場合、PostgreSQL のエラーハンドリング機構により例外が発生する。

保守担当者向けメモ

  • 本関数は、構造体の固定サイズに加えて、可変長文字列の格納に必要な追加領域を動的に計算する。新しい文字列型オプションを追加し、かつ特殊なサイズ計算が必要な場合は fill_cb の実装を確認すること。
