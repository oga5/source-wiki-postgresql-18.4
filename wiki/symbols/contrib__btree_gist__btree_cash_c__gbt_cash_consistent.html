gbt_cash_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_cash.c:135

目的

GiST インデックス検索において、money 型のインデックスキーがクエリ条件に適合するかを判定します。

シグネチャまたは形状

Datum gbt_cash_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • entry: GISTENTRY* (インデックスエントリ)
  • query: Cash (検索条件値)
  • strategy: StrategyNumber (比較演算子の種類)
  • recheck: bool* (出力用フラグ)
  • **出力**:
  • bool (適合する場合は true、しない場合は false)
  • *recheck: 常に false を設定（インデックスによる判定が完全であることを示す）

前提条件と不変条件

  • entry->key は cashKEY 構造体として解釈可能であること。
  • tinfo（gbtree_ninfo 構造体）が money 型の特性を正しく保持していること。

アルゴリズムまたは振る舞い

1. *recheck を false に設定し、インデックスによる判定が正確であることを保証します。 2. entry->key から cashKEY を取得し、GBT_NUMKEY_R 構造体に下限値と上限値をマッピングします。 3. gbt_num_consistent を呼び出し、tinfo に定義された比較関数を用いて、クエリ条件とインデックスキーの適合性を判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス検索エンジン（直接の呼び出し元コードはソース抜粋に含まれていません）。
  • **呼び出し先**: gbt_num_consistent（btree_utils_num.c で定義された汎用関数）。

関連データ構造

  • cashKEY: money 型の範囲（lower, upper）を保持する構造体。
  • GBT_NUMKEY_R: 数値範囲を表現するための汎用構造体。
  • tinfo: money 型の比較ロジックを定義した gbtree_ninfo 構造体。

失敗モードと境界条件

  • 本関数は常に recheck = false を返すため、インデックス検索結果に対する再チェックは不要です。
  • 内部で呼び出される gbt_num_consistent が、money 型の比較演算（gbt_cashgt 等）において適切な結果を返すことに依存しています。

保守担当者向けメモ

  • 本関数は btree_utils_num.h の汎用ロジックを利用するアダプターです。動作の詳細は btree_utils_num.c を参照してください。
  • money 型は内部的に 64bit 整数として扱われます。比較ロジックに変更を加える場合は、tinfo に登録されている比較関数群を確認してください。
