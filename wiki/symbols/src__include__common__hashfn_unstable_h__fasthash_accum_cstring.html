fasthash_accum_cstring

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/common/hashfn_unstable.h:288

目的

NUL終端文字列をハッシュ状態に蓄積し、文字列の長さをハッシュ計算の「tweak（調整値）」として利用することで、効率的かつハッシュテスト（SMHasher等）に適したハッシュ値を生成します。

シグネチャまたは形状

static inline size_t fasthash_accum_cstring(fasthash_state *hs, const char *str)

入力と出力

  • hs: ハッシュ計算の進行状況を保持する fasthash_state 構造体へのポインタ。
  • str: ハッシュ対象のNUL終端文字列。
  • 戻り値: 文字列の長さ（size_t）。

前提条件と不変条件

  • 入力文字列 str はNUL終端されている必要があります。
  • 本関数は「不安定」なハッシュアルゴリズムの一部であり、プラットフォームのエンディアンに依存します。永続化（ディスク保存）が必要なデータには使用できません。

アルゴリズムまたは振る舞い

1. **最適化判定**: 64bit環境（SIZEOF_VOID_P >= 8）かつ文字列ポインタが uint64 にアライメントされている場合、fasthash_accum_cstring_aligned を使用してワード単位で高速に処理します。 2. **フォールバック**: 32bit環境、またはアライメントされていない文字列の場合は、fasthash_accum_cstring_unaligned を使用してバイト単位で処理します。 3. **デバッグ検証**: USE_ASSERT_CHECKING が有効な場合、アライメント処理の結果と非アライメント処理の結果を比較し、ハッシュ状態の整合性を検証します。

関連データ構造

  • fasthash_state: ハッシュ計算の進行状況を保持する構造体。

失敗モードと境界条件

  • 文字列がNUL終端されていない場合、メモリ境界外アクセスが発生する可能性があります。
  • 32bit環境では最適化が適用されず、常に非アライメント処理が選択されます。

保守担当者向けメモ

  • 本関数は fasthash_accum_cstring_aligned を呼び出す際、アドレスサニタイザー（ASan）の誤検知を避けるための属性保護が適用されている可能性があるため、実装変更時は注意してください。
  • 文字列長をハッシュ計算に含める設計は、ハッシュの品質を維持するために不可欠です。
