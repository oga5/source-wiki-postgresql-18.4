recordMultipleDependencies

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_depend.c:58

目的

複数のオブジェクトに対する依存関係をシステムカタログ pg_depend に一括して永続化します。

シグネチャまたは形状

void recordMultipleDependencies(const ObjectAddress *depender,
                               const ObjectAddress *referenced,
                               int nreferenced,
                               DependencyType behavior)

入力と出力

  • depender: 依存元オブジェクトのアドレス。
  • referenced: 依存先オブジェクトのアドレス配列。
  • nreferenced: 依存先オブジェクトの数。
  • behavior: 依存関係の性質（DependencyType）。
  • 出力: なし（pg_depend テーブルへの挿入のみ）。

前提条件と不変条件

  • ブートストラップ処理モード（IsBootstrapProcessingMode）中は何も行われません。
  • nreferenced <= 0 の場合は即座にリターンします。
  • 依存先オブジェクトがシステムによって固定（Pinned）されている場合、そのオブジェクトへの依存関係は記録されません。

アルゴリズムまたは振る舞い

1. pg_depend テーブルを RowExclusiveLock で開きます。 2. メモリ効率を考慮し、MAX_CATALOG_MULTI_INSERT_BYTES に基づいてバッファリング用のスロットを確保します。 3. 各 referenced オブジェクトを順次処理し、isObjectPinned で除外判定を行います。 4. 依存関係情報をタプルスロットに格納し、バッファが満杯になるか全処理が完了した時点で CatalogTuplesMultiInsertWithInfo を呼び出し、一括挿入を実行します。 5. 最後にインデックスを閉じ、テーブルロックを解除してリソースを解放します。 6. 重複する依存関係のチェックは行いません。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/dependency.c 内の複数箇所、および src/backend/catalog/pg_depend.c 内。
  • 呼び出し先: table_open, isObjectPinned, MakeSingleTupleTableSlot, CatalogTuplesMultiInsertWithInfo, CatalogOpenIndexes, CatalogCloseIndexes, table_close 等。

関連データ構造

  • ObjectAddress: オブジェクトを特定するための識別子（classId, objectId, objectSubId）。
  • pg_depend: 依存関係を保持するシステムカタログ。
  • TupleTableSlot: タプル挿入用の中間バッファ。

失敗モードと境界条件

  • nreferenced が 0 以下の場合、処理はスキップされます。
  • ブートストラップ中は pg_depend が存在しない可能性があるため、処理を回避します。
  • メモリ不足時には palloc がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数は CatalogTuplesMultiInsertWithInfo を利用して効率化されています。新しい依存関係タイプを追加する際は、pg_depend のスキーマ定義と整合性が取れているか確認してください。
  • isObjectPinned のチェックは pg_depend の肥大化を防ぐ重要な最適化であるため、削除しないでください。
