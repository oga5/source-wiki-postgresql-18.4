SPI_commit

定義

  • 種別: function
  • 定義箇所: src/backend/executor/spi.c:319

目的

SPI（Server Programming Interface）経由で実行中のトランザクションをコミットします。

シグネチャまたは形状

void SPI_commit(void)

入力と出力

  • 入力: なし
  • 出力: なし（成功時）

前提条件と不変条件

  • SPI 接続が確立されている必要があります。
  • トランザクション制御が許可されているコンテキスト（非アトミックなコンテキスト）で呼び出される必要があります。

アルゴリズムまたは振る舞い

内部関数 _SPI_commit(false) を呼び出し、現在のトランザクションをコミットします。引数 false は、コミット後に新しいトランザクションを自動的に開始しない（チェーンしない）ことを示します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/pl/plperl/plperl.c
  • src/pl/plpgsql/src/pl_exec.c
  • src/pl/plpython/plpy_spi.c
  • src/pl/tcl/pltcl.c
  • 呼び出し先:
  • _SPI_commit（内部関数）

関連データ構造

  • SPI 関連の内部スタック（_SPI_stack）がトランザクション状態の管理に使用されます。

失敗モードと境界条件

  • トランザクション制御が許可されていないコンテキストで呼び出された場合、エラーが発生します。

保守担当者向けメモ

  • SPI_commit_and_chain とは異なり、本関数はコミット後に新しいトランザクションを開始しません。
  • 内部実装の詳細は _SPI_commit に依存しているため、トランザクション制御のロジック変更時は _SPI_commit を参照してください。
