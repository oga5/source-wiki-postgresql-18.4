SplitGUCList

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varlena.c:3828

目的

区切り文字で区切られた文字列（識別子やファイル名のリスト）を解析し、PostgreSQLの List 構造に変換する。

シグネチャまたは形状

bool SplitGUCList(char *rawstring, char separator, List **namelist);

入力と出力

  • **入力**:
  • rawstring: 解析対象の文字列（破壊的に変更される）。
  • separator: 要素を区切る文字（例: ,）。
  • **出力**:
  • namelist: 解析された文字列のリストが格納されるポインタ。
  • 戻り値: 成功時に true、構文エラー（引用符の不一致や空の識別子など）時に false を返す。

前提条件と不変条件

  • rawstring は呼び出し元で確保されたメモリ領域を指す必要がある。
  • 内部で rawstring を直接書き換えるため、呼び出し元は元の文字列を保持する必要がある場合は事前にコピーを作成すること。
  • 引用符で囲まれた識別子内の "" は、単一の " として扱われる。

アルゴリズムまたは振る舞い

1. 先頭の空白をスキップする。空文字列の場合は true を返す。 2. 文字列を順次走査し、以下のいずれかの形式で識別子を抽出する：

  • **引用符付き**: " で囲まれた範囲。内部の "" は " に置換（collapse）される。
  • **引用符なし**: separator または空白文字が出現するまでを識別子とする。

    • 3. 抽出した識別子を lappend を用いて namelist に追加する。 4. 識別子直後の空白をスキップし、separator があれば次の要素へ、終端であれば終了する。 5. 構文エラー（引用符の閉じ忘れ、空の識別子、予期しない文字）が検出された場合は直ちに false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/hba.c
  • src/backend/postmaster/postmaster.c
  • src/backend/storage/file/fd.c
  • src/backend/utils/adt/ruleutils.c
  • src/bin/pg_dump/dumputils.c
  • **呼び出し先**:
  • scanner_isspace (マクロ)
  • strchr
  • memmove
  • strlen
  • lappend

関連データ構造

  • List: PostgreSQLの汎用リスト構造。

失敗モードと境界条件

  • **引用符の不一致**: 閉じ引用符が見つからない場合は false を返す。
  • **空の識別子**: 引用符なしで separator が連続する場合など、識別子が空になる場合は false を返す。
  • **不正な構文**: 識別子と区切り文字の間に予期しない文字がある場合は false を返す。

保守担当者向けメモ

  • 本関数は破壊的読み込みを行うため、引数 rawstring が定数文字列（const char *）を指していないことを確認すること。
  • scanner_isspace はロケール依存の可能性があるため、使用環境の文字エンコーディングに注意が必要。
