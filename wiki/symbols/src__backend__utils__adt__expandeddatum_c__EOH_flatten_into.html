EOH_flatten_into

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/expandeddatum.c:80

目的

展開済みオブジェクト（ExpandedObjectHeader）の内容を、指定されたメモリ領域へフラットな（ディスク保存可能な）形式で書き出します。

シグネチャまたは形状

void EOH_flatten_into(ExpandedObjectHeader *eohptr, void *result, Size allocated_size)

入力と出力

  • eohptr: 展開済みオブジェクトのヘッダーへのポインタ。
  • result: フラット化されたデータを格納するターゲットメモリ領域。
  • allocated_size: result に割り当てられたメモリサイズ。
  • 出力: なし（result が指すメモリ領域にデータが書き込まれます）。

前提条件と不変条件

  • eohptr は有効な ExpandedObjectHeader を指している必要があります。
  • result が指すターゲットアドレスは、最大アライメント（maxaligned）されていることが期待されます。
  • allocated_size は、対象オブジェクトのフラット化に必要なサイズ以上である必要があります。

アルゴリズムまたは振る舞い

1. eohptr 内のメソッドテーブル（eoh_methods）にアクセスします。 2. 型固有の flatten_into メソッドを呼び出し、展開済みオブジェクトのデータを result が指すメモリ領域へコピー・変換します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/common/detoast.c:91
  • src/backend/access/common/heaptuple.c:339
  • src/backend/executor/execTuples.c:248
  • src/backend/utils/adt/datum.c:152, 534
  • src/pl/plpgsql/src/pl_exec.c:882
  • 呼び出し先:
  • eohptr->eoh_methods->flatten_into（型固有の実装へ委譲）

関連データ構造

  • ExpandedObjectHeader: 展開済みオブジェクトの共通ヘッダー。
  • ExpandedObjectMethods: 型固有のメソッドテーブル（flatten_into を保持）。

失敗モードと境界条件

  • allocated_size が不足している場合、ターゲットメモリのバッファオーバーフローが発生する可能性があります。
  • eohptr が無効なポインタである場合、セグメンテーション違反が発生します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際のフラット化ロジックは各データ型（array_expanded.c や expandedrecord.c 等）で定義された flatten_into メソッドに依存します。
  • 呼び出し元で EOH_get_flat_size を使用して必要なサイズを事前に確認してから本関数を呼び出す設計が一般的です。
