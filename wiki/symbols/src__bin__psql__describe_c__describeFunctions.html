describeFunctions

定義

  • 種別: function
  • 定義箇所: src/bin/psql/describe.c:294

目的

psql のメタコマンド \df を実行し、データベース内の関数、プロシージャ、集約、ウィンドウ関数、トリガーに関する情報をシステムカタログから取得して整形表示する。

シグネチャまたは形状

bool describeFunctions(const char *functypes, const char *func_pattern,
                       char **arg_patterns, int num_arg_patterns,
                       bool verbose, bool showSystem)

入力と出力

  • **入力**:
  • functypes: フィルタリング対象の文字（a, n, p, t, w 等）。
  • func_pattern: 関数名のパターン。
  • arg_patterns: 引数の型パターン。
  • num_arg_patterns: 引数パターンの数。
  • verbose: 詳細表示フラグ。
  • showSystem: システムオブジェクトを含めるかどうかのフラグ。
  • **出力**:
  • 成功時に true を返す。

前提条件と不変条件

  • PostgreSQL 9.2 以降のサーバー接続を前提とする。
  • functypes に指定可能な文字は anptwSx+ に限定される。
  • p（プロシージャ）オプションは PostgreSQL 11.0 以降でのみ有効。

アルゴリズムまたは振る舞い

1. 引数 functypes の妥当性を検証し、エラー時はログを出力して終了する。 2. サーバーバージョン（pset.sversion）に基づき、クエリ生成ロジックを切り替える。 3. initPQExpBuffer を用いて SQL クエリを構築する。

  • pg_catalog.pg_proc を主軸に、必要に応じて pg_namespace や pg_type を結合する。
  • verbose モード時は、Volatility、Parallel、Owner、Security、Leakproof、Language、Description 等の情報を追加取得する。

    • 4. num_arg_patterns に基づき、引数の型を絞り込むための LEFT JOIN を動的に追加する。 5. 構築したクエリを実行し、結果を整形して出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c:1328（\df コマンドのディスパッチャー）。
  • **呼び出し先**: initPQExpBuffer, printfPQExpBuffer, appendPQExpBuffer, pg_log_error, printACLColumn 等の psql 内部ユーティリティ。

関連データ構造

  • PQExpBufferData: SQL クエリ構築用バッファ。
  • PGresult: クエリ実行結果の保持。
  • printQueryOpt: 出力オプション。

失敗モードと境界条件

  • 不正な functypes が渡された場合、エラーメッセージを出力する。
  • サーバーバージョンが 11 未満で p オプションを指定した場合、エラーメッセージを出力する。
  • データベース接続が切断されている場合や、カタログへのアクセス権限がない場合はクエリ実行が失敗する可能性がある。

保守担当者向けメモ

  • PostgreSQL 11 で導入された prokind 列への移行により、クエリ生成ロジックがバージョン分岐している。新しいオブジェクトタイプを追加する際は、この分岐構造を維持する必要がある。
  • verbose モード時の列追加は、translate_columns 配列の定義と整合させる必要がある。
