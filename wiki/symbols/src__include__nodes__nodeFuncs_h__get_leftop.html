get_leftop

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/nodes/nodeFuncs.h:84

目的

演算子式（OpExpr）ノードから、その左辺のオペランド（Node）を抽出する。

シグネチャまたは形状

Node *get_leftop(const void *clause)

入力と出力

  • **入力**: clause - OpExpr 構造体へのポインタ（void * として受け取る）。
  • **出力**: 演算子の左辺ノードへのポインタ。引数が空のリストを持つ場合は NULL を返す。

前提条件と不変条件

  • clause は OpExpr 構造体としてキャスト可能である必要がある。
  • OpExpr 構造体の args フィールドが NIL でない場合、リストの先頭要素が左辺オペランドであると仮定する。

アルゴリズムまたは振る舞い

1. 入力された clause を OpExpr * にキャストする。 2. expr->args が NIL かどうかを判定する。 3. NIL でない場合、linitial() マクロを使用してリストの最初の要素を Node * として返す。 4. NIL の場合は NULL を返す。

関連データ構造

  • OpExpr: 演算子式を表すノード構造体。
  • Node: PostgreSQLのノードツリーの基底構造体。

失敗モードと境界条件

  • clause が OpExpr 以外の型である場合、キャスト後のアクセスにより未定義の動作を引き起こす可能性がある。
  • args リストが空の場合、安全に NULL を返す。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、頻繁に呼び出されることを想定している。
  • OpExpr の構造に依存しているため、primnodes.h での OpExpr 定義変更時には本関数のロジックも確認が必要である。
