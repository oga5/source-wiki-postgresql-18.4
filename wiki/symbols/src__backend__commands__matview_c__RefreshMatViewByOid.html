RefreshMatViewByOid

定義

  • 種別: function
  • 定義箇所: src/backend/commands/matview.c:164

目的

マテリアライズド・ビュー（MV）のデータ再構築および更新処理を統括する。

シグネチャまたは形状

ObjectAddress RefreshMatViewByOid(Oid matviewOid, bool is_create, bool skipData,
                                  bool concurrent, const char *queryString,
                                  QueryCompletion *qc)

入力と出力

  • **入力**:
  • matviewOid: 対象MVのOID。
  • is_create: 作成時か否かのフラグ。
  • skipData: データを生成せずに定義のみ更新するか否か。
  • concurrent: 同時実行更新（CONCURRENTLY）を行うか否か。
  • queryString: 実行元のクエリ文字列。
  • qc: クエリ完了状態を格納する構造体。
  • **出力**:
  • ObjectAddress: 更新されたMVのオブジェクトアドレス。

前提条件と不変条件

  • 対象リレーションが RELKIND_MATVIEW であること。
  • CONCURRENTLY 指定時は、MVに「WHERE句のない一意なインデックス」が存在すること。
  • CONCURRENTLY と WITH NO DATA は同時に指定できない。
  • MVには単一の SELECT INSTEAD OF ルールが定義されていること。

アルゴリズムまたは振る舞い

1. **権限昇格**: MV所有者のユーザーIDおよびセキュリティコンテキストへ一時的に切り替える。 2. **検証**: MVのメタデータ、ルール、およびインデックス構成が更新要件を満たしているか確認する。 3. **競合チェック**: CheckTableNotInUse により、現在のトランザクション内で対象テーブルが使用中ではないことを確認する。 4. **状態設定**: SetMatViewPopulatedState でMVの有効状態を更新する。 5. **一時テーブル作成**: make_new_heap を呼び出し、データ生成用の一時テーブルを作成する。 6. **データ生成**: skipData が偽の場合、refresh_matview_datafill を介してクエリ結果を一時テーブルへ流し込む。 7. **更新適用**:

  • concurrent が真の場合: refresh_by_match_merge を呼び出し、差分更新を実行する。

呼び出し元と呼び出し先

関連データ構造

  • Relation: MVのメタデータ保持に使用。
  • Query: MV定義時のクエリ情報。
  • DestReceiver: 一時テーブルへのデータ挿入用インターフェース。

失敗モードと境界条件

  • MVではないリレーションが指定された場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラー。
  • CONCURRENTLY 指定時に適切な一意インデックスがない場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラー。
  • ルール定義が不正な場合、elog(ERROR) により中断。

保守担当者向けメモ

  • concurrent モードでの更新は refresh_by_match_merge に委譲されており、この関数内では PG_TRY ブロックによる例外安全性が確保されている。
  • SetUserIdAndSecContext を使用しているため、呼び出し終了後のコンテキスト復元が確実に行われるよう注意が必要。
