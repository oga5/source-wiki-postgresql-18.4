RunObjectTruncateHook

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaccess.c:75

目的

データベースオブジェクトの切り詰め（TRUNCATE）操作が発生した際に、登録された外部フック関数を呼び出し、拡張機能や監査プラグインにイベントを通知します。

シグネチャまたは形状

void RunObjectTruncateHook(Oid objectId)

入力と出力

  • **入力**: objectId (Oid) - 切り詰め対象となるリレーションのオブジェクト識別子。
  • **出力**: なし（void）。

前提条件と不変条件

  • グローバル変数 object_access_hook が非NULLである必要があります。本関数内で Assert(object_access_hook != NULL) により検証されます。

アルゴリズムまたは振る舞い

1. object_access_hook が設定されていることをアサートで確認します。 2. 登録されているフック関数を以下の引数で呼び出します。

  • イベントタイプ: OAT_TRUNCATE
  • クラスID: RelationRelationId
  • オブジェクトID: objectId
  • サブID: 0
  • 追加引数: NULL

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の明示的な呼び出し元は提供されていません。
  • **呼び出し先**: グローバル関数ポインタ object_access_hook が指す任意のプラグイン関数。

関連データ構造

  • OAT_TRUNCATE: catalog/objectaccess.h で定義されるイベントタイプ定数。
  • RelationRelationId: catalog/pg_class.h で定義されるリレーションカタログのOID。

失敗モードと境界条件

  • object_access_hook が NULL の状態で呼び出された場合、アサートが有効なビルドではプロセスが異常終了します。

保守担当者向けメモ

  • 本関数はフックの実行口です。実際の処理はロードされた共有ライブラリ内のフック関数に依存します。
  • 名前ベースの識別が必要な場合は、対となる RunObjectTruncateHookStr の利用を検討してください。
