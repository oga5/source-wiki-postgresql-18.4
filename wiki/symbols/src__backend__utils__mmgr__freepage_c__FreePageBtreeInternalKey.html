FreePageBtreeInternalKey

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/mmgr/freepage.c:86

目的

メモリ管理における空き領域検索用の B-tree において、内部ノード（非リーフノード）のキーとして機能します。子ノードへのポインタと、その子ノードが管理する範囲の下限値を保持します。

シグネチャまたは形状

typedef struct FreePageBtreeInternalKey
{
    Size        first_page;        /* 子ページ内のキーの下限値 */
    RelptrFreePageBtree child;     /* 子ノードへの相対ポインタ */
} FreePageBtreeInternalKey;

入力と出力

本構造体はデータ保持を目的としており、関数としての入力・出力はありません。メモリ操作の対象として、B-tree のノード配列内で使用されます。

前提条件と不変条件

  • first_page は、child が指すサブツリー内の最小のページ番号を表します。
  • child は RelptrFreePageBtree 型であり、共有メモリ環境下で有効な相対ポインタとして解決される必要があります。

アルゴリズムまたは振る舞い

B-tree の内部ノードにおいて、キーの検索および挿入・削除操作時に、この構造体の配列が走査されます。特定のページ番号を含むサブツリーを特定するために、first_page の値が比較に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mmgr/freepage.c 内の B-tree 操作関数（ノードの挿入、削除、分割、検索処理など）が、この構造体の配列を直接操作します。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • FreePageBtree: B-tree ノード全体の構造体。
  • RelptrFreePageBtree: 子ノードへの参照に使用される相対ポインタ型。
  • FreePageBtreeLeafKey: リーフノードで使用される対となるキー構造体。

失敗モードと境界条件

  • **メモリ不足**: B-tree のノード操作時に、ノード内の配列サイズ（FPM_ITEMS_PER_INTERNAL_PAGE）を超えてキーを追加しようとした場合、ノードの分割が必要です。
  • **相対ポインタの無効化**: child が指す先が正しく初期化されていない場合、メモリ破壊を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は sizeof(FreePageBtreeInternalKey) を用いてメモリレイアウトが計算されています。構造体のメンバを変更する場合は、関連する memcpy やポインタ演算のオフセット計算に影響がないか確認してください。
  • FPM_ITEMS_PER_INTERNAL_PAGE に基づく配列操作が freepage.c 内の複数箇所（735行目、760行目、907行目など）で行われているため、変更時はこれらの箇所をすべて精査してください。
