postmaster_child_launch

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/launch_backend.c:228

目的

PostgreSQLのPostmasterプロセスから、バックエンドプロセスや補助プロセスを起動するための抽象化レイヤーを提供します。

シグネチャまたは形状

pid_t postmaster_child_launch(BackendType child_type, int child_slot, void *startup_data, size_t startup_data_len, ClientSocket *client_sock)

入力と出力

  • **入力**:
  • child_type: 起動するプロセスの種類（BackendType）。
  • child_slot: プロセス管理用のスロット番号。
  • startup_data: プロセス起動に必要なデータ構造へのポインタ。
  • startup_data_len: startup_data のサイズ。
  • client_sock: クライアント接続用ソケット（存在する場合）。
  • **出力**:
  • 起動した子プロセスのPID（pid_t）。

前提条件と不変条件

  • IsPostmasterEnvironment が真であり、かつ IsUnderPostmaster が偽であること（Postmasterプロセス自身から呼び出される必要がある）。

アルゴリズムまたは振る舞い

1. **ログ記録**: 外部接続バックエンドの場合、startup_data に起動開始時刻を記録します。 2. **プロセス生成**:

  • EXEC_BACKEND 定義時: internal_forkexec を呼び出し、プラットフォーム固有の手段でプロセスを生成します。
  • それ以外: fork_process() を実行します。

    • 3. **子プロセス側の処理（fork 後）**:

  • 外部接続バックエンドの場合、接続タイミング情報を記録します。
  • ClosePostmasterPorts を呼び出し、不要なソケットを閉じます。
  • InitPostmasterChild を呼び出し、Postmasterからの分離を行います。
  • child_process_kinds の設定に基づき、共有メモリ（DSMおよびPG共有メモリ）のデタッチを行います。
  • TopMemoryContext へ切り替え、MyPMChildSlot と MyClientSocket を設定します。
  • 対応する main_fn を実行し、プロセスを移行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/postmaster.c
  • src/backend/postmaster/syslogger.c
  • **呼び出し先**:
  • internal_forkexec (EXEC_BACKEND時)
  • fork_process
  • ClosePostmasterPorts
  • InitPostmasterChild
  • dsm_detach_all / PGSharedMemoryDetach
  • child_process_kinds[child_type].main_fn

関連データ構造

  • BackendType: プロセスの種類を識別する列挙型。
  • BackendStartupData: 起動時に引き継がれるデータ構造。
  • child_process_kinds: プロセスタイプごとのメイン関数や共有メモリ設定を保持するテーブル。

失敗モードと境界条件

  • EXEC_BACKEND モードでは、ハンドルやソケットの複製失敗がプロセス起動失敗の原因となる場合があります。
  • main_fn は戻り値を持たない（pg_unreachable()）設計であり、正常終了時はプロセス自体が終了します。

保守担当者向けメモ

  • EXEC_BACKEND モードと通常モードで起動パスが分岐するため、変更時は両方のパスで環境の引き継ぎが正しく行われるか確認してください。
  • startup_data は PostmasterContext に割り当てられているため、子プロセス側で即座に解放してはなりません。
