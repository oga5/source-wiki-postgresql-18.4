GinScanEntry

定義

  • 種別: typedef (ポインタ型)
  • 定義箇所: src/include/access/gin_private.h:267
  • 実体: struct GinScanEntryData へのポインタ

目的

GIN インデックススキャンにおいて、クエリ条件を満たす個々のエントリの状態を管理・保持するために使用されます。

シグネチャまたは形状

typedef struct GinScanEntryData *GinScanEntry;

入力と出力

  • 入力: GinScanKeyData 内の requiredEntries や additionalEntries 配列として保持されます。
  • 出力: ginget.c 内の各関数において、インデックスツリーの走査状態や取得済みアイテムのリストを保持するコンテキストとして機能します。

前提条件と不変条件

  • GinScanEntry は GinScanKeyData の一部として管理される必要があります。
  • 走査対象のインデックススキャンにおいて、有効な GinState および Snapshot が提供されていることが前提となります。

アルゴリズムまたは振る舞い

  • ginget.c において、startScanEntry により初期化され、entryGetItem や entryLoadMoreItems を通じて、インデックスページから一致する TID（タプル識別子）を順次取得します。
  • ポスティングツリーの走査（scanPostingTree）においても、現在の走査位置やバッファ状態を保持するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: 主に src/backend/access/gin/ginget.c 内の関数群（scanPostingTree, startScanEntry, entryLoadMoreItems, entryGetItem など）。
  • **呼び出し先**: 構造体内部のフィールドを介して、GinState やバッファ管理関数と連携します。

関連データ構造

  • GinScanEntryData: 本ポインタが指す実体構造体。
  • GinScanKeyData: スキャン時のクエリ条件を保持し、本構造体を配列として管理する親構造体。
  • GinState: インデックス操作のコンテキスト。

失敗モードと境界条件

  • メモリ割り当て失敗: palloc を使用して配列を確保する際、メモリ不足により失敗する可能性があります。
  • 走査終了: entryLoadMoreItems 等において、これ以上の一致アイテムが存在しない場合の終了条件を適切に処理する必要があります。

保守担当者向けメモ

  • 本シンボルは GinScanEntryData へのポインタであるため、実体の定義（GinScanEntryData）が gin_private.h 内のどこで定義されているかを確認し、フィールドの変更時には注意してください。
  • ginget.c でのメモリ管理（palloc）と密接に関連しているため、スキャン終了時のメモリ解放漏れに注意してください。
