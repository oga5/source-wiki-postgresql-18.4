logfile_open

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/syslogger.c:1218

目的

PostgreSQL のログファイルを安全なパーミッションで開き、バッファリング設定および Windows 環境での改行コード調整を適用してファイルポインタを返す。

シグネチャまたは形状

static FILE *logfile_open(const char *filename, const char *mode, bool allow_errors)

入力と出力

  • **入力**:
  • filename: 開く対象のファイルパス。
  • mode: fopen に渡すモード文字列（"a" や "w"）。
  • allow_errors: true の場合、オープン失敗時に LOG レベルで報告し NULL を返す。false の場合は FATAL エラーとしてプロセスを終了させる。
  • **出力**: 成功時は開かれた FILE*、失敗時は NULL（allow_errors が true の場合）。

前提条件と不変条件

  • Log_file_mode グローバル変数に基づき、ファイル作成時のパーミッションが制御される。
  • S_IWUSR（所有者の書き込み権限）は、設定に関わらず常に付与される。

アルゴリズムまたは振る舞い

1. umask を一時的に変更し、Log_file_mode に基づくパーミッションでファイルを開く。 2. fopen 実行後、直ちに元の umask を復元する。 3. 成功した場合:

  • setvbuf を使用して行バッファリング（PG_IOLBF）を有効にする。
  • Windows 環境では _setmode を呼び出し、ファイル記述子をテキストモード（_O_TEXT）に設定して CRLF 改行を適用する。

    • 4. 失敗した場合:

  • allow_errors の値に応じて ereport を発行し、エラーを記録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/syslogger.c 内の以下の箇所から呼び出される。
  • syslogFile、csvlogFile、jsonlogFile の初期化時（mode="a", allow_errors=false）。
  • ログローテーション処理時（mode="w" または "a", allow_errors=true）。
  • **呼び出し先**: fopen, umask, setvbuf, ereport, _setmode（Windows のみ）。

関連データ構造

  • FILE: 標準 C ライブラリのファイルストリーム。

失敗モードと境界条件

  • ファイルオープンに失敗した場合、allow_errors が false であれば FATAL エラーとなり、PostgreSQL の Syslogger プロセスが停止する。
  • allow_errors が true の場合は LOG レベルの報告に留まり、呼び出し元へ NULL が返される。

保守担当者向けメモ

  • 本関数は syslogger.c 内でのみ使用される静的関数である。
  • Windows 特有の改行コード処理が含まれているため、プラットフォーム依存の挙動を変更する際は注意が必要。
