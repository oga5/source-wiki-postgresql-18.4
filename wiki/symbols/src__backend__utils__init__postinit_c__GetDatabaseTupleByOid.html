GetDatabaseTupleByOid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/postinit.c:147

目的

指定されたデータベースOID（dboid）に対応する pg_database システムカタログのタプルを取得する。

シグネチャまたは形状

static HeapTuple GetDatabaseTupleByOid(Oid dboid)

入力と出力

  • **入力**: dboid (Oid) - 検索対象のデータベースOID。
  • **出力**: HeapTuple - 見つかった場合は pg_database タプルのコピー。見つからない場合は NULL（HeapTupleIsValid が偽となるタプル）。

前提条件と不変条件

  • pg_database リレーションに対する AccessShareLock を取得して読み取りを行う。
  • 共有リレーションキャッシュ（criticalSharedRelcachesBuilt）の状態に応じて、インデックススキャンまたはヒープスキャンが選択される。
  • 返却されるタプルはメモリ上にコピーされているため、呼び出し元で適切に解放する必要がある。

アルゴリズムまたは振る舞い

1. pg_database の oid カラムを検索キーとして初期化する。 2. table_open で pg_database を開く。 3. systable_beginscan を使用し、DatabaseOidIndexId インデックス（または criticalSharedRelcachesBuilt が偽の場合はヒープ）を用いてスキャンを開始する。 4. systable_getnext でタプルを取得する。 5. タプルが存在する場合、heap_copytuple を呼び出してメモリコピーを作成する。 6. systable_endscan および table_close を実行してリソースを解放する。 7. 取得したタプル（または無効なタプル）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/init/postinit.c:1070 での呼び出しが確認されている。
  • **呼び出し先**: ScanKeyInit, table_open, systable_beginscan, systable_getnext, heap_copytuple, systable_endscan, table_close。

関連データ構造

  • HeapTuple: データベースの行データ。
  • Relation: pg_database システムカタログへのアクセス用。
  • SysScanDesc: システムカタログスキャンの状態管理。
  • ScanKeyData: 検索条件（OID）の保持。

失敗モードと境界条件

  • 指定したOIDが存在しない場合、systable_getnext は無効なタプルを返し、関数はそのまま無効なタプルを返す。
  • criticalSharedRelcachesBuilt が偽の状態で共有リレーションキャッシュが構築されていない場合、インデックスを使用せずヒープスキャンにフォールバックする。

保守担当者向けメモ

  • 本関数は pg_database のタプルをコピーして返すため、呼び出し元でのメモリリークに注意すること。
  • criticalSharedRelcachesBuilt フラグの依存関係により、システム起動時の初期化フェーズにおける呼び出し順序に制約がある。
