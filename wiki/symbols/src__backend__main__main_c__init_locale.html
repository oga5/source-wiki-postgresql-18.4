init_locale

定義

  • 種別: function
  • 定義箇所: src/backend/main/main.c:364

目的

PostgreSQL プロセスのロケール設定を初期化し、指定されたロケールが適用できない場合にフォールバックとして "C" ロケールを適用する。

シグネチャまたは形状

static void init_locale(const char *categoryname, int category, const char *locale)

入力と出力

  • categoryname: ロケールカテゴリの名称（ログ出力用）。
  • category: 設定対象のロケールカテゴリ（LC_COLLATE, LC_CTYPE 等）。
  • locale: 設定するロケール名。
  • 出力: なし（成功時はロケールが設定され、失敗時は elog(FATAL, ...) によりプロセスが終了する）。

前提条件と不変条件

  • pg_perm_setlocale が利用可能であること。
  • 指定されたロケール、または "C" ロケールのいずれかがシステムでサポートされている必要がある。

アルゴリズムまたは振る舞い

1. pg_perm_setlocale を呼び出し、指定された category に locale を適用する。 2. 適用に失敗した場合、再度 pg_perm_setlocale を呼び出し、同じ category に "C" ロケールを適用する。 3. "C" ロケールの適用にも失敗した場合、elog(FATAL) を呼び出し、エラーメッセージを出力してプロセスを異常終了させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/main/main.c 内の main 関数（LC_COLLATE, LC_CTYPE, LC_MESSAGES, LC_MONETARY, LC_NUMERIC, LC_TIME の各カテゴリに対して呼び出される）。
  • 呼び出し先: pg_perm_setlocale（ロケール設定用）、elog（エラー報告用）。

関連データ構造

  • なし（標準的なロケールカテゴリ定数を使用）。

失敗モードと境界条件

  • 指定されたロケールと "C" ロケールの両方が適用できない場合、FATAL エラーとなりサーバーは起動しない。

保守担当者向けメモ

  • 本関数は main 関数の初期化フェーズで呼び出されるため、ここで発生する FATAL エラーはサーバー起動の失敗を意味する。
  • ロケール設定の永続化には pg_perm_setlocale が使用されている点に注意すること。
