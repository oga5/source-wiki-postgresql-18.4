expand_inherited_rtentry

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/inherit.c:85

目的

クエリ内の継承またはパーティション分割されたテーブル（RangeTblEntry）を、個別の物理的な子テーブルまたはパーティションに展開し、プランナが処理可能な形式へ変換する。

シグネチャまたは形状

void expand_inherited_rtentry(PlannerInfo *root, RelOptInfo *rel, RangeTblEntry *rte, Index rti)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel: 展開対象の RelOptInfo。
  • rte: 展開対象の RangeTblEntry。
  • rti: rte の範囲テーブルインデックス。
  • **出力**: なし（副作用として root 内の append_rel_list や processed_tlist 等を更新）。

前提条件と不変条件

  • rte->inh が真であること（呼び出し元で保証される）。
  • rte が RTE_SUBQUERY または RTE_RELATION であること。
  • 継承セットのメンバーは、親テーブルと同じロックモードでロックされていること。

アルゴリズムまたは振る舞い

1. **サブクエリの処理**: rte->rtekind == RTE_SUBQUERY の場合、expand_appendrel_subquery を呼び出して終了する。 2. **行マークの準備**: 親テーブルが FOR UPDATE/SHARE 等でロックされている場合、PlanRowMark を更新し、子テーブル用のマーク生成に備える。 3. **展開処理**:

  • **パーティションテーブル**: expand_partitioned_rtentry を呼び出し、再帰的な展開とプルーニングを行う。
  • **継承テーブル**: find_all_inheritors で全子テーブルを取得し、expand_single_inheritance_child を用いて各子テーブルの RTE および AppendRelInfo を生成する。

    • 4. **ターゲットリストの更新**: 行マークが必要な場合、必要に応じて ctid 等のジャンク列を root->processed_tlist に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/initsplan.c
  • src/backend/optimizer/util/inherit.c (再帰的呼び出し)
  • **呼び出し先**:
  • expand_appendrel_subquery
  • expand_partitioned_rtentry
  • expand_single_inheritance_child
  • find_all_inheritors
  • build_simple_rel
  • table_open / table_close

関連データ構造

  • PlannerInfo: プランニングの全体状態。
  • RangeTblEntry: 範囲テーブルエントリ。
  • RelOptInfo: リレーションの最適化情報。
  • PlanRowMark: 行ロック管理用構造体。

失敗モードと境界条件

  • **一時テーブル**: 他のバックエンドの一時テーブルは無視される。
  • **子テーブルなし**: find_all_inheritors は親自身を必ず含むため、子が存在しない場合でも親のみが処理される。

保守担当者向けメモ

  • securityQuals は親から子へ伝播されるが、子固有の制約は apply_child_basequals で処理される点に注意。
  • FOR UPDATE/SHARE の行マーク処理において、old_allMarkTypes を保存し、子テーブル追加後にターゲットリストを調整するロジックが含まれている。
