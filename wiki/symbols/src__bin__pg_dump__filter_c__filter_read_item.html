filter_read_item

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/filter.c:391

目的

フィルタファイルから1行ずつ読み込み、コマンド（include/exclude）、オブジェクト型、および対象パターンを解析して抽出する。

シグネチャまたは形状

bool filter_read_item(FilterStateData *fstate,
                      char **objname,
                      FilterCommandType *comtype,
                      FilterObjectType *objtype)

入力と出力

  • **入力**:
  • fstate: フィルタ処理の状態を保持する FilterStateData 構造体へのポインタ。
  • **出力**:
  • objname: 解析されたオブジェクト名（文字列）へのポインタ。
  • comtype: 抽出されたコマンド型（FILTER_COMMAND_TYPE_INCLUDE または FILTER_COMMAND_TYPE_EXCLUDE）。
  • objtype: 抽出されたオブジェクト型（FilterObjectType）。
  • **戻り値**: 項目が正常に読み込まれた場合は true、ファイルの終端に達した場合は false を返す。

前提条件と不変条件

  • fstate->fp が有効なファイルポインタとしてオープンされていること。
  • 構文エラーが発生した場合、fstate->exit_nicely コールバックを通じてプログラムが終了される。

アルゴリズムまたは振る舞い

1. pg_get_line_buf を使用してファイルから1行読み込む。 2. 空行または # で始まるコメント行はスキップする。 3. 行の先頭から filter_get_keyword を使用してコマンドとオブジェクト型を順次取得する。 4. get_object_type を用いてキーワードを列挙型にマッピングする。 5. read_pattern を呼び出し、残りの文字列をオブジェクトパターンとして解析・正規化する。 6. 読み込み失敗時や構文エラー時には、pg_log_filter_error でエラーを出力し、fstate->exit_nicely を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c
  • src/bin/pg_dump/pg_restore.c
  • src/bin/pg_dump/pg_dumpall.c
  • **呼び出し先**:
  • pg_get_line_buf
  • filter_get_keyword
  • get_object_type
  • read_pattern
  • pg_log_filter_error
  • initPQExpBuffer

関連データ構造

  • FilterStateData: フィルタ処理の現在の状態（ファイルポインタ、行番号、バッファ等）を保持。
  • FilterCommandType: include または exclude を識別する列挙型。
  • FilterObjectType: オブジェクトの種類を識別する列挙型。
  • PQExpBufferData: パターン文字列を構築するための動的バッファ。

失敗モードと境界条件

  • **ファイル読み込みエラー**: ferror が真の場合、エラーログを出力して終了する。
  • **構文エラー**: コマンドやオブジェクト型が不明な場合、またはキーワードが欠落している場合、エラーログを出力して終了する。
  • **EOF**: ファイルの終端に達した場合は false を返し、呼び出し元に通知する。

保守担当者向けメモ

  • 本関数は pg_get_line_buf を介して行単位で処理を行うため、複数行にまたがる引用符付きパターンの解析は read_pattern 内のロジックに依存している。
  • exit_nicely は外部から注入されるコールバックであるため、エラー時の終了挙動を調査する際は呼び出し元の設定を確認すること。
