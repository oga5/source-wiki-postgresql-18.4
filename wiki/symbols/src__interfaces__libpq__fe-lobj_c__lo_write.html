lo_write

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-lobj.c:294

目的

データベース内のラージオブジェクトに対して、指定されたバッファからデータを書き込みます。

シグネチャまたは形状

int lo_write(PGconn *conn, int fd, const char *buf, size_t len);

入力と出力

  • **入力**:
  • conn: PostgreSQL接続オブジェクト。
  • fd: 書き込み対象のラージオブジェクト記述子。
  • buf: 書き込むデータを含むバッファ。
  • len: 書き込むバイト数。
  • **出力**:
  • 成功時: 書き込まれたバイト数を返します。
  • 失敗時: -1 を返します。

前提条件と不変条件

  • lo_initialize(conn) が成功している必要があります。
  • len は INT_MAX 以下である必要があります（これを超える場合はエラーとなります）。
  • 内部的に PQfn（Fastpath）を使用するため、現在のトランザクション内で実行される必要があります。

アルゴリズムまたは振る舞い

1. lo_initialize を呼び出し、サーバー側関数のOIDを解決します。 2. len が INT_MAX を超えていないか検証し、超える場合はエラーを報告して終了します。 3. PQArgBlock を使用して、ファイル記述子とバッファポインタを引数として設定します。 4. PQfn を介してサーバー側のラージオブジェクト書き込み関数を実行します。 5. サーバーからの応答（PGresult）を確認し、成功した場合は書き込みバイト数を返し、失敗した場合は -1 を返します。いずれの場合も PQclear で結果を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_archiver.c、src/test/examples/testlo.c、および src/interfaces/libpq/fe-lobj.c 内の他の関数から呼び出されます。
  • **呼び出し先**: lo_initialize、libpq_append_conn_error、PQfn、PQresultStatus、PQclear。

関連データ構造

  • PGconn: 接続状態およびラージオブジェクト関数OIDを保持。
  • PQArgBlock: PQfn に渡す引数構造体。

失敗モードと境界条件

  • len が INT_MAX を超える場合、エラーとなり -1 を返します。
  • サーバー側でエラーが発生した場合、PQresultStatus が PGRES_COMMAND_OK 以外となり、-1 を返します。
  • トランザクションがアボート状態にある場合、書き込みは失敗します。

保守担当者向けメモ

  • 本関数は size_t を引数に取りますが、内部的には int32 にキャストされるため、非常に大きなデータサイズを扱う際には注意が必要です。
  • サーバー側のラージオブジェクト関数は int32 の長さを期待するため、この制約は意図的なものです。
