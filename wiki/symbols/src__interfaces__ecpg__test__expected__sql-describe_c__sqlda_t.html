sqlda_t

定義

  • 定義箇所: src/interfaces/ecpg/test/expected/sql-describe.c:32
  • ソース定義: typedef struct sqlda_compat sqlda_t;

目的

ECPG（Embedded SQL for C）において、動的 SQL 文の実行結果やメタデータ（列情報など）を保持するための SQL 記述子領域（SQL Descriptor Area）として機能します。

シグネチャまたは形状

sqlda_compat 構造体の別名です。具体的な内部フィールド構成は、コンパイル時に選択される sqlda-compat.h または sqlda-native.h に依存します。

入力と出力

  • **入力**: ECPGdescribe() 関数により、SQL 文の実行結果メタデータがこの構造体へロードされます。
  • **出力**: 構造体内のフィールド（列数や列名など）をアプリケーション側で参照し、結果セットの解析に使用します。

前提条件と不変条件

  • 使用前にメモリの確保（malloc 等）と初期化（memset 等）が必要です。
  • ECPGdescribe() を呼び出す前に、対象となる SQL 文が PREPARE されている必要があります。

アルゴリズムまたは振る舞い

1. アプリケーションが sqlda_t 構造体をメモリ上に確保します。 2. ECPGdescribe() を呼び出し、SQL 文のメタデータを構造体に格納します。 3. 構造体内のフィールド（sqld フィールド等）を読み取り、列数や各列の属性を確認します。 4. 処理終了後、確保したメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/compat_informix-sqlda.c 等のテストコード内で、動的 SQL のメタデータ取得のために使用されています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。ECPGdescribe() 等の ECPG ライブラリ関数がこの構造体を操作します。

関連データ構造

  • sqlvar_t: SQLDA 内で個々の列情報を保持するために使用される構造体。
  • sqlda-compat.h / sqlda-native.h: 構造体の定義元ヘッダー。

失敗モードと境界条件

  • メモリ確保に失敗した場合、未定義の動作を引き起こす可能性があります。
  • ECPGdescribe() がエラーを返した場合、構造体内のデータは更新されないか、不整合な状態となります。sqlca.sqlcode によるエラーチェックが必須です。
