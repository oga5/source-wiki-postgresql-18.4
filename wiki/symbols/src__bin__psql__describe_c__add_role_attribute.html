add_role_attribute

定義

  • 種別: function
  • 定義箇所: src/bin/psql/describe.c:3846

目的

ロールの属性情報をカンマ区切りの文字列としてバッファに追記し、ロールの属性リストを整形するために使用されます。

シグネチャまたは形状

static void add_role_attribute(PQExpBuffer buf, const char *const str);

入力と出力

  • buf: 属性文字列を蓄積する PQExpBuffer へのポインタ。
  • str: 追加する属性名（例: "Superuser"）。
  • 出力: なし（buf の内容が更新される）。

前提条件と不変条件

  • buf は初期化済みであること。
  • str は有効な文字列ポインタであること。

アルゴリズムまたは振る舞い

1. buf->len が 0 より大きい場合（既に属性が追加されている場合）、区切り文字として ", " をバッファに追記します。 2. 指定された文字列 str をバッファに追記します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/describe.c 内のロール情報を表示する関数群（3782 行目から 3801 行目にかけて、"Superuser", "No inheritance", "Create role", "Create DB", "Cannot login", "Replication", "Bypass RLS" などの属性追加時に呼び出されます）。
  • 呼び出し先: appendPQExpBufferStr（libpq のユーティリティ関数）。

関連データ構造

  • PQExpBuffer: libpq で定義される動的文字列バッファ構造体。

失敗モードと境界条件

  • buf が NULL の場合、appendPQExpBufferStr の内部実装に依存した動作となります。
  • バッファのメモリ確保に失敗した場合、libpq の規定に従いプログラムが終了する可能性があります。

保守担当者向けメモ

  • 本関数は static であり、describe.c 内でのみ使用されます。
  • ロール属性の表示形式を変更する場合（例: カンマ以外の区切り文字にする場合）は、この関数を修正してください。
