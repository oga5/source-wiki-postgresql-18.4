PLy_spi_subtransaction_abort

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_spi.c:581

目的

SPI 操作中に発生した PostgreSQL エラーを捕捉し、サブトランザクションを安全にロールバックした上で、対応する Python 例外へ変換して送出する。

シグネチャまたは形状

void PLy_spi_subtransaction_abort(MemoryContext oldcontext, ResourceOwner oldowner)

入力と出力

  • **入力**:
  • oldcontext: エラー発生前のメモリコンテキスト。
  • oldowner: エラー発生前のリソースオーナー。
  • **出力**: なし（void）。副作用として Python 例外が設定される。

前提条件と不変条件

  • 本関数は、PostgreSQL のエラーハンドリングブロック（PG_CATCH）内で呼び出されることを想定している。
  • 呼び出し前に CopyErrorData() を介してエラー情報を保存し、FlushErrorState() でエラー状態をクリアする必要がある。

アルゴリズムまたは振る舞い

1. oldcontext へメモリコンテキストを切り替え、CopyErrorData() でエラー情報を取得し、FlushErrorState() を実行する。 2. RollbackAndReleaseCurrentSubTransaction() を呼び出し、現在のサブトランザクションを破棄する。 3. メモリコンテキストとリソースオーナーを、エラー発生前の状態（oldcontext, oldowner）に復元する。 4. PLy_spi_exceptions ハッシュテーブルからエラーコードに対応する Python 例外クラスを検索する。該当しない場合は PLy_exc_spi_error を使用する。 5. PLy_spi_exception_set() を呼び出し、Python インタプリタに対して例外を発生させる。 6. FreeErrorData() で保存していたエラー情報を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_spi.c 内の各 SPI 実行関数（PG_CATCH ブロック内）。
  • src/pl/plpython/plpy_cursorobject.c 内のカーソル操作関数（PG_CATCH ブロック内）。
  • **呼び出し先**:
  • CopyErrorData, FlushErrorState, RollbackAndReleaseCurrentSubTransaction, MemoryContextSwitchTo, hash_search, PLy_spi_exception_set, FreeErrorData

関連データ構造

  • ErrorData: PostgreSQL のエラー情報構造体。
  • PLyExceptionEntry: SQL エラーコードと Python 例外オブジェクトを紐付けるハッシュテーブルエントリ。
  • PLy_spi_exceptions: エラーコードと例外の対応を保持するハッシュテーブル。

失敗モードと境界条件

  • hash_search でエラーコードが見つからない場合、デフォルトの PLy_exc_spi_error が使用される。
  • メモリコンテキストやリソースオーナーの復元に失敗した場合、後続の処理でメモリリークやリソースの不整合が発生する可能性がある。

保守担当者向けメモ

  • 本関数は longjmp を伴う PostgreSQL のエラーハンドリングと Python の例外機構を橋渡しする重要な役割を持つ。
  • PG_CATCH ブロック内でのみ使用すること。それ以外の場所で呼び出すとトランザクション状態が破壊される。
