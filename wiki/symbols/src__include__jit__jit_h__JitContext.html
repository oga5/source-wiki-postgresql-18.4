JitContext

定義

  • 種別: struct
  • 定義箇所: src/include/jit/jit.h:57

目的

PostgreSQL の JIT コンパイル処理における実行状態と、それに付随する統計情報を保持するための共通データ構造です。プロバイダ非依存のインターフェースとして機能します。

シグネチャまたは形状

typedef struct JitContext
{
    int flags;              /* PGJIT_* ビットフラグ */
    JitInstrumentation instr; /* JIT 処理の計測データ */
} JitContext;

入力と出力

  • **入力**: 構造体生成時に flags を通じて JIT の動作（式コンパイル、タプルデフォーム等）が指定されます。
  • **出力**: 実行過程で instr フィールドが更新され、コード生成や最適化にかかるコストが記録されます。

前提条件と不変条件

  • 本構造体は jit_release_context を通じて解放される必要があります。
  • instr フィールドは JitInstrumentation 型であり、計測データの一貫性が維持されます。

アルゴリズムまたは振る舞い

  • JIT プロバイダ（LLVM 等）は、この構造体を基底として拡張（例: LLVMJitContext）し、具体的なコンパイル状態を管理します。
  • 実行エンジンは、このコンテキストを通じて JIT の有効化・無効化や統計情報の集計を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execGrouping.c や src/include/nodes/execnodes.h（es_jit フィールドとして保持）から参照されます。
  • **呼び出し先**: jit_release_context 関数が本構造体を引数として受け取り、リソース解放を行います。また、llvm_release_context 等のプロバイダ固有の実装が本構造体を操作します。

関連データ構造

  • JitInstrumentation: 統計情報保持用。
  • LLVMJitContext: JitContext を内包する LLVM 実装固有の構造体。
  • ExprState: 実行ノード内で本コンテキストを保持する場合があります。

失敗モードと境界条件

  • JIT が無効な場合やプロバイダのロードに失敗した場合、コンテキストの生成や操作が適切にハンドリングされる必要があります。
  • 構造体の解放漏れはリソースリークの原因となります。

保守担当者向けメモ

  • 本構造体はプロバイダ非依存であるため、LLVM 特有のフィールドを追加しないでください。LLVM 固有のデータが必要な場合は、llvmjit.h で定義される派生構造体を使用してください。
