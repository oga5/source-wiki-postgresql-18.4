pg_encoding

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/encode.c:33

目的

バイナリデータとテキスト表現間の相互変換アルゴリズム（HEX、Base64、Escape等）を抽象化し、統一的なインターフェースで管理するための関数ポインタテーブルを提供します。

シグネチャまたは形状

struct pg_encoding
{
    uint64 (*encode_len) (const char *data, size_t dlen);
    uint64 (*decode_len) (const char *data, size_t dlen);
    uint64 (*encode) (const char *data, size_t dlen, char *res);
    uint64 (*decode) (const char *data, size_t dlen, char *res);
};

入力と出力

  • **入力**: 変換対象のデータポインタ（data）およびその長さ（dlen）。
  • **出力**:
  • _len 系関数: 変換後のデータに必要なバッファサイズを uint64 で返します。
  • encode/decode 関数: 変換後のデータを res バッファに書き込み、実際に書き込んだバイト数を uint64 で返します。

前提条件と不変条件

  • encode_len および decode_len は、変換実行前に必要なメモリサイズを算出するために使用されます。
  • 変換後のサイズが MaxAllocSize を超える場合、安全なエラーハンドリングを行う必要があります。

アルゴリズムまたは振る舞い

1. pg_find_encoding を介して特定のエンコーディング方式に対応する本構造体を取得します。 2. encode_len または decode_len で必要なバッファサイズを計算します。 3. palloc でメモリを確保します。 4. encode または decode を呼び出し、データを変換してバッファへ書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/encode.c 内の binary_encode および binary_decode 関数が、この構造体の関数ポインタを介して変換処理を実行します。
  • **呼び出し先**: 各エンコーディング方式（hex, base64等）に対応する具体的な実装関数が代入されます。

関連データ構造

  • enclist: サポートされているエンコーディング方式を保持する静的配列。

失敗モードと境界条件

  • decode 処理において入力データが不正な場合、ereport を発行してトランザクションを中断する設計となっています。
  • 32bit環境等でのメモリ割り当てオーバーフローを防ぐため、サイズ計算結果の厳密なチェックが求められます。

保守担当者向けメモ

  • 新しいエンコーディング方式を追加する場合は、enclist 配列に新しいエントリを追加し、本構造体の各関数ポインタに対応する実装を定義してください。
