filter_get_keyword

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/filter.c:178

目的

フィルタ設定ファイルの文字列バッファから、次のキーワード（空白で区切られた連続する非空白文字の塊）を抽出します。

シグネチャまたは形状

static const char *filter_get_keyword(const char **line, int *size)

入力と出力

  • **入力**:
  • line: 解析対象の文字列へのポインタ。関数終了後、読み取ったキーワードの直後の位置まで更新されます。
  • size: キーワードの長さを格納するための出力用整数ポインタ。
  • **出力**:
  • 戻り値: 抽出されたキーワードの開始位置を指す const char *。キーワードが見つからない場合は NULL を返します。
  • size: 抽出されたキーワードの長さ（バイト数）。見つからない場合は 0 が設定されます。

前提条件と不変条件

  • line が指すポインタおよび line 自体は NULL であってはなりません（Assert により検証）。
  • 抽出されるキーワードは、空白文字（isspace で判定）を含まない連続した文字列です。

アルゴリズムまたは振る舞い

1. size を 0 に初期化します。 2. line が指す位置から、先頭の空白文字をスキップします。 3. 空白以外の文字に到達した場合、そこをキーワードの開始位置とします。 4. 次の空白文字または文字列終端（\0）に到達するまでポインタを進め、その間の長さを計算して size に格納します。 5. line をキーワードの直後の位置に更新し、キーワードの開始位置を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/filter.c 内の行解析処理（420行目、439行目など）。
  • **呼び出し先**: 標準ライブラリの isspace を使用。

関連データ構造

  • なし（文字列ポインタと整数による直接的な操作のみ）。

失敗モードと境界条件

  • 文字列が空白のみ、または空文字列の場合: size は 0 となり、戻り値は NULL となります。
  • 文字列の末尾にキーワードがある場合: 文字列終端までを正しくキーワードとして抽出します。

保守担当者向けメモ

  • 本関数は filter_read_item 等のパーサーから呼び出され、フィルタファイルのトークン化を補助します。
  • isspace の挙動は現在のロケールに依存するため、フィルタファイルのエンコーディングと実行環境のロケール設定に注意してください。
