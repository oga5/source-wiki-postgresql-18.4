quoteAclUserName

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/dumputils.c:584

目的

データベースのユーザー名やロール名を SQL 文中で安全に使用できるよう、必要に応じて識別子をダブルクォートで囲み、内部のダブルクォートをエスケープする。

シグネチャまたは形状

void quoteAclUserName(PQExpBuffer output, const char *input)

入力と出力

  • output (入出力): 処理結果を追記する PQExpBuffer。
  • input (入力): クォート対象のユーザー名文字列。

前提条件と不変条件

  • input が英数字およびアンダースコアのみで構成されている場合は、そのまま出力される。
  • それ以外の文字が含まれる場合、全体がダブルクォートで囲まれる。
  • input 内に含まれるダブルクォートは、"" にエスケープされる。
  • 本関数は putid() と同様の識別子判定ロジックに従う必要がある。

アルゴリズムまたは振る舞い

1. input を走査し、英数字およびアンダースコア以外の文字が含まれるか判定する。 2. 特殊文字が含まれる場合、safe フラグを false に設定する。 3. safe が false の場合、output に開始の " を追加する。 4. input を再度走査し、各文字を output に追加する。その際、" が出現した場合は "" として出力する。 5. safe が false の場合、output に終了の " を追加する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/pg_dump.c 内の複数箇所で、ロール名を出力する際に利用されている。
  • 呼び出し先: appendPQExpBufferChar (libpq のバッファ操作関数)。

関連データ構造

  • PQExpBuffer: PostgreSQL の動的文字列バッファ。

失敗モードと境界条件

  • 空文字列が渡された場合、そのまま（あるいは空のクォートなしで）処理される。
  • input にダブルクォートのみが含まれる場合、"" にエスケープされ、全体が " で囲まれる。

保守担当者向けメモ

  • 本関数は dequoteAclUserName と対になる処理を行う。
  • 識別子の安全性の判定ロジック（isalnum および _ のチェック）を変更する場合は、バックエンド側の putid() との整合性を維持すること。
