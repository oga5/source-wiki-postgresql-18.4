CatalogTupleInsertWithInfo

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/indexing.c:255

目的

システムカタログへのタプル挿入において、呼び出し元が提供するインデックス状態（CatalogIndexState）を再利用することで、インデックスオープン処理のオーバーヘッドを削減しつつ、ヒープ挿入とインデックス更新を統合的に実行する。

シグネチャまたは形状

void CatalogTupleInsertWithInfo(Relation heapRel, HeapTuple tup, CatalogIndexState indstate)

入力と出力

  • **入力**:
  • heapRel: 挿入対象のシステムカタログリレーション記述子。
  • tup: 挿入するヒープタプル。
  • indstate: 既にオープンされているインデックス情報を含む CatalogIndexState。
  • **出力**: なし（void）。

前提条件と不変条件

  • indstate は、対象リレーションに対して事前に CatalogOpenIndexes を呼び出して取得された有効な状態であること。
  • システムカタログ特有の制約（NOT NULL 等）が適用される。

アルゴリズムまたは振る舞い

1. CatalogTupleCheckConstraints を呼び出し、タプルがカタログの制約を満たしているか検証する。 2. simple_heap_insert を呼び出し、ヒープリレーションへタプルを挿入する。 3. CatalogIndexInsert を呼び出し、提供された indstate を使用して関連するすべてのインデックスを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c
  • src/backend/commands/analyze.c
  • src/backend/storage/large_object/inv_api.c
  • **呼び出し先**:
  • CatalogTupleCheckConstraints
  • simple_heap_insert
  • CatalogIndexInsert

関連データ構造

  • Relation: ヒープリレーションの記述子。
  • HeapTuple: 挿入されるタプルデータ。
  • CatalogIndexState: インデックス更新に必要な情報を保持する不透明な構造体（ResultRelInfo をラップ）。

失敗モードと境界条件

  • カタログ制約違反が発生した場合、CatalogTupleCheckConstraints によりエラーがスローされる。
  • インデックス更新時にカタログの整合性が保てない場合、CatalogIndexInsert 経由でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、複数行の挿入を行う際に CatalogOpenIndexes の呼び出し回数を最小化するために使用する。
  • システムカタログでは式インデックスや部分インデックスはサポートされていないため、これらを使用するような拡張は行わないこと。
