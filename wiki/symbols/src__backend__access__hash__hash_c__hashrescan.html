hashrescan

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash.c:403

目的

ハッシュインデックススキャンの状態をリセットし、新しいスキャンキーを使用してスキャンを再開可能にする。

シグネチャまたは形状

void hashrescan(IndexScanDesc scan, ScanKey scankey, int nscankeys,
                ScanKey orderbys, int norderbys)

入力と出力

  • **入力**:
  • scan: 現在のスキャン状態を保持する IndexScanDesc 構造体。
  • scankey: 新しいスキャンキー（NULL の場合、既存のキーが維持される）。
  • nscankeys: スキャンキーの数。
  • orderbys, norderbys: ハッシュインデックスでは使用されない。
  • **出力**: なし（副作用として scan 構造体内部の状態を更新）。

前提条件と不変条件

  • scan->opaque は HashScanOpaque 型としてキャスト可能であること。
  • スキャン終了時に _hash_dropscanbuf を呼び出し、保持しているバッファを解放する。

アルゴリズムまたは振る舞い

1. 現在のスキャン位置（so->currPos）が有効な場合、_hash_kill_items を呼び出して削除対象（killed）としてマークされたアイテムを処理する。 2. _hash_dropscanbuf を呼び出し、現在保持しているバッファを解放する。 3. HashScanPosInvalidate を使用してスキャン位置を無効化し、次回の hashgettuple 呼び出し時に _hash_first が再実行されるようにする。 4. 新しい scankey が提供されている場合、scan->keyData を更新する。 5. スキャン状態のフラグ hashso_buc_populated および hashso_buc_split を false にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のインデックス AM インターフェース経由で呼び出される（amroutine->amrescan に登録）。
  • **呼び出し先**:
  • _hash_kill_items
  • _hash_dropscanbuf
  • HashScanPosIsValid / HashScanPosInvalidate（マクロ）

関連データ構造

  • IndexScanDesc: スキャン全体のメタデータ。
  • HashScanOpaque: ハッシュインデックス固有のスキャン状態（so->currPos, so->hashso_buc_populated 等）。
  • ScanKeyData: スキャン条件。

失敗モードと境界条件

  • scankey が NULL の場合、キーの更新はスキップされる。
  • numKilled > 0 の場合のみ _hash_kill_items が実行される。

保守担当者向けメモ

  • 本関数は IndexAmRoutine の amrescan コールバックとして機能する。
  • スキャン位置の無効化は、次回のタプル取得時にバケットの再走査を強制するための重要なステップである。
