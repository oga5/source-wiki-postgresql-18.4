btoptions

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtutils.c:3681

目的

B-tree インデックスのオプション（fillfactor、vacuum_cleanup_index_scale_factor、deduplicate_items）を解析し、BTOptions 構造体に基づいた bytea 形式のオプションデータを生成します。

シグネチャまたは形状

bytea *btoptions(Datum reloptions, bool validate)

入力と出力

  • **入力**:
  • reloptions: 解析対象のオプションデータ（Datum）。
  • validate: オプションの妥当性を検証するかどうかを示すフラグ。
  • **出力**:
  • 解析結果を格納した bytea 型のポインタ。

前提条件と不変条件

  • BTOptions 構造体で定義されたフィールド（fillfactor 等）が、relopt_parse_elt 配列内のオフセットと一致している必要があります。
  • build_reloptions 関数が RELOPT_KIND_BTREE を正しく処理できる環境である必要があります。

アルゴリズムまたは振る舞い

1. fillfactor、vacuum_cleanup_index_scale_factor、deduplicate_items を定義した静的な relopt_parse_elt 配列を構築します。 2. build_reloptions を呼び出し、提供された reloptions を BTOptions 構造体のレイアウトに従って解析・検証します。 3. 解析された結果を bytea 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtree.c:154（amroutine->amoptions への代入として参照）
  • **呼び出し先**:
  • build_reloptions（PostgreSQL 内部の汎用オプション解析関数）

関連データ構造

  • BTOptions: B-tree インデックス固有のオプションを保持する構造体。
  • relopt_parse_elt: オプションの名称、型、構造体内のオフセットを定義するメタデータ構造。

失敗モードと境界条件

  • validate が true の場合、不正なオプション値が渡されると build_reloptions 内部でエラーが発生し、処理が中断されます。

保守担当者向けメモ

  • 本関数は amoptions インターフェースの一部です。新しい B-tree オプションを追加する場合は、tab 配列に要素を追加し、対応するフィールドを BTOptions 構造体に追加してください。
