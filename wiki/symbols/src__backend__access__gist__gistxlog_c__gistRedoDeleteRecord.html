gistRedoDeleteRecord

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistxlog.c:171

目的

GiSTインデックスの削除操作に関するWALレコードをリプレイし、対象ページから指定されたタプルを削除してインデックスの整合性を復元する。

シグネチャまたは形状

static void gistRedoDeleteRecord(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）。
  • **出力**: なし（副作用としてバッファの更新とロック解除を行う）。

前提条件と不変条件

  • record は gistxlogDelete 構造体を含む有効なWALレコードであること。
  • ホットスタンバイ環境下では、ページ更新前に ResolveRecoveryConflictWithSnapshot による競合解決が必須である。

アルゴリズムまたは振る舞い

1. WALレコードから gistxlogDelete データを抽出する。 2. ホットスタンバイ環境であれば、ResolveRecoveryConflictWithSnapshot を呼び出し、削除操作がスタンバイ側のクエリと競合しないことを保証する。 3. XLogReadBufferForRedo を使用して対象ページを読み込む。 4. BLK_NEEDS_REDO が返された場合:

  • PageIndexMultiDelete を実行し、指定されたオフセットのタプルを削除する。
  • GistClearPageHasGarbage および GistMarkTuplesDeleted を呼び出し、ページの状態を更新する。
  • ページのLSNを更新し、MarkBufferDirty でダーティ状態にする。

    • 5. バッファが有効であれば、UnlockReleaseBuffer でロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: gist_redo（src/backend/access/gist/gistxlog.c 内のディスパッチャ）。
  • **呼び出し先**:
  • ResolveRecoveryConflictWithSnapshot
  • XLogReadBufferForRedo
  • PageIndexMultiDelete
  • GistClearPageHasGarbage
  • GistMarkTuplesDeleted
  • PageSetLSN
  • MarkBufferDirty
  • UnlockReleaseBuffer

関連データ構造

  • gistxlogDelete: 削除対象のオフセット情報や競合解決に必要なメタデータを含む構造体。
  • XLogReaderState: WALレコードの読み取り状態を保持する構造体。

失敗モードと境界条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返さない場合、ページ更新はスキップされる。
  • ホットスタンバイ環境での競合解決に失敗した場合、リカバリプロセスが停止または待機する可能性がある。

保守担当者向けメモ

  • 本関数はGiSTの削除操作に特化している。バキューム処理による削除と混同しないこと（バキュームの競合解決は別途 XLOG_HEAP2_PRUNE_VACUUM_SCAN 等で処理済みであるため）。
  • ページ内のタプル削除後に GistClearPageHasGarbage を呼び出すことで、ページ内の不要領域フラグを適切にクリアしている点に注意。
