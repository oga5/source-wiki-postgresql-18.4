PLy_rollback

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_spi.c:492

目的

Python コードから PostgreSQL のトランザクションロールバックを実行し、発生したエラーを Python 例外として再送出する。

シグネチャまたは形状

PyObject *PLy_rollback(PyObject *self, PyObject *args)

入力と出力

  • **入力**: self (モジュールまたはオブジェクトインスタンス), args (引数タプル)。いずれも使用されない。
  • **出力**: 成功時は Py_None を返し、失敗時は Python 例外を発生させ NULL を返す。

前提条件と不変条件

  • PLy_current_execution_context() が有効な実行コンテキストを返す必要がある。
  • PG_TRY ブロック内で実行され、PostgreSQL のエラーハンドリング機構（longjmp）と統合されている。

アルゴリズムまたは振る舞い

1. 現在のメモリコンテキストを保存する。 2. PG_TRY ブロック内で SPI_rollback() を呼び出し、トランザクションをロールバックする。 3. 成功時は exec_ctx->scratch_ctx を NULL にリセットし、Py_None を返す。 4. PG_CATCH ブロック（エラー発生時）:

  • CopyErrorData() でエラー情報を取得し、FlushErrorState() でエラー状態をクリアする。
  • exec_ctx->scratch_ctx を NULL にリセットする。
  • PLy_spi_exceptions ハッシュテーブルからエラーコードに対応する Python 例外クラスを検索する（見つからない場合は PLy_exc_spi_error を使用）。
  • PLy_spi_exception_set() を呼び出し、Python 側で例外を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_plpymodule.c:98 にて、Python モジュールのメソッドとして登録されている。
  • **呼び出し先**: SPI_rollback, PLy_current_execution_context, CopyErrorData, FlushErrorState, hash_search, PLy_spi_exception_set, FreeErrorData。

関連データ構造

  • PLyExecutionContext: 実行コンテキストの管理に使用。
  • PLyExceptionEntry: SQL エラーコードと Python 例外のマップに使用。
  • ErrorData: PostgreSQL のエラー情報保持用。

失敗モードと境界条件

  • SPI_rollback() が失敗した場合、PG_CATCH ブロックへ遷移し、PostgreSQL のエラーが Python 例外としてラップされる。
  • カスタムエラーコードで対応する例外が見つからない場合、デフォルトの SPIError が使用される。

保守担当者向けメモ

  • PG_TRY ブロック内でのメモリ管理に注意すること。エラー発生時には MemoryContextSwitchTo を使用してコンテキストを復元し、FreeErrorData でメモリリークを防いでいる。
