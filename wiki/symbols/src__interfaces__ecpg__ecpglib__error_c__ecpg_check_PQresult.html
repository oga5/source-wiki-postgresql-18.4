ecpg_check_PQresult

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/error.c:280

目的

libpq から返された PGresult のステータスを判定し、ECPG のエラーハンドリング機構へ適切に引き渡すフィルタリングを行う。

シグネチャまたは形状

bool ecpg_check_PQresult(PGresult *results, int lineno, PGconn *connection, enum COMPAT_MODE compat);

入力と出力

  • **入力**:
  • results: libpq の実行結果（PGresult）。
  • lineno: エラー発生時のソースコード行番号。
  • connection: データベース接続ハンドル（PGconn）。
  • compat: 互換モード（COMPAT_MODE）。
  • **出力**:
  • bool: 成功（PGRES_TUPLES_OK, PGRES_COMMAND_OK, PGRES_COPY_OUT）の場合は true、それ以外は false を返す。

前提条件と不変条件

  • results が NULL の場合、connection から取得したエラーメッセージをログ出力し、ecpg_raise_backend を呼び出す。
  • エラー発生時は PQclear(results) を呼び出し、結果セットのメモリを解放する。

アルゴリズムまたは振る舞い

1. results が NULL か判定し、異常時は ecpg_raise_backend を経由してエラー処理へ移行する。 2. PQresultStatus(results) を評価する:

  • 成功系（PGRES_TUPLES_OK, PGRES_COMMAND_OK, PGRES_COPY_OUT）: true を返す。
  • PGRES_EMPTY_QUERY: ecpg_raise で内部エラーを報告し、PQclear 後 false を返す。
  • エラー系（PGRES_NONFATAL_ERROR, PGRES_FATAL_ERROR, PGRES_BAD_RESPONSE）: ログ出力後、ecpg_raise_backend を呼び出し、PQclear 後 false を返す。
  • PGRES_COPY_IN: PQendcopy を実行し、PQclear 後 false を返す。
  • デフォルト: 未知のステータスとしてログ出力し、ecpg_raise_backend を呼び出し、PQclear 後 false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/connect.c
  • src/interfaces/ecpg/ecpglib/descriptor.c
  • src/interfaces/ecpg/ecpglib/execute.c
  • **呼び出し先**:
  • ecpg_log
  • ecpg_raise
  • ecpg_raise_backend
  • PQerrorMessage / PQresultErrorMessage
  • PQresultStatus
  • PQclear
  • PQendcopy

関連データ構造

  • PGresult (libpq)
  • PGconn (libpq)
  • COMPAT_MODE (ECPG)

失敗モードと境界条件

  • results が NULL の場合、libpq の接続エラーとして処理される。
  • PGRES_COPY_IN 状態では PQendcopy が呼び出され、データ転送の中断が試みられる。
  • 未知の PQresultStatus が返された場合、デフォルトの失敗処理が適用される。

保守担当者向けメモ

  • 本関数はエラーハンドリングのゲートウェイであるため、新しい PGresult ステータスが libpq に追加された場合は、switch 文の網羅性を確認すること。
  • ecpg_raise_backend への引き渡しは、sqlca 構造体へのエラー反映を伴うため、呼び出し順序や引数の整合性に注意が必要。
