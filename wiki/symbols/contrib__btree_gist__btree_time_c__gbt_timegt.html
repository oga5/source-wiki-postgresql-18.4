gbt_timegt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_time.c:41

目的

2つの TimeADT 値を比較し、第1引数が第2引数より大きいかどうかを判定します。

シグネチャまたは形状

static bool gbt_timegt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の TimeADT 値へのポインタ。
  • flinfo: 関数マネージャ情報（使用されません）。
  • **出力**:
  • bool: *a > *b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は有効な TimeADT 値を指している必要があります。
  • 内部的に PostgreSQL の time_gt 関数を利用するため、時間型の比較ルールに従います。

アルゴリズムまたは振る舞い

1. 入力された void * ポインタを TimeADT * にキャストします。 2. TimeADTGetDatumFast マクロを使用して、各値を Datum 形式に変換します。 3. DirectFunctionCall2 を介して PostgreSQL の標準関数 time_gt を呼び出し、比較結果を取得します。 4. DatumGetBool を使用して結果を bool 型に変換し、呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_time.c 内の静的テーブル定義（gbt_timekey_cmp 等で使用される比較関数群の一部）から参照されています。
  • **呼び出し先**: time_gt (PostgreSQL コア関数)。

関連データ構造

  • TimeADT: PostgreSQL の時間型を表現する内部データ構造。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、セグメンテーション違反が発生する可能性があります。
  • time_gt 関数がエラーを発生させた場合、例外として伝播します。

保守担当者向けメモ

  • 本関数は btree_gist モジュールにおける時間型の比較ロジックの一部です。同様の比較関数（gbt_timege, gbt_timeeq 等）と一貫した実装パターンを持っています。
  • flinfo 引数は現在の実装では使用されていませんが、将来的な拡張や DirectFunctionCall の要件に合わせてシグネチャに含まれています。
