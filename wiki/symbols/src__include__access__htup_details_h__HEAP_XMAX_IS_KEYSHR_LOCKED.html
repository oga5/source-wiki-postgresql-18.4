HEAP_XMAX_IS_KEYSHR_LOCKED

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/htup_details.h:278

目的

ヒープタプルの infomask を検査し、タプルが「キー共有ロック（Key Share Lock）」状態にあるかどうかを判定する。

シグネチャまたは形状

static inline bool HEAP_XMAX_IS_KEYSHR_LOCKED(int16 infomask)

入力と出力

  • **入力**: int16 infomask（タプルの t_infomask フィールド）
  • **出力**: bool（HEAP_XMAX_KEYSHR_LOCK と一致する場合は true、それ以外は false）

前提条件と不変条件

  • 入力される infomask は、有効なヒープタプルの t_infomask であること。
  • HEAP_LOCK_MASK によるビットマスク処理が適用されるため、他のロック状態と混同されないことが保証されている。

アルゴリズムまたは振る舞い

1. 引数 infomask に対して HEAP_LOCK_MASK を適用し、ロック関連のビットのみを抽出する。 2. 抽出された値が HEAP_XMAX_KEYSHR_LOCK と等しいかを比較する。 3. 一致すれば true を返し、そうでなければ false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgrowlocks/pgrowlocks.c: pgrowlocks 関数内
  • src/backend/access/heap/heapam.c: heap_lock_tuple、heap_update、heap_delete 等のタプル操作関数内
  • **呼び出し先**:
  • 本関数は単一の論理演算を行うインライン関数であり、他の関数を呼び出さない。

関連データ構造

  • HeapTupleHeaderData: タプルヘッダー構造体。t_infomask を保持する。
  • HEAP_LOCK_MASK: ロック状態を識別するためのビットマスク定数。
  • HEAP_XMAX_KEYSHR_LOCK: キー共有ロック状態を示すビット値。

失敗モードと境界条件

  • infomask が不正な値（ビットが壊れている等）である場合、誤った判定を返す可能性がある。
  • 本関数自体はメモリ安全性やポインタの妥当性を検証しないため、呼び出し側で適切なタプルヘッダーが参照されている必要がある。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、インライン化によるオーバーヘッド削減が意図されている。
  • ロック状態の定義（HEAP_XMAX_KEYSHR_LOCK 等）が変更された場合、本関数のロジックも影響を受ける可能性があるため注意すること。
