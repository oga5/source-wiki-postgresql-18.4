boot_openrel

定義

  • 種別: function
  • 定義箇所: src/backend/bootstrap/bootstrap.c:439

目的

ブートストラップモードにおいて、指定された名前のリレーションを開き、その属性情報をキャッシュ（attrtypes）に読み込むことで、以降のタプル挿入操作を可能にする。

シグネチャまたは形状

void boot_openrel(char *relname)

入力と出力

  • **入力**: relname (開く対象のリレーション名。NAMEDATALEN を超える場合は切り詰められる)
  • **出力**: なし（副作用としてグローバル変数 boot_reldesc および attrtypes を更新する）

前提条件と不変条件

  • pg_type が構築済みであること（未構築の場合は populate_typ_list() が呼び出される）。
  • 既に別のリレーションが開かれている場合は、closerel(NULL) により適切に閉じられる。

アルゴリズムまたは振る舞い

1. relname の長さをチェックし、必要に応じて切り詰める。 2. Typ リストが空であれば populate_typ_list() を呼び出し、型情報を初期化する。 3. 既に boot_reldesc が設定されている場合は closerel() を実行して閉じる。 4. table_openrv を使用してリレーションを開き、boot_reldesc に格納する。 5. リレーションの属性数（numattr）を取得する。 6. 各属性について、attrtypes 配列の要素を確保（AllocateAttribute）し、リレーションの記述子から属性情報をコピーする。 7. デバッグログ（DEBUG4）に属性情報を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/bootstrap/bootstrap.c 内のBKIコマンド処理ロジックから呼び出される（直接的な呼び出し元関数の詳細は提供されていない）。
  • **呼び出し先**: populate_typ_list, closerel, table_openrv, makeRangeVar, RelationGetNumberOfAttributes, AllocateAttribute, TupleDescAttr, memmove, elog。

関連データ構造

  • boot_reldesc: 現在操作中のリレーション記述子。
  • attrtypes: 属性情報を保持するグローバル配列。
  • Typ: 型情報のリスト。

失敗モードと境界条件

  • relname が NAMEDATALEN を超える場合、末尾が切り詰められる。
  • table_openrv がリレーションを見つけられない場合、基盤となるアクセスメソッドの挙動に従いエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はブートストラップ時の「鶏と卵」問題を解決するため、pg_type の状態に依存した初期化を行う。
  • attrtypes へのコピーには ATTRIBUTE_FIXED_PART_SIZE が使用されており、属性定義の構造変更時には注意が必要である。
