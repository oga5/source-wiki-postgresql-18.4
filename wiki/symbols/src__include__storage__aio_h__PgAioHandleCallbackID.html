PgAioHandleCallbackID

定義

  • 種別: enum
  • 定義箇所: src/include/storage/aio.h:192

目的

非同期 I/O (AIO) 完了時に実行されるコールバック関数を識別するための列挙型です。共有メモリ内での関数ポインタの安定性を確保し、特に EXEC_BACKEND 環境下でのポインタ不一致問題を回避するために使用されます。

シグネチャまたは形状

typedef enum PgAioHandleCallbackID
{
    PGAIO_HCB_INVALID = 0,
    PGAIO_HCB_MD_READV,
    PGAIO_HCB_SHARED_BUFFER_READV,
    PGAIO_HCB_LOCAL_BUFFER_READV,
} PgAioHandleCallbackID;

入力と出力

  • 入力: pgaio_io_register_callbacks 関数において、コールバックを登録する際の識別子として使用されます。
  • 出力: I/O 完了処理の内部で、実行すべき具体的なコールバック関数を特定するためのキーとして機能します。

前提条件と不変条件

  • PGAIO_HCB_INVALID は初期状態または無効なコールバックを指します。
  • 登録された ID は、対応する PgAioHandle のライフサイクルを通じて一貫して保持される必要があります。

アルゴリズムまたは振る舞い

I/O 操作の完了時、システムは登録された PgAioHandleCallbackID を参照し、対応する処理ロジック（md、shared_buffer、local_buffer 等の読み込み処理）をディスパッチします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/aio/aio_callback.c 内の各関数（pgaio_io_register_callbacks 等）がこの ID を使用してコールバックを管理・参照しています。
  • 呼び出し先: 直接的な関数呼び出しではなく、この ID をキーとして適切な完了処理ルーチンへ制御が移ります。

関連データ構造

  • PgAioHandle: I/O 操作のコンテキストを保持し、この ID を内部配列等で管理します。
  • PgAioHandleCallbacks: 実際の関数ポインタ群を保持する構造体ですが、本 ID はこれらへの間接参照として機能します。

失敗モードと境界条件

  • PGAIO_HCB_INVALID が指定された場合、コールバック登録や実行時にエラーが発生する可能性があります。
  • 範囲外の ID を指定した場合の挙動は未定義です。

保守担当者向けメモ

  • 新しい I/O 完了処理を追加する場合は、この列挙型に新しいメンバを追加し、対応するディスパッチロジックを aio_callback.c に実装してください。
  • EXEC_BACKEND 環境におけるポインタの安定性を維持するため、直接関数ポインタを共有メモリに格納せず、必ずこの ID を介して参照してください。
