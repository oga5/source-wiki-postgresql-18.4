InvalMessageArray

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/cache/inval.c:175

目的

キャッシュ無効化メッセージを格納するための動的配列を管理し、トランザクションやコマンド境界で発生する無効化イベントを一時的に保持する。

シグネチャまたは形状

typedef struct InvalMessageArray
{
	SharedInvalidationMessage *msgs;	/* palloc'd array (can be expanded) */
	int			maxmsgs;		/* current allocated size of array */
} InvalMessageArray;

入力と出力

  • **入力**: SharedInvalidationMessage 型のメッセージを配列に追加する。
  • **出力**: msgs ポインタを通じて、蓄積された無効化メッセージのリストを提供し、必要に応じて配列の拡張を行う。

前提条件と不変条件

  • msgs は palloc によってメモリ確保されている必要がある。
  • maxmsgs は現在確保されている配列の要素数を保持する。
  • 配列は必要に応じて拡張可能である。

アルゴリズムまたは振る舞い

  • キャッシュ無効化イベントが発生した際、この構造体を使用してメッセージを蓄積する。
  • src/backend/utils/cache/inval.c 内で静的配列 InvalMessageArrays[2] としてインスタンス化され、サブグループごとのメッセージ管理に使用される。
  • メッセージの追加時に容量が不足した場合は、配列を再確保して拡張する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/inval.c 内のキャッシュ無効化処理ロジック。
  • **呼び出し先**: palloc 等のメモリ管理関数（配列の拡張時）。
  • **注記**: 具体的な関数呼び出しの連鎖については、提供されたソース excerpt には詳細な記述がない。

関連データ構造

  • SharedInvalidationMessage: 配列に格納される個々の無効化メッセージの型。
  • InvalidationMsgsGroup: 本構造体を利用してメッセージをグループ化する上位構造体。

失敗モードと境界条件

  • メモリ不足: palloc が失敗した場合、PostgreSQL の標準的なエラー処理（ereport(ERROR, ...)）によりトランザクションがアボートされる。
  • 配列拡張の失敗: メモリ確保に失敗した場合、システムは整合性を維持するために処理を停止する。

保守担当者向けメモ

  • InvalMessageArrays は static な配列として定義されており、グローバルな状態管理の一部となっている。
  • 本構造体は inval.c 内のキャッシュ無効化ライフサイクル（コマンド境界およびトランザクション終了時の処理）において重要な役割を果たすため、変更時は AtEOXact_Inval 等のトランザクション処理への影響を考慮すること。
