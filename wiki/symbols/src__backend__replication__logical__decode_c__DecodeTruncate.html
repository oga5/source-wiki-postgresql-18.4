DecodeTruncate

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/decode.c:1073

目的

WALの XLOG_HEAP_TRUNCATE レコードを解析し、論理デコーディング用の変更情報として ReorderBuffer にキューイングする。

シグネチャまたは形状

static void DecodeTruncate(LogicalDecodingContext *ctx, XLogRecordBuffer *buf)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • buf: 解析対象のWALレコードおよび関連情報を含む XLogRecordBuffer。
  • **出力**: なし（副作用として ReorderBuffer への変更追加を行う）。

前提条件と不変条件

  • buf->record は XLOG_HEAP_TRUNCATE 型のWALレコードを保持している必要がある。
  • ctx->slot が有効であり、データベースIDが一致する場合のみ処理が継続される。

アルゴリズムまたは振る舞い

1. WALレコードから xl_heap_truncate 構造体を取得する。 2. レコードの dbId が現在のスロットのデータベースIDと一致しない場合は終了する。 3. FilterByOrigin を呼び出し、レプリケーションオリジンによるフィルタリング対象であれば終了する。 4. ReorderBufferAllocChange を使用して REORDER_BUFFER_CHANGE_TRUNCATE 型の変更オブジェクトを確保する。 5. レコード内のフラグ（XLH_TRUNCATE_CASCADE, XLH_TRUNCATE_RESTART_SEQS）を変換し、変更オブジェクトに設定する。 6. 対象となるリレーションIDの配列を ReorderBufferAllocRelids で確保し、レコードからコピーする。 7. ReorderBufferQueueChange を呼び出し、トランザクションIDとLSN（buf->origptr）を紐付けて変更をキューに登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/decode.c 内のディスパッチャ（LogicalDecodingProcessRecord からの呼び出しが示唆される）。
  • **呼び出し先**:
  • XLogRecGetData: WALレコードデータへのアクセス。
  • FilterByOrigin: オリジンフィルタリング。
  • ReorderBufferAllocChange: 変更オブジェクトの確保。
  • ReorderBufferAllocRelids: リレーションID配列の確保。
  • ReorderBufferQueueChange: 変更のキューイング。

関連データ構造

  • LogicalDecodingContext: デコーディング状態。
  • XLogRecordBuffer: WALレコードのバッファ。
  • xl_heap_truncate: WAL上の物理的なTRUNCATEレコード構造。
  • ReorderBufferChange: 論理デコーディングで扱われる変更単位。

失敗モードと境界条件

  • データベースIDが不一致の場合、処理は早期リターンされる。
  • FilterByOrigin が真を返す場合、処理は早期リターンされる。
  • メモリ不足時は ReorderBufferAllocChange 等の内部関数がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は xl_heap_truncate 構造体のレイアウトに強く依存している。WALフォーマットの変更時には src/include/access/heapam_xlog.h の定義と整合性を確認すること。
  • ReorderBufferQueueChange の第5引数（false）は、この変更が即時適用ではなくトランザクションの一部として蓄積されることを示す。
