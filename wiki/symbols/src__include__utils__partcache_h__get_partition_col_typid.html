get_partition_col_typid

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/partcache.h:85

目的

パーティションキーの指定された列に対応するデータ型ID（Oid）を取得します。

シグネチャまたは形状

Oid get_partition_col_typid(PartitionKey key, int col)

入力と出力

  • **入力**:
  • key: パーティション情報を保持する PartitionKey 構造体へのポインタ。
  • col: 取得対象の列インデックス（0から始まる整数）。
  • **出力**:
  • 指定された列のデータ型を表す Oid を返します。

前提条件と不変条件

  • key は有効な PartitionKey 構造体を指している必要があります。
  • col は有効な列範囲内（0 以上、かつパーティションキーの列数未満）である必要があります。
  • パーティションキーはリレーションを開いている間は不変であるため、取得される型IDもその期間中有効です。

アルゴリズムまたは振る舞い

key->parttypid 配列の指定されたインデックス col に格納されている Oid を直接参照して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execPartition.c:1674
  • src/backend/parser/parse_utilcmd.c:4366
  • src/backend/parser/parse_utilcmd.c:4521
  • **呼び出し先**:
  • 本関数はインライン関数であり、他の関数を呼び出しません。

関連データ構造

  • PartitionKey: パーティションキーのメタデータを保持する構造体。本関数はこの構造体内の parttypid 配列を参照します。

失敗モードと境界条件

  • col が不正な値（負数や列数以上）である場合、メモリの不正アクセスが発生する可能性があります。本関数自体には境界チェックは含まれていません。

保守担当者向けメモ

  • 本関数は partcache.h で定義されたインライン関数群の一部であり、PartitionKey 構造体の内部実装をカプセル化しています。将来的に PartitionKeyData の構造が変更された場合、このアクセサを介することで影響範囲を限定できます。
