astreamer_lz4_frame

定義

  • 種別: struct
  • 定義箇所: src/fe_utils/astreamer_lz4.c:28

目的

LZ4アルゴリズムを用いたデータの圧縮および伸長処理において、ストリーマーの内部状態を保持するための構造体です。

シグネチャまたは形状

typedef struct astreamer_lz4_frame
{
    astreamer    base;

    LZ4F_compressionContext_t cctx;
    LZ4F_decompressionContext_t dctx;
    LZ4F_preferences_t prefs;

    size_t       bytes_written;
    bool         header_written;
} astreamer_lz4_frame;

入力と出力

本構造体は関数引数として渡される astreamer ポインタをキャストして利用されます。直接的な入出力関数ではなく、状態管理のためのデータコンテナです。

前提条件と不変条件

  • astreamer 基底構造体（base）が先頭メンバとして配置されており、アップキャストが可能です。
  • USE_LZ4 が定義されている環境でのみ有効です。
  • 圧縮または伸長のいずれかのコンテキスト（cctx または dctx）が適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

  • astreamer フレームワークのパイプライン処理において、圧縮・伸長中のコンテキスト、設定、および進捗状況（書き込みバイト数やヘッダーの有無）を追跡します。
  • 圧縮時には LZ4F_compressionContext_t を、伸長時には LZ4F_decompressionContext_t を使用して、ストリームデータの断片的な処理を可能にします。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer_lz4_compressor_new および astreamer_lz4_decompressor_new にてメモリ確保（palloc0）が行われます。また、各ストリーマーの content、finalize、free 関数内でキャストして参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • astreamer: 基底インターフェース構造体。
  • LZ4F_compressionContext_t / LZ4F_decompressionContext_t: LZ4ライブラリのコンテキスト。
  • LZ4F_preferences_t: LZ4圧縮設定。

失敗モードと境界条件

  • メモリ割り当て失敗時（palloc0）にプロセスが終了する可能性があります。
  • 圧縮・伸長処理中にバッファが不足した場合、astreamer の仕組みを通じて動的に拡張されますが、その際のメモリ制限に依存します。

保守担当者向けメモ

  • 本構造体は astreamer_lz4.c 内でのみ使用されます。
  • 構造体のメンバを変更する場合は、astreamer_lz4_compressor_new 等の初期化ロジックと、各処理関数でのキャスト後のアクセス整合性を確認してください。
