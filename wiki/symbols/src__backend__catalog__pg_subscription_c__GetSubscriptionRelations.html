GetSubscriptionRelations

定義

  • **種別**: function
  • **定義箇所**: src/backend/catalog/pg_subscription.c:546

目的

指定されたサブスクリプションに関連付けられたテーブルの同期状態（SubscriptionRelState）のリストを取得します。

シグネチャまたは形状

List *GetSubscriptionRelations(Oid subid, bool not_ready)

入力と出力

  • **入力**:
  • subid (Oid): 対象のサブスクリプションID。
  • not_ready (bool): true の場合、状態が SUBREL_STATE_READY ではないレコードのみを抽出します。
  • **出力**:
  • List *: SubscriptionRelState 構造体のリスト。各要素には relid（テーブルID）、state（同期状態）、lsn（LSN）が含まれます。

前提条件と不変条件

  • pg_subscription_rel システムカタログに対する AccessShareLock を保持して操作を行います。
  • 呼び出し元は、返されたリストに対して適切なメモリ解放を行う必要があります。

アルゴリズムまたは振る舞い

1. pg_subscription_rel を AccessShareLock でオープンします。 2. subid をキーとしてスキャンを開始します。not_ready が true の場合は、srsubstate が SUBREL_STATE_READY ではないタプルに絞り込みます。 3. スキャン結果を順次走査し、各タプルから srrelid、srsubstate、srsublsn を抽出します。 4. 抽出した値を SubscriptionRelState 構造体に格納し、リストに追加します。 5. スキャンを終了し、リレーションをクローズしてリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/subscriptioncmds.c (複数箇所)
  • src/backend/replication/logical/tablesync.c
  • **呼び出し先**:
  • table_open, table_close
  • ScanKeyInit, systable_beginscan, systable_getnext, systable_endscan
  • palloc, lappend
  • SysCacheGetAttr, DatumGetLSN

関連データ構造

  • SubscriptionRelState: テーブルの同期状態を保持する構造体。
  • pg_subscription_rel: 参照先のシステムカタログ。

失敗モードと境界条件

  • 指定された subid に該当するレコードが存在しない場合、空のリスト（NIL）を返します。
  • srsublsn が NULL の場合、lsn には InvalidXLogRecPtr が設定されます。

保守担当者向けメモ

  • 本関数は pg_subscription_rel を直接スキャンします。頻繁に呼び出される場合は、カタログアクセスのコストに注意してください。
  • not_ready フラグによるフィルタリングは、初期同期プロセスにおいて特定の状態のテーブルを特定するために使用されます。
