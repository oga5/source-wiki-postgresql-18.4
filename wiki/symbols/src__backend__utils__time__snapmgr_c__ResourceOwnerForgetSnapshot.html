ResourceOwnerForgetSnapshot

定義

  • 種別: function
  • 定義箇所: src/backend/utils/time/snapmgr.c:238

目的

指定された ResourceOwner が保持しているスナップショットの追跡を解除します。

シグネチャまたは形状

static inline void ResourceOwnerForgetSnapshot(ResourceOwner owner, Snapshot snap)

入力と出力

  • **入力**:
  • owner: スナップショットを所有しているリソースオーナー。
  • snap: 追跡を解除する対象のスナップショット構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • snap は、事前に ResourceOwnerRememberSnapshot を介して当該 owner に登録されている必要があります。
  • snapshot_resowner_desc がリソースオーナーの管理用記述子として定義されている必要があります。

アルゴリズムまたは振る舞い

1. snap ポインタを Datum 型に変換します。 2. ResourceOwnerForget を呼び出し、リソースオーナーの管理下から当該スナップショットの参照を削除します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/time/snapmgr.c:882 (直接の呼び出しを確認)
  • **呼び出し先**:
  • ResourceOwnerForget (リソースオーナー管理機能)

関連データ構造

  • ResourceOwner: リソースの所有権を管理する構造体。
  • Snapshot: 可視性判定に使用されるスナップショットデータ。
  • snapshot_resowner_desc: リソースオーナーがスナップショットを識別・管理するための記述子。

失敗モードと境界条件

  • owner が無効な場合や、snap が当該 owner に登録されていない場合の挙動は ResourceOwnerForget の実装に依存します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、スナップショットのライフサイクル管理におけるリソースリーク防止のために使用されます。
  • ResourceOwnerRememberSnapshot と対になる操作であるため、修正時は必ずセットで確認してください。
