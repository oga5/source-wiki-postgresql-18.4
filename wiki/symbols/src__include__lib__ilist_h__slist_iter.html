slist_iter

定義

  • 種別: struct
  • 定義箇所: src/include/lib/ilist.h:257

目的

単一連結リスト（slist）を走査するためのイテレータ構造体です。

シグネチャまたは形状

typedef struct slist_iter
{
    slist_node *cur;
} slist_iter;

入力と出力

  • **入力**: 走査対象の slist_head を保持するリスト構造。
  • **出力**: 走査中の現在のノード（slist_node *cur）。

前提条件と不変条件

  • cur は、リスト内の有効なノードを指すか、リストの終端を示す NULL である必要があります。
  • イテレータを使用する間、リストの構造が外部から破壊的に変更されないことが前提です（走査中の削除が必要な場合は slist_mutable_iter を使用してください）。

アルゴリズムまたは振る舞い

  • スタック上に確保され、slist_foreach マクロを通じてリストの先頭から順次 next ポインタを辿るために使用されます。
  • cur メンバが現在の走査位置を保持し、リストの終端に達するまで更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/event_trigger.c
  • src/backend/utils/cache/catcache.c
  • **呼び出し先**:
  • src/include/lib/ilist.h 内の slist_foreach マクロ（型チェック等で利用）。

関連データ構造

  • slist_node: リストの各要素を構成するリンク構造。
  • slist_head: リストの起点。
  • slist_mutable_iter: 走査中にノードを削除するための代替イテレータ。

失敗モードと境界条件

  • リストが空の場合、初期化直後の cur は NULL となり、走査は即座に終了します。
  • 走査中にリストのノードを free() すると、次のノードへのポインタが不正になり、未定義動作を引き起こします。

保守担当者向けメモ

  • 本構造体はメモリを動的に割り当てません。スタック上で宣言して使用してください。
  • ILIST_DEBUG を定義すると、リスト操作時の整合性チェックが強化されます。
