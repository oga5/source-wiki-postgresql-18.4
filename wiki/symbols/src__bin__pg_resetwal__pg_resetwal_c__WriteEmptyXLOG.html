WriteEmptyXLOG

定義

  • 種別: function
  • 定義箇所: src/bin/pg_resetwal/pg_resetwal.c:1116

目的

データベースの WAL（Write-Ahead Log）を初期化し、新しいチェックポイントレコードを含む空の WAL セグメントファイルを生成する。

シグネチャまたは形状

static void WriteEmptyXLOG(void)

入力と出力

  • **入力**: グローバル変数 ControlFile（制御情報）および WalSegSz、newXlogSegNo を参照。
  • **出力**: なし（副作用としてファイルシステム上に WAL セグメントファイルを生成）。

前提条件と不変条件

  • ControlFile が有効な値で初期化されていること。
  • 呼び出し前に KillExistingXLOG() 等により古い WAL ファイルが削除されていること。
  • ファイルシステムへの書き込み権限があること。

アルゴリズムまたは振る舞い

1. **ヘッダー構築**: PGAlignedXLogBlock を使用し、XLOG_PAGE_MAGIC を含むページヘッダーを初期化。ControlFile からタイムライン ID やシステム識別子を設定。 2. **チェックポイント作成**: 初期チェックポイントレコード（XLOG_CHECKPOINT_SHUTDOWN）を構築し、ControlFile.checkPointCopy をデータとして格納。 3. **CRC 計算**: チェックポイントレコードの CRC を計算し、ヘッダーに格納。 4. **ファイル生成**: XLogFilePath で決定したパスに新しい WAL セグメントファイルを作成。 5. **書き込み**: 最初のページ（ヘッダーとチェックポイント）を書き込み、残りの領域をゼロ埋めする。 6. **同期**: fsync を実行してディスクへの書き込みを確定させ、ファイルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_resetwal/pg_resetwal.c 内の main 関数（525行目付近）。
  • **呼び出し先**: pg_fatal（エラー時）、open, write, fsync, close 等の標準システムコール。

関連データ構造

  • ControlFileData（ControlFile グローバル変数）
  • XLogPageHeader
  • XLogRecord
  • CheckPoint

失敗モードと境界条件

  • ファイル作成失敗（open が負を返す場合）: pg_fatal によりエラー終了。
  • ディスク容量不足（write が失敗し errno が設定される場合）: pg_fatal によりエラー終了。
  • fsync 失敗: pg_fatal によりエラー終了。

保守担当者向けメモ

  • 本関数は pg_resetwal の最終段階で実行される。
  • WAL セグメントサイズ（WalSegSz）やブロックサイズ（XLOG_BLCKSZ）の変更が正しく反映されるよう、ControlFile の整合性に依存している点に注意すること。
