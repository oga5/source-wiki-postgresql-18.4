getsubdfa

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexec.c:373

目的

正規表現のツリー構造（subre ノード）に対応する DFA（決定性有限オートマトン）を取得または生成します。

シグネチャまたは形状

static struct dfa *getsubdfa(struct vars *v, struct subre *t);

入力と出力

  • **入力**:
  • v: マッチング状態を保持する vars 構造体へのポインタ。
  • t: 対象となる subre ツリーノードへのポインタ。
  • **出力**:
  • 成功時: 対応する dfa 構造体へのポインタ。
  • 失敗時: NULL（メモリ不足等）。

前提条件と不変条件

  • v->subdfas 配列が subre ノードの ID をインデックスとして DFA をキャッシュするために使用されます。
  • t がバックリファレンスノード（t->op == 'b'）である場合、生成された DFA にバックリファレンス関連の制約情報が設定されます。

アルゴリズムまたは振る舞い

1. v->subdfas[t->id] を参照し、既に DFA が存在すればそれを返します。 2. 存在しない場合、newdfa を呼び出して新しい DFA を構築します。 3. newdfa が成功し、かつノードがバックリファレンス（op == 'b'）であれば、t から backno、backmin、backmax を DFA にコピーします。 4. 生成した DFA を v->subdfas[t->id] に保存し、呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regexec.c 内の cdissect 関連処理（849, 851, 930, 932, 1098, 1173, 1380 行目付近）から頻繁に呼び出されます。
  • **呼び出し先**: newdfa（rege_dfa.c で定義）。

関連データ構造

  • struct vars: マッチング全体のコンテキスト。
  • struct subre: 正規表現のツリー構造ノード。
  • struct dfa: 状態遷移を管理する DFA 構造体。

失敗モードと境界条件

  • newdfa がメモリ不足等で NULL を返した場合、本関数も NULL を返します。
  • t->id が v->subdfas 配列の範囲外である場合の挙動については、本ソース抜粋からは確認できません。

保守担当者向けメモ

  • 本関数は遅延評価 DFA のキャッシュ機構の一部です。
  • バックリファレンスノードの属性（backno 等）は、DFA 構築時にのみこの関数を介して反映されます。
