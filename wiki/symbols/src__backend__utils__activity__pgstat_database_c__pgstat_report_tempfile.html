pgstat_report_tempfile

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_database.c:210

目的

一時ファイルの生成に関する統計情報を、現在のデータベースの統計エントリに記録する。

シグネチャまたは形状

void pgstat_report_tempfile(size_t filesize)

入力と出力

  • **入力**: filesize (生成された一時ファイルのサイズ、単位はバイト)。
  • **出力**: なし。

前提条件と不変条件

  • グローバル変数 pgstat_track_counts が真である場合にのみ統計が更新される。
  • MyDatabaseId を用いて現在のデータベースの統計エントリが取得可能であること。

アルゴリズムまたは振る舞い

1. pgstat_track_counts を確認し、偽であれば即座に終了する。 2. pgstat_prep_database_pending(MyDatabaseId) を呼び出し、現在のデータベースに対応する PgStat_StatDBEntry 構造体を取得する。 3. 取得したエントリの temp_bytes に filesize を加算する。 4. 取得したエントリの temp_files を 1 インクリメントする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/file/fd.c:1534 (直接的な証拠あり)。
  • **呼び出し先**: pgstat_prep_database_pending (直接的な証拠あり)。

関連データ構造

  • PgStat_StatDBEntry: データベース単位の統計情報を保持する構造体。

失敗モードと境界条件

  • pgstat_track_counts が無効な場合、統計は記録されない。
  • pgstat_prep_database_pending が失敗した場合の挙動については、提供されたソースコード内には記述がない。

保守担当者向けメモ

  • 本関数は pgstat_track_counts の設定値に依存するため、統計収集がオフの環境ではオーバーヘッドを最小限に抑える設計となっている。
  • データベース単位の統計を更新する関数であり、テーブルやインデックス単位の統計とは分離されている。
