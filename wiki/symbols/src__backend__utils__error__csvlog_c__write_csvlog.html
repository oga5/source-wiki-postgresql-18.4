write_csvlog

定義

  • 種別: function
  • 定義箇所: src/backend/utils/error/csvlog.c:62

目的

ErrorData 構造体に含まれるエラー情報を CSV 形式の文字列に変換し、ログ出力先へ送信する。

シグネチャまたは形状

void write_csvlog(ErrorData *edata)

入力と出力

  • **入力**: ErrorData *edata（エラー詳細情報）
  • **出力**: なし（副作用としてログバッファへの書き込みおよび出力先への転送を行う）

前提条件と不変条件

  • プロセスごとの静的変数 log_line_number は、プロセス ID (MyProcPid) が変更された際にリセットされる。
  • CSV リテラルへの変換は appendCSVLiteral を介して行われ、ダブルクォーテーションのエスケープが保証される。

アルゴリズムまたは振る舞い

1. **初期化**: プロセス ID を確認し、変更があれば log_line_number をリセットして開始時刻を更新する。 2. **メタデータ構築**: タイムスタンプ、ユーザー名、DB 名、PID、セッション ID、行番号、PS 表示、トランザクション情報などを StringInfo バッファに順次追加する。 3. **エラー詳細変換**: ErrorData のメッセージ、詳細、ヒント、SQL ステート等を CSV 形式に変換してバッファに追加する。 4. **クエリ情報**: check_log_of_query の判定に基づき、必要に応じて実行中のクエリ文字列とカーソル位置を追加する。 5. **出力**: 構築した CSV 文字列を、プロセス種別に応じて syslogger へ転送または直接書き込みを行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/error/elog.c
  • **呼び出し先**: appendCSVLiteral, get_formatted_log_time, get_ps_display, GetTopTransactionIdIfAny, error_severity, unpack_sql_state, check_log_of_query, get_backend_type_for_log 等。

関連データ構造

  • ErrorData: エラーの各属性を保持する構造体。
  • StringInfoData: ログ文字列構築用バッファ。
  • PGPROC: プロセス情報およびトランザクション ID 保持用。

失敗モードと境界条件

  • MyProcPort が NULL の場合、ユーザー名やホスト名などの接続情報は出力されない。
  • edata->detail_log が存在する場合はそちらを優先し、なければ edata->detail を使用する。
  • Log_error_verbosity が PGERROR_VERBOSE 以上の場合のみ、関数名やファイル名などの詳細なエラー位置情報が付与される。

保守担当者向けメモ

  • 本関数は elog.c から呼び出されるログ出力の主要なエントリポイントである。
  • CSV フォーマットの変更は appendCSVLiteral のエスケープルールと整合性を取る必要がある。
  • プロセス間通信（パイプ経由）の設計上、ログ出力によるブロッキングを避ける実装となっている。
