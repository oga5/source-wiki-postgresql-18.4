px_resolve_alias

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/px.c:129

目的

暗号アルゴリズム名や関連する識別子に対して、定義済みのエイリアスリストを照合し、正規化された名称を返します。

シグネチャまたは形状

const char *px_resolve_alias(const PX_Alias *list, const char *name);

入力と出力

  • **入力**:
  • list: PX_Alias 構造体の配列へのポインタ。name メンバが NULL である要素で終端されている必要があります。
  • name: 解決対象の文字列（アルゴリズム名など）。
  • **出力**:
  • エイリアスが一致した場合、対応する正規名称（list->name）。
  • 一致しない場合、入力された name そのもの。

前提条件と不変条件

  • list は name メンバが NULL で終端された有効な配列を指している必要があります。
  • 比較は pg_strcasecmp を使用するため、大文字小文字を区別しません。

アルゴリズムまたは振る舞い

1. list の各要素を順次走査します。 2. 現在の要素の alias メンバと、引数 name を大文字小文字を区別せずに比較します。 3. 一致した場合は、その要素の name メンバを即座に返します。 4. list->name が NULL に到達するまで一致が見つからない場合、入力された name をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/openssl.c 内の ossl_aliases を用いた呼び出しが確認されています。
  • **呼び出し先**: pg_strcasecmp（PostgreSQL 標準ライブラリ関数）。

関連データ構造

  • PX_Alias: エイリアス名と正規名称のペアを保持する構造体（px.h で定義）。

失敗モードと境界条件

  • list が空（最初の要素の name が NULL）の場合、常に name を返します。
  • name が NULL の場合の挙動は、pg_strcasecmp の実装に依存します。

保守担当者向けメモ

  • 本関数は破壊的な操作を行わず、入力された文字列のポインタをそのまま返す設計です。
  • 新しいアルゴリズムを追加する際は、対応する PX_Alias 配列を更新することで本関数による解決が可能になります。
