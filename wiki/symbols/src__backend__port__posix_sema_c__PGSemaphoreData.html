PGSemaphoreData

定義

  • **種別**: struct
  • **定義箇所**: src/backend/port/posix_sema.c:52

目的

PostgreSQL のプロセス間同期用セマフォ（PGSemaphore）を保持するためのデータ構造です。共有メモリ上に配置されるセマフォ配列の各要素として機能します。

シグネチャまたは形状

typedef struct PGSemaphoreData
{
    SemTPadded sem_padded;
} PGSemaphoreData;

入力と出力

  • **入力**: 構造体内部の sem_padded フィールド（SemTPadded 型）が、プラットフォーム固有のセマフォ実体を保持します。
  • **出力**: なし（データコンテナ）。

前提条件と不変条件

  • 共有メモリ上に PGSemaphoreData の配列として確保される必要があります。
  • posix_sema.c においては、SemTPadded を介して POSIX セマフォ（sem_t）をラップする形式をとります。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持に特化しており、直接的なアルゴリズムは持ちません。この構造体へのポインタが PGSemaphore 型として扱われ、PGSemaphoreLock や PGSemaphoreUnlock などの関数を通じて、内部の sem_t に対する操作が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/port/posix_sema.c 内の PGSemaphoreShmemSize 関数が、必要なメモリサイズ計算のために本構造体を参照しています。
  • 同ファイル内の sharedSemas 配列定義において使用されています。
  • **呼び出し先**:
  • 本構造体はデータ構造であるため、関数を呼び出すことはありません。

関連データ構造

  • SemTPadded: 本構造体が内部で保持する、パディングを考慮したセマフォ型。
  • PGSemaphore: PGSemaphoreData へのポインタとして定義される抽象型。

失敗モードと境界条件

  • **メモリ確保**: 共有メモリの不足により PGSemaphoreData の配列確保が失敗する可能性があります。
  • **プラットフォーム依存**: EXEC_BACKEND 環境下では、名前付きセマフォを使用する場合に本構造体の共有メモリ配置と競合する制約があります。

保守担当者向けメモ

  • sysv_sema.c にも同名の構造体が存在しますが、実装の詳細（内部の型）が異なります。変更時はプラットフォームごとの差異に注意してください。
  • PGSemaphoreData のサイズは PGSemaphoreShmemSize で計算されるため、構造体定義を変更した場合は必ずこの計算ロジックを再確認してください。
