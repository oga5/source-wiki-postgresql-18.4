pg_logical_slot_peek_changes

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/logicalfuncs.c:339

目的

論理レプリケーションスロットからWALの変更内容を読み取るためのSQLインターフェース関数です。スロットの読み取り位置を更新（確認）せずに、変更内容を「覗き見（peek）」するために使用されます。

シグネチャまたは形状

Datum pg_logical_slot_peek_changes(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL経由で渡されるスロット名、LSN、アップストリームのオプション等）。
  • **出力**: Datum（デコードされたWAL変更内容を含む結果セット）。

前提条件と不変条件

  • 本関数は pg_logical_slot_get_changes_guts を呼び出すラッパーです。
  • 内部的に confirm = false を指定して呼び出すため、実行後にレプリケーションスロットの永続的な読み取り位置は更新されません。

アルゴリズムまたは振る舞い

1. pg_logical_slot_get_changes_guts を呼び出します。 2. 第2引数に false を渡し、読み取り位置の確認（confirm）を行わないように制御します。 3. 第3引数に false を渡し、バイナリ形式ではなくテキスト形式での出力を指定します。 4. 取得した変更内容を Tuplestore 形式で返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（ユーザーからのクエリ実行）。
  • **呼び出し先**: pg_logical_slot_get_changes_guts。

関連データ構造

  • DecodingOutputState: 出力プラグインから生成されたデータを保持し、SQLの戻り値へ変換するために使用されます。

失敗モードと境界条件

  • 存在しないスロットを指定した場合や、権限不足の場合はエラーが発生します。
  • pg_logical_slot_get_changes_guts 内の PG_TRY/PG_CATCH ブロックにより、エラー発生時にはリソースリークを防ぐためのクリーンアップが行われます。

保守担当者向けメモ

  • 本関数は単なるラッパーです。ロジックの変更が必要な場合は、核となる pg_logical_slot_get_changes_guts を参照してください。
  • 読み取り位置を更新する pg_logical_slot_get_changes との挙動の違い（confirm 引数の有無）を維持することが重要です。
