preprocess_aggrefs_walker

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepagg.c:343

目的

クエリツリーを再帰的に走査し、遭遇した Aggref ノードに対して集約の前処理（preprocess_aggref）を適用します。

シグネチャまたは形状

static bool preprocess_aggrefs_walker(Node *node, PlannerInfo *root);

入力と出力

  • **入力**:
  • node: 走査対象のノード（Node *）。
  • root: プランニング情報（PlannerInfo *）。
  • **出力**:
  • 走査を継続する場合は false、停止する場合は true を返します（expression_tree_walker の規約に従う）。

前提条件と不変条件

  • パーサーにより、集約の引数、直接引数、およびフィルタ句内に（現在のレベルの）集約関数が含まれていないことが保証されている必要があります。
  • SubLink ノードが含まれていないことを前提としています（Assert で検証）。

アルゴリズムまたは振る舞い

1. node が NULL の場合は false を返して終了します。 2. node が Aggref 型の場合:

  • preprocess_aggref(aggref, root) を呼び出し、当該集約ノードを処理します。
  • パーサーの保証により、集約の内部構造（引数やフィルタ）を再帰的に走査する必要がないため、false を返して走査を停止します。

    • 3. node が Aggref 以外の場合:

  • expression_tree_walker を使用して、子ノードに対して再帰的に本関数を適用します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/prep/prepagg.c 内の preprocess_aggrefs（直接の呼び出しを確認）。
  • 本関数自身（再帰呼び出し）。
  • **呼び出し先**:
  • preprocess_aggref
  • expression_tree_walker

関連データ構造

  • Aggref: 処理対象となる集約関数ノード。
  • PlannerInfo: プランニングの全体状態を保持する構造体。

失敗モードと境界条件

  • SubLink ノードに遭遇した場合、Assert により異常終了します。

保守担当者向けメモ

  • 本関数は expression_tree_walker を利用したツリー走査のコールバックです。
  • Aggref ノードを見つけた際に再帰を停止する最適化が行われています。これは、集約関数がネストしないというクエリの性質に基づいています。
