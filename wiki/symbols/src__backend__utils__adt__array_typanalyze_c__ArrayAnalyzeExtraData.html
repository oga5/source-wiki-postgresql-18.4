ArrayAnalyzeExtraData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/array_typanalyze.c:36

目的

配列型のカラムに対する ANALYZE 処理において、配列要素の型情報や統計計算に必要な関数ポインタ、および標準的な統計解析の状態を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    Oid type_id;
    Oid eq_opr;
    Oid coll_id;
    bool typbyval;
    int16 typlen;
    char typalign;
    FmgrInfo *cmp;
    FmgrInfo *hash;
    AnalyzeAttrComputeStatsFunc std_compute_stats;
    void *std_extra_data;
} ArrayAnalyzeExtraData;

入力と出力

  • 本構造体は ANALYZE 実行時に palloc を介してメモリ確保され、配列要素型のキャッシュ情報や標準的な統計解析関数のポインタで初期化されます。
  • 出力として、compute_array_stats 等の関数がこの構造体を参照し、統計情報の収集を行います。

前提条件と不変条件

  • cmp および hash フィールドに格納される FmgrInfo は、対応する型の typcache エントリに由来しており、ANALYZE 実行期間中有効であることが期待されます。
  • std_extra_data は std_typanalyze() から引き継がれた状態を保持します。

アルゴリズムまたは振る舞い

  • 配列要素の型に関する物理的プロパティ（typlen, typalign 等）を保持し、配列のデコードや要素の比較・ハッシュ計算を支援します。
  • 標準的な統計解析（std_typanalyze）の結果をラップし、配列特有の統計（最頻要素や要素数ヒストグラム）を収集するためのコンテキストとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/array_typanalyze.c 内の array_typanalyze および compute_array_stats 関数。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありませんが、保持する FmgrInfo ポインタを通じて型固有の比較・ハッシュ関数が実行されます。

関連データ構造

  • FmgrInfo: 関数マネージャ情報。
  • AnalyzeAttrComputeStatsFunc: 標準的な統計計算関数の型定義。
  • TrackItem: Lossy Counting アルゴリズムで使用されるハッシュテーブルエントリ。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • typcache エントリが無効な場合、cmp や hash フィールドの参照は未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は ANALYZE のライフサイクルに依存します。CurrentMemoryContext 上で確保されるため、明示的な解放は不要です。
  • 統計情報の精度が不足している場合は、保持されている cmp や hash 関数が正しく設定されているかを確認してください。
