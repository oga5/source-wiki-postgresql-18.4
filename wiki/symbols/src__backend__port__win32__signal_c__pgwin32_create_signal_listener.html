pgwin32_create_signal_listener

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/signal.c:226

目的

Windows環境において、特定のプロセスID（PID）に対するシグナル要求を受け取るための名前付きパイプを作成する。

シグネチャまたは形状

HANDLE pgwin32_create_signal_listener(pid_t pid)

入力と出力

  • **入力**: pid_t pid - シグナルリスナーを関連付ける対象プロセスのID。
  • **出力**: HANDLE - 作成された名前付きパイプのハンドル。

前提条件と不変条件

  • 指定されたPIDに基づき、\\.\pipe\pgsignal_<pid> という形式の名前付きパイプが生成される。
  • パイプは双方向通信（PIPE_ACCESS_DUPLEX）およびメッセージモード（PIPE_TYPE_MESSAGE）で構成される。

アルゴリズムまたは振る舞い

1. 引数の pid を用いてパイプ名文字列を構築する。 2. Windows API CreateNamedPipe を呼び出し、パイプを作成する。 3. 作成に失敗した場合（INVALID_HANDLE_VALUE）、ereport(ERROR, ...) を通じてエラーを報告し、処理を中断する。 4. 成功した場合、作成したパイプハンドルを呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c 内のコードから呼び出されることが確認されている。
  • **呼び出し先**: Windows API CreateNamedPipe および ereport（PostgreSQLのエラー報告関数）。

関連データ構造

  • HANDLE: Windows APIのパイプハンドル型。

失敗モードと境界条件

  • CreateNamedPipe が失敗した場合、GetLastError() を含むエラーメッセージとともに ereport(ERROR, ...) が発生し、プロセスが異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は、Windows上でPOSIXシグナルをエミュレートするための基盤の一部である。
  • パイプ名はPIDに依存するため、同一PIDで複数のリスナーを生成しようとした場合の挙動は、Windowsの CreateNamedPipe の仕様に準拠する。
