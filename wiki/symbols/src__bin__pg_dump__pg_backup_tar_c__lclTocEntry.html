lclTocEntry

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_backup_tar.c:91

目的

TAR形式のアーカイブにおいて、目次（TOC）の各エントリに関連付けられたファイル情報とTARメンバー情報を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	TAR_MEMBER *TH;
	char	   *filename;
} lclTocEntry;

入力と出力

  • **入力**: pg_malloc0 等によりメモリが確保され、te->formatData（TocEntry構造体のメンバ）に格納されます。
  • **出力**: te->formatData を通じて、アーカイブの読み込みや書き込み処理中に参照されます。

前提条件と不変条件

  • TH は TAR_MEMBER 構造体へのポインタであり、有効なTARヘッダー情報を指している必要があります。
  • filename は、アーカイブ内の該当メンバーに対応するファイルパスを指す文字列である必要があります。

アルゴリズムまたは振る舞い

本構造体は、pg_dump のアーカイブ処理において、特定のTOCエントリがどのTARメンバーに対応するかを追跡するために利用されます。pg_backup_custom.c 等の他モジュールからも te->formatData をキャストして参照されることがあり、アーカイブのメタデータと物理的なファイル構成を紐付ける役割を果たします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_custom.c 内の複数の箇所で、te->formatData からキャストして利用されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • TAR_MEMBER: TARアーカイブ内の個々のメンバー（ファイル）のメタデータを管理する構造体。
  • TocEntry (暗黙的): formatData メンバを通じて本構造体を保持する上位の構造体。

失敗モードと境界条件

  • メモリ確保失敗: pg_malloc0 が失敗した場合、プログラムは終了します。
  • 不正なポインタ: te->formatData が適切に初期化されていない状態でアクセスした場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は pg_backup_tar.c で定義されていますが、pg_backup_custom.c からも参照されています。変更を加える際は、これら両方のモジュールにおける整合性に注意してください。
  • TH メンバが指す TAR_MEMBER のライフサイクルと、lclTocEntry のライフサイクルが一致していることを確認してください。
