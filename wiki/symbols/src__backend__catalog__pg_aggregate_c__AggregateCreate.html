AggregateCreate

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_aggregate.c:45

目的

PostgreSQLのシステムカタログ pg_aggregate への集約関数定義の登録、および関連するサポート関数との依存関係構築を行う。

シグネチャまたは形状

ObjectAddress AggregateCreate(const char *aggName, Oid aggNamespace, bool replace, char aggKind, int numArgs, int numDirectArgs, oidvector *parameterTypes, Datum allParameterTypes, Datum parameterModes, Datum parameterNames, List *parameterDefaults, Oid variadicArgType, List *aggtransfnName, List *aggfinalfnName, List *aggcombinefnName, List *aggserialfnName, List *aggdeserialfnName, List *aggmtransfnName, List *aggminvtransfnName, List *aggmfinalfnName, bool finalfnExtraArgs, bool mfinalfnExtraArgs, char finalfnModify, char mfinalfnModify, List *aggsortopName, Oid aggTransType, int32 aggTransSpace, Oid aggmTransType, int32 aggmTransSpace, const char *agginitval, const char *aggminitval, char proparallel)

入力と出力

  • **入力**: 集約関数の名称、名前空間、引数定義、遷移関数・最終関数・結合関数等の名前リスト、および遷移型（aggTransType）や初期値等のメタデータ。
  • **出力**: 作成された集約関数の ObjectAddress を返す。

前提条件と不変条件

  • aggName および aggtransfnName は必須。
  • 引数の数（numArgs）は FUNC_MAX_ARGS - 1 以下であること。
  • 遷移型（aggTransType）が多相型の場合、集約の引数も多相型である必要がある。
  • AGGKIND_HYPOTHETICAL（仮説的集合集約）の場合、直接引数と集約引数の型整合性が必須。

アルゴリズムまたは振る舞い

1. **入力検証**: 引数の数、多相型の一貫性、および集約の種類に応じた制約（AGGKIND_HYPOTHETICAL 等）をチェックする。 2. **関数検索**: 遷移関数や最終関数等のサポート関数を検索し、OIDを取得する（ソース抜粋以降の処理）。 3. **カタログ登録**: pg_proc への登録後、pg_aggregate テーブルにタプルを挿入する。 4. **依存関係構築**: record_object_address_dependencies を呼び出し、集約関数とサポート関数間の依存関係を pg_depend に記録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/aggregatecmds.c（CREATE AGGREGATE コマンド処理）。
  • **呼び出し先**: ソース抜粋内では check_valid_polymorphic_signature を呼び出している。以降の処理で lookup_agg_function、ProcedureCreate、table_open、heap_form_tuple 等を呼び出すことが示唆されている。

関連データ構造

  • pg_aggregate: 集約関数の詳細を保持するシステムカタログ。
  • ObjectAddress: 作成されたオブジェクトの識別子。
  • oidvector: パラメータ型を保持する構造体。

失敗モードと境界条件

  • 引数の数が上限を超えた場合、ERRCODE_TOO_MANY_ARGUMENTS を送出。
  • 多相型の整合性が取れない場合、ERRCODE_INVALID_FUNCTION_DEFINITION を送出。
  • AGGKIND_HYPOTHETICAL で引数の型が一致しない場合、エラーを送出。

保守担当者向けメモ

  • 本関数は非常に多くの引数を取るため、シグネチャ変更時には pg_aggregate.h の定義と整合性を確認すること。
  • internal 型の扱いは厳格であるため、サポート関数の検証ロジックを変更する際は注意が必要。
  • 依存関係の登録は pg_depend を介して行われるため、カタログ更新後の整合性維持に留意すること。
