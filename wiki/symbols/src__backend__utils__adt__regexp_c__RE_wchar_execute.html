RE_wchar_execute

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/regexp.c:281

目的

pg_wchar 形式のデータに対して、コンパイル済みの正規表現を適用し、マッチングを実行する。

シグネチャまたは形状

bool RE_wchar_execute(regex_t *re, pg_wchar *data, int data_len, int start_search, int nmatch, regmatch_t *pmatch)

入力と出力

  • **入力**:
  • re: コンパイル済みの正規表現オブジェクト (regex_t*)。
  • data: 検索対象のワイド文字配列 (pg_wchar*)。
  • data_len: data の長さ。
  • start_search: 検索を開始するオフセット位置。
  • nmatch: pmatch 配列のサイズ。
  • pmatch: マッチング結果（位置情報）を格納する構造体配列。
  • **出力**:
  • bool: マッチした場合は true、マッチしなかった場合は false を返す。

前提条件と不変条件

  • re は事前に pg_regcomp 等で正しく初期化されている必要がある。
  • pmatch 配列は nmatch 分の領域が確保されている必要がある。

アルゴリズムまたは振る舞い

1. pg_regexec を呼び出し、正規表現エンジンによるマッチングを実行する。 2. pg_regexec の戻り値を確認する。

  • REG_OKAY または REG_NOMATCH 以外が返された場合、pg_regerror でエラーメッセージを取得し、ereport(ERROR, ...) を発行して処理を中断する。

    • 3. regexec_result が REG_OKAY である場合に true を返し、それ以外（REG_NOMATCH）の場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/regexp.c 内の複数箇所（正規表現マッチング関数群）。
  • **呼び出し先**:
  • pg_regexec (正規表現エンジン本体)
  • pg_regerror (エラーメッセージ生成)
  • ereport (PostgreSQL エラー報告メカニズム)

関連データ構造

  • regex_t: POSIX 正規表現コンパイル済みオブジェクト。
  • pg_wchar: PostgreSQL で使用されるワイド文字型。
  • regmatch_t: マッチした部分文字列の開始・終了位置を保持する構造体。

失敗モードと境界条件

  • **正規表現エンジンエラー**: pg_regexec が REG_OKAY および REG_NOMATCH 以外の値を返した場合、ERRCODE_INVALID_REGULAR_EXPRESSION エラーとして処理が停止する。
  • **メモリ不足**: ereport 経由でバックエンドのメモリ管理に依存する。

保守担当者向けメモ

  • 本関数は pg_wchar 形式のデータ専用である。マルチバイト文字列を扱う際は、事前に適切な変換が行われていることを確認すること。
  • pg_regexec の第5引数（details）には NULL が渡されており、詳細なマッチング情報は取得されない仕様となっている。
