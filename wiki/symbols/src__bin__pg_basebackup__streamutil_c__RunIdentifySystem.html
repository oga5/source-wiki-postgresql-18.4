RunIdentifySystem

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/streamutil.c:408

目的

PostgreSQLサーバーに対して IDENTIFY_SYSTEM コマンドを発行し、システム識別子、タイムラインID、WAL開始位置、およびデータベース名を取得します。

シグネチャまたは形状

bool RunIdentifySystem(PGconn *conn, char **sysid, TimeLineID *starttli, XLogRecPtr *startpos, char **db_name);

入力と出力

  • **入力**:
  • conn: 確立済みの PGconn 接続オブジェクト。
  • sysid, starttli, startpos, db_name: 各値を受け取るためのポインタ（不要な場合は NULL を指定可能）。
  • **出力**:
  • 成功時に true、失敗時に false を返します。
  • 各ポインタ引数には、サーバーから取得した値が格納されます（sysid と db_name は pg_strdup で確保されるため、呼び出し元で解放が必要）。

前提条件と不変条件

  • conn は NULL であってはなりません（Assert で検証）。
  • サーバーとの接続がレプリケーションモードで確立されている必要があります。

アルゴリズムまたは振る舞い

1. IDENTIFY_SYSTEM コマンドをサーバーに送信し、結果セットを検証します。 2. サーバーから返された行（1行）とフィールド（最低3つ、9.4以降は4つ）を確認します。 3. フィールドから以下の情報を抽出します：

  • sysid: システム識別子文字列。
  • starttli: タイムラインID（整数値）。
  • startpos: WAL位置（%X/%X 形式の文字列を uint64 に変換）。
  • db_name: サーバーバージョンが 9.4 以上の場合のみ、4番目のフィールドから取得。

    • 4. 処理終了後、PQclear を呼び出して結果セットを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_basebackup/pg_receivewal.c
  • src/bin/pg_basebackup/pg_recvlogical.c
  • src/bin/pg_basebackup/receivelog.c
  • **呼び出し先**:
  • libpq 関数群 (PQexec, PQresultStatus, PQntuples, PQnfields, PQgetvalue, PQgetisnull, PQserverVersion, PQclear, PQerrorMessage)
  • pg_strdup, pg_log_error (PostgreSQLフロントエンドユーティリティ)

関連データ構造

  • PGconn, PGresult (libpq)
  • TimeLineID, XLogRecPtr (PostgreSQL内部型)

失敗モードと境界条件

  • 接続が切断されている場合やコマンド実行に失敗した場合は false を返します。
  • サーバーから期待される行数やフィールド数が返されない場合、エラーログを出力して false を返します。
  • WAL位置のパースに失敗した場合、エラーログを出力して false を返します。
  • 9.4 未満のサーバーに対して db_name を要求した場合、そのフィールドは取得されません。

保守担当者向けメモ

  • 本関数はフロントエンドコードであるため、バックエンドの関数を直接呼び出せません。
  • サーバーバージョンによるフィールド数の差異（9.4以降の db_name 追加）を考慮した実装になっています。新しいフィールドが追加される場合は、同様のバージョンチェックが必要です。
