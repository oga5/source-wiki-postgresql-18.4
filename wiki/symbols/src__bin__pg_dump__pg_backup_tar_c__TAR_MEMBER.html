TAR_MEMBER

定義

  • **種別**: struct
  • **定義箇所**: src/bin/pg_dump/pg_backup_tar.c:66

目的

TAR形式のアーカイブ内における個々のファイルメンバー（データまたはメタデータ）の状態と位置情報を管理する構造体です。

シグネチャまたは形状

typedef struct {
    FILE       *nFH;          /* 現在のファイルハンドル */
    FILE       *tarFH;        /* アーカイブ全体のファイルハンドル */
    FILE       *tmpFH;        /* 一時ファイルハンドル */
    char       *targetFile;   /* 対象ファイル名 */
    char        mode;         /* モード（読み込み/書き込み） */
    pgoff_t     pos;          /* 現在のオフセット位置 */
    pgoff_t     fileLen;      /* ファイル長 */
    ArchiveHandle *AH;        /* アーカイブハンドラへのポインタ */
} TAR_MEMBER;

入力と出力

本構造体は、pg_dump のTAR形式アーカイブ処理における入出力操作のコンテキストとして機能します。各フィールドは、アーカイブ内の特定のメンバーに対する読み書きの進捗や、一時ファイルとの連携を制御するために使用されます。

前提条件と不変条件

  • ArchiveHandle を介してアーカイブ全体の管理状態と紐付いている必要があります。
  • 書き込み時には tmpFH を使用してデータを蓄積し、最終的に _tarAddFile を通じてメインアーカイブへ統合されるというライフサイクルを持ちます。

アルゴリズムまたは振る舞い

  • **シークと読み込み**: _tarPositionTo によりアーカイブ内の特定メンバーまで移動し、tarRead を用いてデータを抽出します。
  • **書き込み**: _StartData で一時ファイルを作成し、_WriteData でデータを蓄積後、_EndData でTARヘッダーと共にメインアーカイブへ書き出します。
  • **終了処理**: tarClose により、使用中のファイルハンドルを適切にクローズします。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_backup_tar.c 内の各アーカイブ操作関数（tarOpen, tarClose, tarRead, tarPrintf, _tarAddFile, _tarPositionTo など）が本構造体を引数として受け取ります。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ArchiveHandle: アーカイブ全体の抽象化レイヤー。
  • lclContext: TAR形式全体のコンテキストを保持する構造体。

失敗モードと境界条件

  • 圧縮はサポートされていません（gzdopen 等のバッファリングがファイル位置の正確な制御を阻害するため）。
  • 一時ファイルの作成失敗や、アーカイブのシーク失敗時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は pg_backup_tar.c 内の内部的な状態管理に特化しています。
  • メンバーの追加や変更を行う際は、pg_backup_tar.h で定義されるTARヘッダー構造との整合性に注意してください。
