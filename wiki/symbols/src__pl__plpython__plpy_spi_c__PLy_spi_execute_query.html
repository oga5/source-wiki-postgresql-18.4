PLy_spi_execute_query

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_spi.c:293

目的

Python から渡された SQL クエリ文字列を PostgreSQL の SPI (Server Programming Interface) を介して実行し、その結果を Python オブジェクトとして返却する。

シグネチャまたは形状

static PyObject *PLy_spi_execute_query(char *query, long limit)

入力と出力

  • **入力**:
  • query: 実行する SQL クエリ文字列。
  • limit: クエリ結果の最大行数。
  • **出力**:
  • 成功時: クエリ結果を格納した PyObject* (通常は PLyResultObject)。
  • 失敗時: NULL (Python 側で例外として処理される)。

前提条件と不変条件

  • 実行前に PLy_spi_subtransaction_begin を呼び出し、メモリコンテキストとリソースオーナーを保護する必要がある。
  • クエリ文字列は pg_verifymbstr によりマルチバイト文字列として検証される。
  • SPI 実行は PG_TRY ブロック内で保護され、エラー発生時にはサブトランザクションがアボートされる。

アルゴリズムまたは振る舞い

1. 現在のメモリコンテキストとリソースオーナーを保存する。 2. PLy_spi_subtransaction_begin でサブトランザクションを開始する。 3. PG_TRY ブロック内で以下を実行する:

  • pg_verifymbstr でクエリの妥当性を確認。
  • SPI_execute を呼び出し、クエリを実行する。
  • PLy_spi_execute_fetch_result を呼び出し、SPI の結果セットを Python オブジェクトに変換する。
  • PLy_spi_subtransaction_commit でサブトランザクションをコミットする。

    • 4. PG_CATCH ブロック内で PLy_spi_subtransaction_abort を呼び出し、エラーを処理する。 5. SPI の戻り値 rv が負（エラー）の場合、PLy_exception_set を通じて Python 例外を発生させ、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_spi.c 内の関数（PLy_spi_execute からの呼び出しが確認されている）。
  • **呼び出し先**: PLy_spi_subtransaction_begin, PG_TRY, PLy_current_execution_context, pg_verifymbstr, SPI_execute, PLy_spi_execute_fetch_result, PLy_spi_subtransaction_commit, PG_CATCH, PLy_spi_subtransaction_abort, PLy_exception_set, SPI_result_code_string。

関連データ構造

  • PLyExecutionContext: 現在の実行コンテキスト。
  • SPI_tuptable: SPI 実行後のタプル結果セット。
  • SPI_processed: SPI 実行により処理された行数。

失敗モードと境界条件

  • SPI_execute が負の値を返した場合、SPI エラーとして処理される。
  • PostgreSQL 内部でエラーが発生した場合、PG_CATCH ブロックへ遷移し、サブトランザクションがアボートされる。
  • メモリ不足やクエリの構文エラーは、PostgreSQL のエラーハンドリング機構により longjmp を介して PG_CATCH へ転送される。

保守担当者向けメモ

  • 本関数は volatile なメモリコンテキストとリソースオーナーを保持しており、PG_TRY ブロック内でのメモリ管理に注意が必要である。
  • SPI_execute の結果は SPI_tuptable などのグローバル変数に格納されるため、変換処理は PG_TRY ブロック内で完結させる必要がある。
