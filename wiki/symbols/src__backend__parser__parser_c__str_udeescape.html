str_udeescape

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parser.c:371

目的

SQLのUnicodeエスケープ文字列（U&'...' 形式）に含まれるエスケープシーケンスを解析し、サーバーのエンコーディングに変換した文字列を生成する。

シグネチャまたは形状

static char *str_udeescape(const char *str, char escape, int position, core_yyscan_t yyscanner);

入力と出力

  • **入力**:
  • str: 解析対象の文字列リテラル。
  • escape: エスケープ文字（通常はバックスラッシュ）。
  • position: エラー報告用の開始位置オフセット。
  • yyscanner: 字句解析器の状態。
  • **出力**: 変換後の文字列を保持するメモリ領域（palloc で確保）。

前提条件と不変条件

  • 入力文字列内のエスケープシーケンスは \XXXX または \+XXXXXX の形式である必要がある。
  • UTF-16サロゲートペアが使用される場合、上位サロゲートの直後に下位サロゲートが続く必要がある。
  • 変換処理中、scanner_errposition_callback を用いてエラー発生時の位置情報を保持する。

アルゴリズムまたは振る舞い

1. 入力文字列を走査し、エスケープ文字を検出する。 2. \XXXX（4桁）または \+XXXXXX（6桁）の形式を識別し、16進数値をコードポイントに変換する。 3. check_unicode_value で値の妥当性を検証する。 4. サロゲートペアを検出した場合、ペアを結合して単一のコードポイントに変換する。 5. pg_unicode_to_server を呼び出し、コードポイントをサーバーのエンコーディングに変換して出力バッファに書き込む。 6. エスケープ文字自体が連続する場合（\\）は、エスケープ文字として扱う。 7. 変換後の文字列を格納するバッファが不足した場合は repalloc で拡張する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parser.c 内の構文解析処理（raw_parser のフローに関連する箇所）。
  • **呼び出し先**: palloc, repalloc, strlen, isxdigit, hexval, check_unicode_value, is_utf16_surrogate_first, is_utf16_surrogate_second, surrogate_pair_to_codepoint, pg_unicode_to_server, setup_scanner_errposition_callback, cancel_scanner_errposition_callback, ereport, scanner_errposition。

関連データ構造

  • ScannerCallbackState: エラー位置報告用コールバックの状態管理。
  • pg_wchar: Unicodeコードポイントを保持する型。

失敗モードと境界条件

  • **構文エラー**: 形式が不正なエスケープシーケンス（\XXXX 以外）や、サロゲートペアの不整合（上位のみ、下位のみ、順序不正）が発生した場合、ereport(ERROR, ...) により処理を中断する。
  • **メモリ不足**: palloc / repalloc が失敗した場合はシステム標準のメモリ不足エラーが発生する。

保守担当者向けメモ

  • 本関数は U& 構文の処理に特化している。
  • エラー発生時には scanner_errposition を使用して、SQL文字列内の正確な位置をユーザーに通知する設計となっている。
  • サロゲートペアの処理において、pair_first 変数が状態保持に使用されているため、ループ内での状態遷移に注意すること。
