pe_test_config

定義

  • 種別: struct
  • 定義箇所: src/test/modules/test_escape/test_escape.c:24

目的

test_escape テストスイート全体で共有される実行時設定およびテスト状態を保持するための構造体です。

シグネチャまたは形状

typedef struct pe_test_config
{
	int			verbosity;
	bool		force_unsupported;
	const char *conninfo;
	PGconn	   *conn;

	int			test_count;
	int			failure_count;
} pe_test_config;

入力と出力

  • **入力**: 実行時のコマンドライン引数や接続情報が格納されます。
  • **出力**: テスト実行中に test_count（実行数）および failure_count（失敗数）が更新され、最終的なテスト結果の集計に使用されます。

前提条件と不変条件

  • conn フィールドは、テスト実行前に確立された有効な PGconn オブジェクトを指す必要があります。
  • test_count および failure_count は、テストの進行に伴い非負の整数として加算されます。

アルゴリズムまたは振る舞い

本構造体は、テストスイートのグローバルな状態管理を行います。main 関数で初期化された後、各テスト関数（test_gb18030_page_multiple 等）にポインタとして渡され、テストの実行状況の追跡や、接続情報の参照、エラー報告の制御に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数にて pe_test_config tc = {0}; として初期化されます。
  • **呼び出し先**: 以下の関数が引数として受け取ります。
  • report_result
  • test_gb18030_page_multiple
  • test_gb18030_json
  • test_psql_parse
  • test_one_vector_escape
  • test_one_vector

関連データ構造

  • PGconn (libpq): データベース接続状態を保持するために使用されます。

失敗モードと境界条件

  • conn が NULL の状態でテスト関数が実行された場合、libpq 関連のテストでセグメンテーションフォールトが発生する可能性があります。
  • failure_count がインクリメントされる条件は、各テスト関数内での検証ロジック（report_result 等）に依存します。

保守担当者向けメモ

  • 本構造体はテストスイートの「コンテキスト」として機能します。新しいテスト項目を追加する際は、この構造体を通じて設定や状態を共有してください。
  • verbosity フィールドの具体的な利用範囲については、各テスト関数内のログ出力ロジックを参照してください。
