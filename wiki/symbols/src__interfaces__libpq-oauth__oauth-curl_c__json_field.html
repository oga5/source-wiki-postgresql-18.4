json_field

定義

  • **種別**: struct
  • **定義箇所**: src/interfaces/libpq-oauth/oauth-curl.c:435

目的

OAuth 2.0 認可サーバーから返される JSON レスポンスをパースするためのスキーマ定義を提供します。各フィールドのキー名、期待される型、格納先、および必須フラグを保持します。

シグネチャまたは形状

struct json_field {
    const char *name;
    JsonTokenType type;
    union {
        char **scalar;
        struct curl_slist **array;
    } target;
    bool required;
};

入力と出力

  • **入力**: parse_oauth_json 関数等において、パース対象の JSON 構造を定義する配列として使用されます。
  • **出力**: パース成功時、target メンバ（scalar または array）を介して、抽出された値が呼び出し元のコンテキストへ書き込まれます。

前提条件と不変条件

  • type は JSON_TOKEN_STRING、JSON_TOKEN_NUMBER、または JSON_TOKEN_ARRAY_START のいずれかである必要があります。
  • target 共用体は type に応じて適切に選択される必要があります（ARRAY_START の場合は curl_slist、それ以外は char ポインタ）。

アルゴリズムまたは振る舞い

parse_oauth_json 関数がこの構造体の配列を走査し、JSON のキーと name を照合します。一致した場合、type に基づいて値をパースし、対応する target ポインタ先に値を格納します。required が true のフィールドが JSON 内に見つからない場合、パース処理は失敗とみなされます。

呼び出し元と呼び出し先

  • **呼び出し元**: parse_oauth_json 関数がこの構造体を参照してパース処理を実行します。また、oauth-curl.c 内の各ステップ（Discovery, Device Authorization, Token Request 等）で、レスポンス解析用の定義配列として静的に初期化されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • JsonTokenType: common/jsonapi.h で定義されるトークン型。
  • curl_slist: libcurl の文字列リスト構造体。
  • async_ctx: 本構造体を用いたパース結果を保持するコンテキスト構造体。

失敗モードと境界条件

  • required フィールドが欠落している場合、パース処理はエラーを返します。
  • type がサポート外のトークン型である場合、パース処理は適切に処理できない可能性があります。
  • target ポインタが指すメモリ領域が適切に確保されていない場合、書き込み時に未定義動作が発生します。

保守担当者向けメモ

  • 新しい JSON フィールドを追加する際は、type に応じて target 共用体の適切なメンバを指定してください。
  • required フラグの検証ロジックは parse_oauth_json 内に実装されています。変更時は当該関数の整合性を確認してください。
