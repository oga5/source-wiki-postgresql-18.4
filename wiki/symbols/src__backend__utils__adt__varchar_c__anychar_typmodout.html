anychar_typmodout

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varchar.c:71

目的

char(n) および varchar(n) 型の型修飾子（typmod）を、SQLの CREATE TABLE 等で表示可能な文字列形式に変換します。

シグネチャまたは形状

char *anychar_typmodout(int32 typmod)

入力と出力

  • **入力**: typmod (int32型)。内部的な型修飾子値。
  • **出力**: char *。palloc で確保された、(n) 形式の文字列、または修飾子がない場合は空文字列。

前提条件と不変条件

  • typmod は VARHDRSZ を加算した値として保持されていることを前提としています。
  • 戻り値の文字列は palloc で確保されるため、呼び出し元で適切に管理する必要があります。

アルゴリズムまたは振る舞い

1. 64バイトのメモリ領域を palloc で確保します。 2. typmod が VARHDRSZ より大きい場合、typmod - VARHDRSZ を計算し、(n) という形式の文字列をバッファに書き込みます。 3. typmod が VARHDRSZ 以下の場合、バッファの先頭にヌル文字を書き込み、空文字列とします。 4. 生成した文字列バッファを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/varchar.c:429
  • src/backend/utils/adt/varchar.c:660
  • **呼び出し先**:
  • palloc (メモリ確保)
  • snprintf (文字列整形)

関連データ構造

  • VARHDRSZ: 可変長データのヘッダーサイズ。typmod からこの値を差し引くことで、ユーザーが指定した長さ n を復元します。

失敗モードと境界条件

  • palloc がメモリ不足により失敗した場合、PostgreSQL の標準的なエラーハンドリングにより例外が発生します。
  • typmod が負の値や異常に小さい値の場合、else 分岐により空文字列が返されます。

保守担当者向けメモ

  • 本関数は char(n) と varchar(n) の両方で共通して使用されます。
  • 出力形式を変更する場合は、SQL 標準の型修飾子表示との整合性に注意してください。
