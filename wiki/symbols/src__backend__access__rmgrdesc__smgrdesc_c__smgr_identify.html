smgr_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/smgrdesc.c:43

目的

ストレージマネージャ（smgr）に関連するWALレコードの info フィールドに基づき、操作の種類（CREATE または TRUNCATE）を識別する文字列を返します。

シグネチャまたは形状

const char *smgr_identify(uint8 info)

入力と出力

  • **入力**: uint8 info - WALレコードの info フィールド。
  • **出力**: const char * - 操作名を表す静的文字列（"CREATE" または "TRUNCATE"）。該当しない場合は NULL を返します。

前提条件と不変条件

  • 入力された info ビットから XLR_INFO_MASK を除去した値が、XLOG_SMGR_CREATE または XLOG_SMGR_TRUNCATE のいずれかと一致する必要があります。

アルゴリズムまたは振る舞い

1. info に対してビットマスク ~XLR_INFO_MASK を適用し、サブタイプを抽出します。 2. switch 文を用いて抽出した値を判定します。

  • XLOG_SMGR_CREATE の場合: "CREATE" を返します。
  • XLOG_SMGR_TRUNCATE の場合: "TRUNCATE" を返します。

    • 3. いずれにも該当しない場合、初期値である NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h において、RmgrData 構造体の登録時に参照されています。
  • **呼び出し先**: 本関数内での関数呼び出しはありません。

関連データ構造

  • XLR_INFO_MASK: WALレコードのサブタイプを分離するために使用されるビットマスク。
  • XLOG_SMGR_CREATE, XLOG_SMGR_TRUNCATE: ストレージ操作を識別するための定数。

失敗モードと境界条件

  • info が定義された操作のいずれにも一致しない場合、戻り値は NULL となります。

保守担当者向けメモ

  • 本関数は pg_waldump 等のツールがWALの内容を人間が読める形式に変換する際に利用されます。
  • 新しいストレージ操作が追加された場合は、switch 文に新しい case を追加する必要があります。
