pgwin32_ReserveSharedMemoryRegion

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/port/win32_shmem.c:572

目的

Windows環境において、子プロセス（バックエンド）が起動する前に、親プロセス（Postmaster）から子プロセスのアドレス空間内の特定領域を予約する。これにより、共有メモリが意図したアドレスにマッピングされることを保証し、他のDLL等によるアドレス空間の占有を防ぐ。

シグネチャまたは形状

int pgwin32_ReserveSharedMemoryRegion(HANDLE hChild)

入力と出力

  • **入力**: hChild (予約対象となる子プロセスのプロセスハンドル)
  • **出力**: 成功した場合は true (1)、失敗した場合は false (0) を返す。

前提条件と不変条件

  • ShmemProtectiveRegion が NULL でないこと。
  • UsedShmemSegAddr が NULL でないこと。
  • UsedShmemSegSize が 0 でないこと。
  • 実行コンテキストは Postmaster プロセス内であること。

アルゴリズムまたは振る舞い

1. VirtualAllocEx を使用し、ShmemProtectiveRegion で指定されたアドレス範囲を PAGE_NOACCESS として予約する。 2. 予約されたアドレスが期待値と一致するか確認する。 3. VirtualAllocEx を使用し、UsedShmemSegAddr で指定された共有メモリ領域を PAGE_READWRITE として予約する。 4. 予約されたアドレスが期待値と一致するか確認する。 5. いずれかのステップで予約に失敗、またはアドレスの不一致が発生した場合は、エラーをログ出力し false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c (子プロセス起動処理内)
  • **呼び出し先**: Windows API VirtualAllocEx

関連データ構造

  • ShmemProtectiveRegion: アドレス空間の断片化を防ぐための保護領域のベースアドレス。
  • UsedShmemSegAddr: 共有メモリセグメントのベースアドレス。
  • UsedShmemSegSize: 共有メモリセグメントのサイズ。

失敗モードと境界条件

  • VirtualAllocEx が NULL を返した場合、メモリ予約に失敗したとみなされる。
  • 予約されたアドレスが期待値（ShmemProtectiveRegion または UsedShmemSegAddr）と異なる場合、アドレス空間の競合が発生したとみなされ、失敗として扱われる。
  • 失敗時は FATAL ではなく LOG レベルでエラーを出力し、処理を継続する。

保守担当者向けメモ

  • 本関数は Windows 特有のメモリ管理ハックであり、共有メモリのアドレス固定を目的としている。
  • PROTECTIVE_REGION_SIZE は、共有メモリ領域の直前を保護するために使用される。
  • 予約失敗時に FATAL を使用しないのは、Postmaster プロセスを停止させないための設計である。
