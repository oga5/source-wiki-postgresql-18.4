compare

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_pool.c:144

目的

遺伝的アルゴリズムにおける個体（Chromosome）の適応度（worth）を比較し、qsort 等のソート関数で利用可能な形式で順序を決定します。

シグネチャまたは形状

static int compare(const void *arg1, const void *arg2)

入力と出力

  • **入力**: 比較対象となる2つの Chromosome 構造体へのポインタ（void * として受け取り、内部でキャスト）。
  • **出力**:
  • 0: chromo1->worth == chromo2->worth の場合。
  • 1: chromo1->worth > chromo2->worth の場合。
  • -1: chromo1->worth < chromo2->worth の場合。

前提条件と不変条件

  • 入力される arg1 および arg2 は、有効な Chromosome 構造体を指している必要があります。
  • Chromosome 構造体には worth フィールドが存在し、比較可能な値が格納されている必要があります。

アルゴリズムまたは振る舞い

1. 引数 arg1 と arg2 を Chromosome * 型にキャストします。 2. 両者の worth フィールドを比較します。 3. worth が小さいほど「優秀」と見なされるため、昇順ソートを行うとコストの低い順（優秀な順）に並びます。

呼び出し元と呼び出し先

  • **呼び出し元**: sort_pool 関数から qsort の比較関数として呼び出されます。
  • **呼び出し先**: 該当なし（純粋な比較ロジックのみ）。
  • 注記: ツールが提示した contrib/ 配下の compare への言及は、本シンボルとは別の関数を指すものであり、本シンボルの呼び出し元ではありません。

関連データ構造

  • Chromosome: 個体の遺伝情報と適応度（worth）を保持する構造体。

失敗モードと境界条件

  • arg1 または arg2 が NULL の場合、デリファレンスによりクラッシュします。
  • worth が DBL_MAX のような無効値である場合も、数値としてそのまま比較されます。

保守担当者向けメモ

  • 本関数は qsort 用の標準的な比較関数として実装されています。
  • 評価値が小さいほど優秀であるという前提に基づいているため、ソート順序を変更する際は注意してください。
