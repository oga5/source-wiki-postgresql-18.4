SharedRecordTableEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/typcache.c:211

目的

並列クエリ環境において、レコード型の情報を共有メモリ上で管理するためのハッシュテーブルエントリとして機能します。

シグネチャまたは形状

typedef struct SharedRecordTableEntry
{
	SharedRecordTableKey key;
} SharedRecordTableEntry;

入力と出力

  • 入力: SharedRecordTableKey を保持するコンテナ構造体。
  • 出力: なし（データ構造定義）。

前提条件と不変条件

  • SharedRecordTableKey を唯一のメンバとして持ち、ハッシュテーブルのキーとエントリが一体化した構造です。

アルゴリズムまたは振る舞い

  • 本構造体は、dshash（共有ハッシュテーブル）の要素として使用されます。
  • typcache.c 内の共有レコード型レジストリにおいて、型修飾子（typmod）とレコード型定義の対応関係を保持するために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/utils/cache/typcache.c:277 (サイズ計算)
  • src/backend/utils/cache/typcache.c:2235, 2947 (ポインタ変数宣言)
  • src/backend/utils/cache/typcache.c:2960, 3010 (型キャストによる代入)
  • 呼び出し先: 情報なし。

関連データ構造

  • SharedRecordTableKey: 本構造体が内包するキー構造体。
  • SharedRecordTypmodRegistry: 本構造体が格納される共有ハッシュテーブルの管理構造。

失敗モードと境界条件

  • 共有メモリの確保失敗や、ハッシュテーブルの衝突解決に関する境界条件は dshash の仕様に依存します。

保守担当者向けメモ

  • 本構造体は SharedRecordTableKey をラップしているのみですが、将来的に共有メモリ上で保持すべき追加のメタデータが必要になった場合、この構造体にメンバを追加することで拡張可能です。
