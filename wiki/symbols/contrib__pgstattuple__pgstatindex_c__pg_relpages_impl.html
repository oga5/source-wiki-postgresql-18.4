pg_relpages_impl

定義

  • 種別: function
  • 定義箇所: contrib/pgstattuple/pgstatindex.c:452

目的

指定されたリレーションの物理的なページ総数を取得する。

シグネチャまたは形状

int64 pg_relpages_impl(Relation rel)

入力と出力

  • **入力**: Relation rel（対象リレーションの構造体ポインタ）
  • **出力**: int64（リレーションを構成するブロックの総数）

前提条件と不変条件

  • 入力された rel は、RELKIND_HAS_STORAGE マクロにより、物理ストレージを持つリレーション（テーブルやインデックス等）であることが検証される。
  • 関数内で relation_close(rel, AccessShareLock) が実行されるため、呼び出し側は本関数を呼び出した時点でリレーションの所有権を放棄することになる。

アルゴリズムまたは振る舞い

1. RELKIND_HAS_STORAGE を使用して、対象リレーションがストレージを持つか確認する。ストレージを持たない場合、ERRCODE_WRONG_OBJECT_TYPE でエラーを報告する。 2. RelationGetNumberOfBlocks(rel) を呼び出し、リレーションの全ブロック数を取得する。 3. relation_close を呼び出し、AccessShareLock を解放してリレーションを閉じる。 4. 取得したブロック数を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • pg_relpages
  • pg_relpages_v1_5
  • pg_relpagesbyid
  • pg_relpagesbyid_v1_5

    • (いずれも contrib/pgstattuple/pgstatindex.c 内)

  • **呼び出し先**:
  • RelationGetNumberOfBlocks
  • relation_close
  • ereport (エラー発生時)

関連データ構造

  • Relation: PostgreSQLのシステムカタログにおけるリレーション情報を保持する構造体。

失敗モードと境界条件

  • ストレージを持たないリレーション（ビューや外部テーブルなど）が渡された場合、エラーが発生する。
  • 一時テーブルに対しては正常に動作する。

保守担当者向けメモ

  • 本関数はリレーションを閉じる責務を持つため、呼び出し側で二重にクローズしないよう注意が必要である。
