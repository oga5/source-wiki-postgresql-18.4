gistFreeBuildBuffers

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/gist/gistbuildbuffers.c:506

目的

GiST インデックスのバッファリングビルドで使用された一時ファイルリソースを解放し、ビルドバッファに関連する後処理を行う。

シグネチャまたは形状

void gistFreeBuildBuffers(GISTBuildBuffers *gfbb)

入力と出力

  • **入力**: gfbb - 解放対象の GISTBuildBuffers 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • gfbb は有効な GISTBuildBuffers 構造体を指している必要がある。
  • 本関数呼び出し後、gfbb->pfile が指すファイルハンドルは無効となる。

アルゴリズムまたは振る舞い

1. BufFileClose を呼び出し、gfbb->pfile に関連付けられた一時ファイルをクローズする。 2. メモリ上のデータ構造（ハッシュテーブルやバッファノード等）については、本関数では明示的に解放せず、それらが割り当てられているメモリコンテキストの解放に委ねる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistbuild.c:325
  • **呼び出し先**: BufFileClose（storage/buffile.h）

関連データ構造

  • GISTBuildBuffers: バッファリングビルドの状態を保持するメイン構造体。

失敗モードと境界条件

  • gfbb が NULL の場合、または gfbb->pfile が既にクローズされている場合は未定義の動作となる可能性がある。

保守担当者向けメモ

  • 本関数はリソースのクリーンアップの一部のみを担う。メモリ上のオブジェクトは gfbb->context が管理するメモリコンテキストの破棄時に一括解放される設計であるため、メモリリークの懸念がある場合はコンテキストのライフサイクルを確認すること。
