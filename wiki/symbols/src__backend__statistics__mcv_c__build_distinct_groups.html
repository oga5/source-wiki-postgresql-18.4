build_distinct_groups

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/mcv.c:423

目的

ソート済みのデータセットから重複する値の組み合わせをグループ化し、各グループの出現頻度をカウントした上で、頻度の高い順に並べ替えたリストを生成する。

シグネチャまたは形状

static SortItem *build_distinct_groups(int numrows, SortItem *items, MultiSortSupport mss, int *ndistinct)

入力と出力

  • **入力**:
  • numrows: 入力配列 items の要素数。
  • items: ソート済みの SortItem 配列。
  • mss: 比較操作を制御する MultiSortSupport 構造体。
  • **出力**:
  • 戻り値: 頻度順にソートされた SortItem の配列。
  • ndistinct: 検出された一意なグループ数（出力引数）。

前提条件と不変条件

  • 入力 items は、mss に基づいて昇順にソートされている必要がある。
  • multi_sort_compare が items に対して昇順の順序を維持していることを前提とする。

アルゴリズムまたは振る舞い

1. count_distinct_groups を呼び出し、一意なグループ数を事前に算出する。 2. 算出されたグループ数分のメモリを palloc で確保する。 3. ソート済み items を走査し、multi_sort_compare を用いて隣接する要素が異なる場合に新しいグループとして登録する。 4. 各グループの出現回数（count）をカウントする。 5. 最後に qsort_interruptible を使用し、compare_sort_item_count を介してグループを頻度の降順にソートする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/mcv.c 内の統計情報構築処理（例: 208行目付近）。
  • **呼び出し先**:
  • count_distinct_groups
  • multi_sort_compare
  • qsort_interruptible
  • compare_sort_item_count

関連データ構造

  • SortItem: 統計対象のデータ項目と出現回数を保持する構造体。
  • MultiSortSupport: 複数列の比較ロジックを保持する構造体。

失敗モードと境界条件

  • numrows が 0 の場合、groups[0] へのアクセスで境界外参照が発生する可能性がある（呼び出し元で numrows > 0 が保証されている必要がある）。
  • メモリ不足時には palloc がエラーをスローする。

保守担当者向けメモ

  • 本関数は items がソート済みであることを前提としているため、呼び出し元でソート順序が崩れていないことを確認すること。
  • Assert 文により、グループ化の整合性とソート順序の不変条件をチェックしている。
