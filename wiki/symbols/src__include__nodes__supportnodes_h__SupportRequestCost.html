SupportRequestCost

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/supportnodes.h:131

目的

特定のSQL関数呼び出しに対して、プランナーがコスト見積もり（起動コストおよびタプルあたりのコスト）を算出するためのリクエストを定義します。

シグネチャまたは形状

typedef struct SupportRequestCost
{
    NodeTag     type;
    /* 入力フィールド */
    struct PlannerInfo *root;
    Oid         funcid;
    Node       *node;
    /* 出力フィールド */
    Cost        startup;
    Cost        per_tuple;
} SupportRequestCost;

入力と出力

  • **入力**:
  • root: プランナーのインフラストラクチャ情報（NULLの可能性あり）。
  • funcid: コスト見積もりの対象となる関数のOID。
  • node: 関数を呼び出しているパースノード（NULLの可能性あり）。
  • **出力**:
  • startup: 関数の実行に伴う一回限りの起動コスト。
  • per_tuple: タプルごとに発生する評価コスト。

前提条件と不変条件

  • サポート関数がコスト見積もりを提供できない場合、NULLを返す必要があります。
  • 構造体自体は一時的なものであり、サポート関数内で永続的なメモリ確保を行う必要はありません。

アルゴリズムまたは振る舞い

1. プランナーまたは実行エンジンが最適化のためにコスト情報を必要とする際、本構造体を生成します。 2. 対象関数に紐付けられたサポート関数が本構造体を引数として呼び出されます。 3. サポート関数は、算出結果を startup および per_tuple フィールドに格納します。 4. 呼び出し元は、返された結果に基づきプランのコスト見積もりを調整します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/plancat.c にて本構造体が生成・利用されている証拠があります。また、src/test/regress/regress.c で型チェックが行われています。
  • **呼び出し先**: 本構造体はデータ保持用であり、特定の関数を呼び出すものではありません。サポート関数（SQLの prosupport 属性に登録された関数）が本構造体を受け取り、値を設定します。

関連データ構造

  • NodeTag: ノードの種類を識別するためのタグ。
  • PlannerInfo: プランナーのコンテキスト情報。
  • Cost: コスト計算に使用される型。

失敗モードと境界条件

  • サポート関数がリクエストを処理できない場合は NULL を返すことがプロトコルです。
  • root や node が NULL であっても、サポート関数は適切に処理を継続する必要があります。

保守担当者向けメモ

  • 本構造体はプランナーだけでなく実行時（Executor）にも利用されるため、プランナー専用のロジックと限定しないよう注意してください。
  • 将来的なリクエストタイプの追加に対して後方互換性を保つため、処理不能時はエラーではなく NULL を返す設計となっています。
