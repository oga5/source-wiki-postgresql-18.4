dshash_table_control

定義

  • 種別: struct
  • 定義箇所: src/backend/lib/dshash.c:83

目的

動的共有メモリ（DSA）領域上に配置される、共有ハッシュテーブルの制御ブロック。プロセス間で共有されるハッシュテーブルのメタデータと状態を保持する。

シグネチャまたは形状

typedef struct dshash_table_control
{
    dshash_table_handle handle;
    uint32      magic;
    dshash_partition partitions[DSHASH_NUM_PARTITIONS];
    int         lwlock_tranche_id;
    size_t      size_log2;
    dsa_pointer buckets;
} dshash_table_control;

入力と出力

  • 本構造体は、dsa_allocate を介して共有メモリ上に確保される。
  • size_log2 および buckets は、ハッシュテーブルの動的リサイズ時に更新される。

前提条件と不変条件

  • size_log2 および buckets への書き込みは、すべてのパーティションロックが保持されている場合にのみ許可される。
  • 読み取りは、いずれか一つのパーティションロックが保持されていれば安全に行える。

アルゴリズムまたは振る舞い

  • **リサイズ管理**: ハッシュテーブルの負荷が閾値を超えた際、全パーティションのロックを取得した上で size_log2 を更新し、新しいバケット配列（buckets）を割り当てる。
  • **並行制御**: partitions 配列を通じて、ロックの細分化とエントリ管理を行う。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/lib/dshash.c 内の dsa_allocate 呼び出し箇所（215行目）にてインスタンス化される。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数呼び出しの対象ではない。

関連データ構造

  • dshash_partition: パーティションごとのロックとエントリ数を管理。
  • dshash_table: バックエンドローカルな状態を保持する構造体。
  • dsa_pointer: 共有メモリ上のバケット配列を指すポインタ。

失敗モードと境界条件

  • 共有メモリ不足時には dsa_allocate が失敗する可能性がある。
  • リサイズ操作中に全パーティションロックを取得するため、頻繁なリサイズはシステム全体のパフォーマンス低下を招く。

保守担当者向けメモ

  • 共有メモリ上の構造体であるため、直接的なメモリポインタの保持は避け、必ず dsa_pointer を使用すること。
  • 複数のロックを取得する際は、デッドロックを回避するためパーティション番号の昇順で取得すること。
