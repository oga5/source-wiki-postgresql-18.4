pg_crc32c_armv8_available

定義

  • 種別: function
  • 定義箇所: src/port/pg_crc32c_armv8_choose.c:44

目的

実行環境の CPU が ARMv8 CRC 拡張命令をサポートしているかどうかを判定し、CRC-32C 計算の最適化実装を選択するための判断材料を提供します。

シグネチャまたは形状

static bool pg_crc32c_armv8_available(void)

入力と出力

  • 入力: なし
  • 出力: bool 型。サポートされている場合は true、それ以外（判定不能または非サポート）の場合は false を返します。

前提条件と不変条件

  • 本関数は、OS が提供する CPU 機能フラグ（HWCAP_CRC32 等）または sysctl インターフェースに依存します。
  • NetBSD 環境では、cpu0 の機能がシステム内の全 CPU を代表していると仮定して判定を行います。

アルゴリズムまたは振る舞い

実行環境に応じて以下のいずれかの手法で判定を行います。

1. **ELF Auxiliary Vector (HAVE_ELF_AUX_INFO または HAVE_GETAUXVAL)**:

  • AT_HWCAP または AT_HWCAP2 を参照し、HWCAP_CRC32 / HWCAP2_CRC32 ビットが立っているかを確認します。

    • 2. **NetBSD (__NetBSD__)**:

  • sysctlbyname を使用して machdep.cpu0.cpu_id (aarch64) または machdep.id_isar を取得します。
  • 取得したレジスタ値から CRC32 関連のビットフィールドを抽出し、値が 0 でない（サポートされている）かを判定します。

    • 3. **その他**:

  • 上記のいずれにも該当しない場合、false を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/pg_crc32c_armv8_choose.c 内の pg_comp_crc32c_choose 関数から呼び出されます。
  • 呼び出し先: OS 提供のシステムコール（elf_aux_info, getauxval, sysctlbyname）および memset を使用します。

関連データ構造

  • sysctlbuf: NetBSD 環境でレジスタ情報を格納するために使用される uint64 型のバッファ。
  • aarch64_sysctl_cpu_id: NetBSD (aarch64) でレジスタ構造を解釈するために使用される構造体。

失敗モードと境界条件

  • sysctlbyname が失敗した場合、または取得したデータのサイズが期待値と異なる場合は false を返します。
  • 判定ロジックがサポートされていない環境では、安全側に倒して false を返します。

保守担当者向けメモ

  • NetBSD における sysctl インターフェースは 32bit と 64bit で大きく異なるため、修正時は両アーキテクチャへの影響を確認してください。
  • 将来的に CRC32 拡張の新しい値が定義された場合、現在の「0 以外はサポート」という判定ロジックが有効であるか再評価が必要です。
