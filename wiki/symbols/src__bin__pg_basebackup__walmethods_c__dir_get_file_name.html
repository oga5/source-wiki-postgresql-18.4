dir_get_file_name

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/walmethods.c:101

目的

WAL保存用ディレクトリメソッドにおいて、圧縮アルゴリズムと一時的な接尾辞を考慮したファイルパス文字列を生成する。

シグネチャまたは形状

static char *dir_get_file_name(WalWriteMethod *wwmethod, const char *pathname, const char *temp_suffix)

入力と出力

  • **入力**:
  • wwmethod: WalWriteMethod 構造体へのポインタ。圧縮アルゴリズムの判定に使用。
  • pathname: 基本となるファイルパス。
  • temp_suffix: ファイル名に付与する一時的な接尾辞（NULL許容）。
  • **出力**:
  • 生成されたファイルパスを保持する char * 型のメモリ領域（呼び出し元で解放が必要）。

前提条件と不変条件

  • pg_malloc0 を使用して MAXPGPATH バイトのメモリを確保する。
  • wwmethod が有効なポインタであること。

アルゴリズムまたは振る舞い

1. MAXPGPATH サイズのメモリを確保する。 2. wwmethod->compression_algorithm を参照し、以下の拡張子を付与する。

  • PG_COMPRESSION_GZIP: .gz
  • PG_COMPRESSION_LZ4: .lz4
  • その他: 拡張子なし

    • 3. temp_suffix が指定されている場合、その文字列を末尾に連結する。 4. 構築された文字列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/walmethods.c 内のディレクトリメソッド実装（dir_open_for_write 等）から複数回呼び出されている。
  • **呼び出し先**:
  • pg_malloc0 (メモリ確保)
  • snprintf (文字列構築)

関連データ構造

  • WalWriteMethod: 圧縮設定を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には pg_malloc0 経由でプロセスが終了する可能性がある。
  • pathname と temp_suffix の合計長が MAXPGPATH を超える場合、snprintf により切り詰めが発生する可能性がある。

保守担当者向けメモ

  • 本関数はディレクトリメソッド専用である。tarメソッド用のパス生成には tar_get_file_name が別途存在する。
  • 戻り値のメモリは呼び出し側で管理（解放）される必要がある。
