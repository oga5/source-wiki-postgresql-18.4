CopyReadLineText

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfromparse.c:1233

目的

入力ストリームから1行分のデータを読み込み、line_buf へ転送する。テキスト形式およびCSV形式の解析をサポートする。

シグネチャまたは形状

static bool CopyReadLineText(CopyFromState cstate, bool is_csv)

入力と出力

  • **入力**:
  • cstate: CopyFromState 構造体（入力バッファおよび解析状態を保持）。
  • is_csv: CSVモードかどうかを示すフラグ。
  • **出力**:
  • 戻り値: bool 型。EOFに到達した場合は true、それ以外は false を返す。

前提条件と不変条件

  • 入力データは既にサーバーエンコーディングに変換済みであること。
  • マルチバイト文字の各バイトは上位ビットがセットされているため、改行やエスケープ文字がマルチバイトシーケンス内に混入することはない（バイト単位の処理が可能）。
  • CSVモードでは、クォート内の改行はデータの一部として扱われる。

アルゴリズムまたは振る舞い

1. **初期化**: CSVモードの場合、クォート文字とエスケープ文字を特定する。 2. **ループ処理**:

  • REFILL_LINEBUF および CopyLoadInputBuf を使用して、必要に応じて入力バッファを補充する。
  • バッファが空の場合はEOFとして終了する。
  • 1バイトずつ読み込み、CSVモードの場合はクォート状態（in_quote）とエスケープ状態（last_was_esc）を更新する。
  • 改行文字（\r または \n）を検出し、行末を判定する。
  • \r を検出した際、\r\n のペアであるかを確認し、必要に応じて eol_type を設定する。

    • 3. **終了**: 行末に到達した時点で処理を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfromparse.c 内の CopyReadLine から呼び出される。
  • **呼び出し先**: CopyLoadInputBuf、ereport（エラー報告用マクロ）。

関連データ構造

  • CopyFromState: COPY FROM 操作の全体的な状態を保持する構造体。

失敗モードと境界条件

  • **EOF**: 入力データが尽きた場合、true を返して終了する。
  • **不正な改行**: \r が単独で出現し、かつ期待される改行形式と一致しない場合、ERRCODE_BAD_COPY_FILE_FORMAT でエラーを報告する。
  • **バッファ補充**: IF_NEED_REFILL_AND_NOT_EOF_CONTINUE マクロにより、データ不足時の制御フローが管理される。

保守担当者向けメモ

  • is_csv が true の場合、CSV特有のステートマシンが動作する。クォートとエスケープが同一文字である場合の処理に注意すること。
  • cur_lineno の更新は、CSVモードにおけるクォート内改行を考慮した論理的な行数カウントとして実装されている。
