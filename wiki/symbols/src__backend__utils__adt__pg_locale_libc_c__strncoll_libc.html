strncoll_libc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_libc.c:567

目的

OSのlibcロケール機能を利用して、指定されたロケール設定に基づき2つの文字列を比較する。

シグネチャまたは形状

int strncoll_libc(const char *arg1, ssize_t len1, const char *arg2, ssize_t len2, pg_locale_t locale)

入力と出力

  • **入力**:
  • arg1, arg2: 比較対象の文字列。
  • len1, len2: 文字列の長さ（-1 の場合はNUL終端文字列として扱う）。
  • locale: COLLPROVIDER_LIBC を使用するロケール情報構造体。
  • **出力**:
  • int: strcoll_l の結果（比較結果）。

前提条件と不変条件

  • locale->provider が COLLPROVIDER_LIBC であること（Assert で検証）。
  • len1 または len2 が -1 でない場合、指定された長さ分のメモリ領域が有効であること。

アルゴリズムまたは振る舞い

1. 文字列の長さに応じて、スタック上の TEXTBUFLEN バッファ、または必要に応じてヒープ（palloc）から作業用メモリを確保する。 2. len が -1 でない場合、入力文字列を作業用バッファにコピーし、NUL終端を付与する。 3. strcoll_l を呼び出し、ロケールハンドル locale->info.lt を用いて比較を実行する。 4. ヒープを確保した場合は pfree で解放し、比較結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/pg_locale_libc.c 内の collate_methods_libc 構造体から参照されている。
  • **呼び出し先**: strcoll_l（libc関数）、palloc、pfree（PostgreSQLメモリ管理）。

関連データ構造

  • pg_locale_t: ロケール情報を保持する構造体。
  • collate_methods_libc: libc プロバイダーのメソッドを定義する構造体。

失敗モードと境界条件

  • bufsize1 + bufsize2 が TEXTBUFLEN を超える場合、ヒープメモリ確保が発生する。
  • len が -1 の場合はNUL終端文字列として処理される。

保守担当者向けメモ

  • 本関数は strcoll_l をラップするブリッジ層である。
  • TEXTBUFLEN（1024バイト）を超える文字列の処理において、ヒープ確保と解放のオーバーヘッドが発生する点に注意すること。
