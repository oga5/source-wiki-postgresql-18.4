secure_open_gssapi

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-gssapi.c:502

目的

GSSAPI（Kerberos）認証および暗号化セッションの初期化を行い、クライアントとのハンドシェイクを通じてセキュアな通信路を確立する。

シグネチャまたは形状

ssize_t secure_open_gssapi(Port *port);

入力と出力

  • **入力**: Port *port（接続状態を保持する構造体）
  • **出力**: 成功時は 0、失敗時は -1 を返す。非ブロッキングI/Oの待機が必要な場合は、read_or_wait 等の戻り値が伝播される。

前提条件と不変条件

  • Port 構造体内の gss メンバが初期化される。
  • 通信バッファ（PqGSSSendBuffer, PqGSSRecvBuffer, PqGSSResultBuffer）が PQ_GSS_AUTH_BUFFER_SIZE サイズで割り当てられる。
  • 環境変数 KRB5_KTNAME が設定されている場合、サーバーのキーファイルパスが反映される。

アルゴリズムまたは振る舞い

1. **初期化**: GSSAPI用のメモリ領域を確保し、送受信バッファを初期化する。 2. **認証ループ**: クライアントからのパケットを読み込み、gss_accept_sec_context を呼び出してコンテキストをネゴシエートする。 3. **パケット処理**: 4バイトの長さヘッダーを読み取り、続く暗号化データを復号する。 4. **応答**: 認証に必要な応答がある場合、ヘッダーを付与してクライアントへ送信する。 5. **委任**: 委任された認証情報がある場合、pg_store_delegated_credential を呼び出して保存する。 6. **完了**: コンテキスト確立が完了するまでこのプロセスを繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/backend_startup.c
  • **呼び出し先**: read_or_wait, gss_accept_sec_context, pg_store_delegated_credential, secure_raw_write, WaitLatchOrSocket 等。

関連データ構造

  • Port: 接続情報を保持する構造体。
  • pg_gssinfo: GSSAPIコンテキストや認証情報を保持する構造体。
  • PqGSSSendBuffer, PqGSSRecvBuffer: 通信用の静的バッファ。

失敗モードと境界条件

  • **メモリ不足**: バッファ確保失敗時に FATAL エラーを発生させる。
  • **パケットサイズ超過**: クライアントまたはサーバーが PQ_GSS_AUTH_BUFFER_SIZE を超えるパケットを送信しようとした場合、COMMERROR を発生させ接続を終了する。
  • **GSSAPIエラー**: gss_accept_sec_context がエラーを返した場合、エラーログを出力して終了する。

保守担当者向けメモ

  • 本関数は静的バッファを使用するため、単一の接続を前提としている。
  • エラー発生時は elog(FATAL) を避け、elog(COMMERROR) を使用して無限再帰を防止すること。
  • パケットサイズ制限はプロトコル仕様に直結するため、変更時はクライアントとの互換性に注意が必要。
