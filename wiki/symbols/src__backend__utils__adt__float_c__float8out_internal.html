float8out_internal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/float.c:536

目的

double 型の値を PostgreSQL の文字列形式（cstring）に変換します。

シグネチャまたは形状

char *float8out_internal(double num)

入力と出力

  • **入力**: num (変換対象の double 値)
  • **出力**: 変換された文字列を保持する palloc 済みの char * バッファ。

前提条件と不変条件

  • 呼び出し元は、返された文字列に対して pfree を行う責任があります。
  • palloc(32) により、出力文字列用に 32 バイトのメモリが確保されます。

アルゴリズムまたは振る舞い

1. 32 バイトのメモリを確保します。 2. GUC パラメータ extra_float_digits の値に基づき、以下のいずれかの変換を行います。

  • extra_float_digits > 0 の場合: double_to_shortest_decimal_buf を使用し、最短の 10 進数表現を生成します。
  • それ以外の場合: pg_strfromd を使用し、DBL_DIG + extra_float_digits の精度で文字列化します。

    • 3. 生成された文字列へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/float.c: float8out 関数経由で PG_RETURN_CSTRING に渡されます。
  • src/backend/utils/adt/geo_ops.c: 幾何学型の座標値出力に使用されます。
  • contrib/cube/cube.c: cube 型の文字列表現生成に使用されます。
  • **呼び出し先**:
  • palloc (メモリ確保)
  • double_to_shortest_decimal_buf (高精度変換)
  • pg_strfromd (標準的な浮動小数点変換)

関連データ構造

  • double: 入力値の型。
  • extra_float_digits: 出力精度を制御するグローバルな設定値。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • double が NaN や Inf の場合、pg_strfromd および double_to_shortest_decimal_buf の実装に依存した文字列が生成されます。

保守担当者向けメモ

  • 本関数は float8out の実体（guts）であり、SQL インターフェースと直接的な変換ロジックを分離しています。
  • 精度制御は extra_float_digits に依存しているため、出力形式の変更が必要な場合はこのパラメータの仕様を確認してください。
