llvm_create_types

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit.c:994

目的

PostgreSQLのデータ構造定義をLLVM IR上で利用可能な型としてロードし、グローバル変数へキャッシュする初期化処理を行います。

シグネチャまたは形状

static void llvm_create_types(void)

入力と出力

  • 入力: なし（pkglib_path および llvm_context を参照）
  • 出力: なし（副作用としてグローバル変数 llvm_types_module および各種 Type* / Attribute* 変数を初期化）

前提条件と不変条件

  • llvm_context が初期化済みであること。
  • pkglib_path に llvmjit_types.bc が存在すること。

アルゴリズムまたは振る舞い

1. pkglib_path から llvmjit_types.bc ファイルのパスを構築します。 2. LLVMCreateMemoryBufferWithContentsOfFile を使用してビットコードファイルをメモリバッファに読み込みます。 3. LLVMParseBitcodeInContext2 を呼び出し、llvm_context 内に llvm_types_module を生成します。 4. LLVMDisposeMemoryBuffer でバッファを解放します。 5. llvm_pg_var_type および load_return_type を使用して、特定の構造体や型を llvm_types_module から検索し、対応するグローバル変数に割り当てます。 6. LLVMGetNamedFunction を使用して、テンプレート関数（AttributeTemplate 等）を取得します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/jit/llvm/llvmjit.c 内の初期化フロー（llvm_session_initialize 等）から呼び出されます。
  • 呼び出し先: LLVM C API (LLVMCreateMemoryBufferWithContentsOfFile, LLVMParseBitcodeInContext2, LLVMDisposeMemoryBuffer, LLVMGetNamedFunction) および内部ヘルパー関数 (llvm_pg_var_type, load_return_type)。

関連データ構造

  • llvm_types_module: ロードされたビットコードを保持する LLVMModuleRef。
  • 各種 Type* グローバル変数: PostgreSQLの構造体に対応するLLVM型定義。

失敗モードと境界条件

  • ビットコードファイルの読み込み失敗時、またはパース失敗時に elog(ERROR) を発行し、PostgreSQLのエラー処理系へ制御を移します。

保守担当者向けメモ

  • 本関数は llvmjit_types.bc の構造に強く依存しています。src/backend/jit/llvm/llvmjit_types.c で定義されている型構造を変更した場合は、本関数でのマッピング処理も整合性を確認する必要があります。
