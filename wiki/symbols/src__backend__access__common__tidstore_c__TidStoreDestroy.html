TidStoreDestroy

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tidstore.c:316

目的

TidStore 構造体によって保持されているリソースを解放し、メモリから破棄します。

シグネチャまたは形状

void TidStoreDestroy(TidStore *ts)

入力と出力

  • **入力**: ts (TidStore *) - 破棄対象の TidStore インスタンスへのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • ts は有効な TidStore を指している必要があります。
  • 共有メモリ版の TidStore を破棄する場合、呼び出し元のバックエンドは、その TidStore に対して他のプロセスがアクセスしていないことを保証する必要があります。

アルゴリズムまたは振る舞い

1. TidStoreIsShared(ts) を用いて、ストレージが共有メモリかローカルメモリかを判定します。 2. **共有メモリの場合**:

  • shared_ts_free を呼び出し、内部のラディックツリーを解放します。
  • dsa_detach を呼び出し、DSA（Dynamic Shared Area）領域からデタッチします。

    • 3. **ローカルメモリの場合**:

  • local_ts_free を呼び出し、内部のラディックツリーを解放します。
  • MemoryContextDelete を呼び出し、ts->rt_context に割り当てられたメモリコンテキストを削除します。

    • 4. 最後に pfree(ts) を呼び出し、TidStore 構造体自体のメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/vacuumlazy.c
  • src/backend/commands/vacuumparallel.c
  • src/test/modules/test_tidstore/test_tidstore.c
  • **呼び出し先**:
  • shared_ts_free / local_ts_free
  • dsa_detach
  • MemoryContextDelete
  • pfree

関連データ構造

  • TidStore: 破棄対象のメイン構造体。
  • dsa_area: 共有メモリ管理に使用される領域。

失敗モードと境界条件

  • 無効なポインタを渡した場合の動作は未定義です。
  • 共有メモリ版において、他のプロセスが使用中に本関数を呼び出した場合の整合性は保証されません。

保守担当者向けメモ

  • 本関数を呼び出したバックエンドは、以降その TidStore を参照してはなりません。
  • 共有メモリ版の破棄は、すべての関連プロセスが TidStoreDetach を完了した後に実行されることを想定しています。
