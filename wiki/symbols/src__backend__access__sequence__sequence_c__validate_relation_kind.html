validate_relation_kind

定義

  • 種別: function
  • 定義箇所: src/backend/access/sequence/sequence.c:69

目的

対象のリレーションがシーケンス（RELKIND_SEQUENCE）であることを検証し、型安全性を保証します。

シグネチャまたは形状

static inline void validate_relation_kind(Relation r)

入力と出力

  • **入力**: Relation r（検証対象のリレーション構造体）
  • **出力**: なし（正常時は何も返さない）

前提条件と不変条件

  • r は有効な Relation 構造体を指している必要があります。
  • 本関数は static inline で定義されており、コンパイル時に最適化されることが期待されています。

アルゴリズムまたは振る舞い

1. 引数 r の rd_rel->relkind フィールドを参照します。 2. 値が RELKIND_SEQUENCE と一致するかを確認します。 3. 一致しない場合、ereport を呼び出し、ERROR レベルで処理を中断します。 4. エラー時には、対象リレーション名とサポートされていないリレーション種別を報告します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/sequence/sequence.c 内のシーケンス操作関数から呼び出されます。
  • ツールによるヒューリスティックな解析では、src/backend/access/index/indexam.c および src/backend/access/table/table.c にも同名の関数定義および呼び出しの痕跡が見られますが、これらが本関数と同一の実装を指しているか、あるいは同名の別関数であるかはソースコードの文脈上、断定できません。
  • **呼び出し先**:
  • ereport（エラー報告用マクロ）
  • RelationGetRelationName（リレーション名取得用マクロ）
  • errdetail_relkind_not_supported（エラー詳細生成用関数）

関連データ構造

  • Relation: utils/rel.h で定義されるリレーション構造体。
  • Form_pg_class（rd_rel 経由でアクセスされる構造体）

失敗モードと境界条件

  • **失敗モード**: rd_rel->relkind が RELKIND_SEQUENCE 以外の場合、ERRCODE_WRONG_OBJECT_TYPE エラーが発生し、トランザクション処理が中断されます。

保守担当者向けメモ

  • 本関数はシーケンス専用の検証ロジックです。他のリレーション種別に対する検証が必要な場合は、それぞれのモジュールで定義された関数を使用してください。
  • ツールが検出した他ファイル（indexam.c, table.c）での同名関数の存在は、各モジュールが独立して同様の検証ロジックを保持している可能性を示唆しています。
