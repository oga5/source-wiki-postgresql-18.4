pg_uuid_t

定義

  • 種別: struct
  • 定義箇所: src/include/utils/uuid.h:20

目的

PostgreSQLの uuid 型の物理的なデータ構造を保持し、システム定義の uuid_t との名称衝突を回避するために定義されています。

シグネチャまたは形状

typedef struct pg_uuid_t
{
	unsigned char data[UUID_LEN];
} pg_uuid_t;

入力と出力

  • 本構造体は、PG_GETARG_UUID_P マクロを介してSQL関数の引数から取得され、PG_RETURN_UUID_P マクロを介して結果として返されます。
  • 内部データは unsigned char の配列として保持されます。

前提条件と不変条件

  • UUID_LEN（16バイト）の固定長データ構造です。
  • メモリ割り当てには通常 palloc が使用されます。

アルゴリズムまたは振る舞い

  • 本構造体自体はデータ保持のためのコンテナであり、比較や生成などのロジックは src/backend/utils/adt/uuid.c 内の関数によって提供されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照例）**:
  • contrib/btree_gin/btree_gin.c: メモリ確保に使用。
  • contrib/btree_gist/btree_uuid.c: 比較関数 uuid_internal_cmp の引数や、DatumGetUUIDP を介した変換に使用。
  • contrib/uuid-ossp/uuid-ossp.c: uuid_generate_v35_internal の引数として使用。
  • **呼び出し先**:
  • 本構造体はデータ構造定義であるため、直接的な関数呼び出しは行いません。

関連データ構造

  • UUID_LEN: 構造体のサイズを規定する定数。
  • Datum: PostgreSQLの汎用データ型。UUIDPGetDatum および DatumGetUUIDP を通じて相互変換されます。

失敗モードと境界条件

  • 固定長（16バイト）であるため、データ長に関する境界条件は構造体定義レベルでは発生しません。メモリ確保失敗時は palloc の挙動に従います。

保守担当者向けメモ

  • システムヘッダーの uuid_t との衝突を避けるため、必ず pg_uuid_t を使用してください。
  • ロジックの変更が必要な場合は、本ヘッダーではなく src/backend/utils/adt/uuid.c を拡張してください。
