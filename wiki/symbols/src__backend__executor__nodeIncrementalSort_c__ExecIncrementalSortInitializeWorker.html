ExecIncrementalSortInitializeWorker

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIncrementalSort.c:1218

目的

パラレルクエリ実行時、各ワーカープロセスにおいて IncrementalSortState ノードの共有情報を初期化し、自身がワーカーであることを設定する。

シグネチャまたは形状

void ExecIncrementalSortInitializeWorker(IncrementalSortState *node, ParallelWorkerContext *pwcxt)

入力と出力

  • node: 初期化対象の IncrementalSortState 構造体へのポインタ。
  • pwcxt: パラレル実行コンテキスト。
  • 出力: なし（node 構造体のメンバを更新）。

前提条件と不変条件

  • node は有効な IncrementalSortState を指している必要がある。
  • pwcxt は有効なパラレルコンテキストである必要がある。

アルゴリズムまたは振る舞い

1. pwcxt->toc から、現在のプランノード ID (node->ss.ps.plan->plan_node_id) に対応する共有情報を shm_toc_lookup を用いて検索し、node->shared_info に格納する。 2. node->am_worker フラグを true に設定し、当該ノードがワーカープロセスで動作していることを示す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execParallel.c:1393
  • 呼び出し先: shm_toc_lookup（PostgreSQL 共有メモリ管理機能）

関連データ構造

  • IncrementalSortState: インクリメンタルソートノードの実行状態を保持する構造体。
  • ParallelWorkerContext: パラレルクエリ実行時の共有コンテキスト。

失敗モードと境界条件

  • shm_toc_lookup が指定された ID を見つけられない場合、true 引数によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はパラレルクエリのワーカー初期化プロセスの一部である。共有メモリ上の統計情報等にアクセスする前に必ず呼び出される必要がある。
