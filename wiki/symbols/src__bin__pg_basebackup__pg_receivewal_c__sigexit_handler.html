sigexit_handler

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_receivewal.c:616

目的

SIGINT および SIGTERM シグナルを受信した際に、プログラムを安全に終了させるためのフラグを設定する。

シグネチャまたは形状

static void sigexit_handler(SIGNAL_ARGS)

※ SIGNAL_ARGS は PostgreSQL のシグナルハンドラ用マクロ。

入力と出力

  • 入力: SIGNAL_ARGS（シグナル情報）
  • 出力: なし（副作用としてグローバル変数を更新）

前提条件と不変条件

  • グローバル変数 time_to_stop が定義されていること。
  • 本関数がシグナルハンドラとして登録されていること。

アルゴリズムまたは振る舞い

1. グローバル変数 time_to_stop を true に設定する。 2. これにより、メインループやストリーミング処理が終了条件を検知し、処理を中断する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（シグナル発生時）
  • 呼び出し先: なし
  • 補足: pg_receivewal.c 内で pqsignal(SIGINT, sigexit_handler) および pqsignal(SIGTERM, sigexit_handler) により登録される。また、pg_recvlogical.c でも同様のハンドラとして利用されている。

関連データ構造

  • time_to_stop: 終了要求状態を保持するグローバルなブール型変数。

失敗モードと境界条件

  • シグナルハンドラ内での実行であるため、非同期シグナル安全（async-signal-safe）な操作のみを行う必要がある。本関数は単純なフラグ代入のみを行うため、この要件を満たす。

保守担当者向けメモ

  • 本関数は pg_receivewal と pg_recvlogical の両方で利用されている。変更を加える際は、両方のツールにおける終了処理の整合性に注意すること。
