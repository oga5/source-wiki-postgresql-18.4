ScanTypeControl

定義

  • 種別: enum
  • 定義箇所: src/backend/optimizer/path/indxpath.c:46

目的

インデックスパス生成時において、対象のインデックスがサポートすべきスキャン方式（amgettuple または amgetbitmap）を制御・指定するために使用されます。

シグネチャまたは形状

typedef enum
{
    ST_INDEXSCAN,   /* amgettuple をサポートする必要がある */
    ST_BITMAPSCAN,  /* amgetbitmap をサポートする必要がある */
    ST_ANYSCAN,     /* どちらでも許容される */
} ScanTypeControl;

入力と出力

  • 本シンボルは列挙型であり、関数への引数として渡されることで、パス生成ロジックの動作を切り替えるフラグとして機能します。

前提条件と不変条件

  • ST_INDEXSCAN を指定した場合、対象のインデックスアクセスメソッドは amgettuple を実装している必要があります。
  • ST_BITMAPSCAN を指定した場合、対象のインデックスアクセスメソッドは amgetbitmap を実装している必要があります。

アルゴリズムまたは振る舞い

  • インデックスパス生成プロセスにおいて、要求されるスキャン方式を明示することで、不適切なスキャン方式の生成を抑制します。
  • ST_ANYSCAN が指定された場合、プランナはインデックスの機能に応じて柔軟にスキャン方式を選択可能です。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/path/indxpath.c 内のパス生成関数（get_index_paths 等）において、引数として渡されることが確認されています。
  • 呼び出し先: 本シンボル自体は列挙型であり、直接的な呼び出し先はありません。

関連データ構造

  • IndexPath: 通常のインデックススキャン用パス構造体。
  • BitmapHeapPath: ビットマップスキャン用パス構造体。

失敗モードと境界条件

  • インデックスが指定された ScanTypeControl に対応するアクセスメソッド（amgettuple / amgetbitmap）を欠いている場合、パス生成時に実行時エラーや不正なプラン生成を招く可能性があります。

保守担当者向けメモ

  • 本列挙型は indxpath.c 内のパス生成ロジックのインターフェースの一部です。新しいスキャン方式が追加される場合、この列挙型に値を追加し、各パス生成関数での条件分岐を更新する必要があります。
