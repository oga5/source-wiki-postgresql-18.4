bitno_to_blkno

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hashovfl.c:34

目的

ハッシュインデックスのオーバーフローページ管理において、ビットマップ上の0から始まるビット番号を、物理的な絶対ブロック番号へ変換する。

シグネチャまたは形状

static BlockNumber bitno_to_blkno(HashMetaPage metap, uint32 ovflbitnum)

入力と出力

  • **入力**:
  • metap: ハッシュインデックスのメタページへのポインタ。
  • ovflbitnum: 変換対象となる0ベースのビット番号。
  • **出力**:
  • 変換後の絶対ブロック番号 (BlockNumber)。

前提条件と不変条件

  • metap->hashm_ovflpoint は1以上である必要がある。
  • ovflbitnum は、metap->hashm_spares 配列によって定義されるスプリットポイントの範囲内で適切にマッピングされる。

アルゴリズムまたは振る舞い

1. ovflbitnum を1ベースのページ番号に変換する。 2. metap->hashm_spares 配列を走査し、現在のスプリットポイント (hashm_ovflpoint) に基づいて、該当するページがどのスプリット範囲に属するかを特定する。 3. _hash_get_totalbuckets(i) を使用して、特定したスプリットポイント以前に存在するバケットページ数を取得する。 4. 取得したバケットページ数に1ベースのページ番号を加算し、絶対ブロック番号として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/hash/hashovfl.c 内の複数箇所（オーバーフローページの検索や新規バッファ取得時）。
  • **呼び出し先**:
  • _hash_get_totalbuckets（インライン関数またはマクロと推測されるが、本ソース内では定義元不明）。

関連データ構造

  • HashMetaPage: ハッシュインデックスのメタデータ構造体。hashm_ovflpoint および hashm_spares を保持する。

失敗モードと境界条件

  • ovflbitnum が不正な値の場合、hashm_spares の範囲外を参照する可能性がある。
  • _hash_get_totalbuckets の計算結果が BlockNumber の範囲を超える場合の挙動は、本関数内では明示されていない。

保守担当者向けメモ

  • 本関数は _hash_ovflblkno_to_bitno と対になる逆変換関数である。
  • スプリットポイントの計算ロジックは hashm_spares 配列の構造に強く依存しているため、メタページのレイアウト変更時には注意が必要である。
