TestFunc

定義

  • 種別: typedef
  • 定義箇所: src/common/unicode/case_test.c:35

目的

PostgreSQL の Unicode 大文字・小文字変換関数のテストにおいて、変換ロジックを抽象化するための関数ポインタ型です。

シグネチャまたは形状

typedef size_t (*TestFunc) (char *dst, size_t dstsize, const char *src, ssize_t srclen);

入力と出力

  • **入力**:
  • dst: 変換結果を格納するバッファへのポインタ。
  • dstsize: dst バッファのサイズ。
  • src: 変換対象の文字列。
  • srclen: src の長さ（バイト数）。
  • **出力**:
  • 変換後の文字列長（size_t）を返します。

前提条件と不変条件

  • src は有効な文字列である必要があります。
  • 変換関数は、指定された dstsize の範囲内で変換を行い、結果の長さを返す必要があります。

アルゴリズムまたは振る舞い

この型は、tfunc_lower、tfunc_upper、tfunc_title、tfunc_fold といった具体的な変換関数を統一的に扱うために使用されます。テスト実行時にこれらの関数を TestFunc 型として受け取り、test_convert 関数内で呼び出すことで、多様な変換ロジックを共通のテストフレームワークで検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/unicode/case_test.c 内の test_convert 関数が、引数として TestFunc を受け取り呼び出します。
  • **呼び出し先**: 情報なし（この型は関数ポインタの定義であり、直接呼び出し先を持つものではありません）。

関連データ構造

  • tfunc_lower, tfunc_upper, tfunc_title, tfunc_fold: この型に適合する具体的な変換関数群。

失敗モードと境界条件

  • dstsize が変換結果を格納するのに不十分な場合、各実装関数がどのように振る舞うか（切り詰めやエラー報告）は、各実装に依存します。
  • srclen が負の値や不正な値である場合の挙動は、各実装関数の仕様に従います。

保守担当者向けメモ

  • 新しい変換ロジックを追加する際は、このシグネチャに適合させることで test_convert による網羅的なバッファ境界テストを再利用可能です。
