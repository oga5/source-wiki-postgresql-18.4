PLy_init_plpy

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_plpymodule.c:139

目的

PL/Python 拡張において、Python 側の plpy モジュールを初期化し、PostgreSQL の実行環境と統合するための設定を行う。

シグネチャまたは形状

void PLy_init_plpy(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として Python インタプリタ内の __main__ モジュールに plpy が登録される）

前提条件と不変条件

  • Python インタプリタが既に初期化されていること。
  • PostgreSQL の SPI（Server Programming Interface）が利用可能な状態であること。

アルゴリズムまたは振る舞い

1. **型初期化**: PLy_plan_init_type, PLy_result_init_type, PLy_subtransaction_init_type, PLy_cursor_init_type を順次呼び出し、PL/Python で使用する各 Python 型オブジェクトを初期化する。 2. **モジュール作成**: PyModule_Create を呼び出し、PLy_module 定義に基づいて plpy モジュールを生成する。 3. **名前空間への登録**:

  • __main__ モジュールを取得する。
  • plpy モジュールを取得し、__main__ モジュールの辞書に plpy として登録する。

    • 4. **エラーチェック**: モジュールの取得失敗や Python 側の例外発生時には PLy_elog(ERROR, ...) を通じて PostgreSQL 側のエラーとして報告する。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpython/plpy_main.c:123
  • 呼び出し先:
  • PLy_plan_init_type, PLy_result_init_type, PLy_subtransaction_init_type, PLy_cursor_init_type
  • PyModule_Create, PyImport_AddModule, PyModule_GetDict, PyDict_SetItemString, PyErr_Occurred（Python C API）
  • PLy_elog

関連データ構造

  • PLy_module: plpy モジュールの定義構造体。

失敗モードと境界条件

  • plpy モジュールのインポートに失敗した場合、PLy_elog(ERROR, ...) により PostgreSQL 側でエラーが発生し、処理が中断される。
  • Python 側で例外が発生している場合、PyErr_Occurred() により検知され、同様にエラーとして処理される。

保守担当者向けメモ

  • 本関数は Python C API を使用しているため、参照カウントの管理に注意が必要である。
  • PLy_module の定義内容や、__main__ への登録ロジックを変更する場合は、Python のモジュールロード順序に影響を与える可能性がある。
