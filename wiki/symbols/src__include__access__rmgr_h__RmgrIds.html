RmgrIds

定義

  • 種別: enum
  • 定義箇所: src/include/access/rmgr.h:25

目的

PostgreSQLのWAL（Write-Ahead Log）を処理する各リソースマネージャ（Resource Manager）を識別するための列挙型です。

シグネチャまたは形状

typedef enum RmgrIds
{
    /* access/rmgrlist.h の内容が展開される */
    RM_NEXT_ID
} RmgrIds;

入力と出力

  • 入力: access/rmgrlist.h 内の定義。
  • 出力: RmgrIds 列挙型の各メンバおよび終端を示す RM_NEXT_ID。

前提条件と不変条件

  • RmgrId は uint8 型として扱われるため、リソースマネージャの総数は最大256個に制限されます。
  • rmgrlist.h を介して定義が生成されるため、直接この列挙型を編集するのではなく、rmgrlist.h を更新する必要があります。

アルゴリズムまたは振る舞い

1. access/rmgrlist.h を PG_RMGR マクロを用いて展開し、組み込みのリソースマネージャIDを列挙します。 2. 最後に RM_NEXT_ID を配置することで、定義されたIDの総数や範囲の境界を特定可能にします。 3. 実行時には、このIDをキーとして RmgrTable（src/backend/access/transam/rmgr.c）から対応するメソッド（redo, desc 等）をルックアップします。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • 呼び出し先: access/rmgrlist.h をインクルードして定義を構築します。

関連データ構造

  • RmgrId: RmgrIds の各要素を保持する uint8 型のエイリアス。
  • RmgrTable: RmgrIds をインデックスとして使用するリソースマネージャのコールバックテーブル。

失敗モードと境界条件

  • IDの枯渇: uint8 の上限（256）を超えるリソースマネージャの定義は不可能です。
  • 範囲外のID: RM_MIN_CUSTOM_ID 未満のIDをカスタムリソースマネージャとして使用した場合、組み込みIDと衝突するリスクがあります。

保守担当者向けメモ

  • 新しいリソースマネージャを追加する際は、rmgrlist.h にエントリを追加してください。RmgrIds は自動的に更新されます。
  • 拡張機能でカスタムWALを使用する場合は、必ず RM_MIN_CUSTOM_ID 以上の値を使用するようにしてください。
