pull_vars_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/util/var.c:46

目的

式ツリーの走査中に、特定のクエリレベルに属する Var ノードや PlaceHolderVar ノードを収集するための状態保持用コンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    List   *vars;           /* 収集されたノードのリスト */
    int     sublevels_up;   /* 走査対象とするクエリのネストレベル */
} pull_vars_context;

入力と出力

  • **入力**: 走査対象の式ツリーおよび、収集対象の sublevels_up 値。
  • **出力**: vars フィールドに格納された、条件に合致するノードのリスト。

前提条件と不変条件

  • vars は NIL で初期化される必要があります。
  • sublevels_up は、走査対象のクエリレベルと一致するように設定されている必要があります。

アルゴリズムまたは振る舞い

  • expression_tree_walker を利用した走査において、この構造体をコンテキストとして渡します。
  • pull_vars_walker 関数がツリーを再帰的に探索し、現在の sublevels_up と一致する Var ノードを見つけると、そのポインタを vars リストに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/var.c 内の関数（例: pull_vars_of_level 等）からインスタンス化され、pull_vars_walker に渡されます。
  • **呼び出し先**: pull_vars_walker がこの構造体の vars リストを更新します。

関連データ構造

  • List: 収集されたノードを保持するために使用されます。
  • Var, PlaceHolderVar: 収集対象となるノード型です。

失敗モードと境界条件

  • メモリ不足により List への追加が失敗する可能性があります。
  • sublevels_up の設定が不適切な場合、意図しないレベルの変数が収集されるか、あるいは何も収集されない可能性があります。

保守担当者向けメモ

  • 本構造体は expression_tree_walker パターン専用のコンテキストです。
  • PlaceHolderVar も Var と同様に扱われるため、収集対象に含まれる点に注意してください。
