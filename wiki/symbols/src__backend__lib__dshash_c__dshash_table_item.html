dshash_table_item

定義

  • 種別: struct
  • 定義箇所: src/backend/lib/dshash.c:44

目的

動的共有メモリ（DSA）上に構築されたハッシュテーブルにおいて、個々のエントリを管理するためのヘッダー構造体です。

シグネチャまたは形状

struct dshash_table_item
{
    dsa_pointer next;    /* 同一バケット内の次のアイテムへのポインタ */
    dshash_hash hash;    /* 再計算を避けるためのハッシュ値 */
    /* ユーザー定義のエントリオブジェクトがこの直後に続く */
};

入力と出力

  • 本構造体自体はデータ保持用であり、関数ではないため入力・出力はありません。
  • ユーザーデータは本構造体の直後に配置され、ENTRY_FROM_ITEM マクロ等を用いてアクセスされます。

前提条件と不変条件

  • next フィールドは dsa_pointer であり、共有メモリ上の有効な領域を指す必要があります。
  • ユーザーデータは dshash_table_item の直後に配置されるため、メモリ配置は MAXALIGN で調整されている必要があります。

アルゴリズムまたは振る舞い

  • ハッシュテーブルのバケット内連結リストのノードとして機能します。
  • hash フィールドを保持することで、衝突解決時のキー比較コストを削減します。
  • 共有メモリ上で動作するため、直接的なポインタではなく dsa_pointer を使用してリンクを管理します。

呼び出し元と呼び出し先

  • 呼び出し元: dshash.c 内のバケット操作関数（find_in_bucket, insert_into_bucket 等）が本構造体を操作します。
  • 呼び出し先: 本構造体自体はデータ構造であり、特定の関数を呼び出すことはありません。

関連データ構造

  • dshash_table: ハッシュテーブル全体の管理構造体。
  • dsa_pointer: 共有メモリ上のアドレスを指すための抽象化されたポインタ。

失敗モードと境界条件

  • 共有メモリが枯渇した場合、本構造体を含むエントリの確保に失敗します。
  • dsa_pointer が無効な場合、リストの走査時に不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体の直後にユーザーデータを配置する設計のため、構造体のサイズ変更やフィールド追加を行う際は、ENTRY_FROM_ITEM および ITEM_FROM_ENTRY マクロの計算式（MAXALIGN の使用箇所）に影響がないか確認してください。
  • 共有メモリ上のデータ構造であるため、デバッグ時には dsa_pointer の指す先が適切にマッピングされているか注意が必要です。
