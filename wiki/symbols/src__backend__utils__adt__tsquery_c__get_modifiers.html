get_modifiers

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery.c:113

目的

tsquery のトークンに付与された修飾子（重み付けおよびプレフィックス指定）を解析し、内部表現へ変換する。

シグネチャまたは形状

char *get_modifiers(char *buf, int16 *weight, bool *prefix)

入力と出力

  • **入力**:
  • buf: 解析対象の文字列バッファ。
  • weight: 重み付け情報を格納する int16 へのポインタ。
  • prefix: プレフィックス指定（*）の有無を格納する bool へのポインタ。
  • **出力**:
  • 戻り値: 修飾子の解析が終了した後の、バッファ内の次の位置を指すポインタ。

前提条件と不変条件

  • weight および prefix は、関数呼び出し時にそれぞれ 0 および false に初期化される。
  • buf が : で始まらない場合、修飾子なしとみなされ、入力された buf がそのまま返される。

アルゴリズムまたは振る舞い

1. buf が : で始まるかを確認し、そうでなければ即座に終了する。 2. : 以降の文字を順次走査し、以下の修飾子を判定する。

  • a / A: weight の第4ビット（1 << 3）をセット。
  • b / B: weight の第3ビット（1 << 2）をセット。
  • c / C: weight の第2ビット（1 << 1）をセット。
  • d / D: weight の第1ビット（1）をセット。
  • *: prefix を true に設定。

    • 3. 認識できない文字が現れた時点で走査を停止し、その位置を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsquery.c 内の解析処理（state->buf = get_modifiers(...)）から呼び出される。
  • **呼び出し先**: t_iseq（マクロ）、pg_mblen_cstr。

関連データ構造

  • int16: 重み付けビットマスクの保持に使用。

失敗モードと境界条件

  • マルチバイト文字が含まれる場合、pg_mblen_cstr(buf) == 1 の条件により、シングルバイト文字のみが修飾子として処理される。
  • 認識できない文字が入力された場合、エラーにはならず、その文字の位置で解析を終了して制御を戻す。

保守担当者向けメモ

  • 本関数は tsquery の構文解析において、トークンの直後に続く修飾子を抽出する役割を持つ。
  • 重み付けのビット割り当て（a=8, b=4, c=2, d=1）は、tsvector の重み付け定義と整合している必要がある。
