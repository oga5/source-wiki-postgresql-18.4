PGEventResultCopy

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/libpq-events.h:58

目的

PGresult オブジェクトが複製される際に、イベントハンドラへ通知を行うためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	const PGresult *src;
	PGresult   *dest;
} PGEventResultCopy;

入力と出力

  • src: 複製元の PGresult オブジェクト（読み取り専用）。
  • dest: 複製先の PGresult オブジェクト。

前提条件と不変条件

  • 本構造体は、PGresult のコピー処理に関連するイベントが発生した際に、イベントハンドラへ渡される引数の一部として使用されます。
  • src および dest は有効な PGresult へのポインタである必要があります。

アルゴリズムまたは振る舞い

libpq 内部で PGresult が複製される際、登録されたイベントハンドラに対してこの構造体が渡されます。ハンドラはこの構造体を参照し、必要に応じて src に関連付けられたカスタムデータ（PQresultInstanceData 等で管理されるもの）を dest に引き継ぐなどの処理を行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-exec.c 内の PGresult 複製処理に関連する箇所で本構造体がインスタンス化されます。
  • 呼び出し先: 登録済みの PGEventProc 型のイベントハンドラ関数。

関連データ構造

  • PGresult: libpq のクエリ結果を保持する構造体。
  • PGEventProc: イベントハンドラの関数ポインタ型。

失敗モードと境界条件

  • src または dest が NULL の場合の挙動については、libpq のイベント処理ロジックに依存します。
  • イベントハンドラ内でのメモリ割り当てに失敗した場合、適切なエラーハンドリングを行わないとメモリリークや不正な状態を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は PGresult のライフサイクル管理の一部です。イベントハンドラ内で dest に対してカスタムデータを設定する際は、PQsetResultInstanceData を使用してください。
