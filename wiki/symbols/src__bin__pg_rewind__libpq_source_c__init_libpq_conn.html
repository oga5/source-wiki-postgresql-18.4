init_libpq_conn

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/libpq_source.c:109

目的

リモートの PostgreSQL サーバーとの接続を pg_rewind のデータ取得に適した状態に初期化し、ファイル取得用のプリペアドステートメントを準備する。

シグネチャまたは形状

static void init_libpq_conn(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn - 初期化対象となる libpq 接続ハンドル。
  • **出力**: なし（副作用として接続状態を変更し、サーバー側でステートメントを準備する）。

前提条件と不変条件

  • 接続 conn は有効であること。
  • サーバー側で full_page_writes が有効であること（無効な場合は致命的エラーとして終了する）。

アルゴリズムまたは振る舞い

1. **タイムアウトの無効化**: 以下のセッション設定を 0（無制限）に設定する。

  • statement_timeout, lock_timeout, idle_in_transaction_session_timeout, transaction_timeout

    • 2. **読み取り専用モード**: default_transaction_read_only を on に設定する。 3. **セキュリティ設定**: ALWAYS_SECURE_SEARCH_PATH_SQL を実行し、search_path を安全な値にリセットする。 4. **整合性チェック**: SHOW full_page_writes を実行し、値が on であることを確認する。 5. **ステートメント準備**: fetch_chunks_stmt という名前で、pg_read_binary_file を呼び出す SQL を準備する。このステートメントは、パス、開始位置、長さを引数として受け取り、バイナリチャンクを取得する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/libpq_source.c 内の init_libpq_source から呼び出される。
  • **呼び出し先**: run_simple_command, run_simple_query, PQexec, PQprepare, pg_fatal 等。

関連データ構造

  • PGconn: libpq の接続オブジェクト。
  • PGresult: クエリ実行結果を保持する構造体。

失敗モードと境界条件

  • **致命的エラー (pg_fatal)**:
  • search_path のクリアに失敗した場合。
  • full_page_writes が on ではない場合。
  • fetch_chunks_stmt の準備に失敗した場合。

保守担当者向けメモ

  • 本関数は接続の所有権を変更せず、設定の適用とステートメントの準備のみを行う。
  • fetch_chunks_stmt で使用される pg_read_binary_file は、リモートサーバー上のファイル読み取り権限に依存する。
