SyncRepUpdateSyncStandbysDefined

定義

  • 種別: function
  • 定義箇所: src/backend/replication/syncrep.c:963

目的

同期レプリケーションの設定変更（synchronous_standby_names）を共有メモリの状態に反映し、待機中のバックエンドの整合性を維持する。

シグネチャまたは形状

void SyncRepUpdateSyncStandbysDefined(void)

入力と出力

  • **入力**: なし（内部的に SyncStandbysDefined() および共有メモリ WalSndCtl を参照）。
  • **出力**: なし（共有メモリ WalSndCtl->sync_standbys_status を更新）。

前提条件と不変条件

  • SyncRepLock を排他モードで取得して共有メモリを保護する。
  • 待機キューは常にLSN順にソートされている必要がある。

アルゴリズムまたは振る舞い

1. 現在の同期スタンバイ定義状態と、共有メモリ上の sync_standbys_status を比較する。 2. 状態に変更がある場合、SyncRepLock を取得する。 3. 同期スタンバイが未定義（空）になった場合、待機中の全バックエンドを SyncRepWakeQueue で解放する。 4. WalSndCtl->sync_standbys_status を更新し、同期スタンバイが定義されているかどうかのフラグを反映させる。このフラグは、バックエンドが誤って待機キューに入るのを防ぐインターロックとして機能する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/checkpointer.c:1380（直接の呼び出しを確認）。
  • **呼び出し先**: SyncStandbysDefined(), LWLockAcquire(), LWLockRelease(), SyncRepWakeQueue(), Assert()。

関連データ構造

  • WalSndCtl: 同期レプリケーションのグローバル状態を保持する共有メモリ構造体。
  • SyncRepLock: 同期レプリケーションの待機キューおよび状態を保護するLWLock。

失敗モードと境界条件

  • **競合状態の防止**: 同期スタンバイが定義されていない状態でバックエンドが待機キューに入り、永久にウェイクアップされない事態を、sync_standbys_status の更新とインターロックによって防ぐ。
  • **初期化の重複回避**: SYNC_STANDBY_INIT フラグにより、初期化処理が二重に行われないよう制御されている。

保守担当者向けメモ

  • 本関数は、GUC設定変更時にバックエンドが永久待機に陥るのを防ぐための重要な同期ポイントである。
  • SyncRepWaitForLSN() との整合性が極めて重要であるため、変更時は必ず待機キューの解放ロジックを確認すること。
