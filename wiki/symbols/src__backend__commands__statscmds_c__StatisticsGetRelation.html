StatisticsGetRelation

定義

  • 種別: function
  • 定義箇所: src/backend/commands/statscmds.c:936

目的

統計オブジェクトのOIDから、その統計情報が定義されているテーブル（リレーション）のOIDを取得する。

シグネチャまたは形状

Oid StatisticsGetRelation(Oid statId, bool missing_ok)

入力と出力

  • **入力**:
  • statId: 検索対象の統計オブジェクトのOID。
  • missing_ok: true の場合、統計オブジェクトが見つからなくてもエラーにせず InvalidOid を返す。false の場合はエラーを投げる。
  • **出力**:
  • 対象テーブルのOID。見つからない場合は InvalidOid（missing_ok が true の場合）。

前提条件と不変条件

  • STATEXTOID システムキャッシュを使用して検索を行う。
  • 取得したタプルから抽出した stxrelid を返す。
  • 正常終了時には ReleaseSysCache を呼び出し、取得したタプルを解放する。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を用いて STATEXTOID から統計オブジェクトのタプルを検索する。 2. タプルが無効な場合:

  • missing_ok が true なら InvalidOid を返す。
  • false なら elog(ERROR) で処理を中断する。

    • 3. タプルが有効な場合、GETSTRUCT で Form_pg_statistic_ext 構造体を取得する。 4. 取得した構造体の oid が入力された statId と一致することを Assert で確認する。 5. stxrelid フィールドを結果として保持する。 6. ReleaseSysCache でタプルを解放し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c:15551 (直接の呼び出しを確認)
  • **呼び出し先**:
  • SearchSysCache1
  • HeapTupleIsValid
  • GETSTRUCT
  • ReleaseSysCache
  • elog

関連データ構造

  • Form_pg_statistic_ext: pg_statistic_ext システムカタログのタプル構造体。stxrelid フィールドを保持する。

失敗モードと境界条件

  • 指定された statId が存在しない場合、missing_ok が false であればエラー（ERROR）が発生する。
  • 内部的な整合性チェックとして、キャッシュから取得したタプルのOIDが一致しない場合にアサーション失敗が発生する可能性がある。

保守担当者向けメモ

  • 本関数は pg_statistic_ext カタログへの読み取り専用アクセスを行うユーティリティである。
  • 統計オブジェクトの削除や変更を行う際は、本関数で取得したリレーションOIDを用いて適切なロックや権限チェックを行う必要がある。
