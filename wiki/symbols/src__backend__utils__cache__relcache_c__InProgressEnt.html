InProgressEnt

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/relcache.c:164

目的

リレーションの構築中に発生する再帰的な呼び出しや、構築プロセス中の無効化イベントを追跡するために使用される構造体です。

シグネチャまたは形状

typedef struct inprogressent
{
	Oid			reloid;			/* 構築中のリレーションのOID */
	bool		invalidated;	/* 構築中に無効化イベントが到着したか */
} InProgressEnt;

入力と出力

本構造体はデータ保持を目的としており、関数としての入力・出力はありません。

前提条件と不変条件

  • reloid は現在構築中のリレーションを一意に識別します。
  • invalidated は、リレーションの構築処理が完了する前に、当該リレーションに対する無効化通知が届いた場合に true に設定されます。

アルゴリズムまたは振る舞い

リレーション構築処理において、構築中のリレーションを管理するためのリスト（in_progress_list）の要素として利用されます。構築中に無効化イベントが発生した場合、このフラグを立てることで、構築完了後に再構築が必要であることを記録します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/relcache.c 内の in_progress_list を操作する関数群。
  • 呼び出し先: 情報なし。

関連データ構造

  • in_progress_list: InProgressEnt 型の要素を保持する静的リスト（src/backend/utils/cache/relcache.c:170）。

失敗モードと境界条件

  • 構築中のリレーションが再帰的に参照される際、この構造体を用いて状態を管理しない場合、無効化イベントを見落とすリスクがあります。

保守担当者向けメモ

  • 本構造体は relcache.c 内の静的リストで使用されています。構築処理の再帰的な複雑さを管理するための重要なフラグであるため、構築ロジックを変更する際は invalidated フラグの更新タイミングを慎重に確認してください。
