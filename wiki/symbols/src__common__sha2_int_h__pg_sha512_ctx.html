pg_sha512_ctx

定義

  • 種別: struct
  • 定義箇所: src/common/sha2_int.h:61

目的

SHA-384 および SHA-512 ハッシュ計算アルゴリズムの内部状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct pg_sha512_ctx
{
	uint64		state[8];
	uint64		bitcount[2];
	uint8		buffer[PG_SHA512_BLOCK_LENGTH];
} pg_sha512_ctx;

入力と出力

  • **入力**: state（64ビットハッシュ状態変数）、bitcount（処理済みビット数）、buffer（未処理の入力データブロック）。
  • **出力**: ハッシュ計算完了時に pg_sha512_final を通じて最終的なハッシュ値が生成されます。

前提条件と不変条件

  • 64ビット単位の状態変数を使用します。
  • buffer のサイズは PG_SHA512_BLOCK_LENGTH に固定されています。
  • 外部ライブラリ（OpenSSL等）が利用できない環境でのフォールバック実装として使用されます。

アルゴリズムまたは振る舞い

1. **初期化**: pg_sha512_init によりコンテキストがリセットされます。 2. **更新**: pg_sha512_update に入力データを渡すと、内部バッファが満たされるごとに SHA512_Transform が呼び出され、ハッシュ計算が進行します。 3. **終了**: pg_sha512_final が残りのバッファを処理し、最終結果を書き出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/cryptohash.c および src/common/sha2.c 内の各関数。
  • **呼び出し先**: SHA512_Transform および SHA512_Last が本構造体を引数として受け取り、計算を実行します。

関連データ構造

  • pg_sha384_ctx: 本構造体と同一の定義を流用しています。
  • common/sha2.h: 定数定義（ブロック長など）を提供します。

失敗モードと境界条件

  • メモリ上のコンテキストがゼロクリアされていない場合、不正なハッシュ値が生成される可能性があります（memset による初期化が推奨されます）。
  • 外部モジュールから直接操作せず、cryptohash.h 経由の抽象化インターフェースを使用することが推奨されます。

保守担当者向けメモ

  • 本構造体は Aaron D. Gifford 氏の実装に基づいています。
  • SHA-384 は SHA-512 の構造体を流用する設計です。
  • 内部実装用ヘッダーであるため、外部モジュールからの直接参照は避けてください。
