mytype

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/preproc/struct.h:1

目的

ECPG（Embedded SQL C Preprocessor）のプリプロセッサテストにおいて、SQLのホスト変数として使用される標準的な構造体定義を提供します。C言語の構造体メンバーとSQLデータ型とのマッピングおよびメモリレイアウトの検証を目的としています。

シグネチャまたは形状

struct mytype {
    int    id;
    char   t[64];
    double d1;
    double d2;
    char   c[30];
};

入力と出力

本シンボルはデータ定義であり、関数ではないため直接的な入力・出力はありません。ECPGプリプロセッサがこの構造体を読み込むことで、SQL文内でのホスト変数としてのバインド処理が行われます。

前提条件と不変条件

  • 構造体メンバーの型とサイズ（char[64]、char[30]等）は、ECPGがホスト変数として認識する際の型変換ルールに準拠している必要があります。
  • テストコードにおいて、MYTYPE として型定義（typedef）されることが一般的です。

アルゴリズムまたは振る舞い

本シンボル自体に実行時のアルゴリズムは存在しません。ECPGプリプロセッサがこの定義をメタデータとして参照し、SQL文中のホスト変数とC言語のメモリ領域を対応付けるための型情報を提供します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/interfaces/ecpg/test/expected/preproc-outofscope.c: ECPGのテスト出力コードにおいて、ECPGt_int や ECPGt_double 等の型識別子と共に、構造体メンバーへのポインタがSQLバインド処理に渡されています。
  • **呼び出し先**:
  • 本シンボルはデータ定義であるため、関数呼び出しは行いません。

関連データ構造

  • MYTYPE: struct mytype の別名（typedef）。
  • mynulltype: mytype の各フィールドに対応するNULLインジケータを管理するための対となる構造体。

失敗モードと境界条件

  • 構造体のメモリレイアウトがECPGの期待するアライメントと一致しない場合、SQLバインド時にデータ破損や型不整合が発生する可能性があります。
  • char 配列のサイズ（64や30）を超えるデータを格納した場合、バッファオーバーフローのリスクがあります。

保守担当者向けメモ

  • 本ファイルはプロダクションコードではなく、ECPGのプリプロセッサの挙動を検証するためのテスト用定義です。
  • 構造体のメンバー構成を変更する場合は、対応する .pgc テストファイルおよび expected ディレクトリ内の生成済みソースコードの整合性を確認してください。
