目的

Informix 互換の decimal 型ライブラリの機能検証を行う回帰テストのメインエントリポイントです。文字列変換、算術演算、型変換、および NULL 値処理の正確性を検証します。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（グローバル配列 decs[] を参照）
  • 出力: 成功時に 0 を返却。標準出力にテスト結果の詳細を出力。

前提条件と不変条件

  • decs[] 配列が有効な文字列ポインタのリストとして定義されていること。
  • PGTYPESdecimal_new() で確保したメモリは、テスト終了時に PGTYPESdecimal_free() で解放されること。
  • deccmp 等の関数は NULL 値に対して 0 を返す仕様であるため、演算結果の出力パラメータは事前に初期化されていること。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、テスト用配列 decarr を準備。 2. **変換テスト**: decs[] を走査し、deccvasc で decimal 型へ変換後、dectoasc を用いて異なる精度で文字列に戻し、結果を検証。 3. **型変換テスト**: dectolong / dectoint / dectodbl を経由した数値型との相互変換を検証。 4. **NULL テスト**: rsetnull で NULL を設定し、risnull で判定可能かを確認。 5. **算術演算テスト**: decarr 内の全組み合わせに対し、decadd, decsub, decmul, decdiv を実行し、結果を比較。 6. **終了処理**: 確保した全メモリを解放し、終了。

呼び出し元と呼び出し先

  • 呼び出し元: 不明（テスト実行時に OS またはテストハーネスから呼び出される）
  • 呼び出し先: PGTYPESdecimal_new, deccvasc, check_errno, dectoasc, dectolong, deccvlong, deccmp, dectoint, deccvint, dectodbl, print_double, PGTYPESdecimal_free, rsetnull, risnull, decadd, decsub, decmul, decdiv, calloc, realloc, free, strlen, printf

関連データ構造

  • decimal: Informix 互換の数値型構造体。
  • decarr: テスト対象の decimal ポインタを保持する動的配列。

失敗モードと境界条件

  • deccvasc 失敗時: check_errno() を呼び出し、該当インデックスをスキップ。
  • dectoasc 失敗時: check_errno() を呼び出し、エラーを記録。
  • i == 6 の場合: dectodbl の精度問題（アンダーフロー）を避けるため、浮動小数点変換テストをスキップ。
  • NULL 値: risnull により判定され、算術演算時は出力パラメータが初期化された状態で処理される。

保守担当者向けメモ

  • print_double はプラットフォーム間の指数表記の差異を吸収するために使用されています。
  • decarr の再確保には realloc を使用しており、メモリ管理に注意してください。
  • 本関数はテストコードであるため、標準出力への出力形式は回帰テストの期待値と厳密に一致させる必要があります。
