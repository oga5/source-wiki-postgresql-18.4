assign_log_timezone

定義

  • 種別: function
  • 定義箇所: src/backend/commands/variable.c:455

目的

GUC（Grand Unified Configuration）パラメータ log_timezone の値を、検証済みのデータからグローバル変数へ適用する。

シグネチャまたは形状

void assign_log_timezone(const char *newval, void *extra)

入力と出力

  • newval: 設定されるタイムゾーン名の文字列（本関数内では未使用）。
  • extra: check_log_timezone によって事前に検証・解析された pg_tz 構造体へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • extra 引数は、pg_tz 構造体へのポインタを指している必要がある。
  • 本関数は、GUCサブシステムが提供する assign_hook として呼び出されることを前提としている。

アルゴリズムまたは振る舞い

1. extra 引数を pg_tz ** 型にキャストする。 2. キャストされたポインタが指す値を参照し、グローバル変数 log_timezone に代入する。

呼び出し元と呼び出し先

  • 呼び出し元: GUCサブシステム（src/backend/utils/misc/guc_tables.c でフックとして登録）。
  • 呼び出し先: なし。

関連データ構造

  • pg_tz: タイムゾーン情報を保持する構造体。
  • log_timezone: ログ出力に使用されるタイムゾーンを保持するグローバル変数。

失敗モードと境界条件

  • extra が NULL である場合、または不正なメモリ領域を指している場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は単純な代入を行うのみである。検証ロジックは check_log_timezone に集約されている。
  • log_timezone の変更は、ログ出力のタイムスタンプ形式に直接影響を与える。
