FindRegisteredWorkerBySlotNumber

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/bgworker.c:220

目的

共有メモリ上の特定のバックグラウンドワーカー用スロット番号に対応する、Postmaster管理下の RegisteredBgWorker 構造体を検索する。

シグネチャまたは形状

static RegisteredBgWorker *FindRegisteredWorkerBySlotNumber(int slotno)

入力と出力

  • **入力**: slotno (int) - 検索対象の共有メモリ上のスロット番号。
  • **出力**: RegisteredBgWorker* - 指定されたスロット番号と一致する rw_shmem_slot を持つ構造体へのポインタ。該当するものがない場合は NULL を返す。

前提条件と不変条件

  • BackgroundWorkerList が初期化済みであること。
  • 呼び出し側は、リストの走査中に BackgroundWorkerList が変更されないことを保証する必要がある（共有メモリのロック状態については本関数内では制御していない）。

アルゴリズムまたは振る舞い

1. BackgroundWorkerList を dlist_foreach を用いて先頭から順に走査する。 2. 各要素（RegisteredBgWorker）の rw_shmem_slot メンバを確認する。 3. slotno と一致する要素が見つかった場合、そのポインタを即座に返す。 4. リストの末尾まで走査しても一致する要素が見つからない場合、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/bgworker.c 内の関数（ソースコード上の参照として src/backend/postmaster/bgworker.c:285 が確認されている）。
  • **呼び出し先**: dlist API（dlist_foreach, dlist_container）を使用。

関連データ構造

  • RegisteredBgWorker: Postmasterが保持するバックグラウンドワーカーのプライベートな管理構造体。
  • BackgroundWorkerList: 登録済みワーカーを管理する双方向リスト。

失敗モードと境界条件

  • 指定された slotno に対応するワーカーが登録されていない場合、NULL を返す（エラーではなく正常な検索結果として扱われる）。
  • リストが空の場合、常に NULL を返す。

保守担当者向けメモ

  • 本関数は Postmaster のプライベートリストを走査するものであり、共有メモリ上の BackgroundWorkerArray を直接参照するものではない。
  • 検索は線形探索であるため、ワーカー数が増加した際のパフォーマンスに留意すること。
