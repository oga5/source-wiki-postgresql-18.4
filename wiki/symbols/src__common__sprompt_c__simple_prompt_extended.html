simple_prompt_extended

定義

  • 種別: function
  • 定義箇所: src/common/sprompt.c:52

目的

対話的なユーザー入力（パスワード等）を、割り込み（SIGINT）によるキャンセル処理をサポートした状態で安全に取得する。

シグネチャまたは形状

char *simple_prompt_extended(const char *prompt, bool echo, PromptInterruptContext *prompt_ctx);

入力と出力

  • **入力**:
  • prompt: 表示するプロンプト文字列。
  • echo: false の場合、端末のエコーバックを無効化する。
  • prompt_ctx: 割り込み発生時に longjmp を行うための制御構造体（NULL可）。
  • **出力**:
  • ユーザーが入力した文字列（改行除去済み）。malloc されたメモリを返すため、呼び出し側で free が必要。入力が失敗した場合は空文字列を返す。

前提条件と不変条件

  • 端末（/dev/tty または Windows の CONIN$/CONOUT$）がオープン可能であること。オープン失敗時は stdin/stderr にフォールバックする。
  • echo が false の場合、プラットフォーム固有の API（termios または SetConsoleMode）を用いてエコーを制御する。

アルゴリズムまたは振る舞い

1. 端末デバイスをオープンする。失敗時は stdin/stderr を使用する。 2. echo が false の場合、端末設定を変更してエコーを無効化する。 3. prompt が指定されていれば出力し、フラッシュする。 4. pg_get_line を呼び出し、入力を取得する。 5. 入力結果から改行コード（CR/LF）を除去する。 6. echo が false であった場合、端末設定を復元し、改行を出力する。 7. prompt_ctx が指定されており、かつキャンセルが発生した場合は改行を出力する。 8. 端末デバイスをクローズし、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/command.c（パスワード入力等の対話処理）
  • src/common/sprompt.c（simple_prompt 関数からのラッパー）
  • **呼び出し先**:
  • pg_get_line（入力取得）
  • pg_strdup（メモリ確保）
  • pg_strip_crlf（文字列整形）
  • tcgetattr/tcsetattr（POSIX エコー制御）
  • GetConsoleMode/SetConsoleMode（Windows エコー制御）

関連データ構造

  • PromptInterruptContext: 割り込み発生時の制御用構造体。

失敗モードと境界条件

  • 端末オープン失敗時: stdin/stderr にフォールバックするが、この場合エコー抑止は機能しない。
  • 入力取得失敗時: 空文字列を返す。
  • Windows 環境: MSYS 等の特定の環境ではコンソール I/O が正しく動作しない場合がある。

保守担当者向けメモ

  • 返される文字列は malloc されているため、メモリリークを防ぐため呼び出し側での free を徹底すること。
  • Windows における CONIN$/CONOUT$ の w+ モードオープンは、コンソールとして認識させるための必須の回避策である。
