r_consonant_pair

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_danish.c:189

目的

単語の末尾付近にある二重子音ペアを検出し、その一方を削除することで語幹を正規化する。

シグネチャまたは形状

static int r_consonant_pair(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（Snowballステミング環境）
  • 出力: 成功時に 1、失敗または条件不一致時に 0、エラー発生時に負の値を返す。

前提条件と不変条件

  • z->I[1] に格納された領域境界（R1等）より後ろの範囲で処理を行う。
  • z->p（文字列バッファ）の末尾から遡ってマッチングを行う。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置から z->I[1] までの範囲で、末尾の文字が 'd' (100) または 't' (116) であるかを確認する。 2. a_1 配列（among 構造体）を用いて、末尾の二重子音パターンを後方検索する。 3. マッチした場合、その直前の文字を slice_del を用いて削除する。 4. 処理中、z->lb（下限）を一時的に z->I[1] に変更して範囲を制限し、終了後に復元する。

呼び出し元と呼び出し先

  • 呼び出し元: danish_ISO_8859_1_stem（および同等のノルウェー語・スウェーデン語用ステミング関数）から呼び出される。
  • 呼び出し先: find_among_b（後方検索）、slice_del（文字列削除）。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_1: 二重子音パターンを定義した among 構造体の配列。

失敗モードと境界条件

  • z->c < z->I[1]: 処理対象領域が不十分な場合は 0 を返す。
  • z->c <= z->lb: 削除対象の文字が存在しない場合は 0 を返す。
  • slice_del: 内部的なメモリ操作でエラーが発生した場合は負の値を返す。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによる自動生成コードである。ロジックの変更が必要な場合は、直接編集せず元の .sbl ファイルを修正すること。
  • a_1 配列の定義が変更された場合、マッチング対象となる子音ペアのルールも変わるため注意が必要。
