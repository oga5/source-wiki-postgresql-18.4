set_config_by_name

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/guc_funcs.c:331

目的

SQLインターフェースを通じて、指定されたGUC（Grand Unified Configuration）パラメータの値を設定またはリセットし、その結果として適用された新しい値を返します。

シグネチャまたは形状

Datum set_config_by_name(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • arg0 (text): 設定するパラメータ名。
  • arg1 (text): 設定する値（NULLの場合はリセット）。
  • arg2 (bool): is_local フラグ。trueの場合は SET LOCAL として動作。
  • **出力**:
  • Datum (text): 設定適用後のパラメータの現在値。

前提条件と不変条件

  • 第1引数（パラメータ名）がNULLの場合、エラーが発生します。
  • is_local がNULLの場合は false として扱われます。
  • 実行ユーザーの権限に応じて、PGC_SUSET（スーパーユーザー）または PGC_USERSET（一般ユーザー）のコンテキストで設定が適用されます。

アルゴリズムまたは振る舞い

1. 引数からパラメータ名、値、および is_local フラグを抽出します。 2. set_config_option を呼び出し、指定された値を適用します。この際、is_local がtrueであれば GUC_ACTION_LOCAL を、そうでなければ GUC_ACTION_SET を指定します。 3. GetConfigOptionByName を使用して、設定適用後の現在の値を取得します。 4. 取得した値を text 型に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL層からの呼び出し（具体的な呼び出し元コードの詳細は提供されていません）。
  • **呼び出し先**:
  • set_config_option (GUCのコアロジック)
  • GetConfigOptionByName (設定値の取得)
  • superuser (権限判定)
  • cstring_to_text (型変換)

関連データ構造

  • PG_FUNCTION_ARGS: SQL関数インターフェース用の引数構造体。

失敗モードと境界条件

  • パラメータ名がNULLの場合: ERRCODE_NULL_VALUE_NOT_ALLOWED でエラー終了。
  • 不正なパラメータ名や値が指定された場合: set_config_option 内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はSQLから直接呼び出されるラッパーです。設定の検証や保持のロジックは guc.c に依存しているため、動作変更時は guc.c の仕様を確認してください。
  • SET DEFAULT の挙動は value 引数にNULLを渡すことで実現されています。
