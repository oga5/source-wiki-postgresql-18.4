is_foreign_expr

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/deparse.c:241

目的

与えられた式（Expr）が、リモートの PostgreSQL サーバー上で安全かつ正確に実行可能かどうかを判定する。

シグネチャまたは形状

bool is_foreign_expr(PlannerInfo *root, RelOptInfo *baserel, Expr *expr)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • baserel: 式が評価される対象のリレーション情報。
  • expr: リモート実行の可否を判定する対象の式ツリー。
  • **出力**:
  • bool: リモート実行が安全であれば true、そうでなければ false を返す。

前提条件と不変条件

  • baserel->fdw_private に PgFdwRelationInfo が格納されている必要がある。
  • IS_UPPER_REL(baserel) が真の場合、fpinfo->outerrel が有効である必要がある。

アルゴリズムまたは振る舞い

1. **コンテキスト初期化**: foreign_glob_cxt および foreign_loc_cxt を初期化し、リレーションの識別子（relids）を設定する。 2. **再帰的走査**: foreign_expr_walker を呼び出し、式ツリー内の各ノードがリモート実行可能か（出荷可能性や照合順序の整合性）を検証する。 3. **照合順序チェック**: loc_cxt.state が FDW_COLLATE_UNSAFE である場合、照合順序の不整合によりリモート実行不可と判断する。 4. **可変性チェック**: contain_mutable_functions を呼び出し、式に可変関数（now() 等）が含まれていないか確認する。これらは結果が安定しないためリモート送信できない。 5. **判定**: 上記のすべてのチェックを通過した場合にのみ true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/deparse.c
  • contrib/postgres_fdw/postgres_fdw.c（多数の箇所でクエリ条件やターゲットリストの検証に使用）
  • **呼び出し先**:
  • foreign_expr_walker
  • contain_mutable_functions

関連データ構造

  • PlannerInfo
  • RelOptInfo
  • PgFdwRelationInfo
  • foreign_glob_cxt
  • foreign_loc_cxt

失敗モードと境界条件

  • 式ツリー内にリモートサーバーでサポートされていない関数や演算子が含まれる場合、foreign_expr_walker が false を返し、本関数も false を返す。
  • 照合順序がローカルとリモートで競合する可能性がある場合、安全のために false を返す。
  • 可変関数が含まれる場合、実行結果の安定性が保証されないため false を返す。

保守担当者向けメモ

  • 本関数はコストの高いカタログ検索（contain_mutable_functions）を最後に行うことで最適化されている。
  • 将来的にリモートサーバーの機能が拡張された場合、foreign_expr_walker のロジック更新が必要になる可能性がある。
