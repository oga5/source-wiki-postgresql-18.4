OnConflict_CheckForSerializationFailure

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/predicate.c:4535

目的

直列化可能スナップショット分離（SSI）において、トランザクション間の読み取り/書き込み依存関係（rw-conflict）を分析し、直列化不可能なサイクルや危険な構造を検知して、必要に応じてトランザクションを強制終了（ロールバック）する。

シグネチャまたは形状

static void OnConflict_CheckForSerializationFailure(const SERIALIZABLEXACT *reader, SERIALIZABLEXACT *writer)

入力と出力

  • **入力**:
  • reader: 読み取り側トランザクションのポインタ。
  • writer: 書き込み側トランザクションのポインタ。
  • **出力**: なし（副作用としてトランザクションの DOOMED 状態設定または ereport(ERROR) による中断が発生）。

前提条件と不変条件

  • SerializableXactHashLock が呼び出し元によって保持されていること。

アルゴリズムまたは振る舞い

1. **書き込み側の分析**: writer が既にコミット済み、または out-conflict を持つ場合、危険な構造（R -> W -> T2）と判断し failure を真とする。 2. **ピボットの検証**: writer がピボット（中間ノード）として機能している場合、outConflicts リストを走査し、コミット順序やスナップショットの整合性を確認して直列化不可能な条件を満たすか判定する。 3. **読み取り側の分析**: reader がピボットとして機能している場合、inConflicts リストを走査し、同様に直列化不可能な条件を判定する。 4. **終了処理**:

  • failure が真の場合、自身が writer であれば即座にエラーを報告。
  • writer が既に準備済み（Prepared）であれば、reader をエラーで終了させる。
  • それ以外の場合は writer を SXACT_FLAG_DOOMED に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/lmgr/predicate.c 内の競合発生箇所（例: CheckForSerializableConflictIn 等から呼び出される）。
  • **呼び出し先**: ereport（エラー発生時）、LWLockRelease。

関連データ構造

  • SERIALIZABLEXACT: 直列化可能トランザクションの状態を保持。
  • RWConflictData: トランザクション間の依存関係を保持。
  • SerializableXactHashLock: 共有メモリ上のハッシュテーブルを保護するロック。

失敗モードと境界条件

  • writer が既に PREPARED 状態である場合、writer をロールバックできないため、代わりに reader を強制終了させる。
  • メモリ圧迫等によりロックが昇格されている場合、偽陽性の競合検知が発生する可能性がある。

保守担当者向けメモ

  • dlist_foreach を使用する際、reader の inConflicts リストに対して unconstify が使用されている点に注意。
  • 本関数は直列化可能性を保証するための最終防衛線であり、ここで検知される競合はアプリケーション側でのリトライが必要となる。
