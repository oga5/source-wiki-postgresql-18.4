pullup_replace_vars_context

定義

  • **種別**: struct
  • **定義箇所**: src/backend/optimizer/prep/prepjointree.c:69

目的

サブクエリを親クエリへ引き上げる（Pull-up）処理において、サブクエリ内の変数を親クエリの式に適切に置換・マッピングするための状態を保持するコンテキスト構造体です。

シグネチャまたは形状

typedef struct pullup_replace_vars_context {
    PlannerInfo *root;
    List        *targetlist;        /* 引き上げ対象サブクエリのターゲットリスト */
    RangeTblEntry *target_rte;      /* サブクエリのRTE */
    int          result_relation;   /* 書き換え後のクエリにおける結果リレーションのインデックス */
    Relids       relids;            /* 引き上げ後のサブクエリ内リレーションID（lateral時のみ設定） */
    nullingrel_info *nullinfo;      /* RTEごとのnullingrel情報（lateral時のみ設定） */
    bool        *outer_hasSubLinks; /* 親クエリのhasSubLinksへのポインタ */
    int          varno;             /* サブクエリのvarno */
    ReplaceWrapOption wrap_option;  /* 特定の出力をPHV（PlaceHolderVar）にする必要があるか */
    Node       **rv_cache;          /* PHVを含む結果のキャッシュ */
} pullup_replace_vars_context;

入力と出力

本構造体は関数引数として渡され、引き上げ処理の過程で targetlist や nullinfo などの情報が参照されます。また、rv_cache を通じて置換結果のキャッシュが更新されます。

前提条件と不変条件

  • target_rte->lateral が真の場合に限り、relids および nullinfo が有効な値として設定されます。
  • outer_hasSubLinks は親クエリの状態を正しく反映している必要があります。

アルゴリズムまたは振る舞い

サブクエリの引き上げ時に pullup_replace_vars 関数へ渡され、以下の役割を果たします。 1. サブクエリの出力変数を親クエリの式へ変換する際のマッピング情報を提供。 2. 外結合の null 伝播を考慮した PlaceHolderVar の挿入判断。 3. rv_cache を利用した置換結果の効率的な再利用。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/prep/prepjointree.c 内の pull_up_subqueries や pull_up_sublinks 等の引き上げ処理関数。
  • **呼び出し先**: 本構造体は pullup_replace_vars 関数へ渡され、同関数内で参照されます。

関連データ構造

  • PlannerInfo: プランナのグローバル情報。
  • RangeTblEntry: 範囲テーブルエントリ。
  • nullingrel_info: 外結合の null 伝播情報。
  • ReplaceWrapOption: PHV へのラップ要否を制御する列挙型。

失敗モードと境界条件

  • target_rte が適切に設定されていない場合、変数置換が正しく行われずプランナが不正な状態に陥る可能性があります。
  • 深いジョインツリーに対する再帰処理が含まれるため、スタックオーバーフローの制約を受けます。

保守担当者向けメモ

  • 本構造体は pullup_replace_vars の動作の核であり、特に外結合の null 伝播を扱う際の PlaceHolderVar 挿入ロジックと密接に関連しています。
  • 処理順序が厳密であるため、フィールドの追加や変更を行う際は、pullup_replace_vars 内の参照順序に注意してください。
