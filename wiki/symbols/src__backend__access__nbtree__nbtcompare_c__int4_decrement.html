int4_decrement

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtcompare.c:223

目的

B-tree インデックスのスキップスキャンにおいて、int4 型の値を 1 減算するロジックを提供します。

シグネチャまたは形状

static Datum int4_decrement(Relation rel, Datum existing, bool *underflow)

入力と出力

  • **入力**:
  • rel: 対象のインデックスリレーション（未使用）。
  • existing: 減算対象の int4 値（Datum 型）。
  • underflow: アンダーフロー発生時に true を書き込むためのポインタ。
  • **出力**:
  • 減算後の Datum 値。アンダーフロー時は (Datum) 0 を返します。

前提条件と不変条件

  • existing は int32 として解釈されます。
  • underflow ポインタは有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. existing を int32 に変換します。 2. 値が PG_INT32_MIN と等しい場合、アンダーフローと判定し、*underflow を true に設定して (Datum) 0 を返します。 3. それ以外の場合、*underflow を false に設定し、値から 1 を引いた結果を Datum に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtcompare.c 内で、SkipSupport 構造体の decrement フィールドに本関数が代入されています。
  • **呼び出し先**: 直接的な関数呼び出しはソース内に見当たりません。

関連データ構造

  • SkipSupport: インデックススキップスキャン用の増減ロジックを保持する構造体。

失敗モードと境界条件

  • **アンダーフロー**: 入力値が PG_INT32_MIN の場合、減算は行われず underflow フラグが立てられます。

保守担当者向けメモ

  • 本関数は int4 型専用です。他の整数型については、対応する int2_decrement や int8_decrement を参照してください。
  • rel 引数は現在の実装では使用されていませんが、将来的な拡張のためにシグネチャに含まれています。
