range_typanalyze

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_typanalyze.c:45

目的

範囲型（Range Types）カラムに対する統計情報収集（ANALYZE）処理を初期化し、統計計算関数を登録する。

シグネチャまたは形状

Datum range_typanalyze(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（VacAttrStats 構造体へのポインタを含む）。
  • **出力**: Datum 型として true を返す。

前提条件と不変条件

  • stats->attrtypid は有効な範囲型（またはそのドメイン）であること。
  • range_get_typcache を介して型キャッシュが取得可能であること。

アルゴリズムまたは振る舞い

1. range_get_typcache を呼び出し、対象の型情報を取得する。 2. stats->attstattarget が負の値であれば、default_statistics_target で初期化する。 3. stats->compute_stats に統計計算関数 compute_range_stats を割り当てる。 4. stats->extra_data に取得した型キャッシュ情報を格納する。 5. stats->minrows を 300 * stats->attstattarget に設定し、サンプリングの最小行数を確保する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の明示的な呼び出し元情報は提供されていません。
  • **呼び出し先**:
  • range_get_typcache
  • getBaseType
  • compute_range_stats（関数ポインタとして設定）

関連データ構造

  • VacAttrStats: ANALYZE処理中にカラムの統計情報を保持する構造体。
  • TypeCacheEntry: 範囲型のメタデータ（境界値の型や比較関数など）を保持するキャッシュ構造体。

失敗モードと境界条件

  • stats->attrtypid が範囲型ではない場合、range_get_typcache 内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は std_typanalyze と同様の初期化パターンに従っています。
  • minrows の計算式（300倍）は、範囲型の統計精度を維持するためのヒューリスティックな値です。
