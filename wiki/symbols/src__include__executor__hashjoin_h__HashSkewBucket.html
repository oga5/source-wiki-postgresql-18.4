HashSkewBucket

定義

  • 種別: struct
  • 定義箇所: src/include/executor/hashjoin.h:113

目的

ハッシュ結合において、頻出値（MCV: Most Common Values）に起因するデータスキューを最適化するための専用バケット構造体です。メインのハッシュテーブルとは別に管理されます。

シグネチャまたは形状

typedef struct HashSkewBucket
{
	uint32		hashvalue;		/* common hash value */
	HashJoinTuple tuples;		/* linked list of inner-relation tuples */
} HashSkewBucket;

入力と出力

  • **入力**: 頻出するハッシュ値（hashvalue）およびそれに対応する内部リレーションのタプル群（tuples）。
  • **出力**: なし（構造体として状態を保持）。

前提条件と不変条件

  • tuples は、同一のハッシュ値を持つ内部リレーションのタプルの連結リストを指します。
  • SKEW_BUCKET_OVERHEAD マクロにより、この構造体のサイズ（アライメント調整済み）が定義されています。

アルゴリズムまたは振る舞い

  • ハッシュ結合の構築フェーズにおいて、特定のハッシュ値が頻出する場合、メインのハッシュテーブルではなく本構造体を用いてタプルを格納します。
  • プローブフェーズでは、まずこのスキューバケットを確認することで、頻出値に対する検索効率を向上させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHash.c 内のハッシュテーブル構築および検索ロジック。
  • **呼び出し先**: 直接的な関数呼び出しは行いませんが、HashJoinTuple 型を介してタプルデータと相互作用します。

関連データ構造

  • HashJoinTableData: 本構造体を保持するメインのハッシュテーブル管理構造体。
  • HashJoinTuple: 本構造体が管理するタプルリストの要素型。

失敗モードと境界条件

  • メモリ不足時: HashJoinTableData の spaceUsedSkew を通じてメモリ消費が追跡されます。
  • 衝突: 複数の異なるハッシュ値が同じスキューバケットインデックスにマッピングされる可能性があります。

保守担当者向けメモ

  • nodeHash.c 内で hashtable->skewBucket として配列管理されています。
  • 並列ハッシュ結合（Parallel Hash Join）との相互作用については、ParallelHashJoinState の設計と照らし合わせて確認してください。
