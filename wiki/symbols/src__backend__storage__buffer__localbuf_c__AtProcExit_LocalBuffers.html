AtProcExit_LocalBuffers

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/localbuf.c:1003

目的

バックエンドプロセスの終了時に、ローカルバッファのピン留め（pin）がすべて解放されていることを確認する。

シグネチャまたは形状

void AtProcExit_LocalBuffers(void)

入力と出力

  • 入力: なし
  • 出力: なし（正常終了時）

前提条件と不変条件

  • プロセス終了時に、保持しているローカルバッファのピンがすべて解放されている必要がある。

アルゴリズムまたは振る舞い

1. CheckForLocalBufferLeaks() を呼び出し、リークしている（ピン留めされたままの）ローカルバッファがないか検査する。 2. ピンが残っている場合、アサーションが有効であればエラーとなり、無効であれば後続の DropRelationBuffers 呼び出し時に一時テーブルの削除処理で失敗する可能性がある。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/buffer/bufmgr.c:4049
  • 呼び出し先: CheckForLocalBufferLeaks()

関連データ構造

  • LocalBufferDescriptors: ローカルバッファのメタデータを保持する配列。
  • LocalRefCount: 各ローカルバッファの参照カウントを保持する配列。

失敗モードと境界条件

  • ピン留めされたままのバッファが存在する場合、CheckForLocalBufferLeaks() によりリークが検出される。

保守担当者向けメモ

  • 本関数はバックエンド終了時のクリーンアップ処理の一部である。
  • 共有バッファマネージャとは異なり、プロセスローカルなリソース管理に特化している。
