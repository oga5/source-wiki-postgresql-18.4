relptr_store_eval

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/relptr.h:63

目的

共有メモリ等の動的領域において、絶対アドレスをベースアドレスからの相対オフセット値に変換して格納するための計算を行う。

シグネチャまたは形状

Size relptr_store_eval(char *base, char *val)

入力と出力

  • **入力**:
  • base: 相対参照の起点となるベースアドレス（char *）。
  • val: 格納対象の絶対アドレス（char *）。
  • **出力**:
  • val が NULL の場合は 0 を返す。
  • それ以外の場合は、val - base + 1 を計算したオフセット値を返す。

前提条件と不変条件

  • val が NULL でない場合、val >= base である必要がある（Assert により検証される）。
  • 0 は NULL ポインタを表す予約値として扱われるため、実際のオフセット値には 1 が加算される。

アルゴリズムまたは振る舞い

1. val が NULL かどうかを判定する。 2. NULL であれば 0 を返す。 3. NULL でなければ、val が base 以降のアドレスであることを確認し、val から base を引いた差分に 1 を加えた値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/utils/relptr.h 内の relptr_store マクロ（行79および87）から呼び出される。
  • **呼び出し先**:
  • Assert マクロ（標準的なデバッグ用検証）。

関連データ構造

  • relptr（相対ポインタを保持する共用体）: 本関数で計算されたオフセット値が、この構造体の relptr_off メンバに格納される。

失敗モードと境界条件

  • val < base の場合、Assert が有効なビルドではアサーション失敗によりプログラムが停止する。
  • val が NULL の場合、オフセット 0 が返されるため、呼び出し側はこれを NULL として扱う必要がある。

保守担当者向けメモ

  • 本関数は relptr_store マクロの内部実装の一部である。
  • オフセット計算時に +1 しているため、生のオフセット値を取り出す際は relptr_offset() 等の適切なアクセサを使用し、直接計算を行わないこと。
  • base 引数には必ず char * 型を渡す必要がある（マクロ側で型検証が行われる）。
