ExecReScanIndexScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexscan.c:552

目的

インデックススキャンノードの状態をリセットし、再スキャン可能な状態にする。

シグネチャまたは形状

void ExecReScanIndexScan(IndexScanState *node)

入力と出力

  • **入力**: IndexScanState *node（スキャン対象のインデックススキャン状態構造体）
  • **出力**: なし（副作用として node の内部状態が更新される）

前提条件と不変条件

  • node が有効な IndexScanState を指していること。
  • ランタイムキーが存在する場合、iss_RuntimeContext が初期化されていること。

アルゴリズムまたは振る舞い

1. **ランタイムキーの再計算**: iss_NumRuntimeKeys が 0 でない場合、iss_RuntimeContext をリセットし、ExecIndexEvalRuntimeKeys を呼び出して新しいキー値を計算する。完了後、iss_RuntimeKeysReady を true に設定する。 2. **再ソートキューのフラッシュ**: iss_ReorderQueue が存在する場合、ペアリングヒープ内の全タプルをポップしてメモリを解放する。 3. **インデックススキャンのリセット**: iss_ScanDesc が存在する場合、index_rescan を呼び出し、スキャンキーおよびオーダーバイキーを再適用する。 4. **フラグのリセット**: iss_ReachedEnd を false に設定する。 5. **共通スキャン状態のリセット**: ExecScanReScan を呼び出し、共通のスキャン状態を更新する。

関連データ構造

  • IndexScanState: インデックススキャンの実行状態を保持する構造体。
  • ExprContext: ランタイムキー評価用の式コンテキスト。
  • pairingheap: ORDER BY 句の再ソートに使用される優先度付きキュー。

失敗モードと境界条件

  • node->iss_ScanDesc が NULL の場合、index_rescan はスキップされる。
  • ランタイムキーがない場合、キー計算処理はスキップされる。

保守担当者向けメモ

  • 本関数は、クエリ実行中にパラメータが変更された際などに呼び出される。
  • pairingheap のメモリリークを防ぐため、再スキャン時には必ずキューを空にする必要がある。
