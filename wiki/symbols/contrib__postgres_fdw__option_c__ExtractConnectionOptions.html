ExtractConnectionOptions

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/option.c:422

目的

外部データラッパー（FDW）のオプションリストから、libpq が認識可能な接続パラメータのみを抽出して配列に格納する。

シグネチャまたは形状

int ExtractConnectionOptions(List *defelems, const char **keywords, const char **values)

入力と出力

  • **入力**:
  • defelems: 抽出対象となる DefElem 構造体のリスト。
  • keywords: 抽出されたオプション名を格納するための文字列配列（呼び出し元で確保）。
  • values: 抽出されたオプション値を格納するための文字列配列（呼び出し元で確保）。
  • **出力**:
  • 戻り値: 抽出されたオプションの総数（int）。

前提条件と不変条件

  • InitPgFdwOptions() が内部で呼び出され、オプションのメタデータが初期化されていること。
  • keywords および values 配列は、defelems の要素数以上のサイズが確保されていること。

アルゴリズムまたは振る舞い

1. InitPgFdwOptions() を呼び出し、内部のオプションリストを構築する。 2. defelems リストを走査する。 3. 各要素に対して is_libpq_option() を実行し、libpq 関連のオプションであるか判定する。 4. libpq 関連であれば、その名前を keywords に、値を defGetString() で取得して values に格納する。 5. 抽出した件数をカウントし、最終的にその数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/connection.c:506 (サーバーオプションの抽出)
  • contrib/postgres_fdw/connection.c:508 (ユーザーマッピングオプションの抽出)
  • **呼び出し先**:
  • InitPgFdwOptions()
  • is_libpq_option()
  • defGetString()

関連データ構造

  • DefElem: SQL で指定されたオプションのキーと値を保持する構造体。
  • List: PostgreSQL のリスト構造。

失敗モードと境界条件

  • defelems が空の場合、戻り値は 0 となり、配列への書き込みは発生しない。
  • defGetString() が失敗した場合、PostgreSQL のエラー処理メカニズムにより処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数は libpq 接続パラメータのフィルタリングに特化している。新しい libpq オプションが追加された場合、is_libpq_option のロジックが正しく更新されていることを確認すること。
