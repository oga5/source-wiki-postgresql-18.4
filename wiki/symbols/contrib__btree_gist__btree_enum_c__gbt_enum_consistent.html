gbt_enum_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_enum.c:122

目的

GiST インデックス検索において、列挙型（ENUM）のインデックスキーがクエリ条件を満たすかどうかを判定します。

シグネチャまたは形状

Datum gbt_enum_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY *entry（インデックスキーを含むエントリ）
  • PG_GETARG_OID(1): Oid query（検索対象の列挙値）
  • PG_GETARG_UINT16(2): StrategyNumber strategy（比較演算子）
  • PG_GETARG_POINTER(4): bool *recheck（再チェックフラグのポインタ）
  • **出力**:
  • Datum: bool 型（一致判定結果）
  • *recheck: false に設定（本関数による判定は完全であるため）

前提条件と不変条件

  • ENUM 型は内部的に OID として扱われ、順序付けが可能である必要があります。
  • recheck は常に false に設定されます。これは、本関数がインデックス内の範囲情報を用いて正確な判定を行うことを前提としているためです。

アルゴリズムまたは振る舞い

1. entry から oidKEY 構造体を取り出し、GBT_NUMKEY_R 形式に変換します。 2. recheck を false に設定します。 3. gbt_num_consistent ユーティリティ関数を呼び出し、以下の情報を渡して判定を委譲します。

  • 変換済みのキー範囲 (key)
  • クエリ値 (query)
  • 比較戦略 (strategy)
  • リーフノード判定 (GIST_LEAF(entry))
  • 比較ロジック定義 (tinfo)

    • 4. gbt_num_consistent の戻り値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス検索エンジン（直接の呼び出し元はソースコード上では明示されていません）。
  • **呼び出し先**: gbt_num_consistent（contrib/btree_gist/btree_utils_num.c で定義された汎用ユーティリティ）。

関連データ構造

  • oidKEY: ENUM の範囲（lower, upper）を保持する構造体。
  • GBT_NUMKEY_R: ユーティリティ関数用の範囲表現構造体。
  • tinfo: gbtree_ninfo 構造体。ENUM 型の比較関数やサイズ情報を保持する設定テーブル。

失敗モードと境界条件

  • 本関数は recheck を false に固定するため、インデックスが保持する範囲情報が正確であることを前提としています。
  • 比較戦略がサポートされていない場合、gbt_num_consistent 内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は oidKEY を介して OID ベースの範囲検索ロジックを再利用しています。
  • ENUM 型の比較ロジックを変更する場合は、tinfo 構造体で参照されている比較関数（gbt_enumgt 等）を確認してください。
  • KNN 検索は現在サポートされていません。
