BrinMemTuple

定義

  • 種別: struct
  • 定義箇所: src/include/access/brin_tuple.h:44

目的

BRIN（Block Range Index）インデックスの処理中に、メモリ上でページ範囲の要約情報を保持・操作するための構造体です。

シグネチャまたは形状

typedef struct BrinMemTuple
{
    bool        bt_placeholder;
    bool        bt_empty_range;
    BlockNumber bt_blkno;
    MemoryContext bt_context;
    Datum      *bt_values;
    bool       *bt_allnulls;
    bool       *bt_hasnulls;
    BrinValues  bt_columns[FLEXIBLE_ARRAY_MEMBER];
} BrinMemTuple;

入力と出力

  • brin_deform_tuple の出力先として、bt_values、bt_allnulls、bt_hasnulls が使用されます。
  • bt_columns は可変長配列として各カラムの要約値を保持します。

前提条件と不変条件

  • bt_columns は構造体の末尾に配置される必要があります。
  • bt_context は bt_columns 内の値を保持するメモリコンテキストを指す必要があります。

アルゴリズムまたは振る舞い

  • BRIN インデックスの更新時、brin_form_tuple を介してディスク格納用の BrinTuple へ変換されます。
  • インデックス検索時には、brin_deform_tuple を使用してディスク上の BrinTuple から本構造体へデータが展開されます。
  • bt_info フィールド（BrinTuple 側）と連携し、NULL 状態やプレースホルダー、空範囲のフラグ管理を行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/brin/brin.c 内の union_tuples、add_values_to_range 等、および contrib/pageinspect/brinfuncs.c。
  • 呼び出し先: 本構造体自体はデータ保持用であり、操作には brin_tuple.c で定義された変換関数群が使用されます。

関連データ構造

  • BrinTuple: ディスク上の物理的な表現。
  • BrinValues: 各カラムの要約値を保持する構造体。
  • BrinDesc: BRIN インデックスの記述子。

失敗モードと境界条件

  • TOAST された値を含む場合、インデックスエントリのデトースト処理が必要です。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、動的なメモリ確保とサイズ計算が必須です。

保守担当者向けメモ

  • メモリ上の操作とディスク上の表現を分離して管理しています。外部コードは本構造体を扱い、変換処理は brin_tuple.c に委譲してください。
  • NULL ビットマップ（hasnulls および allnulls）は BrinTuple のデータ領域先頭に格納されるため、変換時のビット操作に注意してください。
