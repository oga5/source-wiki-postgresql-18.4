LogicalRepInfo

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_basebackup/pg_createsubscriber.c:53

目的

データベースごとの論理レプリケーション設定（パブリケーション、サブスクリプション、レプリケーションスロット）およびその作成状態を管理する。

シグネチャまたは形状

struct LogicalRepInfo
{
    char       *dbname;             /* データベース名 */
    char       *pubconninfo;        /* パブリッシャー接続文字列 */
    char       *subconninfo;        /* サブスクライバー接続文字列 */
    char       *pubname;            /* パブリケーション名 */
    char       *subname;            /* サブスクリプション名 */
    char       *replslotname;       /* レプリケーションスロット名 */
    bool        made_replslot;      /* レプリケーションスロット作成フラグ */
    bool        made_publication;   /* パブリケーション作成フラグ */
};

入力と出力

本構造体は、pg_createsubscriber ユーティリティ内での状態保持に使用される。入力として各フィールドが設定され、処理の進行に伴い made_replslot や made_publication が更新される。

前提条件と不変条件

  • dbname、pubconninfo、subconninfo 等の文字列ポインタは、適切に割り当てられたメモリを指す必要がある。
  • made_replslot および made_publication は、実際に該当オブジェクトが作成された場合にのみ true に設定される。

アルゴリズムまたは振る舞い

本構造体は、物理スタンバイから論理サブスクライバーへの変換プロセスにおいて、パブリッシャー側で作成されたオブジェクトを追跡する。これにより、エラー発生時や終了時に cleanup_objects_atexit を通じて作成したオブジェクトを適切に削除（クリーンアップ）することが可能となる。

呼び出し元と呼び出し先

  • **呼び出し元**: store_pub_sub_info により初期化される。
  • **呼び出し先**: check_publisher, setup_publisher, check_subscriber, setup_subscriber, setup_recovery, drop_primary_replication_slot, drop_failover_replication_slots, drop_replication_slot, create_publication 等の関数に引数として渡され、各処理の対象データベース情報として参照される。

関連データ構造

  • CreateSubscriberOptions: コマンドライン引数や全体設定を保持する構造体。

失敗モードと境界条件

  • 文字列フィールドが NULL の場合、参照先の関数でセグメンテーション違反が発生する可能性がある。
  • made_ フラグが正しく管理されていない場合、クリーンアップ処理が不完全となり、パブリッシャー側に不要なオブジェクトが残存する可能性がある。

保守担当者向けメモ

  • 本構造体は pg_createsubscriber の変換プロセスにおける状態管理の要である。
  • 新たな論理レプリケーション関連オブジェクトを追加する場合は、本構造体にフィールドを追加し、クリーンアップ処理（cleanup_objects_atexit）を更新する必要がある。
