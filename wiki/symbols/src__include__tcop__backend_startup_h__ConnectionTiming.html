ConnectionTiming

定義

  • 種別: struct
  • 定義箇所: src/include/tcop/backend_startup.h:97

目的

バックエンドプロセスの起動および接続確立プロセスにおける各フェーズの時間を計測・記録するための構造体です。接続のパフォーマンス分析やデバッグログ出力に使用されます。

シグネチャまたは形状

typedef struct ConnectionTiming
{
    TimestampTz socket_create;
    TimestampTz ready_for_use;
    TimestampTz fork_start;
    TimestampTz fork_end;
    TimestampTz auth_start;
    TimestampTz auth_end;
} ConnectionTiming;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。各フィールドは utils/timestamp.h で定義される TimestampTz 型を保持します。

前提条件と不変条件

  • socket_create と ready_for_use の差分は、接続確立およびセットアップの合計時間を表します。
  • 各タイムスタンプは、バックエンドの初期化プロセス中に順次更新されることが想定されています。

アルゴリズムまたは振る舞い

接続確立の各段階（ソケット作成、プロセス生成、認証）において、対応するフィールドに現在のタイムスタンプを記録します。これにより、接続の各フェーズにかかった時間を後続の処理で算出可能にします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tcop/backend_startup.c にて、接続確立の各フェーズで本構造体のインスタンスが更新されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。
  • 外部参照: src/include/tcop/backend_startup.h:25 にて extern PGDLLIMPORT struct ConnectionTiming conn_timing; として宣言されており、グローバルな接続タイミング情報として参照されます。

関連データ構造

  • TimestampTz: utils/timestamp.h で定義される時間型。
  • BackendStartupData: バックエンド起動時に引き継がれる関連情報。

失敗モードと境界条件

  • 接続が正常に完了しない場合、ready_for_use などのフィールドが初期値（TIMESTAMP_MINUS_INFINITY 等）のままとなる可能性があります。

保守担当者向けメモ

  • 本構造体の各フィールドは、log_connections GUC設定が有効な場合にログ出力の根拠として利用されます。
  • 認証後のクエリ処理フローについては本構造体の範囲外であり、tcopprot.h を参照してください。
