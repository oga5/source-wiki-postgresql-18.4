IsPostmasterChildWalSender

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/pmsignal.c:270

目的

指定されたスロット番号に対応する子プロセスが、現在 WALSENDER 状態であるかどうかを判定します。

シグネチャまたは形状

bool IsPostmasterChildWalSender(int slot)

入力と出力

  • **入力**: slot (int) - 検査対象の子プロセススロット番号（1始まり）。
  • **出力**: bool - 指定されたスロットが PM_CHILD_WALSENDER 状態であれば true、それ以外であれば false を返します。

前提条件と不変条件

  • slot は 1 以上 num_child_flags 以下である必要があります（Assert により検証）。
  • PMSignalState が初期化済みであり、有効な共有メモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. 引数 slot が有効範囲内にあるかを確認します。 2. 内部的なインデックス調整のため、slot をデクリメントします。 3. 共有メモリ上の PMSignalState->PMChildFlags 配列の該当インデックスを参照します。 4. 値が PM_CHILD_WALSENDER と一致するかを評価し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/postmaster.c (行3481および3915): 子プロセスの終了処理や状態監視の文脈で呼び出されています。
  • **呼び出し先**:
  • 本関数内での関数呼び出しはありません。

関連データ構造

  • PMSignalState: 子プロセスの状態フラグを保持する共有メモリ上の構造体。
  • PMChildFlags: PMSignalState 内の配列で、各スロットの状態を保持します。

失敗モードと境界条件

  • slot が範囲外（<= 0 または > num_child_flags）の場合、アサーションエラーが発生しプロセスが異常終了します。

保守担当者向けメモ

  • 本関数は sig_atomic_t 等で管理される共有メモリフラグを直接読み取るため、ロックを取得せずに実行されます。
  • num_child_flags は Postmaster のローカル変数であり、共有メモリ内の状態と同期している前提で動作します。
