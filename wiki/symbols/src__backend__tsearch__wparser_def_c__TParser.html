TParser

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/wparser_def.c:204

目的

PostgreSQL の全文検索におけるデフォルト・テキスト検索パーサーの解析状態を保持するメイン構造体です。入力文字列の解析位置、現在の状態、および解析に必要なコンテキストを管理します。

シグネチャまたは形状

ソースコード上の定義は struct TParser です。内部には解析対象の文字列、現在の解析状態、および状態遷移やトークン抽出に必要な情報が含まれます。

入力と出力

  • **入力**: 解析対象となる文字列（TParserCopyInit を使用する場合は他の TParser が持つ文字列を参照）。
  • **出力**: 解析の進行に伴い、抽出されたトークンや現在の解析位置が更新されます。

前提条件と不変条件

  • TParserCopyInit で生成されたコピーを使用する場合、コピー元の TParser を先にクローズしてはなりません。
  • 解析には palloc0 によるメモリ確保が前提となります。

アルゴリズムまたは振る舞い

1. **初期化**: TParserInit により構造体が作成され、入力文字列がワイド文字形式に変換されます。 2. **状態遷移**: TParserGet 関数が現在の状態に基づき、TParserStateActionItem テーブルを参照して次の状態へ遷移します。 3. **トークン抽出**: 文字クラス判定関数（p_is*）を用いて入力を読み進め、特定のフラグ（A_BINGO）に達した時点でトークンを確定させます。 4. **スタック管理**: URL や XML タグなどの複雑な構造を解析するため、内部で TParserPosition スタックを用いた再帰的な状態管理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: TParserInit、TParserCopyInit、TParserGet 等の関数が本構造体を操作します。
  • **呼び出し先**: TParserCharTest 型の関数ポインタ（p_is* 系関数）や TParserSpecial 型の特殊ハンドラが呼び出されます。

関連データ構造

  • TParserState: 解析の現在地を示す列挙型。
  • TParserStateActionItem: 状態遷移とアクションを定義するテーブル要素。
  • TParserPosition: 解析スタックを管理するための構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる可能性があります。
  • TParserCopy を使用する際、元の TParser が破棄された後にコピーを参照すると不正なメモリ参照が発生します。

保守担当者向けメモ

  • 本構造体は wparser_def.c 内の巨大な状態遷移テーブルと密接に連携しています。
  • WPARSER_TRACE マクロを有効にすることで、解析過程の詳細なデバッグ出力が可能です。
  • 新しいトークンタイプを追加する際は、p_is* 関数群と状態遷移テーブルの整合性に注意してください。
