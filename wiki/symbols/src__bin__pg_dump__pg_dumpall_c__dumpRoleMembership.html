dumpRoleMembership

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dumpall.c:1030

目的

クラスタ内のロールメンバーシップ（権限付与関係）を抽出し、適切な順序で SQL スクリプトとして出力する。

シグネチャまたは形状

static void dumpRoleMembership(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn（データベース接続ハンドル）
  • **出力**: なし（直接 OPF ストリームへ SQL を書き出す）

前提条件と不変条件

  • server_version に基づき、PostgreSQL 16 以降であれば grantor（付与者）および権限オプション（inherit_option, set_option）をダンプ対象とする。
  • ロールメンバーシップのダンプ順序は、依存関係（ADMIN OPTION を持つロールが先にダンプされる必要がある）を考慮して決定される。

アルゴリズムまたは振る舞い

1. pg_auth_members システムカタログからメンバーシップ情報を取得するクエリを生成・実行する。 2. ロールごとにメンバーシップをグループ化し、ハッシュテーブルを用いて「すでにダンプ可能な権限付与者」を追跡する。 3. 依存関係を解決するため、すべてのメンバーシップがダンプされるまで、許可された付与者を持つエントリを繰り返しスキャンする。 4. 循環参照や解決不能な順序が発生した場合はエラーを出力して終了する。 5. 存在しないロールや無効な付与者については警告を出力し、可能な限りダンプを継続する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dumpall.c 内のメイン処理フローから呼び出される。
  • **呼び出し先**: executeQuery, createPQExpBuffer, appendPQExpBuffer, rolename_create, PQntuples, PQgetvalue, pg_log_warning, pg_log_error 等のユーティリティ関数。

関連データ構造

  • PQExpBuffer: SQL クエリ構築用バッファ。
  • rolename_hash: メンバー名を管理するためのハッシュテーブル。
  • PGresult: クエリ結果を保持する構造体。

失敗モードと境界条件

  • **循環依存**: 権限付与の順序が解決できない場合、pg_log_error を出力し、exit_nicely(1) で異常終了する。
  • **古いサーバーバージョン**: 16 未満のバージョンでは grantor や権限オプションのダンプをスキップする。
  • **欠損データ**: 存在しないロールへの参照がある場合、警告を出力して該当エントリをスキップする。

保守担当者向けメモ

  • 本関数は pg_dumpall のダンプ順序制御において重要な役割を果たす。特に ADMIN OPTION の伝播順序を考慮したループ構造となっているため、修正時は依存関係のグラフ構造を破壊しないよう注意すること。
  • role_catalog 変数を使用してクエリを構築しているため、カタログ名が変更された場合の影響範囲を確認すること。
