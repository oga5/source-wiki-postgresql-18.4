pg_hmac_ctx

定義

  • 種別: struct
  • 定義箇所: src/common/hmac_openssl.c:59

目的

OpenSSL の HMAC_CTX をラップし、PostgreSQL のバックエンドおよびフロントエンド環境で HMAC 計算の状態を保持・管理する。

シグネチャまたは形状

struct pg_hmac_ctx
{
    HMAC_CTX   *hmacctx;
    pg_cryptohash_type type;
    pg_hmac_errno error;
    const char *errreason;

#ifdef USE_RESOWNER_FOR_HMAC
    ResourceOwner resowner;
#endif
};

入力と出力

  • **入力**: pg_hmac_create を通じて生成され、pg_hmac_init でハッシュアルゴリズムと鍵が設定される。
  • **出力**: pg_hmac_final を通じて計算結果の MAC 値が生成される。

前提条件と不変条件

  • USE_RESOWNER_FOR_HMAC が定義されている環境（バックエンド）では、ResourceOwner を介したリソース管理が行われる。
  • pg_hmac_free 呼び出し時に explicit_bzero を使用してコンテキスト内の機密情報を消去する。

アルゴリズムまたは振る舞い

1. **作成**: pg_hmac_create でメモリを割り当て、HMAC_CTX を初期化する。 2. **初期化**: pg_hmac_init で指定されたハッシュアルゴリズムと秘密鍵を OpenSSL のコンテキストに適用する。 3. **更新**: pg_hmac_update により、入力データをストリームとして処理する。 4. **終了**: pg_hmac_final で最終的な MAC 値を算出し、バッファサイズを検証する。 5. **解放**: pg_hmac_free で OpenSSL のコンテキストを破棄し、構造体メモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-scram.c（pg_hmac_create を使用）、src/common/hmac.c（コンテキストの生成、初期化、更新、終了、解放の各関数で利用）。
  • **呼び出し先**: OpenSSL ライブラリの HMAC_CTX 関連関数。

関連データ構造

  • HMAC_CTX (OpenSSL)
  • pg_cryptohash_type
  • pg_hmac_errno
  • ResourceOwner (バックエンド環境のみ)

失敗モードと境界条件

  • OpenSSL のエラー発生時は ERR_get_error() を通じてエラーキューを監視し、pg_hmac_error() で PostgreSQL 形式のエラーメッセージに変換する。
  • メモリ割り当て失敗時は ALLOC マクロの挙動に従う。

保守担当者向けメモ

  • 本構造体は src/common/hmac.c が提供する抽象化レイヤーの OpenSSL 具体実装である。
  • バックエンド環境では TopMemoryContext を使用し、フロントエンドでは malloc を使用するよう抽象化されている。
