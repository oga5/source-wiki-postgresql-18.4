plpgsql_exec_function

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_exec.c:492

目的

コンパイル済みの PL/pgSQL 関数を実行するためのメインエントリーポイント。実行状態の初期化、引数のローカル変数への割り当て、ステートメントの実行、および戻り値の処理を統括する。

シグネチャまたは形状

Datum plpgsql_exec_function(
    PLpgSQL_function *func,
    FunctionCallInfo fcinfo,
    EState *simple_eval_estate,
    ResourceOwner simple_eval_resowner,
    ResourceOwner procedure_resowner,
    bool atomic
)

入力と出力

  • **入力**:
  • func: 実行対象のコンパイル済み関数構造体。
  • fcinfo: 関数呼び出し情報（引数や戻り値情報を含む）。
  • simple_eval_estate / simple_eval_resowner: 式評価用の共有実行状態とリソースオーナー。
  • procedure_resowner: プロシージャ実行用のリソースオーナー。
  • atomic: トランザクションの原子性フラグ。
  • **出力**:
  • Datum: 関数の実行結果。

前提条件と不変条件

  • plpgsql_estate_setup により PLpgSQL_execstate が初期化されていること。
  • error_context_stack にエラーコールバックが登録され、実行中のエラー追跡が可能であること。
  • 関数終了時に exec_toplevel_block が PLPGSQL_RC_RETURN を返さない場合、制御が関数の末尾に到達したとみなされエラーが発生する。

アルゴリズムまたは振る舞い

1. **初期化**: plpgsql_estate_setup で実行状態を構築し、エラーコールバックを設定する。 2. **データ準備**: copy_plpgsql_datums でローカル変数のコピーを作成する。 3. **引数割り当て**: func->fn_nargs 分の引数をループし、PLPGSQL_DTYPE_VAR（単純変数）または PLPGSQL_DTYPE_REC（レコード）としてローカル変数に格納する。この際、R/W 拡張オブジェクトの所有権取得や配列の展開を行う。 4. **実行**: exec_set_found で FOUND 変数を初期化し、プラグインが有効であれば func_beg を呼び出す。その後、exec_toplevel_block を実行する。 5. **戻り値処理**: 戻り値の型変換を行い、集合を返す関数（Set-returning function）の場合は ReturnSetInfo を検証して SFRM_Materialize モードで結果を処理する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpgsql/src/pl_handler.c (PL/pgSQL ハンドラ)
  • **呼び出し先**:
  • plpgsql_estate_setup, copy_plpgsql_datums, assign_simple_var, exec_move_row_from_datum, exec_move_row, exec_eval_cleanup, exec_set_found, exec_toplevel_block, TransferExpandedObject, expand_array など。

関連データ構造

  • PLpgSQL_execstate: 実行中の関数の状態を保持する。
  • PLpgSQL_function: コンパイル済みの関数定義。
  • FunctionCallInfo: PostgreSQL の関数呼び出しインターフェース。

失敗モードと境界条件

  • **戻り値なし**: exec_toplevel_block が PLPGSQL_RC_RETURN を返さずに終了した場合、ERRCODE_S_R_E_FUNCTION_EXECUTED_NO_RETURN_STATEMENT エラーを発生させる。
  • **型不一致**: 引数や戻り値の型変換に失敗した場合、エラーが発生する。
  • **集合返却の不整合**: 集合を返さないコンテキストで集合を返そうとした場合、ERRCODE_FEATURE_NOT_SUPPORTED エラーを発生させる。

保守担当者向けメモ

  • varlena 型の変数に対しては、R/W 拡張オブジェクトの最適化（TransferExpandedObject）や配列の強制展開が行われる。このロジックはパフォーマンスに直結するため、変更時はメモリコンテキストの管理に注意すること。
  • exec_eval_cleanup は exec_move_row 等の後に呼び出され、一時的なメモリを解放する。呼び出し漏れはメモリリークの原因となる。
