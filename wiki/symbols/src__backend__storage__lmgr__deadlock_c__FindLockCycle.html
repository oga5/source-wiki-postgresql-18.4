FindLockCycle

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/deadlock.c:445

目的

指定されたプロセス（checkProc）を起点として、ロック待ちグラフ上の循環参照（デッドロック）を探索します。

シグネチャまたは形状

static bool FindLockCycle(PGPROC *checkProc, EDGE *softEdges, int *nSoftEdges)

入力と出力

  • **入力**:
  • checkProc: 探索を開始する対象のプロセス（PGPROC）。
  • softEdges: 検出されたソフトエッジ（解決可能な競合）を格納する配列へのポインタ。
  • nSoftEdges: 検出されたソフトエッジの数を格納する整数へのポインタ。
  • **出力**:
  • 戻り値: 循環参照が検出された場合は true、それ以外は false。
  • softEdges および nSoftEdges: 探索中に特定されたソフトエッジ情報が書き込まれます。

前提条件と不変条件

  • 探索開始前に nVisitedProcs および nDeadlockDetails が 0 にリセットされます。
  • nSoftEdges は探索開始時に 0 に初期化されます。

アルゴリズムまたは振る舞い

1. 探索用カウンタ（nVisitedProcs、nDeadlockDetails）を初期化します。 2. nSoftEdges を 0 に設定します。 3. FindLockCycleRecurse を呼び出し、再帰的にロック待ちグラフを探索して循環参照の有無を判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/deadlock.c 内の各所（DeadLockCheck 等から、デッドロック検出および制約の検証のために呼び出されます）。
  • **呼び出し先**:
  • FindLockCycleRecurse（直接の再帰探索を実行）。

関連データ構造

  • PGPROC: ロック待ち状態を保持するプロセス構造体。
  • EDGE: 待ちグラフ上のエッジ（waiterとblockerの関係）。

失敗モードと境界条件

  • 循環参照が存在しない場合、false を返します。
  • 探索中にメモリやスタックの制限に達した場合の挙動については、本関数単体では定義されていません。

保守担当者向けメモ

  • 本関数は DeadLockCheck の主要な探索ロジックの一部です。
  • 探索結果として得られる softEdges は、デッドロック解消のための待ち行列順序の再構成に使用されます。
