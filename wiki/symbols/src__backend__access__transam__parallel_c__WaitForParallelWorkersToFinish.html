WaitForParallelWorkersToFinish

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/parallel.c:802

目的

並列クエリ実行において、起動されたすべてのワーカープロセスが終了するまで待機し、必要に応じてワーカーから伝播されたエラーを処理し、最終的なトランザクション状態を同期する。

シグネチャまたは形状

void WaitForParallelWorkersToFinish(ParallelContext *pcxt)

入力と出力

  • **入力**: pcxt (並列実行コンテキストを保持する ParallelContext 構造体へのポインタ)
  • **出力**: なし（正常終了時はワーカーの完了を待機し、エラー時は例外をスローする）

前提条件と不変条件

  • pcxt は有効な並列コンテキストであること。
  • リーダープロセスは、本関数を呼び出す前にワーカーを起動済みであること。
  • ワーカーがエラーを報告した場合、本関数内で ereport(ERROR, ...) が発生し、処理が中断される。

アルゴリズムまたは振る舞い

1. **ループ待機**: すべてのワーカーが終了するまで以下の処理を繰り返す。

  • CHECK_FOR_INTERRUPTS() を呼び出し、保留中のメッセージ処理や割り込みチェックを行う。
  • 各ワーカーの error_mqh を確認し、終了済みか稼働中かを判定する。
  • すべてのワーカーが終了していればループを抜ける。
  • 稼働中のワーカーがないにもかかわらず終了済みとみなせない場合、ワーカーの起動失敗や異常終了を検知し、必要に応じてエラーをスローする。
  • 待機が必要な場合は WaitLatch を使用して待機し、ラッチをリセットする。

    • 2. **状態同期**: ループ終了後、pcxt->toc が存在する場合、共有メモリ上の FixedParallelState から last_xlog_end を取得し、リーダーの XactLastRecEnd を更新する。

関連データ構造

  • ParallelContext: 並列実行の全体状態を管理する構造体。
  • FixedParallelState: ワーカーとリーダー間で共有される固定サイズのトランザクション状態。
  • shm_mq: ワーカーとの通信に使用される共有メモリキュー。

失敗モードと境界条件

  • **ワーカーの初期化失敗**: ワーカーがエラーキューにアタッチする前に終了した場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラーをスローする。
  • **ワーカーの異常終了**: ワーカーがエラーを報告した場合、リーダー側でそのエラーが伝播される。
  • **ポストマスターの終了**: WL_EXIT_ON_PM_DEATH により、ポストマスターが終了した場合は待機を中断する。

保守担当者向けメモ

  • 本関数はリーダープロセスがワーカーの完了を待つための主要な同期ポイントである。
  • CHECK_FOR_INTERRUPTS() をループの先頭で呼び出すことで、ワーカーから送られてきたエラーメッセージを即座に処理する設計となっている。
  • XactLastRecEnd の更新は、並列実行中にワーカーが行った WAL 書き込みをリーダーに反映させるために不可欠である。
