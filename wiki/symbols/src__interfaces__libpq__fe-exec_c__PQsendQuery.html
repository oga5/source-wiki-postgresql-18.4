PQsendQuery

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-exec.c:1432

目的

PostgreSQLバックエンドに対してSQLクエリを非同期に送信します。

シグネチャまたは形状

int PQsendQuery(PGconn *conn, const char *query);

入力と出力

  • **入力**:
  • conn: 確立されたデータベース接続を指す PGconn ポインタ。
  • query: 実行するSQLコマンド文字列。
  • **出力**:
  • 成功した場合は 1、失敗した場合は 0 を返します。

前提条件と不変条件

  • conn は有効な接続状態である必要があります。
  • 内部的に PQsendQueryInternal を呼び出し、第3引数に true を渡して処理を委譲します。

アルゴリズムまたは振る舞い

1. 内部関数 PQsendQueryInternal を呼び出します。 2. 接続状態の確認およびプロトコルに応じたメッセージ構築を行い、バックエンドへクエリを送信します。 3. 非同期実行のため、クエリ送信直後に制御を呼び出し元へ戻します。結果の取得には別途 PQgetResult を使用する必要があります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • contrib/postgres_fdw/connection.c
  • contrib/postgres_fdw/postgres_fdw.c
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_upgrade/task.c
  • src/bin/pgbench/pgbench.c
  • **呼び出し先**:
  • PQsendQueryInternal (直接の内部委譲先)

関連データ構造

  • PGconn: 接続状態を保持する構造体。

失敗モードと境界条件

  • 接続が確立されていない場合や、送信中にエラーが発生した場合は 0 を返します。
  • 詳細は PQerrorMessage を通じて取得可能です。

保守担当者向けメモ

  • 本関数は PQsendQueryInternal のラッパーです。ロジックの変更が必要な場合は、fe-exec.c 内の PQsendQueryInternal の実装を確認してください。
  • 非同期インターフェースであるため、呼び出し側で PQgetResult をループさせて結果を完全に消費する責任があります。
