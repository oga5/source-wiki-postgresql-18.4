replace_token

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/initdb.c:472

目的

文字列配列（char **）内の各要素に含まれる特定のトークンを、指定した置換文字列に置き換える。

シグネチャまたは形状

static char **replace_token(char **lines, const char *token, const char *replacement)

入力と出力

  • **入力**:
  • lines: NULL で終端された文字列の配列。
  • token: 検索対象の文字列。
  • replacement: 置換後の文字列。
  • **出力**:
  • 置換処理済みの lines ポインタを返す。

前提条件と不変条件

  • lines は NULL で終端されている必要がある。
  • lines 内の各文字列は malloc で確保されたメモリ領域である必要がある（関数内で free されるため）。

アルゴリズムまたは振る舞い

1. token と replacement の長さの差分（diff）を計算する。 2. lines 配列を順に走査し、各文字列内に token が含まれているか確認する。 3. token が見つかった場合、以下の手順で新しい文字列を生成する：

  • strlen(lines[i]) + diff + 1 のサイズでメモリを確保する。
  • 元の文字列の token 出現位置までをコピーする。
  • replacement をコピーする。
  • token 以降の残りの文字列をコピーする。

    • 4. 元の文字列を free し、新しい文字列で置き換える。 5. 走査終了後、更新された lines を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/initdb/initdb.c 内の複数箇所（conflines や bki_lines の処理）。
  • **呼び出し先**: strlen, strstr, pg_malloc, memcpy, strcpy, free（標準ライブラリおよび PostgreSQL ユーティリティ関数）。

関連データ構造

  • char **: 文字列のリストを保持する配列。

失敗モードと境界条件

  • lines が空の場合、ループは実行されずそのまま返される。
  • token が文字列内に存在しない場合、その要素は変更されない。
  • pg_malloc が失敗した場合、プログラムは終了する（pg_malloc の仕様による）。

保守担当者向けメモ

  • 本関数は lines 配列内の各文字列をインプレースで更新する。呼び出し側は lines のメモリ管理に注意すること。
  • 文字列の再確保を行っているため、頻繁な呼び出しはパフォーマンスに影響する可能性がある。
