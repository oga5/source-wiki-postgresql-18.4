gbt_oid_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_oid.c:135

目的

GiSTインデックスにおいて、Oid 型のインデックスキーとクエリ条件を照合し、検索対象のノードが条件に適合するかを判定します。

シグネチャまたは形状

Datum gbt_oid_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY 構造体（インデックスキーを含む）。
  • PG_GETARG_OID(1): 検索対象の Oid 値。
  • PG_GETARG_UINT16(2): 比較演算の種類を示す StrategyNumber。
  • PG_GETARG_POINTER(4): recheck フラグへのポインタ。
  • **出力**:
  • Datum (bool): 適合する場合は true、しない場合は false。
  • *recheck: 常に false を設定（本関数による判定が完全であるため）。

前提条件と不変条件

  • oidKEY 構造体は Oid の範囲（lower, upper）を保持します。
  • リーフノードにおいては lower == upper であることが期待されます。

アルゴリズムまたは振る舞い

1. GISTENTRY から oidKEY を取得し、GBT_NUMKEY_R 構造体に変換します。 2. recheck フラグを false に設定します。 3. btree_utils_num.h で定義された汎用関数 gbt_num_consistent を呼び出し、範囲とクエリ値、戦略を照合します。 4. 判定結果を PG_RETURN_BOOL で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのGiSTインデックスエンジン（直接の呼び出し元情報はソースコード上では明示されていません）。
  • **呼び出し先**: gbt_num_consistent（contrib/btree_gist/btree_utils_num.c 内の汎用関数）。

関連データ構造

  • oidKEY: Oid の範囲を保持する構造体。
  • GBT_NUMKEY_R: 数値範囲を扱うための汎用構造体。
  • tinfo: Oid 型固有の比較関数を保持する gbtree_ninfo 構造体。

失敗モードと境界条件

  • 本関数は常に recheck = false を返すため、インデックスによる判定結果は完全であるとみなされます。
  • 入力値が不正な場合、PostgreSQLの標準的なエラーハンドリング（PG_GETARG マクロ等）により例外が発生します。

保守担当者向けメモ

  • ほとんどのロジックは btree_utils_num.c に委譲されています。複雑な判定ロジックの変更が必要な場合は、まず btree_utils_num.c を確認してください。
