test_file_descriptor_sync

定義

  • 種別: function
  • 定義箇所: src/bin/pg_test_fsync/pg_test_fsync.c:502

目的

異なるファイル記述子（file descriptor）を介して書き込まれたデータに対して fsync を実行した際の挙動と効率を検証する。これは、マルチプロセス環境で同一ファイルに対して fsync を行う際の性能特性を評価することを目的としている。

シグネチャまたは形状

static void test_file_descriptor_sync(void)

入力と出力

  • 入力: なし（グローバル変数 filename および buf を使用）
  • 出力: なし（標準出力へテスト結果を表示）

前提条件と不変条件

  • filename で指定されたファイルが書き込み可能であること。
  • buf に書き込み用のデータが準備されていること。
  • alarm_triggered フラグにより、一定時間の計測が制御されること。

アルゴリズムまたは振る舞い

1. **ベースライン計測**: ファイルを開き、書き込み、fsync を実行して閉じるという一連の操作を、計測期間終了まで繰り返す。 2. **記述子跨ぎの計測**: ファイルを開き、書き込み、閉じる。その後、再度ファイルを開き直して fsync を実行し、閉じるという操作を繰り返す。 3. 各計測フェーズにおいて、START_TIMER から STOP_TIMER までの間に実行された操作回数（ops）を計測する。 4. 処理中にエラーが発生した場合は die() を呼び出し、プログラムを終了させる。

呼び出し元と呼び出し先

  • 呼び出し元: main 関数（src/bin/pg_test_fsync/pg_test_fsync.c:136）から呼び出される。
  • 呼び出し先: open, write, fsync, close, printf, fflush, die などの標準ライブラリおよびユーティリティ関数。

関連データ構造

  • filename: テスト対象のファイルパスを保持する文字列。
  • buf: 書き込みに使用されるデータバッファ。
  • alarm_triggered: 計測期間の終了を判定するためのブール値フラグ。

失敗モードと境界条件

  • ファイルのオープン失敗: open() が -1 を返した場合、die() によりエラー終了する。
  • 書き込み失敗: write() が期待されるバイト数（XLOG_BLCKSZ）を書き込めなかった場合、die() によりエラー終了する。
  • 同期失敗: fsync() が 0 以外を返した場合、die() によりエラー終了する。

保守担当者向けメモ

  • 本関数は、プラットフォームごとの fsync の挙動差異を可視化する重要なテストケースである。
  • writethrough サポート環境での挙動については、将来的な拡張の余地がある旨のコメントがソース内に存在する。
