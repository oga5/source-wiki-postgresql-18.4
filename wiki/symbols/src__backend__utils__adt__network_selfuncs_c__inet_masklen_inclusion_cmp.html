inet_masklen_inclusion_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_selfuncs.c:922

目的

ネットワークアドレス（inet型）のマスク長に基づき、指定された演算子が包含関係を満たすかどうかを判定し、比較結果を返します。

シグネチャまたは形状

static int inet_masklen_inclusion_cmp(inet *left, inet *right, int opr_codenum)

入力と出力

  • **入力**:
  • left, right: 比較対象の inet 型ポインタ。
  • opr_codenum: 演算子の種類を示す数値（inet_opr_codenum で変換されたもの）。
  • **出力**:
  • 0: 演算子がマスク長の包含関係を満たす場合。
  • 0以外: 満たさない場合。opr_codenum の値をそのまま返します。

前提条件と不変条件

  • opr_codenum が 0（オーバーラップ）の場合、マスク長の大小に関わらず常に 0 を返します。
  • ip_bits() 関数により取得されるマスク長が比較の基準となります。

アルゴリズムまたは振る舞い

1. left と right のマスク長の差（order）を算出します。 2. 以下の条件のいずれかを満たす場合、包含関係が成立すると見なし 0 を返します。

  • order > 0 かつ opr_codenum >= 0
  • order == 0 かつ opr_codenum が -1 から 1 の範囲内
  • order < 0 かつ opr_codenum <= 0

    • 3. 上記以外の場合は、opr_codenum を返します。これは呼び出し側で包含関係の成否を判断するための指標として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network_selfuncs.c 内の複数の箇所（行70, 848, 894, 908, 960付近）から参照・呼び出しされています。
  • **呼び出し先**:
  • ip_bits()（utils/inet.h 経由で提供されるマクロまたは関数と推測されますが、本ソース内での定義詳細は不明です）。

関連データ構造

  • inet: PostgreSQL のネットワークアドレス型。

失敗モードと境界条件

  • opr_codenum が 0 の場合は常に成功（0 を返却）とみなされます。
  • マスク長の差が 0 の場合、opr_codenum が -1 から 1 の範囲外であれば包含関係なしと判定されます。

保守担当者向けメモ

  • 本関数は inet_hist_match_divider と共に、ネットワーク統計の包含関係推定において重要な役割を果たします。
  • 戻り値の opr_codenum は、単なる真偽値ではなく、呼び出し側で包含関係の方向性を区別するために利用されている点に注意してください。
