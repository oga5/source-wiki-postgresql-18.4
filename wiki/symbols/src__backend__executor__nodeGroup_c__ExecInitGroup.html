ExecInitGroup

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGroup.c:160

目的

Group ノードの実行状態（GroupState）を初期化し、ソート済み入力に対するグループ化処理の準備を行う。

シグネチャまたは形状

GroupState *ExecInitGroup(Group *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の Group プランノード。
  • estate: 実行状態（EState）。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの GroupState 構造体へのポインタ。

前提条件と不変条件

  • 入力となる子プランは、グループ化キーに従ってソート済みである必要がある。
  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK が含まれていないこと（アサーションで検証）。

アルゴリズムまたは振る舞い

1. GroupState 構造体を割り当て、ExecGroup を実行関数として設定する。 2. 式コンテキスト（ExprContext）を作成する。 3. ExecInitNode を呼び出し、子プランノードを初期化する。 4. 子プランの出力スロットに基づき、スキャン用スロットを作成する。 5. 結果タプルスロットと射影情報（ProjectionInfo）を初期化する。 6. HAVING 句（qual）を初期化する。 7. execTuplesMatchPrepare を呼び出し、グループ化キー比較用の関数ポインタ（eqfunction）を事前計算してキャッシュする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execProcnode.c:336（ExecInitNode 経由での呼び出しが示唆される）
  • **呼び出し先**:
  • ExecInitNode（子プランの初期化）
  • ExecAssignExprContext
  • ExecCreateScanSlotFromOuterPlan
  • ExecInitResultTupleSlotTL
  • ExecAssignProjectionInfo
  • ExecInitQual
  • execTuplesMatchPrepare

関連データ構造

  • GroupState: グループ化処理の実行状態を保持する構造体。
  • Group: プランノードの型。
  • TupleTableSlotOps: タプル操作のインターフェース。

失敗モードと境界条件

  • eflags にサポート外のフラグ（EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK）が渡された場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数はソート済み入力を前提としている。プランナがソートを保証できない場合は、上位で Sort ノードが挿入されている必要がある。
  • eqfunction のキャッシュは、実行時のループ内での比較コストを削減するために重要である。
