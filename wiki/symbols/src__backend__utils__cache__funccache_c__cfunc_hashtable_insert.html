cfunc_hashtable_insert

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/funccache.c:166

目的

コンパイル済み関数オブジェクトをグローバルな関数キャッシュハッシュテーブルに登録し、キャッシュエントリと関数構造体間の相互参照を確立する。

シグネチャまたは形状

static void cfunc_hashtable_insert(CachedFunction *function, CachedFunctionHashKey *func_key)

入力と出力

  • **入力**:
  • function: 登録対象の CachedFunction 構造体へのポインタ。
  • func_key: キャッシュの検索キーとなる CachedFunctionHashKey 構造体へのポインタ。
  • **出力**: なし（副作用としてハッシュテーブルの状態を更新）。

前提条件と不変条件

  • cfunc_hashtable が初期化されていない場合、内部で cfunc_hashtable_init() が呼び出される。
  • func_key に callResultType が含まれる場合、その内容は TopMemoryContext にコピーされる。

アルゴリズムまたは振る舞い

1. ハッシュテーブルが未初期化であれば初期化する。 2. hash_search を使用して func_key でハッシュテーブルを検索し、エントリ（hentry）を確保する（HASH_ENTER）。 3. 既に同一キーのエントリが存在する場合は WARNING をログ出力する。 4. func_key->callResultType が存在する場合、メモリコンテキストを TopMemoryContext に切り替え、CreateTupleDescCopy を用いてタプル記述子を永続メモリ領域へコピーする。 5. hentry->function に function ポインタを格納する。 6. function->fn_hashkey にハッシュテーブル内のキーへのポインタを設定し、相互参照を確立する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/funccache.c 内の cached_function_compile から呼び出されることが確認されている。
  • **呼び出し先**:
  • cfunc_hashtable_init()
  • hash_search()
  • MemoryContextSwitchTo()
  • CreateTupleDescCopy()
  • elog()

関連データ構造

  • CachedFunction: キャッシュされる関数の実体。
  • CachedFunctionHashKey: キャッシュ検索用のキー構造体。
  • CachedFunctionHashEntry: ハッシュテーブルに格納されるエントリ構造体。
  • cfunc_hashtable: グローバルなハッシュテーブル変数。

失敗モードと境界条件

  • hash_search によるエントリ確保失敗時の挙動は基盤ライブラリに依存する。
  • callResultType のコピー時にメモリ不足が発生した場合、hentry->key.callResultType は NULL に設定される。この状態ではキャッシュヒットが発生しなくなる可能性がある。
  • 既に存在するキーを挿入しようとした場合、WARNING が発生するが処理は継続される。

保守担当者向けメモ

  • 本関数は TopMemoryContext を操作するため、メモリリークを防ぐためにコンテキストの切り替えが正しく行われていることを確認すること。
  • fn_hashkey へのバックリンク設定は、キャッシュエントリの生存期間と関数オブジェクトの生存期間が一致することを前提としている。
