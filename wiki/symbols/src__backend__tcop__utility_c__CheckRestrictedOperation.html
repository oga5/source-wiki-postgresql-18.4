CheckRestrictedOperation

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/utility.c:458

目的

セキュリティ制限下にある操作（security-restricted operation）の実行を禁止し、該当するコマンドが呼び出された場合にエラーを発生させる。

シグネチャまたは形状

static void CheckRestrictedOperation(const char *cmdname)

入力と出力

  • **入力**: cmdname (実行しようとしているSQLコマンド名を示す文字列)
  • **出力**: なし（正常時は復帰、違反時はエラーをスロー）

前提条件と不変条件

  • InSecurityRestrictedOperation() が真を返す状態（セキュリティ制限下）では、本関数を呼び出したコマンドの実行は許可されない。

アルゴリズムまたは振る舞い

1. InSecurityRestrictedOperation() を呼び出し、現在の実行コンテキストがセキュリティ制限下にあるかを確認する。 2. 制限下にある場合、ereport を使用して ERRCODE_INSUFFICIENT_PRIVILEGE エラーを発生させる。 3. エラーメッセージには、引数として渡された cmdname が含まれる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c 内の各コマンド実行処理。具体的には以下のコマンド実行時に呼び出されることが確認されている。
  • CLOSE (700行目)
  • PREPARE (747行目)
  • DEALLOCATE (760行目)
  • LISTEN (809行目)
  • UNLISTEN (835行目)
  • DISCARD (888行目)
  • **呼び出し先**: InSecurityRestrictedOperation() (外部関数)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • **失敗モード**: InSecurityRestrictedOperation() が真を返す場合、常に ERROR レベルで処理が中断される。

保守担当者向けメモ

  • 本関数は、セキュリティ上の懸念があるコマンドを特定のコンテキスト（例：イベントトリガー実行中など）で実行させないためのガードレールとして機能する。
  • 新しいコマンドを追加する際、セキュリティ制限下での実行が不適切であると判断される場合は、本関数を呼び出すこと。
