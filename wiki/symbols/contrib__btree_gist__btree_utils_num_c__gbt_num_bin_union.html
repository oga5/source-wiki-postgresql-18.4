gbt_num_bin_union

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_utils_num.c:227

目的

GiST インデックスのノード分割時などに、既存の数値範囲（Datum）と新しい数値キーを統合し、包含する最小の範囲を更新します。

シグネチャまたは形状

void gbt_num_bin_union(Datum *u, GBT_NUMKEY *e, const gbtree_ninfo *tinfo, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • u: 更新対象の範囲データ（Datum）。初期状態ではポインタが未割り当て（!DatumGetPointer(*u)）である可能性があります。
  • e: 統合対象の新しい数値キー（GBT_NUMKEY）。
  • tinfo: データ型固有の比較関数やサイズ情報を含む gbtree_ninfo 構造体。
  • flinfo: 関数呼び出し情報（FmgrInfo）。
  • **出力**:
  • u: 統合後の範囲データが格納されます。

前提条件と不変条件

  • u が指す Datum は、初期化されていない場合は palloc0 によって tinfo->indexsize 分のメモリが確保されます。
  • e は tinfo で定義されたサイズに基づき、下限値と上限値のペアとして解釈されます。

アルゴリズムまたは振る舞い

1. e から下限値（lower）と上限値（upper）を抽出します。 2. u が未初期化の場合:

  • tinfo->indexsize 分のメモリを確保し、e の値をコピーして初期範囲とします。

    • 3. u が既に初期化されている場合:

  • 現在の範囲（ur）と新しいキー（rd）を比較します。
  • tinfo->f_gt を使用して ur.lower が rd.lower より大きい場合、ur.lower を rd.lower で更新します。
  • tinfo->f_lt を使用して ur.upper が rd.upper より小さい場合、ur.upper を rd.upper で更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_utils_num.c 内の gbt_num_picksplit 関数から、インデックス分割時の左右のページ範囲を計算するために呼び出されます。
  • **呼び出し先**:
  • tinfo 構造体内の関数ポインタ（f_gt, f_lt）を介して、各データ型固有の比較関数を呼び出します。

関連データ構造

  • gbtree_ninfo: 数値型の比較関数やサイズ情報を保持する構造体。
  • GBT_NUMKEY: インデックスに格納される数値キーの型。
  • GBT_NUMKEY_R: 範囲（下限・上限）を扱うための内部構造体。

失敗モードと境界条件

  • palloc0 によるメモリ確保が失敗した場合、PostgreSQL の標準的なエラーハンドリングにより例外が発生します。
  • tinfo が正しく設定されていない場合、比較関数呼び出し時に未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は gbt_num_picksplit における範囲結合のコアロジックです。
  • unconstify マクロを使用して const 修飾を解除し、範囲の更新を行っています。
  • 新しい数値型を追加する際は、gbtree_ninfo の比較関数が正しく実装されていることを確認してください。
