_dumpPreparedQueries

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_dump/pg_backup.h:64

目的

pg_dump において実行されるプリペアドクエリを識別するための列挙型です。各定数は、特定のデータベースオブジェクト（型、関数、テーブル等）のダンプ処理に関連するクエリを指します。

シグネチャまたは形状

enum _dumpPreparedQueries
{
    PREPQUERY_DUMPAGG,
    PREPQUERY_DUMPBASETYPE,
    PREPQUERY_DUMPCOMPOSITETYPE,
    PREPQUERY_DUMPDOMAIN,
    PREPQUERY_DUMPENUMTYPE,
    PREPQUERY_DUMPFUNC,
    PREPQUERY_DUMPOPR,
    PREPQUERY_DUMPRANGETYPE,
    PREPQUERY_DUMPTABLEATTACH,
    PREPQUERY_GETATTRIBUTESTATS,
    PREPQUERY_GETCOLUMNACLS,
    PREPQUERY_GETDOMAINCONSTRAINTS,
};

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし（列挙型定義）

前提条件と不変条件

  • 本列挙型の各要素は、プリペアドクエリの準備状態を管理するためのインデックスとして使用されます。

アルゴリズムまたは振る舞い

  • 本列挙型自体はロジックを持ちませんが、bool *is_prepared 配列のインデックスとして使用され、特定のクエリが既に準備済みであるかを追跡するために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/pg_backup.h:255 にて、is_prepared 配列のインデックスとして参照されています。
  • 呼び出し先: なし（列挙型定義のため）

関連データ構造

  • Archive 構造体（pg_backup.h 内で定義）: 本列挙型を用いた準備状態管理フラグを保持するコンテキストとして関連します。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）

保守担当者向けメモ

  • 新しいプリペアドクエリを pg_dump に追加する際は、本列挙型に新しい要素を追加し、それに対応する is_prepared 配列のサイズや初期化処理が適切に行われているか確認してください。
