lquery_in

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_io.c:758

目的

PostgreSQL の lquery 型に対する標準入力関数として、文字列形式のクエリパターンを内部バイナリ形式に変換する。

シグネチャまたは形状

Datum lquery_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_POINTER(0) を介して取得される char * 型の文字列（クエリパターン）。
  • **出力**: Datum 型として返される lquery *（内部バイナリ形式の構造体）。解析失敗時は NULL を返す。

前提条件と不変条件

  • 入力文字列は lquery 型として妥当な構文（メタ文字 *, |, !, {} 等を含む）である必要がある。
  • parse_lquery がメモリ確保を行うため、PostgreSQL のメモリコンテキストが適切に設定されている必要がある。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) で入力文字列を取得する。 2. parse_lquery を呼び出し、文字列を解析して lquery 構造体を構築する。 3. 解析が成功した場合は、構築された lquery 構造体へのポインタを PG_RETURN_POINTER で返す。 4. 解析が失敗した場合は、PG_RETURN_NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の型システム（SQL インターフェース経由）。PG_FUNCTION_INFO_V1(lquery_in) により登録されている。
  • **呼び出し先**: parse_lquery（文字列解析および構造体構築を担当）。

関連データ構造

  • lquery: 解析結果を格納する内部構造体。
  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約に基づく引数コンテキスト。

失敗モードと境界条件

  • 入力文字列が lquery の構文規則に違反している場合、parse_lquery 内でエラーが発生する。
  • メモリ不足等の理由で parse_lquery が NULL を返した場合、本関数も NULL を返す。

保守担当者向けメモ

  • 本関数は lquery 型の入出力インターフェースの入り口である。構文解析の詳細は parse_lquery に委譲されているため、解析ロジックの変更はそちらを参照すること。
  • lquery は複雑なメタ文字をサポートしているため、解析ロジックの変更時には ltree.h の構造体定義との整合性に注意が必要である。
