FileSetDelete

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/fileset.c:135

目的

FileSet 内の指定された名前の一時ファイルをファイルシステムから削除する。

シグネチャまたは形状

bool FileSetDelete(FileSet *fileset, const char *name, bool error_on_failure)

入力と出力

  • **入力**:
  • fileset: 対象となる FileSet 構造体へのポインタ。
  • name: 削除対象のファイル名。
  • error_on_failure: 削除に失敗した場合にエラーを発生させるかどうかを示すフラグ。
  • **出力**:
  • bool: 削除が成功した場合は true、失敗した場合は false を返す。

前提条件と不変条件

  • fileset は FileSetInit によって初期化済みであること。
  • 指定された name は、当該 FileSet 内で作成された一時ファイルに対応していること。

アルゴリズムまたは振る舞い

1. FilePath を呼び出し、fileset と name から物理的なファイルパスを構築する。 2. 構築したパスを引数として PathNameDeleteTemporaryFile を呼び出し、実際のファイル削除を実行する。 3. PathNameDeleteTemporaryFile の戻り値をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/file/buffile.c (複数の箇所で呼び出しを確認)
  • **呼び出し先**:
  • FilePath (同ファイル内)
  • PathNameDeleteTemporaryFile (外部モジュール)

関連データ構造

  • FileSet: 一時ファイルの集合を管理する構造体。

失敗モードと境界条件

  • 指定されたファイルが存在しない場合、error_on_failure が true であればエラーが発生し、false であれば false が返される。
  • ファイルシステム上の権限不足やI/Oエラーが発生した場合、PathNameDeleteTemporaryFile の挙動に従い、error_on_failure に応じた処理が行われる。

保守担当者向けメモ

  • 本関数はトランザクション終了時に自動実行されないため、呼び出し側で明示的にリソース解放を行う必要がある。
  • PathNameDeleteTemporaryFile の仕様変更が本関数の挙動に直接影響を与える。
