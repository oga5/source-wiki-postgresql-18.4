gbt_float4ge

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_float4.c:33

目的

GiSTインデックスにおいて、2つの float4 値を比較し、左辺が右辺以上であるか（>=）を判定します。

シグネチャまたは形状

bool gbt_float4ge(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の float4 値へのポインタ。
  • b: 比較対象の float4 値へのポインタ。
  • flinfo: 関数呼び出し情報（本関数内では未使用）。
  • **出力**:
  • bool: *a >= *b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • ポインタ a および b は、有効な float4 型のメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. ポインタ a および b を const float4 * 型にキャストします。 2. それぞれのポインタが指す値をデリファレンスし、C言語の >= 演算子を用いて比較結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_float4.c 内の静的構造体 tinfo の初期化時に参照されています（84行目）。
  • **呼び出し先**: 直接的な関数呼び出しはありません。

関連データ構造

  • float4: PostgreSQLの単精度浮動小数点数型。
  • FmgrInfo: PostgreSQLの関数マネージャ情報構造体。

失敗モードと境界条件

  • a または b が NULL ポインタである場合、セグメンテーション違反が発生する可能性があります。
  • float4 値が NaN（Not a Number）である場合の比較結果は、標準的な浮動小数点演算の規則に従います。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で提供される汎用的な数値比較ロジックを補完するために定義されたラッパーです。
  • float4 型の比較ロジックを変更する場合は、本関数と対になる他の比較関数（gbt_float4gt, gbt_float4le, gbt_float4lt, gbt_float4eq）との整合性に注意してください。
