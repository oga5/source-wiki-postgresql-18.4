hlCheck

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/wparser_def.c:1966

目的

checkcondition_HL 関数において、ハイライト対象となる単語リストとその長さを保持するためのコールバックデータ構造です。

シグネチャまたは形状

typedef struct {
    HeadlineWordEntry *words;
    int         len;
} hlCheck;

入力と出力

  • words: HeadlineWordEntry 型の配列へのポインタ。検索対象の単語情報を保持します。
  • len: words 配列に含まれる要素数。

前提条件と不変条件

  • words は有効なメモリ領域を指している必要があります。
  • len は 0 以上の整数である必要があります。

アルゴリズムまたは振る舞い

本構造体は、検索結果のハイライト生成プロセスにおいて、条件判定用のコンテキストとして利用されます。opaque ポインタを介してキャストされ、checkcondition_HL 内で検索条件と単語リストを照合するために参照されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tsearch/wparser_def.c 内の checkcondition_HL 関数および関連するハイライト処理ロジック（2153行目、2696行目などでインスタンス化）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • HeadlineWordEntry: ハイライト対象の単語情報を格納する構造体。

失敗モードと境界条件

  • words が NULL の状態で参照された場合、セグメンテーション違反が発生する可能性があります。
  • len が実際の配列サイズと一致しない場合、範囲外アクセスが発生するリスクがあります。

保守担当者向けメモ

  • 本構造体は checkcondition_HL の実行コンテキストに強く依存しています。この構造体のフィールドを変更する場合は、必ず checkcondition_HL 内の参照ロジックと整合性を確認してください。
