hstore_subscript_fetch

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_subs.c:93

目的

hstore 型のデータに対して添字演算（hstore_val['key']）が行われた際、指定されたキーに対応する値を抽出して実行エンジンに返す。

シグネチャまたは形状

static void hstore_subscript_fetch(ExprState *state, ExprEvalStep *op, ExprContext *econtext)

入力と出力

  • **入力**:
  • op->resvalue: 処理対象の hstore データ（Datum）。
  • sbsrefstate->upperindex[0]: 添字として指定されたキー（text 型）。
  • **出力**:
  • op->resvalue: 抽出された値（text 型）。
  • op->resnull: キーが存在しない、または添字が NULL の場合に true に設定される。

前提条件と不変条件

  • 呼び出し時点で、ソースとなる hstore が NULL でないことが前提（Assert により検証）。
  • 添字（キー）は単一の text 型である必要がある。

アルゴリズムまたは振る舞い

1. 添字（キー）が NULL かどうかを確認し、NULL であれば結果を NULL として終了する。 2. DatumGetHStoreP を使用して hstore を取得し、DatumGetTextPP でキーを取り出す。 3. hstoreFindKey を呼び出し、hstore 内からキーを検索する。 4. キーが見つからない、または値が NULL の場合は op->resnull を true に設定して終了する。 5. キーが見つかった場合、cstring_to_text_with_len を用いて値を text 型に変換し、op->resvalue に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（SubscriptingRef ノードの評価時）。
  • **呼び出し先**: hstoreFindKey（hstore 内のキー検索）、cstring_to_text_with_len（結果の構築）。

関連データ構造

  • HStore: 操作対象の hstore 構造体。
  • SubscriptingRefState: 添字演算の状態を保持する構造体。
  • HEntry: hstore 内の各エントリを指す構造体。

失敗モードと境界条件

  • **添字が NULL**: 結果として NULL を返す。
  • **キーが存在しない**: 結果として NULL を返す。
  • **値が NULL**: 結果として NULL を返す。

保守担当者向けメモ

  • 本関数は hstore_fetchval() と同様のロジックで実装されている。
  • hstore は不変（immutable）なデータ構造であるため、本関数は読み取り専用の操作を行う。
  • 添字演算の結果は常に単一の text 型として扱われるため、配列のようなスライスや多次元の考慮は不要である。
