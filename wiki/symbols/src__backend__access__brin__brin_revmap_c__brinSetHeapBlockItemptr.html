brinSetHeapBlockItemptr

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_revmap.c:154

目的

BRINインデックスのリバースレンジマップ（revmap）ページ内の特定の項目を更新し、ヒープブロック範囲と対応する要約タプル（BrinTuple）のTIDを紐付ける。

シグネチャまたは形状

void brinSetHeapBlockItemptr(Buffer buf, BlockNumber pagesPerRange, BlockNumber heapBlk, ItemPointerData tid)

入力と出力

  • buf: 更新対象のrevmapページを保持するバッファ。
  • pagesPerRange: 1つのBRIN要約タプルがカバーするヒープブロック数。
  • heapBlk: 対象となるヒープブロック番号。
  • tid: ヒープブロック範囲に対応する要約タプルのTID。無効なTIDが渡された場合は、対応関係を無効化する。
  • 出力: なし（副作用としてバッファの内容を更新）。

前提条件と不変条件

  • 引数 buf で指定されたバッファは、呼び出し元によって既にピン留めされ、排他ロックされている必要がある。

アルゴリズムまたは振る舞い

1. BufferGetPage を使用してバッファからページを取得する。 2. PageGetContents を介してrevmapのデータ領域にアクセスする。 3. HEAPBLK_TO_REVMAP_INDEX マクロを用いて、指定された heapBlk に対応するインデックス位置を算出する。 4. tid が有効な場合、算出した位置に ItemPointerSet を用いてTIDを書き込む。 5. tid が無効な場合、ItemPointerSetInvalid を用いて該当位置を無効化する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/brin/brin_pageops.c
  • src/backend/access/brin/brin_revmap.c
  • src/backend/access/brin/brin_xlog.c
  • 呼び出し先:
  • BufferGetPage
  • PageGetContents
  • HEAPBLK_TO_REVMAP_INDEX
  • ItemPointerIsValid, ItemPointerSet, ItemPointerSetInvalid

関連データ構造

  • RevmapContents: revmapページ内のデータ構造。
  • ItemPointerData: ヒープブロックと要約タプルを対応付けるためのTID情報。

失敗モードと境界条件

  • 呼び出し元がバッファのロックを保持していない場合、競合やデータ破損の可能性がある。
  • heapBlk が現在のrevmapの容量を超えている場合、不正なメモリ領域へのアクセスが発生する可能性がある（呼び出し元で適切なページ拡張が行われていることが前提）。

保守担当者向けメモ

  • 本関数はWAL記録そのものは行わない。WAL記録が必要な場合は、呼び出し元で MarkBufferDirty や XLogInsert を適切に処理すること。
