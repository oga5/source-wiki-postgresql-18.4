extract_actual_join_clauses

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/restrictinfo.c:512

目的

RestrictInfo のリストを、現在の結合レベルで評価すべき条件（joinquals）と、下位へプッシュダウン可能な条件（otherquals）に分離します。

シグネチャまたは形状

void extract_actual_join_clauses(List *restrictinfo_list,
                                 Relids joinrelids,
                                 List **joinquals,
                                 List **otherquals)

入力と出力

  • **入力**:
  • restrictinfo_list: 処理対象の RestrictInfo ノードのリスト。
  • joinrelids: 現在の結合対象リレーションの集合。
  • **出力**:
  • *joinquals: 現在の結合レベルで評価されるべき Expr のリスト。
  • *otherquals: 下位へプッシュダウン可能な Expr のリスト。

前提条件と不変条件

  • joinquals および otherquals は、関数呼び出し前に空のリスト（NIL）として初期化されます。
  • joinquals に分類される条件は、pseudoconstant であってはなりません。

アルゴリズムまたは振る舞い

1. restrictinfo_list を走査し、各 RestrictInfo を取得します。 2. RINFO_IS_PUSHED_DOWN マクロを用いて、その条件が joinrelids に対してプッシュダウン可能か判定します。 3. プッシュダウン可能な場合:

  • pseudoconstant ではなく、かつ rinfo_is_constant_true が真でない場合に限り、その clause を otherquals に追加します。

    • 4. プッシュダウン不可能な場合:

  • pseudoconstant でないことを Assert で確認し、rinfo_is_constant_true が真でない場合に限り、その clause を joinquals に追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/createplan.c (複数箇所)
  • **呼び出し先**:
  • RINFO_IS_PUSHED_DOWN (マクロ)
  • rinfo_is_constant_true
  • lappend

関連データ構造

  • RestrictInfo: 条件式とプランナ用メタデータを保持する構造体。
  • Relids: リレーション識別子の集合。
  • List: PostgreSQL のリスト構造。

失敗モードと境界条件

  • rinfo->pseudoconstant が真である条件が joinquals に分類されようとした場合、Assert により停止します。
  • rinfo_is_constant_true が真を返す条件は、いずれのリストにも追加されません。

保守担当者向けメモ

  • 本関数は、プラン作成の最終段階で RestrictInfo から生の Expr を抽出する際に使用されます。
  • RINFO_IS_PUSHED_DOWN の判定ロジックが変更された場合、本関数の分類結果も直接影響を受けます。
