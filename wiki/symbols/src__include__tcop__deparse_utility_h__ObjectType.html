ObjectType

定義

  • 種別: struct
  • 定義箇所: src/include/tcop/deparse_utility.h:99

目的

データベースオブジェクトの種類を識別するための型定義です。主にDDLコマンドの追跡や、権限チェック処理において対象オブジェクトを特定するために使用されます。

シグネチャまたは形状

struct {
    ObjectType objtype;
} defprivs;

※上記は deparse_utility.h 内での使用例であり、ObjectType 自体は列挙型として定義されています。

入力と出力

  • 入力: 権限チェック関数やオブジェクト検索関数において、操作対象の種別を指定する引数として渡されます。
  • 出力: 該当なし（型定義のため）。

前提条件と不変条件

  • CollectedCommand 構造体の一部として使用される場合、type フィールドによる識別と整合性が取れている必要があります。

アルゴリズムまたは振る舞い

  • 本シンボルはデータ型であり、直接的なアルゴリズムは持ちません。
  • 権限チェック（pg_aclmask）やオブジェクト名からOIDへの変換（objectNamesToOids）において、処理対象を分類するための識別子として機能します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/catalog/aclchk.c 内の多数の関数（objectNamesToOids, objectsInSchemaToOids, pg_aclmask, aclcheck_error, aclcheck_error_col など）。
  • 呼び出し先:
  • 該当なし（型定義のため）。

関連データ構造

  • CollectedCommand: 本型をメンバとして保持する構造体。
  • ObjectAddress: オブジェクトを一意に特定するための構造体。
  • AclMode: 権限チェックの結果として使用される型。

失敗モードと境界条件

  • 未定義の ObjectType が渡された場合、権限チェック関数や検索関数内で予期せぬエラーや不正な動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本シンボルは src/include/tcop/deparse_utility.h で定義されていますが、実質的には src/backend/catalog/aclchk.c における権限管理ロジックと密接に結合しています。変更時はカタログ定義との整合性に注意してください。
