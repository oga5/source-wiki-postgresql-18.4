btree_xlog_reuse_page

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtxlog.c:1006

目的

B-tree インデックスのページ再利用に関する WAL レコードをリプレイし、ホットスタンバイ環境におけるリカバリ競合を解決する。

シグネチャまたは形状

static void btree_xlog_reuse_page(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象の WAL レコード）
  • **出力**: なし（副作用としてリカバリ競合の解決を実行）

前提条件と不変条件

  • record から取得されるデータは xl_btree_reuse_page 構造体として解釈可能であること。

アルゴリズムまたは振る舞い

1. XLogRecGetData を使用して、WAL レコードから xl_btree_reuse_page 構造体を取得する。 2. グローバル変数 InHotStandby が真である場合、ResolveRecoveryConflictWithSnapshotFullXid を呼び出し、以下の情報を渡してリカバリ競合を解決する。

  • xlrec->snapshotConflictHorizon
  • xlrec->isCatalogRel
  • xlrec->locator

呼び出し元と呼び出し先

  • **呼び出し元**: btree_redo（src/backend/access/nbtree/nbtxlog.c 内のディスパッチ処理）
  • **呼び出し先**: ResolveRecoveryConflictWithSnapshotFullXid（外部関数）

関連データ構造

  • xl_btree_reuse_page: WAL レコード内に格納されるページ再利用情報。
  • XLogReaderState: WAL レコードの読み取り状態を保持する構造体。

失敗モードと境界条件

  • InHotStandby が偽の場合、本関数は何も行わず正常終了する。
  • 呼び出し先である ResolveRecoveryConflictWithSnapshotFullXid が内部でエラーを発生させた場合、リカバリ処理全体が中断される可能性がある。

保守担当者向けメモ

  • 本関数は物理的なページ内容の変更を行わず、主にスタンバイサーバーでの整合性維持（リカバリ競合の解消）に特化している。
  • xl_btree_reuse_page 構造体の定義については src/include/access/nbtxlog.h を参照のこと。
