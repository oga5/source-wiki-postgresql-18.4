hash_xlog_add_ovfl_page

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash_xlog.c:172

目的

ハッシュインデックスにおけるオーバーフローページの追加操作をWALリプレイ時に再現し、関連するバケットチェーン、ビットマップページ、およびメタページの状態を整合させる。

シグネチャまたは形状

static void hash_xlog_add_ovfl_page(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）
  • **出力**: なし（副作用としてバッファプール上のページを更新）

前提条件と不変条件

  • WALリプレイ中であるため、他のプロセスによる同時アクセスは発生しない前提で動作する。
  • XLogRecGetBlockTag により、対象となるブロック（オーバーフローページ、左隣接ページ、ビットマップページ、メタページ）が正しく特定できること。

アルゴリズムまたは振る舞い

1. **オーバーフローページの初期化**: XLogInitBufferForRedo で新しいオーバーフローページを初期化し、hasho_prevblkno を設定して左隣接ページとリンクさせる。 2. **リンク更新**: 左隣接ページが存在する場合、XLogReadBufferForRedo で読み込み、hasho_nextblkno を新しいオーバーフローページに更新する。 3. **ビットマップ更新**: WALレコードにビットマップの参照が含まれる場合、該当ビットをセットする。 4. **新規ビットマップページ作成**: 必要に応じて新しいビットマップページを初期化し、メタページへの登録準備を行う。 5. **メタページ更新**: hashm_firstfree を更新し、必要に応じて hashm_spares や hashm_mapp（新しいビットマップページへの参照）を更新する。 6. **LSN更新と永続化**: 変更した全ページに現在のWAL LSNを書き込み、MarkBufferDirty を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/hash/hash_xlog.c 内の hash_redo 関数（WALレコード種別 XLOG_HASH_ADD_OVFL_PAGE に基づくディスパッチ）。
  • **呼び出し先**: XLogRecGetBlockTag, XLogInitBufferForRedo, XLogReadBufferForRedo, _hash_initbuf, _hash_initbitmapbuffer, MarkBufferDirty, UnlockReleaseBuffer 等のバッファ管理およびハッシュインデックス操作関数。

関連データ構造

  • xl_hash_add_ovfl_page: WALレコードのデータ構造。
  • HashPageOpaque: ページヘッダーのハッシュインデックス固有情報。
  • HashMetaPage: インデックスのメタデータ構造。

失敗モードと境界条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返さない場合、そのページの更新はスキップされる。
  • ビットマップページやメタページの参照がないWALレコードの場合、該当する更新処理は実行されない。

保守担当者向けメモ

  • 本関数は冪等である必要がある。
  • 通常のインデックス操作とは異なり、リプレイ中はロックの保持期間が最適化されている（詳細はソース内のコメント参照）。
