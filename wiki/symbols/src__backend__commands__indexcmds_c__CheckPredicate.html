CheckPredicate

定義

  • 種別: function
  • 定義箇所: src/backend/commands/indexcmds.c:1844

目的

インデックスの述語（WHERE 句）が、インデックス定義として許容される式であるかを検証します。

シグネチャまたは形状

static void CheckPredicate(Expr *predicate)

入力と出力

  • **入力**: Expr *predicate - 検証対象となるインデックスの述語式。
  • **出力**: なし。検証に失敗した場合はエラーを報告し、処理を中断します。

前提条件と不変条件

  • 本関数が呼び出される前に、transformExpr() によってサブクエリ、集約関数、ウィンドウ関数が含まれていないことが確認されている必要があります。
  • インデックス述語には IMMUTABLE な関数のみが許可されます。

アルゴリズムまたは振る舞い

1. contain_mutable_functions_after_planning(predicate) を呼び出し、述語内に可変（mutable）な関数が含まれているかを確認します。 2. 可変関数が含まれている場合、ERRCODE_INVALID_OBJECT_DEFINITION エラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/indexcmds.c 内の DefineIndex 関数（stmt->whereClause を引数として渡す箇所）から呼び出されます。
  • **呼び出し先**: contain_mutable_functions_after_planning を呼び出します。

関連データ構造

  • Expr: PostgreSQLの式ツリーを表す構造体。

失敗モードと境界条件

  • 述語内に IMMUTABLE ではない関数が含まれている場合、ERROR レベルで処理が停止します。

保守担当者向けメモ

  • 本関数は CREATE INDEX における述語の妥当性チェックを担当しています。
  • インデックス式と同様の制約（副作用の禁止）を強制するためのガードレールとして機能しています。
