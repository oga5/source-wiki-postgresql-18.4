hstore_to_plpython

定義

  • 種別: function
  • 定義箇所: contrib/hstore_plpython/hstore_plpython.c:82

目的

PostgreSQL の hstore 型データを、対応する Python の辞書型（dict）オブジェクトへ変換する。

シグネチャまたは形状

Datum hstore_to_plpython(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される hstore 型のデータ。
  • **出力**: Datum 型として返される Python 辞書オブジェクト。

前提条件と不変条件

  • _PG_init によって、必要な plpython および hstore の外部関数ポインタが解決済みであること。
  • 入力データは有効な hstore 構造体であること。

アルゴリズムまたは振る舞い

1. PyDict_New() を呼び出し、空の Python 辞書を生成する。メモリ確保に失敗した場合は ERROR を報告する。 2. hstore 構造体のエントリ数（HS_COUNT）分だけループ処理を行う。 3. 各エントリについて、PLyUnicode_FromStringAndSize を使用してキーを Python 文字列に変換する。 4. 値が NULL の場合は Py_None を、それ以外の場合は PLyUnicode_FromStringAndSize で変換した文字列を、キーと共に PyDict_SetItem で辞書に格納する。 5. 変換に使用した一時的な Python オブジェクト（キーおよび値）に対して Py_XDECREF を呼び出し、参照カウントを適切に管理する。 6. 構築した辞書を PointerGetDatum で変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の SQL インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • PLyUnicode_FromStringAndSize（plpython モジュール由来）
  • PyDict_New, PyDict_SetItem, Py_XDECREF（Python C API）
  • HS_COUNT, STRPTR, ARRPTR, HSTORE_KEY, HSTORE_KEYLEN, HSTORE_VAL, HSTORE_VALLEN, HSTORE_VALISNULL（hstore モジュール由来のマクロ）

関連データ構造

  • HStore: hstore 型の内部表現。
  • PyObject: Python オブジェクトの汎用型。
  • Datum: PostgreSQL の汎用データ型。

失敗モードと境界条件

  • **メモリ不足**: PyDict_New が失敗した場合、ereport により ERRCODE_OUT_OF_MEMORY エラーが発生する。
  • **NULL 値**: hstore 内の NULL 値は、Python 側の None オブジェクトとして正しく処理される。

保守担当者向けメモ

  • 本関数は Python C API を直接操作するため、参照カウント（Py_XDECREF）の管理が重要である。
  • plpython および hstore の内部関数は動的にロードされるため、これらのモジュールの API 変更が本関数の動作に影響を与える可能性がある。
