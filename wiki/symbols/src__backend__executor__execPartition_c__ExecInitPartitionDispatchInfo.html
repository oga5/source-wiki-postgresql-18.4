ExecInitPartitionDispatchInfo

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execPartition.c:1101

目的

パーティション化されたテーブルの階層構造において、特定のパーティションレベルでのルーティング情報を保持する PartitionDispatch 構造体を初期化および登録します。

シグネチャまたは形状

static PartitionDispatch ExecInitPartitionDispatchInfo(
    EState *estate,
    PartitionTupleRouting *proute,
    Oid partoid,
    PartitionDispatch parent_pd,
    int partidx,
    ResultRelInfo *rootResultRelInfo
)

入力と出力

  • **入力**:
  • estate: 実行状態。パーティションディレクトリの管理に使用。
  • proute: ルーティング環境を保持するメイン構造体。
  • partoid: 初期化対象のパーティションの OID。
  • parent_pd: 親パーティションのディスパッチ情報（ルートの場合は NULL）。
  • partidx: 親における当該パーティションのインデックス。
  • rootResultRelInfo: ルートテーブルの ResultRelInfo。
  • **出力**:
  • 初期化された PartitionDispatch 構造体へのポインタ。

前提条件と不変条件

  • proute->memcxt が現在のメモリコンテキストとして使用されます。
  • estate->es_partition_directory が初期化されていない場合、本関数内で CreatePartitionDirectory が呼び出されます。
  • parent_pd が指定されている場合、親の indexes[partidx] は -1 である必要があります。

アルゴリズムまたは振る舞い

1. **ディレクトリ準備**: パーティションディレクトリが存在しない場合、作成します。 2. **リレーション取得**: ルートテーブル以外は RowExclusiveLock で開きます。 3. **構造体割り当て**: PartitionDispatchData をメモリ確保し、パーティション数に応じたインデックス配列を初期化（-1 で埋める）します。 4. **サブパーティション対応**: parent_pd が存在する場合、親と子の型変換が必要であれば tupmap と tupslot を構築します。 5. **管理配列の更新**: proute 内の partition_dispatch_info 配列を必要に応じて拡張し、新しい pd を登録します。 6. **ResultRelInfo 構築**: サブパーティションテーブルの場合、制約チェック用に ResultRelInfo を作成し、proute->nonleaf_partitions に格納します。 7. **リンク設定**: parent_pd がある場合、親の indexes[partidx] を新しい dispatchidx に更新し、階層を辿れるようにします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execPartition.c 内のルーティング初期化処理（ExecSetupPartitionTupleRouting 等）から呼び出されます。
  • **呼び出し先**: table_open, PartitionDirectoryLookup, RelationGetPartitionKey, build_attrmap_by_name_if_req, MakeSingleTupleTableSlot, InitResultRelInfo 等。

関連データ構造

  • PartitionDispatchData: ルーティング階層の各ノード情報。
  • PartitionTupleRouting: ルーティング全体の管理構造体。
  • ResultRelInfo: パーティションごとの実行時情報。

失敗モードと境界条件

  • proute->max_dispatch の上限に達した場合、repalloc により配列が拡張されます。
  • partoid がルートテーブルと一致する場合、table_open はスキップされます。

保守担当者向けメモ

  • 本関数は、パーティション階層を再帰的に構築する際の「ノード作成」を担当します。
  • tupslot は TTSOpsVirtual を使用して作成されるため、仮想タプルスロットとして動作します。
  • indexes 配列の -1 は、そのパーティションがまだ初期化されていないことを示す重要な不変条件です。
