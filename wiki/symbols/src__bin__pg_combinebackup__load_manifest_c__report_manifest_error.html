report_manifest_error

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/load_manifest.c:227

目的

バックアップマニフェストのパース処理中に発生した致命的なエラーを報告し、プログラムを異常終了させるためのエラーハンドラです。

シグネチャまたは形状

static void report_manifest_error(JsonManifestParseContext *context, const char *fmt, ...);

入力と出力

  • **入力**:
  • context: パース処理の状態を保持する JsonManifestParseContext 構造体へのポインタ。
  • fmt: エラーメッセージのフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**: なし（関数は復帰せず exit(1) を呼び出します）。

前提条件と不変条件

  • 本関数は JsonManifestParseContext のエラーコールバックとして設定されることを想定しています。
  • 呼び出し後、プロセスは直ちに終了するため、呼び出し元での後続処理は実行されません。

アルゴリズムまたは振る舞い

1. va_start を使用して可変長引数リストを初期化します。 2. pg_log_generic_v を呼び出し、PG_LOG_ERROR レベルでエラーメッセージをログ出力します。 3. va_end で引数リストをクリーンアップします。 4. exit(1) を実行し、プロセスを終了させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_combinebackup/load_manifest.c 内で、パースコンテキストの error_cb フィールドに本関数が代入されています。
  • src/bin/pg_verifybackup/pg_verifybackup.c においても同様の用途で参照されています。
  • **呼び出し先**:
  • pg_log_generic_v (PostgreSQL ログ出力関数)
  • exit (標準ライブラリ)

関連データ構造

  • JsonManifestParseContext: パース処理の状態管理に使用される構造体。

失敗モードと境界条件

  • 本関数自体が失敗することはありませんが、ログ出力先が書き込み不可である場合や、メモリ不足等の極端な状況下では pg_log_generic_v の挙動に依存します。

保守担当者向けメモ

  • 本関数は pg_verifybackup と共通のシグネチャを持っています。修正を行う際は、両方のバイナリへの影響を考慮してください。
  • pg_noreturn 属性が宣言されているため、コンパイラは本関数以降のコードが到達不能であることを認識します。
