BufferCachePagesContext

定義

  • 種別: struct
  • 定義箇所: contrib/pg_buffercache/pg_buffercache_pages.c:63

目的

pg_buffercache_pages 関数が SRF（Set-Returning Function）として呼び出される際、複数回の呼び出しにわたって状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct
{
	TupleDesc	tupdesc;
	BufferCachePagesRec *record;
} BufferCachePagesContext;

入力と出力

  • tupdesc: SRF が返すタプルの記述子を保持します。
  • record: バッファキャッシュの情報を格納する BufferCachePagesRec 型の配列（またはポインタ）を指します。

前提条件と不変条件

  • pg_buffercache_pages 関数の実行中に palloc を介してメモリが割り当てられ、SRF の user_fctx として管理されることを前提とします。

アルゴリズムまたは振る舞い

  • SRF の初回呼び出し時に palloc でメモリ確保され、以降の呼び出しでバッファキャッシュのメタデータを順次読み出すための状態保持領域として機能します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pg_buffercache/pg_buffercache_pages.c 内の pg_buffercache_pages 関数において、fctx としてインスタンス化されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • BufferCachePagesRec: バッファキャッシュの個別のメタデータを保持する構造体。
  • TupleDesc: PostgreSQL のタプル定義情報。

失敗モードと境界条件

  • メモリ不足: palloc 失敗時にエラーが発生します。
  • ライフサイクル: SRF の呼び出しが終了する前にコンテキストが破棄された場合、後続のアクセスで不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は pg_buffercache_pages の実行コンテキストに特化しています。他の関数（pg_buffercache_numa_pages など）では別のコンテキスト構造体が使用されるため、混同しないよう注意してください。
