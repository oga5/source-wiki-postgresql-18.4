init_missing_cache

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/heaptuple.c:125

目的

欠損値（missing values）を管理するためのハッシュテーブル missing_cache を初期化する。

シグネチャまたは形状

static void init_missing_cache(void)

入力と出力

  • 入力: なし
  • 出力: なし（グローバル変数 missing_cache を初期化する副作用を持つ）

前提条件と不変条件

  • TopMemoryContext が初期化済みであること。
  • missing_hash および missing_match 関数が定義済みであること。

アルゴリズムまたは振る舞い

1. HASHCTL 構造体を設定し、キーサイズとエントリサイズを missing_cache_key に合わせる。 2. メモリコンテキストとして TopMemoryContext を指定する。 3. hash_create を呼び出し、"Missing Values Cache" という名前でサイズ 32 のハッシュテーブルを作成する。 4. 作成したハッシュテーブルのポインタをグローバル変数 missing_cache に格納する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/common/heaptuple.c 内の関数（ソースコード上の参照として行126および185が確認されている）。
  • 呼び出し先: hash_create（PostgreSQLのハッシュテーブル管理関数）。

関連データ構造

  • missing_cache_key: ハッシュテーブルのキーおよびエントリとして使用される構造体。
  • missing_cache: 初期化対象となるハッシュテーブルのポインタ。

失敗モードと境界条件

  • hash_create はメモリ不足時にエラーを発生させる可能性がある。
  • 本関数は冪等ではないため、複数回呼び出された場合の挙動については、呼び出し元でのガードが必要となる。

保守担当者向けメモ

  • 本関数は静的関数であり、モジュール内でのみ使用される。
  • キャッシュサイズは固定値（32）で初期化されているため、将来的に負荷が増大した場合は調整を検討すること。
