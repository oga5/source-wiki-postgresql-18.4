leftmostvalue_int2

定義

  • 種別: function
  • 定義箇所: contrib/btree_gin/btree_gin.c:209

目的

int2 型（2バイト整数）における最小値を返し、GINインデックスを用いた範囲検索（特に BTLess 系戦略）において、スキャン開始位置を決定するための基準値を提供します。

シグネチャまたは形状

static Datum leftmostvalue_int2(void)

入力と出力

  • **入力**: なし
  • **出力**: Datum 型として表現された int16 の最小値（SHRT_MIN）。

前提条件と不変条件

  • 本関数は int2 型の最小値を返すことが保証されています。
  • GINインデックスのサポート関数として、GIN_SUPPORT マクロを通じて呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. limits.h で定義された SHRT_MIN を取得します。 2. Int16GetDatum マクロを使用して、取得した値を PostgreSQL の Datum 形式に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gin/btree_gin.c 内の GIN_SUPPORT(int2, ...) マクロ定義において参照されています。
  • **呼び出し先**:
  • Int16GetDatum（PostgreSQLの型変換マクロ）を呼び出します。

関連データ構造

  • Datum: PostgreSQLの汎用データ表現型。
  • GIN_SUPPORT: 本関数を登録し、GINインデックスの比較ロジックに組み込むためのマクロ。

失敗モードと境界条件

  • 本関数自体は定数を返すため、実行時の失敗モードはありません。
  • 境界条件として、int2 型の表現可能な最小値（SHRT_MIN）を正確に返すことで、インデックススキャンの開始点を正しく定義します。

保守担当者向けメモ

  • 新しい整数型を追加する際は、本関数を参考に leftmostvalue_[type] を定義し、GIN_SUPPORT マクロに登録してください。
  • SHRT_MIN は limits.h に依存しているため、環境による型のサイズ定義に変更がないか注意してください。
