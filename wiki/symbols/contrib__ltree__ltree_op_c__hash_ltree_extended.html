hash_ltree_extended

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_op.c:167

目的

ltree 型のデータに対して、シード値（seed）を考慮したハッシュ値を計算します。ハッシュインデックス等の PostgreSQL 内部のハッシュ計算処理で使用されます。

シグネチャまたは形状

Datum hash_ltree_extended(PG_FUNCTION_ARGS)
  • 引数 0: ltree * (対象のパス)
  • 引数 1: uint64 (ハッシュ計算に使用するシード値)
  • 戻り値: uint64 (計算されたハッシュ値)

入力と出力

  • **入力**: ltree 構造体へのポインタと、uint64 型のシード値を受け取ります。
  • **出力**: 各レベルのラベルを hash_any_extended でハッシュ化し、それらを累積した uint64 型のハッシュ値を返します。

前提条件と不変条件

  • 入力された ltree が空（レベル数 0）の場合、1 + seed を返します。これにより、シード値が 0 の場合に hash_ltree と互換性のある結果を維持しつつ、シード値がある場合には異なる値を生成します。
  • PG_FREE_IF_COPY により、入力された ltree がコピーである場合は適切にメモリ解放を行います。

アルゴリズムまたは振る舞い

1. ltree のレベル数 an を取得し、最初のレベル al を特定します。 2. レベル数が 0 の場合、1 + seed を返して終了します。 3. レベル数が 0 より大きい間、以下の処理を繰り返します：

  • hash_any_extended を用いて現在のラベル名 al->name のハッシュ値を計算します。
  • result = (result << 5) - result + levelHash の式で累積ハッシュ値を更新します。
  • 次のレベルへポインタを進めます。

    • 4. 最終的な result を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数登録マクロ PG_FUNCTION_INFO_V1(hash_ltree_extended) により、SQL インターフェースから呼び出し可能として登録されています。具体的な SQL クエリやインデックス操作からの直接的な呼び出し元は、提供されたソース内には明示されていません。
  • **呼び出し先**: hash_any_extended (PostgreSQL 内部のハッシュ関数) を呼び出します。

関連データ構造

  • ltree: 階層構造を持つラベルパスを保持する構造体。
  • ltree_level: ltree 内の個々のラベル情報を保持する構造体。

失敗モードと境界条件

  • ltree が空の場合、計算ロジックをスキップして即座に値を返します。
  • メモリ管理については PG_FREE_IF_COPY を使用しており、PostgreSQL のメモリ管理規約に従います。

保守担当者向けメモ

  • 本関数は hash_ltree と異なり、シード値を受け取る拡張版です。ハッシュインデックスのサポート関数として機能するため、ロジック変更時は hash_ltree との整合性に注意してください。
  • LEVEL_NEXT マクロを使用してレベルを走査しているため、ltree.h で定義されている構造に変更があった場合は影響を受けます。
