int_query_opr_selec

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_selfuncs.c:255

目的

query_int 型の論理クエリ構造を再帰的に走査し、統計情報（MCE: Most Common Elements）に基づいてクエリ全体の選択率（Selectivity）を推定する。

シグネチャまたは形状

Selectivity int_query_opr_selec(ITEM *item, Datum *mcelems, float4 *mcefreqs, int nmcelems, float4 minfreq)

入力と出力

  • **入力**:
  • item: 評価対象の ITEM 構造体（クエリツリーのノード）。
  • mcelems, mcefreqs, nmcelems: pg_statistic から取得した頻出要素の配列、その頻度、および要素数。
  • minfreq: 頻出要素に含まれない場合の最小頻度推定値。
  • **出力**:
  • Selectivity (float): 計算された選択率（0.0 から 1.0 の範囲）。

前提条件と不変条件

  • 再帰呼び出しを行うため、スタックオーバーフローを防ぐ目的で check_stack_depth() が呼び出される。
  • 最終的な選択率は CLAMP_PROBABILITY マクロにより 0.0 から 1.0 の範囲に収められる。
  • item->type が OPR の場合、演算子に応じて item - 1 や item + item->left を参照して再帰的に計算を行う。

アルゴリズムまたは振る舞い

1. **VAL ノードの場合**:

  • mcelems 内を bsearch で検索。見つかれば対応する頻度を返し、見つからなければ minfreq / 2 を返す。

    • 2. **OPR ノードの場合**:

  • ! (NOT): 1.0 - s1 を計算。
  • & (AND): s1 * s2 を計算。
  • | (OR): s1 + s2 - s1 * s2 を計算。

    • 3. **エラー処理**: 未知の型や演算子に対しては elog(ERROR) を発行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_selfuncs.c 内の _int_matchsel 関数から呼び出される（直接の証拠あり）。
  • **呼び出し先**:
  • 自身を再帰的に呼び出す。
  • bsearch (標準ライブラリ)。
  • compare_val_int4 (比較関数)。
  • check_stack_depth (PostgreSQL 内部関数)。

関連データ構造

  • ITEM: query_int の構文木を構成するノード構造体。
  • Selectivity: 選択率を表す浮動小数点型。

失敗モードと境界条件

  • **スタックオーバーフロー**: 非常に深いクエリツリーに対しては check_stack_depth() によりエラーとなる。
  • **未知の演算子/型**: item->val が !, &, | 以外の場合、または item->type が不正な場合はエラーとなる。
  • **統計情報欠如**: mcelems が NULL の場合、デフォルト値 DEFAULT_EQ_SEL を返す。

保守担当者向けメモ

  • 本関数は再帰的であるため、クエリの複雑さに応じてスタック消費量が増大する点に注意すること。
  • 統計情報が不完全な場合のヒューリスティック（minfreq / 2）は、統計情報の精度に依存する。
