SubTransSetParent

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/subtrans.c:84

目的

指定されたサブトランザクション ID (xid) とその親トランザクション ID (parent) を関連付け、サブトランザクションログ（SUBTRANS）に記録します。

シグネチャまたは形状

void SubTransSetParent(TransactionId xid, TransactionId parent)

入力と出力

  • **入力**:
  • xid: 親子関係を設定する対象のサブトランザクション ID。
  • parent: xid の親として設定するトランザクション ID。
  • **出力**: なし（副作用として共有メモリ上の SUBTRANS ページを更新）。

前提条件と不変条件

  • parent は有効なトランザクション ID であること (TransactionIdIsValid)。
  • xid は parent よりも後に生成された ID であること (TransactionIdFollows)。
  • 既に設定済みの親 ID を別の有効な ID に書き換えることは禁止されており、アサーションにより保護されています。

アルゴリズムまたは振る舞い

1. TransactionIdToPage および TransactionIdToEntry を用いて、対象 XID が格納されるページ番号とエントリ位置を特定します。 2. SimpleLruGetBankLock で該当ページに対する排他ロック (LW_EXCLUSIVE) を取得します。 3. SimpleLruReadPage を呼び出し、対象ページをメモリ上のスロットに読み込みます。 4. 該当エントリ (ptr) が InvalidTransactionId であることを確認し、指定された parent を書き込みます。 5. ページが更新されたことを示すため、SubTransCtl->shared->page_dirty[slotno] を true に設定します。 6. ロックを解放して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/twophase.c: 準備済みトランザクションの復元時。
  • src/backend/access/transam/xact.c: サブトランザクション開始時。
  • src/backend/storage/ipc/procarray.c: プロセス配列の走査時。
  • **呼び出し先**:
  • TransactionIdToPage, TransactionIdToEntry (マクロ)
  • SimpleLruGetBankLock, SimpleLruReadPage (SLRU 管理関数)
  • LWLockAcquire, LWLockRelease (ロック管理関数)

関連データ構造

  • SubTransCtl: SUBTRANS の SLRU 管理を行う共有メモリ制御構造体。
  • TransactionId: トランザクションを一意に識別する 32bit ID。

失敗モードと境界条件

  • **アサーション失敗**: xid が parent より前である場合や、既に異なる親が設定されている場合にプロセスが停止します。
  • **メモリ不足**: SLRU ページ読み込み時にバッファスロットが確保できない場合、SimpleLruReadPage 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は一時的な親子関係を保持するものであり、システム再起動時にはデータがクリアされます。永続的なトランザクション状態の記録には使用しないでください。
  • SubTransCtl を介した SLRU 操作は、ページ単位のロック制御に依存しているため、競合が発生しやすい箇所です。
