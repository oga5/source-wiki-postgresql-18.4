ginRedoDeletePage

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginxlog.c:476

目的

GINインデックスにおいて、ページ削除操作に関連するWALレコードをリプレイし、インデックスの物理構造（左隣接ページ、削除対象ページ、親ページ）を整合性のある状態へ復元します。

シグネチャまたは形状

static void ginRedoDeletePage(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）。
  • **出力**: なし（副作用としてバッファの内容を更新）。

前提条件と不変条件

  • XLogReadBufferForRedo を介して取得した各ページは、GinPageIsData マクロによりGINデータページであることが保証される必要があります。
  • 親ページ（pbuffer）に対しては、!GinPageIsLeaf であることがアサーションにより検証されます。
  • デッドロックを回避するため、左隣接ページ（lbuffer）を最初にロックする順序が守られます。

アルゴリズムまたは振る舞い

1. **左隣接ページの更新**: XLogReadBufferForRedo で取得した左隣接ページの rightlink を、WALレコード内の data->rightLink で更新します。 2. **削除対象ページの更新**: 削除対象ページに GinPageSetDeleted を適用し、削除時のトランザクションID（deleteXid）を記録します。 3. **親ページの更新**: 親ページから data->parentOffset で指定されたアイテムを GinPageDeletePostingItem により削除します。 4. **共通処理**: 各ステップで変更が行われた場合、PageSetLSN でLSNを更新し、MarkBufferDirty を呼び出します。 5. **終了処理**: 取得したすべてのバッファに対して UnlockReleaseBuffer を実行し、ロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginxlog.c 内の gin_redo 関数から呼び出されます。
  • **呼び出し先**: XLogRecGetData、XLogReadBufferForRedo、BufferGetPage、GinPageIsData、GinPageGetOpaque、PageSetLSN、MarkBufferDirty、GinPageSetDeleted、GinPageSetDeleteXid、GinPageIsLeaf、GinPageDeletePostingItem、BufferIsValid、UnlockReleaseBuffer。

関連データ構造

  • ginxlogDeletePage: WALレコードから読み取られるデータ構造。rightLink、deleteXid、parentOffset を保持します。
  • XLogReaderState: WALリプレイの状態管理構造体。

失敗モードと境界条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返さない場合、そのバッファに対する更新処理はスキップされます。
  • 各バッファは BufferIsValid を確認した後にのみ UnlockReleaseBuffer が実行されます。

保守担当者向けメモ

  • 本関数は ginStepRight() とのデッドロックを防ぐため、左隣接ページを優先的にロックする設計となっています。変更を加える際はロック順序に注意してください。
  • GinPageIsData や GinPageIsLeaf などのマクロは access/gin_private.h で定義されています。
