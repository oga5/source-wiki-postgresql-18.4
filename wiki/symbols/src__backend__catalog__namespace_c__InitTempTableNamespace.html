InitTempTableNamespace

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/namespace.c:4392

目的

現在のセッションにおける一時テーブル用名前空間（pg_temp_N）および対応する TOAST 用名前空間（pg_toast_temp_N）を初期化または作成する。

シグネチャまたは形状

static void InitTempTableNamespace(void)

入力と出力

  • 入力: なし（グローバル変数 MyProcNumber 等を参照）
  • 出力: なし（副作用としてグローバル変数 myTempNamespace, myTempToastNamespace, myTempNamespaceSubID を更新）

前提条件と不変条件

  • myTempNamespace が未初期化（!OidIsValid）であること。
  • 呼び出し元は、一時テーブル作成の要求が発生した時点であること。
  • データベースに対する ACL_CREATE_TEMP 権限が必要。
  • リカバリ中（Hot Standby）またはパラレルワーカープロセスでは実行不可。

アルゴリズムまたは振る舞い

1. **権限チェック**: データベースに対する一時テーブル作成権限を確認し、不可ならエラーを投げる。 2. **環境チェック**: リカバリ中またはパラレルワーカーの場合はエラーを投げる。 3. **名前空間の特定/作成**:

  • pg_temp_<MyProcNumber> を検索し、存在しなければ NamespaceCreate で作成する。存在する場合は RemoveTempRelations で既存のリレーションを削除する。
  • 同様に pg_toast_temp_<MyProcNumber> を検索・作成する。

    • 4. **状態更新**:

  • myTempNamespace および myTempToastNamespace に OID を格納。
  • MyProc->tempNamespaceId を更新し、他プロセスから使用中であることを認識可能にする。
  • myTempNamespaceSubID を現在のサブトランザクション ID に設定。
  • baseSearchPathValid および searchPathCacheValid を false に設定し、検索パスの再計算を強制する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/namespace.c 内の、一時テーブル作成に関連する処理（直接の呼び出し箇所は src/backend/catalog/namespace.c:4385 等）。
  • 呼び出し先: object_aclcheck, RecoveryInProgress, IsParallelWorker, get_namespace_oid, NamespaceCreate, CommandCounterIncrement, RemoveTempRelations, GetCurrentSubTransactionId。

関連データ構造

  • MyProc: プロセスごとの一時名前空間 ID を保持する構造体。
  • myTempNamespace, myTempToastNamespace: セッション内の名前空間 OID を保持するグローバル変数。

失敗モードと境界条件

  • 権限不足: ERRCODE_INSUFFICIENT_PRIVILEGE を発生させる。
  • 読み取り専用状態: リカバリ中やパラレルワーカーの場合、ERRCODE_READ_ONLY_SQL_TRANSACTION を発生させる。
  • 既に初期化済みの場合: Assert により停止する。

保守担当者向けメモ

  • 本関数はトランザクションのコミット前に実行されるため、ロールバック時には AtEOXact_Namespace 等による適切なクリーンアップが必要となる。
  • pg_temp_N の所有者は常にスーパーユーザーであり、権限チェックは pg_namespace_aclmask 等でセッション中に再評価される。
