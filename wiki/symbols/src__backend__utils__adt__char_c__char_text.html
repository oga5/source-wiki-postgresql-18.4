char_text

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/char.c:227

目的

PostgreSQLの内部型 "char"（1バイト文字型）を text 型へ変換します。

シグネチャまたは形状

Datum char_text(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CHAR(0) を介して "char" 型の値を受け取ります。
  • **出力**: 変換後の text 型を PG_RETURN_TEXT_P マクロで返します。

前提条件と不変条件

  • palloc を使用してメモリを確保するため、メモリコンテキストが有効である必要があります。
  • charout と同様の変換規則に従いますが、0x00（ヌルバイト）は空の text 値として扱われます。

アルゴリズムまたは振る舞い

1. 入力値が IS_HIGHBIT_SET マクロにより上位ビットがセットされていると判定された場合、\ooo 形式の8進数エスケープ文字列（4バイト）として text 構造体に格納します。 2. 上位ビットがセットされておらず、かつ '\0' でない場合、その1バイトをそのまま text 構造体に格納します。 3. 入力値が '\0' の場合、データ部を持たない空の text 構造体を返します。 4. 最終的に PG_RETURN_TEXT_P を通じて結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • **呼び出し先**: palloc（メモリ確保）、SET_VARSIZE、VARDATA、IS_HIGHBIT_SET、TOOCTAL、PG_RETURN_TEXT_P などのマクロおよび関数を使用します。

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • "char": 1バイトの内部文字型。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • 0x00 は空の文字列として扱われるため、データ長が0の text 型が生成されます。

保守担当者向けメモ

  • 本関数は charout と変換ロジックを共有していますが、0x00 の扱いにおいて明示的な区別を行っています。
  • IS_HIGHBIT_SET マクロは src/include/postgres.h で定義されています。
