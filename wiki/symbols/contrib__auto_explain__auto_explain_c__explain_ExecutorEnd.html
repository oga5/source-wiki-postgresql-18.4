explain_ExecutorEnd

定義

  • 種別: function
  • 定義箇所: contrib/auto_explain/auto_explain.c:370

目的

クエリ実行終了時に、実行時間が設定された閾値を超えている場合に実行計画をログ出力する。

シグネチャまたは形状

static void explain_ExecutorEnd(QueryDesc *queryDesc)

入力と出力

  • **入力**: QueryDesc *queryDesc（実行中のクエリ記述子）
  • **出力**: なし（副作用としてログ出力を行う）

前提条件と不変条件

  • queryDesc->totaltime が有効であること。
  • auto_explain_enabled() が真であること。
  • 処理は queryDesc->estate->es_query_cxt メモリコンテキスト内で実行される。

アルゴリズムまたは振る舞い

1. **統計の確定**: InstrEndLoop を呼び出し、実行統計を確定させる。 2. **閾値判定**: 実行時間（ミリ秒）が auto_explain_log_min_duration を超えているか確認する。 3. **計画の生成**: 閾値を超えた場合、ExplainState を初期化し、設定（analyze, verbose, format 等）に基づいて実行計画を文字列化する。 4. **フォーマット調整**: JSON 形式の場合は先頭と末尾を {} で囲む等の調整を行う。 5. **ログ出力**: ereport を使用して、実行時間と生成された計画文字列をログに記録する。 6. **フックの連鎖**: 最後に prev_ExecutorEnd（存在する場合）または standard_ExecutorEnd を呼び出し、標準の終了処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecutorEnd フックとして登録されており、PostgreSQL の実行エンジンから呼び出される。
  • **呼び出し先**:
  • InstrEndLoop, NewExplainState, ExplainBeginOutput, ExplainQueryText, ExplainQueryParameters, ExplainPrintPlan, ExplainPrintTriggers, ExplainPrintJITSummary, ExplainEndOutput
  • prev_ExecutorEnd または standard_ExecutorEnd

関連データ構造

  • QueryDesc: クエリ実行状態を保持する構造体。
  • ExplainState: EXPLAIN 出力を制御するための状態構造体。

失敗モードと境界条件

  • queryDesc->totaltime が NULL の場合はログ出力を行わない。
  • パラレルワーカー内での二重ログ出力については、モジュール全体で配慮されている（ソース内のコメント参照）。
  • ログ出力用の文字列生成はクエリコンテキストで行われるため、クエリ終了時に自動的に解放される。

保守担当者向けメモ

  • 本関数は ExecutorEnd フックとして機能するため、他の拡張モジュールとの共存を考慮し、必ず prev_ExecutorEnd を呼び出す必要がある。
  • ExplainState の設定は、GUC 変数（auto_explain_log_*）に依存している。
