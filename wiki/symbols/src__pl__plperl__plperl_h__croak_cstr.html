croak_cstr

定義

  • 種別: static inline function
  • 定義箇所: src/pl/plperl/plperl.h:174

目的

データベースエンコーディングの文字列を引数として受け取り、Perl の croak を呼び出すことで、PostgreSQL のエラーハンドリングメカニズムと整合した例外を安全に発生させる。

シグネチャまたは形状

static inline void croak_cstr(const char *str)

入力と出力

  • **入力**: str (データベースエンコーディングでエンコードされた C 文字列)
  • **出力**: なし（Perl インタプリタを介して例外をスローし、制御を PostgreSQL のエラーハンドラへ戻す）

前提条件と不変条件

  • dTHX マクロにより、Perl インタプリタコンテキストが利用可能であること。
  • str は有効な C 文字列であること。

アルゴリズムまたは振る舞い

1. croak_sv が定義されている場合:

  • cstr2sv(str) で文字列を Perl の SV に変換し、sv_2mortal で一時的なオブジェクトとして登録した後、croak_sv を呼び出す。

    • 2. croak_sv が定義されていない場合:

  • utf_e2u(str) を使用して文字列を UTF-8 に変換する。
  • mess() を使用してエラー位置情報を含む SV を作成し、UTF-8 フラグを立てる。
  • pfree() で変換後の文字列メモリを解放する。
  • get_sv("@", GV_ADD) で取得した Perl のエラー変数 $@ に値を設定し、croak(NULL) を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plperl/plperl.c 内の複数の箇所（edata->message を引数として渡す形で頻繁に利用されている）。
  • **呼び出し先**:
  • cstr2sv (条件付き)
  • utf_e2u (条件付き)
  • mess (条件付き)
  • croak / croak_sv (Perl API)
  • pfree (PostgreSQL メモリ管理)

関連データ構造

  • SV (Perl Scalar Value)
  • errsv (Perl のエラー変数 $@)

失敗モードと境界条件

  • str が NULL の場合、内部の変換関数（cstr2sv や utf_e2u）の挙動に依存する。
  • Perl インタプリタが初期化されていない環境で呼び出された場合、dTHX マクロにより未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は plperl.c からのエラー報告の標準的な手段である。
  • croak_sv の有無による分岐は、Perl のバージョン差異を吸収するための互換性維持コードである。
  • utf_e2u で確保したメモリは pfree で明示的に解放する必要がある点に注意すること。
