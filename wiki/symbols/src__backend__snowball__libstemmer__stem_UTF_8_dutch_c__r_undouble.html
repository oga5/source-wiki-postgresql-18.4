r_undouble

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_dutch.c:335

目的

ステミング処理において、接尾辞削除後に残った二重子音を解消し、単語の綴りを正規化する。

シグネチャまたは形状

static int r_undouble(struct SN_env * z)

入力と出力

  • 入力: SN_env 構造体（ステミング環境）。
  • 出力: 処理が成功した場合は 1、失敗した場合は 0 または負の値を返す。

前提条件と不変条件

  • z->p（文字列バッファ）が有効であること。
  • a_2 テーブルに基づく二重子音のパターンマッチングが可能な状態であること。

アルゴリズムまたは振る舞い

1. 現在位置の直前の文字が二重子音を形成しているかを確認する。 2. find_among_b を使用して、a_2 テーブル内の特定の二重子音パターンと後方マッチングを行う。 3. マッチした場合、ket（削除開始位置）を現在位置に設定する。 4. skip_b_utf8 を用いて、重複している文字を一つスキップして bra（削除終了位置）を特定する。 5. slice_del を呼び出し、該当する重複文字を削除する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_ISO_8859_1_danish.c、stem_ISO_8859_1_dutch.c、stem_ISO_8859_2_hungarian.c、stem_UTF_8_danish.c 等の各言語ステマー（直接の呼び出し関係はソースコード内の静的解析による）。
  • 呼び出し先: find_among_b、skip_b_utf8、slice_del（いずれも Snowball ランタイム関数）。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • a_2: 二重子音の検索に使用される among テーブル。

失敗モードと境界条件

  • z->c - 1 <= z->lb: 文字列の先頭に達している場合は処理を中断する。
  • find_among_b がマッチしない場合、処理を中断する。
  • skip_b_utf8 または slice_del が負の値を返した場合、そのエラーコードを伝播する。
