pg_time_usec_t

定義

  • 種別: typedef
  • 定義箇所: src/bin/pgbench/pgbench.c:372

目的

マイクロ秒単位の時間を表現するための型です。pgbench におけるトランザクションのスケジュール、レイテンシ計測、およびスレッドの実行時間管理に使用されます。

シグネチャまたは形状

typedef int64 pg_time_usec_t;

入力と出力

  • 入力: 64ビット整数（int64）として値を保持します。
  • 出力: 時間計測やスケジュール計算の基準値として、関数（executeMetaCommand 等）へポインタ経由で渡されます。

前提条件と不変条件

  • 符号付き64ビット整数であるため、マイクロ秒単位で非常に長い期間を表現可能です。
  • 時間の計測や比較の基準として一貫して使用されることが期待されます。

アルゴリズムまたは振る舞い

  • pgbench 内の各状態（CState や TState）において、イベントの発生時刻やスケジュール時刻を保持するために使用されます。
  • executeMetaCommand や processXactStats などの関数において、現在の時刻（now）を指す引数として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: pgbench.c 内の複数の箇所で、時刻管理用変数（txn_scheduled, sleep_until, stmt_begin 等）の型として参照されています。
  • 呼び出し先: 本シンボル自体は型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • CState: クライアントの状態を保持する構造体。txn_scheduled や stmt_begin などのフィールドで使用。
  • TState: スレッドの状態を保持する構造体。create_time や bench_start などのフィールドで使用。

失敗モードと境界条件

  • 整数オーバーフロー: int64 の範囲を超える極端な長時間計測を行う場合、オーバーフローのリスクがあります。
  • 精度: マイクロ秒単位の精度に依存しており、それ以下の精度を必要とする計測には適していません。

保守担当者向けメモ

  • pgbench.c 内で時刻を扱う際の標準的な型として定着しています。時刻計算のロジックを変更する際は、この型が int64 であることを考慮してください。
