ItemPointerSetOffsetNumber

定義

  • 種別: function
  • 定義箇所: src/include/storage/itemptr.h:157

目的

ItemPointerData 構造体内のオフセット番号（ip_posid）を特定の値に設定する。

シグネチャまたは形状

static inline void ItemPointerSetOffsetNumber(ItemPointerData *pointer, OffsetNumber offsetNumber)

入力と出力

  • **入力**:
  • pointer: 設定対象の ItemPointerData 構造体へのポインタ。
  • offsetNumber: 設定する OffsetNumber 型の値。
  • **出力**: なし（副作用として pointer が指すメモリの内容を更新）。

前提条件と不変条件

  • pointer は有効なメモリ領域を指している必要がある（Assert(PointerIsValid(pointer)) により検証）。
  • offsetNumber は OffsetNumber 型として妥当な値であること。

アルゴリズムまたは振る舞い

1. pointer が NULL でないことを Assert で確認する。 2. pointer が指す構造体の ip_posid メンバに、引数 offsetNumber を代入する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistutil.c
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/heapam_handler.c
  • src/backend/access/heap/heapam_xlog.c
  • src/backend/access/nbtree/nbtutils.c
  • src/include/access/ginblock.h
  • src/include/access/gist_private.h
  • src/include/access/nbtree.h
  • **呼び出し先**:
  • Assert マクロ（デバッグビルド時のみ）。

関連データ構造

  • ItemPointerData: タプル位置を保持する 6 バイトの構造体。
  • OffsetNumber: ページ内のアイテム位置を示す型。

失敗モードと境界条件

  • pointer が NULL の場合、デバッグビルドでは Assert によりアボートする。
  • offsetNumber に MaxOffsetNumber を超える値を設定した場合、システムはそれを通常のタプル位置ではなく、移動済みパーティションや推論的挿入トークンなどの特殊状態として解釈する可能性がある。

保守担当者向けメモ

  • 本関数はインライン関数であり、パフォーマンスを考慮して実装されている。
  • ItemPointerData のメモリレイアウトは固定されているため、構造体のメンバ順序や型定義を変更する際は注意が必要である。
