exec_check_rw_parameter

定義

  • src/pl/plpgsql/src/pl_exec.c:8400

目的

PL/pgSQL の式評価において、展開されたオブジェクト（expanded object）を読み書き可能（read/write）なパラメータとして渡すことが可能かどうかを判定し、最適化の可否を決定します。

シグネチャまたは形状

static void exec_check_rw_parameter(PLpgSQL_expr *expr, int paramid)

入力と出力

  • **入力**:
  • expr: 評価対象の PLpgSQL_expr 構造体。
  • paramid: 割り当て対象のパラメータ ID。
  • **出力**: なし（副作用として expr の expr_rwopt および expr_rw_param フィールドを更新）。

前提条件と不変条件

  • expr->expr_simple_expr が NULL であってはならない（単純な式であること）。
  • paramid は expr->target_param + 1 と一致しなければならない。

アルゴリズムまたは振る舞い

1. **初期化**: expr->expr_rwopt を PLPGSQL_RWOPT_NOPE に設定し、最適化不可として初期化します。 2. **ローカル変数最適化**: expr->target_is_local が真の場合、式内のパラメータ参照数をカウントします。参照がちょうど1つであれば、PLPGSQL_RWOPT_TRANSFER を適用し、そのパラメータを expr_rw_param に設定して終了します。 3. **関数ベースの最適化**: 式のトップレベルが FuncExpr, OpExpr, または SubscriptingRef である場合、対応するサポート関数（get_func_support）を呼び出します。 4. **サポート関数の検証**: サポート関数が T_SupportRequestModifyInPlace リクエストに対して有効な Param を返した場合、PLPGSQL_RWOPT_INPLACE を適用し、そのパラメータを expr_rw_param に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/pl_exec.c 内の式処理に関連する箇所から呼び出されます。
  • **呼び出し先**: count_param_references, get_func_support, OidFunctionCall1 等を呼び出します。

関連データ構造

  • PLpgSQL_expr: 式の定義と最適化状態を保持。
  • count_param_references_context: パラメータ参照カウント用のコンテキスト構造体。
  • SupportRequestModifyInPlace: サポート関数へのインプレース更新リクエスト用構造体。

失敗モードと境界条件

  • 式が単純な式（expr_simple_expr）でない場合はアサーションエラーとなります。
  • サポート関数が定義されていない、またはインプレース更新を許可しない場合は最適化されません。
  • SubscriptingRef の場合、refexpr と refassgnexpr のみがサポート関数の判断材料として使用されます。

保守担当者向けメモ

  • 本関数は式評価の最適化パスの一部です。サポート関数が返す Param が API に準拠しているか（PARAM_EXTERN かつ paramid が一致するか）を厳格に検証しています。
  • SubscriptingRef の引数リストの構築方法については、将来的に拡張が必要になる可能性があります。
