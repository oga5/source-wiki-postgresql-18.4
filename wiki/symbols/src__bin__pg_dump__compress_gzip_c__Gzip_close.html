Gzip_close

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_gzip.c:339

目的

GZIP 圧縮ファイルハンドルを閉じ、関連するリソースを解放します。

シグネチャまたは形状

static bool Gzip_close(CompressFileHandle *CFH)

入力と出力

  • **入力**: CompressFileHandle *CFH - 閉じる対象のファイルハンドル構造体。
  • **出力**: bool - 正常に閉じられた場合は true、それ以外（zlib エラー等）は false を返します。

前提条件と不変条件

  • CFH->private_data に有効な gzFile ハンドルが格納されている必要があります。
  • 実行後、CFH->private_data は NULL に設定されます。

アルゴリズムまたは振る舞い

1. CFH->private_data から gzFile ハンドルを取得します。 2. CFH->private_data を NULL にクリアします。 3. gzclose(gzfp) を呼び出し、戻り値が Z_OK であるかを確認して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: gzclose (zlib ライブラリ関数)
  • **呼び出し元**:
  • src/bin/pg_dump/compress_gzip.c 内で CFH->close_func に代入されています。
  • 直接的な呼び出し元については、提供されたソース excerpt に記載がありません。

関連データ構造

  • CompressFileHandle: 圧縮 I/O の抽象化に使用される構造体。private_data メンバを介して gzFile を保持します。

失敗モードと境界条件

  • gzclose が Z_OK 以外の値を返した場合、関数は false を返します。
  • CFH が NULL の場合の動作については、提供されたソース excerpt にガードロジックが含まれていないため、未定義です。

保守担当者向けメモ

  • 本関数は CompressFileHandle インターフェースの一部として実装されています。
  • gzclose の戻り値チェックにより、圧縮ストリームの終了処理におけるエラーを呼び出し元へ伝播させます。
