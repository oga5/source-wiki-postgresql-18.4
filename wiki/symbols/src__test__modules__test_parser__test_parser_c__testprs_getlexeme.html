testprs_getlexeme

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_parser/test_parser.c:58

目的

PostgreSQLのカスタムパーサーインターフェースにおいて、入力テキストから次のトークンを抽出し、そのトークン型を判定して返す。

シグネチャまたは形状

Datum testprs_getlexeme(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): ParserState 構造体へのポインタ（解析状態）。
  • PG_GETARG_POINTER(1): トークンの開始位置を格納するための char ** 型の出力用ポインタ。
  • PG_GETARG_POINTER(2): トークン長を格納するための int * 型の出力用ポインタ。
  • **出力**:
  • 戻り値: トークン型を表す int32（空白: 12, 単語: 3, 終了: 0）。
  • 副作用: *t にトークンの開始アドレスが設定され、*tlen にトークン長が設定される。また、ParserState 内の pos が更新される。

前提条件と不変条件

  • ParserState は testprs_start によって適切に初期化されている必要がある。
  • pst->pos は常に 0 以上 pst->len 以下の範囲内であること。

アルゴリズムまたは振る舞い

1. 現在の解析位置（pst->pos）からトークンの開始位置を特定する。 2. 現在位置の文字が空白か否かでトークン型を決定する。

  • 空白の場合: 型 12 を割り当て、空白が続く限り pst->pos を進める。
  • 空白以外の場合: 型 3 を割り当て、空白が現れるまで pst->pos を進める。

    • 3. pst->pos の移動量からトークン長（*tlen）を算出する。 4. トークン長が 0 の場合は解析終了とみなし、型を 0 に設定する。 5. 決定した型を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの全文検索エンジン（ts_parser インターフェース経由）。ソースコード内では PG_FUNCTION_INFO_V1(testprs_getlexeme) による登録が確認できる。
  • **呼び出し先**: PostgreSQLの関数マネージャ（PG_GETARG_POINTER, PG_RETURN_INT32 等のマクロ経由）。

関連データ構造

  • ParserState: 解析中のバッファ、バッファ長、現在位置を保持する構造体。

失敗モードと境界条件

  • pst->pos が pst->len に達している場合、トークン長は 0 となり、型 0 が返される。
  • 入力バッファが空の場合、即座に型 0 が返される。

保守担当者向けメモ

  • 本関数は ts_parser インターフェースの仕様に従い、トークン型として 3（単語）と 12（空白）を返します。これらはPostgreSQL標準パーサーと互換性があります。
  • トークン型の追加や変更を行う場合は、testprs_lextype の定義と整合性を取る必要があります。
