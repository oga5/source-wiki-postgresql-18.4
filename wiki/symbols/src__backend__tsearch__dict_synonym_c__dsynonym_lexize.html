dsynonym_lexize

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/dict_synonym.c:211

目的

全文検索における単語の置換処理。入力された単語を辞書データと照合し、一致した場合は対応する同義語を返します。

シグネチャまたは形状

Datum dsynonym_lexize(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): DictSyn 構造体（辞書データ）。
  • PG_GETARG_POINTER(1): 入力単語文字列。
  • PG_GETARG_INT32(2): 入力単語の長さ。
  • **出力**:
  • 一致した場合は TSLexeme 配列（置換語とフラグを含む）へのポインタ。
  • 一致しない場合や入力が無効な場合は NULL。

前提条件と不変条件

  • 入力単語の長さ len が 0 以下、または辞書のサイズ d->len が 0 以下の場合は処理を行わず NULL を返します。
  • 辞書データは初期化時に qsort でソート済みである必要があります。

アルゴリズムまたは振る舞い

1. 辞書の case_sensitive フラグに基づき、入力単語をそのまま使用するか、str_tolower で小文字化して検索キーを作成します。 2. bsearch を使用し、compareSyn 関数を介してソート済み辞書配列から一致する Syn 構造体を検索します。 3. 検索キーとして使用した一時メモリを pfree で解放します。 4. 一致が見つかった場合、palloc0 で TSLexeme 配列を確保し、置換語とフラグをコピーして返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの全文検索エンジン（ts_lexize インターフェース経由）。具体的な呼び出し元コードの詳細は提供されていません。
  • **呼び出し先**: bsearch（標準ライブラリ）、compareSyn、str_tolower、palloc0、pfree、pnstrdup。

関連データ構造

  • DictSyn: 辞書全体の状態を保持。
  • Syn: 単語のペア（入力語と出力語）を保持。
  • TSLexeme: 検索結果として返される語彙情報。

失敗モードと境界条件

  • 入力単語が辞書に存在しない場合、NULL を返します。
  • 辞書が空の場合、検索を行わずに NULL を返します。
  • メモリ不足時は palloc0 等がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 辞書検索には bsearch を使用しているため、辞書定義のソート順序が維持されていることが重要です。
  • d->len のチェックは、Solaris環境等における bsearch のバグ（要素数0の際の挙動）を回避するための安全策です。
