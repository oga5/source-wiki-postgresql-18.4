ExecMarkPos

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execAmi.c:326

目的

プランノードの現在のスキャン位置を記録し、後続の ExecRestrPos による復元を可能にするためのインターフェースを提供します。

シグネチャまたは形状

void ExecMarkPos(PlanState *node)

入力と出力

  • **入力**: PlanState *node - スキャン位置を記録対象とするプランノードの状態構造体。
  • **出力**: なし（副作用としてノード内部の状態が更新されます）。

前提条件と不変条件

  • 引数 node は有効な PlanState を指している必要があります。
  • すべてのノードタイプがマーク機能をサポートしているわけではありません。サポート外のノードに対して呼び出された場合、デバッグログが出力されるのみで、マーク操作は行われません。

アルゴリズムまたは振る舞い

1. nodeTag(node) を使用して、プランノードの型を判定します。 2. 判定された型に応じて、以下の専用関数を呼び出します。

  • T_IndexScanState → ExecIndexMarkPos
  • T_IndexOnlyScanState → ExecIndexOnlyMarkPos
  • T_CustomScanState → ExecCustomMarkPos
  • T_MaterialState → ExecMaterialMarkPos
  • T_SortState → ExecSortMarkPos
  • T_ResultState → ExecResultMarkPos

    • 3. 上記以外のノードタイプの場合は、DEBUG2 レベルのログを出力して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeMergejoin.c (複数箇所)
  • src/backend/executor/nodeResult.c
  • **呼び出し先**:
  • 各ノード専用のマーク関数（ExecIndexMarkPos, ExecMaterialMarkPos 等）が呼び出されます。

関連データ構造

  • PlanState: 実行中のプランノードの状態を保持する基底構造体。

失敗モードと境界条件

  • 未対応のノードタイプが渡された場合、エラーにはならず DEBUG2 ログが出力されます。
  • 呼び出し元が ExecRestrPos を要求する前にマーク操作が成功していることを前提とする場合、この関数がマークを記録できないノードに対して呼び出されると、後の復元処理で問題が発生する可能性があります。

保守担当者向けメモ

  • 新しいプランノードタイプを追加し、そのノードがマーク/リストア機能をサポートする場合は、本関数の switch 文に新しい case を追加してください。
  • ExecSupportsMarkRestore 関数と整合性を保つように注意してください。
