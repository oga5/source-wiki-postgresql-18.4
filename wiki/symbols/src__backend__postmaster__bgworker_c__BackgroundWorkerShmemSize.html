BackgroundWorkerShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/bgworker.c:145

目的

バックグラウンドワーカー機能が共有メモリ上で必要とする総バイト数を算出する。

シグネチャまたは形状

Size BackgroundWorkerShmemSize(void)

入力と出力

  • 入力: なし（グローバル変数 max_worker_processes を参照）
  • 出力: Size 型の共有メモリ所要バイト数

前提条件と不変条件

  • max_worker_processes が適切に設定されていること。
  • 共有メモリのサイズ計算において、BackgroundWorkerArray のヘッダーサイズと、全スロット分の BackgroundWorkerSlot サイズの合計が算出される。

アルゴリズムまたは振る舞い

1. BackgroundWorkerArray 構造体の slot メンバーまでのオフセットを取得する。 2. max_worker_processes に sizeof(BackgroundWorkerSlot) を乗算し、スロット配列全体のサイズを計算する。 3. 上記の合計値を add_size 関数を用いて安全に加算し、結果を返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/ipc/ipci.c: 共有メモリの初期化プロセス（CreateSharedMemoryAndSemaphores）から呼び出される。
  • src/backend/postmaster/bgworker.c: 内部的な初期化処理（BackgroundWorkerShmemInit）から参照される。
  • 呼び出し先:
  • add_size (PostgreSQL ユーティリティ関数)
  • mul_size (PostgreSQL ユーティリティ関数)

関連データ構造

  • BackgroundWorkerArray: 共有メモリ上のワーカー管理用配列構造体。
  • BackgroundWorkerSlot: 各ワーカーの状態を保持する個別のスロット構造体。

失敗モードと境界条件

  • max_worker_processes が極端に大きい場合、mul_size および add_size によるオーバーフローチェックが行われる。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウトを決定する重要な関数である。BackgroundWorkerArray 構造体の定義を変更した場合は、必ず本関数の計算ロジックを更新すること。
