dblink_init

定義

  • 種別: function
  • 定義箇所: contrib/dblink/dblink.c:267

目的

dblink モジュールで使用される待機イベントの登録および、匿名接続用構造体 pconn の初期化を行う。

シグネチャまたは形状

static void dblink_init(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 pconn および dblink_we_get_result を初期化）

前提条件と不変条件

  • pconn が NULL である場合にのみ初期化処理が実行される。
  • pconn は TopMemoryContext 上に割り当てられる。

アルゴリズムまたは振る舞い

1. pconn が未初期化か確認する。 2. dblink_we_get_result が未設定の場合、WaitEventExtensionNew("DblinkGetResult") を呼び出し、待機イベントを登録する。 3. TopMemoryContext を使用して remoteConn 構造体分のメモリを確保し、pconn に割り当てる。 4. pconn のメンバを初期化する（conn = NULL, openCursorCount = 0, newXactForCursor = false）。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/dblink/dblink.c 内の多数の関数（dblink_connect, dblink_exec, dblink_record 等、計12箇所で呼び出しを確認）。
  • 呼び出し先: WaitEventExtensionNew（PostgreSQL 内部 API）、MemoryContextAlloc。

関連データ構造

  • remoteConn: 接続状態を保持する構造体。
  • TopMemoryContext: メモリ割り当てに使用される永続的なメモリコンテキスト。

失敗モードと境界条件

  • MemoryContextAlloc がメモリ不足により失敗した場合、PostgreSQL のエラーハンドリング機構により例外がスローされる。

保守担当者向けメモ

  • 本関数は pconn が一度初期化されると再実行されないため、匿名接続のライフサイクル管理の起点となっている。
  • 待機イベントの登録は初回実行時のみ行われる。
