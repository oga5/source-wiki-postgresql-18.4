gbt_inetge

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_inet.c:33

目的

GiST インデックス内において、inet 型のスカラ変換値（double）を比較し、「以上（greater than or equal to）」の条件判定を行う。

シグネチャまたは形状

static bool gbt_inetge(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象となる double 型値へのポインタ（void * として渡される）。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • bool: *a >= *b が真であれば true、そうでなければ false を返す。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な double 型の値を指している必要がある。
  • 本関数は inet 型が double に変換された後の比較を行うため、変換後の値の順序関係が元の inet 型の順序を保持していることが前提となる。

アルゴリズムまたは振る舞い

1. a および b を double 型のポインタとしてキャストする。 2. それぞれのポインタが指す値をデリファレンスし、比較演算子 >= を用いて大小比較を行う。 3. 比較結果をそのまま戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_inet.c 内の静的配列（tinfo 構造体の初期化箇所）から参照されている。
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わない。

関連データ構造

  • double: inet 型の値をインデックス内で比較可能なスカラ値として表現するために使用される。

失敗モードと境界条件

  • ポインタが NULL の場合、デリファレンスによりセグメンテーション違反が発生する可能性がある。
  • double 型への変換精度により、非常に大きなネットワーク範囲や複雑なサブネットマスクにおいて比較結果が期待通りにならない可能性がある。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で定義される汎用的な GiST 比較ロジックの一部として利用される。
  • inet 型の比較は double への変換を伴うため、精度の限界について contrib/btree_gist/btree_inet.c のファイル概要を参照すること。
