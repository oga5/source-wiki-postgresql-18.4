ForeignTable

定義

  • 種別: struct
  • 定義箇所: src/include/foreign/foreign.h:53

目的

外部テーブルのメタデータ（リレーションID、サーバーID、およびオプション設定）を保持し、システムキャッシュから取得した情報を管理するための構造体です。

シグネチャまたは形状

typedef struct ForeignTable
{
	Oid			relid;			/* relation Oid */
	Oid			serverid;		/* server Oid */
	List	   *options;		/* ftoptions as DefElem list */
} ForeignTable;

入力と出力

  • 本構造体は、システムカタログから情報を取得する関数（GetForeignTable等）によってインスタンス化され、呼び出し元に返されます。
  • options フィールドは DefElem のリストとして保持されます。

前提条件と不変条件

  • システムキャッシュから情報をコピーして作成されるため、使用後はメモリ管理（pfree等）が必要です。
  • options は各FDWの実装側で解析されることを前提としています。

アルゴリズムまたは振る舞い

  • 外部データラッパー（FDW）が外部テーブルの属性（どのサーバーに属しているか、どのようなオプションが設定されているか）を特定するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • contrib/file_fdw/file_fdw.c
  • contrib/postgres_fdw/deparse.c
  • contrib/postgres_fdw/postgres_fdw.c
  • 呼び出し先: 情報なし。

関連データ構造

  • List（nodes/parsenodes.hで定義）: options フィールドの格納に使用。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • 構造体内の各フィールドはシステムカタログのデータに依存します。カタログ情報が存在しない場合の挙動は、本構造体を取得する関数の flags 引数（MISSING_OK等）によって制御されます。

保守担当者向けメモ

  • 本構造体は src/backend/foreign/foreign.c で実装される関数群と対で使用されます。
  • 構造体内の options は DefElem のリストであるため、アクセス時には適切なリスト走査が必要です。
