GenericXLogRegisterBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/generic_xlog.c:298

目的

汎用WALレコード構築において、変更対象のバッファを登録し、編集可能なページイメージを呼び出し元に提供します。

シグネチャまたは形状

Page GenericXLogRegisterBuffer(GenericXLogState *state, Buffer buffer, int flags)

入力と出力

  • **入力**:
  • state: 構築中のWALレコードの状態を保持する GenericXLogState へのポインタ。
  • buffer: 登録対象のバッファ。
  • flags: ページ処理に関するフラグ（例: GENERIC_XLOG_FULL_IMAGE）。
  • **出力**:
  • 編集可能なページイメージ（Page 型）。登録されたバッファのコピーが返されます。

前提条件と不変条件

  • state は GenericXLogStart() によって初期化されている必要があります。
  • 登録可能なバッファ数は MAX_GENERIC_XLOG_PAGES に制限されています。
  • 既に登録済みのバッファが指定された場合、新しいイメージを作成せず、既存のイメージを返します。

アルゴリズムまたは振る舞い

1. state->pages 配列を走査し、指定された buffer が既に登録されているか確認します。 2. 登録済みであれば、そのバッファに対応する page->image を返します。 3. 未登録の空きスロットが見つかった場合、そのスロットに buffer を割り当て、BufferGetPage(buffer) の内容を page->image にコピーして返します。 4. MAX_GENERIC_XLOG_PAGES を超えて登録しようとした場合、elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/bloom/blinsert.c、contrib/bloom/blutils.c、contrib/bloom/blvacuum.c 等のインデックスアクセス手法の実装から呼び出されます。
  • **呼び出し先**: BufferGetPage（マクロ）を呼び出し、バッファからページイメージを取得します。

関連データ構造

  • GenericXLogState: 汎用WALレコードの構築状態を管理する構造体。
  • GenericXLogPageData: 各バッファの登録状態とページイメージを保持する構造体。

失敗モードと境界条件

  • **エラー**: 登録済みバッファ数が MAX_GENERIC_XLOG_PAGES を超えた場合、エラーが発生します。
  • **無効なバッファ**: BufferIsInvalid を用いて空きスロットを判定しており、無効なバッファが渡された場合の挙動は未定義です。

保守担当者向けメモ

  • 本関数は、ページ変更の差分計算（デルタ圧縮）の起点となります。ここで取得した Page に対して行った変更は、最終的に GenericXLogFinish() が呼び出された際に元のページと比較され、差分がWALレコードに記録されます。
