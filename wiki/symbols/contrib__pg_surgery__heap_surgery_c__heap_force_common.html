heap_force_common

定義

  • 種別: function
  • 定義箇所: contrib/pg_surgery/heap_surgery.c:84

目的

破損したヒープテーブルに対し、指定された TID リストに基づいて強制的な状態変更（Kill または Freeze）を共通ロジックで実行する。

シグネチャまたは形状

static Datum heap_force_common(FunctionCallInfo fcinfo, HeapTupleForceOption heap_force_opt)

入力と出力

  • **入力**:
  • fcinfo: SQL 関数呼び出し情報（リレーション OID と TID 配列を含む）。
  • heap_force_opt: 実行する操作の種類（HEAP_FORCE_KILL または HEAP_FORCE_FREEZE）。
  • **出力**:
  • Datum: 処理結果（通常は PG_RETURN_VOID() に相当する値）。

前提条件と不変条件

  • リカバリ中（RecoveryInProgress()）ではないこと。
  • 対象リレーションがヒープ AM を使用していること。
  • 実行者が対象リレーションの所有者またはスーパーユーザーであること。
  • 対象リレーションに対して RowExclusiveLock を取得すること。

アルゴリズムまたは振る舞い

1. **入力検証**: TID 配列をソートし、ページ単位で処理できるように準備する。 2. **ページ走査**: find_tids_one_page を用いて、対象ブロック内の TID を特定する。 3. **バッファ操作**: 対象ブロックを ReadBuffer で読み込み、LockBufferForCleanup で排他ロックを取得する。 4. **タプル選別**: ページ内の各 TID について、有効なタプルかを確認し、処理対象をマークする。 5. **状態変更**:

  • KILL の場合: ItemIdSetDead を使用してタプルを無効化し、必要に応じて可視性マップを更新する。
  • FREEZE の場合: タプルヘッダーを直接書き換え、FrozenTransactionId を設定する。

    • 6. **コミット**: 変更がある場合、MarkBufferDirty を呼び出し、WAL ログを生成してロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • heap_force_kill
  • heap_force_freeze
  • **呼び出し先**:
  • sanity_check_tid_array, relation_open, qsort, tidcmp, find_tids_one_page, ReadBuffer, LockBufferForCleanup, BufferGetPage, PageGetMaxOffsetNumber, PageGetItemId, ItemIdSetDead, visibilitymap_pin, START_CRIT_SECTION 等。

関連データ構造

  • HeapTupleForceOption: 操作種別を定義する列挙型。
  • ItemPointer: TID 配列のデータ構造。
  • Buffer: バッファ管理用ハンドル。

失敗モードと境界条件

  • **エラー**: リカバリ中の実行、非ヒープテーブルへの操作、権限不足、無効なブロック番号やオフセット番号の指定。
  • **通知**: 範囲外のブロックや無効な TID が指定された場合、NOTICE を発行してスキップする。

保守担当者向けメモ

  • 本関数は PostgreSQL の整合性チェックをバイパスして直接データを書き換えるため、緊急時の最終手段としてのみ使用すること。
  • 変更は WAL に記録されるため、PITR やレプリケーションには反映される。
