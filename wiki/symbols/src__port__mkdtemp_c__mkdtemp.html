mkdtemp

定義

  • 種別: function
  • 定義箇所: src/port/mkdtemp.c:285

目的

プラットフォーム標準の mkdtemp(3) が欠如している環境において、指定されたテンプレート文字列に基づき、一意な名前を持つディレクトリをモード 0700 で安全に作成します。

シグネチャまたは形状

char *mkdtemp(char *path);

入力と出力

  • **入力**: path (ディレクトリ名テンプレートへのポインタ。末尾に X を含む必要があります)
  • **出力**: 成功時は変更された path へのポインタを返し、失敗時は NULL を返します。

前提条件と不変条件

  • path は NULL であってはなりません（_DIAGASSERT による検証）。
  • テンプレート内の X は、プロセス ID やカウンタを用いて一意な文字列に置換されます。
  • ディレクトリ作成は排他的に行われます。

アルゴリズムまたは振る舞い

1. 内部補助関数 GETTEMP(path, NULL, 1) を呼び出します。 2. GETTEMP はテンプレートの X を解析し、mkdir(2) を用いてディレクトリ作成を試みます。 3. EEXIST エラーが発生した場合は、テンプレート内の文字をインクリメントして別の名前を生成し、成功するまで再試行します。 4. 成功した場合、呼び出し元に path を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/regress/pg_regress.c:506 での利用が確認されています。
  • **呼び出し先**: 内部関数 GETTEMP を呼び出します。

関連データ構造

  • 特になし（標準的な C 文字列およびシステムコールを使用）。

失敗モードと境界条件

  • path が無効な場合、アサーションエラーが発生する可能性があります。
  • 一意なディレクトリ名を作成できない場合や、mkdir が失敗した場合は NULL を返します。
  • HAVE_MKDTEMP が定義されている環境では、本実装はコンパイルされません。

保守担当者向けメモ

  • 本実装は NetBSD 由来です。OpenBSD 版と比較して暗号学的依存関係を避ける設計となっています。
  • mkdtemp は mktemp と異なり、ディレクトリ作成がアトミックに行われるため、競合状態に対して安全です。
