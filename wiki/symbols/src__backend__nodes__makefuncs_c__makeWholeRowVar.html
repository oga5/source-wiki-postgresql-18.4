makeWholeRowVar

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/makefuncs.c:136

目的

行全体（Whole-Row）を参照する Var ノードを生成します。RTE（Range Table Entry）の種類に応じて適切な型解決を行い、クエリツリー構築時に必要なノード構造を抽象化します。

シグネチャまたは形状

Var *makeWholeRowVar(RangeTblEntry *rte, int varno, Index varlevelsup, bool allowScalar)

入力と出力

  • **入力**:
  • rte: 参照対象の RangeTblEntry。
  • varno: 生成する Var ノードの varno。
  • varlevelsup: 外部クエリレベルへの参照深度。
  • allowScalar: RTE_FUNCTION の場合にスカラー値を許容するかどうかのフラグ。
  • **出力**:
  • 構築された Var ノードへのポインタ。

前提条件と不変条件

  • RTE_RELATION の場合、対象リレーションが複合型（Composite Type）を持つ必要があります。持たない場合はエラーとなります。
  • RTE_FUNCTION において allowScalar が false の場合、関数がスカラーを返しても RECORDOID 型の Var が生成されます。

アルゴリズムまたは振る舞い

rte->rtekind に基づき以下の処理を行います。 1. **RTE_RELATION**: リレーションの型OIDを取得し、makeVar を呼び出して Var を生成します。 2. **RTE_SUBQUERY**:

  • ビュー展開済みであればその型を使用。
  • セットを返す関数の場合はその型を解決。
  • 通常のサブクエリは RECORDOID を使用。

    • 3. **RTE_FUNCTION**:

  • 複数関数や順序付き関数の場合は RECORDOID を強制。
  • 単一関数の場合、戻り値が複合型ならその型を、スカラーかつ allowScalar が真ならその型を、それ以外は RECORDOID を使用して makeVar を呼び出します。

    • 4. **その他（JOIN, VALUES等）**: RECORDOID を使用して makeVar を呼び出します。

関連データ構造

  • Var: 生成されるノードの型。
  • RangeTblEntry: 入力として使用される範囲テーブルエントリ。
  • RangeTblFunction: RTE_FUNCTION 処理時に参照される関数情報。

失敗モードと境界条件

  • get_rel_type_id が有効なOIDを返さない場合、ERRCODE_WRONG_OBJECT_TYPE でエラー終了します。
  • RTE_FUNCTION で allowScalar が false の際に、関数がスカラーを返す場合は RECORDOID にフォールバックされます。

保守担当者向けメモ

  • 生成される Var ノードの location フィールドは本関数では設定されません。必要に応じて呼び出し側で補完してください。
  • RTE_FUNCTION の処理ロジックは、関数の戻り値が複合型かスカラーかによって挙動が分岐するため、変更時は注意が必要です。
