ConfigData

定義

  • 種別: struct
  • 定義箇所: src/include/common/config_info.h:12

目的

PostgreSQL のビルド時設定やインストールディレクトリ情報を保持するためのデータ構造です。設定項目名とそれに対応する値をペアとして管理します。

シグネチャまたは形状

typedef struct ConfigData
{
	char	   *name;
	char	   *setting;
} ConfigData;

入力と出力

  • 本構造体は、get_configdata() 関数によって生成される配列の要素として使用されます。
  • name および setting は、palloc によって確保された文字列へのポインタです。

前提条件と不変条件

  • get_configdata() が返す ConfigData 配列および各メンバ文字列は、palloc で確保されます。
  • 呼び出し元は、使用後に必ず pfree を用いてメモリを解放する責任を負います。

アルゴリズムまたは振る舞い

  • get_configdata() 関数において、BINDIR や DOCDIR などのディレクトリパス、および VAL_CC などのビルド時定数が本構造体のインスタンスに格納されます。
  • 呼び出し元は、この構造体の配列を走査することで、実行環境の設定情報を取得します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/pg_config.c
  • src/bin/pg_config/pg_config.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。生成元として src/common/config_info.c 内の get_configdata() が関与しています。

関連データ構造

  • get_configdata(): 本構造体の配列を生成・返却する関数。

失敗モードと境界条件

  • メモリ不足時: palloc が失敗した場合、システムはエラーを発生させます。
  • 配列の拡張: 新しい設定項目を追加する場合、src/common/config_info.c 内の配列サイズ定義とループ処理を更新する必要があります。

保守担当者向けメモ

  • 本構造体は主に pg_config ユーティリティ向けですが、バックエンドでも利用可能です。
  • メモリ管理の責任が呼び出し元にある点に注意してください。
