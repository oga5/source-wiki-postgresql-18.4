count_nonjunk_tlist_entries

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/tlist.c:194

目的

ターゲットリスト（List）に含まれる TargetEntry のうち、resjunk フラグが立っていない（非ジャンクな）エントリの総数をカウントします。

シグネチャまたは形状

int count_nonjunk_tlist_entries(List *tlist)

入力と出力

  • **入力**: tlist - TargetEntry ノードを要素として持つリスト。
  • **出力**: int - リスト内の非ジャンクエントリの個数。

前提条件と不変条件

  • tlist は TargetEntry ノードのリストである必要があります。
  • resjunk が true のエントリは、クエリの最終結果には含まれない補助的な列（ソート用キーなど）として扱われます。

アルゴリズムまたは振る舞い

1. カウンタ len を 0 に初期化します。 2. foreach マクロを使用して、リスト tlist の各要素を走査します。 3. 各要素を TargetEntry 型にキャストし、resjunk フィールドを確認します。 4. resjunk が偽（false）であれば、len をインクリメントします。 5. リストの走査完了後、最終的な len を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_expr.c: maref->ncolumns との比較や、ターゲットリストの要素数検証に使用。
  • src/backend/utils/adt/ruleutils.c: サブリンク内のターゲットリストの列数取得に使用。
  • **呼び出し先**:
  • lfirst マクロ（リスト要素へのアクセス）。

関連データ構造

  • List: PostgreSQL の汎用リスト構造。
  • TargetEntry: クエリのターゲットリストを構成するノード。resjunk フィールドを保持します。

失敗モードと境界条件

  • tlist が空リスト（NIL）の場合、0 を返します。
  • tlist 内のすべてのエントリが resjunk = true である場合、0 を返します。

保守担当者向けメモ

  • 本関数は、クエリの解析段階やルール展開時に、ユーザーが指定した列数と実際のターゲットリストの整合性を確認するために頻繁に使用されます。
  • resjunk フラグの定義については src/include/nodes/primnodes.h の TargetEntry 構造体定義を参照してください。
