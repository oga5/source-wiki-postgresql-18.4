toastid_valueid_exists

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/toast_internals.c:506

目的

指定されたTOASTテーブル内に、特定の valueid を持つデータが存在するかを確認する。

シグネチャまたは形状

bool toastid_valueid_exists(Oid toastrelid, Oid valueid)

入力と出力

  • **入力**:
  • toastrelid (Oid): 検索対象のTOASTテーブルのOID。
  • valueid (Oid): 検索対象のデータ識別子。
  • **出力**:
  • bool: 該当するデータが存在する場合は true、存在しない場合は false。

前提条件と不変条件

  • toastrelid で指定されたリレーションは、有効なTOASTテーブルである必要がある。
  • 呼び出し元は、適切なトランザクションコンテキストおよびスナップショットを保持している必要がある。

アルゴリズムまたは振る舞い

1. table_open を使用し、AccessShareLock を取得して対象のTOASTテーブルを開く。 2. toastrel_valueid_exists を呼び出し、実際の検索処理を委譲する。 3. table_close を使用し、取得したロックを解放する。 4. 検索結果を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/toast_internals.c:287 (推測される利用箇所)
  • **呼び出し先**:
  • table_open
  • toastrel_valueid_exists
  • table_close

関連データ構造

  • Relation: TOASTテーブルのメタデータ管理に使用。
  • Oid: テーブルおよびデータ識別子の型。

失敗モードと境界条件

  • toastrelid が無効な場合、table_open 内でエラーが発生する可能性がある。
  • ロック取得時にデッドロックやタイムアウトが発生する可能性がある。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的な検索ロジックは toastrel_valueid_exists に依存している。
  • AccessShareLock を使用しているため、読み取り専用の存在確認として設計されている。
