gbt_date_fetch

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_date.c:142

目的

GiST インデックスのリーフノードから date 型の値をフェッチ（取得）し、インデックスの圧縮表現から元のデータ形式へ変換します。

シグネチャまたは形状

Datum gbt_date_fetch(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体へのポインタを受け取ります。
  • **出力**: Datum 型として、フェッチされた日付データを返します。

前提条件と不変条件

  • btree_utils_num.h で定義された tinfo（gbtree_ninfo 構造体）が、date 型の処理に必要なメタデータとして利用可能である必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から GISTENTRY 構造体を取得します。 2. gbt_num_fetch 関数を呼び出し、tinfo を渡して数値型 GiST インデックス用の汎用的なフェッチ処理を実行します。 3. gbt_num_fetch の結果を PG_RETURN_POINTER を通じて呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックスマネージャ（具体的な呼び出し元コードは提供されていません）。
  • **呼び出し先**: gbt_num_fetch（btree_utils_num.c 内の関数）。

関連データ構造

  • GISTENTRY: GiST インデックスのノードエントリを保持する構造体。
  • tinfo: date 型の処理に必要な関数ポインタやメタデータを保持する gbtree_ninfo 構造体のインスタンス。

失敗モードと境界条件

  • GISTENTRY が無効な場合や、tinfo が正しく初期化されていない場合に予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.c が提供する汎用ロジックのラッパーです。date 型固有の複雑な処理は tinfo を介して gbt_num_fetch に委譲されています。
