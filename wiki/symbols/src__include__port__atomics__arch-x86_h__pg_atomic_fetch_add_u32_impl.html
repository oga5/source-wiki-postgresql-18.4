pg_atomic_fetch_add_u32_impl

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/port/atomics/arch-x86.h:182

目的

x86アーキテクチャにおいて、指定されたメモリ領域の値に対してアトミックに加算を行い、加算前の値を返す。

シグネチャまたは形状

uint32 pg_atomic_fetch_add_u32_impl(volatile pg_atomic_uint32 *ptr, int32 add_)

入力と出力

  • **入力**:
  • ptr: 操作対象となる pg_atomic_uint32 型のポインタ。
  • add_: 加算する値（int32）。
  • **出力**:
  • 加算操作が行われる直前の ptr->value の値を uint32 として返す。

前提条件と不変条件

  • ptr は有効なメモリ領域を指している必要がある。
  • x86の lock プレフィックス付き命令を使用するため、マルチコア環境においてもキャッシュコヒーレンシが保証される。

アルゴリズムまたは振る舞い

1. インラインアセンブラを用いて xaddl 命令を実行する。 2. lock プレフィックスを付与することで、命令の不可分性を保証する。 3. add_ の値を ptr->value に加算し、その結果を ptr->value に格納する。 4. 加算前の元の値を res レジスタに取得し、呼び出し元へ戻り値として返す。 5. コンパイラに対して memory バリアおよび cc（フラグレジスタ）の変更を通知する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/atomics.h および src/include/port/atomics/generic.h からの呼び出しが確認されている。
  • **呼び出し先**: 本関数はインラインアセンブラを直接実行するため、他の関数を呼び出さない。

関連データ構造

  • pg_atomic_uint32: アトミック操作対象となる32ビット整数を保持する構造体。

失敗モードと境界条件

  • 渡されたポインタがアライメントされていない場合、ハードウェア例外が発生する可能性がある。
  • add_ の値が int32 の範囲を超えて加算された場合のオーバーフロー挙動は、CPUの標準的な整数演算に従う。

保守担当者向けメモ

  • 本実装はx86専用である。他アーキテクチャへの移植時には、各アーキテクチャ固有のアトミック命令（例: ARMの LDREX/STREX 等）を使用する実装が必要となる。
  • volatile 修飾子により、コンパイラによる最適化でメモリ読み書きが省略されないよう制御されている。
