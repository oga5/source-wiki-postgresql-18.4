AutoVacuumWorkItemType

定義

  • 種別: enum
  • 定義箇所: src/include/postmaster/autovacuum.h:23

目的

自動バキュームシステムが処理するバックグラウンド作業の種類を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
	AVW_BRINSummarizeRange,
} AutoVacuumWorkItemType;

入力と出力

  • **入力**: 列挙型定数 AVW_BRINSummarizeRange を指定して作業を要求します。
  • **出力**: なし（列挙型定義のため）。

前提条件と不変条件

  • 現在定義されている値は AVW_BRINSummarizeRange のみです。
  • 将来的に他のバックグラウンド保守タスクへ拡張可能な設計となっています。

アルゴリズムまたは振る舞い

  • 他のプロセスから AutoVacuumRequestWork 関数を呼び出す際に、実行すべき作業の種類を特定するための識別子として機能します。
  • 共有メモリ上に登録された作業項目をワーカープロセスが読み取る際、この型に基づいて実行する処理を分岐させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/autovacuum.c 内の AutoVacuumRequestWork 関数が引数として受け取ります。
  • **呼び出し先**: 本シンボル自体は列挙型であるため、直接の呼び出し先はありません。

関連データ構造

  • AutoVacuumRequestWork: 本型を引数として受け取り、作業を登録する関数。

失敗モードと境界条件

  • 未定義の列挙値を指定した場合の動作は定義されていません。
  • 現在は AVW_BRINSummarizeRange 以外の値が渡された場合のハンドリングについては、ソースコードの文脈からは不明です。

保守担当者向けメモ

  • 本型は拡張可能です。新しい自動バキューム作業を追加する場合は、この列挙型に新しい定数を追加してください。
  • 実際の作業ロジックは src/backend/postmaster/autovacuum.c に実装されています。
