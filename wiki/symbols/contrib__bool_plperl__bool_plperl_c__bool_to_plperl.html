bool_to_plperl

定義

  • 種別: function
  • 定義箇所: contrib/bool_plperl/bool_plperl.c:14

目的

PostgreSQL の boolean 型の値を、PL/Perl で利用可能な Perl の真偽値オブジェクト（SV）へ変換します。

シグネチャまたは形状

Datum bool_to_plperl(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される PostgreSQL の bool 型引数。
  • **出力**: Datum 型として返される、Perl の真偽値定数（&PL_sv_yes または &PL_sv_no）へのポインタ。

前提条件と不変条件

  • dTHX マクロにより、現在のスレッドにおける Perl インタプリタコンテキストが正しく取得されている必要があります。
  • PostgreSQL の関数呼び出し規約（PG_FUNCTION_ARGS）に従う必要があります。

アルゴリズムまたは振る舞い

1. dTHX を呼び出し、Perl インタプリタのコンテキストを取得します。 2. PG_GETARG_BOOL(0) を使用して、PostgreSQL 側の引数を bool 型として取得します。 3. 三項演算子を用い、値が真であれば Perl のグローバル変数 PL_sv_yes を、偽であれば PL_sv_no を選択します。 4. 選択した Perl オブジェクトへのポインタを PointerGetDatum で Datum 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の PG_FUNCTION_INFO_V1(bool_to_plperl) マクロによる登録が確認されています。その他の具体的な呼び出し元は提供されたソースからは不明です。
  • **呼び出し先**:
  • dTHX (Perl マクロ)
  • PG_GETARG_BOOL (PostgreSQL マクロ)
  • PointerGetDatum (PostgreSQL マクロ)

関連データ構造

  • SV (Perl のスカラ値構造体)
  • Datum (PostgreSQL の汎用データ型)

失敗モードと境界条件

  • Perl インタプリタが初期化されていない環境や、コンテキストが不正な状態での呼び出しは未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数はマルチスレッド環境下での動作を前提としており、dTHX マクロの呼び出しが必須です。
  • Perl の真偽値定数（PL_sv_yes / PL_sv_no）を直接参照しているため、Perl インタプリタのライフサイクルに依存します。
