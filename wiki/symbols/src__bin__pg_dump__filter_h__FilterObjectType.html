FilterObjectType

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_dump/filter.h:47

目的

pg_dump および pg_restore において、フィルタファイル（除外・包含リスト）で指定されるデータベースオブジェクトの型を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
	FILTER_OBJECT_TYPE_NONE,
	FILTER_OBJECT_TYPE_TABLE_DATA,
	FILTER_OBJECT_TYPE_TABLE_DATA_AND_CHILDREN,
	FILTER_OBJECT_TYPE_DATABASE,
	FILTER_OBJECT_TYPE_EXTENSION,
	FILTER_OBJECT_TYPE_FOREIGN_DATA,
	FILTER_OBJECT_TYPE_FUNCTION,
	FILTER_OBJECT_TYPE_INDEX,
	FILTER_OBJECT_TYPE_SCHEMA,
	FILTER_OBJECT_TYPE_TABLE,
	FILTER_OBJECT_TYPE_TABLE_AND_CHILDREN,
	FILTER_OBJECT_TYPE_TRIGGER,
} FilterObjectType;

入力と出力

  • 入力: フィルタファイル内のキーワード文字列。
  • 出力: get_object_type 関数により、キーワードに対応する FilterObjectType 値が抽出されます。また、filter_object_type_name 関数により、列挙値から人間が読める文字列へ変換されます。

前提条件と不変条件

  • フィルタファイル内で新しいオブジェクト型を追加する場合、本列挙型と filter.c 内の get_object_type 関数を同時に更新する必要があります。

アルゴリズムまたは振る舞い

  • フィルタ解析処理において、対象オブジェクトの種別を特定するための識別子として機能します。
  • filter_read_item() を通じて、ファイルから読み取られた項目がどのオブジェクト型に該当するかを判定する際に利用されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照）:
  • src/bin/pg_dump/filter.c: get_object_type および filter_object_type_name 内で使用。
  • src/bin/pg_dump/pg_dump.c, pg_dumpall.c, pg_restore.c: 各アプリケーションのメイン処理で変数として宣言・利用。
  • 呼び出し先: 情報なし。

関連データ構造

  • FilterCommandType: フィルタ操作（INCLUDE/EXCLUDE）を定義する対となる列挙型。
  • FilterStateData: 解析状態を保持する構造体。

失敗モードと境界条件

  • 未定義または不明なオブジェクト型が指定された場合、get_object_type 関数がどのようにエラーをハンドリングするかは当該関数の実装に依存します。

保守担当者向けメモ

  • 本モジュールは pg_dump と pg_restore で共通利用されるため、変更時は両アプリケーションへの影響を考慮してください。
  • 列挙型の追加時は、対応する文字列変換関数 filter_object_type_name の更新を忘れないようにしてください。
