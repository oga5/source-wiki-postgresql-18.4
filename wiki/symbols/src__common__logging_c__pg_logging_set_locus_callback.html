pg_logging_set_locus_callback

定義

  • 種別: function
  • 定義箇所: src/common/logging.c:201

目的

ログ出力時に使用される、動的な位置情報（ファイル名および行番号）を取得するためのコールバック関数を登録します。

シグネチャまたは形状

void pg_logging_set_locus_callback(void (*cb) (const char **filename, uint64 *lineno))

入力と出力

  • **入力**:
  • cb: ログ出力時に呼び出される関数ポインタ。この関数は、出力されるメッセージに関連付けられたファイル名（const char **）と行番号（uint64 *）を書き込む必要があります。
  • **出力**: なし（void）。

前提条件と不変条件

  • 本関数は src/common/logging.c 内の静的変数 log_locus_callback を更新します。
  • 登録されたコールバックは、ログ出力処理（pg_log_generic_v 等）の内部で、位置情報が必要なタイミングで実行されます。

アルゴリズムまたは振る舞い

1. 引数として受け取った関数ポインタ cb を、モジュール内の静的変数 log_locus_callback に代入します。 2. 以降、ログ出力フレームワークが位置情報を必要とする際、この登録されたコールバックを通じてファイル名と行番号を取得します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/startup.c:135 で呼び出されていることが確認されています。
  • **呼び出し先**: 登録されたコールバック関数（cb）が、ログ出力処理の過程で間接的に呼び出されます。

関連データ構造

  • log_locus_callback: 本関数によって設定される、位置情報取得用コールバックを保持する静的変数。

失敗モードと境界条件

  • cb に NULL を渡した場合、log_locus_callback は NULL に設定されます。この場合、ログ出力時にコールバックが実行されない（または実行前にチェックされる）ことが想定されます。

保守担当者向けメモ

  • 本関数はフロントエンドプログラム専用です。バックエンドコードからは使用しないでください。
  • ログ出力時にファイル名や行番号を動的に追跡する必要があるツール（psql 等）でのみ使用されることを想定しています。
