exec_check_assignable

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_exec.c:8557

目的

指定されたデータム（datum）に対して代入が可能かどうかを検証します。

シグネチャまたは形状

static void exec_check_assignable(PLpgSQL_execstate *estate, int dno)

入力と出力

  • estate: 実行状態を保持する PLpgSQL_execstate 構造体へのポインタ。
  • dno: 検証対象のデータムを指すインデックス。
  • 戻り値: なし。代入不可の場合はエラーを報告し、処理を中断します。

前提条件と不変条件

  • dno は 0 以上 estate->ndatums 未満である必要があります（Assert により検証）。

アルゴリズムまたは振る舞い

1. estate->datums[dno] から対象のデータムを取得します。 2. データムの型（dtype）に応じて以下の処理を行います：

  • PLPGSQL_DTYPE_VAR, PLPGSQL_DTYPE_PROMISE, PLPGSQL_DTYPE_REC: 変数が isconst（定数）として宣言されている場合、ERRCODE_ERROR_IN_ASSIGNMENT エラーを発生させます。
  • PLPGSQL_DTYPE_ROW: 代入可能とみなします（メンバ変数の検証はコンパイル時に完了しているため）。
  • PLPGSQL_DTYPE_RECFIELD: 親レコードが代入可能かを確認するため、再帰的に exec_check_assignable を呼び出します。
  • 上記以外: 未知の型として elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpgsql/src/pl_exec.c 内の複数箇所（2373, 2969, 4727, 4803 行目など）。
  • 呼び出し先: exec_check_assignable（再帰呼び出し）。

関連データ構造

  • PLpgSQL_execstate: 実行状態を管理する構造体。
  • PLpgSQL_datum: 検証対象のデータム。
  • PLpgSQL_variable: 定数フラグ（isconst）を確認するために使用。
  • PLpgSQL_recfield: レコードフィールドの親レコードを参照するために使用。

失敗モードと境界条件

  • 定数（CONSTANT）として宣言された変数への代入を試みた場合、エラーが発生します。
  • 未知の dtype が渡された場合、内部エラーとして処理されます。
  • dno が範囲外の場合、アサーション違反となります。

保守担当者向けメモ

  • 本関数は代入操作の直前に呼び出されるガード関数です。
  • PLPGSQL_DTYPE_RECFIELD の再帰呼び出しにより、ネストされたレコード構造の代入可能性を正しく伝播させる設計となっています。
