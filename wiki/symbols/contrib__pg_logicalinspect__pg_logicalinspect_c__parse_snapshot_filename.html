parse_snapshot_filename

定義

  • 種別: function
  • 定義箇所: contrib/pg_logicalinspect/pg_logicalinspect.c:57

目的

論理デコーディングのスナップショットファイル名（%X-%X.snap 形式）を解析し、対応するLSN（Log Sequence Number）を抽出・検証する。

シグネチャまたは形状

static XLogRecPtr parse_snapshot_filename(const char *filename)

入力と出力

  • **入力**: filename (スナップショットファイル名を表す文字列)
  • **出力**: 解析に成功した場合は XLogRecPtr 型のLSN。失敗した場合は ereport(ERROR) により処理を中断する。

前提条件と不変条件

  • 入力文字列は %X-%X.snap という特定のフォーマットに従っている必要がある。
  • sscanf による解析後、生成された文字列と入力文字列を strcmp で比較し、フォーマットの厳密な一致を保証する。

アルゴリズムまたは振る舞い

1. sscanf を使用して、入力文字列から16進数の上位32ビット（hi）と下位32ビット（lo）を抽出する。 2. 抽出した値から sprintf を用いて一時的なファイル名文字列を再構築する。 3. 再構築した文字列と元の入力文字列を strcmp で比較し、形式が完全に一致するか検証する。 4. 一致する場合、hi と lo を結合して uint64 型のLSNを生成し、返す。 5. 形式が不正な場合は ereport(ERROR) を発行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_logicalinspect/pg_logicalinspect.c:115
  • contrib/pg_logicalinspect/pg_logicalinspect.c:150

  • strcmp (標準ライブラリ)
  • ereport (PostgreSQLエラー報告マクロ)

関連データ構造

  • XLogRecPtr: LSNを保持するための型。

失敗モードと境界条件

  • 入力文字列が %X-%X.snap 形式に適合しない場合、ereport(ERROR) によりエラーが発生する。
  • ファイル名のサフィックスが .snap でない場合や、数値部分が16進数として解釈できない場合もエラーとなる。

保守担当者向けメモ

  • 本関数はファイル名のフォーマットを厳密に検証する。将来的にスナップショットのファイル命名規則が変更された場合、本関数の sscanf および sprintf のフォーマット指定子を更新する必要がある。
