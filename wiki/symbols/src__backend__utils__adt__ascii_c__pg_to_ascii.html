pg_to_ascii

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ascii.c:28

目的

指定されたマルチバイトエンコーディング（LATIN1, LATIN2, LATIN9, WIN1250）のバイト列を、近似的なASCII表現に変換します。

シグネチャまたは形状

static void pg_to_ascii(unsigned char *src, unsigned char *src_end, unsigned char *dest, int enc)

入力と出力

  • **入力**:
  • src: 変換元バッファの開始ポインタ。
  • src_end: 変換元バッファの終了ポインタ。
  • dest: 変換先バッファのポインタ。
  • enc: エンコーディング識別子（PG_LATIN1, PG_LATIN2, PG_LATIN9, PG_WIN1250）。
  • **出力**: なし（dest が指すメモリ領域に変換結果を書き込む）。

前提条件と不変条件

  • dest は、入力文字列の長さを収容するのに十分なメモリ領域を確保している必要があります。
  • サポートされていないエンコーディングが指定された場合、関数はエラーを報告し、処理を中断します。

アルゴリズムまたは振る舞い

入力バイト列を走査し、以下のルールで変換を行います。 1. **128未満**: そのままコピーします。 2. **128以上かつ range 未満**: スペース（' '）に置換します。

  • range はエンコーディングにより128または160となります。

    • 3. **range 以上**: ハードコードされた変換テーブル ascii を参照し、対応するASCII文字に変換します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/ascii.c 内のSQLインターフェース関数（to_ascii 等）から呼び出されます。
  • **呼び出し先**: ereport（エラー発生時）、pg_encoding_to_char（エラーメッセージ生成時）。

関連データ構造

  • ascii (内部変換テーブル): 各エンコーディングに対応する文字マッピングを保持する unsigned char 配列。

失敗モードと境界条件

  • **サポート外のエンコーディング**: enc が定義済みの4種以外の場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラーを発生させます。
  • **バッファオーバーフロー**: dest のサイズが不足している場合、メモリ破壊が発生する可能性があります（呼び出し元で適切なサイズ確保が求められます）。

保守担当者向けメモ

  • 本関数はインプレース変換を想定した設計ではありません。
  • 変換テーブルは固定値であり、動的なエンコーディング追加には対応していません。
  • RANGE_128 (128) および RANGE_160 (160) は、各エンコーディングの変換開始オフセットを定義する定数です。
