PublicationRelInfo

定義

  • 種別: struct
  • 定義箇所: src/include/catalog/pg_publication.h:137

目的

論理レプリケーションにおいて、特定のパブリケーションに関連付けられたリレーション（テーブル）の構成情報を保持するために使用されるメモリ内データ構造です。

シグネチャまたは形状

typedef struct PublicationRelInfo
{
	Relation	relation;
	Node	   *whereClause;
	List	   *columns;
} PublicationRelInfo;

入力と出力

  • **入力**: パブリケーションへのリレーション追加時や、既存のパブリケーション定義の更新時に、リレーション情報、行フィルタリング条件（whereClause）、および列リスト（columns）が格納されます。
  • **出力**: レプリケーションロジックにおいて、どのテーブルをどの条件でパブリッシュすべきかを判定するためのメタデータとして利用されます。

前提条件と不変条件

  • relation は有効なリレーション記述子を指している必要があります。
  • whereClause は行フィルタリングのための式ツリー、またはフィルタリングがない場合は NULL です。
  • columns はパブリッシュ対象の列リスト、または全列対象の場合は NIL です。

アルゴリズムまたは振る舞い

パブリケーションに対するリレーションの追加や変更処理において、カタログ情報から読み取られた情報を一時的に保持します。この構造体を通じて、パブリケーションごとの列フィルタリングや行フィルタリングの適用状況が管理されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_publication.c: publication_add_relation 関数にて使用。
  • src/backend/commands/publicationcmds.c: パブリケーションの作成・変更コマンド（ALTER PUBLICATION 等）の処理過程で使用。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • Relation: リレーションの物理的なメタデータを保持する構造体。
  • Node: whereClause を表現するための汎用ノード。
  • List: columns を保持するためのリスト構造。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性があります。
  • relation が無効な場合や、whereClause に不正な式が含まれる場合、後続のレプリケーション処理でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は pg_publication_rel カタログと密接に関連しています。列リストや行フィルタリングの仕様変更を行う際は、本構造体のフィールド定義と、それらを処理する src/backend/commands/publicationcmds.c 内のロジックを併せて確認してください。
