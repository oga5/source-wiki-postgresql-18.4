in_range_int4_int4

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/int.c:651

目的

ウィンドウ関数のフレーム境界計算において、int4 型の値が指定された範囲内にあるかどうかを判定します。

シグネチャまたは形状

Datum in_range_int4_int4(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • val (int32): 比較対象の値。
  • base (int32): 範囲の基準値。
  • offset (int32): 基準値からのオフセット量。
  • sub (bool): true の場合、オフセットを減算します。
  • less (bool): true の場合、val <= (base ± offset) を判定します。
  • **出力**: bool (Datum形式): 判定結果。

前提条件と不変条件

  • offset が負の値である場合、ERRCODE_INVALID_PRECEDING_OR_FOLLOWING_SIZE エラーが発生します。
  • 算術演算においてオーバーフローが発生した場合、その結果は「無限大」または「負の無限大」として扱われ、less フラグに基づいた論理判定が行われます。

アルゴリズムまたは振る舞い

1. offset が負であればエラーを報告します。 2. sub が true の場合、offset を反転させます。 3. base と offset の加算を行い、pg_add_s32_overflow でオーバーフローをチェックします。 4. オーバーフローが発生した場合:

  • sub が false ならば、合計値は val より大きいとみなし、less の値をそのまま返します。
  • sub が true ならば、合計値は val より小さいとみなし、!less を返します。

    • 5. オーバーフローしない場合:

  • less が true ならば val <= sum を、false ならば val >= sum を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/int.c 内で直接参照されています。
  • **呼び出し先**: pg_add_s32_overflow (インライン関数) を呼び出します。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約に従う引数構造体。

失敗モードと境界条件

  • offset < 0: ereport によるエラー終了。
  • 整数オーバーフロー: pg_add_s32_overflow により安全に処理され、論理的な境界値として判定されます。

保守担当者向けメモ

  • 本関数は int4 型専用です。他の整数型については、対応する in_range_intX_intY 関数を参照してください。
  • オーバーフロー時の挙動はウィンドウ関数の仕様に依存しているため、変更時は関連するSQL標準の仕様を確認してください。
