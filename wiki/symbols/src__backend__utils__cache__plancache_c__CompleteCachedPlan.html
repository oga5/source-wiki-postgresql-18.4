CompleteCachedPlan

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/plancache.c:390

目的

CreateCachedPlan で作成された CachedPlanSource に対し、解析・書き換え済みのクエリツリーを格納し、キャッシュエントリを完成させる。

シグネチャまたは形状

void CompleteCachedPlan(CachedPlanSource *plansource,
                        List *querytree_list,
                        MemoryContext querytree_context,
                        Oid *param_types,
                        int num_params,
                        ParserSetupHook parserSetup,
                        void *parserSetupArg,
                        int cursor_options,
                        bool fixed_result)

入力と出力

  • **入力**:
  • plansource: 対象となる CachedPlanSource 構造体。
  • querytree_list: 解析・書き換え済みのクエリツリーリスト。
  • querytree_context: クエリツリーを保持するメモリコンテキスト（NULL の場合は新規作成）。
  • param_types, num_params: パラメータの型情報と個数。
  • parserSetup, parserSetupArg: パーサー設定用フック。
  • cursor_options: カーソルオプション。
  • fixed_result: 結果のタプル記述子が固定かどうかを示すフラグ。
  • **出力**: なし（plansource 構造体の内部状態を更新）。

前提条件と不変条件

  • plansource->magic が CACHEDPLANSOURCE_MAGIC であること。
  • plansource->is_complete が false であること（二重完了の禁止）。
  • ワンショット計画（is_oneshot）の場合、querytree_list は呼び出し元で十分に生存期間が確保されているとみなされる。

アルゴリズムまたは振る舞い

1. **メモリ管理**: querytree_context を plansource->context の子として再配置、または新規作成して querytree_list をコピーする（ワンショット計画を除く）。 2. **依存関係の抽出**: ワンショット計画ではなく、かつ再検証が必要な場合、extract_query_dependencies を呼び出してリレーション OID、無効化項目、RLS 依存情報を抽出する。 3. **メタデータ保存**: 現在の search_path、ユーザー ID、RLS 設定を保存する。 4. **パラメータと属性の保存**: パラメータ型配列を plansource->context にコピーし、その他の引数および結果のタプル記述子（PlanCacheComputeResultDesc で算出）を格納する。 5. **状態更新**: is_complete および is_valid を true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/prepare.c
  • src/backend/executor/functions.c
  • src/backend/executor/spi.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • MemoryContextSetParent, MemoryContextSwitchTo, AllocSetContextCreate
  • copyObject
  • StmtPlanRequiresRevalidation, extract_query_dependencies
  • GetUserId, GetSearchPathMatcher
  • PlanCacheComputeResultDesc

関連データ構造

  • CachedPlanSource: キャッシュのソース情報を保持する主要構造体。

失敗モードと境界条件

  • is_oneshot が true の場合、メモリコピーをスキップし、呼び出し元のメモリコンテキストをそのまま利用する。
  • トランザクション制御コマンドなど、再検証が不要な場合は依存関係の抽出をスキップする。

保守担当者向けメモ

  • 本関数は CreateCachedPlan の直後に呼び出されることを想定している。
  • querytree_context の管理はメモリリークを防ぐため重要であり、plansource のライフサイクルと同期させる必要がある。
