varchar_1

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-array_of_struct.c:35

目的

ECPG（Embedded SQL for C）において、可変長文字列（VARCHAR）を表現するためのデータ構造。

シグネチャまたは形状

struct varchar_1 {
    int len;
    char arr[50];
};

入力と出力

  • 入力: SQL の文字列データを受け取り、len に長さ、arr に文字列本体を格納する。
  • 出力: ECPG ライブラリがデータベースとの通信時にこの構造体を参照し、SQL の VARCHAR 型と相互変換を行う。

前提条件と不変条件

  • arr のサイズは 50 バイト固定である。
  • len は arr に格納された有効な文字列の長さを保持する。

アルゴリズムまたは振る舞い

ECPG プリプロセッサによって生成されるコードにおいて、ホスト変数として使用される。SQL 文の実行時に、データベースの VARCHAR 型と C 言語の構造体間でデータをマッピングする際の型定義として機能する。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/test/expected/preproc-cursor.c 内の複数の箇所で、ECPGt_varchar 型としてこの構造体のサイズ（sizeof(struct varchar_1)）が参照されている。
  • 呼び出し先: データベースとの通信を行う ECPG ライブラリ関数（ECPGdo 等）が、この構造体のメモリレイアウトを介してデータ転送を行う。

関連データ構造

  • varchar_2, varchar_3, varchar_4, varchar_5: 同様の目的で定義された他の可変長文字列構造体。

失敗モードと境界条件

  • arr のサイズ（50 バイト）を超える文字列を格納しようとした場合、バッファオーバーフローや切り捨てが発生する可能性がある。
  • len が 50 を超える値に設定された場合、ECPG ライブラリの期待する動作と矛盾し、未定義の挙動を引き起こす可能性がある。

保守担当者向けメモ

  • 本構造体は ECPG プリプロセッサが自動生成するコードの一部であるため、手動での変更は推奨されない。
  • preproc-cursor.c 等のテストコードから参照されているため、定義を変更する場合は関連するテストスイートへの影響を確認すること。
