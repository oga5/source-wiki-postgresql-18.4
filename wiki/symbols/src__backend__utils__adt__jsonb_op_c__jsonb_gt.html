jsonb_gt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_op.c:179

目的

2つの jsonb 型の値を受け取り、左側の値が右側の値よりも大きいかどうかを判定する比較演算子（>）の実装。

シグネチャまたは形状

Datum jsonb_gt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの Jsonb 構造体へのポインタを受け取る。
  • **出力**: bool 型の Datum。左側の値が右側の値より大きい場合は true、それ以外は false を返す。

前提条件と不変条件

  • 入力された Jsonb ポインタは、PG_GETARG_JSONB_P マクロにより適切に取得される必要がある。
  • 比較処理終了後、PG_FREE_IF_COPY を用いてメモリの解放（必要な場合）を行う必要がある。

アルゴリズムまたは振る舞い

1. PG_GETARG_JSONB_P を使用して、引数から2つの Jsonb オブジェクト（jba, jbb）を取得する。 2. compareJsonbContainers 関数を呼び出し、両者の root 構造を比較する。 3. 比較結果が 0 より大きい（> 0）場合に true を、そうでない場合に false を res に格納する。 4. PG_FREE_IF_COPY で一時的なコピーを解放する。 5. PG_RETURN_BOOL を使用して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（fmgr）を介してSQLから呼び出される。ソースコード上の直接的な呼び出し元情報は提供されていない。
  • **呼び出し先**: compareJsonbContainers（src/backend/utils/adt/jsonb_util.c で定義されていると推測される補助関数）。

関連データ構造

  • Jsonb: jsonb 型の内部表現。
  • JsonbContainer: Jsonb 構造体内の root フィールドとして保持されるデータ構造。

失敗モードと境界条件

  • 入力が無効な jsonb 構造である場合、compareJsonbContainers の挙動に依存する。
  • メモリ不足等のシステムエラーが発生した場合、PostgreSQLの標準的なエラーハンドリングにより中断される。

保守担当者向けメモ

  • 本関数はB-Treeインデックスのソート順を決定する比較ロジックの一部である。
  • 比較ロジックの変更が必要な場合は、compareJsonbContainers の実装を確認すること。
