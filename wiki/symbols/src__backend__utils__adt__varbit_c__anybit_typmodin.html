anybit_typmodin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varbit.c:89

目的

BIT および BIT VARYING 型の型修飾子（typmod）を解析・検証し、内部表現として使用可能な整数値に変換します。

シグネチャまたは形状

int32 anybit_typmodin(ArrayType *ta, const char *typename)

入力と出力

  • **入力**:
  • ta: 型修飾子を含む配列型（ArrayType）。
  • typename: エラーメッセージに使用する型名文字列。
  • **出力**:
  • 成功時: 検証済みのビット長（int32）。
  • 失敗時: エラーをスローし、処理を中断。

前提条件と不変条件

  • ta から取得される修飾子の数は1つである必要があります。
  • ビット長は1以上である必要があります。
  • ビット長は MaxAttrSize * BITS_PER_BYTE を超えてはなりません。

アルゴリズムまたは振る舞い

1. ArrayGetIntegerTypmods を呼び出し、配列から修飾子リストを取得します。 2. 修飾子の数が1つでない場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告します。 3. 修飾子値が1未満、または最大許容ビット長を超えている場合、それぞれ ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告します。 4. 検証を通過した値を int32 として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/varbit.c:433 (bit 型の型修飾子処理)
  • src/backend/utils/adt/varbit.c:778 (varbit 型の型修飾子処理)
  • **呼び出し先**:
  • ArrayGetIntegerTypmods (配列からの修飾子抽出)
  • ereport (エラー報告)

関連データ構造

  • ArrayType: 型修飾子を保持する配列構造体。

失敗モードと境界条件

  • **修飾子の数が不正**: n != 1 の場合にエラー。
  • **長さが範囲外**: 1未満、または MaxAttrSize * BITS_PER_BYTE を超える場合にエラー。

保守担当者向けメモ

  • 本関数は bit と varbit の両方で共通の検証ロジックとして使用されています。
  • エラーメッセージの文言は、呼び出し元から渡される typename に依存します。
