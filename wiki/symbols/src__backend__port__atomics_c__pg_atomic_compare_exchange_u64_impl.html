pg_atomic_compare_exchange_u64_impl

定義

  • 種別: function
  • 定義箇所: src/backend/port/atomics.c:33

目的

ハードウェアによるネイティブな 64 ビットアトミック操作がサポートされていない環境において、スピンロックを用いて「強い（strong）」比較交換（CAS）操作をシミュレートする。

シグネチャまたは形状

bool pg_atomic_compare_exchange_u64_impl(volatile pg_atomic_uint64 *ptr, uint64 *expected, uint64 newval)

入力と出力

  • ptr: 操作対象となる 64 ビットアトミック変数のポインタ。
  • expected: 比較対象の値へのポインタ。成功時はそのまま、失敗時は現在の値で更新される。
  • newval: 比較が成功した場合に書き込まれる新しい値。
  • 戻り値: 比較が成功し、値が更新された場合は true、失敗した場合は false を返す。

前提条件と不変条件

  • ptr が指す構造体は、事前に pg_atomic_init_u64_impl 等でスピンロックが初期化されている必要がある。
  • スピンロックによる排他制御を行うため、呼び出し元はスピンロックの再帰的取得によるデッドロックを避ける必要がある。

アルゴリズムまたは振る舞い

1. ptr->sema に格納されたスピンロックを SpinLockAcquire で取得する。 2. ptr->value と *expected を比較する。 3. 一致する場合:

  • ptr->value に newval を代入する。
  • 戻り値を true に設定する。

    • 4. 一致しない場合:

  • *expected を現在の ptr->value で更新する。
  • 戻り値を false に設定する。

    • 5. SpinLockRelease でスピンロックを解放し、結果を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/port/atomics/generic.h 内の汎用アトミック操作関数群から呼び出される。
  • 呼び出し先: SpinLockAcquire および SpinLockRelease（src/include/storage/spin.h）。

関連データ構造

  • pg_atomic_uint64: 64 ビット値とスピンロック（sema）を保持する構造体。

失敗モードと境界条件

  • スピンロックの競合が激しい場合、パフォーマンスが低下する可能性がある。
  • 本関数は「強い」CAS を提供するため、偽の失敗（spurious failure）は発生しない。

保守担当者向けメモ

  • 本関数は PG_HAVE_ATOMIC_U64_SIMULATION が定義されている環境でのみ使用される。
  • パフォーマンスが極めて重要なコードパスでは、ネイティブなアトミック命令が利用可能かどうかを確認すること。
