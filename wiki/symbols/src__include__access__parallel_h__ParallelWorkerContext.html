ParallelWorkerContext

定義

  • 種別: struct
  • 定義箇所: src/include/access/parallel.h:50

目的

並列クエリ実行において、バックグラウンドワーカーが共有メモリリソースへアクセスするためのコンテキストを保持します。

シグネチャまたは形状

typedef struct ParallelWorkerContext
{
    dsm_segment *seg;
    shm_toc    *toc;
} ParallelWorkerContext;

入力と出力

  • **入力**: 実行ノード（PlanState 等）の初期化時に、リーダープロセスからワーカーへ渡される共有メモリセグメントおよびテーブル・オブ・コンテント（TOC）へのポインタ。
  • **出力**: なし（構造体自体が共有リソースへのアクセス経路を提供）。

前提条件と不変条件

  • seg は有効な動的共有メモリ（DSM）セグメントを指している必要があります。
  • toc は seg 内に配置された有効な共有メモリテーブル・オブ・コンテントを指している必要があります。

アルゴリズムまたは振る舞い

ワーカープロセスは、この構造体を通じてリーダーが準備した共有メモリ領域（seg）およびデータ構造の目次（toc）にアタッチし、並列実行に必要な共有データへアクセスします。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • ExecParallelInitializeWorker (src/backend/executor/execParallel.c)
  • ExecAggInitializeWorker (src/backend/executor/nodeAgg.c)
  • ExecAppendInitializeWorker (src/backend/executor/nodeAppend.c)
  • ExecHashInitializeWorker (src/backend/executor/nodeHash.c)
  • ExecIncrementalSortInitializeWorker (src/backend/executor/nodeIncrementalSort.c)
  • その他、nodeBitmapHeapscan.c, nodeBitmapIndexscan.c, nodeCustom.c, nodeForeignscan.c, nodeHashjoin.c, nodeIndexonlyscan.c 等の各実行ノード。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • dsm_segment: 動的共有メモリセグメント。
  • shm_toc: 共有メモリ内の動的メモリ管理用テーブル・オブ・コンテント。

失敗モードと境界条件

  • 共有メモリへのアタッチに失敗した場合、ワーカーは適切にエラーを報告する必要があります。
  • 構造体内のポインタが NULL の場合、アクセス時にセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体はバックエンドの並列クエリ実行専用です。src/bin/pg_dump/parallel.c で使用される並列処理とは無関係であるため、混同しないよう注意してください。
  • 実行ノードの初期化フローにおいて、リーダーからワーカーへこのコンテキストが正しく伝播されているかを確認してください。
