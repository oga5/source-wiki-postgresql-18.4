get_parallel_tables_list

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/reindexdb.c:628

目的

並列再構築処理（REINDEX）の対象となるテーブルのリストを、システムカタログから取得して構築する。

シグネチャまたは形状

SimpleStringList *get_parallel_tables_list(PGconn *conn, ReindexType type, SimpleStringList *user_list, bool echo);

入力と出力

  • **入力**:
  • conn: データベース接続ハンドル。
  • type: 再構築の種類（REINDEX_DATABASE または REINDEX_SCHEMA）。
  • user_list: スキーマ名リスト（REINDEX_SCHEMA の場合に使用）。
  • echo: クエリ実行時にログを出力するかどうかのフラグ。
  • **出力**:
  • 対象テーブルの修飾名（schema.table）を格納した SimpleStringList 構造体。対象がない場合は NULL を返す。

前提条件と不変条件

  • type が REINDEX_DATABASE の場合、user_list は NULL であること。
  • type が REINDEX_SCHEMA の場合、user_list は有効なリストであること。
  • REINDEX_INDEX、REINDEX_SYSTEM、REINDEX_TABLE が指定された場合はアサーションエラーとなる。
  • 取得されるテーブルは、一時テーブル（relpersistence != 't'）を除外する。

アルゴリズムまたは振る舞い

1. type に応じてシステムカタログ（pg_class, pg_namespace）からテーブル名とスキーマ名を取得するクエリを構築する。 2. REINDEX_DATABASE の場合は pg_catalog 以外の全テーブルを、REINDEX_SCHEMA の場合は指定されたスキーマ内のテーブルを対象とする。 3. クエリ結果は relpages の降順でソートされ、大きなテーブルから順に処理されるよう設計されている。 4. 取得した結果セットを反復処理し、fmtQualifiedIdEnc を用いて各テーブルの修飾名を生成し、SimpleStringList に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/scripts/reindexdb.c 内の reindex_one_database 関数から呼び出される。
  • **呼び出し先**: initPQExpBuffer, appendPQExpBufferStr, executeQuery, termPQExpBuffer, PQntuples, PQgetvalue, simple_string_list_append, fmtQualifiedIdEnc 等のユーティリティ関数。

関連データ構造

  • SimpleStringList: テーブル名のリストを保持するために使用。
  • PGconn: データベース接続状態。
  • ReindexType: 再構築対象の種別を定義する列挙型。

失敗モードと境界条件

  • 該当するテーブルが存在しない場合、NULL を返す。
  • データベース接続エラーやクエリ実行失敗時は、executeQuery 内部でエラーハンドリングが行われる（通常はプログラム終了）。

保守担当者向けメモ

  • 本関数は relpages によるソートを行っているため、並列実行時の負荷分散に寄与している。
  • 新しい relkind（例: パーティションテーブル等）を対象に含める場合は、クエリの c.relkind IN (...) 句を更新する必要がある。
