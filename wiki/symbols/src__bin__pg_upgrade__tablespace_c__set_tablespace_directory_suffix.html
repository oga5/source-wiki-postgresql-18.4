set_tablespace_directory_suffix

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/tablespace.c:102

目的

PostgreSQLのアップグレードプロセスにおいて、各クラスタのバージョンおよびカタログバージョンに基づいた、テーブルスペース内のサブディレクトリパスを生成・設定します。

シグネチャまたは形状

static void set_tablespace_directory_suffix(ClusterInfo *cluster);

入力と出力

  • **入力**: ClusterInfo *cluster（対象となるクラスタ情報構造体へのポインタ）
  • **出力**: なし（副作用として cluster->tablespace_suffix を更新）

前提条件と不変条件

  • 引数 cluster が指す構造体内の major_version_str および controldata.cat_ver が初期化済みであること。
  • psprintf を使用してメモリを動的に確保するため、呼び出し元で適切なメモリ管理が行われること。

アルゴリズムまたは振る舞い

1. cluster->major_version_str（メジャーバージョン文字列）と cluster->controldata.cat_ver（カタログバージョン番号）を取得します。 2. psprintf を使用し、/PG_<major_version>_<cat_ver> という形式の文字列を生成します。 3. 生成された文字列を cluster->tablespace_suffix に格納します。この先頭の / は、ディレクトリパスを連結する際の区切り文字として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/tablespace.c 内の初期化処理において、old_cluster および new_cluster に対してそれぞれ呼び出されます。
  • **呼び出し先**: psprintf（PostgreSQLのメモリ管理付き文字列フォーマット関数）。

関連データ構造

  • ClusterInfo: pg_upgrade で使用されるクラスタ情報を保持する構造体。tablespace_suffix フィールドを介して本関数と連携します。

失敗モードと境界条件

  • psprintf 内部でメモリ不足が発生した場合、プログラムは終了する可能性があります。
  • major_version_str や cat_ver が正しく設定されていない場合、不正なパス文字列が生成されます。

保守担当者向けメモ

  • 本関数はテーブルスペースの物理パスを構築するための補助的なロジックです。
  • 生成されるサフィックスは、PostgreSQLのバージョンアップグレードにおけるデータディレクトリの互換性維持に不可欠です。変更時は、新旧クラスタ間でのパス整合性に影響がないか注意してください。
