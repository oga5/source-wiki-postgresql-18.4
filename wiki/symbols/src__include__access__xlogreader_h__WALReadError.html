WALReadError

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlogreader.h:382

目的

WAL（Write-Ahead Log）の読み込み操作中に発生したエラーに関する詳細情報を保持し、エラーハンドリングや診断を支援するために使用されます。

シグネチャまたは形状

typedef struct WALReadError
{
	int			wre_errno;		/* errno set by the last pg_pread() */
	int			wre_off;		/* Offset we tried to read from. */
	int			wre_req;		/* Bytes requested to be read. */
	int			wre_read;		/* Bytes read by the last read(). */
	WALOpenSegment wre_seg;		/* Segment we tried to read from. */
} WALReadError;

入力と出力

本構造体は関数引数として渡され、エラー発生時に呼び出し元へ詳細情報を伝達する出力コンテナとして機能します。

前提条件と不変条件

  • wre_errno は pg_pread() 等のシステムコールが設定したエラーコードを反映している必要があります。
  • wre_read は実際に読み込まれたバイト数を示し、要求された wre_req と比較することで読み込み不足を判定可能です。

アルゴリズムまたは振る舞い

WAL 読み込み処理においてエラーが発生した際、関連するコンテキスト情報（オフセット、要求サイズ、セグメント情報）を本構造体に格納し、上位レイヤーへ通知します。特に WALReadRaiseError 関数を通じてエラーを報告する際に利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な言及）**:
  • src/backend/access/transam/xlogreader.c
  • src/backend/access/transam/xlogutils.c
  • src/backend/postmaster/walsummarizer.c
  • src/backend/replication/walsender.c
  • src/bin/pg_waldump/pg_waldump.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。ただし、WALReadRaiseError 関数が本構造体を引数として受け取り、エラー処理を実行します。

関連データ構造

  • WALOpenSegment: エラーが発生した対象の WAL セグメント情報を保持します。

失敗モードと境界条件

  • wre_read が wre_req より小さい場合、部分的な読み込み失敗または EOF 到達を示唆します。
  • wre_errno が 0 の場合、システムコールレベルでのエラーは発生していない可能性があります。

保守担当者向けメモ

  • 本構造体はバックエンドおよびフロントエンド（pg_waldump）の両方で使用されるため、変更時は互換性に注意してください。
  • エラー発生時の診断情報を拡充する必要がある場合は、フィールドの追加を検討してください。
