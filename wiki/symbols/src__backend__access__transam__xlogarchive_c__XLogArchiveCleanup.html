XLogArchiveCleanup

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogarchive.c:711

目的

指定されたWALセグメントに関連するアーカイブ状態ファイル（.done および .ready）を削除し、アーカイブ管理システムから当該セグメントの追跡情報を除去します。

シグネチャまたは形状

void XLogArchiveCleanup(const char *xlog)

入力と出力

  • **入力**: xlog (対象となるWALセグメントのファイル名)
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • xlog 引数は有効なWALセグメント名である必要があります。
  • pg_wal/archive_status ディレクトリ内に対応する状態ファイルが存在する場合、それらは削除されます。

アルゴリズムまたは振る舞い

1. StatusFilePath マクロを使用して、対象WALセグメントの .done ファイルのパスを構築します。 2. unlink() を呼び出し、.done ファイルを削除します。 3. 同様に、対象WALセグメントの .ready ファイルのパスを構築します。 4. unlink() を呼び出し、.ready ファイルを削除します。 5. 各 unlink 操作の失敗については、現在の実装では明示的なエラー報告を行いません。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c: WALの削除やリサイクル処理の過程で、不要になったセグメントのアーカイブ状態をクリーンアップするために呼び出されます。
  • **呼び出し先**:
  • unlink() (標準ライブラリ関数)
  • StatusFilePath() (内部マクロ)

関連データ構造

  • pg_wal/archive_status/ 配下のファイル（.ready および .done サフィックスを持つファイル）が本関数の直接的な操作対象です。

失敗モードと境界条件

  • **ファイル不在**: unlink 対象のファイルが存在しない場合、unlink は失敗しますが、本関数はそのエラーを無視して処理を継続します。
  • **権限不足**: プロセスが pg_wal/archive_status ディレクトリ内のファイルを削除する権限を持たない場合、削除は失敗します。

保守担当者向けメモ

  • 本関数は unlink の戻り値をチェックしていません。アーカイブ状態ファイルの削除失敗がシステム全体の整合性に与える影響については、呼び出し元の文脈（xlog.c）で考慮する必要があります。
  • .ready ファイルは通常、アーカイブプロセスが処理を開始する前に存在しますが、本関数はクリーンアップを目的としているため、存在する場合は強制的に削除します。
