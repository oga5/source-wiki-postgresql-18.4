DatumGetEOHP

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/expandeddatum.c:28

目的

Datum 形式で渡された外部展開済みデータ（Expanded Object）から、対応する ExpandedObjectHeader 構造体へのポインタを安全に抽出します。

シグネチャまたは形状

ExpandedObjectHeader *DatumGetEOHP(Datum d)

入力と出力

  • **入力**: Datum d （展開済みデータ型を指す TOAST ポインタを含む Datum）
  • **出力**: ExpandedObjectHeader * （抽出された展開済みオブジェクトのヘッダーへのポインタ）

前提条件と不変条件

  • 入力 d は、VARATT_IS_EXTERNAL_EXPANDED マクロにより、外部展開済みデータであることが保証されていなければなりません（内部で Assert されます）。
  • 抽出されたポインタは、VARATT_IS_EXPANDED_HEADER マクロにより、有効な展開済みヘッダーであることが保証されます。

アルゴリズムまたは振る舞い

1. Datum を varattrib_1b_e 型へキャストし、外部展開済みデータであることを検証します。 2. VARDATA_EXTERNAL を介してデータ領域から varatt_expanded 構造体をコピーします。この際、アライメントが保証されない可能性があるため memcpy を使用します。 3. コピーされた構造体内の eohptr が有効なヘッダーであることを検証します。 4. 取得した ExpandedObjectHeader へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/detoast.c
  • src/backend/access/common/heaptuple.c
  • src/backend/executor/execExprInterp.c
  • src/backend/executor/execTuples.c
  • src/backend/executor/nodeWindowAgg.c
  • src/backend/utils/adt/array_expanded.c
  • **呼び出し先**:
  • DatumGetPointer (マクロ)
  • VARATT_IS_EXTERNAL_EXPANDED (マクロ)
  • VARDATA_EXTERNAL (マクロ)
  • memcpy (標準ライブラリ)
  • VARATT_IS_EXPANDED_HEADER (マクロ)

関連データ構造

  • ExpandedObjectHeader: 展開済みオブジェクトの共通ヘッダー。
  • varattrib_1b_e: 外部展開済みデータ用の可変長属性構造体。
  • varatt_expanded: 外部展開済みデータ内に格納されるポインタ構造体。

失敗モードと境界条件

  • 入力が展開済みデータでない場合、Assert によりアボートします（デバッグビルド時）。
  • 抽出されたポインタが不正なヘッダーを指している場合、Assert によりアボートします。

保守担当者向けメモ

  • memcpy を使用しているのは、TOAST ポインタがメモリ上で適切にアライメントされていない可能性があるためです。この実装は、データ構造のバイナリレイアウトに強く依存しています。
