ReinitializeParallelDSM

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/transam/parallel.c:507

目的

並列クエリ実行コンテキスト（ParallelContext）を再利用可能な状態に初期化します。主に、以前の並列実行で使用したワーカーのリソースを解放し、共有メモリ上のエラーキューを再構築するために使用されます。

シグネチャまたは形状

void ReinitializeParallelDSM(ParallelContext *pcxt)

入力と出力

  • **入力**: ParallelContext *pcxt - 再初期化対象の並列コンテキスト。
  • **出力**: なし（副作用として pcxt が指す共有メモリ領域および構造体が更新されます）。

前提条件と不変条件

  • pcxt は有効な ParallelContext を指している必要があります。
  • TopTransactionContext が有効である必要があります（関数内で一時的にメモリコンテキストを切り替えるため）。

アルゴリズムまたは振る舞い

1. **メモリコンテキストの切り替え**: TopTransactionContext へ一時的に切り替えます。 2. **ワーカーの終了待機**: nworkers_launched > 0 の場合、WaitForParallelWorkersToFinish および WaitForParallelWorkersToExit を呼び出し、既存のワーカーが完全に終了するまで待機します。その後、関連する追跡用メモリ（known_attached_workers）を解放します。 3. **固定状態のクリア**: 共有メモリ上の FixedParallelState を検索し、last_xlog_end を 0 にリセットします。 4. **エラーキューの再作成**: nworkers > 0 の場合、共有メモリ上のエラーキュー領域を再スキャンし、shm_mq_create を用いて各ワーカー用のメッセージキューを再初期化します。各キューは MyProc を受信者として設定し、shm_mq_attach で再接続します。 5. **コンテキスト復元**: 元のメモリコンテキストに戻します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c
  • src/backend/commands/vacuumparallel.c
  • src/backend/executor/execParallel.c
  • **呼び出し先**:
  • WaitForParallelWorkersToFinish
  • WaitForParallelWorkersToExit
  • shm_toc_lookup
  • shm_mq_create
  • shm_mq_set_receiver
  • shm_mq_attach
  • MemoryContextSwitchTo

関連データ構造

  • ParallelContext: 並列実行のライフサイクルを管理する構造体。
  • FixedParallelState: 共有メモリ上の固定サイズ状態。
  • shm_mq: 共有メモリベースのメッセージキュー。

失敗モードと境界条件

  • shm_toc_lookup が指定されたキー（PARALLEL_KEY_FIXED または PARALLEL_KEY_ERROR_QUEUE）を見つけられない場合、エラーが発生します。
  • pcxt が適切に初期化されていない場合、セグメンテーション違反やメモリ破壊の可能性があります。

保守担当者向けメモ

  • 本関数は、並列クエリを繰り返し実行する際（例：パラレルVACUUMやパラレルスキャン）のオーバーヘッドを削減するために重要です。
  • エラーキューの再作成ロジックは、ParallelContext のワーカー数設定と共有メモリ上のレイアウトに強く依存しているため、構造を変更する際は注意してください。
