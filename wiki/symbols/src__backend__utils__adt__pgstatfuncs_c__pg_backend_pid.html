pg_backend_pid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pgstatfuncs.c:689

目的

現在のバックエンドプロセス（セッション）のプロセスID（PID）をSQLインターフェースを通じて取得します。

シグネチャまたは形状

Datum pg_backend_pid(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（SQL関数呼び出し規約に基づく引数）
  • 出力: int32 型のプロセスIDを Datum として返却します。

前提条件と不変条件

  • 呼び出し元のバックエンドプロセスが有効である必要があります。
  • グローバル変数 MyProcPid が現在のプロセスIDを保持していることを前提としています。

アルゴリズムまたは振る舞い

1. グローバル変数 MyProcPid を参照します。 2. PG_RETURN_INT32 マクロを使用して、取得したPIDをSQLエンジンへ返却します。

呼び出し元と呼び出し先

  • 呼び出し元: SQLクエリから直接呼び出されるほか、src/backend/utils/adt/lockfuncs.c:456 において並列クエリ実行中のクライアントPIDリスト取得の文脈で言及されています。
  • 呼び出し先: PG_RETURN_INT32 マクロ（内部的な値の返却）。

関連データ構造

  • MyProcPid: 現在のバックエンドプロセスのPIDを格納するグローバル変数。

失敗モードと境界条件

  • プロセスが初期化されていない場合や、バックエンド外のコンテキストから呼び出された場合の挙動については、本ソースコードからは明示されていません。

保守担当者向けメモ

  • 本関数は非常に軽量なラッパーです。PIDの取得元である MyProcPid の定義や管理ライフサイクルに変更がある場合は注意してください。
