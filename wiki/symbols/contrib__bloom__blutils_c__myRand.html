myRand

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blutils.c:226

目的

PostgreSQLの標準乱数生成器から独立した擬似乱数生成を行い、インデックスのオンディスク構造が環境やバージョン変更の影響を受けないようにする。

シグネチャまたは形状

int32 myRand(void)

入力と出力

  • 入力: なし（内部状態変数 next を使用）
  • 出力: [0, 0x7ffffffd] の範囲の int32 値

前提条件と不変条件

  • 内部状態 next は、呼び出し前に mySrand 等で初期化されている必要がある。
  • 計算過程において、next は [1, 0x7ffffffe] の範囲にあることが前提となる。

アルゴリズムまたは振る舞い

Park and Millerによる線形合同法（LMM）に基づき、以下の手順で乱数を生成する。 1. next を用いて hi（商）と lo（剰余）を算出する。 2. x = 16807 * lo - 2836 * hi を計算し、オーバーフローを回避する。 3. x が負の場合は 0x7fffffff を加算して補正する。 4. 新しい next を x に更新する。 5. 戻り値として x - 1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/bloom/blutils.c 内の mySrand
  • contrib/bloom/blutils.c 内の署名生成処理（nBit の算出など）
  • 呼び出し先: なし

関連データ構造

  • 内部状態変数: next（ファイルスコープの静的変数）

失敗モードと境界条件

  • next が適切に初期化されていない場合、予測不能な値が生成される可能性がある。
  • x の計算において、31ビットの範囲を超えないよう設計されている。

保守担当者向けメモ

  • 本関数はPark and Millerの論文（Communications of the ACM, 1988）に基づく実装である。
  • 構造変更時は、ビット位置の計算に使用される SIGNWORDBITS との整合性に注意すること。
