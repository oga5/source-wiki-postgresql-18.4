XLogWalRcvSendReply

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walreceiver.c:1091

目的

スタンバイサーバーの WAL 受信状況（書き込み、フラッシュ、適用位置）をプライマリサーバーへ報告し、レプリケーションの進捗を同期させる。

シグネチャまたは形状

static void XLogWalRcvSendReply(bool force, bool requestReply)

入力と出力

  • **入力**:
  • force (bool): true の場合、ステータス間隔の制限を無視して強制的に送信する。
  • requestReply (bool): プライマリに対して応答を要求するかどうか。
  • **出力**: なし（副作用としてプライマリへメッセージを送信）。

前提条件と不変条件

  • wal_receiver_status_interval が 0 以下かつ force が false の場合、処理は即座に終了する。
  • LogstreamResult 構造体が最新の書き込み・フラッシュ位置を保持していること。

アルゴリズムまたは振る舞い

1. **ガードチェック**: force が偽かつステータス報告間隔内であれば早期リターンする。 2. **間隔チェック**: 前回の送信から wal_receiver_status_interval が経過していない場合、かつ書き込み/フラッシュ位置に変更がない場合はリターンする。 3. **更新予約**: WalRcvComputeNextWakeup を呼び出し、次回の報告タイミングをスケジュールする。 4. **メッセージ構築**:

  • 現在の LogstreamResult から writePtr と flushPtr を取得。
  • GetXLogReplayRecPtr(NULL) で現在の適用位置 (applyPtr) を取得。
  • StringInfo バッファにメッセージ種別 'r' と各 LSN、タイムスタンプ、requestReply フラグを書き込む。

    • 5. **送信**: walrcv_send を介してプライマリへメッセージを送信する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/walreceiver.c 内の各所（WalReceiverMain ループ内やメッセージ処理プロセスなど、計7箇所で確認）。
  • **呼び出し先**:
  • GetCurrentTimestamp()
  • WalRcvComputeNextWakeup()
  • GetXLogReplayRecPtr()
  • resetStringInfo() / pq_sendbyte() / pq_sendint64()
  • elog()
  • walrcv_send()

関連データ構造

  • LogstreamResult: WAL の書き込みおよびフラッシュ位置を保持するグローバル変数。
  • reply_message: 送信用の StringInfo バッファ。
  • wrconn: プライマリとの接続ハンドル。

失敗モードと境界条件

  • walrcv_send が失敗した場合、呼び出し元へエラーが伝播する可能性がある。
  • applyPtr の取得にはスピンロックが必要なため、負荷が高い環境では報告がわずかに遅延する可能性がある。

保守担当者向けメモ

  • 本関数は libpqwalreceiver を通じて通信を行うため、ネットワークの切断やタイムアウトは walrcv_send 側で処理される。
  • force 引数は、接続確立時やエラーリカバリ時など、即時同期が必要な場面で使用される。
