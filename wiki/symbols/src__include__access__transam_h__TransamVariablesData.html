TransamVariablesData

定義

  • 種別: struct
  • 定義箇所: src/include/access/transam.h:209

目的

PostgreSQLサーバー全体で共有されるトランザクションID（XID）の生成状態、OID生成状態、およびトランザクションのラップアラウンド防止に関連する制限値を保持する共有メモリ構造体です。

シグネチャまたは形状

typedef struct TransamVariablesData {
    Oid         nextOid;
    uint32      oidCount;
    FullTransactionId nextXid;
    TransactionId oldestXid;
    TransactionId xidVacLimit;
    TransactionId xidWarnLimit;
    TransactionId xidStopLimit;
    TransactionId xidWrapLimit;
    Oid         oldestXidDB;
    TransactionId oldestCommitTsXid;
    TransactionId newestCommitTsXid;
    FullTransactionId latestCompletedXid;
    uint64      xactCompletionCount;
    TransactionId oldestClogXid;
} TransamVariablesData;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入出力はありません。各フィールドは共有メモリ上の変数として、バックエンドプロセスによって参照および更新されます。

前提条件と不変条件

各フィールドは、以下の特定のロックによって保護されています。

  • OidGenLock: nextOid, oidCount
  • XidGenLock: nextXid, oldestXid, xidVacLimit, xidWarnLimit, xidStopLimit, xidWrapLimit, oldestXidDB
  • CommitTsLock: oldestCommitTsXid, newestCommitTsXid
  • ProcArrayLock: latestCompletedXid, xactCompletionCount
  • XactTruncationLock: oldestClogXid

アルゴリズムまたは振る舞い

  • nextXid は次に割り当てられる64ビットのトランザクションIDを保持します。
  • xidVacLimit から xidWrapLimit までのフィールドは、トランザクションIDの枯渇（ラップアラウンド）を防ぐための閾値として機能し、自動バキュームのトリガーやトランザクション停止の判断に使用されます。
  • xactCompletionCount はサーバー起動以降のトップレベルトランザクションの完了数を追跡し、スナップショットの再計算が必要かどうかの判定に利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/varsup.c にて初期化および参照が行われています。
  • 呼び出し先: 構造体自体はデータ保持用であり、関数呼び出しは含みません。

関連データ構造

  • FullTransactionId: 64ビットのトランザクションID。
  • TransactionId: 32ビットのトランザクションID。
  • Oid: オブジェクト識別子。

失敗モードと境界条件

  • xidWrapLimit に到達すると、システムはトランザクションIDの循環によるデータ破損を防ぐために新たなトランザクションの開始を拒否します。
  • 共有メモリ上のデータであるため、ロックの取得を怠ると競合状態が発生します。

保守担当者向けメモ

  • 本構造体は PGDLLIMPORT を通じて外部から参照されるグローバル変数 TransamVariables として定義されています。
  • 32ビットの古いXIDインターフェースと64ビットの新しいインターフェースが混在しているため、修正時にはエポックの扱いに注意してください。
