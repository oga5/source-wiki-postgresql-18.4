tuplesort_begin_cluster

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplesortvariants.c:252

目的

テーブルのクラスタリング（物理的な並び替え）を行うための Tuplesortstate を初期化し、指定されたインデックス定義に基づいたソート環境を構築する。

シグネチャまたは形状

Tuplesortstate *tuplesort_begin_cluster(TupleDesc tupDesc,
                                        Relation indexRel,
                                        int workMem,
                                        SortCoordinate coordinate,
                                        int sortopt)

入力と出力

  • **入力**:
  • tupDesc: ソート対象タプルの記述子。
  • indexRel: クラスタリングの基準となるインデックスリレーション。
  • workMem: ソートに使用可能なメモリ量。
  • coordinate: 並列ソート等の実行環境情報。
  • sortopt: ソートオプション（ランダムアクセス可否など）。
  • **出力**:
  • 初期化済みの Tuplesortstate 構造体へのポインタ。

前提条件と不変条件

  • indexRel->rd_rel->relam が BTREE_AM_OID であること（B-treeインデックスのみサポート）。
  • indexRel の各キー属性に対して SortSupport が準備される。

アルゴリズムまたは振る舞い

1. tuplesort_begin_common を呼び出し、共通状態を初期化する。 2. TuplesortClusterArg を割り当て、インデックス情報（BuildIndexInfo）を格納する。 3. クラスタリング専用の関数ポインタ（comparetup_cluster, writetup_cluster 等）を Tuplesortstate に登録する。 4. インデックスの式（ii_Expressions）が存在する場合、評価用の EState と TupleTableSlot を作成する。 5. インデックスの各キーに対して PrepareSortSupportFromIndexRel を実行し、比較ロジックを構築する。 6. 最初のキーが単純な属性である場合、省略キー（Abbreviated Keys）最適化を有効化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c:730
  • **呼び出し先**:
  • tuplesort_begin_common
  • BuildIndexInfo
  • _bt_mkscankey

  • PrepareSortSupportFromIndexRel
  • MemoryContextSwitchTo

関連データ構造

  • Tuplesortstate: ソートのメイン状態管理構造体。
  • TuplesortClusterArg: クラスタリング固有の引数と状態を保持する構造体。
  • SortSupportData: 各カラムの比較ロジックを保持する構造体。
  • IndexInfo: インデックスの属性や式に関する情報。

失敗モードと境界条件

  • インデックスがB-tree以外の場合、Assert により停止する。
  • メモリ不足時は palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は tuplesort.c のエンジンに対し、クラスタリング特有の比較・入出力ロジックを注入する役割を持つ。
  • インデックス式を含む場合、EState を作成して評価を行うため、メモリコンテキストの管理に注意が必要。
