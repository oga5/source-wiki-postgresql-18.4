match_prosrc_to_query

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_proc.c:1089

目的

関数定義のソース文字列（prosrc）が、元のSQLクエリ文字列（queryText）内のどこに位置するかを特定し、エラー発生時のカーソル位置を変換するために使用されます。

シグネチャまたは形状

static int match_prosrc_to_query(const char *prosrc, const char *queryText, int cursorpos)

入力と出力

  • **入力**:
  • prosrc: 検索対象の関数ソース文字列。
  • queryText: 関数定義を含む元のSQLコマンド文字列。
  • cursorpos: 関数ソース内でのエラー発生位置（オフセット）。
  • **出力**:
  • int: queryText 内での対応するエラー位置（オフセット）。一致しない、または複数箇所で一致した場合は 0 を返します。

前提条件と不変条件

  • queryText 内で prosrc が一意に特定できる必要があります。
  • ドル引用符（$foo$）または単一引用符（'foo'）で囲まれたリテラルとして検索されます。

アルゴリズムまたは振る舞い

1. queryText を走査し、$prosrc$ または 'prosrc' のパターンを探索します。 2. ドル引用符形式の場合、開始位置を特定し、pg_mbstrlen_with_len を用いてマルチバイト文字を考慮したオフセット計算を行います。 3. 単一引用符形式の場合、match_prosrc_to_literal を呼び出し、エスケープされた引用符やバックスラッシュを考慮した位置補正を行います。 4. 走査中に複数のマッチが見つかった場合は、曖昧さを避けるため直ちに 0 を返して失敗とみなします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/pg_proc.c 内の関数定義エラー処理ロジックから呼び出されます。
  • **呼び出し先**:
  • match_prosrc_to_literal（単一引用符形式の解析用）
  • pg_mbstrlen_with_len（文字列長計算用）

関連データ構造

  • 特になし（標準的なC文字列および整数型を使用）。

失敗モードと境界条件

  • **複数マッチ**: queryText 内に同一の prosrc が複数存在する場合、誤判定を防ぐため 0 を返します。
  • **不一致**: パターンが見つからない場合、0 を返します。

保守担当者向けメモ

  • 本関数は完全な構文解析を行わず、文字列スキャンによるヒューリスティックなマッチングを行います。複雑なネストや特殊な構文状況下では誤判定の可能性があるため、修正時には match_prosrc_to_literal との整合性に注意してください。
