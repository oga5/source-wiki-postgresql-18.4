ExecHashTableReset

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHash.c:2326

目的

ハッシュ結合の構築フェーズにおいて、前回のパスで使用されたハッシュバケットとタプルを解放し、新しいパスのためにハッシュテーブルの状態を初期化する。

シグネチャまたは形状

void ExecHashTableReset(HashJoinTable hashtable)

入力と出力

  • **入力**: hashtable (HashJoinTable) - リセット対象のハッシュ結合テーブル構造体。
  • **出力**: なし（副作用として hashtable の内部状態が更新される）。

前提条件と不変条件

  • hashtable は有効な HashJoinTable 構造体を指している必要がある。
  • hashtable->batchCxt が初期化済みであること。

アルゴリズムまたは振る舞い

1. hashtable->batchCxt を MemoryContextReset でリセットし、前回のパスで確保されたメモリ（タプルやバケット）を解放する。 2. hashtable->batchCxt にメモリコンテキストを切り替え、新しいハッシュバケットヘッダー配列を palloc0_array で再確保する。 3. hashtable->spaceUsed を 0 にリセットする。 4. hashtable->chunks を NULL に設定し、古いメモリチャンクへの参照を破棄する。 5. 元のメモリコンテキストへ戻す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHashjoin.c:1221 からの呼び出しが確認されている。
  • **呼び出し先**: MemoryContextReset, MemoryContextSwitchTo, palloc0_array などのメモリ管理関数。

関連データ構造

  • HashJoinTable: ハッシュ結合の全状態を保持する主要なデータ構造。
  • HashJoinTuple: ハッシュテーブル内のタプル表現。

失敗モードと境界条件

  • メモリ不足時、palloc0_array がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は nbatch > 1 の状況で、バッチを切り替える際に呼び出される。
  • hashtable->buckets.unshared の再確保を行っているため、並列実行時の共有メモリハッシュテーブル（shared）との挙動の違いに注意すること。
