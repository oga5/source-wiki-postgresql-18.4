calculate_toast_table_size

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/dbsize.c:395

目的

指定されたTOASTテーブルの物理的なディスクサイズ（ヒープおよび関連するすべてのインデックスの合計）を計算します。

シグネチャまたは形状

int64 calculate_toast_table_size(Oid toastrelid)

入力と出力

  • **入力**: toastrelid (Oid) - サイズを計算する対象のTOASTテーブルのOID。
  • **出力**: int64 - 対象テーブルおよびそのインデックスが占有する合計バイト数。

前提条件と不変条件

  • toastrelid は有効なリレーションを指している必要があります。
  • 呼び出し元は、対象のリレーションに対して適切なロック（AccessShareLock）を保持しているか、本関数内で取得されることを前提とします。

アルゴリズムまたは振る舞い

1. relation_open を使用して対象のTOASTテーブルを開きます。 2. MAX_FORKNUM までのすべてのフォーク（ヒープ、FSM、VM等）について、calculate_relation_size を呼び出し、サイズを累積します。 3. RelationGetIndexList を使用して、TOASTテーブルに関連付けられたすべてのインデックスのリストを取得します。 4. 各インデックスに対して relation_open を行い、同様にすべてのフォークのサイズを合計に加算します。 5. 取得したインデックスリストを解放し、開いたリレーションをすべて relation_close して合計サイズを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/dbsize.c 内の関数（例: calculate_table_size 等の文脈で呼び出される）。
  • **呼び出し先**:
  • relation_open / relation_close
  • calculate_relation_size
  • RelationGetIndexList
  • lfirst_oid
  • list_free

関連データ構造

  • Oid: リレーション識別子。
  • Relation: リレーション記述子。
  • ForkNumber: ファイルのフォーク種別（メイン、FSM、VMなど）。
  • List: インデックスのリスト。

失敗モードと境界条件

  • 指定された toastrelid が存在しない場合、relation_open がエラーを発生させます。
  • ファイルシステム上のサイズ取得に失敗した場合、calculate_relation_size の挙動に依存します。

保守担当者向けメモ

  • 本関数はTOASTテーブルの全フォークを走査するため、大規模なテーブルでは処理に時間がかかる可能性があります。
  • calculate_relation_size が内部で CHECK_FOR_INTERRUPTS() を呼び出していることを前提としています。
