ShutdownInformation

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/parallel.c:148

目的

並列処理の終了処理において、対象となる並列実行状態（ParallelState）とアーカイブハンドル（Archive）を保持し、終了処理関数へコンテキストを渡すために使用されます。

シグネチャまたは形状

typedef struct ShutdownInformation
{
	ParallelState *pstate;
	Archive    *AHX;
} ShutdownInformation;

入力と出力

  • **入力**: 構造体初期化時に ParallelState ポインタおよび Archive ポインタが格納されます。
  • **出力**: 終了処理ルーチン内で、これらのポインタを介して並列実行環境のクリーンアップやリソース解放が行われます。

前提条件と不変条件

  • pstate および AHX は、終了処理が実行される時点で有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、終了処理の引数としてキャストされて渡されます（src/bin/pg_dump/parallel.c:343）。
  • 静的変数 shutdown_info として定義されており（src/bin/pg_dump/parallel.c:154）、プログラム終了時のシグナルハンドリング等で参照されるコンテキストを保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/parallel.c 内の終了処理ロジック。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すものではありません。

関連データ構造

  • ParallelState: 並列実行全体の管理構造体。
  • Archive: アーカイブ処理のハンドル。

失敗モードと境界条件

  • 構造体内のポインタが NULL の状態で終了処理が実行された場合、参照先でのセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は parallel.c 内で静的変数として管理されているため、並列処理の終了シーケンスにおけるグローバルな状態管理の一部として機能しています。変更時は、並列実行のライフサイクル全体への影響を考慮してください。
