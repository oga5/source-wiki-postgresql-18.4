pgstat_report_checksum_failures_in_db

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/activity/pgstat_database.c:165

目的

データベースにおけるチェックサムエラーの発生回数と、最後に発生した時刻を統計情報として記録する。

シグネチャまたは形状

void pgstat_report_checksum_failures_in_db(Oid dboid, int failurecount)

入力と出力

  • **入力**:
  • dboid (Oid): 対象データベースのOID。
  • failurecount (int): 報告するチェックサムエラーの発生数。
  • **出力**: なし。

前提条件と不変条件

  • pgstat_track_counts が有効であること。
  • 対象データベースの統計エントリが、事前に pgstat_prepare_report_checksum_failure() によって作成されていること。
  • メモリ確保を避ける設計のため、クリティカルセクション内での呼び出しが可能。

アルゴリズムまたは振る舞い

1. pgstat_track_counts が無効な場合は即座に終了する。 2. pgstat_get_entry_ref を呼び出し、共有メモリ上のデータベース統計エントリへの参照を取得する（create=false を指定し、メモリ確保を回避）。 3. エントリが存在しない場合は警告を出力して終了する。 4. pgstat_lock_entry で対象エントリをロックする。 5. 共有メモリ上の checksum_failures に failurecount を加算し、last_checksum_failure を現在のタイムスタンプで更新する。 6. pgstat_unlock_entry でロックを解除する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c
  • src/backend/catalog/storage.c
  • src/backend/storage/buffer/bufmgr.c
  • **呼び出し先**:
  • pgstat_get_entry_ref
  • pgstat_lock_entry
  • pgstat_unlock_entry
  • GetCurrentTimestamp

関連データ構造

  • PgStatShared_Database: 共有メモリ上に保持されるデータベース統計構造体。
  • PgStat_EntryRef: 統計エントリへの参照を保持する構造体。

失敗モードと境界条件

  • pgstat_track_counts が無効な場合、統計は更新されない。
  • 事前に pgstat_prepare_report_checksum_failure() が呼ばれていない場合、entry_ref が取得できず、警告ログが出力されて処理が中断される。

保守担当者向けメモ

  • 本関数はクリティカルセクション内での実行を想定しているため、メモリ確保を伴う操作は行わないこと。
  • チェックサムエラーは頻繁に発生する事象ではないため、共有メモリへの直接更新を行っている。
