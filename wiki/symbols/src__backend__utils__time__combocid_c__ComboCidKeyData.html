ComboCidKeyData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/time/combocid.c:56

目的

同一トランザクション内でタプルの挿入と削除が連続して行われる際、cmin と cmax のペアを単一のキーとして識別し、Combo CID（コマンドIDの抽象化）へのマッピングを管理するために使用されます。

シグネチャまたは形状

typedef struct
{
    CommandId    cmin;
    CommandId    cmax;
} ComboCidKeyData;

入力と出力

  • **入力**: cmin および cmax の CommandId 値。
  • **出力**: 構造体として保持されるコマンドIDのペア。

前提条件と不変条件

  • 構造体内にパディングが存在しないことを前提としてハッシュ操作が行われます。この前提を崩す変更は避ける必要があります。

アルゴリズムまたは振る舞い

  • comboHash ハッシュテーブルのキーとして使用され、cmin/cmax のペアから対応する Combo CID を高速に検索するために利用されます。
  • comboCids 配列の要素として格納され、Combo CID をインデックスとして実値を参照する際にも使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/time/combocid.c 内のハッシュテーブル操作関数および配列管理ロジック。
  • 呼び出し先: 特定の関数呼び出し関係はソースから直接特定できませんが、メモリ確保（repalloc）やハッシュテーブルのキーサイズ設定（hash_ctl.keysize）の文脈で利用されています。

関連データ構造

  • ComboCidKey: ComboCidKeyData へのポインタ型。
  • ComboCidEntryData: Combo CID 管理のためのエントリ構造体。

失敗モードと境界条件

  • 構造体のパディングに関する前提が満たされない場合、ハッシュ計算やメモリ上のデータ解釈において不整合が生じるリスクがあります。

保守担当者向けメモ

  • 本構造体は TopTransactionContext に確保されるメモリ領域の一部として扱われます。
  • トランザクション終了時にメモリが解放されるため、明示的な pfree は不要です。
  • パラレルクエリ等でのシリアライズ/デシリアライズ時にも、この構造体のサイズが計算の基準となります。
