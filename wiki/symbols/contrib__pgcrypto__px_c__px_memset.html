px_memset

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/px.c:123

目的

機密情報（暗号鍵やパスワードなど）が格納されたメモリ領域を、コンパイラの最適化によって削除されることなく確実にゼロクリアするために使用されます。

シグネチャまたは形状

void px_memset(void *ptr, int c, size_t len)

入力と出力

  • ptr: クリア対象のメモリ領域へのポインタ。
  • c: メモリに書き込む値（通常は 0）。
  • len: クリアするバイト数。
  • 戻り値: なし。

前提条件と不変条件

  • ptr は有効なメモリ領域を指している必要があります。
  • len は ptr が指すバッファのサイズを超えてはなりません。

アルゴリズムまたは振る舞い

標準ライブラリの memset を呼び出します。コンパイラが「使用されないメモリへの書き込み」と判断して最適化で削除することを防ぐためのラッパーとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/crypt-blowfish.c
  • contrib/pgcrypto/crypt-md5.c
  • contrib/pgcrypto/crypt-sha.c
  • contrib/pgcrypto/mbuf.c
  • **呼び出し先**:
  • memset (標準ライブラリ)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • ptr が NULL の場合や、len が確保されたメモリ領域を超える場合は、セグメンテーション違反などの未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は、機密情報の漏洩を防ぐセキュリティ上の重要な役割を担っています。将来的にコンパイラの最適化挙動が変化した場合でも、メモリクリアが省略されないことを保証する実装を維持してください。
