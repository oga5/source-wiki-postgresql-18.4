deccvasc

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/compatlib/informix.c:197

目的

ASCII 文字列を Informix 互換の decimal 構造体に変換します。

シグネチャまたは形状

int deccvasc(const char *cp, int len, decimal *np)

入力と出力

  • **入力**:
  • cp: 変換対象の文字列へのポインタ。
  • len: 文字列の長さ。
  • np: 変換結果を格納する decimal 構造体へのポインタ。
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時は ECPG_INFORMIX_NUM_OVERFLOW、ECPG_INFORMIX_BAD_NUMERIC、または ECPG_INFORMIX_BAD_EXPONENT などのエラーコードを返します。

前提条件と不変条件

  • np は rsetnull により初期化され、NULL 値として設定されます。
  • cp が NULL 値（risnull で判定）である場合、変換は行われず 0 が返されます。

アルゴリズムまたは振る舞い

1. rsetnull を呼び出し、出力先 np を NULL 状態に初期化します。 2. ecpg_strndup を使用して入力文字列 cp を複製します。メモリ確保に失敗した場合は ECPG_INFORMIX_NUM_UNDERFLOW を返します。 3. PGTYPESnumeric_from_asc を呼び出し、文字列を PostgreSQL の numeric 型へ変換します。 4. 変換失敗時は errno を参照し、PGTYPES_NUM_OVERFLOW、PGTYPES_NUM_BAD_NUMERIC に応じて適切な Informix エラーコードへマッピングします。 5. 変換成功時は PGTYPESnumeric_to_decimal を使用して numeric を decimal 構造体に格納し、一時的な numeric オブジェクトを解放します。 6. 最後に複製した文字列を free し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/test/expected/compat_informix-dec_test.c（テストコードからの呼び出しを確認）
  • **呼び出し先**:
  • rsetnull, risnull, ecpg_strndup, PGTYPESnumeric_from_asc, PGTYPESnumeric_to_decimal, PGTYPESnumeric_free, free

関連データ構造

  • decimal: Informix 互換の数値型構造体。
  • numeric: PostgreSQL 内部の数値型構造体。

失敗モードと境界条件

  • 文字列のメモリ確保失敗: ECPG_INFORMIX_NUM_UNDERFLOW を返します。
  • 数値のオーバーフロー: PGTYPESnumeric_to_decimal または PGTYPESnumeric_from_asc の結果に基づき ECPG_INFORMIX_NUM_OVERFLOW を返します。
  • 不正な数値形式: ECPG_INFORMIX_BAD_NUMERIC を返します。
  • 不正な指数表記: ECPG_INFORMIX_BAD_EXPONENT を返します。

保守担当者向けメモ

  • 本関数は PGTYPES ライブラリを介して変換を行うため、PGTYPES 側のエラー定義と Informix 互換エラーコードの対応関係を維持する必要があります。
  • ecpg_strndup で確保したメモリは必ず解放される設計となっています。
