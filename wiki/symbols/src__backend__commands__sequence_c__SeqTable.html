SeqTable

定義

  • 種別: typedef
  • 定義箇所: src/backend/commands/sequence.c:89

目的

セッション内でアクセスされたシーケンスのキャッシュ状態を管理するためのポインタ型です。

シグネチャまたは形状

typedef SeqTableData *SeqTable;

SeqTableData 構造体へのポインタとして定義されています。

入力と出力

  • 入力: なし（型定義のため）
  • 出力: なし

前提条件と不変条件

  • SeqTable は、現在のセッションにおいて一度でもアクセスされたシーケンスの状態を保持する seqhashtab ハッシュテーブルのエントリを指すために使用されます。

アルゴリズムまたは振る舞い

  • 本シンボル自体は型定義であり、具体的なロジックは持ちません。
  • SeqTable 型の変数は、シーケンスのキャッシュ状態（last 値や cached 値など）を追跡するために、init_sequence や lock_and_open_sequence などの関数内で参照されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/sequence.c 内の複数の関数（init_sequence, lock_and_open_sequence 等）で、シーケンスの状態を保持する変数として利用されています。
  • 呼び出し先: 型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • SeqTableData: SeqTable が指し示す実体となる構造体。
  • seqhashtab: SeqTable アイテムを格納するセッション内のハッシュテーブル。

失敗モードと境界条件

  • 該当なし（型定義のため）。

保守担当者向けメモ

  • SeqTable はセッション固有のキャッシュを管理する重要な型です。シーケンスのキャッシュロジックを変更する際は、SeqTableData の定義と併せて確認してください。
