libpqrcv_get_option_from_conninfo

定義

  • 種別: function
  • 定義箇所: src/backend/replication/libpqwalreceiver/libpqwalreceiver.c:492

目的

接続文字列（conninfo）から指定されたキーワードに対応する値を取得する。

シグネチャまたは形状

char *libpqrcv_get_option_from_conninfo(const char *connInfo, const char *keyword)

入力と出力

  • **入力**:
  • connInfo: 解析対象の接続文字列。
  • keyword: 取得したいオプションのキー名。
  • **出力**:
  • 指定されたキーワードに対応する値（文字列）。該当する値が存在しない場合は NULL を返す。

前提条件と不変条件

  • connInfo は PQconninfoParse で解析可能な形式であること。
  • 戻り値として返される文字列は pstrdup によってメモリ確保されているため、呼び出し側で適切に解放する必要がある。

アルゴリズムまたは振る舞い

1. PQconninfoParse を使用して接続文字列を解析する。解析に失敗した場合は ereport(ERROR, ...) を発行して処理を中断する。 2. 解析結果のオプションリストを走査する。 3. 指定された keyword と一致し、かつ値が設定されているオプションを探す。 4. 同一キーワードが複数存在する場合は、リスト内の最後に出現した値を採用する。 5. PQconninfoFree を呼び出して解析用メモリを解放し、取得した値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • libpqrcv_get_dbname_from_conninfo (dbname取得用)
  • libpqrcv_connect 内部のオプション解析処理
  • **呼び出し先**:
  • PQconninfoParse
  • PQfreemem
  • PQconninfoFree
  • pstrdup / pfree (PostgreSQLメモリ管理関数)

関連データ構造

  • PQconninfoOption: libpq が提供する接続オプションの構造体。

失敗モードと境界条件

  • **構文エラー**: connInfo が不正な形式の場合、ereport(ERROR) によりトランザクションまたはプロセスが異常終了する。
  • **メモリ不足**: PQconninfoParse が失敗した場合、エラーメッセージを生成して終了する。
  • **値の欠如**: 指定したキーワードが存在しない、または値が空の場合は NULL を返す。

保守担当者向けメモ

  • 本関数は libpq の PQconninfoParse をラップしたユーティリティである。
  • 戻り値のメモリ管理は PostgreSQL のメモリコンテキストに依存するため、呼び出し側での pfree を忘れないこと。
