libpqrcv_identify_system

定義

  • 種別: function
  • 定義箇所: src/backend/replication/libpqwalreceiver/libpqwalreceiver.c:420

目的

プライマリサーバーに対して IDENTIFY_SYSTEM コマンドを発行し、システム識別子（System ID）およびタイムラインID（Timeline ID）を取得する。

シグネチャまたは形状

char *libpqrcv_identify_system(WalReceiverConn *conn, TimeLineID *primary_tli)

入力と出力

  • **入力**:
  • conn: 接続状態を保持する WalReceiverConn 構造体へのポインタ。
  • primary_tli: 取得したタイムラインIDを格納するための TimeLineID 型へのポインタ。
  • **出力**:
  • 戻り値: プライマリサーバーのシステム識別子（文字列）。呼び出し元で解放が必要。

前提条件と不変条件

  • conn が有効な接続状態を保持していること。
  • プライマリサーバーが IDENTIFY_SYSTEM コマンドをサポートしていること。

アルゴリズムまたは振る舞い

1. libpqsrv_exec を使用してプライマリサーバーへ IDENTIFY_SYSTEM コマンドを送信する。 2. 応答が PGRES_TUPLES_OK であることを確認する。失敗した場合は ereport(ERROR, ...) で処理を中断する。 3. 応答結果が少なくとも3列以上かつ1行であることを検証する。条件を満たさない場合はプロトコル違反としてエラーを報告する。 4. 応答の第1列からシステム識別子を抽出し、メモリを確保してコピーする。 5. 応答の第2列からタイムラインIDをパースし、primary_tli に格納する。 6. PQclear で結果セットを解放し、システム識別子を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/libpqwalreceiver/libpqwalreceiver.c 内の PQWalReceiverFunctions 構造体（.walrcv_identify_system フィールド）を通じて間接的に呼び出される。
  • **呼び出し先**: libpqsrv_exec（libpq経由のコマンド実行）、PQresultStatus, PQnfields, PQntuples, PQgetvalue, PQclear, pstrdup, pg_strtoint32, ereport。

関連データ構造

  • WalReceiverConn: 接続情報を保持する内部構造体。
  • PGresult: libpqのクエリ結果構造体。
  • TimeLineID: タイムライン識別子。

失敗モードと境界条件

  • **プロトコルエラー**: サーバーからの応答が期待される形式（行数や列数）と異なる場合、ERRCODE_PROTOCOL_VIOLATION でエラーとなる。
  • **通信エラー**: libpqsrv_exec が失敗した場合、エラーメッセージと共に処理が中断される。

保守担当者向けメモ

  • 本関数は IDENTIFY_SYSTEM の応答形式（9.3以前は3列、9.4以降は4列以上）の変化に対応するため、列数の最小値チェックを行っている。
  • 戻り値の文字列は pstrdup で確保されているため、呼び出し側で適切にメモリ管理を行う必要がある。
