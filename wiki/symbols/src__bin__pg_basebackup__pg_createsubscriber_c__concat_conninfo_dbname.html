concat_conninfo_dbname

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_createsubscriber.c:438

目的

既存の接続文字列（conninfo）に特定のデータベース名（dbname）パラメータを追加・更新し、新しい接続文字列を生成する。

シグネチャまたは形状

char *concat_conninfo_dbname(const char *conninfo, const char *dbname)

入力と出力

  • **入力**:
  • conninfo: ベースとなる接続文字列（const char *）。
  • dbname: 追加または上書きするデータベース名（const char *）。
  • **出力**:
  • 編集済みの接続文字列を保持する動的に確保されたメモリ領域（char *）。呼び出し元で解放する必要がある。

前提条件と不変条件

  • conninfo は NULL であってはならない（Assert により検証）。
  • 内部で PQExpBuffer を使用して文字列を構築する。

アルゴリズムまたは振る舞い

1. createPQExpBuffer でバッファを初期化する。 2. conninfo をバッファにコピーする。 3. appendConnStrItem を使用して、dbname パラメータをバッファに追加（または既存の値を更新）する。 4. pg_strdup を用いてバッファの内容を複製し、結果として返す。 5. destroyPQExpBuffer でバッファを破棄する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_createsubscriber.c 内の複数箇所（パブリッシャーおよびサブスクライバーの接続情報構築時）。
  • **呼び出し先**:
  • createPQExpBuffer, appendPQExpBufferStr, appendConnStrItem, pg_strdup, destroyPQExpBuffer（いずれもPostgreSQLフロントエンドライブラリ関数）。

関連データ構造

  • PQExpBuffer: 文字列構築用のバッファ構造体。

失敗モードと境界条件

  • メモリ不足時には pg_strdup や createPQExpBuffer 内部でエラーが発生し、プログラムが終了する可能性がある。
  • conninfo が空文字列の場合の挙動は appendConnStrItem の仕様に依存する。

保守担当者向けメモ

  • 本関数は pg_createsubscriber における接続文字列操作のユーティリティである。
  • 戻り値は pg_strdup で確保されているため、呼び出し側で pfree 等によるメモリ解放を適切に行うこと。
