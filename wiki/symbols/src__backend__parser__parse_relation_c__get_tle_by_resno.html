get_tle_by_resno

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_relation.c:3544

目的

指定された属性番号（resno）に対応する TargetEntry をターゲットリスト（tlist）から検索して返します。

シグネチャまたは形状

TargetEntry *get_tle_by_resno(List *tlist, AttrNumber resno)

入力と出力

  • **入力**:
  • tlist: 検索対象となる TargetEntry のリスト。
  • resno: 検索する属性番号（AttrNumber）。
  • **出力**:
  • 一致する TargetEntry 構造体へのポインタ。見つからない場合は NULL を返します。

前提条件と不変条件

  • tlist は TargetEntry ノードを含むリストである必要があります。
  • resno は TargetEntry の resno フィールドと比較可能な値である必要があります。

アルゴリズムまたは振る舞い

1. tlist を先頭から順に走査します。 2. 各要素（TargetEntry）の resno フィールドを確認します。 3. resno が引数と一致した場合、その TargetEntry を即座に返します。 4. リストの末尾まで一致するものが見つからなかった場合、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/explain.c
  • src/backend/optimizer/plan/createplan.c
  • src/backend/parser/analyze.c
  • src/backend/parser/parse_target.c
  • **呼び出し先**:
  • lfirst マクロ（リスト要素の取得に使用）。

関連データ構造

  • TargetEntry: クエリのターゲットリストを構成するノード。
  • List: PostgreSQLの汎用リスト構造。

失敗モードと境界条件

  • tlist が空（NIL）の場合、ループは実行されず NULL が返されます。
  • リスト内に指定された resno を持つ要素が存在しない場合、NULL が返されます。

保守担当者向けメモ

  • 本関数は単純な線形探索を行うため、tlist が非常に長い場合にはパフォーマンスに影響を与える可能性があります。
  • 呼び出し元で NULL が返された場合のハンドリングが適切に行われているか確認してください。
