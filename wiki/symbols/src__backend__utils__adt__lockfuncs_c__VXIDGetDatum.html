VXIDGetDatum

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/lockfuncs.c:73

目的

仮想トランザクションID（VXID）を、SQLで表示可能なテキスト形式（"<procNumber>/<lxid>"）の Datum に変換します。

シグネチャまたは形状

Datum VXIDGetDatum(ProcNumber procNumber, LocalTransactionId lxid)

入力と出力

  • **入力**:
  • procNumber (ProcNumber): プロセス番号。
  • lxid (LocalTransactionId): ローカルトランザクションID。
  • **出力**:
  • Datum: 変換されたテキストデータ。

前提条件と不変条件

  • procNumber と lxid は、それぞれ10進数および符号なし10進数として文字列化されます。
  • vxidstr バッファサイズ（32バイト）は、この形式の文字列を格納するのに十分な長さです。

アルゴリズムまたは振る舞い

1. snprintf を使用し、procNumber と lxid を "procNumber/lxid" という形式の文字列に整形します。 2. CStringGetTextDatum を呼び出し、生成された文字列を PostgreSQL の text 型の Datum に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/lockfuncs.c:305
  • src/backend/utils/adt/lockfuncs.c:355
  • src/backend/utils/adt/lockfuncs.c:421
  • **呼び出し先**:
  • snprintf (標準ライブラリ)
  • CStringGetTextDatum (PostgreSQL内部関数)

関連データ構造

  • ProcNumber: プロセスを識別する数値型。
  • LocalTransactionId: ローカルトランザクションを識別する型。
  • Datum: PostgreSQLの汎用データ表現。

失敗モードと境界条件

  • snprintf のバッファサイズ（32バイト）を超える入力が与えられた場合、文字列は切り詰められますが、通常のVXIDの範囲内では発生しません。
