int8_increment

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtcompare.c:327

目的

B-tree インデックスのスキップスキャンにおいて、int64 型のキー値をインクリメントするために使用されます。

シグネチャまたは形状

static Datum int8_increment(Relation rel, Datum existing, bool *overflow)

入力と出力

  • **入力**:
  • rel: 対象となるリレーション（未使用）。
  • existing: インクリメント対象の int64 値（Datum 型）。
  • overflow: オーバーフロー発生時に状態を書き込むためのブール型ポインタ。
  • **出力**:
  • 戻り値: インクリメントされた Datum 値。オーバーフロー時は (Datum) 0。
  • *overflow: インクリメントが PG_INT64_MAX を超える場合に true、それ以外は false。

前提条件と不変条件

  • 入力 existing は int64 として解釈可能であること。
  • overflow ポインタは有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. existing を int64 に変換します。 2. 値が PG_INT64_MAX と等しい場合、*overflow を true に設定し、戻り値として (Datum) 0 を返します。 3. それ以外の場合、*overflow を false に設定し、値に 1 を加算して Datum 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtcompare.c 内の SkipSupport 構造体初期化処理において、関数ポインタとして登録されます。
  • **呼び出し先**: 外部関数への直接的な呼び出しは確認されていません。

関連データ構造

  • SkipSupport: インデックススキップスキャン用の増減ロジックを保持する構造体。

失敗モードと境界条件

  • **オーバーフロー**: 入力値が PG_INT64_MAX の場合、インクリメントは不可能であり、overflow フラグが立てられます。この際、戻り値は未定義（0）となります。

保守担当者向けメモ

  • 本関数は SkipSupport インターフェースの一部として実装されています。
  • rel 引数は現在の実装では使用されていませんが、将来的な拡張のためにシグネチャに含まれています。
