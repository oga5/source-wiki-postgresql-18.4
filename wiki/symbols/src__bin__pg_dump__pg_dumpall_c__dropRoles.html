dropRoles

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dumpall.c:775

目的

データベースクラスタからロールを削除するための SQL 文（DROP ROLE）を生成し、出力ストリーム（OPF）へ書き出す。

シグネチャまたは形状

static void dropRoles(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn（データベース接続ハンドル）。
  • **出力**: なし（副作用としてグローバルな出力ストリーム OPF に SQL を書き出す）。

前提条件と不変条件

  • OPF が書き込み可能な状態でオープンされていること。
  • role_catalog が有効なカタログ名を参照していること。
  • server_version および if_exists グローバル変数が適切に設定されていること。

アルゴリズムまたは振る舞い

1. role_catalog からロール名（rolname）を取得するクエリを構築する。

  • server_version が 9.6 以上の場合、pg_ で始まるシステムロールを除外する。

    • 2. executeQuery を用いてデータベースからロール名の一覧を取得する。 3. ロールが存在する場合、出力ストリームに -- Drop roles というヘッダーコメントを書き出す。 4. 取得した各ロール名に対し、以下の形式で SQL を出力する。

  • DROP ROLE [IF EXISTS] <rolename>;
  • fmtId() を使用してロール名を適切にクォートする。

    • 5. 使用したリソース（PGresult および PQExpBuffer）を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dumpall.c 内のメイン処理フロー（643行目付近）。
  • **呼び出し先**: createPQExpBuffer, printfPQExpBuffer, executeQuery, PQfnumber, PQntuples, fprintf, PQgetvalue, fmtId, PQclear, destroyPQExpBuffer（いずれも PostgreSQL フロントエンドライブラリ関数）。

関連データ構造

  • PGconn: データベース接続状態を保持。
  • PQExpBuffer: SQL クエリ構築用バッファ。
  • PGresult: クエリ結果セット。

失敗モードと境界条件

  • データベース接続が切断されている場合、executeQuery 内部でエラーが発生する可能性がある。
  • role_catalog が空の場合、何も出力されない。
  • if_exists フラグが偽の場合、ロールが存在しないと SQL 実行時にエラーとなる可能性がある。

保守担当者向けメモ

  • 本関数は pg_dumpall の共有オブジェクトダンプフェーズの一部である。
  • server_version による条件分岐は、古い PostgreSQL バージョンとの互換性を維持するために重要である。
  • fmtId() によるエスケープ処理は、識別子に特殊文字が含まれる場合に必須であるため、変更時は注意すること。
