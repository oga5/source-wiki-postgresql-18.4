tar_get_file_name

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/walmethods.c:824

目的

WALのtar保存メソッドにおいて、指定されたパスと一時サフィックスを結合し、ファイル名文字列を生成する。

シグネチャまたは形状

static char *tar_get_file_name(WalWriteMethod *wwmethod, const char *pathname, const char *temp_suffix)

入力と出力

  • **入力**:
  • wwmethod: WalWriteMethod 構造体へのポインタ（本関数内では未使用）。
  • pathname: ベースとなるファイルパス。
  • temp_suffix: ファイル名に付与する一時的なサフィックス（NULL許容）。
  • **出力**:
  • 生成されたファイル名を含む char * 型のメモリ領域（呼び出し元で解放が必要）。

前提条件と不変条件

  • pg_malloc0 を使用して MAXPGPATH バイトのメモリを確保する。
  • pathname と temp_suffix を結合した文字列が MAXPGPATH に収まる必要がある。

アルゴリズムまたは振る舞い

1. pg_malloc0 を呼び出し、MAXPGPATH サイズのゼロ初期化されたメモリを確保する。 2. snprintf を使用し、pathname に temp_suffix（存在する場合）を連結した文字列を確保したメモリに書き込む。 3. 生成された文字列へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/walmethods.c:899 にて直接呼び出しを確認。
  • src/bin/pg_basebackup/walmethods.c:684 にて WalWriteMethodOps 構造体の get_file_name フィールドに登録されている。
  • **呼び出し先**:
  • pg_malloc0
  • snprintf

関連データ構造

  • WalWriteMethod: WAL書き込みメソッドの抽象基底構造体。

失敗モードと境界条件

  • pg_malloc0 がメモリ不足により失敗した場合、プログラムは終了する。
  • pathname と temp_suffix の合計長が MAXPGPATH を超える場合、snprintf により文字列が切り詰められる可能性がある。

保守担当者向けメモ

  • 本関数は WalWriteMethodOps インターフェースの一部として実装されている。
  • wwmethod 引数は現在の実装では使用されていないが、インターフェースの整合性のために保持されている。
