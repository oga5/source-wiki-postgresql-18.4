expression_returns_set

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/nodeFuncs.c:762

目的

与えられたノードツリー（式）の中に、集合を返す関数（Set-Returning Function: SRF）が含まれているかどうかを判定する。

シグネチャまたは形状

bool expression_returns_set(Node *clause)

入力と出力

  • **入力**: clause (Node*) - 検査対象となる式ツリーのルートノード。
  • **出力**: bool - 式ツリー内に集合を返す要素が含まれていれば true、そうでなければ false を返す。

前提条件と不変条件

  • 入力 clause は有効なノードツリーである必要がある。
  • 内部的に expression_returns_set_walker を呼び出すため、再帰的な走査が可能な構造であること。

アルゴリズムまたは振る舞い

1. 引数として受け取った clause を、補助関数である expression_returns_set_walker に渡す。 2. expression_returns_set_walker は、ツリーを再帰的に走査し、集合を返す性質を持つノードを探索する。 3. 探索結果（bool 値）をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeProjectSet.c
  • src/backend/optimizer/path/allpaths.c
  • src/backend/optimizer/path/equivclass.c
  • src/backend/optimizer/plan/planner.c
  • src/backend/optimizer/prep/prepjointree.c
  • **呼び出し先**:
  • expression_returns_set_walker (同ファイル内)

関連データ構造

  • Node: PostgreSQL の構文解析木や計画木を構成する汎用ノード構造体。

失敗モードと境界条件

  • clause が NULL の場合、expression_returns_set_walker の実装に依存する（通常は false を返す）。
  • ツリーが非常に深い場合、再帰呼び出しによりスタックオーバーフローが発生する可能性がある（nodeFuncs.c 全体で check_stack_depth() による保護が推奨されている）。

保守担当者向けメモ

  • 本関数は単なるラッパーであるため、ロジックの変更が必要な場合は expression_returns_set_walker を修正すること。
  • 新しいノード型を追加した際は、expression_returns_set_walker の switch 文に該当ノードの処理を追加する必要がある。
