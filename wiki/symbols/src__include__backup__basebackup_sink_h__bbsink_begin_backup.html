bbsink_begin_backup

定義

  • 種別: function
  • 定義箇所: src/include/backup/basebackup_sink.h:174

目的

バックアップ処理におけるシンク（bbsink）の初期化を行い、状態の割り当てとバッファの準備を行う。

シグネチャまたは形状

static inline void bbsink_begin_backup(bbsink *sink, bbsink_state *state, int buffer_length)

入力と出力

  • sink: 初期化対象の bbsink 構造体へのポインタ。
  • state: バックアップ全体で共有される bbsink_state へのポインタ。
  • buffer_length: 使用するバッファの長さ（バイト単位）。

前提条件と不変条件

  • sink は NULL であってはならない。
  • buffer_length は 0 より大きい必要がある。
  • 実行後、sink->bbs_buffer は NULL 以外であること。
  • sink->bbs_buffer_length は BLCKSZ の倍数であること。

アルゴリズムまたは振る舞い

1. 引数として渡された state を sink->bbs_state に格納する。 2. buffer_length を sink->bbs_buffer_length に設定する。 3. sink->bbs_ops->begin_backup(sink) を呼び出し、具体的なシンク実装側の初期化処理（バッファの割り当て等）を実行する。 4. 最後に、バッファが正常に割り当てられていることと、バッファ長が BLCKSZ の倍数であることをアサーションで検証する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c:320
  • src/backend/backup/basebackup_gzip.c:106
  • src/backend/backup/basebackup_lz4.c:125
  • src/backend/backup/basebackup_sink.c:28
  • src/backend/backup/basebackup_zstd.c:151
  • **呼び出し先**:
  • sink->bbs_ops->begin_backup(sink)（bbsink_ops 構造体で定義された関数ポインタ）

関連データ構造

  • bbsink: シンクの基本構造体。
  • bbsink_state: バックアップ全体の進捗やメタデータを保持する共有状態。
  • bbsink_ops: 各種バックアップイベントに対するコールバック関数テーブル。

失敗モードと境界条件

  • sink が NULL の場合、アサーションにより異常終了する。
  • buffer_length が 0 以下の場合、アサーションにより異常終了する。
  • begin_backup 実行後に sink->bbs_buffer が割り当てられていない場合、アサーションにより異常終了する。
  • buffer_length が BLCKSZ の倍数でない場合、アサーションにより異常終了する。

保守担当者向けメモ

  • 本関数はインライン関数であり、シンクチェーンの構築時に各レイヤーで再帰的または順次的に呼び出される。
  • bbs_buffer の割り当ては bbs_ops->begin_backup 内で行われるため、この関数を呼び出す前に bbs_ops が正しく設定されている必要がある。
