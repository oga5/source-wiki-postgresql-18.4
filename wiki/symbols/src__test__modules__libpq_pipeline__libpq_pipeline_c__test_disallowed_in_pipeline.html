test_disallowed_in_pipeline

定義

  • 種別: function
  • 定義箇所: src/test/modules/libpq_pipeline/libpq_pipeline.c:407

目的

パイプラインモードにおける制約事項の検証。パイプラインモード下で許可されない関数の挙動、およびモード遷移の冪等性を確認する。

シグネチャまたは形状

static void test_disallowed_in_pipeline(PGconn *conn)

入力と出力

  • 入力: PGconn *conn（アクティブなデータベース接続）
  • 出力: なし（検証失敗時は pg_fatal を呼び出しプロセスを終了する）

前提条件と不変条件

  • 接続 conn はブロッキングモードであること。
  • パイプラインモードへの移行および終了が正常に行えること。

アルゴリズムまたは振る舞い

1. **モード移行**: PQenterPipelineMode を呼び出し、パイプラインモードへ移行する。 2. **禁止操作の検証**:

  • PQexec を実行し、PGRES_FATAL_ERROR が返されること、および特定の禁止エラーメッセージが含まれることを確認する。
  • PQsendQuery を実行し、失敗すること、および特定の禁止エラーメッセージが含まれることを確認する。

    • 3. **冪等性の検証**:

  • 既にパイプラインモードである状態で再度 PQenterPipelineMode を呼び出し、成功することを確認する。

    • 4. **状態確認**: PQisBusy がアイドル状態で 0 を返すことを確認する。 5. **モード終了**: PQexitPipelineMode を呼び出し、通常モードへ復帰する。 6. **復帰確認**: 終了後に PQexec が正常に動作することを確認する。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/libpq_pipeline/libpq_pipeline.c:2360（main 関数内と推測される）
  • 呼び出し先: PQisnonblocking, PQenterPipelineMode, PQpipelineStatus, PQexec, PQresultStatus, PQerrorMessage, PQsendQuery, PQisBusy, PQexitPipelineMode, pg_fatal

関連データ構造

  • PGconn: libpq の接続ハンドル。

失敗モードと境界条件

  • 接続が非ブロッキングモードの場合、pg_fatal で終了する。
  • パイプラインモードへの移行・終了に失敗した場合、pg_fatal で終了する。
  • 期待されるエラーメッセージと異なる場合、pg_fatal で終了する。
  • パイプラインモード終了後に PQexec が失敗した場合、pg_fatal で終了する。

保守担当者向けメモ

  • 本関数は libpq のパイプラインモードにおける API 制約のテストケースである。
  • PQexec および PQsendQuery のエラーメッセージ文字列は libpq 本体の実装と密結合しているため、ライブラリ側のメッセージ変更時には本テストの修正が必要となる可能性がある。
