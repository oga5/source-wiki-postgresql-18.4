ResOwnerReleaseHMAC

定義

  • 種別: function
  • 定義箇所: src/common/hmac_openssl.c:365

目的

ResourceOwner によって管理されている HMAC コンテキストを解放し、リソースのクリーンアップを行う。

シグネチャまたは形状

static void ResOwnerReleaseHMAC(Datum res)

入力と出力

  • **入力**: res (Datum) - 解放対象の pg_hmac_ctx 構造体へのポインタを保持する Datum 型の値。
  • **出力**: なし（void）。

前提条件と不変条件

  • res は有効な pg_hmac_ctx 構造体を指している必要がある。
  • 本関数は ResourceOwner のコールバックとして呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. DatumGetPointer(res) を使用して、Datum から pg_hmac_ctx 構造体へのポインタを取得する。 2. コンテキスト内の resowner フィールドを NULL に設定し、リソース所有者との関連付けを解除する。 3. pg_hmac_free(ctx) を呼び出し、OpenSSL のコンテキスト破棄およびメモリ解放を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/hmac_openssl.c 内の ResourceOwner 登録処理（.ReleaseResource = ResOwnerReleaseHMAC として参照されている）。
  • **呼び出し先**: pg_hmac_free。

関連データ構造

  • pg_hmac_ctx: HMAC 計算の状態を保持する構造体。
  • ResourceOwner: PostgreSQL のバックエンド環境でリソースのライフサイクルを管理する仕組み。

失敗モードと境界条件

  • res が NULL の場合、ポインタ参照時にクラッシュする可能性がある。
  • pg_hmac_free 内部でのエラーハンドリングに依存する。

保守担当者向けメモ

  • 本関数は USE_RESOWNER_FOR_HMAC が有効なバックエンド環境でのみ機能する。
  • pg_hmac_free 内で explicit_bzero が実行されるため、機密情報（鍵など）のメモリ残留は防止される。
