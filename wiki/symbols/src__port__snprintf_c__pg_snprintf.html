pg_snprintf

定義

  • 種別: function
  • 定義箇所: src/port/snprintf.c:201

目的

プラットフォームごとの snprintf 実装の差異を吸収し、PostgreSQL で一貫した動作と POSIX 準拠のフォーマット機能（%n$ による引数順序制御など）を提供します。

シグネチャまたは形状

int pg_snprintf(char *str, size_t count, const char *fmt, ...);

入力と出力

  • **入力**:
  • str: 出力先バッファへのポインタ。
  • count: バッファの最大サイズ。
  • fmt: フォーマット文字列。
  • ...: 可変長引数。
  • **出力**:
  • 戻り値: 書き込まれた（または書き込まれるべきだった）バイト数。

前提条件と不変条件

  • C99 標準に従い、バッファサイズを超過する場合でも、切り捨てられた後の「本来書き込まれるべきだったバイト数」を返します。
  • va_list を用いて内部の pg_vsnprintf へ処理を委譲します。

アルゴリズムまたは振る舞い

1. va_start を使用して可変長引数リストを初期化します。 2. 内部関数 pg_vsnprintf を呼び出し、フォーマット処理を実行します。 3. va_end で引数リストを終了します。 4. pg_vsnprintf が返した書き込みバイト数を呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: pg_vsnprintf
  • **呼び出し元**:
  • contrib/pgcrypto/crypt-gensalt.c
  • src/bin/pgbench/pgbench.c
  • その他、snprintf を pg_snprintf に置換している各ソースファイル。

関連データ構造

  • PrintfTarget: 出力先と状態を管理する構造体（内部処理で使用）。

失敗モードと境界条件

  • バッファサイズ count が不足している場合、出力は切り捨てられますが、戻り値は切り捨て前の必要なサイズを返します。

保守担当者向けメモ

  • 本関数は標準ライブラリの snprintf を置き換える目的で設計されています。
  • このファイル内では標準ライブラリの printf 系関数が #undef されているため、直接呼び出す際は注意が必要です。
  • %n$ 指定子を使用する場合、PG_NL_ARGMAX (31) までの引数順序制御が可能です。
