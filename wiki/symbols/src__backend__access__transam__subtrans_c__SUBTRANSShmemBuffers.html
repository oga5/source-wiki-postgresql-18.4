SUBTRANSShmemBuffers

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/subtrans.c:200

目的

サブトランザクションログ（SUBTRANS）の SLRU キャッシュで使用する共有メモリバッファ数を決定する。

シグネチャまたは形状

int SUBTRANSShmemBuffers(void)

入力と出力

  • **入力**: なし（グローバル変数 subtransaction_buffers を参照）。
  • **出力**: 決定されたバッファ数（int 型）。

前提条件と不変条件

  • subtransaction_buffers が 0 の場合、自動チューニングが適用される。
  • 返り値は SLRU_MAX_ALLOWED_BUFFERS を超えない。

アルゴリズムまたは振る舞い

1. subtransaction_buffers が 0 の場合、SimpleLruAutotuneBuffers(512, 1024) を呼び出し、共有バッファ数に基づいて自動的に値を決定する。 2. subtransaction_buffers が 0 以外の場合、16 と subtransaction_buffers の大きい方を採用し、それを SLRU_MAX_ALLOWED_BUFFERS と比較して小さい方を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/subtrans.c 内の SUBTRANSShmemSize（共有メモリサイズ計算時）
  • src/backend/access/transam/subtrans.c 内の SUBTRANSShmemInit（共有メモリ初期化時）
  • **呼び出し先**:
  • SimpleLruAutotuneBuffers (条件付き)
  • Min / Max マクロ

関連データ構造

  • SubTransCtl: 本関数で決定されたバッファ数を用いて初期化される SLRU 制御構造体。

失敗モードと境界条件

  • subtransaction_buffers に極端に小さい値が設定された場合でも、最小値 16 が保証される。
  • SLRU_MAX_ALLOWED_BUFFERS を上限として制限されるため、設定値が過大であってもメモリ枯渇を防ぐ設計となっている。

保守担当者向けメモ

  • 本関数は起動時の共有メモリ割り当てに直結するため、変更時は SUBTRANSShmemSize および SUBTRANSShmemInit への影響を考慮すること。
