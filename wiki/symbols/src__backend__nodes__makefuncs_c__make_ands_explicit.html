make_ands_explicit

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/makefuncs.c:798

目的

論理積（AND）条件のリストを、単一の Expr ノードに正規化して変換します。リストの要素数に応じて、適切な形式（定数、単一式、または BoolExpr）を選択します。

シグネチャまたは形状

Expr *make_ands_explicit(List *andclauses)

入力と出力

  • **入力**: andclauses (論理積として結合されるべき Expr ノードの List)
  • **出力**: 変換後の Expr ノードへのポインタ。

前提条件と不変条件

  • 入力リストが空（NIL）の場合、true を示す BoolConst が返されます。
  • 入力リストの要素が1つの場合、リスト内の最初の要素がそのまま返されます。
  • 入力リストの要素が2つ以上の場合、make_andclause を介して BoolExpr（AND_EXPR）として構築されます。

アルゴリズムまたは振る舞い

1. andclauses が NIL の場合、makeBoolConst(true, false) を呼び出し、真の定数ノードを返します。 2. list_length(andclauses) が 1 の場合、リストの先頭要素（linitial）をそのまま返します。 3. それ以外（要素が2つ以上）の場合、make_andclause(andclauses) を呼び出し、論理積ノードを生成して返します。

関連データ構造

  • List: 条件式のリストを保持するために使用されます。
  • Expr: 戻り値の型であり、クエリツリー内の式ノードの基底型です。
  • BoolExpr: make_andclause 経由で生成される論理演算ノード。

失敗モードと境界条件

  • andclauses が NIL の場合、エラーにはならず true の定数ノードを返します。
  • メモリ不足時には makeNode 経由でメモリ確保に失敗し、システム全体としてエラー終了する可能性があります。

保守担当者向けメモ

  • 本関数は、暗黙的な論理積リストを明示的な BoolExpr 構造に変換する際に使用されます。
  • 呼び出し側でリストの長さを事前にチェックする必要はありません。本関数内で最適化（リストが1つの場合にノードを生成しない等）が行われます。
