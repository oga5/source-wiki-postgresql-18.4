overexplain_debug

定義

  • 種別: function
  • 定義箇所: contrib/pg_overexplain/pg_overexplain.c:284

目的

PlannedStmt 構造体の内部情報を EXPLAIN 出力に詳細なデバッグ情報として追加する。

シグネチャまたは形状

static void overexplain_debug(PlannedStmt *plannedstmt, ExplainState *es)

入力と出力

  • **入力**:
  • plannedstmt: 解析対象の PlannedStmt 構造体へのポインタ。
  • es: EXPLAIN コマンドの出力状態を管理する ExplainState 構造体。
  • **出力**: なし（副作用として es を通じて出力ストリームに情報を書き込む）。

前提条件と不変条件

  • es は有効な ExplainState であること。
  • plannedstmt は NULL でないこと。
  • EXPLAIN の出力フォーマット（TEXT, JSON, XML, YAML）に応じて、ExplainPropertyText 等の API が適切に処理を行う。

アルゴリズムまたは振る舞い

1. PlannedStmt グループを開始する。 2. plannedstmt->commandType に基づき、コマンドの種類（SELECT, UPDATE 等）を文字列化して出力する。 3. PlannedStmt の各種フラグ（hasReturning, parallelModeNeeded 等）をカンマ区切りの文字列として生成し出力する。 4. 補助関数 overexplain_bitmapset および overexplain_intlist を呼び出し、サブプランの巻き戻し情報、リレーション OID、実行時パラメータ型を出力する。 5. stmt_location および stmt_len を解析し、クエリ内の位置情報を出力する。 6. PlannedStmt グループを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_overexplain/pg_overexplain.c 内の overexplain_debug_handler（推論される呼び出し元）。
  • **呼び出し先**:
  • ExplainOpenGroup, ExplainCloseGroup, ExplainPropertyText, ExplainIndentText (PostgreSQL commands/explain.h API)
  • overexplain_bitmapset, overexplain_intlist (同モジュール内の補助関数)
  • initStringInfo, appendStringInfoString, psprintf (PostgreSQL 文字列操作 API)

関連データ構造

  • PlannedStmt: クエリ実行計画のルート構造体。
  • ExplainState: EXPLAIN コマンドの実行状態と出力バッファを保持する構造体。

失敗モードと境界条件

  • plannedstmt->stmt_location が -1 の場合、位置情報は「Unknown」と出力される。
  • plannedstmt->stmt_len が 0 の場合、位置情報は「～ to end」と出力される。
  • フラグが一つも立っていない場合、フラグ項目には「none」と出力される。

保守担当者向けメモ

  • 本関数は EXPLAIN (debug) オプションが指定された際に呼び出される。
  • ExplainPropertyText を使用しているため、出力フォーマットの変更（JSON 等）には自動的に追従する。
  • PlannedStmt のフィールドが追加された場合は、フラグのリスト生成ロジックを更新する必要がある。
