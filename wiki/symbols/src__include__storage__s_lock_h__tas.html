tas

定義

  • 種別: function
  • 定義箇所: src/include/storage/s_lock.h:133

目的

ハードウェアレベルの原子操作（Test-and-Set）を実行し、スピンロックの取得を試みる低レベル関数です。

シグネチャまたは形状

static __inline__ int tas(volatile slock_t *lock)

入力と出力

  • **入力**: lock - 操作対象となる slock_t 型のポインタ（volatile 修飾あり）。
  • **出力**: ロックが既に取得されていた場合は非ゼロ（1）、ロックの取得に成功した場合は 0 を返します。

前提条件と不変条件

  • lock は有効なメモリ領域を指している必要があります。
  • 本関数は x86 アーキテクチャにおけるインラインアセンブラ実装であり、lock プレフィックスを用いたアトミックな交換命令に依存します。

アルゴリズムまたは振る舞い

1. ロック値に対して、まずロックを伴わない cmpb 命令によるテストを行い、既にロックされているかを確認します。 2. ロックされていないと判断された場合、lock xchgb 命令を実行してメモリ上の値をアトミックに更新します。 3. memory および cc（条件コード）を破壊対象として指定し、コンパイラや CPU による命令の並び替えを防止します。 4. 最終的な結果（成功または失敗）を整数値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/storage/s_lock.h 内の TAS(lock) マクロを通じて、PostgreSQL のスピンロック取得処理から間接的に呼び出されます。
  • **呼び出し先**: CPU 命令（cmpb, lock xchgb）を直接実行します。関数呼び出しとしての外部依存はありません。

関連データ構造

  • slock_t: スピンロックの状態を保持する型。

失敗モードと境界条件

  • ロックが既に他者によって保持されている場合、本関数は即座に失敗（非ゼロを返却）します。
  • 本関数自体はリトライを行いません。リトライ戦略は呼び出し側の s_lock() 関数等で管理されます。

保守担当者向けメモ

  • 本関数は直接呼び出さず、必ず spin.h を介して利用してください。
  • x86 以外のアーキテクチャへの移植時には、そのプラットフォームに適した TAS 実装が必要です。
  • 現在の cmpb による事前テストの有効性については、プラットフォーム間で性能差があることが示唆されていますが、実証データがないため現状のまま維持されています。
