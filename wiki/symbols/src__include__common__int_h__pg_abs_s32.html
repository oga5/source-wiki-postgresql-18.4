pg_abs_s32

定義

  • 種別: function
  • 定義箇所: src/include/common/int.h:220

目的

符号付き32ビット整数（int32）の絶対値を、オーバーフローを回避して符号なし32ビット整数（uint32）として取得する。

シグネチャまたは形状

static inline uint32 pg_abs_s32(int32 a)

入力と出力

  • **入力**: int32 a（絶対値を求める対象の整数）
  • **出力**: uint32（a の絶対値）

前提条件と不変条件

  • 入力値が INT32_MIN であっても、オーバーフローせずに正しく絶対値（2147483648）を返却する。

アルゴリズムまたは振る舞い

1. 引数 a を int64 型にキャストし、64ビット環境での絶対値計算関数 i64abs() に渡す。 2. int64 型で計算することで、int32 の範囲を超える INT32_MIN の絶対値も安全に保持する。 3. 計算結果を uint32 型にキャストして返却する。

呼び出し元と呼び出し先

  • **呼び出し先**: i64abs()
  • **呼び出し元**:
  • src/backend/utils/adt/jsonfuncs.c (複数箇所)
  • src/backend/utils/adt/numeric.c (11584行目)

関連データ構造

  • int32, int64, uint32: PostgreSQLで定義された標準的な整数型。

失敗モードと境界条件

  • INT32_MIN を入力した場合でも、内部で int64 を経由するため、オーバーフローや未定義動作は発生しない。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、ヘッダーファイル内で完結している。
  • i64abs() への依存があるため、利用時には適切な環境定義が必要である。
