RI_QueryHashEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/ri_triggers.c:151

目的

参照整合性（RI）チェックのためにキャッシュされたSPI（Server Programming Interface）プランを保持するためのハッシュテーブルエントリ構造体です。

シグネチャまたは形状

typedef struct RI_QueryHashEntry
{
    RI_QueryKey key;    /* ハッシュテーブルの検索キー */
    SPIPlanPtr  plan;   /* キャッシュされたSPIプラン */
} RI_QueryHashEntry;

入力と出力

本構造体は直接的な関数ではありませんが、ハッシュテーブル操作において以下の役割を果たします。

  • **入力**: hash_search 関数等を通じて、RI_QueryKey をキーとして本構造体を取得または作成します。
  • **出力**: キャッシュされた SPIPlanPtr を保持し、後続の制約チェック処理で再利用可能にします。

前提条件と不変条件

  • ri_query_cache ハッシュテーブルの要素として使用されます。
  • plan フィールドは SPI_keepplan() によって永続化されたプランを指すことが想定されています。

アルゴリズムまたは振る舞い

  • 外部キー制約の検証時に、ri_BuildQueryKey で生成したキーを用いてハッシュテーブルを検索します。
  • エントリが存在しない場合は新たに作成され、ri_PlanCheck 等で生成されたプランが plan フィールドに格納されます。
  • 一度生成されたプランはバックエンドの生存期間中、ハッシュテーブルを通じて保持されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/ri_triggers.c 内のハッシュテーブル操作関数（hash_search を呼び出す箇所）が本構造体を参照・操作します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • RI_QueryKey: 本構造体の検索キーとして使用されます。
  • SPIPlanPtr: executor/spi.h で定義される、実行可能なクエリプランへのポインタです。

失敗モードと境界条件

  • ハッシュテーブルのメモリ不足により、エントリの作成が失敗する可能性があります。
  • SPI_keepplan() が失敗した場合、キャッシュされたプランが不正な状態になる可能性があります。

保守担当者向けメモ

  • 本構造体は ri_query_cache のエントリサイズとして sizeof(RI_QueryHashEntry) が指定されています。
  • キャッシュされたプランの破棄ロジックは限定的であるため、大量の制約変更が発生する環境ではメモリ使用量に注意してください。
