RelationMapInitializePhase3

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/cache/relmapper.c:691

目的

PostgreSQLの起動プロセスにおいて、ローカルのマップファイル（pg_filenode.map）をメモリ上に読み込み、カタログのOIDと物理ファイル番号の対応関係を初期化する。

シグネチャまたは形状

void RelationMapInitializePhase3(void)

入力と出力

  • **入力**: なし（内部状態およびファイルシステム上のマップファイルに依存）
  • **出力**: なし（副作用としてメモリ上のマップ状態を更新）

前提条件と不変条件

  • ブートストラップモード（IsBootstrapProcessingMode()）で実行されている場合は、マップファイルが存在しないため何もしない。
  • 正常系では、load_relmap_file を呼び出し、ファイル読み込みに失敗した場合はプロセスを終了（die）させる。

アルゴリズムまたは振る舞い

1. IsBootstrapProcessingMode() をチェックし、真であれば即座にリターンする。 2. load_relmap_file(false, false) を呼び出し、ローカルマップファイルをロードする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/relcache.c:4117
  • **呼び出し先**:
  • load_relmap_file（relmapper.c 内の静的関数）
  • IsBootstrapProcessingMode（miscadmin.h）

関連データ構造

  • RelMapFile: マップファイルの内容を保持する構造体。
  • local_map: メモリ上にロードされたローカルマップデータ。

失敗モードと境界条件

  • **ファイル読み込み失敗**: load_relmap_file 内部でエラーが発生した場合、PostgreSQLプロセスは致命的なエラー（FATAL）として終了する。
  • **ブートストラップ時**: マップファイルが存在しない環境下では、本関数は何も行わず正常終了する。

保守担当者向けメモ

  • 本関数は初期化フェーズの第3段階として位置付けられている。
  • マップファイルの整合性チェック（CRC検証など）は load_relmap_file 内で行われるため、本関数は呼び出しの制御に専念している。
