fillFakeState

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgxlog.c:34

目的

WAL リプレイ処理中に、インデックス操作に必要な最小限の SpGistState 構造体を構築します。

シグネチャまたは形状

static void fillFakeState(SpGistState *state, spgxlogState stateSrc);

入力と出力

  • **入力**:
  • state: 初期化対象となる SpGistState 構造体へのポインタ。
  • stateSrc: WAL レコードから読み取られた spgxlogState 型のソースデータ。
  • **出力**: なし（state 引数を介して副作用として値を設定）。

前提条件と不変条件

  • state 引数は有効なメモリ領域を指している必要があります。
  • palloc0 を使用して deadTupleStorage を割り当てるため、呼び出し時に適切なメモリコンテキストがアクティブである必要があります。

アルゴリズムまたは振る舞い

1. memset を使用して state 構造体全体を 0 で初期化します。 2. stateSrc から redirectXid および isBuild フラグをコピーします。 3. SGDTSIZE 分のメモリを palloc0 で確保し、state->deadTupleStorage に割り当てます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgxlog.c 内の各リプレイ関数（spgRedoAddLeaf、spgRedoMoveLeafs、spgRedoAddNode、spgRedoSplitTuple 等）から呼び出されます。
  • **呼び出し先**:
  • memset
  • palloc0

関連データ構造

  • SpGistState: SP-GiST インデックス操作の実行状態を保持する構造体。
  • spgxlogState: WAL レコード内に格納された、リプレイに必要な状態情報。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。

保守担当者向けメモ

  • 本関数は WAL リプレイ専用の簡易的な状態初期化を行うものです。通常のインデックス操作で使用される SpGistState の完全な初期化とは異なる点に注意してください。
