lclContext

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_backup_tar.c:78

目的

TAR形式のアーカイブ処理における、現在のアーカイブ状態、ファイルハンドル、および位置情報を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    int         hasSeek;
    pgoff_t     filePos;
    TAR_MEMBER *loToc;
    FILE       *tarFH;
    pgoff_t     tarFHpos;
    pgoff_t     tarNextMember;
    TAR_MEMBER *FH;
    int         isSpecialScript;
    TAR_MEMBER *scriptTH;
} lclContext;

入力と出力

本構造体は ArchiveHandle の formatData メンバとして保持され、TAR形式の入出力処理関数間で共有されます。

前提条件と不変条件

  • pg_backup_tar.c 内の各関数において、ArchiveHandle->formatData を lclContext* にキャストして使用します。
  • tarFH はアーカイブファイルへの有効なファイルポインタである必要があります。

アルゴリズムまたは振る舞い

  • アーカイブ内の現在位置（tarFHpos）や、次に読み込むべきメンバーの位置（tarNextMember）を追跡します。
  • ラージオブジェクト（loToc）や復元用スクリプト（scriptTH）のメタデータを個別に管理し、アーカイブ内の特定位置へのシークや読み込みをサポートします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_custom.c 内の複数の関数（_getFilePos 等）が、ArchiveHandle を介して本構造体を参照しています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • TAR_MEMBER: アーカイブ内の個々のファイルメンバーを管理するために使用されます。
  • ArchiveHandle: 本構造体を保持する上位のアーカイブ管理構造体です。

失敗モードと境界条件

  • hasSeek フラグにより、シーク不可能なストリームに対する挙動を制御します。
  • ファイルポインタ tarFH が NULL の場合、アーカイブ操作は失敗します。

保守担当者向けメモ

  • 本構造体は pg_backup_tar.c の内部実装に強く依存しています。
  • src/bin/pg_dump/pg_backup_custom.c からの参照が見られますが、これは pg_backup_archiver.c が提供する抽象化レイヤーを通じた一般的なアクセスパターンです。
