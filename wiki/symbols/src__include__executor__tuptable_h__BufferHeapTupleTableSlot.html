BufferHeapTupleTableSlot

定義

  • **種別**: struct
  • **定義箇所**: src/include/executor/tuptable.h:271

目的

バッファ上のヒープタプルを保持し、そのバッファに対するピン（pin）を管理するためのタプルテーブルスロット構造体です。

シグネチャまたは形状

typedef struct BufferHeapTupleTableSlot {
    pg_node_attr(abstract)
    HeapTupleTableSlot base;
    Buffer buffer;
} BufferHeapTupleTableSlot;

入力と出力

  • **入力**: HeapTupleTableSlot（基底構造体）および Buffer（バッファ識別子）。
  • **出力**: なし（構造体定義）。

前提条件と不変条件

  • buffer が InvalidBuffer でない場合、スロットは該当するバッファページに対してピンを保持している必要があります。
  • バッファへの参照を解放する際は、保持しているピンを解放する必要があります。
  • TTS_FLAG_SHOULDFREE フラグは、バッファ内のタプルを指している場合には設定してはなりません。

アルゴリズムまたは振る舞い

  • 本構造体は HeapTupleTableSlot を継承しており、物理的なヒープタプルをバッファ経由で参照します。
  • バッファ管理とタプルアクセスの抽象化レイヤーとして機能し、実行エンジンが効率的にバッファ上のデータにアクセスできるようにします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_handler.c および src/backend/access/index/genam.c において、slot から本構造体へのキャストが行われています。
  • **呼び出し先**: 構造体定義のため、直接的な関数呼び出し関係はありません。

関連データ構造

  • HeapTupleTableSlot: 基底となる構造体。
  • Buffer: バッファ管理に使用される識別子。

失敗モードと境界条件

  • buffer が InvalidBuffer の場合、ピンの解放処理は不要となります。
  • 不適切なピン管理はバッファのリークや早期解放を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は abstract 属性を持つため、直接インスタンス化するのではなく、スロット操作関数を通じて操作されることを想定しています。
  • base.tuple がバッファ内を指しているという前提条件を維持するため、タプルの生存期間管理には注意してください。
