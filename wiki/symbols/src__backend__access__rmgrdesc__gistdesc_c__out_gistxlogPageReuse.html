out_gistxlogPageReuse

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/gistdesc.c:25

目的

GiSTインデックスのページ再利用（Page Reuse）に関するWALレコードの内容を、人間が読める形式に変換してバッファへ出力します。

シグネチャまたは形状

static void out_gistxlogPageReuse(StringInfo buf, gistxlogPageReuse *xlrec)

入力と出力

  • buf (入出力): 変換後の文字列が追記される StringInfo バッファ。
  • xlrec (入力): 解析対象の gistxlogPageReuse 構造体へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • xlrec は有効な gistxlogPageReuse 構造体を指している必要があります。
  • buf は appendStringInfo が呼び出し可能な初期化済み状態である必要があります。

アルゴリズムまたは振る舞い

1. xlrec 内の locator（spcOid, dbOid, relNumber）および block 番号を抽出します。 2. snapshotConflictHorizon から Epoch と Xid をそれぞれ算出します。 3. isCatalogRel フラグに基づき、カタログリレーションか否かを 'T' または 'F' として判定します。 4. 上記の情報を appendStringInfo を用いて buf に整形して書き込みます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/rmgrdesc/gistdesc.c:72 （gist_desc 関数内からの呼び出しが示唆されています）。
  • 呼び出し先: appendStringInfo（文字列整形用ユーティリティ）、EpochFromFullTransactionId、XidFromFullTransactionId。

関連データ構造

  • StringInfo: 文字列操作用バッファ。
  • gistxlogPageReuse: GiSTのページ再利用WALレコード構造体。

失敗モードと境界条件

  • buf の容量不足が発生した場合、appendStringInfo の内部仕様に従いメモリ確保が試行されます。
  • xlrec が NULL の場合、プログラムはクラッシュします。

保守担当者向けメモ

  • 本関数はGiSTのWALデコード専用です。新しいフィールドが gistxlogPageReuse 構造体に追加された場合は、本関数の出力フォーマットも更新してください。
