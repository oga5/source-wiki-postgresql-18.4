PageOutput

定義

  • 種別: function
  • 定義箇所: src/fe_utils/print.c:3088

目的

出力先が端末である場合に、設定に応じてページャ（moreやless等）を起動し、そのパイプを書き込み用ストリームとして返却する。

シグネチャまたは形状

FILE *PageOutput(int lines, const printTableOpt *topt)

入力と出力

  • **入力**:
  • lines: 出力予定の行数。
  • topt: ページャ設定を含む printTableOpt 構造体へのポインタ。
  • **出力**:
  • 成功時: ページャのパイプ（FILE *）または標準出力（stdout）。
  • 失敗時: popen が失敗した場合は stdout を返す。

前提条件と不変条件

  • stdin および stdout が端末（isatty）である必要がある。
  • topt->pager が有効（非ゼロ）であること。
  • ページャプログラムの環境変数（PSQL_PAGER または PAGER）が設定されていること。

アルゴリズムまたは振る舞い

1. ページャ使用の可否を判定（topt->pager が有効かつ stdin/stdout が端末であるか）。 2. TIOCGWINSZ が利用可能な環境では、端末の行数（ws_row）と lines を比較し、ページャ起動の必要性を判断する。 3. ページャプログラムを環境変数から特定する。空または空白のみの場合は stdout を返す。 4. fflush(NULL) でバッファをフラッシュし、disable_sigpipe_trap() を呼び出して SIGPIPE トラップを無効化する。 5. popen でページャを起動し、成功すればそのパイプを返す。 6. popen 失敗時は restore_sigpipe_trap() でシグナル設定を復元し、stdout を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/command.c
  • src/bin/psql/common.c
  • src/bin/psql/help.c
  • src/bin/psql/input.c
  • src/fe_utils/print.c
  • **呼び出し先**:

関連データ構造

  • printTableOpt: ページャ設定（pager フラグ、pager_min_lines）を保持する構造体。

失敗モードと境界条件

  • popen が失敗した場合、エラーを報告せず stdout にフォールバックする。
  • ページャプログラムが空白のみの場合、ページャを起動せず stdout を返す。
  • TIOCGWINSZ が定義されていない環境では、行数による自動判定はスキップされる。

保守担当者向けメモ

  • popen で起動したページャは、後続の ClosePager 関数で適切に閉じる必要がある。
  • SIGPIPE のハンドリングが disable_sigpipe_trap / restore_sigpipe_trap によって制御されているため、このペアを維持すること。
