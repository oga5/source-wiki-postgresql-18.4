ssl_client_cert_present

定義

  • 種別: function
  • 定義箇所: contrib/sslinfo/sslinfo.c:99

目的

現在のセッションにおいて、クライアント証明書が提示され、かつ有効であるかどうかを判定し、その真偽値を返します。

シグネチャまたは形状

Datum ssl_client_cert_present(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（PostgreSQLの関数呼び出し規約に基づく引数）。
  • **出力**: bool 型（PG_RETURN_BOOL を介して返される）。クライアント証明書が有効な場合は true、それ以外は false。

前提条件と不変条件

  • グローバル変数 MyProcPort が初期化されている必要があります。
  • 本関数は、PostgreSQLのバックエンドプロセス内で実行されることを前提としています。

アルゴリズムまたは振る舞い

1. 現在のセッションのポート情報を保持するグローバル変数 MyProcPort にアクセスします。 2. MyProcPort->peer_cert_valid の値を参照します。 3. 参照した値を PG_RETURN_BOOL マクロを通じてSQL層へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: 外部の関数呼び出しは行いません。MyProcPort 構造体のメンバを直接参照します。

関連データ構造

  • Port（libpq/libpq-be.h で定義）: MyProcPort の型であり、SSL接続状態や証明書の検証結果を保持します。

失敗モードと境界条件

  • MyProcPort が NULL の場合、セグメンテーション違反が発生する可能性があります（通常、バックエンドプロセス内では常に有効です）。
  • SSL接続が確立されていない場合、peer_cert_valid は false を保持するため、本関数は正しく false を返します。

保守担当者向けメモ

  • 本関数は MyProcPort の状態を直接読み取る軽量なラッパーです。
  • 証明書の詳細な検証ロジックは、本関数ではなく、PostgreSQLのバックエンド（src/backend/libpq/be-secure-openssl.c 等）におけるSSLハンドシェイク時に実行されます。
