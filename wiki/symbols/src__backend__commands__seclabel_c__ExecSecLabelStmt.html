ExecSecLabelStmt

定義

  • 種別: function
  • 定義箇所: src/backend/commands/seclabel.c:114

目的

SECURITY LABEL SQL コマンドの実行を制御し、指定されたセキュリティプロバイダーを介してデータベースオブジェクトにラベルを付与・更新します。

シグネチャまたは形状

ObjectAddress ExecSecLabelStmt(SecLabelStmt *stmt)

入力と出力

  • **入力**: SecLabelStmt *stmt（解析済みの SECURITY LABEL 文の構造体）
  • **出力**: ObjectAddress（ラベルが適用された対象オブジェクトのアドレス）

前提条件と不変条件

  • label_provider_list にプロバイダーが登録されていること。
  • プロバイダーが指定されていない場合、label_provider_list に単一のプロバイダーのみが存在すること。
  • 対象オブジェクトに対する所有権（check_object_ownership）を有していること。
  • 対象オブジェクトの型が SecLabelSupportsObjectType によりサポートされていること。

アルゴリズムまたは振る舞い

1. **プロバイダーの特定**: stmt->provider が指定されていれば一致するものを検索し、未指定であれば単一の登録済みプロバイダーを選択する。 2. **オブジェクトの特定**: get_object_address を呼び出し、対象オブジェクトを特定すると同時に ShareUpdateExclusiveLock を取得する。 3. **権限チェック**: check_object_ownership により、実行ユーザーが対象オブジェクトの所有者であることを確認する。 4. **型固有の検証**: OBJECT_COLUMN の場合、対象が許可されたリレーション種別（テーブル、ビュー等）であるかを確認する。 5. **プロバイダーによる検証**: 登録された provider->hook を呼び出し、ラベルの妥当性を検証する（エラー時はここで中断）。 6. **永続化**: SetSecurityLabel を呼び出し、システムカタログを更新する。 7. **終了処理**: relation_close を呼び出し、取得したリレーションを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（SQL 文の実行パス）
  • **呼び出し先**:
  • SecLabelSupportsObjectType
  • get_object_address
  • check_object_ownership
  • provider->hook
  • SetSecurityLabel
  • relation_close

関連データ構造

  • SecLabelStmt: SQL 文の解析結果を保持する構造体。
  • LabelProvider: セキュリティラベルプロバイダーの定義（フック関数を含む）。
  • ObjectAddress: 操作対象のオブジェクトを識別する構造体。

失敗モードと境界条件

  • プロバイダーがロードされていない、または複数存在して指定がない場合にエラー。
  • 指定されたプロバイダー名がロード済みリストに存在しない場合にエラー。
  • サポート外のオブジェクト型に対して実行した場合にエラー。
  • オブジェクトが存在しない、または所有権がない場合にエラー。
  • プロバイダーのフック関数がエラーを投げた場合に処理が中断される。

保守担当者向けメモ

  • 本関数はラベルの永続化を担当しますが、ラベル自体の妥当性検証は provider->hook に完全に委譲されています。
  • get_object_address で取得したロックは、トランザクション終了時まで保持されます。
