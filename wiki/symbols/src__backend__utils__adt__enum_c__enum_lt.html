enum_lt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/enum.c:305

目的

2つの列挙型（ENUM）値の大小を比較し、左辺が右辺より小さいかどうかを判定するSQL演算子関数です。

シグネチャまたは形状

Datum enum_lt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの列挙型OID（a, b）を受け取ります。
  • **出力**: bool 型の Datum を返します。左辺が右辺より小さい場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力されたOIDは、システムカタログ pg_enum に存在する有効な列挙値である必要があります。
  • 比較ロジックは enum_cmp_internal に依存しており、列挙型の定義順序に基づいた比較が行われます。

アルゴリズムまたは振る舞い

1. PG_GETARG_OID を使用して、引数から2つの列挙型OIDを取得します。 2. enum_cmp_internal(a, b, fcinfo) を呼び出し、比較結果を取得します。 3. 比較結果が 0 未満（a < b）である場合に true を返し、それ以外の場合に false を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: enum_cmp_internal（比較の共通エンジン）。
  • **呼び出し元**:
  • contrib/btree_gist/btree_enum.c:61 からの呼び出しが確認されています。
  • その他、SQLレベルの < 演算子として呼び出されます。

関連データ構造

  • pg_enum: 列挙型の定義を保持するシステムカタログ。
  • Datum: PostgreSQLの汎用データ表現。

失敗モードと境界条件

  • 引数が無効なOIDである場合、enum_cmp_internal 内部でエラーが発生する可能性があります。
  • 列挙型の順序が定義されていない、またはキャッシュ整合性が取れていない場合、予期しない比較結果となる可能性があります。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の比較ロジックは enum_cmp_internal に集約されています。比較ロジックの変更が必要な場合は enum_cmp_internal を参照してください。
  • check_safe_enum_use による検証が呼び出し元（SQL層）で行われていることを前提としています。
