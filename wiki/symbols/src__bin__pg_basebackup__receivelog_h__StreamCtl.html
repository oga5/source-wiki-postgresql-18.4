StreamCtl

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_basebackup/receivelog.h:29

目的

PostgreSQL のストリーミングレプリケーションプロトコルを介した WAL（Write-Ahead Log）受信処理を制御するための設定構造体です。

シグネチャまたは形状

typedef struct StreamCtl
{
    XLogRecPtr    startpos;
    TimeLineID    timeline;
    char         *sysidentifier;
    int           standby_message_timeout;
    bool          synchronous;
    bool          mark_done;
    bool          do_sync;
    stream_stop_callback stream_stop;
    pgsocket      stop_socket;
    WalWriteMethod *walmethod;
    char         *partial_suffix;
    char         *replication_slot;
} StreamCtl;

入力と出力

  • **入力**: ストリーミング開始位置 (startpos)、タイムラインID (timeline)、システム識別子 (sysidentifier)、レプリケーションスロット名 (replication_slot) 等の制御パラメータ。
  • **出力**: walmethod を通じて WAL データが指定されたストレージ（ファイルやアーカイブ）へ書き込まれます。

前提条件と不変条件

  • walmethod は、WAL データの書き込み先を抽象化する有効な WalWriteMethod 構造体を指している必要があります。
  • stream_stop コールバックは、ストリーミングの継続可否を判定するために定期的に呼び出されます。

アルゴリズムまたは振る舞い

1. ReceiveXlogStream() 関数に本構造体を渡すことでストリーミングが開始されます。 2. サーバーに対して START_REPLICATION コマンドを発行し、WAL ストリームを受信します。 3. synchronous が true の場合、WAL 書き込みのたびに即時フラッシュを行います。 4. stop_socket が有効な場合、そのソケットの入力を監視し、stream_stop コールバックと併せて停止条件を評価します。 5. 受信したデータは walmethod を介して永続化されます。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_basebackup.c および pg_receivewal.c でインスタンス化され、ReceiveXlogStream() へ渡されます。
  • **呼び出し先**: receivelog.c 内の各内部関数（HandleCopyStream, ProcessXLogDataMsg, open_walfile 等）が、この構造体のフィールドを参照して処理を制御します。

関連データ構造

  • XLogRecPtr, TimeLineID: WAL の位置とタイムラインを定義する基本型。
  • WalWriteMethod: WAL 書き込みの抽象化レイヤー。
  • stream_stop_callback: ストリーミング停止判定用コールバック関数型。

失敗モードと境界条件

  • stream_stop コールバックで重い処理を行うと、ストリームの遅延が発生します。
  • synchronous を有効にすると、ディスクへのフラッシュ頻度が増加し、パフォーマンスに影響を与える可能性があります。

保守担当者向けメモ

  • walmethod はストリームの出力先を柔軟に変更するための重要な抽象化です。新しい書き込み方式を追加する場合は walmethods.h を参照してください。
  • replication_slot が NULL の場合、スロットを使用せずにストリーミングを行います。
