CompareTSQ

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_op.c:188

目的

2つの TSQuery 型オブジェクトを比較し、その順序関係（大小関係または等価性）を判定する。

シグネチャまたは形状

static int CompareTSQ(TSQuery a, TSQuery b)

入力と出力

  • **入力**:
  • a: 比較対象の TSQuery ポインタ。
  • b: 比較対象の TSQuery ポインタ。
  • **出力**:
  • 0: 両者が等価である場合。
  • -1: a が b より小さいと判定された場合。
  • 1: a が b より大きいと判定された場合。

前提条件と不変条件

  • 入力される TSQuery は有効なメモリ領域を指している必要がある。
  • 内部的に QT2QTN を使用してツリー構造へ変換するため、メモリ不足時にはエラーが発生する可能性がある。

アルゴリズムまたは振る舞い

1. **ノード数比較**: a->size と b->size を比較し、異なる場合はその大小を返す。 2. **物理サイズ比較**: VARSIZE(a) と VARSIZE(b) を比較し、異なる場合はその大小を返す。 3. **構造比較**: 両者のノード数が 0 でない場合、QT2QTN を用いてそれぞれを QTNode ツリー構造に展開する。 4. **ツリー比較**: QTNodeCompare を呼び出し、ツリー構造としての比較結果を取得する。 5. **メモリ解放**: 展開した QTNode ツリーを QTNFree で解放し、比較結果を返す。 6. **デフォルト**: ノード数が 0 の場合は等価（0）とみなす。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery_op.c 内の比較演算関数（tsquery_cmp 等）から呼び出される。
  • **呼び出し先**:
  • QT2QTN: TSQuery をツリー構造へ変換。
  • QTNodeCompare: ツリー構造の比較。
  • QTNFree: ツリー構造のメモリ解放。
  • VARSIZE: PostgreSQL の可変長データサイズ取得マクロ。

関連データ構造

  • TSQuery: 全文検索クエリの内部表現。
  • QTNode: TSQuery を操作するために展開されたツリー構造。

失敗モードと境界条件

  • a または b が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • ノード数が 0 の場合、物理サイズや構造に関わらず等価と判定される。

保守担当者向けメモ

  • 本関数は QT2QTN を介してツリー構造を生成するため、頻繁な呼び出しはオーバーヘッドとなる。
  • QTNFree によるメモリ解放が確実に行われるよう、ロジック変更時には注意が必要である。
