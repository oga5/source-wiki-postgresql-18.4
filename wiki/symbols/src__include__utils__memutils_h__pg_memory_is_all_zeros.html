pg_memory_is_all_zeros

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/memutils.h:218

目的

指定されたメモリ領域がすべてゼロで埋められているかを高速に判定します。

シグネチャまたは形状

bool pg_memory_is_all_zeros(const void *ptr, size_t len)

入力と出力

  • **入力**:
  • ptr: 検査対象のメモリ領域の先頭ポインタ。
  • len: 検査するバイト数。
  • **出力**:
  • 領域内の全バイトが 0 の場合は true。
  • ゼロ以外の値が一つでも含まれる場合は false。

前提条件と不変条件

  • ptr は有効なメモリ領域を指している必要があります。
  • 比較処理において、aligned_end は常に end を超えないことが保証されています。

アルゴリズムまたは振る舞い

メモリ領域のサイズに応じて最適化された手法で比較を行います。 1. **小規模領域**: len < sizeof(size_t) の場合、バイト単位で順次比較します。 2. **中規模領域**: sizeof(size_t) 以上かつ sizeof(size_t) * 8 未満の場合、ポインタが size_t アライメントに一致するまでバイト単位で比較し、その後 size_t 単位で比較します。 3. **大規模領域**: sizeof(size_t) * 8 以上の場合、アライメント調整後に 8 * sizeof(size_t) 単位でループを展開して比較します。この際、ビット単位の OR を使用して短絡評価を回避し、コンパイラによる SIMD 命令の生成を促進します。 4. 最後に残った端数分をバイト単位で比較します。

呼び出し元と呼び出し先

  • **呼び出し先**: 外部関数への依存はありません。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • len が 0 の場合、ループに入らず直ちに true を返します。
  • ポインタのアライメントが size_t に一致しない場合でも、適切に調整して比較を行うため、境界条件によるメモリ破壊のリスクはありません。

保守担当者向けメモ

  • 本関数はパフォーマンスが重要視される箇所で使用されるため、ループの展開や SIMD 最適化を阻害するような変更には注意してください。
  • size_t 単位の比較は、アーキテクチャのワードサイズに依存します。
