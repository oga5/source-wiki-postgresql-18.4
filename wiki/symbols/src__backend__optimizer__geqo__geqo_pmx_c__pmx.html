pmx

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_pmx.c:48

目的

遺伝的アルゴリズムを用いたクエリ最適化において、2つの親個体（tour1, tour2）から部分一致交叉（Partially Mapped Crossover: PMX）を用いて子個体（offspring）を生成する。

シグネチャまたは形状

void pmx(PlannerInfo *root, Gene *tour1, Gene *tour2, Gene *offspring, int num_gene)

入力と出力

  • **入力**:
  • root: クエリ最適化のプランナ情報。
  • tour1, tour2: 交叉対象となる親個体の遺伝子配列。
  • num_gene: 遺伝子の数。
  • **出力**:
  • offspring: 交叉結果として生成された子個体（関数内で書き換えられる）。

前提条件と不変条件

  • num_gene は正の整数であること。
  • tour1, tour2, offspring は少なくとも num_gene 分のメモリ領域を確保していること。
  • 関数内で palloc を使用するため、メモリ不足時には PostgreSQL のエラーハンドリングが呼び出される。

アルゴリズムまたは振る舞い

1. **初期化**: 交叉範囲（left, right）をランダムに決定し、offspring に tour2 をコピーする。 2. **範囲コピー**: tour1 の指定範囲を offspring に上書きし、由来を記録する。 3. **STEP 1 (マッピング)**: tour1 の範囲内の遺伝子と tour2 の対応関係に基づき、offspring 内の競合を解決する。 4. **STEP 2 (未解決の補正)**: マッピングで解決できなかった遺伝子を再配置する。 5. **STEP 3 (最終調整)**: check_list を用いて重複を検出し、未使用の遺伝子で置き換えることで順列の正当性を確保する。 6. **終了処理**: 内部で使用した一時配列（failed, from, indx, check_list）を pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_main.c:198
  • **呼び出し先**:
  • geqo_randint（乱数生成）
  • palloc / pfree（メモリ管理）

関連データ構造

  • Gene: 遺伝子を表す型。
  • PlannerInfo: クエリプランナのコンテキスト情報。
  • 内部補助配列: failed, from, indx, check_list（いずれも num_gene + 1 のサイズ）。

失敗モードと境界条件

  • left と right が逆転している場合は入れ替えて処理する。
  • mx_fail が 0 より大きい場合、STEP 2 で未解決の遺伝子を補正するロジックが実行される。
  • 遺伝子の重複が残る場合は STEP 3 で強制的に置換される。

保守担当者向けメモ

  • 本関数は #if defined(PMX) で囲まれており、ビルド設定により有効化される。
  • palloc を多用するため、大規模な遺伝子数を扱う際はメモリ消費量に留意すること。
  • アルゴリズムは「Genitor」の実装を参考にしている。
