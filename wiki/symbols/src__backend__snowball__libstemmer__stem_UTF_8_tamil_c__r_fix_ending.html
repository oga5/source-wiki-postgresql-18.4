r_fix_ending

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_tamil.c:808

目的

タミル語の語幹抽出処理において、単語末尾の接尾辞を特定のルールに基づいて除去または置換し、単語を正規化する。

シグネチャまたは形状

static int r_fix_ending(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境構造体。処理対象の文字列 z->p を含む）
  • 出力: int（成功時に 1、失敗または条件不一致時に 0、エラー発生時に負の値を返す）

前提条件と不変条件

  • len_utf8(z->p) が 3 以下の場合、処理を行わず 0 を返す。
  • 処理終了後、z->c（カーソル位置）は初期値（z->lb）に復元される。

アルゴリズムまたは振る舞い

1. 文字列長が 3 を超える場合、末尾から前方へ向かって a_5 配列内のパターンを照合する。 2. マッチしたパターンに応じて以下のいずれかの操作を行う:

  • slice_del: 接尾辞の削除。
  • slice_from_s: 接尾辞を特定の文字列に置換。
  • 条件付き処理: z->I[0] の値や、追加の among 配列（a_2, a_3, a_4 等）を用いた文脈判定を行い、削除や置換を決定する。

    • 3. a_5 にマッチしない場合、末尾の特定の文字列（s_13）を起点として、a_6 から a_9 のパターンを用いた別の削除ルールを適用する。 4. 処理の過程で slice_del や slice_from_s が失敗（負の戻り値）した場合は、その値をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_tamil.c 内の複数箇所（行 776, 1058, 1348）。
  • 呼び出し先: len_utf8, find_among_b, slice_del, slice_from_s, eq_s_b（いずれも header.h 経由のランタイム関数）。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_2 ～ a_9: 接尾辞パターンを定義した among 構造体の配列。

失敗モードと境界条件

  • 文字列長が 3 以下の場合は即座に終了する。
  • slice_del や slice_from_s がメモリ不足等の理由で負の値を返した場合、関数は即座にエラーを伝播させる。
  • パターンマッチングが失敗した場合、カーソル位置を適切に復元して終了する。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードであるため、直接編集せず、元の .sbl ファイルを修正すること。
  • among 配列のインデックスや s_* 文字列定数は、タミル語の UTF-8 バイトシーケンスに依存しているため、変更時は注意が必要。
