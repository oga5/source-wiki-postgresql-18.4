ExplainPropertyUInteger

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_format.c:235

目的

EXPLAIN コマンドの出力において、符号なし64ビット整数（uint64）のプロパティを、指定されたフォーマット（TEXT, JSON, XML, YAML）で出力バッファに書き込む。

シグネチャまたは形状

void ExplainPropertyUInteger(const char *qlabel, const char *unit, uint64 value, ExplainState *es)

入力と出力

  • qlabel: プロパティのラベル名（例: "Peak Memory Usage"）。
  • unit: プロパティの単位（例: "kB"）。単位がない場合は NULL を指定可能。
  • value: 出力対象の uint64 値。
  • es: 出力状態を管理する ExplainState 構造体へのポインタ。
  • 出力: 戻り値なし。es が保持するバッファにフォーマット済みの文字列が追加される。

前提条件と不変条件

  • es は ExplainBeginOutput によって初期化済みであること。
  • value は uint64 型の範囲内であること。

アルゴリズムまたは振る舞い

1. snprintf を使用し、UINT64_FORMAT マクロに従って value を32バイトの固定長バッファ buf に文字列として変換する。 2. 変換した文字列を ExplainProperty 関数へ引き渡す。 3. ExplainProperty 内で、es->format に基づき、JSON/YAML/XML 等の各フォーマット固有のエスケープ処理やタグ付けが行われる。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/commands/explain.c (実行計画の統計情報出力時)
  • contrib/pg_overexplain/pg_overexplain.c
  • 呼び出し先:
  • ExplainProperty (内部で呼び出し)
  • snprintf (標準ライブラリ)

関連データ構造

  • ExplainState: 出力フォーマット、インデントレベル、グループ状態を保持する中心的な構造体。

失敗モードと境界条件

  • buf のサイズ（32バイト）を超える uint64 値は想定されていない。
  • qlabel や unit が NULL の場合の挙動は、内部で呼び出される ExplainProperty の実装に依存する。

保守担当者向けメモ

  • 本関数は ExplainProperty のラッパーであり、数値の文字列化を担う。フォーマット固有の出力ロジックを変更する場合は ExplainProperty を参照すること。
  • 新しいフォーマットを追加する際は、ExplainProperty 側の switch 文を更新する必要がある。
