pthread_mutex_t

定義

  • 種別: struct
  • 定義箇所: src/port/pthread-win32.h:9

目的

Windows 環境において、POSIX スレッドのミューテックス機能をエミュレートするための構造体です。Windows ネイティブの CRITICAL_SECTION をラップし、クロスプラットフォームな同期制御を提供します。

シグネチャまたは形状

typedef struct pthread_mutex_t
{
    LONG         initstate;
    CRITICAL_SECTION csection;
} pthread_mutex_t;

入力と出力

  • 入力: pthread_mutex_t 構造体へのポインタ（pthread_mutex_lock 等の関数へ渡される）。
  • 出力: なし（構造体内部の initstate および csection が更新される）。

前提条件と不変条件

  • initstate は以下の状態を保持する:
  • 0: 未初期化
  • 1: 初期化完了
  • 2: 初期化中
  • 使用前に PTHREAD_MUTEX_INITIALIZER による静的初期化、または pthread_mutex_init による明示的な初期化が必須です。
  • 構造体を手動で直接操作することは禁止されています。

アルゴリズムまたは振る舞い

  • pthread_mutex_lock 呼び出し時に遅延初期化（Double-Checked Locking に類似）が行われます。
  • initstate が 0 の場合、InterlockedExchange を用いて排他的に InitializeCriticalSection を実行し、状態を 1 に遷移させます。
  • 実際の排他制御は Windows の CRITICAL_SECTION に委譲されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/include/port/pg_pthread.h
  • src/interfaces/ecpg/ecpglib/connect.c
  • src/interfaces/ecpg/ecpglib/misc.c
  • src/interfaces/ecpg/include/ecpg-pthread-win32.h
  • 呼び出し先:
  • Windows API (InitializeCriticalSection 等) が内部的に使用されます。

関連データ構造

  • PTHREAD_MUTEX_INITIALIZER: 本構造体を初期化するための静的マクロ。
  • CRITICAL_SECTION: Windows API が提供する同期プリミティブ。

失敗モードと境界条件

  • 初期化が完了していない状態でロック操作を行うと、initstate の競合や未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は src/port/pthread-win32.c の実装と密接に結合しています。
  • initstate の管理はスレッドセーフに行われていますが、構造体のメモリレイアウトを変更する場合は pthread-win32.c 側の初期化ロジックとの整合性を確認してください。
