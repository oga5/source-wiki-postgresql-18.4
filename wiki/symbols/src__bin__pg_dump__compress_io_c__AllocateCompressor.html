AllocateCompressor

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_io.c:122

目的

指定された圧縮仕様に基づき、データストリームの圧縮・解凍処理を行うための CompressorState を生成・初期化するファクトリ関数です。

シグネチャまたは形状

CompressorState *AllocateCompressor(const pg_compress_specification compression_spec,
                                   ReadFunc readF, WriteFunc writeF);

入力と出力

  • **入力**:
  • compression_spec: 圧縮アルゴリズムやパラメータを保持する pg_compress_specification 構造体。
  • readF: 読み込み処理を行うコールバック関数。
  • writeF: 書き込み処理を行うコールバック関数。
  • **出力**:
  • 初期化済みの CompressorState 構造体へのポインタ。

前提条件と不変条件

  • pg_malloc0 を使用してメモリを確保するため、メモリ不足時にはプロセスが終了する可能性があります。
  • readF および writeF は、後続のデータ処理時に使用されるため、有効な関数ポインタである必要があります。

アルゴリズムまたは振る舞い

1. CompressorState 構造体用にメモリを確保し、ゼロクリアします。 2. 引数として渡された readF および writeF を構造体に格納します。 3. compression_spec.algorithm の値に基づき、以下のいずれかの初期化関数を呼び出してコンテキストを構築します。

  • PG_COMPRESSION_NONE: InitCompressorNone
  • PG_COMPRESSION_GZIP: InitCompressorGzip
  • PG_COMPRESSION_LZ4: InitCompressorLZ4
  • PG_COMPRESSION_ZSTD: InitCompressorZstd

    • 4. 初期化済みの CompressorState を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_custom.c (複数箇所)
  • **呼び出し先**:
  • pg_malloc0
  • InitCompressorNone / InitCompressorGzip / InitCompressorLZ4 / InitCompressorZstd (アルゴリズムに応じていずれか)

関連データ構造

  • CompressorState: 圧縮処理のコールバックと状態を保持する構造体。
  • pg_compress_specification: 圧縮アルゴリズムの指定を保持する構造体。

失敗モードと境界条件

  • 未知の圧縮アルゴリズムが指定された場合、CompressorState は初期化されず、各アルゴリズム固有の初期化関数も呼び出されません。

保守担当者向けメモ

  • 新しい圧縮アルゴリズムを追加する場合は、本関数の if-else ブロックに新しい条件分岐を追加し、対応する InitCompressor* 関数を実装する必要があります。
