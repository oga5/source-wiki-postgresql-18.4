shimBoolConsistentFn

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginlogic.c:107

目的

GINインデックスにおいて、三値論理（GinTernaryValue）のみを実装している演算子クラスに対し、二値論理（bool）を期待するスキャンエンジンとの互換性を提供するためのアダプター関数です。

シグネチャまたは形状

static bool shimBoolConsistentFn(GinScanKey key)

入力と出力

  • **入力**: GinScanKey key（整合性チェックに必要な状態と関数ポインタを保持する構造体）
  • **出力**: bool（インデックスエントリが検索条件に合致するかどうか）

前提条件と不変条件

  • key->triConsistentFmgrInfo に有効な三値論理関数が設定されている必要があります。
  • 呼び出しにより key->recheckCurItem が副作用として更新されます。

アルゴリズムまたは振る舞い

1. key->triConsistentFmgrInfo を介して、演算子クラスの三値論理関数を呼び出します。 2. 結果が GIN_MAYBE の場合:

  • key->recheckCurItem を true に設定し、true を返します（再チェックが必要な合致とみなす）。

    • 3. 結果が GIN_MAYBE 以外の場合:

  • key->recheckCurItem を false に設定し、結果の真偽値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginlogic.c 内の ginInitConsistentFunction において、key->boolConsistentFn に関数ポインタとして登録されます。
  • **呼び出し先**: FunctionCall7Coll を介して、key->triConsistentFmgrInfo が指す演算子クラス固有の三値論理関数を呼び出します。

関連データ構造

  • GinScanKey: 整合性チェックの状態を保持する構造体。
  • GinTernaryValue: 三値論理（TRUE, FALSE, MAYBE）を表現する型。

失敗モードと境界条件

  • GIN_MAYBE が返された場合、スキャンエンジン側で recheck が発生します。これは演算子クラスが三値論理をサポートしている場合に発生する正常な動作です。

保守担当者向けメモ

  • 本関数は key->entryRes を一時的に変更する可能性があるため、並列スキャン実装時にはスレッドセーフティに注意が必要です。
  • MAYBE の判定は演算子クラス側の実装に依存します。パフォーマンス上の懸念がある場合は、演算子クラス側で二値論理関数を直接実装することが推奨されます。
