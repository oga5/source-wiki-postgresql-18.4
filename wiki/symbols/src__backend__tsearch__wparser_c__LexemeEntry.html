LexemeEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/wparser.c:147

目的

全文検索の解析処理において、トークン化された個々の語彙（lexeme）とその型情報を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	int			type;
	char	   *lexeme;
} LexemeEntry;

入力と出力

  • type: トークン型を示す整数値。
  • lexeme: 解析された語彙文字列へのポインタ。

前提条件と不変条件

  • wparser.c 内の解析処理において、動的に確保された配列（PrsStorage 等の list メンバ）の要素として使用されます。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持用であり、解析結果をリスト形式で管理するために利用されます。palloc や repalloc を用いてメモリが確保・拡張される対象となります。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tsearch/wparser.c 内の解析ロジック（st->list として参照）。
  • 呼び出し先: 情報なし。

関連データ構造

  • PrsStorage: LexemeEntry の配列を保持する構造体。

失敗モードと境界条件

  • メモリ不足時に palloc または repalloc が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は ts_parse 系関数の実行中に一時的に生成されるトークンリストの各要素です。メモリ管理は wparser.c 内の関数に依存しているため、リストの拡張や解放の際には PrsStorage の状態管理と整合性を取る必要があります。
