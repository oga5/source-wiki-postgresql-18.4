get_code_decomposition

定義

  • 種別: function
  • 定義箇所: src/common/unicode_norm.c:134

目的

Unicode 文字の分解情報を取得し、分解後のコードポイント配列とサイズを返します。

シグネチャまたは形状

static const pg_wchar *get_code_decomposition(const pg_unicode_decomposition *entry, int *dec_size)

入力と出力

  • **入力**:
  • entry: 分解情報を含む pg_unicode_decomposition 構造体へのポインタ。
  • dec_size: 分解後のコードポイント数を格納するための出力用整数ポインタ。
  • **出力**:
  • 分解されたコードポイント配列の先頭アドレス（pg_wchar *）。

前提条件と不変条件

  • entry は有効な pg_unicode_decomposition 構造体を指している必要があります。
  • インライン分解の場合、DECOMPOSITION_SIZE(entry) は 1 である必要があります（Assert による検証あり）。

アルゴリズムまたは振る舞い

1. DECOMPOSITION_IS_INLINE(entry) を判定します。 2. **インライン分解の場合**:

  • dec_size に 1 を設定します。
  • 静的変数 x に entry->dec_index を代入し、そのアドレスを返します。

    • 3. **外部テーブル参照の場合**:

  • dec_size に DECOMPOSITION_SIZE(entry) を設定します。
  • UnicodeDecomp_codepoints 配列の entry->dec_index 番目の要素から始まるアドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/unicode_norm.c 内の正規化処理フロー（unicode_normalize 等の内部処理）から複数回呼び出されています。
  • **呼び出し先**:
  • 外部関数への呼び出しはありません。内部マクロ DECOMPOSITION_IS_INLINE および DECOMPOSITION_SIZE を使用します。

関連データ構造

  • pg_unicode_decomposition: 分解情報を保持する構造体。
  • UnicodeDecomp_codepoints: 分解されたコードポイントを格納する静的データテーブル。
  • pg_wchar: ワイド文字型。

失敗モードと境界条件

  • entry が無効なポインタである場合、未定義の動作を引き起こします。
  • インライン分解時にサイズが 1 でない場合、Assert が有効なビルドではアボートします。

保守担当者向けメモ

  • 本関数は静的変数 x を使用してインライン分解の結果を返しているため、呼び出し元で返り値を保持する際は注意が必要です。
  • UnicodeDecomp_codepoints テーブルの整合性は common/unicode_norm_table.h に依存しています。
