ExecRunCompiledExpr

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_expr.c:2987

目的

JITコンパイルされた式評価関数を初回実行し、以降の呼び出しを直接実行へ切り替えるためのラッパー関数です。

シグネチャまたは形状

static Datum ExecRunCompiledExpr(ExprState *state, ExprContext *econtext, bool *isNull)

入力と出力

  • **入力**:
  • state: 式評価の状態を保持する ExprState 構造体。
  • econtext: 式評価のコンテキスト。
  • isNull: 結果が NULL かどうかを格納するフラグへのポインタ。
  • **出力**:
  • 評価結果の Datum。

前提条件と不変条件

  • state->evalfunc_private に CompiledExprState が格納されている必要があります。
  • CheckExprStillValid により、式が実行可能な状態であることが検証されます。

アルゴリズムまたは振る舞い

1. CheckExprStillValid を呼び出し、式の有効性を確認します。 2. llvm_enter_fatal_on_oom を呼び出し、メモリ不足時のエラーハンドリングを準備します。 3. llvm_get_function を使用して、cstate->context からコンパイル済み関数ポインタを取得します。 4. llvm_leave_fatal_on_oom を呼び出し、エラーハンドリングを終了します。 5. state->evalfunc を取得した関数ポインタで上書きし、次回以降の呼び出しで本関数を介さないように最適化します。 6. 取得した関数を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit_expr.c:2966 にて state->evalfunc に代入されています。
  • **呼び出し先**:
  • CheckExprStillValid
  • llvm_enter_fatal_on_oom / llvm_leave_fatal_on_oom
  • llvm_get_function
  • func (コンパイル済み関数)

関連データ構造

  • ExprState: 式評価の実行状態を管理。
  • CompiledExprState: JIT コンパイル済み式のコンテキスト情報（state->evalfunc_private として保持）。

失敗モードと境界条件

  • llvm_get_function が失敗した場合、Assert(func) によりアサーションエラーが発生します。
  • メモリ不足が発生した場合、llvm_enter_fatal_on_oom の設定に従い致命的エラーとして処理されます。

保守担当者向けメモ

  • 本関数は「初回実行時のみ」呼ばれることを前提としています。state->evalfunc を直接書き換えることで、以降のオーバーヘッドを排除する設計になっています。
  • llvm_get_function の戻り値が NULL でないことを前提としています。
