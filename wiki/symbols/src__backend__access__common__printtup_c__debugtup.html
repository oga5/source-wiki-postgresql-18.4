debugtup

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/printtup.c:461

目的

対話型バックエンドにおいて、タプル（行）の内容を標準出力へ整形表示するためのデバッグ用出力関数です。

シグネチャまたは形状

bool debugtup(TupleTableSlot *slot, DestReceiver *self);

入力と出力

  • **入力**:
  • slot: 表示対象のタプルを含む TupleTableSlot。
  • self: DestReceiver 構造体へのポインタ（本関数内では使用されません）。
  • **出力**:
  • 常に true を返します。

前提条件と不変条件

  • slot は有効な TupleTableSlot である必要があります。
  • slot->tts_tupleDescriptor が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. slot からタプル記述子を取得し、全属性をループ処理します。 2. 各属性について slot_getattr を呼び出し、値を取得します。 3. 値が NULL の場合はスキップします。 4. getTypeOutputInfo を使用して、属性の型に応じた出力関数（typoutput）を特定します。 5. OidOutputFunctionCall で値を文字列に変換します。 6. printatt を呼び出し、属性番号、属性情報、変換後の文字列を標準出力へ書き出します。 7. 全属性の処理終了後、区切り線 \t----\n を出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/print.c (直接呼び出し)
  • src/backend/tcop/dest.c (DestReceiver 経由の登録)
  • src/include/executor/execdebug.h (マクロ MJ_debugtup 経由)
  • **呼び出し先**:
  • slot_getattr
  • getTypeOutputInfo
  • OidOutputFunctionCall
  • printatt

関連データ構造

  • TupleTableSlot: タプルデータを保持する構造体。
  • DestReceiver: 出力先を抽象化する構造体。

失敗モードと境界条件

  • slot が空の場合や、属性が NULL の場合は適切にスキップされます。
  • printatt などの出力関数が失敗した場合の挙動は、それらの関数に依存します。

保守担当者向けメモ

  • 本関数は DestReceiver インターフェースを実装していますが、self 引数は使用していません。
  • printtup とは異なり、クライアントへのプロトコル送信ではなく、標準出力へのデバッグ表示を目的としています。
