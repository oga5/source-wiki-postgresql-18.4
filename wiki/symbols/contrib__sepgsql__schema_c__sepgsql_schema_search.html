sepgsql_schema_search

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/schema.c:208

目的

指定されたスキーマ（namespaceId）に対して、現在のセッションが SEARCH 権限を持っているかを SELinux ポリシーに基づいて検証します。

シグネチャまたは形状

bool sepgsql_schema_search(Oid namespaceId, bool abort_on_violation)

入力と出力

  • **入力**:
  • namespaceId (Oid): 検証対象のスキーマの OID。
  • abort_on_violation (bool): 権限がない場合にエラーを発生させるかどうか。
  • **出力**:
  • bool: 権限が許可されている場合は true、拒否された場合は false（abort_on_violation が false の場合）。

前提条件と不変条件

  • check_schema_perms 関数が利用可能であること。
  • SEPG_DB_SCHEMA__SEARCH 定数が定義されていること。

アルゴリズムまたは振る舞い

1. check_schema_perms を呼び出し、対象スキーマに対する SEPG_DB_SCHEMA__SEARCH 権限をチェックします。 2. その際、引数 abort_on_violation をそのまま check_schema_perms に渡します。 3. 権限チェックの結果をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/hooks.c:269
  • **呼び出し先**: check_schema_perms

関連データ構造

  • Oid: PostgreSQL のオブジェクト識別子。
  • SEPG_DB_SCHEMA__SEARCH: スキーマ検索権限を表す定数。

失敗モードと境界条件

  • abort_on_violation が true の場合、権限チェックに失敗すると PostgreSQL のエラーメカニズムにより処理が中断されます。
  • abort_on_violation が false の場合、権限がない場合は単に false を返します。

保守担当者向けメモ

  • 本関数は check_schema_perms のラッパーです。権限チェックのロジックを変更する場合は check_schema_perms を参照してください。
