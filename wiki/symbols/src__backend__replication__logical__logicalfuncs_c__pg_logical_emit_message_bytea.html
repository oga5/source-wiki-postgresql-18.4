pg_logical_emit_message_bytea

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/logicalfuncs.c:367

目的

ユーザー定義のバイナリメッセージを論理レプリケーションのWAL（Write-Ahead Log）に挿入するためのSQLインターフェースを提供します。

シグネチャまたは形状

Datum pg_logical_emit_message_bytea(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • transactional (bool): メッセージをトランザクションに関連付けるかどうか。
  • prefix (text): メッセージを識別するためのプレフィックス。
  • data (bytea): 書き込むバイナリデータ。
  • flush (bool): 書き込み後にWALをフラッシュするかどうか。
  • **出力**:
  • lsn (XLogRecPtr): メッセージが書き込まれたWALのログシーケンス番号。

前提条件と不変条件

  • 入力された data は bytea 型として適切に処理される必要があります。
  • prefix は内部で text_to_cstring を用いてC文字列に変換されます。

アルゴリズムまたは振る舞い

1. 引数から transactional、prefix、data、flush の各値を取得します。 2. LogLogicalMessage 関数を呼び出し、指定されたプレフィックスとバイナリデータをWALに書き込みます。 3. 書き込み結果として得られた XLogRecPtr（LSN）を戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます（直接的なC言語レベルの呼び出し元情報はソース内に明示されていません）。
  • **呼び出し先**: LogLogicalMessage を呼び出します。

関連データ構造

  • XLogRecPtr: WAL上の位置を示すデータ型。
  • bytea: PostgreSQLの可変長バイナリデータ型。

失敗モードと境界条件

  • LogLogicalMessage の内部処理に依存します。メモリ不足やWAL書き込みエラーが発生した場合、PostgreSQLの標準的なエラーハンドリングメカニズムにより例外がスローされます。

保守担当者向けメモ

  • 本関数はSQL関数として定義されており、PG_FUNCTION_ARGS を介して引数を受け取ります。
  • メッセージの書き込みロジックの詳細は LogLogicalMessage に委譲されているため、WALへの書き込み動作を変更する場合はそちらを確認してください。
