ConstructTupleDescriptor

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/index.c:280

目的

インデックス作成時に、そのインデックスのタプル記述子（TupleDesc）を構築します。ヒープリレーションの属性情報や、インデックス式、演算子クラス（opclass）の定義に基づき、インデックスの各列の型や属性を決定します。

シグネチャまたは形状

static TupleDesc ConstructTupleDescriptor(Relation heapRelation,
                                         const IndexInfo *indexInfo,
                                         const List *indexColNames,
                                         Oid accessMethodId,
                                         const Oid *collationIds,
                                         const Oid *opclassIds)

入力と出力

  • **入力**:
  • heapRelation: 対象となるヒープリレーション。
  • indexInfo: インデックスの属性数、キー属性数、インデックス式を含むメタデータ。
  • indexColNames: インデックス列名のリスト。
  • accessMethodId: インデックスアクセスメソッドのOID。
  • collationIds: 各インデックス列の照合順序（Collation）OID配列。
  • opclassIds: 各インデックス列の演算子クラスOID配列。
  • **出力**:
  • 構築された TupleDesc 構造体。

前提条件と不変条件

  • indexInfo に含まれる属性数と indexColNames の要素数は一致している必要があります。
  • インデックス式（ii_Expressions）が存在する場合、対応するリスト要素が提供されている必要があります。
  • 演算子クラスや型情報の検索において、システムキャッシュ（TYPEOID, CLAOID）が有効である必要があります。

アルゴリズムまたは振る舞い

1. GetIndexAmRoutineByAmId を呼び出し、アクセスメソッドのAPI構造体を取得します。 2. CreateTemplateTupleDesc で指定された属性数分のテンプレートを作成します。 3. 各属性について以下の処理をループで実行します：

  • 属性名、照合順序を設定します。
  • 属性が単純な列（atnum != 0）の場合、ヒープリレーションの属性定義をコピーします。
  • 属性が式（atnum == 0）の場合、式の結果型を exprType で特定し、pg_type キャッシュから型情報を取得して設定します。
  • 演算子クラスの opckeytype が指定されている場合、それを優先してキー型として適用します。
  • ANYELEMENT 型などの特殊なケースでは、基底型を解決します。

    • 4. 最終的に構築された TupleDesc を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/index.c 内の index_create 関数から呼び出されます。
  • **呼び出し先**:
  • GetIndexAmRoutineByAmId
  • RelationGetDescr
  • CreateTemplateTupleDesc
  • SearchSysCache1 / ReleaseSysCache
  • exprType / exprTypmod
  • CheckAttributeType
  • get_base_element_type

関連データ構造

  • TupleDesc: 構築されるタプル記述子。
  • IndexInfo: インデックスの定義情報。
  • IndexAmRoutine: インデックスアクセスメソッドのルーチン定義。
  • FormData_pg_attribute: 属性定義のカタログデータ。

失敗モードと境界条件

  • colnames リストや indexprs リストの要素数が不足している場合、elog(ERROR) を発生させます。
  • 存在しない型OIDや演算子クラスOIDを参照した場合、キャッシュルックアップ失敗として elog(ERROR) を発生させます。
  • インデックスとして安全でない型（匿名レコード型など）が式の結果として指定された場合、CheckAttributeType によりエラーとなります。

保守担当者向けメモ

  • 本関数はインデックスの物理的な列定義を決定する重要な箇所です。CONCURRENTLY なインデックス作成時など、カタログ操作の順序に依存する処理が含まれるため、変更時は index_create 全体のフローへの影響を考慮してください。
  • attrelid はこの時点では InvalidOid に設定され、後続の InitializeAttributeOids で修正される設計となっています。
