parse_bool_with_len

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/bool.c:36

目的

文字列をブール値（true または false）に変換します。多様な入力形式を許容する柔軟なパーサーとして機能します。

シグネチャまたは形状

bool parse_bool_with_len(const char *value, size_t len, bool *result);

入力と出力

  • value: 解析対象の文字列へのポインタ。
  • len: 文字列の長さ。
  • result: 解析結果を格納する bool 型へのポインタ（NULL の場合は結果の書き込みをスキップ）。
  • 戻り値: 解析に成功した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • value は有効なメモリ領域を指している必要があります。
  • result が NULL でない場合、解析の成否に関わらず、失敗時には false が書き込まれます。

アルゴリズムまたは振る舞い

1. value の先頭文字に基づき、switch 文で候補を絞り込みます。 2. pg_strncasecmp を使用して、以下の形式を大文字小文字を区別せずに照合します：

  • true, false
  • yes, no
  • on, off
  • 1, 0（長さが1の場合のみ）

    • 3. 一致した場合は result に値を代入し、true を返します。 4. いずれにも一致しない場合は result に false を代入し、false を返します。

関連データ構造

  • bool (C言語の論理型)

失敗モードと境界条件

  • 入力文字列が上記リストのいずれにも該当しない場合、false を返します。
  • 1 または 0 は、長さが1の場合のみ有効と判定されます。
  • on/off は、長さが2より大きい場合、その長さまで含めて比較されます。

保守担当者向けメモ

  • 本関数は設定ファイルの解析など、PostgreSQLの広範な箇所で再利用される重要なロジックです。
  • result が NULL の場合でも、解析の成否（戻り値）は正しく返されます。
