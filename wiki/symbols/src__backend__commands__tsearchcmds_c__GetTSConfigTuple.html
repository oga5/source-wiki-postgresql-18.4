GetTSConfigTuple

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tsearchcmds.c:786

目的

指定された名前（List *names）に対応するテキスト検索設定（TS Configuration）のシステムカタログタプルを取得します。

シグネチャまたは形状

HeapTuple GetTSConfigTuple(List *names)

入力と出力

  • **入力**: names (テキスト検索設定名を指定する List 構造体)
  • **出力**: HeapTuple (成功時)、または NULL (指定された設定が見つからない場合)

前提条件と不変条件

  • 呼び出し元は、取得したタプルに対して適切なタイミングで ReleaseSysCache を呼び出す責任があります。
  • pg_ts_config システムカタログがアクセス可能である必要があります。

アルゴリズムまたは振る舞い

1. get_ts_config_oid(names, true) を呼び出し、名前から設定の OID を解決します。 2. OID が無効な場合は NULL を返します。 3. SearchSysCache1 を使用して TSCONFIGOID キャッシュからタプルを検索します。 4. キャッシュからタプルが取得できない場合（論理的に到達不能な状態）、elog(ERROR) を発生させます。 5. 取得した HeapTuple を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/tsearchcmds.c 内のテキスト検索設定を操作する関数（例: AlterTSConfiguration 等）。
  • **呼び出し先**:
  • get_ts_config_oid
  • SearchSysCache1
  • elog

関連データ構造

  • HeapTuple: システムカタログの行データ。
  • TSCONFIGOID: pg_ts_config システムカタログを対象とするシステムキャッシュ識別子。

失敗モードと境界条件

  • 指定された名前の設定が存在しない場合、NULL を返します。
  • キャッシュ検索が失敗した場合（システムカタログの不整合など）、ERROR レベルでログを出力し、トランザクションを中断します。

保守担当者向けメモ

  • 本関数は pg_ts_config への読み取り専用アクセスを前提としています。
  • キャッシュの解放漏れはメモリリークの原因となるため、呼び出し側での ReleaseSysCache の実施を徹底してください。
