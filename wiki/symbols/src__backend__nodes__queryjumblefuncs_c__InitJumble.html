InitJumble

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/queryjumblefuncs.c:179

目的

クエリのフィンガープリント（queryId 生成）処理で使用する作業用状態構造体 JumbleState を初期化し、メモリを割り当てる。

シグネチャまたは形状

static JumbleState *InitJumble(void)

入力と出力

  • **入力**: なし
  • **出力**: 初期化済みの JumbleState 構造体へのポインタ。

前提条件と不変条件

  • palloc を使用するため、PostgreSQL のメモリコンテキストが有効である必要がある。
  • 戻り値の JumbleState は、クエリ走査中にハッシュバッファや定数位置情報を保持するために使用される。

アルゴリズムまたは振る舞い

1. JumbleState 構造体用にメモリを確保する。 2. ハッシュ計算用の作業用バッファ（jumble）を JUMBLE_SIZE 分確保する。 3. 定数位置記録用の配列（clocations）を初期サイズ 32 で確保する。 4. 各種カウンタ（jumble_len, clocations_count, highest_extern_param_id, pending_nulls）を 0 に初期化する。 5. has_squashed_lists フラグを false に設定する。 6. USE_ASSERT_CHECKING が定義されている場合、total_jumble_len を 0 に初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: JumbleQuery（src/backend/nodes/queryjumblefuncs.c:143）から呼び出される。
  • **呼び出し先**: palloc（メモリ割り当て関数）。

関連データ構造

  • JumbleState: クエリのフィンガープリント生成に必要な状態を保持する構造体。
  • LocationLen: 定数の位置と長さを記録するための構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • clocations 配列は初期サイズ 32 で確保されるが、必要に応じて後続の処理で拡張される前提となっている。

保守担当者向けメモ

  • 本関数で確保されたメモリは、クエリのフィンガープリント処理終了後に適切に解放される必要がある。
  • JumbleState のフィールド構成を変更した場合は、必ず本関数での初期化ロジックを更新すること。
