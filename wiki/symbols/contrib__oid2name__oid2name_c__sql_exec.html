sql_exec

定義

  • 種別: function
  • 定義箇所: contrib/oid2name/oid2name.c:371

目的

データベースに対して SQL クエリを実行し、その結果セットを整形して標準出力（stdout）に表形式で表示する汎用関数です。

シグネチャまたは形状

int sql_exec(PGconn *conn, const char *todo, bool quiet)

入力と出力

  • **入力**:
  • conn: 確立済みの PGconn 接続オブジェクト。
  • todo: 実行する SQL 文（文字列）。
  • quiet: true の場合、ヘッダー行の出力を抑制するフラグ。
  • **出力**: 成功時は 0 を返します。

前提条件と不変条件

  • conn は有効な接続状態である必要があります。
  • クエリ実行結果の各列の幅は、列名および各行のデータ値の最大長に基づいて動的に計算されます。

アルゴリズムまたは振る舞い

1. PQexec を使用して SQL を実行します。 2. クエリ結果が空、またはエラー（PQresultStatus > 2）の場合、エラーログを出力してプログラムを終了（exit(1)）します。 3. 結果セットの各列について、列名と全行のデータ値を走査し、表示に必要な最大幅を算出します。 4. quiet が false の場合、列名ヘッダーと区切り線を出力します。 5. 各行のデータを、算出された幅に合わせて整形して出力します。 6. PQclear で結果セットを解放し、メモリをクリーンアップします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/oid2name/oid2name.c 内の sql_exec_dumpalldbs, sql_exec_dumpalltables, sql_exec_searchtables, sql_exec_dumpalltbspc 等から呼び出されます。
  • **呼び出し先**:
  • libpq 関数群 (PQexec, PQresultStatus, PQerrorMessage, PQclear, PQfinish, PQntuples, PQnfields, PQfname, PQgetvalue)。
  • メモリ管理関数 (pg_malloc, free)。
  • ログ出力関数 (pg_log_error, pg_log_error_detail)。

関連データ構造

  • PGconn, PGresult: libpq ライブラリが提供する接続および結果保持用の構造体。

失敗モードと境界条件

  • クエリ実行に失敗した場合、PQerrorMessage を用いてエラー詳細を出力し、PQfinish で接続を閉じた上でプロセスを終了します。
  • メモリ割り当て失敗時は pg_malloc の内部挙動に依存します。

保守担当者向けメモ

  • 本関数は列幅の動的計算を行うため、データ量が多い場合はメモリ消費量に注意してください。
  • 出力フォーマットは固定幅の表形式であり、可読性を優先した設計となっています。
