read_backend_variables

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/launch_backend.c:902

目的

EXEC_BACKEND モードにおいて、親プロセスから引き継がれたバックエンド起動パラメータを読み込み、プロセス内の状態を復元します。

シグネチャまたは形状

static void read_backend_variables(char *id, void **startup_data, size_t *startup_data_len);

入力と出力

  • id: 起動パラメータを保持するファイルパス（非Windows）または共有メモリハンドルID（Windows）。
  • startup_data: 読み込まれた起動データへのポインタを格納する出力引数。
  • startup_data_len: 読み込まれた起動データのサイズを格納する出力引数。

前提条件と不変条件

  • EXEC_BACKEND が有効な環境で呼び出されることを前提とします。
  • 読み込み対象のファイルまたは共有メモリハンドルが、親プロセスによって正しく作成・設定されている必要があります。

アルゴリズムまたは振る舞い

1. **プラットフォーム別の読み込み**:

  • **非Windows**: id で指定されたファイルを AllocateFile で開き、BackendParameters 構造体と起動データを読み込みます。読み込み後、ファイルを閉じ、unlink で削除します。
  • **Windows**: id をハンドルとして解釈し、MapViewOfFile で共有メモリをマッピングします。構造体と起動データをコピーした後、UnmapViewOfFile および CloseHandle を実行してリソースを解放します。

    • 2. **状態の復元**: 読み込んだ BackendParameters を引数として restore_backend_variables() を呼び出し、プロセス内のグローバル変数や状態を再構築します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c 内の SubPostmasterMain から呼び出されます。
  • **呼び出し先**: restore_backend_variables（内部関数）。

関連データ構造

  • BackendParameters: プロセス間で受け渡される状態情報の構造体。

失敗モードと境界条件

  • ファイルのオープン失敗、読み込み失敗、または共有メモリのマッピング失敗が発生した場合、write_stderr でエラーを出力し、exit(1) でプロセスを終了します。
  • startup_data_len が 0 の場合、startup_data には NULL が設定されます。

保守担当者向けメモ

  • Windows環境では id を文字列から数値へ変換する際、_WIN64 の定義有無に応じて _atoi64 または atol を使い分ける必要があります。
  • restore_backend_variables の呼び出し順序は、プロセスの初期化において重要です。
