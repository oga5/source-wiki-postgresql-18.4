dataIsMoveRight

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/gindatapage.c:233

目的

GINポスティングツリーの走査において、現在のページから右側の兄弟ページへ移動すべきかどうかを判定します。

シグネチャまたは形状

static bool dataIsMoveRight(GinBtree btree, Page page)

入力と出力

  • **入力**:
  • btree: 現在のB-tree走査状態を保持する構造体。
  • page: 判定対象のGINデータページ。
  • **出力**:
  • bool: 右側へ移動する必要がある場合は true、そうでない場合は false を返します。

前提条件と不変条件

  • page は有効なGINデータページである必要があります。
  • btree->itemptr には、現在探索中のアイテムポインタが格納されている必要があります。

アルゴリズムまたは振る舞い

1. GinDataPageGetRightBound(page) を呼び出し、現在のページの右境界値（最大アイテムポインタ）を取得します。 2. GinPageRightMost(page) が真の場合、右側にページが存在しないため false を返します。 3. GinPageIsDeleted(page) が真の場合、ページが削除済みであるため true を返します。 4. 上記以外の場合、ginCompareItemPointers を使用して btree->itemptr と右境界値を比較します。探索対象のアイテムポインタが右境界値より大きい場合、右側のページへ移動すべきと判断し true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gindatapage.c:1891 にて、btree->isMoveRight 関数ポインタに代入されています。
  • **呼び出し先**:
  • GinDataPageGetRightBound
  • GinPageRightMost
  • GinPageIsDeleted
  • ginCompareItemPointers

関連データ構造

  • GinBtree: GINインデックスの走査状態を管理する構造体。
  • Page: PostgreSQLのバッファページ構造。

失敗モードと境界条件

  • ページがインデックスの最右端（RightMost）である場合、移動は発生しません。
  • ページが削除済み（Deleted）である場合、強制的に右側へ移動します。

保守担当者向けメモ

  • 本関数は GinBtree 構造体のメソッドとして利用されることを想定しています。
  • 比較ロジックは ginCompareItemPointers に依存しているため、アイテムポインタの順序定義に変更があった場合は注意が必要です。
