scram_SaltedPassword

定義

  • 種別: function
  • 定義箇所: src/common/scram-common.c:37

目的

PBKDF2（RFC 2898準拠）アルゴリズムを用いて、パスワードとソルトからSCRAM認証用のソルト付きハッシュ（SaltedPassword）を生成する。

シグネチャまたは形状

int scram_SaltedPassword(const char *password,
                         pg_cryptohash_type hash_type, int key_length,
                         const uint8 *salt, int saltlen, int iterations,
                         uint8 *result, const char **errstr);

入力と出力

  • **入力**:
  • password: 正規化済みのパスワード文字列。
  • hash_type: 使用するハッシュアルゴリズムの型。
  • key_length: 生成するキーの長さ。
  • salt, saltlen: ソルトのデータとその長さ。
  • iterations: PBKDF2の反復回数。
  • **出力**:
  • result: 生成されたハッシュ値を格納するバッファ。
  • errstr: エラー発生時にエラーメッセージへのポインタが設定される。
  • **戻り値**: 成功時に 0、失敗時に -1 を返す。

前提条件と不変条件

  • 入力パスワードは、本関数呼び出し前にSASLprepによる正規化が完了していること。
  • result バッファは、少なくとも key_length 分のメモリが確保されていること。
  • サーバー環境（FRONTEND マクロが未定義）では、長時間計算に対する応答性を確保するため、内部で CHECK_FOR_INTERRUPTS() が実行される。

アルゴリズムまたは振る舞い

1. pg_hmac_create を用いてHMACコンテキストを初期化する。 2. **初回反復**: パスワード、ソルト、およびビッグエンディアンの整数 1 を入力としてHMACを計算し、結果を result にコピーする。 3. **後続反復**: iterations - 1 回のループを実行する。

  • 前回のHMAC結果を新たな入力としてHMACを計算する。
  • 計算結果と現在の result をXOR演算し、累積する。

    • 4. 最後に pg_hmac_free でコンテキストを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth-scram.c
  • src/interfaces/libpq/fe-auth-scram.c
  • **呼び出し先**:
  • pg_hmac_create, pg_hmac_init, pg_hmac_update, pg_hmac_final, pg_hmac_free, pg_hmac_error (HMAC計算用)
  • pg_hton32 (エンディアン変換)

関連データ構造

  • pg_hmac_ctx: HMAC計算の状態を保持するコンテキスト構造体。

失敗モードと境界条件

  • pg_hmac_create が失敗した場合（メモリ不足等）、-1 を返し errstr にエラー内容を設定する。
  • 各HMAC操作（init/update/final）が失敗した場合、コンテキストを解放し -1 を返す。
  • iterations が極端に大きい場合、サーバープロセスでは CHECK_FOR_INTERRUPTS() により処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数はフロントエンドとバックエンドで共有されるため、修正時は双方の互換性に注意すること。
  • SCRAM_MAX_KEY_LEN を超えるキー長はサポートされない可能性があるため、利用時は定数定義を確認すること。
