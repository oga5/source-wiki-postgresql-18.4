WinSetMarkPosition

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeWindowAgg.c:3331

目的

ウィンドウ関数実行時における WindowObject のマーク位置（mark position）を更新し、関連する tuplestore の読み取りポインタを同期させる。

シグネチャまたは形状

void WinSetMarkPosition(WindowObject winobj, int64 markpos)

入力と出力

  • winobj: 操作対象の WindowObject。
  • markpos: 設定する新しいマーク位置（行番号）。
  • 戻り値: なし。

前提条件と不変条件

  • winobj は有効であること（WindowObjectIsValid）。
  • markpos は現在の winobj->markpos より小さい値であってはならない（後方移動は不可）。

アルゴリズムまたは振る舞い

1. winobj に関連付けられた WindowAggState を取得する。 2. markpos が現在の winobj->markpos より小さい場合、エラーを発生させる。 3. markpos が現在の位置より大きい場合:

  • winstate->buffer（tuplestore）の読み取りポインタを winobj->markptr に切り替え、markpos - winobj->markpos 分だけタプルをスキップし、winobj->markpos を更新する。

    • 4. 同様に、winstate->buffer の読み取りポインタを winobj->readptr に切り替え、markpos が現在の winobj->seekpos より大きい場合は、markpos - winobj->seekpos 分だけスキップし、winobj->seekpos を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeWindowAgg.c (複数箇所)
  • src/backend/utils/adt/windowfuncs.c (複数箇所)
  • **呼び出し先**:
  • tuplestore_select_read_pointer
  • tuplestore_skiptuples
  • elog (エラー発生時)

関連データ構造

  • WindowObject: ウィンドウ関数実行時のコンテキストを保持する構造体。
  • WindowAggState: ウィンドウ集約ノードの実行時状態。
  • tuplestore: パーティション内のタプルを保持するストレージ。

失敗モードと境界条件

  • markpos に現在の位置より小さい値を指定した場合、elog(ERROR) により処理が中断される。
  • winobj が無効な場合、Assert によりデバッグビルドで停止する。

保守担当者向けメモ

  • 本関数は tuplestore の読み取りポインタを操作するため、呼び出し順序やポインタの状態に依存する。
  • markptr と readptr の両方を更新する設計となっているため、ウィンドウ関数のフレーム移動ロジックと密接に関連している。
