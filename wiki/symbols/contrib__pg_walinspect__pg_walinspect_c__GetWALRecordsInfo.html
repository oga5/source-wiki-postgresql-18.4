GetWALRecordsInfo

定義

  • 種別: function
  • 定義箇所: contrib/pg_walinspect/pg_walinspect.c:539

目的

指定されたLSN範囲内のWALレコードを走査し、各レコードの情報を抽出してテーブル形式（tuplestore）で結果セットとして構築する。

シグネチャまたは形状

static void GetWALRecordsInfo(FunctionCallInfo fcinfo, XLogRecPtr start_lsn, XLogRecPtr end_lsn)

入力と出力

  • **入力**:
  • fcinfo: SQL関数呼び出し情報。結果セットの格納先（tuplestore）を含む。
  • start_lsn: 読み取りを開始するWALのLSN。
  • end_lsn: 読み取りを終了するWALのLSN。
  • **出力**: なし（fcinfo を通じて結果セットを返却）。

前提条件と不変条件

  • start_lsn <= end_lsn であること。
  • InitMaterializedSRF により、fcinfo がマテリアライズドSRFとして初期化されていること。

アルゴリズムまたは振る舞い

1. InitXLogReaderState を呼び出し、WAL読み取り用の状態を初期化する。 2. 一時的なメモリコンテキスト（tmp_cxt）を作成する。 3. ReadNextXLogRecord を用いて、end_lsn に達するまでWALレコードを順次読み取る。 4. ループ内で以下の処理を繰り返す：

  • tmp_cxt に切り替え、GetWALRecordInfo でレコード情報を抽出。
  • tuplestore_putvalues で結果セットに格納。
  • tmp_cxt をリセットしてメモリを解放し、CHECK_FOR_INTERRUPTS で割り込みを確認する。

    • 5. ループ終了後、tmp_cxt の削除、xlogreader の解放、および関連するメモリの解放を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • pg_get_wal_records_info
  • pg_get_wal_records_info_till_end_of_wal
  • **呼び出し先**:
  • InitMaterializedSRF
  • InitXLogReaderState
  • ReadNextXLogRecord
  • GetWALRecordInfo
  • tuplestore_putvalues
  • XLogReaderFree

関連データ構造

  • XLogReaderState: WAL読み取り状態を保持する構造体。
  • FunctionCallInfo: SQL関数呼び出しのコンテキスト。
  • ReturnSetInfo: SRFの結果セット情報。

失敗モードと境界条件

  • start_lsn が無効な場合や、WALの読み取り中にエラーが発生した場合は、PostgreSQLの標準的なエラーメカニズムにより処理が中断される。
  • end_lsn に達した時点で読み取りは終了する。

保守担当者向けメモ

  • ループ内でのメモリリークを防ぐため、tmp_cxt を使用してレコードごとのメモリ解放を確実に行う設計となっている。
  • PG_GET_WAL_RECORDS_INFO_COLS（11列）の定義は関数ローカルであり、変更時は GetWALRecordInfo の引数と整合性を取る必要がある。
