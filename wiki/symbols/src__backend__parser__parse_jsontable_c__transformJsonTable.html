transformJsonTable

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/parser/parse_jsontable.c:73

目的

SQLの JSON_TABLE 構文解析木（JsonTable ノード）を、プランナが実行可能な TableFunc 構造体へ変換し、RangeTableEntry として登録する。

シグネチャまたは形状

ParseNamespaceItem *transformJsonTable(ParseState *pstate, JsonTable *jt)

入力と出力

  • **入力**:
  • pstate: 現在のパーサ状態。
  • jt: 解析対象の JsonTable 構文解析木。
  • **出力**: 変換された TableFunc を含む ParseNamespaceItem。

前提条件と不変条件

  • rootPathSpec->string は A_Const 型の文字列であること。
  • pstate->p_lateral_active は実行開始時に false であること（関数内で一時的に true に設定される）。
  • ON ERROR 句には EMPTY、EMPTY ARRAY、または ERROR のみが許可される。

アルゴリズムまたは振る舞い

1. **初期化**: JsonTableParseContext を作成し、パス名の重複チェックを行う。 2. **LATERAL設定**: pstate->p_lateral_active を有効化し、JSON_TABLE 内の列参照を可能にする。 3. **ノード変換**:

  • JsonFuncExpr を構築し、transformExpr を通じて TableFunc.docexpr を生成する。
  • transformJsonTableColumns を呼び出し、カラム定義を再帰的に処理して TableFunc.plan を構築する。

    • 4. **メタデータ設定**: PASSING 句の引数を TableFunc にコピーし、序数カラム（ordinality）を初期化する。 5. **登録**: addRangeTableEntryForTableFunc を呼び出し、RangeTableEntry を生成して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_clause.c:1114
  • **呼び出し先**:
  • generateJsonTablePathName
  • CheckDuplicateColumnOrPathNames
  • transformExpr
  • transformJsonTableColumns
  • copyObject
  • contain_vars_of_level
  • addRangeTableEntryForTableFunc

関連データ構造

  • JsonTable: 入力となる構文解析木。
  • TableFunc: 出力となる実行用構造体。
  • JsonTableParseContext: 解析中に共有されるコンテキスト。

失敗モードと境界条件

  • ON ERROR 句に不正な動作が指定された場合、ereport(ERROR, ...) により構文エラーを発生させる。
  • カラム名やパス名が重複している場合、CheckDuplicateColumnOrPathNames によりエラーが発生する。

保守担当者向けメモ

  • 本関数は LATERAL 結合の挙動を前提としており、pstate->p_lateral_active の制御が重要である。
  • NESTED 句の再帰処理は transformJsonTableColumns に委譲されている。
  • PASSING 句の引数は TableFunc の docexpr とは別に評価されるため、tf->passingvalexprs へのコピーが必須である。
