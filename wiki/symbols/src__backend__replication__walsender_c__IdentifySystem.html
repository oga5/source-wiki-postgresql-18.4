IdentifySystem

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walsender.c:396

目的

レプリケーション接続において、プライマリサーバーのシステム識別子、現在のタイムラインID、最新のWAL位置、および接続先データベース名をクライアントへ通知する。

シグネチャまたは形状

static void IdentifySystem(void)

入力と出力

  • 入力: なし（グローバルなシステム状態および接続状態を参照）
  • 出力: なし（DestRemoteSimple を介して、4列のタプルをクライアントへ送信する）

前提条件と不変条件

  • 呼び出し時に有効なレプリケーション接続が確立されている必要がある。
  • データベース名を取得する場合、MyDatabaseId が有効である必要がある。

アルゴリズムまたは振る舞い

1. GetSystemIdentifier() を用いてシステムIDを取得する。 2. RecoveryInProgress() を確認し、カスケードレプリケーション中であればスタンバイのフラッシュ位置を、そうでなければ自身のフラッシュ位置を logptr に取得する。 3. MyDatabaseId が有効な場合、一時的にトランザクションを開始してデータベース名を取得し、メモリコンテキストを跨いで保持する。 4. 4列（systemid, timeline, xlogpos, dbname）のタプル記述子を作成する。 5. 取得した情報を do_tup_output を通じてクライアントへ送信する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/walsender.c 内のレプリケーションコマンド処理フロー（exec_replication_command 等から参照される）。
  • 呼び出し先: GetSystemIdentifier, RecoveryInProgress, GetStandbyFlushRecPtr, GetFlushRecPtr, StartTransactionCommand, get_database_name, CommitTransactionCommand, CreateDestReceiver, do_tup_output 等。

関連データ構造

  • XLogRecPtr: WAL位置の保持に使用。
  • TupOutputState: タプル出力の管理に使用。
  • TupleDesc: 出力するタプルのスキーマ定義に使用。

失敗モードと境界条件

  • データベースに接続していない場合、dbname 列は NULL として送信される。
  • トランザクション環境外でデータベース名を取得しようとした場合、syscache アクセスによりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は IDENTIFY_SYSTEM コマンドの実行実体である。
  • 出力フォーマット（列数や型）を変更する場合は、クライアント側の互換性に注意すること。
  • dbname の取得時にトランザクションを明示的に開始・終了している点に留意すること。
