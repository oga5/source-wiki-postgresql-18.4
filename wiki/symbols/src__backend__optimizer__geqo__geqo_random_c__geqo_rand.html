geqo_rand

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_random.c:27

目的

GEQO（遺伝的クエリ最適化）モジュールにおいて、セッション固有の乱数状態に基づいた浮動小数点数の乱数を生成する。

シグネチャまたは形状

double geqo_rand(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root - クエリ最適化の情報を保持する構造体。内部の join_search_private を介して乱数状態を参照する。
  • **出力**: double - 0.0 以上 1.0 未満の範囲の擬似乱数値。

前提条件と不変条件

  • root->join_search_private が GeqoPrivateData 型として正しく初期化されている必要がある。
  • pg_prng の乱数状態がセッションごとに独立して管理されている。

アルゴリズムまたは振る舞い

1. 引数 root から GeqoPrivateData 構造体を取得する。 2. 構造体内の random_state を参照し、pg_prng_double を呼び出して乱数を生成する。 3. 生成された値を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_selection.c において、選択アルゴリズムの確率計算に使用されている。
  • **呼び出し先**: pg_prng_double（src/backend/utils/misc/pg_prng.c で定義される PostgreSQL 標準の擬似乱数生成関数）。

関連データ構造

  • PlannerInfo: クエリ最適化のコンテキスト。
  • GeqoPrivateData: GEQO 固有のプライベートデータ。乱数生成器の状態（random_state）を保持する。

失敗モードと境界条件

  • 戻り値の範囲は [0.0, 1.0) である。
  • PlannerInfo が NULL の場合や、join_search_private が初期化されていない場合はセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は pg_prng をラップしたインターフェースである。
  • 遺伝的アルゴリズムの再現性を確保したい場合は、geqo_set_seed を使用して初期状態を固定すること。
