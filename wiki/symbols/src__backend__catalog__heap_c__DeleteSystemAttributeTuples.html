DeleteSystemAttributeTuples

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/heap.c:1655

目的

指定されたリレーション（OID）に関連付けられているシステム属性（attnum <= 0）のタプルを pg_attribute カタログから削除します。

シグネチャまたは形状

void DeleteSystemAttributeTuples(Oid relid)

入力と出力

  • **入力**: relid (Oid) - 削除対象のシステム属性を持つリレーションのOID。
  • **出力**: なし（副作用として pg_attribute カタログが更新されます）。

前提条件と不変条件

  • 本関数はトランザクション内で呼び出される必要があります。
  • pg_attribute に対する RowExclusiveLock を取得します。

アルゴリズムまたは振る舞い

1. table_open を使用して pg_attribute を RowExclusiveLock モードで開きます。 2. ScanKey を設定し、対象リレーションの OID かつ属性番号が 0 以下のタプルを検索対象とします。 3. systable_beginscan を用いてインデックス（AttributeRelidNumIndexId）経由でスキャンを開始します。 4. systable_getnext で該当するタプルを順次取得し、CatalogTupleDelete を呼び出して削除します。 5. スキャン終了後、systable_endscan および table_close を実行してリソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていませんが、heap.c 内のテーブル削除処理に関連する文脈で使用されます。
  • **呼び出し先**: table_open, ScanKeyInit, systable_beginscan, systable_getnext, CatalogTupleDelete, systable_endscan, table_close。

関連データ構造

  • pg_attribute: 属性情報を保持するシステムカタログ。
  • ScanKeyData: スキャン条件を定義する構造体。
  • SysScanDesc: システムカタログスキャンの状態を保持する記述子。

失敗モードと境界条件

  • 対象の relid が存在しない場合、スキャンは何もヒットせず、削除処理も行われません。
  • カタログロックの競合が発生した場合、PostgreSQL の標準的なロック待機またはデッドロック検出メカニズムが働きます。

保守担当者向けメモ

  • 本関数はシステム属性のみを対象としています。ユーザー定義属性の削除には別の関数を使用してください。
  • pg_attribute のインデックススキャンに依存しているため、カタログのインデックス定義に変更を加える際は注意が必要です。
