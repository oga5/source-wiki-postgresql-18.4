AdvanceNextFullTransactionIdPastXid

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/varsup.c:303

目的

システム内の次期トランザクションID（nextXid）を、指定されたトランザクションID（xid）以降の値に強制的に進める。主にリカバリ処理や状態同期において、観測された最新のXIDに内部状態を追従させるために使用される。

シグネチャまたは形状

void AdvanceNextFullTransactionIdPastXid(TransactionId xid)

入力と出力

  • **入力**: xid (TransactionId) - 進めるべき基準となるトランザクションID。
  • **出力**: なし（副作用として共有メモリ上の TransamVariables->nextXid を更新）。

前提条件と不変条件

  • 呼び出し元は、スタートアッププロセスであるか、シングルプロセスモード（!IsUnderPostmaster）でなければならない（Assertによる検証）。
  • TransamVariables は初期化済みであること。

アルゴリズムまたは振る舞い

1. 現在の nextXid を取得し、指定された xid がそれより進んでいない場合は即座に終了する。 2. xid を TransactionIdAdvance マクロでインクリメントする。 3. 現在の epoch を保持しつつ、xid がラップアラウンドした（xid < next_xid）場合は epoch を加算する。 4. 計算された新しい FullTransactionId を生成する。 5. XidGenLock を排他モードで取得し、TransamVariables->nextXid を更新してロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/multixact.c
  • src/backend/access/transam/twophase.c
  • src/backend/access/transam/xact.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/storage/ipc/procarray.c
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease (排他制御)
  • TransactionIdAdvance (マクロ)
  • XidFromFullTransactionId / EpochFromFullTransactionId / FullTransactionIdFromEpochAndXid (型変換ヘルパー)

関連データ構造

  • TransamVariables: 共有メモリ上のグローバル状態を保持する構造体。
  • FullTransactionId: 64bitのトランザクションID（エポックと32bit XIDの組み合わせ）。
  • XidGenLock: nextXid の更新を保護する軽量ロック。

失敗モードと境界条件

  • **境界条件**: 指定された xid が現在の nextXid より小さい場合、状態は変更されない。
  • **ラップアラウンド**: 32bit XIDの境界を跨ぐ場合、エポックが正しくインクリメントされる設計となっている。

保守担当者向けメモ

  • 本関数は、WALリプレイやリカバリ時にシステム状態を整合させるための重要な役割を担う。
  • XidGenLock を取得する前に nextXid を読み取る処理は、スタートアッププロセス等の単一実行環境であることを前提としているため、マルチプロセス環境で呼び出す場合は注意が必要である。
