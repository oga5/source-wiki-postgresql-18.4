list_delete_cell

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/list.c:840

目的

指定されたリストから特定のセル（ListCell）を削除し、更新されたリストのポインタを返します。

シグネチャまたは形状

List *list_delete_cell(List *list, ListCell *cell);

入力と出力

  • **入力**:
  • list: 対象となる List 構造体へのポインタ。
  • cell: 削除対象の ListCell へのポインタ。
  • **出力**:
  • 削除後の新しい List 構造体へのポインタ。リストの再配置により元のポインタと異なる場合があります。

前提条件と不変条件

  • cell は必ず list 内の有効な要素を指している必要があります。
  • リストは配列ベースで実装されているため、削除操作に伴い内部配列のメモリ再配置が発生する可能性があります。

アルゴリズムまたは振る舞い

1. cell ポインタと list->elements の差分から、削除対象のインデックスを算出します。 2. 算出したインデックスを引数として list_delete_nth_cell を呼び出します。 3. list_delete_nth_cell が返す更新後のリストポインタをそのまま返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/foreigncmds.c:161
  • src/backend/optimizer/plan/analyzejoins.c:145
  • src/backend/nodes/list.c 内の各削除関数（list_delete_ptr, list_delete_int 等）
  • **呼び出し先**:
  • list_delete_nth_cell

関連データ構造

  • List: リストのヘッダ構造体。
  • ListCell: リスト内の各要素を指すポインタ。

失敗モードと境界条件

  • list が NULL の場合や、cell が当該リストの範囲外を指している場合の挙動は定義されていません（セグメンテーション違反の可能性があります）。

保守担当者向けメモ

  • 本関数は list_delete_nth_cell のラッパーです。インデックス計算に依存しているため、リストのメモリレイアウトが変更された場合は注意が必要です。
  • リスト操作中に ListCell へのポインタを保持し続けることは避けてください。配列の再配置によりポインタが無効化されるリスクがあります。
