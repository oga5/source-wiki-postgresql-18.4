ST_SWAPN

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/lib/sort_template.h:281

目的

ST_POINTER_TYPE 型の要素を保持する2つの配列間で、指定された個数（n）の要素を要素単位で交換する。

シグネチャまたは形状

static inline void ST_SWAPN(ST_POINTER_TYPE * a, ST_POINTER_TYPE * b, size_t n)

入力と出力

  • 入力:
  • a: 交換対象となる配列のポインタ。
  • b: 交換対象となるもう一方の配列のポインタ。
  • n: 交換する要素数。
  • 出力: なし（戻り値は void）。

前提条件と不変条件

  • a および b は、少なくとも n 個の要素を持つ有効なメモリ領域を指している必要がある。
  • ST_POINTER_TYPE および ST_SWAP マクロが、本関数を呼び出すコンテキストにおいて適切に定義されていること。

アルゴリズムまたは振る舞い

1. 0 から n-1 までのインデックス i についてループを実行する。 2. 各反復において、ST_SWAP(&a[i], &b[i]) を呼び出し、配列 a の i 番目の要素と配列 b の i 番目の要素を交換する。

呼び出し元と呼び出し先

  • 呼び出し先: ST_SWAP マクロ。
  • 呼び出し元:
  • DO_SWAPN マクロ（src/include/lib/sort_template.h:243, 248）を介して間接的に利用される。
  • 直接的な呼び出し元はソースコードの文脈からは特定できない。

関連データ構造

  • ST_POINTER_TYPE: ソート対象の要素型を指す型定義。

失敗モードと境界条件

  • n が 0 の場合、ループは実行されず何も行われない。
  • a または b が NULL である場合、あるいは n が指し示すメモリ領域のサイズを超えている場合、未定義動作を引き起こす。

保守担当者向けメモ

  • 本関数は sort_template.h のマクロ展開プロセスの一部として生成される。
  • ST_SWAP の実装に依存しているため、型ごとの交換コストは ST_SWAP の定義に左右される。
