gbt_intv_compress

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_interval.c:142

目的

GiST インデックスに格納する interval 型の値を、インデックスノード内で範囲を表現するための intvKEY 形式（またはその内部表現）に変換します。

シグネチャまたは形状

Datum gbt_intv_compress(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体を受け取ります。
  • **出力**: 変換後の GISTENTRY を Datum として返します。

前提条件と不変条件

  • INTERVALSIZE は 16 バイトに固定されています。
  • リーフノード（leafkey）または INTERVALSIZE が sizeof(Interval) と一致しない場合に変換処理が実行されます。

アルゴリズムまたは振る舞い

1. entry->leafkey が真の場合、interval 値を読み取り、その値を下限および上限として 2 * INTERVALSIZE のメモリ領域にコピーします。 2. entry->leafkey が偽（内部ノード）の場合、既存の intvKEY から下限および上限の値を読み取り、同様に 2 * INTERVALSIZE のメモリ領域にコピーします。 3. gistentryinit を使用して、変換されたデータを含む新しい GISTENTRY を初期化し、返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス構築プロセスから呼び出されます（直接的な呼び出し元関数の詳細はソースコード内に明示されていません）。
  • **呼び出し先**: palloc, DatumGetIntervalP, memcpy, DatumGetPointer, gistentryinit などの PostgreSQL 内部関数およびマクロ。

関連データ構造

  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。
  • intvKEY: インデックスノード内で範囲（下限・上限）を保持するための構造体。
  • Interval: PostgreSQL の interval 型。

失敗モードと境界条件

  • メモリ割り当て（palloc）が失敗した場合、PostgreSQL のエラーハンドリングにより中断されます。
  • INTERVALSIZE が sizeof(Interval) と異なる場合、強制的に変換処理が走る設計となっています。

保守担当者向けメモ

  • 本関数は btree_gist モジュールにおける interval 型の圧縮ロジックの基点です。
  • intvKEY 構造体のレイアウト変更や interval 型の内部表現に変更があった場合は、memcpy のオフセット計算に影響が出る可能性があるため注意してください。
