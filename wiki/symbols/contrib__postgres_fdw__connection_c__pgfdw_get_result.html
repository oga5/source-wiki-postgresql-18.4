pgfdw_get_result

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/connection.c:958

目的

リモートの PostgreSQL サーバーからクエリ実行結果（PGresult）を取得します。

シグネチャまたは形状

PGresult *pgfdw_get_result(PGconn *conn)

入力と出力

  • **入力**: conn (リモートサーバーへの接続を保持する PGconn ポインタ)
  • **出力**: 取得した結果を指す PGresult ポインタ。

前提条件と不変条件

  • conn は有効な PGconn オブジェクトである必要があります。
  • 内部的に libpqsrv_get_result_last を呼び出すため、その関数が要求する状態（pgfdw_we_get_result）が満たされている必要があります。

アルゴリズムまたは振る舞い

1. libpqsrv_get_result_last 関数を呼び出し、引数として conn と pgfdw_we_get_result を渡します。 2. リモートサーバーから返された結果をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/connection.c 内の複数箇所
  • contrib/postgres_fdw/postgres_fdw.c 内の複数箇所（クエリ実行結果の取得時）
  • **呼び出し先**:
  • libpqsrv_get_result_last

関連データ構造

  • PGconn (libpq の接続構造体)
  • PGresult (libpq の結果構造体)

失敗モードと境界条件

  • conn が無効な場合や、リモートサーバーとの通信が切断されている場合の挙動は、依存する libpqsrv_get_result_last の実装に依存します。

保守担当者向けメモ

  • 本関数は libpqsrv_get_result_last のラッパーです。結果取得のロジックを変更する場合は、pgfdw_we_get_result の定義および libpqsrv 側の仕様を確認してください。
