ForceTransactionIdLimitUpdate

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/varsup.c:516

目的

トランザクションID（XID）の枯渇を防ぐための制限データ（Wraparound制限）を更新する必要があるかどうかを判定します。

シグネチャまたは形状

bool ForceTransactionIdLimitUpdate(void)

入力と出力

  • **入力**: なし（共有メモリ上の TransamVariables を参照）。
  • **出力**: bool 型。更新が必要な場合は true、不要な場合は false を返します。

前提条件と不変条件

  • XidGenLock を共有モードで取得し、共有メモリ上の状態を読み取ります。
  • oldestXid が正常な値であること、および xidVacLimit が有効であることが前提です。

アルゴリズムまたは振る舞い

1. XidGenLock を取得し、TransamVariables から nextXid、xidVacLimit、oldestXid、oldestXidDB を読み取ります。 2. 以下のいずれかの条件に該当する場合、更新が必要と判断し true を返します。

  • oldestXid が正常ではない。
  • xidVacLimit が無効である。
  • 現在の nextXid が xidVacLimit 以上に達している。
  • oldestXidDB に対応するデータベースがシステムカタログ（pg_database）に存在しない。

    • 3. 上記のいずれにも該当しない場合、false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/vacuum.c:1819
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease (共有メモリ保護)
  • XidFromFullTransactionId (変換)
  • TransactionIdIsNormal / TransactionIdIsValid / TransactionIdFollowsOrEquals (判定マクロ)
  • SearchSysCacheExists1 (カタログ検索)

関連データ構造

  • TransamVariables: ID生成状態を保持する共有メモリ構造体。

失敗モードと境界条件

  • oldestXid が異常な値である場合や、データベースが削除された直後などでカタログ検索が失敗する場合、安全側に倒して true を返します。

保守担当者向けメモ

  • 本関数は VACUUM 処理の判断基準として利用されます。
  • XidGenLock の取得は最小限の範囲で行われていますが、頻繁に呼び出される可能性があるため、処理の追加には注意が必要です。
