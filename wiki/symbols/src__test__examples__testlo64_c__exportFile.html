exportFile

定義

  • 種別: function
  • 定義箇所: src/test/examples/testlo64.c:172

目的

PostgreSQL のラージオブジェクトの内容を、指定されたローカルの Unix ファイルへエクスポートする。

シグネチャまたは形状

static void exportFile(PGconn *conn, Oid lobjId, char *filename)

入力と出力

  • **入力**:
  • conn: データベース接続を保持する PGconn オブジェクト。
  • lobjId: エクスポート対象のラージオブジェクトの Oid。
  • filename: 書き出し先のファイルパスを示す文字列。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • conn は有効なデータベース接続であること。
  • lobjId はデータベース内に存在する有効なラージオブジェクトであること。
  • プロセスが filename に対して書き込み権限を持つこと。

アルゴリズムまたは振る舞い

1. lo_open を使用し、lobjId を読み取り専用モード (INV_READ) で開く。 2. open を使用し、filename を作成または上書きモード (O_CREAT | O_WRONLY | O_TRUNC) で開く。 3. lo_read を用いてラージオブジェクトから BUFSIZE 単位でデータを読み込み、write を用いてローカルファイルへ順次書き出す。 4. 読み込みが完了するまでループを継続する。 5. 最後に lo_close でラージオブジェクトを閉じ、close でローカルファイルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/examples/testlo64.c および src/test/examples/testlo.c 内で参照されている（ただし、提供されたソース内ではコメントアウトされている箇所が主であり、直接的な実行フローの詳細は不明）。
  • **呼び出し先**: lo_open, open, lo_read, write, lo_close, close, fprintf。

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • Oid: PostgreSQL のオブジェクト識別子。
  • BUFSIZE: 読み込みバッファサイズ（定義値はソース内に依存）。

失敗モードと境界条件

  • lo_open が失敗した場合（戻り値が 0 未満）、エラーメッセージを標準エラー出力に出力する。
  • open が失敗した場合、エラーメッセージを標準エラー出力に出力する。
  • write による書き込みバイト数が nbytes に満たない場合、エラーメッセージを標準エラー出力に出力する。

保守担当者向けメモ

  • 本関数は libpq のラージオブジェクト API を使用した基本的なファイル転送処理である。
  • エラー発生時に exit_nicely 等でプログラムを終了させる処理は含まれておらず、エラーメッセージ出力後も処理が継続される点に注意すること。
