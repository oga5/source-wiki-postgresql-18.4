pathkey_is_redundant

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/pathkeys.c:158

目的

与えられた PathKey が、既存の PathKey リストに対して冗長であるかどうかを判定します。冗長と判定された場合、そのパスキーはリストに追加されません。

シグネチャまたは形状

static bool pathkey_is_redundant(PathKey *new_pathkey, List *pathkeys)

入力と出力

  • **入力**:
  • new_pathkey: 判定対象の PathKey 構造体へのポインタ。
  • pathkeys: 既存の PathKey を格納した List。
  • **出力**:
  • bool: 冗長であれば true、そうでなければ false を返します。

前提条件と不変条件

  • new_pathkey および pathkeys 内の各要素は、正規化（Canonical）されている必要があります。
  • EquivalenceClass（EC）のポインタ比較によって意味的な同一性を判定するため、正規化が前提となります。

アルゴリズムまたは振る舞い

1. **定数判定**: new_pathkey が属する EquivalenceClass が定数を含む場合、無条件に冗長とみなして true を返します（EC_MUST_BE_REDUNDANT マクロを使用）。 2. **リスト走査**: pathkeys リストを順次走査し、各要素の pk_eclass と new_pathkey->pk_eclass を比較します。 3. **一致判定**: ポインタが一致（同一の EquivalenceClass）すれば冗長とみなし true を返します。 4. **終了**: リストの最後まで一致が見つからなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/pathkeys.c 内の以下の箇所から呼び出されています。
  • 117, 801, 961, 1232, 1431, 1826, 1928 行目。
  • **呼び出し先**:
  • EC_MUST_BE_REDUNDANT マクロ（詳細な定義は本抜粋外）。
  • lfirst マクロ（リスト要素へのアクセス）。

関連データ構造

  • PathKey: 順序を表現する構造体。pk_eclass フィールドを介して EquivalenceClass を参照します。
  • EquivalenceClass: 意味的に等価な式の集合。

失敗モードと境界条件

  • pathkeys が空リストの場合、常に false を返します。
  • new_pathkey が NULL の場合の挙動は定義されていません（セグメンテーション違反の可能性があります）。

保守担当者向けメモ

  • 本関数はポインタ比較による高速な判定を行っています。もし EquivalenceClass の正規化ルールが変更された場合、この判定ロジックも影響を受ける可能性があります。
  • EC_MUST_BE_REDUNDANT は定数を含む等価クラスを冗長とみなすため、定数伝播後の最適化において重要な役割を果たします。
