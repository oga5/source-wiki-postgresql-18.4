sampler_random_fract

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/sampling.c:240

目的

0から1の範囲（0を含まない）の一様な乱数を生成します。対数計算など、0が入力されるとエラーや不整合を引き起こす計算処理の入力値として利用されます。

シグネチャまたは形状

double sampler_random_fract(pg_prng_state *randstate)

入力と出力

  • **入力**: pg_prng_state *randstate （乱数生成器の状態を保持する構造体へのポインタ）
  • **出力**: double 型の乱数値（範囲: 0.0 < result < 1.0）

前提条件と不変条件

  • randstate は初期化済みである必要があります。
  • 戻り値は常に 0.0 より大きいことが保証されます。

アルゴリズムまたは振る舞い

1. pg_prng_double(randstate) を呼び出し、[0.0, 1.0) の範囲の乱数を取得します。 2. 取得した値が 0.0 である場合、ループして再度乱数を生成します。 3. 0.0 以外の値が得られた時点でその値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/sampling.c 内の各サンプリングロジック（BlockSampler関連処理やリザーバーサンプリング処理）
  • src/backend/commands/analyze.c
  • contrib/file_fdw/file_fdw.c
  • contrib/postgres_fdw/postgres_fdw.c
  • contrib/tsm_system_rows/tsm_system_rows.c
  • contrib/tsm_system_time/tsm_system_time.c
  • **呼び出し先**:
  • pg_prng_double （src/include/common/pg_prng.h で定義される乱数生成関数）

関連データ構造

  • pg_prng_state: 乱数生成器の状態を保持する構造体。

失敗モードと境界条件

  • randstate が NULL の場合、pg_prng_double の内部実装に依存した動作（通常はセグメンテーション違反）となります。
  • 乱数生成器が極めて低い確率で 0.0 を返し続ける場合、理論上は無限ループに陥る可能性がありますが、pg_prng の特性上、実用上の問題とはなりません。

保守担当者向けメモ

  • 本関数は 0.0 を除外する設計です。対数計算（log()）を行う箇所で 0.0 が渡されることを防ぐための重要なガードとなっています。
  • 乱数生成器の変更や更新を行う際は、pg_prng_double の戻り値範囲が変更されていないか確認してください。
