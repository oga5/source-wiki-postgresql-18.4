PrepareQuery

定義

  • 種別: function
  • 定義箇所: src/backend/commands/prepare.c:58

目的

SQLの PREPARE 文を処理し、指定されたクエリを解析・書き換えした上で、セッション内で再利用可能なプリペアドステートメントとしてキャッシュに保存します。

シグネチャまたは形状

void PrepareQuery(ParseState *pstate, PrepareStmt *stmt, int stmt_location, int stmt_len)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態。
  • stmt: PREPARE 文の構文解析木。
  • stmt_location: クエリ文字列内での開始位置。
  • stmt_len: クエリ文字列の長さ。
  • **出力**: なし（副作用としてセッション内の prepared_queries ハッシュテーブルを更新）。

前提条件と不変条件

  • stmt->name が空文字列であってはならない（プロトコルレベルの無名ステートメントと競合するため）。
  • pstate は有効な解析状態であること。

アルゴリズムまたは振る舞い

1. **検証**: ステートメント名が空でないことを確認し、空の場合はエラーを報告する。 2. **ノード作成**: 内部処理のため、対象クエリを RawStmt ノードでラップする。 3. **キャッシュ準備**: CreateCachedPlan を呼び出し、解析前の状態を保持する CachedPlanSource を作成する。 4. **型変換**: stmt->argtypes に含まれる型名を Oid の配列に変換する。 5. **解析と書き換え**: pg_analyze_and_rewrite_varparams を使用して、クエリの解析と書き換えを実行する。 6. **キャッシュ完了**: CompleteCachedPlan を呼び出し、解析結果を CachedPlanSource に格納する。 7. **保存**: StorePreparedStatement を呼び出し、名前付きステートメントとしてハッシュテーブルに登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c (SQLコマンド実行のディスパッチ経由)
  • **呼び出し先**:
  • CreateCachedPlan
  • typenameTypeId
  • pg_analyze_and_rewrite_varparams
  • CompleteCachedPlan
  • StorePreparedStatement

関連データ構造

  • PrepareStmt: PREPARE 文の構文解析結果を保持する構造体。
  • CachedPlanSource: プリペアドステートメントの計画キャッシュの基盤となる構造体。
  • prepared_queries: セッション内でプリペアドステートメントを管理するハッシュテーブル。

失敗モードと境界条件

  • ステートメント名が空の場合、ERRCODE_INVALID_PSTATEMENT_DEFINITION エラーが発生する。
  • 型名の解決失敗やクエリの解析・書き換えエラーが発生した場合、ereport(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は CachedPlanSource のライフサイクル管理に深く関与しています。
  • pg_analyze_and_rewrite_varparams を使用しているため、パラメータの型情報が不明な場合でもコンテキストから推論が試みられます。
  • 実行時のパラメータ評価は EvaluateParams が担当するため、本関数は計画の生成と保存に特化しています。
