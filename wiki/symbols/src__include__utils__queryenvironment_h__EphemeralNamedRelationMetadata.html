EphemeralNamedRelationMetadata

定義

  • 種別: typedef
  • 定義箇所: src/include/utils/queryenvironment.h:44

目的

カタログに存在しない一時的な名前付きリレーション（ENR）のメタデータを保持し、クエリ計画や実行時にリレーションのスキーマ情報（TupleDesc）を解決するためのインターフェースを提供します。

シグネチャまたは形状

typedef EphemeralNamedRelationMetadataData *EphemeralNamedRelationMetadata;

EphemeralNamedRelationMetadataData 構造体へのポインタ型です。

入力と出力

  • **入力**: 本型を引数として受け取る関数（例: ENRMetadataGetTupDesc）は、ENR のメタデータ構造体を参照します。
  • **出力**: ENRMetadataGetTupDesc を通じて、リレーションの行構造を定義する TupleDesc を返します。

前提条件と不変条件

  • QueryEnvironment 内で管理される ENR の一部として存在します。
  • カタログ外のリレーションを表現するため、必要に応じてカタログ参照を伴う動的な解決が行われます。

アルゴリズムまたは振る舞い

  • 本型は、リレーション名、型、および TupleDesc（またはそれを導出するための OID）を保持します。
  • ENRMetadataGetTupDesc() 関数に渡されることで、クエリの型チェックや計画作成に必要なスキーマ情報を取得する起点となります。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/parser/parse_enr.c
  • src/backend/parser/parse_relation.c
  • src/backend/utils/misc/queryenvironment.c
  • src/include/parser/parse_enr.h
  • **呼び出し先**:
  • 本型を引数として受け取る関数として ENRMetadataGetTupDesc が定義されています。

関連データ構造

  • EphemeralNamedRelationMetadataData: 本型が指し示す実体。
  • QueryEnvironment: ENR を保持するコンテナ。
  • TupleDesc: src/include/access/tupdesc.h で定義される、リレーションの行構造情報。

失敗モードと境界条件

  • 該当する ENR が環境内に存在しない場合、検索関数（get_visible_ENR_metadata 等）は適切なエラーまたは NULL を返します。
  • メタデータから TupleDesc を導出できない場合、カタログ参照の失敗や型解決エラーが発生する可能性があります。

保守担当者向けメモ

  • QueryEnvironment は不透明な型として設計されており、本メタデータへのアクセスは提供された API を介して行うことが推奨されます。
  • 現在の実装はリストベースの検索であるため、登録数が多い環境ではパフォーマンス上の制約を受ける可能性があります。
