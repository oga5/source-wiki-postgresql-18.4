目的

PostgreSQL のデータベースエンコーディング変換サブシステムにおいて、SHIFT_JIS_2004 から EUC_JIS_2004 への文字コード変換を実行する。

シグネチャまたは形状

Datum shift_jis_2004_to_euc_jis_2004(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を受け取る。
  • src: 変換元文字列（CSTRING）
  • dest: 変換先バッファ（CSTRING）
  • len: 変換対象の長さ（INT32）
  • noError: 不正なバイト列に対するエラー報告を抑制するかどうかのフラグ（BOOL）
  • **出力**: 変換されたバイト数（INT32）を PG_RETURN_INT32 で返す。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_SHIFT_JIS_2004, PG_EUC_JIS_2004) マクロにより、エンコーディング ID が正しく設定されている必要がある。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得する。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングの整合性を検証する。 3. 変換ロジック本体である shift_jis_20042euc_jis_2004 を呼び出し、変換を実行する。 4. 変換されたバイト数を戻り値として返す。

失敗モードと境界条件

  • 不正なバイトシーケンスが入力された場合、noError フラグが false であればエラーを報告する。true の場合は変換を中断する。
