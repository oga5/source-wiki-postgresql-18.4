sepgsql_needs_fmgr_hook

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/label.c:267

目的

指定された関数（functionId）の実行において、SELinux のフック（sepgsql_fmgr_hook）による介入が必要かどうかを判定する。

シグネチャまたは形状

static bool sepgsql_needs_fmgr_hook(Oid functionId)

入力と出力

  • **入力**: functionId (Oid) - 判定対象となる関数の識別子。
  • **出力**: bool - フックが必要な場合は true、不要な場合は false を返す。

前提条件と不変条件

  • 外部の next_needs_fmgr_hook が設定されている場合、その判定結果が優先される。
  • 権限チェックは ProcedureRelationId を対象として行われる。

アルゴリズムまたは振る舞い

1. **外部フックの確認**: next_needs_fmgr_hook が存在し、かつ true を返す場合は即座に true を返す。 2. **信頼されたプロシージャの確認**: sepgsql_avc_trusted_proc(functionId) が非 NULL を返す場合、ドメイン遷移を伴う信頼されたプロシージャと見なし true を返す。 3. **権限チェック**: db_procedure:{execute} および db_procedure:{entrypoint} 権限を sepgsql_avc_check_perms で確認する。権限がない（チェックが失敗する）場合は、後続の実行失敗を予期して true を返す。 4. 上記いずれにも該当しない場合、false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/label.c:427 (初期化時にフック関数として登録)
  • **呼び出し先**:
  • next_needs_fmgr_hook (外部フック)
  • sepgsql_avc_trusted_proc
  • sepgsql_avc_check_perms

関連データ構造

  • Oid: 関数識別子。
  • ObjectAddress: 権限チェック対象のオブジェクト情報を保持。
  • ProcedureRelationId: pg_proc カタログを参照するための定数。

失敗モードと境界条件

  • sepgsql_avc_check_perms が権限不足を検出した場合、関数実行自体は後続の処理で失敗するが、本関数はフックを有効にするために true を返す。

保守担当者向けメモ

  • 本関数は、関数実行時のセキュリティドメイン遷移や権限検証のトリガーを制御する重要な判定ロジックである。
  • 権限チェックの失敗時に true を返す仕様は、実行時のエラーハンドリングをフック経由で一貫させるための設計である。
