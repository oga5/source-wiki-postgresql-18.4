AsyncShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/commands/async.c:500

目的

非同期通知機能（NOTIFY等）で使用する共有メモリ上の制御構造体（AsyncQueueControl）の確保・初期化、およびSLRU（Simple Least Recently Used）バッファの初期化を行う。

シグネチャまたは形状

void AsyncShmemInit(void)

入力と出力

  • 入力: なし（グローバルな共有メモリ状態に依存）
  • 出力: なし（副作用として共有メモリ領域の確保と初期化を実行）

前提条件と不変条件

  • PostgreSQLの共有メモリ初期化プロセスの一部として呼び出される必要がある。
  • MaxBackends が定義されている環境で実行される。

アルゴリズムまたは振る舞い

1. AsyncQueueControl 構造体に必要な共有メモリサイズを計算し、ShmemInitStruct を用いて確保または既存領域へのアタッチを行う。 2. 共有メモリが新規作成された場合（found == false）、AsyncQueueControl 内のヘッド・テール位置、リスナーリスト、および各バックエンドの状態を初期値でリセットする。 3. NotifyCtl（SLRU制御構造体）に対して asyncQueuePagePrecedes をページ比較関数として設定し、SimpleLruInit を呼び出してSLRUバッファを初期化する。 4. 共有メモリが新規作成された場合、SlruScanDirectory を使用して pg_notify ディレクトリ内の既存ファイルをすべて削除し、クリーンな状態から開始する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c（共有メモリ初期化シーケンス内）
  • 呼び出し先: ShmemInitStruct, SimpleLruInit, SlruScanDirectory, asyncQueuePagePrecedes

関連データ構造

  • AsyncQueueControl: 通知キューの全体管理構造体。
  • QueueBackendStatus: 各バックエンドの通知状態を保持する構造体。
  • NotifyCtl: SLRU管理用の制御構造体。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、PostgreSQLの起動プロセスが停止する可能性がある。
  • found フラグにより、再起動時と初回起動時の処理を分岐させており、初回起動時のみディレクトリのクリーンアップを行う。

保守担当者向けメモ

  • pg_notify ディレクトリのクリーンアップは、システム起動時または再起動時にのみ行われる。
  • NotifyCtl の初期化時に使用される LWTRANCHE_NOTIFY_BUFFER および LWTRANCHE_NOTIFY_SLRU は、ロック管理において重要である。
