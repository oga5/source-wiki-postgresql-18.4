PipeProtoHeader

定義

  • 種別: struct
  • 定義箇所: src/include/postmaster/syslogger.h:44

目的

PostgreSQLのsysloggerプロセスにおいて、パイプ経由で転送されるログメッセージのチャンク構造を定義し、非同期的なログ書き込みの整合性とメッセージの再構築を可能にする。

シグネチャまたは形状

typedef struct
{
	char		nuls[2];
	uint16		len;
	int32		pid;
	bits8		flags;
	char		data[FLEXIBLE_ARRAY_MEMBER];
} PipeProtoHeader;

入力と出力

  • **入力**: 各バックエンドプロセスがログメッセージを分割し、本ヘッダーを付与してパイプへ書き込む。
  • **出力**: Sysloggerプロセスがパイプから読み取り、pid ごとにバッファリングして元のメッセージを再構築する。

前提条件と不変条件

  • PIPE_BUF の制限を考慮し、アトミックな書き込みを保証する設計となっている。
  • nuls フィールドは常に \0\0 である必要がある。
  • data フィールドは可変長配列（FLEXIBLE_ARRAY_MEMBER）として定義されている。

アルゴリズムまたは振る舞い

1. 送信側はログメッセージを PIPE_CHUNK_SIZE 以下のチャンクに分割する。 2. 各チャンクに PipeProtoHeader を付与し、パイプへ送信する。 3. 受信側（syslogger）はパイプを監視し、ヘッダーを解析して pid を特定する。 4. 特定した pid に基づき、受信データをバッファへ蓄積してメッセージを再構築する。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/postmaster/syslogger.c: ログ受信ループ内でのヘッダー解析およびデータコピーに使用。
  • src/include/postmaster/syslogger.h: PIPE_HEADER_SIZE マクロの定義や、他の構造体（PipeProtoChunk 等）のメンバとして使用。
  • **呼び出し先**:
  • 本構造体自体はデータ定義であり、特定の関数を呼び出すものではない。

関連データ構造

  • PipeProtoChunk: 本ヘッダーを含むログ転送用チャンク構造体。
  • PIPE_HEADER_SIZE: offsetof(PipeProtoHeader, data) として定義されるヘッダーサイズ。

失敗モードと境界条件

  • PIPE_CHUNK_SIZE を超えるデータ転送は、プロトコル上の整合性を損なう可能性がある。
  • nuls フィールドが期待値と異なる場合、プロトコルの同期が取れなくなる可能性がある。

保守担当者向けメモ

  • 本構造体はPOSIXの PIPE_BUF 制限に依存している。メッセージの分割・結合ロジックを変更する際は、構造体のパディングやアライメントに注意すること。
  • EXEC_BACKEND 環境（Windows等）では、プロセス間でのパイプ共有に関連するため、構造の変更は慎重に行う必要がある。
