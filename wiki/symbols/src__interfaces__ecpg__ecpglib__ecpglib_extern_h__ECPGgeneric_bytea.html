ECPGgeneric_bytea

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/ecpglib/ecpglib_extern.h:42

目的

ECPGライブラリにおいて、バイナリデータ（bytea型）を保持するための汎用的な構造体を提供します。

シグネチャまたは形状

struct ECPGgeneric_bytea
{
    int         len;
    char        arr[FLEXIBLE_ARRAY_MEMBER];
};

入力と出力

  • **入力**: 構造体の len メンバにバイナリデータの長さを格納し、arr メンバに実際のバイナリデータを保持します。
  • **出力**: len を参照することで、格納されているバイナリデータのサイズを取得可能です。

前提条件と不変条件

  • arr は FLEXIBLE_ARRAY_MEMBER を使用した可変長配列として定義されており、構造体の末尾に配置されます。
  • len は格納されているバイナリデータのバイト数を表します。

アルゴリズムまたは振る舞い

  • ECPGランタイムにおいて、SQLの bytea 型とC言語のホスト変数間でのデータ受け渡しに使用されます。
  • 実行時に len を参照してバイナリデータの長さを特定し、arr を通じてデータ本体にアクセスします。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/interfaces/ecpg/ecpglib/data.c: 変数へのキャストおよびアクセス。
  • src/interfaces/ecpg/ecpglib/descriptor.c: 記述子内の値の取得。
  • src/interfaces/ecpg/ecpglib/execute.c: SQL実行時のバイナリデータ長取得および値の参照。
  • src/interfaces/ecpg/ecpglib/misc.c: 構造体の初期化および長さの検証。
  • **呼び出し先**: 本構造体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • variable: ホスト変数とSQL変数のマッピングを保持する構造体。本構造体は variable->value を通じて参照されることが多いです。

失敗モードと境界条件

  • len が 0 の場合、データが空であることを示します（misc.c での検証ロジックが存在）。
  • 可変長配列であるため、メモリ割り当て時に len に基づく適切なサイズ計算が行われない場合、バッファオーバーフローのリスクがあります。

保守担当者向けメモ

  • 本構造体は ecpglib 内部でバイナリデータを扱う際の標準的なコンテナです。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体自体のサイズではなく、動的に確保されたメモリ領域のサイズに注意して操作してください。
