pcb_error_callback

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_node.c:169

目的

SQL 解析中にエラーが発生した際、エラー発生箇所を特定し、ユーザーに通知するためのエラーコールバック関数です。

シグネチャまたは形状

static void pcb_error_callback(void *arg);

入力と出力

  • **入力**: arg - ParseCallbackState 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • arg は ParseCallbackState 型としてキャスト可能である必要があります。
  • ParseCallbackState 内の pstate および location が有効な状態である必要があります。

アルゴリズムまたは振る舞い

1. 引数 arg を ParseCallbackState * にキャストします。 2. 現在のエラーコードが ERRCODE_QUERY_CANCELED（クエリキャンセル）でないことを確認します。 3. クエリキャンセルでない場合、parser_errposition を呼び出し、pstate と location を用いてエラー位置を報告します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_node.c 内で、errcallback.callback に関数ポインタとして設定されます。
  • **呼び出し先**:
  • geterrcode() (PostgreSQL エラーハンドリング基盤)
  • parser_errposition() (エラー位置報告用関数)

関連データ構造

  • ParseCallbackState: エラー報告に必要な ParseState とソーステキスト上の位置情報 (location) を保持する構造体。

失敗モードと境界条件

  • クエリがキャンセルされている場合（ERRCODE_QUERY_CANCELED）、エラー位置の報告はスキップされます。

保守担当者向けメモ

  • 本関数は setup_parser_errposition_callback を介してエラーコンテキストに登録されることを前提としています。直接呼び出すことは想定されていません。
