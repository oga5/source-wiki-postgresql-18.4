getRelationDescription

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaddress.c:4102

目的

指定されたリレーション（テーブル、インデックス、ビュー等）のOIDに基づき、その種類と修飾名を含む人間が読みやすい説明文を生成し、提供された StringInfo バッファに追記します。

シグネチャまたは形状

static void getRelationDescription(StringInfo buffer, Oid relid, bool missing_ok)

入力と出力

  • **入力**:
  • buffer: 説明文を格納する StringInfo へのポインタ。
  • relid: 対象リレーションのOID。
  • missing_ok: true の場合、リレーションが見つからなくてもエラーにせず終了する。false の場合はエラーを発生させる。
  • **出力**: なし（buffer を直接更新）。

前提条件と不変条件

  • relid に対応する pg_class エントリがシステムカタログに存在する必要があります（missing_ok が false の場合）。
  • 呼び出し側は、この関数が内部で RELOID システムキャッシュを使用し、終了時に ReleaseSysCache を呼び出して解放することを前提とします。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用して RELOID キャッシュからリレーションのタプルを取得します。 2. タプルが見つからない場合、missing_ok に応じてエラーを投げるか、そのままリターンします。 3. RelationIsVisible を用いて、リレーションが現在の検索パスで可視かどうかを判定します。可視でない場合は get_namespace_name でスキーマ名を取得します。 4. quote_qualified_identifier を使用して、スキーマ名とリレーション名を適切に引用符で囲んだ文字列を生成します。 5. relkind（リレーションの種類）に基づき、appendStringInfo を使用して「table」「index」「view」などの接頭辞を付与した説明文をバッファに書き込みます。 6. 最後に ReleaseSysCache を呼び出し、取得したタプルを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/objectaddress.c 内の複数の箇所から、オブジェクトの識別や説明生成の文脈で呼び出されています（例: getObjectDescription 関連の処理）。
  • **呼び出し先**: SearchSysCache1, HeapTupleIsValid, GETSTRUCT, RelationIsVisible, get_namespace_name, quote_qualified_identifier, appendStringInfo, ReleaseSysCache。

関連データ構造

  • StringInfo: 文字列構築用のバッファ構造体。
  • Form_pg_class: pg_class システムカタログの行構造体。
  • Oid: オブジェクト識別子。

失敗モードと境界条件

  • missing_ok が false で、かつ relid が存在しない場合、elog(ERROR, ...) によりトランザクションが中断されます。
  • relkind が定義されたケースに該当しない場合、デフォルトとして「relation」という汎用的な名称が使用されます。

保守担当者向けメモ

  • 新しいリレーションタイプが pg_class に追加された場合は、switch 文に新しい case を追加して適切な説明文を定義してください。
  • 本関数はシステムキャッシュを操作するため、キャッシュの整合性が保たれている環境で呼び出される必要があります。
