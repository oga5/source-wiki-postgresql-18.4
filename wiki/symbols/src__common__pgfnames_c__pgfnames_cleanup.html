pgfnames_cleanup

定義

  • 種別: function
  • 定義箇所: src/common/pgfnames.c:85

目的

pgfnames 関数によって動的に確保されたファイル名リスト（文字列配列）および各文字列要素を解放し、メモリリークを防止する。

シグネチャまたは形状

void pgfnames_cleanup(char **filenames)

入力と出力

  • **入力**: filenames - pgfnames が返した char* の配列（NULL 終端）。
  • **出力**: なし（void）。

前提条件と不変条件

  • filenames は pgfnames によって確保された有効なポインタであること。
  • リストは NULL ポインタで終端されていること。
  • 本関数呼び出し後、filenames が指すメモリ領域および各文字列要素へのアクセスは無効となる。

アルゴリズムまたは振る舞い

1. filenames が指す配列を走査する。 2. 各要素（ファイル名文字列）に対して pfree を実行する。 3. 最後に配列自体に対して pfree を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/initdb/findtimezone.c:725 での呼び出しが確認されている。
  • **呼び出し先**: pfree（メモリ解放用ユーティリティ）。

関連データ構造

  • char **: pgfnames が生成する動的配列。

失敗モードと境界条件

  • filenames が NULL の場合、あるいは既に解放済みのポインタを渡した場合の挙動は未定義（pfree の仕様に依存）。

保守担当者向けメモ

  • pgfnames を使用するすべての箇所で、本関数による解放が必須である。
  • pfree を使用しているため、PostgreSQL のメモリコンテキスト管理下で動作する。
