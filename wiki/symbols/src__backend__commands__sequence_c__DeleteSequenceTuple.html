DeleteSequenceTuple

定義

  • 種別: function
  • 定義箇所: src/backend/commands/sequence.c:569

目的

システムカタログから特定のシーケンス定義（タプル）を物理的に削除する。

シグネチャまたは形状

void DeleteSequenceTuple(Oid relid)

入力と出力

  • **入力**: relid (削除対象シーケンスのOID)
  • **出力**: なし（成功時）。失敗時はエラーをスローする。

前提条件と不変条件

  • SequenceRelationId（pg_sequenceカタログ）に対するRowExclusiveLockが取得可能であること。
  • 指定されたrelidに対応するタプルがシステムキャッシュ（SEQRELID）に存在すること。

アルゴリズムまたは振る舞い

1. table_open を使用して pg_sequence カタログを RowExclusiveLock モードで開く。 2. SearchSysCache1 を用いて、指定された relid に対応するタプルを検索する。 3. タプルが見つからない場合は elog(ERROR) を発生させる。 4. CatalogTupleDelete を呼び出し、カタログから該当タプルを削除する。 5. ReleaseSysCache でキャッシュエントリを解放し、table_close でカタログリレーションを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/dependency.c:1383（オブジェクト削除処理の一部として参照されている）。
  • **呼び出し先**: table_open, SearchSysCache1, CatalogTupleDelete, ReleaseSysCache, table_close。

関連データ構造

  • Relation: シーケンスカタログへのアクセスに使用。
  • HeapTuple: カタログ内のシーケンス定義行を保持。
  • SEQRELID: システムキャッシュの検索キー。

失敗モードと境界条件

  • 指定された relid が存在しない場合、elog(ERROR) によりトランザクションがアボートされる。

保守担当者向けメモ

  • 本関数はカタログ上の定義を削除するのみであり、物理的なシーケンスファイル（ストレージ）の削除は呼び出し元または別のメカニズムで行われる必要がある。
  • pg_sequence カタログの構造変更時には、SearchSysCache1 のキーや CatalogTupleDelete の引数に影響が出る可能性がある。
