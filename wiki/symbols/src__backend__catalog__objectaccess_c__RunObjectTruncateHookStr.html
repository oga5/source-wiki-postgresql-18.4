RunObjectTruncateHookStr

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaccess.c:201

目的

データベースオブジェクトの切り詰め（Truncate）操作が発生した際、名前ベースで外部のフック関数を呼び出し、監視や介入を行うためのエントリーポイントを提供します。

シグネチャまたは形状

void RunObjectTruncateHookStr(const char *objectName)

入力と出力

  • **入力**: objectName (const char *) - 切り詰め対象となるオブジェクトの名前。
  • **出力**: なし（void）。

前提条件と不変条件

  • グローバル変数 object_access_hook_str が非NULLである必要があります。関数内で Assert(object_access_hook_str != NULL) により検証されます。

アルゴリズムまたは振る舞い

1. object_access_hook_str が設定されていることを確認します。 2. 登録されたフック関数に対し、以下の引数で呼び出しを行います。

  • イベントタイプ: OAT_TRUNCATE
  • クラスID: RelationRelationId
  • オブジェクト名: objectName
  • 追加フラグ: 0
  • 補助データ: NULL

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は、提供された情報からは特定できません。
  • **呼び出し先**: グローバル関数ポインタ object_access_hook_str に登録された外部プラグイン関数。

関連データ構造

  • OAT_TRUNCATE: catalog/objectaccess.h で定義される ObjectAccessType 列挙型の一つ。
  • RelationRelationId: catalog/pg_class.h で定義されるカタログOID定数。

失敗モードと境界条件

  • object_access_hook_str が NULL の状態で呼び出された場合、アサーション違反によりプロセスが停止します（デバッグビルド時）。

保守担当者向けメモ

  • 本関数は名前ベースのフック呼び出し用です。OIDが確定している場合は、OIDベースの RunObjectTruncateHook を使用してください。
  • フック関数側で発生したエラーや副作用は、本関数を介して呼び出し元へ伝播します。
