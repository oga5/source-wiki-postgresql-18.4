r_residual_suffix

目的

カタルーニャ語のステミング処理において、単語末尾に残存する特定の接尾辞を検索し、条件に応じて削除または置換を行う。

シグネチャまたは形状

static int r_residual_suffix(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（ステミング環境構造体）。現在の単語データおよびカーソル位置（c）を含む。
  • **出力**: int 型。処理が成功した場合は 1、接尾辞が見つからない場合は 0、エラー発生時は負の値を返す。

前提条件と不変条件

  • z->c は単語の末尾を指している必要がある。
  • find_among_b を使用するため、単語は後方から前方へスキャンされる。
  • 接尾辞の削除または置換には、r_R1 関数による領域チェックが必須である。

アルゴリズムまたは振る舞い

1. z->ket を現在のカーソル位置 z->c に設定する。 2. a_4 テーブル（22個の要素）を用いて、後方から接尾辞を検索する。 3. 接尾辞が見つからない場合は 0 を返して終了する。 4. 接尾辞が見つかった場合、z->bra を現在のカーソル位置に設定する。 5. among_var の値に基づき分岐する:

  • **ケース 1**: r_R1 を確認後、slice_del で接尾辞を削除する。
  • **ケース 2**: r_R1 を確認後、slice_from_s を用いて接尾辞を s_9 に置換する。

    • 6. 成功時に 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: catalan_ISO_8859_1_stem（ステミング処理のメインフロー内）。
  • **呼び出し先**: find_among_b, r_R1, slice_del, slice_from_s。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • a_4: 接尾辞マッチング用の among 構造体テーブル。
  • s_9: 置換対象となる文字列定数。

失敗モードと境界条件

  • r_R1 が失敗した場合、関数は r_R1 の戻り値（負または0）をそのまま返す。
  • slice_del または slice_from_s が失敗した場合、負の値を返す。
  • among_var が 0 の場合（マッチなし）、何も変更せず 0 を返す。
