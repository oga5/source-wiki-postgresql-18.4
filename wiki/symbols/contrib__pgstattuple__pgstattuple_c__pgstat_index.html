pgstat_index

定義

  • 種別: function
  • 定義箇所: contrib/pgstattuple/pgstattuple.c:536

目的

インデックスリレーションを走査し、ライブ/デッドタプルの統計情報を算出するための汎用的なループ処理を提供します。

シグネチャまたは形状

static Datum pgstat_index(Relation rel, BlockNumber start, pgstat_page pagefn, FunctionCallInfo fcinfo)

入力と出力

  • **入力**:
  • rel: 走査対象のインデックスリレーション。
  • start: 走査を開始するブロック番号。
  • pagefn: 各ページを処理するためのコールバック関数。
  • fcinfo: SQL 関数呼び出し情報。
  • **出力**:
  • Datum: build_pgstattuple_type によって生成された、統計情報を含む複合型。

前提条件と不変条件

  • 呼び出し元は対象リレーションに対して適切なロックを保持している必要があります。
  • pagefn は pgstat_page 型のシグネチャに従い、pgstattuple_type 構造体を更新する責任を持ちます。
  • 走査中は AccessShareLock が保持され、終了時に relation_close によって解放されます。

アルゴリズムまたは振る舞い

1. GetAccessStrategy(BAS_BULKREAD) を使用してバッファアクセス戦略を初期化します。 2. 無限ループ内で以下の処理を繰り返します。

  • LockRelationForExtension を用いて現在のリレーションのブロック数を取得します。
  • 現在のブロック番号 blkno が総ブロック数 nblocks に達した場合、stat.table_len を設定してループを終了します。
  • blkno から nblocks までページを順次走査し、各ページに対して pagefn を呼び出します。
  • 各ページ処理の合間に CHECK_FOR_INTERRUPTS() を実行し、クエリキャンセルに対応します。

    • 3. 最後に relation_close でリレーションを閉じ、build_pgstattuple_type を呼び出して結果を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgstattuple/pgstattuple.c 内の各インデックス統計関数（B-tree, Hash, GiST 用のラッパー関数から呼び出されます）。
  • **呼び出し先**:
  • GetAccessStrategy
  • LockRelationForExtension / UnlockRelationForExtension
  • RelationGetNumberOfBlocks
  • CHECK_FOR_INTERRUPTS
  • pagefn (引数で渡された関数)
  • relation_close
  • build_pgstattuple_type

関連データ構造

  • pgstattuple_type: 統計情報を保持する構造体。
  • Relation: PostgreSQL のリレーション記述子。
  • BufferAccessStrategy: バッファ読み込み戦略。

失敗モードと境界条件

  • 大規模なインデックスに対して実行した場合、全ページを物理的に読み込むため I/O 負荷が高くなります。
  • CHECK_FOR_INTERRUPTS() により、ユーザーによるクエリキャンセルが発生した場合は処理が中断されます。
  • start 引数に不正なブロック番号が指定された場合の挙動は、pagefn の実装に依存します。

保守担当者向けメモ

  • 本関数は汎用的なループ構造を提供しており、特定のインデックス型（B-tree, Hash, GiST 等）に依存しない設計になっています。
  • 統計情報の集計ロジックを変更する場合は、pagefn に渡される pgstattuple_type 構造体の定義と整合性を取る必要があります。
