StringInfo

定義

  • 種別: typedef
  • 定義箇所: src/include/lib/stringinfo.h:54

目的

StringInfoData 構造体へのポインタ型であり、PostgreSQL における可変長文字列バッファの管理および操作のインターフェースとして機能します。

シグネチャまたは形状

typedef StringInfoData *StringInfo;

入力と出力

  • **入力**: StringInfoData 構造体のアドレス。
  • **出力**: なし（ポインタ型定義のため）。

前提条件と不変条件

  • StringInfo が指す StringInfoData は、initStringInfo() 等で適切に初期化されている必要があります。
  • 読み取り専用モード（maxlen == 0）の場合、バッファの拡張や変更を伴う操作はアサーションエラーを引き起こします。

アルゴリズムまたは振る舞い

  • StringInfoData 構造体（data, len, maxlen, cursor）を介して、動的なメモリ再割り当て（repalloc）を伴う文字列の追加や、バイナリデータの保持を抽象化します。
  • 実際の操作は appendStringInfo などの API を通じて行われ、必要に応じてバッファの自動拡張が行われます。

呼び出し元と呼び出し先

  • **呼び出し元（参照例）**:
  • contrib/cube/cube.c
  • contrib/dblink/dblink.c
  • contrib/hstore/hstore_io.c
  • contrib/ltree/ltree_io.c
  • contrib/pg_trgm/trgm_regexp.c
  • contrib/pgcrypto/crypt-sha.c
  • **呼び出し先**: 本シンボル自体は型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • StringInfoData: バッファの実体となる構造体。

失敗モードと境界条件

  • メモリ不足時: palloc/repalloc が失敗した場合、システムはエラーをスローします。
  • 読み取り専用バッファへの書き込み: maxlen が 0 の状態で変更操作を行うとアサーションエラーが発生します。

保守担当者向けメモ

  • StringInfo は StringInfoData へのポインタであるため、関数に渡す際は値渡し（ポインタのコピー）となります。
  • makeStringInfo() で作成した場合は destroyStringInfo() による解放が必要ですが、スタック上の StringInfoData を指している場合は data フィールドの解放のみを行うなど、メモリ管理のライフサイクルに注意してください。
