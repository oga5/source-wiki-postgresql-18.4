current_user

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/name.c:262

目的

現在のセッションにおける有効なユーザー名（ロール名）を name 型として取得します。

シグネチャまたは形状

Datum current_user(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出し規約に基づく引数）。
  • **出力**: 現在のユーザー名を表す name 型の Datum。

前提条件と不変条件

  • GetUserId() を通じて現在のセッションユーザーIDが取得可能であること。
  • namein 関数が正常に動作し、ユーザー名を name 型の固定長バッファに変換できること。

アルゴリズムまたは振る舞い

1. GetUserId() を呼び出し、現在のセッションユーザーの OID を取得します。 2. GetUserNameFromId(oid, false) を呼び出し、OID に対応するユーザー名を文字列（cstring）として取得します。 3. DirectFunctionCall1 を介して namein 関数を呼び出し、取得した文字列を name 型の Datum に変換します。 4. 変換された Datum を PG_RETURN_DATUM マクロで返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c: 式評価エンジン内で実行時に呼び出されます。
  • src/backend/parser/parse_target.c および src/backend/utils/adt/acl.c: コード内のコメントやドキュメントにおいて、本関数が暗黙的に参照されるケースが言及されています。
  • **呼び出し先**:
  • GetUserId(): 現在のユーザーIDを取得。
  • GetUserNameFromId(): ユーザーIDからユーザー名文字列を取得。
  • namein(): 文字列を name 型へ変換。

関連データ構造

  • name: PostgreSQL のシステムカタログ等で使用される固定長（NAMEDATALEN）の識別子型。
  • Datum: PostgreSQL の汎用データ表現型。

失敗モードと境界条件

  • GetUserNameFromId が指定された OID を解決できない場合、エラーが発生する可能性があります（第2引数に false を指定しているため）。
  • ユーザー名が NAMEDATALEN を超える場合、namein の仕様に従い切り詰められる可能性があります。

保守担当者向けメモ

  • 本関数はセッション状態に依存するため、トランザクションやセッションの切り替えに伴うユーザー権限の変更が正しく反映されることを確認してください。
  • name 型の固定長制限（NAMEDATALEN）に依存しているため、将来的にユーザー名の最大長が変更される場合は注意が必要です。
