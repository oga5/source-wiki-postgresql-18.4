PLy_result_str

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_resultobject.c:255

目的

Pythonの PLyResult オブジェクトを文字列形式に変換し、デバッグや表示に適した表現を生成します。

シグネチャまたは形状

static PyObject *PLy_result_str(PyObject *arg)

入力と出力

  • **入力**: arg (Pythonオブジェクト。内部で PLyResultObject へのキャストが行われる)
  • **出力**: PyObject (PythonのUnicode文字列オブジェクト)

前提条件と不変条件

  • 入力 arg は PLyResultObject 型のインスタンスである必要があります。
  • PLyResultObject 構造体内の status, nrows, rows フィールドが有効なPythonオブジェクトを保持していることが前提です。

アルゴリズムまたは振る舞い

1. 引数 arg を PLyResultObject * にキャストします。 2. PyUnicode_FromFormat を使用し、以下の形式の文字列を構築します。 "<PLyResult status=%S nrows=%S rows=%S>"

  • %S 指定子により、内部の status, nrows, rows フィールドの文字列表現が埋め込まれます。

    • 3. 生成されたUnicodeオブジェクトを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Pythonの型システム（Py_tp_str スロット経由で、Python側から str() や print() が呼ばれた際に間接的に呼び出されます）。
  • **呼び出し先**: PyUnicode_FromFormat (Python C API)。

関連データ構造

  • PLyResultObject: クエリ結果を保持するC構造体。本関数はこの構造体のメンバを参照します。

失敗モードと境界条件

  • arg が PLyResultObject ではない場合、不正なメモリ参照が発生する可能性があります。
  • PyUnicode_FromFormat がメモリ不足等で失敗した場合、NULLを返します。

保守担当者向けメモ

  • 本関数は PLyResult_spec 内の Py_tp_str スロットに登録されています。
  • 文字列フォーマットを変更する場合は、PyUnicode_FromFormat の仕様に従い、適切な型指定子を使用してください。
