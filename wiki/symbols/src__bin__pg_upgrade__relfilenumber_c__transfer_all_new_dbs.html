transfer_all_new_dbs

定義

  • **種別**: function
  • **定義箇所**: src/bin/pg_upgrade/relfilenumber.c:170

目的

旧クラスタ内の全データベースを走査し、新クラスタとの対応関係（FileNameMap）を生成した上で、リレーションファイルの移行処理を実行します。

シグネチャまたは形状

void transfer_all_new_dbs(DbInfoArr *old_db_arr, DbInfoArr *new_db_arr,
                          char *old_pgdata, char *new_pgdata, char *old_tablespace)

入力と出力

  • **入力**:
  • old_db_arr, new_db_arr: 旧・新クラスタのデータベース情報配列。
  • old_pgdata, new_pgdata: 各クラスタのデータディレクトリパス。
  • old_tablespace: 移行対象のテーブルスペースパス（指定がない場合は NULL）。
  • **出力**: なし（副作用としてファイルシステム上のデータ移行を実行）。

前提条件と不変条件

  • old_db_arr に含まれるすべてのデータベースが new_db_arr に存在する必要があります。存在しない場合は pg_fatal により処理が中断されます。
  • 呼び出し終了時に、保留されていた同期キュー（sync_queue）が破棄されます。

アルゴリズムまたは振る舞い

1. old_db_arr をループで走査し、対応する new_db_arr 内のデータベースを名前一致で検索します。 2. gen_db_file_maps を呼び出し、旧・新データベース間のファイル対応表（FileNameMap）を生成します。 3. マッピングが存在する場合、transfer_single_new_db を呼び出して個別のファイル転送を実行します。 4. ループ終了後、sync_queue_sync_all を呼び出し、swap モード等で保留されていたファイル同期（fsync）を確定させます。 5. 最後に sync_queue_destroy を呼び出し、同期キューのリソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/parallel.c 内の複数箇所（並列実行制御ロジックから呼び出されます）。
  • **呼び出し先**:
  • gen_db_file_maps
  • transfer_single_new_db
  • sync_queue_sync_all
  • sync_queue_destroy
  • pg_fatal, pg_free

関連データ構造

  • DbInfoArr: データベース情報の配列構造体。
  • FileNameMap: 旧ファイルと新ファイルの対応を保持する構造体。

失敗モードと境界条件

  • **データベース不一致**: 旧クラスタに存在するデータベースが新クラスタで見つからない場合、pg_fatal で異常終了します。
  • **メモリ管理**: n_maps が 0 の場合でも mappings 配列は割り当てられるため、pg_free による解放が必須です。

保守担当者向けメモ

  • 本関数は parallel.c からの並列実行と、単一プロセス実行の両方から呼び出される可能性があります。
  • sync_queue_sync_all は swap モード以外では実質的な操作を行わない（no-op）設計となっています。
