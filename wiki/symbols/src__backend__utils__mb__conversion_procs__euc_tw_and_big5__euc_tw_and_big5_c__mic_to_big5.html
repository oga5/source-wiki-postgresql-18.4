mic_to_big5

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_tw_and_big5/euc_tw_and_big5.c:131

目的

PostgreSQLの MULE_INTERNAL (MIC) エンコーディングから BIG5 エンコーディングへの文字列変換を行うためのSQL呼び出し可能関数です。

シグネチャまたは形状

Datum mic_to_big5(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の引数を受け取ります。
  • src (CSTRING): 変換元のMIC文字列。
  • dest (CSTRING): 変換後のBIG5文字列を格納するバッファ。
  • len (INT32): 入力文字列のバイト長。
  • noError (BOOL): 変換エラー発生時の挙動フラグ。
  • **出力**: Datum 型として、変換されたバイト数を返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_MULE_INTERNAL, PG_BIG5) により、変換元が PG_MULE_INTERNAL、変換先が PG_BIG5 であることが検証されます。

アルゴリズムまたは振る舞い

1. 引数から変換元ポインタ、変換先ポインタ、長さ、エラー制御フラグを取得します。 2. エンコーディング変換の妥当性をチェックします。 3. 内部関数 mic2big5 を呼び出し、実際のバイト変換を実行します。 4. 変換されたバイト数を戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 により登録）。
  • **呼び出し先**: mic2big5（同ファイル内の静的関数）。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約で使用される引数構造体。

失敗モードと境界条件

  • noError が true の場合、変換エラーが発生しても例外を発生させず、処理を中断してその時点までの変換バイト数を返します。
  • noError が false の場合、変換エラー発生時に例外がスローされる可能性があります（mic2big5 の実装に依存）。

保守担当者向けメモ

  • 本関数は変換のインターフェース層であり、実際の変換ロジックは mic2big5 に委譲されています。
  • 変換テーブルや具体的なコードポイントの対応関係については、mic2big5 の実装および関連するヘルパー関数を参照してください。
