xact_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/xactdesc.c:438

目的

トランザクション（XACT）に関連するWALレコードを人間が読める形式に変換し、指定されたバッファへ出力する。

シグネチャまたは形状

void xact_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 解析結果を格納する StringInfo バッファ。
  • record: 解析対象のWALレコードを保持する XLogReaderState 構造体。
  • **出力**: なし（buf を通じて文字列が追記される）。

前提条件と不変条件

  • record は有効な XLogReaderState を指している必要がある。
  • XLogRecGetInfo(record) により取得される情報ビットは、XLOG_XACT_OPMASK でマスクされた範囲で評価される。

アルゴリズムまたは振る舞い

1. XLogRecGetData でレコードのデータポインタを取得し、info ビットを抽出する。 2. info ビットの値に基づき、以下のいずれかの処理へディスパッチする：

  • **COMMIT / COMMIT_PREPARED**: xact_desc_commit を呼び出す。
  • **ABORT / ABORT_PREPARED**: xact_desc_abort を呼び出す。
  • **PREPARE**: xact_desc_prepare を呼び出す。
  • **ASSIGNMENT**: xl_xact_assignment 構造体へキャストし、トップレベルXIDをバッファに書き込んだ後、xact_desc_assignment を呼び出す。
  • **INVALIDATIONS**: xl_xact_invals 構造体へキャストし、standby_desc_invalidations を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h にて RM_XACT_ID リソースマネージャの記述子として登録されており、WAL解析プロセスから呼び出される。
  • **呼び出し先**: xact_desc_commit, xact_desc_abort, xact_desc_prepare, xact_desc_assignment, standby_desc_invalidations。

関連データ構造

  • StringInfo: 出力用バッファ。
  • XLogReaderState: WALレコードの読み取り状態。
  • xl_xact_commit, xl_xact_abort, xl_xact_prepare, xl_xact_assignment, xl_xact_invals: 各種トランザクションWALレコードの構造体。

失敗モードと境界条件

  • info ビットが定義されたいずれのケースにも該当しない場合、関数は何も行わずに終了する。
  • レコードのデータポインタが不正な場合、キャスト後の構造体アクセスで未定義動作が発生する可能性がある（呼び出し元での検証に依存）。

保守担当者向けメモ

  • 本関数はバックエンドと pg_waldump の両方から利用される共有コードであるため、依存関係の追加には注意が必要。
  • XLOG_XACT_OPMASK によるビットマスク処理は、WALレコードの構造変更時に整合性を維持する必要がある。
