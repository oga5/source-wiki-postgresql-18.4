op_in_opfamily

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/lsyscache.c:66

目的

指定された演算子（opno）が、特定の演算子ファミリー（opfamily）に含まれているかどうかを判定する。

シグネチャまたは形状

bool op_in_opfamily(Oid opno, Oid opfamily)

入力と出力

  • **入力**:
  • opno: 演算子のOID。
  • opfamily: 演算子ファミリーのOID。
  • **出力**:
  • bool: 演算子が当該ファミリーのメンバーであれば true、そうでなければ false を返す。

前提条件と不変条件

  • AMOPOPID システムキャッシュが利用可能であること。
  • AMOP_SEARCH 定数が有効な検索キーとして定義されていること。

アルゴリズムまたは振る舞い

1. SearchSysCacheExists3 を呼び出し、AMOPOPID キャッシュを検索する。 2. 検索キーとして、演算子OID、AMOP_SEARCH（検索用フラグ）、演算子ファミリーOIDを使用する。 3. キャッシュ内に該当するエントリが存在するかどうかを判定し、その結果を直接返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/indxpath.c
  • src/backend/optimizer/util/relnode.c
  • src/backend/partitioning/partprune.c
  • src/backend/utils/adt/like_support.c
  • **呼び出し先**:
  • SearchSysCacheExists3 (システムキャッシュ検索API)

関連データ構造

  • pg_amop: 演算子ファミリーのメンバーシップ情報を保持するシステムカタログ。

失敗モードと境界条件

  • キャッシュ検索自体は失敗しないが、無効なOIDが渡された場合は false を返す。
  • データベースのカタログ情報が整合性を欠いている場合、誤った判定結果を返す可能性がある。

保守担当者向けメモ

  • 本関数は単なるキャッシュ検索のラッパーであり、副作用はない。
  • 頻繁に呼び出されるため、パフォーマンスへの影響は最小限である。
