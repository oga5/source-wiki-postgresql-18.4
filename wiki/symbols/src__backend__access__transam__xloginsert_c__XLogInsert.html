XLogInsert

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xloginsert.c:473

目的

構築済みの WAL レコードを WAL バッファへ書き込み、そのレコードの終了位置（LSN）を返します。

シグネチャまたは形状

XLogRecPtr XLogInsert(RmgrId rmid, uint8 info)

入力と出力

  • **入力**:
  • rmid: リソースマネージャ ID。
  • info: レコードの種類を示すフラグ（XLR_RMGR_INFO_MASK 等の制約あり）。
  • **出力**:
  • 戻り値: 書き込まれた WAL レコードの終了位置を示す XLogRecPtr。

前提条件と不変条件

  • XLogBeginInsert() が事前に呼び出されている必要があります。
  • XLogRegisterBuffer() 等でレコードの内容が作業領域に登録されている必要があります。
  • 呼び出し元は、対象となるバッファに対して適切なロックを保持している必要があります。

アルゴリズムまたは振る舞い

1. begininsert_called フラグを確認し、未初期化であればエラーを発生させます。 2. info 引数のマスクチェックを行い、不正なフラグが含まれていれば PANIC を発生させます。 3. ブートストラップモードの場合、RM_XLOG_ID 以外はログを書き込まず、ダミーの EndPos を返して終了します。 4. do-while ループ内で以下を実行します：

  • GetFullPageWriteInfo() でフルページ書き込みの要否を判定。
  • XLogRecordAssemble() を呼び出し、登録されたデータから WAL レコードを組み立てます。
  • XLogInsertRecord() を呼び出し、組み立てたレコードを WAL バッファへ書き込みます。
  • 書き込みが競合等で失敗した場合（InvalidXLogRecPtr が返された場合）、ループを継続して再試行します。

    • 5. 最後に XLogResetInsertion() を呼び出し、作業領域をクリアして終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_visibility/pg_visibility.c
  • src/backend/access/brin/brin.c
  • src/backend/access/brin/brin_pageops.c
  • src/backend/access/brin/brin_revmap.c
  • src/backend/access/gin/ginbtree.c
  • src/backend/access/gin/gindatapage.c
  • **呼び出し先**:
  • XLogRecordAssemble()
  • XLogInsertRecord()
  • XLogResetInsertion()
  • GetFullPageWriteInfo()
  • IsBootstrapProcessingMode()

関連データ構造

  • XLogRecData: 組み立てられたレコードのデータチェーン。
  • XLogRecPtr: WAL 上の論理位置を示すポインタ。

失敗モードと境界条件

  • XLogBeginInsert が未実行の場合: ERROR をスロー。
  • info 引数に予約済みフラグが含まれる場合: PANIC をスロー。
  • WAL 書き込み時に競合が発生した場合: XLogInsertRecord が InvalidXLogRecPtr を返し、ループ内で再試行されます。

保守担当者向けメモ

  • 本関数は START_CRIT_SECTION() 内で呼び出されることが想定されています。
  • XLogInsertRecord が内部でロックを取得するため、本関数の実行中にフルページ書き込みの要否判定条件が変化する可能性がありますが、XLogInsertRecord 側で再チェックが行われます。
