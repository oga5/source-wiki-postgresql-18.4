sentinel_ok

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/memdebug.h:60

目的

メモリ領域の末尾に配置されたセンチネル値（0x7E）を検証し、バッファオーバーランが発生していないかを確認する。

シグネチャまたは形状

static inline bool sentinel_ok(const void *base, Size offset)

入力と出力

  • **入力**:
  • base: 検証対象のメモリ領域の開始ポインタ。
  • offset: センチネル値が配置されていると期待されるオフセット位置。
  • **出力**:
  • bool: センチネル値が 0x7E であれば true、そうでなければ false を返す。

前提条件と不変条件

  • base から offset バイト進んだ位置に、set_sentinel によって 0x7E が書き込まれている必要がある。
  • 本関数はデバッグビルド時（MEMORY_CONTEXT_CHECKING 有効時）に利用されることを想定している。

アルゴリズムまたは振る舞い

1. base ポインタに offset を加算し、対象アドレスを算出する。 2. VALGRIND_MAKE_MEM_DEFINED を呼び出し、対象の1バイトをValgrindに対してアクセス可能（定義済み）として通知する。 3. 対象アドレスの値が 0x7E と一致するかを判定する。 4. VALGRIND_MAKE_MEM_NOACCESS を呼び出し、対象の1バイトを再びアクセス不可としてValgrindに通知する。 5. 判定結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/alignedalloc.c
  • src/backend/utils/mmgr/aset.c
  • src/backend/utils/mmgr/generation.c
  • src/backend/utils/mmgr/slab.c
  • **呼び出し先**:
  • VALGRIND_MAKE_MEM_DEFINED (Valgrindマクロ)
  • VALGRIND_MAKE_MEM_NOACCESS (Valgrindマクロ)

関連データ構造

  • MemoryContext 関連のチャンク構造体（aset.c 等の Chunk 構造体）において、メモリ境界の検証に使用される。

失敗モードと境界条件

  • offset が確保されたメモリ領域の範囲外を指している場合、誤ったメモリ位置を読み取る可能性がある。
  • Valgrindが有効でない環境では、VALGRIND_... マクロは空の定義に置換されるため、メモリ保護機能は動作しない。

保守担当者向けメモ

  • 本関数は src/include/utils/memdebug.h で定義されており、メモリ管理サブシステムのデバッグにおいて重要な役割を果たす。
  • センチネル値 0x7E の変更や検証ロジックの修正を行う際は、set_sentinel との整合性を維持すること。
