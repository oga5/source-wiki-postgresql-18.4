quote_literal_internal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/quote.c:46

目的

SQL文字列リテラルを安全にエスケープし、単一引用符で囲んだ形式に変換するための内部ヘルパー関数です。

シグネチャまたは形状

static size_t quote_literal_internal(char *dst, const char *src, size_t len)

入力と出力

  • **入力**:
  • dst: 変換後の文字列を格納するバッファ（十分な領域が確保されていること）。
  • src: エスケープ対象のソース文字列。
  • len: src のバイト長。
  • **出力**:
  • 戻り値: dst に書き込まれた総バイト数。

前提条件と不変条件

  • dst は、最悪のケース（全文字が引用符で、かつバックスラッシュが含まれる場合）を考慮した十分なメモリ領域を指している必要があります。
  • standard_conforming_strings の設定に関わらず、バックスラッシュが含まれる場合は ESCAPE_STRING_SYNTAX を付与して安全性を担保します。

アルゴリズムまたは振る舞い

1. **バックスラッシュの検出**: src を走査し、バックスラッシュが含まれている場合は ESCAPE_STRING_SYNTAX を dst に書き込みます。 2. **引用符の付与**: 変換後の文字列の先頭に ' を書き込みます。 3. **エスケープ処理**: src の各文字を順次処理します。

  • SQL_STR_DOUBLE マクロが真を返す文字（単一引用符など）は、二重化（''）して書き込みます。
  • その他の文字はそのまま書き込みます。

    • 4. **終了処理**: 最後に ' を書き込み、書き込んだ総バイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/quote.c 内の quote_literal および quote_literal_cstr から呼び出されています。
  • **呼び出し先**:
  • マクロ SQL_STR_DOUBLE を使用しています。

関連データ構造

  • text 型（可変長データ型）の操作に関連します。

失敗モードと境界条件

  • dst バッファのサイズが不足している場合、バッファオーバーランが発生します。呼び出し側で適切なメモリ確保（最悪ケースを想定した palloc 等）が行われている必要があります。

保守担当者向けメモ

  • 本関数は standard_conforming_strings の設定に依存しない動作を強制しています。これは、生成された文字列がどの環境で実行されるか不明な場合でも安全性を担保するためです。
  • 識別子のクォートには使用しないでください（識別子用には quote_ident を使用してください）。
