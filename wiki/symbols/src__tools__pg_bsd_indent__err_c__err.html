err

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/err.c:42

目的

システムエラー（errno）を伴う致命的なエラーメッセージを標準エラー出力（stderr）へ出力し、指定された終了コードでプロセスを終了させる。

シグネチャまたは形状

void err(int eval, const char *fmt, ...);

入力と出力

  • **入力**:
  • eval: プロセス終了時に使用する終了ステータスコード。
  • fmt: メッセージのフォーマット文字列（printf 形式）。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**: なし（関数は exit を呼び出し、呼び出し元へは戻らない）。

前提条件と不変条件

  • errno が呼び出し時点でのシステムエラー状態を保持していること。
  • fmt が NULL の場合、システムエラー文字列のみが出力される。
  • pg_noreturn 属性が付与されているため（err.h 定義による）、本関数呼び出し後のコードは到達不能とみなされる。

アルゴリズムまたは振る舞い

1. 現在の errno をローカル変数 code に保存する。 2. va_start を使用して可変長引数リストを初期化する。 3. fmt が NULL でない場合、vfprintf を用いてフォーマットされたメッセージを stderr に出力し、続けて : を出力する。 4. strerror(code) を用いてシステムエラーに対応する文字列を取得し、stderr に出力して改行する。 5. va_end で引数リストを終了する。 6. exit(eval) を実行し、プロセスを終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報には、本関数を呼び出している具体的な箇所は記載されていない。ツールによる検索結果（contrib/ 配下のコード）は、本関数 err ではなく、変数名や他の文脈での一致であり、本関数への直接的な参照ではない。
  • **呼び出し先**: vfprintf, fprintf, strerror, exit（いずれも標準 C ライブラリ関数）。

関連データ構造

  • va_list: 可変長引数を処理するための型。

失敗モードと境界条件

  • fmt が NULL の場合、メッセージのプレフィックス出力はスキップされ、システムエラー文字列のみが出力される。
  • exit を呼び出すため、本関数を呼び出した時点でプログラムの実行は即座に停止する。

保守担当者向けメモ

  • 本関数は BSD 由来の err(3) の最小限の実装である。
  • 複雑なエラーハンドリングは行わず、致命的なエラー発生時の終了処理に特化している。
  • err.h に定義されている pg_noreturn 属性により、コンパイラは本関数以降のコードを最適化対象から除外する。
