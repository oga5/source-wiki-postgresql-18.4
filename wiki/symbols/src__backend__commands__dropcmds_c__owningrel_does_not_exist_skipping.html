owningrel_does_not_exist_skipping

定義

  • 種別: function
  • 定義箇所: src/backend/commands/dropcmds.c:138

目的

IF EXISTS 指定の DROP 文において、削除対象の親リレーション（スキーマまたはテーブル）が存在しない場合に、適切な警告メッセージを生成して処理をスキップすべきかどうかを判定します。

シグネチャまたは形状

static bool owningrel_does_not_exist_skipping(List *object, const char **msg, char **name)

入力と出力

  • **入力**:
  • object: 削除対象オブジェクトの修飾名リスト。
  • msg: 警告メッセージのテンプレートを格納するためのポインタ。
  • name: 存在しないオブジェクトの名前を格納するためのポインタ。
  • **出力**:
  • 戻り値: 親リレーションが存在しない場合に true、存在する場合に false。

前提条件と不変条件

  • object リストは、少なくとも親リレーションを特定できる長さである必要があります。
  • msg および name は、呼び出し元で確保されたポインタのアドレスである必要があります。

アルゴリズムまたは振る舞い

1. object リストから末尾の要素を除いた部分を parent_object として抽出します。 2. schema_does_not_exist_skipping を呼び出し、スキーマが存在しない場合はその結果を返します。 3. parent_object を RangeVar に変換し、RangeVarGetRelid を用いてリレーションの存在を確認します。 4. リレーションが存在しない場合、msg に "relation \"%s\" does not exist, skipping" を設定し、name に parent_object の文字列表現を設定して true を返します。 5. リレーションが存在する場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/dropcmds.c 内の各削除処理（413, 422, 435 行付近）。
  • **呼び出し先**:
  • list_copy_head
  • schema_does_not_exist_skipping
  • makeRangeVarFromNameList
  • RangeVarGetRelid
  • gettext_noop
  • NameListToString

関連データ構造

  • List: オブジェクト名の修飾リスト。
  • RangeVar: リレーションを特定するための構造体。

失敗モードと境界条件

  • object リストが空、または親リレーションを特定できないほど短い場合、list_copy_head や makeRangeVarFromNameList の挙動に依存します。
  • RangeVarGetRelid の第3引数に true を渡しているため、リレーション不在時にエラーを発生させず InvalidOid を返します。

保守担当者向けメモ

  • 本関数は IF EXISTS 処理の共通ヘルパーの一部です。同様のロジックを持つ schema_does_not_exist_skipping 等と併せて、一貫したエラーメッセージの構築を維持してください。
