dlsym

定義

  • 種別: function
  • 定義箇所: src/port/win32dlopen.c:60

目的

Windows環境において、POSIX標準の dlsym 関数をエミュレートし、ロード済みの共有ライブラリ（DLL）から指定したシンボルのメモリアドレスを取得します。

シグネチャまたは形状

void *dlsym(void *handle, const char *symbol);

入力と出力

  • **入力**:
  • handle: dlopen によって返されたDLLのハンドル。
  • symbol: 取得対象のシンボル名（文字列）。
  • **出力**:
  • 成功時: 指定されたシンボルのアドレス（void *）。
  • 失敗時: NULL。

前提条件と不変条件

  • handle は有効なDLLハンドルである必要があります。
  • 内部エラーバッファ last_dyn_error を使用するため、スレッドセーフではありません。

アルゴリズムまたは振る舞い

1. Windows APIの GetProcAddress を呼び出し、handle が指すDLLから symbol に対応するアドレスを検索します。 2. アドレスの取得に失敗した場合（ptr が NULL の場合）:

  • set_dl_error() を呼び出し、Windowsのシステムエラー情報を内部バッファに記録します。
  • NULL を返します。

    • 3. 成功した場合:

  • 内部エラーバッファ last_dyn_error の先頭をヌル文字でクリアし、取得したアドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit.c
  • src/backend/utils/fmgr/dfmgr.c
  • src/interfaces/libpq/fe-auth-oauth.c
  • **呼び出し先**:
  • GetProcAddress (Windows API)
  • set_dl_error (内部ヘルパー関数)

関連データ構造

  • last_dyn_error: 直近のエラーメッセージを保持する静的バッファ。

失敗モードと境界条件

  • 指定されたシンボルがDLL内に存在しない場合、GetProcAddress が失敗し、dlsym は NULL を返します。
  • エラー発生時は dlerror() を通じてエラー詳細を確認可能です。

保守担当者向けメモ

  • 本関数はWindowsビルド時のみ使用されます。
  • last_dyn_error が静的変数であるため、マルチスレッド環境での同時呼び出しには注意が必要です。
