pgsymlink

定義

  • 種別: function
  • 定義箇所: src/port/dirmod.c:218

目的

Windows 環境において、ディレクトリに対するジャンクションポイントを作成し、POSIX の symlink 関数と同等の機能を提供します。

シグネチャまたは形状

int pgsymlink(const char *oldpath, const char *newpath)

入力と出力

  • **入力**:
  • oldpath: リンク先となるターゲットパス。
  • newpath: 作成するジャンクションポイントのパス。
  • **出力**:
  • 成功時: 0
  • 失敗時: -1（errno が適切に設定される）

前提条件と不変条件

  • Windows 環境（WIN32）でのみ有効です。
  • oldpath が \??\ で始まらない場合、自動的に付与されます。
  • パス内のスラッシュ（/）はバックスラッシュ（\）に置換されます。

アルゴリズムまたは振る舞い

1. CreateDirectory を呼び出し、newpath にディレクトリを作成します。 2. CreateFile を FILE_FLAG_OPEN_REPARSE_POINT および FILE_FLAG_BACKUP_SEMANTICS フラグ付きで呼び出し、ディレクトリのハンドルを取得します。 3. REPARSE_JUNCTION_DATA_BUFFER 構造体を初期化し、ターゲットパスをワイド文字列に変換して格納します。 4. DeviceIoControl を FSCTL_SET_REPARSE_POINT コードで呼び出し、リパースポイントを設定します。 5. 設定に失敗した場合はエラーメッセージを生成し、作成したディレクトリを削除してハンドルを閉じます。 6. 成功時はハンドルを閉じて終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • 直接的な呼び出し元はソースコードから特定できませんが、src/include/port.h および src/include/port/win32_port.h にて symlink マクロとして定義されており、プラットフォーム非依存のコードから利用されることが想定されています。
  • **呼び出し先**:
  • Windows API: CreateDirectory, CreateFile, DeviceIoControl, MultiByteToWideChar, FormatMessage, LocalFree, CloseHandle, RemoveDirectory。
  • ユーティリティ: _dosmaperr, snprintf, strchr, strlcpy, strlen, memcmp。

関連データ構造

  • REPARSE_JUNCTION_DATA_BUFFER: ジャンクションポイントのメタデータを保持する構造体。

失敗モードと境界条件

  • CreateFile が INVALID_HANDLE_VALUE を返した場合、_dosmaperr を経由してエラーを返します。
  • DeviceIoControl が失敗した場合、Windows のエラーコードを errno に変換し、RemoveDirectory で作成したディレクトリをクリーンアップした上で -1 を返します。

保守担当者向けメモ

  • FSCTL_SET_REPARSE_POINT の定義は SDK バージョンに依存するため、CTL_CODE を用いた直接的な呼び出しを行っています。
  • FRONTEND 定義の有無により、エラー報告先が ereport（バックエンド）か stderr（フロントエンド）に分岐します。
