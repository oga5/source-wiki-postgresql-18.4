escape_replace

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_escape/test_escape.c:349

目的

入力文字列をSQLリテラルとしてエスケープし、指定されたバッファに格納する。

シグネチャまたは形状

static bool escape_replace(PGconn *conn, PQExpBuffer target,
                           const char *unescaped, size_t unescaped_len,
                           PQExpBuffer escape_err);

入力と出力

  • **入力**:
  • conn: データベース接続（本関数内では未使用）。
  • target: エスケープ結果を書き込む PQExpBuffer。
  • unescaped: エスケープ対象の文字列。
  • unescaped_len: 入力文字列の長さ。
  • escape_err: エラーメッセージ格納用バッファ（本関数内では未使用）。
  • **出力**:
  • 常に true を返す。

前提条件と不変条件

  • target バッファは呼び出し元で初期化されている必要がある。
  • unescaped は少なくとも unescaped_len バイトのメモリ領域を指している必要がある。

アルゴリズムまたは振る舞い

1. target バッファの先頭にシングルクォート ' を追加する。 2. unescaped を先頭から unescaped_len 分だけループ処理する。

  • 文字が ' の場合、target に '' を追加する。
  • それ以外の文字はそのまま target に追加する。

    • 3. 最後に target バッファの末尾にシングルクォート ' を追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_escape/test_escape.c 内の pe_test_escape_func 構造体定義（428行目）で関数ポインタとして参照されている。
  • **呼び出し先**: appendPQExpBufferChar および appendPQExpBufferStr を呼び出し、バッファ操作を行う。

関連データ構造

  • PQExpBuffer: PostgreSQLの動的文字列バッファ。
  • PGconn: libpqの接続ハンドル。

失敗モードと境界条件

  • unescaped_len が 0 の場合、単一の空リテラル '' が生成される。
  • 本関数は常に true を返すため、メモリ不足等のエラー発生時は appendPQExpBuffer 側の内部挙動に依存する。

保守担当者向けメモ

  • 本関数はテスト用であり、SQLインジェクション対策としての完全なエスケープ処理を目的としたものではない。
  • conn 引数は現在の実装では使用されていない。
