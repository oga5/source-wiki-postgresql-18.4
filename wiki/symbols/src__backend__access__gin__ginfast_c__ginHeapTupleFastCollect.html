ginHeapTupleFastCollect

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginfast.c:482

目的

GINインデックスの高速挿入（Fast Insert）処理において、挿入対象の値を抽出し、ペンディングリストへ追加するための IndexTuple を生成・収集する。

シグネチャまたは形状

void ginHeapTupleFastCollect(GinState *ginstate,
                             GinTupleCollector *collector,
                             OffsetNumber attnum, Datum value, bool isNull,
                             ItemPointer ht_ctid)

入力と出力

  • **入力**:
  • ginstate: GINインデックスの状態管理構造体。
  • collector: 収集したタプルを保持する GinTupleCollector 構造体。
  • attnum, value, isNull: 挿入対象の属性番号、値、およびNULLフラグ。
  • ht_ctid: ヒープタプルを指す ItemPointer。
  • **出力**: なし（collector 構造体の tuples 配列および関連フィールドを更新）。

前提条件と不変条件

  • collector->tuples は palloc または repalloc によって管理される動的配列である。
  • collector->ntuples は現在保持しているタプル数を示す。
  • collector->lentuples は割り当て済みの配列容量を示す。
  • nentries（抽出されたキー数）が MaxAllocSize を超える場合はエラーとなる。

アルゴリズムまたは振る舞い

1. ginExtractEntries を呼び出し、入力値からインデックスに格納すべきキーの配列を抽出する。 2. collector の容量が不足している場合、pg_nextpower2_32 を用いて2のべき乗サイズに拡張（palloc または repalloc）する。 3. 抽出された各キーに対し GinFormTuple を実行して IndexTuple を作成する。 4. 作成したタプルの t_tid に ht_ctid を設定し、collector->tuples に格納する。 5. collector->ntuples をインクリメントし、collector->sumsize を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gininsert.c:885
  • **呼び出し先**:
  • ginExtractEntries
  • GinFormTuple
  • palloc_array / repalloc_array
  • pg_nextpower2_32
  • IndexTupleSize

関連データ構造

  • GinState: GINインデックスの操作状態。
  • GinTupleCollector: 収集された IndexTuple の配列、カウント数、合計サイズを保持する構造体。
  • IndexTuple: GINインデックスの物理的なタプル形式。

失敗モードと境界条件

  • nentries が負の値、または合計サイズが MaxAllocSize を超える場合、elog(ERROR) により処理が中断される。
  • メモリ割り当て失敗時は palloc/repalloc の標準的なエラー挙動に従う。

保守担当者向けメモ

  • 本関数はペンディングリストへの挿入準備を行うためのユーティリティであり、実際のディスク書き込みは行わない。
  • collector->tuples のリサイズ戦略として2のべき乗を採用しており、メモリ効率と再割り当て回数のバランスを考慮している。
