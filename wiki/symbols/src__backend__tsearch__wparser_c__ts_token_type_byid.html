ts_token_type_byid

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/wparser.c:105

目的

指定されたパーサーIDに対応するトークン型の一覧をSQLクエリに対して提供するセット返却関数（SRF）です。

シグネチャまたは形状

Datum ts_token_type_byid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) により、対象となるパーサーのOIDを受け取ります。
  • **出力**: FuncCallContext を介して、トークン型情報を含むタプルを順次返します。

前提条件と不変条件

  • PostgreSQLのSRF（Set-Returning Function）として実装されており、呼び出し間で FuncCallContext を保持する必要があります。
  • 初回呼び出し時に tt_setup_firstcall が実行され、パーサー定義がキャッシュからロードされる必要があります。

アルゴリズムまたは振る舞い

1. **初回呼び出し**: SRF_IS_FIRSTCALL() が真の場合、SRF_FIRSTCALL_INIT() でコンテキストを初期化し、tt_setup_firstcall を呼び出してパーサーのトークン型情報をメモリ上に展開します。 2. **反復処理**: SRF_PERCALL_SETUP() でコンテキストを取得し、tt_process_call を呼び出します。 3. **データ返却**: tt_process_call が有効なタプルを返す間は SRF_RETURN_NEXT を継続し、データが尽きた時点で SRF_RETURN_DONE を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（SRFとして呼び出されます）。
  • **呼び出し先**:
  • tt_setup_firstcall（初期化時）
  • tt_process_call（各反復時）

関連データ構造

  • FuncCallContext: SRFの実行状態を保持するために使用されます。

失敗モードと境界条件

  • 指定されたOIDに対応するパーサーが存在しない場合、tt_setup_firstcall 内でエラーが発生する可能性があります。
  • 大量のトークン型を持つパーサーの場合、メモリ消費量が増大する可能性があります。

保守担当者向けメモ

  • 本関数はSQLインターフェース層であり、実際のロジックは tt_setup_firstcall および tt_process_call にカプセル化されています。
  • 修正を行う際は、src/backend/tsearch/wparser.c 内の関連するヘルパー関数との整合性に注意してください。
