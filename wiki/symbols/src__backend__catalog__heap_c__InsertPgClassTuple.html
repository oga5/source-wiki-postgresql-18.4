InsertPgClassTuple

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/heap.c:923

目的

新しいリレーション（テーブルやインデックス等）のメタデータを pg_class システムカタログに挿入する。

シグネチャまたは形状

void InsertPgClassTuple(Relation pg_class_desc,
                        Relation new_rel_desc,
                        Oid new_rel_oid,
                        Datum relacl,
                        Datum reloptions)

入力と出力

  • **入力**:
  • pg_class_desc: pg_class リレーションのディスクリプタ。
  • new_rel_desc: 挿入対象となる新しいリレーションのディスクリプタ。
  • new_rel_oid: 新しいリレーションに割り当てられた OID。
  • relacl: アクセス制御リスト（Datum）。指定がない場合は (Datum) 0。
  • reloptions: リレーションオプション（Datum）。指定がない場合は (Datum) 0。
  • **出力**: なし（副作用としてカタログへ挿入）。

前提条件と不変条件

  • トランザクション内で呼び出される必要がある。
  • pg_class_desc は pg_class カタログを指している必要がある。
  • relpartbound 列は常に NULL として挿入される（後続の更新処理で設定される前提）。

アルゴリズムまたは振る舞い

1. pg_class の全属性に対応する values 配列と nulls 配列を初期化する。 2. new_rel_desc->rd_rel から各フィールド値を抽出し、values 配列に格納する。 3. relacl および reloptions が提供されている場合は値をセットし、そうでない場合は nulls フラグを立てる。 4. heap_form_tuple を使用して、pg_class の記述子に基づいたタプルを構築する。 5. CatalogTupleInsert を呼び出し、タプルを挿入してインデックスを更新する。 6. heap_freetuple で構築したタプルを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c (複数箇所)
  • src/backend/catalog/index.c:1016
  • **呼び出し先**:
  • heap_form_tuple
  • CatalogTupleInsert
  • heap_freetuple
  • RelationGetDescr

関連データ構造

  • Relation: リレーションのメタデータ構造体。
  • Form_pg_class: pg_class カタログのタプル構造。
  • HeapTuple: 挿入対象のヒープタプル。

失敗モードと境界条件

  • relacl や reloptions が (Datum) 0 の場合、対応するカタログ列は NULL として扱われる。
  • カタログ挿入時にインデックスの制約違反が発生した場合、CatalogTupleInsert 経由でエラーがスローされる可能性がある。

保守担当者向けメモ

  • pg_class のスキーマ変更（列の追加・削除）を行う際は、values 配列のインデックスと Anum_pg_class_... 定数の整合性を確認すること。
  • relpartbound は本関数では設定されず、別途更新される仕様である点に注意。
