gzip_palloc

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_gzip.c:380

目的

zlib ライブラリが内部メモリ確保に使用するカスタムアロケータ関数として機能し、PostgreSQL のメモリ管理システム（palloc）を介してメモリを割り当てる。

シグネチャまたは形状

static void *gzip_palloc(void *opaque, unsigned items, unsigned size);

入力と出力

  • **入力**:
  • opaque: zlib から渡される不透明なポインタ（本実装では使用されない）。
  • items: 確保する要素数。
  • size: 各要素のバイトサイズ。
  • **出力**:
  • 確保されたメモリ領域へのポインタ。

前提条件と不変条件

  • palloc が利用可能なメモリコンテキスト内で呼び出される必要がある。
  • items * size の計算結果がメモリ確保の制限を超えないこと。

アルゴリズムまたは振る舞い

1. items と size を乗算し、必要な合計バイト数を算出する。 2. PostgreSQL の palloc 関数を呼び出し、メモリを確保してそのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し先**: palloc
  • **呼び出し元**:
  • src/fe_utils/astreamer_gzip.c:253（zs->zalloc への代入）
  • src/backend/backup/basebackup_gzip.c:122（zs->zalloc への代入）
  • ※その他、src/backend/backup/basebackup_gzip.c および src/fe_utils/astreamer_gzip.c 内で関数ポインタとして参照されている。

関連データ構造

  • z_stream（zlib の構造体。zalloc フィールドに本関数が設定される）

失敗モードと境界条件

  • palloc がメモリ不足を検知した場合、PostgreSQL のエラーハンドリング機構によりエラーがスローされる（本関数自体はエラーを戻り値として返さない）。

保守担当者向けメモ

  • 本関数は zlib の zalloc インターフェースに適合させるためのラッパーである。
  • opaque 引数は zlib の仕様上存在するが、本実装では使用していない。
  • 同様の関数が src/backend/backup/basebackup_gzip.c にも存在するため、修正時は両方の実装に注意すること。
