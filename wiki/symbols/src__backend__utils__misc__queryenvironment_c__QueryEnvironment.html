QueryEnvironment

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/misc/queryenvironment.c:32

目的

クエリ実行時やプランニング時に、カタログに存在しない一時的な名前付きリレーション（Ephemeral Named Relation: ENR）を保持・管理するためのコンテナです。トリガーの遷移テーブルなどをクエリ内で参照可能にします。

シグネチャまたは形状

struct QueryEnvironment
{
	List	   *namedRelList;
};

入力と出力

  • 入力: 本構造体は create_queryEnv() によって生成されます。
  • 出力: 内部の namedRelList を通じて、登録された EphemeralNamedRelation オブジェクトを管理します。

前提条件と不変条件

  • 外部からは不透明な構造体（Opaque Structure）として扱われることが推奨されます。
  • 内部の namedRelList は、ENR の数が少ないことを前提とした線形探索用のリストです。

アルゴリズムまたは振る舞い

  • **管理**: register_ENR() によりリストへ追加され、unregister_ENR() により削除されます。
  • **検索**: get_ENR() を呼び出すことで、名前を指定してリストを線形探索し、該当する ENR を取得します。
  • **抽象化**: 実装の詳細（リスト構造など）は隠蔽されており、将来的にハッシュテーブル等へ変更しても呼び出し側に影響を与えない設計となっています。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下のコンポーネントで QueryEnvironment ポインタが参照されています。
  • contrib/pg_overexplain
  • contrib/pg_stat_statements
  • contrib/sepgsql/hooks.c
  • src/backend/commands/createas.c
  • src/backend/commands/explain.c
  • src/backend/parser/analyze.c
  • src/backend/parser/parse_relation.c
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。操作には register_ENR や get_ENR 等の関連関数が使用されます。

関連データ構造

  • EphemeralNamedRelation: 本構造体が管理する ENR の実体。
  • List: namedRelList の型。

失敗モードと境界条件

  • ENR の数が非常に多い場合、線形探索によるパフォーマンス低下が発生する可能性があります。
  • ENRMetadataGetTupDesc 等でリレーションにアクセスする際、適切なロックが保持されていない場合は予期せぬ動作を引き起こす可能性があります（table_open に NoLock を指定しているため）。

保守担当者向けメモ

  • 現在は List による単純な実装ですが、パフォーマンス上の課題が生じた場合はこの構造体および関連関数を修正することで最適化が可能です。
  • デバッグ時は、ENRMetadataGetTupDesc が呼び出される時点で適切なロックが既に取得されている前提であることを考慮してください。
