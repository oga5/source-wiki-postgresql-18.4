downcase_convert

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pgsql.c:287

目的

入力されたバイト列を小文字に変換し、新しいメモリ領域にコピーして返します。主にPGP関連のオプション文字列の正規化に使用されます。

シグネチャまたは形状

char *downcase_convert(const uint8 *s, int len)

入力と出力

  • **入力**:
  • s: 変換対象のバイト列（const uint8 *）。
  • len: 変換対象の長さ（int）。
  • **出力**:
  • 変換後の文字列を格納した、pallocで確保されたヌル終端文字列（char *）。

前提条件と不変条件

  • len は s の有効な長さを指している必要があります。
  • 戻り値のメモリは呼び出し元で解放（pfree）する必要があります。

アルゴリズムまたは振る舞い

1. len + 1 バイトのメモリを palloc で確保します。 2. 入力バイト列 s を先頭から順に走査します。 3. 各バイトが 'A' から 'Z' の範囲内であれば、対応する小文字（ASCII値の差分を加算）に変換します。 4. 変換後の値を確保したメモリ領域に格納します。 5. 最後にヌル終端文字（0）を付与し、文字列のポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-pgsql.c 内の parse_args 関数から呼び出されます。
  • **呼び出し先**: PostgreSQLのメモリ管理関数 palloc を使用します。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • palloc がメモリ不足により失敗した場合、PostgreSQLの標準的なエラーハンドリングにより例外が発生します。
  • 入力 len が 0 の場合、ヌル終端のみの空文字列が返されます。

保守担当者向けメモ

  • 本関数はASCII文字の小文字化のみを想定しています。マルチバイト文字の変換には対応していません。
  • 変換後の文字列は palloc されているため、メモリリークを防ぐために呼び出し元での適切な解放を確認してください。
