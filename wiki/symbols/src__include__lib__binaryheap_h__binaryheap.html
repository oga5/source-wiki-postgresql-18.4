binaryheap

定義

  • 種別: struct
  • 定義箇所: src/include/lib/binaryheap.h:42

目的

メモリ効率を考慮した可変長配列ベースの二分ヒープ構造を提供し、優先度付きキューやソート処理における効率的な要素抽出を可能にする。

シグネチャまたは形状

typedef struct binaryheap
{
    int            bh_size;
    int            bh_space;
    bool           bh_has_heap_property;
    binaryheap_comparator bh_compare;
    void          *bh_arg;
    bh_node_type   bh_nodes[FLEXIBLE_ARRAY_MEMBER];
} binaryheap;

入力と出力

  • **入力**: bh_compare（比較関数）および bh_arg（比較関数に渡す補助引数）を保持する。
  • **出力**: bh_nodes フレキシブル配列メンバを通じて、ヒープ特性に従って整列された要素を保持する。

前提条件と不変条件

  • bh_nodes は連続したメモリ領域として確保される。
  • bh_has_heap_property はデバッグ用のフラグであり、binaryheap_add_unordered 使用後は binaryheap_build を呼び出してヒープ特性を再構築する必要がある。
  • 比較関数は a < b で負、a == b で 0、a > b で正を返す必要がある。

アルゴリズムまたは振る舞い

  • bh_nodes 配列内でインデックス計算（2*i+1 等）を行い、sift_up / sift_down 操作によってヒープ特性を維持する。
  • binaryheap_allocate で初期化し、binaryheap_free で解放する。
  • binaryheap_add で逐次追加、または binaryheap_add_unordered で全要素投入後に binaryheap_build を呼び出すことで O(n) でヒープを構築する。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • src/backend/executor/nodeGatherMerge.c
  • src/backend/executor/nodeMergeAppend.c
  • src/backend/postmaster/pgarch.c
  • src/backend/replication/logical/reorderbuffer.c
  • src/backend/storage/buffer/bufmgr.c
  • src/bin/pg_dump/pg_backup_archiver.c
  • **呼び出し先**:
  • 比較関数（bh_compare）を介して外部ロジックを呼び出す。
  • 実装本体である src/common/binaryheap.c 内の sift_up および sift_down 関数が内部ロジックとして動作する。

関連データ構造

  • bh_node_type: 環境に応じて Datum または void * に解決されるノード型。
  • binaryheap_comparator: ヒープの順序を決定するコールバック関数の型。
  • pairingheap: 用途や計算量特性に応じて選択可能な別のヒープ実装。

失敗モードと境界条件

  • binaryheap_allocate で確保したメモリを解放し忘れるとメモリリークが発生する。
  • binaryheap_add_unordered を使用した後に binaryheap_build を呼び出さない場合、ヒープ特性が保証されない。

保守担当者向けメモ

  • bh_nodes はフレキシブル配列メンバであるため、構造体自体のサイズではなく、確保時に必要な容量を考慮すること。
  • 最小ヒープと最大ヒープの切り替えは、比較関数の実装によって制御される。
