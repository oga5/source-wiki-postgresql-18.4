EndCommand

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/dest.c:168

目的

クエリ実行終了時に出力先（CommandDest）の状態を整理し、必要に応じてクライアントへコマンド完了通知を送信する。

シグネチャまたは形状

void EndCommand(const QueryCompletion *qc, CommandDest dest, bool force_undecorated_output)

入力と出力

  • qc: 完了したコマンドのタグ情報を含む QueryCompletion 構造体へのポインタ。
  • dest: 出力先を示す CommandDest 列挙型。
  • force_undecorated_output: 出力文字列の装飾を強制的に抑制するかどうかを示すフラグ。
  • 出力: なし（戻り値は void）。

前提条件と不変条件

  • BeginCommand と対になる関数であり、SQLクエリごとに1回実行される。
  • dest がリモート通信系（DestRemote, DestRemoteExecute, DestRemoteSimple）の場合、pq_putmessage を介してクライアントへメッセージが送信される。

アルゴリズムまたは振る舞い

1. dest の値に基づき switch 文で分岐する。 2. リモート出力系（DestRemote 等）の場合:

  • BuildQueryCompletionString を呼び出し、qc から完了タグ文字列を生成する。
  • pq_putmessage を使用して PqMsg_CommandComplete メッセージをクライアントへ送信する。

    • 3. その他の出力先（DestNone, DestSPI 等）の場合:

  • 何も行わず break する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/replication/walsender.c
  • src/backend/tcop/postgres.c
  • 呼び出し先:
  • BuildQueryCompletionString
  • pq_putmessage

関連データ構造

  • QueryCompletion: コマンド完了タグを保持する構造体。
  • CommandDest: 出力先を識別する列挙型。

失敗モードと境界条件

  • dest がリモート通信系以外の場合は、メッセージ送信処理がスキップされる。
  • 通信エラーが発生した場合の挙動については、本関数内では明示的なエラーハンドリングは行われていない。

保守担当者向けメモ

  • 新しい出力先を CommandDest に追加した場合は、本関数の switch 文にケースを追加する必要があるか検討すること。
  • EndReplicationCommand は本関数の簡易版として実装されているため、変更時は整合性に注意すること。
