ProcessSlotSyncMessage

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/slotsync.c:1251

目的

スタンバイサーバーの昇格（Promotion）がトリガーされた際に、実行中のレプリケーションスロット同期処理を停止または終了させる。

シグネチャまたは形状

void ProcessSlotSyncMessage(void)

入力と出力

  • 入力: なし（グローバル状態 SlotSyncShutdownPending およびプロセス種別を参照）
  • 出力: なし（成功時はプロセス終了またはエラー送出により制御が戻らない）

前提条件と不変条件

  • 本関数は、昇格処理が開始されたことを示すシグナルやメッセージを受信したコンテキストで呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. SlotSyncShutdownPending フラグを false にリセットする。 2. 現在のプロセスが AmLogicalSlotSyncWorkerProcess()（論理スロット同期ワーカー）である場合：

  • ログに停止メッセージを出力し、proc_exit(0) を呼び出して正常終了する。

    • 3. それ以外（SQL 関数経由の呼び出し等）の場合：

  • IsSyncingReplicationSlots() を確認し、同期が既に完了していれば何もせずリターンする。
  • 同期中であれば、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE エラーを送出し、昇格による同期停止を通知する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/replication/logical/slotsync.c (内部シグナルハンドラ等)
  • src/backend/tcop/postgres.c:3541
  • 呼び出し先:
  • AmLogicalSlotSyncWorkerProcess()
  • IsSyncingReplicationSlots()
  • ereport()
  • proc_exit()

関連データ構造

  • SlotSyncShutdownPending (グローバル変数)

失敗モードと境界条件

  • 同期ワーカープロセス以外から呼び出された場合、同期が進行中であればエラーを送出する。
  • 同期が既に完了している場合は、エラーを発生させずに安全に終了する。

保守担当者向けメモ

  • 本関数は昇格時の安全な停止処理を担う。同期ワーカーと SQL 呼び出し元で終了時の挙動（正常終了かエラー送出か）が異なる点に注意すること。
