record_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rowtypes.c:479

目的

複合型（Composite Types）のバイナリ形式入力をデシリアライズし、HeapTupleHeader として構築する。

シグネチャまたは形状

Datum record_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): StringInfo 型のバイナリデータバッファ。
  • PG_GETARG_OID(1): 対象レコードの型OID。
  • PG_GETARG_INT32(2): 対象レコードの型修飾子（typmod）。
  • **出力**:
  • 構築された HeapTupleHeader を Datum として返す。

前提条件と不変条件

  • check_stack_depth() により、再帰的なレコード型定義に対するスタックオーバーフロー保護が行われる。
  • 匿名レコード型（RECORDOID）かつ typmod < 0 の場合、入力はサポートされずエラーとなる。
  • 削除された列（attisdropped）はスキップされ、NULLとして扱われる。

アルゴリズムまたは振る舞い

1. **型情報の取得**: lookup_rowtype_tupdesc を使用して対象の型定義（TupleDesc）を取得する。 2. **キャッシュ管理**: fcinfo->flinfo->fn_extra を使用し、型定義に対応する RecordIOData（各列のI/O関数情報）をキャッシュする。 3. **列数の検証**: 入力データ内の列数と、型定義上の非削除列数が一致するかを確認する。 4. **各列の処理**:

  • 各列の型OID、データ長を読み取る。
  • 長さが -1 の場合はNULLとして処理する。
  • それ以外の場合は、ReceiveFunctionCall を呼び出し、各列の型に対応するバイナリ入力関数を実行して値を構築する。

    • 5. **タプル構築**: 収集した値とNULLフラグから heap_form_tuple を用いてタプルを生成する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのバイナリプロトコル経由のデータ入力処理（直接的な呼び出し元はソースコード上では明示されていない）。
  • **呼び出し先**: lookup_rowtype_tupdesc, pq_getmsgint, getTypeBinaryInputInfo, ReceiveFunctionCall, heap_form_tuple 等。

関連データ構造

  • RecordIOData: 各列のI/O関数情報や型情報を保持するキャッシュ構造体。
  • ColumnIOData: 個別の列に対するI/O関数ポインタやパラメータを保持する構造体。
  • TupleDesc: レコードの構造定義。

失敗モードと境界条件

  • **ERRCODE_FEATURE_NOT_SUPPORTED**: 匿名レコード型で typmod が指定されていない場合。
  • **ERRCODE_DATATYPE_MISMATCH**: 入力データの列数が期待値と異なる場合、または組み込み型同士で型OIDが一致しない場合。
  • **ERRCODE_INVALID_BINARY_REPRESENTATION**: 入力データの長さが不正な場合。

保守担当者向けメモ

  • fn_extra キャッシュは、レコード型が呼び出し中に変更されないことを前提としている。
  • 組み込み型以外の型不一致については、セキュリティ上の理由から厳密なチェックを行わず、型ごとの受信関数に処理を委ねている。
  • 削除された列の処理において、物理的な列位置と論理的な列順序の整合性に注意すること。
