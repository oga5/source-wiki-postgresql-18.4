ssl_cipher

定義

  • 種別: function
  • 定義箇所: contrib/sslinfo/sslinfo.c:76

目的

現在のセッションで使用されているSSL暗号スイート（Cipher Suite）の名前を文字列として取得する。

シグネチャまたは形状

Datum ssl_cipher(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（PostgreSQLの関数呼び出し規約に基づく引数）
  • **出力**: Datum 型（SSL暗号スイート名を表す text 型データ、またはSSLが使用されていない場合は NULL）

前提条件と不変条件

  • グローバル変数 MyProcPort が初期化されていること。
  • SSL接続が確立されている場合、MyProcPort->ssl_in_use が真であること。

アルゴリズムまたは振る舞い

1. MyProcPort->ssl_in_use を確認し、SSLが使用されていない場合は PG_RETURN_NULL() を返す。 2. be_tls_get_cipher(MyProcPort) を呼び出し、現在の暗号スイート名を取得する。 3. 取得した文字列が NULL の場合は PG_RETURN_NULL() を返す。 4. 取得した文字列を cstring_to_text でPostgreSQLの text 型に変換し、PG_RETURN_TEXT_P を通じて返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（PG_FUNCTION_INFO_V1(ssl_cipher) により登録）。
  • **呼び出し先**: be_tls_get_cipher（src/backend/libpq/be-secure-openssl.c で定義される低レイヤー関数）。

関連データ構造

  • Port 構造体（MyProcPort として参照）

失敗モードと境界条件

  • SSL接続が確立されていない場合、NULL を返す。
  • be_tls_get_cipher が暗号スイート名を特定できない場合、NULL を返す。

保守担当者向けメモ

  • 本関数は MyProcPort を介してOpenSSLのセッション情報を取得するブリッジである。
  • 戻り値のメモリ管理はPostgreSQLの text 型変換プロセスに依存する。
  • be_tls_get_cipher の実装変更が本関数の挙動に直接影響を与える。
