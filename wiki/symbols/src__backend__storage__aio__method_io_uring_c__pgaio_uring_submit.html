pgaio_uring_submit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_io_uring.c:404

目的

PgAioHandle の配列を io_uring の Submission Queue (SQ) に変換し、カーネルへ I/O 投入を行う。

シグネチャまたは形状

int pgaio_uring_submit(uint16 num_staged_ios, PgAioHandle **staged_ios)

入力と出力

  • **入力**:
  • num_staged_ios: 投入する I/O の数（PGAIO_SUBMIT_BATCH_SIZE 以下であること）。
  • staged_ios: 投入対象の PgAioHandle ポインタ配列。
  • **出力**:
  • 成功時: 投入した I/O 数（num_staged_ios と一致）。
  • 失敗時: PANIC によりプロセス終了。

前提条件と不変条件

  • pgaio_my_uring_context が初期化済みであること。
  • num_staged_ios は PGAIO_SUBMIT_BATCH_SIZE を超えないこと。
  • 呼び出し元は、I/O 完了待ちによるブロックが許容されるコンテキストであること。

アルゴリズムまたは振る舞い

1. io_uring_get_sqe を用いて各 PgAioHandle 分の SQE を取得する。 2. pgaio_io_prepare_submit および pgaio_uring_sq_from_io を呼び出し、SQE に I/O 情報を設定する。 3. バッファード I/O かつ既に飛行中の I/O が 4 件以上ある場合、IOSQE_ASYNC フラグを付与してカーネル側のワーカーへオフロードさせる。 4. io_uring_submit を呼び出し、カーネルへ通知する。 5. EINTR の場合は再試行し、それ以外の負の戻り値は PANIC とする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/aio/method_io_uring.c 内の pgaio_uring_ops 構造体を通じて AIO サブシステムから呼び出される。
  • **呼び出し先**: io_uring_get_sqe, pgaio_io_prepare_submit, pgaio_uring_sq_from_io, io_uring_sqe_set_flags, pgstat_report_wait_start/end, io_uring_submit。

関連データ構造

  • PgAioHandle: 投入される I/O の抽象ハンドル。
  • io_uring: liburing が提供するコンテキスト構造体。
  • io_uring_sqe: io_uring の Submission Queue Entry。

失敗モードと境界条件

  • **SQE 不足**: io_uring_get_sqe が NULL を返した場合、elog(ERROR) を発行。
  • **システムコールエラー**: io_uring_submit が負の値を返した場合、PANIC を発行。
  • **部分投入**: 投入数が要求と一致しない場合、PANIC を発行。

保守担当者向けメモ

  • 本関数は io_uring の非同期性を活用するため、バッファード I/O の場合は IOSQE_ASYNC を動的に付与してパフォーマンスを最適化している。
  • EAGAIN 発生時の再試行は、呼び出し元がロックを保持している可能性を考慮し、本関数内では行わず PANIC としている。
