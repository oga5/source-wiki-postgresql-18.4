pg_cryptohash_error

定義

  • 種別: function
  • 定義箇所: src/common/cryptohash_openssl.c:348

目的

暗号学的ハッシュ計算コンテキストに関連付けられたエラーメッセージを取得する。

シグネチャまたは形状

const char *pg_cryptohash_error(pg_cryptohash_ctx *ctx);

入力と出力

  • **入力**: pg_cryptohash_ctx *ctx（ハッシュ計算コンテキストへのポインタ）
  • **出力**: エラー内容を示す文字列（const char *）。エラーがない場合は "success" を返す。

前提条件と不変条件

  • ctx が NULL の場合、メモリ不足を示す "out of memory" を返す。
  • コンテキスト内に errreason が設定されている場合、その文字列を優先して返す。
  • 内部エラー状態が PG_CRYPTOHASH_ERROR_NONE の場合、成功として "success" を返す。

アルゴリズムまたは振る舞い

1. ctx が NULL かを確認し、該当すれば "out of memory" を返す。 2. ctx->errreason が存在すれば、その値を返す。 3. ctx->error の値に基づき、以下のいずれかの文字列を返す：

  • PG_CRYPTOHASH_ERROR_NONE: "success"
  • PG_CRYPTOHASH_ERROR_DEST_LEN: "destination buffer too small"
  • PG_CRYPTOHASH_ERROR_OPENSSL: "OpenSSL failure"

    • 4. 上記のいずれにも該当しない場合、Assert(false) により停止する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/uuid-ossp/uuid-ossp.c
  • src/backend/backup/backup_manifest.c
  • src/backend/utils/adt/cryptohashfuncs.c
  • **呼び出し先**: 外部関数呼び出しはなし。

関連データ構造

  • pg_cryptohash_ctx: ハッシュ計算の状態を保持する構造体。

失敗モードと境界条件

  • ctx が NULL の場合、安全に "out of memory" を返す。
  • 未定義の ctx->error 値が渡された場合、アサーション違反によりプログラムが停止する。

保守担当者向けメモ

  • 本関数はOpenSSL固有のエラー詳細を直接返さず、PG_CRYPTOHASH_ERROR_OPENSSL という汎用的なエラーコードを返す設計となっている。詳細なOpenSSLエラーが必要な場合は、別途 ERR_get_error() 等のOpenSSL APIを確認する必要がある。
