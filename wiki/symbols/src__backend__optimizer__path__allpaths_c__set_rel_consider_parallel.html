set_rel_consider_parallel

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/allpaths.c:588

目的

リレーション（ベースリレーションまたは継承の子リレーション）が並列クエリ実行に適しているかを判定し、RelOptInfo->consider_parallel フラグを設定する。

シグネチャまたは形状

static void set_rel_consider_parallel(PlannerInfo *root, RelOptInfo *rel, RangeTblEntry *rte)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel: 判定対象のリレーション情報。
  • rte: リレーションに対応するレンジテーブルエントリ。
  • **出力**: なし（副作用として rel->consider_parallel を更新）。

前提条件と不変条件

  • rel->consider_parallel は初期状態で false であること。
  • クエリ全体で並列モードが許可されていること (root->glob->parallelModeOK)。
  • 対象はベースリレーションまたは継承の子リレーションであること (IS_SIMPLE_REL(rel))。

アルゴリズムまたは振る舞い

1. **RTE種別ごとの検証**: rte->rtekind に基づき、並列実行を阻害する要素（一時テーブル、CTE、非並列安全な関数やテーブルサンプルなど）がないか確認する。 2. **制約の評価**: リレーションの baserestrictinfo および reltarget->exprs が並列安全であるかを is_parallel_safe で検証する。 3. **フラグ設定**: 上記のすべてのチェックを通過した場合に限り、rel->consider_parallel を true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/allpaths.c 内の set_base_rel_pathlists および set_append_rel_pathlist から呼び出されることが確認されている。
  • **呼び出し先**:
  • is_parallel_safe: 式やリストの並列安全性を判定。
  • get_rel_persistence: リレーションの永続性を取得。
  • func_parallel: 関数の並列安全性を取得。
  • limit_needed: サブクエリのLIMIT/OFFSET有無を確認。
  • rel->fdwroutine->IsForeignScanParallelSafe: FDW固有の並列安全性を確認。

関連データ構造

  • RelOptInfo: パス生成対象のリレーション情報。
  • RangeTblEntry: クエリ内のテーブルやサブクエリの定義情報。
  • PlannerInfo: プランナのグローバル状態。

失敗モードと境界条件

  • **早期リターン**: 一時テーブル、CTE、TABLEFUNC、NAMEDTUPLESTORE、または並列制限のある関数や式が含まれる場合、直ちに処理を中断し consider_parallel は false のままとなる。
  • **RTE_JOIN**: ベースリレーションのみを対象とするため、RTE_JOIN が渡された場合はアサーションエラーとなる。

保守担当者向けメモ

  • 本関数は並列クエリの可否を決定する重要なゲートキーパーである。新しい rtekind や並列実行を制限する要素を追加する際は、必ず本関数の switch 文に適切なチェックを追加すること。
  • baserestrictinfo の評価は、将来的に並列実行の制約を緩和する余地があるが、現在は保守的な設計となっている。
