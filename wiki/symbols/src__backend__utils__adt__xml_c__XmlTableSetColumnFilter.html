XmlTableSetColumnFilter

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xml.c:4845

目的

XMLTABLE 演算子において、特定の列に対応する XPath 式をコンパイルし、実行コンテキストに登録します。

シグネチャまたは形状

static void XmlTableSetColumnFilter(TableFuncScanState *state, const char *path, int colnum)

入力と出力

  • state: TableFuncScanState 構造体へのポインタ。XMLTABLE の実行状態を保持します。
  • path: 列抽出に使用する XPath 式（文字列）。
  • colnum: フィルタを適用する対象の列番号。
  • 出力: なし（副作用として state 内の XmlTableBuilderData を更新）。

前提条件と不変条件

  • USE_LIBXML が定義されている必要があります。
  • path は空文字列であってはなりません。
  • XmlTableSetDocument が既に呼び出され、xtCxt->xpathcxt が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. GetXmlTableBuilderPrivateData を呼び出し、TableFuncScanState から XmlTableBuilderData を取得します。 2. path が空でないことを検証し、空の場合はエラーを報告します。 3. path を libxml2 形式の文字列に複製します。 4. xmlXPathCtxtCompile を使用して XPath 式をコンパイルし、結果を xtCxt->xpathscomp[colnum] に格納します。 5. コンパイル失敗時、または libxml2 内部でエラーが発生した場合は xml_ereport を通じて PostgreSQL のエラーとして報告します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/xml.c 内の XmlTableRoutine 構造体（.SetColumnFilter フィールド）を通じて呼び出されます。
  • 呼び出し先: GetXmlTableBuilderPrivateData, pg_xmlCharStrndup, xmlXPathCtxtCompile, xml_ereport。

関連データ構造

  • TableFuncScanState: 実行ノードの状態を保持する構造体。
  • XmlTableBuilderData: XMLTABLE のビルドに必要な XPath コンテキストやコンパイル済み式を保持する内部構造体。

失敗モードと境界条件

  • path が空文字列の場合: ERRCODE_INVALID_ARGUMENT_FOR_XQUERY でエラー。
  • XPath 式が不正な場合: xmlXPathCtxtCompile が失敗し、ERRCODE_INVALID_ARGUMENT_FOR_XQUERY でエラー。
  • USE_LIBXML が未定義の場合: NO_XML_SUPPORT() マクロによりエラーとなります。

保守担当者向けメモ

  • 本関数は libxml2 の XPath コンパイル機能に強く依存しています。
  • colnum の範囲チェックは本関数内では行われていないため、呼び出し側で xpathscomp 配列のサイズを超えないよう保証する必要があります。
