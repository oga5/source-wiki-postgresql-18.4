EncodeDateTime

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/dt_common.c:752

目的

struct tm 形式の日付・時刻情報を、指定されたスタイル（ISO, SQL, German, Postgres）に基づき文字列へ変換（エンコード）します。

シグネチャまたは形状

void EncodeDateTime(struct tm *tm, fsec_t fsec, bool print_tz, int tz, const char *tzn, int style, char *str, bool EuroDates)

入力と出力

  • **入力**:
  • tm: 変換対象の日付・時刻構造体。
  • fsec: 小数秒（fractional seconds）。
  • print_tz: タイムゾーン情報を出力するかどうかのフラグ。
  • tz: タイムゾーンオフセット（秒単位）。
  • tzn: タイムゾーン名（省略可）。
  • style: 出力フォーマット（USE_ISO_DATES, USE_SQL_DATES, USE_GERMAN_DATES, USE_POSTGRES_DATES）。
  • str: 出力先バッファ（呼び出し側で十分なサイズを確保する必要あり）。
  • EuroDates: 欧州形式（DD/MM）を使用するかどうかのフラグ。
  • **出力**: なし（str バッファへの書き込み）。

前提条件と不変条件

  • tm->tm_isdst < 0 の場合、print_tz は強制的に false となります。
  • tm->tm_year <= 0 の場合、歴史的な「0年」を考慮し、BC（紀元前）表記が付与されます。
  • fsec != 0 の場合、小数秒が追加され、末尾の不要なゼロは TrimTrailingZeros によって削除されます。

アルゴリズムまたは振る舞い

1. style 引数に基づき switch 文でフォーマットを分岐します。 2. 各スタイルに応じた順序（ISO: YYYY-MM-DD, SQL/German: DD/MM または MM/DD 等）で sprintf を使用してバッファへ書き込みます。 3. print_tz が有効な場合、タイムゾーン名があればそれを付与し、なければオフセットを +HH:MM または +HH 形式で計算・追記します。 4. USE_POSTGRES_DATES の場合、date2j を用いて曜日を算出し、文字列形式に含めます。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し関係は、提供された抜粋からは特定できません。
  • **呼び出し先**: TrimTrailingZeros, date2j（USE_POSTGRES_DATES 時）、標準ライブラリの sprintf, strlen, memcpy, strcpy, abs。

関連データ構造

  • struct tm: 標準的な時刻構造体。
  • fsec_t: 小数秒を保持する型。
  • datetkn: 日付・時刻キーワード管理用構造体（本関数内では直接使用されませんが、モジュール内で関連）。

失敗モードと境界条件

  • **バッファオーバーフロー**: str バッファのサイズが不足している場合、sprintf によるメモリ破壊の危険があります。
  • **タイムゾーン名**: tzn が非ASCII文字を含む場合、%.*s による切り詰め処理がエンコーディングを考慮しないため、不正な文字列になる可能性があります（ただし、IANAデータベースのTZ略称はすべてASCIIであると想定されています）。

保守担当者向けメモ

  • 本関数はECPGライブラリの一部であり、バックエンド側の同名関数とは引数構成（EuroDates の有無など）が異なります。
  • 出力バッファのサイズ管理は呼び出し側の責任であるため、変更を加える際はバッファ長に関するドキュメントを更新してください。
