append_num_word

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/cash.c:38

目的

数値を英語の単語表現（例: 123 → "one hundred twenty three"）に変換し、指定された StringInfo バッファへ追加します。主に通貨の読み上げ表現を生成するために使用されます。

シグネチャまたは形状

static void append_num_word(StringInfo buf, Cash value)

入力と出力

  • **入力**:
  • buf: 文字列を格納する StringInfo 構造体へのポインタ。
  • value: 変換対象の Cash 型（int64）の値。
  • **出力**: なし（buf の内容を更新する副作用を持つ）。

前提条件と不変条件

  • value は非負の整数として扱われます。
  • 内部的に small 配列（0〜20および10刻みの単語）を参照するため、value が負の値である場合の動作は定義されていません。

アルゴリズムまたは振る舞い

1. **基底ケース**: value が 20 以下の場合は、small 配列から直接文字列を取得してバッファに追加します。 2. **100の倍数**: 100 で割り切れる場合は、「[数値] hundred」の形式で追加します。 3. **100以上**:

  • 100 以上の場合は「[数値] hundred」を先頭に追加し、残りの下2桁（tu）に対して再帰的または条件分岐による処理を行います。
  • 下2桁が 10 より大きく 10 の倍数の場合は「[百の位] hundred [十の位]」と出力します。
  • 下2桁が 20 未満の場合は「[百の位] hundred and [下2桁]」と出力します。
  • それ以外の場合は「[百の位] hundred [十の位] [一の位]」と出力します。

    • 4. **100未満**: 100 未満の数値に対して、同様の条件分岐を用いて単語をバッファに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/cash.c 内の cash_words 関数（直接の言及はないが、同ファイル内の複数箇所から参照されている）。
  • **呼び出し先**: appendStringInfo および appendStringInfoString（StringInfo 操作用マクロ/関数）。

関連データ構造

  • Cash: int64 の別名。
  • StringInfo: 文字列構築用の動的バッファ構造体。

失敗モードと境界条件

  • **境界条件**: value が 0 の場合、"zero" と出力されます。
  • **制限**: 内部ロジックは 1000 未満の数値を想定した構成となっており、それ以上の値に対する正確な英語表現の生成は保証されていません。

保守担当者向けメモ

  • 本関数は cash_words 関数から呼び出される補助関数です。
  • small 配列のインデックス計算は value の値に依存しているため、ロジックを変更する際は tu（下2桁）の計算と配列アクセスの整合性に注意してください。
