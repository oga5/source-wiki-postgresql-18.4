FileFdwExecutionState

定義

  • 種別: struct
  • 定義箇所: contrib/file_fdw/file_fdw.c:113

目的

外部データラッパー（FDW）によるスキャン実行時のコンテキストを保持するための構造体です。COPY コマンドのインフラストラクチャを利用して外部ソースからデータを読み込む際の実行状態を管理します。

シグネチャまたは形状

typedef struct FileFdwExecutionState
{
	char	   *filename;		/* 読み込み対象のファイルパスまたはプログラム名 */
	bool		is_program;		/* true の場合、filename は OS コマンドとして扱われる */
	List	   *options;		/* filename と is_program を除く、マージ済みの COPY オプション */
	CopyFromState cstate;		/* COPY コマンドの実行状態を保持する内部構造体 */
} FileFdwExecutionState;

入力と出力

  • **入力**: 外部テーブル定義時に指定されたオプションおよび実行時のコンテキスト情報。
  • **出力**: cstate を通じて COPY API に渡され、最終的に TupleTableSlot へデータがマッピングされます。

前提条件と不変条件

  • cstate は BeginCopyFrom によって初期化され、EndCopyFrom によって解放される必要があります。
  • options リストには、filename および is_program 以外の COPY 関連オプションが含まれます。

アルゴリズムまたは振る舞い

1. **初期化**: fileBeginForeignScan 内で palloc によりメモリが確保され、BeginCopyFrom を呼び出してストリームを開きます。 2. **データ取得**: fileIterateForeignScan がこの構造体を参照し、NextCopyFrom を呼び出して外部ソースから 1 行ずつデータを取得します。 3. **終了処理**: fileEndForeignScan が EndCopyFrom を呼び出し、リソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/file_fdw/file_fdw.c 内の fileBeginForeignScan（生成）、fileIterateForeignScan（参照）、fileEndForeignScan（参照および解放）。
  • **呼び出し先**: commands/copy.h で定義される CopyFromState 関連の API（BeginCopyFrom, NextCopyFrom, EndCopyFrom）。

関連データ構造

  • CopyFromState: COPY コマンドの実行状態を管理するコア構造体。
  • FileFdwPlanState: プランニング段階で利用される対応する構造体。

失敗モードと境界条件

  • filename が無効な場合や、is_program が true で実行権限がない場合、BeginCopyFrom の呼び出し時にエラーが発生します。
  • COPY オプションの解析に失敗した場合、実行開始時にエラーが送出されます。

保守担当者向けメモ

  • 本構造体は COPY コマンドの内部 API に強く依存しています。src/backend/commands/copyfrom.c の仕様変更が直接影響するため、修正時は注意してください。
  • cstate のライフサイクル管理（メモリリーク防止）は、fileEndForeignScan で確実に行われる必要があります。
