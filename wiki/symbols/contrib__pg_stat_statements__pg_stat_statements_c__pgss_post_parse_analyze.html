pgss_post_parse_analyze

定義

  • 種別: function
  • 定義箇所: contrib/pg_stat_statements/pg_stat_statements.c:836

目的

PostgreSQLの解析フェーズ終了時にフックされ、クエリの正規化と統計情報の初期登録を行う。特に、定数を含むクエリの正規化文字列を早期に保存するために使用される。

シグネチャまたは形状

static void pgss_post_parse_analyze(ParseState *pstate, Query *query, JumbleState *jstate)

入力と出力

  • **入力**:
  • pstate: 解析状態（ソーステキストを含む）。
  • query: 解析済みのクエリツリー。
  • jstate: クエリの正規化に必要な定数位置情報を含む JumbleState。
  • **出力**: なし（副作用として共有メモリ上のハッシュテーブルを更新）。

前提条件と不変条件

  • pgss（共有状態）および pgss_hash が初期化されていること。
  • pgss_enabled(nesting_level) が真であること（再帰呼び出しの深さが追跡対象範囲内であること）。

アルゴリズムまたは振る舞い

1. **フックの連鎖**: prev_post_parse_analyze_hook が設定されている場合、先にそれを実行する。 2. **ガード節**: 共有メモリが未初期化、または追跡が無効な場合は即座にリターンする。 3. **EXECUTE文の処理**: pgss_track_utility が有効かつ ExecuteStmt の場合、queryId を 0 にリセットして終了する（基底となる PREPARE 文の統計を優先するため）。 4. **正規化情報の保存**: jstate に定数の位置情報（clocations_count > 0）が含まれる場合、pgss_store を呼び出し、正規化されたクエリ文字列をハッシュテーブルに登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの post_parse_analyze_hook を介して呼び出される。
  • **呼び出し先**:
  • prev_post_parse_analyze_hook（存在する場合）
  • pgss_store（正規化が必要な場合）
  • pgss_enabled（マクロまたは関数）

関連データ構造

  • ParseState: 解析中のクエリ情報。
  • Query: 解析されたクエリツリー。
  • JumbleState: クエリの正規化（定数置換）に必要なメタデータ。

失敗モードと境界条件

  • jstate が NULL の場合、または定数を含まないクエリの場合、pgss_store は呼び出されず、正規化文字列の早期登録は行われない。
  • pgss_track_utility が無効な場合、ExecuteStmt の queryId リセット処理はスキップされる。

保守担当者向けメモ

  • 本関数は post_parse_analyze_hook に登録されるため、他の拡張機能との競合を避けるよう prev_post_parse_analyze_hook の呼び出し順序を維持すること。
  • pgss_store への引数は多いため、変更時は pg_stat_statements.c 内の定義と整合性を確認すること。
