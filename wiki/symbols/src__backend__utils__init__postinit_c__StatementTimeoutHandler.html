StatementTimeoutHandler

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/postinit.c:1359

目的

ステートメント実行時間が制限を超過した際に、バックエンドプロセスに対してシグナルを送信し、処理を中断させるためのハンドラです。

シグネチャまたは形状

static void StatementTimeoutHandler(void)

入力と出力

  • 入力: なし（グローバル変数 ClientAuthInProgress および MyProcPid を参照）
  • 出力: なし（副作用としてシグナルを送信）

前提条件と不変条件

  • RegisterTimeout(STATEMENT_TIMEOUT, StatementTimeoutHandler) を通じてタイムアウト機構に登録されている必要があります。
  • MyProcPid が現在のバックエンドプロセスの PID を正しく保持している必要があります。

アルゴリズムまたは振る舞い

1. 認証処理中（ClientAuthInProgress が真）であれば、送信するシグナルを SIGTERM に設定します。それ以外の場合は SIGINT を使用します。 2. HAVE_SETSID が定義されている環境では、プロセスグループ全体に対して kill(-MyProcPid, sig) を試みます。 3. 最後に、現在のプロセス自身に対して kill(MyProcPid, sig) を実行し、シグナルを発生させます。

呼び出し元と呼び出し先

  • 呼び出し元: タイムアウト管理サブシステム（RegisterTimeout により登録）。
  • 呼び出し先: kill() システムコール。

関連データ構造

  • ClientAuthInProgress: 認証処理中かどうかを示すグローバルフラグ。
  • MyProcPid: 現在のバックエンドプロセスのプロセスID。

失敗モードと境界条件

  • kill() システムコールが失敗した場合、シグナルは送信されません。
  • プロセスグループへのシグナル送信（kill(-MyProcPid, ...)）は、環境が HAVE_SETSID をサポートしている場合にのみ実行されます。

保守担当者向けメモ

  • 本関数は postinit.c 内で RegisterTimeout に登録されるコールバックとして設計されています。
  • 認証フェーズと通常のクエリ実行フェーズで送信するシグナルを切り替えることで、認証タイムアウトとステートメントタイムアウトの両方に対応しています。
