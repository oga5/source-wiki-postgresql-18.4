build_startup_packet

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-protocol3.c:2428

目的

PostgreSQL プロトコル 3.0 における接続開始用パケット（StartupMessage）を構築する。

シグネチャまたは形状

static size_t build_startup_packet(const PGconn *conn, char *packet, const PQEnvironmentOption *options)

入力と出力

  • **入力**:
  • conn: 接続状態を保持する PGconn 構造体。
  • packet: パケットを書き込むバッファ（NULL の場合はサイズ計算のみを行う）。
  • options: 環境変数から GUC 設定を読み取るための PQEnvironmentOption 配列。
  • **出力**:
  • 構築されたパケットの合計サイズ（バイト単位）。エラー発生時は 0 を返す。

前提条件と不変条件

  • packet が NULL でない場合、十分なメモリ領域が確保されている必要がある。
  • pg_add_size_overflow を使用して、パケット構築中のサイズ計算におけるオーバーフローを防止している。

アルゴリズムまたは振る舞い

1. プロトコルバージョン番号をネットワークバイトオーダーに変換してパケット先頭に書き込む。 2. user, database, replication, options, application_name, client_encoding の順で、conn 内の値をパケットに追加する。 3. options 配列に基づき、環境変数から取得した GUC 設定をパケットに追加する（値が "default" の場合はスキップ）。 4. 各オプションは key と value のペアとして、それぞれヌル終端文字列として書き込まれる。 5. 最後にパケット全体をヌル終端する。 6. packet が NULL の場合は、書き込みを行わずサイズ計算のみを実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-protocol3.c 内の接続開始処理箇所（2404行目および2413行目）。
  • **呼び出し先**: pg_hton32, pg_add_size_overflow, strcpy, strlen, getenv, pg_strcasecmp。

関連データ構造

  • PGconn: 接続パラメータを保持する構造体。
  • PQEnvironmentOption: 環境変数と GUC 名の対応を定義する構造体。

失敗モードと境界条件

  • サイズ計算中にオーバーフローが発生した場合、0 を返して処理を中断する。
  • conn 内の各文字列フィールドが空の場合、その項目はパケットに追加されない。

保守担当者向けメモ

  • 本関数はプロトコル 3.0 専用である。
  • ADD_STARTUP_OPTION マクロは、packet が NULL の場合でもサイズ計算を正しく行うよう設計されている。変更時はこのマクロの不変条件に注意すること。
