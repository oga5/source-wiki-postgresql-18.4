ReadBuffer_common

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/bufmgr.c:1192

目的

PostgreSQLのバッファ読み込み処理における共通ロジックを実装し、指定されたブロックのバッファ取得、初期化、またはディスクからの読み込みを統合的に管理します。

シグネチャまたは形状

static pg_attribute_always_inline Buffer ReadBuffer_common(
    Relation rel, 
    SMgrRelation smgr, 
    char smgr_persistence,
    ForkNumber forkNum,
    BlockNumber blockNum, 
    ReadBufferMode mode,
    BufferAccessStrategy strategy
)

入力と出力

  • **入力**:
  • rel: 対象のリレーション（NULL可）。
  • smgr: ストレージマネージャのハンドル。
  • smgr_persistence: リレーションがNULLの場合の永続性フラグ。
  • forkNum: 読み込むフォーク番号。
  • blockNum: 読み込むブロック番号（P_NEWを指定すると新規拡張）。
  • mode: 読み込みモード（ロックの有無やエラー時の挙動を制御）。
  • strategy: バッファアクセス戦略。
  • **出力**: 取得または読み込みが完了した Buffer（バッファID）。

前提条件と不変条件

  • blockNum が P_NEW の場合、ExtendBufferedRel が呼び出されます。
  • mode が RBM_ZERO_AND_CLEANUP_LOCK または RBM_ZERO_AND_LOCK の場合、ディスク読み込みを行わず、バッファのピン留めとゼロクリア・ロック処理が優先されます。

アルゴリズムまたは振る舞い

1. **新規拡張**: blockNum == P_NEW の場合、ExtendBufferedRel を呼び出し即時終了します。 2. **ゼロクリアモード**: RBM_ZERO_AND_... モードの場合、PinBufferForBlock でバッファを確保し、ZeroAndLockBuffer で初期化して返します。 3. **標準読み込み**: 上記以外の場合、StartReadBuffer を呼び出し、同期読み込みフラグを立てて WaitReadBuffers で完了を待機します。 4. **戻り値**: 最終的に確保された Buffer を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/bufmgr.c 内の各種 ReadBuffer 系関数（ReadBufferExtended 等）。
  • **呼び出し先**: ExtendBufferedRel, PinBufferForBlock, ZeroAndLockBuffer, StartReadBuffer, WaitReadBuffers。

関連データ構造

  • Buffer: バッファID。
  • ReadBuffersOperation: I/O 操作の状態を保持する構造体。
  • Relation, SMgrRelation: リレーションおよびストレージ管理情報。

失敗モードと境界条件

  • blockNum が P_NEW の際の拡張失敗は、呼び出し先の ExtendBufferedRel に依存します。
  • mode が RBM_ZERO_ON_ERROR の場合、読み込みエラー時にバッファをゼロクリアするフラグが設定されます。

保守担当者向けメモ

  • 本関数は pg_attribute_always_inline が指定されており、インライン展開を前提としています。
  • P_NEW の処理については、拡張ロックの競合を避けるため、可能な限り ExtendBufferedRel を直接利用することが推奨されています。
