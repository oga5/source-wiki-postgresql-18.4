strncoll_icu_utf8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_icu.c:472

目的

UTF-8エンコーディング環境において、ICUライブラリの照合機能を用いて2つの文字列を比較する。

シグネチャまたは形状

int strncoll_icu_utf8(const char *arg1, ssize_t len1, const char *arg2, ssize_t len2, pg_locale_t locale)

入力と出力

  • **入力**:
  • arg1, arg2: 比較対象の文字列（UTF-8）。
  • len1, len2: 各文字列の長さ。
  • locale: ICUプロバイダーを含むロケール情報構造体。
  • **出力**:
  • 比較結果を整数で返す（負: arg1 < arg2、0: arg1 == arg2、正: arg1 > arg2）。

前提条件と不変条件

  • locale->provider が COLLPROVIDER_ICU であること。
  • データベースのエンコーディングが PG_UTF8 であること。

アルゴリズムまたは振る舞い

1. ICUの ucol_strcollUTF8 関数を呼び出し、指定されたロケール設定に基づいて文字列を比較する。 2. ICUの内部状態（UErrorCode）を監視し、エラーが発生した場合は ereport(ERROR, ...) を通じてPostgreSQLのトランザクションメカニズムへ制御を戻す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale_icu.c 内のメソッドテーブル collate_methods_icu_utf8 から参照されている。
  • **呼び出し先**:
  • ucol_strcollUTF8 (ICUライブラリ関数)
  • u_errorName (ICUライブラリ関数)
  • ereport (PostgreSQLエラー報告マクロ)

関連データ構造

  • pg_locale_t: ロケール情報を保持する構造体。
  • UCollator: ICUの照合順序オブジェクト（locale->info.icu.ucol 経由でアクセス）。

失敗モードと境界条件

  • ICUの照合処理でエラーが発生した場合、U_FAILURE が判定され、エラーメッセージと共にトランザクションがアボートされる。

保守担当者向けメモ

  • 本関数はUTF-8環境専用の最適化実装である。
  • ICU 53未満のバージョンでは ucol_strcollUTF8 が不安定であるため、ビルド時のマクロ HAVE_UCOL_STRCOLLUTF8 による制御が前提となっている。
