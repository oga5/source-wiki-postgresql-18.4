lazy_scan_prune

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/vacuumlazy.c:1943

目的

ヒープページ内のHOT更新チェーンの剪定（Pruning）およびタプルの凍結（Freeze）を行い、不要になったタプルを特定して可視性マップ（VM）を更新する。

シグネチャまたは形状

static int lazy_scan_prune(LVRelState *vacrel, Buffer buf, BlockNumber blkno, Page page, Buffer vmbuffer, bool all_visible_according_to_vm, bool *has_lpdead_items, bool *vm_page_frozen)

入力と出力

  • **入力**:
  • vacrel: VACUUM実行状態を保持する構造体。
  • buf, page: 対象ヒープページのバッファとページポインタ。
  • vmbuffer: 可視性マップのバッファ。
  • all_visible_according_to_vm: VM上の可視性フラグ状態。
  • **出力**:
  • 戻り値: 削除されたタプル数（int）。
  • has_lpdead_items: ページ内に削除対象のデッドタプルが存在するかを示すフラグ。
  • vm_page_frozen: ページが新たに凍結されたかを示すフラグ。
  • vacrel 内部の統計情報および TidStore が更新される。

前提条件と不変条件

  • BufferGetBlockNumber(buf) == blkno であること。
  • vacrel->NewRelminMxid および vacrel->NewRelfrozenXid が有効であること。
  • インデックスが存在しない場合、HEAP_PAGE_PRUNE_MARK_UNUSED_NOW オプションが適用される。

アルゴリズムまたは振る舞い

1. heap_page_prune_and_freeze を呼び出し、ページ内のタプル剪定と凍結を実行する。 2. 剪定結果に基づき、vacrel の統計情報（削除数、凍結数、ライブタプル数等）を更新する。 3. 削除対象のデッドタプル（LP_DEAD）がある場合、そのオフセットをソートして dead_items_add を通じ TidStore に蓄積する。 4. ページが全可視（all-visible）と判定された場合、VMのビットを更新し、必要に応じてバッファをダーティにする。 5. デバッグビルドでは、heap_page_is_all_visible を用いて剪定結果との整合性を検証する。

呼び出し元と呼び出し先

  • **呼び出し元**: lazy_scan_heap（src/backend/access/heap/vacuumlazy.c:1391 等）。
  • **呼び出し先**: heap_page_prune_and_freeze, qsort, dead_items_add, PageSetAllVisible, MarkBufferDirty, visibilitymap_set, heap_page_is_all_visible（デバッグ時）。

関連データ構造

  • LVRelState: VACUUMの全体状態を管理。
  • PruneFreezeResult: 剪定および凍結処理の結果を保持。
  • TidStore: 削除対象TIDの格納先。

失敗モードと境界条件

  • ページ内にタプルが存在する場合、vacrel->nonempty_pages が更新される。
  • VMビットとページ状態の不整合が検出された場合、VMの再設定が行われる。
  • presult.lpdead_items > 0 の場合、インデックス削除のための追跡対象となる。

保守担当者向けメモ

  • 本関数は lazy_scan_heap のメインループ内で呼び出される。
  • dead_items_add に渡すオフセットはソート済みである必要があるため、qsort が必須である。
  • VMの更新ロジックは、クラッシュ後のリカバリ状況を考慮し、ビットがクリアされている場合でも強制的に同期を行う設計となっている。
