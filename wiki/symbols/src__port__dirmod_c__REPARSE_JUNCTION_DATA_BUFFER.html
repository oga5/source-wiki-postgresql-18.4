REPARSE_JUNCTION_DATA_BUFFER

定義

  • 種別: struct
  • 定義箇所: src/port/dirmod.c:196

目的

Windows ファイルシステムにおけるジャンクションポイント（ディレクトリのシンボリックリンク相当）のメタデータを保持し、DeviceIoControl を介したリパースポイント操作を可能にするための構造体です。

シグネチャまたは形状

typedef struct {
    DWORD        ReparseTag;
    WORD         ReparseDataLength;
    WORD         Reserved;
    WORD         SubstituteNameOffset;
    WORD         SubstituteNameLength;
    WORD         PrintNameOffset;
    WORD         PrintNameLength;
    WCHAR        PathBuffer[FLEXIBLE_ARRAY_MEMBER];
} REPARSE_JUNCTION_DATA_BUFFER;

入力と出力

  • **入力**: ジャンクションポイント作成または読み取り時に、パス情報（SubstituteName および PrintName）を PathBuffer に格納して使用します。
  • **出力**: DeviceIoControl のバッファとして渡され、ファイルシステムに対してジャンクションポイントの定義を伝達します。

前提条件と不変条件

  • Windows 環境（WIN32）でのみ使用されます。
  • PathBuffer は可変長配列として定義されており、構造体サイズは offsetof(REPARSE_JUNCTION_DATA_BUFFER, PathBuffer) を含めて計算する必要があります。

アルゴリズムまたは振る舞い

  • pgsymlink および pgreadlink において、ジャンクションポイントのターゲットパスを操作するためのメモリレイアウトとして利用されます。
  • DeviceIoControl を呼び出す際、FSCTL_SET_REPARSE_POINT 制御コードと共にこの構造体のインスタンスが渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/dirmod.c 内の pgsymlink および pgreadlink 関数が、この構造体をローカルバッファとしてキャストして使用しています。
  • **呼び出し先**: Windows API の DeviceIoControl に渡されます。

関連データ構造

  • MAX_PATH: PathBuffer のサイズ確保時に使用される定数。
  • WCHAR: Windows のワイド文字型。

失敗モードと境界条件

  • PathBuffer の領域確保が不十分な場合、バッファオーバーフローのリスクがあります。実装では MAX_PATH * sizeof(WCHAR) を用いて十分な領域を確保しています。
  • Windows SDK のバージョンや環境により、ReparseTag の定義が期待通りに機能しない可能性があります。

保守担当者向けメモ

  • 本構造体は Windows の低レベルなファイルシステム API に依存しています。修正を行う際は winioctl.h の定義と整合性が取れているか確認してください。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズ計算には必ず offsetof を使用してください。
