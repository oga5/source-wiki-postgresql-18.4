TuplesortPublic

定義

  • 種別: struct
  • 定義箇所: src/include/utils/tuplesort.h:164

目的

ソート対象のデータ型に依存する処理（比較、読み書き、リソース解放）を抽象化し、汎用的なソートロジックから切り離すための関数ポインタ群および状態管理用メタデータを保持する。

シグネチャまたは形状

typedef struct {
    SortTupleComparator comparetup;
    SortTupleComparator comparetup_tiebreak;
    void (*removeabbrev) (Tuplesortstate *state, SortTuple *stups, int count);
    void (*writetup) (Tuplesortstate *state, LogicalTape *tape, SortTuple *stup);
    void (*readtup) (Tuplesortstate *state, SortTuple *stup, LogicalTape *tape, unsigned int len);
    void (*freestate) (Tuplesortstate *state);
    MemoryContext maincontext;
    MemoryContext sortcontext;
    MemoryContext tuplecontext;
    bool haveDatum1;
    int nKeys;
    SortSupport sortKeys;
    SortSupport onlyKey;
    int sortopt;
    bool tuples;
    void *arg;
} TuplesortPublic;

入力と出力

  • **入力**: tuplesort_begin_xxx 系関数によって初期化され、各ソートバリアントに応じた関数ポインタとコンテキストが設定される。
  • **出力**: ソート処理の各フェーズ（比較、テープへの書き出し、メモリへの読み込み）において、登録された関数ポインタを介して具体的な操作が実行される。

前提条件と不変条件

  • comparetup は qsort_arg_comparator 規約に従う必要がある。
  • maincontext, sortcontext, tuplecontext はソートのライフサイクルを通じて適切に管理される必要がある。
  • arg フィールドには、各ソートバリアント固有のデータ構造が格納される。

アルゴリズムまたは振る舞い

  • ソートロジックの抽象化: Tuplesortstate が保持する本構造体を通じて、ヒープタプル、インデックス用タプル、Datum 等の型を意識せずにソートを実行する。
  • 外部ソート対応: メモリ制限を超えた場合、writetup および readtup を用いて一時ファイル（テープ）との間でタプルの転送を行う。
  • リソース管理: freestate を通じて、arg フィールドに確保されたバリアント固有のリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/tuplesort.c および src/backend/utils/sort/tuplesortvariants.c 内の各ソート処理関数。
  • **呼び出し先**: tuplesort_begin_xxx によって設定された各バリアント固有の関数実装。

関連データ構造

  • Tuplesortstate: 本構造体を内部に保持するソート状態管理構造体。
  • SortTuple: ソート対象のデータ単位。
  • SortSupport: ソートキーの比較サポート情報。
  • LogicalTape: 外部ソート用の一時ファイル操作インターフェース。

失敗モードと境界条件

  • haveDatum1 が false の場合、一部のソート最適化機能が無効化される。
  • freestate が NULL の場合、バリアント固有の解放処理はスキップされる。
  • onlyKey が設定されていない場合、一部の比較最適化が利用できない可能性がある。

保守担当者向けメモ

  • 本構造体は Tuplesortstate の一部として機能する。TuplesortstateGetPublic(state) を介してアクセスされることが多い。
  • arg フィールドに格納される構造体は、CLUSTER、IndexTuple、Datum 等の各ケースで異なるため、変更時は tuplesortvariants.c の整合性に注意すること。
