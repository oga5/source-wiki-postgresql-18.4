HeapTupleData

定義

  • 種別: struct
  • 定義箇所: src/include/access/htup.h:62

目的

PostgreSQLのヒープタプルをメモリ上で管理するためのハンドル構造体です。ディスク上のデータページとメモリ上のタプル表現の橋渡しを行い、タプルの位置情報やメタデータを保持します。

シグネチャまたは形状

typedef struct HeapTupleData
{
    uint32          t_len;      /* t_dataの長さ */
    ItemPointerData t_self;     /* 自身の物理位置（ItemPointer） */
    Oid             t_tableOid; /* タプルが属するテーブルのOID */
    HeapTupleHeader t_data;     /* タプルヘッダーおよびデータへのポインタ */
} HeapTupleData;

入力と出力

  • **入力**: 構造体生成時に t_len（データ長）、t_self（位置）、t_tableOid（テーブルID）、t_data（実データへのポインタ）が設定されます。
  • **出力**: 本構造体を通じて、タプルのヘッダー情報や実データにアクセス可能です。

前提条件と不変条件

  • t_data が指すメモリ領域が、バッファ上のデータなのか palloc されたメモリなのかを本構造体自体は判別できません。利用側でメモリの生存期間を管理する必要があります。
  • heap_form_tuple 等で生成される場合、HeapTupleData とタプル本体が連続したメモリ領域として確保されることがあります。

アルゴリズムまたは振る舞い

  • タプルを指し示すハンドルとして機能します。
  • t_data を介して HeapTupleHeaderData（トランザクション情報やNULLビットマップ等）へアクセスし、タプルの属性値を取得します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • contrib/hstore/hstore_io.c
  • contrib/pg_visibility/pg_visibility.c
  • contrib/pgstattuple/pgstatapprox.c
  • src/backend/access/heap/heapam.c
  • **呼び出し先**:
  • 本構造体はデータ構造定義であり、直接的な関数呼び出しは行いません。タプル操作には heap_form_tuple や各種アクセサ関数が利用されます。

関連データ構造

  • HeapTuple: HeapTupleData へのポインタ型。
  • HeapTupleHeader: タプルのヘッダー部分を指す型。
  • ItemPointerData: タプルの物理的な位置情報。
  • HeapTupleHeaderData: タプルの内部構造（src/include/access/htup_details.h で定義）。

失敗モードと境界条件

  • t_data が指すポインタが無効なメモリ領域を指している場合、セグメンテーション違反やデータ破損を引き起こします。
  • タプルがディスクバッファから解放された後に t_data を参照すると、不正なデータにアクセスする可能性があります。

保守担当者向けメモ

  • 本構造体はタプルの「ハンドル」であり、実体（t_data）のメモリ管理責任は呼び出し側にあります。
  • タプルの詳細な内部構造については src/include/access/htup_details.h を参照してください。
