pg_mb_radix_conv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conv.c:372

目的

ラディックスツリー（基数木）構造を用いて、マルチバイト文字を効率的に検索および変換します。

シグネチャまたは形状

static inline uint32 pg_mb_radix_conv(const pg_mb_radix_tree *rt, int l, unsigned char b1, unsigned char b2, unsigned char b3, unsigned char b4)

入力と出力

  • **入力**:
  • rt: 変換に使用する pg_mb_radix_tree 構造体へのポインタ。
  • l: 文字のバイト長（1〜4）。
  • b1〜b4: 入力文字の各バイト値。
  • **出力**: 変換後のコードポイント（uint32）。変換失敗時は 0 を返します。

前提条件と不変条件

  • rt は有効な pg_mb_radix_tree 構造体を指している必要があります。
  • l が 1〜4 の範囲外である場合、または指定されたバイト列がツリーの有効範囲外である場合、関数は 0 を返します。

アルゴリズムまたは振る舞い

1. **バイト長判定**: l に基づき、4バイトから1バイトまでの処理分岐を行います。 2. **妥当性検証**: 指定されたバイト値が、ツリー内の各階層で定義された下限（_lower）および上限（_upper）の範囲内にあるかを確認します。範囲外であれば即座に 0 を返します。 3. **ツリー探索**:

  • rt->chars32 が存在する場合は uint32 配列として、存在しない場合は rt->chars16 を uint16 配列として参照します。
  • 各バイトをインデックスとして、ツリーのルートから順次オフセットを計算し、次の階層のインデックスを特定します。

    • 4. **結果取得**: 最終的な階層に到達した時点で、対応するコードポイントを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mb/conv.c 内の変換処理（行653、788付近）から呼び出されます。
  • **呼び出し先**: 直接的な関数呼び出しはありませんが、引数として渡された pg_mb_radix_tree 構造体のメンバ配列（chars32 または chars16）にアクセスします。

関連データ構造

  • pg_mb_radix_tree: 変換テーブルおよびツリー構造を保持する構造体（src/include/mb/pg_wchar.h で定義）。

失敗モードと境界条件

  • **変換失敗**: 入力バイト列がツリーの範囲外である場合、または対応するエントリが存在しない場合、0 を返します。
  • **境界条件**: l が 1〜4 以外の値である場合、またはツリーの構造が不整合な場合、予期せぬ動作や 0 の返却が発生します。

保守担当者向けメモ

  • 本関数はパフォーマンスに非常に敏感なため、インライン展開されることを前提としています。
  • chars32 と chars16 の両方に対応しているため、ツリーのメモリレイアウトを変更する際は、インデックス計算のロジック（bN + idx - rt->bN_M_lower）が整合しているか確認してください。
