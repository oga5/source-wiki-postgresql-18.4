XLogRecordBlockHeader

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlogrecord.h:103

目的

WAL（Write-Ahead Logging）レコードに関連付けられた各データブロックのメタデータを保持し、WALレコードの物理的な構成を定義します。

シグネチャまたは形状

typedef struct XLogRecordBlockHeader
{
    uint8       id;             /* ブロック参照ID */
    uint8       fork_flags;     /* リレーション内のフォークおよびフラグ */
    uint16      data_length;    /* ペイロードのバイト数（ページイメージを除く） */
} XLogRecordBlockHeader;

入力と出力

本構造体はデータ構造の定義であり、関数ではないため直接的な入出力はありません。WAL生成時には XLogInsert 等の処理を通じて値が書き込まれ、WAL解析時には XLogReader がこの構造体を用いてストリームをデコードします。

前提条件と不変条件

  • **アライメント**: 本構造体はメモリ上で整列（アライメント）されていません。メモリ操作を行う際は、必ず整列されたバッファにコピーしてからアクセスする必要があります。
  • **後続データ**: 構造体の直後に以下のデータが続く場合があります（フラグに依存）：
  • BKPBLOCK_HAS_IMAGE が設定されている場合: XLogRecordBlockImageHeader
  • BKPBLOCK_SAME_REL が設定されていない場合: RelFileLocator
  • 最後に BlockNumber が続きます。

アルゴリズムまたは振る舞い

WALレコードの階層構造の一部として機能します。XLogRecord の直後に配置され、レコードが参照する各ブロックのメタデータを記述します。data_length はページイメージ以外のペイロードサイズを管理し、fork_flags はリレーション内のフォーク情報や特殊な更新フラグを保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xloginsert.c において、WALレコード構築時に本構造体のフィールドへの書き込みが行われます。
  • **呼び出し先**: src/backend/access/xlogreader.c において、WALストリームの解析時に本構造体が参照されます。
  • ※具体的な関数呼び出し関係の詳細は、提供されたソース抜粋からは特定できません。

関連データ構造

  • XLogRecord: レコード全体の固定長ヘッダー。
  • XLogRecordBlockImageHeader: フルページイメージが含まれる場合の追加ヘッダー。
  • RelFileLocator: BKPBLOCK_SAME_REL が設定されていない場合に続くリレーション識別子。
  • BlockNumber: 構造体の末尾に続くブロック番号。

失敗モードと境界条件

  • **ブロック数制限**: 1つのレコードが参照可能なブロック数は XLR_MAX_BLOCK_ID（32）に制限されています。
  • **データ長**: data_length はページイメージを含まないペイロードのサイズを指します。

保守担当者向けメモ

  • 本構造体はアライメントされていないため、直接ポインタキャストしてアクセスするとプラットフォームによってはクラッシュやパフォーマンス低下を招く恐れがあります。必ずバッファコピー経由で操作してください。
  • xl_info の上位4ビットはリソースマネージャが自由に使用可能です。
