BufferAlloc

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/bufmgr.c:1999

目的

共有バッファプールから指定されたブロックに対応するバッファ記述子（BufferDesc）を取得または割り当てます。

シグネチャまたは形状

static BufferDesc *BufferAlloc(SMgrRelation smgr, char relpersistence, ForkNumber forkNum,
                               BlockNumber blockNum, BufferAccessStrategy strategy,
                               bool *foundPtr, IOContext io_context)

入力と出力

  • **入力**:
  • smgr: 対象リレーションのストレージマネージャ。
  • relpersistence: リレーションの永続性（RELPERSISTENCE_PERMANENT 等）。
  • forkNum, blockNum: 取得対象の識別情報。
  • strategy: バッファ置換戦略（BufferAccessStrategy）。
  • io_context: I/O コンテキスト。
  • **出力**:
  • 戻り値: 取得したバッファの BufferDesc へのポインタ。
  • foundPtr: 既にバッファプール内に存在した場合は true、新規割り当ての場合は false を設定。

前提条件と不変条件

  • CurrentResourceOwner がピン留めを記録できる状態であること。
  • 呼び出し前に ReservePrivateRefCountEntry() が実行され、リソース所有権が確保されていること。
  • 新規割り当て時、対象バッファは参照カウントが 1 であり、かつ BM_TAG_VALID 等のフラグが立っていないクリーンな状態であること。

アルゴリズムまたは振る舞い

1. **検索**: BufTableLookup を使用し、指定されたタグがバッファプールに存在するか確認する。 2. **ヒット時**: PinBuffer でピン留めし、バッファを返す。 3. **ミス時**:

  • GetVictimBuffer で置換候補のバッファを取得する。
  • BufTableInsert でハッシュテーブルに新しいタグを挿入する。
  • 挿入競合が発生した場合は、確保したバッファを解放し、既存のバッファを再利用する。
  • 競合がない場合、バッファヘッダーをロックし、タグを更新して BM_TAG_VALID を付与する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/bufmgr.c 内の ReadBuffer_common 等から呼び出される。
  • **呼び出し先**: BufTableLookup, BufTableInsert, PinBuffer, GetVictimBuffer, StrategyFreeBuffer, LockBufHdr, UnlockBufHdr 等。

関連データ構造

  • BufferDesc: バッファの状態を管理する記述子。
  • BufferTag: バッファを識別する一意のタグ。
  • BufferAccessStrategy: バッファ置換の制御構造。

失敗モードと境界条件

  • ハッシュテーブルへの挿入時に他プロセスとの競合が発生した場合、確保した victim_buffer を解放し、競合相手のバッファを優先して使用する。
  • foundPtr が false の場合、バッファの内容は未初期化であるため、呼び出し側でディスクからの読み込みが必要となる。

保守担当者向けメモ

  • 本関数は LWLock（パーティションロック）の取得・解放を伴うため、デッドロックの回避に注意すること。
  • BM_PERMANENT フラグの付与条件（relpersistence や INIT_FORKNUM）はチェックポイント処理に直結するため、変更時は慎重に行うこと。
