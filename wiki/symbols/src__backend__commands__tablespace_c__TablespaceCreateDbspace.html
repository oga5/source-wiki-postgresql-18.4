TablespaceCreateDbspace

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tablespace.c:111

目的

指定されたテーブルスペース内に、特定のデータベース用のサブディレクトリを作成します。

シグネチャまたは形状

void TablespaceCreateDbspace(Oid spcOid, Oid dbOid, bool isRedo)

入力と出力

  • **入力**:
  • spcOid: 対象テーブルスペースの OID。
  • dbOid: 作成対象データベースの OID。
  • isRedo: WAL リプレイ中かどうかを示すフラグ。
  • **出力**: なし（成功時はディレクトリを作成し、失敗時はエラーを報告）。

前提条件と不変条件

  • spcOid および dbOid は有効な OID である必要があります。
  • GLOBALTABLESPACE_OID が指定された場合、本関数は何も行わず即座に復帰します（グローバルテーブルスペースにはデータベース別サブディレクトリが存在しないため）。

アルゴリズムまたは振る舞い

1. GetDatabasePath を使用して、対象データベースのパス文字列を取得します。 2. stat でディレクトリの存在を確認します。 3. ディレクトリが存在しない場合:

  • TablespaceCreateLock を排他モードで取得します。
  • ロック取得後に再確認を行い、他プロセスによって作成されていないことを確認します。
  • MakePGDirectory でディレクトリ作成を試みます。失敗時、isRedo が真であれば pg_mkdir_p を用いて再試行します。

    • 4. ディレクトリが既に存在する場合、それがディレクトリであることを確認します（ディレクトリでない場合はエラー）。 5. 最後に取得したパス文字列を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/smgr/md.c 内のコードから呼び出されることが確認されています。
  • **呼び出し先**:
  • GetDatabasePath, LWLockAcquire, LWLockRelease, MakePGDirectory, pg_mkdir_p, stat, pfree, ereport 等。

関連データ構造

  • Oid: データベースおよびテーブルスペースの識別子。
  • struct stat: ファイルシステム上の属性確認に使用。

失敗モードと境界条件

  • **ディレクトリ作成失敗**: MakePGDirectory または pg_mkdir_p が失敗した場合、エラーが発生します。ただし、isRedo が真かつ errno == ENOENT の場合は、WAL リプレイ時のディレクトリ欠損とみなして処理を継続します。
  • **型不一致**: 指定されたパスが既に存在し、かつディレクトリではない場合、ERRCODE_WRONG_OBJECT_TYPE でエラーとなります。
  • **競合**: TablespaceCreateLock により、DROP TABLESPACE 等との同時実行が制御されます。

保守担当者向けメモ

  • 本関数は、データベース作成時に物理的なディレクトリ構造を保証する役割を担います。
  • WAL リプレイ時の挙動（isRedo）は、スタンバイ側でのディレクトリ同期を考慮した設計となっています。
