domain_check_safe

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/domains.c:354

目的

ドメイン型に対して、例外を発生させずに制約（NOT NULL や CHECK 制約）の検証を行い、その結果を真偽値で返すための安全なインターフェースを提供します。

シグネチャまたは形状

bool domain_check_safe(Datum value, bool isnull, Oid domainType,
                       void **extra, MemoryContext mcxt,
                       Node *escontext);

入力と出力

  • **入力**:
  • value: 検証対象の Datum 値。
  • isnull: 値が NULL かどうかを示すフラグ。
  • domainType: 対象となるドメイン型の Oid。
  • extra: キャッシュされたドメイン情報を保持するためのポインタ（DomainIOData への参照）。
  • mcxt: 式評価に使用するメモリコンテキスト。
  • escontext: エラー報告用のコンテキスト（Node*）。
  • **出力**:
  • bool: 制約をすべて満たせば true、違反がある場合は false を返します。

前提条件と不変条件

  • extra 引数は、呼び出し元で初期化された DomainIOData 構造体へのポインタを保持している必要があります。
  • escontext が提供されている場合、検証失敗時のエラー詳細はそのコンテキストに記録されます。

アルゴリズムまたは振る舞い

内部関数 domain_check_internal を呼び出し、以下の処理を順次実行します。 1. DomainIOData を使用して、対象ドメインに関連付けられた制約リストを取得します。 2. NOT NULL 制約がある場合、isnull が true であれば検証失敗とみなします。 3. CHECK 制約がある場合、ExprContext を用いて式を評価します。 4. 評価結果が制約を満たさない場合、escontext にエラー情報を設定し、false を返します。 5. すべての制約を通過した場合、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c
  • src/backend/utils/adt/jsonfuncs.c
  • **呼び出し先**:
  • domain_check_internal（直接の委譲先）

関連データ構造

  • DomainIOData: ドメインの制約情報や評価用コンテキストをキャッシュする構造体。
  • ExprContext: CHECK 制約の評価に使用される実行コンテキスト。

失敗モードと境界条件

  • domainType が無効な場合や、キャッシュの不整合が発生した場合は予期せぬ動作を引き起こす可能性があります。
  • escontext が NULL の場合、検証失敗時の詳細なエラー報告が抑制される可能性があります。

保守担当者向けメモ

  • 本関数は domain_check() と共通のロジック（domain_check_internal）を利用しています。修正を行う際は、例外を投げる必要がある domain_check() との挙動の差異に注意してください。
  • extra 引数によるキャッシュ機構は、カタログアクセスを削減するために重要です。メモリ管理（ReScanExprContext 等）が適切に行われているか確認してください。
