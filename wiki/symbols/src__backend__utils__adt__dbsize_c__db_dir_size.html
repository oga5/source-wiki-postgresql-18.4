db_dir_size

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/dbsize.c:73

目的

指定されたディレクトリパス内の全ファイルのサイズを合計し、その総バイト数を返します。

シグネチャまたは形状

static int64 db_dir_size(const char *path)

入力と出力

  • **入力**: path (走査対象のディレクトリパス)
  • **出力**: int64 (ディレクトリ内の全ファイルサイズの合計値。ディレクトリが開けない場合は 0 を返す)

前提条件と不変条件

  • path で指定されたディレクトリが存在し、読み取り権限があること。
  • ループ内では CHECK_FOR_INTERRUPTS() が呼び出され、クエリのキャンセル要求を処理する。
  • . および .. ディレクトリは集計対象から除外される。

アルゴリズムまたは振る舞い

1. AllocateDir を使用してディレクトリを開く。失敗した場合は 0 を返す。 2. ReadDir を使用してディレクトリ内のエントリを反復処理する。 3. 各エントリに対して stat() を実行し、ファイルサイズ (st_size) を dirsize に加算する。 4. stat() 失敗時、ENOENT（ファイル消失）であれば無視し、それ以外は ereport(ERROR, ...) で処理を中断する。 5. 走査完了後、FreeDir でディレクトリを解放し、合計値を返す。

  • src/backend/utils/adt/dbsize.c:143

関連データ構造

  • struct dirent: ディレクトリ内のエントリ情報。
  • struct stat: ファイルのメタデータ（サイズ情報を含む）。
  • DIR: ディレクトリ記述子。

失敗モードと境界条件

  • **ディレクトリ不在/アクセス不可**: AllocateDir が NULL を返した場合、0 を返す。
  • **ファイル消失**: stat 実行時にファイルが削除された場合（ENOENT）、そのファイルは無視して処理を継続する。
  • **権限エラー**: stat が ENOENT 以外のエラーを返した場合、ereport(ERROR) によりトランザクションが中断される。

保守担当者向けメモ

  • 本関数は物理的なファイルシステム走査を行うため、大規模なディレクトリでは時間がかかる可能性があります。必ず CHECK_FOR_INTERRUPTS() を維持してください。
  • MAXPGPATH * 2 のバッファサイズは、パス連結時のオーバーフローを防ぐための設計です。
