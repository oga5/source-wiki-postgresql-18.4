BuildIndexValueDescription

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/genam.c:177

目的

インデックスのキー値を人間が読み取り可能な文字列形式（(col1, col2)=(val1, val2)）に変換する。主にエラーメッセージやログ出力での利用を想定している。

シグネチャまたは形状

char *BuildIndexValueDescription(Relation indexRelation, const Datum *values, const bool *isnull)

入力と出力

  • **入力**:
  • indexRelation: 対象インデックスのリレーション記述子。
  • values: インデックスキー値の配列。
  • isnull: 各値が NULL かどうかを示すブール値の配列。
  • **出力**: 変換された文字列（palloc 済み）。権限不足や RLS（行レベルセキュリティ）により情報開示が制限される場合は NULL を返す。

前提条件と不変条件

  • 呼び出し元は、values および isnull 配列が indexRelation のキー属性数と一致していることを保証する必要がある。
  • インデックスのキー属性に対して、現在のユーザーが SELECT 権限を持っているか、テーブル全体への SELECT 権限を持っている必要がある。

アルゴリズムまたは振る舞い

1. **権限チェック**: RLS が有効な場合、またはテーブルおよび全キー列に対する SELECT 権限がない場合は NULL を返す。式インデックス（attnum == InvalidAttrNumber）が含まれる場合も安全のため NULL を返す。 2. **文字列構築**: StringInfo を初期化し、インデックスの列名リストを付与する。 3. **値の変換**: 各キー値について、インデックスの演算クラス（rd_opcintype）に基づき、適切な出力関数（getTypeOutputInfo）を呼び出して文字列化する。 4. **結合**: 変換された値をカンマ区切りで連結し、最終的な文字列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtinsert.c
  • src/backend/executor/execIndexing.c
  • **呼び出し先**:
  • IndexRelationGetNumberOfKeyAttributes
  • check_enable_rls, pg_class_aclcheck, pg_attribute_aclcheck
  • initStringInfo, appendStringInfo, appendStringInfoString, appendStringInfoChar
  • getTypeOutputInfo, OidOutputFunctionCall

関連データ構造

  • Relation: インデックスのメタデータ保持に使用。
  • StringInfoData: 文字列構築用バッファ。
  • Form_pg_index: インデックスの属性情報（indkey 等）を参照。

失敗モードと境界条件

  • **権限不足**: ユーザーがデータにアクセスできない場合、情報漏洩を防ぐため NULL を返す。
  • **式インデックス**: 複雑な式を含むインデックスの場合、列単位の権限判定が困難なため NULL を返す。
  • **疑似型**: ANYARRAY や RECORD などの疑似型が含まれる場合、現在の実装では標準的な出力関数に依存しており、将来的に拡張が必要になる可能性がある。

保守担当者向けメモ

  • 本関数は MVCC やトランザクションの状態に関わらず、提供された values を単純に文字列化する。
  • 権限チェックのロジックは execMain.c 等の類似関数と整合性を保つ必要がある。
  • rd_opcintype を使用している点は、インデックスの物理的な格納型ではなく、演算クラスの入力型に基づいた変換を行うための重要な設計である。
