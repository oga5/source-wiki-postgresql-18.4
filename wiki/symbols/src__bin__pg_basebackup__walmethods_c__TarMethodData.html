TarMethodData

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_basebackup/walmethods.c:699

目的

pg_basebackup 等において、ストリーミング受信した WAL データを単一の tar ファイルとして永続化するための状態管理構造体です。

シグネチャまたは形状

typedef struct TarMethodData
{
    WalWriteMethod base;          /* 基底インターフェース */
    char          *tarfilename;   /* tar ファイルのパス */
    int            fd;            /* ファイル記述子 */
    TarMethodFile *currentfile;   /* 現在書き込み中のファイル情報 */
#ifdef HAVE_LIBZ
    z_streamp      zp;            /* Gzip 圧縮用ストリーム */
    void          *zlibOut;       /* 圧縮用出力バッファ */
#endif
} TarMethodData;

入力と出力

  • 入力: CreateWalTarMethod により初期化され、WalWriteMethod インターフェースを通じてデータが供給されます。
  • 出力: 指定されたパスの tar ファイルへ、WAL データが順次書き込まれます。

前提条件と不変条件

  • tar メソッドの制約として、同時にオープンできるファイルは 1 つのみです。
  • HAVE_LIBZ が定義されている場合、Gzip 圧縮が有効となります。

アルゴリズムまたは振る舞い

  • WalWriteMethod の具象実装として、ファイルオープン、書き込み、同期、クローズの各操作を管理します。
  • 圧縮が有効な場合、z_streamp を介してデータを圧縮してから tar ファイルへ書き込みます。
  • tar 形式特有のヘッダー生成やパディング処理を内部的に保持します。

呼び出し元と呼び出し先

  • 呼び出し元: CreateWalTarMethod（インスタンス生成）、および walmethods.c 内の各メソッド関数（tar_write_compressed_data 等）。
  • 呼び出し先: zlib ライブラリ（圧縮時）、および標準的なファイルシステム操作関数。

関連データ構造

  • WalWriteMethod: 基底構造体。
  • TarMethodFile: 現在書き込み中のファイル状態を保持する構造体。

失敗モードと境界条件

  • ファイルオープン失敗や書き込みエラーが発生した場合、WalWriteMethod のエラーハンドリング機構（lasterrstring 等）を通じて通知されます。
  • 圧縮ライブラリの初期化や処理に失敗した場合、エラーが返されます。

保守担当者向けメモ

  • 本構造体は pg_basebackup の WAL 受信処理に特化しています。
  • tar メソッドは Gzip のみをサポートしており、ディレクトリメソッドがサポートする LZ4 は利用できません。
  • currentfile は一度に 1 つのファイルしか保持しないため、複数の WAL ファイルを同時に扱うような拡張を行う場合は注意が必要です。
