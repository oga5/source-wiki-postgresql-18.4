ER_mc_callback

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/expandedrecord.c:901

目的

展開されたレコード（ExpandedRecordHeader）が保持するタプル記述子（TupleDesc）の参照カウントを適切に減らし、必要に応じてメモリを解放するためのコールバック関数です。

シグネチャまたは形状

static void ER_mc_callback(void *arg)

入力と出力

  • **入力**: arg - ExpandedRecordHeader 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • arg は ExpandedRecordHeader 型としてキャスト可能であること。
  • erh->er_tupdesc が指す TupleDesc は、本関数によって参照カウントが管理されていること。

アルゴリズムまたは振る舞い

1. 引数 arg を ExpandedRecordHeader 型にキャストします。 2. er_tupdesc が存在する場合、その参照カウント（tdrefcount）を確認します。 3. 参照カウントが 0 より大きい場合、デクリメントします。 4. デクリメントの結果、参照カウントが 0 に達した場合は FreeTupleDesc を呼び出してタプル記述子を解放します。 5. 処理後、erh->er_tupdesc を NULL に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/expandedrecord.c 内の各生成関数（make_expanded_record_from_typeid, make_expanded_record_from_tupdesc, make_expanded_record_from_exprecord, make_expanded_record_from_datum）において、erh->er_mcb.func に本関数が登録されます。
  • **呼び出し先**:
  • FreeTupleDesc（src/include/utils/expandedrecord.h 等で定義されるタプル記述子解放関数）。

関連データ構造

  • ExpandedRecordHeader: レコードのメタデータおよびタプル記述子を保持する構造体。
  • TupleDesc: タプルの構造を定義する記述子。

失敗モードと境界条件

  • er_tupdesc が NULL の場合、何も行わずに終了します。
  • 参照カウントが 0 以下の場合は解放処理を行いません。

保守担当者向けメモ

  • 本関数はメモリコンテキストのコールバックとして登録されることを前提としています。
  • er_tupdesc = NULL への代入は、二重解放や不正な参照を防ぐための安全策として機能します。
