PLySubtransactionData

定義

  • 種別: struct
  • 定義箇所: src/pl/plpython/plpy_subxactobject.h:24

目的

PostgreSQL のサブトランザクション開始前のメモリコンテキストおよびリソースオーナーの状態を保持し、サブトランザクション終了時にそれらを復元するためのコンテキスト情報を格納する。

シグネチャまたは形状

typedef struct PLySubtransactionData
{
	MemoryContext oldcontext;
	ResourceOwner oldowner;
} PLySubtransactionData;

入力と出力

  • **入力**: サブトランザクション開始時に、現在の MemoryContext および ResourceOwner が本構造体に保存される。
  • **出力**: サブトランザクション終了時に、保存された oldcontext および oldowner が復元に使用される。

前提条件と不変条件

  • explicit_subtransactions リストを通じて管理される。
  • サブトランザクションのネスト構造を維持するため、スタックの先頭（linitial）から取得されることが想定されている。

アルゴリズムまたは振る舞い

1. サブトランザクション開始時、現在のコンテキストとリソースオーナーを本構造体に記録する。 2. サブトランザクションの実行中、このデータは explicit_subtransactions リストに保持される。 3. サブトランザクション終了時、本構造体に保存された値を用いて、PostgreSQL の実行環境を元の状態へ復元する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_exec.c
  • src/pl/plpython/plpy_subxactobject.c
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接関数を呼び出すものではない。

関連データ構造

  • explicit_subtransactions: アクティブなサブトランザクションを追跡するグローバルなリスト。
  • PLySubtransactionObject: Python 側から見えるサブトランザクションオブジェクト。

失敗モードと境界条件

  • リストが空の状態で linitial(explicit_subtransactions) を呼び出した場合の挙動については、本ソースからは明示されていない。

保守担当者向けメモ

  • explicit_subtransactions リストを操作する際は、メモリコンテキストの切り替えによる整合性破壊に注意すること。
  • 本構造体は plpy_subxactobject.c 内でメモリ確保が行われるため、ライフサイクル管理は当該実装に依存する。
