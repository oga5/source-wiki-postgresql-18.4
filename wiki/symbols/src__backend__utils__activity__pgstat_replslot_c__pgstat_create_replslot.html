pgstat_create_replslot

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_replslot.c:110

目的

レプリケーションスロットの作成時に、対応する統計情報を初期化（ゼロクリア）するために使用されます。

シグネチャまたは形状

void pgstat_create_replslot(ReplicationSlot *slot)

入力と出力

  • **入力**: ReplicationSlot *slot（統計情報を初期化する対象のレプリケーションスロット構造体）
  • **出力**: なし（副作用として共有メモリ上の統計領域を更新）

前提条件と不変条件

  • 呼び出し元は ReplicationSlotAllocationLock を排他モード（LW_EXCLUSIVE）で保持している必要があります。

アルゴリズムまたは振る舞い

1. ReplicationSlotAllocationLock が排他ロックされていることを Assert で検証します。 2. pgstat_get_entry_ref_locked を呼び出し、指定されたスロットインデックスに対応する共有メモリ上の統計エントリを取得します。 3. 取得した統計エントリの stats 領域を memset によりゼロクリアします。これにより、過去のクラッシュ等で残存していた古い統計情報が破棄されます。 4. pgstat_unlock_entry を呼び出し、統計エントリのロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/slot.c:491
  • **呼び出し先**:
  • pgstat_get_entry_ref_locked
  • pgstat_unlock_entry
  • ReplicationSlotIndex (マクロ)

関連データ構造

  • ReplicationSlot: レプリケーションスロットの実体。
  • PgStatShared_ReplSlot: 共有メモリ上に配置されるレプリケーションスロットの統計構造体。
  • PgStat_EntryRef: 統計エントリへの参照を管理する構造体。

失敗モードと境界条件

  • 呼び出し元が ReplicationSlotAllocationLock を保持していない場合、Assert により異常終了します。
  • 過去のセッションで同名のスロットが削除されずに残存していた場合、本関数は古い統計情報を上書きして初期化します。

保守担当者向けメモ

  • 本関数は論理レプリケーションスロットの統計初期化に特化しています。
  • 統計情報の永続化は名前ベースで行われるため、インデックスの再利用による統計の混同を防ぐ目的で、作成時に必ずゼロクリアを行う設計となっています。
