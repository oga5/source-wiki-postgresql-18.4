WaitEventCustomShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/wait_event.c:102

目的

カスタム待機イベントの管理に必要な共有メモリ領域の合計サイズを計算する。

シグネチャまたは形状

Size WaitEventCustomShmemSize(void)

入力と出力

  • 入力: なし
  • 出力: 必要な共有メモリサイズ（Size 型）

前提条件と不変条件

  • WAIT_EVENT_CUSTOM_HASH_MAX_SIZE（現在は128）に基づいた静的な見積もりを行う。
  • add_size 関数を用いて、オーバーフローを考慮した安全な加算を行う。

アルゴリズムまたは振る舞い

1. WaitEventCustomCounterData 構造体に必要なサイズを MAXALIGN でアライメントして初期値とする。 2. WaitEventCustomEntryByInfo を格納するハッシュテーブルのサイズを見積もり、合計に加算する。 3. WaitEventCustomEntryByName を格納するハッシュテーブルのサイズを見積もり、合計に加算する。 4. 計算された合計サイズを返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:149（PostgreSQLの共有メモリ初期化プロセスの一部として呼び出される）
  • 呼び出し先: add_size, hash_estimate_size

関連データ構造

  • WaitEventCustomCounterData: カスタムイベントのカウンタ用データ構造。
  • WaitEventCustomEntryByInfo: IDベースのハッシュテーブル用エントリ。
  • WaitEventCustomEntryByName: 名前ベースのハッシュテーブル用エントリ。

失敗モードと境界条件

  • WAIT_EVENT_CUSTOM_HASH_MAX_SIZE を超えるカスタムイベントの登録はサポートされない。
  • メモリ計算時に add_size を使用しているため、サイズ計算のオーバーフローは適切に処理される。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウトを決定する重要な箇所である。ハッシュテーブルの構造体定義や WAIT_EVENT_CUSTOM_HASH_MAX_SIZE を変更した場合は、必ず本関数の計算ロジックを更新すること。
