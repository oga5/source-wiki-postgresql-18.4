PageIndexMultiDelete

定義

  • 種別: function
  • 定義箇所: src/backend/storage/page/bufpage.c:1159

目的

ページ内の複数のアイテム（タプル）を一度に削除し、ページ内の断片化を解消して領域を再利用可能にする。

シグネチャまたは形状

void PageIndexMultiDelete(Page page, OffsetNumber *itemnos, int nitems)

入力と出力

  • **入力**:
  • page: 操作対象のページへのポインタ。
  • itemnos: 削除するアイテム番号の配列。
  • nitems: 削除するアイテムの数。
  • **出力**: なし（副作用としてページ内容を直接更新する）。

前提条件と不変条件

  • nitems は MaxIndexTuplesPerPage 以下であること。
  • ページヘッダーの pd_lower、pd_upper、pd_special が整合していること（破損チェックあり）。
  • itemnos 配列内の各アイテム番号は、ページ内に存在し、かつ昇順で提供される必要がある（nextitm != nitems の場合にエラー）。

アルゴリズムまたは振る舞い

1. **最適化**: 削除対象が2つ以下の場合、PageIndexTupleDelete を逆順で呼び出して終了する。 2. **検証**: ページヘッダーのポインタ値が妥当か確認する。 3. **走査**: ページ内の全アイテムを走査し、削除対象外のアイテムのみを一時リスト（newitemids および itemidbase）に退避する。この際、アイテムのオフセットが妥当か検証する。 4. **更新**: ページヘッダーの pd_linp を更新し、pd_lower を再計算する。 5. **デフラグ**: compactify_tuples を呼び出し、ページ内のタプルデータを物理的に詰め直す。アイテムが残らない場合は pd_upper を pd_special にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gist.c
  • src/backend/access/gist/gistvacuum.c
  • src/backend/access/gist/gistxlog.c
  • src/backend/access/hash/hash.c
  • src/backend/access/hash/hash_xlog.c
  • src/backend/access/hash/hashinsert.c
  • src/backend/access/hash/hashovfl.c
  • src/backend/access/nbtree/nbtpage.c
  • **呼び出し先**:
  • PageIndexTupleDelete (小規模削除時)
  • compactify_tuples (タプル再配置時)
  • ereport/elog (エラー発生時)

関連データ構造

  • PageHeaderData: ページヘッダー構造体。
  • ItemIdData: ページ内のアイテムポインタ。
  • itemIdCompactData: 断片化解消時に使用される一時的なアイテム配置情報。

失敗モードと境界条件

  • **データ破損**: ページヘッダーのポインタ（pd_lower 等）やアイテムのオフセットが範囲外の場合、ERRCODE_DATA_CORRUPTED でエラーを報告する。
  • **入力不整合**: itemnos に無効なオフセットが含まれる場合、または totallen が利用可能領域を超える場合にエラーとなる。

保守担当者向けメモ

  • presorted フラグは現在常に true とされているが、将来的な最適化の余地がある。
  • 削除対象のアイテム番号は昇順で渡されることを前提としている。
