partition_bounds_create

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partbounds.c:298

目的

PartitionBoundSpec の配列を、実行時に効率的に利用可能な内部表現である PartitionBoundInfo 構造体へ変換するエントリポイントです。

シグネチャまたは形状

PartitionBoundInfo partition_bounds_create(PartitionBoundSpec **boundspecs, int nparts, PartitionKey key, int **mapping)

入力と出力

  • **入力**:
  • boundspecs: パーティション境界定義の配列。
  • nparts: パーティションの総数。
  • key: パーティションキーの定義。
  • mapping: 呼び出し元で確保された int* へのポインタ。
  • **出力**:
  • 戻り値: 生成された PartitionBoundInfo 構造体。
  • mapping: 元のリストにおける各パーティションのインデックスから、正規化されたインデックスへの変換テーブルが格納されます。

前提条件と不変条件

  • nparts > 0 であること。
  • key->strategy が PARTITION_STRATEGY_HASH, LIST, RANGE のいずれかであること。

アルゴリズムまたは振る舞い

1. mapping 配列を nparts 分確保し、初期値 -1 で埋めます。 2. key->strategy に基づき、以下のいずれかのサブルーチンを呼び出します。

  • create_hash_bounds
  • create_list_bounds
  • create_range_bounds

    • 3. 各サブルーチン内で、境界値のソート、重複排除、および PartitionBoundInfo の datums 配列と indexes 配列の構築が行われます。 4. 最終的に構築された PartitionBoundInfo を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/partitioning/partdesc.c:311
  • **呼び出し先**:
  • create_hash_bounds
  • create_list_bounds
  • create_range_bounds

関連データ構造

  • PartitionBoundInfo: 境界情報の主要なコンテナ。
  • PartitionBoundSpec: パーサーノード表現の境界定義。
  • PartitionKey: パーティション戦略やキー定義を保持する構造体。

失敗モードと境界条件

  • nparts <= 0 の場合、Assert により停止します。
  • 未定義の戦略が渡された場合、Assert(false) により停止します。

保守担当者向けメモ

  • 本関数は単なるディスパッチャーであり、実際の境界構築ロジックは各戦略ごとの create_*_bounds 関数に委譲されています。
  • mapping 配列のメモリ管理は呼び出し元に依存するため、メモリリークや不正なポインタ参照に注意してください。
