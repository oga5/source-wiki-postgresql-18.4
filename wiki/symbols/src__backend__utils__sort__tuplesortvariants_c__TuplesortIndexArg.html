TuplesortIndexArg

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/sort/tuplesortvariants.c:123

目的

インデックス構築時のソート処理において、対象となるテーブル（ヒープ）と構築中のインデックスリレーションを保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	Relation	heapRel;		/* table the index is being built on */
	Relation	indexRel;		/* index being built */
} TuplesortIndexArg;

入力と出力

本構造体はデータ構造であり、関数ではありません。heapRel および indexRel は Relation 型のポインタとして保持されます。

前提条件と不変条件

  • heapRel および indexRel は、ソート処理の実行中、有効なリレーション記述子を指している必要があります。

アルゴリズムまたは振る舞い

本構造体自体はロジックを持ちませんが、tuplesortvariants.c 内の各ソート関数において、Tuplesortstate の arg フィールドにキャストして格納され、型固有の処理（インデックス構築時の比較や変換など）を行う際のコンテキストとして参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/tuplesortvariants.c 内のインデックスソート関連関数（tuplesort_begin_index_btree 等）が本構造体を初期化します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。
  • **参照箇所**: src/backend/utils/sort/tuplesortvariants.c の行 841, 1562, 1827 等で、base->arg からキャストして参照されています。

関連データ構造

  • Relation: PostgreSQLのテーブルやインデックスのメタデータを保持する構造体。
  • Tuplesortstate: ソート処理の全状態を管理する構造体。本構造体はこの内部で arg として保持されます。

失敗モードと境界条件

  • heapRel または indexRel が NULL の状態でソート処理が開始された場合、後続の処理で不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体はインデックス構築専用の引数コンテナです。新しいインデックス型を追加する際、追加のメタデータが必要な場合は本構造体の拡張を検討してください。
