pgaio_io_reopen

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_target.c:115

目的

プロセス間でのI/O引き継ぎ時において、I/Oハンドルに関連付けられたターゲット固有のファイル記述子（FD）を再構築（再オープン）するために使用されます。

シグネチャまたは形状

void pgaio_io_reopen(PgAioHandle *ioh)

入力と出力

  • **入力**: PgAioHandle *ioh - 再オープン対象のI/Oハンドルへのポインタ。
  • **出力**: なし（副作用としてターゲットの再オープン処理を実行）。

前提条件と不変条件

  • ioh->target が有効な範囲（PGAIO_TID_INVALID より大きく PGAIO_TID_COUNT より小さい）にあること。
  • ioh->op が有効な範囲（PGAIO_OP_INVALID より大きく PGAIO_OP_COUNT より小さい）にあること。
  • 上記条件は Assert により実行時に検証されます。

アルゴリズムまたは振る舞い

1. ioh のターゲットおよび操作タイプが有効であることを検証します。 2. pgaio_target_info 配列から、現在の ioh->target に対応するターゲット情報を取得します。 3. 取得したターゲット情報の reopen メソッドを呼び出し、対象リソースの再オープン処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/method_worker.c:530
  • src/backend/storage/aio/aio_target.c:100（pgaio_io_can_reopen 内での参照）
  • **呼び出し先**:
  • pgaio_target_info に登録された各ターゲット固有の reopen 関数。

関連データ構造

  • PgAioHandle: I/O操作の状態を保持する主要構造体。
  • PgAioTargetInfo: ターゲットごとのメソッド（reopen 等）を定義する構造体。

失敗モードと境界条件

  • ioh->target が無効な場合、Assert によりプロセスが停止します。
  • ターゲット固有の reopen 実装内でエラーが発生した場合、その実装に従った例外処理が行われます。

保守担当者向けメモ

  • 本関数は、プロセス境界を跨いでI/Oを継続する際の重要な仲介層です。
  • 新しいI/Oターゲットを追加する場合は、PgAioTargetInfo 構造体を通じて適切な reopen 実装を提供する必要があります。
  • method_worker.c から呼び出す際は、割り込みを防止するなどの排他制御が必要となる場合があります。
