avw_dbase

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/autovacuum.c:180

目的

自動バキューム処理の対象となるデータベースの情報を保持し、バキュームのスケジュール管理や実行判断に必要なメタデータを集約する。

シグネチャまたは形状

typedef struct avw_dbase
{
    Oid                adw_datid;       /* データベースのOID */
    char              *adw_name;        /* データベース名 */
    TransactionId      adw_frozenxid;   /* 凍結済みトランザクションID */
    MultiXactId        adw_minmulti;    /* 最小マルチトランザクションID */
    PgStat_StatDBEntry *adw_entry;      /* 統計情報エントリへのポインタ */
} avw_dbase;

入力と出力

本構造体はメモリ上に動的に割り当てられ（palloc）、主に自動バキュームランチャーがデータベースリストを構築する際に使用される。

前提条件と不変条件

  • adw_datid は有効なデータベースOIDであること。
  • adw_entry は pgstat サブシステムによって管理される統計情報と整合していること。

アルゴリズムまたは振る舞い

自動バキュームランチャーは、この構造体を使用して各データベースのトランザクションID周回防止状況や統計情報を評価し、バキュームが必要なデータベースを特定する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/autovacuum.c 内のデータベースリスト走査処理（lfirst を用いたリスト操作）および palloc によるメモリ確保箇所。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行わない。

関連データ構造

  • PgStat_StatDBEntry: 統計情報管理に使用。
  • Oid, TransactionId, MultiXactId: PostgreSQLのシステム識別子およびトランザクション管理型。

失敗モードと境界条件

  • メモリ不足時: palloc 失敗によるエラー発生の可能性がある。
  • データベース削除時: adw_datid が無効化された場合、統計情報との不整合が生じる可能性がある。

保守担当者向けメモ

  • 本構造体は autovacuum.c 内でリスト管理される。リストの走査や要素の解放を行う際は、メモリリークや無効なポインタ参照に注意すること。
  • 構造体のフィールド追加時は、統計情報やトランザクション管理の更新ロジックへの影響を考慮すること。
