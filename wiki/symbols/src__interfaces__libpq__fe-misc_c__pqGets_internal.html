pqGets_internal

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-misc.c:108

目的

PGconn の入力バッファからヌル終端文字列を検索し、指定された PQExpBuffer へコピーする内部ユーティリティ関数です。

シグネチャまたは形状

static int pqGets_internal(PQExpBuffer buf, PGconn *conn, bool resetbuffer);

入力と出力

  • **入力**:
  • buf: 抽出した文字列を格納する PQExpBuffer 構造体へのポインタ。
  • conn: 通信状態を保持する PGconn 構造体へのポインタ。
  • resetbuffer: true の場合、コピー前に buf をリセットします。
  • **出力**:
  • 成功時: 0 を返し、conn->inCursor を文字列の終端（ヌル文字の次）まで進めます。
  • 失敗時: EOF を返します（バッファ内にヌル文字が見つからない場合）。

前提条件と不変条件

  • conn->inBuffer に有効なデータが格納されていること。
  • conn->inCursor は現在の読み出し開始位置を指していること。

アルゴリズムまたは振る舞い

1. conn->inBuffer 内の inCursor から inEnd までを走査し、ヌル文字（\0）を探します。 2. ヌル文字が見つからない場合、EOF を返して終了します。 3. ヌル文字が見つかった場合、resetbuffer が true なら buf をリセットします。 4. conn->inCursor からヌル文字までのバイト列を appendBinaryPQExpBuffer を用いて buf に追加します。 5. conn->inCursor をヌル文字の次の位置まで進めます。 6. 0 を返して正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq/fe-misc.c 内の関数（直接の呼び出しを確認）。
  • **呼び出し先**:
  • resetPQExpBuffer
  • appendBinaryPQExpBuffer

関連データ構造

  • PGconn: libpq の接続状態を管理する構造体。inBuffer（受信バッファ）、inCursor（読み出し位置）、inEnd（データ終端）を使用します。
  • PQExpBuffer: 動的に拡張可能な文字列バッファ。

失敗モードと境界条件

  • **EOF**: inCursor が inEnd に達してもヌル文字が見つからない場合、文字列が不完全であると判断し EOF を返します。

保守担当者向けメモ

  • 本関数は conn->inCursor を直接操作するため、呼び出し側でバッファの状態が整合していることを確認してください。
  • resetbuffer 引数は、呼び出し元がバッファを累積的に使用するか、単発で使用するかを制御するために重要です。
