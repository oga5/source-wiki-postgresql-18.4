FilterByOrigin

定義

  • 種別: static inline function
  • 定義箇所: src/backend/replication/logical/decode.c:573

目的

論理デコーディングにおいて、指定されたレプリケーションオリジン（origin_id）に基づくレコードのフィルタリングが必要かどうかを判定し、必要に応じてコールバックを実行する。

シグネチャまたは形状

static inline bool FilterByOrigin(LogicalDecodingContext *ctx, RepOriginId origin_id)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト（LogicalDecodingContext *）。
  • origin_id: 判定対象のレプリケーションオリジン ID（RepOriginId）。
  • **出力**:
  • bool: フィルタリングによりレコードをスキップすべき場合は true、それ以外は false を返す。

前提条件と不変条件

  • ctx が有効なポインタであること。
  • ctx->callbacks.filter_by_origin_cb が設定されている場合、そのラッパー関数 filter_by_origin_cb_wrapper が呼び出される。

アルゴリズムまたは振る舞い

1. ctx->callbacks.filter_by_origin_cb が NULL であるかを確認する。 2. コールバックが定義されていない場合は、フィルタリングを行わず false を返す。 3. コールバックが定義されている場合は、filter_by_origin_cb_wrapper を介してその結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/decode.c 内の各デコード関数（DecodeInsert, DecodeUpdate, DecodeDelete, DecodeTruncate, DecodeMultiInsert, DecodeSpecConfirm など）から、WAL レコードのオリジンを確認するために呼び出される。
  • **呼び出し先**: filter_by_origin_cb_wrapper（ctx->callbacks.filter_by_origin_cb を実行する内部関数）。

関連データ構造

  • LogicalDecodingContext: 論理デコーディングの実行状態とコールバック関数を保持する構造体。
  • RepOriginId: レプリケーションオリジンを識別する型。

失敗モードと境界条件

  • ctx->callbacks.filter_by_origin_cb が NULL の場合、本関数は常に false を返し、フィルタリングは行われない。

保守担当者向けメモ

  • 本関数は static inline で定義されており、頻繁に呼び出されるためオーバーヘッドを最小限に抑える設計となっている。
  • フィルタリングのロジック自体は ctx 内のコールバックに依存しているため、動作の詳細はプラグイン側の実装を確認すること。
