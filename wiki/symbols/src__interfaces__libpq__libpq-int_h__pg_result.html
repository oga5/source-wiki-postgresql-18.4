pg_result

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/libpq-int.h:164

目的

PostgreSQL クライアントライブラリ（libpq）において、サーバーから返されたクエリ実行結果、メタデータ、およびエラー情報を保持するための内部構造体です。

シグネチャまたは形状

struct pg_result {
    int ntups;
    int numAttributes;
    PGresAttDesc *attDescs;
    PGresAttValue **tuples;
    int tupArrSize;
    int numParameters;
    PGresParamDesc *paramDescs;
    ExecStatusType resultStatus;
    char cmdStatus[CMDSTATUS_LEN];
    int binary;
    PGNoticeHooks noticeHooks;
    PGEvent *events;
    int nEvents;
    int client_encoding;
    char *errMsg;
    PGMessageField *errFields;
    char *errQuery;
    char null_field[1];
    PGresult_data *curBlock;
    int curOffset;
    int spaceLeft;
    size_t memorySize;
};

入力と出力

  • **入力**: サーバーからの応答データ（プロトコル解析を経て格納）。
  • **出力**: libpq-fe.h で定義される PGresult 型として公開され、アプリケーションがクエリ結果を取得するために使用されます。

前提条件と不変条件

  • tuples は tupArrSize に基づいて動的に再割り当て可能なメモリブロックを指します。
  • attDescs およびエラー関連フィールドは、curBlock が管理するメモリブロック内に配置されます。
  • null_field は、クエリ結果内のすべての NULL 属性が参照する固定の終端文字列として機能します。

アルゴリズムまたは振る舞い

  • **メモリ管理**: curBlock、curOffset、spaceLeft を使用して、結果データ用のメモリを効率的に割り当てます。
  • **独立性**: 生成元の PGconn から必要な情報（noticeHooks 等）をコピーすることで、PGresult 単体での操作を可能にしています。
  • **エラー処理**: エラー発生時は errMsg にメッセージが格納され、詳細なフィールド情報は errFields にリスト形式で保持されます。

呼び出し元と呼び出し先

  • **呼び出し元**: fe-exec.c 等の内部関数が結果の生成と管理を行います。
  • **呼び出し先**: 提供されたソース情報からは、特定の関数呼び出し関係は特定できません。

関連データ構造

  • PGresult (typedef: struct pg_result)
  • PGresAttDesc, PGresAttValue, PGresParamDesc
  • PGMessageField
  • PGresult_data

失敗モードと境界条件

  • errMsg が NULL の場合、エラーが発生していないことを示します。
  • tuples のメモリ割り当て失敗や、curBlock のメモリ不足が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は非公開 API です。libpq-fe.h を介さない直接的なアクセスは避けてください。
  • PGresult のメモリ管理は pqResultAlloc などの内部関数に依存しているため、構造体を直接操作する際はメモリリークや不正なポインタ参照に注意してください。
