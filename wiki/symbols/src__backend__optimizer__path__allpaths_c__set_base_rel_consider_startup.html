set_base_rel_consider_startup

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/allpaths.c:246

目的

SEMI または ANTI 結合の右辺（RHS）となるベースリレーションに対し、パラメータ化されたパスの「高速起動（fast-start）」を考慮すべきかどうかを判定し、フラグを設定する。

シグネチャまたは形状

static void set_base_rel_consider_startup(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root（現在のクエリのプランナ情報）
  • **出力**: なし（副作用として、root 内の対象リレーションの consider_param_startup フラグを更新する）

前提条件と不変条件

  • パラメータ化されたパスは通常、ネストループ結合の内側でのみ使用されるため、高速起動プランの価値は限定的である。
  • 本関数は、計画作成時間の増大を避けるため、RHS が単一のベースリレーションである場合にのみ適用される。
  • 結合リレーション（joinrels）や Append リレーションには適用されない。

アルゴリズムまたは振る舞い

1. root->join_info_list を走査し、各 SpecialJoinInfo を取得する。 2. 結合タイプが JOIN_SEMI または JOIN_ANTI であるかを確認する。 3. bms_get_singleton_member を使用し、RHS が単一のベースリレーション（varno）で構成されているか判定する。 4. 条件を満たす場合、find_base_rel で該当リレーションを取得し、rel->consider_param_startup を true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/allpaths.c 内のメイン処理フロー（allpaths.c:178）から呼び出される。
  • **呼び出し先**:
  • find_base_rel（root からリレーション情報を取得）
  • bms_get_singleton_member（ビットマップセットから単一メンバーを抽出）

関連データ構造

  • PlannerInfo: クエリ全体のプランナ情報を保持。
  • SpecialJoinInfo: SEMI/ANTI 結合などの特殊な結合情報を保持。
  • RelOptInfo: リレーションの最適化情報を保持し、consider_param_startup フラグを格納する。

失敗モードと境界条件

  • RHS が単一のベースリレーションではない場合（結合リレーションなど）、本関数はフラグを設定しない。
  • Append リレーションは本関数の対象外である。

保守担当者向けメモ

  • 本関数は、SEMI/ANTI 結合において「最初の1タプルさえ取得できればよい」という特性を活かすための最適化である。
  • 計画作成コストを抑制するため、対象を単一ベースリレーションに限定している点に注意すること。
