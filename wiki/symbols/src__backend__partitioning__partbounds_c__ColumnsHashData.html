ColumnsHashData

定義

  • 種別: struct
  • 定義箇所: src/backend/partitioning/partbounds.c:4773

目的

ハッシュパーティションにおけるカラムのハッシュ計算に必要なメタデータおよび関数情報を保持する構造体です。実行時の関数呼び出しにおいて、ハッシュ計算に必要な情報をキャッシュするために使用されます。

シグネチャまたは形状

typedef struct ColumnsHashData
{
    Oid         relid;
    int         nkeys;
    Oid         variadic_type;
    int16       variadic_typlen;
    bool        variadic_typbyval;
    char        variadic_typalign;
    Oid         partcollid[PARTITION_MAX_KEYS];
    FmgrInfo    partsupfunc[FLEXIBLE_ARRAY_MEMBER];
} ColumnsHashData;

入力と出力

本構造体はデータ構造であり、直接的な入力や出力は持ちません。ただし、fcinfo->flinfo->fn_extra を通じて関数呼び出しのコンテキストとして利用されます。

前提条件と不変条件

  • partsupfunc は FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体の割り当て時には offsetof(ColumnsHashData, partsupfunc) を用いた動的なメモリ確保が必要です。
  • partcollid 配列は PARTITION_MAX_KEYS のサイズを持ち、パーティションキーの照合順序を保持します。

アルゴリズムまたは振る舞い

ハッシュパーティションの計算時に、fn_extra から本構造体をキャストして取得し、格納されている partsupfunc（ハッシュ関数情報）を使用して各キーのハッシュ値を計算します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/partitioning/partbounds.c 内の関数（fn_extra を介してアクセス）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありませんが、保持している partsupfunc がハッシュ計算関数を指します。

関連データ構造

  • FmgrInfo: 関数呼び出し情報を保持する標準的なPostgreSQL構造体。
  • PartitionHashBound: ハッシュパーティションの境界定義。

失敗モードと境界条件

  • メモリ割り当て不足: partsupfunc のサイズ計算を誤ると、バッファオーバーフローが発生する可能性があります。
  • fn_extra の型不一致: fcinfo->flinfo->fn_extra が本構造体として初期化されていない状態でキャストを行うと、未定義動作を引き起こします。

保守担当者向けメモ

  • partsupfunc が可変長配列であるため、構造体のサイズを計算する際は必ず offsetof マクロを使用してください。
  • 本構造体は実行時のキャッシュとして機能するため、パーティション定義が変更された場合は、関連するキャッシュの無効化と再構築が適切に行われているか確認してください。
