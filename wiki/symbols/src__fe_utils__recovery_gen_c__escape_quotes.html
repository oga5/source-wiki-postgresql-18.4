escape_quotes

定義

  • 種別: function
  • 定義箇所: src/fe_utils/recovery_gen.c:163

目的

入力文字列内のシングルクォートをエスケープし、設定ファイル等で安全に使用可能な文字列を生成します。

シグネチャまたは形状

static char *escape_quotes(const char *src);

入力と出力

  • **入力**: src (エスケープ対象の文字列へのポインタ)
  • **出力**: エスケープ処理済みの新しい文字列（呼び出し元で解放が必要）。メモリ不足時は pg_fatal によりプロセスが終了します。

前提条件と不変条件

  • 入力文字列 src は NULL 終端されている必要があります。
  • 戻り値が NULL になることはありません（メモリ不足時は即座に終了するため）。

アルゴリズムまたは振る舞い

1. escape_single_quotes_ascii(src) を呼び出し、シングルクォートをエスケープした文字列を生成します。 2. 戻り値が NULL の場合、pg_fatal を呼び出してエラーメッセージを出力し、プログラムを終了します。 3. 成功した場合は、エスケープ済みの文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: escape_single_quotes_ascii（fe_utils/string_utils.h 由来）
  • **呼び出し元**:
  • src/fe_utils/recovery_gen.c 内で宣言されています。
  • ツールによるヒューリスティックな参照情報として、src/bin/initdb/initdb.c および src/bin/scripts/vacuumdb.c に同名の関数定義および呼び出しの痕跡がありますが、これらが本ファイル内の escape_quotes と同一の静的関数を直接呼び出しているかは、リンクの文脈上、明示的な証拠が不足しています。

関連データ構造

  • なし（標準的な C 文字列および pg_fatal によるエラーハンドリングを使用）。

失敗モードと境界条件

  • **メモリ不足**: escape_single_quotes_ascii がメモリ確保に失敗した場合、pg_fatal を通じて致命的エラーとして処理されます。

保守担当者向けメモ

  • 本関数は static で定義されており、src/fe_utils/recovery_gen.c 内でのみ使用されることを意図しています。
  • 他のソースファイル（initdb.c 等）にも同名の関数が存在しますが、これらは独立した実装である可能性が高いため、修正時は本ファイルのスコープに注意してください。
