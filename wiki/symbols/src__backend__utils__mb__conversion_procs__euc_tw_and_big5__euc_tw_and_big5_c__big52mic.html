big52mic

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_tw_and_big5/euc_tw_and_big5.c:448

目的

BIG5 エンコーディングの文字列を MULE_INTERNAL (MIC) エンコーディングへ変換する。

シグネチャまたは形状

static int big52mic(const unsigned char *big5, unsigned char *p, int len, bool noError)

入力と出力

  • **入力**:
  • big5: 変換元 BIG5 文字列へのポインタ。
  • p: 変換先バッファへのポインタ。
  • len: 入力文字列のバイト長。
  • noError: 変換エラー時に例外を投げず処理を中断するかを指定するフラグ。
  • **出力**:
  • 戻り値: 変換処理したバイト数。
  • p が指すバッファに変換後の MIC 文字列が書き込まれ、ヌル終端が付与される。

前提条件と不変条件

  • 入力文字列は BIG5 エンコーディングに従っている必要がある。
  • noError が false の場合、不正な文字や変換不能な文字に遭遇するとエラーを報告する。

アルゴリズムまたは振る舞い

1. 入力文字列を1文字ずつ走査する。 2. ASCII文字（高位ビットが立っていない場合）はそのままコピーする。 3. pg_encoding_verifymbchar を用いて文字境界を検証する。 4. BIG5toCNS 関数を呼び出し、BIG5コードをCNSコードとプレーン番号（lc）に変換する。 5. 変換された lc に基づき、MIC形式のバイト列を構築する。

  • プレーン3または4の場合は、LCPRV2_B を先行して書き込む。
  • プレーン番号とCNSコードポイントを順次書き込む。

    • 6. 処理したバイト数を累積し、最後にヌル終端を付与して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mb/conversion_procs/euc_tw_and_big5/euc_tw_and_big5.c:126（big5_to_mic 関数内）。
  • **呼び出し先**:
  • pg_encoding_verifymbchar: 文字の妥当性検証。
  • BIG5toCNS: BIG5からCNSへの変換。
  • report_invalid_encoding / report_untranslatable_char: エラー報告（noError が false の場合）。

関連データ構造

  • PG_BIG5, PG_MULE_INTERNAL: エンコーディング識別子。
  • LC_CNS11643_3, LC_CNS11643_4: MULE_INTERNALのプレーン識別子。
  • LCPRV2_B: 補助文字セットを示すプレフィックス。

失敗モードと境界条件

  • 入力文字列が不正なバイト列である場合、noError が false ならエラーを報告し、true ならその時点までの変換バイト数を返す。
  • 変換不能な文字に遭遇した場合、同様にエラー報告または処理中断を行う。
  • 入力長 len が 0 になるとループを終了する。

保守担当者向けメモ

  • 本関数は変換の橋渡し役であり、具体的な変換ロジックは BIG5toCNS に依存している。
  • noError フラグの扱いは COPY コマンド等の柔軟な処理をサポートするための仕様である。
