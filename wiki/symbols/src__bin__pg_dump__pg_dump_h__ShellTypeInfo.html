ShellTypeInfo

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_dump.h:232

目的

PostgreSQLのダンプ処理において、シェル型（プレースホルダーとして作成された型）の情報を保持し、関連するベース型とのリンクを管理する。

シグネチャまたは形状

typedef struct _shellTypeInfo
{
	DumpableObject dobj;
	TypeInfo   *baseType;
} ShellTypeInfo;

入力と出力

  • 入力: pg_dump 実行時にシステムカタログから読み取られたシェル型情報。
  • 出力: dumpShellType 関数を通じて、ダンプ出力先（Archive）へシェル型の定義を書き出すために使用される。

前提条件と不変条件

  • dobj フィールドにより、DumpableObject としての共通属性（IDや依存関係など）を保持する。
  • baseType は、このシェル型が関連付けられているベース型（TypeInfo）へのポインタである。

アルゴリズムまたは振る舞い

  • pg_dump の収集フェーズでインスタンス化され、メモリ上に保持される。
  • ダンプ順序決定時、DumpableObject の依存関係グラフの一部として扱われる。
  • dumpShellType 関数が呼び出されると、この構造体の情報を基にシェル型の定義がダンプストリームへ出力される。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/pg_dump.c 内のダンプ処理ロジック（dumpShellType 関数など）。
  • 呼び出し先: pg_malloc（メモリ確保時）、dumpShellType（ダンプ実行時）。

関連データ構造

  • DumpableObject: 基底構造体。
  • TypeInfo: 関連付けられたベース型の定義。
  • Archive: ダンプ出力先を管理する構造体。

失敗モードと境界条件

  • メモリ不足時: pg_malloc によるメモリ確保失敗が発生する可能性がある。
  • 依存関係の不整合: baseType が正しく解決されていない場合、ダンプ順序や出力内容に影響を及ぼす可能性がある。

保守担当者向けメモ

  • 本構造体は pg_dump の実行期間中生存する。
  • 新しい型定義を追加する際は、pg_dump_sort.c の依存関係テーブルとの整合性に注意すること。
