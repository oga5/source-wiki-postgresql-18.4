pg_lfind32

定義

  • 種別: function
  • 定義箇所: src/include/port/pg_lfind.h:152

目的

uint32 型の配列から指定された値（key）を高速に検索する。SIMD 命令を利用可能な環境ではベクトル演算を用いて並列比較を行い、効率を最大化する。

シグネチャまたは形状

static inline bool pg_lfind32(uint32 key, const uint32 *base, uint32 nelem)

入力と出力

  • **入力**:
  • key: 検索対象の uint32 値。
  • base: 検索対象となる uint32 配列の先頭ポインタ。
  • nelem: 配列内の要素数。
  • **出力**:
  • bool: key が配列内に存在する場合は true、存在しない場合は false を返す。

前提条件と不変条件

  • USE_NO_SIMD が定義されていない場合、SIMD 命令セット（Vector32）が利用可能であること。
  • USE_ASSERT_CHECKING が有効な場合、SIMD 処理の結果は逐次検索（pg_lfind32_one_by_one_helper）の結果と一致しなければならない。

アルゴリズムまたは振る舞い

1. **初期化**: key を vector32_broadcast でベクトルレジスタに展開する。 2. **フォールバック判定**: 要素数 nelem が SIMD 処理の最小単位（4 レジスタ分）に満たない場合、pg_lfind32_one_by_one_helper を呼び出して逐次検索を行う。 3. **SIMD ループ**: 4 つのベクトルレジスタ単位で配列を走査し、pg_lfind32_simd_helper を用いて比較を行う。 4. **境界処理**: ループ終了後、配列の末尾に対して再度 pg_lfind32_simd_helper を実行する。これにより、配列の末尾部分を重複してチェックするが、境界チェックの分岐を削減し性能を向上させている。 5. **SIMD 無効時**: USE_NO_SIMD が定義されている場合、常に pg_lfind32_one_by_one_helper を使用する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/procarray.c
  • src/backend/storage/lmgr/predicate.c
  • src/backend/utils/time/snapmgr.c
  • src/test/modules/test_lfind/test_lfind.c
  • **呼び出し先**:
  • pg_lfind32_simd_helper (SIMD 処理時)
  • pg_lfind32_one_by_one_helper (フォールバック時および境界チェック時)

関連データ構造

  • Vector32: port/simd.h で定義される SIMD ベクトル型。

失敗モードと境界条件

  • nelem が 0 の場合、検索は行われず false を返す。
  • 配列の末尾処理において、配列サイズが nelem_per_iteration 未満の場合、重複チェックが発生するが、これは意図的な最適化であり正当性に影響しない。

保守担当者向けメモ

  • USE_ASSERT_CHECKING が有効な環境では、SIMD 処理と逐次処理の結果を比較する Assert が含まれている。デバッグ時はこの検証コストに留意すること。
  • SIMD 処理の末尾チェックは、境界外アクセスを防ぐために配列の末尾を重複して走査する設計となっている。この挙動を変更する場合は、境界チェックのオーバーヘッドとのトレードオフを再評価する必要がある。
