CopyFormatOptions

定義

  • **種別**: struct
  • **定義箇所**: src/include/commands/copy.h:59

目的

SQLの COPY コマンドで指定されたオプション（フォーマット、区切り文字、エラー処理方針など）を、構文解析後に保持・伝達するための構造体です。

シグネチャまたは形状

typedef struct CopyFormatOptions {
    int         file_encoding;
    bool        binary;
    bool        freeze;
    bool        csv_mode;
    CopyHeaderChoice header_line;
    char       *null_print;
    int         null_print_len;
    char       *null_print_client;
    char       *default_print;
    int         default_print_len;
    char       *delim;
    char       *quote;
    char       *escape;
    List       *force_quote;
    bool        force_quote_all;
    bool       *force_quote_flags;
    List       *force_notnull;
    bool        force_notnull_all;
    bool       *force_notnull_flags;
    List       *force_null;
    bool        force_null_all;
    bool       *force_null_flags;
    bool        convert_selectively;
    CopyOnErrorChoice on_error;
    CopyLogVerbosityChoice log_verbosity;
    int64       reject_limit;
    List       *convert_select;
} CopyFormatOptions;

入力と出力

  • **入力**: ProcessCopyOptions 関数により、SQL文の解析結果（List *options）から本構造体へ値が変換・設定されます。
  • **出力**: BeginCopyFrom や BeginCopyTo などの初期化関数、および CopyFromGetRoutine / CopyToGetRoutine へ渡され、データ転送処理の挙動を制御します。

前提条件と不変条件

  • delim、quote、escape はそれぞれ1バイトの文字である必要があります。
  • null_print はサーバーエンコーディングで保持されます。
  • force_quote 等のリストは、カラム名指定がない場合は NIL となります。

アルゴリズムまたは振る舞い

本構造体は、COPY 実行時の設定値を保持するコンテナとして機能します。実行フェーズでは、これらのフラグや文字列に基づき、CSVのパース処理、バイナリ変換の有無、エラー発生時の挙動（on_error）が決定されます。

呼び出し元と呼び出し先

  • **呼び出し元**: ProcessCopyOptions（設定の生成）、src/backend/commands/copy.c、src/backend/commands/copyto.c 等。
  • **呼び出し先**: CopyFromGetRoutine、CopyToGetRoutine 等のルーチン取得関数。

関連データ構造

  • CopyHeaderChoice, CopyOnErrorChoice, CopyLogVerbosityChoice: 各オプションの列挙型。
  • List: カラム名リストを保持するために使用。

失敗モードと境界条件

  • file_encoding が指定されていない場合、値は -1 となります。
  • reject_limit はエラー許容回数を制御し、これを超えると処理が中断される可能性があります。

保守担当者向けメモ

  • 本構造体は非常に多くのフィールドを持つため、新しい COPY オプションを追加する際は、ProcessCopyOptions でのパース処理と、各実行ルーチンでの参照箇所を漏れなく更新してください。
  • force_quote_flags 等のポインタ配列は、カラム数に応じて動的に割り当てられるため、メモリ管理に注意してください。
