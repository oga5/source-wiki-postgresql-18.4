mark_file_as_archived

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/receivelog.c:52

目的

指定されたWALファイルまたは履歴ファイルに対して、アーカイブ完了を示す「.done」ファイルを生成し、アーカイブ処理が完了したことを記録する。

シグネチャまたは形状

static bool mark_file_as_archived(StreamCtl *stream, const char *fname);

入力と出力

  • **入力**:
  • stream: ストリーミング制御構造体（WalWriteMethod を介したファイル操作に使用）。
  • fname: アーカイブ完了をマークする対象のファイル名。
  • **出力**:
  • bool: 成功した場合は true、失敗した場合は false を返す。

前提条件と不変条件

  • stream->walmethod が初期化されており、ops->open_for_write および ops->close が有効であること。
  • fname は archive_status/ ディレクトリ内に .done サフィックスを付与して作成されるため、書き込み権限が必要である。

アルゴリズムまたは振る舞い

1. archive_status/<fname>.done というパス文字列を生成する。 2. stream->walmethod->ops->open_for_write を呼び出し、対象の .done ファイルを新規作成する。 3. ファイル作成に失敗した場合はエラーログを出力し、false を返す。 4. stream->walmethod->ops->close を呼び出し、ファイルを正常にクローズする。 5. クローズ処理に失敗した場合はエラーログを出力し、false を返す。 6. 全工程が成功した場合に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/receivelog.c 内の以下の箇所から呼び出されていることが確認されている。
  • WALファイル処理完了時（244行目付近）
  • タイムライン履歴ファイル処理完了時（325行目付近）
  • **呼び出し先**:
  • stream->walmethod->ops->open_for_write
  • stream->walmethod->ops->close
  • pg_log_error（エラー発生時）

関連データ構造

  • StreamCtl: ストリーミングの動作設定および書き込みメソッドを保持する構造体。
  • Walfile: WalWriteMethod によって抽象化されたファイルハンドル。

失敗モードと境界条件

  • **ファイル作成失敗**: ディレクトリの書き込み権限不足やディスクフルにより open_for_write が NULL を返す場合。
  • **クローズ失敗**: ファイルシステムエラー等により close が非ゼロを返す場合。
  • いずれの失敗時も GetLastWalMethodError を使用して詳細なエラーメッセージがログ出力される。

保守担当者向けメモ

  • 本関数は WalWriteMethod の抽象化レイヤーを利用しているため、直接的な fopen や close は行わない。ファイルシステム以外のストレージ（tarアーカイブ等）への書き込み時も、この抽象化を通じて処理される。
