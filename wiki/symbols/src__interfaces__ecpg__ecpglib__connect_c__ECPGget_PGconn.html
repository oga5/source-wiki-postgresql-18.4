ECPGget_PGconn

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/connect.c:724

目的

指定された接続名に対応する、基盤となる libpq の PGconn オブジェクトを取得します。

シグネチャまたは形状

PGconn *ECPGget_PGconn(const char *connection_name)

入力と出力

  • **入力**: connection_name (接続名を示す文字列、または NULL)
  • **出力**: 成功時は対応する PGconn* を返し、該当する接続が見つからない場合は NULL を返します。

前提条件と不変条件

  • ecpg_get_connection がスレッドセーフな接続検索を行うため、本関数もマルチスレッド環境下で安全に呼び出し可能です。
  • 戻り値の PGconn は、ECPGライブラリが管理する内部構造体 connection に保持されているポインタです。

アルゴリズムまたは振る舞い

1. ecpg_get_connection(connection_name) を呼び出し、指定された名前（またはカレント接続）に対応する内部の connection 構造体を取得します。 2. 取得した構造体が NULL であれば、そのまま NULL を返します。 3. 構造体が存在する場合、そのメンバである con->connection（PGconn 型）を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: ecpg_get_connection
  • **呼び出し元**: 提供されたソース excerpt 内には具体的な呼び出し元に関する情報は含まれていません。

関連データ構造

  • struct connection: ECPG内部でデータベース接続状態を管理する構造体。PGconn を保持します。

失敗モードと境界条件

  • 指定された connection_name に一致する接続が存在しない場合、またはカレント接続が設定されていない場合、ecpg_get_connection が NULL を返し、本関数も NULL を返します。

保守担当者向けメモ

  • 本関数は libpq の機能を直接利用したいアプリケーション向けに、ECPGの抽象化レイヤーから生の PGconn を露出させるためのインターフェースです。
  • 戻り値の PGconn に対して直接 PQfinish 等を呼び出すことは、ECPGの接続管理リストとの不整合を招くため避けてください。接続の終了には ECPGdisconnect を使用してください。
