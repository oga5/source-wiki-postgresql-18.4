citext_hash

定義

  • 種別: function
  • 定義箇所: contrib/citext/citext.c:143

目的

citext 型のデータに対して、大文字・小文字を区別しないハッシュ値を計算し、ハッシュインデックス等での利用を可能にする。

シグネチャまたは形状

Datum citext_hash(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される citext 型のデータ（text 型として取得）。
  • **出力**: Datum 型のハッシュ値。

前提条件と不変条件

  • 入力データは PG_GETARG_TEXT_PP を介して取得され、必要に応じて TOAST 展開される。
  • str_tolower により、入力文字列は DEFAULT_COLLATION_OID を用いて小文字に正規化される。
  • 処理終了時には、正規化のために確保したメモリ（str）および TOAST 展開された入力データ（txt）が適切に解放される必要がある。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP(0) を呼び出し、入力引数を text ポインタとして取得する。 2. str_tolower を使用し、入力文字列のデータ部分を小文字に変換した新しい文字列を生成する。 3. hash_any を呼び出し、小文字化された文字列のハッシュ値を計算する。 4. pfree を呼び出し、小文字化文字列のメモリを解放する。 5. PG_FREE_IF_COPY を呼び出し、TOAST された入力データのメモリを解放する。 6. PG_RETURN_DATUM を通じて計算されたハッシュ値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(citext_hash) マクロにより、PostgreSQL の関数マネージャから呼び出される。
  • **呼び出し先**:
  • PG_GETARG_TEXT_PP / PG_RETURN_DATUM / PG_FREE_IF_COPY (PostgreSQL FMGR マクロ)
  • str_tolower (文字列正規化)
  • hash_any (ハッシュ計算)
  • pfree (メモリ解放)

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • Datum: PostgreSQL の汎用データ型。

失敗モードと境界条件

  • 入力データが NULL の場合、PG_GETARG_TEXT_PP の挙動に従う。
  • TOAST されたデータが入力された場合、PG_FREE_IF_COPY を使用しないとメモリリークが発生する可能性がある。

保守担当者向けメモ

  • 本関数は DEFAULT_COLLATION_OID を固定で使用している。これは大文字・小文字の変換ルールを照合順序に依存させないための設計である。
  • citext_hash_extended と同様のロジックを共有している可能性があるため、変更時は整合性に注意すること。
