r_undouble_kpt

目的

エストニア語のステミング処理において、語幹末尾における子音（k, p, t）の重複を解消し、正規化を行う。

シグネチャまたは形状

static int r_undouble_kpt(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（Snowball ランタイム環境構造体）
  • 出力: int（成功時に 1、失敗または条件不一致時に 0、エラー発生時に負の値を返す）

前提条件と不変条件

  • z->I[0] が現在の処理位置の境界として設定されていること。
  • g_V1 グルーピングに含まれる文字が直前に存在しないこと。
  • 処理対象の文字が a_9 配列で定義された特定の文字パターンに一致すること。

アルゴリズムまたは振る舞い

1. in_grouping_b_U を使用し、g_V1 に該当する文字が直前にないことを確認する。 2. 現在のカーソル位置 z->c が z->I[0] 以降であることを確認する。 3. z->ket を現在の位置に設定し、末尾の文字が特定のビットマスク条件を満たすか判定する。 4. find_among_b を用いて a_9 配列から接尾辞を検索する。 5. 一致した場合、z->bra を設定し、among_var の値（1, 2, 3）に応じて slice_from_s を実行し、該当する文字列を置換する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_estonian.c 内のステミングパイプライン（1404行目付近）。
  • 呼び出し先: in_grouping_b_U, find_among_b, slice_from_s（いずれも Snowball ランタイム関数）。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_9: 重複解消ルールを定義した among 構造体の配列。
  • g_V1: 母音グループを定義した文字グループ。

失敗モードと境界条件

  • g_V1 に該当する文字が直前にある場合、処理を中断し 0 を返す。
  • z->I[0] よりカーソル位置が前にある場合、処理を中断し 0 を返す。
  • find_among_b で一致が見つからない場合、処理を中断し 0 を返す。
  • slice_from_s が失敗した場合、その戻り値（負の値）をそのまま返す。
