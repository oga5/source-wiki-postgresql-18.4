xl_heap_truncate

定義

  • 種別: struct
  • 定義箇所: src/include/access/heapam_xlog.h:134

目的

ヒープテーブルの切り詰め（TRUNCATE）操作をWAL（Write-Ahead Logging）に記録するためのデータ構造です。リカバリ時に、どのデータベースのどのリレーションが切り詰められたかを特定するために使用されます。

シグネチャまたは形状

typedef struct xl_heap_truncate
{
    Oid         dbId;
    uint32      nrelids;
    uint8       flags;
    Oid         relids[FLEXIBLE_ARRAY_MEMBER];
} xl_heap_truncate;

入力と出力

  • **入力**: dbId（データベースOID）、nrelids（対象リレーション数）、flags（操作フラグ）、relids（対象リレーションOIDの可変長配列）。
  • **出力**: なし（本構造体はWALレコードのペイロードとして書き込まれます）。

前提条件と不変条件

  • relids は nrelids で指定された数だけ存在する必要があります。
  • SizeOfHeapTruncate マクロ（offsetof(xl_heap_truncate, relids)）を使用して、ヘッダー部分のサイズを計算可能です。

アルゴリズムまたは振る舞い

1. **記録**: tablecmds.c 等でTRUNCATE操作が発生した際、対象リレーションのOIDリストを本構造体に格納し、WALレコードとして書き出します。 2. **再生**: リカバリ処理（heap_redo 等）において、WALレコードから本構造体を読み込み、指定されたリレーションに対して切り詰め処理を再適用します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c: TRUNCATE実行時に本構造体を初期化・使用。
  • src/backend/replication/logical/decode.c: 論理デコード時にWALレコードから本構造体を読み取り。
  • src/backend/access/rmgrdesc/heapdesc.c: WALレコードのデコード（説明出力）時に使用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。
  • FLEXIBLE_ARRAY_MEMBER: C言語の可変長配列定義用マクロ。

失敗モードと境界条件

  • nrelids が 0 の場合、対象リレーションが存在しない不正なレコードとして扱われる可能性があります。
  • relids 配列のメモリレイアウトは FLEXIBLE_ARRAY_MEMBER に依存するため、構造体のサイズ計算には必ず SizeOfHeapTruncate マクロを使用する必要があります。

保守担当者向けメモ

  • flags フィールドには8ビットの領域が確保されており、将来的な拡張が可能です。
  • 本構造体は src/include/access/heapam_xlog.h 内で定義されており、リカバリロジックの実装は src/backend/access/heap/heapam_xlog.c に依存しています。変更時は両ファイルの整合性に注意してください。
