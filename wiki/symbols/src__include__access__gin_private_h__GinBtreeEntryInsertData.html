GinBtreeEntryInsertData

定義

  • 種別: struct
  • 定義箇所: src/include/access/gin_private.h:184

目的

GIN（Generalized Inverted Index）インデックスのエントリツリーに対する挿入操作のパラメータを保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	IndexTuple	entry;			/* 挿入するタプル */
	bool		isDelete;		/* 同一オフセットの古いタプルを削除するかどうか */
} GinBtreeEntryInsertData;

入力と出力

本構造体は、GIN インデックスの挿入処理において、挿入対象の IndexTuple と、既存タプルの置換（削除を伴う挿入）の要否を制御するフラグを保持します。

前提条件と不変条件

  • entry は有効な IndexTuple である必要があります。
  • isDelete が true の場合、挿入位置にある既存のタプルが削除対象として扱われます。

アルゴリズムまたは振る舞い

GIN インデックスの挿入処理において、エントリツリーへの挿入ロジック（ginentrypage.c 等）に渡されるペイロードとして機能します。挿入関数は本構造体を参照し、新しいタプルの書き込みと、必要に応じた既存タプルの削除処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/gininsert.c および src/backend/access/gin/ginentrypage.c 内の複数の関数が、この構造体をインスタンス化または参照しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しを直接行うものではありません。

関連データ構造

  • IndexTuple: GIN インデックスの各ノードに格納されるデータ形式。
  • GinBtreeData: GIN のエントリツリー操作のためのメソッドテーブル。

失敗モードと境界条件

  • メモリ割り当て失敗: palloc を使用して本構造体を生成する箇所（ginentrypage.c:711）において、メモリ不足が発生する可能性があります。
  • 不正なタプル: entry フィールドに不正な IndexTuple が設定された場合、インデックスの破損を招く恐れがあります。

保守担当者向けメモ

  • 本構造体は ginentrypage.c におけるエントリ挿入処理のインターフェースとして多用されています。変更を加える際は、同ファイル内の挿入ロジックへの影響を確認してください。
