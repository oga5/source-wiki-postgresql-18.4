InitCatCachePhase2

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/catcache.c:1234

目的

システムカタログキャッシュの初期化プロセスにおける第2段階を実行し、必要に応じてキャッシュに関連付けられたインデックスの整合性を検証します。

シグネチャまたは形状

void InitCatCachePhase2(CatCache *cache, bool touch_index)

入力と出力

  • **入力**:
  • cache: 初期化対象の CatCache 構造体へのポインタ。
  • touch_index: インデックスの検証を行うかどうかを指定するフラグ。
  • **出力**: なし（副作用としてキャッシュの初期化およびインデックス検証を行う）。

前提条件と不変条件

  • ConditionalCatalogCacheInitializeCache(cache) が呼び出され、キャッシュの基本構造が初期化済みであること。
  • touch_index が true の場合、対象のキャッシュが AMOID または AMNAME ではないこと。

アルゴリズムまたは振る舞い

1. ConditionalCatalogCacheInitializeCache を呼び出し、キャッシュの初期化を保証します。 2. touch_index が true かつ対象が AMOID/AMNAME 以外の場合、以下の手順でインデックスを検証します。

  • デッドロックを回避するため、対象カタログに対して AccessShareLock を取得します。
  • index_open を使用してインデックスを開きます。
  • インデックスがユニークであり、かつ即時制約（indimmediate）であることを Assert で検証します。
  • インデックスを閉じ、カタログのロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/syscache.c: InitCatCachePhase2(SysCache[cacheId], true/false) として呼び出し。
  • **呼び出し先**:
  • ConditionalCatalogCacheInitializeCache
  • LockRelationOid / UnlockRelationOid
  • index_open / index_close

関連データ構造

  • CatCache: キャッシュ管理構造体。
  • Relation: インデックス記述子。

失敗モードと境界条件

  • Assert 失敗: インデックスがユニークでない、または即時制約でない場合に発生します。
  • ロック競合: カタログに対するロック取得時に、他のプロセスとのデッドロックが発生する可能性があります（カタログを先にロックすることで回避しています）。

保守担当者向けメモ

  • 本関数は syscache.c からの呼び出しが主です。
  • AMOID および AMNAME はインデックス検証の対象外としてハードコードされています。
  • インデックスの検証は定義ミスを早期に発見するためのものであり、運用環境でのパフォーマンスへの影響は限定的です。
