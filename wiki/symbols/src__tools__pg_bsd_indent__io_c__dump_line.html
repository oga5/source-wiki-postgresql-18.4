dump_line

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/io.c:60

目的

整形されたソースコードの各セクション（ラベル、コード、コメント）を、適切なインデントを適用して出力ストリームへ書き出す。

シグネチャまたは形状

void dump_line(void)

入力と出力

  • **入力**: グローバル変数（ps 構造体、s_lab/e_lab、s_code/e_code、s_com/e_com 等）から現在の状態とバッファ内容を取得する。
  • **出力**: output ストリームへ整形済みテキストを書き出す。戻り値なし。

前提条件と不変条件

  • s_lab から e_lab、s_code から e_code、s_com から e_com がそれぞれ各セクションのバッファを指していること。
  • 実行終了後、すべてのバッファポインタはリセットされ、空の状態になる。

アルゴリズムまたは振る舞い

1. **空行処理**: 全バッファが空の場合、suppress_blanklines に応じて空行を抑制またはカウントする。 2. **整形制御**: inhibit_formatting が無効な場合、以下の順序で出力を行う。

  • **ラベル**: compute_label_target() で位置を計算し出力。#else/#endif の特殊処理を含む。
  • **コード**: compute_code_target() で位置を計算し、pad_output() を経由して出力。
  • **コメント**: ps.com_col と ps.comment_delta に基づき配置し、必要に応じて改行を挿入。

    • 3. **状態更新**: 出力後、ps（パーサー状態）のインデントレベルやステートメント継続フラグを更新し、バッファをクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/tools/pg_bsd_indent/indent.c 内の複数箇所から呼び出される。
  • **呼び出し先**: compute_label_target(), compute_code_target(), pad_output(), count_spaces(), putc(), fprintf(), fwrite() 等。

関連データ構造

  • parser_state ps: 整形状態を保持するグローバル構造体。
  • s_lab, e_lab, s_code, e_code, s_com, e_com: 各セクションのバッファ境界を示すポインタ。

失敗モードと境界条件

  • cur_col > target の場合、コメントは強制的に次行へ送られる。
  • comment_open フラグが立っている場合、出力前に .*/ を補完して閉じる。
  • ps.use_ff が真の場合、改行の代わりにフォームフィード（\014）を出力する。

保守担当者向けメモ

  • s_code 等のポインタ操作は indent_globs.h で定義されたグローバルバッファに依存している。
  • pad_output のタブ計算には postgres_tab_rules に関連する特殊なロジックが含まれるため、変更時は注意が必要。
  • not_first_line は静的変数として保持され、初回行以降の処理を制御する。
