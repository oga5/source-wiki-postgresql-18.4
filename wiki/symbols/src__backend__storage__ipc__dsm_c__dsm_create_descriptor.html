dsm_create_descriptor

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/dsm.c:1200

目的

動的共有メモリ（DSM）セグメントを管理するためのローカルな記述子（dsm_segment 構造体）をメモリ上に確保し、初期化する。

シグネチャまたは形状

static dsm_segment *dsm_create_descriptor(void)

入力と出力

  • 入力: なし
  • 出力: 初期化済みの dsm_segment 構造体へのポインタ。

前提条件と不変条件

  • TopMemoryContext が有効であること。
  • CurrentResourceOwner が設定されている場合、リソース所有者への登録が成功すること。
  • 戻り値の seg->handle は、呼び出し元で別途初期化される必要がある。

アルゴリズムまたは振る舞い

1. CurrentResourceOwner が存在する場合、リソース所有者の容量を拡張する。 2. TopMemoryContext から dsm_segment サイズ分のメモリを割り当てる。 3. 割り当てたセグメントをグローバルな dsm_segment_list の先頭に追加する。 4. 以下のフィールドを初期化する:

  • control_slot: INVALID_CONTROL_SLOT
  • impl_private: NULL
  • mapped_address: NULL
  • mapped_size: 0
  • resowner: CurrentResourceOwner

    • 5. CurrentResourceOwner が存在する場合、ResourceOwnerRememberDSM を呼び出し、リソース所有者にセグメントを登録する。 6. デタッチ時のコールバックリスト on_detach を初期化する。 7. 初期化された dsm_segment ポインタを返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/dsm.c 内の dsm_create (537行目) および dsm_attach (696行目) から呼び出される。
  • 呼び出し先: ResourceOwnerEnlarge, MemoryContextAlloc, dlist_push_head, ResourceOwnerRememberDSM, slist_init。

関連データ構造

  • dsm_segment: 本関数が生成・初期化する管理構造体。
  • dsm_segment_list: 全セグメントを追跡するグローバルリスト。
  • ResourceOwner: リソースのライフサイクル管理に使用される。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAlloc がエラーを投げる可能性がある。
  • ResourceOwner 関連の操作でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は dsm_segment のメモリ確保とリストへの登録を行うのみであり、セグメントのハンドルや実際のメモリマッピングは呼び出し元で処理される。
  • seg->handle の初期化忘れは、後続のデタッチ処理等で不整合を引き起こす可能性があるため注意すること。
