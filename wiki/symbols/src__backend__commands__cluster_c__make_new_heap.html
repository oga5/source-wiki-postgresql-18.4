make_new_heap

定義

  • 種別: function
  • 定義箇所: src/backend/commands/cluster.c:704

目的

テーブル再構築（CLUSTER や VACUUM FULL 等）の過程で、データ移行先となる新しい空のヒープリレーションを作成する。

シグネチャまたは形状

Oid make_new_heap(Oid OIDOldHeap, Oid NewTableSpace, Oid NewAccessMethod,
                  char relpersistence, LOCKMODE lockmode)

入力と出力

  • **入力**:
  • OIDOldHeap: 元となるテーブルのOID。
  • NewTableSpace: 新しいテーブルを配置するテーブルスペースのOID。
  • NewAccessMethod: 新しいテーブルで使用するアクセスメソッドのOID。
  • relpersistence: リレーションの永続性（RELPERSISTENCE_TEMP 等）。
  • lockmode: 元のテーブルに対して取得するロックモード。
  • **出力**:
  • Oid: 新規作成されたヒープリレーションのOID。

前提条件と不変条件

  • OIDOldHeap で指定されたリレーションは、呼び出し元で既に適切な lockmode でロックされている必要がある。
  • 新しいヒープは元のテーブルのデフォルト値や制約を引き継がない。
  • 新しいヒープの名称は pg_temp_<OIDOldHeap> という形式で生成される。

アルゴリズムまたは振る舞い

1. table_open で元のテーブルを開き、そのスキーマ（TupleDesc）と reloptions を取得する。 2. relpersistence に応じて適切な名前空間（pg_temp または元の名前空間）を決定する。 3. heap_create_with_catalog を呼び出し、新しいヒープを作成する。この際、元のテーブルのマップ状態を継承する。 4. CommandCounterIncrement を実行し、作成したリレーションを後続の処理から参照可能にする。 5. 元のテーブルに TOAST テーブルが存在する場合、NewHeapCreateToastTable を呼び出して新しいヒープ用の TOAST テーブルを作成する。 6. 元のテーブルを閉じて、新しいヒープの OID を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/cluster.c (rebuild_relation 内)
  • src/backend/commands/matview.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • table_open, table_close
  • heap_create_with_catalog
  • NewHeapCreateToastTable
  • CommandCounterIncrement
  • SearchSysCache1, SysCacheGetAttr, ReleaseSysCache

関連データ構造

  • Relation: 操作対象のテーブル構造体。
  • TupleDesc: テーブルの列定義情報。
  • pg_class: システムカタログ（reloptions の取得元）。

失敗モードと境界条件

  • SearchSysCache1 が失敗した場合、elog(ERROR) によりトランザクションが中断される。
  • OIDNewHeap が InvalidOid の場合は Assert により停止する。
  • 一時テーブル名（pg_temp_...）の衝突リスクがあるが、現在の実装では許容されている。

保守担当者向けメモ

  • 本関数はファイルスワップの準備段階を担う。swap_relation_files との整合性を保つため、マッピング状態の継承には注意が必要。
  • TOAST テーブルの再作成ロジックは、元のテーブルの reloptions を引き継ぐ仕様となっている。
