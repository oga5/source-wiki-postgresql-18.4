WaitEventAdjustPoll

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/waiteventset.c:789

目的

WaitEventSet 内の特定のイベント設定に基づき、OS の poll() システムコールで使用される pollfd 構造体の内容を更新・同期します。

シグネチャまたは形状

static void WaitEventAdjustPoll(WaitEventSet *set, WaitEvent *event)

入力と出力

  • **入力**:
  • set: 監視対象のイベント群を保持する WaitEventSet 構造体へのポインタ。
  • event: 更新対象の WaitEvent 構造体へのポインタ。
  • **出力**: なし（set->pollfds 配列内の該当要素を直接書き換えます）。

前提条件と不変条件

  • event->pos は set->pollfds 配列の有効なインデックスであること。
  • event->fd は PGINVALID_SOCKET ではないこと（関数末尾の Assert で検証）。
  • event->events が WL_LATCH_SET の場合、set->latch が非 NULL であること。

アルゴリズムまたは振る舞い

1. set->pollfds 配列から event->pos に対応する pollfd 構造体を取得します。 2. pollfd->revents を 0 にリセットし、pollfd->fd を event->fd に設定します。 3. event->events の種類に応じて pollfd->events（監視フラグ）を決定します：

  • WL_LATCH_SET: POLLIN を設定。
  • WL_POSTMASTER_DEATH: POLLIN を設定。
  • その他（ソケット関連）: WL_SOCKET_READABLE なら POLLIN、WL_SOCKET_WRITEABLE なら POLLOUT、WL_SOCKET_CLOSED なら（POLLRDHUP が定義されていれば）POLLRDHUP をそれぞれビット論理和で設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/waiteventset.c 内の AddWaitEventToSet および ModifyWaitEvent から呼び出されます。
  • **呼び出し先**: OS 提供の poll.h で定義された定数（POLLIN, POLLOUT, POLLRDHUP）を使用します。

関連データ構造

  • WaitEventSet: 待機セット全体を管理する構造体。
  • WaitEvent: 個別のイベント情報を保持する構造体。
  • pollfd: poll() システムコールで使用される標準的な構造体。

失敗モードと境界条件

  • event->fd が無効な場合、Assert によりデバッグビルドで停止します。
  • POLLRDHUP がプラットフォームでサポートされていない場合、WL_SOCKET_CLOSED フラグは無視されます。

保守担当者向けメモ

  • 本関数は poll() を使用する環境専用です。epoll や kqueue を使用する環境では、それぞれ WaitEventAdjustEpoll や WaitEventAdjustKqueueAdd が使用されます。
  • pollfd の設定は event->events のビットマスクに基づいて行われるため、新しいイベントタイプを追加する際は、本関数の条件分岐を適切に拡張する必要があります。
