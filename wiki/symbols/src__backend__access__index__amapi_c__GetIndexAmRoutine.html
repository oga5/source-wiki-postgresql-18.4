GetIndexAmRoutine

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/amapi.c:32

目的

指定されたインデックス・アクセスメソッド（AM）のハンドラ関数を呼び出し、そのAMが提供する IndexAmRoutine 構造体を取得します。

シグネチャまたは形状

IndexAmRoutine *GetIndexAmRoutine(Oid amhandler)

入力と出力

  • **入力**: amhandler (Oid) - 呼び出すべきAMハンドラ関数のOID。
  • **出力**: IndexAmRoutine * - AMの機能が格納された構造体へのポインタ。

前提条件と不変条件

  • amhandler は有効なハンドラ関数を指している必要があります。
  • 返される構造体は呼び出し元で palloc されるため、不要になった時点で適切に解放する必要があります。
  • システムカタログのブートストラップ中であっても安全に使用可能です。

アルゴリズムまたは振る舞い

1. OidFunctionCall0 を使用して、指定された amhandler 関数を実行します。 2. 実行結果（Datum）を IndexAmRoutine 構造体へのポインタに変換します。 3. 戻り値が NULL でないこと、および IsA マクロを用いて正しい型（IndexAmRoutine）であることを検証します。 4. 検証に失敗した場合は elog(ERROR) を発生させます。 5. 検証済みの IndexAmRoutine ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/index/amapi.c
  • src/backend/commands/indexcmds.c
  • src/backend/utils/adt/ruleutils.c
  • src/backend/utils/cache/relcache.c
  • **呼び出し先**:
  • amhandler (引数として渡された関数ポインタ)
  • OidFunctionCall0 (PostgreSQL内部API)
  • DatumGetPointer (PostgreSQL内部API)
  • IsA (PostgreSQL内部API)
  • elog (PostgreSQL内部API)

関連データ構造

  • IndexAmRoutine: インデックスAMの関数ポインタを保持する構造体。
  • Oid: 関数識別子。

失敗モードと境界条件

  • amhandler が NULL を返した場合、または IndexAmRoutine 型ではないデータを返した場合はエラーとなります。

保守担当者向けメモ

  • 本関数は pg_am カタログの amhandler 列から取得したOIDを引数として受け取ります。
  • B-treeなどの主要なAMについては、カタログ検索を回避する最適化パスが上位層で考慮される場合がありますが、本関数自体は純粋にハンドラ関数の実行と型チェックに専念します。
