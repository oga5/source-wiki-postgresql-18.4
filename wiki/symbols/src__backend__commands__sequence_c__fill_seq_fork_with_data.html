fill_seq_fork_with_data

定義

  • 種別: function
  • 定義箇所: src/backend/commands/sequence.c:358

目的

シーケンスリレーションの指定されたフォーク（MAIN_FORKNUM または INIT_FORKNUM）の先頭ページを初期化し、シーケンスデータを格納する。

シグネチャまたは形状

static void fill_seq_fork_with_data(Relation rel, HeapTuple tuple, ForkNumber forkNum)

入力と出力

  • **入力**:
  • rel: 対象となるシーケンスのリレーション記述子。
  • tuple: シーケンスの初期状態を保持する HeapTuple。
  • forkNum: 初期化対象のフォーク番号。
  • **出力**: なし（副作用としてストレージへの書き込みを行う）。

前提条件と不変条件

  • ExtendBufferedRel により、対象フォークのブロック番号 0 が確保される。
  • シーケンスは VACUUM の対象外であるため、タプルは FrozenTransactionId を持つよう強制的に設定される。
  • トランザクションがアボートした場合、このタプルは他のトランザクションから参照されないことが保証されている。

アルゴリズムまたは振る舞い

1. ExtendBufferedRel を呼び出し、ブロック 0 を取得する。 2. PageInit を使用してページを初期化し、sequence_magic（SEQ_MAGIC）をスペシャル領域に設定する。 3. 渡された tuple のヘッダー情報を、シーケンス用に凍結（FrozenTransactionId 等）および初期化する。 4. RelationNeedsWAL が真の場合、トップトランザクションIDを取得する。 5. クリティカルセクションを開始し、PageAddItem でタプルをページに追加する。 6. WALが必要な場合、または INIT_FORKNUM の場合は、XLogInsert を用いてシーケンスの更新をログに記録し、ページに LSN を設定する。 7. クリティカルセクションを終了し、バッファをアンロック・解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/sequence.c 内のシーケンス作成処理（MAIN_FORKNUM および INIT_FORKNUM を引数として呼び出し）。
  • **呼び出し先**: ExtendBufferedRel, PageInit, PageGetSpecialPointer, HeapTupleHeaderSetXmin, PageAddItem, XLogInsert 等のストレージおよびWAL管理関数。

関連データ構造

  • Relation: シーケンスのメタデータ管理に使用。
  • HeapTuple: シーケンスのデータ本体。
  • sequence_magic: ページ先頭の識別用構造体。
  • xl_seq_rec: WAL記録用の構造体。

失敗モードと境界条件

  • PageAddItem が FirstOffsetNumber を返さない場合、elog(ERROR) により処理が中断される。
  • RelationNeedsWAL の判定に基づき、WAL書き込みの有無が分岐する。

保守担当者向けメモ

  • 本関数はシーケンスの物理的な初期化を担うため、WALリプレイ時の整合性に直結する。
  • VACUUM がシーケンスを処理しないという前提に基づいたタプルの凍結処理が含まれているため、この前提を変更する場合は注意が必要である。
