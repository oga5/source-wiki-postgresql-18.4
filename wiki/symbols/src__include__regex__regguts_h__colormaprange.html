colormaprange

定義

  • **種別**: struct
  • **定義箇所**: src/include/regex/regguts.h:229

目的

正規表現エンジンにおいて、高位の文字コード（chr）の範囲を抽象化し、それらが属する色（color）を効率的に特定するためのマッピング情報を保持します。

シグネチャまたは形状

typedef struct colormaprange
{
    chr         cmin;
    chr         cmax;
    int         rownum;
} colormaprange;

入力と出力

  • **入力**: cmin および cmax により、文字コードの包含的な範囲を定義します。
  • **出力**: rownum により、hicolormap 配列内の対応する行インデックスを提供します。

前提条件と不変条件

  • rownum は常に 1 以上である必要があります。
  • cmin から cmax までの範囲は、特定の文字クラス（色）に対応します。

アルゴリズムまたは振る舞い

正規表現の実行時、特定の文字コードがどの範囲に含まれるかを判定するために使用されます。colormap 構造体内の cmranges 配列として管理され、高位文字の検索時にこの範囲情報を参照して対応する色を特定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regc_color.c 内の関数群が、colormap の構築や範囲の拡張時に本構造体を操作・参照しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • colormap: 本構造体の配列（cmranges）を保持する親構造体です。
  • chr: 正規表現エンジンで使用される文字型です。

失敗モードと境界条件

  • MALLOC_ARRAY を用いたメモリ確保時に、numcmranges に基づく領域不足が発生する可能性があります。
  • 範囲の重複や不整合が発生した場合、文字から色へのマッピングが正しく行われない可能性があります。

保守担当者向けメモ

  • 本構造体は、主に高位文字（high-valued chr）の処理に使用されます。
  • regc_color.c での配列操作（cmranges の拡張）を行う際は、メモリ確保のロジックと numcmranges の整合性に注意してください。
