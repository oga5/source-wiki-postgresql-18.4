sepgsql_set_client_label

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/label.c:110

目的

現在のクライアントプロセスのセキュリティラベルを一時的に変更し、トランザクションのコミットまでその状態を保持する。

シグネチャまたは形状

static void sepgsql_set_client_label(const char *new_label)

入力と出力

  • **入力**: new_label (設定するセキュリティコンテキスト文字列。NULL の場合は初期ラベルへリセットされる)
  • **出力**: なし（成功時は内部状態を更新、失敗時はエラーをスロー）

前提条件と不変条件

  • new_label が指定された場合、SELinux によって有効なセキュリティコンテキストである必要がある。
  • 呼び出し元は、現在のクライアントプロセスに対して process:{setcurrent} および process:{dyntransition} 権限を有している必要がある。
  • 変更内容は CurTransactionContext に紐付けられ、トランザクションのコミットまたはロールバックまで保持される。

アルゴリズムまたは振る舞い

1. **ラベル検証**: new_label が提供されている場合、security_check_context_raw() を呼び出し、無効なラベルであれば ERROR を発生させる。NULL の場合は初期ラベル (client_label_peer) を使用する。 2. **権限チェック**: sepgsql_avc_check_perms_label() を使用し、現在のラベルから対象ラベルへの遷移に必要な process:{setcurrent} および process:{dyntransition} 権限を検証する。 3. **状態の蓄積**: CurTransactionContext メモリコンテキストへ切り替え、pending_label 構造体を割り当てる。現在のサブトランザクションIDを記録し、client_label_pending リストに追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/label.c:565 (具体的な関数名はソース抜粋からは特定不可)
  • **呼び出し先**:
  • security_check_context_raw()
  • sepgsql_avc_check_perms_label()
  • sepgsql_get_client_label()
  • MemoryContextSwitchTo()
  • palloc0()
  • GetCurrentSubTransactionId()
  • pstrdup()
  • lappend()

関連データ構造

  • pending_label: トランザクションごとのラベル変更履歴を保持する構造体。
  • client_label_pending: 未コミットのラベル変更を格納するリスト。

失敗モードと境界条件

  • **無効なラベル**: security_check_context_raw() が失敗した場合、ERRCODE_INVALID_NAME でエラー終了する。
  • **権限不足**: sepgsql_avc_check_perms_label() が権限違反を検知した場合、エラー終了する。
  • **メモリ不足**: palloc0 等のメモリ割り当てが失敗した場合、PostgreSQL の標準的なメモリ管理機構によりエラー終了する。

保守担当者向けメモ

  • 本関数による変更はトランザクションに依存するため、sepgsql_xact_callback および sepgsql_subxact_callback が正しく処理を行う前提で設計されている。
  • pending_label のメモリ管理は CurTransactionContext に依存しているため、コンテキストの切り替え漏れに注意すること。
