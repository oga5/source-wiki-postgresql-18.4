ResetCatalogCachesExt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/catcache.c:814

目的

システム全体で管理されているすべてのカタログキャッシュをリセットし、メモリ上のキャッシュ状態を初期化または破棄するために使用されます。

シグネチャまたは形状

void ResetCatalogCachesExt(bool debug_discard)

入力と出力

  • **入力**:
  • debug_discard (bool): キャッシュ破棄時にデバッグ用の挙動を制御するためのフラグ。
  • **出力**: なし（void）

前提条件と不変条件

  • CacheHdr が初期化されており、有効なキャッシュリスト (ch_caches) を保持している必要があります。
  • 本関数は、システムカタログのキャッシュ整合性を再構築する際や、特定のトランザクション境界でキャッシュをクリアする際に呼び出されます。

アルゴリズムまたは振る舞い

1. CACHE_elog を使用して、リセット処理の開始をデバッグログに出力します。 2. CacheHdr->ch_caches に登録されているすべての CatCache エントリを slist_foreach で反復処理します。 3. 各 CatCache に対して ResetCatalogCache(cache, debug_discard) を呼び出し、個別のキャッシュをリセットします。 4. 最後に、処理終了をデバッグログに出力して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/catcache.c (内部的なラッパー関数)
  • src/backend/utils/cache/inval.c
  • **呼び出し先**:
  • ResetCatalogCache (各キャッシュ個別のリセット処理)

関連データ構造

  • CatCache: キャッシュ管理構造体。
  • CacheHdr: キャッシュリストを保持するグローバルなヘッダー構造体。

失敗モードと境界条件

  • CacheHdr が未初期化の状態で呼び出された場合、セグメンテーション違反が発生する可能性があります。
  • キャッシュリストが空の場合、ループは実行されず即座に終了します。

保守担当者向けメモ

  • 本関数はシステム全体のキャッシュ状態に影響を与えるため、呼び出しタイミングには注意が必要です。
  • debug_discard フラグは、キャッシュの破棄プロセスにおいて詳細なログ出力や検証を行うために使用されます。
