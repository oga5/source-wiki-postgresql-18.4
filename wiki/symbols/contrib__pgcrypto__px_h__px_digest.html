px_digest

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/px.h:107

目的

ハッシュ関数（ダイジェスト）アルゴリズムを抽象化し、共通のインターフェースを通じて暗号操作を実行するための構造体です。

シグネチャまたは形状

struct px_digest {
    unsigned (*result_size) (PX_MD *h);
    unsigned (*block_size) (PX_MD *h);
    void     (*reset) (PX_MD *h);
    void     (*update) (PX_MD *h, const uint8 *data, unsigned dlen);
    void     (*finish) (PX_MD *h, uint8 *dst);
    void     (*free) (PX_MD *h);
    union {
        unsigned code;
        void    *ptr;
    } p;
};

入力と出力

  • result_size, block_size: PX_MD 構造体へのポインタを入力とし、それぞれハッシュ値のサイズおよびブロックサイズを unsigned 型で返します。
  • update: ハッシュ対象のデータ (uint8 *) とその長さ (unsigned) を受け取り、内部状態を更新します。
  • finish: 計算結果を格納するバッファ (uint8 *) を受け取り、最終的なハッシュ値を書き込みます。
  • free: 構造体が保持するリソースを解放します。

前提条件と不変条件

  • PX_MD は px_digest の typedef です。
  • 関数ポインタは、対応するアルゴリズムの実装（OpenSSL等）によって初期化されている必要があります。

アルゴリズムまたは振る舞い

本構造体は関数ポインタのテーブルとして機能し、ポリモーフィズムを実現します。 1. reset: ハッシュ計算の状態を初期化します。 2. update: 任意の長さのデータを逐次的に処理します。 3. finish: 最終的なハッシュ値を生成し、呼び出し元が提供したメモリ領域に格納します。 4. free: 割り当てられたコンテキストを破棄します。 p メンバは、具体的なアルゴリズムの実装詳細（識別コードまたは内部状態へのポインタ）を保持します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/px.h 内で PX_MD として定義されています。具体的な呼び出し元に関する情報は提供されていません。
  • 呼び出し先: 提供されたソース excerpt には、この構造体を利用する具体的な関数呼び出しの記述はありません。

関連データ構造

  • PX_MD: px_digest の別名（typedef struct px_digest PX_MD;）。
  • uint8: データ処理に使用される符号なし8ビット整数型。

失敗モードと境界条件

  • 提供されたソース excerpt には、エラーハンドリングや境界条件に関する明示的な記述はありません。

保守担当者向けメモ

  • 本構造体は関数ポインタを多用しているため、デバッグ時には p.ptr が指す具体的な実装コンテキストを確認してください。
  • px.h 内の他の抽象化構造体（PX_Cipher, PX_HMAC 等）と同様の設計パターンに従っています。
