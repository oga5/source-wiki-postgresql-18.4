ExecAsyncResponse

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execAsync.c:116

目的

非同期実行リクエストの完了通知を受け取り、リクエスタ（要求元）ノードに対して適切な応答処理をディスパッチする。

シグネチャまたは形状

void ExecAsyncResponse(AsyncRequest *areq)

入力と出力

  • **入力**: AsyncRequest *areq - 完了した非同期リクエストを保持する構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 areq は有効なポインタであること。
  • areq->requestor に格納されたノードが非同期実行をサポートしていること。

アルゴリズムまたは振る舞い

1. areq->requestor のノード型（nodeTag）を判定する。 2. T_AppendState の場合、ExecAsyncAppendResponse(areq) を呼び出し、Append ノード固有の応答処理を実行する。 3. 非同期実行をサポートしないノード型が渡された場合、elog(ERROR) を発生させ処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execAsync.c 内の各関数
  • contrib/postgres_fdw/postgres_fdw.c
  • **呼び出し先**:
  • ExecAsyncAppendResponse（T_AppendState の場合）

関連データ構造

  • AsyncRequest: 非同期リクエストの状態を管理する構造体。
  • AppendState: 非同期実行の結果を統合する親ノード。

失敗モードと境界条件

  • nodeTag(areq->requestor) が T_AppendState 以外の場合、elog(ERROR) によりクエリ実行が異常終了する。

保守担当者向けメモ

  • 新しい非同期対応ノードを追加する場合は、本関数の switch 文に新しい nodeTag を追加し、対応する ExecAsync...Response 関数を実装すること。
