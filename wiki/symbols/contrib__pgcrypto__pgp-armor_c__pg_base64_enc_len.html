pg_base64_enc_len

定義

  • contrib/pgcrypto/pgp-armor.c:162

目的

Base64 エンコード後のデータに必要なバッファサイズを算出します。

シグネチャまたは形状

static unsigned pg_base64_enc_len(unsigned srclen)

入力と出力

  • **入力**: srclen (エンコード対象のバイト数)
  • **出力**: エンコード後の文字列に必要なバイト数（改行コードを含む）

前提条件と不変条件

  • 3バイトの入力データが4バイトの Base64 文字列に変換されることを前提としています。
  • 76文字ごとに改行コードが挿入されることを前提としています。

アルゴリズムまたは振る舞い

以下の式を用いて計算を行います。 1. (srclen + 2) / 3 * 4: Base64 変換後のデータ長を算出します。 2. srclen / (76 * 3 / 4): 76文字ごとに挿入される改行コードの数を加算します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-armor.c 内の関数（pgp_armor_encode 等）から呼び出されています。
  • **呼び出し先**: 本関数は計算のみを行うため、他の関数を呼び出しません。
  • **注記**: src/backend/utils/adt/encode.c に同名の関数が存在しますが、本シンボルとの直接的な呼び出し関係はソースコード上確認できません。

関連データ構造

  • なし

失敗モードと境界条件

  • 入力が 0 の場合、計算結果は 0 となります。
  • 非常に大きな srclen を指定した場合、符号なし整数のオーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 本関数は contrib/pgcrypto 内部の独自実装です。他のモジュールにある Base64 実装と重複しているため、変更時は整合性に注意してください。
