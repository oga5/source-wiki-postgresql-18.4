ExecIncrementalSort

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIncrementalSort.c:494

目的

インクリメンタルソートを実行するエグゼキュータノードのメイン処理関数。入力データが既にソートキーの接頭辞でソートされていることを利用し、ストリーミング的にソート済みタプルを返却する。

シグネチャまたは形状

TupleTableSlot *ExecIncrementalSort(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（IncrementalSortState 構造体を含む）
  • **出力**: ソート済みの TupleTableSlot *。ソートが完了したタプルを順次返却する。

前提条件と不変条件

  • EXEC_FLAG_BACKWARD や EXEC_FLAG_MARK はサポートしていない。
  • fullsort_state は常に最初に初期化される。
  • node->group_pivot が存在する場合、次のグループのソート開始時にそのタプルを再利用する。

アルゴリズムまたは振る舞い

1. **状態確認**: INCSORT_READFULLSORT または INCSORT_READPREFIXSORT の場合、既存のソート済みバッチからタプルを返却する。 2. **モード遷移**:

  • n_fullsort_remaining > 0 の場合、switchToPresortedPrefixMode を呼び出し、次の接頭辞キーグループを抽出する。
  • ソート済みタプルが枯渇し、遷移中でない場合は INCSORT_LOADFULLSORT へ移行する。

    • 3. **データロード**: INCSORT_LOADFULLSORT モードでは、tuplesort_begin_heap を用いてソート構造を初期化（またはリセット）し、outerNode からタプルを読み込んで蓄積する。 4. **境界条件処理**: bounded モードの場合、currentBound に基づいて tuplesort_set_bound を適用する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていないが、incrsortstate->ss.ps.ExecProcNode に割り当てられる。
  • **呼び出し先**:
  • preparePresortedCols
  • tuplesort_begin_heap / tuplesort_reset / tuplesort_gettupleslot / tuplesort_puttupleslot
  • switchToPresortedPrefixMode
  • ExecGetResultType / outerPlanState

関連データ構造

  • IncrementalSortState: ノードの状態を保持する主要構造体。
  • Tuplesortstate: ソート処理の内部状態（fullsort_state, prefixsort_state）。
  • TupleTableSlot: タプルデータの格納用スロット。

失敗モードと境界条件

  • outerNode がタプルを返さなくなった場合、outerNodeDone フラグにより終了を検知する。
  • bounded ソート時、currentBound が DEFAULT_MIN_GROUP_SIZE 未満の場合は tuplesort_set_bound を適用する。
  • group_pivot が NULL でない場合、グループ境界をまたぐタプルを保持し、次グループのソートに含める。

保守担当者向けメモ

  • 本関数は EXEC_FLAG_BACKWARD をサポートしていないため、順方向スキャンのみを前提としている。
  • n_fullsort_remaining の管理は、switchToPresortedPrefixMode との連携において重要である。
  • 統計情報の収集は INSTRUMENT_SORT_GROUP マクロ等を通じて行われる。
