pid_lock_file_exists

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/exec.c:232

目的

指定されたデータディレクトリ内に postmaster.pid ファイルが存在するかを確認し、PostgreSQL サーバーの稼働状態を判定する。

シグネチャまたは形状

bool pid_lock_file_exists(const char *datadir)

入力と出力

  • **入力**: datadir (PostgreSQL データディレクトリへのパスを示す文字列)
  • **出力**: ファイルが存在すれば true、存在しなければ false を返す。

前提条件と不変条件

  • datadir 引数は有効なディレクトリパスであること。
  • ファイルシステムへの読み取りアクセス権限が必要。

アルゴリズムまたは振る舞い

1. datadir を基に postmaster.pid へのフルパスを構築する。 2. open() システムコールを使用して、対象ファイルを読み取り専用モードで開く。 3. ファイルのオープンに成功した場合、直ちに close() を呼び出し、true を返す。 4. ファイルのオープンに失敗した場合:

  • それ以外のエラーが発生した場合は、pg_fatal を呼び出してプログラムを終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/pg_upgrade.c:366 (旧クラスタのチェック)
  • src/bin/pg_upgrade/pg_upgrade.c:390 (新クラスタのチェック)
  • **呼び出し先**:
  • snprintf (標準ライブラリ)
  • open (システムコール)
  • close (システムコール)
  • pg_fatal (ユーティリティ関数)

関連データ構造

  • なし（パス文字列とファイル記述子のみを使用）。

失敗モードと境界条件

  • postmaster.pid 以外の理由でファイルが開けない場合（権限不足など）、pg_fatal により致命的エラーとして処理される。
  • ENOENT および ENOTDIR 以外のエラーは、ファイルシステムまたは権限の異常とみなされる。

保守担当者向けメモ

  • 本関数はサーバーの稼働状態を判定する簡易的な手段であり、ファイルが存在してもロックが有効であるとは限らない点に注意すること。
  • Windows 環境におけるファイルロックの挙動については、exec.c の他の関数（exec_prog 等）で考慮されている排他制御ロジックとの整合性に留意すること。
