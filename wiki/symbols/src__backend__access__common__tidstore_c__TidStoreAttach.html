TidStoreAttach

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tidstore.c:243

目的

既存の共有メモリ上に構築された TidStore に、現在のバックエンドから接続（アタッチ）するために使用されます。

シグネチャまたは形状

TidStore *TidStoreAttach(dsa_handle area_handle, dsa_pointer handle)

入力と出力

  • **入力**:
  • area_handle: 共有メモリ領域（DSA）を識別するハンドル。
  • handle: 共有メモリ上のラディックツリーを指すポインタ。
  • **出力**:
  • 接続された TidStore 構造体へのポインタを返します。

前提条件と不変条件

  • area_handle は DSA_HANDLE_INVALID であってはなりません。
  • handle は有効な DSA ポインタである必要があります。

アルゴリズムまたは振る舞い

1. palloc0 を使用して、現在のバックエンド用の TidStore 状態構造体をメモリ上に確保します。 2. dsa_attach を呼び出し、指定された area_handle に基づいて共有メモリ領域へ接続します。 3. 共有メモリ上のラディックツリーを shared_ts_attach を介して取得し、TidStore 構造体の tree.shared フィールドに格納します。 4. 接続した dsa_area への参照を ts->area に保持し、初期化済みの TidStore ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/vacuumparallel.c: 並列 VACUUM 処理において、共有メモリ上の TidStore を取得する際に使用されます。
  • **呼び出し先**:
  • dsa_attach
  • shared_ts_attach (内部関数)
  • palloc0

関連データ構造

  • TidStore: 本関数が初期化・返却するメイン構造体。
  • dsa_area: 共有メモリ領域を管理する DSA 構造体。
  • dsa_handle / dsa_pointer: 共有メモリ上のリソースを特定するための識別子。

失敗モードと境界条件

  • 無効な area_handle や handle が渡された場合、Assert により異常終了します。
  • dsa_attach が失敗した場合の挙動は、DSA ライブラリの仕様に依存します。

保守担当者向けメモ

  • 本関数は共有メモリ版の TidStore 専用です。ローカルメモリ版の作成には TidStoreCreateLocal を使用してください。
  • 接続された TidStore を解放する際は、対応する TidStoreDetach を呼び出す必要があります。
