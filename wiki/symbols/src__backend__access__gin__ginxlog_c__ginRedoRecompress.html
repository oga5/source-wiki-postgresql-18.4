ginRedoRecompress

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginxlog.c:116

目的

GINインデックスのリーフページにおけるポスティングリストの変更（挿入、置換、追加、削除）をWALレコードに基づいて再適用し、ページの状態を復元します。

シグネチャまたは形状

static void ginRedoRecompress(Page page, ginxlogRecompressDataLeaf *data)

入力と出力

  • **入力**:
  • page: 更新対象のGINインデックスページ。
  • data: 適用すべき変更内容を含むWALレコード構造体。
  • **出力**: なし（副作用として page を直接更新）。

前提条件と不変条件

  • ページが圧縮形式でない場合（pre-9.4形式）、処理の開始時に自動的に新形式へ変換されます。
  • ページ内のポスティングリストの変更は、WALレコード内のアクション順序に従って逐次適用されます。
  • 変更適用中、ページ末尾のデータが上書きされる可能性がある場合、tailCopy を使用して一時的に退避されます。

アルゴリズムまたは振る舞い

1. **形式変換**: ページが非圧縮の場合、ginCompressPostingList を用いて圧縮形式へ変換し、メタデータを更新します。 2. **WAL解析**: data から各アクション（GIN_SEGMENT_INSERT, REPLACE, ADDITEMS, DELETE）を順次読み取ります。 3. **セグメント処理**:

  • ADDITEMS の場合、既存のセグメントをデコードし、新しいアイテムとマージして再圧縮します。
  • 変更対象のセグメント以降のデータは、必要に応じて tailCopy に退避し、整合性を保ちながらページ上に新しいセグメントを書き込みます。

    • 4. **ページ更新**: 変更後のセグメントをページ上の適切な位置に配置し、ポインタを進めます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginxlog.c 内のWALリプレイ処理（gin_redo 等の文脈から呼び出される）。
  • **呼び出し先**:
  • ginCompressPostingList, ginPostingListDecode, ginMergeItemPointers（ポスティングリスト操作）
  • palloc（メモリ確保）
  • memcpy（データコピー）

関連データ構造

  • ginxlogRecompressDataLeaf: WALレコードのデータ構造。
  • GinPostingList: GINインデックスのポスティングリスト形式。
  • Page: PostgreSQLの標準的なページ構造。

失敗モードと境界条件

  • **ページ形式**: pre-9.4形式のページに対する変換処理が含まれており、空のリーフページも適切に処理されます。
  • **メモリ**: tailCopy の確保時にメモリ不足が発生する可能性があります。
  • **アサーション**: Assert により、セグメントの順序やサイズ、ページ境界（PageGetSpecialPointer）を超えないことが検証されています。

保守担当者向けメモ

  • 本関数はGINのWALリプレイにおいて最も複雑なロジックの一つです。特に tailCopy を用いたページ末尾の保護と、ADDITEMS 時の再圧縮ロジックは、インデックスの物理構造に深く依存しています。
  • 変更を加える際は、gin_mask による整合性チェックが正しく機能するよう、ページ内の変更箇所が適切にマスクされているか確認してください。
