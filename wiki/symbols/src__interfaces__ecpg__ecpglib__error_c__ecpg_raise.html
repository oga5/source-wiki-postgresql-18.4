ecpg_raise

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/error.c:12

目的

ECPGライブラリ内部で発生したエラー情報を、アプリケーションが参照可能な sqlca（SQL Communication Area）構造体に設定し、診断メッセージを生成する。

シグネチャまたは形状

void ecpg_raise(int line, int code, const char *sqlstate, const char *str)

入力と出力

  • **入力**:
  • line: エラーが発生したソースコードの行番号。
  • code: ecpgerrno.h で定義されたECPG内部エラーコード。
  • sqlstate: SQL標準の5文字のSQLSTATEコード。
  • str: エラーメッセージに埋め込むための補足文字列（不要な場合は NULL）。
  • **出力**: なし（戻り値は void）。副作用として sqlca 構造体の内容が更新される。

前提条件と不変条件

  • ECPGget_sqlca() が有効な sqlca_t 構造体へのポインタを返すこと。
  • sqlca が NULL の場合、関数はログを出力して即座に終了する。

アルゴリズムまたは振る舞い

1. ECPGget_sqlca() を呼び出し、現在のスレッドまたはコンテキストの sqlca を取得する。 2. sqlca が取得できない場合、ecpg_log でエラーを記録し、ECPGfree_auto_mem() を実行して終了する。 3. sqlca->sqlcode に code を、sqlca->sqlstate に sqlstate をそれぞれ格納する。 4. code に基づき、ecpg_gettext を用いてローカライズされたエラーメッセージを構築し、sqlca->sqlerrm.sqlerrmc に書き込む。この際、line や str をメッセージ内に埋め込む。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/connect.c および src/interfaces/ecpg/ecpglib/data.c の各関数から、エラー発生時に呼び出される。
  • **呼び出し先**:
  • ECPGget_sqlca()
  • ecpg_log()
  • ECPGfree_auto_mem()
  • ecpg_gettext()
  • snprintf / strncpy

関連データ構造

  • sqlca_t: エラー情報を保持する構造体（sqlca.h で定義）。

失敗モードと境界条件

  • sqlca 構造体が取得できない場合、エラー情報は記録されず、メモリ解放処理のみが行われる。
  • sqlerrmc のバッファサイズ（149文字）を超えるメッセージは切り詰められる。
  • str が必要なエラーコード（例: ECPG_UNSUPPORTED）で str が NULL の場合、動作は未定義（snprintf の挙動に依存）。

保守担当者向けメモ

  • メッセージ文字列は ecpg_gettext を通じて国際化されているため、変更時は翻訳ファイルへの影響を考慮すること。
  • エラー発生時に ECPGfree_auto_mem() が呼ばれるため、本関数の呼び出し後にメモリを参照し続ける実装は避けること。
