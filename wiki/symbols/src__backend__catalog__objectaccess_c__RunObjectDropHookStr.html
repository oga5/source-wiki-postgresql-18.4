RunObjectDropHookStr

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaccess.c:179

目的

データベースオブジェクトの削除操作（DROP）が発生した際、名前ベースの識別子を用いて外部のフック関数（object_access_hook_str）を呼び出し、監査やセキュリティ拡張機能に通知を行う。

シグネチャまたは形状

void RunObjectDropHookStr(Oid classId, const char *objectName, int subId, int dropflags)

入力と出力

  • **入力**:
  • classId: 削除対象オブジェクトのカタログOID。
  • objectName: 削除対象オブジェクトの名前。
  • subId: サブオブジェクトID（該当しない場合は0）。
  • dropflags: 削除操作に関連するフラグ（dropflags）。
  • **出力**: なし（void）。

前提条件と不変条件

  • グローバル変数 object_access_hook_str が NULL であってはならない（Assert により検証）。
  • 呼び出し元は、フックが登録されていることを確認することが推奨される。

アルゴリズムまたは振る舞い

1. object_access_hook_str が非 NULL であることをアサートする。 2. ObjectAccessDrop 構造体をスタック上に確保し、ゼロクリアする。 3. 入力された dropflags を構造体に格納する。 4. 登録されているフック関数 object_access_hook_str を呼び出す。その際、イベントタイプとして OAT_DROP を指定し、オブジェクト情報と構造体へのポインタを渡す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の具体的な呼び出し元は、提供された抜粋からは特定できません。
  • **呼び出し先**: グローバル関数ポインタ object_access_hook_str が指す外部のプラグイン関数。

関連データ構造

  • ObjectAccessDrop: 削除操作のメタデータを保持する構造体。
  • OAT_DROP: ObjectAccessType 列挙型の一つで、削除イベントを示す定数。

失敗モードと境界条件

  • object_access_hook_str が NULL の状態で呼び出された場合、アサーションエラーによりプロセスが停止する（デバッグビルド時）。

保守担当者向けメモ

  • 本関数は名前ベースの識別が必要なケース専用です。OIDベースの識別が可能な場合は RunObjectDropHook を使用してください。
  • フック関数側での処理内容は、ロードされた共有ライブラリに依存します。
