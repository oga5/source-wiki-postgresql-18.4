FileFdwOption

定義

  • 種別: struct
  • 定義箇所: contrib/file_fdw/file_fdw.c:53

目的

file_fdw において、外部テーブル定義時に指定可能なオプションのメタデータを保持するために使用されます。

シグネチャまたは形状

struct FileFdwOption
{
	const char *optname;
	Oid			optcontext;
};

入力と出力

  • optname: オプション名を指す文字列ポインタ。
  • optcontext: オプションが適用可能なカタログの OID（ForeignTableRelationId 等）。

前提条件と不変条件

  • valid_options 配列の要素として定義され、オプションの検証プロセスで使用されます。

アルゴリズムまたは振る舞い

  • 外部テーブルの作成や変更時に指定されたオプションが、valid_options 内の FileFdwOption 構造体と照合され、妥当性が検証されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/file_fdw/file_fdw.c 内の検証ロジック（valid_options 配列の定義および file_fdw_validator 等の関数内での参照）。
  • 呼び出し先: 情報なし。

関連データ構造

  • valid_options: FileFdwOption 型の静的配列。有効なオプションのリストを保持します。

失敗モードと境界条件

  • オプション名が valid_options に存在しない場合、または指定されたコンテキスト（optcontext）が不適切な場合、検証エラーが発生します。

保守担当者向けメモ

  • 新しいオプションを追加する場合は、valid_options 配列に FileFdwOption 構造体を追加してください。
  • optcontext は、そのオプションがサーバー、データベース、テーブルのいずれのレベルで有効かを制御します。
