ExecHashInitializeWorker

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHash.c:2804

目的

並列クエリ実行時、各ワーカープロセスにおいてハッシュ結合の統計情報を共有メモリ領域へ蓄積するための初期化を行う。

シグネチャまたは形状

void ExecHashInitializeWorker(HashState *node, ParallelWorkerContext *pwcxt)

入力と出力

  • **入力**:
  • node: 実行中の HashState 構造体へのポインタ。
  • pwcxt: 並列ワーカーのコンテキスト情報。
  • **出力**: なし（副作用として node->hinstrument を設定）。

前提条件と不変条件

  • node->ps.instrument が有効（非NULL）であること。無効な場合は処理をスキップする。
  • pwcxt->toc に、現在のプランノード ID に対応する SharedHashInfo が登録されていること。

アルゴリズムまたは振る舞い

1. node->ps.instrument を確認し、計測が不要な場合は直ちに終了する。 2. pwcxt->toc から shm_toc_lookup を使用して、共有メモリ上の SharedHashInfo を取得する。 3. 取得した共有情報内の、現在のワーカー番号（ParallelWorkerNumber）に対応する統計格納領域を指すポインタを node->hinstrument に代入する。これにより、以降の処理で統計情報が共有領域に直接書き込まれるようになる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execParallel.c:1385
  • **呼び出し先**:
  • shm_toc_lookup (PostgreSQL 共有メモリ管理ユーティリティ)

関連データ構造

  • HashState: ハッシュ結合ノードの実行状態を保持。
  • SharedHashInfo: 並列実行時に共有されるハッシュ結合の統計情報構造体。
  • ParallelWorkerContext: 並列ワーカーの実行コンテキスト。

失敗モードと境界条件

  • shm_toc_lookup が失敗した場合（false 指定のため）はエラーが発生する可能性がある。
  • 計測（instrument）が有効でない場合、本関数は何も行わない。

保守担当者向けメモ

  • 本関数は並列実行時の統計収集に特化している。ハッシュテーブルの再構築や終了処理時に、このポインタを通じて統計が共有メモリへ反映される。
