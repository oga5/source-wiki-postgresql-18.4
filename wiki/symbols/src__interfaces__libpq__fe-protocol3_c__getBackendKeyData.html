getBackendKeyData

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-protocol3.c:1570

目的

サーバーから送信されたバックエンドのプロセスID（PID）およびキャンセルキー（BackendKeyData）を受信し、接続オブジェクト（PGconn）に格納する。

シグネチャまたは形状

static int getBackendKeyData(PGconn *conn, int msgLength)

入力と出力

  • **入力**:
  • conn: 接続状態を保持する PGconn 構造体へのポインタ。
  • msgLength: 受信したメッセージの長さ。
  • **出力**:
  • 成功時は 0 を返す。
  • 失敗時（読み込みエラーやメモリ不足等）は EOF または 0 を返し、内部で handleFatalError を呼び出して接続を終了させる場合がある。

前提条件と不変条件

  • conn は有効な PGconn 構造体であること。
  • プロトコルバージョン 3.0 において、キャンセルキーの長さは 4 バイトである必要がある。

アルゴリズムまたは振る舞い

1. 既存の conn->be_cancel_key があれば解放し、初期化する。 2. pqGetInt を使用して、メッセージから 4 バイトの be_pid を読み取る。 3. メッセージの残り長からキャンセルキーのサイズを算出する。 4. プロトコル 3.0 の場合、キャンセルキーのサイズが 4 バイト以外であればエラーとして処理する。 5. サイズが 4 バイト未満または 256 バイトを超える場合はエラーとして処理する。 6. malloc でメモリを確保し、pqGetnchar でキャンセルキー本体を読み込む。 7. 成功した場合、conn->be_cancel_key にデータを格納し、conn->be_cancel_key_len を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-protocol3.c 内のメッセージ解析ループ（pqParseInput3 等から間接的に呼び出される）。
  • **呼び出し先**:
  • pqGetInt (低レベルバッファ読み込み)
  • pqGetnchar (低レベルバッファ読み込み)
  • libpq_append_conn_error (エラー報告)
  • handleFatalError (致命的エラー時の接続終了)
  • malloc, free (メモリ管理)

関連データ構造

  • PGconn: libpq の接続状態を管理する構造体。be_pid, be_cancel_key, be_cancel_key_len フィールドが更新される。

失敗モードと境界条件

  • **メモリ不足**: malloc が失敗した場合、libpq_append_conn_error を呼び出し handleFatalError で接続を閉じる。
  • **プロトコル違反**: キャンセルキーの長さが 4 バイトでない（プロトコル 3.0）、または 4 バイト未満、256 バイト超の場合、エラーとして接続を閉じる。
  • **読み込みエラー**: pqGetInt や pqGetnchar が EOF を返した場合、処理を中断する。

保守担当者向けメモ

  • 本関数はプロトコル 3.0 専用の実装である。
  • キャンセルキーの最大長（256バイト）はハードコードされている。
  • handleFatalError を呼び出すと接続が強制終了されるため、呼び出し側での状態管理に注意すること。
