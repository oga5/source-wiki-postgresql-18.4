proclist_delete_offset

定義

  • 種別: function
  • 定義箇所: src/include/storage/proclist.h:114

目的

共有メモリ上の双方向連結リストから、指定されたプロセス番号（procno）に対応するノードを削除します。

シグネチャまたは形状

static inline void proclist_delete_offset(proclist_head *list, int procno, size_t node_offset)

入力と出力

  • list: 削除対象のリストを管理する proclist_head 構造体へのポインタ。
  • procno: リストから削除するプロセスの番号。
  • node_offset: PGPROC 構造体内の proclist_node メンバへのオフセット。
  • 戻り値: なし。

前提条件と不変条件

  • 対象ノードはリストに含まれている必要があります（node->next または node->prev が INVALID_PROC_NUMBER ではないこと）。
  • 削除対象ノードがリストの先頭である場合、list->head は procno と一致しなければなりません。
  • 削除対象ノードがリストの末尾である場合、list->tail は procno と一致しなければなりません。

アルゴリズムまたは振る舞い

1. proclist_node_get を使用して、指定された procno と node_offset からノードの実体を取得します。 2. ノードがリストの先頭にある場合、list->head をノードの next に更新します。それ以外の場合は、前方のノードの next ポインタを現在のノードの next に書き換えます。 3. ノードがリストの末尾にある場合、list->tail をノードの prev に更新します。それ以外の場合は、後方のノードの prev ポインタを現在のノードの prev に書き換えます。 4. 最後に、削除したノードの next および prev を 0 にリセットします。

呼び出し元と呼び出し先

  • 呼び出し先: proclist_node_get（ノードの解決に使用）。
  • 呼び出し元:
  • src/include/storage/proclist.h 内のリスト操作マクロから直接的または間接的に呼び出されます。
  • 具体的な呼び出し箇所として、リストの先頭削除や特定のノード削除を行うコードパスが確認されています。

関連データ構造

  • proclist_head: リストの先頭と末尾を管理する構造体。
  • proclist_node: リストの前後関係を保持する構造体。
  • PGPROC: プロセス情報を保持する構造体（node_offset の基点）。

失敗モードと境界条件

  • リストに含まれていないノードを削除しようとした場合、Assert により停止する可能性があります。
  • INVALID_PROC_NUMBER を含むノードの前後関係を正しく処理するため、条件分岐によってリストの端（head/tail）の更新を制御しています。

保守担当者向けメモ

  • 本関数はインライン関数であり、共有メモリ上のデータ構造を直接操作します。
  • node_offset を使用することで、PGPROC 構造体の特定のメンバをリストとして再利用可能にしています。
  • 呼び出し元で offsetof(PGPROC, link_member) を指定するパターンが一般的です。
