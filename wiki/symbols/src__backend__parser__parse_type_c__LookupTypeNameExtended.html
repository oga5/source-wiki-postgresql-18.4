LookupTypeNameExtended

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_type.c:72

目的

SQL文中の TypeName 構造体を解決し、対応するシステムカタログ pg_type のエントリ（Type / HeapTuple）および型修飾子（typmod）を取得します。

シグネチャまたは形状

Type LookupTypeNameExtended(ParseState *pstate,
                            const TypeName *typeName, int32 *typmod_p,
                            bool temp_ok, bool missing_ok);

入力と出力

  • **入力**:
  • pstate: パーサー状態（エラー位置報告用）。
  • typeName: 解決対象の型名構造体。
  • typmod_p: 型修飾子を格納するポインタ（不要な場合はNULL可）。
  • temp_ok: 一時テーブルの型検索を許可するか。
  • missing_ok: 型が見つからない場合にエラーとせず NULL を返すか。
  • **出力**:
  • 成功時: Type（HeapTuple へのポインタ）。
  • 失敗時: NULL（missing_ok が真の場合）。

前提条件と不変条件

  • 戻り値として返される Type（HeapTuple）は、呼び出し元で ReleaseSysCache を実行して解放する必要があります。
  • typeName->pct_type が真の場合、arrayBounds は NIL である必要があります。

アルゴリズムまたは振る舞い

1. **型名解決**:

  • typeName->typeOid が設定済みであればそれを使用。
  • %TYPE 参照の場合、指定されたリレーションのフィールド型を検索。
  • 通常の型名の場合、スキーマ修飾の有無に応じて LookupExplicitNamespace または TypenameGetTypidExtended で typoid を特定。
  • 配列指定がある場合、get_array_type で配列型 OID に変換。

    • 2. **キャッシュ検索**: TYPEOID システムキャッシュから typoid に対応する HeapTuple を取得。 3. **型修飾子解決**: typenameTypeMod を呼び出し、構文解析された typmods を内部表現に変換して typmod_p に格納。 4. **返却**: 取得したタプルを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_func.c:1890
  • src/backend/parser/parse_type.c:41
  • **呼び出し先**:
  • RangeVarGetRelid, get_attnum, get_atttype (%TYPE 解決時)
  • DeconstructQualifiedName, LookupExplicitNamespace, TypenameGetTypidExtended
  • get_array_type, SearchSysCache1, typenameTypeMod

関連データ構造

  • TypeName: 構文解析された型名情報。
  • ParseState: パーサーのコンテキスト。
  • pg_type: システムカタログ（TYPEOID キャッシュの対象）。

失敗モードと境界条件

  • 型が見つからない場合: missing_ok が偽なら ereport(ERROR) を発生。
  • %TYPE 参照で列が存在しない場合: missing_ok が偽なら ERRCODE_UNDEFINED_COLUMN でエラー。
  • 名前リストの長さが不正な場合: ERRCODE_SYNTAX_ERROR でエラー。
  • キャッシュ検索失敗時: elog(ERROR) で異常終了。

保守担当者向けメモ

  • 本関数は HeapTuple を返すため、メモリリークを防ぐために呼び出し側での ReleaseSysCache が必須です。
  • %TYPE 参照の解決時にはロックを取得しないため、同時実行される DDL との競合が発生する可能性があります。
