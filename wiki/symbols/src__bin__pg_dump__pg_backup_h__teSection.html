teSection

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_dump/pg_backup.h:55

目的

バックアップおよびリストア処理における、データベースオブジェクトの論理的な処理フェーズを分類するために使用されます。

シグネチャまたは形状

typedef enum _teSection
{
    SECTION_NONE = 1,
    SECTION_PRE_DATA,
    SECTION_DATA,
    SECTION_POST_DATA,
} teSection;

入力と出力

  • **入力**: なし（列挙型定義）。
  • **出力**: SECTION_NONE（1）、SECTION_PRE_DATA（2）、SECTION_DATA（3）、SECTION_POST_DATA（4）のいずれかの整数値。

前提条件と不変条件

  • 各値は、ダンプ対象のオブジェクトがバックアップのどの段階で処理されるべきかを示す契約として機能します。
  • SECTION_NONE は、コメントやACLなど、特定のフェーズに依存しないオブジェクトに使用されます。

アルゴリズムまたは振る舞い

  • _tocEntryRequired 関数において、現在の処理セクション（curSection）と対象オブジェクトのセクションを比較し、処理の要否を判定する基準として利用されます。
  • ダンプ/リストアの進行に伴い、この列挙値を用いて処理対象をフィルタリングします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_archiver.c 内の _tocEntryRequired 関数が、この列挙型を引数として受け取り、処理の可否を判断します。
  • **呼び出し先**: なし（列挙型のため）。
  • **参照箇所**:
  • src/bin/pg_dump/pg_backup_archiver.c (変数 curSection として利用)
  • src/bin/pg_dump/pg_backup_archiver.h (構造体メンバ section として利用)
  • src/bin/pg_dump/pg_dump.h (構造体メンバ section として利用)

関連データ構造

  • TocEntry: アーカイブ内の各エントリを保持する構造体。teSection 型のメンバ section を持ちます。

失敗モードと境界条件

  • 該当なし。

保守担当者向けメモ

  • 新しいバックアップフェーズを追加する場合は、この列挙型を拡張し、_tocEntryRequired 等の判定ロジックを更新する必要があります。
  • SECTION_NONE は他のセクションと排他的ではない可能性があるため、判定ロジックの実装時には注意してください。
