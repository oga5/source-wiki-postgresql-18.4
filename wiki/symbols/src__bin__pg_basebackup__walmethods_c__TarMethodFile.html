TarMethodFile

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_basebackup/walmethods.c:691

目的

tar 形式で WAL を保存する際、現在書き込み中の個別のファイルに関するメタデータと状態を保持するために使用されます。

シグネチャまたは形状

typedef struct TarMethodFile
{
    Walfile     base;
    pgoff_t     ofs_start;      /* ヘッダーの開始位置 */
    char        header[TAR_BLOCK_SIZE];
    size_t      pad_to_size;
} TarMethodFile;

入力と出力

  • 入力: Walfile 型の基底構造体を含む。
  • 出力: tar アーカイブ内のファイルヘッダー情報およびパディングサイズを管理する。

前提条件と不変条件

  • Walfile を継承しており、WalWriteMethod の tar 実装において使用される。
  • tar メソッドは「一度に1つのファイルのみオープン可能」という制約があるため、本構造体もそのコンテキスト下で管理される。

アルゴリズムまたは振る舞い

  • ofs_start により、アーカイブ内での当該ファイルの tar ヘッダー位置を追跡する。
  • header バッファに tar ヘッダー情報を保持し、必要に応じてチェックサムの計算や更新を行う。
  • pad_to_size を用いて、tar 形式に必要なパディング処理を制御する。

呼び出し元と呼び出し先

  • 呼び出し元: tar_write_padding_data、CreateWalTarMethod（pg_malloc0 による割り当て）、および tar メソッドの各操作関数。
  • 呼び出し先: tarChecksum（ヘッダー計算時）、lseek（ファイル位置の復元時）。

関連データ構造

  • Walfile: 基底構造体。
  • TarMethodData: tar メソッド全体の状態を保持する構造体（currentfile メンバとして本構造体を保持）。

失敗モードと境界条件

  • lseek 失敗時: lseek の戻り値と ofs_start の不一致を検出するロジックが存在する。
  • メモリ割り当て: pg_malloc0 による確保が失敗した場合はプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本構造体は walmethods.c 内の tar 実装専用である。
  • header の更新や ofs_start の操作を行う際は、tar アーカイブの整合性を保つため、必ず walmethods.c 内の tar 関連関数を経由すること。
