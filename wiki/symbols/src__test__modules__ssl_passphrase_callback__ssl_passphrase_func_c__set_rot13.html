set_rot13

定義

  • 種別: function
  • 定義箇所: src/test/modules/ssl_passphrase_callback/ssl_passphrase_func.c:55

目的

SSLコンテキストに対して、ROT13変換を行うカスタムパスフレーズコールバック関数を登録する。

シグネチャまたは形状

static void set_rot13(SSL_CTX *context, bool isServerStart)

入力と出力

  • **入力**:
  • context: 設定対象の SSL_CTX 構造体へのポインタ。
  • isServerStart: サーバー起動時かどうかを示すフラグ（本関数内では未使用）。
  • **出力**: なし（void）。

前提条件と不変条件

  • ssl_passphrase_command が設定されている場合、警告を発する。
  • OpenSSLの SSL_CTX_set_default_passwd_cb を使用して、コールバック関数 rot13_passphrase を登録する。

アルゴリズムまたは振る舞い

1. グローバル変数 ssl_passphrase_command が空でないか確認し、値が存在する場合は ereport で警告を出力する。 2. SSL_CTX_set_default_passwd_cb を呼び出し、SSLコンテキストのデフォルトパスフレーズコールバックとして rot13_passphrase を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • _PG_init（openssl_tls_init_hook への代入を通じて間接的に呼び出される）。
  • **呼び出し先**:
  • ereport（PostgreSQLのログ出力関数）。
  • SSL_CTX_set_default_passwd_cb（OpenSSLのAPI）。
  • rot13_passphrase（コールバックとして登録）。

関連データ構造

  • SSL_CTX: OpenSSLのSSLコンテキスト構造体。

失敗モードと境界条件

  • ssl_passphrase_command が設定されている場合、処理は継続されるが警告メッセージがログに出力される。

保守担当者向けメモ

  • 本関数はテスト用モジュールの一部であり、実運用環境でのセキュリティ保護を目的としたものではない。
  • openssl_tls_init_hook を介して呼び出されるため、シグネチャの変更には注意が必要。
