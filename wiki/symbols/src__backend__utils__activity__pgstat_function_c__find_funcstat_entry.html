find_funcstat_entry

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_function.c:222

目的

指定された関数OIDに対応する、現在のバックエンドにおける統計情報（PgStat_FunctionCounts）を取得します。

シグネチャまたは形状

PgStat_FunctionCounts *find_funcstat_entry(Oid func_id)

入力と出力

  • **入力**: func_id (Oid) - 統計情報を取得したい対象関数のOID。
  • **出力**: PgStat_FunctionCounts * - 該当する統計構造体へのポインタ。見つからない場合は NULL を返します。

前提条件と不変条件

  • pgstat_fetch_pending_entry を介して、現在のデータベース（MyDatabaseId）における関数統計エントリを検索します。

アルゴリズムまたは振る舞い

1. pgstat_fetch_pending_entry を呼び出し、PGSTAT_KIND_FUNCTION 種別かつ指定された func_id に一致するエントリ参照を取得します。 2. エントリ参照が存在する場合、その内部にある pending フィールド（PgStat_FunctionCounts）を返します。 3. エントリ参照が存在しない場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pgstatfuncs.c (関数統計情報を参照するSQL関数内)
  • **呼び出し先**:
  • pgstat_fetch_pending_entry (統計サブシステム内部関数)

関連データ構造

  • PgStat_FunctionCounts: 関数統計（呼び出し回数、実行時間など）を保持する構造体。
  • PgStat_EntryRef: 統計エントリへの参照を管理する構造体。

失敗モードと境界条件

  • 指定された func_id に対応する統計エントリが現在のバックエンドの保留中統計リストに存在しない場合、NULL を返します。これはエラーではなく、統計がまだ収集されていない状態として正常に処理されます。

保守担当者向けメモ

  • 本関数は pgstat_fetch_pending_entry のラッパーとして機能しています。統計情報の取得ロジックに変更がある場合は、基盤となる pgstat_fetch_pending_entry の仕様を確認してください。
