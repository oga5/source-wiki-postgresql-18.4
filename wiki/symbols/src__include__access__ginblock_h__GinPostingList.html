GinPostingList

定義

  • 種別: struct
  • 定義箇所: src/include/access/ginblock.h:336

目的

GINインデックスにおいて、圧縮されたポストリスト（Posting List）のセグメントを保持するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    ItemPointerData first;                     /* リスト内の最初のアイテム（非圧縮） */
    uint16          nbytes;                    /* 後続するバイト数 */
    unsigned char   bytes[FLEXIBLE_ARRAY_MEMBER]; /* 可変長バイト符号化されたアイテム */
} GinPostingList;

入力と出力

  • **入力**: 構造体内の bytes フィールドに格納された可変長バイト符号化データ。
  • **出力**: first フィールドによるリストの開始位置と、符号化された後続のアイテム群。

前提条件と不変条件

  • bytes フィールドは FLEXIBLE_ARRAY_MEMBER として定義されており、構造体末尾に可変長のデータが続くことを前提としています。
  • nbytes は bytes 配列に格納されている有効なデータ長を示します。

アルゴリズムまたは振る舞い

  • GINインデックスのデータページにおいて、ポストリストを複数のセグメントに分割して保持するために使用されます。
  • ginPostingListDecode 等の関数を通じて、符号化された bytes を展開し、元の ItemPointer（TID）のリストを復元します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • contrib/amcheck/verify_gin.c: ginPostingListDecode を介した検証。
  • contrib/pageinspect/ginfuncs.c: インデックス内容の検査。
  • src/backend/access/gin/gindatapage.c: GINデータページの走査および操作（GinDataLeafPageGetPostingList 等を使用）。
  • **呼び出し先**:
  • ItemPointerData（データ型として利用）。

関連データ構造

  • ItemPointerData: ポストリスト内の個々のTIDを指すために使用されます。
  • GinPageOpaqueData: GINページの特殊領域を管理する構造体。

失敗モードと境界条件

  • nbytes の値が実際のメモリ確保サイズと不一致である場合、メモリ破壊や不正な読み取りが発生する可能性があります。
  • 符号化データが破損している場合、復号処理（ginPostingListDecode）が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は可変長配列を含むため、メモリ割り当て時には sizeof(GinPostingList) + nbytes のような計算が必要です。
  • GINインデックスのバージョン管理やページレイアウトの変更に伴い、符号化方式が変更される可能性があるため、関連する gindatapage.c のロジックと併せて確認してください。
