dropDBs

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dumpall.c:1485

目的

PostgreSQL クラスタ内のデータベースを削除するための SQL コマンド（DROP DATABASE）を生成し、出力ストリーム（OPF）に書き出す。

シグネチャまたは形状

static void dropDBs(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn（データベース接続ハンドル）。
  • **出力**: なし（副作用としてグローバルな出力ストリーム OPF に SQL を書き出す）。

前提条件と不変条件

  • conn は有効なデータベース接続であること。
  • dumpDatabases() と同様のロジックでデータベースを選択する必要がある。
  • template1、template0、postgres は削除対象から除外される。

アルゴリズムまたは振る舞い

1. pg_database システムカタログから、接続可能な（datallowconn が真かつ datconnlimit != -2）データベース名の一覧を昇順で取得する。 2. 取得したデータベースが 1 つ以上存在する場合、出力ストリームにヘッダーコメントを書き出す。 3. 取得した各データベース名に対し、template1、template0、postgres 以外の名前であれば DROP DATABASE 文を生成する。 4. グローバル変数 if_exists が真の場合、IF EXISTS 句を付与する。 5. fmtId() を使用してデータベース名をクォートし、SQL を出力する。 6. 最後に PQclear で結果セットを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dumpall.c 内のメイン処理フロー（main 関数等から呼び出される）。
  • **呼び出し先**: executeQuery（クエリ実行）、PQntuples、PQgetvalue、PQclear（libpq 関数）、fmtId（識別子整形）。

関連データ構造

  • PGconn: データベース接続状態を保持する libpq 構造体。
  • PGresult: クエリ結果を保持する libpq 構造体。
  • OPF: 出力先となるグローバルなファイルポインタ。

失敗モードと境界条件

  • データベース一覧の取得に失敗した場合、executeQuery 経由でエラーが発生する可能性がある。
  • 接続が切断されている場合、PQgetvalue 等で未定義の動作を引き起こす可能性がある。
  • template0 は datallowconn の状態に関わらず明示的に除外される。

保守担当者向けメモ

  • 本関数は dumpDatabases() とデータベース選択ロジックを同期させる必要がある。変更時は両方の関数を確認すること。
  • fmtId() による識別子のクォートが適切に行われているため、特殊文字を含むデータベース名も安全に処理される。
