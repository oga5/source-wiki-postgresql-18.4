TransApplyAction

定義

  • 種別: enum
  • 定義箇所: src/backend/replication/logical/worker.c:259

目的

論理レプリケーションにおけるトランザクション適用時の処理モードを定義します。非ストリーミングトランザクションと、ストリーミングトランザクション（シリアライズや並列適用）の挙動を区別するために使用されます。

シグネチャまたは形状

typedef enum
{
    TRANS_LEADER_APPLY,           /* 非ストリーミングトランザクション用 */
    TRANS_LEADER_SERIALIZE,       /* ストリーミング用: シリアライズ */
    TRANS_LEADER_SEND_TO_PARALLEL, /* ストリーミング用: 並列ワーカーへ転送 */
    TRANS_LEADER_PARTIAL_SERIALIZE, /* ストリーミング用: 部分シリアライズ */
    TRANS_PARALLEL_APPLY,         /* 並列適用ワーカーによる適用 */
} TransApplyAction;

入力と出力

本シンボル自体は列挙型であり、入力や出力は持ちません。ただし、get_transaction_apply_action 関数などがこの型を戻り値として返し、ワーカー内の各適用処理関数がこの値を保持して動作を分岐させます。

前提条件と不変条件

  • 論理レプリケーションの適用ワーカーにおいて、トランザクションの性質に応じて適切なアクションが選択される必要があります。
  • ストリーミング関連のアクション（SERIALIZE 等）は、トランザクションがストリーミングモードである場合にのみ適用されます。

アルゴリズムまたは振る舞い

ワーカーは受信したトランザクションに対して get_transaction_apply_action を呼び出し、その結果に基づいて以下のいずれかの処理を実行します。

  • **直接適用**: TRANS_LEADER_APPLY の場合、受信した変更を即座にローカルへ適用します。
  • **ストリーミング処理**: TRANS_LEADER_SERIALIZE や TRANS_LEADER_SEND_TO_PARALLEL 等の場合、メモリ制限を回避するために一時ファイルへの退避や並列ワーカーへの転送を行います。
  • **並列適用**: TRANS_PARALLEL_APPLY の場合、並列適用ワーカーが受信した変更を適用します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/worker.c 内の複数の箇所（apply_action 変数として保持）および src/backend/replication/logical/applyparallelworker.c:147 で参照されています。
  • **呼び出し先**: 本シンボルは列挙型であるため、直接の呼び出し先はありません。

関連データ構造

  • ApplyExecutionData: 適用処理中の実行状態を保持する構造体であり、この中で apply_action が管理されることがあります。

失敗モードと境界条件

  • ストリーミングトランザクションにおいて、一時ファイルへの書き出し（SERIALIZE）が失敗した場合、適用処理は中断されます。
  • 不適切なアクションが選択された場合、論理レプリケーションの適用ロジックが整合性を失う可能性があります。

保守担当者向けメモ

  • 本シンボルは worker.c 内の get_transaction_apply_action 関数によって決定されます。ストリーミング処理の複雑なロジックを理解する際は、この列挙値がどのように分岐のトリガーとなっているかを追跡してください。
