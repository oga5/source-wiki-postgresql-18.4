free_provider

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq-oauth/oauth-curl.c:122

目的

provider 構造体が保持する動的メモリおよび libcurl 関連のリソースを解放する。

シグネチャまたは形状

static void free_provider(struct provider *provider);

入力と出力

  • **入力**: struct provider *provider - 解放対象の provider 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 provider は有効なメモリ領域を指している必要がある。
  • 構造体内の各メンバ（issuer, token_endpoint, device_authorization_endpoint）は、malloc 等で確保されたメモリ、または NULL である必要がある。

アルゴリズムまたは振る舞い

1. provider->issuer を解放する。 2. provider->token_endpoint を解放する。 3. provider->device_authorization_endpoint を解放する。 4. curl_slist_free_all を呼び出し、provider->grant_types_supported が保持する curl_slist 構造体全体を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq-oauth/oauth-curl.c:332 において、actx->provider を引数として呼び出されている。
  • **呼び出し先**: free (標準ライブラリ), curl_slist_free_all (libcurl)。

関連データ構造

  • struct provider: OAuth 2.0 認可サーバーのメタデータを保持する構造体。

失敗モードと境界条件

  • 引数 provider が NULL の場合、構造体メンバへのアクセスによりセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は provider 構造体自体のメモリ解放は行わない。呼び出し側で構造体本体のメモリ管理を行う必要がある。
