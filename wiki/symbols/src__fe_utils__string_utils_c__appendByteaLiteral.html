appendByteaLiteral

定義

  • 種別: function
  • 定義箇所: src/fe_utils/string_utils.c:526

目的

バイナリデータ（bytea）をSQLリテラル形式に変換し、指定されたバッファに追加します。

シグネチャまたは形状

void appendByteaLiteral(PQExpBuffer buf, const unsigned char *str, size_t length, bool std_strings)

入力と出力

  • buf: 変換後の文字列を格納する PQExpBuffer。
  • str: 変換対象のバイナリデータ。
  • length: str のバイト長。
  • std_strings: 標準文字列構文を使用するかどうかを示すフラグ。
  • 出力: なし（buf の内容が更新される）。

前提条件と不変条件

  • buf は有効な PQExpBuffer である必要があります。
  • 内部で enlargePQExpBuffer を呼び出し、必要なバッファサイズ（2 * length + 5）を確保します。

アルゴリズムまたは振る舞い

1. enlargePQExpBuffer でバッファを拡張します。失敗した場合は何もせず終了します。 2. リテラルの開始として ' を追加します。 3. std_strings が false の場合、エスケープ用の \ を追加します。 4. バイナリデータを16進数形式（\x プレフィックス付き）に変換し、バッファに書き込みます。 5. 最後に終了の ' を追加し、ヌル終端文字を付与して buf->len を更新します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_dump/pg_backup_archiver.h:443
  • 呼び出し先:
  • enlargePQExpBuffer（バッファ拡張用）

関連データ構造

  • PQExpBuffer: 文字列バッファを管理する構造体。

失敗モードと境界条件

  • enlargePQExpBuffer がメモリ不足等で失敗した場合、関数は何も処理を行わずに戻ります。

保守担当者向けメモ

  • 本関数は16進数フォーマット（\x...）を強制的に使用します。これは、出力先のサーバーバージョンが不明な状況で安全性を確保するためです。
  • 以前の古いエスケープ形式を使用する方が望ましいケースがある可能性がソース内のコメントで示唆されています。
