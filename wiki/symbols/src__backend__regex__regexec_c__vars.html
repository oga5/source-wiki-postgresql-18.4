vars

定義

  • 種別: struct
  • 定義箇所: src/backend/regex/regexec.c:106

目的

正規表現のマッチング実行中における状態とコンテキストを保持する構造体です。文字列の範囲、DFA（決定性有限オートマトン）のキャッシュ、エラー状態、およびサブマッチの結果を管理します。

シグネチャまたは形状

struct vars {
    regex_t    *re;
    struct guts *g;
    int         eflags;
    size_t      nmatch;
    regmatch_t *pmatch;
    rm_detail_t *details;
    chr        *start;
    chr        *search_start;
    chr        *stop;
    int         err;
    struct dfa **subdfas;
    struct dfa **ladfas;
    struct sset **lblastcss;
    chr       **lblastcp;
    struct smalldfa dfa1;
    struct smalldfa dfa2;
};

入力と出力

  • **入力**: pg_regexec 等の関数によって初期化され、マッチング対象の文字列情報やコンパイル済み正規表現（regex_t）が格納されます。
  • **出力**: マッチング結果（pmatch 配列への書き込み）および実行中のエラーコード（err）が更新されます。

前提条件と不変条件

  • dfa1 および dfa2 は、小規模な正規表現においてスタックメモリ上で完結させるための最適化として保持されます。
  • バックリファレンスを含む複雑な正規表現の場合、subdfas や ladfas を介して動的な DFA 管理が行われます。

アルゴリズムまたは振る舞い

1. **初期化**: マッチング開始時に pg_regexec によってコンテキストが構築されます。 2. **状態管理**: マッチングの進行に伴い、start から stop までの範囲で探索が行われます。 3. **DFA 活用**: rege_dfa.c が提供するロジックにより、必要に応じて DFA 状態が生成・キャッシュされます。 4. **サブマッチ特定**: バックリファレンスが存在する場合、再帰的な探索（cdissect）を通じて pmatch に正確な境界が格納されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regexec.c 内の pg_regexec が本構造体を生成・管理します。
  • **呼び出し先**: 本構造体自体はデータ構造であり、find、cfind、cdissect などのマッチング関数に引数として渡され、参照・更新されます。
  • **注意**: ツールが検出した contrib/postgres_fdw や heap.c 内の vars は、本構造体とは異なる変数名（リスト型）であり、本シンボルとは無関係です。

関連データ構造

  • regex_t: コンパイル済み正規表現の外部インターフェース。
  • struct guts: 正規表現エンジンの内部的な共通データ。
  • struct dfa: 遅延評価 DFA の状態管理構造体。
  • struct smalldfa: 小規模な DFA を保持するための構造体。
  • regmatch_t: マッチしたサブ式の位置情報。

失敗モードと境界条件

  • err フィールドが 0 以外の場合、マッチング中にリソース不足やスタックオーバーフロー等のエラーが発生したことを示します。
  • 文字列の範囲（start から stop）が不正な場合、マッチングは正しく動作しません。

保守担当者向けメモ

  • 本構造体は regexec.c 内の正規表現実行エンジンの心臓部です。フィールドの追加や変更は、rege_dfa.c や cdissect のロジックに直接影響を与えるため注意してください。
  • dfa1 と dfa2 の二重構造は、メモリ割り当てを最小限にするための最適化です。大規模な正規表現での挙動を確認する際は、これらのフィールドがどのように動的メモリへ切り替わるかを追跡してください。
