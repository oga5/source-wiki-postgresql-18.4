pg_lsn_lt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_lsn.c:135

目的

2つのログシーケンス番号（pg_lsn）を比較し、左側の値が右側の値よりも小さいかどうかを判定します。

シグネチャまたは形状

Datum pg_lsn_lt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの pg_lsn 型の引数を受け取ります。
  • **出力**: bool 型。左側のLSNが右側のLSNより小さい場合に true、それ以外の場合に false を返します。

前提条件と不変条件

  • 入力される pg_lsn は、内部的には64ビット整数（XLogRecPtr）として表現されている必要があります。
  • PG_GETARG_LSN マクロを使用して、入力された Datum から XLogRecPtr を正しく抽出できる必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_LSN(0) および PG_GETARG_LSN(1) を呼び出し、2つのLSN値を XLogRecPtr 型として取得します。 2. 取得した2つの値に対して、C言語の比較演算子 < を適用します。 3. 比較結果を PG_RETURN_BOOL マクロを介して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（演算子 < の実装として呼び出されます）。
  • **呼び出し先**: PG_GETARG_LSN および PG_RETURN_BOOL マクロ。
  • 補足: ソースコード内での直接的な関数呼び出し関係は、提供された情報からは確認できません。

関連データ構造

  • XLogRecPtr: LSNの内部表現である64ビット整数型。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • 引数が有効な pg_lsn 型でない場合、PG_GETARG_LSN マクロの内部でエラーが発生する可能性があります。
  • 64ビット整数の範囲内での比較であるため、数値的なオーバーフローやアンダーフローの懸念はありません。

保守担当者向けメモ

  • 本関数はB-treeインデックスの比較演算子として利用されることを想定しています。
  • 比較ロジックは単純な64ビット整数の比較であるため、計算コストは極めて低いです。
