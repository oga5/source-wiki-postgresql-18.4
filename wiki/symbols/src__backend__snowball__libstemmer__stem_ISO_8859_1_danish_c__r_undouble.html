r_undouble

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_danish.c:257

目的

単語の末尾にある二重子音を解消し、語幹を正規化する。

シグネチャまたは形状

static int r_undouble(struct SN_env * z)

入力と出力

  • 入力: SN_env 構造体（現在の処理対象文字列およびカーソル位置を含む）。
  • 出力: 成功時に 1、処理対象外または失敗時に 0、メモリ確保失敗等のエラー時に負の値を返す。

前提条件と不変条件

  • z->I[1] に格納された領域境界より後ろの文字を対象とする。
  • g_c グルーピング（子音）に含まれる文字が対象。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置 z->c が z->I[1] より前であれば終了する。 2. z->I[1] を探索の下限（z->lb）として設定する。 3. g_c グルーピングに含まれる文字を後方から検索し、見つかった位置を z->bra とする。 4. その文字を z->S[0] に一時保存する。 5. 保存した文字と直前の文字が一致する場合、その文字を削除（slice_del）する。 6. 処理終了後、元の z->lb を復元する。

呼び出し元と呼び出し先

  • 呼び出し元: danish_ISO_8859_1_stem（および他言語のステミング関数）。
  • 呼び出し先: in_grouping_b, slice_to, eq_v_b, slice_del（いずれもSnowballランタイム関数）。

関連データ構造

  • SN_env: ステミング処理の環境状態を保持する構造体。
  • g_c: 子音を定義するグループ。

失敗モードと境界条件

  • z->c < z->I[1]: 処理対象領域外のため 0 を返す。
  • slice_to 失敗: メモリ不足等により -1 を返す。
  • slice_del 失敗: 内部エラーにより負の値を返す。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによって自動生成されたコードである。ロジックの変更はソースとなる .sbl ファイルに対して行うこと。
  • z->I[1] は r_mark_regions で設定される領域境界であるため、本関数を呼び出す前に領域が正しくマークされている必要がある。
