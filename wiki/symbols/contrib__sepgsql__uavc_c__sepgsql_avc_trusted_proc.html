sepgsql_avc_trusted_proc

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/uavc.c:444

目的

指定されたプロシージャ（関数）に関連付けられたセキュリティラベルに基づき、実行時に適用すべき新しいセキュリティコンテキスト（ncontext）を取得します。

シグネチャまたは形状

char *sepgsql_avc_trusted_proc(Oid functionId)

入力と出力

  • **入力**: functionId (Oid) - 対象となるプロシージャのオブジェクト識別子。
  • **出力**: char * - プロシージャ実行時に切り替えるべきセキュリティコンテキスト文字列。

前提条件と不変条件

  • sepgsql_get_client_label() により、現在のクライアントのセキュリティラベルが取得可能であること。
  • GetSecurityLabel を用いて、対象プロシージャのセキュリティラベルが取得可能であること。
  • キャッシュの整合性を保つため、sepgsql_avc_check_valid() を用いたポリシー更新チェックのループ内で実行されること。

アルゴリズムまたは振る舞い

1. 現在のクライアントラベル（scontext）と、対象プロシージャのセキュリティラベル（tcontext）を取得します。 2. sepgsql_avc_check_valid() を呼び出し、キャッシュの有効性を確認します。 3. do-while ループ内で以下を実行します：

  • tcontext が存在する場合はそのラベルを、存在しない場合は sepgsql_avc_unlabeled() を使用します。
  • sepgsql_avc_lookup() を呼び出し、SEPG_CLASS_DB_PROCEDURE クラスに対するキャッシュエントリを検索します。
  • ポリシーが更新された場合（sepgsql_avc_check_valid() が偽を返す場合）、検索を再試行します。

    • 4. 取得したキャッシュエントリから ncontext を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/label.c:282
  • contrib/sepgsql/label.c:332
  • **呼び出し先**:
  • sepgsql_get_client_label()
  • GetSecurityLabel()
  • sepgsql_avc_check_valid()
  • sepgsql_avc_lookup()
  • sepgsql_avc_unlabeled()

関連データ構造

  • avc_cache: キャッシュエントリを保持する構造体。ncontext フィールドを介して結果を返します。
  • ObjectAddress: プロシージャを特定するために使用されます。

失敗モードと境界条件

  • 対象プロシージャにセキュリティラベルが設定されていない場合、sepgsql_avc_unlabeled() が提供するデフォルトのラベルを使用して検索が行われます。
  • ポリシーのリロードが発生した場合、ループによって最新のキャッシュ状態への同期が試みられます。

保守担当者向けメモ

  • 本関数は DB_PROCEDURE クラスの権限チェックにおいて、単なる許可判定だけでなく、実行時のセキュリティラベル切り替え（ドメイン遷移）を制御するために使用されます。
  • キャッシュの整合性維持のため、必ず sepgsql_avc_check_valid() を含むループ構造を維持してください。
