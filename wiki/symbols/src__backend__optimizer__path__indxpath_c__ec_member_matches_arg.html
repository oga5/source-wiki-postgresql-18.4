ec_member_matches_arg

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/path/indxpath.c:72

目的

等価クラス（EquivalenceClass）のメンバーとインデックス列のマッチング処理において、対象となるインデックス情報と列番号を保持するための引数構造体です。

シグネチャまたは形状

typedef struct
{
	IndexOptInfo *index;		/* 検討中のインデックス */
	int			indexcol;		/* マッチ対象のインデックス列番号 */
} ec_member_matches_arg;

入力と出力

本構造体は関数への引数として渡されるデータコンテナです。

  • 入力: IndexOptInfo へのポインタおよび int 型の列番号。
  • 出力: なし（データ保持用）。

前提条件と不変条件

  • index は有効な IndexOptInfo 構造体を指している必要があります。
  • indexcol は、指定された index が持つ列の範囲内である必要があります。

アルゴリズムまたは振る舞い

この構造体は、コールバック関数等へ複数のパラメータをまとめて渡すための「引数パケット」として機能します。処理の過程で index および indexcol フィールドが読み取られ、インデックス列とクエリ条件の適合判定に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/indxpath.c 内の関数（例: 2527行目付近でインスタンス化）。
  • **呼び出し先**: 本構造体を参照する関数（例: 4094-4095行目付近で、キャストされた arg から各フィールドが抽出される）。

関連データ構造

  • IndexOptInfo: インデックスのメタデータと統計情報を保持する構造体。

失敗モードと境界条件

  • arg が NULL の状態でキャストしてアクセスした場合、セグメンテーション違反が発生します。
  • indexcol がインデックスの定義範囲外である場合、後続の処理で不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は、特定のコールバック関数（check_index_only 等の述語関数）にコンテキストを渡すために利用されています。
  • 構造体のフィールドを追加・変更する場合は、この構造体を利用しているすべての箇所（特にキャストを行っている箇所）で整合性が取れているか確認してください。
