SerializeParamList

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/params.c:228

目的

ParamListInfo 構造体に保持されたパラメータリストを、プロセス間通信や永続化のためにバイナリ形式へシリアライズします。

シグネチャまたは形状

void SerializeParamList(ParamListInfo paramLI, char **start_address)

入力と出力

  • **入力**:
  • paramLI: シリアライズ対象の ParamListInfo 構造体。
  • start_address: シリアライズ先メモリ領域の開始アドレスを指すポインタ。関数実行後、書き込み済みのサイズ分だけアドレスが進められます。
  • **出力**: なし（start_address を介した副作用のみ）。

前提条件と不変条件

  • paramLI が NULL または numParams <= 0 の場合、パラメータ数は 0 として扱われます。
  • paramFetch フックが設定されている場合、各パラメータの取得にはそのフックが優先的に使用されます。
  • typeOid が無効な場合、Datum は値渡し（by-value）として扱われます。

アルゴリズムまたは振る舞い

1. パラメータ数（nparams）を書き込み、start_address を進めます。 2. 各パラメータについて以下の順序でシリアライズを実行します：

  • paramFetch フックまたは params 配列から ParamExternData を取得します。
  • パラメータの型 OID を書き込みます。
  • パラメータのフラグ（pflags）を書き込みます。
  • get_typlenbyval を用いて型の長さを特定し、datumSerialize を呼び出して Datum 値と NULL フラグを書き込みます。

    • 3. 各ステップで start_address を書き込みサイズ分だけインクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execParallel.c:774 (パラレルクエリ実行時のパラメータ転送)
  • **呼び出し先**:
  • paramLI->paramFetch (動的パラメータ取得時)
  • get_typlenbyval (型情報の取得)
  • datumSerialize (データ本体のシリアライズ)

関連データ構造

  • ParamListInfo: パラメータリストを保持する主要構造体。
  • ParamExternData: 個々のパラメータの型、値、フラグを保持する構造体。

失敗モードと境界条件

  • paramLI が NULL の場合、パラメータ数 0 として正常に処理されます。
  • typeOid が無効な場合、datumSerialize は sizeof(Datum) を長さとして使用します。
  • ログ出力用の paramValuesStr はシリアライズ対象外であり、復元されません。

保守担当者向けメモ

  • 本関数は EstimateParamListSpace と対になる実装である必要があります。シリアライズ対象のフィールドを追加・変更した場合は、見積もりロジックも更新してください。
  • paramFetch フックを使用している場合、シリアライズ中に動的な値が変化する可能性がある点に注意してください。
