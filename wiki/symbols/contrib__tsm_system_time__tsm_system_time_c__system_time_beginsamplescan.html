system_time_beginsamplescan

定義

  • 種別: function
  • 定義箇所: contrib/tsm_system_time/tsm_system_time.c:189

目的

TABLESAMPLE SYSTEM_TIME メソッドにおけるサンプリングスキャンの実行状態を初期化し、指定された時間制限と乱数シードを適用します。

シグネチャまたは形状

static void system_time_beginsamplescan(SampleScanState *node, Datum *params, int nparams, uint32 seed)

入力と出力

  • node: サンプリング状態を保持する SampleScanState 構造体へのポインタ。
  • params: ユーザーが指定したサンプリングパラメータ（params[0] はミリ秒単位の時間制限）。
  • nparams: パラメータ数。
  • seed: 乱数生成に使用するシード値。
  • 出力: なし（node->tsm_state 内の SystemTimeSamplerData を更新）。

前提条件と不変条件

  • params[0] に指定されたミリ秒数は負の値であってはならず、また NaN であってはならない。
  • node->tsm_state は SystemTimeSamplerData 型としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. params[0] からミリ秒単位の時間制限を取得し、妥当性を検証する。不正な場合は ERRCODE_INVALID_TABLESAMPLE_ARGUMENT でエラーを報告する。 2. node->tsm_state を SystemTimeSamplerData として取得する。 3. 取得した seed と millis を構造体に格納する。 4. スキャン状態の初期値として lt（最終タプルオフセット）を InvalidOffsetNumber に、doneblocks（完了ブロック数）を 0 にリセットする。 5. start_time およびブロック選択用の変数は、最初の system_time_nextsampleblock 呼び出し時に初期化されるため、ここでは変更しない。

呼び出し元と呼び出し先

  • 呼び出し元: tsm_system_time_handler 内で TsmRoutine の BeginSampleScan コールバックとして登録され、クエリ実行エンジンから呼び出される。
  • 呼び出し先: ereport（エラー発生時）。

関連データ構造

  • SystemTimeSamplerData: サンプリングの実行状態を保持する構造体。
  • SampleScanState: PostgreSQL のテーブルスキャン状態を管理する構造体。

失敗モードと境界条件

  • millis < 0 または isnan(millis) の場合、エラーを発生させて処理を中断する。

保守担当者向けメモ

  • 本関数はスキャンの開始準備のみを行い、実際の時間計測やブロック選択の初期化は system_time_nextsampleblock に委譲されています。状態の追加や変更を行う際は、後続の nextsampleblock への影響を考慮してください。
