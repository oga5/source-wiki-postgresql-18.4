InitCompressFileHandleGzip

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_gzip.c:418

目的

CompressFileHandle 構造体に対し、GZIP 圧縮形式を扱うための関数ポインタ群を割り当て、初期化を行う。

シグネチャまたは形状

void InitCompressFileHandleGzip(CompressFileHandle *CFH, const pg_compress_specification compression_spec)

入力と出力

  • **入力**:
  • CFH: 初期化対象となる CompressFileHandle 構造体へのポインタ。
  • compression_spec: 圧縮設定を保持する pg_compress_specification 構造体。
  • **出力**: なし（CFH のメンバを直接書き換える）。

前提条件と不変条件

  • CFH は有効なメモリ領域を指している必要がある。
  • HAVE_LIBZ が定義されている環境で動作することを前提とする（未定義時の挙動についてはソースコードのガードロジックを参照）。

アルゴリズムまたは振る舞い

1. CFH の各関数ポインタメンバに、GZIP 固有の実装関数（Gzip_open, Gzip_read, Gzip_write, Gzip_close 等）を代入する。 2. 引数 compression_spec を CFH->compression_spec に格納する。 3. CFH->private_data を NULL に初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_io.c:203 から呼び出されることが確認されている。
  • **呼び出し先**: 本関数自体は関数ポインタの代入のみを行う。代入される各 Gzip_* 関数が後続の処理で呼び出される。

関連データ構造

  • CompressFileHandle: 圧縮ファイル操作のインターフェースを定義する構造体。
  • pg_compress_specification: 圧縮レベルやアルゴリズム等の設定を保持する構造体。

失敗モードと境界条件

  • HAVE_LIBZ が定義されていない場合、本関数はスタブとして動作し、エラーを発生させる可能性がある（詳細は compress_gzip.c 全体の実装に依存）。

保守担当者向けメモ

  • 本関数は compress_io.c が提供する汎用インターフェースと、zlib を利用した GZIP 実装を繋ぐブリッジである。
  • 新たなファイル操作関数を追加した場合は、本関数で適切にポインタを割り当てる必要がある。
