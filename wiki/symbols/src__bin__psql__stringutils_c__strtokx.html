strtokx

定義

  • 種別: function
  • 定義箇所: src/bin/psql/stringutils.c:51

目的

引用符（クォート）やエスケープシーケンスを考慮し、入力文字列をトークン単位に分割する。標準の strtok と異なり、入力文字列を破壊せずにトークンを切り出す。

シグネチャまたは形状

char *strtokx(const char *s, const char *whitespace, const char *delim, const char *quote, char escape, bool e_strings, bool del_quotes, int encoding);

入力と出力

  • **入力**:
  • s: 解析対象の文字列（初回呼び出し時のみ指定、以降は NULL）。
  • whitespace: トークン間の空白文字セット。
  • delim: トークンとして扱う区切り文字セット。
  • quote: 引用符として扱う文字セット。
  • escape: エスケープ文字。
  • e_strings: E'...' 構文を許可するか。
  • del_quotes: トークンから引用符を除去するか。
  • encoding: マルチバイト文字対応のためのエンコーディング指定。
  • **出力**: 切り出されたトークンのポインタ。解析終了時は NULL を返す。

前提条件と不変条件

  • 静的変数 storage を使用して入力文字列のコピーを保持するため、スレッドセーフではない。
  • e_strings が true の場合、E'...' 構文が優先的に処理される。
  • e_strings と del_quotes を同時に true にすることはサポートされていない。

アルゴリズムまたは振る舞い

1. **初期化**: s が指定された場合、内部バッファ storage にコピーを作成する。 2. **スキップ**: 先頭の whitespace を読み飛ばす。 3. **区切り文字判定**: delim に該当する場合、その位置で文字列を終端し、トークンを返す。 4. **引用符処理**: quote に該当する場合、対応する閉じ引用符までをトークンとして走査する。エスケープ文字や二重引用符を適切に処理する。 5. **通常トークン**: 上記以外の場合、次の whitespace、delim、または quote までをトークンとして切り出す。 6. **後処理**: del_quotes が有効な場合、strip_quotes を呼び出して引用符を除去する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/copy.c 内の各箇所で、COPY コマンドの引数解析に使用されている。
  • **呼び出し先**: pg_malloc（メモリ確保）、strspn/strcspn/strchr（文字列検索）、memmove（バッファ調整）、PQmblenBounded（マルチバイト文字走査）、strip_quotes（引用符除去）。

関連データ構造

  • 内部静的変数: storage（入力文字列のコピー）、string（解析継続位置のポインタ）。

失敗モードと境界条件

  • 入力文字列が空、または解析完了時は NULL を返す。
  • 引用符が閉じられていない場合、文字列の末尾までをトークンとして扱う。
  • storage のメモリ確保に失敗した場合は NULL を返す。

保守担当者向けメモ

  • psql はシングルスレッド前提のため、静的変数による状態保持は許容されている。
  • トークン化の際、隣接するトークンのために 2 * strlen(s) + 1 のバッファを確保しており、メモリ効率よりも実装の単純さを優先している。
