AddFileToBackupManifest

定義

  • 種別: function
  • 定義箇所: src/backend/backup/backup_manifest.c:101

目的

バックアップ・マニフェスト（JSON形式）に対し、個別のファイル情報を追加します。ファイルパス、サイズ、最終更新時刻、およびチェックサム情報を記録し、バックアップの整合性検証を可能にします。

シグネチャまたは形状

void AddFileToBackupManifest(backup_manifest_info *manifest, Oid spcoid,
                             const char *pathname, size_t size, pg_time_t mtime,
                             pg_checksum_context *checksum_ctx)

入力と出力

  • **入力**:
  • manifest: マニフェスト生成状態を保持する構造体。
  • spcoid: テーブルスペースOID（指定時はパス解決に使用）。
  • pathname: ファイルのパス。
  • size: ファイルサイズ。
  • mtime: 最終更新時刻。
  • checksum_ctx: チェックサム計算コンテキスト。
  • **出力**: なし（副作用として manifest が保持する一時ファイルにJSONエントリを追記）。

前提条件と不変条件

  • IsManifestEnabled(manifest) が真であること。
  • マニフェストはUTF-8で記録されるため、パス名がUTF-8として不正な場合はエンコード処理が行われる。
  • マニフェスト内の各ファイルエントリはJSONオブジェクトとして記述され、カンマで区切られる。

アルゴリズムまたは振る舞い

1. **パスの正規化**: spcoid が有効な場合、パスをテーブルスペースディレクトリ配下の相対パスに変換する。 2. **JSON構築**: StringInfo を使用してJSONエントリを生成する。

  • 初回エントリ以外には先行するカンマと改行を挿入する。
  • パス名がUTF-8として妥当か判定し、必要に応じて Encoded-Path として16進数エンコードを行う。
  • サイズおよびGMT形式の最終更新時刻をJSONに追加する。

    • 3. **チェックサム処理**: checksum_ctx が有効な場合、pg_checksum_final で値を確定させ、アルゴリズム名と共に16進数文字列として追加する。 4. **書き込み**: AppendStringToManifest を呼び出し、構築したJSON文字列を一時ファイルへ出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c (バックアップのメインループ内)
  • **呼び出し先**:
  • IsManifestEnabled
  • initStringInfo, appendStringInfo, appendStringInfoChar, appendStringInfoString
  • escape_json_with_len, hex_encode
  • pg_verify_mbstr, pg_strftime, pg_gmtime
  • pg_checksum_final, pg_checksum_type_name
  • AppendStringToManifest

関連データ構造

  • backup_manifest_info: マニフェスト生成のメタデータ（first_file フラグや一時ファイルへの参照を保持）。
  • pg_checksum_context: チェックサム計算の状態管理。

失敗モードと境界条件

  • pg_checksum_final が失敗した場合、elog(ERROR) により処理を中断する。
  • パス名がUTF-8として不正かつ force_encode が指定されていない場合、自動的にエンコード処理へフォールバックする。

保守担当者向けメモ

  • 本関数はメモリリークを防ぐため、最後に pfree(buf.data) を実行している。
  • JSONの構文（カンマの制御）は manifest->first_file フラグに依存しているため、このフラグの状態を外部から変更しないこと。
