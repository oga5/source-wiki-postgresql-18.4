RT_NODE_256

定義

  • 種別: struct
  • 定義箇所: src/include/lib/radixtree.h:568

目的

適応型基数木（ART）において、256個の全スロットを直接保持する最大サイズのノード構造を提供します。子ノードの密度が高い場合に、検索のオーバーヘッドを最小化するために使用されます。

シグネチャまたは形状

typedef struct RT_NODE_256
{
    RT_NODE         base;
    bitmapword      isset[RT_BM_IDX(RT_FANOUT_256)];
    RT_PTR_ALLOC    children[RT_FANOUT_256];
} RT_NODE_256;

入力と出力

  • **入力**: 8ビットのチャンク値（キーの一部）をインデックスとして使用し、children 配列から対応する子ノードポインタを取得します。
  • **出力**: RT_PTR_ALLOC 型の子ノードポインタ、または空の場合は無効なポインタを返します。

前提条件と不変条件

  • base フィールドとして RT_NODE を先頭に含み、ノード共通のメタデータ（種別やサイズクラス情報）を保持します。
  • isset ビットマップは、children 配列の各スロットが有効かどうかを追跡するために使用されます。
  • RT_FANOUT_256（256）個の固定長スロットを保持します。

アルゴリズムまたは振る舞い

  • 検索時、キーのチャンク値を直接インデックスとして使用するため、定数時間 $O(1)$ で子ノードにアクセス可能です。
  • ノードが満杯に近い状態、あるいは特定の閾値を超えた際に、より小さなノード（RT_NODE_48 等）から昇格して生成されます。
  • RT_NODE_256_GET_CHILD 等のマクロを通じて、ビットマップの確認とポインタ取得が統合的に行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: 基数木の挿入・検索・削除ロジック（src/include/lib/radixtree.h 内の各関数）から、ノードの型キャストや操作を通じて参照されます。
  • **呼び出し先**: 直接的な関数呼び出しではなく、メモリ確保やポインタ操作を伴うマクロ（RT_NODE_256_GET_CHILD など）が本構造体を利用します。

関連データ構造

  • RT_NODE: 基底構造体。
  • RT_NODE_4, RT_NODE_16, RT_NODE_48: 他の適応型ノードサイズクラス。
  • RT_RADIX_TREE: 本ノードを階層的に保持するメイン構造体。

失敗モードと境界条件

  • メモリ不足時: RT_PTR_ALLOC の確保に失敗する可能性があります。
  • 境界条件: 256個を超える子ノードは保持できません。これを超える場合は、基数木の階層構造が深くなることで対応します。

保守担当者向けメモ

  • 本構造体は RT_FANOUT_256 に基づく固定サイズであるため、メモリ消費量が大きくなります。ノードの密度が低い場合は、より小さなノード種別への降格が検討される設計となっています。
  • offsetof(RT_NODE_256, children) を使用したメモリ初期化が行われているため、構造体のレイアウト変更時には注意が必要です。
