dumpRoles

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dumpall.c:822

目的

PostgreSQL クラスタ内のロール定義を抽出し、SQL スクリプトとして出力する。

シグネチャまたは形状

static void dumpRoles(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn（データベース接続ハンドル）
  • **出力**: なし（直接 OPF ストリームへ SQL を書き出す）

前提条件と不変条件

  • role_catalog が有効なカタログ名を参照していること。
  • server_version に応じて適切なクエリが選択されること。
  • pg_ で始まるロール名はシステムロールとして除外される。

アルゴリズムまたは振る舞い

1. サーバーバージョンに基づき、pg_authid 等からロール属性（権限、パスワード、有効期限等）を取得するクエリを構築・実行する。 2. 結果セットが存在する場合、出力ストリームに -- Roles ヘッダーを書き出す。 3. 各ロールについて以下の処理を繰り返す：

  • pg_ で始まる名前の場合は警告を出してスキップする。
  • binary_upgrade モード時は、OID を保存するための binary_upgrade_set_next_pg_authid_oid を出力する。
  • CREATE ROLE 文を生成する（現在のユーザー自身の場合はスキップされる場合がある）。
  • ALTER ROLE 文を生成し、スーパーユーザー権限、継承、ログイン可否、接続制限、パスワード、有効期限等の属性を順次追加する。
  • コメント（COMMENT ON ROLE）およびセキュリティラベル（buildShSecLabels）を必要に応じて出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dumpall.c 内のメイン処理フローから呼び出される。
  • **呼び出し先**: executeQuery, printfPQExpBuffer, appendPQExpBuffer, fmtId, buildShSecLabels 等のユーティリティ関数。

関連データ構造

  • PGconn: データベース接続状態。
  • PQExpBuffer: SQL 文構築用バッファ。
  • PGresult: クエリ結果保持用。

失敗モードと境界条件

  • PQntuples(res) == 0: ロールが存在しない場合は何も出力しない。
  • binary_upgrade モード: 現在の接続ユーザーに対する CREATE ROLE を抑制する。
  • no_role_passwords / no_comments / no_security_labels: 各フラグが有効な場合、対応する SQL 出力を抑制する。

保守担当者向けメモ

  • rolconfig（ロールごとの設定）は本関数ではダンプされず、後続の処理で扱われる点に注意すること。
  • ロールの依存関係やメンバーシップについては dumpRoleMembership が別途担当する。
