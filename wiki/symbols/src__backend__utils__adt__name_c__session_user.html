session_user

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/name.c:268

目的

現在のセッションユーザー名を取得し、PostgreSQL の name 型として返却します。

シグネチャまたは形状

Datum session_user(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL 関数呼び出し規約に基づく引数）。
  • **出力**: Datum 型。現在のセッションユーザー名を表す name 型のデータ。

前提条件と不変条件

  • 内部的に GetSessionUserId() を呼び出すため、セッション状態が初期化されている必要があります。
  • 返却されるユーザー名は name 型の制約（NAMEDATALEN）に従います。

アルゴリズムまたは振る舞い

1. GetSessionUserId() を呼び出し、現在のセッションユーザーの OID を取得します。 2. GetUserNameFromId(oid, false) を呼び出し、OID に対応するユーザー名を文字列（cstring）として取得します。 3. DirectFunctionCall1 を介して namein 関数を呼び出し、取得した文字列を name 型の Datum に変換します。 4. 変換された Datum を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c:3244（エグゼキュータによる式評価）
  • **呼び出し先**:
  • GetSessionUserId()
  • GetUserNameFromId()
  • namein()

関連データ構造

  • name: PostgreSQL の固定長識別子型。
  • Datum: PostgreSQL の汎用的なデータ表現型。

失敗モードと境界条件

  • GetUserNameFromId の第2引数に false を指定しているため、対応するユーザーが存在しない場合はエラーが発生します。

保守担当者向けメモ

  • 本関数は SQL レベルの SESSION_USER 関数を実装しています。
  • namein を介して変換を行うため、name 型のメモリ管理（palloc0 等）は namein 側の実装に依存します。
