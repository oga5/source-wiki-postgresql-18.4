r_emphasis

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_estonian.c:881

目的

エストニア語のステミング処理において、単語内の強調表現に関連する接尾辞を特定し、条件に応じて削除する。

シグネチャまたは形状

static int r_emphasis(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（Snowball ランタイム環境構造体）
  • 出力: 処理が成功した場合は 1、失敗または条件不一致の場合は 0、エラー発生時は負の値を返す。

前提条件と不変条件

  • z->c（現在のカーソル位置）が z->I[0]（領域境界）以上であること。
  • among_var の判定には a_0 配列が使用される。
  • 処理対象の単語末尾に特定の文字（UTF-8）が存在することが前提となる。

アルゴリズムまたは振る舞い

1. **境界チェック**: 現在のカーソル位置が領域境界 z->I[0] より前であれば終了する。 2. **接尾辞検索**: find_among_b を使用し、a_0 配列に基づいて後方から接尾辞を検索する。 3. **条件分岐**:

  • among_var == 1: g_GI グループの文字が直前に存在し、かつ r_LONGV が適用されない場合に slice_del で削除する。
  • among_var == 2: g_KI グループの文字が直前に存在する場合に slice_del で削除する。

    • 4. **UTF-8 処理**: skip_b_utf8 を用いてマルチバイト文字の境界を適切にスキップする。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_estonian.c 内のステミングパイプライン（行1382付近）。
  • 呼び出し先: find_among_b, skip_b_utf8, in_grouping_b_U, r_LONGV, slice_del。

関連データ構造

  • SN_env: ステミング処理の環境状態を保持する構造体。
  • a_0: among 構造体の配列（接尾辞定義）。
  • g_GI, g_KI: 文字グループ定義。

失敗モードと境界条件

  • find_among_b が一致する接尾辞を見つけられない場合は 0 を返す。
  • skip_b_utf8 が不正な UTF-8 シーケンスを検出した場合、負の値を返す。
  • slice_del がメモリ操作等で失敗した場合、負の値を返す。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードであるため、直接編集せず元の .sbl ファイルを修正すること。
  • z->I[0] は r_mark_regions で設定される領域境界を指す。デバッグ時は SN_env のカーソル位置と領域境界の整合性に注意すること。
