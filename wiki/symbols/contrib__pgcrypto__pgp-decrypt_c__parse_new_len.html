parse_new_len

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-decrypt.c:51

目的

OpenPGP パケットの「新しい形式（New Packet Format）」における長さ情報を解析し、パケットのデータ長またはストリーム形式であることを特定します。

シグネチャまたは形状

static int parse_new_len(PullFilter *src, int *len_p)

入力と出力

  • **入力**:
  • src: データを読み出すための PullFilter ポインタ。
  • len_p: 解析した長さを格納するための整数ポインタ。
  • **出力**:
  • 成功時: パケットタイプ（PKT_NORMAL または PKT_STREAM）を返します。
  • 失敗時: PXE_PGP_CORRUPT_DATA を返します。

前提条件と不変条件

  • src から GETBYTE マクロを用いてバイト単位の読み取りを行います。
  • 算出された長さ len は、0 以上 MAX_CHUNK 以下である必要があります。

アルゴリズムまたは振る舞い

1. src から最初のバイト b を読み取ります。 2. b の値に応じて長さを決定します：

  • b <= 191: 長さは b です。
  • 192 <= b <= 223: 長さは ((b - 192) << 8) + 192 + 次のバイト です。
  • b == 255: 続く 4 バイトをビッグエンディアンの整数として読み取り、長さを決定します。
  • 上記以外: 1 << (b & 0x1F) を長さとし、パケットタイプを PKT_STREAM に設定します。

    • 3. 長さが範囲外（負または MAX_CHUNK 超過）の場合はエラーを返します。 4. 正常終了時は len_p に長さを書き込み、パケットタイプを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-decrypt.c 内のパケット解析処理（151行目、192行目など）から呼び出されます。
  • **呼び出し先**: GETBYTE マクロ（src からの読み取り）、px_debug（エラーログ出力）。

関連データ構造

  • PullFilter: データのストリーム読み取りに使用される構造体。
  • PKT_NORMAL, PKT_STREAM: パケットの形式を示す定数。

失敗モードと境界条件

  • GETBYTE が失敗した場合の挙動は PullFilter の実装に依存します。
  • 長さが MAX_CHUNK を超える場合、または負の値になる場合は PXE_PGP_CORRUPT_DATA を返します。

保守担当者向けメモ

  • 本関数は RFC 4880 のパケット長エンコーディング規則に基づいています。
  • px_debug によるエラー出力は、タイミング攻撃対策として詳細な理由を隠蔽する方針に従っています。
