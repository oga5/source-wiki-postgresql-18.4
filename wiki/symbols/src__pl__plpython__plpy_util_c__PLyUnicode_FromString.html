PLyUnicode_FromString

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_util.c:115

目的

PostgreSQLのサーバーエンコーディングで表現されたC文字列を、PythonのUnicodeオブジェクトへ変換します。

シグネチャまたは形状

PyObject *PLyUnicode_FromString(const char *s)

入力と出力

  • **入力**: s (ヌル終端されたC文字列)
  • **出力**: 変換されたPython Unicodeオブジェクト（PyObject*）。失敗時はPython例外が設定され、NULLが返される可能性があります。

前提条件と不変条件

  • 入力文字列 s は、現在のPostgreSQLサーバーエンコーディングに従っている必要があります。
  • 内部的に strlen(s) を使用するため、s は有効なヌル終端文字列である必要があります。

アルゴリズムまたは振る舞い

1. 入力文字列 s の長さを strlen(s) で算出します。 2. PLyUnicode_FromStringAndSize(s, strlen(s)) を呼び出し、実際の変換処理（サーバーエンコーディングからUTF-8への変換およびPythonオブジェクトの生成）を委譲します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_elog.c: PLy_elog 関連処理
  • src/pl/plpython/plpy_exec.c: トリガーデータや識別子のPythonオブジェクト変換
  • **呼び出し先**:
  • PLyUnicode_FromStringAndSize (同一ファイル内)

関連データ構造

  • PyObject: Pythonの汎用オブジェクト型。

失敗モードと境界条件

  • 文字列のエンコーディング変換中にエラーが発生した場合、PostgreSQL側のエラー報告メカニズム（PLy_elog）が呼び出されるか、Pythonの例外が設定されます。
  • メモリ不足時にはPythonの例外が発生し、関数は NULL を返します。

保守担当者向けメモ

  • 本関数は PLyUnicode_FromStringAndSize のラッパーです。変換ロジックの変更は PLyUnicode_FromStringAndSize 側で行ってください。
  • Pythonオブジェクトの参照カウント管理は、呼び出し先である PLyUnicode_FromStringAndSize 内の PG_TRY/PG_CATCH ブロックによって保護されています。
