do_swap

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/relfilenumber.c:452

目的

pg_upgrade の --swap モードにおいて、旧クラスタと新クラスタ間のディレクトリ構造の入れ替えおよびカタログファイルの差し替えを統括します。

シグネチャまたは形状

static void do_swap(FileNameMap *maps, int size, char *old_tablespace)

入力と出力

  • **入力**:
  • maps: 移行対象のファイル対応表（FileNameMap 配列）。
  • size: maps 配列の要素数。
  • old_tablespace: 特定のテーブルスペースのみを対象とする場合のパス（指定がない場合は NULL）。
  • **出力**: なし（副作用としてファイルシステム上のディレクトリ移動およびカタログファイルの差し替えを実行）。

前提条件と不変条件

  • maps は FileNameMapCmp 関数を用いて relfilenumber 順にソートされる必要があります（本関数内で実行）。
  • prepare_for_swap が成功した場合にのみ、swap_catalog_files が呼び出されます。

アルゴリズムまたは振る舞い

1. qsort を使用して maps を relfilenumber 順に並べ替えます。 2. old_tablespace が指定されている場合、そのテーブルスペースに対して prepare_for_swap を実行し、成功すれば swap_catalog_files を呼び出します。 3. old_tablespace が指定されていない場合、まず old_cluster.pgdata に対して同様の処理を行い、続いて os_info.old_tablespaces に含まれるすべてのテーブルスペースに対して順次処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の記述から、pg_upgrade のメイン処理フローの一部として呼び出されます。
  • **呼び出し先**:
  • qsort (標準ライブラリ)
  • FileNameMapCmp
  • prepare_for_swap
  • swap_catalog_files

関連データ構造

  • FileNameMap: 旧ファイルと新ファイルの対応を保持する構造体。

失敗モードと境界条件

  • prepare_for_swap が失敗（戻り値が偽）した場合、その対象に対する swap_catalog_files はスキップされます。
  • maps が空または size が不適切な場合、qsort や配列アクセスで予期せぬ動作をする可能性があります。

保守担当者向けメモ

  • 本関数は --swap モード専用のコードパスです。通常のファイル転送処理とは独立しているため、変更時は prepare_for_swap および swap_catalog_files との整合性に注意してください。
  • maps のソート順序は swap_catalog_files の処理において重要であるため、qsort の呼び出しは必須です。
