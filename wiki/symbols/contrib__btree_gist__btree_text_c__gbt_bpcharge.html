gbt_bpcharge

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_text.c:106

目的

bpchar（空白埋め固定長文字）型に対する「以上（greater than or equal to）」の比較を、GiSTインデックスのサポート関数として提供します。

シグネチャまたは形状

static bool gbt_bpcharge(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象のデータへのポインタ。
  • collation: 照合順序（Collation）のOID。
  • flinfo: 関数呼び出し情報（使用されていません）。
  • **出力**:
  • bool: a >= b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は bpchar 型の有効なデータポインタである必要があります。
  • 内部で bpcharge 関数を呼び出すため、PostgreSQLの標準的な bpchar 比較ロジックに依存します。

アルゴリズムまたは振る舞い

1. PointerGetDatum を使用して、入力されたポインタ a と b を Datum 型に変換します。 2. DirectFunctionCall2Coll を介して、PostgreSQLの標準関数 bpcharge を呼び出します。 3. bpcharge の戻り値を DatumGetBool で論理値に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し先**: bpcharge（PostgreSQL標準の比較関数）。
  • **呼び出し元**: ソースコード内の静的配列（bptinfo 構造体の初期化箇所など）から関数ポインタとして参照されています。直接的な関数呼び出しの詳細は提供されていません。

関連データ構造

  • gbtree_vinfo: contrib/btree_gist/btree_text.c 内で定義されている、bpchar 型の比較関数群を保持する構造体で使用されます。

失敗モードと境界条件

  • a または b が NULL の場合、DirectFunctionCall2Coll の挙動に依存します。
  • 照合順序（collation）が不適切な場合、bpcharge の実行時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は bpchar 型専用のラッパーです。ロジックの変更が必要な場合は、src/backend/utils/adt/varlena.c にある基底の bpcharge 関数を確認してください。
  • gbt_textge と同様のパターンで実装されています。
