TmFromChar

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/formatting.c:426

目的

文字列から日時データ型へ変換する際（TO_DATE, TO_TIMESTAMP 等）の解析結果を一時的に保持するための構造体です。

シグネチャまたは形状

typedef struct {
    FromCharDateMode mode;
    int hh, pm, mi, ss, ssss, d, dd, ddd, mm, ms, year, bc, ww, w, cc, j, us, yysz, clock, tzsign, tzh, tzm, ff;
    bool has_tz;
    int gmtoffset;
    pg_tz *tzp;
    char *abbrev;
} TmFromChar;

入力と出力

  • **入力**: DCH_from_char() 関数において、フォーマット文字列の解析結果を格納する出力先として使用されます。
  • **出力**: 解析された各日時フィールド（年、月、日、時、分、秒、タイムゾーン等）がメンバ変数に格納されます。

前提条件と不変条件

  • ZERO_tmfc(_X) マクロを使用して、使用前に memset でゼロ初期化することが推奨されます。
  • d フィールドは 1-7 の値（1=日曜日）をとり、0 は未設定を意味します。

アルゴリズムまたは振る舞い

1. DCH_from_char() が入力文字列を走査し、フォーマット指定子に対応する値を本構造体の各メンバに書き込みます。 2. from_char_set_mode() を通じて、解析モード（FromCharDateMode）の整合性が検証されます。 3. 最終的に、本構造体に格納された値が解析され、PostgreSQL 内部の日時表現へ変換されます。

呼び出し元と呼び出し先

  • **呼び出し元**: DCH_from_char() がこの構造体を引数として受け取り、値を設定します。また、TO_TIMESTAMP 系の関数実装内でローカル変数として宣言されます。
  • **呼び出し先**: from_char_set_mode() がこの構造体を参照し、モード設定を行います。

関連データ構造

  • FromCharDateMode: 解析モードを定義する列挙型。
  • FormatNode: フォーマット文字列の解析ツリー。
  • pg_tz: タイムゾーン情報。

失敗モードと境界条件

  • タイムゾーンフィールド（TZH, TZM）が存在しない場合、tzsign は 0 となります。
  • フォーマット文字列に該当する情報が含まれない場合、各メンバは初期値（0）のまま保持されます。

保守担当者向けメモ

  • 新しい日時フォーマット指定子を追加する場合、本構造体にフィールドを追加し、DCH_from_char 内での代入処理を更新する必要があります。
  • abbrev や tzp は動的なメモリ管理を伴う可能性があるため、ライフサイクルに注意してください。
