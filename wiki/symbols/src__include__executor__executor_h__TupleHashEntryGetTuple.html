TupleHashEntryGetTuple

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/executor/executor.h:173

目的

ハッシュテーブルエントリから、格納されている MinimalTuple を取得する。

シグネチャまたは形状

static inline MinimalTuple TupleHashEntryGetTuple(TupleHashEntry entry)

入力と出力

  • **入力**: TupleHashEntry entry（ハッシュテーブル内のエントリへのポインタ）
  • **出力**: MinimalTuple（エントリに関連付けられたタプル）

前提条件と不変条件

  • 引数 entry は有効な TupleHashEntry 構造体を指している必要がある。
  • entry->firstTuple が初期化済みであること。

アルゴリズムまたは振る舞い

entry 構造体が保持する firstTuple メンバを直接返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAgg.c
  • src/backend/executor/nodeSetOp.c
  • src/backend/executor/nodeSubplan.c
  • **呼び出し先**: なし（インライン関数として直接メンバを参照）

関連データ構造

  • TupleHashEntry: ハッシュテーブル内のエントリを表現する構造体。
  • MinimalTuple: PostgreSQLの実行エンジン内で使用される、最小限のオーバーヘッドを持つタプル形式。

失敗モードと境界条件

  • entry が NULL の場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数はパフォーマンス向上のために static inline として定義されている。
  • 構造体 TupleHashEntry の定義変更に伴い、メンバ名 firstTuple が変更された場合は本関数の修正が必要となる。
