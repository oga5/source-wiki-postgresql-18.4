ExecEndLimit

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeLimit.c:533

目的

Limit 実行ノードおよびその配下のサブプランノードを終了（シャットダウン）させ、リソースを解放する。

シグネチャまたは形状

void ExecEndLimit(LimitState *node)

入力と出力

  • **入力**: node (LimitState *) - 終了対象となる Limit ノードの状態構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • node は有効な LimitState 構造体を指している必要がある。
  • 内部で outerPlanState(node) を介してサブプランの状態を取得するため、サブプランが初期化済みであること。

アルゴリズムまたは振る舞い

1. outerPlanState(node) を呼び出し、Limit ノードの直下にあるサブプランの実行状態を取得する。 2. ExecEndNode を呼び出し、取得したサブプランの終了処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:750 からの呼び出しが確認されている。
  • **呼び出し先**: ExecEndNode（サブプランの終了処理）。

関連データ構造

  • LimitState: Limit ノードの実行状態を保持する構造体。src/include/nodes/execnodes.h に定義。

失敗モードと境界条件

  • node が NULL の場合、outerPlanState マクロの挙動に依存する（通常はセグメンテーション違反を引き起こす可能性がある）。

保守担当者向けメモ

  • 本関数は Limit ノード固有の終了処理を担うが、現状ではサブプランの終了処理を委譲するのみである。
  • WITH TIES 等で追加のメモリリソースを確保している場合、将来的にこの関数内でそれらの解放処理が必要になる可能性がある。
