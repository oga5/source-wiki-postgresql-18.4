UnicodeNormalizationQC

定義

  • 種別: enum
  • 定義箇所: src/include/common/unicode_norm.h:28

目的

Unicode 文字列の正規化状態を判定する「クイックチェック（Quick Check）」アルゴリズムの結果を表現するために使用されます。

シグネチャまたは形状

typedef enum
{
	UNICODE_NORM_QC_NO = 0,
	UNICODE_NORM_QC_YES = 1,
	UNICODE_NORM_QC_MAYBE = -1,
} UnicodeNormalizationQC;

入力と出力

  • 本シンボル自体は列挙型であり、関数等の戻り値として正規化判定結果を返します。
  • YES (1): 文字列が指定された正規化形式に適合している。
  • NO (0): 文字列が指定された正規化形式に適合していない。
  • MAYBE (-1): 文字列が正規化されているか判断できず、詳細な正規化処理が必要である。

前提条件と不変条件

  • src/include/common/unicode_normprops_table.h 内で、ビットフィールド（signed int quickcheck:4）として本型が参照される場合があります。

アルゴリズムまたは振る舞い

  • 文字列全体を走査する前に、正規化プロパティテーブルを参照して正規化の必要性を高速に判定する際に利用されます。
  • MAYBE が返された場合、より詳細な正規化ロジック（src/common/unicode_norm.c 内の分解・結合処理）へ移行するトリガーとなります。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/varlena.c および src/common/unicode_norm.c 内の各関数。
  • 呼び出し先: 本シンボルは型定義であるため、直接の呼び出し先はありません。

関連データ構造

  • UnicodeNormalizationForm: 正規化形式を指定する列挙型。
  • unicode_is_normalized_quickcheck(): 本型を戻り値として使用する関数。

失敗モードと境界条件

  • MAYBE の判定は、正規化の可否を即座に確定できないケースを指します。これはエラーではなく、アルゴリズム上の最適化による中間状態です。

保守担当者向けメモ

  • 本型はフロントエンドとバックエンドの両方で利用されます。変更時は src/common/unicode_norm.c および関連するテーブル定義との整合性に注意してください。
  • MAYBE の値が -1 であるため、ビットフィールド等で扱う際は符号付き整数として定義されていることを確認してください。
