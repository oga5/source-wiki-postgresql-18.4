gbt_cashkey_cmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_cash.c:55

目的

GiST インデックス内における money 型の範囲キー（cashKEY）同士を比較し、ソート順序を決定する。

シグネチャまたは形状

int gbt_cashkey_cmp(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の Nsrt 構造体へのポインタ。内部の t メンバが cashKEY としてキャストされる。
  • flinfo: 関数呼び出し情報（本関数内では未使用）。
  • **出力**:
  • 0: 両方のキーが等しい場合。
  • 1: a が b より大きい場合。
  • -1: a が b より小さい場合。

前提条件と不変条件

  • 入力 a および b は Nsrt 構造体として適切に初期化されている必要がある。
  • cashKEY 構造体は lower および upper メンバを持つ。

アルゴリズムまたは振る舞い

1. a と b を Nsrt 構造体経由で cashKEY 型にキャストする。 2. まず lower 値を比較する。

  • ia->lower > ib->lower であれば 1 を返す。
  • ia->lower < ib->lower であれば -1 を返す。

    • 3. lower 値が等しい場合、upper 値を比較する。

  • ia->upper > ib->upper であれば 1 を返す。
  • ia->upper < ib->upper であれば -1 を返す。

    • 4. 両方の値が等しければ 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_cash.c 内のソート処理に関連する箇所（ソースコード上の直接的な参照として contrib/btree_gist/btree_cash.c:89 に記述あり）。
  • **呼び出し先**: 本関数内での明示的な関数呼び出しはない。

関連データ構造

  • cashKEY: money 型の範囲（lower, upper）を保持する構造体。
  • Nsrt: ソート処理のためにキーを保持するラッパー構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合の挙動は定義されていない。
  • money 型の内部表現である 64bit 整数値の範囲外の入力に対する動作は、標準的な整数比較に依存する。

保守担当者向けメモ

  • 本関数は money 型の GiST インデックス構築時のソート順を定義する。
  • 比較ロジックは lower を優先し、同値の場合に upper を比較する辞書順に近い順序を採用している。
