pq_sendfloat8

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqformat.c:275

目的

float8 型の値をバイナリ形式に変換し、StringInfo バッファへ追加します。

シグネチャまたは形状

void pq_sendfloat8(StringInfo buf, float8 f)

入力と出力

  • buf (入力/出力): データを追加する対象の StringInfo バッファ。
  • f (入力): 送信する float8 型の値。
  • 戻り値: なし。

前提条件と不変条件

  • buf は初期化済みである必要があります。
  • 内部で pq_sendint64 を呼び出すため、StringInfo のバッファ容量が不足した場合はメモリ拡張が行われます。

アルゴリズムまたは振る舞い

1. float8 型の値を int64 型として解釈するための共用体（union）を使用します。 2. 入力値 f を共用体に代入します。 3. 共用体内の int64 値を pq_sendint64 に渡し、ネットワークバイトオーダー（ビッグエンディアン）に変換した上でバッファへ書き込みます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/cube/cube.c
  • src/backend/utils/adt/float.c
  • src/backend/utils/adt/geo_ops.c
  • 呼び出し先:
  • pq_sendint64 (内部で呼び出し)

関連データ構造

  • StringInfo: メッセージ構築に使用される動的バッファ構造体。
  • float8: 8バイト浮動小数点数。

失敗モードと境界条件

  • StringInfo のメモリ確保に失敗した場合、palloc 経由でエラーが発生します。
  • pq_sendint64 の仕様に基づき、バッファへの書き込み時にエラーが発生した場合は処理が中断されます。

保守担当者向けメモ

  • 本関数は float8 をビットパターンを維持したまま int64 として送信します。プラットフォーム間での浮動小数点表現の互換性は、このビットパターンの転送に依存します。
