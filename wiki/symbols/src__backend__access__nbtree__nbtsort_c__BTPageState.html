BTPageState

定義

  • 種別: struct
  • 定義箇所: src/backend/access/nbtree/nbtsort.c:231

目的

B-Treeインデックスのバルクロード（構築）プロセスにおいて、現在構築中の各ツリーレベルの状態を保持するために使用されます。

シグネチャまたは形状

typedef struct BTPageState
{
    BulkWriteBuffer btps_buf;    /* ページ構築用ワークスペース */
    BlockNumber btps_blkno;      /* ページ書き込み先のブロック番号 */
    IndexTuple  btps_lowkey;     /* ページの厳密な下限ピボットタプル */
    OffsetNumber btps_lastoff;   /* 最後にロードされたアイテムのオフセット */
    Size        btps_lastextra;  /* 最後のアイテムの余分なポスティングリスト領域 */
    uint32      btps_level;      /* ツリーレベル（0 = リーフ） */
    Size        btps_full;       /* この値未満の空き容量で「フル」と判定 */
    struct BTPageState *btps_next; /* 親レベルへのリンク（存在する場合） */
} BTPageState;

入力と出力

  • 入力: _bt_pagestate 関数によって初期化され、_bt_buildadd を通じてインデックス構築中のタプル情報が更新されます。
  • 出力: _bt_uppershutdown を通じて、構築されたページが物理的なインデックスファイルへ書き出されます。

前提条件と不変条件

  • btps_level が 0 の場合はリーフページを指します。
  • btps_next は、現在のレベルが溢れた際に上位レベルのページを構築するための再帰的なリンク構造を保持します。
  • btps_full は、フィルファクタに基づいたページ充填の閾値として機能します。

アルゴリズムまたは振る舞い

1. **初期化**: _bt_pagestate により、指定されたレベルの BTPageState が割り当てられます。 2. **構築**: _bt_buildadd がソート済みタプルを順次受け取り、btps_buf を使用してページを充填します。 3. **ページ分割（上位構築）**: ページが btps_full に達すると、親レベルの BTPageState に対して再帰的にタプルが追加され、ツリーが上方向に成長します。 4. **終了**: _bt_uppershutdown が呼び出されると、残りのページがフラッシュされ、ツリーの構築が完了します。

呼び出し元と呼び出し先

  • **呼び出し元**: _bt_load（メインループ）、_bt_buildadd、_bt_uppershutdown、_bt_sort_dedup_finish_pending。
  • **呼び出し先**: _bt_pagestate（コンストラクタ）。

関連データ構造

  • BTWriteState: 構築全体の書き込み状態を管理する構造体。
  • BulkWriteBuffer: ページ構築用のバッファ領域。
  • IndexTuple: インデックスに格納される個々のデータ行。

失敗モードと境界条件

  • メモリ不足: palloc0 による構造体確保時に発生する可能性があります。
  • ページ溢れ: btps_full を超えるデータが追加された場合、親レベルへのリンク（btps_next）が適切に処理されないとツリー構造が破壊されます。

保守担当者向けメモ

  • 本構造体はバルクロード専用です。通常の挿入処理（nbtinsert.c）とは異なるページ管理ロジックである点に注意してください。
  • btps_next を介した再帰的なページ構築は、インデックスの高さが動的に決定される仕組みの核心です。
