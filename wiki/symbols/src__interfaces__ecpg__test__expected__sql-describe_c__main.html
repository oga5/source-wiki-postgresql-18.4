目的

ECPG（Embedded SQL for C）における DESCRIBE 文の動作を検証する回帰テスト用プログラムのエントリポイント。SQL記述子（Descriptor）および SQLDA（SQL Descriptor Area）を用いたメタデータ取得機能の整合性を検証する。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム内で定義された SQL 文を使用）
  • 出力: 標準出力へのメタデータ情報、および終了コード（成功時 0、失敗時 1）

前提条件と不変条件

  • データベース ecpg1_regression への接続が成功していること。
  • テスト用テーブル descr_t2 が作成され、データが挿入されていること。
  • 各 SQL 操作後に sqlca.sqlcode を確認し、エラー発生時には exit(1) で終了する。

アルゴリズムまたは振る舞い

1. **初期化**: データベースへ接続し、descr_t2 テーブルの作成とデータの挿入を行う。 2. **準備**: DESCRIBE 対象となる SQL 文を PREPARE する。 3. **記述子準備**: ECPGallocate_desc を使用して desc1, desc2 を確保する。 4. **メタデータ取得**:

  • ECPGdescribe を呼び出し、SQL 記述子（desc1, desc2）および SQLDA（sqlda1, sqlda2, sqlda3）にメタデータをロードする。

    • 5. **検証**: ECPGget_desc_header を使用して取得した列数と、SQLDA 内の情報を比較し、整合性を確認する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先: ECPGconnect, ECPGdo, ECPGtrans, ECPGallocate_desc, ECPGprepare, ECPGdescribe, ECPGget_desc_header 等の ECPG ライブラリ関数。

関連データ構造

  • sqlda_t: SQLDA 構造体（列情報を保持）。
  • sqlca_t: SQLCA 構造体（エラーハンドリングに使用）。

失敗モードと境界条件

  • データベース接続失敗、テーブル作成失敗、または SQL 実行エラー時に sqlca.sqlcode < 0 となり、exit(1) で異常終了する。
  • sqlda が NULL の場合、エラーメッセージを出力して終了する。
