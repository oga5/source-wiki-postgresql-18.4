set_val_in_shmem

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_dsm_registry/test_dsm_registry.c:51

目的

共有メモリ上に配置された TestDSMRegistryStruct の値を、SQLインターフェースを通じて更新する。

シグネチャまたは形状

Datum set_val_in_shmem(PG_FUNCTION_ARGS)

入力と出力

  • 入力: SQL関数引数として int32 型の値を受け取る（PG_GETARG_INT32(0)）。
  • 出力: なし（PG_RETURN_VOID()）。

前提条件と不変条件

  • 実行前に tdr_attach_shmem() が呼び出され、共有メモリセグメントへのアクセス権と tdr_state ポインタが確立されている必要がある。
  • tdr_state が指す構造体内の lck（LWLock）が初期化済みであること。

アルゴリズムまたは振る舞い

1. tdr_attach_shmem() を呼び出し、共有メモリへのアタッチを行う。 2. tdr_state->lck に対して LW_EXCLUSIVE モードで排他ロックを取得する。 3. 引数として渡された値を tdr_state->val に書き込む。 4. tdr_state->lck を解放する。 5. 処理を終了し、voidを返す。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLの関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • 呼び出し先:
  • tdr_attach_shmem()
  • LWLockAcquire()
  • LWLockRelease()
  • PG_GETARG_INT32()
  • PG_RETURN_VOID()

関連データ構造

  • TestDSMRegistryStruct: 共有メモリ上に配置されるデータ構造。val（値）と lck（LWLock）を保持する。

失敗モードと境界条件

  • 共有メモリへのアタッチ失敗時、tdr_attach_shmem 内部でエラーが発生する可能性がある。
  • ロック取得時にデッドロックが発生する可能性がある（本関数単体では排他ロックのみを取得する）。

保守担当者向けメモ

  • 本関数はテスト用モジュールの一部であり、共有メモリ操作の最小構成例として実装されている。
  • tdr_state は静的ポインタとして定義されており、本モジュール内の関数間で共有される。
