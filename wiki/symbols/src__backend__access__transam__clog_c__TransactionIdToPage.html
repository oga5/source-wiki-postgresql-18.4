TransactionIdToPage

定義

  • 種別: static inline function
  • 定義箇所: src/backend/access/transam/clog.c:81

目的

指定されたトランザクションID（XID）が格納されている CLOG（Commit Log）のページ番号を算出する。

シグネチャまたは形状

int64 TransactionIdToPage(TransactionId xid)

入力と出力

  • **入力**: xid (TransactionId) - 対象のトランザクションID。
  • **出力**: int64 - CLOG 内のページ番号。

前提条件と不変条件

  • CLOG_XACTS_PER_PAGE が定義されている環境で動作する。
  • 入力される xid は有効なトランザクションIDであることを想定している。

アルゴリズムまたは振る舞い

xid を定数 CLOG_XACTS_PER_PAGE で除算し、その商をページ番号として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/clog.c 内の複数の関数（TransactionIdSetTreeStatus, TransactionIdGetStatus, TransactionIdSetStatusBit 等）から、対象 XID のページ位置を特定するために頻繁に呼び出されている。
  • **呼び出し先**:
  • なし（算術演算のみを実行するインライン関数）。

関連データ構造

  • CLOG_XACTS_PER_PAGE: 1ページあたりのトランザクション数を定義する定数。
  • TransactionId: トランザクションを一意に識別する型。

失敗モードと境界条件

  • xid が 0（InvalidTransactionId）や特殊な予約済み XID の場合、計算結果は 0 ページ目となる。
  • 除数である CLOG_XACTS_PER_PAGE が 0 の場合はゼロ除算が発生する（通常、PostgreSQL のビルド設定により防止されている）。

保守担当者向けメモ

  • 本関数は非常に頻繁に呼び出されるため、static inline として定義されている。変更を加える際は、パフォーマンスへの影響を考慮すること。
  • ページ番号の計算ロジックは clog.c 全体で一貫している必要があるため、この関数を介さずに直接計算を行うコードを混在させないこと。
