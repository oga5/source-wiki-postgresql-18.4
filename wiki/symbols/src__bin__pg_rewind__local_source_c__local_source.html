local_source

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_rewind/local_source.c:19

目的

pg_rewind において、ローカルファイルシステム上のデータディレクトリをソース（同期元）として扱うための状態を保持する構造体です。

シグネチャまたは形状

typedef struct
{
	rewind_source common;		/* 共通インターフェース関数 */
	const char *datadir;		/* ソースデータディレクトリへのパス */
} local_source;

入力と出力

本構造体はデータ保持用であり、直接的な入出力関数ではありません。init_local_source によって初期化され、datadir フィールドにソースディレクトリのパスが格納されます。

前提条件と不変条件

  • common フィールドは rewind_source インターフェースを実装している必要があります。
  • datadir は有効なローカルファイルシステム上のディレクトリパスを指している必要があります。

アルゴリズムまたは振る舞い

本構造体は、pg_rewind のソース抽象化レイヤーの一部として機能します。local_source.c 内の各関数（local_traverse_files や local_queue_fetch_range 等）は、rewind_source * 型の引数を本構造体へキャストし、datadir を参照してファイル操作を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: init_local_source が本構造体のメモリを確保（pg_malloc0）し、初期化します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出す主体ではありません。各操作関数が本構造体の datadir を読み取ります。

関連データ構造

  • rewind_source: 本構造体が継承（包含）する抽象インターフェース。

失敗モードと境界条件

  • datadir が無効なパスである場合、後続のファイル操作関数（slurpFile や traverse_datadir）が失敗します。
  • メモリ確保失敗時は pg_malloc0 がエラーを発生させます。

保守担当者向けメモ

  • 本構造体は rewind_source の具現化であるため、インターフェースの変更時には rewind_source.h との整合性を確認してください。
  • local_source へのキャストは、source 引数が実際に local_source 型であることを前提としています。
