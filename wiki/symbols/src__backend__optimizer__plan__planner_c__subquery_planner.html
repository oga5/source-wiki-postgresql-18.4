subquery_planner

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/optimizer/plan/planner.c:668

目的

サブクエリ単位のプランニングにおける前処理を統括し、Query ツリーを最適化可能な状態へ正規化する。

シグネチャまたは形状

PlannerInfo *subquery_planner(PlannerGlobal *glob, Query *parse, PlannerInfo *parent_root,
                              bool hasRecursion, double tuple_fraction,
                              SetOperationStmt *setops)

入力と出力

  • **入力**:
  • glob: サブクエリ間で共有されるグローバル状態。
  • parse: プランニング対象の Query ツリー。
  • parent_root: 親クエリの PlannerInfo（存在する場合）。
  • hasRecursion: 再帰的クエリ（CTE）かどうかのフラグ。
  • tuple_fraction: 推定されるタプル取得率。
  • setops: 集合演算（UNION等）の定義。
  • **出力**:
  • 構築された PlannerInfo 構造体へのポインタ。

前提条件と不変条件

  • PlannerInfo 構造体が初期化され、query_level が親クエリに基づいて適切に設定されること。
  • join_domains が初期化されていること。

アルゴリズムまたは振る舞い

1. **初期化**: PlannerInfo を作成し、クエリレベルやパラメータ、メモリコンテキストを初期化する。 2. **CTE処理**: SS_process_ctes を呼び出し、WITH 句をサブクエリまたは initplan に変換する。 3. **MERGE変換**: transform_MERGE_to_join により MERGE 文を結合形式に変換する。 4. **正規化**:

  • replace_empty_jointree: 空の FROM 句をダミーの RTE_RESULT に置換。
  • pull_up_sublinks: ANY/EXISTS サブリンクを結合へ変換。
  • preprocess_function_rtes: 関数 RTE の定数畳み込みとインライン化。
  • expand_virtual_generated_columns: 生成列の式を展開。
  • pull_up_subqueries: サブクエリの引き上げ。
  • flatten_simple_union_all: UNION ALL のフラット化。

    • 5. **範囲テーブル調査**: rtable を走査し、結合の種類や継承関係、Lateral 参照の有無を判定して PlannerInfo にフラグを立てる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • planner() (同ファイル内)
  • src/backend/optimizer/path/allpaths.c
  • src/backend/optimizer/plan/planner.c (再帰呼び出し)
  • **呼び出し先**:
  • SS_process_ctes, transform_MERGE_to_join, replace_empty_jointree, pull_up_sublinks, preprocess_function_rtes, expand_virtual_generated_columns, pull_up_subqueries, flatten_simple_union_all 等。

関連データ構造

  • PlannerInfo: プランニング中のクエリ状態を保持する主要構造体。
  • PlannerGlobal: クエリ全体で共有される情報。
  • Query: 解析済みのクエリツリー。
  • RangeTblEntry: 範囲テーブルの各エントリ。

失敗モードと境界条件

  • 継承関係の判定において、かつて子テーブルが存在したが現在は存在しない場合に has_subclass が偽陽性を返す可能性がある（ソースコード内のコメントによる）。

保守担当者向けメモ

  • 本関数はプランニングの「前処理」を担当する。実際のパス生成やプラン生成は、この後に続くフェーズで行われる。
  • rtable の走査は、すべての RTE 追加が完了した後に実行する必要がある。
