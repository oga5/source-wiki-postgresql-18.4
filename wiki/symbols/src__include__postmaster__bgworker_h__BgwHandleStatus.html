BgwHandleStatus

定義

  • 種別: enum
  • 定義箇所: src/include/postmaster/bgworker.h:103

目的

バックグラウンドワーカー（BgWorker）の現在の実行状態を追跡・表現するために使用されます。

シグネチャまたは形状

typedef enum BgwHandleStatus
{
    BGWH_STARTED,           /* ワーカーは実行中 */
    BGWH_NOT_YET_STARTED,   /* ワーカーは未起動 */
    BGWH_STOPPED,           /* ワーカーは終了済み */
    BGWH_POSTMASTER_DIED,   /* Postmasterが終了し、状態が不明 */
} BgwHandleStatus;

入力と出力

本シンボル自体は列挙型であり、関数ではありません。主に BackgroundWorkerHandle を介してワーカーの状態を問い合わせる際の戻り値として利用されます。

前提条件と不変条件

  • BackgroundWorkerHandle を通じて取得される状態値です。
  • Postmasterが生存している限り、状態は BGWH_STARTED、BGWH_NOT_YET_STARTED、BGWH_STOPPED のいずれかに遷移します。

アルゴリズムまたは振る舞い

  • BGWH_STARTED: ワーカープロセスが正常に起動し、実行中であることを示します。
  • BGWH_NOT_YET_STARTED: Postmasterがワーカーをまだ起動していない状態を示します。
  • BGWH_STOPPED: ワーカープロセスが終了したことを示します。
  • BGWH_POSTMASTER_DIED: 親プロセスであるPostmasterが異常終了したため、ワーカーの状態を正確に判定できない状態を示します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/bgworker.c、src/backend/access/transam/parallel.c、src/backend/storage/ipc/shm_mq.c、contrib/pg_prewarm/autoprewarm.c 等の各モジュールが、ワーカーの状態監視のためにこの型を参照しています。
  • **呼び出し先**: 本シンボルは列挙型であり、直接的な呼び出し先はありません。

関連データ構造

  • BackgroundWorkerHandle: この列挙型を保持し、ワーカーのライフサイクル管理に使用される構造体。

失敗モードと境界条件

  • BGWH_POSTMASTER_DIED は、Postmasterのクラッシュというシステム全体の異常系を示します。この状態になった場合、ワーカーの生存確認は不可能となります。

保守担当者向けメモ

  • 新しい状態を追加する場合は、BackgroundWorkerHandle を利用するすべての箇所で switch 文の網羅性が維持されているか確認してください。
