PortalCreateHoldStore

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/portalmem.c:330

目的

WITH HOLD カーソルがトランザクション終了後もデータを保持できるように、専用のメモリコンテキストとタプルストア（tuplestore）を初期化する。

シグネチャまたは形状

void PortalCreateHoldStore(Portal portal)

入力と出力

  • **入力**: portal (対象となるポータル構造体へのポインタ)
  • **出力**: なし（副作用として portal 構造体のメンバを更新）

前提条件と不変条件

  • portal->holdContext が NULL であること。
  • portal->holdStore が NULL であること。
  • portal->holdSnapshot が NULL であること。

アルゴリズムまたは振る舞い

1. TopPortalContext を親として、PortalHoldContext という名称の新しいメモリコンテキストを作成し、portal->holdContext に割り当てる。このコンテキストはポータル自身の portalContext の子ではない。 2. portal->holdContext にメモリコンテキストを切り替える。 3. tuplestore_begin_heap を呼び出し、タプルストアを初期化して portal->holdStore に格納する。

  • カーソルのオプション（CURSOR_OPT_SCROLL）に基づき、ランダムアクセス（スクロール）の可否を決定する。
  • トランザクションを跨ぐ一時ファイルの使用を有効化する。

    • 4. 元のメモリコンテキストへ復帰する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/pquery.c:1001
  • src/backend/utils/mmgr/portalmem.c:642
  • **呼び出し先**:
  • AllocSetContextCreate
  • MemoryContextSwitchTo
  • tuplestore_begin_heap

関連データ構造

  • Portal: ポータル状態を保持する構造体。
  • MemoryContext: メモリ管理コンテキスト。
  • tuplestore: 永続化対象のタプルを格納するストア。

失敗モードと境界条件

  • メモリ不足時には AllocSetContextCreate または tuplestore_begin_heap がエラーを発生させる可能性がある。
  • work_mem の設定値がタプルストアの動作に影響を与える。

保守担当者向けメモ

  • portal->holdContext は TopPortalContext の直下で作成されるため、ポータルが破棄される際に明示的な解放が必要となる点に注意すること。
  • tuplestore のサイズ制限に maintenance_work_mem を使用すべきか否かという検討事項（XXXコメント）がソースコード内に存在する。
