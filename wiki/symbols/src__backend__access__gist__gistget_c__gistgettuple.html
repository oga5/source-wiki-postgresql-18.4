gistgettuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistget.c:611

目的

GiSTインデックススキャンにおけるメインエントリポイントとして、検索条件に合致する次のタプルを取得し、呼び出し元へ返却する。

シグネチャまたは形状

bool gistgettuple(IndexScanDesc scan, ScanDirection dir)

入力と出力

  • **入力**:
  • scan: インデックススキャンの状態を保持する IndexScanDesc 構造体。
  • dir: スキャン方向（ForwardScanDirection のみサポート）。
  • **出力**:
  • bool: 次のタプルが見つかった場合は true、それ以外は false。
  • scan->xs_heaptid: ヒープタプルの TID。
  • scan->xs_recheck: 再評価が必要かどうかのフラグ。
  • scan->xs_hitup: インデックスのみのスキャン（Index-Only Scan）時に返却されるタプルデータ。

前提条件と不変条件

  • dir は ForwardScanDirection である必要がある。
  • so->qual_ok が false の場合、検索条件が満たされないため直ちに false を返す。
  • 初回呼び出し時はルートページから走査を開始する。

アルゴリズムまたは振る舞い

1. **初期化**: 初回呼び出し時、ルートページを gistScanPage に渡し、スキャン状態を初期化する。 2. **順序付きスキャン**: numberOfOrderBys > 0 の場合、getNextNearest を呼び出し、距離順にタプルを抽出する。 3. **通常スキャン**:

  • 現在のページ（so->pageData）に未処理のタプルがあれば、それを返却する。
  • ページ内のタプルを使い切った場合、getNextGISTSearchItem で次の検索対象ページを取得し、gistScanPage でページを走査して一致タプルをキューイングする。
  • スキャン中に不要と判断されたタプルがある場合、gistkillitems を呼び出して LP_DEAD フラグを管理する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gist.c 内の amgettuple インターフェース経由で呼び出される。
  • **呼び出し先**:
  • gistScanPage: インデックスページの走査とキューイング。
  • getNextNearest: 順序付きスキャンの実行。
  • getNextGISTSearchItem: 次の検索対象アイテムの取得。
  • gistkillitems: 不要なインデックスエントリのマーク付け。
  • MemoryContextReset / MemoryContextSwitchTo: メモリ管理。

関連データ構造

  • GISTScanOpaque: スキャン状態を保持する構造体。
  • IndexScanDesc: インデックススキャンの共通構造体。
  • GISTSearchItem: 検索対象のインデックスページ情報。

失敗モードと境界条件

  • ForwardScanDirection 以外が指定された場合、elog(ERROR) を発生させる。
  • 検索対象がなくなった場合、false を返す。
  • kill_prior_tuple が有効な場合、直前に返したタプルを killedItems に追加する処理を行う。

保守担当者向けメモ

  • 本関数は GiST スキャンの中心的なループを制御する。並行性制御のため、gistScanPage 内での LSN チェックや gistkillitems による安全な削除ロジックが組み込まれている点に注意すること。
  • メモリ管理は so->pageDataCxt 等のコンテキストを介して行われるため、メモリリークの防止には注意が必要である。
