is_xlogfilename

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_receivewal.c:115

目的

与えられたファイル名が WAL（Write-Ahead Log）セグメントファイルであるかを判定し、その状態（部分書き込みか否か）および圧縮アルゴリズムを特定する。

シグネチャまたは形状

static bool is_xlogfilename(const char *filename, bool *ispartial, pg_compress_algorithm *wal_compression_algorithm);

入力と出力

  • **入力**:
  • filename: 判定対象のファイル名文字列。
  • **出力**:
  • ispartial: 部分書き込みファイル（.partial 接尾辞あり）であれば true、それ以外は false を格納。
  • wal_compression_algorithm: 検出された圧縮形式（PG_COMPRESSION_NONE, PG_COMPRESSION_GZIP, PG_COMPRESSION_LZ4）を格納。
  • **戻り値**: WAL ファイルとして有効な形式であれば true、そうでなければ false。

前提条件と不変条件

  • ファイル名の先頭 XLOG_FNAME_LEN 文字分が 16 進数（0-9, A-F）で構成されている必要がある。
  • ispartial および wal_compression_algorithm は、関数が true を返す場合にのみ有効な値が書き込まれる。

アルゴリズムまたは振る舞い

1. ファイル名の長さと、先頭部分が 16 進数のみで構成されているかを確認する。 2. ファイル名の接尾辞を順次照合し、以下のいずれかに該当するかを判定する：

  • 通常ファイル: (なし)
  • 圧縮ファイル: .gz, .lz4
  • 部分書き込みファイル: .partial
  • 圧縮済み部分書き込みファイル: .gz.partial, .lz4.partial

    • 3. 該当する形式が見つかった場合、対応する ispartial フラグと圧縮アルゴリズムを設定して true を返す。いずれにも一致しない場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/pg_receivewal.c 内のディレクトリ走査処理（FindStreamingStart 等）から参照されている。
  • **呼び出し先**: 標準ライブラリ関数 strlen, strspn, strcmp を使用。

関連データ構造

  • pg_compress_algorithm: 圧縮アルゴリズムを識別する列挙型。

失敗モードと境界条件

  • ファイル名が XLOG_FNAME_LEN 未満、または 16 進数以外の文字を含む場合は false を返す。
  • 認識できない接尾辞を持つファイルは false を返す。

保守担当者向けメモ

  • 新しい圧縮アルゴリズムを追加する場合は、本関数の判定ロジックに新しい接尾辞のパターンを追加する必要がある。
  • XLOG_FNAME_LEN は access/xlog_internal.h で定義されている定数に依存している。
