xl_multixact_truncate

定義

  • 種別: struct
  • 定義箇所: src/include/access/multixact.h:83

目的

マルチトランザクション（MultiXact）のSLRU領域において、不要になった古いセグメントを切り詰める（Truncate）際に、その操作内容をWAL（Write-Ahead Log）に記録するための構造体です。

シグネチャまたは形状

typedef struct xl_multixact_truncate
{
    Oid            oldestMultiDB;
    MultiXactId    startTruncOff;
    MultiXactId    endTruncOff;
    MultiXactOffset startTruncMemb;
    MultiXactOffset endTruncMemb;
} xl_multixact_truncate;

入力と出力

  • **入力**: VACUUM 等のクリーンアップ処理において、削除対象となるマルチトランザクションIDの範囲およびメンバーオフセットの範囲が本構造体に格納されます。
  • **出力**: WALレコードとして書き出され、リカバリ時に pg_multixact のSLRU領域を切り詰めるための指示として利用されます。

前提条件と不変条件

  • startTruncOff から endTruncOff までの範囲が、マルチトランザクションのオフセットとして削除可能である必要があります。
  • startTruncMemb から endTruncMemb までの範囲が、マルチトランザクションのメンバー情報として削除可能である必要があります。

アルゴリズムまたは振る舞い

1. VACUUM 処理が不要な古いマルチトランザクションを特定します。 2. 本構造体に切り詰め範囲を設定し、WALレコードを生成します。 3. リカバリ時、このレコードを読み込むことで、システムは対応するSLRUセグメントを安全に解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/multixact.c 内の TruncateMultiXact 関数が、この構造体を使用してWALレコードを作成します。
  • **呼び出し先**: src/backend/access/rmgrdesc/mxactdesc.c がWALのデコード処理において本構造体を参照します。

関連データ構造

  • MultiXactId: マルチトランザクションを識別するID。
  • MultiXactOffset: マルチトランザクションのメンバー情報を指すオフセット。

失敗モードと境界条件

  • 範囲指定が不正な場合、SLRUの整合性が損なわれる可能性があります。
  • 呼び出し元と呼び出し先の詳細なエラーハンドリングについては、ソースコードの文脈からは明示されていません。
