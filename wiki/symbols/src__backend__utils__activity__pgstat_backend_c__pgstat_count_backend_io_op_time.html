pgstat_count_backend_io_op_time

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_backend.c:55

目的

バックエンドプロセスにおける I/O 操作の実行時間を計測し、プロセスローカルな統計バッファに加算する。

シグネチャまたは形状

void pgstat_count_backend_io_op_time(IOObject io_object, IOContext io_context, IOOp io_op, instr_time io_time)

入力と出力

  • **入力**:
  • io_object: 対象となる I/O オブジェクト（ファイル、テーブル等）。
  • io_context: I/O が発生したコンテキスト。
  • io_op: 実行された I/O 操作の種類。
  • io_time: 計測された経過時間（instr_time 型）。
  • **出力**: なし（副作用として内部状態を更新）。

前提条件と不変条件

  • track_io_timing または track_wal_io_timing が有効であること。
  • pgstat_tracks_io_op が現在のバックエンド種別に対して真を返すこと。

アルゴリズムまたは振る舞い

1. 現在のバックエンド種別が統計収集対象外であれば、即座に終了する。 2. PendingBackendStats.pending_io.pending_times の該当するインデックス（io_object, io_context, io_op）に対し、INSTR_TIME_ADD を用いて io_time を加算する。 3. backend_has_iostats フラグを true に設定する。 4. pgstat_report_fixed フラグを true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat_io.c 内のロジックから呼び出されることが確認されている。
  • **呼び出し先**: INSTR_TIME_ADD マクロ（計測時間の加算）、pgstat_tracks_backend_bktype（統計対象判定）、pgstat_tracks_io_op（操作対象判定）。

関連データ構造

  • PendingBackendStats: プロセスローカルな統計保持用構造体。
  • IOObject, IOContext, IOOp: I/O 統計を分類するための列挙型。

失敗モードと境界条件

  • 統計収集が無効なバックエンド種別から呼び出された場合、何も行わずに戻る。
  • Assert 文により、無効な操作や設定の組み合わせはデバッグビルド時に停止する。

保守担当者向けメモ

  • 本関数はメモリ割り当てを行わない設計となっており、クリティカルセクション内からの呼び出しを想定している。
  • 統計はプロセス終了時に共有メモリへフラッシュされるため、本関数単体では永続化されない。
