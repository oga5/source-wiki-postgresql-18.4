generate_base_implied_equalities

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/equivclass.c:1187

目的

等価クラス（EquivalenceClass: EC）に基づき、ベースリレーションに対する制約（制限句）を導出し、プランナの最適化情報を更新する。

シグネチャまたは形状

void generate_base_implied_equalities(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root（クエリ全体のプランナ情報）
  • **出力**: なし（副作用として root 内の ec_merging_done フラグの更新、RestrictInfo の生成、および各 RelOptInfo の eclass_indexes と has_eclass_joins の更新を行う）

前提条件と不変条件

  • 呼び出し時点で、クエリ内の等価関係の吸収が完了している必要がある。
  • root->eq_classes 内の各 EquivalenceClass は ec_merged == NULL であり、ec_broken が偽であることを前提とする。

アルゴリズムまたは振る舞い

1. root->ec_merging_done を true に設定し、EC のマージフェーズ終了を宣言する。 2. root->eq_classes を走査し、メンバー数が 2 以上の EC に対して以下を実行する：

  • 定数を含む場合は generate_base_implied_equalities_const を、含まない場合は generate_base_implied_equalities_no_const を呼び出し、制約を導出する。
  • 導出に失敗した場合は generate_base_implied_equalities_broken でリカバリを行う。

    • 3. 各 EC に含まれるベースリレーションを特定し、そのリレーションの eclass_indexes に現在の EC インデックスを追加する。 4. 複数のリレーションにまたがる EC の場合、該当するリレーションの has_eclass_joins フラグを立てる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planmain.c:205
  • src/backend/optimizer/plan/analyzejoins.c:1900（推論）
  • **呼び出し先**:
  • generate_base_implied_equalities_const
  • generate_base_implied_equalities_no_const
  • generate_base_implied_equalities_broken

関連データ構造

  • PlannerInfo: プランナの全体状態を保持。
  • EquivalenceClass: 等価な式の集合。
  • RelOptInfo: ベースリレーションの情報。

失敗モードと境界条件

  • メンバー数が 1 の EC は制約導出の対象外となる。
  • ec_broken フラグが立った場合、リカバリ処理が実行される。
  • root->group_rtindex に該当するリレーションや、外部結合（Outer Join）に関連するリレーションは、インデックス更新処理から除外される。

保守担当者向けメモ

  • 本関数実行後、get_eclass_for_sort_expr() を通じて新しい単一メンバーの EC が追加される可能性がある点に注意すること。
  • rel->eclass_indexes への追加は、後続の検索処理を高速化するための最適化である。
