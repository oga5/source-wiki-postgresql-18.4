simple_table_tuple_update

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/tableam.c:335

目的

テーブル内のタプルを更新するための簡易ラッパー関数です。複雑な引数やエラーハンドリングを抽象化し、呼び出し側がトランザクションの整合性を前提とした操作を行えるようにします。

シグネチャまたは形状

void simple_table_tuple_update(Relation rel, ItemPointer otid,
                               TupleTableSlot *slot,
                               Snapshot snapshot,
                               TU_UpdateIndexes *update_indexes)

入力と出力

  • **入力**:
  • rel: 対象のテーブルリレーション。
  • otid: 更新対象タプルの物理位置（ItemPointer）。
  • slot: 更新後のデータを含むタプルスロット。
  • snapshot: 可視性判定に使用するスナップショット。
  • update_indexes: インデックス更新に関する制御情報。
  • **出力**: なし（成功時は正常終了、失敗時は elog(ERROR) によりトランザクションを中断）。

前提条件と不変条件

  • トランザクションの整合性が保証されている環境で使用すること。
  • 内部で GetCurrentCommandId(true) を使用するため、アクティブなトランザクション内である必要があります。

アルゴリズムまたは振る舞い

1. table_tuple_update を呼び出し、現在のコマンドIDと指定されたスナップショットを用いて更新を試みます。 2. 更新結果（TM_Result）を評価します。

  • TM_Ok: 正常終了。
  • TM_SelfModified: 自トランザクションによる更新済み（エラー）。
  • TM_Updated: 他トランザクションによる同時更新（エラー）。
  • TM_Deleted: 他トランザクションによる同時削除（エラー）。
  • その他: 未定義のステータス（エラー）。

    • 3. 異常系では elog(ERROR) を発行し、処理を停止します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execReplication.c:696
  • **呼び出し先**:
  • table_tuple_update（src/backend/access/table/tableam.c 内の内部関数）
  • GetCurrentCommandId
  • elog

関連データ構造

  • Relation: テーブルのメタデータ構造。
  • TupleTableSlot: タプルデータを保持するスロット。
  • TM_Result: 更新操作の結果コード。
  • TU_UpdateIndexes: インデックス更新制御構造体。

失敗モードと境界条件

  • 同時更新や同時削除が発生した場合、elog(ERROR) を発行して処理を中断します。
  • 自身による更新が重複した場合も同様にエラーとなります。

保守担当者向けメモ

  • 本関数はエラーハンドリングを簡略化しているため、呼び出し側でエラーをキャッチしてリカバリするような用途には適していません。
  • 内部で table_tuple_update を呼び出しており、具体的なAM（Access Method）の挙動は rel->rd_tableam に依存します。
