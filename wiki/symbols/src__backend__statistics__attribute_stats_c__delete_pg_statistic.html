delete_pg_statistic

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/attribute_stats.c:872

目的

システムカタログ pg_statistic から、指定されたリレーションの特定の属性（カラム）に関連付けられた統計情報行を削除します。

シグネチャまたは形状

static bool delete_pg_statistic(Oid reloid, AttrNumber attnum, bool stainherit)

入力と出力

  • **入力**:
  • reloid (Oid): 対象テーブルのOID。
  • attnum (AttrNumber): 対象属性の番号。
  • stainherit (bool): 継承された統計情報を対象とするかどうか。
  • **出力**:
  • bool: 統計情報行が見つかり、削除に成功した場合は true、存在しなかった場合は false を返します。

前提条件と不変条件

  • pg_statistic テーブルに対する RowExclusiveLock を取得して操作を行います。
  • 操作完了後、CommandCounterIncrement() を呼び出し、変更を可視化します。

アルゴリズムまたは振る舞い

1. pg_statistic テーブルを RowExclusiveLock で開きます。 2. SearchSysCache3 を使用し、STATRELATTINH キャッシュから指定された reloid、attnum、stainherit に一致するタプルを検索します。 3. タプルが存在する場合、CatalogTupleDelete を用いてシステムカタログから削除し、キャッシュを解放します。 4. 最後にテーブルを閉じ、コマンドカウンタをインクリメントして制御を戻します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/attribute_stats.c 内の関数から呼び出されています（行986など）。
  • **呼び出し先**:
  • table_open, table_close
  • SearchSysCache3, ReleaseSysCache
  • CatalogTupleDelete
  • CommandCounterIncrement

関連データ構造

  • pg_statistic (システムカタログ)
  • HeapTuple

失敗モードと境界条件

  • 指定された条件に一致する統計情報が存在しない場合、削除は行われず false を返します。
  • システムカタログのロック取得や削除処理において、データベースの整合性制約や権限によるエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は単一属性の統計削除に特化しています。拡張統計（複数列）の削除には使用できません。
  • CommandCounterIncrement() が含まれているため、本関数を呼び出すループ処理等では、トランザクションの進行状況に注意してください。
