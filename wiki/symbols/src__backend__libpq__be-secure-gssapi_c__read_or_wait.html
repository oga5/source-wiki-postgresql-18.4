read_or_wait

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-gssapi.c:430

目的

GSSAPI通信において、指定されたバイト数（len）が受信バッファ（PqGSSRecvBuffer）に蓄積されるまで、ソケットからの読み込みを継続または待機します。

シグネチャまたは形状

static ssize_t read_or_wait(Port *port, ssize_t len);

入力と出力

  • **入力**:
  • port: 通信対象の Port 構造体。
  • len: 読み込むべき目標バイト数。
  • **出力**:
  • 成功時: 要求されたバイト数 len を返す。
  • 失敗時: -1 を返す（EOFまたは回復不能なエラー発生時）。

前提条件と不変条件

  • PqGSSRecvBuffer および PqGSSRecvLength が、GSSAPI受信状態を管理する静的変数として適切に初期化・維持されていること。
  • secure_raw_read が port からの低レベル読み込みを提供できること。

アルゴリズムまたは振る舞い

1. PqGSSRecvLength が len に達するまでループを実行します。 2. secure_raw_read を呼び出し、データをバッファに追加します。 3. エラー発生時、EWOULDBLOCK、EAGAIN、EINTR 以外であれば即座に -1 を返します。 4. 読み込み結果が 0 または負の場合、WaitLatchOrSocket を使用してソケットが読み込み可能になるまで待機します。 5. 待機後に再度読み込みを行い、それでも 0 バイトであればクライアントによる切断（EOF）と判断して -1 を返します。 6. 読み込みが完了した時点で len を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-secure-gssapi.c:567

  • **呼び出し先**:

関連データ構造

  • Port (通信状態を保持)
  • PqGSSRecvBuffer (受信データ格納用静的バッファ)
  • PqGSSRecvLength (現在のバッファ内データ長)

失敗モードと境界条件

  • **EOF**: 待機後の再試行でも読み込みが 0 バイトの場合、クライアント切断として -1 を返します。
  • **回復不能なエラー**: secure_raw_read が EWOULDBLOCK 等以外のエラーを返した場合、即座に -1 を返します。

保守担当者向けメモ

  • 本関数はブロッキング待機を伴うため、呼び出し元のコンテキストが待機可能であることを確認してください。
  • PqGSSRecvBuffer は静的変数であるため、接続ごとの排他制御や状態管理がこのモジュール内の設計に依存している点に注意してください。
