option

定義

  • 種別: struct
  • 定義箇所: src/include/getopt_long.h:17

目的

コマンドライン引数解析において、GNU 形式のロングオプション（例: --help）の仕様を定義するために使用されます。

シグネチャまたは形状

struct option
{
    const char *name;
    int         has_arg;
    int        *flag;
    int         val;
};

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。各フィールドの役割は以下の通りです。

  • name: オプション名（例: "help"）。
  • has_arg: 引数の要否（no_argument, required_argument, optional_argument のいずれか）。
  • flag: NULL でない場合、オプション検出時に val の値をこのポインタが指す先に書き込みます。
  • val: flag が NULL の場合、getopt_long がこのオプションを検出した際に返す値。

前提条件と不変条件

  • getopt_long 関数に渡す配列の末尾には、全フィールドが 0 または NULL で初期化された番兵要素が必要です。

アルゴリズムまたは振る舞い

getopt_long 関数が引数ベクトルを走査する際、この構造体の配列を参照して一致するロングオプションを特定します。一致した場合、flag が指定されていればその値を更新し、指定されていなければ val を戻り値として返します。

呼び出し元と呼び出し先

  • 呼び出し元: getopt_long を利用するアプリケーションコード（PostgreSQL の各種ユーティリティなど）。
  • 呼び出し先: 本構造体自体はデータ定義であり、関数呼び出しは行いません。
  • 補足: ツールが検出した contrib/dblink/dblink.c 等の参照は、本構造体とは異なる文脈の「option」という名称の変数やフィールドを指している可能性が高く、本構造体との直接的な関係は確認できません。

関連データ構造

  • no_argument / required_argument / optional_argument: has_arg フィールドに指定する定数。

失敗モードと境界条件

  • 配列の終端に番兵（全フィールド 0/NULL）がない場合、getopt_long の走査がメモリ範囲外へ逸脱する可能性があります。

保守担当者向けメモ

  • 本構造体は getopt_long.c で実装される解析ロジックと密結合しています。
  • 構造体の定義は HAVE_STRUCT_OPTION マクロによって制御される場合があるため、環境依存の定義状況に注意してください。
