euc_jis_2004_to_shift_jis_2004

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc2004_sjis2004/euc2004_sjis2004.c:42

目的

PostgreSQL のエンコーディング変換サブシステムにおいて、EUC_JIS_2004 から SHIFT_JIS_2004 への文字コード変換を実行するインターフェース関数です。

シグネチャまたは形状

Datum euc_jis_2004_to_shift_jis_2004(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを介して以下の引数を受け取ります。
  • src (CSTRING): 変換元の文字列バッファ。
  • dest (CSTRING): 変換後の文字列を格納するバッファ。
  • len (INT32): 変換対象のバイト長。
  • noError (BOOL): 不正なバイトシーケンス検出時にエラーを報告するか（false）スキップするか（true）を制御するフラグ。
  • **出力**: Datum 型として、変換されたバイト数を返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_EUC_JIS_2004, PG_SHIFT_JIS_2004) マクロにより、エンコーディング ID の整合性が検証されます。
  • 変換処理は euc_jis_20042shift_jis_2004 関数に委譲されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得します。 2. エンコーディング変換の引数チェックを実行します。 3. 変換ロジック本体である euc_jis_20042shift_jis_2004 を呼び出し、変換を実行します。 4. 変換されたバイト数を PG_RETURN_INT32 を介して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（fmgr）経由で SQL から呼び出されます。また、PG_FUNCTION_INFO_V1(euc_jis_2004_to_shift_jis_2004) により登録されています。
  • **呼び出し先**: euc_jis_20042shift_jis_2004（変換ロジック本体）。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQL の関数引数管理用構造体。
  • Datum: PostgreSQL の汎用データ型。

失敗モードと境界条件

  • 不正なバイトシーケンスが入力された場合、noError フラグの状態に応じて、report_invalid_encoding によるエラー発生または変換の中断が行われます。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の変換ロジックは euc_jis_20042shift_jis_2004 に実装されています。
  • 変換ロジックの修正が必要な場合は、euc_jis_20042shift_jis_2004 を参照してください。
