StrategyNotifyBgWriter

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/freelist.c:430

目的

バックグラウンドライター（bgwriter）のプロセス番号を共有メモリ上の BufferStrategyControl に設定またはクリアし、バッファ割り当て時の通知先を管理します。

シグネチャまたは形状

void StrategyNotifyBgWriter(int bgwprocno)

入力と出力

  • **入力**:
  • bgwprocno (int): 設定するバックグラウンドライターのプロセス番号。クリアする場合は -1 を指定します。
  • **出力**: なし。

前提条件と不変条件

  • StrategyControl が初期化済みであること。
  • buffer_strategy_lock を使用して、StrategyGetBuffer からの参照に対してアトミックな更新を保証します。

アルゴリズムまたは振る舞い

1. StrategyControl->buffer_strategy_lock スピンロックを取得します。 2. StrategyControl->bgwprocno に引数 bgwprocno を代入します。 3. スピンロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/bgwriter.c: バックグラウンドライターのメインループ内で、自身のプロセス番号の登録や終了時のクリアに使用されます。
  • **呼び出し先**:
  • SpinLockAcquire
  • SpinLockRelease

関連データ構造

  • BufferStrategyControl: 共有メモリ上に存在するバッファ置換戦略の制御構造体。bgwprocno フィールドを保持します。

失敗モードと境界条件

  • 本関数自体はエラーを返しません。
  • スピンロックを使用しているため、呼び出し元はロックの競合が発生する可能性を考慮する必要がありますが、bgwriter による呼び出し頻度は低いため、パフォーマンスへの影響は限定的です。

保守担当者向けメモ

  • 本関数は、StrategyGetBuffer がバッファ不足時に bgwriter を適切に通知できるようにするための補助的な役割を果たします。
  • 変更を加える際は、buffer_strategy_lock の競合が他のバッファ管理処理に与える影響を考慮してください。
