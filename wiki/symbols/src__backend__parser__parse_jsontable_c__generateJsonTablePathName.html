generateJsonTablePathName

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_jsontable.c:228

目的

JSON_TABLE の解析プロセスにおいて、一意なパス名（json_table_path_N）を生成し、解析コンテキストに登録します。

シグネチャまたは形状

static char *generateJsonTablePathName(JsonTableParseContext *cxt)

入力と出力

  • **入力**: JsonTableParseContext *cxt（解析状態を保持するコンテキスト構造体）
  • **出力**: 生成された一意なパス名文字列（char *）

前提条件と不変条件

  • 引数 cxt は有効な JsonTableParseContext を指している必要があります。
  • cxt->pathNameId は、呼び出しごとにインクリメントされ、一意な識別子を提供します。

アルゴリズムまたは振る舞い

1. namebuf を使用して "json_table_path_%d" 形式の文字列を生成します。この際、cxt->pathNameId を使用し、直後にインクリメントします。 2. pstrdup を用いて生成された文字列をメモリに複製します。 3. 複製した文字列を cxt->pathNames リストに追加します。 4. 最終的に、複製された文字列へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_jsontable.c:98
  • src/backend/parser/parse_jsontable.c:474
  • **呼び出し先**:
  • snprintf (標準ライブラリ)
  • pstrdup (PostgreSQL メモリ管理)
  • lappend (PostgreSQL リスト操作)

関連データ構造

  • JsonTableParseContext: 解析中に共有されるコンテキスト。pathNameId（カウンタ）および pathNames（生成済みパス名のリスト）を保持します。

失敗モードと境界条件

  • メモリ不足が発生した場合、pstrdup または lappend 内部でエラーが発生し、処理が中断される可能性があります。
  • namebuf のサイズ（32バイト）を超えるパス名IDが生成された場合、snprintf は切り詰めを行いますが、IDの増加により実用上の制限に達する可能性は極めて低いです。

保守担当者向けメモ

  • 本関数は JSON_TABLE の解析において、パス名の一意性を保証するために不可欠です。
  • cxt->pathNames に登録された文字列は、解析終了後に適切に解放される必要があります（JsonTableParseContext のライフサイクルに依存）。
