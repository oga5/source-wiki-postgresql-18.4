enum_le

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/enum.c:314

目的

2つの列挙型（ENUM）値が「小なりまたは等しい（<=）」という関係にあるかを判定します。

シグネチャまたは形状

Datum enum_le(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される2つの Oid 型の値（比較対象の列挙値）。
  • **出力**: bool 型（Datum 形式）。第1引数が第2引数以下であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力される Oid は、有効な列挙型値に対応している必要があります。
  • 比較ロジックは enum_cmp_internal に依存しており、列挙型の定義順序に基づいた比較が行われます。

アルゴリズムまたは振る舞い

1. PG_GETARG_OID を使用して、引数から2つの Oid を取得します。 2. enum_cmp_internal を呼び出し、2つの値の比較結果（整数）を取得します。 3. 比較結果が 0 以下（<= 0）であるかどうかを判定し、その結果を PG_RETURN_BOOL で返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_enum.c:54（直接的な呼び出しの証拠あり）
  • **呼び出し先**:
  • enum_cmp_internal（列挙型の比較エンジン）

関連データ構造

  • Oid: 列挙値の識別子として使用。
  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約に従う引数コンテナ。

失敗モードと境界条件

  • enum_cmp_internal が内部で参照するシステムキャッシュや型キャッシュの整合性に依存します。
  • 未コミットの列挙値が使用された場合、check_safe_enum_use（本関数内では直接呼び出されていませんが、列挙型操作の全体的な制約として）による検証が別途必要となる場合があります。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、比較ロジックの変更は enum_cmp_internal に集約されています。
  • contrib/btree_gist からの呼び出しがあるため、インデックス操作における挙動変更には注意が必要です。
