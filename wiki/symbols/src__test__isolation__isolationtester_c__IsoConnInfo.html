IsoConnInfo

定義

  • **種別**: struct
  • **定義箇所**: src/test/isolation/isolationtester.c:25

目的

PostgreSQL 分離レベルテストにおいて、各データベース接続の状態を管理するための構造体です。セッションごとの接続情報、実行中のステップ、および通知メッセージの追跡を保持します。

シグネチャまたは形状

typedef struct IsoConnInfo
{
    PGconn           *conn;            /* libpq 接続オブジェクト */
    int               backend_pid;     /* バックエンド PID (数値) */
    const char       *backend_pid_str; /* バックエンド PID (文字列) */
    const char       *sessionname;     /* 関連付けられたセッション名 */
    PermutationStep  *active_step;     /* 現在実行中のステップ (アイドル時は NULL) */
    int               total_notices;   /* 受信した NOTICE メッセージの総数 */
} IsoConnInfo;

入力と出力

  • **入力**: libpq 接続およびテスト仕様に基づくステップ情報。
  • **出力**: 接続状態の保持、および total_notices による通知数のカウント。

前提条件と不変条件

  • conn は有効な libpq 接続オブジェクトを指す必要があります。
  • active_step が NULL の場合、その接続はアイドル状態であるとみなされます。

アルゴリズムまたは振る舞い

  • テスト実行時に conns 配列として確保され、各セッションの接続状態を追跡します。
  • try_complete_step 等の関数において、特定の接続（IsoConnInfo）が現在どのステップを実行中か、またロック待ちや通知の発生状況を監視するために参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/isolation/isolationtester.c 内の各関数（run_permutation, try_complete_steps 等）が、conns 配列を介してこの構造体を参照・操作します。
  • **呼び出し先**: libpq ライブラリ（PGconn を通じて）および PermutationStep 構造体。

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • PermutationStep: 現在実行中のテストステップを定義する構造体。

失敗モードと境界条件

  • 接続が切断された場合、conn が無効になる可能性があります。
  • total_notices は通知の受信に伴いインクリメントされますが、オーバーフローの考慮はソースコード上では明示されていません。

保守担当者向けメモ

  • conns 配列のインデックスは 1 + step->session のように計算されることが多いため、配列の境界外アクセスに注意してください。
  • backend_pid_str は backend_pid の文字列形式であり、整合性を維持する必要があります。
