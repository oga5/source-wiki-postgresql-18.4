libpqrcv_connect

定義

  • 種別: function
  • 定義箇所: src/backend/replication/libpqwalreceiver/libpqwalreceiver.c:147

目的

プライマリサーバーへのレプリケーション接続を確立し、WalReceiverConn 構造体を初期化して返す。

シグネチャまたは形状

WalReceiverConn *libpqrcv_connect(const char *conninfo, bool replication, bool logical,
                                  bool must_use_password, const char *appname, char **err);

入力と出力

  • **入力**:
  • conninfo: 接続文字列またはURI。
  • replication: レプリケーション接続を行うか否か。
  • logical: 論理レプリケーションを行うか否か。
  • must_use_password: パスワード認証を必須とするか。
  • appname: アプリケーション名。
  • err: エラーメッセージ格納用ポインタ。
  • **出力**: 成功時は WalReceiverConn*、失敗時は NULL。

前提条件と不変条件

  • replication が false の場合、logical は false でなければならない（Assert による検証）。
  • 接続文字列の妥当性は libpqrcv_check_conninfo により事前に検証される。

アルゴリズムまたは振る舞い

1. libpqrcv_check_conninfo を呼び出し、接続文字列を検証する。 2. replication モードに応じて、dbname、replication、client_encoding、options などの接続パラメータを構築する。

  • 論理レプリケーション時は、datestyle や intervalstyle 等のGUC設定を強制するオプションを付与する。

    • 3. libpqsrv_connect_params を使用してプライマリサーバーへ接続する。 4. must_use_password が指定されている場合、PQconnectionUsedPassword を確認し、パスワードが使用されていない場合はエラーとして接続を切断する。 5. レプリケーション接続でない、または論理レプリケーションの場合は、セキュリティのため ALWAYS_SECURE_SEARCH_PATH_SQL を実行して検索パスを固定する。 6. 成功時に WalReceiverConn を返し、失敗時は libpqsrv_disconnect を経て NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/libpqwalreceiver/libpqwalreceiver.c 内の PQWalReceiverFunctions 構造体（.walrcv_connect フィールド）から参照されている。
  • **呼び出し先**: libpqrcv_check_conninfo, libpqsrv_connect_params, libpqsrv_disconnect, libpqsrv_exec, PQstatus, PQconnectionUsedPassword, PQerrorMessage, PQclear 等。

関連データ構造

  • WalReceiverConn: 接続状態（PGconn）や論理レプリケーションフラグを保持する構造体。

失敗モードと境界条件

  • 接続失敗時: PQstatus が CONNECTION_OK でない場合、エラーメッセージを取得して接続をクリーンアップし NULL を返す。
  • 認証失敗時: must_use_password が真でパスワードが使用されなかった場合、ereport(ERROR, ...) により処理を中断する。
  • SQL実行失敗時: 検索パスの設定クエリが失敗した場合、エラーメッセージをセットして接続をクリーンアップし NULL を返す。

保守担当者向けメモ

  • 本関数は libpq の COPY モードを利用する前提で設計されている。
  • 論理レプリケーション時のGUC設定強制は pg_dump の挙動に準拠しているため、変更時は整合性に注意すること。
  • options パラメータの構築時にメモリ確保（psprintf）を行っているため、pfree による解放漏れがないか確認すること。
