ginVacuumItemPointers

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginvacuum.c:46

目的

GINインデックスのタプル（TID）配列から、バキューム対象として不要になった項目を特定して除去し、残存する項目を再構築します。

シグネチャまたは形状

ItemPointer ginVacuumItemPointers(GinVacuumState *gvs, ItemPointerData *items, int nitem, int *nremaining)

入力と出力

  • **入力**:
  • gvs: バキュームの状態とコールバック関数を保持する GinVacuumState 構造体。
  • items: 検査対象の ItemPointerData 配列。
  • nitem: items 配列の要素数。
  • **出力**:
  • 戻り値: 削除後に残存した項目を格納した新しい配列（メモリは palloc で確保）。削除が発生しなかった場合は NULL。
  • nremaining: 残存した項目数へのポインタ。

前提条件と不変条件

  • gvs->callback は、各 ItemPointer が削除対象であるかを判定するために使用されます。
  • 削除対象が一つも存在しない場合、関数は NULL を返し、nremaining には元の nitem が設定されます。

アルゴリズムまたは振る舞い

1. items 配列を順次走査します。 2. 各項目に対し gvs->callback を実行します。

  • **削除対象の場合**: gvs->result->tuples_removed をインクリメントします。最初の削除対象が見つかった時点で、残存項目を格納するためのメモリを palloc で確保し、それまでの走査済み項目をコピーします。
  • **保持対象の場合**: gvs->result->num_index_tuples をインクリメントします。メモリ確保済みであれば、その配列に項目を格納し、remaining カウンタを進めます。

    • 3. 最後に *nremaining を更新し、確保した配列（または NULL）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gindatapage.c:763
  • src/backend/access/gin/ginvacuum.c:500
  • **呼び出し先**:
  • gvs->callback（GinVacuumState 内の関数ポインタ）
  • palloc（メモリ確保）

関連データ構造

  • GinVacuumState: バキューム処理の状態管理およびコールバック関数を保持。
  • ItemPointerData: GINインデックス内のTIDを表現する構造体。

失敗モードと境界条件

  • nitem が 0 の場合、ループは実行されず NULL が返されます。
  • メモリ不足時には palloc がエラーを発生させます。
  • すべての項目が削除対象となった場合、nremaining は 0 となり、空の配列が返されます。

保守担当者向けメモ

  • 本関数は、削除対象が見つかった場合にのみメモリを動的に確保する最適化を行っています。
  • gvs->callback の実装内容によってバキュームの判定ロジックが決定されるため、呼び出し側のコンテキストに依存します。
