ExecProjectSet

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeProjectSet.c:41

目的

ターゲットリスト内に集合を返す関数（SRF）が含まれるクエリにおいて、入力タプルから複数の出力行を生成・投影する実行ノードのメイン処理を行う。

シグネチャまたは形状

TupleTableSlot *ExecProjectSet(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（ProjectSetState 構造体としてキャストされる実行状態）。
  • **出力**: 投影された結果を含む TupleTableSlot。処理すべき行がなくなった場合は NULL を返す。

前提条件と不変条件

  • pstate は ProjectSetState 型であること。
  • SRF はターゲットリストのトップレベルにあることを前提とする。
  • argcontext は SRF の引数評価用メモリコンテキストとして管理される。

アルゴリズムまたは振る舞い

1. **初期処理**: CHECK_FOR_INTERRUPTS() を実行し、ResetExprContext で式評価用メモリを解放する。 2. **SRF 継続評価**: pending_srf_tuples が真の場合、前回の入力タプルから残りの行を生成するため ExecProjectSRF を呼び出し、結果があれば即座に返す。 3. **新規タプル取得ループ**:

  • argcontext をリセットし、前回のタプルサイクルで使用したメモリを解放する。
  • ExecProcNode を呼び出し、子ノードから新しい入力タプルを取得する。タプルが空（NULL）であれば終了する。
  • econtext->ecxt_outertuple に取得したタプルをセットする。
  • ExecProjectSRF を呼び出し、投影を実行する。
  • 結果が得られればそれを返し、得られなければ（空セットの場合など）ループを継続して次の入力タプルを試行する。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（ExecProcNode 経由で呼び出される）。
  • **呼び出し先**:
  • ExecProjectSRF（SRF の評価と行生成）
  • ExecProcNode（子ノードからのタプル取得）
  • ResetExprContext / MemoryContextReset（メモリ管理）

関連データ構造

  • ProjectSetState: ノードの実行時状態を保持する構造体。
  • TupleTableSlot: 投影結果を格納するスロット。
  • ExprContext: 式評価のためのコンテキスト。

失敗モードと境界条件

  • 子ノードからタプルが取得できない場合、NULL を返して終了する。
  • SRF が行を生成しない場合、ループを継続して次の入力タプルを処理する。
  • SRF がメモリリークを起こす可能性があるため、ループの各段階で適切なコンテキストリセットが行われる。

保守担当者向けメモ

  • pending_srf_tuples フラグの状態が、SRF の評価が中断中か否かを決定する鍵となる。
  • argcontext と econtext のリセットタイミングは、SRF が引数を参照するライフサイクルに依存するため、変更時は注意が必要である。
