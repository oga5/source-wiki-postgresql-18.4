do_like_escape

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like_match.c:392

目的

ユーザーが指定したエスケープ文字を、PostgreSQL 内部で標準的に使用されるバックスラッシュ（\）形式に正規化する。

シグネチャまたは形状

static text *do_like_escape(text *pat, text *esc)

入力と出力

  • **入力**:
  • pat: パターン文字列（text *）。
  • esc: エスケープ文字として指定された文字列（text *）。
  • **出力**:
  • エスケープ処理が適用された新しい text オブジェクトを返す。

前提条件と不変条件

  • esc が空でない場合、その長さは必ず1文字でなければならない。それ以外の場合は ERRCODE_INVALID_ESCAPE_SEQUENCE エラーが発生する。
  • 戻り値のメモリ領域は palloc によって確保される。

アルゴリズムまたは振る舞い

1. **初期化**: パターン文字列の長さの最大2倍のメモリを確保する。 2. **エスケープ指定なし (elen == 0)**: パターン内の \ を \\ に置換する。 3. **エスケープ指定あり**:

  • 指定されたエスケープ文字が \ の場合、パターンをそのままコピーして返す。
  • それ以外の場合、パターンを走査し、以下の変換を行う：
  • 指定されたエスケープ文字を \ に置換する。
  • 既存の \ は \\ に置換する（ただし、直前がエスケープ文字である場合を除く）。

    • 4. **終了**: 最終的な文字列長を SET_VARSIZE で設定し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/like.c（マクロ do_like_escape を介して呼び出される）。
  • **呼び出し先**: palloc, ereport, VARDATA_ANY, VARSIZE_ANY_EXHDR, NextChar, CopyAdvChar, CHAREQ, SET_VARSIZE などのマクロおよび関数。

関連データ構造

  • text: PostgreSQL の可変長文字列型。

失敗モードと境界条件

  • esc が2文字以上の場合、ereport によりエラー終了する。
  • パターンが空の場合、適切に処理され空の文字列が返される。

保守担当者向けメモ

  • 本関数はテンプレートファイル like_match.c 内で定義されており、like.c からインクルードされる際にマクロ定義によって制御される。
  • NextChar や CopyAdvChar などのマクロは、インクルード元の like.c で定義されているため、変更時は依存関係に注意すること。
