QueryRepresentationOperand

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsrank.c:546

目的

全文検索における近接度ランキング（Cover Density）計算時、クエリ内の各オペランド（検索語）に関連する文書内の出現位置情報を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    bool        operandexists;
    bool        reverseinsert;
    uint32      npos;
    WordEntryPos pos[MAXQROPOS];
} QueryRepresentationOperand;

入力と出力

本構造体はデータ保持用であり、関数としての入出力は持ちません。主に QueryRepresentation 構造体の一部としてメモリ確保され、近接度計算の過程で WordEntryPos 配列が充填されます。

前提条件と不変条件

  • pos 配列のサイズは MAXQROPOS で固定されています。
  • npos は pos 配列内に格納された有効な位置情報の数を表します。

アルゴリズムまたは振る舞い

  • operandexists: クエリ内の該当オペランドが文書内に存在するかを示します。
  • reverseinsert: true の場合、位置情報が降順で挿入されていることを示します。
  • pos: 文書内での単語の出現位置（WordEntryPos）を格納するバッファです。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/tsrank.c 内の get_docrep 関数等で、クエリサイズに基づいて palloc0 により動的に配列として確保されます。
  • 呼び出し先: 本構造体自体はデータコンテナであり、直接的な関数呼び出しは行いません。

関連データ構造

  • QueryRepresentation: 本構造体を配列として保持する親構造体。
  • WordEntryPos: pos 配列の要素型。文書内の単語位置を表現します。

失敗モードと境界条件

  • MAXQROPOS を超える位置情報が文書内に存在する場合、配列オーバーフローの可能性があるため、実装側での npos のチェックが不可欠です。

保守担当者向けメモ

  • 本構造体は ts_rank_cd（近接度ランキング）の計算において、文書の単語位置を一時的にキャッシュする役割を担います。
  • メモリ割り当ては query->size に依存するため、非常に大きなクエリを扱う際はメモリ消費量に注意してください。
