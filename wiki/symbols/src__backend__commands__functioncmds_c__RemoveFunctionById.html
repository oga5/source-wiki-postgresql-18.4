RemoveFunctionById

定義

  • 種別: function
  • 定義箇所: src/backend/commands/functioncmds.c:1310

目的

指定されたOIDを持つ関数またはプロシージャをシステムカタログから削除し、関連する統計情報を更新する。

シグネチャまたは形状

void RemoveFunctionById(Oid funcOid)

入力と出力

  • **入力**: funcOid (削除対象の関数のOID)
  • **出力**: なし（成功時はカタログ更新および統計削除を実行、失敗時はエラーをスロー）

前提条件と不変条件

  • funcOid は pg_proc システムカタログに存在する有効なOIDである必要がある。
  • 呼び出し元は適切なロックおよび権限チェックを完了していること。

アルゴリズムまたは振る舞い

1. pg_proc テーブルを RowExclusiveLock で開く。 2. PROCOID システムキャッシュを使用して対象関数のタプルを取得する。存在しない場合はエラーをスローする。 3. 取得したタプルから prokind（関数種別）を抽出する。 4. CatalogTupleDelete を呼び出し、pg_proc から該当タプルを削除する。 5. pgstat_drop_function を呼び出し、統計情報を削除する。 6. prokind が PROKIND_AGGREGATE（集約関数）である場合:

  • pg_aggregate テーブルを RowExclusiveLock で開く。
  • AGGFNOID システムキャッシュを使用して対応する集約情報を取得し、削除する。

    • 7. 開いたリレーションをクローズする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/dependency.c:1388 (依存関係解消プロセスからの呼び出し)
  • **呼び出し先**:
  • table_open, table_close
  • SearchSysCache1, ReleaseSysCache
  • CatalogTupleDelete
  • pgstat_drop_function
  • elog (エラー発生時)

関連データ構造

  • pg_proc (システムカタログ)
  • pg_aggregate (システムカタログ)
  • Form_pg_proc (タプル構造体)

失敗モードと境界条件

  • 指定された funcOid が pg_proc に存在しない場合、elog(ERROR, ...) によりトランザクションがアボートされる。
  • 集約関数として削除を試みる際、pg_aggregate に対応するエントリが見つからない場合も同様にエラーとなる。

保守担当者向けメモ

  • 本関数はカタログの物理的な削除を行う低レイヤーの処理である。依存関係のチェックや権限確認は、呼び出し元（主に dependency.c 等）で事前に行われていることを前提としている。
  • prokind に基づく条件分岐により、集約関数特有のメタデータ削除を制御している。新しい関数種別が追加された場合は、ここでの処理拡張が必要になる可能性がある。
