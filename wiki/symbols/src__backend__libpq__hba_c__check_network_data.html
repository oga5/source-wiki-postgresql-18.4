check_network_data

定義

  • 種別: struct
  • 定義箇所: src/backend/libpq/hba.c:56

目的

クライアントの接続要求時、IPアドレスの照合処理において、比較手法、対象アドレス、および照合結果を保持するために使用されるデータ構造です。

シグネチャまたは形状

typedef struct check_network_data
{
	IPCompareMethod method;		/* test method */
	SockAddr   *raddr;			/* client's actual address */
	bool		result;			/* set to true if match */
} check_network_data;

入力と出力

  • **入力**: 構造体初期化時に method（比較手法）と raddr（クライアントのソケットアドレス）が設定されます。
  • **出力**: 照合処理の結果が result フィールドに真偽値として格納されます。

前提条件と不変条件

  • raddr は有効な SockAddr 構造体を指している必要があります。
  • result は照合処理の実行前に初期化されることが想定されます。

アルゴリズムまたは振る舞い

この構造体は、IPアドレスの照合ロジックにおいて、コールバックデータや一時的な状態保持用として利用されます。照合処理の過程で result が更新され、最終的に接続が許可されるべきかどうかの判定に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/hba.c 内の関数（1186行目および1212行目付近）で、照合処理のコンテキストとしてインスタンス化および参照が行われています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • IPCompareMethod: 比較手法を定義する列挙型。
  • SockAddr: クライアントのネットワークアドレス情報を保持する構造体。

失敗モードと境界条件

  • raddr が NULL の状態で参照された場合、セグメンテーション違反を引き起こす可能性があります。
  • 比較手法（method）が未定義または不正な値である場合、照合結果（result）が意図しない値になる可能性があります。

保守担当者向けメモ

  • 本構造体は hba.c 内のローカルな照合ロジックに特化しています。
  • 構造体のフィールド追加や変更を行う際は、hba.c 内の照合関数（check_ip 等）との整合性に注意してください。
