libpqsrv_connect_internal

定義

  • 種別: function
  • 定義箇所: src/include/libpq/libpq-be-fe-helpers.h:159

目的

PostgreSQL バックエンドプロセスにおいて、非同期な libpq 接続確立プロセスを安全に実行するためのヘルパー関数です。接続待機中にバックエンドの割り込み処理を可能にし、リソース（ファイル記述子）の適切な解放を保証します。

シグネチャまたは形状

static inline void libpqsrv_connect_internal(PGconn *conn, uint32 wait_event_info)

入力と出力

  • conn: 接続対象の PGconn オブジェクト。
  • wait_event_info: 待機イベントの識別子（WaitLatchOrSocket で使用）。
  • 戻り値: なし（void）。成功時は接続が確立され、失敗時は例外がスローされるか、接続がクローズされます。

前提条件と不変条件

  • conn が NULL の場合、ReleaseExternalFD() を呼び出してファイル記述子を解放し、即座に終了します。
  • PQstatus(conn) が CONNECTION_BAD の場合、接続確立を試みずに終了します。
  • 接続確立プロセスは PG_TRY/PG_CATCH ブロックで保護されており、エラー発生時には ReleaseExternalFD() と PQfinish(conn) が確実に実行されます。

アルゴリズムまたは振る舞い

1. PGRES_POLLING_OK または PGRES_POLLING_FAILED に達するまで、PQconnectPoll をループ実行します。 2. ループ内では WaitLatchOrSocket を使用し、ソケットの準備状況（読み込み可能/書き込み可能）を監視します。 3. WaitLatchOrSocket が WL_LATCH_SET を返した場合、CHECK_FOR_INTERRUPTS() を呼び出し、バックエンドの割り込み要求（停止要求など）を処理します。 4. ソケットが準備完了状態であれば、PQconnectPoll を進めます。 5. 処理中に例外が発生した場合、PG_CATCH ブロックでリソースを解放し、エラーを再スローします。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/libpq/libpq-be-fe-helpers.h 内の接続関連関数（libpqsrv_connect 等）から参照されています。
  • 呼び出し先: ReleaseExternalFD, PQstatus, PQsocket, WaitLatchOrSocket, ResetLatch, CHECK_FOR_INTERRUPTS, PQconnectPoll, PQfinish, PG_RE_THROW。

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • MyLatch: バックエンドのプロセス間通信用ラッチ。

失敗モードと境界条件

  • conn が NULL の場合: FD を解放して正常終了。
  • PQstatus が CONNECTION_BAD の場合: 何もせず終了。
  • 接続確立失敗時: PG_CATCH ブロックにより FD 解放と PQfinish が行われ、例外が再スローされます。
  • 割り込み発生時: CHECK_FOR_INTERRUPTS() により、バックエンドのシグナル処理が優先されます。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結しています。
  • WaitLatchOrSocket の待機イベント設定には wait_event_info を使用するため、呼び出し側で適切なイベントタイプを指定してください。
  • Windows 環境では CONNECTION_STARTED 状態に対して WL_SOCKET_CONNECTED を使用する特殊な分岐が含まれています。
