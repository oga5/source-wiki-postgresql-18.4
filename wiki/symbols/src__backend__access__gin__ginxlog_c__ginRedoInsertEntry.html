ginRedoInsertEntry

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginxlog.c:70

目的

GINインデックスのページに対して、WALリプレイ時にエントリの挿入（および必要に応じた削除やリンク更新）を適用する。

シグネチャまたは形状

static void ginRedoInsertEntry(Buffer buffer, bool isLeaf, BlockNumber rightblkno, void *rdata)

入力と出力

  • buffer: 操作対象のインデックスページを含むバッファ。
  • isLeaf: ページがリーフページかどうかを示すフラグ。
  • rightblkno: 分割後の右側ページのブロック番号（分割が発生していない場合は InvalidBlockNumber）。
  • rdata: ginxlogInsertEntry 構造体へのポインタ。挿入するタプルやオフセット情報を含む。
  • 戻り値: なし。失敗時は elog(ERROR) によりリプレイを中断する。

前提条件と不変条件

  • buffer は呼び出し元で適切にロックおよび取得されている必要がある。
  • rightblkno が指定されている場合、対象ページは非リーフページでなければならない（Assert(!GinPageIsLeaf(page))）。
  • data->isDelete が真の場合、対象ページはリーフページでなければならない。

アルゴリズムまたは振る舞い

1. **右側リンクの更新**: rightblkno が有効な場合、指定された offset にある既存タプルのダウンリンクを rightblkno に更新する。 2. **既存タプルの削除**: data->isDelete が真の場合、指定された offset のタプルをページから削除する。 3. **新規タプルの挿入**: data->tuple を指定された offset に挿入する。 4. **エラーハンドリング**: PageAddItem が失敗した場合は、対象の物理的な位置情報を出力してエラーを発生させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/gin/ginxlog.c:391（WALレコードの解析フローから呼び出される）。
  • 呼び出し先: BufferGetPage, PageGetItem, GinSetDownlink, GinPageIsLeaf, PageIndexTupleDelete, PageAddItem, BufferGetTag, elog。

関連データ構造

  • ginxlogInsertEntry: WALレコードに含まれる挿入情報を保持する構造体。
  • Buffer: PostgreSQLのバッファ管理構造体。
  • IndexTuple: GINインデックスのタプル形式。

失敗モードと境界条件

  • PageAddItem が InvalidOffsetNumber を返した場合、インデックスページへの追加失敗としてエラー終了する。
  • offset がページ内の有効な範囲外である場合、アサーション違反が発生する。

保守担当者向けメモ

  • 本関数はWALリプレイ専用であり、インデックスの物理構造を直接変更する。
  • isLeaf フラグは現在の実装では主にアサーションチェックに使用されている。
  • ページ分割後のリンク更新と新規挿入が同一の関数内で行われるため、WALレコードの順序と整合性に注意すること。
