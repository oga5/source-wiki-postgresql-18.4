CteItem

定義

  • 種別: struct
  • 定義箇所: src/backend/parser/parse_cte.c:63

目的

WITH 句（CTE）の解析過程において、各CTEの依存関係を管理し、トポロジカルソートを行うための補助的なデータ構造です。

シグネチャまたは形状

typedef struct CteItem
{
    CommonTableExpr *cte;       /* 解析対象のCTE */
    int             id;         /* 依存関係解決用のID */
    Bitmapset       *depends_on; /* 自身を除く依存先CTEの集合 */
} CteItem;

入力と出力

本構造体はデータ保持用であり、関数ではないため直接的な入出力はありません。解析フェーズにおいて CteState 内の配列として生成され、依存関係の特定およびソート処理に使用されます。

前提条件と不変条件

  • depends_on は、当該CTEが参照する他のCTEのIDを保持する Bitmapset です。
  • id は、解析対象のCTE群の中で一意である必要があります。

アルゴリズムまたは振る舞い

  • makeDependencyGraph によって各CTEの依存関係が特定され、depends_on に格納されます。
  • TopologicalSort 関数において、この構造体の配列が入力として利用され、依存関係に基づいた順序付けが行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_cte.c 内の transformWithClause（配列の確保）、TopologicalSort（ソート処理）。
  • **呼び出し先**: 本構造体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • CommonTableExpr: CTEの定義を保持するノード。
  • CteState: 解析状態を保持する構造体で、CteItem の配列を管理します。
  • Bitmapset: 依存関係の集合を表現するために使用されます。

失敗モードと境界条件

  • メモリ不足時に palloc0 が失敗する可能性があります。
  • 循環参照が存在する場合、TopologicalSort が適切に処理を行う必要があります（詳細は TopologicalSort の実装に依存）。

保守担当者向けメモ

  • depends_on は Bitmapset であるため、CTEの数が非常に多い場合でも効率的に依存関係を管理可能です。
  • 本構造体は parse_cte.c 内での解析専用であり、外部モジュールへ公開されることは想定されていません。
