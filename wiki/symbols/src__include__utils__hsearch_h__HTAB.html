HTAB

定義

  • 種別: struct
  • 定義箇所: src/include/utils/hsearch.h:61

目的

PostgreSQL の動的ハッシュテーブル（dynahash）を管理するための不透明（opaque）な構造体です。メモリ内での効率的なデータ検索、挿入、削除を行うための基盤を提供します。

シグネチャまたは形状

typedef struct HTAB HTAB;

※構造体の内部定義は公開されておらず、不透明なポインタとして扱われます。

入力と出力

  • **入力**: hash_create() を通じて生成され、HASHCTL 構造体によって初期設定が行われます。
  • **出力**: hash_search() 等の関数にポインタとして渡すことで、ハッシュテーブルに対する操作の対象となります。

前提条件と不変条件

  • 呼び出し側は、本構造体の内部メンバに直接アクセスせず、提供される API を介して操作する必要があります。
  • 共有メモリ上で使用する場合は、作成時に HASH_SHARED_MEM フラグを指定する必要があります。

アルゴリズムまたは振る舞い

  • 動的に拡張可能なハッシュテーブルとして機能します。
  • hash_search() を用いて、指定されたキーに対する検索、挿入、削除を行います。
  • hash_seq_init() および hash_seq_search() を用いた全要素のシーケンシャルスキャンをサポートします。
  • hash_destroy() により、関連するメモリリソースが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照例）**:
  • contrib/dblink/dblink.c
  • contrib/pg_stat_statements/pg_stat_statements.c
  • contrib/pg_trgm/trgm_regexp.c
  • contrib/postgres_fdw/connection.c および shippable.c
  • contrib/tablefunc/tablefunc.c
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、具体的な呼び出し先（関数）は src/backend/utils/hash/dynahash.c に実装されています。

関連データ構造

  • HASHCTL: ハッシュテーブルの作成時に設定を指定するために使用されます。
  • HASH_SEQ_STATUS: シーケンシャルスキャンの状態を保持するために使用されます。
  • HASHACTION: 検索操作の動作（検索、挿入、削除など）を定義します。

失敗モードと境界条件

  • メモリ不足時には、hash_create() 等の操作が失敗する可能性があります。
  • 共有メモリ使用時のフラグ指定ミスや、必須である HASH_ELEM フラグの欠如は、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は dynahash のインターフェースの一部です。内部実装の詳細については src/backend/utils/hash/dynahash.c を参照してください。
  • ユーザーデータは HASHELEMENT の直後に配置され、ハッシュキーはユーザーデータの先頭にあることが期待される設計となっています。
