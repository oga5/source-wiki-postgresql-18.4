mbstr_verifier

定義

  • 種別: typedef
  • 定義箇所: src/include/mb/pg_wchar.h:390

目的

マルチバイト文字列が、特定のエンコーディング規則に従って妥当であるかを検証するための関数ポインタ型です。

シグネチャまたは形状

typedef int (*mbstr_verifier) (const unsigned char *mbstr, int len);

入力と出力

  • **入力**:
  • mbstr: 検証対象のマルチバイト文字列へのポインタ。
  • len: 文字列の長さ（バイト単位）。
  • **出力**:
  • int: 検証結果を返します。一般的に、妥当な場合は 0、不正なバイト列が含まれる場合は非ゼロの値を返します（具体的な戻り値の仕様は実装に依存します）。

前提条件と不変条件

  • mbstr は少なくとも len バイト以上のメモリ領域を指している必要があります。
  • この型は pg_wchar_tbl 構造体の一部として保持され、各エンコーディング固有の検証ロジックを抽象化するために使用されます。

アルゴリズムまたは振る舞い

  • 指定されたエンコーディングのルールに基づき、mbstr から始まる文字列を走査します。
  • 文字列内にエンコーディングとして不正なバイトシーケンスが存在するかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/mb/pg_wchar.h 内の pg_wchar_tbl 構造体において、mbverifystr フィールドとして定義されています。具体的な呼び出し箇所については、本ソース excerpt には含まれていません。
  • **呼び出し先**: 不明（各エンコーディングの実装関数に依存します）。

関連データ構造

  • pg_wchar_tbl: 各エンコーディングの検証関数を保持するテーブル構造体。

失敗モードと境界条件

  • len が 0 の場合、または負の値の場合の挙動は、各実装関数に依存します。
  • 不正なバイト列が検出された場合、呼び出し元に対してエラーを示す値を返すことが期待されます。

保守担当者向けメモ

  • 本シンボルは関数ポインタ型であるため、実際の検証ロジックは src/common/wchar.c 等で定義される各エンコーディングごとの関数実装を参照してください。
  • pg_wchar_tbl を介して動的に呼び出されるため、新しいエンコーディングを追加する際は、この型に適合する検証関数を実装し、テーブルに登録する必要があります。
