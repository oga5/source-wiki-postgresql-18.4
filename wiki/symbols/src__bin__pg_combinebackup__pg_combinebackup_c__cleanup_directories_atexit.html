cleanup_directories_atexit

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/pg_combinebackup.c:693

目的

シグネチャまたは形状

static void cleanup_directories_atexit(void)

入力と出力

  • 入力: グローバル変数 cleanup_dir_list（削除対象ディレクトリのリスト）。
  • 出力: なし（副作用としてディレクトリの削除を実行）。

前提条件と不変条件

  • cleanup_dir_list が有効な cb_cleanup_dir 構造体のリストを指していること。
  • リスト内の各要素は、rmtree 関数で削除可能なパスを保持していること。

アルゴリズムまたは振る舞い

1. cleanup_dir_list が空になるまでループを実行します。 2. 各要素について rmtree を呼び出します。

  • dir->rmtopdir が真の場合、ディレクトリ自体を削除します。
  • dir->rmtopdir が偽の場合、ディレクトリ内のコンテンツのみを削除します。

    • 3. 削除処理の成否を pg_log_info および pg_log_error で記録します。 4. 処理済みのリスト要素を pfree で解放し、次の要素へ進みます。

呼び出し元と呼び出し先

  • 呼び出し元: atexit() を介してプログラム終了時に呼び出されます（src/bin/pg_combinebackup/pg_combinebackup.c:329）。
  • 呼び出し先: rmtree()（ディレクトリ削除用）、pg_log_info / pg_log_error（ログ出力用）、pfree()（メモリ解放用）。
  • 注意: ツールにより initdb や pg_basebackup からの参照も検出されていますが、これらは同名の関数を持つ別実装である可能性が高く、本モジュール内での直接的な呼び出し関係は pg_combinebackup.c 内の atexit 登録のみです。

関連データ構造

  • cb_cleanup_dir: 削除対象のパスと、ディレクトリ自体を削除するか否かのフラグ（rmtopdir）を保持する構造体。

失敗モードと境界条件

  • rmtree が失敗した場合、エラーログが出力されますが、プログラム終了処理の一部であるため、それ以上の回復処理は行われません。
  • リストが空の場合、何も行わずに終了します。

保守担当者向けメモ

  • 本関数は atexit で登録されるため、シグナルハンドラ等から直接呼び出すことは想定されていません。
  • cleanup_dir_list の管理（追加処理）が他の関数で行われているため、リストの整合性が維持されていることを確認してください。
