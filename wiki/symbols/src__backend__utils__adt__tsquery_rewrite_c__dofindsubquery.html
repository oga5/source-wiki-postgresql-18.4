dofindsubquery

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_rewrite.c:205

目的

tsquery の木構造（QTNode）を再帰的に走査し、指定された部分木パターン（ex）と一致するノードを置換（subs）し、置換後に不要となった演算子ノードを削除・簡略化する。

シグネチャまたは形状

static QTNode *dofindsubquery(QTNode *root, QTNode *ex, QTNode *subs, bool *isfind)

入力と出力

  • root: 走査対象の QTNode 木のルート。
  • ex: 検索対象となる部分木パターン。
  • subs: 置換後の新しい部分木。
  • isfind: 置換が発生したかどうかを記録するフラグへのポインタ。
  • **戻り値**: 処理後の QTNode 木のルート（簡略化によりノードが削除された場合は NULL を返す可能性がある）。

前提条件と不変条件

  • QTNode の木構造は、事前に QTNSort 等で正規化されている必要がある。
  • 再帰的な処理を行うため、スタック深さの制限（check_stack_depth）および割り込みチェック（CHECK_FOR_INTERRUPTS）が実行される。

アルゴリズムまたは振る舞い

1. findeq を呼び出し、現在のノードが置換対象と一致するか判定する。 2. 一致しなかった場合、かつノードが演算子（QI_OPR）であり、QTN_NOCHANGE フラグが設定されていない場合に限り、子ノードに対して再帰的に dofindsubquery を実行する。 3. 子ノードの置換結果が NULL（削除）である場合、親ノードの子供リストから除外する。 4. 子ノードの数が 0 になった場合、現在の演算子ノード自体を解放し NULL を返す。 5. 子ノードの数が 1 になり、かつ演算子が NOT ではない場合、演算子ノードを削除し、残った子ノードを親に昇格させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsquery_rewrite.c 内の findsubquery および自身（再帰呼び出し）。
  • **呼び出し先**: findeq（一致判定）、dofindsubquery（再帰）、QTNFree（メモリ解放）、pfree（メモリ解放）。

関連データ構造

  • QTNode: tsquery を表現する木構造のノード。
  • QI_OPR: QTNode の型が演算子であることを示す識別子。
  • QTN_NOCHANGE: 置換対象外であることを示すフラグ。

失敗モードと境界条件

  • **スタックオーバーフロー**: 深い木構造に対して再帰を行うため、check_stack_depth() により保護されている。
  • **クエリキャンセル**: 長時間の処理を防ぐため、CHECK_FOR_INTERRUPTS() が呼び出される。
  • **ノード削除**: 子ノードがすべて削除された場合、親ノードも再帰的に削除される。

保守担当者向けメモ

  • 本関数は tsquery の書き換えにおける簡略化フェーズの核となる。
  • QTN_NOCHANGE フラグの管理が最適化の鍵であり、誤ったフラグ設定は置換漏れや無限ループの原因となる可能性がある。
  • NOT 演算子の場合は子ノードが 1 つであっても削除できないため、簡略化ロジックに例外処理が含まれている点に注意すること。
