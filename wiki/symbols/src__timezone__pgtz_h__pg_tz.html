pg_tz

定義

  • 種別: struct
  • 定義箇所: src/timezone/pgtz.h:65

目的

タイムゾーンの正規化された名称と、その内部状態（遷移情報や時刻型情報）を保持し、PostgreSQLのタイムゾーン処理サブシステムにおいて時刻変換の基盤となるデータ構造を提供します。

シグネチャまたは形状

struct pg_tz
{
    char        TZname[TZ_STRLEN_MAX + 1];
    struct state state;
};

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。tzload 関数によってファイルから読み込まれたデータが state フィールドに格納され、TZname には対応するタイムゾーン名が設定されます。

前提条件と不変条件

  • TZname は TZ_STRLEN_MAX + 1 バイトの固定長配列であり、正規化されたタイムゾーン名が格納されます。
  • state フィールドは tzfile 形式のバイナリデータをメモリ上に展開した構造体であり、時刻変換に必要な遷移情報や閏秒情報を含みます。

アルゴリズムまたは振る舞い

本構造体自体はデータコンテナですが、以下のライフサイクルに関与します。 1. pg_open_tzfile によりファイルが特定される。 2. tzload によりファイル内容が state 構造体に展開される。 3. pg_tz インスタンスがセッションやログのタイムゾーン設定として保持され、時刻計算時に参照される。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）:**
  • src/backend/commands/variable.c: セッションおよびログのタイムゾーン設定の保持に使用。
  • src/backend/utils/adt/datetime.c: DetermineTimeZoneOffsetInternal や FetchDynamicTimeZone 等の時刻変換処理において、タイムゾーン情報の参照先として使用。
  • **呼び出し先:**
  • 本構造体はデータ構造であるため、直接的な関数呼び出しは行いません。

関連データ構造

  • struct state: pg_tz 内で保持される、タイムゾーンの遷移情報や閏秒情報を含む構造体。
  • pgtime.h: 関連する時刻型定義を提供。
  • tzfile.h: タイムゾーンファイルのバイナリフォーマット定義を提供。

失敗モードと境界条件

  • TZname の長さが TZ_STRLEN_MAX を超える場合、切り詰めやバッファオーバーフローのリスクがあるため、定義された定数に従う必要があります。
  • state 構造体の各フィールドは tzfile の仕様に依存しており、不正なタイムゾーンファイルが読み込まれた場合、時刻変換処理において予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は src/timezone/pgtz.h で定義されていますが、pgtime.h:50 にも同名の定義が存在するため、参照時にはインクルードパスとスコープに注意してください。
  • 外部モジュールからタイムゾーン機能を利用する場合は、本構造体を直接操作せず、pgtime.h で提供される公開インターフェースを使用することを推奨します。
