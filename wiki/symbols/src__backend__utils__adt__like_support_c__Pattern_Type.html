Pattern_Type

定義

  • 種別: enum
  • 定義箇所: src/backend/utils/adt/like_support.c:58

目的

プランナがパターンマッチング演算子（LIKE、正規表現等）の種類を識別し、適切なインデックスサポート関数や選択率推定ロジックを選択するための識別子として機能します。

シグネチャまたは形状

typedef enum
{
    Pattern_Type_Like,
    Pattern_Type_Like_IC,
    Pattern_Type_Regex,
    Pattern_Type_Regex_IC,
    Pattern_Type_Prefix,
} Pattern_Type;

入力と出力

  • 本シンボルは列挙型であり、関数への引数として渡されることで、対象となるパターンマッチングの特性を伝達します。

前提条件と不変条件

  • like_support.c 内の各サポート関数（like_regex_support 等）は、この型を使用して演算子の挙動（大文字小文字の区別の有無など）を判断します。

アルゴリズムまたは振る舞い

  • Pattern_Type_Like / Pattern_Type_Like_IC: LIKE 演算子（大文字小文字の区別あり/なし）。
  • Pattern_Type_Regex / Pattern_Type_Regex_IC: 正規表現演算子（大文字小文字の区別あり/なし）。
  • Pattern_Type_Prefix: 固定文字列の接頭辞マッチング。

呼び出し元と呼び出し先

  • **呼び出し元**: like_regex_support、patternsel、patternjoinsel、pattern_fixed_prefix 等のプランナサポート関数が引数として受け取ります。
  • **呼び出し先**: 本シンボル自体はデータ型定義であり、呼び出し先はありません。

関連データ構造

  • Node: プランナのサポート要求（rawreq）を保持する構造体。
  • Const: パターン文字列を保持する構造体。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 新しいパターンマッチング手法を追加する場合は、この列挙型に新しいメンバを追加し、like_regex_support 等の分岐処理を更新する必要があります。
