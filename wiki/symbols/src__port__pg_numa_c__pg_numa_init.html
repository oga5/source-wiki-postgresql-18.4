pg_numa_init

定義

  • 種別: function
  • 定義箇所: src/port/pg_numa.c:47

目的

PostgreSQL における NUMA 機能の利用可否を判定し、初期化を行います。

シグネチャまたは形状

int pg_numa_init(void)

入力と出力

  • 入力: なし
  • 出力: int 型。NUMA が利用可能な場合は 0 以上の値、利用不可またはエラーの場合は -1 を返します。

前提条件と不変条件

  • Linux 環境において libnuma が利用可能であることを前提としています。
  • get_mempolicy システムコールが実行可能である必要があります。

アルゴリズムまたは振る舞い

1. get_mempolicy(NULL, NULL, 0, 0, 0) を呼び出し、権限チェックを行います。 2. もし get_mempolicy が失敗し、かつ errno が EPERM（権限不足）である場合、NUMA は利用不可と判断し -1 を返します。これは、seccomp プロファイル等により get_mempolicy が制限されているコンテナ環境での誤動作を防ぐための措置です。 3. 上記以外の場合、numa_available() を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/pg_buffercache/pg_buffercache_pages.c
  • src/backend/storage/ipc/shmem.c
  • 呼び出し先:
  • get_mempolicy (システムコール)
  • numa_available (libnuma ライブラリ関数)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • get_mempolicy が EPERM を返す環境では、NUMA 機能が無効化されます。
  • libnuma がシステムに存在しない、または初期化できない場合、numa_available() の戻り値に従いエラーとなります。

保守担当者向けメモ

  • 本関数は libnuma バージョン 2.0.19 未満における EPERM 処理の不備を回避するワークアラウンドを含んでいます。
  • USE_LIBNUMA が定義されていない環境では、本関数はダミーとして動作します。
