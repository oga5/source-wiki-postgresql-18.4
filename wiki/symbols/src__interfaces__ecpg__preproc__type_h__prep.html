prep

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/preproc/type.h:108

目的

ECPGプリプロセッサにおいて、プリペアードステートメント（準備済み文）のメタデータを保持するために使用される構造体です。

シグネチャまたは形状

struct prep
{
	const char *name;
	const char *stmt;
	const char *type;
};

入力と出力

  • **入力**: プリプロセッサが解析するSQL文中のプリペアードステートメント定義情報。
  • **出力**: 構造体の各フィールド（名前、SQL文本体、型情報）として保持されるデータ。

前提条件と不変条件

  • 各フィールドは const char * であり、文字列リテラルまたは動的に確保された文字列を指すことが想定されます。
  • プリプロセッサの実行状態において、定義されたプリペアードステートメントを識別するための名前（name）が一意である必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、ECPGプリプロセッサがソースコードを解析する際、PREPARE 文によって定義されたステートメントを追跡・管理するためのデータコンテナとして機能します。
  • プリプロセッサは、この構造体を通じてステートメント名と対応するSQL文を関連付け、後の実行時処理に必要な情報を保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは、具体的な呼び出し元は特定できません。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。
  • **注意**: ツールが検出した optimizer/prep.h 等への参照は、PostgreSQLバックエンド側のコードであり、本ECPGプリプロセッサの prep 構造体とは無関係です。

関連データ構造

  • struct variable: プリプロセッサが管理するホスト変数定義。
  • ECPGtype: ホスト変数の型情報を保持する構造体。

失敗モードと境界条件

  • メモリ不足: 構造体自体は静的または動的に確保されますが、フィールドの文字列が動的に確保される場合、メモリ割り当て失敗の可能性があります。
  • 識別子の重複: 同一スコープ内で同じ name を持つプリペアードステートメントが定義された場合の挙動は、プリプロセッサの解析ロジックに依存します。

保守担当者向けメモ

  • 本構造体はECPGプリプロセッサ専用です。PostgreSQLバックエンドの optimizer/prep.h と混同しないよう注意してください。
  • 構造体のメンバーはすべてポインタであるため、ライフサイクル管理（特にメモリの解放）には注意が必要です。
