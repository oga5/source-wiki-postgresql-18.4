init_ts_config_cache

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/ts_cache.c:361

目的

全文検索設定（Tsearch configuration）を管理するためのハッシュテーブルを初期化し、システムカタログの変更時にキャッシュを無効化するためのコールバックを登録します。

シグネチャまたは形状

static void init_ts_config_cache(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 TSConfigCacheHash の初期化およびシステムキャッシュコールバックの登録を行う）

前提条件と不変条件

  • CacheMemoryContext が存在しない場合は、本関数内で CreateCacheMemoryContext() を呼び出して作成されます。
  • TSConfigCacheHash は本関数によって初期化されるまで使用できません。

アルゴリズムまたは振る舞い

1. HASHCTL 構造体を用いて、キーサイズを Oid、エントリサイズを TSConfigCacheEntry に設定します。 2. hash_create を呼び出し、"Tsearch configuration cache" という名前でハッシュテーブルを作成します。 3. CacheRegisterSyscacheCallback を使用して、pg_ts_config および pg_ts_config_map システムカタログの変更を監視し、変更時に InvalidateTSCacheCallBack が実行されるように登録します。 4. CacheMemoryContext が未作成であれば作成します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/ts_cache.c 内の各所（lookup_ts_config_cache 等のキャッシュアクセス関数から呼び出されることが示唆されています）。
  • 呼び出し先: hash_create, CacheRegisterSyscacheCallback, InvalidateTSCacheCallBack, CreateCacheMemoryContext。

関連データ構造

  • TSConfigCacheHash: 全文検索設定を保持するハッシュテーブル。
  • TSConfigCacheEntry: キャッシュエントリの構造体。
  • TSCONFIGOID, TSCONFIGMAP: システムカタログのキャッシュID。

失敗モードと境界条件

  • メモリ不足時には hash_create や CreateCacheMemoryContext がエラーを発生させます。

保守担当者向けメモ

  • 本関数はキャッシュの遅延初期化を前提としています。
  • InvalidateTSCacheCallBack は、カタログ変更時にハッシュテーブル全体を無効化するために使用されます。
