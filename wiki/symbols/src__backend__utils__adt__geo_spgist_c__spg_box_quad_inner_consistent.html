spg_box_quad_inner_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_spgist.c:552

目的

SP-GiSTインデックスの内部ノードを探索する際、検索条件に基づいて不要な象限を枝刈り（フィルタリング）し、次に探索すべきノードを決定する。

シグネチャまたは形状

Datum spg_box_quad_inner_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgInnerConsistentIn 構造体を受け取る。これにはインデックスのプレフィックス、スキャンキー、探索状態（traversalValue）が含まれる。
  • **出力**: spgInnerConsistentOut 構造体を介して、次に探索すべきノード番号のリスト、各ノードに対応する新しい traversalValue（RectBox）、および必要に応じて距離情報が返される。

前提条件と不変条件

  • in->traversalValue が存在しない場合は、初期状態の RectBox が生成される。
  • 探索範囲の計算には RectBox を使用し、4次元空間（x_low, x_high, y_low, y_high）での比較を行う。
  • in->allTheSame が真の場合、すべてのノードを探索対象として報告する。

アルゴリズムまたは振る舞い

1. **初期化**: 探索状態（RectBox）を維持または初期化する。 2. **全ノード探索**: allTheSame フラグが立っている場合は、全ノードを探索対象として返し、距離計算が必要であれば実行する。 3. **枝刈り**: 各象限（0-15）に対し、nextRectBox を用いて次の探索範囲を計算する。 4. **条件判定**: スキャンキーの戦略（overlap, contain, left, right 等）に基づき、nextRectBox が検索条件を満たすか判定する。 5. **結果の格納**: 条件を満たした象限のみを out->nodeNumbers に追加し、対応する nextRectBox を out->traversalValues に保存する。条件を満たさない象限の nextRectBox はメモリリークを防ぐため pfree される。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiSTインデックスの走査プロセスから呼び出される（具体的な呼び出し元関数名はソース内に明示なし）。
  • **呼び出し先**: initRectBox, getRangeBox, spg_box_quad_get_scankey_bbox, nextRectBox, overlap4D, contain4D, contained4D, left4D, overLeft4D, right4D, overRight4D, above4D, overAbove4D, below4D, overBelow4D, pointToRectBoxDistance 等。

関連データ構造

  • RectBox: 4次元空間における現在の探索範囲を保持する構造体。
  • RangeBox: 矩形の範囲を保持する構造体。
  • spgInnerConsistentIn / spgInnerConsistentOut: SP-GiSTの内部整合性チェック用インターフェース構造体。

失敗モードと境界条件

  • 未知の StrategyNumber が渡された場合、elog(ERROR) により処理を中断する。
  • 探索範囲の計算において、メモリコンテキストの切り替え（MemoryContextSwitchTo）を適切に行わない場合、メモリ管理上の不整合が発生する可能性がある。

保守担当者向けメモ

  • 本関数は traversalMemoryContext を使用して nextRectBox を割り当てるため、不要になった場合は必ず pfree する必要がある。
  • 4次元空間の各戦略（left4D 等）は、nextRectBox と queries の比較に基づいている。新しい幾何演算を追加する際は、対応する 4D 判定関数が必要となる。
