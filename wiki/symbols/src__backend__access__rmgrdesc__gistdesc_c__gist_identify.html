gist_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/gistdesc.c:89

目的

GiSTインデックスのWALレコードの info ビットを解析し、対応する操作名を文字列として返します。主に pg_waldump 等のツールでWALの内容を可読化するために使用されます。

シグネチャまたは形状

const char *gist_identify(uint8 info);

入力と出力

  • **入力**: uint8 info （WALレコードの info フィールド）
  • **出力**: const char * （操作名を表す文字列。該当するものがない場合は NULL）

前提条件と不変条件

  • 入力された info ビットから XLR_INFO_MASK を除外した値に基づいて判定を行います。
  • GiST固有のWALレコードタイプ（XLOG_GIST_*）のみを対象とします。

アルゴリズムまたは振る舞い

1. 入力された info から XLR_INFO_MASK をマスクして除去します。 2. 残った値に対して switch 文を用い、以下のいずれかの定数と照合します。

  • XLOG_GIST_PAGE_UPDATE -> "PAGE_UPDATE"
  • XLOG_GIST_DELETE -> "DELETE"
  • XLOG_GIST_PAGE_REUSE -> "PAGE_REUSE"
  • XLOG_GIST_PAGE_SPLIT -> "PAGE_SPLIT"
  • XLOG_GIST_PAGE_DELETE -> "PAGE_DELETE"
  • XLOG_GIST_ASSIGN_LSN -> "ASSIGN_LSN"

    • 3. 一致した操作名を返し、一致しない場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h 内の PG_RMGR マクロ定義を通じて、PostgreSQLのWALリソースマネージャインターフェースから参照されます。
  • **呼び出し先**: 本関数は直接他の関数を呼び出しません。

関連データ構造

  • access/gistxlog.h: GiSTのWALレコードタイプ定数（XLOG_GIST_*）が定義されています。

失敗モードと境界条件

  • 定義されていない info ビットが渡された場合、id は初期値の NULL のまま返されます。

保守担当者向けメモ

  • 新しいGiSTのWALレコードタイプを追加した場合は、必ず本関数の switch 文に新しいケースを追加してください。
