pg_rightmost_one_pos32

定義

  • 種別: static inline function
  • 定義箇所: src/include/port/pg_bitutils.h:110

目的

32bit 符号なし整数において、最も右側（最下位）にある 1 のビット位置を 0 から 31 の範囲で返します。

シグネチャまたは形状

int pg_rightmost_one_pos32(uint32 word)

入力と出力

  • **入力**: uint32 word（対象となる 32bit 整数）
  • **出力**: int（最下位の 1 のビット位置。0 はビット 0、31 はビット 31 に対応）

前提条件と不変条件

  • 入力 word は 0 であってはなりません（Assert(word != 0) により検証されます）。0 を渡した場合の動作は未定義です。

アルゴリズムまたは振る舞い

環境に応じて以下のいずれかの手法でビット位置を特定します。 1. **HAVE__BUILTIN_CTZ が定義されている場合**: コンパイラの組み込み関数 __builtin_ctz を使用します。 2. **MSVC 環境 (_MSC_VER) の場合**: _BitScanForward 命令を使用してビット位置をスキャンします。 3. **上記以外の場合**: 8bit ずつシフトしながら 0 でないバイトを探し、ルックアップテーブル pg_rightmost_one_pos を参照して残りのビット位置を特定するフォールバック実装を使用します。

  • src/include/lib/radixtree.h:1219

  • **呼び出し先**:

関連データ構造

  • pg_rightmost_one_pos: フォールバック実装で使用されるルックアップテーブル（src/port/pg_bitutils.c で定義）。

失敗モードと境界条件

  • word == 0: アサーション違反が発生します。呼び出し側で 0 でないことを保証する必要があります。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスが重視される箇所で使用されます。
  • 新しいプラットフォームで最適化を追加する場合は、pg_bitutils.c にあるルックアップテーブルとの整合性を維持してください。
