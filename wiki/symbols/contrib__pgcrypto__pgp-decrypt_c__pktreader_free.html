pktreader_free

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-decrypt.c:208

目的

PGP パケット読み取り用フィルタに関連付けられたメモリリソースを解放し、機密情報を含む可能性のあるデータ領域をゼロクリアする。

シグネチャまたは形状

static void pktreader_free(void *priv)

入力と出力

  • **入力**: priv (解放対象の PktData 構造体へのポインタ)
  • **出力**: なし

前提条件と不変条件

  • priv は PktData 構造体として初期化されている必要がある。
  • pfree を使用するため、PostgreSQL のメモリコンテキスト内で呼び出される必要がある。

アルゴリズムまたは振る舞い

1. priv を PktData 型としてキャストする。 2. px_memset を呼び出し、PktData 構造体の全領域を 0 で上書きする（機密情報の消去）。 3. pfree を呼び出し、構造体に割り当てられたメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: pgp_create_pkt_reader 内で、フィルタ構造体の解放関数として登録される。
  • **呼び出し先**: px_memset (メモリ消去用)、pfree (メモリ解放用)。

関連データ構造

  • PktData: パケット読み取りの状態を保持する構造体。

失敗モードと境界条件

  • priv が NULL の場合、px_memset および pfree の動作に依存する（通常はセグメンテーション違反を引き起こす可能性がある）。

保守担当者向けメモ

  • 本関数は PullFilter の解放処理の一部として設計されている。機密情報（セッション鍵等）がメモリ上に残存することを防ぐため、解放前に必ず px_memset を実行すること。
