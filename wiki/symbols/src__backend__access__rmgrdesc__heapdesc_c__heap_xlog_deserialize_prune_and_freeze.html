heap_xlog_deserialize_prune_and_freeze

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/heapdesc.c:104

目的

ヒープのPRUNEおよびFREEZE操作に関連するWALレコードのバイナリデータをデシリアライズし、各要素（FREEZE計画、リダイレクト、削除済みアイテム、未使用アイテム、FREEZEオフセット）を個別のポインタ変数へ抽出する。

シグネチャまたは形状

void heap_xlog_deserialize_prune_and_freeze(
    char *cursor, 
    uint8 flags,
    int *nplans, xlhp_freeze_plan **plans,
    OffsetNumber **frz_offsets,
    int *nredirected, OffsetNumber **redirected,
    int *ndead, OffsetNumber **nowdead,
    int *nunused, OffsetNumber **nowunused
)

入力と出力

  • **入力**:
  • cursor: 解析対象のWALレコードデータへのポインタ。
  • flags: 含まれるデータ構造を特定するためのビットフラグ（XLHP_HAS_FREEZE_PLANS, XLHP_HAS_REDIRECTIONS 等）。
  • **出力**:
  • 各引数（nplans, plans, frz_offsets, nredirected, redirected, ndead, nowdead, nunused, nowunused）を介して、解析されたデータ数およびデータ配列へのポインタを書き戻す。

前提条件と不変条件

  • flags に対応するデータが存在する場合、各サブレコードの ntargets は 0 より大きい必要がある（Assert による検証）。
  • cursor は、各フラグに応じたデータ構造のサイズ分だけ順次進められる。

アルゴリズムまたは振る舞い

1. flags を確認し、以下の各データ構造の有無を判定する。

  • XLHP_HAS_FREEZE_PLANS: xlhp_freeze_plans 構造体を読み取り、計画数と配列ポインタを抽出。
  • XLHP_HAS_REDIRECTIONS: xlhp_prune_items 構造体を読み取り、リダイレクト対象数とオフセット配列を抽出。
  • XLHP_HAS_DEAD_ITEMS: xlhp_prune_items 構造体を読み取り、削除対象数とオフセット配列を抽出。
  • XLHP_HAS_NOW_UNUSED_ITEMS: xlhp_prune_items 構造体を読み取り、未使用対象数とオフセット配列を抽出。

    • 2. 各ステップで cursor を適切なオフセット分進める。 3. 最後に、残りの cursor 位置を frz_offsets として設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_xlog.c:94
  • src/backend/access/rmgrdesc/heapdesc.c:304
  • **呼び出し先**:
  • 本関数はデータ構造のデシリアライズを行うユーティリティであり、特定の関数を呼び出すことはない。

関連データ構造

  • xlhp_freeze_plan
  • xlhp_freeze_plans
  • xlhp_prune_items
  • OffsetNumber

失敗モードと境界条件

  • flags に基づいてデータが存在すると期待されているにもかかわらず、メモリ上のデータが破損している場合、Assert によりプロセスが停止する可能性がある。

保守担当者向けメモ

  • 本関数はWALの再実行（Redo）ロジックと解析ロジックで共通のデータ構造を扱うために分離されている。
  • 新しいフラグやデータ構造を追加する場合は、cursor の進め方と flags の判定順序に注意すること。
