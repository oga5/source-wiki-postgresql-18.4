AllocSetFree

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/aset.c:1061

目的

AllocSet メモリコンテキスト内のメモリチャンクを解放し、再利用可能なフリーリストへ戻すか、巨大なチャンクの場合は即座にメモリを解放する。

シグネチャまたは形状

void AllocSetFree(void *pointer)

入力と出力

  • **入力**: pointer (解放対象のメモリ領域へのポインタ)
  • **出力**: なし（副作用としてメモリの解放またはフリーリストへの登録を行う）

前提条件と不変条件

  • pointer は AllocSetAlloc によって割り当てられた有効なチャンクを指している必要がある。
  • MEMORY_CONTEXT_CHECKING が有効な場合、二重解放（double-pfree）やチャンク境界を超えた書き込みが検出されるとエラーまたは警告が発生する。

アルゴリズムまたは振る舞い

1. PointerGetMemoryChunk を使用してチャンクヘッダーを取得する。 2. チャンクが「外部管理（巨大）」か判定する:

  • **外部管理の場合**: チャンクを含むブロックを特定し、AllocSet のブロックリストから取り除いた後、free() でメモリを解放する。
  • **通常の場合**: チャンクサイズに応じたフリーリストのインデックスを特定し、チャンクをそのリストの先頭に繋ぐ。

    • 3. デバッグオプション（CLOBBER_FREED_MEMORY）が有効な場合、解放されたメモリ領域を上書きして破壊を検出しやすくする。 4. Valgrind 用のメモリ状態管理（VALGRIND_MAKE_MEM_DEFINED / NOACCESS）を適切に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c (メモリコンテキストのインターフェース経由)
  • その他、メモリ管理を必要とする各サブシステム（tuplesort.c 等での言及あり）
  • **呼び出し先**:
  • free() (標準ライブラリ)
  • elog() (エラー報告)
  • wipe_mem() (デバッグ時)
  • VALGRIND_MAKE_MEM_* マクロ

関連データ構造

  • AllocSetContext: メモリコンテキストの管理構造体。
  • MemoryChunk: チャンクヘッダー。
  • AllocBlock: メモリブロックのヘッダー。
  • AllocFreeListLink: フリーリストのリンク構造。

失敗モードと境界条件

  • **二重解放**: requested_size が InvalidAllocSize である場合、ERROR を発生させる。
  • **メモリ破壊**: チャンク末尾を超えた書き込みが検出された場合、WARNING を発生させる。
  • **不正なブロック**: 外部管理チャンクのブロックヘッダーが検証に失敗した場合、ERROR を発生させる。

保守担当者向けメモ

  • 本関数は AllocSet のホットスポットであるため、パフォーマンスが重視されている。
  • ERROR と WARNING の使い分けは、メモリ破壊の連鎖を防ぐための設計意図に基づいている。
  • MemoryChunkIsExternal の判定ロジックは、巨大な割り当てと通常の割り当てを分離する重要な分岐点である。
