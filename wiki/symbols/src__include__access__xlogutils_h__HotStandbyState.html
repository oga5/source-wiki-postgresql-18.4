HotStandbyState

定義

  • 種別: enum
  • 定義箇所: src/include/access/xlogutils.h:50

目的

PostgreSQL のリカバリプロセスにおける Hot Standby 機能の進行状況を追跡・管理するために使用されます。

シグネチャまたは形状

typedef enum
{
	STANDBY_DISABLED,
	STANDBY_INITIALIZED,
	STANDBY_SNAPSHOT_PENDING,
	STANDBY_SNAPSHOT_READY,
} HotStandbyState;

入力と出力

  • 本シンボルは列挙型であり、直接的な入力や出力は持ちません。
  • 状態変数 standbyState（extern PGDLLIMPORT）の値として保持されます。

前提条件と不変条件

  • 本列挙型は、リカバリプロセス（スタートアッププロセス）においてのみ有効な状態を定義します。
  • InHotStandby マクロの判定基準として利用されます。

アルゴリズムまたは振る舞い

  • STANDBY_DISABLED: Hot Standby が無効な状態。
  • STANDBY_INITIALIZED: Hot Standby が初期化された状態。
  • STANDBY_SNAPSHOT_PENDING: スナップショットの取得を待機している状態。
  • STANDBY_SNAPSHOT_READY: スナップショットが準備完了し、読み取り専用クエリを受け入れ可能な状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlogutils.c にて standbyState 変数の初期値として参照されています。
  • 呼び出し先: 提供されたソース情報には、本シンボルが直接呼び出す関数や、本シンボルを引数として受け取る関数の具体的な記述はありません。

関連データ構造

  • standbyState: HotStandbyState 型のグローバル変数。

失敗モードと境界条件

  • 提供されたソース情報には、本シンボルの失敗モードや境界条件に関する記述はありません。

保守担当者向けメモ

  • 本シンボルはスタートアッププロセスでのみ有効です。他のプロセスからリカバリ状態を確認する場合は、本シンボルを直接参照せず RecoveryInProgress() を使用してください。
