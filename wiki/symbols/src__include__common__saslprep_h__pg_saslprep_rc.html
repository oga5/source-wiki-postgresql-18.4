pg_saslprep_rc

定義

  • 種別: enum
  • 定義箇所: src/include/common/saslprep.h:20

目的

RFC 4013 (SASLprep) に準拠した文字列正規化処理の結果ステータスを表現するために使用されます。

シグネチャまたは形状

typedef enum
{
	SASLPREP_SUCCESS = 0,
	SASLPREP_OOM = -1,
	SASLPREP_INVALID_UTF8 = -2,
	SASLPREP_PROHIBITED = -3,
} pg_saslprep_rc;

入力と出力

  • 入力: なし（列挙型の定義）
  • 出力: 処理結果を示す整数値（成功時は 0、エラー時は負の値）

前提条件と不変条件

  • SASLPREP_SUCCESS は処理が正常に完了したことを示します。
  • SASLPREP_OOM はフロントエンド環境においてメモリ確保に失敗したことを示します。
  • SASLPREP_INVALID_UTF8 は入力文字列が不正な UTF-8 シーケンスであることを示します。
  • SASLPREP_PROHIBITED は出力文字列に RFC 4013 で禁止されている文字が含まれることを示します。

アルゴリズムまたは振る舞い

pg_saslprep() 関数の戻り値として使用され、呼び出し元に対して正規化処理の成否およびエラーの種類を伝達します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/libpq/auth-scram.c
  • src/common/saslprep.c
  • src/interfaces/libpq/fe-auth-scram.c
  • 呼び出し先: なし（列挙型のため）

関連データ構造

  • pg_saslprep(): この列挙型を戻り値として使用するメインの正規化関数。

失敗モードと境界条件

  • SASLPREP_OOM: メモリ不足発生時。
  • SASLPREP_INVALID_UTF8: 入力文字列の UTF-8 検証失敗時。
  • SASLPREP_PROHIBITED: 禁止文字チェック失敗時。

保守担当者向けメモ

  • 本列挙型はバックエンドとフロントエンドの両方で共通して利用されます。
  • SASLPREP_OOM はフロントエンドでのみ発生する可能性がある点に注意してください。
