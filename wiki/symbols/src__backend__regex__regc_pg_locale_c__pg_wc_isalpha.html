pg_wc_isalpha

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_pg_locale.c:325

目的

指定されたワイド文字（pg_wchar）が、現在のロケール設定においてアルファベット（isalpha）であるかを判定します。

シグネチャまたは形状

int pg_wc_isalpha(pg_wchar c)

入力と出力

  • **入力**: pg_wchar c（判定対象の文字コード）
  • **出力**: int（アルファベットであれば非ゼロ、そうでなければ 0）

前提条件と不変条件

  • pg_regex_strategy が適切に初期化されている必要があります。
  • pg_wchar は符号なし型として扱われます。
  • PG_REGEX_STRATEGY_ICU を使用する場合、USE_ICU が定義されている必要があります。

アルゴリズムまたは振る舞い

現在のロケール戦略（pg_regex_strategy）に基づき、以下のいずれかの方法で判定を行います。

1. **PG_REGEX_STRATEGY_C**: pg_char_properties 配列を参照し、ASCII範囲内（127以下）のプロパティを確認します。 2. **PG_REGEX_STRATEGY_BUILTIN**: pg_u_isalpha(c) を呼び出し、組み込みのUnicodeテーブルで判定します。 3. **PG_REGEX_STRATEGY_LIBC_WIDE**: wchar_t のサイズに応じて iswalpha_l を使用します。 4. **PG_REGEX_STRATEGY_LIBC_1BYTE**: UCHAR_MAX 以下の文字に対し isalpha_l を使用します。 5. **PG_REGEX_STRATEGY_ICU**: u_isalpha(c) を呼び出します（ICU有効時）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_locale.c（pg_ctype_get_cache への引数として使用）
  • src/backend/regex/regc_locale.c（文字クラス判定の条件式内）
  • **呼び出し先**:
  • pg_u_isalpha
  • iswalpha_l
  • isalpha_l
  • u_isalpha

関連データ構造

  • pg_wchar: 判定対象の文字型。
  • PG_Locale_Strategy: 判定戦略を決定する列挙型。
  • pg_regex_strategy: 現在の戦略を保持するグローバル変数。
  • pg_regex_locale: ロケール情報を保持する構造体。

失敗モードと境界条件

  • pg_regex_strategy が予期しない値の場合、関数は 0 を返します。
  • PG_REGEX_STRATEGY_LIBC_WIDE において、wchar_t が4バイト未満かつ c が 0xFFFF を超える場合、判定結果は戦略の定義に依存します。
  • ICU が無効な環境で PG_REGEX_STRATEGY_ICU が選択された場合の挙動は、コンパイル時の条件分岐により制御されます。

保守担当者向けメモ

  • 本関数は regcomp.c からインクルードされることを前提としています。
  • 新しいロケール戦略を追加する場合は、switch 文に新しい case を追加してください。
  • pg_char_properties は ASCII 範囲の判定にのみ使用されるため、マルチバイト文字の判定には適していません。
