pg_current_wal_insert_lsn

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogfuncs.c:294

目的

現在のWAL（Write-Ahead Log）挿入位置を示すLSN（Log Sequence Number）をSQL層へ返却します。

シグネチャまたは形状

Datum pg_current_wal_insert_lsn(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出し規約に基づく引数）
  • **出力**: Datum 型（LSN値として解釈される値）

前提条件と不変条件

  • サーバーがリカバリ中（RecoveryInProgress()）である場合、本関数は実行できません。
  • 実行時にエラーが発生した場合、ereport により処理が中断されます。

アルゴリズムまたは振る舞い

1. RecoveryInProgress() を呼び出し、リカバリ中であればエラーを報告して終了します。 2. GetXLogInsertRecPtr() を呼び出し、現在のWAL挿入位置を取得します。 3. 取得したLSN値を PG_RETURN_LSN マクロを使用して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • SQLインターフェース経由でユーザーから呼び出されます。
  • src/bin/pg_rewind/libpq_source.c において、リモートシステム上の本関数を呼び出すクエリが発行されています。
  • **呼び出し先**:
  • RecoveryInProgress()（リカバリ状態確認）
  • GetXLogInsertRecPtr()（WAL挿入位置の取得）

関連データ構造

  • XLogRecPtr: LSNを保持するためのデータ型。

失敗モードと境界条件

  • リカバリ中に呼び出された場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE エラーが発生します。

保守担当者向けメモ

  • 本関数はSQLインターフェースであり、内部的なWAL管理ロジック（xlog.c）へのブリッジとして機能します。
  • 戻り値のLSNは、現在挿入中のWALレコードの末尾位置を示します。
