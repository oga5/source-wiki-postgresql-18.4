regex_fixed_prefix

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like_support.c:1090

目的

正規表現パターンから固定の接頭辞（prefix）を抽出し、インデックススキャンで利用可能な形式に変換する。また、パターン全体の選択率を推定する。

シグネチャまたは形状

Pattern_Prefix_Status regex_fixed_prefix(Const *patt_const, bool case_insensitive, Oid collation, Const **prefix_const, Selectivity *rest_selec);

入力と出力

  • **入力**:
  • patt_const: パターン文字列を含む Const ノード。
  • case_insensitive: 大文字小文字を区別しない検索かどうか。
  • collation: 照合順序（Collation）の OID。
  • **出力**:
  • prefix_const: 抽出された固定接頭辞（抽出できない場合は NULL）。
  • rest_selec: パターンマッチングの残りの部分に対する選択率（Selectivity）。
  • 戻り値: Pattern_Prefix_Status（Pattern_Prefix_None, Pattern_Prefix_Exact, Pattern_Prefix_Partial のいずれか）。

前提条件と不変条件

  • patt_const の型が BYTEAOID である場合、エラーを報告する（bytea 型の正規表現演算子は未定義のため）。
  • rest_selec が NULL でない場合、正規表現の選択率計算が行われる。

アルゴリズムまたは振る舞い

1. regexp_fixed_prefix を呼び出し、パターンから固定接頭辞を抽出する。 2. 接頭辞が抽出できない場合:

  • prefix_const を NULL に設定する。
  • rest_selec が指定されていれば、regex_selectivity を用いてパターン全体の選択率を計算する。
  • Pattern_Prefix_None を返す。

    • 3. 接頭辞が抽出できた場合:

  • string_to_const を用いて接頭辞を Const ノードに変換し、prefix_const に格納する。
  • rest_selec が指定されていれば、完全一致（exact）かどうかに応じて選択率を計算する（完全一致なら 1.0、部分一致なら regex_selectivity を使用）。
  • 抽出結果に応じて Pattern_Prefix_Exact または Pattern_Prefix_Partial を返す。

関連データ構造

  • Pattern_Prefix_Status (列挙型)
  • Const (PostgreSQL ノード構造体)
  • Selectivity (double 型のエイリアス)

失敗モードと境界条件

  • BYTEAOID が渡された場合、ereport(ERROR, ...) により処理が中断される。
  • 正規表現エンジンが接頭辞を特定できない場合、Pattern_Prefix_None を返し、インデックスによる最適化を断念する。

保守担当者向けメモ

  • 本関数は bytea 型に対して安全ではない（ソース内のコメント参照）。
  • rest_selec の計算において、regex_selectivity を呼び出す際にパターン文字列のメモリ確保と解放（pfree）を行っているため、メモリリークに注意すること。
