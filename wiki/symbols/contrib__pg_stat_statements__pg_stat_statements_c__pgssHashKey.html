pgssHashKey

定義

  • 種別: struct
  • 定義箇所: contrib/pg_stat_statements/pg_stat_statements.c:143

目的

pg_stat_statements モジュールにおいて、共有メモリ上のハッシュテーブルで個々のクエリ統計（pgssEntry）を一意に識別するための検索キーとして使用されます。

シグネチャまたは形状

typedef struct pgssHashKey
{
	Oid			userid;			/* ユーザー OID */
	Oid			dbid;			/* データベース OID */
	int64		queryid;		/* クエリ識別子 */
	bool		toplevel;		/* トップレベルで実行されたクエリか */
} pgssHashKey;

入力と出力

本構造体はデータ保持用であり、関数ではありません。ハッシュテーブル検索時には、この構造体の各フィールドが検索キーとして入力されます。

前提条件と不変条件

  • pgssEntry 構造体の先頭メンバとして配置される必要があります（entry_alloc 等でのキャスト利用のため）。
  • ハッシュテーブルの検索や新規作成時には、memset 等でゼロ初期化された後に各フィールドが設定される必要があります。

アルゴリズムまたは振る舞い

  • ユーザーID、データベースID、クエリID、およびトップレベルフラグの組み合わせにより、クエリの実行コンテキストを特定します。
  • ハッシュテーブルのキーサイズとして sizeof(pgssHashKey) が使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • entry_alloc（新規エントリ作成時）
  • pg_stat_statements 関連の統計取得関数内（pgssHashKey key として宣言し、検索キーとして利用）
  • **呼び出し先**:
  • 本構造体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • pgssEntry: 本構造体をキーとして保持するハッシュテーブルの要素。
  • pgssSharedState: 本構造体を利用するハッシュテーブルを管理する共有状態構造体。

失敗モードと境界条件

  • 構造体内の各フィールドが正しく設定されていない場合、ハッシュテーブルの検索ミスや、誤った統計エントリへのアクセスが発生します。
  • memset による初期化を怠ると、パディング領域の不定値によりハッシュ計算が不安定になる可能性があります。

保守担当者向けメモ

  • pgssEntry の先頭メンバであるという制約は、メモリレイアウトに依存するため、構造体の順序変更には注意が必要です。
  • toplevel フラグは、再帰的なクエリ呼び出しとトップレベルのクエリを区別するために重要です。
