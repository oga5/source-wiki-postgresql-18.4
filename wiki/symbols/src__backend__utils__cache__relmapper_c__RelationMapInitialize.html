RelationMapInitialize

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/relmapper.c:650

目的

リレーションマッピング管理システムで使用されるメモリ上のデータ構造を初期化し、未ロード状態に設定する。

シグネチャまたは形状

void RelationMapInitialize(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として静的変数の状態をリセットする）

前提条件と不変条件

  • 本関数は、システム起動時または初期化フェーズにおいて、マップデータ構造をクリーンな状態にするために呼び出される。
  • 静的変数の初期値がゼロであることを前提としているが、明示的にゼロクリアを行うことで安全性を担保している。

アルゴリズムまたは振る舞い

1. 共有マップ (shared_map) およびローカルマップ (local_map) の magic 番号を 0 に設定し、未ロード状態であることを示す。 2. 両マップの num_mappings を 0 にリセットする。 3. トランザクションに関連する更新状態（active_shared_updates, active_local_updates, pending_shared_updates, pending_local_updates）の num_mappings をすべて 0 にリセットする。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/relcache.c:4033
  • 呼び出し先: なし（本関数は内部変数のリセットのみを行う）

関連データ構造

  • shared_map / local_map: マップファイルの内容を保持する構造体。
  • active_shared_updates / active_local_updates: トランザクション内で変更されたマッピング情報。
  • pending_shared_updates / pending_local_updates: 保留中のマッピング更新情報。

失敗モードと境界条件

  • 本関数自体は単純な代入操作であるため、失敗モードは存在しない。

保守担当者向けメモ

  • 本関数は初期化専用である。マップの再読み込みが必要な場合は RelationMapInvalidate 等の他の関数を使用すること。
  • magic 番号を 0 に設定することで、後続の処理がマップの未ロード状態を正しく判定できるようにしている。
