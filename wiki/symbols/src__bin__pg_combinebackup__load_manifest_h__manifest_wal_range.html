manifest_wal_range

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_combinebackup/load_manifest.h:44

目的

バックアップマニフェストに含まれる WAL（Write-Ahead Log）のタイムラインおよび LSN（Log Sequence Number）範囲を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct manifest_wal_range
{
    TimeLineID    tli;
    XLogRecPtr    start_lsn;
    XLogRecPtr    end_lsn;
    struct manifest_wal_range *next;
    struct manifest_wal_range *prev;
} manifest_wal_range;

入力と出力

  • 入力: マニフェストファイルから読み込まれた WAL 範囲情報。
  • 出力: 双方向リストの要素として、manifest_data 構造体内の first_wal_range および last_wal_range を介して管理される。

前提条件と不変条件

  • next および prev ポインタにより、WAL 範囲の双方向リストを構成する。
  • tli は TimeLineID 型、start_lsn と end_lsn は XLogRecPtr 型であり、access/xlogdefs.h の定義に依存する。

アルゴリズムまたは振る舞い

  • load_manifest.c において、マニフェスト解析時に palloc を用いてメモリが割り当てられ、リストに追加される。
  • write_manifest.c において、リストを走査して WAL 範囲情報を出力するために利用される。

呼び出し元と呼び出し先

  • 呼び出し元（生成）: src/bin/pg_combinebackup/load_manifest.c
  • 呼び出し元（参照）: src/bin/pg_combinebackup/write_manifest.c、src/bin/pg_verifybackup/pg_verifybackup.c
  • 呼び出し先: 特になし（データ保持用構造体）。

関連データ構造

  • manifest_data: 本構造体のリストを保持するルート構造体。
  • manifest_file: マニフェスト内のファイル情報を管理する構造体。

失敗モードと境界条件

  • メモリ不足時: palloc によるメモリ割り当て失敗が発生する可能性がある。
  • リストの整合性: リストの連結処理において、next および prev ポインタの更新が正しく行われない場合、リストの走査が不能になる。

保守担当者向けメモ

  • simplehash を使用したファイル管理とは異なり、WAL 範囲は単純な双方向リストとして管理されている点に注意すること。
  • write_manifest.h でも前方宣言されているため、マニフェストの読み込みと書き込みの両方で共通のデータ構造として扱われる。
