pg_base64_encode

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/encode.c:287

目的

バイナリデータを Base64 エンコードされたテキスト形式に変換し、76文字ごとに改行を挿入する。

シグネチャまたは形状

static uint64 pg_base64_encode(const char *src, size_t len, char *dst)

入力と出力

  • **入力**:
  • src: エンコード対象のバイナリデータへのポインタ。
  • len: src のバイト長。
  • dst: エンコード結果を格納するバッファへのポインタ（十分なサイズが確保されていること）。
  • **出力**:
  • 戻り値: dst に書き込まれた実際のバイト数（uint64）。

前提条件と不変条件

  • dst バッファは、エンコード後のデータ（改行を含む）を格納するために十分なサイズが確保されている必要がある。
  • 内部で使用される _base64 配列が有効であること。

アルゴリズムまたは振る舞い

1. 入力データを3バイトずつ読み込み、4つの6ビット値に分割して _base64 テーブルから対応する文字を選択する。 2. 76文字出力するごとに改行文字 (\n) を挿入する。 3. 入力データの末尾で3バイトに満たない場合、パディング文字 (=) を使用してBase64仕様に従い補完する。 4. 最終的な書き込みバイト数を計算して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/encode.c: pg_base64_enc_len 等と共にエンコーディングテーブルの一部として参照されている。
  • contrib/pgcrypto/pgp-armor.c: PGPアーマー処理において利用されている。
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わず、静的配列 _base64 を参照する。

関連データ構造

  • _base64: Base64変換用の文字テーブル（ソースファイル内で定義）。

失敗モードと境界条件

  • dst バッファが不足している場合、バッファオーバーランが発生する可能性がある（呼び出し元でサイズ計算を行う必要がある）。
  • 入力データが空の場合、戻り値は 0 となる。

保守担当者向けメモ

  • 本関数は76文字ごとの改行挿入を強制する仕様である。改行が不要な用途には適さない。
  • contrib/pgcrypto からの呼び出しと、src/backend/utils/adt/encode.c 内での利用の両面で整合性を保つこと。
