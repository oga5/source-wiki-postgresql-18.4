mic2euc_kr

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_kr_and_mic/euc_kr_and_mic.c:126

目的

MULE_INTERNAL（MIC）形式のバイト列を EUC_KR 形式に変換する内部変換ロジックを提供します。

シグネチャまたは形状

static int mic2euc_kr(const unsigned char *mic, unsigned char *p, int len, bool noError)

入力と出力

  • **入力**:
  • mic: 変換元の MULE_INTERNAL バイト列。
  • p: 変換後の EUC_KR を格納するバッファ。
  • len: 入力バイト列の長さ。
  • noError: 変換エラー時に例外を投げず処理を中断するかどうかを指定するフラグ。
  • **出力**:
  • 戻り値: 処理した入力バイト数。
  • p: 変換結果が格納され、末尾にヌル終端が付与されます。

前提条件と不変条件

  • MULE_INTERNAL 形式のマルチバイト文字は、特定の識別子（LC_KS5601 等）を先頭に持つ構造である必要があります。
  • noError が true の場合、不正なエンコーディングや変換不能な文字に遭遇した時点で処理を停止し、それまでに変換したバイト数を返します。

アルゴリズムまたは振る舞い

1. 入力バイト列をループ処理し、各文字を検証します。 2. ASCII文字（高位ビットが未設定）はそのまま出力バッファへコピーします。 3. LC_KS5601 プレフィックスを検出した場合、後続の2バイトを抽出し EUC_KR 形式として出力します。 4. 不正な文字や変換不能な文字を検出した場合、noError フラグに応じてエラーを報告するか、処理を中断します。 5. 最後にバッファをヌル終端し、消費した入力バイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: mic_to_euc_kr（src/backend/utils/mb/conversion_procs/euc_kr_and_mic/euc_kr_and_mic.c:70）
  • **呼び出し先**:
  • pg_encoding_verifymbchar: 文字の妥当性検証。
  • report_invalid_encoding / report_untranslatable_char: エラー報告（noError が false の場合）。

関連データ構造

  • PG_MULE_INTERNAL: MULE_INTERNAL エンコーディング識別子。
  • PG_EUC_KR: EUC_KR エンコーディング識別子。
  • LC_KS5601: KS X 1001 文字セットを示す MULE_INTERNAL のエスケープシーケンス。

失敗モードと境界条件

  • 入力長 len が 0 に達した時点でループを終了します。
  • c1 == 0（ASCIIのヌル）や、pg_encoding_verifymbchar が負の値を返す不正なシーケンス、または LC_KS5601 以外の変換不能な文字に遭遇した場合、noError が false であればエラーを報告します。

保守担当者向けメモ

  • 本関数は mic_to_euc_kr から呼び出される内部ロジックです。
  • MULE_INTERNAL のエスケープシーケンス構造に依存しているため、新しい文字セットのサポートを追加する際は、プレフィックスの判定ロジックを更新する必要があります。
