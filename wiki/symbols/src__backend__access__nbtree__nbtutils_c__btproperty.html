btproperty

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtutils.c:3704

目的

B-tree インデックスに関する特定のプロパティ（属性）の真偽値を問い合わせるために使用されます。

シグネチャまたは形状

bool btproperty(Oid index_oid, int attno, IndexAMProperty prop, const char *propname, bool *res, bool *isnull)

入力と出力

  • **入力**:
  • index_oid: 対象インデックスの OID。
  • attno: 属性番号（0 はインデックス全体を指す）。
  • prop: 問い合わせるプロパティの種類（IndexAMProperty 型）。
  • propname: プロパティ名（文字列）。
  • **出力**:
  • res: プロパティが真の場合に true が格納される。
  • isnull: 本関数内では使用されていない。
  • **戻り値**: プロパティが B-tree でサポートされている場合は true、それ以外は false。

前提条件と不変条件

  • attno == 0 の場合、インデックス全体に対する問い合わせとみなされ、AMPROP_RETURNABLE に対しては false を返します。

アルゴリズムまたは振る舞い

1. prop 引数に基づき switch 文で分岐します。 2. AMPROP_RETURNABLE（インデックスから値を取得可能か）の場合:

  • attno が 0 でなければ、B-tree は常にデータを返せるため *res = true を設定し true を返します。
  • attno が 0 の場合は false を返します。

    • 3. その他のプロパティについては、汎用コードへ処理を委譲するため false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtree.c 内で amroutine->amproperty に割り当てられています。
  • **呼び出し先**: 本関数が直接呼び出す関数はありません。

関連データ構造

  • IndexAMProperty: プロパティの種類を定義する列挙型。

失敗モードと境界条件

  • 未知の prop が渡された場合、false を返して処理を終了します。

保守担当者向けメモ

  • 本関数は IndexAmRoutine 構造体の amproperty フィールドとして登録されるインターフェース関数です。
  • 現在は AMPROP_RETURNABLE のみに対応しています。新しいプロパティを追加する場合は switch 文を拡張してください。
