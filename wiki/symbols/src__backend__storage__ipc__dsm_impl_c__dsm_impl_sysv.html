dsm_impl_sysv

定義

  • **種別**: function
  • **定義箇所**: src/backend/storage/ipc/dsm_impl.c:422

目的

System V 共有メモリセグメントの作成、アタッチ、デタッチ、および破棄を行う低レベルな実装を提供します。

シグネチャまたは形状

static bool dsm_impl_sysv(dsm_op op, dsm_handle handle, Size request_size,
                          void **impl_private, void **mapped_address, Size *mapped_size,
                          int elevel);

入力と出力

  • **入力**:
  • op: 実行する操作（DSM_OP_CREATE, DSM_OP_ATTACH, DSM_OP_DETACH, DSM_OP_DESTROY）。
  • handle: セグメントを識別する一意なハンドル。
  • request_size: 作成時の要求サイズ。
  • impl_private: 実装固有の識別子（shmget の結果である ident）を保持するポインタ。
  • mapped_address: マッピング済みアドレス。
  • mapped_size: マッピング済みサイズ。
  • elevel: エラー報告レベル。
  • **出力**: 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • dsm_handle は key_t にキャストして使用されます。
  • IPC_PRIVATE キーは使用できません。
  • impl_private が NULL の場合、shmget を呼び出して新しい識別子を取得し、メモリを割り当てます。

アルゴリズムまたは振る舞い

1. **キー生成**: dsm_handle を key_t に変換し、System V 共有メモリキーを生成します。 2. **識別子取得**: *impl_private が空であれば shmget で識別子を取得し、キャッシュします。 3. **操作実行**:

  • DETACH/DESTROY: shmdt でメモリをデタッチし、DESTROY の場合は shmctl(IPC_RMID) でセグメントを削除します。
  • ATTACH: shmctl(IPC_STAT) でサイズを確認し、shmat でマッピングします。
  • CREATE: shmget でセグメントを作成し、shmat でマッピングします。

    • 4. **マッピング**: shmat を使用してプロセスのアドレス空間にセグメントをマッピングし、結果を mapped_address に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/dsm_impl.c 内の dsm_impl_op 関数から呼び出されます。
  • **呼び出し先**: shmget, shmdt, shmctl, shmat などの System V 共有メモリシステムコール。

関連データ構造

  • dsm_op: 操作種別を定義する列挙型。
  • dsm_handle: セグメント識別子。
  • impl_private: int 型の識別子を保持するメモリ領域。

失敗モードと境界条件

  • key が IPC_PRIVATE と一致する場合、エラーとして扱われます。
  • shmget や shmat が失敗した場合、ereport を通じてエラーを報告し false を返します。
  • CREATE 操作中にマッピングに失敗した場合、作成したセグメントを shmctl(IPC_RMID) で破棄するクリーンアップ処理が行われます。

保守担当者向けメモ

  • dsm_handle から key_t への変換時にビットの切り捨てが発生する可能性がありますが、同一ハンドルに対しては常に一貫した変換が行われるため、セグメントの識別には影響しません。
  • shmget のサイズ指定は、既存セグメントへのアクセス時には 0 である必要があります。
