check_exclusion_constraint

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execIndexing.c:963

目的

テーブルへのタプル挿入時に、排他制約（Exclusion Constraint）の違反がないかを検証します。

シグネチャまたは形状

void check_exclusion_constraint(Relation heap, Relation index,
                                IndexInfo *indexInfo,
                                ItemPointer tupleid,
                                const Datum *values, const bool *isnull,
                                EState *estate, bool newIndex)

入力と出力

  • **入力**:
  • heap: 対象のヒープリレーション。
  • index: 検証対象のインデックスリレーション。
  • indexInfo: インデックスに関するメタ情報。
  • tupleid: 挿入されたタプルの物理位置（TID）。
  • values, isnull: インデックスキーの値とNULLフラグの配列。
  • estate: 実行状態（EState）。
  • newIndex: 新規作成されたインデックスかどうかを示すフラグ。
  • **出力**: なし（戻り値 void）。制約違反時はエラーを発生させます。

前提条件と不変条件

  • 本関数は、ヒープへのタプル挿入が完了した後に呼び出されることを想定しています。
  • 内部的に CEOUC_WAIT モードで排他制約チェックを実行するため、競合が発生した場合は待機処理が行われます。

アルゴリズムまたは振る舞い

内部関数 check_exclusion_or_unique_constraint を呼び出し、以下の引数を固定して排他制約の検証を行います。

  • wait_mode: CEOUC_WAIT（競合時に待機するモード）。
  • is_unique: false（排他制約として処理）。
  • conflict_excl_info: NULL。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/index.c
  • src/backend/commands/constraint.c
  • src/backend/commands/indexcmds.c
  • src/backend/executor/execIndexing.c
  • **呼び出し先**:
  • check_exclusion_or_unique_constraint（本ファイル内の内部コア関数）。

関連データ構造

  • Relation: リレーション記述子。
  • IndexInfo: インデックスの構成情報。
  • EState: クエリ実行状態。
  • ItemPointer: タプルの物理位置（TID）。

失敗モードと境界条件

  • 排他制約に違反する既存タプルが存在する場合、競合が解消されるまで待機します。
  • 待機中にデッドロックが発生する可能性があるため、トランザクション管理下での実行が前提となります。

保守担当者向けメモ

  • 本関数は check_exclusion_or_unique_constraint のラッパーです。排他制約チェックの挙動を変更する場合は、呼び出し先のコア関数を確認してください。
  • 投機的挿入（INSERT ... ON CONFLICT）に関連する競合処理については、CEOUC_WAIT_MODE の定義と併せて参照してください。
