addFkConstraintSides

定義

  • 種別: enum
  • 定義箇所: src/backend/commands/tablecmds.c:353

目的

外部キー制約の追加処理において、制約の対象となるテーブル側（参照側または被参照側）を識別するために使用されます。

シグネチャまたは形状

typedef enum addFkConstraintSides
{
	addFkReferencedSide,
	addFkReferencingSide,
	addFkBothSides,
} addFkConstraintSides;

入力と出力

  • 本シンボル自体は列挙型であり、関数への引数として渡されることで、addFkConstraint 関数の動作を制御します。

前提条件と不変条件

  • 外部キー制約の追加処理において、どのテーブルに対して制約の検証やカタログ更新を行うかを決定するフラグとして機能します。

アルゴリズムまたは振る舞い

  • addFkReferencedSide: 被参照テーブル（親テーブル）側の処理を指定します。
  • addFkReferencingSide: 参照テーブル（子テーブル）側の処理を指定します。
  • addFkBothSides: 両方のテーブルに対する処理を指定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/tablecmds.c 内の addFkConstraint 関数を呼び出す箇所で使用されています。
  • **呼び出し先**: 本シンボル自体はデータ型であり、関数を呼び出すことはありません。

関連データ構造

  • addFkConstraint: この列挙型を引数として受け取る関数。

失敗モードと境界条件

  • 列挙型として定義されているため、定義外の値が渡された場合の挙動は、呼び出し側の実装に依存します。

保守担当者向けメモ

  • 本シンボルは src/backend/commands/tablecmds.c 内の外部キー制約追加ロジックに特化した制御用フラグです。変更時は addFkConstraint 関数のロジックと整合性をとる必要があります。
