rewriteVisibilityMap

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/file.c:215

目的

旧バージョンの PostgreSQL で作成された可視性マップ（Visibility Map）ファイルを、現在の形式（1 ページを 2 ページに拡張する形式）へ変換して書き出します。

シグネチャまたは形状

void rewriteVisibilityMap(const char *fromfile, const char *tofile,
                          const char *schemaName, const char *relName)

入力と出力

  • **入力**:
  • fromfile: 変換元の旧形式可視性マップファイルパス。
  • tofile: 変換後の新形式可視性マップ出力先パス。
  • schemaName, relName: エラーログ出力に使用されるスキーマ名およびリレーション名。
  • **出力**: なし（ファイルシステムへの書き込みが副作用）。

前提条件と不変条件

  • fromfile は読み取り可能であること。
  • tofile は新規作成可能であること。
  • BLCKSZ（ブロックサイズ）は旧形式と新形式で共通であること。
  • 変換処理中、旧形式のページヘッダーは新形式の各ページにそのままコピーされます。

アルゴリズムまたは振る舞い

1. 旧ファイルをページ単位で読み込みます。 2. 1 ページを 2 ページに分割する計算を行い、旧形式のビット情報を新形式（VISIBILITYMAP_ALL_VISIBLE）へマッピングします。 3. 各ビットを BITS_PER_HEAPBLOCK 単位で展開し、新形式のページバッファへ書き込みます。 4. 最終ページの末尾が空である場合は、不要なページ生成を避けるため書き出しをスキップします。 5. データチェックサムが有効な場合、pg_checksum_page を用いて新ページごとにチェックサムを再計算します。 6. 変換後のデータを tofile へ書き出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/relfilenumber.c:617 から呼び出されることが確認されています。
  • **呼び出し先**:
  • 標準ライブラリ: open, fstat, read, write, close
  • 内部関数: pg_fatal（エラー時）, pg_checksum_page（チェックサム計算時）

関連データ構造

  • PGIOAlignedBlock: ページ読み書き用のアライメント済みバッファ。
  • PageHeaderData: ページヘッダー情報の保持に使用。
  • new_cluster.controldata.data_checksum_version: チェックサム計算の要否判定に使用。

失敗モードと境界条件

  • ファイルのオープン、読み込み、書き込み失敗時には pg_fatal を呼び出し、システムエラーメッセージと共に処理を中断します。
  • 読み込み時にページサイズが BLCKSZ に満たない場合、部分的なページとしてエラーを報告します。
  • 書き込み時にディスク容量が不足している場合、ENOSPC を想定してエラー終了します。

保守担当者向けメモ

  • 本関数は catversion 201603011 以前からのアップグレード時に必須です。
  • rewriteVmBytesPerPage は (BLCKSZ - SizeOfPageHeaderData) / 2 として計算され、旧形式の 1 ページを新形式の 2 ページに変換するロジックの根幹となります。
