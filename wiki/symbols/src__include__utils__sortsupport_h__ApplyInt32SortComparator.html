ApplyInt32SortComparator

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/sortsupport.h:301

目的

int32 型のデータ（Datum）を比較し、NULL 値の扱いとソート順序（昇順/降順）を考慮した比較結果を返す。

シグネチャまたは形状

static inline int ApplyInt32SortComparator(Datum datum1, bool isNull1,
                                           Datum datum2, bool isNull2,
                                           SortSupport ssup)

入力と出力

  • **入力**:
  • datum1, datum2: 比較対象の int32 値。
  • isNull1, isNull2: 各値が NULL かどうかを示すフラグ。
  • ssup: ソート設定を保持する SortSupport 構造体（ssup_nulls_first, ssup_reverse を参照）。
  • **出力**:
  • int: 比較結果。datum1 < datum2 なら -1、datum1 > datum2 なら 1、等価なら 0 を返す。

前提条件と不変条件

  • ssup は初期化済みであること。
  • datum1, datum2 は int32 として解釈可能であること。

アルゴリズムまたは振る舞い

1. **NULL 判定**:

  • 両方が NULL の場合は 0 を返す。
  • 片方のみ NULL の場合、ssup->ssup_nulls_first に基づき、NULL を先頭（-1）または末尾（1）として扱う。

    • 2. **値の比較**:

  • NULL でない場合、DatumGetInt32 を用いて値を抽出し、標準的な比較を行う。

    • 3. **順序の反転**:

  • ssup->ssup_reverse が真の場合、比較結果を INVERT_COMPARE_RESULT マクロで反転させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/tuplesort.c:546 での呼び出しが確認されている。
  • **呼び出し先**: DatumGetInt32 マクロおよび INVERT_COMPARE_RESULT マクロを使用。

関連データ構造

  • SortSupportData (引数 ssup の型): ソートの実行状態を管理する構造体。

失敗モードと境界条件

  • datum が int32 以外の型である場合、DatumGetInt32 によるキャストの結果は未定義。
  • NULL 判定ロジックは ssup_nulls_first の設定に完全に依存する。

保守担当者向けメモ

  • 本関数はインライン化されており、ソートのループ内での呼び出しコストを最小化するように設計されている。
  • 比較ロジックを変更する場合は、src/include/utils/sortsupport.h 内の他の Apply...SortComparator 関数との整合性に注意すること。
