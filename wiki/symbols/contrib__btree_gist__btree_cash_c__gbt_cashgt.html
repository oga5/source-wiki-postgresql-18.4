gbt_cashgt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_cash.c:29

目的

money 型の2つの値に対して「より大きい（greater than）」という比較を行い、その真偽を判定します。

シグネチャまたは形状

static bool gbt_cashgt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の Cash 型へのポインタ（void * として渡される）。
  • b: 比較対象の Cash 型へのポインタ（void * として渡される）。
  • flinfo: 関数呼び出し情報（本関数内では未使用）。
  • **出力**:
  • bool: *a > *b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な Cash 型の値を指している必要があります。
  • money 型は内部的に 64bit 整数として扱われます。

アルゴリズムまたは振る舞い

1. 引数 a および b を const Cash * 型にキャストします。 2. キャストされたポインタが指す値をデリファレンスし、C言語の標準的な比較演算子 > を用いて大小比較を行います。 3. 比較結果をそのまま戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_cash.c 内の tinfo 構造体（gbtree_ninfo）の初期化時に、比較関数として参照されています。
  • GiST インデックスの検索処理において、gbt_num_consistent 経由で間接的に呼び出されます。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません（組み込みの比較演算子を使用）。

関連データ構造

  • Cash: money 型の内部表現。
  • gbtree_ninfo: 本関数が登録される money 型の特性定義構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • money 型の値が比較不能な状態（未初期化など）である場合の動作は定義されていません。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で定義された汎用 GiST 数値インデックスロジックから呼び出されることを前提としています。
  • 比較ロジックを変更する場合は、他の比較関数（gbt_cashlt, gbt_casheq 等）との整合性に注意してください。
