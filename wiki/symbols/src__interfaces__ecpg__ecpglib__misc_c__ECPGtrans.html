ECPGtrans

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/ecpg/ecpglib/misc.c:159

目的

ECPGアプリケーションから発行されるトランザクション制御コマンド（BEGIN, COMMIT, ROLLBACK等）をデータベースに対して実行する。

シグネチャまたは形状

bool ECPGtrans(int lineno, const char *connection_name, const char *transaction)

入力と出力

  • **入力**:
  • lineno: 呼び出し元のソースコード行番号（ログ出力用）。
  • connection_name: 接続識別子（NULLの場合はデフォルト接続）。
  • transaction: 実行するSQLコマンド文字列。
  • **出力**:
  • bool: 成功時に true、失敗時に false を返す。

前提条件と不変条件

  • ecpg_init によるライブラリの初期化が成功していること。
  • 接続が存在しない場合、コマンドは実行されずシミュレートされる。

アルゴリズムまたは振る舞い

1. ecpg_get_connection で対象の接続を取得し、ecpg_init で初期化を行う。 2. 実行内容をデバッグログに出力する。 3. 接続が確立されている場合、以下の条件を満たせば自動的に begin transaction を発行する：

  • 現在のトランザクション状態が PQTRANS_IDLE である。
  • autocommit が無効である。
  • 指定されたコマンドが begin, start, commit prepared, rollback prepared ではない。

    • 4. 指定された transaction コマンドを PQexec で実行する。 5. ecpg_check_PQresult を用いて実行結果を検証し、成功すれば PQclear で結果を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPGで生成されたCコード（例: compat_informix-describe.c, compat_informix-rnull.c 等のテストコード）。
  • **呼び出し先**:
  • ecpg_get_connection
  • ecpg_init
  • ecpg_log
  • PQtransactionStatus
  • PQexec
  • ecpg_check_PQresult
  • PQclear

関連データ構造

  • struct connection: 接続状態を保持する構造体。
  • PGresult: libpqのクエリ結果構造体。

失敗モードと境界条件

  • ecpg_init が失敗した場合、即座に false を返す。
  • PQexec によるコマンド実行が失敗した場合、ecpg_check_PQresult がエラーを検出し、関数は false を返す。
  • 接続がNULLの場合、コマンド実行はスキップされ、関数は true を返す。

保守担当者向けメモ

  • 本関数はトランザクションの暗黙的な開始を制御するため、autocommit 設定との相互作用に注意すること。
  • PQexec の結果検証には ecpg_check_PQresult を介しており、エラーハンドリングのロジックは同関数に依存している。
