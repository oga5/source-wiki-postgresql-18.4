SetQuitSignalReason

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/pmsignal.c:201

目的

システムシャットダウンの理由を共有メモリに記録し、Postmasterから子プロセスへその理由を伝達するために使用されます。

シグネチャまたは形状

void SetQuitSignalReason(QuitSignalReason reason)

入力と出力

  • **入力**: reason (QuitSignalReason 型) - シャットダウンの理由を示す列挙値。
  • **出力**: なし（副作用として共有メモリの状態を更新）。

前提条件と不変条件

  • PMSignalState が初期化済みであること。
  • 本関数は、共有メモリ上の PMSignalData 構造体内の sigquit_reason フィールドを更新します。

アルゴリズムまたは振る舞い

1. グローバルポインタ PMSignalState が指す共有メモリ領域にアクセスします。 2. 引数 reason を PMSignalState->sigquit_reason に代入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/postmaster.c:2206
  • src/backend/postmaster/postmaster.c:2693
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • PMSignalData: 共有メモリ上に配置される構造体で、sigquit_reason を保持します。
  • QuitSignalReason: シャットダウン理由を定義する列挙型。

失敗モードと境界条件

  • PMSignalState が初期化されていない状態で呼び出された場合、セグメンテーションフォールトが発生する可能性があります。

保守担当者向けメモ

  • 本関数は、Postmasterが子プロセスに対して SIGQUIT を送出する直前または送出時に、その意図を共有メモリ経由で同期するために使用されます。
  • 関連する読み取り関数 GetQuitSignalReason と対で使用されることを想定しています。
