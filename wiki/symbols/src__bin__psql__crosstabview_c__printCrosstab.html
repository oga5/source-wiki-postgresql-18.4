printCrosstab

定義

  • 種別: function
  • 定義箇所: src/bin/psql/crosstabview.c:285

目的

クエリ結果セットをクロス集計（ピボット）形式に変換し、printTable API を用いて整形出力する。

シグネチャまたは形状

static bool printCrosstab(const PGresult *result,
                          int num_columns, pivot_field *piv_columns, int field_for_columns,
                          int num_rows, pivot_field *piv_rows, int field_for_rows,
                          int field_for_data)

入力と出力

  • **入力**:
  • result: libpq のクエリ結果セット。
  • piv_columns, piv_rows: 抽出済みの列および行ヘッダー情報。
  • field_for_columns, field_for_rows, field_for_data: 各役割に対応する列インデックス。
  • **出力**: 成功時に true、重複データ検出等のエラー発生時に false を返す。

前提条件と不変条件

  • piv_columns および piv_rows は、事前に avl_tree 等を用いて重複排除およびランク付けが完了していること。
  • num_columns および num_rows は、ヘッダー配列のサイズと一致していること。

アルゴリズムまたは振る舞い

1. **初期化**: printTableInit で出力用バッファを確保。 2. **ヘッダー構築**:

  • 最初の列に field_for_rows の名前を設定。
  • piv_columns のランク順に列ヘッダーを配置。

    • 3. **行ラベル配置**: piv_rows のランクに基づき、各行の先頭セルにラベルを配置。 4. **データマッピング**: result を走査し、bsearch で行・列のランクを特定。該当する cont.cells インデックスにデータを格納。 5. **出力**: 未設定セルを空文字で埋め、printTable で出力。最後に printTableCleanup でメモリを解放。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/crosstabview.c 内の \crosstabview コマンド処理フローから呼び出される。
  • **呼び出し先**: printTableInit, printTableAddHeader, printTable, printTableCleanup (いずれも fe_utils/print.h 関連)、bsearch, pg_malloc, pg_free。

関連データ構造

  • pivot_field: 行・列のヘッダー値、ソート値、ランクを保持。
  • printTableContent: psql のテーブル出力用構造体。

失敗モードと境界条件

  • **重複データ**: 同一の行・列の組み合わせに対して複数のデータが存在する場合、エラーログを出力して false を返す。
  • **検索失敗**: bsearch で行または列が見つからない場合、Assert により異常終了する。

保守担当者向けメモ

  • cont.cells 配列は (num_columns + 1) * num_rows のサイズで確保される。インデックス計算 1 + col_number + row_number * (num_columns + 1) に注意すること。
  • horiz_map は piv_columns のランク順アクセスを効率化するための一時的な逆引きテーブルである。
