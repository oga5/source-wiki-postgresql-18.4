r_aditzak

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_basque.c:1005

目的

バスク語のステミング処理において、単語末尾の動詞接尾辞を識別し、削除または置換を行う。

シグネチャまたは形状

static int r_aditzak(struct SN_env * z)

入力と出力

  • 入力: SN_env 構造体（ステミング環境）。
  • 出力: 処理が成功した場合は 1、失敗または適用対象外の場合は 0、エラー発生時は負の値を返す。

前提条件と不変条件

  • z->c（現在のカーソル位置）が有効な範囲内であること。
  • find_among_b を用いた後方検索により、接尾辞が a_0 テーブルに存在することが条件となる。

アルゴリズムまたは振る舞い

1. 単語末尾の文字が特定の条件（z->p[z->c - 1] のビットマスク判定）を満たすか確認し、満たさない場合は即座に終了する。 2. find_among_b を使用して、a_0 テーブルから接尾辞を後方検索する。 3. マッチした接尾辞のインデックス（among_var）に応じて以下の処理を行う:

  • 1: r_RV 領域内であることを確認し、接尾辞を削除する。
  • 2: r_R2 領域内であることを確認し、接尾辞を削除する。
  • 3, 4, 5: それぞれ対応する文字列（s_0, s_1, s_2）に置換する。

    • 4. 処理完了後、成功値 1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_ISO_8859_1_basque.c および src/backend/snowball/libstemmer/stem_UTF_8_basque.c 内のメインステミング処理から呼び出される。
  • 呼び出し先: r_RV, r_R2, slice_del, slice_from_s, find_among_b。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_0: 動詞接尾辞の探索テーブル。

失敗モードと境界条件

  • find_among_b がマッチしない場合、0 を返す。
  • r_RV や r_R2 の領域チェックに失敗した場合、その戻り値をそのまま返す。
  • 文字列操作関数（slice_del, slice_from_s）がエラーを返した場合、その負の値を伝播させる。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによる自動生成コードであるため、直接編集しないこと。修正が必要な場合は元の .sbl ファイルを修正すること。
  • a_0 テーブルの定義や領域判定ロジックは、バスク語の文法規則に依存している。
