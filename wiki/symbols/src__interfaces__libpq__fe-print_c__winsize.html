winsize

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/fe-print.c:101

目的

端末（ターミナル）の表示サイズ（行数および列数）を保持するための構造体です。

シグネチャまたは形状

struct winsize
{
    int ws_row; /* 行数 */
    int ws_col; /* 列数 */
} screen_size;

入力と出力

  • 入力: 端末制御（ioctl 等）から取得した行数および列数の値。
  • 出力: 構造体メンバ ws_row および ws_col への格納。

前提条件と不変条件

  • sys/ioctl.h および termios.h 等のシステムヘッダーが適切にインクルードされている環境で使用されます。
  • ページャ起動時の端末サイズ判定において、この構造体の値が参照されます。

アルゴリズムまたは振る舞い

  • fe-print.c 内では、クエリ結果の出力時に端末のサイズを把握し、ページャ（PAGER）を起動するかどうかの判断や、出力レイアウトの調整に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-print.c 内の出力処理ロジック。
  • 参照元（直接的な証拠）:
  • src/interfaces/libpq/fe-print.c:99
  • src/bin/psql/help.c:609
  • src/fe_utils/print.c:812, 1452, 3098
  • 呼び出し先: システムの ioctl システムコール（TIOCGWINSZ）を介して端末情報を取得する処理が関連します。

関連データ構造

  • PQprintOpt: fe-print.c で使用される出力オプション構造体。

失敗モードと境界条件

  • 端末サイズが取得できない場合や、非対話型シェルからの実行時には、値が正しく設定されない可能性があります。

保守担当者向けメモ

  • 本構造体はプラットフォーム依存の winsize 定義と競合する可能性があるため、コンパイル環境におけるヘッダーの依存関係に注意してください。
  • psql や fe_utils と定義が重複しているため、共通化の検討対象となり得ます。
