strlower_builtin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_builtin.c:80

目的

PostgreSQLの組み込みロケールプロバイダにおいて、指定されたロケール設定に基づき、入力文字列を小文字に変換する。

シグネチャまたは形状

size_t strlower_builtin(char *dest, size_t destsize, const char *src, ssize_t srclen, pg_locale_t locale)

入力と出力

  • dest: 変換後の文字列を格納するバッファ。
  • destsize: dest バッファのサイズ。
  • src: 変換対象のソース文字列。
  • srclen: src の長さ（バイト単位）。
  • locale: 組み込みロケール情報を含む pg_locale_t 構造体。
  • 戻り値: 変換後の文字列の長さ（バイト単位）。

前提条件と不変条件

  • locale 引数は、組み込みロケールプロバイダによって初期化された有効なポインタであること。
  • locale->info.builtin.casemap_full が、Unicode 変換テーブルへの有効な参照を保持していること。

アルゴリズムまたは振る舞い

1. locale 構造体内の info.builtin.casemap_full フラグを参照する。 2. common/unicode_case.h で定義された unicode_strlower 関数を呼び出し、変換処理を委譲する。 3. 変換結果の長さを呼び出し元に返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/pg_locale.c 内のロケール抽象化レイヤーから呼び出される。
  • 呼び出し先: unicode_strlower（common/unicode_case.h）。

関連データ構造

  • pg_locale_t: ロケール固有の情報を保持する構造体。本関数は内部の builtin メンバを使用する。

失敗モードと境界条件

  • destsize が変換後の文字列を格納するのに不十分な場合、unicode_strlower の仕様に従った切り詰めやエラーが発生する可能性がある（詳細は unicode_strlower の定義に依存）。

保守担当者向けメモ

  • 本関数は libc や ICU に依存せず、PostgreSQL 内部の Unicode 変換テーブルを使用する。
  • PG_UNICODE_FAST ロケールが指定されている場合、高速なケースマッピングが適用される。
