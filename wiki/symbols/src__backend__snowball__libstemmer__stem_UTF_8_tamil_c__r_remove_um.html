r_remove_um

目的

タミル語のステミング処理において、単語末尾の特定の接尾辞（"um" に相当するパターン）を除去し、必要に応じて語幹の正規化を行う。

シグネチャまたは形状

static int r_remove_um(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング処理環境へのポインタ）
  • 出力: 成功時に 1、失敗または条件不一致時に 0、エラー発生時に負の値を返す。

前提条件と不変条件

  • r_has_min_length(z) が真（戻り値が 0 より大きい）であること。
  • SN_env 構造体が初期化されており、処理対象の文字列が保持されていること。

アルゴリズムまたは振る舞い

1. r_has_min_length を呼び出し、最小長を満たさない場合は終了する。 2. 検索範囲を単語末尾に設定する。 3. eq_s_b を使用して、定義された接尾辞パターン（s_22）と末尾を照合する。 4. マッチした場合、slice_from_s を用いて当該部分を別の文字列（s_23）に置換する。 5. 最後に r_fix_ending を呼び出し、置換後の語幹の整合性を調整する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_tamil.c:1375（直接の呼び出しが確認されている）
  • 呼び出し先: r_has_min_length, eq_s_b, slice_from_s, r_fix_ending

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • s_22, s_23: 内部的に参照される文字列定数（among 構造体に関連）。

失敗モードと境界条件

  • r_has_min_length が 0 以下を返した場合、処理を中断する。
  • eq_s_b でパターンが一致しない場合、何も変更せず 0 を返す。
  • slice_from_s または r_fix_ending が負の値を返した場合、そのエラーコードを伝播させる。
