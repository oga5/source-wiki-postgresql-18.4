fsm_search_avail

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/fsmpage.c:157

目的

FSM（空き領域マップ）ページ内において、指定された空き容量（minvalue）以上の空きを持つスロットを効率的に探索する。

シグネチャまたは形状

int fsm_search_avail(Buffer buf, uint8 minvalue, bool advancenext, bool exclusive_lock_held)

入力と出力

  • **入力**:
  • buf: 検索対象のFSMページバッファ。
  • minvalue: 検索する最小の空き容量。
  • advancenext: 探索後に fp_next_slot を進めるかどうか。
  • exclusive_lock_held: 現在排他ロックを保持しているかを示すフラグ。
  • **出力**:
  • 条件を満たすスロット番号（0以上）。条件を満たすスロットが見つからない場合は -1 を返す。

前提条件と不変条件

  • 呼び出し元は、対象のバッファに対して少なくとも共有ロックを保持している必要がある。
  • ページ内のバイナリツリー構造が整合していること（破損時は内部で再構築される）。
  • ルートノード（fp_nodes[0]）の値が minvalue 未満であれば、探索は即座に終了する。

アルゴリズムまたは振る舞い

1. **初期チェック**: ルートノードを確認し、条件を満たすリーフが存在しない場合は -1 を返す。 2. **探索開始**: fp_next_slot をヒントとして探索を開始する。 3. **探索三角形の拡大**: parentof(rightneighbor(nodeno)) を辿ることで、右方向へ移動しつつ親ノードへ登り、探索範囲を対数時間で拡大する。 4. **降下**: 条件を満たすノードに到達後、左の子ノードを優先してリーフレベルまで降下する。 5. **破損検知と修復**: 降下中に期待値と実際の値が一致しない（torn page）場合、fsm_rebuild_page を呼び出してページを再構築し、探索をリトライする。 6. **ポインタ更新**: 探索終了後、fp_next_slot を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/freespace/freespace.c 内の関数（fsm_search_avail を呼び出し、FSM 全体の管理を行う）。
  • **呼び出し先**:
  • parentof, rightneighbor, leftchild（マクロ）
  • fsm_rebuild_page（破損検知時）
  • LockBuffer, BufferGetTag, elog, MarkBufferDirtyHint

関連データ構造

  • FSMPage: ページ内のノード配列（fp_nodes）およびヒント情報（fp_next_slot）を保持する構造体。

失敗モードと境界条件

  • **破損ページ**: ページが部分的に書き込まれた状態（torn page）の場合、fsm_rebuild_page を実行して修復を試みる。
  • **スロットなし**: 条件を満たすスロットが存在しない場合、-1 を返す。
  • **ラップアラウンド**: fp_next_slot がページ末尾に達した場合、適切にラップアラウンドして探索を継続する。

保守担当者向けメモ

  • fp_next_slot はあくまで探索のヒントであり、厳密な順序を保証するものではない。
  • 更新を伴う操作ではない場合でも、fp_next_slot の更新は共有ロック下で行われる可能性がある。
  • 破損検知時の fsm_rebuild_page はコストが高いため、発生頻度を監視すること。
