vector32_load

定義

  • 種別: static inline function
  • 定義箇所: src/include/port/simd.h:120

目的

メモリ上の uint32 配列からデータを読み込み、プラットフォーム固有の SIMD ベクトル型 Vector32 に格納します。

シグネチャまたは形状

static inline void vector32_load(Vector32 *v, const uint32 *s)

入力と出力

  • v (出力): 読み込んだデータを格納する Vector32 型へのポインタ。
  • s (入力): 読み込み元の uint32 型配列へのポインタ。
  • 戻り値: なし（v が指すメモリ領域が更新されます）。

前提条件と不変条件

  • USE_SSE2 または USE_NEON が定義されている環境でのみ有効です。
  • SIMD が利用できない環境では、本関数は実装されません。

アルゴリズムまたは振る舞い

コンパイル時の機能フラグに応じて、以下のいずれかの命令を実行します。

  • USE_SSE2 が有効な場合: _mm_loadu_si128 を使用し、非整列アクセスでデータをロードします。
  • USE_NEON が有効な場合: vld1q_u32 を使用してデータをロードします。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/port/pg_lfind.h 内の複数の箇所で、ベクトル単位のデータ処理のために呼び出されています。
  • 呼び出し先: プラットフォーム固有のヘッダー（emmintrin.h または arm_neon.h）で定義された組み込み関数。

関連データ構造

  • Vector32: SIMD 演算用の抽象型。プラットフォームに応じて __m128i や uint32x4_t 等にマッピングされます。

失敗モードと境界条件

  • SIMD 非対応環境ではコンパイルエラーとなります。
  • ポインタ v および s が NULL の場合の動作は未定義です。

保守担当者向けメモ

  • 本関数は SIMD が利用できない環境では実装を提供していません。これは 64 ビット整数でのエミュレーションが効率的ではないためです。
  • USE_ASSERT_CHECKING が有効な場合、SIMD 演算の整合性検証が行われる可能性があります。
