CollationCreate

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_collation.c:41

目的

システムカタログ pg_collation に新しい照合順序（Collation）定義を登録し、関連する依存関係を構築します。

シグネチャまたは形状

Oid CollationCreate(const char *collname, Oid collnamespace, Oid collowner, 
                    char collprovider, bool collisdeterministic, int32 collencoding, 
                    const char *collcollate, const char *collctype, 
                    const char *colllocale, const char *collicurules, 
                    const char *collversion, bool if_not_exists, bool quiet);

入力と出力

  • **入力**: 照合順序名、名前空間、所有者、プロバイダー、決定論的フラグ、エンコーディング、ロケール情報（collcollate, collctype, colllocale 等）、および if_not_exists / quiet フラグ。
  • **出力**: 作成された照合順序の Oid。重複時に if_not_exists が真であれば InvalidOid を返します。

前提条件と不変条件

  • collprovider が COLLPROVIDER_LIBC の場合、collcollate と collctype が必須であり、colllocale は指定不可。それ以外の場合は colllocale が必須。
  • collencoding == -1 は「全エンコーディング対応」を意味します。
  • 実行には ShareRowExclusiveLock を取得し、カタログの整合性を保護します。

アルゴリズムまたは振る舞い

1. **重複検証**: GetSysCacheOid3 を使用し、同一名・同一エンコーディングの照合順序が既に存在しないか確認します。 2. **ロック取得**: table_open で pg_collation をロックします。 3. **シャドウイング検証**: 特定エンコーディングと「全エンコーディング」の照合順序が互いに隠蔽しないよう検証します。 4. **タプル構築**: heap_form_tuple を用いてカタログレコードを生成します。 5. **挿入**: CatalogTupleInsert でシステムカタログへ書き込みます。 6. **依存関係登録**: 名前空間や所有者との依存関係を記録します（ソース抜粋では名前空間への依存登録まで記述）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/collationcmds.c 内の各コマンド実行関数。
  • **呼び出し先**: GetSysCacheOid3, table_open, table_close, GetNewOidWithIndex, heap_form_tuple, CatalogTupleInsert, checkMembershipInCurrentExtension, ereport 等。

関連データ構造

  • pg_collation: 照合順序を保持するシステムカタログテーブル。
  • ObjectAddress: 依存関係管理に使用される構造体。

失敗モードと境界条件

  • **重複**: 同一の照合順序が既に存在する場合、if_not_exists が偽であれば ERRCODE_DUPLICATE_OBJECT エラーを発生させます。
  • **シャドウイング**: 特定エンコーディングと全エンコーディングの照合順序が競合する場合、エラーとなります。
  • **拡張機能**: 拡張機能スクリプト内での重複時、既存オブジェクトがその拡張機能のメンバーでない場合はセキュリティリスクとして扱われます。

保守担当者向けメモ

  • collencoding == -1 の特殊な扱いと、プロバイダーごとの引数制約（Assert で定義）が実装の要です。
  • 依存関係の登録処理は、ソース抜粋の末尾以降も pg_depend への記録が続くため、完全な依存関係構築には後続のコードを参照してください。
