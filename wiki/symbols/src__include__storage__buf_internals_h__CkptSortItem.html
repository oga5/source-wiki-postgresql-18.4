CkptSortItem

定義

  • 種別: struct
  • 定義箇所: src/include/storage/buf_internals.h:391

目的

チェックポイント処理において、バッファを効率的に書き出すためのソート用データ構造です。バッファIDと、そのバッファが保持するページを特定するための識別情報を保持します。

シグネチャまたは形状

typedef struct CkptSortItem
{
	Oid			tsId;
	RelFileNumber relNumber;
	ForkNumber	forkNum;
	BlockNumber blockNum;
	int			buf_id;
} CkptSortItem;

入力と出力

  • 本構造体は、バッファマネージャがチェックポイント時に書き出すべきバッファのリストを管理するために使用されます。
  • buf_id は、対応するバッファ記述子を特定するためのインデックスとして機能します。

前提条件と不変条件

  • CkptBufferIds 配列として NBuffers 分の領域が確保され、チェックポイント処理のソート対象として利用されます。

アルゴリズムまたは振る舞い

  • チェックポイント時に、バッファを物理的な配置順序（tsId, relNumber, forkNum, blockNum の順）でソートするために使用されます。
  • src/backend/storage/buffer/bufmgr.c 内の ckpt_buforder_comparator 関数により、この構造体を用いた比較が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/buf_init.c にて共有メモリ領域の確保および初期化が行われます。
  • **呼び出し先**: src/backend/storage/buffer/bufmgr.c 内のソート処理（ckpt_buforder_comparator）で利用されます。

関連データ構造

  • BufferDesc: バッファのメタデータを保持する構造体。buf_id を介して関連付けられます。
  • CkptBufferIds: CkptSortItem 型のグローバル配列（extern PGDLLIMPORT）。

失敗モードと境界条件

  • 共有メモリの確保失敗（NBuffers に基づくサイズ計算時）が境界条件となります。
  • ソート時の比較関数において、各フィールドの順序が正しく定義されていない場合、チェックポイントのI/O効率が低下する可能性があります。

保守担当者向けメモ

  • 本構造体のフィールドを変更する場合、ckpt_buforder_comparator の比較ロジックも併せて更新する必要があります。
  • CkptBufferIds は共有メモリ上に配置されるため、構造体のサイズ変更は共有メモリのレイアウトに影響を与えます。
