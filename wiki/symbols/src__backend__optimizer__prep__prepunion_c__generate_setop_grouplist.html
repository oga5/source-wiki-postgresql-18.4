generate_setop_grouplist

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepunion.c:1593

目的

集合演算（UNION, INTERSECT, EXCEPT）において、ターゲットリストの各列とグループ化句（groupClauses）を対応付け、実行時に使用可能なグループ化リストを生成します。

シグネチャまたは形状

List *generate_setop_grouplist(SetOperationStmt *op, List *targetlist)

入力と出力

  • **入力**:
  • op: 集合演算の定義を含む SetOperationStmt 構造体。
  • targetlist: 集合演算結果のターゲットリスト（TargetEntry のリスト）。
  • **出力**:
  • grouplist: op->groupClauses をコピーし、各 SortGroupClause の tleSortGroupRef をターゲットリストの参照番号で更新したリスト。

前提条件と不変条件

  • targetlist 内の各 TargetEntry は resjunk であってはならない。
  • targetlist 内の各 TargetEntry の ressortgroupref は、自身の resno と一致している必要がある。
  • op->groupClauses の要素数は targetlist の要素数と一致している必要がある。
  • 更新前の SortGroupClause において、tleSortGroupRef は 0 である必要がある。

アルゴリズムまたは振る舞い

1. op->groupClauses のディープコピーを作成し、作業用リスト grouplist を生成します。 2. targetlist を走査し、各 TargetEntry に対して以下の処理を行います。

  • 対応する SortGroupClause を grouplist から取得します。
  • SortGroupClause の tleSortGroupRef に、TargetEntry の ressortgroupref を代入します。

    • 3. 全てのターゲット列とグループ化句の対応付けが完了したことを確認し、完成した grouplist を返します。

関連データ構造

  • SetOperationStmt: 集合演算の構文木ノード。
  • TargetEntry: クエリのターゲットリスト要素。
  • SortGroupClause: ソートまたはグループ化の条件を保持する構造体。

失敗モードと境界条件

  • targetlist が空の場合、あるいは groupClauses との要素数が一致しない場合、Assert によりプログラムが停止する可能性があります。
  • resjunk な列が含まれている場合、Assert により停止します。

保守担当者向けメモ

  • 本関数は、集合演算のプランニング過程でターゲットリストの列順序とグループ化条件を整合させるために不可欠です。
  • tleSortGroupRef の更新は、後のプランニングフェーズでグループ化や重複排除を行う際に、どの列をキーとして使用するかを決定するために利用されます。
