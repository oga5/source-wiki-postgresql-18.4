Chromosome

定義

  • 種別: struct
  • 定義箇所: src/include/optimizer/geqo_gene.h:32

目的

PostgreSQLの遺伝的クエリ最適化（GEQO）において、クエリの結合順序を表現する個体（染色体）を保持するために使用されます。

シグネチャまたは形状

typedef struct Chromosome
{
	Gene	   *string;
	Cost		worth;
} Chromosome;

入力と出力

  • string: 結合対象となるリレーションの順序を示す Gene 型の配列へのポインタ。
  • worth: 当該染色体が示す結合順序のコスト評価値。

前提条件と不変条件

  • string は Gene 型の配列を指す必要があります。
  • worth はクエリ最適化プロセスにおけるコスト評価の結果を保持します。

アルゴリズムまたは振る舞い

GEQOアルゴリズムにおいて、この構造体は交叉（Crossover）や突然変異（Mutation）の操作対象となります。個体群（Pool）内で管理され、最終的に worth に基づいて最適解が選択されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照・操作）**:
  • src/backend/optimizer/geqo/geqo_copy.c: geqo_copy 関数で個体の複製に使用。
  • src/backend/optimizer/geqo/geqo_main.c: 遺伝的アルゴリズムのメインループ内で個体（momma, daddy, kid）として使用。
  • src/backend/optimizer/geqo/geqo_pool.c: Pool 内のデータ配列としての管理、および qsort によるソート操作に使用。
  • **呼び出し先**:
  • 本構造体自体はデータ構造定義であり、直接的な関数呼び出しは行いません。

関連データ構造

  • Gene: 結合対象リレーションの識別子（int）。
  • Pool: 複数の Chromosome を管理する個体群構造体。

失敗モードと境界条件

  • string が指すメモリ領域が適切に割り当てられていない場合、GEQOの操作中にメモリ破壊が発生する可能性があります。
  • worth の計算ロジックが不適切な場合、最適化結果が収束しない可能性があります。

保守担当者向けメモ

  • Gene 型は Relid を表現する int です。実装変更時はこの前提がシステム全体に影響することに留意してください。
  • 構造体の操作ロジックについては、geqo_recombination.c や geqo_selection.c などの実装ファイルを参照してください。
