simple_string_list_member

定義

  • 種別: function
  • 定義箇所: src/fe_utils/simple_list.c:86

目的

SimpleStringList 内に特定の文字列が存在するかを検索し、存在する場合はその要素の「検索済み（touched）」フラグを更新します。

シグネチャまたは形状

bool simple_string_list_member(SimpleStringList *list, const char *val)

入力と出力

  • **入力**:
  • list: 検索対象の SimpleStringList 構造体へのポインタ。
  • val: 検索する文字列（const char *）。
  • **出力**:
  • bool: 文字列がリスト内に見つかった場合は true、見つからなかった場合は false を返します。

前提条件と不変条件

  • list は有効な SimpleStringList 構造体を指している必要があります。
  • リスト内の各セルは strcmp による比較が可能な文字列を保持しています。

アルゴリズムまたは振る舞い

1. リストの head から順に各セルを走査します。 2. 各セルの保持する文字列と val を strcmp で比較します。 3. 一致する文字列が見つかった場合、そのセルの touched フラグを true に設定し、直ちに true を返します。 4. リストの末尾まで走査しても一致しない場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_createsubscriber.c
  • src/bin/pg_dump/pg_backup_archiver.c
  • src/bin/pg_dump/pg_dumpall.c
  • **呼び出し先**:
  • strcmp (標準ライブラリ関数)

関連データ構造

  • SimpleStringList: 文字列リストを管理する構造体。
  • SimpleStringListCell: リストの各要素を保持する構造体。touched フラグをメンバとして持ちます。

失敗モードと境界条件

  • リストが空の場合、走査は行われず false を返します。
  • 要素数が多い場合、線形探索のため $O(N)$ の計算量を要します。

保守担当者向けメモ

  • 本関数は検索と同時に touched フラグを更新する副作用を持ちます。このフラグは後に simple_string_list_not_touched で未処理要素を特定するために利用されます。
  • ポインタリストとは異なり、文字列の比較には strcmp を使用するため、メモリ上の文字列内容が比較対象となります。
