gseg_consistent

定義

  • 種別: function
  • 定義箇所: contrib/seg/seg.c:199

目的

GiST インデックスを用いた検索において、インデックス内のノードがクエリ条件を満たすかどうかを判定する。

シグネチャまたは形状

Datum gseg_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY *entry（インデックス内のエントリ）
  • PG_GETARG_DATUM(1): query（検索条件）
  • PG_GETARG_UINT16(2): strategy（演算子戦略番号）
  • PG_GETARG_POINTER(4): bool *recheck（再チェックフラグ）
  • **出力**:
  • Datum: 判定結果（真偽値）。

前提条件と不変条件

  • 本関数が提供するすべての判定は「完全（exact）」であるため、*recheck は常に false に設定される。

アルゴリズムまたは振る舞い

1. recheck フラグを false に初期化する。 2. GIST_LEAF(entry) を用いて、エントリがリーフノードか内部ノードかを判定する。 3. リーフノードの場合は gseg_leaf_consistent を呼び出し、結果を返す。 4. 内部ノードの場合は gseg_internal_consistent を呼び出し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス管理機構（直接の呼び出し元はソースコード上では明示されていない）。
  • **呼び出し先**:
  • gseg_leaf_consistent
  • gseg_internal_consistent

関連データ構造

  • GISTENTRY: GiST インデックスのエントリ構造体。
  • StrategyNumber: 演算子戦略を識別する型。

失敗モードと境界条件

  • 供給されたソースコード内には、明示的なエラーハンドリングや境界条件の記述はない。

保守担当者向けメモ

  • 本関数は GiST インデックスのインターフェースとして機能するディスパッチャーである。具体的な判定ロジックは gseg_leaf_consistent および gseg_internal_consistent に委譲されているため、ロジックの変更が必要な場合はそちらを参照すること。
