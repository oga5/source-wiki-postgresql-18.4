DatumGetNumeric

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/numeric.h:60

目的

PostgreSQLの Datum 型として保持されている数値を、内部表現である Numeric 型（NumericData 構造体へのポインタ）へ変換します。

シグネチャまたは形状

static inline Numeric DatumGetNumeric(Datum X)

入力と出力

  • **入力**: Datum X - 変換対象の Datum 型データ。
  • **出力**: Numeric - NumericData 構造体へのポインタ。

前提条件と不変条件

  • 入力 X は、有効な Numeric 型の Datum である必要があります。
  • PG_DETOAST_DATUM マクロを使用しているため、入力がトースト（圧縮や外部化）されている場合は、必要に応じてメモリ上で展開されます。

アルゴリズムまたは振る舞い

1. 入力された Datum X に対して PG_DETOAST_DATUM を実行し、必要に応じてデトースト処理を行います。 2. 展開されたデータを Numeric 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_numeric.c
  • contrib/hstore/hstore_io.c
  • contrib/jsonb_plperl/jsonb_plperl.c
  • contrib/jsonb_plpython/jsonb_plpython.c
  • src/backend/utils/adt/date.c
  • src/backend/utils/dbsize.c
  • **呼び出し先**:
  • PG_DETOAST_DATUM (マクロ)

関連データ構造

  • Numeric: NumericData 構造体への不透明ポインタ。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • 入力が Numeric 型として解釈できない場合や、メモリ不足によりデトーストに失敗した場合は、PostgreSQLの標準的なエラーハンドリング機構により例外がスローされます。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル経由でインライン展開されます。
  • Numeric の内部構造は隠蔽されているため、本関数で取得したポインタのメンバに直接アクセスせず、numeric.c で提供される公開関数を使用してください。
