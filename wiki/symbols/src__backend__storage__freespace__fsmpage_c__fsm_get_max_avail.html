fsm_get_max_avail

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/fsmpage.c:137

目的

FSM（空き領域マップ）ページ全体における最大の空き容量を取得します。

シグネチャまたは形状

uint8 fsm_get_max_avail(Page page)

入力と出力

  • **入力**: Page page - 操作対象となる FSM ページへのポインタ。
  • **出力**: uint8 - ページ内の全スロットにおける最大空き容量。

前提条件と不変条件

  • 入力された page は有効な FSM ページ構造である必要があります。
  • ページ内のバイナリツリー構造が整合性を保っていることを前提とします。

アルゴリズムまたは振る舞い

1. PageGetContents(page) を介して、ページ内の FSM 構造体（FSMPage）を取得します。 2. FSM ページのバイナリツリーのルートノードである fp_nodes[0] に格納されている値を返します。この値は、ツリー全体で保持されている最大値です。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/freespace/freespace.c:711
  • src/backend/storage/freespace/freespace.c:914
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行わず、マクロ PageGetContents を使用してページ内容にアクセスします。

関連データ構造

  • FSMPage: storage/fsm_internals.h で定義される、FSM ページ内のノード配列を保持する構造体。
  • Page: PostgreSQL の標準的なページ構造。

失敗モードと境界条件

  • 本関数は単一バイトの読み取りを行うため、ロックは不要です。
  • ページが破損している場合、fp_nodes[0] に格納されている値が実際の最大空き容量と一致しない可能性があります。

保守担当者向けメモ

  • 本関数は非常に軽量なアクセサです。
  • ページ内のツリー構造が更新された際、ルートノード（fp_nodes[0]）が常に最新の最大値を保持していることが、他の関数（fsm_set_avail 等）によって保証されている必要があります。
