A_Expr

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/parsenodes.h:345

目的

SQL構文解析の初期段階において、演算子を用いた式（例: a + b や x = ANY(y)）を表現するためのノード構造体です。

シグネチャまたは形状

typedef struct A_Expr {
    NodeTag     type;
    A_Expr_Kind kind;
    List       *name;
    Node       *lexpr;
    Node       *rexpr;
    ParseLoc    rexpr_list_start;
    ParseLoc    rexpr_list_end;
    ParseLoc    location;
} A_Expr;

入力と出力

  • **入力**: パーサー（gram.y）がSQL文から生成する各要素。
  • **出力**: 解析器（parse_analyze）へ渡されるパースツリーの一部として機能します。

前提条件と不変条件

  • type フィールドは T_A_Expr である必要があります。
  • location はSQL文字列内のバイトオフセットを指し、不明な場合は -1 となります。
  • rexpr がリスト形式の場合、rexpr_list_start および rexpr_list_end によってその範囲が追跡されます。

アルゴリズムまたは振る舞い

  • パーサーによって生成された後、意味解析の過程で演算子の解決や型チェックが行われます。
  • pg_node_attr(custom_read_write) 属性により、ノードの読み書きにカスタム処理が適用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/makefuncs.c にてノードの生成が行われています。また、src/backend/parser/parse_clause.c 等で利用されます。
  • **呼び出し先**: src/backend/nodes/nodeFuncs.c（ノード操作）、src/backend/nodes/outfuncs.c（シリアライズ）、src/backend/nodes/readfuncs.c（デシリアライズ）等で参照・処理されます。

関連データ構造

  • A_Expr_Kind: 式の種類を定義する列挙型。
  • Node: 汎用的なノード基底型。
  • List: 演算子名やリスト形式の右辺を保持するために使用。

失敗モードと境界条件

  • rexpr がリストでない場合、rexpr_list_start/end の値は意味を持ちません。
  • 不明な A_Expr_Kind が指定された場合、readfuncs.c 等でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は「生のパースツリー」の一部であり、解析後のクエリツリー（Expr ノード等）とは区別されます。
  • location フィールドはエラーメッセージのカーソル位置表示に利用されるため、パーサー修正時には正確な値を設定してください。
