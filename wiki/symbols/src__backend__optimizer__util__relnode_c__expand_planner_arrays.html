expand_planner_arrays

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/relnode.c:162

目的

プランナが保持するリレーション管理用配列（simple_rel_array 等）のサイズを動的に拡張し、新しいリレーションや継承関係の情報を格納するための領域を確保する。

シグネチャまたは形状

void expand_planner_arrays(PlannerInfo *root, int add_size)

入力と出力

  • **入力**:
  • root: 現在のクエリプランニング状態を保持する PlannerInfo 構造体へのポインタ。
  • add_size: 追加する配列要素数（正の整数である必要がある）。
  • **出力**: なし（root が指す構造体内の配列が再割り当てされ、サイズ情報が更新される）。

前提条件と不変条件

  • add_size > 0 であること（Assert により検証される）。
  • root->simple_rel_array_size が現在の配列の有効なサイズを保持していること。

アルゴリズムまたは振る舞い

1. new_size を現在のサイズに add_size を加算して算出する。 2. root->simple_rel_array および root->simple_rte_array を repalloc0_array を用いて新しいサイズに拡張し、追加領域をゼロ初期化する。 3. root->append_rel_array が既に存在する場合は repalloc0_array で拡張し、存在しない場合は palloc0_array で新規確保する。 4. 最後に root->simple_rel_array_size を new_size に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/inherit.c:184
  • src/backend/optimizer/util/inherit.c:363
  • **呼び出し先**:
  • repalloc0_array（メモリ再割り当て）
  • palloc0_array（メモリ新規確保）

関連データ構造

  • PlannerInfo: プランナのグローバル状態を保持する構造体。
  • RelOptInfo: リレーションノード。
  • RangeTblEntry: レンジテーブルエントリ。
  • AppendRelInfo: 継承やパーティションに関する情報。

失敗モードと境界条件

  • メモリ不足時には palloc/repalloc 経由でエラーが発生する。
  • add_size に 0 以下が渡された場合、Assert が有効なビルドではアボートする。

保守担当者向けメモ

  • 本関数は、継承テーブルやパーティションテーブルの展開時に、プランナが認識すべきリレーション数が増加する際呼び出される。
  • append_rel_array の初期化ロジックは、既存の有無によって動的に切り替わるため、呼び出し順序に注意すること。
