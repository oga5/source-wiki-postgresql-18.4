read_block

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/reconstruct.c:776

目的

再構築対象のソースファイルから、指定されたオフセット位置にある単一のデータブロックを読み込み、提供されたバッファへ格納します。

シグネチャまたは形状

static void read_block(rfile *s, off_t off, uint8 *buffer)

入力と出力

  • **入力**:
  • s: 読み込み対象のソースファイル情報を保持する rfile 構造体へのポインタ。
  • off: 読み込みを開始するファイル内のオフセット。
  • buffer: 読み込んだデータを格納する uint8 型のバッファ（BLCKSZ 分の容量が必要）。
  • **出力**: なし（成功時はバッファを更新、失敗時は pg_fatal によりプロセスを終了）。

前提条件と不変条件

  • s->fd が有効なファイル記述子であること。
  • buffer が少なくとも BLCKSZ バイトのメモリ領域を確保していること。

アルゴリズムまたは振る舞い

1. pg_pread を使用し、s->fd の指定オフセット off から BLCKSZ バイトを buffer に読み込みます。 2. 読み込みバイト数が BLCKSZ と一致しない場合、エラー処理を行います。

  • 読み込み結果が負（エラー）の場合、pg_fatal でエラーメッセージを出力し終了します。
  • 読み込みバイト数が BLCKSZ 未満の場合、読み込めたバイト数と期待値を報告し、pg_fatal で終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_combinebackup/reconstruct.c 内の write_reconstructed_file 関数（685行目、720行目付近）。
  • **呼び出し先**: pg_pread（システムコール）。

関連データ構造

  • rfile: ソースファイルのメタデータおよびファイル記述子を保持する構造体。
  • BLCKSZ: storage/block.h で定義されるブロックサイズ定数。

失敗モードと境界条件

  • ファイルの読み込みエラー（pg_pread が負を返す）が発生した場合、%m を含むエラーメッセージと共に異常終了します。
  • 指定されたオフセットから BLCKSZ 分のデータが読み込めない（EOF に達するなど）場合、読み込み不足として異常終了します。

保守担当者向けメモ

  • 本関数は dry-run モードの判定ロジックを含んでいません（ソースコード上のコメントには言及がありますが、実装上は常に読み込みを試みます）。
  • pg_pread を使用しているため、ファイルポインタの状態に依存せず、スレッドセーフな読み込みが可能です。
