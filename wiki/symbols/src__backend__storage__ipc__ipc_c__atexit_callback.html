atexit_callback

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/ipc.c:300

目的

プロセスが意図せず exit() を呼び出した際に、PostgreSQLのクリーンアップ処理を強制的に実行するためのバックストップ（安全装置）として機能します。

シグネチャまたは形状

static void atexit_callback(void)

入力と出力

  • 入力: なし
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • 本関数は atexit() を通じて登録されます。
  • 終了コードを特定できない環境で呼び出されるため、内部的に固定値 -1 を使用します。

アルゴリズムまたは振る舞い

1. proc_exit_prepare(-1) を呼び出します。 2. これにより、終了処理中フラグのセットや割り込みの無効化など、プロセス終了に必要な準備処理が実行されます。

呼び出し元と呼び出し先

  • 呼び出し元: C標準ライブラリの exit() が呼び出された際、atexit() に登録されたハンドラとして自動的に呼び出されます。
  • 呼び出し先: proc_exit_prepare

関連データ構造

  • なし

失敗モードと境界条件

  • 終了コードが不明な状態で呼び出されるため、常に -1 を引数として渡します。
  • atexit() の仕様上、本関数はプロセス終了シーケンスの最終段階で実行されます。

保守担当者向けメモ

  • 本関数は proc_exit() を直接呼び出すのではなく、proc_exit_prepare() を呼び出すことで、終了処理の整合性を保っています。
  • atexit() への登録は、ipc.c 内の複数の箇所で行われています。
