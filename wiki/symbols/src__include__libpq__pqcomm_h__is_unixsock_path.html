is_unixsock_path

定義

  • 種別: static inline function
  • 定義箇所: src/include/libpq/pqcomm.h:66

目的

与えられた文字列がUnixドメインソケットのパスとして有効な形式であるかを判定します。

シグネチャまたは形状

static inline bool is_unixsock_path(const char *path)

入力と出力

  • **入力**: path (Unixドメインソケットのパスを示す文字列へのポインタ)
  • **出力**: bool (パスが絶対パスであるか、または抽象名前空間ソケットを示す @ で始まる場合に true を返す)

前提条件と不変条件

  • path はヌル終端文字列である必要があります。
  • is_absolute_path() 関数が別途定義されている環境で利用可能です。

アルゴリズムまたは振る舞い

1. is_absolute_path(path) を呼び出し、パスが絶対パス形式（/ で始まる等）であるかを確認します。 2. または、path[0] == '@' を評価し、Linuxの抽象名前空間（abstract namespace）ソケットであるかを確認します。 3. いずれかの条件が真であれば true を返し、それ以外は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/server.c
  • src/bin/psql/command.c
  • src/bin/psql/prompt.c
  • src/interfaces/libpq/fe-connect.c
  • **呼び出し先**:
  • is_absolute_path() (外部定義)

関連データ構造

  • sockaddr_un (Unixドメインソケットのアドレス構造体に関連)

失敗モードと境界条件

  • path が NULL の場合、未定義の動作（セグメンテーション違反）を引き起こす可能性があります。
  • 空文字列が渡された場合、is_absolute_path の実装に依存しますが、通常は false を返します。

保守担当者向けメモ

  • 本関数は static inline で定義されているため、ヘッダーをインクルードする各翻訳単位に展開されます。
  • 抽象名前空間ソケット（@ で始まるパス）のサポートは、プラットフォームがこれをサポートしている場合にのみ有効に機能します。
