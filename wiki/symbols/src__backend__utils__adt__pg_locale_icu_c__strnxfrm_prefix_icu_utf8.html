strnxfrm_prefix_icu_utf8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_icu.c:538

目的

UTF-8エンコーディング環境において、ICUの照合順序に基づいた文字列のソートキーの一部を生成します。

シグネチャまたは形状

size_t strnxfrm_prefix_icu_utf8(char *dest, size_t destsize, const char *src, ssize_t srclen, pg_locale_t locale)

入力と出力

  • **入力**:
  • dest: ソートキーを格納するバッファ。
  • destsize: dest バッファのサイズ。
  • src: 変換対象のUTF-8文字列。
  • srclen: src の長さ。
  • locale: ICUプロバイダーを含むロケール情報。
  • **出力**:
  • 戻り値: 生成されたソートキーのバイト数。

前提条件と不変条件

  • locale->provider が COLLPROVIDER_ICU であること。
  • データベースのエンコーディングが PG_UTF8 であること。

アルゴリズムまたは振る舞い

1. uiter_setUTF8 を使用して、入力文字列 src をICUの UCharIterator に設定します。 2. ucol_nextSortKeyPart を呼び出し、ICUの照合順序（locale->info.icu.ucol）に基づいてソートキーを生成し、dest バッファに書き込みます。 3. ICUの処理結果（status）を検証し、エラーが発生した場合は ereport(ERROR, ...) を発行してトランザクションを中断します。 4. 生成されたソートキーのサイズを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale_icu.c 内のメソッドテーブル collate_methods_icu_utf8 で参照されています。
  • **呼び出し先**:
  • uiter_setUTF8 (ICU)
  • ucol_nextSortKeyPart (ICU)
  • ereport (PostgreSQL)

関連データ構造

  • pg_locale_t: ロケールおよび照合順序情報を保持する構造体。
  • UCharIterator: ICUの文字列反復子。
  • UErrorCode: ICUの処理状態を示す列挙型。

失敗モードと境界条件

  • ICUのソートキー生成処理が失敗した場合、ereport によりエラーが報告されます。
  • destsize が不足している場合、ucol_nextSortKeyPart の仕様に従い、必要なバッファサイズが戻り値として返されます。

保守担当者向けメモ

  • 本関数はUTF-8環境専用の最適化実装です。
  • ICUのバージョン差異による挙動の変化については、pg_locale_icu.c 全体のマクロ定義やバージョンチェックの文脈を確認してください。
