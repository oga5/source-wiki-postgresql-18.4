pg_clock_gettime_ns

定義

  • 種別: static inline function
  • 定義箇所: src/include/portability/instr_time.h:110

目的

プラットフォーム固有の clock_gettime を利用して、現在時刻をナノ秒単位の instr_time 構造体として取得する。

シグネチャまたは形状

static inline instr_time pg_clock_gettime_ns(void)

入力と出力

  • 入力: なし
  • 出力: instr_time 構造体（内部の ticks メンバにナノ秒単位の時刻が格納される）

前提条件と不変条件

  • <time.h> がインクルードされていること。
  • PG_INSTR_CLOCK がシステムで定義された有効なクロック識別子であること。
  • NS_PER_S 定数がナノ秒から秒への変換係数として定義されていること。

アルゴリズムまたは振る舞い

1. struct timespec 型の変数 tmp を宣言する。 2. clock_gettime(PG_INSTR_CLOCK, &tmp) を呼び出し、システム時刻を取得する。 3. tmp.tv_sec * NS_PER_S + tmp.tv_nsec の計算により、秒とナノ秒をナノ秒単位の単一の int64 値（ticks）に変換する。 4. 計算結果を保持した instr_time 構造体を返す。

呼び出し元と呼び出し先

  • 呼び出し先: clock_gettime (システムライブラリ関数)
  • 呼び出し元:
  • src/include/portability/instr_time.h 内の INSTR_TIME_SET_CURRENT マクロ定義において、現在時刻の代入に使用されている。

関連データ構造

  • instr_time: 時間を保持する不透明な構造体。
  • struct timespec: clock_gettime が使用する標準的な時間構造体。

失敗モードと境界条件

  • clock_gettime が失敗した場合の挙動については、本関数内では明示的なエラーハンドリングを行っていない。
  • ticks の計算において、tv_sec が非常に大きい場合や NS_PER_S との積が int64 の範囲を超える場合のオーバーフローについては、本ソースコード内では考慮されていない。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダファイル内で完結している。
  • PG_INSTR_CLOCK の定義がプラットフォームごとに適切であることを確認すること。
  • instr_time 構造体の内部メンバ ticks に直接依存しているため、構造体の定義変更時には注意が必要である。
