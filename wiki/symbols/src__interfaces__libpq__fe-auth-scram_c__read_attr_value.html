read_attr_value

定義

  • **種別**: function
  • **定義箇所**: src/interfaces/libpq/fe-auth-scram.c:307

目的

SCRAM 認証メッセージ文字列から、指定された属性キーに対応する値（key=value 形式の value 部分）を抽出します。

シグネチャまたは形状

char *read_attr_value(char **input, char attr, PQExpBuffer errorMessage);

入力と出力

  • **入力**:
  • input: 解析対象の文字列へのポインタ。解析が進むと、次の属性の開始位置までポインタが更新されます。
  • attr: 検索対象の属性キー（1文字）。
  • errorMessage: エラー発生時に詳細を記録するためのバッファ。
  • **出力**:
  • 成功時: 抽出された値の先頭アドレスを返します。
  • 失敗時: NULL を返し、errorMessage にエラー内容を追記します。

前提条件と不変条件

  • input が指す文字列は、key=value 形式がカンマ区切りで並んでいることを前提とします。
  • この関数は input が指す文字列を破壊的に変更します（値の末尾に \0 を挿入します）。

アルゴリズムまたは振る舞い

1. input の現在位置が期待される attr と一致するか確認し、一致しなければエラーを返します。 2. 次に = 文字が存在するか確認し、なければエラーを返します。 3. = の直後から、文字列の終端または次のカンマ（,）までの範囲を値として特定します。 4. カンマが見つかった場合、その位置を \0 で置換して文字列を分割し、input ポインタを次の属性の先頭へ進めます。 5. 抽出した値の先頭アドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-auth-scram.c 内の各メッセージ解析関数（read_server_first_message 等の文脈で利用）。
  • **呼び出し先**: libpq_append_error（エラー報告用）。

関連データ構造

  • PQExpBuffer: エラーメッセージを保持するバッファ。

失敗モードと境界条件

  • **属性不一致**: 指定した attr が現在の位置にない場合、エラーとなります。
  • **形式不正**: = が欠落している場合、エラーとなります。

保守担当者向けメモ

  • input 文字列は破壊的に変更されるため、呼び出し側で元の文字列を保持する必要がある場合は注意してください。
  • サーバー側の実装（src/backend/libpq/auth-scram.c）でも同名の関数が使用されていますが、引数構成が異なるため混同しないよう注意してください。
