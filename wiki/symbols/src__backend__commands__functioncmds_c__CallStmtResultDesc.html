CallStmtResultDesc

定義

  • 種別: function
  • 定義箇所: src/backend/commands/functioncmds.c:2382

目的

CALL 文によって実行されるプロシージャの戻り値の型記述子（TupleDesc）を生成・調整する。

シグネチャまたは形状

TupleDesc CallStmtResultDesc(CallStmt *stmt)

入力と出力

  • **入力**: CallStmt *stmt（実行対象のプロシージャ情報を含む構文木）
  • **出力**: TupleDesc（プロシージャの出力引数に対応するタプル記述子。出力引数がない場合は NULL）

前提条件と不変条件

  • stmt->funcexpr が有効な FuncExpr を指していること。
  • PROCOID システムキャッシュから対象プロシージャの情報を取得できること。
  • tupdesc の列数は stmt->outargs のリスト長と一致すること（Assert で検証）。

アルゴリズムまたは振る舞い

1. stmt->funcexpr からプロシージャの OID を取得し、PROCOID システムキャッシュを検索する。 2. build_function_result_tupdesc_t を呼び出し、カタログ情報に基づいた初期 TupleDesc を作成する。 3. キャッシュを解放する。 4. stmt->outargs が存在する場合、多相型（polymorphic types）などの型情報を正確に反映させるため、各列の型を exprType(outarg) で上書き更新する。

  • atttypmod は -1、attcollation は型のデフォルト値に固定される。

    • 5. 最終的な TupleDesc を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c:2089
  • **呼び出し先**:
  • SearchSysCache1
  • build_function_result_tupdesc_t
  • ReleaseSysCache
  • TupleDescAttr
  • TupleDescInitEntry
  • exprType

関連データ構造

  • CallStmt: 実行されるプロシージャの式と引数リストを保持するノード。
  • TupleDesc: プロシージャの出力引数の構造を定義する記述子。
  • pg_proc: プロシージャのメタデータが格納されるシステムカタログ。

失敗モードと境界条件

  • プロシージャの OID がシステムキャッシュに見つからない場合、elog(ERROR) により処理を中断する。
  • 出力引数がない場合、tupdesc は NULL となり、そのまま NULL を返す。

保守担当者向けメモ

  • 本関数は、カタログ上の宣言型と、実行時の outargs に含まれる実際の型情報の乖離を埋める役割を持つ。
  • atttypmod や attcollation を固定値で上書きしているため、将来的にプロシージャの出力引数でこれらを動的に扱う必要が生じた場合は修正が必要となる。
