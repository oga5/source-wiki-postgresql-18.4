sql_exec_searchtables

定義

  • 種別: function
  • 定義箇所: contrib/oid2name/oid2name.c:500

目的

ユーザーが指定したOID、テーブル名、またはファイルノード番号に基づき、PostgreSQLのシステムカタログから該当するデータベースオブジェクト情報を検索し、結果を表示する。

シグネチャまたは形状

void sql_exec_searchtables(PGconn *conn, struct options *opts)

入力と出力

  • **入力**:
  • conn: データベース接続を保持する PGconn ポインタ。
  • opts: 検索条件（OID、テーブル名、ファイルノード）および出力オプションを保持する options 構造体。
  • **出力**: なし（内部で sql_exec を呼び出し、結果を標準出力へ表示する）。

前提条件と不変条件

  • opts 内の検索条件（oids, tables, filenumbers）は、get_comma_elts で処理可能な形式であること。
  • データベース接続 conn は確立済みであること。

アルゴリズムまたは振る舞い

1. get_comma_elts を使用して、opts 内の各検索条件をカンマ区切りの文字列に変換する。 2. 変換された文字列を結合し、SQLの WHERE 句で使用する qualifiers 文字列を構築する。 3. pg_class, pg_namespace, pg_database, pg_tablespace を結合した SELECT 文を生成する。

  • relkind がテーブル、マテリアライズドビュー、インデックス、シーケンス、TOAST値であるものに限定する。
  • opts->extended が真の場合、OID、スキーマ名、テーブルスペース名を追加で取得する。

    • 4. 生成したSQLクエリを sql_exec に渡し、実行および結果表示を行う。 5. 構築に使用した一時的なメモリ（qualifiers, comma_oids 等）を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/oid2name/oid2name.c:626（main 関数内と推測される箇所）。
  • **呼び出し先**:
  • get_comma_elts
  • pg_malloc
  • sprintf
  • psprintf
  • free
  • sql_exec

関連データ構造

  • struct options: 検索条件および実行オプションを保持。
  • struct eary: 検索対象のリストを管理する動的配列。

失敗モードと境界条件

  • メモリ不足時: pg_malloc または psprintf 内部でエラーが発生する可能性がある。
  • 検索条件が空の場合: qualifiers が空の状態でSQLが構築される可能性がある（SQL構文エラーの可能性がある）。
  • データベース接続エラー: sql_exec 内部で処理される。

保守担当者向けメモ

  • 本関数は pg_class の relkind をハードコードしているため、将来的に新しいリレーションタイプをサポートする場合はこのリストを更新する必要がある。
  • qualifiers のバッファサイズ計算（+ 80）は、クエリの複雑化に伴い不足しないよう注意が必要。
