serializeAnalyzeDestroy

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_dr.c:264

目的

EXPLAIN (SERIALIZE) オプションで使用される DestReceiver オブジェクトを破棄し、関連するメモリを解放する。

シグネチャまたは形状

static void serializeAnalyzeDestroy(DestReceiver *self)

入力と出力

  • **入力**: self (DestReceiver *) - 破棄対象のレシーバー構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • self は CreateExplainSerializeDestReceiver によって割り当てられた有効なメモリ領域を指している必要がある。
  • 本関数呼び出し後、self が指すメモリ領域は無効となる。

アルゴリズムまたは振る舞い

1. pfree(self) を呼び出し、レシーバー構造体自体が使用していたメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/explain_dr.c 内の DestReceiver インターフェース経由で呼び出される（self->pub.rDestroy に本関数が登録されている）。
  • **呼び出し先**: pfree（PostgreSQL のメモリ管理関数）。

関連データ構造

  • DestReceiver: 本関数が破棄対象とする基底構造体。

失敗モードと境界条件

  • self が NULL の場合、pfree の仕様に依存する（通常はセグメンテーション違反となる可能性がある）。

保守担当者向けメモ

  • 本関数は DestReceiver の標準的な破棄インターフェースを実装している。
  • SerializeDestReceiver 構造体内部で追加のメモリ確保を行っている場合、pfree(self) の前にそれらの解放処理を追加する必要があるが、現状のソース excerpt では self 自体の解放のみが行われている。
