should_allow_existing_directory

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_file.c:308

目的

バックアップ展開時に、既にファイルシステム上に存在していてもエラーとみなすべきではない特定のディレクトリパスを判定します。

シグネチャまたは形状

static bool should_allow_existing_directory(const char *pathname);

入力と出力

  • **入力**: pathname (検証対象のディレクトリパス文字列)
  • **出力**: true (許容する場合)、false (許容しない場合)

前提条件と不変条件

  • pathname は有効な文字列ポインタである必要があります。
  • last_dir_separator 関数が pathname 内のディレクトリ区切り文字を正しく特定できることを前提としています。

アルゴリズムまたは振る舞い

1. pathname の末尾のディレクトリ名（filename）を抽出します。 2. filename が以下のいずれかの固定名称と一致する場合、true を返します。

  • pg_wal, pg_xlog, archive_status, summaries, pg_tblspc

    • 3. filename が数値のみで構成されている場合、その親ディレクトリが /pg_tblspc/ であるかを確認し、該当すれば true を返します。 4. 上記いずれにも該当しない場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/astreamer_file.c 内のディレクトリ作成処理において、EEXIST エラーが発生した際のガード条件として呼び出されます。
  • **呼び出し先**: last_dir_separator, strcmp, strspn, strlen, strstr (標準ライブラリおよびPostgreSQLユーティリティ関数)。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • pathname にディレクトリ区切り文字が含まれない場合、last_dir_separator の戻り値に依存した未定義の動作が発生する可能性があります。
  • 数値のみのディレクトリ名（テーブルスペースIDなど）は、pg_tblspc 配下にある場合のみ許容されます。

保守担当者向けメモ

  • 本関数はバックアップ展開時の安全性を担保する重要なガードロジックです。新しいシステムディレクトリを許容する必要が生じた場合は、固定名称リストへの追加を検討してください。
