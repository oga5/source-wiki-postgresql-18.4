ExecScanSubPlan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSubplan.c:203

目的

ハッシュ化戦略が適用できないサブクエリ（SubPlan）に対し、クエリ実行のたびにサブプランを再スキャンして評価を行う。

シグネチャまたは形状

static Datum ExecScanSubPlan(SubPlanState *node, ExprContext *econtext, bool *isNull)

入力と出力

  • **入力**:
  • node: 実行対象の SubPlanState。
  • econtext: 式評価コンテキスト。
  • **出力**:
  • 戻り値: サブクエリの評価結果（Datum）。
  • isNull: 結果が NULL かどうかを示すフラグ。

前提条件と不変条件

  • econtext->ecxt_per_query_memory が有効であること。
  • subplan->parParam に含まれるパラメータが変更された場合、planstate->chgParam に反映されていること。
  • EXPR_SUBLINK および MULTIEXPR_SUBLINK では、サブクエリが複数行を返すとエラー（ERRCODE_CARDINALITY_VIOLATION）となる。

アルゴリズムまたは振る舞い

1. **初期化**: ARRAY_SUBLINK の場合、initArrayResultAny を呼び出し配列構築状態を準備する。 2. **コンテキスト切替**: econtext->ecxt_per_query_memory へメモリコンテキストを切り替える。 3. **再スキャン準備**: subplan->parParam を走査し、親プランのパラメータ変更を planstate->chgParam に記録した後、ExecReScan(planstate) を実行する。 4. **反復評価**: ExecProcNode(planstate) を用いてサブクエリのタプルを順次取得する。

  • EXISTS_SUBLINK: 1行でも取得できれば true を返し終了。
  • EXPR_SUBLINK: 取得したタプルの第1カラムを結果とし、複数行取得時はエラーとする。
  • MULTIEXPR_SUBLINK: 取得したタプルの各カラムを setParam にセットし、複数行取得時はエラーとする。
  • ARRAY_SUBLINK: 取得したタプルの第1カラムを配列に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecSubPlan（ソースコード内の参照より）。
  • **呼び出し先**: ExecProcNode, ExecReScan, initArrayResultAny, accumArrayResultAny, heap_getattr, ExecCopySlotHeapTuple, heap_freetuple 等。

関連データ構造

  • SubPlanState: サブプランの実行状態を保持。
  • SubPlan: サブプランの定義情報。
  • ExprContext: 式評価のコンテキスト。
  • TupleTableSlot: サブクエリから返されるタプルを保持。

失敗モードと境界条件

  • **基数違反**: EXPR_SUBLINK, MULTIEXPR_SUBLINK, ROWCOMPARE_SUBLINK において、サブクエリが複数行を返すと ERROR を発生させる。
  • **メモリ管理**: EXPR_SUBLINK 等で pass-by-ref 型の結果を扱う際、ExecCopySlotHeapTuple でコピーを作成し、node->curTuple で管理することで生存期間を保証する。

保守担当者向けメモ

  • 本関数はハッシュ化されないサブクエリのフォールバックとして機能する。
  • ARRAY_SUBLINK の挙動は 8.3 以降、結果が 0 行の場合に NULL ではなく空配列を返すように変更されている点に注意。
  • node->curTuple は、EXPR_SUBLINK や MULTIEXPR_SUBLINK でのメモリリークを防ぐために使用されている。
