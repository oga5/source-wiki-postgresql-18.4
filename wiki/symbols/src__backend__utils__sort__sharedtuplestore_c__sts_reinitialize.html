sts_reinitialize

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/sharedtuplestore.c:233

目的

共有タプルストアにおける全参加者の読み取り位置をリセットし、並列スキャンを再開可能な状態にする。

シグネチャまたは形状

void sts_reinitialize(SharedTuplestoreAccessor *accessor)

入力と出力

  • **入力**: SharedTuplestoreAccessor *accessor - 操作対象の共有タプルストアへのアクセス権を持つ構造体。
  • **出力**: なし（副作用として共有メモリ上の状態を更新）。

前提条件と不変条件

  • accessor が有効な SharedTuplestore を指していること。
  • 共有メモリ上の participants 配列が初期化済みであること。

アルゴリズムまたは振る舞い

1. accessor->sts->nparticipants で定義される全参加者に対してループ処理を行う。 2. 各参加者の read_page を 0 にリセットする。 3. これにより、次回スキャン時に各参加者のファイルの先頭から読み取りが開始される状態となる。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は特定されていません。
  • **呼び出し先**: 共有メモリ構造体 accessor->sts への直接アクセスのみであり、他の関数呼び出しは行いません。

関連データ構造

  • SharedTuplestoreAccessor: 操作の起点となる構造体。
  • SharedTuplestore: 共有メモリ上に配置されるメインの制御構造体。
  • SharedTuplestoreParticipant: 各参加者の状態（read_page を含む）を保持する構造体。

失敗モードと境界条件

  • accessor が NULL の場合、あるいは accessor->sts が初期化されていない場合はセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は読み取り位置のみをリセットします。書き込み済みのファイルデータやチャンクのメタデータは破棄されません。
  • 並列スキャン中に本関数を呼び出す場合、他のプロセスとの同期（ロック）が適切に管理されていることを確認してください。
