WalSyncMethod

定義

  • 種別: enum
  • 定義箇所: src/include/access/xlog.h:22

目的

WAL（先行書き込みログ）データを物理ディスクへ永続化する際に使用する同期手法を定義します。

シグネチャまたは形状

enum WalSyncMethod
{
    WAL_SYNC_METHOD_FSYNC = 0,
    WAL_SYNC_METHOD_FDATASYNC,
    WAL_SYNC_METHOD_OPEN,        /* for O_SYNC */
    WAL_SYNC_METHOD_FSYNC_WRITETHROUGH,
    WAL_SYNC_METHOD_OPEN_DSYNC   /* for O_DSYNC */
};

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし（列挙型定義）

前提条件と不変条件

  • 各列挙値は、OS およびファイルシステムがサポートする特定の同期プリミティブに対応します。
  • システムの構成に応じて、適切な同期手法が選択されます。

アルゴリズムまたは振る舞い

  • WAL_SYNC_METHOD_FSYNC: 標準的な fsync() システムコールを使用します。
  • WAL_SYNC_METHOD_FDATASYNC: fdatasync() を使用し、メタデータの更新を最小限に抑えます。
  • WAL_SYNC_METHOD_OPEN: ファイルオープン時に O_SYNC フラグを使用します。
  • WAL_SYNC_METHOD_FSYNC_WRITETHROUGH: ライトスルーキャッシュを強制する fsync 関連の挙動を指します。
  • WAL_SYNC_METHOD_OPEN_DSYNC: ファイルオープン時に O_DSYNC フラグを使用します。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数情報は、提供されたソース抜粋には含まれていません。

関連データ構造

  • src/include/access/xlog.h 内で定義される WAL 管理システム全般。

失敗モードと境界条件

  • OS が指定された同期手法をサポートしていない場合、システム起動時や設定変更時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本シンボルは WAL の耐久性を決定する重要な設定値です。新しい同期手法を追加または変更する場合は、src/backend/access/transam/xlog.c における実装との整合性を確認してください。
