r_standard_suffix

定義

  • **種別**: function
  • **定義箇所**: src/backend/snowball/libstemmer/stem_ISO_8859_1_portuguese.c:638

目的

ポルトガル語のステミング処理において、単語末尾の標準的な接尾辞を特定し、削除または置換を行う。

シグネチャまたは形状

static int r_standard_suffix(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（Snowball ランタイム環境構造体）
  • **出力**: int（成功時に 1、失敗または条件不一致時に 0、エラー時に負の値を返す）

前提条件と不変条件

  • z は有効な SN_env 構造体を指していること。
  • 単語内の領域（R1, R2）が事前に r_mark_regions 等で計算されていること。
  • 処理対象の単語末尾が a_5 配列に定義された接尾辞パターンに一致する必要がある。

アルゴリズムまたは振る舞い

1. z->ket を現在のカーソル位置に設定し、末尾の文字が特定の条件を満たすか確認する。 2. find_among_b を使用して、後方から a_5 配列内の接尾辞を検索する。 3. マッチした接尾辞の種類（among_var）に応じて以下の処理を行う：

  • **ケース 1-4, 6-8**: r_R2 を確認し、条件を満たせば接尾辞を削除または指定文字列へ置換する。
  • **ケース 5**: r_R1 を確認して削除後、さらに特定の接尾辞（a_2）を再帰的に検索・削除する。
  • **ケース 9**: r_RV を確認し、末尾の 'e' を削除して置換する。

    • 4. 成功時は 1 を返し、各ステップでエラーが発生した場合はその値を伝播させる。

呼び出し元と呼び出し先

  • **呼び出し元**: portuguese_ISO_8859_1_stem（直接の呼び出し関係はソースコードの構造から推測されるが、本抜粋内では明示されていない）。
  • **呼び出し先**: find_among_b, r_R1, r_R2, slice_del, slice_from_s, eq_s_b。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_5 (および a_2, a_3, a_4): 接尾辞マッチング用の among 構造体配列。

失敗モードと境界条件

  • 単語が短すぎる場合や、領域（R1, R2）の条件を満たさない場合は 0 を返して終了する。
  • slice_del や slice_from_s が失敗した場合（負の戻り値）、そのエラーコードをそのまま返す。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードである。ロジックの変更が必要な場合は、直接編集せず元の .sbl スクリプトを修正すること。
  • among_var の各ケースにおける r_R2 等のチェックは、ポルトガル語の文法規則に基づいた領域制約を強制している。
