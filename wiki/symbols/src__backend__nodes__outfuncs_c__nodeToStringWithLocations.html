nodeToStringWithLocations

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/outfuncs.c:810

目的

PostgreSQLのノードツリーをASCII文字列形式に変換します。特に、パース時の位置情報（location）を含めたシリアライズを行うための公開APIです。

シグネチャまたは形状

char *nodeToStringWithLocations(const void *obj);

入力と出力

  • **入力**: obj (シリアライズ対象のノード構造体へのポインタ)
  • **出力**: ノードの文字列表現を格納した動的に割り当てられた char * 型の文字列。

前提条件と不変条件

  • obj は有効なノード構造体である必要があります。
  • 内部で nodeToStringInternal を呼び出し、位置情報を含めるフラグを true に固定して処理します。

アルゴリズムまたは振る舞い

1. 引数 obj を受け取り、nodeToStringInternal(obj, true) を呼び出します。 2. nodeToStringInternal は、ノードのタグに基づいて再帰的にノードを走査し、各フィールドを StringInfo バッファに書き出します。 3. 位置情報を含める設定（true）により、ノード内の location フィールドがシリアライズ結果に含まれます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/print.c (デバッグ出力用)
  • src/backend/tcop/postgres.c (クエリ解析結果やプランのシリアライズ用)
  • **呼び出し先**:
  • nodeToStringInternal (直接の内部処理委譲)

関連データ構造

  • StringInfo: 文字列構築に使用されるバッファ構造体。
  • Node: シリアライズ対象となるPostgreSQLの内部ノード。

失敗モードと境界条件

  • メモリ不足時には、内部の StringInfo 構築プロセスでエラーが発生する可能性があります。
  • 循環参照を持つノード構造体に対しては、再帰的な走査によりスタックオーバーフローが発生するリスクがあるため、check_stack_depth() による保護が適用されます。

保守担当者向けメモ

  • 本関数は nodeToString と異なり、位置情報を強制的に出力します。デバッグやパース結果の完全な再現が必要な場合に利用されます。
  • 新しいノード型を追加した際は、outfuncs.switch.c 側の対応が必要です。
