astreamer_gzip_decompressor_new

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_gzip.c:235

目的

gzip形式のデータストリームを解凍するための astreamer インターフェースを初期化し、構築します。

シグネチャまたは形状

astreamer *astreamer_gzip_decompressor_new(astreamer *next);

入力と出力

  • **入力**: next (解凍後のデータを渡す後続の astreamer インターフェースへのポインタ)
  • **出力**: 初期化された astreamer 構造体へのポインタ。失敗時は pg_fatal によりプロセスが終了します。

前提条件と不変条件

  • next は NULL であってはなりません。
  • ビルド時に HAVE_LIBZ が定義されている必要があります。
  • z_stream の解凍状態は inflateInit2 を用いて初期化され、gzipヘッダーを処理するように設定されます。

アルゴリズムまたは振る舞い

1. astreamer_gzip_decompressor 構造体をメモリ確保し、astreamer_gzip_decompressor_ops をメソッドテーブルとして設定します。 2. initStringInfo で内部バッファを初期化します。 3. z_stream のメモリ割り当て関数を gzip_palloc / gzip_pfree に設定します。 4. inflateInit2(zs, 15 + 16) を呼び出し、gzip形式の解凍を有効化します。 5. 成功した場合、構築された astreamer インターフェースを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c:1268
  • src/bin/pg_verifybackup/pg_verifybackup.c:1308
  • **呼び出し先**:
  • palloc0 (メモリ確保)
  • initStringInfo (バッファ初期化)
  • inflateInit2 (zlib解凍初期化)
  • pg_fatal (エラーハンドリング)

関連データ構造

  • astreamer: ストリーミング処理の基底インターフェース。
  • astreamer_gzip_decompressor: 解凍器の内部状態を保持する構造体。
  • z_stream: zlib の解凍状態を管理する構造体。

失敗モードと境界条件

  • HAVE_LIBZ が未定義の場合、実行時に pg_fatal を発生させます。
  • inflateInit2 が Z_OK 以外の値を返した場合、pg_fatal を発生させます。
  • メモリ不足時は palloc0 がエラーを発生させます。

保守担当者向けメモ

  • 本関数は gzFile を直接扱う writer とは異なり、z_stream を介してメモリ上のバッファを処理します。
  • inflateInit2 の引数 15 + 16 は、gzipヘッダーを認識するために必要な設定です。変更時は zlib の仕様を確認してください。
