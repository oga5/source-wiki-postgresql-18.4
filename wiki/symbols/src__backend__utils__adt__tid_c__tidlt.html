tidlt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tid.c:193

目的

2つの TID (Tuple Identifier) を比較し、第1引数が第2引数よりも小さいかどうかを判定する。

シグネチャまたは形状

Datum tidlt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの ItemPointer (TID) を受け取る。
  • **出力**: bool 型の Datum を返す。第1引数が第2引数より小さい場合は true、それ以外は false。

前提条件と不変条件

  • 入力される ItemPointer は有効なメモリ領域を指している必要がある。
  • ItemPointerCompare の仕様に基づき、ブロック番号、次にオフセット番号の順で比較が行われる。

アルゴリズムまたは振る舞い

1. PG_GETARG_ITEMPOINTER マクロを使用して、引数から2つの ItemPointer を取得する。 2. ItemPointerCompare 関数を呼び出し、両者の大小関係を比較する。 3. 比較結果が 0 未満（第1引数が小さい）であれば true を、そうでなければ false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL 演算子 < の実装として呼び出される（直接の呼び出し元情報はソースコード上では明示されていない）。
  • **呼び出し先**: ItemPointerCompare

関連データ構造

  • ItemPointer: TID の内部表現。BlockIdData（ブロック番号）と OffsetNumber（オフセット番号）で構成される。

失敗モードと境界条件

  • ItemPointerCompare の挙動に依存する。無効なポインタが渡された場合の動作は未定義。

保守担当者向けメモ

  • 本関数は TID 型の比較演算子の一部である。比較ロジックを変更する場合は、ItemPointerCompare の実装を確認すること。
