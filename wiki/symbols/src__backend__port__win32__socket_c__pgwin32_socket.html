pgwin32_socket

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/socket.c:290

目的

Windows環境において、POSIX準拠の socket() システムコールをエミュレートし、PostgreSQLが利用するソケットを非ブロッキングモードで初期化します。

シグネチャまたは形状

SOCKET pgwin32_socket(int af, int type, int protocol);

入力と出力

  • **入力**:
  • af: アドレスファミリー（例: AF_INET）。
  • type: ソケットタイプ（例: SOCK_STREAM）。
  • protocol: プロトコル（例: IPPROTO_TCP）。
  • **出力**:
  • 成功時: 作成されたソケットのハンドル（SOCKET 型）。
  • 失敗時: INVALID_SOCKET を返し、内部で TranslateSocketError() を呼び出して errno を設定します。

前提条件と不変条件

  • 作成されるソケットは、常に WSA_FLAG_OVERLAPPED を指定して作成されます。
  • 作成直後に ioctlsocket を用いて FIONBIO（非ブロッキングモード）が有効化されます。

アルゴリズムまたは振る舞い

1. WSASocket を呼び出し、オーバーラップI/O対応のソケットを作成します。 2. 失敗した場合は TranslateSocketError() でエラーコードを変換し、即座に終了します。 3. ioctlsocket を使用して、ソケットを非ブロッキングモードに設定します。 4. 設定に失敗した場合は、作成したソケットを closesocket で解放し、TranslateSocketError() を呼び出した後、INVALID_SOCKET を返します。 5. 成功時は errno を 0 にリセットし、ソケットハンドルを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/win32_port.h にて socket() マクロとして定義されており、PostgreSQLのネットワーク通信を行う各箇所から呼び出されます。
  • **呼び出し先**:
  • WSASocket (Win32 API)
  • ioctlsocket (Win32 API)
  • TranslateSocketError (内部ユーティリティ)
  • closesocket (Win32 API)

関連データ構造

  • SOCKET: Winsock2のソケットハンドル型。

失敗モードと境界条件

  • WSASocket が失敗した場合、TranslateSocketError により適切な errno が設定されます。
  • 非ブロッキング設定（ioctlsocket）に失敗した場合、ソケットはクローズされ、リソースリークを防ぎます。

保守担当者向けメモ

  • 本関数は socket() マクロを通じて呼び出されるため、直接 socket() を呼び出すコードはすべてこの実装を経由します。
  • Windowsのソケットは常に非ブロッキングで開かれますが、PostgreSQLのロジックがブロッキングを期待する場合、他の関数（pgwin32_waitforsinglesocket 等）が内部的に待機処理を管理します。
