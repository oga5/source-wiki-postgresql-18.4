WordBoundaryState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/pg_locale_builtin.c:38

目的

Unicode文字列のタイトルケース変換（strtitle_builtin）において、文字列走査中の現在の位置、単語境界の判定状態、および走査対象のメタデータを保持するために使用されます。

シグネチャまたは形状

struct WordBoundaryState
{
	const char *str;      /* 走査対象の文字列 */
	size_t		len;      /* 文字列の長さ */
	size_t		offset;   /* 現在の走査位置 */
	bool		posix;    /* POSIX準拠フラグ */
	bool		init;     /* 初期化済みフラグ */
	bool		prev_alnum; /* 直前の文字が英数字であったか */
};

入力と出力

本構造体は、タイトルケース変換処理におけるイテレータの状態管理として機能します。入力として文字列情報を受け取り、走査の進捗に応じて offset や prev_alnum が更新されます。

前提条件と不変条件

  • init フラグが true である場合、構造体は有効な走査状態にあるとみなされます。
  • prev_alnum は、initcap_wbnext 関数が単語境界を判定する際の基準として使用されます。

アルゴリズムまたは振る舞い

strtitle_builtin 関数において、この構造体は initcap_wbnext を用いた単語境界検出のコンテキストとして渡されます。initcap_wbnext は、pg_u_isalnum を利用して現在の文字と直前の文字の英数字属性を比較し、単語の区切りを特定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale_builtin.c 内の strtitle_builtin 関数。
  • src/common/unicode/case_test.c 内のテストコード。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。ただし、initcap_wbnext 関数が本構造体のインスタンスを引数として受け取り、その内容を更新します。

関連データ構造

  • initcap_wbnext: 本構造体を操作するイテレータ関数。
  • pg_locale_t: ロケール情報を保持する上位構造体。

失敗モードと境界条件

  • 文字列が空の場合や、offset が len を超える場合の挙動は、initcap_wbnext の実装に依存します。
  • posix フラグの設定により、単語境界の判定ロジックが切り替わる可能性があります。

保守担当者向けメモ

  • 本構造体は pg_locale_builtin.c 内の内部的な状態管理用です。外部モジュールからの直接的な操作は推奨されません。
  • prev_alnum の更新ロジックを変更する場合は、initcap_wbnext の単語境界判定アルゴリズムとの整合性に注意してください。
