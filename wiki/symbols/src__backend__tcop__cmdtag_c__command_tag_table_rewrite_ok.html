command_tag_table_rewrite_ok

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/cmdtag.c:71

目的

指定されたコマンドタグが、テーブルの書き換え（table rewrite）を伴う操作として許可されているかどうかを判定する。

シグネチャまたは形状

bool command_tag_table_rewrite_ok(CommandTag commandTag)

入力と出力

  • **入力**: CommandTag 型の列挙値。
  • **出力**: bool 型。true の場合はそのコマンドがテーブル書き換えを許可されていることを示し、false の場合は許可されていないことを示す。

前提条件と不変条件

  • commandTag は有効な CommandTag 列挙値である必要がある。
  • tag_behavior 配列が初期化済みであること（cmdtaglist.h により静的に定義される）。

アルゴリズムまたは振る舞い

1. 引数として受け取った commandTag をインデックスとして、グローバルな tag_behavior 配列にアクセスする。 2. 対応する CommandTagBehavior 構造体の table_rewrite_ok メンバの値を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/event_trigger.c:252
  • src/backend/commands/event_trigger.c:677
  • **呼び出し先**:
  • 本関数は tag_behavior 配列を参照するのみであり、他の関数を呼び出さない。

関連データ構造

  • CommandTag: コマンドの種類を識別する列挙型。
  • CommandTagBehavior: 各コマンドタグの属性を保持する構造体。table_rewrite_ok メンバを含む。

失敗モードと境界条件

  • commandTag に範囲外の値を渡した場合、tag_behavior 配列の境界外アクセスが発生する可能性がある。

保守担当者向けメモ

  • 本関数は tag_behavior 配列の特定のメンバを読み取るだけの単純なアクセサである。
  • 新しいコマンドタグを追加する際は、src/include/tcop/cmdtaglist.h を更新することで、本関数が参照する tag_behavior 配列が自動的に更新される。
