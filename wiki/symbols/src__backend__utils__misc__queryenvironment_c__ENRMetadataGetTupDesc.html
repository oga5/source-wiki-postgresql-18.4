ENRMetadataGetTupDesc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/queryenvironment.c:124

目的

一時的な名前付きリレーション（Ephemeral Named Relation: ENR）のメタデータから、リレーションの構造定義である TupleDesc を取得します。

シグネチャまたは形状

TupleDesc ENRMetadataGetTupDesc(EphemeralNamedRelationMetadata enrmd)

入力と出力

  • **入力**: EphemeralNamedRelationMetadata 型のポインタ enrmd。
  • **出力**: 該当するリレーションの TupleDesc 構造体。

前提条件と不変条件

  • enrmd->reliddesc（OID）と enrmd->tupdesc（TupleDesc）のうち、いずれか一方のみが有効（非NULL/非InvalidOid）でなければならないという不変条件があります（Assert により検証）。
  • reliddesc を使用してカタログからリレーションを開く際、ロックは取得されません（NoLock）。呼び出し側で既に適切なロックが保持されている必要があります。

アルゴリズムまたは振る舞い

1. enrmd->tupdesc が設定されている場合、その値をそのまま返します。 2. enrmd->tupdesc が NULL の場合、enrmd->reliddesc を使用して table_open を呼び出し、カタログからリレーションを開きます。 3. 開いたリレーションの rd_att フィールドから TupleDesc を取得します。 4. table_close を呼び出してリレーションを閉じ、取得した TupleDesc を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeNamedtuplestorescan.c:111
  • src/backend/parser/parse_relation.c:2520
  • **呼び出し先**:
  • table_open (カタログからのリレーション取得)
  • table_close (リレーションの解放)

関連データ構造

  • EphemeralNamedRelationMetadata: ENR のメタデータを保持する構造体。
  • TupleDesc: リレーションの列定義を保持する構造体。
  • Relation: リレーションの物理的・論理的属性を保持する構造体。

失敗モードと境界条件

  • enrmd が指す構造体の reliddesc と tupdesc の両方が空、または両方が設定されている場合、アサーション違反によりプロセスが停止する可能性があります。
  • reliddesc が無効な OID を指している場合、table_open の内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は NoLock モードで table_open を実行します。この関数を呼び出すコードパスにおいて、対象リレーションに対する適切なロックが確実に保持されていることを確認してください。
  • 実装は List による線形探索を前提とした環境の一部ですが、TupleDesc の解決ロジック自体は独立しており、将来的なデータ構造の変更時にも影響を受けにくい設計となっています。
