lookup

定義

  • 種別: struct
  • 定義箇所: src/timezone/zic.c:303

目的

タイムゾーン定義ファイル（.zi）の構文解析において、キーワード文字列とそれに対応する整数値をマッピングするためのデータ構造です。

シグネチャまたは形状

struct lookup
{
	const char *l_word;
	const int	l_value;
};

入力と出力

  • l_word: 比較対象となるキーワード文字列（入力）。
  • l_value: キーワードに関連付けられた整数値（出力）。

前提条件と不変条件

  • l_word は定数文字列へのポインタである必要があります。
  • 構造体の配列として定義され、終端を示すために l_word が NULL である要素が配置されることが一般的です。

アルゴリズムまたは振る舞い

zic.c 内で、入力された文字列をこの構造体の配列と照合し、一致する l_word を見つけることで、対応する l_value を取得する検索処理に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/timezone/zic.c 内の構文解析関数群（具体的な関数名はソース抜粋からは特定できません）。
  • 呼び出し先: なし（データ構造のため）。
  • 注記: ツールが提示した contrib/ 配下の各ファイルにおける lookup への言及は、本構造体とは無関係な一般的な検索処理を指しており、本シンボルの呼び出し元ではありません。

関連データ構造

  • struct rule
  • struct zone
  • struct link

失敗モードと境界条件

  • 一致する l_word が存在しない場合、検索処理側で適切なエラーハンドリングが必要です。
  • 配列の終端（l_word == NULL）を正しく判定できない場合、メモリ境界外へのアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は IANA の zic ユーティリティ由来のものです。
  • 新しいキーワードを追加する際は、対応する配列の末尾に要素を追加し、必ず NULL 終端を維持してください。
