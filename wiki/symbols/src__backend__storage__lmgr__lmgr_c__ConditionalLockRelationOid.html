ConditionalLockRelationOid

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lmgr.c:150

目的

指定されたリレーション（OID）に対して、待機せずに条件付きでロックを取得します。ロックが即座に取得できない場合は待機せず、呼び出し元へ制御を戻します。

シグネチャまたは形状

bool ConditionalLockRelationOid(Oid relid, LOCKMODE lockmode)

入力と出力

  • **入力**:
  • relid: ロック対象のリレーションOID。
  • lockmode: 取得するロックのモード（LOCKMODE）。
  • **出力**:
  • bool: ロックの取得に成功した場合は true、競合により取得できなかった場合は false を返します。

前提条件と不変条件

  • ロック取得に成功した場合、必要に応じて AcceptInvalidationMessages() を呼び出し、リレーションキャッシュの整合性を保証します。
  • LOCKACQUIRE_ALREADY_CLEAR でない限り、取得後に MarkLockClear(locallock) を実行し、キャッシュ状態を更新します。

アルゴリズムまたは振る舞い

1. SetLocktagRelationOid を使用して、リレーションOIDから LOCKTAG を構築します。 2. LockAcquireExtended を呼び出し、conditional 引数を true に設定してロックを試みます。 3. LOCKACQUIRE_NOT_AVAIL が返された場合、即座に false を返します。 4. ロック取得成功時、キャッシュが最新でない可能性がある場合（LOCKACQUIRE_ALREADY_CLEAR 以外）、AcceptInvalidationMessages() を実行してインバリデーションメッセージを処理し、MarkLockClear を呼び出します。 5. 最終的に true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/namespace.c
  • src/backend/commands/lockcmds.c
  • src/backend/commands/tablecmds.c
  • src/backend/commands/vacuum.c

関連データ構造

  • LOCKTAG: ロック対象を識別する構造体。
  • LOCALLOCK: プロセスローカルなロック管理構造体。
  • LOCKMODE: ロックの強度を定義する列挙型。

失敗モードと境界条件

  • 他のセッションが競合するロックを保持している場合、待機せずに false を返します。
  • ロック取得処理自体が内部エラー（メモリ不足や不正なタグ等）に遭遇した場合は、LockAcquireExtended の挙動に従いエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は LockRelationOid と異なり、待機を行わないため、デッドロックの懸念がある箇所や、ロック取得を試行して失敗した際に別の処理へフォールバックしたい場合に利用してください。
  • AcceptInvalidationMessages の呼び出しは、ロック取得中に他セッションがスキーマ変更を行った場合に備えるために必須です。削除や変更は避けてください。
