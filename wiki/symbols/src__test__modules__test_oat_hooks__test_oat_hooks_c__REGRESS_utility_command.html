REGRESS_utility_command

定義

  • **種別**: function
  • **定義箇所**: src/test/modules/test_oat_hooks/test_oat_hooks.c:378

目的

PostgreSQLの ProcessUtility_hook を介して、DDL等のユーティリティコマンド実行時に権限チェックと監査ログ出力を行うためのフック関数です。

シグネチャまたは形状

static void REGRESS_utility_command(PlannedStmt *pstmt,
                                    const char *queryString,
                                    bool readOnlyTree,
                                    ProcessUtilityContext context,
                                    ParamListInfo params,
                                    QueryEnvironment *queryEnv,
                                    DestReceiver *dest,
                                    QueryCompletion *qc);

入力と出力

  • **入力**: 実行されるユーティリティコマンドの PlannedStmt 構造体および関連する実行コンテキスト情報。
  • **出力**: なし（void）。ただし、権限チェック失敗時には ereport(ERROR, ...) を通じて処理を中断します。

前提条件と不変条件

  • _PG_init によって ProcessUtility_hook に本関数が登録されている必要があります。
  • 処理の最後には必ず next_ProcessUtility_hook または standard_ProcessUtility を呼び出し、フックチェーンを維持する必要があります。

アルゴリズムまたは振る舞い

1. **監査開始**: audit_attempt を呼び出し、実行されるコマンド名をログ出力します。 2. **権限チェック**: GUC変数 REGRESS_deny_utility_commands が有効かつ実行ユーザーがスーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラーを発生させます。 3. **処理の転送**: 登録されている next_ProcessUtility_hook が存在すればそれを呼び出し、存在しなければ standard_ProcessUtility を呼び出して本来の処理を実行します。 4. **監査終了**: audit_success を呼び出し、処理完了を記録します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのコアシステム（ProcessUtility 実行時）。
  • **呼び出し先**:
  • audit_attempt / audit_success（監査ログ出力）
  • next_ProcessUtility_hook（チェーン内の次のフック）
  • standard_ProcessUtility（コアの標準処理）

関連データ構造

  • PlannedStmt: 実行されるユーティリティ文を保持する構造体。
  • ProcessUtilityContext: ユーティリティコマンドの実行コンテキスト。

失敗モードと境界条件

  • **権限拒否**: 非スーパーユーザーが REGRESS_deny_utility_commands 有効時にユーティリティコマンドを実行すると、ereport(ERROR) により処理が中断されます。

保守担当者向けメモ

  • pstrdup(action) を使用してコマンド名をコピーしていますが、これは監査ログ出力関数側でのメモリ管理要件に依存します。
