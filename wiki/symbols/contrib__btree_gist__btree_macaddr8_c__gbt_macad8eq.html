gbt_macad8eq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_macaddr8.c:40

目的

PostgreSQL の macaddr8 型に対する等価比較（equality）を GiST インデックスのコンテキストで実行する。

シグネチャまたは形状

static bool gbt_macad8eq(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**: 比較対象となる2つのポインタ a, b（macaddr8 型のデータ）、および関数呼び出し情報 flinfo。
  • **出力**: a と b が等しい場合に true、それ以外の場合に false を返す。

前提条件と不変条件

  • 入力ポインタ a および b は有効な macaddr8 型のデータ領域を指している必要がある。
  • macaddr8_eq 関数がシステムカタログに登録されており、DirectFunctionCall2 で呼び出し可能であること。

アルゴリズムまたは振る舞い

1. DirectFunctionCall2 を使用して、PostgreSQL のネイティブ関数 macaddr8_eq を呼び出す。 2. a と b を Datum 型に変換して比較を行う。 3. 比較結果を DatumGetBool で bool 型に変換して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_macaddr8.c 内の静的構造体（tinfo 等）から参照されている。
  • **呼び出し先**: macaddr8_eq（PostgreSQL ネイティブの比較関数）。

関連データ構造

  • macaddr8: 比較対象となる PostgreSQL のネットワークアドレス型。
  • FmgrInfo: 関数呼び出し情報を保持する構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、DirectFunctionCall2 の内部で例外が発生する可能性がある。
  • macaddr8_eq が予期しないデータ形式を受け取った場合、PostgreSQL のエラーハンドリング機構により処理が中断される。

保守担当者向けメモ

  • 本関数は btree_gist モジュールにおける macaddr8 型の比較ロジックのラッパーである。
  • 比較ロジックを変更する場合は、macaddr8_eq の仕様を確認すること。
  • 直接的な呼び出し元として、contrib/btree_gist/btree_macaddr8.c:81 での参照が確認されている。
