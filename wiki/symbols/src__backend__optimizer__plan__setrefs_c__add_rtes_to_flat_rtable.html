add_rtes_to_flat_rtable

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/setrefs.c:395

目的

クエリのレンジテーブル（RTE）をフラット化し、PlannerGlobal に格納することで、実行エンジンがパーミッションチェックやリレーション参照を正しく行えるようにする。

シグネチャまたは形状

static void add_rtes_to_flat_rtable(PlannerInfo *root, bool recursing)

入力と出力

  • **入力**:
  • root: 現在のクエリレベルの PlannerInfo。
  • recursing: 再帰呼び出し中かどうかを示すフラグ。
  • **出力**: なし（root->glob 内のフラット化されたレンジテーブルを更新する副作用を持つ）。

前提条件と不変条件

  • root->parse->rtable が有効であること。
  • トップレベル呼び出し時、フラット化されたレンジテーブルのインデックスは元のインデックスと一致する必要がある。
  • 再帰呼び出し時は、リレーションRTEおよび特定のサブクエリRTEのみを抽出する。

アルゴリズムまたは振る舞い

1. **RTEの追加**: root->parse->rtable を走査し、条件（トップレベルまたは特定のRTE種別）を満たすものを add_rte_to_flat_rtable でフラット化リストに追加する。 2. **デッドサブクエリの処理**: プランツリーから参照されないサブクエリ内のRTEを特定し、パーミッションチェック漏れを防ぐために個別にフラット化リストへ追加する。 3. **再帰的走査**: サブクエリがダミーリレーションである場合や、再帰呼び出し中である場合、add_rtes_to_flat_rtable を再帰的に呼び出し、サブクエリ内のRTEを抽出する。 4. **順序の維持**: ライブRTEの番号付けを維持するため、RTEの追加処理は2段階に分けて行われる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/setrefs.c 内の set_plan_references（初期化時）。
  • src/backend/optimizer/plan/setrefs.c 内の add_rtes_to_flat_rtable（再帰呼び出し）。
  • **呼び出し先**:
  • add_rte_to_flat_rtable
  • flatten_unplanned_rtes
  • fetch_upper_rel

関連データ構造

  • PlannerInfo: クエリのプランニング状態を保持。
  • PlannerGlobal: プラン全体で共有されるグローバル状態（フラット化されたRTEリストを含む）。
  • RangeTblEntry: レンジテーブルの各要素。
  • RelOptInfo: リレーションの最適化情報。

失敗モードと境界条件

  • rel->subroot が存在しない場合、flatten_unplanned_rtes を使用して未計画のRTEを処理する。
  • 継承親（Inheritance-parent）RTEは、内容が既にプルアップされているため無視される。
  • サブクエリRTEで対応する RelOptInfo がない場合、既にプルアップ済みとみなして無視される。

保守担当者向けメモ

  • 本関数はプランツリーの走査前に行われる重要な初期化処理である。
  • ライブRTEの番号付けを破壊しないよう、RTEの追加順序には注意が必要である。
  • recursing フラグは、上位クエリレベルが現在のレベルをダミーとして扱う場合に、プランツリーを走査せずにRTEを抽出するために使用される。
