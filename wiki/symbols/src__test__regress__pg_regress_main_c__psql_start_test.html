psql_start_test

定義

  • 種別: function
  • 定義箇所: src/test/regress/pg_regress_main.c:28

目的

個別のテストケース（.sql ファイル）を実行するための psql サブプロセスを構築および起動します。

シグネチャまたは形状

static PID_TYPE psql_start_test(const char *testname, _stringlist **resultfiles, _stringlist **expectfiles, _stringlist **tags)

入力と出力

  • **入力**:
  • testname: 実行するテストの名称。
  • resultfiles, expectfiles: ファイルパスを格納するリストへのポインタ。
  • tags: テストタグ（本関数内では未使用）。
  • **出力**:
  • 起動したサブプロセスのプロセスID (PID_TYPE) を返却。

前提条件と不変条件

  • outputdir, inputdir, expecteddir, bindir, dblist などのグローバル変数が初期化されていること。
  • psql コマンドが実行可能であること。

アルゴリズムまたは振る舞い

1. testname に基づき、入力SQLファイル、出力結果ファイル、期待値ファイルのパスを特定します（outputdir を優先検索）。 2. resultfiles および expectfiles リストに特定したパスを追加します。 3. StringInfo を用いて psql 実行コマンド文字列を構築します。この際、HIDE_TABLEAM=on および HIDE_TOAST_COMPRESSION=on を設定し、ストレージエンジンによる差異を抑制します。 4. 環境変数 PGAPPNAME を pg_regress/<testname> に設定します。 5. spawn_process を呼び出し、構築したコマンドを実行します。 6. プロセス起動後、PGAPPNAME を解除し、PID を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/regress/pg_regress_main.c 内のテスト実行ループから呼び出されます。
  • **呼び出し先**: spawn_process, initStringInfo, appendStringInfo, add_stringlist_item, setenv, unsetenv 等。

関連データ構造

  • StringInfoData: コマンド文字列の動的構築に使用。
  • _stringlist: ファイルパスの管理に使用。
  • PID_TYPE: プロセス識別子。

失敗モードと境界条件

  • ファイルが見つからない場合、パス検索ロジックにより代替ディレクトリ（inputdir）を探索します。
  • spawn_process が INVALID_PID を返した場合、エラーメッセージを標準エラー出力に出力し、exit(2) でプログラムを終了します。

保守担当者向けメモ

  • 本関数は pg_regress のテスト実行におけるサブプロセス生成の責務を負います。
  • HIDE_TABLEAM 等のオプションは、ストレージエンジン依存のテスト結果差異を吸収するための重要な設定です。変更時はテスト結果への影響を考慮してください。
