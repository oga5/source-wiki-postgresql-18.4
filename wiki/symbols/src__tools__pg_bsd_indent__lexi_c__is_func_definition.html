is_func_definition

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/lexi.c:159

目的

現在のトークン位置以降を先読みし、その識別子が関数定義の開始であるか、あるいは単なる宣言であるかを判定する。

シグネチャまたは形状

static int is_func_definition(char *tp)

入力と出力

  • **入力**: tp (現在のバッファ内の解析開始位置を示すポインタ)
  • **出力**: true (関数定義であると判断した場合)、false (宣言である、または判断不能な場合)

前提条件と不変条件

  • lookahead_reset() を呼び出し、先読み状態を初期化してから解析を開始する。
  • バッファの終端 (buf_end) を超える場合は lookahead() を使用して入力を継続する。

アルゴリズムまたは振る舞い

1. 括弧の深さ (paren_depth) とコメント状態を追跡しながら、文字を順次読み込む。 2. コメント内（/* ... */ または // ...）の文字は無視する。 4. paren_depth が 0 の状態で以下の文字に遭遇した場合に判定を行う:

  • {: 関数定義の本体開始とみなし true を返す。
  • ; または ,: 宣言の終了とみなし false を返す。

    • 5. EOF に到達した場合は、定義ではないとみなして false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/tools/pg_bsd_indent/lexi.c 内の字句解析ロジック（buf_ptr を引数として渡す）。
  • **呼び出し先**: lookahead_reset() および lookahead()。

関連データ構造

  • buf_end: 現在の入力バッファの終端を示すグローバル変数。

失敗モードと境界条件

  • **コメント**: コメント内に { や ; が含まれる場合、正しく無視して解析を継続する。

保守担当者向けメモ

  • 本関数は lexi() 内の字句解析において、識別子の文脈を確定させるために使用される。
  • lookahead() 関数と密接に連携しているため、バッファ管理に変更を加える際は先読みロジックの整合性に注意すること。
