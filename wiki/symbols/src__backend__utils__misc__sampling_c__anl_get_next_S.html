anl_get_next_S

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/sampling.c:295

目的

レガシーなAPIとして、リザーバーサンプリングにおいて次にスキップすべきレコード数を計算し、状態を更新する。

シグネチャまたは形状

double anl_get_next_S(double t, int n, double *stateptr);

入力と出力

  • t (double): 現在の処理済みレコード数。
  • n (int): 抽出対象のサンプル数。
  • stateptr (double *): サンプリング状態（W値）を保持するポインタ。入力時に現在の状態を読み取り、出力時に更新後の状態を書き戻す。
  • 戻り値 (double): 次に選択すべきレコードまでのスキップ数。

前提条件と不変条件

  • stateptr は有効なメモリ領域を指している必要がある。
  • 内部で oldrs.W を介して状態を管理するため、呼び出し間で stateptr が指す値が適切に保持されている必要がある。

アルゴリズムまたは振る舞い

1. stateptr が指す値を oldrs.W に代入する。 2. reservoir_get_next_S を呼び出し、現在の状態とパラメータに基づいて次のスキップ数を計算する。 3. 計算結果を result に格納する。 4. reservoir_get_next_S によって更新された oldrs.W を stateptr に書き戻す。 5. 計算されたスキップ数 result を返す。

呼び出し元と呼び出し先

  • 呼び出し先: reservoir_get_next_S
  • 呼び出し元: ソースコード上の直接的な呼び出し元は提供された情報に含まれていない。

関連データ構造

  • oldrs: reservoir_get_next_S で使用される内部的なサンプリング状態構造体。

失敗モードと境界条件

  • stateptr が NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • 内部で呼び出される reservoir_get_next_S の仕様に依存する。

保守担当者向けメモ

  • 本関数はレガシーなAPIである。新規コードでは ReservoirState を直接使用する新しいAPIを優先すること。
  • oldrs は静的またはグローバルなスコープで定義されていると推測されるが、本ソース抜粋内では定義が明示されていない。
