GinBtreeStack

定義

  • 種別: struct
  • 定義箇所: src/include/access/gin_private.h:130

目的

GIN インデックスの B-tree 構造を走査する際に、現在位置のパス（親ノードへのリンクを含む）を保持するためのスタック構造体です。

シグネチャまたは形状

typedef struct GinBtreeStack
{
    BlockNumber blkno;
    Buffer      buffer;
    OffsetNumber off;
    ItemPointerData iptr;
    uint32      predictNumber;
    struct GinBtreeStack *parent;
} GinBtreeStack;

入力と出力

本構造体は関数間でポインタとして受け渡されます。

  • 入力: GIN インデックス走査関数（ginFindParents 等）において、現在の走査状態を保持するために使用されます。
  • 出力: 走査の完了後、freeGinBtreeStack を通じてメモリが解放されます。

前提条件と不変条件

  • parent フィールドは、現在のノードの親ノードを指すか、ルートノードの場合は NULL となります。
  • buffer は、対応する blkno が読み込まれている有効なバッファである必要があります。

アルゴリズムまたは振る舞い

  • GIN インデックスのツリー走査において、ノードの分割や挿入位置の特定を行う際、再帰的に親ノードを辿るための履歴をスタックとして管理します。
  • predictNumber は、現在のレベルにおけるページ数の予測値を保持し、最適化に利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/gin/ginbtree.c 内の ginFindParents、ginPlaceToPage、ginFinishSplit、ginFinishOldSplit 等がこの構造体を操作します。
  • 呼び出し先: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • GinBtree: GIN のエントリツリーおよびポスティングツリーを操作するためのメソッドテーブル。
  • Buffer: storage/bufmgr.h で定義されるバッファ管理構造体。

失敗モードと境界条件

  • メモリ不足: palloc によるスタック確保時に発生する可能性があります。
  • 無効なスタックポインタ: parent を辿る際に NULL チェックを行わずにアクセスするとセグメンテーション違反が発生します。

保守担当者向けメモ

  • 本構造体は ginbtree.c 内で動的に割り当てられるため、スタックの各要素を解放する際は freeGinBtreeStack を使用して再帰的にメモリを解放してください。
  • predictNumber の更新ロジックは GIN インデックスの性能に直結するため、変更時は関連するツリー操作アルゴリズムへの影響を考慮してください。
