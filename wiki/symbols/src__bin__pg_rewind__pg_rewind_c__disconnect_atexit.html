disconnect_atexit

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/pg_rewind.c:1201

目的

プログラム終了時に、確立されている libpq 接続を安全に切断するために使用されるクリーンアップ関数です。

シグネチャまたは形状

static void disconnect_atexit(void)

入力と出力

  • 入力: なし
  • 出力: なし

前提条件と不変条件

  • グローバル変数 conn が libpq の接続状態を保持していることを前提とします。
  • atexit() を介して登録されることを想定しています。

アルゴリズムまたは振る舞い

1. グローバル変数 conn が NULL でないかを確認します。 2. conn が存在する場合、PQfinish(conn) を呼び出して接続を解放します。

呼び出し元と呼び出し先

  • 呼び出し元: atexit(disconnect_atexit) を通じてプログラム終了時に呼び出されます（src/bin/pg_rewind/pg_rewind.c:300 で登録）。
  • 呼び出し先: PQfinish（libpq ライブラリ関数）。

関連データ構造

  • PGconn (libpq の接続ハンドル、グローバル変数 conn として参照)

失敗モードと境界条件

  • conn が NULL の場合、何も行わずに正常終了します。

保守担当者向けメモ

  • 本関数は pg_rewind 固有のロジックではなく、PostgreSQL の他のツール（pg_basebackup 等）でも同様の名称と目的で定義されています。変更を加える際は、他のツールとの整合性に注意してください。
