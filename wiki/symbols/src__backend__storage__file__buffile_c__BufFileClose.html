BufFileClose

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/buffile.c:411

目的

BufFile 構造体によって管理されるバッファ付きファイルを閉じ、関連するすべての物理ファイル記述子を解放し、メモリリソースを解放する。

シグネチャまたは形状

void BufFileClose(BufFile *file)

入力と出力

  • **入力**: BufFile *file - クローズ対象のバッファ付きファイルへのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • 引数 file は有効な BufFile 構造体を指している必要がある。
  • BufFile が保持するすべての物理ファイル記述子は、fd.c によって管理されている必要がある。

アルゴリズムまたは振る舞い

1. BufFileFlush(file) を呼び出し、バッファ内に残っている未書き込みデータを物理ファイルへフラッシュする。 2. file->files 配列をループし、各物理ファイル記述子に対して FileClose() を実行してクローズする。 3. file->files 配列自体を pfree() で解放する。 4. file 構造体本体を pfree() で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistbuildbuffers.c
  • src/backend/backup/backup_manifest.c
  • src/backend/executor/nodeHash.c
  • src/backend/executor/nodeHashjoin.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • BufFileFlush
  • FileClose
  • pfree

関連データ構造

  • BufFile: 複数の物理ファイルセグメントを論理的に管理する構造体。

失敗モードと境界条件

  • file が NULL の場合、ポインタ参照エラーが発生する可能性がある。
  • FileClose 内部でエラーが発生した場合、その後の pfree が実行されずメモリリークが発生する可能性がある（PostgreSQL のエラーハンドリング機構に依存）。

保守担当者向けメモ

  • 本関数は BufFile のライフサイクルの終端を担う。リソース所有権（ResourceOwner）の観点から、トランザクション終了時に確実に呼び出される必要がある。
  • pfree を使用しているため、呼び出し元は BufFile が割り当てられたメモリコンテキストが有効であることを前提とする必要がある。
