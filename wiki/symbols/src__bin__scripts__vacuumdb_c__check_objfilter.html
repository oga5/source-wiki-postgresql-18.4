check_objfilter

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/vacuumdb.c:449

目的

ユーザーが指定したコマンドラインオプション（objfilter）の組み合わせが論理的に矛盾していないかを検証し、不正な指定がある場合にエラーを報告する。

シグネチャまたは形状

void check_objfilter(void)

入力と出力

  • 入力: グローバル変数 objfilter（ビットマスク形式のフィルタ種別）。
  • 出力: なし（戻り値なし）。
  • 副作用: 矛盾するオプションが検出された場合、pg_fatal を呼び出してプログラムを終了させる。

前提条件と不変条件

  • objfilter 変数が、ユーザーのコマンドライン引数に基づいて適切に設定されていること。

アルゴリズムまたは振る舞い

以下の組み合わせが同時に指定されている場合、pg_fatal を呼び出してエラーメッセージを表示する： 1. 全データベース (OBJFILTER_ALL_DBS) と特定のデータベース (OBJFILTER_DATABASE)。 2. スキーマ内の全テーブル (OBJFILTER_SCHEMA) と特定のテーブル (OBJFILTER_TABLE)。 3. 特定のテーブル (OBJFILTER_TABLE) とスキーマ除外 (OBJFILTER_SCHEMA_EXCLUDE)。 4. スキーマ内の全テーブル (OBJFILTER_SCHEMA) とスキーマ除外 (OBJFILTER_SCHEMA_EXCLUDE)。

呼び出し元と呼び出し先

  • 呼び出し元: main 関数 (src/bin/scripts/vacuumdb.c:320)。
  • 呼び出し先: pg_fatal（エラー報告および終了処理）。

関連データ構造

  • VacObjFilter: 処理対象を絞り込むためのフィルタ種別を定義する列挙型またはビットマスク。

失敗モードと境界条件

  • 矛盾するオプションが指定された場合、常に pg_fatal を経由して終了する。
  • 正常な組み合わせの場合は、何も行わず正常に復帰する。

保守担当者向けメモ

  • 新たなフィルタ種別が VacObjFilter に追加された場合、本関数に適切な矛盾チェックロジックを追加する必要がある。
