cmpLexemeInfo

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/dict_thesaurus.c:333

目的

2つの LexemeInfo 構造体を比較し、ソート順序を決定するための比較関数です。

シグネチャまたは形状

static int cmpLexemeInfo(LexemeInfo *a, LexemeInfo *b)

入力と出力

  • **入力**: 比較対象となる LexemeInfo 構造体へのポインタ a および b。
  • **出力**:
  • a と b が等しい場合は 0。
  • a が b より大きい場合は 1。
  • a が b より小さい場合は -1。
  • 入力ポインタのいずれかが NULL の場合は 0。

前提条件と不変条件

  • LexemeInfo 構造体には idsubst（置換ID）、posinsubst（置換内位置）、tnvariant（バリアント番号）の各フィールドが存在し、これらが比較の優先順位として使用されます。

アルゴリズムまたは振る舞い

以下の優先順位でフィールドを比較し、辞書順序を決定します。 1. idsubst が異なる場合、その大小を比較します。 2. idsubst が等しい場合、posinsubst を比較します。 3. posinsubst も等しい場合、tnvariant を比較します。 4. すべてのフィールドが一致する場合、0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/dict_thesaurus.c:387
  • src/backend/tsearch/dict_thesaurus.c:476
  • **呼び出し先**: 本関数は比較ロジックのみを実行し、他の関数を呼び出しません。

関連データ構造

  • LexemeInfo: 比較対象となる構造体。

失敗モードと境界条件

  • 引数に NULL が渡された場合、比較を行わず 0 を返します。

保守担当者向けメモ

  • 本関数は qsort 等のソート関数で利用されることを想定した標準的な比較インターフェースに従っています。比較フィールドの優先順位を変更する場合は、この階層構造を維持してください。
