crosstab

定義

  • 種別: function
  • 定義箇所: contrib/tablefunc/tablefunc.c:357

目的

SQL クエリの結果（縦持ちデータ）を再構成し、クロス集計形式（横持ちデータ）のセットとして返す。

シグネチャまたは形状

Datum crosstab(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_FUNCTION_ARGS: SQL クエリ文字列を含む引数。
  • **出力**:
  • Datum: Tuplestore に格納されたクロス集計結果の行セット。

前提条件と不変条件

  • **入力クエリの制約**: ソースデータとなる SQL クエリは、必ず「行識別子 (rowname)」「カテゴリ (category)」「値 (values)」の順で 3 列を返す必要がある。
  • **データ順序**: 入力データは rowname でソートされている必要がある。
  • **実行コンテキスト**: ReturnSetInfo をサポートするコンテキスト（SFRM_Materialize モード）で呼び出される必要がある。
  • **型互換性**: 戻り値の型定義（TupleDesc）と、SPI から取得したデータ構造に互換性が必要。

アルゴリズムまたは振る舞い

1. **SPI 接続**: SPI_connect を介してソース SQL を実行し、結果セットを取得する。 2. **検証**: 戻り値の型が複合型であることを確認し、compatCrosstabTupleDescs で SPI 結果との互換性を検証する。 3. **初期化**: tuplestore_begin_heap を使用して、結果を蓄積するための Tuplestore を初期化する。 4. **変換処理**:

  • SPI 結果をループし、rowname が変わるごとに新しい行を生成する。
  • rowname が同一の行については、カテゴリに基づいて値を横方向に配置する。
  • firstpass フラグと lastrowid を使用して、重複行のスキップや行の境界を制御する。

    • 5. **返却**: 構築した Tuplestore を ReturnSetInfo に格納して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL エンジン（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • SPI_connect, SPI_execute, SPI_getvalue (SPI インターフェース)
  • tuplestore_begin_heap (Tuplestore 操作)
  • compatCrosstabTupleDescs (型互換性チェック)
  • TupleDescGetAttInMetadata (メタデータ取得)

関連データ構造

  • ReturnSetInfo: 関数呼び出しのコンテキストと結果セットの受け渡しに使用。
  • Tuplestorestate: 変換後の結果を保持する一時ストレージ。
  • SPITupleTable: SPI 実行結果の保持。
  • AttInMetadata: C 文字列からタプルを構築するためのメタデータ。

失敗モードと境界条件

  • **列数不一致**: ソースクエリが 3 列を返さない場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー終了する。
  • **型不一致**: 戻り値の型が複合型でない場合、または SPI 結果と互換性がない場合、エラー終了する。
  • **コンテキスト不一致**: Materialize モードが許可されていない場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラー終了する。
  • **空の結果**: クエリ結果が空の場合、ExprEndResult を返して終了する。

保守担当者向けメモ

  • 本関数は入力データの順序に強く依存するため、利用側で ORDER BY が指定されていることが前提となる。
  • crosstab_hash と比較して、こちらはハッシュテーブルを使用しないため、メモリ消費は抑えられるが、入力データのソート順序が厳密に守られている必要がある。
