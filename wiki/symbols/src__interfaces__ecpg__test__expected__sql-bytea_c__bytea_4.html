bytea_4

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-bytea.c:63

目的

ECPG における bytea 型のバイナリデータ送受信を検証するための、バッファサイズが制限された構造体定義です。

シグネチャまたは形状

struct bytea_4 {
    int len;
    char arr[DATA_SIZE - LACK_SIZE];
} recv_short_buf;

入力と出力

  • **入力**: SQL の bytea 型データ。
  • **出力**: ECPGdo 関数を介した SQL 文実行時のホスト変数としてのデータ格納先。

前提条件と不変条件

  • len メンバには、arr に格納されるデータの有効長が保持される必要があります。
  • arr のサイズは DATA_SIZE - LACK_SIZE であり、通常のバッファよりも意図的に小さく設定されています。

アルゴリズムまたは振る舞い

ECPG プリプロセッサが生成するコードにおいて、SQL の bytea 型と C 言語の構造体をマッピングするために使用されます。ECPGdo 呼び出し時に、この構造体のアドレスとサイズ情報が渡され、データベースからのバイナリデータが arr に書き込まれます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/sql-bytea.c 内の ECPGdo 呼び出し箇所（154行目、282行目、343行目）。
  • **呼び出し先**: 情報なし。

関連データ構造

  • bytea_1, bytea_2, bytea_3: 同様の目的で定義された他の bytea 用構造体。
  • recv_short_buf: bytea_4 型のインスタンス変数。

失敗モードと境界条件

  • LACK_SIZE 分だけバッファが不足しているため、データ長が DATA_SIZE - LACK_SIZE を超える場合にバッファオーバーフローや切り捨てが発生する境界条件のテストに使用されます。

保守担当者向けメモ

  • 本構造体は回帰テスト用であり、手動編集は不要です。
  • ECPGdebug(1, stderr) を有効にすることで、実行時の通信ログから本構造体へのデータ転送状況を確認可能です。
