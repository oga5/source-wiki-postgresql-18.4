print

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-whenever.c:27

目的

ECPGの回帰テストにおいて、SQL実行エラー発生時にエラーメッセージと詳細なSQLエラー情報を標準エラー出力へ出力するカスタムハンドラです。

シグネチャまたは形状

static void print(const char *msg)

入力と出力

  • 入力: msg (エラーが発生したSQL文やコンテキストを示す文字列へのポインタ)
  • 出力: なし（標準エラー出力への書き込みのみ）

前提条件と不変条件

  • stderr が有効であること。
  • sqlprint() 関数が呼び出し可能な状態であること。

アルゴリズムまたは振る舞い

1. fprintf を使用し、引数 msg を含むエラーメッセージを標準エラー出力 (stderr) に出力します。 2. 続いて sqlprint() を呼び出し、ECPGライブラリが保持する詳細なSQLエラー情報を出力して終了します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内には明示的な呼び出し箇所は記載されていません。
  • 呼び出し先: fprintf (標準ライブラリ), sqlprint() (ECPGライブラリ)

関連データ構造

  • sqlca: SQL通信領域（sqlprint() が内部的に参照する構造体）。

失敗モードと境界条件

  • msg が NULL の場合、fprintf の動作に依存します。

保守担当者向けメモ

  • 本関数はECPGプリプロセッサのテスト生成物の一部です。エラーハンドリングのロジックを変更する場合は、src/interfaces/ecpg/preproc/ 内のプリプロセッサ変換ロジックとの整合性に注意してください。
