pg_reg_getfinalstate

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexport.c:63

目的

コンパイル済みの正規表現オブジェクトから、NFA（非決定性有限オートマトン）の終了状態（post）を取得する。

シグネチャまたは形状

int pg_reg_getfinalstate(const regex_t *regex)

入力と出力

  • **入力**: regex_t *regex（初期化済みの正規表現オブジェクトへのポインタ）
  • **出力**: int（NFA における終了状態の識別子）

前提条件と不変条件

  • regex は NULL であってはならない。
  • regex->re_magic が REMAGIC と一致している必要がある（assert による検証）。

アルゴリズムまたは振る舞い

1. 引数として渡された regex オブジェクトの re_guts を struct guts * にキャストする。 2. 内部の search 構造体（struct cnfa）にアクセスする。 3. cnfa 内の post メンバ（終了状態）の値を返し、呼び出し元に提供する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_trgm/trgm_regexp.c:1058
  • contrib/pg_trgm/trgm_regexp.c:2148
  • **呼び出し先**:
  • 特筆すべき関数呼び出しなし（構造体メンバへの直接アクセスのみ）。

関連データ構造

  • regex_t: 正規表現の公開用構造体。
  • struct guts: 正規表現エンジンの内部状態を保持する構造体。
  • struct cnfa: 正規表現の NFA 構造を定義する構造体。

失敗モードと境界条件

  • regex が NULL または不正な re_magic を持つ場合、アサーション違反によりプログラムが異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は regex->re_guts を struct guts として扱うため、regguts.h の定義変更に依存する。
  • LACON（先読み・後読み制約）の処理は本関数には含まれず、純粋にコンパイル時に決定された終了状態を返す。
