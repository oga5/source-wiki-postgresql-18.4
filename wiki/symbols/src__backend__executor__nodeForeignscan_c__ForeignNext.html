ForeignNext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeForeignscan.c:40

目的

外部テーブルスキャン実行ノードにおいて、FDW（Foreign Data Wrapper）を介して次のタプルを取得し、実行エンジンへ提供する。

シグネチャまたは形状

TupleTableSlot *ForeignNext(ForeignScanState *node)

入力と出力

  • **入力**: ForeignScanState *node（現在のスキャン状態を保持する構造体）
  • **出力**: TupleTableSlot *（取得したタプル、またはスキャン終了時の NULL）

前提条件と不変条件

  • node->fdwroutine が適切に初期化されていること。
  • econtext->ecxt_per_tuple_memory がタプル処理用のメモリコンテキストとして有効であること。
  • CMD_SELECT 以外の操作（直接更新）を行う場合、es_epq_active が NULL であること（再評価が不可能なため）。

アルゴリズムまたは振る舞い

1. econtext->ecxt_per_tuple_memory にメモリコンテキストを切り替え、短命なメモリ領域で処理を行う。 2. plan->operation が CMD_SELECT であれば fdwroutine->IterateForeignScan を呼び出し、それ以外（直接更新）であれば fdwroutine->IterateDirectModify を呼び出す。 3. メモリコンテキストを元に戻す。 4. システム列 tableoid の設定が必要な場合（plan->fsSystemCol が真かつタプルが取得できた場合）、現在のリレーションIDを slot->tts_tableOid に格納する。 5. 取得した slot を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeForeignscan.c 内の ExecScan 経由で呼び出される（直接の呼び出し元は ExecScanAccessMtd テーブルへの登録箇所）。
  • **呼び出し先**: node->fdwroutine に定義された IterateForeignScan または IterateDirectModify 関数。

関連データ構造

  • ForeignScanState: スキャンノードの実行状態。
  • ForeignScan: 実行計画ノード。
  • FdwRoutine: FDW固有の関数ポインタ群。
  • TupleTableSlot: タプルを保持するコンテナ。

失敗モードと境界条件

  • IterateDirectModify 呼び出し時に es_epq_active がアクティブな場合、アサーション違反により停止する。
  • タプルが取得できない場合（TupIsNull(slot)）、tableoid の設定はスキップされる。

保守担当者向けメモ

  • 本関数は ExecScan のアクセスメソッドとして登録されている。
  • 実際のデータ取得ロジックはすべて fdwroutine 経由で外部モジュールに委譲されているため、デバッグ時は対象のFDW実装を確認すること。
