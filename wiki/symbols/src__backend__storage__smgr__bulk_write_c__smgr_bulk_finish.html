smgr_bulk_finish

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/bulk_write.c:129

目的

バルク書き込み操作を終了し、保留中のページをフラッシュして、必要に応じてリレーションの永続性を保証（fsync またはチェックポイントへの登録）する。

シグネチャまたは形状

void smgr_bulk_finish(BulkWriteState *bulkstate)

入力と出力

  • **入力**: BulkWriteState *bulkstate（書き込み状態を保持する構造体へのポインタ）
  • **出力**: なし（副作用としてストレージへの書き込みと同期処理を実行）

前提条件と不変条件

  • bulkstate は smgr_bulk_start_* によって初期化済みであること。
  • 本関数呼び出し時点で、対象リレーションに対する他のバックエンドからのアクセスがないこと。
  • MyProc->delayChkptFlags の操作を行うため、適切なプロセスコンテキストが必要。

アルゴリズムまたは振る舞い

1. **フラッシュ**: smgr_bulk_flush(bulkstate) を呼び出し、キュー内の全ページをストレージへ書き出す。 2. **同期処理の分岐**:

  • **一時リレーション**: 何もせず終了。
  • **WALログなし（非ログ記録または wal_level=minimal）**: smgrregistersync を呼び出し、チェックポイントでのフラッシュを予約する。
  • **WALログあり**:
  • チェックポイントとの競合を防ぐため DELAY_CHKPT_START フラグをセットする。
  • 書き込み開始時と現在の RedoRecPtr を比較し、チェックポイントが介在した場合は smgrimmedsync で即時 fsync を実行する。
  • 競合がない場合は smgrregistersync でチェックポイントに登録する。
  • 最後に DELAY_CHKPT_START フラグを解除する。

関連データ構造

  • BulkWriteState: 書き込みの進捗、WALの要否、対象リレーション情報を保持。

失敗モードと境界条件

  • チェックポイントが書き込み中に発生した場合、smgrimmedsync による強制同期が行われる。
  • wal_level=minimal の場合、smgrDoPendingSyncs がコミット時に同期を担うため、本関数内では同期処理をスキップする。

保守担当者向けメモ

  • MyProc->delayChkptFlags の操作は、チェックポイントによる fsync 漏れを防ぐための重要な排他制御である。
  • use_wal が偽の場合、一時リレーションと永続リレーション（wal_level=minimal）を区別できないため、保守的な実装（smgrregistersync）となっている。
