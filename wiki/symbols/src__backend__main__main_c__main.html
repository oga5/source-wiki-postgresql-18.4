main

定義

  • 種別: function
  • 定義箇所: src/backend/main/main.c:70

目的

PostgreSQL サーバープロセスのエントリーポイントとして、環境の初期化、セキュリティチェック、および起動モードに応じた適切なメインルーチンへのディスパッチを行う。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**:
  • argc: コマンドライン引数の数。
  • argv: コマンドライン引数の配列。
  • **出力**:
  • 正常終了時は各モードのメイン関数へ制御を移すため戻らない。異常時は abort() を呼び出す。

前提条件と不変条件

  • プロセス起動直後に呼び出される必要がある。
  • save_ps_display_args を早期に実行し、プロセス名表示用の引数を確保しなければならない。
  • MemoryContextInit 実行前は elog/ereport の使用に制限がある。

アルゴリズムまたは振る舞い

1. **初期化**: プラットフォーム固有のハック（Windowsのクラッシュハンドラ等）を実行し、プロセス引数を保存する。 2. **サブシステム起動**: メモリ管理 (MemoryContextInit)、スタック深さチェック、ロケール設定を行う。 3. **引数解析**: --help や --version 等の標準オプションを判定し、該当する場合は処理して終了する。 4. **セキュリティ**: 必要に応じて check_root を呼び出し、ルート権限での実行を禁止する。 5. **ディスパッチ**: parse_dispatch_option で起動モードを特定し、switch 文で PostmasterMain、BootstrapModeMain、PostgresSingleUserMain 等の適切な関数へ制御を移す。

呼び出し元と呼び出し先

  • **呼び出し元**: OS（プロセスの起動時）。
  • **呼び出し先**: startup_hacks, save_ps_display_args, MemoryContextInit, set_stack_base, set_pglocale_pgservice, init_locale, help, check_root, parse_dispatch_option, BootstrapModeMain, SubPostmasterMain, GucInfoMain, PostgresSingleUserMain, PostmasterMain。
  • *注: ツールが提示した contrib/amcheck 等の参照は、本関数の呼び出し元ではなく、ソースコード内の文字列「main」に対する一致である可能性が高く、本関数の呼び出し元ではない。*

関連データ構造

  • DispatchOption: 起動モードを定義する列挙型。
  • MyProcPid: プロセスIDを保持するグローバル変数。

失敗モードと境界条件

  • ルート権限で実行された場合、check_root により終了する（一部の例外を除く）。
  • 不明な引数や不正なモード指定時は、適切なディスパッチ先へ遷移できない可能性がある。
  • 起動処理の最後に到達した場合は abort() を呼び出す。

保守担当者向けメモ

  • 本関数は「何をするか」よりも「どこへ処理を渡すか」を決定するゲートウェイである。
  • save_ps_display_args は環境変数の移動を伴う可能性があるため、他の処理よりも優先して実行すること。
  • LC_ALL は環境変数の設定を有効にするため、初期化後に unsetenv で削除される。
