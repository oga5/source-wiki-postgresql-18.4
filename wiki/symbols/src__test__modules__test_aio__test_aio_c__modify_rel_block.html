modify_rel_block

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_aio/test_aio.c:197

目的

指定されたリレーションのブロックを読み込み、意図的にヘッダー破損やチェックサムエラーを注入してディスクへ書き戻すためのテスト用関数です。

シグネチャまたは形状

Datum modify_rel_block(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • relid (Oid): 対象リレーションのOID。
  • blkno (BlockNumber): 対象ブロック番号。
  • zero (bool): ページをゼロクリアするかどうか。
  • corrupt_header (bool): ページヘッダーを破損させるかどうか。
  • corrupt_checksum (bool): チェックサムを不正な値に書き換えるかどうか。
  • **出力**: void (PG_RETURN_VOID)

前提条件と不変条件

  • 呼び出し元は AccessExclusiveLock を取得する権限を持つ必要があります。
  • 本関数は共有バッファの状態を直接操作する「ハック」を含んでおり、本番環境での使用は想定されていません。

アルゴリズムまたは振る舞い

1. 対象リレーションを AccessExclusiveLock で開く。 2. ReadBufferExtended で対象ブロックを読み込み、内容をローカルメモリにコピーした後、バッファを解放・破棄（InvalidateLocalBuffer または EvictUnpinnedBuffer）する。 3. zero フラグが真であればページをゼロクリアする。 4. corrupt_header が真であれば、pd_special を不正な値に設定する。 5. corrupt_checksum が真であれば、チェックサムが一致しなくなるまで最大100回値をインクリメントして試行する。偽であれば PageSetChecksumInplace で正しいチェックサムを再計算する。 6. smgrwrite を使用して、加工したページをディスクへ直接書き出す。 7. リレーションを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由での呼び出しを想定（PG_FUNCTION_INFO_V1 による登録あり）。
  • **呼び出し先**: relation_open, ReadBufferExtended, LockBuffer, BufferGetPage, ReleaseBuffer, InvalidateLocalBuffer, EvictUnpinnedBuffer, PageInit, pg_checksum_page, PageSetChecksumInplace, smgrwrite, relation_close 等。

関連データ構造

  • Relation, Buffer, Page, PageHeader

失敗モードと境界条件

  • チェックサムの破損試行が100回失敗した場合、ERROR を発生させて停止する。
  • PG_IOV_MAX を超えるブロック操作には対応していない。
  • 本関数は排他ロックを保持してバッファを破棄するため、並行実行時には競合や予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数はテスト専用であり、非同期I/Oの異常系テストを目的としています。
  • バッファの破棄処理は競合（race condition）の可能性があるため、本番コードへの流用は厳禁です。
