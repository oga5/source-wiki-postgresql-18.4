unify_hypothetical_args

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_func.c:1740

目的

WITHIN GROUP 句を伴う集約関数（hypothetical-set aggregate）において、仮説引数（hypothetical argument）と対応する集約引数（aggregated argument）の型を統一する。

シグネチャまたは形状

static void unify_hypothetical_args(ParseState *pstate,
                                    List *fargs,
                                    int numAggregatedArgs,
                                    Oid *actual_arg_types,
                                    Oid *declared_arg_types)

入力と出力

  • pstate: パーサの状態。
  • fargs: 関数引数のリスト。
  • numAggregatedArgs: 集約引数の数。
  • actual_arg_types: 引数の実際の型（入出力用）。
  • declared_arg_types: 関数定義上の引数型。

前提条件と不変条件

  • numNonAggregatedArgs（直接引数と仮説引数の合計）が負になることはない。
  • 仮説引数と対応する集約引数の宣言された型は一致している必要がある。

アルゴリズムまたは振る舞い

1. 引数リストから直接引数および仮説引数の位置を特定する。 2. 各仮説引数について、対応する集約引数との型チェックを行う。 3. 宣言された型が ANYOID の場合、select_common_type および select_common_typmod を使用して共通の型を決定する。 4. 決定された共通型に基づき、coerce_type を用いて仮説引数と集約引数の両方を暗黙的にキャストし、actual_arg_types を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_func.c 内の関数（ParseFuncOrColumn からの呼び出しが確認されている）。
  • **呼び出し先**: select_common_type, select_common_typmod, coerce_type。

関連データ構造

  • ParseState: パーサのコンテキスト。
  • List: 引数リスト。
  • Oid: 型識別子。

失敗モードと境界条件

  • numNonHypotheticalArgs < 0: 集約関数の宣言が不正な場合にエラーを発生させる。
  • declared_arg_types の不一致: 仮説引数と集約引数の宣言型が異なる場合にエラーを発生させる。

保守担当者向けメモ

  • 本関数は WITHIN GROUP 構文の型解決に特化している。
  • ANYOID 以外の型が指定されている場合は、make_fn_arguments による後続の型変換に委ねるため、本関数ではスキップされる。
