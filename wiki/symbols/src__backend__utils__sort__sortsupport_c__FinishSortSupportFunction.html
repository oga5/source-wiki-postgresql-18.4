FinishSortSupportFunction

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/sortsupport.c:93

目的

演算子ファミリー（opfamily）から最適な比較関数を特定し、SortSupportData 構造体の比較ロジック（comparator）を初期化します。

シグネチャまたは形状

static void FinishSortSupportFunction(Oid opfamily, Oid opcintype, SortSupport ssup)

入力と出力

  • **入力**:
  • opfamily: 検索対象の演算子ファミリーのOID。
  • opcintype: 比較対象のデータ型のOID。
  • ssup: 初期化対象の SortSupportData 構造体へのポインタ。
  • **出力**: なし（ssup->comparator を直接更新する副作用を持つ）。

前提条件と不変条件

  • ssup は呼び出し前に有効なメモリ領域を指している必要があります。
  • BTSORTSUPPORT_PROC または BTORDER_PROC が指定された opfamily 内に存在する必要があります。

アルゴリズムまたは振る舞い

1. BTSORTSUPPORT_PROC を検索し、存在する場合はその関数を呼び出して ssup の設定を試みます。 2. 上記の結果、ssup->comparator が設定されなかった場合、フォールバックとして BTORDER_PROC（従来のB-tree比較関数）を検索します。 3. BTORDER_PROC が見つかった場合、PrepareSortSupportComparisonShim を呼び出し、従来の比較関数をラップする「Shim」を ssup->comparator に割り当てます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/sort/sortsupport.c 内の PrepareSortSupportFromOrderingOp および関連する初期化関数から呼び出されます。
  • **呼び出し先**:
  • get_opfamily_proc: 演算子ファミリーからサポート関数を検索。
  • OidFunctionCall1: BTSORTSUPPORT_PROC を実行。
  • PrepareSortSupportComparisonShim: 比較関数が見つからない場合のShim構築。
  • elog: 必要な比較関数が見つからない場合の致命的エラー報告。

関連データ構造

  • SortSupportData (SortSupport): ソートの比較状態を保持する主要構造体。
  • BTSORTSUPPORT_PROC / BTORDER_PROC: 検索対象となるサポート関数のID。

失敗モードと境界条件

  • BTORDER_PROC が見つからない場合、elog(ERROR, ...) により処理が中断されます。
  • BTSORTSUPPORT_PROC が存在しても ssup->comparator が設定されない場合、必ず BTORDER_PROC によるフォールバックが行われます。

保守担当者向けメモ

  • 本関数は、新しい高速化API（SortSupport）と従来の比較関数（BTORDER）の橋渡しを行う重要なポイントです。
  • ssup->comparator が設定されないまま終了することはありません。必ずいずれかの比較ロジックが割り当てられます。
