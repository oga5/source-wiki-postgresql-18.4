drop_descriptor

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/descriptor.c:93

目的

ECPGプリプロセッサが管理する記述子リストから、指定された名前と接続情報に一致する記述子を検索し、メモリから解放（破棄）します。

シグネチャまたは形状

void drop_descriptor(const char *name, const char *connection)

入力と出力

  • name: 破棄対象の記述子名（文字列）。" で始まらない場合は即座に終了します。
  • connection: 記述子がバインドされている接続名（文字列、または NULL）。
  • 出力: なし（戻り値 void）。

前提条件と不変条件

  • name が " で始まっている必要があります。
  • 記述子リスト descriptors はグローバルに管理されています。
  • 該当する記述子が見つからない場合、mmerror を通じて警告（ET_WARNING）が出力されます。

アルゴリズムまたは振る舞い

1. name の先頭文字が " でない場合、何もせず終了します。 2. グローバルな descriptors リストを走査します。 3. name が一致し、かつ connection が一致（または両方とも NULL）する記述子を探します。 4. 一致する記述子が見つかった場合、リストからその要素を切り離し、connection、name、および記述子構造体自体のメモリを free して終了します。 5. リストの最後まで見つからなかった場合、接続情報に基づいた警告メッセージを mmerror で報告します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • 呼び出し先: strcmp（標準ライブラリ）、free（標準ライブラリ）、mmerror（プリプロセッサのエラー報告関数）。

関連データ構造

  • struct descriptor: 記述子情報を保持する構造体。name、connection、および次の要素へのポインタ next を含みます。

失敗モードと境界条件

  • 記述子が存在しない場合: mmerror により警告が発行されます。
  • name が " で始まらない場合: 何も行われません。
  • connection が NULL の場合: デフォルト接続にバインドされた記述子として検索されます。

保守担当者向けメモ

  • 本関数は descriptors リストを直接操作するため、リストの整合性を維持する必要があります。
  • mmerror は PARSE_ERROR を引数に取りますが、警告（ET_WARNING）として処理されるため、プリプロセス自体は中断されません。
