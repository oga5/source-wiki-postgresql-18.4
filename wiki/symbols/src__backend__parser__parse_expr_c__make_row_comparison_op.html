make_row_comparison_op

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/parser/parse_expr.c:2837

目的

SQLの行比較（例: (a, b) = (c, d)）を解析し、意味的に確定した式ツリー（RowCompareExpr または単一の OpExpr）を生成する。

シグネチャまたは形状

Node *make_row_comparison_op(ParseState *pstate, List *opname, List *largs, List *rargs, int location)

入力と出力

  • **入力**:
  • pstate: パーサーの状態管理構造体。
  • opname: 比較演算子の名前（例: =）。
  • largs, rargs: 比較対象となる左辺および右辺の式のリスト。
  • **出力**: 変換済みの Node（RowCompareExpr または OpExpr）。

前提条件と不変条件

  • largs と rargs の要素数は一致していなければならない。
  • 行の長さが0であってはならない。
  • 各要素の比較演算子は、ブール値を返すものでなければならない（集合を返す演算子は不可）。
  • 比較演算子はB-tree演算子族に関連付けられている必要がある。

アルゴリズムまたは振る舞い

1. **検証**: 行の長さが一致しているか、0でないかを確認する。 2. **ペアごとの演算子生成**: make_op を呼び出し、各要素ペアに対する演算子式（OpExpr）を生成する。 3. **単一要素の最適化**: 行の長さが1の場合、生成された単一の OpExpr をそのまま返す。 4. **意味解析**: 各演算子のB-tree演算子族を特定し、行比較のセマンティクス（COMPARE_EQ, COMPARE_NE など）を決定する。 5. **式構築**:

  • COMPARE_EQ の場合: 各要素の比較を AND で結合。
  • COMPARE_NE の場合: 各要素の比較を OR で結合。
  • その他: 演算子族を特定し、RowCompareExpr を構築する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_expr.c 内の複数箇所（transformExprRecurse 等から呼び出される）。
  • **呼び出し先**: make_op, get_op_index_interpretation, makeBoolExpr, ereport 等。

関連データ構造

  • RowCompareExpr: 行比較を表すノード。
  • OpExpr: 単一の演算子式。
  • OpIndexInterpretation: 演算子のインデックス解釈情報。
  • ParseState: パーサーの状態。

失敗モードと境界条件

  • **エラー**:
  • 行の要素数が不一致の場合（ERRCODE_SYNTAX_ERROR）。
  • 行の長さが0の場合（ERRCODE_FEATURE_NOT_SUPPORTED）。
  • 演算子がブール値を返さない、または集合を返す場合（ERRCODE_DATATYPE_MISMATCH）。
  • 演算子の解釈（B-treeセマンティクス）が特定できない場合（ERRCODE_FEATURE_NOT_SUPPORTED）。

保守担当者向けメモ

  • make_op が型強制（coercion）を挿入する可能性があるため、入力された largs/rargs リストをそのまま再利用してはならない。
  • 複数の解釈が存在する場合、現在の実装ではインデックスの比較タイプ番号が最小のものを優先的に選択する。
