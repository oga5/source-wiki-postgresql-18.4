transformTableConstraint

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_utilcmd.c:1034

目的

テーブル定義に含まれる制約（Constraint ノード）を解析し、CreateStmtContext が保持する適切なリストへ分類・格納する。

シグネチャまたは形状

static void transformTableConstraint(CreateStmtContext *cxt, Constraint *constraint)

入力と出力

  • **入力**:
  • cxt: 変換処理の状態を保持する CreateStmtContext へのポインタ。
  • constraint: 解析対象の Constraint 構造体。
  • **出力**: なし（副作用として cxt 内の各制約リストを更新する）。

前提条件と不変条件

  • cxt は有効な CreateStmtContext を指している必要がある。
  • constraint->contype に基づいて分類が行われる。

アルゴリズムまたは振る舞い

1. constraint->contype を判定し、以下のいずれかのリストに constraint を追加する：

  • ixconstraints (インデックス関連): CONSTR_PRIMARY, CONSTR_UNIQUE, CONSTR_EXCLUSION
  • ckconstraints (チェック制約): CONSTR_CHECK
  • nnconstraints (NOT NULL制約): CONSTR_NOTNULL
  • fkconstraints (外部キー制約): CONSTR_FOREIGN

    • 2. 外部テーブル（cxt->isforeign が真）の場合、PRIMARY, UNIQUE, EXCLUSION, FOREIGN 制約はエラーとして報告する。 3. パーティションテーブルにおいて、NOT NULL 制約かつ NO INHERIT が指定されている場合はエラーとする。 4. NULL, DEFAULT などの属性関連制約や、未知の制約型が渡された場合は elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_utilcmd.c 内のテーブル要素走査処理から呼び出される。
  • src/backend/commands/tablecmds.c:15766（主キー再作成時）からの呼び出しが確認されている。
  • **呼び出し先**:
  • lappend: リストへの追加に使用。
  • ereport / elog: エラー報告に使用。

関連データ構造

  • CreateStmtContext: 変換中の状態管理用構造体。
  • Constraint: 解析対象の制約ノード。

失敗モードと境界条件

  • **エラー**:
  • 外部テーブルに対してサポート外の制約（PK, UNIQUE, EXCLUSION, FK）を適用しようとした場合。
  • パーティションテーブルに対して NOT NULL かつ NO INHERIT を指定した場合。
  • コンテキストとして不適切な制約型（DEFAULT 等）が渡された場合。
  • 未知の制約型が渡された場合。

保守担当者向けメモ

  • 本関数は CreateStmtContext の各リスト（ixconstraints, ckconstraints 等）を直接操作するため、呼び出し側で cxt が正しく初期化されていることを前提としている。
  • 新しい制約型を追加する場合は、本関数の switch 文にケースを追加し、適切なリストへ振り分ける必要がある。
