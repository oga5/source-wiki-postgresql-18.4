gbt_macad8_union

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_macaddr8.c:143

目的

GiST インデックス構築時、複数の macaddr8 型のキーを包含する最小の範囲（Union）を計算する。

シグネチャまたは形状

Datum gbt_macad8_union(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（GistEntryVector を含む）
  • 出力: mac8KEY 構造体へのポインタ（Datum として返却）

前提条件と不変条件

  • PG_GETARG_POINTER(0) が GistEntryVector を指していること。
  • PG_GETARG_POINTER(1) が出力サイズを格納する領域を指していること。
  • tinfo（gbtree_ninfo 型）が macaddr8 型の比較ロジックを保持していること。

アルゴリズムまたは振る舞い

1. palloc0 を使用して mac8KEY 構造体分のメモリを確保する。 2. 第2引数に sizeof(mac8KEY) を設定し、呼び出し元にサイズを通知する。 3. gbt_num_union を呼び出し、entryvec 内の全キーを包含する最小の mac8KEY を計算する。この際、tinfo に定義された比較関数を使用して範囲を決定する。 4. 計算結果を返却する。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQL の GiST インデックス構築プロセス（直接的な呼び出し元コードはソース抜粋内に明示なし）。
  • 呼び出し先: gbt_num_union（btree_utils_num.c で定義された汎用関数）。

関連データ構造

  • mac8KEY: macaddr8 型の範囲を保持する構造体。
  • GistEntryVector: インデックス対象のキー集合を保持する構造体。
  • tinfo: macaddr8 型の GiST サポート関数や比較ロジックを定義した静的変数。

失敗モードと境界条件

  • メモリ不足時: palloc0 がエラーを発生させる可能性がある。
  • entryvec が空の場合: gbt_num_union の内部実装に依存する。

保守担当者向けメモ

  • 本関数は btree_utils_num.c の汎用ロジックに強く依存している。
  • macaddr8 は 8 バイトのデータ型であり、gbt_num_union を通じて数値的な範囲として処理される。
  • 比較ロジックの変更が必要な場合は、tinfo の定義を確認すること。
