xidout

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xid.c:42

目的

内部的な TransactionId 型の値を、人間が読み取り可能な文字列形式（C文字列）に変換します。

シグネチャまたは形状

Datum xidout(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取った TransactionId 型の値。
  • **出力**: Datum 型として返される、トランザクションIDの文字列表現（char *）。

前提条件と不変条件

  • PG_GETARG_TRANSACTIONID(0) を使用して引数を取得するため、本関数はPostgreSQLの関数呼び出し規約に従う環境で実行される必要があります。
  • 文字列格納用に16バイトのメモリを palloc で確保します。

アルゴリズムまたは振る舞い

1. PG_GETARG_TRANSACTIONID(0) により、引数から TransactionId を取得します。 2. palloc(16) を呼び出し、結果格納用のメモリ領域を確保します。 3. snprintf を使用し、TransactionId を符号なし長整数（unsigned long）として書式化し、確保したバッファに書き込みます。 4. PG_RETURN_CSTRING(result) を介して、変換された文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報には、本関数を直接呼び出している特定の関数名は記載されていません。
  • **呼び出し先**:
  • PG_GETARG_TRANSACTIONID (マクロ)
  • palloc (メモリ確保)
  • snprintf (文字列変換)
  • PG_RETURN_CSTRING (マクロ)

関連データ構造

  • TransactionId: PostgreSQLのトランザクション識別子。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • palloc がメモリ不足により失敗した場合、PostgreSQLの標準的なエラーハンドリングによりアボートされます。
  • TransactionId の値が unsigned long の範囲を超える場合、snprintf の動作に依存しますが、本実装では16バイトのバッファを使用しており、標準的な TransactionId のサイズに対しては十分な領域が確保されています。

保守担当者向けメモ

  • 本関数は xid 型の出力関数として機能します。
  • 64bitトランザクションID（xid8）を扱う場合は、xid8out 等の別関数が使用されるため、本関数は従来の32bit TransactionId 専用である点に注意してください。
