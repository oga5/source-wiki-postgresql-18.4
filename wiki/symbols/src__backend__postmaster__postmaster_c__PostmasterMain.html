PostmasterMain

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/postmaster.c:493

目的

PostgreSQL サーバープロセスのメインエントリーポイントとして、プロセスの初期化、シグナルハンドラの登録、およびコマンドライン引数の解析を行い、サーバーの稼働準備を整える。

シグネチャまたは形状

void PostmasterMain(int argc, char *argv[])

入力と出力

  • **入力**: argc (引数の数), argv (引数文字列の配列)。
  • **出力**: なし（プロセス終了まで制御を保持する）。

前提条件と不変条件

  • InitProcessGlobals() が呼び出され、プロセスグローバル変数が初期化されていること。
  • PostmasterContext が TopMemoryContext の下に作成され、メモリ管理の基盤が確立されていること。
  • umask が PG_MODE_MASK_OWNER に設定され、ファイル作成時の権限が制限されていること。

アルゴリズムまたは振る舞い

1. **初期化**: プロセスグローバル変数の設定、Win32環境でのシグナル初期化、メモリコンテキストの作成を行う。 2. **シグナル設定**: pqinitmask でシグナルマスクを初期化し、pqsignal を用いてシャットダウン、再読み込み、子プロセス終了などの各種シグナルハンドラを登録する。 3. **環境構築**: InitializeWaitEventSupport および InitProcessLocalLatch を呼び出し、待機イベントとラッチを準備する。 4. **オプション解析**: getopt を使用してコマンドライン引数を解析し、SetConfigOption を通じて設定値を適用する。 5. **終了処理**: 正常終了またはエラー発生時にプロセスを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/main/main.c の main 関数から呼び出される。
  • **呼び出し先**: InitProcessGlobals, AllocSetContextCreate, getInstallationPaths, pqinitmask, pqsignal, InitializeWaitEventSupport, InitProcessLocalLatch, InitializeGUCOptions, getopt, SetConfigOption, parse_dispatch_option 等。

関連データ構造

  • PostmasterContext: Postmaster プロセス用のメモリコンテキスト。
  • PMState: サーバーの状態管理に使用される列挙型（本関数内で初期化・遷移の準備が行われる）。

失敗モードと境界条件

  • **引数エラー**: 不正なオプションや必須値の欠如（例: -c オプションの値欠如）が発生した場合、ereport(ERROR, ...) によりプロセスが終了する。
  • **ディスパッチエラー**: -- で始まる特殊なオプションが正しく解析できない場合、構文エラーとして報告される。

保守担当者向けメモ

  • シグナルハンドラの変更時は、子プロセス（postgres.c, bgwriter.c, autovacuum.c 等）への副作用を必ず確認すること。
  • コマンドラインオプションの追加・変更時は、tcop/postgres.c および main/main.c のヘルプ表示との整合性を維持すること。
