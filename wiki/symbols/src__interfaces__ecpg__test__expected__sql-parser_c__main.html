目的

ECPG（Embedded SQL for C）プリプロセッサのパーサー機能を検証するための回帰テスト用エントリポイント。SQL文の解析能力およびデータベース操作の整合性を確認する。

シグネチャまたは形状

int main()

入力と出力

  • 入力: なし（プログラム実行時にデータベースへ接続）。
  • 出力: 成功時に 0 を返却。標準出力にテスト結果を表示。

前提条件と不変条件

  • 実行環境においてデータベース ecpg1_regression への接続が可能であること。
  • sqlca（SQL Communication Area）が各SQL実行後のステータス監視に使用されること。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、ECPGconnect でデータベースに接続。ECPGsetcommit でコミットモードを有効化。 2. **テーブル構築**: CREATE TABLE T を実行し、generate_series を用いてテストデータを投入。 3. **データ取得**: SELECT 文を実行し、結果をホスト変数 item 配列および ind インジケータ配列に格納。 4. **検証**: 取得したデータを標準出力へ出力し、期待値と比較。 5. **スキーマ変更**: ALTER TABLE を用いてカラムの型変更をテスト。 6. **終了処理**: DROP TABLE で環境をクリーンアップし、ECPGdisconnect で接続を終了。 7. **エラーハンドリング**: 各SQL実行後に sqlca.sqlcode < 0 をチェックし、エラー発生時は sqlprint() を呼び出す。

呼び出し元と呼び出し先

  • 呼び出し元: テスト実行環境（OSまたはテストハーネス）。
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGsetcommit, ECPGdo, ECPGdisconnect, sqlprint, printf。

関連データ構造

  • sqlca: SQL実行後のエラーコードや警告を保持する構造体。
  • item / ind: SQLの結果セットを格納するホスト変数配列。

失敗モードと境界条件

  • SQL実行時にエラー（sqlca.sqlcode < 0）が発生した場合、sqlprint() が呼び出され、標準エラー出力に詳細が出力される。
  • sqlca.sqlwarn[0] == 'W' の場合、警告が出力される。
