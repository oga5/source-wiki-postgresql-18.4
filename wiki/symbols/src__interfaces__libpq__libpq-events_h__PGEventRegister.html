PGEventRegister

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/libpq-events.h:37

目的

イベントハンドラが登録される際に、対象となるデータベース接続（PGconn）をイベントプロシージャへ渡すためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	PGconn	   *conn;
} PGEventRegister;

入力と出力

  • **入力**: 構造体内の conn メンバは、イベント登録対象の有効な PGconn ポインタを保持します。
  • **出力**: なし（本構造体はイベント発生時にハンドラへ情報を伝達するコンテナとして機能します）。

前提条件と不変条件

  • conn は、イベントハンドラが呼び出された時点で有効な PGconn オブジェクトを指している必要があります。

アルゴリズムまたは振る舞い

  • PQregisterEventProc を介してイベントハンドラが登録される際、libpq はこの構造体を初期化し、対象の接続情報を格納してハンドラに渡します。
  • ハンドラはこの構造体を通じて、登録対象の接続コンテキストにアクセスします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/libpq-events.c 内のイベント管理ロジックから参照・生成されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • PGconn: データベース接続を管理する主要な構造体。
  • PGEventProc: 本構造体を引数として受け取るイベントハンドラの型定義。

失敗モードと境界条件

  • conn が NULL の場合、ハンドラ側で適切なエラーハンドリングが必要です。

保守担当者向けメモ

  • 本構造体は PGEVT_REGISTER イベント発生時に使用されます。将来的に登録イベントに付随する情報が増えた場合、この構造体にメンバが追加される可能性があります。
