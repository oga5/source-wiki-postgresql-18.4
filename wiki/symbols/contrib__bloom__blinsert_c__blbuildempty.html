blbuildempty

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blinsert.c:164

目的

Bloom 索引のメタページを初期化し、空のインデックス状態を作成します。

シグネチャまたは形状

void blbuildempty(Relation index)

入力と出力

  • **入力**: index (Relation) - 初期化対象となる Bloom 索引のリレーション情報。
  • **出力**: なし（副作用としてメタページが初期化されます）。

前提条件と不変条件

  • index は有効な Bloom 索引のリレーションである必要があります。
  • INIT_FORKNUM を使用して、インデックスの初期フォークに対してメタページの書き込みが行われます。

アルゴリズムまたは振る舞い

1. BloomInitMetapage 関数を呼び出します。 2. BloomInitMetapage は、指定されたリレーションのメタページを初期化し、Bloom 索引の管理に必要な初期構造をディスク上に構築します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/bloom/blutils.c 内の amroutine->ambuildempty フィールドを通じて、PostgreSQL のインデックス構築インターフェースから呼び出されます。
  • **呼び出し先**: BloomInitMetapage（bloom.h または関連するユーティリティで定義）。

関連データ構造

  • Relation: PostgreSQL のリレーション記述子。
  • BloomMetaPageData: メタページ内に格納される Bloom 索引の制御情報（BloomInitMetapage によって初期化される対象）。

失敗モードと境界条件

  • メタページの書き込み権限がない場合や、ディスク領域の不足が発生した場合は、基盤となるバッファマネージャまたはストレージ層によりエラーがスローされます。

保守担当者向けメモ

  • 本関数は ambuildempty インターフェースの実装です。インデックスの再構築や初期化時に呼び出されるため、メタページのレイアウト変更時には BloomInitMetapage と併せて確認が必要です。
