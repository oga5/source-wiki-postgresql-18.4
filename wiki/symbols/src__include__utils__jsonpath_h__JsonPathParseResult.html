JsonPathParseResult

定義

  • 種別: struct
  • 定義箇所: src/include/utils/jsonpath.h:272

目的

SQL/JSON Path 言語の解析過程において、文字列から変換された中間表現を保持するための構造体です。

シグネチャまたは形状

typedef struct JsonPathParseResult
{
	JsonPathParseItem *expr;
	bool		lax;
} JsonPathParseResult;

入力と出力

  • **入力**: parsejsonpath 関数等により、SQL 文字列から解析された結果が本構造体に格納されます。
  • **出力**: 本構造体は、最終的なバイナリ形式の JsonPath を構築するためのソースとして利用されます。

前提条件と不変条件

  • expr は JsonPathParseItem へのポインタであり、パス式の抽象構文木（AST）的な構造を指します。
  • lax は SQL/JSON Path のモード（Lax モードか Strict モードか）を示します。

アルゴリズムまたは振る舞い

1. parsejsonpath 関数が SQL 文字列を解析し、JsonPathParseResult を生成します。 2. 生成された中間表現は、その後バイナリ形式の JsonPath へとシリアライズされます。 3. 解析プロセスでは jsonpath_yyparse 等のパーサーが本構造体を介して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonpath.c: parsejsonpath 関数内で本構造体が生成されます。
  • src/backend/utils/adt/jsonpath_internal.h: パーサー（jsonpath_yyparse）が結果を受け取る引数として使用します。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • JsonPathParseItem: パス式の各要素を構成する中間表現ノード。
  • JsonPath: 本構造体から構築される、オンディスク形式のバイナリ JSON Path。

失敗モードと境界条件

  • 解析エラー発生時、jsonpath_yyerror が呼び出され、解析プロセスが中断される可能性があります。

保守担当者向けメモ

  • 本構造体は解析時の中間表現であり、実行時の JsonPathItem とは役割が異なります。
  • lax フラグの変更や expr の構造変更は、シリアライズロジック（jsonpath.c 内）への影響を伴うため注意してください。
