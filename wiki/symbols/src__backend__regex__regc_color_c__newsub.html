newsub

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_color.c:389

目的

指定された色（color）に対して、必要に応じて新しいサブカラー（subcolor）を割り当て、その識別子を返します。

シグネチャまたは形状

static color newsub(struct colormap *cm, color co)

入力と出力

  • **入力**:
  • cm: カラーマップ構造体へのポインタ。
  • co: 対象となる色（color 型）。
  • **出力**:
  • 成功時: 既存のサブカラー、または新しく生成されたサブカラーの識別子（color 型）。
  • 失敗時: COLORLESS。

前提条件と不変条件

  • cm は有効なカラーマップ構造体を指している必要があります。
  • co は cm 内で有効な色である必要があります。
  • サブカラーが生成された場合、そのサブカラーは自身を指すように設定されます（cm->cd[sco].sub = sco）。

アルゴリズムまたは振る舞い

1. 指定された色 co が既にサブカラーを持っているか確認します。 2. サブカラーが存在しない場合、以下の最適化を試みます。

  • その色が単一の文字（nschrs + nuchrs == 1）のみを参照している場合、サブカラーを生成せず co 自身を返します。

    • 3. サブカラーが必要な場合、newcolor(cm) を呼び出して新しい色を割り当てます。 4. 新しいサブカラーの割り当てに失敗した場合は COLORLESS を返し、成功した場合は cm->cd[co].sub を更新して新しいサブカラーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_color.c 内の関数（行345, 373）。
  • **呼び出し先**:
  • newcolor（src/backend/regex/regc_color.c）
  • CISERR（マクロ）

※注: src/backend/commands/event_trigger.c や src/backend/replication/logical/worker.c に存在する newsub という名前の変数は、本関数とは無関係です。

関連データ構造

  • colormap: カラーマップ全体を管理する構造体。
  • colordesc: 各色の詳細情報を保持する構造体（cm->cd 配列の要素）。

失敗モードと境界条件

  • newcolor が COLORLESS を返した場合、本関数も COLORLESS を返します。この際、CISERR() アサーションによりエラー状態であることが保証されます。

保守担当者向けメモ

  • 本関数は正規表現コンパイル時の色管理において、冗長なサブカラー生成を抑制する最適化ロジックを含んでいます。
  • cm->cd[co].sub の値が NOSUB であることが、サブカラー未割り当ての判定基準となっています。
