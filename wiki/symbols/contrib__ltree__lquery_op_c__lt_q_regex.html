lt_q_regex

定義

  • 種別: function
  • 定義箇所: contrib/ltree/lquery_op.c:301

目的

ltree 型の値が、lquery 型の配列に含まれるいずれかのパターンにマッチするかを判定します。

シグネチャまたは形状

Datum lt_q_regex(PG_FUNCTION_ARGS)

入力と出力

  • 入力:
  • PG_GETARG_LTREE_P(0): 照合対象の ltree 構造体。
  • PG_GETARG_ARRAYTYPE_P(1): 検索条件となる lquery の配列。
  • 出力:
  • bool: いずれかのパターンにマッチした場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力された配列は1次元である必要があります。
  • 配列内に NULL 要素を含めることはできません。
  • 上記の条件を満たさない場合、エラーが発生します。

アルゴリズムまたは振る舞い

1. 入力された lquery 配列の要素数を取得します。 2. 配列の次元数および NULL の有無を検証し、不正な場合はエラーを報告します。 3. 配列内の各 lquery 要素に対して、ltq_regex 関数を呼び出し、ltree との照合を試みます。 4. いずれかの要素でマッチが確認された時点でループを終了し、結果を true とします。 5. 最後に、コピーされた入力引数のメモリを解放し、結果を返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/ltree/ltree_gist.c:706 (GIST インデックス検索時)
  • 呼び出し先:
  • ltq_regex (各要素の照合に使用)
  • ArrayGetNItems, ARR_NDIM, ARR_DIMS, ARR_DATA_PTR, array_contains_nulls, DirectFunctionCall2, NEXTVAL (PostgreSQL 内部マクロおよび関数)

関連データ構造

  • ltree: 階層パスを保持するデータ構造。
  • lquery: 検索パターンを保持するデータ構造。
  • ArrayType: lquery を格納する PostgreSQL の配列型。

失敗モードと境界条件

  • 配列が多次元の場合: ERRCODE_ARRAY_SUBSCRIPT_ERROR を発生させます。
  • 配列内に NULL が含まれる場合: ERRCODE_NULL_VALUE_NOT_ALLOWED を発生させます。

保守担当者向けメモ

  • 本関数は lquery の配列を反復処理するラッパーです。個別のマッチングロジックは ltq_regex に依存しています。
  • NEXTVAL(query) を使用して配列内の lquery ポインタを順次進めているため、メモリレイアウトの整合性に注意してください。
