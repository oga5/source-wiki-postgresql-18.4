ReturnSetInfo

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/execnodes.h:347

目的

集合を返す関数（Set-returning function）の実行時状態を管理し、呼び出し元と関数間での戻り値の受け渡しを仲介します。

シグネチャまたは形状

typedef struct ReturnSetInfo
{
    NodeTag             type;
    ExprContext        *econtext;
    TupleDesc           expectedDesc;
    int                 allowedModes;
    SetFunctionReturnMode returnMode;
    ExprDoneCond        isDone;
    Tuplestorestate    *setResult;
    TupleDesc           setDesc;
} ReturnSetInfo;

入力と出力

  • **入力（呼び出し元が設定）**:
  • econtext: 関数が実行される式コンテキスト。
  • expectedDesc: 呼び出し元が期待するタプル記述子。
  • allowedModes: 呼び出し元が処理可能な戻りモードのビットマスク。
  • **出力（関数が設定）**:
  • returnMode: 実際に使用された戻りモード。
  • isDone: ValuePerCall モード時の完了状態。
  • setResult: Materialize モード時に全タプルを保持するストア。
  • setDesc: 返されるタプルの実際の記述子。

前提条件と不変条件

  • returnMode および isDone は、関数呼び出し前に呼び出し元によって初期化されている必要があります。
  • Materialize モードを使用する場合、関数は setResult にタプルを格納し、setDesc を設定する必要があります。

アルゴリズムまたは振る舞い

1. 呼び出し元は ReturnSetInfo を準備し、fcinfo->resultinfo を通じて関数に渡します。 2. 関数は allowedModes を確認し、適切な戻りモードを選択します。 3. ValuePerCall モードの場合、関数は呼び出されるたびに1行ずつ返し、isDone で終了を通知します。 4. Materialize モードの場合、関数は一度の呼び出しで全結果を setResult に格納して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/dblink/dblink.c、contrib/hstore/hstore_op.c、contrib/pageinspect/ 配下の各関数など、fcinfo->resultinfo を介して本構造体を参照する関数群。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数呼び出しの引数として渡されることで機能します。

関連データ構造

  • ExprContext: 式評価のコンテキスト。
  • TupleDesc: タプルの構造定義。
  • Tuplestorestate: 結果セットを保持するためのストレージ。
  • SetFunctionReturnMode, ExprDoneCond: 戻り値の制御用列挙型。

失敗モードと境界条件

  • fcinfo->resultinfo が NULL である、または ReturnSetInfo 型ではない場合、呼び出し元での型キャストやアクセス時に不正なメモリ参照が発生する可能性があります（contrib/dblink/dblink.c 等でチェックが行われています）。

保守担当者向けメモ

  • 本構造体は実行時専用であり、copyObject や equal といったノードサポート関数はサポートしていません。
  • Materialize モードと ValuePerCall モードの切り替えロジックは、呼び出し元と関数の実装間で整合性を保つ必要があります。
