shdepReassignOwned_InitAcl

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_shdepend.c:1733

目的

REASSIGN OWNED コマンド実行時、pg_init_privs カタログ内のロール参照を新しいロールへ置換します。

シグネチャまたは形状

static void shdepReassignOwned_InitAcl(Form_pg_shdepend sdepForm, Oid oldrole, Oid newrole)

入力と出力

  • sdepForm: 対象となる共有依存関係レコードの構造体ポインタ。
  • oldrole: 置換対象の古いロール OID。
  • newrole: 置換後の新しいロール OID。
  • 戻り値: なし（void）。

前提条件と不変条件

  • sdepForm は有効な pg_shdepend レコードを指している必要があります。
  • 本関数は REASSIGN OWNED の処理フローの一部として呼び出されます。

アルゴリズムまたは振る舞い

1. sdepForm から取得した classid、objid、objsubid を使用します。 2. ReplaceRoleInInitPriv を呼び出し、pg_init_privs 内の oldrole への言及を newrole へ置換します。 3. この処理は、オブジェクトの通常の ACL に対する REASSIGN OWNED の挙動と整合させるために行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/pg_shdepend.c 内の REASSIGN OWNED 処理ルーチン（shdepReassignOwned 等から間接的に参照）。
  • 呼び出し先: ReplaceRoleInInitPriv。

関連データ構造

  • Form_pg_shdepend: pg_shdepend カタログの行構造体。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • pg_init_privs に該当するエントリが存在しない場合、ReplaceRoleInInitPriv の内部挙動に依存します。
  • 本関数は pg_init_privs の履歴的記録を上書きする性質があるため、pg_dump 等の外部ツールとの整合性に関する制約がソースコード内で指摘されています。

保守担当者向けメモ

  • pg_init_privs は本来 CREATE EXTENSION 時の履歴を保持すべきですが、現状の REASSIGN OWNED はロール参照を置換する実装となっています。この設計上のトレードオフについてはソース内のコメントを参照してください。
