BogusFree

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/mcxt.c:291

目的

無効なポインタに対して pfree が呼び出された際に、エラーを発生させて処理を停止させるためのフォールバック関数です。

シグネチャまたは形状

static void BogusFree(void *pointer)

入力と出力

  • 入力: void *pointer - 解放しようとした無効なメモリ領域へのポインタ。
  • 出力: なし（elog(ERROR, ...) により呼び出し元へは戻りません）。

前提条件と不変条件

  • 本関数は、メモリコンテキストのメソッドテーブルにおいて、正常な解放処理が定義できない（または不正な）状況で呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. GetMemoryChunkHeader(pointer) を使用して、対象ポインタに関連付けられたチャンクヘッダー情報を取得します。 2. elog(ERROR, ...) を呼び出し、無効なポインタのアドレスと取得したヘッダー情報をログに出力した上で、エラーを発生させます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/mmgr/mcxt.c 内の MemoryContextMethods 構造体の初期化箇所（line 42）で、無効なメソッドとして参照されています。
  • 呼び出し先: GetMemoryChunkHeader（インライン関数）、elog（エラー報告関数）。

関連データ構造

  • MemoryContextMethods: メモリコンテキストの操作メソッドを保持する構造体。本関数はこの構造体の free_p フィールドに割り当てられることがあります。

失敗モードと境界条件

  • 本関数自体がエラーハンドリングの終端であるため、本関数内での失敗はプロセス全体のエラー処理フローに委ねられます。
  • pointer が NULL である場合や、メモリレイアウトが破壊されている場合でも、GetMemoryChunkHeader の挙動に依存してエラーメッセージが生成されます。

保守担当者向けメモ

  • 本関数は、メモリ管理の整合性が失われた際のデバッグ用セーフティネットです。直接呼び出すことは想定されていません。
