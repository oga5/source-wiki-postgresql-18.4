SubTransGetParent

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/subtrans.c:121

目的

指定されたトランザクション ID (XID) の親トランザクション ID をサブトランザクションログ (SUBTRANS) から取得する。

シグネチャまたは形状

TransactionId SubTransGetParent(TransactionId xid)

入力と出力

  • **入力**: xid (対象のトランザクション ID)
  • **出力**: 親トランザクションの TransactionId。親が存在しない場合や無効な場合は InvalidTransactionId を返す。

前提条件と不変条件

  • xid は TransactionXmin 以上である必要がある（Assert による検証）。
  • xid が正規のトランザクション ID でない場合（Bootstrap や Frozen XID）、親は存在しないものとして扱う。

アルゴリズムまたは振る舞い

1. TransactionIdToPage および TransactionIdToEntry を用いて、対象 XID が格納されている SLRU ページとエントリ番号を特定する。 2. SimpleLruReadPage_ReadOnly を呼び出し、該当ページを読み込んでスロットを取得する（この際、内部でロックが獲得される）。 3. ページバッファ内の該当エントリから親 XID を読み取る。 4. SimpleLruGetBankLock で取得したロックを解放する。 5. 読み取った親 XID を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/subtrans.c (再帰的な探索用)
  • src/backend/access/transam/transam.c
  • **呼び出し先**:
  • SimpleLruReadPage_ReadOnly
  • SimpleLruGetBankLock
  • LWLockRelease

関連データ構造

  • SubTransCtl: SLRU 管理のための共有メモリ制御構造体。

失敗モードと境界条件

  • xid が TransactionXmin より古い場合、アサーション違反が発生する。
  • xid が正規のトランザクション ID でない場合、InvalidTransactionId が返される。

保守担当者向けメモ

  • 本関数は読み取り専用のアクセスを行う。
  • SubTransCtl を介した SLRU ページアクセスは、ページ境界を跨ぐ可能性があるが、SLRU 機構がこれを抽象化している。
  • 戻り値が InvalidTransactionId であることは、必ずしもエラーを意味せず、親が存在しない（ルートトランザクションである等）ことを示す場合がある。
