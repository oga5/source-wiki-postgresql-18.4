fsm_rebuild_page

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/fsmpage.c:341

目的

FSM（空き領域マップ）ページ内のバイナリツリー構造を全ノードにわたって再計算し、整合性を強制的に修正する。

シグネチャまたは形状

bool fsm_rebuild_page(Page page)

入力と出力

  • **入力**: Page page - 操作対象となるFSMページ。
  • **出力**: bool - ページ内のノード値に変更があった場合は true、変更がなかった場合は false を返す。

前提条件と不変条件

  • 対象の page は有効なFSMページ構造として初期化されている必要がある。
  • 本関数を呼び出す際は、対象ページに対する適切なバッファロックが保持されていること。

アルゴリズムまたは振る舞い

1. ページ内の最下位非リーフノードから開始し、ルートノードに向かって逆順に走査する。 2. 各非リーフノードについて、その左右の子ノード（leftchild および rightchild）の値を参照する。 3. 左右の子ノードの値のうち、大きい方を当該ノードの新しい値として設定する。 4. ノードの値が更新された場合、changed フラグを true に設定する。 5. 全ノードの再計算完了後、変更の有無を示すフラグを返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/freespace/fsmpage.c 内の複数箇所（破損検知時のリカバリ用）。
  • **呼び出し先**:
  • leftchild() マクロ（ツリー構造のナビゲーション）。
  • Max() マクロ（子ノード値の比較）。
  • PageGetContents()（ページデータへのアクセス）。

関連データ構造

  • FSMPage: ページ内のノード配列 fp_nodes を保持する構造体。
  • NodesPerPage: ページ内の総ノード数。
  • NonLeafNodesPerPage: ページ内の非リーフノード数。

失敗モードと境界条件

  • **境界条件**: 最初の数個の非リーフノードは子ノードを1つしか持たない場合があるため、lchild および rchild が NodesPerPage の範囲内にあるかを確認するガード条件が含まれている。
  • **失敗モード**: ページが破損している場合、本関数はツリーを再構築して整合性を回復させるが、ページ自体が物理的に破壊されている場合は正常に動作しない可能性がある。

保守担当者向けメモ

  • 本関数はコストが高い操作であるため、通常運用時ではなく、探索中に期待値と実際の値が一致しない等の破損が疑われる場合にのみ呼び出すこと。
