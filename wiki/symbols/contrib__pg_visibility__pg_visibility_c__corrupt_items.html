corrupt_items

定義

  • 種別: struct
  • 定義箇所: contrib/pg_visibility/pg_visibility.c:40

目的

可視性マップ（VM）の検証プロセスにおいて、破損していると判定されたタプルの識別子（TID）を一時的に保持・管理するためのデータ構造です。

シグネチャまたは形状

typedef struct corrupt_items
{
    BlockNumber next;
    BlockNumber count;
    ItemPointer tids;
} corrupt_items;

入力と出力

  • 本構造体は、collect_corrupt_items 関数内でメモリ確保および初期化が行われます。
  • record_corrupt_item 関数を通じて、検出された破損タプルの TID が tids 配列に格納されます。

前提条件と不変条件

  • tids は、破損タプルを格納するために動的に割り当てられた ItemPointer の配列を指す必要があります。
  • count は、現在格納されているタプル数、あるいは配列の容量を管理するために使用されます。

アルゴリズムまたは振る舞い

  • collect_corrupt_items がリレーションを走査し、VMビットと実際のタプル状態が矛盾する場合に本構造体へ情報を蓄積します。
  • FuncCallContext の user_fctx に格納され、SRF（Set-Returning Function）の実行中に状態を維持するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: collect_corrupt_items（生成）、pg_check_frozen および pg_check_visible（user_fctx 経由での参照）。
  • **呼び出し先**: record_corrupt_item（データ追加時に呼び出される）。

関連データ構造

  • ItemPointer: タプルを一意に特定するためのデータ型。
  • FuncCallContext: SRF の実行状態を保持する PostgreSQL の構造体。

失敗モードと境界条件

  • メモリ不足: palloc0 によるメモリ確保失敗時にエラーが発生する可能性があります。
  • 配列のオーバーフロー: tids 配列の容量を超えてタプルが検出された場合の挙動については、record_corrupt_item の実装に依存します。

保守担当者向けメモ

  • 本構造体は pg_check_frozen および pg_check_visible 関数の実行期間中のみ有効なメモリ領域に存在します。
  • tids 配列のサイズ管理については、collect_corrupt_items 内での初期化ロジックを参照してください。
