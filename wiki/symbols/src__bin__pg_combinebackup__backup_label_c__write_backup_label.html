write_backup_label

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/backup_label.c:126

目的

指定されたバッファの内容からバックアップラベルファイルを生成し、指定ディレクトリへ書き出す。その際、増分バックアップに関連する行を除外し、チェックサム計算とマニフェストへの登録を行う。

シグネチャまたは形状

void write_backup_label(char *output_directory, StringInfo buf,
                        pg_checksum_type checksum_type, manifest_writer *mwriter)

入力と出力

  • output_directory: 書き出し先のディレクトリパス。
  • buf: ラベルファイルの内容を保持する StringInfo。
  • checksum_type: チェックサム計算に使用するアルゴリズム。
  • mwriter: マニフェスト書き込み用オブジェクト（NULL可）。
  • 出力: 指定ディレクトリ内に backup_label ファイルを作成する。

前提条件と不変条件

  • output_directory に backup_label という名前のファイルが既に存在してはならない（O_EXCL フラグによる排他制御）。
  • buf は有効なバックアップラベル形式の文字列を保持している必要がある。

アルゴリズムまたは振る舞い

1. pg_checksum_init でチェックサムコンテキストを初期化する。 2. output_directory/backup_label を作成し、ファイル記述子を取得する。 3. buf を行単位で走査し、INCREMENTAL FROM LSN: および INCREMENTAL FROM TLI: で始まる行をスキップする。 4. スキップ対象外の行をファイルへ書き込み、同時に pg_checksum_update でチェックサムを更新する。 5. ファイルをクローズし、pg_checksum_final で最終的なチェックサムを算出する。 6. mwriter が提供されている場合、stat でファイルサイズと更新時刻を取得し、add_file_to_manifest を呼び出してマニフェストに登録する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_combinebackup/pg_combinebackup.c
  • 呼び出し先: get_eol_offset, line_starts_with, pg_checksum_init, pg_checksum_update, pg_checksum_final, add_file_to_manifest, pg_fatal (エラー時)

関連データ構造

  • StringInfo: ラベルファイルの内容を保持するバッファ。
  • pg_checksum_context: チェックサム計算の状態管理。
  • manifest_writer: マニフェスト登録用インターフェース。

失敗モードと境界条件

  • ファイル作成失敗（既存ファイル存在時や権限不足）: pg_fatal で終了。
  • 書き込み失敗（ディスクフル等）: pg_fatal で終了。
  • チェックサム更新失敗: pg_fatal で終了。
  • stat 失敗: pg_fatal で終了。

保守担当者向けメモ

  • 本関数は buf->cursor を直接操作してバッファを走査するため、呼び出し側でバッファの状態を破壊しないよう注意が必要。
  • 増分バックアップ行の除外ロジックは、結合後のバックアップを完全なものとして扱うための仕様である。
