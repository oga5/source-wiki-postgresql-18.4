printResult

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-declare.c:607

目的

ECPG 回帰テストにおいて、テストケースの実行結果（f1, f2, f3 配列の内容）を標準出力に整形して表示する。

シグネチャまたは形状

void printResult(char *tc_name, int loop)

入力と出力

  • **入力**:
  • tc_name: テストケース名を示す文字列（NULL の場合はヘッダーを表示しない）。
  • loop: 表示する配列要素の数。
  • **出力**: なし（標準出力への書き出しのみ）。

前提条件と不変条件

  • グローバル変数として f1（int型配列）、f2（int型配列）、f3（文字列型配列）が定義されている必要がある。
  • loop 引数は、上記配列の有効なインデックス範囲内であること。

アルゴリズムまたは振る舞い

1. tc_name が指定されている場合、****<tc_name> test results:**** という形式でヘッダーを出力する。 2. 0 から loop - 1 までループし、各インデックス i について f1[i], f2[i], f3[i] の値を f1=%d, f2=%d, f3=%s\n という形式で出力する。 3. 最後に改行を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/test/expected/sql-declare.c 内の各テストケース実行箇所（testcase1 から testcase4 までの実行後）。
  • **呼び出し先**:
  • 標準ライブラリの printf 関数。

関連データ構造

  • f1, f2, f3: 本関数が参照するグローバルなデータ配列。

失敗モードと境界条件

  • tc_name が NULL の場合、ヘッダー出力はスキップされる。
  • loop が 0 の場合、ループ内の出力は行われず、改行のみが出力される。
  • 配列のサイズが loop より小さい場合、メモリ破壊や不正な読み取りが発生する可能性がある（呼び出し側の不変条件に依存）。

保守担当者向けメモ

  • 本関数はテストの期待値確認用であり、製品コードには含まれない。
  • f1, f2, f3 の定義が変更された場合、本関数の出力フォーマットも追従する必要がある。
