InjectionPointLoad

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/injection_point.c:531

目的

指定された名前のインジェクションポイントをローカルキャッシュに読み込み、実行可能な状態にするために使用されます。

シグネチャまたは形状

void InjectionPointLoad(const char *name);

入力と出力

  • **入力**: name (インジェクションポイントの識別名)
  • **出力**: なし（成功時）。失敗時はエラーをスローします。

前提条件と不変条件

  • ビルド時に --enable-injection-points が有効である必要があります。
  • 共有メモリ上のインジェクションポイント情報が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. USE_INJECTION_POINTS マクロが定義されている場合、InjectionPointCacheRefresh(name) を呼び出し、共有メモリからローカルキャッシュへ情報を同期します。 2. マクロが定義されていない場合、elog(ERROR, ...) を発行し、本機能がサポートされていない旨を通知します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_aio/test_aio.c (テストモジュールによる直接呼び出し)
  • src/include/utils/injection_point.h (マクロ INJECTION_POINT_LOAD 経由)
  • **呼び出し先**:
  • InjectionPointCacheRefresh (キャッシュの更新処理)

関連データ構造

  • InjectionPointCacheEntry (ローカルキャッシュエントリ)

失敗モードと境界条件

  • **ビルド設定**: --enable-injection-points 無効時に呼び出すと、ERROR レベルのログが発生し処理が中断されます。
  • **キャッシュ更新失敗**: InjectionPointCacheRefresh 内部で発生する動的ライブラリのロード失敗やメモリ不足等のエラーが伝播します。

保守担当者向けメモ

  • 本関数はテストおよびデバッグ専用のインターフェースです。
  • INJECTION_POINT_LOAD マクロ経由での利用が推奨されます。
