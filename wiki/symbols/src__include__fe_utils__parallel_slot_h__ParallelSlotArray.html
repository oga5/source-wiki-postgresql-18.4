ParallelSlotArray

定義

  • 種別: struct
  • 定義箇所: src/include/fe_utils/parallel_slot.h:36

目的

PostgreSQLのフロントエンドツールにおいて、複数のデータベース接続を並列に管理・利用するためのコンテナ構造体です。

シグネチャまたは形状

typedef struct ParallelSlotArray
{
	int			numslots;
	ConnParams *cparams;
	const char *progname;
	bool		echo;
	const char *initcmd;
	ParallelSlot slots[FLEXIBLE_ARRAY_MEMBER];
} ParallelSlotArray;

入力と出力

  • **入力**: 接続パラメータ（ConnParams）、プログラム名、エコー設定、初期化コマンド。
  • **出力**: 内部に保持する ParallelSlot 配列を介した、並列接続の管理状態。

前提条件と不変条件

  • FLEXIBLE_ARRAY_MEMBER を使用しているため、インスタンス化には offsetof(ParallelSlotArray, slots) を用いた動的なメモリ確保が必要です。
  • slots メンバは可変長配列として扱われます。

アルゴリズムまたは振る舞い

  • 複数の ParallelSlot を保持し、接続のプール管理、空きスロットの検索、および並列実行時の同期制御を抽象化します。
  • src/fe_utils/parallel_slot.c で提供される関数群（ParallelSlotsGetIdle 等）により、接続のライフサイクル管理やクエリ結果のハンドリングが行われます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照）**:
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/scripts/reindexdb.c
  • src/bin/scripts/vacuumdb.c
  • **呼び出し先（操作関数）**:
  • src/fe_utils/parallel_slot.c 内の find_matching_idle_slot, find_unconnected_slot, find_any_idle_slot, wait_on_slots, connect_slot, ParallelSlotsGetIdle 等。

関連データ構造

  • ParallelSlot: 個別の PGconn を保持する構造体。
  • ConnParams: 接続パラメータを定義する構造体（fe_utils/connect_utils.h）。
  • ParallelSlotResultHandler: クエリ結果処理用の関数ポインタ型。

失敗モードと境界条件

  • Windows環境では FD_SETSIZE の制限に注意が必要です。
  • 接続スロットの枯渇や、並列処理中の割り込み（キャンセル）発生時に適切なエラーハンドリングが必要です。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を含んでいるため、メモリ確保時に sizeof(ParallelSlotArray) を直接使用しないよう注意してください。
  • ParallelSlotResultHandler を実装する際は、PQclear による結果解放の責務がハンドラ側にあるか呼び出し元にあるかを明確にしてください。
