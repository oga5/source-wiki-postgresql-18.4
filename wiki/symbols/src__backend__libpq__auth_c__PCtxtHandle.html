PCtxtHandle

定義

  • 種別: typedef
  • 定義箇所: src/backend/libpq/auth.c:195

目的

Windowsセキュリティサポートプロバイダーインターフェース（SSPI）において、セキュリティコンテキストを識別するためのハンドルを定義します。

シグネチャまたは形状

Windows APIの PCtxtHandle 型（CtxtHandle 構造体へのポインタ）として定義されています。

入力と出力

本シンボル自体は型定義ですが、関連する関数ポインタ型 QUERY_SECURITY_CONTEXT_TOKEN_FN の引数として使用されます。

  • 入力: セキュリティコンテキストを参照するハンドル。
  • 出力: なし（型定義のため）。

前提条件と不変条件

  • Windows環境でのコンパイル時のみ有効な型定義です。
  • SSPIの仕様に基づき、有効なセキュリティコンテキストを指している必要があります。

アルゴリズムまたは振る舞い

本シンボルは、認証プロセスにおいてセキュリティコンテキストを保持・参照するための不透明なハンドルとして機能します。具体的な値の生成や破棄は、WindowsのSSPI関数群によって行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/libpq/auth.c:196 において、QUERY_SECURITY_CONTEXT_TOKEN_FN 型の定義の一部として参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • QUERY_SECURITY_CONTEXT_TOKEN_FN: PCtxtHandle を引数に取る関数ポインタ型。
  • CtxtHandle: Windows SDKで定義される実際の構造体。

失敗モードと境界条件

  • 無効なハンドルが渡された場合、SSPI関数側でエラーが発生します。
  • Windows以外のプラットフォームでは本シンボルは定義されません。

保守担当者向けメモ

  • 本シンボルはWindows固有の認証処理（GSSAPI/SSPI）に関連しています。
  • src/backend/libpq/auth.c 内の他の認証方式（SCRAMやLDAPなど）とは独立した、Windowsプラットフォーム特有の依存関係です。
