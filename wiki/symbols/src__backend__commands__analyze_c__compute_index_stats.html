compute_index_stats

定義

  • 種別: function
  • 定義箇所: src/backend/commands/analyze.c:864

目的

テーブルのサンプル行に基づき、インデックス（式インデックスや部分インデックスを含む）の統計情報を計算し、VacAttrStats 構造体を更新する。

シグネチャまたは形状

static void compute_index_stats(Relation onerel, double totalrows,
                                AnlIndexData *indexdata, int nindexes,
                                HeapTuple *rows, int numrows,
                                MemoryContext col_context)

入力と出力

  • **入力**:
  • onerel: 対象テーブルのリレーション記述子。
  • totalrows: テーブルの推定全行数。
  • indexdata: インデックスごとの解析情報を保持する配列。
  • nindexes: インデックスの数。
  • rows: サンプル行の配列。
  • numrows: サンプル行数。
  • col_context: 統計計算用のメモリコンテキスト。
  • **出力**: なし（副作用として indexdata 内の統計情報が更新される）。

前提条件と不変条件

  • anl_context がグローバルまたは上位スコープで初期化されている必要がある。
  • indexdata 配列は nindexes 分の要素を持ち、有効な IndexInfo を保持していること。

アルゴリズムまたは振る舞い

1. **コンテキスト作成**: インデックス解析専用の ind_context を作成し、メモリ管理を行う。 2. **インデックス走査**: 各インデックスに対して以下を実行:

  • EState を作成し、式評価や部分インデックスの述語（ii_Predicate）評価の準備を行う。
  • サンプル行 (rows) を順次スキャンし、ExecQual で部分インデックスの条件を満たすか判定する。
  • 条件を満たす行に対し、FormIndexDatum を用いてインデックスキーの値を計算・抽出する。
  • 抽出した値を exprvals にコピーし、統計計算のために保持する。

    • 3. **統計計算**:

  • インデックス内の有効行数 (numindexrows) を基に、テーブル全体に対するインデックスのタプル占有率 (tupleFract) を算出。
  • stats->compute_stats を呼び出し、各列の統計（ヒストグラム等）を生成する。

    • 4. **クリーンアップ**: 各インデックス処理後に EState を破棄し、ind_context をリセットしてメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/analyze.c 内の do_analyze_rel から呼び出される。
  • **呼び出し先**: CreateExecutorState, ExecPrepareQual, FormIndexDatum, vacuum_delay_point, datumCopy, stats->compute_stats 等。

関連データ構造

  • AnlIndexData: インデックス解析に必要な情報を保持する構造体。
  • VacAttrStats: 列ごとの統計情報を保持する構造体。
  • IndexInfo: インデックスのメタデータ（述語など）。

失敗モードと境界条件

  • attr_cnt == 0 かつ部分インデックスでない場合、そのインデックスの解析はスキップされる。
  • numindexrows == 0 の場合、統計計算処理はスキップされる。
  • メモリ不足時は palloc 等によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は anl_context を利用してメモリを管理している。
  • compute_stats 関数ポインタ経由で型固有の解析ロジックを呼び出すため、新しいデータ型を追加する際は typanalyze の実装に注意すること。
  • 拡張統計情報（extended_stats.c）との処理の差異に留意すること。
