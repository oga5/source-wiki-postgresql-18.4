parse_manifest_file

定義

  • 種別: function
  • 定義箇所: src/bin/pg_verifybackup/pg_verifybackup.c:391

目的

JSON 形式の backup_manifest ファイルを読み込み、解析結果を manifest_data 構造体として構築・返却する。

シグネチャまたは形状

manifest_data *parse_manifest_file(char *manifest_path)

入力と出力

  • **入力**: manifest_path (解析対象のファイルパス)
  • **出力**: manifest_data 構造体へのポインタ。解析されたファイル一覧やメタデータを含む。

前提条件と不変条件

  • 指定された manifest_path が読み取り可能であること。
  • マニフェストファイルが有効な JSON 形式であること。
  • ファイルサイズに基づき、ハッシュテーブルの初期サイズが動的に決定される。

アルゴリズムまたは振る舞い

1. ファイルを開き、fstat でサイズを取得する。 2. ファイルサイズに基づき、manifest_files_create でハッシュテーブルを初期化する。 3. JsonManifestParseContext を設定し、コールバック関数（verifybackup_version_cb 等）を登録する。 4. ファイルサイズが READ_CHUNK_SIZE 以下であれば一括で読み込み json_parse_manifest を実行する。 5. ファイルサイズが大きい場合は、json_parse_manifest_incremental_init を使用し、チャンク単位で読み込みと解析を繰り返す。 6. 解析完了後、メモリを解放し、構築された manifest_data を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_verifybackup/pg_verifybackup.c 内の main 関数（またはその周辺の制御フロー）。
  • **呼び出し先**: manifest_files_create, json_parse_manifest, json_parse_manifest_incremental_init, json_parse_manifest_incremental_chunk, json_parse_manifest_incremental_shutdown 等。

関連データ構造

  • manifest_data: 解析結果を保持するメイン構造体。
  • manifest_files_hash: ファイル情報を格納するハッシュテーブル。
  • JsonManifestParseContext: JSON 解析時の状態とコールバックを保持する構造体。

失敗モードと境界条件

  • ファイルが開けない場合や fstat に失敗した場合は report_fatal_error により致命的エラーとして終了する。
  • 読み込み中にエラーが発生した場合は pg_fatal により終了する。
  • チャンク処理において、最後のチャンクサイズが不十分にならないよう、bytes_left < 2 * chunk_size の場合に読み込みサイズを調整するロジックが含まれる。

保守担当者向けメモ

  • 本関数はマニフェストのサイズに応じて一括読み込みとインクリメンタル読み込みを自動的に切り替える。
  • 解析エラーは report_manifest_error コールバックを通じて処理される。
  • 呼び出し元との詳細な依存関係については、ソースコード内の main 関数における呼び出し箇所を参照のこと。
