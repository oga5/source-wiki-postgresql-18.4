chareq

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/char.c:126

目的

PostgreSQLの1バイト文字型（"char"）の2つの値が等しいかどうかを判定する。

シグネチャまたは形状

Datum chareq(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る2つの "char" 型の値。
  • **出力**: Datum 型として返される真偽値（bool）。arg1 と arg2 が等しい場合は true、それ以外は false を返す。

前提条件と不変条件

  • 入力値は PG_GETARG_CHAR マクロにより取得される。
  • 比較は1バイトの文字データとして行われる。

アルゴリズムまたは振る舞い

1. PG_GETARG_CHAR(0) および PG_GETARG_CHAR(1) を使用して、引数から2つの文字データを取得する。 2. 取得した arg1 と arg2 を比較演算子 == で評価する。 3. PG_RETURN_BOOL を使用して、比較結果をブール値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（演算子呼び出し経由）。ソースコード内では直接的な呼び出し元は明示されていない。
  • **呼び出し先**: PG_GETARG_CHAR および PG_RETURN_BOOL マクロを使用。

関連データ構造

  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • 特筆すべき失敗モードはない。1バイトの全範囲（0-255）に対して等価比較を行う。

保守担当者向けメモ

  • 本関数は "char" 型（1バイト）専用である。SQL標準の CHAR(n)（bpchar）とは異なる型であることに注意すること。
  • 比較演算は単純なビット比較であるが、他の比較関数（charlt 等）では uint8 としてキャストして比較が行われる点と整合しているか確認が必要である。
