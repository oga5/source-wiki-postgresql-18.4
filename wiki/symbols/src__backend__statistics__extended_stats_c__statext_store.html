statext_store

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/extended_stats.c:768

目的

計算された拡張統計情報（NDistinct, 依存関係, MCV, 式）をシステムカタログ pg_statistic_ext_data に永続化します。

シグネチャまたは形状

static void statext_store(Oid statOid, bool inh,
                          MVNDistinct *ndistinct, MVDependencies *dependencies,
                          MCVList *mcv, Datum exprs, VacAttrStats **stats)

入力と出力

  • **入力**:
  • statOid: 対象となる統計オブジェクトのOID。
  • inh: 継承された統計かどうかを示すフラグ。
  • ndistinct, dependencies, mcv: 各種統計情報のポインタ（NULL可）。
  • exprs: 式統計のデータム。
  • stats: 属性統計情報の配列。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • pg_statistic_ext_data テーブルに対する RowExclusiveLock を保持している必要があります。
  • 渡される統計データ構造は、対応するシリアライズ関数（statext_ndistinct_serialize 等）で処理可能な形式である必要があります。

アルゴリズムまたは振る舞い

1. pg_statistic_ext_data をオープンし、挿入用の values 配列と nulls 配列を初期化します。 2. 各統計種別（ndistinct, dependencies, mcv）が提供されている場合、それぞれのシリアライズ関数を呼び出し、結果を values に格納します。 3. exprs が提供されている場合は、そのまま values に格納します。 4. 既存の統計データを RemoveStatisticsDataById を用いて削除します（更新ではなく削除・再挿入の戦略をとります）。 5. heap_form_tuple で新しいタプルを生成し、CatalogTupleInsert でカタログに書き込みます。 6. 最後にタプルを解放し、テーブルをクローズします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/extended_stats.c 内の BuildRelationExtStatistics から呼び出されます。
  • **呼び出し先**:
  • statext_ndistinct_serialize, statext_dependencies_serialize, statext_mcv_serialize（シリアライズ用）
  • RemoveStatisticsDataById, CatalogTupleInsert（カタログ操作用）

関連データ構造

  • MVNDistinct, MVDependencies, MCVList: 統計情報の保持用構造体。
  • VacAttrStats: 属性統計情報。
  • pg_statistic_ext_data: 統計データが格納されるシステムカタログ。

失敗モードと境界条件

  • 各統計データが NULL の場合、カタログ上の対応するカラムは NULL として扱われます。
  • RemoveStatisticsDataById が失敗した場合、トランザクション全体がアボートされる可能性があります。

保守担当者向けメモ

  • 本関数は「更新」ではなく「削除して再挿入」という戦略をとっています。これはカタログ操作の複雑さを避けるための設計です。
  • 新しい統計種別を追加する場合は、本関数の values 配列の構築ロジックおよびシリアライズ処理を拡張する必要があります。
