ArrayMapState

定義

  • 種別: struct
  • 定義箇所: src/include/utils/array.h:251

目的

配列の変換操作（array_map）を実行する際に必要なメタデータおよび作業領域を保持するための構造体です。

シグネチャまたは形状

typedef struct ArrayMapState
{
	ArrayMetaState inp_extra;
	ArrayMetaState ret_extra;
} ArrayMapState;

入力と出力

  • **入力**: array_map 関数等において、入力配列の型情報（inp_extra）および変換後の出力型情報（ret_extra）を保持するために使用されます。
  • **出力**: 変換処理の過程で、各要素の型変換に必要なキャッシュ情報として利用されます。

前提条件と不変条件

  • ArrayMetaState 型のフィールドを保持しており、各要素の型情報が正しく初期化されている必要があります。
  • 変換処理の呼び出しごとに、初期化された（ゼロクリアされた）状態の構造体が渡されることが想定されています。

アルゴリズムまたは振る舞い

  • array_map 処理において、入力配列の要素型と出力先の型に関するメタデータをキャッシュし、反復的な型変換処理を効率化します。
  • 実行時に palloc0 等でメモリ確保され、変換処理のワークスペースとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExpr.c:1731: 実行式コンテキスト内でインスタンスが確保されます。
  • src/backend/utils/adt/arrayfuncs.c:3203: array_map 関連の関数で引数として受け取られます。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ArrayMetaState: 配列の要素型に関するメタデータ（型ID、サイズ、アライメント等）を保持する構造体。本構造体の構成要素です。

失敗モードと境界条件

  • メモリ不足時に palloc0 が失敗する可能性があります。
  • 渡される ArrayMapState が適切に初期化されていない場合、型変換処理において予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は array_map の実行状態を管理する重要なワークスペースです。変更を加える際は、src/backend/utils/adt/arrayfuncs.c 内の関連ロジックとの整合性に注意してください。
  • ArrayMetaState の定義変更が本構造体に直接影響を与えるため、型情報のキャッシュ戦略を変更する際は両方の構造体を確認してください。
