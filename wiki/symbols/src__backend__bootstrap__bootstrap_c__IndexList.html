IndexList

定義

  • 種別: struct
  • 定義箇所: src/backend/bootstrap/bootstrap.c:162

目的

ブートストラップモードにおいて、システムカタログのインデックス構築情報を一時的に保持するための連結リスト構造体です。カタログ定義時に登録されたインデックス情報を蓄積し、全定義完了後に一括で構築するために使用されます。

シグネチャまたは形状

typedef struct _IndexList
{
	Oid			il_heap;          /* インデックス対象のヒープリレーションのOID */
	Oid			il_ind;           /* インデックスリレーションのOID */
	IndexInfo  *il_info;          /* インデックスの属性や構成情報 */
	struct _IndexList *il_next;   /* リストの次要素へのポインタ */
} IndexList;

入力と出力

  • 入力: インデックス登録時に il_heap、il_ind、il_info が設定されます。
  • 出力: 構築処理時にリストを辿り、各要素が保持する情報に基づいてインデックスが物理的に構築されます。

前提条件と不変条件

  • ILHead（静的変数）を起点とするリスト構造で管理されます。
  • リストの各要素は palloc によって動的に割り当てられます。

アルゴリズムまたは振る舞い

1. **登録**: index_register（推定）等の処理により、新しいインデックス情報が IndexList 構造体に格納され、ILHead リストに追加されます。 2. **保持**: システムカタログの定義が進行する間、このリストに情報が蓄積されます。 3. **構築**: 全てのカタログ定義が完了した段階で、リストを走査し、各 IndexList 要素が持つ情報を用いてインデックスの構築が実行されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/bootstrap/bootstrap.c 内のインデックス登録処理（index_register 等）にてインスタンス化されます。
  • 呼び出し先: IndexInfo 構造体（access/genam.h 等で定義）を介して、インデックス構築のためのアクセスメソッド層と連携します。

関連データ構造

  • IndexInfo: インデックスの定義詳細を保持する構造体。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • メモリ不足: palloc 失敗時にプロセスが終了する可能性があります。
  • 順序依存性: インデックス構築はカタログ定義の完了後に行う必要があるため、構築タイミングを誤るとシステムカタログの整合性が損なわれます。

保守担当者向けメモ

  • 本構造体はブートストラップ専用のメモリ管理下にあるため、通常のバックエンドのメモリコンテキストとはライフサイクルが異なります。
  • il_next による連結リスト操作において、メモリリークが発生しないよう ILHead の管理に注意してください。
