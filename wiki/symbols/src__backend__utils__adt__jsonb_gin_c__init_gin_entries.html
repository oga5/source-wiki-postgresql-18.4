init_gin_entries

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_gin.c:162

目的

GINインデックス構築時に使用するエントリ格納用構造体（GinEntries）を初期化します。

シグネチャまたは形状

static void init_gin_entries(GinEntries *entries, int preallocated)

入力と出力

  • **入力**:
  • entries: 初期化対象となる GinEntries 構造体へのポインタ。
  • preallocated: 事前に確保するエントリ数。
  • **出力**: なし（entries が指す構造体のメンバを更新）。

前提条件と不変条件

  • preallocated が 0 より大きい場合、palloc を用いて Datum 配列のメモリ領域が確保されます。
  • entries->count は必ず 0 に初期化されます。

アルゴリズムまたは振る舞い

1. entries->allocated に preallocated の値を設定します。 2. preallocated が 0 でない場合、sizeof(Datum) * preallocated 分のメモリを palloc で確保し、entries->buf に格納します。0 の場合は entries->buf を NULL に設定します。 3. entries->count を 0 に設定し、エントリの追加準備を完了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonb_gin.c:247
  • src/backend/utils/adt/jsonb_gin.c:1110
  • **呼び出し先**:
  • palloc (PostgreSQL メモリ管理関数)

関連データ構造

  • GinEntries: エントリのバッファ、確保済みサイズ、現在のカウントを保持する構造体。

失敗モードと境界条件

  • preallocated に非常に大きな値を指定した場合、palloc によるメモリ不足（out of memory）が発生する可能性があります。

保守担当者向けメモ

  • 本関数は GinEntries のメモリ管理を担います。palloc を使用しているため、呼び出し元で適切なメモリコンテキストがアクティブであることを確認してください。
