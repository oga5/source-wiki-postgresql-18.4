HeapScanDescData

定義

  • 種別: struct
  • 定義箇所: src/include/access/heapam.h:54

目的

PostgreSQL のヒープテーブルスキャンにおける実行状態を保持する構造体です。スキャン位置、バッファ管理、およびスキャン戦略の状態を管理します。

シグネチャまたは形状

typedef struct HeapScanDescData {
    TableScanDescData rs_base;
    BlockNumber rs_nblocks;
    BlockNumber rs_startblock;
    BlockNumber rs_numblocks;
    bool        rs_inited;
    OffsetNumber rs_coffset;
    BlockNumber rs_cblock;
    Buffer      rs_cbuf;
    BufferAccessStrategy rs_strategy;
    HeapTupleData rs_ctup;
    ReadStream *rs_read_stream;
    ScanDirection rs_dir;
    BlockNumber rs_prefetch_block;
    ParallelBlockTableScanWorkerData *rs_parallelworkerdata;
    uint32      rs_cindex;
    uint32      rs_ntuples;
    OffsetNumber rs_vistuples[MaxHeapTuplesPerPage];
} HeapScanDescData;

入力と出力

  • 入力: スキャン開始時に rs_base（AM 共通部）およびスキャン範囲（rs_nblocks, rs_startblock）が設定されます。
  • 出力: スキャン進行に伴い、rs_cbuf（バッファ）、rs_ctup（現在のタプル）、rs_vistuples（ページ内の可視タプルオフセット）が更新されます。

前提条件と不変条件

  • rs_cbuf が InvalidBuffer でない場合、当該バッファに対するピンを保持している必要があります。
  • rs_inited が false の場合、スキャンは初期化前であることを示します。
  • rs_numblocks が InvalidBlockNumber の場合、テーブル全体をスキャン対象とみなします。

アルゴリズムまたは振る舞い

  • スキャン開始時に ReadStream を割り当て、スキャン方向（rs_dir）の変化やページ遷移に応じてバッファの解放とプリフェッチブロックの再設定を行います。
  • ページ単位のスキャンでは rs_vistuples を使用してページ内の可視タプルを順次処理します。
  • パラレルスキャン時には rs_parallelworkerdata を使用してページ割り当て情報を共有します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/heapam.c 内の heap_beginscan 等でインスタンス化されます。
  • 呼び出し先: 提供されたソース excerpt には具体的な関数呼び出しの記述はありません。

関連データ構造

  • TableScanDescData: 継承元の AM 共通構造体。
  • HeapTupleData: 現在のタプル情報を保持。
  • ReadStream: 読み込みストリーム管理用。
  • ParallelBlockTableScanWorkerData: パラレルスキャン時のワーカーデータ。

失敗モードと境界条件

  • rs_cbuf が無効な状態でのアクセスは未定義の振る舞いを引き起こす可能性があります。
  • スキャン方向の変更時に ReadStream のリセット処理が正しく行われない場合、プリフェッチの整合性が失われる可能性があります。

保守担当者向けメモ

  • 本構造体は HeapScanDesc 型（ポインタ）として頻繁に参照されます。
  • rs_vistuples 配列のサイズは MaxHeapTuplesPerPage に依存しているため、ページレイアウトの変更時には注意が必要です。
