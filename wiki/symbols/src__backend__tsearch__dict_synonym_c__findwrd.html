findwrd

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/dict_synonym.c:45

目的

文字列から単語を切り出し、末尾にワイルドカード（*）が含まれている場合はプレフィックスフラグを付与して解析する。

シグネチャまたは形状

char *findwrd(char *in, char **end, uint16 *flags)

入力と出力

  • **入力**:
  • in: 解析対象の文字列ポインタ。
  • end: 単語の終了位置を格納するためのポインタ。
  • flags: プレフィックスフラグ（TSL_PREFIX）を格納するためのポインタ。
  • **出力**:
  • 戻り値: 切り出された単語の開始位置を示すポインタ。空行の場合は NULL を返す。
  • end: 単語の末尾位置（* がある場合はその位置）を指す。
  • flags: 単語末尾が * の場合は TSL_PREFIX が設定され、それ以外は 0 が設定される。

前提条件と不変条件

  • in はヌル終端文字列であること。
  • flags が NULL の場合、プレフィックス判定はスキップされる。

アルゴリズムまたは振る舞い

1. 先頭の空白文字を isspace および pg_mblen_cstr を用いてスキップする。 2. 文字列が空であれば *end = NULL とし、NULL を返す。 3. 空白文字が現れるまで単語をスキャンする。 4. 単語の末尾が * であるかを確認する。

  • * であれば、flags に TSL_PREFIX を設定し、*end を * の位置に更新する。
  • それ以外であれば、flags を 0 に初期化し、*end を単語の終了位置に更新する。

    • 5. 単語の開始位置を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • dsynonym_init (src/backend/tsearch/dict_synonym.c:141, 154)
  • **呼び出し先**:
  • pg_mblen_cstr
  • isspace
  • t_iseq

関連データ構造

  • TSL_PREFIX (フラグ定数)

失敗モードと境界条件

  • 入力文字列が空白のみ、または空の場合、NULL を返す。
  • マルチバイト文字を含む文字列に対して pg_mblen_cstr を使用して正しく文字境界を処理する。

保守担当者向けメモ

  • 本関数は辞書ファイルのパース時に使用される。* をプレフィックスとして扱うロジックは、同義語辞書の機能要件に直結しているため、変更時は dsynonym_lexize での検索ロジックとの整合性に注意すること。
