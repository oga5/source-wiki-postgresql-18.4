pg_ultoa_n

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/numutils.c:1054

目的

符号なし32ビット整数（uint32）を文字列形式に変換し、その長さを返します。

シグネチャまたは形状

int pg_ultoa_n(uint32 value, char *a)

入力と出力

  • **入力**:
  • value: 変換対象の uint32 値。
  • a: 変換後の文字列を格納するバッファへのポインタ。
  • **出力**:
  • 戻り値: 変換された文字列の長さ（桁数）。
  • a が指すメモリ領域: 変換結果の文字列が書き込まれます（NUL終端は付与されません）。

前提条件と不変条件

  • a は、変換後の数値が収まる十分なサイズのバッファを指している必要があります。
  • 本関数は NUL 終端を書き込まないため、呼び出し側で必要に応じて終端処理を行う必要があります。

アルゴリズムまたは振る舞い

1. value が 0 の場合、'0' を書き込み 1 を返します。 2. decimalLength32 を使用して必要な桁数を算出します。 3. value が 10000 以上の場合、10000 で割るループを回し、DIGIT_TABLE を用いて一度に2桁ずつバッファへ書き込みます。 4. 残りの数値に対して、100以上、10以上の場合の処理を順次行い、最後に1桁の数値を書き込みます。 5. 最終的に算出した桁数を返します。

  • memcpy（標準ライブラリ）

関連データ構造

  • DIGIT_TABLE: 00〜99 の数値ペアを保持するルックアップテーブル。

失敗モードと境界条件

  • 入力が 0 の場合は正しく '0' を出力します。
  • バッファサイズが不足している場合、メモリ破壊が発生します。

保守担当者向けメモ

  • 本関数はパフォーマンス重視で設計されており、標準の sprintf よりも高速です。
  • DIGIT_TABLE を利用した最適化が行われているため、変更時はテーブルの整合性に注意してください。
  • NUL 終端を行わない仕様は、バッファ操作の効率化を目的としています。
