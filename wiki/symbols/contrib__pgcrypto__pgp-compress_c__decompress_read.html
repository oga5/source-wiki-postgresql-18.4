decompress_read

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-compress.c:235

目的

PullFilter インターフェースを介して、ソースから読み込んだ圧縮データを zlib を用いて伸長し、要求元へ提供する。

シグネチャまたは形状

static int decompress_read(void *priv, PullFilter *src, int len, uint8 **data_p, uint8 *buf, int buflen);

入力と出力

  • **入力**:
  • priv: DecomprData 構造体（伸長状態を保持）。
  • src: 読み込み元の PullFilter。
  • len: 要求された読み込みサイズ。
  • buf, buflen: 伸長結果を格納する一時バッファとそのサイズ。
  • **出力**:
  • 戻り値: 実際に読み込まれたバイト数、またはエラーコード（負の値）。
  • data_p: 伸長されたデータへのポインタを格納。

前提条件と不変条件

  • priv は DecomprData 型として初期化済みであること。
  • zlib のストリーム状態（dec->stream）が有効であること。
  • 内部バッファ dec->buf にデータが残っている場合は、優先的にそのデータが返される。

アルゴリズムまたは振る舞い

1. **バッファ確認**: 内部バッファに伸長済みデータがあれば、要求サイズ分を切り出して即座に返す。 2. **データ読み込み**: 内部ストリームの入力が空の場合、pullf_read を使用してソースからデータを取得する。 3. **伸長処理**: inflate() を呼び出し、データを伸長する。

  • 入力データがある場合は Z_SYNC_FLUSH を使用。
  • 入力データがない場合は Z_FINISH を使用。

    • 4. **終端処理**: Z_STREAM_END に到達した場合、PGPストリームの仕様に従い、後続に余分なデータがないかを確認し、dec->eof をセットする。 5. **再帰的解決**: 処理完了後、restart ラベルへ戻り、要求されたサイズを満たすまで処理を繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**: pgp_decompress_filter 関数内で PullFilter のメソッドとして登録される。
  • **呼び出し先**:
  • pullf_read: ソースからのデータ取得。
  • inflate: zlib による伸長処理。
  • px_debug: エラーログ出力。

関連データ構造

  • DecomprData: 伸長処理の状態（z_stream、バッファ、EOFフラグ等）を保持する構造体。
  • PullFilter: フィルタパイプラインの抽象インターフェース。

失敗モードと境界条件

  • **伸長エラー**: inflate() が Z_OK または Z_STREAM_END 以外を返した場合、PXE_PGP_CORRUPT_DATA を返す。
  • **データ破損**: ストリーム終了後に余分なデータが検出された場合、PXE_PGP_CORRUPT_DATA を返す。
  • **ソース読み込みエラー**: pullf_read が負の値を返した場合、その値をそのまま伝播させる。

保守担当者向けメモ

  • zlib の inflate 呼び出し時に Z_SYNC_FLUSH を使用しているのは、PGPのパケット構造に合わせ、可能な限り出力を得るためである。
  • ストリーム終了後の pullf_read は、PGP特有の終端パケットを消費するための措置であり、削除しないこと。
