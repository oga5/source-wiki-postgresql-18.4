SingleBoundSortItem

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/rangetypes_gist.c:55

目的

GiSTインデックスの構築や分割処理において、範囲型の境界（RangeBound）をソート可能な形式で保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	int			index;
	RangeBound	bound;
} SingleBoundSortItem;

入力と出力

  • index: 関連付けられたデータ項目を識別するための整数値。
  • bound: 比較対象となる RangeBound 型の境界値。

前提条件と不変条件

  • 本構造体は、主に qsort_arg を用いたソート操作の対象として利用されます。
  • 比較関数内では、SingleBoundSortItem へのポインタとしてキャストされた引数が渡されることが前提となります。

アルゴリズムまたは振る舞い

  • インデックスページ内の各範囲の境界を抽出し、本構造体の配列に格納することで、効率的なソートや分割アルゴリズム（range_gist_picksplit 等）の実行を可能にします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/rangetypes_gist.c 内のインデックス分割ロジック（range_gist_picksplit 等）から、メモリ確保（palloc）およびソート（qsort_arg）の対象として参照されています。
  • **呼び出し先**: 比較関数（src/backend/utils/adt/rangetypes_gist.c:1733-1734 付近）において、本構造体へのポインタがキャストされて利用されます。

関連データ構造

  • RangeBound: 範囲型の境界を定義する構造体。
  • range_gist_picksplit: 本構造体を利用してインデックスの分割候補を評価する関数。

失敗モードと境界条件

  • palloc によるメモリ確保失敗時にエラーが発生する可能性があります。
  • ソート対象の要素数（maxoff）が0の場合の挙動については、呼び出し側のロジックに依存します。

保守担当者向けメモ

  • 本構造体は rangetypes_gist.c 内でのみ利用されるローカルなデータ構造です。
  • 比較ロジックを変更する際は、qsort_arg に渡される比較関数のキャストと、本構造体のフィールド構成が一致していることを確認してください。
