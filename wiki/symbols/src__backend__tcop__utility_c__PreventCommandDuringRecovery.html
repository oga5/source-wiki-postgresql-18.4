PreventCommandDuringRecovery

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/utility.c:440

目的

データベースがリカバリモード（スタンバイ状態など）にある際、実行が許可されていない特定のSQLコマンドをブロックし、エラーを発生させるために使用されます。

シグネチャまたは形状

void PreventCommandDuringRecovery(const char *cmdname)

入力と出力

  • **入力**: cmdname (const char *) - エラーメッセージに含めるコマンド名。
  • **出力**: なし。リカバリ中であればエラーを投げて処理を中断します。

前提条件と不変条件

  • RecoveryInProgress() が真を返す環境下で呼び出された場合、必ず ERRCODE_READ_ONLY_SQL_TRANSACTION エラーが発生します。

アルゴリズムまたは振る舞い

1. RecoveryInProgress() を呼び出し、現在のシステムがリカバリ中かどうかを確認します。 2. リカバリ中の場合、ereport を使用して ERROR レベルの例外を発生させます。 3. エラーメッセージには「cannot execute %s during recovery」という形式で、引数として渡された cmdname が埋め込まれます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/async.c: NOTIFY コマンド実行時
  • src/backend/tcop/utility.c: standard_ProcessUtility 内のコマンド実行時
  • src/backend/utils/adt/xid8funcs.c: pg_current_xact_id() 実行時
  • **呼び出し先**:
  • RecoveryInProgress() (システム状態確認)
  • ereport() (エラー報告)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • **失敗モード**: システムがリカバリ中であると判定された場合、例外が発生し、呼び出し元の処理は中断されます。
  • **境界条件**: リカバリ中でない場合は何も行わず、正常に制御を戻します。

保守担当者向けメモ

  • 本関数は、読み取り専用のリカバリ環境で実行すべきではないコマンドのガードとして機能します。新しいコマンドを追加し、それがリカバリ中に許可されない性質のものである場合は、この関数を適切に呼び出す必要があります。
