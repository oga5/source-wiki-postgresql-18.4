xl_relmap_update

定義

  • 種別: struct
  • 定義箇所: src/include/utils/relmapper.h:27

目的

マップされたカタログ（Mapped Catalogs）の更新情報をWAL（Write-Ahead Log）に記録するためのデータ構造です。システムカタログの物理ファイル位置が変更された際、リカバリ時にその変更を再適用するために使用されます。

シグネチャまたは形状

typedef struct xl_relmap_update
{
    Oid         dbid;           /* データベースID、共有マップの場合は0 */
    Oid         tsid;           /* データベースのテーブルスペース、またはpg_global */
    int32       nbytes;         /* relmapデータのサイズ */
    char        data[FLEXIBLE_ARRAY_MEMBER];
} xl_relmap_update;

入力と出力

  • **入力**: 構造体の各フィールド（dbid, tsid, nbytes）および data 配列に格納されるマッピング情報。
  • **出力**: WALレコードの一部として書き込まれ、リカバリ時に relmap_redo 関数によって読み取られます。

前提条件と不変条件

  • dbid が 0 の場合、共有マップ（pg_global）を指します。
  • data フィールドは可変長配列として扱われ、nbytes で指定されたサイズのデータが後続します。
  • マップファイル更新はアトミックに行われる必要があり、本構造体はその永続化の整合性を保証するWALログの基盤となります。

アルゴリズムまたは振る舞い

1. RelationMapUpdateMap が呼び出されると、メモリ上のマッピングが更新されます。 2. 更新内容をWALに記録するため、本構造体が初期化されます。 3. XLogRecGetData を通じてWALレコードから本構造体が復元されます。 4. リカバリ時には relmap_redo がこの構造体を参照し、物理ファイル（pg_filenode.map）を再構築します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/relmapper.c: マッピング更新時に本構造体をインスタンス化し、WALに書き込みます。
  • **呼び出し先**:
  • src/backend/access/rmgrdesc/relmapdesc.c: WALレコードのデコードおよびデバッグ出力時に本構造体をキャストして参照します。

関連データ構造

  • MinSizeOfRelmapUpdate: offsetof(xl_relmap_update, data) として定義されており、構造体の最小サイズを算出するために使用されます。

失敗モードと境界条件

  • nbytes の値が不正な場合、WALの読み込み時にメモリ破壊やパースエラーが発生する可能性があります。
  • マップファイル自体が破損している場合、本構造体を用いたリカバリ処理が失敗し、データベースの起動が阻害される可能性があります。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズ計算には sizeof(xl_relmap_update) ではなく MinSizeOfRelmapUpdate を使用してください。
  • relmap_redo の実装を変更する際は、本構造体のフィールド定義と整合性が取れていることを確認してください。
