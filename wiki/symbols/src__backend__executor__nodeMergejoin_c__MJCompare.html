MJCompare

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMergejoin.c:385

目的

マージ結合において、2つの入力タプルの結合キーを比較し、結合の進行方向（outer/innerのどちらをスキップするか）を決定する。

シグネチャまたは形状

static int MJCompare(MergeJoinState *mergestate)

入力と出力

  • **入力**: MergeJoinState *mergestate（結合状態および比較対象のタプルデータを含む構造体）
  • **出力**: int 型の比較結果。
  • 0: 両方のキーが等しい。
  • > 0: outer側がinner側より大きい（innerをスキップ）。
  • < 0: outer側がinner側より小さい（outerをスキップ）。

前提条件と不変条件

  • mergestate->js.ps.ps_ExprContext が有効であること。
  • 比較対象のタプルは、MergeJoinClause に格納されたソート順序に従って準備されていること。
  • 比較処理は econtext->ecxt_per_tuple_memory コンテキスト内で実行される。

アルゴリズムまたは振る舞い

1. ResetExprContext を呼び出し、式評価コンテキストをリセットする。 2. mergestate->mj_Clauses に格納された結合句を順次走査する。 3. 各結合句について、ApplySortComparator を用いて ldatum と rdatum を比較する。 4. NULL同士の比較（NULL = NULL）が発生した場合は、nulleqnull フラグを立てて継続する。 5. 比較結果が 0 以外であればループを終了する。 6. 全ての比較が終了した時点で、result == 0 かつ（nulleqnull が真、または mj_ConstFalseJoin が真）の場合、強制的に result = 1 に書き換える。これにより、NULL同士や定数偽の結合条件を「不一致」として扱い、inner側をスキップさせる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeMergejoin.c 内のメインループ（ExecMergeJoin 等のステートマシン処理）。
  • **呼び出し先**:
  • ApplySortComparator（比較ロジックの実行）
  • ResetExprContext / MemoryContextSwitchTo（メモリ管理）

関連データ構造

  • MergeJoinState: 結合ノードの状態を保持。
  • MergeJoinClause: 結合キーの式や比較関数（SortSupport）を保持。

失敗モードと境界条件

  • **NULLの扱い**: NULL同士の比較は等価とはみなされず、result = 1 を返すことでinner側をスキップする挙動をとる。
  • **メモリリーク**: 比較関数がメモリを消費する可能性があるため、短命なメモリコンテキスト（ecxt_per_tuple_memory）で実行される。

保守担当者向けメモ

  • 本関数はマージ結合の核心的な比較ロジックである。デバッグ時には MJ_dump マクロによるトレースが有効である。
  • mj_ConstFalseJoin が設定されている場合、結合条件が常に偽となるため、強制的に不一致として扱う必要がある点に注意すること。
