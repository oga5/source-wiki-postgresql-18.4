logicalrep_read_begin

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/proto.c:62

目的

論理レプリケーションの通信プロトコルにおける BEGIN メッセージをデシリアライズし、トランザクション開始に必要な情報を構造体に格納します。

シグネチャまたは形状

void logicalrep_read_begin(StringInfo in, LogicalRepBeginData *begin_data)

入力と出力

  • **入力**:
  • in: 受信したメッセージデータを含む StringInfo バッファ。
  • begin_data: 読み取ったデータを格納する LogicalRepBeginData 構造体へのポインタ。
  • **出力**: なし（begin_data を介して値を返却）。

前提条件と不変条件

  • in バッファには、プロトコル定義に従った BEGIN メッセージのバイナリデータが格納されている必要があります。
  • final_lsn は InvalidXLogRecPtr であってはなりません。

アルゴリズムまたは振る舞い

1. pq_getmsgint64 を使用して final_lsn を読み取ります。 2. final_lsn が InvalidXLogRecPtr の場合、エラーを発生させます。 3. pq_getmsgint64 を使用して committime を読み取ります。 4. pq_getmsgint を使用して 4 バイトの xid を読み取ります。 5. 読み取った各値を begin_data 構造体の対応するフィールドに代入します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/worker.c:992 にて呼び出されていることが確認されています。
  • **呼び出し先**: pq_getmsgint64 および pq_getmsgint を呼び出します。

関連データ構造

  • LogicalRepBeginData: トランザクション開始情報を保持する構造体。
  • StringInfo: 通信バッファ。

失敗モードと境界条件

  • final_lsn が InvalidXLogRecPtr である場合、elog(ERROR) により処理が中断されます。
  • StringInfo の読み込み順序がプロトコル定義と一致しない場合、不正なデータが読み取られるか、バッファオーバーフロー等のエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は logicalproto.h で定義されたプロトコル仕様に依存しています。メッセージ形式を変更する場合は、対応する logicalrep_write_begin 関数と同期をとる必要があります。
