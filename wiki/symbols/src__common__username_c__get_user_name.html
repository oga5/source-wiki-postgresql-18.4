get_user_name

定義

  • 種別: function
  • 定義箇所: src/common/username.c:30

目的

実行中のプロセスに関連付けられた現在のユーザー名を取得します。プラットフォーム固有のユーザーID解決メカニズムを抽象化し、PostgreSQLのフロントエンドおよびバックエンド双方で利用可能なインターフェースを提供します。

シグネチャまたは形状

const char *get_user_name(char **errstr);

入力と出力

  • **入力**: char **errstr - エラー発生時に詳細なエラーメッセージを格納するためのポインタ。
  • **出力**: 成功時はユーザー名を表す const char * を返します。失敗時は NULL を返し、errstr に palloc されたエラーメッセージを格納します。

前提条件と不変条件

  • errstr は呼び出し元で適切に解放する必要があります。
  • 本実装はスレッドセーフではありません。
  • Windows環境では静的バッファを使用しているため、複数回呼び出すと以前の戻り値が上書きされる可能性があります。

アルゴリズムまたは振る舞い

1. errstr を NULL に初期化します。 2. プラットフォームに応じて以下の処理を行います。

  • **POSIX**: geteuid() でユーザーIDを取得し、getpwuid() でパスワードデータベースを検索します。成功時は pw_name を返します。
  • **Windows**: GetUserName() を呼び出し、静的バッファにユーザー名を格納して返します。

    • 3. 取得に失敗した場合は、psprintf を用いてエラーメッセージを生成し、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/util.c:335
  • src/common/username.c:79 (get_user_name_or_exit 内)
  • src/test/regress/pg_regress.c:1029
  • **呼び出し先**:
  • POSIX環境: geteuid(), getpwuid()
  • Windows環境: GetUserName(), GetLastError()
  • 共通: psprintf()

関連データ構造

  • struct passwd (POSIX環境で使用)

失敗モードと境界条件

  • ユーザー情報の検索に失敗した場合、NULL を返し、errstr にエラー詳細を格納します。
  • Windows環境において、バッファサイズ（256+1）を超えるユーザー名は取得できません。

保守担当者向けメモ

  • errstr に返されるメッセージは palloc されているため、メモリリークを防ぐために呼び出し元での解放が必須です。
  • Windows環境の静的バッファの制約により、並行実行環境での利用には注意が必要です。
