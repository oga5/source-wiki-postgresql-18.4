spgist_name_choose

定義

  • 種別: function
  • 定義箇所: src/test/modules/spgist_name_ops/spgist_name_ops.c:123

目的

SP-GiSTインデックスにおいて、挿入対象の name 型値をどのノードへ配置するか、あるいはインデックス構造の分割（Split）が必要かを決定する。

シグネチャまたは形状

Datum spgist_name_choose(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgChooseIn（挿入対象データ、現在のレベル、プレフィックス情報等）および spgChooseOut（決定されたアクション）を受け取る。
  • **出力**: void（結果は spgChooseOut 構造体に格納される）。

前提条件と不変条件

  • name 型の値は内部的に text 型として扱われる。
  • 照合順序（Collation）は考慮せず、バイナリ比較を行う。
  • reconstructedValue はロング形式の varlena ヘッダーを持つものとして処理される。

アルゴリズムまたは振る舞い

1. **プレフィックス照合**: in->hasPrefix が真の場合、既存のプレフィックスと入力値の共通部分を commonPrefix で算出する。

  • 一致しない場合、spgSplitTuple を選択し、プレフィックスを分割して新しいノードを構成する。

    • 2. **ノード選択**: プレフィックスの直後のバイト（nodeChar）を特定し、既存のノードラベル配列を searchChar で検索する。

  • 見つかった場合: spgMatchNode を選択し、該当ノードへ降下する。
  • 見つからない場合:
  • allTheSame が真の場合: spgSplitTuple を選択し、ダミーラベル -2 を用いて分割する。
  • それ以外の場合: spgAddNode を選択し、新しいノードを追加する。

    • 3. **戻り値**: 決定したアクション（spgMatchNode, spgSplitTuple, spgAddNode）を out に設定して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのSP-GiSTインデックスマネージャ（直接の呼び出し元はソースコード上では明示されていないが、PG_FUNCTION_INFO_V1 マクロにより登録されている）。
  • **呼び出し先**: commonPrefix, formTextDatum, searchChar, palloc。

関連データ構造

  • spgChooseIn, spgChooseOut: SP-GiSTの選択フェーズ用構造体。
  • Name: PostgreSQLの name 型データ。

失敗モードと境界条件

  • **プレフィックス不一致**: 入力値が現在のプレフィックスと完全に一致しない場合、spgSplitTuple による構造変更が発生する。
  • **allTheSame**: ノードラベルが重複している場合、spgAddNode が使用できないため、強制的に spgSplitTuple へ移行する。

保守担当者向けメモ

  • 本関数は src/backend/access/spgist/spgtextproc.c のロジックを name 型向けに移植したものである。
  • allTheSame フラグが立っている際の分割処理など、SP-GiST特有の複雑なエッジケースが含まれるため、コアのテキスト実装と照らし合わせて変更を行うこと。
