ReindexType

定義

  • 種別: enum
  • 定義箇所: src/bin/scripts/reindexdb.c:27

目的

reindexdb ユーティリティにおいて、再構築（REINDEX）の対象となるオブジェクトの種類を識別するために使用されます。

シグネチャまたは形状

typedef enum ReindexType
{
	REINDEX_DATABASE,
	REINDEX_INDEX,
	REINDEX_SCHEMA,
	REINDEX_SYSTEM,
	REINDEX_TABLE,
} ReindexType;

入力と出力

  • 本シンボルは列挙型であり、関数への引数として渡されることで、後続の SQL 生成や処理分岐の制御フラグとして機能します。

前提条件と不変条件

  • REINDEX_SYSTEM は並列実行をサポートしていません。

アルゴリズムまたは振る舞い

  • reindexdb の各処理関数（reindex_one_database, gen_reindex_command, run_reindex_command 等）において、この列挙値に基づいて実行する SQL 文の構築や、対象オブジェクトのリスト取得方法が切り替えられます。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数および reindex_one_database 等の制御関数から、処理対象を特定する引数として渡されます。
  • **呼び出し先**: gen_reindex_command や run_reindex_command、get_parallel_tables_list など、reindexdb.c 内の主要な処理関数がこの型を引数として受け取ります。

関連データ構造

  • ConnParams: データベース接続パラメータ。
  • ParallelSlotArray: 並列実行時の接続管理構造体。

失敗モードと境界条件

  • サーバーバージョンが古い場合、特定の ReindexType に対して CONCURRENTLY などのオプションを付与しようとすると、gen_reindex_command 内で pg_fatal が呼び出され、プログラムが停止する可能性があります。

保守担当者向けメモ

  • 新しい再構築対象の種類を追加する場合は、この列挙型を拡張し、gen_reindex_command 内の SQL 生成ロジックを更新する必要があります。
