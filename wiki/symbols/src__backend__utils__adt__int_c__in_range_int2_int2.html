in_range_int2_int2

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/int.c:767

目的

int2 型の値を対象とした範囲内判定（in_range）処理を実行します。

シグネチャまたは形状

Datum in_range_int2_int2(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して5つの引数を受け取ります。
  • 第1〜第5引数は Datum 型として渡されます。
  • 第3引数は内部的に int16 として取得され、int32 にキャストされます。
  • **出力**: Datum 型を返します。

前提条件と不変条件

  • 本関数は in_range_int2_int4 関数へのラッパーとして機能します。
  • 呼び出し元は PostgreSQL の関数呼び出し規約（PG_FUNCTION_ARGS）に従う必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_DATUM マクロを使用して引数を取得します。 2. 第3引数（int16）を int32 にキャストします。 3. DirectFunctionCall5 を使用して in_range_int2_int4 を呼び出し、その結果をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し先**: in_range_int2_int4
  • **呼び出し元**: ソースコード内に明示的な呼び出し元の記述はありません。

関連データ構造

  • Datum: PostgreSQL の汎用データ表現型。
  • int2 / int16: 2バイト整数型。
  • int32: 4バイト整数型。

失敗モードと境界条件

  • 内部で呼び出される in_range_int2_int4 の失敗モードに依存します。
  • 引数の型変換（int16 から int32）は安全に行われます。

保守担当者向けメモ

  • コードの重複を避けるため、in_range_int2_int4 への委譲を行っています。ロジックの変更が必要な場合は、呼び出し先の関数を確認してください。
