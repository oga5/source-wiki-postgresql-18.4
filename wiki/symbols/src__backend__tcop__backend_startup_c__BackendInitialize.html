BackendInitialize

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/backend_startup.c:140

目的

PostgreSQLのバックエンドプロセス起動直後において、クライアントとの通信確立、プロトコルネゴシエーション、および接続パラメータの収集を行う。

シグネチャまたは形状

static void BackendInitialize(ClientSocket *client_sock, CAC_state cac);

入力と出力

  • **入力**:
  • client_sock: クライアントとの通信用ソケット。
  • cac: データベースの状態（CAC_state）。
  • **出力**: なし（成功時は後続処理へ移行、失敗時は FATAL エラーまたは _exit で終了）。

前提条件と不変条件

  • 共有メモリへのアクセスは禁止（InitProcess() 未実行のため）。
  • エラー発生時は proc_exit ではなく _exit(1) を使用すること。
  • TopMemoryContext 上で Port 構造体が割り当てられる。

アルゴリズムまたは振る舞い

1. **初期設定**: ReserveExternalFD() でファイル記述子を確保し、PreAuthDelay が設定されていれば待機する。 2. **通信準備**: pq_init() で通信層を初期化し、MyProcPort を設定。 3. **シグナル設定**: SIGTERM を process_startup_packet_die に割り当て、タイムアウト用ハンドラを登録。 4. **ホスト情報取得**: pg_getnameinfo_all() で接続元情報を取得し、Port 構造体に格納。 5. **接続ログ**: log_connections 設定に基づき接続イベントを記録。 6. **ネゴシエーション**: ProcessSSLStartup() でSSLハンドシェイクを行い、ProcessStartupPacket() でスタートアップパケットを解析。 7. **状態検証**: データベースの状態（cac）を確認し、接続不可であれば FATAL エラーを発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/backend_startup.c 内の BackendMain から呼び出される。
  • **呼び出し先**: pq_init, pg_getnameinfo_all, ProcessSSLStartup, ProcessStartupPacket, RegisterTimeout 等。

関連データ構造

  • Port: 接続情報を保持する構造体。
  • ClientSocket: クライアント接続用ソケット。
  • CAC_state: データベースの接続可否状態。

失敗モードと境界条件

  • **タイムアウト**: スタートアップパケットの受信が AuthenticationTimeout を超えた場合、接続を強制終了する。
  • **データベース状態**: 起動中やスタンバイモード等の理由で接続が許可されない場合、FATAL エラーで終了する。
  • **通信エラー**: pg_getnameinfo_all の失敗は警告として記録される。

保守担当者向けメモ

  • 本関数内では共有メモリに触れてはならない。
  • PostgresMain が再度 InitializeTimeouts を呼び出すため、本関数で登録した STARTUP_PACKET_TIMEOUT は後続処理には引き継がれない。
  • USE_INJECTION_POINTS が有効な場合、テスト用のエラー注入ポイントが存在する。
