brin_revmap_data

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/brinfuncs.c:383

目的

BRIN（Block Range Index）のリビジョンマップページから、格納されている ItemPointer（TID）の配列を抽出し、SQL 結果セットとして列挙する。

シグネチャまたは形状

Datum brin_revmap_data(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: bytea 型の生ページデータ（PG_GETARG_BYTEA_P(0)）。
  • **出力**: ItemPointer 型のセット（SRF_RETURN_NEXT を通じて順次返却）。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要。
  • 入力された bytea データは、verify_brin_page により BRIN_PAGETYPE_REVMAP 型として検証される必要がある。
  • ページが新規（PageIsNew）である場合、結果は NULL となる。

アルゴリズムまたは振る舞い

1. **権限チェック**: superuser() を呼び出し、非特権ユーザーからのアクセスを拒否する。 2. **初期化**: SRF_IS_FIRSTCALL() ブロックにて、FuncCallContext を作成し、メモリコンテキストを切り替える。 3. **ページ検証**: verify_brin_page を使用して、対象ページがリビジョンマップであることを確認する。 4. **データ抽出**: ページ内容から RevmapContents 構造体を取得し、rm_tids 配列へのポインタを保持する。 5. **反復処理**: SRF_PERCALL_SETUP() を経由し、REVMAP_PAGE_MAXITEMS に達するまで state->tids の各要素を順次返す。 6. **終了**: 全要素の列挙完了後、SRF_RETURN_DONE() を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • superuser()
  • SRF_FIRSTCALL_INIT(), SRF_PERCALL_SETUP(), SRF_RETURN_NEXT(), SRF_RETURN_DONE()
  • verify_brin_page()
  • PageIsNew(), PageGetContents()
  • MemoryContextSwitchTo()

関連データ構造

  • ItemPointerData: 抽出対象となる TID のデータ構造。
  • RevmapContents: リビジョンマップページ内のデータを保持する構造体。
  • FuncCallContext: 複数回の関数呼び出し間で状態（現在のインデックス idx 等）を保持するために使用。

失敗モードと境界条件

  • **権限不足**: スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー終了する。
  • **ページ型不一致**: verify_brin_page が失敗した場合、エラーが発生する。
  • **空ページ**: PageIsNew の場合、NULL を返す。
  • **範囲外アクセス**: state->idx が REVMAP_PAGE_MAXITEMS を超える前に列挙を終了する。

保守担当者向けメモ

  • 本関数は pageinspect 拡張の一部であり、デバッグ用途に限定される。
  • FuncCallContext を使用したセットリターニング関数であるため、メモリ管理（multi_call_memory_ctx）に注意すること。
