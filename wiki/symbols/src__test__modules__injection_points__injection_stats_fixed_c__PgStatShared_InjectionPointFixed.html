PgStatShared_InjectionPointFixed

定義

  • 種別: struct
  • 定義箇所: src/test/modules/injection_points/injection_stats_fixed.c:37

目的

PostgreSQLのインジェクションポイントにおける「固定数統計」を共有メモリ上で管理するための構造体です。統計カウンタの排他制御と、リセット操作を考慮した累積値の保持を目的とします。

シグネチャまたは形状

typedef struct PgStatShared_InjectionPointFixed
{
    LWLock        lock;             /* 全カウンタを保護 */
    uint32        changecount;      /* 変更回数 */
    PgStat_StatInjFixedEntry stats;         /* 現在の統計値 */
    PgStat_StatInjFixedEntry reset_offset;  /* リセット時のオフセット値 */
} PgStatShared_InjectionPointFixed;

入力と出力

本構造体はデータ保持用であり、直接的な関数入出力は持ちません。各フィールドは pgstat フレームワークのコールバック関数を通じて読み書きされます。

前提条件と不変条件

  • lock フィールドにより、stats および reset_offset へのアクセスは排他制御される必要があります。
  • reset_offset は、統計リセット時に stats の現在の値をコピーすることで更新され、以降の統計算出時の基準値として機能します。

アルゴリズムまたは振る舞い

  • **統計更新**: pgstat_report_inj_fixed が呼び出されると、lock を取得した状態で stats 内のカウンタがインクリメントされます。
  • **スナップショット生成**: injection_points_stats_fixed 経由で呼び出されるスナップショット処理において、stats から reset_offset を差し引くことで、リセット以降の純粋な統計値が算出されます。
  • **リセット**: injection_stats_fixed_reset_all_cb が実行されると、現在の stats の値が reset_offset にコピーされます。

呼び出し元と呼び出し先

  • **呼び出し元**: pgstat_register_inj_fixed（初期化時）、pgstat_report_inj_fixed（更新時）、injection_stats_fixed_snapshot_cb（スナップショット取得時）、injection_stats_fixed_reset_all_cb（リセット時）などで参照されます。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • PgStat_StatInjFixedEntry: 統計の各項目（attach, detach, run, cached, loaded）を保持する構造体。
  • LWLock: 共有メモリ上のデータ保護に使用される軽量ロック。

失敗モードと境界条件

  • **ロック競合**: 高頻度でインジェクションポイントが実行される環境では、lock による競合が発生する可能性があります。
  • **オーバーフロー**: PgStat_StatInjFixedEntry 内の各カウンタが uint32 等の型制限を超えた場合の挙動は、基盤となる pgstat の仕様に依存します。

保守担当者向けメモ

  • 本構造体は pgstat のカスタム統計機能を利用しています。構造体のレイアウトを変更する場合は、pgstat への登録情報（shared_size や shared_data_off）との整合性に注意してください。
  • FIXED_COMP マクロが本ファイル内で定義されており、スナップショット生成時のオフセット補正ロジックで使用されています。
