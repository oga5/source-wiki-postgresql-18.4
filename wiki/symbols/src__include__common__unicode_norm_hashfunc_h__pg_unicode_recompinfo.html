目的

Unicode 正規化処理における文字の再合成（Recomposition）を高速に行うための、ハッシュ関数とルックアップテーブルを保持する構造体です。

シグネチャまたは形状

typedef struct
{
	const uint16	*inverse_lookup;
	cp_hash_func	hash;
	int		num_recomps;
} pg_unicode_recompinfo;

入力と出力

本構造体自体はデータ保持用であり、直接的な入出力は持ちません。保持する hash 関数ポインタは、文字コード（key）を入力としてハッシュ値を計算し、inverse_lookup テーブルと組み合わせて変換情報を取得するために使用されます。

前提条件と不変条件

  • 本構造体は、完全ハッシュ関数（Recomp_hash_func）と整合するテーブル構成を前提としています。
  • 保持するハッシュ関数は、特定のデータセットに対して衝突が発生しない「完全ハッシュ」として設計されています。

アルゴリズムまたは振る舞い

1. hash 関数により、入力された文字コードからハッシュ値を算出します。 2. 算出されたハッシュ値をインデックスとして、inverse_lookup テーブルを参照します。 3. 取得したオフセット値を用いて、Unicode 再合成に必要な情報を特定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/unicode_norm.c 内で UnicodeRecompInfo として参照されています。
  • 呼び出し先: src/include/common/unicode_norm_hashfunc.h 内で定義される cp_hash_func 型の関数（Recomp_hash_func 等）と連携します。

関連データ構造

  • cp_hash_func: ハッシュ計算を行う関数ポインタ型。
  • UnicodeRecompInfo: 本構造体の静的インスタンス。
  • common/unicode_norm_table.h: 再合成に必要なメインデータ構造を定義しており、本構造体と組み合わせて使用されます。
