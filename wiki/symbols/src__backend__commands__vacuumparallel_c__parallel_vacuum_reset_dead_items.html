parallel_vacuum_reset_dead_items

定義

  • 種別: function
  • 定義箇所: src/backend/commands/vacuumparallel.c:474

目的

並列VACUUM処理において、共有メモリ上の削除対象タプル格納領域（TidStore）を破棄し、新しい領域を再確保して初期化する。

シグネチャまたは形状

void parallel_vacuum_reset_dead_items(ParallelVacuumState *pvs)

入力と出力

  • **入力**: ParallelVacuumState *pvs - 並列VACUUMの現在の状態を保持する構造体。
  • **出力**: なし（副作用として pvs 内の dead_items および共有メモリ上の制御情報が更新される）。

前提条件と不変条件

  • pvs が有効な ParallelVacuumState を指していること。
  • pvs->shared が初期化済みであること。
  • TidStore の再作成時に dead_items_info->max_bytes が保持されていること。

アルゴリズムまたは振る舞い

1. 現在の TidStore を TidStoreDestroy で破棄し、関連するDSAセグメントを解放する。 2. dead_items_info->max_bytes を使用して、新しい共有 TidStore を TidStoreCreateShared で作成する。 3. 新しく作成された TidStore のDSAハンドルおよびストアハンドルを pvs->shared に格納し、共有メモリ上のポインタを更新する。 4. dead_items_info->num_items を 0 にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/vacuumlazy.c:3564
  • **呼び出し先**: TidStoreDestroy, TidStoreCreateShared, dsa_get_handle, TidStoreGetDSA, TidStoreGetHandle

関連データ構造

  • ParallelVacuumState: 並列VACUUMの全体状態を管理する構造体。
  • VacDeadItemsInfo: 共有メモリ上の削除対象タプルに関するメタデータ。
  • TidStore: 削除対象タプルを保持する共有メモリ上のデータ構造。

失敗モードと境界条件

  • 共有メモリ（DSA）の枯渇により TidStoreCreateShared が失敗する可能性がある。

保守担当者向けメモ

  • 本関数は、インデックス処理の各フェーズ間で削除対象タプルリストをリセットする際に使用される。
  • LWTRANCHE_PARALLEL_VACUUM_DSA を使用してメモリを確保しているため、ロックの競合やメモリ管理の整合性に注意すること。
