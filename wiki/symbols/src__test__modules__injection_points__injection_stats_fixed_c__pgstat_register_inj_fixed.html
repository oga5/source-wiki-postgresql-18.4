pgstat_register_inj_fixed

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_stats_fixed.c:128

目的

PostgreSQLの統計フレームワーク（pgstat）に対し、インジェクションポイントの固定数統計（PGSTAT_KIND_INJECTION_FIXED）を登録し、統計機能の有効化状態をマークします。

シグネチャまたは形状

void pgstat_register_inj_fixed(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数および統計フレームワークの状態を更新）

前提条件と不変条件

  • 本関数はモジュールロード時に呼び出されることを想定しています。
  • injection_stats_fixed 構造体が定義済みであり、pgstat フレームワークに登録可能である必要があります。

アルゴリズムまたは振る舞い

1. pgstat_register_kind を呼び出し、PGSTAT_KIND_INJECTION_FIXED 種別を injection_stats_fixed コールバック構造体と共に登録します。 2. グローバル変数 inj_fixed_loaded を true に設定し、統計がロード済みであることを示します。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/injection_points/injection_points.c:575
  • 呼び出し先: pgstat_register_kind（pgstat フレームワークの関数）

関連データ構造

  • injection_stats_fixed: 統計種別のコールバックを保持する構造体。
  • PGSTAT_KIND_INJECTION_FIXED: 登録対象の統計種別識別子。

失敗モードと境界条件

  • 統計フレームワークが初期化されていない状態で呼び出された場合の挙動は、pgstat_register_kind の実装に依存します。

保守担当者向けメモ

  • 本関数は injection_stats_fixed.c 内の初期化ロジックの一部です。統計の登録順序やタイミングを変更する場合は、pgstat フレームワークのライフサイクルを確認してください。
