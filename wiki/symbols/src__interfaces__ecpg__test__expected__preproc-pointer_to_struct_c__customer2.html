customer2

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c:63

目的

ECPG（Embedded SQL for C）プリプロセッサの型変換および構造体ポインタのハンドリングを検証するためのテスト用データ構造。

シグネチャまたは形状

typedef struct {
    struct varchar_2 { int len; char arr[50]; } name;
    int phone;
} customer2;

入力と出力

  • 入力: SQLクエリの結果セットをマッピングするためのホスト変数として使用される。
  • 出力: ECPGdo 関数を介してデータベースからフェッチされた値が、この構造体のメンバ（name および phone）に格納される。

前提条件と不変条件

  • 構造体内の name メンバは、ECPGが期待する varchar 型の形式（len と arr）に従う必要がある。
  • ECPGdo 実行時に sizeof(customer2) を用いてメモリレイアウトが正しく指定されることが前提となる。

アルゴリズムまたは振る舞い

  • malloc を使用してヒープ上に確保されたメモリ領域に対し、SQLのフェッチ結果を直接書き込む対象として利用される。
  • プリプロセッサによって、SQLのデータ型とC言語の構造体メンバ間のマッピングが自動生成される。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c:97（メモリ確保）
  • src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c:196, 198（ECPGdo 内での型指定）
  • src/interfaces/ecpg/test/expected/preproc-array_of_struct.c:68, 93, 192, 194（別テストファイルでの参照）
  • 呼び出し先:
  • 構造体自体はデータ保持用であり、特定の関数を呼び出すものではない。

関連データ構造

  • varchar_2: customer2 内部で定義されているネストされた構造体。
  • customer / customer3 / customer4: 同様の目的で定義された他のテスト用構造体。

失敗モードと境界条件

  • malloc によるメモリ確保失敗時、後続の ECPGdo 実行でセグメンテーション違反が発生する可能性がある。
  • varchar_2 の arr[50] を超えるデータがフェッチされた場合、バッファオーバーフローのリスクがある。

保守担当者向けメモ

  • 本ファイルはプリプロセッサの出力結果であるため、直接編集しないこと。
  • 構造体のメンバ構成を変更する場合は、対応する .pgc ファイル（ソース）を修正し、プリプロセッサを再実行して期待値を更新する必要がある。
