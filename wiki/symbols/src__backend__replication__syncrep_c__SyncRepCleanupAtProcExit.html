SyncRepCleanupAtProcExit

定義

  • 種別: function
  • 定義箇所: src/backend/replication/syncrep.c:415

目的

バックエンドプロセス終了時に、同期レプリケーションの待機キューから当該プロセスを安全に除去し、共有メモリ上の整合性を維持する。

シグネチャまたは形状

void SyncRepCleanupAtProcExit(void)

入力と出力

  • 入力: なし（グローバルな MyProc を参照）
  • 出力: なし

前提条件と不変条件

  • MyProc が初期化されていること。
  • SyncRepLock が利用可能であること。
  • 呼び出し側はプロセス終了処理のコンテキストにあること。

アルゴリズムまたは振る舞い

1. MyProc->syncRepLinks がキューから切り離されているかを確認する（ロック取得によるオーバーヘッドを避けるための最適化）。 2. 切り離されていない場合、SyncRepLock を排他モードで取得する。 3. 再度 dlist_node_is_detached を確認し、他プロセスによって既に除去されていないか検証する。 4. キューに存在する場合は dlist_delete_thoroughly を実行してリストから完全に削除する。 5. SyncRepLock を解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/lmgr/proc.c:931（プロセス終了処理の一部として呼び出される）
  • 呼び出し先: LWLockAcquire, LWLockRelease, dlist_node_is_detached, dlist_delete_thoroughly

関連データ構造

  • PGPROC: MyProc を通じてプロセス固有の待機状態を保持。
  • SyncRepLock: 同期レプリケーションの待機キューを保護する LWLock。
  • dlist_node: syncRepLinks として PGPROC 内に保持され、待機キューの双方向リストを構成する。

失敗モードと境界条件

  • 既にキューから除去されている場合、ロックを取得せずに終了するため、競合やデッドロックのリスクを回避している。
  • ロック取得後の再チェックにより、ロック待ちの間に別プロセスがキュー操作を行った場合でも安全に処理される。

保守担当者向けメモ

  • 本関数はプロセス終了時のクリーンアップ用であり、通常の待機解除フロー（SyncRepReleaseWaiters）とは独立して動作する。
  • dlist_delete_thoroughly を使用しているため、リストの整合性が破壊されることはない。
