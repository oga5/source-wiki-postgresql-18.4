spg_box_quad_leaf_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_spgist.c:740

目的

SP-GiSTインデックスのリーフノードにおいて、格納されたデータ（BOX）が検索条件（scankeys）を満たすかどうかを判定する。

シグネチャまたは形状

Datum spg_box_quad_leaf_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgLeafConsistentIn（検索条件、リーフデータ）および spgLeafConsistentOut（結果、再チェックフラグ）を受け取る。
  • **出力**: bool 型の判定結果（PG_RETURN_BOOL）。out->recheck は常に false に設定される。

前提条件と不変条件

  • すべてのテストは「完全（exact）」であると見なされ、out->recheck は false に固定される。
  • in->returnData が真の場合、out->leafValue にリーフデータが格納される。
  • 距離計算（orderbys）が存在する場合、多角形（F_DIST_POLYP）との距離計算時にのみ out->recheckDistances が有効化される。

アルゴリズムまたは振る舞い

1. out->recheck を false に初期化する。 2. in->nkeys 個の検索条件（scankeys）を順次評価する。 3. spg_box_quad_get_scankey_bbox を用いて各キーを BOX 型に変換する。 4. strategy（RTOverlapStrategyNumber 等）に基づき、geo_ops.c で定義された対応する幾何演算関数（box_overlap, box_contain 等）を呼び出す。 5. いずれかの条件で flag が false になれば即座にループを終了する。 6. 全条件を満たし、かつ in->norderbys > 0 の場合、spg_key_orderbys_distances を用いて距離を計算する。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiSTインデックスの走査プロセス（直接の呼び出し元情報はソースコード上では未特定）。
  • **呼び出し先**:
  • spg_box_quad_get_scankey_bbox
  • box_overlap, box_contain, box_contained, box_same, box_left, box_overleft, box_right, box_overright, box_above, box_overabove, box_below, box_overbelow
  • spg_key_orderbys_distances

関連データ構造

  • spgLeafConsistentIn, spgLeafConsistentOut: SP-GiSTのリーフ判定用インターフェース構造体。
  • BOX: 幾何学的な矩形データ型。

失敗モードと境界条件

  • 未定義の strategy が渡された場合、elog(ERROR) によりエラーを発生させる。
  • in->norderbys > 0 の場合、距離計算関数が F_DIST_POLYP であれば recheckDistances が有効になる。

保守担当者向けメモ

  • 本関数は box および polygon の両方の演算クラスで使用される。多角形の場合、リーフデータ型が BOX とは異なる可能性があるため、leafValue の返却には注意が必要である。
  • 検索条件の追加や変更を行う際は、switch 文の各 strategy ケースを適切に更新すること。
