XLogSetRecordFlags

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xloginsert.c:455

目的

現在構築中の WAL レコードに対して、追加のフラグ（XLOG_INCLUDE_ORIGIN や XLOG_MARK_UNIMPORTANT など）を設定します。

シグネチャまたは形状

void XLogSetRecordFlags(uint8 flags)

入力と出力

  • **入力**: flags (uint8) - 追加するフラグのビットマスク。
  • **出力**: なし。内部状態 curinsert_flags が更新されます。

前提条件と不変条件

  • XLogBeginInsert() が呼び出され、レコード構築セッションが開始されている必要があります（begininsert_called が真であること）。

アルゴリズムまたは振る舞い

1. begininsert_called のアサーションチェックを行い、構築セッションが有効か確認します。 2. 現在のレコード構築用フラグ curinsert_flags に対して、引数 flags をビット単位の OR 演算で追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistxlog.c
  • src/backend/access/heap/heapam.c
  • src/backend/access/transam/twophase.c
  • src/backend/access/transam/xact.c
  • src/backend/access/transam/xlog.c
  • **呼び出し先**:
  • なし（本関数は内部状態を更新するのみです）。

関連データ構造

  • curinsert_flags: 現在構築中のレコードのフラグを保持する内部変数。

失敗モードと境界条件

  • XLogBeginInsert() を呼び出す前に本関数を呼び出した場合、アサーション違反によりプロセスが異常終了します。

保守担当者向けメモ

  • 本関数は XLogInsert() が実行される前のレコード構築フェーズで呼び出される必要があります。
  • フラグは累積されるため、一度設定したフラグを本関数で取り消すことはできません。
