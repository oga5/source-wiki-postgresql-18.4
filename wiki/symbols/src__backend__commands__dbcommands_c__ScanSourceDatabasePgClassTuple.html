ScanSourceDatabasePgClassTuple

定義

  • 種別: function
  • 定義箇所: src/backend/commands/dbcommands.c:391

目的

データベース複製処理において、pg_class のタプルから対象リレーションの情報を抽出し、CreateDBRelInfo 構造体を構築する。

シグネチャまたは形状

CreateDBRelInfo *ScanSourceDatabasePgClassTuple(HeapTupleData *tuple, Oid tbid, Oid dbid, char *srcpath)

入力と出力

  • **入力**:
  • tuple: 処理対象の pg_class タプル。
  • tbid: デフォルトのテーブルスペース OID。
  • dbid: データベースの OID。
  • srcpath: ソースデータベースのパス。
  • **出力**:
  • 成功時: 構築された CreateDBRelInfo 構造体へのポインタ。
  • 失敗時（コピー不要なオブジェクトの場合）: NULL。

前提条件と不変条件

  • 入力タプルは pg_class の有効なタプルであること。
  • 一時リレーション（RELPERSISTENCE_TEMP）は除外される。
  • 共有オブジェクト（GLOBALTABLESPACE_OID）やストレージを持たないリレーションはコピー対象外として NULL を返す。

アルゴリズムまたは振る舞い

1. GETSTRUCT(tuple) を用いて Form_pg_class を取得する。 2. 共有オブジェクト、ストレージ非保持、または一時リレーションであれば NULL を返す。 3. relfilenode が有効であればそれを採用し、無効であれば RelationMapOidToFilenumberForDatabase を呼び出してマッピングを取得する。 4. relfilenumber が取得できない場合はエラー（elog(ERROR)）を発生させる。 5. CreateDBRelInfo 構造体をメモリ確保し、テーブルスペース OID、データベース OID、リレーション番号、OID を設定する。 6. リレーションが永続的（RELPERSISTENCE_PERMANENT）かどうかを判定し、構造体に格納して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/dbcommands.c 内のデータベーススキャン処理（ScanSourceDatabasePgClass 等）から呼び出される。
  • **呼び出し先**: RelationMapOidToFilenumberForDatabase（リレーションマップの解決）、palloc（メモリ確保）、elog（エラー報告）。

関連データ構造

  • CreateDBRelInfo: 複製対象リレーションのメタデータを保持する構造体。
  • Form_pg_class: pg_class カタログのタプル構造。
  • RelFileNumber: リレーションの物理ファイル識別子。

失敗モードと境界条件

  • RelFileNumber が解決できない場合、elog(ERROR) によりトランザクションがアボートされる。
  • 一時リレーションや共有オブジェクトは明示的にスキップされる。

保守担当者向けメモ

  • 本関数は CREATE DATABASE の複製プロセスにおいて、個々のリレーションをコピー対象とするか判断するフィルタリングと情報抽出を担う。
  • relfilenode の解決ロジックは、カタログと物理ファイルの対応関係に依存するため、リレーションマップの変更時には注意が必要である。
