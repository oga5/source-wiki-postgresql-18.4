FormatNode

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/formatting.c:153

目的

フォーマット文字列（TO_CHAR 等で使用されるテンプレート）を解析した結果を保持するノード構造体です。解析済みのノードをリスト化することで、変換処理時の再利用を可能にします。

シグネチャまたは形状

typedef struct {
    uint8       type;
    char        character[MAX_MULTIBYTE_CHAR_LEN + 1];
    uint8       suffix;
    const KeyWord *key;
} FormatNode;

入力と出力

  • **入力**: parse_format() 関数により、フォーマット文字列から各フィールドが設定されます。
  • **出力**: DCH_to_char()、DCH_from_char()、NUM_processor() 等の変換関数が、この構造体を参照してデータ変換を実行します。

前提条件と不変条件

  • type フィールドは NODE_TYPE_XXX に定義された値のいずれかをとります。
  • type が CHAR の場合、character フィールドに文字データが格納されます。
  • type が ACTION の場合、key フィールドが有効な KeyWord を指します。
  • suffix フィールドは、キーワードに対する接頭辞や接尾辞のコードを保持します。

アルゴリズムまたは振る舞い

1. **解析**: parse_format() がフォーマット文字列を走査し、トークンごとに FormatNode を生成してリストを作成します。 2. **キャッシュ**: 生成された FormatNode の配列は、DCHCacheEntry や NUMCacheEntry 内でキャッシュされ、同一フォーマット文字列の再解析を回避します。 3. **処理**: 変換関数はリストを順次走査し、type に応じて文字の出力やキーワードに基づくデータ変換を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: parse_format() がノードを生成します。また、NUMDesc_prepare、DCH_to_char、DCH_from_char、dump_node が引数として受け取ります。
  • **呼び出し先**: KeyWord 構造体を参照します。

関連データ構造

  • KeyWord: フォーマット指定子（YYYY, MM 等）の定義。
  • DCHCacheEntry / NUMCacheEntry: FormatNode の配列を保持するキャッシュ構造体。
  • NUMDesc: 数値フォーマットの属性を保持する構造体。

失敗モードと境界条件

  • **キャッシュ制限**: フォーマット文字列がキャッシュサイズ（DCH_CACHE_SIZE 等）を超える場合、キャッシュは使用されず、毎回解析が行われます。
  • **メモリ**: FormatNode の配列サイズは、キャッシュオーバーヘッドを考慮した計算式（src/backend/utils/adt/formatting.c:387-389）に基づき確保されます。

保守担当者向けメモ

  • 新しいフォーマット指定子を追加する場合は、KeyWord テーブルの更新と併せて、FormatNode の type や suffix の処理ロジックを各変換関数に追加する必要があります。
  • MAX_MULTIBYTE_CHAR_LEN に依存しているため、マルチバイト文字の取り扱いには注意してください。
