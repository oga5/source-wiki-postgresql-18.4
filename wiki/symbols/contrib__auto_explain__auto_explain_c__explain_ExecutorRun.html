explain_ExecutorRun

定義

  • 種別: function
  • 定義箇所: contrib/auto_explain/auto_explain.c:327

目的

クエリ実行のネストレベルを追跡し、実行エンジン（Executor）の実行フェーズをフックして標準的な実行処理を呼び出す。

シグネチャまたは形状

static void explain_ExecutorRun(QueryDesc *queryDesc, ScanDirection direction, uint64 count)

入力と出力

  • **入力**:
  • queryDesc: 実行対象のクエリ記述子。
  • direction: スキャン方向。
  • count: 処理するタプルの数。
  • **出力**: なし（void）。

前提条件と不変条件

  • nesting_level グローバル変数が、クエリのネスト深さを正確に反映している必要がある。
  • PG_TRY ブロックにより、実行中のエラー発生時にも必ず nesting_level がデクリメントされることが保証される。

アルゴリズムまたは振る舞い

1. nesting_level をインクリメントする。 2. PG_TRY ブロックを開始する。 3. prev_ExecutorRun が設定されていればそれを呼び出し、設定されていなければ standard_ExecutorRun を呼び出して標準の実行処理を行う。 4. PG_FINALLY ブロックにて、nesting_level をデクリメントする。 5. PG_END_TRY で処理を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • PostgreSQL の実行エンジン（ExecutorRun_hook を介して呼び出される）。
  • **呼び出し先**:
  • prev_ExecutorRun（以前に設定されたフック関数）または standard_ExecutorRun（PostgreSQL 標準の実行関数）。

関連データ構造

  • QueryDesc: クエリの実行状態を保持する構造体。
  • nesting_level: モジュール内で定義された、クエリのネスト深さを追跡する静的変数。

失敗モードと境界条件

  • PG_TRY ブロックにより、内部でエラーが発生した場合でも nesting_level の整合性が維持されるよう設計されている。
  • 呼び出し先関数がエラーをスローした場合、制御は PG_FINALLY を経由して上位へ伝播する。

保守担当者向けメモ

  • 本関数は ExecutorRun_hook に登録されることで機能する。
  • 他の拡張モジュールとの共存のため、必ず prev_ExecutorRun を経由して呼び出しチェーンを維持すること。
