brin_minmax_opcinfo

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_minmax.c:33

目的

BRIN（Block Range Index）の最小値・最大値演算クラスにおいて、インデックス構築や操作に必要なメタデータ構造体（BrinOpcInfo）を初期化し、提供します。

シグネチャまたは形状

Datum brin_minmax_opcinfo(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介して、対象となる属性の型OID（typoid）を受け取ります。
  • **出力**: BrinOpcInfo 構造体へのポインタを Datum として返します。

前提条件と不変条件

  • BrinOpcInfo 構造体には、最小値と最大値を保持するために oi_nstored = 2 が設定されます。
  • oi_opaque メンバには、MinmaxOpaque 構造体がメモリ配置されます。
  • oi_typcache には、指定された typoid に基づく型キャッシュが設定されます。

アルゴリズムまたは振る舞い

1. palloc0 を使用して、BrinOpcInfo 構造体と MinmaxOpaque 構造体を格納するのに十分なメモリを確保します。 2. oi_nstored を 2 に設定し、oi_regular_nulls を true に設定します。 3. oi_opaque ポインタを、確保したメモリ内の適切な位置（BrinOpcInfo の直後）にアライメントを考慮して配置します。 4. lookup_type_cache を呼び出し、指定された型のキャッシュ情報を取得して oi_typcache に格納します。 5. 初期化された BrinOpcInfo を返します。なお、MinmaxOpaque 内の戦略関数キャッシュは、この時点では遅延初期化のために未設定（fn_oid が InvalidOid）の状態です。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は明示されていませんが、BRIN演算クラスのインターフェースとして brin.c 等から間接的に呼び出される設計です。
  • **呼び出し先**: palloc0, lookup_type_cache を呼び出します。

関連データ構造

  • BrinOpcInfo: BRIN演算クラスのメタデータ保持用構造体。
  • MinmaxOpaque: 最小値・最大値演算クラス固有のキャッシュ構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • 無効な typoid が渡された場合、lookup_type_cache の挙動に依存します。

保守担当者向けメモ

  • MinmaxOpaque の戦略関数キャッシュは遅延初期化される設計であるため、本関数で fn_oid を初期化する必要はありません。
  • inclusion_get_strategy_procinfo と設計が共通化されているため、修正時は brin_inclusion.c の関連コードとの整合性に注意してください。
