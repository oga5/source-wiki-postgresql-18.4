sqlca_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/include/sqlca.h:19

目的

ECPG（Embedded SQL for C）アプリケーションにおいて、SQL文の実行結果、エラー情報、警告、および処理行数などの診断情報を保持するための標準的なデータ構造です。

シグネチャまたは形状

struct sqlca_t {
    char        sqlcaid[8];
    long        sqlabc;
    long        sqlcode;
    struct {
        int     sqlerrml;
        char    sqlerrmc[SQLERRMC_LEN];
    }           sqlerrm;
    char        sqlerrp[8];
    long        sqlerrd[6];
    char        sqlwarn[8];
    char        sqlstate[5];
};

入力と出力

  • **入力**: ECPGランタイムライブラリ（libecpg）がSQL実行後に各フィールドを更新します。
  • **出力**: アプリケーションコードが sqlcode（エラーコード）、sqlstate（SQL標準ステータス）、sqlerrd[2]（処理行数）などを読み取り、エラーハンドリングや制御フローの判定に使用します。

前提条件と不変条件

  • sqlwarn[0] は、他の警告フラグが設定されている場合に 'W' となります。
  • sqlerrd[2] は、INSERT、UPDATE、DELETE文の実行後に処理された行数を保持します。
  • sqlstate は5文字のSQL標準準拠のステータスコードを保持します。

アルゴリズムまたは振る舞い

ECPGプリプロセッサが生成するコードは、SQL文の実行直後にこの構造体を更新します。アプリケーションは、グローバル変数 sqlca を直接参照するか、ECPGget_sqlca() 関数を介してポインタを取得し、実行結果を検証します。

呼び出し元と呼び出し先

  • **呼び出し先**: ECPGget_sqlca() 関数により、現在のコンテキストにおける sqlca_t インスタンスへのポインタが取得されます。
  • **呼び出し元**: src/interfaces/ecpg/ecpglib/ 配下の各ファイル（connect.c, data.c, descriptor.c 等）および src/interfaces/ecpg/compatlib/informix.c が、SQL実行結果の書き込みや初期化のために本構造体を参照・操作しています。

関連データ構造

  • SQLERRMC_LEN: エラーメッセージバッファの最大長（150バイト）。

失敗モードと境界条件

  • sqlcode は古い形式の判定基準であり、詳細な診断には sqlstate の使用が推奨されます。
  • マルチスレッド環境では、直接的なグローバル変数アクセスではなく、必ず ECPGget_sqlca() マクロ経由でアクセスする必要があります。

保守担当者向けメモ

  • 本構造体はSQL標準のSQLCA定義に準拠しており、移植性を維持するため構造の変更には注意が必要です。
  • sqlerrd 配列の各要素には特定の用途（OIDや処理行数など）が割り当てられており、インデックスの誤用を避けてください。
