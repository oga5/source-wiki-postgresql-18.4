tsquery_phrase_distance

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_op.c:113

目的

2つの tsquery を指定された距離（単語間の許容間隔）で結合し、フレーズ検索用の新しい tsquery を生成します。

シグネチャまたは形状

Datum tsquery_phrase_distance(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TSQUERY_COPY(0): 第1引数 (tsquery 型)
  • PG_GETARG_TSQUERY_COPY(1): 第2引数 (tsquery 型)
  • PG_GETARG_INT32(2): 距離 (int32 型)
  • **出力**:
  • Datum: 結合された新しい tsquery 型のポインタ

前提条件と不変条件

  • 距離引数は 0 以上 MAXENTRYPOS 以下である必要があります。これに違反するとエラーが発生します。
  • 入力された tsquery が空（size == 0）の場合、もう一方の tsquery をそのまま返します。

アルゴリズムまたは振る舞い

1. 引数から tsquery と距離を取得します。 2. 距離が範囲外であれば ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告します。 3. いずれかの tsquery が空であれば、空でない方を返却し、メモリを適切に解放します。 4. join_tsqueries を呼び出し、OP_PHRASE 演算子と指定距離を用いて QTNode ツリーを構築します。 5. QTN2QT を使用してツリーを TSQuery 形式にシリアライズします。 6. 一時的なツリー構造を QTNFree で解放し、入力引数のメモリを PG_FREE_IF_COPY で管理した上で、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery_op.c:152 (DirectFunctionCall3経由の呼び出し)
  • **呼び出し先**:
  • join_tsqueries (ツリー構築)
  • QTN2QT (シリアライズ)
  • QTNFree (メモリ解放)
  • ereport (エラー報告)

関連データ構造

  • TSQuery: PostgreSQL の全文検索クエリ型。
  • QTNode: tsquery を操作するための内部ツリー構造。

失敗モードと境界条件

  • **距離の不正**: distance < 0 または distance > MAXENTRYPOS の場合、エラーとなります。
  • **メモリ管理**: PG_GETARG_TSQUERY_COPY を使用しているため、不要になった入力引数は PG_FREE_IF_COPY で解放する必要があります。

保守担当者向けメモ

  • 本関数は tsquery を一度 QTNode ツリーに展開してから再構築するため、複雑なクエリではオーバーヘッドが発生します。
  • QTN_NEEDFREE フラグを持つノードの解放漏れがないよう、QTNFree の呼び出し位置には注意してください。
