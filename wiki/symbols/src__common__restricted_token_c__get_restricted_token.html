get_restricted_token

定義

  • 種別: function
  • 定義箇所: src/common/restricted_token.c:128

目的

Windows環境において、プロセスが制限付きトークン（Restricted Token）で実行されていることを保証します。制限付きでない場合は、自身を再実行して権限を降格させます。

シグネチャまたは形状

void get_restricted_token(void)

入力と出力

  • 入力: 環境変数 PG_RESTRICT_EXEC および現在のプロセスコマンドライン引数。
  • 出力: なし（成功時は子プロセスとして再実行され、親プロセスは終了する）。

前提条件と不変条件

  • Windows環境（WIN32 マクロ定義時）でのみ機能します。
  • フロントエンドプロセスでの使用を想定しています。

アルゴリズムまたは振る舞い

1. 環境変数 PG_RESTRICT_EXEC を確認し、既に "1" であれば何もしません。 2. 未設定の場合、現在のコマンドライン引数を取得し、環境変数を "1" に設定します。 3. CreateRestrictedProcess を呼び出し、制限付きトークンを用いて自身を再起動します。 4. 再起動に成功した場合、親プロセスは WaitForSingleObject で子プロセスの終了を待ち、子プロセスの終了コードを引き継いで終了します。 5. 失敗した場合はエラーログを出力します。

呼び出し元と呼び出し先

  • 呼び出し元（直接の証拠）:
  • src/bin/initdb/initdb.c
  • src/bin/pg_basebackup/pg_createsubscriber.c
  • src/bin/pg_resetwal/pg_resetwal.c
  • src/bin/pg_rewind/pg_rewind.c
  • src/bin/pg_upgrade/pg_upgrade.c
  • src/test/regress/pg_regress.c
  • 呼び出し先:
  • CreateRestrictedProcess (同ファイル内)
  • GetCommandLine, setenv, pg_strdup, pg_log_error, CloseHandle, WaitForSingleObject, GetExitCodeProcess, exit (Windows APIおよびPostgreSQLユーティリティ)

関連データ構造

  • PROCESS_INFORMATION (Windows API構造体)

失敗モードと境界条件

  • CreateRestrictedProcess が失敗した場合、エラーコードと共にログ出力されますが、プロセスは終了しません。
  • GetExitCodeProcess が失敗した場合、pg_fatal によりプロセスが強制終了されます。
  • 非Windows環境では空の関数として動作します。

保守担当者向けメモ

  • 本関数はプロセスを再帰的に再起動するため、デバッグ時にプロセスツリーが深くなる点に注意してください。
  • FRONTEND マクロが定義されている環境でのみコンパイルされるため、バックエンドコードには含めないでください。
