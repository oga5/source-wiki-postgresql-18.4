pgwin32_signal_initialize

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/signal.c:78

目的

Windows環境におけるPostgreSQLのシグナル処理サブシステムを初期化し、シグナル受信用のバックグラウンドスレッドおよびコンソール制御ハンドラをセットアップする。

シグネチャまたは形状

void pgwin32_signal_initialize(void)

入力と出力

  • 入力: なし
  • 出力: なし（成功時）
  • 失敗時: ereport(FATAL, ...) によりプロセスを終了させる。

前提条件と不変条件

  • 本関数は、シグナル処理に関連するグローバル変数（pg_signal_array, pg_signal_mask, pg_signal_queue 等）を初期化する。
  • pg_signal_crit_sec（クリティカルセクション）が初期化され、スレッド間の排他制御が可能な状態になる。

アルゴリズムまたは振る舞い

1. pg_signal_crit_sec を初期化する。 2. シグナル配列（pg_signal_array）をデフォルト値（SIG_DFL）でリセットし、シグナルマスクとキューをクリアする。 3. シグナル通知用のWindowsイベントオブジェクト（pgwin32_signal_event）を作成する。 4. シグナル待機用のバックグラウンドスレッド（pg_signal_thread）を起動する。 5. コンソール制御ハンドラ（pg_console_handler）を登録し、Ctrl-C等のイベントを捕捉可能にする。 6. 上記の各リソース作成に失敗した場合は、FATALエラーとして報告する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/postmaster/postmaster.c:512
  • src/backend/utils/init/miscinit.c:106
  • src/backend/utils/init/miscinit.c:185
  • 呼び出し先:
  • InitializeCriticalSection
  • CreateEvent
  • CreateThread（pg_signal_thread を実行）
  • SetConsoleCtrlHandler（pg_console_handler を登録）
  • ereport

関連データ構造

  • pg_signal_array: シグナルハンドラ、マスク、フラグを保持する配列。
  • pg_signal_crit_sec: シグナル関連データへのアクセスを保護するクリティカルセクション。
  • pgwin32_signal_event: メインスレッドにシグナル到着を通知するためのイベントハンドル。

失敗モードと境界条件

  • CreateEvent が失敗した場合、システムリソース不足として FATAL エラーとなる。
  • CreateThread が失敗した場合、スレッド生成不可として FATAL エラーとなる。
  • SetConsoleCtrlHandler が失敗した場合、コンソール制御不可として FATAL エラーとなる。

保守担当者向けメモ

  • 本関数はPostgreSQLの起動プロセスにおいて早期に呼び出される必要がある。
  • pg_signal_thread が正常に動作するためには、本関数で作成されるイベントオブジェクトとクリティカルセクションが正しく初期化されていることが不可欠である。
