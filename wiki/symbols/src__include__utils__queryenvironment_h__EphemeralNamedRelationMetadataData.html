EphemeralNamedRelationMetadataData

定義

  • 種別: struct
  • 定義箇所: src/include/utils/queryenvironment.h:32

目的

クエリ実行中にのみ存在する一時的な名前付きリレーション（ENR）のメタデータを保持し、プランナやエグゼキュータがカタログ外のデータ構造（トリガーの遷移テーブルなど）を解決できるようにする。

シグネチャまたは形状

typedef struct EphemeralNamedRelationMetadataData
{
    char       *name;           /* リレーション識別名 */
    Oid         reliddesc;      /* TupleDesc取得用OID */
    TupleDesc   tupdesc;        /* 行構造の記述 */
    EphemeralNameRelationType enrtype; /* リレーション種別 */
    double      enrtuples;      /* 推定タプル数 */
} EphemeralNamedRelationMetadataData;

入力と出力

  • **入力**: 登録時に name、reliddesc または tupdesc、enrtype、enrtuples が設定される。
  • **出力**: ENRMetadataGetTupDesc() を介して、tupdesc フィールドまたは reliddesc から導出された TupleDesc が提供される。

前提条件と不変条件

  • reliddesc と tupdesc のいずれか一方のみを使用すること。
  • QueryEnvironment 内で管理され、クエリ実行ライフサイクルに紐付く。

アルゴリズムまたは振る舞い

1. **メタデータ保持**: リレーション名、型、推定サイズ、およびスキーマ情報（TupleDesc または Oid）を構造体として保持する。 2. **スキーマ解決**: ENRMetadataGetTupDesc() が呼び出された際、tupdesc が設定されていればそれを返し、未設定であれば reliddesc を用いてカタログから解決を試みる。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定の呼び出し元は特定できない。
  • **呼び出し先**: access/tupdesc.h で定義された TupleDesc 型に依存する。

関連データ構造

  • EphemeralNamedRelationMetadata: 本構造体へのポインタ型。
  • EphemeralNamedRelationData: 本メタデータと実行時のデータを保持する構造体。
  • QueryEnvironment: 本構造体を管理するコンテナ。
  • EphemeralNameRelationType: リレーションの種別を定義する列挙型。

失敗モードと境界条件

  • reliddesc と tupdesc の両方が設定されていない、あるいは両方が設定されている場合の動作は定義されていない。
  • reliddesc を使用する場合、カタログ参照が失敗するとスキーマ取得が失敗する可能性がある。

保守担当者向けメモ

  • 本構造体は queryenvironment.c で管理されるリスト構造の一部として利用される。
  • 将来的に登録数が増加した場合は、リスト検索からハッシュテーブル検索への変更が検討される可能性があるため、直接のメンバアクセスは避け、提供されている API を介した操作を推奨する。
