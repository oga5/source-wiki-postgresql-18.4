PQconnectStart

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-connect.c:947

目的

PostgreSQL データベースへの非同期接続プロセスを開始し、接続状態を管理する PGconn 構造体を初期化・設定する。

シグネチャまたは形状

PGconn *PQconnectStart(const char *conninfo);

入力と出力

  • **入力**: conninfo (接続パラメータを含む文字列、または NULL)
  • **出力**: 初期化された PGconn 構造体へのポインタ。メモリ確保に失敗した場合は NULL を返す。

前提条件と不変条件

  • conn->errorMessage は空の状態で初期化されることが期待される。
  • 接続プロセスの各ステップで発生するエラーメッセージは、conn->errorMessage に追記される。

アルゴリズムまたは振る舞い

1. pqMakeEmptyPGconn() を呼び出し、PGconn 構造体を確保・初期化する。 2. connectOptions1() を呼び出し、conninfo 文字列を解析してパラメータを構造体に格納する。 3. pqConnectOptions2() を呼び出し、ホストリストの展開やポート決定などの派生オプションを計算する。 4. pqConnectDBStart() を呼び出し、実際の接続試行を開始する。 5. pqConnectDBStart() が失敗した場合は、接続状態を CONNECTION_BAD に設定する。 6. 設定済みの PGconn ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/task.c
  • src/include/libpq/libpq-be-fe-helpers.h
  • src/interfaces/libpq/fe-connect.c (内部的な再帰的またはテスト用途の呼び出し)
  • src/test/modules/oauth_validator/oauth_hook_client.c
  • **呼び出し先**:
  • pqMakeEmptyPGconn
  • connectOptions1
  • pqConnectOptions2
  • pqConnectDBStart

関連データ構造

  • PGconn: 接続状態、エラーメッセージ、パラメータを保持する主要な構造体。

失敗モードと境界条件

  • pqMakeEmptyPGconn がメモリ不足等で NULL を返した場合、関数は直ちに NULL を返す。
  • connectOptions1 または pqConnectOptions2 が失敗した場合、その時点までの設定状態を保持したまま PGconn を返す。
  • pqConnectDBStart が失敗した場合、conn->status が CONNECTION_BAD に設定される。

保守担当者向けメモ

  • 本関数は非同期接続の開始点であり、接続完了には PQconnectPoll を使用したステートマシンの駆動が必要である。
  • 接続失敗時のエラーハンドリングは conn->errorMessage に集約されるため、デバッグ時はこのバッファを確認すること。
