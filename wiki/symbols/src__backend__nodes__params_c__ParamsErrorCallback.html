ParamsErrorCallback

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/params.c:406

目的

エラー発生時のコンテキスト情報として、クエリ実行に関連付けられたパラメータの値をログ出力するために使用されるコールバック関数です。

シグネチャまたは形状

void ParamsErrorCallback(void *arg)
  • arg: ParamsErrorCbData 構造体へのポインタ。

入力と出力

  • **入力**: ParamsErrorCbData 型の引数。これには portalName（ポータル名）および params（ParamListInfo 構造体）が含まれます。
  • **出力**: なし（戻り値は void）。副作用として errcontext を通じてエラーメッセージを構築します。

前提条件と不変条件

  • 引数 arg が指す ParamsErrorCbData 構造体において、params および params->paramValuesStr が有効である必要があります。
  • paramValuesStr は、ログ出力用に事前に生成された文字列である必要があります。

アルゴリズムまたは振る舞い

1. 引数 arg が NULL、または params や paramValuesStr が NULL の場合は何もせず終了します。 2. portalName が存在し、空文字列でない場合は、ポータル名を含めたエラーコンテキストを記録します。 3. portalName が存在しない（または空の）場合は、「unnamed portal」としてエラーコンテキストを記録します。 4. いずれの場合も、params->paramValuesStr に格納されている文字列をエラーメッセージに付与します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/postgres.c:1998
  • src/backend/tcop/postgres.c:2266

    • （params_errcxt.callback に設定される形で呼び出されます）

  • **呼び出し先**:
  • errcontext（PostgreSQL のエラー報告用マクロ）

関連データ構造

  • ParamsErrorCbData: 本関数の引数として渡される構造体。
  • ParamListInfo: パラメータ情報を保持する構造体。paramValuesStr フィールドを介して本関数と連携します。

失敗モードと境界条件

  • data が NULL の場合、あるいは params や paramValuesStr が初期化されていない（NULL である）場合、本関数は安全に早期リターンし、エラー報告を行いません。

保守担当者向けメモ

  • 本関数は errcontext を使用するため、エラー発生時にのみ実行されます。
  • paramValuesStr はシリアライズ対象外のキャッシュであるため、再構築時には自動的に復元されない点に注意してください。
