AnonymousShmemDetach

定義

  • 種別: function
  • 定義箇所: src/backend/port/sysv_shmem.c:674

目的

プロセス終了時に、mmap によって確保された匿名共有メモリブロックを解放（デタッチ）するために使用されます。

シグネチャまたは形状

static void AnonymousShmemDetach(int status, Datum arg);

入力と出力

  • **入力**:
  • status: プロセス終了時のステータスコード（on_shmem_exit コールバック規約による）。
  • arg: ユーザー定義データ（本関数では使用されません）。
  • **出力**: なし（void）。

前提条件と不変条件

  • グローバル変数 AnonymousShmem が指すメモリ領域が、mmap によって確保されている必要があります。
  • AnonymousShmemSize に、確保されたメモリブロックの正しいサイズが格納されている必要があります。

アルゴリズムまたは振る舞い

1. AnonymousShmem ポインタが NULL でないかを確認します。 2. munmap(AnonymousShmem, AnonymousShmemSize) を呼び出し、メモリ領域を解放します。 3. munmap が失敗した場合は elog(LOG, ...) を通じてエラーログを出力します。 4. 解放後、AnonymousShmem を NULL にリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**: on_shmem_exit に登録されたコールバックとして、プロセス終了時に呼び出されます（src/backend/port/sysv_shmem.c:743）。
  • **呼び出し先**: munmap（システムコール）、elog（PostgreSQL エラーログ出力）。

関連データ構造

  • AnonymousShmem: 匿名共有メモリの開始アドレスを保持するグローバル変数。
  • AnonymousShmemSize: 匿名共有メモリのサイズを保持するグローバル変数。

失敗モードと境界条件

  • munmap が失敗した場合、エラーログが出力されますが、プロセス終了処理自体は継続されます。
  • AnonymousShmem が既に NULL の場合、本関数は何もしません。

保守担当者向けメモ

  • 本関数は on_shmem_exit を介して実行されるため、シグナルハンドラや終了処理の文脈で安全に動作するように設計されています。
  • 匿名共有メモリの管理は、System V 共有メモリとは独立した mmap ベースの領域に対して行われる点に注意してください。
