SubOpts

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/subscriptioncmds.c:84

目的

論理レプリケーションの CREATE SUBSCRIPTION および ALTER SUBSCRIPTION コマンドで指定されたオプションを保持し、解析結果をコマンド処理関数へ伝達するためのデータ構造です。

シグネチャまたは形状

typedef struct SubOpts
{
	bits32		specified_opts;
	char	   *slot_name;
	char	   *synchronous_commit;
	bool		connect;
	bool		enabled;
	bool		create_slot;
	bool		copy_data;
	bool		refresh;
	bool		binary;
	char		streaming;
	bool		twophase;
	bool		disableonerr;
	bool		passwordrequired;
	bool		runasowner;
	bool		failover;
	char	   *origin;
	XLogRecPtr	lsn;
} SubOpts;

入力と出力

  • **入力**: SQL コマンドの DefElem リストから parse_subscription_options 関数によって値が抽出・設定されます。
  • **出力**: CreateSubscription や AlterSubscription などのコマンド実行関数が、この構造体を参照してシステムカタログの更新やリモートノードとの通信パラメータを決定します。

前提条件と不変条件

  • 使用前に memset(opts, 0, sizeof(SubOpts)) 等で初期化されることが一般的です。
  • specified_opts ビットフィールドにより、どのオプションが明示的に指定されたかが追跡されます。

アルゴリズムまたは振る舞い

  • コマンド解析フェーズにおいて、ユーザーが指定したオプション（slot_name, copy_data, two_phase 等）を構造体の各フィールドにマッピングします。
  • 相互排他チェックやデフォルト値の適用を経て、最終的な実行パラメータとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/subscriptioncmds.c 内の CreateSubscription および AlterSubscription 関数が、この構造体をローカル変数として宣言し、解析関数へ渡しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • DefElem: SQL 解析時のオプション定義。
  • pg_subscription: 本構造体の値が反映されるシステムカタログ。

失敗モードと境界条件

  • オプションの解析時に不正な値や相互排他違反が検出された場合、parse_subscription_options 経由でエラーが発生します。
  • 構造体内のポインタ型フィールド（slot_name 等）は、メモリ管理の観点から適切なライフサイクルで割り当てられる必要があります。

保守担当者向けメモ

  • 新しいサブスクリプションオプションを追加する場合は、本構造体へのフィールド追加と、parse_subscription_options での解析ロジックの更新が必要です。
  • specified_opts のビット定義と整合性を保つよう注意してください。
