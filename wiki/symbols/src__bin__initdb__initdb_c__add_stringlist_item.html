add_stringlist_item

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/initdb.c:444

目的

文字列リスト（_stringlist 型の連結リスト）の末尾に新しい要素を追加します。

シグネチャまたは形状

void add_stringlist_item(_stringlist **listhead, const char *str)

入力と出力

  • **入力**:
  • listhead: リストの先頭ポインタへのポインタ。
  • str: リストに追加する文字列。
  • **出力**: なし（listhead が指すリストが更新されます）。

前提条件と不変条件

  • listhead は有効なポインタである必要があります。
  • str は NULL であってはなりません。
  • リストの各要素は pg_malloc で確保され、文字列は pg_strdup で複製されます。

アルゴリズムまたは振る舞い

1. pg_malloc を使用して新しい _stringlist エントリを確保します。 2. pg_strdup を使用して入力文字列 str を新しいエントリにコピーします。 3. 新しいエントリの next ポインタを NULL に設定します。 4. listhead が指すリストが空（NULL）の場合、新しいエントリを先頭に設定します。 5. リストが空でない場合、リストの末尾まで走査し、最後の要素の next ポインタを新しいエントリに設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/initdb/initdb.c: extra_guc_names および extra_guc_values の構築時に呼び出されます。
  • src/interfaces/ecpg/test/pg_regress_ecpg.c: テスト結果ファイルやタグの管理に使用されています。
  • **呼び出し先**:
  • pg_malloc
  • pg_strdup

関連データ構造

  • _stringlist: 文字列を保持する連結リスト構造体。

失敗モードと境界条件

  • メモリ不足が発生した場合、pg_malloc および pg_strdup の内部実装によりプログラムが終了する可能性があります。
  • 空のリストに対して呼び出した場合、正しく先頭要素として追加されます。

保守担当者向けメモ

  • 本関数はリストの末尾を探索するために $O(N)$ の計算量を要します。リストが非常に長くなる場合はパフォーマンスに影響する可能性があります。
  • _stringlist 構造体の定義は initdb.c 内に存在します。
