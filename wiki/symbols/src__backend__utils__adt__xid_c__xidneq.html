xidneq

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xid.c:92

目的

2つのトランザクション識別子（TransactionId）が等しくないかどうかを判定し、SQLレベルでの不等価演算子（<>）のバックエンド実装として機能します。

シグネチャまたは形状

Datum xidneq(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの TransactionId を受け取ります。
  • **出力**: 2つのIDが等しくない場合は true、等しい場合は false を bool 型で返します。

前提条件と不変条件

  • 入力される引数は PG_GETARG_TRANSACTIONID マクロで正しく取得可能である必要があります。
  • 比較ロジックは TransactionIdEquals に依存しており、トランザクションIDの等価性判定ルールに従います。

アルゴリズムまたは振る舞い

1. PG_GETARG_TRANSACTIONID(0) および (1) を使用して、2つのトランザクションIDを抽出します。 2. TransactionIdEquals(xid1, xid2) を呼び出し、両者が等しいかを確認します。 3. その結果を反転させ、PG_RETURN_BOOL を介して不等価の結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（不等価演算子 <> の実行時）。
  • **呼び出し先**: TransactionIdEquals（access/transam.h で定義されたプリミティブ）。

関連データ構造

  • TransactionId: PostgreSQLのトランザクションを識別するための32bit整数型。

失敗モードと境界条件

  • 特殊なトランザクションID（InvalidTransactionId, BootstrapTransactionId 等）が渡された場合でも、TransactionIdEquals の定義に基づいた比較が行われます。

保守担当者向けメモ

  • 本関数は単なる等価比較の否定です。トランザクションIDの論理的な前後関係（TransactionIdPrecedes 等）を判定するものではないため、順序比較が必要な場合は別の関数を使用してください。
