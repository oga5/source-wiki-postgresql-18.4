hemdist

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_intbig_gist.c:232

目的

2つの GISTTYPE オブジェクト間のハミング距離を計算し、GiST インデックスにおけるノードの類似度や挿入コストを評価します。

シグネチャまたは形状

int hemdist(GISTTYPE *a, GISTTYPE *b, int siglen)

入力と出力

  • **入力**:
  • a, b: 比較対象の GISTTYPE ポインタ。
  • siglen: ビットベクトルの長さ。
  • **出力**:
  • 2つのオブジェクト間のハミング距離を表す整数値。

前提条件と不変条件

  • a および b は有効な GISTTYPE 構造体である必要があります。
  • siglen はビットベクトルの長さを正しく反映している必要があります。

アルゴリズムまたは振る舞い

1. a または b が ISALLTRUE（全ビットがセットされた状態）である場合、特殊な計算を行います。

  • 両方が ISALLTRUE ならば距離は 0 を返します。
  • 片方のみが ISALLTRUE ならば、SIGLENBIT(siglen) からもう一方のビットベクトルのセットされているビット数（sizebitvec）を引いた値を返します。

    • 2. 両方が通常の状態である場合、hemdistsign を呼び出し、それぞれのビットベクトル（GETSIGN）間のハミング距離を計算して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_intbig_gist.c 内の g_intbig_penalty、g_intbig_picksplit 等。
  • contrib/hstore/hstore_gist.c 内の各 GiST 操作関数。
  • **呼び出し先**:
  • sizebitvec
  • hemdistsign
  • GETSIGN（マクロ）
  • ISALLTRUE（マクロ）
  • SIGLENBIT（マクロ）

関連データ構造

  • GISTTYPE: GiST インデックスで扱われるデータ型。
  • BITVECP: ビットベクトルを指すポインタ型。

失敗モードと境界条件

  • siglen が不適切な場合、ビット演算の結果が不正になる可能性があります。
  • ISALLTRUE 判定により、全ビットが立っている特殊なノードに対する距離計算が最適化されています。

保守担当者向けメモ

  • 本関数は intarray モジュールだけでなく hstore モジュールからも参照されています。変更時は両方の GiST インデックス実装への影響を考慮してください。
  • ISALLTRUE 状態の扱いは、インデックスの圧縮効率に直結するため、ロジックの変更には注意が必要です。
