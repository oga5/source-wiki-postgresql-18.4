parse_filename_for_nontemp_relation

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/reinit.c:379

目的

ファイル名がPostgreSQLのリレーションデータファイルであるかを判定し、そのメタデータ（リレーション番号、フォーク種別、セグメント番号）を抽出するユーティリティ関数です。

シグネチャまたは形状

bool parse_filename_for_nontemp_relation(const char *name, RelFileNumber *relnumber, ForkNumber *fork, unsigned *segno);

入力と出力

  • **入力**:
  • name: 解析対象のファイル名文字列。
  • **出力**:
  • relnumber: 抽出されたリレーション番号（成功時）。
  • fork: 抽出されたフォーク種別（成功時）。
  • segno: 抽出されたセグメント番号（成功時）。
  • **戻り値**:
  • true: 有効なリレーションファイルとして解析できた場合。
  • false: ファイル名がリレーションの命名規則に従っていない場合。

前提条件と不変条件

  • ファイル名の先頭は '1' から '9' の数字である必要があります（先頭が '0' の場合はリレーションファイルではないとみなされます）。
  • リレーション番号およびセグメント番号は PG_UINT32_MAX を超えてはなりません。
  • 成功時には、出力パラメータに有効な値が設定されます。

アルゴリズムまたは振る舞い

1. **リレーション番号の解析**: 先頭の数字文字列を strtoul で解析します。 2. **フォーク種別の判定**: リレーション番号の直後に _ が続く場合、forkname_chars を用いてフォーク名を特定します。_ がない場合は MAIN_FORKNUM とみなします。 3. **セグメント番号の判定**: フォーク名の後に . が続く場合、続く数字をセグメント番号として解析します。先頭が '0' のセグメント番号は拒否されます。 4. **終端チェック**: 解析終了時に文字列が終端（\0）に達していることを確認します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/file/reinit.c 内の各関数（ResetUnloggedRelations 等）
  • src/backend/backup/basebackup.c
  • src/bin/pg_upgrade/relfilenumber.c
  • **呼び出し先**:
  • forkname_chars（フォーク名の特定に使用）
  • strtoul（数値変換に使用）

関連データ構造

  • RelFileNumber
  • ForkNumber

失敗モードと境界条件

  • ファイル名が数字で始まらない場合、または先頭が '0' の場合は false を返します。
  • 数値が範囲外（PG_UINT32_MAX 超過）の場合や、解析中にエラーが発生した場合は false を返します。
  • ファイル名の形式が期待されるパターン（relnumber、_fork、.segno）に一致しない場合、または余分な文字が含まれる場合は false を返します。

保守担当者向けメモ

  • 本関数は厳格な命名規則に基づいています。新しいフォーク種別を追加する際は forkname_chars の更新が必要になる可能性があります。
  • 一時ファイル（t で始まるもの等）は本関数の対象外です。
