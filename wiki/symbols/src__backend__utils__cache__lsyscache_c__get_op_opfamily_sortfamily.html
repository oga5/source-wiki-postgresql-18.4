get_op_opfamily_sortfamily

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/lsyscache.c:108

目的

指定された演算子（opno）が特定の演算子ファミリー（opfamily）内で順序付け演算子（AMOP_ORDER）として定義されている場合、その演算子に関連付けられたソート用演算子ファミリーのOIDを取得します。

シグネチャまたは形状

Oid get_op_opfamily_sortfamily(Oid opno, Oid opfamily)

入力と出力

  • **入力**:
  • opno: 検索対象の演算子OID。
  • opfamily: 検索対象の演算子ファミリーOID。
  • **出力**:
  • 成功時: 対応するソート用演算子ファミリーのOID。
  • 失敗時: InvalidOid（該当するエントリが見つからない場合）。

前提条件と不変条件

  • AMOPOPID システムキャッシュが利用可能であること。
  • 戻り値のOIDは、システムカタログ pg_amop の amopsortfamily 列から取得されます。

アルゴリズムまたは振る舞い

1. SearchSysCache3 を使用し、AMOPOPID キャッシュに対して opno、AMOP_ORDER（戦略番号）、および opfamily をキーとして検索を行います。 2. キャッシュヒットしない（HeapTupleIsValid が偽）場合は InvalidOid を返します。 3. キャッシュヒットした場合、GETSTRUCT マクロで Form_pg_amop 構造体にキャストし、amopsortfamily フィールドの値を抽出します。 4. ReleaseSysCache を呼び出して取得したタプルの参照カウントを解放します。 5. 抽出したOIDを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/indxpath.c 内で、インデックスパスの生成時に呼び出されています。
  • **呼び出し先**:
  • SearchSysCache3（システムキャッシュ検索）
  • ReleaseSysCache（キャッシュ解放）
  • GETSTRUCT（タプル構造体への変換）

関連データ構造

  • Form_pg_amop: pg_amop システムカタログの行構造体。
  • HeapTuple: システムキャッシュから返されるタプル。

失敗モードと境界条件

  • 指定された opno と opfamily の組み合わせが pg_amop に存在しない場合、InvalidOid を返します。
  • キャッシュ検索失敗時のエラーハンドリングは行われず、単に InvalidOid が返されます。

保守担当者向けメモ

  • 本関数は pg_amop カタログの amopstrategy が AMOP_ORDER であるエントリのみを対象としています。
  • 戻り値が InvalidOid であることは、必ずしもエラーではなく、該当するソートファミリーが存在しない（または定義されていない）状態を指す場合があります。
