btree_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/nbtdesc.c:138

目的

B-treeインデックスのWALレコードに含まれる info フィールドを解析し、その操作内容を人間が判読可能な文字列として識別する。

シグネチャまたは形状

const char *btree_identify(uint8 info)

入力と出力

  • **入力**: uint8 info （WALレコードの info ビットフィールド）
  • **出力**: 操作の種類を示す文字列定数（例: "INSERT_LEAF", "VACUUM" 等）。該当する操作がない場合は NULL を返す。

前提条件と不変条件

  • 入力された info から XLR_INFO_MASK を除外した値を用いて判定を行う。
  • 戻り値は静的な文字列リテラルであり、呼び出し側で解放してはならない。

アルゴリズムまたは振る舞い

1. info ビットマスクから XLR_INFO_MASK を除去し、純粋な操作タイプを抽出する。 2. switch 文を用いて、抽出された値に対応するB-tree操作（INSERT_LEAF, SPLIT_L, VACUUM, DELETE 等）を特定する。 3. 特定された操作に対応する文字列を id に代入する。 4. 最終的に特定された文字列、または一致しない場合は初期値の NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h で定義されているリソースマネージャ（RM_BTREE_ID）のメソッドとして、WAL解析ツール（pg_waldump 等）から間接的に呼び出される。
  • **呼び出し先**: 本関数内での関数呼び出しは存在しない。

関連データ構造

  • access/nbtxlog.h: B-treeのWALレコードタイプ定義（XLOG_BTREE_INSERT_LEAF 等）が格納されている。

失敗モードと境界条件

  • 定義されていない info 値が渡された場合、id は NULL のまま返される。

保守担当者向けメモ

  • 新しいB-tree WALレコードタイプを追加した際は、本関数の switch 文にも対応するケースを追加すること。
  • XLR_INFO_MASK を除外する処理は、他の rmgrdesc 実装と共通の慣習であるため、変更時は注意が必要。
