pg_console_handler

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/signal.c:376

目的

Windowsコンソールイベント（Ctrl+C、終了要求など）を捕捉し、PostgreSQL内部のシグナル処理系へ SIGINT として伝達する。

シグネチャまたは形状

static BOOL WINAPI pg_console_handler(DWORD dwCtrlType)

入力と出力

  • **入力**: dwCtrlType (Windowsのコンソール制御イベントの種類を示す DWORD)
  • **出力**: BOOL (イベントを処理した場合は TRUE、それ以外は FALSE)

前提条件と不変条件

  • SetConsoleCtrlHandler を介してWindows OSに登録されている必要がある。
  • 内部で呼び出される pg_queue_signal がスレッドセーフであること。

アルゴリズムまたは振る舞い

1. 引数 dwCtrlType が以下のいずれかであるか判定する:

  • CTRL_C_EVENT
  • CTRL_BREAK_EVENT
  • CTRL_CLOSE_EVENT
  • CTRL_SHUTDOWN_EVENT

    • 2. 上記のいずれかに該当する場合、pg_queue_signal(SIGINT) を呼び出し、シグナルキューに SIGINT を追加する。 3. イベントを処理したことを示すため TRUE を返す。 4. 上記以外のイベントの場合は FALSE を返し、OS側のデフォルト動作に委ねる。

呼び出し元と呼び出し先

  • **呼び出し元**: Windows OSのコンソールイベントハンドラ（SetConsoleCtrlHandler による登録経由）。
  • **呼び出し先**: pg_queue_signal

関連データ構造

  • SIGINT (シグナル定数)

失敗モードと境界条件

  • dwCtrlType が対象外のイベントである場合、何もせず FALSE を返す。
  • pg_queue_signal が失敗した場合の戻り値は本関数内では考慮されていない。

保守担当者向けメモ

  • 本関数はWindows APIの HandlerRoutine コールバックとして動作するため、実行コンテキストはメインスレッドとは異なるスレッドで実行される可能性がある。
  • pg_queue_signal 内部での排他制御（pg_signal_crit_sec）が正しく機能していることを前提としている。
