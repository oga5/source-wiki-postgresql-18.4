hstore_subscript_transform

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_subs.c:41

目的

SQLの添字構文（hstore_val['key']）を解析し、SubscriptingRef ノードを検証および変換して、実行エンジンが利用可能な形式に整形する。

シグネチャまたは形状

static void hstore_subscript_transform(SubscriptingRef *sbsref, List *indirection, ParseState *pstate, bool isSlice, bool isAssignment)

入力と出力

  • **入力**:
  • sbsref: 変換対象の SubscriptingRef ノード。
  • indirection: 添字式を含むリスト。
  • pstate: パーサの状態。
  • isSlice: スライス操作かどうかを示すフラグ。
  • isAssignment: 代入操作かどうかを示すフラグ。
  • **出力**: なし（sbsref ノードのフィールドを直接更新する）。

前提条件と不変条件

  • 添字は単一であること（isSlice が false かつ indirection の長さが 1 であること）。
  • 添字の型は text 型であるか、text 型へ強制変換可能であること。
  • 変換後の sbsref->refrestype は常に TEXTOID となる。

アルゴリズムまたは振る舞い

1. **検証**: スライス操作や複数添字が指定されている場合はエラーを発生させる。 2. **変換**: indirection から添字式を取り出し、transformExpr を用いて解析する。 3. **型強制**: coerce_to_target_type を使用し、添字式を TEXTOID へ変換する。変換失敗時はエラーを発生させる。 4. **ノード設定**: 変換された式を sbsref->refupperindexpr に格納し、reflowerindexpr を NIL に設定する。 5. **結果型設定**: sbsref->refrestype を TEXTOID に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_subs.c 内の hstore_subscript_handler が参照する SubscriptRoutines 構造体から間接的に呼び出される。
  • **呼び出し先**:
  • transformExpr (PostgreSQL パーサ)
  • coerce_to_target_type (PostgreSQL パーサ)
  • ereport (エラー報告)

関連データ構造

  • SubscriptingRef: 添字操作を表現するノード。
  • A_Indices: 添字の構文解析結果を保持する構造体。
  • TEXTOID: text 型の OID。

失敗モードと境界条件

  • **エラー**:
  • hstore がサポートしないスライス操作や複数添字指定時に ERRCODE_FEATURE_NOT_SUPPORTED を発生させる。
  • 添字式が text 型に変換できない場合に ERRCODE_DATATYPE_MISMATCH を発生させる。

保守担当者向けメモ

  • 本関数は hstore の添字が常に単一の text キーを対象とするという制約に基づいている。
  • hstore は不変データ構造であるため、代入操作（isAssignment）の実行時には新しい hstore が生成されるが、本関数はその前段階の構文解析のみを担当する。
