gistplacetopage

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gist.c:229

目的

GiSTインデックスの指定ページへタプルを挿入します。ページ容量が不足している場合はページ分割を実行し、必要に応じて再帰的なツリー更新のための情報を生成します。

シグネチャまたは形状

bool gistplacetopage(Relation rel, Size freespace, GISTSTATE *giststate,
                     Buffer buffer, IndexTuple *itup, int ntup, 
                     OffsetNumber oldoffnum, BlockNumber *newblkno,
                     Buffer leftchildbuf, List **splitinfo,
                     bool markfollowright, Relation heapRel, bool is_build)

入力と出力

  • **入力**:
  • buffer: 操作対象のバッファ。
  • itup, ntup: 挿入するタプルの配列とその数。
  • oldoffnum: 更新対象の古いタプル位置（存在する場合）。
  • **出力**:
  • 戻り値: ページ分割が発生した場合は true、それ以外は false。
  • splitinfo: 分割が発生した場合、親ノード更新に必要な情報が格納されるリスト。
  • newblkno: 分割後の新しいブロック番号。

前提条件と不変条件

  • 操作対象のページが不完全な分割状態（F_FOLLOW_RIGHT フラグがセットされている状態）であってはならない。
  • 削除済みページへの挿入は禁止されている。
  • ページ分割が発生した場合、分割後のページ数は GIST_MAX_SPLIT_PAGES を超えてはならない。

アルゴリズムまたは振る舞い

1. **事前チェック**: ページが不完全な分割状態でないか確認し、削除済みでないことを検証する。 2. **容量確認**: gistnospace を使用し、タプル挿入の余地があるか判定する。 3. **ガベージコレクション**: リーフページで容量不足かつゴミデータがある場合、gistprunepage で領域を解放し、再度容量を確認する。 4. **ページ分割**:

  • gistextractpage で全タプルを抽出し、挿入対象と結合する。
  • gistSplit を呼び出し、分割レイアウトを決定する。
  • 新しいバッファを割り当て、分割後の各ページを初期化する。
  • ルート分割の場合は、新しいルートページを構築し、子ノードへのダウンリンクを配置する。

    • 5. **完了**: 分割情報を splitinfo に格納し、呼び出し元へ制御を戻す。

呼び出し元と呼び出し先

  • **呼び出し元**: gistinsert（gist.c）、gistinserttuple（gist.c）、gistbuild（gistbuild.c）などから呼び出される。
  • **呼び出し先**: gistnospace, gistprunepage, gistextractpage, gistjoinvector, gistSplit, gistNewBuffer, GISTInitBuffer, PredicateLockPageSplit 等。

関連データ構造

  • GISTSTATE: インデックス操作の状態管理構造体。
  • SplitPageLayout: ページ分割時のレイアウト情報を保持する構造体。
  • IndexTuple: インデックスに格納されるタプル。

失敗モードと境界条件

  • **同時実行エラー**: ページが不完全な分割状態にある場合、ERROR をスローする。
  • **分割過多**: 分割によって生成されたページ数が GIST_MAX_SPLIT_PAGES を超える場合、ERROR をスローする。
  • **ルート分割**: ルートページが分割される場合、特殊なルート再構築処理が実行される。

保守担当者向けメモ

  • 本関数はページ分割の複雑なロジックを内包しており、特に並行処理時の整合性維持（NSNやF_FOLLOW_RIGHT）に注意が必要。
  • is_build フラグが true の場合、WALログの扱いが通常と異なるため、修正時は gistxlog.c のリプレイロジックとの整合性を確認すること。
