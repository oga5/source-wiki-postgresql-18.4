src/test/modules/test_ddl_deparse

システム内での役割

本モジュールは、PostgreSQL の DDL デパース（deparse）機能の検証を目的としたテスト用拡張です。ディレクトリ境界に基づき構成されています。

責務

DDL イベントトリガー等でキャプチャされた CollectedCommand 構造体の内容を SQL レベルで検査可能にし、内部的な DDL 解析結果が期待通りであるかを回帰テストで検証します。

このモジュールのファイル

  • src/test/modules/test_ddl_deparse/test_ddl_deparse.c

主要なデータ構造

  • CollectedCommand: DDL 操作の内容を保持する内部構造体。

主なフロー

1. **入力**: PG_GETARG_POINTER を通じて CollectedCommand を受け取ります。 2. **処理**:

  • コマンド種別やタグの抽出（get_command_type, get_command_tag）。
  • ALTER TABLE のサブコマンドリストを走査し、SRF（Set Returning Function）として結果を構築（get_altertable_subcmdinfo）。

    • 3. **出力**: tuplestore_putvalues を介して SQL クエリの結果セットとして返却します。

外部依存

  • postgres.h, funcapi.h
  • nodes/execnodes.h
  • tcop/deparse_utility.h, tcop/utility.h
  • utils/builtins.h

内部構成

  • 子モジュールなし。単一のソースファイルで構成されるテスト用ユーティリティ。

推奨読書順

1. src/test/modules/test_ddl_deparse/test_ddl_deparse.c

  • 新規 ALTER TABLE サブコマンド追加時の get_altertable_subcmdinfo 内の網羅的マッピングの更新プロセス。
