src/include/portability

システム内での役割

src/include/portability は、PostgreSQL が多様なプラットフォーム上で動作するために必要な、OS 固有の差異を吸収する抽象化レイヤーです。このモジュールはディレクトリ境界に基づいて定義されており、システムコールや定数のプラットフォーム依存性を隠蔽し、上位レイヤーに一貫したインターフェースを提供します。

責務

  • **時間計測の抽象化**: プラットフォーム固有のタイマー API を隠蔽し、高精度な時間計測機能を提供する。
  • **メモリ管理の正規化**: メモリ割り当てや共有メモリ操作に関連するフラグ、定数、アクセス権限をプラットフォーム間で統一する。
  • **移植性の確保**: OS 間の差異をプリプロセッサレベルで吸収し、メインロジックの可搬性を維持する。

このモジュールのファイル

  • src/include/portability/instr_time.h: 高精度タイマーの抽象化。
  • src/include/portability/mem.h: メモリ関連定数および IPC フラグの正規化。

主要なデータ構造

  • instr_time: 時間計測用の不透明な構造体。内部的には int64 のティック値を保持し、マクロ経由で操作されます。

主なフロー

本モジュールは主にヘッダーファイルで構成されており、実行時のフローよりもコンパイル時の解決が中心です。 1. **時間計測**: INSTR_TIME_SET_CURRENT 等のマクロにより、OS 固有の関数（clock_gettime 等）を呼び出し、instr_time 構造体に値を格納・演算・変換します。 2. **メモリ定義**: コンパイル時に各 OS のヘッダーを評価し、MAP_FAILED や PG_MMAP_FLAGS 等の定数が未定義であれば適切なデフォルト値を割り当てます。

外部依存

  • <time.h>: Unix 系環境における時間計測機能の利用。
  • 各プラットフォームのシステムヘッダ: OS 固有のメモリ管理およびタイマー API 定義。

内部構成

  • 子モジュールは存在しません。

推奨読書順

1. src/include/portability/instr_time.h: 時間計測の抽象化手法を理解する。 2. src/include/portability/mem.h: メモリ関連の定数正規化戦略を確認する。
