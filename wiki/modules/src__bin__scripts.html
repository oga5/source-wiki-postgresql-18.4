src/bin/scripts

システム内での役割

src/bin/scripts は、PostgreSQL のデータベース管理者がシェルから直接実行するクライアントサイドのメンテナンスユーティリティ群です。データベースの作成・削除、ロール管理、統計情報の更新、インデックスの再構築、およびサーバーの稼働確認といった定型的な運用タスクを自動化します。

責務

  • ユーザー入力を適切な SQL 文（VACUUM, REINDEX, CREATE/DROP 等）へ変換し、サーバーへ発行する。
  • libpq を介したデータベース接続管理と、共通のオプション解析・ログ出力の提供。
  • データベースカタログの動的なクエリによる処理対象オブジェクトの特定。
  • 並列実行（--jobs）の制御と、サーバーバージョンに応じた機能のバリデーション。

このモジュールのファイル

  • **共通基盤**: common.c, common.h
  • **メンテナンスツール**: vacuumdb.c, reindexdb.c, clusterdb.c
  • **管理ツール**: createdb.c, dropdb.c, createuser.c, dropuser.c
  • **監視ツール**: pg_isready.c

主要なデータ構造

  • vacuumingOptions: vacuumdb の実行オプションを保持。
  • ReindexType: reindexdb における再構築対象の種類を定義。
  • VacObjFilter: 処理対象を絞り込むためのフィルタ種別。
  • ConnParams: 接続先データベースや認証情報を保持する構造体。

主なフロー

1. **初期化**: getopt_long を用いてコマンドライン引数を解析し、接続パラメータを決定。 2. **接続**: connectMaintenanceDatabase を介して対象サーバーへ接続。 3. **対象特定**: 必要に応じてカタログをクエリし、処理対象のリスト（テーブルやインデックス）を生成。 4. **コマンド構築**: PQExpBuffer を使用し、セキュリティを考慮した SQL 文を動的に生成。 5. **実行**: PQexec または並列スロット（ParallelSlots）を用いて SQL を発行し、結果を検証。

外部依存

  • libpq: データベース接続およびクエリ実行の基盤。
  • fe_utils: クライアントツール共通の接続管理、並列処理、文字列操作、クエリ構築ユーティリティ。
  • common: ログ出力や接続補助などの低レベルユーティリティ。

内部構成

本モジュールは、各ツールが common.c および fe_utils を共有するフラットな構成です。子モジュールは存在せず、各ソースファイルが個別のバイナリに対応しています。

推奨読書順

1. src/bin/scripts/common.h (インターフェースの理解) 2. src/bin/scripts/common.c (共通ロジックの実装) 3. src/bin/scripts/vacuumdb.c (並列処理とカタログ操作の代表例) 4. src/bin/scripts/reindexdb.c (並列処理の応用) 5. src/bin/scripts/createdb.c (基本的な SQL 生成パターン) 6. src/bin/scripts/pg_isready.c (疎通確認の軽量な実装)
