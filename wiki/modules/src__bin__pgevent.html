src/bin/pgevent

システム内での役割

Windows 環境において、PostgreSQL が Windows イベントログシステムと連携するための DLL（pgevent.dll）を提供するモジュールです。本モジュールはディレクトリ境界に基づき定義されています。

責務

  • Windows イベントログのソースとして PostgreSQL を登録・管理する。
  • レジストリ（SYSTEM\CurrentControlSet\Services\EventLog\Application）へのイベントソース情報の書き込みおよび削除。
  • Windows メッセージコンパイラが期待するメッセージ識別子の定義。

このモジュールのファイル

  • src/bin/pgevent/pgevent.c: DLL のエントリポイントおよびレジストリ操作の実装。
  • src/bin/pgevent/pgmsgevent.h: イベントログ用メッセージ識別子（PGWIN32_EVENTLOG_MSG）の定義。

主要なデータ構造

  • PGWIN32_EVENTLOG_MSG: Windows イベントログに記録されるメッセージのベースとなる 32 ビット識別子。
  • g_module: DLL のモジュールハンドルを保持するグローバル変数。

主なフロー

1. **登録**: regsvr32 を通じて DllRegisterServer が呼び出され、DLL のパスやイベントソース情報をレジストリに書き込む。 2. **インストール制御**: DllInstall が呼び出されると、必要に応じてイベントソース名を更新し、登録処理を制御する。 3. **削除**: DllUnregisterServer が呼び出され、レジストリからイベントソース情報を削除する。

外部依存

  • postgres_fe.h
  • Windows API（レジストリ操作関数、olectl.h 等）

内部構成

  • 子モジュール数: 0

推奨読書順

1. src/bin/pgevent/pgmsgevent.h（メッセージ定義の理解） 2. src/bin/pgevent/pgevent.c（レジストリ操作と DLL 実装の理解）
