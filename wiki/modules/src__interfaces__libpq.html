src/interfaces/libpq

システム内での役割

src/interfaces/libpq は、PostgreSQL の公式クライアントライブラリである libpq の実装ディレクトリです。アプリケーションが PostgreSQL サーバーと通信するための公開 API (libpq-fe.h) と、それを支える内部実装、認証メカニズム、プラットフォーム抽象化層を包括しています。このモジュールはディレクトリ境界に基づき定義されています。

責務

  • **接続管理**: サーバーへの接続確立、認証ネゴシエーション、SSL/TLS および GSSAPI によるセキュア通信の提供。
  • **クエリ実行**: SQL の送信、結果セット (PGresult) の構築、パイプラインモードの制御。
  • **プロトコル解析**: PostgreSQL プロトコル v3.0 のメッセージ送受信および解析。
  • **プラットフォーム抽象化**: Windows (Winsock/SSPI) と Unix 間のソケット I/O やスレッド処理の差異の吸収。
  • **拡張性**: イベント API によるライフサイクル管理や、SASL/OAuth 認証のプラグイン可能なアーキテクチャの提供。

このモジュールのファイル

本モジュールには、認証ロジック (fe-auth-*.c)、接続管理 (fe-connect.c)、クエリ実行 (fe-exec.c)、通信 I/O (fe-misc.c)、セキュリティ抽象化 (fe-secure*.c)、および公開ヘッダー (libpq-fe.h) を含む 31 個のソースファイルが含まれます。

主要なデータ構造

  • PGconn: データベース接続の全状態を保持するメイン構造体。
  • PGresult: クエリ実行結果、メタデータ、エラー情報を保持する構造体。
  • PQExpBuffer: 動的に拡張可能な文字列バッファ。
  • SASLStatus / fe_oauth_state: 認証プロセス中の状態管理用構造体。
  • PGEventProc: 接続や結果セットのライフサイクルをフックするためのイベントハンドラ。

主なフロー

1. **接続**: PQconnectdb が PGconn を初期化し、PQconnectPoll を通じてソケット接続と認証（SSL/SASL等）を非同期に実行。 2. **実行**: PQsendQuery がコマンドを構築し、fe-protocol3.c を介してサーバーへ送信。 3. **受信**: PQgetResult がサーバーからの応答を解析し、fe-exec.c が PGresult を生成。 4. **終了**: PQfinish が接続を閉じ、関連リソースを解放。

外部依存

  • postgres_fe.h, postgres_ext.h: バックエンド共通型定義。
  • OpenSSL: セキュア通信用。
  • GSSAPI: 認証用。
  • win32.h / pthread: プラットフォーム固有の互換性レイヤー。

内部構成

  • **通信層**: fe-misc.c (低レベル I/O), fe-protocol3.c (プロトコル解析)。
  • **認証層**: fe-auth.c (ディスパッチャ), fe-auth-sasl.h (SASL 抽象化), fe-auth-oauth.c (OAuth 実装)。
  • **セキュリティ層**: fe-secure.c (抽象化), fe-secure-openssl.c (OpenSSL 実装)。
  • **ユーティリティ**: pqexpbuffer.c (バッファ管理), fe-trace.c (デバッグトレース)。

推奨読書順

1. libpq-fe.h (公開 API の全体像) 2. libpq-int.h (内部構造体 PGconn の理解) 3. fe-connect.c (接続確立フロー) 4. fe-exec.c (クエリ実行と結果管理) 5. fe-protocol3.c (プロトコル解析の仕組み) 6. fe-auth.c (認証ディスパッチ) 7. fe-secure.c (セキュリティ抽象化) 8. pqexpbuffer.c (文字列操作ユーティリティ)
