contrib/pageinspect

システム内での役割

contrib/pageinspect は、PostgreSQL の物理ストレージ層を低レベルで検査するための拡張モジュールです。データベースのページ（ヒープや各種インデックス）をバイナリデータとして抽出し、その内部構造を SQL インターフェースを通じて可視化・解析する機能を提供します。

責務

  • 物理ページ（Page）のバイナリ抽出およびアライメント調整。
  • ヒープおよび各インデックス（B-tree, GIN, GiST, Hash, BRIN）の内部構造のデコード。
  • ページヘッダー、タプル、メタデータ、空き領域マップ（FSM）の統計情報および内容の可視化。
  • 物理データへの直接アクセスに伴うスーパーユーザー権限の強制と安全なバリデーション。

このモジュールのファイル

  • **共通基盤**: pageinspect.h, rawpage.c
  • **解析ロジック**: heapfuncs.c, btreefuncs.c, ginfuncs.c, gistfuncs.c, hashfuncs.c, brinfuncs.c, fsmfuncs.c

主要なデータ構造

  • PageHeaderData: ページ共通のヘッダー情報。
  • BTPageStat, HashPageStat: 各インデックスの統計情報を保持する構造体。
  • heap_page_items_state, gin_leafpage_items_state: SRF（Set Returning Function）実行時の状態保持用コンテキスト。
  • brin_column_state: BRIN インデックスの出力制御用構造体。

主なフロー

1. **ページ取得**: SQL 関数が get_raw_page() を呼び出し、共有バッファから対象ブロックを bytea として取得。 2. **変換**: get_page_from_raw() により、bytea をメモリ上で適切にアライメントされた Page 構造体に変換。 3. **解析**: 各 *funcs.c がページタイプに応じた構造体（Opaque 領域など）へキャストし、アイテムやメタデータを抽出。 4. **出力**: FuncCallContext を利用した SRF により、解析結果をテーブル形式で返却。

外部依存

  • storage/bufpage.h, storage/bufmgr.h: ページレイアウトおよびバッファ管理。
  • access/: 各アクセスメソッド（nbtree.h, gin_private.h, gist.h, hash.h, brin_internal.h 等）の内部定義。
  • funcapi.h: SQL 関数としての SRF 実装。

内部構成

本モジュールはディレクトリ境界によって定義されており、各ソースファイルは特定のアクセスメソッドに対応する解析関数を独立して実装しています。

推奨読書順

1. contrib/pageinspect/pageinspect.h (共通定義の理解) 2. contrib/pageinspect/rawpage.c (ページ取得の基盤) 3. contrib/pageinspect/btreefuncs.c (インデックス解析の代表例) 4. contrib/pageinspect/heapfuncs.c (ヒープ解析) 5. contrib/pageinspect/hashfuncs.c / brinfuncs.c / gistfuncs.c / ginfuncs.c (各インデックス特有の解析) 6. contrib/pageinspect/fsmfuncs.c (FSM 解析)
