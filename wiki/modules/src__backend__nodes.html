src/backend/nodes

システム内での役割

src/backend/nodes は、PostgreSQL のクエリ解析、最適化、および実行フェーズで使用される「ノード（Node）」データ構造の基盤です。クエリツリーやプランツリーを構成するノードの生成、操作、シリアライズ、比較、およびメモリ管理を一元的に提供します。

責務

  • **ノードのライフサイクル管理**: ノードの生成（コンストラクタ）、ディープコピー、およびメモリ解放。
  • **ツリー操作の抽象化**: 式ツリーやプランツリーの再帰的な走査（Walker）および変形（Mutator）。
  • **シリアライズとデシリアライズ**: デバッグやプランの永続化のためのテキスト形式への変換と復元。
  • **集合演算の提供**: List や Bitmapset を用いた効率的な集合管理と演算。
  • **拡張性の提供**: 外部モジュールが独自のノード型を登録・利用するためのレジストリ機能。

このモジュールのファイル

  • **コンテナ・集合**: list.c, bitmapset.c, multibitmapset.c, tidbitmap.c
  • **ノード操作・変換**: copyfuncs.c, equalfuncs.c, nodeFuncs.c, makefuncs.c
  • **シリアライズ**: outfuncs.c, readfuncs.c, read.c, print.c
  • **メタデータ・拡張**: extensible.c, params.c, queryjumblefuncs.c, value.c

主要なデータ構造

  • Node: 全ノードの基底構造体（NodeTag を保持）。
  • List: 動的配列ベースの汎用コンテナ。
  • Bitmapset / TIDBitmap: 整数集合およびタプルID集合を効率的に管理するビットマップ。
  • ParamListInfo: クエリ実行時の外部パラメータ管理構造。
  • ExtensibleNode: 拡張機能が独自ノードを定義するためのインターフェース。

主なフロー

1. **生成**: makefuncs.c や value.c を用いてパーサーがノードツリーを構築。 2. **走査・変形**: nodeFuncs.c の Walker/Mutator を用いて、プランナーがツリーを最適化・変形。 3. **シリアライズ**: デバッグやログ出力時に outfuncs.c で文字列化し、print.c で整形出力。 4. **比較・コピー**: equalfuncs.c や copyfuncs.c を用いて、プランの同一性判定や複製を実行。

外部依存

  • postgres.h (基本型・エラー処理)
  • utils/memutils.h (メモリコンテキスト管理)
  • common/hashfn.h (ハッシュ計算)
  • port/pg_bitutils.h (ビット操作)

内部構成

本モジュールは、ノードの型ごとに処理を分岐する switch 文を多用する設計です。一部の比較・コピー・読み込みロジックは、保守性のために自動生成された補助ファイル（*.switch.c や *.funcs.c）に分割されています。

推奨読書順

1. list.c (コンテナの基本) 2. bitmapset.c (集合演算の基本) 3. nodeFuncs.c (ツリー走査の基本) 4. makefuncs.c (ノード生成の基本) 5. copyfuncs.c / equalfuncs.c (ノードの基本操作) 6. outfuncs.c / readfuncs.c (シリアライズ) 7. tidbitmap.c (高度なデータ構造) 8. extensible.c (拡張機能の仕組み)

未解決の問い

  • copyfuncs.c 等の自動生成コードと手書きコードの境界の明確化。
