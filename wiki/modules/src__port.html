src/port

システム内での役割

src/port は、PostgreSQL のクロスプラットフォームな移植性を支える基盤モジュールです。OS やアーキテクチャ間で異なるシステムコール、ライブラリ関数、ハードウェア機能を抽象化し、PostgreSQL のコアおよびフロントエンドツールが一貫したインターフェースで動作できるようにします。

責務

  • **プラットフォーム抽象化**: Windows と Unix 系 OS 間でのファイル操作、スレッド、ネットワーク、ロケール処理の差異を吸収する。
  • **ハードウェア最適化**: CPU 命令セット（AVX-512, ARMv8 CRC 等）の実行時判定と動的ディスパッチによる高速化。
  • **標準ライブラリの補完**: POSIX 準拠の snprintf や qsort など、OS 標準の実装が欠如または不完全な場合の代替実装の提供。

このモジュールのファイル

本モジュールは 64 個のソースファイルで構成されており、主に以下のカテゴリに分類されます。

  • **Windows 互換レイヤー**: win32*.c ファイル群（win32stat.c, win32dlopen.c, dirmod.c 等）。
  • **ビット操作・最適化**: pg_bitutils.c, pg_popcount_*.c, pg_crc32c_*.c。
  • **文字列・フォーマット**: snprintf.c, pgstrcasecmp.c, strerror.c。
  • **パス・ファイル操作**: path.c, tar.c。
  • **ロケール・エンコーディング**: chklocale.c, pg_localeconv_r.c。

主要なデータ構造

  • PrintfTarget: snprintf 実装における出力先管理構造体。
  • lconv_member_info: ロケール情報のコピーと管理のためのメタデータ。
  • REPARSE_JUNCTION_DATA_BUFFER: Windows ジャンクションポイント操作用構造体。
  • pthread_mutex_t (Windows): Windows ネイティブの CRITICAL_SECTION をラップしたスレッド同期用構造体。

主なフロー

1. **初期化とディスパッチ**: pg_popcount や pg_comp_crc32c などの最適化関数は、初回呼び出し時に CPU 機能を判定し、最適な実装へ関数ポインタを書き換える（Function Pointer Swapping）。 2. **プラットフォーム抽象化**: stat や rename などの標準関数呼び出しは、Windows 環境では _pgstat64 や pgrename といったラッパーを経由し、リトライロジックやエラーコード変換を経て実行される。 3. **ロケール処理**: pg_localeconv_r は、スレッドセーフではない標準の localeconv を、OS 固有のロックやスレッドローカルなロケール設定を用いて安全にラップする。

外部依存

  • **OS API**: windows.h (Windows), sys/stat.h, unistd.h, pthread.h 等。
  • **PostgreSQL 共通ヘッダー**: c.h, postgres.h, mb/pg_wchar.h。
  • **ハードウェアヘッダー**: immintrin.h (AVX-512), arm_neon.h (AArch64)。

内部構成

本モジュールは、特定のサブシステムに依存せず、PostgreSQL 全体から利用されるユーティリティの集合体です。ディレクトリ境界に基づき、機能ごとに独立したファイルとして構成されています。

推奨読書順

1. src/port/path.c (パス操作の基本) 2. src/port/snprintf.c (移植性レイヤーの代表例) 3. src/port/pg_bitutils.c (動的ディスパッチの設計) 4. src/port/dirmod.c (Windows 互換性の核心) 5. src/port/qsort.c (テンプレート利用の設計)
