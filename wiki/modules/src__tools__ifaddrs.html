src/tools/ifaddrs

システム内での役割

本モジュールは、PostgreSQL が提供するネットワークインターフェース列挙 API の動作検証およびデバッグを目的としたスタンドアロンのテストツール群です。ディレクトリ境界に基づき定義されています。

責務

  • pg_foreach_ifaddr() API の疎通確認。
  • システム上のネットワークインターフェースおよびサブネットマスクの列挙と表示。
  • プラットフォーム依存（特に Windows の WSAStartup 等）のネットワーク初期化処理の検証。

このモジュールのファイル

  • src/tools/ifaddrs/test_ifaddrs.c: ネットワークインターフェース列挙機能の検証用メインプログラム。

主要なデータ構造

  • sockaddr: インターフェースのアドレス情報を保持する標準的なソケット構造体。

主なフロー

1. **初期化**: プラットフォームに応じたソケットライブラリのセットアップ。 2. **列挙**: pg_foreach_ifaddr() を呼び出し、OS のネットワークインターフェースリストを走査。 3. **コールバック**: インターフェースごとに callback 関数が起動。 4. **変換・出力**: getnameinfo を用いて sockaddr を文字列化し、標準出力へ表示。

外部依存

  • postgres.h: PostgreSQL の基本定義および抽象化レイヤー。
  • libpq/ifaddr.h: 検証対象である pg_foreach_ifaddr の宣言。
  • 標準ネットワークヘッダー (netinet/in.h, arpa/inet.h 等)。

内部構成

  • 子モジュールなし。単一のテストツールで構成される。

推奨読書順

1. src/tools/ifaddrs/test_ifaddrs.c: テストツールの実装と API の使用例を確認。 2. src/include/libpq/ifaddr.h: 検証対象 API の定義を確認。

未解決の問い

  • 本ツールがカバーするプラットフォームの網羅性や、将来的なテストケース拡張の必要性。
