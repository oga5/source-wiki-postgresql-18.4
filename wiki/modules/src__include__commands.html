src/include/commands

システム内での役割

src/include/commands は、PostgreSQL における DDL（データ定義言語）および主要なメンテナンスコマンドの実行インターフェースを定義するモジュールです。SQL パーサーが生成した構文解析ツリー（Stmt）を、バックエンドの実行ロジックへと橋渡しする役割を担います。

責務

  • データベースオブジェクト（テーブル、インデックス、シーケンス、型、トリガー等）のライフサイクル管理。
  • VACUUM、CLUSTER、COPY などのメンテナンスおよびデータ転送コマンドの仕様定義。
  • 外部セキュリティプロバイダーやイベントトリガー、拡張機能のためのフックインターフェースの提供。
  • WAL（Write-Ahead Logging）リプレイ時に必要な物理操作のレコード構造定義。

このモジュールのファイル

本ディレクトリには 39 個のヘッダーファイルが含まれており、主に src/backend/commands/ 以下の実装と対になっています。

  • **主要カテゴリ**: copy.h (データ転送), vacuum.h (メンテナンス), trigger.h (トリガー), sequence.h (シーケンス), explain.h (クエリ計画), tablespace.h (ストレージ管理), typecmds.h (型定義), user.h (ロール管理), event_trigger.h (イベントトリガー) 等。

主要なデータ構造

  • CopyFromStateData / CopyToState: COPY コマンドの実行状態を保持。
  • VacuumParams / ClusterParams: メンテナンスコマンドの動作オプション。
  • TriggerData: トリガー実行時のコンテキスト情報。
  • ExplainState: EXPLAIN コマンドの出力設定と状態管理。
  • PreparedStatement: プリペアドステートメントのメタデータ管理。
  • xl_dbase_drop_rec 等: WAL リプレイ用の物理レコード構造。

主なフロー

1. **解析**: SQL 文がパーサーにより Stmt 構造体（CreateStmt, VacuumStmt 等）に変換される。 2. **ディスパッチ**: ProcessUtility 等のディスパッチャが、本モジュールで定義された関数（DoCopy, vacuum, DefineTrigger 等）を呼び出す。 3. **実行**: 各コマンドの実装がシステムカタログ（pg_class, pg_trigger 等）を更新し、必要に応じて物理ストレージ操作を行う。 4. **記録**: 変更内容は WAL に書き込まれ、クラッシュリカバリやレプリケーションに備える。

外部依存

  • catalog/: システムカタログ定義（pg_type, pg_trigger 等）。
  • nodes/: パーサーおよびエグゼキューターのノード定義（parsenodes.h, execnodes.h）。
  • access/: テーブルアクセス手法（AM）および WAL 関連（xlogreader.h）。
  • storage/: ロック管理およびバッファ管理。
  • utils/: リレーションキャッシュや GUC 設定。

内部構成

本モジュールはディレクトリ境界によって定義されており、特定のサブモジュール構造は持ちません。各ヘッダーファイルは、対応する src/backend/commands/ 内のソースファイルと密接に結合しています。

推奨読書順

1. progress.h (進捗報告の基盤) 2. trigger.h (トリガーの基本) 3. vacuum.h (メンテナンスの基本) 4. sequence.h (シーケンス管理) 5. copy.h / copyfrom_internal.h (データ転送) 6. explain.h / explain_state.h (クエリ解析) 7. tablespace.h (ストレージ管理) 8. cluster.h (物理再配置) 9. event_trigger.h (DDL フック) 10. user.h (セキュリティ管理)
