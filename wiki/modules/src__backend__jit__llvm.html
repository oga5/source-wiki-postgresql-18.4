src/backend/jit/llvm

システム内での役割

PostgreSQL の JIT コンパイルを支える LLVM プロバイダーの実装です。クエリ実行時の式評価やタプルデフォーム処理を LLVM IR に変換し、ネイティブ機械語として実行することで、クエリ実行のオーバーヘッドを削減します。

責務

  • **JIT コンパイルのライフサイクル管理**: LLVM コンテキストの初期化、最適化、および ORC JIT を介した実行可能コードへの変換。
  • **IR 生成と最適化**: 式評価（ExprState）やタプルデフォーム処理の LLVM IR への変換、およびモジュール境界を越えた関数インライン展開による最適化。
  • **メモリ管理**: JIT 生成コードの実行可能メモリ領域の確保と、ARM 等のアーキテクチャに適合したメモリ保護の適用。
  • **エラーハンドリング**: LLVM 内部の致命的エラーやメモリ不足を PostgreSQL のエラーシステムへ安全に伝播させるブリッジ機能。
  • **型情報の同期**: PostgreSQL の C 構造体定義を LLVM が認識可能な型情報として提供。

このモジュールのファイル

  • SectionMemoryManager.cpp: JIT 生成コードのメモリ管理と保護。
  • llvmjit.c: JIT プロバイダーのコア実装とコンテキスト管理。
  • llvmjit_deform.c: タプルデフォーム処理の JIT コンパイル。
  • llvmjit_error.cpp: LLVM エラーと PostgreSQL エラーのブリッジ。
  • llvmjit_expr.c: 式評価の JIT コンパイル。
  • llvmjit_inline.cpp: モジュール境界を越えた関数インライン展開。
  • llvmjit_types.c: LLVM 用の型定義カタログ。
  • llvmjit_wrap.cpp: LLVM C API と C++ API を繋ぐラッパー。

主要なデータ構造

  • LLVMJitContext: JIT コンパイル状態を保持するコンテキスト。
  • CompiledExprState: コンパイル済み式の状態管理。
  • InlineWorkList / FunctionInlineState: インライン化計画とコスト管理。
  • SectionMemoryManager: JIT コードのメモリ配置マネージャ。
  • ImportMapTy: インライン化対象のモジュールマップ。

主なフロー

1. **初期化**: llvm_session_initialize によりターゲットマシンと ORC JIT インスタンスを構築。 2. **コンパイル**: llvm_compile_expr 等が式やデフォーム処理を IR に変換し、llvm_inline で最適化。 3. **実行**: llvm_get_function を通じて ORC JIT が機械語を生成し、関数ポインタを解決。 4. **メモリ確定**: SectionMemoryManager が生成コードをメモリに配置し、実行権限を付与。

外部依存

  • **LLVM**: Core, Orc, ExecutionEngine, Bitcode 等のライブラリ。
  • **PostgreSQL Core**: postgres.h, executor/execExpr.h, utils/resowner.h 等の実行エンジンおよびメモリ管理基盤。

内部構成

本モジュールはディレクトリ境界により定義されており、LLVM の C API を活用する C/C++ 実装群で構成されます。

推奨読書順

1. llvmjit.c (全体像とライフサイクル) 2. llvmjit_expr.c (式評価のコンパイル) 3. llvmjit_deform.c (タプルデフォームの最適化) 4. llvmjit_inline.cpp (インライン化戦略) 5. llvmjit_types.c (型定義のブリッジ) 6. SectionMemoryManager.cpp (メモリ管理) 7. llvmjit_error.cpp (エラーハンドリング) 8. llvmjit_wrap.cpp (API ラッパー)
