src/test/examples

システム内での役割

本モジュールは、PostgreSQL の C 言語用クライアントライブラリである libpq の主要な機能と使用パターンを実証するためのサンプルプログラム群です。開発者が libpq を利用したアプリケーションを構築する際の参照実装として機能します。

責務

  • libpq の基本 API（接続、クエリ実行、結果取得）の提示。
  • ラージオブジェクト（Large Object）操作の標準および 64 ビット API の実演。
  • パラメータ化クエリやバイナリデータ送受信などの高度な機能の解説。
  • 非同期通知（LISTEN/NOTIFY）やマルチコネクション管理といった応用的な接続パターンの提供。

このモジュールのファイル

  • testlibpq.c: 基本的な接続とカーソル操作。
  • testlibpq2.c: 非同期通知の受信とイベントループ。
  • testlibpq3.c: パラメータ化クエリとバイナリデータ処理。
  • testlibpq4.c: 複数データベース接続の管理。
  • testlo.c: ラージオブジェクトの基本操作。
  • testlo64.c: 64 ビットオフセットを用いたラージオブジェクト操作。

主要なデータ構造

  • PGconn: データベース接続状態を保持する構造体。
  • PGresult: クエリ実行結果を保持する構造体。
  • PGnotify: 非同期通知メッセージを格納する構造体。

主なフロー

1. **接続**: PQconnectdb 等でデータベースへ接続し、セキュリティのため search_path を設定。 2. **トランザクション**: BEGIN / END を発行し、一連の操作をアトミックに実行。 3. **操作**: 各サンプルの目的に応じ、SQL 実行、ラージオブジェクトの読み書き、またはイベント待機を行う。 4. **解放**: PQclear で結果セットを、PQfinish で接続を確実に解放し、メモリリークを防止。

外部依存

  • libpq-fe.h: libpq の主要 API。
  • libpq/libpq-fs.h: ラージオブジェクト操作用 API。
  • 標準 C ライブラリ（stdio.h, stdlib.h, string.h 等）。
  • ネットワーク関連（netinet/in.h 等、バイトオーダー変換用）。

内部構成

本モジュールはディレクトリ境界に基づいたフラットな構成であり、各ファイルは独立したサンプルプログラムとして実装されています。

推奨読書順

1. testlibpq.c（基本の理解） 2. testlibpq3.c（パラメータ化クエリとバイナリ I/O） 3. testlibpq2.c（非同期処理） 4. testlibpq4.c（マルチコネクション） 5. testlo.c（ラージオブジェクトの基本） 6. testlo64.c（64 ビット API による拡張操作）
