src/bin/pgbench

システム内での役割

pgbench は PostgreSQL の標準的な負荷テスト・ベンチマークツールです。マルチスレッドによる並行クライアントの実行、SQL スクリプトの解析・実行、および詳細な統計情報の収集とレポート出力を行います。

責務

  • **負荷生成**: 複数のクライアント接続を並行管理し、libpq を介してデータベースへクエリを送信する。
  • **スクリプト実行**: SQL 文および \set や \if などのメタコマンドを解釈・実行する。
  • **式評価**: ベンチマークスクリプト内の計算式を抽象構文木（AST）として解析し、動的に評価する。
  • **統計計測**: トランザクションのレイテンシ、TPS（Transactions Per Second）、エラー率を計測する。
  • **環境構築**: ベンチマーク用スキーマの初期化およびデータロード。

このモジュールのファイル

  • src/bin/pgbench/pgbench.c: メインプログラム。ステートマシンによる非同期 I/O 処理と実行制御の中核。
  • src/bin/pgbench/pgbench.h: 式評価エンジンおよび共通データ構造の定義。

主要なデータ構造

  • **実行コンテキスト**: CState（クライアント接続状態）、TState（スレッドごとの実行コンテキスト）。
  • **式評価**: PgBenchExpr（AST ノード）、PgBenchValue（評価結果を保持する共用体）。
  • **統計**: StatsData（レイテンシや成功・失敗の集計）。
  • **その他**: socket_set（I/O 多重化用）、ConnectionStateEnum（クライアントのライフサイクル状態）。

主なフロー

1. **初期化**: 引数解析後、スレッドとクライアントを初期化。 2. **スレッド実行**: threadRun() がメインループとなり、poll/select でソケットを監視。 3. **状態遷移**: イベント発生時に advanceConnectionState() が呼ばれ、CState を更新。 4. **クエリ処理**: sendCommand() でクエリ送信、readCommandResponse() で結果受信と変数代入を行う。 5. **終了処理**: 全クライアント完了後、printResults() で統計情報を集計・出力。

外部依存

  • libpq: データベース接続およびクエリ実行。
  • fe_utils: PostgreSQL フロントエンド共通ユーティリティ（cancel.h, option_utils.h 等）。
  • common: pg_prng.h（乱数生成）、logging.h 等の共通ライブラリ。

内部構成

本モジュールはディレクトリ境界により定義されています。pgbench.c が巨大なため、機能（式評価、ステートマシン、統計）ごとの分割が課題となっています。また、exprparse.y（パーサー）および exprscan.l（レクサー）と密接に連携して動作します。

推奨読書順

1. src/bin/pgbench/pgbench.h（データ構造と式評価の基盤を理解） 2. src/bin/pgbench/pgbench.c（ステートマシンと実行フローを理解）

未解決の問い

  • 式評価ロジックとステートマシンロジックの分離によるリファクタリングの検討。
  • 巨大な pgbench.c の機能分割による保守性の向上。
