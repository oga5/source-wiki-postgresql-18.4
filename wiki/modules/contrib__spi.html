contrib/spi

システム内での役割

contrib/spi は、PostgreSQL の SPI (Server Programming Interface) を活用したトリガーベースの拡張機能群を提供するモジュールです。データベースの自動化処理（自動採番、更新日時記録、ユーザー名記録、参照整合性チェック）を実装するためのサンプルおよび実用的なユーティリティとして機能します。

責務

  • **自動化処理の提供**: トリガー関数を通じて、テーブル操作時に特定のカラムを自動更新・検証する。
  • **SPI の活用**: SQL 実行やタプル操作を伴う複雑なロジックを、SPI を介してデータベースエンジン内部で実行する。
  • **実行計画の管理**: 繰り返し実行されるクエリの計画をキャッシュし、パフォーマンスを最適化する。

このモジュールのファイル

  • contrib/spi/autoinc.c: シーケンス値による自動採番。
  • contrib/spi/insert_username.c: 挿入・更新時のユーザー名自動記録。
  • contrib/spi/moddatetime.c: 更新時のタイムスタンプ自動更新。
  • contrib/spi/refint.c: トリガーベースの参照整合性制約の実装。

主要なデータ構造

  • EPlan: SPIPlanPtr（実行計画）と識別子を保持し、セッション内での計画再利用を管理する構造体。
  • TriggerData: トリガー起動時に渡される、テーブル情報やタプルデータを含むコンテキスト構造体。

主なフロー

1. **トリガー起動**: BEFORE トリガーとして関数が呼び出され、TriggerData を取得。 2. **引数解析**: トリガー定義から対象カラム名やシーケンス名などの引数を解析。 3. **値の生成・検証**: 必要に応じて nextval() やユーザー名取得、あるいは SPI_execp を用いた参照先テーブルの検証を実行。 4. **タプル更新**: heap_modify_tuple_by_cols を使用して、変更後のタプルを構築しトリガーマネージャへ返却。

外部依存

  • **SPI インターフェース**: executor/spi.h
  • **トリガー管理**: commands/trigger.h
  • **基本型・ユーティリティ**: postgres.h, utils/builtins.h, utils/rel.h, access/htup_details.h
  • **その他**: commands/sequence.h, catalog/pg_type.h, utils/fmgrprotos.h, miscadmin.h

内部構成

本モジュールはディレクトリ境界によって定義されており、各ソースファイルは独立したトリガー関数として実装されています。共有ライブラリとしての内部的な階層構造はありません。

推奨読書順

1. contrib/spi/refint.c (SPI によるクエリ実行と計画キャッシュの理解) 2. contrib/spi/autoinc.c (シーケンス操作とタプル更新の基本) 3. contrib/spi/insert_username.c (シンプルなメタデータ注入の例) 4. contrib/spi/moddatetime.c (タイムスタンプ操作と型チェック)

未解決の問い

  • 現代の PostgreSQL において標準機能（FOREIGN KEY 制約や GENERATED カラムなど）で代替可能な機能が多いため、本モジュールの将来的なメンテナンス方針や、学習用リソースとしての位置付けを明確にする必要がある。
