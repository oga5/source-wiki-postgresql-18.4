src/include/lib

システム内での役割

src/include/lib は、PostgreSQL のバックエンドおよび共有メモリ環境で利用される、汎用的かつ高性能なデータ構造とアルゴリズムのライブラリ群です。ディレクトリ境界に基づき、再利用性の高いコンポーネントを統合しています。

責務

  • 高度なデータ構造（木、ヒープ、ハッシュ、リスト）の提供。
  • メモリ効率と実行速度を最適化したアルゴリズムの実装。
  • 共有メモリ（DSA）およびローカルメモリ（MemoryContext）環境への対応。
  • マクロによるメタプログラミングを用いた、型安全かつインライン展開可能なコード生成。

このモジュールのファイル

  • **データ構造**: binaryheap.h, pairingheap.h, rbtree.h, radixtree.h, ilist.h, dshash.h, simplehash.h, integerset.h, bloomfilter.h, hyperloglog.h
  • **アルゴリズム・ユーティリティ**: sort_template.h, bipartite_match.h, knapsack.h, qunique.h, stringinfo.h

主要なデータ構造

  • **ハッシュ**: dshash_table (共有メモリ対応), simplehash (型特化型)
  • **木構造**: RT_RADIX_TREE (適応型基数木), RBTree (赤黒木)
  • **ヒープ**: binaryheap, pairingheap
  • **リスト**: dlist, slist (侵入型連結リスト)
  • **その他**: StringInfo (可変長文字列), IntegerSet (圧縮整数集合), bloom_filter

主なフロー

各データ構造は、利用側でマクロを定義しヘッダーをインクルードすることで、特定の型に最適化された関数群を生成します。 1. **初期化**: _create や _allocate 関数でメモリを確保。 2. **操作**: _insert, _find, _delete 等の API を通じてデータを操作。 3. **走査**: 提供されるイテレータマクロを用いて要素を順次処理。 4. **解放**: _free または _destroy によりリソースを解放。

外部依存

  • utils/dsa.h (共有メモリ管理)
  • utils/memutils.h (メモリコンテキスト)
  • storage/lwlock.h (並行制御)
  • port/pg_bitutils.h (ビット操作)
  • nodes/bitmapset.h (集合操作)

内部構成

本モジュールは個別のヘッダーファイルが独立した機能を提供しており、階層的な子モジュールは持ちません。

推奨読書順

1. radixtree.h (高度な適応型構造) 2. ilist.h (基礎的な侵入型リスト) 3. simplehash.h (高性能ハッシュの設計思想) 4. sort_template.h (メタプログラミング手法) 5. dshash.h (共有メモリ対応の並行制御) 6. pairingheap.h / binaryheap.h (ヒープ実装の比較) 7. stringinfo.h (文字列バッファの基本) 8. rbtree.h (標準的な平衡木) 9. qunique.h / bipartite_match.h (アルゴリズム) 10. bloomfilter.h / hyperloglog.h / integerset.h / knapsack.h (特殊用途)
