src/include/storage

システム内での役割

src/include/storage は、PostgreSQL の物理ストレージ管理、バッファキャッシュ、同時実行制御、およびプロセス間通信（IPC）の基盤となるヘッダー群を格納するディレクトリです。本モジュールは、データベースのデータページへのアクセスから、低レベルなロック管理、非同期 I/O まで、PostgreSQL のストレージサブシステム全体を支える定義を提供します。

責務

  • **データアクセス層の抽象化**: ディスクページ（bufpage.h）やブロック（block.h）の物理レイアウトを定義し、バッファマネージャ（bufmgr.h）を介した効率的なキャッシュアクセスを可能にする。
  • **同時実行制御の提供**: 軽量ロック（lwlock.h）、重量ロック（lock.h）、スピンロック（s_lock.h）を通じた共有リソースの保護。
  • **プロセス管理と通信**: バックエンドプロセスの状態管理（proc.h）および共有メモリを介したプロセス間通信（dsm_impl.h, sinval.h）の定義。
  • **I/O サブシステムの管理**: 非同期 I/O（aio.h）およびファイル記述子管理（fd.h）による、スケーラブルなストレージアクセス。

このモジュールのファイル

65個のヘッダーファイルで構成されており、主に以下のカテゴリに分類されます。

  • **バッファ管理**: bufmgr.h, buf_internals.h, bufpage.h, itemid.h, itemptr.h
  • **ロック・同期**: lock.h, lwlock.h, s_lock.h, spin.h, predicate.h
  • **プロセス・IPC**: proc.h, dsm.h, sinval.h, shmem.h, latch.h
  • **I/O・ファイル**: aio.h, fd.h, smgr.h, block.h

主要なデータ構造

  • BufferDesc: 共有バッファのメタデータと状態を管理する構造体。
  • LOCKTAG / LOCK: 重量ロックの対象識別子と共有メモリ上のロックオブジェクト。
  • PGPROC: 各バックエンドプロセスの共有メモリ上の状態構造体。
  • LWLock: キャッシュライン境界を意識した軽量ロック構造体。
  • ItemPointerData: 物理的なタプル位置（TID）を保持する 6 バイト構造体。
  • PgAioHandle: 非同期 I/O 操作のコンテキストを保持するハンドル。

主なフロー

1. **ページアクセス**: 上位層が ReadBuffer を呼び出し、バッファマネージャがハッシュテーブルを検索。キャッシュミス時は smgr を通じてディスクから読み込み、BufferDesc を更新する。 2. **ロック取得**: 共有リソースへのアクセス前に LWLockAcquire や LockAcquire を呼び出し、競合時は PGPROC を待機キューにリンクしてスリープする。 3. **キャッシュ無効化**: システムカタログ等の変更時、SendSharedInvalidMessages で無効化メッセージを共有キューに書き込み、他プロセスが ReceiveSharedInvalidMessages で自身のキャッシュを同期する。

外部依存

  • lib/ilist.h: 共有メモリ上のリスト管理。
  • port/atomics.h: バッファ状態やロック状態のアトミック更新。
  • access/xlogdefs.h: ページヘッダーの LSN 管理。
  • storage/procnumber.h: プロセス識別子管理。

内部構成

本ディレクトリは、PostgreSQL のストレージ層の「定義」を集約したものであり、実装の多くは src/backend/storage/ 配下に存在します。大規模なサブシステム（バッファ、ロック、AIO）が混在しており、ディレクトリ境界によって論理的にグループ化されています。

推奨読書順

1. src/include/storage/bufmgr.h / buf_internals.h（バッファ管理の基礎） 2. src/include/storage/bufpage.h / itemid.h（ページレイアウト） 3. src/include/storage/lwlock.h / lock.h（同時実行制御） 4. src/include/storage/proc.h（プロセス管理） 5. src/include/storage/aio.h（非同期 I/O） 6. src/include/storage/fd.h（ファイル管理）
