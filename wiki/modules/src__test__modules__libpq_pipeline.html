src/test/modules/libpq_pipeline

本モジュールはディレクトリ境界に基づき定義されています。

システム内での役割

PostgreSQL の libpq が提供するパイプラインモード（非同期的なクエリ送信と結果取得）の機能検証を行うためのテストスイートです。libpq のパイプライン API を利用するクライアントプログラムのリファレンス実装としての側面も持ちます。

責務

  • libpq パイプライン API（PQenterPipelineMode 等）の正常系・異常系動作の検証。
  • 非ブロッキング I/O とパイプライン処理の統合動作の確認。
  • クエリキャンセル、プロトコルバージョンネゴシエーション、エラーハンドリング（PGRES_PIPELINE_ABORTED 状態からの復旧）の網羅的なテスト。

このモジュールのファイル

  • src/test/modules/libpq_pipeline/libpq_pipeline.c

主要なデータ構造

  • PipelineInsertStep: test_pipelined_insert における複雑なパイプライン状態遷移を管理するための列挙型。

主なフロー

1. **接続と開始**: PQconnectdb で接続後、PQenterPipelineMode でパイプラインモードへ移行。 2. **クエリ送信**: PQsendQueryParams 等でクエリを送信し、PQflush でバッファをフラッシュ。 3. **結果処理**: PQgetResult をループで呼び出し、PGRES_PIPELINE_SYNC やエラー状態を処理。 4. **非ブロッキング制御**: select を用いてソケットの読み書き準備を判定し、デッドロックを回避しながら結果を消費。 5. **終了**: PQexitPipelineMode でモードを終了し、PQfinish で接続をクローズ。

外部依存

  • libpq-fe.h: 主要な libpq API。
  • postgres_fe.h: フロントエンド共通定義。
  • catalog/pg_type_d.h: OID 定数。
  • pg_getopt.h: 引数解析。
  • sys/select.h, sys/time.h: I/O 多重化用。
