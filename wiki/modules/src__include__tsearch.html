src/include/tsearch

システム内での役割

PostgreSQL の全文検索（Tsearch）サブシステムにおける、データ型定義、解析エンジン、辞書管理、および実行時キャッシュのインターフェースを統合するディレクトリです。

責務

  • 全文検索の基盤となるデータ型（tsvector, tsquery）のメモリレイアウト定義。
  • テキスト解析、辞書処理、およびクエリ実行のための共通 API の提供。
  • ロケール依存の文字処理および設定ファイル読み込みの抽象化。
  • 検索パフォーマンス向上のためのメタデータキャッシュ構造の管理。

このモジュールのファイル

  • ts_cache.h: パーサー・辞書・設定のキャッシュ管理。
  • ts_locale.h: ロケール対応の文字処理およびファイル読み込みユーティリティ。
  • ts_public.h: 全文検索コンポーネント間の公開インターフェース。
  • ts_type.h: tsvector および tsquery のバイナリ構造定義。
  • ts_utils.h: 解析・実行・クエリ操作のための汎用ユーティリティ。

主要なデータ構造

  • **データ型**: TSVectorData, TSQueryData, WordEntry, QueryItem
  • **キャッシュ**: TSParserCacheEntry, TSDictionaryCacheEntry, TSConfigCacheEntry
  • **解析・実行**: TSVectorParseState, TSQueryParserState, QTNode, ExecPhraseData, HeadlineParsedText
  • **ユーティリティ**: tsearch_readline_state, StopList, TSLexeme

主なフロー

1. **解析**: ts_utils.h のパーサー状態管理を用い、テキストをトークン化して tsvector へ変換。 2. **辞書処理**: ts_public.h のインターフェースを介し、単語の正規化やストップワード判定を実行。 3. **検索実行**: TS_execute がクエリツリーを走査し、ts_cache.h で管理されるキャッシュを利用して高速に一致判定を行う。 4. **キャッシュ管理**: システムカタログの更新時に InvalidateTSCacheCallBack を経由してキャッシュを無効化し、整合性を維持。

外部依存

  • fmgr.h (関数呼び出し)
  • nodes/pg_list.h (リスト構造)
  • mb/pg_wchar.h (マルチバイト文字処理)
  • utils/pg_locale.h (ロケール処理)
  • utils/memutils.h (メモリ管理)
  • lib/stringinfo.h (文字列バッファ)

内部構成

本ディレクトリは、PostgreSQL の全文検索機能のヘッダー群を収容する論理的な境界として構成されています。

推奨読書順

1. ts_type.h（データ構造の理解） 2. ts_utils.h（解析・実行フローの把握） 3. ts_public.h（コンポーネント間連携の理解） 4. ts_locale.h（文字処理の抽象化） 5. ts_cache.h（パフォーマンス最適化層）
