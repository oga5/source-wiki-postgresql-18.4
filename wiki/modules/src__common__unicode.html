src/common/unicode

システム内での役割

本モジュールは、PostgreSQL における Unicode 関連機能（大文字・小文字変換、カテゴリ判定、正規化）の正確性を検証するためのテストスイート群です。ディレクトリ境界に基づき構成されており、主に ICU（International Components for Unicode）ライブラリの出力と PostgreSQL の内部実装を比較することで、Unicode 仕様への準拠を保証します。

責務

  • **品質保証**: Unicode バージョン更新や変換ロジック変更時における回帰テストの実行。
  • **整合性検証**: PostgreSQL 内部の変換テーブルおよびプロパティ判定ロジックと、業界標準である ICU との網羅的な照合。
  • **仕様準拠**: Unicode コンソーシアムが提供する正規化テストデータを用いた仕様適合性の確認。

このモジュールのファイル

  • src/common/unicode/case_test.c: 大文字・小文字変換（Case Mapping）の検証。
  • src/common/unicode/category_test.c: Unicode カテゴリおよび文字プロパティの検証。
  • src/common/unicode/norm_test.c: Unicode 正規化（NFC/NFD/NFKC/NFKD）の検証。

主要なデータ構造

  • TestFunc: 変換関数を抽象化し、テスト実行時に呼び出すための構造体。
  • WordBoundaryState: タイトルケース変換時に単語境界を判定するための状態保持構造体。
  • UnicodeNormalizationTests: 正規化テストの期待値と入力値を保持するデータ配列。

主なフロー

1. **初期化**: 各テストプログラムが PostgreSQL 内部の Unicode バージョンと、環境に応じて ICU のバージョンを解析・表示。 2. **網羅的走査**: 全 Unicode コードポイント（0x000000 - 0x10FFFF）をループし、PostgreSQL の関数と ICU の関数を比較（case_test および category_test）。 3. **正規化検証**: 定義済みのテストテーブルをイテレートし、各正規化形式に対して期待値との比較を実行（norm_test）。 4. **検証と終了**: 不一致が検出された場合は詳細なデバッグ情報を出力し、exit(1) で異常終了。全テスト通過時は成功メッセージを出力。

外部依存

  • postgres_fe.h: フロントエンド用共通定義。
  • common/unicode_*.h: PostgreSQL 側の Unicode 実装ヘッダー群。
  • unicode/*.h: ICU ライブラリ（USE_ICU 定義時のみ）。
  • norm_test_table.h: 正規化テスト用のデータセット。

内部構成

本モジュールはスタンドアロンのテストプログラム群で構成されており、サーバー本体の実行時には直接リンクされません。子モジュールは存在せず、各ファイルが独立したテストツールとして機能します。

推奨読書順

1. src/common/unicode/case_test.c（変換ロジックの検証手法を理解する） 2. src/common/unicode/category_test.c（プロパティ判定の網羅的テスト手法を理解する） 3. src/common/unicode/norm_test.c（外部データセットを用いた正規化テストの仕組みを理解する）

未解決の問い

  • 現在はテストプログラムのみが配置されているが、将来的に共通のユーティリティ関数やデータ構造が抽出される可能性がある。
  • ICU が利用できない環境におけるテストカバレッジの限界と、その補完策の検討。
