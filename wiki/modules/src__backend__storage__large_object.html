src/backend/storage/large_object

本モジュールは、ディレクトリ境界に基づき定義されています。PostgreSQLにおけるラージオブジェクト（LO）のストレージ管理およびアクセス層を担います。

システム内での役割

PostgreSQLの「インバージョン（Inversion）」ファイルシステムに基づき、ユーザーレベルのラージオブジェクト操作をバックエンドで実現します。SQLインターフェース（lo_open等）とシステムカタログ（pg_largeobject）を繋ぐブリッジとして機能します。

責務

  • **LOのライフサイクル管理**: 作成、オープン、クローズ、削除の制御。
  • **データアクセス**: pg_largeobject テーブルに対するページ単位の読み書きおよび切り詰め処理。
  • **整合性と権限**: 操作時のACL検証、TOASTデータのデコード、およびページ間の「穴」のゼロ埋め処理。

このモジュールのファイル

  • src/backend/storage/large_object/inv_api.c

主要なデータ構造

  • LargeObjectDesc: オープンされたLOのID、オフセット、フラグ、スナップショット状態を保持する管理構造体。

主なフロー

1. **初期化**: open_lo_relation() により pg_largeobject テーブルおよびインデックスを現在のトランザクションでオープン。 2. **スキャン**: systable_beginscan_ordered を用い、loid と pageno をキーにページを順次取得。 3. **データ処理**: 読み込み時はTOASTデータの展開とゼロ埋め、書き込み時はカタログの更新（挿入/更新）を実行。 4. **終了**: CommandCounterIncrement() により変更を確定し、トランザクション終了時にリソースを解放。

外部依存

  • postgres.h
  • access/ 系（detoast.h, genam.h, htup_details.h, table.h, xact.h）
  • catalog/ 系（dependency.h, indexing.h, objectaccess.h, pg_largeobject.h）

内部構成

  • 子モジュールなし。inv_api.c が単一の主要実装として機能。
