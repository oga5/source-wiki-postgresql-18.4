src/bin/pg_amcheck

システム内での役割

pg_amcheck は、PostgreSQL データベース内のヒープテーブルおよび B-tree 索引の破損を検出するためのコマンドラインツールです。本モジュールはディレクトリ境界に基づき定義されており、contrib/amcheck 拡張機能が提供する検証関数を並列実行するためのフロントエンドとして機能します。

責務

  • ユーザー指定のパターンに基づき、検証対象となるデータベース、スキーマ、リレーションを SQL クエリで動的に特定する。
  • fe_utils を活用し、複数のデータベース接続を用いた並列検証を実行する。
  • ヒープおよび B-tree 用の検証 SQL を構築・発行し、結果の集約と破損状況の報告を行う。

このモジュールのファイル

  • src/bin/pg_amcheck/pg_amcheck.c: プログラムのライフサイクル管理、引数解析、並列実行制御、結果処理を担う唯一のソースファイル。

主要なデータ構造

  • AmcheckOptions: ユーザー指定の実行オプションを保持。
  • RelationInfo: 検証対象リレーションのメタデータ（OID、型等）を管理。
  • PatternInfo / PatternInfoArray: ユーザー指定の検索パターンを保持。
  • DatabaseInfo: 検証対象データベースの情報を保持。

主なフロー

1. **初期化**: コマンドライン引数を解析し AmcheckOptions を設定。 2. **対象特定**: SQL クエリを用いて検証対象のデータベースおよびリレーションのリストを構築。 3. **並列実行**: ParallelSlotsSetup を経て、各リレーションに対する検証 SQL を非同期に発行。 4. **結果処理**: コールバック関数（verify_heap_slot_handler 等）が各スロットの実行結果を解析し、破損情報を出力。 5. **終了**: 全スロットの完了を待ち、終了コードを返却。

外部依存

  • postgres_fe.h: フロントエンド共通ヘッダー。
  • fe_utils/: 並列実行（parallel_slot.h）、SQL 実行（query_utils.h）、引数解析（option_utils.h）等のユーティリティ。
  • catalog/: pg_am_d.h 等のシステムカタログ定義。
