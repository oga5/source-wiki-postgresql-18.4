src/test/modules/oauth_validator/t

システム内での役割

本モジュールは、PostgreSQL の OAuth 2.0 認証機能のテストを支援するモックサーバー環境を提供します。ディレクトリ境界に基づき定義されたテスト用ユーティリティ群です。

責務

  • OAuth 2.0 認証フロー（Discovery, Authorize, Token）のシミュレーション。
  • テストケースに応じた動的なレスポンス制御（エラー注入、レスポンスサイズ変更、リトライ検証）。
  • テスト実行プロセス（Perl）に対するバックエンドサーバーの提供。

このモジュールのファイル

  • src/test/modules/oauth_validator/t/oauth_server.py: OAuth 2.0 モックサーバーの実装。

主要なデータ構造

  • OAuthHandler: http.server.BaseHTTPRequestHandler を継承したリクエスト処理クラス。
  • _TokenState: トークンリクエストのリトライ回数や待機時間を管理する状態保持クラス。

主なフロー

1. **起動**: main() がエフェメラルポートで HTTPServer を起動し、ポート番号を親プロセスへ通知。 2. **リクエスト処理**: do_GET/do_POST がリクエストを捕捉し、_check_issuer() でパスを正規化。 3. **パラメータ解析**: client_id にエンコードされたテスト用パラメータを抽出し、レスポンスの挙動を動的に決定。 4. **検証**: token() メソッド内でクライアントの挙動（リトライ間隔等）を検証し、JSON レスポンスを返却。

外部依存

  • http.server, urllib.parse, base64, json, collections.defaultdict（標準ライブラリのみ）。

内部構成

  • 単一の Python スクリプトで構成されており、Perl 製のテストスイート（OAuth/Server.pm）から制御されることを前提としています。
