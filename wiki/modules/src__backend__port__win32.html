src/backend/port/win32

システム内での役割

本モジュールは、PostgreSQLをWindows環境で動作させるための移植性レイヤー（Porting Layer）です。POSIX準拠のAPIが存在しないWindowsにおいて、Unixライクなシグナル処理、ソケット通信、タイマー機能、およびクラッシュ時の診断機能を提供します。

責務

  • **POSIX APIのエミュレーション**: シグナル（kill, sigaction等）、ソケット操作、タイマー（setitimer）のWin32 APIによる再実装。
  • **Windows固有の統合**: Winsock2やWindowsイベントオブジェクトを用いた、PostgreSQLのメインスレッドとバックグラウンド処理間の同期。
  • **診断と堅牢性**: 未処理例外の捕捉とミニダンプ生成による、Windows環境での障害解析支援。

このモジュールのファイル

  • crashdump.c: 例外ハンドラによるクラッシュダンプ生成。
  • signal.c: 名前付きパイプとイベントを用いたPOSIXシグナルエミュレーション。
  • socket.c: Winsock2を用いたソケット通信の抽象化とシグナル割り込み対応。
  • timer.c: 専用スレッドを用いた setitimer の実装。

主要なデータ構造

  • CallbackParam: クラッシュダンプ生成時のコールバック用パラメータ。
  • timerCA: タイマー設定をメインスレッドとタイマー管理スレッド間で共有する構造体。
  • pg_signal_queue: シグナルスレッドとメインスレッド間で共有されるシグナルビットマスク。

主なフロー

1. **シグナル処理**: 外部からのシグナル要求を名前付きパイプで受信し、pg_signal_queue を更新してイベントオブジェクトをセット。メインスレッドが待機ポイントでこれを検知し、ハンドラをディスパッチする。 2. **ソケット待機**: WaitForMultipleObjectsEx を使用し、ソケットの準備完了とシグナルイベントの双方を監視することで、ブロッキング操作中もシグナル割り込みを可能にする。 3. **タイマー処理**: setitimer 呼び出しによりタイマー管理スレッドが起動。タイムアウト時に SIGALRM をキューイングし、メインスレッドへ通知する。

外部依存

  • postgres.h: 基本的な型定義およびエラー報告機能。
  • libpq/pqsignal.h: シグナル関連の定数および関数宣言。
  • Windows API (windows.h, winsock2.h, dbghelp.h): OSネイティブ機能へのアクセス。

内部構成

本モジュールはディレクトリ境界に基づいて定義されており、各ファイルは独立した移植性機能を提供します。子モジュールは存在しません。

推奨読書順

1. src/backend/port/win32/signal.c（シグナル機構の理解が他機能の基盤となるため） 2. src/backend/port/win32/socket.c 3. src/backend/port/win32/timer.c 4. src/backend/port/win32/crashdump.c
