contrib/uuid-ossp

システム内での役割

PostgreSQL において UUID を生成するための拡張モジュールです。OSSP UUID、BSD UUID、または E2FS UUID ライブラリをラップし、SQL 関数として UUID 生成機能を提供します。本モジュールはディレクトリ境界に基づき定義されています。

責務

  • **抽象化**: コンパイル時に選択された外部 UUID ライブラリの差異を吸収し、統一された SQL インターフェースを提供します。
  • **UUID 生成**: v1（時間ベース）、v3/v5（名前空間ベース）、v4（乱数ベース）の生成をサポートします。
  • **リソース管理**: OSSP ライブラリ使用時にセッション単位で UUID オブジェクトをキャッシュし、パフォーマンスと一貫性を維持します。

このモジュールのファイル

  • contrib/uuid-ossp/uuid-ossp.c

主要なデータ構造

  • dce_uuid_t: BSD/E2FS 環境で UUID の内部構造を扱うための構造体。

主なフロー

1. **初期化**: PG_FUNCTION_INFO_V1 により SQL 関数として登録。 2. **分岐**: コンパイル時の設定に基づき、OSSP ライブラリまたは OS 組み込み関数を選択。 3. **生成**: uuid_generate_internal を経由して UUID を生成。OSSP 使用時はキャッシュされたオブジェクトを利用し、非 OSSP 時は pg_cryptohash や OS の乱数生成器を活用。 4. **変換**: 生成された値を PostgreSQL の Datum 型に変換して SQL レイヤーへ返却。

外部依存

  • postgres.h, fmgr.h (PostgreSQL コア)
  • common/cryptohash.h, common/sha1.h (ハッシュ計算)
  • utils/uuid.h (PostgreSQL 内部 UUID 型)
  • port/pg_bswap.h, varatt.h
  • 外部 UUID ライブラリヘッダー (ossp/uuid.h 等)
