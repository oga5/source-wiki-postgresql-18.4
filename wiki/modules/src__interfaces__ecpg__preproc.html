src/interfaces/ecpg/preproc

システム内での役割

src/interfaces/ecpg/preproc は、PostgreSQL の埋め込み SQL プリプロセッサ（ecpg）の中核を担うモジュールです。C 言語ソースコード内に埋め込まれた SQL 文を解析し、ecpglib が実行可能な C コードへ変換する一連のプロセスを統括します。

責務

  • **構文解析と変換**: C 言語と SQL が混在するソースコードを解析し、SQL 文をランタイムライブラリ呼び出し（ECPGdo 等）に変換する。
  • **型システム管理**: C 言語の変数型と SQL データ型のマッピング、およびホスト変数のスコープ管理。
  • **コード生成**: 変換後の C コード生成、エラー処理（WHENEVER）の注入、およびデバッグ用の #line ディレクティブ管理。
  • **字句解析サポート**: SQL および C 言語のキーワード識別と、パーサーの曖昧さを解消するためのトークンフィルタリング。

このモジュールのファイル

  • **メイン制御**: ecpg.c
  • **解析・変換**: parser.c, output.c, descriptor.c
  • **型・変数管理**: type.c, type.h, variable.c
  • **キーワード定義**: c_keywords.c, c_kwlist.h, ecpg_keywords.c, ecpg_kwlist.h, keywords.c
  • **ユーティリティ**: util.c, preproc_extern.h

主要なデータ構造

  • ECPGtype: ホスト変数の型情報を保持する中心的な構造体。
  • ECPGstruct_member: 構造体や共用体のメンバーを管理する連結リスト。
  • variable: プリプロセッサが追跡するホスト変数の定義。
  • YYLTYPE: パーサーが使用する位置情報追跡用の型。
  • COMPAT_MODE: ターゲットデータベース（Informix, Oracle 等）の互換性モード。

主なフロー

1. **初期化**: ecpg.c がコマンドライン引数を解析し、互換モードやインクルードパスを設定。 2. **解析**: parser.c がスキャナからのトークンをフィルタリングし、base_yyparse() が構文木を構築。 3. **変数解決**: variable.c がスコープに基づき変数を検索・登録し、type.c が型情報を構築。 4. **コード生成**: output.c が解析結果を基に C コードを生成し、descriptor.c が記述子操作を変換。 5. **クリーンアップ**: util.c の reclaim_local_storage を呼び出し、解析サイクルごとのメモリを解放。

外部依存

  • postgres_fe.h: PostgreSQL フロントエンド共通定義。
  • common/keywords.h: SQL キーワード定義。
  • getopt_long.h: CLI 引数解析。

内部構成

本モジュールはディレクトリ境界に基づき構成されており、単一のディレクトリ内でプリプロセッサの全機能を完結させています。

推奨読書順

1. preproc_extern.h（全体像と共有定義の把握） 2. type.h / type.c（型システムの理解） 3. variable.c（変数管理ロジック） 4. parser.c（字句解析フィルタリング） 5. output.c（コード生成の仕組み） 6. ecpg.c（メインループと制御フロー） 7. descriptor.c（記述子操作） 8. util.c（メモリ管理とエラー処理） 9. keywords.c 関連ファイル（キーワード検索の仕組み）
