contrib/isn

システム内での役割

contrib/isn は、PostgreSQL において国際標準番号（ISBN, ISMN, ISSN, EAN13, UPC）をデータベース型として扱うための拡張モジュールです。これらの識別子を内部的に統一された ean13 型（uint64）として管理し、検証およびフォーマット変換機能を提供します。

責務

  • **データ型定義**: 国際標準番号を格納するための共通型 ean13 の提供。
  • **バリデーション**: 各規格固有のチェックデジット計算および範囲テーブルに基づく妥当性検証。
  • **フォーマット変換**: 内部表現とハイフン付きの人間が読みやすい文字列形式との相互変換。
  • **GUC 管理**: チェックデジットが不正な入力を許容するかどうかの制御（isn.weak）。

このモジュールのファイル

  • isn.c: モジュールの中核となる実装。
  • isn.h: 共通の型定義およびマクロ。
  • EAN13.h, ISBN.h, ISMN.h, ISSN.h, UPC.h: 各識別子固有の範囲定義テーブル。

主要なデータ構造

  • ean13: uint64 のエイリアス。ISN データの内部表現であり、最下位ビットはチェックデジットの有効性フラグとして使用されます。
  • _range / _index テーブル: 各ヘッダーファイルで定義される、識別子ごとの有効範囲やインデックス情報。

主なフロー

1. **入力**: string2ean() が文字列を受け取り、規格を特定してチェックデジットを検証し、ean13 型へ変換します。 2. **検証**: 各規格の _range テーブルを参照し、入力値が有効な範囲内にあるかを確認します。 3. **出力**: ean2string() が ean13 型を読み込み、規格に応じたハイフン挿入ルールを適用して文字列化します。

外部依存

  • postgres.h, fmgr.h: PostgreSQL のコア機能および関数マネージャ。
  • utils/guc.h: GUC（Grand Unified Configuration）システムによる設定管理。

内部構成

本モジュールはディレクトリ境界に基づいて構成されており、サブモジュールは持ちません。各ヘッダーファイルは isn.c から参照される静的データ定義として機能します。

推奨読書順

1. contrib/isn/isn.h（型定義とマクロの理解） 2. contrib/isn/isn.c（コアロジックとフローの把握） 3. contrib/isn/ISBN.h（範囲定義の具体例として） 4. その他各規格のヘッダーファイル（必要に応じて）

未解決の問い

  • 現在の範囲テーブル（特に UPC）はプレースホルダー的な状態であるため、最新の仕様への更新が必要か。
  • static const 定義がヘッダーファイルに存在するため、将来的なビルド構成の変更によるリンクエラーのリスクをどう管理するか。
