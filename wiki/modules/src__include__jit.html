src/include/jit

システム内での役割

src/include/jit は、PostgreSQL の JIT（Just-In-Time）コンパイル機能のインターフェースおよび共通定義を管理するディレクトリです。主に LLVM を利用した実行時のコード生成を抽象化し、実行エンジンと JIT プロバイダ間の仲介役を果たします。

責務

  • **プロバイダ抽象化**: 特定の JIT 実装（LLVM 等）に依存しない共通インターフェース（JitProviderCallbacks）の提供。
  • **LLVM 統合**: PostgreSQL の内部データ構造を LLVM IR にマッピングし、コード生成を支援するユーティリティの提供。
  • **メモリ管理**: JIT 生成コードを安全に配置・実行するためのカスタムメモリマネージャの定義。
  • **環境適応**: LLVM バージョンやアーキテクチャ間の差異を吸収する互換レイヤーの提供。

このモジュールのファイル

  • src/include/jit/SectionMemoryManager.h: JIT 生成コード用のカスタムメモリマネージャ。
  • src/include/jit/jit.h: プロバイダ非依存の JIT 共通インターフェース。
  • src/include/jit/llvmjit.h: LLVM JIT の中核となるコンテキスト管理と型定義。
  • src/include/jit/llvmjit_backport.h: LLVM バージョン差異を吸収する互換性定義。
  • src/include/jit/llvmjit_emit.h: LLVM IR 生成を簡略化するヘルパー関数群。

主要なデータ構造

  • JitContext / LLVMJitContext: JIT コンパイルのライフサイクルと状態管理。
  • JitProviderCallbacks: JIT プロバイダが実装すべき関数ポインタ群。
  • JitInstrumentation: JIT 処理のコスト計測用構造体。
  • SectionMemoryManager: LLVM のメモリ管理を拡張するクラス。
  • MemoryGroup / FreeMemBlock: メモリ割り当て管理用構造。

主なフロー

1. **初期化**: jit.c を通じてプロバイダがロードされ、JitProviderCallbacks が設定されます。 2. **コンテキスト生成**: llvm_create_context により LLVMJitContext が初期化されます。 3. **コード生成**: llvm_compile_expr 等が呼び出され、llvmjit_emit.h のヘルパーを用いて LLVM IR が構築されます。 4. **メモリ配置**: SectionMemoryManager がコードをメモリに配置し、finalizeMemory で実行権限を付与します。 5. **実行**: ORC JIT エンジンを介して機械語として実行されます。

外部依存

  • **LLVM C API**: llvm-c/Core.h, llvm-c/Target.h 等。
  • **PostgreSQL コア**: executor/instrument.h, utils/resowner.h, access/tupdesc.h, fmgr.h, nodes/pg_list.h。

内部構成

本モジュールはディレクトリ境界により定義されており、直接的なサブモジュール構造は持ちません。LLVM 関連機能は USE_LLVM マクロによって条件付きで有効化されます。

推奨読書順

1. src/include/jit/jit.h (全体像とインターフェースの理解) 2. src/include/jit/llvmjit.h (LLVM 統合の基盤) 3. src/include/jit/llvmjit_emit.h (IR 生成の実践) 4. src/include/jit/SectionMemoryManager.h (メモリ管理の詳細) 5. src/include/jit/llvmjit_backport.h (環境依存の吸収層)

未解決の問い

  • LLVM 以外の JIT プロバイダが将来的に導入される場合のインターフェース拡張性の検証。
