src/pl/tcl

システム内での役割

PostgreSQL に Tcl 言語による手続き型言語（PL）機能を提供するためのモジュールです。本モジュールはディレクトリ境界に基づいて定義されており、PostgreSQL のバックエンドと Tcl インタプリタを橋渡しする役割を担います。

責務

  • **Tcl インタプリタの管理**: ユーザーごとのインタプリタ生成、キャッシュ、およびライフサイクル管理。
  • **SQL 関数・トリガーの実行**: SQL 関数、トリガー、イベントトリガーの実行ハンドラを提供。
  • **SPI 統合**: Tcl スクリプトから PostgreSQL の SPI（Server Programming Interface）を呼び出し、SQL の実行や準備を可能にする。
  • **データ変換**: PostgreSQL のタプル・データ型と Tcl オブジェクト間の相互変換（UTF-8 エンコーディング対応を含む）。
  • **エラーハンドリング**: Tcl のエラーを PostgreSQL の ereport へ変換し、SQLSTATE を適切に伝播させる。

このモジュールのファイル

  • src/pl/tcl/pltcl.c: Tcl 統合の中心的な実装ファイル。

主要なデータ構造

  • pltcl_interp_desc: ユーザーごとのインタプリタ状態とクエリキャッシュを保持。
  • pltcl_proc_desc: コンパイル済み関数のキャッシュ情報（メモリコンテキスト管理を含む）。
  • pltcl_proc_htab: 関数 OID とユーザー ID をキーとする関数キャッシュハッシュテーブル。
  • pltcl_call_state: 関数呼び出し時の状態管理。
  • Tcl_Size: Tcl API との互換性のための型定義。

主なフロー

1. **初期化**: _PG_init にて Tcl の Notifier サブシステムを PostgreSQL 向けに調整。 2. **呼び出し**: pltcl_call_handler がエントリポイントとなり、現在のユーザーに対応するインタプリタを取得。 3. **コンパイル**: 未キャッシュの関数を Tcl プロシージャとしてコンパイル。 4. **実行**: Tcl_EvalObjEx を介して Tcl コードを実行。SPI 経由の SQL 実行時はサブトランザクションで保護。 5. **変換**: PostgreSQL のタプルと Tcl のデータ形式間で相互変換を実行。

外部依存

  • **PostgreSQL コア**: postgres.h, fmgr.h, executor/spi.h, access/ 関連ヘッダなど。
  • **Tcl ライブラリ**: tcl.h。

内部構成

本モジュールは pltcl.c を中心とした単一の構成であり、サブディレクトリは持ちません。

推奨読書順

1. src/pl/tcl/pltcl.c: モジュールの全機能が実装されているため、ここから読み進めるのが最適です。
