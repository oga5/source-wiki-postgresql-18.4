src/test/modules/plsample

システム内での役割

PostgreSQL の手続き型言語（PL）ハンドラを実装するための最小限の参照実装です。新しい PL を開発する際のテンプレートとして機能し、関数およびトリガー呼び出しの基本的なインターフェースを提供します。本モジュールはディレクトリ境界に基づき定義されています。

責務

  • **ハンドラのディスパッチ**: SQL からの呼び出しを判別し、関数またはトリガーのハンドラへ転送する。
  • **メタデータ取得**: システムカタログ（pg_proc 等）から関数定義や情報を抽出する。
  • **実行コンテキスト管理**: メモリコンテキストの作成や SPI（Server Programming Interface）の接続・切断を管理する。
  • **デバッグ支援**: 実行時の引数やトリガー情報を NOTICE レベルでログ出力する。

このモジュールのファイル

  • src/test/modules/plsample/plsample.c

主要なデータ構造

  • TriggerData: トリガー呼び出し時のイベント情報（INSERT/UPDATE 等）を保持する構造体。
  • FunctionCallInfo: 関数呼び出し時の引数やコンテキスト情報。

主なフロー

1. **エントリ**: plsample_call_handler が呼び出され、呼び出し種別を判定。 2. **メタデータ抽出**: SearchSysCache1 を使用して対象関数の情報を取得。 3. **処理実行**:

  • 関数呼び出し: 引数を解析し、値を変換して出力。
  • トリガー呼び出し: SPI_connect() で SPI を初期化し、トリガーイベントを処理。

    • 4. **終了**: PG_TRY/PG_FINALLY ブロックにより、エラー発生時を含め安全にリソースを解放。
