src/test/modules/test_regex

システム内での役割

PostgreSQL の正規表現エンジン（Spencer's regex）の動作を検証・デバッグするためのテスト用ハーネスです。Tcl の正規表現テストスイートの挙動を模倣し、正規表現のコンパイル結果やマッチングの詳細を SQL から取得可能にします。

責務

  • pg_regcomp および pg_regexec のラップによる正規表現エンジンのテスト実行。
  • テスト用フラグ（g, i, n 等）の解析と適用。
  • マッチング位置やメタ情報を含む詳細な結果を text[] 形式で提供する SRF（Set Returning Function）の実装。

このモジュールのファイル

  • src/test/modules/test_regex/test_regex.c

主要なデータ構造

  • test_regex_ctx: 複数回の SRF 呼び出し間でマッチング状態や位置情報を保持するコンテキスト。
  • test_re_flags: 正規表現のコンパイルおよび実行オプションを格納する構造体。
  • infoname: 正規表現のメタ情報に関連する識別子。

主なフロー

1. **初期化 (SRF_IS_FIRSTCALL)**: フラグの解析、正規表現のコンパイル、対象文字列に対するマッチングの実行と結果のキャッシュ。 2. **反復処理**: test_regex_ctx を参照し、マッチング結果を順次構築して返す。 3. **終了**: 全ての結果を返却後、SRF_RETURN_DONE を発行。

外部依存

  • postgres.h, funcapi.h: PostgreSQL の基本定義および SRF 実装用 API。
  • regex/regex.h: PostgreSQL 正規表現エンジン API。
  • utils/array.h, utils/builtins.h: 配列構築および文字列操作用ユーティリティ。

内部構成

  • 子モジュールなし（単一のソースファイルで構成）。
