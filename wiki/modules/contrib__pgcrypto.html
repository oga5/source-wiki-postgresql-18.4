contrib/pgcrypto

システム内での役割

contrib/pgcrypto は、PostgreSQL データベース内で暗号化、ハッシュ化、および OpenPGP 形式のデータ処理を可能にする拡張モジュールです。SQL インターフェースを通じて、データの保護やセキュアな通信をサポートします。

責務

  • **暗号化・ハッシュ化**: SQL 関数（digest, hmac, encrypt 等）の提供。
  • **PGP 処理**: RFC 4880 に準拠したデータの暗号化・復号、および鍵管理。
  • **パスワードハッシュ**: crypt(3) 互換のパスワードハッシュ生成と検証。
  • **抽象化**: OpenSSL などのバックエンド実装を隠蔽し、一貫した暗号プリミティブインターフェースを提供。

このモジュールのファイル

本モジュールは、暗号化の抽象化層（px.*）、PGP 処理層（pgp-*）、パスワードハッシュ層（crypt-*）、および SQL インターフェース層（pgcrypto.c, pgp-pgsql.c）で構成される 30 個のソースファイル群です。

主要なデータ構造

  • **PX_Cipher, PX_MD, PX_Combo**: 暗号化・ハッシュ処理を抽象化する関数ポインタテーブル。
  • **PGP_Context**: PGP 処理のセッション状態を保持する主要構造体。
  • **MBuf**: ストリーム処理のためのメモリバッファ管理構造体。
  • **PushFilter / PullFilter**: フィルタチェーンによるデータ変換パイプライン。

主なフロー

1. **初期化**: SQL 関数が呼び出されると、px_find_* を通じて適切な暗号アルゴリズムプロバイダーを検索・初期化。 2. **データ処理**: MBuf とフィルタチェーン（PushFilter/PullFilter）を用いて、圧縮・暗号化・MAC 計算などを逐次適用。 3. **終了**: ResourceOwner を介して OpenSSL コンテキスト等のリソースを確実に解放し、機密情報をゼロクリアして終了。

外部依存

  • **OpenSSL**: 暗号化・ハッシュ化の主要バックエンド（EVP API）。
  • **zlib**: PGP データの圧縮・伸長処理。
  • **PostgreSQL Core**: fmgr.h, utils/resowner.h, mb/pg_wchar.h 等のコアライブラリ。

内部構成

  • **抽象化層 (px)**: 暗号アルゴリズムのバックエンドを切り替えるための基盤。
  • **PGP 層 (pgp)**: RFC 4880 に基づくパケット解析、鍵管理、ストリーム処理。
  • **パスワード層 (crypt)**: Blowfish, DES, MD5 等を用いたハッシュ生成。

推奨読書順

1. contrib/pgcrypto/px.h (抽象化の基盤) 2. contrib/pgcrypto/pgp.h (PGP 処理の定義) 3. contrib/pgcrypto/mbuf.h (ストリーム処理の基盤) 4. contrib/pgcrypto/pgcrypto.c (SQL インターフェースの起点) 5. contrib/pgcrypto/openssl.c (バックエンド実装例) 6. contrib/pgcrypto/pgp-encrypt.c / pgp-decrypt.c (PGP パイプライン)
