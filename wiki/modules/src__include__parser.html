src/include/parser

システム内での役割

src/include/parser は、PostgreSQL の SQL 解析パイプラインにおける最上流のサブシステムです。SQL 文字列を字句解析および構文解析して未解析の構文木（Raw Parse Tree）を生成し、さらに意味解析（Semantic Analysis）を通じて実行可能なクエリツリー（Query 構造体）へと変換する一連の処理を統括します。

責務

  • **字句・構文解析**: SQL 文字列をトークン化し、文法定義に基づいた構文木を構築する。
  • **意味解析**: 構文木に対して型チェック、名前解決、演算子・関数解決を行い、実行可能なクエリツリーを生成する。
  • **解析状態の管理**: 解析プロセス全体で共有されるコンテキスト（ParseState）の保持と、名前空間の管理。
  • **型変換の制御**: SQL の型システムに基づいた暗黙的・明示的な型強制（Coercion）の判定とノード構築。

このモジュールのファイル

本ディレクトリには、解析の各フェーズ（スキャン、構文解析、意味解析）を担う 22 個のヘッダーファイルが含まれます。

  • parser.h, scanner.h, scansup.h: 字句解析および Raw パーサーのインターフェース。
  • analyze.h, parsetree.h: 意味解析のメインエントリーとパースツリー操作。
  • parse_*.h: 各 SQL 句（FROM, WHERE, GROUP BY 等）や要素（式、型、関数、演算子、CTE 等）ごとの意味解析ロジック。

主要なデータ構造

  • ParseState: 解析中のクエリ状態（名前空間、範囲テーブル、パラメータ等）を保持する中心的なコンテキスト。
  • Query: 意味解析の結果として生成される、実行エンジンが扱うクエリツリーのルート構造体。
  • RawStmt: Raw パーサーが生成する未解析の構文木ノード。
  • ParseExprKind: 式が使用されるコンテキスト（WHERE 句、SELECT リスト等）を識別し、構文制限を適用するための列挙型。
  • CoercionPathType: 型変換の手法（関数呼び出し、バイナリ互換等）を分類する列挙型。

主なフロー

1. **Raw Parsing**: raw_parser() が SQL 文字列を List 形式の RawStmt に変換。 2. **Analysis**: parse_analyze() が RawStmt を受け取り、ParseState を初期化。 3. **Transformation**: transformStmt() が再帰的に各句（transformFromClause, transformTargetList 等）を処理。 4. **Resolution**: 式の解析過程で parse_func.h や parse_oper.h を呼び出し、システムカタログを参照して関数や演算子を特定。 5. **Collation**: 解析の最終段階で assign_query_collations() が照合順序を確定。

外部依存

  • nodes/parsenodes.h: 解析結果として生成されるノード構造の定義。
  • catalog/: システムカタログ（pg_type, pg_proc, pg_operator 等）へのアクセス。
  • utils/: リレーションキャッシュやクエリ環境管理ユーティリティ。
  • common/keywords.h: SQL キーワードリスト。

内部構成

本モジュールはディレクトリ境界によって定義されており、src/backend/parser/ 配下の実装と密接に連携しています。各ヘッダーは特定の解析ドメイン（型、式、句など）ごとに責務が分離されています。

推奨読書順

1. src/include/parser/parser.h (解析の入り口) 2. src/include/parser/parse_node.h (解析状態の基盤) 3. src/include/parser/analyze.h (意味解析の全体像) 4. src/include/parser/parse_expr.h (式解析の核心) 5. src/include/parser/parse_relation.h (名前解決とリレーション) 6. src/include/parser/parse_coerce.h (型変換ロジック) 7. src/include/parser/scanner.h (字句解析の基礎)
