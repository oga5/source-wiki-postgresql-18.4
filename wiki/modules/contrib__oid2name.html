contrib/oid2name

システム内での役割

contrib/oid2name は、PostgreSQL の物理ファイルと論理的なデータベースオブジェクト（テーブル、インデックス、シーケンス等）を相互にマッピングするためのコマンドラインユーティリティです。本モジュールはディレクトリ境界に基づき定義されています。

責務

  • データベースオブジェクトの OID、ファイルノード番号、および名前の相互変換。
  • libpq を利用したデータベース接続とシステムカタログ（pg_class 等）の照会。
  • 検索条件に応じた動的な SQL クエリの生成と、結果の整形出力。

このモジュールのファイル

  • contrib/oid2name/oid2name.c

主要なデータ構造

  • options: 実行時のオプションや接続パラメータを保持する構造体。
  • eary: コマンドラインで指定された複数の検索対象を管理し、SQL の IN 句へ変換するための動的配列構造体。

主なフロー

1. **初期化**: main 関数にて options を初期化し、引数を解析。 2. **接続**: sql_conn を介して libpq でデータベースへ接続。 3. **クエリ実行**: ユーザー指定に基づき sql_exec_searchtables 等で適切な SQL を構築・実行。 4. **出力**: sql_exec が結果セットの列幅を計算し、表形式で標準出力へ表示。 5. **終了**: PQfinish で接続を解放。

外部依存

  • libpq-fe.h (データベース接続)
  • postgres_fe.h (フロントエンド共通定義)
  • getopt_long.h / pg_getopt.h (引数解析)
  • catalog/pg_class_d.h (システムカタログ定数)
  • common/ 配下のログ・接続ユーティリティ
