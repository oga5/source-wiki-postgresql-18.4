src/pl/plperl

システム内での役割

src/pl/plperl は、PostgreSQL に Perl 言語による手続き型言語機能（PL/Perl）を提供するモジュールです。本モジュールはディレクトリ境界に基づき定義されています。

責務

  • PostgreSQL の関数ハンドラとして、Perl インタプリタのライフサイクル管理とサンドボックス化を行う。
  • PostgreSQL のデータ型と Perl のデータ型（SV/AV/HV）間の双方向変換を仲介する。
  • SPI (Server Programming Interface) を介したデータベース操作を Perl 空間へ公開する。
  • Perl のバージョン間差異を吸収し、安定した実行環境を維持する。

このモジュールのファイル

  • src/pl/plperl/plperl.c: コア実装（インタプリタ管理、型変換、SPI 連携）。
  • src/pl/plperl/plperl.h: 共通インターフェースと型変換ユーティリティ。
  • src/pl/plperl/plperl_system.h: Perl システムヘッダーの安全なインクルードと名前空間保護。
  • src/pl/plperl/ppport.h: Perl API の移植性レイヤー（自動生成）。

主要なデータ構造

  • plperl_proc_desc: コンパイル済み関数のメタデータキャッシュ。
  • plperl_interp_desc: Perl インタプリタのコンテキスト管理。
  • plperl_call_data: 実行中のコールスタックとコンテキスト情報。
  • pTHX / pTHX_: Perl インタプリタのスレッドコンテキスト。

主なフロー

1. **初期化**: _PG_init により GUC 設定と初期インタプリタの準備を行う。 2. **関数実行**: plperl.c が関数呼び出しをハンドリングし、必要に応じてインタプリタをキャッシュから取得・初期化する。 3. **データ変換**: plperl.h の関数群を用い、PostgreSQL の Datum を Perl の SV へ（またはその逆）変換する。 4. **SPI 実行**: Perl からの SQL 実行要求を SPI を介して PostgreSQL エンジンへ委譲し、結果を Perl オブジェクトとして返す。

外部依存

  • Perl インタプリタ（perl.h, EXTERN.h, XSUB.h）
  • PostgreSQL コア（postgres.h, executor/spi.h, catalog/pg_proc.h 等）
  • ロケールおよびマルチバイト処理（locale.h, mb/pg_wchar.h）

内部構成

本モジュールは単一のディレクトリで構成され、サブモジュールは持ちません。

推奨読書順

1. src/pl/plperl/ppport.h（互換性レイヤーの理解） 2. src/pl/plperl/plperl_system.h（環境構築のハック） 3. src/pl/plperl/plperl.h（データ変換インターフェース） 4. src/pl/plperl/plperl.c（メインロジック）
