src/bin/pg_basebackup

システム内での役割

本モジュールは、PostgreSQL の物理バックアップおよびレプリケーション運用を支えるクライアントツール群（pg_basebackup, pg_receivewal, pg_recvlogical, pg_createsubscriber）の集合体です。サーバーからのデータストリーミング、WAL の永続化、およびバックアップ後のリカバリ設定注入を担う中心的なサブシステムです。

責務

  • **物理バックアップ**: サーバーからのデータディレクトリおよび WAL のストリーミング受信とアーカイブ生成。
  • **WAL ストリーミング**: リアルタイムな WAL 受信と、ディレクトリまたは tar 形式での永続化。
  • **論理レプリケーション支援**: 論理デコーディングストリームの受信および物理スタンバイからサブスクライバーへの変換。
  • **共通基盤の提供**: サーバー接続管理、レプリケーションプロトコル制御、および圧縮・同期処理の抽象化。

このモジュールのファイル

  • **メインツール**: pg_basebackup.c, pg_receivewal.c, pg_recvlogical.c, pg_createsubscriber.c
  • **ストリーミング制御**: receivelog.c, receivelog.h
  • **書き込み抽象化**: walmethods.c, walmethods.h
  • **ユーティリティ**: streamutil.c, streamutil.h
  • **アーカイブ処理**: astreamer_inject.c, astreamer_inject.h

主要なデータ構造

  • StreamCtl: WAL ストリーミングの動作を制御するメイン構造体。
  • WalWriteMethod: WAL の保存先（ディレクトリ/tar）を抽象化するインターフェース。
  • ArchiveStreamState: バックアップストリームの処理状態を保持。
  • LogicalRepInfo: 論理レプリケーション移行時の設定情報を管理。
  • astreamer_recovery_injector: リカバリ設定をストリームに動的注入するための状態保持構造体。

主なフロー

1. **接続確立**: streamutil を介してサーバーとレプリケーション接続を確立。 2. **コマンド発行**: BASE_BACKUP や START_REPLICATION 等のプロトコルコマンドを送信。 3. **ストリーム処理**: receivelog が COPY プロトコルでデータを受信し、astreamer パイプラインや walmethods を通じてファイルシステムへ永続化。 4. **終了処理**: 停止条件（LSN 到達やシグナル）に基づき、サーバーへ終了通知を行いリソースを解放。

外部依存

  • libpq-fe.h: サーバーとの通信基盤。
  • access/xlogdefs.h: WAL 関連の基本型定義。
  • zlib.h / lz4frame.h: 圧縮処理。
  • fe_utils/astreamer.h: アーカイブストリーム処理フレームワーク。

内部構成

本モジュールはディレクトリ境界に基づき構成されており、各ツールが共通のストリーミング・接続基盤を共有するフラットな構造です。

推奨読書順

1. walmethods.h (書き込み抽象化の理解) 2. receivelog.h (ストリーミング制御の理解) 3. streamutil.h (接続・共通基盤の理解) 4. astreamer_inject.h (リカバリ設定注入の理解) 5. pg_basebackup.c (メインオーケストレーター) 6. pg_receivewal.c / pg_recvlogical.c / pg_createsubscriber.c (各ツールの詳細)
