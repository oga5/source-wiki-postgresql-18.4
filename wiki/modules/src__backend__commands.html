src/backend/commands

システム内での役割

src/backend/commands は、PostgreSQL における SQL コマンド（DDL および一部のユーティリティ）の実行エンジンを格納する中核モジュールです。パーサーが生成した構文木を受け取り、システムカタログの更新、物理ストレージの操作、およびトランザクションの整合性管理を統括します。

責務

  • **DDL コマンドのディスパッチ**: CREATE, ALTER, DROP, TRUNCATE 等のコマンドを適切な処理関数へ振り分ける。
  • **オブジェクトライフサイクル管理**: テーブル、インデックス、型、データベース、拡張機能などの作成から削除までの状態遷移を制御する。
  • **カタログとストレージの同期**: システムカタログ（pg_class 等）の更新と、物理的なファイル操作（smgr）の整合性を保証する。
  • **メンテナンス処理の統合**: VACUUM, ANALYZE, EXPLAIN 等の観測・保守コマンドの実行基盤を提供する。

このモジュールのファイル

本モジュールは 52 個のソースファイルで構成されています。主要なファイルは以下の通りです：

  • tablecmds.c: テーブル、インデックス、ビュー等の構造変更（DDL）の中核。
  • trigger.c: トリガーの定義と実行時ディスパッチ。
  • async.c: LISTEN/NOTIFY システムの実装。
  • explain.c: クエリ実行計画の可視化。
  • extension.c: 拡張機能のインストールと管理。
  • vacuum.c / analyze.c: メンテナンスおよび統計情報収集の基盤。
  • copyfrom.c / copyto.c: COPY コマンドによるデータ入出力。

主要なデータ構造

  • AlteredTableInfo: ALTER TABLE 実行時の変更状態を保持。
  • CopyToStateData / CopyFromState: COPY コマンド実行中の状態管理。
  • ExplainState: EXPLAIN 出力時のフォーマットや状態を保持。
  • AsyncQueueControl: 非同期通知システムの共有メモリ制御構造体。
  • VacuumParams: VACUUM コマンドのオプション設定。

主なフロー

1. **コマンド受信**: ProcessUtility を経由して各コマンドのメイン関数（例: DefineRelation, CreateTrigger）が呼び出される。 2. **検証とロック**: 対象オブジェクトの権限確認と適切なロック（AccessExclusiveLock 等）の取得。 3. **カタログ更新**: catalog/ 配下の関数を呼び出し、システムカタログを更新。 4. **物理操作**: storage/ 配下の関数を呼び出し、実際のファイルやインデックスの再構築を実行。 5. **キャッシュ無効化**: CacheInvalidateRelcache 等を呼び出し、全バックエンドに定義変更を伝播。

外部依存

  • catalog/: システムカタログの定義と操作。
  • storage/: ロック管理（lmgr）、バッファ管理、物理ファイル操作。
  • access/: テーブルアクセス（tableam）、トランザクション管理（xact）。
  • utils/: メモリ管理、システムキャッシュ、GUC パラメータ。

内部構成

本モジュールはディレクトリ境界に基づいた集合体であり、機能ごとに高度にモジュール化されています。

  • **DDL 実行系**: tablecmds.c, indexcmds.c, typecmds.c, dbcommands.c
  • **データ移動系**: copyfrom.c, copyto.c, copyfromparse.c
  • **メンテナンス・観測系**: vacuum.c, analyze.c, explain.c
  • **イベント・通知系**: trigger.c, async.c, event_trigger.c

推奨読書順

1. tablecmds.c (DDL の基本構造を理解するため) 2. trigger.c (イベント駆動処理の理解) 3. vacuum.c (メンテナンス処理の基盤) 4. copyfrom.c / copyto.c (データ入出力の仕組み) 5. explain.c (クエリ実行の観測) 6. extension.c (拡張機能の管理)
