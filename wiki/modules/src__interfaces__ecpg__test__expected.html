システム内での役割

本モジュールは、PostgreSQL の ECPG（Embedded SQL for C）プリプロセッサの回帰テストスイートにおける「期待値（expected output）」を格納するディレクトリです。.pgc ファイルからプリプロセッサによって生成された C ソースコード群であり、ECPG の各機能が正しく C コードへ変換され、実行時に期待通りの動作をすることを保証するリファレンスとして機能します。

責務

  • ECPG プリプロセッサの変換ロジックの正当性を検証するための期待値の提供。
  • 動的 SQL、SQLDA、カーソル操作、マルチスレッド環境、および各種データ型（bytea, JSON 等）の動作検証。
  • プリプロセッサが生成する C コードの構造と、ECPG ランタイムライブラリ（ecpglib）とのインターフェースの整合性維持。

このモジュールのファイル

本ディレクトリには 68 個の C ソースファイルが含まれており、主に以下のカテゴリに分類されます。

  • **動的 SQL / 記述子**: sql-sqlda.c, sql-dynalloc.c, sql-dyntest.c 等
  • **プリプロセッサ機能**: preproc-variable.c, preproc-cursor.c, preproc-type.c 等
  • **互換性テスト**: compat_informix-sqlda.c, compat_oracle-char_array.c 等
  • **マルチスレッド**: thread-alloc.c, thread-prep.c, thread-descriptor.c 等
  • **データ型**: sql-bytea.c, sql-sqljson.c, sql-array.c 等

主要なデータ構造

  • sqlca_t: SQL 通信領域（SQLCA）。実行結果やエラーコードを保持。
  • sqlda_t / sqlvar_t: SQLDA（SQL Descriptor Area）の構造体。動的 SQL の列情報やバインドデータを管理。
  • customer / personal_struct: テスト用のカスタム構造体。ホスト変数としてのマッピングを検証。
  • varchar: ECPG で定義される可変長文字列型。

主なフロー

1. **初期化**: ECPGconnect によるデータベース接続とテスト用テーブルの作成。 2. **SQL 実行**: ECPGdo を中心とした SQL 文の送信と結果のバインド。 3. **データ検証**: 取得したデータを printf 等で標準出力し、期待値との比較を実行。 4. **終了処理**: ECPGdisconnect による接続終了とリソース（メモリ、記述子）の解放。 ※ 各 SQL 操作の直後には、sqlca.sqlcode をチェックするエラーハンドリングが定型的に挿入されています。

外部依存

  • ecpglib.h: ECPG ランタイムライブラリのコア機能。
  • sqlca.h: SQLCA 構造体の定義。
  • sqlda-compat.h / sqlda-native.h: 記述子構造体の定義（モード依存）。
  • pgtypes.h: 数値や日付型などの PostgreSQL データ型定義。

内部構成

本モジュールはディレクトリ境界によって定義されており、サブモジュールは持ちません。各ファイルは独立したテストケースとして機能し、プリプロセッサの特定の変換ルールを検証します。

推奨読書順

1. compat_informix-sqlda.c（動的 SQL の基本） 2. sql-sqlda.c（標準的な SQLDA テスト） 3. preproc-variable.c（ホスト変数とスコープ） 4. preproc-array_of_struct.c（構造体マッピング） 5. thread-alloc.c（マルチスレッド環境のメモリ管理） 6. sql-bytea.c（バイナリデータの取り扱い）
