src/interfaces/ecpg/compatlib

システム内での役割

本モジュールは、ECPG（Embedded SQL for C）において Informix データベース製品との互換性を提供するためのライブラリです。ECPG で生成されたコードが Informix 固有の API を呼び出す際、その要求を PostgreSQL の pgtypes ライブラリへ変換・転送する仲介役を担います。

責務

  • **Informix 互換 API の提供**: decimal 型の演算、日付・時刻操作、および sqlca（SQL Communication Area）の管理。
  • **型変換の抽象化**: Informix 固有のデータ型と PostgreSQL 内部表現（numeric 等）の相互変換。
  • **エラーマッピング**: pgtypes ライブラリから返されるエラーを、Informix 互換の ECPG_INFORMIX_* エラーコードへ変換して伝播させる。

このモジュールのファイル

  • src/interfaces/ecpg/compatlib/informix.c: Informix 互換関数群の実装。

主要なデータ構造

  • sqlca_t: Informix 互換の SQL 通信領域構造体。
  • decimal: Informix 固有の数値型（内部的には pgtypes の numeric 型へ変換して処理）。

主なフロー

1. **演算フロー**: decadd 等の関数が呼び出されると、内部ラッパー（deccall2/deccall3）が decimal を numeric に変換し、PGTYPES ライブラリの関数を実行後、結果を decimal に戻して返却します。 2. **変換フロー**: 文字列や数値から decimal への変換は、PGTYPESnumeric_from_asc 等を用いて解析・格納されます。 3. **エラーハンドリング**: 各関数は errno を監視し、Informix 互換のエラーコードへマッピングします。

外部依存

  • postgres_fe.h
  • pgtypes_*.h（numeric, date, timestamp 等の変換ライブラリ）
  • ecpg_informix.h, sqlca.h, sqltypes.h
  • 標準 C ライブラリ（math.h, ctype.h 等）

内部構成

本モジュールはディレクトリ境界に基づいて定義されており、informix.c が唯一のソースファイルです。

推奨読書順

1. src/interfaces/ecpg/compatlib/informix.c
