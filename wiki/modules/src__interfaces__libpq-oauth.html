src/interfaces/libpq-oauth

システム内での役割

libpq-oauth は、libpq における OAuth 2.0 Device Authorization Grant (RFC 8628) を実現するための拡張モジュールです。libpq 本体と疎結合を保ちつつ、認証フローに必要な HTTP 通信やステート管理を非同期に実行します。

責務

  • **OAuth 認証フローの実行**: Discovery からトークン取得までのステートマシン管理。
  • **非同期 I/O 統合**: libcurl を利用した通信を libpq のポーリングループに統合。
  • **疎結合な連携**: libpq 内部構造への直接依存を避け、実行時に注入される関数ポインタを介した安全なアクセス層（シム）の提供。

このモジュールのファイル

  • oauth-curl.c / oauth-curl.h: OAuth 認証のメインロジックおよび公開 API。
  • oauth-utils.c / oauth-utils.h: libpq との通信用ブリッジおよびユーティリティ。

主要なデータ構造

  • **ステート管理**: OAuthStep (フローの現在地), async_ctx (認証コンテキスト)。
  • **OAuth エンティティ**: provider, device_authz, token。
  • **ユーティリティ**: PGTernaryBool (状態管理用), json_field (JSON 解析用)。

主なフロー

1. **初期化**: libpq が libpq_oauth_init() を呼び出し、関数ポインタを注入してモジュールを有効化。 2. **認証駆動**: pg_fe_run_oauth_flow() がポーリングループ内で呼び出され、libcurl を用いて認可サーバーと通信。 3. **イベント処理**: epoll/kqueue を用いて非同期 I/O を多重化し、ステート遷移を制御。 4. **終了処理**: pg_fe_cleanup_oauth_flow() により、認証完了またはエラー時にリソースを解放。

外部依存

  • libcurl: HTTP 通信エンジン。
  • libpq 関連: libpq-fe.h, fe-auth-oauth.h 等。
  • システム: sys/epoll.h / sys/event.h (I/O 多重化), common/jsonapi.h (JSON 解析)。

内部構成

本モジュールはディレクトリ境界により定義されており、libpq の動的ロード可能な拡張として機能します。

推奨読書順

1. oauth-utils.h: libpq とのインターフェース定義を理解する。 2. oauth-curl.h: 公開 API を確認する。 3. oauth-curl.c: ステートマシンと非同期 I/O の実装を追う。 4. oauth-utils.c: 注入された関数ポインタによるブリッジ実装を確認する。
