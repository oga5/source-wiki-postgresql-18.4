src/include

システム内での役割

src/include は PostgreSQL の全ソースコードが参照する中心的なヘッダーディレクトリです。プラットフォーム間の差異を吸収し、バックエンドおよびフロントエンドで共通の型定義、マクロ、およびインターフェースを提供することで、システム全体の整合性と移植性を担保します。

責務

  • **環境抽象化**: OS やコンパイラの違いを吸収し、標準的な開発環境を定義する。
  • **型定義の統一**: Datum や Oid など、PostgreSQL のデータモデルの基盤となる型を定義する。
  • **API の提供**: 関数マネージャ（fmgr）、ウィンドウ関数、統計システム、可変長データ（TOAST）操作などの主要サブシステムのインターフェースを公開する。
  • **ビルド設定の管理**: システムの物理的制限やデバッグフラグなどの静的設定を定義する。

このモジュールのファイル

  • c.h, fmgr.h, funcapi.h, getopt_long.h, miscadmin.h, pg_config_manual.h, pg_getopt.h, pg_trace.h, pgstat.h, pgtar.h, pgtime.h, port.h, postgres.h, postgres_ext.h, postgres_fe.h, varatt.h, windowapi.h

主要なデータ構造

  • **基本型**: Datum, Oid, pg_time_t, Pointer
  • **整数型**: int8〜int64, uint8〜uint64
  • **関数・実行管理**: FunctionCallInfo, FmgrInfo, FuncCallContext, WindowObject
  • **データ構造**: varlena（可変長データ）、PgStat_Counter（統計カウンタ）

主なフロー

本モジュールはヘッダーファイルの集合体であり、実行フローを持ちません。各ソースファイルは postgres.h（バックエンド）または postgres_fe.h（フロントエンド）を起点として本ディレクトリのヘッダーを読み込み、コンパイル時の定義注入やインライン関数による型変換を行います。

外部依存

  • pg_config.h 関連（環境設定）
  • 標準 C ライブラリ（stdint.h, stdio.h, string.h 等）
  • src/port/（プラットフォーム互換実装）
  • src/backend/ 下の各サブシステム（access, executor, utils 等）

内部構成

ディレクトリ境界に基づき、32個のサブモジュール（ディレクトリ）と連携するヘッダー群で構成されています。

推奨読書順

1. c.h（最上位の環境定義） 2. postgres.h / postgres_fe.h（バックエンド/フロントエンドの基盤） 3. fmgr.h（関数呼び出し規約） 4. varatt.h（データレイアウト） 5. miscadmin.h（プロセス管理） 6. port.h（移植性レイヤー） 7. pgstat.h（統計システム） 8. funcapi.h / windowapi.h（拡張 API） 9. pg_config_manual.h（静的設定） 10. postgres_ext.h（外部公開型）
