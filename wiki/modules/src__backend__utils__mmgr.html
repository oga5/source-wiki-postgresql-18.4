src/backend/utils/mmgr

システム内での役割

src/backend/utils/mmgr は、PostgreSQL のメモリ管理サブシステムの中枢です。プロセスローカルなメモリコンテキスト（MemoryContext）の階層管理から、共有メモリ領域（DSA）の管理まで、PostgreSQL の実行に必要なあらゆるメモリ割り当て戦略を抽象化・提供します。

責務

  • **メモリコンテキストの抽象化**: mcxt.c を中心に、用途に応じた多様な割り当て戦略（aset, slab, generation, bump）を統一インターフェースで提供。
  • **共有メモリ管理**: プロセス間共有が可能な動的共有メモリ（DSA）およびページ単位の管理（freepage.c）の提供。
  • **ライフサイクル管理**: コンテキストの親子関係に基づく再帰的なメモリ解放と、ポータル等のリソース管理。
  • **デバッグと安全性**: メモリ破壊検知、Valgrind 連携、およびクリティカルセクション内での割り当て制限の強制。

このモジュールのファイル

  • **基盤**: mcxt.c (ディスパッチャー), memdebug.c (デバッグ支援)
  • **コンテキスト実装**: aset.c (汎用), slab.c (固定サイズ), generation.c (世代別), bump.c (短命・高速)
  • **共有メモリ・低レベル**: dsa.c (共有ヒープ), freepage.c (ページ管理), alignedalloc.c (アライメント調整)
  • **リソース管理**: portalmem.c (ポータル管理)

主要なデータ構造

  • MemoryContext / MemoryContextMethods: 抽象化されたメモリコンテキストの基盤。
  • AllocSetContext / SlabContext / GenerationContext / BumpContext: 各アロケータ固有の管理構造体。
  • dsa_area / FreePageManager: 共有メモリおよびページ管理のための制御構造体。
  • MemoryChunk: 割り当てられたメモリの直前に配置されるメタデータヘッダー。

主なフロー

1. **割り当て**: palloc() 等の API が CurrentMemoryContext を参照し、mcxt_methods テーブル経由で各アロケータの alloc メソッドを呼び出す。 2. **解放**: ポインタから MemoryChunk ヘッダーを特定し、コンテキストを介さずに適切な解放処理（pfree）を実行する。 3. **階層管理**: MemoryContextDelete() が呼び出されると、ツリー構造をトラバースし、子コンテキストを含めて再帰的にメモリを解放する。 4. **共有メモリ**: dsa_allocate_extended() が freepage を利用して共有領域からメモリを確保し、プロセス間で共有可能な擬似ポインタを返す。

外部依存

  • storage/dsm.h, storage/lwlock.h: 共有メモリとロック制御。
  • utils/memutils.h, utils/memutils_internal.h: メモリ管理の公開・内部 API。
  • port/atomics.h, port/pg_bitutils.h: 低レベルな同期とビット操作。
  • lib/ilist.h: 内部管理用の双方向リスト。

内部構成

本モジュールは、mcxt.c が提供する抽象化層の下に、複数の独立したアロケータ実装がプラグインとして存在する構成です。ディレクトリ境界により、メモリ管理に関連するすべての実装がこの一箇所に集約されています。

推奨読書順

1. mcxt.c (全体像とディスパッチの理解) 2. aset.c (標準的なメモリ管理の理解) 3. dsa.c (共有メモリ管理の理解) 4. freepage.c (低レベルページ管理の理解) 5. slab.c / generation.c / bump.c (用途別アロケータの比較) 6. portalmem.c (リソース管理への応用) 7. alignedalloc.c / memdebug.c (補助機能とデバッグ)

未解決の問い

  • dsa と MemoryContext の統合的なライフサイクル管理における、より詳細なエラーハンドリングの精緻化。
