contrib/tablefunc

システム内での役割

本モジュールは、標準 SQL では記述が困難なデータ操作を可能にする PostgreSQL 拡張機能です。主にクロス集計（Crosstab）や階層データの探索（Connectby）といった、セットを返す関数（SRF）を提供します。本ページはディレクトリ境界に基づき構成されています。

責務

  • **クロス集計**: 縦持ちデータを横持ちに変換する crosstab および crosstab_hash の提供。
  • **階層探索**: 親子関係を持つテーブルから再帰的にツリー構造を抽出する connectby の提供。
  • **統計関数**: ガウス分布に従う乱数を生成する normal_rand の提供。
  • **SPI 活用**: Server Programming Interface (SPI) を用いた動的な SQL 実行と結果のタプル変換。

このモジュールのファイル

  • contrib/tablefunc/tablefunc.c

主要なデータ構造

  • normal_rand_fctx: normal_rand 用の関数コンテキスト。
  • crosstab_cat_desc: クロス集計時のカテゴリ記述子。
  • crosstab_HashEnt: カテゴリ名と属性インデックスをマッピングするハッシュテーブルエントリ。

主なフロー

1. **初期化**: SRF_FIRSTCALL_INIT および SPI_connect により実行コンテキストと SPI 接続を確立。 2. **データ取得**: SPI_execute を介してソースとなる SQL を実行。 3. **変換処理**: SPI の結果セットをループし、Tuplestore を用いて目的の形式（クロス集計や階層構造）へ再構築。 4. **出力**: tuplestore_puttuple により構築したタプルを蓄積し、呼び出し元へ返却。

外部依存

  • postgres.h, fmgr.h, funcapi.h（基本機能）
  • executor/spi.h（内部 SQL 実行）
  • access/htup_details.h（タプル操作）
  • common/pg_prng.h（乱数生成）
  • catalog/pg_type.h, lib/stringinfo.h, miscadmin.h, utils/builtins.h
