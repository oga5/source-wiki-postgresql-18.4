src/interfaces/ecpg/ecpglib

システム内での役割

ecpglib は、ECPG（Embedded SQL for C）プリプロセッサによって生成された C コードと、PostgreSQL バックエンド（libpq）を仲介するランタイムライブラリです。SQL 文の実行、データ型変換、接続管理、およびエラーハンドリングを抽象化し、C 言語アプリケーションからデータベースを操作するための基盤を提供します。

責務

  • **SQL 実行ライフサイクル管理**: SQL 文の準備、パラメータバインド、実行、および結果の取得・格納。
  • **データ型変換**: C 言語のホスト変数と PostgreSQL のデータ型（OID）間の相互変換。
  • **接続と状態の管理**: データベース接続の維持、スレッドセーフな状態管理（TSD）、およびトランザクション制御。
  • **エラーハンドリング**: サーバーからのエラーやライブラリ内部の異常を sqlca（SQL Communication Area）へ反映。
  • **互換性サポート**: Informix や Oracle モードなど、異なる SQL 方言や互換性要件の吸収。

このモジュールのファイル

  • connect.c: 接続のライフサイクルとスレッドセーフな接続管理。
  • data.c: PGresult からホスト変数へのデータ変換・パース。
  • descriptor.c: SQL 標準記述子（Descriptor）の管理と操作。
  • ecpglib_extern.h: 内部インターフェース、構造体、関数プロトタイプのハブ。
  • error.c: エラー情報の集約と sqlca へのマッピング。
  • execute.c: SQL 実行のメインエンジンとパラメータ構築。
  • memory.c: スレッドローカルな自動メモリ管理機能。
  • misc.c: ユーティリティ、デバッグログ、スレッド初期化。
  • prepare.c: プリペアドステートメントのキャッシュとライフサイクル。
  • sqlda.c: SQLDA 構造体の構築とデータ配置。
  • typename.c: C 型と PostgreSQL 型（OID）のマッピング管理。

主要なデータ構造

  • statement: SQL 実行のライフサイクルを管理する中心構造体。
  • connection: データベース接続と関連するキャッシュを保持。
  • descriptor / descriptor_item: 動的 SQL 用のメタデータ保持構造。
  • sqlca_t: SQL 通信領域（エラー状態保持）。
  • COMPAT_MODE: 動作モード（PostgreSQL, Informix, Oracle）の切り替えフラグ。

主なフロー

1. **初期化**: ecpg_init によりスレッドごとの sqlca やメモリ管理領域が準備される。 2. **実行**: ECPGdo が呼び出され、statement 構造体が生成される。 3. **準備・バインド**: ecpg_auto_prepare がクエリをキャッシュし、ecpg_build_params がホスト変数をパラメータ配列へ変換する。 4. **通信**: libpq を介してクエリを送信し、結果を PGresult として受け取る。 5. **変換・格納**: ecpg_process_output が結果を解析し、ecpg_get_data を通じてホスト変数へ値をコピーする。 6. **終了**: ecpg_do_epilogue でメモリ解放とロケールの復元が行われる。

外部依存

  • libpq-fe.h: PostgreSQL 通信層。
  • pthread: スレッドセーフな状態管理（Win32 互換層含む）。
  • ecpgtype.h, sqlca.h, sqlda.h: SQL 標準および ECPG 固有の型定義。
  • catalog/pg_type_d.h: PostgreSQL システムカタログの型定義。

内部構成

本モジュールはディレクトリ境界に基づき構成されており、サブモジュールは存在しません。各ソースファイルは ecpglib_extern.h を中心に密結合しており、スレッドローカルストレージ（TLS）を多用してマルチスレッド環境での安全性を確保しています。

推奨読書順

1. ecpglib_extern.h（全体像の把握） 2. misc.c（初期化と基盤機能） 3. execute.c（SQL 実行のメインフロー） 4. prepare.c（キャッシュ機構） 5. descriptor.c（動的 SQL の管理） 6. connect.c（接続管理） 7. memory.c（メモリ管理ポリシー） 8. data.c（型変換ロジック） 9. sqlda.c（SQLDA 変換） 10. error.c（エラーハンドリング） 11. typename.c（型マッピング）
