contrib/file_fdw

システム内での役割

contrib/file_fdw は、PostgreSQL の外部データラッパー（FDW）インターフェースを実装するモジュールです。サーバー上のフラットファイル（CSV、テキスト等）や外部プログラムの出力を、PostgreSQL のテーブルとして透過的に読み込む機能を提供します。本モジュールはディレクトリ境界に基づき定義されています。

責務

  • **FDW ハンドラの提供**: プランナやエグゼキュータが外部データにアクセスするためのコールバック関数群（FdwRoutine）を登録します。
  • **オプション検証**: 外部テーブル定義時に指定された filename や program などのオプションが妥当か、および実行権限（pg_read_server_files 等）が適切かを検証します。
  • **プランニングとコスト見積もり**: 外部ファイルの統計情報を推定し、クエリ実行計画を作成します。
  • **データスキャン実行**: COPY コマンドのインフラストラクチャを再利用し、外部ソースからデータを読み込んでタプル形式へ変換します。

このモジュールのファイル

  • contrib/file_fdw/file_fdw.c

主要なデータ構造

  • FileFdwOption: 外部テーブルのオプション情報を保持。
  • FileFdwPlanState: プランニング段階で保持されるファイルパスや統計情報。
  • FileFdwExecutionState: 実行段階で保持される CopyFromState を含む実行コンテキスト。

主なフロー

1. **プランニング**: fileGetForeignRelSize 等によりファイルサイズから統計情報を推定し、スキャンパスを生成します。 2. **実行開始**: fileBeginForeignScan が BeginCopyFrom を呼び出し、ファイルまたはプログラムのストリームを開きます。 3. **データ取得**: fileIterateForeignScan が NextCopyFrom を介して外部データを 1 行ずつ取得し、TupleTableSlot にマッピングします。 4. **終了**: fileEndForeignScan が EndCopyFrom を呼び出し、リソースを解放します。
