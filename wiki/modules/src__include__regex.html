src/include/regex

システム内での役割

本モジュールは、PostgreSQL の正規表現エンジン（Henry Spencer 氏のパッケージをベースとしたもの）のインターフェースおよび内部構造を定義します。ディレクトリ境界に基づき、正規表現のコンパイル、実行、およびエラーハンドリングに必要なヘッダーファイル群を集約しています。

責務

  • 正規表現エンジンの公開 API（POSIX 互換）の提供。
  • PostgreSQL のメモリ管理や文字処理環境への抽象化レイヤーの提供。
  • コンパイル済み NFA（非決定性有限オートマトン）構造の内部定義と外部公開。
  • 正規表現処理におけるエラーコードとメッセージの管理。

このモジュールのファイル

  • regcustom.h: PostgreSQL 環境への適合（メモリ管理・文字型定義）。
  • regerrs.h: エラーコードとメッセージの定義テーブル。
  • regex.h: 公開インターフェースと主要なデータ構造の定義。
  • regexport.h: NFA 構造を外部（プランナ等）へ公開するためのインターフェース。
  • regguts.h: エンジン内部の低レベルデータ構造とアルゴリズム定義。

主要なデータ構造

  • **公開型**: pg_regex_t（コンパイル済み状態）、pg_regmatch_t（マッチ結果）。
  • **内部型**: guts（実行時状態）、nfa / cnfa（状態遷移グラフ）、colormap（文字クラス抽象化）、subre（サブ式ツリー）。
  • **その他**: chr（文字型）、regex_arc_t（遷移アーク）。

主なフロー

1. **コンパイル**: pg_regcomp が正規表現を解析し、colormap を用いて文字を抽象化し、nfa を構築・最適化して cnfa を生成します。 2. **実行**: pg_regexec が cnfa を参照し、入力文字列を「色（color）」に変換しながら状態遷移を行います。複雑な制約は subre ツリーを辿って解決されます。 3. **割り込み**: 実行中、regcustom.h で定義されたマクロにより、定期的にクエリキャンセルのチェックが行われます。

外部依存

  • postgres.h / miscadmin.h: メモリ管理（palloc）および割り込みチェック用。
  • mb/pg_wchar.h: PostgreSQL のマルチバイト文字処理用。
  • regex.h (システム標準): POSIX 互換インターフェースのベース。

内部構成

本モジュールはヘッダーファイルのみで構成されており、実装は src/backend/regex/ に存在します。

推奨読書順

1. regguts.h（エンジンの核心となるデータ構造とアルゴリズムの理解） 2. regex.h（公開 API と主要な型定義の確認） 3. regcustom.h（PostgreSQL 環境との統合・抽象化の理解） 4. regexport.h（外部コンポーネントとの連携インターフェース） 5. regerrs.h（エラーハンドリングの定義）
