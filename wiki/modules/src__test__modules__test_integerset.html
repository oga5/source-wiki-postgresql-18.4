src/test/modules/test_integerset

システム内での役割

本モジュールは、PostgreSQL の IntegerSet データ構造の機能と堅牢性を検証するためのテストスイートです。ディレクトリ境界に基づき定義されたテスト用モジュールであり、SQL インターフェースを通じて IntegerSet の実装を検証します。

責務

  • lib/integerset.h で定義された API の正当性確認。
  • 境界値（空集合、最大値付近）および Simple-8b エンコーディングの限界値に対する動作検証。
  • 整数集合の追加、メンバ判定、およびイテレーション処理の網羅的なテスト。
  • デバッグやパフォーマンスチューニングのための統計情報出力。

このモジュールのファイル

  • src/test/modules/test_integerset/test_integerset.c

主要なデータ構造

  • test_spec: テストパターンを定義し、検証対象のデータセットを保持する構造体。

主なフロー

1. **初期化**: SQL から test_integerset が呼び出され、小規模なテストケースを実行。 2. **パターン実行**: test_specs 配列に基づき test_pattern をループ実行。 3. **データ構築と検証**: intset_add_member で集合を構築し、intset_is_member によるランダムアクセスおよびイテレータによる全走査で整合性を確認。 4. **クリーンアップ**: メモリコンテキストを解放し、テストを終了。

外部依存

  • postgres.h, fmgr.h: PostgreSQL の基本定義および SQL インターフェース。
  • lib/integerset.h: テスト対象のデータ構造 API。
  • common/pg_prng.h: テスト用乱数生成。
  • utils/memutils.h, utils/timestamp.h: メモリ管理および時間計測。
