src/test/isolation

システム内での役割

PostgreSQL のトランザクション分離レベルや並行実行時の挙動を検証するためのテストスイートです。pg_regress フレームワーク上で動作し、複数のデータベース接続を並行制御することで、デッドロックや競合状態の再現テストを可能にします。

責務

  • .spec ファイルで定義されたテスト仕様の解析と実行。
  • 複数の libpq 接続を用いた並行ステップの実行と監視。
  • ロック待ち状態の判定およびタイムアウトによる異常系処理の制御。
  • テスト実行結果と期待値の比較（pg_regress 経由）。

このモジュールのファイル

  • src/test/isolation/isolation_main.c: テストランチャー（pg_regress ラッパー）。
  • src/test/isolation/isolationtester.c: テスト実行エンジンの中核。
  • src/test/isolation/isolationtester.h: テスト仕様のデータ構造定義。

主要なデータ構造

  • TestSpec: テスト全体の定義（セッション、セットアップ、順列）。
  • Session: 独立したデータベース接続の単位。
  • Step: 実行される個別の SQL コマンド。
  • PermutationStep: 順列内で実行されるステップとブロック条件のラッパー。
  • IsoConnInfo: 各セッションの接続状態やアクティブなステップを保持する実行コンテキスト。

主なフロー

1. **初期化**: isolation_main が pg_regress を介して起動し、isolationtester バイナリを特定。 2. **解析**: パーサー（specparse.y）が .spec ファイルを読み込み、TestSpec 構造体を構築。 3. **検証**: check_testspec がステップ間の依存関係やブロック条件を解決し、実行可能な状態にリンクを構築。 4. **実行**: run_permutation が各セッションに対して非同期に SQL を発行し、try_complete_step を通じてロック待ちや通知をポーリングしながら順列を消化。

外部依存

  • libpq: データベース接続および非同期クエリ実行。
  • pg_regress: テストスイートのライフサイクル管理。
  • postgres_fe.h: フロントエンド用共通定義。
  • sys/select.h: 非同期 I/O 監視。

内部構成

本モジュールは単一のディレクトリで構成され、サブモジュールは持ちません。

推奨読書順

1. src/test/isolation/isolationtester.h（データモデルの理解） 2. src/test/isolation/isolationtester.c（実行ロジックの理解） 3. src/test/isolation/isolation_main.c（テスト起動プロセスの理解）
