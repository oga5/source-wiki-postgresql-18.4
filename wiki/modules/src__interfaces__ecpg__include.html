src/interfaces/ecpg/include

システム内での役割

本モジュールは、PostgreSQL の埋め込み SQL (ECPG) ライブラリにおけるヘッダーファイルの集合体です。ECPG プリプロセッサが生成する C コードと、ランタイムライブラリ（libecpg）および pgtypeslib との間で、データ型、エラーコード、および通信領域（SQLCA）の定義を共有するインターフェース層として機能します。

※本モジュールはディレクトリ境界に基づき定義されています。

責務

  • **型定義の集約**: SQL 型と C 言語型のマッピング、および Informix 互換型定義の提供。
  • **エラーハンドリングの標準化**: ECPG 固有のエラーコードおよび SQL 標準の診断領域（SQLCA）の定義。
  • **プラットフォーム抽象化**: Windows 環境におけるスレッドセーフな処理（pthread 互換）の提供。
  • **動的 SQL サポート**: SQLDA（SQL Descriptor Area）のネイティブおよび互換形式の定義。

このモジュールのファイル

  • **型定義**: pgtypes.h, pgtypes_date.h, pgtypes_interval.h, pgtypes_numeric.h, pgtypes_timestamp.h, datetime.h, decimal.h, ecpgtype.h, sql3types.h, sqltypes.h
  • **通信・エラー**: sqlca.h, ecpgerrno.h, pgtypes_error.h
  • **動的 SQL**: sqlda.h, sqlda-native.h, sqlda-compat.h
  • **互換・環境**: ecpglib.h, ecpg_informix.h, ecpg-pthread-win32.h

主要なデータ構造

  • sqlca_t: SQL 実行結果やエラー状態を保持する通信領域。
  • sqlda_t / sqlda_struct: 動的 SQL 用の記述子領域。
  • numeric / decimal: 高精度数値計算用の構造体。
  • interval / timestamp: 日時演算用の構造体。
  • pthread_mutex_t (Win32): Windows 環境でのスレッド同期用構造体。

主なフロー

1. **プリプロセス**: プリプロセッサがソースコードを解析し、ECPGttype 等を用いてホスト変数の型を特定。 2. **実行**: ECPGdo 関数が SQL 文とホスト変数を libpq を介してサーバーへ送信。 3. **結果処理**: 実行結果が sqlca 構造体に書き込まれ、アプリケーションが SQLCODE や SQLSTATE を参照してエラーハンドリングを行う。 4. **型変換**: pgtypeslib が提供する関数群により、SQL 型と C 言語型間の相互変換や算術演算を実行。

外部依存

  • libpq-fe.h: PostgreSQL フロントエンドライブラリ。
  • errno.h, limits.h, stdint.h: 標準 C ライブラリ。
  • Windows API (windows.h 相当): スレッド処理用。

内部構成

本モジュールはフラットなヘッダーファイルの集合であり、サブモジュールは持ちません。各ヘッダーは必要に応じて相互にインクルードされ、型定義の整合性を保っています。

推奨読書順

1. ecpglib.h (全体像の把握) 2. sqlca.h (エラーハンドリングの基本) 3. ecpgtype.h (型システムの理解) 4. sqlda.h (動的 SQL の構造) 5. pgtypes_numeric.h / pgtypes_timestamp.h (主要なデータ型操作) 6. ecpg_informix.h (互換性レイヤー)
