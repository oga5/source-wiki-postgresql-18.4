src/pl/plpgsql/src

システム内での役割

本モジュールは、PostgreSQL の手続き型言語である PL/pgSQL の実装基盤です。ディレクトリ境界に基づき、言語のコンパイル、実行、構文解析、および関数管理を行う一連のソースコードを包含しています。

責務

  • **言語ハンドラ**: CREATE FUNCTION や DO ブロックの実行エントリーポイントの提供。
  • **コンパイル**: ソースコードを解析し、実行可能な抽象構文木（AST）へ変換。
  • **実行エンジン**: SPI（Server Programming Interface）を介した SQL 実行、変数管理、例外処理の統括。
  • **字句・構文解析**: キーワード管理およびトークン化によるパーサーへの入力供給。
  • **リソース管理**: 関数実行時のメモリコンテキストおよび名前空間（スコープ）の管理。

このモジュールのファイル

  • plpgsql.h: コアとなるデータ構造とインターフェース定義。
  • pl_handler.c: 言語ハンドラおよびバリデーターの実装。
  • pl_comp.c: コンパイラおよび名前解決ロジック。
  • pl_exec.c: 実行エンジンおよび式評価ロジック。
  • pl_funcs.c: AST 走査や名前空間管理のユーティリティ。
  • pl_scanner.c: 字句解析器。
  • pl_reserved_kwlist.h / pl_unreserved_kwlist.h: 予約語および非予約語の定義。

主要なデータ構造

  • PLpgSQL_function: コンパイル済みの関数全体を保持する構造体。
  • PLpgSQL_execstate: 関数の実行時状態（変数、スタック、メモリ）を保持。
  • PLpgSQL_stmt: 全ての文（Statement）の基底構造体。
  • PLpgSQL_datum: 変数やレコードなど、値を持つ要素の基底構造体。
  • PLpgSQL_expr: SQL クエリや式を保持し、SPI を通じて実行するための構造体。

主なフロー

1. **初期化**: pl_handler.c が関数呼び出しを受け取り、SPI 接続を確立。 2. **コンパイル**: pl_comp.c がソースを解析し、pl_scanner.c を用いてトークン化。AST を構築し、pl_funcs.c の名前空間管理機能を利用してスコープを解決。 3. **実行**: pl_exec.c が PLpgSQL_execstate を初期化し、AST を再帰的に辿りながらステートメントを実行。 4. **終了**: PG_TRY/PG_FINALLY ブロックにより、メモリコンテキストや SPI 接続を確実に解放。

外部依存

  • executor/spi.h: SQL 実行の基盤。
  • postgres.h: 基本的な型定義とマクロ。
  • utils/memutils.h: メモリコンテキスト管理。
  • catalog/pg_proc.h / pg_type.h: システムカタログ情報。

内部構成

本モジュールは、PL/pgSQL サブシステムを構成する単一のディレクトリとして定義されています。

推奨読書順

1. plpgsql.h: 全体像とデータ構造の把握。 2. pl_reserved_kwlist.h / pl_unreserved_kwlist.h: 言語仕様の確認。 3. pl_exec.c: 実行エンジンの理解。 4. pl_funcs.c: 管理ユーティリティの確認。 5. pl_comp.c: コンパイルロジックの詳細。 6. pl_scanner.c: 字句解析の実装。 7. pl_handler.c: システムとの接続点。
