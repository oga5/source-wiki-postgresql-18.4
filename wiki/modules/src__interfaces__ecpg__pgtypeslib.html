src/interfaces/ecpg/pgtypeslib

システム内での役割

pgtypeslib は、ECPG（Embedded SQL for C）アプリケーションが PostgreSQL のデータ型（numeric, date, timestamp, interval 等）をクライアントサイドで安全かつ透過的に扱うための型変換・演算ライブラリです。

責務

  • データベースの内部表現と C 言語のデータ型間の相互変換。
  • 日付・時刻の解析、フォーマット変換、および算術演算の提供。
  • 数値型（NUMERIC）の固定小数点演算およびメモリ管理。
  • ライブラリ全体で一貫したメモリ確保・解放インターフェースの提供。

このモジュールのファイル

  • common.c: メモリ管理および書式変換ユーティリティ。
  • datetime.c: date 型の操作と変換。
  • dt.h: 日付・時刻処理の共通定数・構造体定義。
  • dt_common.c: 日付・時刻の解析・エンコード・カレンダー計算の共通基盤。
  • interval.c: interval 型の解析・変換。
  • numeric.c: numeric 型の演算・変換。
  • pgtypeslib_extern.h: 公開関数・型定義のヘッダー。
  • timestamp.c: timestamp 型の操作と変換。

主要なデータ構造

  • numeric: 数値の桁数や符号を保持する可変長構造体。
  • interval: 時間間隔を保持する構造体。
  • datetkn: 日付・時刻解析用のトークン構造体。
  • fsec_t: 小数秒を扱う int32 型。
  • un_fmt_comb: 型変換時に値を保持する共用体。

主なフロー

1. **解析**: 文字列入力に対し ParseDateTime 等でトークン化し、Decode 系関数で内部表現（struct tm, int64 等）へ変換。 2. **演算**: TMODULO マクロや add_interval 等を用い、正規化や算術演算を実行。 3. **変換・出力**: Encode 系関数や pgtypes_fmt_replace を経由し、指定されたスタイル（ISO, SQL 等）の文字列へシリアライズ。

外部依存

  • postgres_fe.h（フロントエンド共通定義）
  • 標準ライブラリ: time.h, math.h, ctype.h, limits.h
  • 内部ヘッダー: pgtypes_*.h（各型の定義）

内部構成

本モジュールはディレクトリ境界により定義されており、サブモジュールは存在しません。各実装ファイルは dt.h や pgtypeslib_extern.h を通じて密接に連携しています。

推奨読書順

1. dt.h（基本データ構造とマクロ） 2. pgtypeslib_extern.h（公開インターフェース） 3. common.c（メモリ管理と変換基盤） 4. dt_common.c（日付・時刻処理の共通ロジック） 5. numeric.c（数値演算） 6. datetime.c（日付操作） 7. interval.c（期間操作） 8. timestamp.c（タイムスタンプ操作）
