src/test/modules/test_cloexec

このモジュールはディレクトリ境界に基づき定義されています。

システム内での役割

Windows環境における O_CLOEXEC フラグの動作を検証するための回帰テストモジュールです。PostgreSQLの移植性レイヤーが、Windowsのプロセス継承制御を正しく扱えているかを確認します。

責務

  • 親プロセスとして、O_CLOEXEC の有無が異なるファイルを開き、そのOSハンドルを子プロセスへ継承させる。
  • 子プロセスとして、受け取ったハンドルへの書き込みを試行し、継承の可否を検証する。
  • 検証結果を終了コードとして報告し、移植性レイヤーの妥当性を保証する。

このモジュールのファイル

  • src/test/modules/test_cloexec/test_cloexec.c

主要なデータ構造

  • 本モジュールは単一の実行ファイルであり、特定のデータ構造よりも、Windows APIの HANDLE 型やファイル記述子を介したプロセス間通信に依存しています。

主なフロー

1. **親プロセス起動**: open() でファイルを作成し、_get_osfhandle() でWindowsネイティブハンドルを取得。 2. **プロセス生成**: CreateProcess を bInheritHandles = TRUE で呼び出し、自身を子プロセスとして起動。 3. **検証**: 子プロセスが WriteFile を実行し、O_CLOEXEC が付与されたハンドルへの書き込みが失敗することを確認。 4. **判定**: 親プロセスが子プロセスの終了コードを回収し、テストの成否を判定。

外部依存

  • postgres.h: PostgreSQLの基本定義。
  • windows.h: Windows API (CreateProcess, HANDLE 等) の利用。
  • common/file_utils.h, port.h: PostgreSQLの移植性ユーティリティ。

内部構成

  • 子モジュール数: 0
  • Windows専用のテストであり、非Windows環境では実行されません。
