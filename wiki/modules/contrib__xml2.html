contrib/xml2

システム内での役割

contrib/xml2 は、PostgreSQL において libxml2 および libxslt を直接操作し、XML ドキュメントの高度な処理機能を提供する拡張モジュールです。SQL インターフェースを通じて、XML のクエリ（XPath）および変換（XSLT）を可能にします。

責務

  • **XPath 評価**: XML ドキュメントに対する XPath クエリの実行と、結果の PostgreSQL データ型への変換。
  • **XSLT 変換**: スタイルシートを用いた XML ドキュメントの変換と、セキュリティを考慮したリソース管理。
  • **リソース管理**: PostgreSQL のメモリ管理外にある libxml2/libxslt のオブジェクトを、エラー発生時を含めて安全に解放する。
  • **エラーハンドリング**: libxml2 のエラーを PostgreSQL の elog(ERROR) へ適切にマッピングする。

このモジュールのファイル

  • contrib/xml2/xpath.c: XML パース、XPath 評価、および共通の初期化・エラーハンドリング基盤。
  • contrib/xml2/xslt_proc.c: libxslt を用いた XML 変換機能の実装。

主要なデータ構造

  • xpath_workspace: 評価中の XML ドキュメント、XPath コンテキスト、結果オブジェクトを保持する管理構造体。

主なフロー

1. **初期化**: pgxml_parser_init により libxml2 のパーサーとエラーハンドリングを準備。 2. **処理実行**:

  • XPath の場合: pgxml_xpath が DOM ツリーを構築し、XPath を評価して結果をテキストへ変換。
  • XSLT の場合: xslt_process がスタイルシートをコンパイルし、セキュリティ設定を適用した上で変換を実行。

    • 3. **終了**: PG_TRY/CATCH ブロック内で確実にメモリ領域を解放し、コンテキストをクリーンアップ。

外部依存

  • **PostgreSQL**: postgres.h, fmgr.h, executor/spi.h, utils/xml.h 等。
  • **外部ライブラリ**: libxml2 (DOM/XPath 操作), libxslt (XSLT 変換)。

内部構成

本モジュールはディレクトリ境界に基づき構成されており、xpath.c が提供する共通基盤を xslt_proc.c が利用する依存関係にあります。

推奨読書順

1. contrib/xml2/xpath.c: モジュールの基盤となる初期化とメモリ管理を理解する。 2. contrib/xml2/xslt_proc.c: 変換レイヤーの実装とセキュリティ設計を確認する。

未解決の問い

  • 大規模データセットに対する xpath_table のパフォーマンス最適化の余地。
  • PostgreSQL 本体に統合された XML 機能との機能重複および将来的な棲み分け。
