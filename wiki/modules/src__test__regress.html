src/test/regress

システム内での役割

src/test/regress は、PostgreSQL の回帰テストを実行するための駆動プログラム pg_regress を提供するモジュールです。本モジュールは、テスト環境の構築、テストケースの並列実行、および結果の検証を自動化し、PostgreSQL の品質保証を支える基盤として機能します。

責務

  • **テスト実行エンジン**: スケジュールファイルに基づいたテストの並列実行とプロセス管理。
  • **環境構築**: 一時的なデータベースインスタンスの作成およびテスト実行に必要な環境変数の構成。
  • **結果検証**: 期待値ファイルと実行結果の比較、および TAP 形式でのレポート生成。
  • **SQL 拡張テスト**: 標準 SQL では検証困難な内部挙動（アトミック操作やデータ型）を検証するための C 関数提供。

このモジュールのファイル

  • pg_regress.c: テスト実行のコアロジックとライフサイクル管理。
  • pg_regress.h: モジュール間のインターフェース、データ構造、およびプラットフォーム抽象化の定義。
  • pg_regress_main.c: プログラムのエントリポイントおよび psql を用いたテスト実行手順の注入。
  • regress.c: SQL レベルから呼び出し可能なテスト用補助関数群。

主要なデータ構造

  • _stringlist: テスト名や引数リストを管理する連結リスト。
  • WIDGET: テスト専用のカスタムデータ型。
  • PID_TYPE: OS 依存性を吸収したプロセス ID 型。
  • init_function / test_start_function / postprocess_result_function: テストのライフサイクルを制御するコールバック関数型。

主なフロー

1. **初期化**: main が regression_main を呼び出し、環境変数や一時インスタンスを準備。 2. **実行**: スケジュールファイルに基づき、psql を用いたテストプロセスを並列生成。 3. **検証**: 各テスト終了後、diff を用いて期待値との差異を判定し、TAP 形式で出力。 4. **終了**: 全テスト完了後に一時インスタンスを停止し、クリーンアップを実行。

外部依存

  • libpq-fe.h: データベース接続確認用。
  • postgres_fe.h: フロントエンド用共通定義。
  • unistd.h / sys/wait.h: プロセス管理用。
  • lib/stringinfo.h: 動的文字列構築用。

内部構成

本モジュールはディレクトリ境界に基づき構成されており、pg_regress.c がエンジンとして機能し、regress.c が SQL 拡張機能を提供するという明確な役割分担がなされています。

推奨読書順

1. src/test/regress/pg_regress.h（インターフェースの理解） 2. src/test/regress/pg_regress.c（コアエンジンの理解） 3. src/test/regress/pg_regress_main.c（実行フローの確認） 4. src/test/regress/regress.c（SQL 拡張テストの実装例）
