src/test/modules/test_aio

本モジュールは、ディレクトリ境界に基づき定義された PostgreSQL のテスト用拡張モジュールです。

システム内での役割

PostgreSQL の非同期 I/O (AIO) サブシステムの堅牢性を検証するためのテスト用ヘルパーです。本番環境では使用されないデバッグ専用の SQL インターフェースを提供します。

責務

  • **SQL インターフェースの提供**: 低レベルな AIO 操作（バッファ作成、I/O 発行、ハンドル管理）を SQL から実行可能にする。
  • **エラー注入**: InjectionPoint を活用し、短縮読み込みや再オープン失敗などの異常系シナリオをシミュレートする。
  • **バッファ操作**: 共有バッファの状態を直接操作し、テストに必要な前提条件を強制的に構築する。

このモジュールのファイル

  • src/test/modules/test_aio/test_aio.c

主要なデータ構造

  • InjIoErrorState: エラー注入の状態を保持・管理するための共有メモリ構造体。

主なフロー

1. **初期化**: _PG_init により共有メモリを確保し、InjectionPoint を登録。 2. **操作実行**: SQL から read_rel_block_ll 等を呼び出し、pgaio API を介して I/O を発行。 3. **エラー注入**: I/O 処理中に InjectionPoint がトリガーされると、InjIoErrorState の設定に基づき意図的なエラーや結果の改ざんを行う。

外部依存

  • storage/aio.h, storage/aio_internal.h (AIO コア API)
  • storage/bufmgr.h, storage/buf_internals.h (バッファ管理)
  • utils/injection_point.h (テスト用注入機能)
  • その他 postgres.h, fmgr.h 等の基本ヘッダー

内部構成

  • 子モジュールなし（単一ファイル構成）

推奨読書順
