src/interfaces/libpq/test

システム内での役割

本モジュールは、PostgreSQL クライアントライブラリである libpq の機能検証およびビルド環境の疎通確認を行うためのテストプログラム群を格納しています。ディレクトリ境界に基づき構成された、libpq 開発における補助的なテストスイートです。

責務

  • libpq の公開 API（SSL 関連や接続文字列解析など）の動作検証。
  • ビルド環境やライブラリのリンク状態が正常であるかの確認。
  • 接続オプション解析ロジックの回帰テスト。

このモジュールのファイル

  • src/interfaces/libpq/test/libpq_testclient.c: SSL ライブラリのリンク確認などを行う最小限のテストクライアント。
  • src/interfaces/libpq/test/libpq_uri_regress.c: PQconninfoParse を用いた接続文字列解析の回帰テストツール。

主要なデータ構造

  • PQconninfoOption: libpq の接続オプションを保持する構造体（libpq-fe.h で定義）。

主なフロー

1. **環境検証 (libpq_testclient)**: コマンドライン引数を受け取り、PQsslAttribute を介してリンクされている SSL ライブラリ情報を出力する。 2. **解析検証 (libpq_uri_regress)**:

  • 引数から接続文字列を PQconninfoParse で解析。
  • PQconndefaults で得たデフォルト値と比較し、差異がある項目のみを標準出力へ表示。
  • 接続先（local/inet）を判定して結果を出力。

外部依存

  • postgres_fe.h: PostgreSQL フロントエンド用共通ヘッダー。
  • libpq-fe.h: libpq の公開 API 定義。

内部構成

本ディレクトリはテスト用ユーティリティの集合体であり、ライブラリ本体のコードは含みません。各ファイルは独立した実行バイナリとして機能します。

推奨読書順

1. src/interfaces/libpq/test/libpq_testclient.c（基本的な API 呼び出しの確認） 2. src/interfaces/libpq/test/libpq_uri_regress.c（接続ロジックの検証手法の確認）
