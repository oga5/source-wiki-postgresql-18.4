src/bin/psql

システム内での役割

src/bin/psql は、PostgreSQL の対話型ターミナルクライアント psql の全ソースコードを格納するディレクトリです。このモジュールは、ユーザー入力の解析、データベース接続の管理、メタコマンド（バックスラッシュコマンド）の実行、およびクエリ結果の整形表示を担う、PostgreSQL クライアントエコシステムの中心的なコンポーネントです。

※本モジュール構成はディレクトリ境界に基づき推論されています。

責務

  • **対話型インターフェースの提供**: readline 等を用いた入力補完、履歴管理、およびプロンプト表示。
  • **メタコマンド処理**: \d, \c, \copy などのバックスラッシュコマンドの解析と実行。
  • **クエリ実行エンジン**: libpq を介した SQL 送信、結果の取得、およびエラーハンドリング。
  • **状態管理**: PsqlSettings（pset）を通じた接続状態、変数、出力フォーマットの統合管理。
  • **メタデータ表示**: システムカタログをクエリし、データベースオブジェクトの詳細を整形して出力。

このモジュールのファイル

  • **初期化・制御**: startup.c, mainloop.c
  • **コマンド処理**: command.c, command.h, copy.c, describe.c, large_obj.c
  • **入力・補完**: input.c, tab-complete.in.c, psqlscanslash.h
  • **ユーティリティ・状態**: common.c, settings.h, variables.c, stringutils.c, prompt.c, crosstabview.c
  • 各種ヘッダーファイル（.h）

主要なデータ構造

  • PsqlSettings (pset): psql の全実行状態を保持するグローバル構造体。
  • VariableSpace: \set コマンド等で管理される変数リポジトリ。
  • backslashResult: メタコマンド実行後の制御フロー（終了、継続、クエリ送信など）を決定する列挙型。
  • printQueryOpt: クエリ結果の表示形式（テーブル、CSV等）を制御する構造体。
  • ConditionalStack: \if 等の条件付き実行を管理するスタック。

主なフロー

1. **起動**: startup.c が引数を解析し、pset を初期化してデータベースへ接続。 2. **入力**: mainloop.c が input.c を介してユーザー入力を取得。 3. **解析**: psqlscanslash.h および stringutils.c を用いて、入力が SQL かメタコマンドかを判別。 4. **実行**:

  • メタコマンドの場合: command.c がディスパッチし、各機能（describe.c 等）を実行。
  • SQL の場合: common.c の SendQuery が libpq を経由してサーバーへ送信。

    • 5. **表示**: fe_utils の機能を利用し、結果を整形して出力。

外部依存

  • libpq: PostgreSQL サーバーとの通信用ライブラリ。
  • fe_utils: クエリ結果の整形やスキャンなど、フロントエンド共通のユーティリティ。
  • readline / editline: 入力補完および履歴管理。

内部構成

本ディレクトリはフラットな構成であり、機能ごとにソースファイルが分割されています。pset グローバル変数がモジュール間の状態共有のハブとして機能しています。

推奨読書順

1. settings.h（全体像の把握） 2. startup.c（起動と初期化） 3. mainloop.c（メインループの構造） 4. command.c（メタコマンドのディスパッチ） 5. common.c（クエリ実行の基盤） 6. variables.c（変数管理） 7. describe.c（メタデータ表示の実装） 8. tab-complete.in.c（補完ロジック）
