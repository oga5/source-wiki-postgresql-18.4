src/backend/access/tablesample

システム内での役割

src/backend/access/tablesample は、PostgreSQL の TABLESAMPLE 句をサポートするサンプリングアルゴリズムの基盤モジュールです。クエリ実行エンジンに対し、テーブルスキャン時に指定された割合でデータを抽出するためのプラグイン可能なインターフェースを提供します。

責務

  • **サンプリングメソッドの提供**: SYSTEM および BERNOULLI サンプリングアルゴリズムの実装。
  • **実行時インターフェースの管理**: TsmRoutine を介したサンプリングロジックの動的ディスパッチ。
  • **再現性の保証**: ハッシュ関数を用いた、クエリ間で一貫したサンプリング結果の提供。
  • **プランナへの統計情報提供**: サンプリング後の推定ページ数およびタプル数の算出。

このモジュールのファイル

  • bernoulli.c: 各タプルに対して独立した確率判定を行う BERNOULLI メソッドの実装。
  • system.c: ページ単位でサンプリングを行い、選択されたページ内の全タプルを返す SYSTEM メソッドの実装。
  • tablesample.c: 各サンプリングメソッドのハンドラを動的に解決するゲートウェイ機能。

主要なデータ構造

  • TsmRoutine: サンプリングメソッドが実装すべき API を定義する構造体。
  • BernoulliSamplerData: BERNOULLI メソッド実行時の状態保持用構造体。
  • SystemSamplerData: SYSTEM メソッド実行時の状態保持用構造体。

主なフロー

1. **初期化**: クエリ実行エンジンが GetTsmRoutine を呼び出し、対象メソッドのハンドラを取得。 2. **スキャン開始**: beginsamplescan コールバックにより、サンプリング率からハッシュのカットオフ値を算出。 3. **サンプリング実行**:

  • SYSTEM の場合: nextsampleblock でハッシュ値に基づきブロックを選択し、その中の全タプルを返す。
  • BERNOULLI の場合: nextsampletuple で各タプルごとにハッシュ値を計算し、選択可否を判定する。

外部依存

  • access/tsmapi.h: サンプリング API の定義。
  • common/hashfn.h: 再現性のあるサンプリングのためのハッシュ関数（hash_any）。
  • optimizer/optimizer.h: プランナ用統計情報ユーティリティ。
  • postgres.h: 基本定義。

内部構成

本モジュールはディレクトリ境界により定義されており、各サンプリングメソッドは独立した実装として提供されます。

推奨読書順

1. src/backend/access/tablesample/tablesample.c（全体的なゲートウェイの理解） 2. src/backend/access/tablesample/system.c（ブロック単位のサンプリングロジック） 3. src/backend/access/tablesample/bernoulli.c（タプル単位のサンプリングロジック）
