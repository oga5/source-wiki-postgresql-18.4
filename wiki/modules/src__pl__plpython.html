src/pl/plpython

システム内での役割

src/pl/plpython は、PostgreSQL の手続き型言語（PL）として Python を統合するためのモジュールです。SQL 関数やトリガーの内部で Python コードを実行可能にし、データベースのデータ型と Python オブジェクト間の相互変換、および SPI（Server Programming Interface）を通じたクエリ実行機能を提供します。

責務

  • **言語ハンドラ**: PostgreSQL の関数呼び出しを Python インタプリタへ橋渡しする。
  • **データマーシャリング**: SQL の Datum と Python の PyObject 間の双方向変換（型変換）。
  • **SPI 統合**: Python から PostgreSQL のクエリ実行、プリペアドステートメント、カーソル操作を可能にする。
  • **エラーハンドリング**: Python の例外と PostgreSQL のエラー報告システム（ereport）の相互変換。
  • **キャッシュ管理**: プロシージャ定義や実行計画のメモリ内キャッシュによるオーバーヘッドの削減。

このモジュールのファイル

本モジュールは、ディレクトリ境界に基づき以下の 26 個のソースファイルで構成されています。

  • plpy_cursorobject.[ch]: カーソル操作の定義。
  • plpy_elog.[ch]: エラーおよび例外処理。
  • plpy_exec.[ch]: Python コード実行のメインエンジン。
  • plpy_main.[ch]: ハンドラのエントリポイントと初期化。
  • plpy_planobject.[ch]: 実行計画（プリペアドステートメント）の管理。
  • plpy_plpymodule.[ch]: Python 側 plpy モジュールの定義。
  • plpy_procedure.[ch]: プロシージャのキャッシュ管理。
  • plpy_resultobject.[ch]: クエリ結果の Python オブジェクト表現。
  • plpy_spi.[ch]: SPI 呼び出しのインターフェース。
  • plpy_subxactobject.[ch]: サブトランザクション制御。
  • plpy_typeio.[ch]: データ型変換ロジック。
  • plpy_util.[ch]: 文字列変換等のユーティリティ。
  • plpython.h, plpython_system.h: Python 環境との統合およびビルド設定。

主要なデータ構造

  • PLyProcedure: プロシージャの実行に必要なメタデータとキャッシュ。
  • PLyExecutionContext: 再帰呼び出しをサポートする実行スタック。
  • PLyDatumToOb / PLyObToDatum: SQL-Python 間の型変換メタデータ。
  • PLyResultObject: クエリ実行結果を保持する Python オブジェクト。
  • PLyPlanObject: プリペアドステートメントのラッパー。
  • PLySubtransactionObject: Python の with 文で利用可能なサブトランザクション制御オブジェクト。

主なフロー

1. **初期化**: plpython3_call_handler が呼び出され、Python インタプリタが起動・設定される。 2. **プロシージャ取得**: PLy_procedure_get により、キャッシュから実行対象の関数定義を取得（なければシステムカタログから構築）。 3. **実行準備**: PLy_exec_function 等が引数を Python オブジェクトに変換し、実行コンテキストをスタックに積む。 4. **コード実行**: Python インタプリタがコードを評価。SPI 経由のクエリ実行時は plpy_spi.c が介在。 5. **結果変換**: 戻り値を PLy_output_convert で SQL 型に変換し、PostgreSQL へ返却。

外部依存

  • Python.h (Python C API)
  • postgres.h (PostgreSQL コア)
  • executor/spi.h (SPI)
  • utils/memutils.h (メモリコンテキスト管理)

内部構成

本モジュールは単一のディレクトリで構成されており、サブモジュールは存在しません。各機能は plpy_ プレフィックスを持つファイル群によって論理的に分割されています。

推奨読書順

1. plpython.h / plpython_system.h (基盤設定) 2. plpy_main.h / plpy_main.c (エントリポイント) 3. plpy_procedure.h / plpy_procedure.c (キャッシュ管理) 4. plpy_exec.h / plpy_exec.c (実行エンジン) 5. plpy_typeio.h / plpy_typeio.c (型変換) 6. plpy_spi.h / plpy_spi.c (SPI 統合) 7. plpy_plpymodule.h / plpy_plpymodule.c (Python 側モジュール定義)
