src/timezone/pgtz.c

概要

PostgreSQL のタイムゾーン処理における統合レイヤーです。OS のタイムゾーンデータベース（tzdb）と PostgreSQL 内部の時刻計算ロジックを橋渡しし、タイムゾーンの読み込み、キャッシュ、および列挙機能を提供します。

モジュール内での役割

src/timezone サブシステムの中核であり、pgtz.h で定義された pg_tz 構造体を管理します。ファイルシステム上のタイムゾーンファイルを読み込み、localtime.c が提供する低レベルな解析関数を呼び出して、PostgreSQL が利用可能な形式に変換します。

責務

  • **タイムゾーンのキャッシュ管理**: ハッシュテーブルを用いて、一度読み込んだタイムゾーン定義をメモリ上に保持し、再読み込みのコストを削減します。
  • **ファイルシステム抽象化**: タイムゾーン名から対応するファイルパスを特定し、大文字小文字を区別しない検索をサポートします。
  • **初期化**: セッションおよびログ用のデフォルトタイムゾーン（通常は GMT）をシステム起動時に設定します。
  • **列挙機能**: 利用可能なタイムゾーンの一覧を取得するためのイテレータを提供します。

重要なシンボル

  • pg_tzset(const char *tzname): 指定された名前のタイムゾーンをロードし、pg_tz 構造体を返します。キャッシュヒット時は即座に返却します。
  • pg_open_tzfile(const char *name, char *canonname): タイムゾーンファイルを開くための低レベル関数。大文字小文字を区別しない検索を行います。
  • session_timezone / log_timezone: 現在のセッションおよびログ出力で使用されるタイムゾーンを指すグローバルポインタ。
  • pg_tzenumerate_next: 利用可能なタイムゾーンを再帰的に走査して列挙するイテレータ。

制御フローとデータフロー

1. **要求**: pg_tzset が呼ばれると、まず大文字に変換した名前で timezone_cache ハッシュテーブルを検索します。 2. **ロード**: キャッシュにない場合、tzload (localtime.c) を呼び出してファイルシステムからデータを読み込みます。 3. **正規化**: pg_open_tzfile はディレクトリを再帰的に走査し、ファイルシステム上の正しいパスと正規化された名前を特定します。 4. **保存**: 読み込まれた pg_tz 構造体はキャッシュに格納され、以降のアクセスを高速化します。

関連コード

  • src/timezone/localtime.c: tzload や tzparse を実装しており、実際のバイナリ解析を担当します。
  • src/timezone/tzfile.h: tzdb ファイルフォーマットの構造定義。
  • src/include/pgtime.h: タイムゾーン関連の公開 API 定義。

読者向けメモ

  • **大文字小文字の扱い**: タイムゾーン名はハッシュテーブル検索のために内部で大文字に変換されます。
  • **GMT の特別扱い**: GMT はファイルシステムに依存せず、常に tzparse を通じて初期化されるため、ブートストラップ時でも安全に利用可能です。
  • **セキュリティ**: scan_directory_ci 内で . で始まるファイルやディレクトリをスキップすることで、タイムゾーンディレクトリ外へのアクセスを防いでいます。
  • **列挙の注意**: pg_tzenumerate_next は palloc を使用するため、メモリコンテキストの管理に注意が必要です。
