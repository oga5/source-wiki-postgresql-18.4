src/fe_utils/connect_utils.c

概要

PostgreSQLのフロントエンドツール（pg_dump, pg_restore 等）がデータベースへの接続および切断を行うための共通ユーティリティを提供します。

モジュール内での役割

src/fe_utils モジュールの一部として、接続パラメータの管理、対話的なパスワード入力、および安全な接続確立プロセスを抽象化します。個別のツールが接続ロジックを再実装する手間を省き、一貫した接続体験を提供します。

責務

  • **接続の確立**: ConnParams 構造体に基づき、libpq を使用してデータベース接続を生成します。
  • **認証の自動化**: 必要に応じてパスワードを対話的に要求し、再利用可能な場合はキャッシュします。
  • **メンテナンス接続の解決**: メンテナンス用DB（postgres または template1）への接続を自動的にフォールバックして試行します。
  • **安全な切断**: 接続終了時にアクティブなトランザクションがあればキャンセルを試み、リソースを解放します。

重要なシンボル

  • connectDatabase: 接続のメイン関数。パスワードプロンプトの制御や接続パラメータの構築を行います。
  • connectMaintenanceDatabase: メンテナンス用DBへの接続を試みるラッパー。
  • disconnectDatabase: 接続を安全に閉じるための関数。
  • ConnParams: 接続先ホスト、ポート、ユーザー名、DB名などを保持する設定構造体。

制御フローとデータフロー

1. **初期化**: connectDatabase は ConnParams を受け取り、必要に応じて静的変数 password を管理します。 2. **接続試行**: PQconnectdbParams を呼び出し、接続を試みます。 3. **認証ループ**: 接続失敗時にパスワードが必要な場合、simple_prompt を呼び出して再試行ループを回します。 4. **セキュリティ設定**: 接続成功後、executeQuery を介して ALWAYS_SECURE_SEARCH_PATH_SQL を実行し、検索パスを保護します。 5. **切断**: disconnectDatabase は PQtransactionStatus を確認し、アクティブな場合は PQcancelBlocking でキャンセル処理を行ってから PQfinish を呼び出します。

直接依存

  • libpq-fe.h (via connect_utils.h): PostgreSQLクライアントライブラリ。
  • common/connect.h: 接続関連の共通定義。
  • common/logging.h: pg_fatal 等のログ出力。
  • fe_utils/query_utils.h: セキュリティ設定のためのクエリ実行。
  • common/string.h

関連コード

  • src/include/fe_utils/connect_utils.h: 公開インターフェースとデータ構造の定義。
  • src/fe_utils/query_utils.c: 接続直後に実行されるセキュリティクエリの定義。

読者向けメモ

  • **パスワードの再利用**: allow_password_reuse を true に設定すると、静的変数にパスワードが保持されます。異なるホストやユーザーへの接続で誤って使用しないよう注意してください。
  • **メンテナンスDBの優先順位**: connectMaintenanceDatabase は postgres を優先し、失敗時に template1 を試すというPostgreSQLの標準的な慣習に従っています。
  • **エラーハンドリング**: fail_ok フラグが true の場合、接続失敗時に pg_fatal で終了せず NULL を返します。呼び出し元で適切にチェックしてください。
