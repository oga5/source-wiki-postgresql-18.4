src/fe_utils/print.c

概要

PostgreSQLフロントエンドツール（psql等）におけるクエリ結果の出力整形を担うモジュールです。多様なフォーマット（aligned, unaligned, CSV, HTML, LaTeX, Troff等）でのレンダリング、端末幅に応じた折り返し、ページャ制御、およびロケール対応の数値フォーマットを提供します。

モジュール内での役割

src/fe_utils のプレゼンテーション層を構成します。libpq から取得した PGresult を、人間が読みやすい形式または機械可読な形式へ変換するパイプラインの終端として機能します。

責務

  • **フォーマット変換**: 内部のテーブル構造（printTableContent）を各出力形式へシリアライズ。
  • **端末表示最適化**: 端末幅の自動検出、カラムの折り返し、ページャ（more/less）の自動起動。
  • **ロケール対応**: LC_NUMERIC に基づく数値の桁区切りや小数点変換。
  • **シグナル管理**: SIGPIPE のトラップ制御および cancel_pressed フラグによる出力中断処理。

重要なシンボル

データ構造

  • printTableContent: 出力対象のテーブルデータ（ヘッダー、セル、フッター、配置情報）を保持するメイン構造体。
  • printTableOpt: 出力形式、境界線、ページャ設定などのレンダリングオプション。
  • printTextFormat: 各フォーマットの罫線やマークアップ定義。

状態管理

  • cancel_pressed: ユーザーによる割り込み（Ctrl+C）を検知するグローバル変数。
  • always_ignore_sigpipe: SIGPIPE を無視するかどうかの状態フラグ。

公開関数

  • printQuery: PGresult を受け取り、printTable を呼び出して出力する高レベルAPI。
  • printTableInit/printTableCleanup: printTableContent のライフサイクル管理。
  • printTableAddHeader/printTableAddCell/printTableAddFooter: テーブル構築用API。
  • PageOutput/ClosePager: ページャのパイプ管理。
  • setDecimalLocale: ロケール設定の初期化。

制御フローとデータフロー

1. **初期化**: printQuery が printTableInit でバッファを確保し、PQgetvalue 等でデータをセルに追加。 2. **フォーマット決定**: printTable が opt->format に基づき、print_aligned_text や print_csv_text 等の具体的なレンダラへディスパッチ。 3. **レンダリング**:

  • print_aligned_text は pg_wcssize で表示幅を計算し、必要に応じて width_wrap を調整。
  • strlen_max_width を用いてマルチバイト文字の表示幅を考慮した文字列切り出しを行う。

    • 4. **終了処理**: printTableCleanup で動的確保されたメモリ（セル、ヘッダー、フッター）を解放。

読者向けメモ

  • **メモリ所有権**: printTableContent に追加されたセルやヘッダーは、printTableCleanup が呼ばれるまで呼び出し側が生存期間を保証する必要があります。mustfree フラグが true の場合のみ、printTableCleanup が free() を実行します。
  • **ページャ**: PageOutput は isatty() を使用して端末かどうかを判定します。パイプ経由の出力や非TTY環境ではページャは起動しません。
  • **未実装・浅い部分**: latex_escaped_print 等の各エスケープ関数は、基本的なASCII文字の変換に留まっており、複雑な非ASCII文字のLaTeX/Troff変換は限定的です。
  • **注意点**: cancel_pressed は各行の出力完了時にチェックされます。長大な行の出力中には即座に中断されない場合があります。
