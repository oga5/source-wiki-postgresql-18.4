src/bin/pg_amcheck/pg_amcheck.c

概要

pg_amcheck は、PostgreSQL データベース内のリレーション（ヒープテーブルおよび B-tree 索引）の破損を検出するためのコマンドラインツールです。contrib/amcheck 拡張機能が提供する検証関数を呼び出し、並列処理を用いて効率的に整合性チェックを行います。

モジュール内での役割

本ファイルは src/bin/pg_amcheck モジュールの唯一のソースファイルであり、CLI の引数解析からデータベースへの接続、対象リレーションの特定、並列実行の制御、および結果の出力まで、プログラムの全ライフサイクルを管理します。

責務

  • **対象の特定**: ユーザー指定のパターン（データベース、スキーマ、リレーション）に基づき、チェック対象のリストを SQL クエリで動的に生成する。
  • **並列実行管理**: fe_utils/parallel_slot.h を利用し、複数のデータベース接続を並列に管理して検証コマンドを発行する。
  • **検証の実行**: ヒープ用（verify_heapam）および B-tree 用（bt_index_check 等）の SQL を構築・実行する。
  • **結果の集約と報告**: 検証結果を解析し、破損が検出された場合は詳細を標準出力に表示し、進捗状況を標準エラーに出力する。

重要なシンボル

  • AmcheckOptions: ユーザーのコマンドライン引数（パターン、並列数、チェックオプション）を保持する構造体。
  • RelationInfo: チェック対象となる個々のリレーションのメタデータ（OID、型、所属 DB）を保持する。
  • main: プログラムのメインループ。接続の確立、対象リストのコンパイル、並列スロットの駆動を行う。
  • verify_heap_slot_handler / verify_btree_slot_handler: 並列スロットから返された SQL 結果を処理するコールバック関数。

制御フローとデータフロー

1. **初期化**: 引数を解析し、AmcheckOptions を設定。 2. **対象リストの構築**: compile_database_list および compile_relation_list_one_db が SQL を発行し、チェック対象のデータベースとリレーションを特定する。 3. **並列実行**: ParallelSlotsSetup でスロットを準備し、run_command で各リレーションに対する検証 SQL を非同期に発行する。 4. **結果処理**: ハンドラ関数が各スロットの PGresult を受け取り、破損メッセージを解析・出力する。 5. **終了**: 全スロットの完了を待ち、終了コード（成功、破損あり、エラー）を返す。

読者向けメモ

  • 本ツールは contrib/amcheck 拡張がインストールされていることを前提としています。
  • 破損検出時の挙動は、on_error_stop オプションや、ヒープと B-tree での検証関数の戻り値の扱いの違い（ヒープは行を返し、B-tree はエラーを投げる）に注意して読んでください。
  • 大規模なデータベースを扱うため、リレーションのリストアップは SQL の CTE を駆使して効率化されています。複雑な SQL 構築ロジックを追う際は、PQExpBuffer の内容をデバッグ出力すると理解が早まります。
