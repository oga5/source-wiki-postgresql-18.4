src/bin/pg_dump/compress_io.c

概要

compress_io.c は、pg_dump および pg_restore における圧縮データストリーム処理の抽象化レイヤーです。Gzip、LZ4、Zstd などの複数の圧縮アルゴリズムを統一的なインターフェースで扱えるようにし、アーカイブの読み書きを簡素化します。

モジュール内での役割

pg_dump サブシステムにおいて、圧縮アルゴリズムごとの実装（compress_gzip.c 等）と、それを利用する上位のアーカイブ処理（pg_backup_archiver.c 等）を仲介するハブの役割を果たします。

責務

  • **圧縮アルゴリズムの抽象化**: 圧縮方式に依存しない汎用的な API を提供する。
  • **ストリーム管理**: 圧縮・解凍コンテキスト（CompressorState）のライフサイクル管理。
  • **ファイル操作の自動検出**: ファイル名や拡張子に基づき、適切な圧縮方式を自動的に選択・初期化する。
  • **エラーハンドリング**: 圧縮ライブラリの有無に応じたサポート状況の検証。

重要なシンボル

  • AllocateCompressor: 圧縮アルゴリズムに応じたコンテキストを生成するファクトリ関数。
  • InitDiscoverCompressFileHandle: ファイルパスから圧縮形式を推論し、適切な CompressFileHandle を構築する。
  • CompressorState: 圧縮・解凍処理に必要なコールバック関数と状態を保持する構造体。
  • CompressFileHandle: ファイル I/O を抽象化するための関数ポインタ群を保持する構造体。

制御フローとデータフロー

1. **初期化**: AllocateCompressor が呼び出されると、指定されたアルゴリズムに基づき、対応する InitCompressor* 関数が CompressorState を初期化します。 2. **データ処理**: writeData / readData が呼び出されると、内部で保持するコールバック関数（writeF / readF）を介して、圧縮済みデータがストリームへ流し込まれます。 3. **自動検出**: InitDiscoverCompressFileHandle は、ファイル拡張子を確認するか、ファイルシステム上の存在確認（stat）を試行して、最適な圧縮方式を決定します。 4. **終了**: EndCompressor または EndCompressFileHandle が呼び出され、バッファのフラッシュとメモリ解放が行われます。

読者向けメモ

  • 本ファイルは「圧縮ストリーム API」と「ファイルハンドル API」の2つの異なる抽象化を提供しています。前者はメモリ上のデータストリーム処理に、後者はファイル I/O に適しています。
  • 新しい圧縮アルゴリズムを追加する場合は、AllocateCompressor および InitCompressFileHandle 内の条件分岐に新しいアルゴリズムを追加し、対応する compress_*.c を実装する必要があります。
  • free_keep_errno は、メモリ解放時に errno を破壊しないためのユーティリティであり、エラーハンドリングの整合性を保つために多用されています。
