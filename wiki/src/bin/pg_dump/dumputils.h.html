src/bin/pg_dump/dumputils.h

概要

dumputils.h は、PostgreSQL のバックアップツールである pg_dump および pg_dumpall で共通して利用されるユーティリティ関数群を定義するヘッダーファイルです。SQL ダンプ生成時の文字列処理、権限（ACL）管理、設定変更コマンドの構築など、ダンプ出力の整合性を保つための補助機能を提供します。

モジュール内での役割

pg_dump サブシステムにおいて、特定のバックアップ形式（tar, custom 等）に依存しない、汎用的な文字列操作や SQL 生成ロジックを切り出した共通基盤です。dumputils.c と対になっており、ダンプ出力の整形やセキュリティ上のサニタイズを担います。

責務

  • **SQL インジェクション対策**: ダンプ内のオブジェクト名やコメントに含まれる改行コードの除去（sanitize_line）。
  • **権限管理**: データベースから取得した ACL 文字列を解析し、適切な GRANT/REVOKE コマンドを構築。
  • **設定管理**: GUC（設定パラメータ）のリスト処理や、ALTER コマンドの生成。
  • **環境適応**: プラットフォームごとのタイムスタンプ形式の差異を吸収。

重要なシンボル

  • PGDUMP_STRFTIME_FMT: ダンプ出力用のタイムスタンプフォーマット。Windows 環境でのエンコーディング問題を回避するため、プラットフォームにより定義が切り替わります。
  • sanitize_line(): 文字列中の改行をスペースに置換し、ダンプファイルの構造破壊を防ぐ関数。
  • buildACLCommands(): ACL 文字列から SQL コマンドを生成する中核関数。
  • SplitGUCList(): 設定パラメータのリストを解析するためのユーティリティ。

制御フローとデータフロー

本ファイルはインターフェース定義のみですが、実装（dumputils.c）における主なフローは以下の通りです： 1. **入力**: データベースから取得した ACL やオブジェクト名などの文字列を受け取る。 2. **処理**: PQExpBuffer を用いて SQL コマンドをバッファに蓄積する。この際、fmtId() 等を用いて識別子を適切にクォートする。 3. **出力**: 構築された SQL コマンド文字列をダンプストリームへ書き出す。

直接依存

  • libpq-fe.h: PostgreSQL クライアントライブラリの基本定義。
  • pqexpbuffer.h: 動的文字列バッファ管理機能。

関連コード

  • src/bin/pg_dump/dumputils.c: 本ヘッダーで宣言された関数の実装。
  • src/fe_utils/string_utils.h: 識別子のクォート処理など、より低レイヤーの文字列操作ユーティリティ。
  • src/bin/pg_dump/pg_dump.c: 本ユーティリティを呼び出し、ダンプ処理のメインループを実行するモジュール。

読者向けメモ

  • **セキュリティ**: sanitize_line はダンプファイルの整合性を維持するために不可欠です。オブジェクト名に改行が含まれる場合、ダンプのリストア時に SQL コマンドが分断されるリスクがあるため、必ずこの関数を通す必要があります。
  • **移植性**: Windows 環境でのタイムスタンプ出力には注意が必要です。PGDUMP_STRFTIME_FMT を使用することで、ロケール依存の文字化けを回避する設計になっています。
  • **拡張**: 新しいオブジェクトタイプに対する権限管理を追加する場合は、buildACLCommands の解析ロジックを拡張する必要があります。
