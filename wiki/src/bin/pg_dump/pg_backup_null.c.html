src/bin/pg_dump/pg_backup_null.c

概要

本ファイルは、pg_dump がアーカイブファイルを作成せず、直接プレーンテキスト形式の SQL スクリプトを出力するための「Null アーカイブ」実装です。

モジュール内での役割

pg_dump のアーカイブフォーマット抽象化レイヤーの一部です。pg_backup_archiver.c が定義するインターフェースを実装し、他のフォーマット（tar, custom, directory）と同様に、データ出力のバックエンドとして機能します。

責務

  • アーカイブ保存処理の無効化（_WriteByte, _WriteBuf 等を空実装）。
  • pg_dump が生成するデータを、そのまま標準出力（または指定ファイル）へ SQL として書き出す。
  • ラージオブジェクト（LO）のダンプ時に、適切な SQL コマンド（lo_open, lo_close 等）を挿入する。

重要なシンボル

  • InitArchiveFmt_Null: アーカイブハンドル（ArchiveHandle）の関数ポインタを本モジュールの関数で初期化するエントリポイント。
  • _WriteData: 通常のデータダンプ時に呼び出され、内容をそのまま出力する。
  • _WriteLOData: ラージオブジェクトのデータダンプ時に呼び出され、バイナリデータを lo_write 用の SQL リテラルに変換する。
  • _PrintTocData: TOC（目次）エントリに基づき、ダンプ対象のデータを順次出力する。

制御フローとデータフロー

1. **初期化**: InitArchiveFmt_Null が呼ばれ、ArchiveHandle の各ポインタが本モジュールの関数に差し替えられます。 2. **データ出力**: _PrintTocData が各 TOC エントリを走査し、dataDumper を呼び出します。 3. **変換**:

  • 通常データは _WriteData を経由してそのまま出力されます。
  • ラージオブジェクトの場合、_StartLO で WriteDataPtr が _WriteLOData に切り替わり、データが SQL リテラルとして整形されます。

    • 4. **終了**: _EndLO で WriteDataPtr が元に戻り、トランザクションのコミットやクローズ処理が行われます。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用共通ヘッダー。
  • fe_utils/string_utils.h: 文字列操作およびエスケープ用。
  • libpq/libpq-fs.h: ラージオブジェクト操作用定数。
  • pg_backup_archiver.h: アーカイブハンドルの構造体定義。
  • pg_backup_utils.h: ユーティリティ関数。

関連コード

  • pg_backup_archiver.c: 本モジュールが実装するインターフェースの定義元。
  • pg_dump.c: pg_dump のメイン処理。アーカイブフォーマットの選択と初期化を制御する。

読者向けメモ

  • 本モジュールは「読み込み（pg_restore）」をサポートしていません。archModeRead で初期化しようとすると pg_fatal で停止します。
  • _WriteByte や _WriteBuf が何もしないのは、プレーンテキスト出力時にはメタデータやヘッダーをファイルに書き込む必要がないためです。
  • ラージオブジェクトの処理において、古いアーカイブバージョンとの互換性維持のための分岐が存在することに注意してください。
