src/bin/pg_dump/pg_backup.h

概要

pg_backup.h は、PostgreSQL のバックアップおよびリストア処理におけるアーカイバの公開インターフェースを定義するヘッダーファイルです。pg_dump および pg_restore が共通して利用するデータ構造、列挙型、および関数プロトタイプを定義しています。

モジュール内での役割

pg_dump サブシステムにおける「共通言語」としての役割を果たします。バックアップの形式（カスタム、tar、ディレクトリ等）や、ダンプ/リストア時のオプション、データベース接続パラメータを抽象化し、各バックアップ形式の実装（pg_backup_custom.c 等）が従うべき契約を提供します。

責務

  • **設定の抽象化**: ダンプおよびリストア時の詳細なオプション（DumpOptions, RestoreOptions）の定義。
  • **アーカイバの管理**: Archive 構造体を通じたバックアップセッションの保持と状態管理。
  • **オブジェクト識別**: CatalogId（システムカタログ上のID）と DumpId（ダンプ処理上の論理ID）の定義によるオブジェクト管理の基盤提供。
  • **インターフェース定義**: アーカイブの作成、オープン、書き込み、リストアを行うための公開 API の提供。

重要なシンボル

  • Archive: ダンプ/リストア処理のメインコンテキストを保持する構造体。
  • ArchiveFormat: サポートされるアーカイブ形式（archCustom, archTar, archDirectory 等）の列挙。
  • DumpId: ダンプ対象オブジェクトを識別するための内部的な連番 ID。
  • DumpOptions / RestoreOptions: コマンドライン引数や設定を保持する構造体。
  • ArchiveMode: アーカイブの操作モード（読み込み、書き込み、追記）。

制御フローとデータフロー

本ファイルはインターフェース定義のため、直接的なロジックは持ちません。一般的なフローは以下の通りです： 1. CreateArchive または OpenArchive で Archive ハンドルを初期化。 2. SetArchiveOptions でオプションを適用。 3. WriteData や RestoreArchive を通じて、アーカイブ形式に応じた具体的な処理（pg_backup_custom.c 等）へ委譲。 4. CloseArchive でリソースを解放。

直接依存

  • common/compression.h: 圧縮仕様の定義。
  • common/file_utils.h: ファイル操作ユーティリティ。
  • fe_utils/simple_list.h: リスト管理用データ構造。
  • libpq-fe.h: PostgreSQL クライアントライブラリ（接続管理用）。

関連コード

  • pg_backup_archiver.h: アーカイバの内部的な実装詳細や、より具体的なメソッド定義。
  • pg_dump.h: pg_dump 特有のダンプロジックと本ヘッダーの連携。
  • pg_backup_custom.c / pg_backup_tar.c: 本ヘッダーで定義されたインターフェースを実装する各バックアップ形式のソース。

読者向けメモ

  • DumpId は CatalogId とは異なり、pg_dump が実行時に割り当てる論理的な識別子です。依存関係の解決やダンプ順序の制御に不可欠です。
  • Archive 構造体は非常に巨大ですが、その多くは pg_backup_archiver.c などの内部実装で利用されます。外部モジュールは、主に Archive ポインタを介して API を呼び出すことに集中してください。
  • appendStringLiteralAH マクロは、Archive のエンコーディング設定に基づいた安全な文字列エスケープを行うための重要なユーティリティです。
