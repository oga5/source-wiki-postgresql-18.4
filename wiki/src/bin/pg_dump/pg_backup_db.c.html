src/bin/pg_dump/pg_backup_db.c

概要

pg_backup_db.c は、pg_dump および pg_restore におけるデータベース接続と SQL 実行の抽象化レイヤーです。アーカイブハンドル（ArchiveHandle）を介して、サーバーへの接続管理、トランザクション制御、および SQL クエリの実行を担います。

モジュール内での役割

pg_dump モジュールにおいて、低レベルな libpq 操作をラップし、アーカイブ処理（pg_backup_archiver.c）がデータベースの状態を意識せずに SQL を発行できるようにする役割を持ちます。

責務

  • **接続管理**: データベースへの接続、再接続、および切断の管理。
  • **SQL 実行**: クエリの実行、結果の取得、およびエラーハンドリング。
  • **COPY モード制御**: COPY コマンドの開始・終了処理。
  • **ラージオブジェクト操作**: ラージオブジェクトに対する ACL 設定や削除コマンドの生成。
  • **互換性チェック**: 接続先サーバーのバージョン確認とリカバリモードの判定。

重要なシンボル

  • ConnectDatabaseAhx: データベース接続を確立し、初期設定（検索パス等）を行う。
  • ExecuteSqlCommandBuf: ahwrite インターフェースとして機能し、データ種別に応じて SQL 実行や COPY データのストリーミングを行う。
  • ExecuteSqlQueryForSingleRow: 単一の結果行を期待するクエリを実行する。
  • ReconnectToServer: 既存の接続を閉じ、新しい接続を確立する。
  • EndDBCopyMode: COPY 操作を終了し、libpq をアイドル状態に戻す。

制御フローとデータフロー

1. **接続**: ConnectDatabaseAhx が libpq を呼び出し、接続後に _check_database_version でサーバーバージョンを検証します。 2. **クエリ実行**: ExecuteSqlStatement や ExecuteSqlQuery が PQexec をラップし、失敗時に die_on_query_failure を通じて終了処理を行います。 3. **データ復元**: ExecuteSqlCommandBuf が呼び出されると、ArchiveHandle の状態（OUTPUT_COPYDATA か OUTPUT_OTHERDATA か）に基づき、COPY データとして流すか、ExecuteSimpleCommands で SQL コマンドとして解析・実行するかを振り分けます。 4. **解析**: ExecuteSimpleCommands は、バッファリングされた SQL を字句解析し、セミコロン単位でコマンドを切り出して実行します。

読者向けメモ

  • 本ファイルは ArchiveHandle 構造体を Archive 型からキャストして操作します。
  • ExecuteSimpleCommands は、SQL コマンドがバッファ境界で分割されることを想定した簡易的なステートマシン（SQL_SCAN, SQL_IN_SINGLE_QUOTE 等）を実装しています。複雑な SQL 解析は行わないため、特殊な構文には注意が必要です。
  • pg_fatal や warn_or_exit_horribly が多用されており、エラー発生時は即座にプロセスが終了する設計になっています。
