src/bin/pg_dump/compress_none.h

概要

compress_none.h は、pg_dump における非圧縮（生データ）入出力処理を定義するヘッダーファイルです。圧縮アルゴリズムを使用しない場合のデータストリーム操作を、compress_io.h で定義された共通インターフェースに適合させます。

モジュール内での役割

pg_dump の圧縮サブシステムにおいて、圧縮を行わない「パススルー」の実装を提供します。他の圧縮方式（Gzip, LZ4, Zstd）と同様に、CompressorState および CompressFileHandle の各関数ポインタを初期化する役割を担います。

責務

  • 非圧縮データ用の CompressorState 初期化関数の宣言。
  • 非圧縮ファイル操作用の CompressFileHandle 初期化関数の宣言。
  • 圧縮サブシステムに対する、透過的なデータ入出力の提供。

重要なシンボル

  • InitCompressorNone: CompressorState 構造体の関数ポインタを、非圧縮処理用の関数（ReadDataFromArchiveNone 等）で埋めます。
  • InitCompressFileHandleNone: ファイル操作用の関数ポインタ（read_none, write_none 等）をセットアップし、標準的なファイル I/O をラップします。

制御フローとデータフロー

本ファイル自体は宣言のみですが、対応する compress_none.c では以下のフローが実行されます。 1. **初期化**: InitCompressorNone が呼び出され、ArchiveHandle が非圧縮モードに設定されます。 2. **データ転送**: ReadDataFromArchiveNone が cs->readF を介してデータを読み込み、ahwrite でアーカイブへ書き出します。 3. **ファイル操作**: CompressFileHandle を通じて、標準的な fread/fwrite が呼び出され、バッファリングなしの直接的なデータ転送が行われます。

直接依存

  • compress_io.h: CompressorState および CompressFileHandle 構造体の定義元。

関連コード

  • src/bin/pg_dump/compress_io.c: 圧縮サブシステムの基盤となる実装。
  • src/bin/pg_dump/compress_gzip.h, compress_lz4.h, compress_zstd.h: 他の圧縮方式のインターフェース定義。これらと並列して使用されます。

読者向けメモ

  • 本モジュールは「圧縮しない」という選択肢を、他の圧縮方式と同一の API で扱うためのアダプターです。
  • InitCompressorNone および InitCompressFileHandleNone は、pg_dump が出力形式を決定する際に、圧縮オプションが指定されていない場合に呼び出されます。
  • 複雑なロジックは含まれておらず、標準的なファイル操作を CompressFileHandle の規約に従ってラップしているだけである点に注目してください。
