src/bin/pg_dump/pg_backup_archiver.c

概要

pg_backup_archiver.c は、PostgreSQL の論理バックアップおよびリストア処理における「アーカイブエンジン」の中核です。ダンプ・リストアの実行状態を管理する ArchiveHandle を中心に、アーカイブ形式（Custom, Tar, Directory）の抽象化、TOC（Table of Contents）の管理、並列処理のオーケストレーション、およびデータベース接続のライフサイクルを制御します。

モジュール内での役割

pg_dump サブシステムにおいて、このファイルは「高レベルな制御層」として機能します。具体的なデータ抽出（pg_dump.c）や、特定のアーカイブ形式への物理的な書き込み（pg_backup_custom.c 等）を抽象化し、一貫したインターフェースを提供します。

責務

  • **アーカイブ管理**: ArchiveHandle を通じたアーカイブのオープン、クローズ、および入出力の抽象化。
  • **TOC 管理**: ダンプ対象オブジェクトのリスト（TOC）の構築、依存関係の解決、およびリストア順序の制御。
  • **並列リストア**: ParallelState を用いたマルチプロセスによるリストアの実行と、依存関係に基づくジョブスケジューリング。
  • **セッション制御**: リストア時の SET SESSION AUTHORIZATION、search_path、default_tablespace 等のセッション状態の管理。
  • **ラージオブジェクト（LO）処理**: LO のダンプ・リストアにおけるバッファリングとストリーミング。

重要なシンボル

アーカイブ管理

  • CreateArchive / OpenArchive: アーカイブハンドルの初期化と生成。
  • CloseArchive: リソースの解放と最終的な I/O 終了処理。
  • RestoreArchive: リストア処理のメインエントリーポイント。

TOC 操作

  • ArchiveEntry: 新しい TOC エントリをアーカイブに追加。
  • PrintTOCSummary: アーカイブのメタデータとエントリ一覧の出力。
  • SortTocFromFile: 外部ファイルに基づいた TOC の順序変更。

並列処理・依存関係

  • WriteDataChunks: 並列ダンプ時のジョブディスパッチ。
  • restore_toc_entries_parallel: 並列リストアのメインループ。
  • fix_dependencies: オブジェクト間の依存関係グラフの構築と調整。
  • reduce_dependencies: 完了したジョブに基づき、依存する後続ジョブの実行可能フラグを更新。

状態管理・ユーティリティ

  • ArchiveHandle: アーカイブの全状態を保持する構造体。
  • ahprintf / ahwrite: アーカイブ出力へのフォーマット済み書き込み。
  • _doSetFixedOutputState: リストア開始時に必要なセッションパラメータ（statement_timeout 等）の設定。

制御フローとデータフロー

1. **初期化**: OpenArchive が ArchiveHandle を生成し、アーカイブ形式に応じた関数ポインタ（InitArchiveFmt_*）をセットアップします。 2. **依存関係構築**: fix_dependencies が TOC エントリ間の依存関係をグラフ化し、depCount を計算します。 3. **リストア実行**:

  • **直列**: restore_toc_entry を RESTORE_PASS ごとに呼び出します。
  • **並列**: restore_toc_entries_parallel が binaryheap を使用して、依存関係が解決された（depCount == 0）エントリを優先的にワーカーへディスパッチします。

    • 4. **データフロー**: WriteData / ahwrite が呼び出されると、ArchiveHandle 内の関数ポインタを通じて、圧縮層（CompressFileHandle）または直接データベース接続（ExecuteSqlCommandBuf）へデータが流れます。

読者向けメモ

  • **メモリ管理**: ArchiveHandle は非常に大きな構造体であり、CloneArchive で複製される際は、接続状態やバッファが適切に分離されるよう注意が必要です。
  • **並列リストアの制約**: 並列リストアは ArchiveHandle の ClonePtr が実装されている形式（Custom, Directory）でのみサポートされます。
  • **セッション状態**: _selectOutputSchema や _becomeUser は、不要な SET コマンドを避けるために現在の状態をキャッシュしています。このキャッシュをクリアするタイミング（_reconnectToDB 等）が重要です。
  • **未実装・浅い部分**: SortTocFromFile による順序変更は、並列リストア時の依存関係グラフと競合する可能性があるため、複雑な依存関係を持つアーカイブでは注意が必要です。
