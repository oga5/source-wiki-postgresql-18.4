src/bin/pg_dump/compress_none.c

概要

本ファイルは、pg_dump における非圧縮（None）データストリームの読み書きを抽象化するための実装です。圧縮を行わない場合の入出力処理を、compress_io モジュールが期待するインターフェースに適合させます。

モジュール内での役割

pg_dump の圧縮サブシステムの一部として、Gzip や Zstd と同等の API を提供します。圧縮が指定されていない場合、このモジュールが選択され、標準的なファイル I/O 操作を透過的に実行します。

責務

  • **Compressor API の実装**: アーカイブデータの読み書きにおける「圧縮なし」の挙動（単なる転送）を定義します。
  • **Compress File API の実装**: FILE* をラップし、fread/fwrite 等の標準関数を CompressFileHandle 構造体経由で利用可能にします。
  • **エラーハンドリング**: ファイル操作失敗時の致命的なエラーを pg_fatal で報告します。

重要なシンボル

  • InitCompressorNone: CompressorState に非圧縮用の関数ポインタを割り当てます。
  • InitCompressFileHandleNone: CompressFileHandle にファイル操作用の関数ポインタを割り当てます。
  • ReadDataFromArchiveNone / WriteDataToArchiveNone: 圧縮を介さずデータを直接転送するコアロジックです。

追加の索引済みシンボル

  • EndCompressorNone
  • close_none
  • eof_none
  • get_error_none
  • getc_none
  • gets_none
  • read_none
  • write_none

制御フローとデータフロー

1. **初期化**: InitCompressorNone または InitCompressFileHandleNone が呼び出され、関数ポインタが構造体にセットされます。 2. **データ転送**:

  • 読み込み時: read_none が fread を呼び出し、データをバッファへ格納します。
  • 書き込み時: write_none が fwrite を呼び出し、データを直接出力先へ書き出します。

    • 3. **終了**: close_none が呼び出され、内部の FILE* がクローズされます。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用共通定義。
  • unistd.h: dup 等の POSIX システムコール用。
  • compress_none.h: 本モジュールの公開ヘッダー。
  • pg_backup_utils.h: pg_fatal や pg_log_error などのエラー報告ユーティリティ。

関連コード

  • compress_io.c: 本モジュールが適合する上位の I/O 抽象化レイヤー。
  • compress_gzip.c, compress_lz4.c, compress_zstd.c: 他の圧縮アルゴリズムの実装（比較対象）。
  • pg_backup_archiver.c: アーカイブ処理のメインロジック。

読者向けメモ

  • 本ファイルは非常に単純なラッパーであり、複雑な状態管理は行いません。
  • private_data メンバには常に FILE* が格納されます。
  • open_none では fd が渡された場合に dup を使用してファイル記述子を複製し、fdopen でストリーム化する点に注意してください。
