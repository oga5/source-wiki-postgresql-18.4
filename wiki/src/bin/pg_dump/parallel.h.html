src/bin/pg_dump/parallel.h

概要

parallel.h は、pg_dump および pg_restore における並列処理のインターフェースを定義するヘッダーファイルです。マルチプロセス（Unix）またはマルチスレッド（Windows）環境で、アーカイブのダンプやリストアを効率的に実行するための制御構造を提供します。

モジュール内での役割

pg_dump サブシステムにおいて、ジョブの分散と同期を司る並列処理層の最上位定義です。実際のロジックは parallel.c に実装されており、本ファイルはリーダープロセス（メイン）とワーカープロセス間の通信および状態管理のための型と関数を公開します。

責務

  • 並列ジョブの実行状態を保持する ParallelState 構造体の定義。
  • ワーカーのライフサイクル管理（開始、ジョブ割り当て、終了）のための API 提供。
  • ジョブ完了時のコールバック関数シグネチャの定義。
  • プラットフォーム依存の並列数制限（PG_MAX_JOBS）の抽象化。

重要なシンボル

  • ParallelState: 並列処理全体の管理構造体。ワーカー数や各ワーカーの現在のタスク（TocEntry）を保持します。
  • ParallelCompletionPtr: ジョブ完了時にリーダープロセスで実行されるコールバック関数の型定義。
  • WFW_WaitOption: WaitForWorkers 関数における待機条件（全アイドル待ち、いずれか1つ待ち等）を指定する列挙型。
  • ParallelBackupStart() / ParallelBackupEnd(): 並列処理の開始と終了を制御する主要関数。
  • DispatchJobForTocEntry(): 特定の TocEntry をワーカーに割り当て、処理を開始させる関数。

追加の索引済みシンボル

  • ParallelSlot

制御フローとデータフロー

1. **初期化**: ParallelBackupStart が呼び出され、指定された数のワーカーが生成されます。 2. **ジョブ投入**: リーダープロセスが DispatchJobForTocEntry を呼び出し、処理対象の TocEntry をワーカーに送ります。 3. **実行**: ワーカーは ArchiveHandle を通じてダンプ/リストアを実行します。 4. **同期**: リーダーは WaitForWorkers を使用してワーカーの状態を監視し、完了時にコールバックを介して状態を更新します。 5. **終了**: 全ジョブ完了後、ParallelBackupEnd によりワーカーが解放されます。

直接依存

  • limits.h: PG_MAX_JOBS の定義に使用。
  • pg_backup_archiver.h: ArchiveHandle や TocEntry など、アーカイブ処理の基盤構造体を参照。

関連コード

  • src/bin/pg_dump/parallel.c: 本ヘッダーで宣言された関数の実装および、ワーカーのパイプ通信ロジック。
  • src/bin/pg_dump/pg_backup_archiver.c: 並列処理のコールバックが実際に状態を更新する先となるメインのアーカイブ制御ロジック。

読者向けメモ

  • ParallelSlot は不透明ポインタ（opaque struct）として定義されており、内部構造は parallel.c でのみ参照可能です。
  • Windows 環境では MAXIMUM_WAIT_OBJECTS の制限により、並列数は最大 64 に制限されます。
  • ワーカープロセスは、リーダーがカタログを読み込んだ後にフォークされるため、カタログ情報へのアクセス権を継承します。Windows ではスレッドとして動作し、ArchiveHandle のクローンを操作します。
