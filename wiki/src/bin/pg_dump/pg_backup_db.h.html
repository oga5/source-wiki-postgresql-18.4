src/bin/pg_dump/pg_backup_db.h

概要

pg_backup_db.h は、pg_dump および pg_restore におけるデータベース接続および SQL 実行の抽象化レイヤーを定義するヘッダーファイルです。アーカイブ処理中に必要な SQL コマンドの実行、トランザクション制御、およびデータベース接続の管理機能を提供します。

モジュール内での役割

src/bin/pg_dump モジュールにおいて、低レベルな libpq の操作を ArchiveHandle 構造体と統合する役割を担います。pg_backup_db.c とペアで動作し、アーカイバがデータベースの状態を意識せずに SQL を発行できるインターフェースを提供します。

責務

  • **SQL 実行の抽象化**: SQL 文の実行、結果の取得、および単一行取得のヘルパー関数の提供。
  • **トランザクション管理**: pg_dump の処理単位となるトランザクションの開始とコミットの制御。
  • **接続管理**: データベース接続の確立、再接続、および接続状態の維持。
  • **エラーハンドリング**: 実行結果のステータスチェックと、データベースバージョン不整合の検証。

重要なシンボル

  • ExecuteSqlStatement: 指定した SQL 文を実行し、エラー発生時に適切に処理を中断します。
  • ExecuteSqlQuery: SQL を実行し、期待される ExecStatusType を検証して PGresult を返します。
  • ExecuteSqlQueryForSingleRow: 単一の結果行を期待するクエリを実行するための簡便なラッパーです。
  • StartTransaction / CommitTransaction: アーカイブ処理におけるトランザクション境界を定義します。
  • EndDBCopyMode: COPY モードでのデータ転送を終了させるための制御関数です。

制御フローとデータフロー

1. **接続**: ConnectDatabaseAhx が libpq 接続を確立し、ArchiveHandle に保持します。 2. **検証**: _check_database_version がサーバーバージョンを確認し、互換性がない場合は処理を停止します。 3. **実行**: ExecuteSqlStatement 等を通じて SQL が発行されます。これらは内部で libpq を呼び出し、結果を ArchiveHandle を介して管理します。 4. **終了**: EndDBCopyMode 等により、特定のデータ転送フェーズを安全に終了させます。

直接依存

  • pg_backup.h: Archive 構造体の定義およびアーカイバの基本型定義に依存しています。

関連コード

  • src/bin/pg_dump/pg_backup_db.c: 本ヘッダーで宣言された関数の実装が含まれます。
  • src/bin/pg_dump/connectdb.c: 実際の接続ロジック（ConnectDatabase）が定義されています。
  • src/bin/pg_dump/pg_backup_archiver.h: ArchiveHandle 構造体の詳細が定義されており、本モジュールが操作する主要なデータ構造です。

読者向けメモ

  • 本ファイルで宣言されている関数は、主に pg_dump のアーカイバがデータベースと対話する際に使用されます。
  • ExecuteSqlQuery などの関数は、呼び出し側で PGresult の解放（PQclear）を行う必要がある点に注意してください。
  • データベース接続の再利用や再接続を行う際は、ReconnectToServer を通じて接続状態が正しく更新されることを確認してください。
