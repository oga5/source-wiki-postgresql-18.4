src/bin/pg_dump/dumputils.c

概要

dumputils.c は、pg_dump および pg_dumpall で共通して利用される SQL 生成および文字列操作のユーティリティ関数群を提供します。主にデータベースオブジェクトの権限（ACL）管理、設定パラメータ（GUC）のダンプ、およびディレクトリ操作を抽象化します。

モジュール内での役割

pg_dump サブシステムにおいて、SQL 文の構築ロジックをメインのダンプ処理から分離する役割を担います。特に、複雑な ACL 文字列の解析や、環境依存のディレクトリ作成処理をカプセル化することで、メインロジックの可読性と保守性を向上させています。

責務

  • **ACL 管理**: データベースから取得した ACL 文字列を解析し、適切な GRANT/REVOKE 文を生成する。
  • **GUC 設定のダンプ**: ALTER ... SET 文の生成および、リスト形式の GUC 変数の安全なパース。
  • **セキュリティラベル**: 共有オブジェクトに対するセキュリティラベルのクエリ構築と出力。
  • **ユーティリティ**: SQL コメント用の文字列サニタイズや、ディレクトリの作成・検証。

重要なシンボル

  • buildACLCommands: オブジェクトの ACL を比較し、必要な GRANT/REVOKE 文を PQExpBuffer に構築する主要関数。
  • parseAclItem: ACL 文字列を解析し、権限、被付与者、付与者を抽出する内部ヘルパー。
  • makeAlterConfigCommand: ALTER DATABASE/ROLE SET 文を生成する。
  • SplitGUCList: GUC リスト変数を解析し、識別子リストに分割する。
  • sanitize_line: 改行コードをスペースに置換し、ダンプ出力の整合性を保つ。

制御フローとデータフロー

1. **ACL 生成**: buildACLCommands が呼び出されると、parsePGArray で ACL を配列化し、現在の ACL とベース ACL を比較して差分（grantitems/revokeitems）を特定します。 2. **コマンド構築**: 特定された差分に対し、parseAclItem を用いて個別の権限を解析し、PQExpBuffer を介して SQL 文を連結します。 3. **GUC 処理**: makeAlterConfigCommand は、設定値が GUC_LIST_QUOTE に該当する場合、SplitGUCList で分解した後に個別にクォートして SQL を生成します。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用共通ヘッダー。
  • dumputils.h: 本ファイルのインターフェース定義。
  • fe_utils/string_utils.h: fmtId や appendStringLiteralConn などの文字列操作ユーティリティ。
  • common/file_perm.h, common/logging.h: ファイル権限管理およびログ出力用。
  • ctype.h

関連コード

  • src/backend/utils/adt/acl.c: バックエンド側の ACL 処理ロジック。本ファイルの解析ロジックと整合性を保つ必要がある。
  • src/backend/utils/misc/guc.c: GUC 変数の定義と管理。variable_is_guc_list_quote のリストはこれと同期させる必要がある。

読者向けメモ

  • 本ファイル内の ACL 解析ロジックは、PostgreSQL のバージョン間で ACL 形式が変更された場合、慎重な調整が必要です。
  • sanitize_line は SQL インジェクションを防ぐための重要な防壁です。ダンプ出力に名前を含める際は必ず経由してください。
  • create_or_open_dir は pg_check_dir を利用しており、ディレクトリが空でない場合にエラーを投げるなど、ダンプファイルの破損を防ぐための厳格なチェックを行っています。
