src/bin/scripts/createuser.c

概要

PostgreSQL のコマンドラインユーティリティ createuser の実装です。指定されたオプションに基づき、サーバーに対して CREATE ROLE SQL コマンドを発行し、新しいロールを作成します。

モジュール内での役割

src/bin/scripts モジュールの一部として、データベース管理者がシェルからロール管理を行うためのフロントエンドツールを提供します。他の create... 系ツールと同様、libpq を利用してメンテナンスデータベースに接続し、操作を実行します。

責務

  • コマンドライン引数の解析と、ロール作成に必要な属性（superuser, createdb, login 等）の収集。
  • 対話モード（--interactive）におけるユーザー入力の処理。
  • サーバーへの接続確立と、構築した CREATE ROLE 文の実行。
  • パスワードの暗号化処理（PQencryptPasswordConn を使用）。

重要なシンボル

  • main: プログラムのメインエントリポイント。引数解析から SQL 発行までの一連の流れを制御します。
  • long_options: getopt_long で使用されるコマンドラインオプションの定義。
  • SimpleStringList: ロール名やメンバーシップ情報を保持するためのリスト構造。
  • trivalue: 属性の指定状態（デフォルト、YES、NO）を管理する列挙型。

追加の索引済みシンボル

制御フローとデータフロー

1. **初期化**: pg_logging_init 等で環境を整え、getopt_long でオプションを解析します。 2. **入力補完**: 引数で指定されなかった属性やロール名について、対話モードであれば simple_prompt を通じてユーザーに入力を求めます。 3. **接続**: connectMaintenanceDatabase を呼び出し、サーバーへの接続を確立します。 4. **SQL 構築**: PQExpBuffer を使用して、解析したオプションに基づき CREATE ROLE ... 文を動的に組み立てます。 5. **実行**: PQexec で SQL を送信し、結果を確認して終了します。

読者向けメモ

  • 本ファイルは CREATE ROLE 文を生成するロジックが中心です。SQL の構文や属性の詳細は PostgreSQL の公式ドキュメントを参照してください。
  • trivalue 型の使用は、ユーザーが明示的に指定した値とデフォルト値を区別するために重要です。
  • パスワードの取り扱いには注意が必要であり、PQencryptPasswordConn を介して安全に暗号化された文字列を SQL に埋め込んでいます。
