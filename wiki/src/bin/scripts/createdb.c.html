src/bin/scripts/createdb.c

概要

createdb.c は、PostgreSQL のデータベース作成コマンド createdb を実装するクライアントサイドのユーティリティです。SQL の CREATE DATABASE 文を生成し、サーバーへ送信することでデータベース作成を自動化します。

モジュール内での役割

src/bin/scripts モジュールの一部として、ユーザーがコマンドラインからデータベースを管理するためのインターフェースを提供します。他の createuser や dropdb と同様、libpq を利用してサーバーと通信するフロントエンドツール群の一つです。

責務

  • コマンドライン引数の解析（getopt_long を使用）。
  • 接続先データベース（メンテナンス用DB）への接続確立。
  • ユーザー指定のオプション（所有者、テーブルスペース、ロケール、エンコーディング等）に基づいた CREATE DATABASE SQL 文の構築。
  • サーバーへの SQL 送信と実行結果の検証。
  • オプションで指定されたデータベースコメントの追加（COMMENT ON DATABASE）。

重要なシンボル

  • main: プログラムのメインエントリポイント。引数解析から SQL 実行までの一連の処理を制御します。
  • help: コマンドラインオプションのヘルプメッセージを表示します。
  • long_options: getopt_long で使用される、コマンドライン引数の定義構造体。

制御フローとデータフロー

1. **初期化**: pg_logging_init 等で環境を整え、getopt_long でオプションを解析します。 2. **接続**: connectMaintenanceDatabase を呼び出し、サーバーへの接続を確立します。 3. **SQL 生成**: PQExpBuffer を使用して、解析したオプションを基に CREATE DATABASE 文を動的に構築します。 4. **実行**: PQexec で SQL を送信します。成功後、必要に応じて COMMENT ON DATABASE 文を同様の手順で実行します。 5. **終了**: PQfinish で接続を閉じ、終了ステータスを返します。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド開発用の基本ヘッダー。
  • common.h: src/bin/scripts 共通のユーティリティ関数群。
  • common/logging.h: ログ出力用インターフェース。
  • fe_utils/option_utils.h: コマンドラインオプション処理の補助。
  • fe_utils/string_utils.h: 文字列操作および SQL エスケープの補助。

関連コード

  • src/bin/scripts/common.c: 接続処理や共通のヘルパー関数が定義されており、createdb の動作の基盤となっています。
  • libpq: PostgreSQL の C 言語用クライアントライブラリ。PQexec や PQExpBuffer などの通信・バッファ操作の根幹です。

読者向けメモ

  • 本ファイルは SQL を直接構築するため、fmtId や appendStringLiteralConn を使用して SQL インジェクションを防ぐ設計になっています。
  • データベース名が指定されない場合、デフォルトで現在の OS ユーザー名が使用される仕様です。
  • ロケール関連のオプション（--locale, --icu-locale 等）が非常に多いため、新しいロケール設定を追加する際は main 内の switch 文と help 関数双方の更新が必要です。
