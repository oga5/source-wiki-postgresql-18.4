src/bin/scripts/reindexdb.c

概要

reindexdb は、PostgreSQL データベース内のインデックスを再構築するためのコマンドラインユーティリティです。SQL の REINDEX コマンドをラップし、データベース全体、特定のテーブル、スキーマ、またはインデックス単位での再構築をサポートします。

モジュール内での役割

src/bin/scripts モジュールの一部として、データベース管理者が運用保守を行うためのクライアントツールを提供します。他の vacuumdb や clusterdb と同様に、fe_utils ライブラリを活用して接続管理や並列処理を実現しています。

責務

  • コマンドライン引数の解析と、対象オブジェクト（DB、テーブル、インデックス等）の特定。
  • REINDEX SQL 文の生成と発行。
  • 並列実行（-j オプション）時の接続管理とタスクのスケジューリング。
  • サーバーバージョンに応じた機能（CONCURRENTLY や TABLESPACE）の妥当性チェック。

重要なシンボル

  • main: 引数解析と実行モード（単一DB vs 全DB）の決定を行うエントリーポイント。
  • ReindexType: 再構築対象の種類を定義する列挙型。
  • reindex_one_database: 特定のデータベースに対する再構築処理のメインループ。
  • gen_reindex_command: 実行する REINDEX SQL 文を構築するヘルパー。
  • ParallelSlotArray: fe_utils を利用した並列実行のための接続管理構造体。

制御フローとデータフロー

1. **初期化**: main で引数を解析し、対象が単一DBか全DBかを判定。 2. **接続**: connectDatabase を使用して対象DBへ接続。 3. **リスト構築**: 並列実行が指定されている場合、get_parallel_tables_list や get_parallel_tabidx_list を呼び出し、カタログから対象オブジェクトのリストを取得・ソートする。 4. **実行**: ParallelSlots を用いて、空きスロットがある限り run_reindex_command で SQL を非同期発行。 5. **完了**: 全てのタスクが終了するまで待機し、エラーがあれば終了コードを返す。

読者向けメモ

  • **並列実行の制約**: REINDEX SYSTEM は並列実行できません。また、インデックス単位の並列再構築では、同一テーブルに属するインデックスが同じジョブで処理されるよう、get_parallel_tabidx_list でソートとグループ化が行われています。
  • **SQL 生成**: gen_reindex_command は、サーバーバージョンに応じて CONCURRENTLY や TABLESPACE オプションの配置を調整します。古いサーバーに対して無効なオプションが渡された場合は pg_fatal で停止します。
  • **拡張性**: 新しい REINDEX オプションを追加する場合は、main の getopt_long と gen_reindex_command の両方を更新する必要があります。
