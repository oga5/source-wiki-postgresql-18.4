src/bin/scripts/pg_isready.c

概要

pg_isready は、PostgreSQL サーバーが接続を受け入れ可能な状態にあるかを判定するコマンドラインユーティリティです。サーバーの稼働状況を監視するスクリプトや、起動待ち合わせの自動化ツールでの利用を想定しています。

モジュール内での役割

src/bin/scripts ディレクトリに配置されたクライアントツールの一つです。他のツール（vacuumdb 等）がデータベース操作を行うのに対し、本ファイルは接続の可否確認のみに特化しています。

責務

  • コマンドライン引数（ホスト、ポート、ユーザー、タイムアウト等）の解析。
  • libpq の PQpingParams を利用したサーバーへの疎通確認。
  • 接続状態に応じた終了ステータス（PQPING_*）の返却。
  • ユーザーへの接続結果の出力（--quiet オプション指定時は抑制）。

重要なシンボル

  • main: プログラムのメインエントリポイント。引数解析から PQpingParams の呼び出しまでを制御します。
  • PQpingParams: サーバーへの ping を実行する libpq 関数。
  • DEFAULT_CONNECT_TIMEOUT: 接続試行のデフォルトタイムアウト値（3秒）。
  • PARAMS_ARRAY_SIZE: 接続パラメータを保持する配列のサイズ。

追加の索引済みシンボル

  • help

制御フローとデータフロー

1. **初期化**: pg_logging_init 等で環境をセットアップし、getopt_long で引数を解析します。 2. **パラメータ構築**: 指定された接続情報を keywords と values 配列に格納します。 3. **接続情報の解決**: PQconndefaults を使用して、明示的な指定がない場合のホストやポートのデフォルト値を特定します。 4. **疎通確認**: PQpingParams を実行し、サーバーの応答を rv に格納します。 5. **結果出力と終了**: quiet モードでなければ結果を標準出力に表示し、rv を終了コードとしてプロセスを終了します。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用共通ヘッダー。
  • common.h: スクリプト共通のユーティリティ関数群。
  • common/logging.h: ログ出力用インターフェース。
  • fe_utils/option_utils.h: コマンドラインオプション解析の補助。

関連コード

  • libpq: 本ファイルが依存する PostgreSQL クライアントライブラリ。PQpingParams や PQconndefaults の詳細は libpq のドキュメントを参照してください。
  • src/bin/scripts/common.c: 他のスクリプトツールと共有される共通処理が定義されています。

読者向けメモ

  • 本プログラムは、データベースへの接続を確立するのではなく、サーバーが接続を受け付けているかを確認するだけである点に注意してください。
  • 終了ステータスは libpq-fe.h で定義される PGPing 列挙型と一致しており、シェルスクリプト等で if pg_isready; then ... のように直接利用可能です。
  • 接続文字列（postgresql://...）が dbname 引数に渡された場合、PQconninfoParse を介して適切にパースされる設計になっています。
