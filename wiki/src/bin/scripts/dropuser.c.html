src/bin/scripts/dropuser.c

概要

PostgreSQL のロールを削除するためのコマンドラインユーティリティ dropuser の実装です。SQL の DROP ROLE コマンドをラップし、対話的な確認や接続オプションの管理を簡素化します。

モジュール内での役割

src/bin/scripts モジュールの一部として、データベース管理者がコマンドラインから直接ロールを削除するためのインターフェースを提供します。他の dropdb や createuser と同様、共通の接続管理ロジックを利用して実行されます。

責務

  • コマンドライン引数の解析（接続先、ロール名、対話モード、IF EXISTS オプション）。
  • 削除対象ロール名の取得（引数がない場合は対話的にプロンプトを表示）。
  • 実行前の対話的な確認（--interactive 指定時）。
  • 適切な DROP ROLE SQL 文の構築と実行。

重要なシンボル

  • main: プログラムのエントリポイント。引数解析、接続確立、SQL 実行の全フローを制御します。
  • if_exists: --if-exists オプションの状態を保持する静的変数。
  • help: ユーザー向けのヘルプメッセージを表示する関数。
  • ConnParams: 接続パラメータを保持する構造体（common.h で定義）。

制御フローとデータフロー

1. **初期化**: pg_logging_init 等で環境を整え、getopt_long でオプションを解析します。 2. **引数検証**: 削除対象のロール名が指定されているか確認し、不足している場合は対話的に入力を求めます。 3. **接続**: connectMaintenanceDatabase を呼び出し、PostgreSQL サーバーへの接続を確立します。 4. **SQL 生成**: PQExpBuffer を使用して DROP ROLE [IF EXISTS] <name>; を構築します。fmtIdEnc を介してロール名を適切にエスケープします。 5. **実行**: PQexec で SQL を送信し、結果を確認して終了します。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用共通ヘッダー。
  • common.h: 接続管理や共通ユーティリティの定義。
  • common/logging.h: ログ出力用。
  • fe_utils/option_utils.h: コマンドラインオプション解析用。
  • fe_utils/string_utils.h: 文字列操作および識別子のエスケープ用。
  • common/string.h

関連コード

  • src/bin/scripts/common.c: connectMaintenanceDatabase や yesno_prompt など、本ファイルが依存する共通関数群が定義されています。
  • src/bin/scripts/createuser.c: 対照的な操作を行うツールであり、実装パターンが類似しています。

読者向けメモ

  • 本ツールは DROP ROLE を実行するものであり、DROP USER と同義です。
  • fmtIdEnc による識別子のエスケープ処理は、SQL インジェクションを防ぐために重要です。
  • 接続先データベースは connectMaintenanceDatabase によって自動的に決定されます（通常は postgres データベース）。
