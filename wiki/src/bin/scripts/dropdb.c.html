src/bin/scripts/dropdb.c

概要

dropdb.c は、PostgreSQL データベースを削除するためのコマンドラインユーティリティ dropdb の実装です。SQL の DROP DATABASE コマンドをラップし、接続管理や対話的な確認処理を簡略化します。

モジュール内での役割

src/bin/scripts モジュールの一部として、データベース管理タスクを自動化するクライアントツール群の一つです。他の createdb や reindexdb と同様に、fe_utils ライブラリを活用して一貫した接続オプションとエラーハンドリングを提供します。

責務

  • コマンドライン引数の解析（接続先、データベース名、削除オプションの指定）。
  • 削除対象データベースへの接続管理（メンテナンス用データベースへの接続）。
  • 削除前の対話的確認（-i オプション時）。
  • DROP DATABASE SQL 文の構築と実行。
  • 実行結果のフィードバックとエラー報告。

重要なシンボル

  • main: プログラムのエントリポイント。引数解析から SQL 実行までの一連の流れを制御します。
  • help: ユーザー向けのコマンド使用方法を表示する関数。
  • ConnParams: 接続パラメータを保持する構造体（common.h で定義）。
  • PQExpBuffer: SQL 文を動的に構築するためのバッファ構造体。

制御フローとデータフロー

1. **初期化**: pg_logging_init 等で環境を整え、getopt_long で引数を解析します。 2. **検証**: データベース名が指定されているか確認し、interactive モードであればユーザーの入力を待ちます。 3. **接続**: connectMaintenanceDatabase を呼び出し、適切なメンテナンス用 DB（デフォルトは postgres）へ接続します。 4. **SQL 生成**: fmtIdEnc を使用してデータベース名をエスケープし、IF EXISTS や FORCE オプションを付与した DROP DATABASE 文を構築します。 5. **実行**: PQexec でサーバーへコマンドを送信し、結果を確認して終了します。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用共通ヘッダー。
  • common.h: ユーティリティ関数や ConnParams 等の定義。
  • common/logging.h: ログ出力用インターフェース。
  • fe_utils/option_utils.h: コマンドラインオプション解析の補助。
  • fe_utils/string_utils.h: 文字列操作および識別子エスケープ用。

関連コード

  • src/bin/scripts/common.c: connectMaintenanceDatabase や yesno_prompt など、このディレクトリ内のツールで共有される共通関数が実装されています。
  • src/bin/scripts/createdb.c: 対照的な操作を行うツールであり、実装パターンが類似しています。

読者向けメモ

  • **接続先**: dropdb は削除対象のデータベースそのものには接続できません。そのため、デフォルトで postgres データベースに接続し、そこから DROP DATABASE を発行します。
  • **セキュリティ**: fmtIdEnc を使用してデータベース名を適切にエスケープしているため、SQL インジェクションのリスクは排除されています。
  • **拡張性**: 新しいオプションを追加する場合は、long_options 配列と getopt_long の switch 文を更新し、SQL 生成ロジックに反映させる必要があります。
