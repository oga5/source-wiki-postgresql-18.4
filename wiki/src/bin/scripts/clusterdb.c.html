src/bin/scripts/clusterdb.c

概要

clusterdb は、PostgreSQL データベース内のテーブルを物理的に再配置（クラスタリング）するためのコマンドラインユーティリティです。SQL の CLUSTER コマンドをラップし、単一データベースまたは全データベースに対して一括で実行する機能を提供します。

モジュール内での役割

src/bin/scripts モジュールの一部として、データベース管理者がシェルから直接実行するメンテナンスツール群を構成します。vacuumdb や reindexdb と同様の設計パターンに従い、共通の接続管理やオプション解析ロジックを共有しています。

責務

  • コマンドライン引数（対象DB、テーブル、接続オプション）の解析。
  • 接続先データベースの決定と接続の確立。
  • CLUSTER SQL コマンドの構築と実行。
  • 全データベース対象時の対象リスト取得とループ処理。

重要なシンボル

  • main: 引数解析、接続パラメータの初期化、および実行モード（単一/全DB）の分岐を行うエントリポイント。
  • cluster_one_database: 特定のデータベースに対して CLUSTER コマンドを発行する。
  • cluster_all_databases: pg_database をクエリし、接続可能な全データベースに対して順次 cluster_one_database を呼び出す。

追加の索引済みシンボル

  • help

制御フローとデータフロー

1. **初期化**: getopt_long を使用してオプションを解析し、ConnParams 構造体に接続情報を格納。 2. **分岐**:

  • --all 指定時: cluster_all_databases が呼び出され、メンテナンスDBから対象DB一覧を取得。
  • 通常時: 指定された（またはデフォルトの）DBに対して cluster_one_database を実行。

    • 3. **実行**: cluster_one_database 内で PQExpBuffer を用いて CLUSTER [VERBOSE] [table_name]; を構築し、executeMaintenanceCommand を介してサーバーへ送信。 4. **終了**: 処理完了後、接続をクローズし終了コードを返す。

関連コード

  • src/bin/scripts/common.c: 接続管理やコマンド実行の共通実装が含まれており、本ファイルの動作の基盤となっています。
  • src/bin/scripts/vacuumdb.c / reindexdb.c: 類似の設計思想で実装されたツールであり、比較することで一貫した実装パターンを理解できます。

読者向けメモ

  • 本ファイルは libpq を直接操作するのではなく、fe_utils 経由で抽象化されたインターフェースを利用しています。
  • CLUSTER コマンドはテーブルのロックを伴うため、実行時には対象テーブルのアクセス状況に注意が必要です。
  • データベース名が指定されない場合、環境変数 PGDATABASE や現在のユーザー名がデフォルトとして使用されます。
