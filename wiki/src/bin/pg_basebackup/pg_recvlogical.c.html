src/bin/pg_basebackup/pg_recvlogical.c

概要

pg_recvlogical は、PostgreSQL の論理デコーディングストリームを制御し、受信したデータをローカルファイルまたは標準出力に書き出すためのクライアントツールです。

モジュール内での役割

src/bin/pg_basebackup モジュールの一部として、物理的なバックアップ（pg_basebackup）や WAL の受信（pg_receivewal）とは異なり、論理レプリケーションスロットを介したデータストリーミングに特化しています。

責務

  • **スロット管理**: 論理レプリケーションスロットの作成、削除、およびストリーミング開始の制御。
  • **データストリーミング**: サーバーからの論理デコーディングデータを COPY プロトコル経由で受信し、ファイルへ書き出す。
  • **フィードバック制御**: 受信位置（LSN）をサーバーに定期的に通知し、スロットの進捗を維持する。
  • **ライフサイクル管理**: 接続の維持、再接続ロジック、およびシグナルによる安全な終了処理。

重要なシンボル

  • StreamLogicalLog(): ストリーミングのメインループ。COPY プロトコルによるデータ受信とファイル書き込みを行う。
  • sendFeedback(): サーバーへ現在の受信・フラッシュ位置を通知する。
  • output_written_lsn / output_fsync_lsn: 現在の書き込みおよびフラッシュ済み LSN を保持するグローバル変数。
  • time_to_abort: シグナル受信時にループを終了させるフラグ。

制御フローとデータフロー

1. **初期化**: コマンドライン引数を解析し、GetConnection() でサーバーと接続。 2. **コマンド実行**: START_REPLICATION ... LOGICAL コマンドを送信し、COPY_BOTH モードへ移行。 3. **メインループ**:

  • select() を使用してソケットを監視し、データ受信またはタイムアウト（フィードバック送信や fsync のタイミング）を待機。
  • 受信した XLogData メッセージを解析し、ファイルへ書き込む。
  • fsync_interval に基づき、定期的に fsync() を実行してディスクへ永続化。

    • 4. **終了処理**: シグナル受信や endpos 到達時に prepareToTerminate() を呼び出し、サーバーへ終了を通知して接続を閉じる。

読者向けメモ

  • 本ファイルは COPY プロトコルを直接操作しているため、libpq の COPY 関連 API（PQgetCopyData 等）の挙動を理解することが重要です。
  • fsync のタイミングは fsync_interval オプションで制御されており、データ損失を防ぐための重要な設計ポイントです。
  • Windows 環境ではシグナルハンドリングが制限されるため、#ifndef WIN32 で囲まれたハンドラの実装に注意してください。
