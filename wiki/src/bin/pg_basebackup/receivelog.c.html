src/bin/pg_basebackup/receivelog.c

概要

PostgreSQLのストリーミングレプリケーションプロトコルを使用して、WAL（Write-Ahead Log）データをリモートサーバーから受信し、ローカルストレージに書き出すためのコアロジックを提供します。

モジュール内での役割

pg_basebackup や pg_receivewal といったツールが、サーバーからWALを継続的に取得する際のバックエンドエンジンとして機能します。streamutil.c で定義された抽象化レイヤー（WalWriteMethod）を利用し、ファイルシステムやアーカイブへの書き込みを抽象化します。

責務

  • **WALストリームの管理**: START_REPLICATION コマンドの発行と、COPY モードでのデータ受信。
  • **WALファイルのライフサイクル管理**: セグメント単位でのファイルのオープン、書き込み、同期（fsync）、クローズ、およびアーカイブ完了フラグの作成。
  • **フィードバック制御**: サーバーへの進捗報告（Standby Status Update）の送信と、同期レプリケーション時のフラッシュ位置の通知。
  • **タイムライン追跡**: タイムライン切り替え時の履歴ファイル取得と、新しいタイムラインへのシームレスな移行。

重要なシンボル

  • ReceiveXlogStream: ストリーミングのメインエントリーポイント。接続確立後の全プロセスを制御します。
  • StreamCtl: ストリーミングの動作（開始位置、タイムライン、同期設定、停止条件など）を定義する構造体。
  • HandleCopyStream: COPY プロトコルによるデータ受信ループを管理する中心関数。
  • walfile: 現在書き込み中のWALファイルハンドルを保持する静的変数。

制御フローとデータフロー

1. **初期化**: ReceiveXlogStream がサーバーバージョンを確認し、START_REPLICATION を発行。 2. **受信ループ**: HandleCopyStream が CopyStreamReceive を介してサーバーからのメッセージを待機。 3. **メッセージ処理**:

  • w (XLogData): WALデータを受信し、open_walfile / close_walfile を経由してディスクへ書き込み。
  • k (Keepalive): サーバーからの生存確認に応答し、必要に応じてフィードバックを送信。

    • 4. **終了処理**: 停止条件（stream_stop コールバック）を満たすか、サーバーから終了通知を受けた際に COPY モードを終了。

読者向けメモ

  • **抽象化の理解**: walmethod->ops を通じた操作は、単なるファイル書き込みだけでなく、圧縮やtarアーカイブへの変換も含まれるため、直接的な write() 呼び出しを探さないように注意してください。
  • **停止条件**: stream_stop コールバックは、ストリーミングをいつ終了させるかを決定する重要なフックです。この関数が true を返すと、ストリーミングは正常終了します。
  • **同期書き込み**: synchronous フラグが有効な場合、WALデータが書き込まれるたびに fsync が発生するため、パフォーマンスに大きな影響を与えます。
