src/bin/pg_test_timing/pg_test_timing.c

概要

pg_test_timing.c は、PostgreSQL が動作する環境におけるシステムクロックの精度とオーバーヘッドを測定するためのコマンドラインツールです。特に、システム時刻が単調増加（monotonic）であるかを確認し、時刻取得処理にかかるコストを評価します。

モジュール内での役割

本ファイルは pg_test_timing モジュールの唯一のソースコードであり、独立した実行バイナリを生成します。データベースサーバーの外部から、OS レベルの時刻取得機能の信頼性を検証する役割を担います。

責務

  • **時刻取得のオーバーヘッド測定**: 指定された時間内での時刻取得ループ回数を計測し、1 回あたりの平均コストを算出する。
  • **単調性の検証**: 時刻取得時に値が過去に戻る（Time warp）現象が発生しないかを監視する。
  • **分布の可視化**: 時刻取得にかかった時間の分布を 2 のべき乗単位のヒストグラムとして出力する。

重要なシンボル

  • test_timing(): 計測のメインループを実行する関数。instr_time を用いて時刻を繰り返し取得し、差分を計算する。
  • histogram[]: 時刻差分を 2 のべき乗（マイクロ秒単位）で分類して格納する配列。
  • INSTR_TIME_* マクロ: portability/instr_time.h で定義された、プラットフォーム非依存の時刻取得・演算用マクロ。

制御フローとデータフロー

1. **初期化**: main が引数を解析し、テスト期間（デフォルト 3 秒）を決定する。 2. **計測ループ**: test_timing 内で INSTR_TIME_SET_CURRENT を呼び出し、現在時刻を継続的に取得する。

  • 前回の時刻との差分（diff）を計算。
  • diff < 0 の場合は時刻の逆転とみなし、エラー終了する。
  • diff のビット長を計算し、対応する histogram のバケットをインクリメントする。

    • 3. **集計と出力**: ループ終了後、総経過時間と総ループ回数から平均オーバーヘッドを算出し、output 関数がヒストグラムを整形して表示する。

直接依存

  • postgres_fe.h: フロントエンド用共通ヘッダー。
  • getopt_long.h: コマンドライン引数解析用。
  • portability/instr_time.h: プラットフォームごとの高精度タイマー抽象化レイヤー。
  • limits.h

関連コード

  • src/include/portability/instr_time.h: INSTR_TIME_* マクロの定義元。各プラットフォームでの時刻取得の実装詳細を確認する際に参照する。

読者向けメモ

  • 本ツールは、PostgreSQL のパフォーマンスチューニングや、特定の環境で時刻取得がボトルネックになっていないかを調査する際に有用です。
  • diff が 0 になることは、システムクロックの分解能が測定ループの実行速度よりも粗いことを示唆しています。
  • ヒストグラムのバケットは 2 のべき乗（1, 2, 4, 8... μs）で構成されており、システムクロックの「跳ね」や「遅延」を視覚的に把握できるよう設計されています。
