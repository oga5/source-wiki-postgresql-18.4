src/bin/pgbench/pgbench.h

概要

pgbench.h は、PostgreSQL のベンチマークツールである pgbench における式評価エンジン（Expression Evaluation Engine）のコア定義を保持するヘッダーファイルです。主にスクリプト内の式を解析・評価するためのデータ構造と、パーサー・レクサーのインターフェースを定義しています。

モジュール内での役割

pgbench のメインロジックである pgbench.c が利用する、式解析のための共通定義を提供します。このファイルは、exprscan.l（レクサー）や exprparse.y（パーサー）と密接に連携し、ベンチマークスクリプト内の計算式を抽象構文木（AST）として表現するための基盤となります。

責務

  • **型定義の提供**: ベンチマーク実行中に扱われる値（PgBenchValue）およびその型（PgBenchValueType）の定義。
  • **AST 構造の定義**: 式を構成するノード（PgBenchExpr）および関数呼び出しのリスト構造の定義。
  • **パーサーインターフェースの公開**: exprparse.y および exprscan.l が生成する関数群の宣言。
  • **ユーティリティ宣言**: 文字列から数値への変換や、構文エラー報告のための共通関数の提供。

重要なシンボル

  • PgBenchValue: 式評価の結果を保持する構造体。int64, double, bool をサポート。
  • PgBenchExpr: 式のノード。定数、変数、関数呼び出しのいずれかとして表現される。
  • PgBenchFunction: サポートされている演算子や組み込み関数（ADD, RANDOM, HASH_FNV1A 等）の列挙型。
  • yyscan_t: Flex が生成するスキャナー状態を保持するための型定義（void *）。

制御フローとデータフロー

本ファイルはデータ構造の定義が主であり、直接的な制御フローは持ちません。 1. **解析フェーズ**: expr_scanner_init でスキャナーを初期化し、expr_yyparse を呼び出すことで、入力文字列が PgBenchExpr のツリー構造に変換されます。 2. **評価フェーズ**: 生成された PgBenchExpr ツリーを pgbench.c 側で再帰的に辿り、PgBenchValue を計算します。

直接依存

  • fe_utils/psqlscan.h: SQL スキャン状態を管理するための共通ユーティリティ。

関連コード

  • src/bin/pgbench/exprparse.y: 本ヘッダーで定義された構造体を使用して、式を構文解析するパーサー。
  • src/bin/pgbench/exprscan.l: 式をトークン化するレクサー。
  • src/bin/pgbench/pgbench.c: 本ヘッダーの定義を利用して、ベンチマークの実行と式の評価を行うメインプログラム。

読者向けメモ

  • **Flex/Bison との統合**: yyscan_t を void * として再定義しているのは、Flex の内部定義を直接参照せずに型安全性を保つための工夫です。
  • **拡張性**: 新しい関数や演算子を追加する場合は、PgBenchFunction 列挙型にシンボルを追加し、評価ロジックを pgbench.c に実装する必要があります。
  • **型安全性**: PgBenchValue は共用体（union）を使用しているため、アクセス時には必ず type フィールドを確認してください。
