src/bin/pgbench/pgbench.c

概要

pgbench.c は、PostgreSQL の負荷テストツール pgbench のメインプログラムです。マルチスレッドによる並行クライアントの実行、SQL スクリプトの解析・実行、統計情報の収集、およびデータベースの初期化を担います。非常に大規模なファイルであり、ステートマシンによる非同期 I/O 処理と、式評価エンジンが中心的な役割を果たします。

モジュール内での役割

pgbench モジュールの中核であり、pgbench.h で定義されたデータ構造と式評価エンジンを利用してベンチマークを実行します。exprparse.y および exprscan.l と連携してスクリプトを解釈し、libpq を介してデータベースと通信します。

責務

  • **実行制御**: スレッド管理、クライアントの並行実行、および poll/select を用いた非同期 I/O 多重化。
  • **スクリプト実行**: SQL およびメタコマンド（\set, \if 等）の解釈と実行。
  • **式評価**: スクリプト内の計算式を AST として評価。
  • **統計収集**: トランザクションのレイテンシ、TPS、エラー発生率の計測とレポート出力。
  • **データベース初期化**: ベンチマーク用スキーマの構築および高速なデータロード。

重要なシンボル

状態管理

  • CState: 個々のクライアント接続のコンテキスト。状態遷移、変数、統計情報、接続ハンドルを保持。
  • TState: スレッドごとの実行コンテキスト。複数の CState を管理し、スレッド固有の乱数生成器や統計情報を保持。
  • ConnectionStateEnum: クライアントのライフサイクル状態（CSTATE_START_TX, CSTATE_WAIT_RESULT 等）。

実行・評価

  • advanceConnectionState(): クライアントのステートマシンを駆動する中心関数。
  • evaluateExpr(): PgBenchExpr を再帰的に評価し、PgBenchValue を返す。
  • executeMetaCommand(): \set や \sleep などのメタコマンドを実行。

データ構造

  • PgBenchValue: 式評価の結果を保持する共用体（int64, double, bool）。
  • Command: スクリプト内の個々のコマンド（SQL またはメタコマンド）の定義。
  • StatsData: トランザクションの成功・失敗・レイテンシなどの統計情報を集計する構造体。

制御フローとデータフロー

1. **初期化**: main() で引数を解析し、スレッドとクライアントを初期化。 2. **スレッド実行**: threadRun() が各スレッドのメインループとなり、poll/select でソケットを監視。 3. **状態遷移**: イベント発生時に advanceConnectionState() が呼ばれ、CState の状態を更新。 4. **クエリ送信**: sendCommand() が libpq を介してクエリを送信。 5. **結果受信**: readCommandResponse() が結果を解析し、\gset 等の変数代入を行う。 6. **終了処理**: 全クライアント完了後、printResults() が統計情報を集計して出力。

読者向けメモ

  • **メモリ管理**: CState や TState は動的に確保されます。pg_malloc 系関数が使用されており、エラー時は pg_fatal で終了します。
  • **非同期 I/O**: poll/select の抽象化層（socket_set）が実装されており、プラットフォーム依存の I/O 多重化を吸収しています。
  • **式評価**: evaluateExpr は再帰的です。複雑な式を評価する際はスタックオーバーフローに注意が必要ですが、実用上の制限は MAX_FARGS 等で管理されています。
  • **未実装・浅い部分**: pgbench.c は非常に巨大であるため、機能ごとにファイルを分割するリファクタリングが推奨されます。特に式評価ロジックとステートマシンロジックの分離が課題です。
