src/bin/pg_test_fsync/pg_test_fsync.c

概要

pg_test_fsync は、PostgreSQL がサポートする様々な fsync 手法（fsync, fdatasync, open_datasync, open_sync 等）のパフォーマンスを比較・測定するためのコマンドラインツールです。

モジュール内での役割

本ファイルは src/bin/pg_test_fsync ディレクトリの唯一のソースであり、PostgreSQL の WAL（Write Ahead Log）書き込みにおいて、どの同期手法が最も効率的かをユーザーが判断するためのベンチマークを提供します。

責務

  • 指定された期間（デフォルト5秒）の各同期手法の実行回数（ops/sec）を計測する。
  • OS やファイルシステムがサポートする同期オプションの可否を判定する。
  • 異なる書き込みサイズや、ファイル記述子を跨いだ同期の挙動を検証する。
  • テスト用の一時ファイルを作成・管理し、終了時にクリーンアップする。

重要なシンボル

  • test_sync(): 主要な同期手法（open_datasync, fdatasync, fsync 等）を順次実行するメインループ。
  • START_TIMER / STOP_TIMER: alarm() またはスレッドを用いた時間計測用マクロ。
  • alarm_triggered: 計測期間の終了を通知するシグナルフラグ。
  • pg_pwrite(): ファイルの特定オフセットへ書き込むためのラッパー。

制御フローとデータフロー

1. **初期化**: handle_args() で引数を解析し、prepare_buf() でランダムデータを含むバッファを準備。 2. **シグナル設定**: SIGINT 等の割り込みを捕捉し、一時ファイルを削除する signal_cleanup を登録。 3. **テスト実行**:

  • test_open(): 基本的な書き込みと同期の可否を確認。
  • test_sync(): 1ブロックおよび2ブロック書き込みで各手法を比較。
  • test_open_syncs(): 書き込みサイズを変えて open_sync のコストを評価。
  • test_file_descriptor_sync(): 異なるファイル記述子間での同期挙動を検証。
  • test_non_sync(): 同期なしのベースラインを測定。

    • 4. **終了**: 一時ファイルを削除して終了。

読者向けメモ

  • **プラットフォーム差異**: Windows と Unix 系でタイマー処理（alarm vs CreateThread）が大きく異なります。
  • **O_DIRECT**: O_DIRECT をサポートする環境では、ファイルシステムやマウントオプションによって書き込みが失敗する可能性があるため、結果の n/a* 表示に注意してください。
  • **計測の正確性**: 本ツールはあくまで「同期手法の比較」を目的としており、実際のデータベース負荷を完全にシミュレートするものではありません。
