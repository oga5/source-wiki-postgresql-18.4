src/bin/pg_rewind/local_source.c

概要

local_source.c は、pg_rewind においてソース（同期元）となるデータディレクトリがローカルファイルシステム上に存在する場合のデータアクセス手段を提供します。

モジュール内での役割

pg_rewind はソースデータへのアクセスを抽象化しており、本ファイルはその「ローカルディレクトリ版」の実装です。libpq_source.c がリモート接続（ネットワーク経由）を担当するのに対し、本ファイルは直接的なファイル I/O を担当します。

責務

  • rewind_source インターフェースの具現化。
  • ローカルデータディレクトリ内のファイル走査（traverse_datadir のラップ）。
  • ソースファイルからターゲットディレクトリへのデータコピー（ファイル全体または指定範囲）。
  • 読み込み中のエラーハンドリングと整合性チェック。

重要なシンボル

  • init_local_source: local_source 構造体を初期化し、関数ポインタをセットアップするコンストラクタ。
  • local_source: ソースディレクトリのパスを保持するプライベートな構造体。
  • local_queue_fetch_file / local_queue_fetch_range: ソースからデータを読み込み、ターゲットへ書き出すメイン処理。

追加の索引済みシンボル

  • local_destroy
  • local_fetch_file

制御フローとデータフロー

1. **初期化**: init_local_source が呼び出され、rewind_source インターフェースに関数ポインタが登録されます。 2. **走査**: local_traverse_files がディレクトリを再帰的に探索し、各ファイルに対してコールバックを実行します。 3. **データ転送**:

  • local_queue_fetch_file はファイルを全コピーします。
  • local_queue_fetch_range は lseek を使用して特定のオフセットから指定バイト数のみを読み込みます。

    • 4. **同期**: 読み込んだデータは file_ops.c の write_target_range を通じてターゲットディレクトリへ書き込まれます。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用共通ヘッダー。
  • file_ops.h: ターゲットディレクトリへの書き込み操作（open_target_file, write_target_range 等）。
  • rewind_source.h: 抽象インターフェース定義。
  • fcntl.h, unistd.h: 低レベルファイル I/O 操作。
  • common/logging.h

関連コード

  • libpq_source.c: 同等の機能をネットワーク経由で提供する対照的な実装。
  • file_ops.c: 本ファイルが依存するターゲット側のファイル操作の実装。
  • rewind_source.h: rewind_source 構造体の定義とインターフェース仕様。

読者向けメモ

  • 本ファイルの実装は同期的です。local_queue_fetch_range 等は呼び出された時点で即座にファイル I/O を完了させます。
  • local_finish_fetch は空の実装ですが、これは非同期的なフェッチを行う他のソース実装（将来的な拡張やリモートソース）との互換性を保つためのインターフェース上の要請です。
  • ソースファイルが読み込み中に変更されることは想定されておらず、サイズ不一致が発生した場合は pg_fatal で即座に終了します。
