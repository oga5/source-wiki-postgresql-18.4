src/bin/psql/command.c

概要

command.c は、psql のメタコマンド（バックスラッシュコマンド、例: \d, \c, \pset）の解析と実行を統括する中心的なファイルです。ユーザーが入力したコマンドを解析し、適切なサブルーチンへディスパッチする役割を担います。

モジュール内での役割

psql のメインループ (mainloop.c) から呼び出される HandleSlashCmds を起点とし、各メタコマンドのロジックを分離・管理します。command.h で定義された backslashResult を用いて、コマンド実行後の制御フロー（クエリ送信、終了、エラーなど）をメインループに伝達します。

責務

  • **コマンドディスパッチ**: 入力されたバックスラッシュコマンドを解析し、対応する exec_command_* 関数へ振り分ける。
  • **条件付き実行の管理**: \if, \elif, \else, \endif による条件分岐のスタック状態を管理し、非アクティブなブロック内のコマンドを適切にスキップする。
  • **状態管理**: pset (PsqlSettings) を介した出力設定、接続情報、変数、およびパイプラインモードの状態更新。
  • **外部リソース操作**: ファイルの読み込み (\i)、エディタの起動 (\e)、シェルコマンドの実行 (\!)。

重要なシンボル

ディスパッチ・制御

  • HandleSlashCmds: メタコマンド処理のメインエントリーポイント。
  • exec_command: コマンド名に基づき各サブルーチンを呼び出すディスパッチャ。
  • backslashResult: コマンド実行後の動作（PSQL_CMD_SEND, PSQL_CMD_TERMINATE 等）を規定する列挙型。

コマンド実行サブルーチン

  • exec_command_d: \d 系コマンド（オブジェクト表示）の解析とディスパッチ。
  • exec_command_connect: \c によるデータベース接続管理。
  • exec_command_pset: \pset による出力フォーマット設定。
  • exec_command_edit: \e によるクエリバッファの編集。
  • exec_command_if / elif / else / endif: 条件付き実行スタックの操作。

状態管理・ユーティリティ

  • do_pset: pset 構造体の設定値を更新し、変更を反映する。
  • do_connect: データベース接続の確立と、接続失敗時のリトライロジック。
  • SyncVariables / UnsyncVariables: 接続状態に応じた内部変数の同期。

制御フローとデータフロー

1. **解析**: HandleSlashCmds が psql_scan_slash_command でコマンド名を特定。 2. **条件判定**: conditional_active(cstack) で現在のブロックが実行対象か確認。非アクティブな場合は ignore_slash_* 系関数で引数を読み飛ばす。 3. **実行**: exec_command が対応する関数を呼び出し、pset や query_buf を操作。 4. **結果返却**: backslashResult を通じて、メインループがクエリをサーバーへ送るべきか、あるいは入力を継続すべきかを判断。

読者向けメモ

  • **メモリ管理**: psql_scan_slash_option で取得した文字列は呼び出し側で free() する必要があります。
  • **制限モード**: restricted 変数が true の場合、\unrestrict 以外のコマンドは拒否されます。
  • **条件分岐**: \if ブロック内のコマンドは、非アクティブ時でも字句解析器を正しく進めるために ignore_slash_* 関数を通す必要があります。これを怠ると、次のコマンドの解析が正しく行われません。
  • **パイプラインモード**: PQpipelineStatus をチェックし、パイプラインモードで許可されていないコマンド（\gexec 等）を適切にエラーにする必要があります。
  • **省略箇所**: 本ファイルは非常に長いため、各 exec_command_* 関数の詳細な実装ロジックは省略していますが、命名規則により機能は明確です。
