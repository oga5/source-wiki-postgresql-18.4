src/bin/psql/large_obj.c

概要

本ファイルは、psql におけるラージオブジェクト（Large Object: LO）操作コマンド（\lo_export, \lo_import, \lo_unlink）の実装を提供します。

モジュール内での役割

psql のメタコマンド処理の一部として、データベース内のラージオブジェクトに対するファイル入出力および削除操作を抽象化します。libpq のラージオブジェクト関数を直接呼び出すためのラッパーとして機能します。

責務

  • **トランザクション管理**: ラージオブジェクト操作はトランザクション内で行う必要があるため、必要に応じて BEGIN/COMMIT/ROLLBACK を自動的に制御します。
  • **エラーハンドリング**: libpq からのエラーを捕捉し、適切なログ出力とトランザクションの終了処理を行います。
  • **結果出力**: 操作結果を標準出力またはログファイルへ整形して出力します。

重要なシンボル

  • do_lo_export: ファイルからラージオブジェクトをエクスポートします。
  • do_lo_import: ファイルをラージオブジェクトとしてインポートし、必要に応じてコメントを付与します。
  • do_lo_unlink: 指定された OID のラージオブジェクトを削除します。
  • start_lo_xact: 操作開始前にトランザクション状態を確認し、必要なら開始します。

追加の索引済みシンボル

  • fail_lo_xact
  • finish_lo_xact
  • print_lo_result

制御フローとデータフロー

1. **開始**: 各 do_lo_* 関数が start_lo_xact を呼び出し、現在のトランザクション状態を確認します。 2. **実行**: SetCancelConn で割り込みを許可し、libpq の lo_* 関数を実行します。 3. **終了**: 成功時は finish_lo_xact で必要に応じてコミットし、失敗時は fail_lo_xact でロールバックします。 4. **結果**: print_lo_result を通じて、HTML 形式や通常のテキスト形式で結果を報告します。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用共通ヘッダー。
  • large_obj.h: 本モジュールの公開インターフェース。
  • settings.h: pset 構造体（接続情報や出力設定）へのアクセス。
  • fe_utils/cancel.h: 操作中の割り込み制御。
  • common.h
  • common/logging.h

関連コード

  • src/bin/psql/large_obj.h: 公開関数の宣言。
  • src/bin/psql/command.c: メタコマンドのディスパッチ処理。
  • libpq (外部ライブラリ): lo_import, lo_export, lo_unlink 等の低レベル API を提供。

読者向けメモ

  • **トランザクションの重要性**: ラージオブジェクト操作は必ずトランザクションブロック内で行う必要があります。本ファイルは autocommit 設定を考慮し、ユーザーが明示的にトランザクションを開始していない場合に自動的に BEGIN を発行します。
  • **LASTOID 変数**: do_lo_import 成功時、生成された OID が LASTOID 変数にセットされます。スクリプト等で利用可能です。
  • **エラー処理**: libpq のラージオブジェクト関数は戻り値の定義が曖昧な場合があるため、PQerrorMessage を併用して詳細なエラーをログに出力しています。
