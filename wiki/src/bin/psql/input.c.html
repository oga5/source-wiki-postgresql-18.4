src/bin/psql/input.c

概要

input.c は、psql における対話的および非対話的な入力処理を抽象化するモジュールです。主に readline ライブラリとの統合、コマンド履歴の管理（保存・読み込み・エンコーディング）、および標準入力からの行読み込みを担当します。

モジュール内での役割

psql のメインループ（mainloop.c）がユーザー入力を受け取る際の窓口となります。readline が利用可能な環境では、コマンド補完や履歴操作を透過的に提供し、利用不可能な環境では標準的な fgets ベースの入力にフォールバックします。

責務

  • **入力の抽象化**: gets_interactive を通じて、環境に応じた適切な入力メソッドを提供。
  • **履歴管理**: ユーザーの入力履歴を readline の履歴リストへ追加し、終了時にファイル（.psql_history）へ永続化。
  • **マルチライン対応**: 履歴内の改行を特殊文字（NL_IN_HISTORY）に変換して保存し、読み込み時に復元することで、複数行のクエリを履歴として保持。
  • **シグナル制御**: 入力待機中に SIGINT を適切に処理するためのフラグ管理。

重要なシンボル

  • gets_interactive(): 対話モードでの入力取得関数。readline が有効ならそれを使用。
  • gets_fromFile(): ファイルまたは標準入力から行単位で読み込む汎用関数。
  • pg_append_history() / pg_send_history(): 入力されたクエリを履歴バッファに蓄積し、readline の履歴リストへコミットする一連の処理。
  • initializeInput(): 起動時に履歴ファイルパスの決定や readline の初期化を行う。
  • printHistory(): \s コマンド等で履歴をコンソールまたはファイルに出力する。

制御フローとデータフロー

1. **初期化**: initializeInput が環境変数 PSQL_HISTORY やホームディレクトリから履歴ファイルパスを特定し、read_history で読み込みます。 2. **入力**: gets_interactive が呼び出されると、readline が入力を待ち受けます。入力された行は pg_append_history でバッファリングされます。 3. **履歴コミット**: クエリ実行のタイミングで pg_send_history が呼ばれ、履歴の重複排除やスペース無視のフィルタリングを経て add_history に渡されます。 4. **終了**: atexit に登録された finishInput が、saveHistory を呼び出し、履歴をファイルへ書き出します。この際、encode_history により改行がエスケープされます。

読者向けメモ

  • **libedit への対応**: readline と libedit の挙動差異（特に履歴のイテレーション方向）を吸収するためのマクロ BEGIN_ITERATE_HISTORY が定義されています。この複雑なロジックは、移植性維持のための重要なハックです。
  • **SIGINT の扱い**: 入力中に SIGINT が発生した場合、sigint_interrupt_enabled フラグを介して longjmp を行う設計になっています。このため、入力関数を呼び出す際は必ず sigint_interrupt_jmp が設定されている必要があります。
  • **履歴の永続化**: saveHistory は、可能な限り append_history を使用して既存の履歴ファイルを破壊しないように設計されています。
