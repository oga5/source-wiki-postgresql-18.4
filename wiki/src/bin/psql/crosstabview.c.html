src/bin/psql/crosstabview.c

概要

crosstabview.c は、psql の \crosstabview コマンドを実装するモジュールです。クエリ結果をピボットテーブル（クロス集計表）形式に変換し、行と列のヘッダーを動的に生成して出力します。

モジュール内での役割

src/bin/psql サブシステムにおいて、クエリ結果の表示形式を拡張する機能を提供します。標準的な行指向の出力ではなく、指定された列を軸にデータを再配置する役割を担います。

責務

  • クエリ結果から一意な行・列ヘッダーを抽出する（AVL木による重複排除）。
  • ユーザー指定の列名やインデックスから対象カラムを特定する。
  • オプションによるソート順の適用（rankSort）。
  • libpq の結果セットを printTable 形式へマッピングし、クロス集計表として出力する。

重要なシンボル

  • PrintResultInCrosstab: 本モジュールのメインエントリポイント。
  • pivot_field: ヘッダー値、ソート値、ランクを保持する構造体。
  • avl_tree: ヘッダー値の重複排除とソート済み配列生成のための平衡二分探索木。
  • printCrosstab: 抽出されたヘッダーに基づき、最終的なテーブルを構築・出力する関数。

制御フローとデータフロー

1. **初期化**: avl_tree を初期化し、引数（行・列・データ・ソート列）を解析・検証します。 2. **集計**: 全タプルを走査し、行と列のヘッダー候補を avl_tree に挿入して一意な値のセットを作成します。 3. **ソート**: avlCollectFields で木を走査して配列化し、必要に応じて rankSort で列の順序を調整します。 4. **マッピング**: printCrosstab が printTable API を使用してヘッダーを配置し、各セルにデータを埋め込みます。 5. **出力**: 構築された printTableContent を出力し、メモリを解放します。

読者向けメモ

  • **メモリ制限**: CROSSTABVIEW_MAX_COLUMNS (1600) を超える列が生成される場合、パフォーマンスとメモリ保護のためエラーを返します。
  • **AVL木**: この実装は挿入専用の最小限の AVL 木です。削除や検索機能は含まれていません。
  • **データ整合性**: 1つのセルに複数の値がマッピングされる場合、\crosstabview はエラーを報告して中断します。
  • **ソート**: ソート列が指定された場合、その値は整数として解釈されます。数値以外の値や不正な形式はランク 0 として扱われます。
