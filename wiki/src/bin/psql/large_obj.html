src/bin/psql/large_obj.h

概要

PostgreSQL のラージオブジェクト（Large Object: LO）を操作するためのインターフェースを定義するヘッダーファイルです。psql クライアントからラージオブジェクトのインポート、エクスポート、削除を行うための公開関数を提供します。

モジュール内での役割

src/bin/psql モジュールにおいて、ユーザーが対話的にラージオブジェクトを管理するためのバックエンド処理を抽象化します。このヘッダーで宣言された関数は、主に psql のコマンド処理ロジックから呼び出されます。

責務

  • ラージオブジェクト操作（import/export/unlink）の関数シグネチャの提供。
  • psql のコマンドラインインターフェースと、libpq が提供するラージオブジェクト API との橋渡し。

重要なシンボル

  • do_lo_export: データベース内のラージオブジェクトをローカルファイルへ書き出します。
  • do_lo_import: ローカルファイルをデータベースへラージオブジェクトとして読み込みます。
  • do_lo_unlink: データベースから指定した OID のラージオブジェクトを削除します。

制御フローとデータフロー

本ファイルはインターフェース定義のみですが、実装である large_obj.c では以下のフローが共通化されています。 1. **トランザクション管理**: start_lo_xact を呼び出し、現在のトランザクション状態を確認。必要に応じて BEGIN を発行します。 2. **操作実行**: libpq の lo_* 関数を呼び出し、結果を判定します。 3. **終了処理**: 成功時は finish_lo_xact で必要に応じて COMMIT し、失敗時は fail_lo_xact で ROLLBACK を行います。

直接依存

なし。

関連コード

  • src/bin/psql/large_obj.c: 本ヘッダーで宣言された関数の実装。トランザクション制御やエラーハンドリングの詳細が含まれます。
  • libpq-fe.h: lo_import や lo_export などの基盤となる API を提供するライブラリ。

読者向けメモ

  • **トランザクションの必須性**: ラージオブジェクトの操作は必ずトランザクションブロック内で行う必要があります。実装側で自動的にトランザクションを開始・終了する仕組みが組み込まれているため、呼び出し側はトランザクションの状態を過度に意識する必要はありません。
  • **エラー処理**: 各関数は成功時に true、失敗時に false を返します。失敗時には libpq のエラーメッセージが標準エラー出力に記録されます。
