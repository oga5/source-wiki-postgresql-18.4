src/bin/psql/describe.c

概要

src/bin/psql/describe.c は、psql のメタコマンド（\d 系コマンド）のバックエンドとなる表示ロジックを実装するファイルです。データベースのシステムカタログ（pg_catalog）をクエリし、取得したメタデータを整形してユーザーに提示する役割を担います。

モジュール内での役割

psql のコマンド処理モジュール（command.c）から呼び出される、メタデータ表示機能の主要な実装群です。libpq を介したクエリ実行と、fe_utils/print.h を介した結果の整形表示を橋渡しします。

責務

  • **メタデータ取得**: システムカタログに対する複雑な SQL クエリの構築と実行。
  • **表示整形**: printQuery や printTable を利用した、人間が読みやすい形式への変換。
  • **互換性維持**: PostgreSQL のバージョン差異（pset.sversion）に応じたクエリの動的生成。
  • **パターンマッチング**: ユーザー指定のパターン（正規表現等）を SQL の WHERE 句に安全に変換。

重要なシンボル

メタコマンドハンドラ（公開関数）

  • describeAggregates, describeFunctions, describeTypes: 関数や型に関する情報を表示。
  • describeTableDetails, listTables: テーブル、ビュー、シーケンス等のリレーション情報を表示。
  • describeRoles, listAllDbs: ロールやデータベースの一覧を表示。
  • permissionsList, listDefaultACLs: アクセス権限（ACL）情報を表示。
  • objectDescription: 制約やトリガーなどのオブジェクトコメントを表示。

内部ヘルパー

  • validateSQLNamePattern: ユーザー入力を SQL パターンに変換し、バリデーションを行う。
  • describeOneTableDetails: 特定のテーブルの詳細情報を構築する複雑なロジック。
  • add_tablespace_footer, add_role_attribute: テーブルやロールの表示に付加情報を追加するユーティリティ。
  • map_typename_pattern: 型名のエイリアスを正規化するマッピング関数。

制御フローとデータフロー

1. **ディスパッチ**: command.c がメタコマンドを解析し、対応する describe* 関数を呼び出す。 2. **クエリ構築**: initPQExpBuffer でバッファを確保し、printfPQExpBuffer で SQL を組み立てる。この際、pset.sversion を参照してバージョンごとの差異を吸収する。 3. **実行**: PSQLexec を通じてサーバーへクエリを送信。 4. **表示**: printQuery または printTable を使用し、pset.popt（出力設定）に基づいて結果を整形して pset.queryFout に出力。 5. **クリーンアップ**: termPQExpBuffer でバッファを解放し、PQclear で結果セットを破棄。

読者向けメモ

  • **メモリ管理**: PQExpBuffer を多用しており、init と term のペアを忘れるとメモリリークの原因となります。
  • **バージョン互換性**: 多くの関数で pset.sversion をチェックしています。古いサーバーに対するフォールバック処理が随所に含まれているため、修正時は注意が必要です。
  • **SQL インジェクション**: validateSQLNamePattern を経由せずにユーザー入力をクエリに含めることは禁止されています。
  • **複雑性**: describeOneTableDetails は非常に巨大で複雑な関数です。テーブル、ビュー、インデックス、パーティションなど、多岐にわたるリレーションの種類を単一の関数で処理しているため、変更時は影響範囲の確認が必須です。
