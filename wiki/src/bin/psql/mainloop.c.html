src/bin/psql/mainloop.c

概要

psql の対話的および非対話的な入力処理の中核を担うメインループの実装です。ユーザーからの入力（SQL文やバックスラッシュコマンド）を読み取り、解析し、実行する一連のライフサイクルを管理します。

モジュール内での役割

src/bin/psql における実行制御の心臓部です。input.c から入力を受け取り、psqlscan（字句解析）を介してコマンドを特定し、command.c の機能を利用して実行します。再帰的に呼び出し可能であり、\i コマンドによるスクリプト実行時にも使用されます。

責務

  • **入力の集約**: 複数行にわたる SQL クエリをバッファ（query_buf）に蓄積する。
  • **コマンド解析**: 字句解析器を用いて SQL 文とバックスラッシュコマンドを識別する。
  • **実行制御**: SendQuery による SQL 実行や、HandleSlashCmds による内部コマンドのディスパッチ。
  • **状態管理**: \if ブロックのネスト状態、履歴の保存、およびシグナル（Ctrl-C）割り込み時のクリーンアップ。

重要なシンボル

  • MainLoop(FILE *source): 本ファイルのメインエントリポイント。指定されたストリームから入力を処理する。
  • psqlscan_callbacks: 字句解析器が変数展開を行う際に使用するコールバック構造体。
  • query_buf: 現在蓄積中のクエリを保持する PQExpBuffer。
  • cond_stack: \if ブロックのネスト状態を追跡するスタック。

制御フローとデータフロー

1. **初期化**: 入力ソースを pset に設定し、解析用バッファとスタックを生成。 2. **入力取得**: gets_interactive または gets_fromFile を呼び出し、1行ずつ読み込む。 3. **解析**: psql_scan を呼び出し、セミコロン（SQL実行）またはバックスラッシュ（コマンド実行）を検出。 4. **実行**:

  • SQL の場合: SendQuery を呼び出し、結果を処理。
  • コマンドの場合: HandleSlashCmds を呼び出し、必要に応じて SendQuery や再スキャンを行う。

    • 5. **終了処理**: EOF 到達時や \q コマンド時にバッファを解放し、呼び出し元のコンテキスト（pset）を復元して終了。

読者向けメモ

  • **再入可能性**: MainLoop は再入可能です。\i コマンドでファイルを実行する際、新しい MainLoop インスタンスが生成され、終了後に元のコンテキストへ戻ります。
  • **シグナル処理**: sigsetjmp を使用して Ctrl-C による割り込みを処理しています。ループの各イテレーションで sigint_interrupt_jmp を再設定している点に注意してください。
  • **バッファ管理**: query_buf と previous_buf はポインタ交換によって効率的に管理されています。\e 等のコマンドで直前のクエリを参照する仕組みです。
