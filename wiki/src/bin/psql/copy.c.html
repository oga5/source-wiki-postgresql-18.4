src/bin/psql/copy.c

概要

psql における \copy メタコマンドの実装および、サーバーとの間で COPY データを転送するための低レベルなストリーム処理を提供します。

モジュール内での役割

src/bin/psql の中で、クライアントサイドのデータインポート/エクスポート機能を担当します。サーバーサイドの COPY コマンドをラップし、ローカルファイルシステムやパイプとデータベース接続間の橋渡しを行います。

責務

  • \copy コマンドの構文解析（parse_slash_copy）。
  • ローカルファイル、パイプ（PROGRAM）、または標準入出力のオープンと管理。
  • libpq の COPY プロトコル（COPY_IN/COPY_OUT）を用いたデータ転送の実行。
  • 対話型入力時のプロンプト表示と、\. による終了シーケンスの制御。

重要なシンボル

  • do_copy: \copy コマンドのメインエントリーポイント。
  • handleCopyOut: サーバーからデータを受け取り、ローカルストリームへ書き出す。
  • handleCopyIn: ローカルストリームからデータを読み込み、サーバーへ送信する。
  • struct copy_options: \copy の解析結果を保持する内部構造体。

制御フローとデータフロー

1. **解析**: do_copy が parse_slash_copy を呼び出し、引数を解析して copy_options を生成します。 2. **準備**: do_copy が対象ファイルまたはパイプをオープンし、COPY クエリ文字列を構築します。 3. **実行**: SendQuery で COPY コマンドをサーバーに送信します。 4. **転送**:

  • COPY TO の場合: handleCopyOut が PQgetCopyData をループで呼び出し、データをストリームへ流し込みます。
  • COPY FROM の場合: handleCopyIn がストリームからバッファ（COPYBUFSIZ）へ読み込み、PQputCopyData でサーバーへ送ります。

    • 5. **終了**: 転送完了後、PQgetResult で最終ステータスを確認し、リソースを解放します。

読者向けメモ

  • handleCopyIn 内の sigsetjmp は、ユーザーによる割り込み（Ctrl+C）発生時に COPY モードから安全に脱出するために使用されています。
  • handleCopyIn はテキストモードにおいて \. を検知するロジックを含んでおり、これが入力ストリームの終端を決定します。
  • PROGRAM オプション使用時は popen を利用するため、シグナルハンドリング（sigpipe）の制御に注意が払われています。
