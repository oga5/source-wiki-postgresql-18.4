src/bin/psql/input.h

概要

src/bin/psql/input.h は、psql におけるユーザー入力処理のインターフェースを定義するヘッダーファイルです。主に readline ライブラリ（または editline）の抽象化と、履歴管理機能の提供を目的としています。

モジュール内での役割

psql のメインループ（mainloop.c）がユーザーからクエリやコマンドを受け取る際の抽象化レイヤーとして機能します。環境に応じた入力ライブラリの差異を吸収し、psql の他のコンポーネントが入力ソースを意識せずに済むように設計されています。

責務

  • **入力抽象化**: readline が利用可能な環境ではそれを使用し、そうでない場合は標準入力から読み取るための共通インターフェースを提供。
  • **履歴管理**: 入力されたクエリの履歴への追加、保存、および重複排除やスペースによる無視といった制御機能の提供。
  • **環境適応**: readline や editline のヘッダーパスの差異を吸収し、ビルド時の条件分岐を統一。

重要なシンボル

  • USE_READLINE: readline 機能が有効な場合に定義されるマクロ。条件付きコンパイルのフラグとして使用。
  • gets_interactive(): 対話モードでの入力取得用関数。readline を呼び出し、必要に応じてタブ補完用のバッファをセットアップする。
  • pg_append_history() / pg_send_history(): 入力された文字列を履歴バッファに蓄積し、最終的に readline の履歴リストへコミットする一連の処理を担う。
  • gets_fromFile(): ファイルやパイプからの入力を読み取るための汎用関数。

制御フローとデータフロー

1. **初期化**: initializeInput() により、履歴ファイルの読み込みや readline の設定が行われます。 2. **入力取得**: gets_interactive() が呼び出されると、readline がプロンプトを表示し、ユーザー入力を待ち受けます。 3. **履歴蓄積**: 入力された行は pg_append_history() を通じて PQExpBuffer に蓄積されます。 4. **履歴確定**: クエリ実行後、pg_send_history() が呼び出され、蓄積されたバッファの内容が add_history() を介して履歴リストに登録されます。

直接依存

  • readline/readline.h / editline/readline.h: 入力補完および編集機能。
  • readline/history.h / editline/history.h: 履歴管理機能。
  • pqexpbuffer.h: 入力文字列および履歴バッファの動的管理に使用。
  • history.h
  • readline.h

関連コード

  • src/bin/psql/input.c: 本ヘッダーで宣言された関数の実装。
  • src/bin/psql/mainloop.c: gets_interactive を呼び出す主要なクライアント。
  • src/bin/psql/tab-complete.h: readline の補完機能と連携する定義。

読者向けメモ

  • 本ファイルは、readline 関連の複雑な #ifdef を隠蔽しています。psql の入力処理を追う際は、まず input.c の gets_interactive を確認してください。
  • readline が利用できない環境（または無効化された場合）のフォールバック動作は gets_fromFile に集約されています。
  • 履歴の重複排除やスペースによる無視といったロジックは pg_send_history 内で制御されています。
