src/bin/psql/copy.h

概要

src/bin/psql/copy.h は、psql における \copy メタコマンドの実行を制御するためのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

psql のコマンド処理サブシステムの一部として、クライアントサイドのデータ転送（COPY コマンド）を管理します。サーバーサイドの COPY とは異なり、クライアント側のファイルシステムと psql プロセス間でデータをストリーミングする機能を提供します。

責務

  • \copy コマンドの解析および実行の起点となる関数の宣言。
  • サーバーとのデータ転送（入出力）を処理する低レベル関数のインターフェース提供。
  • libpq の接続状態を利用したデータストリームの制御。

重要なシンボル

  • do_copy(const char *args): \copy コマンドのメインハンドラ。引数を解析し、適切な転送処理へルーティングします。
  • handleCopyOut(...): サーバーからクライアント（ファイルや標準出力）へのデータ転送を処理します。
  • handleCopyIn(...): クライアント（ファイルや標準入力）からサーバーへのデータ転送を処理します。

制御フローとデータフロー

1. psql のメインループが \copy コマンドを検知し、do_copy を呼び出します。 2. do_copy は copy.c 内の解析ロジック（parse_slash_copy）を用いて引数を構造体へ変換します。 3. 転送方向に応じて handleCopyIn または handleCopyOut が呼び出されます。 4. これらの関数は libpq の COPY プロトコルを使用して、サーバーとの間でストリームデータを送受信します。

直接依存

  • libpq-fe.h: PostgreSQL クライアントライブラリの基本機能（PGconn, PGresult 等）に依存しています。

関連コード

  • src/bin/psql/copy.c: 本ヘッダーで宣言された関数の実装が含まれます。parse_slash_copy による構文解析の詳細が記述されています。
  • src/bin/psql/command.c: \copy コマンドが psql のコマンドディスパッチャからどのように呼び出されるかを確認できます。

読者向けメモ

  • 本ファイルはインターフェース定義のみを保持しています。実装の詳細（特に COPY オプションの解析ロジックや FILE ストリームの扱い）については copy.c を参照してください。
  • handleCopyIn/Out は libpq の COPY モードを直接操作するため、エラーハンドリングやシグナル処理（割り込み）の挙動を理解する際は、copy.c 内の libpq 呼び出し箇所を重点的に確認することをお勧めします。
