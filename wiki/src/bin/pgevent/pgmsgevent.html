src/bin/pgevent/pgmsgevent.h

概要

Windows イベントログシステムと連携するためのメッセージ定義ヘッダーファイルです。PostgreSQL が Windows のイベントビューアにログを出力する際に使用するメッセージ識別子を定義しています。

モジュール内での役割

src/bin/pgevent モジュールにおいて、イベントログのメッセージ構造を定義する基盤となります。pgevent.c がイベントログへの書き込みを行う際、このヘッダーで定義された識別子を参照します。

責務

  • Windows メッセージコンパイラ（MC）が期待する 32 ビットのメッセージ識別子フォーマットの定義。
  • イベントログ出力に使用する定数 PGWIN32_EVENTLOG_MSG の提供。

重要なシンボル

  • PGWIN32_EVENTLOG_MSG: イベントログに記録されるメッセージのベースとなる識別子。Windows API を通じてログを送信する際のキーとして機能します。

制御フローとデータフロー

本ファイルは純粋な定義ファイルであり、実行時の制御フローは持ちません。

  • **データフロー**: PGWIN32_EVENTLOG_MSG が pgevent.c にインポートされ、ReportEvent 等の Windows API を呼び出す際の引数として利用されます。

直接依存

  • 特になし（標準的な C ヘッダーガードや外部ライブラリへの依存は明示されていません）。

関連コード

  • src/bin/pgevent/pgevent.c: 本ヘッダーで定義されたシンボルを利用して、実際に Windows イベントログへメッセージを送信する実装が含まれています。

読者向けメモ

  • 本ファイル内のコメントにあるビットレイアウト図は、Windows のメッセージ定義ファイル（.mc ファイル）の仕様に基づいています。
  • PGWIN32_EVENTLOG_MSG は、Windows イベントログのメッセージリソースにおいて、プレースホルダー %1（ログメッセージ本体）を受け入れるための定義です。
  • Windows 以外のプラットフォームでは使用されません。ビルド環境が Windows であることを前提としています。
