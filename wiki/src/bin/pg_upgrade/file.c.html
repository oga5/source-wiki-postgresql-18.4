src/bin/pg_upgrade/file.c

概要

file.c は、pg_upgrade におけるデータディレクトリ間のファイル転送および変換処理を担う低レベルなユーティリティモジュールです。物理的なファイルコピー、ハードリンク、クローン（reflink）、および旧バージョン形式の可視性マップ（Visibility Map）の変換機能を提供します。

モジュール内での役割

pg_upgrade のデータ移行プロセスにおいて、データベースファイルを新しいクラスタへ移動・複製する際の「物理的なデータ転送層」として機能します。exec.c や pg_upgrade.c から呼び出され、OS 依存のファイル操作を抽象化します。

責務

  • **ファイル転送**: コピー、ハードリンク、copy_file_range、およびファイルクローン（reflink）による高速なデータ移行。
  • **データ変換**: 旧バージョンの可視性マップを、現在の PostgreSQL 形式（1 ページから 2 ページへの拡張など）に変換して書き出す。
  • **機能検証**: 実行環境が特定の転送手法（ハードリンクやクローン等）をサポートしているかを事前に確認するテスト関数の提供。

重要なシンボル

  • cloneFile(): OS がサポートする場合、reflink を使用して高速にファイルを複製する。
  • copyFile(): 汎用的なファイルコピー処理。Windows 環境では CopyFile API を使用。
  • rewriteVisibilityMap(): 旧形式の可視性マップを読み込み、新しい形式に変換して書き出す。
  • check_hard_link() / check_file_clone(): 移行モードが実行可能かを確認する診断関数。

制御フローとデータフロー

1. **転送処理**: 呼び出し元が指定した転送モード（リンク、コピー、クローン）に基づき、各関数がファイル記述子を開き、OS 固有のシステムコール（link, ioctl(FICLONE), copy_file_range 等）を呼び出します。 2. **可視性マップ変換**:

  • 旧ファイルをページ単位で読み込み、ヘッダーを保持したままデータを再配置します。
  • BITS_PER_BYTE をループし、旧形式のビットを新しい形式（VISIBILITYMAP_ALL_VISIBLE）へマッピングします。
  • 必要に応じてチェックサムを再計算し、新しいファイルへ書き出します。

読者向けメモ

  • **プラットフォーム依存性**: このファイルは #ifdef を多用しており、Linux の FICLONE や macOS の copyfile など、OS 固有の高速化機能を積極的に利用しています。
  • **可視性マップの変換**: rewriteVisibilityMap は、catversion 201603011 以前からのアップグレード時に必須となる重要な処理です。この変換を行わないと、アップグレード後に全テーブルの VACUUM が必要となり、移行コストが大幅に増大します。
  • **エラーハンドリング**: ファイル操作失敗時は pg_fatal を使用して即座に処理を中断し、エラーメッセージにシステムエラー（%m）を出力する設計になっています。
