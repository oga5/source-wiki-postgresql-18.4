src/bin/pg_upgrade/task.c

概要

task.c は、pg_upgrade において各データベースに対して実行される一連のタスクを並列処理するための汎用フレームワークを提供します。libpq の非同期 API を活用し、指定された並列数（user_opts.jobs）で効率的にクエリを実行します。

モジュール内での役割

pg_upgrade の各フェーズ（チェックやデータ移行など）で発生する「全データベースに対する同一処理」を抽象化する基盤です。個別の処理ロジックをコールバックとして注入することで、接続管理や並列実行の複雑さを隠蔽します。

責務

  • **タスクの定義と管理**: UpgradeTask 構造体を通じたクエリステップの登録と保持。
  • **並列実行制御**: UpgradeTaskSlot を用いたステートマシンによる接続・クエリ実行の管理。
  • **非同期 I/O**: select() を用いた多重化により、複数のデータベース接続を効率的に待機・処理する。
  • **リソースのライフサイクル管理**: 接続の確立、クエリ結果の処理、および接続の解放。

重要なシンボル

  • UpgradeTask: 実行すべきクエリステップの集合を保持するメイン構造体。
  • upgrade_task_add_step(): タスクにクエリと処理用コールバックを追加する。
  • upgrade_task_run(): 指定された並列数でタスクを実行するメインループ。
  • UpgradeTaskSlot: 各並列スロットの状態（CONNECTING, RUNNING_QUERIES 等）を管理する構造体。

制御フローとデータフロー

1. **初期化**: upgrade_task_create() でタスクを生成し、upgrade_task_add_step() でクエリを登録。 2. **実行開始**: upgrade_task_run() がスロット配列を初期化し、メインループに入る。 3. **ステートマシン**: process_slot() が各スロットの状態を遷移させる。

  • FREE: 次のデータベースを割り当て PQconnectStart を実行。
  • CONNECTING: PQconnectPoll で接続完了を待機。
  • RUNNING_QUERIES: PQsendQuery でクエリを送信し、結果をコールバックで処理。

    • 4. **待機**: wait_on_slots() が select() を呼び出し、I/O 準備が整ったスロットのみを process_slot() に渡す。 5. **終了**: 全データベースの処理が完了するまでループを継続。

  • fe_utils/string_utils.h: 文字列操作およびクエリバッファ管理。
  • pg_upgrade.h: ClusterInfo や DbInfo などのデータ構造定義。

読者向けメモ

  • 本ファイルは libpq の非同期 API（PQconnectStart, PQsendQuery 等）の典型的な実装例です。
  • process_slot 内のステートマシンは、接続からクエリ完了までを非ブロッキングで処理するように設計されています。
  • 各タスクの最初のステップとして ALWAYS_SECURE_SEARCH_PATH_SQL が自動的に追加される点に注意してください。
  • 複数のステップを追加した場合、それらは同一の接続上で順次実行されます。
