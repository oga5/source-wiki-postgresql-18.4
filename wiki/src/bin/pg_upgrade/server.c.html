src/bin/pg_upgrade/server.c

概要

本ファイルは、pg_upgrade におけるデータベースサーバー（Postmaster）の起動、停止、および接続管理を行うためのユーティリティ関数群を提供します。アップグレード対象となる新旧両方のクラスターに対して、安全かつ一貫した接続環境を構築する役割を担います。

モジュール内での役割

pg_upgrade の中核となるサーバー操作レイヤーです。pg_ctl を介したプロセス制御と、libpq を介したデータベース接続の抽象化を行い、他のモジュールがサーバーの状態を意識せずにクエリを実行できるようにします。

責務

  • **サーバーライフサイクル管理**: pg_ctl を使用したサーバーの起動・停止、および atexit フックによる異常終了時のクリーンアップ。
  • **接続の確立**: 適切な接続文字列の生成と、接続失敗時のエラーハンドリング。
  • **環境の検証**: PGHOST 等の環境変数がリモートサーバーを指していないかのチェック。
  • **バージョン情報の取得**: PG_VERSION ファイルの読み込みと解析。

重要なシンボル

  • connectToServer: 指定したデータベースへの接続を確立し、失敗時はプログラムを終了させる。
  • start_postmaster: pg_ctl を呼び出し、サーバーを起動する。新クラスターの場合はパフォーマンス向上のためのオプション（fsync=off 等）を付与する。
  • stop_postmaster: 実行中のサーバーを停止する。
  • executeQueryOrDie: クエリを実行し、失敗した場合はエラーログを出力して終了する。
  • get_major_server_version: PG_VERSION ファイルからメジャーバージョンを抽出する。

追加の索引済みシンボル

  • check_pghost_envvar
  • cluster_conn_opts

  • stop_postmaster_atexit

制御フローとデータフロー

1. **接続**: get_db_conn が ClusterInfo から接続文字列を構築し、PQconnectdb を呼び出します。 2. **起動**: start_postmaster は pg_ctl コマンドを構築して exec_prog で実行し、その後 get_db_conn で接続確認を行います。 3. **終了**: stop_postmaster は os_info.running_cluster を参照して対象を特定し、pg_ctl stop を実行します。 4. **エラー**: 多くの関数は失敗時に pg_fatal や exit(1) を呼び出し、アップグレードプロセス全体を即座に停止させる設計となっています。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用共通ヘッダー。
  • common/connect.h: 接続管理用ユーティリティ。
  • fe_utils/string_utils.h: 文字列操作およびシェルエスケープ用。
  • libpq/pqcomm.h: libpq 通信定義。
  • pg_upgrade.h: モジュール共通のデータ構造（ClusterInfo 等）と定数。

関連コード

  • src/bin/pg_upgrade/exec.c: exec_prog 関数を提供し、外部コマンド実行の基盤となる。
  • src/bin/pg_upgrade/util.c: pg_log や pg_fatal などのログ出力・エラー処理関数を提供。
  • src/bin/pg_upgrade/pg_upgrade.c: 本モジュールの関数を呼び出し、アップグレードのメインフローを制御する。

読者向けメモ

  • 本ファイル内の start_postmaster は、新クラスター起動時に fsync=off や synchronous_commit=off を強制的に付与します。これはアップグレード速度を優先するためであり、クラッシュ時には再作成が必要であるという前提に基づいています。
  • check_pghost_envvar は、誤ってリモートサーバーに対して pg_upgrade を実行することを防ぐための重要な安全装置です。
  • executeQueryOrDie は、実行するクエリの戻り値（PGresult）を呼び出し側で PQclear する必要がある点に注意してください。
