src/bin/pg_upgrade/function.c

概要

function.c は、pg_upgrade において旧クラスタで使用されている共有ライブラリ（C言語関数や論理レプリケーションの出力プラグイン）が、新クラスタでも利用可能であることを検証するためのモジュールです。

モジュール内での役割

pg_upgrade のアップグレードプロセスにおいて、外部ライブラリの依存関係を解決する役割を担います。旧クラスタからライブラリ情報を収集し、新クラスタで LOAD コマンドを実行することで、バイナリ互換性と存在確認を行います。

責務

  • **ライブラリ情報の収集**: 旧クラスタの pg_proc カタログおよび論理レプリケーションスロットから、必要なライブラリ名を抽出する。
  • **検証の実行**: 新クラスタに対して LOAD コマンドを発行し、ライブラリのロード可否をテストする。
  • **エラーレポート**: ロードに失敗したライブラリと、それを使用しているデータベース名をログファイルに出力する。

重要なシンボル

  • get_loadable_libraries(): 旧クラスタからライブラリ名の一覧を取得し、os_info.libraries に格納するメイン関数。
  • check_loadable_libraries(): 収集したライブラリに対して LOAD を試行し、検証を行う。
  • library_name_compare(): ライブラリ名をソートするための比較関数。依存関係の解決順序を安定させるために使用される。
  • loadable_libraries_state: 収集プロセス中の状態を保持する内部構造体。

追加の索引済みシンボル

制御フローとデータフロー

1. **収集フェーズ**: get_loadable_libraries が upgrade_task を使用して各DBから probin をクエリし、結果を state 構造体に一時保存する。 2. **統合**: 収集したライブラリ名と論理レプリケーションプラグイン名を os_info.libraries 配列に集約する。 3. **ソート**: check_loadable_libraries 内で qsort を呼び出し、名前の長さと辞書順でライブラリを整列させる。 4. **検証フェーズ**: ソートされたリストを順次 LOAD し、失敗した場合は loadable_libraries.txt に詳細を記録して pg_fatal で終了する。

直接依存

  • postgres_fe.h: フロントエンド用共通ヘッダー。
  • access/transam.h: トランザクション関連の定数。
  • catalog/pg_language_d.h: 言語OID定義。
  • common/int.h: 整数比較ユーティリティ。
  • pg_upgrade.h: pg_upgrade の主要構造体およびグローバル変数定義。

関連コード

  • src/bin/pg_upgrade/task.c: upgrade_task の実行基盤。
  • src/bin/pg_upgrade/server.c: connectToServer 関数を提供し、新クラスタへの接続を管理。
  • src/bin/pg_upgrade/info.c: DbInfo や os_info などのクラスタ情報管理。

読者向けメモ

  • 本ファイルは、ライブラリのロード失敗を「致命的なエラー」として扱います。アップグレード前に、旧クラスタで使用しているすべての共有ライブラリが新環境にインストールされていることを確認してください。
  • library_name_compare でのソート順序は、ライブラリ間の依存関係（例: hstore と hstore_plpython）を考慮した設計になっています。順序変更には注意が必要です。
