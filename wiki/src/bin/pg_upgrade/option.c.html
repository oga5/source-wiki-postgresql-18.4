src/bin/pg_upgrade/option.c

概要

option.c は pg_upgrade のコマンドライン引数および環境変数の解析、検証、および設定を行うモジュールです。ユーザーが指定した旧クラスタと新クラスタのパス、ポート、実行モードなどの情報を収集し、グローバルな設定構造体に格納します。

モジュール内での役割

pg_upgrade の実行開始時に呼び出され、プログラムの動作を決定づける設定を確定させます。他のモジュール（check.c や server.c など）が参照する user_opts や ClusterInfo 構造体の初期化を担う、システムの入り口となるファイルです。

責務

  • **引数解析**: getopt_long を用いたコマンドラインオプションのパース。
  • **環境変数との統合**: 引数が未指定の場合に環境変数（PGDATAOLD 等）から値を補完。
  • **パスの正規化**: 指定されたディレクトリパスの検証と正規化。
  • **設定の強制**: 互換性維持のためのデフォルト設定（default_transaction_read_only=false の強制適用など）の注入。
  • **環境チェック**: 実行ユーザーが root でないことの確認や、Windows 環境での実行ディレクトリ制限の検証。

重要なシンボル

  • user_opts: ユーザーが指定した実行オプション（転送モード、並列数、統計情報有無など）を保持するグローバル変数。
  • parseCommandLine(): 引数解析のメイン関数。
  • adjust_data_dir(): 設定ファイルのみが配置されたディレクトリ（config-only）から実際のデータディレクトリを特定する関数。
  • get_sock_dir(): クラスタのソケットディレクトリを特定する関数。

制御フローとデータフロー

1. **初期化**: parseCommandLine が呼び出され、デフォルト値が設定されます。 2. **解析**: getopt_long ループにより引数を順次処理し、user_opts や old_cluster/new_cluster 構造体に値を格納します。 3. **検証・補完**: check_required_directory を通じて必須ディレクトリが特定されているか確認し、不足があれば環境変数から補完します。 4. **環境調整**: PGOPTIONS に読み取り専用モードを無効化するフラグを強制的に追加します。 5. **最終化**: adjust_data_dir や get_sock_dir を呼び出し、実行に必要な詳細パスを確定させます。

読者向けメモ

  • **設定の優先順位**: コマンドライン引数が環境変数よりも優先されます。
  • **config-only ディレクトリ**: adjust_data_dir は、postgresql.conf が存在するが PG_VERSION が存在しない場合に、postgres -C data_directory を実行して実際のデータパスを特定します。この際、対象のバイナリパスが正しく設定されている必要があります。
  • **Windows 特有の挙動**: Windows 環境では、実行ディレクトリが新クラスタのデータディレクトリ内にあると権限エラーになる可能性があるため、パスのプレフィックスチェックが行われています。
