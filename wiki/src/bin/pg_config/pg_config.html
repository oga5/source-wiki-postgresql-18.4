src/bin/pg_config/pg_config.c

概要

pg_config.c は、インストールされた PostgreSQL のビルド設定やディレクトリ構成情報を出力するコマンドラインツールです。外部パッケージが PostgreSQL と連携する際、ヘッダーファイルやライブラリのパス、コンパイルオプションを動的に取得するために使用されます。

モジュール内での役割

src/bin/pg_config ディレクトリにおける唯一のメインプログラムです。PostgreSQL のインストール先が元の設定から移動（リロケーション）された場合でも、実行ファイルの場所を基準に正しいパスを算出する役割を担います。

責務

  • **設定情報の提供**: ビルド時の環境変数（CC, CFLAGS 等）やディレクトリパス（BINDIR, PKGLIBDIR 等）の表示。
  • **引数解析**: ユーザーからのオプション入力を受け付け、対応する設定項目をフィルタリングして出力。
  • **実行環境の特定**: find_my_exec を用いて自身のインストールパスを特定し、設定情報の検索起点とする。

重要なシンボル

  • InfoItem: コマンドライン引数（--bindir 等）と、内部的な設定項目名（BINDIR 等）を紐付ける構造体。
  • info_items[]: サポートする全オプションの定義テーブル。
  • main(): プログラムの起点。引数解析、設定データの取得、出力処理を統括。
  • show_item(): 指定された設定項目を ConfigData 配列から検索し、標準出力へ書き出す関数。

追加の索引済みシンボル

  • advice
  • help
  • main
  • show_item

制御フローとデータフロー

1. **初期化**: set_pglocale_pgservice でロケールを設定し、find_my_exec で自身の実行パスを取得。 2. **データ取得**: get_configdata を呼び出し、インストールパスに基づいた設定情報のリスト（ConfigData）をメモリ上にロード。 3. **出力処理**:

  • 引数がない場合: 全設定項目を key = value 形式で出力。
  • 引数がある場合: 各引数を info_items と照合し、一致する項目のみを show_item で出力。

    • 4. **終了**: 不明な引数が渡された場合はエラーメッセージを表示して終了。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用共通ヘッダー。
  • common/config_info.h: 設定情報を取得するための共通インターフェース（ConfigData 構造体や get_configdata 関数など）。

関連コード

  • src/common/config_info.c: get_configdata の実体が含まれており、設定情報の算出ロジックの詳細を確認する際に参照すべきファイルです。

読者向けメモ

  • 本プログラムは、かつてシェルスクリプトで実装されていたものを C 言語に移植したものです。
  • info_items テーブルと help() 関数内の出力内容は同期している必要があるため、新しいオプションを追加する際は両方の修正を忘れないようにしてください。
  • リロケーション対応のため、パス情報はハードコードされず、実行時の実行ファイル位置から相対的に計算される点に注意してください。
