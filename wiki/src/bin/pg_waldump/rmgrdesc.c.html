src/bin/pg_waldump/rmgrdesc.c

概要

rmgrdesc.c は、WAL（Write-Ahead Log）レコードを解析する際に、各リソースマネージャ（rmgr）固有の情報を出力するための記述子テーブルを定義するファイルです。pg_waldump が WAL レコードの内容を人間が読める形式に変換する際の中心的な役割を担います。

モジュール内での役割

pg_waldump モジュールにおいて、WAL レコードの型（rmid）に基づき、適切なデコード関数や識別関数をルックアップするためのインターフェースを提供します。rmgrdesc.h で定義された RmgrDescData 構造体の実装を保持しています。

責務

  • **組み込みリソースマネージャの管理**: access/rmgrlist.h を利用して、PostgreSQL 標準のリソースマネージャ（Heap, B-tree, XACT 等）の記述子テーブルを静的に構築する。
  • **カスタムリソースマネージャのフォールバック**: ロードされていないカスタムリソースマネージャに対し、custom### という形式の識別子を動的に生成し、デフォルトのデコード処理を割り当てる。
  • **ルックアップ機能の提供**: 指定された RmgrId に対応する記述子を返す GetRmgrDesc 関数の提供。

重要なシンボル

  • RmgrDescTable: 組み込みリソースマネージャの関数ポインタを保持する静的配列。
  • GetRmgrDesc(RmgrId rmid): 指定された ID に対応する RmgrDescData を返す公開関数。
  • PG_RMGR: rmgrlist.h を展開する際に使用されるマクロ。各リソースマネージャの desc および identify 関数をテーブルに登録します。

制御フローとデータフロー

1. pg_waldump が WAL レコードを読み込むと、そのレコードの RmgrId をキーとして GetRmgrDesc を呼び出します。 2. GetRmgrDesc は、ID が組み込み範囲内であれば RmgrDescTable から直接エントリを返します。 3. ID がカスタム範囲（RM_MIN_CUSTOM_ID 以上）の場合、初回のみ initialize_custom_rmgrs が実行され、CustomRmgrDesc 配列が初期化されます。 4. 返された RmgrDescData 内の関数ポインタ（rm_desc）を介して、レコードの詳細情報が StringInfo バッファに書き込まれます。

読者向けメモ

  • 本ファイルは FRONTEND マクロを定義してコンパイルされるため、バックエンドのメモリ管理やロック機能には依存せず、純粋なデータ変換ロジックに集中しています。
  • カスタムリソースマネージャのデコード関数は、pg_waldump 実行時にはロードされないため、デフォルトの default_desc が使用され、単に ID が表示される仕様になっています。
