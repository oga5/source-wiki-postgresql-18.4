src/bin/pg_combinebackup/write_manifest.h

概要

write_manifest.h は、pg_combinebackup ツールにおいて、バックアップの整合性を保証するための backup_manifest ファイルを生成・書き出すためのインターフェースを定義しています。

モジュール内での役割

本モジュールは、複数のバックアップソースを結合した後に、新しいバックアップセットのメタデータ（ファイルリスト、サイズ、チェックサム等）を JSON 形式で出力する役割を担います。load_manifest.h が既存のバックアップ情報を読み込むのに対し、本モジュールは新しいバックアップの正当性を記録します。

責務

  • バックアップマニフェスト作成用のコンテキスト（manifest_writer）の初期化。
  • 個別のファイルエントリをマニフェストに追加する処理。
  • マニフェストの最終化（WAL 範囲の記録とファイルのクローズ）。

重要なシンボル

  • manifest_writer: マニフェスト生成の状態を保持する不透明な構造体。バッファリングやチェックサム計算のコンテキストを管理します。
  • create_manifest_writer(): 指定ディレクトリに backup_manifest を作成するための初期化関数。
  • add_file_to_manifest(): ファイルのメタデータとチェックサムをマニフェストに追加します。
  • finalize_manifest(): マニフェストの末尾を閉じ、WAL 範囲を書き込んでファイルを完成させます。

制御フローとデータフロー

1. create_manifest_writer で JSON のヘッダー部分をバッファに書き込みます。 2. add_file_to_manifest が呼び出されるたびに、ファイル情報を JSON オブジェクトとしてバッファに追加します。バッファが一定サイズを超えると、適宜ディスクへフラッシュされます。 3. すべてのファイル処理後、finalize_manifest が呼び出され、残りの WAL 範囲情報と JSON の閉じ括弧を書き込み、ファイルをクローズします。

直接依存

  • common/checksum_helper.h: ファイルのチェックサム計算およびアルゴリズムの定義に使用。

関連コード

  • src/bin/pg_combinebackup/write_manifest.c: 本ヘッダーの実装。JSON のエスケープ処理やディスク I/O を担当。
  • src/bin/pg_combinebackup/load_manifest.h: 既存のバックアップマニフェストを解析する対となるモジュール。

読者向けメモ

  • 本モジュールはフロントエンド（pg_combinebackup）専用の実装です。バックエンドの AddFileToBackupManifest と機能は似ていますが、環境の違いから独自の実装となっています。
  • manifest_writer 構造体はヘッダー内では不透明（opaque）として扱われており、詳細は .c ファイル側で定義されています。
  • マニフェストのパスは常に backup_manifest という固定名で生成されます。
