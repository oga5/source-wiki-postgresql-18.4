src/bin/pg_combinebackup/write_manifest.c

概要

write_manifest.c は、バックアップの整合性を保証するための backup_manifest ファイルを生成・書き出すためのモジュールです。PostgreSQL のバックアッププロセスにおいて、バックアップに含まれる全ファイルの情報と WAL レンジを JSON 形式で記録します。

モジュール内での役割

pg_combinebackup ツールの一部として、複数のバックアップソースを統合した際に、新しいバックアップセットのメタデータを作成します。load_manifest.c が既存のバックアップから情報を読み取るのに対し、本モジュールは新しいマニフェストを構築・出力する役割を担います。

責務

  • **マニフェストの構築**: JSON 形式のバックアップマニフェストをメモリ上のバッファで組み立てる。
  • **ファイル情報の追加**: 各ファイルのパス、サイズ、更新日時、チェックサムを JSON エントリとして追加する。
  • **WAL レンジの記録**: バックアップに関連する WAL のタイムラインと LSN 範囲を記録する。
  • **整合性保護**: マニフェスト自体の SHA-256 チェックサムを計算し、ファイルの末尾に付与する。
  • **ディスクへのフラッシュ**: メモリバッファが一定サイズを超えた際、または終了時にディスクへ書き出す。

重要なシンボル

  • manifest_writer: マニフェスト生成の状態を保持する構造体。ファイル記述子、バッファ、チェックサムコンテキストを含む。
  • create_manifest_writer(): ライターを初期化し、JSON のヘッダー部分を生成する。
  • add_file_to_manifest(): 個別のファイル情報を JSON エントリとしてバッファに追加する。
  • finalize_manifest(): WAL レンジの書き込み、マニフェスト全体のチェックサム計算、およびファイルのクローズを行う。

制御フローとデータフロー

1. **初期化**: create_manifest_writer でバッファを確保し、JSON の開始タグを書き込む。 2. **データ追加**: add_file_to_manifest が呼ばれるたびに、ファイル情報を JSON 文字列として StringInfo バッファに追記する。 3. **バッファ管理**: バッファが 128KB を超えると flush_manifest が呼ばれ、ディスクへの書き出しと同時に、その内容でマニフェスト全体のチェックサム（manifest_ctx）を更新する。 4. **終了処理**: finalize_manifest で WAL レンジを追記し、最終的なチェックサムを計算して書き込み、ファイルを閉じる。

読者向けメモ

  • 本モジュールは、バックエンドの AddFileToBackupManifest と類似していますが、フロントエンドツールとして動作するために StringInfo を用いたバッファリングと手動の JSON エスケープ処理を行っています。
  • 非 UTF-8 のファイルパスが含まれる場合、Encoded-Path として 16 進数エンコードされる点に注意してください。
  • flush_manifest は、チェックサム計算の整合性を保つため、書き出したデータのみを逐次的に manifest_ctx に反映させる設計になっています。
