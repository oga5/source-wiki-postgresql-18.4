src/bin/pg_combinebackup/copy_file.c

概要

copy_file.c は、pg_combinebackup においてバックアップファイルをコピーするための抽象化レイヤーを提供します。プラットフォーム固有の最適化されたコピー手法（クローン、ハードリンク、copy_file_range 等）を統一的なインターフェースで利用可能にします。

モジュール内での役割

pg_combinebackup のバックアップ再構築プロセスにおいて、物理的なファイル転送を担当します。他のモジュールが「どのファイルをどうコピーするか」を決定した後、このモジュールが実際の I/O を実行し、必要に応じてチェックサム計算を並行して行います。

責務

  • **戦略的コピー**: CopyMethod に基づき、最適なシステムコールを選択して実行する。
  • **チェックサム計算**: コピー中、またはコピー後にファイル内容の整合性を検証する。
  • **ドライラン対応**: 実際のコピーを行わずに、ファイルアクセス権限の確認やログ出力のみを行うモードの提供。
  • **エラーハンドリング**: I/O エラー発生時の致命的な例外処理とログ出力。

重要なシンボル

  • copy_file(): 本モジュールのメインエントリーポイント。
  • CopyMethod: コピー手法を定義する列挙型（CLONE, COPY, LINK 等）。
  • checksum_file(): コピー手法がチェックサムを自動計算しない場合に呼び出される補助関数。
  • copy_file_blocks(): 最も基本的なブロック単位の読み書きによるコピー実装。

制御フローとデータフロー

1. **初期化**: copy_file() が呼び出され、dry_run モードであればファイル存在確認のみを行う。 2. **戦略決定**: copy_method に応じて、適切な実装関数（copy_file_clone 等）を関数ポインタに割り当てる。 3. **実行**: 選択された関数がソースからデスティネーションへデータを転送する。 4. **検証**:

  • copy_file_blocks は転送と同時にチェックサムを更新する。
  • その他の手法（clone, link 等）は、転送完了後に checksum_file() を呼び出して検証を行う。

    • 5. **終了**: 成功またはエラー発生時にログを出力し、ファイルディスクリプタを閉じる。

読者向けメモ

  • **プラットフォーム依存性**: copy_file_clone や copy_file_by_range は、OS が対応していない場合に pg_fatal を発生させます。利用環境のカーネルサポート状況に注意してください。
  • **チェックサムの挙動**: copy_file_blocks は転送と計算を同時に行いますが、他の手法はコピー後に別途読み込みが発生するため、パフォーマンス特性が異なります。
  • **ドライラン**: dry_run モードは単なるシミュレーションではなく、ソースファイルのオープンが可能かどうかのチェックを含みます。
