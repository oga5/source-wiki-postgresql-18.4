src/include/fmgr.h

概要

PostgreSQL の関数マネージャ（fmgr）のインターフェースを定義する中心的なヘッダーファイルです。バックエンド内の C 関数呼び出し、動的ロードされた関数の実行、およびそれらに関連するメタデータ管理の規約を規定します。

モジュール内での役割

src/backend/utils/fmgr/ サブシステムの公開 API として機能します。PostgreSQL の実行エンジン（Executor）と、ユーザー定義関数や組み込み関数を結びつけるための抽象化レイヤーを提供します。

責務

  • **呼び出し規約の定義**: C 関数が PostgreSQL の実行エンジンから呼び出される際の引数受け渡し、戻り値、およびエラー処理の標準化。
  • **メタデータ管理**: 関数の OID や属性（strict, set-returning 等）を保持する FmgrInfo 構造体の定義と管理。
  • **データ変換と操作**: Datum 型の操作、TOAST データの展開（detoast）、および引数取得のためのマクロ群の提供。
  • **ABI 互換性チェック**: 動的ロードされるモジュールが現在の PostgreSQL バージョンやビルド設定と整合しているかを検証する仕組み（PG_MODULE_MAGIC）。
  • **フック機構**: セキュリティポリシーやデバッグ目的で関数呼び出しの開始・終了・異常終了を監視するためのフックインターフェースの提供。

重要なシンボル

関数呼び出し管理

  • FunctionCallInfo: 実行時の引数やコンテキストを保持する構造体へのポインタ。
  • FmgrInfo: 関数呼び出しに必要な静的情報を保持する構造体。
  • PG_FUNCTION_ARGS: 関数定義時に使用する引数リストの型。
  • LOCAL_FCINFO(name, nargs): スタック上に FunctionCallInfo を安全に確保するマクロ。

データ操作・変換

  • PG_GETARG_xxx(n): n 番目の引数を特定の型として取得するマクロ群。
  • PG_RETURN_xxx(x): 結果を Datum として返すマクロ群。
  • PG_DETOAST_DATUM(datum): TOAST されたデータを展開するマクロ。
  • PG_FREE_IF_COPY(ptr, n): detoast されたコピーをメモリ解放するユーティリティ。

モジュール・ABI 管理

  • PG_FUNCTION_INFO_V1(funcname): C 関数をバージョン 1 呼び出し規約として登録するマクロ。
  • PG_MODULE_MAGIC: 動的ロードモジュールの ABI 整合性を検証するマジックブロックを生成するマクロ。

制御フローとデータフロー

1. **初期化**: 関数呼び出し前に fmgr_info() を呼び出し、FmgrInfo を構築してキャッシュします。 2. **呼び出し**: FunctionCallInvoke(fcinfo) マクロ、または OidFunctionCallNColl 系の関数を介して実行されます。 3. **データフロー**:

  • FunctionCallInfoBaseData が引数配列（args）を保持します。
  • 呼び出された関数は PG_GETARG_xxx マクロで args から値を取り出し、PG_RETURN_xxx で Datum を返します。
  • isnull フラグにより NULL 値の伝播を制御します。

直接依存

本ファイルはスタンドアロンで動作するよう設計されており、直接的な #include は最小限に抑えられています。ただし、Datum 型や Oid 型などの基本型定義に依存しています。

関連コード

  • src/backend/utils/fmgr/fmgr.c: 関数マネージャの主要な実装。
  • src/backend/utils/fmgr/dfmgr.c: 動的ライブラリのロードと管理。
  • src/backend/executor/nodeAgg.c: 集約関数特有のコンテキスト管理。

読者向けメモ

  • **メモリ所有権**: FmgrInfo 内の fn_extra は、fn_mcxt で指定されたメモリコンテキストに割り当てられる必要があります。
  • **アライメント**: LOCAL_FCINFO はスタック上のアライメントを保証するために共用体を使用しています。
  • **制約**: FunctionCallInfoBaseData の args 配列は可変長です。直接構造体を確保せず、必ず SizeForFunctionCallInfo マクロを使用してサイズ計算を行ってください。
  • **省略された詳細**: 本ファイルはインターフェース定義に特化しており、具体的な関数検索アルゴリズムやハッシュテーブルの実装（fmgr.c 側）は省略されています。
