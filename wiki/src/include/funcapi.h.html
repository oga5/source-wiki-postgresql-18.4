src/include/funcapi.h

概要

funcapi.h は、PostgreSQL の関数管理システム（fmgr）において、複合型を返す関数や集合を返す関数（SRF: Set Returning Functions）、および可変長引数（VARIADIC）を扱う関数を実装するための API を定義するヘッダーファイルです。

モジュール内での役割

src/include モジュールにおいて、ユーザー定義関数（UDF）や内部関数が複雑なデータ構造を安全かつ効率的に扱うためのインターフェースを提供します。fmgr.h を補完し、実行時の型解決やメモリ管理の抽象化を担います。

責務

  • **複合型サポート**: TupleDesc や AttInMetadata を用いた、C 文字列からタプルへの変換および構築の支援。
  • **SRF 状態管理**: 複数回呼び出される関数（ValuePerCall モード）における、呼び出し回数やコンテキストの保持。
  • **型解決**: 実行時の引数や戻り値の型（ポリモーフィック型など）の動的な特定。
  • **マクロによる抽象化**: SRF 実装の定型処理を簡略化する一連の SRF_* マクロの提供。

重要なシンボル

  • FuncCallContext: SRF の呼び出し間で状態を保持するための構造体。
  • AttInMetadata: C 文字列からタプルを構築する際に必要な型情報を保持する構造体。
  • SRF_FIRSTCALL_INIT() / SRF_RETURN_NEXT(): SRF 実装の定型処理を隠蔽する主要マクロ。
  • get_call_result_type(): 実行時に戻り値の型や TupleDesc を取得する関数。

制御フローとデータフロー

1. **初期化**: SRF_IS_FIRSTCALL() で初回呼び出しを判定し、SRF_FIRSTCALL_INIT() で FuncCallContext を作成・初期化します。 2. **メモリ管理**: multi_call_memory_ctx を使用して、複数呼び出しにまたがるデータを保持します。 3. **データ生成**: BuildTupleFromCStrings() 等を用いてタプルを構築し、SRF_RETURN_NEXT() で結果を返します。 4. **終了**: 全ての行を返し終えたら SRF_RETURN_DONE() を呼び出し、リソースの解放と終了フラグの設定を行います。

直接依存

  • access/tupdesc.h: タプル記述子の定義。
  • executor/executor.h / executor/tuptable.h: 実行コンテキストおよびタプルテーブルの操作。
  • fmgr.h: 関数呼び出しの基本インターフェース。

関連コード

  • src/backend/utils/fmgr/funcapi.c: 本ヘッダーで宣言された関数の実装。
  • src/backend/executor/execTuples.c: BlessTupleDesc 等のタプル操作の実装。

読者向けメモ

  • **SRF の注意点**: ValuePerCall モードの SRF は、クエリの途中で中断される可能性があるため、SRF_RETURN_DONE() に依存したクリーンアップは避けてください。永続的なリソース管理が必要な場合は、InitMaterializedSRF() を使用したマテリアライズドモードを検討してください。
  • **メモリコンテキスト**: FuncCallContext 内の multi_call_memory_ctx は、複数回呼び出しにまたがるデータの生存期間を管理するために必須です。これ以外のメモリは、呼び出しごとにリセットされる可能性があるため注意が必要です。
