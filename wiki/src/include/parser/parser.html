src/include/parser/parser.h

概要

parser.h は、PostgreSQL の構文解析プロセスにおける「Raw Parser（生パーサー）」の外部インターフェースを定義するヘッダーファイルです。SQL 文字列を字句解析（Lexical Analysis）および構文解析（Grammatical Analysis）し、未解析の構文木（Raw Parse Tree）を生成するための主要な API を提供します。

モジュール内での役割

PostgreSQL の解析パイプラインの最上流に位置します。ここで生成される構文木は、後続の analyze.c 等による意味解析（Semantic Analysis）を経て、実行可能なクエリツリーへと変換されます。本ヘッダーは、パーサーの動作モードを制御する列挙型と、解析の起点となる関数を定義しています。

責務

  • **解析モードの定義**: SQL 文全体、型名、PL/pgSQL 式など、解析対象に応じたモード（RawParseMode）の提供。
  • **エントリーポイントの公開**: SQL 文字列を解析するための raw_parser 関数の宣言。
  • **GUC 変数の管理**: 字句解析器（scan.l）の挙動を制御する設定値（backslash_quote 等）の外部公開。

重要なシンボル

  • raw_parser(const char *str, RawParseMode mode): 解析のメイン関数。文字列を受け取り、List 形式の構文木を返します。
  • RawParseMode: 解析対象の文脈を指定する列挙型。デフォルトの SQL 解析から PL/pgSQL 特有の構文解析までを切り替えます。
  • BackslashQuoteType: 文字列リテラル内でのバックスラッシュの扱いを制御する設定値。

制御フローとデータフロー

1. **入力**: raw_parser が SQL 文字列と RawParseMode を受け取ります。 2. **字句解析**: scanner_init を通じて Flex ベースの字句解析器が初期化されます。 3. **構文解析**: Bison ベースのパーサー（base_yyparse）が呼び出され、トークン列を RawStmt ノード等のツリー構造に変換します。 4. **出力**: 解析結果として、List 構造の未解析構文木が返されます。この段階ではテーブルアクセスは行われません。

直接依存

  • nodes/parsenodes.h: 構文解析の結果として生成されるノード構造体（RawStmt, TypeName 等）の定義。

関連コード

  • src/backend/parser/parser.c: raw_parser の実装本体。字句解析器と構文解析器の橋渡しを行います。
  • src/backend/parser/gram.y: Bison の文法定義ファイル。
  • src/backend/parser/scan.l: Flex の字句解析定義ファイル。
  • src/backend/parser/analyze.c: 本パーサーが生成したツリーを意味解析する後続モジュール。

読者向けメモ

  • **トランザクションの独立性**: ここで行われる「Raw Parsing」はテーブルアクセスを伴わないため、トランザクションがアボート状態であっても実行可能です。
  • **Raw Parse Tree**: 本パーサーの出力は「未解析」である点に注意してください。型チェックや名前解決は行われておらず、後続の analyze.c による変換が必須です。
  • **GUC 変数**: backslash_quote 等の変数は scan.l で使用されます。これらはグローバルな設定に依存するため、解析の挙動が環境によって変わる可能性があることに留意してください。
