src/include/parser/parse_merge.h

概要

src/include/parser/parse_merge.h は、PostgreSQL の MERGE 文を解析（Transform）するためのインターフェースを定義するヘッダーファイルです。SQL の MERGE ステートメントを内部的な Query 構造体へ変換する処理の入り口を提供します。

モジュール内での役割

src/include/parser モジュールの一部として、構文解析フェーズにおける MERGE 文特有の変換ロジックをカプセル化しています。パーサーのメイン処理から呼び出され、MERGE 文の複雑な条件分岐（MATCHED/NOT MATCHED）を解決する役割を担います。

責務

  • MERGE 文の構文解析結果（MergeStmt）を、実行可能な Query ノードへ変換する関数の宣言。
  • MERGE 文特有のスコープ管理（ターゲットテーブルとソーステーブルの可視性制御）に必要なインターフェースの提供。

重要なシンボル

  • transformMergeStmt: MERGE 文を解析し、Query 構造体を生成する主要なエントリポイントです。

制御フローとデータフロー

1. パーサーが MERGE 文を検出すると、transformMergeStmt が呼び出されます。 2. ParseState を介して、ターゲットテーブルとソーステーブルの RangeTblEntry を特定します。 3. MergeWhenClause の種類（MATCHED, NOT MATCHED BY TARGET, NOT MATCHED BY SOURCE）に応じて、ParseState の名前空間（Namespace）の可視性を動的に切り替えます。 4. 各アクション（INSERT/UPDATE/DELETE）の式を変換し、最終的な Query ノードを構築します。

直接依存

  • parser/parse_node.h: ParseState や MergeStmt などの基本的な解析用データ構造の定義に依存しています。

関連コード

  • src/backend/parser/parse_merge.c: 本ヘッダーで宣言された関数の実装が含まれており、名前空間の制御やアクションの変換ロジックが記述されています。
  • src/include/nodes/parsenodes.h: MergeStmt や MergeWhenClause などの構文解析ツリーノードの定義が含まれています。

読者向けメモ

  • MERGE 文は、ターゲットとソースという2つの異なるリレーションを同時に扱うため、他の DML 文よりも名前空間の管理が複雑です。
  • transformMergeStmt を読む際は、setNamespaceForMergeWhen がどのように各アクションの可視性を制御しているかに注目すると、PostgreSQL がどのように MERGE のスコープを解決しているか理解しやすくなります。
