src/include/parser/parse_expr.h

概要

parse_expr.h は、PostgreSQLのパーサーにおいて、構文解析ツリー（Raw Parse Tree）上の式（Expression）を、意味解析済みのノードツリー（Expression Tree）へ変換するためのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

src/include/parser モジュールにおいて、SQL文中の式（WHERE句、SELECTリスト、JOIN条件など）を処理する中心的な役割を担います。Rawパーサーが生成した未解決のノードを、型チェックや名前解決が完了した実行可能な形式へ変換する「意味解析（Semantic Analysis）」の入り口となります。

責務

  • **式の変換**: transformExpr を通じて、任意の式ノードを意味解析済みのノードへ再帰的に変換する。
  • **コンテキスト管理**: ParseExprKind を用いて、式がどの場所（SELECTリスト、WHERE句など）で使われているかを追跡し、適切なバリデーションを行う。
  • **GUCパラメータの公開**: Transform_null_equals のような、式変換の挙動を制御するグローバル設定の管理。

重要なシンボル

  • transformExpr: 式変換のメインエントリーポイント。再帰的に各ノードの変換関数を呼び出します。
  • ParseExprKind: 式が使用される場所を示す列挙型。エラーメッセージの生成や、特定の場所でのみ許可される構文の制限に使用されます。
  • Transform_null_equals: x = NULL を x IS NULL として扱うかを制御するGUCパラメータ。

制御フローとデータフロー

1. **呼び出し**: parse_analyze などの上位モジュールが transformExpr を呼び出します。 2. **再帰的変換**: transformExpr は parse_expr.c 内の内部関数 transformExprRecurse を呼び出し、ノードの型に応じて transformFuncCall や transformColumnRef などの専用関数へディスパッチします。 3. **状態更新**: ParseState を介して、現在のクエリのスコープや名前空間情報を参照・更新しながら、ノードを変換します。 4. **結果**: 意味解析済みの Node ツリーが返されます。

直接依存

  • parser/parse_node.h: ParseState や ParseExprKind などの基本的なデータ構造定義を提供します。

関連コード

  • src/backend/parser/parse_expr.c: 本ヘッダーで宣言された関数の実装本体。
  • src/include/parser/parse_coerce.h: 型変換（Coercion）のロジック。式変換中に頻繁に呼び出されます。
  • src/include/parser/parse_func.h: 関数呼び出しの解決ロジック。
  • src/include/parser/parse_oper.h: 演算子の解決ロジック。

読者向けメモ

  • 本ファイル自体は非常に簡潔ですが、実装である parse_expr.c は非常に巨大です。特定の式（CASE文、SubLink、FuncCallなど）の変換ロジックを追う際は、parse_expr.c 内の transformExprRecurse からの分岐を起点にすると理解が早いです。
  • ParseExprKind は、エラー発生時に「SELECTリストではこの関数は使えません」といった詳細なメッセージを出すために不可欠な情報です。新しい構文を追加する際は、この列挙型を適切に活用してください。
