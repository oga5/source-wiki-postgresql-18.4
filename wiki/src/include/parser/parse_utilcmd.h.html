src/include/parser/parse_utilcmd.h

概要

parse_utilcmd.h は、PostgreSQLにおけるDDL（データ定義言語）などのユーティリティコマンドに対する「解析（Parse Analysis）」処理のインターフェースを定義するヘッダーファイルです。

モジュール内での役割

パーサーサブシステムにおいて、SELECT や INSERT といったDMLの解析とは異なり、CREATE TABLE や ALTER TABLE などのユーティリティコマンドは、実行直前の段階で解析を行う必要があります。本ファイルは、それらのコマンドを変換・検証するための関数群を公開しています。

責務

  • ユーティリティコマンド（CREATE, ALTER, INDEX 等）の構文木を、実行可能な形式へ変換する関数の宣言。
  • テーブル定義、インデックス定義、パーティション境界などの複雑な構文要素を解析するためのAPI提供。
  • 実行時に必要となる「事前処理（before）」および「事後処理（after）」のリスト生成をサポートするインターフェースの定義。

重要なシンボル

  • transformCreateStmt: CREATE TABLE 文を解析し、実行に必要なリストを生成します。
  • transformAlterTableStmt: ALTER TABLE 文を解析し、実行前後の処理リストを構築します。
  • transformIndexStmt: インデックス作成文の解析を行います。
  • transformPartitionBound: パーティションテーブルの境界定義を解析・検証します。
  • expandTableLikeClause: CREATE TABLE ... (LIKE ...) 構文における定義の展開を処理します。

制御フローとデータフロー

本ファイルで宣言された関数は、主に src/backend/parser/parse_utilcmd.c で実装されています。 1. **入力**: パーサーが生成した生の構文木（CreateStmt 等）を受け取ります。 2. **変換**: カタログ情報（pg_type, pg_class 等）を参照し、型チェックや制約の検証を行います。 3. **出力**: 実行エンジンが処理可能な形式（List や IndexStmt 等）に変換されたノードを返します。一部の関数は、実行前後に実行すべき追加のコマンドリストをポインタ経由で返します。

直接依存

  • parser/parse_node.h: 解析状態を保持する ParseState 構造体などの定義に依存しています。

関連コード

  • src/backend/parser/parse_utilcmd.c: 本ヘッダーで宣言された関数の実装本体。
  • src/include/nodes/parsenodes.h: 解析対象となる各種 Stmt 構造体の定義元。
  • src/backend/commands/: 解析後のコマンドを実行する各コマンドハンドラ（tablecmds.c など）。

読者向けメモ

  • **実行タイミングの重要性**: ユーティリティコマンドの解析は、クエリツリーのキャッシュ問題を防ぐため、解析時ではなく実行開始時に行われます。この設計意図を理解することが、実装を追う際の鍵となります。
  • **構造体の共有**: 実装ファイル（.c）内では CreateStmtContext などの内部的な共有構造体が定義されており、複雑な解析処理をサブルーチン間で受け渡すために使用されています。
