src/include/parser/parsetree.h

概要

parsetree.h は、PostgreSQLの解析フェーズで生成される「パースツリー（Query 構造体）」の各コンポーネントにアクセスするためのユーティリティ関数およびマクロを定義するヘッダーファイルです。

モジュール内での役割

パーサーサブシステムにおいて、解析結果である Query 構造体やその内部の RangeTblEntry（RTE）、TargetEntry などのノード構造を操作するためのインターフェースを提供します。複雑なノード構造を直接走査するのではなく、特定の属性や情報を抽出するための抽象化レイヤーとして機能します。

責務

  • **レンジテーブル（Range Table）操作**: インデックスに基づいた RTE の取得および属性情報の抽出。
  • **ターゲットリスト（Target List）操作**: 結果セットの列（TargetEntry）を属性番号で検索。
  • **ロック情報管理**: FOR UPDATE/SHARE 句に関連する行マーク情報の取得。

重要なシンボル

  • rt_fetch(index, rangetable): レンジテーブルのインデックスから対応する RangeTblEntry を取得するマクロ。1始まりのインデックスをリストの0始まりに変換します。
  • get_rte_attribute_name(): RTE から特定の属性名を取得します。
  • get_rte_attribute_is_dropped(): 属性が削除済みかどうかを判定します。
  • get_tle_by_resno(): ターゲットリストから指定された結果番号（resno）に対応する TargetEntry を検索します。
  • get_parse_rowmark(): クエリ内の特定の RTE に対する RowMarkClause を取得します。

制御フローとデータフロー

本ファイルは関数定義の宣言とマクロのみで構成されており、直接的なロジックフローは持ちません。

  • **データフロー**: 呼び出し元から Query 構造体や List 構造体が渡され、特定のノードや属性情報が返されます。
  • **安全性**: rt_fetch マクロは範囲外アクセスに対してガードがないため、呼び出し側でインデックスの妥当性を保証する必要があります。

直接依存

  • nodes/parsenodes.h: Query, RangeTblEntry, TargetEntry, RowMarkClause などの主要なパースツリーノード定義を提供。

関連コード

  • src/include/nodes/parsenodes.h: パースツリーの構造そのものを定義する基盤ファイル。
  • src/backend/parser/parsetree.c: 本ヘッダーで宣言された関数の実装が含まれます。
  • src/include/parser/parse_node.h: パースツリー構築時の状態管理（ParseState）に関連する定義。

読者向けメモ

  • 本ファイルは、パーサーの出力である Query 構造体を解析・変換するフェーズ（analyze や rewrite）で頻繁に利用されます。
  • rt_fetch は非常に頻繁に使用されるマクロですが、インデックスエラー時のクラッシュリスクがあるため、使用時は必ずインデックスが有効範囲内であることを確認してください。
  • 複雑なノード操作を行う前に、まずはこのファイルで提供されているアクセサ関数で目的のデータが取得できないか確認することをお勧めします。
