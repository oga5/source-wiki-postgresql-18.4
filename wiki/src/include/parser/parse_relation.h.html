src/include/parser/parse_relation.h

概要

本ファイルは、PostgreSQLのパーサーがSQL文内のリレーション（テーブル、ビュー、サブクエリ、CTE、関数など）を解決・管理するためのインターフェースを定義しています。クエリ解析の過程で、名前解決や範囲テーブル（Range Table）の構築を行うための主要な関数群を提供します。

モジュール内での役割

src/include/parser モジュールにおいて、パーサーが参照する名前空間（Namespace）とリレーションの対応付けを担う中心的なヘッダーです。parse_node.h で定義された ParseState を受け取り、クエリ内の識別子を具体的なリレーションやカラムへ変換する役割を果たします。

責務

  • **名前空間管理**: refname（テーブル名やエイリアス）から ParseNamespaceItem を検索・解決する。
  • **範囲テーブル構築**: addRangeTableEntry 系関数群を通じ、各種ソース（テーブル、JOIN、CTE等）を RangeTblEntry (RTE) として ParseState に登録する。
  • **カラム解決**: colNameToVar 等により、クエリ内の列参照を Var ノードへ変換する。
  • **権限・属性管理**: テーブルのオープン、列の権限チェック、属性情報の取得を行う。

重要なシンボル

  • ParseNamespaceItem: 名前空間内のエントリを抽象化した構造体。
  • addRangeTableEntry*: 各種ソースからRTEを作成し、名前空間に追加する一連の関数群。
  • colNameToVar: 名前から列参照を解決し、適切な Var ノードを生成する。
  • expandRTE: RTEが保持する列リストを展開する（SELECT * 等の処理で使用）。
  • errorMissingRTE / errorMissingColumn: 名前解決失敗時の標準的なエラー報告関数。

制御フローとデータフロー

1. **名前解決**: パーサーが識別子に遭遇すると、refnameNamespaceItem や scanNSItemForColumn を呼び出し、現在の ParseState 内の名前空間を走査します。 2. **RTE登録**: FROM 句などの解析時、addRangeTableEntry 系関数が呼び出され、ParseState の p_rtable リストに新しいエントリが追加されます。 3. **ノード変換**: 解決された列は Var ノードに変換され、必要に応じて markNullableIfNeeded 等で属性が付与されます。

直接依存

  • parser/parse_node.h: ParseState や ParseNamespaceItem 等の基本構造体の定義。

関連コード

  • src/backend/parser/parse_relation.c: 本ヘッダーで宣言された関数の実装。
  • src/include/nodes/parsenodes.h: RangeTblEntry や Query 構造体など、パーサーが扱う主要なノード定義。
  • src/include/parser/parse_expr.h: 式の解析に関連する補助機能。

読者向けメモ

  • 本ファイルは非常に多くの関数をエクスポートしていますが、主に「名前空間の検索」「RTEの追加」「列の解決」の3つのカテゴリに分類できます。
  • sublevels_up 引数は、サブクエリやCTEなど、現在のスコープから見て何階層上の名前空間を参照しているかを示す重要なパラメータです。
  • addRangeTableEntry 系関数は、クエリ解析の初期段階で RangeTblEntry を構築するために頻繁に使用されます。実装を読む際は、まず addRangeTableEntry の呼び出し元を追うのが効率的です。
