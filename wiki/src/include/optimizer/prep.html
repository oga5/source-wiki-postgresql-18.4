src/include/optimizer/prep.h

概要

src/include/optimizer/prep.h は、クエリ最適化の初期段階である「前処理（Preprocessing）」フェーズで実行される関数のプロトタイプを定義するヘッダーファイルです。

モジュール内での役割

本ファイルは src/backend/optimizer/prep/ ディレクトリに格納された実装群のインターフェースを提供します。パーサーが生成した初期の Query 構造体を、プランナが最適化しやすい形式へと変換・正規化する役割を担います。

責務

  • **クエリの正規化**: サブクエリの引き上げ（pull-up）、結合の簡略化、不要な RTE（Range Table Entry）の削除など、クエリ構造の最適化。
  • **ターゲットリストの処理**: 更新対象カラムの抽出やターゲットリストの正規化。
  • **集約関数の準備**: 集約コストの算出や集約参照の事前処理。
  • **集合演算の計画**: UNION, INTERSECT, EXCEPT などの集合演算をプランツリーに変換するための準備。

重要なシンボル

  • pull_up_subqueries: サブクエリを親クエリに統合し、最適化の範囲を広げる重要な関数。
  • reduce_outer_joins: 外部結合を内部結合に変換可能か判定し、結合順序の自由度を高める。
  • preprocess_targetlist: クエリのターゲットリストをプランナが扱いやすい形式に変換する。
  • plan_set_operations: 集合演算（UNION等）を処理するためのエントリポイント。

制御フローとデータフロー

本ファイル自体は宣言のみですが、ここに含まれる関数群は planner() 関数が呼び出された直後の「前処理フェーズ」で順次実行されます。 1. **変換**: Query 構造体を受け取り、内部の jointree や rtable を書き換えます。 2. **伝播**: PlannerInfo 構造体を介して、前処理の結果を後続のパス探索フェーズへ引き継ぎます。

直接依存

  • nodes/pathnodes.h: PlannerInfo や RelOptInfo などの最適化用データ構造の定義。
  • nodes/plannodes.h: PlanRowMark や AggClauseCosts などのプラン生成用構造体の定義。

関連コード

  • src/backend/optimizer/prep/: 本ヘッダーで宣言された関数の実体が含まれるディレクトリ。
  • src/include/nodes/parsenodes.h: 前処理の対象となる Query 構造体が定義されている。

読者向けメモ

  • 本ファイルは、PostgreSQLのクエリ最適化における「論理的な書き換え」の境界線です。ここで実行される処理は、コストベースの最適化（パス探索）の前に行われるため、クエリの実行計画に劇的な影響を与える可能性があります。
  • pull_up_subqueries や reduce_outer_joins は、複雑なクエリのパフォーマンスを左右する重要な最適化ステップです。特定のクエリが期待通りに最適化されない場合、これらの前処理ロジックを確認することが推奨されます。
