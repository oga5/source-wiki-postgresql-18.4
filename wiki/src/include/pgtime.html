src/include/pgtime.h

概要

pgtime.h は、PostgreSQL 内部で使用されるタイムゾーンおよび時刻変換ライブラリのインターフェースを定義するヘッダーファイルです。標準 C ライブラリの時刻関数をラップし、64 ビットの時刻表現や PostgreSQL 固有のタイムゾーン管理機能を提供します。

モジュール内での役割

src/timezone/ ディレクトリに実装されたタイムゾーン処理ロジックを、データベースエンジン全体から利用可能にするための公開 API です。時刻計算やタイムゾーン変換を行う際の標準的なデータ型と関数群を定義しています。

責務

  • **時刻表現の抽象化**: 64 ビット符号付き整数 pg_time_t を用いた時刻管理。
  • **構造化時刻の提供**: pg_tm 構造体による、年・月・日・時・分・秒の分解表現の提供。
  • **タイムゾーン操作**: タイムゾーンのロード、列挙、およびオフセット計算のインターフェース提供。
  • **フォーマット変換**: pg_strftime による時刻の文字列化。

重要なシンボル

  • pg_time_t: 64 ビット幅の時刻型。プラットフォーム依存を排除し、長期的な時刻計算を保証します。
  • pg_tm: struct tm に似た構造体ですが、PostgreSQL の日付時刻関数との整合性を考慮した定義になっています。
  • pg_tz: タイムゾーン情報を保持する不透明な（opaque）構造体。
  • session_timezone / log_timezone: 現在のセッションおよびログ出力で使用されるタイムゾーンへのポインタ。

追加の索引済みシンボル

  • pg_tzenum

制御フローとデータフロー

  • **初期化**: pg_timezone_initialize() が起動時に呼ばれ、タイムゾーン環境をセットアップします。
  • **変換**: pg_localtime() や pg_gmtime() を介して、pg_time_t から pg_tm への変換が行われます。
  • **列挙**: pg_tzenumerate_start/next/end を使用して、システムで利用可能なタイムゾーンを反復処理します。
  • **解決**: pg_tzset() により、文字列名から pg_tz 構造体を取得し、以降の時刻計算に使用します。

直接依存

本ファイルは標準的な C ヘッダーや他の PostgreSQL ヘッダーを直接インクルードしていません。純粋なインターフェース定義として構成されています。

関連コード

  • src/timezone/localtime.c: pg_localtime 等の時刻変換ロジックの実装。
  • src/timezone/strftime.c: pg_strftime の実装。
  • src/timezone/pgtz.c: タイムゾーンのロードや管理、pg_tz 構造体の操作ロジック。

読者向けメモ

  • **注意**: pg_tm 構造体の tm_mon（月）や tm_year（年）の扱いは、IANA タイムゾーンライブラリの慣習（0 起点や 1900 年基準）に従っています。PostgreSQL の他の日付時刻関数（1 起点や紀元前基準）とは異なるため、相互運用時には必ず調整が必要です。
  • **不透明性**: pg_tz や pg_tzenum の内部構造は外部に公開されていません。これらを操作する際は、必ず提供されている API 関数を経由してください。
