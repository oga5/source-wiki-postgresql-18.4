src/include/jit/llvmjit_emit.h

概要

llvmjit_emit.h は、PostgreSQL の JIT コンパイル処理において、LLVM IR（中間表現）を生成するためのラッパー関数群を提供するヘッダーファイルです。LLVM C API を直接呼び出す際の冗長な記述を簡略化し、コードの可読性と保守性を向上させることを目的としています。

モジュール内での役割

src/include/jit モジュールにおいて、LLVM IR 生成のフロントエンド的な役割を果たします。他の JIT 関連ソースコードが LLVM の機能を活用する際、定型的な命令生成を抽象化するユーティリティ層として機能します。

責務

  • **定数生成の簡略化**: 整数型やポインタ型の定数を生成するヘルパーを提供。
  • **メモリ操作の抽象化**: GEP (GetElementPtr) や Load 命令を組み合わせた構造体アクセスを簡略化。
  • **制御フロー構築**: 基本ブロック（Basic Block）の動的な生成と命名をサポート。
  • **属性付与**: 関数呼び出しサイトへの最適化属性（alwaysinline 等）の付与。

重要なシンボル

  • l_ptr_const, l_int*_const: LLVM IR 上の定数値を生成する関数群。
  • l_struct_gep, l_load_struct_gep: 構造体メンバーへのアクセスとロードを統合した関数。
  • l_bb_before_v, l_bb_append_v: printf 形式のフォーマットで名前を指定して基本ブロックを生成する関数。
  • l_callsite_alwaysinline: LLVM の呼び出しサイトにインライン化属性を付与する関数。

制御フローとデータフロー

本ファイルはインライン関数のみで構成されており、呼び出し側のコンテキストで直接展開されます。

  • **データフロー**: LLVMBuilderRef を介して命令が順次生成され、LLVMValueRef として結果が返されます。
  • **制御フロー**: l_bb_* 系関数により、LLVM の基本ブロックチェーンを動的に構築します。また、l_funcnull 等の関数は、FunctionCallInfoData 構造体から引数の値や NULL 状態を抽出するデータフローを抽象化しています。

直接依存

  • llvm-c/Core.h: LLVM IR 生成のコア API。
  • llvm-c/Target.h: ターゲット固有の定義。
  • jit/llvmjit.h: PostgreSQL 側の JIT コンテキストや型定義。

関連コード

  • src/include/jit/llvmjit.h: JIT コンテキストや共通の型定義が保持されており、本ファイルの関数が依存する型（TypeSizeT 等）の定義元です。
  • src/backend/jit/llvm/: 実際にこれらのヘルパーを使用して LLVM IR を生成する実装コード群。

読者向けメモ

  • 本ファイルは USE_LLVM が定義されている環境でのみ有効です。
  • pg_attribute_printf を使用している関数は、定義と宣言を分離しているため注意してください。
  • 構造体アクセス（l_struct_gep 等）を行う際は、llvmjit.h で定義されたフィールド番号定数（FIELDNO_...）と組み合わせて使用するのが一般的です。
  • LLVM C API のバージョンアップに伴う変更を吸収する層でもあるため、複雑な命令生成を行う場合は、まず本ファイルにヘルパーが存在しないか確認することを推奨します。
