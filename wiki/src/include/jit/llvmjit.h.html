src/include/jit/llvmjit.h

概要

PostgreSQL の LLVM JIT コンパイル機能における中核的なヘッダーファイルです。LLVM の C API をラップし、PostgreSQL の内部データ構造（タプル、式状態、関数呼び出し情報など）を LLVM の型システムにマッピングするための定義を提供します。

モジュール内での役割

src/backend/jit/llvm/llvmjit.c と対をなし、JIT コンパイルのセッション管理、型定義の共有、およびコード生成のためのインターフェースを定義します。JIT サブシステム全体の「型定義のカタログ」および「コンテキスト管理の窓口」として機能します。

責務

  • **型定義の共有**: PostgreSQL の C 構造体に対応する LLVM 型（StructExprState 等）の宣言。
  • **コンテキスト管理**: LLVMJitContext を通じた、LLVM モジュール、リソース所有者、およびコンパイル状態の追跡。
  • **API 抽象化**: LLVM の複雑な C API を PostgreSQL の実行エンジンから扱いやすくするためのラッパー関数の提供。
  • **エラー・メモリ管理**: OOM（Out of Memory）発生時の安全な終了処理や、リソース解放の制御。

重要なシンボル

  • LLVMJitContext: JIT コンパイルのライフサイクルを管理する主要な構造体。
  • TypePGFunction, StructExprState 等: PostgreSQL の内部構造を LLVM IR 上で表現するための型定義群。
  • llvm_create_context(): 新しい JIT コンテキストを初期化するエントリポイント。
  • llvm_compile_expr(): 式の評価を LLVM IR にコンパイルする主要な関数。

制御フローとデータフロー

  • **初期化**: llvm_create_context が呼ばれると、LLVMJitContext が生成され、リソース所有者（ResourceOwner）に登録されます。
  • **コード生成**: llvm_mutable_module を介してモジュールを取得し、llvm_compile_expr 等の関数が LLVM IR を生成します。
  • **実行**: 生成された IR は LLVM の ORC JIT エンジンによって機械語に変換され、llvm_get_function を通じて実行可能な関数ポインタとして取得されます。

直接依存

  • jit/llvmjit_backport.h: LLVM バージョン間の差異を吸収する互換レイヤー。
  • llvm-c/Types.h: LLVM C API の基本型定義。
  • access/tupdesc.h, fmgr.h, nodes/pg_list.h: PostgreSQL のコアデータ構造。
  • jit/jit.h
  • llvm-c/OrcEE.h

関連コード

  • src/backend/jit/llvm/llvmjit.c: 本ヘッダーで宣言された関数の実装および静的変数の管理。
  • src/include/jit/llvmjit_emit.h: LLVM IR を生成するための具体的なヘルパー関数群。
  • src/include/jit/jit.h: JIT サブシステムの汎用的なインターフェース定義。

読者向けメモ

  • **LLVM 非依存性**: USE_LLVM マクロで囲まれており、LLVM がインストールされていない環境でもコンパイルが通るよう設計されています。
  • **C++ 互換性**: extern "C" ブロックにより、C++ コードからも安全にインクルード可能です。
  • **型定義の注意**: ここで宣言されている Struct... 型は、PostgreSQL の C 構造体とメモリレイアウトが一致している必要があります。変更時は必ず対応する C 構造体定義と整合性を確認してください。
