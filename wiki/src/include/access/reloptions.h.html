src/include/access/reloptions.h

概要

reloptions.h は、PostgreSQL のリレーション（テーブル、インデックス等）やテーブルスペースに対して設定可能なオプション（pg_class.reloptions や pg_tablespace.spcoptions）を定義・管理するためのコアフレームワークを提供します。

モジュール内での役割

src/include/access モジュールにおいて、各アクセスメソッド（AM）やストレージ層が独自のパラメータを定義し、それらを一貫した方法で解析・検証・保持するための基盤となります。

責務

  • **オプション定義の型システム**: bool, int, real, enum, string 型のオプション定義構造体を提供。
  • **名前空間と種類の管理**: relopt_kind を用いて、オプションがどのオブジェクト（Heap, B-tree, BRIN 等）に適用可能かを識別。
  • **解析と検証**: ユーザーが指定したオプション文字列を解析し、型や範囲の妥当性をチェックする関数のインターフェースを提供。
  • **ロックレベルの管理**: オプション変更時に必要な LOCKMODE を定義し、同時実行時の整合性を保証。

重要なシンボル

  • relopt_kind: オプションの適用対象を分類するビットマスク。
  • relopt_gen: 全てのオプション型が共通して持つヘッダー構造体。
  • relopt_value: 解析済みのオプション値を保持する構造体。
  • add_*_reloption: グローバルなオプション定義を追加するための関数群。
  • build_reloptions: Datum 形式のオプションを解析し、構造体に変換するメイン関数。

制御フローとデータフロー

1. **定義**: 各モジュール（例: heapam.c）が add_..._reloption を呼び出し、サポートするオプションを登録します。 2. **解析**: ALTER TABLE SET (...) 等でユーザーがオプションを指定すると、transformRelOptions がリストを生成します。 3. **変換**: build_reloptions が登録された定義に基づき、入力された Datum を C 構造体（StdRdOptions 等）にマッピングします。 4. **利用**: 各モジュールは、変換後の構造体から値を読み取り、動作を動的に変更します。

直接依存

  • access/amapi.h: アクセスメソッドのインターフェース定義。
  • access/htup.h, access/tupdesc.h: タプルおよびタプル記述子の操作。
  • nodes/pg_list.h: オプションリストの管理。
  • storage/lock.h: オプション変更時のロックレベル定義。

  • src/include/access/amapi.h: インデックス AM が amoptions 関数を定義する際に参照。

読者向けメモ

  • **ロックレベルの選択**: 新規オプション追加時は、そのパラメータが実行計画やデータ読み取りにどう影響するかを考慮し、適切な LOCKMODE を選択してください（多くは ShareUpdateExclusiveLock で十分ですが、クエリ結果に影響する場合は AccessExclusiveLock が必須です）。
  • **文字列オプション**: 文字列オプションを扱う際は、validate_string_relopt コールバックによる検証を必ず実装してください。
  • **メモリレイアウト**: build_reloptions を使用する際は、relopt_parse_elt を用いて、オプション名と構造体内のオフセットを正確にマッピングする必要があります。
