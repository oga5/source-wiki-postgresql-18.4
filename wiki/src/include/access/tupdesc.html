src/include/access/tupdesc.h

概要

tupdesc.h は、PostgreSQL における「タプル記述子（Tuple Descriptor）」の定義と操作インターフェースを提供します。タプル記述子は、リレーションやクエリ結果の行構造（列数、各列の型、制約など）を記述するメタデータ構造体です。

モジュール内での役割

src/include/access モジュールにおいて、タプルの物理的な構造を解釈するための基盤となります。実行エンジン（Executor）やアクセスメソッド（AM）が、メモリ上のタプルを読み書きする際に必ず参照する中心的なデータ構造です。

責務

  • **行構造の定義**: タプルの列数、型情報、および NOT NULL 制約やデフォルト値などの制約情報の保持。
  • **効率的なメタデータアクセス**: CompactAttribute 構造体を通じた、タプル変形（Deformation）に必要な属性情報の高速アクセス。
  • **メモリ管理**: 参照カウント（tdrefcount）によるキャッシュ内タプル記述子のライフサイクル管理。

重要なシンボル

  • TupleDescData / TupleDesc: タプルの構造を定義するメイン構造体。
  • CompactAttribute: FormData_pg_attribute の軽量版。タプル変形時のパフォーマンス向上のために使用されます。
  • TupleDescAttr(): FormData_pg_attribute 配列へのアクセス用インライン関数。
  • TupleDescCompactAttr(): CompactAttribute 配列へのアクセス用インライン関数。
  • PinTupleDesc / ReleaseTupleDesc: 参照カウントの増減を行うマクロ。

追加の索引済みシンボル

  • AttrDefault
  • ConstrCheck
  • TupleConstr
  • TupleDescAttr
  • TupleDescCompactAttr

制御フローとデータフロー

1. **初期化**: CreateTupleDesc 等により生成され、natts（属性数）に基づいたメモリ領域が確保されます。 2. **同期**: FormData_pg_attribute（カタログ由来の定義）が変更された場合、populate_compact_attribute() を呼び出して compact_attrs を更新する必要があります。 3. **アクセス**: 実行エンジンは TupleDescCompactAttr() を介して属性の型やアライメント情報を取得し、タプルのバイト列を解析（変形）します。 4. **解放**: 参照カウントが 0 になると FreeTupleDesc() が呼ばれ、メモリが解放されます。

直接依存

  • access/attnum.h: 属性番号の定義。
  • catalog/pg_attribute.h: FormData_pg_attribute の定義。
  • nodes/pg_list.h: リスト構造のサポート。

関連コード

  • src/include/access/htup_details.h: タプル（HeapTuple）自体の構造定義。

読者向けメモ

  • **パフォーマンス**: CompactAttribute は 16 バイトに抑えられています。この構造体を拡張する場合は、タプル変形処理の性能に直結するため細心の注意が必要です。
  • **整合性**: FormData_pg_attribute を直接書き換えた後は、必ず populate_compact_attribute() を呼び出してください。さもないと、CompactAttribute との不整合が発生します。
  • **参照カウント**: tdrefcount が -1 の場合は、キャッシュ管理対象外（スタック上や一時的なメモリコンテキスト）であることを示します。この場合、Pin/Release は何もしません。
