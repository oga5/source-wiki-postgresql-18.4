src/include/access/attmap.h

概要

attmap.h は、異なる行型（TupleDesc）間での属性（列）の対応関係を管理するためのデータ構造とインターフェースを定義するヘッダーファイルです。

モジュール内での役割

src/include/access モジュールにおいて、主に継承やパーティショニング、あるいはテーブルの再構築時に発生する「論理的に等価だが物理的な列順序や削除済み列の有無が異なる」タプル間の変換を支援します。

責務

  • 属性マッピングを保持する AttrMap 構造体の定義。
  • 変換元と変換先の TupleDesc を比較し、列の対応表を生成する関数の宣言。
  • マッピング構造体のメモリ確保および解放の管理。

重要なシンボル

  • AttrMap: 属性番号の対応表を保持する構造体。attnums 配列は出力側の各属性に対応する入力側の属性番号（1始まり）を格納し、削除済み列や対応なしの場合は 0 となります。
  • build_attrmap_by_position(): 位置ベースで列を照合し、型の一致を確認してマッピングを構築します。
  • build_attrmap_by_name(): 列名ベースで照合を行い、より柔軟なマッピングを構築します。

制御フローとデータフロー

1. **構築**: build_attrmap_by_... 関数が、入力と出力の TupleDesc を走査します。 2. **照合**: 削除済み列（attisdropped）をスキップしつつ、型や名前の整合性を検証します。 3. **格納**: 適合する列番号を AttrMap->attnums に記録します。 4. **利用**: 生成された AttrMap は、タプル変換処理（tupconvert.c など）において、実際のデータコピー時のインデックス参照に使用されます。

直接依存

  • access/attnum.h: AttrNumber 型の定義。
  • access/tupdesc.h: TupleDesc 構造体の定義。

関連コード

  • src/backend/access/common/attmap.c: 本ヘッダーで宣言された関数の実装。
  • src/include/access/tupconvert.h: AttrMap を利用して実際にタプルを変換する機能群。

読者向けメモ

  • AttrMap の attnums 配列のインデックスは出力側の属性位置に対応しています。
  • attnums[i] == 0 は、その列が削除済みであるか、あるいは対応する入力列が存在しないことを示します。
  • 継承やパーティションの DDL 操作を追う際に、列の物理的なズレをどう吸収しているかを理解するための基点となります。
