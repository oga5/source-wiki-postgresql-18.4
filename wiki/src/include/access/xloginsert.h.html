src/include/access/xloginsert.h

概要

xloginsert.h は、PostgreSQL の WAL (Write-Ahead Logging) レコードを構築するための公開インターフェースを定義するヘッダーファイルです。データベースの変更操作を WAL レコードとしてシリアライズし、ディスクへ書き出す前段階の準備プロセスを提供します。

モジュール内での役割

src/include/access モジュールにおいて、WAL 生成の「構築フェーズ」を担います。低レベルなストレージ操作（バッファの変更など）と、高レベルな WAL 書き込み処理（xlog.c）を繋ぐ役割を果たします。

責務

  • WAL レコード構築のための作業領域の管理と初期化。
  • 変更対象のバッファ（ページ）の登録と、それに対するフルページイメージの要否判定。
  • レコード本体のデータ（XLogRegisterData）およびページ固有のデータ（XLogRegisterBufData）の登録。
  • 構築されたデータを最終的な WAL レコード形式に組み立てるための API 提供。

重要なシンボル

  • XLogBeginInsert(): WAL レコード構築の開始を宣言します。
  • XLogRegisterBuffer(): 変更対象のバッファを登録し、REGBUF_* フラグでフルページイメージの取得方法を制御します。
  • XLogInsert(): 構築したデータを WAL レコードとして確定させ、ログシーケンス番号 (LSN) を返します。
  • REGBUF_* マクロ群: ページイメージの扱い（強制取得、スキップ、初期化など）を定義するフラグです。
  • XLR_NORMAL_MAX_BLOCK_ID / XLR_NORMAL_RDATAS: 通常のレコード構築で利用されるデフォルトの作業領域サイズです。

制御フローとデータフロー

1. **開始**: XLogBeginInsert() を呼び出し、内部の作業領域をリセットします。 2. **登録**: XLogRegisterBuffer() で変更対象のページを、XLogRegisterData() でレコード本体のデータを登録します。必要に応じて XLogEnsureRecordSpace() で領域を拡張します。 3. **組み立て**: XLogInsert() が呼び出されると、xloginsert.c 内でこれらが XLogRecData 構造体のチェーンとして連結され、WAL レコードヘッダーが付与されます。 4. **書き込み**: 組み立てられたレコードが WAL バッファにコピーされ、ディスクへのフラッシュ待ち状態となります。

読者向けメモ

  • WAL レコードの構築は、必ず XLogBeginInsert から始まり XLogInsert で終わる一連のシーケンスとして実行する必要があります。
  • 多くのページを一度に変更する場合、XLR_NORMAL_MAX_BLOCK_ID を超える可能性があるため、事前に XLogEnsureRecordSpace を呼び出すことが推奨されます。
  • REGBUF_WILL_INIT などのフラグは、リカバリ時の挙動に直結するため、ページの状態を正確に反映させる必要があります。
