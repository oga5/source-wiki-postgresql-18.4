src/include/access/tsmapi.h

概要

tsmapi.h は、PostgreSQL の TABLESAMPLE 句で利用されるサンプリングメソッドを定義するためのインターフェース（API）を規定するヘッダーファイルです。

モジュール内での役割

src/include/access モジュールにおいて、テーブルスキャン時のデータ抽出ロジックを拡張可能にするための抽象化レイヤーを提供します。特定のサンプリングアルゴリズム（例: BERNOULLI や SYSTEM）を実装するハンドラ関数が、このヘッダーで定義された構造体を通じてシステムに機能を提供します。

責務

  • サンプリングメソッドが実装すべきコールバック関数のシグネチャを定義する。
  • プランナおよびエグゼキュータがサンプリングメソッドを呼び出すための共通構造体 TsmRoutine を提供する。
  • メソッドの特性（再現性など）をメタデータとして保持する。

重要なシンボル

  • TsmRoutine: サンプリングメソッドのハンドラが返す構造体。プランナとエグゼキュータが必要とする関数ポインタとプロパティを保持します。
  • SampleScanGetSampleSize_function: プランナがコスト見積もりのために呼び出す関数。
  • NextSampleBlock_function / NextSampleTuple_function: エグゼキュータが実際にスキャンするブロックやタプルを決定するための反復関数。
  • GetTsmRoutine(): 指定された OID に対応するサンプリングメソッドの TsmRoutine を取得する関数。

追加の索引済みシンボル

  • BlockNumber
  • OffsetNumber
  • SampleScanState

制御フローとデータフロー

1. **プランニング**: プランナは SampleScanGetSampleSize を呼び出し、サンプリング後の推定ページ数とタプル数を取得してコスト計算を行います。 2. **初期化**: エグゼキュータは InitSampleScan および BeginSampleScan を呼び出し、サンプリング状態を初期化します。 3. **実行**: エグゼキュータは NextSampleBlock で次に読み込むブロックを、NextSampleTuple でそのブロック内のタプルを順次取得します。 4. **終了**: スキャン終了時に EndSampleScan が呼び出され、リソースが解放されます。

直接依存

  • nodes/execnodes.h: SampleScanState 構造体の定義に依存。
  • nodes/pathnodes.h: PlannerInfo や RelOptInfo 構造体の定義に依存。

関連コード

  • src/backend/access/tablesample/tablesample.c: GetTsmRoutine の実装およびサンプリングメソッドの管理ロジックが含まれます。
  • tablesample-method.sgml: 本 API を利用したサンプリングメソッドの実装に関する公式ドキュメント。

読者向けメモ

  • TsmRoutine を初期化する際は、必ず makeNode(TsmRoutine) を使用してください。これにより、将来的にフィールドが追加された場合でも、未定義のポインタによるクラッシュを防ぐことができます。
  • NextSampleBlock が NULL の場合、システムはデフォルトのシーケンシャルスキャン動作を想定します。特定のアルゴリズムでブロック選択を制御したい場合のみ実装してください。
