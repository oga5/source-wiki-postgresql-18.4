src/include/access/table.h

概要

src/include/access/table.h は、PostgreSQL における「テーブル」リレーションを扱うための汎用的なインターフェースを定義するヘッダーファイルです。インデックスや複合型などの他のリレーション種別と区別し、テーブルに対する安全なアクセス操作を提供します。

モジュール内での役割

src/include/access モジュール内で、テーブルアクセス層（Table Access Method: TAM）の入り口として機能します。relation_open などの低レベルなリレーション操作をラップし、対象がテーブルであることを保証するバリデーション層の役割を担います。

責務

  • テーブルのオープンおよびクローズ操作の提供。
  • リレーションを開く際に、対象がインデックスや複合型ではないことを検証する（relkind チェック）。
  • Oid または RangeVar を用いた柔軟なリレーション解決のサポート。

重要なシンボル

  • table_open / try_table_open: 指定された OID からテーブルリレーションを開く。try_ 版は存在しない場合にエラーではなく NULL を返す。
  • table_openrv / table_openrv_extended: RangeVar（名前空間やテーブル名）からテーブルを開く。
  • table_close: 開いたテーブルリレーションを閉じ、必要に応じてロックを解放する。

制御フローとデータフロー

1. 呼び出し元が table_open 等を呼び出す。 2. 内部で relation_open（relcache）を呼び出し、リレーション記述子を取得する。 3. validate_relation_kind がリレーションの relkind を検査し、インデックスや複合型であればエラーを投げる。 4. 検証を通過した Relation 構造体が呼び出し元に返される。

直接依存

  • nodes/primnodes.h: RangeVar 構造体の定義に使用。
  • storage/lockdefs.h: LOCKMODE 定数の定義に使用。
  • utils/relcache.h: Relation 構造体およびリレーションキャッシュ操作関数へのアクセスに使用。

関連コード

  • src/backend/access/table/table.c: 本ヘッダーで宣言された関数の実装。
  • src/include/access/tableam.h: テーブルアクセスメソッド（TAM）の具体的な API 定義。テーブルに対するスキャンや挿入などの操作は、この tableam.h で定義されたインターフェースを通じて行われる。

読者向けメモ

  • 本ファイルは、テーブルに対する「オープン/クローズ」という高レベルなライフサイクル管理に特化しています。
  • テーブル内のデータ操作（スキャン、挿入、更新など）を行いたい場合は、table.h ではなく tableam.h を参照してください。
  • validate_relation_kind によるチェックは、ビューや外部テーブルの扱いについては呼び出し元で別途考慮が必要な場合があります。
