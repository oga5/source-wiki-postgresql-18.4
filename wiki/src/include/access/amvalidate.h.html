src/include/access/amvalidate.h

概要

インデックスアクセスメソッド（AM）の整合性検証および構成管理を支援するためのユーティリティ関数群を定義するヘッダーファイルです。主に CREATE OPERATOR CLASS や CREATE OPERATOR FAMILY 実行時の検証処理で使用されます。

モジュール内での役割

src/include/access モジュールにおいて、インデックスのメタデータ（演算子クラスや演算子族）が論理的に正しいかを検証する層を担います。カタログキャッシュ（CatCList）から取得した情報を解析し、インデックスが期待する型や関数シグネチャを満たしているかを判定します。

責務

  • 演算子族（OpFamily）内の演算子とサポート関数のグループ化。
  • 演算子およびサポート関数のシグネチャ（引数の型や戻り値）の妥当性検証。
  • 特定のデータ型に対する演算子クラスの存在確認やソート可能性の判定。

重要なシンボル

  • OpFamilyOpFuncGroup: 演算子族内の特定の型ペアに対する演算子と関数のセットを保持する構造体。ビットマスク（uint64）を用いて、最大63番までの戦略番号や関数番号を効率的に管理します。
  • identify_opfamily_groups(): カタログから取得した演算子リストと関数リストをマージし、型ペアごとにグループ化して返す主要関数。
  • check_amproc_signature() / check_amop_signature(): 指定された関数や演算子が、AMの要件を満たすシグネチャを持つか検証する関数。

制御フローとデータフロー

1. **入力**: syscache から取得した CatCList（pg_amop および pg_amproc のエントリ）を受け取ります。 2. **グループ化**: identify_opfamily_groups がリストを走査し、lefttype と righttype の組み合わせに基づいて OpFamilyOpFuncGroup のリストを構築します。 3. **検証**: 各グループに対して、ビットマスクを確認したり、個別のシグネチャ検証関数を呼び出すことで、インデックス定義の不整合を検出します。

直接依存

  • utils/catcache.h: カタログキャッシュのデータ構造（CatCList）を利用するために必要です。

関連コード

  • src/backend/access/index/amvalidate.c: 本ヘッダーで宣言された関数の実装が含まれます。
  • src/include/catalog/pg_amop.h / pg_amproc.h: 検証対象となるシステムカタログの定義。
  • src/backend/commands/indexcmds.c: インデックス作成時にこれらの検証関数を呼び出す上位ロジックが含まれます。

読者向けメモ

  • 本ファイルで定義される関数は、主に amvalidate インデックスメソッド関数（各AMが実装する検証ロジック）から呼び出されることを想定しています。
  • uint64 のビットマスクを使用しているため、戦略番号や関数番号が64以上になるような拡張には対応していません。
  • identify_opfamily_groups を呼び出す際は、入力となる CatCList が適切にソートされていることが前提となります。
