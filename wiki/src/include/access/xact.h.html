src/include/access/xact.h

概要

src/include/access/xact.h は、PostgreSQL のトランザクション管理システムにおける中核的な定義ファイルです。トランザクションの状態遷移、分離レベル、WAL（Write-Ahead Logging）へのコミット/アボート記録の構造、およびトランザクション制御のための公開 API を定義しています。

モジュール内での役割

本ファイルは、src/backend/access/transam/xact.c と対をなし、バックエンドプロセス全体で共有されるトランザクション状態のインターフェースを提供します。トランザクションの開始・終了、サブトランザクション、セーブポイント、および 2 フェーズコミット（2PC）の制御を抽象化し、実行エンジンやコマンド処理層に対して一貫したトランザクションセマンティクスを提供します。

責務

  • **トランザクション状態管理**: トランザクションの分離レベル、読み取り専用フラグ、および実行状態の保持。
  • **WAL レコード定義**: コミット、アボート、準備（Prepare）時の WAL 物理レイアウトの定義。
  • **イベント通知**: トランザクションのライフサイクル（コミット/アボート）にフックするためのコールバック機構の提供。
  • **並列処理制御**: 並列クエリ実行時のトランザクション状態の同期と伝播。
  • **フラグ管理**: トランザクション単位での一時オブジェクトアクセスやロック状態の追跡。

重要なシンボル

トランザクション状態と分離レベル

  • XactIsoLevel: 現在のトランザクション分離レベル（READ COMMITTED, REPEATABLE READ, SERIALIZABLE）。
  • IsolationUsesXactSnapshot(): トランザクション全体で単一スナップショットを使用するか判定するマクロ。
  • MyXactFlags: トランザクションの属性（一時名前空間アクセス、排他ロック取得など）を保持するビットフラグ。

WAL レコード構造体

  • xl_xact_commit, xl_xact_abort: WAL に書き込まれる基本レコード構造。
  • xl_xact_parsed_commit, xl_xact_parsed_abort: WAL レコードを解析し、扱いやすくした構造体。
  • XLOG_XACT_*: WAL レコードタイプを識別するためのオペコード。

トランザクション制御 API

  • StartTransactionCommand(), CommitTransactionCommand(), AbortCurrentTransaction(): トランザクションの基本ライフサイクル制御。
  • BeginInternalSubTransaction(), ReleaseCurrentSubTransaction(): サブトランザクション制御。
  • RegisterXactCallback(), RegisterSubXactCallback(): トランザクションイベントへのフック登録。

制御フローとデータフロー

  • **トランザクション開始/終了**: StartTransactionCommand から始まり、xact.c 内の内部状態を更新。終了時には XactLogCommitRecord を経由して WAL を生成し、ディスクへのフラッシュを待機します。
  • **WAL 解析**: リカバリ時やレプリケーション時には、xact_redo が WAL レコードを読み込み、ParseCommitRecord 等で解析された構造体を用いてトランザクションの効果を再適用します。
  • **フラグ伝播**: MyXactFlags はトランザクションの生存期間中、特定のイベント（一時テーブル作成など）が発生するたびに更新され、コミット時のチェックや制約判定に使用されます。

直接依存

  • access/transam.h: トランザクション ID（XID）の基本定義。
  • access/xlogreader.h: WAL 解析用インターフェース。
  • storage/sinval.h: キャッシュ無効化メッセージの定義。
  • nodes/pg_list.h: リスト構造の利用。

関連コード

  • src/backend/access/transam/xact.c: 本ヘッダーで定義された関数の実装。
  • src/backend/access/transam/xactdesc.c: WAL レコードのデバッグ出力および解析ロジック。
  • src/backend/access/transam/subtrans.c: サブトランザクションの管理。
  • src/backend/access/transam/twophase.c: 2 フェーズコミットの実装。

読者向けメモ

  • **メモリとライフタイム**: xl_xact_parsed_* 構造体は、WAL レコードの解析時に一時的に生成されるものであり、永続的なメモリ管理は呼び出し側に依存します。
  • **アライメント**: WAL レコード構造体は、sizeof(int) の倍数でアライメントされるよう設計されています。これはクロスプラットフォームでの WAL 互換性を維持するためです。
  • **省略された詳細**: 本ファイルはインターフェース定義に特化しており、具体的なロック獲得ロジックや CLOG（Commit Log）への書き込み詳細は xact.c および clog.c に委譲されています。
  • **注意点**: MyXactFlags はグローバル変数ですが、バックエンドプロセスごとに独立したメモリ空間に存在するため、プロセス間での直接的な共有は行われません。
