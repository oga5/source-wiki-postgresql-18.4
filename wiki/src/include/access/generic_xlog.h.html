src/include/access/generic_xlog.h

概要

generic_xlog.h は、PostgreSQL の汎用的な WAL（Write-Ahead Logging）記録メカニズムを定義するヘッダーファイルです。特定のデータ構造に依存せず、ページレベルの変更を効率的に WAL に記録するための API を提供します。

モジュール内での役割

src/include/access モジュールにおいて、インデックスアクセス手法（AM）などが WAL 記録を簡素化するために利用する高レベルな抽象化レイヤーです。低レベルな xlog.h や xloginsert.h をラップし、開発者が個別の WAL レコード形式を定義することなく、ページ変更を安全に記録できるようにします。

責務

  • 汎用 WAL レコードの構築ライフサイクル（開始・登録・終了・破棄）の管理。
  • ページ変更の差分（デルタ）計算の抽象化。
  • ページ全体のイメージ（Full Page Image）の強制記録フラグの提供。
  • リソースマネージャ（rmgr）がリカバリ時に使用する再実行（Redo）関数のインターフェース定義。

重要なシンボル

  • GenericXLogState: WAL 記録プロセス中の状態を保持する不透明な構造体。
  • GenericXLogStart(): 記録セッションを開始し、状態を初期化する。
  • GenericXLogRegisterBuffer(): 変更対象のバッファを登録し、変更可能なページイメージを返す。
  • GenericXLogFinish(): 変更を確定し、WAL レコードを書き出して終了する。
  • GENERIC_XLOG_FULL_IMAGE: 差分ではなくページ全体を記録させるためのフラグ。

制御フローとデータフロー

1. **開始**: GenericXLogStart で状態を確保。 2. **登録**: GenericXLogRegisterBuffer を呼び出し、対象バッファを登録。返されたページポインタに対してメモリ上で変更を加える。 3. **終了**: GenericXLogFinish を呼び出すと、システムは登録されたバッファの変更内容を比較し、差分（デルタ）を計算して WAL レコードとして書き出す。 4. **リカバリ**: generic_redo が WAL レコードを読み込み、記録されたデルタをページに適用する。

直接依存

  • access/xlog.h: WAL 関連の基本定義。
  • access/xlogreader.h: WAL レコード読み込み用。
  • access/xloginsert.h: WAL 挿入用 API。
  • storage/bufpage.h: ページ構造の定義。
  • utils/rel.h: リレーション情報の参照。

関連コード

  • src/backend/access/transam/generic_xlog.c: 本ヘッダーで宣言された API の実装。デルタ計算アルゴリズムや applyPageRedo が含まれる。

読者向けメモ

  • 汎用 WAL は、ページ内の変更が比較的小規模な場合に最適です。変更がページ全体に及ぶ場合や、非常に頻繁な更新が発生する場合は、オーバーヘッドを考慮する必要があります。
  • GenericXLogRegisterBuffer で取得したページポインタに対してのみ変更を行ってください。それ以外の場所でページを直接変更すると、WAL への記録が漏れ、リカバリ時にデータ破損を引き起こします。
  • デルタ計算は MATCH_THRESHOLD に基づいて最適化されており、隣接する変更はマージされます。
