src/include/access/gist_private.h

概要

GiST (Generalized Search Tree) インデックスの内部実装に必要な非公開のデータ構造、マクロ、および関数プロトタイプを定義するヘッダーファイルです。公開 API (gist.h) とは異なり、インデックスの構築、検索、分割、WAL 記録といった低レベルな処理の詳細を規定します。

モジュール内での役割

src/include/access モジュール内において、GiST インデックスのエンジン部分を構成する各ソースファイル（gist.c, gistutil.c, gistbuild.c 等）の共通基盤として機能します。

責務

  • **状態管理**: インデックス操作中のコンテキストや関数ポインタを保持する GISTSTATE の定義。
  • **検索・挿入の補助**: 検索時のキュー管理 (GISTSearchItem) や、挿入時のスタック構造 (GISTInsertStack) の提供。
  • **ページ分割の制御**: インデックスページ分割時のレイアウト定義 (SplitPageLayout) と、バッファリング構築アルゴリズムの管理。
  • **内部定数**: ページ分割の制限値や、無効なタプルを識別するためのフラグ定義。

重要なシンボル

  • GISTSTATE: インデックスのサポート関数（consistent, union, penalty 等）の呼び出し情報や、タプル記述子を保持する中心的な構造体。
  • GISTScanOpaqueData: スキャン実行時の状態（検索キュー、メモリコンテキスト、距離情報）を保持。
  • GISTInsertStack: 挿入時にツリーを降下する際のパスを記録し、ページ分割発生時の再試行を制御。
  • GISTBuildBuffers: インデックス構築時のバッファリング処理（大量挿入の最適化）を管理する構造体。
  • GIST_MAX_SPLIT_PAGES: 1回の分割操作で許容される最大ページ数（75）。

制御フローとデータフロー

  • **検索**: GISTScanOpaqueData 内の pairingheap を使用して、距離順または深さ優先順にインデックスページやヒープタプルを探索します。
  • **挿入**: gistdoinsert が GISTInsertStack を構築しながらツリーを降下します。ページ分割が必要な場合は gistSplit が呼び出され、SplitPageLayout を通じて新しいページレイアウトが決定されます。
  • **構築**: GISTBuildBuffers を介して、インデックス構築時に一時ファイルを利用したバッファリングを行い、ランダム I/O を削減します。

直接依存

  • access/amapi.h, access/gist.h: GiST の基本定義と API。
  • access/itup.h: インデックス用タプルの操作。
  • lib/pairingheap.h: 検索順序制御のためのヒープ構造。
  • storage/bufmgr.h, storage/buffile.h: バッファ管理および一時ファイル操作。
  • utils/hsearch.h: バッファ管理用のハッシュテーブル。
  • access/genam.h

関連コード

  • src/backend/access/gist/: 本ヘッダーで宣言された関数の実装群（gist.c, gistutil.c, gistxlog.c 等）。
  • src/include/access/gist.h: GiST の公開インターフェース。

読者向けメモ

  • 本ファイルは PostgreSQL の内部実装に深く依存しています。特に GISTSTATE の各フィールドは、インデックスの演算子クラス（opclass）が提供するサポート関数と密接に関連しています。
  • TUPLE_IS_INVALID 等の定義は、古いバージョンの PostgreSQL との互換性維持のための歴史的経緯が含まれています。新規開発においてこれらのフラグを直接操作することは避けてください。
  • インデックスの構築アルゴリズム（バッファリング）を理解したい場合は、GISTBuildBuffers 構造体とその関連関数から読み進めることを推奨します。
