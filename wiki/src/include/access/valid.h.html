src/include/access/valid.h

概要

src/include/access/valid.h は、PostgreSQL のヒープタプルが特定の検索条件（スキャンキー）を満たすかどうかを判定するためのインライン関数を提供するヘッダーファイルです。

モジュール内での役割

access モジュールにおいて、タプルアクセスの検証ロジックを抽象化する役割を担います。主にヒープスキャン処理において、タプルが検索条件に合致するかを高速に判定するために使用されます。

責務

  • ヒープタプルとスキャンキーの照合ロジックの提供。
  • ScanKey に基づく属性値の抽出と、対応する比較関数の実行。
  • 検索条件（ScanKey）の評価結果に基づくブール値の返却。

重要なシンボル

  • HeapKeyTest: 本ファイルで定義される唯一の関数。タプルが指定された複数のスキャンキーすべてに合致するかを判定します。

制御フローとデータフロー

1. **入力**: HeapTuple（対象タプル）、TupleDesc（タプル記述子）、nkeys（キー数）、ScanKey（キー配列）。 2. **ループ処理**: nkeys 分だけ ScanKey を順次走査します。 3. **属性抽出**: heap_getattr を呼び出し、タプルから対象属性の値を抽出します。 4. **評価**: FunctionCall2Coll を使用して、抽出した値と ScanKey 内の定数を比較関数で評価します。 5. **判定**: 途中で一つでも条件を満たさない（または NULL 判定で失敗する）場合は即座に false を返し、すべて通過すれば true を返します。

直接依存

  • access/htup.h: HeapTuple 構造体の定義。
  • access/htup_details.h: heap_getattr 等のヒープタプル操作関数。
  • access/skey.h: ScanKey 構造体および関連定義。
  • access/tupdesc.h: TupleDesc 構造体の定義。

関連コード

  • src/include/access/heapam.h: ヒープアクセスメソッドの主要インターフェース。
  • src/include/access/relscan.h: スキャン状態を管理する構造体定義。
  • src/backend/access/common/scankey.c: スキャンキーの初期化や管理ロジック。

読者向けメモ

  • 本ファイルは static inline 関数のみで構成されており、ヘッダーをインクルードするだけで利用可能です。
  • HeapKeyTest は非常に頻繁に呼び出されるため、パフォーマンスに直結します。修正を加える際は、インライン展開によるバイナリサイズへの影響を考慮してください。
  • NULL 値の扱いは厳格であり、キーが NULL を期待していない場合に属性値が NULL であれば、即座に不一致とみなされます。
