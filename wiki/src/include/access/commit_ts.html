src/include/access/commit_ts.h

概要

commit_ts.h は、PostgreSQL におけるトランザクションのコミット時刻（Commit Timestamp）を追跡・管理するためのインターフェースを定義するヘッダーファイルです。各トランザクションがいつコミットされたか、およびレプリケーション元の情報を永続化し、照会可能にする仕組みを提供します。

モジュール内での役割

src/include/access モジュールの一部として、トランザクション管理（transam）サブシステムに属します。CLOG（コミットログ）がトランザクションのコミット状態（コミット済みか否か）を管理するのに対し、本モジュールはコミット時刻というメタデータを管理する補助的な役割を担います。

責務

  • トランザクションのコミット時刻とレプリケーション元 ID の記録および取得。
  • コミット時刻管理用共有メモリの初期化とサイズ計算。
  • チェックポイント処理やトランザクションの切り捨て（Truncate）に伴う管理データの整理。
  • WAL（Write-Ahead Logging）への記録およびリカバリ時の再実行（Redo）処理の定義。

重要なシンボル

  • track_commit_timestamp: 機能の有効/無効を制御する GUC 変数。
  • TransactionTreeSetCommitTsData: トランザクション（およびサブトランザクション）のコミット時刻を記録する主要関数。
  • TransactionIdGetCommitTsData: 指定した XID のコミット時刻とレプリケーション元 ID を取得する関数。
  • xl_commit_ts_set / xl_commit_ts_truncate: WAL ログに記録されるデータ構造。

制御フローとデータフロー

1. **記録**: トランザクション終了時に TransactionTreeSetCommitTsData が呼ばれ、共有メモリ上の SLRU（Simple Least Recently Used）キャッシュを介してコミット時刻が書き込まれます。 2. **取得**: ユーザーが pg_xact_commit_timestamp() 等の関数を呼び出すと、TransactionIdGetCommitTsData が SLRU から該当データを読み取ります。 3. **リカバリ**: commit_ts_redo が WAL レコードを解析し、クラッシュリカバリ時にコミット時刻の状態を復元します。

直接依存

  • access/xlog.h: WAL 関連の定義。
  • datatype/timestamp.h: TimestampTz 型の定義。
  • replication/origin.h: レプリケーション元 ID (RepOriginId) の定義。
  • storage/sync.h: ファイル同期関連の定義。

関連コード

  • src/backend/access/transam/commit_ts.c: 本ヘッダーで宣言された関数の実装。
  • src/backend/access/transam/slru.c: データを永続化するための SLRU キャッシュ機構。
  • src/backend/access/transam/xact.c: トランザクションのコミット処理中に本モジュールを呼び出す側。

読者向けメモ

  • 本機能は track_commit_timestamp が有効な場合にのみ動作します。無効な状態でアクセスしようとするとエラーが発生します。
  • 内部的には SLRU を使用しているため、古いトランザクションのコミット時刻は、TruncateCommitTs によって定期的に削除（切り捨て）されます。
  • 読み取り時は、対象の XID がまだ切り捨てられていないか注意が必要です。
