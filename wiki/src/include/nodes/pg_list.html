src/include/nodes/pg_list.h

概要

PostgreSQLにおける汎用リストデータ構造の定義と操作インターフェースを提供します。歴史的にはLispのコンスセルを模倣していましたが、現在はパフォーマンス最適化のため、拡張可能な配列（expansible array）として実装されています。

モジュール内での役割

src/include/nodes サブシステムにおいて、クエリツリーやプランツリーなどのノードを保持するための最も基本的なコンテナとして機能します。

責務

  • **データ構造の定義**: ポインタ、整数、OID、XIDを保持するリスト構造（List）とセル（ListCell）の定義。
  • **メモリ管理の抽象化**: リストの拡張やメモリ再配置を隠蔽し、安全な操作を提供。
  • **反復処理の提供**: foreach 等の強力なマクロ群による、リスト走査の標準化。
  • **リスト操作API**: 追加、削除、結合、検索、ソートなどの操作関数の宣言。

重要なシンボル

  • List: リストのヘッダ構造体。length（現在の要素数）と elements（配列）を保持。
  • ListCell: リスト内の各要素を保持する共用体。
  • NIL: 空リストを表すマクロ（NULLポインタ）。
  • foreach(cell, lst): リストを走査するための標準的なマクロ。
  • lfirst(lc): ListCell からポインタ値を取り出すマクロ。

制御フローとデータフロー

  • **リストの生成**: list_makeN 系マクロや lcons 等を用いてリストを生成します。
  • **走査**: foreach マクロは内部で ForEachState を使用し、リストのインデックスを順次インクリメントして ListCell へのポインタを更新します。
  • **変更**: lappend や list_delete_nth_cell 等の関数は、必要に応じて elements 配列の再割り当てを行い、新しいリストポインタを返します。

直接依存

  • nodes/nodes.h: NodeTag や castNode 等のノード基盤定義に依存。

関連コード

  • src/backend/nodes/list.c: 本ヘッダーで宣言された関数の実体。
  • src/include/nodes/nodes.h: リストが保持するノードの型定義。

読者向けメモ

  • **空リストの扱い**: 空リストは必ず NIL で表現されます。length == 0 のリストという概念は存在しません。
  • **走査中の変更**: foreach ループ内でリストの要素を削除する場合は、必ず foreach_delete_current を使用してください。通常の削除関数を使用すると、イテレータの整合性が崩れます。
  • **型安全性**: lfirst は void * を返すため、ノードを扱う際は lfirst_node(type, lc) を使用して型キャストを行うことが推奨されます。
  • **メモリ**: リストのヘッダと要素配列は、リストが空でない限り再配置されないことが保証されています。
