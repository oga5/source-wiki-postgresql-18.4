src/include/nodes/nodeFuncs.h

概要

PostgreSQLのクエリツリーや式ツリー（Node構造体）を操作するための汎用的なユーティリティ関数群を定義するヘッダーファイルです。ツリーの走査、変形、および特定のノード型の判定や属性抽出を目的としています。

モジュール内での役割

src/include/nodes モジュールにおいて、構文解析や最適化の過程で生成されるノードツリーを扱うための「標準ライブラリ」として機能します。nodeFuncs.c と対になっており、ツリー構造を再帰的に処理する複雑なロジックを抽象化します。

責務

  • **ツリー走査・変形**: walker および mutator パターンを用いた、クエリツリー全体の再帰的な探索と書き換えの提供。
  • **式属性の抽出**: exprType や exprCollation など、式ノードから型や照合順序などのメタデータを取得する機能。
  • **ノード判定・操作**: is_funcclause や get_leftop といった、特定のノード型を判定・分解するためのインライン関数群の提供。

重要なシンボル

  • **expression_tree_walker / mutator**: ツリーを再帰的に走査・変形するための基盤マクロ。
  • **query_tree_walker / mutator**: Query 構造体全体を対象とした走査・変形用。QTW_* フラグで探索範囲を制御可能。
  • **exprType**: 式ノードの評価結果の型（Oid）を返す。
  • **is_funcclause / is_opclause**: ノードが特定の式型であるかを判定する軽量なインライン関数。

制御フローとデータフロー

本ファイルは主に「コールバック駆動型」のフローを提供します。 1. **走査**: walker 関数がノードを訪問し、ユーザー定義のコールバックを呼び出します。コールバックが true を返すと走査を中断します。 2. **変形**: mutator 関数はツリーを再帰的に辿り、コールバックの戻り値でノードを置換します。 3. **フラグ制御**: query_tree_walker 等に渡される QTW_* フラグにより、サブクエリや範囲テーブル（Range Table）の探索をスキップするなどの制御が可能です。

直接依存

  • nodes/parsenodes.h: Query や RangeTblEntry などの主要なノード定義を参照するために必須。

関連コード

  • src/backend/nodes/nodeFuncs.c: 本ヘッダーで宣言された関数の実装本体。
  • src/include/nodes/nodes.h: Node 構造体の基本定義と IsA マクロ。
  • src/include/nodes/primnodes.h: OpExpr, FuncExpr などの式ノードの定義。

読者向けメモ

  • **型安全性**: walker/mutator のコールバックは void *context を受け取ります。Cの型安全性を補うため、マクロ内でキャストを行っています。
  • **インライン関数**: is_funcclause 等のインライン関数は、頻繁に呼ばれるためパフォーマンスを考慮してヘッダーに定義されています。
  • **再帰の深さ**: ツリーが非常に深い場合、スタックオーバーフローのリスクがあるため、再帰的な処理を行う際は注意が必要です。
