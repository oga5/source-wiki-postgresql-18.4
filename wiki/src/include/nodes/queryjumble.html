src/include/nodes/queryjumble.h

概要

queryjumble.h は、PostgreSQL におけるクエリの正規化およびフィンガープリント（Query ID）生成のためのデータ構造とインターフェースを定義するヘッダーファイルです。

モジュール内での役割

src/include/nodes サブシステムの一部として、パーサーが生成した Query ツリーを解析し、クエリの構造的同一性を識別するためのハッシュ値を計算する機能を提供します。主に pg_stat_statements などの拡張機能が、異なる定数を持つ同一構造のクエリをグループ化するために利用されます。

責務

  • クエリの構造をハッシュ化するための作業状態（JumbleState）の管理。
  • クエリ内の定数（定数リテラル）の位置と長さの追跡。
  • クエリ識別子（Query ID）計算の有効・無効を判定するロジックの提供。

重要なシンボル

  • JumbleState: クエリのハッシュ化プロセス中に保持される作業用構造体。ハッシュバッファや定数の位置情報を含みます。
  • LocationLen: クエリテキスト内の定数の位置と長さを保持する構造体。正規化プロセスで定数をマスクする際に使用されます。
  • ComputeQueryIdType: compute_query_id GUC パラメータの設定値（OFF, ON, AUTO, REGRESS）を定義する列挙型。
  • IsQueryIdEnabled(): 現在のシステム設定で Query ID の計算が必要かどうかを判定するインライン関数。

追加の索引済みシンボル

  • IsQueryIdEnabled

制御フローとデータフロー

1. **初期化**: JumbleQuery(Query *query) が呼び出され、JumbleState が初期化されます。 2. **トラバース**: クエリツリーを再帰的に走査し、ノードの型や構造を JumbleState のバッファに書き込みます。 3. **定数記録**: 走査中に見つかった定数の位置情報を clocations 配列に記録します。 4. **ハッシュ生成**: 最終的なバッファ内容からクエリのフィンガープリントが計算されます。 5. **正規化**: 必要に応じて、記録された位置情報を用いて元のクエリテキストから定数を除去し、正規化されたクエリ文字列を生成します。

直接依存

  • nodes/parsenodes.h: Query 構造体の定義を参照するために必須です。

関連コード

  • src/backend/nodes/queryjumble.c: 本ヘッダーで宣言された関数の実装が含まれます。
  • src/backend/utils/misc/guc.c: compute_query_id GUC パラメータの定義場所。
  • contrib/pg_stat_statements/: 本機能の主要な利用者であり、Query ID を用いて統計情報を集計します。

読者向けメモ

  • JumbleState 内の pending_nulls は、ハッシュ化の効率化のために NULL ノードの連続を遅延処理する仕組みです。
  • squashed フラグは、リスト構造が圧縮された場合にパラメータ番号の再割り当てが必要であることを示します。
  • IsQueryIdEnabled は頻繁に呼び出される可能性があるため、インライン関数として定義されています。
