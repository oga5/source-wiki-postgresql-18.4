src/include/nodes/miscnodes.h

概要

miscnodes.h は、PostgreSQL の主要なツリー構造（パースツリー、プランツリー、実行状態ツリー）のいずれにも分類されない、特殊な目的を持つノード型を定義するヘッダーファイルです。

モジュール内での役割

src/include/nodes サブシステムにおいて、特定のカテゴリに収まらない「ユーティリティ的なノード」の置き場所として機能します。主に API 間で渡されるコンテキスト情報の型安全性を確保するために NodeTag を利用します。

責務

  • 汎用的なノード分類に当てはまらないデータ構造の定義。
  • 「ソフトエラー（Soft Error）」を捕捉・伝播するための標準的なコンテキスト構造の提供。
  • IsA() マクロを用いた、実行時の型判別を可能にするための基盤提供。

重要なシンボル

  • ErrorSaveContext: 関数呼び出し間でエラー情報を安全に伝播させるための構造体。
  • SOFT_ERROR_OCCURRED(escontext): ErrorSaveContext を受け取り、エラーが発生したかどうかを簡潔に判定するマクロ。

制御フローとデータフロー

本ファイルは直接的なロジックを持ちませんが、以下のフローを支えます： 1. **初期化**: 呼び出し元が ErrorSaveContext を作成し、FunctionCallInfo.context 等にセットします。 2. **エラー捕捉**: 下位の関数がエラーを検出した際、例外を投げる代わりに ErrorSaveContext を更新し、error_occurred を true に設定します。 3. **判定**: 呼び出し元は SOFT_ERROR_OCCURRED マクロを使用して、エラーの有無をチェックし、必要に応じて error_data を参照します。

直接依存

  • nodes/nodes.h: NodeTag および IsA() マクロの定義を提供するために必須です。

関連コード

  • src/include/nodes/nodes.h: ノードシステムの基盤。すべてのノード型定義の起点。
  • src/include/utils/elog.h: ErrorData 構造体やエラー報告に関連する機能が定義されており、ErrorSaveContext と密接に関係します。

読者向けメモ

  • このファイルで定義されるノードは、SQL ツリーの構築には使用されません。主に C 言語レベルの API 設計において、関数の戻り値以外で「状態」や「エラー情報」を呼び出し元に返すための補助的な仕組みです。
  • ErrorSaveContext を使用する際は、呼び出し先がエラー情報を書き込むメモリコンテキストの生存期間に注意してください。通常は短命なコンテキストでの利用が想定されています。
