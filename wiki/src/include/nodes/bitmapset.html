src/include/nodes/bitmapset.h

概要

bitmapset.h は、PostgreSQL における非負整数の集合を効率的に表現・操作するための「Bitmapset」パッケージの定義ファイルです。主にクエリ最適化やプランナにおいて、少数の整数（数百程度まで）を扱う集合演算を高速に行うために使用されます。

モジュール内での役割

src/include/nodes サブシステムの一部として、ノード構造体とは独立した汎用的なデータ構造を提供します。他のノード構造体と異なり、メモリ効率と演算速度を最優先した設計となっており、プランナがテーブルのインデックスや属性の集合を管理する際に多用されます。

責務

  • **データ構造の定義**: Bitmapset 構造体およびその構成要素である bitmapword の定義。
  • **集合演算のインターフェース提供**: 和集合、積集合、差集合、包含関係の判定などの関数プロトタイプ定義。
  • **最適化されたビット操作**: CPUアーキテクチャに応じたビット操作（popcount等）の抽象化。
  • **メモリ管理の規約**: 空集合を NULL ポインタとして扱うことで、メモリ消費を最小化する規約の強制。

重要なシンボル

  • Bitmapset: 可変長配列を持つ構造体。nwords で保持するワード数を管理します。
  • bitmapword: アーキテクチャに応じて 32bit または 64bit を選択する型。
  • BMS_Comparison / BMS_Membership: 集合の比較結果やメンバーシップ状態を表す列挙型。
  • bms_is_empty(a): a == NULL をチェックするマクロ。空集合の判定に必須です。
  • bmw_popcount(w): ワード内のビット数を数える最適化関数。

追加の索引済みシンボル

  • signedbitmapword

制御フローとデータフロー

  • **空集合の扱い**: 空集合は NULL として表現されます。そのため、集合を操作する関数は、入力が NULL の場合を適切に処理し、結果が空になる場合は NULL を返す必要があります。
  • **メモリの再利用**: bms_add_member 等の関数は、既存の集合を拡張または変更して返す設計です。呼び出し元は、戻り値を必ず元のポインタに再代入しなければなりません（インプレース更新の保証はないため）。
  • **イテレーション**: bms_next_member を使用して、集合内の整数を順次走査するフローが一般的です。

直接依存

  • nodes/nodes.h: NodeTag 等の基本定義を利用するためにインクルードしています。

関連コード

  • src/backend/nodes/bitmapset.c: 本ヘッダーで宣言された関数の実体。
  • src/port/pg_bitutils.h: bmw_popcount 等の低レイヤーなビット操作の実装元。
  • src/include/nodes/multibitmapset.h: より大規模な集合を扱うための拡張モジュール。

読者向けメモ

  • **NULL 規約**: Bitmapset を扱う際、常に NULL が空集合であることを意識してください。bms_is_empty マクロの使用を推奨します。
  • **戻り値の保持**: 集合を変更する関数（bms_add_member 等）の戻り値は、必ず元の変数に代入してください。これは、メモリの再配置（pfree 後の再割り当て）が発生する可能性があるためです。
  • **デバッグ**: REALLOCATE_BITMAPSETS を有効にしてビルドすると、メモリ解放後の不正アクセスを検出しやすくなります。開発環境でのデバッグに非常に有用です。
