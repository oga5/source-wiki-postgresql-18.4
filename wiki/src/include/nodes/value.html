src/include/nodes/value.h

概要

value.h は、PostgreSQLのパーサーが字句解析時や構文解析時に使用する「定数リテラル」を表現するためのノード構造体を定義するヘッダーファイルです。

モジュール内での役割

nodes サブシステムの一部として、構文解析ツリー（Parse Tree）内で使用される基本的なデータコンテナを提供します。これらは主に、SQLクエリ内のリテラル値を List 構造に格納して受け渡すために利用されます。

責務

  • 整数、浮動小数点数、真偽値、文字列、ビット文字列を表現するノード構造体の定義。
  • 各ノード型から実際の値を取り出すためのアクセスマクロの提供。
  • ノード生成用コンストラクタ関数の宣言。

重要なシンボル

  • **構造体 (Integer, Float, String 等)**: 各リテラル型を保持するノード。NodeTag を含み、PostgreSQLのノードシステムと統合されています。
  • **アクセスマクロ (intVal, floatVal, strVal 等)**: ノードポインタから内部の値を安全に取り出すためのラッパー。castNode を介して型チェックが行われます。
  • **コンストラクタ (makeInteger, makeString 等)**: ノードをメモリ上に割り当て、値を初期化して返す関数群。

追加の索引済みシンボル

  • BitString
  • Boolean

制御フローとデータフロー

1. **生成**: パーサーが字句解析でリテラルを検出すると、makeXXX 関数を呼び出し、対応するノード構造体をヒープ上に作成します。 2. **保持**: 作成されたノードは List 構造などに格納され、構文解析の各フェーズで受け渡されます。 3. **抽出**: 意味解析や計画作成の段階で、intVal() などのマクロを使用してノードから実際の値（int や char*）を取り出し、実行可能な形式へ変換します。

直接依存

  • nodes/nodes.h: NodeTag や castNode マクロなど、PostgreSQLのノードシステムの基盤定義を提供します。

関連コード

  • src/backend/nodes/value.c: 本ヘッダーで宣言された makeXXX 関数の実装が含まれています。
  • src/include/nodes/pg_list.h: これらの値ノードが格納される List 構造の定義。

読者向けメモ

  • **浮動小数点の扱い**: Float ノードは数値を double ではなく char* として保持します。これは、解析段階で精度を失わないようにするためであり、必要に応じて後続の処理で NUMERIC 型等へ変換されます。
  • **歴史的経緯**: かつてはこれら全てを包含する Value という単一の構造体が存在していましたが、現在は個別の構造体に分離されています。
  • **メモリ管理**: makeString や makeFloat に渡す文字列は、呼び出し側で palloc されたものである必要があります。
