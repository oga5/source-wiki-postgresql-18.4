src/include/nodes/parsenodes.h

概要

PostgreSQLのパーサーが生成する「パースツリー」および「クエリツリー」を構成するデータ構造を定義するヘッダーファイルです。SQL文を解析した結果として生成されるノードの型定義が網羅されています。

モジュール内での役割

src/include/nodes モジュールにおいて、SQL構文解析の出力結果を保持する中心的な役割を果たします。パーサー（gram.y）が生成した生のパースツリーから、解析器（analyze.c）が生成する意味的に整理されたクエリツリーまで、SQL処理の初期段階で必要となるすべてのノード構造を定義しています。

責務

  • SQL文の各要素（SELECT, INSERT, UPDATE, DELETE, DDL等）を表現する構造体の定義。
  • パーサーが生成する「生のパースツリー」と、解析後の「クエリツリー」の定義。
  • エラー位置特定のための location フィールドの管理。
  • 権限チェックやクエリの同一性判定（Jumbling）に必要なメタデータの保持。

重要なシンボル

  • Query: 解析後のクエリツリーのルートノード。
  • RawStmt: パーサーが生成する個々の文のコンテナ。
  • RangeTblEntry (RTE): FROM句に現れるテーブルやサブクエリ等の参照先情報。
  • A_Expr, FuncCall: 式や関数呼び出しを表現するノード。
  • AclMode: 権限ビットマスク。

制御フローとデータフロー

1. **パース**: gram.y がSQLを読み込み、RawStmt を含むパースツリーを生成します。 2. **解析**: parse_analyze() が RawStmt を受け取り、意味解析を行って Query 構造体に変換します。 3. **変換**: Query 構造体は、リライターやプランナーによって処理され、最終的に PlannedStmt（plannodes.h で定義）へと変換されます。 4. **実行**: エグゼキューターは Query 構造体ではなく、プランナーが生成したプランツリーを使用します。

直接依存

  • common/relpath.h: ファイルパス関連の定義。
  • nodes/bitmapset.h: 権限管理等で使用するビットマップセット。
  • nodes/lockoptions.h: ロックモードの定義。
  • nodes/primnodes.h: 式や定数などのプリミティブなノード定義。
  • nodes/value.h: 定数値を保持するノード。
  • partitioning/partdefs.h: パーティショニング関連の定義。

関連コード

  • src/backend/parser/gram.y: 本ファイルで定義された構造体を生成するパーサー。
  • src/backend/parser/analyze.c: パースツリーをクエリツリーに変換する解析器。
  • src/include/nodes/plannodes.h: クエリツリーから変換された実行計画を保持するノード群。
  • src/include/nodes/primnodes.h: 式ツリー（Expr）の詳細定義。

読者向けメモ

  • 本ファイルは非常に巨大です。特定の文（例: CREATE TABLE）の構造を知りたい場合は、CreateStmt 構造体から検索を開始してください。
  • pg_node_attr マクロは、PostgreSQLのノード自動生成ツール（gen_node_support.pl）が使用する属性指定です。query_jumble_ignore などは、クエリの同一性判定時に無視されるフィールドを示します。
  • location フィールドは、SQL文字列内のバイトオフセットを指します。エラーメッセージのカーソル位置表示に不可欠です。
