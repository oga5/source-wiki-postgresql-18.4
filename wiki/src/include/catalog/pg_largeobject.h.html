src/include/catalog/pg_largeobject.h

概要

pg_largeobject.h は、PostgreSQL のラージオブジェクト（Large Object: LO）機能において、実際のデータ本体を格納するシステムカタログ pg_largeobject のスキーマ定義を保持するヘッダーファイルです。

モジュール内での役割

src/include/catalog モジュールの一部として、ラージオブジェクトの物理的なデータ構造を定義します。このカタログは、ラージオブジェクトの各ページを個別の行として保持する「チャンク化」されたストレージモデルを提供します。

責務

  • pg_largeobject システムカタログのテーブル構造（OID、ページ番号、データ本体）の定義。
  • ラージオブジェクトのデータアクセスに必要なインデックス（pg_largeobject_loid_pn_index）の宣言。
  • ラージオブジェクトのライフサイクル管理（作成・削除・存在確認）を行う関数のインターフェース提供。

重要なシンボル

  • FormData_pg_largeobject: カタログの行構造を定義する構造体。loid（オブジェクトID）と pageno（ページ番号）の組み合わせで一意性が保証されます。
  • Form_pg_largeobject: FormData_pg_largeobject へのポインタ型。
  • LargeObjectCreate / LargeObjectDrop: ラージオブジェクトのメタデータおよびデータページを操作するための主要な API。

制御フローとデータフロー

  • **データ格納**: ラージオブジェクトは pg_largeobject_metadata（メタデータ）と pg_largeobject（実データ）の2つのカタログに分離されています。
  • **データフロー**: LargeObjectCreate が呼ばれると、まずメタデータが作成され、データ書き込み時に pg_largeobject へページ単位でデータが挿入されます。削除時は、LargeObjectDrop がメタデータと関連するすべてのデータページを pg_largeobject から一括削除します。

直接依存

  • catalog/genbki.h: BKI（Backend Interface）生成用マクロを提供。
  • catalog/pg_largeobject_d.h: 定数やOID定義などの派生情報。
  • utils/snapshot.h: スナップショットを用いた存在確認に使用。

関連コード

  • src/backend/catalog/pg_largeobject.c: 本ヘッダーで宣言された関数の実装。
  • src/include/catalog/pg_largeobject_metadata.h: ラージオブジェクトの所有者やACLを管理するメタデータカタログの定義。
  • src/backend/storage/large_object/inv_api.c: ラージオブジェクトの読み書きを行う上位レイヤーの API 実装。

読者向けメモ

  • **チャンク構造**: ラージオブジェクトは巨大なデータを一度に保持せず、pageno を用いてページ単位で分割管理されています。このため、ランダムアクセスや部分的な更新が効率的に行えます。
  • **キャッシュの回避**: pg_largeobject は非常に巨大になる可能性があるため、システムカタログキャッシュ（syscache）には載せられません。実装を読む際は、キャッシュを利用しない直接的なテーブルスキャンが行われている点に注意してください。
  • **BKI定義**: このファイルは Catalog.pm によって解析されるため、構造体の定義変更には注意が必要です。
