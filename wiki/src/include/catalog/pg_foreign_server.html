src/include/catalog/pg_foreign_server.h

概要

pg_foreign_server.h は、PostgreSQL の外部サーバー（Foreign Server）情報を管理するシステムカタログ pg_foreign_server のスキーマ定義ファイルです。外部データラッパー（FDW）を介して接続するリモートサーバーのメタデータを保持します。

モジュール内での役割

src/include/catalog ディレクトリの一部として、PostgreSQL のカタログ定義メカニズム（genbki.h）に基づき、サーバーの構成情報を定義します。このファイルは、サーバーの作成、検索、および権限管理の基盤となります。

責務

  • pg_foreign_server カタログの物理的な行構造（FormData_pg_foreign_server）の定義。
  • カタログの OID、一意制約（名前および OID）、および TOAST テーブルの宣言。
  • システムキャッシュ（SysCache）の定義による、外部サーバー情報の高速なルックアップ機能の提供。

重要なシンボル

  • FormData_pg_foreign_server: カタログの各行を表す構造体。サーバー名、所有者、FDW の OID、型、バージョン、ACL、オプションを保持します。
  • Form_pg_foreign_server: FormData_pg_foreign_server へのポインタ型。カタログ操作時に頻繁に使用されます。
  • FOREIGNSERVEROID / FOREIGNSERVERNAME: SysCache で使用される識別子。それぞれ OID とサーバー名による高速検索を可能にします。

制御フローとデータフロー

本ファイルは宣言のみを行うヘッダーファイルであり、直接的な実行フローは持ちません。

  • **データフロー**: Catalog.pm スクリプトがこのファイルを読み取り、データベース初期化時（initdb）のカタログスキーマを生成します。
  • **実行時**: バックエンドプロセスが SearchSysCache 等を通じてこの定義を参照し、外部サーバーの接続情報を取得します。

直接依存

  • catalog/genbki.h: カタログ定義用のマクロを提供。
  • catalog/pg_foreign_server_d.h: 定数（OID 等）の定義を分離したヘッダー。

関連コード

  • src/include/catalog/pg_foreign_data_wrapper.h: 外部サーバーが依存する FDW の定義。
  • src/include/catalog/pg_user_mapping.h: 外部サーバーに対するユーザーごとの認証情報を管理するカタログ。
  • src/backend/catalog/pg_foreign_server.c: （存在する場合）カタログ操作の実装ロジック。

読者向けメモ

  • 本ファイル内の BKI_LOOKUP マクロは、pg_authid や pg_foreign_data_wrapper との外部キー関係を宣言しています。
  • CATALOG_VARLEN ブロック内のフィールドは可変長データであり、TOAST テーブル（DECLARE_TOAST）によって管理されます。
  • 外部サーバーの定義を変更する際は、pg_foreign_server_d.h にある OID 定数との整合性に注意してください。
