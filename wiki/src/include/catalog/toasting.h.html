src/include/catalog/toasting.h

概要

toasting.h は、PostgreSQL における TOAST（The Oversized-Attribute Storage Technique）テーブルの作成と管理を支援するためのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

src/include/catalog ディレクトリの一部として、テーブルの物理的な格納構造を拡張する TOAST メカニズムのカタログ操作層を提供します。主に src/backend/catalog/toasting.c で実装される関数の宣言を保持しています。

責務

  • 新規テーブル作成時や ALTER TABLE 実行時に、必要に応じて TOAST テーブルを自動生成する関数の公開。
  • データベースのブートストラップ処理中に、特定の OID を持つ TOAST テーブルを生成するためのインターフェースの提供。

重要なシンボル

  • NewRelationCreateToastTable: 新規リレーション作成時に TOAST テーブルが必要か判断し、作成する。
  • AlterTableCreateToastTable: 既存テーブルに対して明示的に TOAST テーブルを作成する。
  • BootstrapToastTable: システムカタログの初期化（ブートストラップ）時に、固定 OID を指定して TOAST テーブルを作成する。

制御フローとデータフロー

本ファイルは宣言のみを提供します。実際のフローは toasting.c 内の CheckAndCreateToastTable を経由して以下の順序で実行されます： 1. 対象リレーションのロック取得。 2. needs_toast_table() による TOAST 作成の必要性判定。 3. create_toast_table() によるカタログ更新および物理テーブルの生成。 4. CommandCounterIncrement による変更の確定。

直接依存

  • storage/lock.h: ロックモード（LOCKMODE）の定義を利用するためにインクルードされています。

関連コード

  • src/backend/catalog/toasting.c: 本ヘッダーで宣言された関数の実装本体。
  • src/include/catalog/heap.h: テーブル作成全般を司る関連モジュール。
  • src/include/access/toast_compression.h: TOAST データの圧縮設定に関連する定義。

読者向けメモ

  • 本ファイルは TOAST テーブルの「作成」に特化しています。TOAST データの読み書きや圧縮アルゴリズムの詳細は src/backend/access/toast 以下のモジュールを参照してください。
  • BootstrapToastTable は特殊な用途（初期化時）専用であり、通常の SQL 操作からは呼び出されません。
