src/include/catalog/pg_index.h

概要

pg_index.h は、PostgreSQL のシステムカタログ pg_index のスキーマ定義を保持するヘッダーファイルです。このカタログは、データベース内のすべてのインデックスに関するメタデータ（インデックスの対象テーブル、列、制約属性、有効性など）を管理します。

モジュール内での役割

src/include/catalog モジュールの一部として、PostgreSQL のブートストラッププロセスおよびカタログ定義の基盤を担います。Catalog.pm スクリプトによって読み取られ、データベース初期化時に必要なシステムテーブルの構造を生成するために使用されます。

責務

  • pg_index システムカタログの物理的な行構造（FormData_pg_index）の定義。
  • インデックスの特性（一意性、プライマリキー、有効状態、レプリケーション識別子など）を保持するフィールドの定義。
  • インデックスの構成要素（列番号、照合順序、演算子クラス、式、述語）の管理。
  • システムカタログとしてのインデックス（pg_index_indrelid_index 等）の宣言。

重要なシンボル

  • FormData_pg_index: pg_index テーブルの各行の構造体。
  • Form_pg_index: FormData_pg_index へのポインタ型。
  • INDOPTION_DESC / INDOPTION_NULLS_FIRST: インデックスのソート順序を制御するためのビットフラグ。
  • CATALOG(pg_index, ...): genbki.h を通じてテーブル定義を生成するためのマクロ。

制御フローとデータフロー

このファイル自体は実行コードを含まず、データ構造の定義に特化しています。 1. **ビルド時**: Catalog.pm がこのファイルを解析し、pg_index テーブルの初期化用 SQL や C 構造体を生成します。 2. **実行時**: インデックスの作成や変更時に、Form_pg_index 構造体を通じてカタログデータが読み書きされます。特に indkey や indclass などの配列フィールドは、インデックスの検索やプランナの最適化において頻繁に参照されます。

直接依存

  • catalog/genbki.h: カタログ定義用マクロの提供。
  • catalog/pg_index_d.h: 定数や OID 定義など、自動生成された補助定義。

関連コード

  • src/include/catalog/index.h: インデックス操作の関数定義や定数。
  • src/backend/catalog/index.c: pg_index を操作するバックエンド側の実装。
  • src/include/catalog/pg_class.h: インデックス対象となるテーブル（pg_class）の定義。

読者向けメモ

  • **構造の解釈**: indkey が 0（InvalidAttrNumber）の場合、その列は単純な列参照ではなく、indexprs に格納された式であることを示します。
  • **可変長フィールド**: CATALOG_VARLEN ブロック内のフィールドは、行の末尾に配置される可変長データです。これらは heap_form_tuple 等で適切に処理される必要があります。
  • **拡張性**: indoption はインデックスアクセスメソッド（AM）ごとに意味が異なる場合がありますが、INDOPTION_DESC と INDOPTION_NULLS_FIRST は順序付きスキャンをサポートする AM において共通の規約として扱われます。
