src/include/catalog/pg_inherits.h

概要

pg_inherits.h は、テーブルの継承関係を管理するシステムカタログ pg_inherits のスキーマ定義と、それに関連する操作関数の宣言を行うヘッダーファイルです。

モジュール内での役割

src/include/catalog モジュールの一部として、PostgreSQL の継承およびパーティショニング機能の基盤となるメタデータ構造を定義します。このファイルは、Catalog.pm によるスキーマ生成と、バックエンドの継承関連ロジックの橋渡しをします。

責務

  • pg_inherits システムカタログの物理的な行構造（FormData_pg_inherits）の定義。
  • 継承関係を検索・検証するための公開 API の提供。
  • 継承に関連するインデックス（pg_inherits_relid_seqno_index 等）の定義。

重要なシンボル

  • FormData_pg_inherits: pg_inherits カタログの各行の構造体。子テーブル（inhrelid）と親テーブル（inhparent）の OID、および継承順序（inhseqno）を保持します。
  • find_inheritance_children: 指定した親テーブルの直下の子テーブル（またはパーティション）の OID リストを返します。
  • find_all_inheritors: 再帰的にすべての子孫テーブルを検索します。
  • has_subclass: 指定した OID が何らかのテーブルの親であるかを判定する高速なチェック関数です。

追加の索引済みシンボル

  • Form_pg_inherits

制御フローとデータフロー

このファイル自体は宣言のみですが、定義されている関数は主に以下のフローで動作します： 1. has_subclass で継承関係の有無を高速判定。 2. systable_beginscan を使用して pg_inherits をスキャン。 3. inhparent インデックスを利用して、特定の親に紐づく子テーブルの OID を抽出。 4. 必要に応じて inhdetachpending フラグを確認し、デタッチ処理中のパーティションをフィルタリング。

直接依存

  • catalog/genbki.h: カタログ定義用マクロを提供。
  • catalog/pg_inherits_d.h: 自動生成された定数定義。
  • nodes/pg_list.h: 継承関係のリスト管理に使用。
  • storage/lock.h: 継承関係検索時のロックモード定義。

関連コード

  • src/backend/catalog/pg_inherits.c: 本ヘッダーで宣言された関数の実装。
  • src/backend/commands/tablecmds.c: テーブルの作成や継承関係の変更（ALTER TABLE 等）を行う際に、pg_inherits を更新するロジックが含まれています。

読者向けメモ

  • 本ファイルで定義される inhdetachpending は、パーティションのデタッチ処理が進行中であることを示す重要なフラグです。パーティション管理のロジックを追う際は、このフラグの可視性に注意してください。
  • find_inheritance_children などの関数は、呼び出し側で親テーブルのロックを保持していることを前提としている場合が多いため、利用時は注意が必要です。
