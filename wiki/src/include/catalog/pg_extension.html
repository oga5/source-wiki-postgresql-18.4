src/include/catalog/pg_extension.h

概要

PostgreSQLの拡張機能（Extension）を管理するシステムカタログ pg_extension のスキーマ定義ファイルです。

モジュール内での役割

src/include/catalog ディレクトリの一部として、PostgreSQLのメタデータ管理層に位置します。データベース内にインストールされた拡張機能の属性（所有者、名前空間、バージョン、設定テーブルなど）を定義し、システム全体で拡張機能の状態を追跡するための基盤を提供します。

責務

  • pg_extension カタログの物理的な行構造（FormData_pg_extension）の定義。
  • 拡張機能の識別子（OID）および名前に対する一意制約インデックスの宣言。
  • システムキャッシュ（SysCache）の定義による、拡張機能メタデータの高速な検索サポート。
  • Catalog.pm によるスキーマ生成のためのメタデータ提供。

重要なシンボル

  • FormData_pg_extension: pg_extension カタログの1行を表現する構造体。
  • Form_pg_extension: FormData_pg_extension へのポインタ型。
  • CATALOG(pg_extension, ...): BKI（Bootstrap Knowledge Information）ファイル生成用のマクロ。
  • MAKE_SYSCACHE(...): 拡張機能情報をメモリ上で高速検索するためのキャッシュ定義。

制御フローとデータフロー

本ファイルは宣言のみを行うヘッダーであり、直接的な実行フローは持ちません。

  • **データフロー**: Catalog.pm がこのファイルを読み取り、データベース初期化時（initdb）のスキーマ定義や、C言語レベルでのカタログアクセス用コードを生成します。
  • **実行時**: pg_extension へのアクセスが発生すると、定義された SysCache（EXTENSIONOID または EXTENSIONNAME）を通じて、カタログテーブルからメモリ上のキャッシュへデータがロードされます。

直接依存

  • catalog/genbki.h: カタログ定義用マクロの提供。
  • catalog/pg_extension_d.h: 拡張機能関連の定数（OID等）の定義。

関連コード

  • src/backend/catalog/pg_extension.c: pg_extension カタログを操作する実処理（拡張機能の作成・削除など）が記述されています。
  • src/backend/commands/extension.c: CREATE EXTENSION や ALTER EXTENSION などのコマンド実行ロジックが含まれます。

読者向けメモ

  • 本ファイル内の BKI_LOOKUP マクロは、外部キー制約のような役割を果たし、他のカタログ（pg_authid や pg_namespace）との整合性を保証します。
  • CATALOG_VARLEN 以降のフィールドは可変長データであり、extconfig や extcondition は拡張機能が管理する設定テーブルのダンプ制御に使用されます。
  • 構造体定義を変更した場合は、genbki.pl を通じて関連するヘッダーやSQL定義が正しく更新されることを確認してください。
