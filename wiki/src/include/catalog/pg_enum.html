src/include/catalog/pg_enum.h

概要

pg_enum.h は、PostgreSQL の列挙型（ENUM）の各値を管理するシステムカタログ pg_enum のスキーマ定義と、それに関連するインデックス、および操作用関数のプロトタイプを定義するヘッダーファイルです。

モジュール内での役割

src/include/catalog モジュールの一部として、PostgreSQL のメタデータ管理層に位置します。このファイルは Catalog.pm によって読み取られ、データベース初期化時に必要なカタログ定義（BKIファイル）を生成するためのソースとして機能します。

責務

  • pg_enum カタログの物理的な行構造（FormData_pg_enum）の定義。
  • 列挙型の値の順序や所属を管理するためのインデックス（OID、型ID+ラベル、型ID+順序）の宣言。
  • システムキャッシュ（syscache）の定義による、列挙型情報の高速なルックアップの提供。
  • pg_enum.c で実装される列挙型操作関数（作成、削除、ラベル変更等）のインターフェース提供。

重要なシンボル

  • FormData_pg_enum: pg_enum カタログの行構造体。enumtypid（所属する型）、enumsortorder（ソート順）、enumlabel（ラベル名）を保持します。
  • Form_pg_enum: FormData_pg_enum へのポインタ型。カタログ操作時に頻繁に使用されます。
  • CATALOG(pg_enum, ...): BKI 生成用マクロ。カタログ名と OID を定義します。
  • MAKE_SYSCACHE(...): システムキャッシュを生成し、特定のインデックスを使用して高速検索を可能にします。

制御フローとデータフロー

本ファイル自体は宣言のみですが、定義された関数群は以下のフローで利用されます： 1. **DDL実行時**: CREATE TYPE AS ENUM や ALTER TYPE ADD VALUE が実行されると、EnumValuesCreate や AddEnumLabel が呼び出されます。 2. **カタログ更新**: これらの関数は pg_enum テーブルに対して行の挿入・更新を行い、同時にインデックスを更新します。 3. **トランザクション管理**: AtEOXact_Enum がトランザクション終了時に呼び出され、未コミットの列挙型情報のクリーンアップや整合性チェックを行います。

直接依存

  • catalog/genbki.h: カタログ定義用マクロの提供。
  • catalog/pg_enum_d.h: 列挙型に関連する定数定義（OID等）。
  • nodes/pg_list.h: 列挙値のリスト処理に使用。

関連コード

  • src/backend/catalog/pg_enum.c: 本ヘッダーで宣言された関数の実装。トランザクション中の列挙型管理（ハッシュテーブルによる追跡）の詳細が含まれます。
  • src/include/catalog/pg_type.h: 列挙型そのものの定義を保持する pg_type カタログの定義。

読者向けメモ

  • enumsortorder は float4 型です。これは、既存の列挙値の間に新しい値を挿入する際、既存の順序値を変更せずに中間の値を割り当てやすくするための設計です。
  • pg_enum の変更はトランザクションの整合性に強く依存します。特に AddEnumLabel 等の操作は、インデックスの不整合を防ぐため、トランザクションのコミット状態を考慮した制御が必要です。詳細は pg_enum.c のコメントを参照してください。
