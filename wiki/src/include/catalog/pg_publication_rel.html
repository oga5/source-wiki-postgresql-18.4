src/include/catalog/pg_publication_rel.h

概要

pg_publication_rel.h は、PostgreSQL の論理レプリケーション機能において、パブリケーション（Publication）とリレーション（Table）の関連付けを管理するシステムカタログ pg_publication_rel のスキーマ定義ファイルです。

モジュール内での役割

src/include/catalog モジュールの一部として、論理レプリケーションのメタデータ層を構成します。パブリケーションがどのテーブルを公開対象としているかという多対多の関連を保持する役割を担います。

責務

  • pg_publication_rel カタログの構造定義（列構成、型、制約）。
  • カタログに対するインデックス（OID、リレーションとパブリケーションのペア、パブリケーション単位）の宣言。
  • システムキャッシュ（SysCache）の定義による、カタログ検索の高速化サポート。
  • Catalog.pm によるスキーマ生成のためのメタデータ提供。

重要なシンボル

  • FormData_pg_publication_rel: カタログの行構造を定義する構造体。
  • Form_pg_publication_rel: FormData_pg_publication_rel へのポインタ型。
  • prpubid: パブリケーションの OID を保持する外部キー。
  • prrelid: 公開対象テーブルの OID を保持する外部キー。
  • prqual: 行フィルタリングのための式（pg_node_tree）。
  • prattrs: 列フィルタリングのための列リスト（int2vector）。

制御フローとデータフロー

本ファイルは宣言のみを行うヘッダーであり、実行時の制御フローは持ちません。

  • **データフロー**: Catalog.pm がこのファイルを読み取り、PostgreSQL のブートストラップ時に必要な SQL スキーマや C 言語の構造体を生成します。
  • **アクセス**: 実行時には、PUBLICATIONREL および PUBLICATIONRELMAP という SysCache を通じて、特定のテーブルがどのパブリケーションに含まれているか、あるいは特定のパブリケーションがどのテーブルを保持しているかを高速に検索するために利用されます。

直接依存

  • catalog/genbki.h: カタログ定義用マクロの提供。
  • catalog/pg_publication_rel_d.h: 自動生成された定数や識別子の定義。

関連コード

  • src/include/catalog/pg_publication.h: パブリケーション自体の定義。
  • src/include/catalog/pg_class.h: 公開対象となるテーブルの定義。
  • src/backend/catalog/pg_publication_rel.c: （存在する場合）このカタログを操作するバックエンド関数群。

読者向けメモ

  • 本カタログには prqual（行フィルタ）や prattrs（列フィルタ）といった可変長フィールドが含まれており、これらは CATALOG_VARLEN マクロによって保護されています。
  • BKI_LOOKUP マクロが付与されたフィールドは、他のカタログ（pg_publication, pg_class）との参照整合性を保つためのヒントとして機能します。
  • 変更を加える際は、Catalog.pm が正しく解釈できるよう、既存の定義スタイルを厳密に踏襲してください。
