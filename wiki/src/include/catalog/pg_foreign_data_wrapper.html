src/include/catalog/pg_foreign_data_wrapper.h

概要

本ファイルは、PostgreSQLのシステムカタログ pg_foreign_data_wrapper のスキーマ定義を保持するヘッダーファイルです。外部データラッパー（FDW）のメタデータを管理するための構造体定義およびカタログ定義を含みます。

モジュール内での役割

src/include/catalog モジュールの一部として、PostgreSQLのデータディクショナリを構成する役割を担います。Catalog.pm スクリプトによって読み取られ、データベース初期化時（initdb）のカタログスキーマ生成に使用されます。

責務

  • pg_foreign_data_wrapper テーブルの物理的な行構造（FormData_pg_foreign_data_wrapper）の定義。
  • FDWに関連するシステムカタログのインデックス定義（OIDおよび名前による検索用）。
  • システムキャッシュ（SysCache）の登録定義。
  • TOASTテーブルの宣言。

重要なシンボル

  • FormData_pg_foreign_data_wrapper: カタログの1行を表現する構造体。FDWの所有者、ハンドラ関数、バリデータ関数、ACL、オプション情報を含みます。
  • Form_pg_foreign_data_wrapper: 上記構造体へのポインタ型。
  • CATALOG(...): PostgreSQLのビルドシステムがカタログを認識するためのマクロ。
  • MAKE_SYSCACHE(...): FDW情報をメモリ上で高速に検索するためのシステムキャッシュ定義。

制御フローとデータフロー

本ファイル自体は宣言のみを行うため、実行時の制御フローは持ちません。

  • **データフロー**: Catalog.pm が本ファイルを解析し、SQLの CREATE TABLE 文や関連するカタログ操作コードを生成します。実行時には、バックエンドがこの構造体定義を用いてシステムカタログから読み出したタプルをキャストし、FDWの属性（ハンドラやバリデータ）にアクセスします。

直接依存

  • catalog/genbki.h: カタログ定義用マクロを提供。
  • catalog/pg_foreign_data_wrapper_d.h: 定数（OID等）の定義を分離したヘッダー。

関連コード

  • src/backend/catalog/pg_foreign_data_wrapper.c: カタログ操作の実装（挿入・削除・更新）。
  • src/include/catalog/pg_foreign_server.h: FDWを利用する外部サーバーの定義。
  • src/include/catalog/pg_user_mapping.h: FDWとユーザーを紐付けるマッピング定義。

読者向けメモ

  • 本ファイルで定義されている fdwhandler と fdwvalidator は、それぞれ pg_proc を参照するOIDです。これらはFDWの動作を制御する関数を指します。
  • CATALOG_VARLEN ブロック内のフィールドは可変長データであるため、直接アクセスする際は heap_getattr 等のアクセサ関数を介して取得する必要があります。
  • 構造体定義を変更した場合は、catversion.h の更新が必要になる点に注意してください。
