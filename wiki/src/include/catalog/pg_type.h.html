src/include/catalog/pg_type.h

概要

pg_type.h は、PostgreSQL のデータ型システムを管理するシステムカタログ pg_type のスキーマ定義ファイルです。データベース内のすべてのデータ型（基本型、複合型、ドメイン、列挙型、範囲型など）のメタデータを保持します。本ファイルは genbki.pl によって解析され、システム初期化用の BKI ファイルや C 言語用の構造体定義を生成する基盤となります。

モジュール内での役割

src/include/catalog モジュールにおいて、型システムの中核を担います。pg_attribute と密接に関連しており、テーブルの列定義と型定義の整合性を維持する役割を持ちます。また、パーサやプランナが型変換や演算子の解決を行う際の参照先となります。

責務

  • **カタログスキーマの定義**: FormData_pg_type 構造体を通じた pg_type テーブルの物理レイアウトの定義。
  • **型メタデータの管理**: 型のサイズ、アライメント、I/O 関数、カテゴリ、配列・ドメイン関係の保持。
  • **定数定義**: 型の分類（TYPTYPE_*）、カテゴリ（TYPCATEGORY_*）、アライメント（TYPALIGN_*）などの定数提供。
  • **型操作のインターフェース**: 型の作成、名前変更、依存関係生成のための関数宣言。

重要なシンボル

データ構造

  • FormData_pg_type: pg_type テーブルの行構造体。
  • Form_pg_type: FormData_pg_type へのポインタ型。

型分類・属性マクロ

  • TYPTYPE_*: 型の性質（ベース、複合、ドメイン、列挙、範囲など）を識別。
  • TYPCATEGORY_*: パーサが暗黙の型変換を決定するためのカテゴリ分類。
  • TYPALIGN_*: ストレージ上のアライメント要件（c, s, i, d）。
  • TYPSTORAGE_*: TOAST 戦略（p, e, x, m）。

型操作関数

  • TypeShellMake: 型のプレースホルダ（シェル）を作成する。
  • TypeCreate: 型をカタログに登録し、必要な依存関係を構築する。
  • GenerateTypeDependencies: 型に関連する依存関係（所有者、名前空間、I/O 関数など）を生成する。

制御フローとデータフロー

  • **型作成フロー**: TypeShellMake で仮の行を挿入し、TypeCreate で詳細情報を更新する。この際、pg_proc（I/O 関数）や pg_namespace（名前空間）への参照が解決されます。
  • **依存関係フロー**: GenerateTypeDependencies が呼び出されることで、型とそれに関連するオブジェクト（関数、型、クラス）間の pg_depend エントリが作成され、整合性が保証されます。
  • **データアクセス**: SysCache（TYPEOID, TYPENAMENSP）を通じて、カタログ行がメモリ上にキャッシュされ、高速な型検索が実現されます。

直接依存

  • catalog/genbki.h: カタログ定義の基盤マクロ。
  • catalog/objectaddress.h: オブジェクト識別用構造体。
  • catalog/pg_type_d.h: 自動生成された定数定義。
  • nodes/nodes.h: pg_node_tree 型の定義。

関連コード

  • src/backend/catalog/pg_type.c: 本ヘッダーで宣言された関数の実装。
  • src/backend/commands/typecmds.c: CREATE TYPE などの DDL コマンドの実装。
  • src/include/utils/lsyscache.h: キャッシュされた型情報へのアクセス関数群。

読者向けメモ

  • **メモリレイアウト**: FormData_pg_type はシステムテーブルの物理構造と一致させる必要があるため、フィールドの順序や型は厳密に管理されています。
  • **アライメント**: typalign はディスク上の配置に直結するため、変更には細心の注意が必要です。
  • **シェル型**: typisdefined が false の場合、その型は未定義のプレースホルダです。これは循環参照を解決するために使用されます。
  • **省略された詳細**: 本ファイルはカタログ定義が主目的であるため、具体的な型変換ロジックや演算子解決のアルゴリズムについては、src/backend/utils/adt/ 以下の各型実装ファイルを参照してください。
