src/include/catalog/pg_seclabel.h

概要

pg_seclabel システムカタログのスキーマ定義を保持するヘッダーファイルです。PostgreSQLにおけるセキュリティラベル（SELinuxやAppArmor等の外部セキュリティプロバイダが提供するラベル）をデータベースオブジェクトに関連付けるための構造を定義します。

モジュール内での役割

src/include/catalog ディレクトリの一部として、PostgreSQLのメタデータ管理システムにおける「セキュリティラベル」の永続化層を担います。Catalog.pm によるBKI（Backend Interface）生成の入力ソースとして機能します。

責務

  • pg_seclabel テーブルの物理構造（列定義）の定義。
  • セキュリティラベルを特定するためのユニークインデックス（pg_seclabel_object_index）の宣言。
  • TOASTテーブルの定義による、長大なラベル文字列の格納サポート。

重要なシンボル

  • FormData_pg_seclabel: pg_seclabel テーブルの1行を表すC構造体。
  • objoid, classoid, objsubid: ラベル対象のオブジェクトを一意に特定するための識別子。
  • provider: ラベルを管理するセキュリティプロバイダ名（例: "selinux"）。
  • label: 実際に適用されるセキュリティラベル文字列。

制御フローとデータフロー

本ファイルは定義のみを提供します。実際のデータ操作は、src/backend/catalog/pg_seclabel.c 等の関数を通じて行われます。 1. **挿入/更新**: objoid, classoid, objsubid の組み合わせと provider をキーとして、特定のオブジェクトに対するラベルを検索・更新します。 2. **参照**: セキュリティチェックの際、システムはこれらの値を読み取り、プロバイダに照会します。

直接依存

  • catalog/genbki.h: BKI生成用マクロの定義。
  • catalog/pg_seclabel_d.h: 定数（OID等）の定義。

関連コード

  • src/include/catalog/pg_shseclabel.h: 共有オブジェクト（データベース全体やロールなど）に対するセキュリティラベルを管理するカタログ。
  • src/backend/catalog/pg_seclabel.c: セキュリティラベルの操作ロジックを実装するソース。

読者向けメモ

  • 本ファイルで定義される CATALOG マクロは、ビルド時に Catalog.pm によって解析され、SQLの CREATE TABLE 文や関連するインデックス定義に変換されます。
  • objsubid は、テーブルの列に対するラベル付けを行う際に使用されます（0の場合はテーブル全体や関数などのオブジェクト全体を指します）。
  • セキュリティラベルはプロバイダごとに管理されるため、provider 列が複合キーの一部に含まれている点に注意してください。
