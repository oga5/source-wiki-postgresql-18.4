src/include/catalog/pg_user_mapping.h

概要

pg_user_mapping.h は、PostgreSQL のシステムカタログ pg_user_mapping のスキーマ定義を保持するヘッダーファイルです。このカタログは、ローカルのデータベースユーザーと外部サーバー（Foreign Server）との間のマッピング情報を管理します。

モジュール内での役割

src/include/catalog モジュールの一部として、外部データラッパー（FDW）機能における認証情報の紐付けを定義します。SQL の CREATE USER MAPPING コマンドによって操作されるデータの永続化構造を規定しています。

責務

  • pg_user_mapping テーブルの物理的な行構造（FormData_pg_user_mapping）の定義。
  • カタログの OID、インデックス、およびシステムキャッシュ（SysCache）の宣言。
  • Catalog.pm によるスキーマ生成のためのメタデータ提供。

重要なシンボル

  • FormData_pg_user_mapping: テーブルの行構造を定義する構造体。umuser（ユーザーID）と umserver（外部サーバーID）を保持します。
  • Form_pg_user_mapping: FormData_pg_user_mapping へのポインタ型。
  • USERMAPPINGUSERSERVER: umuser と umserver の組み合わせを高速検索するためのシステムキャッシュ識別子。

制御フローとデータフロー

このファイル自体はデータ構造の定義のみを行うため、実行時の制御フローは持ちません。ただし、MAKE_SYSCACHE マクロにより、バックエンドが pg_user_mapping を参照する際に、インデックスを介した高速なキャッシュアクセスが可能になるよう設計されています。

直接依存

  • catalog/genbki.h: BKI（Backend Interface）ファイル生成のためのマクロ定義。
  • catalog/pg_user_mapping_d.h: 定数や OID 定義など、自動生成された補助情報。

関連コード

  • src/backend/catalog/pg_user_mapping.c: このカタログを操作する関数群の実装。
  • src/include/catalog/pg_foreign_server.h: マッピング先となる外部サーバーの定義。
  • src/include/catalog/pg_authid.h: マッピング元となるユーザー情報の定義。

読者向けメモ

  • umuser が InvalidOid の場合、そのマッピングは PUBLIC（全ユーザー対象）として扱われます。
  • umoptions は可変長フィールド（text）であり、CATALOG_VARLEN マクロによって TOAST テーブルの対象となることが示されています。
  • 本ファイル内の CATALOG(...) 宣言は、PostgreSQL のビルドプロセス中に genbki.pl によって解析され、データベース初期化時のカタログ定義に変換されます。直接的な構造体操作よりも、カタログ定義の宣言としての役割が主です。
