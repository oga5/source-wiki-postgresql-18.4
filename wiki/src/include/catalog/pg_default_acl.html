src/include/catalog/pg_default_acl.h

概要

pg_default_acl.h は、PostgreSQL における「デフォルト権限（Default ACL）」を管理するシステムカタログ pg_default_acl のスキーマ定義ファイルです。ユーザーが新しいオブジェクトを作成する際に自動的に適用される権限設定を保持します。

モジュール内での役割

src/include/catalog ディレクトリの一部として、PostgreSQL のメタデータ管理層に位置します。ALTER DEFAULT PRIVILEGES コマンドによって操作される情報の永続化先を定義しており、権限チェックのプロセスにおいて重要な役割を果たします。

責務

  • pg_default_acl カタログの物理構造（テーブル定義）の提供。
  • オブジェクト種別（テーブル、関数、型など）を識別する定数の定義。
  • システムカタログとしてのインデックス定義およびキャッシュ（SysCache）の登録。

重要なシンボル

  • FormData_pg_default_acl: カタログの行構造を定義する構造体。
  • defaclobjtype: デフォルト権限が適用される対象オブジェクトの種類を示す文字コード。
  • DEFACLOBJ_* マクロ: defaclobjtype に格納される値（例: DEFACLOBJ_RELATION はテーブル/ビュー）。
  • DEFACLROLENSPOBJ: pg_default_acl を効率的に検索するためのシステムキャッシュ識別子。

制御フローとデータフロー

本ファイルは宣言のみを行うヘッダーファイルです。 1. **定義**: CATALOG(...) マクロにより、PostgreSQL のビルドプロセス（Catalog.pm）がこの定義を読み取り、実際のテーブル作成用 SQL を生成します。 2. **検索**: 実行時に SearchSysCache1 等の関数が DEFACLROLENSPOBJ キャッシュを使用して、特定のロールや名前空間に対するデフォルト権限を高速に取得します。 3. **適用**: 新規オブジェクト作成時、システムはこのカタログを参照し、defaclacl に格納された権限を新しいオブジェクトの ACL にマージします。

直接依存

  • catalog/genbki.h: カタログ定義用マクロの提供。
  • catalog/pg_default_acl_d.h: 関連する定数や ID 定義の分離ファイル。

関連コード

  • src/backend/catalog/aclchk.c: このカタログに格納された情報を実際に読み込み、権限チェックを行うロジックが含まれます。
  • src/backend/commands/user.c: ALTER DEFAULT PRIVILEGES コマンドの実行時にこのカタログを更新する処理が含まれます。

読者向けメモ

  • BKI_LOOKUP 属性が付与されたフィールドは、外部キー制約のように他のシステムカタログ（pg_authid や pg_namespace）を参照していることを示します。
  • defaclnamespace が 0 の場合、そのロールのすべての名前空間に適用されるデフォルト権限であることを意味します。
  • このファイルは Catalog.pm によって解析されるため、構造体のフィールド順序やマクロの記述形式には厳密なルールがあります。変更時は既存のパターンに従ってください。
