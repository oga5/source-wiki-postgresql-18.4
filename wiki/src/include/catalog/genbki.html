src/include/catalog/genbki.h

概要

genbki.h は、PostgreSQL のシステムカタログ定義を記述するためのメタデータ用ヘッダーファイルです。このファイルで定義されるマクロ群は、C コンパイラに対しては無害なコード（主に extern 宣言）として振る舞いますが、ビルド時に genbki.pl スクリプトによって解析され、カタログの初期化データ（BKIファイル）やヘッダーファイルを生成するための指示子として機能します。

モジュール内での役割

src/include/catalog 配下の各カタログ定義ヘッダー（例: pg_class.h）が必ずインクルードする基盤ファイルです。カタログの構造定義、インデックス、外部キー関係、TOAST テーブルの生成ルールを宣言的に記述するための共通言語を提供します。

責務

  • **カタログ構造の定義**: CATALOG() マクロによるテーブル構造の定義。
  • **メタデータの付与**: BKI_LOOKUP や BKI_DEFAULT などの属性による、カタログ列の制約や参照関係の明示。
  • **ビルド時生成の制御**: インデックス作成、TOAST テーブル定義、外部キー関係のメタデータ化を genbki.pl に伝える。
  • **C コンパイラとの互換性**: マクロを空定義や無害な宣言に置き換え、通常の C コンパイルを阻害しないようにする。

重要なシンボル

  • CATALOG(name, oid, oidmacro): カタログのテーブル構造を定義する起点。
  • BKI_LOOKUP(catalog): 特定の列が別のカタログの OID を参照していることを示し、ビルド時の整合性チェックや名前解決に利用される。
  • DECLARE_UNIQUE_INDEX(...): カタログに対するインデックス作成を指示する。
  • DECLARE_FOREIGN_KEY(...): カタログ間の論理的な外部キー関係を定義する。
  • EXPOSE_TO_CLIENT_CODE: クライアントライブラリ（libpq等）にも公開すべき定義を囲むための条件分岐用マクロ。

制御フローとデータフロー

1. **定義**: 開発者が pg_*.h ファイル内で CATALOG や DECLARE_INDEX マクロを使用してカタログを記述する。 2. **解析**: ビルドプロセスにおいて genbki.pl がこれらのヘッダーファイルを読み込み、マクロ引数を抽出する。 3. **生成**: 抽出された情報に基づき、以下のファイルが生成される。

  • postgres.bki: バックエンド起動時にカタログを初期化するためのスクリプト。
  • *_d.h: C コードからカタログの OID や構造を参照するためのヘッダーファイル。

    • 4. **コンパイル**: 生成された *_d.h を含むソースコードがコンパイルされ、実際のバイナリに組み込まれる。

直接依存

このファイル自体は他のヘッダーに依存していません。純粋なマクロ定義の集合体です。

関連コード

  • src/backend/catalog/genbki.pl: 本ヘッダーで定義されたマクロを解釈し、BKI ファイルや派生ヘッダーを生成する Perl スクリプト。
  • src/backend/bootstrap/bootstrap.c: genbki.pl が生成した BKI ファイルを読み込み、初期データベースを作成するバックエンドのブートストラップ処理。

読者向けメモ

  • 本ファイル内のマクロは、C 言語の文法としては extern int no_such_variable 等に展開されるため、実行時のロジックには一切関与しません。
  • BKI_ で始まるマクロは、主に genbki.pl がカタログの整合性を検証したり、pg_attribute などのシステムカタログを自動生成したりするために使用されます。
  • カタログ定義を変更する際は、必ず genbki.pl が正しく解釈できる形式（本ファイルで定義されたマクロの規約）に従う必要があります。
