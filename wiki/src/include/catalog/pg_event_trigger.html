src/include/catalog/pg_event_trigger.h

概要

PostgreSQLのシステムカタログ pg_event_trigger のスキーマ定義を保持するヘッダーファイルです。イベントトリガー（DDLコマンドの実行時に起動するトリガー）のメタデータを管理します。

モジュール内での役割

src/include/catalog ディレクトリの一部として、PostgreSQLのブートストラッププロセスおよびカタログ定義システム（Catalog.pm）と密接に連携します。このファイルは、イベントトリガーの永続的な構造を定義し、システムキャッシュの生成元となります。

責務

  • pg_event_trigger テーブルの物理的な行構造（FormData_pg_event_trigger）の定義。
  • カタログのTOASTテーブル定義。
  • イベントトリガー検索用のインデックス（名前およびOID）の宣言。
  • システムキャッシュ（SysCache）の定義。

重要なシンボル

  • FormData_pg_event_trigger: カタログの各行を表現する構造体。evtname（トリガー名）、evtevent（対象イベント）、evtfoid（実行関数）などが含まれます。
  • Form_pg_event_trigger: FormData_pg_event_trigger へのポインタ型。
  • CATALOG(pg_event_trigger, ...): genbki.h を通じて、PostgreSQLのブートストラップ時にテーブルを作成するためのマクロ。
  • MAKE_SYSCACHE: イベントトリガー情報をメモリ上で高速検索するためのシステムキャッシュを定義します。

制御フローとデータフロー

本ファイルは宣言のみを行うため、実行時の制御フローは持ちません。

  • **データフロー**: Catalog.pm がこのファイルを読み取り、SQLの CREATE TABLE 文やカタログ初期化用のBKIファイルを生成します。
  • **実行時**: バックエンドプロセスがイベントトリガーを検索する際、MAKE_SYSCACHE で定義されたキャッシュを通じて pg_event_trigger テーブルのデータにアクセスします。

直接依存

  • catalog/genbki.h: カタログ定義用マクロの提供。
  • catalog/pg_event_trigger_d.h: 定数（OID等）の定義。

関連コード

  • src/backend/commands/event_trigger.c: イベントトリガーの実行ロジックやDDLフックの実装。
  • src/include/catalog/pg_proc.h: イベントトリガーが呼び出す関数の定義。
  • src/include/catalog/pg_authid.h: トリガー所有者の参照先。

読者向けメモ

  • このファイルは直接編集するだけでなく、Catalog.pm による自動生成プロセスを意識する必要があります。
  • evttags フィールドは可変長データ（CATALOG_VARLEN）として定義されており、TOASTテーブルが関連付けられている点に注意してください。
  • BKI_LOOKUP マクロは、外部カタログ（pg_authid や pg_proc）との整合性を保つための制約として機能します。
