src/include/catalog/pg_rewrite.h

概要

pg_rewrite.h は、PostgreSQL のルールシステム（ビューや条件付き更新を実現する仕組み）を管理するシステムカタログ pg_rewrite のスキーマ定義ファイルです。

モジュール内での役割

src/include/catalog ディレクトリの一部として、PostgreSQL のメタデータ管理層に位置します。このファイルは、ルール定義を永続化するためのテーブル構造を定義し、システム起動時やカタログ操作時に参照されます。

責務

  • pg_rewrite システムカタログの物理的な行構造（FormData_pg_rewrite）の定義。
  • ルール定義の識別子（OID）およびテーブルとルール名の組み合わせに対する一意制約の定義。
  • システムキャッシュ（RULERELNAME）の定義による、ルール検索の高速化支援。
  • Catalog.pm によるスキーマ生成のためのメタデータ提供。

重要なシンボル

  • FormData_pg_rewrite: pg_rewrite テーブルの各行を表現する構造体。
  • Form_pg_rewrite: FormData_pg_rewrite へのポインタ型。
  • CATALOG(pg_rewrite, ...): BKI（Backend Interface）ファイル生成用のマクロ。テーブル名とOIDを定義します。
  • ev_class: ルールが適用される対象テーブル（pg_class）のOID。
  • ev_qual / ev_action: ルールの条件式と実行内容を格納する pg_node_tree（シリアライズされたノードツリー）。

制御フローとデータフロー

本ファイルは定義のみを行うヘッダーであり、実行時の制御フローは持ちません。データフローとしては、SQLで CREATE RULE が実行される際、この構造体に基づいてタプルが構築され、pg_rewrite テーブルに挿入されます。その後、クエリ書き換えエンジンがこのカタログを読み取り、ev_qual や ev_action を復元してクエリツリーを変換します。

直接依存

  • catalog/genbki.h: BKIマクロ定義を提供。
  • catalog/pg_rewrite_d.h: 定数（OID等）の定義を提供。

関連コード

  • src/backend/rewrite/: ルール書き換えエンジンの実装。本カタログのデータを実際に利用します。
  • src/include/catalog/pg_class.h: ルールが紐付く対象テーブルの定義。
  • src/backend/catalog/: カタログ操作（挿入・更新）を行う関数群。

読者向けメモ

  • ルール名はテーブル単位で一意であればよいため、pg_rewrite_rel_rulename_index が複合キー（ev_class, rulename）で定義されています。
  • ev_qual と ev_action は可変長フィールドであり、CATALOG_VARLEN マクロによって適切に処理されます。
  • このファイルの内容を変更した場合は、make を実行してカタログ定義の整合性を再確認する必要があります。
