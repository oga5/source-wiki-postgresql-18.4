src/include/catalog/pg_parameter_acl.h

概要

pg_parameter_acl.h は、PostgreSQL の設定パラメータに対するアクセス制御リスト（ACL）を保持するシステムカタログ pg_parameter_acl のスキーマ定義ファイルです。

モジュール内での役割

src/include/catalog ディレクトリの一部として、システムカタログの構造を定義します。このファイルは Catalog.pm によって読み取られ、データベース初期化時に必要なメタデータや C 言語用の構造体定義を生成するために使用されます。

責務

  • pg_parameter_acl システムカタログのテーブル構造（列定義）の宣言。
  • カタログに対する一意制約インデックス（OID およびパラメータ名）の定義。
  • システムキャッシュ（SysCache）の登録。
  • パラメータ ACL を操作するための公開 API の宣言。

重要なシンボル

  • CATALOG(pg_parameter_acl, ...): テーブルの物理構造を定義するマクロ。
  • FormData_pg_parameter_acl: カタログの各行を表現する構造体。
  • Form_pg_parameter_acl: FormData_pg_parameter_acl へのポインタ型。
  • ParameterAclLookup(): パラメータ名から対応する OID を取得する関数。
  • ParameterAclCreate(): 新しいパラメータ ACL エントリをカタログに挿入する関数。

制御フローとデータフロー

本ファイルは定義体であるため、実行時のフローは持ちません。ただし、定義された MAKE_SYSCACHE マクロにより、バックエンドプロセスは PARAMETERACLNAME をキーとしてシステムキャッシュから高速に ACL 情報へアクセス可能です。データ操作は src/backend/catalog/pg_parameter_acl.c 内の関数を通じて行われます。

直接依存

  • catalog/genbki.h: BKI（Backend Interface）生成用マクロの定義。
  • catalog/pg_parameter_acl_d.h: 定数（OID 等）の分離定義。

関連コード

  • src/backend/catalog/pg_parameter_acl.c: 本ヘッダーで宣言された関数の実装。
  • src/include/catalog/pg_parameter_acl_d.h: 本カタログに関連する OID 定数群。

読者向けメモ

  • 本カタログは BKI_SHARED_RELATION として定義されており、データベース間で共有されるカタログです。
  • parname 列は text 型であり、GUC（Grand Unified Configuration）パラメータ名が格納されます。
  • paracl 列は aclitem 配列であり、可変長フィールドとして扱われます。
  • 新規追加やスキーマ変更を行う際は、Catalog.pm が正しく認識できるよう、このファイルの記述ルール（特にマクロの使用方法）に従う必要があります。
