src/include/lib/dshash.h

概要

dshash.h は、動的共有メモリ（Dynamic Shared Memory: DSM）領域上に構築される、プロセス間で共有可能な並行ハッシュテーブルの実装を提供します。複数のバックエンドプロセスが同時にアクセスすることを前提としており、ロックパーティション分割による高い並行性を実現しています。

モジュール内での役割

src/include/lib モジュール群の中で、共有メモリを基盤としたデータ構造を提供します。dsa.h（Dynamic Shared Area）をバックエンドとして利用し、プロセス間でメモリ領域を共有・再配置する仕組みを抽象化しています。

責務

  • **共有ハッシュテーブルのライフサイクル管理**: 作成、アタッチ、デタッチ、破棄のインターフェース提供。
  • **並行アクセス制御**: パーティション単位のロック管理による、競合の最小化。
  • **動的拡張**: ハッシュテーブルのサイズが不足した際の自動的なバケット分割。
  • **汎用的な操作API**: キーによる検索、挿入、削除、およびシーケンシャルスキャン機能の提供。

重要なシンボル

  • dshash_table: ハッシュテーブルを表現する不透明な構造体。
  • dshash_table_handle: 他プロセスと共有するための識別子（dsa_pointer）。
  • dshash_parameters: ハッシュ関数や比較関数など、テーブルの挙動を定義する設定構造体。
  • dshash_find_or_insert: 検索と挿入をアトミックに行う主要な操作関数。
  • dshash_seq_status: シーケンシャルスキャン時の状態保持用構造体。

制御フローとデータフロー

1. **初期化**: dshash_create で共有メモリ上にテーブルを確保し、dshash_table_handle を取得します。他プロセスは dshash_attach でこのハンドルを使用してアクセスを開始します。 2. **アクセス**: 各操作（find, insert, delete）はキーからハッシュ値を計算し、対応するパーティションの LWLock を取得します。 3. **拡張**: テーブルの負荷が高まると、全パーティションのロックを取得してバケットの再配置（リサイズ）が行われます。 4. **終了**: dshash_detach でプロセスごとのリソースを解放し、最終的に dshash_destroy で共有メモリ上の領域を解放します。

直接依存

  • utils/dsa.h: 共有メモリ領域の管理とポインタ操作の基盤。

関連コード

  • src/backend/lib/dshash.c: 本ヘッダーで宣言された関数の実装。内部構造体（dshash_table_control 等）の詳細が定義されています。
  • src/include/storage/lwlock.h: 並行制御に使用される軽量ロックの定義。

読者向けメモ

  • **関数ポインタの注意**: dshash_parameters で渡す関数ポインタは、プロセス間で共有できないため、各プロセスが dshash_attach する際に再指定する必要があります。
  • **ロックの粒度**: 通常の操作はパーティション単位のロックで完結しますが、リサイズ時には全パーティションのロックが必要となるため、頻繁なリサイズはパフォーマンスに影響します。
  • **不透明型**: dshash_table や dshash_seq_status は公開されていますが、内部メンバへの直接アクセスは避け、提供されている API を通じて操作してください。
