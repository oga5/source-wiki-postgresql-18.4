src/include/lib/radixtree.h

概要

radixtree.h は、PostgreSQL のメモリ内データベース機能向けに設計された「適応型基数木（Adaptive Radix Tree: ART）」を生成するためのヘッダー専用テンプレートです。マクロによるメタプログラミングを用いて、特定のデータ型やメモリ環境（ローカルメモリまたは共有メモリ）に最適化された基数木の実装を生成します。

モジュール内での役割

本ファイルは src/include/lib モジュールにおいて、高度なインデックス構造を提供します。ハッシュテーブルと比較して、キーの順序保持、範囲クエリの効率性、およびメモリ使用量の漸進的な増加という利点を持ちます。

責務

  • **適応型ノード管理**: 子ノード数に応じてノード構造（Node4, 16, 48, 256）を動的に切り替え、メモリ効率を最大化する。
  • **メモリ環境の抽象化**: RT_SHMEM マクロの有無により、ローカルメモリ（MemoryContext）と共有メモリ（DSA）の両方に対応する。
  • **型安全な生成**: マクロ定義により、特定の RT_VALUE_TYPE に特化した型安全な API を提供する。
  • **並行制御**: 共有メモリ環境下での LWLock を用いたリーダー・ライターロックの提供。

重要なシンボル

コンテナ管理

  • RT_RADIX_TREE: 基数木のハンドル構造体。
  • RT_CREATE / RT_FREE: 木の生成と破棄。
  • RT_MEMORY_USAGE: メモリ消費量の統計取得。

データ操作

  • RT_FIND: キーによる値の検索。
  • RT_SET: キーと値の挿入または更新。
  • RT_DELETE: キーの削除（RT_USE_DELETE 定義時）。

イテレータ

  • RT_BEGIN_ITERATE / RT_ITERATE_NEXT / RT_END_ITERATE: キー順序に従った全要素の走査。

共有メモリ用 API

  • RT_ATTACH / RT_DETACH: 既存の共有メモリ上の木への接続。
  • RT_LOCK_EXCLUSIVE / RT_LOCK_SHARE / RT_UNLOCK: 並行制御用ロック。

制御フローとデータフロー

  • **挿入フロー**: RT_SET がキーのチャンクを抽出し、RT_GET_SLOT_RECURSIVE を通じて適切なノードを探索します。ノードが満杯の場合、RT_GROW_NODE_* 関数が呼び出され、より大きなノード種別へ昇格します。
  • **適応型検索**: RT_NODE_SEARCH はノードの kind に応じて、線形探索（Node4/16）、SIMD 比較（Node16）、ビットマップ索引（Node48/256）を使い分けます。
  • **メモリレイアウト**: 値のサイズがポインタサイズ以下の場合、子ポインタスロットに直接埋め込まれます（タグ付きポインタ）。それ以外は独立したリーフノードとして割り当てられます。

読者向けメモ

  • **メモリ所有権**: ローカルメモリ使用時は MemoryContext が、共有メモリ使用時は DSA が所有権を持ちます。RT_FREE はこれらを適切に解放します。
  • **制約**: 現在の実装は 64bit 符号なし整数キーに限定されています。
  • **未実装機能**: 論文で言及されている「パス圧縮」や「遅延パス展開」は、本実装では「貧者のパス圧縮（キーの先頭バイトがゼロの場合の高さ調整）」という限定的な形式でのみサポートされています。
  • **パフォーマンス**: 高並行環境での性能向上のため、将来的に「楽観的ロック結合（Optimistic Lock Coupling）」への移行が検討されています。
  • **注意**: マクロによる生成のため、デバッグ時は生成された関数名（RT_PREFIX + 関数名）でシンボルを追跡してください。
