src/include/lib/stringinfo.h

概要

StringInfo は、PostgreSQL における可変長文字列バッファを管理するためのデータ型および API です。最大 1GB までの文字列やバイナリデータを効率的に保持・拡張でき、主に動的なクエリ構築やバッファリングに使用されます。

モジュール内での役割

src/include/lib 内で最も汎用的なユーティリティの一つです。メモリ管理（palloc）と密接に統合されており、文字列操作のオーバーヘッドを最小限に抑えつつ、安全なバッファ拡張を提供します。

責務

  • **動的バッファ管理**: 必要に応じたメモリの再割り当て（repalloc）による自動拡張。
  • **データ型の抽象化**: C 文字列（null 終端）とバイナリデータの両方を同一のインターフェースで扱う。
  • **パフォーマンス最適化**: appendStringInfoCharMacro 等のインライン化されたマクロによる高速な文字追加。

重要なシンボル

  • StringInfoData: バッファの実体。data（ポインタ）、len（現在長）、maxlen（容量）、cursor（走査用）を保持。
  • StringInfo: StringInfoData へのポインタ型。
  • initReadOnlyStringInfo: 既存のメモリを再利用する読み取り専用モードの初期化。
  • appendStringInfo: printf スタイルのフォーマット付き文字列追加。

追加の索引済みシンボル

  • initStringInfoFromString

制御フローとデータフロー

1. **初期化**: makeStringInfo() または initStringInfo() でメモリを確保し、StringInfoData を準備します。 2. **操作**: appendStringInfo などの関数を呼び出し、データを追加します。内部で enlargeStringInfo が呼ばれ、容量不足の場合は repalloc でバッファが拡張されます。 3. **終了**: destroyStringInfo() を呼び出し、確保したメモリを解放します。 4. **特殊ケース**: 読み取り専用モード（maxlen=0）の場合、append 操作は禁止され、既存のバッファを直接参照します。

直接依存

本ヘッダーファイル自体は他のヘッダーへの依存を持ちませんが、実装ファイル（stringinfo.c）は以下のメモリ管理機能に依存します。

  • utils/memutils.h (palloc, repalloc, pfree)

関連コード

  • src/common/stringinfo.c: 本ヘッダーで宣言された関数の実装。
  • src/include/utils/memutils.h: メモリ割り当ての基盤。

読者向けメモ

  • **読み取り専用モード**: maxlen が 0 の場合、resetStringInfo や append 系関数を呼び出すとアサーションエラーになります。既存のバッファをラップする際は注意してください。
  • **マクロの副作用**: appendStringInfoCharMacro は引数 str を複数回評価するため、副作用のある式を渡さないでください。
  • **メモリ管理**: makeStringInfo で作成したオブジェクトは destroyStringInfo で解放する必要がありますが、スタック上に確保した StringInfoData の場合は、その data フィールドのみを pfree する必要があります。
