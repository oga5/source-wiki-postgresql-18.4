src/include/lib/binaryheap.h

概要

binaryheap.h は、PostgreSQL の汎用的な二分ヒープ（Binary Heap）実装のインターフェースを定義するヘッダーファイルです。メモリ効率を考慮した可変長配列ベースのヒープ構造を提供し、最小ヒープまたは最大ヒープのいずれとしても利用可能です。

モジュール内での役割

src/include/lib モジュールの一部として、優先度付きキューやソート処理など、効率的な要素抽出が必要なアルゴリズムの基盤を提供します。

責務

  • 二分ヒープのデータ構造 binaryheap の定義。
  • ヒープの構築、要素の追加（順序維持あり/なし）、およびルート要素の取得・削除操作の宣言。
  • バックエンド（Datum）とフロントエンド（void *）の両環境に対応した型抽象化の提供。

重要なシンボル

  • binaryheap: ヒープの状態を保持する構造体。bh_nodes にフレキシブル配列メンバを使用し、メモリを連続領域として確保します。
  • bh_node_type: 環境に応じて Datum または void * に解決されるノード型。
  • binaryheap_comparator: ヒープの順序を決定するコールバック関数の型定義。
  • binaryheap_add / binaryheap_add_unordered: 前者はヒープ特性を維持しながら追加し、後者は高速な一括追加（その後の binaryheap_build が必須）を可能にします。

制御フローとデータフロー

1. **初期化**: binaryheap_allocate で指定容量のメモリを確保し、比較関数を登録します。 2. **構築**: binaryheap_add で逐次追加するか、binaryheap_add_unordered で全要素を投入後に binaryheap_build を呼び出し、O(n) でヒープを構築します。 3. **操作**: binaryheap_first でルートを参照し、binaryheap_remove_first でルートを削除してヒープを再構成します。 4. **データフロー**: bh_nodes 配列内でインデックス計算（2*i+1 等）を行い、sift_up / sift_down 関数（binaryheap.c 内）によってヒープ特性を維持します。

直接依存

本ファイルは標準的な C ヘッダーおよび PostgreSQL の基本型定義に依存していますが、明示的な #include はありません。

関連コード

  • src/common/binaryheap.c: 実装本体。sift_up や sift_down などの内部ロジックが含まれます。
  • src/include/lib/pairingheap.h: 別のヒープ実装。用途や計算量特性に応じて選択されます。

読者向けメモ

  • **ヒープ特性**: bh_has_heap_property フラグはデバッグ用であり、binaryheap_add_unordered を使用した後は必ず binaryheap_build を呼び出す必要があります。
  • **メモリ管理**: binaryheap_allocate で確保したメモリは、必ず binaryheap_free で解放してください。
  • **比較関数**: 比較関数は a < b で負、a == b で 0、a > b で正を返すように実装することで、最小/最大ヒープを制御します。
