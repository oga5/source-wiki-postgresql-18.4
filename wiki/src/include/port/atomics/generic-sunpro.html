src/include/port/atomics/generic-sunpro.h

概要

本ファイルは、Oracle Solaris Studio (SunPro) コンパイラ環境におけるアトミック操作の実装を提供します。Solaris 標準の <atomic.h> および <mbarrier.h> をラップし、PostgreSQL のアトミック操作インターフェースへ適合させるためのヘッダーファイルです。

モジュール内での役割

src/include/port/atomics サブシステムの一部として、プラットフォーム固有のプリミティブを抽象化する役割を担います。generic.h が定義する共通インターフェースに対し、SunPro コンパイラ特有の関数やマクロをマッピングします。

責務

  • **メモリバリアの提供**: <mbarrier.h> を利用し、コンパイラおよびハードウェアレベルのメモリバリア（読み取り/書き込み/フルバリア）を定義します。
  • **アトミック型の定義**: pg_atomic_uint32 および pg_atomic_uint64 構造体を定義し、Solaris のアトミック操作に適したメモリ配置を確保します。
  • **アトミック操作の実装**: atomic_cas_32/64 や atomic_swap_32/64 を利用して、比較交換（CAS）および交換操作を実装します。

重要なシンボル

  • pg_memory_barrier_impl(): フルメモリバリアを実行します。
  • pg_atomic_uint32 / pg_atomic_uint64: アトミック操作対象の値を保持する構造体。
  • pg_atomic_compare_exchange_u32_impl(): Solaris の atomic_cas_32 をラップした比較交換関数。
  • pg_atomic_exchange_u32_impl(): Solaris の atomic_swap_32 をラップした交換関数。

制御フローとデータフロー

1. **初期化**: コンパイル環境が HAVE_MBARRIER_H および HAVE_ATOMIC_H をサポートしているかを確認します。 2. **バリア定義**: mbarrier.h の関数を PostgreSQL 共通の pg_*_barrier_impl マクロに割り当てます。 3. **操作のラップ**: pg_atomic_*_impl 関数群は、ポインタ経由で渡された pg_atomic_uint* 構造体の value フィールドに対し、Solaris のシステム関数を直接呼び出します。 4. **結果の返却**: CAS 操作では、期待値と現在の値が一致したかどうかの真偽値を返し、不一致の場合は expected 引数を現在の値で更新します。

読者向けメモ

  • 本ファイルは Solaris 環境でのみ有効です。
  • pg_atomic_uint64 のアライメントには pg_attribute_aligned(8) が使用されていますが、コンパイラのバージョンによっては厳密な検証が必要です。
  • Solaris の atomic_cas は、期待値と現在の値が一致しない場合に現在の値を返す仕様であるため、PostgreSQL の期待する「期待値を更新する」動作と整合させています。
