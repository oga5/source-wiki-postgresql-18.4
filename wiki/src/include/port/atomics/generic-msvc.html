src/include/port/atomics/generic-msvc.h

概要

本ファイルは、Microsoft Visual C++ (MSVC) コンパイラ環境におけるアトミック操作の実装を提供します。PostgreSQL のアトミック操作抽象化レイヤーの一部であり、Windows プラットフォーム特有の組み込み関数（Interlocked API）をラップして、クロスプラットフォームなインターフェースに適合させます。

モジュール内での役割

src/include/port/atomics モジュールにおいて、MSVC を使用するビルド時のバックエンドとして機能します。atomics.h からインクルードされることを前提としており、コンパイラ固有のプリミティブを PostgreSQL 標準の pg_atomic_* 関数群へ変換する役割を担います。

責務

  • MSVC の Interlocked ファミリー関数を用いたアトミックな読み書き・演算の提供。
  • コンパイラおよびメモリバリアの定義（_ReadWriteBarrier, MemoryBarrier）。
  • 32bit および 64bit 整数に対するアトミック操作のサポート判定と実装。
  • Windows 環境におけるアーキテクチャ依存（特に 64bit ビルドの有無）の吸収。

重要なシンボル

  • pg_atomic_uint32 / pg_atomic_uint64: アトミック操作対象となる値を保持する構造体。
  • pg_atomic_compare_exchange_u32_impl: InterlockedCompareExchange をラップした比較交換操作。
  • pg_atomic_fetch_add_u32_impl: InterlockedExchangeAdd を用いた加算操作。
  • pg_compiler_barrier_impl: コンパイラの最適化を抑止するバリア。

追加の索引済みシンボル

  • pg_atomic_compare_exchange_u64_impl
  • pg_atomic_exchange_u32_impl
  • pg_atomic_exchange_u64_impl
  • pg_atomic_fetch_add_u64_impl

制御フローとデータフロー

本ファイルは関数定義の集合体であり、直接的な実行フローは持ちません。 1. atomics.h からのインクルード時に、PG_HAVE_ATOMIC_* マクロを定義することで、特定の操作が利用可能であることを上位レイヤーに通知します。 2. 各 *_impl 関数は、引数として渡された pg_atomic_* 構造体のポインタを介して、MSVC の組み込み関数を呼び出します。 3. 64bit 操作の一部は _WIN64 マクロで囲まれており、64bit ビルド時のみ有効化されます。

直接依存

  • <intrin.h>: MSVC の組み込み関数（Interlocked API 等）を利用するために必須のヘッダー。
  • intrin.h

関連コード

  • src/include/port/atomics/generic.h: アトミック操作の共通インターフェース定義。本ファイルが実装する関数のプロトタイプや期待される動作が定義されています。
  • atomics.h: 本ファイルをインクルードし、プラットフォームに応じた実装を選択するディスパッチャー。

読者向けメモ

  • **インクルードガードの欠如**: 本ファイルは単体でのインクルードを意図しておらず、必ず atomics.h を経由する必要があります。
  • **Interlocked API**: 実装の多くは Windows の Interlocked シリーズに依存しています。MSDN のドキュメントを参照する際は、これらの関数がメモリバリアを内包している点に注意してください。
  • **64bit 制限**: _InterlockedExchange64 等の関数は 64bit ビルド環境でのみ利用可能なため、条件付きコンパイルが適用されています。
