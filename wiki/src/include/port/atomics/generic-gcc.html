src/include/port/atomics/generic-gcc.h

概要

GCC（および互換コンパイラ）の組み込み関数を利用して、PostgreSQLの原子操作（Atomic Operations）を実装するためのヘッダーファイルです。

モジュール内での役割

src/include/port/atomics サブシステムの一部として、プラットフォーム固有の最適化が適用されない場合の「汎用的なGCC実装」を提供します。atomics.h からインクルードされることを前提としており、コンパイラが提供する __atomic または __sync 組み込み関数をラップします。

責務

  • コンパイラバリアおよびメモリバリア（Read/Write/Full）の定義。
  • pg_atomic_flag、pg_atomic_uint32、pg_atomic_uint64 のデータ構造定義。
  • コンパイラの機能フラグ（HAVE_GCC__ATOMIC_* 等）に基づいた、原子操作関数の実装提供。

重要なシンボル

  • pg_compiler_barrier_impl(): コンパイラによる命令の並び替えを防止する空のasmブロック。
  • pg_memory_barrier_impl(): __atomic_thread_fence または __sync_synchronize を用いたメモリバリア。
  • pg_atomic_uint32 / pg_atomic_uint64: 原子操作対象の値を保持する構造体。
  • pg_atomic_compare_exchange_u32_impl(): CAS（Compare-and-Swap）操作のGCC実装。

制御フローとデータフロー

本ファイルは関数定義の集合体であり、直接的な実行フローは持ちません。 1. **条件分岐**: HAVE_GCC__ATOMIC_* や HAVE_GCC__SYNC_* マクロの定義状況を確認し、利用可能な組み込み関数を選択します。 2. **実装のラップ**: __atomic API（より新しいメモリモデル対応）が利用可能な場合は優先的に使用し、古い __sync API はフォールバックとして使用します。 3. **インライン展開**: 実装された関数は static inline として定義され、呼び出し元で直接展開されます。

直接依存

  • atomics.h: 本ファイルは atomics.h からのみインクルードされるよう設計されており、INSIDE_ATOMICS_H マクロによるガードが存在します。

関連コード

  • src/include/port/atomics/generic.h: 原子操作のインターフェース定義。
  • src/include/port/atomics/arch-x86.h: アーキテクチャ固有の最適化実装（本ファイルよりも優先される場合がある）。

読者向けメモ

  • **インクルードガードの欠如**: 本ファイルには意図的にインクルードガードがありません。必ず atomics.h を経由して読み込んでください。
  • **メモリモデル**: __atomic 組み込み関数を使用する際は、可能な限り適切なメモリ順序（__ATOMIC_SEQ_CST 等）を指定していますが、一部の実装では保守的なモデルが選択されています。
  • **64bitサポート**: PG_DISABLE_64_BIT_ATOMICS が定義されている場合、64bitの原子操作は無効化されます。
