src/include/port/win32_port.h

概要

PostgreSQL を Windows 環境（MSVC および MinGW）でビルド・実行するための互換性レイヤーを定義するヘッダーファイルです。POSIX 準拠のシステムコールやデータ型が存在しない Windows において、PostgreSQL のコアコードが共通のインターフェースで動作するように、マクロ置換やラッパー関数を提供します。

モジュール内での役割

src/include/port ディレクトリ内で、Windows 特有の差異を吸収する中心的な役割を担います。他のプラットフォーム用ヘッダー（linux.h, darwin.h 等）と同様に、プラットフォーム依存の差異を抽象化し、メインのソースコードが OS の違いを意識せずに記述できるようにします。

責務

  • **API の抽象化**: fsync を _commit に置き換えるなど、POSIX 関数を Windows API でエミュレートします。
  • **ソケットの再定義**: Winsock API を POSIX ソケット API と互換性のある形にラップします。
  • **型と定数の補完**: ssize_t, mode_t, pid_t などの不足している型定義や、POSIX 準拠の定数（S_IRUSR 等）を提供します。
  • **エラーコードの変換**: Winsock エラーコードを POSIX エラーコードへマッピングし、strerror 等で適切に扱えるようにします。

重要なシンボル

  • ENABLE_SSPI: Windows のセキュリティサポートプロバイダー（SSPI）を有効化します。
  • pgwin32_socket, pgwin32_select: Winsock を PostgreSQL の要件に合わせてラップした関数群。
  • struct stat: MSVC の 32 ビット制限を回避し、64 ビット対応の __stat64 と互換性を持たせた構造体。
  • pgwin32_setlocale: Windows のロケール処理のバグを回避するためのラッパー。

制御フローとデータフロー

このファイルは、コンパイル時にプリプロセッサによって読み込まれます。 1. **ヘッダーのインクルード**: windows.h 等のシステムヘッダーを読み込む前に、競合を避けるためのマクロ（WIN32_LEAN_AND_MEAN 等）を定義します。 2. **マクロ置換**: stat や fstat などの関数を、src/port/ 内の独自実装関数へリダイレクトします。 3. **ラッパーの適用**: socket や bind などのネットワーク関数を、pgwin32_* プレフィックスを持つ関数へ置き換えます。 4. **型定義**: コンパイラ（MSVC/MinGW）に応じて不足している型を補完します。

直接依存

  • winsock2.h, ws2tcpip.h: ネットワーク通信用。
  • windows.h, ntstatus.h, winternl.h: Windows カーネルおよびシステム API 用。
  • process.h, signal.h, direct.h, sys/stat.h: POSIX 互換性のための標準ヘッダー。

関連コード

  • src/port/win32stat.c: stat 関連の独自実装。
  • src/backend/port/win32/socket.c: ソケットラッパーの実装。
  • src/backend/port/win32/signal.c: Windows におけるシグナルエミュレーション。
  • src/port/win32error.c: エラーコード変換ロジック。

読者向けメモ

  • 本ファイルは Cygwin ビルドでは読み込まれません。MinGW およびネイティブ Windows ビルド専用です。
  • windows.h をインクルードする前に特定の定義（WIN32_LEAN_AND_MEAN 等）を置く必要があるため、インクルード順序が非常に厳格です。修正を加える際は、既存の順序を崩さないよう注意してください。
  • Windows の stat 構造体は MSVC では 32 ビットであるため、本ファイルで 64 ビット対応の構造体に再定義しています。このため、sys/stat.h のインクルード前後でマクロのアンデファインと再定義を行う複雑な処理が含まれています。
