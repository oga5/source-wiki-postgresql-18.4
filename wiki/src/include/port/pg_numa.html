src/include/port/pg_numa.h

概要

pg_numa.h は、PostgreSQL における NUMA (Non-Uniform Memory Access) アーキテクチャ対応のためのポータビリティ層を定義するヘッダーファイルです。主に Linux 環境での libnuma を活用したメモリ配置の確認や制御を抽象化します。

モジュール内での役割

src/include/port ディレクトリの一部として、プラットフォーム固有のシステムコールやライブラリの差異を吸収し、PostgreSQL のコアエンジンが NUMA 認識型のメモリ管理を行えるようにするインターフェースを提供します。

責務

  • NUMA サブシステムの初期化と利用可否の判定。
  • プロセスが使用しているメモリページの配置ノードを照会する機能の提供。
  • NUMA 環境下で正しくメモリ情報を取得するための、ページフォールト誘発（メモリタッチ）処理の抽象化。

重要なシンボル

  • pg_numa_init(): NUMA サブシステムを初期化し、利用可能かを確認します。
  • pg_numa_query_pages(): 指定されたメモリページがどの NUMA ノードに配置されているかを照会します。
  • pg_numa_touch_mem_if_required(): move_pages(2) 等のシステムコールが正確な結果を返すよう、対象メモリを強制的にページフォールトさせます。

追加の索引済みシンボル

  • pg_numa_touch_mem_if_required

制御フローとデータフロー

1. **初期化**: pg_numa_init を呼び出し、libnuma の可用性と権限を確認します。 2. **準備**: メモリ配置を照会する前に、pg_numa_touch_mem_if_required を使用して対象メモリにアクセスし、物理メモリへのマッピングを確定させます。 3. **照会**: pg_numa_query_pages が内部で numa_move_pages を呼び出し、カーネルからページごとのノード情報を取得します。この際、カーネルのバグ回避のため、チャンク単位で処理を分割して実行します。

直接依存

  • c.h (PostgreSQL の基本型定義)
  • libnuma (Linux 環境で USE_LIBNUMA が定義されている場合)

関連コード

  • src/port/pg_numa.c: 上記関数の具体的な実装が含まれています。特に numa_move_pages のチャンク処理や、非対応環境での空実装（スタブ）が記述されています。

読者向けメモ

  • **プラットフォーム依存**: 現在、実質的な機能は Linux の libnuma に依存しています。他プラットフォームでは空の関数として動作し、NUMA 最適化は無効化されます。
  • **カーネルバグへの対応**: pg_numa_query_pages 内のチャンクサイズ制御は、特定のカーネルバージョンにおける do_pages_stat のバグを回避するための重要なワークアラウンドです。
  • **メモリタッチの重要性**: pg_numa_touch_mem_if_required を省略すると、ページが物理メモリに割り当てられていない場合、正しい NUMA ノード情報が取得できない可能性があるため注意してください。
