src/include/port/netbsd.h

概要

src/include/port/netbsd.h は、PostgreSQL を NetBSD 環境でコンパイルおよび実行するために必要な、プラットフォーム固有の定義や互換性レイヤーを提供するヘッダーファイルです。

モジュール内での役割

src/include/port ディレクトリ群の一部として、OS 間の差異を吸収する「ポート層」を構成します。他の OS（Linux, FreeBSD, Windows 等）向けヘッダーと同様に、ビルドシステムがターゲット OS を判別した際にインクルードされ、標準ライブラリの欠落や挙動の差異を補完します。

責務

  • NetBSD 特有のシステムヘッダーのインクルード管理。
  • PostgreSQL のコアコードが期待する標準的な型やマクロが、NetBSD 上で正しく定義されることを保証する。
  • 移植性が必要な機能（ネットワーク、スレッド、ファイル操作等）における NetBSD 固有の制約の吸収。

重要なシンボル

このファイルは通常、以下の役割を果たすシンボルを定義または再定義します。

  • **プラットフォーム識別マクロ**: NetBSD 環境であることを示すフラグ。
  • **互換性マクロ**: _NETBSD_SOURCE などの機能テストマクロの定義。
  • **欠落機能の代替**: NetBSD の標準ライブラリで不足している関数や定数の定義。

制御フローとデータフロー

本ファイルは実行コードを含まず、プリプロセッサによる条件分岐が主となります。 1. ビルド時に PORTNAME が netbsd と判定される。 2. コンパイル時に本ヘッダーが読み込まれる。 3. プリプロセッサが NetBSD 固有のシステム定義をロードし、PostgreSQL の共通コードが使用する型や関数を適切な NetBSD 実装にマッピングする。

直接依存

  • NetBSD のシステムヘッダー（<sys/...> や <netinet/...> など）。
  • PostgreSQL の共通ヘッダー（c.h 等）から間接的に参照されることが前提。

関連コード

  • src/include/port/freebsd.h, src/include/port/openbsd.h: 他の BSD 系 OS 用のヘッダー。実装の共通点が多く、比較対象として有用です。
  • src/include/c.h: PostgreSQL の全ソースがインクルードする基底ヘッダー。ここから各 OS 用のポートヘッダーが呼び出されます。

読者向けメモ

  • 本ファイルは非常に軽量であり、主に「NetBSD 上で PostgreSQL をビルドする際の環境調整」に特化しています。
  • NetBSD 特有の挙動や、特定のバージョンで発生する問題については、このファイル内のコメントや定義を確認してください。
  • 複雑なロジックはここには記述されず、多くの場合、configure スクリプトの判定結果に基づいてマクロが定義される仕組みになっています。
