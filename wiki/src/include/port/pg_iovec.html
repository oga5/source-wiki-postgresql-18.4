src/include/port/pg_iovec.h

概要

pg_iovec.h は、PostgreSQL におけるベクトル I/O（preadv/pwritev）のクロスプラットフォームな抽象化レイヤーです。OS 固有の差異を吸収し、Windows を含むすべてのサポート環境で一貫したインターフェースを提供します。

モジュール内での役割

src/include/port ディレクトリの一部として、プラットフォーム依存のシステムコールをラップし、コードベース全体で移植性の高い I/O 操作を可能にします。特に、標準的な sys/uio.h が存在しない環境や、機能が不完全な環境を補完します。

責務

  • **プラットフォーム抽象化**: Windows 環境における struct iovec の定義と、POSIX 非準拠環境での preadv/pwritev のエミュレーション。
  • **定数定義**: IOV_MAX が未定義の環境に対するフォールバック値の提供と、スタック利用に適した安全な上限値 PG_IOV_MAX の定義。
  • **I/O 最適化**: preadv/pwritev が利用可能な環境では直接呼び出し、利用不可な環境では pread/pwrite を用いたループによる代替実装を提供。

重要なシンボル

  • struct iovec: 複数のバッファを一度の I/O 操作で扱うための構造体。
  • PG_IOV_MAX: スタック上に iovec 配列を確保する際の安全な上限値（Min(IOV_MAX, 128)）。
  • pg_preadv() / pg_pwritev(): ベクトル I/O を実行するラッパー関数。

追加の索引済みシンボル

  • pg_preadv
  • pg_pwritev

制御フローとデータフロー

  • **ネイティブサポート時**: HAVE_DECL_PREADV 等が定義されている場合、カーネルの preadv/pwritev を直接呼び出します。iovcnt == 1 の場合はオーバーヘッドを避けるため、単一バッファ用の pread/pwrite に最適化されます。
  • **エミュレーション時**: preadv/pwritev が存在しない環境では、ループ内で pg_pread/pg_pwrite を順次呼び出します。各呼び出し後にオフセットを更新し、部分的な読み書きが発生した場合はその時点までの合計バイト数を返します。

直接依存

  • <limits.h>: IOV_MAX の定義確認に使用。
  • <sys/uio.h>: POSIX 環境での標準定義。
  • <unistd.h>: POSIX 環境での標準 I/O 関数定義。
  • limits.h
  • sys/uio.h
  • unistd.h

関連コード

  • src/include/port/win32.h: Windows 環境特有の定義や互換性レイヤー。
  • src/port/preadv.c / src/port/pwritev.c (存在する場合): 本ヘッダーで宣言された関数の実体や、関連する移植用実装。

読者向けメモ

  • **Windows の挙動**: Windows 環境では pg_preadv/pg_pwritev を使用すると、ファイルポインタが移動する副作用がある点に注意してください。
  • **スタック利用**: PG_IOV_MAX はスタックオーバーフローを防ぐための安全策です。これを超える数の iovec を扱う場合は、ヒープメモリの使用を検討してください。
  • **IWYU**: sys/uio.h は IWYU (Include What You Use) のエクスポート対象としてマークされており、このヘッダーをインクルードすることで必要な型定義が確実に利用可能になります。
