src/include/common/jsonapi.h

概要

jsonapi.h は、PostgreSQL における JSON 解析および字句解析のための汎用インターフェースを定義するヘッダーファイルです。バックエンドとフロントエンド（libpq 等）の両方で利用可能な、イベント駆動型の JSON パーサーを提供します。

モジュール内での役割

src/include/common モジュールの一部として、JSON データの検証、抽出、および変換を行うための基盤を提供します。jsonapi.c と対になっており、メモリ管理や文字列操作の抽象化レイヤーを介して、環境依存性を排除した解析ロジックを実装しています。

責務

  • **字句解析の提供**: 入力ストリームから JSON トークン（文字列、数値、構造記号など）を順次切り出す。
  • **イベント駆動型解析**: JsonSemAction 構造体を通じて、解析の各段階（オブジェクト開始/終了、フィールド開始など）でコールバック関数を呼び出す。
  • **増分解析のサポート**: pg_parse_json_incremental を通じ、断片的な入力データに対するストリーミング解析を可能にする。
  • **エラーハンドリング**: 解析エラーの分類と、診断メッセージの生成。

重要なシンボル

  • JsonLexContext: 解析状態を保持するメイン構造体。入力バッファ、現在のトークン、解析レベル、メモリ管理フラグを含む。
  • JsonSemAction: 解析中に実行されるコールバック関数のテーブル。ユーザーはこれを定義して JSON 構造からデータを抽出する。
  • pg_parse_json: 完全な JSON 文字列を解析するためのメインエントリーポイント。
  • JsonTokenType: JSON の各要素を識別する列挙型。
  • JsonParseErrorType: 解析結果やエラーの種類を示す列挙型。

制御フローとデータフロー

1. **初期化**: makeJsonLexContext... 関数で JsonLexContext を作成し、必要に応じてエスケープ解除（need_escapes）の有無を指定する。 2. **解析実行**: pg_parse_json または pg_parse_json_incremental を呼び出し、JsonSemAction で定義したコールバックを登録する。 3. **コールバック**: パーサーがトークンを認識するたびに、対応するアクション関数（object_start, scalar 等）が実行され、ユーザー定義の semstate にデータが蓄積される。 4. **終了処理**: freeJsonLexContext を呼び出し、解析に使用したリソースを解放する。

直接依存

本ファイルは標準的な C ヘッダー以外への直接的な依存関係を持ちません。ただし、実装ファイル jsonapi.c は postgres.h または postgres_fe.h を含み、環境に応じて StringInfo または PQExpBuffer を使い分けます。

  • src/interfaces/libpq/pqexpbuffer.h: フロントエンドでの文字列バッファ管理に使用。

読者向けメモ

  • **メモリ管理**: need_escapes を true にすると、文字列のデコード処理が発生し、メモリ消費量が増加します。不要な場合は false を推奨します。
  • **トークン所有権**: setJsonLexContextOwnsTokens を使用することで、コールバックに渡される文字列トークンの解放責任をパーサー側に委譲できます。長期実行されるプロセスでは、メモリリークを防ぐために重要です。
  • **エラー報告**: JSON_SEM_ACTION_FAILED が返された場合、エラーの詳細はコールバック関数内で報告済みであると見なされます。
