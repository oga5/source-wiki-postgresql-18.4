src/include/common/unicode_version.h

概要

PostgreSQL がサポートする Unicode 標準のバージョンを定義するヘッダーファイルです。

モジュール内での役割

src/include/common ディレクトリ内で、Unicode 関連の機能（正規化、カテゴリ判定、大文字小文字変換など）が準拠すべきバージョン情報を一元管理する役割を担います。

責務

  • PostgreSQL 全体で使用される Unicode バージョンを単一のソースとして定義する。
  • バージョンアップ時の変更箇所を最小限に抑える。

重要なシンボル

  • PG_UNICODE_VERSION: 現在の PostgreSQL が準拠している Unicode バージョンを示す文字列マクロです。

制御フローとデータフロー

本ファイルは純粋な定義ファイルであり、実行時の制御フローは存在しません。コンパイル時にこのマクロが参照され、Unicode 関連の処理を行う各モジュールでバージョン情報として利用されます。

直接依存

  • なし

関連コード

  • src/include/common/unicode_norm.h: Unicode 正規化機能。
  • src/include/common/unicode_category.h: Unicode 文字カテゴリ判定機能。
  • src/include/common/unicode_case.h: Unicode 大文字小文字変換機能。
  • 各種 unicode_*_table.h: 本ファイルで定義されたバージョンに基づいて生成されたデータテーブル群。

読者向けメモ

  • 本ファイルは非常にシンプルですが、PostgreSQL の国際化対応（I18N）の基盤となる重要な定数です。
  • Unicode 標準の更新に伴い、このバージョン番号を変更する際は、関連するテーブル生成スクリプトやデータテーブルも併せて更新する必要がある点に注意してください。
