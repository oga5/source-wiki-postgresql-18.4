src/include/common/file_perm.h

概要

PostgreSQLのデータディレクトリ（PGDATA）配下に作成されるファイルやディレクトリのアクセス権限（パーミッション）を管理するための定義とインターフェースを提供します。

モジュール内での役割

src/include/common モジュールの一部として、ファイルシステム操作を行うフロントエンドツールやバックエンドプロセスに対し、一貫したセキュリティポリシーを適用するための基盤を提供します。

責務

  • ファイルおよびディレクトリ作成時のデフォルトモード（mode_t）の定義。
  • umask に適用すべきマスク値の定義。
  • 既存のデータディレクトリの権限に基づき、新規作成時の権限設定を動的に決定する機能の提供。

重要なシンボル

  • PG_DIR_MODE_OWNER / PG_FILE_MODE_OWNER: 所有者のみにアクセスを許可するデフォルトの権限。
  • PG_DIR_MODE_GROUP / PG_FILE_MODE_GROUP: グループによる読み取り/実行権限を許可する拡張権限。
  • SetDataDirectoryCreatePerm(int): 指定されたモードに基づいてグローバルな権限変数を更新する関数。
  • GetDataDirectoryCreatePerm(const char*): 指定パスのディレクトリ権限を読み取り、適切な作成モードを自動設定する関数。

制御フローとデータフロー

1. **初期化**: プロセス起動時、グローバル変数（pg_dir_create_mode 等）はデフォルトの OWNER モードで初期化されます。 2. **動的設定**: GetDataDirectoryCreatePerm が呼び出されると、stat() を用いて対象ディレクトリの現在の権限を取得します。 3. **権限伝播**: SetDataDirectoryCreatePerm が呼び出されると、ディレクトリの権限にグループアクセスが含まれているか判定し、グローバル変数を GROUP モードへ昇格させます。 4. **適用**: ファイル作成処理を行う各モジュールは、これらのグローバル変数を参照して mkdir や open を実行します。

直接依存

  • <sys/stat.h>: S_IRWXU などの標準的なファイルモード定数を利用するために必須です。
  • sys/stat.h

関連コード

  • src/common/file_perm.c: 本ヘッダーで宣言された関数の実装およびグローバル変数の実体が含まれます。

読者向けメモ

  • 本モジュールは、PostgreSQLがデータディレクトリの権限を「所有者のみ」から「グループアクセス許可」へ柔軟に切り替えられるように設計されています。
  • Windows環境では umask や Unix 形式のパーミッションが限定的であるため、GetDataDirectoryCreatePerm の実装にはプラットフォーム依存の条件分岐が含まれています。
  • 新規にファイル作成処理を実装する際は、ハードコードされたモード値を使用せず、必ず本モジュールで定義されたグローバル変数を使用してください。
