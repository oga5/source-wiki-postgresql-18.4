src/include/common/unicode_case_table.h

概要

Unicode 文字の大文字・小文字変換およびケースフォールディング（Case Folding）のための静的ルックアップテーブルと、それらを参照するためのインデックスロジックを定義するヘッダーファイルです。Perl スクリプトによって自動生成されるデータ構造であり、手動編集は禁止されています。

モジュール内での役割

src/include/common モジュールの一部として、PostgreSQL のバックエンドおよびフロントエンドの両方で利用される Unicode 文字処理の基盤を提供します。unicode_case.h で定義されたインターフェースを実装するための具体的なデータを提供します。

責務

  • **ケースマッピングテーブルの保持**: 小文字、タイトルケース、大文字、フォールディングの各変換テーブルの提供。
  • **特殊ケースの管理**: 単一のコードポイントから複数のコードポイントへ変換されるような、単純なマッピングでは表現できない特殊なケースの定義。
  • **インデックス計算**: 任意の Unicode コードポイントを、変換テーブルのオフセットへ変換するロジックの提供。

重要なシンボル

定数・フラグ

  • MAX_CASE_EXPANSION: 1 文字から変換される最大コードポイント数（3）。
  • PG_U_FINAL_SIGMA: ギリシャ文字のシグマ（σ）の語末形判定に使用するフラグ。

型定義

  • CaseKind: 変換の種類（Lower, Title, Upper, Fold）を識別する列挙型。
  • pg_special_case: 単純なマッピングで表現できない特殊なケースを保持する構造体。

データテーブル

  • special_case: 特殊ケースマッピングの配列。
  • case_map_lower, case_map_title, case_map_upper, case_map_fold: 各変換種別に対応するマッピングテーブル。
  • case_map_special: special_case 配列へのインデックスを保持するテーブル。
  • case_map: コードポイントからテーブルのインデックスを引くためのルックアップテーブル。

制御フローとデータフロー

1. **インデックス取得**: case_index(pg_wchar cp) 関数が、入力されたコードポイント cp を引数に受け取り、case_map 配列から対応するインデックスを返します。 2. **マッピング参照**: 取得したインデックスを使用して、case_map_lower 等の各テーブルから変換後のコードポイントを取得します。 3. **特殊ケース判定**: case_map_special テーブルを参照し、値が 0 でなければ special_case 配列から詳細なマッピング情報を取得します。

  • mb/pg_wchar.h: PostgreSQL のワイド文字型（pg_wchar）の定義。

読者向けメモ

  • **自動生成**: 本ファイルは自動生成されるため、直接修正しないでください。修正が必要な場合は、生成元の Perl スクリプトを編集してください。
  • **メモリ**: すべてのテーブルは static const として定義されており、読み取り専用のデータセグメントに配置されます。実行時のメモリ割り当ては発生しません。
