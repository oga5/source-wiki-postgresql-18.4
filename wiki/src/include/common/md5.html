src/include/common/md5.h

概要

MD5 メッセージダイジェストアルゴリズムを PostgreSQL 全体（フロントエンドおよびバックエンド）で利用するための共通定義およびインターフェースを提供します。

モジュール内での役割

src/include/common モジュールの一部として、認証やデータ整合性チェックに必要な MD5 ハッシュ計算の標準的な入り口を定義します。実装の詳細（md5.c）を隠蔽し、呼び出し元に対して一貫した API を提供します。

責務

  • MD5 アルゴリズムに関連する定数（ダイジェスト長、ブロックサイズ等）の定義。
  • パスワードのハッシュ化およびバイナリ変換のための関数プロトタイプの提供。
  • フロントエンドとバックエンドの両方で利用可能な、移植性の高い MD5 操作の抽象化。

重要なシンボル

  • MD5_DIGEST_LENGTH: MD5 ハッシュ結果のバイト数（16バイト）。
  • pg_md5_hash(): 入力バッファから16進数文字列形式のハッシュを生成する主要関数。
  • pg_md5_binary(): 入力バッファからバイナリ形式のハッシュを生成する関数。
  • pg_md5_encrypt(): パスワードとソルトを用いて MD5 ベースの暗号化を行う関数。

制御フローとデータフロー

本ファイルはヘッダー定義のみを行うため、制御フローは存在しません。実際の計算処理は、pg_md5_hash 等の関数が呼び出された際に、対応する実装ファイル（src/common/md5.c や md5_common.c）へ委譲されます。データは呼び出し元からバッファとして渡され、指定された形式（文字列またはバイナリ）で出力バッファに書き込まれます。

直接依存

  • なし（純粋なヘッダーファイルであり、他のヘッダーへの依存関係は明示されていません）。

関連コード

  • src/common/md5.c: MD5 アルゴリズムの低レベルな実装。
  • src/include/common/cryptohash.h: より現代的な暗号学的ハッシュ関数（SHA-2等）を扱うためのインターフェース。

読者向けメモ

  • 本ヘッダーは RFC 1321 に準拠した MD5 実装を前提としています。
  • セキュリティ上の理由から、新規の認証メカニズムには MD5 ではなく、cryptohash.h で提供されるより強力なハッシュアルゴリズム（SHA-256 等）の使用が推奨されます。
  • MD5_PASSWD_LEN は、PostgreSQL の MD5 パスワード形式（md5 + 32文字の16進数）に基づいた定数です。
