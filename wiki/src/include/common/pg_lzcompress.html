src/include/common/pg_lzcompress.h

概要

PostgreSQL 組み込みの LZ（Lempel-Ziv）圧縮アルゴリズムのインターフェースを定義するヘッダーファイルです。主に TOAST（The Oversized-Attribute Storage Technique）におけるデータ圧縮に使用されます。

モジュール内での役割

src/common/pg_lzcompress.c で実装される圧縮・伸長ロジックに対する公開 API を提供します。データ圧縮の戦略（PGLZ_Strategy）を定義し、圧縮処理の可否や効率を制御するための設定を提供します。

責務

  • 圧縮アルゴリズムの挙動を制御する戦略構造体 PGLZ_Strategy の定義。
  • 圧縮後のバッファサイズを算出するマクロ PGLZ_MAX_OUTPUT の提供。
  • 圧縮および伸長関数のプロトタイプ宣言。

重要なシンボル

  • PGLZ_Strategy: 圧縮の閾値やマッチングの探索範囲を制御するパラメータ群。
  • PGLZ_MAX_OUTPUT(_dlen): 圧縮結果を格納するために必要なバッファサイズを計算するマクロ。
  • pglz_compress(): データを圧縮するメイン関数。
  • pglz_decompress(): 圧縮されたデータを元の形式に復元する関数。
  • PGLZ_strategy_default: TOAST で推奨される標準的な圧縮戦略。

制御フローとデータフロー

本ファイルはインターフェース定義のみですが、実際のフローは以下の通りです： 1. **圧縮時**: pglz_compress が入力データと PGLZ_Strategy を受け取り、履歴テーブルを用いてマッチングを探索。制御バイトとタグ（オフセットと長さ）を生成して出力バッファに書き込みます。 2. **伸長時**: pglz_decompress が制御バイトを読み取り、ビットが立っていればタグを解析して過去の出力データからコピーし、立っていなければリテラルとしてコピーします。

直接依存

  • なし（標準的な型定義 int32 等は PostgreSQL の基本ヘッダー経由で利用されます）。

関連コード

  • src/common/pg_lzcompress.c: 本ヘッダーで宣言された関数の実体。アルゴリズムの詳細な実装が含まれます。
  • src/include/common/compression.h: PostgreSQL 全体の圧縮フレームワークを扱う上位のヘッダー。

読者向けメモ

  • **バッファサイズ**: pglz_compress を呼び出す際は、必ず PGLZ_MAX_OUTPUT を使用して十分なメモリを確保してください。
  • **タグの制限**: この実装はオフセット最大 4095 バイト、長さ最大 273 バイトまでのコピーをサポートしています。
  • **戦略の選択**: PGLZ_strategy_default は一般的な用途向けですが、特定のデータ特性に合わせて PGLZ_Strategy をカスタマイズすることが可能です。
