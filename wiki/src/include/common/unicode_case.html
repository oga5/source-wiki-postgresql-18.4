src/include/common/unicode_case.h

概要

Unicode 文字列の大文字・小文字変換およびケースフォールディング（正規化のための比較用変換）を行うためのインターフェースを定義するヘッダーファイルです。フロントエンドとバックエンドの両方から利用可能です。

モジュール内での役割

src/include/common モジュールの一部として、Unicode 処理の基盤を提供します。文字単位の単純な変換から、文字列全体の変換までをカバーし、ロケールに依存しない標準的な Unicode ケースマッピングを実装するための規約を定めています。

責務

  • 単一の pg_wchar（Unicode コードポイント）に対する単純なケース変換関数の提供。
  • UTF-8 エンコードされた文字列に対する、バッファサイズを考慮した安全なケース変換関数の提供。
  • タイトルケース変換に必要な単語境界判定のためのコールバック定義。

重要なシンボル

  • pg_wchar: PostgreSQL で使用される Unicode コードポイント型。
  • WordBoundaryNext: タイトルケース変換時に単語の境界を特定するための関数ポインタ型。
  • unicode_strlower / unicode_strupper / unicode_strfold: 文字列全体を変換する主要関数。
  • unicode_titlecase_simple: 単語境界を考慮したタイトルケース変換の基盤。

制御フローとデータフロー

1. **単一文字変換**: unicode_lowercase_simple 等は、unicode_case_table.h で定義されたマッピングテーブルを参照し、対応するコードポイントを返します。 2. **文字列変換**: unicode_strlower 等は、入力文字列を走査し、各文字に対して casemap（実装ファイル側）を呼び出します。 3. **バッファ管理**: dstsize が 0 の場合、変換後の必要なバイト数を計算して返すため、呼び出し側は事前にメモリ確保量を決定できます。 4. **タイトルケース**: unicode_strtitle は wbnext コールバックを通じて単語の開始位置を逐次取得し、境界に応じた変換を適用します。

直接依存

  • mb/pg_wchar.h: pg_wchar 型の定義を提供。

関連コード

  • src/common/unicode_case.c: 本ヘッダーで宣言された関数の実装。
  • src/include/common/unicode_case_table.h: 変換に使用される静的なマッピングテーブル。
  • src/include/common/unicode_category.h: 文字の属性判定に使用。

読者向けメモ

  • **フル変換の指定**: full 引数を true に設定すると、Unicode 標準で定義される「特殊なケースマッピング（例: ドイツ語の 'ß' が 'SS' になる等）」が適用されます。
  • **メモリ安全性**: dstsize を超える変換が発生した場合、結果は切り詰められます。呼び出し側は、戻り値が dstsize より大きい場合にバッファ不足を検知する必要があります。
  • **UTF-8 前提**: 入力文字列 src は常に UTF-8 であることが期待されています。
