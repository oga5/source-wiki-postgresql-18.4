src/include/common/file_utils.h

概要

PostgreSQLにおけるファイル操作およびディレクトリ走査のための汎用ユーティリティ関数群を定義するヘッダーファイルです。主にフロントエンドツール（pg_basebackup 等）やサーバープロセスが、堅牢なファイルシステム操作を行うために使用します。

モジュール内での役割

src/include/common モジュールの一部として、プラットフォーム間の差異を吸収し、ファイルシステムの整合性を保つための低レベルな抽象化を提供します。特に、データディレクトリの同期や、一時ファイルの管理といったインフラストラクチャ層の基盤を担います。

責務

  • **ファイルシステム同期**: データディレクトリや特定のパスに対する fsync や syncfs の実行。
  • **ディレクトリ走査**: 再帰的なディレクトリ探索と、各エントリに対するアクションの適用。
  • **堅牢なファイル操作**: durable_rename によるアトミックなファイル置換や、書き込みリトライ処理の提供。
  • **メタデータ取得**: dirent 構造体からのファイル種別判定。

重要なシンボル

  • PGFileType: ファイル種別（通常ファイル、ディレクトリ、シンボリックリンク等）を識別する列挙型。
  • DataDirSyncMethod: データディレクトリ同期戦略（fsync または syncfs）の選択肢。
  • sync_pgdata(): データディレクトリ全体を同期するための主要関数。
  • pg_pwritev_with_retry(): 中断された書き込みを再試行するラッパー関数。
  • PG_TEMP_FILES_DIR: 一時ファイルが格納される標準ディレクトリ名。

制御フローとデータフロー

本ファイルはインターフェース定義が主です。実際の処理フローは src/common/file_utils.c に委譲されています。 1. **同期フロー**: sync_pgdata が呼び出されると、指定された sync_method に応じてディレクトリツリーを走査し、各ファイルに対して fsync を発行するか、ファイルシステム単位の syncfs を実行します。 2. **書き込みフロー**: pg_pwritev_with_retry は、システムコールが部分的な書き込みで終了した場合に、残りのデータを計算して再試行を繰り返すループ構造を持ちます。

直接依存

  • <dirent.h>: ディレクトリ操作のための標準ヘッダー。
  • struct iovec (前方宣言): port/pg_iovec.h を含めずに iovec を扱うための前方宣言。
  • dirent.h

関連コード

  • src/common/file_utils.c: 本ヘッダーで宣言された関数の実装。
  • src/include/common/relpath.h: PostgreSQLのパス命名規則や相対パス計算に関連する定義。
  • src/include/common/logging.h: フロントエンドツールでのエラー報告に使用。

読者向けメモ

  • **FRONTEND マクロ**: FRONTEND が定義されている場合のみ利用可能な関数（sync_pgdata 等）があるため、サーバー側コードとクライアント側コードのどちらから参照しているかに注意してください。
  • **移植性**: syncfs などの高度な同期メソッドはプラットフォーム依存であるため、ビルド環境の機能フラグ（HAVE_SYNCFS 等）に依存します。
  • **エラーハンドリング**: ファイル操作は失敗する可能性があるため、戻り値のチェックと errno に基づくエラーログ出力が必須です。
