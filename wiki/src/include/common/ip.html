src/include/common/ip.h

概要

src/include/common/ip.h は、PostgreSQL におけるネットワーク通信の抽象化レイヤーを定義するヘッダーファイルです。IPv4、IPv6、および Unix ドメインソケットを透過的に扱うための共通インターフェースを提供します。

モジュール内での役割

src/include/common モジュールの一部として、フロントエンド（クライアントツール）とバックエンド（サーバープロセス）の両方から利用されるネットワークユーティリティを提供します。システム固有のネットワーク API の差異を吸収し、コードの移植性を高める役割を担います。

責務

  • 標準的な getaddrinfo および getnameinfo をラップし、Unix ドメインソケットを統合的に扱えるようにする。
  • ネットワークアドレス情報の取得および解放のための統一的な API を提供する。
  • 通信エラー発生時にも安全なデフォルト値（"???"）を返すことで、呼び出し側の堅牢性を確保する。

重要なシンボル

  • pg_getaddrinfo_all: ホスト名とサービス名からアドレス情報を取得する。Unix ドメインソケットの場合は専用の処理へルーティングする。
  • pg_freeaddrinfo_all: 取得したアドレス情報構造体を解放する。Unix ドメインソケット用と標準の freeaddrinfo を適切に使い分ける。
  • pg_getnameinfo_all: ソケットアドレスからホスト名やサービス名を解決する。標準の getnameinfo を拡張し、エラー時にも文字列バッファを埋める。

制御フローとデータフロー

1. **アドレス解決**: 呼び出し元が pg_getaddrinfo_all を呼び出すと、hintp->ai_family が AF_UNIX かどうかを判定します。 2. **ルーティング**: AF_UNIX であれば ip.c 内の getaddrinfo_unix が実行され、それ以外は標準の getaddrinfo が呼び出されます。 3. **リソース管理**: pg_freeaddrinfo_all は、アドレス情報の生成元（Unix ソケット用か標準 API か）を識別し、適切なメモリ解放処理を行います。 4. **名前解決**: pg_getnameinfo_all は、sockaddr_storage を受け取り、Unix ソケットであれば getnameinfo_unix を、それ以外は標準の getnameinfo を呼び出します。

直接依存

  • <netdb.h>: addrinfo 構造体および関連関数の定義。
  • <sys/socket.h>: sockaddr 関連の基本定義。
  • libpq/pqcomm.h: PostgreSQL の通信プロトコルに関連する定義。
  • netdb.h
  • sys/socket.h

関連コード

  • src/common/ip.c: 本ヘッダーで宣言された関数の実装。Unix ドメインソケットの具体的な処理が含まれます。
  • src/include/libpq/pqcomm.h: 通信に関連する定数や構造体の定義。

読者向けメモ

  • 本モジュールは、PostgreSQL が IPv6 をサポートするために導入された抽象化層です。
  • pg_getnameinfo_all は標準の getnameinfo と異なり、エラー時でも node や service バッファに "???" を書き込む仕様である点に注意してください。
  • Unix ドメインソケットを扱う際は、pg_freeaddrinfo_all に正しい hint_ai_family を渡さないとメモリリークや不正な解放が発生する可能性があります。
