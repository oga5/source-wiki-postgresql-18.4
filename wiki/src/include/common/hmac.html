src/include/common/hmac.h

概要

本ファイルは、RFC 2104 で規定される「Keyed-Hashing for Message Authentication (HMAC)」を PostgreSQL 内で利用するための汎用インターフェースを定義しています。

モジュール内での役割

src/include/common モジュールの一部として、ハッシュ関数（cryptohash.h）の上に構築された認証付きメッセージ認証コードの生成機能を提供します。バックエンドとフロントエンドの両方から利用可能です。

責務

  • HMAC コンテキストのライフサイクル管理（作成、初期化、解放）。
  • メッセージデータの更新および最終的なダイジェストの算出。
  • エラー状態の取得。

重要なシンボル

  • pg_hmac_ctx: HMAC 計算の状態を保持する不透明な（opaque）構造体。実装詳細は hmac.c に隠蔽されています。
  • pg_hmac_create(): 指定されたハッシュアルゴリズム（pg_cryptohash_type）に基づきコンテキストを生成します。
  • pg_hmac_init(): 秘密鍵を用いて HMAC コンテキストを初期化します。
  • pg_hmac_update() / pg_hmac_final(): データの入力と最終的な計算を行います。

制御フローとデータフロー

1. **作成**: pg_hmac_create でコンテキストを確保し、ハッシュタイプを決定します。 2. **初期化**: pg_hmac_init で鍵を処理します。鍵がブロックサイズより長い場合はハッシュ化して短縮します。 3. **更新**: pg_hmac_update を繰り返し呼び出し、メッセージデータを入力します。 4. **終了**: pg_hmac_final で内部的な ipad/opad を用いた計算を完了し、結果を dest に書き出します。 5. **解放**: pg_hmac_free でメモリを解放します。

直接依存

  • common/cryptohash.h: 基盤となるハッシュアルゴリズムの型定義（pg_cryptohash_type）を提供します。

関連コード

  • src/common/hmac.c: 本ヘッダーの関数実装。MD5, SHA1, SHA2 ファミリーのハッシュ関数を内部的に呼び出します。
  • src/include/common/cryptohash.h: HMAC の基礎となるハッシュ計算インターフェース。

読者向けメモ

  • **メモリ管理**: バックエンドでは palloc、フロントエンドでは malloc が使用されるよう hmac.c 内で抽象化されています。
  • **エラーハンドリング**: pg_hmac_error() を通じて、計算中に発生したエラー理由を取得可能です。
  • **実装の安全性**: pg_hmac_ctx は不透明な構造体として定義されているため、呼び出し側は内部の k_ipad や k_opad に直接触れる必要はありません。
