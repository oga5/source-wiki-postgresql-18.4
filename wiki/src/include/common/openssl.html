src/include/common/openssl.h

概要

PostgreSQL のフロントエンドおよびバックエンド間で共有される、OpenSSL ライブラリ利用のための共通定義を提供します。主に TLS プロトコルバージョンの抽象化と、OpenSSL 依存機能の条件付きコンパイルを制御します。

モジュール内での役割

src/include/common モジュールの一部として、暗号化通信や認証に関連するコンポーネントに対し、OpenSSL のバージョン互換性を吸収した定数を提供します。これにより、各サブシステムが個別に OpenSSL のバージョン判定を行う重複を排除します。

責務

  • OpenSSL が有効な環境下での TLS プロトコルバージョンの定義。
  • LibreSSL 等のバリエーションを含む、OpenSSL 互換ライブラリ間でのバージョン判定の標準化。
  • USE_OPENSSL マクロによる、OpenSSL 依存コードの安全なガード。

重要なシンボル

  • MIN_OPENSSL_TLS_VERSION: PostgreSQL がサポートする最小の TLS バージョン（"TLSv1"）。
  • MAX_OPENSSL_TLS_VERSION: コンパイル時に検出された、利用可能な最大の TLS バージョン。
  • USE_OPENSSL: OpenSSL がビルド時に有効化されているかを示すフラグ。

制御フローとデータフロー

本ファイルは実行時のフローを制御するものではなく、コンパイル時のプリプロセッサ定義を提供します。 1. USE_OPENSSL が定義されている場合、<openssl/ssl.h> をインクルードします。 2. コンパイル環境のヘッダーファイルが提供するマクロ（TLS1_3_VERSION 等）を評価し、MAX_OPENSSL_TLS_VERSION を決定します。 3. この定数は、接続確立時の SSL コンテキスト設定において、プロトコル範囲を指定するために利用されます。

直接依存

  • <openssl/ssl.h>: OpenSSL の主要な API を提供する外部ライブラリヘッダー。
  • openssl/ssl.h

関連コード

  • contrib/pgcrypto/openssl.c: 本ヘッダーで定義された OpenSSL 機能を活用し、暗号化処理の実装を行うモジュール。
  • src/include/common/cryptohash.h: OpenSSL を利用したハッシュ計算のインターフェース。
  • src/backend/libpq/be-secure-openssl.c: バックエンドでの SSL 接続処理の実装。

読者向けメモ

  • 本ファイルは OpenSSL のバージョン判定を簡略化するためのものです。openssl.cnf による実行時の制限は考慮されないため、セキュリティポリシーの適用には注意が必要です。
  • LibreSSL を含む環境では、ライブラリが提供するマクロの差異により MAX_OPENSSL_TLS_VERSION が変動する可能性があるため、特定のバージョンに依存したロジックを組む際は注意してください。
