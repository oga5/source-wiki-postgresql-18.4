src/include/common/unicode_normprops_table.h

概要

本ファイルは、Unicode 正規化（Normalization）における「クイックチェック（Quick Check）」処理で使用される静的なルックアップテーブルと、それに関連するハッシュ関数を定義しています。

モジュール内での役割

src/common/unicode 配下の生成スクリプトによって自動生成されるデータ定義ファイルです。PostgreSQL の Unicode 正規化サブシステムにおいて、特定のコードポイントが正規化済みであるか、あるいは正規化が必要かを高速に判定するためのデータを提供します。

責務

  • NFC（Normalization Form C）および NFKC（Normalization Form KC）のクイックチェック用データの保持。
  • 各正規化形式に対応する完全ハッシュ関数（Perfect Hash Function）の提供。
  • 効率的なルックアップのためのデータ構造（pg_unicode_normprops 等）の定義。

重要なシンボル

  • pg_unicode_normprops: コードポイントとクイックチェック状態（YES/NO/MAYBE）を保持する構造体。
  • pg_unicode_norminfo: ルックアップテーブル、ハッシュ関数、要素数を含む管理構造体。
  • UnicodeNormProps_NFC_QC / UnicodeNormProps_NFKC_QC: 各正規化形式のプロパティテーブル。
  • NFC_QC_hash_func / NFKC_QC_hash_func: 該当テーブルに対する完全ハッシュ関数。
  • UnicodeNormInfo_NFC_QC / UnicodeNormInfo_NFKC_QC: 外部から参照されるルックアップ用インターフェース。

追加の索引済みシンボル

  • qc_hash_func

制御フローとデータフロー

1. 正規化処理が必要な際、呼び出し元は UnicodeNormInfo_NFC_QC（または NFKC_QC）を参照します。 2. 対象のコードポイントを hash 関数（NFC_QC_hash_func 等）に渡し、インデックスを算出します。 3. 算出されたインデックスを用いて normprops 配列から該当する pg_unicode_normprops を取得します。 4. 取得した quickcheck 値に基づき、正規化の要否を判断します。

直接依存

  • common/unicode_norm.h: 正規化に関連する定数や型定義（UNICODE_NORM_QC_MAYBE 等）を提供。

関連コード

  • src/common/unicode/norm_test.c (または関連する正規化実装): 本テーブルを利用して実際の正規化を行うコード。

読者向けメモ

  • **自動生成**: 本ファイルは手動で編集しないでください。変更が必要な場合は、生成スクリプト generate-unicode_normprops_table.pl を修正し、再生成してください。
  • **完全ハッシュ**: ここで定義されているハッシュ関数は、特定のデータセットに対して衝突が発生しないように設計された完全ハッシュです。ハッシュ関数の定数やテーブル h は、生成スクリプトによって最適化されています。
  • **メモリ効率**: pg_unicode_normprops 構造体ではビットフィールドを使用してメモリ使用量を最小限に抑えています。
