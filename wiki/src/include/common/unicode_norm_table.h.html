src/include/common/unicode_norm_table.h

概要

Unicode 正規化（Normalization）処理で使用される、文字の分解（Decomposition）および合成（Composition）のための静的データテーブルを定義するヘッダーファイルです。Perl スクリプトによって自動生成されるデータ構造であり、PostgreSQL の Unicode 処理の基盤となります。

モジュール内での役割

src/include/common モジュールの一部として、Unicode 文字列の正規化ロジック（主に unicode_norm.c 等）にデータを提供します。このファイルはロジックを含まず、検索対象となる静的なデータ構造のみを保持しています。

責務

  • **正規化データの提供**: Unicode コードポイントとそれに対応する分解情報をマッピングする。
  • **分解属性の保持**: 互換分解（Compatibility decomposition）やインライン分解のフラグ管理。
  • **合成の制御**: 特定の文字を合成対象から除外するためのフラグ（DECOMP_NO_COMPOSE）の提供。

重要なシンボル

データ構造

  • struct pg_unicode_decomposition: 各文字の分解情報を保持する構造体。
  • codepoint: 対象の Unicode コードポイント。
  • comb_class: 文字の結合クラス（Combining Class）。
  • dec_size_flags: 分解のサイズおよびフラグ（インライン、互換分解、合成禁止など）。
  • dec_index: UnicodeDecomp_codepoints へのインデックス、またはインライン分解時のコードポイント。

定数テーブル

  • UnicodeDecompMain: 全ての分解情報を格納した静的配列。
  • UnicodeDecomp_codepoints: 分解後のコードポイント列を格納したフラットな配列。

マクロ（フラグとアクセサ）

  • DECOMP_NO_COMPOSE, DECOMP_INLINE, DECOMP_COMPAT: 分解の性質を定義するビットフラグ。
  • DECOMPOSITION_SIZE(x): 分解後の文字数を取得するアクセサ。
  • DECOMPOSITION_IS_INLINE(x): 分解がインラインで保持されているか判定する。

制御フローとデータフロー

1. **検索**: 正規化対象のコードポイントを UnicodeDecompMain 配列に対してバイナリサーチ（または同等の検索ロジック）で特定します。 2. **分解**:

  • DECOMP_INLINE フラグが立っている場合、dec_index から直接分解後のコードポイントを取得します。
  • それ以外の場合、dec_index をオフセットとして UnicodeDecomp_codepoints 配列から分解後のシーケンスを読み取ります。

    • 3. **合成**: DECOMP_NO_COMPOSE フラグを確認し、合成処理を行うべきか判断します。

直接依存

本ファイルはスタンドアロンのデータ定義であり、他のヘッダーファイルへの依存はありません。

読者向けメモ

  • **自動生成**: このファイルは手動で編集しないでください。変更が必要な場合は、生成元の Perl スクリプトを修正してください。
  • **メモリレイアウト**: static const として定義されているため、プログラムのデータセグメントに配置されます。実行時のメモリ割り当ては発生しません。
  • **インクルードガード**: このファイルには意図的に #ifndef ガードがありません。これは、複数の場所でテーブルをインクルードして静的配列を定義する特殊な利用パターンを想定しているためです。
  • **網羅性**: このテーブルは Unicode 標準の正規化に必要な情報を網羅していますが、特定の文字範囲については最適化のためにインライン分解が多用されています。
