src/include/common/restricted_token.h

概要

Windows環境において、プロセスが制限された権限（Restricted Token）で実行されていることを保証するためのユーティリティヘッダーです。主にセキュリティ強化を目的として、管理者権限やPower Users権限を剥奪した状態で子プロセスを起動する機能を提供します。

モジュール内での役割

src/include/common モジュールの一部として、フロントエンドアプリケーション（FRONTEND定義時）が外部プロセスを安全に起動するための共通基盤を提供します。

責務

  • 現在のプロセスが制限されたトークンで実行されているかを確認する。
  • 制限されていない場合、環境変数 PG_RESTRICT_EXEC を用いて自身を再実行し、制限付きトークンを適用する。
  • Windows API を直接操作し、特定のSID（管理者、Power Users）を無効化した新しいプロセスを生成する。

重要なシンボル

  • get_restricted_token(void): プロセス起動時に呼び出すメインのエントリポイント。プラットフォームがWindows以外の場合は何もしません。
  • CreateRestrictedProcess(char *cmd, PROCESS_INFORMATION *processInfo): Windows専用の内部関数。制限付きトークンを作成し、指定されたコマンドをその権限で実行します。

制御フローとデータフロー

1. get_restricted_token() が呼び出されると、環境変数 PG_RESTRICT_EXEC をチェックします。 2. 未設定であれば、CreateRestrictedProcess() を呼び出し、現在のプロセスをサスペンド状態で再起動します。 3. CreateRestrictedProcess() は、現在のプロセスのトークンを取得し、CreateRestrictedToken API を用いて管理者権限等を剥奪したトークンを生成します。 4. 生成されたトークンを使用して CreateProcessAsUser を実行し、子プロセスを起動します。

直接依存

  • 本ヘッダーファイル自体は他のヘッダーへの依存を持ちません。
  • 実装ファイル（src/common/restricted_token.c）は postgres_fe.h および common/logging.h に依存しています。

関連コード

  • src/common/restricted_token.c: 上記関数の実装が含まれています。WIN32 マクロによりプラットフォーム固有の処理が分離されています。
  • src/include/common/logging.h: エラー発生時のログ出力に使用されます。

読者向けメモ

  • **プラットフォーム制限**: この機能は Windows 環境でのみ有効です。他のプラットフォームでは get_restricted_token() は空の関数として動作します。
  • **フロントエンド専用**: 実装ファイル内のガードにより、バックエンドコード（サーバープロセス）でのコンパイルは明示的に禁止されています。
  • **再帰的実行**: get_restricted_token() は自身を再実行する性質があるため、呼び出しタイミングには注意が必要です。通常はアプリケーションの初期化フェーズで一度だけ呼び出されます。
