src/include/common/int.h

概要

src/include/common/int.h は、PostgreSQL における安全な整数演算および比較のためのヘッダーファイルです。コンパイラ依存の未定義動作を避け、オーバーフローを明示的に検出するインライン関数群を提供します。

モジュール内での役割

src/include/common ディレクトリに配置されており、バックエンドおよびフロントエンドの両方から利用可能な共通ユーティリティとして機能します。整数演算の安全性と移植性を担保する低レイヤーの基盤です。

責務

  • **オーバーフロー検出**: 加算、減算、乗算、否定演算において、結果が型の範囲を超える場合に true を返す関数群の提供。
  • **安全な絶対値計算**: 符号付き整数から符号なし整数への変換を伴う、オーバーフローを回避した絶対値計算。
  • **比較ルーチン**: qsort 等の比較関数で利用可能な、オーバーフローリスクのない整数比較関数の提供。

重要なシンボル

  • pg_add_*_overflow, pg_sub_*_overflow, pg_mul_*_overflow: 各種整数型（s16, s32, s64, u16, u32, u64, size_t）に対するオーバーフローチェック付き演算。
  • pg_neg_*_overflow: 符号反転時のオーバーフローチェック（例：INT_MIN の反転）。
  • pg_abs_*: 符号付き整数から符号なし整数への安全な絶対値変換。
  • pg_cmp_*: 比較関数用ルーチン（a > b と a < b の差分を返すことで、オーバーフローを回避しつつ比較結果を算出）。

制御フローとデータフロー

  • **演算**: 可能であれば __builtin_add_overflow 等のコンパイラ組み込み関数を使用し、利用できない場合は型を拡張（例：int16 を int32 にキャスト）して計算し、範囲チェックを行います。
  • **結果の格納**: 演算が成功した場合は *result に値を書き込み false を返し、オーバーフローした場合は *result にダミー値（0x5EED）を書き込み true を返します。
  • **比較**: 32bit 以上の型では (a > b) - (a < b) というイディオムを使用し、減算によるオーバーフローを回避しています。

直接依存

  • 本ファイルはスタンドアロンで設計されており、直接的な外部ヘッダー依存はありません。
  • 標準的な整数型（int16, int32, int64, uint16, uint32, uint64）および bool 型の定義に依存します。

関連コード

  • src/backend/utils/adt/int.c: 本ヘッダーで定義された関数を利用し、SQL レベルの整数型演算や変換を実装している主要な対となるソースファイルです。

読者向けメモ

  • **移植性**: 本ファイルは -fwrapv などのコンパイラフラグに依存せず、C言語標準に準拠した安全なコードを記述することを目的としています。
  • **パフォーマンス**: 64bit 演算では HAVE_INT128 が定義されている環境で最適化されます。
  • **比較関数の注意**: pg_cmp_* 関数は qsort 用に設計されていますが、インライン展開される場合、単純な if 文による比較の方が高速なコードを生成するコンパイラもあります。必要に応じてプロファイルを確認してください。
