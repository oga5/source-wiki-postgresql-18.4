src/include/common/unicode_category.h

概要

Unicode 文字の「一般カテゴリ（General Category）」および各種プロパティを判定するためのインターフェースを定義するヘッダーファイルです。フロントエンド・バックエンド双方から利用可能です。

モジュール内での役割

src/include/common 内の Unicode 関連機能の一部であり、文字の分類や属性判定の基盤を提供します。文字コード（pg_wchar）を引数に取り、Unicode 標準に基づいた分類結果を返します。

責務

  • Unicode 一般カテゴリの列挙型定義（pg_unicode_category）。
  • 文字コードに対するカテゴリ取得関数の宣言。
  • 文字のプロパティ（Alphabetic, Lowercase, Uppercase 等）判定関数の宣言。
  • POSIX 互換性を含む、標準的な文字種判定関数（isalpha, isdigit 等）の Unicode 版の提供。

重要なシンボル

  • pg_unicode_category: Unicode 標準の一般カテゴリ値を保持する列挙型。ICU の UCharCategory と値を一致させています。
  • unicode_category(pg_wchar code): 指定されたコードポイントのカテゴリを返すメイン関数。
  • pg_u_prop_* / pg_u_is*: 文字の属性や種別を判定する一連の述語関数群。

制御フローとデータフロー

本ファイルはインターフェース定義のみを行います。実際の判定ロジックは src/common/unicode_category.c に実装されており、以下のフローで動作します。 1. unicode_category() が呼び出されると、まず ASCII 範囲（0x80 未満）を高速なルックアップテーブルで判定します。 2. それ以外の範囲については、unicode_category_table.h に定義されたデータ構造に対してバイナリサーチを行い、カテゴリを特定します。

直接依存

  • mb/pg_wchar.h: PostgreSQL の文字コード表現である pg_wchar 型の定義に依存しています。

関連コード

  • src/common/unicode_category.c: 実装本体。
  • src/include/common/unicode_category_table.h: 判定に使用される静的なデータテーブル。
  • src/include/common/unicode_case.h: 大文字・小文字変換に関連する機能。

読者向けメモ

  • 本ヘッダーで定義されるカテゴリ値は Unicode 安定性ポリシーにより固定されており、将来的に変更されることはありません。
  • pg_u_is* 系の関数には bool posix 引数を取るものがあり、これは POSIX 規格の厳密な定義に従うか、Unicode の広範な定義に従うかを制御します。
  • パフォーマンスが重要な箇所では、unicode_category.c で定義されている PG_U_*_MASK マクロを使用して、複数のカテゴリをビット演算で効率的に判定することが推奨されます。
