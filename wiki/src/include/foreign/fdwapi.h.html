src/include/foreign/fdwapi.h

概要

PostgreSQL の外部データラッパー（FDW: Foreign Data Wrapper）が実装すべきインターフェースを定義するヘッダーファイルです。外部テーブルに対するクエリの計画、実行、更新、および分析（ANALYZE）に必要なコールバック関数のシグネチャと、それらを保持する FdwRoutine 構造体を定義しています。

モジュール内での役割

src/include/foreign モジュールにおける中核的な定義ファイルです。FDW を開発する際に、PostgreSQL エンジンが呼び出すべき関数群の契約（API）を規定します。

責務

  • FDW のライフサイクル（計画作成から実行終了まで）を制御する関数ポインタの型定義。
  • 外部テーブルの更新、並列処理、非同期実行、および EXPLAIN サポートのための拡張 API の提供。
  • FDW ハンドラがエンジンに提供すべき FdwRoutine 構造体の定義。

重要なシンボル

  • FdwRoutine: FDW が実装する全コールバック関数を保持する構造体。ハンドラ関数はこの構造体を返す必要があります。
  • GetForeignRelSize / GetForeignPaths / GetForeignPlan: クエリ計画作成フェーズで呼び出される必須の API。
  • IterateForeignScan: 実行フェーズで外部テーブルからタプルを取得するための中心的な関数。
  • ExecForeignInsert / ExecForeignUpdate / ExecForeignDelete: 外部テーブルに対するデータ操作を行うための関数群。

制御フローとデータフロー

1. **計画フェーズ**: エンジンは GetFdwRoutine を通じて FdwRoutine を取得し、GetForeignRelSize や GetForeignPaths を呼び出してコスト見積もりとパス生成を行います。 2. **実行フェーズ**: BeginForeignScan で初期化され、IterateForeignScan が繰り返し呼び出されてタプルが TupleTableSlot に格納されます。最後に EndForeignScan でリソースが解放されます。 3. **更新フェーズ**: ModifyTable ノード経由で ExecForeignInsert 等が呼び出され、外部ソースへの書き込みが行われます。

直接依存

  • access/parallel.h: 並列クエリ実行のためのサポート。
  • nodes/execnodes.h: 実行ノード（ForeignScanState 等）の定義。
  • nodes/pathnodes.h: 計画ノード（ForeignPath 等）の定義。

関連コード

  • src/include/foreign/foreign.h: FDW のルーチンを取得するためのヘルパー関数群が定義されています。
  • src/backend/foreign/foreign.c: GetFdwRoutine などの実装が含まれており、本ヘッダーで定義された API を利用する側のロジックを確認できます。

読者向けメモ

  • FdwRoutine を初期化する際は、必ず makeNode(FdwRoutine) を使用してください。これにより、将来的に API が拡張された際にも未定義フィールドが NULL で初期化されることが保証されます。
  • 多くの関数ポインタはオプションです。実装不要な機能については、FdwRoutine 内で NULL を設定してください。
  • 非同期実行や並列処理など、高度な機能については fdwhandler.sgml（PostgreSQL ドキュメント）を参照することを強く推奨します。
