src/include/pgstat.h

概要

PostgreSQL の累積統計システム（Cumulative Statistics System）のインターフェースを定義する中心的なヘッダーファイルです。バックエンドが実行する各種操作（I/O、トランザクション、関数呼び出し、テーブルアクセス等）の統計情報を収集・集計し、共有メモリまたはディスクへ永続化するためのデータ構造と関数群を宣言します。

モジュール内での役割

src/backend/utils/activity/pgstat.c を中心とする統計サブシステムのパブリック API です。バックエンドプロセスが統計情報を更新するためのマクロや、統計情報を取得・管理するための関数群を公開し、システム全体の監視・分析機能の基盤を提供します。

責務

  • **統計データ構造の定義**: データベース、テーブル、関数、WAL、バックエンドプロセス、SLRU 等の統計用構造体の定義。
  • **バックエンドローカルな集計**: トランザクション中の統計更新を一時的に保持する構造体（PgStat_TableStatus 等）の管理。
  • **統計更新インターフェース**: 統計情報をインクリメント・更新するためのマクロおよび関数の提供。
  • **サブシステム間連携**: 統計情報の永続化、リセット、および外部モジュールによるカスタム統計の登録支援。

重要なシンボル

統計データ構造

  • PgStat_Counter: 統計カウンタの基本型（int64）。
  • PgStat_TableCounts / PgStat_TableStatus: テーブル単位のイベントカウンタとバックエンド内での状態管理。
  • PgStat_FunctionCounts: 関数呼び出し回数および実行時間の計測用。
  • PgStat_BktypeIO / PgStat_PendingIO: I/O 操作（読み込み、書き込み、同期等）の統計管理。
  • PgStat_StatDBEntry 等: 共有メモリ上に保持される各オブジェクトの統計スナップショット。

状態管理・制御用

  • TrackFunctionsLevel: 関数統計の追跡レベル（OFF, PL, ALL）。
  • PgStat_FetchConsistency: 統計取得時の整合性レベル（キャッシュ、スナップショット等）。
  • SessionEndType: セッション終了理由の追跡。

更新用マクロ

  • pgstat_count_heap_scan, pgstat_count_buffer_hit 等: テーブル操作をインクリメントするインラインマクロ。
  • pgstat_should_count_relation: 統計追跡の有効性を判定し、必要に応じて初期化を行うヘルパー。

制御フローとデータフロー

1. **収集**: バックエンドプロセスが pgstat_count_* マクロや関数を呼び出し、ローカルメモリ（PgStat_TableStatus 等）にイベントを蓄積します。 2. **フラッシュ**: pgstat_report_stat() が呼び出されると、ローカルの統計情報が共有メモリ上の統計エントリへ反映されます。 3. **永続化**: チェックポインタプロセスがシャットダウン時に共有メモリ上の統計をディスク（pg_stat/pgstat.stat）へ書き出します。 4. **取得**: 統計ビュー（pg_stat_user_tables 等）がクエリされると、pgstat_fetch_stat_* 関数群が共有メモリまたはスナップショットからデータを読み取ります。

読者向けメモ

  • **メモリ所有権**: 統計情報は「バックエンドローカルな一時保持」と「共有メモリ上の集計値」の二層構造です。ローカルな更新は pgstat_report_stat() で共有メモリへ同期されるまで他プロセスからは見えません。
  • **パフォーマンス**: 頻繁に更新されるカウンタはインラインマクロで実装されており、オーバーヘッドを最小化しています。
  • **制約**: PgStat_TableCounts 等の構造体は、pg_memory_is_all_zeros() を使用して更新の有無を判定するため、カウンタ以外のフィールドを混在させる際は注意が必要です。
  • **省略事項**: 本ヘッダーは統計システムの「定義」に特化しており、共有メモリのハッシュテーブル操作やロック管理の詳細は pgstat.c および pgstat_shmem.c に委譲されています。
