src/include/utils/plancache.h

概要

plancache.h は、PostgreSQL におけるクエリ実行計画のキャッシュ管理（Plan Cache）のインターフェースを定義するヘッダーファイルです。SQL クエリの解析・書き換え・計画作成のコストを削減するため、一度作成した計画を再利用する仕組みを提供します。

モジュール内での役割

src/backend/utils/cache/plancache.c と対をなし、バックエンド全体で共有される計画キャッシュのデータ構造と API を公開します。クエリのライフサイクル管理において、パーサーとエグゼキューターの間に位置する重要なコンポーネントです。

責務

  • **計画のライフサイクル管理**: CachedPlanSource を通じて、クエリの解析結果や計画を保持・管理する。
  • **キャッシュ無効化**: DDL 等によるスキーマ変更を検知し、依存する計画を無効化する。
  • **計画の選択**: 汎用計画（Generic Plan）とカスタム計画（Custom Plan）のどちらを使用すべきかの判断を支援する。
  • **式キャッシュ**: 単純なスカラー式の計画を低コストでキャッシュする機能の提供。

重要なシンボル

  • CachedPlanSource: クエリのソース（テキストや解析木）と、それから生成された計画を管理する構造体。
  • CachedPlan: 実行可能な計画そのもの。参照カウントにより管理される。
  • CachedExpression: スカラー式をキャッシュするための軽量な構造体。
  • GetCachedPlan(): キャッシュされた計画を取得する主要な関数。必要に応じて再計画（Replan）を行う。
  • SaveCachedPlan(): CachedPlanSource を永続化し、キャッシュ無効化イベントの監視対象とする。

制御フローとデータフロー

1. **作成**: CreateCachedPlan 等で CachedPlanSource を生成。 2. **保存**: SaveCachedPlan を呼び出し、バックエンドのグローバルリストに登録。 3. **実行**: GetCachedPlan を呼び出す。内部で is_valid フラグを確認し、無効であれば再解析・再計画を実行。 4. **無効化**: sinval（共有キャッシュ無効化）イベントが発生すると、依存関係に基づいて is_valid が false に設定される。 5. **解放**: ReleaseCachedPlan により参照カウントを減らし、ゼロになった時点でメモリを解放。

読者向けメモ

  • **メモリ管理**: CachedPlan は参照カウント方式です。ResourceOwner を介して適切に解放しないとメモリリークの原因となります。
  • **oneshot クエリ**: is_oneshot フラグが立つクエリは、キャッシュ無効化の対象外であり、呼び出し元のメモリコンテキストに依存します。
  • **マジックナンバー**: 構造体の整合性チェック用に CACHEDPLANSOURCE_MAGIC 等の定数が定義されています。デバッグ時に構造体の破損を検知するために使用されます。
