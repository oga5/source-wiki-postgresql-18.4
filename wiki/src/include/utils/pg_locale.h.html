src/include/utils/pg_locale.h

概要

PostgreSQLにおけるロケールおよび照合順序（Collation）処理の抽象化レイヤーを定義するヘッダーファイルです。OS標準のlibcロケール、ICUライブラリ、およびPostgreSQL組み込みのロケール処理を統一的に扱うためのインターフェースを提供します。

モジュール内での役割

src/include/utils モジュールの一部として、データベースの照合順序や文字変換、数値・時間フォーマットのロケール依存性を管理します。pg_locale.c と対になり、照合順序のプロバイダー（libc, ICU, builtin）を隠蔽する役割を担います。

責務

  • ロケール情報のキャッシュと管理（GUC設定に基づく）。
  • 照合順序プロバイダー（libc/ICU/builtin）の抽象化。
  • 文字列の大小文字変換、照合（coll）、変換（xfrm）の共通インターフェース提供。
  • データベース作成時に固定される LC_COLLATE / LC_CTYPE と、実行時に変更可能な LC_MESSAGES 等の分離管理。

重要なシンボル

  • pg_locale_t: ロケール情報を保持する構造体へのポインタ。プロバイダーの種類や照合ルールをカプセル化します。
  • collate_methods: 照合順序の挙動を定義する関数ポインタのテーブル。
  • pg_newlocale_from_collation(Oid collid): 指定されたOIDからロケールオブジェクトを生成する主要なファクトリ関数。
  • pg_strlower, pg_strupper, pg_strcoll: ロケールを考慮した文字列操作のラッパー関数群。

追加の索引済みシンボル

  • UCollator
  • pg_locale_struct

制御フローとデータフロー

1. **初期化**: init_database_collation() によりデータベースのデフォルトロケールがロードされます。 2. **ロケール生成**: pg_newlocale_from_collation() が呼び出されると、pg_locale_t 構造体が構築され、プロバイダー（libc/ICU/builtin）に応じたメソッドテーブルが割り当てられます。 3. **操作実行**: 文字列操作関数（例: pg_strcoll）は、渡された pg_locale_t のメソッドテーブルを参照し、適切な実装（strncoll_icu や strncoll_libc など）へディスパッチします。 4. **キャッシュ**: GUC（locale_monetary 等）が変更されると、関連するロケールデータが再キャッシュされます。

直接依存

  • unicode/ucol.h: ICUライブラリを使用する場合の照合順序定義。
  • pg_wchar.h: PostgreSQL内部の文字エンコーディング処理。

関連コード

  • src/backend/utils/adt/pg_locale.c: 本ヘッダーで宣言された関数の実装。
  • src/backend/utils/adt/pg_locale_icu.c: ICU固有の照合・変換ロジック。
  • src/backend/utils/adt/pg_locale_libc.c: libc固有のロケール処理。
  • src/catalog/pg_collation.h: 照合順序のカタログ定義。

読者向けメモ

  • pg_locale_t は単なる構造体ではなく、実行時にプロバイダーを切り替えるためのポリモーフィズムを実現しています。
  • collate_is_c や ctype_is_c フラグは、パフォーマンス最適化のために多用されます。これらが true の場合、重いロケール処理をスキップして高速なパスを通るよう設計されています。
  • ICUを使用する場合、U_SHOW_CPLUSPLUS_API を無効化してC APIのみを露出させることで、C++ヘッダーとの競合を回避しています。
