src/include/utils/jsonfuncs.h

概要

jsonfuncs.h は、PostgreSQL における JSON および JSONB データの処理・変換・解析を行うための共通インターフェースを定義するヘッダーファイルです。主に JSON データの反復処理、型変換、およびエラーハンドリングの基盤を提供します。

モジュール内での役割

src/include/utils モジュールの一部として、JSON 関連のデータ型（json, jsonb）を操作するバックエンド関数群（jsonfuncs.c）の公開 API を定義しています。SQL レベルの JSON 関数と、内部的なデータ構造の橋渡しを担います。

責務

  • **JSON 解析の抽象化**: JsonLexContext を用いた解析処理と、エラー発生時の通知メカニズム（pg_parse_json_or_errsave 等）の提供。
  • **反復・変換処理**: JSON/JSONB 内の特定要素（キー、文字列、数値など）を走査し、コールバック関数を適用する仕組みの提供。
  • **型変換**: PostgreSQL のデータ型を JSON/JSONB 形式へ変換するためのカテゴリ分類と変換関数の提供。

重要なシンボル

  • JsonToIndex: 反復処理対象を指定するビットフラグ（キー、文字列、数値、真偽値）。
  • JsonIterateStringValuesAction: 反復処理時に各要素に対して実行されるコールバック関数の型定義。
  • JsonTypeCategory: datum_to_json 等で使用される、PostgreSQL 型から JSON への変換ルールを決定するための列挙型。
  • pg_parse_json_or_ereport: JSON 解析を行い、失敗時にエラーを投げるための便利なマクロ。

追加の索引済みシンボル

  • JsonTransformStringValuesAction

制御フローとデータフロー

1. **解析**: makeJsonLexContext で入力テキストから解析コンテキストを生成し、pg_parse_json_or_errsave を通じてセマンティックアクション（JsonSemAction）を適用します。 2. **反復**: iterate_json_values や iterate_jsonb_values を呼び出し、指定した JsonToIndex フラグに基づいて要素を抽出、コールバック関数へ渡します。 3. **変換**: datum_to_json 等で PostgreSQL の Datum を JSON 形式に変換する際、json_categorize_type で型を判定し、適切な出力関数（outfuncoid）を特定して変換を実行します。

直接依存

  • common/jsonapi.h: JSON 解析の低レベルな API と構造体。
  • nodes/nodes.h: エラーハンドリング用ノード構造体。
  • utils/jsonb.h: JSONB 型の定義と操作関数。

関連コード

  • src/backend/utils/adt/jsonfuncs.c: 本ヘッダーで宣言された関数の実装。
  • src/include/utils/json.h: JSON 型自体の定義。
  • src/include/utils/jsonpath.h: JSONPath 関連の処理。

読者向けメモ

  • 本ファイルは JSON 処理の「インターフェース」です。具体的な実装や、内部的な状態管理構造体（OkeysState や GetState など）は jsonfuncs.c に隠蔽されています。
  • escontext を引数に取る関数は、エラー時に例外を投げずにエラー情報を保存する新しいエラーハンドリング方式に対応しています。既存の ereport を使用するコードと混在しているため、呼び出し時には注意が必要です。
