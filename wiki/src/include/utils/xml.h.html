src/include/utils/xml.h

概要

PostgreSQLにおける xml データ型のサポートを定義するヘッダーファイルです。XMLのパース、生成、およびSQL型との相互変換に必要な型定義と関数インターフェースを提供します。

モジュール内での役割

src/include/utils モジュールの一部として、XML関連のデータ型定義と、バックエンドのXML処理機能（src/backend/utils/adt/xml.c）へのアクセスポイントを提供します。

責務

  • xmltype 型の定義と、PostgreSQLのDatum形式との変換マクロの提供。
  • XML処理の厳密さ（PgXmlStrictness）やバイナリエンコーディング（XmlBinaryType）の制御。
  • XML操作関数（xmlconcat, xmlelement, xmlparse 等）の宣言。
  • SQL識別子とXML名の相互変換ロジックの提供。
  • libxml2 依存の抽象化とエラーハンドリング用コンテキストの管理。

重要なシンボル

  • xmltype: XMLデータを保持する varlena 型のエイリアス。
  • PgXmlStrictness: XML処理時のエラー報告レベルを制御する列挙型。
  • DatumGetXmlP / PG_GETARG_XML_P: XML型をDatumから安全に取り出すためのインライン関数およびマクロ。
  • pg_xml_init / pg_xml_done: XML処理セッションの初期化と終了処理を行う関数。
  • XmlTableRoutine: XMLTABLE 演算子をサポートするためのテーブル関数ルーチン。

追加の索引済みシンボル

  • PgXmlErrorContext
  • XmlBinaryType
  • XmlPGetDatum
  • XmlStandaloneType

制御フローとデータフロー

  • **初期化**: XML処理を行う関数は、まず pg_xml_init を呼び出し、エラーハンドリング用の PgXmlErrorContext を取得します。
  • **データ変換**: SQLのデータ型からXMLへの変換は map_sql_value_to_xml_value を介して行われます。
  • **実行**: xmlparse や xmlelement などの関数が、入力された text や xmltype を処理し、結果として新しい xmltype を返します。
  • **終了**: 処理完了後、pg_xml_done を呼び出してリソースを解放し、エラー状態を確定させます。

直接依存

  • executor/tablefunc.h: XmlTableRoutine の定義に使用。
  • fmgr.h: PostgreSQLの関数マネージャインターフェース。
  • nodes/execnodes.h / nodes/primnodes.h: XML式（XmlExpr）のノード定義に使用。

関連コード

  • src/backend/utils/adt/xml.c: 本ヘッダーで宣言された関数の実装および libxml2 との統合ロジック。
  • src/include/utils/builtins.h: XML関連関数を含むPostgreSQLの組み込み関数群。

読者向けメモ

  • **libxml2依存**: 本ファイルは libxml2 がビルド時に有効化されているかに関わらずインクルード可能ですが、libxml2 が無効な環境では、多くの関数はエラーを返すか動作しません。
  • **メモリ管理**: xml.c 内のコメントにある通り、libxml2 のメモリ割り当てはPostgreSQLのメモリ管理（palloc）と競合する可能性があるため、取り扱いには注意が必要です。
  • **GUC変数**: xmlbinary および xmloption はグローバル変数として公開されており、実行時にXMLの挙動を制御します。
