src/include/utils/sortsupport.h

概要

sortsupport.h は、PostgreSQL における高速なソート処理を実現するためのフレームワークを定義しています。従来の B-tree 比較関数（BTORDER_PROC）を繰り返し呼び出す手法に対し、より低オーバーヘッドな比較メカニズムを提供します。

モジュール内での役割

src/include/utils モジュールの一部として、ソート処理の最適化層を担います。tuplesort.c などのソート実行エンジンが、データ型固有の最適化された比較ロジックを利用するためのインターフェースを提供します。

責務

  • **ソートの抽象化**: SortSupportData 構造体を通じて、比較関数や省略キー（Abbreviated Key）変換ルーチンを保持する。
  • **高速化の提供**: 省略キーを用いた比較や、CPU 命令レベルで最適化された比較関数の統合。
  • **互換性の維持**: 従来の比較関数を SortSupport API に適合させるための「Shim（シム）」機能の提供。
  • **NULL 処理の標準化**: 昇順/降順および NULL の配置（First/Last）を考慮した比較結果の正規化。

重要なシンボル

  • SortSupportData: ソートに必要な状態（比較関数、照合順序、省略キー変換器など）を保持する中心的な構造体。
  • ApplySortComparator: SortSupport を用いて、NULL 判定と順序（昇順/降順）を考慮した比較を行うインライン関数。
  • ApplySortAbbrevFullComparator: 省略キー比較が不確定な場合に呼び出される、信頼できる完全な比較関数。
  • BTSORTSUPPORT_PROC: B-tree オペレータクラスが提供すべき、ソートサポート関数を特定するためのプロシージャ ID。

追加の索引済みシンボル

  • ApplyInt32SortComparator
  • ApplySignedSortComparator
  • ApplyUnsignedSortComparator

制御フローとデータフロー

1. **初期化**: ソート実行前に PrepareSortSupport... 系関数を呼び出し、SortSupportData を構築します。この際、対象の型が BTSORTSUPPORT_PROC を提供していればそれを呼び出し、なければ shim を介して従来の比較関数を登録します。 2. **比較実行**: ソートエンジンは ApplySortComparator を呼び出します。 3. **省略キーの活用**: 省略キーが有効な場合、まず高速な比較を行い、結果が不確定（0）であれば ApplySortAbbrevFullComparator を介して完全な比較を行います。 4. **終了**: ソート完了後、ssup_cxt に割り当てられたメモリは適切に解放されます。

読者向けメモ

  • **省略キーの注意点**: 省略キーは「高速な近似」であり、等価判定には使用できません。必ず abbrev_full_comparator によるフォールバックが必要です。
  • **メモリ管理**: SortSupportData 内の ssup_extra に格納するデータは、必ず ssup_cxt コンテキスト内に割り当てる必要があります。
  • **インライン関数の利用**: ApplySortComparator 等はインライン化されており、ソートのループ内での呼び出しコストを最小限に抑える設計になっています。
