src/include/utils/guc_tables.h

概要

PostgreSQLの「Grand Unified Configuration (GUC)」システムで使用される、設定変数の定義構造とメタデータテーブルを規定するヘッダーファイルです。GUCシステムは、データベースの実行時パラメータを管理する中心的な仕組みです。

モジュール内での役割

src/include/utils モジュールにおいて、設定パラメータのスキーマ定義を担います。guc.h がGUCシステムのAPIを定義するのに対し、本ファイルは設定項目そのものを表現するデータ構造と、それらを保持するテーブルの型定義を提供します。

責務

  • 設定変数の型（bool, int, real, string, enum）に応じた構造体の定義。
  • 設定変数のグループ化（config_group）と表示用メタデータの管理。
  • トランザクションや関数呼び出しによる設定変更を追跡するためのスタック構造（GucStack）の定義。
  • 実行時に設定値を操作・参照するための共通インターフェース（config_generic）の提供。

重要なシンボル

  • struct config_generic: 全てのGUC変数の基底となる構造体。名前、コンテキスト、グループ、現在のソース、スタック情報などを保持します。
  • struct config_bool / int / real / string / enum: 各型固有の構造体。config_generic を継承し、デフォルト値やフック関数（check/assign/show）を保持します。
  • ConfigureNamesBool 等: 組み込みのGUC変数を格納するグローバルな配列（実体は guc_tables.c に存在）。
  • GucStack: SET LOCAL 等による一時的な設定変更を管理するためのスタック構造。

制御フローとデータフロー

本ファイルはデータ定義が主であり、直接的な制御フローは持ちません。 1. **初期化**: build_guc_variables() を通じて、ConfigureNamesXxx 配列が読み込まれ、システム全体で利用可能な設定リストが構築されます。 2. **変更管理**: 設定値が変更される際、GucStack を用いて現在の値が保存され、新しい値が適用されます。 3. **参照**: find_option() を使用して名前から config_generic を検索し、ShowGUCOption() 等で値を取得します。

直接依存

  • lib/ilist.h: dlist_node や slist_node を使用したリスト管理。
  • utils/guc.h: GUCシステムで共通利用される列挙型（GucSource, GucContext 等）やフック関数の型定義。

関連コード

  • src/backend/utils/misc/guc_tables.c: 本ヘッダーで定義された構造体を用いた、実際のGUC変数定義の配列が記述されています。
  • src/backend/utils/misc/guc.c: GUCシステムのロジック本体。本ファイルの構造体を利用して設定の適用や検証を行います。

読者向けメモ

  • 本ファイルは「設定の定義」に特化しています。ロジックを追う際は、まず config_generic 構造体のフィールド構成を理解すると、GUCシステムがどのように設定のソース（ファイル、環境変数、SQLコマンド等）を追跡しているかが把握しやすくなります。
  • config_generic の extra ポインタは、check_hook で生成された複雑な設定値を assign_hook に渡すための重要なブリッジです。
  • GUC_IS_IN_FILE などのフラグは、設定ファイル（postgresql.conf）の再読み込み処理において重要な役割を果たします。
