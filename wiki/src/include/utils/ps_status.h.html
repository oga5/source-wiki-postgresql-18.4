src/include/utils/ps_status.h

概要

ps_status.h は、PostgreSQL バックエンドプロセスが OS 上で表示するプロセス名（ps コマンド等で確認可能）を動的に更新するためのインターフェースを定義しています。

モジュール内での役割

src/include/utils モジュールの一部として、バックエンドの可観測性を高める役割を担います。OS 固有のプロセス名更新メカニズムを抽象化し、バックエンドの各コンポーネントが現在の処理状況を外部から監視できるようにします。

責務

  • プロセス名更新機能の有効/無効を制御する GUC 変数（update_process_title）の提供。
  • プロセス起動時の引数（argv）の保存と管理。
  • プロセス名（タイトル）の初期化、更新、およびサフィックス付与の API 提供。
  • プラットフォームごとの実装差異を隠蔽し、統一的なインターフェースをバックエンドに提供する。

重要なシンボル

  • update_process_title: プロセス名の更新を行うか制御するブール値。
  • save_ps_display_args(): プロセス起動時に呼び出し、argv 領域を保護・管理する。
  • init_ps_display(): プロセス名のベース部分を設定する。
  • set_ps_display(): 現在のバックエンドの活動内容をプロセス名に反映させる。
  • set_ps_display_suffix(): プロセス名の末尾に特定の情報を追加する。

追加の索引済みシンボル

  • set_ps_display

制御フローとデータフロー

1. **初期化**: プロセス起動時に save_ps_display_args を呼び出し、OS の引数領域を確保します。 2. **設定**: init_ps_display でプロセス名の固定部分（例: "postgres: worker process"）を定義します。 3. **更新**: 各バックエンド処理の節目で set_ps_display を呼び出します。この関数は内部で set_ps_display_with_len を呼び出し、プラットフォームに応じた方法（setproctitle の利用や argv 領域の直接書き換え）で OS にプロセス名を通知します。 4. **サフィックス**: 必要に応じて set_ps_display_suffix を使い、動的なステータス（例: 実行中のクエリ情報）を追記します。

直接依存

本ヘッダーファイル自体は他のヘッダーへの直接的な依存を持ちませんが、実装ファイルである ps_status.c は utils/guc.h や miscadmin.h を利用してシステム状態を管理します。

関連コード

  • src/backend/utils/misc/ps_status.c: 本ヘッダーで宣言された関数の実体。プラットフォームごとの #ifdef 分岐による複雑な実装が含まれます。
  • src/include/miscadmin.h: バックエンドのグローバルな管理情報が含まれており、ps_status と密接に関連します。

読者向けメモ

  • **プラットフォーム依存**: プロセス名の更新は OS によって実装が大きく異なります（argv 領域の書き換え、setproctitle API の利用など）。移植性を考慮する場合、ps_status.c の実装詳細に注意してください。
  • **パフォーマンス**: Windows 環境ではプロセス名の更新によるオーバーヘッドが大きいため、デフォルトで無効化されています。
  • **利用上の注意**: set_ps_display は頻繁に呼び出される可能性があるため、インライン化されており、文字列長をコンパイル時に評価できるよう工夫されています。
