src/include/utils/xid8.h

概要

xid8.h は、PostgreSQL における 64 ビットトランザクション ID (FullTransactionId) を扱うための抽象データ型 (ADT) インターフェースを定義するヘッダーファイルです。

モジュール内での役割

src/include/utils ディレクトリ内に位置し、トランザクション管理サブシステム (access/transam) と、SQL インターフェース層（関数呼び出し規約）との間のブリッジとして機能します。

責務

  • FullTransactionId 型と PostgreSQL の Datum 型との相互変換機能の提供。
  • SQL 関数から FullTransactionId を受け取り、また返すための標準的なマクロの提供。

重要なシンボル

  • DatumGetFullTransactionId: Datum を FullTransactionId に変換するインライン関数。
  • FullTransactionIdGetDatum: FullTransactionId を Datum に変換するインライン関数。
  • PG_GETARG_FULLTRANSACTIONID(X): SQL 関数内で引数から FullTransactionId を取得するマクロ。
  • PG_RETURN_FULLTRANSACTIONID(X): SQL 関数から FullTransactionId を戻り値として返すマクロ。

制御フローとデータフロー

本ファイルは主に SQL 関数（C 言語で実装されたもの）の引数処理および戻り値処理のパイプラインに組み込まれます。 1. **入力**: PG_GETARG_DATUM で取得した Datum を DatumGetFullTransactionId に渡し、内部表現である FullTransactionId へ変換します。 2. **出力**: 処理結果の FullTransactionId を FullTransactionIdGetDatum で Datum に変換し、PG_RETURN_DATUM 相当の処理を経て SQL エンジンへ返却します。

直接依存

  • access/transam.h: FullTransactionId 型の定義および FullTransactionIdFromU64 等の変換関数を提供します。

関連コード

  • src/include/access/transam.h: FullTransactionId の構造や、32 ビット XID との関連性、トランザクション ID の生成ロジックを理解するために必須です。

読者向けメモ

  • 本ファイルは非常に軽量なラッパーです。複雑なロジックは含まれていません。
  • FullTransactionId は 64 ビットのトランザクション ID を指します。従来の 32 ビット XID と混同しないよう注意してください。
  • SQL 関数を実装する際、FullTransactionId を引数や戻り値として扱う場合は、必ずこれらのマクロを使用して型安全性を確保してください。
