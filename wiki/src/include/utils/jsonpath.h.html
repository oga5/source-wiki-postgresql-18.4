src/include/utils/jsonpath.h

概要

jsonpath.h は、PostgreSQL における SQL/JSON Path 言語のデータ型定義および操作用インターフェースを提供します。JSON Path 式をバイナリ形式で保持するための構造体と、その解析・実行を支援する関数群を定義しています。

モジュール内での役割

src/include/utils モジュールの一部として、JSONB 関連の機能と連携し、JSONB データに対するクエリ実行エンジンの中核を担います。jsonpath.c と対になり、SQL レベルの jsonpath 型の入出力および評価ロジックをサポートします。

責務

  • **データ構造の定義**: JsonPath（オンディスク形式）および JsonPathItem（実行時操作用）の定義。
  • **型変換とアクセス**: Datum から JsonPath への変換マクロおよびインライン関数の提供。
  • **解析サポート**: 文字列から JsonPath 構造への変換（parsejsonpath）およびその中間表現（JsonPathParseItem）の定義。
  • **実行インターフェース**: JsonPathExists や JsonPathQuery など、JSONB データに対するパス評価関数の宣言。

重要なシンボル

  • JsonPath: 可変長データ型（varlena）として定義された、バイナリ形式の JSON Path。
  • JsonPathItemType: パス内の各要素（演算子、メソッド、リテラル等）を識別する列挙型。
  • JsonPathItem: 実行時にバイナリ形式のパスを走査するための抽象化構造体。
  • jspInit / jspGetNext: JsonPathItem を初期化し、パスのリンクを辿るための基本関数。
  • JsonPathQuery: 指定された JSONB データに対してパスを適用し、結果を返すメイン関数。

制御フローとデータフロー

1. **入力**: SQL 文字列が parsejsonpath を通じて JsonPathParseResult（中間表現）に変換されます。 2. **構築**: 中間表現がバイナリ形式の JsonPath にシリアライズされます。 3. **実行**: JsonPathItem を用いてバイナリ形式を走査します。各ノードは nextPos を持ち、リンクを辿ることでパスを評価します。 4. **評価**: jspGetArg 等のアクセサを使用してオペランドを取得し、演算子（jpiAdd, jpiEqual 等）を適用して結果を算出します。

読者向けメモ

  • **バイナリ形式の不変性**: JsonPathItemType の順序はオンディスク形式に直結するため、後方互換性のために変更・挿入は末尾に行う必要があります。
  • **走査の特性**: パスの走査は、演算子ではなく「左オペランド」を起点とする設計になっています。これにより、JSONB 構造を効率的に降下（descend）できるよう工夫されています。
  • **デバッグ**: jspOperationName を使用すると、列挙型から人間が読める演算子名を取得できるため、デバッグ時に有用です。
