src/include/utils/relptr.h

概要

relptr.h は、PostgreSQL における「相対ポインタ（Relative Pointer）」を実装するためのマクロ群を提供します。これは、絶対アドレスではなく、特定のベースアドレスからのオフセットとしてメモリ位置を保持するための仕組みです。

モジュール内での役割

src/include/utils 内において、共有メモリや動的にマッピングされる領域を扱うための低レイヤーなユーティリティとして機能します。特に、プロセス間でアドレス空間が異なる可能性がある共有メモリセグメント内のデータ構造を安全に参照するために使用されます。

責務

  • **型安全な相対参照の提供**: マクロを通じて、特定の型へのポインタをオフセットとして保持・解決する手段を提供します。
  • **共有メモリ対応**: ベースアドレスからの相対位置を計算することで、異なるプロセス間でのポインタの有効性を保証します。
  • **NULL 許容**: 0 を NULL として扱う規約を強制し、安全なポインタ操作を支援します。

重要なシンボル

  • relptr(type): 相対ポインタを定義するための共用体（union）を生成します。
  • relptr_declare(type, name): pgindent に対応した形式で相対ポインタ型を宣言します。
  • relptr_access(base, rp): ベースアドレスと相対ポインタから、実際の型付きポインタを解決します。
  • relptr_store(base, rp, val): 指定した値を相対オフセットに変換して格納します。

追加の索引済みシンボル

  • relptr_store_eval

制御フローとデータフロー

1. **格納**: relptr_store が呼び出されると、relptr_store_eval が実行されます。ここで val - base + 1 が計算され、1 を加算することで 0 を NULL 状態として予約します。 2. **解決**: relptr_access は、保持されたオフセットに base を加え、-1 することで元の絶対アドレスを復元します。 3. **検証**: コンパイル環境が許せば __builtin_types_compatible_p を使用し、ベースアドレスとポインタ型の整合性をチェックします。

直接依存

  • なし（標準的な C のプリプロセッサマクロとインライン関数のみで構成されています）。

関連コード

  • src/include/utils/dsa.h: 動的共有メモリ領域の管理において、この相対ポインタの仕組みが活用される主要なコンシューマです。

読者向けメモ

  • **オフセットの規約**: この実装では 0 が NULL を表すため、実際のオフセットは (アドレス - ベース) + 1 として格納されます。このため、relptr_offset() を使用して生のオフセットを取得する際は注意が必要です。
  • **型安全性**: relptr は共用体を利用したハックであり、コンパイラの機能（__typeof__ 等）を利用して可能な限り型チェックを行っています。デバッグ時には Assert による型検証が有効に機能します。
  • **使用上の注意**: base 引数には必ず char * 型のベースアドレスを渡す必要があります（マクロ内で AssertVariableIsOfTypeMacro により検証されます）。
