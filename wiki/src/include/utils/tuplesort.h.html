src/include/utils/tuplesort.h

概要

tuplesort.h は、PostgreSQL における汎用的なタプルソート機能のインターフェースを定義するヘッダーファイルです。メモリ内での高速な qsort から、メモリ制限を超えた場合の外部ソート（一時ファイルを用いた k-way マージ）までを透過的に扱います。

モジュール内での役割

src/include/utils モジュールにおいて、クエリ実行エンジンやインデックス構築プロセスが利用するソート処理の抽象化層です。具体的なソートロジックは tuplesort.c および tuplesortvariants.c に実装されており、本ファイルはそれらを利用するための公開 API を提供します。

責務

  • **ソートの抽象化**: ヒープタプル、インデックス用タプル、単一 Datum など、多様なデータ型に対するソートインターフェースの統一。
  • **メモリ管理**: work_mem に基づくメモリ使用量の制御と、外部ソートへの切り替え判断。
  • **並列ソートの調整**: 複数プロセス間でのソート状態の共有と、最終的なマージ処理の管理。

重要なシンボル

  • Tuplesortstate: ソート操作の現在の状態を保持する不透明な構造体。
  • SortTuple: ソート対象のタプルと、比較用のキー（Datum）を保持する最小単位。
  • tuplesort_begin_xxx: 各種データ型に対応したソート開始関数群（_heap, _index_btree, _datum 等）。
  • tuplesort_performsort: ソートを実行する（メモリ内ソートまたは外部ソートの開始）。
  • SortCoordinate: 並列ソート時にリーダーとワーカー間で共有される状態情報。

制御フローとデータフロー

1. **初期化**: tuplesort_begin_xxx を呼び出し、ソート対象に応じた Tuplesortstate を作成。 2. **データ投入**: tuplesort_puttuple_xxx を繰り返し呼び出し、データをメモリに蓄積。 3. **ソート実行**: tuplesort_performsort を呼び出す。メモリ内に収まる場合は qsort で完結し、超える場合は一時ファイルへの書き出しとマージ処理へ移行。 4. **データ取得**: tuplesort_gettuple_xxx を用いてソート済みデータを順次取得。 5. **終了**: tuplesort_end でメモリおよび一時ファイルを解放。

読者向けメモ

  • **並列ソート**: 並列ソートを行う場合は、リーダーとワーカーで SortCoordinate を適切に初期化し、共有メモリを介して連携する必要があります。
  • **メモリ消費**: work_mem はソート操作ごとのメモリ上限です。並列ソート時はワーカーごとにこの制限が適用される点に注意してください。
  • **抽象化**: TuplesortPublic 構造体内の関数ポインタにより、ソート対象の型を意識せずに共通のソートロジックを呼び出せる設計になっています。
