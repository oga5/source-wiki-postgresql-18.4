src/include/utils/combocid.h

概要

combocid.h は、PostgreSQL のタプルヘッダーにおける「Combo Command ID」を管理するためのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

src/include/utils モジュールの一部として、トランザクション内でのタプルの可視性判定を最適化する役割を担います。タプルヘッダーのサイズを削減するため、cmin と cmax を同一フィールドにオーバーレイする仕組みをサポートします。

責務

  • トランザクション終了時（AtEOXact_ComboCid）の Combo CID 状態のクリーンアップ。
  • 並列クエリ実行時やサブトランザクション間での Combo CID 状態のシリアライズおよび復元機能の提供。

重要なシンボル

  • AtEOXact_ComboCid(): トランザクション終了時に、メモリ上に保持された Combo CID のハッシュテーブルと配列を破棄します。
  • SerializeComboCIDState() / RestoreComboCIDState(): 並列ワーカー間で Combo CID のマッピング情報を共有するためのシリアライズ・デシリアライズ関数。
  • EstimateComboCIDStateSpace(): シリアライズに必要なメモリサイズを算出します。

制御フローとデータフロー

本ファイル自体はインターフェース定義のみですが、実際のデータフローは以下の通りです： 1. **挿入と削除の同居**: 同一トランザクション内でタプルを挿入し、かつ削除する場合、combocid.c 内のハッシュテーブルを用いて cmin と cmax のペアを単一の CommandId に変換します。 2. **マッピング**: HeapTupleHeaderGetCmin 等が呼び出されると、HEAP_COMBOCID フラグを確認し、必要に応じて Combo CID を実際の cmin/cmax に復元します。 3. **状態管理**: トランザクションのライフサイクルに合わせて、TopTransactionContext 上で状態が管理されます。

直接依存

本ファイルは直接的な #include を持たず、純粋な関数宣言のみを提供しています。

関連コード

  • src/backend/utils/time/combocid.c: 本ヘッダーで宣言された関数の実装および、ハッシュテーブル・配列の管理ロジック。
  • src/include/access/htup.h: HeapTupleHeaderGetCmin および HeapTupleHeaderGetCmax の関数プロトタイプが定義されており、本機能の利用側となります。

読者向けメモ

  • 本モジュールは、タプルヘッダーのサイズを抑えるための最適化機構です。
  • cmin と cmax の両方が必要な特殊なケース（同一トランザクション内での更新）でのみ使用されます。
  • 状態は TopTransactionContext に保持されるため、トランザクションの境界を意識した実装が必要です。
