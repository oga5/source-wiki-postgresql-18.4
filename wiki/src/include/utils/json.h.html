src/include/utils/json.h

概要

src/include/utils/json.h は、PostgreSQL における json 型のサポート機能を提供するヘッダーファイルです。主に JSON 文字列の構築、エスケープ処理、およびバリデーションに関するインターフェースを定義しています。

モジュール内での役割

src/include/utils モジュールの一部として、JSON 関連のデータ型処理の基盤を提供します。より高度なバイナリ形式である jsonb とは異なり、本ファイルはテキストベースの json 型の操作に特化しています。

責務

  • **JSON エスケープ**: 文字列を JSON 形式として安全に出力するためのエスケープ処理。
  • **JSON 構築**: SQL 関数から呼び出される JSON オブジェクトや配列の構築ロジックの提供。
  • **バリデーション**: 入力されたテキストが有効な JSON 形式であるかの検証。
  • **データ変換**: 日時型などの PostgreSQL 固有の型を JSON 互換の文字列へ変換する補助。

重要なシンボル

  • escape_json / escape_json_text: 文字列を JSON 文字列リテラルとしてバッファに書き出すための関数。
  • json_build_object_worker / json_build_array_worker: json_build_object や json_build_array などの SQL 関数を実装するための内部ワーカ関数。
  • json_validate: JSON 文字列の構文チェックを行う関数。

制御フローとデータフロー

本ファイルは宣言のみを提供し、実際のロジックは src/backend/utils/adt/json.c に実装されています。 1. **構築**: json_build_object_worker 等が引数として受け取った Datum 配列を走査し、StringInfo バッファへ順次シリアライズします。 2. **エスケープ**: シリアライズの過程で、特殊文字が含まれる場合は escape_json を経由してバッファへ書き込まれます。 3. **検証**: json_validate は JsonLexContext を使用して字句解析を行い、構文エラーがないかを確認します。

直接依存

  • lib/stringinfo.h: JSON 文字列構築用の動的バッファ構造体 StringInfo を利用するために必須。

関連コード

  • src/backend/utils/adt/json.c: 本ヘッダーで宣言された関数の実装本体。
  • src/include/utils/jsonb.h: バイナリ形式の JSONB 型を扱うためのヘッダー。
  • src/include/utils/jsonfuncs.h: JSON 関連の SQL 関数インターフェースを定義。

読者向けメモ

  • 本ファイルで定義されている関数は、主に SQL レベルの関数（json_in, json_out 等）から呼び出されることを想定しています。
  • json 型は内部的に text 型として保持されるため、複雑な操作や検索が必要な場合は jsonb の利用が推奨されます。
  • escape_json 系の関数を使用する際は、必ず StringInfo バッファが適切に初期化されていることを確認してください。
