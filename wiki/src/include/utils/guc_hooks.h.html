src/include/utils/guc_hooks.h

概要

PostgreSQL の設定パラメータ（GUC: Grand Unified Configuration）に関連する、システム全体に散在するフック関数の宣言を集約したヘッダーファイルです。

モジュール内での役割

src/include/utils/guc.h が定義する GUC システムのインターフェースを補完します。GUC 変数の変更時に呼び出されるバリデーション（check_）や反映（assign_）関数を個別のソースファイルからこのヘッダーに集約することで、guc.h 自体が肥大化し、無関係なソースファイルへ過剰にインクルードされることを防いでいます。

責務

  • GUC 変数の変更プロセスで使用されるコールバック関数のプロトタイプ宣言の提供。
  • システム内の各コンポーネント（ストレージ、ネットワーク、トランザクション管理など）が実装するフック関数のインターフェース定義。

重要なシンボル

  • check_* 関数群: 設定値が適用される前に妥当性を検証する関数。戻り値 bool で成否を判定します。
  • assign_* 関数群: 検証済みの設定値をシステム内部の状態に反映させる関数。
  • show_* 関数群: 現在の設定値を文字列として表示するためのカスタムロジックを提供する関数。

制御フローとデータフロー

1. **設定変更**: ユーザーが SET コマンドや設定ファイル経由で値を変更します。 2. **検証**: GUC システムは、対応する check_ フックを呼び出し、新しい値が許容範囲内かを確認します。 3. **適用**: 検証が成功した場合、assign_ フックが呼び出され、メモリ上の変数やシステム状態が更新されます。 4. **表示**: SHOW コマンド実行時、必要に応じて show_ フックが呼び出され、動的に生成された文字列が返されます。

直接依存

  • utils/guc.h: フック関数の型定義（GucBoolCheckHook 等）および GucSource 列挙型を利用するために必須です。

関連コード

  • src/backend/utils/misc/guc.c: GUC システムのメインロジック。ここで宣言されたフック関数が実際に登録・利用されます。
  • src/include/utils/guc.h: GUC システムの基盤となるデータ構造とマクロ定義。

読者向けメモ

  • 本ファイルは単なる宣言のリストです。実装は各機能のソースファイル（例: xact.c, pgstat.c など）に分散しています。
  • 新しい GUC 変数を追加し、カスタムフックが必要な場合は、実装後にこのファイルへプロトタイプを追加してください。
  • 宣言はアルファベット順に整理されています。保守の際は順序を維持することが推奨されます。
