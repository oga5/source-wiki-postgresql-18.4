src/include/utils/geo_decls.h

概要

PostgreSQLの幾何データ型（Point, LSEG, PATH, LINE, BOX, POLYGON, CIRCLE）の定義および、それらを扱うための浮動小数点演算ユーティリティを提供するヘッダーファイルです。

モジュール内での役割

src/include/utils モジュールにおいて、幾何学的な計算やデータ構造の基盤を定義します。SQLレベルの幾何型とC言語のデータ構造を橋渡しする役割を担います。

責務

  • 幾何データ型の構造体定義。
  • 浮動小数点数の「曖昧な比較（Fuzzy comparison）」を行うためのインライン関数の提供。
  • PostgreSQLの関数マネージャ（fmgr）を介した、データ型とDatum間の変換マクロおよびインライン関数の提供。

重要なシンボル

  • **EPSILON / FPeq 等**: 浮動小数点演算の誤差を許容するための比較関数群。
  • **幾何構造体**: Point, LSEG, PATH, LINE, BOX, POLYGON, CIRCLE。
  • **PG_GETARG_... マクロ**: 関数呼び出し時に引数から特定の幾何型を取り出すためのインターフェース。

追加の索引済みシンボル

  • FPgt
  • FPle
  • FPlt
  • FPne

制御フローとデータフロー

  • **比較処理**: FPeq 等の関数は、fabs(A - B) <= EPSILON を用いて、計算誤差を考慮した等価判定を行います。
  • **データ変換**: DatumGet...P 関数群は、Datum（PostgreSQLの汎用データ型）から各構造体へのポインタをキャストします。PATH や POLYGON のような可変長データ型（TOAST対象）については、PG_DETOAST_DATUM を経由してメモリ上の実体を取得します。

直接依存

  • math.h: fabs 等の数学関数を利用。
  • fmgr.h: PostgreSQLの関数呼び出し規約および Datum 型の定義を利用。

関連コード

  • src/backend/utils/adt/geo_ops.c: 本ヘッダーで宣言されたデータ型を操作する実際の関数実装が含まれています。
  • src/include/fmgr.h: Datum 型やTOAST処理の基盤となる定義。

読者向けメモ

  • **数値解析の注意**: ソース内のコメントにある通り、これらのルーチンは数値解析の専門家によって書かれたものではありません。特に FPeq 等の比較は推移律を満たさない可能性があるため、厳密な計算が必要な場面では注意が必要です。
  • **NaNの扱い**: ここで定義されている比較関数はNaNを考慮しておらず、NaNが含まれる場合は常に FALSE を返します。
  • **可変長データ**: PATH と POLYGON は varlena 型であり、アクセス時には必ずTOAST解除（PG_DETOAST_DATUM）が必要である点に留意してください。
