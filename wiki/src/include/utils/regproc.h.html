src/include/utils/regproc.h

概要

regproc.h は、PostgreSQL のシステムカタログオブジェクト（関数、演算子、クラスなど）を識別するための「レジストリ型（regproc, regclass, regtype 等）」に関連するユーティリティ関数を定義するヘッダーファイルです。

モジュール内での役割

src/include/utils モジュールの一部として、OID（Object ID）と人間が読み取り可能な名前（スキーマ修飾名など）の相互変換を担います。主に SQL レベルでのオブジェクト参照をシステム内部の OID に解決する際や、その逆の変換に使用されます。

責務

  • オブジェクト名（文字列）から OID への変換支援。
  • OID から人間が理解可能な名前（修飾名）へのフォーマット。
  • オブジェクトの存在確認や、曖昧な参照の解決のためのフラグ制御。

重要なシンボル

  • format_procedure_extended / format_operator_extended: オブジェクトの OID を文字列に変換する汎用関数。flags により、未定義時の挙動（NULL を返すか）やスキーマ修飾の強制を指定可能。
  • FORMAT_PROC_FORCE_QUALIFY: format_procedure 系でスキーマ修飾を強制するためのフラグ。
  • stringToQualifiedNameList: 入力文字列をスキーマ名とオブジェクト名のリストに分解するパーサー。

制御フローとデータフロー

本ファイルはインターフェース定義のみを提供します。実際のデータフローは以下の通りです： 1. **入力**: SQL クエリ等から渡された文字列が regprocin 等の I/O ルーチンへ渡される。 2. **解析**: stringToQualifiedNameList を経由して名前リストが生成される。 3. **検索**: FuncnameGetCandidates 等（regproc.c 内）がシステムカタログを検索し、OID を特定する。 4. **出力**: format_procedure 等が OID を受け取り、カタログ情報を参照して文字列を生成する。

直接依存

  • nodes/pg_list.h: オブジェクト名リスト（List）を扱うために必須。

関連コード

  • src/backend/utils/adt/regproc.c: 本ヘッダーで宣言された関数の実装。I/O ルーチンやカタログ検索ロジックが含まれる。
  • src/include/catalog/: pg_proc, pg_operator などのシステムカタログ定義。変換ロジックの参照先。

読者向けメモ

  • このファイルで定義される関数は、主に SQL の regproc 型や regclass 型の入出力関数から呼び出されます。
  • _extended 接尾辞が付いた関数は、エラーハンドリングやフォーマットの柔軟性を高めるために使用してください。
  • 内部的な OID 変換を行う際は、IsBootstrapProcessingMode() のチェックが必要になるケースがあるため、実装側（regproc.c）の文脈も併せて確認することを推奨します。
