src/include/utils/ruleutils.h

概要

ruleutils.h は、PostgreSQL の内部表現（Query ツリー、式、プランノードなど）を、人間が読み取り可能な SQL ソーステキストに変換（デパース）するための関数群を定義するヘッダーファイルです。

モジュール内での役割

src/include/utils モジュールにおいて、システムカタログやプランナが生成した中間表現を、EXPLAIN 出力や pg_get_viewdef などのメタデータ取得機能を通じて SQL として再構成する役割を担います。

責務

  • **SQL 再構成**: 内部ノードから SQL 文や式を文字列として生成する。
  • **定義の抽出**: インデックス、パーティション、制約などの定義を SQL 形式で取得する。
  • **デパースコンテキストの管理**: 式の評価に必要な名前空間やリレーション情報を保持するコンテキストの構築。

重要なシンボル

  • pg_get_querydef: Query 構造体から SQL 文字列を生成する主要なエントリポイント。
  • deparse_expression: 任意の式ノードを SQL 文字列に変換する。
  • deparse_context_for: デパース時に必要なリレーションの別名や OID を管理するリストを作成する。
  • RULE_INDEXDEF_PRETTY / RULE_INDEXDEF_KEYS_ONLY: インデックス定義取得時のフォーマット制御フラグ。

制御フローとデータフロー

1. **コンテキスト構築**: deparse_context_for 等を用いて、デパース対象のクエリやプランに関連する名前空間（dpcontext）を初期化します。 2. **再帰的変換**: deparse_expression がノードツリーを走査し、StringInfo バッファへ SQL を逐次書き出します。 3. **出力**: 最終的に構築された文字列が呼び出し元へ返されます。この際、pretty フラグに応じてインデントや改行が挿入されます。

直接依存

  • nodes/nodes.h: ノードタグおよび基本構造。
  • nodes/parsenodes.h: Query 構造体などの定義。
  • nodes/pg_list.h: リスト操作用 API。

関連コード

  • src/backend/utils/adt/ruleutils.c: 本ヘッダーで宣言された関数の実装本体。
  • src/include/nodes/parsenodes.h: デパース対象となるクエリツリーの定義。
  • src/include/utils/lsyscache.h: OID からオブジェクト名を取得する際に頻繁に併用される。

読者向けメモ

  • 本ファイルは「デパース（Deparsing）」と呼ばれる処理のインターフェースです。
  • struct Plan や struct PlannedStmt は前方宣言されており、ヘッダーの依存関係を最小限に抑えています。
  • 複雑なクエリのデパースには dpcontext（デパースコンテキスト）の正確な構築が不可欠です。実装を読む際は、ruleutils.c 内の deparse_namespace 構造体の定義を併せて参照してください。
