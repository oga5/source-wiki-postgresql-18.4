src/include/utils/funccache.h

概要

funccache.h は、PostgreSQL における関数実行時のメタデータやコンパイル済み情報をキャッシュするためのインターフェースを定義します。SQL や PL/pgSQL などの言語で、関数呼び出しごとのオーバーヘッドを削減するために使用されます。

モジュール内での役割

src/include/utils モジュールの一部として、関数実行エンジン（fmgr）とキャッシュ管理層の橋渡しをします。特定の OID や引数型、照合順序（Collation）の組み合わせごとにキャッシュを管理することで、ポリモーフィック関数やトリガーの効率的な実行を支えます。

責務

  • 関数キャッシュの検索・登録に使用するハッシュキー構造体（CachedFunctionHashKey）の定義。
  • キャッシュされた関数情報の共通ヘッダー構造体（CachedFunction）の定義。
  • 関数コンパイルおよびキャッシュ破棄時のコールバックインターフェースの提供。
  • キャッシュのライフサイクル管理（コンパイル、解決、削除）のための API 提供。

重要なシンボル

  • CachedFunctionHashKey: キャッシュの識別子。関数 OID、引数型、照合順序、トリガー情報など、実行結果に影響を与えるすべての要素を含みます。
  • CachedFunction: キャッシュエントリの基底構造体。言語固有のデータ構造に埋め込まれることを想定しています。
  • cached_function_compile(): キャッシュを検索し、未登録であればコールバックを介して関数をコンパイル・登録するメイン関数。
  • cfunc_resolve_polymorphic_argtypes(): ポリモーフィック型（anyelement 等）を実際の型に解決するためのユーティリティ。

制御フローとデータフロー

1. **検索**: cached_function_compile が呼び出されると、現在の FunctionCallInfo から CachedFunctionHashKey を生成します。 2. **判定**: ハッシュテーブル内に該当キーが存在するか確認します。 3. **コンパイル**: キャッシュミス時は、提供された CachedFunctionCompileCallback を実行し、言語固有のロジックで関数をコンパイルします。 4. **登録**: コンパイルされた結果をハッシュテーブルに格納し、以降の呼び出しで再利用可能にします。 5. **無効化**: pg_proc の更新（xmin/ctid の変化）を検知し、必要に応じてキャッシュを破棄します。

直接依存

  • access/htup_details.h: HeapTuple 構造体の定義。
  • fmgr.h: 関数呼び出し情報（FunctionCallInfo）の定義。
  • storage/itemptr.h: ItemPointerData（ctid）の定義。

関連コード

  • src/backend/utils/cache/funccache.c: 本ヘッダーで定義された構造体と関数の実装。ハッシュテーブルの管理ロジックが含まれます。
  • src/include/fmgr.h: 関数マネージャの基盤。
  • src/include/utils/syscache.h: pg_proc などのシステムカタログキャッシュへのアクセスに使用。

読者向けメモ

  • CachedFunctionHashKey を変更した場合は、必ず funccache.c 側のハッシュ関数とマッチング関数を更新する必要があります。
  • キャッシュエントリは TopMemoryContext に配置されることが推奨されています。
  • CachedFunction 構造体は、言語固有のデータ構造の先頭に配置（埋め込み）されることを前提として設計されています。
