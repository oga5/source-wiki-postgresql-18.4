src/include/mb/pg_wchar.h

概要

PostgreSQLのマルチバイト文字サポートにおける最下層の定義ファイルです。バックエンドおよびフロントエンド（libpqを除く）で共有され、文字エンコーディングの識別子、変換テーブルの構造、および文字操作のための抽象インターフェースを定義します。

モジュール内での役割

src/include/mb モジュールの中心的なヘッダーであり、エンコーディング変換ロジック、検証ロジック、および文字列操作関数群の型定義を提供します。このファイルは、PostgreSQLがサポートする多様なエンコーディングを統一的に扱うための「契約」を定義しています。

責務

  • **型定義**: pg_wchar（内部文字表現）および pg_enc（エンコーディング識別子）の定義。
  • **エンコーディング管理**: サポートされる全エンコーディングの列挙と、それらを操作するための関数テーブル構造の提供。
  • **変換・検証インターフェース**: 文字列の長さ計算、表示幅算出、検証、およびエンコーディング変換のための関数ポインタ型の定義。
  • **Unicode ユーティリティ**: UTF-8 と Unicode コードポイント間の変換および検証用インライン関数の提供。
  • **MULE 関連定義**: MULE (Multilingual Extension) 内部エンコーディングの構造と定数の定義。

重要なシンボル

型と列挙型

  • pg_wchar: PostgreSQL内部で文字を扱うための unsigned int 型。
  • pg_enc: サポートされる全エンコーディングの列挙型。PG_SQL_ASCII から PG_JOHAB までを網羅。
  • pg_wchar_tbl: エンコーディングごとの操作関数（変換、長さ計算、検証）を保持する構造体。

変換・検証関数ポインタ型

  • mb2wchar_with_len_converter / wchar2mb_with_len_converter: マルチバイト文字列と pg_wchar 間の変換。
  • mblen_converter / mbdisplaylen_converter: 文字のバイト長および表示幅の取得。
  • mbchar_verifier / mbstr_verifier: 文字および文字列の妥当性検証。

変換用データ構造

  • pg_mb_radix_tree: UTF-8 と他エンコーディング間の高速変換用基数木。
  • pg_utf_to_local_combined / pg_local_to_utf_combined: 結合文字の変換用マップ。

制御フローとデータフロー

1. **エンコーディングの解決**: pg_char_to_encoding 等により、文字列名から pg_enc ID を取得します。 2. **関数ディスパッチ**: pg_wchar_table[] を pg_enc ID でインデックスし、該当するエンコーディングの関数ポインタ（mblen, mbverifychar 等）を呼び出します。 3. **変換処理**: pg_do_encoding_conversion が呼び出されると、ソースとターゲットのエンコーディングに基づき、適切な変換関数または基数木（pg_mb_radix_tree）を用いた変換が実行されます。 4. **検証**: pg_verify_mbstr は、各エンコーディングの mbverifystr 関数を呼び出し、入力バイト列がそのエンコーディングとして妥当かをチェックします。

直接依存

  • 本ファイルはスタンドアロンで動作するように設計されており、特定のソースファイルへの直接依存はありません。ただし、lib/stringinfo.h 等の他のモジュールから利用されることを想定しています。

関連コード

  • src/common/wchar.c: pg_wchar_table の実体定義。
  • src/common/encnames.c: pg_enc2name_tbl 等のエンコーディング名変換テーブル。
  • src/backend/utils/mb/mbutils.c: エンコーディング変換の実行時ロジック。

読者向けメモ

  • **メモリとライフサイクル**: 本ファイルで定義される関数は、入力文字列のバッファサイズが MAX_CONVERSION_GROWTH（4倍）を考慮していることを前提としています。
  • **ABIの注意**: pg_enc の列挙順序を変更すると libpq の ABI 互換性が破壊されるため、既存の ID を変更してはいけません。
  • **フロントエンド利用**: libpq クライアントプログラムは本ファイルを直接インクルードしてはいけません。libpq-fe.h を通じて公開されている関数のみを使用してください。
  • **省略された詳細**: 基数木の具体的な構築アルゴリズムや、各エンコーディングごとの詳細な変換ロジックは、このヘッダーではなく各実装ファイル（src/common/wchar.c 等）に委譲されています。
