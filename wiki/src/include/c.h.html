src/include/c.h

概要

c.h は PostgreSQL の全ソースコード（バックエンドおよびフロントエンド）が読み込む最上位のヘッダーファイルです。プラットフォームやコンパイラ間の差異を抽象化し、システム全体で共通して利用される型定義、マクロ、および属性指定を提供します。

モジュール内での役割

PostgreSQL のビルドシステムにおける「環境定義層」の頂点に位置します。postgres.h（バックエンド用）や postgres_fe.h（フロントエンド用）を介してすべての .c ファイルに注入され、移植性の高い開発環境を構築します。

責務

  • **コンパイラ抽象化**: GCC/Clang/MSVC 等の属性（__attribute__ 等）を統一的なマクロとして提供。
  • **標準型定義**: int8〜int64、bool、Pointer、Oid 等の基本型の定義。
  • **メモリ・アライメント**: 構造体のパディングやメモリ配置を制御するアライメントマクロの提供。
  • **アサーション**: デバッグビルド時とリリースビルド時で挙動を切り替えるアサーション機構の提供。
  • **プラットフォーム互換**: OS 固有の挙動（WIN32 のファイルオープンモード等）の吸収。

重要なシンボル

コンパイラ属性マクロ

  • pg_attribute_unused(), pg_nodiscard, pg_noreturn: コンパイラの警告抑制や最適化ヒント。
  • pg_attribute_printf(f, a): printf 形式の引数チェック。
  • pg_attribute_always_inline, pg_noinline: インライン展開の強制または禁止。

基本型とデータ構造

  • int8〜int64, uint8〜uint64: 標準化された整数型。
  • Pointer: char * として定義される汎用ポインタ。
  • varlena: 可変長データ型の基底構造体（vl_len_ フィールドを持つ）。
  • PGAlignedBlock: アライメントを保証したページバッファ用共用体。

アサーションと検証

  • Assert(condition): デバッグビルド時に条件を検証し、失敗時に ExceptionalCondition を呼び出す。
  • StaticAssertDecl(cond, msg): コンパイル時に条件を検証する。
  • PointerIsValid(ptr), OidIsValid(oid): 値の妥当性チェック。

メモリ・アライメント操作

  • MAXALIGN(LEN): システムの最大アライメント境界に切り上げる。
  • MemSet(start, val, len): 標準 memset よりも高速な、ワード単位のゼロクリアマクロ。

制御フローとデータフロー

本ファイルは実行コードを含まないヘッダーですが、以下のフローを制御します。

  • **コンパイル時**: pg_config.h 等の環境設定を読み込み、#define を通じてコンパイル時の挙動を決定します。
  • **データレイアウト**: varlena 構造体や PGAlignedBlock を通じて、メモリ上のデータ配置を強制し、プラットフォーム間のアライメント差異を隠蔽します。
  • **エラーパス**: Assert マクロは、デバッグビルド時に ExceptionalCondition を経由してプロセスを終了させるパスを生成します。

直接依存

  • pg_config.h, pg_config_manual.h, pg_config_os.h: ビルド環境設定。
  • 標準 C ライブラリ: <stdint.h>, <stddef.h>, <stdio.h> 等。
  • postgres_ext.h: 外部公開用の基本型定義。
  • port.h: プラットフォーム固有の互換実装。

関連コード

  • postgres.h: バックエンド用ヘッダー。c.h を含み、さらに高度な型定義を追加。
  • postgres_fe.h: フロントエンド用ヘッダー。c.h を含み、クライアントライブラリ用の定義を提供。
  • src/port/: c.h で宣言された互換関数の実体が含まれるディレクトリ。

読者向けメモ

  • **メモリ所有権**: 本ファイルは型定義のみを提供し、メモリ管理は palloc 等の MemoryContext API に依存します。
  • **アライメントの注意**: MAXALIGN は組み込み型のみを対象としており、バッファの配置には BUFFERALIGN や PGAlignedBlock を使用してください。
  • **未実装・浅い部分**: int128 のサポートはプラットフォーム依存であり、一部の環境では利用できません。
  • **保守上の注意**: c.h は全ファイルに影響するため、変更は極めて慎重に行う必要があります。extern 宣言は原則として禁止されており、必要な場合は ifdef で対象を限定してください。
