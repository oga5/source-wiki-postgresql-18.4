src/include/tsearch/ts_type.h

概要

PostgreSQL の全文検索機能（Tsearch）で使用される主要なデータ型 tsvector および tsquery のメモリレイアウトと構造を定義するヘッダーファイルです。

モジュール内での役割

Tsearch サブシステムの基盤となるデータ構造を定義します。このファイルで定義された構造体は、インデックス作成、検索クエリの解析、および SQL インターフェース（fmgr）を介したデータ変換の共通言語として機能します。

責務

  • tsvector（検索対象のドキュメント）のバイナリ表現の定義。
  • tsquery（検索条件）のバイナリ表現の定義。
  • 各データ型を PostgreSQL の Datum と相互変換するためのインライン関数およびマクロの提供。
  • 語彙（lexeme）の重みや位置情報、クエリ演算子の優先順位を操作するためのユーティリティマクロの提供。

重要なシンボル

  • TSVectorData / TSVector: 語彙と位置情報の配列を保持する構造体。
  • WordEntry: tsvector 内の個々の語彙を管理するメタデータ。
  • TSQueryData / TSQuery: 演算子とオペランドからなるクエリツリーの構造体。
  • QueryItem: tsquery 内のノード（演算子またはオペランド）を表現する共用体。
  • WEP_GETWEIGHT / WEP_GETPOS: WordEntryPos から重みや位置情報を抽出するマクロ。

追加の索引済みシンボル

  • QueryItemType
  • QueryOperand

制御フローとデータフロー

  • **データ構造の配置**: TSVectorData は可変長配列（FLEXIBLE_ARRAY_MEMBER）を使用し、ヘッダー、WordEntry 配列、実際の文字列データ、位置情報ベクトルの順にメモリ上に連続して配置されます。
  • **アクセス**: ARRPTR() や STRPTR() マクロを使用して、メモリ上の特定セクションへ直接ポインタ演算を行います。
  • **変換**: PG_GETARG_TSVECTOR 等の fmgr マクロが、PostgreSQL の Datum から内部構造体へのキャストと、必要に応じたデトースト（TOAST 解除）を仲介します。

直接依存

  • fmgr.h: PostgreSQL の関数呼び出しインターフェース。
  • utils/memutils.h: メモリ管理関連の定義。

関連コード

  • src/backend/tsearch/: このヘッダーで定義された構造体を操作する実装（tsvector_op.c や tsquery_util.c など）。
  • src/include/tsearch/ts_public.h: Tsearch の公開 API 定義。

読者向けメモ

  • **メモリレイアウト**: tsvector および tsquery は、ディスク保存やネットワーク転送を考慮した厳密なバイナリレイアウトを持っています。構造体を直接操作する際は、アライメントや varlena ヘッダーの扱いに注意してください。
  • **アライメント**: TSQuery は 4 バイトアライメントを前提としています。構造体定義に 8 バイトアライメントが必要な型（int64 等）を追加することは避けてください。
  • **可変長データ**: FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズ計算には CALCDATASIZE や COMPUTESIZE マクロを必ず使用してください。
