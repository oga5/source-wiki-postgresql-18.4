src/include/fmgr.h

概要

fmgr.h は、PostgreSQL の関数マネージャ（Function Manager: fmgr）のインターフェースを定義する中心的なヘッダーファイルです。SQL レベルの関数を C 言語で実装・呼び出しするための規約、データ構造、およびマクロを提供します。

モジュール内での役割

PostgreSQL のバックエンドおよび拡張モジュールにおいて、関数呼び出しの抽象化レイヤーとして機能します。SQL 関数と C 言語の関数を橋渡しし、引数の受け渡し、型変換、メモリ管理、および動的ロードされたライブラリの管理を統括します。

責務

  • **関数呼び出し規約の定義**: FunctionCallInfo を介した引数・戻り値の標準的な受け渡し方法の提供。
  • **型変換とデトースト**: PG_GETARG_xxx や PG_DETOAST_DATUM 等のマクロによる、PostgreSQL 内部データ型と C 言語型の安全な変換。
  • **メタデータ管理**: FmgrInfo 構造体による、関数呼び出しに必要なカタログ情報のキャッシュ。
  • **ABI 互換性チェック**: PG_MODULE_MAGIC による、ロードされるモジュールとバックエンド間のバージョン・ABI 整合性の検証。

重要なシンボル

  • FunctionCallInfo: 関数呼び出し時の引数やコンテキストを保持する構造体。
  • FmgrInfo: 関数呼び出しに必要なメタデータ（関数ポインタ、引数数、厳密性など）を保持する構造体。
  • PG_FUNCTION_ARGS: 関数定義時に使用する引数リストの標準マクロ。
  • PG_FUNCTION_INFO_V1: C 言語関数を fmgr 対応（Version 1）として登録するためのマクロ。
  • PG_MODULE_MAGIC: 拡張モジュールがバックエンドと互換性があることを示すための必須マクロ。

制御フローとデータフロー

1. **初期化**: fmgr_info() を呼び出し、OID から FmgrInfo を構築してキャッシュします。 2. **呼び出し**: FunctionCallInvoke マクロまたは FunctionCallNColl 系関数を使用して、FmgrInfo と引数を渡して実行します。 3. **引数取得**: 呼び出された関数内で PG_GETARG_xxx マクロを使用し、fcinfo->args からデータを抽出します。 4. **戻り値**: PG_RETURN_xxx マクロを使用して Datum 型として結果を返します。

直接依存

本ファイルはスタンドアロンで動作するように設計されており、他の PostgreSQL ヘッダーへの直接的な依存を最小限に抑えています。必要な型（Node, Aggref 等）は前方宣言やスタブ定義で対応しています。

読者向けメモ

  • **メモリ管理**: FmgrInfo に付随するデータは、指定された MemoryContext に割り当てられます。長期的なキャッシュを行う場合は fmgr_info_cxt を使用してください。
  • **アライメント**: PG_DETOAST_DATUM 等のデトーストマクロは、アライメントを考慮したアクセスを保証します。直接ポインタをキャストする際は注意が必要です。
  • **NULL 処理**: PG_ARGISNULL で引数の NULL チェックを行い、PG_RETURN_NULL で NULL を返します。strict な関数では fmgr が自動的に処理しますが、それ以外は実装者の責任です。
