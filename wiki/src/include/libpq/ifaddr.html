src/include/libpq/ifaddr.h

概要

ifaddr.h は、PostgreSQL における IP アドレスのサブネット判定およびネットワークインターフェース操作のためのユーティリティ関数群を定義するヘッダーファイルです。主にクライアント認証（pg_hba.conf）における IP 範囲の検証や、接続許可設定の処理をサポートします。

モジュール内での役割

libpq サブシステムの一部として、ネットワーク層の抽象化を提供します。特に、OS 固有のネットワークインターフェース情報やソケットアドレスの操作を、PostgreSQL のバックエンドが扱いやすい形式に変換・比較する役割を担います。

責務

  • **サブネット判定**: 指定された IP アドレスが特定のネットワーク範囲内にあるかの検証。
  • **CIDR マスク生成**: CIDR 表記（例: /24）から、IPv4/IPv6 に対応したソケットアドレス形式のネットマスクを生成。
  • **インターフェース列挙**: システム上のネットワークインターフェースを走査し、コールバック関数を通じてアドレス情報を通知する仕組みの提供。

重要なシンボル

  • pg_range_sockaddr: addr が netaddr/netmask で定義される範囲内にあるかを確認する関数。
  • pg_sockaddr_cidr_mask: CIDR ビット数から sockaddr_storage 形式のマスクを作成する関数。
  • pg_foreach_ifaddr: システムの全インターフェースを走査し、各アドレスに対してコールバックを実行する関数。
  • PgIfAddrCallback: pg_foreach_ifaddr で使用される、インターフェース情報を処理するためのコールバック型。

追加の索引済みシンボル

  • sockaddr

制御フローとデータフロー

1. **範囲検証**: pg_range_sockaddr は sockaddr_storage を受け取り、内部で AF_INET または AF_INET6 に分岐してビット演算による比較を行います。 2. **マスク生成**: pg_sockaddr_cidr_mask は文字列のビット数を数値に変換し、指定されたアドレスファミリーに応じたビットマスクを構築して sockaddr_storage に書き込みます。 3. **インターフェース走査**: pg_foreach_ifaddr は OS のネットワーク API を利用してインターフェースを列挙し、見つかった各アドレスに対して PgIfAddrCallback を呼び出します。

直接依存

  • libpq/pqcomm.h: ソケット通信に関連する基本的なデータ構造（sockaddr_storage 等）の定義を提供。

関連コード

  • src/backend/libpq/ifaddr.c: 本ヘッダーで宣言された関数の実装。プラットフォーム固有のネットワーク API（getifaddrs 等）への依存が含まれます。

読者向けメモ

  • 本モジュールは、IPv4 と IPv6 の両方を透過的に扱えるように設計されています。
  • pg_range_sockaddr を呼び出す際は、事前にアドレスファミリーが一致していることを確認する必要があります。
  • pg_foreach_ifaddr の実装はプラットフォーム依存であるため、移植性を考慮する場合は ifaddr.c の実装詳細を確認してください。
