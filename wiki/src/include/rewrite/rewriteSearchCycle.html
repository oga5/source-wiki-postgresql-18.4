src/include/rewrite/rewriteSearchCycle.h

概要

rewriteSearchCycle.h は、PostgreSQL の再帰的共通テーブル式（CTE）における SEARCH および CYCLE 句の書き換え処理を定義するヘッダーファイルです。

モジュール内での役割

src/include/rewrite モジュールの一部として、クエリ書き換えフェーズにおいて、標準 SQL で定義された SEARCH / CYCLE 句を、PostgreSQL が実行可能な再帰的クエリ構造へ変換するためのインターフェースを提供します。

責務

  • SEARCH 句（深さ優先/幅優先探索）および CYCLE 句（循環検出）を含む CommonTableExpr を、内部的な再帰クエリ表現へ変換する関数の宣言。
  • 変換ロジックの公開 API の提供。

重要なシンボル

  • rewriteSearchAndCycle(CommonTableExpr *cte):
  • 本モジュールの主要関数。与えられた CommonTableExpr を解析し、SEARCH または CYCLE 句が存在する場合、それらを再帰的クエリの追加列や UNION 構造へ書き換えた新しい CommonTableExpr を返します。

制御フローとデータフロー

  • **入力**: CommonTableExpr 構造体（SEARCH または CYCLE 句の情報を含む）。
  • **処理**:
  • rewriteSearchAndCycle が呼び出されると、rewriteSearchCycle.c 内で定義されたロジックにより、CTE の ctequery（UNION 構造）に対して、探索順序を保持するための列や、循環を検出するための配列列を追加します。
  • **出力**: 書き換えられた CommonTableExpr。

直接依存

  • nodes/parsenodes.h: CommonTableExpr 構造体の定義を参照するために必要です。

関連コード

  • src/backend/rewrite/rewriteSearchCycle.c: 本ヘッダーで宣言された関数の実装本体。変換の詳細なアルゴリズム（ROW 型や配列演算を用いたクエリ構築）が記述されています。
  • src/include/nodes/parsenodes.h: CommonTableExpr 構造体の詳細を確認する際に参照します。

読者向けメモ

  • 本ファイルは非常にシンプルですが、実装側の rewriteSearchCycle.c に記述されているコメントには、SEARCH / CYCLE 句が具体的にどのような SQL 構造に展開されるかの詳細な例が記載されています。変換ロジックを理解する際は、まず実装側のファイルヘッダーコメントを読むことを強く推奨します。
  • この変換は、パーサが生成した構文木を、クエリ実行エンジンが理解できる標準的な再帰クエリの形式に「正規化」するプロセスの一部です。
