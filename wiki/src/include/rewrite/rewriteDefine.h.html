src/include/rewrite/rewriteDefine.h

概要

PostgreSQLのルールシステムにおいて、書き換えルール（Rewrite Rule）の定義、変更、および管理を行うためのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

src/include/rewrite モジュールの中で、SQLの CREATE RULE や ALTER RULE 文の実行時に呼び出されるバックエンド処理の入り口を提供します。ルール定義のメタデータ管理を担う中心的な役割を果たします。

責務

  • ルール定義（DefineRule）およびクエリ書き換え定義（DefineQueryRewrite）の提供。
  • 既存ルールの名前変更（RenameRewriteRule）機能の提供。
  • ルールの有効/無効状態の切り替え（EnableDisableRule）。
  • ルール実行時のユーザー権限チェック設定（setRuleCheckAsUser）。

重要なシンボル

  • RULE_FIRES_ON_ORIGIN / ALWAYS / REPLICA / DISABLED: ルールの発火条件を制御するフラグ。pg_rewrite.ev_enabled に格納されます。
  • DefineRule: SQLの RuleStmt を受け取り、ルールをシステムカタログに登録する主要関数。
  • DefineQueryRewrite: ルール定義の低レベルな実装。pg_rewrite カタログへの挿入や更新を直接扱います。

制御フローとデータフロー

1. **定義フロー**: パーサーが生成した RuleStmt が DefineRule に渡されます。 2. **検証と変換**: DefineRule は対象リレーションのロックや権限を確認し、DefineQueryRewrite を呼び出します。 3. **カタログ操作**: DefineQueryRewrite は、ルールのアクションやクエリツリーを文字列化（nodeToString）し、pg_rewrite システムカタログへ挿入または更新を行います。 4. **無効化/有効化**: EnableDisableRule は、指定されたリレーションのルールを検索し、ev_enabled フラグを更新することで動作を制御します。

直接依存

  • catalog/objectaddress.h: ObjectAddress 構造体（オブジェクトの識別に使用）。
  • nodes/parsenodes.h: RuleStmt や CmdType などのノード定義。
  • utils/relcache.h: Relation 構造体（リレーションキャッシュへのアクセス）。

関連コード

  • src/backend/rewrite/rewriteDefine.c: 本ヘッダーで宣言された関数の実装本体。
  • src/backend/rewrite/rewriteHandler.c: 定義されたルールを実際にクエリ実行時に適用するロジック。
  • src/include/catalog/pg_rewrite.h: ルール定義が格納されるシステムカタログのスキーマ定義。

読者向けメモ

  • 本ファイルで定義されるルール状態（RULE_FIRES_...）は、レプリケーション環境におけるルールの挙動を決定する重要な設定です。
  • DefineQueryRewrite は pg_rewrite を直接操作するため、呼び出し元で適切なロック（RowExclusiveLock 等）が確保されていることを前提としています。
  • ルールのアクションは nodeToString を介してテキスト形式で保存されるため、複雑なクエリツリーのシリアライズ/デシリアライズが内部で行われている点に注意してください。
