src/include/windowapi.h

概要

windowapi.h は、PostgreSQLのウィンドウ関数（Window Functions）が実行時にウィンドウ内のデータへアクセスするための公開APIを定義するヘッダーファイルです。

モジュール内での役割

ウィンドウ関数実行エンジン（主に nodeWindowAgg.c）と、ユーザー定義または組み込みのウィンドウ関数実装との間のインターフェースを提供します。ウィンドウ関数は通常の関数とは異なり、引数を直接受け取るのではなく、WindowObject を介してパーティション内の行データや状態を取得します。

責務

  • ウィンドウ関数がパーティション内の特定の行やフレームにアクセスするための抽象化レイヤーの提供。
  • パーティション内の行位置、行数、およびピア（同値）関係の判定機能の提供。
  • ウィンドウ関数ごとのローカルメモリ管理（WinGetPartitionLocalMemory）のサポート。

重要なシンボル

  • WindowObject: ウィンドウ関数の実行コンテキストを保持する不透明なポインタ型。
  • PG_WINDOW_OBJECT(): fcinfo->context から WindowObject を抽出するマクロ。
  • WindowObjectIsValid(): WindowObject の有効性を検証するマクロ。
  • WinGetFuncArgInPartition() / WinGetFuncArgInFrame(): パーティションまたはフレーム内の特定位置にある引数値を取得する関数。
  • WINDOW_SEEK_CURRENT / HEAD / TAIL: 検索位置の基準を指定する定数。

制御フローとデータフロー

1. **初期化**: ウィンドウ関数が呼び出されると、PG_WINDOW_OBJECT() を使用して実行コンテキストを取得します。 2. **データ取得**: 関数は WinGetFuncArgInPartition 等を呼び出し、seektype（相対位置）を指定して必要な行のデータをフェッチします。 3. **状態管理**: 必要に応じて WinGetPartitionLocalMemory を使用して、パーティション全体で共有される計算状態を保持します。 4. **終了**: 実行エンジンが各行に対して関数を呼び出し、結果を収集します。

直接依存

本ファイルはスタンドアロンのヘッダーとして設計されており、明示的な #include はありません。ただし、使用する側は fmgr.h や postgres.h 等の基本的な型定義がロードされている環境で利用することを前提としています。

関連コード

  • src/backend/executor/nodeWindowAgg.c: WindowObjectData 構造体の定義および、本APIで宣言されている関数の実装が含まれる中心的なファイルです。
  • src/include/fmgr.h: PostgreSQLの関数呼び出し規約（V1 calling convention）の定義。

読者向けメモ

  • **V1規約**: 本APIを使用する関数は、必ず PostgreSQL の V1 関数呼び出し規約に従う必要があります。
  • **不透明な構造体**: WindowObjectData の内部構造は nodeWindowAgg.c で隠蔽されています。直接構造体メンバにアクセスせず、必ず提供された Win* 関数経由で操作してください。
  • **メモリ管理**: WinGetPartitionLocalMemory で確保したメモリは、パーティションの処理が終了するまで有効です。一時的な計算結果の保持に最適です。
