src/include/c.h

概要

c.h は PostgreSQL の全ソースファイルが読み込む最上位のヘッダーファイルです。プラットフォームやコンパイラに依存しない「PostgreSQL の標準的な C 言語環境」を定義し、システム全体の整合性を担保します。

モジュール内での役割

PostgreSQL のビルドにおいて、すべての .c ファイルは postgres.h（バックエンド用）または postgres_fe.h（フロントエンド用）を経由して本ファイルをインクルードします。これにより、型定義やマクロの定義が全モジュールで統一されます。

責務

  • **環境抽象化**: pg_config.h 等を読み込み、OS やコンパイラの違いを吸収する。
  • **型定義**: int8〜int64、Size、Pointer などの標準的な型を定義する。
  • **コンパイラ最適化**: inline、noreturn、likely/unlikely などの属性マクロを提供し、コードの可読性と性能を向上させる。
  • **アサーション**: デバッグビルドとリリースビルドで挙動が変わる Assert マクロや、コンパイル時チェックを行う StaticAssert を提供する。
  • **メモリ操作**: MemSet など、標準ライブラリよりも効率的なメモリ操作マクロの提供。

重要なシンボル

  • **Assert(condition)**: デバッグビルド時に条件を検証する。
  • **MAXALIGN(LEN)**: 構造体やバッファをプラットフォームの最大アライメントに合わせる。
  • **int8〜int64**: 標準的な固定長整数型。
  • **pg_attribute_...**: コンパイラ固有の属性（always_inline, cold, nonnull 等）をラップしたマクロ群。
  • **lengthof(array)**: 配列の要素数を取得する。

制御フローとデータフロー

本ファイルは実行コードを含まず、プリプロセッサによる定義の注入が主目的です。 1. **初期化**: pg_config.h 等で環境変数を確定。 2. **属性定義**: コンパイラがサポートする機能を判定し、マクロを定義。 3. **型定義**: 整数型や varlena（可変長データ）構造体を定義。 4. **ユーティリティ**: アサーションやメモリ操作マクロを定義し、後続のコンパイル単位で使用可能にする。

関連コード

  • postgres_fe.h: フロントエンド用ヘッダー。c.h を含み、クライアントツール向けの定義を行う。
  • src/port/: c.h で宣言されたポータビリティ関数の実装が含まれる。

読者向けメモ

  • **命名規則**: pg_ で始まるマクロは PostgreSQL 固有の機能です。
  • **アライメント**: MAXALIGN は非常に重要です。PostgreSQL のデータ構造は、メモリ上の配置がこのアライメントに従うことを前提としています。
  • **extern の禁止**: このファイルは全ファイルにインクルードされるため、原則として extern 宣言は記述しません（二重定義エラーを防ぐため）。
  • **IWYU**: /* IWYU pragma: always_keep */ が付与されており、インクルード・ホワット・ユー・ユース（IWYU）ツールによる削除対象から除外されています。
