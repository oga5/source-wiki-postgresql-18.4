src/include/port.h

概要

src/include/port.h は、PostgreSQL がサポートする多様なプラットフォーム（Unix系およびWindows）間でのシステムコールの差異を吸収し、移植性を確保するための互換レイヤー定義ファイルです。

モジュール内での役割

src/include モジュールにおいて、OS 固有の挙動を抽象化する中心的な役割を担います。src/port/ ディレクトリに実装された互換関数のインターフェースを宣言し、マクロを通じて標準関数を PostgreSQL 独自のラッパー関数へ置き換えることで、コアコードが OS を意識せずに記述できるようにします。

責務

  • **プラットフォーム抽象化**: Windows と Unix 間のファイルパス操作、ソケット処理、プロセス実行の差異をマクロで吸収する。
  • **標準関数の置換**: snprintf や strerror など、プラットフォーム間で挙動が異なる、あるいは不足している標準ライブラリ関数を、堅牢な独自実装へ差し替える。
  • **環境依存の定数定義**: EXE（実行ファイル拡張子）や DEVNULL など、ビルド環境に依存する定数を提供する。

重要なシンボル

  • pgsocket: プラットフォームごとのソケット型を抽象化。
  • IS_DIR_SEP / is_absolute_path: OS 依存のパス区切り文字や絶対パス判定を統一。
  • ALL_CONNECTION_FAILURE_ERRNOS: ネットワーク接続失敗を示すエラーコードの集合。
  • pg_snprintf 等の pg_ 接頭辞関数群: 標準関数の移植性・安全性向上版。
  • qsort_arg: 比較関数にコンテキスト（引数）を渡せる拡張版 qsort。

追加の索引済みシンボル

  • SIGNAL_ARGS
  • qsort_arg_comparator
  • socklen_t

制御フローとデータフロー

本ファイルは主にコンパイル時の条件分岐（#ifdef）によって制御されます。 1. **プラットフォーム判定**: WIN32 定義の有無により、port/win32_port.h の読み込みや、open/fopen などの関数を pgwin32_ 系へリダイレクトします。 2. **マクロ置換**: printf や strerror などの標準関数を、pg_ で始まるラッパー関数へマクロ置換し、実行時に適切な互換実装が呼び出されるようにします。 3. **機能検出**: HAVE_ 系マクロを用いて、OS が提供していない機能を src/port/ 内の代替実装で補完します。

直接依存

  • <ctype.h>, <math.h>: 文字処理および数学関数。
  • <io.h>, <unistd.h>: ファイル操作および POSIX 互換インターフェース。
  • <netinet/in.h>, <arpa/inet.h>: ネットワークアドレス操作。
  • port/win32_port.h: Windows 固有の定義。

関連コード

  • src/port/: 本ファイルで宣言された関数の実体（path.c, exec.c, snprintf.c など）。
  • src/include/postgres_fe.h: フロントエンドアプリケーション向けの共通ヘッダー。
  • src/include/c.h: PostgreSQL の最も基本的な型定義やマクロを含むヘッダー。

読者向けメモ

  • 本ファイルで定義されている pg_ 接頭辞付き関数は、可能な限り直接使用してください。
  • Windows 環境では open や rename などの標準関数がマクロで置き換えられているため、デバッグ時にスタックトレースが pgwin32_ 系関数を指すことに注意してください。
  • printf 系関数のマクロ置換は、関数ポインタの取得時に予期せぬ挙動を引き起こす可能性があるため、注意が必要です。
