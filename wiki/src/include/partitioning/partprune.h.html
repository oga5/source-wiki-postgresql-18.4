src/include/partitioning/partprune.h

概要

partprune.h は、PostgreSQL のパーティションプルーニング（不要なパーティションの除外）機能におけるインターフェースを定義するヘッダーファイルです。クエリのプランニング時および実行時に、クエリ条件に基づいてアクセス不要なパーティションを特定するためのデータ構造と関数プロトタイプを提供します。

モジュール内での役割

partitioning モジュールの一部として、partprune.c で実装されるプルーニングロジックの公開 API を定義します。プランナが生成した「プルーニングステップ」を、実行時に PartitionPruneContext を用いて評価し、最終的にスキャン対象となるパーティションの集合（Bitmapset）を決定する役割を担います。

責務

  • パーティションプルーニングに必要な実行時コンテキスト（PartitionPruneContext）の定義。
  • プランニング段階で生成されたプルーニングステップを評価し、対象パーティションを抽出する関数の提供。
  • パーティションキーの比較やハッシュ計算に必要なメタデータ（FmgrInfo 等）の保持。

重要なシンボル

  • PartitionPruneContext: プルーニング実行時に必要な情報を集約した構造体。戦略、境界情報、式評価用のコンテキストなどを保持します。
  • PruneCxtStateIdx(): stepcmpfuncs や exprstates 配列内の特定のステップおよびキーに対するインデックスを計算するマクロです。
  • get_matching_partitions(): 指定されたプルーニングステップに基づき、条件に合致するパーティションのインデックスを返す主要関数です。

制御フローとデータフロー

1. **プランニング時**: make_partition_pruneinfo が呼び出され、クエリの WHERE 句からパーティションキーを特定し、プルーニングステップを生成します。 2. **実行時**: PartitionPruneContext が初期化され、planstate や exprcontext が設定されます。 3. **評価**: get_matching_partitions が呼び出されると、各ステップが順次評価されます。ステップには「ベース（比較）」と「コンバイン（AND/OR）」の2種類があり、これらを組み合わせて最終的な Bitmapset を算出します。

直接依存

  • nodes/execnodes.h: PlanState や ExprState などの実行時ノード定義に依存。
  • partitioning/partdefs.h: PartitionBoundInfo などのパーティション定義の基本構造に依存。

関連コード

  • src/backend/partitioning/partprune.c: 本ヘッダーで宣言された関数の実装本体。
  • src/include/partitioning/partbounds.h: パーティションの境界値管理に関する定義。
  • src/include/partitioning/partdesc.h: パーティション記述子（PartitionDesc）の定義。

読者向けメモ

  • 本ファイルは struct PlannerInfo や struct RelOptInfo を前方宣言することで、ヘッダー間の依存関係を最小限に抑えています。
  • PruneCxtStateIdx マクロは、ステップとキー番号から配列インデックスを算出する重要な計算式です。この計算順序（ステップ単位で連続していること）は、実装上の重要な不変条件となっています。
  • プルーニングはプランニング時と実行時の両方で行われる可能性があるため、PartitionPruneContext がどちらのフェーズで利用されているか（planstate の有無）を意識してコードを読む必要があります。
