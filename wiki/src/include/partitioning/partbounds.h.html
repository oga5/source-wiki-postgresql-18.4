src/include/partitioning/partbounds.h

概要

partbounds.h は、PostgreSQL のパーティション分割テーブルにおける「境界情報（Partition Bounds）」を定義および操作するためのヘッダーファイルです。パーティションの範囲、リスト、ハッシュの各戦略に基づき、特定のデータがどのパーティションに属するかを判定するためのデータ構造と API を提供します。

モジュール内での役割

partitioning サブシステムにおいて、パーティションの定義（partdefs.h）と、実際の実行時検索（partprune.h）を繋ぐ中核的な役割を担います。プランナやエグゼキュータがパーティションの境界を解釈し、クエリ対象のパーティションを特定するための基盤となります。

責務

  • **境界情報の構造化**: PartitionBoundInfoData を通じて、パーティションの境界値、インデックス、デフォルト値、NULL 許容状態を保持する。
  • **境界の生成と検証**: カタログ情報から境界情報を構築し、新しいパーティション追加時の整合性をチェックする。
  • **検索アルゴリズムの提供**: 二分探索（bsearch）を用いた効率的なパーティション特定ロジックの宣言。
  • **結合時のマージ**: パーティション結合（Partition-wise Join）において、結合対象の境界情報を統合する機能の提供。

重要なシンボル

  • PartitionBoundInfoData: パーティション境界情報を保持するメイン構造体。
  • partition_bound_accepts_nulls(bi): NULL を許容するパーティションが存在するかを判定するマクロ。
  • partition_bound_has_default(bi): デフォルトパーティションが存在するかを判定するマクロ。
  • partition_bounds_create(...): パーティション定義から境界情報を生成する主要関数。
  • partition_list_bsearch / partition_range_datum_bsearch: 各戦略に応じた高速な検索関数。

制御フローとデータフロー

1. **初期化**: partition_bounds_create がカタログ上の PartitionBoundSpec を読み込み、ソートされた datums 配列と対応する indexes を構築します。 2. **検索**: クエリ実行時、partition_xxx_bsearch 関数が提供された値（Datum）を境界配列に対して二分探索し、該当するパーティションインデックスを返します。 3. **結合**: partition_bounds_merge がプランナの結合処理中に呼び出され、左右のパーティション境界を比較・統合して新しい境界情報を生成します。

直接依存

  • fmgr.h: 関数マネージャ（比較関数等の呼び出しに使用）。
  • parser/parse_node.h: パーサノードの定義。
  • partitioning/partdefs.h: パーティション戦略やキーの基本定義。

関連コード

  • src/backend/partitioning/partbounds.c: 本ヘッダーで宣言された関数の実装。
  • src/include/partitioning/partprune.h: 本ヘッダーの境界情報を利用してクエリ実行時に不要なパーティションを排除するロジック。
  • src/include/utils/partcache.h: リレーションキャッシュに格納されたパーティション情報の管理。

読者向けメモ

  • PartitionBoundInfoData は非常に複雑な構造をしています。特に indexes 配列の解釈は戦略（Hash/List/Range）によって異なるため、構造体定義直前のコメントを熟読してください。
  • ndatums と nindexes の関係は戦略ごとに異なります（例：Range では nindexes = ndatums + 1）。
  • 本ファイルはプランナとエグゼキュータの両方から参照されるため、変更時は両方の動作への影響を考慮する必要があります。
