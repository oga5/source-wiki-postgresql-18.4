src/include/executor/nodeSetOp.h

概要

nodeSetOp.h は、PostgreSQL のクエリ実行エンジンにおける SetOp ノードのインターフェースを定義するヘッダーファイルです。SetOp ノードは、SQL の INTERSECT および EXCEPT 操作を処理するために使用されます。

モジュール内での役割

src/include/executor モジュールの一部として、プランツリーの実行ノードの一つを定義します。このノードは、入力された2つのリレーション（集合）に対して、重複排除や差分抽出を行うための実行時状態を管理します。

責務

  • SetOp 実行ノードの初期化、終了、および再スキャン（ReScan）のための関数シグネチャを提供します。
  • 実行エンジンが SetOp ノードをプランツリーの一部として統合・操作するための公開 API を定義します。

重要なシンボル

  • ExecInitSetOp: SetOp ノードの実行状態（SetOpState）を初期化し、メモリコンテキストやハッシュテーブル等の準備を行います。
  • ExecEndSetOp: ノードの実行終了時に、割り当てられたメモリやリソースを解放します。
  • ExecReScanSetOp: ノードの状態をリセットし、再実行を可能にします。

制御フローとデータフロー

本ファイル自体は宣言のみですが、対応する nodeSetOp.c では以下のフローが制御されます。

  • **SORTED モード**: 入力がソート済みの場合、マージジョインのように両入力をスキャンし、各グループの出現回数をカウントして結果を生成します。
  • **HASHED モード**: 入力が未ソートの場合、ハッシュテーブルを構築して各グループの出現回数を集計し、最後に結果を生成します。

直接依存

  • nodes/execnodes.h: SetOp および SetOpState 構造体の定義に依存しています。

関連コード

  • src/backend/executor/nodeSetOp.c: 本ヘッダーで宣言された関数の実装が含まれており、実際のロジック（ハッシュテーブル構築や比較処理）が記述されています。
  • src/include/nodes/execnodes.h: 実行ノードの基底構造体や SetOp 固有のプランノード定義が含まれています。

読者向けメモ

  • SetOp は UNION や UNION ALL には使用されません。これらは別の最適化手法（Unique ノード等）で処理されます。
  • SetOp ノードは射影（Projection）やクオリファイア（Qual）のチェックを行いません。入力されたタプルをそのまま利用するため、前段のプランノードで必要な処理が完了している必要があります。
  • ハッシュテーブルを使用する場合、メモリ消費量に注意が必要です。nodeSetOp.c 内の build_hash_table がその制御を担っています。
