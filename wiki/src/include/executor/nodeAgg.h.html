src/include/executor/nodeAgg.h

概要

nodeAgg.h は、PostgreSQL の集約（Aggregate）演算を実行するためのデータ構造と関数プロトタイプを定義するヘッダーファイルです。SQL の GROUP BY や集約関数（SUM, AVG 等）の実行状態を管理するための主要な構造体を提供します。

モジュール内での役割

src/include/executor モジュールにおいて、集約処理の実行エンジン（nodeAgg.c）が使用する型定義の基盤となります。プランナーが生成した Agg ノードを、実行時に AggState として展開・管理するためのインターフェースを定義しています。

責務

  • 集約関数の状態遷移（Transition）および最終計算（Finalization）に必要なメタデータの保持。
  • グループ化、ハッシュ集約、および並列集約のための状態管理構造の定義。
  • 集約実行時のメモリコンテキストやソート状態の管理に必要な情報の提供。

重要なシンボル

  • AggStatePerTransData: 集約の遷移関数（transfn）実行に必要な情報。入力行から中間状態を更新するロジックを保持します。
  • AggStatePerAggData: 最終関数（finalfn）実行に必要な情報。中間状態から最終的な集約結果を生成します。
  • AggStatePerGroupData: グループごとの作業用状態。現在の遷移値（transValue）や NULL 状態を保持します。
  • AggStatePerPhaseData: 複数フェーズにわたる集約（Grouping Sets 等）の各段階の状態を管理します。
  • AggStatePerHashData: ハッシュ集約モードにおけるハッシュテーブルや関連するハッシュ関数の管理構造。

制御フローとデータフロー

集約処理は通常、以下のフローで実行されます： 1. **初期化**: ExecInitAgg により、プランノードから各集約関数のメタデータ（AggStatePerTransData 等）が構築されます。 2. **遷移**: 入力行が来るたびに、AggStatePerGroupData に保持された中間状態に対し、transfn を適用して値を更新します。 3. **ソート/ハッシュ**: DISTINCT や ORDER BY が必要な場合、Tuplesortstate を介して入力を整列させます。 4. **最終化**: グループの境界で finalfn を呼び出し、最終的な結果を生成します。 5. **並列化**: ExecAggEstimate 等の関数を通じて、並列クエリ実行時のワーカー間での状態共有（DSM）をサポートします。

直接依存

  • access/parallel.h: 並列クエリ実行のためのインフラ定義。
  • nodes/execnodes.h: 実行時ノード（AggState 等）の基本定義。

関連コード

  • src/backend/executor/nodeAgg.c: 本ヘッダーで定義された構造体を使用する実装本体。
  • src/include/nodes/execnodes.h: AggState 構造体そのものが定義されており、本ヘッダーの構造体群をメンバとして保持します。

読者向けメモ

  • 本ファイルは集約処理の「静的な定義」に集中しています。実際のロジック（特に transfn の呼び出しやメモリ管理の複雑な挙動）を理解するには、nodeAgg.c の冒頭コメントと ExecAgg 関数の実装を併せて参照してください。
  • AggStatePerTransData と AggStatePerAggData が分離されているのは、複数の集約関数が同じ遷移状態を共有できるようにするためです。この設計により、メモリ効率と計算効率が最適化されています。
