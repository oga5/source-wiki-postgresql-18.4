src/include/executor/nodeWindowAgg.h

概要

nodeWindowAgg.h は、PostgreSQLのクエリ実行エンジンにおけるウィンドウ関数処理（WindowAgg ノード）の初期化、終了、および再スキャンを行うためのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

src/include/executor モジュールの一部として、プランツリーの実行ノードを管理する役割を担います。nodeWindowAgg.c で実装されるウィンドウ関数処理の公開 API を提供し、エグゼキュータがウィンドウ集約処理を呼び出すための窓口となります。

責務

  • WindowAgg ノードの実行状態（WindowAggState）の初期化。
  • 実行終了時のリソース解放。
  • パラメータ変更等に伴う再スキャン（ExecReScan）のサポート。

重要なシンボル

  • ExecInitWindowAgg: WindowAgg ノードの実行状態を構築し、必要なメモリや tuplestore を初期化します。
  • ExecEndWindowAgg: 実行中に確保されたメモリや tuplestore などのリソースを解放します。
  • ExecReScanWindowAgg: ウィンドウ集約の状態をリセットし、再実行を可能にします。
  • WindowAggState: nodeWindowAgg.c で定義される、実行中のウィンドウ集約ノードの状態を保持する構造体です。

制御フローとデータフロー

本ファイル自体は関数プロトタイプのみを定義しています。実際のフローは nodeWindowAgg.c に依存します： 1. **初期化**: ExecInitWindowAgg が呼ばれ、入力タプルを保持するための tuplestore が準備されます。 2. **実行**: 入力タプルがソート済みであることを前提に、パーティション単位でウィンドウ関数が評価されます。 3. **終了**: ExecEndWindowAgg により、集約に使用したメモリコンテキストや一時ストレージがクリーンアップされます。

直接依存

  • nodes/execnodes.h: WindowAgg ノードおよび WindowAggState 構造体の定義に必要です。

関連コード

  • src/backend/executor/nodeWindowAgg.c: 本ヘッダーで宣言された関数の実体が含まれます。ウィンドウ関数の評価ロジックや WindowObject API の詳細が記述されています。
  • src/include/nodes/execnodes.h: WindowAgg ノードのプラン構造体定義が含まれます。

読者向けメモ

  • 本ファイルは非常に軽量なインターフェース定義です。ウィンドウ関数の具体的な挙動（パーティション分割、フレーム定義、tuplestore の使用方法など）を理解するには、必ず対になる nodeWindowAgg.c を参照してください。
  • ウィンドウ関数は入力がソートされていることを前提としているため、プランナが適切な Sort ノードを挿入しているかどうかが実行時の正当性に直結します。
