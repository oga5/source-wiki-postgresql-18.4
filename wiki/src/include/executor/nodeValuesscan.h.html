src/include/executor/nodeValuesscan.h

概要

nodeValuesscan.h は、SQLの VALUES 句（例: VALUES (1, 'a'), (2, 'b')）をスキャンするための実行ノードを定義するヘッダーファイルです。クエリ実行エンジンにおいて、メモリ上の定数リストをタプルとして順次提供する役割を担います。

モジュール内での役割

src/include/executor モジュールの一部として、スキャン系ノード（Scan）の特殊な実装を提供します。テーブルやインデックスを読み取る一般的なスキャンとは異なり、プランツリー内に埋め込まれた定数データを直接処理します。

責務

  • ValuesScan ノードの初期化および再スキャン（ReScan）に必要なインターフェースの公開。
  • 実行時の状態管理を行う ValuesScanState 構造体（execnodes.h で定義）の操作をサポートする関数の宣言。

重要なシンボル

  • ExecInitValuesScan: ValuesScan プランノードから実行時の状態（ValuesScanState）を構築します。
  • ExecReScanValuesScan: スキャン位置をリセットし、再度最初から VALUES リストを読み取れるようにします。

制御フローとデータフロー

本ヘッダーはインターフェース定義のみを提供します。実際のデータフローは nodeValuesscan.c に委譲されています： 1. **初期化**: ExecInitValuesScan が呼び出され、定数リストを保持する状態構造体が準備されます。 2. **実行**: ValuesNext（nodeValuesscan.c 内の内部関数）が ExprContext を使用して、各行の式を評価し、TupleTableSlot に値を格納します。 3. **メモリ管理**: 各行の評価後に ReScanExprContext を呼び出し、メモリ消費を抑制する設計になっています。

直接依存

  • nodes/execnodes.h: ValuesScan および ValuesScanState 構造体の定義に必須。

関連コード

  • src/backend/executor/nodeValuesscan.c: 本ヘッダーで宣言された関数の実装本体。
  • src/include/nodes/execnodes.h: ValuesScan 構造体（プランノード）と ValuesScanState（実行状態）の定義元。

読者向けメモ

  • このノードは、VALUES リスト内の式を評価する際、必要に応じて ExecInitExprList を用いて動的に式評価状態を構築します。
  • VALUES 句は通常、クエリの実行中に一度だけ評価されることが多いため、JIT コンパイルを抑制するなどの最適化が実装レベルで行われています。
  • ValuesScan はテーブルスキャンではないため、ScanDirection に応じたインデックス操作（curr_idx の増減）のみで完結するシンプルな構造です。
