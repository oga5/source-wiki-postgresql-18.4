src/include/executor/nodeTableFuncscan.h

概要

nodeTableFuncscan.h は、PostgreSQL のクエリ実行エンジンにおいて、TABLE 関数（XMLTABLE や JSON_TABLE など）をスキャンするための実行ノードのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

src/include/executor モジュールの一部として、特定のプランノード（TableFuncScan）を初期化・実行・終了させるための公開 API を提供します。このファイルは、実行エンジンがプランツリーを走査する際に呼び出すノード固有のライフサイクル管理を定義しています。

責務

  • TableFuncScanState ノードの初期化（ExecInitTableFuncScan）
  • 実行終了時のリソース解放（ExecEndTableFuncScan）
  • 実行中の再スキャン処理（ExecReScanTableFuncScan）

重要なシンボル

  • ExecInitTableFuncScan: プランノードから実行状態（State）を生成し、必要なメモリやコンテキストを初期化します。
  • TableFuncScanState: TableFuncScan プランノードに対応する実行時の状態を保持する構造体です（定義は execnodes.h にあります）。

制御フローとデータフロー

本ヘッダー自体は宣言のみですが、対応する nodeTableFuncscan.c では以下のフローが制御されます： 1. **初期化**: ExecInitTableFuncScan が呼ばれ、TableFuncScanState が作成されます。 2. **実行**: ExecTableFuncScan が ExecScan を介して呼び出され、TableFuncNext が tuplestore から行を順次取得します。 3. **データ取得**: 初回呼び出し時に tfuncFetchRows が実行され、関数から全行が取得されて tuplestore に格納されます。以降の呼び出しは tuplestore からの読み出しとなります。

直接依存

  • nodes/execnodes.h: TableFuncScan および TableFuncScanState 構造体の定義に必須です。

関連コード

  • src/backend/executor/nodeTableFuncscan.c: 本ヘッダーで宣言された関数の実装が含まれます。
  • src/include/executor/tablefunc.h: TABLE 関数固有の処理ロジックやデータ構造が定義されています。
  • src/include/executor/execScan.h: 汎用的なスキャン実行ルーチン（ExecScan）を提供しており、本ノードの基盤となっています。

読者向けメモ

  • 本ノードは子プランを持たない（outerPlan / innerPlan が NULL）リーフノードとして動作します。
  • XMLTABLE 等の関数は一度の実行で全行を生成して tuplestore に保持する設計であるため、メモリ使用量に注意が必要です。
  • ExecReScanTableFuncScan は、再スキャン時に tuplestore をリセットして再実行を可能にする役割を担います。
