src/include/executor/nodeSort.h

概要

nodeSort.h は、PostgreSQL のクエリ実行エンジンにおけるソート処理（SORT ノード）のインターフェースを定義するヘッダーファイルです。プランツリー内のソート操作を初期化、実行、終了、および再スキャンするための関数群を公開しています。

モジュール内での役割

executor モジュールの一部として、プラン実行時のソートノードのライフサイクルを管理します。実際のソートロジックはペアとなる nodeSort.c に実装されており、このヘッダーは実行エンジンがソートノードを操作するための公開 API として機能します。

責務

  • ソートノード（SortState）の初期化および終了処理の提供。
  • ソート済みデータに対するマーク/リストア（位置の保存と復帰）機能の定義。
  • 並列クエリ実行時における、ソートノードの統計情報収集および DSM（Dynamic Shared Memory）の初期化サポート。

重要なシンボル

  • ExecInitSort: Sort プランノードから SortState を生成し、実行に必要なメモリや状態を初期化します。
  • ExecEndSort: ソート処理で使用したリソース（Tuplesortstate など）を解放します。
  • ExecReScanSort: ソート済みデータを破棄し、再スキャンを可能にするためのリセット処理を行います。
  • SortState: ソートノードの実行時状態を保持する構造体（nodes/execnodes.h で定義）。

制御フローとデータフロー

1. **初期化**: ExecInitSort が呼ばれ、プランノードの情報に基づき SortState が構築されます。 2. **実行**: ExecSort（nodeSort.c 内）が呼び出されると、子ノードから全タプルを読み込み、tuplesort.c を介してメモリまたは一時ファイル上でソートを実行します。 3. **取得**: ソート完了後、要求に応じてソート済みタプルが順次返されます。 4. **終了**: ExecEndSort により、ソートに使用した一時領域が解放されます。

直接依存

  • access/parallel.h: 並列クエリ実行に必要な並列コンテキスト構造体を提供します。
  • nodes/execnodes.h: SortState や Sort プランノードなど、実行エンジンの中核となるデータ構造を定義しています。

関連コード

  • src/backend/executor/nodeSort.c: 本ヘッダーで宣言された関数の実装が含まれます。
  • src/backend/utils/sort/tuplesort.c: 実際のソートアルゴリズム（クイックソートや外部マージソート）を担うコアモジュールです。
  • src/include/nodes/execnodes.h: SortState 構造体の定義元であり、実行状態の理解に不可欠です。

読者向けメモ

  • 本ファイルはインターフェース定義のみです。ソートの具体的な挙動（Datum ソートとタプルソートの使い分けなど）を理解するには、nodeSort.c を参照してください。
  • 並列実行に関連する関数（ExecSortEstimate 等）は、パラレルクエリの実行計画が選択された場合にのみ呼び出されます。
