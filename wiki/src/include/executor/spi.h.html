src/include/executor/spi.h

概要

spi.h は、PostgreSQL の **Server Programming Interface (SPI)** の公開ヘッダーファイルです。SPI は、サーバー内部のコード（主に PL 言語ハンドラや拡張機能）が SQL コマンドを実行し、その結果を処理するためのインターフェースを提供します。

モジュール内での役割

src/include/executor モジュールにおいて、SPI は実行エンジン（Executor）の機能を外部から安全かつ簡便に利用するためのラッパー層として機能します。SQL の解析、計画作成、実行、および結果の取得という一連のライフサイクルを管理します。

責務

  • **SQL 実行の抽象化**: SQL 文字列の実行や、プリペアドプランの実行を簡略化する API を提供。
  • **リソース管理**: SPI 接続のスタック管理、メモリコンテキストの制御、およびトランザクションの境界管理。
  • **データ操作**: 実行結果（SPITupleTable）の保持と、タプル操作（抽出、変換、メモリ解放）のためのユーティリティ関数の提供。
  • **カーソル操作**: サーバーサイドでのカーソルオープン、フェッチ、クローズの制御。

重要なシンボル

  • SPITupleTable: 実行結果のタプル群とメタデータを保持する構造体。
  • SPIPlanPtr: プリペアドプランを指す不透明なポインタ。
  • SPIExecuteOptions: 実行時のパラメータやオプションをカプセル化した構造体。
  • SPI_connect() / SPI_finish(): SPI 呼び出しの開始と終了を管理する関数。
  • SPI_execute() / SPI_execute_plan(): SQL 実行の主要なエントリーポイント。

追加の索引済みシンボル

  • SPIParseOpenOptions
  • SPIPrepareOptions

制御フローとデータフロー

1. **接続**: SPI_connect() を呼び出し、内部スタックを初期化して実行コンテキストを確立します。 2. **準備/実行**: SPI_prepare() でプランを作成し、SPI_execute_plan() で実行します。この際、SPIExecuteOptions を介してパラメータや実行モードを渡します。 3. **結果処理**: 実行結果は SPI_tuptable グローバル変数に格納されます。呼び出し元は SPITupleTable を通じてタプルデータにアクセスします。 4. **終了**: SPI_finish() を呼び出し、割り当てられたメモリやリソースを解放してスタックをポップします。

直接依存

  • commands/trigger.h: トリガー関連のデータ構造。
  • lib/ilist.h: 内部的なリスト管理。
  • parser/parser.h: SQL 解析のためのフック定義。
  • utils/portal.h: カーソル管理のためのポータル定義。

関連コード

  • src/backend/executor/spi.c: 本ヘッダーで宣言された関数の実装。
  • src/include/executor/spi_priv.h: SPI 内部でのみ使用されるプライベートなデータ構造や関数定義。
  • src/backend/executor/executor.h: 実行エンジンのコア機能。

読者向けメモ

  • **グローバル変数**: SPI_processed や SPI_tuptable などのグローバル変数は、SPI 呼び出しのたびに上書きされる可能性があるため、結果を保持したい場合は速やかにコピーする必要があります。
  • **メモリ管理**: SPI は独自のメモリコンテキストを管理します。SPI_palloc などの専用関数を使用することで、SPI 接続終了時に自動的にメモリが解放されるよう設計されています。
  • **トランザクション**: SPI を使用したトランザクション制御（SPI_commit 等）は、呼び出し元のコンテキストに依存するため、使用時には注意が必要です。
