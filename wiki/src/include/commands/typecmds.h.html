src/include/commands/typecmds.h

概要

PostgreSQL におけるデータ型（基本型、ドメイン、列挙型、範囲型、複合型）の定義および変更を行うためのコマンド処理インターフェースを定義するヘッダーファイルです。

モジュール内での役割

src/backend/commands/typecmds.c の公開インターフェースとして機能します。SQL の CREATE TYPE、ALTER TYPE、CREATE DOMAIN などの DDL 文がパーサーによって解析された後、実際のカタログ操作を行うためのエントリポイントを提供します。

責務

  • データ型の作成（DefineType, DefineDomain, DefineEnum 等）の宣言。
  • データ型の変更（AlterType, AlterDomain 等）の宣言。
  • 型の所有権変更や名前空間（スキーマ）移動の管理。
  • 型に関連する OID（配列型やマルチレンジ型）の割り当て管理。

重要なシンボル

  • DefineType, DefineDomain, DefineEnum: 各種データ型を作成する主要関数。
  • AlterType, AlterDomain...: 既存の型やドメインの属性を変更する関数群。
  • DEFAULT_TYPDELIM: 型の配列表現におけるデフォルトの区切り文字（カンマ）。
  • AssignTypeArrayOid: 型の配列用 OID を生成するユーティリティ。

制御フローとデータフロー

1. **解析**: パーサーが SQL を解析し、CreateTypeStmt や AlterTypeStmt などのノードを生成します。 2. **ディスパッチ**: typecmds.h で宣言された関数が呼び出され、typecmds.c 内のロジックへ制御が移ります。 3. **検証と実行**: 各関数は権限チェックを行い、catalog/ 配下の関数（pg_type 等の更新）を呼び出してカタログを書き換えます。 4. **依存関係**: ObjectAddress を介して、作成・変更されたオブジェクトの情報をシステム全体に伝播させます。

直接依存

  • access/htup.h: HeapTuple 構造体（型定義の検索に使用）。
  • catalog/dependency.h: ObjectAddress や DropBehavior など、オブジェクトの依存関係管理用。
  • parser/parse_node.h: ParseState（解析コンテキストの保持に使用）。

関連コード

  • src/backend/commands/typecmds.c: 本ヘッダーで宣言された関数の実装本体。
  • src/include/catalog/pg_type.h: システムカタログ pg_type の定義。
  • src/include/nodes/parsenodes.h: CreateTypeStmt などの DDL 解析ノード定義。

読者向けメモ

  • 本ファイルは主に DDL の実行層を扱います。型の内部表現や型変換のロジックについては src/include/parser/parse_type.h や src/include/utils/lsyscache.h を参照してください。
  • AlterTypeNamespaceInternal などの Internal サフィックスが付いた関数は、再帰的な型変更やシステム内部からの呼び出しを想定した低レベルなインターフェースです。
  • 型の作成順序には依存関係があるため、typecmds.c の冒頭にあるコメント（create function -> create type -> create operator）を理解しておくことが重要です。
