src/include/commands/createas.h

概要

createas.h は、CREATE TABLE AS (CTAS) および CREATE MATERIALIZED VIEW 文の実行を制御するためのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

src/include/commands モジュールの一部として、SQLコマンドの実行フェーズにおける「クエリ結果を新規テーブルへ書き込む」という特殊な処理を抽象化します。

責務

  • CTAS 実行のメインエントリポイントの提供。
  • クエリ結果をリレーションへ流し込むための DestReceiver の生成。
  • CTAS 実行に必要なフラグや存在チェックのユーティリティ提供。

重要なシンボル

  • ExecCreateTableAs: CTAS 文を実行する主要関数。パーサー状態とステートメントを受け取り、リレーションを作成してデータを投入します。
  • CreateIntoRelDestReceiver: クエリの実行結果をテーブルに書き込むための DestReceiver オブジェクトを生成します。
  • GetIntoRelEFlags: IntoClause に基づき、実行時に必要なフラグ（IntoRelEFlags）を算出します。

制御フローとデータフロー

1. パーサーが CREATE TABLE AS 文を解析し、CreateTableAsStmt ノードを生成します。 2. ExecCreateTableAs が呼び出され、ターゲットとなるリレーションの定義（DefineRelation）を行います。 3. クエリ実行エンジンは CreateIntoRelDestReceiver で作成された DestReceiver を使用し、通常の出力先（クライアント等）の代わりに作成したリレーションへタプルを書き込みます。

直接依存

  • catalog/objectaddress.h: 作成されたリレーションの識別に使用。
  • nodes/params.h: パラメータ付きクエリの実行をサポート。
  • parser/parse_node.h: パーサー状態の保持。
  • tcop/dest.h: DestReceiver インターフェースの定義。
  • utils/queryenvironment.h: クエリ実行環境の管理。

関連コード

  • src/backend/commands/createas.c: 本ヘッダーで宣言された関数の実装。DestReceiver の具体的な実装（DR_intorel）が含まれます。
  • src/include/commands/matview.h: マテリアライズド・ビュー関連の共有ロジック。
  • src/include/commands/tablecmds.h: DefineRelation など、テーブル定義の基盤となる関数群。

読者向けメモ

  • 本ファイルは CTAS の実行ロジックの「入り口」です。データが実際にどのようにテーブルへ書き込まれるか（intorel_receive 等）を追う場合は、ペアとなる createas.c の DestReceiver 実装を参照してください。
  • CTAS は歴史的経緯から SELECT INTO と密接に関連しており、マテリアライズド・ビューの作成もこのコードパスを再利用しています。
