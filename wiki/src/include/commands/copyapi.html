src/include/commands/copyapi.h

概要

copyapi.h は、PostgreSQL の COPY コマンドにおけるデータ形式（CSV, Text, Binary 等）の拡張を可能にするためのプラグインインターフェースを定義するヘッダーファイルです。

モジュール内での役割

src/include/commands モジュールにおいて、copy.h が COPY コマンドの全体的な制御構造を定義するのに対し、本ファイルは個別のデータフォーマットが実装すべき「振る舞い」を抽象化する役割を担います。

責務

  • COPY TO（エクスポート）および COPY FROM（インポート）の各フェーズ（開始、行処理、終了）を定義する関数ポインタ構造体の提供。
  • フォーマット固有のロジックをコアエンジンから分離し、プラグイン可能な設計を実現すること。

重要なシンボル

  • CopyToRoutine: COPY TO を実行するためのコールバック関数群を保持する構造体。
  • CopyFromRoutine: COPY FROM を実行するためのコールバック関数群を保持する構造体。
  • CopyToOneRow / CopyFromOneRow: 各フォーマットにおける行単位の読み書きを司る主要なフック。

制御フローとデータフロー

本ファイル自体はインターフェース定義のみであり、実際の制御フローは以下の通りです： 1. **初期化**: COPY コマンド実行時に、選択されたフォーマットに対応する Routine 構造体が参照されます。 2. **開始**: Start コールバックが呼び出され、フォーマット固有の初期化が行われます。 3. **ループ**: OneRow コールバックが繰り返し呼び出され、データがストリームまたはテーブルスロット間で転送されます。 4. **終了**: End コールバックによりリソースの解放や後処理が行われます。

直接依存

  • commands/copy.h: CopyToState および CopyFromState 型の定義を利用するために必須です。

関連コード

  • src/include/commands/copy.h: COPY コマンドのメイン構造体やオプション定義。
  • src/include/commands/copyfrom_internal.h: COPY FROM の内部的な状態管理や実装詳細。

読者向けメモ

  • **静的割り当て**: ここで定義される Routine 構造体は、サーバーの生存期間中保持されるため、通常は static const として定義する必要があります。
  • **拡張性**: 新しいデータフォーマットを追加する際は、これらの構造体を実装し、コアの COPY ロジックから呼び出されるように登録します。
  • **状態管理**: 各コールバックに渡される cstate は、フォーマット固有のプライベートデータへのアクセスポイントとして機能します。
