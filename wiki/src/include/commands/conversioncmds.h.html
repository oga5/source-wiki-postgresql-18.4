src/include/commands/conversioncmds.h

概要

PostgreSQLにおける文字コード変換（CONVERSION）オブジェクトの作成コマンドをサポートするためのヘッダーファイルです。SQLの CREATE CONVERSION 文を実行する際のインターフェースを定義しています。

モジュール内での役割

src/include/commands モジュールの一部として、DDL（データ定義言語）コマンドの実行ロジックを外部に公開する役割を担います。本ファイルは、パーサーが生成した構文解析ツリーを実際のバックエンド処理へ橋渡しする接点となります。

責務

  • CREATE CONVERSION 文の実行に必要な関数プロトタイプの提供。
  • 関連するデータ構造（CreateConversionStmt）および戻り値型（ObjectAddress）の定義のインポート。

重要なシンボル

  • CreateConversionCommand: CREATE CONVERSION 文を処理するメイン関数。パーサーが生成した CreateConversionStmt 構造体を受け取り、システムカタログへの登録と検証を行います。

制御フローとデータフロー

本ヘッダー自体は宣言のみですが、対応する conversioncmds.c では以下のフローで処理が行われます： 1. **名前解決**: QualifiedNameGetCreationNamespace を使用して変換オブジェクトのスキーマを特定。 2. **権限チェック**: 対象スキーマへの CREATE 権限および変換関数の EXECUTE 権限を確認。 3. **検証**: 指定されたソースおよびターゲットのエンコーディングが有効か、また変換関数が適切なシグネチャを持つかを検証。 4. **登録**: 検証完了後、pg_conversion カタログへ新しい変換定義を挿入。

直接依存

  • catalog/objectaddress.h: ObjectAddress 型（DDL操作の結果として返されるオブジェクト識別子）の定義。
  • nodes/parsenodes.h: CreateConversionStmt 構造体（SQLパーサーの出力）の定義。

関連コード

  • src/backend/commands/conversioncmds.c: 本ヘッダーで宣言された関数の実装。
  • src/include/catalog/pg_conversion.h: 文字コード変換を管理するシステムカタログの定義。
  • src/backend/mb/pg_wchar.c: エンコーディング変換の低レベルな処理ロジック。

読者向けメモ

  • 本ファイルは非常にシンプルで、単一の関数宣言のみを含みます。
  • 変換関数は int4 を返す必要があり、内部的に OidFunctionCall6 を用いて検証されるため、関数定義のシグネチャには厳格な制約がある点に注意してください。
  • SQL_ASCII を含む変換はサポートされていないというビジネスロジックが実装層に存在します。
