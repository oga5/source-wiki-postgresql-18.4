src/include/commands/lockcmds.h

概要

lockcmds.h は、SQLの LOCK TABLE コマンドを実行するためのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

src/include/commands モジュールの一部として、SQLコマンドの解析結果（LockStmt）を実際のロック処理へ橋渡しする役割を担います。

責務

  • LOCK TABLE 文の実行に必要な関数プロトタイプの提供。
  • 解析フェーズで生成された LockStmt 構造体を、バックエンドのロック管理機能へ渡すためのエントリーポイントの公開。

重要なシンボル

  • LockTableCommand(LockStmt *lockstmt): LOCK TABLE コマンドを実行するメイン関数。パーサーが生成した LockStmt を受け取り、対象リレーションのロック取得処理を開始します。

制御フローとデータフロー

1. パーサーが LOCK TABLE 文を解析し、LockStmt ノードを作成します。 2. LockTableCommand が呼び出され、LockStmt 内の対象リレーションリストを走査します。 3. 各リレーションに対して RangeVarGetRelidExtended を呼び出し、対象の OID を特定しつつ、コールバック関数経由で権限チェックとロック取得を行います。 4. 継承関係やビューが含まれる場合、再帰的なロック取得処理が実行されます。

直接依存

  • nodes/parsenodes.h: LockStmt 構造体の定義を参照するために必要です。

関連コード

  • src/backend/commands/lockcmds.c: 本ヘッダーで宣言された関数の実装が含まれます。ロック取得のロジックや、継承ツリーの走査、権限チェックの詳細が記述されています。
  • src/include/storage/lmgr.h: 実際のロック管理（LockRelationOid など）を行う低レイヤーのインターフェースです。

読者向けメモ

  • 本ファイルは非常にシンプルで、LockTableCommand の宣言のみを保持しています。
  • 実際のロック取得ロジック（特に継承テーブルやビューに対する再帰的なロック処理）を理解したい場合は、対応する lockcmds.c を参照してください。
  • LOCK TABLE はトランザクション制御と密接に関係しているため、ロックの解放タイミングやトランザクションの挙動については access/xact.h 関連の知識も役立ちます。
