src/include/commands/explain_format.h

概要

explain_format.h は、PostgreSQL の EXPLAIN コマンドが出力する実行計画のフォーマット処理を抽象化するためのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

commands モジュールにおいて、クエリ実行計画の可視化を担う explain サブシステムの中心的な役割を果たします。ExplainState 構造体を利用して、テキスト、JSON、XML、YAML といった異なる出力形式に対して一貫したインターフェースを提供します。

責務

  • **出力の抽象化**: 実行計画の各プロパティ（数値、文字列、リスト等）を、出力形式に依存しない関数呼び出しで記述可能にする。
  • **階層構造の管理**: EXPLAIN 出力におけるネストされたグループ（ノードやプロパティの集合）の開始・終了・保存・復元を制御する。
  • **フォーマット変換の分離**: 呼び出し側が具体的な出力形式（JSON等）を意識せず、論理的なプロパティ名と値のみを渡せるようにする。

重要なシンボル

  • ExplainState: 出力状態を保持する構造体（前方宣言のみ）。
  • ExplainPropertyList / ExplainPropertyListNested: リスト形式のプロパティを出力する。
  • ExplainPropertyText / ExplainPropertyInteger / ExplainPropertyFloat / ExplainPropertyBool: 基本的なデータ型をフォーマットして出力する。
  • ExplainOpenGroup / ExplainCloseGroup: 出力階層の構造化（JSONのオブジェクトやXMLのタグなど）を制御する。

制御フローとデータフロー

1. **初期化**: ExplainBeginOutput で出力ストリームを準備。 2. **データ出力**: 実行計画の各ノードが ExplainProperty* 関数を呼び出し、ExplainState にデータを蓄積。 3. **階層制御**: ExplainOpenGroup / ExplainCloseGroup を用いて、JSONのネストやXMLのタグ階層を構築。 4. **終了**: ExplainEndOutput で最終的なフォーマット（閉じ括弧や終了タグ）を付与。

直接依存

  • nodes/pg_list.h: プロパティリストを扱うための List 型定義。

関連コード

  • src/backend/commands/explain_format.c: 本ヘッダーで宣言された関数の実装。各フォーマット固有のエスケープ処理や整形ロジックが含まれる。
  • src/include/commands/explain_state.h: ExplainState 構造体の定義。
  • src/include/commands/explain.h: EXPLAIN コマンド全体のメインインターフェース。

読者向けメモ

  • 本ファイルは ExplainState の詳細を隠蔽するため、構造体のポインタのみを引数に取ります。
  • ExplainOpenSetAsideGroup などのグループ操作関数は、複雑な実行計画の出力において、一時的に階層を退避・復元するために使用されます。
  • 新しい出力形式を追加する場合、explain_format.c 内の switch 文を網羅的に更新する必要があります。
