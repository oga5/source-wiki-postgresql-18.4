src/include/commands/proclang.h

概要

本ヘッダーファイルは、PostgreSQLにおける手続き型言語（Procedural Language: PL）の定義および管理に関連する関数プロトタイプを定義しています。

モジュール内での役割

src/include/commands モジュールの一部として、SQLコマンド CREATE LANGUAGE や DROP LANGUAGE 等の実行時に呼び出されるバックエンド処理のインターフェースを提供します。

責務

  • 手続き型言語の作成（CreateProceduralLanguage）のための関数定義。
  • 言語名からそのOID（Object Identifier）を解決するためのユーティリティ関数の提供。

重要なシンボル

  • CreateProceduralLanguage(CreatePLangStmt *stmt): CREATE LANGUAGE 文を処理し、システムカタログ pg_language に新しい言語エントリを登録する主要関数。
  • get_language_oid(const char *langname, bool missing_ok): 指定された言語名のOIDを取得する。存在しない場合の挙動を missing_ok フラグで制御可能。

制御フローとデータフロー

  • **データフロー**: CreatePLangStmt 構造体（パーサーが生成）を引数として受け取り、検証を経て ObjectAddress を返します。
  • **制御フロー**:

    • 1. 呼び出し元が CREATE LANGUAGE 文を解析。 2. 本ヘッダーで宣言された CreateProceduralLanguage を呼び出し。 3. 実装側（proclang.c）でハンドラ関数やバリデータ関数の検証を行い、pg_language テーブルを更新。

直接依存

  • catalog/objectaddress.h: ObjectAddress 型の定義に必要。
  • nodes/parsenodes.h: CreatePLangStmt 構造体の定義に必要。

関連コード

  • src/backend/commands/proclang.c: 本ヘッダーで宣言された関数の実装が含まれる。
  • src/include/catalog/pg_language.h: 手続き型言語を保持するシステムカタログの定義。
  • src/include/nodes/parsenodes.h: CreatePLangStmt 構造体の詳細な定義。

読者向けメモ

  • 本ファイルは非常にシンプルであり、主にインターフェースの公開を目的としています。
  • 手続き型言語の追加にはスーパーユーザー権限が必要であるという制約が実装側に存在するため、呼び出し側で権限チェックを重複させる必要はありませんが、実装の前提として理解しておくことが重要です。
  • 言語ハンドラ（plhandler）やバリデータ（plvalidator）の検証ロジックを追う場合は、proclang.c の実装を参照してください。
