src/include/commands/alter.h

概要

src/include/commands/alter.h は、PostgreSQL におけるデータベースオブジェクトの属性変更（ALTER 文）を処理するための共通インターフェースを定義するヘッダーファイルです。

モジュール内での役割

commands モジュール内で、特定のオブジェクト型に依存しない汎用的な ALTER 操作の実行エンジン（alter.c）に対するプロトタイプを提供します。個別のオブジェクト（テーブル、関数など）に対する具体的な変更ロジックと、共通の変更処理を分離する役割を担います。

責務

  • オブジェクトの名称変更（RENAME）の実行。
  • オブジェクトのスキーマ移動（ALTER ... SET SCHEMA）の実行。
  • オブジェクトの所有者変更（ALTER ... OWNER TO）の実行。
  • 依存関係（DEPENDS ON EXTENSION）の更新処理。

重要なシンボル

  • ExecRenameStmt: RenameStmt を受け取り、オブジェクト名を変更する。
  • ExecAlterObjectSchemaStmt: AlterObjectSchemaStmt を受け取り、スキーマ変更を実行する。
  • ExecAlterOwnerStmt: AlterOwnerStmt を受け取り、所有者を変更する。
  • AlterObjectNamespace_oid: 指定されたオブジェクトを新しい名前空間（スキーマ）へ移動させる低レベル関数。

制御フローとデータフロー

1. パーサーが ALTER 文を解析し、parsenodes.h で定義された Stmt 構造体を生成します。 2. process_utility 経由で、これらの Exec... 関数が呼び出されます。 3. 各関数は ObjectAddress を用いて対象オブジェクトを特定し、カタログ（pg_class 等）を更新します。 4. 変更が完了すると、新しい ObjectAddress が返されます。

直接依存

  • catalog/dependency.h: 依存関係の管理に使用。
  • catalog/objectaddress.h: オブジェクトを一意に特定するための構造体定義。
  • nodes/parsenodes.h: RenameStmt や AlterOwnerStmt などの構文解析ノード定義。

関連コード

  • src/backend/commands/alter.c: 本ヘッダーで宣言された関数の実装本体。
  • src/backend/commands/tablecmds.h: テーブル固有の ALTER 操作の詳細。
  • src/include/catalog/objectaddress.h: オブジェクト識別子（Oid）とクラス ID の管理。

読者向けメモ

  • 本ファイルは「汎用的な」変更処理を扱います。特定のオブジェクト型（テーブル、インデックス等）に特化した複雑なロジックは、各オブジェクト専用の commands ファイル（例: tablecmds.c）に委譲されていることが多い点に注意してください。
  • AlterObjectNamespace_oid は、スキーマ移動に伴う依存関係の再構築や権限チェックを伴うため、呼び出し時には適切なロックが保持されていることを確認してください。
