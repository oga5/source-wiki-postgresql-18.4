src/include/commands/tablecmds.h

概要

本ファイルは、PostgreSQLにおけるテーブルおよび関連するリレーション（インデックス、シーケンス、ビュー等）の定義・変更・削除を司るコマンド群のインターフェースを定義しています。

モジュール内での役割

src/include/commands モジュールにおいて、DDL（Data Definition Language）実行の中核を担うヘッダーです。tablecmds.c と対になっており、パーサーが生成した構文解析ツリー（CreateStmt, AlterTableStmt 等）を実際のカタログ操作やストレージ操作へと変換するブリッジの役割を果たします。

責務

  • **テーブル定義の管理**: DefineRelation による新規作成、RemoveRelations による削除。
  • **スキーマ変更の実行**: AlterTable を中心とした、カラム追加・変更、所有者変更、名前変更、名前空間（スキーマ）移動の提供。
  • **トランザクション制御**: ON COMMIT アクション（TRUNCATE や DROP の遅延実行）の登録とライフサイクル管理。
  • **整合性チェック**: テーブル使用状況の確認や、依存関係の検証。

重要なシンボル

  • DefineRelation: CREATE TABLE 等の実行時に呼び出され、カタログへの登録と初期化を行う。
  • AlterTable: ALTER TABLE 文のディスパッチャー。サブコマンドのリストを解析し、適切な内部関数を呼び出す。
  • ExecuteTruncate: TRUNCATE コマンドの実行エントリーポイント。
  • OnCommitAction: トランザクション終了時に実行されるアクション（PRESERVE ROWS, DELETE ROWS, DROP）の管理用関数群。

制御フローとデータフロー

1. **解析**: パーサーが SQL を Stmt 構造体（CreateStmt, AlterTableStmt 等）に変換。 2. **ディスパッチ**: tablecmds.h で定義された関数が呼び出され、tablecmds.c 内のロジックへ制御が移る。 3. **カタログ操作**: pg_class, pg_attribute, pg_constraint 等のシステムカタログを更新。 4. **依存関係解決**: ObjectAddress を介して、他のオブジェクトとの依存関係を pg_depend に記録。 5. **ストレージ反映**: 必要に応じて smgr（ストレージマネージャ）を呼び出し、物理的なファイル操作を実行。

直接依存

  • access/htup.h: ヒープタプルの操作用。
  • catalog/dependency.h / catalog/objectaddress.h: オブジェクト間の依存関係管理用。
  • nodes/parsenodes.h: 構文解析ツリーの定義。
  • storage/lock.h: DDL実行時のロックモード定義。
  • utils/relcache.h: リレーションキャッシュの操作用。

関連コード

  • src/backend/commands/tablecmds.c: 本ヘッダーで宣言された関数の実装本体。
  • src/include/nodes/parsenodes.h: DDL文を表現するデータ構造の定義。
  • src/backend/catalog/: カタログ操作の低レベルAPI。

読者向けメモ

  • 本ファイルは非常に広範な機能をカバーしています。特定の操作（例：ALTER TABLE の詳細）を追う場合は、tablecmds.c 内の ATExec... で始まる関数群を併せて参照してください。
  • AlterTable は非常に複雑な関数であり、サブコマンドごとに処理が分岐します。デバッグ時は、AlterTableStmt 内の cmds リストがどのように処理されるかを追跡するのが定石です。
  • ON COMMIT 関連の関数は、トランザクションのコミット/アボート時に呼び出されるため、トランザクション管理の理解が不可欠です。
