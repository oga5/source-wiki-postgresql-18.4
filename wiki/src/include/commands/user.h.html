src/include/commands/user.h

概要

user.h は、PostgreSQL におけるロール（旧称：ユーザー）の管理操作を定義するヘッダーファイルです。ロールの作成、変更、削除、および権限付与（GRANT/REVOKE）に関連する SQL コマンドのバックエンド処理インターフェースを提供します。

モジュール内での役割

src/include/commands モジュールの一部として、ロール管理というデータベースのセキュリティ基盤を担います。src/backend/commands/user.c と対になっており、SQL パーサーが生成した構文解析ツリー（Stmt 構造体）を実際のシステムカタログ操作へ変換する役割を持ちます。

責務

  • ロール操作（CREATE, ALTER, DROP）の実行関数の宣言。
  • ロール間の権限付与および取り消し（GRANT, REVOKE）のインターフェース提供。
  • 所有オブジェクトの管理（DROP OWNED, REASSIGN OWNED）の定義。
  • パスワード暗号化設定やロール作成時のフック機構の管理。

重要なシンボル

  • CreateRole, AlterRole, DropRole: ロールのライフサイクルを管理する主要関数。
  • GrantRole: ロール間のメンバーシップ付与を処理する関数。
  • check_password_hook: パスワードの妥当性検証を外部モジュールから注入するためのフック。
  • Password_encryption: パスワードの暗号化アルゴリズムを指定する GUC パラメータ。

追加の索引済みシンボル

  • Datum

制御フローとデータフロー

1. **解析**: SQL 文がパーサーによって CreateRoleStmt や GrantRoleStmt などのノードに変換されます。 2. **ディスパッチ**: user.h で宣言された関数が呼び出され、user.c 内のロジックが実行されます。 3. **カタログ操作**: pg_authid や pg_auth_members などのシステムカタログを更新し、必要に応じて依存関係を解決します。 4. **検証**: check_password_hook が設定されている場合、ロール作成や変更時にパスワードの複雑性チェックが実行されます。

直接依存

  • catalog/objectaddress.h: オブジェクトの識別と依存関係管理に使用。
  • libpq/crypt.h: パスワード暗号化タイプ（PasswordType）の定義。
  • nodes/parsenodes.h: SQL 文に対応する構文解析ノードの定義。
  • parser/parse_node.h: 解析状態（ParseState）の管理。
  • utils/guc.h: 設定パラメータ（GUC）の管理。

関連コード

  • src/backend/commands/user.c: 本ヘッダーで宣言された関数の実装。
  • src/include/catalog/pg_authid.h: ロール情報を保持するシステムカタログの定義。
  • src/include/utils/acl.h: アクセス制御リスト（ACL）および権限チェックの基盤。

読者向けメモ

  • 本ファイルで宣言されている関数は、主に ProcessUtility 経由で呼び出されます。
  • DropOwnedObjects や ReassignOwnedObjects は、ロールを削除する前に所有オブジェクトを整理する際に使用される重要なユーティリティです。
  • ロールの権限付与（GrantRole）は、再帰的な権限取り消しが発生する可能性があるため、実装は user.c 内の複雑な依存関係解決ロジックに依存しています。
