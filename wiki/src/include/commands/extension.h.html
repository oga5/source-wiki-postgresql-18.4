src/include/commands/extension.h

概要

PostgreSQLの拡張機能（Extension）を管理するためのコマンド群を定義するヘッダーファイルです。CREATE EXTENSION、ALTER EXTENSION、DROP EXTENSION などのSQLコマンドの実行に必要なインターフェースを提供します。

モジュール内での役割

src/include/commands モジュールの一部として、拡張機能のライフサイクル管理を担います。SQLパーサーが生成した構文木（Stmt）を実際のカタログ操作へ変換するブリッジの役割を果たします。

責務

  • 拡張機能の作成、変更、削除のための関数定義。
  • 拡張機能のメタデータ（OID、名前、スキーマ）を取得するユーティリティ関数の提供。
  • 拡張機能インストール中であることを示すグローバル状態（creating_extension）の管理。

重要なシンボル

  • creating_extension: 拡張機能のインストール/更新中であることを示すフラグ。このフラグが有効な間、作成されるオブジェクトは自動的に拡張機能のメンバーとして登録されます。
  • CurrentExtensionObject: 現在処理中の拡張機能のOID。
  • CreateExtension(): CREATE EXTENSION コマンドのメインエントリーポイント。
  • ExecAlterExtensionStmt(): ALTER EXTENSION コマンドの実行を制御します。
  • get_extension_oid(): 拡張機能名からOIDを解決する頻出ユーティリティ。

制御フローとデータフロー

1. **コマンド実行**: パーサーが CreateExtensionStmt などのノードを生成し、extension.c 内の対応する関数が呼び出されます。 2. **状態設定**: creating_extension が true に設定され、後続のオブジェクト作成処理（テーブル作成や関数定義など）が、この拡張機能への依存関係を自動的に記録するようにします。 3. **カタログ更新**: InsertExtensionTuple 等を通じて pg_extension カタログが更新され、拡張機能の構成情報が永続化されます。 4. **終了**: コマンド完了後、フラグがリセットされます。

直接依存

  • catalog/objectaddress.h: 拡張機能が管理するオブジェクトの識別に使用。
  • parser/parse_node.h: ParseState 構造体を通じて、SQL解析時のコンテキスト情報を保持。

関連コード

  • src/backend/commands/extension.c: 本ヘッダーで宣言された関数の実装。拡張機能の制御ファイル（.control）の解析や、インストールスクリプトの実行ロジックが含まれます。
  • catalog/pg_extension.h: 拡張機能のシステムカタログ定義。

読者向けメモ

  • creating_extension フラグは、拡張機能のインストールスクリプト実行時に非常に重要な役割を果たします。このフラグが立っている間は、システムが自動的に依存関係を追跡するため、開発者が手動で pg_depend を操作する必要はありません。
  • 拡張機能の更新（ALTER EXTENSION UPDATE）も同様のフローを通るため、更新スクリプトの実行時にもこのフラグが活用されます。
