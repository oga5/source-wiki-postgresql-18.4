src/include/commands/collationcmds.h

概要

本ヘッダーファイルは、PostgreSQLにおける照合順序（Collation）関連のDDLコマンドを処理するための関数プロトタイプを定義しています。

モジュール内での役割

src/include/commands モジュールの一部として、照合順序の定義（CREATE COLLATION）および変更（ALTER COLLATION）をバックエンドの実行系へ橋渡しするインターフェースを提供します。

責務

  • 照合順序の作成、変更、および存在確認を行う関数の宣言。
  • SQLパーサーが生成した構文解析ツリー（AlterCollationStmt等）を、実行エンジンが処理可能な形式へ変換するための入り口を提供。

重要なシンボル

  • DefineCollation: CREATE COLLATION 文を実行するためのメイン関数。
  • AlterCollation: ALTER COLLATION 文を実行するためのメイン関数。
  • IsThereCollationInNamespace: 指定された名前空間内に特定の照合順序が存在するかを検証するユーティリティ。

制御フローとデータフロー

1. SQLパーサーが CREATE または ALTER 文を解析し、構文解析ツリーを生成します。 2. 実行エンジンが本ヘッダーで宣言された関数を呼び出します。 3. DefineCollation は ParseState を受け取り、parameters リストを走査して照合順序の属性（locale, provider, deterministic 等）を抽出します。 4. 最終的に pg_collation カタログへの書き込みを行い、ObjectAddress を返して操作を完了します。

直接依存

  • catalog/objectaddress.h: ObjectAddress 構造体の定義に使用。
  • parser/parse_node.h: ParseState 構造体の定義に使用。

関連コード

  • src/backend/commands/collationcmds.c: 本ヘッダーで宣言された関数の実装本体。
  • src/include/nodes/parsenodes.h: AlterCollationStmt などの構文解析ノード定義。
  • src/include/catalog/pg_collation.h: 照合順序を保持するシステムカタログ定義。

読者向けメモ

  • 本ファイルは非常に軽量であり、主にインターフェースの定義に特化しています。
  • 照合順序の属性（lc_collate, lc_ctype など）の検証ロジックは、すべて collationcmds.c 側に集約されています。
  • if_not_exists フラグの扱いや、FROM 句による既存照合順序のコピー処理については、実装側の DefineCollation を参照してください。
