src/include/commands/dbcommands.h

概要

dbcommands.h は、PostgreSQL におけるデータベースレベルの管理コマンド（作成、削除、名前変更、属性変更）を定義するヘッダーファイルです。データベースのライフサイクル管理に必要なインターフェースを提供します。

モジュール内での役割

src/include/commands モジュールの一部として、SQL コマンドの実行層とバックエンドの物理的なデータベース操作を結びつける役割を担います。実際のロジックは src/backend/commands/dbcommands.c に実装されています。

責務

  • データベースの作成 (createdb) および削除 (dropdb, DropDatabase) の実行。
  • データベースの属性変更（名前変更、所有者変更、設定変更、照合順序の更新）の提供。
  • データベースの OID や名前の解決、権限チェックなどのユーティリティ機能の提供。

重要なシンボル

  • createdb(): データベース作成のメインエントリポイント。
  • dropdb() / DropDatabase(): データベース削除の実行関数。
  • AlterDatabase(): データベースの各種プロパティ変更を処理。
  • get_database_oid() / get_database_name(): データベース名と OID の相互変換を行うアクセサ。

制御フローとデータフロー

SQL パーサーが CREATE DATABASE や DROP DATABASE などの文を解析すると、対応する Stmt 構造体が生成されます。これらの関数は ParseState を引数に取り、システムカタログ（pg_database 等）を参照・更新しながら、物理的なディレクトリ操作やトランザクション制御を行います。

直接依存

  • access/xlogreader.h: WAL 関連の操作用。
  • catalog/objectaddress.h: データベースオブジェクトの識別用。
  • lib/stringinfo.h: 文字列操作用。
  • parser/parse_node.h: パーサーの状態管理用。

関連コード

  • src/backend/commands/dbcommands.c: 本ヘッダーで宣言された関数の実装。
  • src/include/commands/dbcommands_xlog.h: データベース操作に関連する WAL レコードの定義。
  • src/catalog/pg_database.h: データベース情報を保持するシステムカタログの定義。

読者向けメモ

  • データベースの作成・削除は非常に重い操作であり、排他ロック（LockSharedObject）を使用して競合を回避しています。
  • データベースの物理的なコピーには CREATEDB_WAL_LOG と CREATEDB_FILE_COPY の2つの戦略が存在し、実装側で使い分けられています。
  • データベースの削除時には、接続中のバックエンドが存在しないことを確認するプロセスが含まれます。
