src/include/commands/portalcmds.h

概要

portalcmds.h は、PostgreSQL における SQL カーソル操作（DECLARE CURSOR, FETCH, CLOSE）を実装するためのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

src/include/commands モジュールの一部として、SQL コマンドの実行フェーズにおける「ポータル（Portal）」関連のユーティリティ関数を提供します。ポータルは、クエリの実行状態を保持し、クライアントへ結果を段階的に返すための抽象化レイヤーです。

責務

  • カーソルのオープン、フェッチ、クローズといったライフサイクル管理の宣言。
  • ポータルのクリーンアップおよび、トランザクション終了後も結果を保持する「ホールド可能（Holdable）」なポータルの永続化処理の定義。

重要なシンボル

  • PerformCursorOpen: DECLARE CURSOR 文を実行し、新しいポータルを作成・初期化します。
  • PerformPortalFetch: 指定されたポータルからデータを取得し、DestReceiver を介して結果を送信します。
  • PerformPortalClose: 指定された名前のポータルを破棄します。
  • PersistHoldablePortal: トランザクション終了時にポータルの状態をメモリからディスク等へ退避させ、結果を保持可能にします。

制御フローとデータフロー

本ファイルは宣言のみを提供します。実際のフローは src/backend/commands/portalcmds.c に委譲されます。 1. **オープン**: PerformCursorOpen が ParseState と DeclareCursorStmt を受け取り、クエリの書き換えとプラン作成を経て CreatePortal を呼び出します。 2. **フェッチ**: PerformPortalFetch がポータル内のプランを実行し、結果を DestReceiver へ流し込みます。 3. **クローズ**: PerformPortalClose がポータルを検索し、メモリリソースを解放します。

直接依存

  • nodes/parsenodes.h: DeclareCursorStmt や FetchStmt などの構文解析ノード定義。
  • parser/parse_node.h: ParseState 構造体。
  • utils/portal.h: Portal 構造体およびポータル管理用の基本型。

関連コード

  • src/backend/commands/portalcmds.c: 本ヘッダーの関数実装。
  • src/backend/tcop/pquery.c: ポータルの実行ロジック（FE/BE プロトコル層）を担う関連モジュール。
  • src/backend/utils/mmgr/portalmem.c: ポータルのメモリ管理を担当する基盤層。

読者向けメモ

  • 本ファイルで定義される関数は、主に SQL レベルのカーソル操作を対象としています。プロトコルレベルのポータル操作（BIND や EXECUTE メッセージなど）については tcop/pquery.c を参照してください。
  • PerformCursorOpen を読む際は、DeclareCursorStmt のオプション（CURSOR_OPT_HOLD など）がどのようにポータルの属性に反映されるかに注目すると理解が深まります。
