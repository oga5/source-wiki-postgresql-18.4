src/include/commands/explain.h

概要

explain.h は、PostgreSQL の EXPLAIN コマンドを実行するための主要なインターフェースと、外部プラグインが実行計画の出力に介入するためのフックを定義するヘッダーファイルです。

モジュール内での役割

src/include/commands モジュールにおいて、クエリの実行計画を解析・表示する機能の中核を担います。explain_state.h で定義される ExplainState 構造体を利用し、実行計画のツリーを走査して整形された出力を生成する役割を持ちます。

責務

  • EXPLAIN コマンドの実行フロー（ExplainQuery）の定義。
  • 実行計画（PlannedStmt）およびユーティリティ文に対する説明生成のインターフェース提供。
  • 外部拡張モジュールが EXPLAIN の出力内容をカスタマイズするためのフック（ExplainOneQuery_hook 等）の提供。
  • 実行計画の各ノード、JIT 統計、トリガー情報などを出力するための関数群の宣言。

重要なシンボル

  • ExplainState: 実行計画の出力設定やバッファを保持する構造体（詳細は explain_state.h）。
  • ExplainQuery: EXPLAIN 文を処理するエントリーポイント。
  • ExplainOnePlan: 計画済みのクエリ（PlannedStmt）に対して説明を出力する主要関数。
  • ExplainOneQuery_hook_type: 外部プラグインが EXPLAIN の挙動を拡張するための関数ポインタ型。

追加の索引済みシンボル

  • Oid
  • QueryEnvironment

制御フローとデータフロー

1. ExplainQuery が EXPLAIN 文を受け取り、クエリを解析・計画します。 2. ExplainOnePlan が PlannedStmt を受け取り、ExplainState に格納されたフォーマット設定（JSON, YAML, Text 等）に従って計画ツリーを再帰的に走査します。 3. 走査中、必要に応じて explain_per_node_hook などのフックが呼び出され、プラグインによる追加情報の挿入が行われます。 4. 最終的に ExplainState が保持する出力先に結果が書き出されます。

直接依存

  • executor/executor.h: 実行計画のノード構造（PlanState）や実行環境の定義。
  • parser/parse_node.h: 解析状態（ParseState）の定義。

関連コード

  • src/backend/commands/explain.c: 本ヘッダーで宣言された関数の実装本体。
  • src/include/commands/explain_state.h: ExplainState 構造体の定義。
  • src/include/commands/explain_format.h: 出力フォーマット（JSON/XML/YAML）に関する定義。
  • src/include/commands/explain_dr.h: DestReceiver を用いた説明出力の制御。

読者向けメモ

  • 本ファイルはインターフェース定義に特化しています。実装の詳細や ExplainState の内部構造を追う場合は、explain.c および explain_state.h を参照してください。
  • フック（PGDLLIMPORT で宣言された変数）を利用することで、標準の EXPLAIN 出力に独自の統計情報やメタデータを追加可能です。拡張機能開発時には特に重要となります。
