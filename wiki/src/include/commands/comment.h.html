src/include/commands/comment.h

概要

本ファイルは、PostgreSQLのSQLコマンド COMMENT ON を実行するためのバックエンド関数のプロトタイプ定義を提供します。データベースオブジェクトに対するコメントの作成、削除、および取得を行うためのインターフェースを定義しています。

モジュール内での役割

src/include/commands モジュールの一部として、DDLコマンドの実行ロジックを抽象化する役割を担います。src/backend/commands/comment.c と対になっており、パーサーが生成した CommentStmt ノードを実際のカタログ操作へと橋渡しします。

責務

  • COMMENT ON 文の実行エントリーポイントの提供。
  • オブジェクトに対するコメントの永続化（pg_description への書き込み）。
  • クラスタ全体で共有されるオブジェクト（データベース、ロール等）に対するコメント操作（pg_shdescription への書き込み）の分離。
  • 特定オブジェクトに関連付けられたコメントの削除および取得機能の提供。

重要なシンボル

  • CommentObject(CommentStmt *stmt): COMMENT ON コマンドを処理する主要な関数。
  • CreateComments / DeleteComments: 通常のオブジェクトに対するコメント操作関数。
  • CreateSharedComments / DeleteSharedComments: 共有カタログ（pg_shdescription）に対するコメント操作関数。
  • GetComment: 指定されたオブジェクトのコメントを取得する関数。

制御フローとデータフロー

1. **解析**: パーサーが CommentStmt 構造体を生成し、CommentObject に渡されます。 2. **解決**: get_object_address を介して、SQL上のオブジェクト指定がシステムカタログ上の ObjectAddress に変換されます。 3. **検証**: 所有権の確認および、対象オブジェクトがコメントを許可されているか（例: インデックス列へのコメント禁止）のチェックが行われます。 4. **永続化**: オブジェクトの種類に応じて、pg_description または pg_shdescription カタログへコメント文字列が書き込まれます。

直接依存

  • catalog/objectaddress.h: ObjectAddress 構造体の定義。
  • nodes/parsenodes.h: CommentStmt 構造体の定義。

関連コード

  • src/backend/commands/comment.c: 本ヘッダーで宣言された関数の実装。
  • src/include/catalog/pg_description.h: コメントを保持するシステムカタログの定義。
  • src/include/catalog/pg_shdescription.h: 共有オブジェクトのコメントを保持するシステムカタログの定義。

読者向けメモ

  • 本モジュールは、コメントの「作成」と「削除」を同一の関数インターフェース（comment 引数が NULL か否か）で扱う設計になっています。
  • 共有オブジェクト（pg_shdescription）と通常オブジェクト（pg_description）で関数が分かれているため、呼び出し時には対象オブジェクトのスコープを意識する必要があります。
  • CommentObject は内部で get_object_address を呼び出し、対象オブジェクトに対するロックを取得するため、並行する DROP 操作との競合は適切に管理されます。
