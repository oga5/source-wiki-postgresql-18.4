src/include/commands/schemacmds.h

概要

schemacmds.h は、PostgreSQL におけるスキーマ（Schema）の作成、名前変更、および所有者変更を行うためのコマンド実行関数のプロトタイプを定義するヘッダーファイルです。

モジュール内での役割

src/include/commands モジュールの一部として、SQL の CREATE SCHEMA、ALTER SCHEMA 文を処理するためのインターフェースを提供します。このヘッダーは、パーサーが生成した構文木（Parse Tree）を実際のデータベース操作に変換するバックエンド処理（schemacmds.c）への窓口となります。

責務

  • スキーマ操作に関連する公開関数の宣言。
  • スキーマの作成（CreateSchemaCommand）および属性変更（RenameSchema, AlterSchemaOwner）のシグネチャ定義。
  • 関連するデータ構造（CreateSchemaStmt 等）の利用に必要な依存関係の提供。

重要なシンボル

  • CreateSchemaCommand: CREATE SCHEMA 文を実行するメイン関数。スキーマの作成、権限チェック、およびスキーマ内のサブコマンド処理を統括します。
  • RenameSchema: スキーマ名を変更する関数。
  • AlterSchemaOwner: スキーマの所有者を変更する関数。

制御フローとデータフロー

1. **パーサー**: SQL 文を解析し、CreateSchemaStmt などのノードを生成します。 2. **コマンド実行**: CreateSchemaCommand が呼び出され、所有権の検証、名前の妥当性チェック（pg_ 接頭辞の保護など）を行います。 3. **カタログ更新**: 内部で namespace.c やカタログ操作関数を呼び出し、pg_namespace カタログを更新してスキーマを永続化します。

直接依存

  • catalog/objectaddress.h: ObjectAddress 構造体の定義。
  • nodes/parsenodes.h: CreateSchemaStmt などの構文解析ノード定義。

関連コード

  • src/backend/commands/schemacmds.c: 本ヘッダーで宣言された関数の実装本体。
  • src/include/catalog/pg_namespace.h: スキーマ情報を保持するシステムカタログの定義。
  • src/include/catalog/namespace.h: 名前空間（スキーマ）の検索や管理を行うユーティリティ関数群。

読者向けメモ

  • 本ファイルは非常に軽量であり、主にインターフェースの定義に特化しています。
  • スキーマ操作は単なるカタログ更新だけでなく、権限チェック（aclcheck）やイベントトリガーのフックなど、PostgreSQL のセキュリティモデルと深く統合されています。実装の詳細を追う際は、schemacmds.c 内の CreateSchemaCommand から読み始めることを推奨します。
