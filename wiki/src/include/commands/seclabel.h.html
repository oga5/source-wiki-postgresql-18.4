src/include/commands/seclabel.h

概要

PostgreSQLのセキュリティラベル機能（SELinuxやAppArmor等の外部セキュリティモジュールと連携するためのラベル付け機構）を管理するためのヘッダーファイルです。

モジュール内での役割

src/include/commands ディレクトリの一部として、SQLコマンド SECURITY LABEL の実行および、外部セキュリティプロバイダーがデータベースオブジェクトに対してラベルを付与・管理するためのインターフェースを提供します。

責務

  • セキュリティラベルの取得、設定、削除を行うための内部APIの定義。
  • SECURITY LABEL 文を実行するためのエントリポイントの提供。
  • 外部セキュリティプロバイダーが自身のフック関数を登録するためのメカニズムの提供。

重要なシンボル

  • ExecSecLabelStmt: SECURITY LABEL SQL文を処理するメイン関数。
  • register_label_provider: セキュリティラベルプロバイダーが自身の検証ロジック（フック）を登録するために使用する関数。
  • check_object_relabel_type: ラベル変更時に呼び出されるコールバック関数の型定義。
  • GetSecurityLabel / SetSecurityLabel: 特定のオブジェクトに対するラベルの読み書きを行うAPI。

追加の索引済みシンボル

  • ObjectAddress

制御フローとデータフロー

1. **登録**: 拡張モジュールが register_label_provider を呼び出し、プロバイダー名と検証用フックを登録します。 2. **実行**: SECURITY LABEL 文が発行されると、ExecSecLabelStmt が呼び出されます。 3. **検証**: プロバイダーが指定されているか確認し、対象オブジェクトがラベル付けをサポートしているかを検証します。 4. **適用**: 登録されたフック関数を介してラベルの妥当性がチェックされ、システムカタログ（pg_seclabel 等）が更新されます。

直接依存

  • catalog/objectaddress.h: データベースオブジェクトを一意に特定するための ObjectAddress 構造体を提供します。

関連コード

  • src/backend/commands/seclabel.c: 本ヘッダーで宣言された関数の実装が含まれます。
  • src/include/catalog/pg_seclabel.h: セキュリティラベルを保持するシステムカタログの定義。

読者向けメモ

  • 本ファイルは主に拡張モジュール開発者や、セキュリティサブシステムの内部構造を理解したいエンジニア向けです。
  • SecLabelStmt 構造体自体はパーサーによって生成されるノードであり、本ヘッダーには定義されていません（nodes/parsenodes.h を参照してください）。
  • ラベルの付与は、対象オブジェクトの所有者権限や、プロバイダー側の検証ロジックによって厳格に制御されます。
