src/include/commands/explain_dr.h

概要

explain_dr.h は、EXPLAIN (SERIALIZE) オプションで使用される DestReceiver（クエリ結果の宛先受信機）に関連するデータ構造と関数プロトタイプを定義するヘッダーファイルです。

モジュール内での役割

src/include/commands モジュールの一部として、クエリ実行結果の処理を拡張する役割を担います。通常の DestReceiver が結果をクライアントへ送信するのに対し、本モジュールは「シリアライズ処理のオーバーヘッド測定」に特化した特殊な受信機を提供します。

責務

  • EXPLAIN (SERIALIZE) 実行時に、データ変換（deTOASTing や型出力関数）にかかるコストを計測するためのインターフェースを提供。
  • シリアライズされたデータのバイト数、所要時間、バッファ使用量を保持する構造体 SerializeMetrics の定義。
  • DestReceiver を生成し、計測結果を取得するための API の公開。

重要なシンボル

  • SerializeMetrics: シリアライズ処理の計測結果を格納する構造体。
  • CreateExplainSerializeDestReceiver(): 計測機能を持つ DestReceiver を生成するコンストラクタ。
  • GetSerializationMetrics(): 実行済みの DestReceiver から計測結果を抽出する関数。

制御フローとデータフロー

本ファイル自体は定義のみを提供します。実際のフローは src/backend/commands/explain_dr.c に依存します： 1. EXPLAIN (SERIALIZE) が実行されると、CreateExplainSerializeDestReceiver が呼び出され、専用の DestReceiver が初期化されます。 2. クエリ実行中、各タプルが受信機に渡されるたびに、シリアライズ処理（serializeAnalyzeReceive）が実行され、SerializeMetrics に統計情報が蓄積されます。 3. 最終的に GetSerializationMetrics を通じて、収集された統計情報が EXPLAIN の出力結果として利用されます。

直接依存

  • executor/instrument.h: 計測用データ構造（instr_time, BufferUsage）の利用。
  • tcop/dest.h: DestReceiver 基底構造体の定義。

関連コード

  • src/backend/commands/explain_dr.c: 本ヘッダーで宣言された関数の実装。
  • src/include/commands/explain.h: ExplainState 構造体の定義元。
  • src/backend/commands/explain.c: EXPLAIN コマンドのメインロジック。

読者向けメモ

  • 本ヘッダーは explain.h を直接インクルードせず、前方宣言（struct ExplainState）を使用することで依存関係を最小限に抑えています。
  • この受信機はデータをクライアントへ送信しません。あくまで「シリアライズ処理のコストを測定する」という目的のために、ネットワーク送信をシミュレートする特殊な DestReceiver である点に注意してください。
