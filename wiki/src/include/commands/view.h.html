src/include/commands/view.h

概要

src/include/commands/view.h は、PostgreSQL におけるビュー（View）の定義および管理を行うためのインターフェースを定義するヘッダーファイルです。ビューの作成、更新、および内部的なクエリ保存処理の入り口を提供します。

モジュール内での役割

src/include/commands モジュールの一部として、SQL の CREATE VIEW および CREATE OR REPLACE VIEW コマンドの実行をサポートします。ビューの実装は、リライトルール（Rewrite Rules）システムに強く依存しており、本ヘッダーはパーサーが生成した構文木（ViewStmt）を実際のデータベースオブジェクトへ変換する橋渡しを担います。

責務

  • ビュー定義コマンド（ViewStmt）の解析と実行の開始。
  • ビューのクエリ定義をシステムカタログへ永続化するためのインターフェース提供。
  • ビュー作成時に必要なオブジェクトアドレスの生成と管理。

重要なシンボル

  • DefineView: CREATE VIEW コマンドを実行する主要な関数。パーサーから渡された ViewStmt を基にビューを作成します。
  • StoreViewQuery: ビューのクエリ定義をシステムカタログに保存します。CREATE OR REPLACE の際にも呼び出され、既存のルールを更新します。

制御フローとデータフロー

1. **コマンド実行**: パーサーが CREATE VIEW 文を解析し、ViewStmt ノードを生成します。 2. **定義処理**: DefineView が呼び出され、対象のネームスペースの確認や権限チェックを行います。 3. **リライトルール構築**: DefineVirtualRelation（view.c 内の内部関数）がターゲットリストから列定義を生成し、ビューをリレーションとして登録します。 4. **永続化**: StoreViewQuery を通じて、ビューの定義クエリがリライトルールとしてシステムカタログに格納されます。

直接依存

  • catalog/objectaddress.h: 作成されたビューのオブジェクト識別子（ObjectAddress）を扱うために必要です。
  • nodes/parsenodes.h: ViewStmt や Query 構造体など、SQL 解析結果のノード定義に依存しています。

関連コード

  • src/backend/commands/view.c: 本ヘッダーで宣言された関数の実装が含まれます。
  • src/backend/rewrite/rewriteDefine.h: ビューの定義をリライトルールとして保存する際に利用されるリライトシステムの機能群です。
  • src/include/commands/tablecmds.h: ビュー作成時にテーブル関連のユーティリティ関数を利用するために参照されます。

読者向けメモ

  • 本ファイルは非常にシンプルですが、ビューの背後にある「リライトルール」という概念が重要です。ビューは実体を持たないため、クエリ実行時にルールシステムがビューへのアクセスを元のテーブルへのクエリに書き換える仕組みになっています。
  • DefineView を追う際は、view.c 内の DefineVirtualRelation がどのように列定義（ColumnDef）を構築し、型チェックを行っているかに注目してください。
