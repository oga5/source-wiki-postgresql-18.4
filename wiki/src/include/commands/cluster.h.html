src/include/commands/cluster.h

概要

cluster.h は、PostgreSQL の CLUSTER コマンドおよび VACUUM FULL 操作に関連するデータ構造と関数宣言を定義するヘッダーファイルです。テーブルを特定のインデックス順に物理的に再配置する機能のインターフェースを提供します。

モジュール内での役割

src/include/commands モジュールの一部として、テーブルの物理的な再構成（リクラスタリング）を制御します。このファイルは、SQL レベルの CLUSTER 文の解析結果をバックエンドの実行ロジックへ橋渡しする役割を担います。

責務

  • CLUSTER コマンドの実行オプション（ClusterParams）の定義。
  • テーブルの物理再配置に必要な低レベル操作（新しいヒープの作成、ヒープの入れ替え）の公開。
  • インデックスがクラスタリングに適しているかの検証およびメタデータの更新。

重要なシンボル

  • ClusterParams: 実行オプション（VERBOSE 等）を保持する構造体。
  • cluster(): SQL 文を受け取り、クラスタリング処理を開始するメインエントリーポイント。
  • make_new_heap(): 再配置先となる新しい物理テーブルを作成する関数。
  • finish_heap_swap(): 古いヒープと新しいヒープを入れ替え、カタログ情報を更新する最終処理関数。
  • CLUOPT_*: 実行時の挙動を制御するビットフラグ。

制御フローとデータフロー

1. **解析**: cluster() が ClusterStmt を受け取り、オプションを ClusterParams に変換します。 2. **準備**: make_new_heap() を呼び出し、新しいストレージ領域を確保します。 3. **実行**: cluster_rel() が実際のデータコピーとインデックス再構築を管理します。 4. **完了**: finish_heap_swap() により、システムカタログの更新と古いヒープの削除が行われ、アトミックに入れ替えが完了します。

直接依存

  • nodes/parsenodes.h: ClusterStmt 構造体の定義。
  • parser/parse_node.h: ParseState の定義。
  • storage/lock.h: ロックモード（LOCKMODE）の定義。
  • utils/relcache.h: リレーションキャッシュ（Relation）の定義。

関連コード

  • src/backend/commands/cluster.c: 本ヘッダーで宣言された関数の実装。
  • src/backend/commands/tablecmds.h: テーブル操作全般を扱う関連モジュール。
  • src/include/commands/vacuum.h: VACUUM FULL が内部的に cluster.c のロジックを再利用しているため、密接に関連します。

読者向けメモ

  • 本ファイルで定義される関数は、単一テーブルだけでなく、複数のテーブルを順次処理する際にも使用されます。
  • finish_heap_swap() は非常に強力な関数であり、システムカタログの整合性を直接操作するため、呼び出し時には適切なロック管理が必須です。
  • CLUSTER 操作は排他ロックを必要とするため、長時間実行される可能性を考慮し、進捗報告（CLUOPT_VERBOSE）の仕組みが組み込まれています。
