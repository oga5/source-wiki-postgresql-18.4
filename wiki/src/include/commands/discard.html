src/include/commands/discard.h

概要

src/include/commands/discard.h は、PostgreSQL の DISCARD コマンドを処理するためのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

src/include/commands モジュールの一部として、SQL レベルの DISCARD コマンドを実行するためのバックエンド処理への入り口を提供します。

責務

  • DISCARD コマンドの実行に必要な関数プロトタイプの公開。
  • パーサーが生成した DiscardStmt 構造体をバックエンドの実行ロジックへ橋渡しする。

重要なシンボル

  • DiscardCommand: DISCARD コマンドの実行を司るメイン関数。パーサーから渡された DiscardStmt を解析し、対象（ALL, PLANS, SEQUENCES, TEMP）に応じた処理を呼び出します。

制御フローとデータフロー

1. パーサーが SQL を解析し、DiscardStmt 構造体を生成します。 2. DiscardCommand が呼び出され、stmt->target に基づいて適切なリセット処理（ResetPlanCache や DiscardAll など）へ分岐します。 3. DiscardAll が選択された場合、セッション状態の全リセット（準備済み文の破棄、一時テーブルの削除、GUC のリセット等）が実行されます。

直接依存

  • nodes/parsenodes.h: DiscardStmt 構造体の定義を参照するために必要です。

関連コード

  • src/backend/commands/discard.c: 本ヘッダーで宣言された関数の実装が含まれています。DISCARD コマンドの具体的なロジック（DiscardAll 等）はすべてここに記述されています。

読者向けメモ

  • 本ファイルは非常にシンプルであり、主に DiscardCommand 関数のシグネチャを保持する役割に留まります。
  • DISCARD ALL はセッションの状態を初期化する強力なコマンドであるため、トランザクションブロック内での実行が禁止されている点に注意してください（実装詳細は discard.c を参照）。
  • 新しい DISCARD ターゲットを追加する場合は、parsenodes.h の列挙型定義と discard.c の switch 文の両方を更新する必要があります。
