src/include/commands/copyfrom_internal.h

概要

COPY FROM コマンドの実行に必要な内部データ構造と型定義を定義するヘッダーファイルです。PostgreSQL の COPY サブシステムにおいて、データソースから行を読み込み、解析し、テーブルへ挿入するまでの一連の処理状態を保持・管理します。

モジュール内での役割

src/include/commands/copy.h が公開インターフェースを提供するのに対し、本ファイルは COPY FROM の実装詳細（内部状態管理）をカプセル化します。主に copyfromparse.c や copyfrom.c などの実装ファイル間で共有される構造体や定数を提供します。

責務

  • COPY FROM 操作全体の実行状態（CopyFromStateData）の定義。
  • 入力ソース（ファイル、フロントエンド、コールバック）および行終端（EOL）の抽象化。
  • バッファリング戦略（input_buf, raw_buf）の定義と、それらを用いた効率的なデータ読み込みのサポート。
  • エラーハンドリングおよび変換処理に必要なメタデータの保持。

重要なシンボル

  • CopyFromStateData: COPY FROM 実行中の全状態を保持する中心的な構造体。
  • CopySource: データの読み込み元を識別する列挙型。
  • CopyInsertMethod: 挿入戦略（単一行 vs バッチ挿入）を決定する列挙型。
  • INPUT_BUF_SIZE / RAW_BUF_SIZE: 読み込みバッファのサイズ（64KB）。
  • CopyFromState: CopyFromStateData へのポインタ型（実装全体で頻繁に使用）。

追加の索引済みシンボル

  • EolType

制御フローとデータフロー

1. **初期化**: CopyFromStateData が生成され、ソースタイプやフォーマットオプションが設定されます。 2. **読み込み**: raw_buf に生データが読み込まれ、必要に応じてエンコーディング変換を経て input_buf に格納されます。 3. **解析**: line_buf に行単位でデータが切り出され、attribute_buf を用いて各フィールドが抽出されます。 4. **変換・挿入**: CopyFromTextOneRow 等の関数が attribute_buf から Datum を生成し、テーブルへ挿入します。

直接依存

  • commands/copy.h: 基本的な COPY 関連の型定義。
  • commands/trigger.h: トリガー処理用。
  • nodes/miscnodes.h: 構文解析や実行に必要なノード定義。

関連コード

  • src/backend/commands/copyfrom.c: COPY FROM のメインロジック。
  • src/backend/commands/copyfromparse.c: テキスト/CSV/バイナリ形式の解析ロジック。
  • src/include/commands/copyapi.h: COPY の拡張用 API 定義。

読者向けメモ

  • 本ファイルは COPY FROM の内部状態を網羅しているため、COPY の挙動を追跡する際の「辞書」として機能します。
  • attribute_buf や line_buf の再利用はパフォーマンス最適化の要です。これらを直接操作する際は、バッファの境界条件（\0 終端の保証など）に注意してください。
  • CopyFromStateData は非常に巨大な構造体です。関数間で受け渡す際は、スタックオーバーフローを避けるため必ずポインタ（CopyFromState）を使用してください。
