src/include/commands/explain_state.h

概要

explain_state.h は、PostgreSQL の EXPLAIN コマンド実行時に使用される状態管理構造体 ExplainState および関連する型定義を保持するヘッダーファイルです。EXPLAIN の実行オプションや出力形式、プラン実行時の統計情報を保持し、コマンドのライフサイクル全体を通じて状態を管理します。

モジュール内での役割

src/include/commands モジュールにおいて、EXPLAIN コマンドの実行エンジンと、その結果を整形・出力する処理の橋渡しを行います。explain.h がコマンドのインターフェースを定義するのに対し、本ファイルは実行中に必要な「状態」のデータ構造を定義しています。

責務

  • EXPLAIN コマンドのオプション（ANALYZE, BUFFERS, VERBOSE 等）を保持する構造体の定義。
  • 並列クエリ実行時のワーカーごとの状態管理（ExplainWorkersState）。
  • 拡張機能（Extension）が EXPLAIN に独自のオプションを追加し、その状態を保持するための拡張用インターフェースの提供。

重要なシンボル

  • ExplainState: EXPLAIN 実行中の全状態を保持するメイン構造体。
  • ExplainFormat: 出力形式（TEXT, XML, JSON, YAML）を定義する列挙型。
  • ExplainWorkersState: 並列クエリ実行時にワーカーごとの出力バッファや状態を管理する構造体。
  • GetExplainExtensionState / SetExplainExtensionState: 拡張機能が ExplainState 内にプライベートな状態を保持するためのアクセサ。

追加の索引済みシンボル

  • ExplainSerializeOption
  • ParseState

制御フローとデータフロー

1. **初期化**: NewExplainState() が呼び出され、デフォルト値で ExplainState が生成されます。 2. **解析**: ParseExplainOptionList() がユーザー指定のオプションを解析し、ExplainState の各フラグを更新します。 3. **拡張**: 拡張機能が RegisterExtensionExplainOption で独自のオプションを登録し、実行時に ApplyExtensionExplainOption を通じて状態を更新します。 4. **保持**: 実行中、ExplainState はプランツリーの走査に伴い、インデントレベルや出力バッファを更新し続けます。

直接依存

  • nodes/plannodes.h: PlannedStmt などのプランノード定義。
  • parser/parse_node.h: ParseState の定義。

読者向けメモ

  • **拡張性**: ExplainState は固定のフィールドだけでなく、extension_state 配列を通じて動的に拡張可能です。独自の EXPLAIN オプションを追加する際は、この仕組みを利用してください。
  • **メモリ管理**: ExplainState 内の str（出力バッファ）は StringInfo で管理されています。メモリリークを防ぐため、ライフサイクル管理には注意が必要です。
  • **状態の永続性**: ExplainState はプランツリーの走査中、再帰的に渡されるため、関数間で状態を共有する重要なコンテキストとして機能します。
