src/include/commands/copy.h

概要

PostgreSQLの COPY コマンドを実行するためのデータ構造、列挙型、および関数プロトタイプを定義するヘッダーファイルです。テーブルと外部ファイル（またはプログラム）間でのデータ転送を制御するインターフェースを提供します。

モジュール内での役割

src/include/commands モジュールの一部として、SQLの COPY 文の構文解析結果を実際の実行処理へ橋渡しする役割を担います。copy.c を中心としたバックエンドの実装と、パーサーやエグゼキューターを繋ぐ共通定義を提供します。

責務

  • COPY コマンドのオプション（CSV形式、区切り文字、エラー処理など）を保持する構造体の定義。
  • データ転送の開始、実行、終了を制御するAPIの提供。
  • COPY FROM（入力）および COPY TO（出力）の処理状態を管理するための抽象的な型定義。

重要なシンボル

  • CopyFormatOptions: COPY コマンドのオプションをパース後の形式で保持する構造体。
  • CopyFromState / CopyToState: それぞれ COPY FROM および COPY TO の実行状態を保持する不透明なポインタ型。
  • DoCopy: COPY 文を実行するためのメインエントリーポイント。
  • copy_data_source_cb / copy_data_dest_cb: 外部ソースや宛先とのデータ転送を抽象化するためのコールバック関数型。

追加の索引済みシンボル

  • CopyHeaderChoice
  • CopyLogVerbosityChoice
  • CopyOnErrorChoice

制御フローとデータフロー

1. **解析**: ProcessCopyOptions により、SQL文のオプションが CopyFormatOptions に変換されます。 2. **初期化**: BeginCopyFrom または BeginCopyTo が呼び出され、転送状態（State）が初期化されます。 3. **転送**: CopyFrom や DoCopyTo がループ内でデータを処理します。この際、NextCopyFrom 等の関数が個別の行やフィールドを読み書きします。 4. **終了**: EndCopyFrom / EndCopyTo によりリソースが解放されます。

直接依存

  • nodes/execnodes.h: エグゼキューターノードの定義。
  • nodes/parsenodes.h: CopyStmt などの構文解析ツリーノード。
  • parser/parse_node.h: ParseState 構造体。
  • tcop/dest.h: DestReceiver（クエリ結果の宛先）の定義。

関連コード

  • src/backend/commands/copy.c: DoCopy 等の主要な実装が含まれる対となるソースファイル。
  • src/include/commands/copyfrom_internal.h: COPY FROM の内部的な詳細定義。
  • src/include/commands/copyapi.h: COPY の拡張やカスタムデータソースのためのAPI定義。

読者向けメモ

  • 本ファイルは定義のみを提供します。具体的なロジックは src/backend/commands/ 以下の copy.c、copyfrom.c、copyto.c に分割されています。
  • CopyFormatOptions は非常に多くのフィールドを持ちますが、そのほとんどは COPY コマンドの多様なフォーマット指定に対応するためのものです。
  • COPY の処理は、ファイルシステムへのアクセスだけでなく、stdin/stdout やプログラムとのパイプ通信もサポートしている点に注意してください。
