src/include/backup/backup_manifest.h

概要

バックアップ取得時に生成される「バックアップマニフェスト」の構築と管理を行うためのインターフェースを定義するヘッダーファイルです。マニフェストは、バックアップに含まれる全ファイルのメタデータ（サイズ、チェックサム、タイムスタンプ等）を記録した JSON 形式のファイルです。

モジュール内での役割

src/include/backup モジュールにおいて、バックアップの整合性を検証するためのメタデータ生成を担います。basebackup_sink.h で定義されるシンク（出力先）と連携し、バックアップストリームの進行に合わせてマニフェストを逐次構築します。

責務

  • マニフェスト構築に必要な状態（backup_manifest_info）の保持と管理。
  • バックアップ対象ファイルおよび WAL 情報のマニフェストへの追加。
  • マニフェスト全体のチェックサム計算（SHA-256）と最終的な出力処理。
  • マニフェスト生成の有効/無効の切り替えおよびエンコーディング制御。

重要なシンボル

  • backup_manifest_info: マニフェスト生成のコンテキストを保持する構造体。一時ファイル（BufFile）への書き込みや、ハッシュ計算の状態を管理します。
  • backup_manifest_option: マニフェスト生成の要否を指定する列挙型。
  • InitializeBackupManifest: マニフェスト生成の初期化を行い、JSON のヘッダー部分を書き込みます。
  • AddFileToBackupManifest: 個別のファイル情報をマニフェストに追加します。
  • SendBackupManifest: 構築済みのマニフェストを最終的な出力先（bbsink）へ送信します。

制御フローとデータフロー

1. **初期化**: InitializeBackupManifest で backup_manifest_info を初期化し、一時ファイルを作成します。 2. **データ追加**: バックアップ処理中に AddFileToBackupManifest が繰り返し呼ばれ、ファイル情報が JSON 形式で一時ファイルに追記されます。 3. **終了処理**: バックアップ完了後、SendBackupManifest が呼び出され、一時ファイルの内容を読み込み、最終的なチェックサムを付与して出力先へ転送します。 4. **解放**: FreeBackupManifest でハッシュ計算用コンテキストなどのリソースを解放します。

直接依存

  • backup/basebackup_sink.h: 出力先インターフェースの定義。
  • common/checksum_helper.h: チェックサム計算用ユーティリティ。
  • pgtime.h: 時刻型定義。
  • storage/buffile.h: 一時ファイル操作用 API。

関連コード

  • src/backend/backup/backup_manifest.c: 本ヘッダーで宣言された関数の実装。
  • src/backend/backup/basebackup.c: バックアップ処理のメインループ。ここで AddFileToBackupManifest が呼び出されます。

読者向けメモ

  • マニフェスト自体のチェックサムには、データファイルのチェックサムアルゴリズム設定に関わらず、常に SHA-256 が使用される点に注意してください。
  • backup_manifest_info は、マニフェストが不要な場合でも NULL ポインタチェックを減らすために初期化される設計になっています。
  • マニフェストは巨大になる可能性があるため、メモリ上に保持せず BufFile を介して一時ファイルに書き出す実装となっています。
