src/include/snowball/header.h

概要

本ファイルは、外部ライブラリである Snowball ステマー（形態素解析器）を PostgreSQL のバックエンド環境に統合するための「ラッパーヘッダー」です。Snowball が想定する環境と PostgreSQL の実行環境の差異を吸収し、メモリ管理やコンパイル設定を統合します。

モジュール内での役割

Snowball のソースコードは、本来 snowball/libstemmer/header.h を参照するように設計されています。ビルドシステム側でインクルードパスを調整することで、Snowball のコードが本ファイルを読み込むように強制し、PostgreSQL の規約に適合させます。

責務

  • **環境の整合性確保**: pg_config.h を確実に先行読み込みさせ、プラットフォーム固有のヘッダー競合や大型ファイル対応の不整合を回避する。
  • **メモリ管理のフック**: Snowball が使用する標準ライブラリのメモリ関数（malloc, free 等）を、PostgreSQL のメモリコンテキスト管理関数（palloc, pfree 等）へ置換する。
  • **名前空間の保護**: MAXINT や MININT などの競合しやすいマクロを未定義化し、コンパイルエラーを防止する。

重要なシンボル

  • SNOWBALL_HEADR_H: 多重インクルードを防ぐためのガードマクロ。
  • malloc, calloc, realloc, free: PostgreSQL のメモリ管理関数へリダイレクトするためのマクロ定義。

制御フローとデータフロー

本ファイルは実行コードを含まず、プリプロセッサによる置換が主目的です。 1. postgres.h を読み込み、PostgreSQL の基本環境を構築。 2. 既存の MAXINT/MININT を削除し、名前空間の汚染を排除。 3. 本来の snowball/libstemmer/header.h を読み込み、Snowball 側の型定義や関数宣言を取り込む。 4. 最後に標準メモリ関数を palloc 系へ再定義し、以降の Snowball ソースコードが PostgreSQL のメモリ管理下で動作するように強制する。

直接依存

  • postgres.h: PostgreSQL のコア定義およびメモリ管理関数（palloc 等）を提供。
  • snowball/libstemmer/header.h: Snowball 本来のヘッダーファイル。

関連コード

  • src/include/snowball/libstemmer/: Snowball のコアロジックが含まれるディレクトリ。
  • src/backend/snowball/: 本ヘッダーを利用してコンパイルされる Snowball の実装ソース群。

読者向けメモ

  • **非汎用性**: 本ファイルは Snowball 専用です。他のソースコードからインクルードしてはなりません。
  • **規約違反の許容**: 本来、ヘッダーファイル内での postgres.h のインクルードは PostgreSQL のコーディング規約で禁止されていますが、Snowball の機械生成されたソースコードを修正せずに統合するための例外措置です。
  • **メモリ管理**: Snowball が確保するメモリはすべて PostgreSQL のメモリコンテキスト経由となるため、メモリリークの追跡やデバッグは PostgreSQL の標準的な手法で行えます。
