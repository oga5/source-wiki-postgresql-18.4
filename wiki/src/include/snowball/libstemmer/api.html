src/include/snowball/libstemmer/api.h

概要

Snowball ステミングアルゴリズムを実行するための基盤となるデータ構造と、外部インターフェースを定義するヘッダーファイルです。

モジュール内での役割

libstemmer モジュールにおける「実行環境の定義」を担います。各言語ごとのステミングロジック（stem_*.h）が共通して利用する、ステミング処理のコンテキスト（環境）を規定します。

責務

  • ステミング処理に必要な状態保持構造体 SN_env の定義。
  • 文字列データを扱うための型 symbol の抽象化。
  • ステミング環境の生成・破棄および入力文字列の設定を行う API の宣言。

重要なシンボル

  • symbol: ステミング対象の文字型。デフォルトは unsigned char ですが、必要に応じて変更可能です。
  • SN_env: ステミング処理の実行環境を保持する構造体。現在のカーソル位置、境界、動的メモリ領域（S）、整数領域（I）を管理します。
  • SN_create_env: ステミング環境を初期化し、必要なメモリを確保します。
  • SN_set_current: ステミング対象の文字列を環境にロードし、カーソルをリセットします。

制御フローとデータフロー

1. **環境構築**: SN_create_env により SN_env が生成され、アルゴリズムが必要とする作業用メモリ（S配列やI配列）が確保されます。 2. **データ入力**: SN_set_current を通じて、処理対象の文字列が SN_env 内のバッファにコピーされます。 3. **ステミング実行**: 各言語固有のステミング関数が SN_env を参照しながら、カーソル（c）や境界（bra, ket）を操作して文字列を変換します。 4. **終了処理**: SN_close_env により、確保されたメモリが解放されます。

直接依存

本ファイルは標準的な C 言語の型定義のみに依存しており、他のヘッダーファイルへの直接的な #include はありません。

関連コード

  • src/backend/snowball/libstemmer/api.c: 本ヘッダーで宣言された関数の実装が含まれており、メモリ確保や文字列操作の具体的なロジックが記述されています。
  • src/include/snowball/libstemmer/header.h: SN_env のメモリ配置やアライメントに関連する定数定義が含まれており、併せて参照する必要があります。

読者向けメモ

  • symbol 型の定義を変更する場合は、header.h で定義される HEAD とのアライメント制約に注意してください。
  • SN_env 構造体は、ステミングアルゴリズムの実行中に動的に変化する状態（カーソル位置など）を保持するため、スレッドセーフな実装が必要な場合は、環境をスレッドごとに独立させる必要があります。
