src/include/storage/procarray.h

概要

procarray.h は、PostgreSQL の全アクティブなバックエンドプロセスを管理する「プロセス配列（ProcArray）」のインターフェースを定義するヘッダーファイルです。このモジュールは、システム全体で実行中のトランザクションを追跡し、MVCC（多版同時実行制御）に必要なスナップショット生成や、トランザクションの可視性判定の基盤となります。

モジュール内での役割

src/include/storage モジュールにおいて、プロセス管理とトランザクション状態の追跡を担う中心的な役割を果たします。src/backend/storage/ipc/procarray.c と対になっており、共有メモリ上のプロセス情報を操作するための公開 API を提供します。

責務

  • **トランザクション追跡**: 全バックエンドのトランザクション ID (XID) およびサブトランザクション ID の状態管理。
  • **スナップショット生成**: GetSnapshotData を通じて、現在のトランザクションの可視性を決定するためのスナップショットを提供。
  • **ホットスタンバイ対応**: スタンバイ側で実行中のトランザクションを追跡するための KnownAssignedXids 配列の管理。
  • **プロセス検索**: PID やトランザクション ID をキーとした PGPROC 構造体の検索および状態取得。
  • **バックエンド制御**: 特定のデータベースやユーザーに関連するバックエンドのカウント、キャンセル、終了処理。

重要なシンボル

  • ProcArrayShmemInit/Size: 共有メモリ上のプロセス配列の初期化とサイズ計算。
  • GetSnapshotData: 現在のトランザクション可視性を定義するスナップショットを取得する主要関数。
  • TransactionIdIsInProgress: 指定された XID が現在実行中かどうかを判定する。
  • ProcArrayAdd/Remove: バックエンドの開始・終了時にプロセス配列へ登録・削除を行う。
  • PGPROC: 各バックエンドの状態を保持する構造体（proc.h で定義）。

制御フローとデータフロー

1. **登録**: バックエンド起動時に ProcArrayAdd が呼ばれ、共有メモリ上の ProcArrayStruct に登録されます。 2. **スナップショット**: トランザクション開始時、GetSnapshotData が ProcArrayLock を取得し、実行中の全 XID をスキャンしてスナップショットを構築します。 3. **更新**: トランザクション終了時、ProcArrayEndTransaction により XID がクリアされ、可視性判定から除外されます。 4. **メンテナンス**: KnownAssignedTransactionIdsIdleMaintenance 等により、不要になった古いトランザクション情報が定期的に整理されます。

直接依存

  • storage/lock.h: ロック管理機能。
  • storage/standby.h: ホットスタンバイ関連の定義。
  • utils/relcache.h: リレーションキャッシュへのアクセス。
  • utils/snapshot.h: スナップショット構造体の定義。

読者向けメモ

  • **ロックの重要性**: プロセス配列へのアクセスには ProcArrayLock が必須です。共有メモリ上の構造を直接操作する際は、デッドロックや競合に十分注意してください。
  • **ホットスタンバイ**: スタンバイノードでは、プライマリから送られてくる WAL を基に KnownAssignedXids を更新します。この仕組みが MVCC の整合性を維持する鍵となります。
  • **パフォーマンス**: GetSnapshotData は頻繁に呼ばれるため、この関数の効率がシステム全体のパフォーマンスに直結します。
