src/include/storage/s_lock.h

概要

PostgreSQL におけるスピンロックの低レベル実装を提供します。ハードウェア固有の原子操作（Test-and-Set）を抽象化し、プラットフォーム間で一貫したロック取得・解放インターフェースを定義します。

モジュール内での役割

storage サブシステムにおける最も基礎的な同期プリミティブです。spin.h が提供する高レベルなスピンロック API のバックエンドとして機能し、CPU アーキテクチャごとの最適化された命令セットをラップします。

責務

  • **原子操作の抽象化**: TAS() (Test-and-Set) を通じて、CPU 固有の命令（x86 の xchg、ARM の ldrex/strex 等）を呼び出す。
  • **メモリバリアの保証**: コンパイラや CPU による命令の並び替えを防ぎ、クリティカルセクションの前後でメモリの整合性を維持する。
  • **待機戦略の管理**: ロック競合時のスピン待機（SPIN_DELAY）と、バックオフを伴うスリープ処理の制御。

重要なシンボル

  • slock_t: スピンロックの状態を保持する型（プラットフォームにより unsigned char や int 等）。
  • S_LOCK(lock): ロック取得を試みるメインマクロ。失敗時は s_lock() 関数へ委譲。
  • S_UNLOCK(lock): ロックを解放するマクロ。メモリバリアを伴う。
  • TAS(lock): 低レベルな Test-and-Set 命令。直接呼び出しは非推奨。
  • SpinDelayStatus: スピン待機の試行回数や遅延時間を追跡するための構造体。

制御フローとデータフロー

1. **ロック試行**: S_LOCK が呼ばれると、まず TAS で即時取得を試みる。 2. **競合時**: 取得失敗時、s_lock() 関数が呼び出される。 3. **待機ループ**: s_lock() 内で TAS_SPIN をループ実行し、perform_spin_delay() を介して CPU に負荷をかけない待機（PAUSE 命令や isb 等）を行う。 4. **タイムアウト**: 規定回数（NUM_DELAYS）を超えても取得できない場合、デッドロックやスタックと見なして PANIC を発生させる。

直接依存

  • intrin.h: Windows 環境におけるコンパイラ組み込み関数（_mm_pause 等）の利用。

関連コード

読者向けメモ

  • **直接使用禁止**: 本ファイルの各マクロは直接呼び出さず、必ず spin.h を経由してください。
  • **移植性**: 新しいアーキテクチャへの移植時には、HAS_TEST_AND_SET を定義し、TAS と S_UNLOCK を適切に実装する必要があります。
  • **メモリバリア**: S_UNLOCK は単なる代入ではなく、メモリバリアを含む必要がある点に注意してください。デフォルト実装は非常に限定的な環境でのみ安全です。
