src/include/storage/pg_shmem.h

概要

pg_shmem.h は、PostgreSQL がプラットフォーム間で共有メモリを扱うための抽象化レイヤーを定義するヘッダーファイルです。OS 固有の共有メモリ実装（System V、mmap、Windows）を隠蔽し、PostgreSQL の共有メモリセグメントの初期化と管理に必要な共通インターフェースを提供します。

モジュール内での役割

src/include/storage モジュールにおいて、共有メモリの基盤となる低レベルなセグメント管理を担当します。上位層の shmem.h（共有メモリ内のデータ構造管理）や dsm.h（動的共有メモリ）が利用する、物理的なメモリ領域の確保とアタッチを行うための最下層の API を提供します。

責務

  • **共有メモリセグメントのヘッダー定義**: 全ての共有メモリ領域の先頭に配置される PGShmemHeader 構造体の定義。
  • **プラットフォーム抽象化**: OS 依存の共有メモリ識別子（ID やハンドル）をラップし、一貫した操作インターフェースを提供。
  • **設定管理**: shared_memory_type や huge_pages といった共有メモリ関連の GUC 変数および列挙型の定義。
  • **ライフサイクル管理**: 共有メモリの作成、アタッチ、デタッチ、および使用状況の確認機能の宣言。

重要なシンボル

  • PGShmemHeader: 共有メモリセグメントのメタデータ（サイズ、マジックナンバー、DSM 制御ハンドル等）を保持する構造体。
  • PGShmemMagic: 共有メモリセグメントが PostgreSQL のものかを確認するための定数。
  • PGSharedMemoryCreate: 共有メモリセグメントを新規作成し、初期化する主要関数。
  • PGSharedMemoryDetach: プロセスが共有メモリから切り離すための関数。
  • HugePagesType: Huge Pages の有効/無効状態を管理する列挙型。

追加の索引済みシンボル

  • PGShmemType

制御フローとデータフロー

1. **初期化**: サーバー起動時に PGSharedMemoryCreate が呼び出され、OS から共有メモリ領域が確保されます。 2. **ヘッダー設定**: 確保された領域の先頭に PGShmemHeader が書き込まれ、マジックナンバーや制御情報が格納されます。 3. **アタッチ**: 子プロセス（EXEC_BACKEND 環境下など）は、既存のセグメントを再アタッチすることでメモリ空間を共有します。 4. **管理**: UsedShmemSegAddr 等のグローバル変数を介して、各プロセスが共有メモリのベースアドレスを保持し、以降のメモリ操作を行います。

直接依存

  • storage/dsm_impl.h: 動的共有メモリの実装詳細への依存。

関連コード

読者向けメモ

  • 本ファイルは「物理的なメモリセグメントの確保」に特化しています。共有メモリ内のメモリ割り当て（ShmemAlloc など）については shmem.h を参照してください。
  • WIN32 定義の有無により、共有メモリの識別子（HANDLE vs unsigned long）やヘッダー内の保持情報が大きく異なります。移植性を考慮する際は注意が必要です。
  • EXEC_BACKEND が定義されている環境（Windows 等）では、プロセス間で共有メモリを再アタッチするフローが必須となるため、関連する関数群の呼び出し順序に注意してください。
