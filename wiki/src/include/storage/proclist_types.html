src/include/storage/proclist_types.h

概要

proclist_types.h は、PostgreSQLのプロセス管理において使用される「プロセス番号（ProcNumber）による双方向連結リスト」のデータ構造を定義するヘッダーファイルです。

モジュール内での役割

src/include/storage モジュール内で、プロセス間通信やロック管理など、複数のプロセスをリスト構造で追跡する必要があるサブシステムのための基盤型を提供します。

責務

  • プロセスリストを構成するためのノード構造体（proclist_node）の定義。
  • リストの先頭と末尾を保持するヘッダー構造体（proclist_head）の定義。
  • リスト走査中に安全な要素削除を可能にするためのイテレータ構造体（proclist_mutable_iter）の定義。

重要なシンボル

  • proclist_node: リスト内の各要素が持つべきリンク情報。next と prev には ProcNumber が格納されます。
  • proclist_head: リストの起点と終点を管理する構造体。空リストは INVALID_PROC_NUMBER で表現されます。
  • proclist_mutable_iter: 走査中に現在の要素が削除されても、次の要素を安全に参照し続けるためのイテレータ。

制御フローとデータフロー

本ファイルは型定義のみを提供するため、直接的な制御フローは持ちません。実際のリスト操作（挿入、削除、走査など）は、この型定義をインクルードする proclist.h 内の関数群によって実行されます。データフローとしては、ProcNumber を介してプロセス配列（PGPROC 配列）上のインデックスを相互参照する形式をとります。

直接依存

  • storage/procnumber.h: ProcNumber 型および INVALID_PROC_NUMBER 定数の定義元。

関連コード

  • src/include/storage/proclist.h: 本ファイルで定義された型を操作するインライン関数やマクロが実装されています。
  • src/include/storage/proc.h: PGPROC 構造体そのものの定義が含まれており、リストの各ノードが指し示す対象となります。

読者向けメモ

  • **メモリ効率**: ポインタではなく ProcNumber（整数インデックス）を使用することで、共有メモリ内での配置や、プロセス間でのリスト構造の共有を容易にしています。
  • **不変条件**: リストに含まれていないノードは next == prev == 0 である必要があります。リスト内のノードが循環参照を持つことは許可されていません。
  • **利用上の注意**: リストの操作を行う際は、必ず proclist.h で提供される API を使用してください。直接構造体のメンバを操作すると、リストの整合性が崩れるリスクがあります。
