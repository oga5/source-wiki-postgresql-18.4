src/include/storage/waiteventset.h

概要

waiteventset.h は、PostgreSQL におけるイベント待機メカニズムの抽象化レイヤーを定義するヘッダーファイルです。複数のソケットやラッチ（Latch）の状態変化を、OS 固有の多重化プリミティブ（epoll, kqueue, poll 等）を用いて効率的に監視するためのインターフェースを提供します。

モジュール内での役割

src/include/storage モジュールにおいて、プロセス間通信（IPC）やネットワーク I/O 待機を統合管理する役割を担います。従来の WaitLatch や WaitLatchOrSocket と比較して、長期間生存するイベントセットを構築することで、システムコールのオーバーヘッドを削減し、スケーラビリティを向上させることを目的としています。

責務

  • 待機対象イベント（ソケットの読み書き、ラッチのセット、ポストマスターの終了など）のビットマスク定義。
  • WaitEventSet 構造体のライフサイクル管理（作成、破棄、イベントの追加・変更）。
  • 発生したイベントを呼び出し元に通知するためのデータ構造（WaitEvent）の提供。
  • OS 依存の待機実装を隠蔽し、一貫した API を提供すること。

重要なシンボル

  • WaitEventSet: 待機対象のイベント群を保持する不透明な構造体。
  • WaitEvent: 発生したイベントの詳細（ファイル記述子、イベント種別、ユーザー定義データ）を格納する構造体。
  • WL_* マクロ: 待機対象イベントの種類を指定するビットマスク（例: WL_LATCH_SET, WL_SOCKET_READABLE）。
  • CreateWaitEventSet() / WaitEventSetWait(): イベントセットの生成および待機実行の主要関数。

制御フローとデータフロー

1. **初期化**: CreateWaitEventSet でセットを確保し、ResourceOwner を紐付けます。 2. **登録**: AddWaitEventToSet を使用して、監視対象のソケットやラッチをセットに追加します。 3. **待機**: WaitEventSetWait を呼び出し、イベントが発生するかタイムアウトするまでブロックします。 4. **処理**: 戻り値として得られた WaitEvent 配列を走査し、各イベントの events ビットを確認して適切な処理を実行します。 5. **解放**: FreeWaitEventSet を呼び出し、リソースを適切にクリーンアップします。

直接依存

  • utils/resowner.h: リソース所有権の管理に使用。

関連コード

  • src/backend/storage/ipc/waiteventset.c: 実装本体。OS 固有の epoll/kqueue/poll の切り替えロジックが含まれます。
  • src/include/storage/latch.h: ラッチの定義。WaitEventSet と密接に関連します。

読者向けメモ

  • **プラットフォーム依存**: 実装は waiteventset.c 内のコンパイル時フラグ（WAIT_USE_EPOLL 等）によって決定されます。
  • **タイムアウト**: WaitEventSetWait のタイムアウト指定は可能な限り避けることが推奨されます。これは、タイムアウト処理が追加のオーバーヘッドを発生させるためです。
  • **リソース管理**: ResourceOwner を指定することで、エラー発生時やトランザクション終了時に自動的にリソースが解放されるよう設計されています。メモリリークを防ぐため、適切なスコープで管理してください。
