src/include/storage/spin.h

概要

src/include/storage/spin.h は、PostgreSQL における最も低レベルな同期プリミティブである「スピンロック」の抽象化レイヤーを提供します。

モジュール内での役割

storage サブシステムにおいて、スピンロックは他の複雑なロック機構（LWLock や重量級ロック）を構築するための基礎となります。本ファイルは、ハードウェア依存の低レベル実装を隠蔽し、PostgreSQL のコード全体で一貫したインターフェースを提供します。

責務

  • スピンロックの初期化、取得、解放、状態確認のための標準 API を定義する。
  • コンパイラバリアを介して、ロック操作前後でのメモリ操作の順序付け（メモリバリア）を保証する。
  • 取得失敗時のタイムアウト処理（通常約1分）を規定する。

重要なシンボル

  • SpinLockInit: ロックを初期状態（アンロック）に設定する。
  • SpinLockAcquire: ロックを取得する。取得できない場合は待機し、一定時間経過後に abort() する。
  • SpinLockRelease: ロックを解放する。
  • SpinLockFree: ロックの状態をテストする（状態変更は行わない）。

制御フローとデータフロー

本ファイルはマクロ定義のラッパーであり、実際の制御フローは s_lock.h で定義されたハードウェア依存のプリミティブに委譲されます。 1. 呼び出し元が SpinLockAcquire を呼び出す。 2. マクロが s_lock.h の S_LOCK を展開し、CPU のアトミック命令（Test-and-Set 等）を実行する。 3. 取得成功までループし、一定回数（または時間）経過後にエラー処理へ移行する。

直接依存

  • storage/s_lock.h: ハードウェア固有の低レベルアトミック操作の実装を提供。

関連コード

  • src/include/storage/lwlock.h: スピンロックよりも高機能な軽量ロック（LWLock）。スピンロックを使用して内部構造を保護する。
  • src/include/storage/s_lock.h: 本ファイルの基盤となる、アーキテクチャ依存のロック実装。

読者向けメモ

  • **短時間保持の原則**: スピンロックは非常に軽量ですが、保持中に他の処理（I/O や長い計算）を行うことは厳禁です。保持時間が長いとシステム全体のパフォーマンスが著しく低下します。
  • **割り込み禁止**: スピンロック保持中に CHECK_FOR_INTERRUPTS() を呼び出してはなりません。
  • **マクロの副作用**: マクロとして実装されているため、引数に副作用のある式（例: lock++）を渡すと、意図せず複数回評価される可能性がある点に注意してください。
  • **メモリバリア**: 本 API を使用することで、ロック操作と保護対象データの間で適切なメモリ順序付けが保証されます。明示的な volatile 修飾は不要です。
