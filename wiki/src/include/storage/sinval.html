src/include/storage/sinval.h

概要

PostgreSQLの共有キャッシュ無効化（Shared Cache Invalidation）メカニズムの定義ファイルです。複数のバックエンド間でシステムカタログやリレーションキャッシュの整合性を保つための通信プロトコルとデータ構造を規定します。

モジュール内での役割

src/include/storage モジュールにおいて、プロセス間通信（IPC）を通じたキャッシュ同期の基盤を提供します。バックエンドが自身のローカルキャッシュを最新の状態に保つための「通知」の形式を定義しています。

責務

  • キャッシュ無効化メッセージの型定義（Catcache, Relcache, Smgr, Snapshot等）。
  • 共有メモリ上の無効化キューを介したメッセージ送受信インターフェースの提供。
  • トランザクションベース（コミット時）および非トランザクションベース（即時）の無効化イベントの識別。

重要なシンボル

  • SharedInvalidationMessage: 各種無効化メッセージを包含する共用体。先頭の id フィールドで型を識別します。
  • SHAREDINVALCATALOG_ID 等: メッセージの種類を識別するための定数。
  • SendSharedInvalidMessages: 共有キューへメッセージを投入する関数。
  • ReceiveSharedInvalidMessages: 共有キューからメッセージを取り出し、各キャッシュの無効化関数を呼び出す関数。
  • catchupInterruptPending: 溜まった無効化メッセージを処理させるための割り込みフラグ。

制御フローとデータフロー

1. **送信**: バックエンドがシステムカタログ等を変更すると、SendSharedInvalidMessages を介して共有メモリ上のキューにメッセージが書き込まれます。 2. **受信**: 他のバックエンドは、ReceiveSharedInvalidMessages を呼び出すことでキューを読み取ります。 3. **処理**: 受信したメッセージの id に基づき、対応するキャッシュ（CatcacheやRelcacheなど）の無効化関数が実行されます。 4. **割り込み**: キューが溢れそうな場合、遅延しているバックエンドに対して PROCSIG_CATCHUP_INTERRUPT が送られ、強制的にキャッシュ同期が行われます。

読者向けメモ

  • **トランザクション性**: CatcacheやRelcacheの無効化はトランザクションに紐付き、コミット時に送信されます。一方、smgr（ストレージマネージャ）やリレーションマップの無効化は即時送信されるため、注意が必要です。
  • **再帰的呼び出し**: ReceiveSharedInvalidMessages は再帰的に呼び出される可能性があるため、実装は静的バッファを使用してメッセージを安全に処理する設計になっています。
  • **ハッシュ衝突**: Catcacheの無効化はハッシュ値ベースで行われるため、稀に無関係なエントリが誤って無効化される可能性がありますが、これはキャッシュの再ロードによって許容されます。
