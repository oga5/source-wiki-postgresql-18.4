src/include/storage/aio.h

概要

src/include/storage/aio.h は、PostgreSQL の非同期 I/O (AIO) サブシステムのメインインターフェースを定義するヘッダーファイルです。このファイルは、バックエンドが非同期に I/O 操作を発行、管理、および待機するための公開 API とデータ構造を提供します。

モジュール内での役割

storage モジュール内において、AIO サブシステムの中核を担います。aio_types.h で定義された基本型を拡張し、I/O ハンドルのライフサイクル管理、操作の開始、コールバックの登録、およびバッチ処理の制御を行うための関数群を宣言しています。

責務

  • **I/O ハンドルの管理**: PgAioHandle の取得、解放、および状態フラグの設定。
  • **操作の定義**: PgAioOp（読み込み/書き込み）および PgAioTargetID（ターゲット種別）の定義。
  • **コールバック機構**: I/O 完了時の処理を定義する PgAioHandleCallbacks と、それを識別するための PgAioHandleCallbackID の提供。
  • **バッチ処理**: 複数の I/O を効率的に発行するためのバッチモード制御。

重要なシンボル

  • PgAioHandle: I/O 操作のコンテキストを保持する主要な構造体。
  • IoMethod: 使用する I/O 実装（sync, worker, io_uring）を選択する列挙型。
  • PgAioHandleCallbacks: I/O 完了時に実行されるステージングおよび完了処理の関数ポインタ群。
  • pgaio_io_acquire() / pgaio_io_release(): ハンドルのライフサイクルを制御する主要関数。
  • pgaio_io_start_readv() / pgaio_io_start_writev(): I/O 操作を開始するためのインターフェース。

制御フローとデータフロー

1. **初期化**: pgaio_io_acquire でハンドルを取得し、pgaio_io_set_target で対象を設定します。 2. **設定**: pgaio_io_register_callbacks で完了時の処理を登録し、pgaio_io_set_flag で挙動を調整します。 3. **発行**: pgaio_io_start_readv 等で操作を定義し、必要に応じてバッチモード（pgaio_enter_batchmode）を使用して pgaio_submit_staged で一括発行します。 4. **待機**: pgaio_wref_wait を使用して、特定の I/O 操作の完了を待機します。

直接依存

  • storage/aio_types.h: 基本的な型定義と構造体定義。
  • storage/procnumber.h: プロセス識別子に関連する定義。

関連コード

  • src/backend/storage/aio/aio.c: 本ヘッダーで宣言された関数の実装。
  • src/include/storage/aio_internal.h: サブシステム内部でのみ使用される定義。
  • src/include/storage/aio_subsys.h: サブシステム全体の初期化やグローバル状態。
  • src/backend/storage/aio/aio_target.c: ターゲット固有の I/O ロジック。

読者向けメモ

  • **EXEC_BACKEND の制約**: io_uring は EXEC_BACKEND 環境と互換性がありません。
  • **コールバックの制限**: PgAioHandleCallbackID を使用するのは、共有メモリ内での関数ポインタの安定性（EXEC_BACKEND 環境下でのポインタの不一致）を回避するためです。
  • **リソース所有権**: pgaio_io_acquire に ResourceOwner を渡すことで、エラー発生時の適切なクリーンアップが保証されます。
