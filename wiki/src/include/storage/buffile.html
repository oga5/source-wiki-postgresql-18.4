src/include/storage/buffile.h

概要

buffile.h は、PostgreSQL における大規模な一時ファイルを効率的に扱うためのバッファリング層を定義します。標準ライブラリの stdio に似たインターフェースを提供し、fd.c が管理する仮想ファイル記述子の上位で動作します。

モジュール内での役割

storage サブシステムの一部として、ソートやハッシュ結合などの操作で発生する巨大な一時データの読み書きを抽象化します。OS のファイルサイズ制限や、単一のファイル記述子では扱えない巨大なデータセットを、複数の物理ファイルに分割して透過的に管理する役割を担います。

責務

  • **バッファリング**: 物理的なファイルアクセス回数を減らし、I/O 効率を向上させる。
  • **巨大ファイルの分割管理**: 1GB 単位のセグメントにファイルを分割し、OS の制限を超えるサイズのデータを扱えるようにする。
  • **ライフサイクル管理**: トランザクション終了時の自動クリーンアップや、FileSet を介した並列クエリ間での一時ファイル共有をサポートする。

重要なシンボル

  • BufFile: 内部構造が隠蔽された不透明な型。バッファリングされたファイルの状態を保持する。
  • BufFileCreateTemp(): 一時ファイルを作成する主要な関数。
  • BufFileRead() / BufFileWrite(): バッファを介した読み書き操作。
  • BufFileSeek() / BufFileTell(): 論理的なファイル位置の操作。
  • BufFileCreateFileSet(): 並列クエリなどで共有可能な一時ファイルを作成する。

制御フローとデータフロー

1. **作成**: BufFileCreateTemp 等で BufFile オブジェクトを生成し、メモリ上にバッファを確保する。 2. **アクセス**: ユーザーは BufFileRead や BufFileWrite を呼び出す。これらは内部バッファを操作し、バッファが溢れるか空になった場合にのみ、fd.c を通じて物理的なファイル I/O を実行する。 3. **セグメント切り替え**: ファイルサイズが上限（1GB）に達すると、自動的に新しい物理ファイルを作成・オープンし、論理的なストリームを継続する。 4. **終了**: BufFileClose により、関連するすべての仮想ファイル記述子を解放し、リソースをクリーンアップする。

直接依存

  • storage/fileset.h: FileSet 構造体および関連するファイル共有機能の定義。

関連コード

  • src/backend/storage/file/buffile.c: 本ヘッダーで宣言された関数の実装。内部構造体 BufFile の定義もここにある。
  • src/include/storage/fd.h: BufFile が基盤とする仮想ファイル記述子管理機能。

読者向けメモ

  • BufFile は stdio の完全な代替ではありません。フォーマット機能（printf 等）は持たず、純粋なバイナリデータのバッファリングに特化しています。
  • BufFile 構造体は palloc によってメモリ確保されるため、明示的な free は不要ですが、BufFileClose を呼び出さないとファイル記述子がリークする可能性があります。
  • 並列実行環境で一時ファイルを使用する場合は、FileSet を介した API を使用してください。
