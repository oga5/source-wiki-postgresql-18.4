src/include/getopt_long.h

概要

getopt_long.h は、PostgreSQL におけるコマンドライン引数解析のためのポータブルなインターフェースを提供します。標準的な getopt に加え、GNU 形式のロングオプション（例: --help）をサポートするためのデータ構造と関数宣言を定義しています。

モジュール内での役割

src/include モジュールの一部として、プラットフォーム間で一貫した引数解析機能を提供します。システムがネイティブな getopt_long を持たない場合に備え、互換性レイヤーとして機能します。

責務

  • ロングオプションを定義するための struct option 構造体の提供。
  • 引数の有無を示す定数（no_argument 等）の定義。
  • getopt_long 関数のプロトタイプ宣言。
  • 環境依存の有無を判定し、必要な場合のみ定義を有効化するガード処理。

重要なシンボル

  • struct option: ロングオプションの名前、引数の要否、フラグの有無を保持する構造体。
  • no_argument / required_argument / optional_argument: オプションが引数を必要とするかを示す定数。
  • getopt_long(): 引数ベクトルを走査し、ロングオプションを解析するメイン関数。

追加の索引済みシンボル

  • option

制御フローとデータフロー

本ファイルはヘッダー定義のみを提供します。実際の解析ロジックは src/port/getopt_long.c に存在します。 1. 呼び出し元は struct option の配列を準備し、getopt_long を呼び出します。 2. getopt_long は内部で optind を管理し、argv を走査します。 3. 非オプション引数に遭遇した場合、argv 内のポインタを並び替えて、オプションと非オプションを分離する処理を行います。

直接依存

  • pg_getopt.h: 標準的な getopt 関連の定義をインポートし、環境に応じた互換性を確保します。

関連コード

  • src/port/getopt_long.c: 本ヘッダーで宣言された getopt_long の実装本体。
  • src/include/pg_getopt.h: getopt 関連の環境依存定義を抽象化するヘッダー。

読者向けメモ

  • 本ヘッダーは HAVE_STRUCT_OPTION や HAVE_GETOPT_LONG マクロによって、OS が提供する標準ライブラリを利用するか、PostgreSQL 側の実装を利用するかが制御されます。
  • getopt_long は argv のポインタを書き換える（非オプションを末尾に移動させる）副作用があるため、呼び出し元の引数配列の扱いに注意してください。
