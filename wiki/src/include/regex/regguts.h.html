src/include/regex/regguts.h

概要

本ファイルは、PostgreSQL の正規表現エンジン（Henry Spencer 氏のパッケージをベースとしたもの）の内部インターフェース定義およびデータ構造を定義するヘッダーファイルです。正規表現のコンパイルおよび実行フェーズで共有される、低レベルなデータ構造やマクロが網羅されています。

モジュール内での役割

src/include/regex モジュールにおける「内部的な基盤」です。regex.h が公開 API を定義するのに対し、regguts.h はコンパイル時の NFA（非決定性有限オートマトン）構築や、実行時の DFA（決定性有限オートマトン）/バックトラッキング処理に必要な詳細構造を定義します。

責務

  • 正規表現エンジン全体で使用されるメモリ管理、アサーション、デバッグ用マクロの抽象化。
  • 文字クラス（Color）の管理およびマッピングロジックの定義。
  • NFA/DFA の状態遷移、アーク（弧）、およびサブ式ツリー（subre）のデータ構造定義。
  • コンパイル済み正規表現の内部状態を保持する guts 構造体の定義。

重要なシンボル

  • struct nfa: コンパイル中に使用される NFA の状態とアークの集合。
  • struct cnfa: 実行時に使用される、メモリ効率を最適化した「コンパクト化された NFA」。
  • struct colormap: 文字コードを内部的な「色（color）」に変換するマッピングテーブル。
  • struct subre: 正規表現の構文解析結果を保持するサブ式ツリー。
  • struct guts: regex_t の内部に隠蔽される、正規表現エンジンの実行時状態。

制御フローとデータフロー

1. **コンパイル**: regcomp が nfa 構造体を構築し、文字を colormap を用いて color に変換します。 2. **最適化**: NFA は cnfa へと変換され、メモリ消費が抑えられます。 3. **実行**: regexec は cnfa を参照し、入力文字列の各文字を GETCOLOR マクロで色に変換しながら状態遷移を行います。 4. **再帰**: 複雑な正規表現（バックリファレンスやルックアラウンド）は subre ツリーを辿ることで解決されます。

直接依存

  • regcustom.h: 環境依存の定義や型定義をオーバーライドするためにインクルードされます。
  • <assert.h>: デバッグ用のアサーション機能を提供します。
  • assert.h

関連コード

  • src/backend/regex/regcomp.c: 本ヘッダーで定義された構造体を使用して NFA を構築します。
  • src/backend/regex/regexec.c: cnfa や guts を使用してマッチングを実行します。
  • src/backend/regex/regc_locale.c: ロケール依存の文字クラス判定ロジックを提供します。

読者向けメモ

  • **メモリ管理**: MALLOC_ARRAY などのマクロは、オーバーフローチェックを伴うメモリ確保を意図していますが、一部の定義には注意が必要です。
  • **Color の概念**: 正規表現エンジンでは、個々の文字を直接扱うのではなく、等価な文字クラスを「色」として抽象化して扱います。WHITE（0）がデフォルトの色である点に注意してください。
  • **構造体の隠蔽**: regex_t の re_fns や re_guts は void * として扱われ、本ファイルの struct guts にキャストされます。このレイヤーを理解することが、エンジン内部のデバッグの鍵となります。
