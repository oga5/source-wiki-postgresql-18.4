src/include/regex/regcustom.h

概要

regcustom.h は、PostgreSQL の正規表現エンジンが外部（PostgreSQL のコア環境）と統合するための抽象化レイヤーです。正規表現ライブラリが環境依存のメモリ管理や文字処理を PostgreSQL の標準機能に委譲するためのブリッジとして機能します。

モジュール内での役割

正規表現エンジン（regguts.h 等）が直接 PostgreSQL の内部関数を呼び出すのではなく、このファイルで定義されたマクロを経由することで、移植性と環境適合性を維持しています。

責務

  • **メモリ管理の抽象化**: palloc/pfree へのマッピング。
  • **文字型の定義**: PostgreSQL の pg_wchar を正規表現エンジンが扱う chr 型として定義。
  • **環境依存のユーティリティ**: CHECK_FOR_INTERRUPTS などの PostgreSQL 特有の制御フローを正規表現エンジンに注入。
  • **文字処理のラップ**: ctype.h の代わりに PostgreSQL のワイド文字対応関数（pg_wc_isalnum 等）をマッピング。

重要なシンボル

  • chr: 正規表現エンジンが扱う文字型（pg_wchar）。
  • MALLOC/FREE/REALLOC: PostgreSQL のメモリコンテキスト管理関数へのエイリアス。
  • INTERRUPT(re): 正規表現の実行中に割り込み（クエリキャンセル等）をチェックするマクロ。
  • CHR_MAX: 正規表現エンジンが扱う文字コードの最大値。
  • MAX_SIMPLE_CHR: カラーマップ（文字クラスの最適化）における「単純な文字」の閾値。

制御フローとデータフロー

本ファイルは実行コードを含まず、マクロ定義の集合体です。 1. **初期化**: 正規表現エンジンが処理を開始する際、メモリ確保が必要になると MALLOC マクロが呼び出され、PostgreSQL の palloc_extended が実行されます。 2. **実行中**: 正規表現の照合ループ内で INTERRUPT マクロが定期的に呼び出され、ユーザーによるキャンセル要求がないかを確認します。 3. **文字処理**: 文字クラスの判定（iscalnum 等）が必要な際、PostgreSQL のワイド文字処理関数が呼び出され、結果がエンジンに返されます。

直接依存

  • postgres.h: PostgreSQL の基本定義（palloc 等）。
  • mb/pg_wchar.h: ワイド文字処理関数。
  • miscadmin.h: CHECK_FOR_INTERRUPTS の提供。
  • regex.h: 本ファイルの最後にインクルードされ、定義されたマクロを利用して正規表現の型を確定させます。
  • ctype.h
  • limits.h
  • wctype.h

関連コード

  • src/include/regex/regguts.h: このファイルで定義されたマクロを実際に消費する正規表現エンジンの内部ヘッダ。
  • src/backend/regex/: 正規表現エンジンの実装本体。

読者向けメモ

  • **コーディング規約の例外**: 本ファイルはヘッダファイルでありながら postgres.h をインクルードしています。これは正規表現ライブラリが PostgreSQL の環境下で動作することを前提とした意図的な設計です。
  • **マクロの多重評価**: CHR_IS_IN_RANGE などのマクロは引数を複数回評価する可能性があるため、副作用のある式を渡さないよう注意してください。
  • **文字型の変更**: chr 型を変更する場合は、CHR_MAX や CHRBITS との整合性に十分注意してください。
