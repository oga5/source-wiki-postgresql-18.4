src/backend/catalog/pg_attrdef.c

概要

本ファイルは、PostgreSQLのシステムカタログ pg_attrdef に対する操作を管理するルーチン群を提供します。pg_attrdef は、テーブルの各列に設定されたデフォルト値（式）を保持するカタログです。

モジュール内での役割

src/backend/catalog モジュールの一部として、テーブル定義の変更（ALTER TABLE 等）に伴うデフォルト値の永続化と、それに関連する依存関係の管理を担います。

責務

  • **デフォルト値の登録**: 列に対するデフォルト式を pg_attrdef に挿入し、対応する pg_attribute のフラグを更新する。
  • **デフォルト値の削除**: 指定された列のデフォルト値をカタログから除去し、依存関係を解消する。
  • **メタデータ検索**: 特定の列や pg_attrdef の OID から、関連する属性情報やデフォルト値の OID を取得する。

重要なシンボル

  • StoreAttrDefault: デフォルト式を文字列化して保存し、pg_attribute を更新して依存関係を記録するメイン関数。
  • RemoveAttrDefault: 特定の列のデフォルト値を削除する。
  • RemoveAttrDefaultById: OID を指定して pg_attrdef エントリを削除する（performDeletion から呼び出される）。
  • GetAttrDefaultOid: 指定した列のデフォルト値の OID を検索する。

追加の索引済みシンボル

  • GetAttrDefaultColumnAddress

制御フローとデータフロー

1. **登録時**: StoreAttrDefault が式を nodeToString でシリアライズし、pg_attrdef に挿入します。その後、pg_attribute の atthasdef を true に更新し、recordDependencyOn を用いてテーブルや式内のオブジェクトとの依存関係を構築します。 2. **削除時**: RemoveAttrDefault が pg_attrdef をスキャンして対象を特定し、performDeletion を介して削除処理を実行します。これにより、依存関係にあるオブジェクトの整合性が保たれます。 3. **検索時**: systable_beginscan を使用して、インデックス（AttrDefaultIndexId 等）経由で効率的にカタログを検索します。

読者向けメモ

  • **生成列の扱い**: 生成列（Generated Column）のデフォルト値は DEPENDENCY_INTERNAL として扱われ、通常のデフォルト値とは削除時の挙動が異なります。
  • **キャッシュの更新**: pg_attrdef を変更すると、pg_attribute の更新を通じてリレーションキャッシュ（relcache）の再構築がトリガーされます。
  • **実装の注意**: ALTER TABLE によるデフォルト値の変更は、内部的に「削除→再作成」のシーケンスで行われるため、フック処理（InvokeObjectPostCreateHookArg）の実装時には注意が必要です。
