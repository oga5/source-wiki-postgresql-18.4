src/backend/catalog/index.c

概要

src/backend/catalog/index.c は、PostgreSQL におけるインデックスのライフサイクル（作成、削除、再構築）を管理する中心的なソースファイルです。システムカタログ（pg_class, pg_index, pg_attribute 等）の更新、物理ストレージの確保、およびインデックス構築処理のディスパッチを担います。

モジュール内での役割

src/backend/catalog モジュールの一部として、DDL コマンド（CREATE INDEX, DROP INDEX, REINDEX）のバックエンド処理を実装します。commands/tablecmds.c などの上位コマンドハンドラと、access/ 以下のインデックスアクセスメソッド（AM）の橋渡しを行います。

責務

  • **インデックス定義の永続化**: index_create を通じたカタログへの登録と、index_drop による削除。
  • **インデックス構築のオーケストレーション**: index_build を介した AM 固有の構築処理の呼び出しと、統計情報の更新。
  • **並行処理（Concurrently）の管理**: CREATE INDEX CONCURRENTLY や DROP INDEX CONCURRENTLY に伴う複雑な状態遷移（indisready, indisvalid 等のフラグ制御）と待機処理。
  • **再構築（REINDEX）の制御**: システムカタログの整合性を維持しつつ、インデックスを再構築する機能。
  • **制約との統合**: PRIMARY KEY や UNIQUE 制約とインデックスの紐付け。

重要なシンボル

インデックス操作（ライフサイクル）

  • index_create(): インデックスのカタログ登録、物理ファイル作成、依存関係の記録を行うメイン関数。
  • index_drop(): インデックスの削除。performDeletion から呼び出されることを想定。
  • index_build(): AM 固有の構築関数を呼び出し、インデックスをデータで埋める。

並行処理（Concurrently）制御

  • index_concurrently_create_copy(): 並行作成用のコピーインデックスを作成。
  • index_concurrently_build(): 並行構築の実行。
  • index_concurrently_swap(): 並行処理の最終段階で、新旧インデックスのカタログ情報を入れ替える。
  • index_set_state_flags(): indisready や indisvalid などのフラグを更新し、インデックスの可視性を制御。

ユーティリティ・情報構築

  • BuildIndexInfo(): インデックスのメタデータ（属性番号、式、述語）を IndexInfo 構造体に変換。
  • FormIndexDatum(): インデックス挿入用のデータ（Datum）を構築。
  • index_check_primary_key(): プライマリキー作成前の制約チェック。

制御フローとデータフロー

1. **作成**: index_create がカタログ（pg_class）を更新し、ConstructTupleDescriptor で属性情報を生成。AppendAttributeTuples で pg_attribute を更新後、UpdateIndexRelation で pg_index にエントリを追加。 2. **構築**: index_build が AM の ambuild 関数を呼び出し、物理ストレージを構築。その後 index_update_stats で pg_class の統計情報を更新。 3. **並行処理**: 複数のトランザクションをまたぎ、index_set_state_flags で段階的にインデックスを「準備完了」「有効」へと遷移させる。 4. **再構築**: reindex_index が既存のインデックスを破棄し、index_build を再実行して物理ファイルを再作成する。

読者向けメモ

  • **メモリ管理**: IndexInfo 構造体は BuildIndexInfo で生成され、インデックス構築のライフサイクル全体で利用されます。
  • **ロック**: インデックス操作は AccessExclusiveLock を多用します。並行処理（Concurrently）では ShareUpdateExclusiveLock を用いて、他の DDL をブロックしつつ DML を許可する設計になっています。
  • **制約**: index_constraint_create はインデックスと制約の依存関係を pg_constraint を通じて管理します。
  • **省略箇所**: 本ファイルは非常に巨大であり、AM 固有の構築ロジック（ambuild）や、カタログ更新の詳細な内部関数（InsertPgClassTuple 等）は、他のモジュールに委譲されています。
