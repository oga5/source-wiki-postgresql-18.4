src/backend/catalog/pg_collation.c

概要

本ファイルは、PostgreSQLのシステムカタログ pg_collation に対する操作を管理するバックエンドモジュールです。主に新しい照合順序（Collation）定義をシステムへ登録するためのロジックを提供します。

モジュール内での役割

src/backend/catalog ディレクトリの一部として、システムカタログの整合性を維持しつつ、DDL（CREATE COLLATION）実行時にカタログテーブルへの書き込みと依存関係の構築を行う役割を担います。

責務

  • pg_collation テーブルへの新規レコード挿入の実行。
  • 照合順序名とエンコーディングの組み合わせにおける一意性制約の検証。
  • 照合順序と名前空間（スキーマ）、所有者、拡張機能間の依存関係の記録。
  • IF NOT EXISTS 句の処理および重複定義時のエラーハンドリング。

重要なシンボル

  • CollationCreate: 本ファイルの主要関数。照合順序の作成に必要な引数を受け取り、カタログへの挿入と依存関係の登録を一括して行います。

制御フローとデータフロー

1. **検証**: GetSysCacheOid3 を使用して、指定された名前とエンコーディングの照合順序が既に存在しないか確認します。 2. **ロック**: table_open で ShareRowExclusiveLock を取得し、競合する同時実行DDLから保護します。 3. **シャドウイングチェック**: 特定エンコーディングと「全エンコーディング（-1）」の照合順序が互いに隠蔽（シャドウイング）しないよう追加検証を行います。 4. **挿入**: heap_form_tuple でタプルを構築し、CatalogTupleInsert でカタログへ書き込みます。 5. **依存関係登録**: recordDependencyOn 等を用いて、名前空間や所有者との関係を pg_depend に記録します。

読者向けメモ

  • 本関数は、libc プロバイダーとそれ以外（icu 等）で必須引数が異なるため、冒頭の Assert でプロバイダーごとの引数整合性をチェックしています。
  • collencoding == -1 は「すべてのエンコーディングで有効」な照合順序を意味します。この特殊なケースと特定のエンコーディングが混在する際の衝突回避ロジックが、このファイルの重要な実装ポイントです。
