src/backend/catalog/objectaddress.c

概要

objectaddress.c は、PostgreSQL のデータベースオブジェクトを抽象的に識別・操作するための中心的なモジュールです。システムカタログ上の任意のオブジェクトを ObjectAddress（classId, objectId, objectSubId の 3 つ組）として表現し、名前解決、権限チェック、およびメタデータ取得の共通インターフェースを提供します。

モジュール内での役割

src/backend/catalog サブシステムにおいて、DDL コマンドや権限管理機能が「どのオブジェクトに対して操作を行うか」を特定するための抽象化レイヤーとして機能します。src/include/catalog/objectaddress.h で定義された構造を実装し、カタログ操作の複雑さを隠蔽します。

責務

  • **オブジェクト識別**: 名前（RangeVar や TypeName）から ObjectAddress への変換。
  • **ロック管理**: オブジェクト解決時に適切なロック（LOCKMODE）を自動的に取得・管理。
  • **メタデータ抽象化**: ObjectPropertyType を用いた、カタログごとの属性（OID, 名前, 所有者, ACL 等）への統一的なアクセス。
  • **権限チェック**: オブジェクト所有権の検証。
  • **説明生成**: ログやエラーメッセージ用の人間が読める形式のオブジェクト記述生成。

重要なシンボル

オブジェクトプロパティ定義

  • ObjectPropertyType: カタログごとのメタデータ（キャッシュID、属性番号、オブジェクト型）を保持する構造体。
  • ObjectProperty: システム内の全オブジェクトクラスを定義した静的配列。

オブジェクト解決・変換

  • get_object_address(): メインのエントリポイント。名前から ObjectAddress を解決し、ロックを取得する。
  • get_object_address_rv(): RangeVar を用いたリレーション関連オブジェクトの解決。
  • read_objtype_from_string(): 文字列から ObjectType 列挙型への変換。

権限・メタデータアクセス

  • check_object_ownership(): 指定されたロールがオブジェクトの所有者であるか検証。
  • get_object_namespace(): オブジェクトが属するスキーマの OID を取得。
  • get_catalog_object_by_oid(): カタログから特定の OID を持つタプルを安全に取得。

制御フローとデータフロー

1. **解決フェーズ**: get_object_address が呼び出されると、objtype に応じて get_relation_by_qualified_name や get_object_address_type などのヘルパー関数が呼び出されます。 2. **ロックフェーズ**: 解決されたオブジェクトに対して、LockDatabaseObject または LockSharedObject を用いてロックを取得します。 3. **再試行ループ**: 同時実行される DDL による競合を防ぐため、ロック取得後に SharedInvalidMessageCounter を確認し、無効化メッセージが処理されていれば再試行します。 4. **メタデータ取得**: get_object_property_data を通じて ObjectProperty 配列から対象カタログの属性情報を引き出し、SearchSysCache を経由してカタログタプルにアクセスします。

読者向けメモ

  • **メモリ管理**: getObjectDescription 等の関数は palloc された文字列を返します。呼び出し側で適切に解放してください。
  • **ロックの生存期間**: get_object_address で取得したロックは、通常トランザクション終了まで保持されることを前提としています。
  • **未実装・浅い部分**: get_object_address_attribute など一部の関数では missing_ok がサポートされておらず、エラーが発生します。また、一部の古いカタログ定義や特殊なオブジェクト型については、ObjectProperty 配列の定義が限定的です。
  • **不変条件**: ObjectAddress の classId は pg_class の OID を指す必要があります。InvalidObjectAddress は初期化やエラー時のデフォルト値として使用されます。
