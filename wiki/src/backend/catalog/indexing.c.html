src/backend/catalog/indexing.c

概要

システムカタログに対するタプル操作（挿入・更新・削除）と、それに伴うインデックスの整合性維持を抽象化するモジュールです。

モジュール内での役割

src/backend/catalog において、カタログ操作の低レベルなインターフェースを提供します。通常のテーブル操作とは異なり、システムカタログ特有の制約や最適化を考慮したラッパーとして機能します。

責務

  • **インデックス管理**: カタログ更新時のインデックス更新処理の自動化。
  • **操作の抽象化**: simple_heap_* 関数群をラップし、インデックス更新ロジックを隠蔽。
  • **パフォーマンス最適化**: *WithInfo 系関数による、インデックスオープン処理のオーバーヘッド削減（複数操作時の再利用）。
  • **整合性チェック**: デバッグビルド時におけるカタログ制約（NOT NULL等）の検証。

重要なシンボル

  • CatalogIndexState: インデックス情報を保持する不透明な構造体（実体は ResultRelInfo）。
  • CatalogOpenIndexes / CatalogCloseIndexes: インデックス更新に必要なリソースの確保と解放。
  • CatalogTupleInsert / CatalogTupleUpdate / CatalogTupleDelete: カタログ操作の主要なエントリーポイント。
  • CatalogTupleInsertWithInfo: 複数行挿入時にインデックス状態を再利用するための最適化版関数。

追加の索引済みシンボル

  • CatalogIndexInsert
  • CatalogTupleCheckConstraints
  • CatalogTupleUpdateWithInfo
  • CatalogTuplesMultiInsertWithInfo

制御フローとデータフロー

1. **初期化**: CatalogOpenIndexes で対象リレーションのインデックス情報をロードし、CatalogIndexState を生成。 2. **操作実行**: simple_heap_* 関数でヒープを更新。 3. **インデックス更新**: CatalogIndexInsert が呼び出され、CatalogIndexState に基づいて各インデックスを更新。

  • HOT更新の場合はインデックス更新をスキップする最適化が含まれる。

    • 4. **終了**: CatalogCloseIndexes でリソースを解放。

直接依存

  • access/heapam.h, access/genam.h: ヒープおよびインデックスアクセスの基本機能。
  • catalog/index.h, catalog/indexing.h: カタログインデックス定義とヘッダー。
  • executor/executor.h: ResultRelInfo 等の実行時情報構造体。
  • utils/rel.h: リレーション記述子の操作。
  • access/htup_details.h
  • access/xact.h
  • postgres.h

関連コード

  • src/include/catalog/indexing.h: 本ファイルの公開インターフェース定義。
  • src/backend/executor/execUtils.c: インデックスオープン/クローズ処理の基盤となる実装。
  • src/backend/access/heap/heapam.c: simple_heap_* 系の実体。

読者向けメモ

  • **制約の制限**: システムカタログでは、式インデックス、部分インデックス、除外制約はサポートされていません。これらは CatalogIndexInsert 内の Assert で保護されています。
  • **HOT更新**: CatalogIndexInsert は HOT (Heap Only Tuple) 更新を検知し、不要なインデックス更新を回避するロジックを持っています。
  • **将来の改善**: 現在、CatalogIndexState のキャッシュ機構は存在しません。複数操作を行う際は *WithInfo 系関数を使用して CatalogOpenIndexes の呼び出し回数を最小限に抑えることが推奨されます。
  • **削除処理**: 現在、カタログの削除操作にはインデックス更新が不要なため、CatalogTupleDelete は単なるラッパーとなっています。
