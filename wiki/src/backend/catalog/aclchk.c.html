src/backend/catalog/aclchk.c

概要

aclchk.c は、PostgreSQL のアクセス制御（ACL: Access Control List）の検証および更新を統括する中心的なファイルです。SQL の GRANT / REVOKE コマンドの実行、ALTER DEFAULT PRIVILEGES の管理、およびシステム内の権限チェック（aclcheck）の基盤を提供します。

モジュール内での役割

src/backend/catalog サブシステムにおいて、権限管理のロジックをカプセル化しています。utils/acl.h で定義された低レベルな ACL 操作関数をラップし、システムカタログ（pg_class, pg_proc, pg_default_acl 等）のメタデータと連携して、オブジェクトごとの権限判定や更新を行います。

責務

  • **権限操作の実行**: GRANT / REVOKE コマンドの解析結果に基づき、対象オブジェクトの ACL を更新。
  • **デフォルト権限の管理**: ALTER DEFAULT PRIVILEGES によるスキーマ単位またはデータベース単位のデフォルト権限設定の永続化。
  • **権限チェックの提供**: 実行時の権限確認（aclcheck）および、特定の権限ビットの有無を判定するマスク処理。
  • **拡張機能の初期権限管理**: pg_init_privs を介した、拡張機能インストール時の初期権限の追跡。

重要なシンボル

権限更新・管理

  • ExecuteGrantStmt: GRANT / REVOKE コマンドのメインエントリーポイント。
  • ExecAlterDefaultPrivilegesStmt: ALTER DEFAULT PRIVILEGES コマンドのハンドラ。
  • merge_acl_with_grant: 既存の ACL に新しい権限を適用（追加/削除）するコア関数。
  • restrict_and_check_grant: 権限付与の妥当性を検証し、警告やエラーを生成する。

権限チェック（マスク処理）

  • pg_aclmask: オブジェクトタイプに応じた権限マスクの計算。
  • object_aclcheck: オブジェクトに対する権限の有無を判定する汎用関数。
  • pg_class_aclmask: テーブル/シーケンスに対する権限判定。
  • pg_attribute_aclmask: 列単位の権限判定。

データ構造

  • InternalGrant: GRANT / REVOKE 文の内部表現。
  • InternalDefaultACL: ALTER DEFAULT PRIVILEGES の情報を保持する構造体。

制御フローとデータフロー

1. **コマンド解析**: ExecuteGrantStmt がパーサーからの GrantStmt を受け取り、InternalGrant 構造体に変換します。 2. **オブジェクト特定**: objectNamesToOids が名前解決を行い、対象オブジェクトの OID リストを生成します。 3. **権限適用**: ExecGrant_Relation や ExecGrant_common などの型別ハンドラが、対象カタログのタプルをロックし、merge_acl_with_grant を呼び出して ACL を更新します。 4. **依存関係更新**: updateAclDependencies を通じて、ACL に含まれるロールの依存関係を pg_shdepend に反映させます。 5. **永続化**: CatalogTupleUpdate によりシステムカタログを更新し、CommandCounterIncrement で変更を可視化します。

読者向けメモ

  • **メモリ管理**: merge_acl_with_grant は古い ACL を pfree し、新しい ACL を返すため、呼び出し側でのメモリリークに注意が必要です。
  • **ロック**: objectNamesToOids は AccessShareLock を使用します。並行 DDL 実行時にタプルが更新されるとエラーになる可能性があるため、必要に応じて呼び出し側でより強いロックを確保してください。
  • **システムカタログ**: pg_init_privs への書き込みは、拡張機能のインストール時や binary_upgrade モード時に限定されます。
  • **省略された詳細**: 本ファイルは非常に多機能であるため、各オブジェクトタイプ（LargeObject, Parameter 等）固有のチェックロジックは、個別の ExecGrant_* 関数に委譲されています。特定のオブジェクトタイプに興味がある場合は、該当する ExecGrant_ 関数を直接参照してください。
