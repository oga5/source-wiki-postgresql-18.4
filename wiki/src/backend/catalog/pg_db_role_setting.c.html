src/backend/catalog/pg_db_role_setting.c

概要

本ファイルは、システムカタログ pg_db_role_setting に対する操作を定義するモジュールです。このカタログは、特定のデータベースまたはロール（ユーザー）に紐付く設定値（GUCパラメータ）を永続化するために使用されます。

モジュール内での役割

src/backend/catalog モジュールの一部として、データベースやロール単位の構成設定の管理を担当します。ALTER ROLE ... SET や ALTER DATABASE ... SET などの SQL コマンドが実行された際に、カタログの更新を仲介する役割を担います。

責務

  • **設定の永続化**: ALTER コマンドによる GUC 設定の追加・更新・削除をカタログに反映する。
  • **設定のクリーンアップ**: ロールやデータベースが削除された際、関連する設定エントリをカタログから除去する。
  • **設定の適用**: セッション開始時などに、カタログから該当する設定を読み込み、現在のプロセスに適用する。

重要なシンボル

  • AlterSetting: ALTER 文による設定変更をカタログに反映するメイン関数。
  • DropSetting: 指定されたデータベースまたはロールに関連する設定をカタログから削除する。
  • ApplySetting: カタログから設定を読み込み、ProcessGUCArray を介して現在のプロセスに適用する。

制御フローとデータフロー

1. **更新フロー (AlterSetting)**:

  • table_open でカタログを開き、対象の (databaseid, roleid) を検索。
  • GUCArrayAdd / GUCArrayDelete / GUCArrayReset を使用して、設定配列をメモリ上で操作。
  • 変更後の配列を CatalogTupleUpdate または CatalogTupleInsert でカタログに書き込む。

    • 2. **適用フロー (ApplySetting)**:

  • 指定された databaseid と roleid をキーに systable_beginscan を実行。
  • 取得したタプルから setconfig（GUC設定配列）を抽出し、ProcessGUCArray を呼び出してセッションに反映する。

関連コード

読者向けメモ

  • 本ファイルは pg_db_role_setting カタログの「操作」に特化しています。設定の「解釈」や「適用」の詳細は utils/misc/guc.c に依存しています。
  • ApplySetting は特定の組み合わせ（DBとロール）に対してのみ動作するため、呼び出し側で必要に応じて InvalidOid を指定し、複数回呼び出す設計になっています。
  • setconfig フィールドは text 型の配列として格納されており、TOAST テーブルが自動的に使用される点に注意してください。
