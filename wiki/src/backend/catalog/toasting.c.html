src/backend/catalog/toasting.c

概要

本ファイルは、PostgreSQLにおけるTOAST（The Oversized-Attribute Storage Technique）テーブルの生成と管理を担うバックエンドモジュールです。テーブルの属性値がページサイズを超過する場合に備え、補助的なTOASTテーブルを自動生成・関連付けるためのロジックを提供します。

モジュール内での役割

src/backend/catalog ディレクトリの一部として、テーブル定義の変更（ALTER TABLE）や新規テーブル作成時に呼び出されます。heap.c 等のテーブル作成処理と密接に連携し、システムカタログ（pg_class）の更新や依存関係の記録を行います。

責務

  • **TOASTテーブルの生成**: 指定されたリレーションに対して、chunk_id, chunk_seq, chunk_data を持つ専用テーブルと、そのインデックスを構築します。
  • **依存関係の管理**: メインテーブルとTOASTテーブル間に DEPENDENCY_INTERNAL を設定し、メインテーブル削除時にTOASTテーブルも連鎖削除されるようにします。
  • **条件判定**: テーブルがTOASTテーブルを必要とするか（needs_toast_table）を判断し、カタログテーブルや共有テーブルに対する不適切な生成を防止します。
  • **ブートストラップ対応**: initdb 中の特殊な生成処理をサポートします。

重要なシンボル

  • NewRelationCreateToastTable: 新規テーブル作成時に呼び出される公開インターフェース。
  • AlterTableCreateToastTable: ALTER TABLE 等で後からTOASTテーブルを追加する際に使用。
  • create_toast_table: 内部的な作業用関数。テーブルとインデックスの作成、カタログ更新、依存関係登録の全工程を統括します。
  • needs_toast_table: TOASTテーブルが必要か否かを判定する述語。

追加の索引済みシンボル

制御フローとデータフロー

1. **呼び出し**: NewRelationCreateToastTable 等がテーブルの Oid を受け取り、CheckAndCreateToastTable を経由して create_toast_table を呼び出します。 2. **判定**: needs_toast_table により、対象がTOASTをサポートすべきか確認します。 3. **構築**: heap_create_with_catalog を用いてTOASTテーブルを作成し、続いて index_create でB-treeインデックスを構築します。 4. **カタログ更新**: pg_class の reltoastrelid を更新し、メインテーブルとTOASTテーブルを紐付けます。 5. **依存登録**: recordDependencyOn を呼び出し、内部依存関係を記録します。 6. **完了**: CommandCounterIncrement() を実行し、変更を可視化します。

読者向けメモ

  • **インデックスの重要性**: TOASTテーブルには chunk_id と chunk_seq をキーとするユニークインデックスが必須です。これはスライスアクセス（部分的なデータ取得）を効率化するためです。
  • **属性のストレージ設定**: TOASTテーブル自体のカラム（chunk_data等）は、再帰的なTOAST発生を防ぐため、強制的に TYPSTORAGE_PLAIN に設定されています。
  • **バイナリアップグレード**: IsBinaryUpgrade フラグが立っている場合、通常の判定ロジックをバイパスし、pg_upgrade から提供されたOIDを使用してTOASTテーブルを復元する特殊なパスを通ります。
