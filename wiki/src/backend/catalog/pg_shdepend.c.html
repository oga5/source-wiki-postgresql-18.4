src/backend/catalog/pg_shdepend.c

概要

pg_shdepend.c は、データベースクラスタ全体で共有されるオブジェクト（ロール、テーブルスペース、データベースなど）と、それらに依存するオブジェクト間の関係を管理する pg_shdepend システムカタログの操作を担います。pg_depend がデータベースローカルな依存関係を扱うのに対し、本ファイルはクラスタ横断的な依存関係（特に所有権や権限）を追跡し、DROP や REASSIGN OWNED などの DDL 操作の整合性を保証します。

モジュール内での役割

src/backend/catalog サブシステムの一部として、オブジェクトのライフサイクル管理における「共有依存関係」のレイヤーを提供します。commands/ 以下の DDL ハンドラ（dbcommands.c, tablecmds.c 等）から呼び出され、オブジェクト削除時の連鎖チェックや、所有者変更時の依存関係更新を抽象化します。

責務

  • **依存関係の記録**: オブジェクト間の共有依存関係（所有者、ACL、テーブルスペース等）の挿入・更新・削除。
  • **削除整合性の検証**: オブジェクト削除時に、依存する共有オブジェクトが存在しないかを確認し、エラーメッセージを生成。
  • **一括操作のサポート**: DROP OWNED や REASSIGN OWNED による、特定のロールに関連する全オブジェクトの特定と一括処理。
  • **テンプレートデータベースの複製**: 新規データベース作成時に、テンプレートデータベースの共有依存関係をコピー。

重要なシンボル

依存関係操作（パブリックAPI）

  • recordSharedDependencyOn: 2つのオブジェクト間の共有依存関係を記録。
  • recordDependencyOnOwner / recordDependencyOnTablespace: 特定の属性（所有者、テーブルスペース）の依存関係を記録するラッパー。
  • changeDependencyOnOwner / changeDependencyOnTablespace: 属性変更に伴う依存関係の更新。
  • updateAclDependencies / updateInitAclDependencies: ACL 更新時の依存関係の同期。

削除・一括処理

  • checkSharedDependencies: オブジェクト削除前に依存関係をチェックし、エラー詳細を生成。
  • shdepDropOwned / shdepReassignOwned: ロールに関連するオブジェクトの削除または所有者変更。
  • dropDatabaseDependencies: データベース削除時の関連レコードクリーンアップ。

内部ヘルパー

  • shdepAddDependency / shdepDropDependency: pg_shdepend への低レベルな挿入・削除。
  • shdepChangeDep: 既存の依存関係を更新するワークホース。
  • shdepLockAndCheckObject: 依存先オブジェクトの存在確認とロック。

制御フローとデータフロー

1. **DDL 実行時**: recordSharedDependencyOn 等が呼ばれ、pg_shdepend を RowExclusiveLock で開きます。 2. **更新フロー**: shdepChangeDep がインデックス（SharedDependDependerIndexId）を使用して既存レコードを検索し、存在すれば CatalogTupleUpdate、なければ CatalogTupleInsert を実行します。 3. **削除チェック**: checkSharedDependencies は SharedDependReferenceIndexId を使用して、削除対象オブジェクトを「参照している」レコードをスキャンします。見つかった依存オブジェクトは ShDependObjectInfo 構造体に格納され、shared_dependency_comparator でソートされた後、エラーメッセージとして整形されます。 4. **一括処理**: shdepDropOwned は、指定されたロールを参照する全レコードをスキャンし、ObjectAddresses リストに収集してから performMultipleDeletions で一括削除します。

読者向けメモ

  • **メモリ管理**: shdepReassignOwned 等のループ処理では、メモリリークを防ぐために AllocSetContextCreate を使用してコンテキストを切り替えています。
  • **ロック**: pg_shdepend への操作は常に適切なロック（RowExclusiveLock 等）を伴います。また、依存先オブジェクトの存在確認には shdepLockAndCheckObject を経由して同時削除を防いでいます。
  • **制約**: pg_shdepend は「共有オブジェクト」への依存を記録するため、dbid が 0 のレコードはクラスタ全体で有効です。
  • **未実装/浅い部分**: remoteDep の管理は単純なリスト構造で行われており、データベース数が多い場合のパフォーマンスは考慮されていません（現状の想定では十分と判断）。
