src/backend/catalog/pg_proc.c

概要

pg_proc.c は、PostgreSQLのシステムカタログ pg_proc（関数、プロシージャ、集約関数を管理するテーブル）に対する操作を定義するモジュールです。関数の作成、更新、および各言語（SQL, C, Internal）に応じたバリデーションロジックを提供します。

モジュール内での役割

src/backend/catalog モジュールの一部として、関数定義の永続化と整合性チェックを担います。pg_proc.h で定義されたスキーマに基づき、システムカタログへの挿入・更新処理を抽象化します。

責務

  • **関数定義の管理**: ProcedureCreate 関数による pg_proc へのレコード挿入および更新。
  • **バリデーション**: 関数作成時に、言語ごとの妥当性（C言語のシンボル解決、SQLの構文解析など）を検証する。
  • **依存関係の追跡**: 関数が依存する型、名前空間、言語、変換ルール等の依存関係を pg_depend に記録する。
  • **エラーハンドリング**: SQL関数定義内の構文エラーを、元の CREATE FUNCTION 文のコンテキストにマッピングする。

重要なシンボル

  • ProcedureCreate: 関数定義の登録・更新を行うメイン関数。
  • fmgr_sql_validator: SQL関数の構文および型チェックを行うバリデータ。
  • fmgr_c_validator: C言語関数のライブラリ存在確認とシンボル解決を行うバリデータ。
  • function_parse_error_transpose: エラー位置を元のクエリ文字列へ変換するユーティリティ。

制御フローとデータフロー

1. **作成/更新**: ProcedureCreate が呼び出されると、引数の妥当性チェック（多相型や内部型の使用可否など）が行われます。 2. **カタログ操作**: SearchSysCache で既存定義を確認し、存在すれば CatalogTupleUpdate、なければ CatalogTupleInsert を実行します。 3. **依存関係記録**: record_object_address_dependencies を通じて、関数が依存するオブジェクト（型、言語など）をカタログに登録します。 4. **バリデーション**: languageValidator が指定されている場合、OidFunctionCall1 を介してバリデータが実行され、関数本体の整合性が検証されます。

読者向けメモ

  • **バリデーションのタイミング**: ProcedureCreate の最後でバリデータが呼ばれますが、check_function_bodies が false の場合（ダンプからのリストア時など）、一部のチェックはスキップされます。
  • **エラー位置の調整**: function_parse_error_transpose は、関数本体の文字列内で発生したエラーを、元のSQL文の行・列に変換する複雑なロジックを含んでいます。デバッグ時にはこのマッピング処理が重要になります。
  • **多相型**: check_valid_polymorphic_signature 等のチェックは、関数定義が実行時に解決可能であることを保証するために不可欠です。
