src/backend/port/atomics.c

概要

atomics.c は、PostgreSQL におけるアトミック操作のうち、ハードウェアによる直接的なサポートが不足している環境で必要となる「フォールバック実装」を提供します。主に 64 ビットアトミック操作をスピンロックを用いてシミュレートする役割を担います。

モジュール内での役割

src/include/port/atomics.h が提供するアトミック操作 API のうち、インライン化できない、あるいはプラットフォーム固有の最適化が困難な 64 ビット操作の実装を補完します。このファイルは、システムが PG_HAVE_ATOMIC_U64_SIMULATION を定義している場合にのみコンパイル・リンクされます。

責務

  • **64 ビットアトミック操作のシミュレーション**: ハードウェアが 64 ビットのアトミック命令をネイティブにサポートしていない環境において、スピンロック（slock_t）で保護された排他制御により、アトミックな読み書き・比較交換（CAS）・加算を実現します。
  • **整合性の保証**: 複数のスレッドやプロセスが競合する環境下で、強整合性を持つ比較交換操作を提供します。

重要なシンボル

  • pg_atomic_init_u64_impl: 64 ビットアトミック変数の初期化。スピンロックの初期化を含みます。
  • pg_atomic_compare_exchange_u64_impl: スピンロックを使用して、アトミックな比較交換（CAS）を実行します。
  • pg_atomic_fetch_add_u64_impl: スピンロックを使用して、アトミックな加算と古い値の取得を実行します。

制御フローとデータフロー

1. **初期化**: pg_atomic_init_u64_impl が呼ばれると、対象構造体内のスピンロックを初期化し、値を設定します。 2. **操作実行**: 各操作関数は、対象の pg_atomic_uint64 構造体に含まれる sema（スピンロック）を SpinLockAcquire で取得します。 3. **排他処理**: ロック保持中に共有メモリ上の値を読み取り、条件判定や算術演算を行い、必要に応じて値を更新します。 4. **解放**: SpinLockRelease を呼び出し、ロックを解放して制御を戻します。

直接依存

  • postgres.h: 基本的な型定義やマクロ。
  • miscadmin.h: システム管理用ユーティリティ。
  • port/atomics.h: アトミック操作のインターフェース定義。
  • storage/spin.h: スピンロックの実装（slock_t, SpinLockAcquire 等）。

関連コード

  • src/backend/storage/lmgr/README.barrier: メモリバリアの概念と PostgreSQL での利用指針。

読者向けメモ

  • 本ファイルは、ネイティブなアトミック命令が利用できない環境での「最後の手段」です。パフォーマンスが重要なパスでは、可能な限りネイティブなアトミック操作が利用されるよう、プラットフォーム側のサポート状況を確認してください。
  • pg_atomic_compare_exchange_u64_impl は、多くのアルゴリズムが期待する「強い（strong）」CAS を提供するために、スピンロックによる完全な排他制御を行っています。
