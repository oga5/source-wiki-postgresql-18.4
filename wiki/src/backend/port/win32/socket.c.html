src/backend/port/win32/socket.c

概要

Windows環境におけるPostgreSQLのソケット通信を抽象化し、POSIX準拠のソケットAPIをWin32 API（Winsock2）上でエミュレートするための実装ファイルです。

モジュール内での役割

src/backend/port/win32 モジュールの一部として、Windows特有の非同期イベントモデルと、PostgreSQLのシグナル処理（pgwin32_signal_event）を統合する役割を担います。

責務

  • **シグナル統合**: ブロッキングが発生するソケット操作中に、PostgreSQLのシグナル割り込みを検知・処理する。
  • **エラー変換**: Winsock特有のエラーコード（WSAxxx）を、PostgreSQLが期待する標準的な errno に変換する。
  • **ソケットエミュレーション**: select や recv/send などの標準的なソケット関数を、Windowsのイベント駆動モデル（WSAEventSelect / WaitForMultipleObjectsEx）を用いて再実装する。

重要なシンボル

  • pgwin32_noblock: ソケット操作を強制的に非ブロッキングモードとして扱うためのグローバルフラグ。
  • pgwin32_waitforsinglesocket: 単一ソケットに対する待機処理とシグナル監視を統合する中核関数。
  • TranslateSocketError: Winsockエラーを errno にマップするユーティリティ。
  • pgwin32_select: select() のWin32移植版。

制御フローとデータフロー

1. **シグナル監視**: 各ソケット関数（recv, send, select 等）の冒頭で pgwin32_poll_signals() を呼び出し、保留中のシグナルがあれば即座に処理します。 2. **待機処理**: ブロッキングが必要な場合、WSAEventSelect でソケットイベントをイベントオブジェクトに関連付け、WaitForMultipleObjectsEx を使用して「ソケットの準備完了」または「シグナルイベント」のいずれかを待機します。 3. **復帰**: 待機が解除された後、WSAEventSelect(s, NULL, 0) を呼び出してイベントの関連付けを解除し、必要に応じてエラー変換や再試行を行います。

直接依存

  • postgres.h: PostgreSQLの基本定義およびエラー報告機能。
  • winsock2.h / windows.h (ペアファイル経由): Win32ネットワークAPI。

関連コード

  • src/include/port/win32/sys/socket.h: Winsockヘッダーのインクルードと競合回避を行うペアファイル。
  • src/backend/port/win32/signal.c: シグナルイベントの生成とディスパッチを担当。

読者向けメモ

  • **非ブロッキングの扱い**: Windowsのソケットは常に非ブロッキングモードで開かれますが、PostgreSQLのロジックがブロッキングを期待する場合、このファイル内の関数が内部的にループと待機を管理してブロッキングをエミュレートしています。
  • **UDPの挙動**: pgwin32_waitforsinglesocket 内には、UDP通信時の高負荷下でのロック問題を回避するための特殊なタイムアウト処理が含まれています。
  • **exceptfds**: pgwin32_select は exceptfds をサポートしていません（PostgreSQLで利用されていないため）。
