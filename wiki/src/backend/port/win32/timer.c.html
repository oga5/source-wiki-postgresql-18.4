src/backend/port/win32/timer.c

概要

Windows環境における setitimer のエミュレーションを提供します。PostgreSQLのバックエンドがPOSIX準拠のタイマー機能（ITIMER_REAL）を必要とする際、Win32 APIを用いてこれを実現します。

モジュール内での役割

src/backend/port/win32 ディレクトリの一部として、Unix系OSとWindows間の移植性レイヤーを構成します。他のプラットフォームで標準的に利用可能な setitimer をWindows上で機能させるためのブリッジです。

責務

  • **タイマー専用スレッドの管理**: バックエンド起動時にタイマー管理用の専用スレッドを生成し、タイムアウトを監視します。
  • **シグナルのエミュレーション**: タイムアウト発生時に pg_queue_signal(SIGALRM) を呼び出し、PostgreSQLのシグナルハンドリング機構へ通知します。
  • **スレッド間同期**: メインスレッドからのタイマー設定変更と、タイマー管理スレッドの待機状態を Event オブジェクトと CriticalSection で同期します。

重要なシンボル

  • setitimer: POSIX互換のインターフェース。本ファイルのエントリポイント。
  • pg_timer_thread: タイマーの待機とタイムアウト処理を行うバックグラウンドスレッド関数。
  • timerCommArea: メインスレッドとタイマー管理スレッド間でタイマー設定を共有するための構造体。

追加の索引済みシンボル

  • timerCA

制御フローとデータフロー

1. **初期化**: setitimer が初めて呼ばれると、CreateEvent と CreateThread により管理スレッドが起動します。 2. **設定更新**: setitimer は timerCommArea を更新し、SetEvent を発行して管理スレッドを起床させます。 3. **待機ループ**: pg_timer_thread は WaitForSingleObjectEx を使用して、イベント通知またはタイムアウトを待機します。 4. **タイムアウト処理**: 待機時間が経過すると SIGALRM をキューイングし、次のタイマー設定が来るまで待機状態（INFINITE）に戻ります。

直接依存

  • postgres.h: PostgreSQLの基本定義および ereport 等のユーティリティ。
  • windows.h (間接的): Win32 API (CreateThread, WaitForSingleObjectEx, CRITICAL_SECTION 等)。

関連コード

  • src/backend/port/win32/signal.c: pg_queue_signal が定義されており、本ファイルから発行されたシグナルを処理します。
  • include/port/win32.h: setitimer や itimerval 構造体の定義元。

読者向けメモ

  • **制限事項**: 本実装は ITIMER_REAL のみに対応しており、インターバルタイマー（it_interval）はサポートされていません。
  • **精度**: WaitForSingleObjectEx を使用しているため、ミリ秒単位の精度となります。
  • **スレッド安全性**: timerCommArea へのアクセスは CRITICAL_SECTION で保護されています。デバッグ時には、スレッド間のデッドロックやイベントの取りこぼしに注意してください。
