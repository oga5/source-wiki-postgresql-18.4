src/backend/regex/regcomp.c

概要

regcomp.c は、PostgreSQL の正規表現エンジンにおけるコンパイルフェーズのメインエントリポイントです。正規表現パターンを解析し、実行時に使用される NFA（非決定性有限オートマトン）およびサブ式ツリー（subre）を構築します。

モジュール内での役割

本ファイルは src/backend/regex モジュールの中心的なハブであり、字句解析、カラーマッピング、NFA 構築、ロケール対応といった各サブコンポーネントを #include して統合します。正規表現のコンパイルプロセス全体を統括するオーケストレーターとして機能します。

責務

  • **パターン解析**: parse() 関数を中心に、正規表現の構文を再帰的に解析し、subre ツリーを生成する。
  • **NFA 構築**: 構文要素を NFA の状態と遷移（アーク）に変換する。
  • **最適化**: 構築された NFA に対して、不要な状態の削除や検索効率向上のための変換（makesearch）を行う。
  • **リソース管理**: コンパイル中に使用されるメモリや一時的なデータ構造の割り当てと解放を管理する。

重要なシンボル

  • pg_regcomp(): 公開されているコンパイル関数。regex_t 構造体を初期化し、解析を開始する。
  • struct vars: コンパイル中の状態（NFA、カラーマップ、ツリー、エラー状態など）を保持するコンテキスト構造体。
  • struct subre: 正規表現の構造を表現するツリーノード。キャプチャグループや量指定子の情報を保持する。
  • parse(): 正規表現のトップレベル解析関数。

制御フローとデータフロー

1. **初期化**: pg_regcomp が vars 構造体をセットアップし、ロケール情報を設定する。 2. **字句解析と構文解析**: lexstart() で字句解析を開始し、parse() が再帰的に呼び出されて subre ツリーと NFA を構築する。 3. **最適化**: optimize() および makesearch() を呼び出し、NFA を実行に適した形式に変換する。 4. **コンパクト化**: compact() を使用して、実行エンジンが扱うための圧縮された NFA（cnfa）を生成する。 5. **終了処理**: freev() を通じて、コンパイル中に使用した一時的な作業領域を解放する。

直接依存

  • regex/regguts.h: 共通のデータ構造とマクロ定義。
  • regc_lex.c, regc_color.c, regc_nfa.c, regc_cvec.c, regc_pg_locale.c, regc_locale.c: コンパイルに必要な各機能モジュール。

関連コード

  • regexec.c: コンパイルされた NFA を使用して実際に文字列マッチングを行う実行エンジン。
  • regfree.c: コンパイルされた regex_t 構造体に関連するメモリを解放する関数。

読者向けメモ

  • 本ファイルは非常に多くの内部関数を抱えていますが、まずは pg_regcomp の流れを追い、次に parse() と parseqatom() がどのように subre ツリーを構築しているかを理解するのが近道です。
  • vars 構造体は多くの関数間で共有されるため、そのフィールドの役割を把握することが重要です。
  • REG_DEBUG マクロを有効にすると、コンパイル過程のダンプ出力が可能になり、複雑なパターンの解析に役立ちます。
