src/backend/regex/regexport.c

概要

regexport.c は、コンパイル済みの正規表現が内部で保持する NFA（非決定性有限オートモーテン）の構造および文字クラス（カラー）情報を、外部から参照可能にするためのアクセサ関数群を提供します。

モジュール内での役割

src/backend/regex モジュールにおいて、正規表現エンジンが構築した内部データ構造を外部（主にクエリ最適化やインデックススキャンなどの目的）へ公開するインターフェース層です。

責務

  • **NFA 構造の公開**: 状態数、初期状態、終了状態、および各状態からの遷移（アーク）情報の提供。
  • **文字クラス（カラー）の公開**: 正規表現内で使用される文字セットの定義と、各カラーに含まれる具体的な文字コードの抽出。
  • **LACON（Lookaround Constraints）の透過的処理**: 内部的な先読み・後読み制約を無視し、実質的な遷移先のみを呼び出し元に返す。

重要なシンボル

  • pg_reg_getnumoutarcs / pg_reg_getoutarcs: 特定の状態から遷移可能なアークを取得する。
  • traverse_lacons: LACON を再帰的に辿り、実質的な遷移先を解決する内部ヘルパー。
  • pg_reg_getcharacters: 特定のカラーに属する文字コードのリストを抽出する。

制御フローとデータフロー

1. **状態遷移の解決**: pg_reg_getoutarcs は traverse_lacons を呼び出します。この関数は、LACON（文字を消費しない遷移）を再帰的に探索し、最終的に文字を消費する「実質的な遷移先」のみを抽出して呼び出し元へ返します。 2. **メモリ管理**: すべての関数は呼び出し元が提供したバッファに値を書き込む形式を採用しており、動的なメモリ確保を回避してエラーハンドリングを簡素化しています。 3. **カラー情報の抽出**: pg_reg_getcharacters は、カラーマップを走査し、指定されたカラー ID に一致する文字コードを収集します。

直接依存

  • regex/regguts.h: 正規表現エンジンの内部構造体（cnfa, colormap, guts 等）の定義。
  • regex/regexport.h: 公開 API の宣言。

関連コード

  • src/include/regex/regexport.h: 本ファイルのヘッダー。API の仕様と設計意図が記述されています。
  • src/backend/regex/regcomp.c: NFA が構築される場所。本ファイルが読み取るデータ構造の生成元です。

読者向けメモ

  • **LACON の扱い**: 本モジュールは LACON を「常に満足されるもの」として扱い、スキップします。これは、正規表現の「マッチングの可能性」を判定する用途には十分であるという設計判断に基づいています。
  • **スタックオーバーフロー**: traverse_lacons は再帰関数であるため、極端に深い LACON 構造を持つ正規表現に対しては check_stack_depth() による保護が行われています。
  • **パフォーマンス**: pg_reg_getcharacters は文字マップを線形走査するため、呼び出し頻度が高い場合は注意が必要です。
