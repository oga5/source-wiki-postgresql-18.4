src/backend/regex/regc_nfa.c

概要

正規表現コンパイルプロセスにおける非決定性有限オートマトン（NFA）の構築、最適化、および管理を行うユーティリティ群です。このファイルは regcomp.c に #include される形で利用されます。

モジュール内での役割

src/backend/regex モジュールにおいて、正規表現の構文解析結果を NFA グラフとして表現し、実行可能な形式（cnfa）へ変換する前段階の「グラフ操作エンジン」として機能します。

責務

  • **NFA 構造の管理**: 状態（state）および遷移（arc）の動的なメモリ割り当て、解放、および再利用。
  • **グラフ最適化**: EMPTY 遷移の除去、制約（^, $, AHEAD, BEHIND）の解消、および不要な状態の削除。
  • **グラフ変換**: 最終的な実行エンジンが解釈可能な cnfa（コンパクト NFA）形式への変換。

重要なシンボル

  • newnfa(): NFA の初期化と基本構造（pre/post 状態）の作成。
  • newstate() / newarc(): 状態と遷移の生成。メモリ効率化のためバッチ割り当て（statebatch / arcbatch）を行う。
  • optimize(): NFA を実行可能な形式に整理するメインパイプライン。
  • compact(): 最適化済み NFA を cnfa 構造体に変換する。
  • moveins() / moveouts(): 状態間の遷移を移動・統合する最適化関数。

制御フローとデータフロー

1. **構築**: regcomp.c からの呼び出しにより、newnfa で初期グラフが作成されます。 2. **最適化**: optimize 関数が以下の順序でグラフを変換します。

  • fixempties: EMPTY 遷移を解消し、グラフを簡素化。
  • fixconstraintloops: 制約遷移による無限ループを検出・解消。
  • pullback / pushfwd: 制約を前後に伝播させ、最終的に PLAIN 遷移へ置換。

    • 3. **変換**: compact がグラフを走査し、cnfa 構造体へシリアライズします。

直接依存

  • regguts.h: NFA 構造体、状態、遷移、およびカラーマップの定義。
  • regcomp.c: 本ファイルがインクルードされる親ファイル。

関連コード

  • regc_color.c: カラーマップの管理。NFA の遷移条件（色）の定義に深く関与。
  • regcomp.c: 構文解析を行い、本ファイルの関数を呼び出して NFA を構築する。
  • rege_dfa.c: 本ファイルで生成された cnfa を用いて実際の正規表現マッチングを行う。

読者向けメモ

  • **メモリ管理**: 状態や遷移は statebatch / arcbatch を用いてバッチ割り当てされます。個別の free は行わず、freenfa で一括解放される設計です。
  • **再帰的処理**: deltraverse や duptraverse など、グラフ走査に再帰を使用しています。スタックオーバーフローを防ぐため STACK_TOO_DEEP マクロによるチェックが随所に挿入されています。
  • **最適化の順序**: 制約の解消（pullback/pushfwd）は、グラフのトポロジーを大きく変更するため、fixempties の後に行う必要があります。
