src/backend/tsearch/ts_parse.c

概要

ts_parse.c は、PostgreSQL の全文検索（tsearch）機能におけるテキスト解析と正規化の核となる実装です。入力されたテキストをトークンに分割し、設定された辞書（Dictionary）を通じて語彙（Lexeme）へと変換する一連のプロセスを管理します。

モジュール内での役割

src/backend/tsearch サブシステムにおいて、生のテキストを検索可能な tsvector や headline 形式へ変換するための「パイプライン」として機能します。パーサー（wparser.c 等）が生成したトークンを受け取り、辞書キャッシュ（ts_cache.h）を利用して正規化を行うブリッジの役割を果たします。

責務

  • **テキストのトークン化と正規化**: パーサーから得られたトークンを辞書で処理し、検索に適した語彙に変換する。
  • **辞書チェーンの管理**: 複数の辞書を順次適用し、単語が正規化されるまで、あるいは辞書が使い果たされるまで処理を継続する。
  • **ヘッドライン生成のサポート**: 検索結果のハイライト表示（Headline）に必要な単語の抽出と、クエリとのマッチング判定を行う。

重要なシンボル

  • LexizeData: 辞書処理の現在の状態（辞書チェーン、作業中のトークンリスト、一時的な結果）を保持する構造体。
  • parsetext: 入力テキストを解析し、ParsedText 構造体に語彙を格納するメイン関数。
  • LexizeExec: 辞書チェーンを走査し、単語を正規化するコアロジック。
  • hlparsetext: ヘッドライン生成用にテキストを解析し、クエリとのマッチング情報を付与する。

制御フローとデータフロー

1. **初期化**: parsetext が LexizeInit を呼び出し、設定（TSConfigCacheEntry）に基づいた辞書処理環境を準備します。 2. **トークン抽出**: prstoken を呼び出し、入力テキストから順次トークンを取得します。 3. **辞書適用**: LexizeExec がトークンを辞書に渡します。辞書が「次の単語も必要（getnext）」と判断した場合、LexizeData 内のスタックにトークンを積み上げ、複数単語を跨ぐ正規化（シソーラス等）を可能にします。 4. **結果格納**: 正規化された語彙は ParsedText 構造体にコピーされ、最終的にインデックスや検索結果として利用されます。

直接依存

  • postgres.h: 基本的な PostgreSQL 型定義とマクロ。
  • tsearch/ts_cache.h: 辞書やパーサーの設定キャッシュへのアクセス。
  • tsearch/ts_utils.h: 全文検索用のユーティリティ関数とデータ構造。
  • varatt.h: 可変長データ型の処理。

関連コード

  • src/backend/tsearch/wparser.c: トークン分割の具体的なロジック。
  • src/backend/tsearch/dict.c: 辞書インターフェースの定義と呼び出し。
  • src/backend/tsearch/to_tsany.c: to_tsvector 等の SQL 関数から本ファイルの関数を呼び出すインターフェース。

読者向けメモ

  • **辞書チェーンの挙動**: LexizeExec は、辞書が単語を認識できない場合や、複数の単語を組み合わせて正規化する場合（シソーラスなど）に複雑な状態遷移を行います。デバッグ時は LexizeData の towork（作業中）と waste（処理済み）リストの状態変化を追うのが有効です。
  • **メモリ管理**: LexizeExec 内では palloc が頻繁に使用されます。特に tmpRes や lastRes を介したメモリの受け渡しがあるため、リークには注意が必要です。
  • **制限事項**: MAXSTRLEN を超える単語は、IGNORE_LONGLEXEME マクロの設定に応じて無視されるかエラーとなります。
