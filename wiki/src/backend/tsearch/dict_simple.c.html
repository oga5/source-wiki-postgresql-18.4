src/backend/tsearch/dict_simple.c

概要

dict_simple.c は、PostgreSQL の全文検索機能における「Simple」辞書の実装です。入力されたトークンを小文字化し、ストップワード（除外語）リストと照合して、その単語を保持するか破棄するかを決定する最も基本的な辞書テンプレートを提供します。

モジュール内での役割

tsearch サブシステムにおいて、辞書テンプレートの一つとして機能します。ユーザーが CREATE TEXT SEARCH DICTIONARY を実行する際、このファイルで定義されたロジックが呼び出され、テキストの正規化とフィルタリングを行います。

責務

  • **辞書設定の初期化**: stopwords（除外語リスト）と accept（ストップワードでない場合に単語をそのまま返すか否か）のパラメータを解析・保持する。
  • **トークンの正規化**: 入力文字列を小文字に変換する。
  • **フィルタリング**: ストップワードリストに基づき、単語を「ストップワード（破棄）」、「受理（保持）」、「認識不能」のいずれかに分類する。

重要なシンボル

  • DictSimple: 辞書の設定（ストップワードリストと受理フラグ）を保持する構造体。
  • dsimple_init: 辞書作成時に呼び出される初期化関数。パラメータの検証と設定の読み込みを行う。
  • dsimple_lexize: 実際の辞書処理関数。入力トークンを小文字化し、ストップワード判定を行う。

制御フローとデータフロー

1. **初期化 (dsimple_init)**:

  • dictoptions を走査し、stopwords ファイルパスと accept フラグを読み込む。
  • readstoplist を呼び出し、ストップワードをメモリ上にロードして DictSimple 構造体に格納する。

    • 2. **辞書処理 (dsimple_lexize)**:

  • 入力文字列を str_tolower で小文字化する。
  • searchstoplist を使用して、小文字化した文字列がストップワードリストに含まれるか確認する。
  • ストップワードであれば空の TSLexeme 配列を返し、そうでなければ accept フラグに応じて単語を返すか NULL を返す。

直接依存

  • postgres.h: 基本的な型定義とマクロ。
  • catalog/pg_collation_d.h: ロケール（照合順序）関連の定数。
  • commands/defrem.h: DefElem 構造体（辞書パラメータの解析に使用）。
  • tsearch/ts_public.h: 全文検索用の共通データ構造（TSLexeme 等）。
  • utils/formatting.h: str_tolower 等の文字列操作関数。
  • utils/fmgrprotos.h

関連コード

  • src/backend/tsearch/dict.c: 辞書全般を管理する基盤コード。
  • src/backend/tsearch/ts_utils.c: readstoplist や searchstoplist など、ストップワード処理の共通ユーティリティが定義されている。

読者向けメモ

  • この辞書は非常にシンプルですが、accept = false に設定することで、特定の単語を「認識不能（辞書に存在しない）」として扱うフィルタとしても利用可能です。
  • str_tolower を使用しているため、照合順序（Collation）に依存した小文字化が行われます。
  • メモリ管理は PostgreSQL の palloc を使用しているため、関数終了時に不要なメモリを pfree するフローが重要です。
