src/backend/tsearch/dict.c

概要

dict.c は、PostgreSQL の全文検索システムにおける「辞書（Dictionary）」機能の標準インターフェースを提供します。主に SQL レベルから特定の辞書を用いて単語を語幹抽出（lexize）するための関数を実装しています。

モジュール内での役割

tsearch サブシステムにおいて、辞書はテキストを正規化・トークン化する重要なコンポーネントです。本ファイルは、キャッシュされた辞書定義を呼び出し、ユーザーが指定した単語に対して辞書処理を適用するゲートウェイの役割を果たします。

責務

  • **辞書呼び出しの仲介**: ts_lexize 関数を通じて、指定された OID に対応する辞書キャッシュを検索し、その辞書固有の lexize メソッドを実行します。
  • **結果の変換**: 辞書から返される内部形式（TSLexeme 配列）を、SQL で利用可能な text 型の配列に変換して返却します。
  • **メモリ管理**: 辞書処理中に動的に割り当てられたメモリ（語彙データや配列構造）の解放を確実に行います。

重要なシンボル

  • ts_lexize: 本ファイルの主要なエントリポイント。SQL 関数 ts_lexize(regdictionary, text) として公開されます。
  • TSDictionaryCacheEntry: 辞書のキャッシュ構造体。辞書固有のメソッドポインタや設定データを含みます。
  • DictSubState: 辞書処理の継続状態を管理する構造体。

制御フローとデータフロー

1. **初期化**: lookup_ts_dictionary_cache を呼び出し、辞書 ID からキャッシュエントリを取得します。 2. **辞書実行**: キャッシュ内の lexize 関数ポインタを呼び出します。必要に応じて DictSubState を用いて複数回の呼び出し（継続処理）を行います。 3. **変換**: 辞書から返された TSLexeme 構造体の配列を走査し、各語彙を CStringGetTextDatum でテキスト型に変換します。 4. **構築と解放**: construct_array_builtin で配列を構築し、一時的に確保したメモリを pfree で解放して結果を返却します。

直接依存

  • tsearch/ts_cache.h: 辞書キャッシュの検索および構造体定義。
  • utils/array.h: SQL 配列の構築用。
  • catalog/pg_type.h: TEXTOID などの型定義。
  • postgres.h: PostgreSQL の基本ヘッダー。
  • tsearch/ts_public.h
  • utils/builtins.h

関連コード

  • src/backend/tsearch/ts_cache.c: 辞書キャッシュの管理ロジック。
  • src/backend/tsearch/dict_*.c: 各種辞書（Simple, Ispell, Synonym, Thesaurus）の実装。これらが ts_lexize から呼び出される実体です。

読者向けメモ

  • 本ファイルは主にデバッグや辞書の動作確認用として設計されています。
  • ts_lexize は、辞書が複数の語彙を返す場合（シノニム展開など）にも対応できるよう、DictSubState を介した反復処理をサポートしています。
  • メモリリークを防ぐため、辞書から返された TSLexeme 配列の各要素と配列自体の解放順序には注意が必要です。
