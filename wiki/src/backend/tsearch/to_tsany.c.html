src/backend/tsearch/to_tsany.c

概要

to_tsany.c は、PostgreSQL の全文検索機能において、入力データ（テキスト、JSON、JSONB）を tsvector 型へ変換する関数群、およびクエリ文字列を tsquery 型へ変換する関数群を実装しています。

モジュール内での役割

tsearch サブシステムにおける「変換層」です。生の入力データを解析し、辞書やパーサーを介して正規化されたトークン列を生成し、最終的な検索用データ構造へと構築する役割を担います。

責務

  • **tsvector 生成**: to_tsvector 関連関数により、テキストや JSON 構造から検索可能なベクトルを生成する。
  • **tsquery 生成**: to_tsquery 関連関数により、ユーザーの検索クエリを解析し、演算子（AND, OR, PHRASE）を含むクエリ構造を構築する。
  • **トークン正規化**: uniqueWORD を用いて、重複する単語の統合や位置情報の管理を行う。
  • **JSON/JSONB 統合**: JSON 構造を走査し、各要素を全文検索対象として抽出する。

重要なシンボル

  • make_tsvector: ParsedText（解析済みテキスト）から最終的な TSVector 構造体を構築する。
  • to_tsvector / to_tsquery: SQL から呼び出されるメインの変換関数。
  • pushval_morph: parse_tsquery がクエリ内の各単語を処理する際に呼び出すコールバック関数。
  • MorphOpaque: クエリ解析時に設定（Config OID）や演算子情報を保持するためのコンテキスト構造体。

制御フローとデータフロー

1. **変換開始**: SQL 関数が呼び出されると、parsetext を使用して入力文字列をトークン化します。 2. **解析**: parsetext は設定された辞書を適用し、単語を正規化して ParsedText 構造体に格納します。 3. **構築**:

  • tsvector の場合: uniqueWORD で重複を排除し、make_tsvector でメモリ上にバイナリ形式の TSVector を構築します。
  • tsquery の場合: parse_tsquery が pushval_morph を呼び出し、トークンを演算子ツリーとしてスタックに積み上げます。

    • 4. **終了**: 構築されたデータ構造が Datum として返されます。

直接依存

  • utils/builtins.h: 組み込み関数定義。
  • utils/jsonfuncs.h: JSON/JSONB 走査用関数。

関連コード

  • src/backend/tsearch/wparser.c: 単語パーサーの定義。

読者向けメモ

  • 本ファイルは、tsvector の内部バイナリレイアウト（WordEntry や WordEntryPos）を直接操作するため、メモリ管理とアライメント（SHORTALIGN）に注意が必要です。
  • uniqueWORD 関数は、単語の重複排除だけでなく、検索位置情報のマージも行うため、全文検索の精度に直結する重要なロジックです。
  • JSON 関連の関数は、iterate_jsonb_values を介して再帰的に値を抽出する設計になっています。
