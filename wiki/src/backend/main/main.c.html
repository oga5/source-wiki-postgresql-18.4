src/backend/main/main.c

概要

PostgreSQL サーバープロセスのエントリーポイントとなるファイルです。実行バイナリ（postgres）が起動した際、最初に呼び出される main() 関数を定義しています。

モジュール内での役割

src/backend/main モジュールの中核であり、PostgreSQL の全実行形態（Postmaster、シングルユーザーモード、ブートストラップモード、フォークされたバックエンドなど）の共通初期化処理を担当します。

責務

  • **環境の初期化**: メモリ管理、ロケール設定、スタック深さチェックなどの基盤サブシステムの起動。
  • **プラットフォーム固有のハック**: Windows 等の OS 特有の初期化処理（ソケット準備やエラーハンドリングの調整）。
  • **セキュリティチェック**: ルート権限での実行を禁止するチェック。
  • **ディスパッチ**: コマンドライン引数に基づき、適切なメインルーチン（PostmasterMain 等）へ処理を委譲する。

重要なシンボル

  • main(): 全プロセスの起点。
  • DispatchOption: 起動モードを決定する列挙型。
  • parse_dispatch_option(): 引数から起動モードを判定するヘルパー関数。
  • reached_main: Sanitizer 関連の初期化タイミングを制御するためのフラグ。

制御フローとデータフロー

1. **初期化**: startup_hacks() でプラットフォーム固有の準備を行い、save_ps_display_args() でプロセス名表示用の引数を保存します。 2. **サブシステム起動**: MemoryContextInit() でメモリ管理を有効化し、ロケールを環境変数から読み込みます。 3. **引数解析**: --help や --version などの標準オプションを処理し、必要に応じて check_root() で権限を確認します。 4. **ディスパッチ**: parse_dispatch_option() でモードを特定し、switch 文で PostmasterMain や BootstrapModeMain 等の各モードのメイン関数へ制御を移します。

読者向けメモ

  • このファイルは「何をするか」よりも「どこへ処理を渡すか」を決定するゲートウェイです。
  • main() 内の save_ps_display_args() は、ps コマンド等でプロセス名を表示するために必須の処理であり、他の処理よりも優先して実行される必要があります。
  • __ubsan_default_options は Sanitizer 実行時の環境変数読み込みに関する特殊なワークアラウンドであり、通常の制御フローとは独立しています。
