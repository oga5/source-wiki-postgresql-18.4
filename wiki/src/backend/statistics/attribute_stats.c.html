src/backend/statistics/attribute_stats.c

概要

本ファイルは、PostgreSQLのシステムカタログ pg_statistic に対するリレーション属性の統計情報の操作（挿入、更新、削除）をサポートする関数群を提供します。主に ANALYZE コマンドが生成する統計情報と同様のデータを、外部から直接インポートまたは操作するためのインターフェースを実装しています。

モジュール内での役割

src/backend/statistics モジュールにおいて、個別の属性（カラム）レベルの統計情報の永続化を担う低レベルなユーティリティ層です。他のファイルが拡張統計（extended_stats.c）や依存関係（dependencies.c）を扱うのに対し、本ファイルは単一属性の統計スロット管理に特化しています。

責務

  • pg_statistic テーブルに対する行の挿入・更新（Upsert）および削除。
  • 属性の型情報（データ型、演算子、照合順序）の導出。
  • 外部から提供されたテキスト形式の統計データ（配列など）の型変換と検証。
  • 統計スロット（stakind）の管理と、空きスロットへの適切な配置。

重要なシンボル

  • pg_restore_attribute_stats: 外部から統計情報をインポートするための公開関数。
  • pg_clear_attribute_stats: 特定の属性の統計情報を削除する公開関数。
  • attribute_statistics_update: 統計情報の更新ロジックをカプセル化した内部関数。
  • set_stats_slot: pg_statistic のスロット（stakind1〜stakind5）を管理するヘルパー。
  • attarginfo: 統計更新関数が受け取る引数の定義テーブル。

制御フローとデータフロー

1. **入力解析**: pg_restore_attribute_stats が引数ペアを解析し、attribute_statistics_update へ渡します。 2. **属性検証**: RangeVarGetRelidExtended で対象テーブルをロックし、属性の存在確認と型情報（get_attr_stat_type）を取得します。 3. **データ変換**: text_to_stavalues を使用し、入力されたテキストデータを属性の型に合わせた配列へ変換します。 4. **スロット更新**: set_stats_slot が pg_statistic の既存スロットを検索し、一致する stakind があれば更新、なければ空きスロットを割り当てます。 5. **永続化**: upsert_pg_statistic が CatalogTupleUpdate または CatalogTupleInsert を呼び出し、システムカタログを更新します。

読者向けメモ

  • **スロットの制約**: pg_statistic はスロット数に制限（STATISTIC_NUM_SLOTS）があります。本ファイルは既存スロットの再利用を試みますが、上限を超えるとエラーになります。
  • **エラーハンドリング**: 統計情報の変換失敗は WARNING として報告され、可能な限り他の統計情報の更新を継続する設計になっています。
  • **システム列**: システム列（attnum < 0）に対する統計操作はサポートされていません。
  • **拡張統計との違い**: 本ファイルは単一属性の統計のみを扱います。複数列にまたがる統計については extended_stats.c を参照してください。
