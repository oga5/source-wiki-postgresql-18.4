src/backend/storage/ipc/ipci.c

概要

PostgreSQL のプロセス間通信（IPC）および共有メモリ領域の初期化を統括する中心的なファイルです。Postmaster 起動時やバックエンドプロセスの開始時に、必要な共有メモリのサイズ計算、確保、および各サブシステムのデータ構造の初期化を行います。

モジュール内での役割

src/backend/storage/ipc モジュールにおける「初期化のハブ」です。各サブシステム（バッファマネージャ、ロックマネージャ、WAL 等）が個別に持つ共有メモリ初期化ロジックを、一箇所に集約して実行する役割を担います。

責務

  • **共有メモリサイズの算出**: 全サブシステムの要求サイズを合算し、必要な共有メモリブロックの総量を決定する。
  • **共有メモリの確保と初期化**: OS から共有メモリセグメントを確保し、ShmemIndex を構築する。
  • **サブシステムの初期化**: 各モジュールの ShmemInit 関数を呼び出し、共有メモリ内のデータ構造をセットアップする。
  • **拡張モジュールへの対応**: RequestAddinShmemSpace を通じて、ロード可能なモジュールが必要とする共有メモリ領域の予約を管理する。

重要なシンボル

  • CreateSharedMemoryAndSemaphores: Postmaster が起動時に呼び出すメインの初期化関数。
  • CalculateShmemSize: 全サブシステムのメモリ要求を合算する関数。
  • CreateOrAttachShmemStructs: 各サブシステムの共有メモリ初期化関数を順次呼び出す内部関数。
  • shmem_startup_hook: 共有メモリ初期化完了後に実行されるフック。拡張モジュールが独自の共有メモリを初期化するために使用する。

追加の索引済みシンボル

  • AttachSharedMemoryStructs
  • InitializeShmemGUCs
  • RequestAddinShmemSpace

制御フローとデータフロー

1. **サイズ計算**: CalculateShmemSize が各サブシステムの ShmemSize 関数を呼び出し、総量を算出する。 2. **確保**: PGSharedMemoryCreate により OS からメモリを確保し、InitShmemAllocation でメモリ管理機構を有効化する。 3. **構造体初期化**: CreateOrAttachShmemStructs が各サブシステムの ShmemInit を呼び出し、共有メモリ上のデータ構造（ハッシュテーブルや制御ブロック）を配置・初期化する。 4. **EXEC_BACKEND 対応**: Windows 等の EXEC_BACKEND 環境では、子プロセスが AttachSharedMemoryStructs を通じて既存の共有メモリに再接続し、ポインタを再設定する。

読者向けメモ

  • このファイルは PostgreSQL の「起動シーケンス」の非常に重要な一部です。新しいサブシステムを追加して共有メモリを使用する場合、必ず CalculateShmemSize と CreateOrAttachShmemStructs の両方に登録が必要です。
  • EXEC_BACKEND（主に Windows）とそれ以外で初期化パスが微妙に異なる点に注意してください。CreateOrAttachShmemStructs は両方のパスから呼ばれるため、冪等性が保たれるように設計されています。
