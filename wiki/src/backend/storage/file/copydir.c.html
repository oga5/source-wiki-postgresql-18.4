src/backend/storage/file/copydir.c

概要

copydir.c は、PostgreSQL のデータディレクトリやテーブルスペースの複製など、ファイルシステムレベルでのディレクトリ再帰コピーを安全かつ堅牢に実行するためのユーティリティを提供します。

モジュール内での役割

src/backend/storage/file モジュールにおいて、ファイルシステム操作の抽象化レイヤー（fd.c）を利用し、高レベルなディレクトリ管理機能を提供します。

責務

  • **ディレクトリの再帰的コピー**: 指定されたディレクトリ構造を維持したまま複製します。
  • **データ整合性の保証**: コピー完了後に fsync を実行し、OS キャッシュからディスクへの書き込みを強制します。
  • **コピー手法の最適化**: file_copy_method GUC に応じ、標準的なコピーまたは OS の高速クローン機能（copy_file_range 等）を選択します。
  • **中断への対応**: CHECK_FOR_INTERRUPTS() を随所に配置し、長時間かかるコピー処理中でもシグナルによる中断を可能にします。

重要なシンボル

  • copydir(): ディレクトリコピーのメインエントリポイント。
  • copy_file(): ファイル単位のコピー処理。バッファリングと定期的なフラッシュを行います。
  • clone_file(): OS の機能を利用した高速なファイルクローン（Reflink 等）。
  • file_copy_method: コピー戦略を制御するグローバル変数。

制御フローとデータフロー

1. **ディレクトリ作成**: MakePGDirectory でコピー先を作成。 2. **走査とコピー**: AllocateDir/ReadDir でソースを走査し、ファイルタイプに応じて copy_file または clone_file を呼び出します。 3. **フラッシュ**: コピー完了後、再度ディレクトリを走査し、すべてのファイルに対して fsync_fname を実行して永続化を保証します。 4. **ディレクトリ同期**: 最後にディレクトリ自体を fsync し、メタデータの更新を確定させます。

読者向けメモ

  • **fsync の重要性**: 本ファイルは単なるコピーではなく、データベースのクラッシュリカバリを前提とした「永続化」を重視しています。fsync をスキップする設定（enableFsync）がある場合、整合性は保証されません。
  • **プラットフォーム依存**: clone_file は OS のサポート状況に強く依存します。macOS や Linux の copy_file_range など、プラットフォーム固有の最適化が条件付きコンパイルで実装されています。
  • **バッファリング**: copy_file 内の FLUSH_DISTANCE は、OS のキャッシュ溢れを防ぎつつ、ディスク書き込みを効率化するためのチューニング値です。
