src/backend/storage/file/buffile.c

概要

buffile.c は、PostgreSQL における大規模な一時ファイルのためのバッファ付き I/O インターフェースを提供します。標準ライブラリの stdio に似た機能を提供しますが、fd.c が管理する仮想ファイル記述子（VFD）上で動作し、OS のファイルサイズ制限を超える巨大なファイルを透過的に扱えるように設計されています。

モジュール内での役割

src/backend/storage/file モジュールの一部として、一時的なデータ保持（ソート、ハッシュ結合、パラレルクエリの共有データなど）の基盤を担います。fd.c が低レベルのファイル記述子管理を行うのに対し、本ファイルはアプリケーション層に近いバッファリングと論理的なファイル操作を提供します。

責務

  • **バッファリング**: 物理的な I/O 回数を減らすためのメモリバッファ管理。
  • **巨大ファイル対応**: MAX_PHYSICAL_FILESIZE（1GB）ごとにファイルを分割し、論理的に単一の巨大ファイルとして扱う。
  • **共有・永続化**: FileSet を介したバックエンド間での一時ファイル共有や、トランザクションを跨ぐ一時ファイルの管理。
  • **リソース管理**: ResourceOwner を用いた、エラー発生時やトランザクション終了時の確実なファイルクローズ。

重要なシンボル

  • BufFile: 複数の物理ファイルから構成される論理的なバッファ付きファイルを表現する構造体。
  • BufFileCreateTemp: 一時ファイルを作成する主要なエントリポイント。
  • BufFileRead / BufFileWrite: バッファを介した読み書き操作。
  • BufFileSeek / BufFileTell: 論理的なファイルポインタの操作。
  • MAX_PHYSICAL_FILESIZE: 物理的な分割単位（1GB）。

制御フローとデータフロー

  • **読み込み**: BufFileRead が呼び出されると、バッファ内にデータがあればコピーし、なければ BufFileLoadBuffer が FileRead を呼び出して物理ファイルからバッファへデータをロードします。
  • **書き込み**: BufFileWrite はデータをバッファに蓄積します。バッファが満杯になると BufFileDumpBuffer が呼び出され、必要に応じて extendBufFile で新しい物理セグメントを作成しつつ、ディスクへフラッシュします。
  • **シーク**: BufFileSeek は論理的なオフセットを計算し、必要であれば BufFileFlush でバッファを書き出した後、物理ファイル記述子を切り替えます。

読者向けメモ

  • BufFile は palloc で割り当てられるため、メモリコンテキストの寿命に注意してください。
  • 巨大なファイルを扱う際、BufFileSeek は論理的なオフセットを int と off_t の組み合わせで管理します。
  • BufFileAppend は物理セグメントのリストを連結するだけであり、データの内容をコピーするわけではないため非常に高速ですが、ファイル間に「穴」が生じる可能性がある点に留意してください。
