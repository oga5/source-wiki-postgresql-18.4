src/backend/storage/file/sharedfileset.c

概要

sharedfileset.c は、複数のバックエンド間で共有可能な一時ファイルセットを管理するための機能を提供します。DSM（動的共有メモリ）セグメントと連携し、共有された一時ファイル群のライフサイクルを管理します。

モジュール内での役割

src/backend/storage/file モジュールにおいて、FileSet を拡張し、プロセス間での共有と自動クリーンアップを実現する上位レイヤーとして機能します。

責務

  • **共有ライフサイクル管理**: 参照カウンタ（refcnt）を用いて、最後にデタッチしたバックエンドがファイルを削除する仕組みを提供します。
  • **DSM 連携**: dsm_segment のデタッチ時に自動的にクリーンアップ処理（SharedFileSetOnDetach）が実行されるようフックを登録します。
  • **名前空間の提供**: 複数のバックエンドが同一のファイル名で一時ファイルにアクセスできるディレクトリ構造を提供します。

重要なシンボル

  • SharedFileSet: 共有ファイルセットを表現する構造体。FileSet を内包し、スピンロックと参照カウンタで保護されます。
  • SharedFileSetInit: 新規作成時に呼び出し、参照カウンタを初期化し、DSM デタッチ時のクリーンアップを登録します。
  • SharedFileSetAttach: 既存のセットに後続のバックエンドが参加する際に呼び出し、参照カウンタをインクリメントします。
  • SharedFileSetOnDetach: DSM デタッチ時に呼ばれる内部コールバック。参照カウンタが 0 になった場合にのみ物理的なファイル削除を実行します。

追加の索引済みシンボル

  • SharedFileSetDeleteAll

制御フローとデータフロー

1. **初期化**: SharedFileSetInit が呼ばれると、refcnt が 1 に設定され、DSM デタッチコールバックが登録されます。 2. **アタッチ**: 他のバックエンドが SharedFileSetAttach を呼び出すと、スピンロックで保護された refcnt がインクリメントされます。 3. **デタッチ**: 各バックエンドが終了または DSM からデタッチする際、SharedFileSetOnDetach が実行されます。 4. **クリーンアップ**: refcnt が 0 に達した最後のバックエンドが FileSetDeleteAll を呼び出し、関連するディレクトリとファイルを物理的に削除します。

読者向けメモ

  • 本モジュールは、エラー発生時のクリーンアップパスでも実行されるため、SharedFileSetOnDetach 内でエラーを発生させることは避ける必要があります。
  • 参照カウンタの操作は必ず mutex（スピンロック）で保護されており、並行アクセス時の整合性が保証されています。
  • FileSet が提供するディレクトリ構造をそのまま利用しているため、ファイルシステムレベルでの一時ファイル管理の詳細は fileset.c を参照してください。
