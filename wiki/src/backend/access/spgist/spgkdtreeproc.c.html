src/backend/access/spgist/spgkdtreeproc.c

概要

本ファイルは、SP-GiSTインデックスにおいて Point 型データに対する k-d 木（k-dimensional tree）アルゴリズムを実装するプロシージャ群を提供します。空間データの効率的な分割と検索を可能にします。

モジュール内での役割

src/backend/access/spgist モジュールの一部として、特定のデータ型（Point）に対するインデックスの挙動を定義する「演算クラス（opclass）」のバックエンド処理を担います。

責務

  • **インデックス構成の定義**: spg_kd_config を通じて、k-d 木がプレフィックスとして float8 を使用し、ノードラベルを必要としないことを宣言します。
  • **空間分割**: spg_kd_picksplit により、データの中央値を用いて空間を再帰的に二分割します。
  • **検索の最適化**: spg_kd_inner_consistent を通じて、クエリ条件に基づき不要なサブツリーを枝刈り（pruning）し、検索範囲を絞り込みます。

重要なシンボル

  • spg_kd_config: インデックスのメタデータを設定する関数。
  • spg_kd_picksplit: データを中央値で分割し、プレフィックスを決定する関数。
  • spg_kd_choose: 挿入時にどのノードへ進むかを決定する関数。
  • spg_kd_inner_consistent: 検索時にどのノードを探索すべきかを判定する関数。
  • SortedPoint: 分割時にソートを行うための補助構造体。

追加の索引済みシンボル

  • getSide
  • x_cmp
  • y_cmp

制御フローとデータフロー

1. **挿入時**: spg_kd_picksplit が呼び出され、入力データを x または y 座標でソートし、中央値をプレフィックスとして決定します。その後、spg_kd_choose が各ポイントを左右のノードへ振り分けます。 2. **検索時**: spg_kd_inner_consistent がクエリ条件（scankeys）を評価します。各ノードがクエリ範囲と重なるか判定し、重ならないノードを探索対象から除外します。ORDER BY 句がある場合は、各ノードの境界ボックス（traversalValues）を計算し、距離を算出します。

読者向けメモ

  • **枝刈りのロジック**: spg_kd_inner_consistent 内の which ビットマスクは、探索すべきノードを管理します。which == 0 となった時点で探索を打ち切る効率的な設計になっています。
  • **境界ボックスの扱い**: ORDER BY 句がある場合、再帰的に境界ボックスを計算して traversalValues に保持します。これは、インデックスの深さが増すにつれて空間の絞り込みを行うために不可欠です。
  • **allTheSameの禁止**: k-d 木では中央値による分割が前提であるため、allTheSame（すべての値が同一）という状態は発生しない設計になっています。
