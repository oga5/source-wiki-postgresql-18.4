src/backend/access/spgist/spgvalidate.c

概要

SP-GiST インデックスの演算子クラス（opclass）および演算子ファミリー（opfamily）の定義が、PostgreSQL の要件を満たしているかを検証するバリデータです。CREATE OPERATOR CLASS や ALTER OPERATOR FAMILY 実行時に呼び出されます。

モジュール内での役割

SP-GiST アクセスメソッドの整合性を担保するゲートキーパーです。ユーザーが定義した関数や演算子が、SP-GiST の内部構造（config, choose, picksplit 等）と矛盾しないかを確認します。

責務

  • **シグネチャ検証**: 登録されたサポート関数が、期待される引数型および戻り値型を持っているかを確認する。
  • **整合性チェック**: 必須のサポート関数がすべて定義されているか、また演算子と関数の組み合わせが論理的に矛盾していないかを検証する。
  • **依存関係の調整**: spgadjustmembers を通じて、演算子や関数がカタログに追加される際の依存関係（ハード/ソフト）を適切に設定する。

重要なシンボル

  • spgvalidate(Oid opclassoid): バリデーションのメインエントリポイント。システムカタログを走査し、定義の妥当性を判定する。
  • spgadjustmembers(...): 演算子クラス作成時に、各メンバー（関数・演算子）の依存関係フラグを調整する。
  • SPGIST_CONFIG_PROC 等の定数: SP-GiST が要求する各サポート関数の識別子。

追加の索引済みシンボル

  • spgadjustmembers
  • spgvalidate

制御フローとデータフロー

1. **カタログ取得**: SearchSysCacheList1 を使用し、対象の演算子ファミリーに関連するすべての演算子と関数をメモリにロードする。 2. **関数検証**: proclist をループし、各関数のシグネチャを check_amproc_signature で確認する。特に SPGIST_CONFIG_PROC は、インデックスの構成情報を取得するために実行され、その結果を用いて他の関数の妥当性を検証する。 3. **演算子検証**: oprlist をループし、戦略番号の範囲や ORDER BY 演算子の戻り値型をチェックする。 4. **グループ整合性**: identify_opfamily_groups で生成されたグループリストを走査し、必須関数が欠落していないか、演算子と関数のペアが揃っているかを最終確認する。

読者向けメモ

  • このファイルは、SP-GiST の「ルール」をコード化したものです。新しいサポート関数を追加する場合や、既存の演算子クラスの挙動を理解したい場合に、何が必須で何がオプションかを知るための仕様書として機能します。
  • spgvalidate 内の ereport(INFO, ...) は、エラーを即座に投げるのではなく、検証結果を false にして処理を継続し、可能な限り多くの不整合を一度に報告する設計になっています。
