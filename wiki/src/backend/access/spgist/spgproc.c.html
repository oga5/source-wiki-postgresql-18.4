src/backend/access/spgist/spgproc.c

概要

SP-GiST 演算子クラス（opclass）が共通して利用する、幾何学的な計算補助ルーチンを提供します。主に距離計算やデータ構造のコピーなど、インデックスの構築やスキャン時に必要となる低レベルなユーティリティ関数を実装しています。

モジュール内での役割

spgquadtreeproc.c や spgkdtreeproc.c といった特定のデータ構造向けの実装に対し、共通の幾何計算ロジックを切り出したヘルパーモジュールです。特に、最近傍探索（Nearest Neighbor Search）における距離計算の標準化を担います。

責務

  • **距離計算の抽象化**: 点と点、点とボックス間の距離計算を統一的に提供します。
  • **順序付けスキャン（Order-by Scan）のサポート**: 複数のスキャンキーに対する距離計算結果を配列として生成します。
  • **メモリ管理**: 幾何データ型（BOX 等）のコピーなど、インデックス操作に必要なメモリ確保を伴うユーティリティを提供します。

重要なシンボル

  • spg_key_orderbys_distances: 順序付けスキャンにおいて、インデックスキー（リーフまたは内部ノード）とスキャンキー間の距離を計算する主要関数です。
  • point_box_distance: 点と軸平行なボックス間の最短距離を計算する内部関数です。
  • point_point_distance: マクロとして定義され、point_distance 関数を呼び出して点間の距離を算出します。

追加の索引済みシンボル

  • box_copy

制御フローとデータフロー

1. **スキャン時の呼び出し**: インデックススキャン中に spg_key_orderbys_distances が呼び出されます。 2. **型判定**: 引数 isLeaf に基づき、キーがリーフ（点）か内部ノード（ボックス）かを判定します。 3. **距離計算**: norderbys 分だけループし、各スキャンキーに対して point_point_distance または point_box_distance を適用します。 4. **結果の返却**: 計算された距離の配列を呼び出し元に返します。

直接依存

  • postgres.h: PostgreSQLの基本定義。
  • math.h: isnan や HYPOT などの数学関数。
  • access/spgist_private.h: SP-GiSTの内部構造体定義。
  • utils/geo_decls.h: Point や BOX 型の定義。
  • utils/float.h
  • utils/fmgrprotos.h

関連コード

  • src/backend/access/spgist/spgquadtreeproc.c: 四分木（Quadtree）の実装。本ファイルの距離計算関数を多用します。
  • src/backend/access/spgist/spgkdtreeproc.c: k-d木の実装。同様に幾何計算のコンテキストで参照されます。
  • src/utils/adt/geo_ops.c: point_distance などの基盤となる幾何演算の実装。

読者向けメモ

  • 本ファイルで提供される point_box_distance は、ボックスが軸に平行であることを前提としています。
  • spg_key_orderbys_distances は、SP-GiSTインデックスにおける最近傍探索（KNN）の効率化において中心的な役割を果たします。
  • 新しい幾何データ型をSP-GiSTでサポートする場合、同様の距離計算ロジックをここに追加するか、あるいは演算子クラス固有のサポート関数として実装することを検討してください。
