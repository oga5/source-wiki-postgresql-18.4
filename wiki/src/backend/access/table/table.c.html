src/backend/access/table/table.c

概要

table.c は、PostgreSQL におけるテーブルアクセスのための汎用的なインターフェースを提供するソースファイルです。特定のテーブルアクセス手法（AM: Access Method）に依存しない、リレーションのオープン・クローズといった共通処理を抽象化しています。

モジュール内での役割

src/backend/access/table サブシステムにおいて、上位層（クエリ実行エンジン等）がテーブルを操作する際の「入り口」として機能します。relation_open などの低レベルなリレーション操作をラップし、対象がテーブルであることを保証するガードレールを提供します。

責務

  • **リレーションのオープン/クローズ**: relation_open 系関数を呼び出し、テーブルとして適切なリレーションを取得・解放する。
  • **型検証**: オープンしたリレーションがインデックスや複合型ではないことを検証し、不正なアクセスを防止する。
  • **一貫したインターフェース**: Oid または RangeVar を用いた多様なオープン形式を統一的に提供する。

重要なシンボル

  • table_open: 指定された OID のリレーションをオープンし、テーブルであることを検証して返す。
  • table_openrv: RangeVar を用いてテーブルをオープンする。
  • table_close: リレーションをクローズし、必要に応じてロックを解放する。
  • validate_relation_kind: 内部的な静的関数。リレーションがテーブルとして有効か（インデックスや複合型でないか）をチェックする。

追加の索引済みシンボル

  • table_openrv_extended
  • try_table_open

制御フローとデータフロー

1. **オープン要求**: 上位層が table_open 等を呼び出す。 2. **低レベルオープン**: relation_open 等の relcache 関連関数を呼び出し、Relation 構造体を取得する。 3. **検証**: validate_relation_kind が rd_rel->relkind を検査する。

  • インデックスや複合型であれば ereport(ERROR) を発行。

    • 4. **返却**: 検証を通過した Relation ポインタを呼び出し元へ返す。 5. **クローズ**: table_close が呼ばれると、内部で relation_close を呼び出し、ロック解除とリソース解放を行う。

直接依存

  • postgres.h: 基本的な型定義とマクロ。
  • access/relation.h: relation_open 等の低レベルリレーション操作関数。
  • access/table.h: 本ファイルの公開インターフェース定義。
  • utils/rel.h: Relation 構造体および関連するアクセスマクロ。

関連コード

  • src/backend/access/table/tableam.c: テーブルアクセス手法の具体的な実装（AM API）が定義されており、本ファイルが提供するインターフェースの背後で実際にデータを操作する。
  • src/include/utils/relcache.h: リレーションキャッシュの管理。table_open が依存する relation_open の基盤。

読者向けメモ

  • 本ファイルは非常に薄いラッパーです。テーブル操作の「入り口」としての役割に特化しており、実際のデータ読み書きロジックは含まれていません。
  • validate_relation_kind は、テーブル以外のリレーション（インデックス等）を誤ってテーブルとして開くことを防ぐ重要なチェックポイントです。
  • table_close は単なる relation_close のエイリアスですが、将来的な拡張性やコードの可読性のためにこの層が維持されています。
