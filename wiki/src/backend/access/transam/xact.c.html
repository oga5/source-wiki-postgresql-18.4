src/backend/access/transam/xact.c

概要

xact.c は、PostgreSQLのトランザクション管理システムの最上位層を実装するファイルです。トランザクションのライフサイクル（開始、コミット、アボート、2PC）、サブトランザクションの階層管理、および並列クエリ実行時の状態同期を統括します。

モジュール内での役割

src/backend/access/transam モジュールにおいて、トランザクションの論理的な状態遷移を制御する中心的な役割を果たします。物理的な永続化（CLOG、サブトランザクションIDの永続化）やWALの書き込みについては、他のサブモジュール（clog.c, subtrans.c, xlog.c）に委譲します。

責務

  • **トランザクション状態管理**: TransactionStateData を用いたスタックベースのトランザクション階層管理。
  • **ライフサイクル制御**: StartTransaction, CommitTransaction, AbortTransaction 等による状態遷移の実行。
  • **並列処理の同期**: 並列ワーカーへのトランザクション状態のシリアライズと、ワーカー内での状態復元。
  • **コールバック管理**: トランザクションの開始・終了時に実行される外部モジュール用フックの管理。
  • **リカバリ時の再実行**: WALレコード（xact_redo）に基づいたトランザクション状態の復元。

重要なシンボル

状態管理

  • CurrentTransactionState: 現在のトランザクション状態を指すポインタ。
  • TransactionStateData: トランザクションの階層、メモリコンテキスト、リソースオーナー、XID等を保持する構造体。
  • TransState / TBlockState: トランザクションの内部状態およびSQLレベルのブロック状態を定義する列挙型。

トランザクション識別子

  • GetTopTransactionId() / GetCurrentTransactionId(): XIDの取得と必要に応じた割り当て。
  • AssignTransactionId(): 新規XIDの生成と、親トランザクションへの伝播。

制御フロー

  • StartTransaction() / CommitTransaction() / AbortTransaction(): トランザクションの基本操作。
  • StartSubTransaction() / CommitSubTransaction() / AbortSubTransaction(): サブトランザクション（SAVEPOINT）の制御。
  • CommitTransactionCommand() / AbortCurrentTransaction(): SQLコマンド単位のトランザクション制御。

制御フローとデータフロー

1. **開始**: StartTransaction がメモリコンテキストとリソースオーナーを初期化し、TRANS_INPROGRESS へ遷移。 2. **XID割り当て**: AssignTransactionId が必要に応じてXIDを生成。サブトランザクションの場合、親のXIDを先に確定させる再帰的処理を行う。 3. **コミット**: CommitTransaction がトリガーやポータルを処理し、RecordTransactionCommit でWALレコードを生成。その後、clog.c を更新して永続化。 4. **アボート**: AbortTransaction がリソースオーナーを解放し、RecordTransactionAbort でアボートをWALに記録。 5. **並列処理**: SerializeTransactionState で状態を共有メモリに書き出し、ワーカー側で StartParallelWorkerTransaction がそれを読み込んで復元する。

読者向けメモ

  • **メモリ所有権**: トランザクション状態は TopTransactionContext に保持されます。AbortTransaction は TransactionAbortContext を使用して、メモリ不足時でもアボート処理が完遂できるように設計されています。
  • **不変条件**: 子トランザクションのXIDは必ず親のXIDより後に割り当てられます。AssignTransactionId はこの順序を保証するために再帰的に親を遡ります。
  • **並列処理の制約**: 並列ワーカー内では、新しいXIDの割り当てやコマンドカウンタの変更は禁止されています。これは、リーダーとワーカー間で状態を同期させるための制約です。
  • **省略された詳細**: 本ファイルは非常に巨大であるため、個別のコールバック関数の詳細や、特定のコマンド（PREPARE 等）の複雑なエラーハンドリングの詳細は省略しています。これらは src/backend/access/transam/README を併読することを推奨します。
