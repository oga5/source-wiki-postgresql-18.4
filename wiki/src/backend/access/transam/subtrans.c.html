src/backend/access/transam/subtrans.c

概要

subtrans.c は、PostgreSQL のサブトランザクション（入れ子になったトランザクション）の親子関係を管理する「サブトランザクションログ（SUBTRANS）」の実装です。各トランザクション ID (XID) に対して、その親トランザクション ID を記録・追跡する機能を提供します。

モジュール内での役割

src/backend/access/transam モジュールの一部として、トランザクション管理の階層構造を維持します。pg_xact (CLOG) がトランザクションのコミット状態を保持するのに対し、本モジュールはトランザクションの「親子関係」という動的な情報を保持します。

責務

  • **親子関係の記録**: SubTransSetParent を通じて、サブトランザクションと親の紐付けを行う。
  • **階層の追跡**: SubTransGetParent や SubTransGetTopmostTransaction を通じて、特定の XID から親や最上位のトランザクションを特定する。
  • **SLRU 管理**: SLRU (Simple Least Recently Used) キャッシュ機構を利用し、メモリ上のページ管理、ディスクへのフラッシュ、不要領域の切り捨てを行う。
  • **起動時初期化**: クラッシュリカバリを必要としないため、起動時にアクティブなページをゼロクリアする。

重要なシンボル

  • SubTransCtl: SLRU 管理のための共有メモリ制御構造体。
  • SubTransSetParent(): 子 XID と親 XID を関連付ける主要関数。
  • SubTransGetTopmostTransaction(): 指定した XID の最上位親を再帰的に探索する関数。
  • ExtendSUBTRANS(): 新しい XID が新しいページを必要とする際に、ページをゼロクリアして拡張する。
  • TruncateSUBTRANS(): 不要になった古いサブトランザクション情報をディスクから削除する。

制御フローとデータフロー

1. **書き込み**: SubTransSetParent が呼ばれると、TransactionIdToPage で対象ページを特定し、SLRU キャッシュからスロットを取得。排他ロック下で親 XID を書き込み、ダーティフラグを立てる。 2. **読み取り**: SubTransGetParent は SimpleLruReadPage_ReadOnly を使用してページを読み込み、該当エントリを返す。 3. **探索**: SubTransGetTopmostTransaction は、TransactionXmin を境界として、親を辿るループを実行する。 4. **保守**: TruncateSUBTRANS はチェックポイント時に呼ばれ、TransactionXmin より古いページを SimpleLruTruncate で解放する。

読者向けメモ

  • **永続性の欠如**: pg_subtrans はクラッシュリカバリを目的としていません。再起動時にはデータがゼロクリアされるため、実行中のトランザクション情報のみを保持する一時的な構造であることを理解してください。
  • **循環参照の防止**: SubTransGetTopmostTransaction 内で親を辿る際、不正なデータ構造による無限ループを防ぐためのチェックが含まれています。
  • **SLRU の特性**: ページサイズは BLCKSZ に依存し、XID は 32bit であるため、ページ番号のラップアラウンドを考慮した設計になっています。
