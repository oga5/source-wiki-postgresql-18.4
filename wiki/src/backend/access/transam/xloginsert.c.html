src/backend/access/transam/xloginsert.c

概要

WAL（Write-Ahead Logging）レコードを構築するための低レベル API を提供するモジュールです。PostgreSQL の各リソースマネージャ（rmgr）が変更を記録する際、このファイルで定義された関数群を使用して、変更内容をメモリ上の作業領域に登録し、最終的に単一の WAL レコードとして組み立てます。

モジュール内での役割

src/backend/access/transam サブシステムの一部として、WAL 書き込みの「構築フェーズ」を担います。xlog.c が WAL の物理的な書き込みや永続化を管理するのに対し、本ファイルは「何を記録するか」という論理的なレコード構成を管理します。

責務

  • **WAL レコードの構築**: XLogBeginInsert から始まる一連の登録処理の管理。
  • **バッファとデータの登録**: 変更対象のページ（XLogRegisterBuffer）や、任意のデータ（XLogRegisterData）を一時的な作業領域に蓄積。
  • **レコードの組み立て**: 登録された情報を XLogRecordAssemble でヘッダー、ブロック情報、データチャンクに整理し、CRC を計算して XLogInsertRecord へ渡す。
  • **圧縮処理**: wal_compression 設定に基づき、フルページイメージ（FPI）の圧縮を行う。

重要なシンボル

  • XLogBeginInsert(): レコード構築の開始を宣言。
  • XLogRegisterBuffer(): 変更対象のバッファを登録。
  • XLogRegisterData(): レコード本体のデータを登録。
  • XLogInsert(): 構築したレコードを WAL に書き込む。
  • registered_buffer: 登録された各ブロックのメタデータと状態を保持する構造体。
  • hdr_scratch: レコードヘッダー構築用のメモリ領域。

制御フローとデータフロー

1. **開始**: XLogBeginInsert() で作業領域を初期化。 2. **登録**: XLogRegister* 関数群を呼び出し、registered_buffers 配列や rdatas チェーンに情報を蓄積。 3. **組み立て**: XLogInsert() が XLogRecordAssemble() を呼び出す。

  • 各ブロックの変更内容を走査し、フルページイメージが必要か判断。
  • 必要に応じて圧縮を行い、ヘッダーとデータを XLogRecData チェーンに連結。

    • 4. **書き込み**: 組み立てられた XLogRecData を xlog.c の XLogInsertRecord() へ渡し、WAL ログとして出力。 5. **リセット**: XLogResetInsertion() で作業領域をクリアし、次のレコード構築に備える。

読者向けメモ

  • **クリティカルセクション**: XLogInsert は通常、START_CRIT_SECTION() 内で呼び出されます。メモリ割り当てやエラー発生時の挙動に注意してください。
  • **バッファの登録**: XLogRegisterBuffer を呼ぶ前に、対象のバッファが排他ロックされていることが前提です。
  • **メモリ管理**: InitXLogInsert で確保される xloginsert_cxt は、バックエンドの生存期間中維持されます。大規模なレコードを構築する場合は XLogEnsureRecordSpace で事前に領域を確保してください。
