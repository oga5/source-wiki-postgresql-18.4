src/backend/access/transam/twophase.c

概要

PostgreSQLの「2フェーズコミット（2PC）」をサポートする中核モジュールです。PREPARE TRANSACTION で保留されたトランザクションの永続化、状態管理、およびリカバリ時の再構築を担います。トランザクションの状態は共有メモリ上の配列と、必要に応じて pg_twophase ディレクトリ内のファイルに保持されます。

モジュール内での役割

src/backend/access/transam モジュールの一部として、トランザクションのライフサイクル管理（xact.c）とWAL管理（xlog.c）を橋渡しします。2PCトランザクションは、通常のトランザクションとは異なり、コミット/アボートの指示が後から来るため、その間の状態を永続化する責務を持ちます。

責務

  • **状態管理**: 準備済みトランザクション（GXACT）の共有メモリ管理と、GID（Global Transaction ID）の競合チェック。
  • **永続化**: 準備済みトランザクションのメタデータとリソースマネージャ（RMGR）データをWALまたはディスクファイルへ書き出し。
  • **リカバリ**: クラッシュリカバリ時やスタンバイモードにおける、準備済みトランザクションの再構築とロックの再獲得。
  • **ライフサイクル制御**: PREPARE から COMMIT PREPARED / ROLLBACK PREPARED までの状態遷移の保護。

重要なシンボル

共有メモリ管理

  • TwoPhaseState: 2PCの全状態を保持する共有メモリ上の構造体。
  • GlobalTransactionData: 個々の準備済みトランザクションを表現する構造体。
  • max_prepared_xacts: 準備済みトランザクションの最大数を制御するGUC変数。

ライフサイクル・状態遷移

  • MarkAsPreparing: GIDを予約し、トランザクションを準備状態へ移行させる。
  • MarkAsPrepared: 準備済みトランザクションを有効化し、ProcArray に登録する。
  • LockGXact: 準備済みトランザクションをコミット/アボートするためにロックする。
  • FinishPreparedTransaction: COMMIT PREPARED または ROLLBACK PREPARED を実行するメイン関数。

リカバリ・永続化

  • StartPrepare / EndPrepare: 2PC状態ファイルの構築とWALへの書き出し。
  • CheckPointTwoPhase: チェックポイント時に、メモリ上の準備済みトランザクションをディスクへフラッシュする。
  • restoreTwoPhaseData: リカバリ開始時に pg_twophase ディレクトリをスキャンし状態を復元する。

制御フローとデータフロー

1. **Prepare**: StartPrepare で状態ファイルをメモリ上に構築し、EndPrepare でWALへ書き出し、MarkAsPrepared で ProcArray に登録して永続化を完了させる。 2. **Commit/Abort**: FinishPreparedTransaction が呼ばれると、WALまたはディスクから状態データを読み込み、ProcessRecords を介して各リソースマネージャのコールバック（twophase_postcommit_callbacks 等）を実行し、リソースを解放する。 3. **Checkpoint**: CheckPointTwoPhase は、WAL上の準備レコードが古い場合に、そのデータをディスクファイルへ書き出し、ondisk フラグを立てる。 4. **Recovery**: restoreTwoPhaseData がディスク上のファイルを読み込み、RecoverPreparedTransactions が各リソースマネージャのリカバリコールバックを呼び出してロックを再構築する。

読者向けメモ

  • **メモリ所有権**: GlobalTransactionData は共有メモリ上に配置され、TwoPhaseStateLock によって保護されます。
  • **レイアウト**: 2PC状態ファイルは TWOPHASE_MAGIC で始まるバイナリ形式で、ヘッダー、サブトランザクションID配列、リソースマネージャデータ、CRCの順で構成されます。
  • **エッジケース**: max_prepared_xacts が0の場合、2PC機能は完全に無効化されます。
  • **浅い領域**: 本ファイルは2PCの「管理」に特化しており、各リソースマネージャ（インデックス、ヒープ等）が具体的に何を準備するかは、各RMGRのコールバック関数に委ねられています。
