src/backend/access/brin/brin_validate.c

概要

BRIN（Block Range Index）アクセスメソッドにおける演算子クラス（opclass）および演算子族（opfamily）の定義が、PostgreSQLの期待する仕様に適合しているかを検証するバリデーターです。

モジュール内での役割

brin モジュールにおいて、ユーザーが定義したカスタム演算子クラスが正しく動作するために必要なサポート関数や演算子がすべて揃っているか、またそれらのシグネチャが正しいかを検証する役割を担います。

責務

  • **サポート関数の検証**: brin が必須とする関数（opcinfo, addvalue, consistent, union 等）のシグネチャが正しいかを確認します。
  • **演算子の検証**: 演算子のストラテジー番号が有効範囲内か、また ORDER BY 演算子が含まれていないか（BRINは非対応）を確認します。
  • **整合性チェック**: 演算子族内のデータ型ペアごとに、必要な演算子とサポート関数のセットが過不足なく定義されているかを検証します。

重要なシンボル

  • brinvalidate(Oid opclassoid): このファイルの唯一の公開関数。CREATE OPERATOR CLASS 実行時に呼び出され、検証結果（bool）を返します。

追加の索引済みシンボル

  • brinvalidate

制御フローとデータフロー

1. **メタデータ取得**: pg_opclass および pg_opfamily システムカタログから、対象の演算子クラスに関連するすべての演算子とサポート関数をリストアップします。 2. **関数シグネチャ検証**: check_amproc_signature を使用し、各サポート関数の引数と戻り値の型が期待通りかを確認します。 3. **演算子検証**: check_amop_signature を使用し、演算子の型適合性を確認します。また、BRINでサポートされない ORDER BY 演算子が含まれていないかをチェックします。 4. **グループ整合性チェック**: identify_opfamily_groups を用いて、データ型ごとの演算子・関数のセットをグループ化し、必須項目が欠けていないかを最終判定します。

読者向けメモ

  • このファイルは CREATE OPERATOR CLASS 時にのみ実行されるため、クエリ実行時のパフォーマンスには影響しません。
  • BRINは「必須のサポート関数」と「演算子」のセットが厳密に決まっています。新しいBRIN演算子クラスを追加する際は、このバリデーターが通るように定義する必要があります。
  • 演算子族（opfamily）レベルでのチェックと、個別の演算子クラス（opclass）レベルでのチェックが混在しているため、エラーメッセージが冗長になる場合がありますが、これは amvalidate APIの仕様によるものです。
