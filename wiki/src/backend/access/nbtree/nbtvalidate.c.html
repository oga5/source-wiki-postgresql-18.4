src/backend/access/nbtree/nbtvalidate.c

概要

B-tree インデックスで使用される演算子クラス（opclass）および演算子族（opfamily）の定義が、PostgreSQL の B-tree 実装の要件を満たしているかを検証するモジュールです。

モジュール内での役割

src/backend/access/nbtree において、ユーザーが定義したインデックス構造の整合性を保証するゲートキーパーの役割を果たします。CREATE OPERATOR CLASS や ALTER OPERATOR FAMILY 実行時に呼び出され、不正な定義がシステムカタログに登録されるのを防ぎます。

責務

  • **定義の妥当性検証**: 演算子およびサポート関数のシグネチャ、戦略番号が B-tree の期待する仕様に合致しているかを確認します。
  • **完全性のチェック**: 特定のデータ型ペアに対して、必須の演算子（<, <=, =, >=, >）とサポート関数（比較関数）がすべて揃っているかを検証します。
  • **依存関係の調整**: 演算子や関数を演算子族に追加する際、それらが「演算子クラス」に強く依存すべきか、「演算子族」に緩く依存すべきかを判定します。

重要なシンボル

  • btvalidate(Oid opclassoid): 指定された演算子クラスの整合性を検証するメイン関数。
  • btadjustmembers(...): 演算子や関数を演算子族に追加する際、依存関係（ハード/ソフト）を決定する関数。
  • BTORDER_PROC 等の定数: B-tree が要求する必須のサポート関数番号。

追加の索引済みシンボル

  • btadjustmembers
  • btvalidate

制御フローとデータフロー

1. **検証 (btvalidate)**:

  • システムカタログから演算子族の全メンバー（演算子・関数）をリストアップします。
  • 各メンバーのシグネチャを check_amproc_signature 等で検証します。
  • identify_opfamily_groups を使用して、データ型ごとのグループに分類します。
  • 各グループ内で必須の演算子・関数が揃っているか、またクロス型演算子が網羅されているかをチェックします。

    • 2. **調整 (btadjustmembers)**:

  • 追加対象が「必須サポート関数」または「クロス型演算子」である場合、演算子族へのソフト依存（ref_is_family = true）を設定します。
  • 同一型演算子の場合、適切な演算子クラスが存在すればハード依存（ref_is_hard = true）を割り当てます。

読者向けメモ

  • このファイルは、B-tree がどのような演算子や関数を必要とするかの「仕様書」としても読めます。特に btvalidate 内の switch 文は、B-tree がサポートする全サポート関数を網羅しています。
  • 演算子族の「完全性」チェックは、プランナが等価クラス（Equivalence Classes）から推論を行うために重要です。一部の演算子が欠けていると、クエリ最適化の機会が失われる可能性があるため、厳格にチェックされています。
