src/backend/access/nbtree/nbtsort.c

概要

B-tree インデックスのバルクロード（CREATE INDEX）を効率的に行うための実装です。ヒープから読み込んだタプルをソートし、リーフページから順次構築することで、ツリーをボトムアップに生成します。バッファキャッシュを介さず smgr を直接利用して物理ページを書き込むため、高速かつ WAL 効率が高いのが特徴です。

モジュール内での役割

nbtree サブシステムにおける「インデックス構築」フェーズを担当します。nbtree.c の btbuild エントリポイントから呼び出され、ソート済みデータから B-tree 構造を物理的に配置する役割を担います。

責務

  • **ソートとスプール**: tuplesort.c を利用したタプルのソートと、一意性チェックのための二重スプール管理。
  • **ページ構築**: リーフページおよび内部ノードの動的生成と、ページ分割（ページが満杯になった際の親へのリンク追加）の制御。
  • **並列処理の調整**: ParallelContext を用いた並列インデックス構築のライフサイクル管理（ワーカーの起動、同期、統計情報の集約）。
  • **物理レイアウトの最適化**: フィルファクタに基づくページ充填率の制御、およびサフィックス切り詰めによるインデックスサイズの削減。

重要なシンボル

構築状態管理

  • BTBuildState: インデックス構築全体のコンテキスト。並列構築時はリーダープロセスが保持。
  • BTSpool: ソート対象のタプルを保持する構造体。一意性制約がある場合は死んだタプル用に spool2 を使用。
  • BTPageState: 構築中の各レベルのページ状態。親レベルへのリンクやページ内のオフセットを管理。
  • BTWriteState: 物理書き込み時のグローバル状態。BulkWriteState を通じて smgr と連携。

並列処理

  • BTShared: 共有メモリ上の状態。ワーカー間の同期、進捗、統計情報の集約に使用。
  • BTLeader: リーダープロセス専用の並列コンテキスト管理構造体。
  • _bt_parallel_build_main: 並列ワーカーのメインエントリポイント。

ページ操作

  • _bt_blnewpage: 新規ページを smgr から取得し、B-tree ヘッダーを初期化。
  • _bt_buildadd: ページにタプルを追加。容量不足時はページ分割を行い、親レベルへ再帰的にリンクを追加。
  • _bt_uppershutdown: 構築完了時にルートページを確定させ、メタページを初期化。

制御フローとデータフロー

1. **初期化**: btbuild が BTBuildState を初期化し、_bt_spools_heapscan を呼び出す。 2. **スキャンとソート**: table_index_build_scan がヒープをスキャンし、_bt_build_callback を経由して tuplesort にタプルを投入。 3. **構築**: _bt_leafbuild が tuplesort_performsort を実行後、_bt_load を呼び出す。 4. **ページ書き込み**: _bt_load はソート済みタプルを _bt_buildadd に渡し、ページが溢れるたびに _bt_pagestate で新規ページを作成し、親レベルを構築する。 5. **終了**: _bt_uppershutdown が最上位レベルを処理し、メタページを書き込んで終了。

読者向けメモ

  • **メモリ所有権**: _bt_blwritepage を呼び出すと、ページバッファの所有権は smgr に移ります。
  • **制約**: 本ファイルは FSM（空き領域マップ）を更新しません。構築後のインデックスは「満杯」状態から開始されるため、構築直後の VACUUM は不要です。
  • **省略箇所**: 並列構築時の詳細なロック競合回避や、tuplesort 内部のメモリ管理の詳細は tuplesort.c に委譲されているため、本ファイルでは高レベルな調整のみを記述しています。
  • **注意**: _bt_buildadd は再帰的に親レベルを構築しますが、これはツリーの高さが低い限りスタックオーバーフローのリスクは極めて低いです。
