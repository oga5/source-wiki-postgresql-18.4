src/backend/access/nbtree/nbtsearch.c

概要

nbtsearch.c は、PostgreSQL B-tree インデックスにおける検索アルゴリズムの中核を担うファイルです。ルートページからリーフページへの降下（Tree Descent）、ページ内での二分探索、および並行性制御のための「右移動（Move Right）」アルゴリズムを実装しています。

モジュール内での役割

nbtree モジュールにおいて、インデックススキャン（IndexScan）の開始点およびページ遷移を制御します。nbtree.c が提供する AM API の検索インターフェースを支える下位層であり、物理的なページ構造を抽象化してスキャン状態（BTScanOpaque）に変換します。

責務

  • **ツリー降下**: ルートからリーフまでのパスを特定し、必要に応じてロックを管理しながら移動する。
  • **ページ内探索**: 二分探索（_bt_binsrch）を用いて、スキャンキーに合致するタプル位置を特定する。
  • **並行性制御**: ページ分割や削除と競合する検索操作において、右兄弟への移動やロックの再取得を制御する。
  • **スキャン状態管理**: スキャン中のページ遷移、バッファのピン留め・ロック解除、および重複排除されたタプル（Posting List）の展開。

重要なシンボル

検索・降下制御

  • _bt_search: インデックスをルートからリーフまで降下し、対象リーフページのバッファをロックして返す。
  • _bt_moveright: ページ分割等により現在のページが古くなった場合、右兄弟へ移動して正しい位置を再特定する。
  • _bt_endpoint: インデックスの端（最初または最後）からスキャンを開始するためのエントリポイント。

ページ内探索

  • _bt_binsrch: ページ内のタプルを二分探索し、指定されたキーに対応する OffsetNumber を返す。
  • _bt_binsrch_insert: 挿入時に使用される、キャッシュ可能な増分二分探索。
  • _bt_binsrch_posting: Posting List（重複排除されたタプル）内部を探索する。

比較・ユーティリティ

  • _bt_compare: スキャンキーとインデックス上のタプルを比較する。内部ページとリーフページで異なる比較ルール（マイナス無限大の扱い等）を適用する。

制御フローとデータフロー

1. **初期化**: _bt_first が _bt_preprocess_keys でキーを最適化し、_bt_search を呼び出す。 2. **降下**: _bt_search は _bt_getroot で開始し、_bt_moveright でページ分割を追いかけながら、_bt_binsrch で次の階層のダウンリンクを特定して下位へ進む。 3. **ページ読み込み**: _bt_readpage がページ内のタプルをスキャンし、_bt_checkkeys で条件判定を行う。 4. **遷移**: _bt_next がページ末尾に達すると _bt_steppage を呼び出し、_bt_readnextpage が右（または左）の兄弟ページへ移動してスキャンを継続する。

直接依存

  • access/nbtree.h: B-tree の主要構造体（BTScanOpaque, BTStack 等）の定義。
  • storage/bufmgr.h: バッファ管理およびロック操作。
  • access/relscan.h: インデックススキャン記述子。

関連コード

  • nbtinsert.c: 挿入時のページ分割やロック管理の対となる実装。
  • nbtpage.c: ページレベルの低レイヤー操作（_bt_getbuf, _bt_relbuf 等）。
  • nbtcompare.c: 比較ロジックの基盤。

読者向けメモ

  • **メモリとライフサイクル**: BTStack は降下中に palloc で確保され、_bt_freestack で解放されます。スキャン中のバッファピンは BTScanOpaque で管理され、_bt_drop_lock_and_maybe_pin によって VACUUM との競合を避けるために適宜解放されます。
  • **並行性**: 本ファイルは Lehman & Yao のアルゴリズムに基づき、ロックを保持したまま右へ移動することで、ページ分割中の検索整合性を保証します。
  • **エッジケース**:
  • _bt_lock_and_validate_left は、逆方向スキャン時にページ削除と競合した場合のリカバリを担当する複雑なロジックです。
  • Posting List（重複排除）の展開は、_bt_readpage 内で _bt_setuppostingitems を通じて行われます。
  • **省略箇所**: _bt_checkkeys や _bt_parallel_* 関連の詳細は、検索ロジックのフローを理解する上では補助的なため、本ファイル内では呼び出し側としての記述に留めています。
