src/backend/access/hash/hashfunc.c

概要

PostgreSQLのハッシュインデックスおよびハッシュ結合で使用される、データ型ごとのハッシュ関数群を定義するファイルです。

モジュール内での役割

src/backend/access/hash サブシステムにおいて、特定のデータ型をハッシュ値へ変換するためのインターフェースを提供します。これらの関数は pg_amproc システムカタログに登録され、ハッシュアクセスメソッドの演算子クラスから呼び出されます。

責務

  • 各データ型（整数、浮動小数点、文字列、OID等）に対するハッシュ計算の実装。
  • ハッシュ結合のシード値に対応した「拡張ハッシュ関数（...extended）」の提供。
  • 浮動小数点の -0.0 と 0.0、あるいは NaN のような、ビット表現は異なるが等価とみなされる値に対して、同一のハッシュ値を生成する正規化処理。
  • ロケール依存の文字列ハッシュ計算（照合順序の考慮）。

重要なシンボル

  • hash_any / hash_any_extended: 汎用的なバイト列ハッシュ関数。多くのデータ型で基盤として使用されます。
  • hash_uint32 / hash_uint32_extended: 整数型用の高速なハッシュ関数。
  • hashfloat4 / hashfloat8: 浮動小数点数特有の正規化（ゼロやNaNの扱い）を行う関数。
  • hashtext: ロケール（照合順序）を考慮した文字列ハッシュ関数。

制御フローとデータフロー

1. **入力**: PG_FUNCTION_ARGS マクロを通じて、SQLレベルの引数が渡されます。 2. **正規化**: 必要に応じて、浮動小数点のゼロ正規化や、文字列のロケール変換（pg_strnxfrm）が行われます。 3. **計算**:

  • 固定長データは hash_uint32 等で直接計算。
  • 可変長データや構造体は hash_any を使用してバイト列としてハッシュ化。

    • 4. **出力**: Datum 型として32ビット（または64ビット）のハッシュ値を返却します。

直接依存

  • common/hashfn.h: ハッシュ計算のコアロジック（hash_any等）。
  • utils/builtins.h, utils/float.h: 基本的な型変換や数学関数。
  • utils/pg_locale.h: ロケール依存の文字列処理。
  • varatt.h: 可変長データ（varlena）の操作。
  • postgres.h
  • utils/fmgrprotos.h

関連コード

  • src/include/common/hashfn.h: ハッシュ関数の定義と実装詳細。
  • src/backend/utils/adt/: 各データ型の型固有の演算子や関数が定義されており、複雑な型はここでハッシュ化されることもあります。
  • src/backend/access/hash/hashvalidate.c: 演算子クラスの整合性チェックに使用されます。

読者向けメモ

  • **一貫性の重要性**: ハッシュ関数は、等価な入力に対して常に同じハッシュ値を返す必要があります。特に浮動小数点数の扱いに注意してください。
  • **拡張関数**: ...extended バージョンの関数は、ハッシュ結合のパフォーマンス向上のためにシード値を受け取ります。新しい型を追加する際は、必ず両方のバージョンを実装することが推奨されます。
  • **文字列ハッシュ**: hashtext はロケールが非決定的な場合に pg_strnxfrm を使用して変換を行うため、他の単純なハッシュ関数よりもコストが高い点に留意してください。
