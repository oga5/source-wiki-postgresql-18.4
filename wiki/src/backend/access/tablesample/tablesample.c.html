src/backend/access/tablesample/tablesample.c

概要

tablesample.c は、PostgreSQL の TABLESAMPLE 句をサポートするための補助的なインターフェースを提供するソースファイルです。主に、各サンプリングメソッド（BERNOULLI や SYSTEM など）のハンドラ関数を呼び出し、必要なルーチン構造体を取得する役割を担います。

モジュール内での役割

tablesample モジュールにおける「ゲートウェイ」として機能します。個別のサンプリングアルゴリズムの実装（bernoulli.c, system.c）とは独立しており、実行時に指定されたサンプリングメソッドのハンドラを動的に解決するための共通基盤を提供します。

責務

  • **ハンドラの検証と取得**: 指定された OID に対応するハンドラ関数を呼び出し、返された TsmRoutine 構造体が正当であるかを検証します。
  • **エラーハンドリング**: ハンドラが期待される形式の構造体を返さない場合、適切なエラーを発生させ、システムの一貫性を保護します。

重要なシンボル

  • GetTsmRoutine(Oid tsmhandler): 本ファイルの唯一の公開関数です。tsmhandler（サンプリングメソッドのハンドラ関数）を引数に取り、そのメソッドが提供する TsmRoutine 構造体へのポインタを返します。

追加の索引済みシンボル

  • GetTsmRoutine

制御フローとデータフロー

1. **呼び出し**: クエリ実行エンジンが TABLESAMPLE 句を処理する際、指定されたメソッドのハンドラ OID を引数に GetTsmRoutine を呼び出します。 2. **関数実行**: OidFunctionCall1 を介してハンドラ関数を呼び出します。この際、引数には NULL が渡されます。 3. **検証**: ハンドラから返された Datum を TsmRoutine 構造体にキャストし、IsA マクロを用いて型チェックを行います。 4. **返却**: 検証に成功すれば構造体ポインタを返し、失敗した場合は elog(ERROR, ...) を通じて処理を中断します。

直接依存

  • postgres.h: PostgreSQL の基本定義および標準マクロ。
  • access/tsmapi.h: TsmRoutine 構造体およびサンプリングメソッドのインターフェース定義。

関連コード

  • src/backend/access/tablesample/system.c: SYSTEM サンプリングメソッドの実装。
  • src/backend/access/tablesample/bernoulli.c: BERNOULLI サンプリングメソッドの実装。
  • src/include/access/tsmapi.h: サンプリングメソッドが実装すべき API 仕様（TsmRoutine の定義）。

読者向けメモ

  • このファイルは非常に小規模であり、主に「サンプリングメソッドの動的ロード」という抽象化層を支えるためのユーティリティです。
  • 新しいサンプリングメソッドを追加する場合、このファイル自体を修正する必要はありません。新しいメソッドの実装側で TsmRoutine を正しく定義し、ハンドラ関数を登録するだけで済みます。
  • GetTsmRoutine が呼び出すハンドラ関数は、PG_FUNCTION_INFO_V1 マクロで定義された標準的な PostgreSQL 関数である必要があります。
