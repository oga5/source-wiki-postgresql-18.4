src/backend/access/rmgrdesc/rmgrdesc_utils.c

概要

rmgrdesc_utils.c は、PostgreSQL の WAL（Write-Ahead Log）レコードの内容を人間が読める形式に変換（デコード）する rmgrdesc ルーチン群を支援するための共通ユーティリティ関数群を提供します。

モジュール内での役割

src/backend/access/rmgrdesc ディレクトリ内の各ファイルは、特定のリソースマネージャ（heap, xact, gist 等）の WAL レコードを記述する関数を実装しています。本ファイルは、これらの実装で頻出する「配列形式のデータ出力」という定型処理を抽象化し、コードの重複を排除する役割を担います。

責務

  • WAL レコード内の配列データを一貫したフォーマット（[elem1, elem2, ...]）で StringInfo バッファへ出力する。
  • OffsetNumber や Oid といった、WAL に頻出する特定のデータ型を文字列化するための標準的なヘルパー関数を提供する。

重要なシンボル

  • array_desc: 配列を走査し、各要素に対して指定されたコールバック関数を適用してバッファに書き出す汎用関数。
  • offset_elem_desc: OffsetNumber を文字列として出力する。
  • redirect_elem_desc: リダイレクト情報（オフセットのペア）を出力する。
  • oid_elem_desc: Oid を文字列として出力する。

制御フローとデータフロー

1. 各リソースマネージャの desc 関数が、WAL レコード内の配列データに対して array_desc を呼び出します。 2. array_desc は、配列の先頭アドレス、要素サイズ、要素数、および各要素を処理するコールバック関数（elem_desc）を受け取ります。 3. array_desc はループ内でポインタ演算を行い、各要素のアドレスを計算してコールバックを呼び出します。 4. コールバック関数（offset_elem_desc 等）は、受け取った要素を StringInfo にフォーマットして書き込みます。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • access/rmgrdesc_utils.h: 本ファイルの関数宣言。
  • storage/off.h: OffsetNumber 型の定義。

関連コード

  • src/include/access/rmgrdesc_utils.h: 本ファイルのインターフェース定義。
  • src/backend/access/rmgrdesc/ 配下の各 *desc.c ファイル: 本ユーティリティを利用するクライアントコード。

読者向けメモ

  • 本ファイルは、WAL のデバッグ出力（pg_waldump 等で使用される）のフォーマットを統一するために存在します。
  • 新しい WAL レコードタイプを追加する際、配列データが含まれる場合は、手動でループを書くのではなく array_desc を活用することで、出力形式の整合性を保つことができます。
  • elem_desc コールバックの void *data 引数は、出力時に追加のコンテキスト情報が必要な場合（例：特定のテーブル定義を参照する場合など）に使用できますが、単純な型変換であれば NULL を渡すことが一般的です。
