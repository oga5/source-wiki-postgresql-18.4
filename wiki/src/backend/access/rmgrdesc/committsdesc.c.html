src/backend/access/rmgrdesc/committsdesc.c

概要

本ファイルは、Commit Timestamp（コミットタイムスタンプ）機能に関連するWAL（Write-Ahead Log）レコードの内容を、人間が読み取り可能な形式に変換するための記述子ルーチンを提供します。

モジュール内での役割

src/backend/access/rmgrdesc/ ディレクトリの一部として、PostgreSQLのWAL解析ツール（pg_waldumpなど）がCommit Timestampのログレコードを解釈する際の補助を行います。

責務

  • WALレコードの情報を解析し、デバッグやログ出力用に文字列化する。
  • WALレコードのタイプ（infoフラグ）を識別し、対応する名称を返す。

重要なシンボル

  • commit_ts_desc: WALレコードのペイロードを解析し、StringInfoバッファに詳細情報を書き出します。
  • commit_ts_identify: WALレコードのinfoビットから、操作の種類（ZEROPAGEまたはTRUNCATE）を文字列として返します。

制御フローとデータフロー

1. **識別**: commit_ts_identify が呼び出され、WALレコードのinfoフィールドから操作タイプを特定します。 2. **解析**: commit_ts_desc が呼び出されると、XLogRecGetData でレコードの生データを取得します。 3. **変換**: info の値に基づき、xl_commit_ts_truncate 構造体へのキャストやメモリコピーを行い、appendStringInfo を通じて人間が理解できる形式（ページ番号やXIDなど）に変換します。

直接依存

  • postgres.h: PostgreSQLの基本定義。
  • access/commit_ts.h: Commit Timestamp機能のデータ構造および定数定義。

関連コード

  • src/backend/access/transam/commit_ts.c: Commit Timestamp機能の本体実装。WALレコードが生成される場所です。
  • src/include/access/rmgrdesc.h: リソースマネージャ記述子インターフェースの定義。

読者向けメモ

  • 本ファイルはWALの「表示」のみを担っており、実際のコミットタイムスタンプの更新ロジックは含みません。
  • COMMIT_TS_ZEROPAGE と COMMIT_TS_TRUNCATE という2つの主要なWAL操作のみを扱っています。これら以外の操作が追加された場合は、本ファイルの switch 文および if 文を拡張する必要があります。
