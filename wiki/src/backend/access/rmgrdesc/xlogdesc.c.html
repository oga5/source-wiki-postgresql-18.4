src/backend/access/rmgrdesc/xlogdesc.c

概要

本ファイルは、PostgreSQLのWAL（Write-Ahead Log）における「XLOG」リソースマネージャの記述子ルーチンを提供します。主に pg_waldump などのツールがWALレコードの内容を人間が読める形式に変換する際や、デバッグ目的でログの詳細を出力する際に使用されます。

モジュール内での役割

src/backend/access/rmgrdesc モジュールの一部として、XLOG固有のWALレコードタイプを識別し、その内容を文字列として整形する責務を担います。他のリソースマネージャ（heapdesc.c や xactdesc.c など）と同様に、WALの可視化を支える重要なコンポーネントです。

責務

  • **WALレコードの識別**: xlog_identify により、WALレコードの info フィールドからレコードの種類（CHECKPOINT, NEXTOID, BACKUP_END 等）を特定する。
  • **レコード内容の記述**: xlog_desc により、特定のWALレコードが保持するデータ（チェックポイント情報やパラメータ変更内容など）を StringInfo バッファに書き出す。
  • **ブロック参照情報の生成**: XLogRecGetBlockRefInfo により、WALレコードに含まれるブロック参照（FPI: Full Page Imageを含む）を詳細または簡潔な形式で出力する。
  • **GUC値の文字列変換**: wal_level などの列挙型設定値を人間が理解可能な文字列に変換する。

重要なシンボル

  • xlog_desc: WALレコードの内容を解析し、人間が読める形式で文字列化するメイン関数。
  • xlog_identify: WALレコードの info ビットから、そのレコードが何を表すかのラベルを返す。
  • XLogRecGetBlockRefInfo: WALレコード内のブロック参照情報を走査し、FPIの有無や圧縮状態を含めて詳細に記述する。
  • wal_level_options: wal_level 設定値と文字列の対応を定義する構造体配列。

追加の索引済みシンボル

  • get_wal_level_string

制御フローとデータフロー

1. **識別**: xlog_identify が info ビットをマスクし、対応する文字列ラベルを返します。 2. **記述**: xlog_desc が XLogRecGetData でレコードのペイロードを取得し、info に基づいて適切な構造体（CheckPoint や xl_parameter_change 等）にキャストして内容をバッファへ追記します。 3. **詳細解析**: XLogRecGetBlockRefInfo は XLogReaderState を走査し、各ブロックのタグ情報やFPIの圧縮方式（pglz, lz4, zstd）を判定して、詳細なレポートを生成します。

  • utils/guc.h: GUC設定値の管理。
  • utils/timestamp.h: タイムスタンプの文字列変換。
  • access/transam.h

関連コード

  • src/bin/pg_waldump/: 本ファイルのルーチンを呼び出してWALの内容を表示する主要なクライアントツール。

読者向けメモ

  • 本ファイルはWALの「解釈」のみを行っており、WALの書き込みやリカバリ処理そのものには関与しません。
  • XLogRecGetBlockRefInfo 内のFPI圧縮判定ロジックは、WALのストレージ効率を理解する上で非常に参考になります。
  • 新しいXLOGレコードタイプを追加した際は、必ず xlog_identify の switch 文と xlog_desc の条件分岐を更新する必要があります。
