src/backend/access/rmgrdesc/gistdesc.c

概要

GiST（Generalized Search Tree）インデックスに関連するWAL（Write-Ahead Log）レコードの内容を、人間が読める形式に変換（デコード）するためのルーチン群を提供します。

モジュール内での役割

src/backend/access/rmgrdesc モジュールの一部として、GiST固有のWALレコードを識別し、pg_waldump などのツールで詳細を表示可能にするためのインターフェースを実装しています。

責務

  • GiSTのWALレコードタイプ（XLOG_GIST_*）の識別。
  • 各WALレコード構造体（gistxlog*）の内容を StringInfo バッファへ整形して出力する。
  • WALログのデバッグおよびリカバリ時の解析支援。

重要なシンボル

  • gist_desc: WALレコードの情報を解析し、詳細な文字列を生成するメイン関数。
  • gist_identify: WALレコードの info ビットから、対応する操作名（文字列）を返す関数。
  • out_gistxlog*: 各種GiST操作（Split, Delete, PageUpdate等）に対応する内部的な整形関数。

追加の索引済みシンボル

  • out_gistxlogDelete
  • out_gistxlogPageDelete
  • out_gistxlogPageReuse
  • out_gistxlogPageSplit
  • out_gistxlogPageUpdate

制御フローとデータフロー

1. **識別**: gist_identify がWALレコードの info ビットを受け取り、操作の種類を特定します。 2. **ディスパッチ**: gist_desc が XLogRecGetInfo で取得した情報に基づき、適切な out_gistxlog* 関数を呼び出します。 3. **整形**: 各 out_gistxlog* 関数が、XLogRecGetData で取得したバイナリデータを StringInfo バッファに書き込みます。

直接依存

  • postgres.h: PostgreSQLの基本定義。
  • access/gistxlog.h: GiSTのWALレコード構造体定義。
  • lib/stringinfo.h: 文字列バッファ操作用ユーティリティ。

関連コード

  • src/backend/access/gist/gistxlog.c: GiSTのWALレコードが実際に生成・適用される場所。
  • src/include/access/gist_xlog.h: GiSTのWALレコード構造体の定義元。

読者向けメモ

  • 本ファイルは「表示用」のコードであり、実際のインデックス操作ロジックは含みません。
  • out_gistxlogPageUpdate は現在空の実装ですが、将来的な拡張やデバッグ情報の追加が想定されています。
  • 新しいGiSTのWALレコードタイプを追加した場合は、必ず gist_desc と gist_identify の両方にケースを追加する必要があります。
