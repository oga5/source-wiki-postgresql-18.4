src/backend/access/rmgrdesc/dbasedesc.c

概要

データベースの作成および削除操作に関連する WAL（Write-Ahead Log）レコードの内容を、人間が読み取り可能な文字列に変換するための記述子ルーチンを提供します。

モジュール内での役割

src/backend/access/rmgrdesc モジュールの一部として、rmgr（Resource Manager）が生成する WAL レコードのデコード処理を担います。主に pg_waldump などのツールが WAL を解析する際に呼び出されます。

責務

  • **WAL レコードの可読化**: dbase_desc 関数を通じて、データベース操作（作成・削除）の WAL レコードから、テーブルスペース ID やデータベース ID などの情報を抽出し、文字列として出力する。
  • **レコードタイプの識別**: dbase_identify 関数を通じて、WAL レコードの info ビットから、それがどのデータベース操作に対応するかを識別する文字列を返す。

重要なシンボル

  • dbase_desc: WAL レコードを解析し、StringInfo バッファに詳細な説明を書き込むメイン関数。
  • dbase_identify: WAL レコードの info フィールドを解析し、操作の種類（CREATE_FILE_COPY, CREATE_WAL_LOG, DROP）を文字列で返す。

制御フローとデータフロー

1. **識別**: dbase_identify が WAL レコードの info ビットをチェックし、操作の種類を特定します。 2. **解析**: dbase_desc が XLogRecGetData を介してレコードのペイロードを取得します。 3. **変換**: info の値に応じて、適切な構造体（xl_dbase_create_file_copy_rec 等）にキャストし、appendStringInfo を用いてバッファへ情報を整形・出力します。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • commands/dbcommands_xlog.h: データベース操作に関連する WAL レコード構造体の定義。
  • lib/stringinfo.h: 文字列操作およびバッファ管理機能。

関連コード

  • src/backend/commands/dbcommands.c: データベース操作の本体。ここで生成された WAL レコードが本ファイルでデコードされます。
  • src/include/access/xlogreader.h: XLogReaderState や XLogRecGetData など、WAL 解析の基盤となる構造体と関数が定義されています。

読者向けメモ

  • 本ファイルは WAL の解析（デバッグやリカバリの追跡）に特化しており、データベース操作のロジック自体は含みません。
  • XLR_INFO_MASK を用いたビットマスク処理は、WAL レコードのフラグ部分を除外して操作タイプを特定するために必須です。
  • 新しいデータベース操作タイプが dbcommands_xlog.h に追加された場合、本ファイルの switch 文および if 分岐を更新する必要があります。
