src/backend/access/gin/gininsert.c

概要

GIN（Generalized Inverted Index）インデックスへのデータ挿入および構築を統括するモジュールです。単一タプルの挿入（gininsert）から、インデックス作成時のバルク構築（ginbuild）、および並列インデックス構築の調整ロジックまでを網羅します。

モジュール内での役割

src/backend/access/gin サブシステムにおける「書き込み」の入り口です。ginbtree.c（B-tree操作）や ginfast.c（ペンディングリスト）といった下位層を呼び出し、インデックスの物理的な更新を制御します。

責務

  • **挿入制御**: FastUpdate（ペンディングリスト）の有無に応じた挿入パスの振り分け。
  • **インデックス構築**: ginbuild を通じた初期構築。メモリ上の BuildAccumulator を用いた効率的なキー集約と、必要に応じたインデックスへのフラッシュ。
  • **並列構築**: GinLeader および GinBuildShared を用いた並列ワーカーの管理、結果のマージ、および共有メモリ上の Tuplesort を介した同期。
  • **リーフタプル管理**: ポスティングリストの圧縮と、リストが溢れた際のポスティングツリーへの昇格ロジック。

重要なシンボル

インデックス構築・状態管理

  • GinBuildState: 構築中の全状態（メモリコンテキスト、BuildAccumulator、並列リーダー情報）を保持するメイン構造体。
  • GinBuildShared: 並列構築時に共有メモリへ配置される、ワーカー間同期のための構造体。
  • GinLeader: 並列構築のリーダープロセスが保持する制御用構造体。

挿入・構築ロジック

  • gininsert: インデックス挿入の公開API。
  • ginbuild: インデックス構築の公開API。
  • ginEntryInsert: 指定されたキーとTIDリストをインデックスへ挿入する中核関数。
  • ginHeapTupleBulkInsert: 構築時にキーを抽出し、BuildAccumulator へ追加する。

並列処理

  • _gin_begin_parallel / _gin_end_parallel: 並列構築のライフサイクル管理。
  • _gin_parallel_merge: リーダープロセスがワーカーからの結果をマージする処理。
  • _gin_parallel_scan_and_build: ワーカープロセスが実行するスキャンとソートのメインループ。

制御フローとデータフロー

1. **挿入フロー**: gininsert → ginHeapTupleFastInsert（高速挿入）または ginHeapTupleInsert → ginEntryInsert → ginInsertValue（B-tree挿入）。 2. **構築フロー**: ginbuild → table_index_build_scan → ginBuildCallback → ginHeapTupleBulkInsert → ginEntryInsert。 3. **並列構築フロー**:

  • リーダーが GinBuildShared を初期化しワーカーを起動。
  • 各ワーカーが ginBuildCallbackParallel を経由して Tuplesort にデータを書き込む。
  • ワーカーが _gin_process_worker_data でデータをマージし、共有 Tuplesort へ出力。
  • リーダーが _gin_parallel_merge で全ワーカーの結果をマージし、最終的なインデックスを構築。

読者向けメモ

  • **メモリ管理**: ginbuild 中は tmpCtx と funcCtx を使い分け、メモリリークを防いでいます。maintenance_work_mem が構築パフォーマンスに直結します。
  • **並列構築の制約**: 並列構築時は allow_sync=true でスキャンするため、TIDの順序が保証されません。そのため、ラップアラウンド検知とマージソートによる再構築が必須となっています。
  • **ポスティングツリーへの昇格**: addItemPointersToLeafTuple は、リーフページに収まらないTIDリストを自動的にポスティングツリーへ変換します。この際、createPostingTree が呼び出されます。
  • **省略箇所**: 本ファイルは並列構築の複雑な同期ロジックを含みますが、Tuplesort 自体の内部実装（tuplesort.c）については詳細を省略しています。
