src/backend/access/gin/ginvalidate.c

概要

GIN（Generalized Inverted Index）インデックスの演算子クラス（opclass）および演算子族（opfamily）の定義が、PostgreSQLのシステムカタログ上で正当であるかを検証するモジュールです。

モジュール内での役割

GINインデックスの作成時や変更時に呼び出され、ユーザーが定義した関数や演算子がGINの期待するシグネチャや制約を満たしているかをチェックします。

責務

  • **検証**: 演算子族に登録されたサポート関数が正しい引数・戻り値の型を持っているかを確認する。
  • **整合性チェック**: 必須のサポート関数（extractValue, extractQuery 等）が定義されているか、また演算子の戦略番号が妥当かを判定する。
  • **依存関係の調整**: CREATE OPERATOR CLASS 等の実行時に、関数や演算子の依存関係（ハード/ソフト）を適切に設定する。

重要なシンボル

  • ginvalidate(Oid opclassoid): インデックス作成時に呼び出されるメインの検証関数。
  • ginadjustmembers(...): 演算子や関数の依存関係をカタログ登録前に調整する関数。
  • GIN_..._PROC: GINがサポートする各サポート関数の識別子（GIN_CONSISTENT_PROC 等）。

追加の索引済みシンボル

  • ginadjustmembers
  • ginvalidate

制御フローとデータフロー

1. ginvalidate が対象の opclassoid を受け取り、システムカタログ（pg_opclass, pg_amproc, pg_amop）から関連情報を取得します。 2. 登録されている全サポート関数をループし、check_amproc_signature を用いてシグネチャを検証します。 3. 登録されている全演算子をループし、戦略番号の範囲と ORDER BY 演算子の禁止事項を確認します。 4. 最後に identify_opfamily_groups でグループ化し、必須関数が欠けていないか最終チェックを行います。 5. 検証失敗時は INFO レベルでエラーを報告し、result = false を返して処理を中断させます。

読者向けメモ

  • GINのサポート関数には「必須」と「オプション」が存在します。ginvalidate 内のループで continue されている関数はオプション扱いであることを示しています。
  • ginadjustmembers は、ALTER OPERATOR FAMILY 等で新しいメンバーを追加する際に、そのメンバーがインデックスの整合性にどの程度重要か（ハード依存かソフト依存か）を決定する重要な役割を担っています。
