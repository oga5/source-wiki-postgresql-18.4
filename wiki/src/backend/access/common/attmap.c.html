src/backend/access/common/attmap.c

概要

attmap.c は、異なる行型（TupleDesc）間で属性番号を変換するためのマッピング構造体 AttrMap を管理するユーティリティモジュールです。主に継承やパーティショニングにおいて、論理的に等価だが物理的な列順序や削除済み列の有無が異なるタプルを変換する際に使用されます。

モジュール内での役割

src/backend/access/common 内において、タプル変換（tupconvert.c）を支える下位層のサポート機能を提供します。タプル変換が必要かどうかの判定や、変換に必要な属性番号の対応表を作成する役割を担います。

責務

  • AttrMap 構造体のメモリ確保および解放。
  • 2つの TupleDesc を比較し、列の位置（Position）または名前（Name）に基づいた属性マッピングの生成。
  • マッピングが不要（変換の必要がない）な場合の最適化判定。

重要なシンボル

  • AttrMap: 属性番号の対応関係を保持する構造体。attnums 配列は出力側の各列に対応する入力側の列番号を格納します。
  • build_attrmap_by_position: 列の物理的な位置に基づいてマッピングを生成します。
  • build_attrmap_by_name: 列名に基づいてマッピングを生成します。
  • build_attrmap_by_name_if_req: マッピング生成後、変換が不要であれば NULL を返すラッパー関数。

追加の索引済みシンボル

  • check_attrmap_match
  • free_attrmap
  • make_attrmap

制御フローとデータフロー

1. **初期化**: make_attrmap で必要なメモリを確保します。 2. **マッピング生成**:

  • build_attrmap_by_position または build_attrmap_by_name を呼び出し、入力と出力の TupleDesc を走査します。
  • 削除済み列（attisdropped）をスキップしつつ、型の一致を確認しながら attrMap->attnums を埋めます。

    • 3. **最適化判定**: check_attrmap_match を呼び出し、マッピングが単なる1対1の対応（変換不要）であるかを確認します。 4. **結果返却**: 変換が不要な場合は free_attrmap でメモリを解放し NULL を返し、必要な場合は生成した AttrMap を返します。

直接依存

  • postgres.h: 基本的な型定義とマクロ。
  • access/attmap.h: AttrMap 構造体および関数プロトタイプ。
  • utils/builtins.h: 型変換やエラー報告のためのユーティリティ。

関連コード

  • src/backend/access/common/tupconvert.c: 本モジュールの主要な利用者。ここで生成された AttrMap を用いて実際のタプル変換が行われます。
  • src/include/access/tupdesc.h: TupleDesc および Form_pg_attribute の定義元。

読者向けメモ

  • 本ファイルは「変換が必要か否か」を判定するロジックを含んでおり、パフォーマンスに直結します。特に check_attrmap_match は、不要なタプルコピーを避けるための重要な最適化ポイントです。
  • attnums 配列のインデックスは出力側の列位置に対応し、値は入力側の列番号（1始まり）です。値が 0 の場合は、その列が削除済みであることを示します。
  • build_attrmap_by_name 内の検索ループは、パーティションテーブルの列順序が一致しているケースを想定して最適化されています。
