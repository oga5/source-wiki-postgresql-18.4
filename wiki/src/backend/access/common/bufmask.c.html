src/backend/access/common/bufmask.c

概要

bufmask.c は、PostgreSQL の WAL（Write-Ahead Logging）の整合性チェックにおいて、ページ比較を容易にするためのユーティリティ関数群を提供します。WAL の生成時と適用（リプレイ）時で値が異なり得るページ内の特定のビットや領域を「マスク（無効化）」することで、誤検知を防ぎます。

モジュール内での役割

src/backend/access/common ディレクトリの一部として、主にストレージ層やアクセスメソッド（AM）が WAL 整合性テストを行う際に利用されます。各リソースマネージャ（rmgr）が個別に実装すべきマスク処理のうち、共通して発生するパターンを抽象化しています。

責務

  • **メタデータの無効化**: LSN やチェックサムなど、WAL 適用時に変化するページヘッダー情報のクリア。
  • **ヒントビットの無視**: WAL を伴わずに更新されるヒントビットやフラグの正規化。
  • **未使用領域のクリア**: ページ内の pd_lower から pd_upper 間のゴミデータの消去。
  • **構造情報の正規化**: インデックスページ等におけるラインポインタフラグやページ内容の強制的なマスク。

重要なシンボル

  • mask_page_lsn_and_checksum: LSN とチェックサムを MASK_MARKER で上書きします。
  • mask_page_hint_bits: pd_prune_xid や PD_PAGE_FULL などのヒント情報をクリアします。
  • mask_unused_space: ページ内の未使用領域をゼロ埋めします。
  • mask_lp_flags: 全ラインポインタのフラグを LP_UNUSED に設定します。
  • mask_page_content: ページヘッダーを除く全領域をマスクし、ページ構造を初期化状態に近づけます。

制御フローとデータフロー

各関数は Page 型のポインタを引数に取り、インプレースでページ内容を書き換えます。 1. **ヘッダー操作**: PageHeader 構造体にキャストし、特定のフィールドを直接 MASK_MARKER で上書きします。 2. **領域操作**: memset を使用して、ページ内の特定のオフセット範囲（pd_lower から pd_upper など）をクリアします。 3. **反復操作**: mask_lp_flags のように、ページ内の全ラインポインタを走査して個別にフラグを更新するパターンも存在します。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • access/bufmask.h: 本ファイルの関数宣言および MASK_MARKER 定義。

関連コード

  • src/include/access/bufmask.h: 本ファイルのヘッダー。
  • 各リソースマネージャ（src/backend/access/nbtree/nbtxlog.c など）: 実際にこれらの関数を呼び出し、WAL 整合性テストを行う側。

読者向けメモ

  • 本ファイルで提供される関数は、あくまで「テスト目的」の正規化ツールです。実稼働時のページ更新処理には使用しないでください。
  • MASK_MARKER は通常 0 です。比較対象のページ間で「値が異なっていても問題ない」箇所を特定し、それらをすべて同じ値（0）に揃えることで、memcmp 等による単純な比較を可能にしています。
  • ページ構造の整合性チェック（pd_lower > pd_upper 等）が含まれているため、デバッグ時のページ破損検知にも役立ちます。
