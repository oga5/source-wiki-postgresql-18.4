src/backend/access/common/tupconvert.c

概要

tupconvert.c は、論理的に等価だが物理的な構造（列の順序や削除済み列の有無）が異なるタプル間での変換をサポートするモジュールです。主に継承関係にあるテーブル間や、スキーマ変更後のデータ移行などで利用されます。

モジュール内での役割

src/backend/access/common ディレクトリにおいて、タプル記述子（TupleDesc）に基づいたデータ変換の抽象化層を提供します。attmap.c が提供する属性マッピング情報を利用し、実際のタプルデータやスロットの変換を実行します。

責務

  • **変換マップの構築**: 入力と出力の TupleDesc を比較し、変換が必要か判断して TupleConversionMap を生成する。
  • **タプル変換の実行**: HeapTuple または TupleTableSlot を対象に、マップに従って属性値を再配置する。
  • **リソース管理**: 変換時に使用するワークスペース（Datum 配列）の確保と、不要になったマップ構造体の解放。

重要なシンボル

  • TupleConversionMap: 変換に必要な属性マッピング情報と、変換作業用のワークスペースを保持する構造体。
  • convert_tuples_by_position / convert_tuples_by_name: 変換マップを生成するファクトリ関数。
  • execute_attr_map_tuple: HeapTuple を新しい構造に変換する。
  • execute_attr_map_slot: TupleTableSlot を対象に、仮想タプルとして変換を実行する。

追加の索引済みシンボル

  • convert_tuples_by_name_attrmap
  • execute_attr_map_cols
  • free_conversion_map

制御フローとデータフロー

1. **初期化**: convert_tuples_by_... 関数が attmap.c を呼び出し、属性の対応関係を検証。物理的に同一であれば NULL を返し、変換不要であることを示す。 2. **準備**: 変換が必要な場合、TupleConversionMap を確保し、Datum 配列等のワークスペースを初期化する。 3. **変換実行**: execute_attr_map_... 関数が、入力側の属性を attrMap に従って出力側のインデックスへコピーする。

  • attrMap の値が 0 の場合、出力側には NULL がセットされる（削除済み列の処理）。

    • 4. **終了**: free_conversion_map により、確保したメモリ領域を解放する。

直接依存

  • access/tupconvert.h: 変換マップ構造体および公開関数の定義。

読者向けメモ

  • **物理的互換性**: 変換関数が NULL を返す場合、変換処理は不要です。呼び出し側でこのチェックを行うことで、不要なメモリ確保を回避できます。
  • **メモリ管理**: TupleConversionMap 内の TupleDesc は参照のみであり、このモジュールでは解放しません。呼び出し側のコンテキストで生存期間を管理してください。
  • **システム列**: execute_attr_map_cols はシステム列（負の属性番号）を考慮したビットマップ変換を行います。通常のタプル変換とは異なるパスを通る点に注意してください。
