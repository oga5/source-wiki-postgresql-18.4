src/backend/access/common/session.c

概要

本ファイルは、PostgreSQLのクライアントセッション全体で共有される状態を管理するためのインフラストラクチャを提供します。主に並列クエリ実行時において、リーダープロセスとワーカープロセス間でセッションスコープのデータを共有することを目的としています。

モジュール内での役割

src/backend/access/common ディレクトリ内で、プロセス間通信（IPC）および共有メモリ管理の抽象化レイヤーとして機能します。他のファイルがタプルやリレーションの操作を扱うのに対し、本ファイルはそれらの操作を支える「セッションの生存期間」に紐づく共有メモリ基盤を提供します。

責務

  • **セッション状態の保持**: CurrentSession グローバル変数を通じて、現在のバックエンドが属するセッション情報を管理する。
  • **共有メモリのライフサイクル管理**: セッションスコープの動的共有メモリ（DSM）セグメントおよび動的共有メモリ領域（DSA）の作成・アタッチ・デタッチを制御する。
  • **共有レジストリの提供**: 並列クエリで必要となる一時的な行型（Record Type）の型修飾子（typmod）レジストリを共有するための基盤を提供する。

重要なシンボル

  • CurrentSession: 現在のバックエンドが使用しているセッション情報を保持するグローバル変数。
  • InitializeSession(): セッションオブジェクトを初期化する。
  • GetSessionDsmHandle(): セッション用DSMセグメントを生成（または取得）し、そのハンドルを返す。
  • AttachSession(): リーダープロセスから提供されたDSMハンドルを用いて、ワーカープロセスがセッション共有メモリに接続する。
  • DetachSession(): セッション共有メモリから明示的にデタッチする。

制御フローとデータフロー

1. **初期化**: InitializeSession により CurrentSession がメモリ上に確保される。 2. **DSM構築**: GetSessionDsmHandle が呼ばれると、shm_toc（テーブル・オブ・コンテンツ）を使用してDSMセグメントを初期化する。この際、DSA領域と型修飾子レジストリ用の領域が確保される。 3. **共有**: リーダープロセスが生成したDSMハンドルをワーカープロセスに伝達し、ワーカーが AttachSession を実行することで、同一のDSA領域およびレジストリにアクセス可能となる。 4. **終了**: バックエンド終了時、または DetachSession 呼び出し時に、登録されたデタッチフックが実行され、リソースが解放される。

直接依存

  • access/session.h: セッション構造体の定義。
  • storage/shm_toc.h: 共有メモリ内のデータ配置管理。
  • storage/lwlock.h: 共有メモリ操作時のロック管理。
  • utils/typcache.h: 共有される型修飾子レジストリの操作。
  • utils/memutils.h: メモリコンテキスト管理。
  • postgres.h

読者向けメモ

  • 本モジュールは現在、主に並列クエリのインフラとして利用されていますが、将来的にコネクションプーリング等の目的でバックエンドとクライアント接続を分離する際の基盤となるよう設計されています。
  • SESSION_MAGIC や SESSION_KEY_* は、共有メモリ内のデータ構造を識別するための定数です。これらを変更・追加する際は、リーダーとワーカー間での整合性に注意してください。
  • dsm_pin_mapping を使用しているため、一度アタッチしたセッションメモリはバックエンドの生存期間中、明示的なデタッチまで保持されます。
