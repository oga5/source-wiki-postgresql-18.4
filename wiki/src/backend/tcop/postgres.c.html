src/backend/tcop/postgres.c

概要

postgres.c は PostgreSQL バックエンドプロセスのメインループ（Traffic Cop）を実装する中心的なファイルです。クライアントからの通信プロトコル（Simple/Extended Query）を解釈し、パーサ、プランナ、エグゼキュータを呼び出してクエリのライフサイクルを管理します。

モジュール内での役割

src/backend/tcop サブシステムにおける最上位の制御層です。PostgresMain がプロセスのメインエントリポイントとなり、クライアントとの通信、トランザクション境界の制御、およびエラー発生時のリカバリ処理を統括します。

責務

  • **メインループ管理**: クライアントからのメッセージ受信とディスパッチ。
  • **プロトコル処理**: Simple Query および Extended Query（Parse/Bind/Execute）プロトコルの実装。
  • **トランザクション制御**: start_xact_command / finish_xact_command によるトランザクション境界の管理。
  • **エラーリカバリ**: sigsetjmp を用いたトップレベルの例外ハンドリングとプロトコル同期の維持。
  • **シグナルハンドリング**: die（終了）、StatementCancelHandler（クエリキャンセル）、quickdie（緊急停止）等の実装。

重要なシンボル

メインエントリとループ

  • PostgresMain: バックエンドプロセスのメインループ。
  • ReadCommand: クライアントまたは標準入力からのメッセージ読み込み。
  • exec_simple_query: Simple Query プロトコルの実行。
  • exec_parse_message / exec_bind_message / exec_execute_message: Extended Query プロトコルの各フェーズ実行。

状態管理変数

  • xact_started: 現在のトランザクション状態を追跡。
  • DoingCommandRead: クライアントからの読み込み中であることを示すフラグ。
  • ignore_till_sync: エラー発生時に次の Sync メッセージまで入力をスキップするためのフラグ。
  • unnamed_stmt_psrc: 未命名のプリペアドステートメントを保持。

ユーティリティ・ヘルパー

  • pg_parse_query, pg_analyze_and_rewrite_*, pg_plan_query: SQL 処理パイプラインの各ステップを実行するラッパー。
  • start_xact_command, finish_xact_command: トランザクションの開始・終了を管理する便利関数。

制御フローとデータフロー

1. **初期化**: PostgresMain がシグナルハンドラを設定し、InitPostgres で接続を確立。 2. **ループ**: for (;;) ループ内で ReadCommand を呼び出し、メッセージタイプを判定。 3. **ディスパッチ**: switch (firstchar) でメッセージタイプに応じた exec_* 関数を呼び出す。 4. **実行**: Portal を作成し、PortalRun を通じて実行。結果は DestReceiver を介してクライアントへ送信。 5. **リカバリ**: sigsetjmp で設定されたスタックにより、ereport(ERROR) 発生時に AbortCurrentTransaction を呼び出し、プロトコル同期を維持しつつループの先頭へ戻る。

読者向けメモ

  • **メモリ管理**: MessageContext は各コマンドメッセージの処理後にリセットされます。永続的なデータは TopMemoryContext 等に配置する必要があります。
  • **エラーハンドリング**: PG_TRY ではなく sigsetjmp を使用しています。これはスタックの最下層でエラーリカバリを行うためです。
  • **プロトコル同期**: ignore_till_sync フラグは、Extended Query プロトコルにおいてエラー発生後にクライアントとバックエンドの同期を保つために重要です。
  • **省略された詳細**: valgrind 関連のデバッグコードや、統計情報（ShowUsage）の詳細は、メインの制御フロー理解には必須ではないため、適宜読み飛ばして構いません。
