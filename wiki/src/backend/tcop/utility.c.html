src/backend/tcop/utility.c

概要

utility.c は、PostgreSQL における「ユーティリティコマンド」（DDL、トランザクション制御、セッション管理など、通常の DML 以外の SQL）の実行を統括するモジュールです。ProcessUtility を中心として、コマンドの分類、実行前後の制約チェック、イベントトリガーのフック、および実行ディスパッチを行います。

モジュール内での役割

src/backend/tcop サブシステムの一部として、postgres.c（メインループ）から呼び出される実行の入り口です。クエリ実行エンジン（Executor）が扱う PlannedStmt のうち、CMD_UTILITY 型のコマンドを専門に処理します。

責務

  • **実行ディスパッチ**: コマンドの種類（NodeTag）に基づき、適切なコマンド実行関数（DefineIndex, AlterTable 等）へ処理を振り分ける。
  • **制約とセキュリティ**: 読み取り専用トランザクション、パラレルモード、リカバリ中、セキュリティ制限下での実行可否を検証する。
  • **イベントトリガー管理**: DDL コマンドの開始・終了をイベントトリガーシステムに通知し、コマンドの収集を行う。
  • **メタデータ生成**: コマンド完了タグ（CommandTag）の生成や、ログレベルの判定を行う。

重要なシンボル

実行ディスパッチャ

  • ProcessUtility: 外部からのメインエントリーポイント。フック（ProcessUtility_hook）の確認と standard_ProcessUtility の呼び出しを行う。
  • standard_ProcessUtility: ユーティリティコマンドの実行フローを制御する中心関数。
  • ProcessUtilitySlow: イベントトリガー対応が必要なコマンドを処理するサブセット。

制約検証

  • CommandIsReadOnly: PlannedStmt が真に読み取り専用か判定する。
  • ClassifyUtilityCommandAsReadOnly: 各コマンドが読み取り専用トランザクションやリカバリ中で実行可能かを分類する。
  • PreventCommandIfReadOnly / PreventCommandIfParallelMode / PreventCommandDuringRecovery: 各種状態に応じた実行拒否エラーを投げる。

メタデータ・ユーティリティ

  • CreateCommandTag: Node からコマンド完了タグを生成する。
  • UtilityReturnsTuples: 結果セットを返すコマンド（FETCH, EXPLAIN 等）を判定する。
  • GetCommandLogLevel: コマンドのログレベルを決定する。

制御フローとデータフロー

1. **初期化**: ProcessUtility が呼び出され、必要に応じて copyObject でツリーを保護する。 2. **検証**: ClassifyUtilityCommandAsReadOnly を呼び出し、現在のトランザクション状態（ReadOnly, Parallel, Recovery）と照合して実行可否を決定する。 3. **ディスパッチ**: standard_ProcessUtility 内の巨大な switch 文でコマンドを分類。

  • イベントトリガー非対応コマンドは直接実行。
  • 対応コマンドは ProcessUtilitySlow へ委譲。

    • 4. **イベントトリガー**: ProcessUtilitySlow 内で EventTriggerDDLCommandStart / End を呼び出し、DDL のライフサイクルを管理する。 5. **完了**: CommandCounterIncrement() を呼び出し、コマンドの副作用を可視化して終了する。

読者向けメモ

  • **メモリ管理**: readOnlyTree が真の場合、copyObject でツリーを複製します。これはパーサの木を破壊しないための安全策です。
  • **再帰呼び出し**: ProcessUtility は再帰的に呼び出される可能性があるため、check_stack_depth() によるスタックオーバーフロー保護が行われています。
  • **イベントトリガーの分離**: standard_ProcessUtility と ProcessUtilitySlow の分離は、トランザクション開始時など、イベントトリガーキャッシュを更新できない状況でトリガーを起動しないための重要な設計です。
  • **網羅性**: ClassifyUtilityCommandAsReadOnly や CreateCommandTag は、新しいコマンドを追加するたびに更新が必要な巨大な switch 文です。追加時は漏れがないよう注意してください。
