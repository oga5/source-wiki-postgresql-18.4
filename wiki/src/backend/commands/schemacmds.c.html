src/backend/commands/schemacmds.c

概要

PostgreSQLにおけるスキーマ（名前空間）の作成、名前変更、および所有者変更を行うコマンドの実装を提供します。SQLの CREATE SCHEMA、ALTER SCHEMA 文のバックエンド処理を担います。

モジュール内での役割

src/backend/commands サブシステムの一部として、データベース内の論理的な名前空間を管理します。他のコマンド群（tablecmds.c など）がスキーマ内にオブジェクトを作成する際、その基盤となる名前空間の整合性を保証します。

責務

  • **スキーマ作成**: CREATE SCHEMA 文の解析結果に基づき、名前空間の作成、所有権の設定、およびスキーマ内でのサブコマンド（テーブル作成など）の逐次実行。
  • **スキーマ操作**: スキーマ名の変更および所有者の変更。
  • **セキュリティ管理**: スキーマ作成・変更時の権限チェック（ACL_CREATE）、所有者変更時のロール切り替え権限の検証。
  • **システム保護**: pg_ で始まる予約済みスキーマ名の保護。

重要なシンボル

  • CreateSchemaCommand: CREATE SCHEMA 文のメインエントリポイント。サブコマンドの実行と検索パスの一時的な変更を管理します。
  • RenameSchema: スキーマ名を変更し、システムカタログを更新します。
  • AlterSchemaOwner: スキーマの所有者を変更し、依存関係を更新します。
  • AlterSchemaOwner_internal: 所有者変更の共通ロジック（ACLの再計算や依存関係の更新）をカプセル化した内部関数。

制御フローとデータフロー

1. **CREATE SCHEMA**:

  • 権限チェック後、NamespaceCreate でカタログにエントリを追加。
  • search_path を一時的に変更し、作成中のスキーマを優先させる。
  • transformCreateSchemaStmtElements でサブコマンドを順序立て、ProcessUtility をループで呼び出して各コマンドを実行。
  • 最後に search_path を復元。

    • 2. **ALTER SCHEMA**:

  • pg_namespace システムカタログを RowExclusiveLock でロック。
  • 権限確認後、CatalogTupleUpdate を使用してカタログタプルを更新。
  • InvokeObjectPostAlterHook を呼び出し、イベントトリガーやキャッシュ無効化を通知。

読者向けメモ

  • CreateSchemaCommand 内のサブコマンド実行は、ProcessUtility を再帰的に呼び出すことで実現されています。これにより、スキーマ作成文の中でテーブル作成などの他のDDLを透過的に扱えます。
  • 所有者変更時、ACL（アクセス制御リスト）の再計算が必要になるため、aclnewowner を介して所有者変更に伴う権限の移行を行っています。
  • if_not_exists の処理は、権限チェックの直後に行われるため、存在しない場合のセキュリティリスク（権限のないユーザーが他人のスキーマ作成を試みる等）を考慮した設計になっています。
