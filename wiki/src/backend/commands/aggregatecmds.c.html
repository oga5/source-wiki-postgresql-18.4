src/backend/commands/aggregatecmds.c

概要

PostgreSQL の CREATE AGGREGATE コマンドを処理するためのバックエンド実装です。SQL パーサーから渡された構文木を解析し、集約関数の定義に必要な属性（遷移関数、最終関数、データ型など）を抽出・検証した上で、カタログ操作を行う関数へ引き渡します。

モジュール内での役割

src/backend/commands モジュールの一部として、集約関数（Aggregate）のメタデータ管理を担当します。SQL レベルの定義をシステムカタログ（pg_aggregate 等）に永続化するためのゲートウェイとして機能します。

責務

  • **構文解析の解釈**: CREATE AGGREGATE 文のパラメータリストを走査し、各属性を適切な内部表現に変換する。
  • **権限と妥当性の検証**: 対象スキーマへの作成権限チェック、および指定された型や関数が定義可能かどうかの検証。
  • **カタログ操作の準備**: AggregateCreate（src/backend/catalog/pg_aggregate.c 等で定義）を呼び出すための引数群を構築する。

重要なシンボル

  • DefineAggregate: 本ファイルのメインエントリポイント。SQL コマンドの実行時に呼び出されます。
  • extractModify: finalfunc_modify 属性の文字列値を内部定数（AGGMODIFY_*）に変換するヘルパー関数。

制御フローとデータフロー

1. **名前解決**: QualifiedNameGetCreationNamespace を使用して、対象の集約関数が作成されるスキーマを特定し、権限を確認します。 2. **パラメータ解析**: foreach ループで parameters リストを走査し、sfunc, stype, finalfunc などの定義を抽出します。 3. **型チェック**: typenameTypeId を用いて、遷移データ型（stype）や入力型が有効か確認します。特に INTERNAL 型の使用にはスーパーユーザー権限を要求する等の制約を適用します。 4. **カタログ登録**: すべての検証が完了した後、収集した情報を AggregateCreate 関数に渡し、システムカタログへの書き込みを委譲します。

読者向けメモ

  • **古い構文への対応**: oldstyle フラグにより、PostgreSQL 8.2 以前の古い定義形式（BASETYPE を使用する形式）との互換性が維持されています。
  • **初期値の扱い**: initcond（初期値）は、実行時の型入力関数で検証されますが、保存は text 型として行われます。これは、now() のような非イミュータブルな関数を初期値に含める可能性があるためです。
  • **エラーハンドリング**: 多くの検証ロジックが DefineAggregate 内に記述されています。新しい集約属性を追加する際は、この関数内のループと検証ロジックを更新する必要があります。
