src/backend/commands/indexcmds.c

概要

インデックスの作成（CREATE INDEX）および再構築（REINDEX）を統括するコマンド実行エンジンです。システムカタログの更新、物理的なインデックス構築、および並行処理（CONCURRENTLY）における複雑な状態遷移と同期制御を担います。

モジュール内での役割

src/backend/commands モジュールの一部として、インデックスという特定のデータベースオブジェクトのライフサイクルを管理します。tablecmds.c がテーブル全体の構造変更を扱うのに対し、本ファイルはインデックス固有の属性（演算子クラス、照合順序、述語）の検証と構築に特化しています。

責務

  • **インデックス定義の検証**: 属性、演算子クラス、述語（WHERE 句）、およびパーティション制約の整合性チェック。
  • **インデックス構築のディスパッチ**: 通常の構築と、CONCURRENTLY モードによる段階的な構築（カタログ更新、スナップショット待機、検証）の制御。
  • **再構築（REINDEX）の管理**: 単一インデックス、テーブル全体、スキーマ、データベース単位の再構築処理。
  • **パーティション対応**: パーティションテーブルに対するインデックスの再帰的作成およびアタッチ処理。

重要なシンボル

インデックス作成・構築

  • DefineIndex(): CREATE INDEX コマンドのメインエントリ。カタログ登録から物理構築までを制御。
  • index_concurrently_build(): 並行構築の物理スキャンと挿入を実行。
  • validate_index(): 並行構築の最終フェーズで、インデックスの整合性を検証。

再構築（REINDEX）

  • ExecReindex(): REINDEX コマンドのディスパッチャ。
  • ReindexIndex() / ReindexTable(): 対象に応じた再構築の実行。
  • ReindexRelationConcurrently(): REINDEX CONCURRENTLY の複雑な状態遷移を管理。

ユーティリティ・検証

  • CheckIndexCompatible(): インデックスの再構築を回避できるか（互換性）を判定。
  • ComputeIndexAttrs(): インデックス属性、演算子クラス、オプションを計算・検証。
  • WaitForOlderSnapshots(): 並行構築時に、古いスナップショットを持つトランザクションの終了を待機。
  • makeObjectName() / ChooseRelationName(): インデックス名の自動生成と衝突回避。

制御フローとデータフロー

1. **初期化**: DefineIndex がパーサーからの IndexStmt を受け取り、テーブルのロックと所有者権限の切り替えを行う。 2. **属性計算**: ComputeIndexAttrs が演算子クラスや照合順序を解決し、IndexInfo 構造体を構築する。 3. **構築フェーズ**:

  • 通常: index_create を呼び出し、カタログ更新と物理構築を同期的に行う。
  • 並行: 複数トランザクションに分割。
  • フェーズ1: カタログにインデックスを登録（indisready=false）。
  • フェーズ2: WaitForLockers で既存の書き込みトランザクションを待機し、物理構築を実行。
  • フェーズ3: validate_index で不足エントリを補完。
  • フェーズ4-6: インデックスの切り替え、古いインデックスの無効化と削除。

読者向けメモ

  • **メモリ管理**: ReindexMultipleTables 等では、トランザクションを跨いで生存させる必要があるデータを PortalContext などの長寿命コンテキストに割り当てています。
  • **並行構築の複雑性**: REINDEX CONCURRENTLY は、デッドロックを避けるためにセッションレベルのロック（LockRelationIdForSession）を使用し、複数のトランザクションに処理を分割しています。このフローを理解するには、WaitForLockers と CommitTransactionCommand の呼び出し順序が重要です。
  • **省略された詳細**: インデックスの物理的なスキャンや挿入の詳細は access/ 以下の各アクセスメソッド（B-tree, GiST 等）に委譲されており、本ファイルはあくまで「コマンドの実行制御」に焦点を当てています。
