src/backend/commands/explain_format.c

概要

explain_format.c は、PostgreSQL の EXPLAIN コマンドの出力形式（TEXT, XML, JSON, YAML）を抽象化し、生成するためのフォーマットルーチンを提供します。

モジュール内での役割

src/backend/commands モジュールの一部として、explain.c が実行計画を走査する際に呼び出される「出力生成層」を担います。ExplainState 構造体を通じて出力バッファと現在のフォーマット状態を管理し、各フォーマット固有の構文（インデント、タグ、引用符、区切り文字）を隠蔽します。

責務

  • **プロパティ出力**: 文字列、数値、真偽値などのプロパティを、指定されたフォーマットに従って出力バッファへ書き込む。
  • **グループ管理**: 階層構造（JSONのオブジェクト/配列、XMLのタグなど）の開始と終了を制御し、ネストレベルを管理する。
  • **フォーマット固有の変換**: 各形式の仕様（JSONのカンマ管理、YAMLの引用符、XMLのタグ名サニタイズなど）を適用する。

重要なシンボル

  • ExplainState: 出力先バッファ、現在のフォーマット、インデントレベル、グループ状態を保持する中心的な構造体。
  • ExplainProperty[Type]: 各データ型（Text, Integer, Float, Bool）に対応する出力関数。
  • ExplainOpenGroup / ExplainCloseGroup: 階層構造の開始と終了を定義する。
  • ExplainPropertyList: リスト形式のプロパティを一括出力する。

制御フローとデータフロー

1. **初期化**: ExplainBeginOutput が呼び出され、フォーマットに応じたルート要素（JSONの [ など）が生成される。 2. **プロパティ出力**: ExplainProperty 経由で値が渡される。内部で es->format を判定し、TEXT ならそのまま、JSON/YAML ならエスケープ処理を経てバッファへ追加される。 3. **グループ化**: ExplainOpenGroup が呼ばれると、es->grouping_stack が更新され、ネストレベルがインクリメントされる。 4. **終了**: ExplainEndOutput でルート要素を閉じ、最終的な文字列が完成する。

読者向けメモ

  • **JSON/YAML の状態管理**: grouping_stack は、現在の階層で「既に要素が出力されたか（カンマが必要か）」を追跡するために使用されます。
  • **XML の制約**: XML タグ名は空白やスラッシュを許容しないため、ExplainXMLTag 内で不正な文字をハイフンに置換する処理が入っています。
  • **拡張性**: 新しいフォーマットを追加する場合、各関数内の switch (es->format) 文にケースを追加する必要があります。
