src/backend/commands/statscmds.c

概要

PostgreSQLの拡張統計情報（Extended Statistics）オブジェクトの作成、変更、および削除を管理するコマンド実装ファイルです。CREATE STATISTICS、ALTER STATISTICS、DROP STATISTICS などのDDL操作のバックエンド処理を担います。

モジュール内での役割

src/backend/commands ディレクトリの一部として、統計情報サブシステムとカタログ（pg_statistic_ext）の橋渡しを行います。オプティマイザが利用する多変量統計情報のライフサイクルを制御します。

責務

  • **統計オブジェクトの作成**: CREATE STATISTICS 文の解析、列や式の妥当性検証、カタログへの登録、および依存関係の構築。
  • **統計オブジェクトの変更**: ALTER STATISTICS による統計ターゲット（stxstattarget）の更新。
  • **統計オブジェクトの削除**: 関連するカタログエントリ（pg_statistic_ext および pg_statistic_ext_data）のクリーンアップとリレーションキャッシュの無効化。
  • **命名規則の管理**: 統計オブジェクト名が重複しないよう、自動生成時の名前解決を行う。

重要なシンボル

  • CreateStatistics: CREATE STATISTICS 文を実行し、カタログへの挿入と依存関係の記録を行うメイン関数。
  • AlterStatistics: 既存の統計オブジェクトの属性（主に統計ターゲット）を更新する。
  • RemoveStatisticsById: 統計オブジェクトを削除し、関連するデータとキャッシュをクリーンアップする。
  • StatisticsGetRelation: 統計オブジェクトのOIDから、それが定義されているテーブルのOIDを取得するユーティリティ。

制御フローとデータフロー

1. **検証**: CreateStatistics は対象リレーションのロックを取得し、所有権やシステムカタログへの操作制限を確認します。 2. **解析**: stmt->exprs を走査し、列参照や式を抽出。型キャッシュを用いてB-tree演算子クラスの存在を確認します。 3. **カタログ操作**: pg_statistic_ext に新しいタプルを挿入し、InvokeObjectPostCreateHook を呼び出します。 4. **依存関係**: recordDependencyOn を使用し、対象テーブルの列や式に対する依存関係を記録します。これにより、テーブルや列の削除時に統計情報も自動的に削除されます。 5. **キャッシュ無効化**: CacheInvalidateRelcache を呼び出し、新しい統計情報がプランナに即座に反映されるようにします。

読者向けメモ

  • 本ファイルはDDLの「定義」を扱うものであり、実際の統計情報の「計算」は ANALYZE 実行時に別のモジュールで行われます。
  • 統計オブジェクトは、テーブルの列や式に対して DEPENDENCY_AUTO で紐付けられています。そのため、テーブルや列を削除すると統計情報も自動的に削除される設計になっています。
  • ChooseExtendedStatisticName は、名前が指定されていない場合に自動生成するロジックですが、同時実行環境での衝突を避けるため、必要に応じて CommandCounterIncrement を挟む設計が推奨されます。
