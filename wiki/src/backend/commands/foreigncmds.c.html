src/backend/commands/foreigncmds.c

概要

PostgreSQL の外部データラッパー（FDW）、外部サーバー、ユーザーマッピング、および外部テーブルの定義・変更・削除を行う DDL コマンドの実行エンジンです。SQL 構文解析後の処理を担い、システムカタログ（pg_foreign_data_wrapper 等）の更新と依存関係の管理を行います。

モジュール内での役割

src/backend/commands サブシステムの一部として、外部データ連携機能のメタデータ管理を担当します。foreign/ ディレクトリ配下の API と連携し、FDW のハンドラやバリデータ関数を呼び出すことで、外部データソースとの接続設定を検証・永続化します。

責務

  • **DDL 実行**: CREATE/ALTER/DROP コマンドの処理およびシステムカタログの更新。
  • **オプション管理**: DefElem リストのテキスト配列形式への変換とバリデーション。
  • **権限管理**: FDW/サーバーに対する USAGE 権限や所有者変更時の特権チェック。
  • **依存関係管理**: pg_depend を介した、関数（ハンドラ/バリデータ）や拡張機能との依存関係の記録。
  • **スキーマインポート**: FDW の ImportForeignSchema API を呼び出し、生成された SQL を実行して外部テーブルを一括定義。

重要なシンボル

オプション処理

  • transformGenericOptions: DefElem リストを text[] に変換し、バリデータ関数を呼び出して検証する中心的な関数。
  • optionListToArray: オプションリストを key=value 形式のテキスト配列に変換する低レベル関数。

FDW/サーバー管理

  • CreateForeignDataWrapper / AlterForeignDataWrapper: FDW の作成・変更。ハンドラとバリデータの OID を管理。
  • CreateForeignServer / AlterForeignServer: 外部サーバーの作成・変更。FDW との紐付けと権限チェックを行う。

ユーザーマッピング管理

  • CreateUserMapping / AlterUserMapping / RemoveUserMapping: ユーザーと外部サーバー間のマッピング管理。user_mapping_ddl_aclcheck で所有者権限を検証。

外部テーブル管理

  • CreateForeignTable: DefineRelation の後に呼び出され、pg_foreign_table カタログにエントリを追加。
  • ImportForeignSchema: FDW の API を利用して外部スキーマをローカルにインポートする。

制御フローとデータフロー

1. **コマンド実行**: ProcessUtility から各 Create/Alter 関数が呼び出される。 2. **検証**: SearchSysCache を用いて既存オブジェクトを検索し、superuser() や object_ownercheck で権限を検証。 3. **オプション変換**: transformGenericOptions が呼び出され、バリデータ関数（FDW 固有）による検証を経て text[] が生成される。 4. **カタログ更新**: heap_form_tuple と CatalogTupleUpdate を使用してシステムカタログを更新。 5. **依存関係記録**: recordDependencyOn を使用して、作成したオブジェクトと関数や拡張機能とのリンクを pg_depend に書き込む。 6. **フック**: InvokeObjectPostCreateHook 等を呼び出し、イベントトリガーや他のサブシステムに通知。

読者向けメモ

  • **メモリ管理**: transformGenericOptions 等で生成されるオプション配列は、現在のメモリコンテキストで割り当てられます。
  • **エラーハンドリング**: ImportForeignSchema 内では、生成された SQL のエラーを適切に報告するために ErrorContextCallback を使用しています。
  • **所有権**: FDW の作成はスーパーユーザーに限定されていますが、サーバーやユーザーマッピングは所有者権限で操作可能です。
  • **省略箇所**: 本ファイルはメタデータ管理に特化しており、実際の外部データ取得（スキャンや結合）のロジックは含まれていません。これらは src/backend/executor/ や各 FDW 実装側に存在します。
