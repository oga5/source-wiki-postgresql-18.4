src/backend/commands/proclang.c

概要

PostgreSQLにおける手続き型言語（Procedural Language: PL）の定義、更新、および検索を管理するコマンド実装ファイルです。CREATE LANGUAGE 文のバックエンド処理を担います。

モジュール内での役割

src/backend/commands モジュールの一部として、システムカタログ pg_language に対する操作を提供します。PLの登録時に必要なハンドラ関数やバリデータ関数の検証、およびそれらに対する依存関係の構築を行います。

責務

  • **PLの作成と更新**: CREATE [OR REPLACE] LANGUAGE 文の実行。
  • **関数の検証**: 指定されたハンドラ、インラインハンドラ、バリデータ関数が正しいシグネチャを持つか確認。
  • **依存関係の管理**: 作成された言語と、関連する関数や拡張機能との間の依存関係（pg_depend）を記録。
  • **権限チェック**: 言語作成に必要なスーパーユーザー権限の強制。

重要なシンボル

  • CreateProceduralLanguage: CREATE LANGUAGE 文のメインエントリポイント。カタログ更新と依存関係構築を統括します。
  • get_language_oid: 言語名から対応する OID を取得するユーティリティ関数。

制御フローとデータフロー

1. **検証**: LookupFuncName を使用して、指定された関数がシステム上に存在し、かつ期待される戻り値や引数型を持つかを確認します。 2. **カタログ操作**: pg_language を RowExclusiveLock で開き、既存の言語であれば heap_modify_tuple で更新、新規であれば heap_form_tuple で挿入します。 3. **依存関係構築**: recordDependencyOn... 系関数を呼び出し、言語とハンドラ関数、拡張機能とのリンクを pg_depend に書き込みます。 4. **フック**: InvokeObjectPostCreateHook を呼び出し、外部モジュールやイベントトリガーに作成を通知します。

読者向けメモ

  • **スーパーユーザー限定**: PLの作成はセキュリティ上の理由からスーパーユーザーに制限されています。
  • **依存関係の重要性**: 言語を削除する際、関連する関数がまだ存在しているとエラーになるのは、ここで構築される DEPENDENCY_NORMAL によるものです。
  • **REPLACEの挙動**: OR REPLACE を指定した場合、OIDや所有者、権限は維持されますが、ハンドラ関数などの定義は更新されます。
