src/backend/commands/copyfromparse.c

概要

copyfromparse.c は、PostgreSQL の COPY FROM コマンドにおける入力データの解析（パース）を担うモジュールです。テキスト、CSV、およびバイナリ形式の入力ストリームを読み込み、行単位で分解し、最終的にデータベースのデータ型（Datum）へ変換するまでの一連のパイプラインを実装しています。

モジュール内での役割

src/backend/commands サブシステムの一部として、copyfrom.c（コマンドの開始・終了管理）と密接に連携します。本ファイルは、低レベルな I/O バッファ管理から、文字エンコーディング変換、エスケープ処理、そして型固有の受信関数呼び出しまでをカプセル化する「データ解析エンジン」として機能します。

責務

  • **ストリーム管理**: ファイル、フロントエンド（クライアント）、またはコールバック関数からのデータ読み込み。
  • **バッファパイプライン**: raw_buf → input_buf → line_buf → attribute_buf という多段階のバッファリングによる効率的なデータ処理。
  • **フォーマット解析**:
  • **Text/CSV**: 行区切り、エスケープ、クォート、区切り文字の処理。
  • **Binary**: ヘッダー検証、フィールド長指定によるバイナリデータの読み込み。
  • **型変換**: 入力文字列またはバイナリデータを、各カラムの型に対応する ReceiveFunctionCall を用いて Datum に変換。
  • **エラーハンドリング**: エンコーディングエラーの検出、不正なフォーマットの報告、および ON_ERROR オプションに基づくスキップ処理。

重要なシンボル

バッファ制御・通信

  • CopyGetData: ソース（ファイル/FE）から raw データを読み込む低レベル関数。
  • CopyLoadRawBuf / CopyLoadInputBuf: バッファの充填と、必要に応じたエンコーディング変換のトリガー。
  • CopyReadBinaryData: バイナリモードで指定バイト数を確実に読み込むためのヘルパー。

解析・変換ロジック

  • NextCopyFrom: 行単位でデータを取得し、デフォルト値の計算を行うメインエントリポイント。
  • CopyReadLine: 入力ストリームから 1 行分を line_buf に読み込む。
  • CopyReadAttributesText / CopyReadAttributesCSV: line_buf をフィールド単位に分割し、エスケープを解除する。
  • CopyReadBinaryAttribute: バイナリ形式のフィールドを読み込み、型変換関数を呼び出す。

内部状態管理

  • CopyConvertBuf: クライアントエンコーディングからサーバーエンコーディングへの変換および検証。
  • CopyFromTextLikeOneRow: テキスト/CSV 形式における行解析の共通ロジック。

制御フローとデータフロー

1. **データロード**: CopyLoadInputBuf が CopyGetData を呼び出し、raw_buf にデータを蓄積。 2. **変換**: CopyConvertBuf がエンコーディング変換を行い、input_buf を準備。 3. **行抽出**: CopyReadLine が input_buf をスキャンし、改行コードやエスケープを考慮して 1 行を line_buf にコピー。 4. **フィールド分割**: CopyReadAttributes... が line_buf を解析し、raw_fields 配列に各フィールドのポインタを格納。 5. **型変換**: NextCopyFrom が CopyFrom...OneRow を呼び出し、raw_fields の文字列を InputFunctionCallSafe 等で Datum に変換。

読者向けメモ

  • **メモリ管理**: line_buf や attribute_buf は StringInfo を使用しており、必要に応じて自動拡張されます。raw_fields 配列は CopyFromState 内で管理され、行ごとに再利用されます。
  • **パフォーマンス**: IF_NEED_REFILL_... マクロは、 tight loop 内での関数呼び出しオーバーヘッドを避けるために使用されています。
  • **制約**: バイナリモードでは line_buf は使用されず、直接 attribute_buf に読み込まれます。
  • **未実装/浅い部分**: 本ファイルは解析に特化しており、テーブルへの実際の挿入処理（CopyFromInsert）は copyfrom.c 側の責務です。また、COPY のプロトコル詳細（特に v2 以前）は省略されています。
