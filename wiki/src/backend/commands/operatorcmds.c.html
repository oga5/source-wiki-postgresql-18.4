src/backend/commands/operatorcmds.c

概要

PostgreSQLにおける演算子（Operator）の定義、変更、および削除を管理するコマンド処理ルーチンを提供します。SQLの CREATE OPERATOR、ALTER OPERATOR、および DROP 関連のバックエンド処理を担います。

モジュール内での役割

src/backend/commands モジュールの一部として、システムカタログ pg_operator に対する操作のフロントエンドとして機能します。パーサーから渡された構文木を解析し、カタログ操作を行う低レベル関数（OperatorCreate 等）へ橋渡しを行います。

責務

  • **構文解析と検証**: CREATE OPERATOR のパラメータ（関数、型、推定関数など）を解析し、整合性を検証します。
  • **権限チェック**: 演算子作成・変更に必要なスキーマ権限や関数実行権限を検証します。
  • **カタログ更新**: pg_operator テーブルの更新を制御し、依存関係の構築を管理します。
  • **推定関数の検証**: 制限（Restriction）および結合（Join）推定関数のシグネチャと戻り値（float8）を厳格にチェックします。

重要なシンボル

  • DefineOperator: CREATE OPERATOR 文のメインエントリポイント。
  • AlterOperator: ALTER OPERATOR 文のメインエントリポイント。
  • RemoveOperatorById: 演算子削除の内部処理。
  • ValidateRestrictionEstimator / ValidateJoinEstimator: 推定関数のシグネチャと権限を検証するヘルパー。
  • ValidateOperatorReference: 既存の演算子参照（commutator/negator）の妥当性を検証。

制御フローとデータフロー

1. **解析**: DefineOperator が List *parameters を走査し、DefElem から演算子の属性（関数名、型、commutator等）を抽出します。 2. **検証**: typenameTypeId や LookupFuncName を使用して、名前をシステムカタログ上の OID に変換し、型や関数の存在と権限を確認します。 3. **実行**: 最終的に OperatorCreate（src/backend/catalog/pg_operator.c 内）を呼び出し、カタログへの挿入を行います。 4. **更新**: AlterOperator は SearchSysCacheCopy1 で既存のタプルを取得し、heap_modify_tuple を経て CatalogTupleUpdate でカタログを更新します。

読者向けメモ

  • **順序の制約**: 演算子を作成する前に、関連する関数や型が定義されている必要があります。
  • **推定関数の制約**: 制限・結合推定関数は float8 を返す必要があり、非組み込み関数の場合はスーパーユーザー権限が必要です。
  • **ALTERの制限**: ALTER OPERATOR では、既存のプランに影響を与える可能性がある属性（関数本体や引数型など）の変更は禁止されています。
  • **シェル演算子**: 演算子作成時に ValidateOperatorReference がシェル（未定義の演算子）を拒否するのは、誤設定を早期に検知するためのポリシーです。
