src/backend/commands/comment.c

概要

PostgreSQLのデータベースオブジェクトに対する「コメント（COMMENT ON ...）」の管理を行うユーティリティモジュールです。システムカタログ pg_description および pg_shdescription への読み書きを抽象化し、SQLコマンドの実行や内部的なコメント操作をサポートします。

モジュール内での役割

src/backend/commands ディレクトリの一部として、DDLコマンドの実行ロジックを提供します。他のコマンドファイルがオブジェクトの作成や削除を行う際、付随するコメントの整合性を維持するために本モジュールの関数が呼び出されます。

責務

  • SQL COMMENT ON 文の解析結果に基づくカタログ更新。
  • オブジェクト所有権の確認および対象オブジェクトのロック管理。
  • pg_description（通常オブジェクト用）と pg_shdescription（共有オブジェクト用）のCRUD操作。
  • 空文字列またはNULLが指定された場合のコメント削除処理の自動化。

重要なシンボル

  • CommentObject: COMMENT ON 文を処理するメインエントリーポイント。
  • CreateComments / CreateSharedComments: 指定されたオブジェクトのコメントを挿入または更新する。
  • DeleteComments / DeleteSharedComments: 指定されたオブジェクトのコメントを削除する。
  • GetComment: 指定されたオブジェクトの現在のコメントを取得する。

制御フローとデータフロー

1. **コマンド処理**: CommentObject が get_object_address を呼び出し、対象オブジェクトのOIDとクラスIDを特定します。 2. **権限チェック**: check_object_ownership により、実行ユーザーが対象オブジェクトの所有者であることを確認します。 3. **カタログ選択**: オブジェクトの種類（データベース、ロール、テーブルスペース等）に応じて、共有カタログか通常カタログかを判定します。 4. **CRUD操作**: systable_beginscan を用いてインデックススキャンを行い、既存のコメントレコードを検索します。

  • 既存レコードがある場合：CatalogTupleUpdate または CatalogTupleDelete を実行。
  • 既存レコードがない場合：CatalogTupleInsert を実行。

読者向けメモ

  • **共有オブジェクト**: データベース、ロール、テーブルスペースはクラスタ全体で共有されるため、pg_shdescription を使用します。これらは subid（サブオブジェクトID）を持たない点に注意してください。
  • **空文字列の扱い**: 本モジュールでは、空文字列のコメントは「コメントの削除」として扱われます。
  • **ロック**: CommentObject は ShareUpdateExclusiveLock を取得し、コメント更新中にオブジェクトが削除されることを防ぎます。
