src/backend/commands/discard.c

概要

discard.c は、PostgreSQL の DISCARD SQL コマンドを実装するモジュールです。セッションの状態を初期化またはリセットするための機能を提供します。

モジュール内での役割

src/backend/commands ディレクトリの一部として、セッション管理に関連するコマンド処理を担います。他のコマンドが特定のオブジェクト（テーブルや関数など）を操作するのに対し、本ファイルはセッション全体の環境をクリーンアップする役割を持ちます。

責務

  • DISCARD コマンドの構文解析結果（DiscardStmt）に基づき、適切なリセット処理へディスパッチする。
  • DISCARD ALL 実行時、セッション内の全リソース（一時テーブル、準備済み文、ロック、GUC設定など）を安全に破棄する。
  • 特定のサブセット（TEMP, PLANS, SEQUENCES）のみをリセットする機能の提供。

重要なシンボル

  • DiscardCommand: DISCARD コマンドのメインエントリーポイント。ターゲットに応じて処理を分岐します。
  • DiscardAll: DISCARD ALL を実行する内部関数。セッション状態を初期状態へ戻す一連のクリーンアップ処理を順次実行します。

制御フローとデータフロー

1. **ディスパッチ**: DiscardCommand が stmt->target を評価し、対象に応じた関数を呼び出します。 2. **検証**: DiscardAll は PreventInTransactionBlock を呼び出し、トランザクションブロック内での実行を禁止します。 3. **リセット処理**: DiscardAll 内で、ポータル、GUC変数、準備済み文、非同期通知（LISTEN）、ロック、プランキャッシュ、一時テーブル、シーケンスの順にリセット処理が実行されます。

直接依存

  • access/xact.h: トランザクション状態のチェックに使用。
  • commands/async.h, prepare.h, sequence.h: 各種リソースのリセット用関数を提供。
  • utils/guc.h, utils/portal.h: セッション設定およびポータルの管理用。
  • catalog/namespace.h
  • commands/discard.h
  • commands/prepare.h
  • commands/sequence.h
  • postgres.h

関連コード

  • src/include/commands/discard.h: DiscardCommand のプロトタイプ定義。
  • src/backend/commands/variable.c: DISCARD 以外のセッション変数操作に関連。
  • src/backend/utils/cache/plancache.c: ResetPlanCache の実装元。

読者向けメモ

  • DISCARD ALL は非常に強力なコマンドであり、実行するとセッション内のすべての状態が失われます。そのため、トランザクションブロック内での実行は意図しない副作用を防ぐために明示的に禁止されています。
  • DiscardAll 内の処理順序は、ポータルのクローズなど、ユーザー定義コードが実行される可能性のある処理を優先するように設計されています。
