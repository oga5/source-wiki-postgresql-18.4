src/backend/commands/functioncmds.c

概要

本ファイルは、PostgreSQL における関数（FUNCTION）、プロシージャ（PROCEDURE）、キャスト（CAST）、および変換（TRANSFORM）の定義・変更・削除を司る DDL コマンドの実装です。構文解析後のノードを受け取り、システムカタログ（pg_proc, pg_cast, pg_transform 等）への反映と整合性チェックを行います。

モジュール内での役割

src/backend/commands サブシステムの一部として、関数オブジェクトのライフサイクルを管理します。パーサーが生成した構文木をカタログ操作可能な形式に変換し、catalog/ 配下の低レベルなカタログ更新関数を呼び出すブリッジの役割を果たします。

責務

  • **関数定義の検証**: 引数リスト、戻り値型、属性（volatility, parallel, strict 等）の妥当性確認。
  • **カタログ操作**: CREATE FUNCTION/PROCEDURE による pg_proc への挿入、DROP による削除。
  • **キャストと変換の管理**: 型変換ルール（pg_cast）および言語間変換（pg_transform）の定義。
  • **実行コンテキストの管理**: CALL 文実行時のアトミック性（トランザクション制御の可否）の判定と実行。

重要なシンボル

関数定義・変更

  • CreateFunction: CREATE FUNCTION/PROCEDURE のメインエントリ。属性解析からカタログ登録までを統括。
  • AlterFunction: ALTER FUNCTION の実装。pg_proc の属性更新と依存関係の再構築を行う。
  • RemoveFunctionById: 関数または集約関数の削除。pg_proc および pg_aggregate からの削除を実行。

パラメータ・属性解析

  • compute_function_attributes: CREATE/ALTER のオプションリストを解析し、関数属性へマッピング。
  • interpret_function_parameter_list: 引数リスト（IN, OUT, VARIADIC）を解析し、oidvector や ArrayType に変換。
  • compute_return_type: 戻り値型の解決。シェル型の作成や検証を行う。

キャスト・変換

  • CreateCast: CREATE CAST の実装。型間のバイナリ互換性や変換関数の妥当性を検証。
  • CreateTransform: CREATE TRANSFORM の実装。言語間変換関数の登録と依存関係の記録。

実行系

  • ExecuteDoStmt: DO ブロックの実行。言語ハンドラの laninline を呼び出す。
  • ExecuteCallStmt: CALL 文の実行。プロシージャの引数評価と実行コンテキスト（アトミック性）の管理。

制御フローとデータフロー

1. **解析フェーズ**: compute_function_attributes 等のヘルパーが DefElem リストを走査し、関数属性（volatility, cost 等）を抽出。 2. **型解決**: LookupTypeName や typenameTypeId を使用して、SQL レベルの型名をカタログ上の Oid に変換。 3. **カタログ更新**: ProcedureCreate（catalog/ 側）を呼び出し、pg_proc への挿入を行う。 4. **依存関係記録**: recordDependencyOn を使用し、関数が依存する型や言語との関係を pg_depend に記録。 5. **実行時フロー**: ExecuteCallStmt は FuncExpr を評価し、fmgr を介してプロシージャを呼び出す。アトミック性が false の場合、トランザクション制御を許可するコンテキストを渡す。

直接依存

  • catalog/: pg_proc, pg_cast, pg_transform 等のカタログ定義と操作関数。
  • parser/: parse_func.h, parse_type.h 等による型・関数名の解決。
  • utils/: syscache.h（カタログキャッシュ）、acl.h（権限チェック）、typcache.h（型情報）。
  • executor/: functions.h（関数実行基盤）。

関連コード

  • src/backend/catalog/pg_proc.c: ProcedureCreate 等のカタログ更新の低レベル実装。
  • src/backend/parser/gram.y: 本ファイルが処理する CreateFunctionStmt 等の構文定義。
  • src/backend/utils/fmgr/: 関数呼び出しの基盤（fmgr_info 等）。

読者向けメモ

  • **メモリ管理**: palloc を多用しており、コマンド実行単位のメモリコンテキストで管理されます。
  • **エラーパス**: ereport(ERROR, ...) が多用されており、検証失敗時はトランザクションがアボートされます。
  • **シェル型**: compute_return_type は、未定義の型を「シェル型」として作成する特殊なロジックを含みます。これは I/O 関数定義時に必要となる循環依存を解決するためです。
  • **アトミック性**: ExecuteCallStmt における atomic フラグは、プロシージャ内での COMMIT/ROLLBACK を許可するかを制御する重要なフラグです。
  • **省略箇所**: pg_aggregate の詳細なロジックは src/backend/catalog/pg_aggregate.c に委譲されており、本ファイルは主に pg_proc 側の操作に集中しています。
