src/backend/commands/functioncmds.c

概要

PostgreSQLにおける関数（FUNCTION）、プロシージャ（PROCEDURE）、キャスト（CAST）、および変換（TRANSFORM）の定義・削除・変更を管理するコマンド実装ファイルです。

モジュール内での役割

src/backend/commands モジュールの一部として、SQLのDDL文（CREATE FUNCTION, ALTER FUNCTION, DROP FUNCTION 等）を解析し、システムカタログ（pg_proc, pg_cast, pg_transform）を操作する責務を担います。

責務

  • **DDL解析と検証**: SQLパーサーから渡された構文木を解析し、引数、戻り値、言語、属性（volatility, cost等）の妥当性を検証する。
  • **カタログ操作**: 適切なシステムカタログ更新ルーチンを呼び出し、オブジェクトの作成・更新・削除を行う。
  • **権限チェック**: オブジェクト作成や実行に必要な権限（ACL）を検証する。
  • **実行コンテキスト管理**: CALL 文実行時のアトミック/非アトミックなコンテキスト管理や、インラインコード（DO 文）の実行をサポートする。

重要なシンボル

  • CreateFunction: CREATE FUNCTION/PROCEDURE 文のメインエントリポイント。
  • AlterFunction: ALTER FUNCTION 文の属性変更を処理する。
  • RemoveFunctionById: 関数や集約関数の削除処理。
  • CreateCast / CreateTransform: 型変換やデータ変換の定義を管理する。
  • ExecuteDoStmt / ExecuteCallStmt: プロシージャやインラインコードの実行を制御する。

制御フローとデータフロー

1. **解析**: CreateFunction 等が ParseState を受け取り、compute_function_attributes でオプションを抽出する。 2. **検証**: interpret_function_parameter_list で引数リストを解析し、型チェックや名前の重複確認を行う。 3. **変換**: interpret_AS_clause で関数本体（prosrc, probin）を確定させる。 4. **永続化**: ProcedureCreate などのカタログ操作関数を呼び出し、システムカタログを更新する。 5. **実行**: ExecuteCallStmt は FuncExpr を評価し、fmgr を通じてプロシージャを呼び出す。

読者向けメモ

  • 本ファイルは非常に多機能です。関数定義の複雑な属性（parallel, leakproof 等）の解析ロジックを追う際は、compute_function_attributes を起点にすると全体像が把握しやすいです。
  • CALL 文の実行（ExecuteCallStmt）は、トランザクション制御を伴う非アトミックな実行コンテキストを扱うため、他のDDL処理とは異なる複雑さを持っています。
  • pg_proc は関数だけでなく、プロシージャや集約関数も格納するため、prokind フィールドによる条件分岐が随所に見られます。
