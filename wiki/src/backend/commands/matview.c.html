src/backend/commands/matview.c

概要

マテリアライズド・ビュー（MV）の作成および更新（REFRESH MATERIALIZED VIEW）を管理するバックエンドモジュールです。MVのデータ再構築、一時テーブルを用いた差分更新、およびMVのメタデータ管理機能を提供します。

モジュール内での役割

src/backend/commands の一部として、SQLコマンドの実行エンジンとストレージ層の橋渡しをします。特に、tablecmds.c が提供するテーブル操作機能を利用し、MV特有の「クエリ結果の永続化」と「整合性維持」を実装しています。

責務

  • **データ再構築**: REFRESH コマンド実行時に、元のクエリを再実行して新しいデータを一時テーブルに生成する。
  • **アトミックな更新**:
  • 非同時実行時：finish_heap_swap を用いて物理ファイルを入れ替え、MVを更新する。
  • 同時実行時：一時テーブルとの差分（FULL JOIN）を計算し、DELETE/INSERT で増分更新を行う。
  • **メタデータ管理**: pg_class の relispopulated フラグを制御し、MVの有効状態を管理する。
  • **セキュリティ制御**: MV所有者の権限でクエリを実行するためのコンテキスト切り替え。

重要なシンボル

  • ExecRefreshMatView: SQLコマンドの入り口。ロックモードを決定し、処理を委譲する。
  • RefreshMatViewByOid: 更新処理のメインロジック。一時テーブルの作成、データ生成、入れ替え/マージのフローを統括する。
  • DR_transientrel: 一時テーブルへのデータ挿入を行うための DestReceiver 構造体。
  • refresh_by_match_merge: CONCURRENTLY 指定時の差分更新ロジック。
  • MatViewIncrementalMaintenanceIsEnabled: 内部的な更新処理中であることを示すフラグ制御。

制御フローとデータフロー

1. **初期化**: ExecRefreshMatView が対象MVをロックし、所有者の権限へ切り替える。 2. **データ生成**: refresh_matview_datafill が DestReceiver を介してクエリ結果を一時テーブルへ流し込む。 3. **更新適用**:

  • 通常更新: refresh_by_heap_swap で物理ファイルを置換。
  • 同時更新: refresh_by_match_merge が一時テーブルとMVの差分を計算し、SPI を通じて DELETE/INSERT を実行。

    • 4. **終了**: 統計情報の更新、一時テーブルの削除、権限コンテキストの復元を行う。

読者向けメモ

  • **同時実行更新の制約**: CONCURRENTLY を使用する場合、MVには「WHERE句のない一意なインデックス」が必須です。これは差分計算（FULL JOIN）のキーとして利用されるためです。
  • **セキュリティ**: REFRESH は所有者の権限で実行されるため、内部で SetUserIdAndSecContext を使用して権限昇格を行っています。
  • **SPIの利用**: refresh_by_match_merge 内では、複雑な差分計算のために SPI（Server Programming Interface）経由でSQLを発行しています。デバッグ時はこの生成されたクエリ文字列に注目してください。
