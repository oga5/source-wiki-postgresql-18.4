src/backend/commands/event_trigger.c

概要

PostgreSQL のイベントトリガー機能のコア実装です。DDL コマンドの実行前後や、特定のイベント（sql_drop, table_rewrite, login）発生時にユーザー定義関数を呼び出すためのディスパッチ、状態管理、およびコマンドの収集（deparsing）を担います。

モジュール内での役割

commands モジュールの一部として、DDL 実行エンジン（utility.c 等）と密接に連携します。イベントトリガーは、DDL コマンドのライフサイクルに介入し、メタデータ操作や監査ログの記録、あるいは特定の操作の禁止を可能にします。

責務

  • **イベントトリガーの管理**: CREATE, ALTER, DROP コマンドによるイベントトリガー定義のカタログ（pg_event_trigger）操作。
  • **実行ディスパッチ**: EventTriggerInvoke を通じたトリガー関数の実行と、実行コンテキスト（EventTriggerData）の提供。
  • **状態管理**: EventTriggerQueryState を用いた、コマンド実行中のドロップ対象オブジェクトや実行済みコマンドの追跡。
  • **DDL 収集**: deparse_utility.h と連携し、実行された DDL コマンドの履歴を保持してトリガー関数から参照可能にする。

重要なシンボル

状態管理

  • currentEventTriggerState: 現在のクエリ実行におけるイベントトリガーの状態を保持するスタックトップ。
  • EventTriggerQueryState: 実行中のコマンドに関連するドロップ対象オブジェクトリストや、収集されたコマンドリストを保持する構造体。

トリガー実行・ディスパッチ

  • EventTriggerDDLCommandStart / End: DDL コマンドの開始・終了時に呼び出されるフック。
  • EventTriggerSQLDrop: オブジェクト削除時に呼び出されるフック。
  • EventTriggerInvoke: リストされたトリガー関数を順次実行するエンジン。

カタログ操作

  • CreateEventTrigger: pg_event_trigger への新規登録。
  • AlterEventTrigger: トリガーの有効/無効状態の更新。
  • SetDatabaseHasLoginEventTriggers: pg_database.dathasloginevt フラグの管理。

ユーティリティ関数

  • pg_event_trigger_dropped_objects: sql_drop トリガー内で削除対象オブジェクトを取得する関数。
  • pg_event_trigger_ddl_commands: 実行された DDL コマンドリストを取得する関数。

制御フローとデータフロー

1. **開始**: ProcessUtility が EventTriggerDDLCommandStart を呼び出し、キャッシュから該当するトリガーを検索。 2. **収集**: コマンド実行中、EventTriggerSQLDropAddObject や EventTriggerCollectSimpleCommand が呼び出され、currentEventTriggerState に情報が蓄積される。 3. **実行**: EventTriggerInvoke がトリガー関数を呼び出す際、EventTriggerData を fcinfo->context にセットし、関数側で CALLED_AS_EVENT_TRIGGER マクロにより情報を取得可能にする。 4. **終了**: EventTriggerEndCompleteQuery により、クエリ終了時にメモリコンテキストが解放され、状態が復元される。

読者向けメモ

  • **メモリ管理**: EventTriggerQueryState は TopMemoryContext ではなく、クエリごとに作成される専用のメモリコンテキストで管理されます。EventTriggerEndCompleteQuery で確実に解放される必要があります。
  • **再入可能性**: イベントトリガー関数自体が DDL を実行する場合、currentEventTriggerState はスタック構造（previous ポインタ）になっており、再帰的な状態管理が可能です。
  • **制約**: イベントトリガーはシングルユーザーモードでは無効化されます。また、event_triggers GUC で全体を無効化可能です。
  • **未実装/浅い部分**: EventTriggerCollectGrant 等の収集関数は、特定の DDL コマンドに対してのみ実装されており、すべての DDL を網羅しているわけではありません。また、FIXME にある通り、トランザクションのサブトランザクションアボート時のコマンド収集の整合性には注意が必要です。
