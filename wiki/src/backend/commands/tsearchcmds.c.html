src/backend/commands/tsearchcmds.c

概要

PostgreSQL の全文検索（Tsearch）機能に関連する DDL コマンド（CREATE/ALTER）の実行ロジックを実装するモジュールです。テキスト検索パーサー、辞書、テンプレート、および設定の定義と変更を管理します。

モジュール内での役割

src/backend/commands ディレクトリの一部として、SQL レベルのテキスト検索オブジェクト定義をシステムカタログ（pg_ts_*）へ反映させる役割を担います。tsearch サブシステムの管理層に位置し、パーサーや辞書のキャッシュ更新や依存関係の解決を仲介します。

責務

  • **オブジェクト定義の永続化**: CREATE 文に基づき、システムカタログへの挿入と依存関係（pg_depend）の構築を行います。
  • **設定の変更**: ALTER 文による辞書オプションの更新や、設定（Configuration）におけるトークンと辞書のマッピング変更を処理します。
  • **検証とシリアライズ**: テンプレートの初期化メソッドの検証や、辞書オプションのシリアライズ/デシリアライズ（pg_dump 対応のため）を管理します。

重要なシンボル

  • DefineTSParser, DefineTSDictionary, DefineTSTemplate, DefineTSConfiguration: 各オブジェクトを作成する主要なエントリポイント。
  • AlterTSDictionary, AlterTSConfiguration: 各オブジェクトの属性やマッピングを変更するエントリポイント。
  • make*Dependencies: 各オブジェクト作成時に pg_depend を構築するヘルパー関数群。
  • serialize_deflist / deserialize_deflist: 辞書の初期化オプションをテキスト形式と List 形式間で変換する関数。

制御フローとデータフロー

1. **解析**: SQL 文から渡された List *parameters を走査し、必要な関数 OID やオプションを抽出します。 2. **検証**: LookupFuncName や verify_dictoptions を使用して、指定された関数やオプションが妥当かを確認します。 3. **カタログ更新**: table_open で対象カタログを開き、heap_form_tuple でタプルを作成して CatalogTupleInsert を実行します。 4. **依存関係登録**: make*Dependencies を呼び出し、作成したオブジェクトと名前空間や関数との依存関係を pg_depend に記録します。 5. **フック**: InvokeObjectPostCreateHook を呼び出し、イベントトリガーやキャッシュ無効化をトリガーします。

読者向けメモ

  • 本ファイルは主に DDL 処理に特化しています。検索実行時の挙動を追う場合は src/backend/tsearch/ を参照してください。
  • serialize_deflist は pg_dump が辞書定義を再構築できるように、SQL 構文に近い形式で文字列化を行う点に注意してください。
  • ALTER 操作において、依存関係の更新が必要な場合と不要な場合（オプション変更のみなど）の判断ロジックが各関数内に含まれています。
