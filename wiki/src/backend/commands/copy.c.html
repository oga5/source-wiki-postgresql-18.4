src/backend/commands/copy.c

概要

copy.c は、PostgreSQL の COPY SQL コマンドの実行を統括するメインエントリーポイントです。テーブルと外部ファイル（またはプログラム、標準入出力）間でのデータ転送を管理し、権限チェック、オプションの解析、および実行フェーズへの橋渡しを行います。

モジュール内での役割

src/backend/commands モジュール内において、COPY コマンドの構文解析後の処理を担います。実際のデータ転送ロジックは copyfrom.c（読み込み）および copyto.c（書き出し）に委譲されており、本ファイルはそれらの上位層として機能します。

責務

  • **権限管理**: ファイルシステムへのアクセス権限（pg_read_server_files 等）の検証。
  • **オプション解析**: COPY コマンドのオプション（FORMAT, DELIMITER, NULL 等）の妥当性検証と構造体への変換。
  • **実行制御**: COPY 対象のテーブルまたはクエリの準備、および適切な実行関数（BeginCopyFrom/BeginCopyTo）の呼び出し。
  • **制約チェック**: 生成列の除外や、RLS（行レベルセキュリティ）が有効な場合のクエリベース転送への切り替え。

重要なシンボル

  • DoCopy: COPY コマンドを実行するメイン関数。
  • ProcessCopyOptions: COPY 文のオプションリストを解析し、CopyFormatOptions 構造体を構築する。
  • CopyGetAttnums: 指定された列名リストから属性番号のリストを生成・検証する。

追加の索引済みシンボル

  • defGetCopyHeaderChoice
  • defGetCopyLogVerbosityChoice
  • defGetCopyOnErrorChoice
  • defGetCopyRejectLimitOption

制御フローとデータフロー

1. **初期化**: DoCopy が ParseState と CopyStmt を受け取り、権限を確認する。 2. **準備**:

  • テーブル対象の場合、table_openrv でロックを取得。
  • RLS が有効な場合、COPY を SELECT クエリに変換して処理を継続する。

    • 3. **オプション解析**: ProcessCopyOptions が DefElem リストを走査し、フォーマットの整合性をチェックする。 4. **実行**:

  • is_from フラグに基づき、CopyFrom または DoCopyTo を呼び出し、実際のデータストリーム処理を開始する。

    • 5. **終了**: table_close でリソースを解放する。

読者向けメモ

  • 本ファイルは主に「コマンドのセットアップ」に特化しています。データ転送のパフォーマンスやパースの詳細を追う場合は、copyfrom.c や copyfromparse.c を参照してください。
  • COPY オプションの追加や変更を行う際は、ProcessCopyOptions 内の整合性チェックロジックが非常に厳格であることに注意してください。
  • RLS（行レベルセキュリティ）が有効な場合、COPY は内部的に SELECT クエリとして再構築されるため、デバッグ時にはクエリ変換のフローを追う必要があります。
