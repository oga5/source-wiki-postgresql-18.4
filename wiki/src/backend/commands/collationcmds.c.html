src/backend/commands/collationcmds.c

概要

本ファイルは、PostgreSQLにおける照合順序（Collation）オブジェクトの作成、変更、およびシステム提供の照合順序のインポートを管理するコマンド処理の実装です。

モジュール内での役割

src/backend/commands モジュールの一部として、SQLの CREATE COLLATION、ALTER COLLATION、およびシステムカタログ pg_collation を操作するユーティリティ関数を提供します。

責務

  • **照合順序の定義**: CREATE COLLATION 文の解析と、プロバイダ（libc, icu, builtin）ごとの妥当性検証およびカタログ登録。
  • **照合順序の更新**: ALTER COLLATION によるバージョン情報の更新処理。
  • **システム照合順序のインポート**: OSやライブラリ（libc, ICU, Win32）から利用可能な照合順序を検出し、pg_collation に一括登録する機能。

重要なシンボル

  • DefineCollation: CREATE COLLATION 文を実行するメイン関数。
  • AlterCollation: 照合順序のバージョン更新を処理する関数。
  • pg_import_system_collations: OS環境から照合順序を自動検出してカタログへ取り込む関数。
  • CollAliasData: libcのロケール名とエイリアスを管理するための内部構造体。

制御フローとデータフロー

1. **定義処理**: DefineCollation が ParseState から引数を抽出し、プロバイダに応じた検証（icu_validate_locale 等）を経て CollationCreate を呼び出しカタログを更新します。 2. **インポート処理**: pg_import_system_collations がプラットフォーム固有のAPI（locale -a, EnumSystemLocalesEx, uloc_getAvailable）を呼び出し、検出したロケールを create_collation_from_locale を介してカタログへ挿入します。 3. **バージョン管理**: get_collation_actual_version を使用して、現在のライブラリ環境における照合順序のバージョンを特定し、カタログの collversion 列を更新します。

読者向けメモ

  • 本ファイルはプラットフォーム依存のコード（#ifdef WIN32 等）を多く含みます。特にロケールの列挙ロジックはOSごとに大きく異なるため、移植や修正の際は注意が必要です。
  • COLLPROVIDER_DEFAULT は特別な扱いであり、ユーザーが直接作成することはできません。
  • ICUプロバイダを使用する場合、非ASCII文字を含むロケール名はカタログ登録時にフィルタリングされる仕様になっています。
