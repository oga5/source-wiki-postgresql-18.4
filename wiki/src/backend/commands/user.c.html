src/backend/commands/user.c

概要

本ファイルは、PostgreSQL におけるロール（ユーザーおよびグループ）のライフサイクル管理および権限付与（GRANT/REVOKE）を実装する中核モジュールです。システムカタログ pg_authid（ロール定義）および pg_auth_members（ロール間のメンバーシップ関係）を操作し、DDL コマンドの実行を担います。

モジュール内での役割

src/backend/commands サブシステムの一部として、認証・認可の基盤となるロール定義の永続化と、複雑なメンバーシップの依存関係管理を提供します。

責務

  • **ロールのライフサイクル管理**: CREATE ROLE, ALTER ROLE, DROP ROLE, RENAME ROLE の実行。
  • **メンバーシップ管理**: GRANT/REVOKE によるロール間の包含関係の制御。
  • **権限検証**: ロール作成・変更・削除に必要な特権（CREATEROLE, SUPERUSER 等）のチェック。
  • **依存関係の解決**: ロール削除時の依存オブジェクトチェックおよび再帰的な権限剥奪（CASCADE 処理）。
  • **所有権の移譲**: DROP OWNED および REASSIGN OWNED によるオブジェクト所有権の管理。

重要なシンボル

ロール操作関数

  • CreateRole: pg_authid への新規挿入と初期メンバーシップ設定。
  • AlterRole: ロール属性（パスワード、接続制限等）の更新。
  • DropRole: ロールおよび関連する依存関係の削除。
  • RenameRole: ロール名の変更と関連するキャッシュの更新。

メンバーシップ管理関数

  • GrantRole: GRANT/REVOKE コマンドのディスパッチャー。
  • AddRoleMems: pg_auth_members へのメンバー追加・更新。
  • DelRoleMems: pg_auth_members からのメンバー削除および再帰的剥奪の計画。

内部状態・補助構造体

  • GrantRoleOptions: GRANT 実行時のオプション（ADMIN, INHERIT, SET）を保持。
  • RevokeRoleGrantAction: 権限剥奪時の再帰的アクション（削除、オプション解除、何もしない）を定義する列挙型。

制御フローとデータフロー

1. **コマンド解析**: ProcessUtility から各 Stmt 構造体を受け取り、オプションを抽出。 2. **権限チェック**: superuser() や have_createrole_privilege() を用いて操作の可否を判定。 3. **カタログ操作**: table_open(AuthIdRelationId, ...) を経由して pg_authid を更新。 4. **メンバーシップ更新**:

  • AddRoleMems は pg_auth_members を更新し、CommandCounterIncrement() を呼び出して変更を可視化。
  • DelRoleMems は initialize_revoke_actions で剥奪計画を立て、plan_recursive_revoke で依存関係を再帰的に解決（CASCADE 処理）。

    • 5. **フック呼び出し**: InvokeObjectPostCreateHook 等により、外部モジュールやイベントトリガーへ変更を通知。

直接依存

  • **カタログ**: pg_authid, pg_auth_members, pg_database 等のシステムカタログ定義。
  • **アクセス層**: access/table.h, access/genam.h（テーブル操作とスキャン）。
  • **ストレージ**: storage/lmgr.h（AccessExclusiveLock 等による排他制御）。
  • **ユーティリティ**: utils/syscache.h（システムキャッシュによる高速なロール検索）。

関連コード

  • src/include/commands/user.h: 公開インターフェース定義。
  • src/backend/catalog/pg_authid.c: ロール関連のカタログ操作補助。
  • src/backend/commands/dbcommands.c: データベース所有権に関連する操作。
  • src/backend/utils/adt/acl.c: 権限チェックの基盤ロジック。

読者向けメモ

  • **メモリ管理**: palloc を多用しており、トランザクション終了時に自動解放されるメモリコンテキストに依存しています。
  • **排他制御**: ロール操作時は AuthIdRelationId に対して RowExclusiveLock を取得し、メンバーシップ変更時は ShareUpdateExclusiveLock を用いて競合を防いでいます。
  • **再帰的剥奪**: plan_recursive_revoke は DROP CASCADE 時に依存する権限を再帰的に探索します。この際、循環参照が発生しないよう is_member_of_role_nosuper 等でガードしています。
  • **省略箇所**: 本ファイル内の check_password_hook 関連の詳細は、外部認証プラグインとの連携用であり、コアロジックの理解には必須ではありません。
