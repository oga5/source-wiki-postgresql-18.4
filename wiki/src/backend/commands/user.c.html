src/backend/commands/user.c

概要

PostgreSQLにおけるロール（旧称：ユーザー）の作成、変更、削除、およびロール間のメンバーシップ管理（GRANT/REVOKE）を担うコマンド実装ファイルです。

モジュール内での役割

src/backend/commands モジュールの一部として、データベースの認証・認可基盤である pg_authid および pg_auth_members カタログを操作する中心的な役割を果たします。

責務

  • **ロール管理**: CREATE ROLE, ALTER ROLE, DROP ROLE, RENAME ROLE の実行。
  • **メンバーシップ管理**: GRANT / REVOKE によるロール間の継承関係や権限付与の管理。
  • **権限チェック**: ロール操作に必要な特権（CREATEROLE 属性や ADMIN OPTION）の検証。
  • **依存関係の解決**: ロール削除時の依存オブジェクトチェックや、メンバーシップ取り消し時の再帰的な権限剥奪（CASCADE処理）。

重要なシンボル

  • CreateRole, AlterRole, DropRole: 主要なDDLコマンドの実行関数。
  • GrantRole: ロールメンバーシップの付与・剥奪を行う関数。
  • AddRoleMems, DelRoleMems: メンバーシップ更新の内部ロジック。
  • RevokeRoleGrantAction: 権限剥奪時の再帰的なアクションを定義する列挙型。
  • GrantRoleOptions: GRANT 時に指定されるオプション（ADMIN, INHERIT, SET）を保持する構造体。

制御フローとデータフロー

1. **DDL実行**: 各関数は ParseState を受け取り、stmt ノードからオプションを抽出します。 2. **権限検証**: superuser() や has_createrole_privilege() を用いて操作の可否を判定します。 3. **カタログ更新**: pg_authid や pg_auth_members を table_open で開き、CatalogTupleInsert / Update / Delete を実行します。 4. **再帰的処理**: DelRoleMems は plan_recursive_revoke を呼び出し、依存関係を辿って権限剥奪の影響範囲を計算します。 5. **フック**: check_password_hook や InvokeObjectPostCreateHook を通じて、外部モジュールやシステムイベントへの通知を行います。

読者向けメモ

  • **再帰的REVOKE**: plan_recursive_revoke は、権限の連鎖（ADMIN OPTIONの継承）を考慮した複雑なロジックです。CASCADE指定時の挙動を理解する際はここが重要です。
  • **ロック**: ロール操作時は LockSharedObject を使用して、並行実行による循環参照や競合を防いでいます。
  • **セキュリティ**: 本ファイルは認証の根幹に関わるため、権限チェックのロジックは非常に厳格です。特に superuser 属性の変更や、pg_ で始まる予約済みロール名の扱いに注意してください。
