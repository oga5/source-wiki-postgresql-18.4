src/backend/commands/seclabel.c

概要

PostgreSQL のセキュリティラベル機能（SECURITY LABEL SQL コマンド）を実装するバックエンドモジュールです。データベースオブジェクトに対して、外部のセキュリティプロバイダーがラベルを付与・管理するための基盤を提供します。

モジュール内での役割

commands サブシステムの一部として、DDL 文の実行とシステムカタログ（pg_seclabel, pg_shseclabel）の操作を仲介します。外部モジュール（SELinux や AppArmor 等の拡張機能）がフックを登録し、ラベルの妥当性を検証するインターフェースを提供します。

責務

  • **DDL 実行:** SECURITY LABEL 文の解析結果を受け取り、対象オブジェクトの特定と所有権チェック、プロバイダーへの検証依頼を行います。
  • **カタログ管理:** セキュリティラベルの追加、更新、削除をシステムカタログに対して行います。
  • **プロバイダー登録:** 外部拡張機能がラベル検証ロジックを登録するための register_label_provider API を提供します。
  • **共有オブジェクト対応:** 共有カタログ（pg_shseclabel）と通常カタログ（pg_seclabel）の差異を抽象化します。

重要なシンボル

  • ExecSecLabelStmt: SECURITY LABEL コマンドのメインエントリポイント。
  • register_label_provider: 拡張機能がラベル検証フックを登録するための関数。
  • SetSecurityLabel / DeleteSecurityLabel: ラベルの永続化・削除を行う低レベル API。
  • label_provider_list: ロードされたプロバイダーを保持する静的リスト。

制御フローとデータフロー

1. **コマンド実行:** ExecSecLabelStmt が呼ばれると、指定されたプロバイダーを label_provider_list から検索します。 2. **検証:** get_object_address で対象を特定し、所有権を確認後、登録されたプロバイダーのフック関数を呼び出してラベルの妥当性をチェックします。 3. **永続化:** 検証が成功すれば、SetSecurityLabel がカタログ（pg_seclabel 等）を更新します。 4. **カタログ操作:** SetSecurityLabel は既存ラベルの有無に応じて CatalogTupleUpdate または CatalogTupleInsert を選択します。

読者向けメモ

  • **プロバイダーの必須性:** プロバイダーが複数ロードされている場合、SQL 文で明示的に指定しないとエラーになります。
  • **フックの役割:** 本ファイルはラベルの「保存場所」を管理しますが、ラベルの「意味」や「妥当性」は登録されたフック関数（check_object_relabel_type）に委譲されています。
  • **ロック:** get_object_address により、操作対象のオブジェクトには ShareUpdateExclusiveLock が取得されます。
