src/backend/commands/async.c

概要

PostgreSQL の非同期通知システム（LISTEN/NOTIFY/UNLISTEN）を実装するモジュールです。ディスクベースの SLRU（Simple Least Recently Used）キューを共有メモリ上にマッピングし、バックエンド間でのイベント伝達を管理します。

モジュール内での役割

src/backend/commands サブシステムの一部として、SQL コマンドの実行とトランザクションライフサイクルに統合された非同期通信機能を提供します。他のバックエンドへの通知送信、および自身のリスニング状態の管理を担います。

責務

  • **通知キューの管理**: SLRU を用いた共有メモリ上の通知キューの読み書きおよびディスクへの永続化（再起動時にクリアされる一時的なデータ）。
  • **トランザクション統合**: NOTIFY はトランザクションコミット時にキューへ書き込まれ、LISTEN 状態の変更もコミット時に反映される。
  • **リスナー管理**: 各バックエンドのリスニング状態（PID、DB OID、読み取り位置）を共有メモリ上で追跡。
  • **シグナル伝達**: PROCSIG_NOTIFY_INTERRUPT を使用して、通知が届いたことを他のバックエンドに通知。

重要なシンボル

キュー制御構造体

  • AsyncQueueControl: 共有メモリ上のメイン制御構造体。キューのヘッド/テール位置、リスナーリストの先頭などを保持。
  • QueueBackendStatus: 各バックエンドのリスニング状態を保持する配列要素。
  • AsyncQueueEntry: キュー内の個々の通知メッセージ。

トランザクション状態管理

  • pendingNotifies: 現在のトランザクションで発行された通知のリスト。コミット時にキューへフラッシュされる。
  • pendingActions: 現在のトランザクションで発行された LISTEN/UNLISTEN アクションのリスト。
  • listenChannels: 現在のバックエンドがリスニング中のチャネル名リスト。

主要なインターフェース

  • Async_Notify, Async_Listen, Async_Unlisten: SQL コマンドのバックエンド実装。
  • PreCommit_Notify, AtCommit_Notify: トランザクションコミット時のフック。
  • HandleNotifyInterrupt, ProcessNotifyInterrupt: 非同期シグナル受信時の処理フロー。

制御フローとデータフロー

1. **NOTIFY 発行**: Async_Notify が pendingNotifies にイベントを蓄積。 2. **コミット時**: PreCommit_Notify が NotifyQueueLock を取得し、pendingNotifies を SLRU キューへ書き込む。 3. **通知送信**: AtCommit_Notify が SignalBackends を呼び出し、リスナーに対して PROCSIG_NOTIFY_INTERRUPT を送信。 4. **受信処理**: シグナルハンドラ HandleNotifyInterrupt がラッチを設定。ProcessNotifyInterrupt が asyncQueueReadAllNotifications を呼び出し、キューから自身の DB OID に一致するメッセージを読み取り、フロントエンドへ送信。

読者向けメモ

  • **メモリとライフサイクル**: pendingNotifies 等のリストは TopTransactionContext に割り当てられ、サブトランザクションのコミット/アボート時に適切にマージまたは破棄されます。
  • **ロック順序**: デッドロック回避のため、NotifyQueueTailLock → NotifyQueueLock → SLRU bank lock の順序で取得する必要があります。
  • **制約**: NOTIFY ペイロードは BLCKSZ（通常 8KB）の SLRU ページ内に収まる必要があります。
  • **未実装/浅い部分**: pg_listening_channels 等の関数は、現在のトランザクション内で listenChannels が不変であることを前提としています。また、AsyncNotifyFreezeXids は VACUUM による CLOG メンテナンス時に古い XID を凍結するために使用されます。
