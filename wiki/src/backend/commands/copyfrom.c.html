src/backend/commands/copyfrom.c

概要

copyfrom.c は、PostgreSQL の COPY <table> FROM コマンドの実行エンジンです。外部ソース（ファイル、プログラム、クライアント）からデータを読み込み、テーブルへ効率的に挿入する責務を負います。単なる挿入だけでなく、パーティションルーティング、トリガー実行、制約チェック、インデックス更新、および FDW（外部データラッパー）との連携を統合的に管理します。

モジュール内での役割

src/backend/commands モジュールの一部として、データ入出力の「入力側」を担います。copyto.c が出力側を担うのに対し、本ファイルは実行時の状態管理（CopyFromState）と、挿入の最適化（マルチインサート）に特化しています。

責務

  • **データソースの抽象化**: ファイル、プログラム、コールバック関数からの入力を統一的に扱う。
  • **挿入の最適化**: CopyMultiInsertInfo を用いたバッチ挿入の制御。パーティションテーブルへのルーティングと、各パーティションごとのバッファ管理。
  • **実行コンテキストの管理**: COPY コマンド実行中のエラーハンドリング、進捗状況の報告、メモリコンテキストの分離。
  • **トリガーと制約の統合**: BEFORE/AFTER トリガー、生成列、チェック制約の適用。

重要なシンボル

状態管理

  • CopyFromState: COPY FROM 実行中の全状態を保持するメイン構造体。
  • CopyFromRoutine: テキスト、CSV、バイナリ形式ごとの処理を抽象化する関数ポインタテーブル。

マルチインサート（バッチ処理）

  • CopyMultiInsertInfo: 複数のパーティションやテーブルに対するバッチ挿入状態を管理する。
  • CopyMultiInsertBuffer: 特定のリレーションに対するタプルバッファ。MAX_BUFFERED_TUPLES（1000）および MAX_BUFFERED_BYTES（65535）でフラッシュタイミングを制御。

主要関数

  • CopyFrom: COPY FROM のメインループ。タプルの読み込みから挿入、トリガー実行までを統括。
  • BeginCopyFrom / EndCopyFrom: 実行コンテキストの初期化およびリソース解放。
  • CopyMultiInsertInfoFlush: バッファされたタプルを物理ストレージへ書き出す。

制御フローとデータフロー

1. **初期化**: BeginCopyFrom が CopyFromState を作成し、フォーマットに応じた CopyFromRoutine を設定。 2. **ループ処理**: CopyFrom 内の無限ループで NextCopyFrom を呼び出し、入力を取得。 3. **ルーティング**: パーティションテーブルの場合、ExecFindPartition で適切なリレーションを特定。 4. **挿入**:

  • **シングル挿入**: table_tuple_insert を直接呼び出す。
  • **マルチ挿入**: CopyMultiInsertInfoStore でバッファに蓄積し、閾値に達した時点で CopyMultiInsertBufferFlush を実行。

    • 5. **後処理**: EndCopyFrom でファイルハンドルやパイプを閉じ、メモリコンテキストを破棄。

読者向けメモ

  • **メモリ管理**: cstate->copycontext を使用して、COPY 実行中のメモリを分離しています。これにより、大量のデータ処理でもメモリリークを抑制します。
  • **マルチインサートの制約**: BEFORE トリガーや INSTEAD OF トリガーが存在する場合、マルチインサートは無効化されます。これはトリガーが挿入済みの行を参照する可能性があるためです。
  • **エラーハンドリング**: CopyFromErrorCallback は、エラー発生時に現在の行番号や列名をコンテキストとして出力するために使用されます。
  • **省略された詳細**: パース処理（NextCopyFrom の詳細）は copyfromparse.c に委譲されており、本ファイルには含まれていません。これは責務の分離によるものです。
