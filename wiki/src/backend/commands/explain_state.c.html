src/backend/commands/explain_state.c

概要

explain_state.c は、PostgreSQL の EXPLAIN コマンド実行時に使用される状態管理構造体 ExplainState の初期化、オプション解析、および拡張機能（Extension）によるカスタムオプションの登録・管理を行うモジュールです。

モジュール内での役割

src/backend/commands サブシステムにおいて、EXPLAIN の実行状態を保持するデータ構造のライフサイクルを管理します。explain.c がプランの出力処理を行うのに対し、本ファイルは出力に必要な設定値や拡張機能用のプライベート領域を準備する役割を担います。

責務

  • ExplainState 構造体の生成とデフォルト値の設定。
  • EXPLAIN コマンドのオプション（ANALYZE, VERBOSE, FORMAT 等）の解析と検証。
  • 拡張機能が独自の EXPLAIN オプションを登録・利用するための API 提供。
  • 拡張機能ごとの状態保持用領域（opaque pointer 配列）の動的管理。

重要なシンボル

  • ExplainState: EXPLAIN の実行全体で共有される状態構造体。
  • NewExplainState(): 構造体の割り当てとデフォルト値の初期化を行うコンストラクタ。
  • ParseExplainOptionList(): DefElem リストを走査し、ExplainState に反映させるメインの解析関数。
  • RegisterExtensionExplainOption(): 拡張機能が独自のオプションハンドラを登録するための関数。
  • GetExplainExtensionId(): 拡張機能に一意の ID を割り当て、状態管理用配列のインデックスを生成する。

制御フローとデータフロー

1. **初期化**: NewExplainState が呼ばれ、palloc0 でメモリを確保し、デフォルトの出力バッファを準備します。 2. **解析**: ParseExplainOptionList が EXPLAIN のオプションリストを反復処理します。組み込みオプションは直接 ExplainState に書き込まれ、未知のオプションは ApplyExtensionExplainOption を通じて登録済みハンドラに委譲されます。 3. **検証**: 解析後、ANALYZE との依存関係（WAL, TIMING 等）の整合性チェックを行い、必要に応じて explain_validate_options_hook を呼び出します。 4. **拡張機能の利用**: 拡張機能は GetExplainExtensionId で取得した ID をキーとして、SetExplainExtensionState / GetExplainExtensionState を経由し、ExplainState 内の extension_state 配列に独自の構造体を格納・取得します。

直接依存

  • postgres.h: 基本的な型定義とメモリ管理。
  • commands/defrem.h: DefElem（オプション定義）の操作。
  • commands/explain.h: EXPLAIN 関連の共通定義。
  • commands/explain_state.h: 本ファイルのプロトタイプと ExplainState 構造体定義。

関連コード

  • src/backend/commands/explain.c: ExplainState を利用してプランツリーを実際に文字列化するメインロジック。
  • src/backend/commands/explain_format.c: 出力フォーマット（JSON, XML 等）の制御。

読者向けメモ

  • **拡張機能の ID**: GetExplainExtensionId で返される ID はバックエンドの生存期間中のみ有効であり、永続的なものではありません。
  • **メモリ管理**: extension_state 配列は必要に応じて repalloc で拡張されるため、拡張機能側でポインタを保持し続ける際は注意が必要です。
  • **フックの利用**: explain_validate_options_hook は、コアのオプション解析後に独自の検証ロジックを追加したい場合に利用します。
