src/backend/commands/define.c

概要

define.c は、PostgreSQL の SQL コマンド解析過程で生成される DefElem（Definition Element）ノードから、特定のデータ型や値を取り出すためのユーティリティ関数群を提供します。

モジュール内での役割

src/backend/commands 内の各コマンド実行ルーチン（tablecmds.c や typecmds.c など）が、WITH 句やオプション引数を解析する際に共通して利用するアクセサ層です。

責務

  • DefElem ノードの arg フィールド（Node 型）を、呼び出し側が期待する型（int32, bool, Oid, List など）へ安全に変換する。
  • 型変換が不可能な場合や、期待される値が欠落している場合に、一貫した構文エラー（ERRCODE_SYNTAX_ERROR）を報告する。
  • 冗長なオプション指定に対するエラー報告の標準化。

重要なシンボル

  • defGetString: DefElem から文字列を抽出。数値や真偽値も文字列に変換して返す汎用的な関数。
  • defGetBoolean: 0/1, true/false, on/off を許容し、bool 型へ変換。
  • defGetNumeric / defGetInt32 / defGetInt64: 数値型への変換。Float として渡された大きな数値を int8 や Oid に変換するロジックを含む。
  • defGetQualifiedName: 名前空間やオブジェクト名を List として取得。
  • errorConflictingDefElem: 構文解析中に重複したオプションが指定された際のエラー報告用。

追加の索引済みシンボル

  • defGetObjectId
  • defGetStringList
  • defGetTypeLength
  • defGetTypeName

制御フローとデータフロー

1. **入力**: パーサーが生成した DefElem 構造体を受け取る。 2. **型判定**: nodeTag(def->arg) を使用して、ノードの実際の型（T_Integer, T_String, T_TypeName 等）をスイッチ文で分岐。 3. **変換**: 各関数内で、必要に応じて intVal() や strVal() などのマクロ、あるいは DirectFunctionCall1 を用いた型変換を実行。 4. **エラー処理**: 期待する型と異なる場合や def->arg が NULL の場合、ereport を発行して処理を中断。

  • catalog/namespace.h: 名前空間関連の定義。

関連コード

  • src/include/commands/defrem.h: 本ファイルの関数プロトタイプ定義。
  • src/backend/parser/gram.y: DefElem が生成される構文解析の起点。

読者向けメモ

  • 本ファイルは純粋なデータ変換ユーティリティであり、データベースの状態変更やカタログ操作は行いません。
  • defGetNumeric や defGetInt64 は、パーサーが数値を Integer ではなく Float として解釈した場合（範囲外の値など）も考慮した実装になっています。
  • 新しいオプション引数を追加する際は、既存の defGet* 関数で対応可能か確認し、必要に応じて本ファイルに新しいアクセサを追加してください。
