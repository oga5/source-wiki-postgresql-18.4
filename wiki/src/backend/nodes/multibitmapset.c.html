src/backend/nodes/multibitmapset.c

概要

multibitmapset.c は、2つの整数（listidx と bitidx）で識別される要素を管理するための「Bitmapset のリスト」を操作するユーティリティ群を提供します。クエリ内の変数（varno と varattno）の集合など、二次元的なインデックスを持つ集合を効率的に扱うために設計されています。

モジュール内での役割

src/backend/nodes サブシステムの一部として、Bitmapset を拡張した高次なデータ構造を提供します。標準的な Bitmapset が単一の整数インデックスを扱うのに対し、本モジュールは List と Bitmapset を組み合わせることで、より複雑な識別子空間をサポートします。

責務

  • **集合演算の提供**: UNION（mbms_add_members）や INTERSECT（mbms_int_members）といった、Bitmapset の操作をリスト全体に適用する機能の提供。
  • **動的な構造管理**: リストの長さを必要に応じて自動的に拡張し、空の要素を適切に処理する。
  • **メンバーシップ判定**: 特定の二次元インデックスが集合に含まれるかの判定。

重要なシンボル

  • mbms_add_member: 指定したインデックスにビットを立てる。リストが不足していれば自動拡張する。
  • mbms_add_members: 2つのマルチビットマップセットの和集合を計算し、左辺を更新する。
  • mbms_int_members: 2つのマルチビットマップセットの積集合を計算し、左辺を更新する。
  • mbms_is_member: 指定したインデックスが集合内に存在するかを判定する。
  • mbms_overlap_sets: 2つのマルチビットマップセットを比較し、共通部分を持つリストインデックスの集合（Bitmapset）を返す。

制御フローとデータフロー

  • **リストの拡張**: mbms_add_member や mbms_add_members は、操作対象のインデックスが現在のリスト長を超えている場合、lappend(a, NULL) を用いてリストを動的に伸長します。
  • **反復処理**: forboth マクロを使用して2つのリストを同時に走査します。リストの長さが異なる場合、短い方に合わせて処理が終了する設計となっています。
  • **インプレース更新**: 多くの関数は、引数として受け取った List *a を直接変更し、結果としてそのポインタを返します。呼び出し側は戻り値でリストを更新する必要があります。

直接依存

  • postgres.h: PostgreSQL の基本定義および elog などのエラー処理機能。
  • nodes/multibitmapset.h: 本モジュールの公開インターフェース定義。

関連コード

  • src/backend/nodes/bitmapset.c: 本モジュールの基礎となる単一ビットマップセットの操作関数群。
  • src/backend/nodes/list.c: List 構造の操作（lappend, list_nth_cell 等）の基盤。

読者向けメモ

  • **空集合の表現**: NIL は空集合を表しますが、リスト内に NULL 要素が含まれる場合があるため、lfirst_node を使用する際は注意が必要です。
  • **インデックスの制約**: 負のインデックスは許可されておらず、エラーとなります。
  • **設計思想**: 現在は必要最小限の機能のみが実装されています。将来的に Bitmapset で利用可能な他の集合演算が必要になった場合、同様のパターンで拡張することが想定されています。
