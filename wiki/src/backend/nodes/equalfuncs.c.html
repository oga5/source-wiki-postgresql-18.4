src/backend/nodes/equalfuncs.c

概要

PostgreSQLの内部データ構造（ノードツリー）が等価であるかを判定するための比較関数群を定義するファイルです。クエリ最適化やプランナにおいて、式やプランノードが同一であるかを検証するために使用されます。

モジュール内での役割

src/backend/nodes サブシステムの一部として、ノードの構造的同一性を保証する役割を担います。copyfuncs.c がノードの複製を担当するのに対し、本ファイルは比較判定のロジックを集中管理します。

責務

  • ノードツリーの再帰的な比較判定。
  • フィールド型に応じた比較マクロの提供（スカラー、ノード、文字列、ビットマップセット等）。
  • 特定のノード型（List, Const, ExtensibleNode 等）に対する特殊な比較ロジックの実装。
  • パース位置（location）など、比較対象外とすべきフィールドの除外管理。

重要なシンボル

  • equal(a, b): 外部から呼び出されるメインの比較関数。ノードの型チェックを行い、適切な比較処理へディスパッチします。
  • COMPARE_NODE_FIELD(fldname): 子ノードを再帰的に比較するためのマクロ。
  • COMPARE_SCALAR_FIELD(fldname): 整数や列挙型などの単純な値を比較するマクロ。
  • _equalList: List 型（T_List, T_IntList 等）専用の比較関数。

追加の索引済みシンボル

  • equal

制御フローとデータフロー

1. **エントリポイント**: equal() が呼び出されると、まずポインタの同一性や NULL チェックを行い、次に nodeTag() で型の一致を確認します。 2. **ディスパッチ**: switch 文により、ノード型に応じた比較関数へ処理を委譲します。

  • 大半のノード型は equalfuncs.switch.c に自動生成されたコードで処理されます。
  • List や Const などの特殊な型は、本ファイル内の専用関数（_equalList 等）で処理されます。

    • 3. **再帰的比較**: COMPARE_NODE_FIELD マクロを通じて、ツリー構造の深部まで再帰的に equal() が呼び出されます。 4. **終了判定**: 途中で不一致が見つかれば即座に false を返し、最後まで一致すれば true を返します。

直接依存

  • postgres.h: 基本的な型定義とマクロ。
  • miscadmin.h: スタック深さチェック（check_stack_depth）用。
  • utils/datum.h: datumIsEqual によるデータ値の比較用。
  • equalfuncs.funcs.c / equalfuncs.switch.c: 自動生成された比較ロジックの断片。

関連コード

  • src/backend/nodes/copyfuncs.c: ノードの複製を行う対照的なモジュール。
  • src/include/nodes/nodes.h: ノードの型定義（NodeTag）の基盤。
  • src/backend/nodes/nodeFuncs.c: ノードツリーを走査する汎用的な関数群。

読者向けメモ

  • **パース位置の無視**: ノード内の location フィールドは比較されません。これは、クエリ内の異なる場所にある同一の式を「等価」とみなすためです。
  • **マクロの規約**: 比較関数内で使用するマクロは、ローカル変数として a と b が存在することを前提としています。
  • **スタック保護**: 再帰的なツリー構造を扱うため、check_stack_depth() を呼び出し、過度に複雑なクエリによるスタックオーバーフローを防止しています。
