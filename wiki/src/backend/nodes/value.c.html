src/backend/nodes/value.c

概要

value.c は、PostgreSQL のパーサーやレクサーで使用される「値ノード（Value Nodes）」を生成するためのファクトリ関数群を実装したファイルです。

モジュール内での役割

src/backend/nodes サブシステムの一部として、構文解析過程で出現するリテラル（整数、浮動小数点数、文字列、真偽値、ビット列）をメモリ上のノード構造として表現する役割を担います。

責務

  • 各種値ノード（Integer, Float, String 等）のメモリ確保と初期化。
  • makeNode() マクロを用いた PostgreSQL のノードシステムへの適合。
  • 呼び出し元から渡された文字列データ（Float, String, BitString 用）の所有権管理（呼び出し元が palloc したメモリをノードに割り当てる）。

重要なシンボル

  • makeInteger(int i): 整数値を保持するノードを作成。
  • makeFloat(char *numericStr): 数値文字列を保持するノードを作成。
  • makeString(char *str): 文字列を保持するノードを作成。
  • makeBoolean(bool val): 真偽値を保持するノードを作成。
  • makeBitString(char *str): ビット列文字列を保持するノードを作成。

追加の索引済みシンボル

  • makeBitString
  • makeBoolean
  • makeFloat
  • makeInteger
  • makeString

制御フローとデータフロー

1. 呼び出し元が特定の make* 関数を呼び出す。 2. makeNode() マクロにより、PostgreSQL のメモリコンテキスト上で該当する構造体のメモリが確保される。 3. 引数として渡された値（または文字列ポインタ）が、確保された構造体のメンバに代入される。 4. 初期化済みのノードポインタが返却される。

直接依存

  • postgres.h: PostgreSQL の基本定義およびメモリ管理機能（palloc 等）へのアクセス。
  • nodes/value.h: ノード構造体の定義および型宣言。

関連コード

  • src/include/nodes/value.h: ノード構造体の定義と、値を取り出すためのマクロ（intVal, strVal 等）が定義されています。
  • src/backend/nodes/makefuncs.c: 他の種類のノードを生成する関数群が含まれており、設計思想が共通しています。

読者向けメモ

  • **メモリ管理**: Float, String, BitString の生成時に渡す文字列は、呼び出し元で palloc されている必要があります。これらはノードの寿命と同期して管理されることが期待されます。
  • **Float の表現**: Float ノードは内部的に double ではなく文字列として保持されます。これは、パーサー段階での精度保持と、後の NUMERIC 型への変換を見越した設計です。
  • **歴史的経緯**: かつては Value という単一の構造体がこれらすべてを包含していましたが、現在は型安全性のために個別の構造体として定義されています。
