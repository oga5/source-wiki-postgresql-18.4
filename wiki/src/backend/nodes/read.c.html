src/backend/nodes/read.c

概要

PostgreSQLの内部ノード構造（Nodeツリー）を、シリアライズされた文字列形式からメモリ上のオブジェクトへ復元するためのパーサーです。主にクエリプランのデシリアライズやデバッグ用途で使用されます。

モジュール内での役割

src/backend/nodes サブシステムにおいて、outfuncs.c が生成したテキスト表現を読み込み、元のデータ構造を再構築する役割を担います。readfuncs.c と連携し、個別のノード型に応じた具体的な解析処理を委譲します。

責務

  • **字句解析**: pg_strtok を用いて、文字列からトークン（括弧、ブレース、値など）を切り出す。
  • **構文解析**: nodeRead を中心とした再帰的な処理により、ノードツリーを構築する。
  • **型判定**: nodeTokenType により、トークンが整数、文字列、リスト、あるいは特殊な構造（{} や ()）かを識別する。
  • **エスケープ処理**: debackslash を用いて、シリアライズ時に付与されたエスケープ文字を除去する。

重要なシンボル

  • stringToNode: 文字列からノードツリーを生成する公開インターフェース。
  • nodeRead: 再帰的にノードを読み込む中核関数。
  • pg_strtok: 文字列を破壊せずにトークンを切り出す字句解析器。
  • pg_strtok_ptr: 現在の解析位置を保持する静的変数。

追加の索引済みシンボル

  • debackslash
  • nodeTokenType
  • stringToNodeInternal
  • stringToNodeWithLocations

制御フローとデータフロー

1. stringToNode が文字列を受け取り、pg_strtok_ptr を初期化して nodeRead を呼び出す。 2. nodeRead は nodeTokenType でトークンの種類を判定する。 3. トークンが { の場合、parseNodeString（readfuncs.c）を呼び出し、ノードのプロパティを読み込む。 4. トークンが ( の場合、リスト（List）やビットマップセット（Bitmapset）として再帰的に要素を読み込む。 5. 値（整数、文字列等）の場合は、対応する makeXXX 関数でノードを生成する。 6. 最終的に再構築された Node ツリーのルートポインタを返す。

直接依存

  • postgres.h: 基本的な型定義とマクロ。
  • nodes/readfuncs.h: parseNodeString 等のノード固有の読み込み関数。
  • nodes/pg_list.h, nodes/value.h: ノード構造の定義。
  • common/string.h: 文字列操作ユーティリティ。
  • ctype.h
  • nodes/bitmapset.h

関連コード

  • src/backend/nodes/outfuncs.c: 本ファイルの逆処理（ノードから文字列への変換）を行う。
  • src/backend/nodes/readfuncs.c: 各ノード型ごとの具体的な読み込みロジックを保持。
  • src/backend/nodes/makefuncs.c: ノード生成の補助関数群。

読者向けメモ

  • **再入可能性**: pg_strtok_ptr を静的変数として保持しているため、stringToNode は再帰呼び出し可能ですが、マルチスレッド環境での同時実行には注意が必要です（PostgreSQLのバックエンドプロセスモデルでは通常問題になりません）。
  • **エスケープ**: pg_strtok はバックスラッシュをそのまま保持します。値の復元時には必ず debackslash を通す必要がある点に注意してください。
  • **デバッグ**: DEBUG_NODE_TESTS_ENABLED が有効な場合、ノードのソース位置情報を復元する機能が追加されます。
