src/backend/nodes/print.c

概要

src/backend/nodes/print.c は、PostgreSQL の内部データ構造（Node）を人間が読みやすい形式で出力するためのデバッグ用ユーティリティ群を提供します。主にバックエンド開発者がクエリツリーやプランツリーの構造を調査する際に使用されます。

モジュール内での役割

src/backend/nodes モジュールにおいて、このファイルは「可視化」の役割を担います。outfuncs.c がノードをシリアライズ可能な文字列に変換するのに対し、本ファイルはそれらの文字列を整形（pretty-print）したり、特定のノード型（RangeTable, TargetList, PathKeys 等）に特化した読みやすい形式で標準出力やログに出力します。

責務

  • **汎用ノードのダンプ**: nodeToString で生成された文字列を、端末表示に適した幅で折り返す。
  • **整形出力**: インデントや改行を挿入し、階層構造を視覚的に分かりやすくする。
  • **型特化型の表示**: RangeTblEntry や TargetEntry など、特定のノード構造を解釈して人間が理解できる形式で出力する。
  • **デバッグログへの統合**: ereport を介して、PostgreSQL のログシステムへ直接デバッグ情報を出力する。

重要なシンボル

  • print(const void *obj): ノードを標準出力にダンプする最も基本的な関数。
  • pprint(const void *obj): ノードを整形して標準出力にダンプする。
  • elog_node_display(...): ログレベルを指定して、ノードをログファイルに出力する。
  • pretty_format_node_dump(const char *dump): 文字列をインデント付きで整形するコアロジック。
  • print_rt, print_expr, print_tl: それぞれ RangeTable, 式, TargetList を人間が読める形式で出力する関数。

制御フローとデータフロー

1. **文字列化**: nodeToStringWithLocations を呼び出し、対象ノードをシリアライズされた文字列に変換します。 2. **整形**: format_node_dump または pretty_format_node_dump に文字列を渡し、StringInfo を用いて改行やインデントを挿入した新しい文字列を生成します。 3. **出力**: 生成された文字列を printf または ereport を通じて出力し、最後に pfree でメモリを解放します。 4. **特化出力**: print_expr 等の関数は、ノードの型を IsA マクロで判定し、再帰的に構造を辿りながら情報を表示します。

直接依存

  • postgres.h: 基本的な型定義とマクロ。
  • lib/stringinfo.h: 整形済み文字列を構築するためのバッファ管理。
  • nodes/nodeFuncs.h: ノードの型判定（IsA）や走査。
  • nodes/pathnodes.h, parser/parsetree.h: 表示対象となるノード構造体の定義。
  • utils/lsyscache.h: 式の表示に必要なカタログ情報（関数名や演算子名の取得）。
  • access/printtup.h
  • nodes/print.h

関連コード

  • src/backend/nodes/outfuncs.c: ノードを文字列に変換する基盤。本ファイルが依存する nodeToString の実装元。
  • src/include/fe_utils/print.h: フロントエンド（psql 等）向けの出力ユーティリティ。名前は似ているが、目的とコンテキストが異なる。

読者向けメモ

  • 本ファイル内の関数は、主にデバッグ目的で設計されています。本番環境のコードから呼び出すことは推奨されません。
  • pretty_format_node_dump は、{, }, (, ) などの記号をヒューリスティックに解析してインデントを決定します。複雑なノード構造では完璧な整形にならない場合があります。
  • print_expr 等の関数は、rtable（Range Table）を引数に取ることで、Var ノードの参照先テーブル名や属性名を解決して表示します。これらを使用する際は、適切なコンテキストの rtable を渡す必要があります。
