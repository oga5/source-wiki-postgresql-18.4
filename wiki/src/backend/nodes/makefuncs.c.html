src/backend/nodes/makefuncs.c

概要

PostgreSQLのクエリ解析および計画フェーズで使用される、各種ノード構造体の生成（コンストラクタ）関数群を定義するファイルです。

モジュール内での役割

src/backend/nodes サブシステムにおいて、ノードのメモリ確保と初期化を抽象化する役割を担います。手動でノード構造体を初期化する際のコード重複を排除し、一貫性のあるデフォルト値を適用することで、パーサーやプランナーの実装を簡素化します。

責務

  • **ノード生成の抽象化**: makeNode() マクロをラップし、特定のノード型（Var, Const, TargetEntry, BoolExpr 等）を生成する。
  • **デフォルト値の適用**: 頻繁に使用されるフィールドに対し、安全な初期値や「不明（-1）」を示す値を自動設定する。
  • **複雑なノード構築の補助**: 複数のサブノードを組み合わせる必要がある複雑な構造（例: makeWholeRowVar や make_opclause）の構築ロジックをカプセル化する。

重要なシンボル

  • makeVar: Var ノードを生成。varno や varattno などの必須情報を引数に取り、他のフィールドをデフォルト値で埋める。
  • makeConst: Const ノードを生成。Datum のデトースト処理など、定数生成に必要な前処理を内包する。
  • makeTargetEntry: TargetEntry ノードを生成。クエリの射影リスト（TargetList）構築に多用される。
  • make_andclause / make_orclause: BoolExpr を生成し、論理演算の木構造を構築する。
  • makeWholeRowVar: 行全体を参照する Var ノードを生成。RTE（Range Table Entry）の種類に応じた複雑な型解決ロジックを隠蔽する。

制御フローとデータフロー

本ファイルは、主にパーサー（gram.y）やプランナーがクエリツリーを構築する際に呼び出されます。 1. **呼び出し**: 上位モジュールが特定のノード生成関数を呼び出す。 2. **メモリ確保**: makeNode() を通じて CurrentMemoryContext にノード分のメモリを確保。 3. **初期化**: 引数で渡された値を構造体にセットし、残りのフィールドをデフォルト値（NULL や InvalidOid 等）で埋める。 4. **返却**: 構築されたノードへのポインタを返す。

直接依存

  • postgres.h: 基本的な型定義と makeNode マクロ。
  • catalog/pg_class.h, catalog/pg_type.h: システムカタログの定数（RECORDOID 等）。
  • nodes/makefuncs.h: 本ファイルのプロトタイプ宣言。
  • nodes/nodeFuncs.h: ノード操作用ユーティリティ。
  • utils/lsyscache.h: 型やリレーションの属性情報を取得するキャッシュ関数。

関連コード

  • src/include/nodes/parsenodes.h: 生成されるノード構造体の定義。
  • src/backend/parser/gram.y: 本ファイルの関数を多用してパースツリーを構築するパーサー。
  • src/backend/nodes/copyfuncs.c: ノードの複製を行うロジック。

読者向けメモ

  • **デフォルト値の注意**: 多くの関数は「一般的なケース」を想定してデフォルト値を設定しています。特殊なフラグや非標準的なフィールド設定が必要な場合は、本関数を呼び出した後に手動でフィールドを上書きする必要があります。
  • **メモリ管理**: 生成されるノードはすべて CurrentMemoryContext に割り当てられます。呼び出し側は、適切なメモリコンテキストがアクティブであることを確認してください。
  • **簡潔さの追求**: 本ファイルは「頻繁に作成されるノード」に特化しています。あまりに特殊なノード生成ロジックは、本ファイルではなく各機能モジュール内に記述されるべきです。
