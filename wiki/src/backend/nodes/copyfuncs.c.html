src/backend/nodes/copyfuncs.c

概要

PostgreSQLのノード構造体（クエリツリーやプランツリーなど）をメモリ上で複製（ディープコピー）するための機能を提供します。

モジュール内での役割

nodes サブシステムにおいて、メモリ上のノードツリーを複製する際の中心的なディスパッチャーです。copyObject() 関数（copyObjectImpl）を通じて、任意のノード型に応じた適切なコピー処理を呼び出します。

責務

  • ノードツリーの再帰的なディープコピーの実行。
  • フィールドの型に応じたコピー処理（スカラー、ポインタ、文字列、Bitmapset等）の抽象化。
  • ExtensibleNode や Const といった特殊なノードに対する個別コピーロジックの提供。
  • スタックオーバーフロー防止のための再帰深度チェック。

重要なシンボル

  • copyObjectImpl: 外部から呼び出されるコピーの起点。
  • COPY_NODE_FIELD: 子ノードを再帰的にコピーするためのマクロ。
  • COPY_SCALAR_FIELD: 値を直接コピーするマクロ。
  • _copyExtensibleNode: 拡張可能なノード型を扱うための専用コピー関数。

制御フローとデータフロー

1. copyObjectImpl が呼び出されると、nodeTag でノードの種類を判定します。 2. copyfuncs.switch.c に記述された switch 文により、各ノード型に対応する _copy... 関数へディスパッチされます。 3. 各 _copy 関数内では、COPY_* マクロを使用して構造体の各フィールドを新しいメモリ領域へコピーします。 4. 子ノードが含まれる場合、COPY_NODE_FIELD が copyObjectImpl を再帰的に呼び出し、ツリー全体を複製します。 5. 最後に、複製されたノードのポインタが呼び出し元に返されます。

直接依存

  • postgres.h: 基本的な型定義とマクロ。
  • miscadmin.h: check_stack_depth() によるスタック保護。
  • utils/datum.h: datumCopy() によるデータムのコピー。
  • copyfuncs.funcs.c, copyfuncs.switch.c: 自動生成または分割されたコピーロジックの断片。

関連コード

  • src/include/nodes/nodes.h: ノードタグと構造体の定義。
  • src/backend/nodes/equalfuncs.c: ノードの比較を行う対となるモジュール。
  • src/backend/nodes/outfuncs.c: ノードを文字列化するモジュール。

読者向けメモ

  • 本ファイルは多くの定型的なコピー処理を含むため、copyfuncs.funcs.c や copyfuncs.switch.c といった生成ファイルにロジックが分割されています。
  • 新しいノード型を追加した際は、これらの生成ファイル側で対応が必要になるケースがほとんどです。
  • COPY_NODE_FIELD は再帰的なコピーを行うため、循環参照を持つデータ構造には使用できません。
