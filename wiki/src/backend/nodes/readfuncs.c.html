src/backend/nodes/readfuncs.c

概要

readfuncs.c は、PostgreSQL の内部ノード構造体（Node ツリー）をシリアライズされた文字列形式からメモリ上のオブジェクトへ復元するためのデシリアライザです。主に stringToNode インターフェースの一部として機能します。

モジュール内での役割

src/backend/nodes モジュールにおいて、outfuncs.c がノードを文字列に変換する役割を担うのに対し、本ファイルは対となる逆変換を担当します。read.c が提供する字句解析器（pg_strtok）を利用して、文字列からノードのフィールドを再構築します。

責務

  • ノードタイプに応じた _read<NodeName> 関数の提供。
  • シリアライズされた文字列から、整数、文字列、ビットマップセット、およびネストされたノードの復元。
  • 拡張可能なノード（ExtensibleNode）のデシリアライズ処理の委譲。
  • デバッグ用位置情報（location）の制御（通常は破棄し -1 に設定）。

重要なシンボル

  • parseNodeString(): 文字列からノードツリーを生成するメインエントリーポイント。
  • READ_LOCALS(nodeTypeName): 各ノード読み込み関数で必要なローカル変数（local_node, token, length）を定義するマクロ。
  • READ_NODE_FIELD(fldname): ネストされたノードを再帰的に読み込むためのマクロ。
  • READ_INT_FIELD 等の READ_*_FIELD マクロ群: 型に応じたフィールドの読み込みと代入を簡略化する一連のヘルパー。
  • readDatum(): Const ノード内の Datum を復元する特殊な関数。

制御フローとデータフロー

1. parseNodeString() が呼び出されると、pg_strtok を用いてノードの型名を取得します。 2. readfuncs.switch.c 内の switch 文により、対応する _read<NodeName> 関数へディスパッチされます。 3. 各 _read 関数は READ_LOCALS でノードを初期化し、READ_*_FIELD マクロを使用してシリアライズされた文字列から順次フィールドを読み取ります。 4. ネストされたノードや配列がある場合、再帰的に nodeRead() や専用の read*Cols 関数を呼び出し、ツリー構造を再構築します。 5. 最後に READ_DONE() マクロで復元されたノードポインタを返します。

読者向けメモ

  • **マクロの規約**: READ_* マクロは、関数内で local_node という名前の変数が定義されていることを前提としています。新しい読み込み関数を追加する際は、必ず READ_LOCALS を使用してください。
  • **位置情報**: location フィールドは通常 -1 にリセットされます。これは、保存されたルールやクエリツリーが現在のクエリテキストと一致しない可能性があるためです。デバッグ目的で保持したい場合は restore_location_fields フラグを確認してください。
  • **拡張性**: ExtensibleNode を扱う際は、GetExtensibleNodeMethods を介して各拡張が定義した nodeRead メソッドが呼び出される点に注意してください。
