src/backend/nodes/nodeFuncs.c

概要

nodeFuncs.c は、PostgreSQL のクエリ解析およびプランニングフェーズにおいて、ノードツリー（式ツリーやプランツリー）に対する汎用的な操作を提供する中心的なモジュールです。ノードの型解決、属性（型・照合順序・位置）の取得・設定、およびツリーの再帰的な走査・変形を行うための基盤機能を提供します。

モジュール内での役割

src/backend/nodes サブシステムにおいて、ノードの構造を理解し、クエリツリーを横断的に操作するための「ユーティリティ層」として機能します。copyfuncs.c や equalfuncs.c がノードのコピーや比較という「静的」な操作を担うのに対し、本ファイルは「意味論的」な操作（型解決やツリーの変形）を担います。

責務

  • **式属性の解決**: 式ツリーの戻り値の型（Oid）、型修飾子（typmod）、照合順序（collation）の抽出と設定。
  • **ツリー走査・変形**: expression_tree_walker / mutator を用いた、式ツリーの再帰的な探索およびノードの置換・修正。
  • **構文解析支援**: エラー報告のための式位置（location）の特定や、暗黙的な型変換の除去。
  • **プラン実行支援**: fix_opfuncids による演算子関数 ID の解決や、プラン状態（PlanState）ツリーの走査。

重要なシンボル

属性解決ファミリー

  • exprType(Node*), exprTypmod(Node*): 式の戻り値の型と修飾子を再帰的に解決。
  • exprCollation(Node*), exprSetCollation(Node*, Oid): 照合順序の取得と設定。
  • exprLocation(Node*): エラー報告用に、式ツリー内の最も左にあるトークンの位置を特定。

ツリー走査・変形（Walker/Mutator）

  • expression_tree_walker_impl: 式ツリーを読み取り専用で走査する汎用エンジン。
  • expression_tree_mutator_impl: 式ツリーを走査しつつ、ノードを置換・修正して新しいツリーを構築するエンジン。
  • query_tree_walker / mutator: Query ノードを起点として、ターゲットリストやジョインツリーを網羅的に走査・変形。

変換・最適化支援

  • applyRelabelType: 式に RelabelType ノードを付与し、型や照合順序を強制的に変更。
  • strip_implicit_coercions: 解析時に挿入された暗黙的な型変換ノードを除去。
  • fix_opfuncids: ツリー内の全演算子ノードに対して、演算子 ID から関数 ID を解決してキャッシュ。

制御フローとデータフロー

  • **走査フロー**: expression_tree_walker は、ノードのタグ（nodeTag）に基づいて switch 文で分岐し、子ノード（List や Expr）に対して再帰的に自身を呼び出します。
  • **変形フロー**: expression_tree_mutator は、ノードのコピーを作成（FLATCOPY）し、その子ノードに対して再帰的に mutator を適用することで、元のツリーを破壊せずに新しいツリーを構築します。
  • **スタック保護**: 再帰的なツリー操作が深くなりすぎないよう、各走査関数内で check_stack_depth() を呼び出し、スタックオーバーフローを防止しています。

読者向けメモ

  • **メモリ管理**: mutator は新しいノードを palloc で割り当てます。呼び出し元は、返されたツリーのメモリコンテキストが適切であることを確認する必要があります。
  • **不変性**: walker はツリーを読み取るだけですが、mutator は新しいノードを生成するため、元のツリーを保持したい場合は mutator の結果を適切に管理してください。
  • **網羅性**: 新しいノード型を nodes/nodes.h に追加した場合は、本ファイルの walker および mutator の switch 文にも必ずケースを追加する必要があります。追加漏れは elog(ERROR) で検出されます。
  • **省略箇所**: raw_expression_tree_walker は、解析前の生のパースツリーを扱うための特殊な走査器であり、主に CTE の解析などで使用されます。通常のプランニング用走査とは対象ノードが異なる点に注意してください。
