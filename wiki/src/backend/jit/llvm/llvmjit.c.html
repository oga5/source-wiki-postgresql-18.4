src/backend/jit/llvm/llvmjit.c

概要

PostgreSQL の LLVM JIT プロバイダーのコア実装です。LLVM のコンテキスト管理、JIT コンパイルのライフサイクル制御、および PostgreSQL の型システムと LLVM IR 間のブリッジ機能を提供します。

モジュール内での役割

src/backend/jit/llvm モジュールの中心的なエントリポイントです。JIT コンパイルの初期化、最適化レベルの管理、および生成されたコードの実行エンジン（ORC JIT）への登録を担います。

責務

  • **セッション管理**: LLVM のターゲットマシン、スレッドセーフコンテキスト、および JIT インスタンスの初期化と終了処理。
  • **コンテキスト管理**: LLVMJitContext の生成と破棄。リソースオーナー（ResourceOwner）と連携し、トランザクション終了時のメモリリークを防止。
  • **型情報の同期**: llvmjit_types.bc から PostgreSQL のデータ構造定義を読み込み、LLVM IR 上で利用可能な型としてキャッシュ。
  • **コード生成と最適化**: モジュールの最適化（Pass Manager の制御）および ORC JIT を介した実行可能コードへの変換。

重要なシンボル

  • LLVMJitContext: JIT コンパイル状態を保持する構造体。
  • llvm_create_context(): JIT コンテキストの初期化関数。
  • llvm_compile_module(): IR モジュールの最適化と JIT への登録を行うメインフロー。
  • llvm_get_function(): JIT コンパイル済みの関数ポインタを取得する関数。
  • llvm_create_types(): C の構造体定義を LLVM 型へマッピングする初期化処理。

制御フローとデータフロー

1. **初期化**: _PG_jit_provider_init が呼ばれ、JIT プロバイダーが登録されます。初回利用時に llvm_session_initialize が実行され、ターゲットマシンと ORC JIT インスタンスが構築されます。 2. **コンテキスト生成**: llvm_create_context が呼ばれると、LLVMJitContext が割り当てられ、リソースオーナーに登録されます。 3. **コンパイル**: llvm_compile_module が呼び出されると、以下の順序で処理が進みます。

  • インライン化（llvm_inline）
  • 最適化（llvm_optimize_module）
  • ORC JIT へのモジュール登録（LLVMOrcLLJITAddLLVMIRModuleWithRT）

    • 4. **実行**: llvm_get_function を通じて、JIT された関数アドレスが解決されます。

読者向けメモ

  • **メモリ管理**: LLVM のコンテキストはメモリ消費が大きいため、LLVMJIT_LLVM_CONTEXT_REUSE_MAX に達すると定期的にリセットされます。
  • **エラーハンドリング**: LLVM 内部でのエラーは elog(ERROR) を通じて PostgreSQL のエラー処理系に伝播しますが、FATAL エラー発生時の再入には注意が必要です。
  • **デバッグ**: jit_dump_bitcode を有効にすると、最適化前後のビットコードがファイルに出力され、生成された IR の確認が可能です。
