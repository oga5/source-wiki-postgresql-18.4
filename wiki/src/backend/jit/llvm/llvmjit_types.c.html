src/backend/jit/llvm/llvmjit_types.c

概要

本ファイルは、PostgreSQL の JIT コンパイル（LLVM）において、C 言語のデータ構造や関数シグネチャを LLVM IR 上で再現するための「型定義のカタログ」です。

モジュール内での役割

JIT コンパイル時に、PostgreSQL の実行エンジンが使用する構造体（TupleTableSlot や ExprState など）のレイアウトを LLVM に認識させるためのブリッジとして機能します。

責務

  • **型情報の提供**: LLVM が生成するコード内で使用する C 構造体や関数の型情報を、ビットコード経由で提供する。
  • **シグネチャの保持**: コンパイル時の属性（インライン化の可否など）を維持するためのテンプレート関数を提供する。
  • **シンボルのエクスポート**: LLVM が参照すべき関数ポインタを referenced_functions 配列に集約し、最適化による削除を防ぐ。

重要なシンボル

  • Struct* 変数群: LLVM が構造体のメモリレイアウトを把握するために使用するダミー変数。
  • AttributeTemplate: 関数属性（呼び出し規約や最適化ヒント）を LLVM に伝えるためのテンプレート。
  • referenced_functions: JIT 実行時に動的にリンクされる可能性のある関数群のリスト。

追加の索引済みシンボル

  • ExecEvalBoolSubroutineTemplate
  • ExecEvalSubroutineTemplate
  • FunctionReturningBool

制御フローとデータフロー

本ファイルは通常のサーバープロセスにはリンクされません。 1. **ビルド時**: Clang によってビットコード（.bc）に変換されます。 2. **初期化時**: LLVM JIT エンジンがこのビットコードをロードします。 3. **実行時**: LLVM はロードされたビットコードを解析し、Struct* 変数から構造体のオフセットを、referenced_functions から関数アドレスを特定して、JIT コンパイルされたコードに埋め込みます。

読者向けメモ

  • **注意**: 本ファイル内の変数は static にしてはいけません。static にすると Clang が不要なコードと判断し、ビットコードから削除してしまうためです。
  • **読み方**: 新しい構造体を JIT で扱いたい場合は、まずこのファイルに Struct 変数を追加し、LLVM 側でその型を認識させる必要があります。
  • **役割の特殊性**: このファイルは「実行されるコード」ではなく「LLVM に型情報を教えるためのメタデータ」であることを理解してください。
