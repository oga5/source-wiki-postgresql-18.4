src/backend/jit/llvm/llvmjit_wrap.cpp

概要

本ファイルは、PostgreSQLのJITコンパイル基盤において、LLVMのC APIでは直接提供されていない、あるいは不足している機能を補完するためのC++ラッパー層です。主にC言語で記述されたPostgreSQLのJITエンジンから、LLVMのC++ APIへアクセスするためのブリッジを提供します。

モジュール内での役割

src/backend/jit/llvm モジュールにおいて、C言語ベースのJIT実行系と、LLVMの強力なC++ APIとの間の「接着剤」として機能します。特に、LLVMの型システムへのアクセスや、JIT実行時のメモリ管理層の構築を支援します。

責務

  • **C APIの拡張**: LLVMのC APIに含まれない、関数型や戻り値型を取得するためのヘルパー関数の提供。
  • **メモリ管理層の構築**: USE_LLVM_BACKPORT_SECTION_MEMORY_MANAGER が有効な環境において、安全なメモリ管理を行う RTDyldObjectLinkingLayer の生成と初期化。
  • **型変換の抽象化**: llvm::wrap / llvm::unwrap を用いた、C APIの不透明ポインタとLLVM C++オブジェクト間の相互変換の管理。

重要なシンボル

  • LLVMGetFunctionReturnType: LLVMValueRef から戻り値の型を抽出するラッパー。
  • LLVMGetFunctionType: LLVMValueRef から関数型全体を抽出するラッパー。
  • LLVMOrcCreateRTDyldObjectLinkingLayerWithSafeSectionMemoryManager: JIT実行時にメモリ保護を考慮したカスタムメモリマネージャを組み込んだオブジェクト層を生成する関数。

制御フローとデータフロー

1. **型情報の抽出**: JITコンパイルされた関数のメタデータが必要な際、PostgreSQLのCコードからこれらのラッパー関数が呼び出されます。内部で llvm::unwrap を使用してLLVMの Function オブジェクトにキャストし、型情報を取得後、llvm::wrap でC API形式に戻して返却します。 2. **メモリ層の初期化**: JITエンジンが実行環境を構築する際、LLVMOrcCreateRTDyldObjectLinkingLayerWithSafeSectionMemoryManager が呼ばれます。ここでは llvm::backport::SectionMemoryManager を使用したラムダ式が RTDyldObjectLinkingLayer に渡され、JITコードのメモリ配置が制御されます。

直接依存

  • postgres.h: PostgreSQLの基本定義。
  • llvm-c/Core.h: LLVM C APIの定義。
  • llvm/IR/Function.h: LLVMの関数表現（C++）。
  • jit/llvmjit.h / jit/llvmjit_backport.h: JIT共通ヘッダー。
  • llvm/ExecutionEngine/Orc/RTDyldObjectLinkingLayer.h: JIT実行時のオブジェクトリンク層。
  • jit/SectionMemoryManager.h: PostgreSQL独自のメモリ管理実装。
  • llvm/ExecutionEngine/SectionMemoryManager.h
  • llvm/Support/CBindingWrapping.h

関連コード

  • src/backend/jit/llvm/SectionMemoryManager.cpp: 本ファイルが利用するメモリ管理の実装詳細。
  • src/backend/jit/llvm/llvmjit.c: JITのメインエントリーポイント。本ファイルのラッパー関数を呼び出す側。

読者向けメモ

  • 本ファイルは「LLVMのバージョン差異」や「C APIの制限」を吸収する場所です。LLVMのバージョンアップに伴い、LLVM_VERSION_MAJOR による条件分岐が増える傾向にあります。
  • DEFINE_SIMPLE_CONVERSION_FUNCTIONS マクロは、LLVMのC APIとC++ APIを繋ぐ定型的な変換関数を生成します。新しい型をC APIに公開する際は、このマクロの利用を検討してください。
  • 非常に限定的な機能しか持たないため、複雑なロジックはここではなく、llvmjit.c や llvmjit_expr.c に記述するのが適切です。
