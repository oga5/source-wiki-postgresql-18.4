src/backend/jit/llvm/llvmjit_deform.c

概要

タプル（行データ）を物理的なメモリレイアウトから TupleTableSlot 形式へ変換（デフォーム）するための LLVM IR コードを生成するモジュールです。

モジュール内での役割

src/backend/jit/llvm サブシステムの一部として、実行時のタプルアクセスを高速化します。コンパイル時にタプルの記述子（TupleDesc）を知ることで、固定幅カラムのオフセット計算や NULL チェックを最適化し、汎用的なデフォーム処理を排除します。

責務

  • 指定された TupleDesc に基づき、タプルをデフォームする専用の LLVM 関数を生成する。
  • 固定幅カラムの配置や NOT NULL 制約を静的に解析し、不要な分岐や計算を排除したコードを構築する。
  • 可変長データ（varlena）や NULL ビットマップの処理を適切にインライン化する。

重要なシンボル

  • slot_compile_deform: 本ファイルのメイン関数。タプルデフォーム用の LLVM 関数を生成し、その LLVMValueRef を返す。

制御フローとデータフロー

1. **初期化**: TupleTableSlot の型（Heap, Minimal 等）を確認し、対象外であれば NULL を返す。 2. **静的解析**: TupleDesc を走査し、どのカラムが NOT NULL かを判定して、オフセット計算の最適化戦略を決定する。 3. **コード生成**:

  • entry ブロックでタプルヘッダや NULL ビットマップへのポインタを取得。
  • カラムごとに attcheckattnoblocks から attstoreblocks までの基本ブロックを生成。
  • switch 命令を用いて、現在の nvalid（既にデフォーム済みのカラム数）から適切な開始位置へジャンプさせる。

    • 4. **ループ処理**: 各カラムに対して、NULL チェック、アライメント調整、データコピー、オフセット更新のコードを順次生成する。 5. **終了**: 最終的な nvalid とオフセットをスロットに書き戻し、関数を終了する。

直接依存

  • llvm-c/Core.h: LLVM IR 生成用 API。
  • access/htup_details.h, access/tupdesc_details.h: タプル構造と記述子の定義。
  • jit/llvmjit.h, jit/llvmjit_emit.h: JIT コンパイルの共通基盤および IR 生成ヘルパー。
  • executor/tuptable.h
  • postgres.h

関連コード

  • src/backend/executor/execTuples.c: 本関数が生成したコードが操作する TupleTableSlot の標準的な実装。
  • src/backend/jit/llvm/llvmjit_expr.c: 式評価の JIT コンパイルを担当する関連モジュール。

読者向けメモ

  • **最適化の鍵**: このコードは「コンパイル時にカラムのオフセットが確定できるか」を重視しています。attguaranteedalign フラグが true の場合、LLVM は定数オフセットを使用してロード命令を生成できるため、非常に高速です。
  • **デバッグ**: 生成されたコードは llvm_mutable_module を通じてモジュールに追加されます。JIT されたコードの挙動を追う場合は、jit_debugging_support を有効にして LLVM IR をダンプすることをお勧めします。
  • **注意点**: 仮想タプル（TTSOpsVirtual）はデフォームが不要なため、この関数は早期リターンします。
