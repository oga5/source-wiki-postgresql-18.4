src/backend/optimizer/path/joinpath.c

概要

本ファイルは、PostgreSQL プランナにおいて、2つのリレーションを結合する際の物理的な実行パス（NestLoop, MergeJoin, HashJoin）を生成・評価する中核モジュールです。結合順序の探索過程で呼び出され、コストモデルに基づいた最適な結合手法の選定と、それらのパスを RelOptInfo に登録する責務を負います。

モジュール内での役割

src/backend/optimizer/path サブシステムの一部として、joinrels.c が決定した結合順序に基づき、具体的な実行アルゴリズムを適用する役割を担います。allpaths.c からの要求を受け、costsize.c でコストを算出し、pathnode.c でパスノードを生成します。

責務

  • **結合パスの網羅的生成**: NestLoop, MergeJoin, HashJoin の各アルゴリズムについて、可能なパスを生成。
  • **コストベースのフィルタリング**: add_path / add_partial_path を通じて、コスト的に劣るパスを早期に排除。
  • **制約の検証**: 結合順序制約（SpecialJoinInfo）やパラメータ化（ParamPathInfo）の整合性チェック。
  • **特殊な結合の処理**: JOIN_UNIQUE_INNER / OUTER などの内部的な結合型の正規化と、それらに伴う一意性保証パスの生成。
  • **拡張性**: set_join_pathlist_hook を通じた外部プラグインによるパス操作の許可。

重要なシンボル

結合パス生成のメインエントリ

  • add_paths_to_joinrel: 結合リレーションに対するパス生成の統括関数。

結合アルゴリズム別生成関数

  • try_nestloop_path / try_partial_nestloop_path: NestLoop パスの生成と登録。
  • try_mergejoin_path / try_partial_mergejoin_path: MergeJoin パスの生成と登録。
  • try_hashjoin_path / try_partial_hashjoin_path: HashJoin パスの生成と登録。

補助・最適化関数

  • get_memoize_path: 結合パス上に Memoize ノードを付与可能か判定し生成。
  • select_mergejoin_clauses: MergeJoin に利用可能な条件式を抽出。
  • allow_star_schema_join: スター型スキーマにおけるパラメータ化パスの制約緩和判定。
  • extract_lateral_vars_from_PHVs: LATERAL 参照をキャッシュキーとして抽出。

制御フローとデータフロー

1. **初期化**: add_paths_to_joinrel が JoinPathExtraData を初期化し、結合のユニーク性やマージ条件を特定。 2. **パス探索**:

  • sort_inner_and_outer: MergeJoin 用にソート順序を考慮したパスを生成。
  • match_unsorted_outer: 既存の順序を活かした NestLoop/MergeJoin を生成。
  • hash_inner_and_outer: ハッシュ結合可能な条件を特定し HashJoin を生成。

    • 3. **コスト評価**: 各 try_*_path 関数内で initial_cost_* を呼び出し、コストの下限値を算出。 4. **登録**: add_path / add_partial_path により、コストが優位なパスのみを RelOptInfo に保持。

直接依存

  • optimizer/pathnode.h: パスノード生成インターフェース。
  • optimizer/cost.h: コスト見積もりモデル。
  • optimizer/restrictinfo.h: 結合条件のメタデータ管理。
  • optimizer/optimizer.h: プランナのグローバル設定。

関連コード

  • joinrels.c: 結合順序の探索制御。
  • pathkeys.c: ソート順序の正規化とマッチング。
  • equivclass.c: 等価条件の推論。
  • src/backend/optimizer/README: 結合パス生成に関する設計思想と制約。

読者向けメモ

  • **メモリ管理**: bms_free を多用し、パス生成の試行段階でメモリリークを防いでいます。
  • **制約**: JOIN_RIGHT_SEMI や JOIN_FULL など、一部の結合型は特定のアルゴリズム（NestLoop等）でサポートされません。
  • **省略された詳細**: 本ファイルはパス生成の「ロジック」に集中しており、実際のコスト計算式（costsize.c）や、プランノード生成（createplan.c）の詳細は含まれていません。
  • **最適化**: add_path_precheck による早期終了は、探索空間が巨大なクエリにおいてプランナの実行時間を抑えるための重要なヒューリスティックです。
