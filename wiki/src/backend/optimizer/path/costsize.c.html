src/backend/optimizer/path/costsize.c

概要

PostgreSQL クエリプランナにおけるコスト見積もりの中心的なモジュールです。物理的な実行パス（Path）の実行コスト（startup_cost, total_cost）およびカーディナリティ（rows）を算出する関数群を提供します。

モジュール内での役割

src/backend/optimizer/path サブシステムの一部として、プランナが生成した各 Path ノードに対して、実行コストモデルに基づいた評価を行います。この評価結果は、add_path() 等で最適な実行計画を選択するための判断基準となります。

責務

  • **コスト計算**: スキャン（Seq, Index, Bitmap, Tid, Subquery, Function 等）、結合（NestLoop, Merge, Hash）、およびソート・集約操作のコスト算出。
  • **カーディナリティ見積もり**: 制限条件や結合条件に基づく行数の推定と、妥当な範囲へのクランプ（clamp_row_est）。
  • **並列実行コストの調整**: 並列ワーカー数やリーダーの参加率を考慮したコストと行数の補正。
  • **メモリ制約の評価**: work_mem を超える場合のディスク I/O コストの加算。

重要なシンボル

コストモデル定数

  • seq_page_cost, random_page_cost: I/O コストの基本単位。
  • cpu_tuple_cost, cpu_index_tuple_cost, cpu_operator_cost: CPU 処理コストの基本単位。
  • parallel_tuple_cost, parallel_setup_cost: 並列実行時のオーバーヘッド。

コスト計算関数（主要なファミリー）

  • **スキャン系**: cost_seqscan, cost_index, cost_bitmap_heap_scan, cost_tidscan。
  • **結合系**: initial_cost_nestloop, final_cost_nestloop, initial_cost_mergejoin, final_cost_mergejoin, initial_cost_hashjoin, final_cost_hashjoin。
  • **操作系**: cost_sort, cost_incremental_sort, cost_agg, cost_windowagg, cost_material。

ユーティリティ

  • clamp_row_est(): 行数見積もりを 1 から MAXIMUM_ROWCOUNT の範囲に収める。
  • cost_qual_eval(): 条件式（WHERE 句等）の評価コストを算出。
  • get_parallel_divisor(): 並列実行時のワーカー数とリーダー参加率から除数を算出。

制御フローとデータフロー

1. **初期化**: プランナが Path を生成する過程で、対応する cost_XXX 関数を呼び出します。 2. **入力**: Path 構造体および関連する RelOptInfo や ParamPathInfo が渡されます。 3. **計算**:

  • ページアクセス数と CPU 処理回数を算出。
  • get_tablespace_page_costs 等で物理デバイスの特性を反映。
  • cost_qual_eval で条件式のコストを加算。

    • 4. **出力**: 計算結果を path->startup_cost, path->total_cost, path->rows に書き込みます。

読者向けメモ

  • **歴史的背景**: 多くの関数が Path を入力として受け取りつつ、内部で必要な情報を個別の引数として要求する設計になっています。これは過去の設計の名残であり、現在のコードベースでは冗長に見える場合があります。
  • **メモリ所有権**: 本モジュールは計算結果を Path 構造体に書き込むだけであり、メモリの割り当てや解放は行いません。
  • **精度**: コストモデルはヒューリスティックな近似であり、特に複雑な結合や並列実行時の見積もりには限界があります。MAXIMUM_ROWCOUNT によるクランプは、double 型のオーバーフローを防ぐための安全策です。
  • **省略された詳細**: cost_rescan 等の内部ヘルパー関数は、特定のプランノードの再スキャンコストを算出するためのものであり、プランナの最適化フェーズで重要な役割を果たしますが、本ページでは主要なコスト計算関数に焦点を当てています。
