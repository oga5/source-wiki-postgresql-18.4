src/backend/optimizer/plan/planner.c

概要

PostgreSQL のクエリ最適化プロセスにおけるメインエントリーポイントを担うファイルです。SQL パーサーが生成した Query ツリーを受け取り、実行エンジンが処理可能な PlannedStmt へ変換するまでのオーケストレーションを行います。

モジュール内での役割

src/backend/optimizer/plan サブシステムの中核であり、プランニングの各フェーズ（前処理、結合順序決定、物理プラン変換、最終化）を呼び出す司令塔です。

責務

  • **プランニングの統括**: planner() および standard_planner() を通じて、クエリのプランニング全体を制御する。
  • **サブクエリの再帰的処理**: subquery_planner() を用いて、クエリツリー内のサブクエリを再帰的にプランニングする。
  • **式の前処理**: preprocess_expression() を通じて、定数畳み込み、演算子解決、サブリンクの変換、相関変数のパラメータ化を行う。
  • **グループ化・集約の最適化**: grouping_planner() を中心に、GROUP BY、HAVING、WINDOW 関数、DISTINCT などの物理パス生成を管理する。
  • **プランの最終化**: set_plan_references() を呼び出し、変数参照の解決やパラメータの確定を行う。

重要なシンボル

エントリーポイント

  • planner(): プランニングの最上位関数。planner_hook をサポートし、プラグインによる介入を可能にする。
  • standard_planner(): プランニングの標準的な実装。グローバル状態の初期化から最終的な PlannedStmt 生成までを行う。
  • subquery_planner(): 特定の Query オブジェクトに対するプランニングのメインロジック。

グループ化・集約関連

  • grouping_planner(): GROUP BY、HAVING、WINDOW、DISTINCT などの処理を物理パスに変換する。
  • create_grouping_paths(): グループ化および集約のためのパスを生成する。
  • create_window_paths(): ウィンドウ関数の評価パスを生成する。
  • create_distinct_paths(): DISTINCT 処理のためのパスを生成する。

式処理・最適化

  • preprocess_expression(): 式ツリーに対して定数畳み込みやサブリンク変換を適用する。
  • find_having_collation_conflicts(): HAVING 句が WHERE 句へ移動可能かを照合順序の観点から判定する。

制御フローとデータフロー

1. **初期化**: standard_planner が PlannerGlobal を作成し、並列クエリの可否を判定する。 2. **再帰的プランニング**: subquery_planner がクエリツリーを走査し、pull_up_subqueries や pull_up_sublinks で論理的な最適化を行う。 3. **式変換**: preprocess_expression により、式が実行可能な形式に正規化される。 4. **パス生成**: grouping_planner が query_planner を呼び出し、スキャン/結合のパスを生成した後、グループ化、ウィンドウ関数、ソート、DISTINCT の順でパスを積み上げる。 5. **最終化**: set_plan_references がプランノード内の変数参照を物理的なインデックスに書き換え、PlannedStmt を完成させる。

読者向けメモ

  • **メモリ管理**: プランニング中に生成されるデータ構造は PlannerInfo->planner_cxt に割り当てられ、プランニング終了後に一括解放される。
  • **フック**: planner_hook はプランニングプロセス全体を置き換える強力なフックであり、拡張機能開発時に注意が必要。
  • **省略された詳細**: 本ファイルは非常に巨大なため、個別の create_*_path 関数の詳細なコスト計算ロジックや、特定の集約最適化（planagg.c への委譲など）の詳細は省略している。これらは各ヘルパー関数の実装を参照すること。
  • **不変条件**: standard_planner は入力された Query ツリーを破壊的に変更するため、再利用する場合はコピーが必要である。
