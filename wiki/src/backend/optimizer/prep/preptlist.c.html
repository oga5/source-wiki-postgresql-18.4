src/backend/optimizer/prep/preptlist.c

概要

本ファイルは、クエリのターゲットリスト（targetList）をプランナが実行可能な形式に変換・補完するためのルーチン群を提供します。SQLのセマンティクスを扱うリライタと異なり、物理的なテーブルアクセスや実行時の行識別子（Junk列）の付与に焦点を当てています。

モジュール内での役割

src/backend/optimizer/prep モジュールの一部として、クエリ最適化の準備段階（Preprocessing）を担当します。パーサから渡されたターゲットリストを、プランナが後続の処理（コスト計算やプラン生成）で利用できるよう、物理的なテーブル定義と整合させる役割を担います。

責務

  • **INSERTの補完**: テーブルの全属性を網羅し、指定のない列にNULLを埋め込む。
  • **UPDATEの正規化**: 更新対象列の番号を抽出し、ターゲットリストを連続した番号に再割り当てする。
  • **Junk列の付与**: UPDATE/DELETE/MERGE や SELECT FOR UPDATE に必要な行識別子（ctid、tableoid、wholerow）をターゲットリストに追加する。
  • **RETURNING句のサポート**: RETURNING 句で使用される変数をプラン実行時に利用可能にする。

重要なシンボル

  • preprocess_targetlist: 本ファイルのメインエントリポイント。クエリタイプに応じたターゲットリストの変換を統括する。
  • expand_insert_targetlist: INSERT文において、テーブル定義順にターゲットリストを整列し、欠落している列にNULLを補完する。
  • extract_update_targetlist_colnos: UPDATE文のターゲットリストから更新対象の列番号を抽出し、リストを再ナンバリングする。
  • get_plan_rowmark: 特定のレンジテーブルインデックスに対応する PlanRowMark を取得する。

制御フローとデータフロー

1. preprocess_targetlist がクエリタイプを判定し、対象リレーションを開く。 2. CMD_INSERT の場合、expand_insert_targetlist を呼び出し、テーブルの全属性を網羅したリストを生成する。 3. CMD_UPDATE 等の場合、extract_update_targetlist_colnos で更新対象列を特定し、リストを正規化する。 4. 必要に応じて、add_row_identity_columns や rowMarks の走査を行い、実行に必要なJunk列（ctid等）を processed_tlist に追加する。 5. 最終的なリストを root->processed_tlist に格納する。

読者向けメモ

  • **歴史的背景**: 本ファイルとリライタの責務分担は歴史的経緯が強く、物理アクセスに近い処理がここに集約されています。
  • **Junk列の扱い**: resjunk = true となっているエントリは、最終的な出力には含まれませんが、プランナやエグゼキュータが内部的に行を特定するために不可欠です。
  • **MERGEの複雑性**: MERGE 文は複数のアクション（INSERT/UPDATE）を持つため、他のコマンドよりもターゲットリストの構築が複雑です。条件式（qual）に含まれる変数も適切に抽出される点に注意してください。
