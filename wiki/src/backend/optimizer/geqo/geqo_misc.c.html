src/backend/optimizer/geqo/geqo_misc.c

概要

本ファイルは、PostgreSQLの遺伝的アルゴリズム（GEQO）オプティマイザにおけるデバッグおよび統計出力のための補助ルーチンを提供します。主に遺伝的アルゴリズムの個体群（Pool）の状態や、エッジテーブルの構造を可視化するために使用されます。

モジュール内での役割

GEQOサブシステムにおいて、アルゴリズムの収束状況や個体群の多様性を追跡するための診断ツールとして機能します。本ファイル内の関数は GEQO_DEBUG マクロが定義されている場合にのみ有効化され、本番環境のパフォーマンスには影響を与えない設計となっています。

責務

  • **個体群の統計計算**: 個体群内の個体の適応度（worth）の平均値を算出する。
  • **個体群の可視化**: 指定した範囲の個体データ（遺伝子配列と適応度）をファイルストリームに出力する。
  • **世代情報の出力**: 特定の世代における最良、最悪、中央値、平均適応度を要約して出力する。
  • **エッジテーブルのダンプ**: 遺伝的アルゴリズムの交叉処理で使用されるエッジテーブル構造の内容を表示する。

重要なシンボル

  • print_pool: 個体群の特定の範囲をダンプし、各個体の遺伝子配列と適応度を表示します。
  • print_gen: 現在の世代番号と、個体群全体の適応度統計（Best/Worst/Mean/Avg）を一行で出力します。
  • print_edge_table: 交叉演算（ERXなど）の内部状態であるエッジテーブルを可視化します。
  • avg_pool: 個体群の平均適応度を計算する内部ヘルパー関数です。

制御フローとデータフロー

  • すべての関数は FILE *fp を引数に取り、外部から指定されたストリーム（通常は stderr やログファイル）へ書き込みを行います。
  • print_gen は内部で avg_pool を呼び出し、個体群の適応度を走査して集計します。
  • print_pool および print_edge_table は、Pool 構造体や Edge 配列を直接参照し、ループ処理によって各要素を整形出力します。
  • 各関数は出力の最後に fflush(fp) を呼び出し、デバッグ出力が即座に反映されることを保証します。

直接依存

  • postgres.h: PostgreSQLの基本定義および elog マクロを提供。
  • optimizer/geqo_misc.h: 本ファイルの関数プロトタイプ定義。

関連コード

  • src/backend/optimizer/geqo/geqo_pool.c: Pool 構造体の定義と管理を行う中心的なファイル。
  • src/backend/optimizer/geqo/geqo_erx.c: Edge 構造体およびエッジテーブルを使用する交叉アルゴリズムの実装。

読者向けメモ

  • 本ファイル内のコードは GEQO_DEBUG が定義されていない限りコンパイルされません。デバッグを行う際は、ビルド設定でこのマクロを有効にする必要があります。
  • avg_pool における平均計算は、オーバーフローを避けるために各要素を個体数で割ってから加算する手法をとっています。精度よりも安全性を優先した実装である点に留意してください。
