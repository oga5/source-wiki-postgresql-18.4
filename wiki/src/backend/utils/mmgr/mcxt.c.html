src/backend/utils/mmgr/mcxt.c

概要

mcxt.c は、PostgreSQL のメモリ管理サブシステムにおける「ディスパッチャー」です。特定のメモリ割り当て戦略（aset, slab 等）に依存しない共通のインターフェースを提供し、実行時に適切なアロケータへ処理を委譲します。メモリコンテキストの階層構造管理、ライフサイクル（作成・リセット・削除）、および統計情報の集計を担う中枢ファイルです。

モジュール内での役割

本ファイルは MemoryContext 構造体と MemoryContextMethods 関数ポインタテーブルを介して、メモリ管理の抽象化層を形成します。各アロケータ実装（aset.c 等）は、このインターフェースに従うことで、呼び出し側（palloc 等）のコードを変更することなく、用途に応じたメモリ戦略を切り替え可能にしています。

責務

  • **コンテキスト管理**: メモリコンテキストの階層（親子関係）の構築、維持、および再帰的な削除・リセット処理。
  • **ディスパッチ**: palloc 等の汎用 API から、各コンテキスト型固有のメソッド（alloc, free, realloc 等）へのルーティング。
  • **安全性と検証**: メモリ割り当て時のクリティカルセクションチェック、Valgrind 連携、および不正なポインタに対するエラーハンドリング。
  • **統計と診断**: コンテキスト階層のトラバース、メモリ消費量の集計、およびデバッグ用統計情報の出力。

重要なシンボル

コンテキスト管理・ディスパッチ

  • mcxt_methods[]: 全アロケータのメソッドを保持する静的テーブル。MCXT_METHOD マクロにより、チャンクヘッダーから取得した ID を基にメソッドを解決します。
  • MemoryContextCreate(): 新規コンテキストの初期化と階層への登録を行う共通関数。
  • MemoryContextTraverseNext(): 再帰を避けてコンテキスト階層を走査するためのヘルパー。

グローバルコンテキスト

  • CurrentMemoryContext: 現在の割り当て先となるデフォルトコンテキスト。
  • TopMemoryContext, ErrorContext: システム起動時に作成される基底コンテキスト。ErrorContext は OOM 時でも動作を保証するため、クリティカルセクション内での割り当てが許可されています。

割り当て API

  • palloc, palloc0, palloc_extended: CurrentMemoryContext に対するメモリ確保。
  • pfree, repalloc: 既存チャンクの解放およびサイズ変更。
  • MemoryContextAllocAligned: 特定のアライメント要件を満たすための特殊な割り当て。

制御フローとデータフロー

1. **割り当て**: palloc(size) は CurrentMemoryContext を参照し、その methods->alloc を呼び出します。 2. **ルーティング**: MCXT_METHOD マクロがチャンク直前のヘッダー（uint64）から MethodID を読み取り、mcxt_methods テーブルから適切な関数を特定します。 3. **階層走査**: MemoryContextDelete や MemoryContextStats は、再帰によるスタックオーバーフローを防ぐため、MemoryContextTraverseNext を用いたループ構造で階層を処理します。 4. **エラーパス**: BogusFree 等のダミー関数が、無効なポインタや不正なヘッダーを持つチャンクに対する pfree 呼び出しを検知し、elog(ERROR) で停止させます。

読者向けメモ

  • **メモリ所有権**: コンテキストは階層構造（ツリー）を形成します。親コンテキストが削除されると、その配下の全子コンテキストも自動的に削除されます。
  • **レイアウト**: 割り当てられたメモリの直前には必ず uint64 のヘッダーが存在し、そこに MethodID が埋め込まれています。このため、pfree はコンテキストを明示的に渡さなくても、ポインタから適切な解放関数を特定できます。
  • **クリティカルセクション**: AssertNotInCriticalSection により、PostgreSQL のクリティカルセクション内でのメモリ割り当ては原則禁止されています。ErrorContext のような例外を除き、このルールを破ると PANIC を引き起こします。
  • **省略された詳細**: 本ファイルはディスパッチャーであるため、各アロケータの内部アルゴリズム（フリーリストの管理方法やページ分割戦略など）については各実装ファイルを参照してください。
