src/backend/utils/misc/pg_config.c

概要

pg_config.c は、PostgreSQL のビルド時設定情報を SQL インターフェースを通じて取得するための関数 pg_config() を提供するソースファイルです。

モジュール内での役割

src/backend/utils/misc ディレクトリにおいて、システム設定や環境情報を提供するユーティリティの一部です。外部コマンドである pg_config が出力する情報を、SQL 関数としてデータベース内から直接参照可能にします。

責務

  • ビルド時に決定された構成パラメータ（コンパイルオプション、パス設定など）の取得。
  • 取得した設定情報を、SQL の集合を返す関数（SRF: Set-Returning Function）として整形して出力する。

重要なシンボル

  • pg_config(PG_FUNCTION_ARGS): SQL から呼び出されるメイン関数。pg_config ビューや関数のバックエンド実装です。

制御フローとデータフロー

1. **初期化**: InitMaterializedSRF を呼び出し、結果を格納するためのタプルストアを準備します。 2. **データ取得**: get_configdata を呼び出し、実行バイナリのパス (my_exec_path) を基に設定情報の配列を取得します。 3. **変換と格納**: 取得した ConfigData 配列をループで走査し、各要素を CStringGetTextDatum でテキスト型に変換します。 4. **出力**: tuplestore_putvalues を使用して、名前と設定値のペアをタプルストアに書き込み、呼び出し元へ返却します。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • common/config_info.h: ConfigData 構造体および get_configdata 関数の定義。
  • funcapi.h: SRF 実装のための関数インターフェース。
  • miscadmin.h: my_exec_path などのグローバル変数へのアクセス。
  • utils/builtins.h: 組み込み関数としての宣言。

関連コード

  • src/common/config_info.c: get_configdata の実体が定義されており、ビルド時の構成情報を生成するロジックが含まれています。
  • src/include/catalog/pg_proc.dat: pg_config() 関数が SQL レベルでどのように定義されているかを確認できます。

読者向けメモ

  • 本ファイルは非常にシンプルで、主に common ライブラリのラッパーとして機能しています。
  • pg_config コマンドと同一の情報を返すため、ビルド環境の差異をデバッグする際に有用です。
  • 戻り値は tuplestore を介した動的な集合であるため、メモリ消費量は設定項目の数に依存します。
