src/backend/utils/misc/guc_funcs.c

概要

本ファイルは、PostgreSQLのGUC（Grand Unified Configuration）パラメータをSQLインターフェース経由で操作・参照するための関数群を実装しています。SET コマンドの実行や SHOW コマンド、および pg_settings ビュー等の基盤となる関数を提供します。

モジュール内での役割

src/backend/utils/misc モジュールにおいて、GUCの内部管理ロジック（guc.c）と、ユーザーがSQLを通じて対話するインターフェースの橋渡しを担います。

責務

  • **SETコマンドの実行**: SET 文の解析結果を受け取り、適切なGUC設定関数を呼び出す。
  • **SHOWコマンドの処理**: 特定のパラメータまたは全パラメータの値をシステムカタログ形式で出力する。
  • **SQL関数インターフェース**: set_config_by_name や show_all_settings など、SQLから直接呼び出し可能な関数の提供。
  • **メタデータ抽出**: パラメータの属性（フラグ、定義値、ソース情報など）をSQL層へ公開する。

重要なシンボル

  • ExecSetVariableStmt: SET コマンドの実行エントリポイント。
  • SetPGVariable: CコードからGUCを設定するための汎用インターフェース。
  • show_all_settings: pg_settings ビューのバックエンドとなるテーブル関数。
  • flatten_set_variable_args: SQLの構文解析結果（リスト）をGUCが処理可能な文字列形式に変換する。
  • ConfigOptionIsVisible: 現在のユーザーが特定のGUC設定を参照可能か判定する。

制御フローとデータフロー

1. **SET処理**: ExecSetVariableStmt がパーサーからの VariableSetStmt を受け取り、flatten_set_variable_args で値を正規化し、set_config_option（guc.c）を呼び出して値を更新します。 2. **SHOW処理**: GetPGVariable が対象を判別し、ShowGUCConfigOption または ShowAllGUCConfig を経由して DestReceiver にタプルを送信します。 3. **テーブル関数**: show_all_settings は SRF（Set Returning Function）として実装され、get_guc_variables で全設定を取得し、GetConfigOptionValues で各行の属性を構築して順次返却します。

読者向けメモ

  • 本ファイルはSQLインターフェースに特化しており、GUCの「値の保持」や「検証ロジック」そのものは guc.c に依存しています。
  • show_all_settings は非常に多くの属性を扱うため、NUM_PG_SETTINGS_ATTS 定数と GetConfigOptionValues 内のスイッチ文が密結合しています。修正時は両方の整合性に注意してください。
  • セキュリティ上の理由から、権限のないユーザーに対しては設定値のソース（ファイル名や行番号）を隠蔽するロジックが含まれています。
