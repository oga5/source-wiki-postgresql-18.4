src/backend/utils/misc/injection_point.c

概要

PostgreSQL の実行コード中に任意のコードを挿入するための「インジェクションポイント」機能を提供します。主にテストやデバッグ目的で、特定のコードパスにフックを仕込み、外部ライブラリの関数を動的に呼び出すために使用されます。

モジュール内での役割

src/backend/utils/misc ディレクトリに配置され、共有メモリを介したプロセス間通信と、各バックエンドプロセス内でのローカルキャッシュ管理を担います。injection_point.h で定義されたマクロインターフェースの実装本体です。

責務

  • **共有メモリ管理**: インジェクションポイントの登録情報を保持する共有メモリ領域の確保と初期化。
  • **動的ライブラリのロード**: 指定されたライブラリと関数を load_external_function で読み込み、実行可能にする。
  • **キャッシュ管理**: 頻繁な検索を避けるため、各バックエンドプロセス内にハッシュテーブル形式のローカルキャッシュを保持する。
  • **スレッドセーフな読み取り**: 共有メモリ上のエントリに対して、ロックフリーな読み取り（世代管理カウンタによる検証）を可能にする。

重要なシンボル

  • InjectionPointEntry: 共有メモリ上の単一のインジェクションポイント定義。
  • InjectionPointsCtl: 共有メモリ上の全インジェクションポイントを管理する構造体。
  • InjectionPointCacheEntry: バックエンドローカルなキャッシュエントリ。
  • InjectionPointRun: 指定された名前のインジェクションポイントを実行するメイン関数。
  • InjectionPointAttach: 新しいインジェクションポイントを登録する関数。

制御フローとデータフロー

1. **登録**: InjectionPointAttach が InjectionPointLock を取得し、共有メモリの空きスロットに情報を書き込み、世代カウンタを更新します。 2. **実行**: InjectionPointRun が呼び出されると、まず InjectionPointCacheRefresh を経由してローカルキャッシュを確認します。 3. **キャッシュ更新**: キャッシュに存在しない、または世代カウンタが古い場合、共有メモリをスキャンして最新情報を取得し、load_external_function で関数ポインタを解決してキャッシュを更新します。 4. **実行**: 解決された関数ポインタを呼び出し、指定された引数を渡します。

読者向けメモ

  • **ビルド要件**: 本機能は --enable-injection-points を指定してビルドした場合のみ有効です。無効な場合はマクロが空の操作に置換されます。
  • **ロックフリーの仕組み**: 共有メモリ上の generation カウンタは、読み取り時にロックを回避するためのものです。読み取り前後のカウンタ値を比較し、不一致であれば再試行または無視する設計になっています。
  • **デバッグ用途**: 本機能は本番環境での使用を想定していません。動的ライブラリのロードを伴うため、セキュリティおよび安定性の観点から慎重に扱う必要があります。
