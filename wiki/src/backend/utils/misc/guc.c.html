src/backend/utils/misc/guc.c

概要

guc.c は、PostgreSQLの「Grand Unified Configuration (GUC)」システムの中心となるエンジンです。サーバー設定パラメータの定義、読み込み、検証、適用、および永続化（ALTER SYSTEM）を管理します。本ファイルは、設定値のライフサイクル管理と、SQLインターフェース（SET/SHOW）のバックエンド処理を担う大規模なインフラストラクチャです。

モジュール内での役割

src/backend/utils/misc サブシステムにおいて、設定管理の「コア」として機能します。

  • guc_tables.c: パラメータの静的定義を提供。
  • guc_funcs.c: SQLレベルのコマンド（SET, SHOW）の入り口。
  • conffiles.c: 設定ファイルのパース処理。
  • 本ファイルは、これらを統合し、メモリ上の状態管理と検証ロジックを統括します。

責務

  • **設定値のライフサイクル管理**: 起動時のデフォルト値適用、設定ファイルからの読み込み、セッション/トランザクション単位の変更管理。
  • **検証と適用**: 型変換、単位変換（kB, MB等）、範囲チェック、およびカスタムフック（check/assign hook）の実行。
  • **トランザクション管理**: SET LOCAL 等によるトランザクション/サブトランザクション単位の変更のスタック管理とロールバック処理。
  • **永続化**: ALTER SYSTEM による postgresql.auto.conf の更新とアトミックな反映。
  • **シリアライズ**: パラレルクエリ実行時のワーカープロセスへの設定値同期。

重要なシンボル

データ構造

  • GUCHashEntry: パラメータ名から定義構造体への高速アクセス用ハッシュテーブルエントリ。
  • GucStack: トランザクション単位の設定変更を保持するスタック構造。
  • unit_conversion: メモリや時間の単位変換テーブル。

主要な管理変数

  • guc_hashtab: 全パラメータを管理するハッシュテーブル。
  • guc_nondef_list: デフォルト値から変更されたパラメータのリスト。
  • guc_stack_list: トランザクションスタックを持つパラメータのリスト。
  • GUCMemoryContext: GUC関連の全データが確保される専用メモリコンテキスト。

主要な関数

  • ProcessConfigFileInternal: 設定ファイルの読み込みと反映のメインロジック。
  • set_config_option: パラメータ設定の汎用インターフェース。
  • find_option: パラメータ名の検索とカスタムプレースホルダーの生成。
  • AtEOXact_GUC: トランザクション終了時の設定値復元処理。

制御フローとデータフロー

1. **初期化**: InitializeGUCOptions が build_guc_variables を呼び出し、ハッシュテーブルを構築してデフォルト値を適用します。 2. **設定変更**: set_config_option が呼ばれると、まず find_option で定義を取得し、parse_and_validate_value で型変換とチェックフックを実行します。 3. **スタック管理**: トランザクション内で変更が行われる場合、push_old_value で現在の値をスタックに退避し、AtEOXact_GUC でコミット/アボートに応じて値を復元します。 4. **永続化**: AlterSystemSetConfigFile は LWLock を取得し、既存の設定をパースして値を置換後、一時ファイル経由で postgresql.auto.conf をアトミックに更新します。

読者向けメモ

  • **メモリ管理**: GUCデータは GUCMemoryContext に集約されており、pfree ではなく guc_free を使用して管理されます。
  • **エラーハンドリング**: check_hook 内でのエラー報告は GUC_check_errcode 等の専用マクロを介して行われます。
  • **並列処理**: パラレルワーカーへの設定同期は SerializeGUCState / RestoreGUCState を通じて行われます。can_skip_gucvar のロジックは非常に繊細であり、変更時は注意が必要です。
  • **省略された詳細**: 単位変換の具体的な計算式や、個別のパラメータの check_hook 実装は各モジュールに分散しているため、本ファイルでは汎用的な呼び出しロジックのみを扱っています。
