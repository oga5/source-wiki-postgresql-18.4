src/backend/utils/misc/guc_tables.c

概要

PostgreSQLの全設定パラメータ（GUC: Grand Unified Configuration）の定義を保持する静的テーブル群を定義するファイルです。各パラメータの型、デフォルト値、有効範囲、説明文、および関連するフック関数を構造体配列として管理しています。

モジュール内での役割

GUCサブシステムにおける「データ定義層」です。guc.c が実行時の設定変更や検証ロジックを担うのに対し、本ファイルは設定のメタデータ（名前、型、カテゴリ、制約）を静的に宣言します。

責務

  • **パラメータ定義**: config_bool, config_int, config_real, config_string, config_enum の各配列による全設定項目の定義。
  • **列挙型オプションの管理**: config_enum_entry を用いた、特定のパラメータで選択可能な値のリスト定義。
  • **表示用メタデータの提供**: GucContext_Names や config_group_names など、SQLインターフェース（pg_settings ビュー等）で利用される表示用文字列の定義。

重要なシンボル

パラメータ定義テーブル

  • ConfigureNamesBool[]: 真偽値設定（例: enable_seqscan）。
  • ConfigureNamesInt[]: 整数値設定（例: max_connections, work_mem）。
  • ConfigureNamesReal[]: 浮動小数点設定（例: seq_page_cost）。
  • ConfigureNamesString[]: 文字列設定（例: archive_command）。
  • ConfigureNamesEnum[]: 列挙型設定（例: wal_level, synchronous_commit）。

表示・メタデータ

  • GucContext_Names[]: 設定の適用タイミング（PGC_USERSET 等）の文字列変換。
  • GucSource_Names[]: 設定値の由来（PGC_S_FILE 等）の文字列変換。
  • config_group_names[]: pg_settings で表示されるカテゴリ名。

列挙型オプション（抜粋）

  • bytea_output_options, client_message_level_options, log_statement_options など、特定のパラメータが取り得る値を定義する静的配列。

制御フローとデータフロー

本ファイルは実行コードをほとんど含まず、主にデータ構造の定義です。

  • **初期化**: guc.c 内の InitializeGUCOptions() がこれらのテーブルを走査し、グローバル変数にデフォルト値を適用します。
  • **検証・適用**: guc.c が SET コマンドを受け取ると、これらのテーブル内の check_hook や assign_hook を呼び出し、値の妥当性チェックと適用を行います。
  • **データフロー**: guc_tables.c で定義された構造体ポインタが guc.c のハッシュテーブルに登録され、SQL経由の参照・更新の基盤となります。

読者向けメモ

  • **静的テーブルの性質**: 配列の最後には必ず {NULL, ...} で終わるマーカーが必要です。追加時は必ず末尾にマーカーを維持してください。
  • **メモリ管理**: config_string のデフォルト値は静的メモリを指すことが一般的ですが、実行時に変更される場合は guc.c 側で適切にメモリ確保が行われます。
  • **制約**: StaticAssertDecl を用いて、列挙型とオプション配列のサイズ不一致をコンパイル時に検出しています。新しい列挙型を追加した際は、このチェックが通ることを確認してください。
  • **省略された詳細**: フック関数の具体的な実装ロジックは guc.c にあるため、本ファイルでは「どの関数が呼ばれるか」という宣言のみを確認してください。
