src/backend/utils/error/jsonlog.c

概要

PostgreSQL のエラーログを JSON 形式で出力するためのユーティリティを提供します。構造化ログを必要とする外部監視ツールやログ収集基盤との統合を目的としています。

モジュール内での役割

src/backend/utils/error モジュールの一部として、elog.c が生成した ErrorData 構造体を JSON 文字列へ変換し、ログ出力パイプラインへ渡す役割を担います。

責務

  • **JSON シリアライズ**: ErrorData の各フィールドを JSON のキー・値ペアに変換し、適切にエスケープ処理を行う。
  • **動的フォーマット**: printf 形式の引数を受け取り、JSON 値として埋め込むための補助関数を提供。
  • **ログ出力のルーティング**: プロセス種別（syslogger か否か）に応じて、直接ファイル書き込みまたはパイプ経由での送信を行う。

重要なシンボル

  • write_jsonlog(ErrorData *edata): 外部から呼び出されるメイン関数。ErrorData を JSON 文字列に変換して出力します。
  • appendJSONKeyValue: JSON のキーと値をバッファに追加する基本関数。値のエスケープ有無を制御可能です。
  • appendJSONKeyValueFmt: printf 形式でフォーマットされた値を JSON 値として追加するラッパー関数。

追加の索引済みシンボル

  • write_jsonlog

制御フローとデータフロー

1. **初期化**: write_jsonlog が呼び出されると、StringInfo バッファを初期化し、JSON の開始文字 { を挿入します。 2. **メタデータ収集**: プロセス情報（PID、ユーザー名、DB名）、セッション情報、トランザクション ID などを順次バッファに追加します。 3. **エラー詳細の追加**: ErrorData に格納されたメッセージ、詳細、ヒント、SQL ステートなどを JSON フィールドとして追加します。 4. **終了処理**: JSON の閉じ括弧 } と改行を付与します。 5. **出力**: MyBackendType を確認し、syslogger プロセスであれば直接ファイルへ、それ以外であればパイプ経由でログ出力プロセスへデータを転送します。

読者向けメモ

  • **エスケープ処理**: appendJSONKeyValue はキーを常にエスケープしますが、値は escape_value フラグによって制御されます。数値やフォーマット済みの文字列など、エスケープが不要な場合は適切にフラグを調整してください。
  • **静的変数**: log_line_number はプロセスごとに管理されており、PID が変更された際にリセットされる設計になっています。
  • **メモリ管理**: StringInfo を使用して動的にバッファを拡張するため、メモリリークを防ぐために pfree の呼び出し位置には注意が必要です。
