src/backend/utils/error/csvlog.c

概要

PostgreSQL のエラーログを CSV 形式で出力するためのロジックを実装したファイルです。ログ収集プロセス（syslogger）が解析しやすいよう、構造化されたフォーマットでログメッセージを生成します。

モジュール内での役割

src/backend/utils/error モジュールの一部として、elog.c が生成した ErrorData 構造体を、CSV 形式の文字列へ変換する役割を担います。

責務

  • **CSV フォーマットの構築**: ErrorData の各フィールドを CSV 形式に変換し、カンマ区切りで連結する。
  • **エスケープ処理**: ログメッセージ内のダブルクォーテーションを適切にエスケープし、CSV の整合性を保つ。
  • **ログのルーティング**: プロセス種別に応じて、syslogger へのパイプ送信または直接書き込みを行う。
  • **メタデータの管理**: セッションごとのログ行番号や開始時刻などの追跡。

重要なシンボル

  • write_csvlog(ErrorData *edata): 外部から呼び出されるメイン関数。エラー情報を CSV 文字列に変換し、出力先へ送る。
  • appendCSVLiteral(StringInfo buf, const char *data): 文字列を CSV リテラルとしてバッファに追加するヘルパー関数。ダブルクォーテーションのエスケープを行う。

追加の索引済みシンボル

  • appendCSVLiteral
  • write_csvlog

制御フローとデータフロー

1. **初期化**: write_csvlog が呼び出されると、プロセスごとのログ行番号（log_line_number）を更新し、StringInfo バッファを初期化します。 2. **データ収集**: タイムスタンプ、ユーザー名、DB 名、セッション ID、トランザクション ID などのメタデータを順次バッファに追加します。 3. **エラー詳細の変換**: ErrorData に含まれるメッセージ、詳細、ヒントなどを appendCSVLiteral を通じてバッファへ追加します。 4. **出力**:

  • MyBackendType が B_LOGGER（syslogger プロセス）の場合、write_syslogger_file を呼び出し直接ファイルへ書き込みます。
  • それ以外の場合、write_pipe_chunks を介して syslogger へデータを転送します。

読者向けメモ

  • **エスケープルール**: PostgreSQL の CSV ログは、ダブルクォーテーションを "" とすることでエスケープします。このルールは appendCSVLiteral に集約されています。
  • **プロセス間通信**: 通常のバックエンドプロセスは直接ファイルに書き込まず、パイプ経由で syslogger にデータを渡す設計になっています。これはログ書き込みによるブロッキングを防ぐためです。
  • **静的変数**: log_line_number はプロセスごとに保持されるため、プロセスが再利用される際や PID が変わる際の初期化ロジックに注意してください。
