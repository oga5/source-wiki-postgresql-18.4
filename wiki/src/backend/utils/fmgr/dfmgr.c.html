src/backend/utils/fmgr/dfmgr.c

概要

PostgreSQL の動的ライブラリ（共有オブジェクト）のロードおよび管理を行うモジュールです。外部関数を呼び出すための共有ライブラリの読み込み、ABI 互換性の検証、およびロード済みライブラリの追跡を担当します。

モジュール内での役割

fmgr（Function Manager）サブシステムの一部として、SQL レベルで定義された外部関数が参照する共有ライブラリを、実行時にプロセス空間へマッピングする役割を担います。

責務

  • **動的ロード**: dlopen を用いた共有ライブラリの読み込み。
  • **ABI 検証**: PG_MODULE_MAGIC マクロによるバージョンおよび ABI 互換性のチェック。
  • **ライブラリ管理**: ロード済みライブラリのリスト保持と重複ロードの防止。
  • **パス解決**: dynamic_library_path 設定に基づいたライブラリの検索とパス展開。
  • **Rendezvous 機構**: 異なる動的ライブラリ間でポインタを共有するための名前付き変数管理。

重要なシンボル

  • load_external_function: 指定したライブラリから特定の関数シンボルを検索・ロードするメイン関数。
  • internal_load_library: ライブラリをロードし、DynamicFileList に登録する内部処理。
  • DynamicFileList: ロード済みライブラリを管理する連結リスト構造体。
  • find_rendezvous_variable: ライブラリ間で情報を共有するためのハッシュテーブルベースの変数管理。
  • Dynamic_library_path: ライブラリ検索パスを保持するグローバル変数。

制御フローとデータフロー

1. **検索**: expand_dynamic_library_name がパスを解決し、絶対パスを取得します。 2. **ロード**: internal_load_library が file_list を走査し、未ロードであれば dlopen を実行します。 3. **検証**: ロード後、PG_MAGIC_FUNCTION_NAME_STRING を通じて ABI 互換性を確認します。失敗時は incompatible_module_error で詳細を報告します。 4. **初期化**: _PG_init 関数が存在すれば実行します。 5. **シンボル解決**: dlsym を用いて目的の関数ポインタを取得します。

読者向けメモ

  • **アンロード不可**: 現在の設計では、一度ロードしたライブラリを安全にアンロードする仕組みはありません。
  • **ABI 互換性**: PG_MODULE_MAGIC を使用していないライブラリはロード時に拒否されます。拡張開発時は必須です。
  • **Rendezvous**: 異なるライブラリ間でグローバル変数を共有したい場合、find_rendezvous_variable が唯一の安全な手段です。
  • **Win32**: Windows 環境では dlfcn.h の代わりに OS 固有の API が使用されますが、SAME_INODE マクロなどで抽象化されています。
