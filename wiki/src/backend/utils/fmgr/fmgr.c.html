src/backend/utils/fmgr/fmgr.c

概要

PostgreSQLの関数管理（Function Manager）の心臓部です。SQLレベルの関数呼び出しを、実際のC言語関数ポインタやPLハンドラへディスパッチする役割を担います。関数メタデータのキャッシュ、動的ライブラリのロード管理、およびセキュリティコンテキストの切り替えを統合的に処理します。

モジュール内での役割

fmgr.h で定義されたインターフェースの実装を提供し、クエリ実行エンジンと関数実装の間の仲介を行います。dfmgr.c（動的ロード）や funcapi.c（SRF/多相型）と連携し、実行時の関数解決を最適化します。

責務

  • **関数解決とキャッシュ**: OIDから関数ポインタを特定し、FmgrInfo 構造体にキャッシュして以降の呼び出しを高速化。
  • **ディスパッチ**: 組み込み関数、C言語関数、SQL関数、およびPLハンドラへの呼び出しを抽象化。
  • **セキュリティと環境制御**: SECURITY DEFINER 関数や proconfig（関数固有のGUC設定）の適用。
  • **データ型I/O**: 型変換関数（Input/Output/Receive/Send）の呼び出しと、エラーハンドリングの抽象化。
  • **メモリ管理**: 関数呼び出しに伴う一時的なメモリコンテキストの管理と、fn_extra を用いた関数固有の永続化データの保持。

重要なシンボル

関数解決・管理

  • fmgr_info(): OIDから関数情報を取得し、FmgrInfo を初期化する主要エントリーポイント。
  • fmgr_info_cxt_security(): セキュリティ定義やGUC設定を考慮した関数解決の内部ロジック。
  • fmgr_isbuiltin(): 組み込み関数テーブルの高速検索。
  • fmgr_symbol(): 関数実装のシンボル名とライブラリパスを解決。

実行ハンドラ

  • fmgr_security_definer(): SECURITY DEFINER や proconfig を持つ関数の実行をラップするハンドラ。GUCのネストレベル管理とユーザーIDの切り替えを行う。

呼び出しインターフェース

  • DirectFunctionCallNColl(): FmgrInfo を介さず、関数ポインタを直接呼び出すための高速なヘルパー群。
  • FunctionCallNColl(): FmgrInfo を使用した標準的な呼び出しインターフェース。
  • InputFunctionCallSafe(): エラー発生時に例外を投げず、ErrorSaveContext を通じてソフトエラーを報告するI/O関数呼び出し。

キャッシュ構造

  • CFuncHash: 外部C言語関数のロード済みアドレスを保持するハッシュテーブル。
  • CFuncHashTabEntry: キャッシュエントリ。OIDとトランザクションID（fn_xmin）を保持し、キャッシュの有効性を検証。

制御フローとデータフロー

1. **初期化**: fmgr_info が呼び出されると、まず fmgr_isbuiltin で組み込みテーブルを検索。見つからない場合は pg_proc を検索。 2. **解決**: prolang に応じて適切なハンドラ（fmgr_info_C_lang 等）を呼び出し、関数ポインタを確定。 3. **実行**: FunctionCallInvoke を通じて関数ポインタへディスパッチ。 4. **セキュリティ**: fmgr_security_definer が介在する場合、fn_extra にキャッシュされた fmgr_security_definer_cache を使用し、GUC設定とユーザーIDを一時的に変更してからターゲット関数を呼び出す。 5. **エラー**: PG_TRY/PG_CATCH ブロックにより、関数実行中のエラー発生時でもセキュリティコンテキストやGUC設定が確実に復元される。

読者向けメモ

  • **メモリ所有権**: FmgrInfo 内の fn_extra は、fn_mcxt で指定されたコンテキストに割り当てられます。永続的なキャッシュが必要な場合は、適切なライフタイムを持つコンテキストを指定してください。
  • **不変性**: FmgrInfo は一度初期化された後は読み取り専用として扱うのが原則です。
  • **エッジケース**: fmgr_security_definer は再入可能ではありません。また、DirectFunctionCall 系はNULL入力を許容しないため、NULLが発生しうる場合は FunctionCall 系を使用してください。
  • **省略された詳細**: 集合を返す関数（SRF）の具体的な実装ロジックは funcapi.c に委譲されており、本ファイルでは主にディスパッチとメタデータ解決に焦点を当てています。
