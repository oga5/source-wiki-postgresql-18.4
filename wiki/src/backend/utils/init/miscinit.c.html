src/backend/utils/init/miscinit.c

概要

PostgreSQLのバックエンドプロセスおよびPostmasterの起動・初期化における低レイヤーの環境構築を担うファイルです。データディレクトリの検証、プロセス間通信（IPC）の初期準備、ユーザーIDの管理、およびロックファイルの管理といった、システム稼働の基盤となる処理を集約しています。

モジュール内での役割

src/backend/utils/init モジュールにおいて、OSレベルの環境設定を行う最下層のレイヤーです。postinit.c がデータベース接続後の論理的な初期化を行うのに対し、本ファイルはプロセスが実行可能になるまでの「物理的・環境的」な準備を担当します。

責務

  • **プロセス環境の初期化**: シグナルマスクの設定、ラッチの初期化、およびPostmaster子プロセスとしての環境構築。
  • **データディレクトリ管理**: ディレクトリの存在確認、所有者・権限チェック、および postmaster.pid ロックファイルの生成・保守。
  • **ユーザーIDコンテキスト管理**: セッションユーザー、認証ユーザー、現在の実行ユーザーIDの追跡と、セキュリティ制限コンテキストの制御。
  • **ライブラリのプリロード**: 起動時に必要な共有ライブラリのロード処理。

重要なシンボル

プロセス環境・状態管理

  • InitPostmasterChild(): Postmasterの子プロセスとして必要なシグナル設定やラッチ初期化を行う。
  • InitStandaloneProcess(): スタンドアロンモード（シングルユーザー）用の環境初期化。
  • MyBackendType: 現在のプロセスの種類（バックエンド、チェックポインター等）を保持。

データディレクトリ・ロック管理

  • checkDataDir(): データディレクトリの整合性、所有者、権限を検証し、適切なパーミッションマスクを設定。
  • CreateDataDirLockFile(): postmaster.pid を作成し、二重起動を防止。
  • AddToDataDirLockFile(): 実行中にロックファイルへ情報を追記する。

ユーザーID管理

  • GetUserId() / SetUserIdAndSecContext(): 現在の実行ユーザーIDおよびセキュリティコンテキスト（SECURITY_LOCAL_USERID_CHANGE 等）の取得・設定。
  • InitializeSessionUserId(): 認証後のセッションユーザーIDを初期化。
  • SetSessionAuthorization() / SetCurrentRoleId(): SET SESSION AUTHORIZATION および SET ROLE の実行フック。

制御フローとデータフロー

1. **起動時**: プロセス起動時に InitPostmasterChild または InitStandaloneProcess が呼ばれ、InitProcessLocalLatch でラッチが初期化されます。 2. **ディレクトリ検証**: checkDataDir が呼ばれ、stat を用いてディレクトリの所有者とパーミッションを検証。問題なければ umask を調整します。 3. **ロックファイル生成**: CreateLockFile が O_EXCL を用いてアトミックにロックファイルを作成。既存ファイルがある場合は kill(pid, 0) でプロセス生存確認を行い、スタックしたロックを自動的にクリーンアップします。 4. **ユーザーコンテキスト**: InitializeSessionUserId がシステムカタログ（pg_authid）を参照し、認証済みユーザーの権限を検証してグローバル変数（CurrentUserId 等）を更新します。

読者向けメモ

  • **メモリ所有権**: DatabasePath や SystemUser などは TopMemoryContext に割り当てられ、プロセス終了まで生存します。
  • **ロックファイルの原子性**: ロックファイルの更新（AddToDataDirLockFile）は、ファイル全体を読み込み、メモリ上で編集し、pg_pwrite で上書きすることで原子性を担保しています。
  • **Windows対応**: WIN32 マクロによる条件分岐が多く、シグナルやファイルパーミッションの扱いがUnix系と異なります。
  • **省略された詳細**: 本ファイルは非常に広範な初期化処理を含むため、各関数の詳細なエラーハンドリングや、特定のOS固有のシグナル処理の詳細は省略しています。これらは src/backend/postmaster/ 以下の各プロセス実装と密接に関連しています。
