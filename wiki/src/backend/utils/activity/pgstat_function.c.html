src/backend/utils/activity/pgstat_function.c

概要

PostgreSQLにおける関数実行統計（呼び出し回数、実行時間）の収集と管理を行うモジュールです。SQL関数の実行コストを可視化するための基盤を提供します。

モジュール内での役割

pgstat.c が提供する統計収集フレームワークを利用し、関数固有の統計データ構造と処理ロジックを分離して実装しています。実行エンジン（Executor）と統計サブシステムを繋ぐインターフェースとして機能します。

責務

  • 関数実行前後の計測処理（pgstat_init_function_usage / pgstat_end_function_usage）の提供。
  • 再帰呼び出しを考慮した実行時間の正確な算出。
  • トランザクション境界での統計情報の整合性維持。
  • 統計情報の共有メモリへのフラッシュ処理（pgstat_function_flush_cb）。

重要なシンボル

  • pgstat_track_functions: 関数統計の収集レベルを制御するGUCパラメータ。
  • pgstat_init_function_usage: 関数実行開始時に呼び出され、計測用コンテキストを初期化する。
  • pgstat_end_function_usage: 関数実行終了時に呼び出され、経過時間を計算して統計を更新する。
  • pgstat_function_flush_cb: バックエンドローカルの統計を共有メモリへ書き出すためのコールバック関数。

制御フローとデータフロー

1. **開始**: pgstat_init_function_usage が実行時に呼ばれ、現在のバックエンド全体の累積時間と対象関数の開始時刻を記録します。 2. **計測**: 関数実行中、他の関数が呼び出されると total_func_time が更新されます。 3. **終了**: pgstat_end_function_usage が呼ばれると、以下の計算が行われます。

  • 経過時間 = 現在時刻 - 開始時刻
  • 自己時間 = 経過時間 - (終了時の累積時間 - 開始時の累積時間)
  • この結果をローカルの統計エントリに加算します。

    • 4. **永続化**: バックエンド終了時や統計収集のタイミングで pgstat_function_flush_cb が呼ばれ、ローカルの累積値が共有メモリ上の統計情報にマージされます。

読者向けメモ

  • **再帰呼び出しの扱い**: 本モジュールは、関数が再帰的に呼び出された場合でも、親関数の「自己時間」が正しく計算されるよう、バックエンド全体の累積時間を用いて差分を算出するロジックを採用しています。
  • **キャッシュ無効化**: 関数が実行中に削除された場合、pgstat_init_function_usage 内で AcceptInvalidationMessages() を呼び出し、システムカタログを再確認して整合性を保つ工夫がなされています。
  • **パフォーマンス**: 統計収集はオーバーヘッドを伴うため、pgstat_track_functions の設定値によって収集対象を制御することが推奨されます。
