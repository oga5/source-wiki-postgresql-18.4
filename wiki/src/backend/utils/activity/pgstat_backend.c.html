src/backend/utils/activity/pgstat_backend.c

概要

各バックエンドプロセス固有の統計情報（I/O および WAL 使用量）を管理するための実装ファイルです。プロセス生存期間中の統計を保持し、共有メモリ上の統計領域へ定期的にフラッシュする役割を担います。

モジュール内での役割

pgstat.c が統計サブシステムの全体的なアクセス・ストレージ管理を行うのに対し、本ファイルは「バックエンド」という特定の統計種別に特化したロジックを分離・カプセル化しています。

責務

  • **ローカル統計の保持**: プロセスごとの I/O および WAL 統計を PendingBackendStats に蓄積し、クリティカルセクション内でも安全に更新可能にする。
  • **共有メモリへのフラッシュ**: 蓄積された統計を共有メモリ上の PgStatShared_Backend 構造体に反映する。
  • **統計対象の判定**: pgstat_tracks_backend_bktype を通じて、どのバックエンド種別が統計収集の対象となるかを制御する。
  • **ライフサイクル管理**: プロセス起動時の初期化および終了時の統計処理（間接的）をサポートする。

重要なシンボル

  • PendingBackendStats: プロセスローカルな統計保持用構造体。メモリ割り当てを避けるため静的領域に配置。
  • pgstat_count_backend_io_op(_time): I/O 統計をローカルバッファに加算する関数。
  • pgstat_flush_backend: 蓄積された統計を共有メモリへ書き出すメイン関数。
  • pgstat_tracks_backend_bktype: 統計収集対象のバックエンド種別を判定するフィルタ。

制御フローとデータフロー

1. **蓄積**: 各種操作（I/O や WAL）が発生すると、pgstat_count_... 系関数が PendingBackendStats を更新し、backend_has_iostats フラグを立てる。 2. **フラッシュ要求**: pgstat_backend_flush_cb が呼び出されると、pgstat_flush_backend が実行される。 3. **同期**: pgstat_get_entry_ref_locked で共有メモリ上のエントリをロックし、pgstat_flush_backend_entry_io および pgstat_flush_backend_entry_wal がローカル値を共有メモリへ加算（Accumulate）する。 4. **リセット**: フラッシュ完了後、ローカルバッファをクリアし、WAL 統計計算用の基準値（prevBackendWalUsage）を更新する。

読者向けメモ

  • **クリティカルセクション**: 本ファイル内の統計更新は、メモリ割り当てを行わない設計になっています。これは、メモリ確保が禁止されているクリティカルセクション内から統計を報告できるようにするためです。
  • **永続化**: バックエンド統計はプロセス終了とともに消滅するため、ディスク上の統計ファイルには書き出されません。
  • **WAL 統計**: prevBackendWalUsage を用いて、前回のフラッシュ時からの差分を計算する差分更新方式を採用しています。
