src/backend/utils/adt/xml.c

概要

PostgreSQL の xml データ型をサポートする実装ファイルです。libxml2 ライブラリをバックエンドで利用し、XML のパース、シリアライズ、XPath 評価、および SQL/XML 規格に基づく SQL データから XML へのマッピング機能を提供します。

モジュール内での役割

src/backend/utils/adt モジュールの一部として、XML 型の入出力関数（xml_in, xml_out 等）および SQL 関数（xmlelement, xpath 等）を定義します。libxml2 がビルド時に有効化されていない場合でも、型定義自体は維持されますが、機能呼び出しはエラーとなります。

責務

  • **XML 型のライフサイクル管理**: libxml2 の初期化、エラーハンドリング、メモリ管理の統合。
  • **XML 変換とシリアライズ**: SQL データ型から XML への変換、および XML 宣言の制御。
  • **XPath 評価**: xpath() や xmlexists() 関数による XML ドキュメントのクエリ。
  • **XMLTABLE 実装**: TableFuncRoutine を介した XMLTABLE 演算子の実行エンジン。
  • **セキュリティ**: 外部エンティティの読み込み禁止など、XML 処理に伴う脆弱性対策。

重要なシンボル

エラーハンドリング

  • PgXmlErrorContext: libxml2 のエラーを PostgreSQL の ereport に変換するためのコンテキスト構造体。
  • pg_xml_init() / pg_xml_done(): エラーハンドラを登録・復元するライフサイクル管理関数。

XMLTABLE サポート

  • XmlTableRoutine: XMLTABLE 演算子の実行に必要なコールバック関数群（InitOpaque, FetchRow, GetValue 等）。
  • XmlTableBuilderData: XMLTABLE 実行中のパース状態や XPath コンテキストを保持する構造体。

SQL/XML マッピング

  • map_sql_value_to_xml_value(): SQL の Datum を XML 表現に変換するコア関数。
  • map_sql_table_to_xmlschema(): SQL テーブル構造を XML Schema にマッピングする関数群。

制御フローとデータフロー

1. **初期化**: 各関数は pg_xml_init() を呼び出し、libxml2 のグローバルエラーハンドラを xml_errorHandler に差し替えます。 2. **パース**: xml_parse() が入力を受け取り、libxml2 の xmlCtxtReadMemory 等を用いてドキュメントツリーを構築します。 3. **XPath 評価**: xpath_internal() が XPath コンテキストを構築し、xmlXPathCompiledEval で評価後、結果を ArrayBuildState に蓄積します。 4. **終了**: PG_TRY/CATCH ブロックの CATCH または終了時に pg_xml_done() を呼び出し、リソースを解放してエラーハンドラを元に戻します。

読者向けメモ

  • **メモリ管理**: libxml2 は独自のメモリ管理を行うため、PostgreSQL の palloc との整合性に注意が必要です。デバッグビルドでは USE_LIBXMLCONTEXT を有効にすることで LibxmlContext に割り当てを制限できます。
  • **セキュリティ**: xmlPgEntityLoader により、外部エンティティの読み込みは空文字列を返すことで無効化されています。
  • **省略された詳細**: libxml2 のバージョン依存の細かいパッチや、特定の SQL/XML 規格のマイナーなエッジケース処理は、コードの可読性を優先して詳細な説明を省略しています。
  • **注意点**: XMLTABLE の実行は TableFuncScanState のライフサイクルに強く依存するため、行単位の呼び出しではなく、一括処理を前提とした設計になっています。
