src/backend/utils/adt/like_match.c

概要

like_match.c は、PostgreSQL の LIKE 演算子および関連するパターンマッチング機能のコアロジックを実装するテンプレートファイルです。このファイルは単体でコンパイルされるのではなく、like.c から異なるエンコーディングや大文字小文字の区別といった条件に応じて複数回インクルード（#include）され、最適化されたマッチング関数を生成します。

モジュール内での役割

src/backend/utils/adt モジュールにおいて、文字列比較やパターンマッチングの低レイヤー処理を担います。like.c が SQL インターフェース（演算子関数）を提供し、本ファイルがその背後で実際のバイト列走査とパターン照合を行うという役割分担になっています。

責務

  • **パターンマッチングの実行**: MatchText 関数を通じ、%（任意の文字列）および _（任意の1文字）を含むパターンと対象文字列の照合を行う。
  • **エスケープ処理**: do_like_escape 関数により、ユーザー指定のエスケープ文字を PostgreSQL 標準のバックスラッシュ（\）形式に変換する。
  • **多言語・多エンコーディング対応**: マルチバイト文字や非決定論的照合順序（Collation）を考慮した走査ロジックの提供。

重要なシンボル

  • MatchText: パターンとテキストを比較する再帰的なメイン関数。LIKE_TRUE, LIKE_FALSE, LIKE_ABORT を返す。
  • do_like_escape: ユーザー定義のエスケープ文字を内部形式へ正規化する関数。
  • GETCHAR: マクロ。大文字小文字の区別が必要な場合に文字変換を適用する。
  • LIKE_ABORT: マッチング失敗時、これ以上の走査が不要であることを示す特殊な戻り値。

制御フローとデータフロー

1. **初期化**: like.c が特定のエンコーディングや大文字小文字区別の有無を定義し、本ファイルをインクルードする。 2. **走査**: MatchText がテキストとパターンを先頭から走査。

  • \ があればリテラルとして扱い、% や _ があれば再帰的にマッチングを試みる。
  • 非決定論的照合順序の場合、サブパターン単位で pg_strncoll を使用して比較を行う。

    • 3. **再帰**: % マッチング時に、残りのパターンがテキストのどこで一致するかを探索するために MatchText が再帰呼び出しされる。 4. **終了**: 全ての文字を消費したか、一致不可能と判断された時点で結果を返す。

直接依存

本ファイルは特定のヘッダーを直接インクルードせず、インクルード元である like.c が提供するマクロ（NextChar, CopyAdvChar 等）に依存する設計です。

関連コード

  • src/backend/utils/adt/like.c: 本ファイルをインクルードし、SQL から呼び出される関数群を定義する親ファイル。
  • src/backend/utils/adt/like_support.c: LIKE パターンに関するオプティマイザのサポート関数群。

読者向けメモ

  • **再帰の深さ**: MatchText は再帰関数であるため、極端に長いパターンや複雑なワイルドカードの組み合わせはスタックオーバーフローのリスクがあります。そのため check_stack_depth() が呼び出されています。
  • **テンプレート方式**: このファイルは「マクロ定義を先に行い、ファイルをインクルードする」という C 言語の古典的なテンプレート手法を採用しています。コードを読む際は、like.c 側でどのようなマクロが定義されているかを先に確認してください。
  • **パフォーマンス**: MatchText は可能な限りバイト単位の走査を行い、マルチバイト文字の境界を意識しつつも、不要な memcpy を避けるよう設計されています。
