src/backend/utils/adt/multirangetypes.c

概要

PostgreSQL のマルチレンジ型（multirange）の内部表現、I/O 処理、および SQL レベルの演算子・集約関数の実装を担うモジュールです。マルチレンジは複数のレンジ型を保持する可変長データ型であり、本ファイルはそれらのソート、マージ、シリアライズ、およびバイナリ走査を管理します。

モジュール内での役割

rangetypes.c が単一の範囲型を扱うのに対し、本ファイルはそれらの集合体としてのマルチレンジを扱います。src/include/utils/multirangetypes.h で定義された MultirangeType 構造体を利用し、PostgreSQL の型システム（typcache）と連携して、動的な型情報に基づいた演算を提供します。

責務

  • **I/O 変換**: 文字列（multirange_in/out）およびバイナリ（multirange_recv/send）形式の相互変換。
  • **データ管理**: 複数のレンジのソートとマージ（multirange_canonicalize）、およびシリアライズされたデータ構造の構築・走査。
  • **演算実装**: 集合演算（和、差、積）、包含判定、比較、および集約関数（range_agg, multirange_agg）の実装。
  • **キャッシュ管理**: MultirangeIOData を用いた型情報の効率的な取得と保持。

重要なシンボル

データ構造

  • MultirangeType: マルチレンジのメモリレイアウトを定義する構造体。
  • MultirangeIOData: I/O 関数で利用されるキャッシュ構造体。typcache と I/O プロシージャを保持。
  • MultirangeParseState: 文字列パース時の状態遷移マシン。

シリアライズ・デシリアライズ

  • make_multirange: レンジの配列から正規化された MultirangeType を生成。
  • multirange_deserialize: MultirangeType を個別の RangeType 配列に分解。
  • write_multirange_data: 内部レイアウト（ヘッダー、アイテム、フラグ、境界値）への書き込み。

内部演算・走査

  • multirange_canonicalize: 入力レンジのソートと重複・隣接レンジのマージ。
  • multirange_get_bounds: 指定インデックスのレンジ境界を抽出。
  • multirange_bsearch_match: 二分探索による包含判定の汎用ルーチン。

制御フローとデータフロー

1. **入力**: multirange_in が文字列をパースし、個別の RangeType を生成。make_multirange を呼び出し、multirange_canonicalize で正規化（ソート・マージ）した後に write_multirange_data でバイナリレイアウトを構築する。 2. **演算**: 演算関数（例: multirange_union）は multirange_deserialize でレンジ配列を取り出し、rangetypes.c の内部関数（range_union_internal 等）を適用して新しいレンジ配列を作成し、再度 make_multirange でシリアライズする。 3. **走査**: multirange_get_bounds は、MultirangeGetItemsPtr 等のマクロを使用して、可変長データ内のオフセットを計算し、境界値に直接アクセスする。

直接依存

  • utils/rangetypes.h: レンジ型の基本定義と操作関数。
  • utils/typcache.h: 型情報の動的キャッシュ。
  • utils/array.h: 集約関数での配列構築用。
  • libpq/pqformat.h: バイナリ I/O 用。

関連コード

  • src/backend/utils/adt/rangetypes.c: 単一レンジ型の演算ロジック。
  • src/include/utils/multirangetypes.h: MultirangeType 構造体定義。
  • src/backend/utils/cache/typcache.c: 型情報のキャッシュ管理。

読者向けメモ

  • **メモリレイアウト**: マルチレンジは MultirangeType ヘッダー、レンジアイテム（オフセット/長さ）、フラグ、境界値の順に並ぶ可変長構造です。MultirangeGetBoundariesPtr マクロによるアライメント計算が重要です。
  • **正規化**: make_multirange は常に multirange_canonicalize を呼び出し、入力レンジをソート・マージします。このため、マルチレンジの内部表現は常に「重複なし・隣接なし」の正規形であることが保証されます。
  • **パフォーマンス**: 二分探索（multirange_bsearch_match）を多用しており、レンジ数が多い場合でも $O(\log N)$ での検索が可能です。
  • **未実装/浅い部分**: 本ファイルは主に multirange 型の演算に特化しており、個々のレンジ型の詳細な演算ロジックは rangetypes.c に委譲されています。
