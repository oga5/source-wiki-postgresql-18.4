src/backend/utils/adt/float.c

概要

PostgreSQL の組み込み浮動小数点型（float4, float8）に対する SQL レベルの演算、変換、および集約関数の実装を担うモジュールです。IEEE 754 浮動小数点数のプラットフォーム非依存な挙動を保証し、算術演算、三角関数、統計集約、および型変換機能を提供します。

モジュール内での役割

src/backend/utils/adt サブシステムの一部として、数値計算のコアロジックを提供します。float.h で定義されたインターフェースを実装し、SQL クエリ実行時の演算子（+, -, *, /）や関数（sin, sqrt, avg 等）のバックエンド処理を完結させます。

責務

  • **I/O 変換**: 文字列とバイナリ形式間の相互変換（in, out, recv, send）。
  • **算術・数学演算**: 基本的な四則演算、超越関数（三角関数、指数、対数、ガンマ関数）。
  • **比較とソート**: B-tree インデックス構築用の比較関数および SortSupport 実装。
  • **統計集約**: AVG, STDDEV, VARIANCE, REGR_* 等の集約関数における、数値的に安定した累積計算（Youngs-Cramer アルゴリズム）。
  • **エラーハンドリング**: 浮動小数点例外（オーバーフロー、アンダーフロー、ゼロ除算）の SQLSTATE 準拠した報告。

重要なシンボル

設定・定数

  • extra_float_digits: 出力精度を制御する GUC パラメータ。
  • degree_c_*: 三角関数の精度を保証するために使用される定数群。

入出力・変換

  • float4in, float8in: 文字列から浮動小数点数への変換。strtof/strtod をラップし、プラットフォーム間の差異を吸収。
  • float4out, float8out: 浮動小数点数から文字列への変換。shortest_decimal 形式を優先。
  • ftod, dtof, dtoi*, ftoi*: 型キャスト関数。

算術・数学関数

  • float*pl, float*mi, float*mul, float*div: 基本演算子。
  • dpow, dexp, dlog1, dsin, dcos, dtan: 超越関数群。
  • dgamma, dlgamma: ガンマ関数関連。

集約関数

  • float8_accum, float4_accum: 集約の遷移関数。Youngs-Cramer アルゴリズムにより N, Sx, Sxx を更新。
  • float8_combine: 並列集約時に中間状態をマージする関数。
  • float8_avg, float8_var_*, float8_stddev_*: 集約の最終計算関数。

制御フローとデータフロー

1. **入力**: _in 関数が文字列を解析し、strtod/strtof を使用して内部表現へ変換。エラー時は ereturn を介して SQL レベルのエラーを発生。 2. **演算**: PG_FUNCTION_ARGS を通じて Datum を受け取り、FPU を利用して計算。結果が無限大や NaN になる場合、float_overflow_error 等のヘルパーでエラーを報告。 3. **集約**: 遷移データとして float8[3]（N, Sx, Sxx）または float8[6]（回帰用）の配列を使用。_accum 関数が新しい値を加えるたびに、数値的に安定した更新式を適用。 4. **出力**: _out 関数が extra_float_digits に基づき、shortest_decimal または pg_strfromd を用いて文字列化。

直接依存

  • math.h: 標準数学関数。
  • common/int.h: オーバーフローチェック付き算術。
  • common/shortest_dec.h: 高精度な浮動小数点出力用。
  • utils/array.h: 集約用配列操作。
  • utils/float.h: 公開インターフェース定義。

関連コード

  • src/include/utils/float.h: 本ファイルの関数プロトタイプとインラインヘルパー。
  • src/backend/utils/adt/numeric.c: 任意精度数値型（numeric）の実装。
  • src/backend/utils/sort/tuplesort.c: SortSupport を利用したソート処理。

読者向けメモ

  • **数値的安定性**: 集約関数では、単純な sum(x^2) ではなく Youngs-Cramer アルゴリズムを採用しており、丸め誤差を最小化しています。
  • **プラットフォーム非依存性**: strtod や pow の挙動は OS/コンパイラにより異なるため、float*in_internal や dpow 内で明示的な NaN/Inf チェックやエラーハンドリングを行っています。
  • **三角関数の定数**: init_degree_constants は、コンパイラによる定数式の事前計算（およびそれによる精度の不一致）を防ぐための工夫です。
  • **省略された詳細**: 本ファイルは非常に広範な関数群を含むため、個別の数学関数の実装詳細（POSIX 準拠のための細かい分岐など）は省略しています。これらは主に math.h のラッパーとして機能します。
