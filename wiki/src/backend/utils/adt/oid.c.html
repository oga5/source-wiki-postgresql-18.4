src/backend/utils/adt/oid.c

概要

PostgreSQLの組み込み型である Oid（Object Identifier）および、その配列型である oidvector の入出力処理、比較演算、およびユーティリティ関数を定義するソースファイルです。

モジュール内での役割

src/backend/utils/adt モジュールの一部として、システムカタログや内部識別子として多用される Oid 型の基本的なデータ型としての振る舞いを提供します。

責務

  • Oid 型のテキスト形式およびバイナリ形式の相互変換（I/Oルーチン）。
  • oidvector 型の構築、検証、およびI/O処理。
  • Oid および oidvector に対する比較演算子（等価、大小比較など）の実装。
  • 構文解析時のノードから Oid を抽出するヘルパー関数の提供。

重要なシンボル

  • oidin / oidout: Oid 型のテキスト入出力。
  • oidrecv / oidsend: Oid 型のバイナリ入出力。
  • oidvector: Oid の固定長配列を表現する構造体。
  • buildoidvector: Oid の配列から oidvector を構築する関数。
  • check_valid_oidvector: oidvector の制約（1次元、NULL不可など）を検証する関数。
  • oid_cmp: qsort 等で使用可能な Oid 用の比較関数。

追加の索引済みシンボル

  • oidparse
  • oidvectorin
  • oidvectorout
  • oidvectorrecv
  • oidvectorsend

制御フローとデータフロー

  • **I/O処理**: PG_FUNCTION_ARGS マクロを介してデータを受け取り、uint32in_subr や pq_getmsgint などの共通関数を用いて変換を行います。
  • **oidvectorの検証**: oidvector は汎用的な配列型からキャストされる可能性があるため、check_valid_oidvector を通じて、次元数やNULLの有無が期待通りであるかを厳格にチェックします。
  • **比較演算**: oideq などの単体比較は直接的な比較を行い、oidvector の比較は btoidvectorcmp を呼び出して結果を判定します。

関連コード

  • src/include/nodes/nodes.h: oidparse で使用されるノードタグ定義。

  • src/backend/utils/adt/int.c: uint32in_subr など、数値変換の基盤となる実装。

読者向けメモ

  • oidvector は歴史的な経緯から、一般的な配列型とは異なり「インデックスの下限が0」である点に注意してください。
  • oidvectorrecv 内では、array_recv を呼び出すために InitFunctionCallInfoData を用いて一時的な関数呼び出し情報を構築する工夫がなされています。
  • oidparse はパーサーが生成したノードから Oid を取り出す際に使用されますが、T_Float 型のノードも許容する柔軟な実装になっています。
