src/backend/utils/adt/pg_locale.c

概要

PostgreSQL のロケールおよび照合順序（Collation）処理の基盤となるユーティリティ群です。データベース作成時に固定される LC_COLLATE / LC_CTYPE と、実行時に変更可能な LC_MESSAGES / LC_MONETARY / LC_NUMERIC / LC_TIME の管理、および ICU/libc/組み込みプロバイダ間の抽象化レイヤーを提供します。

モジュール内での役割

src/backend/utils/adt 内で、文字列の比較、変換、書式設定を行う際の「ロケール依存の振る舞い」を決定する中心的な役割を担います。各データ型（text, numeric, money 等）は、本ファイルが提供する pg_locale_t 構造体を通じて、プロバイダ（libc, ICU, builtin）を意識せずにロケール操作を実行できます。

責務

  • **ロケール管理**: setlocale のラッパー（pg_perm_setlocale）による環境変数の整合性維持と、Windows 特有のロケール名変換。
  • **照合順序キャッシュ**: pg_collation カタログに基づく pg_locale_t の生成と、simplehash を用いたバックエンド生存期間中のキャッシュ管理。
  • **プロバイダ抽象化**: libc, ICU, 組み込みプロバイダの差異を吸収し、文字列操作（大文字小文字変換、照合、変換）の統一インターフェースを提供。
  • **書式設定キャッシュ**: LC_TIME や lconv（通貨・数値書式）の情報をキャッシュし、高コストなロケール切り替えを抑制。

重要なシンボル

ロケール管理・キャッシュ

  • pg_newlocale_from_collation(Oid): 指定された OID に対応する pg_locale_t を取得（キャッシュ利用）。
  • pg_perm_setlocale(int, const char*): 安全な setlocale ラッパー。
  • PGLC_localeconv(): 数値・通貨書式情報の取得とキャッシュ。
  • cache_locale_time(): 日時書式情報のキャッシュ更新。

文字列操作インターフェース

  • pg_strcoll, pg_strncoll: ロケール依存の文字列比較。
  • pg_strnxfrm, pg_strnxfrm_prefix: 比較キーへの変換（インデックス作成等で使用）。
  • pg_strlower, pg_strupper, pg_strtitle, pg_strfold: 大文字小文字変換。

内部構造体

  • struct pg_locale_struct: ロケールプロバイダ、決定論的フラグ、メソッドテーブルを保持する主要構造体。
  • collation_cache_entry: simplehash で管理されるキャッシュエントリ。

制御フローとデータフロー

1. **初期化**: init_database_collation がデータベースのデフォルトロケールを TopMemoryContext にロード。 2. **キャッシュ検索**: pg_newlocale_from_collation が呼ばれると、まず last_collation_cache_oid を確認し、ミスなら CollationCache（ハッシュテーブル）を検索。 3. **ロケール生成**: キャッシュミス時、create_pg_locale が pg_collation カタログを読み込み、プロバイダ（builtin, icu, libc）に応じて適切な初期化関数を呼び出す。 4. **操作実行**: pg_strcoll 等の関数が pg_locale_t 内のメソッドテーブル（collate_methods）を介して、各プロバイダの実装（strncoll_icu 等）へディスパッチする。

読者向けメモ

  • **メモリ管理**: pg_locale_t は CollationCacheContext（TopMemoryContext 配下）で確保され、バックエンド終了まで解放されません。freelocale() を直接呼んではいけません。
  • **Windows 対応**: Windows は LC_MESSAGES の扱いが特殊であり、IsoLocaleName 等の変換ロジックが複雑に含まれています。
  • **省略された詳細**: 本ファイルは各プロバイダの具体的な変換ロジック（strlower_icu 等）の呼び出し元であり、個別のアルゴリズム詳細は各 pg_locale_*.c ファイルに委譲されています。
  • **安全性**: pg_strnxfrm 等はプロバイダによって安全性が異なるため、必ず pg_strxfrm_enabled() で確認してから使用してください。
