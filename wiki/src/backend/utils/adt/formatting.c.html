src/backend/utils/adt/formatting.c

概要

本ファイルは、PostgreSQL におけるデータ型（Timestamp, Numeric, int, float 等）のフォーマット変換関数（to_char, to_timestamp, to_date, to_number）および文字列の大文字・小文字変換関数を実装するモジュールです。Oracle の同名ルーチンに触発された設計となっており、複雑なフォーマット文字列を解析・キャッシュし、内部表現と文字列間の相互変換を行います。

モジュール内での役割

src/backend/utils/adt サブシステムの一部として、SQL レベルの型変換関数を提供します。特に日時型と数値型のシリアライズ・デシリアライズにおいて、ロケールやフォーマット指定子を考慮した柔軟な変換ロジックを担います。

責務

  • **フォーマット解析**: SQL で指定されたフォーマット文字列を FormatNode ツリーに変換し、効率的な変換処理を可能にする。
  • **変換処理**: 内部バイナリ表現（TmToChar 等）から文字列への変換（to_char）、および文字列から内部表現への逆変換（to_timestamp 等）。
  • **キャッシュ管理**: 解析済みのフォーマットツリーを DCHCache および NUMCache に保持し、トランザクションを跨いで再利用する。
  • **文字列操作**: ロケールを考慮した大文字・小文字変換（lower, upper, initcap）の提供。

重要なシンボル

フォーマット解析・キャッシュ

  • FormatNode: フォーマット文字列を分解した最小単位（アクション、文字、セパレータ等）。
  • DCHCacheEntry / NUMCacheEntry: 解析済みフォーマットのキャッシュ構造体。age フィールドによる LRU 的なキャッシュ管理を行う。
  • parse_format(): フォーマット文字列を FormatNode のリストに変換するパーサ。

日時変換

  • TmToChar: to_char 処理用の内部日時構造体。fmt_tm をラップし、タイムゾーンや小数秒を保持。
  • TmFromChar: to_timestamp 等の逆変換用構造体。解析された各フィールド（年、月、日、時、分、秒、TZ 等）を保持。
  • do_to_timestamp(): 逆変換のコアロジック。文字列を解析し、TmFromChar を経由して pg_tm に変換する。

数値変換

  • NUMDesc: 数値フォーマットの属性（精度、スケール、符号、ロケール設定等）を保持する記述子。
  • NUMProc: to_number / to_char 実行時の状態を管理するプロセッサ構造体。
  • NUM_processor(): 数値変換のメインループ。フォーマットノードに従い、数値と文字列を相互に走査・変換する。

制御フローとデータフロー

1. **解析フェーズ**: parse_format() がフォーマット文字列を走査し、KeyWord（DCH_keywords / NUM_keywords）と KeySuffix を照合して FormatNode 配列を構築する。 2. **キャッシュ**: DCH_cache_fetch() / NUM_cache_fetch() がキャッシュを検索し、ヒットしなければ解析を実行してキャッシュに保存する。 3. **変換フェーズ**:

  • to_char: DCH_to_char() が FormatNode を順次走査し、TmToChar の値を文字列バッファに書き出す。
  • to_number: NUM_processor() が FormatNode を走査し、入力文字列から数値を抽出し、NUMDesc に基づいて numeric 型へ変換する。

    • 4. **逆変換**: do_to_timestamp() が DCH_from_char() を呼び出し、入力文字列を TmFromChar に分解した後、最終的な日時型（Timestamp 等）を構築する。

読者向けメモ

  • **メモリ管理**: キャッシュされた FormatNode は TopMemoryContext に割り当てられ、永続的です。キャッシュサイズは DCH_CACHE_SIZE 等の定数で制限されています。
  • **ロケール**: str_tolower 等の関数は、collid を受け取り、C/POSIX ロケールとそれ以外で処理パスを切り替えます。
  • **制限事項**: to_char の数値変換において、EEEE（科学的表記）は他のフォーマット指定子と併用できない制限があります。
  • **未実装**: 数値のスペルアウト（TO_CHAR(123, 'SP')）などは未実装です。
  • **注意**: DCH_from_char 等の逆変換関数は、入力文字列の解析において ErrorSaveContext を介したソフトエラー処理をサポートしており、プランナやバリデーションでの利用を想定しています。
