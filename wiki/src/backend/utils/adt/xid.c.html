src/backend/utils/adt/xid.c

概要

PostgreSQLのトランザクション識別子（TransactionId / xid）およびコマンド識別子（CommandId / cid）を、SQLレベルで扱うためのデータ型関数群を定義するファイルです。

モジュール内での役割

src/backend/utils/adt モジュールの一部として、システム内部の識別子をユーザー空間（SQL）で利用可能にするための「入出力関数」および「比較・演算関数」を提供します。

責務

  • **型変換**: 文字列（in/out）およびバイナリ形式（recv/send）との相互変換。
  • **比較演算**: xid および xid8（64bit拡張トランザクションID）の等価性、大小比較。
  • **ユーティリティ**: トランザクションの経過時間（age）計算や、ソート用の比較関数（qsort用）の提供。
  • **ハッシュ関数**: ハッシュインデックスやハッシュ結合で使用されるハッシュ値の計算。

重要なシンボル

  • xid / xid8 関連関数: SQL型 xid および xid8 の操作用関数群。
  • cid 関連関数: SQL型 cid（コマンドID）の操作用関数群。
  • xidComparator / xidLogicalComparator: ソート時に使用される比較関数。
  • xid_age / mxid_age: トランザクションの経過時間を計算する関数。

制御フローとデータフロー

  • **入出力**: PG_GETARG_* マクロで引数を受け取り、PG_RETURN_* マクロで結果を返します。文字列変換には uint32in_subr 等の共通ヘルパーを利用します。
  • **バイナリ通信**: libpq/pqformat.h の StringInfo を介して、ネットワークバイトオーダーを意識したバイナリデータの送受信を行います。
  • **比較**: TransactionId や FullTransactionId の比較ロジックは、access/transam.h 等で定義されたプリミティブに依存しています。

読者向けメモ

  • **比較の注意**: xid は循環する可能性があるため、単純な大小比較ではなく TransactionIdPrecedes 等の専用関数を使用する必要があります。xidComparator は単なるソート用であり、論理的な前後関係とは異なる場合があります。
  • **xid8**: PostgreSQL 14以降で導入された64bitトランザクションID（FullTransactionId）のサポートが充実しており、新しいコードではこちらが推奨されます。
  • **コマンドID**: CommandId は単なる uint32 であり、現在のトランザクション内でのコマンド順序を追跡するために使用されます。
