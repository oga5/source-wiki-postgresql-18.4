src/backend/utils/adt/jsonfuncs.c

概要

jsonfuncs.c は、PostgreSQL の json および jsonb データ型に対する SQL レベルの操作関数を実装する中心的なファイルです。JSON 文書の走査、パス抽出、要素の取得、およびレコード型へのマッピング（populate_record 等）といった高度な処理を担います。

モジュール内での役割

src/backend/utils/adt モジュールにおいて、JSON 関連のロジックを抽象化・提供します。json.c や jsonb.c が低レベルなバイナリ表現やシリアライズを扱うのに対し、本ファイルは SQL 関数インターフェース（PG_FUNCTION_ARGS）と JSON 構造の橋渡しを行います。

責務

  • **JSON 走査と抽出**: ->, ->>, #>, #>> 演算子に対応するパス抽出ロジック。
  • **レコード変換**: JSON/JSONB を PostgreSQL のレコード型（RECORD）や複合型へ変換する populate_record 関連機能。
  • **集合操作**: json_each, json_array_elements 等のセットを返す関数（SRF）の実装。
  • **JSON 変換・加工**: json_strip_nulls や値の変換（transform_json_string_values）などのユーティリティ。
  • **エラーハンドリング**: JSON 解析時のコンテキスト報告および ErrorSaveContext を用いたソフトエラー処理。

重要なシンボル

状態管理構造体

  • GetState: json_get* 関数群の走査状態を保持。
  • PopulateRecordsetState: json_populate_recordset 等の SRF における状態管理。
  • PopulateArrayContext: JSON 配列から PostgreSQL 配列を構築する際のメタデータと次元管理。
  • OkeysState, EachState, ElementsState: 各 SRF の実行状態を保持。

変換・キャッシュ構造体

  • ColumnIOData / RecordIOData: 複合型や配列の変換に必要な型情報と I/O 関数をキャッシュする構造体。
  • JsValue / JsObject: json（テキスト）と jsonb（バイナリ）の差異を吸収し、統一的に扱うためのラッパー。

主要関数

  • get_worker: JSON パス抽出の共通ロジック。
  • populate_record_field: JSON 値を特定の PostgreSQL 型へ変換する再帰的関数。
  • setPath: jsonb_set や jsonb_insert におけるパス指定による JSONB 構造の変更。

制御フローとデータフロー

1. **解析**: makeJsonLexContext で入力テキストをトークン化し、JsonSemAction（セマンティックアクション）を定義して pg_parse_json を呼び出します。 2. **走査**: 各 SRF は Tuplestore を使用して結果を蓄積します。jsonb の場合は JsonbIterator を用いて直接走査します。 3. **変換**: populate_record 系関数は、JSON のキーと PostgreSQL の属性名を照合し、prepare_column_cache で取得した型情報に基づき InputFunctionCall を実行して Datum を生成します。 4. **エラー**: pg_parse_json_or_errsave を経由し、escontext が提供されていればエラーを保存、なければ ereport で即時終了します。

読者向けメモ

  • **メモリ管理**: SRF 実装では Tuplestore を使用し、tmp_cxt（一時メモリコンテキスト）を適宜リセットすることでメモリ消費を抑制しています。
  • **再帰**: populate_record_field は再帰的に呼び出されるため、check_stack_depth() によるスタックオーバーフロー保護が行われています。
  • **JSONB 優先**: 近年の PostgreSQL では jsonb が推奨されるため、json（テキスト）版の関数は内部的に jsonb へ変換して処理するケースが増えています。
  • **省略**: 本ファイルは非常に巨大であるため、特定の型変換の詳細ロジックや、jsonb_set 内部の複雑なイテレータ操作の詳細は省略しています。これらは jsonb.c のイテレータ仕様に強く依存しています。
