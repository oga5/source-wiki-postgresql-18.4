src/backend/utils/adt/cash.c

概要

PostgreSQLの通貨型（money）の入出力、算術演算、および型変換を実装するソースファイルです。内部的には64ビット整数（int64）として値を保持し、ロケール設定に基づいて通貨記号や小数点、桁区切りを動的に処理します。

モジュール内での役割

src/backend/utils/adt モジュールの一部として、データ型定義に基づいた演算子関数および変換関数を提供します。cash.h で定義された Cash 型（int64の別名）を操作する中心的な役割を担います。

責務

  • **入出力処理**: ロケール（lc_monetary）に応じた通貨文字列のパース（cash_in）およびフォーマット出力（cash_out）。
  • **算術演算**: 通貨型同士の加減算、および数値型（float8, int系）との乗除算。
  • **型変換**: numeric 型や整数型との相互変換。
  • **比較演算**: 通貨値の大小比較および等価判定。

重要なシンボル

  • Cash: int64 として定義された通貨の内部表現。
  • cash_in / cash_out: 通貨型の入出力関数。ロケール情報を取得してフォーマットを決定します。
  • append_num_word: cash_words 関数で使用される、数値を英語の単語に変換する補助関数。
  • cash_pl_cash / cash_mi_cash: オーバーフローチェック付きの加減算ラッパー。

制御フローとデータフロー

  • **入出力**: PGLC_localeconv() を呼び出し、現在のロケール設定（frac_digits, mon_decimal_point 等）を取得します。cash_in は文字列をパースして int64 に変換し、cash_out は int64 をロケールに従って文字列化します。
  • **演算**: ほとんどの演算は int64 の算術演算に帰着しますが、乗除算では float8 へのキャストと rint() による丸め処理が行われます。すべての演算において、オーバーフローやゼロ除算のチェックが ereport を通じて行われます。
  • **変換**: numeric 型との変換では、ロケールの小数桁数に合わせてスケーリング（10のべき乗の乗算）を行い、精度を維持します。

読者向けメモ

  • **ロケール依存**: cash_in と cash_out は lc_monetary に強く依存します。ロケールが変更されると、同じバイナリ値でも出力結果が変わる可能性がある点に注意してください。
  • **オーバーフロー**: 内部表現が int64 であるため、非常に大きな金額を扱う場合はオーバーフローのリスクがあります。演算関数には pg_add_s64_overflow 等のチェックが組み込まれています。
  • **精度**: 小数点以下の扱いはロケールの frac_digits に依存します。デフォルトは2桁ですが、ロケール設定により変動します。
