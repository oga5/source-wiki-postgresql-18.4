src/backend/utils/adt/mcxtfuncs.c

概要

PostgreSQLのバックエンドにおけるメモリコンテキスト（MemoryContext）の状態を可視化・管理するための関数群を提供します。主にSQLインターフェースを通じて、現在のバックエンドのメモリ使用状況を調査したり、特定のプロセスに対してメモリ状況のログ出力を要求したりするために使用されます。

モジュール内での役割

src/backend/utils/adt ディレクトリの一部として、システム管理やデバッグに役立つデータ型・関数を提供します。メモリ管理サブシステム（src/backend/utils/mmgr）の情報を、SQLからクエリ可能な形式（SRF: Set Returning Function）に変換するブリッジの役割を果たします。

責務

  • **メモリ状況の列挙**: pg_get_backend_memory_contexts を通じて、現在のバックエンドの全メモリコンテキストを階層構造とともに取得し、テーブル形式で返します。
  • **識別子の管理**: コンテキストの親子関係を追跡し、一貫性のある context_id を割り当てます。
  • **シグナル送信**: pg_log_backend_memory_contexts を通じて、指定したPIDのプロセスに対し、メモリコンテキストのログ出力を促すシグナル（PROCSIG_LOG_MEMORY_CONTEXT）を送信します。

重要なシンボル

  • pg_get_backend_memory_contexts: メモリコンテキストの統計情報を返すSRF。
  • pg_log_backend_memory_contexts: 指定プロセスにメモリログ出力を指示する関数。
  • MemoryContextId: コンテキストとIDを紐付けるための内部構造体。
  • MEMORY_CONTEXT_IDENT_DISPLAY_SIZE: ログや表示用に切り詰められる識別子の最大長（1024バイト）。

追加の索引済みシンボル

  • PutMemoryContextsStatsTupleStore
  • int_list_to_array

制御フローとデータフロー

1. **列挙**: pg_get_backend_memory_contexts は TopMemoryContext を起点に、幅優先探索（BFS）で全コンテキストを走査します。 2. **ID割り当て**: 走査中にハッシュテーブル（context_id_lookup）を使用して各コンテキストに一意のIDを割り当てます。これにより、親子関係のパスを安定して表現します。 3. **統計取得**: 各コンテキストに対して context->methods->stats を呼び出し、使用量や空き領域を取得します。 4. **出力**: 取得した情報を Tuplestore に格納し、SQLの結果セットとして返却します。

読者向けメモ

  • **BFSの採用理由**: コード内のコメントにある通り、深さ優先ではなく幅優先でIDを割り当てることで、コンテキストの増減によるIDの変動を最小限に抑え、安定した出力を提供する工夫がなされています。
  • **セキュリティ**: pg_log_backend_memory_contexts は、DoS攻撃を防ぐため、デフォルトでスーパーユーザー権限を必要とします。
  • **識別子の切り詰め**: ident フィールド（SQLクエリ文字列など）は非常に長くなる可能性があるため、表示時には pg_mbcliplen を用いて安全に切り詰められます。
