src/backend/utils/adt/inet_net_pton.c

概要

ネットワークアドレス（IPv4/IPv6）を、人間が読みやすい形式（Presentation）から、システム内部で扱うネットワークバイトオーダーのバイナリ形式（Network）へ変換するユーティリティです。

モジュール内での役割

src/backend/utils/adt 内でネットワーク型（inet, cidr）を扱うための低レベルな変換処理を提供します。PostgreSQL のネットワーク型データ構造の構築やパース処理の基盤となります。

責務

  • プレゼンテーション形式（文字列）からバイナリ形式への変換。
  • CIDR 表記（例: 192.168.1.0/24）のパースとビット長（プレフィックス長）の抽出。
  • IPv4/IPv6 それぞれのクラスフル/クラスレスなアドレス指定の解釈。
  • 不正なフォーマットに対するエラーハンドリング（errno の設定）。

重要なシンボル

  • pg_inet_net_pton: 公開されているメインのエントリポイント。アドレスファミリーに応じて適切な変換関数へ振り分けます。
  • inet_cidr_pton_ipv4 / inet_cidr_pton_ipv6: CIDR 表記を含む文字列をパースする中核関数。
  • inet_net_pton_ipv4 / inet_net_pton_ipv6: CIDR 表記がない、あるいは省略された場合の変換関数。

追加の索引済みシンボル

  • getbits
  • getv4

制御フローとデータフロー

1. **エントリ**: pg_inet_net_pton が呼び出され、af（アドレスファミリー）と size（バッファサイズ）を評価します。 2. **振り分け**: size が -1 の場合はホストアドレス用、それ以外は CIDR ネットワーク用として、IPv4 または IPv6 の各関数へ処理を委譲します。 3. **パース**: 各関数は文字列を走査し、10進数（IPv4）または16進数（IPv6）のオクテットをバイナリに変換します。 4. **CIDR 処理**: / 以降の数値を読み取り、ビット長を確定させます。指定がない場合は、IPv4 のクラスフルルールに基づいてデフォルトのビット長を推論します。 5. **完了**: 変換されたバイナリデータを dst に格納し、確定したビット長を返します。

関連コード

  • src/backend/utils/adt/network.c: 本ファイルでパースされたデータを利用して、PostgreSQL の inet 型や cidr 型の演算を行う上位層。
  • src/backend/utils/adt/inet_cidr_ntop.c: 逆方向（バイナリから文字列）の変換を行う対となるファイル。

読者向けメモ

  • 本ファイルは ISC ライセンスに基づく外部由来のコードを含んでおり、PostgreSQL 固有のメモリ管理（palloc 等）ではなく、標準的な errno を用いたエラー制御を行っています。
  • IPv6 のパース処理では、::（ゼロ圧縮）の展開や、IPv4 互換アドレスの処理など、複雑なエッジケースが含まれています。
  • size 引数に -1 を渡す挙動は、このモジュール特有のインターフェース設計です。利用時は呼び出し元の意図を確認してください。
