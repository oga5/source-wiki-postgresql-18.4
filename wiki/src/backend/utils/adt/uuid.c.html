src/backend/utils/adt/uuid.c

概要

PostgreSQLにおける uuid 型のデータ型定義および関連する演算子、関数を実装するモジュールです。RFC 9562 に準拠した UUID の生成（v4, v7）や、型変換、比較、ソートサポートを提供します。

モジュール内での役割

src/backend/utils/adt ディレクトリにおいて、組み込みデータ型 uuid の実体として機能します。uuid.h で定義された pg_uuid_t 構造体を操作し、SQL レベルの関数と C 言語のロジックを橋渡しします。

責務

  • **型変換**: 文字列表現（uuid_in, uuid_out）およびバイナリ表現（uuid_recv, uuid_send）の相互変換。
  • **比較演算**: B-tree インデックス等で使用される比較関数（uuid_eq, uuid_lt 等）の提供。
  • **UUID 生成**: 乱数ベースの v4 および時刻ベースの v7 UUID の生成。
  • **最適化**: ソート時のパフォーマンス向上のための SortSupport および SkipSupport の実装。
  • **メタデータ抽出**: UUID からバージョンやタイムスタンプを抽出する機能。

重要なシンボル

  • pg_uuid_t: UUID を保持する 16 バイトの構造体。
  • uuid_in / uuid_out: SQL 型と文字列間の入出力ハンドラ。
  • gen_random_uuid: v4 UUID を生成する SQL 関数。
  • uuidv7: 現在時刻に基づき v7 UUID を生成する SQL 関数。
  • uuid_sortsupport: ソート時の比較を高速化するためのフック関数。

制御フローとデータフロー

  • **UUID 生成**: get_real_time_ns_ascending() で単調増加するナノ秒単位の時刻を取得し、generate_uuidv7() で RFC 9562 に基づくビット配置と乱数付与を行い、pg_uuid_t を構築します。
  • **ソート**: uuid_sortsupport が呼び出されると、uuid_abbrev_convert を通じて UUID の先頭バイトを Datum（整数）に変換し、比較コストを削減します。
  • **比較**: uuid_internal_cmp が memcmp を用いて 16 バイトのバイナリ比較を直接行います。

読者向けメモ

  • **単調性**: uuidv7 はシステム時刻に依存しますが、get_real_time_ns_ascending により同一トランザクション内や高頻度生成時でも単調増加が保証される設計になっています。
  • **プラットフォーム依存**: SUBMS_MINIMAL_STEP_BITS は OS の時刻精度（macOS 等のマイクロ秒制限）を考慮して動的に調整されます。
  • **ソート最適化**: uuid_abbrev_convert は Datum へのキャストを利用した高速化を行っていますが、リトルエンディアン環境ではバイトスワップが必要である点に注意してください。
