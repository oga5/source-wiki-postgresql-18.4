src/backend/utils/adt/arrayfuncs.c

概要

PostgreSQL の配列型（ARRAY）に対するコアサポート関数群を実装するファイルです。配列の内部表現（ArrayType）の構築、分解、入出力（テキスト/バイナリ）、要素へのアクセス、スライス操作、および配列同士の比較・演算を担います。

モジュール内での役割

src/backend/utils/adt における配列型操作の心臓部です。SQL レベルの配列操作関数（array_in, array_out, array_get_element 等）と、内部的なメモリレイアウト操作を繋ぐレイヤーとして機能します。

責務

  • **I/O 変換**: 配列リテラル文字列のパース（array_in）および文字列化（array_out）、バイナリ形式の送受信（array_recv, array_send）。
  • **要素・スライス操作**: 特定要素の取得・更新、およびスライス（部分配列）の抽出・挿入。
  • **配列構築・分解**: construct_array 等による配列の動的生成と、deconstruct_array による要素への分解。
  • **比較・演算**: 配列同士の等価性判定、大小比較、ハッシュ計算、および unnest や width_bucket 等のユーティリティ関数。
  • **反復処理**: ArrayIterator を用いた配列要素の効率的な走査。

重要なシンボル

配列構築・分解

  • construct_array / construct_md_array: 配列の Datum 配列から ArrayType を生成。
  • deconstruct_array: ArrayType を Datum 配列に分解。
  • ArrayBuildState / ArrayBuildStateArr: 集約関数等で配列を段階的に構築するための作業構造体。

要素・スライス操作

  • array_get_element / array_set_element: 単一要素の読み書き。
  • array_get_slice / array_set_slice: 部分配列の読み書き。

反復処理

  • ArrayIteratorData: 配列走査の状態を保持する構造体。
  • array_create_iterator / array_iterate: 配列を要素単位またはスライス単位で走査するための API。

I/O 処理

  • array_in / array_out: テキスト形式の入出力。
  • array_recv / array_send: バイナリ形式の入出力。

制御フローとデータフロー

1. **入力**: array_in は ReadArrayDimensions で次元情報を読み取り、ReadArrayStr で再帰的にトークンを解析して Datum 配列を構築します。 2. **構築**: CopyArrayEls が Datum 配列を ArrayType のメモリレイアウト（ヘッダー、NULL ビットマップ、データ領域）へコピーします。 3. **アクセス**: array_seek が NULL ビットマップを考慮しながらデータ領域のオフセットを計算し、目的の要素位置を特定します。 4. **更新**: array_set_element 等は、変更後のサイズを計算し、新しい ArrayType を palloc してデータを再配置する（Expanded Array の場合はインプレース更新を試みる）フローをとります。

読者向けメモ

  • **メモリレイアウト**: 配列は ArrayType ヘッダー、NULL ビットマップ（存在する場合）、データ領域の順で構成されます。ARR_DATA_PTR マクロ等を使用してアクセスする必要があります。
  • **メモリ所有権**: palloc されたメモリは呼び出し元が管理します。CopyArrayEls 等の関数は、渡された Datum 配列の所有権を適切に扱う必要があります。
  • **Expanded Array**: VARATT_IS_EXTERNAL_EXPANDED をチェックする箇所があり、メモリ上で展開された配列（ExpandedArrayHeader）に対する最適化パスが存在します。
  • **NULL 処理**: 配列内の NULL はビットマップで管理されます。array_get_isnull / array_set_isnull を介して操作してください。
  • **未実装/浅い部分**: 固定長配列（typlen > 0）に対するスライス操作など、一部の機能は制限されています。また、多次元配列の拡張（array_set_element）は限定的なサポートに留まっています。
