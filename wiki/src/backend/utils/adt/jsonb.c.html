src/backend/utils/adt/jsonb.c

概要

jsonb.c は、PostgreSQL の jsonb 型に対する入出力（I/O）処理、型変換、および集約関数の実装を担う中心的なソースファイルです。JSON テキストからバイナリ形式へのパース、バイナリからテキストへのシリアライズ、および SQL レベルでのデータ型変換（to_jsonb やキャスト）を統括します。

モジュール内での役割

src/backend/utils/adt モジュールにおいて、jsonb 型のライフサイクル管理と SQL インターフェースを提供します。jsonb.h で定義されたバイナリ構造を操作するための高レベルな API を提供し、jsonfuncs.c が提供する走査・検索機能と連携して JSONB の全機能を実現します。

責務

  • **I/O 処理**: SQL 文字列と jsonb バイナリ間の相互変換（in, out, recv, send）。
  • **構築・集約**: jsonb_build_object, jsonb_agg 等の SQL 関数を通じた動的な JSONB 構築。
  • **型変換**: 任意の SQL 型から jsonb への変換、および jsonb からスカラ型（数値、bool 等）へのキャスト。
  • **パース状態管理**: JsonbInState を用いたパース中のツリー構築とバリデーション。

重要なシンボル

パース・構築状態管理

  • struct JsonbInState: パース中の状態を保持する構造体。JsonbParseState をラップし、一意なキーの制約やエラーハンドリングを管理。
  • struct JsonbAggState: 集約関数（jsonb_agg 等）の実行中に累積状態を保持する構造体。

パース・変換関数

  • jsonb_from_cstring: JSON 文字列を jsonb バイナリへ変換するメインエントリーポイント。
  • datum_to_jsonb_internal: 任意の Datum を jsonb 構造へ再帰的に変換するコアロジック。
  • JsonbToCStringWorker: jsonb バイナリを文字列へシリアライズする汎用ワーカー。

SQL インターフェース

  • jsonb_in, jsonb_out: 標準的な型 I/O 関数。
  • jsonb_typeof: JSONB コンテナの型（object, array, number 等）を返す関数。
  • jsonb_agg_transfn / jsonb_object_agg_transfn: 集約関数の遷移関数。

制御フローとデータフロー

1. **パース**: jsonb_in が jsonb_from_cstring を呼び出し、JsonLexContext を用いてトークンを順次処理します。各トークンは jsonb_in_object_start 等のコールバックを通じて pushJsonbValue に渡され、バイナリツリーが構築されます。 2. **構築**: datum_to_jsonb_internal は再帰的に呼び出され、複合型や配列を走査して JsonbInState に要素を追加します。 3. **集約**: jsonb_agg 等の集約関数は、JsonbAggState を使用してメモリコンテキストを切り替えながら、各行の値を累積します。最終的に finalfn が WJB_END_ARRAY を付与して完成させます。

直接依存

  • utils/jsonb.h: Jsonb バイナリ構造とイテレータの定義。
  • utils/json.h: JSON パース用レキサーとセマンティックアクションの定義。
  • utils/typcache.h: 型情報の動的キャッシュ（json_categorize_type 等）。

関連コード

  • src/include/utils/jsonb.h: バイナリレイアウトの仕様と定数。
  • src/backend/utils/adt/jsonfuncs.c: JSONB の検索、抽出、パス操作関数。
  • src/backend/utils/adt/jsonpath_exec.c: JSONPath を用いた高度なクエリ実行。

読者向けメモ

  • **メモリ管理**: jsonb の構築には JsonbInState を使用し、パース終了時に JsonbValueToJsonb で最終的なバイナリを生成します。集約関数では MemoryContext の切り替えが頻繁に行われるため、メモリリークやコンテキストの不整合に注意が必要です。
  • **制約**: checkStringLen により、JSONB 内の文字列長には JENTRY_OFFLENMASK による制限があります。
  • **省略された詳細**: 本ファイルは I/O と構築に特化しており、JSONB のインデックス（GIN）の内部実装や、JSONPath の実行ロジックは別ファイル（jsonfuncs.c 等）に委譲されています。
  • **不変性**: to_jsonb_is_immutable は、特定の型が JSONB 変換後に不変であるかを判定しますが、現在は再帰的な走査が不完全である点に注意してください。
