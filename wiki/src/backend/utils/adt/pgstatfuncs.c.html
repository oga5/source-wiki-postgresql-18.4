src/backend/utils/adt/pgstatfuncs.c

概要

PostgreSQL の統計情報システム（pgstat）が収集したデータを、SQL インターフェース（システムビューや関数）を通じてユーザーに提供するためのバックエンド実装です。統計情報の取得、リセット、および特定のバックエンドやオブジェクトに関する詳細な状態情報の変換を担います。

モジュール内での役割

src/backend/pgstat/ サブシステムと SQL レイヤーの仲介役です。pgstat が共有メモリやローカルメモリに保持している統計構造体を、SQL の Datum 型や Tuple 形式に変換して返却します。

責務

  • **統計情報の公開**: テーブル、インデックス、関数、データベース、バックエンド、WAL、レプリケーションスロット等の統計情報を SQL 関数として提供。
  • **権限管理**: HAS_PGSTAT_PERMISSIONS マクロを用い、統計情報へのアクセス権限（pg_read_all_stats ロールまたは対象オブジェクトの所有者）を検証。
  • **統計情報のリセット**: 統計カウンターの初期化・リセット処理のディスパッチ。
  • **データ変換**: 内部の統計構造体（PgStat_StatTabEntry 等）から SQL で利用可能な型（int64, float8, timestamptz）へのマッピング。

重要なシンボル

統計取得関数（マクロ生成）

  • PG_STAT_GET_RELENTRY_INT64: テーブル統計（analyze_count 等）を int64 で取得する関数群を生成。
  • PG_STAT_GET_RELENTRY_FLOAT8: 時間統計（total_vacuum_time 等）を float8 で取得する関数群を生成。
  • PG_STAT_GET_DBENTRY_INT64: データベース統計（xact_commit 等）を int64 で取得する関数群を生成。

バックエンド・アクティビティ関連

  • pg_stat_get_activity: 全バックエンドの稼働状況（PID, 状態, 待機イベント, クエリ等）を tuplestore に格納して返却。
  • pg_stat_get_progress_info: VACUUM や CREATE INDEX などの進行状況を報告。
  • pg_stat_get_backend_subxact: 特定バックエンドのサブトランザクション情報を取得。

I/O および WAL 統計

  • pg_stat_get_io: 全バックエンドタイプ別の I/O 統計を構築。
  • pg_stat_get_wal: WAL 関連の統計情報を構築。

管理・リセット

  • pg_stat_reset_shared: クラスタ全体の共有統計（bgwriter, checkpointer 等）をリセット。
  • pg_stat_reset_single_table_counters: 特定テーブルの統計をリセット。

制御フローとデータフロー

1. **リクエスト**: SQL 関数が呼び出されると、pgstat_fetch_stat_* 系関数を呼び出し、最新の統計スナップショットを取得します。 2. **権限チェック**: HAS_PGSTAT_PERMISSIONS を使用し、呼び出し元が統計情報を閲覧可能か判定します。権限がない場合、機密情報（クエリ文字列やクライアントアドレス等）は NULL またはマスクされた値に置換されます。 3. **変換**: tuplestore_putvalues を使用して、結果セットを構築します。pg_stat_get_activity のようなセットを返す関数は InitMaterializedSRF で初期化されたコンテキストを使用します。 4. **返却**: 構築された Datum または HeapTuple を SQL エンジンへ返却します。

読者向けメモ

  • **メモリ管理**: tuplestore を使用する関数は、multi_call_memory_ctx を利用してメモリを管理しています。
  • **権限の境界**: pg_stat_get_activity 等では、権限がない場合に一部の列を NULL にする処理がハードコードされています。セキュリティ要件の変更時はここを確認してください。
  • **スナップショット**: pgstat_fetch_stat_* は、呼び出しのたびに最新の統計スナップショットを読み込みます。高頻度な呼び出しはパフォーマンスに影響を与える可能性があります。
  • **省略された詳細**: 本ファイルは統計情報の「表示」に特化しており、統計情報の「収集」ロジック（pgstat_report_* 等）は src/backend/pgstat/ 配下の各ファイルに委譲されています。
