src/backend/utils/adt/rowtypes.c

概要

PostgreSQLにおける「複合型（Composite Types）」および「匿名レコード（RECORD）」の入出力、比較、およびハッシュ計算を実装するモジュールです。

モジュール内での役割

src/backend/utils/adt 内で、ユーザー定義の行型や一時的なレコード構造をSQLレベルで操作するための基盤を提供します。他のデータ型モジュールが単一の値を扱うのに対し、本ファイルは複数のフィールドからなるタプルを再帰的に処理します。

責務

  • **I/O処理**: テキスト形式およびバイナリ形式でのレコードのシリアライズ・デシリアライズ。
  • **比較演算**: 複合型同士の比較（等価性、大小関係）の実行。
  • **ハッシュ計算**: ハッシュ結合やハッシュ集約で使用するためのレコードのハッシュ値生成。
  • **メタデータキャッシュ**: 実行効率向上のため、レコードの構造（TupleDesc）や各フィールドのI/O関数情報を fn_extra にキャッシュする。

重要なシンボル

  • record_in / record_out: テキスト形式の入出力関数。
  • record_recv / record_send: バイナリ形式の入出力関数。
  • record_cmp: 内部的な比較ロジック。record_lt, record_gt 等の基盤。
  • record_eq: 等価性判定。
  • hash_record / hash_record_extended: ハッシュ値生成関数。
  • RecordIOData / RecordCompareData: 実行時の型情報や関数ポインタを保持するキャッシュ構造体。

追加の索引済みシンボル

  • ColumnCompareData
  • ColumnIOData
  • record_ne

制御フローとデータフロー

1. **初期化**: 各関数は lookup_rowtype_tupdesc を呼び出し、対象レコードの型定義（TupleDesc）を取得します。 2. **キャッシュ確認**: fcinfo->flinfo->fn_extra を確認し、既存のキャッシュがあれば再利用します。なければ、各フィールドの型に応じたI/O関数や比較関数をルックアップしてキャッシュします。 3. **処理実行**:

  • 入出力時は heap_form_tuple / heap_deform_tuple を使用してタプルとフィールド配列を変換します。
  • 比較・ハッシュ時は、各フィールドの型に対応する関数を再帰的に呼び出します。

    • 4. **終了**: ReleaseTupleDesc でリソースを解放し、結果を返します。

読者向けメモ

  • **再帰的処理**: レコードの中にレコードが含まれる場合があるため、check_stack_depth() によるスタック保護が行われています。
  • **キャッシュの生存期間**: fn_extra は関数呼び出しのライフサイクルに依存します。型が途中で変更されるケース（ALTER TYPE 等）には注意が必要です。
  • **Dropped Columns**: 削除された列（attisdropped）はスキップされるよう設計されています。比較ロジックでは、物理的な列位置ではなく論理的な列順序で対応付けが行われます。
