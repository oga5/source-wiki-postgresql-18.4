src/backend/utils/adt/numeric.c

概要

numeric.c は、PostgreSQL の NUMERIC 型（任意の精度を持つ数値型）の内部表現、算術演算、および SQL 関数を実装するコアファイルです。この型は、金融計算などで求められる正確な精度を維持するため、内部的に可変長の基数（NBASE=10000）浮動小数点形式で数値を管理します。

モジュール内での役割

src/backend/utils/adt モジュールにおいて、数値計算の基盤を提供します。numeric.h で定義されたインターフェースを実装し、SQL レベルの算術演算子（+, -, *, / 等）や集約関数（SUM, AVG, STDDEV 等）のバックエンド処理を担います。

責務

  • **データ表現**: NumericData（ディスク上の格納形式）と NumericVar（計算用メモリ形式）の相互変換および管理。
  • **算術演算**: 加減乗除、剰余、平方根、指数、対数、べき乗などの高精度演算の実装。
  • **型変換**: 文字列（numeric_in/out）、整数、浮動小数点数との相互変換。
  • **集約処理**: SUM や AVG などの集約関数における中間状態（NumericSumAccum）の管理と高速な加算処理。
  • **プランナ連携**: numeric_support を通じたプランナへの最適化ヒントの提供。

重要なシンボル

データ構造

  • NumericData: ディスク上の可変長表現。NumericShort（短縮形式）または NumericLong（長形式）のいずれかを選択する。
  • NumericVar: 計算用のメモリ内表現。NumericDigit 配列を持ち、算術演算の対象となる。
  • NumericSumAccum: 集約関数用の高速加算器。32bit 整数を用いてキャリー伝播を遅延させる。

算術・変換関数

  • add_var, sub_var, mul_var, div_var: NumericVar に対する基本算術演算。
  • numeric_in, numeric_out: SQL 文字列と内部表現の変換。
  • numeric_recv, numeric_send: バイナリプロトコル用の変換。
  • numeric_round, numeric_trunc: 数値の丸め・切り捨て。

内部ユーティリティ

  • make_result: NumericVar から NumericData を生成する。
  • strip_var: NumericVar の先頭・末尾のゼロを除去して正規化する。
  • apply_typmod: NUMERIC(p, s) の制約を適用する。

制御フローとデータフロー

1. **入力**: numeric_in が文字列を解析し、set_var_from_str を経て NumericVar を生成。 2. **計算**: 演算関数（numeric_add 等）が NumericVar を受け取り、add_var 等の低レベル関数を呼び出す。 3. **正規化**: 計算結果は strip_var で正規化され、make_result でディスク格納形式（NumericData）に変換される。 4. **集約**: numeric_accum 等が NumericSumAccum を更新。accum_sum_carry で必要に応じてキャリーを伝播させ、accum_sum_final で最終結果を NumericVar に変換する。

読者向けメモ

  • **メモリ管理**: NumericVar の digit 配列は palloc されるため、free_var による解放が必須です。
  • **基数**: NBASE=10000 を採用しており、1 つの NumericDigit に 4 桁の 10 進数を格納します。
  • **最適化**: mul_var は NBASE^2 を用いた基数変換により計算量を削減しています。
  • **制約**: NUMERIC_MAX_PRECISION は 1000 です。これを超える精度はサポートされません。
  • **未実装/浅い部分**: dump_numeric 等のデバッグ関数は NUMERIC_DEBUG マクロが有効な場合のみコンパイルされます。また、一部の数学関数は double への変換を介して近似計算を行っています。
