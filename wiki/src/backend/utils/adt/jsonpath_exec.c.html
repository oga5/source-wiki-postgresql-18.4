src/backend/utils/adt/jsonpath_exec.c

概要

SQL/JSON パス言語の実行エンジンを実装するファイルです。jsonb データ型に対して jsonpath を適用し、フィルタリング、抽出、変換、および JSON_TABLE 演算を実行します。PostgreSQL の JSONB 構造を再帰的に走査し、標準仕様に基づいたパス評価を行います。

モジュール内での役割

src/backend/utils/adt モジュールの一部として、JSONB 関連の SQL 関数（jsonb_path_exists, jsonb_path_query 等）のバックエンドロジックを提供します。プランナやエグゼキュータが利用する TableFuncRoutine インターフェースを介して、JSON_TABLE 演算の実行状態管理も担います。

責務

  • **パス実行**: executeJsonPath を起点とした再帰的なパス走査と評価。
  • **型変換と演算**: JSONB 値の数値・文字列・日時型へのキャスト、算術演算、比較演算。
  • **エラーハンドリング**: lax モードと strict モードに応じたエラーの抑制または報告。
  • **JSON_TABLE 実行**: 行パターン生成、ネストされたパスの結合、列値の抽出。

重要なシンボル

実行コンテキスト

  • JsonPathExecContext: パス実行中のグローバル状態（変数、ルート/カレントアイテム、モードフラグ、タイムゾーン設定）。
  • JsonTableExecContext: JSON_TABLE 実行時のプラン状態と列ごとの実行コンテキスト。

実行結果・型

  • JsonPathExecResult: 実行結果（jperOk, jperNotFound, jperError）。
  • JsonPathBool: 述語評価結果（jpbTrue, jpbFalse, jpbUnknown）。
  • JsonValueList: 評価結果のシーケンスを保持するリスト構造。

主要なエントリポイント

  • executeJsonPath: パス実行のメインエントリ。
  • jsonb_path_exists_internal / jsonb_path_match_internal: SQL 関数から呼び出されるラッパー。
  • JsonTableGetValue: JSON_TABLE の列値抽出関数。

制御フローとデータフロー

1. **初期化**: executeJsonPath が JsonPathExecContext を構築し、ルートアイテムをセット。 2. **再帰的走査**: executeItem がパスアイテムの型に応じて分岐。配列のアンラップが必要な場合は executeItemOptUnwrapTarget を経由。 3. **述語評価**: executeBoolItem がフィルタ条件を評価。executePredicate が左右のオペランドをシーケンスとして取得し、比較関数（executeComparison 等）を適用。 4. **結果収集**: 評価された値は JsonValueList に蓄積され、次のパスアイテムへ渡されるか、最終結果として返される。 5. **JSON_TABLE フロー**: JsonTableSetDocument で行パターンをリセットし、JsonTablePlanNextRow が再帰的にネストされたパスを結合して行を生成。

読者向けメモ

  • **メモリ管理**: JsonValueList は palloc を多用します。JSON_TABLE 実行時は MemoryContext を切り替えてメモリリークを防いでいます。
  • **再帰の深さ**: check_stack_depth() が各再帰ステップで呼ばれており、深いパスによるスタックオーバーフローを防止しています。
  • **lax/strict モード**: jspAutoUnwrap などのマクロでモード判定を行います。lax モードでは構造エラーが無視され、配列の自動アンラップが行われます。
  • **省略された詳細**: 正規表現のコンパイル詳細や、各算術演算子の具体的な数値変換ロジックは、PostgreSQL の既存の数値型関数（numeric_add 等）に委譲されているため、本ファイルでは主に呼び出しの制御に焦点を当てています。
