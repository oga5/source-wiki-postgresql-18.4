src/backend/utils/adt/jsonb_util.c

概要

PostgreSQL の jsonb 型における、バイナリ表現とメモリ上の JsonbValue 構造体との相互変換、およびコンテナ（配列・オブジェクト）の反復処理を担うユーティリティモジュールです。

モジュール内での役割

jsonb 型のコアロジックを支える基盤層です。jsonb.c が SQL インターフェースや演算子を定義するのに対し、本ファイルはバイナリデータのパース、シリアライズ、およびコンテナ内の要素検索といった低レベルな操作を提供します。

責務

  • **変換**: Jsonb（ディスク上のバイナリ）と JsonbValue（メモリ上の操作用構造体）の相互変換。
  • **反復**: JsonbIterator を用いたネストされた JSON 構造の走査。
  • **検索**: コンテナ内でのキー検索やインデックス指定による要素取得。
  • **構築**: JsonbParseState を介した、メモリ上での jsonb 構築と正規化（重複キーの排除など）。

重要なシンボル

  • JsonbToJsonbValue / JsonbValueToJsonb: 変換の主要エントリーポイント。
  • JsonbIteratorNext: コンテナを走査するためのイテレータ関数。
  • findJsonbValueFromContainer: 配列やオブジェクトから特定の値を検索する関数。
  • pushJsonbValue: jsonb 構築時の状態遷移を管理する関数。
  • JSONB_MAX_ELEMS / JSONB_MAX_PAIRS: コンテナのサイズ制限を定義する定数。

制御フローとデータフロー

1. **構築**: pushJsonbValue を呼び出し、JsonbParseState をスタックとして利用して階層構造を構築します。WJB_END_OBJECT などのトークンで uniqueifyJsonbObject が呼ばれ、キーの重複排除が行われます。 2. **シリアライズ**: convertToJsonb が StringInfo バッファを確保し、convertJsonbValue が再帰的に各要素をバイナリ形式へ書き込みます。 3. **走査**: JsonbIteratorInit で初期化し、JsonbIteratorNext をループで回します。ネストされたコンテナに遭遇すると、内部で新しいイテレータを生成して再帰的に処理します。

読者向けメモ

  • **メモリ管理**: JsonbIteratorNext はネストされたコンテナを処理する際、内部でイテレータを palloc します。長時間の反復処理でメモリリークを防ぐため、イテレータの親子関係を適切に管理する必要があります。
  • **バイナリ構造**: jsonb のバイナリは JEntry 配列と可変長データ部で構成されます。getJsonbOffset や getJsonbLength は、この構造を解釈して特定の要素の物理位置を特定するために不可欠です。
  • **再帰**: 多くの関数が再帰的です。複雑な JSON 構造を扱う際は check_stack_depth() によるスタックオーバーフロー対策が随所で行われています。
