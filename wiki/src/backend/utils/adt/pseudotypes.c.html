src/backend/utils/adt/pseudotypes.c

概要

PostgreSQLのシステムカタログ（pg_type）に定義される「擬似型（pseudo-types）」および、特殊な扱いを要する型の入出力関数を実装するモジュールです。

モジュール内での役割

PostgreSQLの型システムにおいて、擬似型はテーブル列として使用できない抽象的な型です。しかし、システム内部の整合性や、特定の関数呼び出し（SELECT type_in(...) 等）をサポートするために、pg_type には入出力関数のエントリが必要です。本ファイルは、これらの関数に対して「エラーを投げる」または「適切な代替関数へ委譲する」ための共通基盤を提供します。

責務

  • **擬似型の入出力禁止:** PSEUDOTYPE_DUMMY_... マクロを用いて、サポートされない型に対する入出力操作をエラー（ERRCODE_FEATURE_NOT_SUPPORTED）で拒否する。
  • **例外的な型のサポート:** anyarray や void など、擬似型であっても出力（表示）が必要な型に対して、適切な変換関数を提供・委譲する。
  • **cstring型の完全サポート:** 擬似型として扱われるが、内部的には機能する cstring 型に対して、完全な入出力関数セットを提供する。

重要なシンボル

  • PSEUDOTYPE_DUMMY_IO_FUNCS(typname): 入出力関数をエラーにするためのマクロ。
  • cstring_in / cstring_out: cstring 型の入出力処理。
  • void_in / void_out: void 型の入出力処理。
  • anyarray_out / anyrange_out: 擬似型だが表示が必要な型に対する出力関数。

制御フローとデータフロー

  • **拒否フロー:** ほとんどの擬似型（any, trigger 等）は、マクロにより ereport(ERROR, ...) を呼び出す関数に固定されます。
  • **委譲フロー:** anyarray_out や anyenum_out などは、内部的に array_out や enum_out といった実体型の関数を呼び出し、結果をそのまま返します。
  • **cstring処理:** cstring 型は、pstrdup を用いてメモリを確保し、文字列のコピーを返します。バイナリ送受信（recv/send）では libpq の StringInfo を介して変換を行います。

直接依存

  • postgres.h: PostgreSQLの基本定義。
  • libpq/pqformat.h: バイナリ入出力（StringInfo 等）の操作。
  • utils/fmgrprotos.h: 関数マネージャのプロトタイプ定義。

関連コード

  • src/include/catalog/pg_type.h: 擬似型が定義されているカタログ。
  • src/backend/utils/adt/arrayfuncs.c: anyarray 系の出力処理の委譲先。
  • src/backend/utils/adt/enum.c: anyenum の出力処理の委譲先。

読者向けメモ

  • **エラーメッセージ:** 本ファイル内のエラーメッセージは汎用的です。特定のコンテキストで混乱を招く可能性があるため、将来的な改善の余地（XXX コメント）が残されています。
  • **擬似型と実型:** pg_node_tree のように、擬似型ではないが本ファイルで入出力関数を管理している型もあります。これは、セキュリティ上の理由で入力を禁止しつつ、出力は許可したいという特殊な要件があるためです。
