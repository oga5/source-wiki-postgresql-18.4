src/backend/utils/adt/tsquery_gist.c

概要

tsquery_gist.c は、PostgreSQL の全文検索型 tsquery に対する GiST (Generalized Search Tree) インデックスのサポート機能を提供します。tsquery の検索条件をビットマップ（シグネチャ）に圧縮し、インデックスの構築、検索、分割を効率化します。

モジュール内での役割

src/backend/utils/adt モジュールの一部として、全文検索機能（tsearch）のインデックス最適化を担います。GiST インデックスの標準的なインターフェース（compress, consistent, union, penalty, picksplit）を tsquery 型向けに実装しています。

責務

  • **シグネチャ圧縮**: tsquery を固定長のビットマップ（TSQuerySign）に変換し、インデックスサイズを削減する。
  • **検索の適合性判定**: consistent 関数により、インデックス内のシグネチャと検索クエリの包含関係を判定する。
  • **インデックス構造の維持**: union や picksplit を通じて、GiST ツリーのバランスと検索効率を最適化する。

重要なシンボル

  • gtsquery_consistent: インデックス検索時に呼び出され、条件に合致するかを判定する。常に recheck = true を返し、インデックスによる絞り込み後に詳細な再検証を行う。
  • gtsquery_compress: インデックス登録時に tsquery を TSQuerySign に圧縮する。
  • gtsquery_picksplit: インデックスページが溢れた際の分割アルゴリズム。ハミング距離を用いて、分割後のオーバーラップを最小化する。
  • hemdist: 2つのシグネチャ間のハミング距離を計算し、ペナルティ計算や分割の基準として使用する。

制御フローとデータフロー

1. **圧縮**: gtsquery_compress が tsquery を TSQuerySign に変換し、インデックスに格納する。 2. **検索**: gtsquery_consistent がクエリのシグネチャとインデックスのシグネチャをビット演算（&）で比較し、候補を絞り込む。 3. **分割**: gtsquery_picksplit が最も距離の遠い2つのエントリをシードとして選択し、他のエントリをコスト計算に基づいて左右のページに振り分ける。

読者向けメモ

  • 本ファイルの実装は「不完全（inexact）」なインデックス検索を前提としています。consistent 関数が常に recheck = true を設定するのは、シグネチャによる判定が偽陽性を含む可能性があるためです。
  • gtsquery_consistent_oldsig は古いバージョンの PostgreSQL との互換性維持のための関数であり、新規開発では無視して構いません。
  • ハミング距離（hemdist）の計算は、インデックスの品質（検索効率）を左右する重要なヒューリスティックです。
