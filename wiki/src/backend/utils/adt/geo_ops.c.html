src/backend/utils/adt/geo_ops.c

概要

geo_ops.c は、PostgreSQL における 2 次元幾何データ型（point, line, lseg, box, circle, path, polygon）の内部表現、I/O 処理、および演算子を実装するモジュールです。幾何学的な計算（交差判定、距離計算、包含関係）のバックエンドロジックを担います。

モジュール内での役割

src/backend/utils/adt 内で幾何データ型の演算を統括します。プランナやインデックス（GiST 等）が利用する選択率推定やコスト計算は geo_selfuncs.c に分離されており、本ファイルは純粋なデータ型操作と演算に特化しています。

責務

  • **データ型 I/O**: SQL 文字列形式と内部バイナリ形式の相互変換（_in, _out, _recv, _send）。
  • **幾何演算**: 2 点間の距離、線分の交差、包含判定、面積計算などの幾何学的プリミティブの提供。
  • **算術演算**: 幾何オブジェクトに対する移動、回転、スケーリング等の演算。
  • **正規化**: 内部表現の整合性維持（例：box の座標順序の入れ替え、polygon のバウンディングボックス計算）。

重要なシンボル

型変換・I/O

  • box_in, box_out, box_recv, box_send: BOX 型の入出力。
  • path_in, path_out, path_recv, path_send: PATH 型の入出力。
  • poly_in, poly_out, poly_recv, poly_send: POLYGON 型の入出力。

幾何演算・判定

  • lseg_interpt_lseg: 線分同士の交差判定。
  • box_contain_box: BOX 同士の包含判定。
  • point_inside: POLYGON 内の点判定（レイキャスティング法）。
  • dist_ppoly_internal: 点と多角形の距離計算。

算術・構築

  • line_construct: 2 点から直線式 $Ax + By + C = 0$ を構築。
  • statlseg_construct: 2 点から線分を構築。
  • box_construct: 2 点から BOX を構築（座標の正規化を含む）。

制御フローとデータフロー

1. **入力処理**: _in 関数が文字列をパースし、path_decode や pair_decode を経由して Point 配列等の内部構造体に変換します。 2. **正規化**: box_in や poly_in は、入力後に座標の大小関係を整理（box_construct や make_bound_box）し、一貫した内部表現を保証します。 3. **演算**: SQL 演算子から呼び出される関数は、Point や LSEG などのプリミティブを引数に取り、FPeq や FPlt などの浮動小数点比較マクロを用いて計算します。 4. **出力処理**: _out 関数は path_encode 等を用いて内部構造体を SQL 互換の文字列にシリアライズします。

読者向けメモ

  • **浮動小数点の扱い**: 幾何計算には float8 を使用しており、比較には FPeq 等の epsilon を考慮したマクロが必須です。
  • **メモリ管理**: palloc を多用します。特に PATH や POLYGON は可変長データ型（varlena）として扱われるため、SET_VARSIZE によるサイズ設定が重要です。
  • **未実装・浅い部分**: point_inside はレイキャスティング法を用いていますが、複雑な多角形に対する最適化（空間インデックス等）は含まれていません。
  • **注意点**: line_construct 等で生成される直線式は、垂直線や水平線で特殊な係数（A=0 や B=0）を取るため、ゼロ除算を避けるロジックが随所に含まれています。
