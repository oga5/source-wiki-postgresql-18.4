src/backend/utils/adt/trigfuncs.c

概要

本ファイルは、PostgreSQLのトリガー機能で利用可能な組み込み関数を提供します。現在は、冗長な更新操作を抑制するためのトリガー関数が実装されています。

モジュール内での役割

src/backend/utils/adt ディレクトリの一部として、SQLから呼び出し可能な組み込み関数（ADT: Abstract Data Type）のインターフェースを提供します。トリガーサブシステムとデータ型処理の橋渡しを行う役割を担います。

責務

  • **更新の抑制**: UPDATE 操作において、更新前後のタプルが物理的に同一である場合に、更新処理をスキップさせるトリガーロジックを提供します。
  • **トリガー実行環境の検証**: 関数がトリガーとして正しく呼び出されているか（BEFORE UPDATE かつ FOR EACH ROW であるか）を検証し、不正な呼び出しに対してエラーを返します。

重要なシンボル

  • suppress_redundant_updates_trigger: 本ファイルの唯一の公開関数。CREATE TRIGGER 文で指定することで、変更のない更新を無効化します。

制御フローとデータフロー

1. **検証フェーズ**: fcinfo から TriggerData を取得し、トリガーの呼び出し条件（TRIGGER_FIRED_BY_UPDATE 等）をチェックします。 2. **比較フェーズ**: tg_newtuple（更新後）と tg_trigtuple（更新前）のタプル長、ヘッダー情報、およびペイロード（memcmp）を比較します。 3. **決定フェーズ**:

  • 完全に一致する場合: rettuple を NULL に設定し、トリガー呼び出し元に対して更新の中止を指示します。
  • 差異がある場合: newtuple をそのまま返し、更新を続行させます。

直接依存

  • postgres.h: PostgreSQLの基本定義。
  • access/htup_details.h: ヒープタプルの構造体定義とアクセス用マクロ。
  • commands/trigger.h: トリガー関連のデータ構造（TriggerData）とマクロ。
  • utils/fmgrprotos.h: 関数マネージャのプロトタイプ定義。

関連コード

  • src/backend/commands/trigger.c: トリガーの実行エンジン本体。本ファイルで定義された関数がどのように呼び出されるかを理解する上で重要です。
  • src/backend/executor/execMain.c: UPDATE 操作の実行フローにおいて、トリガーがどのように介入するかの詳細が記述されています。

読者向けメモ

  • 本関数は、更新対象のタプルが「物理的に」同一であるかを判定します。論理的に等価であっても、物理的な表現（例：NULLビットマップやヘッダーの差異）が異なる場合は更新が実行されます。
  • memcmp を使用しているため、非常に高速ですが、タプルの物理レイアウトに依存した実装である点に注意してください。
  • 開発者は、この関数を BEFORE UPDATE トリガーとして定義することで、不要なインデックス更新やWAL書き込みを回避し、パフォーマンスを最適化できます。
