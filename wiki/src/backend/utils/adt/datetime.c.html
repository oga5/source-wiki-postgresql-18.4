src/backend/utils/adt/datetime.c

概要

datetime.c は、PostgreSQL における日時データ型（timestamp, date, time, interval 等）の解析、変換、および書式化を支える中核的なサポートモジュールです。SQL 文字列から内部表現への変換、タイムゾーンの計算、および Julian 日付変換などの低レベルな算術ロジックを所有しています。

モジュール内での役割

src/backend/utils/adt モジュールの一部として、日時関連のデータ型実装（timestamp.c, date.c など）から呼び出される共通関数を提供します。SQL レベルの型変換関数（_in, _out）の背後にある複雑なパースロジックをカプセル化しています。

責務

  • **日時パース**: 入力文字列のトークン化および意味解析（ParseDateTime, DecodeDateTime）。
  • **カレンダー演算**: Julian 日付とカレンダー日付の相互変換、曜日計算。
  • **タイムゾーン処理**: タイムゾーンオフセットの決定、DST（夏時間）境界の計算、動的タイムゾーンの解決。
  • **間隔演算**: interval 型の算術処理と正規化。
  • **書式化**: 内部表現から SQL 準拠の文字列への変換（EncodeDateTime, EncodeInterval）。

重要なシンボル

日付・時間変換

  • date2j, j2date: カレンダー日付と Julian 日付の相互変換。
  • j2day: Julian 日付から曜日（0-6）への変換。

パース・デコード

  • ParseDateTime: 文字列をトークンと型（DTK_DATE, DTK_TIME 等）に分解。
  • DecodeDateTime, DecodeTimeOnly: トークン列を struct pg_tm にマッピング。
  • DecodeInterval: interval 型のパースと算術調整。

タイムゾーン管理

  • DetermineTimeZoneOffset: 指定日時のタイムゾーンオフセットと DST 状態を決定。
  • DecodeTimezoneAbbrev: タイムゾーン略称（PST, JST 等）の解決。
  • zoneabbrevtbl: 構成ファイルからロードされたタイムゾーン略称テーブル。

内部データ構造

  • datetktbl, deltatktbl: 日時キーワードおよび間隔単位の静的ルックアップテーブル。
  • TzAbbrevCache: タイムゾーン略称のルックアップ結果を保持するキャッシュ。

制御フローとデータフロー

1. **入力**: SQL 関数が文字列を ParseDateTime に渡す。 2. **トークン化**: ParseDateTime が文字列を走査し、field[]（文字列）と ftype[]（型）の配列を生成。 3. **デコード**: DecodeDateTime がトークン配列をループし、struct pg_tm に値を埋める。この際、datebsearch を用いてキーワードテーブルを高速検索する。 4. **調整**: ValidateDate や Adjust* 系関数（AdjustMicroseconds 等）が、オーバーフローチェックを行いながら値を正規化する。 5. **出力**: EncodeDateTime 等が struct pg_tm を受け取り、指定された DateStyle に基づいて文字列バッファに書き出す。

読者向けメモ

  • **パフォーマンス**: datebsearch はバイナリサーチを利用しており、キーワードテーブルはアルファベット順にソートされている必要があります。テーブルを編集する際は注意してください。
  • **メモリ管理**: TzAbbrevCache はセッション単位のキャッシュであり、ClearTimeZoneAbbrevCache でリセット可能です。
  • **オーバーフロー**: 日時演算では pg_add_s64_overflow 等の安全な算術関数が多用されています。これは、日時が広範囲な値を取り得るためです。
  • **省略された詳細**: タイムゾーンの内部的な zic 形式のバイナリ解析ロジックは pg_tz モジュールに委譲されており、本ファイルでは主にその結果を利用する上位ロジックに焦点を当てています。
