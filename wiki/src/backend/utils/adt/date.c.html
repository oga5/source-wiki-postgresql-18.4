src/backend/utils/adt/date.c

概要

本ファイルは、SQL 標準で定義される DATE 型および TIME 型（TIME WITHOUT TIME ZONE および TIME WITH TIME ZONE）の内部実装を提供します。PostgreSQL のデータ型システムにおいて、これらの型の入出力、算術演算、比較、および型変換ロジックを担う中心的なソースコードです。

モジュール内での役割

src/backend/utils/adt モジュールの一部として、日時関連のデータ型を管理します。timestamp.c が日時（TIMESTAMP）を扱うのに対し、本ファイルは日付のみ、あるいは時刻のみに特化した型を扱います。これらは Datum として表現され、PostgreSQL の実行エンジンやインデックスアクセスメソッドと連携します。

責務

  • **型変換とI/O**: SQL 文字列と内部表現（DateADT, TimeADT, TimeTzADT）の相互変換（_in, _out, _recv, _send）。
  • **算術演算**: 日付・時刻に対する加減算、および INTERVAL 型との演算。
  • **比較とソート**: B-Tree インデックス等で利用される比較関数および sortsupport 関数の提供。
  • **型修飾子（Typmod）管理**: TIME(n) の精度制限や検証。
  • **型間変換**: DATE / TIME / TIMESTAMP / TIMESTAMPTZ 間の相互変換。

重要なシンボル

内部表現型

  • DateADT: int32。エポック（2000-01-01）からの経過日数。
  • TimeADT: int64。00:00:00 からのマイクロ秒数。
  • TimeTzADT: struct。TimeADT と int32（GMT からのオフセット秒）の組。

型変換・I/O

  • date_in / date_out: DATE 型のテキスト入出力。
  • time_in / time_out: TIME 型のテキスト入出力。
  • timetz_in / timetz_out: TIMETZ 型のテキスト入出力。

演算・サポート関数

  • date_cmp, time_cmp, timetz_cmp: 比較関数。
  • date_sortsupport, date_skipsupport: インデックス最適化用関数。
  • AdjustTimeForTypmod: TIME 型の精度（小数点以下の桁数）を調整するユーティリティ。

制御フローとデータフロー

1. **入力処理**: _in 関数群が ParseDateTime を呼び出し、文字列を pg_tm 構造体に分解します。その後、date2j や tm2time を用いて内部表現（DateADT や TimeADT）へ変換します。 2. **演算処理**: 演算子関数（date_pli 等）は、Datum から値を抽出し、算術演算を行った後、範囲チェック（IS_VALID_DATE 等）を経て結果を返します。 3. **型変換**: date2timestamp_opt_overflow 等の関数が、日付をタイムスタンプへ昇格させます。この際、オーバーフロー発生時の挙動（エラーまたは無限値の返却）を制御します。 4. **精度調整**: AdjustTimeForTypmod が TIME 型の精度を切り捨て/丸め処理し、SQL で指定された精度に合わせます。

読者向けメモ

  • **メモリ管理**: TimeTzADT は palloc を使用してヒープ上に確保されます。DateADT や TimeADT は int32/int64 であり、Datum として直接渡されます。
  • **無限値**: DATEVAL_NOBEGIN / DATEVAL_NOEND は、それぞれ PG_INT32_MIN / PG_INT32_MAX を使用して表現されます。
  • **精度**: TIME 型の精度は MAX_TIME_PRECISION（6）までサポートされています。
  • **注意点**: time_in 等の関数は、typmod を引数として受け取り、入力時に即座に精度調整を行う設計になっています。
  • **省略箇所**: 本ファイルは非常に広範な比較関数群を含みますが、これらは基本的に date_cmp や timetz_cmp_internal をラップした定型的な実装です。個別の比較演算子のロジックは、この内部関数を理解すれば容易に把握可能です。
