src/backend/utils/adt/varchar.c

概要

PostgreSQLの組み込みデータ型である char(n) (BpChar) および varchar(n) の実装を提供します。これらの型は、指定された最大長を持つ可変長文字列として扱われます。

モジュール内での役割

src/backend/utils/adt モジュールにおいて、文字列型の中核を担います。text 型とバイナリ互換性を持ちつつ、挿入時の長さ制限や空白埋め（padding）といった特定のセマンティクスを適用する役割を持ちます。

責務

  • **型変換と検証**: input/output 関数および typmod（型修飾子）の検証。
  • **セマンティクス適用**: char(n) における末尾空白の扱い（比較時の無視など）や、varchar(n) の長さ制限の強制。
  • **比較・ハッシュ演算**: ロケールを考慮した比較関数や、末尾空白を無視するハッシュ関数の提供。
  • **プランナサポート**: varchar 型の長さ変換における不要な処理の最適化。

重要なシンボル

  • bpcharin / varcharin: 文字列から内部表現への変換。
  • bpchar / varchar: 型キャスト関数。指定長への切り詰めや空白埋めを行う。
  • bpchartruelen: 末尾の空白を除いた「真の長さ」を計算するユーティリティ。
  • bpchareq / bpcharcmp: ロケールを考慮した比較関数。
  • hashbpchar: 末尾空白を無視するハッシュ関数。

制御フローとデータフロー

1. **入力**: bpchar_input / varchar_input が文字列を受け取り、atttypmod に基づいて長さを検証します。長すぎる場合はエラーを投げるか、末尾が空白であれば切り詰めます。 2. **変換**: bpchar / varchar 関数が、既存のデータに対してキャスト（長さ変更）を適用します。 3. **比較**: bpchareq 等は bcTruelen を呼び出し、末尾の空白を無視した状態で varstr_cmp を用いてロケール依存の比較を行います。 4. **ハッシュ**: hashbpchar は、ロケールが非決定的な場合、pg_strnxfrm を用いて正規化してからハッシュ値を計算します。

読者向けメモ

  • char(n) と varchar(n) は内部的には varlena 構造体ですが、text 型とは異なり、挿入時に末尾の空白埋めや切り詰めが発生します。
  • 比較関数において、deterministic なロケール（Cロケールなど）ではビット単位の比較を行い、それ以外では strcoll 相当の処理を行うことでパフォーマンスを最適化しています。
  • bpchar の比較において、末尾の空白を無視する仕様は SQL 標準に準拠していますが、実装上は bcTruelen による事前計算が重要です。
