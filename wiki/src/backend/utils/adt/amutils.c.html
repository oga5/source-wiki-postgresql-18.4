src/backend/utils/adt/amutils.c

概要

amutils.c は、PostgreSQL のインデックス・アクセスメソッド（AM）に関する情報を SQL レベルで取得するための API を提供します。主に pg_indexam_has_property などのシステム関数を通じて、インデックスの機能や特性を動的に照会するインターフェースを実装しています。

モジュール内での役割

src/backend/utils/adt モジュール内で、インデックスのメタデータと AM の能力（Capability）を橋渡しする役割を担います。カタログ情報（pg_index, pg_class）と IndexAmRoutine 構造体を統合し、ユーザーがインデックスの挙動を検証するための共通基盤を提供します。

責務

  • **プロパティ照会**: インデックス、インデックス列、または AM 全体に対する特定の属性（orderable, returnable 等）の有無を判定する。
  • **AM 抽象化**: 個別の AM が持つ固有のロジックと、PostgreSQL が提供する汎用的なロジックを統合する。
  • **進捗報告**: インデックス構築時のフェーズ名を取得する API の提供。

重要なシンボル

  • indexam_property: 本ファイルの核心となる内部関数。AM、インデックス、列の各レベルでのプロパティ判定を統合的に処理します。
  • IndexAMProperty: 照会可能なプロパティを定義する列挙型。
  • test_indoption: pg_index.indoption（ソート順や NULL の扱い）をビットマスクで検証するヘルパー関数。
  • pg_indexam_has_property / pg_index_has_property / pg_index_column_has_property: SQL から呼び出される公開関数。

追加の索引済みシンボル

  • am_propname
  • lookup_prop_name
  • pg_indexam_progress_phasename

制御フローとデータフロー

1. **入力解析**: SQL 関数が呼び出されると、indexam_property がプロパティ名を列挙型に変換し、対象の amoid または index_oid を特定します。 2. **AM ルーチンの取得**: GetIndexAmRoutineByAmId を使用して、対象 AM の IndexAmRoutine 構造体を取得します。 3. **AM 優先の判定**: AM 側で amproperty 関数が定義されている場合、まずはそれに処理を委譲します。 4. **汎用ロジックの適用**: AM 側で解決できない場合、attno（列番号）の有無に応じて以下の処理を行います。

  • **列レベル**: pg_index を参照し、test_indoption 等を用いてソート順や検索可能性を判定。
  • **インデックス/AM レベル**: IndexAmRoutine のフラグ（amcanorder, amcanunique 等）を直接参照して結果を返します。

直接依存

  • access/amapi.h: IndexAmRoutine 構造体および AM API 定義。
  • catalog/pg_index.h / pg_class.h: システムカタログの構造体定義。
  • utils/syscache.h: システムカタログのキャッシュアクセス。
  • access/htup_details.h
  • catalog/pg_class.h
  • postgres.h
  • utils/builtins.h

関連コード

  • src/include/access/amapi.h: 各 AM が実装すべきルーチンの定義。
  • src/backend/access/index/indexam.c: インデックスアクセスの実処理。
  • src/backend/catalog/index.c: インデックス作成や管理のロジック。

読者向けメモ

  • 本ファイルは「プロパティの照会」に特化しており、実際のインデックス操作は行いません。
  • amproperty ルーチンが定義されていない場合、汎用的なビットテストやフラグ参照にフォールバックする設計になっています。
  • 新しい AM プロパティを追加する場合は、IndexAMProperty 列挙型と am_propnames 配列の両方を更新する必要があります。
