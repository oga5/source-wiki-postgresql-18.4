src/backend/utils/adt/tsvector_op.c

概要

tsvector_op.c は、PostgreSQL の全文検索用データ型 tsvector に対する演算、操作、および統計処理を実装するモジュールです。tsvector の内部構造（ソートされた語彙と位置情報の配列）を直接操作し、検索クエリとの照合、重み付け、編集、および統計情報の抽出を行います。

モジュール内での役割

全文検索サブシステム（tsearch）のデータ型操作レイヤーとして機能します。tsvector 型の SQL 関数（setweight, strip, delete 等）のバックエンド実装を提供し、インデックス（GIN/GiST）やクエリ実行エンジンが利用する比較・照合ロジックを担います。

責務

  • **データ操作**: tsvector の結合、重み付け、不要な位置情報の削除、特定の語彙の削除。
  • **クエリ照合**: tsquery と tsvector の照合ロジック（TS_execute）。特に位置情報（phrase 演算子）を考慮した複雑な照合。
  • **統計処理**: ts_stat 関数による tsvector カラムの語彙統計情報の集計。
  • **トリガーサポート**: テキストカラムから tsvector を自動生成するトリガー関数の提供。

重要なシンボル

データ構造

  • TSVector: tsvector 型の内部表現（varlena 構造）。
  • CHKVAL: クエリ照合時に tsvector の走査状態を保持するコンテキスト。
  • TSVectorStat: ts_stat 実行時に語彙の出現頻度を保持するツリー構造。
  • ExecPhraseData: phrase 演算子実行時に位置情報とマッチ幅を保持する構造体。

演算・操作関数

  • tsvector_strip: 位置情報を削除し、語彙のみにする。
  • tsvector_setweight: 指定された重みを全語彙に適用。
  • tsvector_delete_str / tsvector_delete_arr: 指定した語彙を削除。
  • tsvector_concat: 2つの tsvector をマージし、位置情報をオフセット調整して結合。

クエリ照合・実行

  • TS_execute: tsquery の論理演算を実行するメインエントリーポイント。
  • TS_phrase_execute: phrase 演算子を含む複雑なクエリの照合と位置情報の計算。
  • ts_match_vq: tsvector と tsquery のマッチングを行う SQL 関数。

制御フローとデータフロー

1. **照合フロー**: ts_match_vq が TS_execute を呼び出し、checkcondition_str コールバックを通じて tsvector のバイナリ配列を二分探索（tsvector_bsearch）します。 2. **位置情報フロー**: phrase 演算子が含まれる場合、TS_phrase_execute が再帰的に呼び出され、ExecPhraseData を介してマッチした位置のリストをマージ・計算します。 3. **統計フロー**: ts_stat は SPI を使用してテーブルを走査し、ts_accum で各行の tsvector を TSVectorStat ツリーに挿入して集計します。

読者向けメモ

  • **メモリ管理**: tsvector は可変長データ型であり、関数内で palloc を使用して新しい TSVector を生成する際は、VARSIZE の計算と SET_VARSIZE が必須です。
  • **位置情報の制約**: tsvector の位置情報は uint16 に格納され、重み（2bit）と位置（14bit）を共有します。このため、位置は MAXENTRYPOS（16383）までに制限されます。
  • **パフォーマンス**: tsvector_concat や tsvector_delete は、内部配列の再構築を伴うため、大きな tsvector に対してはコストが高くなります。
  • **未実装/浅い部分**: tsquery_requires_match は、インデックススキャン最適化のために簡略化されたロジックを使用しており、複雑な NOT 演算のネストについては網羅的ではありません。
