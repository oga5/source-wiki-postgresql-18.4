src/backend/utils/adt/tsrank.c

概要

tsrank.c は、PostgreSQL の全文検索機能において、tsvector（文書）と tsquery（検索条件）の適合度を計算（ランキング）するための関数群を提供します。

モジュール内での役割

src/backend/utils/adt モジュールの一部として、全文検索の検索結果をソートするための SQL 関数（ts_rank および ts_rank_cd）を実装しています。

責務

  • **適合度計算**: 文書内の単語の出現頻度や重みに基づき、スコアを算出する。
  • **近接度ランキング**: ts_rank_cd を通じて、クエリ内の単語が文書内でどれだけ近くに出現しているか（Cover Density）を考慮したランキングを行う。
  • **正規化**: 文書の長さやユニーク単語数に応じたスコアの正規化処理。

重要なシンボル

  • ts_rank_...: 頻度ベースのランキングを行う SQL 公開関数群。
  • ts_rankcd_...: 近接度（Cover Density）ベースのランキングを行う SQL 公開関数群。
  • calc_rank: 頻度ベースのランキングのメインロジック。
  • calc_rank_cd: 近接度ランキングのメインロジック。
  • default_weights: 単語の重み（A, B, C, D）のデフォルト値。

制御フローとデータフロー

1. **入力処理**: getWeights で重み配列を検証し、PG_GETARG_... で引数を取得します。 2. **ランキング計算**:

  • 頻度ベース: calc_rank がクエリの演算子（AND/OR）に応じて calc_rank_and または calc_rank_or を呼び出し、スコアを算出します。
  • 近接度ベース: get_docrep で文書をクエリに適合する形式に変換し、Cover 関数でクエリをカバーする範囲を再帰的に探索してスコアを算出します。

    • 3. **正規化**: 指定された method（ビットフラグ）に基づき、文書の長さやユニーク単語数でスコアを割ることで正規化を行います。

読者向けメモ

  • **ランキング手法の違い**: ts_rank は単語の出現頻度を重視し、ts_rank_cd は単語同士の近接性を重視します。用途に応じて使い分けてください。
  • **再帰呼び出し**: Cover 関数は再帰的に実装されています。非常に複雑なクエリや長い文書ではスタック深さに注意が必要です。
  • **正規化フラグ**: RANK_NORM_... 定数はビットマスクとして組み合わせて使用可能です。デフォルトは RANK_NO_NORM です。
