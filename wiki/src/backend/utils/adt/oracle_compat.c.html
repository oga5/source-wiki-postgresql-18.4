src/backend/utils/adt/oracle_compat.c

概要

Oracle データベースとの互換性を目的とした、文字列操作および変換関数群を実装するソースファイルです。主に text 型および bytea 型を対象とした、大文字小文字変換、パディング、トリミング、文字変換などのユーティリティを提供します。

モジュール内での役割

src/backend/utils/adt モジュールの一部として、SQL 標準関数を補完する形で、Oracle ユーザーが慣れ親しんだ文字列操作関数を提供します。PostgreSQL の内部エンコーディング（特にマルチバイト文字）を意識した実装がなされています。

責務

  • **大文字小文字変換**: lower, upper, initcap, casefold の提供。
  • **文字列パディング**: lpad, rpad による指定長への埋め込みと切り詰め。
  • **トリミング**: btrim, ltrim, rtrim およびそれらの bytea 版による不要文字の除去。
  • **文字変換・生成**: translate による文字置換、ascii によるコードポイント取得、chr によるコードポイントからの文字生成。

重要なシンボル

  • dotrim: btrim, ltrim, rtrim の共通実装。マルチバイト文字を考慮し、文字単位での走査を行う。
  • dobyteatrim: bytea 型に対するトリミングの共通実装。
  • lower, upper, initcap: utils/formatting.h の関数をラップし、PostgreSQL の text 型として入出力を行う。
  • translate: 文字列内の特定の文字セットを別のセットに置換する。

制御フローとデータフロー

1. **入力処理**: PG_GETARG_TEXT_PP 等のマクロを使用して、PostgreSQL の Datum から text 構造体を取り出します。 2. **エンコーディング考慮**: マルチバイト文字環境では、pg_mblen_range 等を用いて文字境界を正確に判定します。 3. **メモリ確保**: palloc を使用して、変換後の文字列を格納する十分な領域を確保します。オーバーフローチェック（pg_mul_s32_overflow 等）が随所で行われます。 4. **処理実行**: 各関数は、入力文字列を走査し、必要に応じて memcpy で結果バッファにコピーします。 5. **出力**: SET_VARSIZE で最終的なサイズを確定し、PG_RETURN_TEXT_P で Datum として返します。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • mb/pg_wchar.h: マルチバイト文字処理用。
  • utils/builtins.h: 組み込み関数プロトタイプ。
  • utils/formatting.h: 文字列変換ユーティリティ。
  • varatt.h: 可変長データ型（text, bytea）の操作用。
  • common/int.h

  • utils/memutils.h

関連コード

  • src/backend/utils/adt/varlena.c: text 型の基本的な操作や repeat などの文字列関数が定義されています。
  • src/backend/utils/adt/formatting.c: str_tolower や str_initcap など、本ファイルが依存する文字列変換のコアロジックが含まれます。

読者向けメモ

  • 本ファイル内の関数は、データベースのエンコーディング設定に強く依存します。特に UTF8 環境下では、ascii や chr 関数が Unicode コードポイントを扱うよう設計されている点に注意してください。
  • マルチバイト文字の処理において、pg_mblen_range を使用した効率的な走査ロジックが dotrim 等で採用されています。新しい文字列関数を追加する際は、このパターンを参考にしてください。
  • translate 関数は、from 文字列が to 文字列より長い場合、対応する文字がない from 文字を削除する仕様です。
