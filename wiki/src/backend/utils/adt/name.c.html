src/backend/utils/adt/name.c

概要

PostgreSQLの組み込みデータ型 name を扱うための関数群を定義するファイルです。name 型はシステムカタログ等で識別子を格納するために使用される固定長（NAMEDATALEN）の文字列型です。

モジュール内での役割

src/backend/utils/adt モジュールの一部として、データ型固有の入出力、比較、およびソートロジックを提供します。システムカタログのメタデータ管理において、識別子を効率的かつ一貫した形式で扱うための基盤となります。

責務

  • **型変換**: cstring と name 型間の相互変換（namein, nameout）およびバイナリ形式の送受信（namerecv, namesend）。
  • **比較・ソート**: name 型同士の比較演算子（=, <>, <, <=, >, >=）および B-tree インデックス用の比較関数とソートサポート関数の提供。
  • **ユーティリティ**: name 型のコピーや比較を行うための内部用ヘルパー関数（namestrcpy, namestrcmp）。
  • **SQL関数**: current_user, session_user, current_schema などのセッション情報を取得する SQL 関数の実装。

重要なシンボル

  • namein / nameout: name 型の標準的な入出力関数。
  • namecmp: 内部的な比較ロジック。照合順序（Collation）を考慮し、C ロケールであれば高速な strncmp を使用します。
  • namestrcpy: name 型のバッファへ安全に文字列をコピーし、終端を保証する関数。
  • nameconcatoid: name 型に OID を付与して一意性を確保する関数（information_schema 等で使用）。

制御フローとデータフロー

  • **入出力**: namein は入力文字列を NAMEDATALEN に収まるよう pg_mbcliplen で切り詰め、palloc0 で確保したメモリにコピーします。
  • **比較**: すべての比較演算子は namecmp を呼び出します。C ロケール以外の場合は varstr_cmp を介して汎用的な文字列比較ロジックへ委譲されます。
  • **セッション関数**: current_user 等は、バックエンドの内部状態（GetUserId 等）から文字列を取得し、namein を直接呼び出して name 型の Datum に変換して返します。

読者向けメモ

  • name 型は歴史的な理由から NAMEDATALEN（通常 64 バイト）の固定長です。この制限はハードコードせず、常に定数を使用してください。
  • namein 等の関数では、palloc0 を使用してメモリをゼロクリアすることで、未初期化領域による比較の不整合を防いでいます。
  • name 型の比較において、NAMEDATALEN を超える部分は無視されます。これはシステムカタログの識別子として十分な長さであるという前提に基づいています。
