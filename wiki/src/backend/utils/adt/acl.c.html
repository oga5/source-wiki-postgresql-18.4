src/backend/utils/adt/acl.c

概要

acl.c は、PostgreSQL のアクセス制御リスト（ACL）のデータ構造操作、および SQL レベルの権限チェック関数を実装する中心的なファイルです。AclItem（権限付与の最小単位）の配列として表現される ACL のパース、マージ、更新、およびロールのメンバーシップ判定を担います。

モジュール内での役割

src/backend/utils/adt モジュールの一部として、SQL 型 aclitem の I/O 関数および演算子を提供します。また、システム全体で利用される権限チェック（has_table_privilege 等のファミリー）のバックエンド実装を提供し、セキュリティサブシステムとデータ型システムを橋渡しします。

責務

  • **ACL 操作**: ACL 配列の作成、コピー、連結、マージ、ソート、および更新。
  • **権限判定**: ロール間のメンバーシップ判定（継承、SET ROLE、管理権限の再帰的探索）。
  • **SQL インターフェース**: has_xxx_privilege ファミリーの関数群による、オブジェクトに対する権限の動的チェック。
  • **文字列変換**: ACL 文字列のパース（aclparse）およびシリアライズ（aclitemout）。

重要なシンボル

ACL データ構造操作

  • AclItem: 権限付与の最小単位（被付与者、付与者、権限ビット）。
  • allocacl, aclcopy, aclconcat, aclmerge: ACL 配列のメモリ管理と集合演算。
  • aclupdate: 権限の追加・削除・置換を行うメインのミューテータ。
  • aclmask: 指定されたロールが持つ権限ビットを計算する。

ロールメンバーシップとキャッシュ

  • cached_roles: ロールメンバーシップの再帰的探索結果を保持するキャッシュ。
  • roles_is_member_of: ロールのメンバーシップを再帰的に探索するコア関数。
  • has_privs_of_role, member_can_set_role, is_admin_of_role: 権限の種類に応じたロール関係の判定。

SQL 呼び出し可能関数

  • aclitemin, aclitemout: aclitem 型の I/O。
  • aclexplode: ACL 配列をテーブル形式に展開する SRF（Set Returning Function）。
  • has_table_privilege_* 等: 各種オブジェクトに対する権限チェックの SQL 関数群。

制御フローとデータフロー

1. **権限チェック**: has_xxx_privilege 関数が呼ばれると、convert_xxx_name で対象 OID を特定し、object_aclcheck を経由して aclmask を呼び出します。 2. **メンバーシップ探索**: aclmask は has_privs_of_role を呼び出し、これは roles_is_member_of を経由してシステムカタログ（pg_auth_members）を走査します。 3. **ACL 更新**: aclupdate は既存の AclItem を検索し、変更を適用して新しい ACL 配列を返します。再帰的な権限剥奪が必要な場合は recursive_revoke が呼び出されます。 4. **キャッシュ**: initialize_acl で登録されたコールバックが、カタログ更新時に cached_roles を無効化します。

読者向けメモ

  • **メモリ管理**: ACL 配列は palloc された可変長データ（varlena）として扱われます。aclupdate 等の関数は新しいメモリ領域を確保して返すため、呼び出し側でのメモリ解放に注意が必要です。
  • **パフォーマンス**: ロールのメンバーシップ探索は再帰的でコストが高いため、cached_roles と Bloom フィルタによる最適化が行われています。
  • **制約**: aclitem のサイズは pg_type.h でハードコードされているため、構造体の変更は厳禁です。
  • **未実装**: aclinsert および aclremove は現在サポートされておらず、エラーを返します。
