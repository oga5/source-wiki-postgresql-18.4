src/backend/utils/adt/regproc.c

概要

regproc.c は、PostgreSQL のシステムカタログ（pg_proc, pg_class, pg_operator 等）を参照する特殊なデータ型（regproc, regprocedure, regclass, regtype, regoperator, regcollation, regconfig, regdictionary, regrole, regnamespace）の入出力および変換ロジックを実装するモジュールです。これらの型は内部的には Oid とバイナリ互換ですが、SQL インターフェース上では名前解決を伴う柔軟な I/O を提供します。

モジュール内での役割

src/backend/utils/adt 内において、システムオブジェクトの OID を人間が読み書き可能な名前（または修飾名）に変換する「メタデータ型」のハブとして機能します。主に SQL レベルでのオブジェクト参照を、システムカタログの OID へ解決する役割を担います。

責務

  • **I/O 変換**: 文字列（名前）から OID への変換（_in 関数）および OID から名前への変換（_out 関数）。
  • **名前解決**: 現在の search_path を考慮したオブジェクト名の検索と、曖昧な参照の解決。
  • **バイナリ互換性**: oidrecv/oidsend を再利用したバイナリ形式のシリアライズ/デシリアライズ。
  • **フォーマット出力**: format_procedure 等の関数を通じた、他モジュール向けの標準的なオブジェクト名生成。

重要なシンボル

入出力関数群 (SQL インターフェース)

  • regprocin/regprocout: regproc 型の入出力。
  • regprocedurein/regprocedureout: regprocedure 型（引数付き関数名）の入出力。
  • regclassin/regclassout: regclass 型（リレーション名）の入出力。
  • regtypein/regtypeout: regtype 型（型名）の入出力。
  • regoperin/regoperout: regoper 型（演算子名）の入出力。
  • regoperatorin/regoperatorout: regoperator 型（引数付き演算子名）の入出力。

ユーティリティ関数

  • format_procedure_extended: regprocedure の名前生成ロジックを外部公開するための拡張関数。
  • format_operator_extended: regoperator の名前生成ロジックを外部公開するための拡張関数。
  • stringToQualifiedNameList: 文字列をスキーマ修飾名リストにパースする。

内部ヘルパー

  • parseDashOrOid: "-" または数値 OID をパースする。
  • parseNameAndArgTypes: 関数/演算子の名前と引数リストをパースする。

制御フローとデータフロー

1. **入力フロー**: _in 関数が文字列を受け取り、parseDashOrOid で数値 OID かどうかを判定。数値でなければ stringToQualifiedNameList で名前を分解し、FuncnameGetCandidates や RangeVarGetRelid 等のカタログ検索関数を呼び出して OID を特定します。 2. **出力フロー**: _out 関数が OID を受け取り、SearchSysCache を使用してカタログから名前を取得。quote_qualified_identifier を用いて、必要に応じてスキーマ修飾付きの文字列を構築します。 3. **エラーハンドリング**: escontext (ErrorSaveContext) をサポートしており、エラー発生時に例外を投げずに NULL を返す安全な呼び出しが可能です。

読者向けメモ

  • **メモリ管理**: palloc された文字列は呼び出し元で適切に解放される前提です。format_ 系関数は palloc された文字列を返します。
  • **ブートストラップモード**: IsBootstrapProcessingMode() チェックが存在し、初期化中の特殊な挙動を制御しています。
  • **設計の意図**: これらの型は Oid のラッパーですが、SQL 上で「名前」を扱えるようにすることで、ユーザーが pg_proc 等を直接クエリしなくてもオブジェクトを特定できるように設計されています。
  • **省略箇所**: regprocrecv 等のバイナリ I/O 関数は、単に oidrecv を呼び出すだけの単純なラッパーであるため、詳細は省略しています。
