src/backend/utils/adt/int.c

概要

PostgreSQL の組み込み整数型（int2 / smallint および int4 / integer）の SQL インターフェースを実装するモジュールです。型変換、算術演算、比較、および generate_series などのセット生成関数を提供します。int8（bigint）は別ファイルで管理されます。

モジュール内での役割

src/backend/utils/adt サブシステムの一部として、SQL レベルの整数演算を C 言語のネイティブな算術演算へマッピングします。common/int.h で定義されたオーバーフロー保護ルーチンを利用し、安全な数値計算を保証する役割を担います。

責務

  • **I/O 処理**: SQL 文字列と内部バイナリ表現（Datum）間の相互変換（in/out/recv/send）。
  • **算術・論理演算**: 加減乗除、剰余、ビット演算、絶対値、GCD/LCM の実装。
  • **型変換**: int2 と int4 間のキャスト、および bool 型との相互変換。
  • **セット生成**: generate_series 関数による整数シーケンスの生成とプランナサポート。
  • **バリデーション**: int2vector 型の構造整合性チェック。

重要なシンボル

I/O ルーチン

  • int2in, int2out, int4in, int4out: テキスト形式の入出力。
  • int2recv, int2send, int4recv, int4send: バイナリ形式の入出力。
  • int2vectorin, int2vectorout: int2vector 型（smallint[] の特殊版）の入出力。

演算・比較

  • int4pl, int4mi, int4mul, int4div: int4 の算術演算。
  • int4eq, int4lt, int4gt 等: int4 および int2 の比較演算子群。
  • int4gcd, int4lcm: 最大公約数および最小公倍数の計算。

ユーティリティ・構造体

  • generate_series_fctx: generate_series の実行状態を保持するコンテキスト構造体。
  • buildint2vector: int2vector を動的に構築するヘルパー。
  • check_valid_int2vector: int2vector のメモリレイアウトが期待通りか検証する内部関数。

制御フローとデータフロー

  • **演算フロー**: すべての演算関数は PG_FUNCTION_ARGS を受け取り、PG_GETARG_INTxx で値を取得します。オーバーフローの可能性がある演算（加減乗）は、pg_add_sxx_overflow 等を呼び出し、オーバーフロー時は ereport(ERROR, ...) でトランザクションを中断します。
  • **セット生成フロー**: generate_series_int4 は SRF_IS_FIRSTCALL() を用いて初回に generate_series_fctx をメモリコンテキストに割り当て、以降の呼び出しで SRF_RETURN_NEXT を用いてシーケンスを返します。
  • **GCD/LCM フロー**: int4gcd_internal はユークリッドの互換法を実装しており、INT_MIN などの境界値を安全に処理するために負の空間で計算を行います。

読者向けメモ

  • **メモリ管理**: int2vector は可変長構造体であり、palloc で確保されます。buildint2vector は SET_VARSIZE を用いてヘッダーを初期化します。
  • **オーバーフロー**: int.c 内の算術演算は、C 言語の未定義動作を避けるため、必ず common/int.h のオーバーフローチェック関数を経由します。
  • **int2vector**: これは歴史的な理由で存在する特殊な配列型です。通常の配列とは異なり、ndim=1, dataoffset=0（NULL 値なし）という不変条件を持ちます。
  • **省略された詳細**: 比較演算子の実装は非常に定型的であるため、本ページでは個別の関数ロジックの詳細は省略しています。これらはすべて PG_RETURN_BOOL を用いた単純な比較です。
