src/backend/utils/adt/like_support.c

概要

like_support.c は、LIKE、正規表現、および関連するパターンマッチング演算子に対するプランナのサポート関数を提供します。これらの関数は、インデックススキャンが直接サポートしていない演算子に対し、インデックスを利用可能な「近似条件」を生成することで、クエリ実行の効率化を図ります。

モジュール内での役割

src/backend/utils/adt モジュールの一部として、主にプランナがインデックスの利用可能性を判断する際のフックとして機能します。実行時のパターンマッチング処理（like.c や regexp.c）とは異なり、本ファイルは「どの範囲をスキャンすればよいか」という最適化の判断に特化しています。

責務

  • **インデックス条件の導出**: パターンから固定の接頭辞（prefix）を抽出し、>= や < といったインデックススキャン可能な条件式へ変換する。
  • **選択率（Selectivity）の推定**: パターンマッチングがどの程度の行にヒットするかを統計情報に基づいて推定し、プランナが最適な実行計画を選択できるようにする。
  • **演算子の抽象化**: LIKE、ILIKE、正規表現などの異なるパターンマッチング手法を、プランナが共通のインターフェースで扱えるよう抽象化する。

重要なシンボル

  • like_regex_support: プランナからの要求（選択率推定またはインデックス条件生成）を振り分けるメインエントリポイント。
  • match_pattern_prefix: パターンから固定接頭辞を抽出し、インデックススキャン用の List を生成する。
  • patternsel_common: パターンマッチングの選択率を計算する汎用ロジック。
  • Pattern_Type: サポートするパターンマッチングの種類（LIKE, Regex, Prefix等）を定義する列挙型。

制御フローとデータフロー

1. **プランナからの呼び出し**: textlike_support 等の関数がプランナから呼ばれ、like_regex_support に処理を委譲します。 2. **条件生成**: match_pattern_prefix がパターンを解析し、固定接頭辞を抽出します。 3. **インデックス適合性の確認**: 抽出した接頭辞と演算子、インデックスの照合順序（Collation）が適合するかを確認します。 4. **クエリの書き換え**: 適合する場合、make_opclause を用いてインデックススキャン用の条件式（>= prefix など）を構築し、プランナへ返します。 5. **選択率計算**: patternsel_common が統計情報（ヒストグラムやMCV）を参照し、パターンマッチングのヒット率を推定します。

読者向けメモ

  • 本ファイルは「インデックスが本来サポートしていない演算子を、インデックスを使って高速化する」という、PostgreSQLの「損失のあるインデックススキャン（Lossy Index Scan）」の考え方を実装しています。
  • make_greater_string 関数は、非Cロケールにおいて「接頭辞より大きい文字列」を生成する複雑なロジックを含んでいます。この部分はロケール依存性が高いため、デバッグ時には注意が必要です。
  • 選択率推定において、ヒストグラムが小さい場合はヒューリスティックな手法にフォールバックする仕組みが組み込まれています。
