src/backend/utils/adt/jsonb_gin.c

概要

本ファイルは、PostgreSQL の jsonb 型に対する GIN（Generalized Inverted Index）インデックスのサポート機能を実装しています。jsonb_ops（キーと値のペアを個別にインデックス化）と jsonb_path_ops（パス全体をハッシュ化してインデックス化）という2つの演算子クラスを提供し、JSONB の包含演算子 (@>) や JSONPath 演算子 (@?, @@) の高速な検索を可能にします。

モジュール内での役割

src/backend/utils/adt モジュールの一部として、jsonb 型のインデックスアクセスメソッドを定義します。GIN インデックスの構築（extract）および検索時の整合性チェック（consistent）を担い、プランナがインデックスを利用したクエリ実行計画を生成するための基盤を提供します。

責務

  • **インデックス構築**: JSONB ドキュメントを走査し、GIN インデックスに格納するキー（Datum）を抽出する。
  • **クエリ変換**: SQL クエリ（JSONPath 等）を解析し、GIN インデックスで検索可能な論理ツリー（JsonPathGinNode）へ変換する。
  • **整合性判定**: インデックスから得られた候補がクエリ条件を満たすか判定する（consistent / triconsistent）。
  • **データ正規化**: インデックスキーとして保存する際の型変換とハッシュ化。

重要なシンボル

インデックス構築・変換用構造体

  • GinEntries: 抽出されたインデックスキーを保持する動的配列バッファ。
  • JsonPathGinNode: JSONPath クエリを GIN 検索用に分解した論理ツリーのノード。
  • JsonPathGinPathItem: jsonb_ops におけるパスアイテムの連結リスト。
  • PathHashStack: jsonb_path_ops におけるネストされたパスのハッシュ計算用スタック。

演算子クラスサポート関数

  • gin_extract_jsonb / gin_extract_jsonb_path: インデックス作成時に JSONB からキーを抽出。
  • gin_extract_jsonb_query / gin_extract_jsonb_query_path: クエリ実行時に検索条件からキーを抽出。
  • gin_consistent_jsonb / gin_consistent_jsonb_path: インデックスの検索結果が条件を満たすか判定。
  • gin_triconsistent_jsonb / gin_triconsistent_jsonb_path: 3値論理（TRUE/FALSE/MAYBE）による判定。

制御フローとデータフロー

1. **抽出フロー**: gin_extract_jsonb が JsonbIterator を用いてドキュメントを走査し、make_scalar_key を介して Datum を生成、add_gin_entry でバッファに蓄積します。 2. **クエリ解析フロー**: extract_jsp_query が JSONPath を解析し、JsonPathGinContext を用いて JsonPathGinNode ツリーを構築します。 3. **実行フロー**: execute_jsp_gin_node が再帰的にツリーを走査し、インデックスの検索結果（check 配列）と照合して最終的な整合性を判定します。

読者向けメモ

  • **メモリ管理**: GinEntries は palloc を使用して動的に拡張されます。JsonPathGinNode ツリーはクエリ実行中のみ有効なメモリコンテキストに割り当てられます。
  • **再チェックの必要性**: GIN インデックスは本質的に「損失あり（lossy）」です。特に jsonb_path_ops はハッシュ衝突や構造情報の欠落があるため、consistent 関数は常に recheck = true を設定し、ヒープ上の実データで再検証する必要があります。
  • **制約**: jsonb_ops はキーの長さが JGIN_MAXLENGTH を超える場合、ハッシュ化して保存します。
  • **省略された詳細**: 本ファイルでは JSONPath の複雑なフィルタ条件の一部（NOT EXISTS 等）はサポートされておらず、これらはインデックスを使用せずフルスキャンにフォールバックされます。
