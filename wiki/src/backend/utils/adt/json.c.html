src/backend/utils/adt/json.c

概要

PostgreSQLにおける json 型の入出力、変換、および集約処理を実装するコアモジュールです。JSONデータの妥当性検証、型変換（DatumからJSONへのシリアライズ）、およびJSONオブジェクト内のキー重複チェック機能を提供します。

モジュール内での役割

src/backend/utils/adt モジュール群の一部として、JSONデータ型に対するSQLインターフェースを定義します。jsonb 型とは異なり、テキストベースのJSON処理に特化しており、主に json 型の型変換関数や集約関数（json_agg, json_object_agg）のバックエンド実装を担います。

責務

  • **JSONシリアライズ**: SQLの各データ型（Datum）をJSON形式の文字列へ変換する。
  • **キー重複チェック**: JSONオブジェクト構築時や解析時に、同一キーの存在を高速に検出する。
  • **集約処理**: json_agg や json_object_agg の状態管理と最終的なJSON文字列の生成。
  • **文字列エスケープ**: JSON仕様に従った特殊文字のエスケープ処理（SIMD最適化を含む）。

重要なシンボル

  • datum_to_json_internal: 任意のDatumをJSON形式に変換する中心的な再帰関数。
  • escape_json_with_len: SIMD命令を活用して高速に文字列をエスケープする関数。
  • JsonAggState: 集約関数実行中の状態を保持する構造体。
  • JsonUniqueParsingState: JSON解析時のキー重複チェック用コンテキスト。
  • json_in / json_out: json 型の標準的な入出力関数。

制御フローとデータフロー

1. **変換**: SQL関数（to_json等）が呼び出されると、datum_to_json_internal が型カテゴリ（JsonTypeCategory）に基づいて適切な変換処理へ分岐します。 2. **再帰**: 配列や複合型（レコード）の場合、array_to_json_internal や composite_to_json が再帰的に呼び出され、ネストされた構造を構築します。 3. **集約**: json_agg_transfn 等の遷移関数が JsonAggState を更新し、最終的に json_agg_finalfn がバッファを text 型として返します。 4. **検証**: json_validate は JsonLexContext を使用して構文解析を行い、必要に応じてハッシュテーブル（JsonUniqueCheckState）を用いてキーの重複を検証します。

読者向けメモ

  • **SIMD最適化**: escape_json_with_len はパフォーマンス向上のためSIMD命令を使用しています。この領域を修正する場合は、ベクトル演算の境界条件に注意してください。
  • **メモリ管理**: 集約関数では MemoryContext の切り替えが頻繁に行われます。JsonAggState のライフサイクルは集約の実行期間に依存するため、メモリリークや不正なポインタ参照に注意が必要です。
  • **キー重複**: json_object_agg 等でキー重複チェックを行う際、ハッシュテーブルのキーとして文字列をコピーする処理が含まれています。これはバッファの再配置によるポインタ無効化を防ぐための設計です。
