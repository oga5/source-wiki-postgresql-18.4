src/backend/utils/adt/tsvector_parser.c

概要

tsvector_parser.c は、PostgreSQL の全文検索機能（Tsearch）において、tsvector および tsquery の入力文字列をトークン単位に分解・解析するための汎用パーサーを提供します。

モジュール内での役割

src/backend/utils/adt モジュール内で、テキスト検索関連のデータ型（tsvector, tsquery）の入力を処理する低レベルなユーティリティとして機能します。主に tsvector のテキスト表現を内部構造へ変換する際のフロントエンドとして利用されます。

責務

  • **トークン解析**: 入力文字列から単語、位置情報、重み（A/B/C/D）を抽出する。
  • **状態管理**: TSVectorParseStateData を通じて、解析の進捗やエラーコンテキストを保持する。
  • **エラーハンドリング**: ErrorSaveContext をサポートし、構文エラーを例外として投げるか、呼び出し元へ安全に通知する。
  • **柔軟な構文対応**: tsvector だけでなく、tsquery や websearch_to_tsquery 特有の構文ルール（演算子や引用符の扱い）をフラグで切り替える。

重要なシンボル

  • TSVectorParseStateData: 解析中の状態を保持する構造体。
  • init_tsvector_parser: パーサーの状態を初期化するコンストラクタ。
  • gettoken_tsvector: 入力文字列から次のトークンを切り出すメインループ関数。
  • close_tsvector_parser: 解析終了後にメモリを解放するデストラクタ。

追加の索引済みシンボル

  • prssyntaxerror
  • reset_tsvector_parser

制御フローとデータフロー

1. **初期化**: init_tsvector_parser で入力文字列と解析フラグをセットし、状態オブジェクトを生成します。 2. **トークン抽出**: gettoken_tsvector が有限オートマトン（WAITWORD, WAITENDWORD, INPOSINFO 等の状態コード）として動作します。 3. **状態遷移**: 入力文字列を1文字ずつ読み進め、現在の状態と文字種に応じて次の状態へ遷移します。 4. **出力**: トークンが確定すると RETURN_TOKEN マクロにより、呼び出し元へ単語文字列と位置情報配列が返されます。 5. **終了**: 入力文字列の終端に達するか、エラーが発生するまでループを繰り返します。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • tsearch/ts_locale.h: ロケール依存の文字処理関数。
  • tsearch/ts_utils.h: tsvector 関連のデータ構造や定数定義。

関連コード

  • src/backend/utils/adt/tsvector.c: 本パーサーを利用して tsvector 型の入力を処理する主要な実装。
  • src/backend/utils/adt/tsquery.c: tsquery 関連の処理。
  • src/include/tsearch/ts_utils.h: パーサーが使用するフラグ（P_TSV_OPR_IS_DELIM 等）が定義されています。

読者向けメモ

  • 本パーサーは tsvector と tsquery の両方で共有されています。is_tsquery フラグは主にエラーメッセージの文言を切り替えるために使用されます。
  • RESIZEPRSBUF マクロは、単語バッファが不足した際に動的に拡張します。
  • escontext を使用したソフトエラー処理は、バリデーション関数などで例外を発生させずにエラーを検知したい場合に重要です。
  • 状態遷移のロジックは複雑なため、新しい構文ルールを追加する際は、既存の WAIT 系状態コードの遷移図を整理してから修正することを推奨します。
