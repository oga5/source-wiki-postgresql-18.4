src/backend/utils/adt/varbit.c

概要

PostgreSQL の BIT および BIT VARYING 型の内部実装を提供します。これらの型は可変長データ型（varlena）として扱われ、ビット単位の操作、型変換、および SQL 標準の文字列操作関数を実装しています。

モジュール内での役割

src/backend/utils/adt モジュールの一部として、SQL レベルのビット列型と PostgreSQL の実行エンジンを仲介します。varlena 構造を拡張した VarBit 構造体を用い、メモリ効率とビット演算のパフォーマンスを両立させています。

責務

  • **型変換と I/O**: SQL 文字列（'101' 等）と内部バイナリ表現の相互変換、およびバイナリ送受信。
  • **ビット操作**: ビット単位の論理演算（AND, OR, XOR, NOT）、シフト、連結、部分抽出。
  • **型修飾子管理**: typmod（ビット長）の検証と適用。
  • **プランナ連携**: varbit_support による型変換の最適化（不要なキャストの削除）。
  • **インデックスサポート**: B-Tree 等の比較演算子（bit_cmp）の提供。

重要なシンボル

データ構造

  • VarBit: ビット列を保持する構造体。vl_len_（ヘッダー）、bit_len（ビット数）、bit_dat（可変長配列）で構成。

I/O および型変換

  • bit_in / varbit_in: 文字列から VarBit への変換。
  • bit_out / varbit_out: VarBit から文字列への変換。
  • bit_recv / bit_send: バイナリ形式の入出力。
  • bittypmodin / varbittypmodin: 型修飾子（長さ）の解析と検証。

ビット操作関数

  • bitcat: ビット列の連結。
  • bitsubstring: ビット列の部分抽出。
  • bit_and, bit_or, bitxor, bitnot: 論理演算。
  • bitshiftleft, bitshiftright: ビットシフト。
  • bitsetbit, bitgetbit: 特定位置のビット操作。

比較演算子

  • bit_cmp: 2つのビット列の比較（bitlt, bitgt 等の基盤）。

制御フローとデータフロー

  • **パディングの不変条件**: VarBit の末尾バイトの未使用ビットは常に 0 でなければなりません。VARBIT_PAD マクロがこの不変条件を維持するために使用されます。
  • **メモリ管理**: palloc を使用して動的にメモリを確保します。PG_GETARG_VARBIT_P でデトースト（必要に応じて展開）されたデータを受け取り、結果は PG_RETURN_VARBIT_P で返されます。
  • **比較フロー**: bit_cmp は memcmp でバイト単位の比較を行い、一致した場合のみビット長を比較します。
  • **シフト操作**: bitshiftleft/right は、バイト境界を跨ぐシフトをループ内で処理し、末尾のパディングを再調整します。

読者向けメモ

  • **メモリレイアウト**: VarBit は varlena 構造を継承していますが、データ領域の先頭に bit_len を持つため、VARHDRSZ に加えて VARBITHDRSZ のオフセットを考慮する必要があります。
  • **パフォーマンス**: ビット列の連結やシフトは、バイト境界を跨ぐ処理が必要なため、単純な memcpy よりも計算コストが高くなります。
  • **未実装/浅い部分**: bit_out は現在 varbit_out を呼び出しており、バイナリ形式の正確な復元を優先しています。16進数出力のロジックはコメントアウトされています。
  • **注意点**: bitgetbit / bitsetbit は 0-based の左から右へのインデックスを使用しますが、これは標準 SQL の SUBSTRING 等の 1-based インデックスと仕様が異なるため注意が必要です。
