src/backend/utils/adt/timestamp.c

概要

PostgreSQL の日時データ型である TIMESTAMP（タイムスタンプ）および INTERVAL（期間）のバックエンド実装です。SQL レベルの型定義と内部表現（int64 マイクロ秒単位）の相互変換、算術演算、比較、および型修飾子（typmod）の処理を担います。

モジュール内での役割

src/include/datatype/timestamp.h で定義されたデータ構造を操作する中心的な ADT（抽象データ型）実装です。datetime.c などの低レベルな日時計算ライブラリを呼び出し、PostgreSQL の実行エンジン（プランナ、エグゼキュータ）が利用可能な Datum インターフェースを提供します。

責務

  • **型変換とI/O**: SQL 文字列と内部表現の相互変換（_in, _out）、バイナリ転送（_recv, _send）。
  • **算術演算**: 加減算、乗除算、および期間の正規化（justify 関数群）。
  • **比較と集約**: B-Tree インデックス用比較関数、ハッシュ関数、および SUM/AVG 集約関数の状態管理。
  • **プランナサポート**: generate_series 等の行数推定や、型修飾子（typmod）の最適化。
  • **タイムゾーン処理**: AT TIME ZONE 構文の実装およびタイムゾーン変換。

重要なシンボル

型定義と状態管理

  • Timestamp, TimestampTz: int64（マイクロ秒単位）で表現される絶対時刻。
  • Interval: time（オフセット）、day、month の3フィールドで構成される期間構造体。
  • IntervalAggState: SUM/AVG 集約時に使用される中間状態構造体。

I/O および変換

  • timestamp_in, timestamp_out: 文字列形式との変換。
  • timestamp_recv, timestamp_send: バイナリ形式との変換。
  • timestamp2tm, tm2timestamp: struct pg_tm（POSIX 形式）と内部表現の変換。

演算と調整

  • AdjustTimestampForTypmod: typmod に基づく精度調整（丸め処理）。
  • timestamp_pl_interval, timestamp_mi_interval: 加減算。
  • interval_justify_hours, interval_justify_days: 期間の単位繰り上げ/繰り下げ。

制御フローとデータフロー

1. **入力**: _in 関数が ParseDateTime を呼び出し、文字列を解析。DecodeDateTime で struct pg_tm に展開後、tm2timestamp で int64 に変換。 2. **演算**: 演算子関数は int64（Timestamp）または Interval 構造体を直接操作。オーバーフローチェックには pg_add_s64_overflow 等の安全な算術関数を使用。 3. **出力**: _out 関数は timestamp2tm で struct pg_tm に戻し、EncodeDateTime で文字列化。 4. **集約**: interval_avg_accum が IntervalAggState を生成し、do_interval_accum で加算。最終的に interval_avg が interval_div を呼び出して平均を算出。

読者向けメモ

  • **メモリレイアウト**: Timestamp は int64 ですが、Interval は構造体であり、palloc されたメモリ領域を指すポインタとして扱われます。
  • **精度**: MAX_TIMESTAMP_PRECISION（6）を超えた精度は切り捨てられます。
  • **無限大**: TIMESTAMP_NOT_FINITE マクロで判定される無限大（+/-Infinity）の扱いには注意が必要です。算術演算で無限大同士の計算が発生するとエラーになります。
  • **浮動小数点**: -ffast-math コンパイルオプションは、日時計算の精度を損なうため禁止されています。
  • **未実装**: interval_abs() は、期間の絶対値の定義が曖昧であるため意図的に実装されていません。
