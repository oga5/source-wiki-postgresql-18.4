src/backend/utils/adt/inet_cidr_ntop.c

概要

ネットワークアドレス（IPアドレス）を、CIDR表記を含むプレゼンテーション形式（文字列）へ変換するためのユーティリティ関数群を提供します。

モジュール内での役割

src/backend/utils/adt モジュールの一部として、ネットワーク関連データ型（inet, cidr）の表示処理を支える低レベルな変換ロジックを担います。

責務

  • ネットワークバイトオーダーのバイナリ形式IPアドレスを、人間が読み取り可能な文字列形式へ変換する。
  • IPv4およびIPv6の両方に対応し、常にCIDRプレフィックス長（例: /24, /64）を付与する。
  • 変換時のバッファオーバーフローを防止し、エラー時には適切な errno を設定する。

重要なシンボル

  • pg_inet_cidr_ntop: 公開関数。アドレスファミリ（IPv4/IPv6）に応じて適切な変換関数へディスパッチする。
  • inet_cidr_ntop_ipv4: IPv4アドレスをCIDR表記に変換する内部関数。
  • inet_cidr_ntop_ipv6: IPv6アドレスをCIDR表記に変換する内部関数。ゼロ圧縮（::）の最適化を含む。

制御フローとデータフロー

1. **ディスパッチ**: pg_inet_cidr_ntop が af 引数を確認し、IPv4 または IPv6 の処理へ分岐する。 2. **正規化**: 各関数は、指定されたビット数（bits）に基づき、ホスト部をゼロクリアした一時的なバイナリ表現を作成する。 3. **文字列構築**:

  • IPv4: オクテット単位で文字列化し、最後に /bits を連結する。
  • IPv6: 16ビット単位で処理し、連続するゼロを :: に短縮するロジックを適用した後、文字列化する。

    • 4. **出力**: 変換後の文字列を dst バッファにコピーし、成功時はポインタを、失敗時は NULL を返す。

直接依存

  • postgres.h: PostgreSQLの基本定義。
  • <sys/socket.h>, <netinet/in.h>, <arpa/inet.h>: 標準的なソケットおよびネットワークアドレス定義。
  • utils/builtins.h, utils/inet.h: PostgreSQL内部のネットワーク型定義と関数プロトタイプ。
  • arpa/inet.h
  • netinet/in.h
  • sys/socket.h

関連コード

  • src/backend/utils/adt/inet_net_pton.c: 本ファイルの逆変換（文字列からバイナリへの変換）を行う対となるファイル。
  • src/backend/utils/adt/network.c: PostgreSQLの inet 型の演算子や関数を実装する上位モジュール。

読者向けメモ

  • 本ファイルはISCライセンスに基づくコードを含んでおり、PostgreSQLの他の部分とはやや異なるコーディングスタイル（SPRINTF マクロの使用など）が見られます。
  • IPv6の変換ロジックは、アドレスの短縮表現（ゼロ圧縮）を考慮した複雑な実装になっています。デバッグや修正を行う際は、bits によるプレフィックス長制限が正しく適用されているか注意してください。
  • size 引数によるバッファサイズチェックが厳密に行われており、領域不足時は EMSGSIZE が返されます。
