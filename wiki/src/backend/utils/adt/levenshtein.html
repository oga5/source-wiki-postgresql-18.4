src/backend/utils/adt/levenshtein.c

概要

文字列間のレーベンシュタイン距離（編集距離）を計算するための実装です。挿入、削除、置換のコストを個別に指定可能であり、マルチバイト文字にも対応しています。

モジュール内での役割

src/backend/utils/adt 内で、文字列の類似度判定や曖昧検索機能の基盤を提供します。このファイルは単体でコンパイルされるのではなく、varlena.c から異なるマクロ定義（LEVENSHTEIN_LESS_EQUAL の有無）で複数回インクルードされることで、最適化されたバリアントを生成します。

責務

  • **距離計算**: 2つの文字列間の最小編集コストを算出する。
  • **メモリ効率**: 動的計画法において、行列全体を保持せず、2行分（現在と直前）のメモリのみを使用して計算する。
  • **計算量削減**: max_d（許容最大距離）が指定された場合、対角線付近のみを計算する枝刈りアルゴリズムを適用し、不要な計算をスキップする。
  • **マルチバイト対応**: 文字列をバイト単位ではなく文字単位で処理し、必要に応じて文字長をキャッシュする。

重要なシンボル

  • varstr_levenshtein: 基本的な編集距離計算関数。
  • varstr_levenshtein_less_equal: max_d を指定し、閾値を超えた時点で計算を打ち切る最適化版。
  • MAX_LEVENSHTEIN_STRLEN: セキュリティ上の理由から、信頼できない入力に対して課される最大文字列長（255文字）。

制御フローとデータフロー

1. **初期化**: 入力文字列の文字数をカウントし、必要に応じて文字長キャッシュを作成します。 2. **メモリ確保**: 計算に必要な2行分の作業領域を palloc で確保します。 3. **ループ処理**:

  • 外側のループでターゲット文字列の各文字を走査します。
  • 内側のループでソース文字列とのコストを比較・更新します。
  • LEVENSHTEIN_LESS_EQUAL が有効な場合、start_column と stop_column を動的に更新し、計算範囲を絞り込みます。

    • 4. **結果返却**: 最終的な編集距離を返します。

直接依存

  • postgres.h: PostgreSQLの基本定義。
  • mb/pg_wchar.h (間接的): 文字列操作およびマルチバイト文字の長さ判定に使用。
  • utils/builtins.h: 関連する組み込み関数の宣言。

関連コード

  • src/backend/utils/adt/varlena.c: 本ファイルをインクルードし、実際のSQL関数（levenshtein 等）のエントリポイントを提供します。

読者向けメモ

  • **インクルードの仕組み**: 本ファイルは LEVENSHTEIN_LESS_EQUAL マクロの有無によって挙動が変わる特殊な構成です。直接呼び出すのではなく、varlena.c 経由で提供される関数を使用してください。
  • **パフォーマンス**: マルチバイト文字が含まれない場合、高速なパス（s_char_len == NULL）が選択されます。
  • **セキュリティ**: trusted フラグが false の場合、入力長が MAX_LEVENSHTEIN_STRLEN を超えるとエラーになります。これは計算量が $O(mn)$ であるため、DoS攻撃を防ぐための制限です。
