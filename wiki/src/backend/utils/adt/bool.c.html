src/backend/utils/adt/bool.c

概要

PostgreSQLの組み込み型 boolean に関連するすべての操作を定義するソースファイルです。型変換、入出力、比較演算、ハッシュ計算、および集約関数（bool_and, bool_or 等）の実装を含みます。

モジュール内での役割

src/backend/utils/adt モジュールの一部として、データ型ごとの基本操作を提供する役割を担います。SQLレベルの boolean 型と、内部的な bool 型（C言語の bool）の橋渡しを行います。

責務

  • **文字列解析**: true, false, yes, no, on, off, 1, 0 などの多様な入力形式の正規化。
  • **I/O処理**: テキスト形式およびバイナリ形式（boolrecv/boolsend）の相互変換。
  • **比較・ハッシュ**: boolean 型の大小比較およびハッシュインデックス用のハッシュ値計算。
  • **集約関数**: bool_and や bool_or などの集約計算に必要な状態管理と計算ロジックの提供。

重要なシンボル

  • parse_bool_with_len: 文字列をブール値に変換するコア関数。柔軟な入力を許容します。
  • boolin / boolout: SQLの boolean 型の入出力関数。
  • booland_statefunc / boolor_statefunc: 標準的な集約関数用の状態遷移関数。
  • BoolAggState: 集約計算中に保持される内部状態構造体。
  • bool_accum / bool_alltrue: 複雑な集約計算（EVERYやANYなど）をサポートするための状態蓄積および最終計算関数。

追加の索引済みシンボル

  • bool_accum_inv
  • bool_anytrue
  • boolrecv
  • parse_bool

制御フローとデータフロー

1. **入力**: boolin が文字列を受け取り、parse_bool_with_len を呼び出して bool 型へ変換します。 2. **演算**: 比較演算子（booleq 等）は、PostgreSQLの Datum インターフェースを通じて引数を受け取り、Cの論理演算結果を PG_RETURN_BOOL で返します。 3. **集約**: bool_accum が呼び出されるたびに BoolAggState が更新され、最終的に bool_alltrue 等がその状態を読み取って結果を算出します。

直接依存

  • postgres.h: PostgreSQLの基本定義。
  • ctype.h: 文字判定（isspace 等）。
  • common/hashfn.h: ハッシュ計算用ユーティリティ。
  • libpq/pqformat.h: バイナリ通信用プロトコル関数。
  • utils/builtins.h: 組み込み関数のプロトタイプ宣言。

関連コード

  • src/include/utils/builtins.h: 本ファイルで定義された関数のSQLインターフェース定義。
  • src/backend/utils/adt/numutils.c: 数値変換ユーティリティ（関連する解析ロジックの参考）。

読者向けメモ

  • parse_bool_with_len は非常に寛容なパーサーであり、設定ファイル（postgresql.conf）の解析など、他の場所でも再利用される重要なロジックです。
  • booltext は boolout とは異なり、SQL標準に準拠した文字列（true/false）を返す必要があるため、個別に実装されています。
  • 集約関数において、bool_accum は bool_accum_inv とペアで動作し、ウィンドウ関数やインクリメンタルな集約をサポートする設計になっています。
