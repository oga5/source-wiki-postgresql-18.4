src/backend/utils/mb/stringinfo_mb.c

概要

本ファイルは、PostgreSQLの文字列バッファ管理機能である StringInfo に対し、マルチバイト文字を考慮した拡張機能を提供します。主にSQLクエリの生成やログ出力において、文字列を安全にクォートしてバッファへ追加する処理を担います。

モジュール内での役割

src/backend/utils/mb サブシステムの一部として、common/stringinfo.c に依存せず、かつマルチバイトエンコーディングの知識を必要とする文字列操作を分離しています。これにより、フロントエンド（クライアント側）のコードが不要なマルチバイト関連の依存関係を取り込むことを防いでいます。

責務

  • **マルチバイト対応のクォート処理**: 入力文字列内のシングルクォートをエスケープ（二重化）し、全体をシングルクォートで囲んで StringInfo に追加する。
  • **長さ制限付きの安全な切り詰め**: 指定されたバイト数（maxlen）を超えない範囲で、マルチバイト文字の境界を考慮して文字列を切り詰める。
  • **省略記号の付与**: 切り詰めが発生した場合に、末尾に ... を付与して可読性を維持する。

重要なシンボル

  • appendStringInfoStringQuoted: 本ファイルの唯一の公開関数。文字列をクォートしてバッファに追加するメインロジック。

制御フローとデータフロー

1. **初期化**: maxlen が指定されている場合、pg_mbcliplen を使用してマルチバイト文字の境界を壊さないように文字列を切り詰め、一時メモリにコピーします。 2. **クォート開始**: バッファの先頭に ' を追加します。 3. **エスケープ処理**: strchr を用いて入力文字列内の ' を検索し、見つかるたびにその直前までの文字列と ' 自体をバッファに追加します。次のループで ' を再度追加することで、結果として ' が '' にエスケープされます。 4. **終了処理**: 最後に残った文字列を追加し、必要に応じて ... を付与した後、末尾に ' を追加して閉じます。

直接依存

  • postgres.h: 基本的な型定義とマクロ。
  • mb/pg_wchar.h: pg_mbcliplen などのマルチバイト文字操作関数。
  • mb/stringinfo_mb.h: 本関数のプロトタイプ宣言。

関連コード

  • lib/stringinfo.h: StringInfo データ構造および appendStringInfoCharMacro などの基本操作の定義元。
  • common/stringinfo.c: StringInfo の汎用的な実装。本ファイルはこれの「マルチバイト対応版」という位置付けです。

読者向けメモ

  • 本関数は、SQLインジェクション対策の完全な代替ではありません。あくまでログ出力やデバッグ、あるいは信頼されたコンテキスト内での文字列構築を想定しています。
  • maxlen の指定はバイト単位であることに注意してください。マルチバイト文字の途中で切れないよう、内部で pg_mbcliplen が適切に処理を行っています。
  • メモリ管理のため、切り詰めが発生した場合は一時的に palloc を使用し、処理終了後に pfree を行っています。
