src/backend/utils/mb/conv.c

概要

PostgreSQLにおける文字エンコーディング変換の低レベルなユーティリティ関数群を提供します。主に、特定の文字セット間での変換や、UTF-8とローカルエンコーディング間の変換を効率的に行うための基盤を提供します。

モジュール内での役割

src/backend/utils/mb サブシステムにおいて、変換テーブルやラディックスツリーを用いた具体的な変換ロジックを実装する層です。mbutils.c が提供する高レベルな変換インターフェースのバックエンドとして機能します。

責務

  • 単一バイト文字セット間の変換（local2local 等）。
  • Mule Internal Code (MIC) との相互変換。
  • ラディックスツリーを用いた高速な文字変換（pg_mb_radix_conv）。
  • UTF-8 とローカルエンコーディング間の双方向変換（UtfToLocal, LocalToUtf）。
  • 変換失敗時のエラー報告および不正なエンコーディングの検出。

重要なシンボル

  • UtfToLocal / LocalToUtf: UTF-8とローカルエンコーディング間の変換を行うメイン関数。
  • pg_mb_radix_conv: ラディックスツリー構造を使用して、マルチバイト文字を効率的に検索・変換する関数。
  • local2local: 単純なテーブル引きによる単一バイト文字セット変換。
  • compare3 / compare4: bsearch 用の比較関数。結合文字の検索に使用されます。

追加の索引済みシンボル

  • latin2mic
  • latin2mic_with_table
  • mic2latin
  • mic2latin_with_table
  • store_coded_char

制御フローとデータフロー

1. **入力処理**: 入力文字列をバイト単位で読み込み、pg_utf_mblen や pg_encoding_verifymbchar で文字境界を特定します。 2. **変換試行**:

  • まず結合文字マップ（cmap）を検索（bsearch を使用）。
  • 次にラディックスツリー（map）を検索（pg_mb_radix_conv を使用）。
  • 最後にアルゴリズム的な変換関数（conv_func）を適用。

    • 3. **出力**: 変換されたコードポイントを store_coded_char でマルチバイトストリームへ書き込みます。 4. **エラーハンドリング**: 変換不可または不正なバイト列が検出された場合、noError フラグに応じてエラーを報告するか、処理を中断します。

直接依存

  • postgres.h: 基本的な型定義やエラー報告マクロ（ereport, elog）を利用。
  • mb/pg_wchar.h: 文字エンコーディングの定義、ラディックスツリー構造体、およびマルチバイト処理用マクロを利用。

関連コード

  • src/backend/utils/mb/mbutils.c: 本ファイルの関数を呼び出す上位層。
  • src/include/mb/pg_wchar.h: 変換に使用されるデータ構造（pg_mb_radix_tree 等）の定義。

読者向けメモ

  • 本ファイルは非常にパフォーマンスに敏感なコードです。新しい変換テーブルを追加する際は、pg_mb_radix_tree の構造を理解することが不可欠です。
  • noError フラグは、変換が途中で失敗してもエラーを投げずに呼び出し元へ制御を戻すために使用されます。このフラグを扱う際は、戻り値（消費されたバイト数）が正しく計算されているか注意してください。
  • 結合文字（Combined characters）の処理は、通常の文字変換よりも優先される場合があるため、変換順序のロジックを読み解く際は注意が必要です。
