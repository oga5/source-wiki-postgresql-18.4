src/backend/utils/mb/conversion_procs/utf8_and_gb18030/utf8_and_gb18030.c

概要

PostgreSQL における GB18030 エンコーディングと UTF-8 エンコーディング間の相互変換を行うためのロード可能なモジュールです。

モジュール内での役割

PostgreSQL の文字コード変換サブシステムの一部として、pg_conversion システムカタログから呼び出される変換プロシージャを提供します。

責務

  • GB18030 と UTF-8 間の双方向変換の実行。
  • GB18030 の 4 バイト文字と Unicode コードポイント間の線形変換（gb_linear / gb_unlinear）。
  • UTF-8 のバイト列と Unicode コードポイント間の変換（unicode_to_utf8word / utf8word_to_unicode）。
  • 変換マッピングテーブル（.map ファイル）を用いた文字コード変換の仲介。

重要なシンボル

  • gb18030_to_utf8: GB18030 から UTF-8 への変換を行う SQL 呼び出し可能な関数。
  • utf8_to_gb18030: UTF-8 から GB18030 への変換を行う SQL 呼び出し可能な関数。
  • conv_18030_to_utf8 / conv_utf8_to_18030: マッピングテーブル外の特殊な範囲（主に 4 バイト GB18030 文字）を処理するロジック。

追加の索引済みシンボル

  • gb_linear
  • gb_unlinear
  • unicode_to_utf8word
  • utf8word_to_unicode

制御フローとデータフロー

1. **エントリポイント**: gb18030_to_utf8 または utf8_to_gb18030 が呼び出されます。 2. **検証**: CHECK_ENCODING_CONVERSION_ARGS マクロで引数の妥当性を確認します。 3. **変換処理**: LocalToUtf または UtfToLocal を呼び出し、変換テーブルと補助的な変換関数（conv_...）を使用してバイト列を変換します。 4. **特殊範囲の計算**: 4 バイト GB18030 文字の場合、gb_linear で線形空間にマッピングし、Unicode コードポイントを算出した後、UTF-8 バイト列へ変換します。

  • mb/pg_wchar.h: マルチバイト文字処理の定義。
  • gb18030_to_utf8.map, utf8_to_gb18030.map: 変換用マッピングテーブル。

読者向けメモ

  • 本ファイルは主に 4 バイト GB18030 文字の計算ロジックを含んでいます。基本的な 1〜2 バイト文字の変換は、インクルードされている .map ファイル内のツリー構造によって処理されます。
  • gb_linear 等の関数は、GB18030 の複雑なバイト構造を計算可能な線形空間に変換するためのものであり、このアルゴリズムは GB18030 の仕様（GB18030-2000）に基づいています。
