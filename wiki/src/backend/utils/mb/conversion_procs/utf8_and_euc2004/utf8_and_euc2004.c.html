src/backend/utils/mb/conversion_procs/utf8_and_euc2004/utf8_and_euc2004.c

概要

本ファイルは、PostgreSQLにおける文字コード変換モジュールの一部であり、EUC_JIS_2004 と UTF8 間の相互変換機能を提供します。PostgreSQLの動的ロード可能な変換プロシージャとして実装されており、データベースのエンコーディング変換処理を担います。

モジュール内での役割

src/backend/utils/mb/conversion_procs/ 配下に配置される変換プラグインの一つです。PostgreSQLの文字コード変換フレームワークに対し、特定のエンコーディングペア（EUC_JIS_2004 ⇔ UTF8）を処理するための関数インターフェースを提供します。

責務

  • EUC_JIS_2004 から UTF8 への変換処理の提供。
  • UTF8 から EUC_JIS_2004 への変換処理の提供。
  • 変換時の引数検証（CHECK_ENCODING_CONVERSION_ARGS）およびエラーハンドリングの制御。
  • 外部定義された変換マップ（.map ファイル）を利用した文字コードのルックアップ。

重要なシンボル

  • euc_jis_2004_to_utf8: EUC_JIS_2004 文字列を UTF8 に変換するPostgreSQL関数。
  • utf8_to_euc_jis_2004: UTF8 文字列を EUC_JIS_2004 に変換するPostgreSQL関数。
  • euc_jis_2004_to_unicode_tree / euc_jis_2004_from_unicode_tree: 変換マップから生成された、高速検索用のツリー構造。

制御フローとデータフロー

1. **呼び出し**: PostgreSQLの変換フレームワークから、引数（ソース文字列、長さ、変換先バッファ等）と共に各関数が呼び出されます。 2. **検証**: CHECK_ENCODING_CONVERSION_ARGS マクロにより、エンコーディングIDが正しいか確認されます。 3. **変換実行**:

  • LocalToUtf または UtfToLocal 汎用関数を呼び出します。
  • 内部でインクルードされた .map ファイル内の変換テーブルとツリー構造を参照し、バイト単位の変換を行います。

    • 4. **戻り値**: 正常に変換されたバイト数を返し、変換フレームワークへ制御を戻します。

直接依存

  • postgres.h, fmgr.h: PostgreSQLのコア機能および関数管理用ヘッダー。
  • mb/pg_wchar.h: マルチバイト文字処理の共通定義。
  • euc_jis_2004_to_utf8.map, utf8_to_euc_jis_2004.map: 変換ロジックの根幹となる文字コード対応表。
  • ../../Unicode/euc_jis_2004_to_utf8.map
  • ../../Unicode/utf8_to_euc_jis_2004.map

関連コード

  • src/backend/utils/mb/: 文字コード変換全般を管理するサブシステム。
  • src/include/mb/pg_wchar.h: 変換関数が利用する LocalToUtf や UtfToLocal の定義元。

読者向けメモ

  • 本ファイル自体は変換ロジックの「接着剤」であり、実際の変換テーブルはインクルードされている .map ファイルに依存しています。
  • 変換テーブルの更新が必要な場合は、直接このCファイルを編集するのではなく、対応するマップ生成スクリプトやソースデータを確認してください。
  • noError フラグが true の場合、変換失敗時にエラーを投げずに処理を継続する挙動をとります。これは変換関数を呼び出す側の要件に従います。
