src/backend/utils/mb/conversion_procs/euc2004_sjis2004/euc2004_sjis2004.c

概要

PostgreSQL における EUC_JIS_2004 と SHIFT_JIS_2004 間の文字コード変換を実装するロード可能なモジュールです。JIS X 0213:2004 規格に基づく相互変換機能を提供します。

モジュール内での役割

PostgreSQL の文字コード変換サブシステムの一部として、動的ロード可能な変換プロシージャを提供します。fmgr（Function Manager）を介して、データベースのエンコーディング変換時に呼び出されます。

責務

  • EUC_JIS_2004 から SHIFT_JIS_2004 への変換。
  • SHIFT_JIS_2004 から EUC_JIS_2004 への変換。
  • 不正なバイトシーケンスの検出と、必要に応じたエラー報告（または noError フラグによるスキップ）。

重要なシンボル

  • euc_jis_2004_to_shift_jis_2004: SQL から呼び出される変換関数（EUC→SJIS）。
  • shift_jis_2004_to_euc_jis_2004: SQL から呼び出される変換関数（SJIS→EUC）。
  • euc_jis_20042shift_jis_2004: EUC から SJIS への変換ロジック本体。
  • shift_jis_20042euc_jis_2004: SJIS から EUC への変換ロジック本体。
  • get_ten: SJIS の第 2 バイトから「点（ten）」と「区（ku）」の偶奇を算出する補助関数。

制御フローとデータフロー

1. **エントリポイント**: PG_FUNCTION_ARGS を介してソース文字列、ターゲットバッファ、長さ、エラーハンドリングフラグを受け取ります。 2. **検証**: CHECK_ENCODING_CONVERSION_ARGS マクロでエンコーディング ID の整合性を確認します。 3. **変換ループ**:

  • 入力文字列を 1 文字ずつ走査します。
  • pg_encoding_verifymbchar でマルチバイト文字の妥当性を検証します。
  • 各エンコーディングの仕様（SS2/SS3 の有無や区点位置）に基づき、ビット演算で変換先のコードポイントを計算します。

    • 4. **終了**: 変換済みバイト数を返し、ターゲットバッファをヌル終端します。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • fmgr.h: 関数マネージャインターフェース。
  • mb/pg_wchar.h: マルチバイト文字処理の定義（SS2, SS3, IS_HIGHBIT_SET 等）。

関連コード

  • src/backend/utils/mb/: 文字コード変換の基盤となるディレクトリ。
  • src/include/mb/pg_wchar.h: エンコーディング ID やマルチバイト文字の検証関数の定義。
  • src/backend/utils/mb/conversion_procs/: 他のエンコーディング変換プロシージャ群。

読者向けメモ

  • 本ファイルは JIS X 0213:2004 の複雑なマッピングを直接実装しています。
  • ku（区）と ten（点）の計算は、JIS コードと Shift_JIS の仕様上のオフセットに基づいています。
  • 不正なバイト列に遭遇した際、noError が true であれば変換を中断して終了しますが、false の場合は report_invalid_encoding を呼び出し、エラーを発生させます。
  • 変換ロジックを変更する際は、JIS X 0213 の規格書と照らし合わせ、特に第 2 水準や第 3/4 水準の扱いが正しいか注意してください。
