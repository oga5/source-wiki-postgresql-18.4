src/backend/parser/parse_clause.c

概要

SQL クエリの FROM 句、WHERE 句、LIMIT 句、GROUP BY 句、ORDER BY 句、およびウィンドウ関数定義のセマンティック解析（意味解析）を担うモジュールです。構文解析器が生成した生の構文木を、プランナが処理可能な Query ツリーへ変換する過程で、リレーションの解決、名前空間の構築、型整合性の検証を行います。

モジュール内での役割

src/backend/parser サブシステムにおいて、クエリの各句を個別に処理する役割を持ちます。特に FROM 句の解析は、ParseState 内の p_namespace（名前空間）や p_rtable（レンジテーブル）を構築する中核的な処理であり、parse_relation.c や parse_expr.c と密接に連携します。

責務

  • **FROM 句の変換**: RangeVar や JoinExpr を解析し、レンジテーブルエントリ（RTE）の生成と名前空間への登録を行う。
  • **JOIN 処理**: NATURAL JOIN や USING 句の展開、ON 句の式変換、および結合による可視性の制御。
  • **句の型解決**: WHERE, LIMIT, GROUP BY, ORDER BY 句の式を変換し、期待される型（boolean, int8 等）へのキャスト挿入や検証を行う。
  • **ウィンドウ関数・集約解析**: PARTITION BY や ORDER BY の解析、およびウィンドウフレームオフセットの型解決。
  • **制約・権限チェック**: ON CONFLICT 句のインデックス推論や、ターゲットテーブルに対する権限検証。

重要なシンボル

FROM 句解析

  • transformFromClause: FROM リストを走査し、各アイテムを変換して名前空間を構築するメインエントリ。
  • transformFromClauseItem: 再帰的に RangeVar, JoinExpr, RangeSubselect 等を処理するディスパッチャ。
  • setTargetTable: INSERT/UPDATE/DELETE のターゲットテーブルをレンジテーブルに追加し、ロックを取得する。

句の変換・検証

  • transformWhereClause: WHERE 句の式を変換し、結果が boolean であることを保証する。
  • transformLimitClause: LIMIT 句を変換し、int8 へのキャストと変数を含まないことの検証を行う。
  • transformGroupClause / transformSortClause: GROUP BY および ORDER BY 句を変換し、SortGroupClause リストを生成する。

内部ヘルパー

  • buildVarFromNSColumn: 名前空間カラムから Var ノードを構築する。
  • buildMergedJoinVar: JOIN の結果としてマージされたカラムの式（COALESCE 等）を生成する。
  • markRelsAsNulledBy: 外部結合において、結合によって NULL 化されるリレーションをマークする。

制御フローとデータフロー

1. **FROM 句**: transformFromClause がリストを左から右へ走査。transformFromClauseItem が各要素を RTE に変換し、p_namespace に追加。JOIN の場合は再帰的に左右のサブツリーを処理し、p_joinexprs を更新する。 2. **JOIN 解決**: transformJoinUsingClause が USING 句を ON 句の等価条件に変換。buildMergedJoinVar が結合後のカラムの型を解決し、必要に応じて COALESCE やキャストを挿入する。 3. **式変換**: 各句の解析関数は transformExpr を呼び出し、式を Query ツリーの一部として変換。その後、coerce_to_boolean や coerce_to_specific_type で型を強制する。 4. **名前空間管理**: setNamespaceLateralState や setNamespaceColumnVisibility を用いて、LATERAL 参照や結合による可視性のスコープを動的に制御する。

読者向けメモ

  • **メモリ管理**: ParseState は解析セッションを通じて生存し、palloc によって割り当てられたメモリは解析終了時に free_parsestate で解放されます。
  • **再帰的解析**: transformFromClauseItem は JoinExpr を再帰的に処理するため、非常に深い結合ツリーではスタックオーバーフローのリスクがあります（check_stack_depth() で保護）。
  • **LATERAL 参照**: LATERAL の可視性は p_namespace の操作と setNamespaceLateralState によって厳密に制御されています。
  • **未実装・浅い部分**: RangeTableFunc は現在 XMLTABLE に特化しており、JSON_TABLE は別ファイルに委譲されています。また、CUBE や ROLLUP の解析は flatten_grouping_sets を経由して複雑な正規化が行われます。
