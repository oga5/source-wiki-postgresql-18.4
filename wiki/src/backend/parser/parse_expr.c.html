src/backend/parser/parse_expr.c

概要

parse_expr.c は、PostgreSQL のパーサーにおいて、SQL 式（Expression）の「意味解析（Transformation）」を担う中核ファイルです。構文解析器（Bison）が生成した生の構文木（A_Expr, FuncCall 等）を、プランナが処理可能な意味的に確定したノード（OpExpr, FuncExpr 等）へ変換します。

モジュール内での役割

src/backend/parser モジュールにおいて、式レベルの型解決、演算子・関数のオーバーロード解決、および名前解決を行うためのディスパッチャーとして機能します。analyze.c から呼び出され、クエリの各句（SELECT リスト、WHERE 句など）に含まれる式を再帰的に変換します。

責務

  • **式変換のディスパッチ**: ノードの種類に応じて適切な変換関数を呼び出す。
  • **型解決とキャスト**: 式の型を決定し、必要に応じて暗黙的な型変換（Coercion）を挿入する。
  • **名前解決**: ColumnRef を Var ノードや関数呼び出しに解決する。
  • **SQL/JSON 変換**: SQL/JSON 関連の式（JSON_VALUE, JSON_TABLE 等）の複雑な型解決とバリデーション。
  • **エラー報告**: 式が許可されていない場所（例: DEFAULT 句内のサブクエリ）でのエラー検出。

重要なシンボル

エントリポイント

  • transformExpr: 式解析の公開インターフェース。ParseState を更新し、再帰的な変換を開始する。
  • transformExprRecurse: 式ノードのタグに基づいて変換関数を振り分けるメインループ。

変換関数群

  • transformColumnRef: 列参照（table.col 等）を解決し、名前空間から Var を生成する。
  • transformFuncCall: 関数呼び出しを解決し、FuncExpr や集約関数へ変換する。
  • transformAExprOp: 演算子式（=, + 等）を解決し、OpExpr 等へ変換する。
  • transformCaseExpr: CASE 文を変換し、型整合性をチェックする。
  • transformSubLink: サブクエリ式を解析し、SubLink ノードを構築する。
  • transformTypeCast: 明示的なキャスト（::）を処理する。

JSON 関連

  • transformJsonFuncExpr: SQL/JSON 関数（JSON_VALUE 等）の変換。
  • transformJsonConstructorExpr: JSON_OBJECT 等のコンストラクタ変換。

制御フローとデータフロー

1. **初期化**: transformExpr が ParseState の p_expr_kind を設定し、解析コンテキストを保持する。 2. **再帰的走査**: transformExprRecurse がノードを走査。スタックオーバーフローを防ぐため check_stack_depth() を呼び出す。 3. **解決**: 各変換関数が catalog を参照し、演算子や関数の OID を特定。型が不一致な場合は parse_coerce.c を利用してキャストを挿入する。 4. **名前空間参照**: transformColumnRef が parse_relation.c の refnameNamespaceItem 等を呼び出し、現在のスコープ内のテーブルや列を特定する。 5. **完了**: 変換されたノードがツリーとして組み立てられ、最終的に Query 構造体の一部となる。

読者向けメモ

  • **メモリ管理**: 本ファイルで生成されるノードは CurrentMemoryContext（通常は ParseState のコンテキスト）に割り当てられます。
  • **再帰の深さ**: 非常に深い式ツリーはスタックオーバーフローを引き起こす可能性があるため、check_stack_depth() が各再帰ステップで呼び出されています。
  • **JSON 変換**: JSON 関連の変換ロジックは非常に複雑で、JsonReturning や JsonFormat といった専用の構造体を介して型とエンコーディングを厳密に管理しています。
  • **未実装/浅い部分**: transformIndirection は配列添字とフィールド選択を分離して処理しますが、複雑な多次元配列のインデックス操作は parse_coerce.c 等のヘルパーに強く依存しています。
