src/backend/parser/gramparse.h

概要

gramparse.h は、PostgreSQL の SQL 解析プロセスにおける「生の解析フェーズ（Raw Parsing）」、すなわち Flex（字句解析）と Bison（構文解析）の間で共有される内部定義を保持するヘッダーファイルです。

モジュール内での役割

本ファイルは、parser.c、gram.y、scan.l といったコア解析コンポーネント間でのみ使用されるプライベートなインターフェースです。解析結果を外部に公開する parser.h とは異なり、解析器内部のステート管理に特化しています。

責務

  • 字句解析器と構文解析器の間で共有されるデータ構造（base_yy_extra_type）の定義。
  • 解析器の初期化および実行に必要な関数プロトタイプの提供。
  • Flex の yyextra フィールドへ効率的にアクセスするためのマクロ定義。

重要なシンボル

  • base_yy_extra_type: 解析器の実行状態を保持する構造体。字句解析のルックアヘッド情報や、最終的な構文解析結果（parsetree）を保持します。
  • pg_yyget_extra(yyscanner): yyextra フィールドに直接アクセスするための最適化されたマクロ。Flex の標準関数よりも高速に動作します。
  • base_yylex() / base_yyparse(): それぞれ字句解析および構文解析のメインエントリーポイント。

制御フローとデータフロー

1. **初期化**: parser_init() を呼び出し、base_yy_extra_type 構造体を解析器に渡します。 2. **字句解析**: base_yylex() がトークンを生成し、必要に応じて base_yy_extra_type 内のルックアヘッド情報を更新します。 3. **構文解析**: base_yyparse() が base_yylex() を呼び出しながら構文木を構築し、最終結果を base_yy_extra_type の parsetree フィールドに格納します。

直接依存

  • nodes/parsenodes.h: 構文木のノード定義。
  • parser/scanner.h: 字句解析器の基本定義（core_yy_extra_type 等）。
  • gram.h: Bison が生成するトークン定義（scanner.h の後にインクルードする必要がある）。

関連コード

  • src/backend/parser/parser.c: 解析器のメイン制御フロー。
  • src/backend/parser/gram.y: SQL 文法定義（Bison）。
  • src/backend/parser/scan.l: SQL 字句定義（Flex）。
  • src/include/parser/parser.h: 解析器のパブリックインターフェース。

読者向けメモ

  • **スコープの制限**: このファイルはコア解析コンポーネント以外からはインクルードしないでください。外部向けの定義は parser.h に記述されています。
  • **インクルード順序**: gram.h は scanner.h の後にインクルードする必要があります。これは scanner.h が YYLTYPE を定義しているためです。
  • **パフォーマンス**: pg_yyget_extra マクロは Flex の構造を前提とした最適化を行っています。構造体のメモリレイアウトに依存しているため、変更時は注意が必要です。
