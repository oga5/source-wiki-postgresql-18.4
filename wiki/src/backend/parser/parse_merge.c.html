src/backend/parser/parse_merge.c

概要

PostgreSQL の MERGE 文を解析し、構文木（MergeStmt）をクエリツリー（Query）へ変換するパーサーモジュールです。

モジュール内での役割

src/backend/parser サブシステムの一部として、MERGE 特有の複雑なスコープ管理（ターゲットテーブルとソースリレーションの可視性制御）を担当します。analyze.c が呼び出す解析プロセスの中で、MERGE 固有のロジックを分離・実装しています。

責務

  • MERGE 文の構文解析結果を Query 構造体にマッピングする。
  • WHEN 句ごとのアクション（INSERT/UPDATE/DELETE）の妥当性検証と権限チェック。
  • アクションごとの名前空間（Namespace）の動的な可視性制御。
  • INSERT および UPDATE アクションにおけるターゲットリストの変換。

重要なシンボル

  • transformMergeStmt: MERGE 文の解析を行うメインエントリポイント。
  • setNamespaceForMergeWhen: WHEN 句の種類に応じて、ターゲットとソースの可視性を切り替えるヘルパー関数。
  • MergeAction: 各 WHEN 句に対応するアクション情報を保持する構造体。

追加の索引済みシンボル

  • setNamespaceVisibilityForRTE

制御フローとデータフロー

1. **初期化**: transformMergeStmt が Query ノードを作成し、WITH 句を処理します。 2. **検証**: WHEN 句の順序（到達不能な句のチェック）と権限を検証します。 3. **ターゲット設定**: setTargetTable を呼び出し、ターゲットリレーションを確定させます。 4. **ソース変換**: transformFromClause でソースリレーションを解析し、名前空間に追加します。 5. **アクション変換**: 各 WHEN 句をループ処理します。

  • setNamespaceForMergeWhen で、そのアクションが参照可能なリレーション（ターゲット/ソース）の可視性を設定します。
  • transformWhereClause で条件式を変換します。
  • CMD_INSERT や CMD_UPDATE に応じて、それぞれのターゲットリストを変換します。

    • 6. **完了**: 最終的な Query 構造体を返します。

読者向けメモ

  • MERGE は INSERT ON CONFLICT と異なり、EXCLUDED 疑似リレーションを持ちません。代わりに、ターゲットとソースの両方が名前空間に存在し、アクションごとに可視性が切り替わる仕組みになっています。
  • setNamespaceVisibilityForRTE は、名前空間内の ParseNamespaceItem を直接操作して可視フラグを書き換えるため、副作用に注意してください。
  • MERGE の結合条件は、transform_MERGE_to_join（本ファイル外）で最終的に構築されるため、ここでは jointree の骨組み作成に留めています。
