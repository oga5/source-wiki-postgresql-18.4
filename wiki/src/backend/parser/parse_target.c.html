src/backend/parser/parse_target.c

概要

SQLクエリのターゲットリスト（SELECT句のリストやINSERT/UPDATEの対象列）を解析・変換するためのモジュールです。構文解析後の抽象構文木（AST）を、プランナーが処理可能な形式（TargetEntryノード）へ変換する役割を担います。

モジュール内での役割

src/backend/parser サブシステムの一部として、クエリ解析の最終段階に近い「ターゲットリストの構築」を担当します。parse_expr.c で個々の式を変換した後、それらをリストとしてまとめ、列名の推論や型変換、アクセス権限のマーク付けを行います。

責務

  • **ターゲットリストの変換**: ResTarget（構文解析結果）を TargetEntry（意味解析結果）へ変換。
  • **ワイルドカード展開**: SELECT * や table.* のような展開を解決し、対象となる全列のリストを生成。
  • **代入処理のサポート**: INSERT や UPDATE における列への代入（サブフィールドや配列添字を含む）の型チェックと coercion（型強制）の実行。
  • **列名の推論**: AS 句がない場合に、式の内容から適切な列名を自動生成。

重要なシンボル

  • transformTargetList: ターゲットリスト全体を変換するメインのエントリポイント。
  • transformTargetEntry: 個別の式を TargetEntry に変換する。
  • transformAssignedExpr: 代入対象の式を型チェックし、必要に応じて FieldStore や SubscriptingRef を構築。
  • FigureColname: 式からデフォルトの列名を推論する。
  • ExpandColumnRefStar: * 展開のロジックを管理。

制御フローとデータフロー

1. **変換開始**: transformTargetList が呼び出され、リスト内の各要素を走査。 2. **展開**: * が含まれる場合、ExpandColumnRefStar 等を呼び出し、対象となる列のリストを動的に生成・結合。 3. **式変換**: transformTargetEntry を通じて各式を transformExpr で変換し、TargetEntry を生成。 4. **代入処理**: UPDATE 等では transformAssignedExpr が呼び出され、ターゲット列の型情報に基づき、代入値の型変換や添字操作（transformAssignmentIndirection）を再帰的に適用。 5. **後処理**: resolveTargetListUnknowns で型不明な列を TEXT に固定し、markTargetListOrigins で列の由来（テーブル/列番号）を記録。

読者向けメモ

  • 本ファイルは「代入」の複雑なロジック（特に FieldStore や SubscriptingRef を伴う再帰的な処理）を含んでいます。これらは INSERT や UPDATE の際に、複合型や配列の特定要素のみを更新するために使用されます。
  • FigureColname は非常に多くのノード型を網羅しており、新しい式ノードを追加した際はここへの追記が必要になる場合があります。
  • * 展開のロジックは、SELECT 句と ROW() などのコンストラクタで共有されており、make_target_entry フラグで出力形式を制御しています。
