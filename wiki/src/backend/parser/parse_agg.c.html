src/backend/parser/parse_agg.c

概要

本ファイルは、SQL 解析プロセスにおける「集約関数（Aggregate）」および「ウィンドウ関数（Window Function）」のセマンティック解析を担います。構文解析後のノードに対し、型解決、配置制限の検証、および GROUP BY 句との整合性チェックを行います。

モジュール内での役割

src/backend/parser モジュールの一部として、parse_expr.c や parse_clause.c から呼び出され、集約特有の複雑な制約（ネストの禁止、GROUP BY への依存関係）を解決します。

責務

  • **集約・ウィンドウ関数の変換**: Aggref や WindowFunc ノードの初期化と、引数のターゲットリストへの変換。
  • **配置制限の検証**: WHERE 句や JOIN 条件など、集約関数が許可されない場所での使用をエラーとして検出。
  • **グループ化の整合性チェック**: GROUP BY 句に含まれない列が SELECT リスト等で使用されている場合の検証と、RTE_GROUP への置換。
  • **集約サポート関数の構築**: プランナが使用する遷移関数（transfn）、シリアライズ関数等の式ツリー生成。

重要なシンボル

集約・ウィンドウ関数変換

  • transformAggregateCall: Aggref を完成させ、引数を直接引数と集約引数に分離し、ORDER BY/DISTINCT を処理する。
  • transformWindowFuncCall: WindowFunc を完成させ、WindowDef を ParseState に登録する。

検証と整合性チェック

  • parseCheckAggregates: クエリ全体の集約とグループ化の整合性を最終確認し、グループ化された列を RTE_GROUP への参照に置換する。
  • check_agglevels_and_constraints: 集約関数がクエリのどのレベルに属するかを判定し、配置場所が妥当かチェックする。
  • check_agg_arguments: 集約関数の引数を走査し、ネストされた集約や不正な変数を検出する。

内部ユーティリティ

  • substitute_grouped_columns: GROUP BY された列を RTE_GROUP 参照に置換する再帰的処理の入り口。
  • expand_grouping_sets: GROUPING SETS 句をフラットなリストに展開する。
  • build_aggregate_transfn_expr: 遷移関数の実行に必要なダミー式ツリーを構築する。

制御フローとデータフロー

1. **変換フェーズ**: transformAggregateCall が Aggref を受け取り、TargetEntry を生成して agg->args に格納。ORDER BY や DISTINCT があれば SortGroupClause を生成。 2. **検証フェーズ**: check_agglevels_and_constraints が ParseExprKind を参照し、現在のコンテキスト（WHERE, JOIN 等）で集約が許可されているか判定。 3. **グループ化置換**: parseCheckAggregates が GROUP BY 句を解析し、RTE_GROUP を作成。substitute_grouped_columns_mutator が式ツリーを走査し、グループ化された列を buildGroupedVar で生成した Var ノード（RTE_GROUP を指す）に置換する。

読者向けメモ

  • **メモリ管理**: copyObject や makeNode を多用し、解析ツリーのノードを生成・複製します。ParseState は解析セッションを通じて生存します。
  • **制約**: 集約関数のネストは SQL 標準および PostgreSQL の実装上禁止されています。check_agg_arguments がこの再帰的制約を管理します。
  • **省略された詳細**: expand_grouping_sets の複雑なビット演算や、cmp_list_len_contents_asc などのソートロジックは、プランナ向けの補助的な実装であり、パーサのメインフローからは独立しています。
  • **注意点**: substitute_grouped_columns は flatten_join_alias_vars を呼び出すことで、JOIN されたテーブルの別名解決を正規化してから比較を行います。この順序は重要です。
