src/backend/parser/parse_node.c

概要

parse_node.c は、PostgreSQL のパーサーがクエリツリーを構築する際に使用する、基本的なノード生成および管理ルーチンを提供します。主に ParseState のライフサイクル管理、エラー位置の報告、および定数や添字（subscripting）ノードの変換を担います。

モジュール内での役割

src/backend/parser サブシステムにおいて、このファイルは「パーサーの基盤」として機能します。他の parse_*.c ファイルが特定の SQL 構文（式、句、関数など）を解析する際、共通のコンテキストである ParseState を介して情報を共有し、エラー報告を行うためのユーティリティを提供します。

責務

  • **ParseState の管理**: 解析セッション全体で共有される状態（ParseState）の生成と破棄。
  • **エラー位置の追跡**: SQL 解析中に発生したエラーに対し、ソーステキスト上の正確な位置を報告するためのコールバック機構の提供。
  • **ノード変換**: 構文解析器から渡された未加工のノード（A_Const など）を、実行可能なクエリツリー用のノード（Const など）へ変換。
  • **コンテナ操作の抽象化**: 配列やその他の添字可能な型に対する添字操作の変換。

重要なシンボル

  • ParseState: 解析中のクエリの状態を保持する構造体。
  • make_parsestate / free_parsestate: ParseState のメモリ管理を行う関数。
  • parser_errposition: エラー発生時にソーステキスト内の位置を特定し、ユーザーに通知する関数。
  • transformContainerSubscripts: 配列等の添字操作を解析し、SubscriptingRef ノードを生成する関数。
  • make_const: 構文解析器からの定数値を、型情報を持つ Const ノードへ変換する関数。

制御フローとデータフロー

1. **初期化**: make_parsestate が呼び出され、解析コンテキストが作成されます。 2. **解析中**: 構文解析器が transformContainerSubscripts や make_const を呼び出し、未加工の構文要素を意味のあるノードへ変換します。 3. **エラー処理**: 外部関数でエラーが発生した場合、setup_parser_errposition_callback で登録されたコールバックが pcb_error_callback を経由して parser_errposition を呼び出し、エラー位置を特定します。 4. **終了**: 解析完了後、free_parsestate がリソースを解放し、ターゲットリレーションのロックを解除します。

読者向けメモ

  • ParseState は再帰的なクエリ（サブクエリなど）を扱うために parentParseState を持つことができます。
  • parser_errposition は、バイトオフセットを文字インデックスに変換するため、マルチバイト文字を考慮した pg_mbstrlen_with_len を使用しています。
  • make_const は、型が明示されていない定数（UNKNOWN 型）を扱う際の重要な入り口です。将来的な型解決のために、まずは UNKNOWN としてマークされます。
