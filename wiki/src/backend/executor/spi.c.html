src/backend/executor/spi.c

概要

spi.c は、PostgreSQL の **Server Programming Interface (SPI)** を実装するモジュールです。SPI は、サーバー内部のコード（PL/pgSQL などの手続き型言語やトリガー関数）から SQL クエリを安全かつ透過的に実行するためのインターフェースを提供します。クエリの解析、計画、実行、およびトランザクション制御をカプセル化し、呼び出し元のコンテキストと実行エンジンを分離します。

モジュール内での役割

src/backend/executor サブシステムの一部として、実行エンジン（execMain.c 等）をラップし、手続き型言語が SQL を実行するための「ゲートウェイ」として機能します。

責務

  • **接続管理**: SPI 呼び出しのネストレベルを管理し、呼び出し元ごとのメモリコンテキストとグローバル状態（SPI_processed 等）を保護する。
  • **クエリ実行**: SQL 文字列の解析・計画・実行、およびプリペアドプランの実行。
  • **データ変換**: 内部的なタプル表現と、呼び出し元が扱うデータ形式間の変換。
  • **トランザクション制御**: 手続き内からのトランザクション終了（コミット/ロールバック）の制御。
  • **リソース管理**: SPI 経由で生成されたタプルテーブルやメモリコンテキストの自動クリーンアップ。

重要なシンボル

状態管理（グローバル変数）

  • SPI_processed: 直近の操作で処理された行数。
  • SPI_tuptable: 直近の操作で返されたタプルのテーブル。
  • SPI_result: 直近の操作のステータスコード。

接続・ライフサイクル

  • SPI_connect() / SPI_finish(): SPI セッションの開始と終了。
  • AtEOXact_SPI() / AtEOSubXact_SPI(): トランザクション/サブトランザクション終了時の SPI 状態のクリーンアップ。

クエリ実行

  • SPI_execute() / SPI_execute_extended(): SQL 文字列の即時実行。
  • SPI_execute_plan() / SPI_execute_plan_extended(): プリペアドプランの実行。
  • SPI_prepare() / SPI_prepare_extended(): クエリの解析と計画の事前作成。

カーソル操作

  • SPI_cursor_open() / SPI_cursor_fetch(): カーソルの作成とフェッチ。

データ操作・ユーティリティ

  • SPI_copytuple() / SPI_modifytuple(): タプルのコピーと変更。
  • SPI_getvalue() / SPI_getbinval(): タプルからの値抽出。
  • SPI_register_relation(): 一時的な名前付きリレーションの登録。

制御フローとデータフロー

1. **接続**: SPI_connect が _SPI_stack を拡張し、新しい _SPI_connection を作成。procCxt（プロシージャ用）と execCxt（実行用）メモリコンテキストを初期化。 2. **実行**: SPI_execute 等が _SPI_prepare_oneshot_plan で解析を行い、_SPI_execute_plan を呼び出す。 3. **実行フロー**: _SPI_execute_plan は GetCachedPlan でプランを取得し、ProcessUtility または _SPI_pquery（ExecutorStart/Run/Finish を呼び出す）を通じてクエリを処理。 4. **結果返却**: spi_dest_startup と spi_printtup が DestReceiver として機能し、実行結果を SPITupleTable に蓄積。 5. **終了**: SPI_finish がメモリコンテキストを解放し、スタックをポップする。

読者向けメモ

  • **メモリ所有権**: SPI は procCxt（プロシージャ終了まで生存）と execCxt（個々のクエリ実行中のみ生存）の二層構造でメモリを管理します。SPI_palloc は procCxt を使用します。
  • **グローバル変数**: SPI_processed 等のグローバル変数は、SPI のネスト呼び出し時に _SPI_connection 構造体に保存・復元されます。
  • **制約**: SPI_commit や SPI_rollback は、アトミックな（関数内の）SPI コンテキストでは呼び出せません。
  • **省略箇所**: 本ファイルは非常に大規模であるため、各 SPI_execute_* 系関数の詳細な引数バリデーションロジックや、spi_printtup の詳細なタプルコピー処理は省略しています。これらは主に execTuples.c の機能に依存しています。
