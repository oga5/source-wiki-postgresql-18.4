src/backend/executor/nodeModifyTable.c

概要

nodeModifyTable.c は、PostgreSQL の実行エンジンにおいて INSERT、UPDATE、DELETE、および MERGE 文を実行するための中心的なプランノードです。プランナが生成した ModifyTable ノードを解釈し、ターゲットリレーション（テーブル、パーティション、外部テーブル、ビュー）に対してデータ操作を適用します。

モジュール内での役割

src/backend/executor モジュール内の「ノード層」に位置し、ExecProcNode から呼び出される実行ノードの一つです。他のノードが主にデータの読み取りを行うのに対し、本ノードはデータ変更という副作用を伴う処理を担います。

責務

  • **DML 操作の実行**: ターゲットリレーションに対する行の挿入、更新、削除の実行。
  • **トリガー管理**: BEFORE/AFTER 行レベルトリガーおよびステートメントレベルトリガーの起動。
  • **制約とポリシーの適用**: CHECK 制約、NOT NULL 制約、RLS (Row Level Security) ポリシー、および WITH CHECK OPTION の検証。
  • **パーティションルーティング**: パーティションテーブルへの挿入時に、適切なリーフパーティションを特定し、必要に応じてタプルを変換してルーティングする。
  • **競合解決**: INSERT ... ON CONFLICT における競合検知と、それに応じた UPDATE または DO NOTHING の実行。
  • **MERGE 処理**: ソースとターゲットの結合結果に基づき、WHEN MATCHED / NOT MATCHED 条件に応じたアクションのディスパッチ。
  • **RETURNING 句の処理**: 操作後のタプルを射影し、呼び出し元へ返却。

重要なシンボル

実行コンテキスト

  • ModifyTableContext: 操作中の状態（EState、ModifyTableState、EPQState）や、操作結果を保持する一時的なスロットをまとめた構造体。
  • UpdateContext: UPDATE 操作特有の状態（パーティション移動の有無、インデックス更新フラグ、ロックモード）を保持。

主要な実行関数

  • ExecModifyTable: ノードのメインループ。サブプランからタプルを取得し、操作タイプに応じて ExecInsert / ExecUpdate / ExecDelete / ExecMerge を呼び出す。
  • ExecInsert / ExecUpdate / ExecDelete: 各操作の核心。トリガー、制約、FDW 呼び出し、インデックス更新を統合する。
  • ExecMerge: MERGE 文のロジック。ExecMergeMatched と ExecMergeNotMatched を用いて条件分岐を行う。

初期化・ユーティリティ

  • ExecInitModifyTable: ノードの初期化。ターゲットリレーションの準備、FDW の初期化、トリガー状態のセットアップを行う。
  • ExecInitGenerated / ExecComputeStoredGenerated: 生成列（Generated Columns）の計算と初期化。
  • ExecInitInsertProjection / ExecInitUpdateProjection: 射影（Projection）の準備。

制御フローとデータフロー

1. **初期化**: ExecInitModifyTable が ResultRelInfo を構築し、ターゲットリレーションをオープン。 2. **ループ処理**: ExecModifyTable がサブプランからタプルをプルアップ。 3. **ルーティング**: パーティションテーブルの場合、ExecPrepareTupleRouting で適切なリーフパーティションを特定。 4. **操作実行**:

  • ExecInsert 等がトリガーを起動し、table_tuple_insert 等の Table AM を呼び出す。
  • INSERT ... ON CONFLICT の場合、競合検知後に ExecOnConflictUpdate を呼び出し、必要に応じて UPDATE パスへ遷移。
  • UPDATE でパーティションキーが変更された場合、ExecCrossPartitionUpdate が DELETE と INSERT を組み合わせてタプルを移動。

    • 5. **RETURNING**: ExecProcessReturning が射影を行い、結果を呼び出し元へ返す。

読者向けメモ

  • **メモリ管理**: ModifyTable はクエリ実行期間中生存する EState を多用します。ExecInit 時に割り当てられたメモリは ExecutorEnd で解放されます。
  • **副作用**: 本ノードはトランザクションの状態を直接変更します。ExecReScanModifyTable は未実装であり、再実行は想定されていません。
  • **複雑性**: MERGE 処理は UPDATE や DELETE と複雑に絡み合っており、特に同時実行制御（EvalPlanQual）のパスが非常に深いため、デバッグ時は TM_Result の戻り値（TM_Updated 等）の追跡が重要です。
  • **省略**: 本ファイルには ExecBatchInsert のような FDW 向けの最適化パスが含まれていますが、これらは特定の FDW 実装に依存するため、詳細な挙動は各 FDW の実装に委ねられています。
