src/backend/executor/nodeWindowAgg.c

概要

nodeWindowAgg.c は、SQL のウィンドウ関数（WindowFunc）およびウィンドウ集約（WindowAgg）を実行するための実行ノードを実装します。入力タプルは PARTITION BY および ORDER BY キーでソートされていることを前提とし、パーティション単位でデータを tuplestore に蓄積してウィンドウフレーム内の演算を行います。

モジュール内での役割

src/backend/executor サブシステムの一部として、プランナが生成した WindowAgg ノードを実行します。他のノード（nodeAgg.c など）と異なり、ウィンドウ関数はフレーム内の任意の行へのランダムアクセスを必要とするため、tuplestore を用いた一時的なデータ保持と、WindowObject API を介した関数へのアクセス提供が中心的な役割です。

責務

  • **パーティション管理**: 入力ストリームからパーティションを検出し、tuplestore へのスプールとメモリコンテキストの切り替えを行う。
  • **ウィンドウフレーム計算**: RANGE / ROWS / GROUPS モードに応じたフレーム境界（head/tail）の動的計算。
  • **集約演算**: 通常の集約関数をウィンドウ関数として実行するための状態管理（WindowStatePerAgg）。
  • **ウィンドウ関数 API**: WindowObject を通じて、関数がパーティション内の任意行にアクセスするためのインターフェースを提供。
  • **最適化**: 連続する行でフレームが変化しない場合の計算スキップや、逆遷移関数（invtransfn）を用いた増分更新。

重要なシンボル

状態管理構造体

  • WindowAggState: ノード全体の実行状態を保持するルート構造体。
  • WindowObjectData: ウィンドウ関数がパーティション内を探索するための API オブジェクト。
  • WindowStatePerFuncData: 各ウィンドウ関数の実行状態（引数、関数ポインタ、結果型情報）。
  • WindowStatePerAggData: 集約関数としての実行状態（遷移関数、現在の集約値、メモリコンテキスト）。

実行制御関数

  • ExecWindowAgg: ノードのメイン実行ループ。タプルのスプール、関数評価、射影を行う。
  • ExecInitWindowAgg / ExecEndWindowAgg: ノードのライフサイクル管理。
  • begin_partition / release_partition: パーティション境界でのリソース初期化・解放。

集約計算関数

  • eval_windowaggregates: 集約関数をウィンドウ関数として評価する。
  • advance_windowaggregate / advance_windowaggregate_base: 遷移関数および逆遷移関数を用いた集約状態の更新。
  • finalize_windowaggregate: 最終関数（finalfn）を適用して結果を算出。

制御フローとデータフロー

1. **初期化**: ExecInitWindowAgg が tuplestore や式コンテキストを準備。 2. **スプール**: spool_tuples が入力プランからタプルを読み込み、tuplestore に格納。 3. **フレーム更新**: update_frameheadpos / update_frametailpos が現在の行に対するフレーム境界を計算。 4. **関数評価**:

  • 通常のウィンドウ関数: eval_windowfunction が WindowObject を介してデータを取得。
  • 集約関数: eval_windowaggregates が遷移関数を呼び出し、必要に応じて advance_windowaggregate_base で古い行を削除。

    • 5. **射影**: ExecProject で最終的な出力タプルを生成。

読者向けメモ

  • **メモリ所有権**: WindowAgg は複数のメモリコンテキスト（partcontext, aggcontext）を使い分けます。特に aggcontext は集約の再計算時にリセットされるため、ポインタの生存期間に注意が必要です。
  • **パフォーマンス**: tuplestore がディスクに溢れると性能が著しく低下します。WinSetMarkPosition を適切に呼び出すことで、不要な行を早期に破棄させることが重要です。
  • **制約**: invtransfn（逆遷移関数）が提供されていない場合、フレームの移動時に集約を全行再計算するため、計算量が $O(N^2)$ になる可能性があります。
  • **省略箇所**: 本ファイル内の tuplestore の詳細な内部実装や、fmgr による関数呼び出しの低レイヤーな詳細は省略しています。これらは src/backend/utils/sort/tuplestore.c および src/backend/utils/fmgr/fmgr.c を参照してください。
