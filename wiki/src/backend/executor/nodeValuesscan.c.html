src/backend/executor/nodeValuesscan.c

概要

nodeValuesscan.c は、SQLの VALUES 句（例: VALUES (1, 'a'), (2, 'b')）によって生成された定数リストをスキャンするための実行エンジンノードを実装します。このノードは、メモリ上の式リストを順次評価し、仮想タプルとして提供します。

モジュール内での役割

src/backend/executor におけるスキャンノードの一種です。テーブルやインデックスを物理的に読み取る他のスキャンノードとは異なり、クエリ実行時にメモリ上に展開された定数式を評価してタプルを生成する役割を担います。

責務

  • **タプル生成**: VALUES 句内の各行を評価し、TupleTableSlot に格納する。
  • **式評価の最適化**: 各行の式を必要に応じて初期化し、メモリ消費を抑えるために評価後の状態を管理する。
  • **スキャン制御**: 前方・後方スキャンをサポートし、再スキャン（ReScan）時のインデックスリセットを行う。

重要なシンボル

  • ExecInitValuesScan: ノードの初期化。式リストの準備とメモリコンテキストの割り当てを行う。
  • ValuesNext: スキャンの中心となる関数。現在のインデックスに基づき、次の行の式を評価して仮想タプルを作成する。
  • ExecValuesScan: ExecScan を呼び出し、スキャン処理を抽象化するインターフェース。
  • ExecReScanValuesScan: スキャン位置を先頭に戻し、状態をリセットする。

追加の索引済みシンボル

  • ValuesRecheck

制御フローとデータフロー

1. **初期化**: ExecInitValuesScan が VALUES 句のリストを配列に変換し、式評価の準備を行う。 2. **実行**: ExecValuesScan が ValuesNext を呼び出す。 3. **評価**: ValuesNext は curr_idx を更新し、該当する行の式を ExecEvalExpr で評価する。 4. **メモリ管理**: 評価結果は MakeExpandedObjectReadOnly を通じて読み取り専用化され、ExecStoreVirtualTuple でスロットに格納される。 5. **終了**: 全ての行を処理すると NULL を返し、スキャンを終了する。

読者向けメモ

  • **メモリ効率**: VALUES 句が非常に長い場合、全ての行の式状態を一度に初期化するとメモリを圧迫するため、必要に応じて遅延初期化を行っています。
  • **JITの無効化**: VALUES 句の式は通常一度しか評価されないため、オーバーヘッドを避けるためにJITコンパイルを意図的に無効化するロジックが含まれています。
  • **仮想タプル**: このノードは物理的なタプルを持たず、評価結果をその場で構築する「仮想タプル」を使用している点に注意してください。
