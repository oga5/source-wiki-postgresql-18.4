src/backend/executor/nodeTableFuncscan.c

概要

nodeTableFuncscan.c は、SQLの XMLTABLE や JSON_TABLE といった「テーブル関数（Table Functions）」を実行するためのスキャンノードを実装しています。これらは入力データ（XMLやJSON）を解析し、仮想的なリレーショナルテーブルとして出力する機能です。

モジュール内での役割

src/backend/executor におけるスキャンノードの一種です。他のスキャンノード（nodeSeqscan.c 等）と同様に、ExecScan インターフェースを介してプラン実行エンジンに統合されています。

責務

  • **初期化**: TableFuncScanState の構築、式評価コンテキストの準備、および入力データ型に応じた FmgrInfo の設定。
  • **データ取得**: 入力ドキュメントを解析し、行を抽出して tuplestore に格納する。
  • **行の生成**: 抽出されたデータに対して列フィルタやデフォルト値の適用を行い、最終的なタプルを生成する。
  • **リソース管理**: 実行中に使用するメモリコンテキスト（perTableCxt）の管理と、tuplestore のクリーンアップ。

重要なシンボル

  • ExecInitTableFuncScan: ノードの初期化と式ツリーの準備を行う。
  • TableFuncNext: ExecScan から呼び出され、tuplestore から次のタプルを取得する。
  • tfuncFetchRows: 最初の呼び出し時にドキュメントを解析し、全行を tuplestore にロードする。
  • tfuncLoadRows: 行ごとの値抽出、デフォルト値の適用、NOT NULL 制約の検証を行う。

制御フローとデータフロー

1. **初期化**: ExecInitTableFuncScan がプランノードを解析し、XML/JSON 用のルーチン（XmlTableRoutine 等）をセットアップする。 2. **初回実行**: TableFuncNext が tupstore が空であることを検知し、tfuncFetchRows を呼び出す。 3. **解析**: tfuncFetchRows が tfuncInitialize を通じて名前空間やフィルタを設定し、tfuncLoadRows で行を反復処理する。 4. **タプル生成**: tfuncLoadRows が routine->GetValue で各列の値を取得し、tuplestore_putvalues で格納する。 5. **スキャン**: 以降の呼び出しでは、tuplestore_gettupleslot を使用してメモリ上の結果を順次返す。

読者向けメモ

  • 本ノードは、一度の実行で全行を tuplestore にロードする「マテリアライズ型」のスキャンです。そのため、入力ドキュメントが巨大な場合はメモリ消費に注意が必要です。
  • opaque ポインタは、XML や JSON の解析エンジンが実行状態を保持するために使用する動的な領域です。
  • perTableCxt は、1つのテーブル関数呼び出しのライフサイクルに紐づくメモリ管理用であり、クエリ全体よりも短く、タプル単位よりも長い生存期間を持ちます。
