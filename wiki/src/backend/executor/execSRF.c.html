src/backend/executor/execSRF.c

概要

execSRF.c は、PostgreSQL における集合を返す関数（SRF: Set-Returning Functions）の実行を管理する API を提供します。主に nodeFunctionscan.c および nodeProjectSet.c から利用され、ReturnSetInfo プロトコルに基づいた関数の呼び出しと結果の処理を抽象化します。

モジュール内での役割

実行エンジン（src/backend/executor）において、SRF の実行状態を保持する SetExprState の初期化と、関数呼び出しの反復処理を担います。特に、関数が一度に全結果を返す（Materialize）のか、呼び出しごとに値を返す（ValuePerCall）のかという差異を吸収し、呼び出し元に統一されたインターフェースを提供します。

責務

  • **SRF 実行状態の管理**: SetExprState の初期化および、関数呼び出しに必要なメタデータ（引数、戻り値型、TupleDesc）の準備。
  • **プロトコル制御**: ReturnSetInfo を介した SRF プロトコルの遵守確認と、エラーハンドリング。
  • **結果の具現化**: 関数が返す結果を Tuplestore に格納し、呼び出し元が反復的に取得できるようにする。
  • **メモリ管理**: SRF 実行に伴うメモリリークを防ぐためのコンテキスト切り替えと、クリーンアップコールバックの登録。

重要なシンボル

  • SetExprState: SRF の実行状態を保持する構造体。
  • ExecInitTableFunctionResult: FROM 句で呼び出されるテーブル関数の初期化用。
  • ExecMakeTableFunctionResult: テーブル関数を実行し、結果を Tuplestore にマテリアライズする。
  • ExecMakeFunctionResultSet: プロジェクション（SELECT リスト等）内の SRF を実行し、値を逐次取得する。
  • ReturnSetInfo: SRF と実行エンジン間で通信するための構造体。

制御フローとデータフロー

1. **初期化**: ExecInit... 関数が呼び出され、SetExprState が作成されます。ここで fmgr_info を用いて関数情報をキャッシュします。 2. **実行**: ExecMake... 関数が呼び出されると、ExprContext を用いて引数を評価し、FunctionCallInvoke で関数を呼び出します。 3. **プロトコル処理**:

  • ValuePerCall モードの場合、関数が ExprMultipleResult を返す限りループし、結果を Tuplestore に蓄積します。
  • Materialize モードの場合、関数が一度に返した Tuplestore をそのまま利用します。

    • 4. **終了処理**: 実行終了時やエラー発生時には、登録された ShutdownSetExpr コールバックが呼び出され、Tuplestore の解放やリソースのクリーンアップが行われます。

読者向けメモ

  • SRF の実行はメモリコンテキストの切り替えが頻繁に行われます。特に argContext（引数用）と econtext->ecxt_per_tuple_memory（関数呼び出し用）の使い分けに注意してください。
  • tupledesc_match は、関数が返す行型とクエリが期待する行型が一致するかを検証します。動的な型（RECORD 型など）を扱う際の重要なチェックポイントです。
  • ShutdownSetExpr は、クエリが途中で中断された場合でもリソースを確実に解放するために重要です。新しい状態を追加する場合は、このコールバックへの登録を忘れないようにしてください。
