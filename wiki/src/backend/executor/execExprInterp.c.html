src/backend/executor/execExprInterp.c

概要

execExprInterp.c は、PostgreSQL の式評価エンジンにおける「インタプリタ」実装です。プランナが生成した ExprState（評価ステップのリスト）を解釈し、CPU 上で直接実行します。パフォーマンスを最大化するため、コンパイラの機能（Computed Gotos）を活用したスレッド化コード（Threaded Code）技術を採用しています。

モジュール内での役割

src/backend/executor モジュールにおいて、式評価の実行フェーズを担います。execExpr.c が式ツリーを評価ステップのリストにコンパイルするのに対し、本ファイルはそのリストを高速に走査・実行します。JIT コンパイルが有効でない場合や、単純な式に対しては本ファイルが主役となります。

責務

  • **命令ディスパッチ**: ExprEvalStep に格納されたオペコードに基づき、適切な評価ロジックへジャンプする。
  • **高速パスの提供**: 頻出する単純な式（変数参照、定数など）に対して、インタプリタのオーバーヘッドを回避する ExecJust* 関数群を提供。
  • **複雑な評価の委譲**: 複雑な関数呼び出しや集約処理など、インライン化に適さない処理を専用のヘルパー関数へルーティング。
  • **実行時検証**: スキーマ変更等による実行計画の不整合を検出するための検証ロジックの実行。

重要なシンボル

ディスパッチ制御

  • EEO_USE_COMPUTED_GOTO: コンパイラがサポートする場合、goto を用いた直接スレッド化（Direct Threading）を有効にするマクロ。
  • ExecInterpExpr: インタプリタのメインループ。ExprEvalStep を順次実行する。
  • ExecReadyInterpretedExpr: 式の初期化を行い、最適化された evalfunc（高速パスまたはインタプリタ）を割り当てる。

高速パス関数

  • ExecJustInnerVar, ExecJustOuterVar, ExecJustScanVar: スロットからの変数抽出を最適化した関数。
  • ExecJustConst: 定数評価のショートカット。
  • ExecJustHash*: ハッシュ結合等で頻出するハッシュ計算の最適化版。

実行ヘルパー

  • ExecEvalArrayCompareInternal: ANY/ALL 比較の共通ロジック。
  • ExecAggPlainTransByVal / ExecAggPlainTransByRef: 集約関数の遷移関数呼び出し（値渡し/参照渡し）。
  • ExecEvalJsonExprPath: SQL/JSON パス式の評価。

制御フローとデータフロー

1. **初期化**: ExecReadyInterpretedExpr が式を解析し、単純なパターンであれば ExecJust* を state->evalfunc_private に設定。それ以外は ExecInterpExpr を設定し、EEO_USE_COMPUTED_GOTO が有効ならオペコードをジャンプ先アドレスに変換。 2. **実行**: ExecInterpExpr が呼び出されると、op ポインタが現在のステップを指し、EEO_DISPATCH() マクロで該当する EEO_CASE へジャンプ。 3. **ステップ実行**: 各 EEO_CASE は ExprEvalStep のデータ（op->d）を参照し、結果を op->resvalue / op->resnull に書き込む。 4. **遷移**: EEO_NEXT() で次のステップへ進むか、EEO_JUMP() で条件分岐やループを行う。 5. **終了**: EEOP_DONE_RETURN に到達すると評価結果を返す。

読者向けメモ

  • **パフォーマンス**: 本ファイルは PostgreSQL のホットパスです。ExecInterpExpr 内の switch 文や goto は、コンパイラの最適化を強く意識して記述されています。
  • **メモリ管理**: 遷移関数の結果などは aggcontext や per-tuple コンテキストを適切に切り替えて管理されます。ExecAggCopyTransValue 等のメモリ操作は、メモリリークや二重解放を防ぐための重要な箇所です。
  • **制約**: EEO_USE_COMPUTED_GOTO が利用できない環境（一部のコンパイラ）では、標準的な switch 文によるディスパッチにフォールバックされます。
  • **未実装/浅い部分**: 一部の複雑なノード（XMLEXPR や JSON 関連）は、コードサイズと複雑性の観点からインライン化されず、外部関数への呼び出しに留まっています。
