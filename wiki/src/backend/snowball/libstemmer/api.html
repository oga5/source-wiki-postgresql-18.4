src/backend/snowball/libstemmer/api.c

概要

Snowball ステミングアルゴリズムを実行するための実行環境（SN_env）のライフサイクル管理および、処理対象文字列のセットアップを行う API 実装です。

モジュール内での役割

libstemmer ライブラリのフロントエンドとして機能します。各言語ごとのステミングロジック（stem_*.c）が共通して利用する、メモリ管理とデータ構造の初期化・破棄の標準インターフェースを提供します。

責務

  • **環境の生成と破棄**: ステミングに必要な動的メモリ（SN_env 構造体および内部の文字列バッファ）の確保と解放。
  • **状態の初期化**: アルゴリズム実行に必要な整数配列（I）および文字列ポインタ配列（S）の割り当て。
  • **入力のセット**: ステミング対象の文字列を環境内にロードし、カーソル位置をリセットする。

重要なシンボル

  • SN_create_env: SN_env 構造体を生成し、必要なバッファを初期化します。
  • SN_close_env: 確保されたすべてのメモリを解放し、リークを防ぎます。
  • SN_set_current: ステミング対象の文字列を環境にコピーし、処理開始位置を先頭に設定します。

制御フローとデータフロー

1. **初期化**: SN_create_env が呼び出されると、calloc で環境を確保し、create_s()（utilities.c 由来）を用いて内部バッファを生成します。エラー発生時は goto error でクリーンアップを行います。 2. **実行**: SN_set_current が呼び出されると、replace_s を介して入力文字列が環境のバッファにコピーされ、カーソル c が 0 にリセットされます。 3. **終了**: SN_close_env は、配列 S 内の各要素と環境本体を順次解放します。

直接依存

  • header.h: SN_env 構造体の定義および symbol 型の定義を提供します。

関連コード

  • src/include/snowball/libstemmer/api.h: 本ファイルの関数プロトタイプと SN_env 構造体の定義が含まれています。
  • utilities.c: create_s, lose_s, replace_s など、メモリ管理や文字列操作の低レイヤー処理を担う関数群が定義されています。

読者向けメモ

  • 本ファイルはメモリ管理を直接行っているため、SN_create_env で確保した環境は必ず SN_close_env で解放する必要があります。
  • SN_env 構造体内の p はメインの文字列バッファ、S はアルゴリズム内で一時的に利用される文字列配列を指します。
  • header.h で定義される symbol 型のサイズ（char か short か）が、システム全体のメモリ配置に影響を与える点に注意してください。
