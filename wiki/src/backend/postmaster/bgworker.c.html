src/backend/postmaster/bgworker.c

概要

PostgreSQLの「バックグラウンドワーカー（BGWorker）」機能の実装ファイルです。ユーザー定義のコードを独立したプロセスとして実行するための基盤を提供します。Postmasterによるプロセス管理と、共有メモリを介した動的なワーカー登録・監視の仕組みを定義しています。

モジュール内での役割

postmaster サブシステムの一部として、Postmasterプロセスと通常のバックエンドプロセス間の橋渡しを行います。Postmasterはワーカーの起動・終了・再起動を管理し、バックエンドはワーカーの登録や状態監視を行います。

責務

  • **ワーカーのライフサイクル管理**: Postmasterによるワーカープロセスのフォーク、死活監視、終了処理。
  • **共有メモリ管理**: BackgroundWorkerArray を用いたワーカー状態の共有と、ロックレスな通信プロトコルの提供。
  • **動的登録**: バックエンドプロセスからのワーカー登録と、それに対するハンドル（BackgroundWorkerHandle）の提供。
  • **関数解決**: ライブラリ名と関数名から実行アドレスを解決するメカニズムの提供。

重要なシンボル

共有メモリ管理

  • BackgroundWorkerArray: 全ワーカーの状態を保持する共有メモリ上の配列。
  • BackgroundWorkerSlot: 個々のワーカーの状態（PID、終了フラグ、登録情報）を保持する構造体。
  • BackgroundWorkerLock: 共有メモリ上のスロットを操作する際に使用する LWLock。

プロセス管理（Postmaster側）

  • BackgroundWorkerList: Postmasterが保持する登録済みワーカーのプライベートリスト。
  • BackgroundWorkerStateChange(): 共有メモリの変更を検知し、Postmasterの内部状態を更新する。
  • ReportBackgroundWorkerPID() / ReportBackgroundWorkerExit(): ワーカーのPID変化を共有メモリに反映し、通知を行う。

ユーザー・バックエンドAPI

  • RegisterDynamicBackgroundWorker(): 実行中のバックエンドからワーカーを動的に登録する。
  • GetBackgroundWorkerPid(): ワーカーの現在のPIDと状態を取得する。
  • WaitForBackgroundWorkerStartup() / WaitForBackgroundWorkerShutdown(): ワーカーの起動・終了を待機する。
  • TerminateBackgroundWorker(): ワーカーに終了を指示する。

制御フローとデータフロー

1. **登録**: バックエンドが RegisterDynamicBackgroundWorker を呼び出し、BackgroundWorkerLock を取得して空きスロットを確保。in_use フラグを立てて PMSIGNAL_BACKGROUND_WORKER_CHANGE シグナルをPostmasterに送る。 2. **起動**: Postmasterが BackgroundWorkerStateChange を実行。in_use が真のスロットを走査し、新規ワーカーであれば fork() を実行。 3. **実行**: 子プロセスは BackgroundWorkerMain を呼び出し、LookupBackgroundWorkerFunction でユーザー関数をロードして実行する。 4. **終了**: ワーカー終了時、Postmasterが ReportBackgroundWorkerExit を呼び出し、必要に応じて再起動またはスロットの解放（in_use = false）を行う。

読者向けメモ

  • **ロックレスプロトコル**: Postmasterはロックを取得できないため、共有メモリ上の in_use フラグとメモリバリア（pg_read_barrier / pg_write_barrier）を用いたロックレスな協調動作が設計されています。
  • **メモリ所有権**: RegisteredBgWorker はPostmasterのプライベートメモリ（PostmasterContext）で管理されます。
  • **制約**: max_parallel_workers の制限は、共有メモリ上のカウンタ（parallel_register_count 等）で管理されます。
  • **省略された詳細**: 本ファイルはプロセス管理のロジックに特化しており、ユーザーコードが実行する具体的なトランザクション処理や、InitPostgres の詳細な内部挙動については他モジュールに委譲されています。
