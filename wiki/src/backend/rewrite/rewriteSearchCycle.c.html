src/backend/rewrite/rewriteSearchCycle.c

概要

本ファイルは、再帰的共通テーブル式（CTE）における SEARCH および CYCLE 句の書き換え処理を実装しています。これらの句は SQL 標準で定義された再帰クエリの探索順序制御や循環検出をサポートするためのもので、本モジュールはこれらを標準的な UNION クエリの構造へと変換します。

モジュール内での役割

src/backend/rewrite サブシステムの一部として、パーサが生成した CommonTableExpr ノードを、実行可能なクエリツリーへと変換する前処理を担います。rewriteHandler.c 等が呼び出す書き換えパイプラインの補助的な役割を果たします。

責務

  • SEARCH 句（BREADTH FIRST / DEPTH FIRST）の変換：探索順序を維持するためのシーケンス列の追加と、再帰ステップごとの深さまたはパスの記録。
  • CYCLE 句の変換：循環検出のためのマーク列とパス列の追加、および循環発生時の停止条件（WHERE 句）の挿入。
  • CommonTableExpr のメタデータ（列名、型、型修飾子）の更新。

重要なシンボル

  • rewriteSearchAndCycle(CommonTableExpr *cte): 本ファイルのメインエントリポイント。CTE を受け取り、書き換え済みの新しい CommonTableExpr を返します。
  • make_path_rowexpr: 指定された列から RowExpr を構築し、パス記録用の基盤を作成します。
  • make_path_cat_expr: 再帰ステップごとにパス配列を連結する ARRAY_CAT 関数呼び出しを生成します。

制御フローとデータフロー

1. **初期化**: 入力された CommonTableExpr をコピーし、UNION 構造の左右のサブクエリを特定します。 2. **列の拡張**: SEARCH や CYCLE で指定された列を CTE の出力列リストに追加します。 3. **サブクエリの再構築**:

  • 左側（初期クエリ）: 探索用の初期値（深さ0や空の配列）をターゲットリストに追加します。
  • 右側（再帰クエリ）: 再帰参照箇所を特定し、パスの更新（ARRAY_CAT）や循環マークの計算（CASE 文）をターゲットリストに追加します。

    • 4. **メタデータ更新**: SetOperationStmt および CommonTableExpr 自体の列定義を更新し、後続の解析フェーズで整合性が取れるようにします。

読者向けメモ

  • 本実装は、SQL 標準が求める複雑な構造を PostgreSQL の内部表現（RowExpr, ArrayExpr）で擬似的に実現しています。
  • sqc.depth のような標準 SQL の記述は、内部的には FieldSelect や FuncExpr を用いて明示的な関数呼び出しに変換されています。
  • 再帰参照が UNION の右側サブクエリのトップレベルに存在しない場合、現在の実装ではエラーとなります。これは将来的な拡張ポイントです。
