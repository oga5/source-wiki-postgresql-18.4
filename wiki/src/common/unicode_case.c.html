src/common/unicode_case.c

概要

Unicode 文字の大文字・小文字変換、タイトルケース変換、およびケースフォールディング（正規化のための比較用変換）を行うための実装ファイルです。PostgreSQL のフロントエンドおよびバックエンドの両方から利用可能です。

モジュール内での役割

src/common モジュールにおいて、Unicode 規格に準拠した文字列操作機能を提供します。unicode_norm.c や unicode_category.c と並び、多言語対応（i18n）の基盤となるユーティリティ群の一部です。

責務

  • 単一の pg_wchar に対する単純なケース変換。
  • UTF-8 文字列全体に対するケース変換（unicode_strlower 等）。
  • Unicode 規格で定義される「特殊ケースマッピング（Special Casing）」の適用（例：ギリシャ語の最終シグマ処理）。
  • タイトルケース変換時の単語境界判定のサポート。

重要なシンボル

  • unicode_strlower, unicode_strupper, unicode_strtitle, unicode_strfold: 文字列変換の公開 API。
  • convert_case: 文字列変換のコアロジックを担う内部関数。
  • casemap: 文字単位の変換ルール（単純変換または特殊変換）を決定する関数。
  • CaseMapResult: 変換結果の状態（自己、単純、特殊）を示す列挙型。

制御フローとデータフロー

1. **API 呼び出し**: unicode_strlower 等の関数が convert_case を呼び出します。 2. **走査**: convert_case が入力文字列を UTF-8 として走査し、文字ごとに casemap を呼び出します。 3. **マッピング判定**:

  • casemap は casekind_map を参照し、単純な変換テーブルを引きます。
  • full フラグが有効な場合、check_special_conditions を通じて文脈依存の特殊マッピング（special_case テーブル）を適用します。

    • 4. **出力**: 変換された文字を UTF-8 にエンコードし、出力バッファに書き込みます。

読者向けメモ

  • **パフォーマンス**: 0x80 未満の ASCII 文字に対しては、テーブル参照による高速パスが用意されています。
  • **特殊ケース**: ギリシャ語の最終シグマ（Final Sigma）など、文脈に依存する変換は check_final_sigma で前後関係を走査して判定されます。
  • **バッファ管理**: dstsize が 0 の場合、必要なバッファサイズを計算するために利用可能です。
  • **タイトルケース**: 単語境界の判定はコールバック関数 wbnext に依存しており、呼び出し側が境界ロジックを制御する設計になっています。
