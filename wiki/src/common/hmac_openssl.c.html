src/common/hmac_openssl.c

概要

OpenSSL ライブラリを利用して HMAC（Hash-based Message Authentication Code）計算を行うための実装です。PostgreSQL のバックエンドおよびフロントエンドの両方で利用可能な共通インターフェースを提供します。

モジュール内での役割

src/common/hmac.c が提供する抽象化レイヤーの OpenSSL 具体実装です。ビルド時に OpenSSL が有効な場合にのみコンパイルされ、暗号学的ハッシュ計算のバックエンドとして機能します。

責務

  • **コンテキスト管理**: HMAC_CTX の生成、初期化、および解放。
  • **リソース保護**: バックエンド環境では ResourceOwner を使用し、エラー発生時やトランザクション終了時のメモリリークを防止。
  • **エラーハンドリング**: OpenSSL のエラーキューを監視し、PostgreSQL 形式のエラーメッセージへ変換。

重要なシンボル

  • pg_hmac_ctx: HMAC 計算の状態を保持する構造体。OpenSSL の HMAC_CTX をラップします。
  • pg_hmac_create/free: コンテキストのライフサイクルを管理する関数。
  • pg_hmac_init/update/final: HMAC 計算の標準的な 3 ステップを実行する関数。
  • USE_RESOWNER_FOR_HMAC: バックエンド環境でリソース所有者管理を有効にするためのマクロ。

制御フローとデータフロー

1. **作成**: pg_hmac_create でコンテキストを割り当て、バックエンドであれば ResourceOwner に登録します。 2. **初期化**: pg_hmac_init でハッシュアルゴリズム（MD5, SHA系）と秘密鍵を設定します。 3. **更新**: pg_hmac_update を繰り返し呼び出し、データをストリームとして処理します。 4. **終了**: pg_hmac_final で最終的な MAC 値を生成し、バッファサイズを確認します。 5. **解放**: pg_hmac_free で OpenSSL のコンテキストを破棄し、メモリを解放します。

読者向けメモ

  • **メモリ管理**: バックエンドでは TopMemoryContext を使用し、フロントエンドでは malloc を使用するよう ALLOC/FREE マクロで抽象化されています。
  • **エラー処理**: OpenSSL のエラーは ERR_get_error() を通じて取得し、pg_hmac_error() で人間が読み取れる形式に変換されます。
  • **安全性**: pg_hmac_free 内で explicit_bzero を使用し、機密情報が含まれる可能性のあるコンテキストを確実に消去しています。
