src/common/exec.c

概要

実行ファイルの検索、検証、およびパスの正規化を行うためのユーティリティ関数群を提供します。PostgreSQLのバックエンドおよびフロントエンドツールが、自身の実行場所や関連するバイナリを特定し、正当性を確認するために使用されます。

モジュール内での役割

src/common モジュールの一部として、OS依存のパス操作やプロセス実行に関連する低レベルな抽象化を提供します。特に、動的ロードや外部コマンド実行の前提条件となる「正しい実行ファイルの特定」を担います。

責務

  • **実行ファイルの検証**: 指定されたパスがファイルとして存在し、実行可能かつ読み取り可能であることを確認する。
  • **実行パスの特定**: argv[0] や環境変数 PATH を用いて、現在のプロセスの絶対パスを解決する。
  • **関連バイナリの検索**: 特定のディレクトリ内で、バージョン整合性を確認しながら別の実行ファイルを探す。
  • **OS固有の処理**: Windowsにおける .exe 拡張子の補完や、セキュリティトークン（DACL）の操作、ASLRの無効化など、プラットフォーム特有の差異を吸収する。

重要なシンボル

  • validate_exec: パスの存在と実行権限を検証する。
  • find_my_exec: 現在のプロセスの絶対パスを取得する。
  • find_other_exec: 同一ディレクトリ内の別バイナリを検索・検証する。
  • pipe_read_line: 外部コマンドを実行し、その出力を1行読み取る。
  • set_pglocale_pgservice: ロケールと設定ディレクトリをパスに基づいて設定する。

制御フローとデータフロー

1. **パス解決**: find_my_exec が argv[0] を受け取り、パス区切り文字の有無を判定。なければ PATH 環境変数を走査し、validate_exec で候補を順次検証する。 2. **正規化**: normalize_exec_path が realpath(3) を呼び出し、シンボリックリンクを解決して絶対パスに変換する。 3. **検証**: validate_exec は stat(2) でファイル属性を確認し、プラットフォームに応じて access(2) またはファイルモードビットで権限をチェックする。 4. **外部連携**: find_other_exec はパスを特定後、-V オプション付きでコマンドを実行し、pipe_read_line でバージョン文字列を比較して整合性を保証する。

直接依存

  • postgres.h / postgres_fe.h: 環境に応じたヘッダーの切り替え。
  • <sys/stat.h>, <unistd.h>, <signal.h>: 標準的なシステムコール。
  • common/string.h: パス操作や文字列処理の補助。
  • sys/personality.h / sys/procctl.h: (EXEC_BACKEND時) ASLR制御用。

読者向けメモ

  • **Windows対応**: Windows環境では .exe の自動補完や _fullpath の使用など、POSIXとは異なるコードパスが多用されています。
  • **エラー報告**: log_error マクロにより、フロントエンド（stderr出力）とバックエンド（ereport）の両方で一貫したエラーハンドリングが行われるよう設計されています。
  • **realpath**: macOS環境では _DARWIN_BETTER_REALPATH を定義することで、より正確なパス解決を保証しています。
