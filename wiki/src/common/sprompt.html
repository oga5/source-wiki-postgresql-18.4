src/common/sprompt.c

概要

sprompt.c は、対話的なユーザー入力（主にパスワードやユーザー名）を安全かつポータブルに取得するためのユーティリティを提供します。特に、パスワード入力時のエコーバック抑止や、割り込み（SIGINT）への対応を抽象化します。

モジュール内での役割

src/common 内のフロントエンド向けユーティリティの一部です。psql や pg_dump などのクライアントアプリケーションが、ユーザーから機密情報を取得する際の標準的なインターフェースとして機能します。

責務

  • **対話型入力の抽象化**: /dev/tty（Windowsでは CONIN$/CONOUT$）を優先的に使用し、標準入出力がリダイレクトされている場合でも対話的なプロンプトを実現します。
  • **エコー制御**: パスワード入力時に端末の表示設定を一時的に変更し、入力内容を隠蔽します。
  • **割り込み処理**: PromptInterruptContext を介したシグナルハンドリングとの連携により、入力中のキャンセルを安全に処理します。
  • **プラットフォーム差異の吸収**: POSIX（termios）と Windows（ConsoleMode）の端末制御の違いを隠蔽します。

重要なシンボル

  • simple_prompt: 基本的な入力取得関数。
  • simple_prompt_extended: 割り込みコンテキストを受け取り、キャンセル処理を可能にする拡張版。
  • PromptInterruptContext: 割り込み発生時に longjmp を行うための制御構造体。

制御フローとデータフロー

1. **端末のオープン**: /dev/tty を優先的に開き、失敗した場合は stdin/stderr にフォールバックします。 2. **エコー設定**: echo 引数が false の場合、プラットフォーム固有の API を使用して端末のエコーを無効化します。 3. **プロンプト表示**: 指定されたプロンプト文字列を termout に出力します。 4. **入力取得**: pg_get_line を呼び出し、入力を取得します。 5. **後処理**: 入力文字列から改行コードを除去し、必要に応じて端末設定を復元（エコー有効化）し、ファイルをクローズして結果を返します。

直接依存

  • c.h: PostgreSQL の基本定義。
  • common/fe_memutils.h: メモリ管理（pg_strdup 等）。
  • common/string.h: 文字列操作（pg_strip_crlf 等）。
  • termios.h: POSIX 端末制御（HAVE_TERMIOS_H 定義時）。

関連コード

  • src/common/pg_get_line.c: 入力行の読み取りロジック。
  • src/common/string.c: 文字列処理ユーティリティ。

読者向けメモ

  • **Windows 特有の挙動**: CONIN$/CONOUT$ を w+ モードで開くのは、Windows API がコンソールとして認識するために必要な回避策です。
  • **メモリ管理**: 返される文字列は malloc されているため、呼び出し側で必ず free する必要があります。
  • **非対話環境**: stdin がパイプされている場合でも動作するように設計されていますが、その場合はエコー抑止は機能しません。
