src/common/percentrepl.c

概要

percentrepl.c は、文字列内の % で始まるプレースホルダーを、指定された値に置換するための汎用的なユーティリティ関数を提供します。主に GUC（Grand Unified Configuration）パラメータの文字列展開に使用されます。

モジュール内での役割

src/common 内の文字列操作ユーティリティの一部であり、バックエンド（PostgreSQLサーバー）とフロントエンド（クライアントツール）の両方から利用可能です。

責務

  • 入力文字列を走査し、% エスケープシーケンスを識別する。
  • 可変長引数で渡された対応値を用いて、プレースホルダーを文字列置換する。
  • %% を単一の % に変換する。
  • 不正なプレースホルダーや、文字列末尾の不完全なエスケープに対してエラーを報告する。

重要なシンボル

  • replace_percent_placeholders: 本ファイルの唯一の公開関数。instr（対象文字列）、param_name（エラー報告用）、letters（置換対象文字のリスト）、および可変長引数（置換後の文字列）を受け取ります。

制御フローとデータフロー

1. initStringInfo で結果格納用のバッファを初期化します。 2. 入力文字列 instr を一文字ずつ走査します。 3. % を検出した場合：

  • 次の文字が % なら、% をバッファに追加してスキップします。
  • それ以外の場合、letters 文字列を走査して対応する va_arg を取得します。
  • 値が見つかればバッファに追加し、見つからない（または値が NULL）場合はエラーを発生させます。

    • 4. % 以外はそのままバッファに追加します。 5. 最終的に構築された result.data を返します。

直接依存

  • postgres.h / postgres_fe.h: 環境に応じた基本定義。
  • common/logging.h: フロントエンド用のエラーログ出力。
  • common/percentrepl.h: 関数プロトタイプ定義。
  • lib/stringinfo.h: 動的文字列構築のための StringInfo API。

関連コード

  • src/include/common/percentrepl.h: インターフェース定義。
  • src/common/stringinfo.c: StringInfo の実装。本関数が依存するバッファ管理の基盤です。

読者向けメモ

  • **エラーハンドリング**: FRONTEND マクロの有無により、ereport（バックエンド）と pg_log_error + exit（フロントエンド）を切り替えています。
  • **制約**: すべての置換値は文字列（char *）である必要があります。また、この関数はすべての置換値が事前に準備できている場合に適しており、複雑な条件分岐や型混在が必要なケースには向きません。
  • **NULL値の扱い**: letters に含まれていても、対応する値が NULL であればエラーとして扱われます。これにより、動的な機能制限を実装可能です。
