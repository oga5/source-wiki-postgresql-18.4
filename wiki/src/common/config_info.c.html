src/common/config_info.c

概要

PostgreSQLのビルド時設定情報（インストールディレクトリやコンパイルオプションなど）を収集し、pg_config ユーティリティ等で利用可能な形式で提供するための共通コードです。

モジュール内での役割

src/common ディレクトリの一部として、フロントエンドおよびバックエンドの両方から利用可能な設定取得インターフェースを提供します。主にインストールパスの動的な算出と、ビルド時に埋め込まれた定数の抽出を担います。

責務

  • **設定情報の集約**: BINDIR や LIBDIR などのディレクトリパス、および CFLAGS や LDFLAGS などのコンパイル時フラグを ConfigData 構造体の配列として構築します。
  • **パスの正規化**: 実行ファイルのパスを基点として、各ディレクトリパスを算出・整形します。
  • **メモリ管理**: 呼び出し元が解放可能なメモリ領域を確保し、設定情報を返却します。

重要なシンボル

  • ConfigData: 設定名（name）と値（setting）のペアを保持する構造体。
  • get_configdata(): 設定情報の配列を生成する唯一の公開関数。呼び出し元は結果を pfree する必要があります。

追加の索引済みシンボル

  • get_configdata

制御フローとデータフロー

1. **初期化**: get_configdata が呼び出されると、必要な項目数（23個）に基づいて palloc_array でメモリを確保します。 2. **パス算出**: 引数 my_exec_path を基に、get_doc_path や get_lib_path などのヘルパー関数を呼び出し、各ディレクトリパスを生成します。 3. **定数抽出**: ビルド時に定義されたマクロ（VAL_CFLAGS 等）が存在する場合、その値をコピーします。未定義の場合は「not recorded」という文字列を格納します。 4. **返却**: 構築された配列を呼び出し元に返します。

直接依存

  • postgres.h / postgres_fe.h: 環境に応じた基本定義。
  • common/config_info.h: ConfigData 構造体および関数プロトタイプの定義。

関連コード

  • src/bin/pg_config/: 本モジュールを利用して設定情報を表示する主要なクライアント。
  • src/include/pg_config_paths.h: インストールパス算出の基盤となる定数定義。

読者向けメモ

  • **メモリ所有権**: get_configdata が返すメモリは palloc されているため、呼び出し側で適切に解放する必要があります。
  • **項目数の変更**: configdata_len を更新する際は、関数内のインデックス i との整合性に注意してください。
  • **フロントエンド利用**: FRONTEND マクロの定義により、バックエンド用とフロントエンド用の両方のビルド環境でコンパイル可能です。
