src/common/unicode/category_test.c

概要

PostgreSQL の Unicode カテゴリおよび文字プロパティの実装が、ICU（International Components for Unicode）ライブラリの定義と一致しているかを検証するためのテストプログラムです。

モジュール内での役割

src/common/unicode モジュールにおける品質保証ツールです。PostgreSQL 内部の Unicode 変換テーブルやプロパティ判定ロジックが、業界標準である ICU と乖離していないかを全コードポイント（0x000000 - 0x10FFFF）走査によって確認します。

責務

  • **整合性検証**: PostgreSQL 独自の Unicode プロパティ関数と ICU の対応する関数を比較。
  • **バージョン差異の許容**: PostgreSQL と ICU で使用している Unicode バージョンが異なる場合に発生する「未割り当てコードポイント」の差異を適切にスキップする。
  • **網羅的テスト**: 全 Unicode コードポイントをループし、カテゴリ、バイナリプロパティ、および文字クラス（alpha, digit, space 等）の判定結果を照合する。

重要なシンボル

  • icu_test(): テストのメインロジック。全コードポイントを走査し、不一致があればエラーを出力して終了する。
  • parse_unicode_version(): バージョン文字列（例: "15.1"）を比較可能な整数値に変換する。
  • pg_unicode_version / icu_unicode_version: テスト対象のバージョンを保持するグローバル変数。

追加の索引済みシンボル

  • icu_test
  • main

制御フローとデータフロー

1. **初期化**: main 関数で PG_UNICODE_VERSION を解析し、ICU が有効な場合は U_UNICODE_VERSION も解析する。 2. **走査**: icu_test 内で 0 から 0x10ffff までループ。 3. **判定**: 各コードポイントに対し、PostgreSQL の pg_u_* 関数群と ICU の u_* 関数群を呼び出し、結果を比較。 4. **例外処理**: バージョン不一致による「未割り当て」の差異を検知した場合は、エラーとせずスキップする。 5. **終了**: 不一致が検出された場合は詳細な情報を出力して exit(1) を実行。

読者向けメモ

  • 本ファイルは USE_ICU が定義されている環境でのみ動作します。
  • テストが失敗する場合、PostgreSQL 側のテーブル生成スクリプトまたはプロパティ判定ロジックに、Unicode バージョン更新に伴う不整合が生じている可能性が高いです。
  • バージョン不一致時のスキップロジックは、古い Unicode バージョンで「未割り当て」だったコードポイントが新しいバージョンで「割り当て済み」になった場合に発生する誤検知を防ぐための重要な安全装置です。
