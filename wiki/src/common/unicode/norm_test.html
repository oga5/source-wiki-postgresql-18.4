src/common/unicode/norm_test.c

概要

norm_test.c は、PostgreSQL の Unicode 正規化機能（unicode_normalize）の正確性を検証するためのスタンドアロンのテストプログラムです。Unicode コンソーシアムが提供する正規化テストデータに基づき、実装が仕様通りに動作するかを確認します。

モジュール内での役割

src/common/unicode ディレクトリ内で、正規化ロジックの回帰テストを担当します。本番コードではなく、ビルドプロセスや開発環境での検証用ツールとして機能します。

責務

  • **テスト実行**: norm_test_table.h に定義されたテストケースを順次読み込み、正規化処理を実行する。
  • **結果検証**: 期待値（output）と実際の出力（result）を比較し、不一致がある場合に詳細なデバッグ情報を出力して終了する。
  • **可視化**: pg_wchar 型の文字列を人間が読みやすい U+XXXX 形式の文字列に変換する。

重要なシンボル

  • main: テストスイートのメインループ。全テストケースと4つの正規化形式（NFC, NFD, NFKC, NFKD）を網羅的に検証する。
  • unicode_normalize: 検証対象となるコア関数（common/unicode_norm.h で定義）。
  • UnicodeNormalizationTests: norm_test_table.h で定義されたテストデータの配列。
  • pg_wcscmp: pg_wchar 文字列を比較する補助関数。

追加の索引済みシンボル

  • print_wchar_str

制御フローとデータフロー

1. **初期化**: main 関数が UnicodeNormalizationTests 配列をイテレートする。 2. **正規化実行**: 各テストケースに対し、0 から 3 の正規化形式を指定して unicode_normalize を呼び出す。 3. **比較**: pg_wcscmp を用いて、計算結果と期待値を比較する。 4. **エラーハンドリング**: 比較失敗時、print_wchar_str を使用して入力・期待値・結果を標準出力にダンプし、exit(1) で異常終了する。 5. **完了**: 全テストを通過した場合、「All tests successful!」と出力して終了する。

直接依存

  • postgres_fe.h: フロントエンド用共通ヘッダー。
  • common/unicode_norm.h: 正規化関数の宣言。
  • norm_test_table.h: テストデータ（UnicodeNormalizationTests）の定義。
  • 標準ライブラリ: stdio.h, stdlib.h, string.h（出力およびメモリ管理用）。

関連コード

  • src/common/unicode/norm_test_table.h: テストケースのデータソース。
  • src/common/unicode/unicode_norm.c: unicode_normalize 関数の実装本体。

読者向けメモ

  • 本ファイルは PostgreSQL のサーバー本体ではなく、スタンドアロンのテストツールとしてビルドされます。
  • print_wchar_str は静的バッファを使用しているため、スレッドセーフではありませんが、本テストプログラムはシングルスレッドで動作するため問題ありません。
  • テストデータは Unicode 正規化テストの仕様に厳密に従っているため、実装のバグ修正や機能追加時の回帰テストとして非常に重要です。
