src/common/unicode/case_test.c

概要

PostgreSQL の Unicode 大文字・小文字変換（Case Mapping）機能の正確性を検証するためのテストプログラムです。PostgreSQL 内部実装と、外部ライブラリである ICU（International Components for Unicode）の出力を比較検証します。

モジュール内での役割

src/common/unicode モジュールにおける品質保証の要です。Unicode のバージョン更新時や、変換ロジックの変更時に、期待される変換結果が維持されているかを自動的に確認する役割を担います。

責務

  • **ICU との整合性検証**: 全ての有効な Unicode コードポイントに対し、PostgreSQL の変換結果が ICU の結果と一致するかを網羅的にテストします。
  • **変換関数の境界値テスト**: unicode_strlower 等の関数に対し、NULL 終端の有無やバッファサイズが異なる多様なケースでの動作を検証します。
  • **ロケール非依存の変換確認**: ロケールに依存しない標準的な Unicode 変換ルールが正しく適用されているかを保証します。

重要なシンボル

  • WordBoundaryState: タイトルケース変換時の単語境界を判定するための状態保持構造体。
  • initcap_wbnext: 単語境界を特定するためのイテレータ関数。
  • icu_test_full: 文字列全体に対する ICU との比較テストを実行する関数。
  • test_convert: 変換関数の引数（ソース・宛先）のバッファ状態を網羅的にテストする汎用テスト関数。

制御フローとデータフロー

1. **初期化**: main 関数で ICU の UCaseMap を初期化し、PostgreSQL と ICU の Unicode バージョンを表示します。 2. **網羅的テスト**: test_icu が全コードポイントを走査し、icu_test_simple で単一文字の変換結果を比較します。 3. **文字列変換テスト**: test_convert_case が特定の文字列セットを定義し、test_convert を通じて tfunc_lower/upper/title/fold を呼び出します。 4. **検証**: 各変換関数は unicode_str* 系関数を呼び出し、結果を期待値と比較します。不一致があれば exit(1) で異常終了します。

読者向けメモ

  • 本ファイルは USE_ICU が定義されている環境で最も強力に機能します。ICU がない環境では、基本的な変換関数のテストのみが実行されます。
  • test_convert 関数は、メモリ管理やバッファの境界条件を意図的に複雑に設定してテストしているため、変換関数の堅牢性を確認する際の参考になります。
  • ICU のバージョンと PostgreSQL が内蔵する Unicode データのバージョンが異なる場合、一部のコードポイントで不一致が発生する可能性があるため、テスト実行時のログ出力には注意してください。
