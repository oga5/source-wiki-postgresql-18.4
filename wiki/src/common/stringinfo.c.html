src/common/stringinfo.c

概要

stringinfo.c は、PostgreSQL における可変長文字列バッファ StringInfo を管理するための汎用ライブラリです。メモリの動的確保を自動的に行い、C 言語の文字列操作に伴うバッファオーバーフローのリスクを軽減します。

モジュール内での役割

src/common ディレクトリに位置し、バックエンド（PostgreSQL サーバ）とフロントエンド（psql や pg_dump 等のクライアントツール）の両方から利用される共通ユーティリティです。メモリ管理には palloc/repalloc を使用し、フロントエンドでは malloc にフォールバックする仕組みを備えています。

責務

  • **動的バッファ管理**: 文字列やバイナリデータの追加時に、必要に応じてバッファを自動拡張（倍増戦略）する。
  • **データ構造の抽象化**: StringInfoData 構造体を通じて、文字列の長さ、最大容量、カーソル位置を統合管理する。
  • **安全性**: 常にヌル終端を維持する（一部のバイナリ用関数を除く）ことで、C 文字列としての安全性を保証する。

重要なシンボル

  • StringInfoData: バッファの実体。data（ポインタ）、len（現在長）、maxlen（容量）、cursor（走査用）を保持。
  • appendStringInfo: printf 形式でフォーマットされた文字列をバッファに追加する主要関数。
  • appendBinaryStringInfo: ヌル文字を含む可能性のあるバイナリデータを追加する関数。
  • enlargeStringInfo: バッファの再確保を行う内部ロジック。
  • makeStringInfo / initStringInfo: オブジェクトの生成・初期化を行うコンストラクタ群。

追加の索引済みシンボル

制御フローとデータフロー

1. **初期化**: initStringInfo 等で palloc を用いて初期バッファを確保。 2. **追加**: appendStringInfo 等が呼ばれると、まず enlargeStringInfo が現在の空き容量を確認。 3. **拡張**: 容量不足の場合、repalloc を用いて maxlen を倍増させ、新しいメモリ領域を確保。 4. **書き込み**: pvsnprintf や memcpy を用いてデータをバッファにコピーし、len を更新。 5. **終了**: destroyStringInfo で pfree を呼び出し、メモリを解放。

直接依存

  • postgres.h / postgres_fe.h: 環境に応じた基本定義。
  • utils/memutils.h: メモリ割り当て関数（palloc, repalloc, pfree）。
  • lib/stringinfo.h: 構造体定義および関数プロトタイプ。

関連コード

  • src/include/lib/stringinfo.h: 本ファイルのヘッダ。利用可能な全 API が定義されている。
  • src/backend/utils/mmgr/: palloc 等のメモリ管理の実装。

読者向けメモ

  • **読み取り専用モード**: maxlen == 0 の StringInfo は読み取り専用です。これに対して reset や append を行うと Assert で停止するため注意してください。
  • **パフォーマンス**: appendStringInfo は va_list の再走査を避けるため、内部で appendStringInfoVA をループ呼び出しする設計になっています。頻繁な追加が発生する場合は、事前に enlargeStringInfo で十分なサイズを確保することでオーバーヘッドを削減可能です。
  • **ヌル終端**: appendBinaryStringInfoNT はヌル終端を保証しません。バイナリデータを扱う際は、用途に応じて NT（No-Terminator）版と使い分けてください。
