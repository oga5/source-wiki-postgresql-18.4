src/common/sha2.c

概要

SHA-224, SHA-256, SHA-384, SHA-512 アルゴリズムのフォールバック実装を提供します。OpenSSL 等の外部ライブラリが利用できない環境や、PostgreSQL の内部的なハッシュ計算処理において使用されます。

モジュール内での役割

src/common 内の暗号ハッシュ関連機能の一部であり、sha2_int.h で定義されたコンテキスト構造体を用いて、データの初期化、更新、最終化を行う一連の API を提供します。

責務

  • SHA-2 ファミリーの各アルゴリズムに対する init, update, final 関数の実装。
  • ビッグエンディアン/リトルエンディアン環境を吸収するためのバイト順序変換。
  • メッセージブロックのパディング処理および変換関数の実行。
  • バックエンド（palloc）とフロントエンド（malloc）のメモリ管理の抽象化。

重要なシンボル

  • pg_sha256_init/update/final: SHA-256 用のメイン API。
  • pg_sha512_init/update/final: SHA-512 用のメイン API。
  • SHA256_Transform / SHA512_Transform: ブロック単位の圧縮関数（内部用）。
  • REVERSE32 / REVERSE64: エンディアン変換用マクロ。
  • Ch / Maj / Sigma / sigma: SHA-2 アルゴリズムの論理演算マクロ。

制御フローとデータフロー

1. **初期化**: init 関数でコンテキストを初期状態（定数 initial_hash_value）でリセットします。 2. **更新**: update 関数で入力データをバッファリングし、ブロック長（64/128バイト）に達するたびに Transform 関数を呼び出して中間状態を更新します。 3. **最終化**: final 関数でパディング（0x80 の付与とビット長情報の埋め込み）を行い、最後の Transform を実行して最終的なダイジェストを生成します。 4. **クリーンアップ**: セキュリティ上の理由から、final 実行後にコンテキストを memset でゼロクリアします。

直接依存

  • postgres.h / postgres_fe.h: 環境に応じた共通定義。
  • sha2_int.h: アルゴリズムごとのコンテキスト構造体定義。

関連コード

  • src/include/common/sha2.h: 公開インターフェースと定数定義。
  • src/common/cryptohash.c: 本ファイルを含むハッシュ関数群を抽象化する高レベル API。

読者向けメモ

  • 本ファイルは Aaron D. Gifford 氏による実装をベースとしています。
  • SHA2_UNROLL_TRANSFORM を定義することで、変換ループをアンロールしてパフォーマンスを最適化可能です。
  • SHA-224 は SHA-256 の、SHA-384 は SHA-512 のラッパーとして実装されており、コードの重複を避ける設計になっています。
  • エンディアン変換マクロは WORDS_BIGENDIAN が未定義の場合のみ有効化されます。
