src/common/wait_error.c

概要

wait_error.c は、wait(2) や waitpid(2)、system(3) 等から返されるプロセス終了ステータスを、人間が判読可能な文字列や標準的な終了コードに変換するためのユーティリティ群を提供します。

モジュール内での役割

src/common 内の他のユーティリティと同様、バックエンドとフロントエンド（クライアントツール）の両方から利用可能な共通機能を提供します。プロセス管理を行うコードにおいて、終了ステータスの解釈を標準化する役割を担います。

責務

  • プロセス終了ステータスの人間向け説明文の生成。
  • シグナルによる終了と、シェル経由の終了（128 + シグナル番号）の抽象化。
  • 終了ステータスから標準的な終了コード（0-255）への正規化。

重要なシンボル

  • wait_result_to_str: ステータスを説明する文字列を返します。palloc されたメモリを返すため、呼び出し側での解放が必要です。
  • wait_result_is_signal: 特定のシグナルで終了したかを判定します。シェル経由の終了コードも考慮します。
  • wait_result_is_any_signal: シグナルによる終了、またはシェル特有の終了コード（126, 127等）を判定します。
  • wait_result_to_exit_code: 複雑な wait ステータスを、シェルが期待する 0-255 の終了コードに変換します。

制御フローとデータフロー

  • 各関数は WIFEXITED や WIFSIGNALED マクロを使用してステータスを分解します。
  • wait_result_to_str は、exitstatus == -1 の場合に errno を参照してエラーメッセージを生成する特殊なパスを持ちます。
  • シェル経由の実行（system 等）を考慮し、終了コード 126（実行不可）や 127（コマンド未検出）を明示的に処理します。

直接依存

  • postgres.h / postgres_fe.h: 環境に応じた共通定義。
  • <signal.h>: シグナル関連の定数。
  • <sys/wait.h>: wait ステータス解析用マクロ（WIFEXITED 等）。
  • signal.h
  • sys/wait.h

関連コード

  • src/common/exec.c: プロセスの実行に関連するコード。
  • src/include/port.h: プラットフォーム依存のプロセス管理差異を吸収する定義。

読者向けメモ

  • **シェル経由の挙動**: POSIX シェルは、プロセスがシグナルで終了した場合に 128 + シグナル番号 を終了コードとして返します。本ファイル内の関数は、この慣習を考慮して設計されています。
  • **メモリ管理**: wait_result_to_str が返す文字列は動的に確保されるため、メモリリークを避けるために呼び出し側で適切に管理してください。
  • **プラットフォーム**: Windows 環境では WTERMSIG の扱いが異なるため、条件付きコンパイルで対応しています。
