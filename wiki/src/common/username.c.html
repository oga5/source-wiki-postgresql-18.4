src/common/username.c

概要

実行中のプロセスに関連付けられた現在のユーザー名を取得するためのユーティリティモジュールです。プラットフォーム固有のユーザーID解決メカニズムを抽象化し、PostgreSQLのフロントエンドおよびバックエンドの両方で利用可能なインターフェースを提供します。

モジュール内での役割

src/common ディレクトリの一部として、OSレベルのユーザー情報を取得する共通機能を提供します。主にクライアントツール（psql や pg_dump 等）が実行ユーザーを特定する際や、バックエンドが認証プロセスでユーザー情報を参照する際に使用されます。

責務

  • **ユーザー名解決**: POSIX環境では getpwuid、Windows環境では GetUserName を使用して現在のユーザー名を特定する。
  • **エラーハンドリング**: ユーザー名取得失敗時に、詳細なエラーメッセージを生成し、呼び出し元へ通知する。
  • **強制終了のサポート**: 取得失敗時にプログラムを即座に終了させるためのヘルパー関数を提供する。

重要なシンボル

  • get_user_name(char **errstr): 現在のユーザー名を返すメイン関数。失敗時は errstr にエラー詳細を格納し NULL を返す。
  • get_user_name_or_exit(const char *progname): get_user_name を呼び出し、失敗した場合は指定されたプログラム名と共にエラーを標準エラー出力へ書き出し、exit(1) を実行する。

追加の索引済みシンボル

  • get_user_name_or_exit

制御フローとデータフロー

1. **呼び出し**: 呼び出し元が get_user_name を呼び出す。 2. **プラットフォーム分岐**:

  • **POSIX**: geteuid() でユーザーIDを取得し、getpwuid() でパスワードデータベースを検索する。
  • **Windows**: GetUserName() を呼び出し、静的バッファにユーザー名を格納する。

    • 3. **エラー処理**: 取得に失敗した場合、psprintf を用いて動的にエラーメッセージを生成し、呼び出し元に返却する。 4. **終了処理**: get_user_name_or_exit が使用された場合、戻り値が NULL であれば即座にプロセスを終了させる。

直接依存

  • postgres.h / postgres_fe.h: 環境に応じたPostgreSQLの基本定義。
  • pwd.h (POSIX): ユーザーデータベースアクセス用。
  • unistd.h (POSIX): geteuid() 用。
  • common/username.h: 本モジュールのヘッダーファイル。

関連コード

  • src/common/psprintf.c: エラーメッセージ生成に使用される安全な文字列フォーマット関数。
  • src/include/common/username.h: 公開インターフェース定義。

読者向けメモ

  • **静的バッファ**: Windows環境では静的バッファを使用しているため、複数回呼び出した場合、以前の戻り値は上書きされる可能性があります。
  • **スレッドセーフ**: 本実装はスレッドセーフを考慮した設計ではありません。主にシングルスレッドのツールや、初期化フェーズでの利用を想定しています。
  • **エラーメッセージ**: errstr に返されるメッセージは palloc されているため、呼び出し元で適切に解放する必要があります。
