src/common/encnames.c

概要

PostgreSQL における文字エンコーディング名と内部 ID の相互変換、および外部ライブラリ（gettext, ICU）とのマッピングを管理するユーティリティファイルです。

モジュール内での役割

src/common 内で、文字エンコーディングに関するメタデータの中央リポジトリとして機能します。クライアントおよびサーバーサイドのエンコーディング検証や、外部システムとの連携に必要な名前変換を提供します。

責務

  • エンコーディング名の正規化（不要な文字の除去と小文字化）。
  • 文字列名から内部 ID (pg_enc) への変換。
  • 内部 ID から文字列名への変換。
  • ICU や gettext が期待するエンコーディング名へのマッピング。
  • クライアント/サーバーエンコーディングの妥当性検証。

重要なシンボル

  • pg_encname_tbl: エンコーディング名（エイリアス含む）と内部 ID の対応表。二分探索用。
  • pg_enc2name_tbl: 内部 ID から公式名およびコードページへのマッピング。
  • pg_char_to_encoding(): 文字列から ID を検索する主要関数。
  • pg_encoding_to_char(): ID から文字列へ変換する主要関数。
  • clean_encoding_name(): 検索キーを正規化する内部ヘルパー。

制御フローとデータフロー

1. **検索**: pg_char_to_encoding が呼び出されると、clean_encoding_name で入力文字列を英数字のみの小文字に変換します。 2. **照合**: 正規化されたキーを用いて pg_encname_tbl に対して二分探索を行い、一致する pg_enc ID を返します。 3. **変換**: pg_encoding_to_char は、ID をインデックスとして pg_enc2name_tbl を直接参照し、定数文字列を返します。 4. **検証**: pg_valid_server_encoding 等の関数は、変換結果が特定のエンコーディング範囲（PG_VALID_BE_ENCODING 等）に収まっているかを確認します。

直接依存

  • c.h: PostgreSQL の基本型定義。
  • ctype.h: isalnum() 等の文字操作。
  • unistd.h: POSIX 準拠の標準定義。
  • mb/pg_wchar.h: pg_enc 型定義およびエンコーディング関連のマクロ。

関連コード

  • src/include/mb/pg_wchar.h: エンコーディング ID の定義や検証マクロの定義元。
  • src/backend/utils/mb/: 実際のエンコーディング変換ロジックが実装されているディレクトリ。

読者向けメモ

  • **検索の仕組み**: pg_encname_tbl はアルファベット順にソートされており、二分探索で効率的に検索されます。エイリアス（例: win -> WIN1251）もこのテーブルで解決されます。
  • **正規化**: clean_encoding_name は isalnum を使用して記号を無視するため、ISO-8859-1 と iso88591 は同じキーとして扱われます。
  • **ICU 対応**: pg_enc2icu_tbl はバックエンドエンコーディングのみを対象としています。StaticAssertDecl により、テーブルのサイズがエンコーディング数と一致していることが保証されています。
