src/common/kwlookup.c

概要

kwlookup.c は、PostgreSQL の構文解析器などで使用されるキーワード検索機能を提供します。SQL 標準に準拠した大文字・小文字を区別しない高速なキーワード照合を実現するための汎用的なユーティリティです。

モジュール内での役割

src/common ディレクトリの一部として、フロントエンドおよびバックエンドの両方から利用可能な共通ライブラリとして機能します。主に src/include/common/kwlookup.h で定義されたデータ構造を操作し、特定のキーワードリストに対する効率的な検索処理を抽象化します。

責務

  • **高速なキーワード照合**: パーフェクトハッシュ関数を利用し、キーワードリスト内での検索を効率化します。
  • **SQL 標準準拠の比較**: ロケールに依存しない、ASCII ベースの「A-Z から a-z への変換」による大文字・小文字を区別しない比較を行います。
  • **境界チェック**: 入力文字列がキーワードの最大長を超えている場合に即座に拒否し、不要な計算を回避します。

重要なシンボル

  • ScanKeywordLookup: 本ファイルの主要関数。入力文字列がキーワードリストに含まれるか判定し、該当する場合はそのインデックスを返します。
  • ScanKeywordList: 検索対象のキーワード群、オフセット情報、ハッシュ関数を保持する構造体。

制御フローとデータフロー

1. **事前チェック**: 入力文字列の長さを max_kw_len と比較し、長すぎる場合は即座に -1 を返します。 2. **ハッシュ計算**: 提供された hash 関数を呼び出し、候補となるキーワードのインデックスを特定します。 3. **検証**: ハッシュ値が有効範囲内であることを確認後、GetScanKeyword を用いて該当するキーワード文字列を取得します。 4. **比較**: 入力文字列を 1 文字ずつ ASCII 範囲内で小文字化しながら、取得したキーワードと照合します。一致すればインデックスを返し、不一致であれば -1 を返します。

直接依存

  • c.h: PostgreSQL の基本型定義やマクロ。
  • common/kwlookup.h: ScanKeywordList 構造体および GetScanKeyword インライン関数の定義。

関連コード

  • src/tools/gen_keywordlist.pl: ScanKeywordList 構造体に必要なデータ（文字列、オフセット、ハッシュ関数）を生成するためのツール。
  • src/common/keywords.c: 実際の SQL キーワードリストの定義や管理を行うファイル。

読者向けメモ

  • **ロケールへの配慮**: 本実装では tolower() を使用していません。これは、トルコ語などの特定のロケールで発生しうる予期せぬ文字変換を避け、SQL 標準が要求する ASCII ベースの比較を保証するためです。
  • **パーフェクトハッシュ**: この関数はハッシュ値が直接インデックスを指すことを前提としています。ハッシュ関数が生成する値が 0 から num_keywords - 1 の範囲外である場合、検索は失敗とみなされます。
