src/common/scram-common.c

概要

本ファイルは、PostgreSQLにおけるSCRAM（Salted Challenge Response Authentication Mechanism）認証に必要な低レベルの暗号学的計算を実装しています。RFC 5802に準拠し、フロントエンド（クライアント）とバックエンド（サーバー）の両方から利用可能な共有コードとして提供されます。

モジュール内での役割

src/common モジュールの一部として、認証サブシステムが依存する暗号化プリミティブを提供します。認証プロトコルそのものの制御ロジックではなく、パスワードハッシュやキー生成といった計算処理に特化しています。

責務

  • **SaltedPasswordの生成**: PBKDF2アルゴリズムを用いたパスワードのハッシュ化。
  • **キー導出**: SCRAM認証に必要な ClientKey および ServerKey の計算。
  • **シークレットの構築**: データベースに保存される形式（SCRAM-SHA-256$iterations:salt$stored_key:server_key）への文字列変換。
  • **環境非依存の提供**: FRONTEND マクロの切り替えにより、クライアントツールとサーバープロセスの双方で同一のロジックを保証。

重要なシンボル

  • scram_SaltedPassword: PBKDF2を用いてパスワードからソルト付きハッシュを生成する。
  • scram_ClientKey / scram_ServerKey: HMACを用いて各キーを導出する。
  • scram_build_secret: 認証情報を保存用文字列にエンコードする。
  • scram_H: 指定されたハッシュ関数で入力をハッシュ化する汎用関数。

制御フローとデータフロー

1. **初期化**: 各関数は pg_hmac_create または pg_cryptohash_create を呼び出し、コンテキストを生成します。 2. **計算**: scram_SaltedPassword は内部でループ処理を行い、HMACを繰り返してパスワードを強化します。バックエンド環境では CHECK_FOR_INTERRUPTS() が呼び出され、長時間計算に対する応答性を確保します。 3. **エンコード**: scram_build_secret は計算されたバイナリキーを pg_b64_encode を用いてBase64形式に変換し、最終的な認証文字列を構築します。 4. **終了**: 各関数はエラー発生時に errstr を設定し、コンテキストを解放して終了します。

直接依存

  • common/hmac.h, common/cryptohash.h: 暗号学的計算の基盤。
  • common/base64.h: 認証情報の文字列化。
  • port/pg_bswap.h: エンディアン変換（pg_hton32）。
  • postgres.h / postgres_fe.h: 環境に応じたヘッダーの切り替え。
  • common/scram-common.h
  • miscadmin.h

関連コード

  • src/common/hmac.c: HMAC計算の実装。
  • src/common/cryptohash.c: SHA-256等のハッシュ計算の実装。
  • src/backend/libpq/auth.c: 本ファイルの関数を呼び出し、実際のSCRAM認証フローを制御するサーバー側のコード。

読者向けメモ

  • **入力の正規化**: 本ファイルに渡されるパスワードは、呼び出し元で事前にSASLprepによる正規化が行われている必要があります。
  • **メモリ管理**: scram_build_secret が返す文字列は呼び出し元で解放（pfree または free）する必要があります。
  • **中断可能性**: scram_SaltedPassword のループ内には CHECK_FOR_INTERRUPTS() が含まれており、サーバープロセスがハングしないよう配慮されています。
