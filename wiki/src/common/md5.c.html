src/common/md5.c

概要

RFC 1321 に準拠した MD5 メッセージダイジェストアルゴリズムのフォールバック実装です。外部ライブラリ（OpenSSL 等）が利用できない環境や、PostgreSQL の内部的なパスワードハッシュ計算のために使用されます。

モジュール内での役割

src/common における暗号ハッシュ機能の一部であり、md5_int.h で定義されたコンテキスト構造体 (pg_md5_ctx) を操作する低レベルな実装を提供します。

責務

  • MD5 アルゴリズムのコア計算（ブロック処理、パディング、最終結果の生成）。
  • エンディアン（バイトオーダー）に依存しないデータ処理の提供。
  • pg_md5_init, pg_md5_update, pg_md5_final を通じた標準的なハッシュ計算インターフェースの提供。

重要なシンボル

  • pg_md5_ctx: ハッシュ計算の状態を保持する構造体。
  • md5_calc: 64バイトのブロックに対して MD5 の変換関数を適用する内部関数。
  • pg_md5_init/update/final: 外部から呼び出される MD5 計算のライフサイクル管理関数。
  • ROUND1 ～ ROUND4: MD5 アルゴリズムの各ラウンドで使用されるマクロ。

追加の索引済みシンボル

  • md5_pad
  • md5_result
  • pg_md5_final
  • pg_md5_init
  • pg_md5_update

制御フローとデータフロー

1. **初期化**: pg_md5_init でコンテキストの初期状態（定数 MD5_A0 等）を設定。 2. **更新**: pg_md5_update が入力データをバッファリングし、64バイトに達するたびに md5_calc を呼び出して中間状態を更新。 3. **終了**: pg_md5_final が md5_pad を呼び出し、パディング処理（メッセージ長の追加など）を行った後に最終的なハッシュ値を md5_result で出力バッファに書き出す。

直接依存

  • postgres.h / postgres_fe.h: 環境に応じた基本定義。
  • md5_int.h: pg_md5_ctx 構造体および関連定数の定義。

関連コード

  • src/include/common/md5.h: 公開 API 定義。
  • src/common/md5_common.c: pg_md5_hash などの高レベルなユーティリティ関数群。
  • src/common/cryptohash.c: PostgreSQL 全体で統一されたハッシュ API のラッパー。

読者向けメモ

  • **実装の性質**: 本ファイルは RFC 1321 に忠実な「純粋な」実装です。パフォーマンスよりも移植性と依存関係の排除を優先しています。
  • **エンディアン**: WORDS_BIGENDIAN マクロを使用して、ビッグエンディアン環境でのバイト順序を適切に調整しています。
  • **注意**: 本実装はパスワードハッシュ等の用途には適していますが、現代のセキュリティ要件（衝突耐性など）が求められる用途には、より強力なハッシュアルゴリズム（SHA-256 等）の使用を検討してください。
