src/common/pgfnames.c

概要

pgfnames.c は、指定されたディレクトリ内のファイル名一覧を取得し、メモリ管理を伴うリストとして返すためのユーティリティ関数群を提供します。PostgreSQLのバックエンドおよびフロントエンドの両方で利用可能です。

モジュール内での役割

src/common ディレクトリの一部として、ファイルシステム操作の抽象化レイヤーを担います。OS固有のディレクトリ走査処理をラップし、呼び出し元がメモリ管理を意識しつつ安全にファイル一覧を扱えるように設計されています。

責務

  • **ディレクトリ走査**: 指定されたパスのディレクトリを開き、. および .. を除外したファイル名リストを構築する。
  • **メモリ管理**: 動的に拡張可能な配列を確保し、取得したファイル名を格納する。また、確保したメモリを解放するためのクリーンアップ関数を提供する。
  • **エラーハンドリング**: ディレクトリのオープン、読み取り、クローズ時のエラーをログ出力（elog または pg_log_warning）する。

重要なシンボル

  • pgfnames(const char *path): ディレクトリ内のファイル名リスト（char **）を返すメイン関数。リストの末尾は NULL で終端されます。
  • pgfnames_cleanup(char **filenames): pgfnames で確保された各文字列および配列自体を解放する関数。

追加の索引済みシンボル

  • pgfnames_cleanup

制御フローとデータフロー

1. **初期化**: opendir でディレクトリを開き、初期サイズ 200 のポインタ配列を palloc で確保します。 2. **走査**: readdir をループで回し、. と .. 以外のエントリを pstrdup でコピーして配列に格納します。配列が不足した場合は repalloc で倍増させます。 3. **終了処理**: ループ終了後、配列の末尾に NULL を代入し、closedir を呼び出します。 4. **解放**: 呼び出し元が pgfnames_cleanup を呼ぶことで、リスト内の各文字列と配列ポインタが pfree されます。

直接依存

  • dirent.h: ディレクトリ操作用標準ヘッダー。
  • postgres.h / postgres_fe.h: 環境に応じた基本定義。
  • common/logging.h: フロントエンド用のログ出力機能。

関連コード

  • src/common/file_utils.c: ファイル操作に関連する他のユーティリティが含まれており、ファイルシステム操作の文脈で併用されることが多いです。

読者向けメモ

  • **メモリ管理の注意**: pgfnames が返すリストは palloc で確保されています。呼び出し元は必ず pgfnames_cleanup を呼び出す必要があります。そうしないとメモリリークが発生します。
  • **環境の抽象化**: FRONTEND マクロの有無により、エラーログの出力先（elog か pg_log_warning か）が切り替わる仕組みになっています。
  • **エラー時の挙動**: ディレクトリが空の場合やエラーが発生した場合は NULL を返す可能性があるため、呼び出し元での戻り値チェックが必須です。
