src/common/string.c

概要

PostgreSQL のバックエンドおよびフロントエンド（クライアントツール）の両方で利用可能な、汎用的な文字列操作ユーティリティを提供します。

モジュール内での役割

src/common ディレクトリの一部として、標準 C ライブラリの文字列関数を補完し、PostgreSQL 特有の要件（ログ出力の安全性確保や入力の正規化など）を満たすための共通基盤を提供します。

責務

  • 文字列の末尾判定や、改行コードの除去といった基本的な文字列処理。
  • strtol のラッパーによる int 型への安全な変換。
  • 非 ASCII 文字をエスケープシーケンス（\xXX）に置換し、ログ出力等の安全性を高める機能。
  • 文字列が ASCII のみで構成されているかの検証。

重要なシンボル

  • pg_str_endswith: 文字列が特定の接尾辞で終わるかを確認します。
  • strtoint: strtol を呼び出し、結果を int にキャストして範囲チェックを行います。
  • pg_clean_ascii: 非 ASCII 文字をエスケープした新しい文字列を生成します。
  • pg_strip_crlf: 文字列末尾の \n および \r を削除します。
  • pg_is_ascii: 文字列が ASCII 文字のみか判定します。

制御フローとデータフロー

  • **メモリ管理**: pg_clean_ascii は FRONTEND マクロの定義に応じて malloc または palloc_extended を使い分け、呼び出し元にメモリの所有権を渡します。
  • **変換処理**: strtoint は errno を利用してオーバーフローを検出し、呼び出し元へエラーを伝播させます。
  • **インプレース操作**: pg_strip_crlf は引数として渡された文字列を直接書き換え、新しい長さを返します。

直接依存

  • postgres.h / postgres_fe.h: 環境に応じた基本定義。
  • common/string.h: 本ファイルの関数宣言および型定義。

関連コード

  • src/include/common/string.h: 本ファイルのインターフェース定義。
  • src/common/stringinfo.c: 動的文字列バッファの管理が必要な場合に使用されます。
  • src/common/wchar.c: マルチバイト文字やワイド文字の処理が必要な場合に関連します。

読者向けメモ

  • pg_clean_ascii は、ログ出力の安全性を確保するための限定的なツールです。汎用的な文字列フィルタリングとして使用すべきではありません。
  • FRONTEND マクロの有無により、メモリ確保戦略が切り替わる点に注意してください。
  • 本ファイル内の関数は、PostgreSQL のバックエンドとフロントエンドの両方からリンクされるため、依存関係は最小限に抑えられています。
