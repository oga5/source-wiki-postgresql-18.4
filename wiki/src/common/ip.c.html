src/common/ip.c

概要

PostgreSQL におけるネットワークアドレス解決および名前解決のための抽象化レイヤーです。標準的な getaddrinfo / getnameinfo API をラップし、IPv4/IPv6 に加えて Unix ドメインソケットを透過的に扱えるようにします。

モジュール内での役割

src/common 内のネットワーク関連ユーティリティとして、フロントエンド（psql 等）とバックエンドの両方から利用されます。OS 依存の差異を吸収し、ネットワーク接続の初期化やログ出力時のアドレス表示を統一する役割を担います。

責務

  • **プロトコル非依存の解決**: AF_UNIX、AF_INET、AF_INET6 を区別せず、単一のインターフェースでアドレス情報を取得・解放する。
  • **Unix ドメインソケットのサポート**: 標準 API が対応していない Unix ソケットの addrinfo 構築および名前解決をエミュレートする。
  • **堅牢なエラーハンドリング**: 名前解決失敗時にも、呼び出し元が安全に処理を継続できるようデフォルト値（"???"）を補完する。

重要なシンボル

  • pg_getaddrinfo_all: ホスト名とサービス名から addrinfo 構造体のリストを生成する。
  • pg_freeaddrinfo_all: 生成された addrinfo 構造体を、その生成元（標準 API か Unix 用か）に応じて適切に解放する。
  • pg_getnameinfo_all: ソケットアドレスからホスト名とサービス名を文字列として取得する。

追加の索引済みシンボル

  • getaddrinfo_unix
  • getnameinfo_unix

制御フローとデータフロー

1. **解決要求**: pg_getaddrinfo_all が呼び出されると、hintp->ai_family を確認し、AF_UNIX であれば内部の getaddrinfo_unix へ、それ以外は標準の getaddrinfo へ処理を委譲します。 2. **Unix ソケットの構築**: getaddrinfo_unix は calloc で addrinfo と sockaddr_un を確保し、パス情報を設定します。特に @ で始まる抽象名前空間（Abstract Namespace）のソケットを適切に処理します。 3. **解放**: pg_freeaddrinfo_all は、hint_ai_family を引数として受け取り、Unix ソケット用であれば手動で free し、それ以外は freeaddrinfo を呼び出します。 4. **名前解決**: pg_getnameinfo_all は、失敗時に node や service バッファを "???" で埋めることで、呼び出し側のクラッシュや未定義動作を防ぎます。

読者向けメモ

  • pg_freeaddrinfo_all に hint_ai_family を渡す必要があるのは、addrinfo 構造体自体からは、それが標準 API によって生成されたものか、getaddrinfo_unix によって生成されたものかを判別できないためです。
  • Unix ドメインソケットのパスが @ で始まる場合、それは Linux 固有の抽象名前空間ソケットとして扱われます。この実装は、その慣習に従い sun_path の先頭をヌル文字に置換して処理します。
