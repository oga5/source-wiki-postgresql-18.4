src/pl/tcl/pltcl.c

概要

PostgreSQL の手続き型言語（PL）として Tcl を統合するための中心的な実装ファイルです。SQL 関数、トリガー、およびイベントトリガーの実行ハンドラを提供し、PostgreSQL の SPI（Server Programming Interface）と Tcl インタプリタ間のブリッジとして機能します。

モジュール内での役割

src/pl/tcl モジュールの核であり、Tcl インタプリタのライフサイクル管理、SQL 呼び出しのディスパッチ、およびデータ型の相互変換を担います。PostgreSQL のバックエンドプロセス内で Tcl を安全かつ効率的に実行するための環境構築を行います。

責務

  • **インタプリタ管理**: ユーザーごとの Tcl インタプリタの生成・キャッシュ。
  • **関数コンパイル**: SQL 関数を Tcl プロシージャへ変換し、ハッシュテーブルで管理。
  • **SPI 統合**: Tcl スクリプトから SQL を実行するためのコマンド（spi_exec, spi_prepare 等）の提供。
  • **データ変換**: PostgreSQL のタプル・データ型と Tcl オブジェクト間の変換（UTF-8 エンコーディングの考慮を含む）。
  • **エラーハンドリング**: Tcl のエラーを PostgreSQL の ereport へ変換し、SQLSTATE を伝播。

重要なシンボル

インタプリタとキャッシュ管理

  • pltcl_interp_desc: ユーザーごとの Tcl インタプリタ状態とクエリキャッシュを保持する構造体。
  • pltcl_proc_desc: コンパイル済み関数のキャッシュ情報。メモリコンテキスト管理と参照カウントを含む。
  • pltcl_proc_htab: 関数 OID とユーザー ID をキーとする関数キャッシュハッシュテーブル。

実行ハンドラ

  • pltcl_call_handler: SQL 関数呼び出しのエントリポイント。
  • pltclu_call_handler: 非信頼（Untrusted）Tcl 関数用のエントリポイント。
  • pltcl_func_handler, pltcl_trigger_handler: それぞれ関数とトリガーの実行ロジック。

SPI 統合コマンド

  • pltcl_SPI_execute, pltcl_SPI_prepare: Tcl から SQL を実行・準備するためのコマンド実装。
  • pltcl_elog: Tcl から PostgreSQL の elog/ereport を呼び出すためのコマンド。

制御フローとデータフロー

1. **初期化**: _PG_init で Tcl の Notifier サブシステムをオーバーライドし、PostgreSQL のバックエンドと競合しないように設定。 2. **呼び出し**: pltcl_call_handler が呼び出されると、pltcl_fetch_interp で現在のユーザーに対応するインタプリタを取得。 3. **コンパイル**: compile_pltcl_function が関数定義を検索し、未キャッシュであれば Tcl プロシージャとして定義。 4. **実行**: Tcl_EvalObjEx で Tcl コードを実行。SPI 経由の SQL 実行時は pltcl_subtrans_begin/commit/abort を使用してサブトランザクションを管理。 5. **変換**: pltcl_build_tuple_argument / pltcl_build_tuple_result を介して、PostgreSQL のタプルと Tcl のリスト形式間でデータを変換。

読者向けメモ

  • **メモリ管理**: pltcl_proc_desc は fn_cxt（メモリコンテキスト）で管理され、参照カウントが 0 になった時点で解放されます。
  • **UTF-8 変換**: UTF_BEGIN/UTF_END マクロは、PostgreSQL の内部エンコーディングと Tcl の UTF-8 間での変換時に発生する一時的なメモリリークを防ぐためのラッパーです。
  • **サブトランザクション**: SPI 操作は常にサブトランザクション内で実行され、エラー発生時に PostgreSQL のトランザクション状態を保護します。
  • **制限事項**: Tcl の Notifier サブシステムを無効化しているため、Tcl のイベントループ機能は PostgreSQL 内では使用できません。
  • **未実装/浅い部分**: pltcl_SPI_prepare で作成されたプランは、関数が再コンパイルされる際にメモリリークする可能性があります。
