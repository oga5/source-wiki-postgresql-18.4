src/pl/plperl/plperl_system.h

概要

plperl_system.h は、PostgreSQL の PL/Perl モジュールにおいて、Perl のシステムヘッダーを安全に読み込むためのラッパーヘッダーです。PostgreSQL と Perl のヘッダー間で発生する名前空間の衝突や、コンパイラの警告を抑制することを目的としています。

モジュール内での役割

PL/Perl の各ソースファイルが Perl API を利用する際の「ゲートウェイ」として機能します。PostgreSQL 固有の宣言を含まず、純粋に Perl 環境との整合性を保つための設定のみを記述することで、依存関係の複雑化を防いでいます。

責務

  • **名前空間の保護**: printf 系関数や _ マクロなど、PostgreSQL と Perl の両方で定義されるシンボルの衝突を回避する。
  • **コンパイラ警告の抑制**: pragma GCC system_header を使用し、Perl ヘッダー由来の不要なコンパイラ警告を無効化する。
  • **プラットフォーム差異の吸収**: Windows (MSVC/MinGW) 環境における inline や isnan の定義差異を調整する。
  • **API 互換性の確保**: ppport.h と連携し、Perl のバージョン間差異を吸収するマクロ定義を提供する。

重要なシンボル

  • PERL_NO_GET_CONTEXT: Perl のスレッドコンテキストを明示的に渡すモードを有効化し、XSUB.h への依存を制御します。
  • HAS_BOOL: Perl が bool 型を再定義するのを防ぎ、PostgreSQL 側の定義を優先させます。
  • pg_attribute_unused(): PERL_UNUSED_DECL を再定義し、GCC 等での警告を回避します。
  • AV_SIZE_MAX: Perl のバージョンに応じて配列インデックスの最大値を適切に定義します。

制御フローとデータフロー

1. **初期化**: コンパイラ警告の抑制設定と、Windows 用の WIN32IO_IS_STDIO 定義。 2. **衝突回避**: Perl ヘッダーが上書きする可能性のある printf 系関数や _ マクロを一旦 undef します。 3. **Perl API 読み込み**: EXTERN.h および perl.h をインクルードします。 4. **再定義**: PostgreSQL 側の pg_printf 等を再定義し、_ マクロを dgettext に割り当て直します。 5. **互換性補完**: ppport.h を読み込み、不足している HeUTF8 や GvCV_set などのマクロを補完します。

直接依存

  • EXTERN.h, perl.h: Perl のコア API。
  • XSUB.h: Perl の拡張モジュール用ヘッダー（PG_NEED_PERL_XSUB_H 定義時のみ）。
  • ppport.h: Perl のバージョン間互換性レイヤー。

関連コード

  • plperl.c: 本ヘッダーをインクルードして Perl API を利用するメイン実装。
  • src/include/port.h: PostgreSQL 側の printf 系関数の定義元。

読者向けメモ

  • 本ファイルは、PostgreSQL のビルドシステムと Perl のヘッダーの間の「緩衝地帯」です。
  • printf 系関数を pg_ プレフィックス付きの関数に差し替える処理は、PostgreSQL の移植性レイヤーと Perl の標準ライブラリ利用を共存させるための重要なハックです。
  • 新しい Perl API を追加する際は、まず ppport.h で対応可能か確認し、それでも不足する場合のみ本ファイルで定義を追加してください。
