src/pl/plperl/plperl.h

概要

plperl.h は、PostgreSQL の PL/Perl 言語ハンドラにおける共通ヘッダーファイルです。Perl と PostgreSQL 間のデータ型変換、エンコーディング処理、および SPI (Server Programming Interface) 呼び出しのためのインターフェースを定義します。

モジュール内での役割

PL/Perl モジュールにおける「ブリッジ」の役割を果たします。plperl.c（メインロジック）と、Perl の XS (eXternal Subroutine) インターフェースを介して呼び出される各ファイルの間で、共通の型変換ユーティリティと関数宣言を提供します。

責務

  • **エンコーディング変換**: データベースの文字エンコーディングと Perl の内部 UTF-8 表現の相互変換。
  • **型変換の抽象化**: PostgreSQL の文字列と Perl の SV (Scalar Value) 間の安全な変換。
  • **エラーハンドリング**: Perl の croak を PostgreSQL のエラー報告メカニズムと整合させるためのラッパー提供。
  • **SPI インターフェースの公開**: Perl 側から PostgreSQL の SPI 関数を呼び出すための関数プロトタイプ定義。

重要なシンボル

  • sv2cstr(SV *sv): Perl の SV を PostgreSQL のメモリ管理下にある char * (データベースエンコーディング) に変換します。
  • cstr2sv(const char *str): PostgreSQL の文字列を Perl が扱える UTF-8 フラグ付きの SV に変換します。
  • croak_cstr(const char *str): データベースエンコーディングの文字列を用いて Perl の croak を呼び出し、安全に例外を発生させます。
  • utf_u2e / utf_e2u: 内部的なエンコーディング変換用インライン関数。

追加の索引済みシンボル

  • croak_cstr
  • cstr2sv
  • sv2cstr

制御フローとデータフロー

本ファイルは主にインライン関数と宣言を提供します。 1. **データ変換フロー**: Perl からデータを受け取る際は sv2cstr を経由し、PostgreSQL のメモリコンテキストで文字列を確保します。逆に PostgreSQL から Perl へ渡す際は cstr2sv を経由して UTF-8 化します。 2. **エラーフロー**: Perl 側でエラーが発生した場合、croak_cstr を通じて PostgreSQL のエラーハンドラへ制御を戻すための準備を行います。

直接依存

  • mb/pg_wchar.h: PostgreSQL のマルチバイト文字処理関数（pg_any_to_server 等）を利用するために必須。
  • plperl_system.h: Perl のヘッダーファイル群をラップし、コンパイル時の名前空間汚染を防ぐためのラッパー。

関連コード

  • plperl.c: 本ヘッダーで宣言された関数の実装および、PL/Perl のメイン制御ロジックが含まれます。
  • plperl_opmask.h: Perl のオペレータ制限に関する定義。
  • perlchunks.h: Perl 側で実行される初期化コードの文字列リテラル。

読者向けメモ

  • **インクルード順序**: postgres.h およびシステムヘッダーの後にインクルードする必要があります。これは Perl のヘッダーがシステム定義と衝突するのを防ぐためです。
  • **メモリ管理**: sv2cstr 等の変換関数は palloc を使用してメモリを確保します。呼び出し側は、PostgreSQL のメモリコンテキストのライフサイクルを意識する必要があります。
  • **SQL_ASCII**: データベースエンコーディングが SQL_ASCII の場合、エンコーディング変換をスキップする最適化が行われています。
