src/pl/plperl/ppport.h

概要

ppport.h は、Perl の移植性レイヤー（Perl/Pollution/Portability）を提供するヘッダーファイルです。Devel::PPPort モジュールによって自動生成されており、異なる Perl バージョン間での API の差異を吸収し、古い Perl 環境でも新しい Perl API を利用可能にするためのラッパーやマクロ定義を提供します。

モジュール内での役割

src/pl/plperl モジュールにおいて、PostgreSQL のビルド環境が想定する Perl バージョンと、実際にインストールされている Perl インタプリタのバージョン間のギャップを埋める役割を担います。これにより、plperl.c などのコアロジックが特定の Perl バージョンに依存することなく、安定してコンパイル・実行されることを保証します。

責務

  • **API 互換性の提供**: Perl のバージョンアップに伴い追加・変更された API を、古いバージョンでも利用できるようにマクロで再定義します。
  • **プラットフォーム差異の吸収**: Perl のビルド環境やプラットフォーム固有の差異を隠蔽します。
  • **非推奨 API の回避**: 古い API の使用を検出し、より安全な代替手段への移行を支援します。
  • **コンテキスト管理**: スレッドセーフな Perl インタプリタ環境（pTHX 等）の受け渡しを簡略化するマクロを提供します。

重要なシンボル

バージョン判定マクロ

  • PERL_VERSION_EQ, PERL_VERSION_LT, PERL_VERSION_GT 等: コンパイル時に Perl のバージョンを比較し、条件付きコンパイルを可能にします。

スレッドコンテキストマクロ

  • pTHX, aTHX, dTHX: Perl インタプリタのコンテキスト（PerlInterpreter*）を関数間で受け渡すための標準的なマクロ群。

互換性ラッパー

  • newCONSTSUB, sv_catpvf_mg, croak_xs_usage: 古い Perl バージョンには存在しない、あるいは挙動が異なる関数を、マクロや静的関数として再実装します。

ユーティリティ

  • isALNUM, isDIGIT, isSPACE: ロケールや UTF-8 処理を考慮した文字種判定マクロ。
  • SvPV_nolen, SvPV_force: Perl スカラー値から文字列バッファを取得する際の安全なラッパー。

追加の索引済みシンボル

  • ARGSproto
  • DPPP_
  • DPPP_dopoptosub_at
  • NV
  • pTHX
  • pTHX_

制御フローとデータフロー

  • **初期化フロー**: ppport.h がインクルードされると、NEED_xxx マクロの定義状況に応じて必要な互換性関数が静的関数として定義されます。
  • **データ変換フロー**: plperl モジュールが PostgreSQL の Datum を Perl の SV* に変換する際、ppport.h が提供する newSVpvn や sv_setpv などのラッパーを経由することで、Perl バージョンを意識せずにデータ操作が可能になります。

直接依存

  • locale.h: ロケール依存の文字処理を行うために使用されます。
  • patchlevel.h: Perl のバージョン情報を取得するために内部的に参照されます。
  • ppport.h

関連コード

  • plperl_system.h: Perl のシステムヘッダーを安全にインクルードするためのラッパー。
  • plperl.c: 本ヘッダーを利用して Perl インタプリタと通信するメインロジック。

読者向けメモ

  • **自動生成ファイル**: このファイルは Devel::PPPort によって生成されています。**直接編集してはいけません**。変更が必要な場合は、生成元のスクリプトや設定を修正する必要があります。
  • **メモリ管理**: ppport.h が提供するラッパー関数（newCONSTSUB 等）は、Perl のメモリ管理ルール（SV の参照カウント等）に従う必要があります。
  • **浅い領域**: 本ファイルは非常に広範な API をカバーしていますが、特定のマイナーな Perl 拡張機能や、非常に古い Perl バージョンでの挙動については、ドキュメント（perldoc ppport.h）を参照してください。
  • **パフォーマンス**: 多くの関数がマクロとして実装されているため、関数呼び出しのオーバーヘッドは最小限に抑えられていますが、複雑な互換性処理が必要な場合は静的関数が呼び出されます。
