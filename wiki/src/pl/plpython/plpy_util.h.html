src/pl/plpython/plpy_util.h

概要

plpy_util.h は、PL/Python モジュール全体で使用される汎用的なユーティリティ関数を定義するヘッダーファイルです。主に Python の文字列オブジェクト（Unicode）と PostgreSQL の内部エンコーディング間の相互変換を抽象化する役割を担います。

モジュール内での役割

PL/Python は PostgreSQL のサーバーエンコーディングと Python の Unicode 文字列の間で頻繁に変換を行う必要があります。本ファイルは、その変換ロジックを各モジュールから共通利用可能なインターフェースとして提供し、エンコーディング変換の複雑さを隠蔽します。

責務

  • Python の PyObject (Unicode) から PostgreSQL のサーバーエンコーディングに準拠したバイト列や C 文字列への変換。
  • PostgreSQL のサーバーエンコーディングから Python の PyObject (Unicode) への変換。
  • エンコーディング変換に伴うメモリ管理（palloc/pfree）の適切な処理。

重要なシンボル

  • PLyUnicode_Bytes: Python Unicode オブジェクトをサーバーエンコーディングのバイト列に変換します。
  • PLyUnicode_AsString: Python Unicode オブジェクトをサーバーエンコーディングの C 文字列（palloc 済み）として取得します。
  • PLyUnicode_FromString / FromStringAndSize: サーバーエンコーディングの C 文字列から Python Unicode オブジェクトを生成します。

制御フローとデータフロー

1. **Python → PG**: PLyUnicode_Bytes が Python オブジェクトを一度 UTF-8 に変換し、必要に応じて pg_any_to_server を経由してサーバーエンコーディングへ変換します。 2. **PG → Python**: PLyUnicode_FromString が pg_server_to_any を使用してサーバーエンコーディングから UTF-8 へ変換し、最終的に Python の Unicode オブジェクトを構築します。 3. **メモリ管理**: palloc されたメモリは、変換完了後に呼び出し元または関数内で適切に解放されます。

直接依存

  • plpython.h: Python C API と PostgreSQL のヘッダーを統合する基盤。

関連コード

  • src/pl/plpython/plpy_util.c: 上記関数の実装が含まれており、pg_any_to_server や pg_server_to_any といった PostgreSQL のエンコーディング変換関数を利用しています。

読者向けメモ

  • 本ファイルで提供される関数は、Python と PostgreSQL の間で文字列をやり取りする際の「標準的な変換経路」です。
  • Python がサポートしていないエンコーディング（EUC_TW 等）を PostgreSQL が扱う場合があるため、UTF-8 を中間形式として利用する設計になっています。
  • PLyUnicode_AsString を使用する場合、戻り値は palloc されているため、呼び出し側で pfree する必要がある点に注意してください。
