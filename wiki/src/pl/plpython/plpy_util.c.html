src/pl/plpython/plpy_util.c

概要

PostgreSQLのサーバーエンコーディングとPythonのUnicodeオブジェクト間の相互変換を行うためのユーティリティ関数群を提供します。

モジュール内での役割

pl/plpython サブシステムにおいて、データベースとPythonインタープリタ間で文字列データを安全に受け渡すためのブリッジ層として機能します。

責務

  • **エンコーディング変換**: PostgreSQLのサーバーエンコーディングとUTF-8（Python側の標準）間の変換を仲介します。
  • **メモリ管理**: Pythonオブジェクトの参照カウント管理と、PostgreSQLのメモリコンテキスト（palloc/pfree）間の整合性を維持します。
  • **エラーハンドリング**: 変換失敗時の PLy_elog を介したPostgreSQLエラー報告。

重要なシンボル

  • PLyUnicode_Bytes: Python Unicodeオブジェクトをサーバーエンコーディングのバイト列に変換します。
  • PLyUnicode_AsString: Python UnicodeオブジェクトをサーバーエンコーディングのC文字列（palloc済み）に変換します。
  • PLyUnicode_FromString / FromStringAndSize: サーバーエンコーディングのC文字列をPython Unicodeオブジェクトに変換します。

追加の索引済みシンボル

  • PLyUnicode_FromStringAndSize

制御フローとデータフロー

1. **Python → DB**: PLyUnicode_Bytes が呼び出されると、まず PyUnicode_AsUTF8String でUTF-8に変換されます。その後、pg_any_to_server を用いてサーバーエンコーディングへ変換し、必要に応じてメモリを解放します。 2. **DB → Python**: PLyUnicode_FromString が呼び出されると、pg_server_to_any でUTF-8に変換された後、Pythonの PyUnicode_FromString を通じてUnicodeオブジェクトが生成されます。 3. **例外処理**: PG_TRY/PG_CATCH ブロックにより、変換中のエラー発生時にもPython側の参照カウント（Py_DECREF）が適切に処理されるよう設計されています。

直接依存

  • postgres.h: PostgreSQLの基本型およびメモリ管理関数。
  • mb/pg_wchar.h: マルチバイト文字変換関数（pg_any_to_server 等）。
  • plpy_elog.h: Python層でのエラー報告用インターフェース。
  • plpy_util.h: 本ファイルの関数宣言。

関連コード

  • src/pl/plpython/plpy_typeio.c: 型変換のロジックにおいて、本ファイルのユーティリティが頻繁に利用されます。
  • src/pl/plpython/plpython.h: Python C APIとの統合定義。

読者向けメモ

  • Pythonは内部的にUnicodeを扱いますが、PostgreSQLはデータベースごとに異なるエンコーディングを持つため、本ファイルは「UTF-8を中間言語」として利用する設計になっています。
  • pg_any_to_server がメモリを動的に割り当てた場合、必ず pfree を呼び出す必要がある点に注意してください。
  • Pythonオブジェクトの参照カウント（Py_DECREF）の管理は、PostgreSQLの例外発生時にも漏れないよう慎重に記述されています。
