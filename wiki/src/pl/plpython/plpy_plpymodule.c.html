src/pl/plpython/plpy_plpymodule.c

概要

本ファイルは、PL/Python 拡張における Python 側の plpy モジュールの定義と初期化を担います。PostgreSQL の内部機能（SPI やログ出力など）を Python スクリプトから利用可能にするためのインターフェースを提供します。

モジュール内での役割

plpy モジュールは、PL/Python のユーザーが SQL を実行したり、データベースのログレベルを制御したりするための「窓口」です。本ファイルは、Python インタプリタがロードされた際に、これらの関数や例外クラスを Python の名前空間に登録する役割を果たします。

責務

  • plpy モジュールおよび plpy.spiexceptions モジュールの定義と初期化。
  • Python から呼び出される plpy.debug から plpy.fatal までのログ出力関数の実装。
  • SQL のエスケープ関数（quote_literal 等）の提供。
  • PostgreSQL のエラーコード（SQLSTATE）を Python の例外クラスにマッピングし、動的に生成する。

重要なシンボル

  • PyInit_plpy: Python モジュールロード時に呼び出されるエントリポイント。
  • PLy_methods: plpy モジュールが公開するメソッド（execute, prepare, cursor 等）の定義テーブル。
  • PLy_spi_exceptions: SQLSTATE と Python 例外オブジェクトを紐付けるハッシュテーブル。
  • PLy_output: Python からのログ出力を PostgreSQL の ereport に変換する共通関数。

制御フローとデータフロー

1. **初期化**: PyInit_plpy が呼ばれ、PLy_module を作成。続いて PLy_add_exceptions が実行されます。 2. **例外生成**: PLy_generate_spi_exceptions が spiexceptions.h を読み込み、SQLSTATE に対応する Python 例外クラスを動的に生成して plpy.spiexceptions に登録します。 3. **関数呼び出し**: Python から plpy.info() 等が呼ばれると、PLy_output を経由して ereport が実行されます。 4. **エラーハンドリング**: PG_TRY/PG_CATCH ブロックにより、PostgreSQL 側のエラーをキャッチし、Python の例外として再送出します。

読者向けメモ

  • 本ファイルは Python の C API を多用しています。Py_INCREF や Py_DECREF による参照カウント管理が随所で行われているため、メモリリークや二重解放には注意が必要です。
  • PLy_output 関数は、Python の可変長引数とキーワード引数を解析し、PostgreSQL の ereport に渡すための複雑な変換を行っています。ここが Python と PostgreSQL の型変換の境界線の一つです。
  • spiexceptions.h はビルド時に生成されるファイルであるため、手動で編集しても反映されません。
