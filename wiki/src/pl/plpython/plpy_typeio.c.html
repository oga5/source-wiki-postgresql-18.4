src/pl/plpython/plpy_typeio.c

概要

plpy_typeio.c は、PostgreSQL のデータ型（Datum）と Python オブジェクト（PyObject）間の双方向変換を担う中核モジュールです。SQL の型システムと Python の動的型システムを橋渡しするため、再帰的な型構造（配列、複合型、ドメイン）の解析と変換ロジックを実装しています。

モジュール内での役割

plpy_exec.c 等の実行エンジンが SQL 関数を呼び出す際、引数や戻り値のマーシャリングを本モジュールに委譲します。型変換のメタデータ（PLyDatumToOb, PLyObToDatum）を保持し、実行時の型チェックや変換関数のディスパッチを行います。

責務

  • **入力変換（SQL → Python）**: Datum を Python の基本型、リスト、辞書に変換。
  • **出力変換（Python → SQL）**: Python オブジェクトを Datum に変換し、必要に応じて record_in や型固有の入力関数を呼び出す。
  • **メタデータ管理**: 型変換に必要な FmgrInfo や TupleDesc のキャッシュと初期化。
  • **再帰的構造の処理**: 多次元配列やネストされた複合型の再帰的な走査。
  • **エラーハンドリング**: 変換失敗時の PostgreSQL エラー（ereport）への変換と、Python 例外のクリーンアップ。

重要なシンボル

変換エントリポイント

  • PLy_input_convert: SQL から Python への変換を開始するトップレベル関数。
  • PLy_output_convert: Python から SQL への変換を開始するトップレベル関数。
  • PLy_input_from_tuple: HeapTuple を Python の dict に変換。

初期化・セットアップ

  • PLy_input_setup_func / PLy_output_setup_func: 型 OID に基づき、変換に必要な構造体（PLyDatumToOb 等）を再帰的に構築。
  • PLy_input_setup_tuple / PLy_output_setup_tuple: 複合型（RECORD 等）の各カラムに対する変換情報を初期化。

変換ロジック（ファミリー）

  • **スカラー**: PLyBool_FromBool, PLyLong_FromInt64, PLyUnicode_FromScalar 等。
  • **配列**: PLyList_FromArray（SQL→Py）, PLySequence_ToArray（Py→SQL）。
  • **複合型**: PLyDict_FromComposite（SQL→Py）, PLyObject_ToComposite（Py→SQL）。

制御フローとデータフロー

1. **セットアップ**: 実行開始時に PLy_*_setup_func が呼ばれ、型 OID に応じた変換関数ポインタとメタデータが PLyDatumToOb 構造体に格納されます。 2. **変換実行**:

  • PLy_input_convert は MemoryContextReset を行い、変換中のメモリリークを防ぎます。
  • 変換関数は再帰的に呼び出されます（例: PLyList_FromArray_recurse）。
  • Python オブジェクトの生成には PyObject_* API を使用し、PostgreSQL 側へ戻す際は InputFunctionCall 等で Datum を生成します。

    • 3. **エラーパス**: PG_TRY / PG_CATCH ブロックにより、変換中のエラー発生時に Python の参照カウント（Py_DECREF）を適切に解放し、PostgreSQL のエラー状態へ復帰します。

読者向けメモ

  • **メモリ管理**: 変換処理は PLyExecutionContext に紐付いたスクラッチメモリコンテキストで行われます。MemoryContextReset が多用されるため、変換関数内で永続的なメモリ確保を行わないよう注意が必要です。
  • **再帰の深さ**: check_stack_depth() が再帰的なセットアップ関数内で呼ばれており、スタックオーバーフローを防止しています。
  • **RECORD 型**: RECORDOID は実行時に TupleDesc が決定するため、他の型よりも複雑なライフサイクル管理（PinTupleDesc / ReleaseTupleDesc）が必要です。
  • **省略された詳細**: 本ファイルは型変換の「ロジック」に集中しており、Python インタプリタ自体の初期化や SPI の詳細については plpy_main.c や plpy_spi.c を参照してください。
