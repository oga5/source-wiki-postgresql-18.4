src/pl/plpython/plpy_exec.h

概要

plpy_exec.h は、PostgreSQL の PL/Python において、Python 関数およびトリガーの実行を制御するためのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

PL/Python モジュールの中核である「実行エンジン」の入り口を提供します。plpy_exec.c と対になっており、PostgreSQL の関数呼び出しインターフェース（FunctionCallInfo）と Python 側の実行ロジックを橋渡しします。

責務

  • PostgreSQL から呼び出される関数実行のメインエントリポイントの宣言。
  • トリガー実行時のエントリポイントの宣言。
  • 実行に必要な PLyProcedure 構造体への依存関係の管理。

重要なシンボル

  • PLy_exec_function: PostgreSQL の関数として定義された Python コードを実行するためのメイン関数。
  • PLy_exec_trigger: トリガーとして定義された Python コードを実行するためのメイン関数。
  • PLyProcedure: 実行対象となる Python プロシージャのメタデータや状態を保持する構造体。

制御フローとデータフロー

本ファイル自体は宣言のみですが、対となる plpy_exec.c では以下のフローが制御されます： 1. **初期化**: PLy_global_args_push により、再帰呼び出しに備えて引数をスタックへ退避。 2. **実行**: PLy_procedure_call を通じて Python インタプリタへ制御を移し、コードを実行。 3. **結果変換**: Python の戻り値を PostgreSQL の Datum または HeapTuple に変換。 4. **後処理**: PG_TRY/PG_CATCH ブロックにより、エラー発生時のサブトランザクションのロールバックやリソース解放を保証。

直接依存

  • plpy_procedure.h: 実行対象のプロシージャ定義（PLyProcedure）を参照するために必須。

関連コード

  • plpy_exec.c: 本ヘッダーで宣言された関数の実装。
  • plpy_procedure.c: プロシージャのコンパイルやキャッシュ管理を担当。
  • plpy_spi.c: Python から PostgreSQL のデータにアクセスするための SPI (Server Programming Interface) 呼び出しを処理。

読者向けメモ

  • 本ファイルは非常にシンプルですが、PLy_exec_function が SRF (Set-Returning Function) をサポートしている点に注意してください。SETOF を返す関数の場合、PLySRFState を用いてイテレータの状態を保持し、複数回の呼び出しを跨いで実行を継続する仕組みになっています。
  • 実行中のエラーハンドリングは PostgreSQL の PG_TRY マクロに依存しているため、Python 側の例外と PostgreSQL のエラー処理の境界を理解する上で plpy_elog.h も併せて参照することをお勧めします。
