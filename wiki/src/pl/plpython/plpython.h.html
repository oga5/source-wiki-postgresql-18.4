src/pl/plpython/plpython.h

概要

plpython.h は、PostgreSQL の手続き型言語である PL/Python モジュールの中心的なヘッダーファイルです。PL/Python を構成する各サブモジュール（plpy_*.h）の共通基盤として機能し、Python インタプリタとの統合に必要な設定を定義します。

モジュール内での役割

本ファイルは、PL/Python サブシステム全体の「共通設定層」です。すべてのサブモジュールがこのファイルをインクルードすることで、Python のヘッダーファイルや PostgreSQL 固有の翻訳ドメイン設定を統一的に適用します。

責務

  • **Python 環境の統合**: plpython_system.h を介して Python ヘッダーを安全に読み込みます。
  • **ビルド設定の制御**: Py_LIMITED_API を定義し、Python の安定した ABI を利用するための制約を管理します。
  • **翻訳ドメインの定義**: TEXTDOMAIN を plpython に設定し、エラーメッセージ等の国際化（i18n）をサポートします。
  • **インクルード順序の強制**: postgres.h が先行してインクルードされていることを検証し、ビルド時の依存関係ミスを防ぎます。

重要なシンボル

  • PLPYTHON_H: 多重インクルードを防ぐためのガードマクロ。
  • Py_LIMITED_API: Python の限定 API を有効化するためのマクロ（MSVC を除く）。
  • TEXTDOMAIN: PL/Python モジュール専用のメッセージ翻訳ドメイン。

制御フローとデータフロー

本ファイルは実行コードを含まないヘッダーファイルであるため、直接的なフローは存在しません。しかし、ビルドプロセスにおいて以下の制約を強制します。 1. **検証**: postgres.h が未定義の場合、コンパイルエラーを発生させます。 2. **包含**: plpython_system.h を通じて Python のシステムヘッダーを読み込みます。 3. **適用**: 各サブモジュールが本ファイルをインクルードすることで、上記の設定が各コンパイル単位に伝播します。

直接依存

  • plpython_system.h: Python ヘッダーのラッパー。

関連コード

  • plpy_*.h: 本ファイルをインクルードするすべてのサブモジュールヘッダー。
  • plpy_*.c: 本ファイルの設定を間接的に利用する実装ファイル。

読者向けメモ

  • **インクルードの原則**: このファイルは各サブモジュールのヘッダーファイルからインクルードされることを前提としています。そのため、.c ファイルから直接インクルードする必要はありません。
  • **Python ヘッダーの扱い**: Python のヘッダーを直接インクルードしようとするとエラーが発生する仕組みになっています。必ず plpython.h を経由してください。
  • **MSVC の例外**: Py_LIMITED_API は MSVC 環境ではビルドエラーを避けるために無効化されています。移植性を考慮する際は注意してください。
