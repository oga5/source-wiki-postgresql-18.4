src/pl/plpython/plpy_elog.h

概要

本ヘッダーファイルは、PL/Python におけるエラーおよび例外処理のインターフェースを定義します。Python 側で発生した例外を PostgreSQL のエラー報告システム（ereport）へ適切に変換・伝播させるための仕組みを提供します。

モジュール内での役割

src/pl/plpython モジュールにおいて、Python の例外状態を PostgreSQL のエラーレベル（ERROR, FATAL 等）と結びつけるブリッジの役割を果たします。

責務

  • Python の例外オブジェクト（PLy_exc_error 等）のグローバルな管理。
  • Python の例外情報を PostgreSQL の ereport に渡すための変換ロジックの提供。
  • コンパイル時の最適化を考慮したエラー報告マクロ（PLy_elog）の提供。

重要なシンボル

  • PLy_elog(elevel, ...): Python の例外を PostgreSQL のエラーとして報告する主要なマクロ。ereport をラップし、エラー発生後の制御フローを適切に終了させます。
  • PLy_exc_error, PLy_exc_fatal, PLy_exc_spi_error: PL/Python で使用される標準的な Python 例外クラスを保持するグローバル変数。
  • PLy_elog_impl(...): マクロから呼び出される実体関数。例外のスタックトレースや詳細情報を収集し、ereport を実行します。
  • PLy_exception_set(...): Python 側で発生させる例外をセットするための関数群。

制御フローとデータフロー

1. **例外発生**: Python コード実行中に例外が発生すると、PyErr_Fetch で例外情報が取得されます。 2. **変換**: PLy_elog_impl 内で PyErr_NormalizeException を経て、PostgreSQL 側で理解可能な形式（SQLSTATE や詳細メッセージ）に変換されます。 3. **報告**: 収集された情報が ereport に渡され、PostgreSQL のエラーログまたはクライアントへの通知として出力されます。 4. **終了**: pg_unreachable() を用いて、ERROR レベル以上のエラー発生時に以降のコードが実行されないことをコンパイラに伝えます。

直接依存

  • plpython.h: Python C API との連携および PostgreSQL の基本定義。

関連コード

  • src/pl/plpython/plpy_elog.c: 本ヘッダーで宣言された関数の実装。例外のスタックトレース解析や属性抽出の詳細が含まれます。

読者向けメモ

  • PLy_elog マクロは、elevel が ERROR 以上の場合に pg_unreachable() を呼び出します。これにより、エラー発生後のコードパスが到達不能であることをコンパイラに明示し、警告を抑制しています。
  • Python の例外を PostgreSQL のエラーに変換する際は、PLy_get_spi_error_data 等の内部関数が、Python オブジェクトから SQLSTATE やエラー詳細を抽出する役割を担っています。
