src/pl/plpython/plpython_system.h

概要

plpython_system.h は、PostgreSQL の PL/Python モジュールにおいて、Python のシステムヘッダー（Python.h）を安全かつ一貫してインクルードするための専用ヘッダーファイルです。

モジュール内での役割

PL/Python モジュール全体における「Python インターフェースのゲートウェイ」として機能します。PostgreSQL 側のコードと Python 側のコードの間で発生しうる、マクロ定義の競合やコンパイラの警告を抑制する役割を担います。

責務

  • **警告の抑制**: pragma GCC system_header を使用し、Python ヘッダー内で発生するコンパイラ警告を無効化します。
  • **マクロ競合の回避**: HAVE_SNPRINTF の定義や、Windows 環境（MSVC）における _DEBUG マクロと errcode 関数の衝突を解決します。
  • **インクルードの抽象化**: PL/Python の各ソースファイルが個別に Python.h を直接インクルードする際の複雑な前処理を隠蔽します。

重要なシンボル

  • PLPYTHON_SYSTEM_H: 多重インクルードを防ぐためのガードマクロ。
  • HAVE_SNPRINTF: Python ヘッダーによる再定義警告を防ぐために強制的に定義されるマクロ。
  • errcode: MSVC 環境下で PostgreSQL の関数名と衝突するのを防ぐため、一時的にリネームされるシンボル。

制御フローとデータフロー

このファイルは実行ロジックを持ちません。プリプロセッサ段階で以下の処理を行います。 1. コンパイラが GCC であれば、以降のヘッダーをシステムヘッダーとして扱い警告を抑制。 2. Python の snprintf 定義との競合を回避。 3. MSVC 環境の場合、_DEBUG の一時的な無効化や errcode のリネームを行い、安全に Python.h を読み込む。 4. 読み込み完了後、必要に応じて元の状態へ復元。

直接依存

  • Python.h: Python のコア API を提供する標準ヘッダー。

関連コード

  • plpython.h: 本ヘッダーをインクルードし、PL/Python の主要なデータ構造や関数定義を保持する上位ヘッダー。
  • plpy_main.c: PL/Python の初期化処理を行うファイル。本ヘッダーを介して Python 環境をロードします。

読者向けメモ

  • **Postgres コードの禁止**: このファイルには PostgreSQL 固有の宣言を含めてはいけません。純粋に Python 環境との橋渡しのみに専念させる設計です。
  • **インクルード順序**: PL/Python のソースファイルを作成・修正する際は、必ず Python.h を直接インクルードせず、この plpython_system.h を経由してください。これにより、プラットフォーム固有のビルド問題を回避できます。
