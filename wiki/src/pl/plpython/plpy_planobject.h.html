src/pl/plpython/plpy_planobject.h

概要

本ファイルは、PL/Python におけるプリペアドステートメント（実行計画）を Python オブジェクトとして表現するためのデータ構造とインターフェースを定義しています。

モジュール内での役割

src/pl/plpython モジュールにおいて、SQL 実行計画を Python 側から操作可能にするためのブリッジ層です。PostgreSQL の SPI（Server Programming Interface）で作成された実行計画を、Python の PLyPlan クラスとしてラップします。

責務

  • **データ構造の定義**: PLyPlanObject 構造体を通じて、SPI 計画、引数の型情報、メモリコンテキストを保持する。
  • **ライフサイクル管理**: Python オブジェクトとしての初期化、型定義の登録、および型チェック機能の提供。

重要なシンボル

  • PLyPlanObject: 実行計画を保持する主要な構造体。SPIPlanPtr を含み、Python の PyObject として振る舞います。
  • PLy_plan_init_type(): Python インタプリタに対して PLyPlan 型を登録する初期化関数。
  • PLy_plan_new(): 新しい PLyPlanObject インスタンスを生成するファクトリ関数。
  • is_PLyPlanObject(): 渡された PyObject が PLyPlan 型であるかを判定する型チェック関数。

制御フローとデータフロー

本ヘッダーは定義のみを提供します。実際のフローは plpy_planobject.c に実装されています。 1. **初期化**: PLy_plan_init_type が呼ばれ、Python の型システムに PLyPlan が組み込まれます。 2. **生成**: PLy_plan_new によりメモリが確保され、各フィールドが初期化されます。 3. **破棄**: PLy_plan_dealloc（実装側）が呼ばれると、保持している SPIPlanPtr と MemoryContext が解放されます。

直接依存

  • executor/spi.h: PostgreSQL の SPI 関連機能（SPIPlanPtr 等）を利用するため。
  • plpy_typeio.h: 引数の型変換（PLyObToDatum）を扱うため。

関連コード

  • src/pl/plpython/plpy_planobject.c: 本ヘッダーで定義された構造体のメソッド実装およびライフサイクル管理の実体。
  • src/pl/plpython/plpy_spi.c: 実行計画の実行（execute）やカーソルの作成など、SPI を介した操作のロジック。

読者向けメモ

  • PLyPlanObject は MemoryContext を保持しており、計画に関連するメモリ管理を PostgreSQL のメモリ管理システムと同期させています。
  • Python 3.8 以降とそれ以前のバージョンで、PyTypeObject の参照カウント管理に差異があるため、実装ファイル（.c）側の #if ディレクティブに注意してください。
  • 本構造体は PyObject_HEAD を含んでいるため、Python の C API 拡張のルールに従ってメモリが確保・解放される必要があります。
