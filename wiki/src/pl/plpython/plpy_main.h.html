src/pl/plpython/plpy_main.h

概要

PL/Python 拡張の実行時状態を管理するための中心的なヘッダーファイルです。主に、Python コード実行時のコンテキスト管理と、グローバルなインタープリタ状態へのアクセスを提供します。

モジュール内での役割

src/pl/plpython モジュールにおける「実行エンジン」の基盤です。plpy_main.c と対になり、PostgreSQL の関数呼び出しから Python インタープリタへの橋渡しを行う際の、スタック管理とメモリ管理の規約を定義します。

責務

  • **実行コンテキストの定義**: Python コードのネストされた呼び出しを追跡するための PLyExecutionContext 構造体の提供。
  • **グローバル状態の公開**: Python インタープリタのグローバル辞書へのアクセス提供。
  • **メモリ管理の補助**: 実行コンテキストに紐付いたスクラッチメモリコンテキストの管理。

重要なシンボル

  • PLyExecutionContext: 現在実行中のプロシージャ情報と、一時的なメモリ割り当て用の scratch_ctx を保持するスタック構造体。
  • PLy_interp_globals: Python インタープリタのグローバル名前空間を指すポインタ。
  • PLy_current_execution_context(): 現在の実行スタックのトップを取得する関数。

制御フローとデータフロー

本ファイルはデータ構造の定義が主であり、直接的な制御フローは持ちません。

  • **データフロー**: PostgreSQL のハンドラが Python コードを呼び出す際、PLyExecutionContext がスタックにプッシュされます。Python コード実行中、このコンテキストを通じて現在のプロシージャ情報やメモリ領域が参照されます。

直接依存

  • plpy_procedure.h: PLyProcedure 構造体の定義を参照するために必要です。

関連コード

  • src/pl/plpython/plpy_main.c: 本ヘッダーで定義された構造体の実体化、および PLy_initialize によるインタープリタの初期化が行われます。
  • src/pl/plpython/plpy_exec.c: 実行コンテキストのプッシュ/ポップ操作や、実際の Python コード実行ロジックが含まれます。

読者向けメモ

  • 本ファイルで定義される PLyExecutionContext は、再帰的な Python 呼び出し（トリガー内での関数呼び出しなど）をサポートするためのスタック構造です。
  • scratch_ctx は、型変換や一時的なデータ処理のために使用されるメモリコンテキストです。メモリリークを防ぐため、コンテキストのライフサイクルと適切に同期させる必要があります。
