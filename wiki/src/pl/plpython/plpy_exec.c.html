src/pl/plpython/plpy_exec.c

概要

plpy_exec.c は、PostgreSQL の PL/Python における Python コード実行の心臓部です。SQL 関数やトリガーとして呼び出された Python コードの実行環境を構築し、PostgreSQL と Python 間のデータ変換、SPI（Server Programming Interface）を介したクエリ実行、およびエラーハンドリングを統括します。

モジュール内での役割

plpy_procedure.c がプロシージャの定義やキャッシュを管理するのに対し、本ファイルは実行時のコンテキスト（引数、戻り値、SRF 状態）を管理します。plpy_spi.c と連携してデータベース操作を行い、plpy_typeio.c を利用して型変換を行います。

責務

  • **関数実行:** PLy_exec_function を通じて、標準関数および集合を返す関数（SRF）の実行を制御する。
  • **トリガー実行:** PLy_exec_trigger を通じて、トリガーデータの構築、Python への引き渡し、および戻り値（SKIP/MODIFY 等）の処理を行う。
  • **引数管理:** 再帰呼び出しに対応するため、プロシージャのグローバル辞書内の引数状態をスタック（PLySavedArgs）で管理する。
  • **リソース保護:** PG_TRY/PG_CATCH ブロックにより、Python 実行中のエラー発生時に SPI 接続の切断やメモリの解放を保証する。

重要なシンボル

  • PLy_exec_function: SQL 関数呼び出しのメインエントリーポイント。
  • PLy_exec_trigger: トリガー呼び出しのメインエントリーポイント。
  • PLySRFState: 集合を返す関数（SRF）のイテレータ状態を保持する構造体。
  • PLy_global_args_push/pop: 再帰呼び出し時に Python グローバル辞書の引数を退避・復元するスタック操作関数。
  • PLy_procedure_call: Python コードを評価し、エラー発生時に PostgreSQL のエラーとして伝播させるラッパー。

制御フローとデータフロー

1. **初期化:** PLy_global_args_push で現在の引数状態をスタックに退避。 2. **実行準備:** PLy_function_build_args または PLy_trigger_build_args で、PostgreSQL の FunctionCallInfo から Python オブジェクトを生成。 3. **Python 実行:** PLy_procedure_call が PyEval_EvalCode を呼び出し、Python インタプリタ上でコードを実行。 4. **結果変換:** 戻り値を PLy_output_convert で PostgreSQL の Datum に変換。 5. **クリーンアップ:** PG_FINALLY または PG_CATCH ブロックで、SPI の終了処理、スタックのポップ、および Python オブジェクトの参照カウント（Py_DECREF）を適切に処理。

読者向けメモ

  • **再帰呼び出し:** PL/Python は関数内で別の PL/Python 関数を呼ぶことが可能です。PLy_global_args_push/pop は、この再帰時にグローバル辞書 proc->globals が汚染されないようにするための重要なメカニズムです。
  • **メモリ管理:** Python オブジェクトの参照カウント管理と PostgreSQL のメモリコンテキスト管理が混在しています。PG_TRY ブロック内での Py_DECREF 漏れはメモリリークに直結するため、PG_CATCH での解放処理が厳密に行われています。
  • **SRF の挙動:** SRF は複数回呼び出されるため、PLySRFState を用いてイテレータの状態を保持しています。MemoryContextRegisterResetCallback を使用して、関数終了時に確実にイテレータが解放されるよう設計されています。
