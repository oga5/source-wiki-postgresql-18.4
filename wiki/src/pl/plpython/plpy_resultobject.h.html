src/pl/plpython/plpy_resultobject.h

概要

plpy_resultobject.h は、PL/Python において PostgreSQL のクエリ実行結果を Python オブジェクトとして表現するためのデータ構造定義と、その初期化インターフェースを提供します。

モジュール内での役割

src/pl/plpython モジュールにおいて、SPI（Server Programming Interface）経由で実行されたクエリの結果を Python 側へ橋渡しする役割を担います。SQL の結果セットを Python のリストや辞書のように操作可能にするための基盤です。

責務

  • **データ構造の定義**: PLyResultObject 構造体を定義し、Python オブジェクトとしてのヘッダと、PostgreSQL のタプル記述子（TupleDesc）、行データ、ステータスを保持する。
  • **ライフサイクル管理**: Python 側から利用可能な PLyResult 型の初期化と、新規インスタンス生成のインターフェースを提供。

重要なシンボル

  • PLyResultObject: クエリ結果を保持するメインの構造体。PyObject_HEAD を含み、Python の拡張型として機能します。
  • PLy_result_init_type(): Python インタプリタ起動時に PLyResult 型を登録するための関数。
  • PLy_result_new(): 新しい PLyResultObject インスタンスを生成し、デフォルト値（空リストや None）で初期化する関数。

制御フローとデータフロー

本ヘッダーはデータ構造の定義に特化しています。実際のデータフロー（クエリ結果の格納や Python 側への変換）は、対となる plpy_resultobject.c 内のメソッドによって制御されます。 1. PLy_result_new でメモリ確保と初期化。 2. SPI 実行結果に基づき、tupdesc や rows が設定される。 3. Python 側からのメソッド呼び出し（colnames や nrows 等）に応じて、PLyResultObject 内のデータが参照される。

直接依存

  • access/tupdesc.h: PostgreSQL のタプル記述子（TupleDesc）を利用するために必要。
  • plpython.h: Python の C API および PL/Python 共通の定義を利用するために必要。

関連コード

  • src/pl/plpython/plpy_resultobject.c: 本ヘッダーで定義された構造体の実装および、Python 側から呼び出されるメソッド群（colnames, nrows 等）が記述されています。
  • src/pl/plpython/plpy_spi.c: クエリ実行結果を PLyResultObject に変換する処理の呼び出し元となる可能性が高いファイルです。

読者向けメモ

  • 本構造体は Python の拡張型として定義されているため、PyObject_HEAD を含んでいます。メモリ管理は Python の参照カウンタ（Py_INCREF/Py_DECREF）と PostgreSQL のメモリコンテキストの双方に注意を払う必要があります。
  • tupdesc は PostgreSQL 側のリソースであるため、PLy_result_dealloc で適切に解放されることを確認してください。
