src/pl/plpython/plpy_spi.h

概要

plpy_spi.h は、PL/Python から PostgreSQL の SPI (Server Programming Interface) を利用するためのインターフェースを定義するヘッダーファイルです。Python スクリプトから SQL クエリの実行やプリペアドステートメントの管理を行うための関数群を公開します。

モジュール内での役割

src/pl/plpython モジュールにおいて、Python レイヤーと PostgreSQL の実行エンジンを橋渡しする役割を担います。特に、Python 側から呼び出される plpy.execute や plpy.prepare といった関数のバックエンド実装をサポートします。

責務

  • **SPI インターフェースの公開**: Python から SQL を実行するための関数定義を提供。
  • **トランザクション制御**: plpy.commit() および plpy.rollback() を介したトランザクション制御の提供。
  • **サブトランザクション管理**: SPI 操作に伴うサブトランザクションの開始・コミット・アボート処理の管理。
  • **例外マッピング**: PostgreSQL のエラーコード（SQLSTATE）と Python の例外クラスを紐付けるための構造定義。

重要なシンボル

  • PLy_spi_prepare: Python から SQL をプリペアドステートメントとして準備する関数。
  • PLy_spi_execute: Python から SQL を即時実行する関数。
  • PLy_spi_subtransaction_*: SPI 操作を安全に行うためのサブトランザクション制御関数群。
  • PLyExceptionEntry: SQLSTATE と Python 例外オブジェクトを対応させるための構造体。

制御フローとデータフロー

本ファイルはインターフェース定義であるため、実際のフローは plpy_spi.c に依存します。 1. Python 側から plpy モジュールの関数が呼び出される。 2. plpy_spi.h で宣言された関数が plpy_spi.c 内で実行される。 3. 必要に応じて PLy_spi_subtransaction_begin が呼ばれ、メモリコンテキストとリソースオーナーが保護される。 4. SPI を通じて PostgreSQL エンジンへクエリが渡され、結果が Python オブジェクトに変換されて返される。

直接依存

  • plpython.h: PL/Python の基本定義および Python C API へのアクセス。
  • utils/resowner.h: リソースオーナー管理（SPI 操作中のリソースリーク防止のため）。

関連コード

  • src/pl/plpython/plpy_spi.c: 本ヘッダーで宣言された関数の実装。
  • src/pl/plpython/plpy_planobject.h: プリペアドステートメントのデータ構造定義。
  • src/pl/plpython/plpy_resultobject.h: SPI 実行結果を保持する Python オブジェクトの定義。

読者向けメモ

  • 本ファイルで定義されている PLy_spi_subtransaction_* 関数は、SPI 操作中にエラーが発生した場合のメモリリークやリソースの不整合を防ぐために重要です。
  • PLyExceptionEntry は、PostgreSQL のエラーを Python の例外として適切にキャッチ・再送出するための基盤となります。
  • SPI を使用する際は、必ず適切なメモリコンテキストの切り替えが行われているかを確認してください。
