src/pl/plpython/plpy_spi.c

概要

PostgreSQL の SPI (Server Programming Interface) と Python インターフェースを仲介するレイヤーです。Python コードから SQL クエリの実行、プリペアドステートメントの管理、およびトランザクション制御を可能にします。

モジュール内での役割

plpy モジュールのコア機能の一部として、データベース操作の実行エンジンを提供します。plpy_exec.c 等が上位の実行ロジックを担うのに対し、本ファイルは SPI 呼び出しの低レイヤーなラップと、PostgreSQL のエラーを Python 例外へ変換する責務を負います。

責務

  • **クエリ実行**: plpy.execute() を通じた SQL クエリの実行と結果の取得。
  • **プリペアドステートメント**: plpy.prepare() によるクエリの事前コンパイルと引数型の解決。
  • **トランザクション管理**: plpy.commit() / rollback() の提供および内部サブトランザクションの安全な管理。
  • **エラー変換**: PostgreSQL の ErrorData を Python の例外オブジェクトへマッピングし、詳細なエラー情報を付与。

重要なシンボル

  • PLy_spi_prepare: plpy.prepare の実装。クエリを解析し PLyPlanObject を生成。
  • PLy_spi_execute: クエリまたはプランを実行するエントリポイント。
  • PLy_spi_subtransaction_begin/commit/abort: SPI 操作をサブトランザクション内で安全に実行するための定型処理。
  • PLy_spi_execute_fetch_result: SPI の結果セット（SPITupleTable）を Python の PLyResultObject に変換。

制御フローとデータフロー

1. **リクエスト**: Python 側から plpy.execute 等が呼ばれると、PLy_spi_subtransaction_begin でサブトランザクションを開始します。 2. **実行**: PG_TRY() ブロック内で SPI_execute または SPI_execute_plan を呼び出します。 3. **変換**: 実行結果が成功すれば PLy_spi_execute_fetch_result で Python オブジェクトへ変換し、失敗すれば PG_CATCH() ブロックへ遷移します。 4. **例外処理**: PG_CATCH() 内で PLy_spi_subtransaction_abort を呼び出し、PostgreSQL のエラー情報を PLy_spi_exception_set で Python 例外として再送出します。

読者向けメモ

  • **メモリ管理**: SPI 操作は PG_TRY ブロック内でサブトランザクションを伴うことが多いため、メモリコンテキストの切り替え（MemoryContextSwitchTo）が頻繁に行われます。リークを防ぐため、PG_CATCH でのクリーンアップが必須です。
  • **エラーハンドリング**: PostgreSQL のエラーは longjmp を伴うため、Python の例外処理と混在させる際は PG_TRY / PG_CATCH を正しく使用する必要があります。
  • **サブトランザクション**: SPI 操作中にエラーが発生した場合、サブトランザクションを適切にアボートしないと、メインのトランザクションまで汚染される可能性がある点に注意してください。
