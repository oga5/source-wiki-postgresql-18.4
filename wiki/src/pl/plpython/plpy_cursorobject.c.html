src/pl/plpython/plpy_cursorobject.c

概要

PostgreSQL のカーソルを Python から操作するための PLyCursor クラスを実装するファイルです。SQL クエリまたはプリペアドプランからカーソルを作成し、データのフェッチやカーソルのクローズを行うインターフェースを提供します。

モジュール内での役割

src/pl/plpython モジュールにおいて、SPI（Server Programming Interface）のカーソル機能を Python オブジェクトとしてラップする役割を担います。plpy_spi.c が提供する低レベルな SPI 操作を、Python のイテレータやメソッド呼び出しとして抽象化します。

責務

  • PLyCursor 型の定義と初期化。
  • SQL クエリまたは PLyPlanObject からのカーソル生成。
  • fetch メソッドによる結果セットの取得と Python オブジェクトへの変換。
  • Python イテレータプロトコル（__next__）の実装による行単位の反復処理。
  • カーソルライフサイクルの管理（クローズおよびメモリコンテキストの解放）。

重要なシンボル

  • PLy_cursor: カーソル作成のメインエントリポイント。
  • PLy_cursor_fetch: 指定行数を取得するメソッド。
  • PLy_cursor_iternext: Python の for ループ等で呼び出されるイテレータ実装。
  • PLyCursorObject: カーソルの状態（ポータル名、メモリコンテキスト等）を保持する構造体。

制御フローとデータフロー

1. **初期化**: PLy_cursor がクエリまたはプランを受け取り、SPI_cursor_open を呼び出してポータルを作成します。 2. **データ取得**: fetch または iternext が呼び出されると、SPI_cursor_fetch を実行してデータを取得します。 3. **変換**: 取得したタプルは PLy_input_from_tuple を介して Python オブジェクトに変換されます。 4. **エラー処理**: 各操作は PG_TRY/PG_CATCH ブロックで囲まれ、SPI 操作失敗時にはサブトランザクションを適切にアボートします。 5. **クリーンアップ**: PLy_cursor_dealloc により、カーソルが閉じられ、関連するメモリコンテキストが解放されます。

読者向けメモ

  • 本ファイルは Python の C API を直接使用しているため、参照カウント（Py_INCREF/Py_DECREF）の管理が重要です。
  • カーソルは PostgreSQL のポータルに依存しているため、サブトランザクションのアボート時にポータルが無効化されるケースを考慮したエラーハンドリングが行われています。
  • PLy_cursor_query および PLy_cursor_plan 内で作成される MemoryContext は、カーソルオブジェクトの生存期間と同期して管理されます。
