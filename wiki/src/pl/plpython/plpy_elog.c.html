src/pl/plpython/plpy_elog.c

概要

Python の例外を PostgreSQL のエラー報告システム（ereport）へ変換し、逆に PostgreSQL のエラー情報を Python の例外オブジェクトへマッピングするためのブリッジ機能を提供します。

モジュール内での役割

plpy モジュールにおけるエラーハンドリングの中核です。Python 側で発生した例外をキャッチし、PostgreSQL のエラーメッセージ、詳細、ヒント、トレースバック情報として適切に整形してログ出力やクライアントへの通知を行います。

責務

  • **例外の変換**: Python の PyErr を PostgreSQL の elevel（ERROR, FATAL 等）に変換。
  • **トレースバックの生成**: Python のスタックトレースを解析し、PostgreSQL のエラーコンテキストとして整形。
  • **SPI エラーの伝播**: SPIError などのカスタム例外から SQLSTATE やエラー詳細情報を抽出。
  • **例外属性の管理**: Python 例外オブジェクトへの属性（detail, hint 等）の読み書き。

重要なシンボル

  • PLy_elog_impl: Python の例外状態を PostgreSQL の ereport に変換するメイン関数。
  • PLy_exception_set_with_details: PostgreSQL の ErrorData を Python 例外オブジェクトに変換してセットする関数。
  • PLy_exc_error, PLy_exc_fatal, PLy_exc_spi_error: Python 側で利用可能な PL/Python 特有の例外クラス。

制御フローとデータフロー

1. **エラー発生時**: PLy_elog_impl が呼び出されると、PyErr_Fetch で現在の Python 例外状態を取得します。 2. **正規化**: PyErr_NormalizeException を経て、例外の種類（SPI 関連か、汎用エラーか）を判定します。 3. **情報抽出**: PLy_traceback がスタックフレームを辿り、エラーメッセージとトレースバック文字列を構築します。 4. **報告**: ereport マクロを使用して PostgreSQL のエラーログシステムへ情報を渡します。 5. **クリーンアップ**: PG_FINALLY ブロック内で、取得した Python オブジェクトの参照カウントを減らし、メモリを解放します。

直接依存

  • postgres.h: PostgreSQL のコア機能および ereport 等の定義。
  • lib/stringinfo.h: 動的文字列バッファ操作。
  • plpy_elog.h, plpy_main.h, plpy_procedure.h, plpy_util.h: PL/Python 内部の共通定義およびユーティリティ。

関連コード

  • src/pl/plpython/plpy_exec.c: SQL 実行中に発生したエラーが本モジュールを介して報告される主要な呼び出し元。

読者向けメモ

  • **メモリ管理**: Python オブジェクトの参照カウント管理と PostgreSQL のメモリコンテキスト（palloc/pfree）が混在するため、PG_TRY/PG_FINALLY を用いた安全なリソース解放が徹底されています。
  • **トレースバック**: Python の traceback モジュールの挙動を模倣するように設計されていますが、PL/Python 特有の実行コンテキスト（PLyExecutionContext）を考慮したカスタム実装になっています。
  • **文字列変換**: PLyUnicode_AsString 等のユーティリティを介して Python 文字列と PostgreSQL の文字列を相互変換しています。
