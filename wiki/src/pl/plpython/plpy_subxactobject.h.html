src/pl/plpython/plpy_subxactobject.h

概要

本ヘッダーファイルは、PL/Python における明示的なサブトランザクション制御のための Python オブジェクト定義を提供します。PostgreSQL のサブトランザクション機能を Python のコンテキストマネージャ（with 文）として利用可能にするためのデータ構造とインターフェースを定義しています。

モジュール内での役割

src/pl/plpython モジュールにおいて、SPI（Server Programming Interface）呼び出しをグループ化し、アトミックな実行を保証するための基盤を提供します。Python 側から plpy.subtransaction() を通じて呼び出されるオブジェクトの型定義を担います。

責務

  • **Python オブジェクト定義**: PLySubtransactionObject 構造体を通じ、Python インタプリタと PostgreSQL のトランザクション状態を橋渡しする。
  • **状態管理**: サブトランザクションの開始・終了状態を保持し、二重実行や不正な操作を防止する。
  • **リソース追跡**: PLySubtransactionData を通じて、サブトランザクション開始前のメモリコンテキストやリソースオーナーを保持し、終了時に適切に復元する。

重要なシンボル

  • PLySubtransactionObject: Python 側から見えるサブトランザクションオブジェクトの構造体。started および exited フラグでライフサイクルを管理。
  • PLySubtransactionData: PostgreSQL 内部でサブトランザクションのコンテキストを保持するための構造体。
  • explicit_subtransactions: 現在アクティブな明示的サブトランザクションを追跡するグローバルなリスト。

制御フローとデータフロー

1. **初期化**: PLy_subtransaction_init_type() により、Python 型として登録される。 2. **生成**: PLy_subtransaction_new() が呼ばれ、Python オブジェクトがインスタンス化される。 3. **実行**: __enter__ メソッド（plpy_subxactobject.c 内）が呼ばれると、BeginInternalSubTransaction が実行され、現在のコンテキストが explicit_subtransactions リストに保存される。 4. **終了**: __exit__ メソッドが呼ばれると、例外の有無に応じてサブトランザクションのコミットまたはロールバックが行われ、リソースが解放される。

直接依存

  • nodes/pg_list.h: サブトランザクションのスタック管理に使用するリスト構造。
  • plpython.h: Python C API および PL/Python 共通定義。
  • utils/resowner.h: PostgreSQL のリソースオーナー管理。

関連コード

  • src/pl/plpython/plpy_subxactobject.c: 本ヘッダーで定義された構造体の実装およびメソッドの実体。
  • src/pl/plpython/plpy_spi.c: サブトランザクション内で実行される SPI 呼び出しの関連ロジック。

読者向けメモ

  • 本オブジェクトは Python のコンテキストマネージャとして設計されています。with 文を使用することで、例外発生時に自動的にサブトランザクションがロールバックされる安全なコードが記述可能です。
  • explicit_subtransactions リストは、ネストされたサブトランザクションの整合性を保つために重要です。直接操作する際はメモリコンテキストの切り替えに注意してください。
