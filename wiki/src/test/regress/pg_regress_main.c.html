src/test/regress/pg_regress_main.c

概要

pg_regress_main.c は、PostgreSQL の回帰テストを実行するためのメイン実行ファイルのエントリポイントです。かつてシェルスクリプトで実装されていたテスト実行ロジックを C 言語に移植したものであり、テストの起動から実行プロセスの管理までを担います。

モジュール内での役割

pg_regress モジュールにおける「メイン実行プログラム」の役割を果たします。pg_regress.c で定義された汎用的なテスト実行フレームワーク（regression_main）に対し、具体的なテスト実行手順（psql を用いた SQL ファイルの実行）を注入する役割を担います。

責務

  • **テスト実行の初期化**: テスト対象のデータベース名などのデフォルト設定を行う。
  • **テストプロセスの生成**: 各テストケース（.sql ファイル）に対して psql コマンドを構築し、サブプロセスとして起動する。
  • **パス管理**: 入力（SQL）、期待値（Expected）、出力（Result）ファイルのパスを解決し、テスト実行環境を整える。
  • **環境変数の制御**: PGAPPNAME を設定し、テスト実行中のログやプロセス監視を容易にする。

重要なシンボル

  • main: プログラムの起点。regression_main を呼び出し、テスト実行の制御をフレームワークへ委譲します。
  • psql_start_test: 個別のテストケースを実行するための関数。psql コマンドラインを組み立て、spawn_process で実行します。
  • psql_init: テスト実行前の初期化関数。デフォルトのデータベース名などを設定します。

制御フローとデータフロー

1. main が起動し、regression_main を呼び出します。 2. regression_main は引数として渡された psql_init を実行し、環境を初期化します。 3. テスト実行フェーズに入ると、regression_main は各テストケースに対して psql_start_test を呼び出します。 4. psql_start_test は以下の処理を行います：

  • inputdir や outputdir を参照して対象ファイルのパスを特定。
  • StringInfo を用いて psql 実行コマンド文字列を構築。
  • spawn_process でプロセスを生成し、PID を返却。

    • 5. フレームワーク側が PID を監視し、テスト完了を待ち合わせます。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用共通ヘッダー。
  • lib/stringinfo.h: 動的文字列バッファ操作用。
  • pg_regress.h: pg_regress モジュール内の共通定義および関数プロトタイプ。

関連コード

  • src/test/regress/pg_regress.c: テスト実行のコアエンジン。regression_main が定義されています。
  • src/test/regress/regress.c: テスト実行に関連する補助的なロジックが含まれます。

読者向けメモ

  • 本ファイルは非常にシンプルであり、pg_regress の全体像を把握するための「実装例」として最適です。
  • psql_start_test 内で HIDE_TABLEAM や HIDE_TOAST_COMPRESSION が設定されている点に注目してください。これは、ストレージエンジンの違いによるテスト結果の差異を吸収するための重要な工夫です。
  • 複雑なテストロジックは pg_regress.c に集約されているため、テスト実行の仕組みを深く理解したい場合はそちらを参照してください。
