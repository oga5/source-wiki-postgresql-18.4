src/test/regress/regress.c

概要

PostgreSQL の回帰テストスイートで使用される、C 言語で記述された補助関数群を定義するソースファイルです。主に SQL レベルから呼び出されるテスト用関数や、内部的なデータ型、アトミック操作の検証ロジックが含まれています。

モジュール内での役割

src/test/regress モジュールの一部として、SQL スクリプト（.sql ファイル）から直接呼び出せる C 関数を提供します。これにより、標準の SQL ではテストが困難な内部挙動（メモリレイアウト、アトミック操作、エンコーディング変換など）の検証を可能にします。

責務

  • **テスト用データ型の定義**: WIDGET 型のようなテスト専用のデータ型とその入出力関数の提供。
  • **内部機能の検証**: アトミック操作、スピンロック、エンコーディング変換、TOAST 処理などの低レイヤー機能の正当性確認。
  • **SQL インターフェースの提供**: PG_FUNCTION_INFO_V1 を用いた、SQL から実行可能なテスト関数のエクスポート。

重要なシンボル

  • WIDGET: テスト用のカスタムデータ型構造体。
  • test_atomic_ops: アトミック操作（フラグ、uint32, uint64）およびスピンロックの正当性を検証する関数。
  • test_enc_conversion: エンコーディング変換ロジックをテストするための関数。
  • make_tuple_indirect: TOAST の間接参照（indirect）を強制的に生成し、検証するための関数。
  • EXPECT_TRUE, EXPECT_EQ_U32/U64: テスト内で条件判定を行い、失敗時に elog(ERROR) を投げるためのマクロ。

制御フローとデータフロー

本ファイル内の関数は、主に SQL の CREATE FUNCTION を通じてデータベースにロードされます。 1. **呼び出し**: SQL クエリから Datum 型の引数を受け取ります。 2. **処理**: PG_GETARG_* マクロで引数を取得し、PostgreSQL の内部 API（palloc, DirectFunctionCall 等）を使用して処理を実行します。 3. **検証**: EXPECT_* マクロや elog を用いて、期待値と異なる場合に即座にエラーを発生させ、テストを中断させます。 4. **返却**: PG_RETURN_* マクロを使用して結果を SQL レベルへ返します。

読者向けメモ

  • 本ファイル内の関数は、本番環境での利用を想定していません。あくまで回帰テスト用です。
  • test_atomic_ops などの検証関数は、プラットフォーム固有の動作を確認するため、移植性の検証にも役立ちます。
  • make_tuple_indirect のように、意図的に内部構造を破壊・操作する関数が含まれているため、PostgreSQL の内部データ構造を学習する際のサンプルとしても有用です。
