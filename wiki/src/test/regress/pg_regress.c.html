src/test/regress/pg_regress.c

概要

pg_regress.c は、PostgreSQL の回帰テストを実行するためのメインドライバです。シェルスクリプトから C 言語へ移植されたこのプログラムは、テスト環境（一時的なデータベースインスタンスの構築や既存環境への接続）の準備、テストケースの並列実行、結果の検証、および TAP 形式でのレポート生成を担います。

モジュール内での役割

src/test/regress モジュールのコアエンジンとして機能します。pg_regress.h で定義されたインターフェースを実装し、テスト実行のライフサイクル（初期化、実行、後処理）を制御します。

責務

  • **テスト実行制御**: スケジュールファイルに基づいたテストの並列実行管理と、プロセス終了の待機。
  • **環境構築**: initdb を用いた一時的な PostgreSQL インスタンスの作成、設定ファイルの調整、および認証設定（SSPI 等）の自動構成。
  • **結果検証**: diff コマンドを用いた期待値との比較と、プラットフォーム固有の差異を吸収するための resultmap 処理。
  • **TAP 出力**: テスト結果を TAP (Test Anything Protocol) 形式で標準出力へ書き出し、ログファイルへ記録。

重要なシンボル

テスト実行・管理

  • regression_main: テスト実行のメインエントリポイント。引数解析、環境構築、テスト実行ループを統括。
  • run_schedule: スケジュールファイル内のテストグループを読み込み、並列実行を制御。
  • spawn_process: OS 依存のプロセス生成（POSIX fork/exec または Windows CreateRestrictedProcess）。
  • wait_for_tests: 実行中のテストプロセスが終了するまで待機し、終了ステータスを収集。

結果検証・出力

  • results_differ: diff を実行し、期待値との差異を判定。resultmap を参照して適切な期待値ファイルを選択。
  • emit_tap_output: TAP プロトコル形式でログを出力。DIAG, NOTE, TEST_STATUS などの型をサポート。
  • test_status_print: 個別のテスト結果（成功/失敗）を整形して出力。

データベース・環境操作

  • create_database / drop_database_if_exists: psql を介してテスト用 DB を管理。
  • make_temp_sockdir: 一時的な Unix ソケットディレクトリを作成し、終了時に自動削除するよう atexit を登録。
  • initialize_environment: テスト実行に必要な環境変数（PGHOST, PGPORT, PGOPTIONS 等）を構成。

制御フローとデータフロー

1. **初期化**: regression_main が引数を解析し、initdb で一時インスタンスを作成するか、既存環境へ接続するかを決定。 2. **環境構築**: initialize_environment が環境変数を設定。temp-instance モードの場合、postmaster を起動し、PQpingParams で接続可能になるまで待機。 3. **テスト実行**: run_schedule がスケジュールファイルを読み込み、startfunc コールバックを介してテストプロセスを起動。wait_for_tests で終了を待機。 4. **検証**: 各テスト終了後、results_differ が diff を呼び出し、結果を比較。失敗した場合は failed_tests に記録。 5. **終了処理**: 全テスト完了後、stop_postmaster でインスタンスを停止し、TAP 形式のサマリーを出力して終了。

読者向けメモ

  • **メモリ管理**: _stringlist や _resultmap などの連結リストは、テスト終了時に free_stringlist 等で明示的に解放されます。
  • **プラットフォーム差異**: Windows と POSIX でプロセス管理（spawn_process）やソケット処理が大きく異なります。特に Windows では diff の改行コード問題への対策が含まれています。
  • **エラーパス**: bail() 関数は致命的なエラー時に呼び出され、atexit で登録されたクリーンアップ処理（stop_postmaster 等）を実行して終了します。
  • **省略された詳細**: string_matches_pattern は簡易的な正規表現エンジンですが、resultmap のマッチングに特化しており、汎用的な正規表現ライブラリではありません。
