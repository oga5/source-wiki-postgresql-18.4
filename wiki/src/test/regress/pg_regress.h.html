src/test/regress/pg_regress.h

概要

pg_regress.h は、PostgreSQL の回帰テスト駆動プログラムである pg_regress のコアインターフェースを定義するヘッダーファイルです。テスト実行に必要なデータ構造、関数シグネチャ、およびプラットフォーム依存の型定義を提供します。

モジュール内での役割

pg_regress モジュールの中心的なコントラクトを定義します。このヘッダーを介して、メインの駆動ロジック（pg_regress.c）と、個別のテスト実行ロジックや初期化処理が疎結合に連携できるよう設計されています。

責務

  • **プラットフォーム抽象化**: Windows と Unix 系 OS 間でのプロセス ID（PID_TYPE）の差異を吸収する。
  • **データ構造の定義**: テスト対象のリストや設定値を保持するための汎用的な文字列リスト構造体（_stringlist）の提供。
  • **コールバック定義**: テスト実行のライフサイクル（初期化、開始、結果の後処理）を制御するための関数ポインタ型の定義。
  • **グローバル設定の公開**: テスト実行に必要なディレクトリパスやフラグを外部モジュールから参照可能にする。

重要なシンボル

  • PID_TYPE / INVALID_PID: OS に依存しないプロセス管理のための型と無効値。
  • _stringlist: テスト名や引数リストを管理するための連結リスト。
  • regression_main(): テスト駆動プログラムのメインエントリポイント。
  • init_function / test_start_function / postprocess_result_function: テストの挙動をカスタマイズするためのコールバック型。

制御フローとデータフロー

本ファイル自体は宣言のみですが、regression_main を呼び出すことで以下のフローが開始されます： 1. init_function による環境設定。 2. test_start_function を用いた各テストプロセスの生成。 3. postprocess_result_function による出力結果の検証。 このフローにおいて、_stringlist がテスト対象の管理データとして各関数間を渡されます。

直接依存

  • <unistd.h>: POSIX 標準のシステムコール定義（主に pid_t の取得）。
  • unistd.h

関連コード

  • src/test/regress/pg_regress.c: 本ヘッダーで定義された関数の実装およびメインロジック。
  • src/test/regress/pg_regress_main.c: regression_main を呼び出す具体的な実行バイナリのソース。

読者向けメモ

  • **プラットフォーム依存性**: WIN32 マクロの有無によって PID_TYPE の実体が pid_t か HANDLE かに切り替わります。移植性を考慮する際は注意してください。
  • **拡張性**: 新しいテストスイートを追加する場合、regression_main に渡すコールバック関数を実装することで、既存のテストエンジンを再利用可能です。
  • **グローバル変数**: dblist や inputdir などの変数は pg_regress.c で定義され、ここで extern 宣言されています。これらはテスト実行中に動的に更新されるため、スレッドセーフではない点に留意してください。
