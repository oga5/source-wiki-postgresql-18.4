src/test/examples/testlibpq3.c

概要

testlibpq3.c は、PostgreSQL の C 言語用クライアントライブラリである libpq を使用し、**パラメータ化クエリ（PQexecParams）**と**バイナリ形式でのデータ送受信**を実証するためのサンプルプログラムです。

モジュール内での役割

src/test/examples ディレクトリ内に位置し、開発者が libpq の高度な機能（バイナリ I/O やアウトオブラインパラメータ）を実装する際の参照実装として機能します。

責務

  • PQexecParams を用いたクエリ実行のデモンストレーション。
  • ネットワークバイトオーダー（ビッグエンディアン）とホストバイトオーダー間の変換処理の提示。
  • バイナリデータ（bytea 型など）の取得と、フィールド長を考慮した安全なデータアクセスの実演。

重要なシンボル

  • PQexecParams: クエリとパラメータを分離して送信するための主要関数。
  • show_binary_results: バイナリ形式で返された PGresult を解析し、標準出力に整形表示するヘルパー関数。
  • ntohl / htonl: ネットワークバイトオーダーとホストバイトオーダー間の変換に使用。

追加の索引済みシンボル

  • exit_nicely
  • main

制御フローとデータフロー

1. **接続確立**: PQconnectdb でデータベースに接続し、SET search_path で環境を整えます。 2. **テキストパラメータの送信**: PQexecParams を使用し、パラメータをテキストとして送信しつつ、結果をバイナリ形式で要求します。 3. **バイナリパラメータの送信**: 整数値を htonl で変換し、paramFormats を 1（バイナリ）に設定して送信します。 4. **結果解析**: show_binary_results 内で PQfnumber を用いてフィールド位置を特定し、PQgetvalue で取得したポインタを適切な型にキャストして処理します。 5. **終了処理**: PQclear で結果セットを解放し、PQfinish で接続を閉じます。

直接依存

  • libpq-fe.h: libpq の主要 API 定義。
  • netinet/in.h / arpa/inet.h: バイトオーダー変換関数（ntohl, htonl）の提供。
  • stdio.h, stdlib.h, string.h, stdint.h: 標準的な C ライブラリ。
  • sys/types.h
  • windows.h

関連コード

  • src/test/examples/testlibpq.c: 基本的なクエリ実行のサンプル。
  • src/test/examples/testlo.c: ラージオブジェクト操作のサンプル。
  • src/test/examples/testlibpq3.sql: 本プログラムを実行するための前提となるスキーマとデータ定義。

読者向けメモ

  • **バイトオーダー**: バイナリ I/O を扱う際は、必ず ntohl/htonl を使用してホストのエンディアンとネットワークのエンディアンを整合させる必要があります。
  • **フィールド長**: bytea 型などのバイナリデータにはヌルバイトが含まれる可能性があるため、strlen ではなく PQgetlength で取得した長さを必ず使用してください。
  • **実行前準備**: 本コードを実行する前に、ソース内のコメントに記載された SQL を実行し、test1 テーブルを作成しておく必要があります。
