src/test/examples/testlibpq.c

概要

PostgreSQL の C 言語用クライアントライブラリである libpq の基本的な使用方法を示すサンプルプログラムです。データベースへの接続、トランザクション内でのカーソル操作、およびクエリ結果の取得と表示という一連のライフサイクルを網羅しています。

モジュール内での役割

src/test/examples ディレクトリに配置された、libpq の機能を学習するための最も標準的な入門用サンプルです。他のサンプル（testlo.c など）がラージオブジェクトや特定の機能に特化しているのに対し、本ファイルはライブラリの基本 API の利用パターンを提示します。

責務

  • データベース接続の確立とステータス確認。
  • セキュリティを考慮した search_path の設定。
  • トランザクションブロック（BEGIN/END）の管理。
  • カーソルを用いたデータ取得と結果セットの反復処理。
  • リソース（PGconn, PGresult）の適切な解放。

重要なシンボル

  • PGconn: データベース接続を保持する不透明な構造体。
  • PGresult: クエリ実行結果を保持する構造体。
  • PQconnectdb(): 接続文字列に基づきデータベースへ接続する。
  • PQexec(): SQL コマンドをサーバーに送信し実行する。
  • PQclear(): PGresult が占有するメモリを解放する（メモリリーク防止に必須）。
  • PQfinish(): 接続を閉じ、関連リソースを解放する。

追加の索引済みシンボル

  • exit_nicely
  • main

制御フローとデータフロー

1. **初期化**: コマンドライン引数から接続文字列を取得し、PQconnectdb で接続を試行します。 2. **セットアップ**: PQexec を使用して search_path を空に設定し、セキュリティを確保します。 3. **トランザクション実行**: BEGIN を発行し、DECLARE CURSOR でカーソルを定義します。 4. **データ取得**: FETCH ALL で結果セットを取得し、PQnfields（列数）と PQntuples（行数）を用いて PQgetvalue で各値を標準出力へ書き出します。 5. **終了処理**: CLOSE でカーソルを閉じ、END でトランザクションを終了し、PQfinish で接続を破棄します。エラー発生時は exit_nicely を経由して安全に終了します。

直接依存

  • stdio.h: 標準入出力関数。
  • stdlib.h: exit() 関数。
  • libpq-fe.h: libpq の公開 API 定義。

関連コード

  • src/test/examples/testlibpq2.c: 非同期処理やより複雑なエラーハンドリングを含む発展的なサンプル。
  • src/test/examples/testlo.c: libpq を用いたラージオブジェクト操作のサンプル。

読者向けメモ

  • **メモリ管理**: PQexec が返す PGresult は、不要になった時点で必ず PQclear を呼び出す必要があります。これを怠るとメモリリークが発生します。
  • **エラーハンドリング**: 本サンプルでは exit_nicely を定義し、接続失敗時に PQfinish を呼んでから終了するよう設計されています。実運用コードでも同様のクリーンアップ戦略が推奨されます。
  • **実行方法**: コンパイルには libpq のヘッダーとライブラリへのパスを通す必要があります。通常は pg_config --cflags --libs を利用してビルドします。
