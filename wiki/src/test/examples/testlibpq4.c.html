src/test/examples/testlibpq4.c

概要

testlibpq4.c は、PostgreSQL のクライアントライブラリである libpq を使用して、同時に複数のデータベース接続を確立・管理する方法を示すサンプルプログラムです。

モジュール内での役割

src/test/examples ディレクトリに含まれる一連のサンプルコードの一つです。他のサンプルが単一接続やラージオブジェクトの操作に焦点を当てる中、本ファイルは「マルチコネクション」という応用的な接続パターンを提示する役割を担います。

責務

  • 2つの異なるデータベースに対して同時に PGconn オブジェクトを生成する。
  • 接続状態の検証と、セキュリティのための search_path 設定を自動化する。
  • トランザクション内でのカーソル操作（DECLARE, FETCH, CLOSE）の基本手順を示す。
  • エラー発生時に複数の接続を安全にクローズして終了する。

重要なシンボル

  • exit_nicely(PGconn*, PGconn*): 複数の接続を保持した状態でエラーが発生した際、メモリリークを防ぎつつ安全に終了するためのヘルパー関数。
  • check_prepare_conn(PGconn*, const char*): 接続の成否を確認し、search_path を空に設定してセキュリティを確保する初期化関数。
  • PQsetdb(...): データベースへの接続を確立するレガシーな関数。
  • main(...): プログラムのメインロジック。2つのデータベース接続を確立し、一方の接続でカーソルを用いたデータ取得を行う。

追加の索引済みシンボル

  • check_prepare_conn
  • exit_nicely
  • main

制御フローとデータフロー

1. **初期化**: コマンドライン引数から接続先データベース名を特定。 2. **接続**: PQsetdb を2回呼び出し、conn1 と conn2 を確立。check_prepare_conn で各接続を検証。 3. **トランザクション**: conn1 に対して BEGIN を発行。 4. **データ操作**: DECLARE CURSOR でカーソルを作成し、FETCH ALL でデータを取得。取得した結果セットを標準出力に整形表示。 5. **終了処理**: CLOSE でカーソルを閉じ、END でトランザクションを終了。最後に PQfinish で両方の接続を解放。

直接依存

  • stdio.h: 標準入出力用。
  • stdlib.h: exit() 等の標準ライブラリ用。
  • libpq-fe.h: PostgreSQL クライアントライブラリの主要ヘッダー。

関連コード

  • src/test/examples/testlibpq.c: 単一接続の基本例。
  • src/test/examples/testlibpq3.c: より詳細なエラーハンドリングを含む接続例。
  • libpq 公式ドキュメント: PQsetdb は現在非推奨であり、新規開発では PQconnectdb の使用が推奨される点に注意。

読者向けメモ

  • 本コードは libpq の古い API (PQsetdb) を使用しています。現代的なアプリケーション開発では、接続文字列を柔軟に扱える PQconnectdb や PQconnectdbParams を使用してください。
  • PQclear を忘れるとメモリリークが発生するため、PGresult を使い終わるたびに必ず呼び出すという libpq の重要なイディオムが実践されています。
  • 複数の接続を扱う際は、本コードのように終了処理を共通化（exit_nicely）しておくと、異常系でのリソース解放漏れを防げます。
