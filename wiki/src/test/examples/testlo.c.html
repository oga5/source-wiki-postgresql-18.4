src/test/examples/testlo.c

概要

PostgreSQL のラージオブジェクト（Large Object）機能を利用するための libpq サンプルプログラムです。ローカルファイルとデータベース内のラージオブジェクト間でのデータ転送、およびオブジェクト内の特定範囲に対する読み書き操作を実演します。

モジュール内での役割

src/test/examples ディレクトリに配置された、libpq の API 使用例の一つです。特にラージオブジェクトのインターフェース（lo_* 関数群）の基本的な使い方を提示する役割を担います。

責務

  • ローカルファイルからデータベースへのラージオブジェクトのインポート。
  • データベース内のラージオブジェクトからローカルファイルへのエクスポート。
  • lo_lseek を用いたラージオブジェクト内の特定位置へのアクセスと、部分的な読み取り・書き込み。
  • トランザクション内でのラージオブジェクト操作の実行。

重要なシンボル

  • importFile / exportFile: ファイルシステムとラージオブジェクト間のデータ転送を実装する補助関数。
  • pickout: ラージオブジェクトの特定範囲を読み取り、標準エラー出力に表示する関数。
  • overwrite: ラージオブジェクトの特定範囲を 'X' で上書きする関数。
  • lo_import / lo_export: libpq が提供するラージオブジェクト転送用の高レベル API。

追加の索引済みシンボル

  • exit_nicely
  • main

制御フローとデータフロー

1. **初期化**: PQsetdb でデータベースに接続し、search_path を安全に設定します。 2. **トランザクション開始**: BEGIN を発行し、ラージオブジェクト操作をトランザクション内で保護します。 3. **インポート**: lo_import を使用してローカルファイルをラージオブジェクトとして登録します。 4. **操作**: pickout で特定範囲を読み取り、overwrite で特定範囲を更新します。これらは内部で lo_open, lo_lseek, lo_read/lo_write を使用します。 5. **エクスポート**: lo_export を使用してラージオブジェクトの内容をファイルに書き出します。 6. **終了**: END でトランザクションをコミットし、PQfinish で接続を閉じます。

直接依存

  • libpq-fe.h: libpq の主要なクライアントライブラリ機能。
  • libpq/libpq-fs.h: ラージオブジェクト操作用 API。
  • 標準 C ライブラリ: stdio.h, stdlib.h, fcntl.h, unistd.h（ファイルシステム操作用）。
  • sys/stat.h
  • sys/types.h

関連コード

  • src/test/examples/testlo64.c: 64ビットオフセットに対応したラージオブジェクト操作のサンプル。より大規模なデータやオフセットを扱う場合の参照先です。

読者向けメモ

  • 本プログラムは、ラージオブジェクトの操作がトランザクションのコンテキスト内で行われる必要があることを示す良い例です。
  • importFile や exportFile は手動実装の例ですが、実際のアプリケーションでは lo_import や lo_export といったライブラリ提供の関数を使用することが推奨されます。
  • エラーハンドリングは最小限であるため、本番環境のコードでは PQerrorMessage を活用したより堅牢なチェックが必要です。
