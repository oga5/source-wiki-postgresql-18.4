src/test/examples/testlo64.c

概要

PostgreSQL のラージオブジェクト（Large Object）機能に対し、64ビットオフセット API を使用した操作を検証するためのサンプルプログラムです。

モジュール内での役割

src/test/examples モジュールにおいて、ラージオブジェクトの境界値や大容量データに対する 64 ビット API（lo_lseek64, lo_truncate64 等）の動作を実証するテストコードとして機能します。

責務

  • ローカルファイルからデータベースへのラージオブジェクトのインポート・エクスポート。
  • 64 ビットオフセットを指定したデータの読み取り（pickout）と書き込み（overwrite）。
  • ラージオブジェクトのサイズ変更（my_truncate）。
  • トランザクション内での一連の操作の実行と接続の管理。

重要なシンボル

  • lo_lseek64: 64 ビットオフセットでのシーク操作。
  • lo_tell64: 現在の 64 ビット位置の取得。
  • lo_truncate64: 64 ビット長でのラージオブジェクトの切り詰め。
  • lo_import / lo_export: libpq が提供する標準的なインポート・エクスポート関数。

追加の索引済みシンボル

  • exit_nicely
  • exportFile
  • importFile
  • main
  • my_truncate
  • overwrite
  • pickout

制御フローとデータフロー

1. **接続確立**: PQsetdb でデータベースに接続し、セキュリティ設定（search_path）を適用。 2. **トランザクション開始**: BEGIN を発行し、一連の操作をアトミックに実行。 3. **操作実行**:

  • lo_import でファイルをラージオブジェクトとして保存。
  • pickout で 4GB を超えるオフセット位置からデータを読み取り。
  • overwrite で特定範囲を上書き。
  • my_truncate でサイズを縮小。

    • 4. **終了処理**: END でトランザクションをコミットし、PQfinish で接続を閉じる。

直接依存

  • libpq-fe.h: PostgreSQL クライアントライブラリの基本機能。
  • libpq/libpq-fs.h: ラージオブジェクト操作用 API。
  • 標準 C ライブラリ: stdint.h, stdio.h, stdlib.h, unistd.h 等（ファイル I/O および型定義）。
  • fcntl.h
  • sys/stat.h
  • sys/types.h

関連コード

  • src/test/examples/testlo.c: 64 ビット API を使用しない、基本的なラージオブジェクト操作のサンプル。
  • libpq ドキュメント: ラージオブジェクトのインターフェース仕様。

読者向けメモ

  • 本プログラムは 4GB を超えるオフセット（例: 4294967000U）を明示的に扱っています。32 ビット環境や古い API では正しく動作しない可能性があるため、64 ビット API の検証用途であることを念頭に置いてください。
  • importFile や exportFile は自作関数ですが、main 関数内ではより簡潔な lo_import / lo_export が優先的に使用されています。
  • エラーハンドリングは fprintf による標準エラー出力への書き出しと、必要に応じた exit_nicely による終了で構成されています。
