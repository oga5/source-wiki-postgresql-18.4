src/test/modules/test_rbtree/test_rbtree.c

概要

PostgreSQL の赤黒木（Red-Black Tree）ライブラリ (lib/rbtree.h) の正当性を検証するためのテストモジュールです。整数値をキーとする木構造を構築し、挿入、検索、削除、および走査の各操作が期待通りに動作するかを網羅的に確認します。

モジュール内での役割

本ファイルは src/test/modules/test_rbtree ディレクトリにおける唯一のソースであり、PostgreSQL の拡張機能としてビルドされます。SQL から test_rb_tree(int) 関数を呼び出すことで、内部的な検証ロジックを実行するテストスイートとして機能します。

責務

  • **データ構造の定義**: IntRBTreeNode を定義し、赤黒木ライブラリが要求するコールバック関数（比較、結合、メモリ確保・解放）を提供します。
  • **検証ロジックの提供**: 以下の操作の正確性を検証します。
  • 順方向・逆方向の走査（LeftRightWalk, RightLeftWalk）
  • 検索操作（rbt_find, rbt_find_less, rbt_find_great）
  • 削除操作（rbt_delete）および木が空になるまでの状態遷移
  • **ランダムテストの実行**: Fisher-Yates シャッフルを用いて挿入順序をランダム化し、エッジケースを網羅します。

重要なシンボル

  • IntRBTreeNode: テスト用のノード構造体。RBTNode を継承し、整数キーを保持します。
  • irbt_cmp / irbt_combine: 赤黒木ライブラリがノードの比較や重複キーの処理に使用するコールバック。
  • test_rb_tree: SQL から呼び出されるメインのエントリポイント。
  • rbt_populate: 指定されたサイズとステップで木を構築するユーティリティ。

制御フローとデータフロー

1. **初期化**: test_rb_tree が引数 size を受け取り、検証を開始します。 2. **構築**: create_int_rbtree で木を作成し、rbt_populate でランダムな順序でノードを挿入します。 3. **検証**: 各テスト関数（testfind, testdelete 等）が木に対して操作を行い、結果が期待値と一致するかを elog(ERROR, ...) でチェックします。 4. **終了**: 全てのテストが成功すれば PG_RETURN_VOID() で正常終了します。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • common/pg_prng.h: 乱数生成器（pg_global_prng_state）。
  • lib/rbtree.h: 検証対象である赤黒木ライブラリの API。
  • utils/memutils.h: メモリ管理（palloc, pfree）。
  • fmgr.h: PostgreSQL の関数管理インターフェース。

関連コード

  • src/include/lib/rbtree.h: 本テストが検証対象としている赤黒木ライブラリのヘッダーファイル。
  • src/backend/lib/rbtree.c: 赤黒木ライブラリの実装本体。

読者向けメモ

  • 本ファイルは、PostgreSQL の内部ライブラリが正しく実装されているかを検証する「ユニットテスト」の好例です。
  • rbt_find_less や rbt_find_great のテストでは、削除操作を途中で混ぜることで、木が動的に再構成される際の整合性も確認しています。
  • 乱数生成には pg_global_prng_state を使用しているため、実行ごとに異なる順序でテストが行われます。
