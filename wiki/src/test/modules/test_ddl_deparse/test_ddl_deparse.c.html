src/test/modules/test_ddl_deparse/test_ddl_deparse.c

概要

PostgreSQL の DDL デパース（deparse）機能のテストを支援するための拡張モジュールです。CollectedCommand 構造体に含まれる DDL 操作の内容を SQL レベルで検証可能にするためのユーティリティ関数を提供します。

モジュール内での役割

このファイルは、DDL イベントトリガー等でキャプチャされた CollectedCommand 構造体を、SQL クエリから検査するためのインターフェースを提供します。主に回帰テストにおいて、内部的な DDL 構造が期待通りに解析されているかを確認するために使用されます。

責務

  • CollectedCommand 型の内部状態（コマンド種別やタグ）の抽出。
  • ALTER TABLE コマンドに含まれるサブコマンドの詳細情報の列挙。
  • 内部的な Node 構造を、人間が読み取り可能なテキスト形式へ変換して SQL に返す。

重要なシンボル

  • get_command_type: CollectedCommand の type フィールドを文字列に変換します。
  • get_command_tag: CollectedCommand 内のパースツリーからコマンドタグ（例: CREATE TABLE）を生成します。
  • get_altertable_subcmdinfo: ALTER TABLE のサブコマンドリストを走査し、各操作の種類と対象オブジェクトをセット値として返します。

制御フローとデータフロー

1. **入力**: 各関数は PG_GETARG_POINTER(0) を介して CollectedCommand 構造体を受け取ります。 2. **処理**:

  • get_command_type / get_command_tag: 構造体内の列挙型やパースツリーを直接参照し、テキストに変換します。
  • get_altertable_subcmdinfo: InitMaterializedSRF を使用して SRF（Set Returning Function）として初期化し、foreach ループでサブコマンドリストを反復処理します。

    • 3. **出力**: tuplestore_putvalues を用いて、サブコマンドの種類とオブジェクト記述をタプルとして構築し、呼び出し元へ返します。

直接依存

  • postgres.h: 基本的な PostgreSQL 型定義。
  • tcop/deparse_utility.h: CollectedCommand 構造体および関連する定義。
  • tcop/utility.h: CreateCommandName 等のユーティリティ関数。
  • funcapi.h: SRF 実装のための API。
  • nodes/execnodes.h
  • utils/builtins.h

関連コード

  • src/backend/tcop/utility.c: DDL 実行のメインロジック。
  • src/include/tcop/deparse_utility.h: CollectedCommand の定義元。
  • src/backend/commands/tablecmds.c: ALTER TABLE のサブコマンド定義（AlterTableCmd 等）。

読者向けメモ

  • 本ファイルはテスト専用のモジュールであり、プロダクション環境での使用は想定されていません。
  • get_altertable_subcmdinfo は非常に長い switch 文を含んでいますが、これは AlterTableType の網羅的なマッピングを行っているためです。新しい ALTER TABLE 操作が追加された場合、ここにも追従が必要です。
  • CollectedCommand は内部構造体であるため、PostgreSQL のバージョンアップに伴い構造が変更される可能性がある点に注意してください。
