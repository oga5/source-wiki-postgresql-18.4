src/test/modules/plsample/plsample.c

概要

PostgreSQL の手続き型言語（PL）ハンドラを実装するための最小限のテンプレートです。新しい PL を開発する際の参照実装として機能し、関数呼び出しおよびトリガー呼び出しの基本的なインターフェースを提供します。

モジュール内での役割

plsample モジュールの唯一のソースファイルであり、PostgreSQL の fmgr（Function Manager）と連携して、SQL から呼び出された関数やトリガーの実行を仲介します。

責務

  • **ハンドラのディスパッチ**: plsample_call_handler を通じて、呼び出し元が通常の関数か、トリガーか、イベントトリガーかを判別し、適切なサブハンドラへ転送します。
  • **メタデータ取得**: pg_proc システムカタログから関数定義やソースコードを取得します。
  • **実行コンテキストの管理**: メモリコンテキストの作成や SPI（Server Programming Interface）の接続・切断を管理します。
  • **デバッグ出力**: 実行時に引数やトリガー情報を NOTICE レベルでログ出力します。

重要なシンボル

  • plsample_call_handler: PostgreSQL から呼び出されるエントリポイント。
  • plsample_func_handler: 通常の関数呼び出しを処理する内部関数。
  • plsample_trigger_handler: トリガー呼び出しを処理する内部関数。

制御フローとデータフロー

1. **エントリ**: plsample_call_handler が呼び出され、fcinfo->context を確認して呼び出し種別を判定します。 2. **メタデータ抽出**: SearchSysCache1(PROCOID, ...) を使用して、実行対象の関数情報を取得します。 3. **処理実行**:

  • 関数呼び出しの場合: 引数を解析し、OutputFunctionCall で値を変換して出力します。
  • トリガー呼び出しの場合: SPI_connect() で SPI を初期化し、TriggerData を通じてトリガーイベント（INSERT/UPDATE 等）を解析します。

    • 4. **終了**: PG_TRY/PG_FINALLY ブロックにより、エラー発生時でも安全にリソースを解放します。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • catalog/pg_proc.h, catalog/pg_type.h: システムカタログ定義。
  • executor/spi.h: トリガー処理のための SPI インターフェース。
  • utils/syscache.h, utils/lsyscache.h: システムキャッシュへのアクセス。
  • funcapi.h: 関数呼び出しのメタデータ処理。
  • commands/event_trigger.h
  • commands/trigger.h
  • utils/fmgrprotos.h

関連コード

  • src/pl/plpgsql/: 本格的な PL 実装の例。より複雑なコンパイルや実行キャッシュの仕組みを確認できます。
  • src/include/commands/trigger.h: トリガー関連のデータ構造定義。

読者向けメモ

  • 本ファイルは「サンプル」であり、実際の言語処理系（インタプリタやコンパイラ）は含まれていません。
  • plsample_func_handler 内の AllocSetContextCreate は、関数実行ごとのメモリ管理の雛形です。
  • トリガーハンドラ内で SPI_connect() を呼ぶ際は、必ず SPI_finish() を呼び出す必要があります。
  • イベントトリガーのハンドラは未実装（TODO）であるため、拡張する際は CALLED_AS_EVENT_TRIGGER の分岐を埋める必要があります。
