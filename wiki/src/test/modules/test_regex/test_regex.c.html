src/test/modules/test_regex/test_regex.c

概要

PostgreSQL の正規表現エンジン（Spencer's regex）の動作を検証するためのテスト用ハーネスです。SQL から呼び出し可能な関数 test_regex を提供し、正規表現のコンパイル結果やマッチングの詳細を配列として返します。

モジュール内での役割

src/test/modules/test_regex モジュールの中心となる実装ファイルです。Tcl の正規表現テストスイートの挙動を模倣するように設計されており、正規表現エンジンのデバッグや回帰テストを目的としています。

責務

  • **正規表現のコンパイルと実行**: pg_regcomp および pg_regexec をラップし、テスト用のフラグ設定を適用する。
  • **テスト結果の構造化**: マッチング位置や正規表現のメタ情報（re_info）を SQL で扱いやすい text[] 形式に変換する。
  • **メモリ管理**: SRF（Set Returning Function）のコンテキストを利用し、複数行にわたるマッチング結果を効率的に保持する。

重要なシンボル

  • test_regex: SQL から呼び出されるメインの SRF。
  • test_regex_ctx: 複数回の呼び出し間でマッチング状態（位置情報やフラグ）を保持するコンテキスト構造体。
  • test_re_flags: 正規表現のコンパイルおよび実行オプションを格納する構造体。
  • parse_test_flags: テスト用の文字列フラグ（g, i, n など）を内部的な cflags/eflags に変換するパーサー。

制御フローとデータフロー

1. **初期化 (SRF_IS_FIRSTCALL)**:

  • parse_test_flags でオプションを解析。
  • test_re_compile でパターンをコンパイル。
  • setup_test_matches で対象文字列全体に対してマッチングを実行し、結果を test_regex_ctx に格納。
  • 最初の行として正規表現のメタ情報（build_test_info_result）を返す。

    • 2. **反復処理**:

  • test_regex が再呼び出しされるたびに、test_regex_ctx から次のマッチング結果を build_test_match_result で構築して返す。

    • 3. **終了**:

  • 全てのマッチングが完了すると SRF_RETURN_DONE を返す。

直接依存

  • utils/array.h / utils/builtins.h: 配列構築および文字列操作用ユーティリティ。

関連コード

  • src/backend/regex/: 正規表現エンジンの本体。
  • src/backend/utils/adt/regexp.c: 本番環境用の正規表現関数（regexp_matches 等）の実装。比較対象として有用。

読者向けメモ

  • 本ファイルはデバッグ目的のため、通常の正規表現関数よりも詳細な情報（マッチングのインデックスや内部フラグの状態）を返すように設計されています。
  • parse_test_flags 内のフラグ定義は Tcl の正規表現テストスイートと互換性を持つよう意図されています。
  • test_re_compile は内部で pg_wchar への変換を行っており、マルチバイト文字を扱う際の挙動を理解する上で参考になります。
