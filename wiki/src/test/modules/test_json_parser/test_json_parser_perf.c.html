src/test/modules/test_json_parser/test_json_parser_perf.c

概要

PostgreSQLのJSONパーサー（再帰下降型とインクリメンタル型）の純粋な解析速度を比較・測定するためのベンチマーク用スタンドアロンプログラムです。

モジュール内での役割

test_json_parser モジュールにおいて、解析アルゴリズム間のパフォーマンス差異を評価する役割を担います。test_json_parser_incremental.c がチャンク分割による動作検証を主目的とするのに対し、本ファイルは入力全体を一括で渡すことで、パーサー自体の計算コストを測定します。

責務

  • 指定されたJSONファイルをメモリ上に読み込む。
  • コマンドライン引数に基づき、再帰下降型パーサーまたはインクリメンタル型パーサーのいずれかを選択する。
  • 指定された回数だけ解析処理を繰り返し実行し、実行時間を測定可能な状態にする。
  • 解析結果の正当性を検証し、エラー発生時には終了する。

重要なシンボル

  • main: プログラムのエントリポイント。引数解析、ファイル読み込み、ループ実行を制御します。
  • BUFSIZE: ファイル読み込み時のバッファサイズ（6000バイト）。
  • pg_parse_json: 再帰下降型パーサーの実行関数。
  • pg_parse_json_incremental: インクリメンタル型パーサーの実行関数。
  • nullSemAction: 解析結果に対して何もしない（パースのみを行う）セマンティックアクション構造体。

制御フローとデータフロー

1. **初期化**: pg_logging_init でログを初期化し、StringInfo を準備します。 2. **入力**: コマンドライン引数から反復回数と対象ファイルを取得し、ファイル内容を StringInfo バッファへ全量読み込みます。 3. **ループ実行**: 指定された回数分、以下の処理を繰り返します。

  • use_inc フラグに応じて makeJsonLexContext... でコンテキストを生成。
  • 各パーサー関数を呼び出し、nullSemAction を渡して解析を実行。
  • freeJsonLexContext でリソースを解放。

    • 4. **終了**: 全反復終了後、正常終了します。

直接依存

  • postgres_fe.h: PostgreSQLフロントエンド用ヘッダー。
  • common/jsonapi.h: JSON解析API定義。
  • common/logging.h: エラーログ出力用。
  • lib/stringinfo.h: 動的文字列バッファ操作。
  • mb/pg_wchar.h: マルチバイト文字処理。
  • stdio.h, string.h: 標準入出力および文字列操作。

関連コード

  • src/test/modules/test_json_parser/test_json_parser_incremental.c: インクリメンタルパーサーの機能テスト用コード。
  • src/backend/utils/adt/jsonfuncs.c: パーサーの実装本体（バックエンド側）。

読者向けメモ

  • 本プログラムは「純粋な解析速度」を測定するため、インクリメンタルパーサーを使用する場合でも、チャンク分割を行わず全データを一度に渡しています。
  • -i オプションの有無でパーサーを切り替えます。
  • 解析結果のセマンティクス（意味解析）は nullSemAction により無効化されているため、純粋に構文解析のオーバーヘッドのみが測定対象となります。
