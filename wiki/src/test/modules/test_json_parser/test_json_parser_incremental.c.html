src/test/modules/test_json_parser/test_json_parser_incremental.c

概要

PostgreSQL のインクリメンタル JSON パーサーの動作を検証するためのテストプログラムです。入力ファイルを小さなチャンクに分割してパーサーに供給することで、パーサーが断片的なデータ入力を正しく処理できるかを確認します。

モジュール内での役割

src/test/modules/test_json_parser モジュールにおいて、パーサーの堅牢性をテストする主要な実行ファイルです。特に、lexer が不完全なチャンクをどのように扱うか、またメモリ管理（トークンの所有権）が適切に行われているかを検証します。

責務

  • **チャンク分割処理**: 指定されたサイズ（デフォルト 60 バイト）でファイルを読み込み、パーサーへ逐次供給する。
  • **セマンティック処理のシミュレーション**: -s オプションにより、パース結果を標準出力へ再構成（エコーバック）し、構造の整合性を検証する。
  • **メモリリーク検証**: -o オプションにより JSONLEX_CTX_OWNS_TOKENS を有効化し、メモリリーク検知ツールとの併用を可能にする。
  • **ストレステスト**: -r オプションによるチャンクサイズの動的変更と反復実行。

重要なシンボル

  • DoState: パース中の状態（バッファや要素順序）を保持する構造体。
  • pg_parse_json_incremental: インクリメンタルパースを実行するコア関数。
  • JsonSemAction sem: パースイベント（オブジェクト開始、スカラ値など）を処理するコールバック関数のテーブル。
  • lex_owns_tokens: トークンのメモリ解放責任をパーサー側に持たせるかを示すフラグ。

制御フローとデータフロー

1. **初期化**: getopt でオプションを解析し、JsonLexContext を準備。 2. **ループ処理**:

  • ファイルを chunk_size 分だけ読み込み、appendBinaryStringInfo でバッファに追加。
  • pg_parse_json_incremental を呼び出し、JSON_INCOMPLETE が返る限り入力を継続。
  • 最終チャンクで is_last = true を指定してパースを完了させる。

    • 3. **セマンティック処理**: パース中に do_object_start や do_scalar などのコールバックが呼び出され、構造を再構築して出力する。 4. **終了処理**: freeJsonLexContext でリソースを解放。

読者向けメモ

  • **テストの意図**: このプログラムは、意図的に「ゴミデータ（trailing junk）」をバッファに混入させています。これは、パーサーがチャンク境界を正しく認識し、不要なデータに惑わされずにパースを継続できるかをテストするためです。
  • **デバッグ**: パースエラーが発生した際は、json_errdetail が詳細なエラー位置と理由を報告するため、これを参照してください。
  • **注意**: -s オプション使用時は、出力結果を jq などの外部ツールにパイプすることで、パース結果の妥当性を容易に確認できます。
