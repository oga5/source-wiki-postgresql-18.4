src/test/modules/test_integerset/test_integerset.c

概要

PostgreSQL の IntegerSet データ構造の機能と堅牢性を検証するためのテストモジュールです。SQL から呼び出し可能な関数を通じて、様々なパターンや境界条件における整数集合の追加、検索、およびイテレーションの正確性を確認します。

モジュール内での役割

src/test/modules/test_integerset ディレクトリにおける唯一のソースファイルであり、lib/integerset.h で定義された API の動作を保証するテストスイートの本体です。

責務

  • **機能検証**: IntegerSet の基本操作（追加、メンバ判定、イテレーション）の正当性確認。
  • **境界値テスト**: 空集合、単一値、および uint64 の最大値付近の極端な値のテスト。
  • **負荷・パターンテスト**: 疎なデータや、Simple-8b エンコーディングの限界（2^60）を超える距離を持つデータに対する動作検証。
  • **統計情報の提供**: 必要に応じて、メモリ使用量や処理時間のベンチマーク情報を出力する機能の提供。

重要なシンボル

  • test_integerset: SQL から呼び出されるメインのエントリポイント。
  • test_spec: テストパターンを定義する構造体。
  • test_pattern(): 定義されたパターンに基づいて IntegerSet を構築・検証する関数。
  • intset_test_stats: 統計出力の有効/無効を切り替えるフラグ。

追加の索引済みシンボル

  • check_with_filler
  • test_empty
  • test_huge_distances

  • test_single_value_and_filler

制御フローとデータフロー

1. **初期化**: test_integerset が呼び出されると、まず test_empty や test_single_value などの小規模なテストを実行します。 2. **パターン実行**: test_specs 配列をループし、test_pattern を呼び出します。 3. **データ構築**: test_pattern 内で IntegerSet を作成し、指定されたビットパターンに従って intset_add_member を繰り返し呼び出します。 4. **検証**:

  • intset_is_member を用いたランダムアクセスによる存在確認。
  • intset_begin_iterate と intset_iterate_next を用いた全走査による順序と値の整合性確認。

    • 5. **クリーンアップ**: MemoryContextDelete を使用して、テスト用に確保したメモリコンテキストを解放します。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • lib/integerset.h: テスト対象のデータ構造 API。
  • common/pg_prng.h: テスト用の乱数生成。
  • utils/memutils.h: メモリコンテキスト管理。
  • utils/timestamp.h: 処理時間計測。
  • fmgr.h: SQL 呼び出しインターフェース。

関連コード

  • src/include/lib/integerset.h: テスト対象の API 定義。
  • src/backend/lib/integerset.c: （実装本体）本テストが検証対象とするロジック。

読者向けメモ

  • 本ファイルは、IntegerSet の内部実装（特にバッファリングや B-tree 構造へのフラッシュ）を網羅的にテストするように設計されています。
  • intset_test_stats を true に変更することで、デバッグやパフォーマンスチューニング時に詳細なメモリ使用量や処理時間を確認できます。
  • test_huge_distances は、Simple-8b エンコーディングの制約（2^60）を意識した重要なテストケースです。実装を変更する際は、この境界条件が正しく処理されているかを確認してください。
