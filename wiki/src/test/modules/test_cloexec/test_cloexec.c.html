src/test/modules/test_cloexec/test_cloexec.c

概要

Windows環境における O_CLOEXEC フラグの動作を検証するための単一ファイルテストプログラムです。ファイル記述子（FD）に O_CLOEXEC を付与して開いた場合、子プロセスにハンドルが継承されないことを確認します。

モジュール内での役割

src/test/modules/test_cloexec モジュールの唯一のソースコードであり、PostgreSQLの移植性レイヤーがWindowsのプロセス継承制御を正しく扱えているかをテストする回帰テストの一部です。

責務

  • **親プロセスの実行**: O_CLOEXEC あり/なしの2つのファイルを開き、そのOSハンドルを子プロセスへ引数として渡して起動する。
  • **子プロセスの実行**: 親から渡されたハンドルに対して書き込みを試行し、継承の可否を判定する。
  • **検証結果の出力**: 期待通りの継承制御が行われているかを標準出力および終了コードで報告する。

重要なシンボル

  • main: 引数の有無により、親プロセス（テスト実行）と子プロセス（検証実行）の役割を切り替えるエントリポイント。
  • run_parent_tests: ファイルを開き、CreateProcess で子プロセスを生成する。bInheritHandles を TRUE に設定するのが肝要。
  • run_child_tests: 親から渡されたハンドル文字列を解析し、書き込みテストを実行する。
  • try_write_to_handle: WriteFile API を用いて、特定のハンドルへの書き込みが可能かを確認する。

制御フローとデータフロー

1. **起動**: 引数なしで実行されると親プロセスとして動作。 2. **準備**: open() で2つのファイルを作成。一方は O_CLOEXEC を付与。 3. **継承**: _get_osfhandle() で取得したハンドルを文字列化し、CreateProcess で自身を子プロセスとして再起動。 4. **検証**: 子プロセスは受け取ったハンドルに対し WriteFile を実行。

  • O_CLOEXEC ありのハンドルへの書き込みは失敗すべき。
  • O_CLOEXEC なしのハンドルへの書き込みは成功すべき。

    • 5. **終了**: 子プロセスの終了コードを親が確認し、テストの成否を判定。

直接依存

  • postgres.h: PostgreSQLの基本定義。
  • fcntl.h, sys/stat.h: ファイル操作用。
  • windows.h: Windows API (CreateProcess, HANDLE 等) の利用に必須。
  • common/file_utils.h, port.h: PostgreSQLの移植性ユーティリティ。

関連コード

  • src/include/port.h: O_CLOEXEC の定義や移植性に関するマクロが定義されています。
  • src/port/: Windows固有のファイル操作実装が含まれており、本テストの検証対象となるロジックが格納されています。

読者向けメモ

  • 本テストは **Windows専用** です。#ifndef WIN32 により、他のプラットフォームでは何も行わずに終了します。
  • CreateProcess の bInheritHandles 引数が TRUE であることが、このテストの前提条件です。これが FALSE だと、O_CLOEXEC の有無に関わらず継承されないため、テストの意味を成しません。
  • _get_osfhandle は、PostgreSQLのファイル記述子からWindowsのネイティブハンドルを取得するために不可欠な関数です。
