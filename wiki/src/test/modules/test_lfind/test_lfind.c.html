src/test/modules/test_lfind/test_lfind.c

概要

PostgreSQLの最適化された線形探索関数（pg_lfind系）の正確性を検証するためのテストモジュールです。SIMD命令等を利用した探索ロジックが、境界条件やデータ長の変化に対して正しく動作することを保証します。

モジュール内での役割

本ファイルは src/test/modules/test_lfind ディレクトリの唯一のソースであり、PostgreSQLの拡張機能としてロード可能なテスト用関数を提供します。SQLインターフェースを通じて、コアライブラリの探索関数の回帰テストを実行する役割を担います。

責務

  • **探索ロジックの検証**: pg_lfind8（8ビット値の検索）および pg_lfind8_le（8ビット値の以下検索）、pg_lfind32（32ビット値の検索）の動作確認。
  • **境界条件の網羅**: ベクトル演算（SIMD）が適用される範囲と、末尾の端数処理（テール処理）が発生する範囲の両方をテスト。
  • **エラー報告**: 期待値と異なる結果が得られた場合に elog(ERROR) を発行し、テスト失敗を通知。

重要なシンボル

  • test_lfind8, test_lfind8_le, test_lfind32: SQLから呼び出し可能なテスト用関数。
  • test_lfind8_internal, test_lfind8_le_internal: 8ビット探索のテストロジックを共通化したヘルパー関数。
  • LEN_NO_TAIL, LEN_WITH_TAIL: ベクトルサイズを考慮したテスト用バッファ長を計算するマクロ。

制御フローとデータフロー

1. **初期化**: テスト関数が呼び出されると、Vector8（または uint32 配列）に基づいたバッファを確保します。 2. **データ配置**: memset でバッファを初期化し、特定のインデックスに検索対象のキーを配置します。 3. **探索実行**: pg_lfind 系関数を呼び出し、戻り値（bool）を検証します。 4. **検証**:

  • 存在すべき値が見つからない場合、または存在しない値が見つかった場合にエラーを発生させます。
  • ベクトル演算の境界（len_no_tail）と、端数を含む範囲（len_with_tail）の両方でテストを行い、ロジックの堅牢性を確認します。

直接依存

  • postgres.h: PostgreSQLの基本型およびエラー報告機能。
  • fmgr.h: PostgreSQLの関数管理インターフェース（PG_FUNCTION_INFO_V1 等）。
  • port/pg_lfind.h: 検証対象である最適化された線形探索関数の定義。

関連コード

  • src/include/port/pg_lfind.h: 本テストが検証している実際の探索アルゴリズムの実装。
  • src/backend/port/pg_lfind.c: （存在する場合）各プラットフォーム固有の最適化実装。

読者向けメモ

  • 本ファイルはユニットテストとして設計されています。実行には CREATE EXTENSION を使用してモジュールをロードし、SQL経由で関数を呼び出す必要があります。
  • LEN_NO_TAIL はSIMDレジスタのサイズを意識した計算を行っています。テストケースを追加する際は、ベクトル境界をまたぐようなインデックス指定を考慮してください。
  • elog(ERROR) を使用しているため、テスト失敗時はトランザクションがアボートされます。CI環境等での利用を想定しています。
