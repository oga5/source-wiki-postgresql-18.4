src/test/modules/test_shm_mq/worker.c

概要

PostgreSQLのバックグラウンドワーカー（Background Worker）機能を利用し、共有メモリメッセージキュー（shm_mq）を介してデータの送受信を行うサンプル実装です。

モジュール内での役割

test_shm_mq モジュールにおける「ワーカープロセス」の本体です。メインプロセス（バックエンド）が作成した共有メモリセグメントに接続し、キューを介してメッセージを中継する役割を担います。

責務

  • **初期化**: DSM（動的共有メモリ）への接続と、shm_toc を用いた制御構造の取得。
  • **同期**: 自身のワーカー番号の取得と、準備完了をメインプロセスへ通知（Latch経由）。
  • **データ処理**: メッセージキューからの読み込みと、別キューへの書き出し（copy_messages）。
  • **終了処理**: シグナルハンドラの適切な設定と、共有メモリからの安全なデタッチ。

重要なシンボル

  • test_shm_mq_main: ワーカープロセスのエントリポイント。
  • attach_to_queues: ワーカー番号に基づき、入力用および出力用のメッセージキューを特定・接続する。
  • copy_messages: メッセージの受信と転送をループ実行するメインロジック。

制御フローとデータフロー

1. **起動**: test_shm_mq_main が起動し、DSMセグメントをアタッチ。 2. **準備**: shm_toc から制御構造を取得し、workers_attached をインクリメントして自身のIDを確定。 3. **接続**: attach_to_queues により、自身のIDに対応する入力キューと出力キューを shm_mq_attach で開く。 4. **通知**: SetLatch を呼び出し、メインプロセスに準備完了を伝える。 5. **ループ**: copy_messages 内で shm_mq_receive と shm_mq_send を繰り返し、データが途切れるまで処理を継続。 6. **終了**: dsm_detach を経てプロセス終了。

直接依存

  • storage/shm_mq.h: メッセージキューの操作。
  • storage/shm_toc.h: 共有メモリ内のデータ構造検索。
  • test_shm_mq.h: モジュール共通のヘッダー（test_shm_mq_header 等）。
  • postgres.h / miscadmin.h: PostgreSQLの基本環境。
  • storage/ipc.h
  • storage/procarray.h
  • tcop/tcopprot.h

関連コード

  • src/test/modules/test_shm_mq/test.c: ワーカーを起動し、メッセージを投入する側の実装。
  • src/backend/storage/ipc/shm_mq.c: 本ファイルが利用するメッセージキューの基盤実装。

読者向けメモ

  • 本ファイルは、バックグラウンドワーカーが共有メモリを介してどのように協調動作するかを示すテンプレートとして設計されています。
  • CHECK_FOR_INTERRUPTS() がループ内に配置されており、SIGTERM 等のシグナルによる停止要求を適切に処理するようになっています。
  • 実際のアプリケーション開発時には、copy_messages 内のロジックを任意の計算処理に置き換えることで、並列計算ワーカーとして拡張可能です。
