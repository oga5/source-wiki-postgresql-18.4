src/test/modules/test_radixtree/test_radixtree.c

概要

PostgreSQL の適応型基数木（Adaptive Radix Tree: ART）実装の正当性を検証するためのテストモジュールです。基数木の各ノードクラス（node-4, 16, 48, 256）の動作、挿入・検索・削除の整合性、およびイテレータの順序性を確認します。

モジュール内での役割

src/test/modules/test_radixtree ディレクトリにおける唯一のソースファイルであり、基数木ライブラリ（lib/radixtree.h）の機能テストを実行するエントリポイントです。

責務

  • **ノードクラスの網羅的テスト**: 各ノードサイズへの遷移を誘発するキー数を用いた挿入・更新・削除の検証。
  • **整合性チェック**: EXPECT_TRUE 等の独自マクロを用いた、期待値と実測値の厳密な比較。
  • **ランダム負荷テスト**: 大量のランダムなキーを用いた挿入・検索・削除のシーケンス実行による堅牢性の確認。
  • **共有メモリ対応**: TEST_SHARED_RT マクロの切り替えにより、共有メモリ（DSA）上での動作検証をサポート。

重要なシンボル

  • test_radixtree: SQL から呼び出されるメインのテスト関数。
  • rt_node_class_tests: 各ノードクラスのテストに必要なキー数を定義した配列。
  • test_basic: 特定のノードクラスとシフト量に対して、基本的な CRUD 操作を検証する関数。
  • test_random: 擬似乱数を用いた大規模なデータセットによるストレステスト関数。
  • EXPECT_* マクロ: テスト失敗時にエラーログを出力し、即座に処理を停止させる検証用ユーティリティ。

制御フローとデータフロー

1. **初期化**: test_radixtree が呼び出されると、まず test_empty で空の木に対する操作を検証します。 2. **構造テスト**: rt_node_class_tests をループし、test_basic を通じて異なる深さ（シフト量）と順序（昇順・降順）で木を構築・検証します。 3. **負荷テスト**: test_random が実行され、pg_prng で生成されたキーを挿入後、ソートされたキー配列と比較して検索・イテレーションの正当性を確認します。 4. **終了**: 各テスト関数内で rt_free を呼び出し、メモリリークを防ぎつつ検証を完了します。

直接依存

  • lib/radixtree.h: テスト対象の基数木実装。
  • postgres.h, fmgr.h: PostgreSQL のコア機能および関数インターフェース。
  • utils/memutils.h: メモリコンテキスト管理。
  • common/pg_prng.h: ランダムテスト用の擬似乱数生成器。
  • common/int.h
  • utils/timestamp.h

関連コード

  • src/include/lib/radixtree.h: 基数木のデータ構造定義と API 仕様。
  • src/backend/lib/radixtree.c: 基数木の実装本体。

読者向けメモ

  • 本ファイルは lib/radixtree.h のマクロ定義（RT_DECLARE, RT_DEFINE 等）を利用して、テスト用の基数木インスタンスを生成しています。
  • TEST_SHARED_RT を有効にすると、共有メモリ（DSA）を利用したテストに切り替わります。開発環境でデバッグを行う際は、このマクロの有無に注意してください。
  • EXPECT_* マクロは elog(ERROR) を発行するため、テスト失敗時はトランザクションがアボートされます。
