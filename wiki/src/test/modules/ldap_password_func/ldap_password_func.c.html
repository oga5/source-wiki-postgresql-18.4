src/test/modules/ldap_password_func/ldap_password_func.c

概要

本ファイルは、PostgreSQLのLDAP認証プロセスにおいて、LDAPバインドパスワードを動的に変換（ミューテート）するためのロード可能なモジュールです。主にテスト目的で、パスワードに対してROT13変換を適用するフックを提供します。

モジュール内での役割

src/test/modules/ldap_password_func モジュールの唯一のソースファイルであり、PostgreSQLの認証フック機構を利用して、外部から提供されたパスワードを加工する役割を担います。

責務

  • **フックの登録**: モジュールロード時に ldap_password_hook へ独自の変換関数を割り当てる。
  • **パスワード変換**: 入力されたパスワード文字列に対してROT13アルゴリズムを適用し、変換後の文字列をメモリ上に確保して返却する。

重要なシンボル

  • _PG_init: モジュールロード時に実行される初期化関数。ldap_password_hook を設定します。
  • rot13_passphrase: ldap_password_hook に登録される静的関数。パスワードの各文字をROT13で置換します。
  • ldap_password_hook: libpq/auth.h で定義されたグローバルな関数ポインタ。

制御フローとデータフロー

1. **初期化**: PostgreSQLがモジュールをロードすると _PG_init が呼び出され、ldap_password_hook に rot13_passphrase のアドレスが格納されます。 2. **フック実行**: LDAP認証時にパスワードが必要になると、コアシステムが ldap_password_hook を介して rot13_passphrase を呼び出します。 3. **変換処理**: rot13_passphrase は palloc でメモリを確保し、入力文字列をコピーした後、アルファベットを13文字分シフトさせて変換します。 4. **返却**: 変換済みの新しい文字列ポインタがコアシステムへ返され、認証処理に使用されます。

直接依存

  • postgres.h: PostgreSQLの基本定義。
  • fmgr.h: 関数マネージャおよびモジュール初期化用マクロ。
  • libpq/auth.h: ldap_password_hook の定義元。
  • float.h, stdio.h: 標準ライブラリ。

関連コード

  • src/backend/libpq/auth.c: ldap_password_hook が実際に呼び出される認証ロジックの本体。
  • src/include/libpq/auth.h: フックの定義とインターフェース仕様。

読者向けメモ

  • 本モジュールは実運用環境での使用を想定したものではなく、認証フックの動作検証やテストを目的としています。
  • palloc を使用しているため、メモリ管理はPostgreSQLのメモリコンテキストに依存します。
  • ROT13変換はアルファベットのみを対象としており、記号や数字はそのまま保持される点に注意してください。
