src/test/modules/test_bloomfilter/test_bloomfilter.c

概要

PostgreSQLのブルームフィルタ実装（src/include/lib/bloomfilter.h）の精度を検証するためのテスト用モジュールです。SQLから呼び出し可能な関数を提供し、指定されたパラメータに基づいてブルームフィルタの偽陽性率を測定します。

モジュール内での役割

本ファイルは src/test/modules/test_bloomfilter ディレクトリにおける唯一のソースコードであり、ブルームフィルタの統計的な妥当性を確認するための回帰テストおよび性能評価の基盤として機能します。

責務

  • **ブルームフィルタの構築と投入**: 指定された要素数とメモリ量でフィルタを初期化し、ダミーデータを投入する。
  • **偽陽性率の測定**: フィルタに存在しないことが保証されたデータセットを検査し、誤って「存在あり」と判定された回数をカウントする。
  • **閾値判定**: 偽陽性率が定義された閾値（1%）を超えた場合に警告（WARNING）を発行する。

重要なシンボル

  • test_bloomfilter: SQLから呼び出されるメインのエントリポイント。テストの実行回数やフィルタサイズを制御する。
  • create_and_test_bloom: 個別のテストケースを実行する内部関数。フィルタの生成から結果の出力までを統括する。
  • populate_with_dummy_strings: フィルタにテスト用の文字列を投入する。
  • nfalsepos_for_missing_strings: 偽陽性の発生数をカウントする。
  • FPOSITIVE_THRESHOLD: 偽陽性率の許容上限（0.01）。

制御フローとデータフロー

1. **初期化**: test_bloomfilter がSQL引数を受け取り、パラメータの妥当性を検証する。 2. **ループ実行**: 指定された回数分 create_and_test_bloom を呼び出す。 3. **フィルタ構築**: bloom_create でメモリを確保し、populate_with_dummy_strings で「i+数値」形式の文字列を投入する。 4. **検証**: nfalsepos_for_missing_strings で「M+数値」形式の文字列を検査し、偽陽性を集計する。 5. **報告**: 偽陽性率を計算し、ereport を介してログまたは警告を出力する。 6. **終了**: bloom_free でメモリを解放する。

直接依存

  • postgres.h: PostgreSQLの基本定義。
  • common/pg_prng.h: 乱数生成器（シード値の生成に使用）。
  • fmgr.h: SQL関数呼び出しインターフェース。
  • lib/bloomfilter.h: テスト対象であるブルームフィルタのAPI。
  • miscadmin.h: CHECK_FOR_INTERRUPTS() 等の管理用マクロ。

関連コード

  • src/include/lib/bloomfilter.h: ブルームフィルタのデータ構造と関数定義。
  • src/backend/lib/bloomfilter.c: ブルームフィルタの実際のアルゴリズム実装。

読者向けメモ

  • 本テストは、ブルームフィルタのビット配列サイズ（power引数）が23から32の範囲であることを要求します。
  • CHECK_FOR_INTERRUPTS() がループ内に配置されているため、長時間実行されるテストでもSQL側からキャンセル可能です。
  • 偽陽性率は確率的な事象であるため、テスト結果が閾値付近で変動する可能性がある点に留意してください。
