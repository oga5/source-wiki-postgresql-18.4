src/test/modules/test_dsa/test_dsa.c

概要

PostgreSQLの動的共有メモリ領域（DSA: Dynamic Shared Memory Area）機能の動作検証を行うためのテストモジュールです。SQL関数として呼び出し可能な形式で、DSAの基本的な割り当て、リソースオーナー管理、およびセグメント生成ロジックを検証します。

モジュール内での役割

src/test/modules/test_dsa ディレクトリの唯一のソースファイルであり、DSAサブシステムのストレステストおよび機能確認用インターフェースを提供します。

責務

  • **基本機能の検証**: DSAの作成、メモリ割り当て、データ書き込み、読み取り、解放、デタッチの一連のライフサイクルをテストする。
  • **リソースオーナー管理の検証**: ResourceOwner を切り替えた際のDSA割り当てと、オーナー解放時の挙動を確認する。
  • **セグメント生成ロジックの検証**: make_new_segment() 内のページマップサイズ計算ロジックを、異なるサイズでの割り当てを通じてテストする。

重要なシンボル

  • test_dsa_basic: DSAの基本的なCRUD操作を検証する関数。
  • test_dsa_resowners: リソースオーナーの入れ替えと、それに伴うメモリ管理の整合性を検証する関数。
  • test_dsa_allocate: 指定されたページ数でDSAを繰り返し作成・解放し、セグメント生成の境界条件をテストする関数。

制御フローとデータフロー

各関数は以下の共通パターンに従います。 1. **初期化**: LWLockNewTrancheId() でロックトランチを確保し、dsa_create() で領域を生成。 2. **操作**: dsa_allocate() でメモリを確保し、dsa_get_address() でポインタを解決してデータを操作。 3. **終了処理**: dsa_free() でメモリを解放し、dsa_detach() で領域を破棄。 ※ test_dsa_resowners では、CurrentResourceOwner を一時的に変更し、リソース解放プロセスを明示的に呼び出すフローが含まれます。

直接依存

  • utils/dsa.h: DSAのコアAPI。
  • storage/lwlock.h: DSAで使用するロックトランチの管理。
  • utils/resowner.h: リソースオーナーの管理と解放処理。
  • storage/dsm_registry.h: DSM（動的共有メモリ）の登録。
  • postgres.h, fmgr.h: PostgreSQLの標準的なモジュール基盤。
  • utils/freepage.h

関連コード

  • src/backend/utils/mmgr/dsa.c: DSAの実際の内部実装。本ファイルのテスト対象。
  • src/backend/storage/ipc/dsm.c: DSAの基盤となる動的共有メモリ管理。

読者向けメモ

  • **メモリリークの警告**: コード内のコメントにある通り、各関数内で LWLockNewTrancheId() によって確保されたトランチは明示的に解放されず、テスト実行プロセス終了までリークします。これはテストモジュールとしての簡便性を優先した実装です。
  • **テストの目的**: 本ファイルはDSAのAPI使用例としても有用ですが、主に内部の複雑なメモリ管理ロジック（特にセグメントの動的拡張）が正しく動作することを保証するための回帰テストとして機能します。
