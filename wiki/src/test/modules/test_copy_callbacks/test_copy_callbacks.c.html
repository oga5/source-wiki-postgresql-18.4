src/test/modules/test_copy_callbacks/test_copy_callbacks.c

概要

PostgreSQLの COPY TO コマンド実行時に、データ出力先としてカスタムコールバック関数を登録・利用する機能を検証するためのテスト用拡張モジュールです。

モジュール内での役割

本モジュールは、PostgreSQLのコア機能である COPY コマンドの拡張インターフェース（コールバック機構）が正しく動作することを検証する単一のテスト用Cソースファイルです。

責務

  • COPY TO 実行時に呼び出されるコールバック関数（to_cb）の実装。
  • SQL関数 test_copy_to_callback を提供し、指定されたテーブルに対して COPY TO を実行しつつ、カスタムコールバックを介したデータ処理フローを実証する。

重要なシンボル

  • to_cb: CopyToState に登録されるコールバック関数。受信したデータを NOTICE レベルでログ出力する。
  • test_copy_to_callback: SQLから呼び出されるエントリポイント。BeginCopyTo / DoCopyTo / EndCopyTo の一連のライフサイクルを管理する。

制御フローとデータフロー

1. **初期化**: test_copy_to_callback が対象テーブルをオープンし、BeginCopyTo を呼び出す。この際、引数として to_cb 関数ポインタを渡す。 2. **実行**: DoCopyTo が内部でテーブルをスキャンし、各行のデータを to_cb に順次渡す。 3. **終了**: EndCopyTo によりリソースが解放され、処理行数がログ出力される。

直接依存

  • postgres.h: PostgreSQLの基本定義。
  • access/table.h: テーブルアクセスのためのインターフェース。
  • commands/copy.h: COPY コマンドの内部API（CopyToState 等）。
  • fmgr.h: PostgreSQLの関数管理インターフェース。
  • utils/rel.h: リレーション（テーブル）操作用ユーティリティ。

関連コード

  • src/backend/commands/copyto.c: COPY TO のコア実装。本テストが検証対象としている BeginCopyTo 等の定義元。
  • src/include/commands/copy.h: COPY 関連のデータ構造定義。

読者向けメモ

  • 本ファイルは、PostgreSQLの拡張機能開発者が「カスタム出力先」を実装する際の最小構成例として参照できます。
  • to_cb 内のデータは char * として扱われていますが、実際の COPY 出力形式（CSV, Text, Binary）に応じたデータ構造の解釈には注意が必要です。
  • テスト実行時は、本モジュールをビルドした上で、SQLから test_copy_to_callback(regclass) を呼び出すことで動作を確認できます。
