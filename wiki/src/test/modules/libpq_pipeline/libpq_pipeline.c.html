src/test/modules/libpq_pipeline/libpq_pipeline.c

概要

本ファイルは、PostgreSQL の libpq が提供するパイプラインモード（非同期的なクエリ送信と結果取得）の機能検証を行うためのテストスイートです。単一の実行ファイルとして、パイプラインの正常系、異常系、キャンセル、プロトコルバージョン、および非ブロッキング I/O との統合動作を網羅的にテストします。

モジュール内での役割

src/test/modules/libpq_pipeline モジュールのメインロジックを担います。libpq のパイプライン API を利用するクライアントプログラムの「リファレンス実装」としての側面を持ち、API の正しい使用順序やエラーハンドリングのパターンを提示します。

責務

  • **パイプライン制御**: PQenterPipelineMode / PQexitPipelineMode を用いた状態遷移の検証。
  • **非同期 I/O**: select や PQflush を組み合わせた非ブロッキング処理の検証。
  • **エラーハンドリング**: PGRES_PIPELINE_ABORTED 状態からの復旧および同期処理の検証。
  • **クエリキャンセル**: PQcancel 関連 API を用いたクエリ中断の検証。
  • **プロトコル検証**: max_protocol_version オプションによる接続ネゴシエーションの検証。

重要なシンボル

テスト実行関数

  • test_cancel, test_pipelined_insert, test_pipeline_abort, test_simple_pipeline: 各種パイプライン機能の検証ロジック。
  • test_protocol_version: プロトコルバージョン指定の検証。

ユーティリティ・ヘルパー

  • pg_fatal_impl: エラー発生時にプログラムを終了させるラッパー。
  • confirm_query_canceled_impl: クエリが期待通りキャンセルされたかを検証。
  • wait_for_connection_state: pg_stat_activity を監視し、特定の接続状態を待機。
  • copy_connection: 既存の接続設定を複製して監視用接続を作成。
  • process_result: test_uniqviol における結果処理の共通化。

状態管理

  • enum PipelineInsertStep: test_pipelined_insert における複雑なパイプライン状態遷移を管理。

制御フローとデータフロー

1. **メインループ**: main 関数が引数からテストケースを選択し、PQconnectdb で接続を確立。 2. **パイプライン開始**: 各テスト関数内で PQenterPipelineMode を呼び出し、パイプラインモードへ移行。 3. **クエリ送信**: PQsendQueryParams 等でクエリを送信し、必要に応じて PQflush でバッファをフラッシュ。 4. **結果取得**: PQgetResult をループで呼び出し、PGRES_PIPELINE_SYNC や PGRES_PIPELINE_ABORTED を含む各結果を処理。 5. **非ブロッキング制御**: test_pipelined_insert や test_uniqviol では、select を用いてソケットの読み書き準備を判定し、デッドロックを回避しながら結果を消費。 6. **終了**: PQexitPipelineMode でモードを終了し、PQfinish で接続をクローズ。

読者向けメモ

  • **メモリ管理**: PQresult は PQclear で解放する必要があります。copy_connection で作成した接続は PQfinish で解放されます。
  • **エラーハンドリング**: パイプラインモードでエラーが発生すると、以降のクエリは PGRES_PIPELINE_ABORTED となり、PQpipelineSync を呼び出すまでこの状態が継続します。
  • **非ブロッキング**: PQsetnonblocking を使用する場合、PQflush の戻り値を確認し、必要に応じて select で待機する実装が必須です。
  • **省略箇所**: 本ファイルはテストプログラムであるため、実運用コードで必要な高度なリトライロジックや接続プール管理は含まれていません。
