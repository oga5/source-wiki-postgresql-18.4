src/test/modules/oauth_validator/t/oauth_server.py

概要

本ファイルは、OAuth 2.0 認証フローのテストを目的としたモック認証サーバーです。Perl で記述されたテストスイートから呼び出されることを想定しており、エフェメラルポートで起動して JSON ベースのレスポンスを返します。

モジュール内での役割

src/test/modules/oauth_validator/t ディレクトリにおける唯一の実行可能スクリプトであり、OAuth サーバーの挙動をシミュレートするバックエンドとして機能します。

責務

  • **OAuth エンドポイントの提供**: /.well-known/openid-configuration (discovery), /authorize, /token の各エンドポイントを実装。
  • **テスト用パラメータの注入**: クライアント ID にエンコードされた JSON を解析し、エラーコードの強制やレスポンスの肥大化（巨大な文字列やネストされた構造）を動的に制御。
  • **状態管理**: 複数回にわたるトークンリクエスト間でのリトライ回数や待機時間の検証。
  • **通信の可視化**: デバッグ用にレスポンス内容を標準出力へログ出力。

重要なシンボル

  • OAuthHandler: http.server.BaseHTTPRequestHandler を継承したリクエスト処理クラス。
  • _check_issuer(): リクエストパスに基づき、標準・代替・パラメータ化された各モードを切り替える。
  • _TokenState: main() 内で定義される、リトライ制御のための状態保持用クラス。
  • authorization() / token(): OAuth 2.0 の主要なフローを処理するメソッド。

制御フローとデータフロー

1. **起動**: main() が HTTPServer をポート 0 で起動し、割り当てられたポート番号を標準出力へ書き出した後、stdout を閉じてデーモン化します。 2. **リクエスト処理**: do_GET または do_POST がリクエストを受け取り、_check_issuer() でパスを正規化します。 3. **パラメータ解析**: do_POST は _parse_params() でボディを解析し、必要に応じて client_id からテスト用パラメータを抽出します。 4. **レスポンス生成**: 各エンドポイントメソッドが JSON を生成し、_send_json() を通じてクライアントへ返却します。 5. **検証**: token() メソッド内で _TokenState を参照し、クライアントが期待される待機時間を守っているか等のアサーションを実行します。

直接依存

  • http.server: サーバー実装の基盤。
  • urllib.parse: URL およびクエリパラメータの解析。
  • base64, json: クライアントからのパラメータデコードおよびレスポンス生成。
  • collections.defaultdict: クライアントごとの状態管理用。

関連コード

  • OAuth/Server.pm: 本スクリプトを起動し、テストを実行する親プロセス側の Perl モジュール。

読者向けメモ

  • **デバッグ**: _log_response() 内でレスポンスサイズが 1024 バイトを超えるとアサーションエラーが発生するようになっています。巨大なレスポンスをテストする場合は注意してください。
  • **ライフサイクル**: 親プロセスが終了シグナルを送るまで serve_forever() で待機します。
  • **互換性**: Python 3.6 以前の環境を考慮した quote_plus の safe 引数指定など、古い Python バージョンへの配慮が見られます。
