src/test/modules/oauth_validator/magic_validator.c

概要

本ファイルは、PostgreSQLのOAuthトークン検証機能におけるABI互換性チェックのテスト用モジュールです。意図的に不正なマジックナンバーを定義することで、検証フックの読み込み時にシステムがどのようにエラーを検知・拒否するかを検証するために使用されます。

モジュール内での役割

oauth_validator テストスイートの一部として、正常系ではなく「異常系（ABI不一致）」をシミュレートする役割を担います。他のバリデーターが正常な動作を検証するのに対し、本ファイルはロード時のバリデーションロジックをテストするための境界条件を提供します。

責務

  • **ABI互換性の破壊**: OAuthValidatorCallbacks 構造体の先頭に不正なマジックナンバー（0xdeadbeef）を配置し、ロード時の拒否を誘発する。
  • **検証フックの提供**: _PG_oauth_validator_module_init を通じて、意図的に無効なコールバックテーブルをエクスポートする。

重要なシンボル

  • validator_callbacks: OAuthValidatorCallbacks 構造体のインスタンス。先頭要素に不正なマジック値を保持しており、これが本ファイルの核心です。
  • _PG_oauth_validator_module_init: モジュールロード時に呼び出されるエントリーポイント。不正な validator_callbacks へのポインタを返します。
  • validate_token: 本来の検証ロジックですが、このモジュールが正しく拒否されれば呼び出されることはありません。

制御フローとデータフロー

1. PostgreSQLサーバーがモジュールをロードし、_PG_oauth_validator_module_init を呼び出します。 2. 関数は不正なマジックナンバーを含む validator_callbacks 構造体のアドレスを返します。 3. サーバー側の検証ロジックがマジックナンバーをチェックし、期待値と一致しないため、モジュールのロードを拒否またはエラーを発生させます。 4. validate_token は到達不能コードとして定義されており、実行されることは想定されていません。

直接依存

  • postgres.h: PostgreSQLの基本定義。
  • fmgr.h: 関数マネージャおよびモジュールロード用マクロ（PG_MODULE_MAGIC）の提供。
  • libpq/oauth.h: OAuthValidatorCallbacks および ValidatorModuleState 等の型定義。

関連コード

  • src/test/modules/oauth_validator/validator.c: 正常なマジックナンバーを定義する標準的なバリデーター実装。比較対象として参照してください。
  • src/include/libpq/oauth.h: OAuthValidatorCallbacks 構造体の定義元。マジックナンバーの期待値を確認するために参照します。

読者向けメモ

  • 本ファイルは「テスト用」であり、実運用環境のコードの参考にはしないでください。
  • elog(FATAL, ...) が含まれていますが、このモジュールが正しく機能（＝ロード拒否）していれば、この行が実行されることはありません。
  • テスト実行時にこのモジュールがロードされた際、ログにABI不一致に関するエラーが出力されることが期待される動作です。
