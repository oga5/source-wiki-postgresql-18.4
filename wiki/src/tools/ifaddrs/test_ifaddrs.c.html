src/tools/ifaddrs/test_ifaddrs.c

概要

test_ifaddrs.c は、PostgreSQL のネットワークインターフェースアドレス取得機能である pg_foreach_ifaddr() の動作を検証するためのスタンドアロンなテストツールです。

モジュール内での役割

src/tools/ifaddrs ディレクトリにおける唯一のソースファイルであり、システムが提供するネットワークインターフェース列挙 API の疎通確認およびデバッグ用ユーティリティとして機能します。

責務

  • pg_foreach_ifaddr() を呼び出し、システム上の全インターフェースアドレスとサブネットマスクを列挙する。
  • 取得した sockaddr 構造体を人間が読める形式（IPアドレス文字列）に変換して標準出力へ表示する。
  • Windows 環境における WSAStartup の初期化処理を含む、プラットフォーム依存のセットアップを行う。

重要なシンボル

  • pg_foreach_ifaddr: 検証対象のコア関数。インターフェースごとにコールバックを実行する。
  • callback: pg_foreach_ifaddr に渡される関数。各インターフェースのアドレスとマスクを受け取り、表示処理を行う。
  • print_addr: sockaddr 構造体を getnameinfo を用いて文字列化するヘルパー関数。

追加の索引済みシンボル

  • main

制御フローとデータフロー

1. **初期化**: main 関数が起動し、Windows 環境であれば WSAStartup でソケットライブラリを初期化する。 2. **列挙開始**: pg_foreach_ifaddr(callback, NULL) を呼び出し、OS のネットワークインターフェースリストを走査する。 3. **コールバック実行**: 各インターフェースが見つかるたびに callback が呼び出される。 4. **アドレス変換**: callback 内で print_addr が呼ばれ、getnameinfo を介してバイナリ形式の sockaddr が数値形式の文字列に変換される。 5. **出力**: 変換されたアドレスとマスクが標準出力に書き出される。

直接依存

  • postgres.h: PostgreSQL の基本定義およびプラットフォーム抽象化レイヤー。
  • netinet/in.h / arpa/inet.h: ソケットアドレス構造体およびネットワーク関連の標準定義。
  • libpq/ifaddr.h: 検証対象である pg_foreach_ifaddr の宣言。

関連コード

  • src/include/libpq/ifaddr.h: pg_foreach_ifaddr のインターフェース定義。
  • src/backend/libpq/ifaddr.c: pg_foreach_ifaddr の実装本体。本テストツールが正しく動作しているかを確認する際の比較対象。

読者向けメモ

  • 本ファイルは PostgreSQL のバックエンドの一部としてビルドされるものではなく、単独のテストツールとして意図されています。
  • getnameinfo を使用しているため、IPv4 (AF_INET) と IPv6 (AF_INET6) の両方に対応した汎用的なアドレス表示が可能です。
  • エラー発生時には %m 指定子（fprintf の拡張）を使用して、OS が返すエラーメッセージを表示するよう設計されています。
