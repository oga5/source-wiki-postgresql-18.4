src/tools/pg_bsd_indent/parse.c

概要

本ファイルは、C言語の構文解析における「状態遷移」と「スタックベースの還元（Reduction）」を管理する中心的なモジュールです。字句解析されたトークンを受け取り、現在のコード構造（if, for, switch, 宣言など）をスタック上で追跡し、適切なインデントレベルを決定します。

モジュール内での役割

indent プログラムの心臓部です。字句解析器（lexi.c）がトークンを識別した後、このモジュールがそのトークンを現在の構文コンテキストに適用し、インデントの計算に必要な状態（parser_state）を更新します。

責務

  • **構文スタックの管理**: トークンに基づき、ps.p_stack を操作して現在のネストレベルや構造を保持する。
  • **インデント計算**: 構文構造の変化に応じて、次の行のインデントレベル（ps.i_l_follow）を算出する。
  • **構文の還元**: reduce() 関数を通じて、if-else や do-while などの複合文を単純な文（stmt）へと畳み込み、スタックを整理する。

重要なシンボル

  • parse(int tk): 外部から呼び出されるメインの構文解析関数。トークンを受け取り、状態を更新する。
  • reduce(void): スタック上のトークンを評価し、ルールに従って文を還元する内部関数。
  • ps (parser_state): 現在の解析状態を保持するグローバル構造体。

追加の索引済みシンボル

  • parse
  • reduce

制御フローとデータフロー

1. **入力**: parse() がトークン tk を受け取る。 2. **前処理**: if 文の未解決な else がある場合、先に還元を行う。 3. **状態遷移**: switch(tk) により、トークンに応じたスタック操作とインデントレベルの調整を行う。 4. **還元**: reduce() を呼び出し、スタックのトップが還元可能なパターン（例: do + stmt → dohead）に一致するかをループで確認する。 5. **出力**: 更新された ps 構造体の値が、後の出力処理（io.c）でインデントとして利用される。

読者向けメモ

  • **スタックの重要性**: ps.p_stack は単なるネスト管理ではなく、インデントの計算根拠そのものです。reduce() 内の switch 文が、このプログラムの「文法」を定義しています。
  • **デバッグ**: #ifdef debug が有効な場合、各トークン処理後にスタックの状態が標準出力にダンプされます。挙動を追う際は非常に有用です。
  • **還元ルール**: reduce() は無限ループで記述されており、スタックが安定するまで還元を繰り返します。このため、新しい構文ルールを追加する際は、無限ループに陥らないよう注意が必要です。
