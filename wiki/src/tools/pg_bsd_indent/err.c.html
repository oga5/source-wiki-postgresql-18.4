src/tools/pg_bsd_indent/err.c

概要

err.c は、pg_bsd_indent ツール内で使用されるエラー報告およびプロセス終了のためのユーティリティ関数群を提供します。標準的なエラー出力（stderr）へのメッセージ出力と、指定された終了コードでのプログラム停止を簡潔に行うためのラッパーです。

モジュール内での役割

本モジュールは、indent の各処理フェーズ（引数解析、字句解析、構文解析など）で発生する致命的なエラーを統一的に扱うための基盤です。他のソースファイルから呼び出され、エラー発生時の終了処理を標準化します。

責務

  • **エラーメッセージの整形と出力**: ユーザーに対してエラー内容を stderr に出力する。
  • **終了コードの管理**: エラー発生時に適切な終了ステータスを伴ってプロセスを終了させる。
  • **errno の統合**: システムエラーが発生した際、errno に基づく説明文を自動的に付与する（err 関数）。

重要なシンボル

  • err(int eval, const char *fmt, ...): システムエラー（errno）を伴うエラー報告用。メッセージの後にシステムエラー文字列を連結して出力し、eval で終了します。
  • errx(int eval, const char *fmt, ...): システムエラーを伴わない汎用的なエラー報告用。メッセージのみを出力し、eval で終了します。

追加の索引済みシンボル

  • err
  • errx

制御フローとデータフロー

1. **呼び出し**: 各モジュールから err または errx が呼び出されます。 2. **可変長引数の処理**: va_start / va_end を使用して、フォーマット文字列と引数を取得します。 3. **出力**: vfprintf を使用して stderr にメッセージを書き込みます。err の場合は、続けて strerror(errno) を出力します。 4. **終了**: exit(eval) を呼び出し、プロセスを即座に終了させます。

読者向けメモ

  • 本ファイルは BSD 由来の err(3) ファミリーの最小限の実装です。
  • pg_noreturn 属性が定義されているため、これらの関数を呼び出した後のコードは到達不能（unreachable）として扱われます。
  • 複雑なエラーハンドリングロジックは含まれておらず、あくまで致命的なエラー発生時の終了処理に特化しています。
