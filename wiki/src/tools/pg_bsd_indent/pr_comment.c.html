src/tools/pg_bsd_indent/pr_comment.c

概要

C言語ソースコード内のコメントを解析し、整形して出力バッファへ書き出すためのメインロジックを担うファイルです。

モジュール内での役割

pg_bsd_indent の整形エンジンの一部として、字句解析器（lexi.c）がコメントの開始を検知した際に呼び出されます。コメントの配置、折り返し、および整形ルール（ボックスコメントか否かなど）を決定し、最終的な出力形式を制御します。

責務

  • **コメントの分類**: コメントが「ボックスコメント（整形不可）」か「通常のテキストコメント（整形可）」かを判定する。
  • **配置の決定**: コードのインデントレベルや宣言の有無に基づき、コメントの開始カラムを計算する。
  • **テキストの整形**: 最大行長（max_col）を超えないように、空白やタブを基準にコメントを適切に折り返す。
  • **バッファ管理**: コメント内容を一時バッファへコピーし、必要に応じて dump_line() を呼び出して出力する。

重要なシンボル

  • pr_comment(void): 本ファイルの唯一の公開関数。コメント処理の全工程を制御する。
  • ps (parser_state): 現在のパーサー状態を保持するグローバル構造体。ps.box_com や ps.com_col など、コメント整形に必要な状態を読み書きする。
  • buf_ptr: 入力バッファの現在位置を指すポインタ。
  • e_com: 出力コメントバッファの末尾を指すポインタ。

追加の索引済みシンボル

  • pr_comment

制御フローとデータフロー

1. **初期化**: ps.box_com の判定を行い、コメントの開始カラム（ps.com_col）を計算する。 2. **前処理**: /* を出力バッファに書き込み、必要に応じて開始行の空白をスキップする。 3. **メインループ**: buf_ptr を進めながら文字単位で処理を行う。

  • \n (改行): ボックスコメントならそのまま出力、通常コメントなら行の継続を考慮して整形する。
  • * (アスタリスク): コメント終了 (*/) かどうかを判定する。
  • default: テキストをバッファへコピー。行長を超えた場合は last_bl（最後に確認した空白）の位置で強制改行（dump_line()）を行う。

    • 4. **終了**: */ を書き込み、状態を復元して呼び出し元へ戻る。

直接依存

  • indent_globs.h: グローバル変数およびパーサー状態の定義。
  • indent.h: indent コマンドの主要な定数や型定義。
  • indent_codes.h: 字句解析用のトークンコード。
  • 標準ライブラリ: stdio.h, string.h, stdlib.h 等。

関連コード

  • indent.c: pr_comment を呼び出すメインループが含まれる。
  • io.c: dump_line() や fill_buffer() など、バッファ操作の低レイヤー関数を提供。
  • lexi.c: コメントの開始を検知し、本関数を起動するトリガーとなる。

読者向けメモ

  • 本ファイルは非常に古いコードベース（1980年代）に由来しており、goto 文や複雑なポインタ操作が含まれます。
  • ps.box_com フラグが true の場合、コメント内の空白や改行は極力維持されます。
  • CHECK_SIZE_COM マクロは、コメントバッファのオーバーフローを防ぐために必須です。修正を加える際は、バッファサイズを常に意識してください。
