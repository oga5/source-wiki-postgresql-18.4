src/tools/pg_bsd_indent/io.c

概要

io.c は、pg_bsd_indent における入出力処理の中核を担うファイルです。整形されたコードの出力、入力バッファの管理、および整形ルールの制御（/*INDENT ON/OFF*/）を実装しています。

モジュール内での役割

本モジュールは、字句解析器（lexi.c）や構文解析器（parse.c）が生成したトークンや状態を、最終的な整形済みテキストとして出力ストリームへ書き出す役割を持ちます。

責務

  • **出力の整形**: dump_line を通じて、ラベル、コード、コメントを適切なインデント位置に配置して出力する。
  • **バッファ管理**: 入力ストリームから行単位でデータを読み込み、必要に応じて動的にバッファを拡張する。
  • **先読み（Lookahead）**: 構文解析のために必要な先読み機能を提供し、入力ストリームの制御を行う。
  • **整形制御**: /*INDENT ON/OFF*/ ディレクティブを検出し、整形処理の有効・無効を切り替える。

重要なシンボル

  • dump_line(): 整形済み行をファイルに出力するメイン関数。
  • fill_buffer(): 入力ストリームから次の行を読み込む。
  • lookahead(): 入力ストリームを先読みする。
  • pad_output(): 指定されたカラム位置までタブやスペースで埋める。
  • compute_code_target(): コードの適切なインデント位置を計算する。

制御フローとデータフロー

1. **入力**: fill_buffer が input からデータを読み込み、in_buffer に格納します。この際、lookahead バッファの内容が優先されます。 2. **処理**: 構文解析器が状態（ps）を更新し、dump_line が呼び出されます。 3. **出力**: dump_line は compute_code_target 等を用いてインデント量を決定し、pad_output を経由して output ストリームへ書き出します。 4. **リセット**: 行出力後、各バッファ（s_lab, s_code, s_com）はクリアされ、次の行の処理へ備えます。

直接依存

  • c.h, ctype.h, err.h, stdio.h, stdlib.h, string.h
  • indent_globs.h (グローバル状態 ps やバッファポインタ)
  • indent.h (定数や関数プロトタイプ)

関連コード

  • lexi.c: 入力ストリームからトークンを切り出す際に本ファイルのバッファ機能を利用する。
  • parse.c: 構文解析の結果として dump_line を呼び出す。
  • indent.c: プログラムのメインループから fill_buffer を呼び出し、処理を開始する。

読者向けメモ

  • 本ファイルは古い BSD スタイルのコードであり、s_code や e_code といったポインタによるバッファ操作が多用されています。これらは indent_globs.h で定義されたグローバルなバッファを指しています。
  • dump_line 内の comment_open フラグや inhibit_formatting フラグは、コメントや整形無効化ブロックの継続状態を管理する重要な変数です。
  • pad_output におけるタブ計算は、postgres_tab_rules に依存した特殊なロジックが含まれている点に注意してください。
