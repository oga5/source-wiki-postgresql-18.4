src/tools/pg_bsd_indent/indent.c

概要

indent.c は、BSD indent ユーティリティのメインエントリポイントおよび中心的な制御ループを実装するファイルです。C 言語ソースコードの字句解析結果に基づき、適切なインデント、改行、空白の挿入を行い、整形されたコードを出力します。

モジュール内での役割

src/tools/pg_bsd_indent モジュールの中核であり、他のコンポーネント（lexi.c による字句解析、parse.c による構文解析、io.c による入出力）を統合してプログラムのライフサイクルを管理します。

責務

  • **初期化**: グローバル状態（parser_state）および各バッファのメモリ確保と初期設定。
  • **コマンドライン解析**: オプションの読み込みとプロファイル設定の適用。
  • **メインループ**: 字句解析器 lexi() を呼び出し、トークン種別に応じて整形処理を分岐・実行する。
  • **バックアップ生成**: 入力ファイルを上書きする場合の .BAK ファイル作成処理。

重要なシンボル

  • main: プログラムの実行起点。初期化、引数解析、メインループの実行を行う。
  • ps (parser_state): 解析中のコードの状態（インデントレベル、括弧の深さ、宣言内か否か等）を保持するグローバル構造体。
  • lexi(): 入力ストリームから次のトークンを読み取り、その型を返す字句解析関数。
  • bakcopy(): 入力ファイルをバックアップし、出力先を元のファイル名に切り替えるためのユーティリティ。
  • indent_declaration(): 宣言文における識別子の位置を揃えるためのインデント計算と挿入を行う関数。

追加の索引済みシンボル

  • bakcopy
  • directives
  • indent_declaration

制御フローとデータフロー

1. **初期化**: 各種バッファ（codebuf, labbuf 等）を確保し、ps をデフォルト値で初期化。 2. **引数処理**: コマンドライン引数を走査し、設定を適用。入力ファイルが開かれていない場合は stdin を使用。 3. **メインループ**:

  • lexi() でトークンを取得。
  • ps.search_brace が真の場合、if や while の直後の括弧を適切に配置するためのバッファリング処理を行う。
  • switch (type_code) により、トークン種別（lparen, semicolon, lbrace 等）に応じた整形ロジックを実行。
  • dump_line() を適宜呼び出し、バッファの内容を整形済み出力として書き出す。

    • 4. **終了**: EOF 到達時に最終行をフラッシュし、エラーコードを返して終了。

直接依存

  • c.h: PostgreSQL 共通の基本定義。
  • indent_globs.h: グローバル変数定義（DECLARE_INDENT_GLOBALS を定義してインクルード）。
  • indent_codes.h: トークン種別の列挙。
  • indent.h: 内部関数プロトタイプおよび構造体定義。
  • 標準ライブラリ: stdio.h, stdlib.h, string.h, ctype.h, unistd.h 等。
  • err.h
  • fcntl.h
  • sys/param.h

関連コード

  • lexi.c: lexi() 関数の実装。トークン化のロジック。
  • parse.c: parse() 関数の実装。構文解析とインデントレベルの更新。
  • io.c: dump_line() や fill_buffer() 等、入出力バッファ管理の実装。
  • pr_comment.c: コメントの整形ロジック。

読者向けメモ

  • 本ファイルは非常に長い switch 文で構成されています。まずは main 関数の初期化部分と、メインループ内の lexi() 呼び出しから始まるフローを追うのが理解の近道です。
  • ps 構造体はプログラム全体で共有されるため、各トークン処理が ps をどのように更新しているかを意識してください。
  • search_brace フラグは、if や else の後の括弧配置（-br, -bl オプション等）を制御するための重要な状態遷移です。
