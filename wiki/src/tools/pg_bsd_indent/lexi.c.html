src/tools/pg_bsd_indent/lexi.c

概要

lexi.c は、indent ユーティリティの字句解析器（レキサー）を実装するファイルです。入力ソースコードからトークンを一つずつ切り出し、その種類を識別して呼び出し元に返します。

モジュール内での役割

indent のメインループにおいて、ソースコードのストリームを意味のある単位（識別子、キーワード、演算子、リテラルなど）に分解する役割を担います。解析結果は parser_state に反映され、後続の整形処理の判断材料となります。

責務

  • **トークン抽出**: 入力バッファから空白をスキップし、次のトークンを特定する。
  • **キーワード識別**: specials テーブルを用いた二分探索により、C言語の予約語や型名を判定する。
  • **型名の動的管理**: typedef 等で定義された新しい型名を登録・保持し、識別子と区別する。
  • **文脈解析**: is_func_definition を用いて、識別子が関数定義か宣言かを推論する。

重要なシンボル

  • lexi(struct parser_state *state): メインの字句解析関数。トークンを読み取り、その型コードを返す。
  • specials[]: C言語の予約語とコードの対応表。二分探索用にアルファベット順で定義されている。
  • typenames: ユーザー定義の型名を保持する動的配列。
  • add_typename(const char *key): 新しい型名を typenames に追加する。
  • is_func_definition(char *tp): 関数定義の開始を判定するための先読みロジック。

制御フローとデータフロー

1. **空白スキップ**: buf_ptr を進めて空白を読み飛ばす。 2. **分類**: chartype 配列を参照し、英数字か記号かを判定する。

  • **英数字の場合**: 数値リテラルまたは識別子として処理。specials テーブルでキーワードか確認し、該当しなければ typenames を検索して型名かどうかを判定する。
  • **記号の場合**: switch 文で演算子や区切り文字を特定する。

    • 3. **状態更新**: 抽出したトークンを token バッファに格納し、state->last_u_d（単項演算子か二項演算子かのフラグ）を更新して整形処理を補助する。 4. **戻り値**: トークンの種類を示す整数コードを返す。

直接依存

  • c.h, err.h, stdio.h, ctype.h, stdlib.h, string.h: 標準ライブラリおよび共通ヘッダー。
  • indent_globs.h: グローバル変数（buf_ptr, token 等）の定義。
  • indent_codes.h: トークン種別コードの定義。
  • indent.h: parser_state 構造体定義。

関連コード

  • indent.c: lexi() を呼び出し、解析結果に基づいて整形を行うメインループ。
  • parse.c: lexi() が提供するトークン情報を元に、構文解析の状態を遷移させる。

読者向けメモ

  • **先読みの注意**: is_func_definition はバッファの境界を越えて先読みを行うため、lookahead() 関数と連携しています。
  • **型名の扱い**: indent は typedef を自動認識しようとしますが、解析が不完全な場合は add_typename を通じて明示的に型名を教える必要があります。
  • **テーブルの順序**: specials 配列は bsearch を使用しているため、要素を追加・変更する際は必ずアルファベット順を維持してください。
