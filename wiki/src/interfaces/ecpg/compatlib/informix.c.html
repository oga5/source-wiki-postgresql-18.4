src/interfaces/ecpg/compatlib/informix.c

概要

本ファイルは、ECPG（Embedded SQL for C）において、Informix データベース製品との互換性を提供するためのライブラリ関数群を実装しています。Informix 固有のデータ型（decimal, date, timestamp 等）の操作や、それらと PostgreSQL の内部表現との変換を仲介します。

モジュール内での役割

src/interfaces/ecpg/compatlib モジュールの中核であり、Informix 互換 API を提供する唯一のソースファイルです。ECPG で生成されたコードが Informix 互換の関数を呼び出す際、このファイルがその要求を PostgreSQL の pgtypes ライブラリへ変換・転送します。

責務

  • **数値演算の抽象化**: decimal 型に対する加減乗除や比較操作を、pgtypes の numeric 型を介して実行する。
  • **型変換**: 文字列、整数、浮動小数点数と decimal 型との相互変換。
  • **日付・時刻操作**: Informix 形式の日付・時刻文字列の解析、フォーマット、および演算。
  • **互換性維持**: sqlca（SQL Communication Area）の初期化や、Informix 固有のユーティリティ関数（rupshift, ldchar 等）の提供。

重要なシンボル

  • sqlca_init: Informix 互換の sqlca_t 構造体の初期状態を定義する静的変数。
  • deccall2 / deccall3: decimal 型の演算を numeric 型の関数へ委譲するための内部ラッパー。
  • decadd, decsub, decmul, decdiv: Informix 互換の算術演算関数。
  • rsetnull / risnull: Informix 形式の NULL 値判定および設定を行う関数。
  • rfmtlong: Informix 形式のフォーマット文字列に基づき、long 型を文字列へ変換する複雑なユーティリティ。

制御フローとデータフロー

1. **演算フロー**: decadd 等の関数が呼び出されると、deccall3 が decimal を numeric に変換し、PGTYPESnumeric_add 等の PostgreSQL 内部関数を実行後、結果を再び decimal に戻して返却します。 2. **変換フロー**: deccvasc 等の変換関数は、入力文字列を ecpg_strndup で複製し、PGTYPESnumeric_from_asc で解析した後、decimal 構造体に格納します。 3. **エラーハンドリング**: 各関数は errno を監視し、PGTYPES ライブラリから返されるエラーコードを Informix 互換の ECPG_INFORMIX_* エラーコードにマッピングして呼び出し元へ伝播させます。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用ヘッダー。
  • pgtypes_*.h: numeric, date, timestamp 等の PostgreSQL 共通型変換ライブラリ。
  • ecpg_informix.h, sqlca.h, sqltypes.h: Informix 互換のための型定義および定数。
  • 標準 C ライブラリ: math.h, ctype.h, limits.h（数値計算および文字列操作用）。
  • ecpgerrno.h
  • ecpgtype.h
  • pgtypes_date.h
  • pgtypes_error.h
  • pgtypes_numeric.h

関連コード

  • src/interfaces/ecpg/pgtypeslib/: 本ファイルが依存している PGTYPES 関連関数の実装元。
  • src/interfaces/ecpg/ecpglib/: ECPG のコアライブラリ。ECPGset_noind_null 等の基盤機能を提供。

読者向けメモ

  • 本ファイル内の deccall2/deccall3 は、Informix の decimal 型と PostgreSQL の numeric 型の間の変換を隠蔽する重要なパターンです。
  • rfmtlong は Informix の複雑なフォーマット指定子を処理するため、他の関数に比べて実装が非常に長くなっています。
  • rgetmsg や rtypalign 等の関数は、現状ではスタブ（ダミー実装）となっており、コンパイラの警告を抑制する処理のみが行われています。
