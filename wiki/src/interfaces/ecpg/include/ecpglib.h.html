src/interfaces/ecpg/include/ecpglib.h

概要

ecpglib.h は、PostgreSQL の埋め込み SQL (ECPG) アプリケーションが利用するクライアントライブラリ（libecpg）の公開インターフェースを定義するヘッダーファイルです。ECPG プリプロセッサによって生成された C コードが、データベース接続、トランザクション制御、SQL 実行を行うための主要な API を提供します。

モジュール内での役割

src/interfaces/ecpg/include モジュールにおいて、本ファイルはアプリケーション開発者が直接利用する関数群を束ねる「メインゲートウェイ」の役割を果たします。他のヘッダーファイルが型定義やエラーコードを個別に提供するのに対し、本ファイルはそれらを統合し、実行時のライブラリ呼び出し規約を規定します。

責務

  • **接続管理**: データベースへの接続、切断、および接続先の切り替え。
  • **SQL 実行**: 静的 SQL (ECPGdo) および動的 SQL の実行と記述子 (Descriptor) の管理。
  • **トランザクション制御**: コミット、ロールバック、およびトランザクション状態の取得。
  • **互換性サポート**: SQLCODE や SQLSTATE などの標準的な SQL 診断領域へのアクセス提供。

重要なシンボル

  • ECPGdo: ECPG の中核となる関数。プリプロセッサが生成した SQL 文を実際に実行します。
  • ECPGconnect / ECPGdisconnect: データベースセッションのライフサイクルを管理します。
  • SQLCODE / SQLSTATE: sqlca 構造体へのショートカットマクロ。SQL 実行結果のステータス確認に使用されます。
  • ECPGget_PGconn: 基盤となる libpq の接続ハンドルを取得し、必要に応じて低レイヤーの操作を可能にします。

制御フローとデータフロー

ECPG プリプロセッサは、埋め込み SQL 文を ECPGdo などのライブラリ関数呼び出しに変換します。 1. **初期化**: ecpg_pthreads_init によりスレッドセーフな環境を準備します。 2. **実行**: ECPGdo が呼び出されると、引数として渡された SQL 文とホスト変数が libpq を介してサーバーへ送信されます。 3. **結果処理**: 実行結果は sqlca 構造体に反映され、アプリケーションは SQLCODE 等を通じてエラー判定を行います。 4. **動的 SQL**: ECPGallocate_desc 等の記述子操作関数を介して、実行時にクエリ構造を構築・取得します。

直接依存

  • stdbool.h, string.h: 標準 C ライブラリ。
  • ecpg_config.h: ビルド環境固有の設定。
  • ecpgtype.h: ECPG で使用される型定義。
  • libpq-fe.h: PostgreSQL のフロントエンドライブラリ（基盤通信）。
  • sqlca.h: SQL 通信領域の定義。

関連コード

  • src/interfaces/ecpg/ecpglib/: 本ヘッダーで宣言された関数の実体が実装されているディレクトリ。
  • src/interfaces/ecpg/preproc/: 本ヘッダーの関数を呼び出す C コードを生成するプリプロセッサ。

読者向けメモ

  • 本ファイルで宣言されている関数は、原則としてアプリケーションコードから直接呼び出すことを想定していません。これらは ECPG プリプロセッサが生成するコードによって呼び出されることを前提としています。
  • extern "C" が指定されているため、C++ プロジェクトからの利用もサポートされています。
  • SQLCODE や SQLSTATE は sqlca 構造体へのアクセスを簡略化するためのものであり、古い SQL 標準との互換性を維持するために重要です。
