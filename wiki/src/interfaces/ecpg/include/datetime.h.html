src/interfaces/ecpg/include/datetime.h

概要

ECPG（Embedded SQL for C）において、日時および期間データを扱うための型定義を提供するヘッダーファイルです。Informix 互換のデータ型を C 言語の型として定義し、プリプロセッサによるコード生成をサポートします。

モジュール内での役割

src/interfaces/ecpg/include モジュールの一部として、ECPG ライブラリおよびプリコンパイラが生成するコード間で、日時関連のデータ構造のシグネチャを統一する役割を担います。

責務

  • dtime_t および intrvl_t 型の定義。
  • ECPG ライブラリ（ecpglib）との型定義の重複回避。
  • Informix 互換の型定義の提供。

重要なシンボル

  • dtime_t: timestamp 型の別名。ECPG での日時データ保持に使用されます。
  • intrvl_t: interval 型の別名。期間データ保持に使用されます。
  • _ECPG_DATETIME_H: 多重インクルードを防ぐためのガードマクロ。

制御フローとデータフロー

本ファイルは純粋な宣言ヘッダーであり、実行時の制御フローは持ちません。ECPG プリプロセッサが SQL 文を C コードに変換する際、これらの型定義を参照して変数宣言を生成します。

直接依存

  • ecpg_informix.h: Informix 互換機能に必要な基本定義を提供します。

関連コード

  • src/interfaces/ecpg/pgtypeslib/datetime.c: 本ヘッダーで定義された型を操作する関数群（PGTYPESdate_... 等）の実装が含まれています。
  • src/interfaces/ecpg/include/pgtypes_timestamp.h: timestamp 型の定義元であり、dtime_t の基盤となります。
  • src/interfaces/ecpg/include/pgtypes_interval.h: interval 型の定義元であり、intrvl_t の基盤となります。

読者向けメモ

  • _ECPGLIB_H が定義されている場合、型定義をスキップする仕組みになっています。これは ecpglib 内部で既にこれらの型が定義されている場合に、再定義エラーを防ぐための安全策です。
  • ECPG アプリケーションを開発する際、Informix 互換モードでコンパイルする場合にこれらの型が利用されます。直接的な型操作が必要な場合は、pgtypeslib の関数群を参照してください。
