src/interfaces/ecpg/include/ecpg_informix.h

概要

本ヘッダーファイルは、ECPG（Embedded SQL for C）において Informix データベースとの互換性を維持するための定義および関数宣言を提供します。Informix 固有のデータ型操作やエラーコードを ECPG 環境で利用可能にすることを目的としています。

モジュール内での役割

src/interfaces/ecpg/include モジュールの一部として、Informix 互換レイヤーのインターフェースを定義します。他の ECPG ライブラリ機能と連携し、Informix アプリケーションを PostgreSQL に移植する際のブリッジとして機能します。

責務

  • **エラーコードの定義**: Informix 固有の SQL エラーコード（SQLNOTFOUND や数値演算エラー等）の定義。
  • **Informix 互換関数の宣言**: 日付操作、文字列操作、数値演算、および decimal 型や datetime 型の変換関数のプロトタイプ宣言。
  • **変数管理**: Informix 互換モードにおける変数保持や sqlca のリセット機能の提供。

重要なシンボル

  • SQLNOTFOUND: Informix 互換の「データなし」を示すステータスコード（100）。
  • ECPG_INFORMIX_*: 数値演算や日付変換に関連する一連の Informix 固有エラーコード。
  • r* 系関数群: rdatestr, rfmtdate 等、Informix の日付・文字列操作を模倣する関数群。
  • dec* 系関数群: decimal 型の算術演算および型変換関数。
  • dt* 系関数群: datetime 型（timestamp）および interval 型の操作関数。

制御フローとデータフロー

本ファイルはインターフェース定義であるため、直接的なロジックフローは持ちません。

  • **データフロー**: アプリケーションが Informix 互換関数を呼び出す際、本ヘッダーで宣言された型（decimal, timestamp 等）を介してデータが渡されます。
  • **制御フロー**: ECPG ライブラリのランタイムが、これらの関数を通じて Informix 互換の挙動をエミュレートします。

直接依存

  • ecpglib.h: ECPG の基本ライブラリ機能。
  • pgtypes_date.h, pgtypes_interval.h, pgtypes_numeric.h, pgtypes_timestamp.h: PostgreSQL の型定義および操作関数。

関連コード

  • src/interfaces/ecpg/pgtypeslib/: 本ヘッダーで宣言されている r*, dec*, dt* 系関数の具体的な実装が含まれています。
  • src/interfaces/ecpg/include/sqlca.h: Informix 互換の SQLCA 構造体定義。

読者向けメモ

  • 本ファイルは Informix アプリケーションの移植性を高めるための「ラッパー」です。PostgreSQL ネイティブな実装とは異なる命名規則やエラーコード体系を持つため、Informix からの移行時以外での利用は推奨されません。
  • extern "C" ブロックにより、C++ からの呼び出しにも対応しています。
  • 実装の詳細を確認する場合は、対応する pgtypeslib 内のソースコードを参照してください。
