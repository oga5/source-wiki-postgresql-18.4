src/interfaces/ecpg/include/sqlda.h

概要

sqlda.h は、ECPG（Embedded SQL for C）において動的 SQL を扱うための「SQLDA（SQL Descriptor Area）」構造体の定義を抽象化するヘッダーファイルです。

モジュール内での役割

ECPG ライブラリが Informix 互換モードで動作しているか、ネイティブモードで動作しているかに応じて、適切な SQLDA 構造体定義を選択するスイッチの役割を果たします。

責務

  • コンパイル時の条件分岐に基づき、sqlvar_t および sqlda_t 型のエイリアスを定義する。
  • ユーザーコードに対して、モードを意識させない統一的なインターフェースを提供する。

重要なシンボル

  • sqlda_t: SQLDA 構造体の汎用的な型名。
  • sqlvar_t: SQLDA 内の各列（変数）情報を保持する構造体の汎用的な型名。
  • _ECPG_INFORMIX_H: Informix 互換モードを有効にするためのプリプロセッサ定義。

制御フローとデータフロー

このファイルは実行時のフローを持たず、コンパイル時のプリプロセッサディレクティブによって動作します。 1. _ECPG_INFORMIX_H が定義されている場合、sqlda-compat.h を読み込み、Informix 互換の構造体を sqlda_t として割り当てます。 2. 定義されていない場合、sqlda-native.h を読み込み、PostgreSQL ネイティブな構造体を sqlda_t として割り当てます。

直接依存

  • sqlda-compat.h: Informix 互換モード用の構造体定義。
  • sqlda-native.h: PostgreSQL ネイティブモード用の構造体定義。

関連コード

  • src/interfaces/ecpg/ecpglib/sqlda.c: 本ヘッダーで定義された構造体を使用して、実際に SQLDA のメモリ確保やデータ変換を行う実装ファイル。

読者向けメモ

  • 本ファイルは単なる型定義のラッパーです。実際の動的 SQL の処理ロジックを追う場合は、sqlda.c を参照してください。
  • Informix 互換モードを使用する場合は、アプリケーションのビルド設定で _ECPG_INFORMIX_H が適切に定義されていることを確認してください。
