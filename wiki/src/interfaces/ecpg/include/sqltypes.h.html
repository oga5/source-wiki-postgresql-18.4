src/interfaces/ecpg/include/sqltypes.h

概要

本ファイルは、ECPG（Embedded SQL for C）におけるデータ型マッピングを定義するヘッダーファイルです。主に Informix 互換のデータ型コードと、ECPG 内部の型定義を橋渡しする役割を担います。

モジュール内での役割

src/interfaces/ecpg/include モジュールにおいて、SQL の型と C 言語の型を対応付けるための定数定義を提供します。主に sqlda（SQL Descriptor Area）構造体を使用する際に、フィールドの型を識別するために参照されます。

責務

  • Informix 互換の型コード（C...TYPE）の定義。
  • SQL 記述子で使用される型識別子（SQL...）の定義。
  • プラットフォーム依存の整数サイズ（long vs long long）を考慮した 64 ビット整数型のマッピング。

重要なシンボル

  • C...TYPE 系マクロ: Informix 互換のアプリケーション用データ型コード。
  • SQL... 系マクロ: sqlda 構造体の sqltype フィールドで使用される標準的な型識別子。
  • SQLINT8 / SQLSERIAL8: コンパイル環境の long サイズに応じて 8 バイト整数型を動的に選択するマクロ。

制御フローとデータフロー

本ファイルは純粋な定数定義ヘッダーであり、実行時の制御フローは持ちません。データフローとしては、ECPG のプリプロセッサやライブラリが、SQL 文の実行結果を C 言語の変数にバインドする際、これらの型コードを参照して適切な変換関数を選択するために使用されます。

直接依存

  • limits.h: 標準 C ライブラリ。整数型のサイズ判定に関連して間接的に利用されます。

関連コード

  • src/interfaces/ecpg/include/ecpgtype.h: ここで定義されている ECPGt_... 型定数の基盤となっています。
  • src/interfaces/ecpg/include/sqlda.h: 本ファイルで定義された SQL... 型コードが実際に利用される構造体定義が含まれています。

読者向けメモ

  • 本ファイルは Informix 互換モードをサポートするための定義が多く含まれています。
  • SQLINT8 などの定義において、SIZEOF_LONG を用いた条件分岐が行われています。移植性を考慮する際は、この判定ロジックが現在のプラットフォームで正しく機能しているか確認してください。
  • 新しい型を追加する場合は、CTYPEMAX の値を更新する必要があります。
