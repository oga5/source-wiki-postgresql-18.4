src/interfaces/ecpg/include/pgtypes_interval.h

概要

本ファイルは、ECPG（Embedded SQL for C）において PostgreSQL の interval 型を扱うためのデータ構造定義と、それらを操作する関数群のプロトタイプを定義するヘッダーファイルです。

モジュール内での役割

src/interfaces/ecpg/include モジュールの一部として、ECPG ライブラリが SQL の時間間隔（Interval）データを C 言語の構造体として表現し、文字列変換やメモリ管理を行うためのインターフェースを提供します。

責務

  • interval 構造体の定義と、その内部表現（int64 による時間単位と long による月単位の分離）の提供。
  • interval 型の動的なメモリ確保・解放機能の宣言。
  • 文字列形式（ISO 8601 等）と内部構造体間の相互変換機能の宣言。

重要なシンボル

  • struct interval: 時間間隔を保持する主要な構造体。time（マイクロ秒単位のオフセット）と month（月・年単位）で構成されます。
  • PGTYPESinterval_from_asc: 文字列から interval 構造体へ変換する関数。
  • PGTYPESinterval_to_asc: interval 構造体を文字列へ変換する関数。
  • PGTYPESinterval_new / PGTYPESinterval_free: interval 型のメモリ管理を行う関数。

制御フローとデータフロー

本ファイル自体は宣言のみを提供します。実際のデータフローは、ECPG アプリケーションが SQL 文を実行する際に、ライブラリ内部でこれらの関数を呼び出し、データベースから取得したバイナリデータや文字列を interval 構造体にマッピングすることで発生します。

直接依存

  • stdint.h: int64_t 型の定義に使用。
  • ecpg_config.h: ECPG のビルド設定（プラットフォーム依存の定義など）。
  • pgtypes.h: ECPG の共通型定義。

関連コード

  • src/interfaces/ecpg/pgtypeslib/interval.c: 本ヘッダーで宣言された関数の実装が含まれています。
  • src/interfaces/ecpg/include/pgtypes_timestamp.h: interval と組み合わせて使用されることが多いタイムスタンプ型の定義。
  • src/interfaces/ecpg/include/pgtypes_date.h: 日付型に関連する定義。

読者向けメモ

  • **構造体の設計**: interval 構造体は、time（月・年以外の単位）と month（月・年）を分けて保持します。これは PostgreSQL 内部の interval 表現と整合させるための設計です。
  • **メモリ管理**: PGTYPESinterval_new で確保したメモリは、必ず PGTYPESinterval_free で解放する必要があります。
  • **C++ 互換性**: extern "C" ブロックにより、C++ からの呼び出しも考慮されています。
