src/interfaces/ecpg/include/ecpg-pthread-win32.h

概要

ECPG（Embedded C for PostgreSQL）ライブラリにおいて、Windows環境下でのスレッド処理をPOSIXスレッド（pthread）APIと互換性を持たせるための互換レイヤーヘッダーです。

モジュール内での役割

ecpglib がプラットフォームに依存せずスレッドセーフな動作を実現するための抽象化層として機能します。非Windows環境では標準の <pthread.h> を利用し、Windows環境では本ファイルが定義するマクロや構造体を通じてネイティブなWin32 API（CRITICAL_SECTION や Tls 系関数）へ変換します。

責務

  • POSIXスレッドAPIのサブセット（mutex、thread-local storage、once実行）のWindows向け再定義。
  • Windows環境におけるスレッド同期および初期化処理の抽象化。
  • コンパイル時のプラットフォーム分岐による、POSIX/Win32 APIの透過的な切り替え。

重要なシンボル

  • pthread_mutex_t: Win32の CRITICAL_SECTION をラップした構造体。初期化状態管理用の initstate を含む。
  • pthread_key_t: DWORD 型として定義されるスレッドローカルストレージのキー。
  • pthread_once_t: 初期化済みフラグを保持する bool 型。
  • pthread_getspecific / pthread_setspecific: TlsGetValue / TlsSetValue へのマッピング。
  • pthread_once: win32_pthread_once を呼び出し、一度だけ実行される処理を保証するマクロ。

制御フローとデータフロー

  • **初期化**: pthread_mutex_t は PTHREAD_MUTEX_INITIALIZER を通じて初期化され、必要に応じて pthread_mutex_init で CRITICAL_SECTION が構築されます。
  • **スレッドローカル**: pthread_key_create は TlsAlloc を呼び出し、OSからスレッドローカルストレージのインデックスを取得します。
  • **一度だけの実行**: pthread_once マクロは、フラグが未設定の場合にのみ win32_pthread_once を呼び出し、スレッドセーフな初期化を実現します。

直接依存

  • pthread.h (非Windows環境時)
  • Windows API (windows.h 相当の機能: CRITICAL_SECTION, TlsAlloc, TlsGetValue, TlsSetValue)

関連コード

  • src/interfaces/ecpg/ecpglib/: 本ヘッダーを利用してスレッドセーフな接続管理やエラー処理を行うECPGライブラリ本体。

読者向けメモ

  • **制限事項**: pthread_key_create のデストラクタ引数は、現在の実装では無視されます（FIXMEコメント参照）。Windows環境ではスレッド終了時に自動的にクリーンアップされないため、メモリリークに注意が必要です。
  • **設計意図**: このファイルは完全なPOSIXスレッド実装を目指すものではなく、ECPGライブラリが動作するために必要な最小限の機能をWin32 APIでエミュレートすることに特化しています。
