src/interfaces/ecpg/include/sqlda-compat.h

概要

本ファイルは、ECPG（Embedded SQL for C）において、Informix 互換の SQLDA（SQL Descriptor Area）構造を定義するヘッダーファイルです。動的な SQL 文の実行において、ホスト変数とデータベース間のデータ受け渡しを仲介するデータ構造を提供します。

モジュール内での役割

ecpg ライブラリにおける動的 SQL サポートの一部であり、特に Informix との互換性が求められる環境で使用されます。sqlda.h や sqlda-native.h と並び、SQLDA のバリエーションを定義する役割を担います。

責務

  • **データ構造の定義**: SQL 文の入出力パラメータを保持するための sqlvar_compat（個々の変数情報）および sqlda_compat（記述子全体）の構造を定義する。
  • **互換性の維持**: Informix 形式の SQLDA 構造を再現し、既存の Informix アプリケーションが ECPG で動作するようにインターフェースを規定する。
  • **拡張性の確保**: 32KB を超えるデータに対応するための sqlilongdata フィールドなど、将来的な拡張や大規模データへの対応を考慮した構造を提供します。

重要なシンボル

  • struct sqlvar_compat: 各列やパラメータの型、長さ、データへのポインタ、インジケータ変数などを保持する構造体。
  • struct sqlda_compat: 記述子名、保持する変数の数、sqlvar_compat 配列へのポインタ、および連結リスト用の desc_next を含む SQLDA のメイン構造体。

追加の索引済みシンボル

  • sqlda_compat

制御フローとデータフロー

本ファイル自体はデータ構造の定義のみであり、ロジックを含みません。

  • **データフロー**: ECPG ランタイムライブラリが SQL 文を実行する際、この構造体を通じてアプリケーションとデータベース間でデータがやり取りされます。アプリケーションは sqlda_compat を準備し、ECPG がその内容を読み取ってバインド変数として使用、あるいは結果セットを格納します。

直接依存

  • なし（純粋な構造体定義のみ）。

関連コード

  • src/interfaces/ecpg/include/sqlda.h: SQLDA の汎用的な定義。
  • src/interfaces/ecpg/include/sqlda-native.h: ECPG ネイティブ形式の SQLDA 定義。
  • src/interfaces/ecpg/include/ecpg_informix.h: Informix 互換機能に関連する他の定義群。

読者向けメモ

  • 本構造体は Informix 互換を目的としているため、フィールドの配置やサイズは Informix の仕様に準拠しています。
  • sqlilongdata は 32KB を超えるデータを取り扱うための拡張フィールドです。古いアプリケーションとの互換性を維持しつつ、現代的なデータサイズに対応するために使用されます。
  • sqlflags や reserved フィールドは内部利用または将来の拡張用であり、アプリケーション側で直接操作すべきではありません。
