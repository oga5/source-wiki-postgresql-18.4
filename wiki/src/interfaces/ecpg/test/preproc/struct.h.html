src/interfaces/ecpg/test/preproc/struct.h

概要

本ファイルは、ECPG（Embedded SQL C Preprocessor）のプリプロセッサテスト用に使用されるデータ構造定義ヘッダーです。主にSQLのホスト変数として構造体を使用する際の、型マッピングとメモリレイアウトの検証を目的としています。

モジュール内での役割

src/interfaces/ecpg/test/preproc ディレクトリにおいて、ECPGがC言語の構造体をどのように認識し、SQLのデータ型とバインドするかをテストするための「型定義の雛形」を提供します。

責務

  • ECPGプリプロセッサが解析可能な構造体定義の提供。
  • 構造体メンバーの型（整数、文字列、浮動小数点数）のバリエーション定義。
  • NULLインジケータを伴う構造体と、そうでない構造体の対比定義。

重要なシンボル

  • struct mytype: ECPGのホスト変数として利用される標準的な構造体。文字列や浮動小数点数を含む。
  • struct mynulltype: 各フィールドに対応するNULLインジケータを模した構造体。ECPGのNULL値処理テストに使用される。

追加の索引済みシンボル

  • MYNULLTYPE
  • MYTYPE
  • mynulltype
  • mytype

制御フローとデータフロー

本ファイルは純粋なデータ定義ヘッダーであるため、実行時の制御フローは存在しません。ECPGプリプロセッサがこのヘッダーを読み込み、構造体のメンバーをSQLのホスト変数として登録する際の「メタデータ」として機能します。

直接依存

  • 特になし（標準的なC言語の構造体定義のみで構成されています）。

関連コード

  • src/interfaces/ecpg/test/preproc/strings.h: 本ファイルと同様に、テストケースで使用されるデータ定義を保持する関連ヘッダー。
  • ECPGのテストスイート内の .pgc ファイル: 本ヘッダーを #include し、実際にSQL文の中でこれらの構造体を利用するテストコード。

読者向けメモ

  • 本ファイルはプロダクションコードではなく、あくまでECPGのプリプロセッサの挙動を検証するためのテスト用定義です。
  • mynulltype の各メンバーは、mytype の各フィールドに対応するNULL状態を管理するためのプレースホルダーとして設計されています。
  • ECPGの型変換ルールを確認する際は、この構造体のメンバー構成と、対応する .pgc ファイル内での EXEC SQL 文の記述を照らし合わせてください。
