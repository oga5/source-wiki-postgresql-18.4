src/interfaces/ecpg/test/printf_hack.h

概要

printf_hack.h は、ECPG（Embedded C for PostgreSQL）のテストスイートにおいて、浮動小数点数（double）の出力形式をプラットフォーム間で統一するためのヘッダーファイルです。

モジュール内での役割

ECPGのテスト結果は、期待値との比較（diff）によって検証されます。しかし、printf("%g", x) の出力形式はOSやライブラリの実装に依存するため、特にWindows環境で指数表記の桁数が異なるとテストが誤って失敗します。本ファイルはこの差異を吸収する役割を担います。

責務

  • double 型の値を標準出力へ書き出す際のフォーマットを正規化する。
  • Windows環境特有の「3桁の指数表記（例: e+001）」を、他プラットフォーム標準の「2桁表記（例: e+01）」に変換する。

重要なシンボル

  • print_double(double x): 本ファイルで定義される唯一の静的関数。printf("%g", x) の代替として使用されます。

追加の索引済みシンボル

  • print_double

制御フローとデータフロー

1. **入力**: double 型の値を受け取ります。 2. **条件分岐**: プリプロセッサマクロ WIN32 の定義有無を確認します。

  • **非Windows環境**: そのまま printf("%g", x) を実行します。
  • **Windows環境**:
  • 一度 sprintf で文字列バッファに変換します。
  • 文字列を走査し、指数部分が3桁（e の後に 0 が続く形式）であるか判定します。
  • 条件に合致する場合、文字列を操作して指数部分を2桁に詰め、最後に printf で出力します。

直接依存

  • 本ファイルはヘッダーファイルですが、内部で標準Cライブラリの関数（sprintf, strlen, printf）を直接呼び出しています。

関連コード

  • src/interfaces/ecpg/test/pg_regress_ecpg.c: ECPGのテスト実行エンジン。テスト出力の比較を行う際に、この print_double を利用してプラットフォーム間の差異を排除します。

読者向けメモ

  • 本ファイルは static 関数として定義されているため、インクルードした各ソースファイル内で個別にコンパイルされます。
  • Windows環境でのみ文字列操作が発生するため、パフォーマンスが重視される箇所での使用には注意が必要ですが、テスト用ユーティリティであるため現状では許容されています。
  • 将来的に他のプラットフォームで同様のフォーマット差異が見つかった場合は、条件分岐の拡張が必要です。
