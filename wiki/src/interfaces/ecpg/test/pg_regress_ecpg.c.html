src/interfaces/ecpg/test/pg_regress_ecpg.c

概要

ECPG（Embedded SQL for C）の回帰テストを実行するためのドライバプログラムです。PostgreSQL の標準的なテストフレームワークである pg_regress を拡張し、ECPG 特有のソースコード変換やテスト結果の正規化を自動化します。

モジュール内での役割

src/interfaces/ecpg/test ディレクトリにおける実行エンジンです。シェルスクリプトで行われていたテスト実行処理を C 言語で再実装したものであり、プラットフォームに依存しない一貫したテスト環境を提供します。

責務

  • **ソースコードの正規化**: #line ディレクティブに含まれる環境依存のパス情報を除去し、比較可能な状態にします。
  • **テスト結果のフィルタリング**: 接続失敗時のエラーメッセージなど、環境によって変動する詳細情報を除去します。
  • **テストプロセスの管理**: 各テストケースの実行、標準出力・標準エラー出力のキャプチャ、および pg_regress への結果通知を行います。

重要なシンボル

  • ecpg_filter_source: ソースコード内の #line 指示子を書き換え、パス部分をファイル名のみに短縮します。
  • ecpg_filter_stderr: テスト実行時のエラーログから、ホスト名やパスなどの可変部分を削除し、結果の差異を抑止します。
  • ecpg_start_test: テスト対象のプログラムを起動し、入出力ファイルを準備します。
  • main: regression_main を呼び出し、テストスイート全体のライフサイクルを制御します。

追加の索引済みシンボル

  • ecpg_init
  • ecpg_postprocess_result

制御フローとデータフロー

1. **初期化**: main が regression_main を呼び出し、テスト実行のメインループを開始します。 2. **テスト実行**: ecpg_start_test が呼び出され、テスト対象プログラムを spawn_process で起動します。この際、ecpg_filter_source を用いてソースコードの正規化が行われます。 3. **結果処理**: テスト終了後、ecpg_postprocess_result が呼び出されます。.stderr ファイルが対象の場合、ecpg_filter_stderr が実行され、内容が正規化されます。 4. **比較**: 正規化された結果と期待値ファイルが pg_regress によって比較されます。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用共通ヘッダ。
  • common/string.h: 文字列操作ユーティリティ。
  • lib/stringinfo.h: 動的文字列バッファ管理。
  • pg_regress.h: pg_regress フレームワークのコア API。

関連コード

  • regression.h: テスト環境の設定や定数定義。
  • printf_hack.h: テスト実行時の出力フォーマット調整用。
  • pg_regress (外部): 本プログラムが利用するテスト実行エンジンの本体。

読者向けメモ

  • 本ファイルは pg_regress のコールバック関数を実装する形式をとっています。テストの追加や変更を行う際は、ecpg_start_test 内のパス生成ロジックと、ecpg_filter_* 関数による正規化ルールが現在のテスト要件に合致しているか確認してください。
  • ecpg_filter_stderr はインプレースでファイルを書き換えるため、処理失敗時のエラーハンドリングに注意が必要です。
