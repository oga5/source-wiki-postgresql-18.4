src/interfaces/ecpg/test/regression.h

概要

本ファイルは、ECPG（Embedded C for PostgreSQL）の回帰テストスイートで使用される共通のデータベース名およびユーザー名を定義するヘッダーファイルです。ECPGのプリプロセッサが解釈可能な EXEC SQL 構文を用いて、テスト環境の定数を一元管理しています。

モジュール内での役割

src/interfaces/ecpg/test ディレクトリにおけるテスト実行時の設定基盤です。テスト対象のデータベースやユーザーをハードコードするのではなく、本ファイルで定義することで、複数のテストケース間で一貫した環境変数を共有する役割を担います。

責務

  • テスト用データベース名（REGRESSDB1, REGRESSDB2）の定義。
  • テスト用ユーザー名（REGRESSUSER1, REGRESSUSER2）の定義。
  • ECPGプリプロセッサが認識可能な形式での定数提供。

重要なシンボル

  • REGRESSDB1, REGRESSDB2: テストで使用されるデータベース識別子。
  • REGRESSUSER1, REGRESSUSER2: テスト実行時に接続権限を持つユーザー識別子。

制御フローとデータフロー

本ファイルは実行コードを含まず、プリプロセッサによるソースコード変換時に参照されます。 1. ECPGプリプロセッサが本ファイルを読み込む。 2. EXEC SQL define 指令により、プリプロセッサのシンボルテーブルに定数が登録される。 3. テストソースコード内でこれらのシンボルが参照されると、プリプロセッサが実際の文字列値に置換し、最終的なCコードが生成される。

直接依存

  • 特になし（純粋な定義ファイルであり、他のヘッダーをインクルードしていません）。

関連コード

  • src/interfaces/ecpg/test/pg_regress_ecpg.c: 本ファイルで定義された定数を利用して、実際のテスト環境の構築や接続処理を行うメインのテストドライバです。

読者向けメモ

  • 本ファイルは標準的なC言語の #define ではなく、ECPG特有の EXEC SQL define を使用しています。そのため、CコンパイラではなくECPGプリプロセッサによって処理される点に注意してください。
  • テスト環境の名称を変更する必要がある場合は、本ファイルを修正することで、依存するすべてのテストコードに一括で変更を反映できます。
