src/interfaces/ecpg/test/expected/thread-alloc.c

概要

本ファイルは、ECPG（Embedded SQL for C）におけるマルチスレッド環境下でのメモリ割り当てと接続管理の安定性を検証するための回帰テスト用ソースコードです。プリプロセッサによって生成されたCコードの期待値として機能します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPGライブラリがスレッドセーフに動作し、かつ動的なメモリ確保・解放を正しく処理できるかを実証する役割を担います。

責務

  • 複数のスレッドを並列起動し、各スレッドで独立したデータベース接続を確立する。
  • SQLクエリの実行結果として動的に割り当てられたメモリ（r）を、各スレッド内で適切に解放する。
  • スレッド間でのリソース競合やメモリリークが発生しないことを確認する。

重要なシンボル

  • THREADS: 同時に実行するスレッド数（16）。
  • REPEATS: 各スレッドが実行するクエリの繰り返し回数（50）。
  • fn(): 各スレッドのエントリポイント。接続の確立、クエリ実行、メモリ解放、切断を行う。
  • ECPGdo(): ECPGの主要なSQL実行関数。ここではSELECT文の実行と結果の取得に使用。
  • sqlca: SQL通信領域。エラーハンドリングに使用。

追加の索引済みシンボル

  • fn
  • main
  • sqlca_t

制御フローとデータフロー

1. **初期化**: main() 関数が THREADS 分のスレッドを生成（Windowsなら _beginthreadex、POSIXなら pthread_create）。 2. **スレッド実行 (fn)**:

  • ECPGconnect で個別の接続を確立。
  • ループ内で ECPGdo を呼び出し、pg_class テーブルからデータを取得。
  • 取得したデータ（r）を free() で解放し、NULL で初期化するサイクルを繰り返す。

    • 3. **終了**: ECPGdisconnect で接続を閉じ、main() 側で全スレッドの終了を待機（WaitForMultipleObjects または pthread_join）。

読者向けメモ

  • 本ファイルは ecpg プリプロセッサの出力結果であるため、手動編集ではなくテスト生成プロセスの一部として扱われます。
  • r ポインタの解放（free(r)）は、ECPGが動的に割り当てたメモリをユーザーコードが責任を持って管理する必要があることを示す重要なパターンです。
  • WindowsとPOSIXの両環境に対応したスレッド実装が含まれており、クロスプラットフォームなテストコードの書き方の参考になります。
