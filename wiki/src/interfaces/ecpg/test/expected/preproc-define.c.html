src/interfaces/ecpg/test/expected/preproc-define.c

概要

本ファイルは、ECPG（Embedded SQL for C）プリプロセッサの機能テスト用ソースコードです。define.pgc から生成された C コードであり、ECPG における EXEC SQL TYPE 指令による型定義や、ホスト変数としての配列の扱いを検証します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内の回帰テストスイートの一部です。プリプロセッサが SQL 文中の型定義を正しく C 言語の型に変換し、データベースとの間で配列データを正しく受け渡しできるかを確認する役割を担います。

責務

  • EXEC SQL TYPE を用いたカスタム型（intarray）の定義と利用。
  • EXEC SQL BEGIN/END DECLARE SECTION 内での配列変数の宣言。
  • データベースに対するテーブル作成、データ挿入、および配列を用いた一括取得の実行。
  • 実行結果の標準出力への表示と、エラー発生時の sqlprint による診断。

重要なシンボル

  • intarray: EXEC SQL TYPE で定義された int[6] 型のエイリアス。
  • ECPGdebug: ECPG ライブラリのデバッグ出力を制御するマクロ。
  • sqlca: SQL 通信領域（SQL Communication Area）。実行結果のステータス確認に使用。
  • ECPGdo: ECPG が生成する、SQL 文を実行するための汎用関数。

追加の索引済みシンボル

  • main
  • string

制御フローとデータフロー

1. **初期化**: ECPGdebug を有効化し、データベースへ接続。 2. **DDL/DML実行**: create table でテーブルを作成し、insert でデータを投入。 3. **データ取得**: select * を実行し、結果をホスト変数（name, amount, letter 配列）に直接バインド。 4. **ループ処理**: sqlca.sqlerrd[2]（処理された行数）に基づき、取得した配列データを反復処理して出力。 5. **終了処理**: テーブルの削除、トランザクションのコミット、および接続の切断。

直接依存

  • ecpglib.h: ECPG ランタイムライブラリのコア機能。
  • ecpgerrno.h: エラーコード定義。
  • sqlca.h: SQLCA 構造体の定義。
  • stdlib.h, string.h, stdio.h: 標準 C ライブラリ。

関連コード

  • src/interfaces/ecpg/test/expected/preproc-type.c: 型定義に関連する他のテストケース。
  • src/interfaces/ecpg/test/expected/preproc-variable.c: ホスト変数の宣言と利用に関するテスト。
  • src/interfaces/ecpg/preproc/: ECPG プリプロセッサ自体のソースコード（本ファイルの生成元）。

読者向けメモ

  • 本ファイルはプリプロセッサによって自動生成された「期待される出力」です。手動で編集するものではありません。
  • EXEC SQL で始まる行は、プリプロセッサによって ECPGdo などのライブラリ関数呼び出しに置換されています。
  • sqlca.sqlcode のチェックは、各 SQL 文の直後で一貫して行われており、ECPG テストにおける標準的なエラーハンドリングパターンを示しています。
