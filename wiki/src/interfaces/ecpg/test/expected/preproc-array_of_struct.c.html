src/interfaces/ecpg/test/expected/preproc-array_of_struct.c

概要

ECPG（Embedded SQL for C）における「構造体の配列」および「構造体内のメンバ」に対する SQL 操作の回帰テスト用ソースコードです。プリプロセッサが構造体定義をどのように C 言語のホスト変数として展開し、データベースとの間でデータをやり取りするかを検証します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内のテストスイートの一つです。ECPG プリプロセッサが生成する C コードの正当性を確認するための期待値（expected output）として機能します。

責務

  • 構造体の配列をホスト変数として使用した SELECT 文の実行。
  • インジケータ変数を用いた NULL 値およびデータ状態のハンドリング。
  • 異なる構造体定義（typedef や匿名構造体）に対する ECPG の型変換の妥当性検証。

重要なシンボル

  • customer / customer2 / customer3 / customer4: テスト対象の構造体。それぞれ異なる定義方法（typedef や構造体宣言）で定義されています。
  • cust_ind: 構造体メンバに対応するインジケータ変数の構造体。
  • ECPGdo: ECPG が SQL 文を実行するために使用する主要なライブラリ関数。
  • sqlca: SQL 通信領域。エラーハンドリングや警告の確認に使用されます。

追加の索引済みシンボル

  • varchar_5

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、テスト用の customers テーブルを作成・挿入します。 2. **データフェッチ**: ECPGdo を呼び出し、SQL の結果セットを C 言語の構造体配列（custs1〜custs4）にバインドします。 3. **検証**: 取得したデータを printf で標準出力に書き出し、期待される結果と一致するかを確認します。 4. **終了**: ECPGdisconnect で接続を解除します。 各 SQL 操作の直後には、sqlca を参照してエラーや警告の有無をチェックする定型的なエラーハンドリングが挿入されています。

直接依存

  • ecpglib.h: ECPG ライブラリの基本関数。
  • ecpgerrno.h: ECPG のエラーコード定義。
  • sqlca.h: SQL 通信領域の定義。
  • stdio.h: 標準入出力関数。

関連コード

  • src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c: 構造体へのポインタを扱うテスト。
  • src/interfaces/ecpg/test/expected/sql-indicators.c: インジケータ変数の詳細な挙動を確認するテスト。
  • src/interfaces/ecpg/test/expected/preproc-variable.c: 基本的なホスト変数の宣言と使用。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成されたコードであるため、手動で編集するものではありません。
  • ECPGdebug(1, stderr) が有効になっており、実行時に詳細なデバッグログが標準エラー出力に出力されます。
  • 構造体のメンバ順序と ECPGdo に渡される引数の順序が、データベースの列定義と一致していることが重要です。
