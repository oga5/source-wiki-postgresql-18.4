src/interfaces/ecpg/test/expected/thread-thread_implicit.c

概要

本ファイルは、ECPG（Embedded SQL for C）におけるマルチスレッド環境での動作を検証するための回帰テストプログラムです。複数のスレッドが並行してデータベース接続を行い、同一テーブルに対して競合なく挿入処理を実行できるかをテストします。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPGのプリプロセッサが生成するCコードの期待値（またはそのソース）を提供します。本ファイルは特に、スレッドセーフな接続管理とトランザクション処理の整合性を確認する役割を担います。

責務

  • **スレッド管理**: pthread（POSIX）または Windows API を使用し、指定された数のスレッドを生成・同期する。
  • **並行接続テスト**: 各スレッドが個別の接続名を使用してデータベースに接続し、独立したトランザクションを実行する。
  • **整合性検証**: 全スレッドの処理終了後、挿入されたレコード数をカウントし、期待値と一致するかを判定する。

重要なシンボル

  • main: テストの全体フロー（テーブル作成、スレッド生成、結果検証）を制御する。
  • test_thread: 各スレッドが実行する関数。個別のデータベース接続を確立し、ループ内で INSERT を実行する。
  • nthreads / iterations: テストの負荷を決定する定数（デフォルトは10スレッド、各20回挿入）。
  • ECPGdo: ECPGライブラリのコア関数。SQL文の実行とホスト変数とのバインドを行う。

制御フローとデータフロー

1. **初期化**: main がテスト用テーブル test_thread を作成し、接続を閉じる。 2. **並行実行**: main が nthreads 分のスレッドを起動する。 3. **スレッド内処理**: 各 test_thread は thread_%03ld という一意な接続名を生成し、データベースへ接続。iterations 回の INSERT を実行し、コミットして切断する。 4. **同期**: main は全スレッドの終了を待機（pthread_join または WaitForMultipleObjects）。 5. **検証**: main が再接続し、テーブルの全行数を取得。期待値（nthreads * iterations）と比較して成功・失敗を出力する。

直接依存

  • ecpglib.h, sqlca.h: ECPGのランタイムライブラリおよびSQL通信領域。
  • pthread.h / windows.h: OS依存のスレッド制御用ヘッダー。
  • ecpg_config.h: ビルド環境設定。
  • ecpgerrno.h
  • locale.h
  • stdint.h
  • stdlib.h

関連コード

  • thread-thread.c: 類似のマルチスレッドテスト。接続管理の細かな挙動の違いを比較する際に有用。
  • sql-insupd.c: 基本的な挿入・更新処理のテスト。
  • src/interfaces/ecpg/ecpglib/: ECPGランタイムライブラリの実装。スレッドセーフな接続管理の仕組みを理解する際に参照。

読者向けメモ

  • 本プログラムは「回帰テスト」用であるため、デバッグ出力（ECPGdebug）は意図的に無効化されています。
  • スレッドの実行順序は不定であるため、テスト結果は「最終的なレコード数」という確定的な指標で評価されています。
  • Windows環境とPOSIX環境の両方に対応するため、スレッド生成と待機処理が #ifndef WIN32 で分岐されています。コードを追う際は、使用しているプラットフォームのブロックに注目してください。
