概要

本ファイルは、ECPG（Embedded C for PostgreSQL）におけるデータベース接続機能の回帰テスト用ソースコードです。ecpg プリプロセッサによって生成された C コードであり、単一のデータベースに対する多様な接続パターンと切断処理の挙動を検証します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPG ライブラリが提供する接続管理 API（ECPGconnect, ECPGdisconnect）の正当性を確認する役割を担います。

責務

  • データベース接続文字列の様々なバリエーション（ホスト名、Unixドメインソケット、接続パラメータ付きURL）のテスト。
  • ユーザー認証情報（ユーザー名、パスワード）の受け渡しテスト。
  • 接続名（Connection Name）を用いた接続・切断の管理。
  • 二重接続や存在しない接続先への切断など、境界条件におけるエラーハンドリングの確認。

重要なシンボル

  • main(): テストのメインエントリポイント。一連の接続・切断シーケンスを実行します。
  • ECPGconnect(): データベースへの接続を確立する ECPG ライブラリ関数。
  • ECPGdisconnect(): 指定した接続名または現在の接続を終了する関数。
  • ECPGdebug(): デバッグ出力を制御するマクロ。本ファイルでは stderr への出力を有効化しています。

制御フローとデータフロー

1. **初期化**: ECPGdebug を有効化し、デバッグ情報を出力可能にします。 2. **接続シーケンス**:

  • 固定文字列および変数を用いた接続を繰り返し実行します。
  • unix:postgresql:// 形式の接続文字列や、connect_timeout 等のパラメータを含むURLの解析をテストします。

    • 3. **切断シーケンス**:

  • ECPGdisconnect を呼び出し、接続名（main）を指定して明示的に切断します。

    • 4. **例外処理**: 存在しない接続名に対する切断試行など、異常系の挙動を確認します。
