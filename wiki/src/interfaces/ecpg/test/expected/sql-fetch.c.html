src/interfaces/ecpg/test/expected/sql-fetch.c

概要

本ファイルは、ECPG（Embedded SQL for C）のプリプロセッサが生成するテストコードの期待値です。SQLのカーソル操作（DECLARE, FETCH, MOVE, CLOSE）が正しくC言語コードへ変換されることを検証するための回帰テスト用ソースです。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPGの機能検証における「期待される出力」を定義します。プリプロセッサの変換ロジックが意図通りに動作しているかを確認する比較対象となります。

責務

  • SQLカーソルを用いたデータ取得の基本フローの提示。
  • FETCH 文におけるホスト変数へのデータバインドの検証。
  • WHENEVER 句によるエラーハンドリングおよび終了条件（NOT FOUND）の変換結果の提示。

重要なシンボル

  • ECPGdo: ECPGが生成する、SQL文を実行するための中心的なライブラリ関数。
  • sqlca: SQL通信領域。実行結果のステータス（sqlcode）や警告（sqlwarn）を保持します。
  • ECPGdebug: デバッグ出力を有効化するマクロ。

追加の索引済みシンボル

  • main

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、テスト用テーブル My_Table を作成・投入します。 2. **カーソル操作**: DECLARE でカーソルを定義し、FETCH をループ内で実行して結果をホスト変数 i および str に格納します。 3. **動的制御**: MOVE コマンドによるカーソル位置の調整や、パラメータ付きの FETCH を実行します。 4. **終了処理**: CLOSE でカーソルを閉じ、テーブル削除後に ECPGdisconnect で接続を終了します。 5. **エラー判定**: 各 ECPGdo 呼び出し直後に sqlca.sqlcode をチェックし、エラー発生時は exit(1) を実行するコードが生成されています。

直接依存

  • ecpglib.h: ECPGランタイムライブラリのインターフェース。
  • ecpgerrno.h: ECPGのエラーコード定義。
  • sqlca.h: SQL通信領域の構造体定義。
  • 標準Cライブラリ (stdio.h, stdlib.h, string.h): 出力およびプロセス制御用。

関連コード

  • src/interfaces/ecpg/test/expected/preproc-cursor.c: カーソル関連機能の別のテストケース。
  • src/interfaces/ecpg/test/expected/sql-declare.c: DECLARE 文の挙動に焦点を当てたテスト。
  • src/interfaces/ecpg/test/expected/sql-execute.c: SQL実行全般の変換結果を確認する際に参照。

読者向けメモ

  • 本ファイルは手書きではなく、ecpg プリプロセッサによって生成されたコードです。そのため、#line 指令が多用されており、元の .pgc ファイルとの対応関係が示されています。
  • ECPGdo 関数の引数は非常に複雑ですが、これはSQL文の文字列と、バインドされるホスト変数の型・サイズ・インジケータ変数をランタイムに伝えるための構造です。
  • sqlca.sqlcode のチェックは、プリプロセッサが WHENEVER 句に基づいて自動挿入する定型パターンです。
