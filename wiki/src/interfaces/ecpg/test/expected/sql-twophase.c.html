src/interfaces/ecpg/test/expected/sql-twophase.c

概要

本ファイルは、ECPG（Embedded C for PostgreSQL）のプリプロセッサが生成するテスト用ソースコードの期待値です。PostgreSQLの「二相コミット（Two-Phase Commit）」機能がECPG経由で正しく動作するかを検証するための回帰テストケースです。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPGプリプロセッサの出力結果が仕様通りであることを保証する役割を担います。このファイルは、プリプロセッサが .pgc ファイルをCコードに変換した後の「正解」として機能します。

責務

  • 二相コミットに関連するSQLコマンド（PREPARE TRANSACTION, COMMIT PREPARED）の実行フローの検証。
  • ECPGライブラリ関数（ECPGconnect, ECPGtrans, ECPGdo 等）の呼び出し形式の確認。
  • SQLエラー発生時の sqlca（SQL Communication Area）によるエラーハンドリングの動作確認。

重要なシンボル

  • main: テストのメインエントリポイント。一連のSQL操作を順次実行します。
  • ECPGdebug: デバッグ出力を制御するマクロ。本ファイルでは (X)+100 として再定義され、テスト実行時の詳細なログ出力を調整しています。
  • sqlca: SQL操作の結果やエラーコードを保持するグローバル構造体。各操作後に sqlcode をチェックします。

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、ECPGsetcommit で自動コミットを無効化します。 2. **準備**: create table を実行し、トランザクションを開始します。 3. **二相コミットの実行**:

  • insert 文を実行。
  • PREPARE TRANSACTION 'ecpg_twophase' を発行してトランザクションを準備状態へ移行。
  • COMMIT PREPARED 'ecpg_twophase' を発行して準備済みトランザクションを確定。

    • 4. **終了**: テーブルの削除と ECPGdisconnect による接続解除を行います。 各ステップで sqlca.sqlcode を確認し、負の値であれば sqlprint() を呼び出してエラーを出力します。

直接依存

  • ecpglib.h: ECPGランタイムライブラリの関数定義。
  • ecpgerrno.h: ECPGのエラーコード定義。
  • sqlca.h: SQL通信領域の構造体定義。
  • stdio.h, stdlib.h, string.h: 標準Cライブラリ。

関連コード

  • src/interfaces/ecpg/test/sql/twophase.pgc: 本ファイルの元となるソースコード（プリプロセッサへの入力）。
  • src/interfaces/ecpg/ecpglib/: ECPGのランタイムライブラリ実装。本ファイルが呼び出す各 ECPG 関数の実体が含まれます。

読者向けメモ

  • 本ファイルは手書きではなく、ecpg プリプロセッサによって自動生成されたコードの期待値です。そのため、コード内の #line 指示子は元の .pgc ファイルの行番号を指しています。
  • ECPGdebug マクロの再定義は、テスト環境においてデバッグ出力を制御するための意図的なオーバーライドです。
  • 二相コミットのテストでは、トランザクション名（ここでは 'ecpg_twophase'）の一貫性が重要です。
