src/interfaces/ecpg/test/expected/compat_informix-intoasc.c

概要

本ファイルは、ECPG（Embedded SQL for C）における Informix 互換ライブラリの intoasc 関数の動作を検証するための回帰テスト用ソースコードです。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPG プリプロセッサが生成する C コードの期待値（出力結果）を定義しています。Informix 互換モードでコンパイルされた際の、型変換関数の挙動を保証する役割を担います。

責務

  • interval 型のデータを文字列形式へ変換する intoasc 関数のテスト。
  • 変換先となるバッファ（dirty_str）のメモリ状態と、変換後の文字列内容の整合性確認。

重要なシンボル

  • intoasc: Informix 互換ライブラリ関数。interval 型を ASCII 文字列に変換します。
  • dirty_str: 変換結果を受け取るための固定長文字配列。
  • interval_ptr: テスト対象の interval 構造体へのポインタ。
  • ECPGdebug: デバッグ出力を制御するマクロ。テスト実行時のトレースに使用されます。

追加の索引済みシンボル

  • main

制御フローとデータフロー

1. **初期化**: interval_ptr にメモリを割り当て、time および month フィールドにテスト用の数値を設定します。 2. **変換実行**: intoasc(interval_ptr, dirty_str) を呼び出し、interval 構造体の値を文字列として dirty_str に書き込みます。 3. **検証**: 変換前後の dirty_str の内容を printf で出力し、期待される文字列形式になっているかを標準出力経由で確認します。

直接依存

  • ecpglib.h, sqlca.h: ECPG の基本ライブラリおよび SQL 通信領域定義。
  • ecpg_informix.h: Informix 互換関数群の宣言。
  • pgtypes_interval.h: interval 型の定義。
  • stdio.h, stdlib.h: 標準入出力およびメモリ管理。
  • ecpgerrno.h

関連コード

  • src/interfaces/ecpg/test/expected/compat_informix-sqlda.c: Informix 互換機能のより広範なテストケース。
  • src/interfaces/ecpg/test/expected/pgtypeslib-dt_test.c: interval 型や日付関連の型変換を扱う他のテスト。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成された「期待値」であるため、手動で編集するものではなく、ECPG のテストスイートの一部として扱われます。
  • intoasc は Informix の ESQL/C との互換性を維持するための関数であり、PostgreSQL ネイティブな実装とは異なるインターフェースを持つ点に注意してください。
