src/interfaces/ecpg/test/expected/compat_informix-rnull.c

概要

Informix 互換モードにおける ECPG の NULL 値処理（risnull および rsetnull）を検証するための回帰テスト用ソースコードです。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPG プリプロセッサが生成する C コードの期待値（テスト結果）を定義します。Informix 互換機能の正当性を確認する役割を担います。

責務

  • 各種データ型（char, short, int, bool, float, long, double, decimal, date, timestamp）に対する NULL 値の判定と設定の検証。
  • データベーステーブルへの挿入および取得を通じた、NULL 値の透過的なハンドリングの確認。

重要なシンボル

  • test_null(int type, char *ptr): 指定された型の変数が NULL かどうかを risnull で判定し、結果を標準出力するヘルパー関数。
  • main(): テストの実行エントリポイント。DB 接続、テーブル作成、データ挿入、データ取得、NULL 判定、テーブル削除の一連の流れを制御します。
  • ECPGdebug: デバッグ出力を有効化するためのマクロ。

追加の索引済みシンボル

  • main
  • test_null

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、検証用の test テーブルを作成します。 2. **データ挿入**:

  • 1回目の挿入: 通常の値を挿入。
  • 2回目の挿入: rsetnull を使用して全フィールドを NULL に設定してから挿入。

    • 3. **検証**:

  • select 文でデータを取得し、test_null を呼び出して各フィールドが期待通り NULL であるか（またはそうでないか）を判定します。

    • 4. **終了**: テーブルを削除し、ECPGdisconnect で接続を終了します。

直接依存

  • ecpglib.h, ecpgerrno.h, sqlca.h: ECPG の基本ライブラリおよびエラーハンドリング。
  • ecpg_informix.h: Informix 互換機能（risnull, rsetnull 等）の定義。
  • sqltypes.h: Informix 互換のデータ型定義。
  • stdlib.h: 標準ライブラリ。

関連コード

  • src/interfaces/ecpg/test/expected/compat_informix-sqlda.c: Informix 互換の SQLDA 構造体に関するテスト。
  • src/interfaces/ecpg/test/expected/compat_informix-test_informix.c: Informix 互換機能全般のテスト。

読者向けメモ

  • 本ファイルは ecpg プリプロセッサによって生成されたコードです。手動で編集する際は、元の .pgc ファイル（rnull.pgc）を修正し、プリプロセッサを再実行する必要があります。
  • risnull と rsetnull は Informix 互換モードでのみ提供される関数であり、標準の ECPG 動作とは異なる点に注意してください。
