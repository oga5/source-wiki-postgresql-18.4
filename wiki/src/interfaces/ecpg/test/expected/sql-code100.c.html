src/interfaces/ecpg/test/expected/sql-code100.c

概要

本ファイルは、ECPG（Embedded SQL for C）の回帰テスト用ソースコードの期待値です。SQL実行時に発生する「データが見つからない（SQLCODE 100）」という警告・エラー状態を、ECPGが正しく検知・処理できるかを検証します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内の他のテストファイルと同様、ECPGプリプロセッサが生成するCコードの正当性を確認するための参照用実装です。

責務

  • 存在しないレコードに対する UPDATE、DELETE、INSERT ... SELECT 操作を実行する。
  • 各操作後に sqlca.sqlcode をチェックし、期待される値（100）が返されることを確認する。
  • データベース接続からテーブル作成、データ挿入、クリーンアップまでの一連のライフサイクルを実証する。

重要なシンボル

  • sqlca: SQL通信領域（SQL Communication Area）。実行結果のステータスコードやエラーメッセージを保持する構造体。
  • ECPGdo: ECPGが生成する、SQL文を実行するための主要なライブラリ関数。
  • ECPGdebug: デバッグ出力を有効にするマクロ。本ファイルでは stderr への出力を強制するよう再定義されています。

追加の索引済みシンボル

  • main
  • sqlca_t

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、テスト用テーブル test を作成します。 2. **データ投入**: for ループを用いて test テーブルに初期データを挿入します。 3. **エラー条件の誘発**:

  • WHERE index = -1 という存在しない条件を指定して UPDATE、DELETE、INSERT を実行します。
  • 各操作後、sqlca.sqlcode が 100 であることを確認し、それ以外の場合のみエラーメッセージを出力します。

    • 4. **終了処理**: テーブルの削除と ECPGdisconnect を行い、接続を終了します。

直接依存

  • ecpglib.h: ECPGランタイムライブラリの関数定義。
  • ecpgerrno.h: ECPG固有のエラーコード定義。
  • sqlca.h: sqlca_t 構造体および sqlca グローバル変数の定義。
  • stdio.h: 標準入出力関数（printf 等）。

関連コード

  • src/interfaces/ecpg/test/sql/code100.pgc: 本ファイルの元となるソースコード（プリプロセス前）。
  • src/interfaces/ecpg/ecpglib/: ECPGランタイムライブラリの実装。ECPGdo や ECPGconnect の挙動を理解する上で重要です。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成されたコードであるため、手動で編集するものではありません。
  • sqlca.sqlcode == 100 は、SQL標準において「データが見つからない（No data found）」を示す成功に近い警告コードです。このテストは、ECPGがこのコードを適切にハンドリングし、アプリケーションが意図しないエラーとして扱わないようにするためのものです。
  • ECPGdebug マクロが定義されているため、実行時に詳細なログが stderr に出力されます。デバッグ時にはこの出力を参照してください。
