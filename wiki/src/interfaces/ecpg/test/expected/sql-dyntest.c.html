src/interfaces/ecpg/test/expected/sql-dyntest.c

概要

本ファイルは、ECPG（Embedded SQL for C）における動的SQLの機能検証を行うための回帰テスト用プログラムです。SQL記述子（Descriptor）を使用して、実行時にクエリ結果のメタデータ（列名、型、精度など）を取得し、その内容を標準出力へ表示する一連の処理を実装しています。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内に配置されたテストスイートの一部であり、ECPGプリプロセッサが生成するCコードの期待値（expected output）のソースです。動的SQLのメタデータ操作に関する機能の正当性を確認する役割を担います。

責務

  • 動的SQL用の記述子（MYDESC）の割り当てと解放。
  • SELECT クエリの準備とカーソルを用いたフェッチ処理。
  • ECPGget_desc を使用した、列の型、長さ、精度、スケール等のメタデータ抽出。
  • 抽出したメタデータに基づいた、動的なデータ型判定と値の表示。

重要なシンボル

  • ECPGallocate_desc / ECPGdeallocate_desc: SQL記述子のメモリ管理を行います。
  • ECPGget_desc: 記述子から特定の列属性やデータを取得します。
  • sqlca: SQL通信領域。各SQL操作後のエラーチェックに使用されます。
  • SQL3_... 系定数: SQL標準の動的データ型コードを定義する列挙型です。

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、テスト用テーブルを作成・挿入します。 2. **準備**: ECPGprepare でクエリを準備し、DECLARE CURSOR でカーソルを定義します。 3. **ループ処理**:

  • FETCH を実行し、結果を MYDESC 記述子に格納します。
  • ECPGget_desc_header で列数を取得します。
  • 各列に対して ECPGget_desc を呼び出し、メタデータを取得します。
  • switch 文で TYPE を判定し、適切な型としてデータを読み取り、標準出力へ整形して出力します。

    • 4. **終了**: カーソルを閉じ、記述子を解放して終了します。

直接依存

  • ecpglib.h: ECPGライブラリの主要関数群。
  • ecpgerrno.h: エラーコード定義。
  • sqlca.h: SQL通信領域の構造体定義。
  • stdio.h, stdlib.h: 標準入出力およびプロセス終了処理。

関連コード

  • src/interfaces/ecpg/test/expected/sql-sqlda.c: SQLDA（SQL Descriptor Area）を用いたより古い形式の動的SQLテスト。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成されたコードであるため、#line 指令が多用されています。
  • ECPGdebug(1, stderr) が有効になっており、実行時に詳細なデバッグログが標準エラーに出力されるよう設計されています。
  • SQL3_ 定数は、PostgreSQLがサポートする動的SQLの型システムを理解する上で重要なリファレンスとなります。
