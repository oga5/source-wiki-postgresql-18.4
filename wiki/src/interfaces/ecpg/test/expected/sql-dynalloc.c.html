src/interfaces/ecpg/test/expected/sql-dynalloc.c

概要

本ファイルは、ECPG（Embedded SQL for C）における動的SQLの割り当ておよび記述子（Descriptor）の使用方法を検証するための回帰テスト用ソースコードです。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPGプリプロセッサが生成するCコードの期待値（ゴールデンファイル）を提供します。プリプロセッサの変換ロジックが正しく動作しているかを検証する役割を担います。

責務

  • SQL記述子（mydesc）の動的割り当てと解放の検証。
  • SELECT 文の結果を記述子経由でホスト変数へ動的に取得するプロセスの実証。
  • NULL 値のインジケータ処理を含む、多様なデータ型（serial, numeric, varchar, timestamptz 等）のハンドリング。

重要なシンボル

  • main: テストの実行起点。データベース接続からクエリ実行、結果の出力、リソース解放までを制御します。
  • sqlca: SQL通信領域。各SQL文実行後のエラーコードや処理行数を確認するために使用されます。
  • ECPGallocate_desc / ECPGget_desc: 記述子の管理と、実行結果の動的取得を行うECPGライブラリ関数。

追加の索引済みシンボル

  • sqlca_t

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、テスト用テーブルを作成・データ投入します。 2. **記述子準備**: ECPGallocate_desc で mydesc を確保します。 3. **クエリ実行**: ECPGdo を使用して SELECT 文を実行し、結果を記述子に格納します。 4. **データ取得**: ECPGget_desc をループまたは個別に呼び出し、記述子からホスト変数（d1〜d9）およびインジケータ変数（i1〜i9）へデータを転送します。 5. **検証**: sqlca.sqlerrd[2]（処理行数）を基にループし、インジケータを確認しながら標準出力へ結果を表示します。 6. **終了処理**: ECPGfree_auto_mem でメモリを解放し、記述子の破棄と切断を行います。

直接依存

  • ecpglib.h: ECPGランタイムライブラリの関数定義。
  • sqlca.h: SQL通信領域の構造体定義。
  • ecpgerrno.h: エラーコード定義。
  • stdio.h, stdlib.h: 標準入出力およびメモリ管理。

関連コード

  • src/interfaces/ecpg/test/expected/sql-dynalloc2.c: 類似した動的割り当てのテストケース。
  • src/interfaces/ecpg/test/expected/sql-sqlda.c: SQLDA（SQL Descriptor Area）を用いたより低レイヤーな動的SQL処理の例。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成されたコードであるため、直接編集するものではありません。
  • ECPGdebug(1, stderr) が有効になっており、実行時に詳細なトレースが標準エラー出力に出力されます。
  • sqlprint() は sqlca のエラー情報を出力するヘルパー関数です。各SQL文の直後に sqlca.sqlcode をチェックする定型パターンが随所に現れます。
