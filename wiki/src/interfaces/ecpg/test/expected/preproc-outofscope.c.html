src/interfaces/ecpg/test/expected/preproc-outofscope.c

概要

ECPG（Embedded SQL for C）プリプロセッサの回帰テスト用ソースコードです。特に、カーソルの宣言と操作が異なる関数スコープにまたがって行われる「スコープ外（out-of-scope）」の動作を検証します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内のテストスイートの一つであり、ECPGが生成するCコードの正当性を確認する役割を担います。

責務

  • 関数間で共有されるカーソル操作の検証。
  • DECLARE、OPEN、FETCH、CLOSE の各SQL操作が、異なる関数スコープから正しく実行できることを確認する。
  • 構造体（MYTYPE）とインジケータ変数（MYNULLTYPE）を用いたデータバインドのテスト。

重要なシンボル

  • MYTYPE: データベースの行データを保持する構造体。
  • MYNULLTYPE: 各フィールドのNULL状態を追跡するためのインジケータ構造体。
  • get_var1(): カーソル操作に必要なメモリを確保し、変数を初期化する関数。
  • open_cur1() / get_record1() / close_cur1(): カーソルライフサイクルを管理する関数群。

制御フローとデータフロー

1. **初期化**: main 関数でデータベース接続とテーブル作成を行います。 2. **スコープ外準備**: get_var1 を呼び出し、ヒープ上に確保された構造体ポインタを main 側の変数に渡します。 3. **カーソル操作**: open_cur1 でカーソルを開き、get_record1 でループしながらデータをフェッチします。この際、ECPGget_var を介してポインタ経由でデータが構造体に書き込まれます。 4. **終了処理**: close_cur1 でカーソルを閉じ、メモリを解放して接続を切断します。

読者向けメモ

  • 本ファイルは ecpg プリプロセッサによって生成された後のコードです。ECPGdo マクロ内の複雑な引数は、プリプロセッサがSQL文をCの関数呼び出しに変換した結果です。
  • ECPGget_var は、ECPGset_var で登録された変数を取得するために使用されています。これは、関数をまたいでSQL変数を参照するためのECPGの内部的な仕組みです。
  • sqlca.sqlcode のチェックは、各SQL操作の直後に行われており、エラーハンドリングの標準的なパターンを示しています。
