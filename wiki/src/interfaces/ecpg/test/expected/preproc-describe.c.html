src/interfaces/ecpg/test/expected/preproc-describe.c

概要

本ファイルは、ECPG（Embedded SQL for C）プリプロセッサの DESCRIBE 文の動作を検証するための回帰テスト用ソースコードです。SQL 文のメタデータ取得機能が正しく動作することを、動的 SQL の準備と記述を通じて確認します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPG プリプロセッサが生成する C コードの期待値を提供します。このファイルは、プリプロセッサが .pgc ファイルをどのように C コードへ変換すべきかのリファレンスとして機能します。

責務

  • 動的 SQL 文（SELECT）の準備と実行計画の記述（DESCRIBE）の検証。
  • SQL 記述子（Descriptor）の割り当て、ヘッダー情報の取得、および各フィールド名の抽出。
  • 正常系における SQL 実行フロー（接続、テーブル作成、データ挿入、コミット、切断）の網羅。

重要なシンボル

  • main: テストの全工程を制御するエントリポイント。
  • ECPGdebug: SQL 実行時のデバッグ出力を有効化するマクロ。
  • sqlca: SQL 通信領域（SQLCA）。各操作後のエラーチェックに使用。
  • ECPGdescribe: SQL 文の情報を記述子に格納する主要関数。

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、テスト用テーブル t1 を作成・投入します。 2. **準備**: ECPGprepare を使用して SQL 文をプリコンパイルし、識別子を割り当てます。 3. **記述**: ECPGallocate_desc で記述子を確保し、ECPGdescribe で SQL 文のメタデータを取得します。 4. **抽出**: ECPGget_desc_header でフィールド数を取得し、ECPGget_desc で各フィールド名を取り出して printf で出力します。 5. **終了**: ECPGdeallocate_desc および ECPGdeallocate でリソースを解放し、ECPGdisconnect で終了します。

直接依存

  • ecpglib.h: ECPG ライブラリのコア関数定義。
  • ecpgerrno.h: エラーコード定義。
  • sqlca.h: SQLCA 構造体定義。
  • stdlib.h, string.h: 標準 C ライブラリ。

関連コード

  • src/interfaces/ecpg/test/expected/sql-describe.c: SQL レベルでの DESCRIBE 動作を確認する類似テスト。
  • src/interfaces/ecpg/test/expected/sql-sqlda.c: SQLDA（SQL Descriptor Area）に関連するより詳細なテスト。

読者向けメモ

  • 本ファイルはプリプロセッサによって自動生成されたコードの「期待値」であるため、手動で編集するものではありません。
  • ECPGdebug(1, stderr) が有効になっており、実行時に詳細な SQL トレースが標準エラー出力に流れます。
  • DESCRIBE の結果として得られるフィールド名が、期待通りに取得できているかを検証する構造になっています。
