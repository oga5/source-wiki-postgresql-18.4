src/interfaces/ecpg/test/expected/compat_informix-describe.c

概要

本ファイルは、ECPG（Embedded SQL for C）における DESCRIBE 文の動作を検証するための回帰テスト用ソースコードです。特に Informix 互換モードにおける SQL 記述子領域（SQLDA）と名前付き記述子の挙動をテストします。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPG プリプロセッサが生成する C コードの期待値を提供します。Informix 互換機能の正当性を確認するテストスイートの一環です。

責務

  • DESCRIBE 文を使用して、準備済みステートメントのメタデータを取得する。
  • SQL 記述子（sqlda_t）と名前付き記述子（DESCRIPTOR）の両方で同じメタデータが正しく取得できることを検証する。
  • 実行結果のメタデータ（フィールド名など）を標準出力し、期待値との比較を可能にする。

重要なシンボル

  • sqlda_t: Informix 互換モードで定義される SQL 記述子構造体。
  • sqlvar_t: sqlda_t 内で各列の情報を保持する構造体。
  • ECPGdescribe(): 準備済みステートメントの情報を記述子に格納する ECPG ライブラリ関数。
  • ECPGget_desc() / ECPGget_desc_header(): 記述子からメタデータを抽出する関数。

追加の索引済みシンボル

  • main

制御フローとデータフロー

1. **初期化**: データベース接続を行い、テスト用のテーブルを作成・挿入する。 2. **準備**: SELECT 文を ECPGprepare で準備する。 3. **記述**: ECPGdescribe を呼び出し、メタデータを名前付き記述子と sqlda_t 構造体の両方に格納する。 4. **検証**: ECPGget_desc_header で列数を取得し、sqlda_t 内の sqld フィールドと比較する。 5. **出力**: 各列のフィールド名を ECPGget_desc および sqlda_t から取得し、比較・表示する。 6. **終了**: 記述子の解放、メモリの解放、テーブルの削除を行い終了する。

直接依存

  • ecpglib.h, ecpgerrno.h, sqlca.h: ECPG の基本ライブラリとエラー処理。
  • ecpg_informix.h: Informix 互換機能の定義。
  • sqlda-compat.h / sqlda-native.h: 記述子構造体の定義（コンパイル環境に依存）。
  • stdlib.h
  • string.h

関連コード

  • src/interfaces/ecpg/test/expected/compat_informix-sqlda.c: SQLDA のより詳細な操作テスト。
  • src/interfaces/ecpg/test/expected/sql-describe.c: 非互換モードにおける DESCRIBE のテスト。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成されたコードであるため、#line 指示子が多用されています。
  • ECPGdebug(1, stderr) が有効になっており、実行時に詳細なトレースが標準エラー出力に出力されます。
  • Informix 互換モードでは DESCRIPTOR と SQLDA が明確に区別されるため、両方のパスをテストしている点に注目してください。
