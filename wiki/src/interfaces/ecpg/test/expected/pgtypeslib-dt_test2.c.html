src/interfaces/ecpg/test/expected/pgtypeslib-dt_test2.c

概要

PostgreSQL の ECPG ライブラリにおける日付・時刻型（pgtypes）の変換関数群を検証するための回帰テスト用ソースコードです。文字列から日付・時刻・期間型への変換、およびその逆変換の正確性を確認します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPG プリプロセッサが生成する C コードの期待値（またはテスト実行用コード）を提供します。pgtypeslib 関連の機能テストにおいて、特に複雑なフォーマットのパース能力を検証する役割を担います。

責務

  • 多様な日付フォーマット（ISO, 米国式, BC表記など）のパーステスト。
  • 時刻およびタイムスタンプの文字列変換の検証。
  • 期間（interval）型に対する加算演算およびコピー操作の検証。
  • メモリ管理（PGTYPESchar_free 等）の正当性の確認。

重要なシンボル

  • dates[], times[], intervals[]: テスト対象となる入力文字列の配列。エッジケースや不正なフォーマットを含みます。
  • PGTYPESdate_from_asc / PGTYPEStimestamp_from_asc: 文字列から各型への変換関数。
  • PGTYPESdate_to_asc / PGTYPEStimestamp_to_asc: 各型から文字列への変換関数。
  • PGTYPEStimestamp_add_interval: タイムスタンプと期間の加算を行う関数。

追加の索引済みシンボル

  • main

制御フローとデータフロー

1. **初期化**: main 関数内で ECPGdebug を有効化し、デバッグ出力を設定します。 2. **日付変換ループ**: dates[] 配列を順次走査し、PGTYPESdate_from_asc でパースを試みます。パース結果と endptr の状態を標準出力に書き出します。 3. **時刻結合テスト**: 日付のパースが成功した場合、times[] 配列の各要素と結合し、タイムスタンプとしてのパースおよび変換を検証します。 4. **期間演算テスト**: intervals[] 配列を走査し、PGTYPESinterval_from_asc で生成した期間を既存のタイムスタンプに加算します。また、PGTYPESinterval_copy を用いたメモリコピーの検証も行います。

直接依存

  • ecpglib.h, sqlca.h: ECPG の基本ライブラリおよびエラーハンドリング。
  • pgtypes_date.h, pgtypes_timestamp.h: 日付・時刻型定義および変換関数群。
  • stdio.h, string.h, stdlib.h, limits.h: 標準 C ライブラリ。
  • ecpgerrno.h

関連コード

  • src/interfaces/ecpg/test/expected/pgtypeslib-dt_test.c: 本ファイルと対になる基本的な日付・時刻テスト。
  • src/interfaces/ecpg/pgtypeslib/: pgtypes ライブラリの実装本体。本テストが検証している関数の定義元です。

読者向けメモ

  • 本ファイルはプリプロセッサによって処理された後のコードであり、#line 指令により元の .pgc ファイルとの対応が維持されています。
  • dates[] 配列の末尾にある長い文字列は、ParseDateTime() のバッファ境界や最大フィールド数をテストするための意図的なデータです。
  • times という変数名が libc の関数と衝突しないよう、static 宣言でスコープを制限しています。
