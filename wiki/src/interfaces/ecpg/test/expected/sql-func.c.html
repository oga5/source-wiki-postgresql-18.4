src/interfaces/ecpg/test/expected/sql-func.c

概要

本ファイルは、ECPG（Embedded SQL for C）プリプロセッサの回帰テスト用ソースコードです。PostgreSQLのストアドプロシージャ（関数）およびトリガーの実行機能が、ECPG経由で正しく動作することを検証します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内のテストスイートの一部であり、ECPGが生成するCコードの期待値（出力結果）を定義しています。プリプロセッサが .pgc ファイルを変換した後の、ライブラリ呼び出しの構造を提示します。

責務

  • データベースオブジェクト（テーブル、関数、トリガー）の動的な作成と削除の検証。
  • INSERT 操作に伴うトリガーの起動と、その結果の取得。
  • ECPGライブラリ関数（ECPGdo, ECPGconnect 等）の適切な呼び出しシーケンスの確認。

重要なシンボル

  • main: テストの実行エントリポイント。
  • ECPGdebug: ECPGライブラリのデバッグ出力を制御するマクロ。
  • sqlca: SQL通信領域。各SQL文実行後のエラーコードや警告を確認するために使用されます。

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、ECPGsetcommit でコミットモードを設定します。 2. **セットアップ**: ECPGdo を介して CREATE TABLE、CREATE FUNCTION、CREATE TRIGGER を順次実行します。 3. **実行**: INSERT 文を実行し、トリガーを介して Log テーブルにデータが書き込まれることを確認します。 4. **検証**: SELECT 文で Log テーブルから結果を取得し、text 変数に格納して printf で出力します。 5. **クリーンアップ**: 作成したトリガー、関数、テーブルを順次 DROP し、ECPGdisconnect で接続を終了します。

直接依存

  • ecpglib.h: ECPGランタイムライブラリの主要関数。
  • ecpgerrno.h: エラーコード定義。
  • sqlca.h: SQLCA構造体定義。
  • stdio.h, stdlib.h, string.h: 標準Cライブラリ。

関連コード

  • func.pgc: 本ファイルの元となるソースコード。
  • src/interfaces/ecpg/test/expected/sql-execute.c: SQL実行の基本動作を確認する関連テスト。
  • src/interfaces/ecpg/test/expected/sql-insupd.c: INSERT/UPDATE 操作の検証用テスト。

読者向けメモ

  • 本ファイルはプリプロセッサによって自動生成されたコードであるため、手動で編集するものではありません。
  • 各SQL文の直後には、sqlca を参照してエラーや警告をチェックする定型的なコードブロックが挿入されています。
  • ECPGdebug マクロが定義されており、テスト実行時に stderr へ詳細なログを出力する設定になっています。
