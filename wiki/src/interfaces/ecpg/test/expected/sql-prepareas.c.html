src/interfaces/ecpg/test/expected/sql-prepareas.c

概要

ECPG（Embedded C for PostgreSQL）における PREPARE および EXECUTE 文の動作検証を行うための回帰テスト用ソースコードです。特に、プリペアドステートメントの識別子に変数や特殊文字（スペース、ダブルクォート）を含む場合の挙動を網羅的にテストします。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリに配置されたテストスイートの一部であり、ECPGプリプロセッサが生成するCコードの期待値（出力結果）を定義します。

責務

  • プリペアドステートメントの多様な定義方法（定数名、変数名、特殊文字を含む識別子）の検証。
  • EXECUTE 文における定数リストおよび式リストの受け渡し動作の確認。
  • PREPARE ... AS 構文と EXECUTE ... (exprlist) 構文の組み合わせテスト。

重要なシンボル

  • main: テストのメインエントリポイント。DB接続からテーブル作成、各種 PREPARE/EXECUTE パターンの実行、後処理までを順次実行します。
  • check_result_of_insert: 挿入されたデータを SELECT で取得し、期待値と一致するかを確認するヘルパー関数。
  • ECPGdebug: ECPGライブラリのデバッグ出力を制御するマクロ。

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、テスト用テーブル test を作成します。 2. **テスト実行**: 以下のパターンを繰り返し実行します。

  • truncate test でテーブルを空にする。
  • ECPGprepare または ECPGdo (prepareモード) でステートメントを準備。
  • ECPGdo (executeモード) でステートメントを実行。
  • check_result_of_insert で結果を検証。

    • 3. **終了処理**: ECPGdeallocate でプリペアドステートメントを解放し、テーブルを削除して commit します。

直接依存

  • ecpglib.h: ECPGランタイムライブラリのコア機能。
  • sqlca.h: SQL通信領域（SQLCA）の定義。
  • ecpgerrno.h: エラーコード定義。
  • stdlib.h, string.h, stdio.h: 標準Cライブラリ。

関連コード

  • src/interfaces/ecpg/test/preproc/prepareas.pgc: 本ファイルの元となるソースコード（ECPGプリプロセッサへの入力）。
  • src/interfaces/ecpg/ecpglib/: ECPGのランタイムライブラリ実装。ECPGdo, ECPGprepare 等の挙動を理解する上で重要です。

読者向けメモ

  • 本ファイルはプリプロセッサによって自動生成されたコードであるため、直接編集するものではありません。
  • ECPGdo の引数には、SQL文の型やバインド変数の情報が詳細に埋め込まれています。ECPGの内部的なデータ受け渡しメカニズムを調査する際の参考資料として適しています。
  • 各テストケースで sqlca.sqlcode をチェックし、エラー発生時に sqlprint() を呼び出すことで、テストの失敗を標準エラー出力に記録する設計になっています。
