src/interfaces/ecpg/test/expected/connect-test6.c

概要

本ファイルは、ECPG（Embedded SQL for C）におけるデータベース接続の異常系テストケースです。特に、接続が切断された状態や存在しない接続名に対してECPG関数を呼び出した際の挙動（セグメンテーションフォールトの回避と適切なエラーコードの返却）を検証します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内の回帰テスト群の一部です。ECPGプリプロセッサが生成するCコードの期待値として機能し、ライブラリの堅牢性を保証する役割を担います。

責務

  • 接続済み状態から切断した後のディスクリプタ操作の検証。
  • 存在しない接続名に対する ECPGdeallocate_all の呼び出しテスト。
  • 存在しない接続名を用いたプリペアドステートメントの実行テスト。
  • 各操作後の sqlca.sqlcode を出力し、エラーハンドリングが正しく機能していることを確認する。

重要なシンボル

  • main: テストの実行エントリポイント。各テストケースを順次実行する。
  • ECPGdebug: デバッグ出力を有効化するマクロ。ここでは標準エラー出力へリダイレクトされる。
  • sqlca: SQL通信領域。実行結果のステータスコード（sqlcode）を確認するために参照される。

制御フローとデータフロー

1. **初期化**: データベースへ接続し、テスト用のテーブルを作成・挿入する。 2. **テスト1**: 接続を ECPGdisconnect で切断した後、ECPGget_desc を呼び出し、エラーが適切に捕捉されるか確認する。 3. **テスト2/3**: 存在しない接続名や NULL 接続に対して ECPGdeallocate_all を実行し、クラッシュせずにエラーコードが返ることを確認する。 4. **テスト4**: 存在しない接続名でプリペアドステートメントを定義・実行し、エラーハンドリングの挙動を検証する。

直接依存

  • ecpglib.h: ECPGライブラリの主要関数定義。
  • ecpgerrno.h: エラーコード定義。
  • sqlca.h: SQLCA構造体の定義。
  • stdlib.h, string.h, stdio.h: 標準Cライブラリ。

関連コード

  • src/interfaces/ecpg/ecpglib/: ECPGライブラリ本体。本テストが検証している ECPGdo, ECPGconnect, ECPGget_desc 等の実装が含まれる。
  • src/interfaces/ecpg/test/expected/connect-test1.c 〜 connect-test5.c: 他の接続関連のテストケース。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成されたコードであるため、#line 指令が多用されています。
  • 過去にセグメンテーションフォールトが発生していた箇所を修正するための回帰テストであるため、エラーハンドリングのロジックを追う際に有用です。
  • ECPGdebug マクロが定義されており、実行時に (X)+100 となる点は、デバッグレベルの調整を意図した実装上の工夫です。
