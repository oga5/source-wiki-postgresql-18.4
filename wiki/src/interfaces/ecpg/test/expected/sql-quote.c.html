src/interfaces/ecpg/test/expected/sql-quote.c

概要

本ファイルは、ECPG（Embedded SQL for C）における文字列リテラルのエスケープ処理および引用符の扱いを検証するための回帰テスト用ソースコードです。ecpg プリプロセッサによって生成された C コードであり、standard_conforming_strings 設定の切り替えに伴う挙動の変化を確認します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPG ライブラリの機能検証を担います。特に、SQL 文中の文字列リテラルがデータベースに対してどのように解釈・挿入されるかをテストする役割を持ちます。

責務

  • standard_conforming_strings の on/off を切り替え、エスケープシーケンスの挙動を検証する。
  • 引用符で囲まれたテーブル名（"My_Table"）に対する操作の正確性を確認する。
  • 挿入したデータが期待通りに格納されているかをカーソル操作を通じて検証する。

重要なシンボル

  • main: テストの実行エントリポイント。一連の SQL 操作を順次実行する。
  • ECPGdebug: ECPG のデバッグ出力を有効化するマクロ。
  • sqlca: SQL 通信領域。各 SQL 文実行後のエラーコードや警告を確認するために使用される。
  • ECPGdo: ECPG が生成する、SQL 文をデータベースへ送信するための汎用関数。

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、テスト用テーブルを作成する。 2. **設定変更**: standard_conforming_strings を off に設定し、文字列リテラルを挿入。その後 on に切り替え、同様の挿入を行う。 3. **検証**: カーソル C を宣言し、fetch を用いてテーブル内の全データを取得・出力する。 4. **終了**: トランザクションのロールバック、テーブルの削除、および ECPGdisconnect による接続解除を行う。 5. **エラーハンドリング**: 各 SQL 操作の直後に sqlca.sqlcode をチェックし、エラー発生時は即座に終了する。

直接依存

  • ecpglib.h: ECPG ランタイムライブラリのコア機能。
  • ecpgerrno.h: ECPG 固有のエラーコード定義。
  • sqlca.h: SQL 通信領域の構造体定義。
  • stdio.h, stdlib.h, string.h: 標準 C ライブラリ。

関連コード

  • src/interfaces/ecpg/test/expected/preproc-strings.c: 文字列処理に関連する他のテストケース。
  • src/interfaces/ecpg/test/expected/sql-parser.c: SQL 文の解析に関連するテスト。

読者向けメモ

  • 本ファイルは ecpg プリプロセッサの出力結果であるため、手動で編集するものではありません。
  • E'...' 構文（エスケープ文字列定数）と通常の文字列リテラルの挙動の違いを理解することが、このテストの核心です。
  • sqlca.sqlwarn[0] == 'W' のチェックは、SQL 実行時の警告を標準出力に流すための定型的な処理です。
