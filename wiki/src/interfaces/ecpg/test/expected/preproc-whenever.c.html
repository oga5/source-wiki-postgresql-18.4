src/interfaces/ecpg/test/expected/preproc-whenever.c

概要

ECPG（Embedded SQL for C）のプリプロセッサが生成する WHENEVER 文の動作を検証するための回帰テスト用ソースコードです。SQL実行時のエラーや警告に対する制御フローの分岐が正しくCコードに変換されるかを確認します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内の他のテストファイルと同様、プリプロセッサの出力結果を保持する「期待値」ファイルです。ECPGコンパイラが WHENEVER 指令をどのようにC言語の if 文や goto 文へ変換するかを定義しています。

責務

  • WHENEVER SQLERROR および SQLWARNING の各アクション（sqlprint, do, call, continue, goto, stop）の変換結果を提示する。
  • データベース接続、テーブル作成、クエリ実行、ロールバックといった一連のSQL操作を通じて、エラーハンドリングの網羅的なテストケースを提供する。

重要なシンボル

  • sqlca: SQL通信領域。SQL実行後のステータス（sqlcode, sqlwarn）を保持する構造体。
  • ECPGdo: ECPGが生成する、SQL文を実行するための主要なライブラリ関数。
  • sqlprint(): ECPGライブラリが提供する、エラー内容を標準エラー出力に表示する関数。
  • print(), print2(), warn(): テスト内で定義された、カスタムエラーハンドラ関数。

追加の索引済みシンボル

制御フローとデータフロー

1. **初期化**: ECPGdebug でデバッグモードを有効にし、データベースへ接続します。 2. **SQL実行**: ECPGdo を介してSQLを発行します。 3. **条件分岐**: 各 ECPGdo 呼び出しの直後に、sqlca.sqlcode や sqlca.sqlwarn をチェックするコードが挿入されます。 4. **ハンドリング**: プリプロセッサが生成した if 文により、エラー発生時には指定されたアクション（goto によるジャンプや exit による終了など）が実行されます。 5. **終了**: 全てのテストケースを完了後、exit(0) で正常終了します。

直接依存

  • ecpglib.h: ECPGライブラリの関数定義。
  • ecpgerrno.h: エラーコード定義。
  • sqlca.h: SQLCA構造体の定義。
  • stdlib.h: exit() 等の標準ライブラリ関数。

関連コード

  • preproc-whenever_do_continue.c: WHENEVER ... DO CONTINUE の挙動に特化したテスト。
  • src/interfaces/ecpg/preproc/: ECPGプリプロセッサ本体のソースコード。WHENEVER 文の変換ロジックが実装されています。

読者向けメモ

  • 本ファイルは手書きのコードではなく、ECPGプリプロセッサによって生成された「期待される出力」です。
  • sqlca.sqlcode < 0 のチェックが各SQL実行後に自動挿入されている点に注目してください。これがECPGにおけるエラーハンドリングの基本メカニズムです。
  • goto error; のような制御フローは、C言語の構造化プログラミングの観点からは非推奨に見えるかもしれませんが、ECPGの仕様上、エラーハンドリングの標準的な実装パターンとなっています。
