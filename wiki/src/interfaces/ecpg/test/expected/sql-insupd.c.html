src/interfaces/ecpg/test/expected/sql-insupd.c

概要

本ファイルは、ECPG（Embedded SQL for C）プリプロセッサの回帰テスト用ソースコードです。SQLの INSERT および UPDATE 文、特に RETURNING 句を伴う操作の動作検証を目的としています。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、プリプロセッサが生成するCコードの期待値（ゴールデンファイル）を提供します。ECPGがSQL文をどのように ECPGdo 関数呼び出しへ変換するかを確認するリファレンスです。

責務

  • INSERT および UPDATE 文の実行と結果の検証。
  • RETURNING 句によるデータ取得のテスト。
  • sqlca（SQL Communication Area）を用いたエラーハンドリングの検証。
  • データベース接続の確立から切断までのライフサイクル管理。

重要なシンボル

  • main: テストの実行エントリポイント。
  • ECPGdo: ECPGが生成する、SQL文を実行するための主要なライブラリ関数。
  • sqlca: SQL実行結果や警告を保持するグローバル構造体。
  • ECPGdebug: デバッグ出力を有効化するマクロ（本ファイルでは +100 のオフセットが適用されています）。

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、デバッグモードを有効化します。 2. **DDL実行**: create table を実行し、テスト用のテーブルを準備します。 3. **DML実行**:

  • INSERT 文でデータを投入し、RETURNING 句で値をホスト変数 i4 に格納します。
  • UPDATE 文でデータを更新し、RETURNING 句で更新後の値を配列 i3 に格納します。

    • 4. **検証**: SELECT 文で結果を配列 i1, i2 に取得し、printf で期待値と一致するかを確認します。 5. **終了**: ECPGdisconnect で接続を閉じます。

直接依存

  • ecpglib.h: ECPGランタイムライブラリの関数定義。
  • sqlca.h: SQLCA構造体定義。
  • ecpgerrno.h: エラーコード定義。
  • 標準Cライブラリ (stdio.h, stdlib.h, string.h)。

関連コード

  • src/interfaces/ecpg/test/expected/sql-execute.c: SQL実行の基本的なパターンを網羅する関連テスト。
  • src/interfaces/ecpg/test/expected/sql-fetch.c: カーソル操作を含むデータ取得のテスト。
  • src/interfaces/ecpg/ecpglib/: ECPGランタイムライブラリの実装（ECPGdo 等の定義元）。

読者向けメモ

  • 本ファイルはプリプロセッサによって自動生成されたコードであるため、手動で編集するものではありません。
  • ECPGdo の引数には、ホスト変数の型、ポインタ、サイズ、インジケータ変数の有無などが詳細にエンコードされています。これらはECPGの型変換ロジックを理解する上で重要なサンプルとなります。
  • sqlca.sqlcode < 0 のチェックは、ECPGにおける標準的なエラーハンドリングパターンです。
