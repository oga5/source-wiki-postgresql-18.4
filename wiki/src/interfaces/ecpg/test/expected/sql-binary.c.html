src/interfaces/ecpg/test/expected/sql-binary.c

概要

本ファイルは、ECPG（Embedded SQL for C）におけるバイナリデータ型（bytea）の取り扱いおよびバイナリカーソルの動作を検証するための回帰テスト用ソースコードです。ecpg プリプロセッサによって生成された C コードであり、データベースとの接続からテーブル操作、バイナリデータのフェッチまでの一連の処理を含みます。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内のテストスイートの一部として、ECPG ライブラリがバイナリデータを正しくシリアライズ・デシリアライズできるかを保証します。

責務

  • bytea 型カラムを含むテーブルの作成とデータ挿入の検証。
  • BINARY カーソルを使用したデータ取得の動作確認。
  • 取得したバイナリデータのメモリ管理（ポインタの解放など）の検証。
  • エンディアン依存性を考慮したデータ出力の制御。

重要なシンボル

  • struct TBempl: データベースのテーブル構造を模した C 言語の構造体。
  • ECPGdebug: ECPG のデバッグ出力を有効化するマクロ。
  • main: テストの実行エントリポイント。
  • sqlca: SQL 通信領域（SQL Communication Area）。各 SQL 文実行後のエラーチェックに使用。

追加の索引済みシンボル

  • TBempl

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、bytea_output を escape モードに設定します。 2. **テーブル操作**: empl テーブルを作成し、bytea 型カラムを含むデータを挿入します。 3. **カーソル操作**:

  • 通常のカーソル (C) でデータを取得し、標準出力に表示します。
  • BINARY カーソル (B, A) を宣言し、バイナリ形式でデータをフェッチします。

    • 4. **データ検証**: 取得したバイナリデータをループで走査し、8進数形式で出力して内容を確認します。 5. **終了処理**: pointer の解放および ECPGdisconnect による接続終了を行います。

直接依存

  • ecpglib.h: ECPG ライブラリの主要関数定義。
  • ecpgerrno.h: ECPG エラーコード定義。
  • sqlca.h: SQLCA 構造体の定義。
  • stdio.h, stdlib.h: 標準入出力およびメモリ管理用。

関連コード

  • src/interfaces/ecpg/test/expected/sql-bytea.c: bytea 型の基本的なテスト。
  • src/interfaces/ecpg/test/expected/sql-fetch.c: カーソル操作の基本テスト。
  • src/interfaces/ecpg/test/expected/sql-describe.c: 動的 SQL における型情報の確認。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成されたコードであるため、直接編集するものではありません。元のソースは binary.pgc です。
  • accs（short 型）の出力が意図的にスキップされているのは、プラットフォーム間のエンディアンの違いによるテストの失敗を防ぐためです。
  • バイナリデータのフェッチ時に pointer を使用している箇所は、動的なメモリ割り当てが行われるため、free() による解放が必須となります。
