src/interfaces/ecpg/test/expected/preproc-autoprep.c

概要

ECPG（Embedded SQL for C）における「自動プリペア（Automatic Prepare）」機能の動作を検証するための回帰テスト用ソースコードです。SQL文が明示的にプリペアされていない場合でも、ECPGライブラリが透過的にプリペア処理を行う挙動を確認します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPGプリプロセッサが生成するCコードの期待値（ゴールデンファイル）を提供します。このファイルは、プリプロセッサが .pgc ファイルを変換した後の最終的なCコードの構造を定義しています。

責務

  • SQL文の自動プリペア機能（ECPGst_prepnormal）の検証。
  • データベース接続、テーブル作成、データ挿入、カーソル操作、および切断の一連のライフサイクルの実行。
  • SQL実行結果のエラーハンドリング（sqlca を使用した警告・エラーの出力）。

重要なシンボル

  • test(): テストの主要ロジックをカプセル化した関数。main から二度呼び出されることで、再実行時の安定性を確認します。
  • ECPGdo(): ECPGのコア関数。SQL文の実行を担当し、第4引数で実行モード（ECPGst_prepnormal 等）を指定します。
  • ECPGprepare(): 明示的なプリペア処理のテストに使用されます。
  • sqlca: SQL通信領域。実行後のエラーコードや警告フラグを保持します。

追加の索引済みシンボル

  • test

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続。 2. **自動プリペア実行**: ECPGst_prepnormal モードで INSERT や SELECT を実行。ECPGライブラリが内部的にプリペア処理を代行します。 3. **明示的プリペア**: ECPGprepare を使用してプリペア済みステートメントを作成し、ECPGst_execute で実行するフローを検証します。 4. **カーソル操作**: DECLARE CURSOR を使用し、動的SQL（$1）を介したカーソル操作の挙動を確認します。 5. **終了**: DROP TABLE を実行し、ECPGdisconnect で接続を終了します。

直接依存

  • ecpglib.h: ECPGランタイムライブラリの関数定義。
  • ecpgerrno.h: ECPG固有のエラーコード定義。
  • sqlca.h: SQLCA構造体の定義。
  • stdio.h, stdlib.h, string.h: 標準Cライブラリ。

関連コード

  • src/interfaces/ecpg/test/expected/sql-prepareas.c: 明示的なプリペア処理のテスト。
  • src/interfaces/ecpg/test/expected/preproc-cursor.c: カーソル操作の詳細なテスト。
  • src/interfaces/ecpg/ecpglib/: ECPGランタイムライブラリの実装（ECPGdo 等の本体）。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成されたコードであるため、手動編集は推奨されません。
  • ECPGdebug(1, stderr) が有効になっており、実行時に詳細なデバッグログが標準エラー出力に出力されます。
  • test() 関数が2回呼ばれているのは、一度プリペアされたステートメントが再利用される際のキャッシュ機構や整合性をテストするためです。
