src/interfaces/ecpg/test/expected/preproc-whenever_do_continue.c

概要

ECPG（Embedded SQL for C）における WHENEVER 文の CONTINUE アクションを検証するための回帰テスト用ソースコードです。SQLエラー発生時にプログラムを停止させず、処理を継続させる挙動を確認します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内の他のテストファイルと同様、ECPGプリプロセッサが生成するCコードの期待値（出力結果）を定義する役割を担います。

責務

  • WHENEVER SQLERROR CONTINUE 指令が、プリプロセス後のCコードでどのようにエラーハンドリングを制御するかを実証する。
  • カーソル操作中のエラー発生時、ループを中断せずに次のイテレーションへ移行するフローの検証。
  • WHENEVER 指令の有効範囲と、明示的な CONTINUE による無効化の確認。

重要なシンボル

  • main: テストの実行起点。データベース接続からテーブル作成、データ挿入、カーソルフェッチ、後処理までの一連の流れを制御します。
  • sqlca: SQL通信領域。ECPGの実行結果（sqlcode）を保持し、WHENEVER 文の条件判定に使用されます。
  • ECPGdo: ECPGライブラリのコア関数。埋め込みSQL文を実際のデータベース操作へ変換します。

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、テスト用テーブル emp を作成します。 2. **データ準備**: INSERT 文によりサンプルデータを投入します。 3. **カーソル操作**: DECLARE でカーソルを定義し、FETCH をループ内で実行します。 4. **エラーハンドリング**: WHENEVER SQLERROR CONTINUE が設定されているため、FETCH 実行時にエラーが発生しても sqlca.sqlcode < 0 のチェックを経て continue 文が実行され、ループが継続されます。 5. **終了処理**: CLOSE および DROP TABLE を実行し、正常終了します。

直接依存

  • ecpglib.h: ECPGライブラリの主要関数定義。
  • ecpgerrno.h: ECPG固有のエラーコード定義。
  • sqlca.h: SQLCA構造体の定義。
  • stdlib.h: exit() 等の標準ライブラリ関数。

関連コード

  • src/interfaces/ecpg/test/expected/preproc-whenever.c: WHENEVER 文の他のアクション（STOP や GOTO）を検証する関連テスト。
  • src/interfaces/ecpg/test/expected/preproc-cursor.c: カーソル操作の基本動作を確認するためのテスト。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成された「期待値」であるため、手動で編集するものではありません。
  • ECPGdebug マクロが定義されており、実行時に stderr へ詳細なログを出力するように設定されています。デバッグ時にはこの出力を参照してください。
  • WHENEVER 文はプリプロセッサがソースコードを走査する際に、後続のすべてのSQL文に対してエラーハンドリングコードを自動挿入する仕組みであることを理解しておくことが重要です。
