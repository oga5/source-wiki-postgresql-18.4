src/interfaces/ecpg/test/expected/thread-descriptor.c

概要

本ファイルは、ECPG（Embedded SQL for C）におけるマルチスレッド環境下での SQL 記述子（Descriptor）の割り当ておよび解放処理の安定性を検証するための回帰テスト用ソースコードです。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPG プリプロセッサが生成する C コードの期待値を提供します。特に、スレッドセーフな記述子操作が正しく行われるかを検証するテストケースです。

責務

  • 複数スレッド（デフォルト 16）を並行起動し、各スレッド内で記述子の動的割り当てと解放を繰り返す。
  • ECPGallocate_desc および ECPGdeallocate_desc の呼び出しが、競合やメモリリークなしに完了することを確認する。
  • SQL エラー発生時に sqlprint() を呼び出し、エラー状態を標準出力へ出力する。

重要なシンボル

  • fn: 各スレッドが実行する関数。記述子の確保と解放を REPEATS 回繰り返す。
  • ECPGallocate_desc: SQL 記述子を割り当てる ECPG ライブラリ関数。
  • ECPGdeallocate_desc: 割り当てられた SQL 記述子を解放する ECPG ライブラリ関数。
  • sqlca: SQL 通信領域（SQLCA）。エラーコードの確認に使用される。

追加の索引済みシンボル

  • main
  • sqlca_t

制御フローとデータフロー

1. **初期化**: main 関数がプラットフォーム（Windows/POSIX）に応じてスレッドを生成。 2. **並行実行**: 各スレッドが fn を実行。 3. **記述子操作**: fn 内で ECPGallocate_desc を呼び出し、直後に ECPGdeallocate_desc を実行。各ステップで sqlca.sqlcode をチェックし、負の値であればエラーを出力。 4. **同期**: main 関数が WaitForMultipleObjects または pthread_join を使用して全スレッドの終了を待機し、正常終了する。

直接依存

  • ecpglib.h: ECPG ライブラリのコア機能。
  • sqlca.h: SQLCA 構造体定義。
  • pthread.h (POSIX) / windows.h (Windows): スレッド制御用ヘッダー。
  • ecpgerrno.h
  • locale.h
  • process.h
  • stdio.h

関連コード

  • thread-alloc.c: 記述子以外のリソース割り当てに関するスレッドテスト。
  • thread-prep.c: プリペアードステートメントのスレッドテスト。
  • sql-desc.c: 記述子操作の基本機能テスト。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成された「期待される出力」であるため、手動で編集するものではありません。
  • THREADS と REPEATS の定数は、負荷テストの規模を決定します。
  • Windows 環境と POSIX 環境の両方に対応するため、条件付きコンパイル（#ifdef WIN32）が多用されています。スレッド生成ロジックの差異を確認する際に注意してください。
