src/interfaces/ecpg/test/expected/sql-define.c

概要

本ファイルは、ECPG（Embedded SQL for C）プリプロセッサの回帰テスト用ソースコードです。特に、コマンドライン引数で定義されたマクロやプリプロセッサの定義が、複数のファイル間で意図せず引き継がれないことを検証するためのテストケースです。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPGプリプロセッサが生成するCコードの期待値（出力結果）を定義しています。このファイルは、プリプロセッサの定義スコープ管理が正しく機能しているかを検証する役割を担います。

責務

  • ECPGの EXEC SQL 文が正しくCコードに変換されていることの確認。
  • データベース接続、テーブル作成、データ挿入、およびクエリ実行の基本フローの検証。
  • コマンドライン経由で渡されたマクロ定義が、ファイル境界を越えて汚染されないことの検証。

重要なシンボル

  • main: テストの実行エントリポイント。
  • sqlca: SQL通信領域（SQL Communication Area）。エラーハンドリングに使用。
  • ECPGdo: ECPGが生成する、SQL文を実行するための主要なライブラリ関数。
  • ECPGdebug: デバッグ出力を有効にするマクロ。

追加の索引済みシンボル

  • sqlca_t

制御フローとデータフロー

1. **初期化**: ECPGdebug を呼び出し、デバッグログを有効化します。 2. **接続**: ECPGconnect を使用してテスト用データベースに接続します。 3. **DDL/DML実行**: ECPGdo を介して CREATE TABLE や INSERT を実行し、状態を構築します。 4. **検証**: SELECT 文を実行し、結果をホスト変数（i, s）に格納して printf で出力します。 5. **スコープ検証**: コマンドラインで定義されたシンボル（CMDLINESYM）の値をクエリ結果として取得し、期待値通りに再定義・リセットされているかを確認します。 6. **終了**: ECPGdisconnect で接続を解除します。

直接依存

  • ecpglib.h: ECPGランタイムライブラリの関数定義。
  • ecpgerrno.h: エラーコード定義。
  • sqlca.h: sqlca_t 構造体および sqlca グローバル変数の定義。

関連コード

  • define.pgc: 本ファイルの元となるプリプロセッサ入力ファイル。
  • define_prelim.pgc: 本テストの前提となる、マクロ定義のスコープをテストするための先行ファイル。
  • src/interfaces/ecpg/ecpglib/: ECPGランタイムライブラリの実装。

読者向けメモ

  • 本ファイルは ecpg プリプロセッサによって自動生成されたコードの「期待値」です。手動で編集するものではなく、テスト実行時に生成されるコードとの比較対象となります。
  • sqlca.sqlcode < 0 のチェックは、各 EXEC SQL 文の直後に挿入される定型的なエラーハンドリングパターンです。
  • マクロのスコープに関するテストであるため、コンパイル時のコマンドラインオプション（-D）がどのようにプリプロセッサに影響を与えるかを理解する上で重要なサンプルです。
