src/interfaces/ecpg/test/expected/sql-show.c

概要

PostgreSQL の埋め込み SQL (ECPG) における SHOW コマンドおよび SET コマンドの動作検証を行うための回帰テスト用ソースコードです。ECPG プリプロセッサによって生成された C コードの期待値として機能します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部であり、ECPG が SQL 文を正しく C 言語の関数呼び出し（ECPGdo）に変換できているかを検証するテストスイートの構成要素です。

責務

  • SET コマンドによるセッションパラメータの設定。
  • SHOW コマンドによるパラメータ値の取得と、ホスト変数への格納。
  • 取得した値の標準出力への表示による検証。
  • 実行時のエラーおよび警告のハンドリング（sqlca の確認）。

重要なシンボル

  • main: テストの実行エントリポイント。
  • ECPGdebug: デバッグ出力を有効にするマクロ（ここでは標準エラー出力へリダイレクト）。
  • ECPGdo: ECPG が生成する、SQL 文を実行するための中心的なライブラリ関数。
  • sqlca: SQL 通信領域。実行結果のステータス（sqlcode）や警告（sqlwarn）を保持します。

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続します。 2. **実行ループ**: 以下のパターンを繰り返します。

  • SET コマンドでパラメータを変更。
  • SHOW コマンドでパラメータを読み取り、ホスト変数 var に結果をバインド。
  • printf で var の内容を出力し、期待値との比較を可能にする。

    • 3. **終了**: ECPGdisconnect で接続を解除します。 4. **エラー処理**: 各 ECPGdo 呼び出しの直後に sqlca をチェックし、異常があれば sqlprint() を実行します。

直接依存

  • ecpglib.h: ECPG ランタイムライブラリのインターフェース。
  • ecpgerrno.h: エラーコード定義。
  • sqlca.h: SQL 通信領域の定義。
  • stdio.h, stdlib.h, string.h: 標準 C ライブラリ。

関連コード

  • src/interfaces/ecpg/preproc/: ECPG プリプロセッサ本体。このファイルが生成される元となる .pgc ファイルの処理ロジックが含まれます。
  • src/interfaces/ecpg/ecpglib/: ECPGdo や ECPGconnect などのランタイム関数の実装。

読者向けメモ

  • 本ファイルはプリプロセッサによって自動生成されたコードであるため、直接編集するものではありません。
  • ECPGdebug(1, stderr) が設定されているため、実行時には詳細な通信ログが標準エラー出力に流れます。
  • sqlca.sqlwarn[0] == 'W' や sqlca.sqlcode < 0 のチェックは、ECPG における標準的なエラーハンドリングの定型パターンです。
