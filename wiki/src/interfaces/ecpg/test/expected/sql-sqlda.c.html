src/interfaces/ecpg/test/expected/sql-sqlda.c

概要

本ファイルは、ECPG（Embedded SQL for C）における SQLDA（SQL Descriptor Area）インターフェースの動作を検証するための回帰テスト用ソースコードです。Informix 互換モードおよびネイティブモードの両方で、動的 SQL の実行と結果取得が正しく行えるかを確認します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPG プリプロセッサが生成するコードの期待値（テストケース）を提供します。SQLDA を用いた動的 SQL のバインドおよびフェッチ処理の標準的な実装例として機能します。

責務

  • SQLDA 構造体を用いた動的 SQL の準備、実行、および結果の取得。
  • sqlda_t 構造体への手動マッピングによる入力パラメータのバインド。
  • 取得した SQLDA データの型に応じたダンプ出力（dump_sqlda 関数）。
  • 複数レコードのフェッチおよびメモリ管理の検証。

重要なシンボル

  • sqlda_t: SQLDA を表現する主要な構造体。
  • sqlvar_t: SQLDA 内の各列情報を保持する構造体。
  • dump_sqlda(): SQLDA 内のデータを型に応じて解析し、標準出力へ書き出すデバッグ用関数。
  • ECPGdo(): ECPG の中心的な実行関数。SQLDA を引数として動的 SQL を処理します。

制御フローとデータフロー

1. **初期化**: データベース接続を行い、テスト用テーブル t1 を作成・挿入します。 2. **動的 SQL 実行**:

  • PREPARE 文で SQL を準備し、DECLARE でカーソルを定義します。
  • FETCH 時に ECPGt_sqlda 型を指定し、結果を outp_sqlda に格納します。

    • 3. **データ解析**: dump_sqlda が sqlda->sqlvar を走査し、sqltype に基づいて値をキャストして表示します。 4. **入力バインド**: inp_sqlda を手動で malloc し、sqlvar にバインド変数のアドレスをセットして EXECUTE を呼び出します。 5. **終了処理**: DEALLOCATE でステートメントを解放し、メモリを free して接続を閉じます。

読者向けメモ

  • 本ファイルは ecpg プリプロセッサによって生成されたコードです。手書きの .pgc ファイルを修正する際は、プリプロセッサの挙動を考慮してください。
  • dump_sqlda 関数は、新しいデータ型を追加する際のテンプレートとして有用です。
  • sqlda_t は _ECPG_INFORMIX_H マクロの有無により定義が切り替わるため、環境依存の挙動に注意してください。
