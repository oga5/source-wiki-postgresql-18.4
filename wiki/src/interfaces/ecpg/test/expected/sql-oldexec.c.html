src/interfaces/ecpg/test/expected/sql-oldexec.c

概要

本ファイルは、ECPG（Embedded SQL for C）の回帰テスト用ソースコードです。主に「動的SQLの実行（EXECUTE IMMEDIATE）」および「プリペアドステートメントを用いたカーソル操作」の動作検証を目的としています。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内の他のテストファイルと同様、ECPGプリプロセッサが生成するCコードの期待値（ゴールデンファイル）としての役割を果たします。

責務

  • 動的SQLの構築と実行の検証。
  • プリペアドステートメントへのパラメータバインドの検証。
  • カーソルを使用した複数行のフェッチと、その結果のホスト変数へのマッピングの検証。
  • SQLトランザクション制御（COMMIT）の確認。

重要なシンボル

  • main: テストの全工程を実行するエントリーポイント。
  • ECPGdebug: ECPGライブラリのデバッグ出力を制御するマクロ。
  • sqlca: SQL通信領域。各SQL操作後のエラーチェックに使用されます。
  • ECPGdo: ECPGの主要な実行関数。SQL文の実行、準備、カーソル操作を抽象化します。

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、テスト用テーブルを作成します。 2. **動的実行**: sprintf で構築したSQL文字列を ECPGst_exec_immediate を介して実行します。 3. **プリペアド実行**: ECPGprepare でSQLを準備し、ECPGst_execute でホスト変数をバインドして実行します。 4. **カーソル操作**: DECLARE CURSOR を使用し、FETCH で取得したデータをホスト変数配列（name, amount, letter）に格納します。 5. **結果出力**: sqlca.sqlerrd[2]（処理された行数）を基にループを回し、取得したデータを標準出力へ書き出します。 6. **終了処理**: テーブルの削除、トランザクションのコミット、接続の切断を行います。

直接依存

  • ecpglib.h: ECPGランタイムライブラリの関数定義。
  • sqlca.h: SQL通信領域の構造体定義。
  • ecpgerrno.h: エラーコード定義。
  • 標準Cライブラリ: stdlib.h, string.h, stdio.h。

関連コード

  • src/interfaces/ecpg/test/expected/sql-execute.c: 動的SQL実行の別のテストケース。
  • src/interfaces/ecpg/test/expected/preproc-cursor.c: カーソル操作の詳細なテスト。
  • src/interfaces/ecpg/ecpglib/: ECPGランタイムライブラリの実装（ECPGdo 等の本体）。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成されたコードであるため、手動で編集するものではありません。
  • sqlca.sqlcode < 0 のチェックは、各SQL操作の直後に挿入されており、ECPGの標準的なエラーハンドリングパターンを示しています。
  • データのフェッチ時に使用されている ECPGt_int や ECPGt_char などの型指定子は、ECPGがC言語の型とSQLの型をどのようにマッピングしているかを理解する上で非常に参考になります。
