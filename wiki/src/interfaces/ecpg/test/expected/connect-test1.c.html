src/interfaces/ecpg/test/expected/connect-test1.c

概要

PostgreSQL の埋め込み SQL（ECPG）におけるデータベース接続機能の回帰テスト用ソースコードです。ecpg プリプロセッサによって .pgc ファイルから生成された C コードであり、接続文字列のバリエーションや認証、接続管理の挙動を検証します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPG ライブラリが提供する接続 API（ECPGconnect / ECPGdisconnect）の期待される動作を定義しています。

責務

  • 様々な接続形式（TCP, Unix ドメインソケット, タイムアウト指定等）のテスト。
  • ユーザー名、パスワード、接続先データベースの組み合わせによる接続成功・失敗の検証。
  • ECPGdisconnect による接続終了処理の確認。

重要なシンボル

  • main: テストの実行エントリポイント。一連の接続・切断シーケンスを順次実行します。
  • ECPGconnect: データベースへの接続を確立する ECPG ライブラリ関数。
  • ECPGdisconnect: 確立された接続を終了する ECPG ライブラリ関数。
  • ECPGdebug: デバッグログ出力を有効化するマクロ。

制御フローとデータフロー

1. **初期化**: ECPGdebug を呼び出し、デバッグ出力を有効にします。 2. **接続テスト**: ECPGconnect を使用し、異なる接続文字列（URL形式、ホスト指定、認証情報）で接続を試みます。 3. **状態管理**: 接続ごとに ECPGdisconnect を呼び出し、接続状態をクリーンアップします。 4. **異常系テスト**: 存在しないデータベースや不正なポート、誤ったパスワードを指定し、接続失敗時の挙動を確認します。 5. **終了**: 全てのテストケース終了後、0 を返して正常終了します。

直接依存

  • ecpglib.h: ECPG ライブラリのコア機能。
  • ecpgerrno.h: エラーコード定義。
  • sqlca.h: SQL 通信領域（SQLCA）の定義。
  • stdlib.h, string.h, stdio.h: 標準 C ライブラリ。

関連コード

  • src/interfaces/ecpg/test/expected/connect-test2.c ～ connect-test6.c: 接続テストのバリエーションを補完する他のテストケース。
  • src/interfaces/ecpg/ecpglib/: ECPGconnect 等の実装が含まれるライブラリ本体。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成された「期待される出力」であるため、直接編集するものではありません。
  • ECPGconnect の引数には、接続名、ユーザー名、パスワード、接続文字列が含まれます。接続名が NULL の場合、デフォルトの接続が使用されます。
  • 接続失敗のテストケースでは、ECPGdisconnect が呼び出されないケースが含まれていますが、これは接続自体が確立されていないためです。
