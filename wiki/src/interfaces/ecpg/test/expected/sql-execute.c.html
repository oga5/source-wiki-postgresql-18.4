src/interfaces/ecpg/test/expected/sql-execute.c

概要

本ファイルは、ECPG（Embedded SQL for C）における動的SQL実行機能の回帰テスト用ソースコードです。ecpg プリプロセッサによって生成されたCコードの期待値として機能し、SQLの実行、準備、カーソル操作の正当性を検証します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPGライブラリが提供する動的SQLインターフェース（EXECUTE IMMEDIATE や PREPARE / EXECUTE）が、仕様通りに動作することを保証する役割を担います。

責務

  • 動的SQL文の構築と実行の検証。
  • PREPARE 文によるクエリの事前コンパイルと、パラメータ付き EXECUTE のテスト。
  • カーソルを用いた結果セットのフェッチと、ホスト変数へのデータマッピングの確認。
  • SQLエラー発生時の sqlca（SQL Communication Area）の挙動確認。

重要なシンボル

  • main: テストの全工程（接続、テーブル作成、データ操作、クエリ、切断）を順次実行するエントリポイント。
  • ECPGdo: ECPGの主要なランタイム関数。SQL文の実行、パラメータのバインド、結果の取得を抽象化します。
  • ECPGprepare / ECPGdeallocate: SQL文の準備および解放を行う関数。
  • sqlca: SQL実行結果のステータスやエラー情報を保持する構造体。

制御フローとデータフロー

1. **初期化**: ECPGdebug を有効化し、データベースへ接続します。 2. **DDL実行**: ECPGdo を用いてテスト用テーブルを作成します。 3. **動的SQL実行**:

  • ECPGst_exec_immediate: 文字列として構築されたSQLを即時実行します。
  • ECPGprepare + ECPGst_execute: パラメータ（$1）を含むクエリを準備し、値をバインドして実行します。

    • 4. **カーソル操作**: DECLARE CURSOR でクエリを定義し、FETCH を通じてホスト変数（name, amount, letter）に結果を格納します。 5. **終了処理**: テーブルの削除、トランザクションのコミット、データベースからの切断を行います。

直接依存

  • ecpglib.h: ECPGランタイムライブラリの関数定義。
  • ecpgerrno.h: ECPG固有のエラーコード定義。
  • sqlca.h: SQLCA構造体の定義。
  • 標準Cライブラリ: stdlib.h, string.h, stdio.h（メモリ操作や出力用）。

関連コード

  • src/interfaces/ecpg/test/expected/sql-dynalloc.c: 動的SQLにおけるメモリ割り当てのテスト。
  • src/interfaces/ecpg/test/expected/sql-prepareas.c: PREPARE 文のより詳細な検証。
  • src/interfaces/ecpg/ecpglib/: ECPGランタイムライブラリの実装（ECPGdo 等の本体）。

読者向けメモ

  • 本ファイルはプリプロセッサの出力結果であるため、手動編集用ではありません。
  • EXECUTE IMMEDIATE での引用符（\042）のエスケープ処理など、プリプロセッサがどのようにSQL文を変換しているかを観察するのに適しています。
  • sqlca.sqlerrd[2] を参照して、影響を受けた行数を確認するパターンは、ECPGテストにおける標準的な検証手法です。
