src/interfaces/ecpg/test/expected/preproc-strings.c

概要

本ファイルは、ECPG（Embedded SQL for C）プリプロセッサの文字列リテラル処理に関する回帰テストの期待値（出力結果）を定義するCソースコードです。strings.pgc からプリプロセスを経て生成されたコードであり、PostgreSQLの多様な文字列リテラル形式が正しくC言語の変数にマッピングされるかを検証します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPGのプリプロセッサが生成するコードの妥当性を確認するためのテストスイートを構成します。

責務

  • PostgreSQL特有の文字列リテラル（標準、エスケープ、Unicodeエスケープ、ドル記号引用符、ビット/16進数文字列）の取得テスト。
  • ECPGライブラリを介したSQL実行と、結果のC言語変数への格納および表示。
  • データベース接続の確立と切断のライフサイクル管理。

重要なシンボル

  • ECPGdebug: ECPGのデバッグ出力を制御するマクロ。テスト実行時に詳細なログを有効化するために使用されます。
  • main: テストの実行エントリポイント。SQLの実行から結果の標準出力への書き出しまでを制御します。
  • s1〜s8: SQLクエリの結果を受け取るために宣言されたC言語の文字列ポインタ変数。

制御フローとデータフロー

1. **初期化**: ECPGdebug を有効化し、ECPGconnect でテスト用データベースに接続します。 2. **環境設定**: set standard_conforming_strings to on を実行し、文字列リテラルの解釈ルールを固定します。 3. **データ取得 (1)**: ECPGdo を使用して、各種文字列リテラル（N'', E'', U&'', $$ 等）を一度に取得し、s1〜s6 に格納します。 4. **出力**: 取得した文字列を printf で標準出力に書き出します。 5. **データ取得 (2)**: ビット文字列および16進数文字列を s7, s8 に取得し、同様に出力します。 6. **終了**: ECPGdisconnect で接続を閉じ、プログラムを終了します。

直接依存

  • ecpglib.h: ECPGのランタイムライブラリ関数を提供。
  • ecpgerrno.h: エラーコード定義。
  • sqlca.h: SQL通信領域（SQLCA）の定義。

関連コード

  • strings.pgc: 本ファイルのソースとなるプリプロセッサ入力ファイル。
  • src/interfaces/ecpg/test/expected/sql-quote.c: SQL引用符の扱いに関する関連テスト。
  • src/interfaces/ecpg/test/expected/preproc-variable.c: 変数宣言とプリプロセッサの挙動に関するテスト。

読者向けメモ

  • 本ファイルは自動生成されたコードであるため、直接編集するのではなく、対応する .pgc ファイルを修正してプリプロセッサを再実行する必要があります。
  • ECPGdo の引数にある ECPGt_char や ECPGt_NO_INDICATOR は、ECPGがSQL型とC型をマッピングする際の内部表現です。これらはプリプロセッサが自動生成するものであり、手動で記述する際は注意が必要です。
