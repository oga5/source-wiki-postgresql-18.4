概要

本ファイルは、ECPG（Embedded SQL for C）における Informix 互換機能の回帰テスト用ソースコードです。プリプロセッサによって生成された C コードであり、データベース接続、テーブル操作、およびタイムスタンプ型の計算処理を検証します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPG のプリプロセッサが正しく SQL 文を C コードに変換できているかを検証する「期待値」としての役割を担います。

責務

  • データベースに対する基本的な CRUD 操作（CREATE, INSERT, SELECT, DROP）の実行。
  • PGTYPES ライブラリを用いたタイムスタンプおよびインターバル型の計算検証。
  • SQL 実行後のエラーハンドリングとトランザクション制御（ROLLBACK/COMMIT）の動作確認。

重要なシンボル

  • sql_check(): SQL 実行後の SQLCODE を監視し、エラー発生時に詳細な診断メッセージを出力してロールバックを行うユーティリティ関数。
  • sqlca: SQL 通信領域（SQLCA）。エラーコードや警告状態を保持するグローバル構造体。
  • ECPGdo(): 埋め込み SQL 文を実行するための主要なライブラリ関数。
  • ECPGdebug(): デバッグログの出力を制御するマクロ。

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、DateStyle を設定。 2. **テーブル操作**: history テーブルを作成し、初期データを挿入。 3. **データ処理**:

  • SELECT max(timestamp) で最新のタイムスタンプを取得。
  • 取得した値に対して PGTYPESinterval_from_asc と PGTYPEStimestamp_add_interval を使用して加算処理を実行。
  • 計算結果を新しいレコードとして INSERT。

    • 4. **終了処理**: トランザクションをコミットし、テーブルを削除して接続を切断。 5. **エラーフロー**: 各 SQL 操作後に sql_check を呼び出し、異常があれば即座に終了する。

読者向けメモ

  • 本ファイルは自動生成されたコード（.pgc ファイルの出力結果）であるため、直接編集するものではありません。
  • sql_check 関数内の ignore 引数は、特定の SQL ステートメントで期待される戻り値（例：データが存在しない場合の 100）を許容するために使用されています。
