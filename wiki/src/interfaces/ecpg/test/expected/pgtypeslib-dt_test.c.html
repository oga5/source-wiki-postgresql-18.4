src/interfaces/ecpg/test/expected/pgtypeslib-dt_test.c

概要

本ファイルは、ECPG（Embedded SQL for C）における日付、タイムスタンプ、および間隔（interval）型を扱う pgtypeslib の機能検証用テストプログラムです。ECPGプリプロセッサによって処理された後のCソースコードであり、回帰テストの一部として実行されます。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の中で、pgtypeslib のデータ型変換およびフォーマット機能の正確性を検証する役割を担います。データベースとの対話だけでなく、ライブラリ単体での文字列変換（asc ⇔ type）の整合性を確認します。

責務

  • ECPG経由でのデータベース接続およびテーブル操作の検証。
  • PGTYPES 系関数（date, timestamp, interval）の変換ロジックの網羅的テスト。
  • PGTYPESdate_fmt_asc や PGTYPEStimestamp_defmt_asc を用いた複雑なフォーマット解析の検証。
  • エラーハンドリング（check_errno）の動作確認。

重要なシンボル

  • main: テストの全工程を制御するエントリポイント。
  • check_errno: errno を解析し、pgtypes 固有のエラーコードを人間が読める形式で出力する補助関数。
  • ECPGdebug: ECPGのデバッグ出力を有効化するマクロ。
  • PGTYPES*: pgtypeslib が提供する日付・時刻操作用API群。

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、テスト用テーブルを作成。 2. **DB操作**: ECPGdo を介してSQLを実行し、日付・時刻データを挿入・選択してECPGの型マッピングを検証。 3. **型変換テスト**:

  • PGTYPES*_from_asc / to_asc を使用し、文字列と内部表現の相互変換を検証。
  • PGTYPESdate_fmt_asc 等でカスタムフォーマットの入出力をテスト。

    • 4. **エラー検証**: 不正な日付文字列を入力し、check_errno で期待されるエラーコードがセットされるかを確認。 5. **終了**: トランザクションのロールバックと接続の切断。

直接依存

  • ecpglib.h, sqlca.h: ECPGのコアライブラリおよびSQL通信領域。
  • pgtypes_date.h, pgtypes_timestamp.h, pgtypes_interval.h: 日付・時刻型定義および変換関数。
  • pgtypes_error.h: pgtypes 固有のエラーコード定義。
  • stdio.h, string.h, stdlib.h: 標準Cライブラリ。
  • ecpgerrno.h

関連コード

  • src/interfaces/ecpg/test/expected/pgtypeslib-dt_test2.c: 補完的な日付・時刻テスト。
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test.c: 数値型（numeric）の変換テスト。
  • src/interfaces/ecpg/pgtypeslib/: PGTYPES 関数の実装本体。

読者向けメモ

  • 本ファイルはプリプロセッサ生成物であるため、EXEC SQL 構文が展開された後のコード構造を理解するのに適しています。
  • regression mode で動作するため、出力結果は固定されており、回帰テストの期待値として機能します。
  • 複雑なフォーマット文字列（fmt）のテストケースが多いため、PGTYPES 関数の仕様を確認する際のドキュメント代わりとしても参照可能です。
