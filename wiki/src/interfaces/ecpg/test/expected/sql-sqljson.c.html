src/interfaces/ecpg/test/expected/sql-sqljson.c

概要

本ファイルは、ECPG（Embedded SQL for C）における SQL/JSON 関数の動作を検証するための回帰テスト用ソースコードです。ECPGプリプロセッサによって .pgc ファイルから生成された C コードであり、PostgreSQL の SQL/JSON 機能が ECPG 経由で正しく利用できるかをテストします。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPG の機能テストスイートを構成します。このファイルは、特定の SQL 構文（JSON 関連）がプリプロセッサによって適切に C コードへ変換され、実行時に期待通りの結果を返すことを保証する役割を担います。

責務

  • SQL/JSON 関数（json_object, json_array, json_scalar, json_serialize 等）の実行。
  • IS JSON 述語を含む複雑なクエリの実行と結果の検証。
  • ホスト変数（char 配列や bool 配列）へのクエリ結果のバインドと出力。
  • 実行時のエラーハンドリング（sqlca.sqlcode のチェック）。

重要なシンボル

  • main: テストのメインエントリポイント。
  • sqlca: SQL 通信領域（SQL Communication Area）。実行結果のステータス確認に使用。
  • ECPGdo: ECPG が生成する、SQL 文を実行するための主要なライブラリ関数。
  • json / is_json: テスト結果を受け取るためのホスト変数。

追加の索引済みシンボル

  • sqlca_t

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、ECPGsetcommit でコミットモードを設定します。 2. **クエリ実行**: ECPGdo を介して SQL/JSON クエリを送信します。 3. **データ取得**: クエリ結果は ECPGdo の引数として渡されたホスト変数（json や is_json）に格納されます。 4. **検証**: printf を使用してホスト変数の内容を標準出力に書き出し、期待値との比較を可能にします。 5. **終了**: ECPGdisconnect で接続を閉じます。

  • ecpgerrno.h: ECPG エラーコード定義。

  • stdio.h: 標準入出力関数。

読者向けメモ

  • 本ファイルは自動生成された「期待される出力」のソースコードです。直接編集するものではなく、ECPG のプリプロセッサの挙動を理解するためのリファレンスとして参照してください。
  • ECPGdo の引数には、SQL 文の文字列だけでなく、ホスト変数の型、サイズ、インジケータ変数の情報が含まれています。これらは ECPG が型安全にデータをやり取りするための重要な仕組みです。
  • 実行時にエラーが発生した場合、sqlca.sqlcode が負の値になるため、各クエリ実行後に sqlprint() を呼び出してエラー内容を確認するパターンが定型化されています。
