src/interfaces/ecpg/test/expected/compat_informix-sqlda.c

概要

本ファイルは、ECPG（Embedded SQL for C）における Informix 互換モードでの SQLDA（SQL Descriptor Area）の動作を検証するための回帰テスト用ソースコードです。動的 SQL の実行において、入力および出力の記述子をどのように扱い、データベースとデータをやり取りするかを実証します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリに配置されたテストスイートの一部であり、プリプロセッサによって生成された C コードの期待値（expected output）を定義します。Informix 互換機能の整合性を担保する役割を担います。

責務

  • sqlda_t および sqlvar_t 構造体を用いた動的 SQL の実行テスト。
  • FETCH 文における USING DESCRIPTOR 句の検証。
  • EXECUTE 文での入力記述子（inp_sqlda）の手動構築と出力記述子（outp_sqlda）の自動取得の検証。
  • 複数接続環境（con2）における記述子操作の正常性確認。

重要なシンボル

  • sqlda_t: SQLDA のメイン構造体。記述子内の列数や各列の属性を保持します。
  • sqlvar_t: 列ごとのデータ型、データポインタ、インジケータを保持する構造体。
  • dump_sqlda(): 記述子内のデータを標準出力にダンプするデバッグ用ヘルパー関数。
  • ECPGdo(): ECPG の中心的な関数。SQL 文の実行と記述子のバインドを仲介します。

追加の索引済みシンボル

  • dump_sqlda
  • main

制御フローとデータフロー

1. **初期化**: データベース接続を行い、テスト用テーブル t1 を作成・挿入します。 2. **動的取得**: PREPARE で SQL 文を準備し、DECLARE でカーソルを定義します。 3. **フェッチ**: FETCH 文で outp_sqlda を指定し、結果セットを記述子経由で取得します。 4. **入力バインド**: malloc で inp_sqlda を手動構築し、EXECUTE 文の入力パラメータとして渡します。 5. **クリーンアップ**: free() を用いて記述子メモリを解放し、接続を終了します。

直接依存

  • ecpglib.h, sqlca.h: ECPG ライブラリのコア機能。
  • ecpg_informix.h: Informix 互換機能の定義。
  • sqlda-compat.h / sqlda-native.h: 記述子構造体の定義（コンパイル環境に依存）。
  • sqltypes.h: SQL 型定数（SQLINT, SQLCHAR 等）の定義。
  • ecpgerrno.h
  • limits.h
  • stdlib.h
  • string.h

関連コード

  • src/interfaces/ecpg/test/expected/sql-sqlda.c: Informix 互換モードではない標準的な SQLDA テスト。
  • src/interfaces/ecpg/test/expected/sql-dynalloc.c: 動的メモリ割り当てに関連するテスト。

読者向けメモ

  • 本ファイルは ecpg プリプロセッサによって生成されたコードです。手動で編集するものではなく、テストの期待値として参照してください。
  • sqlda_t の構造は _ECPG_INFORMIX_H マクロの定義有無により sqlda-compat.h か sqlda-native.h のいずれかに切り替わります。
  • dump_sqlda 関数内の switch 文は、主要な SQL 型の表示ロジックを理解するのに適しています。
