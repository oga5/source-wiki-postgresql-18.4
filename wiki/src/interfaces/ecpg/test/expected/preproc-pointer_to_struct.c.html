src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c

概要

本ファイルは、ECPG（Embedded SQL for C）プリプロセッサが、構造体へのポインタをホスト変数として扱う際の挙動を検証するための回帰テスト用ソースコードです。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、プリプロセッサが生成するCコードの正当性を確認する役割を担います。特に、構造体ポインタを用いたデータフェッチや、インジケータ変数との連携が正しく変換されるかを検証します。

責務

  • 構造体ポインタ（customer*, customer2* 等）を用いたSQLクエリの実行。
  • 異なる構造体定義（varchar型を含むもの、含まないもの）に対するデータマッピングの検証。
  • ECPGdo 関数による動的SQL実行時のメモリレイアウトとポインタ参照の整合性確認。

重要なシンボル

  • customer, customer2, customer3, customer4: テスト対象となる構造体定義。それぞれ異なるメンバ構成を持ち、プリプロセッサの型解析能力を試します。
  • cust_ind: SQLのNULL値等を判定するためのインジケータ構造体。
  • ECPGdo: ECPGが生成する、SQL文を実行するための主要なライブラリ関数。
  • sqlca: SQL通信領域。エラーハンドリングや警告の確認に使用されます。

追加の索引済みシンボル

  • main
  • varchar_1
  • varchar_2
  • varchar_3
  • varchar_4

制御フローとデータフロー

1. **初期化**: malloc を使用して、テスト用の構造体配列をヒープ上に確保します。 2. **データベース操作**: ECPGconnect で接続後、customers テーブルを作成し、サンプルデータを挿入します。 3. **データフェッチ**: 複数の ECPGdo 呼び出しにより、確保した構造体ポインタへ直接データをフェッチします。この際、sizeof を用いたメモリサイズ指定が正しく行われているかを確認します。 4. **出力**: フェッチしたデータを printf で標準出力し、期待値との比較を可能にします。 5. **終了**: ECPGdisconnect で接続を解除し、プロセスを終了します。

直接依存

  • ecpglib.h: ECPGライブラリのコア機能。
  • ecpgerrno.h: エラーコード定義。
  • sqlca.h: SQLCA構造体の定義。
  • stdio.h, stdlib.h: 標準入出力およびメモリ管理。

関連コード

  • preproc-array_of_struct.c: 構造体の配列を扱うテスト。本ファイルと対照的な検証を行います。
  • sql-sqlda.c: 動的SQL記述子領域（SQLDA）を扱うテスト。より複雑なデータ構造のハンドリングを理解する助けになります。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成された「期待される出力」であるため、手動で編集するものではありません。
  • ECPGdebug(1, stderr) が有効になっており、実行時に詳細なデバッグログが標準エラー出力に出力されます。
  • 構造体のメンバに varchar 型が含まれる場合、len と arr の構造がECPGの期待する形式と一致している必要があります。
