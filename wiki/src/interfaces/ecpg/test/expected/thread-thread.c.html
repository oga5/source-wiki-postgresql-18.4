src/interfaces/ecpg/test/expected/thread-thread.c

概要

ECPG（Embedded SQL for C）におけるマルチスレッド環境での動作検証を行うための回帰テストプログラムです。複数のスレッドがそれぞれ独立したデータベース接続を確立し、共有テーブルに対して並行してデータを挿入する動作をシミュレートします。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内のテストスイートの一つであり、ECPGライブラリがスレッドセーフであることを確認する役割を担います。

責務

  • 複数のスレッドを生成し、各スレッドで個別のデータベース接続を管理する。
  • 共有テーブル test_thread に対する並行挿入処理を実行する。
  • 全スレッド終了後に、挿入された行数が期待値と一致するかを検証する。
  • POSIXスレッド（pthread）およびWindowsスレッドの両環境に対応する。

重要なシンボル

  • main: テストのメインエントリポイント。テーブルの初期化、スレッドの生成・待機、最終結果の検証を行う。
  • test_thread: 各スレッドで実行される関数。個別の接続名（thread_XXX）を生成し、ループ内でSQL挿入を実行する。
  • nthreads, iterations: テストの規模を決定するグローバル変数（デフォルトは10スレッド、各20回挿入）。

制御フローとデータフロー

1. **初期化**: main がデータベースに接続し、test_thread テーブルを再作成して切断する。 2. **並行処理**: main が nthreads 分のスレッドを生成し、各スレッドは test_thread を呼び出す。 3. **スレッド内処理**: 各スレッドは ECPGconnect で個別の接続を確立し、INSERT 文を iterations 回実行して COMMIT する。 4. **終了処理**: main は全スレッドの終了を待機（pthread_join または WaitForMultipleObjects）する。 5. **検証**: main が再度接続し、テーブルの全行数をカウントして、期待値（nthreads * iterations）と比較する。

直接依存

  • ecpglib.h, sqlca.h: ECPGランタイムライブラリおよびエラーハンドリング用。
  • pthread.h (POSIX) / windows.h (Windows): スレッド制御用。
  • ecpg_config.h: 環境設定。
  • ecpgerrno.h
  • locale.h
  • stdint.h
  • stdlib.h

関連コード

  • thread-thread_implicit.c: 同様のテストだが、接続を明示的に管理しないケースを扱う。
  • thread-alloc.c, thread-prep.c: スレッド環境における他のECPG機能（メモリ割り当てやプリペアードステートメント）のテスト。

読者向けメモ

  • 本ファイルは .pgc ファイルからECPGプリプロセッサによって生成されたコードです。
  • スレッドごとに異なる接続名を使用することで、ECPGライブラリがスレッドごとのコンテキストを正しく分離できているかを検証しています。
  • デバッグ出力（ECPGdebug）は、スレッドの実行順序が不定であるため、回帰テストでは無効化されています。
