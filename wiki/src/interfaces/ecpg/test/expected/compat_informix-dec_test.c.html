src/interfaces/ecpg/test/expected/compat_informix-dec_test.c

概要

Informix 互換の decimal 型ライブラリの機能検証を行うための回帰テスト用ソースコードです。文字列から decimal 型への変換、算術演算、型変換、および NULL 値の取り扱いを網羅的にテストします。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内のテストスイートの一つであり、ECPG が生成する Informix 互換コードの正確性を保証します。特に pgtypes ライブラリの decimal 関連関数の挙動を確認する役割を担います。

責務

  • 文字列から decimal への変換 (deccvasc) とその逆 (dectoasc) の検証。
  • decimal 型と long / int / double 型間の相互変換の検証。
  • decimal 型に対する加算 (decadd)、減算 (decsub)、乗算 (decmul)、除算 (decdiv)、比較 (deccmp) の実行。
  • risnull および rsetnull を用いた NULL 値の管理と判定のテスト。

重要なシンボル

  • decs[]: テスト対象となる文字列の配列。正数、負数、指数表記、無効な値などを含む。
  • main(): テストのメインループ。decs 配列を走査し、各関数を呼び出して結果を標準出力に書き出す。
  • check_errno(): 発生したエラーコードを解析し、人間が読める形式で出力するヘルパー関数。
  • print_double(): プラットフォーム間で一貫した浮動小数点数の出力を保証するためのラッパー。

追加の索引済みシンボル

  • check_errno
  • main
  • print_double

制御フローとデータフロー

1. **初期化**: decs 配列の各要素に対し、PGTYPESdecimal_new() でメモリを確保。 2. **変換テスト**: deccvasc で文字列を decimal に変換し、dectoasc で異なる精度指定で文字列に戻して結果を比較。 3. **型変換テスト**: dectolong や dectoint を経由して数値型との整合性を確認。 4. **算術演算**: 確保した decimal 配列の全組み合わせに対して四則演算を行い、結果を検証。 5. **NULL テスト**: rsetnull で明示的に NULL を設定し、risnull で正しく判定できるかを確認。 6. **終了処理**: 確保したメモリを PGTYPESdecimal_free() で解放。

直接依存

  • ecpglib.h, ecpg_informix.h: ECPG および Informix 互換機能のコア定義。
  • pgtypes_numeric.h, decimal.h: decimal 型の定義と操作関数。
  • sqlca.h, sqltypes.h: SQL 関連の型とエラー管理。
  • stdio.h, stdlib.h: 標準入出力およびメモリ管理。
  • ecpgerrno.h
  • pgtypes_error.h

関連コード

  • src/interfaces/ecpg/pgtypeslib/: decimal.c など、本テストが検証している実体関数の実装が含まれるディレクトリ。
  • src/interfaces/ecpg/test/expected/compat_informix-sqlda.c: Informix 互換の SQLDA 構造体テスト。
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test.c: numeric 型に特化したテスト。

読者向けメモ

  • 本ファイルは ECPG のプリプロセッサによって処理された後の「期待される出力」として扱われます。
  • print_double 関数は、Windows とそれ以外のプラットフォームで指数表記の桁数が異なる問題を吸収するために存在します。
  • deccmp などの関数は、NULL 値が渡された場合にエラーを返さず 0 を返す仕様があるため、テストコード内では出力パラメータの初期化を行ってから演算を実行しています。
