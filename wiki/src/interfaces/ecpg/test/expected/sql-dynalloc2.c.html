src/interfaces/ecpg/test/expected/sql-dynalloc2.c

概要

本ファイルは、ECPG（Embedded SQL for C）における動的SQL実行および記述子（Descriptor）を使用したデータ取得の回帰テスト用ソースコードです。dynalloc2.pgc からプリプロセッサによって生成されたCコードであり、動的に割り当てられたメモリ領域への結果セットの格納と、NULL値のハンドリングを検証します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内のテストスイートの一部として、ECPGの動的SQL機能が正しく動作することを保証します。特に、DESCRIBE 相当の操作や、実行時に列数が決定するクエリの結果をホスト変数にマッピングする際の挙動を確認します。

責務

  • データベース接続の確立とテスト用テーブルの作成・データ投入。
  • ECPGallocate_desc を用いた記述子の確保。
  • ECPGdo を用いた動的クエリの実行と、結果の記述子への格納。
  • ECPGget_desc を使用した記述子からのデータおよびインジケータ（NULL判定用）の取得。
  • テスト終了後のリソース解放（ECPGfree_auto_mem、ECPGdeallocate_desc）。

重要なシンボル

  • sqlca: SQL通信領域。各SQL文実行後のエラーチェックに使用されます。
  • ECPGallocate_desc / ECPGdeallocate_desc: SQL記述子のライフサイクルを管理します。
  • ECPGget_desc: 記述子から特定の列データやインジケータ値を取り出します。
  • ECPGdo: ECPGの主要な実行関数。SQL文の送信と結果のバインドを行います。

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、テスト用テーブル test を作成します。 2. **データ投入**: INSERT 文により、数値およびNULL値を含むレコードを投入します。 3. **動的取得**: SELECT * を実行し、結果を mydesc という名前の記述子にバインドします。 4. **データ抽出**: ECPGget_desc を呼び出し、各列のデータ（ip1, cp2）とインジケータ（ipointer1, ipointer2）を取得します。 5. **検証**: インジケータを確認し、NULLであれば "NULL" を、値があればその内容を標準出力へ書き出します。 6. **終了処理**: ECPGfree_auto_mem で自動割り当てメモリを解放し、記述子の破棄と切断を行います。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成されたコードであるため、手動で編集するものではありません。
  • ECPGdebug(1, stderr) が有効になっており、実行時に詳細なデバッグログが標準エラー出力に出力されます。
