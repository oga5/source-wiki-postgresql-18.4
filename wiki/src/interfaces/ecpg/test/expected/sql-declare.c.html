src/interfaces/ecpg/test/expected/sql-declare.c

概要

PostgreSQL の ECPG（Embedded SQL for C）プリプロセッサの回帰テスト用ソースコードです。SQL の DECLARE ステートメント、カーソル操作、およびプリペアドステートメントの動作を検証します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリに配置されたテストスイートの一部であり、ECPG が生成する C コードの期待値（またはテスト実行用コード）として機能します。

責務

  • 複数のデータベース接続（con1, con2）間でのカーソルおよびステートメントのスコープ検証。
  • PREPARE、DECLARE CURSOR、FETCH、DESCRIBE ステートメントの統合的な動作確認。
  • SQL 記述子（Descriptor）を用いた動的 SQL 実行のテスト。

重要なシンボル

  • sqlca: ECPG のエラー情報を保持する構造体。各 SQL 操作後に sqlcode をチェックするために使用。
  • ECPGdo: ECPG が生成する SQL 実行の主要なインターフェース関数。
  • ECPGprepare / ECPGdeallocate: プリペアドステートメントのライフサイクル管理。
  • ECPGallocate_desc / ECPGget_desc: SQL 記述子の割り当てと値の取得。

追加の索引済みシンボル

  • commitTable

  • main

制御フローとデータフロー

1. **初期化**: main 関数で con1 と con2 に接続し、テスト用テーブルを作成・挿入します。 2. **テスト実行**: execute_test 関数内で以下のパターンを順次実行します。

  • **ケース 1-2**: カーソルを用いたデータ取得。接続指定の有無による挙動の違いを確認。
  • **ケース 3**: EXECUTE ステートメントによる直接実行。
  • **ケース 4**: 特定接続（con2）でのカーソル操作。
  • **記述子テスト**: DESCRIBE を用いてメタデータを取得し、FETCH 結果を記述子経由で読み取る。

    • 3. **終了処理**: テーブルの削除、接続の切断を行い、リソースを解放します。

直接依存

  • ecpglib.h: ECPG ライブラリのコア機能。
  • sqlca.h: SQL 通信領域の定義。
  • ecpgerrno.h: ECPG 固有のエラーコード定義。
  • 標準 C ライブラリ: locale.h, string.h, stdlib.h。

関連コード

  • src/interfaces/ecpg/test/expected/sql-sqlda.c: SQLDA（SQL Descriptor Area）を用いたより詳細な動的 SQL テスト。
  • src/interfaces/ecpg/test/expected/preproc-cursor.c: カーソル操作に特化したテスト。
  • src/interfaces/ecpg/test/expected/sql-describe.c: DESCRIBE ステートメントの単体テスト。

読者向けメモ

  • 本ファイルは .pgc ファイルからプリプロセッサによって生成された C コードです。#line 指示子が多用されており、元のソースコードとの対応関係を示しています。
  • ECPGdebug(1, stderr) が有効になっており、実行時に詳細なデバッグログが標準エラー出力に出力されます。
  • sqlca.sqlcode のチェックは、ECPG におけるエラーハンドリングの標準的なパターンです。
