src/interfaces/ecpg/test/expected/sql-describe.c

概要

本ファイルは、ECPG（Embedded SQL for C）における DESCRIBE 文の動作を検証するための回帰テスト用ソースコードです。SQL文のメタデータ取得機能が、SQL記述子（Descriptor）および SQLDA（SQL Descriptor Area）の両方で正しく機能することを確認します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内のテストスイートの一部であり、プリプロセッサによって生成された C コードの期待値として機能します。ECPG ライブラリが提供する動的 SQL 記述機能の整合性を保証する役割を担います。

責務

  • DESCRIBE 文を用いたクエリのメタデータ（列名など）の抽出。
  • SQL 記述子（ALLOCATE DESCRIPTOR）と SQLDA（sqlda_t）の双方を用いたメタデータ取得の比較検証。
  • 結果セットを返すクエリと、返さないクエリの両方に対する DESCRIBE の挙動確認。

重要なシンボル

  • sqlda_t: SQLDA 構造体。動的 SQL における列情報を保持します。
  • ECPGdescribe(): DESCRIBE 文を実行し、指定された記述子または SQLDA にメタデータを格納するライブラリ関数。
  • ECPGget_desc() / ECPGget_desc_header(): 記述子からメタデータや列数を取得するための関数。
  • ECPGdebug: デバッグ出力を制御するマクロ。

制御フローとデータフロー

1. **初期化**: データベースへの接続とテスト用テーブルの作成・データ挿入を行います。 2. **準備**: PREPARE 文で SQL 文を準備します。 3. **記述**: ECPGdescribe を呼び出し、desc1, desc2（記述子）および sqlda1, sqlda2, sqlda3（SQLDA）にメタデータをロードします。 4. **検証**: ECPGget_desc_header で取得した列数と、各 SQLDA 内の sqld フィールドを比較し、整合性を確認します。 5. **出力**: 各列のメタデータ（フィールド名）を標準出力に書き出し、期待値との比較を可能にします。 6. **終了**: 記述子の解放、メモリの解放、テーブルの削除、接続の切断を行います。

読者向けメモ

  • 本ファイルは ecpg プリプロセッサによって生成されたコードであるため、直接編集するものではありません。
  • DESCRIBE のテストにおいて、SQL 記述子と SQLDA が混在して使用されている点に注目してください。これは、両方のインターフェースが正しく同期して動作することを検証するためです。
  • sqlca.sqlcode を用いたエラーチェックが各ステップで行われており、ECPG の標準的なエラーハンドリングパターンを学習するのに適しています。
