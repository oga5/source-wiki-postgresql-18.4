概要

本ファイルは、ECPG（Embedded SQL for C）における SQL 記述子（Descriptor）領域の操作を検証するための回帰テスト用ソースコードです。プリプロセッサによって生成された C コードであり、動的 SQL 実行時のパラメータ受け渡しや結果取得の挙動を確認します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPG の機能テストスイートを構成します。特に、SQL 記述子を使用した動的 SQL の実行、カーソル操作、および記述子ヘッダの管理機能が正しく動作することを保証する役割を担います。

責務

  • SQL 記述子の割り当て（ECPGallocate_desc）と解放（ECPGdeallocate_desc）。
  • ECPGset_desc を用いた記述子へのパラメータ設定。
  • ECPGget_desc を用いた実行結果の取得。
  • プリペアドステートメントと記述子を組み合わせたカーソル操作の検証。

重要なシンボル

  • ECPGdebug: デバッグ出力を有効化するマクロ。
  • main: テストのメインエントリポイント。
  • sqlca: SQL 通信領域。各 SQL 文実行後のエラーチェックに使用。
  • ECPGallocate_desc / ECPGdeallocate_desc: 記述子領域のライフサイクル管理関数。
  • ECPGset_desc / ECPGget_desc: 記述子内のデータフィールドへのアクセス関数。

制御フローとデータフロー

1. **初期化**: ECPGallocate_desc で記述子を確保し、ECPGconnect でデータベースに接続します。 2. **準備**: ECPGprepare を使用して SQL 文をプリペアドステートメントとして登録します。 3. **実行**: ECPGdo を呼び出し、ECPGt_descriptor を介して記述子に格納されたパラメータを SQL 文に渡します。 4. **結果取得**: ECPGget_desc を使用して、クエリ結果を C 言語の変数にマッピングします。 5. **終了**: ECPGdeallocate_all および ECPGdisconnect を呼び出し、リソースをクリーンアップします。
