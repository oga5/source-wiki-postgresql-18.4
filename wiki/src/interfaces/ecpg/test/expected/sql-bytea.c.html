src/interfaces/ecpg/test/expected/sql-bytea.c

概要

PostgreSQL の ECPG（Embedded SQL for C）における bytea 型の取り扱いを検証するための回帰テスト用ソースコードです。バイナリデータの挿入、抽出、カーソル操作、および動的 SQL におけるデータ型の整合性を確認します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPG プリプロセッサが生成する C コードの期待値（ゴールデンファイル）を提供します。このファイルは、ECPG が SQL の bytea 型を C 言語の構造体へどのようにマッピングし、通信するかを定義するリファレンスです。

責務

  • bytea 型の静的および動的な SQL 文での送受信テスト。
  • SQL 記述子（Descriptor）を用いたバイナリデータの受け渡し検証。
  • 可変長配列やインジケータ変数を用いたデータ境界のテスト。
  • 実行結果のバイナリダンプによる整合性確認。

重要なシンボル

  • struct bytea_1 ～ bytea_4: bytea 型を C 言語で表現するための構造体。len（データ長）と arr（データ本体）で構成されます。
  • dump_binary(): 取得したバイナリデータを 16 進数形式で標準出力に書き出すデバッグ用関数。
  • ECPGdo(): ECPG の中心的な関数。SQL 文の実行とホスト変数へのバインドを行います。
  • init(): テスト実行ごとにバッファとインジケータ変数をリセットするマクロ。

制御フローとデータフロー

1. **初期化**: init() マクロにより、送受信用の構造体配列とインジケータ変数をクリアします。 2. **データ準備**: send_buf に 0xff から始まるバイナリパターンを生成します。 3. **SQL 実行**:

  • INSERT 文で bytea データをテーブルへ投入。
  • SELECT 文でデータを取得し、recv_buf 等へ格納。
  • カーソル操作や PREPARE された動的 SQL を通じて同様の操作を繰り返します。

    • 4. **検証**: dump_binary() を呼び出し、期待されるバイナリ長と内容が一致しているかを確認します。 5. **終了**: テーブルの削除と接続の切断を行います。

直接依存

  • ecpglib.h: ECPG ランタイムライブラリの基本機能。
  • sqlca.h: SQL 通信領域（SQLCA）の定義。
  • ecpgerrno.h: エラーコード定義。
  • 標準 C ライブラリ: stdlib.h, string.h, stdio.h, time.h。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成されたコードであるため、手動で編集するものではありません。
  • bytea 型を扱う際は、構造体の len メンバが正しく設定されていることが重要です。
  • LACK_SIZE を用いたテストケースは、バッファサイズが不足した場合の挙動や境界条件を確認するためのものです。
  • ECPGdebug(1, stderr) が有効になっているため、実行時には詳細な通信ログが標準エラー出力に表示されます。
