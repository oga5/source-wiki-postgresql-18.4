概要

本ファイルは、ECPG（Embedded SQL for C）プリプロセッサの回帰テスト用ソースコードです。CREATE TABLE AS 文の動作検証を目的としており、プリプロセッサによって変換された後の C コードの期待値（expected output）として機能します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部であり、ECPG が SQL 文を正しく C 言語の関数呼び出し（ECPGdo 等）に変換できているかを検証するテストスイートの構成要素です。

責務

  • CREATE TABLE AS 文の標準的な実行および WITH NO DATA オプションの動作確認。
  • SQL 実行結果のホスト変数への格納と、その値の検証。
  • データベース接続からテーブル作成、データ挿入、クエリ実行、後始末（DROP）までの一連のライフサイクルの実行。

重要なシンボル

  • main: テストの実行エントリポイント。
  • ECPGdo: ECPG が生成する、SQL 文をデータベースへ送信するための汎用関数。
  • sqlca: SQL 通信領域（SQL Communication Area）。エラーや警告の判定に使用されます。
  • ECPGdebug: デバッグ出力を制御するマクロ。

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、ECPGsetcommit でコミットモードを設定します。 2. **テーブル操作**: CREATE TABLE および CREATE TABLE AS を実行し、データを挿入します。 3. **データ取得**: SELECT 文を実行し、結果を C 言語のホスト変数 id にバインドして取得します。 4. **検証**: 取得した id を printf で出力し、期待通りの値であることを確認します。 5. **終了処理**: 作成したテーブルを DROP し、ECPGdisconnect で接続を終了します。各ステップで sqlca.sqlcode をチェックし、エラー発生時は sqlprint を呼び出します。
