概要

本ファイルは、ECPG（Embedded SQL for C）におけるデータベース接続の切断機能（DISCONNECT）の動作を検証するための回帰テスト用ソースコードです。ECPGプリプロセッサによって生成されたCコードの期待値（expected output）として機能します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部であり、ECPGライブラリの接続管理機能が仕様通りに動作することを確認するテストスイートの一環です。

責務

  • 複数のデータベース接続を確立し、CURRENT、DEFAULT、ALL といった異なる引数を用いた DISCONNECT 文の挙動を網羅的にテストする。
  • 接続状態の変化が、その後の SQL 実行（SELECT current_database()）に正しく反映されるかを検証する。

重要なシンボル

  • main: テストの実行起点。一連の接続・切断操作を順次実行する。
  • ECPGconnect: データベースへの接続を確立するライブラリ関数。
  • ECPGdisconnect: データベース接続を切断するライブラリ関数。
  • ECPGdebug: デバッグ出力を有効化するマクロ。

制御フローとデータフロー

1. **初期化**: ECPGdebug を呼び出し、デバッグログを標準エラー出力へ向ける。 2. **接続**: ECPGconnect を用いて複数のデータベース（ecpg1_regression, ecpg2_regression）に接続する。 3. **検証**: ECPGdo を介して SELECT current_database() を実行し、現在の接続先を確認する。 4. **切断テスト**:

  • CURRENT: 直近の接続を閉じる。
  • DEFAULT: デフォルト接続を閉じる。
  • ALL: すべての接続を閉じる。

    • 5. **終了**: 接続が既に存在しない状態での DISCONNECT 呼び出しを行い、エラーハンドリングの挙動を確認して終了する。
