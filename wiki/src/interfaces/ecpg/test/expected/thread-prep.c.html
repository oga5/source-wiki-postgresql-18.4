src/interfaces/ecpg/test/expected/thread-prep.c

概要

ECPG（Embedded SQL for C）におけるマルチスレッド環境下でのプリペアードステートメントの動作を検証するための回帰テスト用ソースコードです。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPGプリプロセッサが生成するCコードの期待値を提供します。本ファイルは、スレッドセーフなSQL実行とプリペアードステートメントのライフサイクル管理をテストする役割を担います。

責務

  • 複数スレッドから同時に同一のテーブルに対して INSERT を実行する環境の構築。
  • ECPGprepare および ECPGdo を用いたプリペアードステートメントの並行実行の検証。
  • スレッドごとの独立したデータベース接続（ECPGconnect / ECPGdisconnect）の管理。

重要なシンボル

  • fn: 各スレッドで実行される関数。個別の接続を確立し、プリペアードステートメントをループ実行します。
  • main: テストのメインエントリポイント。テーブルの初期化を行い、プラットフォーム（Windows/POSIX）に応じてスレッドを生成・同期します。
  • THREADS / REPEATS: テストの並列度と各スレッドでの実行回数を定義する定数。
  • sqlca: SQL実行結果のステータスを保持する構造体。エラーハンドリングに使用されます。

追加の索引済みシンボル

  • sqlca_t

制御フローとデータフロー

1. **初期化**: main 関数が T テーブルを作成し、接続を閉じます。 2. **スレッド生成**: THREADS 分のスレッドを起動し、各スレッドに一意の識別子を渡します。 3. **スレッド内処理 (fn)**:

  • データベースへの接続を確立。
  • ECPGprepare で INSERT 文をプリペア。
  • REPEATS 回数分、ECPGdo でプリペアードステートメントを実行。
  • 終了時に ECPGdeallocate でステートメントを解放し、接続を切断。

    • 4. **同期**: main 関数が全スレッドの終了を待機（WaitForMultipleObjects または pthread_join）して終了します。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成された「期待値」であるため、手動で編集するものではありません。
  • ECPGdebug マクロが定義されており、デバッグ時には出力が調整されるようになっています。
  • WindowsとPOSIXでスレッドAPIが異なるため、#ifdef WIN32 による条件分岐が随所に含まれています。コードを読む際は、ターゲットプラットフォームに合わせて読み飛ばす箇所を判断してください。
