src/interfaces/ecpg/test/expected/preproc-init.c

概要

本ファイルは、ECPG（Embedded SQL for C）プリプロセッサの回帰テスト用ソースコードです。主に、EXEC SQL ブロック内での変数宣言、初期化、および WHENEVER 文によるエラーハンドリングの動作検証を目的としています。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPGプリプロセッサが .pgc ファイルを C コードへ変換した後の「期待される出力」を定義しています。プリプロセッサの構文解析とコード生成の正当性を確認するためのリファレンスです。

責務

  • C 言語の複雑な初期化式（算術演算、構造体メンバ参照、三項演算子など）が ECPG の宣言セクションで正しく処理されることの検証。
  • EXEC SQL WHENEVER 文が、エラー発生時に指定された関数を呼び出すコードへ正しく展開されることの検証。
  • SQLCA（SQL Communication Area）構造体との連携確認。

重要なシンボル

  • sqlca_t: SQL 実行結果やエラー情報を保持する構造体。
  • ECPGdo(): ECPG が生成する SQL 実行用の中核関数。
  • sqlca: ECPGget_sqlca() を通じてアクセスされるグローバルな SQLCA インスタンス。
  • fa, fb, fc, fd, fe: WHENEVER 文のテストで呼び出されるスタブ関数群。

制御フローとデータフロー

1. **宣言セクション**: EXEC SQL BEGIN DECLARE SECTION 内で定義された変数が、C のローカル変数として展開されます。 2. **初期化**: プリプロセッサは、SQL 宣言セクション内の初期化式をそのまま C の初期化コードとして出力します。 3. **SQL 実行**: ECPGdo() が SQL 文（select now()）をデータベースへ送信します。 4. **エラーハンドリング**: WHENEVER 文で指定されたアクションは、ECPGdo() の直後に sqlca.sqlcode をチェックする if 文として展開されます。エラー（sqlcode < 0）が発生した場合、登録された関数が実行されます。

直接依存

  • ecpglib.h: ECPG ライブラリの基本関数定義。
  • ecpgerrno.h: ECPG エラーコード定義。
  • sqlca.h: SQLCA 構造体の定義と sqlca マクロ。

関連コード

  • src/interfaces/ecpg/ecpglib/: ECPG ランタイムライブラリの実装。

  • src/interfaces/ecpg/test/expected/preproc-variable.c: 変数宣言の他のケースを網羅するテスト。

読者向けメモ

  • 本ファイルは自動生成されたコードの「期待値」であるため、手動で編集するものではありません。
  • WHENEVER 文の展開結果を確認する際は、ECPGdo() の呼び出し直後の if (sqlca.sqlcode < 0) ブロックに注目してください。
  • C++ 互換性のための extern "C" や名前空間の処理が含まれており、ECPG がマルチ言語環境をどうサポートしているかの参考になります。
