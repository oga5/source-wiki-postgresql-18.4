src/interfaces/ecpg/test/expected/preproc-variable.c

概要

本ファイルは、ECPG（Embedded SQL for C）プリプロセッサのテストスイートの一部であり、SQL文内でのC言語変数および構造体の取り扱いを検証する回帰テスト用ソースコードです。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内の他のテストファイルと同様に、プリプロセッサが .pgc ファイルを C ソースコードに変換した後の「期待される出力」を定義しています。ECPGの型定義機能や構造体マッピングの正当性を確認する役割を担います。

責務

  • ECPGの EXEC SQL TYPE 指令による型定義の変換確認。
  • 構造体および共用体（ind）を用いたホスト変数とインジケータ変数のバインドテスト。
  • カーソル操作（DECLARE, OPEN, FETCH, CLOSE）を通じたデータ取得の検証。
  • データベース接続からテーブル作成、データ操作、切断までの一連のライフサイクルの実行。

重要なシンボル

  • ECPGdebug: SQL実行時のデバッグ出力を制御するマクロ。
  • sqlca: SQL通信領域。実行結果のステータス（sqlcode）を保持。
  • personal_struct: データベースの行データを受け取るための構造体。
  • personal_indicator: 各フィールドのNULL値や状態を判定するためのインジケータ構造体。
  • ind: EXEC SQL TYPE で定義された共用体型。

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、テスト用テーブル family を作成・挿入します。 2. **カーソル操作**: DECLARE でカーソルを定義し、FETCH ループでデータを取得します。 3. **データバインド**: ECPGdo を呼び出す際、ホスト変数（p->name, p->birth.born 等）とインジケータ変数（i->ind_name 等）をペアで渡します。 4. **判定**: sqlca.sqlcode をチェックし、ECPG_NOT_FOUND でループを終了します。 5. **終了処理**: テーブルの削除、トランザクションのコミット、および ECPGdisconnect による接続終了を行います。

直接依存

  • ecpglib.h: ECPGライブラリのコア関数群。
  • ecpgerrno.h: エラーコード定義。
  • sqlca.h: SQLCA構造体の定義。
  • stdlib.h, string.h: 標準Cライブラリ。

  • src/interfaces/ecpg/test/expected/sql-indicators.c: インジケータ変数の詳細なテスト。

読者向けメモ

  • 本ファイルはプリプロセッサによって生成されたコードであるため、直接編集するものではありません。
  • EXEC SQL で始まるコメント行は、プリプロセッサが処理した元の SQL 文を示しています。
  • ECPGdo の引数リストは非常に長くなる傾向があるため、構造体メンバへのアクセスとインジケータ変数の対応関係を追う際は、ECPGt_ で始まる型指定子に注目してください。
