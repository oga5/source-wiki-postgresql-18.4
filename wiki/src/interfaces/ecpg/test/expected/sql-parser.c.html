src/interfaces/ecpg/test/expected/sql-parser.c

概要

本ファイルは、ECPG（Embedded SQL for C）プリプロセッサのパーサーが、SQL文内のトークン結合や複雑な構文を正しく処理できるかを検証するための回帰テスト用ソースコードです。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリに配置されたテストスイートの一部であり、プリプロセッサによって生成されたCコードの期待値（expected output）として機能します。

責務

  • SQL文中の特殊な構文（nullif や with ordinality など）の解析能力を検証する。
  • ALTER TABLE 文におけるデータ型変更などの構文が、ECPG経由で正しくデータベースへ伝達されることを確認する。
  • 実行結果の整合性を sqlca（SQL Communication Area）を通じて監視する。

重要なシンボル

  • main: テストの実行エントリポイント。データベース接続からテーブル操作、切断までの一連のフローを制御します。
  • ECPGdo: ECPGライブラリの主要関数。埋め込みSQL文をデータベースへ送信し実行します。
  • sqlca: SQL実行後のステータスや警告を保持する構造体。エラーハンドリングの基盤となります。
  • ECPGdebug: デバッグ出力を有効化するマクロ。テスト実行時のトレースに使用されます。

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、ECPGsetcommit でコミットモードを設定します。 2. **テーブル構築**: CREATE TABLE を実行し、generate_series を用いた INSERT 文でテストデータを投入します。 3. **データ取得**: SELECT 文を実行し、結果をホスト変数（item 配列および ind インジケータ配列）に格納します。 4. **検証**: 取得したデータを標準出力に表示し、期待値との比較を行います。 5. **スキーマ変更**: ALTER TABLE を用いてカラムの型変更をテストし、最後に DROP TABLE で環境をクリーンアップして終了します。

直接依存

  • ecpglib.h: ECPGランタイムライブラリの関数定義。
  • ecpgerrno.h: ECPG固有のエラーコード定義。
  • sqlca.h: SQL通信領域の定義。
  • stdio.h, stdlib.h, string.h: 標準Cライブラリ。

関連コード

  • parser.pgc: 本ファイルの元となるプリプロセッサ入力ファイル。
  • src/interfaces/ecpg/lib/: ECPGランタイムライブラリの実装。ECPGdo などの挙動を理解する上で重要です。

読者向けメモ

  • 本ファイルはプリプロセッサによって自動生成されたコードであるため、直接編集するものではありません。
  • ECPGdo の引数にある ECPGt_int や ECPGt_EOIT は、ホスト変数とSQL文の型変換を制御する重要なフラグです。
  • sqlca.sqlcode < 0 のチェックは、ECPGテストにおける標準的なエラーハンドリングパターンです。
