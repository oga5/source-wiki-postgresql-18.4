src/interfaces/ecpg/test/expected/sql-array.c

概要

本ファイルは、ECPG（Embedded SQL for C）における配列型および複合的なデータ型の取り扱いを検証するための回帰テスト用ソースコードです。プリプロセッサによって生成されたCコードであり、PostgreSQLの各種データ型（timestamp, numeric, date, interval 等）とC言語の配列とのマッピングをテストします。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ群の一部として、ECPGプリプロセッサがSQL文を正しくCコードへ変換できているかを確認する「期待値」としての役割を持ちます。

責務

  • ECPGにおける配列変数のバインド変数の受け渡し検証。
  • PGTYPES ライブラリを用いたデータ型の変換（文字列から各型への変換およびその逆）のテスト。
  • SQLの INSERT および SELECT 文における配列データの整合性確認。

重要なシンボル

  • main: テストの実行起点。接続からテーブル作成、データ操作、切断までの一連のフローを制御します。
  • sqlca: SQL通信領域（SQL Communication Area）。各SQL実行後のエラーチェックに使用されます。
  • PGTYPES...: pgtypeslib で定義された関数群。日付、時刻、数値、間隔型のフォーマット変換を行います。

追加の索引済みシンボル

  • sqlca_t

制御フローとデータフロー

1. **初期化**: setlocale および ECPGdebug の設定。 2. **データ準備**: PGTYPES 関数を用いて、C言語側の配列（ts, d, in, n 等）にテストデータを投入します。 3. **データベース操作**:

  • ECPGconnect でデータベースへ接続。
  • CREATE TABLE で配列型を含むテーブルを作成。
  • INSERT 文でC言語の配列をSQLのバインド変数として渡す。
  • SELECT 文でデータを取得し、期待値と一致するかを printf で出力して検証。

    • 4. **終了処理**: DROP TABLE を実行し、ECPGdisconnect で接続を終了します。

直接依存

  • ecpglib.h, sqlca.h: ECPGのコアライブラリおよび通信領域定義。
  • pgtypes_*.h: PostgreSQLの特殊データ型（日付、数値、間隔等）の操作用ヘッダー。
  • locale.h, string.h, stdlib.h: 標準Cライブラリ。
  • ecpgerrno.h
  • pgtypes_date.h
  • pgtypes_interval.h
  • pgtypes_numeric.h
  • pgtypes_timestamp.h

関連コード

  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test.c: numeric 型の単体テスト。
  • src/interfaces/ecpg/test/expected/preproc-array_of_struct.c: 構造体の配列に関するテスト。

読者向けメモ

  • 本ファイルは ecpg プリプロセッサによって自動生成されたコードの「期待値」です。直接編集するものではなく、テスト実行時に生成されるコードとの差分比較に使用されます。
  • ECPGdo マクロ内の引数は、プリプロセッサが生成した複雑なバインド変数リストです。配列のサイズや型指定がSQL側の定義と一致しているかを確認する際の重要な参照先となります。
