src/interfaces/ecpg/test/expected/compat_oracle-char_array.c

概要

本ファイルは、ECPG（Embedded SQL for C）における文字配列の取り扱いと、SQLDA（SQL Descriptor Area）を用いた動的SQL実行の挙動を検証するための回帰テスト用ソースコードです。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内のテストスイートの一部であり、ECPGプリプロセッサが生成するCコードの期待値として機能します。特に、固定長文字配列へのデータ取得時のパディング挙動と、互換モードにおけるSQLDAのデータ型変換を検証します。

責務

  • **文字配列の検証**: varchar 型フィールドを固定長 char 配列へフェッチする際の、空白埋めおよびヌル終端処理の確認。
  • **SQLDAの検証**: 動的SQL実行時に sqlda_t を介して取得したデータの型判定と、numeric 型から文字列への変換処理の確認。
  • **エラーハンドリング**: whenever 文による警告やエラー発生時の制御フローの検証。

重要なシンボル

  • sqlda_t: 動的SQLの結果を保持する構造体。
  • sqlvar_t: sqlda_t 内の各カラム情報を保持する構造体。
  • ECPGdo: ECPGの主要なSQL実行関数。
  • PGTYPESnumeric_to_asc: numeric 型を文字列へ変換するライブラリ関数。

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、テスト用テーブル strdbase を作成・挿入します。 2. **静的フェッチ**: カーソル C を使用し、varchar データを固定長 char 配列へフェッチします。この際、バッファサイズとインジケータ変数を介してデータの切り詰めや空白埋めが適切に行われるかを確認します。 3. **動的フェッチ**: ECPGprepare でクエリを準備し、sqlda_t を使用して結果を取得します。 4. **データ変換**: sqlda->sqlvar をループで走査し、sqltype が ECPGt_numeric の場合は PGTYPESnumeric_to_asc を呼び出して表示用文字列に変換します。 5. **終了処理**: カーソルを閉じ、テーブルを削除して接続を切断します。
