src/interfaces/ecpg/test/expected/sql-indicators.c

概要

ECPG（Embedded SQL for C）における「インジケータ変数」の挙動を検証するための回帰テスト用ソースコードです。SQLのNULL値や値の切り捨てをC言語側でどのように扱うかを実証します。

モジュール内での役割

src/interfaces/ecpg/test/expected ディレクトリ内のテストスイートの一部であり、プリプロセッサが生成するCコードの期待値として機能します。

責務

  • SQLの INSERT, SELECT, UPDATE 文におけるインジケータ変数の利用方法の提示。
  • NULL値の挿入および取得時におけるインジケータ変数の値（-1など）の検証。
  • ECPGライブラリの基本的な接続・トランザクション制御の実行。

重要なシンボル

  • sqlca_t: SQL通信領域。エラーコードや警告、処理された行数などの診断情報が格納されます。
  • ECPGdo(): ECPGの主要な実行関数。SQL文とホスト変数、インジケータ変数をバインドします。
  • intvar, nullind: テスト内で使用されるホスト変数およびインジケータ変数。

追加の索引済みシンボル

  • main

制御フローとデータフロー

1. **初期化**: ECPGconnect でデータベースに接続し、テスト用テーブル indicator_test を作成します。 2. **データ挿入**: nullind を -1 に設定し、NULL値を挿入するテストを行います。 3. **データ取得**: SELECT 文を実行し、結果を intvar に、NULL状態を nullind に格納します。 4. **検証**: printf を使用して、期待通りの値とインジケータ状態になっているかを標準出力へ書き出します。 5. **終了**: テーブルの削除と接続解除を行い、正常終了します。

直接依存

  • ecpglib.h: ECPGランタイムライブラリの関数定義。
  • ecpgerrno.h: ECPGのエラーコード定義。
  • sqlca.h: SQLCA構造体の定義。
  • stdio.h: 標準入出力関数。

関連コード

  • src/interfaces/ecpg/test/expected/sql-sqlda.c: 動的SQLにおける記述子領域の扱いに焦点を当てた関連テスト。
  • src/interfaces/ecpg/ecpglib/: ECPGランタイムライブラリの実装（ECPGdo 等の本体）。

読者向けメモ

  • 本ファイルは ecpg プリプロセッサによって .pgc ファイルから生成された「期待値」です。直接編集するものではなく、テスト実行時の比較対象となります。
  • インジケータ変数が -1 の場合、対応するホスト変数はNULLとして扱われます。
  • ECPGdebug(1, stderr) が有効になっており、実行時に詳細なデバッグログが標準エラー出力に出力されるようになっています。
