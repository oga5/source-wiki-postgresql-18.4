src/interfaces/ecpg/ecpglib/misc.c

概要

本ファイルは、ECPG（Embedded SQL for C）ライブラリの基盤となるユーティリティ関数群を提供します。SQLCA（SQL Communication Area）の管理、デバッグログ出力、スレッドセーフな初期化処理、およびNULL値判定といった、ライブラリ全体で共通して利用される低レイヤーの機能を集約しています。

モジュール内での役割

ecpglib の中核的なサポートモジュールです。他のモジュール（execute.c や connect.c など）がSQL実行や接続管理を行う際に必要とする、スレッドローカルな状態管理やエラーハンドリングの補助を提供します。

責務

  • **SQLCA管理**: スレッドごとに独立した sqlca_t 構造体の生成・初期化・破棄。
  • **デバッグ機能**: ECPGdebug によるログ出力設定と、スレッドセーフなログ出力（ecpg_log）。
  • **NULL値判定**: ECPG型システムにおける「NULL」を表現する特殊値のセットおよび判定。
  • **プラットフォーム互換性**: Windows環境における pthread 互換レイヤーの提供。
  • **国際化対応**: gettext を用いたメッセージのローカライズ処理。

重要なシンボル

  • ECPGget_sqlca(): 現在のスレッドに対応する sqlca_t 構造体を取得・生成する。
  • ECPGdebug(): デバッグモードの有効化と出力先の設定。
  • ecpg_log(): デバッグ情報の出力。
  • ECPGis_noind_null() / ECPGset_noind_null(): インジケータ変数を使用しない場合のNULL値判定と設定。
  • ECPGtrans(): トランザクション制御（BEGIN/COMMIT等）の実行。

制御フローとデータフロー

1. **初期化**: ライブラリ利用時に ECPGget_sqlca が呼ばれ、pthread_once を介してスレッドごとの sqlca 領域が確保されます。 2. **デバッグ**: ECPGdebug で設定された simple_debug フラグに基づき、ecpg_log がログを debugstream に書き出します。この際、PIDの付与やスレッドセーフな排他制御が行われます。 3. **NULL判定**: ECPGis_noind_null は、型に応じて定義された「NULLを表す特殊値（INT_MIN や 0xff の埋め込みなど）」をチェックし、結果を返します。 4. **トランザクション**: ECPGtrans は接続状態を確認後、必要に応じて PQexec を呼び出し、SQLコマンドをデータベースへ送信します。

読者向けメモ

  • **スレッドセーフティ**: 本ファイルはマルチスレッド環境を強く意識しており、pthread_key_t や pthread_mutex_t を多用しています。Windows環境では pthread をエミュレートする独自実装が含まれている点に注意してください。
  • **NULLの表現**: ECPGでは、インジケータ変数がない場合に特定の最小値やビットパターンをNULLとして扱います。このロジックは ECPGis_noind_null に集約されているため、新しい型を追加する際はここを更新する必要があります。
  • **デバッグログ**: ecpg_log 内で malloc を行っているため、極めて頻繁なログ出力はパフォーマンスに影響を与える可能性があります。
