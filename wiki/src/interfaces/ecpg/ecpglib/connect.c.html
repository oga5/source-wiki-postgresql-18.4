src/interfaces/ecpg/ecpglib/connect.c

概要

ECPG（Embedded SQL for C）におけるデータベース接続のライフサイクル管理と、スレッドセーフな接続ハンドリングを担う中核ファイルです。

モジュール内での役割

ecpglib の接続管理レイヤーとして、アプリケーションと libpq の間を仲介します。接続の確立、切断、およびスレッドごとの接続状態（TSD: Thread Specific Data）の追跡を行います。

責務

  • **接続の管理**: ECPGconnect および ECPGdisconnect による接続の生成・破棄。
  • **スレッドセーフなアクセス**: pthread を使用した接続リストの排他制御と、スレッドごとのカレント接続の保持。
  • **通知処理**: ECPGnoticeReceiver によるデータベースからの通知（NOTICE）の捕捉と sqlca へのマッピング。
  • **トランザクション状態の制御**: ECPGsetcommit によるオートコミット設定の切り替え。

重要なシンボル

  • connections_mutex: 全接続リストを保護するグローバルなミューテックス。
  • actual_connection_key: スレッドごとのカレント接続を保持するための pthread_key_t。
  • all_connections: 存在する全接続を管理する連結リストのヘッド。
  • ECPGconnect: データベース接続を初期化し、リストに追加するメイン関数。
  • ecpg_get_connection: 指定された名前またはカレント接続の connection 構造体を取得する。

制御フローとデータフロー

1. **接続取得**: ECPGconnect が呼び出されると、接続パラメータを解析し、connections_mutex でロックを確保した上で connection 構造体を生成・リストに追加します。 2. **スレッドローカル管理**: ECPGsetconn や ECPGconnect は pthread_setspecific を使用し、そのスレッドが使用する接続を actual_connection_key に登録します。 3. **接続の終了**: ECPGdisconnect が ecpg_finish を呼び出し、libpq の PQfinish を実行してメモリを解放し、リストから該当ノードを削除します。 4. **通知**: データベースからの通知は ECPGnoticeReceiver で受け取り、sqlca 構造体の sqlstate や sqlcode を更新してアプリケーションへ伝播させます。

読者向けメモ

  • **スレッド安全性**: ecpg_get_connection は、名前指定がない場合はスレッドローカルな接続を優先し、見つからない場合にグローバルな接続をフォールバックとして返します。この挙動はマルチスレッド環境で特に重要です。
  • **エラーハンドリング**: ECPGconnect 内でのメモリ割り当て失敗や接続失敗時は、リスト操作の前にクリーンアップを行うよう設計されています。
  • **互換性**: INFORMIX_MODE などの互換性フラグに応じて、接続文字列の解釈や環境変数の参照方法が変化します。
