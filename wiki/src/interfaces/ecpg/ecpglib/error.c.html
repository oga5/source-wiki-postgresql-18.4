src/interfaces/ecpg/ecpglib/error.c

概要

ECPG（Embedded SQL for C）ライブラリにおけるエラーハンドリングと診断情報の管理を行うモジュールです。SQL実行中に発生したエラーを捕捉し、sqlca（SQL Communication Area）構造体へ情報を格納することで、アプリケーション側がエラー内容を判別できるようにします。

モジュール内での役割

ecpglib のエラー報告の最終的な出口です。他のソースファイル（execute.c や connect.c など）で発生した異常系を、統一された形式で sqlca に書き出し、ログ出力およびメモリ解放のトリガーを引く役割を担います。

責務

  • **エラー情報の集約**: 内部エラーコードやバックエンド（PostgreSQLサーバー）からのエラー情報を sqlca 構造体にマッピングする。
  • **診断メッセージの生成**: エラー発生行番号や詳細情報を含むメッセージを構築する。
  • **互換性の維持**: Informix などの互換モードを考慮したエラーコードの変換。
  • **リソース管理**: エラー発生時に ECPGfree_auto_mem() を呼び出し、メモリリークを防止する。

重要なシンボル

  • ecpg_raise: ECPGライブラリ内部で発生したエラーを sqlca に設定する汎用関数。
  • ecpg_raise_backend: libpq から返された PGresult を解析し、サーバー側のエラーを sqlca に反映する。
  • ecpg_check_PQresult: SQL実行結果のステータスを判定し、エラーであれば適切なエラー処理へ分岐させるフィルタ関数。
  • sqlprint: sqlca に格納されたエラーメッセージを標準エラー出力に表示するユーティリティ。

制御フローとデータフロー

1. **判定**: ecpg_check_PQresult が libpq の結果を監視し、異常があれば ecpg_raise_backend を呼び出す。 2. **変換**: ecpg_raise_backend は PQresultErrorField を用いて SQLSTATE やエラーメッセージを取得し、sqlca 構造体を更新する。 3. **記録**: ecpg_log を通じてログファイルにエラー詳細を書き出す。 4. **クリーンアップ**: ECPGfree_auto_mem() を呼び出し、エラー発生時に確保されていた一時的なメモリを解放する。

直接依存

  • sqlca.h: エラー情報を保持する sqlca_t 構造体の定義。
  • ecpglib_extern.h: 内部関数プロトタイプとメモリ管理関数の宣言。
  • ecpgerrno.h: ECPG固有のエラーコード定義。
  • postgres_fe.h: libpq（PostgreSQLフロントエンドライブラリ）の基本定義。
  • ecpglib.h
  • ecpgtype.h

関連コード

  • src/interfaces/ecpg/ecpglib/execute.c: SQL実行の主処理。ここで ecpg_check_PQresult が頻繁に呼び出される。
  • src/interfaces/ecpg/ecpglib/memory.c: ECPGfree_auto_mem() の実装元。

読者向けメモ

  • **SQLCAの重要性**: ECPGアプリケーションは sqlca を通じてエラーを検知するため、このファイルはライブラリの信頼性を左右する重要な箇所です。
  • **メモリ管理**: エラー発生時は即座に ECPGfree_auto_mem() が呼ばれるため、エラーハンドリングの途中でメモリを参照し続けるような実装は避ける必要があります。
  • **翻訳**: ecpg_gettext を使用しており、国際化対応がなされています。メッセージを変更する際は注意してください。
