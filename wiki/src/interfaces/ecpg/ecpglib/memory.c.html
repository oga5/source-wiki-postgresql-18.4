src/interfaces/ecpg/ecpglib/memory.c

概要

ECPG（Embedded SQL for C）ライブラリ内におけるメモリ管理機能を提供します。標準的なメモリ割り当て関数のラッパーと、スレッドローカルストレージを利用した自動メモリ解放メカニズムを実装しています。

モジュール内での役割

ecpglib の基盤層として、SQL実行やデータ変換の過程で動的に確保されるメモリのライフサイクルを管理します。特に、エラー発生時やトランザクション終了時にメモリリークを防ぐための追跡機能を提供します。

責務

  • **標準関数のラップ**: malloc, realloc, strdup 等のラッパーを提供し、メモリ不足時に ecpg_raise を通じてECPG固有のエラー処理を統合する。
  • **自動メモリ追跡**: ecpg_auto_alloc を通じて確保されたメモリをリストで保持し、必要に応じて一括解放する。
  • **スレッドセーフな管理**: pthread を利用し、スレッドごとに独立したメモリ追跡リストを保持する。

重要なシンボル

  • ecpg_alloc / ecpg_realloc: メモリ確保の標準ラッパー。失敗時にECPGエラーを発生させる。
  • ecpg_auto_alloc: メモリを確保し、自動解放リストに登録する。
  • ECPGfree_auto_mem: リスト内の全ポインタとリスト構造体自体を解放する。
  • auto_mem_key: スレッドローカルなメモリ追跡リストを特定するためのキー。

制御フローとデータフロー

1. **確保**: ecpg_auto_alloc が呼ばれると、内部で ecpg_alloc でメモリを確保し、ecpg_add_mem で auto_mem 構造体のリストにポインタを追加します。 2. **追跡**: リストは pthread_getspecific を介してスレッドごとに管理されます。 3. **解放**: ECPGfree_auto_mem が呼び出されると、リストを辿り、登録された各ポインタに対して free を実行し、最後にリスト構造体自体を解放します。auto_mem_destructor により、スレッド終了時にも自動的に解放が行われます。

関連コード

  • src/interfaces/ecpg/ecpglib/error.c: メモリ不足エラー発生時の通知処理。

読者向けメモ

  • 本ファイルは、ECPGが生成するCコードがメモリ管理を意識せずに済むよう設計されています。
  • ECPGfree_auto_mem は、特定のスコープ終了時やエラーハンドリングの過程で呼び出されることが想定されています。
  • ecpg_clear_auto_mem は、メモリの追跡リストのみをクリアし、確保した実データは解放しない特殊なケース（所有権の移譲など）で使用されます。使用には注意が必要です。
