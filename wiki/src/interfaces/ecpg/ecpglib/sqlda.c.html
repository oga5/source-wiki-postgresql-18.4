src/interfaces/ecpg/ecpglib/sqlda.c

概要

本ファイルは、ECPG（Embedded SQL for C）における SQLDA（SQL Descriptor Area）構造体の構築およびデータ設定を担うサポートルーチン群を提供します。

モジュール内での役割

ecpglib の一部として、動的 SQL 実行時に取得した PGresult を、Informix 互換モードまたは ECPG ネイティブモードの SQLDA 構造体に変換するブリッジの役割を果たします。

責務

  • SQLDA 構造体に必要なメモリサイズの計算（メタデータおよびデータ領域の算出）。
  • PGresult からのメタデータ（列名、型情報）の抽出と SQLDA へのマッピング。
  • PGresult の各行データを SQLDA 内の適切なオフセット位置へ配置し、型変換を行う。
  • メモリレイアウトの整合性を保つためのアライメント調整。

重要なシンボル

  • ecpg_build_compat_sqlda / ecpg_build_native_sqlda: PGresult から SQLDA 構造体を初期化・構築する関数。
  • ecpg_set_compat_sqlda / ecpg_set_native_sqlda: 構築済みの SQLDA に PGresult の値を格納する関数。
  • ecpg_sqlda_align_add_size: メモリ配置時のアライメントとオフセット計算を行う内部ユーティリティ。

追加の索引済みシンボル

  • sqlda_common_total_size

制御フローとデータフロー

1. **サイズ計算**: sqlda_xxx_total_size が呼び出され、構造体本体、フィールド定義、および各列のデータに必要な合計サイズを算出します。 2. **メモリ確保**: ecpg_alloc を使用して、メタデータとデータ領域を単一の連続したメモリブロックとして確保します。 3. **メタデータ設定**: ecpg_build_xxx_sqlda が列名や型情報を構造体に書き込みます。 4. **データ転送**: ecpg_set_xxx_sqlda が PGresult から値を取り出し、ecpg_get_data を介して SQLDA 内の sqldata ポインタが指す領域へコピーします。この際、ecpg_sqlda_align_add_size で計算されたオフセットに従って配置されます。

読者向けメモ

  • **メモリ管理**: SQLDA はメタデータとデータが同一のメモリブロックに配置されるため、解放時は free(sqlda) だけで完結する設計になっています。
  • **アライメント**: ecpg_sqlda_align_add_size は、C 言語の構造体アライメント規則を模倣しています。特に numeric 型などの複雑な型を扱う際は、ポインタの配置に注意が必要です。
  • **互換性**: Informix 互換モードとネイティブモードで構造体のレイアウトが異なるため、関数が二系統用意されています。修正時は両方の整合性を確認してください。
