src/interfaces/ecpg/ecpglib/descriptor.c

概要

ECPG（Embedded SQL for C）における SQL 記述子（Descriptor）の管理および操作を行うモジュールです。SQL 標準の ALLOCATE DESCRIPTOR、GET DESCRIPTOR、SET DESCRIPTOR、DEALLOCATE DESCRIPTOR などの機能を実装し、動的 SQL 実行時のメタデータや結果セットの受け渡しをサポートします。

モジュール内での役割

ecpglib の一部として、アプリケーションと PostgreSQL サーバー間のデータ型変換や結果セットの保持を仲介します。特に、実行時に列数や型が確定する動的 SQL において、記述子を介した柔軟なデータバインディングを提供します。

責務

  • **記述子のライフサイクル管理**: スレッドローカルストレージ（TLS）を利用した記述子の生成、保持、破棄。
  • **属性の取得・設定**: ECPGget_desc および ECPGset_desc を通じた、記述子内のメタデータ（列名、型、精度など）やデータ値のアクセス。
  • **DESCRIBE 文の処理**: ECPGdescribe によるプリペアドステートメントのメタデータ取得と、記述子または SQLDA への格納。

重要なシンボル

  • ECPGget_desc: 記述子から指定した属性やデータをアプリケーション変数へコピーする。
  • ECPGset_desc: アプリケーションから記述子へ値を設定する。
  • ECPGallocate_desc / ECPGdeallocate_desc: 記述子のメモリ確保と解放。
  • ecpg_find_desc: 名前をキーにスレッドローカルな記述子リストから対象を検索する。
  • descriptor_key: スレッドごとの記述子リストを保持するための pthread_key_t。

制御フローとデータフロー

1. **初期化**: 初回アクセス時に pthread_once を経由して descriptor_key が初期化されます。 2. **検索**: ecpg_find_desc が呼ばれると、pthread_getspecific で現在のスレッドに紐付いた記述子リストを取得し、名前でマッチングを行います。 3. **データ操作**:

  • ECPGget_desc は可変長引数（va_list）を解析し、PGresult から該当する列の情報を抽出し、get_int_item や get_char_item を介して指定された C 変数に変換・格納します。
  • ECPGset_desc は逆に、C 変数から記述子内の descriptor_item 構造体へ値をコピーします。

    • 4. **終了**: スレッド終了時に descriptor_destructor が呼び出され、メモリリークを防ぐために全記述子が解放されます。

読者向けメモ

  • **スレッド安全性**: 本モジュールはスレッドローカルストレージを使用して記述子を管理しているため、マルチスレッド環境下でも記述子の名前空間はスレッドごとに分離されます。
  • **可変長引数**: ECPGget_desc や ECPGset_desc は va_arg を多用しています。引数の順序や型が ECPG プリプロセッサの生成コードと厳密に一致している必要があるため、デバッグ時は注意してください。
  • **ロケール**: 数値変換時に LC_NUMERIC を一時的に "C" ロケールへ変更する処理が含まれており、データベースとの標準的な数値交換を保証しています。
