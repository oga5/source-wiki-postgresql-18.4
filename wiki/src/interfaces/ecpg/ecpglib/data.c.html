src/interfaces/ecpg/ecpglib/data.c

概要

PostgreSQL のクエリ結果（PGresult）から、ECPG アプリケーションのホスト変数へデータを転送・変換するためのコアロジックを提供します。

モジュール内での役割

ecpglib のデータ変換層として機能します。データベースから取得したテキストまたはバイナリ形式のデータを、C 言語の型（int, double, numeric, date 等）へマッピングし、必要に応じてインジケータ変数の更新やエラーハンドリングを行います。

責務

  • **データ変換**: PQgetvalue で取得した値を C 言語の型へパース・変換する。
  • **型安全性と互換性**: COMPAT_MODE（Informix, Oracle 等）に応じた挙動の差異を吸収する。
  • **配列処理**: SQL 配列形式（{...}）のパースおよび要素ごとの展開。
  • **バイナリ/テキスト変換**: 16 進数エンコードされた bytea データのデコード処理。

重要なシンボル

  • ecpg_get_data: 本ファイルのメイン関数。結果セットから指定されたホスト変数へ値をコピーする。
  • garbage_left: パース後の文字列に不正な残余データがないか検証する。
  • check_special_value: NaN や Infinity などの特殊な浮動小数点値を処理する。
  • hex_decode / ecpg_hex_encode: bytea 型の 16 進数変換を担う。

制御フローとデータフロー

1. **初期化**: PQgetvalue で値を取得し、PQfformat でバイナリかテキストかを確認。 2. **インジケータ処理**: PQgetisnull を確認し、NULL であればインジケータ変数に -1 を設定して終了。 3. **型変換**:

  • **バイナリモード**: memcpy で直接コピー。
  • **テキストモード**: switch 文で型を判定し、strtol, strtod, PGTYPES..._from_asc 等を用いて変換。

    • 4. **配列展開**: 配列の場合は array_delimiter や array_boundary を用いてループし、各要素を順次処理。 5. **エラー/警告**: 変換失敗時は ecpg_raise でエラーを報告し、切り捨てが発生した場合は sqlca に警告を設定。

読者向けメモ

  • **Informix モード**: INFORMIX_MODE が有効な場合、数値変換エラー時にエラーを返さず NULL をセットするなどの特殊な挙動があるため注意してください。
  • **配列の扱い**: isarray フラグが ECPG_ARRAY_ARRAY の場合、{ で始まり } で終わる文字列をパースします。このロジックは複雑になりやすいため、デバッグ時は pval のポインタ移動を追跡するのが有効です。
  • **メモリ管理**: PGTYPESnumeric_new 等でメモリを確保した場合は、必ず free されていることを確認してください。
