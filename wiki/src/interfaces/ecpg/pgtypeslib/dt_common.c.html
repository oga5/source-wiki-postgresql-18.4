src/interfaces/ecpg/pgtypeslib/dt_common.c

概要

本ファイルは、ECPG ライブラリにおける日付・時刻処理の共通基盤です。文字列の解析（デコード）、内部表現から文字列への変換（エンコード）、およびカレンダー計算（ユリウス日変換等）を担います。pgtypeslib の日付・時刻関連機能のバックエンドとして機能します。

モジュール内での役割

datetime.c, timestamp.c, interval.c といった上位の型固有実装に対し、共通のトークンテーブル、解析ロジック、および書式変換ユーティリティを提供します。

責務

  • **日付・時刻の解析**: 文字列をトークン化し、struct tm や fsec_t（小数秒）へ変換。
  • **カレンダー計算**: ユリウス日とグレゴリオ暦の相互変換。
  • **書式化**: ISO, SQL, German, Postgres スタイルの日付・時刻文字列生成。
  • **タイムゾーン処理**: POSIX 形式や数値オフセットの解析と変換。
  • **スキャン処理**: strftime に似たフォーマット文字列に基づく入力解析（defmt_scan）。

重要なシンボル

トークンテーブルとキャッシュ

  • datetktbl, deltatktbl: 日付・時刻・期間解析用のキーワードテーブル。
  • datecache, deltacache: 解析効率化のための直近のトークンキャッシュ。

日付・時刻変換

  • date2j, j2date: ユリウス日と暦（年・月・日）の相互変換。
  • abstime2tm: AbsoluteTime を struct tm に変換し、タイムゾーン情報を抽出。
  • dt2time: ユリウス日（double）を時・分・秒・小数秒に分解。

解析・デコード

  • ParseDateTime: 文字列をフィールド単位にトークン化。
  • DecodeDateTime: 解析済みフィールドを struct tm へマッピング。
  • DecodeUnits, DecodeTime, DecodeTimezone: 特定の単位や形式のデコード。

エンコード

  • EncodeDateOnly, EncodeDateTime: struct tm を指定されたスタイルで文字列化。

制御フローとデータフロー

1. **入力解析**: ParseDateTime が入力文字列を field[] 配列に分割し、ftype[] で型を識別します。 2. **デコード**: DecodeDateTime が各フィールドを走査し、DecodeNumber や DecodeSpecial を呼び出して struct tm を構築します。 3. **変換**: abstime2tm 等がシステム時刻を struct tm に変換し、必要に応じてタイムゾーンオフセットを適用します。 4. **出力**: EncodeDateTime が struct tm と fsec_t を受け取り、指定された style に基づいてバッファへシリアライズします。

直接依存

  • dt.h: 日付・時刻関連の定数・構造体定義。
  • pgtypeslib_extern.h: 公開 API 定義。
  • postgres_fe.h: フロントエンド用共通定義。
  • 標準ライブラリ: time.h (時刻操作), math.h (数値計算), ctype.h (文字分類)。

関連コード

  • datetime.c, timestamp.c, interval.c: 本ファイルのユーティリティを利用して各型を実装。
  • common.c: メモリ管理や文字列操作の補助。

読者向けメモ

  • **メモリ管理**: PGTYPEStimestamp_defmt_scan 内で pgtypes_alloc を使用しており、呼び出し元での解放が必要です。
  • **エッジケース**: 紀元前（BC）の扱いや、ユリウス日変換の境界条件（Nov 24, -4713）に注意してください。
  • **制限事項**: DecodeNumberField 等の数値解析ロジックには、一部の桁数制限や丸め処理の簡略化（バックエンドとの差異）が存在します。
  • **スレッド安全性**: localtime_r や gmtime_r を使用しており、スレッドセーフな設計となっています。
  • **省略箇所**: find_end_token 等の内部スキャン関数は、フォーマット解析の複雑な再帰的ロジックを隠蔽していますが、エラーハンドリングは限定的です。
