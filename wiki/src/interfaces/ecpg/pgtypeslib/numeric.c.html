src/interfaces/ecpg/pgtypeslib/numeric.c

概要

PostgreSQL の NUMERIC 型をクライアントサイド（ECPG）で扱うための数値演算ライブラリです。固定小数点数演算を実装し、文字列変換、算術演算、型変換機能を提供します。

モジュール内での役割

src/interfaces/ecpg/pgtypeslib の一部として、データベースから取得した NUMERIC データを C 言語の構造体として操作可能にします。サーバー側の numeric.c とは独立した、クライアント側専用の軽量実装です。

責務

  • **数値の表現**: numeric 構造体を用いた可変長数値の管理。
  • **算術演算**: 加算、減算、乗算、除算の実行。
  • **型変換**: 文字列（ASCII）、int、long、double、decimal との相互変換。
  • **メモリ管理**: 数値バッファの動的確保と解放。

重要なシンボル

  • numeric: 数値を保持する主要構造体（digits 配列、weight、sign 等で構成）。
  • PGTYPESnumeric_new / free: numeric 型のライフサイクル管理。
  • PGTYPESnumeric_from_asc / to_asc: 文字列と数値間の変換。
  • PGTYPESnumeric_add / sub / mul / div: 基本的な算術演算関数。
  • alloc_var: 内部的な数値バッファの確保と初期化。

制御フローとデータフロー

1. **初期化**: alloc_var で必要な桁数分のメモリを確保し、numeric 構造体を初期化します。 2. **変換**: set_var_from_str が文字列を解析し、weight（小数点位置）と digits（数値配列）を決定します。 3. **演算**: add_abs や sub_abs などの低レベル関数が絶対値計算を行い、PGTYPESnumeric_add 等のラッパーが符号を判定して最終結果を構築します。 4. **出力**: get_str_from_var が内部表現を文字列に変換します。この際、必要に応じて四捨五入が行われます。

読者向けメモ

  • **精度と丸め**: get_str_from_var は出力時に丸め処理を行うため、呼び出し元の numeric 構造体が変更される可能性があります。
  • **メモリ管理**: digitbuf_alloc は pgtypes_alloc をラップしています。メモリリークを防ぐため、PGTYPESnumeric_free を適切に使用してください。
  • **除算の制約**: PGTYPESnumeric_div は内部で select_div_scale を呼び出し、精度を自動決定します。非常に大きな数値や複雑なスケール変換を伴う場合は、計算結果の精度に注意が必要です。
