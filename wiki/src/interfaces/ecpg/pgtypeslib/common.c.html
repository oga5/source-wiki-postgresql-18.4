src/interfaces/ecpg/pgtypeslib/common.c

概要

本ファイルは、ECPG（Embedded SQL for C）のデータ型ライブラリ（pgtypeslib）における汎用的なユーティリティ関数群を提供します。メモリ管理の抽象化および、書式付き文字列生成のための補助機能に特化しています。

モジュール内での役割

pgtypeslib 全体で使用される低レベルなメモリ操作と、データ型変換時の文字列バッファ操作を担います。他のモジュール（numeric.c や datetime.c など）が共通して利用する基盤層として機能します。

責務

  • **メモリ管理のラップ**: calloc や strdup を呼び出し、失敗時に errno を設定する安全なラッパーを提供します。
  • **Windows 互換性の確保**: PGTYPESchar_free を通じて、DLL 境界を跨いだメモリ解放を安全に行うためのインターフェースを提供します。
  • **書式付き文字列の構築**: pgtypes_fmt_replace を通じ、多様なデータ型（数値、文字列、文字）を特定のバッファへ安全に書き込むための変換ロジックを統合します。

重要なシンボル

  • pgtypes_alloc: calloc のラッパー。ゼロ初期化されたメモリを確保します。
  • pgtypes_strdup: strdup のラッパー。メモリ確保失敗時のエラーハンドリングを統一します。
  • pgtypes_fmt_replace: 型に応じた文字列変換とバッファへのコピーを行う中心的な関数です。
  • PGTYPESchar_free: free のラッパー。Windows環境でのメモリ解放の整合性を保つために使用されます。

制御フローとデータフロー

pgtypes_fmt_replace が本ファイルの制御フローの中心です。 1. replace_type に基づき、入力された union un_fmt_comb を適切な文字列形式へ変換します。 2. 数値型の場合は一時バッファを確保し、snprintf で文字列化します。 3. 出力先バッファの残容量（pstr_len）を確認し、オーバーフローを防ぎながら memcpy または strcpy で書き込みます。 4. 成功時はポインタと残容量を更新し、失敗時はエラーコードを返します。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用共通定義。
  • pgtypes.h: ライブラリの基本型定義。
  • pgtypeslib_extern.h: 本ライブラリが公開する外部インターフェース定義。

関連コード

  • src/interfaces/ecpg/pgtypeslib/numeric.c: 数値型の変換処理において、本ファイルの pgtypes_alloc 等が頻繁に利用されます。
  • src/interfaces/ecpg/pgtypeslib/pgtypeslib_extern.h: 本ファイルで定義された関数のプロトタイプ宣言が含まれています。

読者向けメモ

  • 本ファイルは、メモリ確保失敗時に errno を設定する規約に従っています。呼び出し元でエラーチェックを行う際は errno を参照してください。
  • pgtypes_fmt_replace は、バッファオーバーフローを防止するために pstr_len を厳密に管理しています。新しい型を追加する際は、必ずバッファサイズのチェックロジックを維持してください。
  • PGTYPESchar_free は、Windows環境でライブラリとアプリケーションが異なるランタイムを使用している場合に発生するメモリ解放エラーを回避するための重要な関数です。必ずこの関数経由で解放してください。
