src/interfaces/ecpg/pgtypeslib/timestamp.c

概要

PostgreSQLの timestamp 型をECPG（Embedded SQL for C）環境で扱うための変換および演算関数群を提供します。内部的にはマイクロ秒単位の int64 として時刻を管理し、C言語の struct tm との相互変換や、文字列とのフォーマット変換を担います。

モジュール内での役割

pgtypeslib の一部として、日付・時刻関連のデータ型処理を完結させます。dt.h で定義された低レベルな日付計算ロジックを利用し、ユーザーが利用可能な高レベルなAPI（PGTYPEStimestamp_*）を公開します。

責務

  • **型変換**: timestamp と struct tm 間の相互変換（tm2timestamp, timestamp2tm）。
  • **文字列処理**: 文字列から timestamp への解析（from_asc）および、指定フォーマットでの文字列化（to_asc, fmt_asc）。
  • **算術演算**: timestamp への interval の加減算および、timestamp 同士の差分計算。
  • **特殊値対応**: infinity や epoch といった特殊なタイムスタンプ値のエンコード・デコード。

重要なシンボル

  • timestamp: int64（マイクロ秒単位）として定義されたメインのデータ型。
  • tm2timestamp() / timestamp2tm(): 内部的な変換の核となる関数。
  • PGTYPEStimestamp_from_asc() / PGTYPEStimestamp_to_asc(): ECPGアプリケーションが直接利用する文字列変換API。
  • PGTYPEStimestamp_add_interval(): タイムスタンプに対する期間演算の主要関数。

制御フローとデータフロー

1. **変換**: tm2timestamp は date2j（ユリウス日変換）を用いて日付を計算し、time2t で時刻をマイクロ秒に変換して合算します。 2. **フォーマット**: PGTYPEStimestamp_fmt_asc は dttofmtasc_replace を呼び出し、strftime に似た形式で文字列を構築します。この際、% プレースホルダーごとに pgtypes_fmt_replace を介して値をバッファへ書き込みます。 3. **演算**: add_interval は、月単位の加算が必要な場合は struct tm に分解して調整を行い、それ以外の時間単位は int64 の直接加算で処理します。

読者向けメモ

  • **精度**: timestamp はマイクロ秒精度です。fsec_t は小数秒を扱いますが、ディスク保存用ではない点に注意してください。
  • **オーバーフロー**: tm2timestamp 等の関数は、計算結果が有効範囲外の場合に -1 を返します。呼び出し側でのエラーチェックが必須です。
  • **ロケール**: dttofmtasc_replace 内のいくつかのフォーマット指定子（%a, %A 等）は、現時点ではロケール対応が不完全な箇所があるため注意が必要です。
