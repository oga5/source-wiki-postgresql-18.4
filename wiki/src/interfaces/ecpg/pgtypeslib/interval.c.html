src/interfaces/ecpg/pgtypeslib/interval.c

概要

本ファイルは、ECPG（Embedded SQL for C）ライブラリにおける interval 型の解析、変換、および文字列フォーマット処理を担います。PostgreSQL バックエンドの datetime.c から移植されたロジックをベースに、ECPG 向けに調整された実装を提供します。

モジュール内での役割

pgtypeslib モジュールの一部として、日付・時刻関連のデータ型変換機能を提供します。特に、SQL 文字列と内部表現（interval 構造体）の相互変換を抽象化し、クライアントアプリケーションが PostgreSQL の時間間隔を容易に扱えるようにします。

責務

  • **文字列解析**: ISO 8601 形式および PostgreSQL 独自形式の文字列を interval 構造体に変換。
  • **フォーマット変換**: 内部の interval 表現を、指定されたスタイル（SQL 標準、ISO 8601、Postgres 形式など）の文字列に変換。
  • **メモリ管理**: interval 型の動的メモリ確保および解放（PGTYPESinterval_new, PGTYPESinterval_free）。
  • **構造変換**: interval 構造体と struct tm + fsec_t（小数秒）間の変換。

重要なシンボル

  • PGTYPESinterval_from_asc: 文字列から interval を生成する主要なエントリポイント。
  • PGTYPESinterval_to_asc: interval を文字列に変換する主要なエントリポイント。
  • DecodeInterval: 文字列トークンを解析し、時間単位（年、月、日、時、分、秒）に分解するコアロジック。
  • EncodeInterval: 内部表現を各種スタイルに従って文字列へシリアライズする関数。
  • interval: ECPG で使用される時間間隔の内部表現構造体。

制御フローとデータフロー

1. **入力**: PGTYPESinterval_from_asc が文字列を受け取り、ParseDateTime でトークン化します。 2. **解析**: DecodeInterval または DecodeISO8601Interval がトークンを走査し、struct tm と fsec_t に値を蓄積します。 3. **変換**: tm2interval が tm 構造体の値を interval 構造体の month（月単位）と time（マイクロ秒単位）に集約します。 4. **出力**: PGTYPESinterval_to_asc は interval2tm で分解した後、EncodeInterval を呼び出し、指定された IntervalStyle に基づいて文字列バッファを生成します。

読者向けメモ

  • **移植の注意点**: 本ファイル内の多くの関数はバックエンドのコードをコピー＆ペーストしたものです。struct pg_tm が struct tm に置き換えられている点に注意してください。
  • **制約**: -ffast-math コンパイルオプションは、浮動小数点演算の精度に影響を与えるため、本コードでは明示的に禁止されています。
  • **スタイル**: IntervalStyle はハードコードされている箇所が多く、ECPG の仕様に合わせて INTSTYLE_POSTGRES_VERBOSE がデフォルトとして使用されています。
