src/interfaces/ecpg/preproc/descriptor.c

概要

SQL記述子領域（Descriptor Area）の管理および、ECPGプリプロセッサにおける記述子関連のコード生成を担うモジュールです。SQL標準の GET DESCRIPTOR および SET DESCRIPTOR 文を、ECPGランタイムライブラリが実行可能なCコードへ変換します。

モジュール内での役割

src/interfaces/ecpg/preproc サブシステムにおいて、記述子に関連する構文解析結果を保持し、適切なランタイム関数呼び出しを出力する役割を持ちます。

責務

  • **記述子の追跡**: プリプロセス中に宣言・使用される記述子の名前と接続情報を管理します。
  • **代入処理の管理**: GET/SET DESCRIPTOR 文で指定された項目（COUNT, DATA 等）のリストを一時的に保持し、コード生成時に消費します。
  • **コード生成**: 記述子操作を ECPGget_desc や ECPGset_desc といったランタイム関数呼び出しに変換して出力します。
  • **型チェック**: 記述子項目に代入される変数が数値型であるか等の妥当性を検証します。

重要なシンボル

  • add_descriptor / drop_descriptor: 記述子の登録と破棄を行います。
  • lookup_descriptor: 既存の記述子を検索し、必要に応じて接続情報を更新します。
  • output_get_descr / output_set_descr: 記述子の取得・設定を行うCコードを生成します。
  • descriptor_variable: 記述子をECPGの変数として扱うための構造体を返します。
  • sqlda_variable: SQLDA（SQL Descriptor Area）用の変数構造体を生成します。

制御フローとデータフロー

1. **蓄積**: 構文解析中に push_assignment が呼ばれ、記述子に対する操作項目がグローバルな assignments リストに蓄積されます。 2. **生成**: output_get_descr 等の関数が呼び出されると、assignments リストを走査し、ECPGdump_a_type を用いて各項目の型情報を出力ストリーム（base_yyout）に書き出します。 3. **クリーンアップ**: 各出力関数の終了時に drop_assignments が呼ばれ、リストが解放されます。 4. **エラーハンドリング**: 記述子項目が未実装であったり、型が不適切な場合は mmerror または mmfatal を通じてプリプロセスエラーを報告します。

直接依存

  • postgres_fe.h: PostgreSQLフロントエンド用の共通定義。
  • preproc_extern.h: プリプロセッサ全体で共有される外部変数や関数プロトタイプ。

関連コード

  • src/interfaces/ecpg/preproc/variable.c: 変数検索（find_variable）の定義。
  • src/interfaces/ecpg/preproc/output.c: コード生成の基盤となる出力ユーティリティ。
  • src/interfaces/ecpg/preproc/type.c: 型情報のダンプ処理。

読者向けメモ

  • 記述子は通常の変数とは異なり、variable リストには含まれません。そのため、descriptor_variable 関数で専用の静的構造体を生成して対応しています。
  • assignments リストは一時的なスタックとして機能します。記述子操作のコード生成が完了するたびに必ず drop_assignments を呼び出し、メモリリークを防ぐ設計になっています。
  • sqlda_variable は loc_alloc を使用しており、現在の文の処理が終われば自動的に解放されるライフサイクルを前提としています。
