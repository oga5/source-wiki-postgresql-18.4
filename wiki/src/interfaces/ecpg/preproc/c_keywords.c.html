src/interfaces/ecpg/preproc/c_keywords.c

概要

ECPG（Embedded SQL for C）プリプロセッサにおいて、C言語の予約語を識別するための字句解析用ルックアップ機能を提供します。

モジュール内での役割

ecpg プリプロセッサがソースコードを解析する際、識別子がC言語の予約語であるかを判定する役割を担います。ecpg_keywords.c がSQLキーワードを扱うのに対し、本ファイルはC言語固有のキーワードを対象としています。

責務

  • C言語キーワードのトークンリストの保持。
  • 入力文字列が予約語であるかの高速な判定（ハッシュ検索）。
  • 予約語と判定された場合の対応するトークン値の返却。

重要なシンボル

  • ScanCKeywordTokens: c_kwlist.h を用いて生成される、各キーワードに対応するトークン値の配列。
  • ScanCKeywordLookup: 指定された文字列が予約語かを確認するメイン関数。

制御フローとデータフロー

1. **入力**: ScanCKeywordLookup に解析対象の文字列が渡されます。 2. **事前判定**: 文字列長が最大キーワード長を超えている場合、即座に -1 を返して終了します。 3. **ハッシュ検索**: ScanCKeywords_hash_func を呼び出し、完全ハッシュを用いて候補インデックス h を算出します。 4. **照合**: GetScanKeyword で取得したキーワードと入力文字列を strcmp で比較します。 5. **出力**: 一致した場合は ScanCKeywordTokens[h] を返し、不一致または範囲外の場合は -1 を返します。

直接依存

  • postgres_fe.h: PostgreSQLフロントエンド用共通ヘッダー。
  • c_kwlist_d.h: ハッシュ関数やキーワードリストの定義データ。
  • c_kwlist.h: キーワード名とトークン値の定義。
  • preproc_extern.h, preproc.h: プリプロセッサ共通のインターフェース定義。

関連コード

  • ecpg_keywords.c: SQLキーワードのルックアップを担当する対となるファイル。
  • keywords.c: 汎用的なキーワード検索ユーティリティ。
  • parser.c: 本ファイルで定義されたルックアップ関数を呼び出し、字句解析を行うメインロジック。

読者向けメモ

  • 本ファイルは、PG_KEYWORD マクロを再定義して c_kwlist.h をインクルードすることで、トークン値の配列を静的に構築しています。
  • SQLキーワードの検索とは異なり、C言語のキーワードは**大文字小文字を区別する**ため、strcmp による厳密な比較が行われています。
  • 完全ハッシュを使用しているため、検索計算量は文字列長に依存するものの、非常に効率的です。
