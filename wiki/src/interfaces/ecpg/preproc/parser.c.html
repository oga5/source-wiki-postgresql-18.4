src/interfaces/ecpg/preproc/parser.c

概要

ECPG（Embedded SQL for C）プリプロセッサにおける字句解析のフィルタリング層です。Bison（構文解析器）が LALR(1) の制約内で動作できるよう、スキャナ（pgc.l）から出力されるトークンを先読みし、必要に応じてトークンの置換や結合を行います。

モジュール内での役割

src/interfaces/ecpg/preproc モジュールにおいて、スキャナとパーサの間に位置する仲介役です。スキャナが生成するトークンストリームを調整し、構文解析器が曖昧さを回避して正しく SQL 文を解釈できるようにします。

責務

  • **先読みによるトークン置換**: FORMAT や NOT など、後続のトークンによって意味が変わるキーワードを、専用のトークン（FORMAT_LA 等）に変換する。
  • **トークンの結合**: UIDENT や USCONST に続く UESCAPE 句を検出し、これらを単一のトークンに統合する。
  • **位置情報の付与**: スキャナが設定しない yylloc（トークンの位置情報）を、yytext や yylval を基に生成・補完する。
  • **正規化**: キーワードの小文字変換を行い、ECPG ライブラリとの互換性を維持する。

重要なシンボル

  • filtered_base_yylex(): パーサから呼び出されるメインのトークン取得関数。先読みロジックを内包する。
  • base_yylex_location(): スキャナ（base_yylex）をラップし、トークンの位置情報（yylloc）を生成する補助関数。
  • have_lookahead: 先読みトークンが有効かどうかを保持するフラグ。
  • lookahead_token / lookahead_yylval: 先読みしたトークンの情報を一時保存する変数群。

追加の索引済みシンボル

制御フローとデータフロー

1. パーサが filtered_base_yylex() を呼び出す。 2. have_lookahead が真であれば、保存済みの先読みトークンを返し、フラグを偽にする。 3. 偽であれば、base_yylex_location() を呼び出し、スキャナから新しいトークンを取得する。 4. 取得したトークンが先読み対象（NOT, WITH 等）の場合、再度 base_yylex_location() を呼び出して次トークンを確認する。 5. 条件に応じてトークンを置換（例: NOT → NOT_LA）し、必要に応じて have_lookahead を真にして次回の呼び出しに備える。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用共通ヘッダ。
  • preproc_extern.h: プリプロセッサ全体で共有される外部変数や関数の宣言。
  • preproc.h: プリプロセッサの定義。

関連コード

  • pgc.l (スキャナ): トークンを生成するソース。本ファイルの base_yylex はここで定義されている。
  • gram.y (パーサ): 本ファイルが提供するトークンを受け取り、構文木を構築する。
  • src/include/parser/parser.h: PostgreSQL 本体のパーサ定義。本ファイルはこれに倣った設計となっている。

読者向けメモ

  • 本ファイルは、スキャナ側でバックトラックを発生させずに文法上の曖昧さを解決するための「フィルタ」です。
  • yylloc の生成において、文字列リテラル以外は小文字化を行っています。これは ECPG の歴史的な互換性維持のための仕様です。
  • UESCAPE の処理は、3 つのトークンを 1 つにまとめる複雑な処理を行っているため、修正時はトークンの消費順序に注意してください。
