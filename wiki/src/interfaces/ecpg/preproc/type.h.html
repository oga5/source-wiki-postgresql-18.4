src/interfaces/ecpg/preproc/type.h

概要

本ヘッダーファイルは、ECPG（Embedded SQL for C）プリプロセッサが扱うデータ型、変数、およびSQL文の構造を定義する中心的なヘッダーです。C言語の型とSQLの型をマッピングするためのメタデータ構造を提供します。

モジュール内での役割

src/interfaces/ecpg/preproc モジュールにおいて、型定義の基盤となります。プリプロセッサがソースコードを解析する際、ホスト変数の型情報や構造体メンバー、カーソル、プリペアードステートメントなどの情報を保持するために使用されます。

責務

  • **型定義の抽象化**: C言語の構造体や配列、単純型をECPGが理解可能な形式（ECPGtype）へ変換・保持する。
  • **メモリ管理のインターフェース**: 型構造体やメンバーリストの生成・解放関数の宣言。
  • **プリプロセッサ状態の保持**: カーソル、定義済みマクロ、変数リスト、SQL文の実行状態などを管理する構造体の定義。

重要なシンボル

  • ECPGtype: ホスト変数の型情報を保持する主要な構造体。
  • ECPGstruct_member: 構造体や共用体のメンバーを管理する連結リスト。
  • ECPGmake_*: ECPGtype や ECPGstruct_member を動的に生成するファクトリ関数群。
  • ECPGdump_a_type: 型情報を出力コードへ書き出すための関数。
  • struct variable: プリプロセッサが追跡するホスト変数の定義。

制御フローとデータフロー

本ファイルはデータ構造の定義が主であり、直接的な制御フローは持ちません。データフローは、type.c で実装された ECPGmake_* 関数群が ECPGtype 構造体を構築し、それを parser.c や variable.c が参照することで、解析中のコードから型情報を抽出・伝播させる形で発生します。

直接依存

  • ecpgtype.h: ECPGがサポートする基本的な型列挙（enum ECPGttype）を提供。

関連コード

  • src/interfaces/ecpg/preproc/type.c: 本ヘッダーで宣言された構造体の生成・解放・ダンプの実装。
  • src/interfaces/ecpg/preproc/variable.c: struct variable を利用したホスト変数の管理ロジック。
  • src/interfaces/ecpg/preproc/output.c: ECPGdump_a_type を利用して、解析結果をCコードとして出力する処理。

読者向けメモ

  • **メモリ管理**: 本ファイルで定義される構造体は、原則として mm_alloc 等で動的に確保されます。ECPGfree_* 関数による適切な解放が求められます。
  • **再帰的構造**: ECPGtype は u.element や u.members を通じて再帰的な構造（配列の中の構造体など）を表現可能です。解析時にはこの再帰構造を辿る必要がある点に注意してください。
  • **マクロ定義**: struct _defines は、コマンドライン引数（-D）とソース内の #define を区別して管理する仕組みを持っており、プリプロセッサのコンパイル単位を跨ぐ状態管理に重要です。
