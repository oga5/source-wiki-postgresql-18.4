src/interfaces/ecpg/preproc/preproc_extern.h

概要

本ファイルは、ECPG（Embedded SQL for C）プリプロセッサ全体で共有されるグローバル変数、関数プロトタイプ、および定数定義を保持する中心的なヘッダーファイルです。

モジュール内での役割

src/interfaces/ecpg/preproc モジュールにおける「インターフェース定義」の役割を担います。各ソースファイル（parser.c, output.c, variable.c など）が互いの関数や状態を参照するための共通基盤を提供します。

責務

  • プリプロセッサの動作状態（フラグ、現在のファイル名、ネストレベル等）を保持するグローバル変数の宣言。
  • パーサー、出力生成、メモリ管理、変数管理に関する主要関数のエクスポート。
  • 互換性モード（Informix, Oracle等）やエラーコードの定義。

重要なシンボル

  • YYLTYPE: パーサーが使用する位置情報追跡用の型。
  • COMPAT_MODE: ターゲットデータベースの互換性モードを制御する列挙型。
  • STRUCT_DEPTH: 構造体解析時のネスト上限。
  • base_yyparse(): プリプロセッサのメインパーサー関数。
  • mmerror() / mmfatal(): プリプロセッサ専用のエラーおよび致命的エラー報告関数。

制御フローとデータフロー

本ファイル自体はロジックを含みませんが、以下のフローを支えています： 1. **初期化**: lex_init() や parser_init() を通じてパーサーと字句解析器の状態を初期化。 2. **解析**: base_yyparse() が base_yylex() を呼び出し、入力ソースをトークン化して解析。 3. **状態管理**: variable.c や type.c で定義された構造体（struct variable, struct typedefs 等）をグローバル変数として保持し、解析の進行に合わせて更新。 4. **出力**: output_statement() 等の関数が、解析結果を基に C コードを生成。

直接依存

  • common/keywords.h: SQL キーワード定義。
  • type.h: ECPG で使用する型定義。
  • <limits.h>: CHAR_BIT 等のシステム制限定義。
  • limits.h

関連コード

  • parser.c: base_yyparse の実装元。
  • variable.c: 本ファイルで宣言された変数管理関数の実装元。
  • output.c: 本ファイルで宣言されたコード生成関数の実装元。

読者向けメモ

  • 本ファイルは非常に多くのグローバル変数を宣言しています。これらはプリプロセッサの「現在の状態」を保持しているため、デバッグ時にはこれらの変数の値を確認することが重要です。
  • mm_alloc や mm_strdup などのメモリ管理関数は、プリプロセッサ内でのメモリリークを防ぐためのラッパーです。メモリ確保が必要な場合はこれらを使用してください。
  • INFORMIX_MODE や ORACLE_MODE マクロは、特定のデータベース互換性が必要なコードパスを分岐させるために頻繁に使用されます。
