src/interfaces/ecpg/preproc/ecpg_keywords.c

概要

ECPG（Embedded SQL for C）プリプロセッサにおいて、入力された文字列がSQLキーワードまたはECPG固有のキーワードであるかを判定し、対応するトークン値を返すためのルックアップ機能を提供します。

モジュール内での役割

src/interfaces/ecpg/preproc モジュールにおける字句解析フェーズの一部を担います。パーサーがソースコードをトークン化する際、識別子とキーワードを区別するための辞書的な役割を果たします。

責務

  • ECPG固有のキーワードリストの管理。
  • PostgreSQLバックエンドの標準SQLキーワードと、ECPG拡張キーワードの二段階検索。
  • 大文字・小文字を区別しないキーワード照合の実行。

重要なシンボル

  • ScanECPGKeywordLookup: 外部から呼び出されるメイン関数。文字列を受け取り、一致するトークンIDまたは -1 を返します。
  • ECPGScanKeywordTokens: ecpg_kwlist.h から生成される、ECPGキーワードに対応するトークン値の配列。
  • PG_KEYWORD: マクロ。ecpg_kwlist.h をインクルードする際にトークン値のリストを構築するために使用されます。

制御フローとデータフロー

1. **入力**: 字句解析器から識別子候補の文字列（text）が渡されます。 2. **バックエンド検索**: ScanKeywordLookup を使用し、まず標準的なSQLキーワード（ScanKeywords）を検索します。一致すればそのトークンを返します。 3. **ECPG拡張検索**: SQLキーワードで見つからなかった場合、ScanECPGKeywords を対象にECPG固有のキーワードを検索します。 4. **出力**: 一致した場合は対応する ECPGScanKeywordTokens の値を返し、いずれにも該当しない場合は -1 を返します。

直接依存

  • postgres_fe.h: PostgreSQLフロントエンド用共通ヘッダー。
  • ecpg_kwlist_d.h: ECPGキーワードの定義データ構造。
  • ecpg_kwlist.h: キーワードリストの定義（マクロ展開用）。
  • preproc_extern.h / preproc.h: プリプロセッサの共通定義および外部インターフェース。
  • ctype.h

関連コード

  • src/interfaces/ecpg/preproc/keywords.c: ECPG全体で使用されるキーワード検索の基盤。
  • src/interfaces/ecpg/preproc/parser.c: 本ファイルで定義されたルックアップ関数を利用して字句解析を行うパーサー本体。
  • src/include/parser/kwlist.h: バックエンド側の標準SQLキーワード定義。

読者向けメモ

  • 本ファイルは、PG_KEYWORD マクロを再定義してヘッダーファイルをインクルードする手法（X-Macroパターン）を採用しています。これにより、キーワードの追加・変更時にリストの整合性を保ちやすくなっています。
  • 検索ロジックはバックエンドのキーワード検索と共通のルール（大文字・小文字の正規化など）に従っているため、バックエンドの仕様変更の影響を受けやすい点に注意してください。
