src/interfaces/ecpg/preproc/ecpg.c

概要

ecpg.c は、PostgreSQL の埋め込み SQL プリプロセッサ（ecpg）のメインエントリポイントです。C 言語ソースコード内の SQL 文を解析し、ecpglib を呼び出す C コードに変換するための初期化、コマンドライン引数の処理、およびファイル処理の制御を行います。

モジュール内での役割

ecpg プリプロセッサの司令塔として機能します。パーサー（parser.c）や出力生成（output.c）などの各コンポーネントを統合し、ユーザーが指定した入力ファイルに対して一連の変換プロセスを適用します。

責務

  • **CLI 解析**: getopt_long を使用し、互換モード（Informix/Oracle）、インクルードパス、プリプロセッサ定義などのオプションを処理する。
  • **環境構築**: プリプロセッサの動作に必要なグローバル状態（カーソル、型定義、定義済みシンボル）の初期化とクリーンアップ。
  • **ファイル制御**: 入力ファイルのオープン、出力ファイルの生成、およびエラー発生時の出力ファイル削除（ロールバック）の管理。
  • **解析実行**: base_yyparse() を呼び出し、字句解析および構文解析プロセスを開始する。

重要なシンボル

  • main: プログラムの実行起点。引数解析から解析ループ、終了処理までを統括する。
  • compat: 互換モード（PostgreSQL, Informix, Oracle）を保持する列挙型。
  • include_paths: EXEC SQL INCLUDE で検索されるディレクトリのリスト。
  • add_preprocessor_define: コマンドラインから渡された -D オプションを内部の定義リストに登録する。
  • ret_value: プリプロセス全体の成功・失敗状態を保持するグローバル変数。

制御フローとデータフロー

1. **初期化**: main が起動し、ロケール設定と実行パスの特定を行う。 2. **引数処理**: getopt_long でオプションを解析し、グローバル変数（compat, include_paths 等）を更新する。 3. **ファイルループ**: 指定された各入力ファイルに対し、以下の処理を繰り返す。

  • 内部状態（カーソル、型、定義）をリセット。
  • base_yyin（入力）と base_yyout（出力）を準備。
  • base_yyparse() を呼び出し、解析を実行。
  • 解析完了後、エラーがあれば出力ファイルを削除し、正常終了ならファイルを閉じる。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用の共通定義。
  • getopt_long.h: コマンドライン引数解析用。
  • preproc_extern.h: プリプロセッサ全体で共有される外部変数や関数宣言。
  • unistd.h

関連コード

  • parser.c: base_yyparse() の実装元。
  • output.c: 解析結果を C コードとして出力するための関数群。
  • type.c / variable.c: 埋め込み SQL で使用される型や変数の管理。
  • util.c: メモリ割り当てや文字列操作などのユーティリティ。

読者向けメモ

  • **状態の保持**: 複数のファイルを処理する際、cur（カーソル）や types（型定義）などのグローバルなデータ構造が適切にリセットされる点に注意してください。
  • **エラー処理**: ret_value が非ゼロの場合、生成された出力ファイルは unlink で削除されます。これはプリプロセッサが不完全なコードを生成したまま残さないための重要な安全策です。
  • **互換モード**: -C オプションによる動作分岐は、compat 変数を通じて各モジュールに伝播します。特定の互換性挙動を追う場合は、この変数の参照箇所を確認してください。
