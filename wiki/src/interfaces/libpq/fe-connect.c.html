src/interfaces/libpq/fe-connect.c

概要

fe-connect.c は、PostgreSQL クライアントライブラリ libpq における接続確立プロセスの中心的な実装です。接続文字列（または URI）のパース、環境変数や設定ファイルからのデフォルト値の解決、物理的なソケット接続、および認証ネゴシエーション（SSL/GSSAPI/SASL）のライフサイクルを管理します。

モジュール内での役割

libpq の接続管理層の基盤であり、PGconn 構造体の初期化と状態遷移を担います。fe-exec.c がクエリ実行を担うのに対し、本ファイルは「接続が確立され、認証が完了し、クエリ実行可能な状態になるまで」の準備を完結させます。

責務

  • **接続パラメータの管理**: PQconninfoOption を用いた接続オプションのパース、検証、および PGconn への適用。
  • **接続ライフサイクル**: 同期・非同期接続の開始、ポーリング（PQconnectPoll）、および接続のクリーンアップ。
  • **認証・セキュリティのネゴシエーション**: SSL/TLS、GSSAPI、SASL 認証のネゴシエーションと状態管理。
  • **環境統合**: .pgpass ファイルの読み込み、LDAP サービスルックアップ、環境変数からのデフォルト値解決。
  • **プラットフォーム抽象化**: Windows (Winsock) と Unix 間のソケット操作や keepalive 設定の差異を吸収。

重要なシンボル

接続管理・ライフサイクル

  • PGconn: 接続状態を保持するメイン構造体。
  • PQconnectdbParams / PQconnectStartParams: 接続開始の公開 API。
  • PQconnectPoll: 非同期接続の進捗を管理する状態マシン。
  • pqDropConnection: 物理的なソケットを閉じ、認証状態をリセットする内部関数。
  • pqClosePGconn: 接続を閉じ、再接続可能な状態へリセットする関数。

オプション管理

  • PQconninfoOptions[]: サポートされる全接続オプションの定義（環境変数、デフォルト値、PGconn へのオフセット）。
  • internalPQconninfoOption: PGconn 構造体へのマッピング情報を含む内部定義。
  • conninfo_parse / conninfo_uri_parse: 接続文字列や URI をパースしてオプション配列を生成する関数群。

認証・セキュリティ

  • supported_sasl_mechs: 利用可能な SASL 認証メカニズムのリスト。
  • init_allowed_encryption_methods: sslmode や gssencmode に基づき、試行すべき暗号化方式を決定する。

制御フローとデータフロー

1. **初期化**: PQconnectStart が pqMakeEmptyPGconn で PGconn を確保し、connectOptions1 でオプションをパース・適用する。 2. **解決**: pqConnectOptions2 がホスト名解決、ポート決定、認証情報の取得（passwordFromFile）を行い、接続先リストを構築する。 3. **接続試行**: PQconnectPoll が状態マシンとして動作し、CONNECTION_NEEDED から順次状態を遷移させる。

  • CONNECTION_STARTED: ソケット作成と connect() の実行。
  • CONNECTION_MADE: 暗号化（SSL/GSSAPI）のネゴシエーション。
  • CONNECTION_AWAITING_RESPONSE: スタートアップパケット送信と認証要求の待機。

    • 4. **終了**: PQfinish が pqClosePGconn を呼び出し、リソースを解放する。

読者向けメモ

  • **メモリ所有権**: PGconn 内の文字列メンバは strdup されており、freePGconn で解放されます。PQconninfoOption 配列は PQconninfoFree で解放する必要があります。
  • **状態マシン**: PQconnectPoll は非ブロッキング I/O を前提としています。select() 等でソケットの状態を監視し、必要に応じてこの関数を再呼び出しする設計です。
  • **省略された詳細**: LDAP 関連の複雑なパース処理や、特定のプラットフォーム固有の keepalive 設定の詳細は、実装の保守性を考慮して関数化されています。
  • **注意点**: PQconnectPoll 内の goto keep_going は、状態遷移をループ内で継続させるための制御構造であり、複雑な状態遷移をフラットに記述するために使用されています。
