src/interfaces/libpq/win32.c

概要

Windows環境における libpq のソケットエラー処理を補完するためのサポートモジュールです。主に、Winsockエラーコードを人間が読み取れる文字列に変換する機能を提供します。

モジュール内での役割

libpq はクロスプラットフォームなライブラリですが、Windows特有のソケットエラー（WSA*系）は標準的な strerror では解決できません。本ファイルは、win32.h で宣言されたインターフェースを実装し、Windows環境下でのエラー診断を可能にします。

責務

  • **エラーコードの変換**: Winsockエラーコードを定義済みのテーブルから検索し、説明文を返します。
  • **動的DLLロード**: テーブルに存在しないエラーの場合、WindowsのシステムDLL（ws2_32.dll 等）を動的にロードし、FormatMessage API を使用してシステムからエラーメッセージを取得します。
  • **フォールバック**: 変換に失敗した場合でも、エラーコードを16進数および10進数でフォーマットして返します。

重要なシンボル

  • winsock_strerror: 本ファイルのメイン関数。エラーコードを文字列に変換します。
  • WSErrors: 一般的なWinsockエラーコードと説明文を保持する静的テーブル。
  • dlls: エラーメッセージ取得のために検索対象となるWindows DLLのリスト。

追加の索引済みシンボル

  • LookupWSErrorMessage
  • MessageDLL
  • WSErrorEntry

制御フローとデータフロー

1. winsock_strerror が呼び出されると、まず LookupWSErrorMessage を介して WSErrors テーブルを線形探索します。 2. テーブルで見つからない場合、dlls 配列内の各DLLを LoadLibraryEx で順次ロードし、FormatMessage を試行します。 3. 成功した場合はメッセージをバッファに書き込み、最後にエラーコードを付加して返します。 4. 全ての試行が失敗した場合は、汎用的な「unrecognized socket error」メッセージを生成します。

直接依存

  • postgres_fe.h: PostgreSQLフロントエンド用共通ヘッダー。
  • win32.h: 本モジュールのインターフェース定義。
  • Windows API: LoadLibraryEx, FormatMessage などのシステム関数。

関連コード

  • src/interfaces/libpq/win32.h: winsock_strerror のプロトタイプ宣言および、Windows環境用の互換性マクロ（close, read 等）が定義されています。

読者向けメモ

  • **パフォーマンス**: エラー発生時のみ実行されるため、テーブルの線形探索やDLLのロードはパフォーマンス上のボトルネックとは見なされていません。
  • **DLLロード**: dlls 配列内のDLLは、一度ロードされるとプロセス終了まで保持されます。
  • **バッファ管理**: winsock_strerror は呼び出し元から提供されたバッファを使用します。バッファサイズが不足しないよう、呼び出し側は十分な領域を確保する必要があります。
