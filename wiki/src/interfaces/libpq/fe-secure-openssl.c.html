src/interfaces/libpq/fe-secure-openssl.c

概要

本ファイルは、libpq における OpenSSL を用いたセキュア通信（TLS/SSL）の実装を提供します。接続の初期化、証明書検証、暗号化通信の I/O 抽象化を担い、PostgreSQL クライアントが安全にサーバーと通信するための基盤となります。

モジュール内での役割

libpq のセキュリティ層の一部であり、fe-secure.c が提供する汎用的なセキュリティインターフェースの OpenSSL 固有の実装です。PGconn 構造体と密接に連携し、ソケット通信を TLS でラップします。

責務

  • **SSL コンテキスト管理**: SSL_CTX の生成、証明書・秘密鍵の読み込み、プロトコルバージョンの制御。
  • **TLS ハンドシェイク**: open_client_SSL による接続確立と、SNI/ALPN の設定。
  • **I/O 抽象化**: OpenSSL の BIO メソッドをカスタマイズし、libpq の非ブロッキング I/O と統合。
  • **証明書検証**: ホスト名と証明書の照合（SAN/CN の確認）、CRL チェック。
  • **エラーハンドリング**: OpenSSL エラーキューの管理と、PGconn へのエラーメッセージ転送。

重要なシンボル

接続・I/O 操作

  • pgtls_open_client: SSL 接続の初期化とハンドシェイクを開始するエントリーポイント。
  • pgtls_read / pgtls_write: SSL_read/SSL_write をラップし、エラーコードを libpq の期待する形式に変換。
  • pgtls_close: SSL オブジェクトと関連リソース（証明書、エンジン）の解放。

証明書・検証

  • pgtls_verify_peer_name_matches_certificate_guts: ホスト名と証明書の照合ロジック。
  • pgtls_get_peer_certificate_hash: サーバー証明書のハッシュ値を取得（SCRAM 認証等で使用）。

設定・フック

  • PQsetSSLKeyPassHook_OpenSSL: クライアント証明書のパスワード取得用フックを設定。
  • initialize_SSL: PGconn の設定に基づき SSL_CTX を構築する内部関数。

制御フローとデータフロー

1. **初期化**: pgtls_open_client が呼ばれると initialize_SSL が実行され、SSL_CTX が作成されます。ここで証明書ファイルや鍵が読み込まれ、SSL_set_bio によりカスタム BIO が設定されます。 2. **I/O フロー**: pgtls_read/write は OpenSSL の SSL_read/write を呼び出します。OpenSSL は内部でカスタム BIO (pgconn_bio_read/write) を経由して、libpq の低レベルソケット関数を呼び出します。 3. **エラー処理**: ERR_clear_error でキューをクリアしてから操作を行い、失敗時は SSL_get_error で詳細を判定して conn->errorMessage に書き込みます。 4. **終了**: pgtls_close が呼ばれると、SSL_shutdown を経て SSL_free が実行され、リソースが解放されます。

読者向けメモ

  • **メモリ所有権**: SSL_CTX は接続ごとに作成され、SSL オブジェクト作成後に即座に解放されます（OpenSSL の参照カウントにより維持されます）。
  • **スレッド安全性**: ssl_config_mutex は BIO_METHOD の初期化を保護するために使用されます。
  • **BIO の役割**: OpenSSL 標準のソケット BIO ではなく、libpq 独自の BIO を使用することで、SIGPIPE の抑制や詳細なエラーログ出力を実現しています。
  • **未実装・浅い部分**: 証明書検証のコールバック (verify_cb) は現在デフォルトの検証をそのまま受け入れる実装となっており、詳細なログ出力は行われません。
