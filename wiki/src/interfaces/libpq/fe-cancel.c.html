src/interfaces/libpq/fe-cancel.c

概要

本ファイルは、PostgreSQLサーバーに対して実行中のクエリをキャンセルするためのリクエスト送信機能を実装しています。非同期（ノンブロッキング）な新しいAPIと、シグナルハンドラ内での呼び出しを想定した従来のブロッキングAPIの両方を提供します。

モジュール内での役割

libpq クライアントライブラリの一部として、接続状態とは独立した「キャンセル専用」の通信経路を確立・管理します。既存の PGconn から必要な認証情報（PIDおよびキャンセルキー）を抽出し、サーバーへキャンセル要求を伝達する役割を担います。

責務

  • **キャンセル情報の抽出**: PQgetCancel を通じて、既存接続からキャンセルに必要なトークンを安全にコピーする。
  • **キャンセル実行**:
  • PQcancelCreate / PQcancelStart / PQcancelPoll: 非同期かつスレッドセーフなキャンセル要求の送信。
  • PQcancel: 従来のシグナル安全なブロッキングキャンセル要求。
  • **リソース管理**: キャンセル用接続の作成、ソケット設定、および終了処理。

重要なシンボル

  • struct pg_cancel: キャンセル要求に必要な情報（PID、キー、ソケット設定）を保持する構造体。
  • PQcancelCreate: キャンセル専用の PGcancelConn を作成する。
  • PQcancel: シグナルハンドラ内での使用を想定した、低レベルなブロッキングキャンセル関数。
  • PQsendCancelRequest: 確立済みのキャンセル用接続を通じてパケットを送信する。

制御フローとデータフロー

1. **初期化**: PQgetCancel または PQcancelCreate により、対象接続の認証情報とアドレスをコピーしたオブジェクトを生成する。 2. **接続**:

  • 非同期フローでは PQcancelStart で接続を開始し、PQcancelPoll でソケットの準備完了を待機する。
  • PQcancel では、直接 socket() -> connect() を行い、ブロッキングで送信する。

    • 3. **送信**: CancelRequestPacket を構築し、サーバーへ送信する。 4. **完了待機**: サーバーがキャンセル要求を受理し、接続を閉じる（EOF）のを待機する。

直接依存

  • postgres_fe.h: フロントエンド用共通定義。
  • libpq-fe.h / libpq-int.h: libpq の公開および内部インターフェース。
  • unistd.h: POSIX システムコール用。
  • port/pg_bswap.h: ネットワークバイトオーダー変換用。

関連コード

  • fe-connect.c: 接続確立のロジックを共有（pqConnectDBStart 等）。
  • fe-misc.c: ソケット操作やメッセージ送信の補助関数を提供。

読者向けメモ

  • **シグナル安全性**: PQcancel はシグナルハンドラ内での呼び出しを考慮し、malloc や sprintf 等の非リエントラントな関数を避けて実装されています。この関数を修正する際は、この制約を維持してください。
  • **Windowsの挙動**: Windows環境ではソケットのEOF検知の挙動が異なるため、PQcancelPoll 等でプラットフォーム固有の分岐が存在します。
  • **非同期APIの推奨**: 新規開発では、スレッドセーフで柔軟な PQcancelCreate を含む非同期API群の使用が推奨されます。
