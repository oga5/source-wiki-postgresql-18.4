src/interfaces/libpq/pthread-win32.c

概要

本ファイルは、Windows環境における libpq のスレッドセーフな動作をサポートするための、限定的な pthread API 実装を提供します。POSIXスレッドライブラリが標準で存在しないWindowsにおいて、最小限の同期プリミティブを抽象化します。

モジュール内での役割

libpq はクロスプラットフォームなライブラリであり、内部でスレッド同期を必要とする箇所があります。本モジュールは、Windows固有のAPI（CRITICAL_SECTION 等）を pthread インターフェースでラップすることで、libpq のメインロジックがプラットフォーム差異を意識せずに同期処理を行えるようにします。

責務

  • **ミューテックスの抽象化**: pthread_mutex_t を用いた排他制御の提供。
  • **遅延初期化**: ミューテックスの初期化を初回ロック時に行うことで、静的初期化の安全性を確保。
  • **スレッド識別**: pthread_self によるスレッドIDの取得。
  • **スタブ提供**: pthread_setspecific / pthread_getspecific のスタブ実装（現状では機能せず）。

重要なシンボル

  • pthread_mutex_lock: CRITICAL_SECTION を用いた排他制御。initstate を用いたスレッドセーフな遅延初期化ロジックを含みます。
  • pthread_mutex_unlock: LeaveCriticalSection を呼び出し、ロックを解放します。
  • pthread_mutex_init: ミューテックス構造体の初期状態を設定します。
  • pthread_self: GetCurrentThreadId() をラップし、現在のスレッドIDを返します。

追加の索引済みシンボル

  • pthread_getspecific
  • pthread_setspecific

制御フローとデータフロー

1. **初期化フロー**: pthread_mutex_lock が呼び出されると、initstate を確認します。未初期化（0）であれば、InterlockedExchange を用いて初期化中（2）に遷移させ、他スレッドとの競合を防ぎつつ InitializeCriticalSection を実行します。 2. **ロックフロー**: 初期化完了後、EnterCriticalSection を呼び出し、OSレベルの排他制御に入ります。 3. **データフロー**: pthread_setspecific 等は現在スタブであり、データは保持されません。

  • pthread-win32.h: 本実装に対応する型定義（pthread_mutex_t 等）および関数プロトタイプ。

読者向けメモ

  • **機能制限**: 本実装は完全な pthread 実装ではありません。特にスレッドローカルストレージ（pthread_setspecific 等）は実装されておらず、呼び出しても何も行われません。
  • **同期の仕組み**: pthread_mutex_lock 内の InterlockedExchange によるスピンロック的な待機処理は、初期化の競合を避けるための重要なパターンです。
  • **利用上の注意**: このコードは libpq の内部的な同期ニーズを満たすためだけに設計されています。汎用的なスレッドライブラリとしての利用は想定されていません。
