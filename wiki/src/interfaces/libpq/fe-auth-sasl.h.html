src/interfaces/libpq/fe-auth-sasl.h

概要

本ヘッダーファイルは、libpq における SASL（Simple Authentication and Security Layer）認証メカニズムのインターフェース定義を提供します。PostgreSQL クライアントがサーバーと SASL 認証を行うための抽象化レイヤーを定義しており、特定の認証方式（SCRAM など）をプラグインとして追加可能にするための構造体を提供します。

モジュール内での役割

src/interfaces/libpq モジュール内において、認証ロジックの拡張性を担保する役割を担います。fe-auth.c などの認証処理本体が、具体的な認証方式に依存せずに SASL 交換を実行できるようにするための契約（コントラクト）を定義しています。

責務

  • **抽象化**: 各 SASL メカニズムが実装すべきコールバック関数のシグネチャを定義する。
  • **状態管理**: 認証プロセス中の状態（SASLStatus）と、メカニズム固有の状態（pg_fe_sasl_mech）の管理規約を定める。
  • **ライフサイクル定義**: 認証の初期化、メッセージ交換、チャネルバインディング確認、リソース解放の各フェーズを規定する。

重要なシンボル

  • SASLStatus: 認証交換の進捗状態を示す列挙型。SASL_COMPLETE（成功）、SASL_FAILED（失敗）、SASL_CONTINUE（継続）、SASL_ASYNC（非同期処理待ち）を定義。
  • pg_fe_sasl_mech: SASL メカニズムを実装するための構造体。init, exchange, channel_bound, free の4つの関数ポインタを保持する。

制御フローとデータフロー

1. **初期化**: init() が呼び出され、メカニズム固有のコンテキストが生成されます。 2. **交換**: exchange() がサーバーからのチャレンジを受け取り、クライアントのレスポンスを生成します。この際、inputlen を用いてバイナリデータ（埋め込み NULL を含む可能性あり）を安全に扱います。 3. **状態遷移**: exchange() の戻り値（SASLStatus）に基づき、認証を継続するか、完了するか、あるいは非同期処理へ移行するかが決定されます。 4. **終了**: 接続終了時に free() が呼ばれ、init() で確保されたリソースが解放されます。

直接依存

  • libpq-fe.h: PGconn 構造体の定義を参照するためにインクルードされています。

関連コード

  • src/include/libpq/sasl.h: サーバー側の SASL 実装との対照となるヘッダー。
  • src/interfaces/libpq/fe-auth-scram.c: 本インターフェースを実装した具体的な認証方式の例。
  • src/interfaces/libpq/fe-auth.c: 本ヘッダーで定義されたインターフェースを利用して認証フローを制御するメインロジック。

読者向けメモ

  • 本ヘッダーは libpq の内部 API であり、アプリケーション開発者が直接利用する公開 API ではありません。
  • exchange() 関数において、input は NULL 終端されていますが、SASL の仕様上バイナリデータが含まれる可能性があるため、必ず inputlen を参照して処理を行う必要があります。
  • 非同期認証（SASL_ASYNC）を実装する場合、conn->async_auth への適切な設定が必須となります。
