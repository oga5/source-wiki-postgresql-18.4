src/interfaces/libpq/fe-trace.c

概要

fe-trace.c は、PostgreSQL のフロントエンド・バックエンド間の通信プロトコルをデバッグ用にトレース出力するための機能を実装しています。接続ごとのトレース設定管理と、バイナリ形式のプロトコルメッセージを人間が読みやすいテキスト形式に変換して出力する役割を担います。

モジュール内での役割

libpq の通信スタックの一部として、プロトコルメッセージの送受信時にフックされるデバッグユーティリティです。PGconn 構造体に紐付いたトレース設定を読み取り、通信内容を FILE * ストリームへ書き出します。

責務

  • **トレース制御**: PQtrace / PQuntrace によるデバッグ出力の有効化・無効化および PQsetTraceFlags による詳細設定。
  • **メッセージ解析**: フロントエンドおよびバックエンドのプロトコルメッセージを解析し、型に応じたフォーマットで出力。
  • **回帰テスト対応**: PQTRACE_REGRESS_MODE を通じて、テスト結果に影響を与える可変情報（タイムスタンプや特定の内部フィールド）を抑制する機能の提供。

重要なシンボル

  • PQtrace: トレース対象の接続と出力先を関連付けるエントリポイント。
  • pqTraceOutputMessage: メッセージの型を識別し、適切な解析関数へディスパッチするメインルーチン。
  • pqTraceOutputNoTypeByteMessage: 型バイトを持たない特殊なメッセージ（StartupMessage, SSLRequest 等）の処理。
  • PGconn.Pfdebug: トレース出力先のファイルポインタ。

制御フローとデータフロー

1. **初期化**: PQtrace が PGconn に出力先を設定。 2. **メッセージ受信/送信**: 通信処理中に pqTraceOutputMessage が呼び出される。 3. **解析**:

  • メッセージの先頭バイト（ID）と長さを読み取る。
  • regress モード判定を行い、必要に応じて出力をサニタイズする。
  • switch 文でメッセージ型を特定し、pqTraceOutput_XXX 系の関数でペイロードをデコードする。

    • 4. **出力**: 解析されたデータが Pfdebug に書き込まれる。

読者向けメモ

  • **メッセージの重複**: プロトコル上、異なるメッセージが同じ型バイトを共有している場合があります（例: Describe と DataRow）。このファイルでは toServer フラグや conn->current_auth_response を用いて文脈を判断しています。
  • **回帰テスト**: regress モードは、ファイル名や行番号など、サーバー側の変更で頻繁に変わる値を隠蔽するために重要です。新しいメッセージ型を追加する際は、このモードでの出力がテストを壊さないか確認してください。
  • **拡張**: 新しいプロトコルメッセージを追加した場合は、pqTraceOutputMessage の switch 文にケースを追加し、必要に応じて専用の pqTraceOutput_XXX 関数を実装してください。
