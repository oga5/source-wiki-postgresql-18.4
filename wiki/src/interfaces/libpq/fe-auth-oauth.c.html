src/interfaces/libpq/fe-auth-oauth.c

概要

本ファイルは、PostgreSQL クライアントライブラリ (libpq) における SASL OAUTHBEARER 認証メカニズムの実装です。サーバーとの認証ネゴシエーション、OAuth 2.0/OIDC 準拠のディスカバリ（.well-known URI の解析）、およびトークン取得のための非同期フロー管理を担います。

モジュール内での役割

libpq の認証サブシステムの一部として、fe-auth.c から SASL メカニズムとして呼び出されます。接続確立時にトークンが不足している場合、サーバーからディスカバリ情報を取得し、外部プラグインまたはアプリケーション定義のコールバックを通じてトークンを再取得する「再試行」ループを制御します。

責務

  • **SASL ステートマシン**: OAUTHBEARER プロトコルに従った初期応答の生成と、サーバーからのエラー応答（ディスカバリ情報を含む）の解析。
  • **OAuth ディスカバリ**: RFC 8414 に基づく .well-known URI の検証と、Issuer ID の導出。
  • **非同期フロー管理**: ユーザー定義の認証フック (PQAUTHDATA_OAUTH_BEARER_TOKEN) または動的ロードされる libpq-oauth プラグインの実行。
  • **セキュリティ**: トークンのメモリ保護（explicit_bzero）と、HTTPS 必須の強制。

重要なシンボル

SASL インターフェース

  • pg_oauth_mech: libpq の認証ディスパッチャに登録される SASL メカニズム構造体。
  • oauth_exchange: 認証のメインステートマシン。FE_OAUTH_INIT から FE_OAUTH_REQUESTING_TOKEN までの遷移を管理。

JSON 解析 (ディスカバリ用)

  • struct json_ctx: サーバーからのエラー応答をパースするためのコンテキスト。
  • oauth_json_scalar: JSON 値を抽出し、status, scope, openid-configuration フィールドを PGconn に格納する。

フロー制御

  • setup_token_request: ユーザー定義フックまたは組み込みフローのいずれかを選択し、非同期実行の準備を行う。
  • use_builtin_flow: libpq-oauth プラグインを dlopen でロードするか、静的リンクされた実装を有効化する。

制御フローとデータフロー

1. **初期化**: oauth_init で fe_oauth_state を作成。 2. **認証試行**: oauth_exchange が呼ばれ、client_initial_response でトークンを送信。 3. **ディスカバリ**: サーバーが invalid_token を返した場合、handle_oauth_sasl_error が JSON を解析し、conn->oauth_discovery_uri 等を更新。 4. **非同期取得**: SASL_ASYNC を返し、run_user_oauth_flow を経由してトークンを取得。 5. **再試行**: トークン取得後、conn->oauth_want_retry をセットして SASL_FAILED を返し、libpq の接続ループで再接続をトリガーする。

読者向けメモ

  • **メモリ所有権**: conn->oauth_token は pqClearOAuthToken で確実にゼロクリアされます。async_ctx に格納されるユーザー定義リクエストは、cleanup_user_oauth_flow で解放されます。
  • **設計上の制約**: セキュリティ上の理由から、Issuer は接続ごとに 1 つに制限されています。
  • **未実装/省略**: 本ファイルは認証プロトコルの実装に特化しており、実際の HTTP 通信（トークン取得）は libpq-oauth プラグインまたはアプリケーション側のフックに委譲されています。
  • **デバッグ**: PGOAUTHDEBUG=UNSAFE を設定することで、開発環境での HTTP 通信許可や詳細なエラーログ出力が可能です。
