src/interfaces/libpq/fe-auth.c

概要

fe-auth.c は、PostgreSQL クライアント（libpq）における認証ネゴシエーションのメインディスパッチャです。サーバーからの認証要求（AuthRequest）を解析し、適切な認証メカニズム（GSSAPI, SSPI, SASL, MD5, Password）へ処理を委譲します。また、パスワードの暗号化やユーザー名の取得といった認証関連のユーティリティ関数も提供します。

モジュール内での役割

libpq の接続確立プロセス（fe-connect.c の PQconnectPoll）において、認証フェーズを担う中核モジュールです。認証プロトコルの詳細実装（SCRAM, OAuth 等）を各 fe-auth-*.c に分離し、本ファイルはそれらを統合するインターフェース層として機能します。

責務

  • **認証ディスパッチ**: サーバーからの AuthRequest に基づき、適切な認証ハンドラを呼び出す。
  • **認証状態管理**: GSSAPI/SSPI/SASL のコンテキスト保持および継続処理の制御。
  • **セキュリティ検証**: channel_binding や require_auth 設定に基づき、認証要求がポリシーに適合しているか検証する。
  • **パスワード管理**: クライアントサイドでのパスワード暗号化（MD5, SCRAM）およびパスワード変更用 SQL の生成。
  • **プラットフォーム抽象化**: Windows (SSPI) と Unix (GSSAPI) の認証差異の吸収。

重要なシンボル

認証ディスパッチ

  • pg_fe_sendauth: 認証要求を処理するメインのエントリポイント。
  • check_expected_areq: 認証要求が接続設定（require_auth 等）と矛盾しないか検証する。

認証メカニズム（内部ハンドラ）

  • pg_GSS_startup / pg_GSS_continue: GSSAPI 認証の開始と継続。
  • pg_SSPI_startup / pg_SSPI_continue: Windows SSPI 認証の開始と継続。
  • pg_SASL_init / pg_SASL_continue: SASL（SCRAM, OAuth）認証の初期化と継続。
  • pg_password_sendauth: レガシーな MD5/平文パスワード認証の処理。

ユーティリティ・公開 API

  • PQencryptPasswordConn: 指定されたアルゴリズムでパスワードを暗号化する。
  • PQchangePassword: ALTER USER 文を発行してパスワードを変更する。
  • pg_fe_getauthname: 現在の認証ユーザー名を取得する。

制御フローとデータフロー

1. **要求受信**: pg_fe_sendauth がサーバーからの AuthRequest を受け取る。 2. **検証**: check_expected_areq が現在の接続状態（SSL使用有無、require_auth 設定）と要求を照合する。 3. **ディスパッチ**: switch 文で認証タイプを判定。

  • **SASL**: pg_SASL_init でメカニズムを選択し、conn->sasl->exchange を通じて SASL ステートマシンを駆動する。
  • **GSS/SSPI**: pg_GSS_startup 等を呼び出し、conn->gctx や conn->sspictx にコンテキストを保持して継続する。

    • 4. **応答送信**: 各ハンドラは pqPacketSend を使用して、サーバーへ認証応答メッセージを送信する。 5. **非同期対応**: async フラグが設定された場合、呼び出し元は conn->async_auth を通じて認証を継続する。

読者向けメモ

  • **メモリ管理**: 認証コンテキスト（gctx, sspictx）やバッファは PGconn 構造体に紐付いており、PQfinish で解放されます。ハンドラ内で malloc された一時バッファは、各関数内で適切に free される必要があります。
  • **スレッド安全性**: pglock_thread() / pgunlock_thread() が GSSAPI/SSPI 処理の前後で使用されており、ライブラリの並行実行時の競合を防いでいます。
  • **省略された詳細**: 本ファイルは認証の「ディスパッチャ」であるため、各認証アルゴリズム（SCRAM のハッシュ計算など）の数学的詳細は各 fe-auth-*.c に委譲されており、本ファイルには含まれていません。
  • **注意点**: channel_binding=require が設定されている場合、SASL 以外の認証要求は即座にエラーとなります。これはセキュリティ上の重要な不変条件です。
