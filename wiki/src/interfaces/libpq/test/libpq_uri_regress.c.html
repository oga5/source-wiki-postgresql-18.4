src/interfaces/libpq/test/libpq_uri_regress.c

概要

libpq_uri_regress.c は、libpq の接続文字列（conninfo）解析機能、特に URI 形式のサポートを検証するための回帰テスト用プログラムです。

モジュール内での役割

src/interfaces/libpq/test ディレクトリにおいて、libpq の接続オプション解析ロジックが期待通りに動作するかを確認する単体テストツールとして機能します。

責務

  • コマンドライン引数から受け取った接続文字列を PQconninfoParse で解析する。
  • 解析結果と PQconndefaults で取得したデフォルト値を比較する。
  • デフォルト値と異なる設定項目のみを標準出力に表示し、接続タイプ（local/inet）を判定する。

重要なシンボル

  • main: プログラムの起点。引数の解析、デフォルト値との比較、結果の出力を行う。
  • PQconninfoParse: 接続文字列を解析し、PQconninfoOption 配列を生成する。
  • PQconndefaults: libpq がデフォルトで使用する接続オプションのリストを取得する。

制御フローとデータフロー

1. **初期化**: 引数から接続文字列を取得し、PQconninfoParse で構造体配列に変換する。 2. **比較**: PQconndefaults を呼び出し、解析結果とデフォルト値を順次比較する。 3. **出力**:

  • opt->val が存在し、かつデフォルト値と異なる場合に keyword=value 形式で出力する。
  • host や hostaddr の値に基づき、接続先が Unix ドメインソケットか TCP/IP かを判定し、(local) または (inet) を出力する。

    • 4. **終了**: 処理完了後、正常終了コードを返す。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用共通ヘッダー。
  • libpq-fe.h: libpq の公開 API 定義。

関連コード

  • src/interfaces/libpq/fe-connect.c: PQconninfoParse および PQconndefaults の実装が含まれており、本テストの検証対象となるロジックの本体です。
  • src/interfaces/libpq/test/libpq_testclient.c: 同じディレクトリにある別のテスト用クライアントです。

読者向けメモ

  • **実装の前提**: このコードは PQconninfoOption 配列内のキーワード順序が解析結果とデフォルト値で一致していることを前提としています。
  • **プラットフォーム依存**: Unix ドメインソケットの判定に '/' の有無を使用しているため、Windows 環境では意図した動作にならない可能性があります（Windows は Unix ドメインソケットをサポートしていないため、テスト対象外として扱われています）。
  • **用途**: 本プログラムは手動実行または回帰テストスクリプトから呼び出されることを想定しており、ライブラリの一部ではありません。
