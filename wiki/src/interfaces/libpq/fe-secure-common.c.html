src/interfaces/libpq/fe-secure-common.c

概要

本ファイルは、PostgreSQLのクライアントライブラリ（libpq）における、SSL/TLS接続時のサーバー証明書検証ロジックを実装しています。特定のSSLバックエンド（OpenSSL等）に依存しない共通の検証ユーティリティを提供します。

モジュール内での役割

libpq のセキュア通信サブシステムの一部であり、fe-secure.c が提供する高レベルなインターフェースと、fe-secure-openssl.c などの実装固有コードの橋渡しを行います。証明書のホスト名やIPアドレスが接続先と一致するかを判定するロジックを抽象化しています。

責務

  • サーバー証明書内の名前（Common Name / SAN）と接続先ホスト名の照合。
  • 証明書内のIPアドレスと接続先IPアドレスの照合。
  • ワイルドカード証明書（*.example.com 等）の妥当性判定。
  • 検証失敗時のエラーメッセージ生成と PGconn への格納。

重要なシンボル

  • pq_verify_peer_name_matches_certificate: 接続先ホスト名と証明書情報を照合するメインのエントリポイント。
  • pq_verify_peer_name_matches_certificate_name: 証明書内の文字列（DNS名）とホスト名を比較。
  • pq_verify_peer_name_matches_certificate_ip: 証明書内のバイナリIPデータとホストIPを比較。
  • wildcard_certificate_match: ワイルドカードパターンのマッチング規則（RFC 2818準拠の独自実装）を適用。

制御フローとデータフロー

1. **検証開始**: pq_verify_peer_name_matches_certificate が sslmode=verify-full を確認し、検証処理を開始します。 2. **バックエンド呼び出し**: pgtls_verify_peer_name_matches_certificate_guts を経由して、各SSLライブラリから抽出された証明書情報を取得します。 3. **照合**:

  • 文字列比較の場合、pq_verify_peer_name_matches_certificate_name がワイルドカード判定を含めて照合します。
  • IPアドレスの場合、pq_verify_peer_name_matches_certificate_ip がネットワークバイトオーダーで比較を行います。

    • 4. **結果処理**: 一致しない場合、libpq_append_conn_error を通じて PGconn に詳細なエラー情報を追加し、呼び出し元へ結果を返します。

直接依存

  • postgres_fe.h: libpq用共通ヘッダー。
  • fe-secure-common.h: 本ファイルの公開インターフェース定義。
  • libpq-int.h: PGconn 構造体およびエラーハンドリング用マクロ。
  • pqexpbuffer.h: エラーメッセージ構築用バッファ操作。
  • <arpa/inet.h>: IPアドレス変換用。
  • arpa/inet.h

関連コード

  • fe-secure.c: セキュア通信のメインインターフェース。
  • fe-secure-openssl.c: OpenSSLを用いた具体的な証明書抽出ロジック。
  • src/include/libpq/pqcomm.h: 通信プロトコル関連の定義。

読者向けメモ

  • **ワイルドカードの制限**: 本実装のワイルドカードマッチングは、RFC 2818に基づき、* は単一のパス名コンポーネントのみにマッチし、ドットを跨ぐことはできません。
  • **セキュリティ**: CVE-2009-4034対策として、証明書内の名前に埋め込まれたNULLバイトを明示的に拒否しています。
  • **拡張性**: 新しいSSLバックエンドを追加する場合、この共通ロジックを再利用することで、検証ロジックの一貫性を保つことが推奨されます。
