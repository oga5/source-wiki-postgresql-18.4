src/interfaces/libpq/libpq-fe.h

概要

libpq-fe.h は、PostgreSQL のフロントエンドアプリケーション（クライアント）がデータベースサーバーと通信するために必要な、公開 API 定義およびデータ構造を網羅したヘッダーファイルです。

モジュール内での役割

libpq モジュールのパブリックインターフェースとして機能します。アプリケーション開発者はこのファイルをインクルードすることで、接続管理、クエリ実行、結果取得、認証フックなどの全機能にアクセスできます。

責務

  • クライアントアプリケーション向けの公開関数プロトタイプの提供。
  • 接続状態 (ConnStatusType) や実行結果 (ExecStatusType) などの列挙型の定義。
  • 不透明ポインタ（PGconn, PGresult 等）による内部実装の隠蔽。
  • コンパイル時の機能判定用マクロ（LIBPQ_HAS_*）の提供。

重要なシンボル

  • **PGconn**: データベース接続を管理する不透明な構造体。
  • **PGresult**: クエリ実行結果を保持する構造体。
  • **ConnStatusType**: 接続の現在の状態を示す列挙型。
  • **ExecStatusType**: クエリ実行結果のステータス（成功、失敗、コピー中など）。
  • **PQconnectdb / PQexec**: 同期的な接続およびクエリ実行の主要関数。
  • **PQsendQuery / PQgetResult**: 非同期クエリ実行のためのインターフェース。

制御フローとデータフロー

1. **接続**: PQconnectdb 等で PGconn を生成し、PQstatus で接続を確認します。 2. **実行**: PQexec または PQsendQuery を通じて SQL を送信します。 3. **結果処理**: PGresult を受け取り、PQresultStatus で判定後、PQgetvalue 等でデータを抽出します。 4. **終了**: PQclear で結果を解放し、PQfinish で接続を閉じます。

直接依存

  • stdint.h: 固定幅整数型の利用。
  • stdio.h: FILE ポインタ（トレース機能等）の利用。
  • postgres_ext.h: Oid などのバックエンド共通型の定義。

関連コード

  • libpq-int.h: libpq 内部でのみ使用される構造体定義やプライベート関数が含まれます。
  • fe-connect.c, fe-exec.c: このヘッダーで宣言されている主要関数の実装が含まれます。
  • pqexpbuffer.h: クエリ構築等で利用される動的バッファ管理。

読者向けメモ

  • **不透明ポインタ**: PGconn や PGresult のメンバに直接アクセスしてはいけません。必ず提供されているアクセサ関数（PQdb(), PQntuples() 等）を使用してください。
  • **後方互換性**: 列挙型の値は追加される可能性がありますが、既存の定数の変更や削除は行われません。
  • **スレッドセーフ**: libpq は基本的にスレッドセーフですが、一部の古い関数や特定の環境下では PQregisterThreadLock による排他制御が必要になる場合があります。
  • **機能判定**: 新しい機能を使用する際は、LIBPQ_HAS_* マクロを確認することで、リンク先のライブラリがその機能をサポートしているかを判定できます。
