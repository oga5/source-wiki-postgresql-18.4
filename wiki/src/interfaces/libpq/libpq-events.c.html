src/interfaces/libpq/libpq-events.c

概要

libpq の「イベント API」を実装するソースファイルです。この API を使用することで、アプリケーションは PGconn や PGresult のライフサイクルイベント（作成、破棄、リセットなど）をフックし、独自のデータ（インスタンスデータ）を関連付けることができます。

モジュール内での役割

libpq の拡張ポイントを提供します。標準的な接続やクエリ実行のフローに介入し、外部ライブラリやアプリケーションが PGconn や PGresult に紐づくリソース管理を行うための基盤となります。

責務

  • イベントプロシージャ（コールバック）の登録管理。
  • PGconn および PGresult へのインスタンスデータの保存と取得。
  • ライフサイクルイベントのトリガー（PGEVT_REGISTER 等の通知）。
  • イベント登録時のメモリ動的確保とクリーンアップ。

重要なシンボル

  • PQregisterEventProc: イベント処理関数を登録し、PGEVT_REGISTER イベントを発火します。
  • PQsetInstanceData / PQinstanceData: PGconn に紐づく任意のデータを管理します。
  • PQresultSetInstanceData / PQresultInstanceData: PGresult に紐づく任意のデータを管理します。
  • PQfireResultCreateEvents: 手動作成された PGresult に対してイベントを通知します。

制御フローとデータフロー

1. **登録**: PQregisterEventProc が呼ばれると、PGconn 内の events 配列にプロシージャが追加され、即座に PGEVT_REGISTER が通知されます。 2. **データ保持**: PQsetInstanceData は、登録済みプロシージャのアドレスをキーとして、PGconn または PGresult 構造体内の events 配列から該当エントリを検索し、data ポインタを更新します。 3. **イベント発火**: libpq 内部の各処理（接続破棄や結果作成など）において、登録されたプロシージャが順次呼び出され、イベント ID と関連情報が渡されます。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド用共通定義。
  • libpq-fe.h: 公開 API 定義。
  • libpq-int.h: PGconn や PGresult の内部構造体定義。

関連コード

  • libpq-events.h: イベント ID (PGEventId) や構造体定義。
  • fe-connect.c / fe-exec.c: 実際に PGEVT_CONNDESTROY や PGEVT_RESULTCREATE などのイベントをトリガーする箇所。

読者向けメモ

  • **識別子としての関数ポインタ**: イベントの識別には PGEventProc のアドレスが使用されます。そのため、同一プロシージャを複数回登録することはできません。
  • **メモリ管理**: PQregisterEventProc は name 引数を内部で strdup します。呼び出し側は元の文字列を保持し続ける必要はありません。
  • **拡張性**: この API は、主に libpq を拡張するライブラリ（例: GSSAPI や特定の認証プラグイン）が、接続や結果セットの生存期間に合わせてリソースをクリーンアップするために利用されます。
