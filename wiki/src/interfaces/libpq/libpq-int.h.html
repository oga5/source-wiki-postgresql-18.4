src/interfaces/libpq/libpq-int.h

概要

libpq-int.h は、PostgreSQL クライアントライブラリ（libpq）の内部的なデータ構造、型定義、およびモジュール間共有関数を定義するプライベートヘッダーです。公開 API である libpq-fe.h とは異なり、アプリケーションからの直接利用は推奨されません。

モジュール内での役割

libpq の中核となるデータ構造（PGconn, PGresult）の定義場所であり、fe-*.c ファイル群が共通して参照する「libpq の設計図」として機能します。

責務

  • **接続状態の保持**: PGconn 構造体を通じた、接続パラメータ、ソケット状態、認証情報、SSL/GSSAPI コンテキストの管理。
  • **クエリ結果の表現**: PGresult 構造体による、バックエンドからの応答データ、メタデータ、エラー情報の保持。
  • **内部インターフェースの公開**: fe-connect.c や fe-exec.c など、各ソースファイル間で共有される内部関数（pq プレフィックス）のプロトタイプ宣言。
  • **プラットフォーム抽象化**: Windows (Win32/SSPI) と Unix 間のソケット処理やスレッドロックの差異を吸収するマクロの提供。

重要なシンボル

  • PGconn: データベース接続の全状態を保持するメイン構造体。
  • PGresult: クエリ実行結果を保持する構造体。
  • PGAsyncStatusType: クエリ実行のステートマシン状態（IDLE, BUSY, COPY_IN 等）。
  • pq* プレフィックス関数: pqReadData, pqFlush, pqParseInput3 など、libpq 内部でのみ使用されるユーティリティ関数群。

制御フローとデータフロー

  • **データフロー**: バックエンドからのデータは inBuffer に読み込まれ、pqParseInput3 等のプロトコル解析関数によって PGresult 構造体へ変換されます。
  • **制御フロー**: PGconn 内の asyncStatus がクエリの進行を管理し、cmd_queue がパイプラインモードでのコマンド実行順序を制御します。

読者向けメモ

  • **非公開 API**: 本ファイルで定義されているシンボルは、PostgreSQL のバージョンアップに伴い予告なく変更される可能性があります。アプリケーション開発者は必ず libpq-fe.h を使用してください。
  • **メモリ管理**: PGresult は独自のメモリ管理（curBlock 等）を行っており、pqResultAlloc などの内部関数がそのライフサイクルを制御しています。
  • **移植性**: Windows 環境特有の処理（WIN32 マクロ）が随所に含まれているため、プラットフォーム依存の挙動を追う際は注意が必要です。
