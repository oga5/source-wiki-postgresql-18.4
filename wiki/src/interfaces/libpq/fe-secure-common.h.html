src/interfaces/libpq/fe-secure-common.h

概要

本ヘッダーファイルは、libpq における SSL/TLS 接続時のサーバー証明書検証に関する共通ロジックを定義しています。特定の SSL ライブラリ（OpenSSL 等）に依存しない、実装非依存の検証ユーティリティを提供します。

モジュール内での役割

libpq のセキュア通信サブシステムにおいて、fe-secure.c が外部インターフェースを担うのに対し、本ファイルと対になる fe-secure-common.c は、各 SSL 実装（fe-secure-openssl.c など）から呼び出される共通の検証ロジックを提供します。

責務

  • サーバー証明書内の名前（Common Name や SAN）と接続先ホスト名の照合。
  • ワイルドカード証明書の検証ルールの適用。
  • 証明書内の IP アドレスと接続先 IP アドレスの照合。
  • 検証失敗時の PGconn へのエラーメッセージ設定。

重要なシンボル

  • pq_verify_peer_name_matches_certificate_name: 証明書内の文字列（DNS名）とホスト名の照合を行います。
  • pq_verify_peer_name_matches_certificate_ip: 証明書内のバイナリ形式の IP アドレスと接続先 IP の照合を行います。
  • pq_verify_peer_name_matches_certificate: 接続済み PGconn を用いて、証明書全体に対する検証プロセスをトリガーします。

制御フローとデータフロー

1. SSL ハンドシェイク後、各 SSL 実装ファイルが証明書から名前や IP 情報を抽出します。 2. 抽出されたデータは、本モジュールの関数（_name または _ip）に渡されます。 3. これらの関数は、PGconn 内の接続先情報と比較し、一致すれば 1、不一致なら 0、エラー時は -1 を返します。 4. 検証結果に基づき、libpq は接続を継続するか、エラーを報告して切断します。

直接依存

  • libpq-fe.h: PGconn 構造体の定義および基本的な libpq インターフェースを提供します。

関連コード

  • src/interfaces/libpq/fe-secure-common.c: 本ヘッダーで宣言された関数の実装が含まれます。
  • src/interfaces/libpq/fe-secure-openssl.c: 本モジュールの関数を利用して OpenSSL 固有の証明書検証を行う実装例です。
  • src/interfaces/libpq/fe-secure.c: libpq のセキュア通信のメインエントリーポイントです。

読者向けメモ

  • 本モジュールは RFC 2818 に準拠した検証を行いますが、ワイルドカードの扱いについてはブラウザの実装と一部異なる独自ルール（ドットの扱いなど）があるため、実装詳細を確認する際は fe-secure-common.c 内の wildcard_certificate_match を参照してください。
  • 証明書から抽出された名前には NULL 文字が含まれる可能性があるため、セキュリティ対策として埋め込み NULL チェックが実装されています。
