src/interfaces/libpq/libpq-events.h

概要

libpq-events.h は、libpq の「イベント API」を定義するヘッダーファイルです。この API を使用することで、アプリケーションは PGconn や PGresult のライフサイクルイベント（接続の破棄や結果の生成など）をフックし、独自のデータや処理を関連付けることができます。

モジュール内での役割

libpq モジュールにおける拡張ポイントを提供します。通常の libpq 利用者には不要ですが、ライブラリの挙動をカスタマイズしたり、特定の接続や結果セットに関連する状態を管理したい高度なアプリケーション向けに設計されています。

責務

  • イベントの種類（PGEventId）の定義。
  • イベント発生時に渡される構造体（PGEventRegister 等）の定義。
  • イベントハンドラ（PGEventProc）の型定義。
  • 接続および結果セットに対するイベント登録と、インスタンスデータの管理（設定・取得）用関数の宣言。

重要なシンボル

  • PGEventId: イベントの種類を識別する列挙型。PGEVT_CONNDESTROY や PGEVT_RESULTDESTROY など、リソース解放のタイミングを捉えるために重要です。
  • PGEventProc: イベントハンドラの関数ポインタ型。evtId に応じて適切な処理を分岐させます。
  • PQregisterEventProc: 特定の PGconn に対してイベントハンドラを登録します。
  • PQinstanceData / PQresultInstanceData: 登録したハンドラに関連付けられたカスタムデータを取得・設定します。

制御フローとデータフロー

1. **登録**: アプリケーションが PQregisterEventProc を呼び出し、ハンドラを登録します。この際、PGEVT_REGISTER イベントが即座に発行されます。 2. **イベント発生**: libpq 内部で接続の切断や結果の生成が発生すると、登録された各 PGEventProc が呼び出されます。 3. **データ管理**: ハンドラ内では PQsetInstanceData を使用して、その接続や結果セットに紐づくメモリ領域や状態を保持します。 4. **クリーンアップ**: PGEVT_CONNDESTROY や PGEVT_RESULTDESTROY イベントを受け取った際に、アプリケーションは確保したメモリを解放する責任を負います。

直接依存

  • libpq-fe.h: PGconn および PGresult 構造体の定義を参照するために必須です。

関連コード

  • src/interfaces/libpq/libpq-events.c: 本ヘッダーで宣言された関数の実装が含まれています。
  • src/interfaces/libpq/libpq-int.h: PGconn や PGresult の内部構造（イベント配列の保持方法など）が定義されており、イベント API がどのように内部で管理されているかを理解するのに役立ちます。

読者向けメモ

  • **メモリ管理の注意**: イベント API を使用する場合、PGEVT_RESULTDESTROY などの破棄イベントで必ずリソースを解放してください。そうしないとメモリリークの原因となります。
  • **一意性**: PGEventProc は関数ポインタのアドレスで識別されるため、同一の関数を複数回登録することはできません。
  • **デバッグ**: PQregisterEventProc の name 引数はデバッグ時に役立つため、識別しやすい名前を付けることを推奨します。
