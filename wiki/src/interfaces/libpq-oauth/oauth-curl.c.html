src/interfaces/libpq-oauth/oauth-curl.c

概要

本ファイルは、PostgreSQL クライアントライブラリ libpq における OAuth 2.0 Device Authorization Grant (RFC 8628) の実装の中核を担います。libcurl を利用した非同期 HTTP 通信を libpq の接続ポーリングループに統合し、Discovery からトークン取得までのステートマシンを管理します。

モジュール内での役割

libpq-oauth モジュールにおける「通信エンジン」です。oauth-utils.c が提供する libpq とのシム層を介して、接続状態やエラー情報を PGconn に反映させます。libcurl のマルチハンドルをラップし、OS 固有の I/O 多重化プリミティブ（epoll または kqueue）を用いて、非同期かつ非ブロッキングな認証フローを実現します。

責務

  • **OAuth ステートマシン管理**: OAuthStep に基づき、Discovery、デバイス認可、トークンポーリングの各フェーズを遷移。
  • **非同期 I/O 統合**: libcurl のソケットイベントとタイマーイベントを epoll/kqueue に集約し、libpq のポーリングループと同期。
  • **JSON 解析**: common/jsonapi.h を利用し、認可サーバーからのレスポンスを型安全にパース。
  • **エラーハンドリング**: 通信エラー、プロトコルエラー、JSON 解析エラーを PQExpBuffer に集約し、libpq のエラーメッセージとして出力。

重要なシンボル

ステートマシン管理

  • enum OAuthStep: 認証フローの現在地（INIT, DISCOVERY, DEVICE_AUTHORIZATION, TOKEN_REQUEST, WAIT_INTERVAL）。
  • struct async_ctx: 認証フローの全コンテキスト。CURLM ハンドル、ソケット多重化用ディスクリプタ、解析済みデータ構造を保持。

データ構造と解放

  • struct provider, struct device_authz, struct token: 認可サーバーのメタデータおよび OAuth レスポンスを保持。各構造体に対応する free_* 関数がメモリ管理を担う。

公開 API

  • pg_fe_run_oauth_flow(): 認証フローのメインエントリポイント。PGconn を引数に取り、ポーリング状態を返す。
  • pg_fe_cleanup_oauth_flow(): 認証完了またはエラー時に、async_ctx と関連リソースを解放。

制御フローとデータフロー

1. **初期化**: pg_fe_run_oauth_flow が initialize_curl を呼び出し、libcurl をスレッドセーフに初期化。 2. **駆動**: pg_fe_run_oauth_flow_impl が async_ctx->step に基づき、start_* 関数でリクエストをキューイングし、drive_request で libcurl のマルチハンドルをポンプする。 3. **イベントループ**: register_socket / register_timer が libcurl からの要求を epoll/kqueue に反映。comb_multiplexer がイベントの整合性を維持。 4. **データフロー**: append_data がレスポンスを async_ctx->work_data に蓄積し、parse_oauth_json が定義済みの json_field 配列に従って構造体へマッピングする。

読者向けメモ

  • **メモリ所有権**: async_ctx は PGconn の sasl_state に紐付き、pg_fe_cleanup_oauth_flow によって所有権が解放されます。
  • **エラーパス**: actx_error マクロでエラーをバッファリングし、PGRES_POLLING_FAILED を返すことで libpq にエラーを伝播させます。
  • **エッジケース**: slow_down レスポンス受信時のポーリング間隔の動的延長や、kqueue 使用時のレベルトリガー維持のための comb_multiplexer 実装など、プラットフォーム依存の複雑な制御が含まれます。
  • **省略箇所**: 本ファイルは libcurl の詳細なオプション設定や JSON 解析の再帰的な深さ制限など、RFC 準拠のための細かいバリデーションを多く含みます。これらはコードの堅牢性を高めるために意図的に詳細に記述されています。
