src/interfaces/libpq-oauth/oauth-utils.c

概要

本ファイルは、libpq-oauth モジュールが libpq の内部機能へ安全にアクセスするための「接着剤（Glue）」となるユーティリティ群を提供します。libpq の内部ヘッダー（libpq-int.h）に直接依存することなく、必要な機能やデータ構造へのアクセスを可能にします。

モジュール内での役割

libpq-oauth は動的ライブラリとしてロードされることを前提としており、ホスト側の libpq との疎結合を維持する必要があります。本ファイルは、初期化時に libpq から渡される関数ポインタを保持し、それらを介して PGconn 構造体の操作やエラー処理を代行します。

責務

  • **API 注入**: libpq_oauth_init を通じて、libpq 内部のアクセサおよびミューテータを登録する。
  • **内部機能の模倣**: libpq が提供するエラーログ出力やシグナル制御（SIGPIPE 対策）などの内部 API を再実装し、モジュール内で利用可能にする。
  • **環境依存の抽象化**: プラットフォームごとのソケットエラー処理や NLS（国際化）対応の差異を吸収する。

重要なシンボル

  • libpq_oauth_init: モジュール起動時に呼び出され、libpq 側の関数ポインタを本モジュールへ注入する。
  • libpq_append_conn_error: libpq のエラーバッファへ翻訳済みのエラーメッセージを安全に追加する。
  • pq_block_sigpipe / pq_reset_sigpipe: ネットワーク通信中の SIGPIPE シグナルを一時的にブロック・復元するユーティリティ。
  • pg_g_threadlock: スレッドセーフではない環境での libcurl 初期化を保護するためのグローバルロック関数。

追加の索引済みシンボル

  • libpq_gettext
  • oauth_unsafe_debugging_enabled

制御フローとデータフロー

1. **初期化**: libpq が libpq_oauth_init を呼び出し、関数ポインタ（conn_errorMessage 等）を静的変数に格納します。 2. **エラー処理**: libpq_append_conn_error が呼ばれると、登録された conn_errorMessage を介して PGconn のバッファを取得し、libpq_gettext で翻訳した文字列を書き込みます。 3. **シグナル制御**: ネットワーク操作の前後で pq_block_sigpipe を呼び出し、現在のシグナルマスクを保存した上で SIGPIPE をブロックします。操作完了後に pq_reset_sigpipe で元の状態へ戻します。

直接依存

  • postgres_fe.h: PostgreSQL フロントエンド開発用の基本定義。
  • signal.h: シグナル処理（sigset_t, pthread_sigmask 等）に使用。
  • oauth-utils.h: 本モジュールで定義される関数プロトタイプと型定義。

関連コード

  • src/interfaces/libpq-oauth/oauth-curl.c: 本ファイルで提供されるユーティリティを利用して、実際の OAuth 認証フロー（HTTP 通信）を実行する。
  • src/interfaces/libpq-oauth/oauth-utils.h: 関数ポインタの型定義や、libpq から注入されるインターフェースの宣言。

読者向けメモ

  • 本ファイルは USE_DYNAMIC_OAUTH が定義されている環境でのみ動作します。静的ビルドには対応していません。
  • libpq-int.h への依存は厳禁です。内部構造にアクセスする必要がある場合は、必ず oauth-utils.h で定義されたアクセサ関数を経由してください。
  • libpq_gettext は、NLS が有効な場合は libpq の実装を呼び出し、無効な場合は引数をそのまま返すラッパーとして機能します。
