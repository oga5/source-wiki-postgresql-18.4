src/interfaces/libpq-oauth/oauth-curl.h

概要

oauth-curl.h は、PostgreSQL の libpq における OAuth 2.0 Device Authorization Grant (RFC 8628) 実装の公開インターフェースを定義するヘッダーファイルです。libpq の認証プロセスに OAuth/OIDC フローを統合するためのエントリポイントを提供します。

モジュール内での役割

src/interfaces/libpq-oauth モジュールにおいて、外部（libpq 本体）と内部（oauth-curl.c 等のロジック）を繋ぐゲートウェイの役割を果たします。libpq が認証フェーズで OAuth を必要とする際、このヘッダーで宣言された関数を呼び出すことで、非同期な認証フローを開始・管理します。

責務

  • OAuth 認証フローの実行およびクリーンアップのための公開 API の提供。
  • libpq-fe.h を介した、PGconn オブジェクトとのインターフェース定義。
  • 動的リンクまたは静的リンク環境下での、認証コールバックのシグネチャの固定。

重要なシンボル

  • pg_fe_run_oauth_flow(PGconn *conn): OAuth 認証フローを駆動するメイン関数。PostgresPollingStatusType を返し、非同期 I/O をサポートします。
  • pg_fe_cleanup_oauth_flow(PGconn *conn): 認証完了後、またはエラー発生時に、関連するリソース（curl ハンドルやメモリ）を解放するためのクリーンアップ関数。

制御フローとデータフロー

本ファイル自体は宣言のみですが、呼び出し元は pg_fe_run_oauth_flow をポーリングループ内で呼び出します。 1. **開始**: libpq が認証要求を受け取ると pg_fe_run_oauth_flow を呼び出す。 2. **実行**: oauth-curl.c が libcurl を使用して認可サーバーと通信し、デバイスコードの取得やトークンポーリングを行う。 3. **終了**: 認証成功または失敗時に pg_fe_cleanup_oauth_flow を呼び出し、接続状態をクリーンにする。

直接依存

  • libpq-fe.h: PostgreSQL クライアントライブラリの基本構造体（PGconn）および型定義（PostgresPollingStatusType）に依存しています。

関連コード

  • src/interfaces/libpq-oauth/oauth-curl.c: 本ヘッダーで宣言された関数の実装本体。
  • src/interfaces/libpq-oauth/oauth-utils.h: 認証フローで使用されるユーティリティ関数やデータ構造の定義。
  • fe-auth-oauth.h: libpq 内部で OAuth 認証を統合するための定義。

読者向けメモ

  • 本ファイルは公開 API を定義しているため、変更は libpq の ABI 互換性に影響を与える可能性があります。
  • USE_DYNAMIC_OAUTH マクロの有無により、libpq-int.h への依存関係が切り替わる設計になっています。実装を読む際は、ビルド構成がどちらであるかを意識してください。
  • 非同期処理を前提としているため、pg_fe_run_oauth_flow の戻り値が PGRES_POLLING_READING や PGRES_POLLING_WRITING を返す場合のハンドリングが重要です。
