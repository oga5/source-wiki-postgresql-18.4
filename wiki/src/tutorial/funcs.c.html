src/tutorial/funcs.c

概要

PostgreSQL の拡張機能としてロード可能な、ユーザー定義関数（UDF）のサンプル実装集です。SQL レベルの関数と C 言語の関数をバインドするための標準的なインターフェースを提供します。

モジュール内での役割

src/tutorial ディレクトリにおいて、PostgreSQL のデータ型（数値、テキスト、複合型）を C 言語でどのように操作するかを示すリファレンス実装の役割を担います。

責務

  • **データ型の変換**: SQL の型と C の内部表現（Datum）間の相互変換。
  • **メモリ管理**: palloc を使用した動的メモリ確保と、PostgreSQL のメモリコンテキストへの準拠。
  • **可変長データの操作**: text 型などの可変長データ（TOAST 可能なデータ）のヘッダー処理とコピー。
  • **複合型の解析**: HeapTupleHeader を介した行データ内の属性値抽出。

重要なシンボル

  • PG_MODULE_MAGIC: 拡張機能の互換性チェック用マクロ。
  • PG_FUNCTION_INFO_V1(funcname): 関数呼び出し規約（V1）を宣言する必須マクロ。
  • PG_GETARG_... / PG_RETURN_...: 引数取得および戻り値設定のための標準マクロ群。
  • Datum: PostgreSQL の汎用データ型表現。

制御フローとデータフロー

1. **呼び出し**: SQL から関数が呼ばれると、PG_FUNCTION_ARGS を通じて引数が渡されます。 2. **抽出**: PG_GETARG_* マクロで Datum を C の型に変換します。 3. **処理**:

  • 値渡し（int32 等）は直接演算。
  • 参照渡し（text 等）は palloc でメモリを確保し、memcpy でデータを構築。
  • 複合型は GetAttributeByName でフィールド値を抽出。

    • 4. **返却**: PG_RETURN_* マクロで結果を Datum に変換して呼び出し元へ戻します。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • executor/executor.h: GetAttributeByName 等の実行エンジン機能。
  • utils/fmgrprotos.h: text_starts_with 等の組み込み関数プロトタイプ。
  • utils/geo_decls.h: Point 型の定義。

関連コード

  • src/tutorial/complex.c: ユーザー定義の複合型（Complex）の定義と操作。
  • src/include/fmgr.h: 関数マネージャのインターフェース定義。

読者向けメモ

  • **メモリ管理**: palloc で確保したメモリは、現在のメモリコンテキストが終了すると自動的に解放されます。手動の free は不要です。
  • **可変長データ**: text 型を扱う際は、VARSIZE_ANY_EXHDR や VARDATA_ANY を使用して、圧縮やショートヘッダーの有無を意識した安全なアクセスが推奨されます。
  • **V1 規約**: 本ファイルは PostgreSQL 7.3 以降の V1 呼び出し規約に従っています。古い規約（V0）との混同に注意してください。
