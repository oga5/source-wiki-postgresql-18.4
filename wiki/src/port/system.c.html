src/port/system.c

概要

Windows環境（MinGW/MSVC等）における system() および popen() 関数のラッパー実装です。Windowsのコマンドプロセッサ（cmd.exe）の特殊な引用符（クォート）処理ルールに対応し、コマンド実行時の引数解釈エラーを回避するために提供されています。

モジュール内での役割

src/port はPostgreSQLの移植性レイヤーを担います。本ファイルは、POSIX準拠の環境では標準関数がそのまま利用できる一方で、Windows特有のコマンドライン解析の癖を吸収し、クロスプラットフォームな動作を保証する役割を持ちます。

責務

  • **コマンド文字列の正規化**: 実行するコマンド文字列全体を二重引用符（"）で囲むことで、cmd.exe が引数を誤って解釈するのを防ぐ。
  • **標準関数のオーバーライド**: system および popen をマクロで無効化し、安全なラッパー関数経由での実行を強制する。

重要なシンボル

  • pgwin32_system: 標準の system() の代替。コマンド文字列を引用符で囲んでから実行します。
  • pgwin32_popen: 標準の popen() の代替。同様に引用符で囲んだコマンドを _popen に渡します。

制御フローとデータフロー

1. **入力**: 実行したいコマンド文字列を受け取ります。 2. **メモリ確保**: 元の文字列長 + 2バイト（引用符用）+ 1バイト（終端文字）のバッファを確保します。 3. **文字列変換**: バッファの先頭と末尾に " を挿入し、中央に元のコマンドをコピーします。 4. **実行**: 変換後の文字列を標準の system() または _popen() に渡します。 5. **後処理**: 実行結果を保持しつつ、確保したバッファを解放し、元の errno を復元して呼び出し元へ返します。

直接依存

  • postgres.h / postgres_fe.h: PostgreSQLの基本定義およびフロントエンド用定義。
  • fcntl.h: ファイル制御関連の標準ヘッダー。

関連コード

  • src/port/win32common.c: Windows環境における他の移植性サポートコード。
  • src/include/c.h: PostgreSQLの移植性に関するマクロ定義（system や popen の再定義箇所）。

読者向けメモ

  • 本ファイルは WIN32 かつ __CYGWIN__ ではない環境でのみ有効です。
  • cmd.exe の引用符ルールは非常に複雑です。この実装は「文字列全体を引用符で囲む」という最も安全なアプローチをとっていますが、既に引用符が含まれる複雑なコマンドを渡す場合は、呼び出し側で適切にエスケープされている必要があります。
  • malloc が失敗した場合は errno を ENOMEM に設定して即座にエラーを返します。
