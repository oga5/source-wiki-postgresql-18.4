src/port/getopt_long.c

概要

POSIX 標準の getopt に加え、GNU スタイルのロングオプション（--option）の解析をサポートする移植用実装です。プラットフォーム側で getopt_long が提供されていない環境において、PostgreSQL のコマンドライン引数解析を統一するために使用されます。

モジュール内での役割

src/port ディレクトリの一部として、OS 間の差異を吸収する役割を担います。主にフロントエンドツールやユーティリティが、標準的な引数解析インターフェースを利用できるようにします。

責務

  • ロングオプション（--name）およびショートオプション（-n）の混在解析。
  • 引数を持たないオプション、必須引数、任意引数のハンドリング。
  • 非オプション引数の自動的な末尾への再配置（argv の書き換え）。
  • 解析終了時の状態リセット（optind 等の管理）。

重要なシンボル

  • getopt_long(): 本ファイルのメイン関数。引数ベクトルを走査し、オプションを識別します。
  • struct option: ロングオプションの定義構造体（getopt_long.h で定義）。
  • optarg, optind, opterr, optopt: getopt 系の標準的なグローバル変数。

追加の索引済みシンボル

  • getopt_long

制御フローとデータフロー

1. **初期化と走査**: place ポインタを用いて現在の引数文字列を追跡します。 2. **非オプションの再配置**: 引数が - で始まらない場合、それを argv の末尾へ移動させ、nonopt_start を更新します。これにより、すべてのオプションが解析された後に非オプション引数が並ぶようになります。 3. **ロングオプション解析**: -- で始まる引数を検知すると、longopts 配列と照合します。一致した場合、引数の有無を確認し、必要に応じて optarg を設定します。 4. **ショートオプション解析**: - で始まる単一文字を optstring と照合し、同様に引数を処理します。 5. **終了処理**: 全ての引数を処理し終えるか、非オプション引数のみになった時点で -1 を返します。

直接依存

  • c.h: PostgreSQL の共通ヘッダー。
  • getopt_long.h: struct option 定義および関数プロトタイプ。

関連コード

  • src/port/getopt.c: getopt_long の基盤となるショートオプション解析の実装。
  • src/include/getopt_long.h: 本関数のインターフェース定義。

読者向けメモ

  • **argv の書き換え**: この実装は argv のポインタ順序を動的に変更します。呼び出し元は、この関数が argv を破壊的に操作することを前提にする必要があります。
  • **再入可能性**: static 変数（place, nonopt_start 等）を使用しているため、スレッドセーフではありません。
  • **エラーハンドリング**: optstring の先頭が : である場合、エラーメッセージを出力せずに BADARG を返すなど、標準的な getopt の挙動に準拠しています。
