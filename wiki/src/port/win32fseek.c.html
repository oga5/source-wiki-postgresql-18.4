src/port/win32fseek.c

概要

Windows環境（特にMSVCコンパイラ）における fseeko および ftello の代替実装を提供します。標準のファイル操作関数がパイプやデバイスに対して不適切な挙動を示す問題を回避し、PostgreSQLの移植性を向上させます。

モジュール内での役割

src/port ディレクトリの一部として、Windows固有のプラットフォーム差異を吸収する役割を担います。主にバックエンドおよびフロントエンドのファイルI/O操作において、64ビットオフセットを安全に扱うためのラッパーとして機能します。

責務

  • **ファイル種別の判定**: _get_osfhandle を介して取得したハンドルから、対象がディスクか、パイプやキャラクタデバイスかを判別します。
  • **安全なシーク操作**: ディスクファイルに対しては _fseeki64 / _ftelli64 を呼び出し、それ以外（パイプ等）に対しては適切なエラーコード（ESPIPE）を設定して失敗させることで、未定義動作を防止します。

重要なシンボル

  • _pgfseeko64: fseeko の代替。ファイル種別を確認した上で、ディスクであれば64ビットオフセットでのシークを実行します。
  • _pgftello64: ftello の代替。現在のファイル位置を64ビット整数で取得します。

制御フローとデータフロー

1. **ハンドル取得**: _fileno と _get_osfhandle を使用して、標準の FILE* から Windows の HANDLE を抽出します。 2. **種別判定**: pgwin32_get_file_type を呼び出し、対象が FILE_TYPE_DISK かどうかを確認します。 3. **分岐**:

  • ディスクの場合: 標準の _fseeki64 / _ftelli64 を呼び出し、結果を返します。
  • パイプ/キャラクタデバイスの場合: errno に ESPIPE をセットし、エラー（-1）を返します。
  • その他: EINVAL をセットし、エラー（-1）を返します。

直接依存

  • postgres_fe.h / postgres.h: 環境に応じたPostgreSQLの基本定義。
  • _MSC_VER: Microsoft Visual C++ コンパイラ環境でのみコンパイルされるようガードされています。

関連コード

  • src/port/win32common.c: pgwin32_get_file_type が定義されている可能性が高い、Windows互換レイヤーの共通関数群。
  • src/include/port/win32.h: Windows環境での型定義やマクロが定義されているヘッダ。

読者向けメモ

  • 本ファイルは _MSC_VER が定義されている環境でのみ有効です。
  • パイプやデバイスに対して fseek を実行しようとすると、標準ライブラリがエラーを返さないケースがあるため、明示的なチェックを行っています。
  • pgoff_t 型が64ビット整数として定義されていることを前提としています。
