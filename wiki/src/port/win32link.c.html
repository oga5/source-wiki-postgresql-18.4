src/port/win32link.c

概要

Windows環境において、POSIX準拠の link() 関数をエミュレートするためのラッパー実装です。

モジュール内での役割

src/port ディレクトリの一部として、PostgreSQLのソースコードがWindowsプラットフォーム上でネイティブなPOSIX APIを欠いている場合に、互換性を提供します。

責務

  • POSIXの link(const char *src, const char *dst) シグネチャをWindowsの CreateHardLinkA APIへ変換する。
  • Windows APIの失敗時に、GetLastError() を取得し、_dosmaperr() を介してPOSIX準拠のエラーコード（errno）へマッピングする。

重要なシンボル

  • link(const char *src, const char *dst): 本ファイルが提供する唯一の公開関数。ハードリンクを作成します。
  • CreateHardLinkA: Windows API。ファイルシステムレベルでハードリンクを作成するために呼び出されます。

追加の索引済みシンボル

  • link

制御フローとデータフロー

1. 呼び出し元から src（既存ファイル）と dst（作成するリンク名）が渡されます。 2. CreateHardLinkA(dst, src, NULL) を実行します。 3. 成功時（戻り値が0以外）は 0 を返します。 4. 失敗時（戻り値が0）は GetLastError() でエラー詳細を取得し、_dosmaperr() で errno を設定した上で -1 を返します。

直接依存

  • c.h: PostgreSQLの標準的なヘッダーであり、プラットフォーム固有の定義や共通マクロを提供します。

関連コード

  • src/port/win32common.c: Windows環境での共通的なエラーハンドリングやユーティリティが含まれる場合があります。
  • src/port/win32stat.c: ファイルシステム関連の互換性レイヤーとして、併せて参照されることが多いファイルです。

読者向けメモ

  • この関数は CreateHardLinkA を使用しているため、ANSI文字列（マルチバイト文字）を扱います。
  • Windowsのハードリンクは、同一ボリューム内でのみ作成可能であるという制約がある点に注意してください。
  • _dosmaperr() は、Windows固有のエラーコードをPOSIXの errno に変換する重要な役割を担っています。
