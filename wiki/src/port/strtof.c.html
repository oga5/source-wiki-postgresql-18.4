src/port/strtof.c

概要

src/port/strtof.c は、プラットフォーム固有の strtof 実装における精度やエラーハンドリングの不備を補完するためのラッパー関数 pg_strtof を提供します。

モジュール内での役割

src/port ディレクトリの一部として、PostgreSQL が移植性を維持するために提供する標準ライブラリ関数の代替実装（またはラッパー）の一つです。

責務

  • プラットフォームの strtof が抱える「二重丸め問題」や不完全なオーバーフロー/アンダーフロー判定を補完する。
  • strtod をフォールバックとして利用し、より正確な浮動小数点変換結果を保証する。
  • 呼び出し元の errno 状態を適切に保持・復元する。

重要なシンボル

  • pg_strtof: 本ファイルの唯一の公開関数。標準の strtof の代わりに使用される。

制御フローとデータフロー

1. **初期化**: 呼び出し元の errno を保存し、errno を 0 にリセットする。 2. **一次変換**: strtof を呼び出し、結果を fresult に格納する。 3. **判定**:

  • エラーが発生した場合はそのまま返す。
  • 解析不能、NaN、または通常の有限値であれば、保存していた errno を復元して返す。

    • 4. **再試行 (フォールバック)**:

  • strtod を使用して double 型で再計算を行う。
  • double の結果と float の結果を比較し、サブノーマル数や無限大の判定を行い、必要に応じて ERANGE を設定する。

直接依存

  • c.h: PostgreSQL の基本定義。
  • <float.h>: FLT_MIN などの浮動小数点定数。
  • <math.h>: isnan, isinf などの数学関数。
  • float.h
  • math.h

関連コード

  • src/port/snprintf.c: 浮動小数点を含む文字列処理に関連する他の移植性レイヤー。
  • strtod(3): 本関数がフォールバックとして利用する標準ライブラリ関数。

読者向けメモ

  • この関数は、Cygwin や Mingw などの特定の環境で発生する浮動小数点変換の挙動不一致を吸収するために存在します。
  • strtof が正しく実装されている環境では、このラッパーはオーバーヘッドとなりますが、移植性の観点から一貫した挙動を強制するために使用されます。
  • errno の操作が複雑であるため、修正を加える際は caller_errno の復元ロジックがすべてのパスで正しく機能しているか注意してください。
