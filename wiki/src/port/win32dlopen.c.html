src/port/win32dlopen.c

概要

Windows環境において、POSIX標準の動的リンクライブラリ操作関数（dlopen, dlsym, dlclose, dlerror）をエミュレートするためのラッパー実装です。

モジュール内での役割

src/port ディレクトリの一部として、PostgreSQLのコアおよび拡張機能がプラットフォーム固有の差異を意識せずに共有ライブラリ（DLL）をロードできるようにする抽象化層を提供します。

責務

  • **POSIX互換インターフェースの提供**: Windows API (LoadLibrary, GetProcAddress, FreeLibrary) をラップし、POSIX準拠のシグネチャで公開します。
  • **エラー管理**: Windows固有のエラーコードを FormatMessage を用いて人間が読める文字列に変換し、dlerror() を通じて提供します。
  • **安全なロード**: DLLロード時のシステムダイアログ（ポップアップ）を抑制し、バックグラウンドでの静かなロードを実現します。

重要なシンボル

  • dlopen: LoadLibrary を呼び出し、DLLをプロセス空間にロードします。
  • dlsym: GetProcAddress を呼び出し、指定したシンボルのアドレスを取得します。
  • dlclose: FreeLibrary を呼び出し、DLLの参照カウントを減らします。
  • dlerror: 直近の操作で発生したエラーメッセージを返します。
  • set_dl_error: 内部ヘルパー関数。Windowsの GetLastError を取得し、静的バッファに格納します。

制御フローとデータフロー

1. **ロード時**: dlopen が SetErrorMode でエラーダイアログを無効化し、LoadLibrary を実行します。成功すればハンドルを返し、失敗すれば set_dl_error でエラーを記録して NULL を返します。 2. **シンボル解決**: dlsym は受け取ったハンドルを使用して GetProcAddress を実行します。 3. **エラー処理**: 各関数は操作失敗時に set_dl_error を呼び出し、静的変数 last_dyn_error を更新します。dlerror はこのバッファの内容を読み取ります。

直接依存

  • c.h: PostgreSQLの基本定義およびプラットフォーム共通のヘッダー。
  • Windows API: windows.h（c.h 経由で間接的に利用される LoadLibrary, GetProcAddress 等）。

関連コード

  • src/include/port.h: これらの関数がPOSIX環境とWindows環境で正しくマッピングされるための宣言が含まれています。
  • src/backend/utils/fmgr/dfmgr.c: 動的ロード機能を利用する主要なコンシューマーです。

読者向けメモ

  • **エラーバッファ**: last_dyn_error は静的変数であるため、スレッドセーフではありません。PostgreSQLの現在の設計では、この制限が許容される範囲で利用されています。
  • **エラー抑制**: SetErrorMode を使用して SEM_FAILCRITICALERRORS を設定しているため、DLLが見つからない場合でもWindowsのシステムエラーダイアログは表示されません。
  • **移植性**: このファイルはWindowsビルド時のみコンパイルされるようにビルドシステムで制御されています。
