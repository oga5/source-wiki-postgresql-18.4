src/port/strlcpy.c

概要

strlcpy.c は、標準ライブラリの strncpy に代わる安全な文字列コピー関数 strlcpy を提供する移植用ソースファイルです。strlcpy が提供されていないプラットフォームにおいて、PostgreSQL が一貫した文字列操作を行えるようにします。

モジュール内での役割

src/port ディレクトリの一部として、OS 間の機能差異を吸収する役割を担います。本ファイルは、OpenBSD 由来のコードをそのまま利用しており、プラットフォーム依存の脆弱性や挙動の不一致を排除するために使用されます。

責務

  • 文字列の安全なコピー：コピー先のバッファサイズ（siz）を超えないようにコピーを行い、常にヌル終端を保証します。
  • 戻り値による切り捨ての通知：コピーしようとした元の文字列の長さを返し、呼び出し元が切り捨ての発生を検知できるようにします。

重要なシンボル

  • strlcpy(char *dst, const char *src, size_t siz): 本ファイルの唯一の公開関数です。strncpy と異なり、バッファが小さい場合に必ずヌル終端を付与し、かつコピー元の全長を返すことで安全性を高めています。

追加の索引済みシンボル

  • strlcpy

制御フローとデータフロー

1. **コピー処理**: siz が 0 でない限り、最大 siz-1 バイトまで src から dst へコピーを試みます。 2. **ヌル終端の保証**:

  • コピー中に src の終端に達した場合は、そのまま終了します。
  • siz バイト分コピーしても終端に達しない場合、dst の末尾にヌル文字を書き込みます。

    • 3. **長さの計算**: 最後に src の残りの長さを走査し、src の全文字数（ヌル文字を含まない）を戻り値として返します。

直接依存

  • c.h: PostgreSQL の基本定義や型定義（size_t 等）を提供します。

関連コード

  • src/port/strlcat.c: strlcpy と対になる、安全な文字列連結関数です。
  • src/port/strnlen.c: 文字列の長さを安全に取得するための補助関数です。

読者向けメモ

  • 本関数は OpenBSD の実装をそのまま採用しています。
  • strncpy とは異なり、戻り値が siz 以上である場合、文字列が切り捨てられたことを意味します。この戻り値を確認することで、バッファオーバーフローやデータ損失を適切にハンドリングすることが推奨されます。
  • siz が 0 の場合、コピーは行われず、単に src の長さのみが返されます。
