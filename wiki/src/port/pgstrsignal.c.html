src/port/pgstrsignal.c

概要

pgstrsignal.c は、Unix シグナル番号に対応する説明文字列を取得するための移植性レイヤーです。プラットフォーム固有の strsignal(3) をラップし、環境による差異を吸収して一貫したインターフェースを提供します。

モジュール内での役割

src/port ディレクトリの一部として、PostgreSQL がサポートする多様なプラットフォーム間でのシステムコールの挙動を標準化する役割を担います。

責務

  • シグナル番号から人間が読める形式の文字列への変換。
  • strsignal(3) が存在しない、あるいは NULL を返す環境に対するフォールバック処理。
  • 呼び出し元に対して、常に有効な文字列ポインタを返すことの保証。

重要なシンボル

  • pg_strsignal(int signum): 本ファイルの唯一の公開関数。シグナル番号を受け取り、その説明文字列を返します。

追加の索引済みシンボル

  • pg_strsignal

制御フローとデータフロー

1. **条件分岐**: コンパイル時に HAVE_STRSIGNAL マクロが定義されているかを確認します。 2. **標準ライブラリ呼び出し**: 定義されている場合、strsignal(signum) を呼び出します。 3. **安全策**: strsignal が NULL を返した場合は、固定文字列 "unrecognized signal" を返します。 4. **フォールバック**: HAVE_STRSIGNAL が未定義の場合、シグナル名が利用不可であることを示す固定文字列を返します。

直接依存

  • c.h: PostgreSQL の基本型定義やプラットフォーム固有の設定（HAVE_STRSIGNAL 等）を含むヘッダー。

関連コード

  • src/port/pqsignal.c: シグナルハンドラの登録など、シグナル関連の他の移植性機能を提供します。
  • strsignal(3): 本関数がラップしている標準 C ライブラリ関数。

読者向けメモ

  • 本関数が返す文字列は、POSIX の仕様上、後続の strsignal 呼び出しによって無効化される可能性があるため、呼び出し直後に使用するかコピーする必要があります。
  • プロジェクトのコーディング規約として、この関数が返す文字列だけでなく、数値のシグナル番号も併せてログ出力することが推奨されています。
  • 過去に存在した sys_siglist[] を利用する複雑な実装は、現代のプラットフォームでは不要と判断され削除されています。
