src/port/pgsleep.c

概要

pgsleep.c は、PostgreSQL におけるプラットフォーム非依存なスリープ（遅延）処理を提供するユーティリティファイルです。マイクロ秒単位での実行停止を可能にします。

モジュール内での役割

src/port ディレクトリの一部として、OS 間の差異を吸収し、フロントエンドツールやバックエンドプロセスが共通のインターフェースでスリープ処理を利用できるようにします。

責務

  • マイクロ秒単位の指定によるプロセスの一時停止。
  • Unix 系 OS における nanosleep のラップ。
  • Windows 環境（フロントエンド）における SleepEx への変換。

重要なシンボル

  • pg_usleep(long microsec): 指定されたマイクロ秒数だけプロセスを停止させるメイン関数。

追加の索引済みシンボル

  • pg_usleep

制御フローとデータフロー

1. **入力検証**: microsec が 0 より大きい場合のみ処理を実行します。 2. **プラットフォーム分岐**:

  • **Unix 系**: microsec を秒とナノ秒に分割し、struct timespec を構築して nanosleep を呼び出します。
  • **Windows (フロントエンド)**: マイクロ秒をミリ秒に換算（端数切り上げ）し、SleepEx を呼び出します。

    • 3. **終了**: 割り込みやカーネルのスケジューリング精度に依存して復帰します。

直接依存

  • c.h: PostgreSQL の基本型定義およびプラットフォーム抽象化マクロ。
  • <time.h>: nanosleep および struct timespec の定義に使用。
  • time.h

関連コード

  • src/backend/port/win32/signal.c: Windows バックエンド環境では、本ファイルの代わりにシグナル対応版のスリープ処理が使用されます。
  • WaitLatch(): バックエンドプロセスにおいて、シグナルやラッチの発生を即座に検知する必要がある場合に推奨される代替手段です。

読者向けメモ

  • **精度の限界**: OS のカーネルティック周期に依存するため、指定した時間よりも長くスリープする可能性があります（最大 20ms 程度の誤差が観測されています）。
  • **バックエンドでの利用注意**: バックエンドプロセスで長時間スリープを行うと、シグナルやラッチへの応答性が低下します。可能な限り WaitLatch() を使用してください。
  • **Windows の挙動**: Windows バックエンドでは本ファイルの実装は使用されません。これは FRONTEND マクロが定義されているか、WIN32 が定義されていない環境でのみコンパイルされるためです。
