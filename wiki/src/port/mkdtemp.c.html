src/port/mkdtemp.c

概要

本ファイルは、プラットフォーム標準の mkdtemp(3) が欠如している環境のために、NetBSD 由来の互換実装を提供するものです。指定されたテンプレート文字列に基づいて、一意な名前を持つディレクトリをモード 0700 で安全に作成します。

モジュール内での役割

src/port モジュールの一部として、PostgreSQL が移植性を維持するために提供する「欠落したシステム関数」の補完機能群の一つです。主にビルド環境やツールチェーンが標準ライブラリで提供しない場合にのみコンパイルされます。

責務

  • **一意なパスの生成**: X で終わるテンプレート文字列を解析し、プロセス ID やカウンタを用いて衝突しないパス名を生成する。
  • **ディレクトリの作成**: mkdir(2) を呼び出し、排他的にディレクトリを作成する。
  • **フォールバック提供**: システムの mkdtemp が利用できない場合に、同等のインターフェースを公開する。

重要なシンボル

  • mkdtemp(char *path): 公開関数。テンプレートパスを受け取り、成功すればパスを返し、失敗すれば NULL を返す。
  • GETTEMP(...): 内部的な補助関数。ファイル作成 (mkstemp) とディレクトリ作成 (mkdtemp) の両方をサポートする汎用的なパス生成・検証ロジック。

追加の索引済みシンボル

  • GETTEMP

制御フローとデータフロー

1. **テンプレート解析**: path 末尾の X の数をカウントし、プロセス ID と静的カウンタ (xtra) を用いて X を文字に置換する。 2. **ディレクトリ検証**: パス内のディレクトリ部分が実際に存在し、かつディレクトリであることを stat(2) で確認する。 3. **作成試行**: mkdir(path, 0700) を実行する。 4. **衝突回避**: EEXIST エラーが発生した場合、テンプレート内の文字をインクリメントして別の名前を生成し、成功するまで再試行する。

読者向けメモ

  • **実装の由来**: このコードは NetBSD の実装をベースにしています。OpenBSD 版の方が DoS 耐性が高いとされていますが、暗号学的依存関係を避けるためにあえて NetBSD 版が採用されています。
  • **条件付きコンパイル**: HAVE_MKDTEMP が定義されている環境では、このファイルの実装は無視されます。
  • **安全性**: mkdtemp は mktemp とは異なり、ディレクトリ作成がアトミックに行われるため、競合状態（Race Condition）に対して安全です。
