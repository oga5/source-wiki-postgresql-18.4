src/port/inet_net_ntop.c

概要

ネットワークアドレス（IPv4/IPv6）を、バイナリ形式から人間が読み取り可能なプレゼンテーション形式（文字列）へ変換するためのユーティリティ関数を提供します。

モジュール内での役割

src/port 内の移植性レイヤーの一部として、システム標準の inet_net_ntop が存在しない、あるいは挙動が異なる環境においても、PostgreSQL が一貫したネットワークアドレス変換を行えるようにします。

責務

  • ネットワークバイトオーダーのバイナリデータから、CIDR 表記を含む文字列への変換。
  • IPv4 および IPv6 アドレスの双方に対応。
  • PostgreSQL 内部の定数（PGSQL_AF_INET 等）とシステム標準の定数の差異を吸収。
  • バッファオーバーフローの防止と適切なエラーコード（EMSGSIZE, EINVAL）の返却。

重要なシンボル

  • pg_inet_net_ntop: 公開インターフェース。アドレスファミリに応じて適切な変換関数へディスパッチします。
  • inet_net_ntop_ipv4: IPv4 用の変換ロジック。
  • inet_net_ntop_ipv6: IPv6 用の変換ロジック。ゼロ圧縮（::）の処理を含みます。
  • decoct: IPv6 内部で IPv4 互換アドレスを扱う際に使用される補助関数。

制御フローとデータフロー

1. **ディスパッチ**: pg_inet_net_ntop が呼び出されると、af 引数に基づき switch 文で IPv4 または IPv6 の処理へ分岐します。 2. **IPv4 変換**: 4オクテットを順次文字列化し、必要に応じて /bits を付与します。 3. **IPv6 変換**:

  • 入力データを 16 ビット単位のワード配列に変換します。
  • ゼロが連続する最長の範囲を特定し、:: による省略表記を決定します。
  • 変換後の文字列を一時バッファ tmp に構築し、最後に dst へコピーします。

    • 4. **エラーハンドリング**: バッファサイズが不足する場合や、ビット数が範囲外の場合は NULL を返し、errno をセットします。

直接依存

  • postgres.h / postgres_fe.h: PostgreSQL の基本定義。
  • <sys/socket.h>, <netinet/in.h>, <arpa/inet.h>: 標準的なネットワークソケット関連の定義。
  • arpa/inet.h
  • netinet/in.h
  • sys/socket.h
  • utils/inet.h

関連コード

  • src/port/inet_aton.c: 逆の変換（文字列からバイナリ）を行うための関連ファイル。
  • src/include/utils/inet.h: PostgreSQL 内部でのネットワーク型定義。

読者向けメモ

  • 本ファイルは ISC ライセンスに基づくコードを含んでおり、歴史的な経緯から sprintf の戻り値の扱いに工夫（SPRINTF マクロ）が見られます。
  • IPv6 のゼロ圧縮ロジックは複雑であるため、変更を加える際は best（最長ゼロラン）の計算ロジックを慎重に追跡してください。
  • PGSQL_AF_INET とシステム側の AF_INET が一致している前提で動作するため、移植先環境でこれらが衝突しないか注意が必要です。
