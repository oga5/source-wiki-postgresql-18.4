src/port/pthread-win32.h

概要

Windows 環境において、PostgreSQL が必要とする最小限の POSIX スレッド（pthread）API を提供するためのヘッダーファイルです。Windows ネイティブの同期プリミティブを POSIX 風のインターフェースでラップし、クロスプラットフォームなコードベースの移植性を維持します。

モジュール内での役割

src/port ディレクトリの一部として、Windows 上でのビルド時に POSIX スレッド機能が欠如している問題を補完します。主に libpq などのフロントエンドライブラリや、スレッドセーフな操作が必要なバックエンドユーティリティから利用されます。

責務

  • Windows の CRITICAL_SECTION を用いたミューテックスの実装定義。
  • スレッド識別子（pthread_self）およびスレッド固有データ（TSD）操作のインターフェース定義。
  • pthread_once_t などの基本的なスレッド制御型の提供。

重要なシンボル

  • pthread_mutex_t: ミューテックスの状態（initstate）と Windows の CRITICAL_SECTION を保持する構造体。
  • PTHREAD_MUTEX_INITIALIZER: ミューテックスを初期化するための静的初期化マクロ。
  • pthread_mutex_lock: 内部で InterlockedExchange を使用し、スレッドセーフな遅延初期化（Double-Checked Locking に類似）を実現する関数。

追加の索引済みシンボル

  • pthread_key_t
  • pthread_once_t

制御フローとデータフロー

本ファイルはインターフェース定義のみを提供します。実際の動作は pthread-win32.c に依存します。

  • **初期化フロー**: pthread_mutex_lock が呼び出されると、initstate を確認します。未初期化（0）であれば InterlockedExchange を用いて排他的に InitializeCriticalSection を実行し、状態を 1 に遷移させます。
  • **データフロー**: pthread_key_t は単なる ULONG として定義されており、現在の実装では TSD（スレッド固有データ）の機能はスタブ化されています。

直接依存

  • Windows API: windows.h（暗黙的）および winnt.h 等の Windows システムヘッダーに依存し、CRITICAL_SECTION や LONG 型を利用します。

関連コード

  • src/interfaces/libpq/pthread-win32.c: 本ヘッダーで宣言された関数の実装が含まれています。
  • src/include/c.h: PostgreSQL のポータビリティ層の基盤であり、Windows 固有の型定義やマクロの調整が行われています。

読者向けメモ

  • **スタブの存在**: pthread_setspecific や pthread_getspecific は現在空の実装（スタブ）です。これらが必要な機能を利用する際は、Windows 上での動作に制限があることに注意してください。
  • **初期化の重要性**: pthread_mutex_t は PTHREAD_MUTEX_INITIALIZER で初期化するか、明示的に pthread_mutex_init を呼び出す必要があります。initstate の管理は pthread-win32.c 内で厳密に行われていますが、手動で構造体を操作することは避けてください。
