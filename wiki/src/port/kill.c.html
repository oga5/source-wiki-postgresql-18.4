src/port/kill.c

概要

Windows環境における kill() システムコールの代替実装を提供します。POSIX準拠のシグナル送信機能をWindowsのプロセス間通信（名前付きパイプ）を用いてエミュレートします。

モジュール内での役割

src/port ディレクトリの一部として、プラットフォーム固有の差異を吸収する役割を担います。PostgreSQLのバックエンドがWindows上でシグナルを適切に受信・処理できるようにするための抽象化レイヤーです。

責務

  • **シグナル送信の抽象化**: pgkill 関数を通じて、Windows上のプロセスに対してシグナルを送信する。
  • **SIGKILLの特別処理**: SIGKILL が指定された場合、Windowsの TerminateProcess を直接呼び出し、即時終了を強制する。
  • **名前付きパイプによる通信**: SIGKILL 以外のシグナルに対しては、ターゲットプロセスが作成した名前付きパイプ (\\.\pipe\pgsignal_<pid>) を介してシグナル番号を伝達する。
  • **エラーマッピング**: Windows特有のエラーコードを、POSIX準拠の errno（ESRCH, EPERM, EINVAL 等）に変換する。

重要なシンボル

  • pgkill(int pid, int sig): 本ファイルの主要関数。Windows環境での kill() の代替。

追加の索引済みシンボル

  • pgkill

制御フローとデータフロー

1. **入力検証**: 引数のシグナル番号とPIDが妥当か確認します。 2. **SIGKILL判定**: SIGKILL の場合は OpenProcess と TerminateProcess を使用してプロセスを強制終了します。 3. **パイプ通信**: それ以外のシグナルは CallNamedPipe を使用し、ターゲットプロセスのパイプへシグナルデータを書き込みます。 4. **結果判定**:

  • 通信成功時: 戻り値を確認し、正常終了を返します。
  • 通信失敗時: GetLastError() を確認し、パイプの状態（終了処理中など）に応じて適切な errno を設定します。

直接依存

  • c.h: PostgreSQLの基本定義およびプラットフォーム依存のヘッダーインクルードを管理する共通ヘッダー。

関連コード

  • src/port/pqsignal.c: シグナルハンドリングに関連する補助機能。
  • src/port/win32error.c: Windowsエラーコードと errno のマッピングに関連するユーティリティ。

読者向けメモ

  • 本ファイルは #ifdef WIN32 で囲まれており、Windowsビルド時のみ有効です。
  • pgkill はプロセスグループへのシグナル送信をサポートしていません（pid <= 0 はエラーとなります）。
  • パイプ通信の失敗時、ERROR_BROKEN_PIPE 等はプロセスが終了処理中であると見なし、POSIXの挙動に合わせて成功（0）を返す設計になっています。
