src/port/chklocale.c

概要

chklocale.c は、OS のロケール設定（LC_CTYPE）から PostgreSQL が内部的に使用する文字エンコーディング（pg_enc）を特定するためのユーティリティ関数群を提供します。

モジュール内での役割

src/port 内において、データベースの初期化（initdb）やクライアント接続時のエンコーディング自動検出をサポートするプラットフォーム抽象化レイヤーの一部です。OS 固有のロケール表現と PostgreSQL 内部のエンコーディング識別子を橋渡しします。

責務

  • OS が提供するロケール名やコードセット名と、PostgreSQL のエンコーディング定数のマッピング。
  • Windows 環境におけるコードページ（CPnnn）から PostgreSQL エンコーディングへの変換。
  • ロケール文字列からシステム固有のコードセット情報を抽出する抽象化。

重要なシンボル

  • encoding_match_list: システムのエンコーディング名と PostgreSQL のエンコーディングコードを対応付ける静的テーブル。
  • pg_get_encoding_from_locale(): ロケール文字列から PostgreSQL エンコーディングを導出する主要な公開関数。
  • win32_get_codeset(): Windows 環境でロケール名からコードページ番号を解決するヘルパー。

追加の索引済みシンボル

制御フローとデータフロー

1. **入力**: pg_get_encoding_from_locale がロケール文字列を受け取ります（NULL の場合は現在の環境設定を使用）。 2. **正規化**:

  • 非 Windows 環境では newlocale と nl_langinfo_l を使用してシステムからコードセット名を取得します。
  • Windows 環境では GetLocaleInfoEx を使用してコードページを特定します。

    • 3. **照合**: 取得した文字列を encoding_match_list と照合します。pg_strcasecmp を用いて大文字小文字を区別せずに検索します。 4. **出力**: 一致した場合は対応する pg_enc を返し、不明な場合は -1 を返します。

直接依存

  • postgres.h / postgres_fe.h: 環境に応じた基本定義。
  • langinfo.h: POSIX 環境でのロケール情報取得用。
  • mb/pg_wchar.h: PostgreSQL のマルチバイト文字エンコーディング定義。

関連コード

  • src/include/mb/pg_wchar.h: pg_enc 列挙型が定義されており、本ファイルのマッピング先となります。
  • src/port/win32setlocale.c: Windows におけるロケール設定の補完を行う関連ファイル。

読者向けメモ

  • **Windows 特有の挙動**: Windows では GetLocaleInfoEx が失敗した場合、文字列の末尾からコードページ番号を推測するフォールバック処理が含まれています。
  • **C/POSIX ロケール**: C または POSIX ロケールが指定された場合、エンコーディングは PG_SQL_ASCII として扱われます。
  • **拡張性**: 新しいエンコーディングや別名を追加したい場合は、encoding_match_list にエントリを追加するだけで対応可能です。
  • **警告**: 認識できないコードセットに遭遇した場合、write_message フラグが真であれば警告ログが出力されます。これは環境設定の不整合をデバッグする際に有用です。
