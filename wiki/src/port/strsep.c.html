src/port/strsep.c

概要

strsep.c は、文字列を特定の区切り文字で分割するための標準的なユーティリティ関数 strsep を提供します。このファイルは、標準ライブラリで strsep が提供されていないプラットフォーム向けに、PostgreSQL の移植性レイヤーとして実装されています。

モジュール内での役割

src/port ディレクトリの一部として、OS 間の差異を吸収し、PostgreSQL のバックエンドおよびフロントエンドコードがプラットフォームに依存せず文字列操作を行えるようにします。

責務

  • 文字列のトークン化（分割）処理の提供。
  • 入力文字列を破壊的に変更し、区切り文字を \0（ヌル文字）に置換することでトークンを生成する。
  • 呼び出し元が保持する文字列ポインタを、次のトークンの開始位置へ更新する。

重要なシンボル

  • strsep(char **stringp, const char *delim):
  • *stringp で指定された文字列を delim に含まれる文字で分割します。
  • 処理が完了すると *stringp は次のトークンの先頭を指すように更新され、トークンが尽きると NULL になります。

追加の索引済みシンボル

  • strsep

制御フローとデータフロー

1. **初期チェック**: *stringp が NULL の場合、即座に NULL を返します。 2. **ループ処理**: 文字列を 1 文字ずつ走査します。 3. **照合**: 現在の文字が delim に含まれるかを確認します。 4. **トークン確定**:

  • 区切り文字が見つかった場合、その位置を \0 で上書きし、*stringp をその次の位置へ更新してトークンを返します。
  • 文字列の終端（\0）に達した場合は、*stringp を NULL に設定して終了します。

直接依存

  • c.h: PostgreSQL の基本的な型定義やマクロを提供します。

関連コード

  • strtok(3): 類似の関数ですが、strsep は空のトークンを正しく処理できる点や、状態を呼び出し元が管理する点で異なります。
  • src/port/strlcpy.c, src/port/strlcat.c: 同じく文字列操作の移植性を高めるためのユーティリティ群です。

読者向けメモ

  • **破壊的変更**: この関数は引数として渡された文字列を直接書き換えます。元の文字列を保持する必要がある場合は、事前にコピーを作成してください。
  • **移植性**: この実装は BSD 由来の標準的な挙動に従っています。システム標準の strsep が存在しない環境でのみ使用されることを想定しています。
