src/port/win32stat.c

概要

Windows 環境において、POSIX 準拠のファイル状態取得関数（stat, lstat, fstat）をエミュレートするための実装です。Windows API の GetFileInformationByHandle を活用し、ファイル属性やタイムスタンプを PostgreSQL が期待する struct stat 形式へ変換します。

モジュール内での役割

src/port 内の Windows 互換レイヤーの一部です。標準の C ライブラリが提供する stat 系関数が Windows のジャンクションポイントや特定のファイル属性を正しく扱えないケースを補完し、PostgreSQL のバックエンドおよびフロントエンドがプラットフォーム差異を意識せずにファイルシステムへアクセスできるようにします。

責務

  • **メタデータ変換**: Windows の FILETIME やファイル属性を、Unix スタイルの time_t や mode_t へ変換する。
  • **シンボリックリンク（ジャンクション）対応**: Windows のジャンクションポイントを POSIX のシンボリックリンクとして解釈する。
  • **ファイル記述子管理**: _get_osfhandle を介して Windows ハンドルを取得し、パイプやキャラクタデバイスを含む多様なファイルタイプを struct stat にマッピングする。

重要なシンボル

  • _pglstat64: lstat の実装。シンボリックリンク自体を対象とし、リンク先を追跡しません。
  • _pgstat64: stat の実装。シンボリックリンクを再帰的に追跡します。
  • _pgfstat64: fstat の実装。ファイル記述子からファイル情報を取得します。
  • fileinfo_to_stat: Windows の BY_HANDLE_FILE_INFORMATION を struct stat に変換する内部ヘルパー。

追加の索引済みシンボル

  • fileattr_to_unixmode
  • filetime_to_time

制御フローとデータフロー

1. **_pglstat64**: pgwin32_open_handle でファイルを開き、fileinfo_to_stat で基本情報を取得します。ディレクトリと判定された場合、readlink を呼び出し、ジャンクションポイントであれば S_IFLNK として情報を上書きします。 2. **_pgstat64**: _pglstat64 を呼び出した後、結果がシンボリックリンクであれば、リンク先を解決するまでループ（最大8回）して再帰的に _pglstat64 を呼び出します。 3. **_pgfstat64**: _get_osfhandle で取得したハンドルから pgwin32_get_file_type で種別を判定し、ディスクファイルであれば fileinfo_to_stat を、パイプやキャラクタデバイスであれば適切な st_mode を設定します。

直接依存

  • c.h: PostgreSQL の基本定義。
  • port/win32ntdll.h: NTDLL 関連の低レベル定義。
  • windows.h: Windows API のコアヘッダー。

関連コード

  • src/port/dirmod.c: ディレクトリ操作や readlink の実装が含まれており、本ファイルと密接に連携します。
  • src/port/win32common.c: Windows 固有の共通ユーティリティ。
  • src/include/port/win32.h: stat 構造体の定義やマクロの再定義が含まれます。

読者向けメモ

  • 本ファイルは _pglstat64 等のプレフィックス付き関数を定義しており、これらは通常 src/include/port/win32.h 内の #define によって標準の stat 等に置き換えられます。
  • ジャンクションポイントの判定には readlink を利用しており、ファイルシステムの実装に依存した挙動が含まれる点に注意してください。
  • fileattr_to_unixmode は所有者の権限のみを考慮しており、グループや他人の権限ビットは設定されません。
