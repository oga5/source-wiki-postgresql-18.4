src/port/path.c

概要

src/port/path.c は、PostgreSQL におけるクロスプラットフォームなパス操作ルーチンを提供します。Windows と Unix 系 OS 間で異なるパス区切り文字、ドライブレターの扱い、およびパスの正規化ルールを抽象化し、一貫したファイルシステム操作を可能にします。

モジュール内での役割

src/port モジュールの一部として、バックエンドおよびフロントエンド（psql 等）の両方から利用されます。特にインストールディレクトリの相対パス計算や、セキュリティ上の観点から安全なパスの検証を行うための基盤を提供します。

責務

  • **パスの正規化**: canonicalize_path を通じて、冗長な区切り文字、.、.. を解決し、一貫した形式に変換します。
  • **プラットフォーム抽象化**: Windows のドライブレターや \ を Unix 形式の / に統一し、必要に応じてネイティブ形式へ戻す処理を行います。
  • **相対パス計算**: インストール場所（bin）を基準とした、データやライブラリディレクトリへの相対パスを算出します。
  • **セキュリティ検証**: 抽出処理時などに、パスがカレントディレクトリを脱出していないかを検証します。

重要なシンボル

  • canonicalize_path: パスを標準化するメイン関数。
  • make_relative_path: インストールディレクトリの再配置をサポートするための相対パス算出。
  • path_is_safe_for_extraction: パスが .. を含まず、安全であることを確認する関数。
  • get_progname: 実行ファイル名からディレクトリパスを除去し、.exe サフィックスを処理する関数。
  • IS_DIR_SEP: OS 依存のパス区切り文字を判定するマクロ。

制御フローとデータフロー

1. **正規化**: canonicalize_path が呼び出されると、まず Windows 特有の \ を / に置換し、末尾の不要な文字を削除します。その後、状態マシン（canonicalize_state）を用いてパスの各コンポーネントを走査し、.. を解決します。 2. **相対パス計算**: make_relative_path は、コンパイル時のパスと実行時のパスを比較し、共通のプレフィックスを特定します。その後、join_path_components を用いて、実行環境の場所を基準とした新しいパスを構築します。 3. **安全確認**: path_is_safe_for_extraction は、入力を正規化した上で path_contains_parent_reference を呼び出し、ディレクトリトラバーサル攻撃を防ぐためのチェックを行います。

直接依存

  • postgres.h / postgres_fe.h: 環境に応じた共通定義。
  • mb/pg_wchar.h: マルチバイト文字のエンコーディング処理用。
  • sys/stat.h, unistd.h, pwd.h: OS 標準のファイルシステム・ユーザー情報操作。
  • shlobj.h (Windows): Windows 特有のパス操作用。
  • ctype.h
  • pg_config_paths.h

関連コード

  • src/include/port.h: 本ファイルで定義されている関数のプロトタイプ宣言が含まれています。
  • src/common/wchar.c: 文字列のエンコーディング関連のより詳細な処理が必要な場合に参照されます。
  • src/port/dirmod.c: ディレクトリ操作に関連する補完的な機能が含まれています。

読者向けメモ

  • **Windows 互換性**: Windows 上では \ と / が混在しやすいため、本ファイルを通した正規化が必須です。
  • **メモリ管理**: make_absolute_path など一部の関数は malloc を使用するため、呼び出し側での free が必要です。
  • **SJIS の扱い**: Windows 環境における Shift-JIS の \（0x5C）問題に対処するため、debackslash_path 内で特殊な文字長計算を行っています。他のエンコーディングを追加する際は注意してください。
