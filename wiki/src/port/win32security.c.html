src/port/win32security.c

概要

Windows 環境における PostgreSQL の実行権限および実行コンテキストを判定するためのセキュリティ補助関数群を提供します。

モジュール内での役割

src/port ディレクトリの一部として、Windows 特有の API を抽象化し、PostgreSQL のバックエンドおよびフロントエンド（pg_ctl 等）が共通して利用できるセキュリティチェック機能を提供します。

責務

  • 現在のプロセスが管理者権限（Administrators または Power Users）で実行されているかの判定。
  • 現在のプロセスが Windows サービスとして実行されているかの判定。
  • 判定失敗時のエラーログ出力（ereport が利用できない初期化フェーズを考慮）。

重要なシンボル

  • pgwin32_is_admin: 現在のユーザーが管理者グループに属しているかを判定します。
  • pgwin32_is_service: 現在のプロセスが Windows サービスとして動作しているかを判定します。
  • log_error: フロントエンドとバックエンドの両方で利用可能な、標準エラー出力への簡易ログ出力関数です。

制御フローとデータフロー

  • **pgwin32_is_admin**:

    • 1. AllocateAndInitializeSid で管理者および Power Users の SID を生成します。 2. CheckTokenMembership を用いて現在のアクセストークンがこれらの SID を含むか確認します。 3. 最後に FreeSid でリソースを解放します。

  • **pgwin32_is_service**:

    • 1. 静的変数 _is_service を使用し、初回実行時のみ判定を行います。 2. 標準エラー出力のハンドル有無、LocalSystem アカウントでの実行、SECURITY_SERVICE_RID のトークン保持状況を順に確認し、サービス実行か否かを決定します。

直接依存

  • postgres.h / postgres_fe.h: ビルドターゲットに応じた共通定義。
  • Windows API (windows.h 相当の機能): AllocateAndInitializeSid, CheckTokenMembership, GetStdHandle 等。

関連コード

  • src/port/win32common.c: Windows 固有の共通ユーティリティ。
  • src/port/win32error.c: Windows エラーコードの変換や処理。

読者向けメモ

  • 本ファイル内の関数は、PostgreSQL の起動直後など、ereport() や write_stderr() がまだ利用できない極めて早い段階で呼び出されることを想定しています。そのため、エラーハンドリングは標準的な PostgreSQL の仕組みを介さず、直接 stderr へ出力する実装になっています。
  • pgwin32_is_service はキャッシュ（static int）を利用するため、一度判定が確定するとその後の呼び出しは高速です。
