src/port/pgmkdirp.c

概要

pg_mkdir_p 関数を提供し、指定されたパスに対して再帰的にディレクトリを作成します。UNIXの mkdir -p コマンドと同等の機能を提供し、既存のディレクトリが存在してもエラーとしない点が特徴です。

モジュール内での役割

src/port は、プラットフォーム間の差異を吸収する移植性レイヤーです。本ファイルは、ファイルシステム操作における標準的なディレクトリ作成処理を、OS依存性を最小限に抑えつつ提供する役割を担います。

責務

  • 指定されたパスの各階層を順次走査し、存在しないディレクトリを作成する。
  • 作成時に適切なパーミッション（omode）を適用する。
  • 親ディレクトリ作成時には、書き込み・実行権限を保証するために umask を一時的に変更する。
  • Windows環境におけるドライブレターやネットワークパスの特殊な形式を適切にスキップする。

重要なシンボル

  • pg_mkdir_p(char *path, int omode): 本ファイルの主要関数。成功時に0、失敗時に-1を返す。

追加の索引済みシンボル

  • pg_mkdir_p

制御フローとデータフロー

1. **パスの正規化（Windows対応）**: Windows環境の場合、ドライブレターやUNCパスのプレフィックスをスキップし、処理対象のパス位置を調整します。 2. **umaskの調整**: umask を一時的に変更し、親ディレクトリ作成時に必要な権限（u+wx）を確保します。 3. **再帰的作成**: パスを / で分割し、各階層ごとに stat で存在確認を行います。

  • 存在しない場合: mkdir を実行。
  • 存在する場合: ディレクトリであることを確認し、そうでなければエラーを返します。

    • 4. **復元**: 処理終了時またはエラー発生時に、元の umask を必ず復元します。

直接依存

  • <sys/stat.h>: stat 構造体および mkdir 関数定義。
  • sys/stat.h

関連コード

  • src/port/dirmod.c: ディレクトリ操作に関連する他の移植性関数が含まれています。
  • src/port/path.c: パス操作に関するユーティリティ関数群。

読者向けメモ

  • 本関数は path 引数を破壊的に変更します（内部で / を \0 に置き換えるため）。呼び出し元でパス文字列を保持する必要がある場合は注意してください。
  • omode は mode_t ではなく int で宣言されています。これは port.h への依存を減らすための設計上の選択です。
  • エラー発生時、path 引数は「問題が発生した階層」までが書き換えられた状態となります。
