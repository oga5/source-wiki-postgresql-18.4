src/port/timingsafe_bcmp.c

概要

timingsafe_bcmp.c は、メモリ領域の比較を「タイミング攻撃」に対して安全に行うための移植性レイヤーです。比較処理にかかる時間を入力データの内容に依存させないことで、サイドチャネル攻撃を防止します。

モジュール内での役割

src/port ディレクトリの一部として、標準ライブラリに同等の機能がない、あるいは環境によって実装が異なる場合に備えて提供されるユーティリティ関数です。主に認証トークンや暗号学的ハッシュの比較など、セキュリティが重要な箇所で使用されます。

責務

  • メモリ領域の比較において、不一致が見つかった時点で処理を中断せず、常に全バイトを走査する。
  • OpenSSL が利用可能な環境では、CRYPTO_memcmp を優先的に使用して最適化を図る。
  • OpenSSL が利用できない環境では、ビット演算を用いた定数時間比較アルゴリズムを提供する。

重要なシンボル

  • timingsafe_bcmp: 本ファイルの唯一の公開関数。memcmp と同様の引数を取り、一致すれば 0、不一致なら非 0 を返します。

制御フローとデータフロー

1. **分岐**: USE_SSL マクロの定義有無により実装が切り替わります。 2. **OpenSSL 版**: CRYPTO_memcmp を呼び出し、その結果をそのまま返します。 3. **フォールバック版**:

  • 入力ポインタを unsigned char として扱います。
  • ret 変数に各バイトの XOR (^) 結果を累積 (|=) します。
  • 途中で不一致があってもループを抜けず、最後まで計算を続行します。
  • 最後に ret が 0 かどうかを判定し、結果を返します。これにより、比較時間がデータ内容に依存しないことを保証します。

直接依存

  • c.h: PostgreSQL の基本定義および移植性マクロ。
  • openssl/crypto.h: USE_SSL 定義時に CRYPTO_memcmp を利用するために必要。

関連コード

  • src/port/explicit_bzero.c: セキュリティ関連のユーティリティとして、メモリの安全なクリアを行う点で概念的に近接しています。
  • src/include/c.h: 本ファイルが依存する基本定義が含まれています。

読者向けメモ

  • この関数は、通常の memcmp とは異なり、パフォーマンスよりも「処理時間の一定性」を優先しています。
  • ret |= *p1++ ^ *p2++ というイディオムは、分岐命令を発生させずに不一致を累積させるための定石です。
  • 比較対象が暗号学的に重要なデータである場合は、必ずこの関数を使用してください。
