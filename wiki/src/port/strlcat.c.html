src/port/strlcat.c

概要

strlcat.c は、安全な文字列連結関数である strlcat を提供する移植性レイヤーのソースファイルです。標準の strncat が抱えるバッファオーバーフローのリスクを排除し、常にヌル終端を保証する実装を提供します。

モジュール内での役割

src/port ディレクトリの一部として、プラットフォーム間で標準ライブラリの挙動を統一するために使用されます。OS 側で strlcat が提供されていない環境において、PostgreSQL のビルドプロセスで代替としてコンパイルされます。

責務

  • 宛先バッファの全サイズ（siz）を考慮した安全な文字列連結。
  • 連結後の文字列が常にヌル終端されることの保証。
  • 連結後の文字列長がバッファサイズを超えた場合の切り詰め（Truncation）の検知と、必要なバッファサイズの算出。

重要なシンボル

  • strlcat(char *dst, const char *src, size_t siz): 本ファイルの唯一の公開関数。dst に src を連結し、連結後の長さを返します。

追加の索引済みシンボル

  • strlcat

制御フローとデータフロー

1. **初期スキャン**: dst の終端を探索し、残りのバッファサイズ n を計算します。 2. **連結処理**: n が 1 より大きい間、src の文字を dst の末尾にコピーします。 3. **終端処理**: 連結完了後、必ず *d = '\0' を実行してヌル終端を強制します。 4. **戻り値**: dst の初期長と src の長さの合計を返します。この値が siz 以上であれば、切り詰めが発生したことを呼び出し元に伝えます。

直接依存

  • c.h: PostgreSQL の移植性確保のための基本ヘッダー。

関連コード

  • src/port/strlcpy.c: 対となる文字列コピー関数。同様に安全なコピーを提供します。
  • src/port/strnlen.c: 文字列の長さを安全に取得するための補助関数。

読者向けメモ

  • strncat とは異なり、第 3 引数 siz は「残りの空き容量」ではなく「バッファ全体のサイズ」である点に注意してください。
  • 戻り値が siz 以上の場合、データが切り詰められたことを意味します。この挙動は OpenBSD の strlcat 仕様に準拠しています。
  • 本実装は、標準ライブラリが strlcat を提供していない環境でのみ使用されることを想定しています。
