src/port/win32setlocale.c

概要

Windows 環境における setlocale() 関数の不具合を吸収するためのラッパー実装です。特定のロケール名が正しく認識されない問題や、非 ASCII 文字を含むロケール名が返される問題を解決し、PostgreSQL が安定してロケールを扱えるようにします。

モジュール内での役割

src/port ディレクトリの一部として、Windows プラットフォーム固有の移植性レイヤーを提供します。標準ライブラリの setlocale を直接呼び出す代わりに、このラッパーを経由させることで、OS 側のバグを透過的に回避します。

責務

  • **入力の正規化**: setlocale() に渡すロケール名にドットが含まれる場合など、Windows が拒絶する形式を許容される別名（エイリアス）に置換します。
  • **出力の正規化**: setlocale() が返すロケール名に非 ASCII 文字が含まれる場合、それを安全な ASCII 文字列に変換し、後続の処理でエンコーディングの問題が発生しないようにします。

重要なシンボル

  • pgwin32_setlocale: 本ファイルのメイン関数。標準の setlocale を置き換えるラッパーです。
  • locale_map_argument: setlocale 呼び出し前に適用される置換ルール（入力用）。
  • locale_map_result: setlocale 呼び出し後に適用される置換ルール（出力用）。
  • map_locale: ロケール名の文字列置換を実行する内部ヘルパー関数。

追加の索引済みシンボル

  • locale_map

制御フローとデータフロー

1. **入力変換**: pgwin32_setlocale が呼び出されると、引数のロケール名を map_locale に渡し、locale_map_argument に基づいて別名に変換します。 2. **実行**: 変換後の文字列を使用して、本来の setlocale() を呼び出します。 3. **出力変換**: setlocale() の戻り値に対し、map_locale を用いて locale_map_result に基づく置換を行い、非 ASCII 文字を排除した文字列を返します。 4. **バッファ管理**: map_locale は静的バッファ aliasbuf を使用して変換後の文字列を保持します。

直接依存

  • c.h: PostgreSQL の共通ヘッダー。unconstify マクロや基本的な型定義を提供します。

関連コード

  • src/include/c.h: setlocale の再定義や unconstify マクロの定義元。
  • Windows SDK の locale.h: 本来の setlocale 関数が定義されている標準ヘッダー。

読者向けメモ

  • 本ファイルは #undef setlocale を行い、標準の setlocale を隠蔽してラッパー関数を優先させています。
  • map_locale 関数は、単純な文字列置換だけでなく、start と end を指定した簡易的な範囲置換もサポートしています。これは、ロケール名の一部のみを動的に書き換える必要がある場合に利用されます。
  • aliasbuf は静的領域であるため、スレッドセーフな実装が必要な環境では注意が必要です（ただし、PostgreSQL の現在の Windows 移植層の設計に依存します）。
