src/port/win32fdatasync.c

概要

Windows環境において、POSIX標準の fdatasync() をエミュレートするための実装です。ファイル記述子（fd）に関連付けられたデータを物理ストレージへフラッシュし、メタデータの更新を最小限に抑えることで同期処理を行います。

モジュール内での役割

src/port ディレクトリの一部として、PostgreSQLがクロスプラットフォームで動作するために必要な、Windows固有のシステムコールラッパーを提供します。

責務

  • **ファイルハンドルの取得**: POSIX形式のファイル記述子（int fd）を、Windowsのネイティブハンドル（HANDLE）へ変換する。
  • **低レベル同期の実行**: Windowsの ntdll.dll が提供する NtFlushBuffersFileEx を呼び出し、データのみの同期（FLUSH_FLAGS_FILE_DATA_SYNC_ONLY）を要求する。
  • **エラーマッピング**: ネイティブな NTSTATUS コードを、標準的なCライブラリの errno に変換する。

重要なシンボル

  • fdatasync(int fd): 本ファイルの唯一の公開関数。POSIX互換のインターフェースを提供します。
  • pg_NtFlushBuffersFileEx: win32ntdll.h で定義された、ntdll の関数への動的リンクポインタ。
  • FLUSH_FLAGS_FILE_DATA_SYNC_ONLY: メタデータ（最終アクセス時刻など）の更新を待たずに、データのみをフラッシュすることを指示するフラグ。

追加の索引済みシンボル

  • fdatasync

制御フローとデータフロー

1. _get_osfhandle(fd) を呼び出し、Cランタイムのファイル記述子からWindowsハンドルを取得する。 2. initialize_ntdll() を実行し、必要なネイティブAPI関数ポインタがロードされていることを確認する。 3. pg_NtFlushBuffersFileEx を呼び出し、同期処理をOSカーネルへ依頼する。 4. 結果が成功（NT_SUCCESS）であれば 0 を返し、失敗であれば pg_RtlNtStatusToDosError でエラーコードを変換し、_dosmaperr で errno を設定して -1 を返す。

直接依存

  • postgres.h / postgres_fe.h: PostgreSQLの基本定義および環境設定。
  • port/win32ntdll.h: WindowsネイティブAPI（ntdll）へのアクセスに必要なラッパー定義。

関連コード

  • src/port/win32ntdll.c: pg_NtFlushBuffersFileEx などの関数ポインタを動的にロードする実装が含まれています。
  • src/port/open.c: Windows環境でのファイルオープン処理に関連する補助関数群。

読者向けメモ

  • この関数は、Windowsの標準APIではなく、ntdll.dll を直接操作する低レベルな実装です。
  • fdatasync は、ファイルの内容のみをディスクに書き込むことを保証しますが、ファイルシステムの実装によってはメタデータも更新される場合があります。
  • initialize_ntdll() の呼び出しは、初回実行時にのみ重い処理（GetModuleHandle や GetProcAddress）が発生するよう設計されています。
