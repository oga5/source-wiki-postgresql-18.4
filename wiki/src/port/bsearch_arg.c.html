src/port/bsearch_arg.c

概要

標準ライブラリの bsearch を拡張し、比較関数に任意のコンテキストポインタ（arg）を渡せるようにした二分探索の実装です。

モジュール内での役割

src/port ディレクトリの一部として、プラットフォーム間で一貫した動作を保証するためのユーティリティ関数を提供します。標準の bsearch では比較関数に引数を渡せないため、状態を保持する必要がある検索処理において不可欠な代替手段となります。

責務

  • ソート済み配列に対する二分探索の実行。
  • 比較関数 compar に対して、検索対象の key、配列要素 p、およびユーザー定義の arg を渡すインターフェースの提供。
  • 検索対象が見つかった場合はそのポインタを返し、見つからない場合は NULL を返す。

重要なシンボル

  • bsearch_arg: 本ファイルの唯一の公開関数。標準の bsearch と同様のシグネチャを持ちつつ、void *arg を追加で受け取ります。

制御フローとデータフロー

1. **初期化**: lim（残りの要素数）を nmemb で初期化します。 2. **ループ**: lim が 0 になるまで、以下の処理を繰り返します。

  • lim >> 1 を用いて中央の要素 p を特定します。
  • compar(key, p, arg) を呼び出し、結果 cmp を取得します。
  • cmp == 0 ならば p を返します。
  • cmp > 0 ならば、探索範囲を右半分に絞り込みます（base を更新し、lim を調整）。
  • cmp < 0 ならば、探索範囲を左半分に絞り込みます（lim を半分にする）。

    • 3. **終了**: ループを抜けた場合は NULL を返します。

直接依存

  • c.h: PostgreSQL の基本型定義やマクロを含む共通ヘッダー。

関連コード

  • qsort_arg.c: 本関数と同様に、比較関数に arg を渡せるように拡張されたクイックソートの実装です。対になる操作として頻繁に併用されます。

読者向けメモ

  • この実装は、lim が奇数か偶数かによって探索範囲の絞り込みロジックを微調整する、古典的かつ効率的なアルゴリズムを採用しています。
  • 比較関数 compar の実装者は、arg を通じて外部の状態（検索条件や環境情報など）に安全にアクセスできるため、グローバル変数を使用せずに検索処理を記述可能です。
