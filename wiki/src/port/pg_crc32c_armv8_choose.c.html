src/port/pg_crc32c_armv8_choose.c

概要

ARMv8 CPU における CRC-32C 計算の最適化実装を選択するためのランタイム判定ロジックを提供します。

モジュール内での役割

src/port 内の CRC-32C 関連ファイル群の一部であり、実行環境が ARMv8 の CRC 拡張命令をサポートしているかを判定し、最適な計算関数へ動的に切り替える役割を担います。

責務

  • 実行時の CPU 機能判定（ARMv8 CRC 拡張の有無）。
  • pg_comp_crc32c 関数ポインタの初期化と、初回呼び出し時の実装切り替え。
  • OS 固有のインターフェース（Linux の getauxval や NetBSD の sysctl）を抽象化し、判定ロジックを隠蔽する。

重要なシンボル

  • pg_comp_crc32c: CRC-32C 計算を行うグローバルな関数ポインタ。初期状態では pg_comp_crc32c_choose を指します。
  • pg_comp_crc32c_choose: 初回呼び出し時に実行される判定関数。判定後、自身を適切な実装（ARMv8 またはソフトウェア実装）に置き換えます。
  • pg_crc32c_armv8_available: CPU が CRC 拡張をサポートしているかを OS 経由で確認する内部関数。

制御フローとデータフロー

1. **初期化**: pg_comp_crc32c は pg_comp_crc32c_choose を指して初期化されます。 2. **初回実行**: 最初の CRC 計算要求時に pg_comp_crc32c_choose が呼ばれます。 3. **判定**: pg_crc32c_armv8_available が OS の機能フラグ（HWCAP_CRC32 等）を読み取り、サポート状況を返します。 4. **切り替え**: 判定結果に基づき、pg_comp_crc32c を pg_comp_crc32c_armv8（ハードウェア）または pg_comp_crc32c_sb8（ソフトウェア）に再割り当てします。 5. **実行**: 選択された関数を呼び出し、結果を返します。以降の呼び出しは判定をスキップし、直接実装関数を呼び出します。

直接依存

  • sys/auxv.h / asm/hwcap.h (Linux 環境での機能判定用)
  • sys/sysctl.h / aarch64/armreg.h (NetBSD 環境での機能判定用)
  • port/pg_crc32c.h (CRC-32C の型定義とインターフェース)
  • postgres.h
  • postgres_fe.h

関連コード

  • src/port/pg_crc32c_armv8.c: ARMv8 命令を使用した高速な CRC-32C 実装。
  • src/port/pg_crc32c_sb8.c: フォールバック用の Slicing-by-8 ソフトウェア実装。
  • src/port/pg_crc32c_sse42_choose.c: x86_64 環境における同様の判定ロジック。

読者向けメモ

  • 本ファイルは「初回呼び出し時に自身を書き換える」というパターン（Function Pointer Swapping）を採用しています。これにより、以降の呼び出しコストを最小化しています。
  • NetBSD における判定は sysctl を使用しており、32bit/64bit アーキテクチャ間でレジスタの読み取り方法が異なる点に注意してください。
  • FRONTEND マクロの有無により、PostgreSQL サーバーとクライアント（pg_dump 等）の両方で利用可能な設計になっています。
