src/port/inet_aton.c

概要

inet_aton.c は、ドット区切りの IPv4 アドレス文字列をバイナリ形式（struct in_addr）に変換する inet_aton() 関数を提供する移植性レイヤーです。

モジュール内での役割

src/port ディレクトリの一部として、標準 C ライブラリに inet_aton() が存在しないプラットフォーム向けに、PostgreSQL が一貫したネットワークアドレス変換機能を利用できるようにします。

責務

  • 文字列形式の IPv4 アドレス（例: 127.0.0.1）の構文解析。
  • 10進数、8進数（0 プレフィックス）、16進数（0x プレフィックス）の混在表記のサポート。
  • 不正な形式や末尾の不要な文字に対する厳密なバリデーション。
  • ネットワークバイトオーダーへの変換と結果の格納。

重要なシンボル

  • inet_aton(const char *cp, struct in_addr *addr): メインの変換関数。成功時に 1、失敗時に 0 を返します。

追加の索引済みシンボル

  • inet_aton

制御フローとデータフロー

1. **解析ループ**: 入力文字列を . で分割し、各パートを数値に変換します。基数（8, 10, 16）はプレフィックスに基づいて動的に決定されます。 2. **検証**: 各パートが 8 ビット（または最後のパートが 8/16/24 ビット）の範囲内であるかを確認します。 3. **構築**: パート数に応じてビットシフトを行い、32 ビットの整数値を組み立てます。 4. **変換**: pg_hton32 を使用してホストバイトオーダーからネットワークバイトオーダーへ変換し、addr に書き込みます。

直接依存

  • c.h: PostgreSQL の基本定義。
  • <netinet/in.h>: struct in_addr の定義。
  • <ctype.h>: 文字種判定（isdigit, isxdigit 等）。
  • port/pg_bswap.h: バイトオーダー変換用マクロ。
  • ctype.h
  • netinet/in.h

関連コード

  • inet_net_ntop.c: 逆にバイナリから文字列への変換を行う関数を提供します。
  • src/include/port.h: inet_aton のプロトタイプ宣言が含まれる場合があります。

読者向けメモ

  • 本関数は GNU C ライブラリの古い実装をベースにしています。
  • inet_addr() と異なり、戻り値で成功・失敗を明確に区別できるため、エラーハンドリングが容易です。
  • 最後のパートに続く「ゴミ（不正な文字）」が含まれている場合、関数は 0 を返して失敗します。
