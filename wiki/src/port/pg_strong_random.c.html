src/port/pg_strong_random.c

概要

暗号学的に安全な乱数（CSPRNG）を生成するためのポータブルなインターフェースを提供します。認証時のソルトやクエリキャンセルキーなど、予測不可能性が求められる用途に使用されます。

モジュール内での役割

src/port ディレクトリの一部として、プラットフォーム固有の乱数生成機能を抽象化し、PostgreSQLのバックエンドおよびフロントエンドの両方から利用可能な共通APIを提供します。

責務

  • **プラットフォーム抽象化**: OpenSSL、Windows API (CryptGenRandom)、または /dev/urandom のいずれかを利用可能な環境に合わせて自動選択します。
  • **安全な乱数生成**: 呼び出し元に対して、暗号学的に十分な強度の乱数バイト列を保証します。
  • **初期化の提供**: 互換性のための初期化関数を提供します。

重要なシンボル

  • pg_strong_random(void *buf, size_t len): 指定されたバッファに安全な乱数を書き込むメイン関数。成功時に true を返します。
  • pg_strong_random_init(void): 互換性のための初期化関数。現在は実質的な処理は行いません。
  • NUM_RAND_POLL_RETRIES: OpenSSL使用時にCSPRNGのシード状態を確認・補完するための試行回数。

追加の索引済みシンボル

  • pg_strong_random
  • pg_strong_random_init

制御フローとデータフロー

1. **初期化**: pg_strong_random_init が呼び出されます（現在は空の操作）。 2. **生成**: pg_strong_random が呼び出されると、ビルド時の設定に応じて以下のいずれかのパスを通ります。

  • **OpenSSL**: RAND_status でシード状態を確認し、必要に応じて RAND_poll を実行後、RAND_bytes で生成。
  • **Windows**: CryptAcquireContext で暗号プロバイダを取得（キャッシュ）し、CryptGenRandom で生成。
  • **その他**: /dev/urandom を開き、read システムコールでデータを読み込みます。

    • 3. **終了**: 成功時は true を返し、失敗時は false を返します。呼び出し元は必ず戻り値をチェックする必要があります。

直接依存

  • c.h: PostgreSQLの基本定義。
  • fcntl.h, unistd.h, sys/time.h: POSIX環境でのファイル操作およびシステムコール用。
  • openssl/rand.h: OpenSSLビルド時のみ使用。
  • wincrypt.h: Windowsビルド時のみ使用。

関連コード

  • src/include/port.h: 本モジュールのプロトタイプ宣言が含まれています。
  • src/backend/libpq/auth.c: 認証プロセスで本関数が頻繁に利用される代表的な箇所です。

読者向けメモ

  • **バックエンド制約**: 本ファイルはPostgreSQLの起動初期段階で呼び出される可能性があるため、elog() や palloc() といったバックエンド固有のインフラに依存してはなりません。
  • **戻り値の重要性**: 乱数生成の失敗はセキュリティ上の脆弱性に直結します。戻り値が false の場合、処理を中断するなどの適切なエラーハンドリングが必須です。
  • **OpenSSLの挙動**: OpenSSL環境では、シードが不十分な場合に RAND_poll() を試行するロジックが含まれていますが、最終的な成功は RAND_bytes() の結果に依存します。
