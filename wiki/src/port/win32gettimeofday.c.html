src/port/win32gettimeofday.c

概要

Windows環境において、POSIX標準の gettimeofday() 関数をエミュレートするための互換レイヤーです。PostgreSQLがクロスプラットフォームで時刻を取得できるように提供されています。

モジュール内での役割

src/port ディレクトリの一部として、Windows固有のAPIをPOSIX準拠のインターフェースに変換する役割を担います。他のプラットフォームでは標準ライブラリが提供する機能を、Windows上で補完します。

責務

  • Windowsの FILETIME 形式（1601年1月1日からの100ナノ秒単位）を、POSIXの timeval 形式（1970年1月1日からの秒およびマイクロ秒）へ変換する。
  • GetSystemTimePreciseAsFileTime を使用して、可能な限り高精度なシステム時刻を取得する。
  • tzp（タイムゾーン情報）引数が渡された場合にアサーションで警告し、非互換な利用を防止する。

重要なシンボル

  • gettimeofday: 本ファイルのメイン関数。struct timeval に現在時刻を書き込む。
  • epoch: 1601年と1970年の差分を保持する定数。FILETIME をUNIXエポックに変換するために使用。
  • FILETIME_UNITS_PER_SEC: FILETIME の単位を秒に換算するための係数。

制御フローとデータフロー

1. **引数チェック**: tzp が NULL であることを Assert で確認します。 2. **時刻取得**: GetSystemTimePreciseAsFileTime を呼び出し、現在のシステム時刻を FILETIME 構造体に格納します。 3. **単位変換**:

  • FILETIME の値を ULARGE_INTEGER にキャストし、epoch を減算して1970年1月1日からの経過時間を算出します。
  • 算出した値を FILETIME_UNITS_PER_SEC で除算して tv_sec（秒）を求めます。
  • 余りを FILETIME_UNITS_PER_USEC で除算して tv_usec（マイクロ秒）を求めます。

    • 4. **戻り値**: 常に 0 を返します。

直接依存

  • c.h: PostgreSQLの基本定義およびアサーションマクロ。
  • sysinfoapi.h: Windows API (GetSystemTimePreciseAsFileTime) の定義。
  • sys/time.h: struct timeval の定義。

関連コード

  • src/port/win32getrusage.c: Windows環境におけるリソース使用量取得の互換実装。
  • src/include/port.h: 各プラットフォームの互換関数の宣言が含まれるヘッダー。

読者向けメモ

  • 本関数は高精度な「経過時間」の計測用ではありません。経過時間の計測には elapsed_time() 等の専用関数を使用してください。
  • tzp 引数はPOSIXでも非推奨であり、PostgreSQL内では使用されないことが前提となっています。
  • GetSystemTimePreciseAsFileTime は比較的新しいWindows APIであり、古い環境では動作しない可能性がある点に注意してください。
