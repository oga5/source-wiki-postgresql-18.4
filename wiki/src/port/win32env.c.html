src/port/win32env.c

概要

Windows 環境における環境変数の操作（putenv, setenv, unsetenv）を、POSIX 互換の挙動で提供するためのラッパーモジュールです。

モジュール内での役割

src/port の一部として、Windows 特有の環境変数管理の不整合を解消します。特に、複数の C ランタイムライブラリ（CRT）が混在する環境において、すべてのライブラリ間で環境変数の状態を同期させる役割を担います。

責務

  • **プロセス環境の更新**: SetEnvironmentVariable を使用し、OS レベルで環境変数を設定します。
  • **CRT 同期**: ロードされている既知のすべての CRT モジュール（msvcrt, ucrtbase 等）を列挙し、それぞれの _putenv を呼び出すことで、リンク先の異なるライブラリ間での不整合を防ぎます。
  • **POSIX 互換インターフェース**: setenv および unsetenv のセマンティクスをエミュレートし、呼び出し元に一貫した API を提供します。

重要なシンボル

  • pgwin32_putenv: 本ファイルの核となる関数。OS 環境と全 CRT の環境変数を更新します。
  • PUTENVPROC: CRT 内の _putenv 関数を動的に呼び出すための関数ポインタ型。
  • pgwin32_setenv / pgwin32_unsetenv: POSIX 準拠のラッパー関数。内部で pgwin32_putenv を呼び出します。

制御フローとデータフロー

1. **環境変数の更新**: pgwin32_putenv が呼び出されると、まず SetEnvironmentVariable でプロセス全体の環境を更新します。 2. **CRT の探索**: modulenames 配列に定義された既知の CRT モジュールを GetModuleHandleEx で検索します。 3. **動的呼び出し**: 見つかったモジュールに対して GetProcAddress で _putenv を取得し、値を同期させます。 4. **自己更新**: 最後に、現在のプロセスがリンクしている CRT の _putenv を呼び出し、処理を完了します。

直接依存

  • c.h: PostgreSQL の基本定義および標準ヘッダーへのアクセスを提供します。

関連コード

  • src/port/win32common.c: Windows 固有の共通ユーティリティ。
  • src/port/win32error.c: Windows エラーコードの変換など、プラットフォーム固有の処理。

読者向けメモ

  • **CRT の混在**: Windows では、異なるライブラリが異なる CRT にリンクしている場合、それぞれの CRT が独自の環境変数キャッシュを持つため、単純な putenv では不十分です。本ファイルはその問題を解決するために、ロード済みの全 CRT を探索する手法をとっています。
  • **メモリ管理**: pgwin32_putenv は引数の文字列を永続化しません。呼び出し元は必要に応じてメモリを管理する必要があります。
  • **注意点**: modulenames に含まれない新しい CRT が将来導入された場合、このリストを更新する必要があります。
