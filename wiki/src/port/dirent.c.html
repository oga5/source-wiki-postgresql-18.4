src/port/dirent.c

概要

Windows環境（MSVC）において、POSIX準拠のディレクトリ操作関数（opendir, readdir, closedir）を提供するためのラッパー実装です。

モジュール内での役割

src/port 内で、プラットフォーム固有の差異を吸収する移植層の一部です。WindowsのネイティブAPIである FindFirstFile/FindNextFile を、PostgreSQLが期待するPOSIXインターフェースへ変換します。

責務

  • ディレクトリのオープン、走査、クローズの抽象化。
  • Windowsの WIN32_FIND_DATA 構造体から、PostgreSQLが定義する struct dirent へのデータ変換。
  • Windows特有のファイル属性（リパースポイント等）をPOSIX風のファイルタイプ（DT_LNK, DT_DIR 等）へマッピング。

重要なシンボル

  • DIR: ディレクトリ走査の状態を保持する構造体。検索ハンドルと現在のエントリ情報を管理します。
  • opendir(): 指定パスがディレクトリか検証し、走査用のハンドルを初期化します。
  • readdir(): 次のファイルエントリを取得し、struct dirent に情報を格納して返します。
  • closedir(): 検索ハンドルを解放し、メモリをクリーンアップします。

追加の索引済みシンボル

  • closedir
  • opendir
  • readdir

制御フローとデータフロー

1. **opendir**: パスに * を付与し、FindFirstFile 用の検索文字列を作成します。 2. **readdir**:

  • 初回呼び出し時は FindFirstFile を実行。
  • 2回目以降は FindNextFile を呼び出し、結果を d->ret にコピーします。
  • dwFileAttributes を評価し、d_type を決定します。

    • 3. **closedir**: FindClose でハンドルを閉じ、DIR 構造体自体を解放します。

直接依存

  • postgres.h / postgres_fe.h: PostgreSQLの基本定義。
  • <dirent.h>: POSIXインターフェース定義。
  • Windows API (windows.h 相当の機能): FindFirstFile, FindNextFile, FindClose, GetFileAttributes 等。
  • dirent.h

関連コード

  • src/include/port/win32_msvc/dirent.h: 本ファイルで使用される struct dirent および DIR の定義元。
  • src/port/dirmod.c: ディレクトリ操作に関連する他の移植用ユーティリティが含まれています。

読者向けメモ

  • 本実装はWindows専用です。FRONTEND マクロの有無により、バックエンド用とフロントエンド用の両方で利用可能です。
  • d_ino や d_reclen はWindows APIで直接取得できないため、0に固定されています。
  • リパースポイント（シンボリックリンクやジャンクション）の判定ロジックが含まれており、Windows特有のファイルシステム構造を考慮した設計になっています。
