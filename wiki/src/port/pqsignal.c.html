src/port/pqsignal.c

概要

PostgreSQL におけるシグナルハンドリングの抽象化レイヤーです。OS 固有のシグナル動作の差異を吸収し、信頼性の高いシグナル処理を提供することを目的としています。

モジュール内での役割

src/port ディレクトリに位置し、バックエンドおよびフロントエンドの両方から利用されるポータブルなシグナル設定インターフェースを提供します。標準の signal(2) の代わりに利用することで、プラットフォーム間での一貫した動作を保証します。

責務

  • **信頼性の高いシグナル設定**: sigaction を使用して、シグナルハンドラがリセットされない、あるいはシグナルがブロックされないといった「信頼できない」シグナル動作を回避します。
  • **ラッパーハンドラの提供**: シグナル発生時に wrapper_handler を介することで、errno の保存・復元や、不正なプロセス（fork 後など）でのシグナル処理を防止する安全策を提供します。
  • **プラットフォーム差異の吸収**: Windows 環境と Unix 環境におけるシグナル API の違いを隠蔽します。

重要なシンボル

  • pqsignal(): シグナルハンドラを登録するためのメイン関数。SA_RESTART フラグをデフォルトで付与します。
  • wrapper_handler(): 実際のハンドラを呼び出す前に実行される中間関数。プロセス ID の検証や errno の保護を行います。
  • pqsignal_handlers[]: 登録されたシグナルハンドラを保持する内部配列。

追加の索引済みシンボル

制御フローとデータフロー

1. **登録**: pqsignal(signo, func) が呼ばれると、指定された func は pqsignal_handlers 配列に格納され、実際のハンドラとして wrapper_handler が sigaction に登録されます。 2. **シグナル発生**: OS からシグナルが通知されると、wrapper_handler が起動します。 3. **検証**: wrapper_handler 内で MyProcPid を確認し、現在のプロセスがシグナルを受け取るべき正当なプロセスであるかを判定します。 4. **実行**: 検証が成功すれば、pqsignal_handlers から元の関数を取り出して実行します。終了時に errno を復元します。

直接依存

  • c.h: PostgreSQL の基本定義。
  • signal.h: 標準シグナル定義。
  • libpq/pqsignal.h: シグナル関連の型定義やマクロ。
  • miscadmin.h: MyProcPid などのプロセス管理情報。
  • unistd.h

関連コード

  • src/include/libpq/pqsignal.h: Windows 環境での sigaction エミュレーションなど、本ファイルのヘッダー定義。
  • src/backend/port/win32/signal.c: Windows 環境におけるバックエンドのシグナル処理実装。

読者向けメモ

  • 本ファイルは、歴史的な経緯から「信頼できない」シグナル実装を回避するために設計されています。
  • wrapper_handler 内の MyProcPid チェックは、fork() 後の子プロセスが誤って共有メモリを操作するのを防ぐ重要なガードレールです。
  • Windows 環境では、フロントエンドとバックエンドでシグナル処理の挙動が大きく異なるため、コードを読む際は #ifdef FRONTEND の条件分岐に注意してください。
