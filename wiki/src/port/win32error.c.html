src/port/win32error.c

概要

Windows API が返すエラーコード（DWORD）を、POSIX 準拠の標準エラーコード（errno）に変換するためのユーティリティファイルです。

モジュール内での役割

src/port ディレクトリの一部として、PostgreSQL が Windows 上で動作する際に、OS 固有のエラーをプラットフォーム非依存の形式へ正規化する役割を担います。

責務

  • Windows 特有のエラーコードと POSIX errno のマッピングテーブルの保持。
  • _dosmaperr 関数を通じた、グローバルな errno 変数の更新。
  • 未定義のエラーコードに対するフォールバック処理（EINVAL への設定）。

重要なシンボル

  • doserrors: Windows エラーコードと POSIX エラーコードを対応付けた静的な構造体配列。
  • _dosmaperr(unsigned long e): 引数に Windows エラーコードを受け取り、対応する errno を設定する公開関数。

追加の索引済みシンボル

  • DWORD

制御フローとデータフロー

1. _dosmaperr が呼び出されると、まずエラーコード 0（成功）かどうかを確認します。 2. doserrors 配列を線形探索し、一致する winerr を探します。 3. 一致した場合、対応する doserr を errno に代入します。 4. 一致しない場合、デバッグログを出力し、デフォルト値として EINVAL を errno に設定します。 5. FRONTEND 定義の有無により、エラー報告の出力先（ereport または stderr）が切り替わります。

直接依存

  • postgres.h / postgres_fe.h: PostgreSQL の基本定義およびフロントエンド/バックエンドの切り替え用ヘッダー。

関連コード

  • src/port/win32common.c: Windows 固有の共通処理を扱う関連ファイル。
  • src/port/win32ntdll.h: Windows NT API を直接扱う際の定義群。

読者向けメモ

  • 本ファイルは Windows 環境でのみ有効です。
  • _dosmaperr は、Windows API を呼び出した直後に GetLastError() の結果を変換するために使用されます。
  • マッピングテーブルは網羅的ではないため、未知のエラーコードが渡された場合は LOG レベルで記録される仕様になっています。
