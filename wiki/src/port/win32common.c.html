src/port/win32common.c

概要

Windows環境におけるPostgreSQLの移植レイヤー（win32*.c）で共通して利用される、低レベルなユーティリティ関数を提供するソースファイルです。

モジュール内での役割

src/port ディレクトリ内の他のWindows固有実装（win32stat.c, win32pread.c 等）から呼び出される、共通の補助ルーチンをホストしています。コードの重複を排除し、Windows APIの呼び出しに伴う定型的なエラーハンドリングを統一する役割を担います。

責務

  • Windows API GetFileType のラッパー提供。
  • API呼び出し時のエラーコード（GetLastError）を、PostgreSQLが扱う errno に変換する処理の集約。

重要なシンボル

  • pgwin32_get_file_type: 指定されたファイルハンドルからファイルタイプを取得する関数。INVALID_HANDLE_VALUE や無効なストリームに対するガード処理を含みます。

制御フローとデータフロー

1. **入力**: Windowsの HANDLE を受け取ります。 2. **バリデーション**: ハンドルが INVALID_HANDLE_VALUE または標準ストリーム未割り当て（-2）であるかを判定し、該当する場合は errno を EINVAL に設定して終了します。 3. **API実行**: GetFileType を呼び出し、結果を保持します。 4. **エラー判定**: GetFileType が FILE_TYPE_UNKNOWN を返した場合、GetLastError を確認します。エラーが NO_ERROR でなければ _dosmaperr を呼び出し、WindowsのエラーコードをPOSIX準拠の errno にマッピングします。 5. **出力**: 成功時はファイルタイプ（FILE_TYPE_DISK, FILE_TYPE_CHAR 等）を返し、失敗時は FILE_TYPE_UNKNOWN を返します。

直接依存

  • postgres_fe.h または postgres.h: ビルドターゲット（フロントエンド/バックエンド）に応じて必要な定義を読み込みます。

関連コード

  • src/port/win32*.c: 本ファイルで定義された関数を利用する、他のWindows移植用ソース群。
  • src/include/port/win32.h: Windows固有の型定義やマクロが定義されているヘッダー。

読者向けメモ

  • 本ファイルは非常に小規模ですが、Windows特有の「ハンドルが有効か否か」や「エラーコードの変換」という、移植レイヤーで頻出する落とし穴を回避するために重要です。
  • _dosmaperr は、Windowsのシステムエラーコードを標準的な errno 値に変換するMSVCランタイムの関数です。Windows移植コードを読む際は、この変換がどこで行われているかを意識すると理解が早まります。
