src/port/explicit_bzero.c

概要

メモリ領域を安全にゼロクリアするためのポータブルな関数 explicit_bzero を提供します。コンパイラの最適化によって「使用されない書き込み」とみなされ、クリア処理が削除されるのを防ぐことを目的としています。

モジュール内での役割

src/port ディレクトリの一部として、プラットフォーム間で一貫したセキュリティ機能を提供します。パスワードや暗号鍵などの機密情報をメモリから消去する際に使用されます。

責務

  • プラットフォーム固有の安全なゼロクリア関数（memset_s や SecureZeroMemory）への抽象化。
  • 上記の関数が利用できない環境において、コンパイラの最適化を回避するフォールバック実装の提供。

重要なシンボル

  • explicit_bzero(void *buf, size_t len): メモリをゼロクリアする公開関数。
  • bzero2: フォールバック環境で使用される内部的なクリア関数。
  • bzero_p: コンパイラの最適化を回避するために使用される volatile な関数ポインタ。

追加の索引済みシンボル

  • explicit_bzero

制御フローとデータフロー

1. **プラットフォーム判定**: コンパイル時に HAVE_DECL_MEMSET_S や WIN32 の定義を確認します。 2. **実装の選択**:

  • memset_s が利用可能な場合: 直接呼び出します。
  • Windows の場合: SecureZeroMemory を呼び出します。
  • その他: volatile な関数ポインタ bzero_p を介して bzero2 を呼び出します。

    • 3. **最適化の回避**: フォールバック環境では、関数ポインタを volatile にすることで、コンパイラが「この関数呼び出しは副作用を持つ可能性がある」と判断し、呼び出しを削除しないように強制します。

直接依存

  • c.h: PostgreSQL の標準的な環境定義およびプラットフォーム依存のヘッダーへのアクセスを提供します。

関連コード

  • timingsafe_bcmp.c: メモリ比較において同様にタイミング攻撃や最適化を考慮した実装が含まれており、セキュリティ関連のユーティリティとして対をなします。

読者向けメモ

  • このファイルは、標準的な memset がコンパイラの最適化によって削除されるリスク（Dead Store Elimination）を回避するためのものです。
  • フォールバック実装の volatile ポインタによる手法は、OpenSSH の実装を参考にしています。
  • 可能な限り、プラットフォームが提供する標準的なセキュリティ関数（memset_s 等）が優先される設計になっています。
