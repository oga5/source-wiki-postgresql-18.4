src/port/pg_localeconv_r.c

概要

pg_localeconv_r.c は、標準 C ライブラリの localeconv() が持つスレッドセーフではないという欠点を補うための、PostgreSQL 独自のラッパー関数を提供します。異なるロケールカテゴリ（LC_NUMERIC と LC_MONETARY）を個別に指定し、スレッド間で競合することなく struct lconv を取得することを目的としています。

モジュール内での役割

src/port 内において、プラットフォーム依存のロケール処理を抽象化する役割を担います。OS ごとに異なるロケール管理手法（Windows のスレッドローカルモード、POSIX の locale_t、あるいはグローバルロック）を吸収し、一貫したインターフェースを提供します。

責務

  • **スレッドセーフなロケール情報の取得**: 呼び出し元が指定したロケールに基づき、安全に struct lconv を構築する。
  • **メモリ管理**: 取得した文字列メンバのコピーを管理し、pg_localeconv_free() を通じて呼び出し元が安全にメモリを解放できるようにする。
  • **プラットフォーム差異の吸収**: Windows、Glibc、macOS/*BSD 間で異なるロケール API の差異を隠蔽する。

重要なシンボル

  • pg_localeconv_r(): メインの公開関数。指定されたロケール名から struct lconv を生成する。
  • pg_localeconv_free(): pg_localeconv_r() が確保した動的メモリを解放する。
  • struct lconv_member_info: lconv 構造体の各メンバをどのようにコピー・管理するかを定義するメタデータテーブル。
  • TRANSLATE_FROM_LANGINFO: Glibc 環境で利用可能な、より効率的なロケール情報取得パスを有効化するマクロ。

制御フローとデータフロー

1. **初期化**: table[] に定義されたメンバ情報に基づき、対象の lconv 構造体を初期化します。 2. **ロケール適用**:

  • **Windows**: _configthreadlocale を使用してスレッドローカルモードに切り替え、一時的にロケールを変更して localeconv() を呼び出します。
  • **POSIX**: newlocale() で作成した locale_t を使用し、localeconv_l() または uselocale() を経由して情報を抽出します。

    • 3. **データコピー**: pg_localeconv_copy_members() が table[] を走査し、各メンバを strdup() でコピーします。 4. **終了処理**: 一時的に変更したロケールを復元し、使用したリソース（locale_t やロック）を解放します。

読者向けメモ

  • **メモリ所有権**: pg_localeconv_r() が返す struct lconv 内の文字列メンバは動的に確保されます。呼び出し側は必ず pg_localeconv_free() を呼び出す必要があります。
  • **POSIX の制約**: POSIX 標準では LC_CTYPE と他のカテゴリのエンコーディングが不一致な場合の挙動が未定義です。本ファイルでは、安全のために LC_ALL を指定してロケールを適用することで、この不整合を回避しています。
  • **ロックの存在**: localeconv_l() が利用できない古い環境では、pthread_mutex_t によるグローバルロックが使用されます。これはパフォーマンス上のボトルネックになる可能性があるため、可能な限り locale_t をサポートする環境での利用が推奨されます。
