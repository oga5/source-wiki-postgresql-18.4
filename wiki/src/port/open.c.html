src/port/open.c

概要

Windows 環境における open() および fopen() のラッパー実装です。POSIX 準拠のファイル操作を Windows のネイティブ API (CreateFile) に変換し、PostgreSQL が必要とする高度なファイル共有やエラーハンドリングを実現します。

モジュール内での役割

src/port 内で、プラットフォーム固有の差異を吸収する「移植層」の一部です。特に Windows 特有のファイルロック問題や、POSIX とは異なるファイル属性の扱いを抽象化し、PostgreSQL のコアおよびフロントエンドツールが共通のインターフェースでファイル操作を行えるようにします。

責務

  • **POSIX 互換のファイルオープン**: open() および fopen() の呼び出しを Windows の CreateFile にマッピングします。
  • **共有違反の緩和**: アンチウイルスソフト等による一時的なファイルロックに対し、最大 30 秒間の再試行ロジックを提供します。
  • **ファイル属性の変換**: O_CREAT や O_TRUNC などのフラグを Windows の CreateFile 用フラグへ変換します。
  • **ハンドル管理**: O_CLOEXEC のサポートや、ファイルハンドルからファイル記述子（fd）への変換を行います。

重要なシンボル

  • pgwin32_open_handle: 内部関数。CreateFile を呼び出し、再試行ロジックと NT ステータスの変換を担います。
  • pgwin32_open: open() の代替。ファイルハンドルを取得し、_open_osfhandle を通じて C ランタイムのファイル記述子を返します。
  • pgwin32_fopen: fopen() の代替。モード文字列を解析して pgwin32_open を呼び出し、_fdopen で FILE* を生成します。
  • openFlagsToCreateFileFlags: POSIX のオープンフラグを Windows の dwCreationDisposition に変換するヘルパー関数。

制御フローとデータフロー

1. **フラグ変換**: openFlagsToCreateFileFlags が POSIX フラグを Windows の作成モードに変換します。 2. **ハンドル取得**: pgwin32_open_handle が CreateFile を実行します。ここで共有モード (FILE_SHARE_READ|WRITE|DELETE) を指定し、同時実行性を確保します。 3. **再試行ループ**: ERROR_SHARING_VIOLATION 等が発生した場合、pg_usleep を用いて最大 30 秒間リトライします。 4. **fd 変換**: 成功したハンドルは _open_osfhandle で POSIX 互換のファイル記述子に変換されます。 5. **モード適用**: _setmode を使用して、バイナリ/テキストモードを適用します。

直接依存

  • windows.h (間接的): CreateFile, HANDLE 等の Windows API。
  • port/win32ntdll.h: NT ステータスコードの取得用。
  • fcntl.h, sys/stat.h: POSIX ファイルフラグ定義。
  • assert.h
  • postgres.h
  • postgres_fe.h

関連コード

  • src/port/win32error.c: _dosmaperr を通じて Windows エラーコードを errno に変換する際に利用されます。
  • src/include/port.h: open や fopen が本ファイルの実装に置換されるためのマクロ定義が含まれています。

読者向けメモ

  • 本ファイルは WIN32 マクロが定義されている場合のみコンパイルされます。
  • O_CLOEXEC の実装において、SetHandleInformation ではなく CreateFile の SECURITY_ATTRIBUTES を利用している点は、マルチスレッド環境での競合を避けるための重要な設計です。
  • STATUS_DELETE_PENDING のハンドリングは、Windows 特有の「削除中だがまだファイルが存在する」状態を POSIX の期待通りに扱うための工夫です。
